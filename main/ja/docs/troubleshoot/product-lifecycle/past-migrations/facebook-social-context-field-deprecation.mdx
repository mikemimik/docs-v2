title: Facebookソーシャルコンテキストフィールドの非推奨化
---

**2020年7月30日**以降、`context` フィールドを要求するFacebook接続は失敗するため、Auth0はすべての接続でこのフィールドを要求しないようになります。

4月30日に、Facebookは新しいアプリケーションでの **Social Context** フィールドの使用を[非推奨](https://developers.facebook.com/docs/graph-api/changelog/4-30-2019-endpoint-deprecations)としました。Auth0は、2019年4月30日より前に作成されたFacebook接続については、引き続きこのフィールドをデフォルトで要求していました。7月30日までにこのフィールドが要求されないようにするには、User Data 接続セクションで **Social context** フィールドのチェックを外してください。

<Frame>![Facebook Context Field Deprecation](/docs/images/cdy7uua7fh8z/4oi3foRuwYYOveDlq2vJM/a57cd954ceeaf6d7722796be7c743b76/facebook-context.png)</Frame>

**Social context** のチェックを外すと、プロファイルデータに `context` フィールドは含まれなくなります。このフィールドには次の内容が含まれます。

```text lines
"context": {
      "mutual_likes": {"data": [],"summary": {"total_count": 0}},
      "id": "dXNlcl9...UZD"
    }
```

<div id="actions">
  ## Actions
</div>

Auth0 が返す Facebook のプロファイル内の `context` フィールドをアプリケーションで使用していない場合、アプリケーションはそのまま引き続き動作します。一方、そのフィールドを使用している場合は、アプリケーションのコードがそれに依存しないよう調整する必要があります。

7 月 30 日の影響を避けるため、Facebook 接続のプロパティで **Social context** フィールドのチェックを外すことをお勧めします。
