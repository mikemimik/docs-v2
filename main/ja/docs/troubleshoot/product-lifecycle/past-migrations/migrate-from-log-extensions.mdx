title: Log Extensions からの移行
---

以下の Auth0 Log Extensions は現在非推奨であり、以下に示す提供終了 (EOL) 日までサポートされます。この非推奨の Log Extensions 一覧から新しい extension を作成することは、現在できません。同等の機能は、ログイベントストリームまたは [Auth0 Marketplace](https://marketplace.auth0.com/) の連携で設定できます。

以下は、Private Cloud で **2023年1月6日** に EOL を迎えます。

* Auth0 Authentication API Webhooks
* Auth0 <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> Webhooks
* Logs to Cloudwatch
* Logs to Logentries
* Logs to Loggly
* Logs to Logstash
* Logs to Papertrail
* Logs to Splunk
* Logs to Sumo Logic

以下は、Public Cloud で **2023年5月2日** に EOL を迎えます。

* Auth0 Authentication API Webhooks
* Auth0 Management API Webhooks
* Logs to Cloudwatch
* Logs to Logentries
* Logs to Loggly
* Logs to Logstash
* Logs to Papertrail
* Logs to Splunk
* Logs to Sumo Logic
* Logs to Segment
* Logs to Mixpanel
* Logs to AppInsights
* Logs to Azure Blob Storage

記載された EOL 日以降、Auth0 は上記一覧のインストール済み log extensions をサポートしなくなります。このページでは、特定の log extensions から移行する手順を案内します。

<div id="auth0-authentication-api-webhooks">
  ## Auth0 Authentication API Webhooks
</div>

送信先でログオブジェクトの配列を処理できることを確認してください。

1. [Auth0 Dashboard にログインします](https://manage.auth0.com/)。
2. **Extensions** に移動し、**Installed Extensions** タブで **Auth0 Authentication API webhook** を選択します。
3. **Webhook URL** と **Authorization Header** (ある場合) をコピーします。
4. 次に、**Monitoring &gt; Streams &gt; New Event Stream** に移動します。
5. [**Custom Webhook**](/docs/ja/customize/log-streams/custom-log-streams) を選択し、名前を入力して **Create** をクリックします。
6. webhook を設定します。

   * **Payload URL** に、拡張機能 からコピーした webhook URL を入力します。
   * ある場合は、**認可トークン** に 拡張機能 からコピーした Authorization Header を設定します。
   * **Content Format** を JSON Array に設定します。
7. **Save** をクリックします。
8. **Extensions &gt; Installed Extensions** に戻り、**Auth0 Authentication API webhooks** 拡張機能 を無効にします。

<div id="cloudwatch">
  ## CloudWatch
</div>

Auth0 Marketplace で提供されている [Amazon EventBridge 連携](https://marketplace.auth0.com/integrations/amazon-log-streaming) を設定します。イベントバスでイベントを利用できるようになったら、[ルールを作成](https://docs.aws.amazon.com/eventbridge/latest/userguide/create-event-bus.html)して、それらのイベントを AWS CloudWatch にマッピングします。**Auth0 Logs to CloudWatch** 拡張機能は必ず無効にしてください。

1. [Auth0 Dashboard にログインします](https://manage.auth0.com/)。
2. **Extensions &gt; Installed Extensions** に移動します。
3. **Auth0 Logs to CloudWatch** 拡張機能を無効にします。

<div id="loggly">
  ## Loggly
</div>

1. [Loggly アカウントにログインします](https://www.loggly.com/login/)。
2. **Logs** メニューを開き、**Source Setup** を選択します。
3. **Customer Tokens** タブに移動します。
4. トークンをコピーします。
5. [Auth0 Dashboard にログインします](https://manage.auth0.com/)。
6. **Monitoring &gt; Streams &gt; New Event Stream** に移動します。
7. [**Custom Webhook**](/docs/ja/customize/log-streams/custom-log-streams) を選択し、名前を入力して **Create** をクリックします。
8. Webhook を設定します。

   * **Payload URL** で、`LOGGLY_TOKEN` を Loggly アカウントからコピーしたトークンに置き換えます: `https://logs-01.loggly.com/bulk/{LOGGLY_TOKEN}/tag/auth0-{TENANT_NAME}/`。
   * **認可トークン** は空欄のままにします。
   * **Content Type** を `application/json` に設定します。
   * **Content Format** を JSON Lines に設定します。
9. **Save** をクリックします。
10. **Extensions &gt; Installed Extensions** に移動し、**Auth0 Logs to Loggly** 拡張機能を無効にします。

<div id="logstash">
  ## Logstash
</div>

1. [Auth0 Dashboard にログインします](https://manage.auth0.com/)。
2. **Monitoring &gt; Streams &gt; New Event Stream** に移動します。
3. [**Custom Webhook**](/docs/ja/customize/log-streams/custom-log-streams) を選択し、名前を入力して **Create** をクリックします。
4. Webhook を設定します。

   * **Payload URL** に、Logstash サーバーの URL を入力します。
   * **認可トークン** に `Basic <CREDENTIALS>` を設定し、`CREDENTIALS` はコロン (`:`) で区切った Logstash のユーザー名とパスワード (`<USER:PASSWORD>`) を Base64 エンコードした値に置き換えます。Logstash の HTTP 資格情報は、Logstash の設定ファイルで確認できます。
5. **Save** をクリックします。
6. **Extensions &gt; Installed Extensions** に移動し、**Auth0 Logs to Logstash** 拡張機能 を無効にします。

<div id="papertrail">
  ## Papertrail
</div>

1. Papertrail にログインします。
2. [Log Destinations](https://papertrailapp.com/account/destinations) に移動し、[Create Log Destination](https://papertrailapp.com/destinations/new) をクリックします。
3. 説明を入力し、destination の設定を行います。
4. **Accept Connections via ...** ペインで、**Token** を選択します。
5. **Create** をクリックします。
6. [Auth0 Dashboard にログイン](https://manage.auth0.com/)します。
7. **Monitoring &gt; Streams &gt; New Event Stream** に移動します。
8. [**Custom Webhook**](/docs/ja/customize/log-streams/custom-log-streams) を選択し、名前を入力して **Create** をクリックします。
9. Webhook を設定します。

   * **Payload URL** を `https://logs.collector.solarwinds.com/v1/logs` に設定します。
   * **認可トークン** を `Basic <CREDENTIALS>` に設定し、`CREDENTIALS` は Papertrail の destination 設定で取得した Papertrail トークンを Base64 エンコードした値に置き換えます。
   * **Content Format** を JSON Lines に設定します。
10. **Save** をクリックします。
11. **Extensions &gt; Installed Extensions** に移動し、**Auth0 Logs to Papertrail** 拡張機能 を無効にします。

<div id="segment">
  ## セグメント
</div>

1. Auth0 Dashboard にログインします。
2. **Extensions &gt; Installed Extensions** で、Segment 拡張機能を探します。
3. 新しい Segment 向けログストリームの設定で必要になるため、**Write Key** を控えておきます。
4. Segment 拡張機能を無効にします。
5. Segment で最後に受信したログの UTC タイムスタンプを控えておきます。このタイムスタンプがわかっていれば、Segment 内でログが重複しないようにしながら新しいストリームを作成できます。
6. **Monitoring &gt; Log Streams** に移動し、Segment 向けの新しいログストリームを作成します。該当するフィールドに **Write Key** を入力し、**Start From** セクションに控えておいたタイムスタンプを入力します。
7. ストリームを作成します

<div id="mixpanel">
  ## Mixpanel
</div>

1. Mixpanel アカウントにログインし、**Project Settings** を開きます。**Project ID** と **Data Residency** (データレジデンシー / リージョン) を控えておきます。
2. Mixpanel では、イベントのインポートにプロジェクトの `token` を使用することは現在推奨されておらず、代わりにサービスアカウントの仕組みを使用することが推奨されています。**Project Settings** ページで **Service Accounts** を選択し、管理者またはオーナー権限を持つ新しいサービスアカウントを作成します。資格情報を控えておきます。
3. Auth0 Dashboard にログインします。
4. **Monitoring &gt; Log Streams** で、新しい Mixpanel ストリームを作成します。プロジェクト ID、リージョン、サービスアカウントの資格情報を入力して、ストリームを作成します。
5. 古い 拡張機能 の無効化も忘れないでください。**Extensions &gt; Installed Extensions** に移動し、**Auth0 Logs to Mixpanel 拡張機能** を無効にします。

<div id="appinsights">
  ## AppInsights
</div>

1. [Azure EventGrid Log Stream](https://marketplace.auth0.com/integrations/azure-log-streaming) を設定するか、既存の EventGrid ストリームを利用します。
2. ログイベントの宛先を設定するには、[Azure が公開している手順](https://urldefense.com/v3/__https://learn.microsoft.com/en-us/azure/event-grid/auth0-log-stream-app-insights__;!!PwKahg!9T9C3qhBRgQQHChQkQqnmHDDnX99yWJ4W8F4rLVs6qrwwbnFCY3JcCfF486d1DLqRFEghVnJZ9gJnronDv0$) に従ってください。
3. Log Streaming ではログが異なる形に変換される場合があります。詳細については、非推奨の [Log Extenstion code](https://github.com/auth0-extensions/auth0-logs-to-provider/blob/master/server/lib/senders/appinsights.js) を参照してください

ログに対して行われる主な変換は次のとおりです。

* appInsight クライアントは上書きされます。

  * カスタム time がログ時刻と同じになるように設定されます
  * OS と OS version がログ情報と同じになるように設定されます
  * Device type tag には、ログの発生元がモバイルかどうかを示す情報が格納されます
  * IP address tag がログ情報と同じになるように設定されます
  * user id、account id、user agent、auth user id の各 tag がログと同じになるように設定されます
* record は整理され、空の場合は次のプロパティが削除されます。

  * ip
  * user&#95;id
  * user&#95;name
  * connection
  * client&#95;name
  * description
* record の isMobile プロパティは、yes または no の文字列を含むように変更されます
* record の details プロパティは、文字列になるように変更されます
* record の details プロパティは、8185 文字に切り詰められます

<div id="azure-blob-storage">
  ## Azure Blob Storage
</div>

1. [Azure EventGrid Log Stream](https://marketplace.auth0.com/integrations/azure-log-streaming)を設定するか、既存のEventGridストリームを利用します。
2. ログイベントの送信先を設定するには、[Azure が公開している手順](https://urldefense.com/v3/__https://learn.microsoft.com/en-us/azure/event-grid/auth0-log-stream-blob-storage__;!!PwKahg!9T9C3qhBRgQQHChQkQqnmHDDnX99yWJ4W8F4rLVs6qrwwbnFCY3JcCfF486d1DLqRFEghVnJZ9gJ5k9lKMM$)に従ってください。
3. Log Streaming では、ログが通常とは異なる形に変換される場合があります。詳細については、非推奨の[Log Extenstion code](https://github.com/auth0-extensions/auth0-logs-to-provider/blob/master/server/lib/senders/blobstorage.js)を参照してください

ログに対して行われる主な変換は次のとおりです。

* レコードの`type_code`と`type`は変更されます。`Type`は`type_code`を人が理解しやすい形で表した説明です
* レコードに`user_agent`情報が含まれている場合、`os`、`os_version`、`device`、`device_version`は変更されます
