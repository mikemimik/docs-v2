---
description: アプリケーションのドメインを移行できるよう、非推奨通知についてテナントログを確認する方法を説明します
title: テナントのホスト名の検証の移行
---

Auth0 は、Public Cloud では 2022 年 6 月 9 日から、Private Cloud では 2022 年 9 月 9 日から、Authentication API にバリデーションのレイヤーを追加します。お使いのアプリケーションから Authentication API への呼び出しがこの変更の影響を受ける可能性があると Auth0 が検出した場合に備え、テナントログに非推奨通知を表示し、この変更に備えるための移行フラグを提供しています。

<div id="affected-endpoints">
  ### 影響を受けるエンドポイント
</div>

Public Cloud では 2022 年 6 月 9 日以降、Private Cloud では 2022 年 9 月 9 日以降、以下の Authentication API エンドポイントへの呼び出しのうち、適切に検証されていないものはすべて拒否されます。Auth0 では、その日付より十分前に、アプリケーションのドメイン URL または API 呼び出しの identifier を同じテナントに移行するための対応を行うことを推奨しています。

影響を受けるエンドポイントは次のとおりです。

* `/oauth/token`
* `/co/authenticate`
* `/userinfo`
* `/login`
* `/oauth/revoke`
* `/mfa/challenge`
* `/p/<connection-type>/<ticket>` (エンタープライズ接続のプロビジョニング エンドポイント)

<div id="review-tenant-logs">
  ## テナントログを確認する
</div>

まず、アプリケーションを移行する必要があるかどうかを確認するため、テナントログで非推奨通知を確認します。

1. [Auth0 Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs) に移動します。

   1. ログで `type:depnote AND description:ignore*request*host*header*` を検索し、影響を受けていて移行が必要なアプリケーションに関する非推奨通知を探します。ログエントリには、影響を受けるアプリケーションの ID と、次のメッセージが含まれます。
      `Ignore request Host header: This feature is being deprecated. Please see https://auth0.com/docs/product-lifecycle/deprecations-and-migrations``/tenant-hostname-migration.`
   2. ログの **Details &gt; Raw** セクションを確認します。ここで、更新が必要なアプリケーションの `client_id`、またはプロビジョニングエンドポイントの場合は `connection_id` を特定できます。

      <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
        `tenant_from_host` フィールドも確認してください。このフィールドには、API 呼び出しで受信したドメインから識別された Auth0 テナントが含まれます。
      </Callout>
2. 該当するすべてのアプリケーションを修正します。

   1. テナントとドメインから判別されたテナントに不一致がある場合は、誤って設定されているその他のリクエストパラメーターとあわせて、送信する識別子またはドメイン URL を修正する必要があります。
   2. ドメインのテナントは、`client_id` または `connection_id` に関連付けられたテナントと一致している必要があります。

該当するすべてのテナントの移行が完了すると、この移行に関連する非推奨通知はテナントログに表示されなくなります。

<div id="verify-migration">
  ## 移行の確認
</div>

アプリケーションの移行を完了し、未検証のホスト名を設定したら、2022年6月9日または2022年9月9日までの任意のタイミングで非推奨の動作を無効にし、変更内容を確認します。

1. [Auth0 Dashboard &gt; テナント設定 &gt; Advanced &gt; Migrations](https://manage.auth0.com/#/tenant/advanced) に移動します。
2. **Ignore request Host header** トグル**.** を無効にします。このトグルを無効にすると、テナントに対して検証が強制され、移行が完了します。

このトグルを無効にしたあと、ホスト名の検証が想定どおりに機能しない場合は、4xx エラーが返され、ドメインのテナントと `client_id` または `connection_id` に関連付けられたテナントが一致していないことが示されます。

すべてのアプリケーションの移行が本番環境で正常に実施され、確認されたら、このスイッチを恒久的に無効にして、非推奨機能が今後使用できないようにできます。Public Cloud では 2022年6月9日以降、Private Cloud では 2022年9月9日以降、Auth0 によってホスト名の検証が強制され、関連するスイッチはテナント設定から削除されます。
