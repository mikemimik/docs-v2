title: Facebook Graph API の変更
---

2018年8月1日以降、Facebook では、リクエスト可能な Facebook Graph API の権限とフィールドが変更されました。Auth0 ではこれらの変更を反映して Facebook 接続 を更新し、わかりやすくするために接続インターフェースも変更しました。詳細と重要な日程については、[Facebook Login Changelog: Recent Changes to Facebook Login](https://developers.facebook.com/docs/facebook-login/changelog#2018-07-02) を参照してください。

<div id="features-affected">
  ## 影響を受ける機能
</div>

この更新では、コードや設定の変更が不要な場合もありますが、既存の権限で許可されていれば、アプリケーションが追加のプロファイルデータを受け取る可能性があります。次の点にご注意ください。

* Facebook 接続が、削除された権限のいずれかを要求するように設定されている場合、あなたの<Tooltip tip="アクセストークン: API へのアクセスに使用される、オペークな文字列または JWT の形式の認可資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip>には、それらがスコープとして含まれません。
* Facebook アプリケーションが「development」としてマークされている場合、接続を試している間は一時的にエラーが表示されることがあります。
* 接続に新しい権限を追加すると、エンドユーザーは次回ログイン時に同意を求められます。特定の権限を持っていないユーザーへの対応方法については、Facebook のドキュメントを参照してください。

<div id="facebook-login-permissions">
  ### Facebookログインの権限
</div>

ユーザーがFacebookを使ってログインするとき、アプリケーションは [Facebook Login permissions](https://developers.facebook.com/docs/facebook-login/permissions) を要求します。ユーザーが初めてログインする場合や、権限が変更されている場合は、要求された新しい権限を示す同意ウィンドウがFacebookに表示されます。これらの権限が付与されると、アプリケーションはFacebookのアクセストークンを使用して、そのユーザーに代わって操作できるようになります。

Facebook 接続インターフェイスは、表示されるすべての権限について、通常名とマシン名の両方を表示するように更新されました。これにより、必要な権限を見つけやすくなり、それらの権限名を使っているコードとの対応付けもしやすくなります。

<div id="permissions-added">
  #### 追加された権限
</div>

Facebook 接続インターフェイスに、以下の権限が追加されました。

* business&#95;management
* groups&#95;access&#95;member&#95;info
* leads&#95;retrieval
* pages&#95;manage&#95;instant&#95;articles
* publish&#95;to&#95;groups
* publish&#95;to&#95;groups
* user&#95;age&#95;range
* user&#95;gender
* user&#95;link

<div id="permissions-removed">
  #### 削除された権限
</div>

以下の権限が Facebook 接続インターフェイスから削除されました。

* read&#95;custom&#95;friendlists
* rsvp&#95;event
* user&#95;about&#95;me
* user&#95;actions-books
* user&#95;actions-fitness
* user&#95;actions-music
* user&#95;actions-news
* user&#95;actions-video
* user&#95;education&#95;history
* user&#95;games&#95;activity
* user&#95;relationship&#95;details
* user&#95;relationships
* user&#95;religion&#95;politics
* user&#95;website
* user&#95;work&#95;history

<div id="permissions-moved-to-deprecated">
  #### Deprecated に移動された権限
</div>

以下の権限は **Deprecated** セクションに移動されており、Graph API の最新バージョンでは使用しないでください。

* publish&#95;actions
* user&#95;managed&#95;groups

<div id="facebook-graph-api-fields">
  ### Facebook Graph API フィールド
</div>

[Facebook Graph API](https://developers.facebook.com/docs/graph-api/reference/v3.2/user) は、ユーザーのログイン後に Auth0 ユーザーのプロファイルデータを取得するために使用されます。要求されるユーザーデータの権限によって、Graph API から取得される情報が決まります。返されるフィールドは、要求された権限と、それらのフィールドが Facebook のユーザープロファイルに存在するかどうかによって異なります。

この変更により、Graph API は v2.8 から v3.2 にアップグレードされ、ログイン時に次のユーザーデータフィールドが要求されます。

* address (追加)
* age&#95;range
* birthday
* context
* cover
* currency (追加)
* devices
* email
* favorite&#95;athletes
* favorite&#95;teams
* first&#95;name
* gender
* hometown
* id
* inspirational&#95;people
* install&#95;type (追加)
* installed
* is&#95;verified
* languages
* last&#95;name
* link
* locale
* location
* meeting&#95;for (追加)
* middle&#95;name
* name
* name&#95;format
* picture
* public&#95;key (追加)
* quotes
* security&#95;settings (追加)
* short&#95;name (追加)
* significant&#95;other
* sports (追加)
* third&#95;party&#95;id
* timezone
* updated&#95;time
* verified
* video&#95;upload&#95;limits (追加)
* viewer&#95;can&#95;send&#95;gift (追加)
