---

無料のサブスクリプションプランをご利用のお客様を含め、すべてのお客様がセキュリティに関する懸念事項を報告できます。報告を提出する前に、詳細について [Auth0 の脆弱性開示ポリシー](https://www.okta.com/content/dam/okta---digital/en_us/legal/okta-vulnerability-disclosure-policy_v2.0.pdf) をご確認ください。

調査結果の公開を希望する場合は、[psirt@okta.com](mailto:psirt@okta.com) まで直接レポートを提出してください。可能であれば、弊社の [GPG 暗号化キー](https://www.okta.com/vulnerability-reporting-policy/okta-pgp-key/) を使用してください。

報奨を希望する場合は、Bugcrowd の [標準開示条件](https://www.bugcrowd.com/resources/essentials/standard-disclosure-terms/) が適用されます。また、報奨を受けるには、[Bugcrowd](https://bugcrowd.com/okta) 経由で提出する必要があります。詳細については、[Auth0 のバグバウンティプログラム](https://bugcrowd.com/auth0-okta) をご覧ください。

法的情報については、[法的情報の詳細](https://auth0.com/legal) をお読みください。
