---

ご利用中のAuth0サブスクリプションプランによって、利用可能なユーザー数の上限や機能が決まります。

<div id="quota-utilization">
  ## クォータ使用状況
</div>

[Reports &gt; Quota Utilization](https://support.auth0.com/reports/quota) にアクセスすると、Auth0 Support Center でサブスクリプションの使用状況を確認できます。使用状況情報では、次の詳細を確認できます。

* ユーザー使用量

  * アクティブユーザー
* 機能使用量

  * エンタープライズ接続
  * マシン間認証
  * 多要素認証

<Frame>![サブスクリプション機能の使用状況を示すレポート](/docs/images/cdy7uua7fh8z/7g0PDAKl5D8YEf4vC5FjSI/188182aabbeb7e0af77b1bc9d292cfad/Screenshot_2023-02-07_at_5.35.40_PM.jpeg)</Frame>

各サマリーカードには、当月の使用状況とサブスクリプションで利用可能な上限が表示されます。また、カードをクリックするとドリルダウンして、Auth0 Private Instance ごとの使用量の内訳 (該当する場合) や、過去 12 か月分までの テナント ごとの使用量などの詳細を確認できます。

必要に応じて、このデータをエクスポートするための CSV をダウンロードすることもできます。

<Frame>![](/docs/images/cdy7uua7fh8z/7bBqDuycQWsnUdd9hlXN10/040a04d458bff06e8392c1432e3308ce/Screenshot_2023-02-07_at_6.07.01_PM.jpeg)</Frame>

<div id="active-users">
  ### アクティブユーザー
</div>

これを使用すると、特定の暦月においてアクティブなイベント (ログインまたはサインアップ) が発生したユニークユーザー数を確認できます。さらに、最大で過去12か月分まで、これらのアクティブユーザーの内訳を テナント および Private Auth0 Instance (該当する場合) ごとにドリルダウンして確認できます。

<div id="enterprise-connections">
  ### エンタープライズ接続
</div>

[エンタープライズ接続](/ja/docs/authenticate/enterprise-connections)は、当月中に次の両方を満たしている場合、アクティブとみなされます。

* いずれかのアプリケーションで有効になっていること。
* ユーザーアクティビティ (たとえば、ログイン、サインアップ、またはトークン交換) があること。

エンタープライズ接続がどのアプリケーションでも一度も有効になっていない場合、または有効になっていても当月中にユーザーアクティビティがなかった場合、その接続はアクティブとはみなされません。

このレポートを使用すると、特定の暦月におけるアクティブなエンタープライズ接続数を確認できます。また、 (該当する場合は) Private Auth0 Instance ごとのアクティブな接続の内訳や、アクティブなエンタープライズ接続およびテナントごとのアクティブユーザー数も確認できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Okta Workforce Enterprise 接続は、アクティブなエンタープライズ接続数の上限にはカウントされません。
</Callout>

<div id="machine-to-machine-auth">
  ### マシン間認証
</div>

このレポートでは、[クライアント認証情報フロー](/ja/docs/get-started/authentication-and-authorization-flow/client-credentials-flow)で発行された<Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報です。" cta="用語集を見る" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip>の数を、Auth0 <Tooltip tip="Management API: お客様が管理タスクを実行するためのプロダクトです。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip>向けに発行されたトークンを除き、暦月ごとに集計して表示します。これは API Authentication Calls とも呼ばれます。

Private Auth0 Instance (該当する場合) およびテナントごとの認証コール数を確認できます。

Auth0 Public Cloud デプロイオプションのテナントでは、このレポートで過去 7 日間のテナント内のApplication別の認証コールの内訳も確認できます。

<div id="multi-factor-auth">
  ### 多要素認証
</div>

多要素認証 (<Tooltip tip="多要素認証（MFA）: SMS 経由のコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=MFA">MFA</Tooltip>) の利用状況には 2 種類のレポートがあり、各暦月における MFA イベント (登録またはチャレンジ) の件数が表示されます。

Enterprise MFA の件数には、プッシュ通知、WebAuthn、SMS、Email、Phone によるワンタイムコードに関連する MFA イベントが含まれます。Pro MFA の件数には、Google Authentication や Cisco DUO などのワンタイムコードアプリに関連する MFA イベントが含まれます。

これら 2 種類のレポートでは、Auth0 Private Instance* (該当する場合) およびテナントごとの MFA 件数の内訳を確認できます。

*Auth0 Private Instance の MFA 利用状況は、2023 年 1 月 1 日以降のデータのみ利用できます。

***

<div id="tenant-usage">
  ## テナントの使用状況
</div>

特定のテナントの総ユーザー数とアクティブユーザー数は、Auth0 Support Center の [Reports &gt; Usage](https://support.auth0.com/reports/usage) にアクセスし、Subscription (セルフサービス サブスクリプションの場合は テナント も) を選択すると、暦月ごとに確認できます。表示される内訳は次のとおりです。

* アクティブユーザー: 月ごとおよびユーザータイプ (Social、Password、<Tooltip tip="パスワードレス: 最初の認証要素としてパスワードに依存しない認証方式。" cta="用語集を表示" href="/ja/docs/glossary?term=Passwordless">パスワードレス</Tooltip>、Enterprise) ごとのアクティブユーザー数と、Total Active の合計数。
* テナント別アクティブユーザー: 月、テナント、ユーザータイプ (Social、Password、Passwordless、Enterprise) ごとのアクティブユーザー数と、Total Active の合計数。
* テナント別総ユーザー数

<Frame>![Support Center -> Reports -> Usage](/docs/images/cdy7uua7fh8z/2RmWp1ZIWwV2JCPJcftll5/acd9c3c71b8dbd9cca12a44588e015af/Screen_Shot_2022-10-27_at_1.21.35_PM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザータイプを特定できない場合、Total Active または Total Users の数が、各ユーザータイプの合計を上回ることがあります。これは、アカウントのリンクが有効になっている場合などに発生する可能性がありますが、これに限りません。
</Callout>
