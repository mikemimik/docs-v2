<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で、Auth0 テナントを削除したり、デフォルトの構成にリセットしたりできます。Auth0 アカウントのデータは、Auth0 Support のサポートを受けて削除できます。

<div id="delete-tenant">
  ## テナントを削除
</div>

<Warning>
  テナントの削除は、元に戻せない破壊的な操作です。一度削除したテナントは復元できず、新しいテナントを作成する際に同じテナント名を再利用することもできません。また、そのテナントに関連するメトリクスも失われます。
</Warning>

テナントを削除する前に、次の選択肢を検討してください。

* [テナントの所有者を変更する](/docs/ja/get-started/manage-dashboard-access)。
* [テナント設定をリセットする](#reset-tenant-configuration)。

<div id="remove-custom-domain">
  ### カスタムドメインを削除する
</div>

削除したいテナントに <Tooltip tip="カスタムドメイン: 固有名（バニティ名）を使用するサードパーティドメイン。" cta="用語集を表示" href="/docs/ja/glossary?term=custom+domain">カスタムドメイン</Tooltip> を設定している場合は、先にそのカスタムドメインを削除する必要があります。削除しないと、そのカスタムドメインを別のテナントに設定できません。

<div id="determine-deletion-process">
  ### 削除方法を確認する
</div>

テナントの削除方法は、関連付けられている[サブスクリプションプラン](https://www.auth0.com/pricing)と、[テナントの種類](/docs/ja/get-started/auth0-overview/create-tenants/child-tenants) (メインまたは子) によって異なります。[Auth0 Dashboard](#delete-tenant-in-auth0-dashboard) で削除できる場合もあれば、[Auth0 Support または Community への問い合わせ](#contact-auth0-support-or-community)が必要な場合もあります。

|                     | Free Plan               | Self-service plan     | Enterprise plan       |
| ------------------- | ----------------------- | --------------------- | --------------------- |
| **メインテナント**         | Auth0 Dashboard         | Auth0 Dashboard       | Auth0 Support に問い合わせる |
| **子テナントを持つメインテナント** | Auth0 Community に問い合わせる | Auth0 Support に問い合わせる | Auth0 Support に問い合わせる |
| **子テナント**           | Auth0 Dashboard         | Auth0 Dashboard       | Auth0 Dashboard       |

<div id="downgrade-tenant">
  ### テナントのダウングレード
</div>

Auth0 Dashboardでテナントを削除するには、事前にFreeプランへダウングレードする必要があります。詳しくは、[Auth0サブスクリプションのダウングレードまたはキャンセル](/docs/ja/troubleshoot/customer-support/manage-subscriptions/downgrade-or-cancel-subscriptions)をご覧ください。

<div id="delete-tenant-in-dashboard">
  ### Auth0 Auth0 Dashboardでテナントを削除する
</div>

[適切なアクセス レベル](/docs/ja/get-started/manage-dashboard-access/feature-access-by-role)があれば、Auth0 Auth0 Dashboardでテナントを削除できます。

1. [Auth0 Dashboard &gt; Settings](https://manage.auth0.com/#/tenant/advanced)に移動し、**Advanced**ビューに切り替えます。
2. ページ下部の**Danger Zone**で、**Delete**を選択します。
3. 確認プロンプトの内容を確認し、テナントを完全に削除することを確認します。
4. **Yes, Delete Tenant**を選択します。

<div id="contact-auth0-support-or-community">
  ### Auth0 Support または Community に問い合わせる
</div>

テナントが Trial、Essential、Professional、または Enterprise プランで、Auth0 Dashboard でテナントを削除できない場合は、サポートを受けるために [Auth0 Support に問い合わせてください](/docs/ja/troubleshoot/customer-support/open-and-manage-support-tickets)。

テナントが Free プランで、Auth0 Dashboard で削除できない場合は、[Auth0 Community に投稿して](https://community.auth0.com/)、サポートを依頼してください。

<div id="reset-tenant-configuration">
  ## テナント設定をリセットする
</div>

[Auth0 Deploy CLI](https://github.com/auth0/auth0-deploy-cli) を使用すると、テナントを既定の設定にリセットできます。

Deploy CLI は、Auth0 テナントの設定管理に役立つツールです。スタンドアロン CLI または Node.js モジュールとして開発ワークフローに組み込めます。Deploy CLI ツールの詳細と利用開始方法については、[Deploy CLI Tool](/docs/ja/deploy-monitor/deploy-cli-tool) を参照してください。

<div id="delete-auth0-account-data">
  ## Auth0 アカウントデータを削除する
</div>

Auth0 アカウントデータ (メールアドレスや請求情報など) を削除するには、テナントを削除する手順に従ったうえで、[Auth0 Community](https://community.auth0.com/) に投稿し、サポートを依頼してください。

データ管理者としての Auth0 の責任について詳しくは、[Okta Privacy Policy](https://www.okta.com/privacy-policy/) をご覧ください。
