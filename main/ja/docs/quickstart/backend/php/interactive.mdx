    Auth0を使用すると、ほぼあらゆる種類のアプリケーションに対して、トークンベースのエンドポイント認可を素早く追加できます。このガイドでは、Auth0 PHP SDKを使ってAuth0を統合し、トークンベースの認可を追加して、アプリケーションのルートを保護する方法を説明します。

    このクイックスタートを使用するには、以下が必要です：

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard にアプリケーションを登録しておく必要があります。Auth0 アプリケーション
      では、プロジェクトの認証の動作を設定します。

      ### アプリケーションを設定する

      インタラクティブセレクターを使用して、新しい Auth0 アプリケーションを作成するか、
      連携したいプロジェクトに対応する既存のアプリケーションを選択します。Auth0 の各アプリケーションには、一意の英数字のクライアント ID が割り当てられ、
      アプリケーションコードはそれを使って SDK 経由で Auth0 API を呼び出します。

      この Quickstart を使用して行った設定は、[Auth0 Dashboard](https://manage.auth0.com/#/) 上のアプリケーションに自動的に反映されます。今後はそこで
      アプリケーションを管理できます。

      設定全体を確認したい場合は、代わりにサンプルアプリケーションを表示することもできます。

      ### API を設定する

      同様に、新しい Auth0 API を作成するか、[Auth0 Dashboard](https://manage.auth0.com/#/) から
      連携するプロジェクトに対応する既存の API を使用する必要があります。API には一意の識別子を選び、それを控えておいてください。以下でアプリケーションを設定するには、
      その識別子が必要です。
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 は、PHP アプリに Auth0認証と認可を実装するプロセスを簡素化するための [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワークリクエストを管理するための、[PSR-17](https://www.php-fig.org/psr/psr-17/) と [PSR-18](https://www.php-fig.org/psr/psr-18/) に対応した HTTP ライブラリをインストールしておく必要があります。利用可能なライブラリが
      ない場合は、ターミナルで次のコマンドを実行して、信頼性の高いライブラリを
      インストールできます。

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      次に、ターミナルで次のコマンドを実行して Auth0 PHP SDK をインストールします。

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      SDK が正しく機能するようにするには、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `strategy`: `strategy` は、アプリケーションのユースケースに応じた SDK の動作を決めるためのものです。この
        場合は、次の定数に設定します。

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain: Auth0 テナントのドメインです。通常、Auth0 Dashboard のアプリケーションの
        設定にある Domain フィールドで確認できます。カスタムドメインを使用している場合は、代わりに
        そのカスタムドメインの値を設定してください。
      * clientId: この Quickstart の前の手順で設定した Auth0 アプリケーションの ID です。これは
        Auth0 Dashboard のアプリケーションの設定にある Client ID フィールドで確認できます。
      * clientSecret: この Quickstart の前の手順で作成した Auth0 アプリケーションのシークレットです。Client secret は
        Auth0 Dashboard のアプリケーションの設定にある Client Secret フィールドにあります。
      * audience: 上で登録した Auth0 API の識別子です。これは配列として指定する必要があります。

      <Note>
        ##### チェックポイント

        これで Auth0 SDK は正しく設定されました。アプリケーションを実行して、次のことを確認してください。

        * SDK が正しく初期化されること。
        * アプリケーションで Auth0 に関連するエラーが発生しないこと。
      </Note>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      次に、ベアラートークンを取得して処理できるよう、アプリケーションを拡張します。ベアラートークンは、クライアントがユーザーに代わって API へのリクエスト時に送信するアクセストークンです。アクセストークンは、アプリケーション内のルートへのアクセスを許可または拒否します。これはエンドポイント認可と呼ばれます。

      リクエストからアクセストークンを取得する最も簡単な方法は、PHP SDK の `getBearerToken()`
      メソッドを使うことです。このメソッドは、GET パラメータ、POST ボディ、リクエストヘッダー、その他のソースからトークンを取得します。この
      場合、PHP SDK は、GET リクエストの `token` パラメータ、または HTTP
      `Authorization` ヘッダーで渡されたトークンを処理します。
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      次に、受信したリクエストをアプリケーションに振り分けるためのルーティングライブラリをインストールします。これは必須の手順ではありませんが、
      このQuickstartでは、アプリケーションの構成を簡潔にするために導入しています。

      ```bash lines
      composer require steampixel/simple-php-router
      ```

      ルートを定義するため、アプリケーションに `router.php` という新しいファイルを作成します。右側のインタラクティブパネルにある **router.php** タブのコードをコピーしてください。
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Auth0アプリケーション、Auth0 PHP SDK を設定し、アプリケーションがリクエストからベアラー
      トークンを取得できるようになったので、次のステップはプロジェクトのエンドポイントの認可を設定することです。上で実装した
      `getBearerToken()` メソッドは、リクエストのアクセスに関する
      詳細を含む `Token` クラスを返します。

      `getBearerToken()` メソッドは受信したリクエストを自動的に検証して確認するため、
      アプリケーションはメソッドのレスポンスを評価して access token の詳細を判断します。レスポンスが
      null の場合、有効な token は渡されていません。そうでない場合は、レスポンスの内容を確認して
      リクエストについてさらに詳しく把握してください。

      右側のインタラクティブパネルでは、レスポンスが null かどうかを確認して、
      `/api/private` ルートへのアクセスを制限する方法を確認できます。
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      場合によっては、access
      token でリクエストされた scopes に基づいて、特定の route へのアクセスを制限したいことがあります。右側の対話型パネルに示されているように、
      `getBearerToken()` メソッドの応答にある &#39;scope&#39; プロパティの内容を確認し、access token に付与された scopes を確認します。
    </Section>
