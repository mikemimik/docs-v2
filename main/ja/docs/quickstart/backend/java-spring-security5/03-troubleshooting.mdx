このドキュメントでは、API から予期しないレスポンスが返された場合に、設定のトラブルシューティングを行う方法を説明します。アカウント用に設定された例を使ってこの Quickstart を進められるよう、ログインしておくことをおすすめします。

JWT のバリデーションが正しく設定されていれば、リクエストを送信した際に API から適切なレスポンスを受け取ることができます。一方で、API から 401 (Unauthorized) レスポンスが返される場合は、JWT ミドルウェアの設定が渡された JWT と一致していないことが原因です。

このドキュメントは、JWT ミドルウェアの設定をトラブルシューティングする際に役立ちます。

<div id="how-does-a-token-get-validated">
  ## トークンはどのように検証されますか？
</div>

JWT を検証する際には、いくつか確認すべき点があります。

1. **トークンは正しい形式ですか？** つまり、このトークンは JSON Web トークン (JWT) の構造に準拠していますか？ JWT の構造について詳しくは、[JWT の構造に関するこちらのセクション](/ja/docs/secure/tokens/json-web-tokens#what-is-the-json-web-token-structure-)を参照してください。
2. **トークンは改ざんされていませんか？** JWT の最後の部分は署名です。この署名は、トークンが実際に送信者によって署名されており、いかなる変更も加えられていないことを確認するために使用されます。
3. **トークンは有効期間内に受信されましたか？** JWT は指定された期間内でのみ有効です (`exp` claim で示されます) 。
4. **トークンは想定された認可主体から送られてきていますか？** これには 2 つの要素があります。

   * **署名の検証**: JWT が、発行主体によって発行されたキーを使って正しく署名されていることを確認できますか？
   * **発行者の値**: 発行者は `iss` claim で定義されます。ここでも、この claim はアプリケーションが想定しているものと一致していますか？
5. **トークンは現在のアプリケーション向けですか？** つまり、JWT の `aud` claim はアプリケーションが期待しているものと一致していますか？

<div id="inspecting-a-token">
  ## トークンを確認する
</div>

JWT を手早く確認する方法の 1 つは、[JWT.io](https://jwt.io/) の Web サイトを使うことです。ここには便利なデバッガーがあり、JWT の形式が正しいかどうかをすばやく確認できるほか、さまざまなクレームの値を調べることもできます。

<Frame>![JWT.io で JWT をデバッグする](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-rs256.png)</Frame>

上のスクリーンショットでは、トークンが **RS256** アルゴリズムで署名されていることがわかります。トークンの **発行者** は **https://jerrie.auth0.com/**、**audience** は **https://rs256.test.api** です。

つまり、JWT バリデーションの設定に登録するこれらの値は、発行者の末尾のスラッシュも含めて、**完全に** 一致している必要があります。たとえば次のようにします

```
audience = "https://rs256.test.api",
domain = "https://jerrie.auth0.com/"
};
```

HS256 で署名されたトークンの場合、デバッガービューは少し異なって表示されます。

<Frame>![JWT.io で JWT をデバッグする](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-hs256.png)</Frame>

上のスクリーンショットでは、トークンが **HS256** アルゴリズムで署名されていることがわかります。トークンの **発行者** は **https://jerrie.auth0.com/**、**audience** は **https://hs256.test.api** です。
