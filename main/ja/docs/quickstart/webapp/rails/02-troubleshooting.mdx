<div id="troubleshooting">
  # トラブルシューティング
</div>

以下は、Railsのクイックスタートを使用する際に発生する可能性がある問題についてのトラブルシューティング トピックです。

<div id="using-a-reverse-proxy">
  ## リバースプロキシを使用する場合
</div>

ログインにリダイレクトするときに OmniAuth が生成する `redirect_uri` パラメータは、Rails に渡される `Host` ヘッダーに基づいています。そのため、この strategy をリバースプロキシと併用すると (OmniAuth 全般でも同様です) 、誤ったコールバック URL が渡されることがあります。次のスニペットを使うと、OmniAuth が使用するホストを調整できます。

```
OmniAuth.config.full_host = lambda do |env|
    scheme         = env['rack.url_scheme']
    local_host     = env['HTTP_HOST']
    forwarded_host = env['HTTP_X_FORWARDED_HOST']
    forwarded_host.blank? ? "#{scheme}://#{local_host}" : "#{scheme}://#{forwarded_host}"
end
```

[詳細については、こちらのStack Overflowのスレッドをご覧ください](https://stackoverflow.com/a/7135029/728480).

<div id="actioncontrollerinvalidauthenticitytoken">
  ## ActionController::InvalidAuthenticityToken
</div>

これは、ログインrequestをPOSTする際に必要なCSRFトークンが欠けていることが原因である可能性があります。ブラウザーでログインボタンを調べると、次のように表示されるはずです。

```erb lines
<a data-method="post" href="auth/auth0">Login</a>
```

... また、ページの`<head>`要素には、次のようなCSRFメタタグが含まれているはずです。

```erb lines
<meta name="csrf-param" content="authenticity_token">
<meta name="csrf-token" content="UY2XpKwxzwBWalxFVJ8yKsao/33it7If09BnZewpHifVPSpFJd2LrA7xgQn6VQrhZNGjgZoLI3kV+bkQHtr+Rw==">
```

これらの要素を追加すると、Rails はログインリンクを、ログインにリダイレクトする前に CSRFトークンをバックエンドへ POST して検証できるよう変換します。

<div id="actiondispatchcookiescookieoverflow">
  ## ActionDispatch::Cookies::CookieOverflow
</div>

このエラーは、Cookie セッションを使用していて、プロファイル全体を保存しているために、最大サイズの 4 kb を超えてしまっていることを意味します。ユーザープロファイルにアクセスできない場合や、`NoMethodError`、`undefined method '[]' for nil:NilClass` のようなエラーが表示される場合は、開発環境では In-Memory store を使用してみてください。

`/config/initializers/session_store.rb` に移動し (存在しない場合は作成して) 、以下を追加します。

```
Rails.application.config.session_store :cache_store
```

`/config/environments/development.rb` を開き、以下を追加します:

```
config.cache_store = :memory_store
```

これらの変更を反映するには、Rails サーバーを再起動してください。

本番環境のアプリケーションでは、MemCached などのメモリストアを使用することを推奨します。

<div id="ssl-issues">
  ## SSL の問題
</div>

一部の環境では、Ruby が認証局証明書 (CA 証明書) を見つけられないことがあります。

CA 証明書バンドルをプロジェクトのディレクトリにダウンロードしてください。

```bash lines
curl -L -o lib/ca-bundle.crt http://curl.haxx.se/ca/ca-bundle.crt
```

このイニシャライザを `config/initializers/fix_ssl.rb` に追加してください:

```rb lines
# config/initializers/fix_ssl.rb

require 'open-uri'
require 'net/https'

module Net
  class HTTP
    alias_method :original_use_ssl=, :use_ssl=

    def use_ssl=(flag)
      path = ( Rails.env == "development") ? "lib/ca-bundle.crt" : "/usr/lib/ssl/certs/ca-certificates.crt"
      self.ca_file = Rails.root.join(path).to_s
      self.verify_mode = OpenSSL::SSL::VERIFY_PEER
      self.original_use_ssl = flag
    end
  end
end
```

<div id="failure-messageinvalid_credentials">
  ## &quot;failure message=invalid_credentials&quot;
</div>

この問題はローカル環境では発生しませんが、ステージング環境や本番環境では発生することがあります。エラーメッセージは次のように表示される場合があります。

```
omniauth: (auth0) Authentication failure! invalid_credentials: OAuth2::Error, server_error: The redirect URI is wrong. You send [wrong url], and we expected [callback url set in your app settings]
```

これを解決するには、以下を`config/environments/staging.rb`または`production.rb`に追加します:

```
OmniAuth.config.full_host = "http://www.example.com"
```

`http://www.example.com` は、アプリケーションで実際に使用する URL に置き換えてください。
