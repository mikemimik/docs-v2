    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションでは、
      開発中のプロジェクトで認証をどのように動作させるかを設定します。

      ### アプリケーションを設定する

      対話型セレクターを使用して新しい Auth0 アプリケーションを作成するか、連携するプロジェクトに対応する既存のアプリケーションを選択します。Auth0 のすべてのアプリケーションには、英数字からなる一意のクライアント ID
      が割り当てられており、アプリケーションのコードは SDK を通じて Auth0 API を呼び出す際にこれを使用します。

      この Quickstart で行った設定は、[Auth0 Dashboard](https://manage.auth0.com/#/) 上のアプリケーションに自動的に反映されます。今後はここで
      アプリケーションを管理できます。

      設定内容をひととおり確認したい場合は、代わりにサンプルアプリケーションを参照できます。

      ### コールバック URL を設定する

      コールバック URL は、ユーザーの認証後に Auth0 がリダイレクトする先として指定する、アプリケーション内の URL です。
      これを設定しない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを
        `http://localhost:3000/auth/auth0/callback` に設定してください。
      </Info>

      ### Logout URL を設定する

      Logout URL は、ユーザーのログアウト後に Auth0 がリダイレクトする先として指定する、アプリケーション内の URL です。これを
      設定しない場合、ユーザーはアプリケーションからログアウトできず、エラーが表示されます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

      ### 許可する Web オリジンを設定する

      許可する Web オリジンとは、認証フローへのアクセスを許可する URL のことです。ここには
      プロジェクトの URL を含める必要があります。適切に設定されていない場合、プロジェクトで認証トークンをサイレントに更新
      できなくなるため、ユーザーは次回アプリケーションにアクセスしたとき、またはページを
      更新したときにログアウトされます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      認証フローを処理するには、カスタムの[OmniAuthストラテジー](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication)である`omniauth-auth0`を使用します。

      次の依存関係を`Gemfile`に追加します。

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # 偽造された認証リクエストを防ぐ
      ```

      gem を追加したら、`bundle install` を実行してインストールします。
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Auth0 Dashboard のアプリケーション **設定** にある Auth0 ドメイン、client ID、client
      secret の値を指定するため、設定ファイル `./config/auth0.yml` を作成します。
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      次の初期化ファイル `./config/initializers/auth0.rb` を作成し、前の手順で作成した設定ファイルを使って **OmniAuth** ミドルウェアを[設定](https://github.com/auth0/omniauth-auth0/blob/master/EXAMPLES.md#send-additional-authentication-parameters)します。

      `callback_path` が、Auth0 アプリケーションの &quot;Allowed Callback URLs&quot; 設定で指定した値と一致していることを確認してください。
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      認証コールバック、`logout` アクション、および
      ログアウト URL を構築するメソッドを処理する Auth0 コントローラーを作成します。

      次のコマンドを実行します。
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`

      コールバックメソッド内で、
      `request.env['omniauth.auth']` として返されるユーザー情報のハッシュを、現在の
      セッションに代入します。

      ログアウトを設定するには、`logout` アクション内で `reset_session`
      メソッドを呼び出し、セッション内に保存されているすべてのオブジェクトをクリアします。その後、
      Auth0 のログアウト エンドポイントにリダイレクトします。`reset_session` の詳細については、
      [Ruby on Rails ActionController documentation](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668) を参照してください。
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      これらのルートを `./config/routes.rb` ファイルに追加します。

      前のステップで作成した Auth0 コントローラーに、さまざまな Auth0 コールバック URL を Rails が適切に振り分けられるよう、ルートを設定しておく必要があります。

      <Info>
        ##### チェックポイント

        アプリケーションを実行し、意図したとおりに動作していること、および Auth0 に関連するエラーが発生していないことを確認してください。
      </Info>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      ユーザーは `/auth/auth0` エンドポイントにアクセスすることで、アプリケーションにログインできるようになります。

      <Warning>
        [偽造された認証リクエストを防ぐ](https://github.com/cookpad/omniauth-rails_csrf_protection)には、`link_to` または
        `button_to` ヘルパーメソッドを `:post` メソッドとともに使用してください。
      </Warning>

      ```ruby lines
      <!-- アプリケーションの任意の場所にログインボタンを配置してください -->
      <%= button_to 'Login', '/auth/auth0', method: :post %>
      ```

      <Info>
        ##### チェックポイント

        選択するとユーザーが `/auth/auth0` エンドポイントにリダイレクトされるボタンを、アプリケーションに追加します。
        Auth0 にリダイレクトされてログインし、認証が成功するとアプリに戻ることを確認してください。
      </Info>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      これで Rails アプリケーションにログインできるようになったので、ログアウトする
      方法も必要です。ユーザーを `auth/logout` アクションにリダイレクトすると、Auth0 のログアウトエンドポイントに
      リダイレクトされ、ログアウトできます。

      <Info>
        前の手順のあとでこれをテストするには、セッションをクリアしてから、ユーザーを
        Auth0 のログアウトエンドポイントにリダイレクトする必要がある場合があります。
      </Info>

      ```ruby lines
      <!-- アプリケーションの任意の場所にログアウトボタンを配置してください -->
      <%= button_to 'Logout', 'auth/logout', method: :get %>
      ```

      <Info>
        ##### チェックポイント

        選択するとユーザーが `/auth/logout` エンドポイントにリダイレクトされるボタンを、アプリケーションに追加します。Auth0 にリダイレクトされたあと、すぐにアプリケーションに戻り、ログイン状態ではなくなっていることを確認してください。
      </Info>
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      ユーザーのプロファイルを表示するには、アプリケーションで保護されたルートを用意する必要があります。[Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) を使うと、複数の
      コントローラーで共有されるルートへのアクセスを制御できます。ユーザーが未認証の場合、Concern は自動的に Auth0 へリダイレクトするようにします。認証済みの場合は、
      Concern が現在のユーザーのプロファイルを返すようにします。

      Concern を用意したら、ログイン済みユーザーが必要なコントローラーにそれをインクルードします。すると、次の例のように
      セッション `session[:userinfo]` からユーザー情報にアクセスできます。

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      ユーザー情報をセッションから読み込んだら、その情報を使ってフロントエンドに表示します:

      ```html lines
      <div>
      <p>正規化されたユーザープロファイル:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>完全なユーザープロファイル:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Info>
        ##### チェックポイント

        アプリに `Secured` concern を追加し、認証済みユーザーによるアクセスが必要なコントローラーに組み込みます。認証済みユーザーがそのコントローラー内のアクションにアクセスできること、また未認証のユーザーが認証のために Auth0 にリダイレクトされることを確認します。
      </Info>
    </Section>
