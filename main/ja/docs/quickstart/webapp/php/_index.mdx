    Auth0を使用すると、ほぼあらゆる種類のアプリケーションに素早く認証を追加できます。このガイドでは、Auth0 PHP SDKを使用して、任意のPHPアプリケーションにAuth0を統合し、認証を追加して、ユーザープロファイル情報を表示する方法を説明します。

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションでは、
      開発中のプロジェクトで認証をどのように機能させるかを設定します。

      ### アプリケーションを設定する

      対話型セレクターを使って新しい Auth0 アプリケーションを作成するか、統合したいプロジェクトに対応する既存のアプリケーションを選択します。Auth0 内のすべてのアプリケーションには、英数字の一意な Client ID
      が割り当てられます。アプリケーションのコードは、この ID を使って SDK 経由で Auth0 API を呼び出します。

      この Quickstart で行った設定は、[Auth0 Dashboard](https://manage.auth0.com/#/) のアプリケーションに自動的に反映されます。以後、アプリケーションの管理はここで行えます。

      設定済みの完全な例を確認したい場合は、代わりにサンプルアプリケーションを参照できます。

      ### コールバック URL を設定する

      コールバック URL は、ユーザーの認証完了後に Auth0 からリダイレクトする先の、アプリケーション内の URL です。これを設定していない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを
        `http://localhost:3000/callback` に設定してください。
      </Info>

      ### ログアウト URL を設定する

      ログアウト URL は、ユーザーのログアウト後に Auth0 からリダイレクトする先の、アプリケーション内の URL です。これを設定していない場合、ユーザーはアプリケーションからログアウトできず、エラーが表示されます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

      ### 許可する Web オリジンを設定する

      許可する Web オリジンは、認証フローへのアクセスを許可する URL です。ここには
      プロジェクトの URL を含める必要があります。正しく設定されていないと、プロジェクトで認証トークンをサイレントに更新できなくなるため、ユーザーは次回アプリケーションにアクセスしたときやページを更新したときに
      ログアウトされます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 は、PHP アプリに Auth0認証と認可を実装する作業を簡単にするための [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワークリクエストを処理するために、[PSR-17](https://www.php-fig.org/psr/psr-17/) および [PSR-18](https://www.php-fig.org/psr/psr-18/) に対応した HTTP ライブラリがインストールされている必要があります。利用可能なライブラリが
      ない場合は、ターミナルで次のコマンドを実行して、信頼性の高いライブラリをインストールできます。

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      次のコマンドをターミナルで実行して、Auth0 PHP SDKをインストールしてください。

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      アプリケーションに `index.php` という新しいファイルを作成し、右側のインタラクティブパネルの **index.php** タブにあるコードをコピーします。

      SDK が正しく動作するように、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `domain`: Auth0 テナントのドメインです。通常、この値は Auth0 Dashboard の
        アプリケーションの設定にある Domain フィールドで確認できます。[カスタムドメイン](https://auth0.com/docs/custom-domains) を使用している場合は、代わりに
        そのカスタムドメインの値を設定してください。
      * `clientId`: この Quickstart の前半で設定した Auth0 アプリケーションの ID です。これは
        Auth0 Dashboard のアプリケーションの設定にある Client ID フィールドで確認できます。
      * `clientSecret`: この Quickstart の前半で設定した Auth0 アプリケーションのシークレットです。これは
        Auth0 Dashboard のアプリケーションの設定にある Client Secret フィールドで確認できます。
      * `redirectUri`: 認証後に Auth0 がユーザーをリダイレクトする先として、
        アプリケーション内で指定する URL です。これは、この Quickstart の前半で設定した callback URL に対応します。この
        値は Auth0 Dashboard のアプリケーションの設定にある Callback URLs フィールドでも確認できます。コードに
        設定する値が前半で指定したものと一致していることを確認してください。一致していない場合、ユーザーにエラーが表示されます。
      * `cookieSecret`: セッション Cookie の暗号化に使用する、十分に長いシークレット値です。適切な文字列は、
        ターミナルで `openssl rand -hex 32` を実行して生成できます。

      <Info>
        ##### チェックポイント

        これで Auth0 SDK は正しく設定されているはずです。アプリケーションを実行して、次のことを確認してください。

        * SDK が正しく初期化されること。
        * アプリケーションで Auth0 に関連するエラーが発生していないこと。
      </Info>

      ユーザーがログインとログアウトを行えるようになったので、次は認証済みユーザーに関連付けられた
      [プロファイル情報](https://auth0.com/docs/users/concepts/overview-user-profile)を取得したいと思うはずです。
      たとえば、プロジェクト内にログイン中のユーザーの名前や
      プロフィール画像を表示したいことがあるでしょう。

      Auth0 PHP SDK では、`getCredentials()` メソッドを通じてユーザー情報を取得できます。使い方の例は、
      インタラクティブパネルで `profile.php` のコードを確認してください。

      このメソッドにはユーザーの本人確認に関する機微な情報が含まれるため、利用できるかどうかは
      ユーザーの認証状態に左右されます。レンダリングエラーを防ぐには、
      アプリケーションで結果を利用する前に、`getCredentials()` メソッドが `object` と `null` のどちらを返すかを必ず確認し、
      Auth0 がユーザーを認証しているかどうかを判断してください。

      <Info>
        ##### チェックポイント

        次のことを確認してください。

        * ログイン後に、`nickname` またはその他のユーザープロパティを正しく表示できること。
      </Info>
