  <Content>
    Auth0を使用すると、ほぼあらゆる種類のアプリケーションに素早く認証を追加できます。このガイドでは、Auth0 PHP SDKを使用して、任意のPHPアプリケーションにAuth0を統合し、認証を追加して、ユーザープロファイル情報を表示する方法を説明します。

    このクイックスタートを使用するには、以下が必要です：

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションは、
      開発中のプロジェクトで authentication をどのように機能させるかを設定する場所です。

      ### アプリケーションを設定する

      対話型セレクターを使って新しい Auth0 アプリケーションを作成するか、連携したいプロジェクトに対応する既存のアプリケーションを選択します。Auth0 の各アプリケーションには、英数字からなる一意のクライアント ID
      が割り当てられており、アプリケーションのコードは SDK を通じて Auth0 API を呼び出す際にこれを使用します。

      この Quickstart を使って行った設定は、[Auth0 Dashboard](https://manage.auth0.com/#/) のアプリケーションに自動的に反映されます。ここでは、
      今後もアプリケーションを管理できます。

      設定内容をひととおり確認したい場合は、代わりにサンプルアプリケーションを参照できます。

      ### コールバック URL を設定する

      コールバック URL は、ユーザーが認証された後に Auth0 がリダイレクトする、アプリケーション内の URL です。設定しない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを
        `http://localhost:3000/callback` に設定してください。
      </Info>

      ### Logout URL を設定する

      ログアウト URL は、ユーザーがログアウトした後に Auth0 がリダイレクトする、アプリケーション内の URL です。設定しない場合、ユーザーはアプリケーションからログアウトできず、エラーが表示されます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

      ### 許可する Web オリジンを設定する

      許可する Web オリジンは、authentication flow へのアクセスを許可する URL です。ここには
      プロジェクトの URL を含める必要があります。適切に設定されていないと、プロジェクトは
      認証トークンをサイレントに更新できなくなり、その結果、ユーザーは次回アプリケーションを開いたときやページを
      更新したときにログアウトされます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 は、PHP アプリに Auth0 の認証と認可を実装する作業を簡単にするための [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワーク要求を処理するために、[PSR-17](https://www.php-fig.org/psr/psr-17/) および [PSR-18](https://www.php-fig.org/psr/psr-18/) に準拠した HTTP ライブラリをインストールしておく必要があります。利用可能なライブラリが
      ない場合は、ターミナルで次のコマンドを実行して、信頼性の高いライブラリをインストールできます。

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      ターミナルで次のコマンドを実行し、Auth0 PHP SDKをインストールします。

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      アプリケーションに `index.php` という新しいファイルを作成し、右側のインタラクティブ
      パネルの **index.php** タブにあるコードをコピーします。

      SDK を正しく動作させるには、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `domain`: Auth0 テナントのドメインです。通常、この値は Auth0 Dashboard の
        アプリケーションの設定にある Domain フィールドで確認できます。[カスタムドメイン](https://auth0.com/docs/custom-domains) を使用している場合は、代わりに
        カスタムドメインの値を設定してください。
      * `clientId`: このクイックスタートの前半で設定した Auth0 アプリケーションの ID です。この値は
        Auth0 Dashboard のアプリケーションの設定にある Client ID フィールドで確認できます。
      * `clientSecret`: このクイックスタートの前半で設定した Auth0 アプリケーションのシークレットです。この値は
        Auth0 Dashboard のアプリケーションの設定にある Client Secret フィールドで確認できます。
      * `redirectUri`: ユーザーの認証後に
        Auth0 がリダイレクトするアプリケーション内の URL です。これは、このクイックスタートの前半で設定したコールバック URL に対応します。この値は
        Auth0 Dashboard のアプリケーションの設定にある Callback URLs フィールドでも確認できます。コードに入力する内容が、前半で設定した値と一致していることを
        必ず確認してください。一致していないと、ユーザーにエラーが表示されます。
      * `cookieSecret`: セッション Cookie の暗号化に使用する、十分に長いシークレット値です。適切な
        文字列は、ターミナルで `openssl rand -hex 32` を実行して生成できます。

      <Info>
        ##### チェックポイント

        これで Auth0 SDK は正しく設定されているはずです。アプリケーションを実行して、次の点を確認してください。

        * SDK が正しく初期化されていること。
        * アプリケーションで Auth0 に関連するエラーが発生していないこと。
      </Info>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      次に、受信したリクエストをアプリケーションに振り分けるためのルーティングライブラリをインストールします。これは必須の手順ではありませんが、
      この Quickstart では、アプリケーションの構成をシンプルにするのに役立ちます。

      ```text lines
      composer require steampixel/simple-php-router
      ```

      アプリケーション内に `router.php` という新しいファイルを作成してルートを定義し、右側のインタラクティブパネルにあるコードを
      コピーして貼り付けます。
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Auth0 アプリケーションと Auth0 PHP SDK の設定が完了したら、次はプロジェクトのログインを設定します。これを行うには、SDK の `login()` メソッドを使って、ユーザーを Auth0 Universal Login ページにリダイレクトするログインボタンを作成します。ユーザーの認証が正常に完了すると、このクイックスタートで前に設定したコールバック URL にリダイレクトされます。

      ログイン処理を行うために、アプリケーションに `login.php` という新しいファイルを作成し、右側のインタラクティブパネルにあるコードをコピーしてください。このコードには、ログインに必要なロジックが含まれています。

      <Info>
        ##### チェックポイント

        これで、ユーザー名とパスワードを使用してログインまたはサインアップできるようになっているはずです。

        ログインリンクをクリックし、次の点を確認してください。

        * アプリケーションが Auth0 Universal Login ページにリダイレクトされる。
        * ログインまたはサインアップできる。
        * SDK の設定に使用した `redirectUri` の値を使って、Auth0 からアプリケーションにリダイレクトされる。
      </Info>
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      プロジェクトにログインするユーザーには、ログアウトする手段も必要です。ここでは、SDK の
      `logout()` メソッドを使ってログアウトボタンを実装します。ユーザーがログアウトすると、[Auth0 logout](https://auth0.com/docs/api/authentication?http#logout) エンドポイントにリダイレクトされ、
      その後、この Quickstart で先ほど設定したログアウト URL にただちにリダイレクトされます。

      この処理を行うため、アプリケーションに `logout.php` という新しいファイルを作成し、対話パネルにある
      コードをコピーしてください。このコードには、ログアウトに必要なロジックが含まれています。続いて、
      `index.php` ファイルを更新し、ログアウトボタンを追加します。

      <Info>
        ##### チェックポイント

        アプリケーションを実行してログアウトボタンをクリックし、次の点を確認してください。

        * アプリケーションが、`Application Settings` の Allowed Logout URLs の 1 つとして指定したアドレスにリダイレクトされること。
        * アプリケーションからログアウトされていること。
      </Info>
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      ユーザーがログインおよびログアウトできるようになったら、次は認証済みユーザーに関連付けられた
      [プロファイル情報](https://auth0.com/docs/users/concepts/overview-user-profile)
      を取得したくなるはずです。たとえば、プロジェクト内でログイン中のユーザーの名前や
      プロフィール画像を表示したい場合があります。

      Auth0 PHP SDK では、`getCredentials()` メソッドを通じてユーザー情報を取得できます。使い方の例は、
      インタラクティブパネルの `profile.php` コードを確認してください。

      このメソッドにはユーザーの本人確認に関わる機密情報が含まれるため、利用できるかどうかは
      ユーザーの認証状態によって決まります。レンダリングエラーを防ぐため、`getCredentials()` メソッドが
      `object` または `null` を返すかどうかを必ず確認し、
      アプリケーションで結果を利用する前に Auth0 がユーザーを認証済みかどうかを判断してください。

      <Info>
        ##### チェックポイント

        次を確認してください:

        * ログイン後に、`nickname` またはその他のユーザープロパティを正しく表示できること。
      </Info>
    </Section>
