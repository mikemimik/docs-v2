  <Content>
    Auth0を使用すると、アプリケーションに認証を追加し、ユーザープロファイル情報へのアクセスが可能になります。このガイドでは、[Authlib](https://authlib.org/) SDKを使用して、Auth0をPython [Django](https://www.djangoproject.com/)アプリケーションに統合する方法を説明します。

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 のアプリケーションでは、開発中のプロジェクトで認証をどのように機能させるかを設定します。

      ### アプリケーションを設定する

      対話型セレクターを使って新しい Auth0 アプリケーションを作成するか、連携したいプロジェクトに対応する既存のアプリケーションを選択します。Auth0 のすべてのアプリケーションには、英数字からなる一意のクライアント ID が割り当てられます。アプリケーションのコードは、この ID を使って SDK 経由で Auth0 API を呼び出します。

      この Quickstart で行った設定は、[Auth0 Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/) のアプリケーションにも自動的に反映されます。今後はそこでアプリケーションを管理できます。

      設定例をひととおり確認したい場合は、代わりにサンプルアプリケーションを参照してください。

      ### コールバック URL を設定する

      コールバック URL とは、ユーザーの認証後に Auth0 がリダイレクトする先として指定する、アプリケーション内の URL です。これを設定しない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを
        `http://localhost:3000/callback` に設定してください。
      </Info>

      ### Logout URL を設定する

      Logout URL とは、ユーザーのログアウト後に Auth0 がリダイレクトする先として指定する、アプリケーション内の URL です。これを設定しない場合、ユーザーはアプリケーションからログアウトできず、エラーが発生します。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      この連携では、Authlib など、いくつかのライブラリの依存関係を追加します。プロジェクトのディレクトリに
      `requirements.txt` ファイルを作成し、次の内容を含めます。

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      これらの依存関係を利用できるようにするには、シェルで次のコマンドを実行します。

      ```text lines
      pip install -r requirements.txt
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      次に、プロジェクトのディレクトリに `.env` ファイルを作成します。このファイルには、クライアントのキーやその他の
      設定情報が格納されます。

      ```text lines
      AUTH0_CLIENT_ID={yourClientId}
      AUTH0_CLIENT_SECRET={yourClientSecret}
      AUTH0_DOMAIN={yourDomain}
      ```
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      すでに Django アプリケーションの設定が完了している場合は、次の手順に進んでください。新しいアプリケーションプロジェクトの場合は、次の
      コマンドを実行します。

      ```text lines
      django-admin startproject webappexample
      ```

      新しいプロジェクトフォルダーへ移動します:

      ```text lines
      cd webappexample
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      `.env` の値を確認するため、`webappexample/settings.py` ファイルを開きます。

      ファイルの先頭に、`os` と `dotenv` のインポートを追加します。

      次に、`BASE_DIR` の定義の下に `TEMPLATE_DIR` 変数を追加します。

      次に、`TEMPLATES` 変数を見つけて、`DIRS` の値を更新し、
      `TEMPLATE_DIR` 文字列を追加します。これにより、今後のステップで作成する
      テンプレートファイルのパスが決まります。この配列のほかの内容はそのままにしてください。

      このファイルの末尾に、Auth0 の設定を読み込むコードを追加します。
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      アプリケーションの作成を始めるには、IDE で `webappexample/views.py` ファイルを開きます。

      アプリケーションに必要なライブラリをすべてインポートします。

      これで、Auth0 でのアプリケーションの認証を処理するよう Authlib を設定できます。

      Authlib の OAuth `register()` メソッドで利用できる設定オプションの詳細は、[公式ドキュメントを参照してください。](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      この例では、アプリケーションに 4 つのルート (login、callback、logout、index) を追加します。

      * `login` - アプリの訪問者が `/login` ルートにアクセスすると、Auth0 に移動して
        認証フローを開始します。
      * `callback` - ユーザーが Auth0 でのログインを完了すると、`/callback` ルートでアプリケーションに
        戻ります。このルートはユーザーのセッションを保存するため、次回アクセス時に
        再度ログインする必要がなくなります。
      * `logout` - `/logout` ルートは、ユーザーをアプリケーションからサインアウトさせます。このルートは
        アプリ内のユーザーセッションを消去し、セッションが保持されないようにするために
        Auth0 の Logout endpoint にリダイレクトします。その後、アプリケーションはユーザーをホームルートにリダイレクトします。
      * `index` - ホームルートでは、認証済みユーザーの詳細をレンダリングするか、訪問者がサインインできるようにします。
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      `webappexample/urls.py` ファイルの内容を右側のコードに置き換えて、
      これらの新しいルートに接続します。

      これにより、`/login`、`/callback`、`/logout`、`/` の各ルートが
      適切なハンドラーに振り分けられます。
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      次に、ホームページのルートで使用するテンプレートファイルを作成します。

      `webappexample` フォルダー内に `templates` という名前の新しいサブディレクトリを作成し、
      `index.html` ファイルを作成します。

      `index.html` ファイルには、ログインしている場合はユーザー情報を表示し、ログアウトしている場合は
      ログインボタンを表示するためのテンプレートコードを記述します。
    </Section>

    <Section id={sections[9].id} title={sections[9].title} stepNumber="10">
      アプリケーションを実行する準備ができました！プロジェクトのディレクトリでシェルを開き、次を実行します:

      ```text lines
      python3 manage.py migrate
      python3 manage.py runserver 3000
      ```

      これで、ブラウザーで [http://localhost:3000](http://localhost:3000) を開けば、アプリケーションを利用できるようになっているはずです。

      <Info>
        ##### チェックポイント

        確認のため、[http://localhost:3000](http://localhost:3000/) にアクセスしてください。Auth0 に移動して login し、その後アプリケーションに戻ってプロファイル情報を表示するログインボタンがあるはずです。
      </Info>
    </Section>
