  <Content>
    Auth0を使用すると、ほぼあらゆる種類のアプリケーションに素早く認証を追加できます。このガイドでは、[Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/)版および[iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS)版のAuth0 SDKを使用して、.NET AndroidおよびiOSアプリケーションへのAuth0の統合、認証の追加、ユーザープロファイル情報の表示を行う方法を説明します。

    <Info>
      このクイックスタートでは、.NET Android と iOS を対象としています。これらは
      `Xamarin.Android` と `Xamarin.iOS` の次世代版だからです。現在も
      `Xamarin.Android` と `Xamarin.iOS` を使用している場合でも、連携は
      同じで SDK にも互換性があるため、このガイドをそのまま利用できます。
    </Info>

    このクイックスタートを使用するには、以下が必要です：

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションでは、
      プロジェクトの認証の動作を設定します。

      ### アプリケーションを設定する

      インタラクティブセレクターを使って新しい &quot;Native Application&quot; を作成するか、連携したいプロジェクトに対応する既存のアプリケーションを選択します。Auth0 のすべてのアプリケーションには英数字の一意な
      client ID が割り当てられ、アプリケーションのコードはそれを使って SDK 経由で Auth0 API を呼び出します。

      この Quickstart で行った設定は、[Auth0 Dashboard](https://manage.auth0.com/#/) 上のアプリケーションに自動的に反映されます。今後はここで
      アプリケーションを管理できます。

      構成全体を確認したい場合は、代わりにサンプルアプリケーションを参照することもできます。

      ### コールバック URL を設定する

      コールバック URL は、ユーザーの
      認証後に Auth0 がリダイレクトするアプリケーション内の URL です。設定しない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、プラットフォームに応じて次のいずれかの URL に設定してください。

        * **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      ### Logout URL を設定する

      Logout URL は、ユーザーのログアウト後に Auth0 がリダイレクトするアプリケーション内の URL です。設定しない場合、ユーザーはアプリケーションからログアウトできず、エラーが表示されます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、プラットフォームに応じて次のいずれかの URL に設定してください。

        * **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      最後に、アプリケーションの **アプリケーションの種類** が [アプリケーションの
      設定](https://manage.auth0.com/#/applications/) で **Native** に設定されていることを確認してください。
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 では、.NET Android および iOS アプリケーションに Auth0 の
      認証を実装する作業を簡単にするために、[Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/) 用と [iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS) 用の SDK を提供しています。

      NuGet パッケージ マネージャー (Tools -&gt; Library Package Manager -&gt; Package Manager Console) を使用して、
      Android アプリケーションをビルドするか iOS アプリケーションをビルドするかに応じて、`Auth0.OidcClient.AndroidX` または `Auth0.OidcClient.iOS` パッケージをインストールします。

      または、NuGet パッケージ マネージャー コンソール (`Install-Package`) や
      `dotnet` CLI (`dotnet add`) を使用することもできます。

      ```
      Install-Package Auth0.OidcClient.AndroidX
      Install-Package Auth0.OidcClient.iOS
      ```

      ```
      dotnet add Auth0.OidcClient.AndroidX
      dotnet add Auth0.OidcClient.iOS
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Auth0 をアプリケーションに統合するには、Auth0 Domain と Client ID を含む `Auth0ClientOptions` のインスタンスを渡して、
      `Auth0Client` クラスのインスタンスを生成します。

      ```csharp
      using Auth0.OidcClient;
      var client = new Auth0Client(new Auth0ClientOptions {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}"
      }, this);
      ```

      デフォルトでは、SDK は Android では Chrome Custom Tabs、iOS では ASWebAuthenticationSession を使用します。

      <Note>
        ##### 確認事項

        これで `Auth0Client` は正しく初期化されているはずです。アプリケーションを実行して、次の点を確認してください。

        * `Auth0Client` が、`Activity`(Android) または
          `UIViewController`(iOS) で正しく初期化されていること。
        * アプリケーションで Auth0 に関連するエラーが発生していないこと。
      </Note>
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      ユーザーの認証が正常に完了すると、この
      Quickstart で先ほど設定したコールバック URL にリダイレクトされます。

      Android デバイスでコールバックを処理するには、このコールバック URL を処理するインテントを登録する必要があります。これを簡単に
      行うには、`LoginAsync` メソッドを呼び出して認証フローを
      開始したのと同じアクティビティにインテントを登録します。

      コード サンプル内の `YOUR_ANDROID_PACKAGE_NAME` は、必ず実際の
      アプリケーションのパッケージ名 (`com.mycompany.myapplication` など) に置き換えてください。また、
      `DataScheme`、`DataHost`、`DataPathPrefix` の
      文字列はすべて小文字にしてください。さらに、Activity には
      `LaunchMode = LaunchMode.SingleTask` を設定してください。設定しないと、コールバック URL が呼び出されるたびに、システムによってアクティビティの新しいインスタンス
      が毎回作成されます。

      さらに、`Activity`
      クラスの `OnNewIntent` イベントでインテントを処理する必要があります。
      `ActivityMediator` シングルトンの `Send`
      メソッドを呼び出して渡された URL をそのまま渡し、認証フローを完了するよう Auth0 OIDC Client に通知する必要があります。

      ```charp
      protected override void OnNewIntent(Intent intent)
      {

            base.OnNewIntent(intent);

            ActivityMediator.Instance.Send(intent.DataString);

      }
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      ユーザーの認証が正常に完了すると、この
      Quickstartで先ほど設定したコールバックURLにリダイレクトされます。

      iOSデバイスでコールバックを処理するには、次の手順に従います。

      1. Visual Studioでアプリケーションの `Info.plist` ファイルを開き、&#42;&#42;Advanced &#42;&#42;タブに移動します。
      2. **URL Types** で、&#42;&#42;Add URL Type &#42;&#42;ボタンをクリックします。
      3. &#42;&#42;Identifier &#42;&#42;を Auth0 に設定し、&#42;&#42;URL Schemes &#42;&#42;をアプリケーションの
         `Bundle Identifier` と同じ値にし、&#42;&#42;Role &#42;&#42;を `None` に設定します。

      以下は、URL
      Typeを追加した後の `info.plist` ファイルのXML表現の例です。

      ```xml
      <key>CFBundleURLTypes</key>
      <array>
      <dict>
      <key>CFBundleTypeRole</key>
      <string>None</string>
      <key>CFBundleURLName</key>
      <string>Auth0</string>
      <key>CFBundleURLSchemes</key>
      <array>
      <string>YOUR_BUNDLE_IDENTIFIER</string>
      </array>
      </dict>
      </array>
      ```

      さらに、`AppDelegate` クラスの `OpenUrl` イベントでコールバック URL を処理する必要があります。受け取った
      URL を渡して `ActivityMediator` シングルトンの `Send` メソッドを呼び出し、認証フローを完了できるよう
      Auth0 OIDC Client に通知する必要があります。
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Auth0 アプリケーションと Auth0 SDK の設定が完了したら、次にプロジェクトでログインを設定します。
      これを行うには、SDK の `LoginAsync()` メソッドを使用して、ユーザーを
      Auth0 Universal Login ページにリダイレクトするログインボタンを作成します。

      ```csharp
      var loginResult = await client.LoginAsync();
      ```

      エラーがなければ、`LoginAsync()` から返される `LoginResult` の
      `User`、`IdentityToken`、
      `AccessToken`、`RefreshToken` にアクセスできます。

      <Note>
        ##### チェックポイント

        これで、ユーザー名とパスワードを使ってログインまたはサインアップできるようになります。

        ログインボタンをクリックして、次のことを確認してください。

        * Android または iOS のアプリケーションから Auth0 Universal Login ページにリダイレクトされること。
        * ログインまたはサインアップできること。
        * Auth0 からアプリケーションにリダイレクトされること。
      </Note>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      プロジェクトにログインするユーザーには、ログアウトする手段も必要です。SDK の
      `LogoutAsync()` メソッドを使って、ログアウト ボタンを作成します。ユーザーがログアウトすると、Auth0 のログアウト エンドポイントにリダイレクトされ、
      その後、この Quickstart で先ほど設定したログアウト URL にすぐにリダイレクトされます。

      ```csharp
      await client.LogoutAsync();
      ```

      <Note>
        ##### 確認ポイント

        アプリケーションを起動し、ログアウトボタンをクリックします。次のことを確認してください。

        * Android または iOS のアプリケーションが、Application Settings の Allowed
          Logout URLs の 1 つとして指定したアドレスにリダイレクトされること。
        * アプリケーションからログアウトされていること。
      </Note>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      これでユーザーがログインおよびログアウトできるようになったので、次は認証済みユーザーに関連付けられたプロファイル情報を取得したくなるでしょう。
      たとえば、プロジェクト内にログイン中のユーザーの名前やプロフィール画像を表示したい場合があります。

      Android と iOS 向けの Auth0 SDK では、`LoginResult.User` プロパティを通じてユーザー情報を利用できます。

      ```csharp
      if loginResult.IsError == false {
            var user = loginResult.User
            var name = user.FindFirst(c => c.Type == "name")?.Value
            var email = user.FindFirst(c => c.Type == "email")?.Value
            var picture = user.FindFirst(c => c.Type == "picture")?.Value
      }
      ```
    </Section>
