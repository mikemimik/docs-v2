title: Tenant Access Control List ルールを設定する方法
sidebarTitle: ルールを設定
description: ルールのシグナル、条件、アクション、スコープ、優先順位を定義して、動作を決めます。Tenant ACL の設定と管理には、Management API または Auth0 の SDK、CLI、Terraform プロバイダーを使用します。
---

Tenant ACL のルールは、次の要素で構成されます。

* **Signal**: シグナルは、IP アドレス、地理位置情報、ユーザーエージェントなど、受信したリクエストから取得される識別情報です。

* **Condition**: 条件は、演算子 (`match` など) と値のセット (IP アドレスのリストなど) を組み合わせたものです。

* **Action**: アクションは、条件が満たされた場合にルールが実行する動作を指定するもので、allow、block、redirect などがあります。

* **Scope**: スコープは、そのルールが適用されるエンドポイントの範囲を示します。これには Authentication API、Management API、またはテナント全体が含まれます。

* **Priority**: 優先順位は、そのルールが他のルールに対してどの順序で実行されるかを定義します。

これらの Tenant ACL ルールのプロパティと使用方法の詳細については、[Management API Endpoint Parameter Reference](./reference) を参照してください。

<Note>
  Tenant ACL では、一貫した[ルール評価ロジック](./rule-evaluation)を使用して、特定のリクエストに対してどのアクションを適用するか、または適用しないかを判断します。

  ユースケースに応じたきめ細かなアクセス制御ポリシーを作成するため、優先順位は慎重に設定してください。
</Note>

<div id="create-and-delete-rules">
  ## ルールの作成と削除
</div>

以下のManagement APIエンドポイントを使用して、Tenant ACLルールを作成および削除できます。指定のスコープを持つ[Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production)が必要です。

| 操作     | エンドポイント                                                                        | 必要なスコープ               |
| ------ | ------------------------------------------------------------------------------ | --------------------- |
| ルールを作成 | [アクセス制御リストを作成](/ja/docs/api/management/v2/network-acls/post-network-acls)         | `create:network_acls` |
| ルールを削除 | [アクセス制御リストを削除](/ja/docs/api/management/v2/network-acls/delete-network-acls-by-id) | `delete:network_acls` |

<div id="modify-rules">
  ## ルールを変更する
</div>

以下の Management API エンドポイントを使用して、Tenant ACL ルールを変更できます。指定のスコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) が必要です。

| Action    | Endpoint                                                                        | Required scope        |
| --------- | ------------------------------------------------------------------------------- | --------------------- |
| ルールを更新する  | [アクセス制御リストを部分更新](/ja/docs/api/management/v2/network-acls/patch-network-acls-by-id) | `update:network_acls` |
| ルールを上書きする | [アクセス制御リストを更新](/ja/docs/api/management/v2/network-acls/put-network-acls-by-id)     | `update:network_acls` |

<div id="enable-or-disable-rules">
  ### ルールを有効または無効にする
</div>

<Tabs>
  <Tab title="Management API">
    Management API の [Update access control list](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) エンドポイントを使用すると、ルールを有効または無効にできます。

    `rule` オブジェクトの `active` プロパティを、それぞれ `true` または `false` に設定します。
  </Tab>

  <Tab title="Auth0 Dashboard">
    Auth0 Dashboard を使用してルールを有効または無効にするには、[Auth0 Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) に移動します。

    既存のルール一覧では、各ルールの横にあるボタンを切り替えることで、特定のルールを有効または無効にできます。

    ![Auth0 Dashboard で Tenant ACL ルールを有効または無効にする](/docs/images/cdy7uua7fh8z/29b92e102142d3cb20e69a40e292f6b7/Screenshot_2025-09-03_at_12.34.05.png)

    **Status** 列には、ルールのステータス (**Active** または **Disabled**) が表示されます。
  </Tab>
</Tabs>

<div id="view-rules">
  ## ルールを表示
</div>

<Tabs>
  <Tab title="Management API">
    以下のManagement API endpointを使用して、既存のTenant ACLルールを確認できます。必要なscopeが付与された[Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production)が必要です。

    | Action     | Endpoint                                                                                | Required scope      |
    | ---------- | --------------------------------------------------------------------------------------- | ------------------- |
    | ルールを表示     | [テナントの特定のアクセス制御リストエントリを取得](/ja/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls` |
    | すべてのルールを表示 | [テナントのすべてのアクセス制御リストエントリを取得](/ja/docs/api/management/v2/network-acls/get-network-acls)      | `read:network_acls` |
  </Tab>

  <Tab title="Auth0 Dashboard">
    既存のルール一覧を表示するには、[Auth0 Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) に移動します。

    | Column name | Description                                                                 | Related API response field |
    | ----------- | --------------------------------------------------------------------------- | -------------------------- |
    | 優先順位         | ルールが評価される順序を示す番号                                                            | `priority`                 |
    | 名前          | ルールの目的や機能の説明                                                                | `description`              |
    | Action      | ルールが実行する[action](/ja/docs/secure/tenant-access-control-list/reference#actions) | `action`                   |
    | ステータス       | ルールが有効か無効かを示す値                                                              | `active`                   |
    | 最終更新        | ルールが最後に更新された日時                                                              | `updated_at`               |

    ![Auth0 DashboardでTenant ACLルールの一覧を表示](/docs/images/cdy7uua7fh8z/4093f9a39aba48bdf8d579c134dc4da5/Screenshot_2025-09-03_at_12.33.47.png)
  </Tab>
</Tabs>
