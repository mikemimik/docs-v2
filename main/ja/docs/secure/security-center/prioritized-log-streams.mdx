title: Prioritized Log Streams
---

**Prioritized Log Streams** は、セキュリティイベントを強調表示する Auth0 Dashboard の機能です。攻撃が発生した場合や、その他の理由でユーザーアクティビティが増加した場合でも、Prioritized Log Streams と事前定義されたイベントは有効なまま維持されるため、下流のセキュリティ自動化やプロセスへの影響を抑えられます。

サポートされているサービスを使用して新しい 優先ログストリーム を作成することも、カスタム webhook を設定して、すでに利用しているサービスにストリーミングすることもできます。また、すべてのログをイベント分析サービスにエクスポートできます。Prioritized Log Streams は、料金プランに記載されている log stream の上限数には含まれません。

<div id="configure-a-prioritized-log-stream">
  #### 優先ログストリーム を設定する
</div>

1. 新しい Log Stream を設定するには、[Log Streams ページ](/ja/docs/customize/log-streams)の手順に従ってください。
2. 新しい stream の設定中に、Auth0 Dashboard の **Settings** タブに移動し、**Prioritized Log** チェックボックスをオンにして、以下で説明する定義済みのセキュリティイベントを有効化します。**Save** を選択します。
3. **Stream** ページに移動し、**Prioritized** ラベルを開きます。

<div id="prioritized-logs-event-types">
  #### 優先ログのイベントタイプ
</div>

以下のイベントコードは、Auth0 によりセキュリティ関連イベントとして分類されています。

| イベントコード           | イベント                     |
| ----------------- | ------------------------ |
| `limit_mu`        | IPアドレスのブロック              |
| `limit_sul`       | アカウントのブロック (IPスロットリング)   |
| `limit_wc`        | アカウントのブロック (ブルートフォース保護)  |
| `pwd_leak`        | Login時の漏えいパスワード          |
| `ublkdu`          | ユーザーのloginブロック解除         |
| `signup_pwd_leak` | Signup時の漏えいパスワード         |
| `reset_pwd_leak`  | リセット時の漏えいパスワード           |
| `gd_send_sms`     | MFA SMSの送信               |
