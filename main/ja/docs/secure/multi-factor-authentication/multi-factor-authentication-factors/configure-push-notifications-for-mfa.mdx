title: MFA のプッシュ通知を設定する
---

[iOS](/docs/ja/secure/multi-factor-authentication/auth0-guardian/guardian-for-ios-sdk) および [Android](/docs/ja/secure/multi-factor-authentication/auth0-guardian/guardian-for-android-sdk) 向けの [Guardian](/docs/ja/secure/multi-factor-authentication/auth0-guardian) SDK を使用すると、ベンダー固有のプッシュ通知サービスに対応した独自開発アプリを利用できます。AWS Simple Notification Service (SNS) プラットフォーム用にプッシュ通知を設定することも、Direct to Vendor サービスを使用して Firebase Cloud Messaging (FCM) と Apple Push Notification (APN) の資格情報を Auth0 に直接追加することもできます。

<div id="aws-sns-configuration">
  ## AWS SNS の設定
</div>

プッシュ通知を設定する前に、以下の前提条件を満たしてください。

1. AWS Management Console を使用して [SNS Platform Application を作成](https://console.aws.amazon.com/sns/v3/home?region=us-east-1#/mobile/push-notifications/platform-applications) し、その ARN を控えておきます。
2. Platform Application のエンドポイントを作成する権限を持つ AWS Access Key ID を作成します。Guardian は、登録が正常に完了すると、その一環として適切なデバイストークンを使用した Platform Application のエンドポイントを自動的に作成します。

<div id="configure-push-notifications-with-aws-sns">
  ### AWS SNS を使用してプッシュ通知を設定する
</div>

Guardian からプッシュ通知を受け取るには、まず Guardian のデフォルトの SNS 設定を上書きする必要があります。その後、Android と iOS のプッシュ通知を設定できます。

1. Auth0 Dashboard で [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa) に移動し、**Push via Auth0 Guardian** を選択します。この方法が無効になっている場合は、右上のトグルを選択して有効にします。
2. **Custom** を選択して、追加の設定を表示します。
3. Push Notification Service セクションで、**Amazon SNS** を選択します。

<Frame>![Auth0 Dashboard > Security > Multi-factor Auth > Guardian > Custom > Amazon SNS](/docs/images/cdy7uua7fh8z/6PKSyH99GJ2CalJoG38sfZ/fbe7229dcc6f7d303a582520001b3dc1/Amazon_SNS.png)</Frame>

1. Amazon SNS Configuration セクションで、次の項目を入力します。

   | 項目                    | 説明                    |
   | --------------------- | --------------------- |
   | AWS Access Key ID     | AWS Access Key ID     |
   | AWS Secret Access Key | AWS Secret Access Key |
   | AWS Region            | AWS アプリケーションのリージョン    |
2. **Save** を選択して変更を保存します。

<div id="configure-push-notifications-for-android-using-fcm">
  ## FCM を使用して Android のプッシュ通知を設定する
</div>

<Tabs>
  <Tab title="FCM の資格情報を取得する">
    Auth0 Dashboard で Android のプッシュ通知を設定するには、まず FCM サーバーの資格情報を取得します。詳細については、Google の [Firebase ドキュメント](https://firebase.google.com/docs/admin/setup#initialize-sdk)を参照してください。

    1. Firebase Console に移動し、該当するプロジェクトを選択します。
    2. **Settings** メニューを開き、**Service Accounts** を選択します。
    3. **Generate New Private Key** を選択し、資格情報ファイルをダウンロードします。

    資格情報を取得したら、Auth0 Dashboard でプッシュ通知を設定できます。
  </Tab>

  <Tab title="Auth0 Dashboard を設定する">
    資格情報を取得したら、Auth0 Dashboard から Android のプッシュ通知を設定できます。

    1. [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian) に移動し、**Push Notification using Auth0 Guardian** を選択します。
    2. Push Notification App セクションで **Custom** を選択し、追加の設定を表示します。
    3. Android App Configuration セクションで、以下を設定します。

       | 設定                    | 説明                                                                                      |
       | --------------------- | --------------------------------------------------------------------------------------- |
       | Android App を有効にする    | Auth0 がカスタム アプリケーションにプッシュ通知を送信できるようにします。このオプションが有効になっていない場合、プッシュ通知は送信されません。             |
       | Google Play Store URL | MFA を設定するためにユーザーがダウンロードする必要があるカスタム アプリケーションへのリンクを指定します。                                 |
       | FCM サーバーの資格情報         | [FCM サーバーの資格情報](https://firebase.google.com/docs/admin/setup#initialize-sdk)をアップロードします。 |
    4. **Save** を選択して変更を保存します。
  </Tab>
</Tabs>

<div id="configure-push-notifications-for-ios-using-apns">
  ## APNs を使用して iOS のプッシュ通知を設定する
</div>

<Tabs>
  <Tab title="APN 証明書を取得">
    Auth0 Dashboard で iOS のプッシュ通知を設定するには、まず APN 証明書を取得します。

    1. Apple Developer アカウントで [新しい APN 証明書](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns#2947597) を生成します。

    2. 証明書をローカルにインストールします。

    3. Keychain Access で **Apple Sandbox Push Services: {AppId}** 証明書を探します。

    4. 証明書と関連する秘密鍵を、パスワードなしの P12 ファイル (デフォルト名: `Certificates.p12`) としてエクスポートします。

    5. RC2 ではなく Triple DES を使用して P12 ファイルを生成するには、次のコマンドを実行します。

       ```bash lines
       openssl pkcs12 -in Certificates.p12 -legacy -nocerts -nodes -out pk.pem -passin pass:"" &&
       openssl pkcs12 -in Certificates.p12 -legacy -nokeys -out cert.crt -passin pass:"" &&
       openssl pkcs12 -export -inkey pk.pem -in cert.crt -descert -out Certificate_3des.p12 -passout pass:"" &&
       rm pk.pem cert.crt
       ```

    6. `Certificates.p12` を削除し、Auth0 Dashboard では `Certificate_3des.p12` を APN 証明書として使用します。

    資格情報を取得したら、Auth0 Dashboard でプッシュ通知を設定できます。
  </Tab>

  <Tab title="Auth0 Dashboard を設定">
    資格情報を取得したら、Auth0 Dashboard から iOS のプッシュ通知を設定できます。

    1. [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian) に移動し、**Push Notification using Auth0 Guardian** を選択します。
    2. Push Notification App セクションで **Custom** を選択し、追加の設定項目を表示します。
    3. iOS App Configuration セクションで、次の項目を設定します。

       | 項目                  | 説明                                                                                                                                                                             |
       | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
       | Enable iOS App      | Auth0 がカスタムアプリケーションにプッシュ通知を送信できるようにします。このオプションが有効になっていない場合、プッシュ通知は送信されません。                                                                                                     |
       | Apple App Store URL | MFA を設定するためにユーザーがダウンロードする必要があるカスタムアプリケーションへのリンクを指定します。                                                                                                                         |
       | APNs Bundle ID      | iOS Developer Console で確認できます。                                                                                                                                                 |
       | APNs Certificate    | [P12 証明書](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns) をアップロードします。 |
       | iOS App Environment | Apple Developer アカウントの設定に基づいて指定します。                                                                                                                                            |
    4. **Save** を選択して変更を保存します。
  </Tab>
</Tabs>
