Auth0 には、[Universal Login](/docs/ja/authenticate/login/auth0-universal-login) を使用した組み込みの <Tooltip tip="多要素認証 (MFA): SMS で送信されるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/docs/ja/glossary?term=MFA">MFA</Tooltip> の登録および認証フローが用意されています。ただし、独自のユーザーインターフェイスを作成する場合は、[MFA API](/docs/ja/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api) を使用して実現できます。

認証要素としてメールが有効になっている場合、メールアドレスが確認済みのすべてのユーザーは、それを使用して MFA を完了できます。

<Card title="利用可否は Auth0 のプランによって異なります">
  この機能が利用できるかどうかは、使用しているログイン実装と Auth0 のプランまたはカスタム契約の両方によって異なります。詳しくは、[Pricing](https://auth0.com/pricing) をご覧ください。
</Card>

<div id="prerequisites">
  ## 前提条件
</div>

MFA API を使用する前に、アプリケーションで MFA グラントタイプを有効にする必要があります。[Auth0 Dashboard &gt; アプリケーション &gt; Advanced Settings &gt; Grant Types](https://manage.auth0.com/#/applications) に移動して、**MFA** を選択します。

* [Email](/docs/ja/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa) を認証要素として、Auth0 Dashboard または [Management API](https://auth0.com/docs/api/management/v2#!/Guardian/put_factors_by_name) で設定します。

<div id="enroll-with-email">
  ## メールを登録する
</div>

ユーザーがプライマリーアイデンティティの確認済みメールアドレスに加えて、別のメールアドレスも登録できるようにするには、次の手順を行います。

<div id="get-mfa-token">
  ### MFAトークンを取得する
</div>

登録を開始するタイミングに応じて、MFA API の使用に必要な<Tooltip tip="Access Token: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を表示" href="/docs/ja/glossary?term=access+token">access token</Tooltip>は、さまざまな方法で取得できます。

* 認証時に登録する場合は、[Authenticate With Resource Owner Password Grant and MFA](/docs/ja/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa)を参照してください。
* ユーザーが任意のタイミングで認証要素を登録できるようにする場合は、[Manage MFA Factor Enrollments](/docs/ja/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authenticator-factors-mfa-api)を参照してください。

<div id="enroll-authenticator">
  ### 認証器を登録
</div>

ユーザーの認証器を登録するには、MFA Associate エンドポイントに `POST` リクエストを送信します。このエンドポイントで必要なベアラートークンは、前のステップで取得した MFAトークン です。

次のパラメーターを使用します。

| Parameter              | Value                             |
| ---------------------- | --------------------------------- |
| `authentication_types` | `[oob]`                           |
| `oob_channels`         | `[email]`                         |
| `email`                | `email@address.com`、ユーザーのメールアドレス。 |

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/mfa/associate' \
    --header 'authorization: Bearer MFA_TOKEN' \
    --header 'content-type: application/json' \
    --data '{ "authenticator_types": ["oob"], "oob_channels": ["email"], "email" : "email@address.com" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/mfa/associate");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer MFA_TOKEN");
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{ "authenticator_types": ["oob"], "oob_channels": ["email"], "email" : "email@address.com" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/mfa/associate"

  	payload := strings.NewReader("{ "authenticator_types": ["oob"], "oob_channels": ["email"], "email" : "email@address.com" }")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("authorization", "Bearer MFA_TOKEN")
  	req.Header.Add("content-type", "application/json")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```



  ```python Python

  ```


</AuthCodeGroup>

成功すると、次のようなレスポンスが返されます。

```json lines
{
    "authenticator_type": "oob",
    "binding_method": "prompt",
    "oob_code" : "Fe26..nWE",
    "oob_channel": "email",
    "recovery_codes": [ "N3BGPZZWJ85JLCNPZBDW6QXC" ]
  }
```

`User is already enrolled` エラーが表示された場合、そのユーザーにはすでに MFA の認証要素が登録されています。ユーザーに別の認証要素を関連付ける前に、既存の認証要素でチャレンジを実行する必要があります。

これがユーザーが初めて認証器を関連付ける場合、レスポンスに `recovery_codes` が含まれていることがわかります。リカバリーコードは、第 2 認証要素の認証に使用しているアカウントやデバイスにアクセスできなくなった場合に、ユーザーがアカウントにアクセスするために使用されます。これらのコードは 1 回限り有効で、必要に応じて新しいコードが生成されます。

<div id="confirm-email-enrollment">
  ### メール登録を確認する
</div>

ユーザーには、アプリケーションに入力する6桁のコードが記載されたメールが届きます。

```json lines
{
  "id_token": "eyJ...i",
  "access_token": "eyJ...i",
  "expires_in": 600,
```

この時点で、認証器の関連付けは完全に完了しており、使用できる状態になっています。また、ユーザーの認証トークンも取得済みです。

認証器が確認済みかどうかは、いつでも MFA Authenticators エンドポイントを呼び出して確認できます。認証器が確認済みであれば、`active` に返される値は `true` です。

必要に応じて、ユーザーが受信するメールをカスタマイズできます。詳しくは、[メールテンプレートをカスタマイズする](/docs/ja/customize/email/email-templates) を参照してください。

<div id="customize-mfa">
  ## MFA をカスタマイズする
</div>

<Warning>
  Resource Owner Password Grant、埋め込み、またはリフレッシュトークンのフローで利用するカスタマイズ可能な MFA は、早期アクセス機能です。この機能を使用することで、Okta の [Master Subscription Agreement](https://www.okta.com/legal/) に定められた該当する Free Trial 条項に同意したものとみなされます。Auth0 のリリースステージの詳細については、[Product Release Stages](/docs/ja/troubleshoot/product-lifecycle/product-release-stages) をご覧ください。早期アクセスへの参加を希望する場合は、[Auth0 Support](https://support.auth0.com/) までお問い合わせください。
</Warning>

カスタマイズ可能な MFA を使用すると、ユーザーはアプリケーションでサポートされている任意の認証要素を登録し、選択した認証要素でチャレンジを受けられます。

`oauth/token` endpoint での authentication 中、レスポンスでは `mfa_required` エラーが返されます。このエラーには、MFA API で使用する `mfa_token` と、認証器の一覧を含む `mfa_requirements` パラメーターが含まれます。

```json
{
  "error": "mfa_required",
  "error_description": "Multifactor authentication required",
  "mfa_token": "Fe26...Ha",
  "mfa_requirements": {
    "challenge": [
      { "type": "otp" },
      { "type": "push-notification" },
      { "type": "phone" },
      { "type": "recovery-code" }
      { "type": "email"} //challengeでのみ使用可能
    ]
  }
}
```

`mfa_token` を使用して [`mfa/authenticator`](/docs/ja/api/authentication/muti-factor-authentication/list-authenticators) endpoint を呼び出し、ユーザーが登録済みのすべての認証要素の一覧を取得したうえで、アプリケーションがサポートする同じ種類のものを特定します。チャレンジを発行するには、対応する `authenticator_type` も取得する必要があります:

```json
[
  {
    "type": "recovery-code",
    "id": "recovery-code|dev_qpOkGUOxBpw6R16t",
    "authenticator_type": "recovery-code",
    "active": true
  },
  {
    "type": "otp",
    "id": "totp|dev_6NWz8awwC8brh2dN",
    "authenticator_type": "otp",
    "active": true
  }
]
```

[`request/mfa/challenge`](/docs/ja/api/authentication/muti-factor-authentication/request-mfa-challenge) endpointを呼び出して、MFAチャレンジを必須にします。

Auth0 Actionsを使用すると、MFAフローをさらにカスタマイズできます。詳細については、[Actions トリガー: post-challenge - API オブジェクト](/docs/ja/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object)をご覧ください。
