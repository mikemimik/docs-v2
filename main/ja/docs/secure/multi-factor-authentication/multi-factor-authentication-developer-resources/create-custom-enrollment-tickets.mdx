ユーザーの [多要素認証 (MFA) ](/ja/docs/secure/multi-factor-authentication) の登録をカスタマイズして管理するには、Custom Enrollment Tickets を使用できます。Auth0 の Custom Enrollment Tickets では 1 回限り有効なリンクが生成され、そのリンクを使って、指定した [認証要素](/ja/docs/secure/multi-factor-authentication/multi-factor-authentication-factors) でユーザーが <Tooltip tip="多要素認証 (MFA): SMS で送信されるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証のプロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> に登録できるよう案内できます。

<div id="use-cases">
  ### ユースケース
</div>

Custom Enrollment Tickets は、次の用途で使用できます。

* 新規ユーザーのオンボーディング
* ユーザー設定の構成
* 登録時に顧客が選択する認証要素を指定する
* ユーザーが複数の認証要素を登録できるようにする

<div id="configure-custom-enrollment-tickets">
  ### Custom Enrollment Tickets を設定する
</div>

<Tooltip tip="Management API: 顧客が管理タスクを実行するための製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> の [`/guardian/post_ticket`](https://auth0.com/docs/api/management/v2#!/Guardian/post_ticket) エンドポイントに POST リクエストを送信します。

</AuthCodeGroup>

| パラメーター                       | 必須 | 説明                                                                                        |
| ---------------------------- | -- | ----------------------------------------------------------------------------------------- |
| `allow_multiple_enrollments` | 任意 | すでに登録済みのユーザーが、email を除く追加の認証要素を登録できるようにします。                                               |
| `factor`                     | 任意 | ユーザーに登録させる認証要素を指定します (email を除く) 。`allow_multiple_enrollments` と併用すると、登録用に複数の認証要素を指定できます。 |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  これらのパラメーターは [Universal Login](/ja/docs/authenticate/login/auth0-universal-login) でのみ使用でき、Classic Login またはカスタム MFA ページでは使用できません。
</Callout>

Management API は、`ticket_id` と `ticket_url` を含む登録用チケットを返します。`ticket_id` は内部使用専用です。登録プロセスを開始するには、`send_email` パラメーターを使用して `ticket_url` をユーザーに送信します。チケットの有効期限は 5 日間で、1 回しか使用できません。チケットを受け取ったユーザーが登録できるのも 1 回のみです。

レスポンス例:

```json lines
{
  "ticket_id": "gten_8b2f5e90d2c848dc9230d34f",
  "ticket_url": "https://guardianadnrdoifcmtest0.eu.auth0.com/guardian/enroll?ticket=gten_8b2f5e90d2c848dc9230d34f"
}
```

[複数のカスタムドメイン](/ja/docs/customize/custom-domains/multiple-custom-domains)を有効にする場合は、HTTPヘッダー `auth0-custom-domain` を含める必要があります。詳細については、[複数のカスタムドメイン](/ja/docs/customize/custom-domains/multiple-custom-domains#customize-email-handling-using-the-management-api)を参照してください。

<div id="custom-enrollment-tickets-with-classic-login">
  ### Classic Login での Custom Enrollment Tickets
</div>

[Classic Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) を使用しており、ユーザーが `ticket_url` にアクセスしたときに表示されるページの見た目をカスタマイズする必要がある場合は、MFA ページを編集できます。

1. [Auth0 Dashboard &gt; Branding &gt; Universal Login &gt; Advanced options &gt; Multi-factor Authentication.](https://manage.auth0.com/#/mfa_page) に移動します。
2. **Customize MFA Page** が選択されていない場合は、このオプションをオンにします。
3. `ticket` 変数を更新します。
   例:

```liquid lines
{% if ticket %}
<h4 class="message">Welcome, {{ userData.email }}, enroll your device below</h4>
{% else %}
<h4 class="message">Welcome back, {{ userData.email }}, authenticate below</h4>
{% endif %}
```
