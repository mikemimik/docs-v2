Auth0 では、お客様とユーザーが <Tooltip tip="多要素認証（MFA）: SMS によるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証のプロセスです。" cta="用語集を見る" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> の認証要素を管理できるよう、2 種類の API を提供しています。

<div id="auth0-authentication-api">
  ## Auth0 Authentication API
</div>

[Authentication API の多要素認証エンドポイント](https://auth0.com/docs/api/authentication#multi-factor-authentication)を使用すると、ユーザー自身が MFA の認証要素を管理できるようになります。

この方法では、<Tooltip tip="リソース所有者: 保護されたリソースへのアクセスを許可できるエンティティ（ユーザーやアプリケーションなど）。" cta="用語集を見る" href="/ja/docs/glossary?term=Resource+Owner">リソース所有者</Tooltip>パスワードグラント (ROPG) 認証フローを使用して認証を行います。詳しくは、[Authentication API を使用して認証要素を管理する](/ja/docs/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authenticator-factors-mfa-api)をご覧ください。

<div id="auth0-management-api">
  ## Auth0 Management API
</div>

ユーザーのMFA認証要素を管理するには、[Management API の Users エンドポイント](https://auth0.com/docs/api/management/v2)を使用します。

この方法では、機密アプリケーションを使用して認証を行います。詳しくは、[Management API を使用して認証方法を管理する](/ja/docs/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authentication-methods-with-management-api)をご覧ください。
