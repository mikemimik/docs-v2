title: クラシックログイン向けにMFAをカスタマイズする
---

[Auth0 Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa) セクションで、<Tooltip tip="多要素認証（MFA）: SMSで送信されるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/docs/ja/glossary?term=multi-factor+authentication">多要素認証</Tooltip>を有効にできます。<Tooltip tip="多要素認証（MFA）: SMSで送信されるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> の基本的なブランディングオプションを使って、MFAページをカスタマイズできます。さらに細かくカスタマイズするには、MFAページ用に独自のHTMLを指定できます。独自のHTMLを使用する場合、クラシックログインでは Auth0 MFA Widget が使用されますが、次の制限があります。

* メールを使ったMFAには対応していません。
* ユーザーが複数の認証要素を登録している場合、どの認証要素を使うかは選択できません。MFA Widget により、最も安全性の高い認証要素でログインするよう求められます。
* <Tooltip tip="Universal Login: アプリケーションは、ユーザーの本人確認のために、Auth0 の認可サーバーでホストされる Universal Login にリダイレクトされます。" cta="用語集を表示" href="/docs/ja/glossary?term=Universal+Login">Universal Login</Tooltip> の国際化機能は利用されません。詳しくは、[Universal Login Internationalization](/docs/ja/customize/internationalization-and-localization/universal-login-internationalization) をご覧ください。

<div id="customize-html-for-the-mfa-page">
  ## MFA ページの HTML をカスタマイズする
</div>

MFA ページをカスタマイズするには、[Auth0 Dashboard &gt; Branding &gt; Universal Login &gt; Multi-factor Authentication](https://manage.auth0.com/#/mfa_page) タブを開き、**Customize MFA Page** トグルを有効にします。次に、テキストエディターを使って HTML を編集し、CSS でページのスタイルを設定し、カスタム変数を取得するための JavaScript を変更できます。変更が完了したら、必ず **変更を保存** をクリックしてください。

MFA ページはカスタマイズしなくても動作し、Auth0 は必要に応じて組み込みスクリプトを更新します。ただし、カスタマイズを有効にすると、Auth0 はそのスクリプトを自動更新できなくなるため、スクリプトの更新と保守 (MFA Widget などのバージョン番号の変更を含む) はお客様の責任となります。

クラシックログインの MFA プロセス内のさまざまな箇所に表示されるテキストを上書きする方法について詳しくは、[MFA テーマの言語辞書](/docs/ja/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary) を参照してください。MFA Widget のテーマオプションについて詳しくは、[MFA Widget のテーマオプション](/docs/ja/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options) を参照してください。

<div id="reset-page">
  ## ページをリセット
</div>

以前のデザインに戻したい場合は、次の 2 つの方法があります。

* **Reset to Last** をクリックして、最後に保存したテンプレートに戻します。
* **Reset to Default** をクリックして、Auth0 が提供するデフォルトのテンプレートに戻します。

<div id="render-enrollment-invitations">
  ## 登録招待をレンダリングする
</div>

このページがレンダリングされるシナリオは 2 つあります。ユーザーが登録のためにこのページへ誘導された場合 (たとえば、登録用リンクを含むメールからアクセスした場合) は、`ticket` プロパティを利用できます。そうでない場合は、`requestToken` プロパティを利用できます。

<div id="html-and-liquid-syntax">
  ## HTML と Liquid の構文
</div>

ホストされたページでは、テンプレートに Liquid 構文を使用します。詳しくは、GitHub の [Liquid for Designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) をご覧ください。ページをレンダリングする際に利用できるパラメーターは次のとおりです。

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

MFA Widget で使用するパラメーターの多くは、カスタマイズ画面で提供されるデフォルトのテンプレートに示されているように、Guardian に渡す必要があります。さらに高度なカスタマイズが必要な場合は、[auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example) を使用できます。

詳しくは、[MFA ユーザーページをカスタマイズする](/docs/ja/secure/multi-factor-authentication/customize-mfa) をご覧ください。
