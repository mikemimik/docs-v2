description: コードとしてテナント設定を表現するための Auth0 Terraform Provider について説明します。
title: Auth0 Terraform Provider
---

Auth0 テナント設定を管理するためのツールは Deploy CLI だけではありません。[公式にサポートされている Terraform Provider](https://github.com/auth0/terraform-provider-auth0) も利用できます。[Terraform](https://terraform.io/) は、クラウドリソースの設定をコードとして表現するためのサードパーティ製ツールです。Auth0 を含む幅広いクラウドプロバイダーに対応した、実績のあるプラグインフレームワークを備えています。

Auth0 Terraform Provider は、オープンソースの Terraform 互換ツールである [OpenTofu](https://opentofu.org/) でも利用できます。Auth0 プロバイダーは [OpenTofu Registry](https://search.opentofu.org/provider/auth0/auth0/latest) で確認できます。

Deploy CLI と Terraform Provider はどちらも Auth0 テナント設定の管理に役立ちますが、それぞれに長所と短所があります。

次のような場合は、Auth0 Terraform Provider の利用を検討するとよいでしょう。

* 開発ワークフローですでに Terraform または OpenTofu を活用している
* テナント管理の要件が細かい、または一部の特定リソースだけを対象としている

次のような場合は、Auth0 Terraform Provider の利用を見送ったほうがよいかもしれません。

* 開発ワークフローで Terraform または OpenTofu を使用しておらず、事前に追加の設定が必要になる
* 開発ワークフローが主に、複数のテナントをまとめて管理することを目的としている
* テナントに既存のリソースが多数あり、「インポート」に多くの手間がかかる可能性がある
