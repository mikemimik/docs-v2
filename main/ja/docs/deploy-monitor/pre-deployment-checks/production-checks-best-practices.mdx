description: アプリケーションがAuth0のベストプラクティスに準拠していることを確認するためのチェック
title: '本番環境準備チェック: ベストプラクティス'
---

以下のチェックは自動化できないため、本番環境にデプロイする前に、これらの項目を手動で確認することをお勧めします。

| Check                                                                                | Description                                                                                                                                |
| ------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| [Configuration Parameters](/ja/docs/authenticate/database-connections/custom-db) を外部化する | ルール、Hooks、またはカスタムデータベース接続の開発時には、認証情報、接続文字列、APIキーなど、すべての設定パラメーターを[ハードコードするのではなく外部化](https://manage.auth0.com/#/connections/database)してください。 |
| [Single Sign-on (SSO)](/ja/docs/glossary?term=Single+Sign-on+%28SSO%29) のタイムアウト値を確認する   | デフォルトの[SSO Cookie のタイムアウト値](https://manage.auth0.com/#/account/advanced)を確認し、要件に合っていることを確認します。                                             |
| テナントと管理者                                                                             | すべてのテナントとテナント管理者を確認し、内容が正しいことを確認します。使用されなくなったテナントは廃止してください。また、テナント管理者は必要なユーザーのみに限定されていることを確認します。                                           |
| アプリケーションコード内のClient IDsを確認する                                                         | アプリケーションコード内のClient IDsが、それぞれのAuth0 Applicationの設定と一致していることを確認します。                                                                         |
| Auth0の公開IPを許可リストに追加する                                                                | ルール、Hooks、またはカスタムデータベースを使用して内部サービスやファイアウォールの背後にあるサービスに接続する場合は、Auth0のIPを許可リストに追加してください。これらの項目を設定する際のツールチップで、IPアドレスの一覧を確認できます。               |
| 攻撃対策を確認する                                                                            | [Auth0の攻撃対策機能と設定を確認する](/ja/docs/secure/attack-protection)ことをお勧めします。                                                                           |
