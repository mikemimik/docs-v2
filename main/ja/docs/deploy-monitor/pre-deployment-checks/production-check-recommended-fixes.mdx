---
description: 本番環境へのデプロイ前に Auth0 アプリケーションを改善するための推奨事項
title: '本番運用準備チェック: 重大ではない修正'
---

以下のチェックでは、本番環境への正常なデプロイに向けた推奨事項 (任意) をすべて完了しているかどうかを確認します。

チェック結果の詳細については、[Run Production Checks](/ja/docs/deploy-monitor/pre-deployment-checks/how-to-run-production-checks)を参照してください。

| チェック                                                                                                                 | 説明                                                                                                                                                                                                                       |
| -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ログストリーミングを設定する                                                                                                       | テナントのログエントリを外部ストレージまたは外部プロバイダーにストリーミングしていることを確認します。ログを 30 日以上保持し、分析やアラートをより効果的に行えるようにするため、[ログストリーミングサービスを設定する](/ja/docs/customize/log-streams)ことを推奨します。                                                                     |
| [Custom Domain](/ja/docs/customize/custom-domains) を設定する必要があります                                                         | エンドユーザーに最もシームレスで安全な体験を提供するため、[Universal Login でカスタムドメインを使用](https://manage.auth0.com/#/tenant/custom_domains)してください。                                                                                                     |
| [Custom Error Page](/ja/docs/customize/login-pages/custom-error-pages) を設定する必要があります                                     | アプリケーション固有の情報と企業ブランドを反映した [Custom Error Page を設定](https://manage.auth0.com/#/account)してください。                                                                                                                             |
| Universal Login を有効にする                                                                                               | [Universal Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login) の最新バージョンを使用していることを確認します。アクセシビリティとパフォーマンスの向上、コード不要のカスタマイズ、新機能の活用のため、ログインページを Classic から Universal Login に更新することを推奨します。 |
| [Email Templates](/ja/docs/customize/email/manage-email-flow) を設定する必要があります                                              | アプリケーション固有の情報と企業ブランドを反映した [カスタムメールテンプレートを設定](https://manage.auth0.com/#/emails)してください。                                                                                                                                   |
| [Multi-factor Auth](/ja/docs/secure/multi-factor-authentication) を設定する必要があります                                           | ユーザーを認証する際のセキュリティ対策として、多要素認証の導入を検討することを推奨します。                                                                                                                                                                            |
| [MFA for Tenant Administrators](/ja/docs/get-started/manage-dashboard-access) が有効になっています                                | テナント管理者に対して[多要素認証を有効に](https://manage.auth0.com/#/account/admins)してください。                                                                                                                                                 |
| [Redirect Logout URL](/ja/docs/authenticate/login/logout#set-the-allowed-logout-urls-at-the-account-level) を定義する必要があります | アプリケーションの [Allowed Redirect Logout URLs](https://manage.auth0.com/#/account/advanced) を確認してください。                                                                                                                         |
| アプリケーションでは JSON Web Token (JWT) の署名アルゴリズムとして HS256 ではなく RS256 を使用する必要があります                                           | 各アプリケーションで、JSON Web Token (JWT) の[署名アルゴリズム](/ja/docs/get-started/apis#signing-algorithms)を HS256 ではなく RS256 に設定してください。                                                                                                      |
| [Application Login URI](/ja/docs/get-started/applications/application-settings#application-uris) を設定する                  | すべての対話型アプリケーションに [Application Login URI](/ja/docs/get-started/applications/application-settings#application-uris) を指定してください。                                                                                                |
| [Tenant Allowed Logout URL](/ja/docs/get-started/applications/application-settings#application-uris) を設定する              | 少なくとも 1 つの一般公開されたログアウトルートを作成し、そこを指すように[テナント設定を更新](/ja/docs/get-started/applications/application-settings#application-uris)してください。                                                                                           |
| [Tenant Login URI](/ja/docs/get-started/tenant-settings#login-and-logout) を設定する                                         | デフォルトのログインルートを公開し、そこを指すように[テナント設定を更新](/ja/docs/get-started/tenant-settings#login-and-logout)してください。                                                                                                                         |
