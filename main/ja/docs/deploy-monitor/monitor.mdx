---
description: Auth0 の使用状況を監視および追跡する方法と、テナントログを外部ツールに送信する方法について説明します。
title: Auth0 を監視する
validatedOn: 2026-02-04
---

Auth0 の導入環境や Auth0 のステータス、各種サービスに加え、テナントログのイベントデータも監視できます。

最新のステータス更新を受け取るには、X で [@auth0status](https://x.com/auth0status) をフォローしてください。また、Auth0 の導入環境を監視して、外部サービスやアプリケーションのステータスと可用性を確認しましょう。

| 読む                                                                                                      | 学べること                                                                                  |
| ------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| [Auth0 のステータスを確認する](/ja/docs/deploy-monitor/monitor/check-auth0-status)                                    | ロケールとリージョンごとの Auth0 サービスの可用性を確認し、サービス更新の受信登録をする方法。                                     |
| [外部サービスのステータスを確認する](/ja/docs/deploy-monitor/monitor/check-external-services-status)                        | 外部アイデンティティプロバイダーのサービスを監視する方法。                                                          |
| [アプリケーションを監視する](/ja/docs/deploy-monitor/monitor/monitor-applications)                                      | ユーザーに影響が及ぶ可能性を把握するために、アプリケーションの稼働状況を監視する方法。                                            |
| [Monitor Auth0 Using System Center Operations Manager](/ja/docs/deploy-monitor/monitor/monitor-using-scom) | System Center Operations Manager (SCOM) または合成トランザクションをサポートする任意のツールを使用して Auth0 を監視する方法。 |
| [ログ](/ja/docs/deploy-monitor/logs)                                                                         | ビジネス要件に応じて分析できる Auth0 のテナントイベントログについて。                                                 |
| [Log Streams](/ja/docs/customize/log-streams)                                                              | AWS EventBridge、Azure Event Grid、Datadog などの外部サービスに向けてテナントログのイベントストリームを作成する方法。         |
