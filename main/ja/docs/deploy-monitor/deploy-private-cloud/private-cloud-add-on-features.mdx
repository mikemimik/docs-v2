Private Cloud のデプロイオプションでは、追加のアドオン機能として ジオフェイルオーバー と PCI compliance を利用できます。これらの機能についてご不明な点がある場合は、[Auth0 Sales までお問い合わせください](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales)。

<div id="private-cloud-geo-failover">
  ## Private Cloud のジオフェイルオーバー
</div>

ジオフェイルオーバーは、アクティブ-パッシブ構成のフェイルオーバー方式で、大規模なリージョン障害が発生した際に使用するセカンダリのフェイルオーバーリージョンをお客様が指定できるようにするものです。一般に、同じクラウドプロバイダー内でレイテンシが 100ms 以内のリージョンであれば、フェイルオーバーリージョンとして選択できます。ただし、クラウドプロバイダーによっては、特定のフェイルオーバーリージョンを推奨する場合があります。Auth0 のオンボーディングチームが、お客様のユースケースに最適なリージョンの選定を支援します。

さらに、当社のデータストアの 1 つでは、3 つ目の「タイブレーク」リージョンが必要です。この 3 つ目のリージョンが本番トラフィックを受けることはありませんが、フェイルオーバー時に新しいリーダーを選出するために必要です。

リージョン障害が検出されると、Auth0 がお客様に代わってジオフェイルオーバーを実行します。いったん開始されると、プロセス全体は自動的に進行します。リージョンで問題が発生していないことが確認され次第、Auth0 はプライマリリージョンにフェイルバックします。

  ## PCI Compliance
</div>

PCI Compliance は、必要なお客様向けにアドオンとして提供されています。
