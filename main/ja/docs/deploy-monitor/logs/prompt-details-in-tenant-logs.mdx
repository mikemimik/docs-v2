---

Auth0テナントログには、ユーザーの認証アクションに関するトランザクションの詳細が記録されます。テナントログには、サインイン、ログイン、検証の各プロセスに関する情報が記録されるため、内容を確認できます。ユーザーがどのように認証されたか、どの認証情報を提供したか、トランザクションの所要時間、接続の状態を確認できます。

以下は、テナントログに記録されるデフォルト値です。

| 属性            | 説明                                                |
| ------------- | ------------------------------------------------- |
| `completedAt` | ユーザー操作時間を含むステージの完了時刻                              |
| `elapsedTime` | `completedAt` と `initiatedAt` の差                  |
| `flow`        | ログインエクスペリエンス (例: Universal または Classic) に応じて異なります |
| `initiatedAt` | ステージの開始時刻                                         |
| `name`        | ステージ名                                             |

<div id="login">
  ## login
</div>

login ステージでは、次の値が提供されます。

| Attribute   | Description                                      |
| ----------- | ------------------------------------------------ |
| `flow`      | Universal Login か Classic Login かに応じて、2 つの値を取ります |
| `user_id`   | このステージで認証されたユーザーの Auth0 ID                       |
| `user_name` | このステージで認証されたユーザーの名前                              |

追加フィールドには、次の情報が含まれます。

| Attribute       | Description            |
| --------------- | ---------------------- |
| `connection`    | ユーザーの認証に使用された接続        |
| `connection_id` | ユーザーの認証に使用された接続の ID    |
| `strategy`      | ユーザーの認証に使用された接続のストラテジー |
| `identity`      | 認証されたユーザーの ID 情報       |

login フロー内の追加イベントの値:

| Name                           | Universal Login Experience | Description                                |
| ------------------------------ | -------------------------- | ------------------------------------------ |
| `federated-authenticate`       | Universal                  | 認証に使用されるフェデレーション接続 (ソーシャル、法務、またはエンタープライズ)  |
| `lock-password-authenticate`   | Classic                    | ユーザー名とパスワードで認証されたユーザー                      |
| `oauth1-authenticate`          | Classic                    | oauth1 フェデレーションで認証されたユーザー                  |
| `oauth2-authenticate`          | Classic                    | oauth2 フェデレーションで認証されたユーザー                  |
| `oidc-authenticate`            | Classic                    | oidc フェデレーションで認証されたユーザー                    |
| `prompt-authenticate`          | Universal                  | ユーザー名とパスワードで認証されたユーザー                      |
| `prompt-authenticate-password` | Universal                  | `idfirst` フローでユーザー名とパスワードを使って認証されたユーザー     |
| `prompt-signup`                | Universal                  | ユーザー名とパスワードを使って登録したユーザー                    |
| `prompt-signup-password`       | Universal                  | `idfirst` フローでユーザー名とパスワードを使って登録したユーザー      |
| `saml-authenticate`            | Classic                    | SAML フェデレーションで認証されたユーザー                    |
| `wsfed-authenticate`           | Classic                    | WS-FED フェデレーションで認証されたユーザー                  |

<div id="consent">
  ## 同意
</div>

同意のステージでは、次の値が提供されます。

| 属性          | 説明                                                  |
| ----------- | --------------------------------------------------- |
| `grantInfo` | ユーザーが権限を付与した API の `{"{myApiIdentifier}"}` またはその識別子 |
| `audience`  | トークンの対象                                             |
| `id`        | データベースに保存されたユーザーの許可情報                               |
| `scope`     | ユーザーが承認したスコープ                                       |

<div id="login-email-verification">
  ## login-email-verification
</div>

メールアドレス確認のプロンプトでは、次の値が提供されます。

| 属性                          | 説明                            |
| --------------------------- | ----------------------------- |
| ` login-email-verification` | メールアドレス確認のプロンプトが表示された場合に含まれます |

<div id="redirect">
  ## リダイレクト
</div>

リダイレクトステージでは、次の値を利用できます。

| 属性    | 説明              |
| ----- | --------------- |
| `URL` | ユーザーのリダイレクト先URL |

<div id="mfa">
  ## mfa
</div>

<Tooltip tip="多要素認証（MFA）: SMS で送信されるコードなど、ユーザー名とパスワードに加えて別の要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=Multi-factor+authentication">多要素認証</Tooltip>には、次の値があります。

| 属性              | 説明                                                                                               |
| --------------- | ------------------------------------------------------------------------------------------------ |
| `flow`          | Classic experience (`mfa`) で使用する場合と、Universal experience (`universal-mfa`) で使用する場合とで、2 つの値を取ります。 |
| `performed_acr` | `id_token` に含まれる `acr`。                                                                          |
| `performed_amr` | `id_token` に含まれる `amr`。                                                                          |
| `provider`      | MFA の実行に使用されるプロバイダー。                                                                             |
