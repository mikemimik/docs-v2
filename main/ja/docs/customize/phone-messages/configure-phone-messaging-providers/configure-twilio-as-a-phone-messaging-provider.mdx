description: Auth0 Dashboard で Twilio を Phone Messaging Provider として設定する方法について説明します。
title: Twilio を Phone Messaging Provider として設定する
---

<div id="prerequisites">
  ## 前提条件
</div>

有効なSMSおよび/または電話の送信オプションが設定された [Twilio](https://twilio.com/) アカウントが必要です。

<div id="configure-twilio">
  ## Twilio を設定する
</div>

1. [Twilio](https://www.twilio.com/login) アカウントにログインします。
2. [Twilio Console](https://console.twilio.com/) に移動します。Auth0 Dashboard で Twilio を設定する際には、Twilio SID と Twilio Auth Token が必要です。
3. Auth0 の [Auth0 Dashboard &gt; Branding &gt; Phone Provider](https://manage.auth0.com/#/branding/phone/provider) に移動します。**Phone Provider** として **Twilio** を選択します。
4. **Delivery Method** を選択します。顧客向けの配信方法として、**Text**、**Voice**、またはその両方を選択できます。
5. **Twilio Provider Settings** で、**SMS Source** を選択します。

   1. **Use From** を選択した場合は、顧客が電話メッセージを受信する際に使用するデフォルトの電話番号を入力します。
   2. [Messaging Service](https://www.twilio.com/docs/messaging/services) を選択した場合は、Twilio Messaging Service SID を入力します。この値は Twilio Console で確認できます。
   3. Twilio Console から Twilio SID と Twilio Auth Token を入力し、**Save** をクリックします。

<div id="test-phone">
  ## 電話をテスト
</div>

**試す**ボタンをクリックして、テスト用の電話メッセージを送信します。Twilio を外部の電話プロバイダーとして正常に設定している場合、Auth0 は選択した配信方法を使用して確認用の電話メッセージを送信します。数分経っても電話メッセージが届かない場合は、失敗がないか [Auth0 のログ](https://manage.auth0.com/#/logs) を確認してください。

Twilio の [Messaging Activity](https://console.twilio.com/us1/develop/sms/overview) ページには、ユーザーに送信されたすべての SMS の配信状況に関する情報が表示されます。
