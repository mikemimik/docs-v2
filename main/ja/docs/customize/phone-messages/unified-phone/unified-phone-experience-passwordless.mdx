---

<Card title="開始前に">
  Unified Phone Experienceに移行するには、次のいずれかを満たしている必要があります。

  * テナントで多要素認証 (MFA) の SMS、音声、または SMS+音声 の通知が有効化され、設定済みであること、または
  * テナントで パスワードレス SMS が有効化され、設定済みであること

  MFA とパスワードレスのどちらも設定していない場合は、[Configure Auth0&#39;s Unified Phone Experience](/ja/docs/customize/phone-messages/unified-phone/configure-unified-phone)を参照し、最初から設定してください。

  MFA とパスワードレスの連携で異なるプロバイダーを使用している場合は、[Custom Phone Provider](/ja/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider)を使用する必要があります。

  Phone as ID を利用している場合は、すでに Auth0 の Unified Phone Experience を使用しているため、追加の対応は不要です。
</Card>

Unified Phone Experience では、テナント内のすべての電話プロバイダー設定を 1 か所で一元管理できます。さまざまな電話の認証要素ごとに、電話プロバイダーを何度も設定する必要はありません。<Tooltip tip="Passwordless: 最初の認証要素としてパスワードに依存しない認証方式です。" cta="用語集を見る" href="/ja/docs/glossary?term=Passwordless">パスワードレス</Tooltip>の電話認証要素について詳しくは、[パスワードレス Authentication with SMS](/ja/docs/authenticate/passwordless/authentication-methods/sms-otp)を参照してください。

すでにパスワードレス用の電話プロバイダーを設定している場合は、その設定を Unified Phone Experience に移行し、テナントレベルで適用できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 は、テナントレベルのプロバイダーで電話メッセージが正常に送信されたかどうかを、エンドユーザーに通知しません。問題を切り分けるには、[テナントログ](/ja/docs/deploy-monitor/logs)を確認してください。
</Callout>

<div id="update-your-phone-provider">
  ## 電話プロバイダーを更新する
</div>

Unified Phone Experience を使用するには、次の手順を行う必要があります。

* 現在の設定を確認する
* プロバイダーを設定する:

  * Twilio を設定する
  * カスタム電話プロバイダーを設定する
* 配信方法を更新する

<div id="verify-current-settings">
  ### 現在の設定を確認する
</div>

1. 現在のパスワードレス設定を確認するには、[**Auth0 Dashboard &gt; Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless) に移動します。
2. **Phone Message** を選択します。
3. Unified Phone Experience を利用できる場合は、テナントレベルの Provider を設定するよう促すバナーが表示されます。

<Frame>![Dashboard > Authentication > Passwordless > SMS](/docs/images/cdy7uua7fh8z/7piF5GX9dZL0EXC8SRtMzE/41737a34205541cee46cdbf364401bbc/2025-02-12_15-56-26.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  電話プロバイダーの設定にレガシーエクスペリエンスを使用する必要があり、テナントレベルのプロバイダーを使用するためのトグルが表示されない場合は、[Auth0 Support](/ja/docs/troubleshoot/customer-support) にお問い合わせください。
</Callout>

<div id="configure-twilio-as-a-tenant-level-provider">
  ### Twilio をテナントレベルのプロバイダーとして設定する
</div>

パスワードレスの電話プロバイダーとして Twilio を使用している場合は、Unified Phone Experience に Twilio SID と AuthToken を追加する必要があります。Twilio SID は既存の設定からコピーできますが、AuthToken の値は Twilio から取得する必要があります。

1. [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider) に移動します。
2. プロバイダーを選択して設定します。

   <Frame>![Branding > Phone Provider > Twilio](/docs/images/cdy7uua7fh8z/6TESTIdmhUL0LEitACadIK/fcdc34e18a6b66b47370f6d754c085b5/2025-02-21_12-49-53.png)</Frame>

   1. 電話プロバイダーとして Twilio を選択します。
   2. 配信方法を選択します。
   3. Twilio の設定を入力します。
3. 変更を **Save** します。
4. 本番環境で使用する前に、**Send Test Message** を選択してテナントレベルのプロバイダーをテストします。
5. テナントレベルのプロバイダーを有効にします。

   1. [**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless) に移動します。
   2. **Use Tenant-level Messaging Provider** を選択します

      <Warning>
        テナントにトグルが表示されない場合は、すでに Unified Phone Experience を使用しています。
      </Warning>
6. 変更を **Save** します。

<div id="create-a-new-custom-phone-provider-action-for-passwordless">
  ### パスワードレス用の新しい custom-phone-provider Action を作成する
</div>

Unified Phone Experience では、パスワードレス SMS 用に設定した [カスタム SMS ゲートウェイ](/ja/docs/authenticate/passwordless/authentication-methods/use-sms-gateway-passwordless) は使用されません。<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用して、Auth0 Actions を使ったカスタム電話プロバイダーを設定できます。詳しくは、[カスタム電話プロバイダーを設定する](/ja/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) をご覧ください。

<div id="configure-passwordless-custom-phone-provider">
  ##### パスワードレスの custom-phone-provider を設定する
</div>

phone provider Action を使用して Custom SMS Gateway の電話プロバイダー設定を行うには、次の手順に従います。

1. [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider) に移動します。
2. プロバイダーを選択します。

   1. **Custom** を選択します。
   2. **Provider Configuration** で、パスワードレス通知をカスタム SMS ゲートウェイに転送する新しい `custom-phone-provider` Action を追加します。
3. 変更を **Save** します。
4. **Send Test Message** を選択し、本番環境で使用する前にテナントレベルのプロバイダーをテストします。
5. **Use Tenant-level Messaging Provider** を有効にするには、[**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless) に移動します。
6. 変更を **Save** します。

<div id="custom-sms-gateway-to-custom-phone-provider-example">
  ##### Custom SMS ゲートウェイから custom-phone-provider を使用する例
</div>

```javascript lines expandable
const request = require('<favorite_http_client>');
/**
* 電話通知の送信時に実行されるハンドラー
* 
@param
 {Event} event - ユーザーおよびログイン時のコンテキストに関する詳細情報。
* 
@param
 {CustomPhoneProviderAPI} api - 電話通知の送信動作を変更するためのメソッドとユーティリティ。
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {
  const notificationType = event.notification.message_type;
  switch (notificationType) {
    case '<otp_verify>': {
      const req = {
        url: event.secrets.GATEWAY_URL,
        method: 'POST',
        headers: {
          'User-Agent': 'Auth0'
        },
        body: {
          recipient: event.notification.recipient,
          body: event.notification.as_text, // または event.notification.as_voice
          sender: event.notification.from, // TODO: sender が含まれていません。追加するかどうか確認が必要です。
          req: event.request || {}, // 有効でない場合は undefined になります
        }
      }
      const response = await request(req);
      console.log(response.statusCode);
      return
    }
    default: {
      // 他の通知を通常どおり処理する
      return;
    }
  }
};
```

<div id="update-delivery-methods">
  ### 配信方法を更新
</div>

| 電話認証要素 | 現在のテナントレベルの配信方法 | 変更後のテナントレベルの配信方法 | ユーザーの対応                                                                 |
| ------ | --------------- | ---------------- | ----------------------------------------------------------------------- |
| SMS    | SMS             | SMS + Voice      | なし                                                                      |
| SMS    | SMS             | Voice            | ユーザーは、パスワードレスの電話認証要素がテナントレベルのプロバイダーで有効になっている配信方法のサブセットとなるように更新する必要があります |
| SMS    | SMS + Voice     | Voice            | ユーザーは、パスワードレスの電話認証要素がテナントレベルのプロバイダーで有効になっている配信方法のサブセットとなるように更新する必要があります |
| SMS    | SMS + Voice     | SMS              | なし                                                                      |

<div id="disable-phone-provider">
  ## 電話プロバイダー を無効にする
</div>

Unified Phone Experience を設定した後で無効にすると、パスワードレスを含む 電話プロバイダー との既存の連携が機能しなくなる可能性があります。
