Auth0 では、ACUL で利用できる複数の SDK を提供しています。

<div id="acul-sdks">
  ## ACUL SDK
</div>

* [Auth0 ACUL React SDK](/docs/ja/libraries/acul/react-sdk)
  React アプリケーション内での連携を簡単にする React Hooks を提供します。

  * [Auth0 ACUL JS SDK](/docs/ja/libraries/acul/js-sdk)
    中核となる JavaScript ライブラリ (TypeScript で記述) で、あらゆるフロントエンドフレームワークやプレーンな JavaScript での使用に適しています。

<div id="acul-sample-applications">
  ## ACUL サンプルアプリケーション
</div>

[ACUL サンプルアプリケーション](https://github.com/auth0-samples/auth0-acul-samples) には、完全なビルドパイプライン、デプロイ ワークフロー、一般的な Universal Login 画面の実装が含まれています。これらのサンプルアプリケーションは、画面の開発に役立ちます。サンプルは、ACUL React SDK または JS SDK を使用する React 向けに提供されています。

`auth0 acul init` コマンドを使用すると、ACUL React SDK または JS SDK を使った React のスタータープロジェクトを初期化できます。詳しくは、GitHub の [Auth0 CLI](https://github.com/auth0/auth0-cli) を参照してください。

<div id="acul-resources">
  ## ACUL リソース
</div>

Auth0 では、Universal Login エクスペリエンスをカスタマイズするための各種リソースを提供しています。

<div id="auth0-management-api">
  ### Auth0 Management API
</div>

Management API を使用すると、個々の Universal Login 画面を更新したり、複数の画面をまとめて設定したりできます。Auth0 Management API を使用するには、[access token](/docs/ja/secure/tokens/access-tokens/management-api-access-tokens) が必要です。

**個別の画面設定**

* [`GET /v2/prompts/{prompt}/screen/{screen}/rendering` エンドポイント](/docs/ja/api/management/v2/prompts/get-rendering) を使用すると、1 つの画面の設定を取得できます。

  * [`PATCH /v2/prompts/{prompt}/screen/{screen}/rendering` エンドポイント](/docs/ja/api/management/v2/prompts/patch-rendering) を使用すると、1 つの画面に対して ACUL を設定できます。

**画面の一括設定**

* [`GET /v2/prompts/rendering` エンドポイント](/docs/ja/api/management/v2/prompts/get-all-rendering) を使用すると、すべての画面の設定を取得できます。

  * [`PATCH /v2/prompts/rendering` エンドポイント](/docs/ja/api/management/v2/prompts/patch-bulk-rendering) を使用すると、画面を一括更新できます。

<div id="auth0-developer-tools">
  ### Auth0 の開発ツール
</div>

Auth0 には、ACUL の設定を含む tenant 管理を簡素化するための各種ツールが用意されています。

* **Auth0 CLI**<p />
  `acul` コマンドを使用して、プロジェクトのひな形を作成し、ローカルで開発し、画面設定をデプロイできます。詳しくは、GitHub の [Auth0 CLI](https://github.com/auth0/auth0-cli) をご覧ください。

  * **Auth0 Deploy CLI**<p />
    JSON ファイルで定義した ACUL のレンダリング設定を含め、tenant 全体の設定をコードとして管理できます。詳しくは、GitHub の [Auth0 Deploy CLI](https://github.com/auth0/auth0-deploy-cli) をご覧ください。

  * **Auth0 Terraform provider**<p />
    ACUL の画面設定を定義できます。詳しくは、Terraform の [`promt_screen_renderer resource`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) をご覧ください。
