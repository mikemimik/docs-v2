イベントは、Auth0 またはサードパーティの<Tooltip tip="アイデンティティプロバイダー（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+providers">アイデンティティプロバイダー</Tooltip> (IdPs) 内で発生する変更を、外部アプリやサードパーティサービスとの間で同期、関連付け、またはオーケストレーションするための API ベースの方法を Auth0 のお客様に提供します。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  イベントストリームを有効にすると、お客様はそのような情報の管理主体となり、どの情報をストリーミングするかやその情報をどのように利用するかを含め、Auth0 Platform からエクスポートされるデータについて責任を負います。
</Callout>

<div id="use-cases">
  ## ユースケース
</div>

イベントは、Auth0 テナント内で発生した特定のアクションに関するリアルタイム通知です。現時点では主にユーザーへの変更が対象ですが、今後はそれ以外の変更も含まれる予定です。これらのイベントを監視するイベントストリームを作成すると、Auth0 内で発生した非同期の変更 (つまり、エンティティやフローに対してすでに完了した変更) をオーケストレーションし、外部で処理するためにさまざまな送信先のいずれかへ転送できます。イベントは、次のようなさまざまなユースケースをサポートします。

* **[システム間でデータを同期する](/docs/ja/customize/events/sync-data-across-systems)** — 外部データベース、データウェアハウス、または検索インデックスを、Auth0 でのアイデンティティの変更と同期した状態に保ちます。
* **[外部システムとアイデンティティ変更を関連付ける](/docs/ja/customize/events/correlate-identity-changes)** — アイデンティティの変更に応じて、CRM プラットフォーム、請求サービス、または SaaS ライセンス管理ツールに更新を反映します。
* **[業務ワークフローをオーケストレーションする](/docs/ja/customize/events/orchestrate-business-workflows)** — 1 つのワークフロー内で、CRM や通知チャネルなど複数の下流システムにアイデンティティの変更を振り分けます。
