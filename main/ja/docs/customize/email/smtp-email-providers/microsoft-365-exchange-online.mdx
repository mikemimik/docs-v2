<div id="prerequisites">
  ## 前提条件
</div>

* 有効なサブスクリプションがある [Microsoft 365](https://www.microsoft.com/en-us/microsoft-365) アカウントが必要です。

* お使いの Microsoft 365 サブスクリプションには、Exchange Online 経由でメールを送信できるライセンスが必要です。Exchange Online で Microsoft Graph API を使用するには、少なくとも Microsoft Business Basic プランの有料サブスクリプションが必要です。

* 特定の [Auth0 IP アドレス](/ja/docs/secure/security-guidance/data-security/allowlist) からの受信接続を許可するように、プロバイダーを設定する必要があります。

<div id="configure-microsoft-365-exchange-online">
  ## Microsoft 365 Exchange Online を設定する
</div>

1. Microsoft 365 アカウントを使用して、[Microsoft Azure](https://azure.microsoft.com/) アカウントにログインします。
2. 現在 [Microsoft Entra ID アプリケーション](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application) リソースをお持ちでない場合は、登録します。
3. Active Directory のアプリケーション リソースに移動します。**Services &gt; App registrations** を選択し、メール送信に使用するアプリケーションを選択します。
4. **Manage &gt; API permissions** に移動して、必要なアプリケーション権限を設定します。
5. **Add a permission &gt; Microsoft Graph &gt; Application permissions** セレクターを選択し、`Mail.Send` [permission](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#api-permissions) を追加します。

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     **Application permissions** を設定する前に、管理者の同意が必要です。
   </Callout>

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     既定では、`Mail.Send` により、テナント内の任意のユーザーとしてメールを送信する権限が付与されます。特定のメールボックスのみにアクセスを制限するには、[Exchange Online の RBAC for Applications](https://learn.microsoft.com/en-us/exchange/permissions-exo/application-rbac) を使用して、その権限を特定のサービス プリンシパルに限定します。
   </Callout>
6. 必要なプロバイダー設定を取得します。

   1. **Application Overview** に移動し、アプリケーションの [Client ID と Tenant ID](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#application-id-client-id) を取得します。
   2. **Manage &gt; Certificates &amp; secrets** に移動し、アプリケーション [secret](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#certificates--secrets) を作成するための **Value** を取得します。
   3. アプリケーション経由でメールを送信する組織の **User Email** を取得します。**Users &gt; Active users** に移動し、承認済みユーザーを選択して、そのプロパティから **User Email** を取得します。
7. Auth0 [Dashboard &gt; Branding &gt; メールプロバイダー](https://manage.auth0.com/#/branding/email_provider) に移動します。
8. **Use my own メールプロバイダー** トグルを有効化します。
9. **Microsoft 365** のロゴをクリックします。
10. **From** メールアドレスを入力します。次に、手順 6 の **Tenant ID**、**Client ID**、**Client Secret** を入力し、**Save** をクリックします。

<div id="send-a-test-email">
  ## テストメールを送信する
</div>

**Send Test Email** ボタンを使用して、テストメールを送信します。設定が正しければ、確認メールが届きます。数分待ってもメールが届かない場合は、エラーコードについて [Auth0ログ](https://manage.auth0.com/#/logs) を確認してください。

設定が完了すると、Microsoft Exchange の [Email Usage](https://admin.microsoft.com/#/reportsUsage/EmailActivity) ページに、ユーザーに送信されたすべてのメールの配信状況に関するインサイトが表示されます。
