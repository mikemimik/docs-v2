<div id="prerequisites">
  ## 前提条件
</div>

* [Resend](https://resend.com) のアカウントが必要です。

* メールを送信するには、事前に Resend で少なくとも 1 つのドメインを検証しておく必要があります。詳しくは、Resend の[ドメイン検証に関するドキュメント](https://resend.com/docs/dashboard/domains/introduction)を参照してください。

* 特定の [Auth0 IP addresses](/ja/docs/secure/security-guidance/data-security/allowlist) からの接続を許可するように、プロバイダーを設定する必要があります。

<div id="configure-resend">
  ## Resend を設定する
</div>

1. [Resend](https://resend.com) アカウントにログインします。
2. **API Keys** に移動し、**Create API Key** をクリックします。
3. キーの名前を入力し、適切な権限レベルとドメインを選択して、**Add** をクリックします。

<Frame>
  <img src="/docs/images/customize/emails/providers/resend-create-api-key.png" />
</Frame>

4. API キーの値をコピーします。後で再度確認することはできません。
5. Auth0 [Dashboard &gt; Branding &gt; メールプロバイダー](https://manage.auth0.com/#/branding/email_provider) に移動します。
6. **Use my own email provider** トグルを有効化します。
7. **Resend** ロゴを選択します。
8. **From** メールアドレスを入力し、コピーした Resend の **API Key** を貼り付けて、**Save** をクリックします。

**From** メールアドレスには、Resend で確認済みのドメインを使用する必要があります。[メールによるパスワードレス認証](/ja/docs/authenticate/passwordless#email) を使用している場合、<Tooltip tip="Passwordless: 最初の認証要素としてパスワードに依存しない認証形式。" cta="用語集を見る" href="/ja/docs/glossary?term=passwordless">パスワードレス</Tooltip>接続の **From** メールアドレスの値は、Resend で使用している **From** メールアドレスの値と一致している必要があります。

<div id="send-a-test-email">
  ## テストメールを送信する
</div>

**Send Test Email** ボタンをクリックして、テストメールを送信します。設定が正しければ、確認メールが届きます。数分経ってもメールが届かない場合は、[Auth0 のログ](https://manage.auth0.com/#/logs) でエラーコードを確認してください。

設定が完了すると、Resend の [Emails](https://resend.com/emails) ページに、ユーザーへ送信されたすべてのメールの配信状況が表示されます。
