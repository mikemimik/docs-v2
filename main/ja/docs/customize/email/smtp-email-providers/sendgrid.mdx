<div id="prerequisites">
  ## 前提条件
</div>

* [SendGrid](https://sendgrid.com) のアカウントが必要です。Microsoft Azureのサブスクリプションがある場合は、Azure マーケットプレイスで無料のアカウントを取得できます。

* 特定の[Auth0 IPアドレス](/docs/ja/secure/security-guidance/data-security/allowlist)からの受信接続を許可するよう、プロバイダーを設定する必要があります。

<div id="configure-sendgrid">
  ## SendGrid を設定する
</div>

1. [SendGrid](https://sendgrid.com) アカウントにログインします。
2. SendGrid の **Settings &gt; API Keys** に移動し、**Create API Key** をクリックします。
3. キーの名前を入力し、**Restricted Access** を選択します。
4. **Access Details** で、**Mail Send** をクリックして展開します。
5. **Mail Send** の権限を **Full Access** に設定します。
6. **Save** をクリックし、キーの値をコピーします。
7. Auth0 の [Auth0 Dashboard &gt; Branding &gt; メールプロバイダー](https://manage.auth0.com/#/branding/email_provider) に移動します。
8. **Use my own email provider** トグルを有効化します。
9. **SendGrid** のロゴをクリックします。
10. **From** のメールアドレスを入力し、コピーした SendGrid の **API Key** を貼り付けて、**Save** をクリックします。

[メールを使用するパスワードレス認証](/docs/ja/authenticate/passwordless#email) を使用している場合、<Tooltip tip="パスワードレス: 第1認証要素としてパスワードに依存しない認証形式。" cta="用語集を見る" href="/docs/ja/glossary?term=passwordless">パスワードレス</Tooltip> 接続の **From** メールアドレスの値は、SendGrid で使用する **From** メールアドレスの値と一致している必要があります。

<div id="send-a-test-email">
  ## テストメールを送信
</div>

**テストメールを送信**ボタンをクリックして、テストメールを送信します。設定が正しければ、確認メールが届きます。数分経ってもメールが届かない場合は、エラーコードが記録されていないか [Auth0 logs](https://manage.auth0.com/#/logs) を確認してください。

設定が完了すると、SendGrid の [Email Activity](https://sendgrid.com/logs/index) ページに、ユーザーに送信されたすべてのメールの配信状況が表示されます。
