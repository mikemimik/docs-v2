title: Authorization Extension の API アクセス を有効にする
---

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 では、ロールベースのアクセス制御 (RBAC) を実装する方法が 2 つ用意されており、API 独自の内部アクセス制御システムの代わりに、またはそれと組み合わせて使用できます。

  * [Authorization Core](/docs/ja/manage-users/access-control/rbac)
  * [Authorization Extension](/docs/ja/customize/extensions/authorization-extension)

  Authorization Core の機能セットは Authorization Extension の機能に対応しているほか、パフォーマンスとスケーラビリティも向上しており、Authorization Extension より柔軟なロールベースのアクセス制御システムを提供します。

  現時点では、どちらも RBAC の主要な機能を実装しており、API に対して定義されたカスタムスコープを、権限としてユーザーに割り当てられているものだけに制限できます。
</Callout>

設定とセットアップが完了すると、拡張機能 にはユーザーに加えて、グループ、ロール、権限が含まれるようになります。拡張機能 の API アクセス を有効にすると、プロビジョニングを自動化し、ユーザーの認可コンテキストをリアルタイムで照会できます。

<div id="enable-api-access">
  ## API アクセスを有効にする
</div>

1. [Auth0 Dashboard &gt; Extensions &gt; Auth0 Authorization](https://manage.auth0.com/#/extensions) に移動します。
2. **API** セクションを開くには、**Authorization Dashboard** の右上にある Auth0 テナント名を選択し、**API** を選択します。

   <Frame>![Dashboard - Extensions - Authorization Extensions Dashboard - API を選択](/docs/images/cdy7uua7fh8z/6cF78dwvbK7Sc0JbuxJF85/fbd895fb6f7338559628ea71b3003f52/click-configuration_select-api.png)</Frame>
3. **設定** ページで、**API Access** スイッチを有効にします。

   <Frame>![Dashboard - Extensions - Authorization Extensions Dashboard - API Access を有効にする](/docs/images/cdy7uua7fh8z/5toxhsSik9EWMnQoK5wTjH/fc40d80a32461e9216f5c69a89e031ec/enable-api-access.png)</Frame>
4. API Access を有効にすると、API が発行する token の一部のパラメーターを表示したり、拡張機能内で制御したりできるようになります。token の**有効期限**を制御できるほか、API にアクセスするための token の **audience**、**発行者**、**URL** も確認できます。

   <Frame>![Dashboard - Extensions - Authorization Extensions Dashboard - API Access が有効](/docs/images/cdy7uua7fh8z/3WfHXaEyi8FvD8pTDlYQpt/cbe92c5b5f4baf9c35a8c686ff4f511d/api-access-enabled.png)</Frame>

<div id="access-extension-api">
  ## 拡張機能のAPIにアクセスする
</div>

拡張機能のAPI アクセスを有効にすると、Auth0 は [Auth0 Dashboard](https://manage.auth0.com/#/apis) で使用できる API を自動的に作成します。API にアクセスするには、Machine to Machine Application を作成する必要があります。これは、API 自体とやり取りするエンティティです。

<div id="create-applications">
  ### アプリケーションを作成する
</div>

1. [Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications) に移動し、**Create Application** を選択します。新しいアプリケーションに名前を付け、タイプとして **Machine to Machine Application** を選択します。続けて **Create** を選択します。
   アプリケーションの **Quick Start** ページにリダイレクトされます。ここでは、そのアプリケーションで使用する API に応じて対話型ドキュメントをカスタマイズできます。Auth0 が拡張機能用に作成した API (**auth0-authorization-extension-api** またはそれに近い名前) を選択してください。
   API とアプリケーションを組み合わせて使用するのは今回が初めてなので、「This application is not authorized for this API.」というメッセージが表示されます。API でこのアプリケーションを使えるようにするには、**Navigate to the API and Authorize** を選択します。

   <Frame>![Dashboard - Authorization Extensions - Client Quick Start](/docs/images/cdy7uua7fh8z/UbMM1C6a1GjMIhNlk8cCD/462b9dcd29e5d7f5bbd261490e6a46b0/Dashboard_-_Authorization_Extensions_-_Client_Quick_Start.png)</Frame>
2. API で使用できる Machine to Machine Applications の一覧が表示されます。先ほど作成したアプリケーションの横にあるスイッチを有効にして、認可します。

   <Frame>![Dashboard - Authorization Extension - Clients for API](/docs/images/cdy7uua7fh8z/4WFJ3BhCf2kZjbRWhQ0sCp/38aeb0ca9360fd75bd7d54f897ac1735/Dashboard_-_Authorization_Extension_-_Clients_for_API.png)</Frame>

   アプリケーションを認可すると、**Grant ID** が表示されます。また、そのアプリケーションに付与する **Scopes** も選択できます。付与するスコープは、アクセスするエンドポイントによって異なります。
3. スコープを変更した場合は、保存するために **Update** を選択します。

   <Frame>![Dashboard - Authorization Extension Dashboard - Client Scopes](/docs/images/cdy7uua7fh8z/2RvMFmLusnUOmBp1uXPh1/a78add868b094d37cec63365f745f196/extensions-auth-api-scopes.png)</Frame>

<div id="get-access-token">
  ### アクセストークンを取得する
</div>

API にアクセスするには、[適切なトークンをリクエストして取得する](/docs/ja/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow)必要があります。

<div id="call-the-api">
  ### API を呼び出す
</div>

API は次の方法で呼び出せます。

* HTML リクエスト
* cURL コマンド

また、[Authorization Extension API Explorer](https://auth0.com/docs/api/authorization-extension) では、エンドポイントの詳細情報に加え、上記の方法を使って各エンドポイントを呼び出すサンプルも確認できます。

さらに、**API Explorer** では、API 経由で 拡張機能 へのアクセスを有効にした後、コマンドラインから実行できるすべての操作を確認できます。

API ドキュメントを表示するには、**Explorer** ページを選択してください。

<Frame>![Authorization Extension API Explorer](/docs/images/cdy7uua7fh8z/5mwKE5muQHYbXKqYZvWOwR/aa0ecec3e5cb803a105d857f24c8b6b7/api-explorer.png)</Frame>
