<Tooltip tip="Single Sign-On（SSO）: ユーザーが1つのアプリケーションにログインすると、ほかのアプリケーションにも自動的にログインできるようにするサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=Single+Sign-on">シングルサインオン</Tooltip> (SSO) Dashboard は、多くの組織に共通する課題を解決するために設計された Web アプリケーションです。組織の規模を問わず、会計、人事、開発、サポートなど、さまざまな業務を担う多様なアプリケーションが利用されています。そうしたすべてのアプリケーションのユーザー名、パスワード、ログイン URL を覚えておくのは大変です。この拡張機能を使用すると、複数のエンタープライズアプリケーションでユーザーの SSO ログインを有効にし、認証エクスペリエンスを簡素化できます。また、ユーザーが SSO ログインで認証できるクラウドサービスの一覧を作成することもできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザーは、適切な権限 (グループ/ロール) を持っていない限り、ダッシュボード経由かどうかにかかわらず、サービスプロバイダーにアクセスできてはなりません。理想的には、アクセス権が付与されていないサービスプロバイダーは、ダッシュボード上に表示されないようにする必要があります。
</Callout>

SSO Dashboard は、次の 2 種類のユーザーをサポートしています。

* **Users**: ダッシュボードにログインし、SSO でサインインするアプリケーションを選択できます。
* **Admins**: ユーザーに表示するアプリケーションを設定できます。

この拡張機能を設定するには、次の手順を実行します。

1. [SSO Dashboard Application を作成する](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/create-sso-dashboard-application)
2. [SSO Dashboard 拡張機能をインストールする](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension)
3. [SSO Dashboard にアプリケーションを追加する](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard)
4. [SSO Dashboard 上のアプリケーションを更新する](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/update-applications-on-the-sso-dashboard)

また、[GitHub でこの SSO Dashboard Extension を確認する](https://github.com/auth0-extensions/auth0-sso-dashboard-extension)こともできます。
