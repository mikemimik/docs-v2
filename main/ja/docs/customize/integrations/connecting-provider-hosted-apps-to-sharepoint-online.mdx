Auth0 を使うと、SharePoint アプリの認証プロセスを大幅に簡素化できます。Auth0 が <Tooltip tip="アクセストークン: API へのアクセスに使用される、ランダムな文字列または JWT 形式の認可資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip> の取得を代行するため、そのトークンを使って SharePoint API を呼び出せます。

特別なライブラリは必要ありません。Auth0 がサポートする任意の SDK を使用できます。

<div id="step-1-register-your-application-in-auth0">
  ## ステップ 1. Auth0 にアプリケーションを登録する
</div>

通常どおり、Auth0 で新しいアプリケーションを登録します: **Applications &gt; NEW**。詳しい手順については、利用可能な SDK のいずれかを選択してください。次のステップで必要になるので、`client_id` は控えておいてください。

<div id="step-2-create-a-package-for-your-application">
  ## ステップ 2. アプリケーション用のパッケージを作成する
</div>

アプリケーション用に、**<Tooltip tip="Client ID: Auth0 から登録済みリソースに付与される識別値。" cta="用語集を見る" href="/ja/docs/glossary?term=Client+ID">Client ID</Tooltip>** と **<Tooltip tip="Client ID: Auth0 から登録済みリソースに付与される識別値。" cta="用語集を見る" href="/ja/docs/glossary?term=Client+Secret">Client Secret</Tooltip>** を取得する必要があります。アプリケーションの登録方法は、想定される利用方法に応じてさまざまです。

[SharePoint Add-in を登録する (Microsoft Docs)](https://docs.microsoft.com/en-us/sharepoint/dev/sp-add-ins/register-sharepoint-add-ins) では、SharePoint でアプリケーションを登録するさまざまな方法が説明されています。このチュートリアルのこの手順では、最も簡単な方法、つまり特定のテナント (ご自身のテナント) でセルフ登録する方法を使用します。

<div id="open-sharepoint-online">
  ### SharePoint Online を開く
</div>

ダッシュボードの URL は `https://{your Office365 tenant}.sharepoint.com/_layouts/15/appregnew.aspx` です

  ### Client ID と Client Secret を生成します:
</div>

<div id="complete-the-information-in-the-form">
  ### フォームに情報を入力します:
</div>

Auth0 はアプリと Office 365 のインフラストラクチャの間に位置するため、アプリにはこの URL を使用する必要があります。

**アプリ ドメイン**:

export const codeExample1 = `{yourDomain}`;

<AuthCodeBlock children={codeExample1} language="http" />

**リダイレクト URI**:

export const codeExample2 = `https://{yourDomain}/login/callback?SP_APP_TOKEN&connection=CONNECTION&client_id={yourClientId}&redirect_uri={https://yourApp/callback}`;

<AuthCodeBlock children={codeExample2} language="http" />

* `connection` は、Auth0 の接続で使う名前です (`"sharepoint"` など) 。
* `client_id` は、Auth0 であなたのアプリを識別する ID です (ステップ 1 で作成) 。
* `redirect_uri` は、一連の処理が完了した後にユーザーが最終的に遷移する、実際のアプリ内の場所です。これを指定しない場合は、Auth0 で定義したアプリのコールバック URL (localhost の場合があります) が常に使用されます。

<div id="package-the-app-and-upload-to-sharepoint">
  ### アプリをパッケージ化して SharePoint にアップロードする:
</div>

Visual Studio でアプリ マニフェストの情報を入力します。

<Frame>![SharePoint - Visual Studio - アプリケーション マニフェスト](/docs/images/cdy7uua7fh8z/36dscLHlLHZjJx1rLgrQ7N/8d64ec7a51f4e053627ecb5dfd3c7707/90SEc.png)</Frame>

`Query string` は、先ほど設定した `Redirect URI` と完全に同じになります。続いて、プロジェクトを右クリックし、`Publish` を選択します。

**Publishing Profile** を作成します (SharePoint のダッシュボードで取得したものと同じ **Client ID** と **Client Secret** を入力する必要があります) 。

**Package** をクリックし、生成されたファイルを SharePoint にアップロードします。

<div id="step-3-create-the-connection-in-auth0">
  ## ステップ 3. Auth0 で接続を作成する
</div>

この連携の最後のステップは、Auth0 に SharePoint 接続を追加することです。

必要なもの:

* `Connection Name`。任意の名前です。ステップ 2 で入力したものと一致している必要があります。
* `Client Id` と `Client Secret`。これらもステップ 2 で入力したものと一致している必要があります。
* `Test SharePoint Site Url`。これは接続のテストに使用する SP サイトの URL です (たとえば、Auth0 Dashboard で「Try」ボタンを押したときなど) 。ユーザーは常に SharePoint 内からサイトへのリンクをたどるため、実行時に使われることはありません。

ユーザーは Office Marketplace からアプリをインストールします。リンクをクリックすると Auth0 にリダイレクトされ、そこでアクセストークンが取得され、最終的にアプリに移動します。アプリは次のような `User Profile` を受け取ります。

<Frame>![Office Marketplace - Redirect -  User Profile Example](/docs/images/cdy7uua7fh8z/4piqwmx1ONWMd3kjTFMq4I/714545f5cda3ab1d1dd5c620bc96b083/8Xp6x.png)</Frame>

次のプロパティが含まれます: `cacheKey`、`refresh_token`、`host`、`site`。これらにより、SharePoint API (リストなど) を呼び出せるようになります。
