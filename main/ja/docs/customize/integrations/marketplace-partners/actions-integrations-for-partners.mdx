title: パートナー向け Actions 連携
---

パートナーの皆様は、Actions 連携 を使用して、自社製品を Auth0 のワークフローに組み込むゼロコードソリューションを構築できます。Actions 連携 を使うと、Auth0 の顧客はコードを一切書かずに、本人確認や同意管理などのソリューションを実装できます。Actions 連携 を作成するのはパートナーである皆様で、顧客はそれを自分のワークフローにドラッグ＆ドロップして利用します。

Actions 連携 は、Auth0 プラットフォーム内の指定したポイントで実行される自己完結型の関数です。JavaScript で記述されており、ソースコードは非公開のため、顧客がコードを変更することはできません。

<div id="create-and-publish-an-actions-integration">
  ## Actions連携を作成して公開する
</div>

Actions連携の作成では、計画、構築、テストを行います。[Auth0 Marketplace](https://marketplace.auth0.com/) でお客様に提供する前に、Auth0 による Actions連携の審査が行われます。Auth0 Marketplace については、[Marketplace Partners](/ja/docs/customize/integrations/marketplace-partners) を参照してください。

<div id="step-1-determine-your-use-case">
  ### ステップ 1. ユースケースを明確にする
</div>

Actions 連携でどのようなユースケースに対応するのかを明確にします。顧客のどのような課題を解決するのでしょうか。どのようにその課題を解決するのでしょうか。詳しくは、[連携のユースケースを定義する](/ja/docs/customize/integrations/marketplace-partners/defining-an-integration-use-case)をお読みください。

<div id="step-2-choose-a-flow">
  ### ステップ 2. フローを選択する
</div>

Action 連携は、1 つの [Action](/ja/docs/customize/actions) で構成されます。Action で実行できることは、Auth0 のランタイム環境内のどこで実行されるか、つまり Action の [フロー](/ja/docs/customize/actions/explore-triggers) によって決まります。利用可能な Action フローは次のとおりです。

* **ログインフロー** は、ユーザーがサインアップまたはログインに成功した後、同期的に実行されます。ログインイベントには、インタラクティブなフロー (ユーザー名/パスワードやフェデレーテッドログインなど) と、非インタラクティブなフロー (セッションチェックや <Tooltip tip="Refresh Token: ユーザーに再度ログインさせることなく、新しいアクセストークンを取得するために使用されるトークン。" cta="用語集を見る" href="/ja/docs/glossary?term=refresh+token">リフレッシュトークン</Tooltip> の交換など) が含まれます。このフローの詳細については、[Login Flow](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) を参照してください。
* **Machine to Machine フロー** は、アプリケーションが client credentials grant を使用して <Tooltip tip="Access Token: API へのアクセスに使用される Authorization 資格情報で、不透明文字列または JWT の形式を取ります。" cta="用語集を見る" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip> を要求したときに、同期的に実行されます。このフローの詳細については、[Machine to Machine Flow](/ja/docs/customize/actions/explore-triggers/machine-to-machine-trigger) を参照してください。
* **Pre User Registration フロー** は、Auth0 がユーザーをデータベースまたは <Tooltip tip="Passwordless: 第1認証要素としてパスワードに依存しない認証形式。" cta="用語集を見る" href="/ja/docs/glossary?term=passwordless">パスワードレス</Tooltip> 接続に追加する前に、同期的に実行されます。このフローは、enterprise 接続やソーシャル接続などのフェデレーテッドログインでは実行されません。このフローの詳細については、[Pre User Registration Flow](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/pre-user-registration-trigger) を参照してください。
* **Post User Registration フロー** は、Auth0 がユーザーをデータベースまたはパスワードレス接続に追加した後に、非同期で実行されます。このフローは、enterprise 接続やソーシャル接続などのフェデレーテッドログインでは実行されません。このフローの詳細については、[Post User Registration Flow](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/post-user-registration-trigger) を参照してください。
* **Post Change Password フロー** は、データベース接続のユーザーがパスワードを変更した後に、非同期で実行されます。このフローは、他の種類の接続でのパスワード変更では実行されません。このフローの詳細については、[Post Change Password Flow](/ja/docs/customize/actions/explore-triggers/password-reset-triggers/post-change-password-trigger) を参照してください。
* **Send Phone Message** **フロー** は、カスタム <Tooltip tip="Multi-factor authentication (MFA): SMS によるコードなど、ユーザー名とパスワードに加えた認証要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> プロバイダーの一部として、電話または SMS でメッセージを送信します。カスタム MFA プロバイダーについては、[Customize Multi-factor Authentication SMS and Voice Messages](/ja/docs/customize/customize-sms-or-voice-messages) を参照してください。このフローの詳細については、[Send Phone Message Flow](/ja/docs/customize/actions/explore-triggers/mfa-notifications-trigger) を参照してください。

Action フローの詳細については、[Explore Flows and Triggers](/ja/docs/customize/actions/explore-triggers) を参照してください。ログイン時にユーザーをエンドポイントにリダイレクトする必要がある場合は、[Redirect Actions for Partners](/ja/docs/customize/integrations/marketplace-partners/redirect-actions-for-partners) を参照してください。

<div id="step-3-build-an-actions-integration">
  ## ステップ 3. Actions 連携を構築する
</div>

Action 連携のユースケースに適したフローを決めたら、テスト用テナントで Action を作成します。手順については、[Write Your First Action](/ja/docs/customize/actions/write-your-first-action)を参照してください。この Action が、Action 連携コードのベースになります。

また、連携の構築に役立つ Action 連携テンプレートも利用できます。

* [Post Login Action 連携テンプレート](https://github.com/auth0-marketplace/TEMPLATE-action-post-login)
* [Machine-to-Machine Action 連携テンプレート](https://github.com/auth0-marketplace/TEMPLATE-action-client-credentials)
* [Pre User Registration Action 連携テンプレート](https://github.com/auth0-marketplace/TEMPLATE-action-pre-user-registration)
* [Post User Registration Action 連携テンプレート](https://github.com/auth0-marketplace/TEMPLATE-action-post-user-registration)
* [Post Change Password Action 連携テンプレート](https://github.com/auth0-marketplace/TEMPLATE-action-post-password-change)
* [Send Phone Message Action 連携テンプレート](https://github.com/auth0-marketplace/TEMPLATE-action-phone-message)

Actions 連携を安全かつ安定して実装する方法については、[Actions Coding Guidelines](/ja/docs/customize/actions/action-coding-guidelines)を参照してください。あわせて、Actions 連携に固有の以下のガイドラインにも従ってください。

* テスト用テナントで Action を作成する際は、顧客が指定する必要のあるすべての値 (識別子、テナント名、しきい値など) に secrets を使用してください。Actions 連携を提出すると、Auth0 は、どのフィールドを保存時に暗号化する必要があるか (secrets のままにするか) 、どのフィールドを編集可能なテキストフィールドにできるかを確認します。
* 設定やトラブルシューティングに役立つよう、ログ出力用のデバッグ / テストモード設定フラグを使用してください。Actions 連携で、すべての実行についてログを出力することはまれであるべきです。
* レビューしやすいように、secret 名は `ALL_CAPS_SNAKE_CASE` で記述してください。
* Action 連携はクローズドソースであり、顧客はコードを閲覧したり変更したりできません。Action 連携を制御するために必要な顧客固有の入力は、secrets または設定から取得する必要があります。

<div id="step-4-test-your-actions-integrations">
  ### Step 4. Actions 連携をテストする
</div>

Auth0 のレビュアーはお客様のサービスにアクセスできないため、レビューに提出する前に Action をテストする必要があります。成功する実行パターンとエッジケースの両方を手動でテストしてください。次の点を確認します。

* Action はサイレントに失敗したり、後続の実行を妨げたりしませんか？
* HTTP リクエストが失敗した場合、Action はどのように動作しますか？
* ユーザーに表示されるエラーメッセージはありますか？ ある場合はどのような内容ですか？
* どのような情報がログに記録されますか？
* Action は回避されたり、なりすまされたりする可能性がありますか？

提出プロセスでは、Auth0 は正常系に対するユニットテストカバレッジを求めます。Auth0 では、これをできるだけ簡単に行えるよう、必要なツールとサンプルをすべて提供しています。

<div id="step-5-submit-your-actions-integration">
  ### ステップ 5. Actions 連携を申請する
</div>

Actions 連携の作成と十分なテストが完了したら、Auth0 にレビューを依頼するため、[Auth0 Marketplace のサービスリクエスト](https://autheco.atlassian.net/servicedesk/customer/portal/1/group/1/create/43)を送信してください。初回レビューを通過すると、その連携はセキュリティ準拠の審査に進みます。

<div id="step-6-publish-your-actions-integration">
  ### ステップ 6. Actions 連携を公開する
</div>

Auth0 による Actions 連携の審査が完了すると、審査の結果と公開時期についてお知らせします。
