`language` 設定オプションを使用すると、Lock の言語を変更できます。これにより、Lock の `i18n` ディレクトリから対応する言語ファイルが読み込まれます。

<div id="set-language-option">
  ## 言語オプションの設定
</div>

次の例を使用するには、まずページに Lock を組み込む必要があります。

`<script src="https://cdn.auth0.com/js/lock/11.14/lock.min.js"></script>`

次に、`options` オブジェクトを定義し、`language` オプションを指定します。`language` オプションには、[`Lock` 内の `i18n` ディレクトリ](https://github.com/auth0/lock/tree/master/src/i18n)にある対応するファイル名と一致する文字列を指定する必要があります。最後に、Lock をインスタンス化します。

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample1 = `// 対応している言語を選択
var options = {
  language: 'es'
};

// Auth0Lock を初期化
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample1} language="javascript" />

Lock の翻訳データは、各種翻訳を表すキーと値のペアで構成された言語ファイルから取得されます。言語によっては一部の値が不足している場合があり、その際は `language does not have property <missing prop>` という警告が表示されます。不足している値を追加するには、[pull request](https://github.com/auth0/lock/tree/master/src/i18n) を送信してください。あるいは、Lock の `options` で不足している値を定義することもできます (以下を参照) 。

<div id="replace-dictionary-terms">
  ## 辞書の文言を置き換える
</div>

`languageDictionary` オプションを使って、特定の辞書項目を独自にカスタマイズすることもできます。これは、対応言語のいずれかを使用しつつ、一部の項目の文言だけを変更したい場合に便利です。たとえば、`title` の文言を変更したり、ウィジェット内の残りのテキストはそのままに、他のラベルの表示方法だけを変えたりできます。

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample2 = `// languageDictionary の一部の属性をカスタマイズ
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "ログイン"
  },
};

// Auth0Lock を初期化
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample2} language="javascript" />

利用可能な `languageDictionary` プロパティ名の例と、`language` ファイルの構成方法については、[Lock の英語辞書ファイル](https://github.com/auth0/lock/blob/master/src/i18n/en.js)を参照してください。

文字列値 (URL) を取る `languageBaseUrl` オプションは、Auth0 が提供する翻訳の言語ソース URL を上書きします。デフォルトでは、提供される言語翻訳の保存先である Auth0 の CDN URL `https://cdn.auth0.com` を使用します。別の値を指定することで、必要に応じて、アプリケーション向けの言語翻訳に独自のソースを使用できます。言語ソースは JavaScript ファイルである必要があります。
