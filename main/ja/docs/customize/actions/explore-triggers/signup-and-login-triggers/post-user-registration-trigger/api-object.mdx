`post-user-registration` Actions トリガーの API オブジェクトには、次の項目が含まれます。

<div id="apicache">
  ## `api.cache`
</div>

複数の実行にわたって保持されるデータを保存・取得します。

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

指定されたキーに対応するキャッシュレコードが存在する場合は、それを削除します。

キャッシュから値が削除された場合は、`type: "success"` を持つ `CacheWriteResult` オブジェクトを返します。操作に失敗した場合は `type: "error"` を返します。エラー時には、返されるオブジェクトに
失敗の内容を示す `code` プロパティが含まれます。

| パラメータ | 説明                              |
| ----- | ------------------------------- |
| `key` | *String*. キャッシュに保存されているレコードのキー。 |

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

指定された
`key` に対応するキャッシュ済みの値のレコードが存在する場合、
それを取得します。レコードが見つかった場合、キャッシュされた値は
返されるオブジェクトの `value` プロパティに格納されています。

指定された
`key` に対応する項目がキャッシュ内に見つかった場合、キャッシュレコードを返します。キャッシュレコードはオブジェクトで、
キャッシュされた値を保持する `value` プロパティと、
Unix エポックからの経過ミリ秒で
レコードの最大有効期限を示す `expires_at` プロパティを
持ちます。

**重要:**
このキャッシュは、短期間だけ保持される一時的なデータ向けに設計されています。項目は、指定された
有効期間内であっても、後続のトランザクションでは利用できない場合があります。

| Parameter | Description                    |
| --------- | ------------------------------ |
| `key`     | *String*。キャッシュに保存されているレコードのキー。 |

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, [options])`
</div>

指定したキーに、文字列値をキャッシュへ保存または更新します。

このキャッシュに保存された値は、設定されたトリガーのスコープ内でのみ有効です。
また、
[Actions Cache Limits](/ja/docs/customize/actions/limitations)
の制限が適用されます。

この方法で保存された値の有効期間は、指定された
`ttl` または `expires_at` までです。有効期間が
指定されていない場合は、デフォルトで 15 分になります。有効期間は、
[Actions Cache Limits](/ja/docs/customize/actions/limitations)
に記載されている最大時間を超えることはできません。

| Parameter            | Description                                                                                                                                                                                         |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `key`                | *String*。キャッシュに保存されるレコードのキーです。                                                                                                                                                                      |
| `value`              | *String*。保存するレコードの値です。                                                                                                                                                                              |
| `options`            | *Optional object*。キャッシュの動作を調整するためのオプションです。                                                                                                                                                          |
| `options.expires_at` | *Optional number*。Unix エポックからの経過時間をミリ秒で表した絶対有効期限です。キャッシュされたレコードはこれより前に削除される場合がありますが、指定した `expires_at` を過ぎて保持されることはありません。<br />*注:* `ttl` も指定する場合、この値は指定しないでください。両方のオプションを指定した場合は、より早い方の有効期限が使用されます。 |
| `options.ttl`        | *Optional number*。このキャッシュエントリの有効期間をミリ秒で表した値です。キャッシュされた値はこれより前に削除される場合がありますが、指定した `ttl` を過ぎて保持されることはありません。<br />*注:* `expires_at` も指定する場合、この値は指定しないでください。両方のオプションを指定した場合は、より早い方の有効期限が使用されます。         |
