title: MFA 通知のトリガー
---

Send Phone Message トリガーを使用すると、[多要素認証 (MFA) ](/docs/ja/secure/multi-factor-authentication)で SMS/Voice を認証要素として使用する際にコードを実行できます。メッセージの送信に[カスタムプロバイダー](/docs/ja/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa#custom-phone-messaging-providers)を使用する場合は、このフローの `send-phone-message` トリガーを使ってカスタムプロバイダーを設定する必要があります。

<Frame>![Actions の Send Phone Message フローの図。](/docs/images/cdy7uua7fh8z/FSkVXDdknDJq1hsK08EYu/e031ec0067a5460afae8d9ed5d462288/send-phone-message-flow.png)</Frame>

このフロー内の Actions はブロッキング (同期) です。つまり、トリガーのプロセスの一部として実行され、Action が完了するまで Auth0 パイプラインの残りの処理は実行されません。

<div id="triggers">
  ## トリガー
</div>

<div id="send-phone-message">
  ### Send Phone Message
</div>

`send-phone-message` トリガーは、登録プロセスとチャレンジプロセス (`event.message_options.action`) で実行されます。また、<Tooltip tip="Universal Login: アプリケーションは、ユーザーの本人確認のために、Auth0 の認可サーバーでホストされている Universal Login にリダイレクトされます。" cta="用語集を見る" href="/docs/ja/glossary?term=Universal+Login">Universal Login</Tooltip> の New experience を使用している場合は、`voice` メッセージタイプ (`event.message_options.message_type === 'voice'`) でも実行されます。

<div id="common-use-cases">
  ## よくあるユースケース
</div>

<div id="use-a-custom-sms-provider">
  ### 独自のSMSプロバイダーを使用する
</div>

```javascript lines
const AWS = require("aws-sdk");

/**
 * SendPhoneMessage フローの実行中に呼び出されるハンドラー。
 *
 * @param {Event} event - ユーザーおよびログイン時のコンテキストに関する詳細。
 */
exports.onExecuteSendPhoneMessage = async (event) => {
  const text = event.message_options.text;
  const recipient = event.message_options.recipient;

  const awsSNS = new AWS.SNS({
    apiVersion: "2010-03-31",
    region: event.secrets.AWS_REGION,
    credentials: new AWS.Credentials(event.secrets.AWS_ACCESS_KEY_ID, event.secrets.AWS_SECRET_ACCESS_KEY)
  });

  const params = { Message: text, PhoneNumber: recipient };

  return awsSNS
    .publish(params)
    .promise();
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  この Action を正しく実行するには、Action に `AWS_REGION`、`AWS_ACCESS_KEY_ID`、`AWS_SECRET_ACCESS_KEY` という名前の secrets が含まれ、`aws-sdk` NPM package への dependency が設定されている必要があります。
</Callout>
