sidebarTitle: 概要
title: トリガーを確認する
---

トリガーは、必要なカスタマイズのために Actions を追加できる Auth0 の処理上のポイントです。1 つのトリガーには複数の Actions を配置でき、順番に実行されます。カスタムトークン交換 を除き、トリガーには同期 (ブロッキング) と非同期 (ノンブロッキング) があります。各トリガーには、それぞれ固有の Objects と API が用意されています。これらは、接続 (たとえば <Tooltip tip="パスワードを第 1 要素として使用しない認証方式。" cta="用語集を見る" href="/ja/docs/glossary?term=passwordless">パスワードレス</Tooltip> 接続) によって異なる場合があります。

特定のトリガーにおける Actions 用の reference Objects と API を確認するには、以下のトリガーを選択してください。あわせて、Auth0 における [パスワードレス接続](/ja/docs/authenticate/passwordless) の定義も確認してください。

<div id="signup-and-login">
  #### サインアップとログイン
</div>

ユーザーがデータベース接続またはパスワードレス接続に追加されるとき (サインアップ) 、またはユーザーの認証トランザクション中 (ログイン) 。

| トリガー                     | 説明                              |
| ------------------------ | ------------------------------- |
| `pre-user-registration`  | ユーザーが作成される前に実行されます。             |
| `post-user-registration` | ユーザーが作成された後に非同期で実行されます。         |
| `post-login`             | ユーザーが認証された後、トークンが発行される前に実行されます。 |

<div id="mfa-notifications">
  #### MFA 通知
</div>

SMS を <Tooltip tip="多要素認証（MFA）: ユーザー名とパスワードに加えて、SMS で送信されるコードなどの要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=Multi-Factor+Authentication">多要素認証</Tooltip> (MFA) の要素として使用する場合や、カスタムプロバイダーを設定する場合。

| トリガー                 | 説明                                             |
| -------------------- | ---------------------------------------------- |
| `send-phone-message` | 登録時およびチャレンジ時のメッセージ送信にカスタムプロバイダーを使用するとトリガーされます。 |

<div id="password-reset">
  #### パスワードリセット
</div>

パスワードリセットのリクエストが開始されたとき。

| トリガー                            | 説明                                    |
| ------------------------------- | ------------------------------------- |
| `post-change-password`          | パスワードの変更後に実行されます。                     |
| `password-reset-post-challenge` | 最初のチャレンジが完了した後、パスワードがリセットされる前に実行されます。 |

<div id="machine-to-machine">
  #### マシン間
</div>

ユーザーではなく、アプリケーションが認証・認可されて API を使用する場合です。

| トリガー                   | 説明                          |
| ---------------------- | --------------------------- |
| `credentials-exchange` | access token が返される前に実行されます。 |

<div id="custom-token-exchange">
  ### カスタムトークン交換
</div>

ユーザーをログインさせるためのカスタムトークン交換リクエストを受信したとき。

| トリガー                    | 説明                                            |
| ----------------------- | --------------------------------------------- |
| `custom-token-exchange` | トランザクションの最初のステップで、`post-login`トリガーより前に実行されます。 |
