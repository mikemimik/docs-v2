Auth0 Multiple <Tooltip tip="カスタムドメイン: 特別な名称、またはバニティ名を持つサードパーティのドメイン。" cta="用語集を見る" href="/ja/docs/glossary?term=Custom+Domains">カスタムドメイン</Tooltip> (MCD) 機能は高い柔軟性を備えており、組織は単一の Auth0 テナントから、多様なブランド展開、地域ごとの要件、パートナー連携を管理できます。これにより、アイデンティティ管理を簡素化しながら、ユーザーにシームレスでブランドに統一感のある体験を提供できます。

<div id="single-brand-multiple-top-level-domains-tlds">
  ### 単一ブランド、複数のトップレベルドメイン (TLD)
</div>

航空会社のように、単一のユーザーベースを持ちながらも、異なるトップレベルドメイン名 (たとえば `.com`、`.co.uk`、`.com.au`) を必要とする企業もあります。これは、多くの場合、各法域の規制や顧客の好みによるものです。MCD を使えば、統合されたユーザーストアを備えた単一のテナントを運用しながら、複数のトップレベルドメインを提供する構成を Auth0 で簡単に実現できます。たとえば、メインの Web サイトが `exampleco.com` であっても、`exampleco.co.uk` や `exampleco.com.au` も運用できます。

<div id="multiple-brands-single-user-base">
  ### 複数ブランドで単一のユーザーベースを共有
</div>

企業が複数のブランドを展開し、それぞれに独自のドメインを持たせながら、単一のユーザーストアや 1 つの Auth0 テナントによる統合機能のメリットを活用するのは一般的です。たとえば小売業では、ある企業が高級住宅家具 (`brand1-home-furniture.com`) とオフィス家具 (`brand2-office-furniture.com`) を扱う別個のブランドを運営している場合があります。MCD を使えば、これらのブランドはそれぞれ別のドメイン名を維持したまま、集中管理されたユーザーストアと Auth0 の構成を共有できます。

<div id="white-labeled-b2b-saas">
  ### ホワイトラベル型のB2B SaaS
</div>

多くのSaaSプロバイダーは、ホワイトラベルモデルでサービスを提供しています。たとえば、求人応募を管理するプラットフォーム `job-saas.com` を考えてみましょう。顧客企業は、`jobs.client-a.com` や `jobs.client-b.com` のように、自社のサブドメインで `job-saas.com` のサービスを提供し、見た目や操作感をカスタマイズできます。MCD を使えば、`job-saas.com` は Auth0 の設定全体を単一のテナントで管理しながら、異なる組織やカスタムドメインを顧客に直接提供できます。
