---

<Tooltip tip="カスタムドメイン: 固有名やバニティ名を使用するサードパーティのドメイン。" cta="用語集を見る" href="/ja/docs/glossary?term=custom+domain">カスタムドメイン</Tooltip>の証明書を Auth0 に管理させる場合は、そのドメインに CNAME レコードを追加するだけで済みます。Auth0 がそのレコードを検証した後、Auth0 のサーバー上で証明書を生成します。これらの証明書は 3 か月ごとに自動更新されます。簡単に設定でき、証明書を自分で管理する必要もありません。

Auth0 管理の証明書を使用してカスタムドメインを設定するには、まずドメイン名を Auth0 に提供し、そのドメインの所有権を確認する必要があります。確認が完了したら、カスタムドメインを使用するように Auth0 の機能を設定する必要があります。

<div id="provide-your-domain-name-to-auth0">
  ## Auth0 にドメイン名を指定する
</div>

1. [Auth0 Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) または [Auth0 Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/custom_domains) に移動します。表示されたボックスにカスタムドメインを入力し、**Auth0 管理の証明書** を選択します。
2. **Add Domain** をクリックします。ドメインを追加した後も **Add Domain** ボタンは引き続き表示されますが、テナントごとに追加できるドメインは 1 つだけです。

<div id="verify-ownership">
  ## 所有権を確認する
</div>

Auth0 でこのドメインを使用する前に、ドメインの所有権を確認する必要があります。

1. [Auth0 Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/branding/custom_domains) に移動し、Auth0 Dashboard に表示されている CNAME 検証レコードをドメインの DNS レコードに追加します。

   <Frame>![Auth0 Branding Custom Domains Auth0-Managed Certificate Verify Domain](/docs/images/cdy7uua7fh8z/3dz8ii70nm0a5amS0PNJ5K/c7b58043042293f5502ec012901f5dac/Custom_Domain_-_Pending_Verification_-_English.png)</Frame>
2. **Verify** をクリックして先に進みます。
   DNS 設定によっては、Auth0 が CNAME レコードを検証できるようになるまで数分かかる場合があります。Auth0 がドメイン名を検証できると、確認ウィンドウが表示されます。これは、検証プロセスが完了したことを意味します。1 ～ 2 分以内に、カスタムドメインを使用できるようになります。

<div id="add-cname-verification-record-to-dns-record">
  ## DNS レコードに CNAME 検証レコードを追加する
</div>

追加後は、証明書の更新時に問題が発生しないよう、CNAME レコードを常に維持しておく必要があります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 では、厳密に必要な場合を除き、CNAME フラット化を無効にすることを推奨しています。Auth0 管理の証明書に対する CNAME フラット化はサポート対象外の構成であり、予告なくカスタムドメインが機能しなくなる可能性があるためです。
</Callout>

Cloudflare で管理されるすべてのサブドメインに対して CNAME フラット化を有効にしつつ、特定のサブドメインを Auth0 の カスタムドメイン として設定する必要がある場合は、Auth0 用にその[サブドメインを別の DNS プロバイダーに委任する](https://developers.cloudflare.com/dns/manage-dns-records/how-to/subdomains-outside-cloudflare/)ことを検討してください。
これにより、Auth0 に使用するサブドメインを除き、すべてのサブドメインで CNAME フラット化を利用できるようになります。

以下の手順は、ドメインホストプロバイダーによって異なる場合があります。

1. ドメイン管理サービスにログインします。
2. 新しいレコードを作成します。

   | パラメーター                 | 値                                                             |
   | ---------------------- | ------------------------------------------------------------- |
   | **レコードタイプ**            | **CNAME**                                                     |
   | **名前**                 | カスタムドメイン 名を入力します (例: **login.northwind.com**) 。          |
   | **Time to Live (TTL)** | デフォルト値を使用します。                                                 |
   | **値**                  | ドメインの DNS レコード用に Auth0 Dashboard から提供された **CNAME** の値を貼り付けます。 |
3. 完了したら、レコードを保存します。

Auth0 がドメイン名を検証できると、確認ウィンドウが表示されます。これは、検証プロセスが完了したことを意味します。1 ～ 2 分以内に、カスタムドメイン を使用できるようになるはずです。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  DNS プロバイダーで CNAME レコードのプロキシがデフォルトで有効になっている場合、カスタムドメイン は保留状態のまま無期限に残ります。DNS プロバイダーの設定を確認し、プロキシを無効にする必要がある場合があります。
</Callout>

検証プロセスを完了できない場合は、これらの手順を再度実行する前に少なくとも 4 時間待ってください。サービスの中断を避けるため、再検証の際に カスタムドメイン を再作成しないでください。
