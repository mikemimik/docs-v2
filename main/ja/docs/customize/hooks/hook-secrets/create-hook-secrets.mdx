<Warning>
  ルールと Hooks のサポート終了 (EOL) 日は**2026年11月18日**です。また、**2023年10月16日**以降に作成された新しいテナントでは、これらはすでに利用できません。有効な Hooks を使用している既存のテナントは、提供終了まで Hooks 機能を引き続き利用できます。

  Auth0 を拡張するには、Actions の使用を強くお勧めします。Actions では、豊富な型情報、インラインドキュメント、公開 `npm` パッケージを利用できるほか、拡張機能の利用体験を高める外部連携にも対応できます。Actions の詳細については、[Auth0 Actions の仕組みを理解する](/ja/docs/customize/actions/actions-overview)をご覧ください。

  移行を支援するために、[ルールから Actions への移行](/ja/docs/customize/actions/migrate/migrate-from-rules-to-actions)と[Hooks から Actions への移行](/ja/docs/customize/actions/migrate/migrate-from-hooks-to-actions)のガイドを用意しています。また、機能比較、[Actions のデモ](https://www.youtube.com/watch?v=UesFSY1klrI)、そのほか移行に役立つリソースをまとめた専用の[Move to Actions](https://auth0.com/extensibility/movetoactions)ページもあります。

  ルールと Hooks の非推奨化について詳しくは、ブログ記事をご覧ください: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/)。
</Warning>

<Warning>
  2026年にルールと Hooks の機能を削除する予定であるため、新しいルールまたは Hook は、Actions への移行をテストする目的に限り、Development 環境でのみ作成してください。

  ルールを Actions に移行する方法については、[Migrate from Rules to Actions](/ja/docs/customize/actions/migrate/migrate-from-rules-to-actions)をご覧ください。Hooks を Actions に移行する方法については、[Migrate from Hooks to Actions](/ja/docs/customize/actions/migrate/migrate-from-hooks-to-actions)をご覧ください。
</Warning>

任意の[Hook](/ja/docs/customize/hooks)に対して、Auth0 Dashboard または<Tooltip tip="Management API: 管理タスクを実行するための製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip>を使用して複数の Hook シークレットを作成できます。

Hook シークレットは、[Deploy Command-Line Interface (CLI) Tool](/ja/docs/deploy-monitor/deploy-cli-tool)を使用してインポートおよびエクスポートすることもできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  1つの Hook につき、最大20個のシークレットを作成できます。
</Callout>

<div id="create-hook-secrets-using-the-dashboard">
  ## Auth0 Dashboard を使用して Hook シークレットを作成する
</div>

1. [Auth0 Dashboard](https://manage.auth0.com/#/) の [Hooks](https://manage.auth0.com/#/hooks) ページに移動し、編集する Hook の横にある鉛筆アイコンをクリックします。
2. Hook エディターでレンチアイコンをクリックし、**Secrets** をクリックします。
3. **Add Secret** をクリックします。
4. シークレットの名前と値を入力し、**Save** をクリックします。

<div id="create-hook-secrets-using-the-management-api">
  ## Management API を使用して Hook シークレットを作成する
</div>

[Add Hook Secrets エンドポイント](https://auth0.com/docs/api/management/v2/#!/Hooks/post_secrets)に `POST` リクエストを送信します。`MGMT_API_ACCESS_TOKEN`、`HOOK_ID`、`HOOK_SECRET_KEY`、`HOOK_SECRET_VALUE` の各プレースホルダー値は、それぞれ Management API の <Tooltip tip="Access Token: API へのアクセスに使用する認可情報で、不透明な文字列または JWT の形式を取ります。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">Access Token</Tooltip>、Hook ID、Hook のキーと値のペアに必ず置き換えてください。

| 値                       | 説明                                                                                                                                                  |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens) のうち、[scope](/ja/docs/glossary?term=scope) `create:hooks` を持つもの。 |
| `HOOK_ID`               | シークレットを追加する対象のフックの ID。                                                                                                                              |
| `HOOK_SECRET_KEY`       | 指定したフックに追加するシークレットの名前。このエンドポイントは、キーと値のペアからなるオブジェクトを受け付けます。                                                                                          |
| `HOOK_SECRET_VALUE`     | 指定したフックに追加するシークレットの値。このエンドポイントは、キーと値のペアからなるオブジェクトを受け付けます。                                                                                           |
