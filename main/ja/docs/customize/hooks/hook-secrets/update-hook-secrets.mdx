<Warning>
  ルールとHooksのサポート終了 (EOL) 日は**2026年11月18日**です。また、**2023年10月16日**以降に作成された新規テナントでは、これらはすでに利用できません。アクティブな Hooks がある既存のテナントでは、提供終了まで Hooks プロダクトを引き続き利用できます。

  Auth0 を拡張するには、Actions の利用を強く推奨します。Actions では、豊富な型情報、インラインドキュメント、公開 `npm` パッケージを利用できるほか、拡張機能の開発体験をさらに高める外部連携にも接続できます。Actions の詳細については、[Understand How Auth0 Actions Work](/ja/docs/customize/actions/actions-overview)をお読みください。

  移行を支援するために、[ルールから Actions への移行](/ja/docs/customize/actions/migrate/migrate-from-rules-to-actions)と[Hooksから Actions への移行](/ja/docs/customize/actions/migrate/migrate-from-hooks-to-actions)のガイドを用意しています。また、機能比較、[Actions のデモ](https://www.youtube.com/watch?v=UesFSY1klrI)、その他の移行に役立つリソースをまとめた専用の[Move to Actions](https://auth0.com/extensibility/movetoactions)ページもあります。

  ルールとHooksの非推奨化についてさらに詳しくは、ブログ記事をご覧ください：[Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/)。
</Warning>

<Warning>
  2026年にルールとHooksの機能を削除する予定であるため、新しいルールやHooksは、Actions への移行をテストする目的で、Development 環境でのみ作成してください。

  ルールを Actions に移行する方法については、[Migrate from Rules to Actions](/ja/docs/customize/actions/migrate/migrate-from-rules-to-actions)をお読みください。Hooksを Actions に移行する方法については、[Migrate from Hooks to Actions](/ja/docs/customize/actions/migrate/migrate-from-hooks-to-actions)をお読みください。
</Warning>

特定の Hook に追加された Hook シークレットは、Auth0 Dashboard または <Tooltip tip="Management API: お客様が管理タスクを実行するための API です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用して更新できます。

Hook シークレットは、Deploy Command-Line Interface (CLI) ツールを使用してインポートおよびエクスポートすることもできます。詳しくは、[Deploy CLI Tool](/ja/docs/deploy-monitor/deploy-cli-tool)をお読みください。

<div id="update-hook-secrets-using-the-dashboard">
  ## Auth0 Dashboard を使用して Hook シークレットを更新する
</div>

1. [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/) に移動し、編集する Hook の横にある鉛筆アイコンをクリックします。
2. Hook エディターでレンチアイコンをクリックし、**Secrets** をクリックします。
3. 編集する Hook シークレットの値の横にある鉛筆と紙のアイコンをクリックします。
4. 選択した Hook シークレットの名前や値を変更し、**Save** をクリックします。

<div id="update-hook-secrets-using-the-management-api">
  ## Management API を使用して Hook シークレットを更新する
</div>

[Hook シークレットを更新するエンドポイント](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_secrets)に対して `PATCH` リクエストを送信します。`HOOK_ID` と `MGMT_API_ACCESS_TOKEN` のプレースホルダー値は、それぞれお使いの Hook ID と Management API の <Tooltip tip="Access Token: API へのアクセスに使用される、opaque な文字列または JWT 形式の Authorization 認証情報。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">Access Token</Tooltip> に必ず置き換えてください。

    --header 'content-type: application/json' \

| 値                       | 説明                                                                                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN` | [scope](/ja/docs/glossary?term=scope) `update:hooks` を含む Management API 用のアクセストークン。詳しくは、[Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens) を参照してください。 |
| `HOOK_ID`               | シークレットを更新する Hook の ID。                                                                                                                                                          |
| `HOOK_SECRET_KEY`       | 指定した Hook で更新するシークレットの名前。このエンドポイントは、キーと値のペアからなるオブジェクトを受け付けます。                                                                                                                   |
| `HOOK_SECRET_VALUE`     | 指定した Hook で更新するシークレットの値。このエンドポイントは、キーと値のペアからなるオブジェクトを受け付けます。                                                                                                                    |

<Warning>
  指定した Hook に設定されている Hook シークレットを取得すると、値にはプレースホルダーテキスト `_VALUE_NOT_SHOWN` が含まれます。更新時にこのプレースホルダーテキストをそのまま返さないよう注意してください。返してしまうと、シークレットの実際の値が上書きされます。
</Warning>
