description: Auth0 Dashboard で利用できる API に関連する設定について説明します。
title: API 設定
---

許可されたアプリケーションから利用する登録済み API を設定するには、[Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) にある <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要な製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> の **設定** タブを使用します。API の設定を行うには、一覧で対象の API の横にある **...** をクリックして **設定** を選択するか、API 名をクリックします。API の作成と登録の方法については、[API を登録する](/ja/docs/get-started/auth0-overview/set-up-apis) を参照してください。

<Frame>![Auth0 Dashboard のアプリケーション API 一覧](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>

<div id="settings">
  ## 設定
</div>

このタブでは、トークンの有効期限、ロールベースのアクセス制御 (RBAC) 、その他のアクセス設定を構成できます。変更を保存するには、タブの下部にある **Save** をクリックします。

<div id="general-settings">
  ### 一般設定
</div>

これらのフィールドは、Auth0 の <Tooltip tip="Management API: 顧客が管理タスクを実行するためのプロダクトです。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を除き、API を最初に登録したときに設定されたものです。変更できるのは **Name** のみです。

<Frame>![Dashboard Applications APIs Settings Tab General Settings](/docs/images/cdy7uua7fh8z/QCaXd3FSeP6SYqLfEJWBr/fbb1bb5b2e12fb099073c33fb39cf44a/dashboard-applications-apis-settings-general-settings.png)</Frame>

* **Id**: Auth0 によって生成される一意の英数字の文字列です。この情報は読み取り専用で、[Auth0&#39;s Management API Resource Servers endpoints](https://auth0.com/docs/api/management/v2#!/Resource_Servers/get_resource_servers_by_id) を直接操作する場合にのみ必要です。
* **Name**: API のフレンドリー名です。機能には影響しません。次の文字は使用できません: `< >`。
* **Identifier**: API の一意の識別子です。この値は API の作成時に設定され、その後は変更できません。URL の使用を推奨しますが、公開された URL である必要はありません。Auth0 が API を呼び出すことはありません。

<div id="token-settings">
  ### トークン設定
</div>

<Frame>![Dashboard - API - Token Settings - Expiration](/docs/images/cdy7uua7fh8z/2FFfqqmntXL9CnGrSxea9M/810e5542553168ffcd6081a10bc2db2f/image2.png)</Frame>

* **最大 <Tooltip tip="Access Token: API へのアクセスに使用される、opaque 文字列または JWT 形式の認可資格情報です。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip> 有効期間 (秒)&#x20;**: アクセストークンの有効期限が切れるまでの時間 (秒) です。デフォルト値は 86400 秒 (24 時間) です。設定できる最大値は 2592000 秒 (30 日) です。
* **Implicit / ハイブリッドフロー アクセストークン有効期間 (秒)&#x20;**: Implicit またはハイブリッドフローで発行されたアクセストークンの有効期限が切れるまでの時間 (秒) です。デフォルト値は 86400 秒 (24 時間) です。この値は最大アクセストークン有効期間を超えることはできません。
* **<Tooltip tip="JSON Web Token (JWT): 2 者間でクレームを安全にやり取りするために使用される、標準的な ID トークン形式（また、多くの場合アクセストークン形式）です。" cta="用語集を見る" href="/ja/docs/glossary?term=JSON+Web+Token">JSON Web トークン</Tooltip> (JWT) プロファイル**: このプロファイルは、API 用に発行されるアクセストークンの形式を決定します。使用可能な値は `Auth0` と `RFC 9068` です。詳細は、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。
* **JSON Web Token (JWT) <Tooltip tip="Signing Algorithm: トークンが改ざんされていないことを保証するために、トークンにデジタル署名する際に使用されるアルゴリズムです。" cta="用語集を見る" href="/ja/docs/glossary?term=Signing+Algorithm">署名アルゴリズム</Tooltip>**: トークンの署名に使用するアルゴリズムです。署名は、JWT の送信者が名乗っている本人であることを検証し、メッセージが途中で改変されていないことを保証するために使われます。使用可能な値は [`HS256`](https://en.wikipedia.org/wiki/Symmetric-key_algorithm) と [`RS256`](https://en.wikipedia.org/wiki/Public-key_cryptography) です。`RS256` (推奨) を選択すると、トークンはテナントの秘密キーで署名されます。この値は API の作成時に設定され、その後は変更できません。署名アルゴリズムと Auth0 での仕組みの詳細は、[Signing Algorithms](/ja/docs/get-started/applications/signing-algorithms) を参照してください。

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    署名は JWT の一部です。JWT の構造に不慣れな場合は、[JSON Web Token Structure](/ja/docs/secure/tokens/json-web-tokens/json-web-token-structure) を参照してください。
  </Callout>
* **JSON Web Encryption (JWE):** 有効にすると、発行されるアクセストークンは JSON Web Encryption (JWE) 形式で暗号化されます。詳細は、[JSON Web Encryption](/ja/docs/secure/tokens/access-tokens/json-web-encryption) を参照してください。

<div id="rbac-settings">
  ### RBAC 設定
</div>

<Frame>![Auth0 Dashboard の API 設定にある RBAC トグル](/docs/images/cdy7uua7fh8z/65tKb6aj0ktc2qXLUVlV3e/641adef615d6af9e5a3b588ff397af87/dashboard-apis-edit_view-settings_rbac-settings.png)</Frame>

* **RBAC を有効にする**: この設定を有効にすると、API に RBAC ポリシーが適用されます。詳しくは、[ロールベースのアクセス制御](/ja/docs/manage-users/access-control/rbac) と [API でロールベースのアクセス制御を有効にする](/ja/docs/get-started/apis/enable-role-based-access-control-for-apis) を参照してください。
* **アクセストークンに Permissions を追加する**: この設定を有効にすると、Permissions claim がアクセストークンに追加されます。この設定は、RBAC を有効にした場合にのみ利用できます。権限は **Permissions** タブで設定できます。

<div id="access-settings">
  ### アクセス設定
</div>

<Frame>![Auth0 Dashboard アプリケーション API 設定タブ アクセス設定](/docs/images/cdy7uua7fh8z/JNlWACqoyfmpE89JwWLKy/5a2b595f939167b0e6e1f72b485b08e7/2025-02-25_14-53-21.png)</Frame>

* **ユーザー同意のスキップを許可**: この設定を有効にすると、「ファーストパーティ」として設定されたアプリケーションでは、API でユーザーの同意を省略できます。
* **オフラインアクセスの許可**: この設定を有効にすると、アプリケーションが API に対して <Tooltip tip="Refresh Token: ユーザーに再度ログインを求めることなく、新しいアクセストークンを取得するために使用されるトークン。" cta="用語集を見る" href="/ja/docs/glossary?term=refresh+tokens">リフレッシュトークン</Tooltip> を要求できるようになります。

<div id="default-policy-setting">
  #### デフォルトポリシーの設定
</div>

<Warning>
  **My Account API** の認証保証は現在、単一オプションのポリシーによる早期アクセスとして提供されています。この機能を使用することで、お客様は Okta の [Master Subscription Agreement](https://www.okta.com/legal) に定める該当する Free Trial 条項に同意したものとみなされます。Auth0 の製品リリースサイクルの詳細については、[Product Release Stages](/ja/docs/troubleshoot/product-lifecycle/product-release-stages) をご覧ください。プログラムに参加するには、[Auth0 Support](https://support.auth0.com) にお問い合わせください。
</Warning>

<Frame>![デフォルトポリシー API 設定](/docs/images/api-settings-access-default-policy.png)</Frame>

* **2FA を必須にする**: 有効にすると、Auth0 は My Account API にデフォルトポリシーを適用し、[ステップアップ認証](/ja/docs/secure/multi-factor-authentication/step-up-authentication) をトリガーします。ユーザーは 15 分以内に第 2 の認証要素で認証する必要があります。動作の詳細と有効化の方法については、[My Account API: デフォルトポリシー](/ja/docs/manage-users/my-account-api#default-policy) を参照してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  デフォルトポリシーはクラシックログインでは利用できません。
</Callout>

<div id="permissions">
  ## 権限
</div>

**Permissions** タブの設定を使って、この API で使用する権限 (スコープ) を定義します。詳しくは、[API の権限を追加する](/ja/docs/get-started/apis/add-api-permissions) と [API の権限を削除する](/ja/docs/get-started/apis/delete-api-permissions) をご覧ください。

<Frame>![Auth0 Dashboard API の権限を追加する API 権限の定義画面](/docs/images/cdy7uua7fh8z/32Pb185OFs2mC6z2fmunEw/2708c52c2869d016066cc456dd00b6a2/dashboard-applications-apis-permissions.png)</Frame>

<div id="machine-to-machine-applications">
  ## Machine-to-Machine アプリケーション
</div>
