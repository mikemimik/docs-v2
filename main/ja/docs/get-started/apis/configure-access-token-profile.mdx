export const codeExample = `curl --request PATCH \\
  --url 'https://{yourDomain}/api/v2/resource-servers/API_ID' \\
  --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \\
  --header 'content-type: application/json' \\
  --data '{"token_dialect": "TOKEN_DIALECT" }'
`;

使用する API の <Tooltip tip="アクセストークン: API へのアクセスに使用される認可資格情報で、opaque な文字列または JWT の形式を取ります。" cta="用語集を見る" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip>プロファイルは、Auth0 トークンプロファイルまたは RFC 9068 トークンプロファイルから選択できます。設定するアクセストークンプロファイルによって、その API 用に発行されるアクセストークンの形式が決まります。

デフォルトでは、Auth0 は Auth0 トークンプロファイルを使用してアクセストークンを発行します。Auth0 トークンプロファイルでは、アクセストークンは[JSON Web トークン (JWTs) ](/ja/docs/secure/tokens/json-web-tokens)形式で発行され、エンティティに関する情報をクレームとして含みます。

Auth0 は RFC 9068 トークンプロファイルもサポートしています。RFC 9068 トークンプロファイルでは、[IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068) に準拠した <Tooltip tip="JSON Web トークン (JWT): 2 者間でクレームを安全に表現するために使われる標準的な ID トークン形式（および多くの場合アクセストークン形式）です。" cta="用語集を見る" href="/ja/docs/glossary?term=JWTs">JWT</Tooltip> 形式のアクセストークンが発行されます。これらのトークンプロファイルの違いについて詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。

[API を登録する](/ja/docs/get-started/auth0-overview/set-up-apis)際には、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> でアクセストークンプロファイルを選択できます。API の登録後は、[Management API](https://auth0.com/docs/api/management/v2) と [Auth0 Dashboard](https://manage.auth0.com/) を使用して、いつでもアクセストークンプロファイルを設定できます。

<div id="configure-access-token-profile-for-an-api">
  ## API のアクセストークンプロファイルを設定する
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) に移動し、詳細を表示する API の名前をクリックします。

    2. **Access Token Settings** までスクロールし、**JSON Web Token (JWT) Profile** でアクセストークンプロファイルを選択します。選択したプロファイルによって、その API に対して発行されるアクセストークンの形式とクレームが決まります。サポートされている値は `Auth0` と `RFC 9068` です。2 つのプロファイルの違いについて詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。

    3. **Save** をクリックします。
  </Tab>

  <Tab title="Management API">
    Management API では、アクセストークンプロファイルをトークンダイアレクトと呼びます。アクセストークンプロファイルを設定するには、Management API を使用して API の `token_dialect` パラメーターを設定します。

    次のコードサンプルは、[Update a resource server endpoint](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id) に PATCH リクエストを送信します。

    次の表の説明に従って、`API_ID`、`MGMT_API_ACCESS_TOKEN`、`TOKEN_DIALECT` をそれぞれ対応する値に置き換えます。

    | Parameter               | Description                                                                                                                 |
    | ----------------------- | --------------------------------------------------------------------------------------------------------------------------- |
    | `API_ID`                | トークンダイアレクトを更新する対象 API の ID。                                                                                                 |
    | `MGMT_API_ACCESS_TOKEN` | scope `update:resource_servers` を持つ [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens)。 |
    | `TOKEN_DIALECT`         | 指定した API のアクセストークンのダイアレクト。詳しくは、[Token dialect options](#token-dialect-options) を参照してください。                                   |

    #### トークンダイアレクトのオプション

    Auth0 では、次のトークンダイアレクトをサポートしています。

    | Value                   | Description                                                                                                                                                                                                                                                |
    | ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `access_token`          | デフォルトの Auth0 トークンプロファイルは、JSON Web トークン (JWT) 形式のアクセストークンを生成します。詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。                                                                                            |
    | `access_token_authz`    | `permissions` クレームを含むデフォルトの Auth0 トークンプロファイル (`access_token`)。RBAC の permissions について詳しくは、[Enable Role-Based Access Control for APIs](/ja/docs/get-started/apis/enable-role-based-access-control-for-apis) を参照してください。                                         |
    | `rfc9068_profile`       | RFC 9068 トークンプロファイルは、[IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068) に準拠した JWT 形式のアクセストークンを生成します。詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。 |
    | `rfc9068_profile_authz` | `permissions` クレームを含む RFC 9068 プロファイル (`rfc9068_profile`)。RBAC の permissions について詳しくは、[Enable Role-Based Access Control for APIs](/ja/docs/get-started/apis/enable-role-based-access-control-for-apis) を参照してください。                                             |
  </Tab>
</Tabs>
