---

Auth0 Dashboard では、アプリケーションを手動で作成することも、[Client ID Metadata Document (CIMD)](/ja/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd) の URL からインポートすることもできます。

<div id="create-manually">
  ## 手動で作成
</div>

アプリケーションを手動で作成すると、Auth0 はそのアプリケーションに <Tooltip tip="Client ID: アプリケーションの一意の識別子として機能する英数字の文字列。" cta="用語集を見る" href="/ja/docs/glossary?term=client+ID">client ID</Tooltip> を割り当てます。これは、アプリケーションを一意に識別する英数字の文字列です。Auth0 API を呼び出すときは、アプリケーションのコード内でこの ID を使用します。このアプリケーション登録プロセスは、手動クライアント登録と呼ばれます。

もう 1 つの重要な情報は、<Tooltip tip="Client Secret: クライアント（アプリケーション）が認可サーバーに対して認証を行うために使用する Secret です。これはクライアントと認可サーバーのみが知っているべきもので、推測されないよう十分なランダム性を備えている必要があります。" cta="用語集を見る" href="/ja/docs/glossary?term=client+secret">Client Secret</Tooltip> です。これは常に機密として扱わなければなりません。第三者がアプリケーションの client secret にアクセスすると、その第三者はアプリケーションになりすまして保護されたリソースにアクセスできてしまいます。特定のセキュリティリスクを Auth0 が確認できるよう、適切なアプリケーションの種類を選択することが重要です。

<div id="import-from-url">
  ## URLからインポート
</div>

URL から外部でホストされている Client ID Metadata Document (CIMD) をインポートして、アプリケーションを登録できます。CIMD は、アプリケーションが管理する安全な HTTPS ドメインでホストされる、クライアントメタデータを含む JSON ファイルです。このアプリケーション登録プロセスは、手動CIMD登録と呼ばれます。詳しくは、[Register Applications with CIMD](/ja/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd) をご覧ください。

<div id="manual-client-registration-vs-manual-cimd-registration">
  ## 手動クライアント登録と手動CIMD登録の比較
</div>

次の表では、手動クライアント登録と手動CIMD登録を比較しています。

| 項目              | 手動クライアント登録              | 手動CIMD登録                                             |
| --------------- | ----------------------- | ---------------------------------------------------- |
| **Client ID**   | Auth0 によって生成            | HTTPS URL (例: `https://yourdomain.com/client.json`)  |
| **認証**          | Client secret または証明書    | 非対称キーのみ (シークレットなし)                                   |
| **検証方法**        | なし                      | HTTPS ホスティングによるドメイン所有権の確認                            |
| **メタデータ管理**     | Auth0 Dashboard/API で更新 | CIMD が正となる情報源                                        |
| **キーのローテーション**  | 手動、単一キーのみ               | `jwks_uri` で複数キーに対応                                  |
| **アプリケーションの種類** | すべての種類                  | サードパーティアプリケーションのみ                                    |
| **最適な用途**       | ファーストパーティアプリ、社内サービス     | MCP クライアント、サードパーティ連携                                 |

<div id="create-application-in-auth0">
  ## Auth0でアプリケーションを作成
</div>

Auth0でアプリケーションを作成するには、次の手順に従います。

1. [Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications) に移動します。
2. **Create Application** を選択します。
3. **手動で作成する** または **Import from URL** を選択します。
   * **Import from URL**: [Register Applications with CIMD](/ja/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd) の手順に従います。
   * **手動で作成する**: 新しいアプリケーションにわかりやすい名前を付けます。次に、アプリの種類を選択します。
     * **ネイティブアプリケーション**: モバイル、デスクトップ、ハイブリッドアプリなど、デバイス上でネイティブに動作するアプリケーションです (例: iOS、Android) 。
     * **シングルページWebアプリケーション**: ユーザーインターフェースロジックの大部分をWebブラウザー内で実行し、主にAPIを使ってWebサーバーと通信するJavaScriptアプリケーションです (例: AngularJS + Node.js、React) 。
     * **従来型Webアプリケーション**: アプリケーションロジックの大部分をサーバー側で実行する従来型のWebアプリケーションです (例: Express.js、ASP.NET) 。
     * **マシンツーマシンアプリケーション**: コマンドラインツール、デーモン、IoTデバイス、バックエンドで実行されるサービスなど、ユーザー操作を伴わないアプリケーションです。
4. **Create** を選択します。
