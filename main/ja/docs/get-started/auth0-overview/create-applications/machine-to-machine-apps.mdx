Auth0 をマシン間 (M2M) アプリケーションと統合するには、まず <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要な製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> を使用してアプリを Auth0 に登録する必要があります。これらのアプリには、コマンドラインツール、デーモン、IoT デバイス、バックエンドで実行されるサービスなどの非対話型アプリケーションが含まれます。

M2M アプリは API とその permissions または [scopes](/ja/docs/get-started/apis/scopes/api-scopes) に関連付けられており、これらは事前定義された値から選択します。この登録を始める前に、関連する API をすでに Auth0 に[登録](/ja/docs/get-started/auth0-overview/set-up-apis)し、その API の scopes を定義していることを確認してください。

アプリケーションが Auth0 の <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> のみにアクセスできるよう認可したい場合は、別の API を登録する必要はありません。Management API はあらかじめ用意されています。

Auth0 をマシン間 (M2M) アプリケーションと統合するには、まず Auth0 Dashboard を使用してアプリを Auth0 に登録する必要があります。

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、**Create Application** をクリックします。
2. アプリケーションが分かりやすい名前を入力し、**Machine to Machine Applications** を選択して、**Create** をクリックします。
3. アプリケーションから呼び出せるようにする API を選択します。
4. API にアクセスする各 M2M アプリには、認証されたユーザーによって付与される permissions (または scopes) のセットを付与する必要があります。permissions の追加方法については、[API の権限を追加する](/ja/docs/get-started/apis/add-api-permissions) を参照してください。アプリケーションの access token の一部として発行したい **Permissions** を選択し、**Authorize** をクリックします。

   <Warning>
     `update:client_grants` scope と `create:client_grants` scope は、クライアントが自身にさらに permissions を付与できるようになるため、高い権限を持つアプリケーションでのみ使用してください。
   </Warning>

   新しいアプリケーションが作成されると、**Application Settings** を設定できます。これには次のタブが含まれます。

   | Settings Tab    | Description                                                                                                                                                                                                |
   | --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start** | ${application_type} アプリケーションで利用可能なすべてのサンプルを表示します。また、さまざまな技術を使って API を呼び出す方法も確認できます。API で Access Tokens を受け入れて検証する方法については、[Backend/API Quickstarts](/ja/docs/quickstart/backend/nodejs/interactive) を参照してください。 |
   | **Settings**    | アプリケーションで利用可能なすべての設定を表示します。これらの設定の大半は、デフォルトで Auth0 によって作成されます。                                                                                                                                             |
   | **Credentials** | アプリケーションの認証方法と設定済みの認証情報を表示します。詳細については、[Application Credentials](/ja/docs/secure/application-credentials) を参照してください                                                                                            |
   | **APIs**        | Application で使用する追加の API を認可できます。                                                                                                                                                                          |

   詳細については、[Application Settings](/ja/docs/get-started/applications/application-settings) を参照してください。

<div id="next-steps">
  ## 次のステップ
</div>

アプリケーションの登録と設定が完了したら、一般的な次のステップは次のとおりです。

* ID プロバイダー接続を設定し、アプリケーションで有効にします (ユースケースに応じて必要な場合) 。
* Auth0 に登録したアプリケーションを利用するように、アプリのコードを変更します。さまざまな技術向けの詳細な手順やサンプルが掲載されている [Auth0 Quickstarts](/ja/docs/quickstarts) をご覧ください。また、ログインやログアウトの実装、ユーザーセッションの処理、ユーザープロフィール情報の取得と表示などについても学べます。
* [Auth0 APIs](/ja/docs/api) を利用します。

  * [Authentication API](https://auth0.com/docs/api/authentication) は、主要な ID 関連機能 (たとえば、ログイン、ログアウト、ユーザープロフィールの取得) をすべて処理します。ほとんどのユーザーは、Quickstarts、[Auth0.js library](/ja/docs/libraries/auth0js)、または [Lock widget](/ja/docs/libraries/lock) を通じてこの API を利用します。ただし、認証 UI をすべて手動で実装する場合は、この API を直接操作する必要があります。
  * [Management API](https://auth0.com/docs/api/management/v2) を使うと、Auth0 の Dashboard からも行えるさまざまな作業 (たとえば、ユーザーの作成やアプリケーションの grant type の設定) を自動化できます。
