title: 従来型Webアプリケーションを登録する
validatedOn: 2026-02-25
---

従来型Webアプリを Auth0 と統合するには、まず <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> を使用して、アプリを Auth0 に登録する必要があります。従来型Webアプリは、アプリケーション ロジックの大部分をサーバー側で実行する従来型の Web アプリケーションです (例: Express.js、ASP.NET) 。アプリケーションの種類について詳しくは、[Application Types](/ja/docs/get-started/applications) を参照してください。

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、**Create Application** をクリックします。
2. アプリケーションがわかりやすい名前を入力し、**従来型Webアプリケーション** を選択して、**Create** をクリックします。

   新しいアプリケーションが作成されると、次のタブを含む **アプリケーション設定** ページを表示できます。

   | Settings Tab         | Description                                                                                                                                                                                                                                                                                                                 |
   | -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**      | アプリケーション タイプで利用できるすべてのドキュメントを表示します。                                                                                                                                                                                                                                                                                         |
   | **Settings**         | アプリケーションで利用可能なすべての設定を表示します。デフォルトでは、ほとんどの設定は自動的に作成されます。詳しくは、[アプリケーション設定](/ja/docs/get-started/applications/application-settings) を参照してください。                                                                                                                                                                           |
   | **Credentials**      | アプリケーションの認証方式と設定済みの認証情報を表示します。詳しくは、[Application Credentials](/ja/docs/secure/application-credentials) を参照してください                                                                                                                                                                                                                |
   | **Add-ons**          | アプリケーションに関連付けられたプラグインを有効にできます。これらは、Auth0 が access token を生成する SAML または WS-Fed の Web アプリです。詳しくは、[Enable SAML2 Web App Addon](/ja/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) と [Configure WS-Fed Applications](/ja/docs/get-started/applications/update-application-connections) を参照してください。 |
   | **Connections**      | アプリケーションのコネクションを有効にできます。コネクションはユーザーのソースであり、複数のアプリケーション間で有効化して共有できます。詳しくは、[Connections](/ja/docs/authenticate/identity-providers) を参照してください。                                                                                                                                                                                    |
   | **Organizations**    | ユーザーを、そのユーザーが所属する Organization に対して認証できます。詳しくは、[Organizations](/ja/docs/manage-users/organizations) を参照してください。                                                                                                                                                                                                                 |
   | **Login Experience** | Organization 内のユーザー向けログイン体験を設定できます。詳しくは、[Understand how Auth0 Organizations Work](/ja/docs/manage-users/organizations/organizations-overview#login-experience) を参照してください。                                                                                                                                                      |

   詳しくは、[アプリケーション設定](/ja/docs/get-started/applications/application-settings) を参照してください。
3. 下にスクロールして **Trust Token Endpoint IP Header** 設定を探します。アプリケーション アーキテクチャでサポートされている場合は、この設定を有効にして **Save Changes** をクリックします。この設定により、Auth0 はプロキシからの IP ヘッダーを信頼できるようになり、ブルートフォース攻撃からの保護に役立ちます。

<div id="next-steps">
  ## 次のステップ
</div>

アプリケーションの登録と設定が完了したら、次は以下の手順に進んでください。

1. 接続を設定し、アプリケーションで有効化します。
2. Auth0 に登録したアプリケーションを使って、アプリのコードに認証を実装します。詳しい手順やさまざまな技術向けのサンプルは、[Auth0 Quickstarts](/ja/docs/quickstarts) を参照してください。ログインとログアウトの実装方法、ユーザーセッションの処理、ユーザープロフィール情報の取得と表示などについても学べます。

* [Auth0 APIs](/ja/docs/api) を利用します。

  * [Authentication API](https://auth0.com/docs/api/authentication) は、主要な ID 関連の機能 (たとえば、ログイン、ログアウト、ユーザープロフィールの取得) をすべて担います。ほとんどのユーザーは、Quickstarts、[Auth0.js library](/ja/docs/libraries/auth0js)、または [Lock widget](/ja/docs/libraries/lock) を通じてこの API を利用します。ただし、認証 UI をすべて手動で構築する場合は、この API を直接利用できます。
  * [Management API](https://auth0.com/docs/api/management/v2) を使うと、Auth0 Dashboard からも行えるさまざまな作業 (たとえば、ユーザーの作成や application grant type の設定) を自動化できます。
