description: Auth0 Dashboard を使用してネイティブアプリケーションを登録および設定する方法を説明します。
title: ネイティブアプリケーションを登録する
validatedOn: 2026-02-25
---

Auth0 をネイティブアプリケーションと統合するには、まず <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> を使用してアプリを Auth0 に登録する必要があります。ネイティブアプリでは、安全に認証を行うために PKCE を使用した Authorization Code フローを使用してください。詳しくは、[Authorization Code Flow with PKCE](/docs/ja/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) をご覧ください。これらのアプリには、デバイス上でネイティブに動作するモバイル、デスクトップ、ハイブリッドの各アプリ (たとえば iOS、Android) が含まれます。

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、**Create Application** をクリックします。
2. アプリケーションがわかりやすい名前を入力し、**Native** を選択して、**Create** をクリックします。

   アプリケーションを作成すると、Auth0 によって **アプリケーション設定** に移動します。ここには次のタブがあります。

   | Settings Tab         | Description                                                                                                                                                                                                                                                                                                                  |
   | -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**      | アプリケーションタイプで利用できるすべてのドキュメントを表示します。                                                                                                                                                                                                                                                                                           |
   | **Settings**         | アプリケーションで利用可能なすべての設定を表示します。既定では、ほとんどの設定は自動的に作成されます。詳しくは、[アプリケーション設定](/docs/ja/get-started/applications/application-settings) をご覧ください。                                                                                                                                                                                 |
   | **Credentials**      | アプリケーションの認証方法と設定済みの認証情報を表示します。詳しくは、[Application Credentials](/docs/ja/secure/application-credentials) をご覧ください                                                                                                                                                                                                                   |
   | **Add-ons**          | アプリケーションに関連付けられたプラグインを有効にできます。Add-ons は主に SAML または WS-Fed の統合に使用され、通常はネイティブアプリケーションには適用されません。詳しくは、[Enable SAML2 Web App Addon](/docs/ja/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) と [Configure WS-Fed Applications](/docs/ja/get-started/applications/update-application-connections) をご覧ください。 |
   | **Connections**      | アプリケーションのコネクションを有効にできます。コネクションはユーザーソースであり、複数のアプリケーション間で有効化して共有できます。詳しくは、[Connections](/docs/ja/authenticate/identity-providers) をご覧ください。                                                                                                                                                                                        |
   | **Organizations**    | ユーザーが所属する組織にログインできるようにします。詳しくは、[Organizations](/docs/ja/manage-users/organizations) をご覧ください。                                                                                                                                                                                                                                    |
   | **Login Experience** | Organization 内のユーザー向けのログイン体験を設定できます。詳しくは、[Understand how Auth0 Organizations Work](/docs/ja/manage-users/organizations/organizations-overview#login-experience) をご覧ください。                                                                                                                                                        |

   詳しくは、[アプリケーション設定](/docs/ja/get-started/applications/application-settings) をご覧ください。
3. モバイルアプリを開発している場合は、**Advanced Settings** で必要な iOS/Android パラメーターを入力し、**Save Changes** をクリックします。

   * iOS アプリの場合は、[**Team ID** と **App Bundle Identifier** を入力します](/docs/ja/get-started/applications/enable-universal-links-support-in-apple-xcode)。
   * Android アプリの場合は、[**App Package Name** と **Key Hashes** を入力します](/docs/ja/get-started/applications/enable-android-app-links-support)。

<div id="next-steps">
  ## 次のステップ
</div>

アプリケーションの登録と設定が完了したら、一般的な次のステップは次のとおりです。

* 接続を設定し、アプリケーションで有効にします。
* Auth0 に登録したアプリケーションを使用するように、アプリのコードを変更します。詳しい手順やさまざまな技術向けのサンプルは、[Auth0 Quickstarts](/docs/ja/quickstarts)をご覧ください。ログインやログアウトの実装方法、ユーザーセッションの処理、ユーザープロフィール情報の取得と表示などについても学べます。
* [Auth0 APIs](/docs/ja/api)を利用します。

  * [Authentication API](https://auth0.com/docs/api/authentication) は、主要なID関連の機能 (たとえば、ログイン、ログアウト、ユーザープロフィールの取得) をすべて担います。ほとんどのユーザーは、Quickstarts、[Auth0.js library](/docs/ja/libraries/auth0js)、または [Lock widget](/docs/ja/libraries/lock) を通じてこの API を利用します。ただし、認証 UI をすべて手作業で構築する場合は、この API を直接利用する必要があります。
  * [Management API](https://auth0.com/docs/api/management/v2) を使用すると、Auth0 の Dashboard からも行えるさまざまなタスク (たとえば、ユーザーの作成や application grant タイプの設定) を自動化できます。
