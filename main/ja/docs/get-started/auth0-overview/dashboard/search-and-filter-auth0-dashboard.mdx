Auth0 では、Auth0 Dashboard 上で検索可能なフィールドに対するフィルター式を作成できる、登録済みのアプリケーションと API 向けの、より直感的な検索機能をテナントユーザーに提供しています。これにより、数百ものリソースが並ぶページ分割された一覧をスクロールする代わりに、テナントユーザーは次のようなフィールドでクエリを実行できます。

* アプリケーション名 (Travel0-native)
* アプリケーションまたはユーザーのメタデータ (permissions)
* アプリケーションの種類 (SPA)
* API 名
* API ID

Auth0 Dashboard では、登録されているすべてのリソースから一致する結果がリアルタイムで表示されます。

<div id="how-it-works">
  ## 仕組み
</div>

Auth0 Dashboard の [アプリケーション](https://manage.auth0.com/#/applications) セクションと [APIs](https://manage.auth0.com/#/apis) セクションの検索バーでは、ガイド付きのフィルターインターフェースとブール検索を使用します。検索バーは、フィルタリング可能なフィールドを一覧表示するドロップダウンメニューとして機能します。フィールドを選択して値を入力すると、その値が結果の上部に削除可能なタグとして適用されます。組み合わせられるフィルターは一度に最大 5 つです。

フィルターを追加するたびに、結果はさらに絞り込まれます。結果を絞り込むには、異なるフィールドの間で `AND/OR` ロジックを使用します。フィルターは URL にクエリパラメータとして反映されます。URL はいつでもコピーしてチームメイトと共有したり、ブックマークして後で同じフィルター表示に戻ったりできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **ブール検索の例**

  **アプリケーションの種類** `Native` OR `SPA` と **ファーストパーティ クライアント** `true` でフィルターすると、結果には Native または SPA で、かつファーストパーティのアプリケーションのみが表示されます。Regular Web Apps やサードパーティのアプリケーションは除外されます。
</Callout>

<div id="search-behavior-by-field-type">
  ### フィールドタイプごとの検索動作
</div>

Auth0 Dashboard の検索では、フィールドタイプに応じて結果が照合されます。

* **String フィールド** (name、identifier) : 部分一致に対応しています。`payments` と入力すると、「Payments-application」や「Corp Payments Service」などが見つかります。語順は関係ありません。特定のタームに一致する結果を除外するには、先頭に `NOT`、`-`、または `No` を付けます。たとえば、name フィールドに `NOT test` と入力すると、名前に「test」を含まないすべてのリソースが返されます。ワイルドカード検索には 3 文字以上が必要です。それより少ない文字数を入力すると、「Minimum 3 characters」というメッセージが表示され、クエリは送信されません。

* **完全一致フィールド**

ID の完全な値を入力する必要があります。ID の一部だけを入力しても結果は返されません。ログやサポートチケットに記載された特定の Client ID がある場合は、これらのフィールドを使用します。

* **メタデータフィールド**

メタデータは、キーと値の組み合わせ、またはキーのみで検索できます。

* **キーと値**: キーと値を入力すると、そのキーがその値に完全一致するリソースを検索できます。たとえば、キー `environment` と値 `production` を指定すると、`app_metadata.environment` が `production` と等しいリソースだけが返されます。値については前方一致もサポートされています。`prod` と入力すると、`production`、`prod-eu`、`prod-us` に一致します。
  * **キーのみ**: 値を入力せずにキーを入力して **Enter** を押します。すると、そのメタデータキーが設定されているすべてのリソースが、値に関係なく返されます。特定のキーが使われているリソースを探すのに便利です。

また、ドット記法を使うことで、最大 2 階層までのネストされたメタデータキーも検索できます。たとえば、キー `tier.bronze` と値 `true` は、`app_metadata.tier.bronze` が `true` と等しいリソースに一致します。

特定のキーと値の組み合わせに一致する結果を除外するには、値の先頭に `NOT` を付けます。たとえば、キー `color` と値 `NOT orange` を指定すると、`color` が `orange` 以外に設定されているすべてのリソースが返されます。キーは完全一致である必要があります。メタデータでは contains や ends-with による一致はサポートされていません。値は最大 64 文字までインデックス化され、それより長い値は検索できません。

* **列挙型フィールド**

定義済みリストから 1 つ以上のアプリケーションタイプを選択します。同じフィルター内で複数選択した場合は OR 条件として扱われます。たとえば、`Native` と `SPA` を選択すると、すべての Native アプリケーションと SPA アプリケーションが返されます。

* **Boolean フィールド**

`true` または `false` を選択して、ファーストパーティのクライアントを検索します。これにより、ファーストパーティ アプリケーション (同意画面をスキップすることが許可されているもの) とサードパーティのものを区別できます。

<div id="searchable-fields">
  ### 検索対象のフィールド
</div>

<div id="applications">
  #### アプリケーション
</div>

| **項目**                                        | **説明**                                            | **一致タイプ**   |
| --------------------------------------------- | ------------------------------------------------- | ----------- |
| **名前**                                        | アプリケーションの名前                                       | 部分一致        |
| **Client ID** (`client_id`)                   | 一意のクライアントID                                       | 完全一致        |
| **External client ID** (`external_client_id`) | アプリケーションの外部識別子                                    | 完全一致        |
| **メタデータ**                                     | アプリケーションに保存されたカスタムのキーと値のメタデータ                     | 完全一致または前方一致 |
| **アプリケーションの種類**                               | Native、SPA、Regular Web App、または Machine to Machine | 列挙値の選択      |
| **ファーストパーティ クライアント**                          | アプリケーションがファーストパーティかサードパーティか                       | ブール値        |

<div id="apis">
  #### API
</div>

| **項目**  | **説明**                                           | **一致の種類** |
| ------- | ------------------------------------------------ | --------- |
| **名前**  | API の名前                                          | 部分一致      |
| **識別子** | 一意の audience 識別子 (例: `https://api.example.com`)  | 部分一致      |
| **Id**  | システム内部の ID                                       | 完全一致      |

<div id="limitations">
  ### 制限事項
</div>

* 1 回の検索につき適用できるフィルターは最大 5 つです。検索バーでは 6 つ目のフィルターは追加できません。
* インデックスは最終的整合性を採用しているため、検索結果に直近数秒の変更が反映されない場合があります。
* 通常は 2 秒以内に結果が返されます。クエリがタイムアウトする場合は、フィルターの数を減らすか、より具体的な検索タームを使用してください。

<div id="example-queries">
  ## クエリ例
</div>

<div id="find-an-application-by-client-id">
  ### クライアントIDでアプリケーションを見つける
</div>

エラーログやサポートチケットにある `client_id` を使って、該当するアプリケーションをすばやく特定したい場合に使用します。

1. **[Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications)** に移動します。
2. 検索バーを選択し、**Client ID** を選びます。
3. `client_id` の値全体を貼り付けます。一致するアプリケーションがすぐに表示されます。

<div id="find-applications-using-metadata">
  ### メタデータを使ってアプリケーションを検索する
</div>

アプリケーションに環境ラベル、プランの種類、外部 ID などのカスタム メタデータが保存されている場合は、それらの値で検索できます。

1. **[Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications)** に移動します。
2. 検索バーを選択し、**メタデータ** を選びます。
3. メタデータのキー (例: `environment`) を入力します。
4. 値 (例: `production`) を入力します。`metadata.environment` が `production` と一致するアプリケーションが結果に表示されます。プレフィックスで一致させるには、`prod` のような値の一部を入力します。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

**結果が見つかりません**

フィルター値のスペルが正しいことを確認してください。文字列フィールドの場合は、より短いターム、またはより一般的なタームを試してください。メタデータフィールドの場合は、正確なキー名を確認してください。キーは大文字と小文字が区別されるため、完全に一致している必要があります。メタデータでは contains 形式の照合はサポートされていないため、代わりに接頭辞を使用してください。アプリケーション ID のような完全一致フィールドでは、ID の完全な値を貼り付けていることを確認してください。

**結果が古いようです**

検索結果の反映には時間差があるため、ここ数秒以内に行われた変更が表示されない場合があります。最近リソースを作成または更新した場合は、数秒待ってから再度検索してください。

**検索がタイムアウトしました**

有効なフィルターの数を減らすか、より具体的な検索タームを使用してください。問題が解決しない場合は、[Auth0 Support](https://support.auth0.com) にお問い合わせください。
