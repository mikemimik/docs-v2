Auth0 Model Context Protocol (MCP) Server は、AI エージェントをお使いの Auth0 テナントに接続し、アプリケーションの作成、ユーザー管理、Actions のデプロイなど、Auth0 での複雑な複数ステップの操作を実行できるようにします。これは、AI エージェントが外部ツールや API と通信してユーザーに代わってタスクを実行できるようにする標準化されたプロトコル、[Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction) を活用しています。

Cursor、Windsurf、Claude Desktop などの AI 搭載ツールを使う開発者向けに設計された Auth0 MCP Server は、開発者のアイデンティティ インフラストラクチャ管理のあり方を一変させ、Auth0 サービスをよりすばやく簡単に扱えるようにします。

<div id="authentication-and-credential-security">
  ## 認証と資格情報のセキュリティ
</div>

Auth0 MCP Server では、認証の管理、request の検証、Auth0 <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> との安全な通信の確保を通じて、高度なセキュリティを実現しています。これには次が含まれます。

* [OAuth 2.0 Device Authorization Flow](/ja/docs/get-started/authentication-and-authorization-flow/device-authorization-flow): Auth0 への request を安全に認証します。
* [資格情報の安全な保存](/ja/docs/get-started/applications/rotate-credentials): 資格情報はシステムのキーチェーンに安全に保存され、平文で公開されることはありません。
* [最小限の API permissions](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens): 必要最小限の permissions のみを request し、最小権限の原則を厳格に順守します。

<div id="supported-tools-and-integrations">
  ## サポート対象のツールと連携
</div>

Auth0 MCP Server は現在、以下を含む [MCP クライアント](https://modelcontextprotocol.io/clients) と連携しています。

* [Claude Desktop](https://claude.ai/download)
* [Cursor](https://www.cursor.com/downloads)
* [Windsurf](https://www.windsurf.com/editor)

今後、主要な新規クライアントの登場に合わせて、連携先が順次追加される予定です。サポート対象ツールの完全な一覧と各機能の詳細については、[Auth0 Model Context Protocol (MCP) Server Tools Reference](/ja/docs/get-started/auth0-mcp-server/auth0-mcp-tools-reference) を参照してください。
