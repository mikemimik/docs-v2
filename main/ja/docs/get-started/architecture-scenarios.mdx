Auth0を利用するお客様には、共通する目標や課題があるのが一般的です。そこで、実際のお客様への導入経験を共有し、効率的なソリューションの実現に役立てていただけるようにしました。

<div id="application-architecture-scenarios">
  ## アプリケーション アーキテクチャのシナリオ
</div>

これらのシナリオでは、アプリケーションで利用できるさまざまな技術アーキテクチャと、Auth0 がどのように役立つかを紹介します。これらのシナリオの目的は、実装プロセスを最初から最後まで順を追って説明することです。

| 読む項目...                                                                                           | 学べること...                                                                               |
| ------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| [複数の組織](/ja/docs/get-started/architecture-scenarios/multiple-organization-architecture)              | 単一のアイデンティティプロバイダーを使用する組織、または複数のアイデンティティプロバイダーを使用する組織のいずれかを利用するアプリケーション アーキテクチャを設定する方法。 |
| [Regular Web Apps 向けシングルサインオン](/ja/docs/get-started/architecture-scenarios/sso-for-regular-web-apps) | Active Directory を使用する従業員と、別のユーザー ストアを使用する契約業者向けに、SSO を使用する Web アプリケーションを設定して実装する方法。   |
| [サーバー アプリケーション + API](/ja/docs/get-started/architecture-scenarios/server-application-api)            | cron ジョブを使用してタイムシート エントリを記録するタイムシート API を設定する方法。                                       |
| [シングルページアプリケーション + API](/ja/docs/get-started/architecture-scenarios/spa-api)                         | シングルページアプリケーションを使用してタイムシート エントリを記録するタイムシート API を設定する方法。                                |
| [モバイル アプリケーション + API](/ja/docs/get-started/architecture-scenarios/mobile-api)                        | モバイル アプリケーションを使用して、一元化されたデータベースにタイムシート エントリを記録するタイムシート API を設定する方法。                    |
