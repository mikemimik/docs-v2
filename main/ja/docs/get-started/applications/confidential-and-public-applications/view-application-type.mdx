title: アプリケーションが Confidential か Public かを確認する
---

アプリケーションが、confidential アプリケーションとして Auth0 に登録されているか、または public アプリケーションとして登録されているかを確認できます。詳細については、[Confidential and Public Application](/ja/docs/get-started/applications/confidential-and-public-applications)を参照してください。

Auth0 では、アプリケーションが Auth0 Authentication API の [Get Token](https://auth0.com/docs/api/authentication/authorization-code-flow/get-token) endpoint に対してどのように認証するかを定義する **Authentication Method** 設定に基づいて、そのアプリケーションが confidential か public かを判断します。

1. Auth0 Dashboard で [Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、表示したいアプリケーションの名前を選択します。
2. **Credentials** ビューが表示されない場合、そのアプリケーションは public アプリケーションです。
3. **Credentials** ビューが表示される場合は、それを選択して **Authentication Method** フィールドを確認します。

   <Frame>![Private Key JWT Authentication を設定する - Auth0 Dashboard の手順](/docs/images/cdy7uua7fh8z/33kfi48tkbMIOQJ8PBxj76/5b79ffda11ad58b27128068057e6e05c/Default_App_-_Creds_-_English.png)</Frame>

該当する方式は次のとおりです。

* **None**: <Tooltip tip="Client Secret: クライアント（アプリケーション）が Authorization Server に対して認証するために使用するシークレット。クライアントと Authorization Server だけが知っている必要があり、推測できないよう十分にランダムでなければなりません。" cta="用語集を表示" href="/ja/docs/glossary?term=client+secret">client secret</Tooltip> を持たない public アプリケーション。
* **Client Secret Post**: POST リクエストの本文パラメーターを使って client secret を送信するアプリケーション。
* **Client Secret Basic**: HTTP BASIC 認証スキームを使って client secret を送信するアプリケーション。
* **Private Key <Tooltip tip="JSON Web Token (JWT): 2 者間で claim を安全に表現するために使用される標準の ID Token 形式（および多くの場合 Access Token 形式）。" cta="用語集を表示" href="/ja/docs/glossary?term=JWT">JWT</Tooltip>**: 非対称認証を使用するアプリケーション。

これらの値は、次のように confidential アプリケーションと public アプリケーションに対応します。

| アプリケーションの種類      | 例                              | Token Endpoint Authentication Method                      |
| ---------------- | ------------------------------ | --------------------------------------------------------- |
| **Public**       | シングルページまたはネイティブ                | **None**                                                  |
| **Confidential** | 通常の Web または machine-to-machine | **Basic**, **Post**, **Private Key JWT**, **Unspecified** |

public アプリケーションは、**Post** や **Basic** などの <Tooltip tip="Token Endpoint: プログラムによって token を要求するために使用される Authorization Server 上の endpoint。" cta="用語集を表示" href="/ja/docs/glossary?term=Token+endpoint">Token endpoint</Tooltip> 認証方式に必要な認証情報の機密性を維持できません。
