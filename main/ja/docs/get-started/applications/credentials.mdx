Auth0 Dashboard の [アプリケーション](http://manage.auth0.com/#/applications) ページで、一覧から対象のアプリケーションを見つけ、その名前をクリックして設定画面を開きます。次に、**資格情報** タブに切り替えます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  パブリックアプリケーションの場合、テナントによっては資格情報タブが表示されないことがあります。機密アプリケーションとパブリックアプリケーションの違いについて詳しくは、[Confidential and Public Applications](/ja/docs/get-started/applications/confidential-and-public-applications) を参照してください。
</Callout>

<div id="authentication-methods">
  ## 認証方式
</div>

Auth0 では、アプリケーションが<Tooltip tip="認可サーバー: ユーザーのアクセス範囲の境界を定義するのに寄与する集中型サーバーです。たとえば、認可サーバーはユーザーが利用できるデータ、タスク、機能を制御できます。" cta="用語集を表示" href="/ja/docs/glossary?term=authorization+server">認可サーバー</Tooltip>に対して認証を行うための方法として、次のものを提供しています。

* **Private Key <Tooltip tip="JSON Web トークン (JWT): 2 者間でクレームを安全に表現するために使用される標準的な ID トークン形式（多くの場合、アクセストークン形式でもあります）。" cta="用語集を表示" href="/ja/docs/glossary?term=JWT">JWT</Tooltip>**: 機密アプリケーション向けの非対称認証方式です。Private Key JWT では、資格情報として使用する公開キーと秘密キーのペアを生成します。公開キーを提供し、秘密キーは Auth0 と共有せずに自社のシステム内で安全に保管します。秘密キーを使用して、認可サーバーに送信するリクエストに署名します。公開キーは、以下の **利用可能な資格情報** セクションで管理できます。
* **<Tooltip tip="クライアントシークレット: クライアント（アプリケーション）が認可サーバーで認証するために使用するSecretです。クライアントと認可サーバーだけが知っているべきであり、推測できないよう十分にランダムでなければなりません。" cta="用語集を表示" href="/ja/docs/glossary?term=Client+Secret">クライアントシークレット</Tooltip>:** 対称認証方式です。クライアントシークレット認証では、アプリケーションの作成時に Auth0 から割り当てられたクライアントシークレットを指定します。このセクションから `client_secret` を直接表示してコピーできます。

  * **クライアントシークレット (Basic)**: `HTTP BASIC` 認証スキームを使用してクライアントシークレットを送信する機密アプリケーションでは、Basic を使用します。
  * **クライアントシークレット (Post)**: リクエストボディパラメーターを使用してクライアントシークレットを送信する機密アプリケーションでは、Post を使用します。

認証方式と関連する資格情報の詳細については、[Application Credentials](/ja/docs/secure/application-credentials)をご覧ください。

<div id="available-credentials">
  ## 利用可能な資格情報
</div>

<Frame>![](/docs/images/cdy7uua7fh8z/7vzjXXZ0A9pDj5qsXmOTwh/b4b299c6316216376f7e2858007df0b1/Private_Key_JWT_-_English.png)</Frame>

**Add New Credential**: アプリケーションの資格情報の最大数 (2) に達していない場合は、一覧に新しい項目を追加します。

* **Name**: 資格情報の名前。例: 4096。
* **Key ID**: Auth0 が生成する一意の資格情報の識別子です。同じ資格情報を複数回アップロードすることはできません。
* **Algorithm**: 各資格情報に対して選択するアルゴリズム。RS256、PS256、RS384 をサポートしています。
* **Expires At**: 指定された用途に対して資格情報が無効になる `datetime`。資格情報は削除されませんが、使用できなくなります。これは、**Set an explicit expiry date for this credential** チェックボックスを有効にすることで UI で設定できます。

サイドメニューから、資格情報の有効化、無効化、削除を行えます。

<div id="rotate-client-secret">
  ## クライアントシークレットのローテーション
</div>

必要に応じて、アプリケーションのクライアントシークレットをローテーションすることがあります。詳しくは、[クライアントシークレットのローテーション](/ja/docs/get-started/applications/rotate-client-secret)をご覧ください。
