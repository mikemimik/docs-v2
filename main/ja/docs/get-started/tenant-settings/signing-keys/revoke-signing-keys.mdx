<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用して、テナントのアプリケーションまたは API の署名鍵を取り消すことができます。この署名鍵は、アプリケーションまたは API に送信される <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/ja/docs/glossary?term=ID+tokens">ID トークン</Tooltip>、<Tooltip tip="ID Token: リソースへのアクセスではなく、クライアント自体を対象としたクレデンシャル。" cta="用語集を見る" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip>、<Tooltip tip="Access Token: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可クレデンシャル。" cta="用語集を見る" href="/ja/docs/glossary?term=SAML">SAML</Tooltip> アサーション、および <Tooltip tip="Security Assertion Markup Language (SAML): パスワードなしで 2 者間で認証情報を交換できるようにする標準化プロトコル。" cta="用語集を見る" href="/ja/docs/glossary?term=WS-Fed">WS-Fed</Tooltip> アサーションの署名に使用されます。詳しくは、[署名鍵](/ja/docs/get-started/tenant-settings/signing-keys) を参照してください。

<div id="prerequisites">
  ## 前提条件
</div>

* これまで使用していた署名鍵を取り消すには、先にそのキーをローテーションしておく必要があります。詳しくは、[署名鍵のローテーション](/ja/docs/get-started/tenant-settings/signing-keys/rotate-signing-keys)を読むか、以下の**署名鍵をローテーションして取り消す**セクションを参照してください。
* 以前のキーを取り消す前に、アプリケーションまたは API が新しいキーを使用するよう更新されていることを確認してください。

<Warning>
  取り消した署名鍵は再利用できないため、取り消して問題ないことを必ず確認してください。
</Warning>

<div id="use-the-dashboard">
  ## Auth0 Dashboard を使う
</div>

<div id="revoke-previously-used-signing-key">
  ### 以前使用した署名鍵を取り消す
</div>

1. [Auth0 Dashboard &gt; 設定 &gt; 署名鍵](https://manage.auth0.com/#/tenant/signing_keys) に移動します。
2. **有効なキーの一覧** セクションで **以前使用した** キーを探し、その他のオプション (**...**) メニューから **キーを取り消す** を選択します。
   **有効なキーの一覧** セクションには、テナントで現在使用されている署名鍵に加え、署名鍵をローテーションした場合に次に割り当てられる署名鍵が表示されます。以前に署名鍵をローテーションしたことがある場合は、このセクションに以前使用したキーも表示されます。
   **取り消されたキーの一覧** セクションには、テナントで取り消された直近 3 つのキーが表示されます。
3. 確認のため、**取り消す** を選択します。

<div id="rotate-and-revoke-signing-key">
  ### 署名鍵をローテーションして取り消す
</div>

1. [Auth0 Dashboard &gt; 設定 &gt; 署名鍵](https://manage.auth0.com/#/tenant/signing_keys) に移動します。
2. **ローテーション設定**セクションで、**署名鍵をローテーションして取り消す**セクションを見つけ、**キーをローテーションして取り消す**を選択します。
3. 確認のため、**ローテーションして取り消す**を選択します。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

<Warning>
  以前使用していた署名鍵のみ取り消すことができます。
</Warning>

1. 署名鍵の一覧を取得するには、[すべてのアプリケーション署名鍵を取得](https://auth0.com/docs/api/management/v2#!/Keys/get_signing_keys) エンドポイントに `GET` リクエストを送信します。
2. [Key ID でアプリケーション署名鍵を取り消す](https://auth0.com/docs/api/management/v2#!/Keys/put_signing_keys) エンドポイントに `PUT` リクエストを送信します。`{yourKeyId}` と `{yourMgmtApiAccessToken}` のプレースホルダーは、それぞれ署名鍵の ID と Management API アクセストークンに必ず置き換えてください。

| 値                       | 説明                                                                                                                                |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `YOUR_KEY_ID`           | 取り消す署名鍵の ID。署名鍵 ID の確認方法については、[Locate JSON Web Key Sets](/ja/docs/secure/tokens/json-web-tokens/locate-json-web-key-sets) を参照してください。 |
| `MGMT_API_ACCESS_TOKEN` | スコープ `update:signing_keys` を持つ [Management API 用のアクセストークン](https://auth0.com/docs/api/management/v2/tokens)。                      |
