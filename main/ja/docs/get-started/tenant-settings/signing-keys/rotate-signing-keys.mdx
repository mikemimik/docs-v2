アプリケーションや API がトークンの検証に使用する JSON Web Key (JWK) キーを変更するために、署名鍵を定期的に手動でローテーションできます。アプリケーションまたは API がこのキー変更に**対応していない**場合、期限切れの署名鍵を使ってトークンを検証しようとすると、認証リクエストは失敗します。

<Warning>
  Auth0 では、まず開発テナントで署名鍵のローテーションを実行し、その後、アプリケーションと API が引き続き想定どおりに動作することを確認することを推奨しています。すべてが正常に動作していることを確認したら、本番テナントでも同じ署名鍵のローテーションを実行してください。
</Warning>

Auth0 は一度に 1 つの署名鍵でしか署名しませんが、テナントの <Tooltip tip="OpenID: ログイン情報を収集・保存することなく、アプリケーションがユーザーの本人確認を行えるようにする認証のためのオープン標準です。" cta="用語集を見る" href="/docs/ja/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) ディスカバリー ドキュメントには、常に複数のキーが含まれます。OIDC ディスカバリー ドキュメントには、現在のキーと次のキーの両方が常に含まれ、前のキーがまだ取り消されていない場合は、そのキーも含まれることがあります。緊急時にもシームレスな動作を実現するため、アプリケーションはドキュメントに記載されているどのキーでも使用できるようにしておく必要があります。OpenID Connect ディスカバリー ドキュメントの詳細については、[JSON Web Key Sets を確認する](/docs/ja/secure/tokens/json-web-tokens/locate-json-web-key-sets) をお読みください。

<Warning>
  新しい署名鍵に合わせてアプリケーションを更新する時間を確保できるよう、前のキーで署名されたすべてのトークンは、そのキーを取り消すまで引き続き有効です。詳しくは、[署名鍵を取り消す](/docs/ja/get-started/tenant-settings/signing-keys/revoke-signing-keys) をお読みください。
</Warning>

テナントのアプリケーション署名鍵は、<Tooltip tip="Auth0 Dashboard: サービスを構成するための Auth0 の主要製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または Auth0 <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を使用してローテーションできます。

<div id="use-the-dashboard">
  ## Auth0 Dashboardを使用する
</div>

1. [Auth0 Dashboard &gt; 設定 &gt; 署名鍵](https://manage.auth0.com/#/tenant/signing_keys) に移動します。

   <Frame>![Auth0 Dashboard テナント設定 署名鍵タブ](/docs/images/cdy7uua7fh8z/7r8t3EGctFmvkCgPrU0i2R/f79ac74dac5cded37b39bd8a8b80951d/dashboard-tenant-settings-signing-keys.png)</Frame>
2. **Rotation Settings** で **Rotate Signing Key** を探し、**Rotate Key** を選択します。
3. 確認するには **Rotate** をクリックします。

   <Frame>![Auth0 Dashboard 設定 署名鍵タブのローテーション確認](/docs/images/cdy7uua7fh8z/6Ofp24mgt8ZrvaxF2wLA5N/5cca1a3267ffdc07f61d7990b4e85989/Rotating_Key_Warning_-_English.png)</Frame>

<div id="use-the-management-api">
  ## Management API を使用する
</div>

1. 署名鍵の一覧を取得するには、[Get all Application Signing Keys](https://auth0.com/docs/api/management/v2#!/Keys/get_signing_keys) エンドポイントに `GET` リクエストを送信します。
2. 署名鍵をローテーションするには、[Rotate the Application Signing Key](https://auth0.com/docs/api/management/v2#!/Keys/post_signing_keys) エンドポイントに `POST` リクエストを送信します。`MGMT_API_ACCESS_TOKEN` プレースホルダーの値は、ご利用の Management API アクセストークンに置き換えてください。

<div id="key-rotation-impact">
  ## キー ローテーションの影響
</div>

<div id="apis-and-api-gateways-accepting-access-tokens">
  ### アクセストークンを受け付けるAPIとAPIゲートウェイ
</div>

ほとんどのミドルウェアやAPIゲートウェイでは、一定間隔でJSON Web Key Set (JWKS) エンドポイントを参照し、現在および今後使用される署名鍵を取得します。ご利用のミドルウェアやAPIゲートウェイがこのエンドポイントに**対応しておらず**、`*.cer` ファイルを手動で設定する必要がある場合は、Auth0での署名鍵ローテーションに合わせて、ミドルウェアやゲートウェイの再設定を調整する必要があります。

<div id="regular-web-applications">
  ### 従来型Webアプリケーション
</div>

Auth0 で署名鍵をローテーションする際は、<Tooltip tip="Web Service Federation (WS-Fed): ドメインをまたいでユーザーIDを管理するためのプロトコル。" cta="用語集を見る" href="/docs/ja/glossary?term=WS-Fed">WS-Fed</Tooltip> または <Tooltip tip="Web Service Federation (WS-Fed): ドメインをまたいでユーザーIDを管理するためのプロトコル。" cta="用語集を見る" href="/docs/ja/glossary?term=SAML">SAML</Tooltip> を利用しているアプリケーションの再構成を調整する必要があります。これは通常、新しい公開証明書をアップロードしたときや、WS-Fed/SAML のメタデータ URL を入力してアプリケーションを再構成したときに発生します。これにより、アプリケーションがトークンの検証に使用する JWKS キーが変更されるため、実装で JWKS キーが変更されないことを前提にしないようにしてください。
