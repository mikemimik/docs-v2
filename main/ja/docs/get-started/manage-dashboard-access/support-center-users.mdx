---

テナント管理者は、同僚にロールを割り当てて、<Tooltip tip="Auth0 Dashboard: サービスを構成するための Auth0 の主要な製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> への限定的なアクセス権を付与できます。これにより、最小権限の原則を守りながら、本番アプリケーションを危険にさらすことなく、テナントメンバーが必要な業務を遂行できるようになります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  すべてのロールで [Support Center](https://support.auth0.com/) を利用できます。
</Callout>

<div id="dashboard-roles">
  ## Auth0 Dashboard のロール
</div>

Auth0 Dashboard へのアクセスには、次のロールを割り当てることができます。

| ロール                     | 権限                                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **Admin**                 | Extensions を含む、Auth0 Dashboard 内のすべてのリソースに対する読み取りおよび書き込みアクセス。                                                                   |
| **編集者 - 接続**            | あらゆる種類の接続に対する読み取り、書き込み、および作成アクセス。                                                                                               |
| **編集者 - キー管理**          | 暗号化キーの作成と管理。                                                                                                                    |
| **編集者 - Organizations** | 組織の設定に対する読み取り、書き込み、および作成アクセス。Users、Roles、Connections に対しては読み取り専用アクセス。                                                           |
| **編集者 - 特定のアプリケーション**   | 特定のアプリケーションのみに対する読み取りおよび書き込みアクセス。                                                                                               |
| **Editor - Users**          | ユーザー管理操作 (作成、削除、ブロック、ブロック解除、MFA のリセット、パスワードのリセット、メタデータの更新、ロールの割り当てなど) およびログへのアクセス。                                              |
| **Viewer - Users**          | ユーザーおよびユーザー関連のログへの読み取り専用アクセス。                                                                                                   |
| **閲覧者 - 設定**            | secrets、請求、ユーザー、ログなどの機密情報を除く、すべての設定 (アプリケーション、API、ルール、セキュリティ設定など) への読み取り専用アクセス。                                                 |
| **サポートアクセス - 非推奨**      | チケットへのアクセス (送信、表示、更新) および集計メトリクス。                                                                                               |
| **Elevated Support Access**          | テナント上の **すべてのユーザー** が作成した **すべての** サポートチケット (Auth0 Support Center の Subscripton Tickets) に対する読み取り、書き込み、および作成アクセス。集計メトリクスへのアクセス。 |

<Card title="利用可否は Auth0 のプランによって異なります">
  ご利用の Auth0 のプランまたはカスタム契約によって、この機能が利用できるかどうかが決まります。詳しくは、[Auth0 の料金ページ](https://auth0.com/pricing)をご覧ください。
</Card>

制限付きアクセスのロールでは、テナントメンバーが Auth0 Dashboard で利用できる範囲が、業務に必要なセクションとアクションのみに制限されます。

各ロールに関連付けられた機能の権限は、以下のとおりです。特定のロールに特定の権限が記載されていない場合、そのロールを付与されたテナントメンバーは Auth0 Dashboard のその領域にはアクセスできません。

<div id="log-events-available-to-user-roles">
  ## ユーザーロールがアクセスできるログイベント
</div>

ログには、シークレットや PII などの機微なデータが含まれる場合があります。そのため、その情報を必要としないロールのユーザーに機微なデータを開示しないことが重要です。一方で、**Editor - Users** または **Viewer - Users** ロールでは、ユーザーの問題を特定するために、ログへの一定のアクセスが必要です。たとえば、ユーザーが正しくサインアップできたか、ブロックされているかどうか、などを確認するためです。

**Editor - Users** と **Viewer - Users** には、ユーザーイベントに関連する限られた種類のログへのアクセスが許可されています。この一覧にあるログイベントは、ユーザーの操作に関する必要な情報を提供しますが、テナント設定のその他の部分に関する機微な情報は開示しません。詳細については、[Log Event Type Codes](/ja/docs/deploy-monitor/logs/log-event-type-codes) を参照してください。

<div id="limitations">
  ## 制限事項
</div>

* **Admin** ロールを持つユーザーは、**Editor - Specific Apps** ユーザーを一度に1つのアプリケーションにしか招待できません。これを回避するには、招待したユーザーが承諾した後に、**Admin** ユーザーが[そのユーザーのロールを編集](/ja/docs/get-started/manage-dashboard-access/edit-dashboard-users)して、複数のアプリケーションを割り当てます。
* **Viewer - Users** ロールと **Editor - Users** ロールでは、Users の **Devices** セクションおよび **Authorized Apps** セクションにアクセスできません。
* [New Activity Page](/ja/docs/get-started/auth0-overview/dashboard/activity) が表示されるのは、**Admins** と **Elevated Support Access** ユーザーのみです。**Editor - Users** と **Viewer - Users** は、[Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Stats/get_daily) を通じて、日次アクティビティ (ログインやサインアップなど) にアクセスできます。

<div id="private-cloud-requirements">
  ## Private Cloud の要件
</div>

**Editor - Users** と **Viewer - Users** のロールを使用するには、お使いの環境で User Search v3 と Logs Search v3 が有効になっている必要があります。お使いの環境がこれらのバージョンに対応していない場合、これら 2 つのロールは利用できません。
