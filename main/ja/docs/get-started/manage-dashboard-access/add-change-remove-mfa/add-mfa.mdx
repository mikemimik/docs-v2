各 <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ユーザーは、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=multi-factor+authentication">多要素認証</Tooltip> (MFA) に各自で登録する必要があります。ほとんどの認証要素は [プロファイル](https://manage.auth0.com/#/profile) で登録できます。ただし、デバイスの生体認証では段階的な登録が必要です。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 は、最も安全で使いやすい認証方法として WebAuthn 認証要素を推奨しています。詳しくは、[WebAuthn を使用した FIDO 認証](/ja/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn) をご覧ください。
</Callout>

<Warning>
  管理者が MFA を使用するには、少なくとも 1 つの認証要素を有効にする必要があります。メインのデバイスを失った場合でもアカウントにアクセスできるよう、Auth0 では複数の認証要素を設定することを強く推奨しています。

  理想的な設定は、次の 3 つの認証要素を使用することです。

  1. WebAuthn、Guardian、または OTP を主な方法として使用する。
  2. バックアップとして 1 つ以上の SMS 受信用電話番号を使用する (サブスクリプションプランに紐づくテナントでのみ利用可能) 。
  3. リカバリーコード。

  MFA トークンを提示できず、適切なバックアップ方法もない場合、アカウントを復旧できない可能性があります。
</Warning>

<div id="add-mfa">
  ## MFA を追加する
</div>

MFA を自身で登録するには、Auth0 Dashboard の各ユーザーが次の手順に従う必要があります。

1. Auth0 Dashboard の右上にある自分のユーザー名をクリックし、[プロファイル](https://manage.auth0.com/#/profile) をクリックします。
2. 利用したい対応済みの方法を見つけ、その行の **+ ADD** をクリックします。

   <Frame>![Dashboard - Profile - Multi-Factor - Authentication](/docs/images/cdy7uua7fh8z/2NyLNvP92RruwZ46dLPm5x/8ece3df2b550af8dd78368ca36d8a661/MFA.png)</Frame>
3. 画面上の案内に従って登録を完了します。

<div id="device-biometrics">
  ## デバイス生体認証
</div>

デバイス生体認証を使うWebAuthnは、Account Settingsページで追加できない唯一の方式です。代わりに、Auth0ではWebAuthn対応デバイスが段階的に登録されます。ほかのMFA方式を登録すると、Auth0からそれらのデバイスを登録するよう求められます。これらの案内は、Auth0 Dashboardにログインするたびに繰り返し表示されます。

<Frame>![undefined](/docs/images/cdy7uua7fh8z/1cBjDSZrIeHAS2Fyzbt5HK/7045d8e2c5b0b0588d5eb88f4a5620b0/deviceenrollment.png)</Frame>

登録の際、Auth0からデバイスに名前を付けるよう求められます。これにより、Account Settingsページから簡単に管理できます。

Javascriptが無効になっているブラウザーや、WebAuthnのプラットフォーム認証器をサポートしていないブラウザーでは、デバイス生体認証を登録したり、その方法で認証したりすることはできません。主要なブラウザーとオペレーティングシステムの最新バージョンでは、WebAuthn with Security Keysがサポートされています。詳しくは、[webauthn.meのブラウザーサポートセクション](https://webauthn.me/browser-support)をご覧ください。

<div id="recovery-codes">
  ## 回復コード
</div>

二要素認証を有効にすると、成功直後に Auth0 からリカバリーコードをコピーするよう求められます。登録済みの認証要素をすべて利用できなくなった場合は、このリカバリーコードを使ってアカウントにログインできます。Auth0 では、リカバリーコードをコピーして印刷するか、パスワードマネージャーなどの安全な場所に保管することを推奨しています。

リカバリーコードを紛失した場合や、新しいコードを生成したい場合は、[プロファイル](https://manage.auth0.com/#/profile) から行えます。

<div id="log-in-to-the-dashboard-with-mfa-enabled">
  ## MFA を有効にして Auth0 Dashboard にログインする
</div>

MFA を有効にしてログインする場合も、通常のログインとほとんど変わりません。管理者アカウントの資格情報を入力すると、有効にしている MFA の認証要素の種類に応じて、2 つ目のプロンプトが表示されます。

ユーザーが主な認証要素を利用できなくなった場合は、**Select Another Method** をクリックして、リカバリーコードを含むほかの認証要素を試すことができます。そのため、アカウントにログインできなくなるのを防ぐには、複数の方法を登録しておくことが非常に重要です。

2 つ目の認証要素の登録が正常に完了したあと、WebAuthn をサポートする新しいデバイスからログインすると、「このデバイスでよりすばやくログイン」というプロンプトが表示されます。これにより、次回からそのデバイスを多要素認証に使用できます。
