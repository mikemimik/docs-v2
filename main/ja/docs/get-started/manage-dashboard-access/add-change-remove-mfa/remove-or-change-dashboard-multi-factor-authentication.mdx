Auth0 Dashboard ユーザーは、利用できなくなった、または不要になった<Tooltip tip="多要素認証（MFA）: SMS で送信されるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=multi-factor+authentication">多要素認証</Tooltip> (MFA) の認証要素を削除または変更できます。Auth0 Dashboard ユーザー向けの MFA の詳細については、[Multi-Factor Authentication for Dashboard Users](/ja/docs/get-started/manage-dashboard-access/add-change-remove-mfa) を参照してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 では、最も安全で使いやすい認証方法として WebAuthn の認証要素を推奨しています。詳しくは、[FIDO Authentication with WebAuthn](/ja/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn) を参照してください。
</Callout>

<Warning>
  Admin が MFA を使用するには、少なくとも 1 つの認証要素を有効にする必要があります。Auth0 では、主要なデバイスを紛失した場合でもアカウントにアクセスできるよう、複数の認証要素を設定することを強く推奨しています。

  理想的な設定は、次の 3 つの認証要素を使用することです。

  1. 主な方法として WebAuthn、Guardian、または OTP。
  2. バックアップとして 1 つ以上の SMS 番号 (サブスクリプションプランが紐付いたテナントでのみ利用可能) 。
  3. リカバリーコード。

  MFA トークンを提示できず、適切なバックアップ方法もない場合は、アカウントを復旧できなくなる可能性があります。
</Warning>

<div id="remove-or-change-an-mfa-factor-from-the-dashboard">
  ## Auth0 Dashboard で MFA 認証要素を削除または変更する
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 Dashboard からは、Device Biometrics 認証要素 (MacBook Touch Bar、Windows Hello、iOS Touch ID または Face ID、Android の指紋認証や顔認証など) を追加できません。Device Biometrics 認証要素の追加方法については、[Dashboard ユーザー向けの多要素認証を追加する](/ja/docs/get-started/manage-dashboard-access/add-change-remove-mfa/add-mfa)を参照してください。
</Callout>

現在の MFA 認証要素でログインできる Auth0 Dashboard ユーザーは、次の手順に従ってください。

1. Auth0 Dashboard の右上にある自分のユーザー名をクリックし、[プロファイル](https://manage.auth0.com/#/profile)をクリックします。

   <Frame>![Auth0 Dashboard - プロファイル - 多要素認証 - 認証](/docs/images/cdy7uua7fh8z/2NyLNvP92RruwZ46dLPm5x/8ece3df2b550af8dd78368ca36d8a661/MFA.png)</Frame>
2. 使用したい新しい認証要素を見つけ、その行の **+ ADD** をクリックします。画面の指示に従って登録を完了します。
3. 同じ[プロファイル](https://manage.auth0.com/#/profile)ページで、使用をやめる認証要素を見つけ、**REMOVE** をクリックします。
4. **Yes** をクリックして削除を確定します。
5. Auth0 から、現在使用している (古い) 認証要素で認証するよう求められます。認証に成功すると、その認証要素が削除されます。

<div id="remove-or-change-a-lost-mfa-factor">
  ## 紛失した MFA 認証要素を削除または変更する
</div>

現在の MFA 認証要素ではログイン**できない** Auth0 Dashboard ユーザーは、次の手順を実行してください。

1. Auth0 Dashboard へのログインを試みます。Auth0 から、現在の認証要素を使用して認証するよう求められます。
2. Auth0 から紛失したデバイスまたは資格情報の提示を求められたら、**Try another method** をクリックします。
3. Other Methods ボックスで、認証に使用する別の方法をクリックします。
4. ログインして Auth0 Dashboard にアクセスします。
5. Auth0 Dashboard の右上隅で自分のユーザー名をクリックし、[プロファイル](https://manage.auth0.com/#/profile) をクリックします。

   <Frame>![Dashboard - Profile - Multi-Factor - Authentication](/docs/images/cdy7uua7fh8z/2NyLNvP92RruwZ46dLPm5x/8ece3df2b550af8dd78368ca36d8a661/MFA.png)</Frame>
6. もう使用できない認証方法を見つけて、**REMOVE** をクリックします。
7. 削除を確認するには **Yes** をクリックします。
8. Auth0 から、現在の認証要素を使用して再度認証するよう求められます。手順 2〜4 を繰り返して本人確認を行うと、Auth0 によって紛失した認証要素が削除されます。

<div id="get-help-from-auth0-support">
  ## Auth0 Supportに問い合わせる
</div>

ロックアウトされていて、有効になっているMFA認証要素のどれにもアクセスできない場合は、アカウントへのアクセスを回復できるとは限りません。別の管理者が、あなたに代わってAuth0 Supportにサポートチケットを提出する必要があります。場合によっては、Auth0がリクエストを確認し、MFAをリセットできることがあります。ただし、アカウントの所有者であることを確認できない場合もあります。そのため、複数の異なる認証要素を有効にしておくことが非常に重要です。

Auth0 Supportは**エンドユーザー**アカウントのリセットには対応していません。アプリケーションとAPIにアクセスするアカウントについては、お客様の責任で管理してください。エンドユーザーアカウントについては、[ユーザーの管理](/ja/docs/manage-users)を参照してください。
