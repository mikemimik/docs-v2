Auth0 テナント管理者は、テナントメンバーの管理を含め、自身の Auth0 アカウントおよびテナント配下で発生するすべてのアクティビティに責任を負います。<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または [Auth0 Teams Dashboard](/docs/ja/get-started/auth0-teams/tenant-member-management) から、テナントメンバー (ダッシュボードユーザー) を追加、変更、削除できます。Auth0 では、Auth0 テナントにアクセスできる Auth0 Dashboard のテナントメンバー一覧を定期的に見直し、次の点を確認することを推奨しています。

* 各ユーザーに、テナントメンバーとしてのアクセス権が業務上必要であること。
* メンバーが会社のアカウントで登録されていること。
* 退職者がアクセスできないこと。
* Dashboard 管理者が複数いること。

以下は、特別なアクセス要件が必要になる可能性があるユーザーの例です。

* アプリのエンドユーザーのログイン問題を調査する必要があるサポート担当者。
* エンドユーザーにロールや権限を割り当てる必要があるサポート担当者や IT 担当者。
* 本番環境で自分のアプリケーションの問題を調査する必要がある開発者。
* 自分のアプリケーションの設定や使用状況を分析する必要があるプロダクトマネージャー。
* 自分のアプリケーションの設定を行う必要がある開発者。
* B2B のユースケースで顧客向けの接続を作成する必要があるサポート担当者や IT 担当者。
