このリファレンスガイドでは、Auth0 MCP Server と任意の [MCP クライアント](https://modelcontextprotocol.io/clients) を使用して Auth0 テナントを操作するための自然言語の例を確認できます。

<div id="application-management">
  ## アプリケーション管理
</div>

| ツール                         | 説明                                         | 自然言語の例                                                                                                                                                                                                     |
| --------------------------- | ------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth0_list_applications`   | Auth0 テナント内のすべてのアプリケーションを一覧表示するか、名前で検索します。 | 「Auth0 アプリケーションをすべて表示して。」<br /><br />「名前に &#39;api&#39; を含むアプリケーションを探して。」<br /><br />「Auth0 テナントにはどんなアプリケーションがありますか？」                                                                                      |
| `auth0_get_application `    | 特定の Auth0 アプリケーションの詳細を取得します。               | 「&#39;Customer Portal&#39; というアプリケーションの詳細を表示して。」<br /><br />「Client ID {clientID} のアプリケーション情報を取得して。」<br /><br />「&#39;Mobile App&#39; のコールバック URL は何ですか？」                                                  |
| `auth0_create_application ` | 新しい Auth0 アプリケーションを作成します。                  | 「&#39;Analytics Dashboard&#39; という名前の新しいシングルページアプリケーションを作成して。」<br /><br />「&#39;iOS Client&#39; という名前の新しい Native モバイルアプリを作成して。」<br /><br />「バックグラウンドサービス用の machine-to-machine アプリケーションを作成して。」              |
| `auth0_update_application`  | 既存の Auth0 アプリケーションを更新します。                  | 「&#39;Web App&#39; のコールバック URL に `https://staging.example.com/callback` を追加して。」<br /><br />「&#39;Customer Portal&#39; のログアウト URL を変更して。」<br /><br />「&#39;Admin Dashboard&#39; アプリケーションに開発環境用のメタデータを追加して。」 |

<div id="resource-server-api-management">
  ## リソースサーバー (API) の管理
</div>

| Tool                           | Description                              | Natural language examples                                                                                                                                     |
| ------------------------------ | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth0_list_resource_servers`  | Auth0 テナント内のすべてのリソースサーバー (API) を一覧表示します。 | 「Auth0 テナント内のすべての API を表示して。」<br /><br />「自分のリソースサーバーを一覧表示して。」<br /><br />「Auth0 で設定している API は何？」                                                             |
| `auth0_get_resource_server`    | 特定の Auth0 リソースサーバーの詳細を取得します。             | 「『User API』の詳細を表示して。」<br /><br />「『Payment API』にはどんなスコープが定義されている？」<br /><br />「識別子 `https://api.example.com` のリソースサーバー情報を取得して。」                               |
| `auth0_create_resource_server` | 新しい Auth0 リソースサーバー (API) を作成します。         | 「read スコープと write スコープを持つ『Inventory API』という新しい API を作成して。」<br /><br />「顧客データ API 用のリソースサーバーを設定して。」<br /><br />「識別子 `https://orders.example.com` の API を作成して。」 |
| `auth0_update_resource_server` | 既存の Auth0 リソースサーバーを更新します。                | 「『User API』に『admin』スコープを追加して。」<br /><br />「『Payment API』のトークンの有効期間を 1 時間に更新して。」<br /><br />「API の署名アルゴリズムを RS256 に変更して。」                                      |

<div id="combining-operations">
  ## 操作を組み合わせる
</div>

  #### クライアントアクセス用の新しいAPIを設定する
</div>

```bash wrap lines
Create a new API called "Inventory API" with read and write scopes, then update my "Admin Dashboard" application to have access to it.
```

</div>

```bash wrap lines
社内ネットワーク外からログインするユーザーにMFAを要求するActionを作成し、ログインフローにデプロイしてください。
```

<div id="best-practices">
  ## Best Practices
</div>

1. リクエストは具体的にする: 特定のリソースに言及する際は、名前、IDs、その他の識別情報を含めてください。
2. 自然な言葉を使う: ツール名やパラメーターを覚える必要はありません。何をしたいのかをそのまま説明してください。
3. 確認前に見直す: Claude が変更を提案した場合は、確定する前に内容を注意深く確認してください。
4. まずはシンプルに始める: まずは簡単なクエリから始めて、Claude がリクエストをどのように解釈するかに慣れてください。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

Claude が操作を実行できない場合は、次の点を確認してください。

1. 認証: Auth0 CLI でログインしていることを確認してください (`auth0 login`) 。
2. 権限: ご利用のアカウントに、Auth0 で必要な権限が付与されていることを確認してください。
3. 具体性: リクエストの内容を、できるだけ具体的にしてみてください。
