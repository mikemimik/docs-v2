  Auth0 Universal Componentsは現在、早期アクセス段階です。これを使用することで、[Oktaのマスターサブスクリプション契約](https://www.okta.com/agreements/)に記載された適用中の無料トライアル条件に同意したものとみなされます。詳しくは、[製品リリース段階](/ja/docs/troubleshoot/product-lifecycle/product-release-stages)をご覧ください。
</Warning>

Auth0 Universal Componentsは、[Tailwind CSS](https://tailwindcss.com/docs/installation/using-vite)と[Radix UI primitives](https://www.radix-ui.com/primitives/docs/overview/introduction)を活用した柔軟なデザインシステムを基盤に構築されています。スタイル設定は、[`Auth0ComponentProvider`](/ja/docs/get-started/universal-components/auth0-component-provider)によって一元的に制御され、高レベルのプリセットを使うことも、CSS変数を細かく上書きすることもできます。

このアーキテクチャにより、認証フローをアプリケーションにネイティブに溶け込む見た目と操作感にできます。

<Tabs>
  <Tab title="React">
    ## はじめに

    1. スタイルシートを読み込みます。

    ```tsx
    import "@auth0/universal-components-react/styles";
    ```

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Tailwind v4をご利用の方:** `@import "@auth0/universal-components-react/tailwind"` を CSS ファイルに追加してください。
    </Callout>

    2. ブランドカラーを追加します。

    ```css
    :root {
      --primary: #4f46e5; /* ブランドカラー */
      --primary-foreground: #ffffff; /* ボタンのテキスト */
    }
    ```

    完了しました。すべてのボタン、リンク、アクティブ状態にブランドが反映されました。

    ***

    ## テーマプリセット

    `themeSettings` を `Auth0ComponentProvider` に渡すと、組み込みテーマを切り替えられます。

    ```tsx App.tsx
    <Auth0ComponentProvider
      themeSettings={{
        mode: "light", // 'light' | 'dark'
        theme: "default", // 'default' | 'minimal' | 'rounded'
      }}
    >
      <App />
    </Auth0ComponentProvider>
    ```

    ## CSS変数の確認

    すべての視覚的プロパティはCSSカスタムプロパティで制御されています。スタイルシートまたは`themeSettings.variables`プロパティで上書きできます。

        ```css
          --primary-foreground: oklch(100% 0 0);  /* プライマリ面上のテキスト */

        /_ サーフェス _/
        --background: oklch(100% 0 0); /_ ページ背景 _/
        --foreground: oklch(9% 0 0); /_ デフォルトテキスト _/

        --input: oklch(100% 0 0); /_ 入力フィールド背景 _/

        /_ セカンダリ & ミュート _/

        --secondary-foreground: oklch(100% 0 0);
        --muted: oklch(96% 0 0); /_ 無効化された背景 _/
        --muted-foreground: oklch(45% 0 0); /_ プレースホルダーテキスト _/
        ```

    ## スタイルコンポーネント

    すべてのコンポーネントは、グローバルスタイルに影響を与えずに個別のオーバーライドを指定できる `styling` プロパティをサポートしています。

    ```tsx
    <SsoProviderTable
      styling={{
        variables: {
          common: {
            "--primary": "#059669", // このコンポーネントにのみ適用するオーバーライド
          },
          light: { "--card": "#f8fafc" },
          dark: { "--card": "#1e293b" },
        },
        classes: {
          "SsoProviderTable-header": "shadow-lg",
          "SsoProviderTable-table": "rounded-xl",
        },
      }}
    />
    ```

    **使用するタイミング：**

    * `variables` - コンポーネント単位で適用される CSS カスタムプロパティの上書き
    * `classes` - コンポーネントの特定の部分に適用される Tailwind またはカスタムクラス

    <Accordion title="コンポーネント別の適用可能なクラス対象">
      各コンポーネントは、主要なセクションごとにクラスターゲットを公開しています。

      **SsoProviderCreate**

      * `SsoProviderCreate-header`, `SsoProviderCreate-wizard`
      * `ProviderSelect-root`, `ProviderDetails-root`, `ProviderConfigure-root`

      **SsoProviderTable**

      * `SsoProviderTable-header`, `SsoProviderTable-table`, `SsoProviderTable-row`

      **SsoProviderEdit**

      * `SsoProviderEdit-header`, `SsoProviderEdit-tabs`
      * `SsoProviderEdit-ssoTab`, `SsoProviderEdit-provisioningTab`, `SsoProviderEdit-domainsTab`

      **DomainTable**

      * `DomainTable-header`, `DomainTable-table`
      * `DomainTable-createModal`, `DomainTable-configureModal`, `DomainTable-deleteModal`

      **OrganizationDetailsEdit**

      * `OrganizationDetailsEdit-header`, `OrganizationDetailsEdit-form`, `OrganizationDetailsEdit-actions`
    </Accordion>

    ## よくあるカスタマイズ

    ### ブランドカラー (Hex)

    ブランドの16進数カラーコードをCSSで使用する形式に変換してください：

    ```css
    :root {
      /* 16進数を直接使用 */
      --primary: #4f46e5;
      --primary-foreground: #ffffff;

      /* またはより良い色操作のためにoklchを使用 */
      --primary: oklch(50% 0.2 265); /* 紫 */
    }
    ```

    ### 角丸

    すべてのコンポーネントに丸みのある外観を適用するには：

    ```css
    :root {
      --radius-lg: 16px; /* ボタン、入力欄 */
      --radius-xl: 20px; /* カード */
      --radius-2xl: 24px; /* モーダル */
    }
    ```

    ### コンパクトなタイポグラフィ

    よりコンパクトなUIの場合：

    ```css
    :root {
      --font-size-page-header: 1.75rem;
      --font-size-heading: 1.25rem;
      --font-size-title: 1rem;
      --font-size-body: 0.875rem;
    }
    ```

    ## ダークモード

    コンポーネントは`themeSettings`の`mode`設定に自動的に対応します。アプリのダークモードと同期するには：

    ```tsx
    function App() {
      const [isDark, setIsDark] = useState(false);

      return (
        <Auth0ComponentProvider
          themeSettings={{
            mode: isDark ? "dark" : "light",
          }}
        >
          <YourApp />
        </Auth0ComponentProvider>
      );
    }
    ```

    またはシステム設定に従う:

    ```tsx
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;

    <Auth0ComponentProvider
      themeSettings={{ mode: prefersDark ? 'dark' : 'light' }}
    >
    ```

    ## はじめに

    1. スタイルシートを読み込みます。

    ルートレイアウトファイルに以下を追加します：

    ```tsx layout.tsx
    import { Auth0ComponentProvider } from "@auth0/universal-components-react/rwa";
    import "@auth0/universal-components-react/styles";

    export default function RootLayout({ children }) {
      return (
        <html lang="en">
          <body>
            <Auth0ComponentProvider
              mode="proxy"
              domain={process.env.NEXT_PUBLIC_AUTH0_DOMAIN}
              proxyConfig={{ baseUrl: "/api/auth" }}
            >
              {children}
            </Auth0ComponentProvider>
          </body>
        </html>
      );
    }
    ```

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">

    すべてのコンポーネントは、グローバルスタイルに影響を与えずに個別のオーバーライドを適用できる `styling` プロパティをサポートしています。

    ```tsx
    <SsoProviderTable
      styling={{
        variables: {
          common: {
            "--primary": "#059669", // このコンポーネントのみオーバーライド
          },
          light: { "--card": "#f8fafc" },
          dark: { "--card": "#1e293b" },
        },
        classes: {
          "SsoProviderTable-header": "shadow-lg",
          "SsoProviderTable-table": "rounded-xl",
        },
      }}
    />
    ```

    **使用するタイミング：**

    * `variables` - コンポーネント単位で適用される CSS カスタムプロパティの上書き
    * `classes` - コンポーネントの特定の部分に適用される Tailwind またはカスタムクラス

    ## よくあるカスタマイズ

    ### ブランドカラー (Hex)

    ブランドの16進数カラーコードをCSSで使用する形式に変換してください：

    ```css
    :root {
      /* 16進数を直接使用 */
      --primary: #4f46e5;
      --primary-foreground: #ffffff;

      /* またはより良い色操作のためにoklchを使用 */
      --primary: oklch(50% 0.2 265); /* 紫 */
    }
    ```

    ### 角丸

    すべてのコンポーネントに丸みのある外観を適用するには：

    ```css
    :root {
      --radius-lg: 16px; /* ボタン、入力欄 */
      --radius-xl: 20px; /* カード */
      --radius-2xl: 24px; /* モーダル */
    }
    ```

    ### コンパクトなタイポグラフィ

    よりコンパクトなUIの場合：

    ```css
    :root {
      --font-size-page-header: 1.75rem;
      --font-size-heading: 1.25rem;
      --font-size-title: 1rem;
      --font-size-body: 0.875rem;
    }
    ```

    ```tsx
      const [isDark, setIsDark] = useState(false);

      return (
        <Auth0ComponentProvider
          themeSettings={{
            mode: isDark ? "dark" : "light",

        >
          <YourApp />
        </Auth0ComponentProvider>
      );
    }
    ```
