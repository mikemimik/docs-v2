Auth0 では、製品やコミュニティを効果的に活用できるよう、さまざまなリソースを提供しています。この一覧では、利用可能なリソースへのリンクをカテゴリ別にご紹介します。

<div id="get-started">
  ## はじめに
</div>

Auth0 の基本を学ぶために役立つリソースには、次のようなものがあります。

* [**Getting Started documentation**](/ja/docs/get-started): <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> や、Auth0 サービスの各コンポーネントで一般的に使われる用語を確認できます。Auth0 の全体像を把握し、Auth0 の担当者と連携するときやドキュメントを読むときによく目にする用語を学べます。
* [**Architecture scenarios**](/ja/docs/get-started/architecture-scenarios): 一般的なアーキテクチャシナリオを確認し、それらを Auth0 でどのように実装するかを学べます。シナリオには、Single-Page Application (SPA) から API を呼び出す場合など、よくあるアーキテクチャパターンのチュートリアルが含まれます。概要レベルの説明はアーキテクトに役立ち、チュートリアルは開発チームの助けになります。
* [**Implementation guides**](/ja/docs/): ユーザー管理や <Tooltip tip="多要素認証（MFA）: SMS で送信されるコードのように、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセスです。" cta="用語集を見る" href="/ja/docs/glossary?term=multi-factor+authentication">多要素認証</Tooltip> (MFA) など、よく使われる機能の実装方法を学べます。この情報は、アーキテクトにも開発者にも役立ちます

<div id="learn">
  ## 学習
</div>

Auth0 では、学習や必要な情報の素早い確認に役立つ、数多くのチュートリアル、ガイド、ホワイトペーパー、ブログ記事を提供しています。

* [**Docs site**](/ja/docs/): ドキュメントを参照して幅広いトピックを確認したり、検索機能を使ってトピックやタームに関連するコンテンツをすばやく見つけたりできます。
* [**Feature descriptions and white papers**](https://auth0.com/learn/): 機能の簡単な説明、業界のケーススタディ、参考資料となるホワイトペーパーを確認できます。Auth0 Learn サイトでは、Auth0 の機能とそのビジネス上の価値を手早く把握できるため、プロジェクトオーナーだけでなく、アーキテクトや開発者にも役立ちます。
* [**Blog posts**](https://auth0.com/blog/): さまざまなトピックについて専門家が執筆したブログ記事を読むことができます。内容は、定番のアドバイスから、アイデンティティ分野の最新ニュースまで多岐にわたります。多くの記事は、アーキテクトや開発者を対象としています。

<div id="run-sample-code">
  ## サンプルコードを試す
</div>

開発チームで構築の準備が整ったら、Auth0 はプロジェクトをすばやく進められるよう、サンプルプログラム、SDK、ライブラリを提供しています。

* [**Quickstarts**](/ja/docs/quickstarts): 認証、セッション管理、プロファイルの更新、logout など、プログラムに組み込みたい主要機能の実装方法を示す、豊富な小規模サンプルプログラムを確認できます。Quickstarts は、開発者がアプリケーションを Auth0 とどのように連携させるかを理解するうえで、よい出発点になります。
* [Libraries and SDKs](/ja/docs/libraries): アイデンティティプロトコルの複雑な部分の多くを抽象化する、幅広いライブラリと SDK を活用することで、独自のアプリケーション開発を簡素化できます。Auth0 SDKs は多数の言語とフレームワークをサポートしており、Auth0 との連携を簡単にします。また、iOS や Android を含む複数のプラットフォームで利用できる、Lock という Login Widget 向けの library も提供しています。
* [**Management API**](https://auth0.com/docs/api/management/v2): Auth0 内のオブジェクト、構成、設定を確認し、操作できます。<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> Explorer を使うと、個々のオブジェクトや設定を必要に応じてすばやく操作でき、アプリケーションにコードとして組み込む前に API 呼び出しをテストできます。
* [**Authentication API**](https://auth0.com/docs/api/authentication): OIDC、<Tooltip tip="OAuth 2.0: 認可プロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を見る" href="/ja/docs/glossary?term=OAuth">OAuth</Tooltip>、および <Tooltip tip="OAuth 2.0: 認可プロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を見る" href="/ja/docs/glossary?term=SAML">SAML</Tooltip> プロトコルを介して、ユーザーの認証と認可を行います。Authentication API Explorer では、認証や <Tooltip tip="Authorization Flow: OAuth 2.0 フレームワークで規定された認可グラント（またはワークフロー）。" cta="用語集を見る" href="/ja/docs/glossary?term=authorization+flows">認可フロー</Tooltip> を試し、アプリケーションにコードとして組み込む前に API 呼び出しをテストできます。

<div id="try-out-features-and-api-calls">
  ## 機能と API 呼び出しを試す
</div>

開発中は、Auth0 のさまざまな製品機能を実際に試したりテストしたりできます。

* **TRY ボタン/リンク**: Auth0 の製品機能をすばやく試せます。**TRY** ボタンは Auth0 全体の各所にあり、接続、ルール、Hooks、メールテンプレートを試すことができます。
* [**Authentication API Explorer**](https://auth0.com/docs/api/authentication#introduction): authentication と認可フローを試し、アプリケーションに実装する前に API 呼び出しをテストできます。
* [**Management API Explorer**](https://auth0.com/docs/api/management/v2): 個々のオブジェクトや設定を必要に応じてすばやく操作し、アプリケーションに実装する前に API 呼び出しをテストできます。

<div id="get-help">
  ## ヘルプを利用する
</div>

実装のトラブルシューティングに役立つ Auth0 のリソースには、次のものがあります。

* [**Auth0 Community forum**](https://community.auth0.com/): Auth0 の投稿、FAQ、コミュニティ Q&amp;A を通じて、Auth0 ユーザーとつながることができます。アーキテクトや開発者にとって、学習や情報交換に加え、問題解決のサポートを得るための貴重な情報源です。
* [**Support Center**](https://support.auth0.com/): サブスクリプションやテナントの表示と管理、サポートリクエストの登録と確認、テナントに対する自動の本番チェックの実行、コンプライアンス情報の確認を行えます。有料プランの利用者にとって、ドキュメントやフォーラム検索だけでは問題や疑問を解決できない場合に、Support Center は非常に役立つリソースです。

  * [**サポートケースを作成する**](/ja/docs/troubleshoot/customer-support/open-and-manage-support-tickets): サポートが必要な場合は、サポートケースを作成して登録します。
  * [**サポートプランとサービスレベル契約**](/ja/docs/troubleshoot/customer-support#defect-responses): 購入可能な複数のサポートレベルについて確認できます。
  * [**トラブルシューティングのヒント**](/ja/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws#what-to-check-before-logging-an-issue) と [**サポートケースに含める情報**](/ja/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws#information-to-provide-when-logging-an-issue): 開発チームやサポートチームが問題を分析する際に役立つアドバイスを確認できます。
* [**サポート対象バージョン**](/ja/docs/troubleshoot/customer-support/product-support-matrix): どの SDK、ブラウザー、言語がサポート対象かを確認できます。アーキテクトや開発者は、Auth0 と連携できる言語、ライブラリ、SDK をプロジェクトで使用していることを確認するために参照してください。
* [**Feedback Portal**](https://auth0.com/feedback): 製品に関する提案を送ることができます。 (過去に登録した内容をより見やすく管理したい場合は、Support Center 経由でも送信できます。) アーキテクトや開発者は、このサイトを使って Auth0 製品へのフィードバックを送り、今後の機能強化に向けた検討材料としてもらえます。
* [**Professional Services**](/ja/docs/get-started/professional-services): 世界各地のプロフェッショナルサービスチームを活用して、プロジェクトの成功を加速させましょう。プロジェクトオーナーにとって、Auth0 のアイデンティティ専門家がプロジェクトの推進や一時的なスキル不足の補完にどのように役立つかを知るうえで有用です。

<div id="set-up-and-monitor-operations">
  ## 運用の設定と監視
</div>

ローンチの計画を立てる準備ができたら、Auth0 では次のリソースを利用できます。

* [**ローンチ前のアドバイス**](/ja/docs/deploy-monitor/pre-deployment-checks) および [**本番チェック**](/ja/docs/deploy-monitor/pre-deployment-checks/how-to-run-production-checks): ローンチの計画に役立つヒントやツールを確認できます。円滑なローンチに向けた知見を得るために、プロジェクトマネージャー、開発チーム、運用チームはこれらのヒントを確認してください。
* [**運用ポリシー**](/ja/docs/troubleshoot/customer-support/operational-policies): Auth0 のポリシーを把握して、運用requestの対応に必要な期間を理解しておきましょう。ポリシーはチーム全体に役立ちますが、特にプロジェクトオーナーと運用チームは Auth0 の運用ポリシーを把握しておく必要があります。
* [**ステータスダッシュボード**](https://status.auth0.com/): Auth0 サービスの可用性をすばやく確認し、ステータス更新を購読できます。サービス停止が発生することはまれですが、万一発生した場合、Auth0 は根本原因分析を実施し、その結果をこのサイトで公開します。運用チームとサポートチームは、Auth0 のステータスを確認する方法を把握しておく必要があります。
* [**監視エンドポイント**](/ja/docs/deploy-monitor): 監視用エンドポイントを監視インフラストラクチャに統合する方法を確認できます。この情報は、特に運用チームとプロジェクトオーナーに役立ちます。
* [**ログデータ**](/ja/docs/deploy-monitor/logs): Auth0 が提供するログの種類、ログデータの保持期間、さらに分析や長期保存のためにログデータを外部の分析ツールにエクスポートする際に使用できるツールについて確認できます。この情報は、開発者や運用チームに加え、データ保持に関心のあるコンプライアンスチームにも役立ちます。
* [**Auth0 Dashboard の通知**](/ja/docs/get-started/architecture-scenarios/business-to-consumer/operations#notifications): Auth0 からの重要なお知らせを把握できます。Auth0 は随時、重要な情報を Auth0 Dashboard 上で通知し、情報の重大度に応じて、登録済みの Auth0 Dashboard 管理者にメールでも通知します。定期的に Auth0 Dashboard にログインし、上部のベルアイコンを確認して重要なお知らせがないか確認してください。
