<div id="endpoint">
  ## エンドポイント
</div>

`POST /oauth/token`

このエンドポイントを使用すると、認可時に取得したリフレッシュトークンを使ってアクセストークンを更新できます。

<div id="learn-more">
  ## 詳細はこちら
</div>

* [リフレッシュトークン](https://auth0.com/docs/secure/tokens/refresh-tokens)

<div id="headers">
  ## ヘッダー
</div>

<ParamField header="DPoP" type="string">
  リクエスト用の DPoP プルーフです。これは省略可能で、アプリケーションで Demonstrating Proof-of-Possession を使用している場合にのみ必要です。
</ParamField>

<div id="body-parameters">
  ## Body Parameters
</div>

<ParamField body="grant_type" type="string" required>
  使用するフローを指定します。アクセストークンを更新する場合は、`refresh_token` に設定する必要があります。

  指定可能な値: `refresh_token`
</ParamField>

<ParamField body="client_id" type="string" required>
  リクエストを送信するアプリケーションを識別するための、アプリケーションのクライアントIDです。
</ParamField>

<ParamField body="client_secret" type="string">
  アプリケーションのクライアントシークレットです。Token Endpoint Authentication Method が `Post` または `Basic` に設定されている場合に必要です。
</ParamField>

<ParamField body="refresh_token" type="string" required>
  以前にクライアントに発行されたリフレッシュトークンです。このトークンは、新しいアクセストークンの取得に使用されます。
</ParamField>

<ParamField body="scope" type="string">
  リクエストするスコープ権限をスペース区切りで指定したリストです。省略した場合は元のスコープが使用されます。指定した場合は、より少ないスコープをリクエストできます。なお、URLエンコードが必要です。
</ParamField>

<div id="response-schema">
  ## レスポンススキーマ
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    新たに発行されたアクセストークン。
  </ResponseField>

  <ResponseField name="refresh_token" type="string">
    リフレッシュトークン。
  </ResponseField>

  <ResponseField name="id_token" type="string">
    該当する場合は ID トークン。
  </ResponseField>

  <ResponseField name="token_type" type="string">
    トークンの種類。通常は `Bearer`。
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    アクセストークンの有効期間 (秒) 。
  </ResponseField>

  <ResponseField name="scope" type="string">
    アクセストークンのスコープ。
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## レスポンスメッセージ
</div>

| Status | Description                                    |
| ------ | ---------------------------------------------- |
| 200    | アクセストークンの更新に成功しました。                       |
| 400    | 不正なリクエストです。必要なパラメータが不足しているか、リクエストの形式が正しくありません。 |
| 401    | 認証されていません。クライアント認証情報またはリフレッシュトークンが無効です。        |
| 403    | アクセスは許可されていません。リフレッシュトークンが無効であるか、有効期限が切れています。  |
