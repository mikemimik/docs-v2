<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> では、アプリケーションのユーザーを管理するための機能が提供されており、次のようなタスクを実行できます。

* 異なるプロバイダーの個別のユーザーアカウントをリンクして、1 つのプロファイルに関連付けます。詳しくは、[ユーザーアカウントのリンク](/docs/ja/manage-users/user-accounts/user-account-linking)をご覧ください。
* ユーザーアカウントのリンク解除を行い、別々のIDに戻します。
* ユーザーのメタデータを更新します。詳しくは、[ユーザープロファイルでメタデータがどのように機能するかを理解する](/docs/ja/manage-users/user-accounts/metadata)をご覧ください。

<div id="initialize-the-usersapiclient">
  ## UsersAPIClient を初期化する
</div>

まず、`account` とプライマリ ID の `token` を渡して、新しい `UsersAPIClient` インスタンスを作成します。ユーザーをリンクする場合、このプライマリ ID は、データを「保持」する対象のユーザープロファイルであり、そこに他の ID をリンクすることになります。

export const codeExample = `val account = Auth0("{yourClientId}", "{yourDomain}")
val client = UsersAPIClient(account, "token")`;

<div id="link-users">
  ## ユーザーをリンクする
</div>

ユーザーアカウントをリンクすると、ユーザーは自分のどのアカウントからでも認証できるようになります。どのアカウントで認証しても、同じプロファイルでログインできます。アカウントをリンクしない場合、Auth0 はそれぞれの異なるアカウントを別々のプロファイルとして扱います。

`link` メソッドは 2 つのパラメーターを受け取ります。プライマリユーザー ID とセカンダリユーザーのトークン (この ID でログインした後に取得されるトークン) です。ここでいうユーザー ID は、このユーザーアカウントの一意の識別子です。ID が `facebook|1234567890` の形式の場合、必要な ID は区切り文字のパイプ以降の部分です。

```kotlin lines
client
    .link("primary user id", "secondary user token")
    .start(object: Callback<List<UserIdentity>, ManagementException>() {
        override fun onSuccess(payload: List<UserIdentity>) {
            // 更新されたIDを取得しました！アカウントがリンクされました。
        }

        override fun onFailure(error: ManagementException) {
            // エラー！
        }
    })
```

<div id="unlink-users">
  ## ユーザーのリンク解除
</div>

ユーザーのリンクを解除すると、各アカウントはそれぞれ別のプロファイルに戻ります。`unlink` メソッドは 3 つのパラメータを受け取ります。プライマリーユーザー ID、セカンダリーユーザー ID、そしてセカンダリープロバイダー (セカンダリーユーザーのプロバイダー) です。

```kotlin lines
users
    .unlink("primary user id", "secondary user id", "secondary provider")
    .start(object: Callback<List<UserIdentity>, ManagementException>() {
        override fun onSuccess(payload: List<UserIdentity>) {
            // 更新されたIDを取得しました！アカウントのリンクが解除されました。
        }

        override fun onFailure(error: ManagementException) {
            // エラー！
        }
    })
```

アカウントをリンクしても、セカンダリアカウントのメタデータはプライマリアカウントのメタデータと統合されません。同様に、2つのアカウントをリンク解除しても、セカンダリアカウントにプライマリアカウントのメタデータが保持されることはありません。

<div id="updating-user-metadata">
  ## ユーザーのメタデータを更新する
</div>

ユーザーのメタデータを更新するには、`metadata` オブジェクトを作成してから、ユーザー ID と `metadata` オブジェクトを渡して `updateMetadata` メソッドを呼び出します。このオブジェクト内の値は、同じキーを持つ既存の値を上書きし、ユーザーのメタデータにまだ存在しないキーについては新しい値を追加します。

```kotlin lines
val metadata = mutableMapOf<String, Any?>()
metadata.put("name", listOf("My", "Name", "Is"));
metadata.put("phoneNumber", "1234567890");

users
    .updateMetadata("user id", metadata)
    .start(object: Callback<UserProfile, ManagementException>() {
        override fun onSuccess(payload: UserProfile) {
            // メタデータが更新されました
        }

        override fun onFailure(error: ManagementException) {
            // エラー！
        }
    })
```
