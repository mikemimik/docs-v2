      <ParamField body="MfaPollingOptions" type="object">
        MFAプッシュ通知のポーリング処理を管理するための設定オプションです。

        ポーリングセッションの開始時にこれらのオプションを指定すると、
        チャレンジエンドポイントへの問い合わせ頻度や、
        成功時または失敗時の処理方法を制御できます。

        <Expandable title="プロパティ">
          <ParamField body="intervalMs?" type="number">
            連続するポーリングリクエスト間の省略可能な間隔 (**ミリ秒**) 。

            * 省略した場合は、SDK の内部デフォルト間隔 (通常は 5000 ミリ秒) が使用されます。
            * 応答性とレート制限への配慮のバランスが取れた値を選択してください。
          </ParamField>

          <ParamField body="onCompleted()?" type="void">
            MFAプッシュチャレンジが正常に承認され、
            ポーリングの完了時に実行される省略可能なコールバック。

            * **1 回だけ**呼び出され、その後ポーリングは自動的に停止します。
            * ログインフローを次に進める際に便利です (たとえば `continue()` の呼び出し) 。
          </ParamField>

          <ParamField body="onError()?" type="void">
            ポーリング中にエラーが発生した場合に呼び出される省略可能なコールバックです。

            以下を含む [ULError](/ja/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) オブジェクトを受け取ります。

            * `status` — 失敗したリクエストの HTTP ステータスコード

            * `responseText` — 利用可能な場合の生のレスポンス本文

            * エラーイベントごとに 1 回呼び出されます。ポーリングはエラーの種類に応じて再試行されることがあります。

            * エラーメッセージの表示や、カスタムの再試行/中止ロジックのトリガーに使用できます。
