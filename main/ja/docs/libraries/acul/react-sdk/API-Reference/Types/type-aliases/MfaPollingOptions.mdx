---
title: "型エイリアス: MfaPollingOptions"
---

```ts Example
type MfaPollingOptions = {
  intervalMs?: number;
  onCompleted?: () => void;
  onError?: (error: Error) => void;
}
```

<ParamField body="MfaPollingOptions" type="object">
  MFA プッシュ通知のポーリング処理を管理するための設定オプションです。

  ポーリングセッションの開始時にこれらのオプションを指定すると、チャレンジエンドポイントへの問い合わせ頻度や、
  成功時・失敗時の処理方法を制御できます。

  <Expandable title="プロパティ">
    <ParamField body="intervalMs" type="number">
      連続するポーリングリクエストの間隔を、**ミリ秒**単位で指定する任意の値です。

      * 省略した場合は、SDK の内部デフォルト間隔が使用されます (通常は 5000 ms) 。
      * 応答性とレート制限への配慮のバランスが取れる値を選択してください。

      ```ts Default Value
      実装によって異なります（一般的には 5000 ms）
      ```
    </ParamField>

    <ParamField body="onCompleted" type="() => void">
      MFA プッシュチャレンジが正常に
      承認され、ポーリングが完了した時点で実行される任意のコールバックです。

      * **1 回だけ**呼び出され、その後ポーリングは自動的に停止します。
      * ログインフローを次に進める際に便利です (例: `continue()` を呼び出す) 。
    </ParamField>

    <ParamField body="onError" type="(error: Error) => void">
      ポーリング中にエラーが発生した場合に呼び出される任意のコールバックです。

      次の内容を含む [ULError](/docs/ja/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) オブジェクトを受け取ります。

      * `status` — 失敗したリクエストの HTTP ステータスコード

      * `responseText` — 利用可能な場合の生のレスポンス本文

      * エラーイベントごとに 1 回呼び出されます。エラーの種類によっては、ポーリングが再試行される場合があります。

      * エラーメッセージを表示したり、カスタムの再試行 / 中止ロジックを実行したりするために使用します。

      **プロパティ**

      <ParamField body="error" type={<a href="/docs/ja/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error">Error</a>}>
        失敗したポーリングリクエストの HTTP ステータスコードと生のレスポンス本文を含むエラーオブジェクトです。
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>
