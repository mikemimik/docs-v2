<ParamField body="continueWithNewNickname" type="Promise<void>">
  WebAuthn セキュリティキーの新しいニックネームを送信します。
  この操作は、ユーザーが新しいニックネームを入力して「Save」または「Continue」ボタンをクリックすることに相当します。
  SDK はこの新しいニックネームを Auth0 の `/u/mfa-webauthn-change-key-nickname` エンドポイントに POST します。

  新しいニックネームが有効で、更新が正常に完了すると、Auth0 は通常、ユーザーを
  次の適切な画面 (たとえば MFA 要素の管理画面に戻る、または成功確認画面) にリダイレクトします。
  ニックネームが無効な場合 (たとえば空、長すぎる、短すぎるなど) 、Auth0 は
  &#39;mfa-webauthn-change-key-nickname&#39; 画面を再レンダリングし、SDK の
  コンテキスト内の `transaction.errors` 配列がバリデーション失敗の詳細で更新されます。

  フォーム送信が開始されると解決される Promise です。
  通常はリダイレクトまたは
  ページの再レンダリングとなるため、解決時にデータが直接返されることはありません。

  #### 例外

  `payload.nickname` が指定されていない場合、または文字列でない場合、
  エラーをスローします。
  また、`FormHandler` が送信中に回復不能な問題 (例: ネットワークエラー) に遭遇した場合もエラーをスローします。
  Auth0 からのバリデーションエラー (無効なニックネームなど) は JavaScript エラーとしてスローされるのではなく、
  操作後に `this.transaction.errors` に反映されます。

  ```typescript Example
  // 'sdk' は MfaWebAuthnChangeKeyNickname のインスタンスであるとします
  const newNickname = "My Favorite YubiKey"; // ユーザー入力の値
  try {
    await sdk.continueWithNewNickname({ nickname: newNickname });
    // 成功した場合、ページはリダイレクトされます。
  } catch (error) {
    // この catch は、SDK 呼び出し自体で発生した予期しないエラーを処理するためのものです。
    console.error("Failed to submit the new nickname:", error);
  }
  // await の後は、サーバー側のバリデーションエラーがないか必ず sdk.transaction.errors を確認してください。
  if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
    sdk.transaction.errors.forEach(err => {
      if (err.field === 'nickname') {
        // ニックネーム入力欄に関連する err.message を表示します。
        // 例: "Name is too short", "Name is required"
      }
    });
  }
  ```
