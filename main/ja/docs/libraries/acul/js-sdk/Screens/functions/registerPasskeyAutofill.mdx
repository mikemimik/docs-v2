---
title: "registerPasskeyAutofill"
---

> **registerPasskeyAutofill**(`inputId`?: `string`): `Promise`&lt;`void`&gt;

`mediation: "conditional"` を指定した `navigator.credentials.get()` を使用して、受動的な WebAuthn 資格情報リクエストを初期化します。サポートされている場合、ブラウザーは保存済みのパスキーをユーザー名フィールドのオートコンプリートのドロップダウンに直接表示できます。パスキーを選択すると、追加のユーザー操作なしで認証が完了します。

ログイン画面の初期化時 (たとえばページの読み込み時) に、これを一度呼び出します。ブラウザーが Conditional Mediation をサポートしていない場合、このメソッドは安全に no-op します。

<div id="parameters">
  ## パラメータ
</div>

<ParamField body="inputId?" type="string">
  ユーザー名用の`<input>`要素の ID (`#`は不要) 。例: `"username"`。指定した場合、SDK はその要素が存在することを確認し、`<input>`要素であることを検証したうえで、`autocomplete`属性を`"username webauthn"`に上書きします。

  省略した場合は、`<input>`要素を手動で設定する必要があります。

  ```html
  <input id="username" autocomplete="username webauthn" />
  ```
</ParamField>

<div id="returns">
  ## 戻り値
</div>

`Promise`&lt;`void`&gt;

Conditional UI の登録が完了すると解決されます。

<div id="example">
  ## 例
</div>

```typescript
import LoginId from '@auth0/auth0-acul-js/login-id';

async function initializeLogin() {
  const loginId = new LoginId();
  // 対応するHTML inputが存在することを確認してください:
  // <input id="username" autocomplete="username webauthn" />
  await loginId.registerPasskeyAutofill('username');
}

initializeLogin().catch(console.error);
```

<div id="gotchas">
  ## 注意点
</div>

* `autocomplete` 属性は、**必ず正確に** `"username webauthn"` である必要があります。`"email"` や `"text"` などの無関係なトークンを含めると、ブラウザーでパスキーのドロップダウンが表示されなくなります。
* この属性を上書きするのは意図的なものであり、ブラウザー間で一貫した動作を実現するために必要です。既存の autocomplete 値を結合したり拡張したりすることを前提にしないでください。
* ブラウザーが Conditional Mediation をサポートしていない場合、SDK は安全に no-op します。

<div id="remarks">
  ## 注記
</div>

このメソッドは内部の `registerPasskeyAutofill()` ユーティリティに処理を委譲し、リクエストのライフタイムを管理するためのバックグラウンド `AbortController` を返します。ページのライフサイクルごとに 1 回だけ呼び出してください。
