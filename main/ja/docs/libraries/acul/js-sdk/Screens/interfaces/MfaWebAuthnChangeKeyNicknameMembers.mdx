   *                 遭遇した場合にエラーをスローします。

<ParamField body="continueWithNewNickname" type="Promise<void>">
  WebAuthn セキュリティキーの新しいニックネームを送信します。
  このアクションは、ユーザーが新しいニックネームを入力して「保存」または「続行」ボタンをクリックする操作に対応します。
  SDK は、この新しいニックネームを Auth0 の `/u/mfa-webauthn-change-key-nickname` エンドポイントに POST します。

  新しいニックネームが有効で、更新に成功した場合、Auth0 は通常、ユーザーを
  次の適切な画面 (例: MFA 認証要素管理画面に戻る、または成功確認画面) にリダイレクトします。
  ニックネームが無効な場合 (例: 空、長すぎる、短すぎる) 、Auth0 は
  `mfa-webauthn-change-key-nickname` 画面を再レンダリングし、SDK の
  コンテキスト内の `transaction.errors` 配列がバリデーション失敗の詳細で更新されます。

  フォーム送信が開始されると解決される Promise です。
  解決時に直接データが返されることはありません。通常はリダイレクトまたは
  ページの再レンダリングが発生するためです。

  #### 例外

  `payload.nickname` が指定されていない場合、または文字列でない場合はエラーをスローします。
  また、送信中に `FormHandler` で回復不能な問題 (例: ネットワークエラー) が発生した場合も同様です。
  Auth0 からのバリデーションエラー (無効なニックネームなど) は JavaScript エラーとしてはスローされず、
  操作後に `this.transaction.errors` に反映されます。

  ```typescript Example
  // 'sdk' が MfaWebAuthnChangeKeyNickname のインスタンスであると仮定します
  const newNickname = "My Favorite YubiKey"; // ユーザー入力の値
  try {
    await sdk.continueWithNewNickname({ nickname: newNickname });
    // 成功した場合、ページはリダイレクトされます。
  } catch (error) {
    // この catch は、SDK 呼び出し自体で発生した予期しないエラーを処理するためのものです。
    console.error("Failed to submit the new nickname:", error);
  }
  // await の後は、サーバー側のバリデーションエラーがないか常に sdk.transaction.errors を確認してください。
  if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
    sdk.transaction.errors.forEach(err => {
      if (err.field === 'nickname') {
        // ニックネーム入力フィールドに関連する err.message を表示します。
        // 例: "Name is too short", "Name is required"
      }
    });
  }
  ```
