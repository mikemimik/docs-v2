ConsentMembers

````ts Example
export interface ConsentMembers extends BaseMembers {
  /**
   * Consentの画面の特定のプロパティとデータへのアクセスを提供します。
   * リクエストされている`scopes`のリストと`hideScopes`フラグを含みます。
   * @type {ScreenMembersOnConsent}
   */
  screen: ScreenMembersOnConsent;

  /**
   * リクエストされた権限を承認（グラント）するユーザーの決定を送信します。
   * このactionは`action: "accept"`とともに`/u/consent` endpointにPOSTします。
   *
   * @param {CustomOptions} [payload] - 省略可能。requestとともに送信するカスタムキーと値のペアを格納するオブジェクト。
   *                                    これらのParametersはサーバーに送信されるフォームデータに含まれます。
   * @returns {Promise<void>} acceptのactionが正常に送信されると解決されるPromise。
   *                          成功した場合、Auth0は通常ユーザーをアプリケーションにリダイレクトするか、
   *                          認証フローの次のステップに進めます。
   * @throws {Error} フォームの送信が失敗した場合（例：ネットワークエラー、無効なstate）にエラーをスローします。
   *                 サーバー側のエラー（"invalid_request"など）は、JavaScriptエラーとしてスローされるのではなく、
   *                 操作後に`this.transaction.errors`に反映されます。
   *
   * @example
   * ```typescript
   * // 'consentManager'がConsentの画面のSDKクラスのインスタンスであると仮定
   * try {
   *   await consentManager.accept();
   *   // 成功した場合、ページは通常リダイレクトされます。
   * } catch (error) {
   *   // 送信中に発生した予期しないエラーを処理します。
   *   console.error("Failed to submit consent acceptance:", error);
   * }
   * // awaitの後、サーバー側のバリデーションの問題についてconsentManager.transaction.errorsを確認してください。
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * リクエストされた権限を拒否（却下）するユーザーの決定を送信します。
   * このactionは`action: "deny"`とともに`/u/consent` endpointにPOSTします。
   *
   * @param {CustomOptions} [payload] - 省略可能。requestとともに送信するカスタムキーと値のペアを格納するオブジェクト。
   *                                    これらのParametersはサーバーに送信されるフォームデータに含まれます。
   * @returns {Promise<void>} denyのactionが正常に送信されると解決されるPromise。
   *                          成功した場合、Auth0は通常ユーザーをリダイレクトし、アクセス拒否メッセージを表示するか、
   *                          アプリケーションにエラーを返す可能性があります。
   * @throws {Error} フォームの送信が失敗した場合（例：ネットワークエラー、無効なstate）にエラーをスローします。
   *                 サーバー側のエラーは`this.transaction.errors`に反映されます。
   *
   * @example
   * ```typescript
   * // 'consentManager'がConsentの画面のSDKクラスのインスタンスであると仮定
   * try {
   *   await consentManager.deny({ reason: "user_declined" }); // カスタムオプションの例
   *   // 成功した場合、ページは通常リダイレクトされます。
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // awaitの後、サーバー側のバリデーションの問題についてconsentManager.transaction.errorsを確認してください。
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}

<ParamField body="accept" type="Promise<void>">
  要求された権限を受け入れる (付与する) というユーザーの決定を送信します。
  このアクションは、`action: "accept"` を付けて `/u/consent` エンドポイントに POST します。

  accept アクションが正常に送信されると解決される Promise です。
  成功すると、Auth0 は通常、ユーザーをアプリケーションにリダイレクトするか、
  認証フローの次のステップへ進めます。

  #### 例外

  フォームの送信に失敗した場合 (たとえば、ネットワークの問題や無効な state) にエラーをスローします。
  サーバー側のエラー (&quot;invalid&#95;request&quot; など) は、JavaScript エラーとしてスローされるのではなく、
  操作後に `this.transaction.errors` に反映されます。

  ```typescript Example
  // 'consentManager' が Consent 画面 SDK クラスのインスタンスであると仮定します
  try {
    await consentManager.accept();
    // 成功した場合、通常はページがリダイレクトされます。
  } catch (error) {
    // 送信処理自体で発生した予期しないエラーを処理します。
    console.error("Failed to submit consent acceptance:", error);
  }
  // await の後で、サーバー側のバリデーションの問題について consentManager.transaction.errors を確認します。
  ```

  このアクションは、`action: "deny"` を付けて `/u/consent` エンドポイントに POST します。
