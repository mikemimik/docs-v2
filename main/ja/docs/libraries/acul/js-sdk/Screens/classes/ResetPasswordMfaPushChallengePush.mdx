<ParamField body="changeLanguage" type="Promise<void>">
  このメソッドは、Universal Login ページの表示言語を変更します。

  ```typescript Example
  import ResetPasswordMfaPushChallengePush from '@auth0/auth0-acul-js/reset-password-mfa-push-challenge-push';
  const resetPasswordMfaPushChallengePushManager = new ResetPasswordMfaPushChallengePush();
  resetPasswordMfaPushChallengePushManager.changeLanguage({
    language: 'fr',
  });
  ```

  **メソッドのパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/ja/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions)。
    </ParamField>

    <ParamField body="language" type="string" required>
      使用する言語の locale コード (例: `'en'`、`'fr'`、`'es'`) 。
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      `'session'` に設定すると、選択した言語は session の間保持されます。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="continue" type="Promise<void>">
  このメソッドは、パスワードリセットフロー中の MFA ステップを続行するために、プッシュ通知チャレンジを承認します。

  ```typescript Example
  import ResetPasswordMfaPushChallengePush from '@auth0/auth0-acul-js/reset-password-mfa-push-challenge-push';
  const resetPasswordMfaPushChallengePushManager = new ResetPasswordMfaPushChallengePush();
  await resetPasswordMfaPushChallengePushManager.continue();
  ```
</ParamField>

<ParamField body="enterCodeManually" type="Promise<void>">
  このメソッドは、プッシュ通知の承認を待つ代わりに、手動でコードを入力する画面に切り替えます。

  ```typescript Example
  import ResetPasswordMfaPushChallengePush from '@auth0/auth0-acul-js/reset-password-mfa-push-challenge-push';
  const resetPasswordMfaPushChallengePushManager = new ResetPasswordMfaPushChallengePush();
  await resetPasswordMfaPushChallengePushManager.enterCodeManually();
  ```
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/ja/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>

<ParamField body="pollingManager" type={<span><a href="/docs/ja/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  このメソッドは、MFA プッシュチャレンジの polling を開始し、管理します。指定した間隔でプッシュ通知が承認されたかどうかを繰り返し確認し、承認が検出されると `onCompleted` コールバックを呼び出します。HTTP 429 の rate limit レスポンスは内部的に処理され、polling は rate limit がリセットされるまで待機してから再試行します。

  返される [`MfaPushPollingControl`](/docs/ja/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) では、polling のライフサイクルを管理するために `startPolling()`、`stopPolling()`、`isRunning()` を利用できます。

  ```typescript Example
  import ResetPasswordMfaPushChallengePush from '@auth0/auth0-acul-js/reset-password-mfa-push-challenge-push';
  const resetPasswordMfaPushChallengePushManager = new ResetPasswordMfaPushChallengePush();
  const control = resetPasswordMfaPushChallengePushManager.pollingManager({
    intervalMs: 5000,
    onCompleted: () => resetPasswordMfaPushChallengePushManager.continue(),
    onError: (error) => console.error('Polling error:', error),
  });

  control.startPolling();

  // 必要に応じて polling を停止します
  control.stopPolling();
  ```

  **メソッドのパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [MfaPollingOptions](/docs/ja/libraries/acul/js-sdk/Screens/interfaces/MfaPollingOptions)。
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      polling の試行間隔 (ミリ秒) 。
    </ParamField>

    <ParamField body="onCompleted?" type="() => void">
      プッシュチャレンジが承認され、polling が自動的に停止したときに呼び出されるコールバック。
    </ParamField>

    <ParamField body="onError?" type="(error: Error) => void">
      polling 中にエラーが発生した場合に呼び出されるコールバック。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="resendPushNotification" type="Promise<void>">

</ParamField>
