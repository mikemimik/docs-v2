---
title: "サインアップ"
description: "Universal Login の `signup` 画面をカスタマイズするために使用できる、すべてのプロパティとメソッドについて説明します。"
---

Signup クラスは、`signup` 画面の機能を提供します。この画面では、ユーザーの識別子とパスワードを収集します。テナントによっては、この識別子としてメールアドレス、電話番号、またはユーザー名を使用できます。

<div id="constructors">
  ## コンストラクター
</div>

サインアップ画面マネージャーのインスタンスを作成します:

```typescript Example
import Signup from '@auth0/auth0-acul-js/signup';
const signupManager = new Signup();
signupManager.signup({
  email: 'test@example.com',
  password: 'P@$wOrd123!',
});
```

<div id="methods">
  ## メソッド
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  このメソッドは、Universal Login ページの表示言語を変更します。

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.changeLanguage({
    language: 'fr',
  });
  ```

  **メソッドのパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [LanguageChangeOptions](/ja/docs/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions)。
    </ParamField>

    <ParamField body="language" type="string" required>
      使用する言語のロケールコードです (例: `'en'`、`'fr'`、`'es'`) 。
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      `'session'` に設定すると、選択した言語はセッション中保持されます。
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      `ulp-` で始まる追加のカスタムフィールドです (例: `'ulp-custom-field'`) 。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedSignup" type="Promise<void>">
  このメソッドは、ソーシャル サインアップ フォームの送信を処理します。[signup](#signup) メソッドと似ていますが、ソーシャル サインアップ 用に使用されます。

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.federatedSignup({
    connection: 'google-oauth2',
  });
  ```

  **メソッドのパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [FederatedSignupPayloadOptions](/ja/docs/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupPayloadOptions)。
    </ParamField>

    <ParamField body="connection" type="string" required>
      使用するソーシャル接続名です。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/ja/docs/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  このメソッドは、コンテキストから トランザクション エラーの配列を取得します。存在しない場合は空の配列を返します。トランザクション コンテキスト内の error object の配列です。
</ParamField>

<ParamField body="getSignupIdentifiers" type={<span><a href="/ja/docs/libraries/acul/js-sdk/Screens/interfaces/Identifier">Identifier</a>[] | null</span>}>
  このメソッドは、サインアップ フォームで有効な 識別子 の一覧を返し、トランザクション の設定に基づいて各項目が必須か任意かを示します。識別子 object の配列です (例: email、phone、username) 。

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const identifiers = signupManager.getSignupIdentifiers();
  // [{ type: 'email', required: true }, { type: 'username', required: false }]
  ```
</ParamField>

<ParamField body="pickCountryCode" type="Promise<void>">
  このメソッドは、ユーザーを国番号の選択リストにリダイレクトし、電話番号の国番号プレフィックスを変更できるようにします。

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.pickCountryCode();
  ```
</ParamField>

<ParamField body="signup" type="Promise<void>">
  このメソッドは、サインアップ フォームの送信を処理します。

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.signup({
    email: 'test@example.com',
    password: 'P@$wOrd123!',
  });
  ```

  **メソッドのパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [SignupPayloadOptions](/ja/docs/libraries/acul/js-sdk/Screens/interfaces/SignupPayloadOptions)。
    </ParamField>

    <ParamField body="email?" type="string">
      ユーザーのメールアドレスです。
    </ParamField>

    <ParamField body="username?" type="string">
      ユーザーの 識別子 です。
    </ParamField>

    <ParamField body="phoneNumber?" type="string">
      ユーザーの電話番号です。
    </ParamField>

    <ParamField body="password?" type="string">
      ユーザーのパスワードです。
    </ParamField>

    <ParamField body="captcha?" type="string">
      captcha コード、または captcha プロバイダーからのレスポンスです。Auth0 テナントで [ボット検出](/ja/docs/secure/attack-protection/bot-detection) が有効になっている場合、このプロパティは必須です。
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      `ulp-` で始まる追加のカスタムフィールドです (例: `'ulp-custom-field'`) 。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validatePassword" type={<span><a href="/ja/docs/libraries/acul/js-sdk/Screens/interfaces/PasswordValidationResult">PasswordValidationResult</a></span>}>
  このメソッドは、パスワードが有効かどうかと、その理由を示す [PasswordValidationResult](/ja/docs/libraries/acul/js-sdk/Screens/interfaces/PasswordValidationResult) 型のオブジェクトを返します。

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const result = signupManager.validatePassword('P@$wOrd123!');
  ```

  **メソッドのパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="password" type="string" required>
      ユーザーのパスワード。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validateUsername" type={<span><a href="/ja/docs/libraries/acul/js-sdk/Screens/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  このメソッドは、指定されたユーザー名を、トランザクションコンテキストで定義された現在のユーザー名ポリシーに対して検証します。ユーザー名が有効かどうかと、その理由を示すオブジェクトを返します。

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const result = signupManager.validateUsername('myusername');
  // result => { valid: true, errors: [] }
  ```

  **メソッドのパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="username" type="string" required>
      バリデーション対象のユーザー名文字列。
    </ParamField>
  </Expandable>
</ParamField>
