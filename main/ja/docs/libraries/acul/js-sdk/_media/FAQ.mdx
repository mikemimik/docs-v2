<div id="where-can-i-see-errors-related-to-my-screen-and-how-to-handle-them">
  ## 画面に関連するエラーはどこで確認でき、どう対処できますか
</div>

SDK には、処理中のエラーを検出して確認するための方法が複数用意されています。

1. **画面固有のエラー向けの `getErrors` メソッド**
   トップレベルのゲッター関数 `getErrors` を使うと、バックエンドの検証が失敗した際に、詳細な認証エラーを取得できます。
   ```javascript
      import Login from "@auth0/auth0-acul-js/login";

      const loginIdManager = new Login();

      const errors = loginIdManager.getErrors();
      if (errors) {
      console.error("Login failed:", errors);
   }
   ```
2. **画面固有のエラー**
   各画面には `transaction.errors` プロパティがあり、その画面に固有のエラー一覧を返します。これらのエラーはリクエストの最終検証後に生成されるため、対応が必要な問題を特定するのに役立ちます。

これらの方法を使うことで、SDK フロー全体を通してエラーを簡単に追跡し、解決できます。

<div id="can-i-use-this-sdk-from-a-cdn">
  ## このSDKはCDN経由で利用できますか？
</div>

現在、このSDKはnpmレジストリからのインストールでのみ利用でき、CDNから直接使用することはできません。

SDKをインストールするには、次のコマンドを実行してください。

```sh
npm install @auth0/auth0-acul-js
```

インストール後、JavaScript、TypeScript、または ReactJS のプロジェクトでインポートして使用できます。

<div id="can-i-build-a-reactjs-application-with-this-sdk">
  ## このSDKでReactJSアプリケーションを構築できますか？
</div>

はい、このSDKはReactJSアプリケーションに簡単に組み込めます。

このSDKはVanilla JavaScriptで記述されていますが、React、Angular、Vueをはじめとする、JavaScriptベースのあらゆるフレームワークと完全に互換性があります。Reactのコンポーネントやフック内でも、SDKをスムーズに利用できます。
