この記事では、シングルページアプリ (SPA) を [auth0.js](/ja/docs/libraries/auth0js) から [auth0-spa-js](/ja/docs/libraries/auth0-single-page-app-sdk) へ移行する方法を紹介します。以下では、auth0.js を使用するシナリオと、それに相当する auth0-spa-js のコードを示します。

<div id="functionality-that-cannot-be-migrated">
  ## 移行できない機能
</div>

auth0.js のすべての機能を auth0-spa-js にそのまま移行できるわけではありません。直接移行できないシナリオには、次のようなものがあります。

* 埋め込みの[ユーザー名/パスワードによるログイン](https://auth0.github.io/auth0.js/WebAuth.html#login)および埋め込みの[パスワードレスログイン](https://auth0.github.io/auth0.js/WebAuth.html#passwordlessLogin)
* ユーザーの[サインアップ](https://auth0.github.io/auth0.js/WebAuth.html#signup)
* [/userinfo エンドポイントからユーザープロファイルを取得する](https://auth0.github.io/auth0.js/Authentication.html#userInfo)
* [ユーザーのパスワード変更用メールをリクエストする](https://auth0.github.io/auth0.js/WebAuth.html#changePassword)
* [Management API でユーザーをリンクする](https://auth0.github.io/auth0.js/Management.html#linkUser)
* [Management API でユーザーを取得する](https://auth0.github.io/auth0.js/Management.html#getUser)
* [Management API でユーザー属性を更新する](https://auth0.github.io/auth0.js/Management.html#patchUserAttributes)
* [Management API でユーザーのメタデータを更新する](https://auth0.github.io/auth0.js/Management.html#patchUserMetadata)
* また、Auth0.js では設定できても、auth0-spa-js には対応するものがないオプションもあります。たとえば `responseType` です。各オプションに 1:1 で対応していないのには理由があります。この場合、`responseType` は不要です。というのも、この SDK は SPA 専用であり、レスポンスタイプを変更する必要がないためです。

<div id="authentication-parameters-are-not-modified-anymore">
  ## 認証パラメーターは変更されなくなりました
</div>

Auth0.js は、設定したカスタムパラメーターを内部で `camelCase` から `snake_case` に変換します。たとえば、`deviceType: 'offline'` を設定すると、auth0.js は実際には `device_type: 'offline'` をサーバーに送信します。

auth0-spa-js を使用する場合、カスタムパラメーターは `camelCase` から `snake_case` に**変換されません**。送信したパラメーターは、そのまま <Tooltip tip="認可サーバー: ユーザーのアクセス範囲の定義に関与する集中管理型サーバーです。たとえば、認可サーバーは、ユーザーが利用できるデータ、タスク、機能を制御できます。" cta="用語集を見る" href="/ja/docs/glossary?term=authorization+server">認可サーバー</Tooltip> に渡されます。

window.addEventListener('load', () => {
  auth0.parseHash({ hash: window.location.hash }, function(err, authResult) {

<div id="get-the-user-information">
  ## ユーザー情報を取得する
</div>

<div id="auth0js">
  ### auth0.js
</div>

`userInfo()` 関数は [`/userinfo` エンドポイント](https://auth0.com/docs/api/authentication#user-profile) にリクエストを送信し、ユーザープロファイルを返します。

* [userInfo()](https://auth0.github.io/auth0.js/Authentication.html#userInfo)

```jsx lines
window.addEventListener('load', () => {
  auth0.client.userInfo(accessToken, function(err, user) {
    console.log(user)
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

auth0.js とは異なり、Auth0 SPA SDK はユーザープロファイルを取得するために [`/userinfo エンドポイント`](https://auth0.com/docs/api/authentication#user-profile) を呼び出しません。代わりに、`Auth0Client.getUser()` はデコードした `id_token` からユーザー情報を返します。

* [Auth0Client.getUser()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getUser)

```jsx lines
window.addEventListener('load', async () => {
  const user = await auth0.getUser();
  console.log(user);
});
```

<div id="refresh-tokens">
  ## リフレッシュトークン
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({}, function(err, authResult) {
    // 認証トークンまたはエラー
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Auth0 SPA SDK は、<Tooltip tip="Access Token: API へのアクセスに使用される認可クレデンシャルで、opaque な文字列または JWT の形式を取ります。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">Access Token</Tooltip> の更新を自動的に処理します。`getTokenSilently` を呼び出すたびに、有効な Access Token が返されるか、Auth0 にセッションがない場合はエラーが返されます。

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently();
});
```

<div id="get-a-token-for-a-different-audience-or-with-more-scopes">
  ## 別のaudience用、または追加のスコープを持つトークンを取得する
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  }, function(err, authResult) {
    // 認証トークンまたはエラー
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Auth0 SPA SDK は、Access Token のリフレッシュを自動的に処理します。`getTokenSilently` を呼び出すたびに、有効な Access Token が返されるか、Auth0 にセッションがない場合はエラーが返されます。

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)
* [GetTokenSilentlyOptions](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```

[getTokenWithPopup](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenWithPopup) を使用してポップアップを開き、ユーザーが新しい API に同意できるようにします。

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenWithPopup({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```
