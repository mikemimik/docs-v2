多くの接続で新しい<Tooltip tip="OAuth 2.0: 認可プロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を見る" href="/docs/ja/glossary?term=OAuth">OAuth</Tooltip>クレデンシャルを作成する際は、アプリケーションの署名に使用している証明書のSHA-1を指定する必要があります。[Auth0 Dashboard](https://manage.auth0.com/#)でアプリケーションの設定を完了する際には、SHA-256の値も指定する必要があります。

アプリケーションの署名に使用している証明書を確認してください。まだない場合は生成できます。本番用アプリケーションでは、この対応を行う必要があります。

開発中は、SDKのインストール時に自動生成されたデフォルトの`android.keystore`証明書でアプリケーションに署名できます。この例では、このデフォルトのキーストアを使用します。このキーストアを使ってキーハッシュを生成するには、以下の例に従ってください。

<div id="generating-your-key-hashes">
  ## キーハッシュの生成
</div>

**Windows の場合:**

`keytool -list -v -keystore "%USERPROFILE%\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android`

**Linux / macOS の場合:**

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

<div id="using-your-key-hashes">
  ## キーハッシュを使用する
</div>

キーハッシュが出力されたら、生成された SHA256 値をコピーし、[Auth0 Dashboard](https://manage.auth0.com/#/applications) でアプリケーションの設定を開きます。「Show Advanced Settings」をクリックし、「Device Settings」タブの「Android」セクションで、「App Package Name」にアプリケーションのパッケージ名を入力し、「キーハッシュ」フィールドにコピーした SHA256 値を入力します。変更を保存することを忘れないでください。

アプリケーションの設定に Callback URL を追加せず、またアプリケーションのデバイス設定に キーハッシュ を追加しない場合、Auth0サーバーは呼び出し結果をアプリケーションに返しません。
