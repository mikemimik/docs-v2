<div id="native-provider-google">
  ## Native Provider - Google
</div>

**Lock** の有無にかかわらず、Google AuthProvider を使ってログインできます。[設定](#setup) セクションの手順に必ず従ってください。

[Lock-Google.Android](https://github.com/okta-graveyard/Lock-Google.Android)
を利用するには、Android API 15 以降と Google Play Services 10.+ が必要です

<div id="latest-version-of-lock-google">
  ### Lock-Google の最新バージョン
</div>

Lock-Google は [Maven Central](https://central.sonatype.com/) から利用できます。インストールするには、`build.gradle` に以下の行を追加するだけです。

`compile 'com.auth0.android:lock-google:1.+'`

最新バージョンは、リポジトリの [README](https://github.com/okta-graveyard/Lock-Google.Android#install) または [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock-google) で確認できます。

<div id="lock-google-setup">
  ### Lock-Google の設定
</div>

<div id="google-cloud-console">
  #### Google Cloud Console
</div>

1. [Google Cloud Console](https://console.cloud.google.com) に移動し、新しいプロジェクトを作成します。
2. 有効なメールアドレスと名前を少なくとも入力して、[OAuth Consent Screen](https://console.cloud.google.com/apis/credentials/consent) の設定を完了します。
3. 左側のナビゲーション ドロワーで、[Credentials](https://console.cloud.google.com/apis/credentials) をクリックします。
4. [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) ボタンをクリックし、**OAuth client ID** を選択して新しい資格情報を作成します。次に **Web アプリケーション** を選択し、&quot;Auth0 Server Google-OAuth&quot; などの名前を付けます。**許可されたリダイレクト URI** には、コールバック URL (`https://{yourDomain}/login/callback` のような形式) を入力します。フィールドを離れる前に必ず ENTER キーを押し、その後 Create ボタンをクリックしてください。後で使用するので、`CLIENT ID` と `CLIENT SECRET` の値を控えておきます。
5. [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) ボタンをクリックし、**OAuth client ID** を選択して新しい資格情報を作成します。次に **Android** を選択し、&quot;Auth0 Android Google-OAuth&quot; などの名前を付けます。アプリケーションの署名に使用している証明書の **SHA-1** を取得し、最初のフィールドに入力します。SHA-1 の取得方法については、[this](#certificate-fingerprints) セクションを参照してください。最後に、Android アプリケーションの **Package Name** を最後のフィールドに入力し、Create ボタンをクリックします。後で使用するので、`CLIENT ID` の値を控えておきます。

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Auth0 Dashboard に移動し、[Social Connections](https://manage.auth0.com/#/connections/social) をクリックします。
2. **Google** をクリックすると、ダイアログが表示されます。
3. 上記の **Google Cloud Console** セクションのステップ 4 で取得した `CLIENT ID` の値を、&quot;Client ID&quot; フィールドに入力します。
4. 上記の **Google Cloud Console** セクションのステップ 4 で取得した `CLIENT SECRET` の値を、&quot;Client Secret&quot; フィールドに入力します。
5. 上記の **Google Cloud Console** セクションのステップ 5 で取得した `CLIENT ID` を、&quot;Allowed Mobile Client IDs&quot; フィールドに入力します。
6. Save ボタンをクリックします。
7. Auth0 Dashboard に移動し、[アプリケーション](https://manage.auth0.com/#/applications) をクリックします。まだ作成していない場合は、先に作成してからアプリケーションの設定ページを開いてください。
8. ページ下部の &quot;Show Advanced Settings&quot; リンクをクリックし、&quot;Device Settings&quot; タブに移動します。
9. Android セクションで、**Package Name** にアプリケーションのパッケージ名を入力します。最後に、アプリケーションの署名に使用している証明書の SHA-256 を **Key Hashes** フィールドに入力します。SHA-256 の取得方法については、[こちら](#certificate-fingerprints)のセクションを参照してください。&quot;変更を保存&quot; ボタンをクリックします。

#### Androidアプリケーション

1. Androidアプリケーションの `res/strings.xml` ファイルに、新しい String リソースを作成します。名前は `google_server_client_id` とし、値には上記の **Google Cloud Console** の設定セクションの手順 5 で取得した `CLIENT_ID` を設定します。
2. `AndroidManifest.xml` ファイルの `Application` タグ内に、Google Play Services のバージョンのメタデータを追加します。

   ```xml lines
   <meta-data
       android:name="com.google.android.gms.version"
       android:value="@integer/google_play_services_version" />
   ```
3. `AndroidManifest.xml` ファイルに、Android のインターネット権限を追加します。

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. `GoogleAuthProvider` の新しいインスタンスを作成する際は、最初のパラメータとして、前の手順で取得した `google_server_client_id` の値を渡します。

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private GoogleAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new GoogleAuthProvider(getString(R.string.google_server_client_id), client);
     }

     // ...
   }
   ```

Google SDK の設定についてさらにサポートが必要な場合は、Google の [Sign-In for Android Guide](https://developers.google.com/identity/android-credential-manager) をご確認ください。

<div id="usage-without-lock">
  ### Lock を使用しない場合
</div>

このプロバイダーを **Lock** なしで使用する場合は、`onActivityResult()` メソッドをオーバーライドし、その呼び出しをプロバイダーのインスタンスにリダイレクトするようにしてください。最後に、認証プロセスを開始するために start を呼び出します。

```kotlin lines
// 独自のリクエストコードを定義する
private static final int RC_PERMISSIONS = 101;
private static final int RC_AUTHENTICATION = 102;

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    if (provider.authorize(requestCode, resultCode, data)) {
        return;
    }
    super.onActivityResult(requestCode, resultCode, data);
}

private void beginAuthentication(){
  provider.start(this, callback, RC_PERMISSIONS, RC_AUTHENTICATION);
}
```

これで完了です。アプリケーションを実行し、Googleネイティブプロバイダーでログインする準備が整いました！！

### 追加オプション

<div id="using-a-custom-connection-name">
  #### カスタム接続名を使用する
</div>

カスタムのソーシャル接続名を使用して Auth0 で認可を行うには、2 番目のコンストラクターを使って GoogleAuthProvider のインスタンスを作成します。

```kotlin wrap lines
GoogleAuthProvider provider = new GoogleAuthProvider("my-connection", "google-server-client-id", client);
```

<div id="send-additional-authentication-parameters">
  #### 認証用の追加パラメータを送信する
</div>

認証時に追加のパラメータを送信するには、`setParameters` を呼び出します。

```kotlin lines
Map<String, Object> parameters = new HashMap<>();
//エントリを追加
provider.setParameters(parameters);
```

<div id="requesting-a-custom-google-scope">
  #### カスタム Google スコープをリクエストする
</div>

デフォルトでは、スコープ `Scopes.PLUS_LOGIN` がリクエストされます。`setScopes` にスコープの一覧を渡して呼び出すことで、スコープをカスタマイズできます。各 Google API (Auth、Drive、Plus..) では、それぞれ独自の Google スコープ一覧が定義されています。

```kotlin wrap lines
provider.setScopes(Arrays.asList(new Scope(Scopes.PLUS_ME), new Scope(Scopes.PLUS_LOGIN)));
```

#### カスタムの Android 実行時権限をリクエストする

このProviderでは、ユーザーを認証するために Android Manifest で特別な権限を要求する必要はありません。ただし、ユースケース上それらが必要な場合は、AuthProvider に処理を任せることができます。`setRequiredPermissions` メソッドを使用してください。

```kotlin wrap lines
provider.setRequiredPermissions(new String[]{"android.permission.GET_ACCOUNTS"});
```

Lock を使用していない場合は、権限リクエストの結果を自分で処理する必要があります。その場合は、対象のアクティビティで `ActivityCompat.OnRequestPermissionsResultCallback` インターフェースを実装してください。`onRequestPermissionsResult` メソッドが呼び出されたら、`provider.onRequestPermissionsResult` を呼び出して、その結果をプロバイダーに渡します。

<div id="log-out-clear-account">
  #### ログアウト / アカウントをクリア。
</div>

次回、ユーザーに資格情報の入力を求めるようにするには、`clearSession` を呼び出してユーザーをログアウトします。これを行うとプロバイダーの状態は無効になるため、結果を `authorize` しようとする前に、再度 `start` を呼び出す必要があります。`stop` を呼び出しても同じ効果があります。

`provider.clearSession();`

<div id="remember-the-last-login">
  #### 最後にログインしたアカウントを記憶する
</div>

既定では、このProviderは最後にログインしたアカウントを記憶します。この動作を変更するには、次のメソッドを使用します。

`provider.rememberLastLogin(false);`

<div id="certificate-fingerprints">
  #### 証明書のフィンガープリント
</div>

Google Cloud Console で新しい <Tooltip tip="OAuth 2.0: 認可プロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を見る" href="/docs/ja/glossary?term=OAuth">OAuth</Tooltip> Credential を作成する際は、アプリケーションの署名に使用している証明書の SHA-1 を指定する必要があります。<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> でアプリケーションの設定を完了する際には、SHA-256 の値も指定する必要があります。以下は、その値を取得するためのターミナルコマンドの例と、その出力例です。

コマンド:

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

出力例:

`no71633JAC3qgzQYCbskprUr55k=`

サポートが必要な場合は、これらの値を取得する方法について、この [Keystores Guide](/docs/ja/libraries/auth0-android/android-development-keystores-hashes) を参照してください。

<div id="lock-facebook-setup">
  ### Lock-Facebook 設定
</div>

<div id="facebook-developers-console">
  #### Facebook Developers Console
</div>

1. [Facebook Developers Console](https://developers.facebook.com/) にアクセスして、新しいアプリを作成します。「Android」を選択し、有効な名前を付けて、「Create new Facebook App ID」をクリックします。
2. アプリケーションの **Package Name** と、プロバイダーを使用する **Activity class** の名前を入力し、Next ボタンをクリックします。
3. アプリケーションの署名に使用している証明書の **SHA-1** Base64 エンコード済みキー ハッシュを追加し、Next ボタンをクリックします。SHA-1 の取得方法については、[こちら](#certificate-fingerprints)のセクションを参照してください。
4. 最後に、ページ上部までスクロールして、Skip Quickstart ボタンをクリックし、Facebook アプリのページに移動します。
5. ページ上部に `APP ID` と `APP SECRET` の値があります。後で必要になるため、保存しておいてください。
6. 左側にナビゲーションドロワーがあります。設定、次に Basic の順にクリックします。**シングルサインオン** スイッチを ON にして、保存ボタンをクリックします。
7. 設定、次に Advanced の順にクリックします。**Native or desktop app?** スイッチを ON にします。

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Auth0 Dashboard に移動し、[Social Connections](https://manage.auth0.com/#/connections/social) をクリックします。
2. **Facebook** をクリックすると、ダイアログが表示されます。
3. 上記の **Facebook Developers Console** セクションのステップ 5 で取得した `APP ID` の値を &quot;App ID&quot; フィールドに入力します。
4. 上記の **Facebook Developers Console** セクションのステップ 5 で取得した `APP SECRET` の値を &quot;App Secret&quot; フィールドに入力します。
5. Save ボタンをクリックします。
6. Auth0 Dashboard に移動し、[アプリケーション](https://manage.auth0.com/#/applications) をクリックします。まだ作成していない場合は、先に作成してからアプリケーションの設定ページを開いてください。
7. ページの下部にある &quot;Show Advanced Settings&quot; リンクをクリックし、&quot;Device Settings&quot; タブに移動します。
8. Android セクションで、**Package Name** フィールドにアプリケーションのパッケージ名を入力します。続いて、アプリケーションの署名に使用している証明書の SHA-256 を **Key Hashes** フィールドに入力します。SHA-256 の取得方法については、[こちら](#certificate-fingerprints) のセクションを参照してください。最後に、&quot;変更を保存&quot; ボタンをクリックします。

<div id="android-application">
  #### Android アプリケーション
</div>

1. Android アプリケーションの `res/strings.xml` ファイルに、新しい String リソースを作成します。名前は `facebook_app_id` とし、値には上記の **Facebook Developers Console** の設定セクションのステップ 5 で取得した `APP ID` を設定します。
2. `AndroidManifest.xml` ファイルの Application タグ内に、`FacebookActivity` と `facebook_app_id` のメタデータを追加します。

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. `FacebookAuthProvider` の新しいインスタンスを作成します。

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
   ```

Facebook SDK の設定についてさらにサポートが必要な場合は、Facebook の [Getting Started Guide](https://developers.facebook.com/docs/android/getting-started) をご確認ください。

このProviderを**Lock**なしで使用する場合は、アクティビティの`onActivityResult()`メソッドをオーバーライドし、その呼び出しをProviderインスタンスに渡すようにしてください。最後に、認証プロセスを開始するにはstartを呼び出します。

```kotlin lines

private static final int RC_PERMISSIONS = 101;

@Override

        return;
    }
    super.onActivityResult(requestCode, resultCode, data);
}

private void beginAuthentication(){
  provider.start(this, callback, RC_PERMISSIONS, RC_AUTHENTICATION);

```

以上で準備完了です。アプリケーションを実行し、Facebook Native Providerを使用してログインできます!!

### 証明書のフィンガープリント

Facebook Developers Console で新しい OAuth Credential を作成する際は、アプリケーションの署名に使用している証明書の SHA-1 を指定する必要があります。さらに、Auth0 Dashboard でアプリケーションの設定を完了する際には、SHA-256 の値も指定する必要があります。以下に、その値を取得するためのターミナルコマンドの例と出力例を示します。

コマンド:

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

出力例:

SHA1: BB:0D:AC:74:D3:21:E1:43:07:71:9B:62:90:AF:A1:66:6E:44:5D:75
SHA256: 15:B9:F9:33:9F:E4:E3:68:C2:10:49:17:5D:A8:77:12:7C:8E:57:E9:FF:B7:23:EA:CC:DD:56:08:06:C9:5E:33

サポートが必要な場合は、これらの値を取得する方法について [Keystores Guide](/docs/ja/libraries/auth0-android/android-development-keystores-hashes) を参照してください。
