---

Auth0 Angular SDK は、Auth0 を使用して Angular アプリに認証および認可を実装するための JavaScript library です。サービス、認証ガード、HTTP インターセプターを提供し、Angular アプリ内で一般的な認証タスクを実行できるようにします。

Auth0 Angular SDK は、グラントやプロトコルの詳細、token の有効期限と更新、さらに token の保存とキャッシュを処理します。内部では、[Universal Login](/ja/docs/authenticate/login/auth0-universal-login) と [Authorization Code Grant Flow with PKCE](/ja/docs/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) を実装しています。

この library は [GitHub で公開](https://github.com/auth0/auth0-angular) されており、[API の詳細](https://auth0.github.io/auth0-angular/) も確認できます。

<div id="installation">
  ## インストール
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Angular SDK は、Angular SPA バージョン 14 以降に対応しています。
</Callout>

[Angular CLI](https://cli.angular.io/) を使用して、プロジェクトに SDK をインストールします。

`ng add @auth0/auth0-angular`

<div id="getting-started">
  ## はじめに
</div>

まず、`AuthModule` をプロジェクトにインポートし、Auth0 クライアントアプリ用に設定して、SDK の連携を開始します。

```jsx lines
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';

// SDKからモジュールをインポートする
import { AuthModule } from '@auth0/auth0-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,

    // モジュールをアプリケーションに設定付きでインポートする
    AuthModule.forRoot({
      domain: '{yourDomain}',
      clientId: 'YOUR_AUTH0_CLIENT_ID',
      authorizationParams: {
        redirect_uri: window.location.origin,
      },
    }),
  ],

  bootstrap: [AppComponent],
})
export class AppModule {}
```

<div id="login">
  ## ログイン
</div>

SDK は Angular の依存性注入システムを完全にサポートしています。まず、`AuthService` 型をコンポーネントにインポートし、コンストラクターに注入します。

次に、ユーザーのログインを開始するには、サービスの `loginWithRedirect` または `loginWithPopup` メソッドを呼び出します。

```jsx lines
import { Component } from '@angular/core';

// SDKからAuthServiceタイプをインポートする
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-login',
  template: '<button (click)="login">Log in</button>',
  styles: [],
})
export class LoginComponent {
  // コンストラクター経由で認証サービスをコンポーネントに注入する
  constructor(public auth: AuthService) {}

  login(): void {
    // これを呼び出してユーザーをログインページにリダイレクトする
    this.auth.loginWithRedirect();
  }
}
```

<div id="logout">
  ## ログアウト
</div>

ユーザーをログアウトするには、`logout` を使用します。`returnTo` が <Tooltip tip="サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> の **Allowed Logout URLs** に設定されていることを確認してください。

```jsx lines
import { Component } from '@angular/core';

// SDKからAuthServiceタイプをインポートする
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-logout',
  template: '<button (click)="logout">Log out</button>',
  styles: [],
})
export class LogoutComponent {
  // コンストラクター経由で認証サービスをコンポーネントに注入する
  constructor(@Inject(DOCUMENT) private doc: Document, public auth: AuthService) {}

  login(): void {
    // これを呼び出してユーザーをログインページにリダイレクトする
    this.auth.logout({ 
      logoutParams: { 
        returnTo: this.doc.location.origin 
      }
    });
  }
}
```

<div id="authentication-state">
  ## 認証状態
</div>

認証状態は、`isAuthenticated$` observable を通じて公開されています。

```html lines
<div *ngIf="auth.isAuthenticated$ | async">
  <app-login></app-login>
</div>
```

<div id="user-profile">
  ## ユーザープロファイル
</div>

認証済みユーザーのユーザープロファイルは、`user$` observable を通じて利用できます。

```html lines
<ul *ngIf="auth.user$ | async as user">
  <li>{{ user.name }}</li>
  <li>{{ user.email }}</li>
</ul>
```

内部では、この observable は `isAuthenticated$` observable からチェーンされているため、ユーザーがログインしているかどうかを手動で確認する必要はありません。認証済みユーザーが利用可能になり次第、値の発行を開始します。

<div id="protect-a-route">
  ## ルートを保護する
</div>

組み込みの `AuthGuard` を使ってルートコンポーネントを保護します。未認証の状態でこのルートにアクセスすると、ユーザーはログインページにリダイレクトされ、ログイン後にこのページへ戻されます。

認証ガードは、`canActivate` Hook (子ルートを使用している場合は `canActivateChild`) を使って、ルート定義に追加する必要があります。

```jsx lines expandable
import { NgModule } from '@angular/core';
import { Routes, RouterModule } from '@angular/router';
import { HomeComponent } from './unprotected/unprotected.component';
import { ProtectedComponent } from './protected/protected.component';

// 認証ガードをインポートする
import { AuthGuard } from '@auth0/auth0-angular';

const routes: Routes = [
  {
    path: 'protected',
    component: ProtectedComponent,

    // `canActivate` フックに認証ガードを登録してルートを保護する
    canActivate: [AuthGuard],
  },
  {
    path: '',
    component: HomeComponent,
    pathMatch: 'full',
  },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule],
})
export class AppRoutingModule {}
```

認証ガードは、ユーザーが未認証の場合に遅延読み込みモジュールの読み込みを防ぐための`canLoad`フックにも対応しています。

## API を呼び出す

SDK には組み込みの `HttpInterceptor` が含まれており、Angular の `HttpClient` サービスを使用してリクエストを送信する際に `Authorization` ヘッダーを注入することで、<Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可情報。" cta="用語集を見る" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip> を自動的に自動付与します。

想定外の送信先へアクセストークンが漏えいするのを防ぐため、このヘッダーを API のどのルートに自動追加するかを指定するように SDK を設定する必要があります。

<div id="import-the-http-interceptor">
  ### HTTPインターセプターをインポートする
</div>

まず、Auth0 の HTTP インターセプターをアプリに統合するために必要な型をインポートします。

```jsx lines
// インターセプターモジュールと必要なAngularの型をインポートする
import { HttpClientModule, HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthHttpInterceptor } from '@auth0/auth0-angular';

// `providers` 配列でインターセプターをアプリモジュールに登録する
@NgModule({
  declarations: [],
  imports: [
    BrowserModule,
    HttpClientModule,     // HttpClientを使ってAPIを呼び出せるようにするために登録する
    AppRoutingModule,
    AuthModule.forRoot(...),
  ],
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthHttpInterceptor, multi: true },
  ],
  bootstrap: [AppComponent],
})
```

**注:** このインターセプターは他のインターセプターと連結して使用することを前提に設計されており、適切な位置に組み込める保証がないため、自動登録ではなく、アプリのモジュールに手動で登録する必要があります。

<div id="configure-the-http-interceptor">
  ### HTTP インターセプターを設定する
</div>

これは、ニーズに応じてさまざまな方法で設定できます。ルート は string で指定することも、送信される token に影響するオプションを含む object で指定することもでき、使用する HTTP メソッドに応じてマッチさせることもできます。さらに、ルート URI には wildcard を含めて、ルート の group を指定することもできます。

**注:** HTTP 呼び出しが行われても、Configuration で指定された ルート に一致しない場合、インターセプターは回避され、`Authorization` ヘッダー は含まれません。

この例では、どの ルート に `Authorization` ヘッダー を含めるかを SDK に指定する一般的な方法を示します。

```jsx lines expandable
// httpInterceptor設定を含めるよう既存のSDK設定を変更する
AuthModule.forRoot({
  domain: '{yourDomain}',
  clientId: 'YOUR_AUTH0_CLIENT_ID',
  authorizationParams: {
    redirect_uri: window.location.origin,
  },

  // AuthHttpInterceptorの設定
  httpInterceptor: {
    allowedList: [
      // '/api'へのすべての呼び出しにアクセストークンを付与する（完全一致）
      '/api',

      // '/api/'で始まるすべての呼び出しにアクセストークンを付与する
      '/api/*',

      // /api/accountsで始まるすべてのリクエストに一致させ、付与するアクセストークンに必要な
      // audienceとscopeも指定する
      {
        uri: '/api/accounts/*',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'read:accounts',
          }
        },
      },

      // HTTPメソッドによるマッチング
      {
        uri: '/api/orders',
        httpMethod: 'post',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'write:orders',
          }
        },
      },

      // 絶対URIを使用する
      {
        uri: 'https://your-domain.auth0.com/api/v2/users',
        tokenOptions: {
          authorizationParams: {
            audience: 'https://your-domain.com/api/v2/',
            scope: 'read:users',
          }
        },
      },
    ],
  },
});
```

**注:** `tokenOptions` は、基盤となる SPA SDK の `getTokenSilently` メソッドに直接渡されます。使用できるプロパティの詳細については、[ドキュメント](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html)を参照してください。

<div id="call-an-api">
  ### API を呼び出す
</div>

Angular&#39;s `HttpClient` サービスを使用して API を呼び出します。設定に一致するルートでは、アクセストークンが `Authorization` ヘッダーを通じて自動的に含まれます。

```jsx lines
export class MyComponent {
  constructor(private http: HttpClient) {}

  callApi(): void {
    this.http.get('/api').subscribe(result => console.log(result));
  }
}
```

なお、呼び出し側で token やその他の認可情報を渡すために、追加で行うことはありません。必要な処理はすべてインターセプターとモジュールの設定で行われます。
