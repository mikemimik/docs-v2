---

**Lock v2 for iOS** では、新しいアカウントを作成する前に、ユーザーに入力を求める追加フィールドを指定できます。追加フィールドは、基本フィールド (メールアドレス、ユーザー名、パスワード) の後に表示されます。

<div id="adding-custom-fields">
  ## カスタムフィールドの追加
</div>

サインアップ時に既定で必要なのは、ユーザーのメールアドレスとパスワードです。必要に応じて、収集するデータを増やすことができます。ここで追加のサインアップ項目を収集すると、それらは `user_metadata` に保存されます。詳しくは、[メタデータ](/docs/ja/manage-users/user-accounts/metadata) を参照してください。

```swift lines
.withOptions {
  $0.customSignupFields = [
    CustomTextField(name: "first_name", placeholder: "First Name", icon: LazyImage(name: "ic_person", bundle: Lock.bundle)),
    CustomTextField(name: "last_name", placeholder: "Last Name", icon: LazyImage(name: "ic_person", bundle: Lock.bundle))
  ]
}
```

カスタムテキストフィールドで使用するアイコンを指定する必要があります。

これで完了です。アプリケーションの Auth0 Dashboard でユーザーのサインアップを有効にしている場合、基本フィールド (email/username、パスワード) を入力して Submit をクリックすると、残りのフィールドの入力を求められます。ユーザーは、サインアップを完了する前に、すべてのカスタムフィールドを入力する必要があります。

ユーザーのサインアップまたはサインインをリクエストすると、追加フィールドはリクエストボディの `user_metadata` 属性に追加されます。これらの値は、ユーザープロファイルを照会すればいつでも確認でき、Auth0 Dashboard の「Users」セクションからも確認できます。
