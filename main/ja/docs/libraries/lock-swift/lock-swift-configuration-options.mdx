以下に、Lock の動作を設定するためのさまざまなオプションを示します。さらに、[Style Customization Options](/ja/docs/libraries/lock-swift/lock-swift-customization) ページでは、Lock の外観やスタイルを変更するためのオプションも多数用意されています。

<div id="configuring-locks-behavior">
  ## Lock の動作を設定する
</div>

設定オプションは、`withOptions` を使って Lock の初期化時に追加できます。

```swift lines
Lock
  .classic()
  .withOptions {
    $0.closable = true
    $0.usernameStyle = [.Username]
    $0.allow = [.Login, .ResetPassword]
  }
  .present(from: self)
```

<div id="behavior-options">
  ## 動作設定
</div>

<div id="closable">
  ### closable
</div>

ユーザーがLockを閉じられるようにします。デフォルトでは `false` です。

```swift lines
.withOptions {
  $0.closable = true
}
```

<div id="scope">
  ### scope
</div>

認証に使用するスコープです。デフォルトは `openid` です。返されるのは **<Tooltip tip="Access Token: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip>** だけでなく、ユーザー情報を含む <Tooltip tip="ID Token: リソースへのアクセスではなく、クライアント自体のための資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=JSON+Web+Token">JSON Web トークン</Tooltip> (JWT) である **<Tooltip tip="ID Token: リソースへのアクセスではなく、クライアント自体のための資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=ID+Token">ID トークン</Tooltip>** も含まれます。認証スコープの詳細については、[Scopes](/ja/docs/get-started/apis/scopes) のドキュメントを参照してください。

```swift lines
.withOptions {
  $0.scope = "openid name email picture"
}
```

<div id="refresh-tokens">
  #### リフレッシュトークン
</div>

Lock オプションで `offline_access` スコープを指定すると、access&#95;token と id&#95;token に加えて [リフレッシュトークン](/ja/docs/secure/tokens/refresh-tokens) を受け取れるようになります。<Tooltip tip="リフレッシュトークン: ユーザーに再度ログインを求めることなく、新しいアクセストークンを取得するために使用されるトークン。" cta="用語集を見る" href="/ja/docs/glossary?term=Refresh+Tokens">リフレッシュトークン</Tooltip>は保存しておき、古いアクセストークンの有効期限が切れたときに新しいアクセストークンを取得するために使用できます。Auth0 認証でリフレッシュトークンを使用する方法の詳細については、リフレッシュトークンの実装に使用する [Auth0.Swift SDK](/ja/docs/libraries/auth0-swift) のリファレンスドキュメント、または Swift 開発での Auth0 の使用例 (リフレッシュトークンの管理を含む) を包括的に紹介している [Swift Quickstart ガイド](/ja/docs/quickstart/native/ios-swift) を参照してください。

<div id="termsofservice">
  ### termsOfService
</div>

デフォルトでは、Lock は Auth0 の利用規約とプライバシーポリシーを使用しますが、別の URL を指定して、他の利用規約やポリシーにリンクすることもできます。

```swift lines
.withOptions {
  $0.termsOfService = "https://mycompany.com/terms"
  $0.privacyPolicy = "https://mycompany.com/privacy"
}
```

<div id="show-terms-of-service">
  ### 利用規約を表示
</div>

データベース接続では、利用規約ダイアログが表示されます。既定値は `true` です。なお、`mustAcceptTerms` フラグが有効な場合、利用規約は常に表示されます。

```swift lines
.withOptions {
    $0.showTerms = true
}
```

<div id="require-users-to-accept-the-terms-of-service">
  ### ユーザーに利用規約への同意を必須にする
</div>

データベース接続では、利用規約に明示的に同意する必要があります。

```swift lines
.withOptions {
    $0.mustAcceptTerms = true
}
```

<div id="web-authentication-options">
  ## Web Authentication オプション
</div>

<div id="leeway">
  ### leeway
</div>

ID トークンのバリデーションで使用するクロックスキューです。サーバー時刻とクライアント時刻の差を考慮して、ID トークンが有効と見なされる時間の許容範囲を広げます。デフォルトは **60000 ミリ秒** (60 秒) です。

```swift lines
.withOptions {
  $0.leeway = 30000 // 30秒
}
```

<div id="maxage">
  ### maxAge
</div>

ユーザーが最後に認証されてからの経過時間として許容される最大値 (ミリ秒) です。ID トークンのバリデーションに使用されます。設定すると、ID トークンには認証時刻を示す `auth_time` クレームが含まれます。デフォルトは `nil` です。

```swift lines
.withOptions {
  $0.maxAge = 86400000 // 1日
}
```

<div id="database-options">
  ## Database のオプション
</div>

<div id="allow">
  ### allow
</div>

アクセス可能な Database の画面を指定します。既定では、`.Login`、`.Signup`、`.ResetPassword` など、すべての画面が有効です。

```swift lines
.withOptions {
  $0.allow = [.Login, .ResetPassword]
}
```

<div id="initialscreen">
  ### initialScreen
</div>

ユーザーに最初に表示される画面です。既定値は `.Login` で、その他のオプションには `.Signup` と `ResetPassword` があります。

```swift lines
.withOptions {
  $0.initialScreen = .Login
}
```

<div id="usernamestyle">
  ### usernameStyle
</div>

ログイン時に必要な識別子の種類を指定します。デフォルトは `[.Username, .Email]` ですが、`[.Username]` または `[.Email]` も指定できます。ただし、このオプションが有効になるのは、[Auth0 Dashboard](https://manage.auth0.com/#/) で `requires_username` フラグを `true` に設定している場合のみです。

```swift lines
.withOptions {
  $0.usernameStyle = [.Username]
}
```

<div id="custom-signup-fields">
  #### カスタム Signup フィールド
</div>

Signup 時に既定で必要な情報は、ユーザーのメールアドレスとパスワードです。必要に応じて、収集する情報を増やすことができます。ここで追加の Signup フィールドを収集すると、それらは `user_metadata` に保存されます。詳しくは [メタデータ](/ja/docs/manage-users/user-accounts/metadata) をご覧ください。なお、カスタムのテキストフィールドで使用するアイコンは指定する必要があります。

```swift lines
.withOptions {
  $0.customSignupFields = [
    CustomTextField(name: "first_name", placeholder: "First Name", icon: LazyImage(name: "ic_person", bundle: Lock.bundle)),
    CustomTextField(name: "last_name", placeholder: "Last Name", icon: LazyImage(name: "ic_person", bundle: Lock.bundle))
  ]
}
```

以下の例のように、ほかのバンドルのアイコンを指定することもできます。
CustomTextField(name: &quot;slack&#95;handle&quot;, placeholder: &quot;Slack Handle&quot;, icon: LazyImage(name: &quot;ic&#95;slack&quot;, bundle: Bundle(identifier: &quot;CustomBundle&quot;)))

<div id="enterprise-options">
  ## エンタープライズのオプション
</div>

エンタープライズ接続固有の設定オプションもあります：

<div id="enterpriseconnectionusingactiveauth">
  ### enterpriseConnectionUsingActiveAuth
</div>

デフォルトでは、エンタープライズ接続は Web Authentication を使用します。ただし、代わりに資格情報を使った認証を使用し、ユーザー名とパスワードの入力を求める接続を指定することもできます。

```swift lines
.withOptions {
  $0.enterpriseConnectionUsingActiveAuth = ["enterprisedomain.com"]
}
```

<div id="activedirectoryemailasusername">
  ### activeDirectoryEmailAsUsername
</div>

クレデンシャル認証モードでは、ユーザーの識別子としてメールアドレスを必須にしますか？デフォルトは `false` で、その場合は代わりにユーザー名が必要です。

```swift lines
.withOptions {
  $0.activeDirectoryEmailAsUsername = true
}
```

<div id="logging-options">
  ## ログオプション
</div>

Lock では、ログ機能のオン/オフや、そのほかのログ関連設定を簡単に変更できます。

<div id="loglevel">
  ### logLevel
</div>

既定では `.off` です。Syslog のログレベルをサポートしています。

```swift lines
.withOptions {
  $0.logLevel = .all
}
```

<div id="loghttprequest">
  ### logHttpRequest
</div>

Auth0.swift API リクエストをログに記録するかどうかを指定します。デフォルトでは `false` です。

```swift lines
.withOptions {
  $0.logHttpRequest = true
}
```

<div id="loggeroutput">
  ### loggerOutput
</div>

ロガーの出力ハンドラーを指定します。デフォルトでは `print` 文を使用します。

```swift lines
.withOptions {
  $0.loggerOutput = CleanroomLockLogger()
}
```

上記のコードでは、loggerOutput が [CleanroomLogger](https://github.com/emaloney/CleanroomLogger) を使用するように設定されています。通常は、loggerOutput プロトコルを実装することでこれを実現できます。もちろん、お好みのロガーライブラリを使用することもできます。以下は、CleanroomLogger で logger 出力を処理する使用例です。

```swift lines
class CleanroomLockLogger: LoggerOutput {
  func message(_ message: String, level: LoggerLevel, filename: String, line: Int) {
    let channel: LogChannel?
    switch level {
    case .debug:
        channel = Log.debug
    case .error:
        channel = Log.error
    case .info:
        channel = Log.info
    case .verbose:
        channel = Log.verbose
    case .warn:
        channel = Log.warning
    default:
        channel = nil
    }
    channel?.message(message, filePath: filename, fileLine: line)
  }
}
```
