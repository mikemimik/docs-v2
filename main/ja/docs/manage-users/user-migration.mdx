Auth0では、カスタムデータベース接続、Auth0 <Tooltip tip="顧客が管理タスクを実行するための製品。" cta="用語集を表示" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip>、またはUser Import/Export Extensionを使用して、外部アプリケーションからユーザーをインポートできます。

<div id="automatic-migrations">
  ## 自動移行
</div>

Auth0 は、[カスタムデータベース接続](/docs/ja/authenticate/database-connections/custom-db) から Auth0 へのユーザーの自動移行をサポートしています。これは、**trickle migration** または **lazy migration** と呼ばれることもあります。この機能を有効にすると、ユーザーには次のことが適用されます。

* 連携を設定した後、初回ログイン時に Auth0 へ移行されます。
* 移行に伴ってパスワードのリセットを求められることはありません。

<Card title="利用可否は Auth0 のプランによって異なります">
  この機能が利用可能かどうかは、ログイン実装の内容と、Auth0 のプランまたはカスタム契約の両方によって決まります。詳しくは、[Pricing](https://auth0.com/pricing) をご覧ください。
</Card>

Auth0 へのインポート対象として指定されたカスタムデータベース接続経由でユーザーが認証されると、次の処理が行われます。

* Auth0 は、移行済みのユーザーを Auth0 データベースに対して認証します。
* ユーザーがまだ移行されていない場合、Auth0 はカスタム Login スクリプトを実行し、ログインに成功すると、そのユーザーを Auth0 データベースに追加します。
* 以降のログインでは、ユーザーの資格情報は **NOT** カスタムデータベースではなく、Auth0 から取得されます。
* 新規ユーザーは自動的に Auth0 データベースに追加されます。

<div id="bulk-user-imports-with-the-management-api">
  ## Management API を使ったユーザーの一括インポート
</div>

すでにユーザーデータベースがある場合は、[`/post_users_imports`](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports) Management API エンドポイントを使用して、その情報をデータベース接続に取り込むことができます。

<div id="migrate-users-with-the-user-importexport-feature">
  ## User Import/Export 機能を使用したユーザーの移行
</div>

User Import/Export 機能を使用すると、次のことができます。

* 既存のデータベースユーザーを Auth0 に一括でインポートする。
* Auth0 のデータベースユーザーの一部またはすべてを検索してエクスポートする。

一括インポートを使用するには、Auth0 Dashboard の Editor - Users または Admin である必要があります。一括エクスポートを使用するには、Auth0 Dashboard の Viewer - Users、Editor - Users、または Admin である必要があります。

ユーザーデータは、ダッシュボードで利用できる [User Import/Export 機能](/docs/ja/manage-users/user-migration/bulk-user-import-export) を使用してインポートおよびエクスポートできます。
