[`GET /api/v2/users` エンドポイント](https://auth0.com/docs/api/management/v2#!/Users/get_users)を使用すると、ユーザーの一覧を取得できます。このエンドポイントでは、次のことができます。

* さまざまな条件に基づいて検索する
* 返されるフィールドを選択する
* 返された結果を並べ替える

このエンドポイントは**結果整合性**であるため、既存ユーザーの表示名の変更など、バックオフィス処理で使用することを推奨します。

<div id="request-example">
  ## リクエストの例
</div>

ユーザーを検索するには、[`/api/v2/users` エンドポイント](https://auth0.com/docs/api/management/v2#!/Users/get_users) に `GET` リクエストを送信します。このリクエストには、[Management API access token](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens) を含める必要があります。検索クエリを `q` パラメータに渡し、`search_engine` パラメータを `v3` に設定します。

たとえば、メールアドレスが `jane@exampleco.com` と完全に一致するユーザーを検索するには、`q=email:"jane@exampleco.com"` を使用します。

<div id="query-examples">
  ## クエリの例
</div>

以下は、<Tooltip tip="Management API: お客様が管理タスクを実行するための製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> で実行できるクエリの例です。

| ユースケース                                                                                                     | クエリ                                                                        |
| ---------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| 名前に &quot;john&quot; を含むすべてのユーザーを検索する                                                                      | `name:\*john\*`                                                            |
| 名前が完全に &quot;jane&quot; と一致するすべてのユーザーを検索する                                                                 | `name:"jane"`                                                              |
| 名前が &quot;john&quot; で始まるすべてのユーザーを検索する                                                                     | `name:john*`                                                               |
| 名前が &quot;jane&quot; で始まり、&quot;smith&quot; で終わるユーザーを検索する                                                  | `name:jane*smith`                                                          |
| メールアドレスが完全に &quot;john@exampleco.com&quot; と一致するすべてのユーザーを検索する                                              | `email:"john@exampleco.com"`                                               |
| `OR` を使って、メールアドレスが完全に &quot;john@exampleco.com&quot; または &quot;jane@exampleco.com&quot; と一致するすべてのユーザーを検索する | `email:("john@exampleco.com" OR "jane@exampleco.com")`                     |
| メールアドレスが未検証のユーザーを検索する                                                                                      | `email_verified:false OR NOT \_exists_\:email_verified`                    |
| `user_metadata` の `full_name` フィールドの値が &quot;John Smith&quot; であるユーザーを検索する                                 | `user_metadata.full_name:"John Smith"`                                     |
| 特定の接続からのユーザーを検索する                                                                                          | `identities.connection:"google-oauth2"`                                    |
| 一度もログインしたことのないすべてのユーザーを検索する                                                                                | `(NOT \_exists_\:logins_count OR logins_count:0)`                          |
| 2018 年より前にログインしたすべてのユーザーを検索する                                                                              | `last_login:[* TO 2017-12-31]`                                             |
| 最終ログインが 2017 年 12 月であるすべてのユーザーを検索する                                                                        | `last_login:[2017-11 TO 2017-12]`, `last_login:[2017-12-01 TO 2017-12-31]` |
| ログイン回数が &gt;= 100 かつ &lt;= 200 のすべてのユーザーを検索する                                                              | `logins_count:[100 TO 200]`                                                |
| ログイン回数が &gt;= 100 のすべてのユーザーを検索する                                                                           | `logins_count:[100 TO *]`                                                  |
| ログイン回数が &gt; 100 かつ &lt; 200 のすべてのユーザーを検索する                                                                | `logins_count:\{100 TO 200}`                                               |
| メールアドレスのドメインが &quot;exampleco.com&quot; のすべてのユーザーを検索する                                                     | `email.domain:"exampleco.com"`                                             |

<div id="limitations">
  ## 制限事項
</div>

* クエリに一致するユーザーがそれ以上いても、このエンドポイントが返すユーザーは最大 50 件です。
* 50 件を超えるユーザーを返す必要がある場合は、`page` パラメータを使って結果のページを追加で表示します。各ページには 50 人のユーザーが含まれます。たとえば、51〜100 件目の結果を表示するには `&page=2`、101〜150 件目の結果を表示するには `&page=3` を指定できます。以降も同様です。ただし、ページングを使用しても、このエンドポイントが同じ検索条件で合計 1000 人を超えるユーザーを返すことはありません。
* インデックス化、クエリ、返却の対象となるユーザーデータには、ユーザーごとに 1 MB の上限があります。対象は [user search endpoint](https://auth0.com/docs/api/management/v2/users/get-users) です。これが 1MB を超えるカスタムメタデータにどう影響するかについて詳しくは、[Metadata Field Names and Data Types](/ja/docs/manage-users/user-accounts/metadata/metadata-fields-data#size-limits-and-storage) を参照してください。サイズ超過のユーザープロファイルについてすべてのユーザー属性を取得するには、[get user endpoint](https://auth0.com/docs/api/management/v2/users/get-users-by-id) を使用する必要があります。
* デフォルトでは、`GET /api/v2/users` エンドポイントは結果を決定的な順序で返すため、同じクエリでは毎回、論理的に同じ順序の結果が得られます。この動作は `primary_order` クエリパラメータで制御されます。

  * `primary_order=true` (デフォルト) : 同一のクエリでは、結果が常に一貫した順序で返されます。
  * `primary_order=false`: 結果は非決定的な順序で返されるため、複雑なクエリではパフォーマンスが向上する場合があります。
* アプリケーションで検索結果の一貫した順序が不要で、かつクエリが短くページネーションに依存しない場合は、クエリパフォーマンスを向上させるために `primary_order=false` を設定してください。
* すべてのユーザーを完全にエクスポートする必要がある場合は、[export job](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_exports) または [User Import / Export](/ja/docs/customize/extensions/user-import-export-extension) 拡張機能を使用してください。
* `414 Request-URI Too Large` エラーが表示された場合は、クエリ文字列がサポートされている長さを超えていることを意味します。この場合は、検索条件を絞り込んでください。

以下の用途でこのエンドポイントを使用することは**推奨しません**。

* 即時整合性が必要な操作。代わりに、[Get Users by Email endpoint](/ja/docs/manage-users/user-search/retrieve-users-with-get-users-by-email-endpoint) または [Get Users by ID endpoint](/ja/docs/manage-users/user-search/retrieve-users-with-get-users-by-id-endpoint) を使用してください。
* ユーザーのエクスポート。代わりに、[User Export endpoint](/ja/docs/manage-users/user-migration/bulk-user-exports) を使用してください。
* 認証プロセスの一部としてユーザー検索を必要とする操作。代わりに、Get Users by Email endpoint または Get Users by ID endpoint を使用してください。
* Email による [Account Linking](/ja/docs/manage-users/user-accounts/user-account-linking) のためのユーザー検索。代わりに、Get Users by Email endpoint を使用してください。
