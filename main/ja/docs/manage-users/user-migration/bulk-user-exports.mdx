`POST /api/v2/jobs/users-exports` エンドポイントを使用すると、[接続](/docs/ja/authenticate/identity-providers)に関連付けられたすべてのユーザー、またはテナント内のすべてのユーザーをエクスポートするジョブを作成できます。

エクスポート可能なユーザープロファイルのフィールド一覧については、[User Profile Structure](/docs/ja/manage-users/user-accounts/user-profiles/user-profile-structure)をご覧ください。

ジョブを作成する際は、次の情報を指定する必要があります。

* ユーザーのエクスポート元となる[接続の ID](/docs/ja/authenticate/identity-providers/locate-the-connection-id) (任意)
* エクスポートファイルの形式 (CSV または JSON 互換)
* エクスポートするユーザーレコードの最大数 (任意。省略した場合はすべてのレコードがエクスポートされます)
* エクスポートに含めるユーザー関連のフィールド (ユーザー ID や名前など)

また、有効な [Management API Access Token](https://auth0.com/docs/api/management/v2/tokens) も必要です。

<div id="create-a-request-body">
  ## リクエストボディを作成する
</div>

必要に応じて、<Tooltip tip="サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>で `connection_id` と Auth0 テナントのドメイン名を確認します。以下のリクエストボディを含む新しいテキストファイルを作成します。

```json lines
{
   "connection_id":"connection_id",
   "format":"csv",
   "limit":20,
   "fields":[
      {
         "name":"user_id"
      },
      {
         "name":"email"
      },
      {
         "name":"user_metadata.country"
      }
   ]
}
```

`connection_id` をご自身のデータベース接続 ID に置き換えるか、削除すると、テナント内のすべてのユーザーをエクスポートできます。

<div id="include-user-metadata-in-exported-csv">
  ## エクスポートするCSVにユーザーメタデータを含める
</div>

CSV形式でユーザーデータをエクスポートする際にメタデータ情報も含める場合は、エクスポートする各メタデータフィールドを指定します。エクスポートできるフィールドは最大30件です。

<Warning>
  `app_metadata` または `user_metadata` 全体をCSVでエクスポートすることはできません。メタデータオブジェクト内のフィールドを明示的に指定する必要があります。

  `app_metadata` または `user_metadata` を単一のオブジェクトとしてエクスポートするには、JSON互換形式を使用し、リクエストボディの `fields` パラメータに必要なフィールドを含めます。たとえば、次のように指定します。

  `{"name":"app_metadata"}`

  エクスポートできるフィールドは最大30件までのため、ユーザーデータにフィールドが多い場合はJSON形式の使用をおすすめします。
</Warning>

たとえば、メタデータが次のような構造になっている場合:

```json lines
{
  "consent": {
      "given": true,
      "date": "01/23/2019",
      "text_details": "{yourURL}"
  }
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザーエクスポートのCSVファイルでは、CSVインジェクションを防ぐため、OWASP標準に準拠してStringデータ型をエスケープします。

  * 二重引用符 (&quot;) の前には、二重引用符が追加されます。
  * 各文字列の先頭には、一重引用符 (&#39;) が追加されます。
  * 各文字列は二重引用符で囲まれます。

  これは、Auth0が生成するISO 8601形式の日付には適用されません。
</Callout>

<div id="json-compatible-format">
  ### JSON互換形式
</div>

JSON互換形式でデータをエクスポートする場合は、`root` プロパティだけを指定すればよく、各内部プロパティを個別に指定する必要はありません。これらは自動的に含まれます。

Auth0 のエクスポートファイルはサイズが大きいため、[NDJSON](https://github.com/ndjson/ndjson-spec) 形式を使用します。一方、インポート機能では JSON ファイルが必要です。

Auth0 が生成したエクスポートを使ってユーザーをインポートするには、まず任意のライブラリ (たとえば [jq](https://stedolan.github.io/jq/)) を使って、ファイルを NDJSON から JSON に変換する必要があります。

<div id="check-export-status">
  ## エクスポートのステータスを確認する
</div>

ユーザーをエクスポートするジョブを作成したら、[Get a Job endpoint](https://auth0.com/docs/api/management/v2#!/Jobs/get_jobs_by_id) を使用してそのステータスを確認できます。

ジョブの作成時にレスポンスで返されたジョブIDを指定します。以下のサンプルリクエストを使用する場合は、プレースホルダー `{yourJobId}` を実際のIDの値に置き換えてください。

<div id="find-export-data">
  ## エクスポートデータを確認する
</div>

エクスポートファイルには、**location** パラメーターの値として提供される URL からアクセスできます。テナント名は、そのままファイル名にもなります。たとえば、テナント名が `auth0docs` の場合、ファイル名は `auth0docs.csv` または `auth0docs.json` です。URL にアクセスすると、ファイルのダウンロードが自動的に開始されます。

ダウンロードリンクの有効期限は 60 秒です。この時間を過ぎた場合でも、ジョブの有効期限が切れるまでの 24 時間以内であれば、再度アクセスできます。

<div id="job-cleanup">
  ## ジョブのクリーンアップ
</div>

ジョブに関連するデータはすべて24時間後に自動的に削除され、それ以降はアクセスできなくなります。したがって、**お好みの保存方法を使ってジョブの結果を保存しておくことを強く推奨します**。

<div id="filter-example">
  ## フィルターの例
</div>

エクスポートした .csv ファイルから、アプリケーションへのユーザーの最終ログイン日時など、データの特定のサブセットだけを抽出したい場合があります。

1. Management API の [Create export user&#39;s job](https://auth0.com/docs/api/management/v2/jobs/post-users-exports) エンドポイントに `POST` リクエストを送信して、ユーザーを `.csv` ファイルにエクスポートします。

   1. 新しい job を作成します。

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/users-exports' \
      -H 'Content-Type: application/json' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN' \
      -d '{"format":"csv"}'
      ```

   2. ステータスを確認します。

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/JOB_ID' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN'
      ```

2. ローカルマシンのダウンロードフォルダーにあるファイルを開きます。

3. コマンドラインツールを使ってフィルターを追加します。この例では、[Miller](https://miller.readthedocs.io/en/latest/installing-miller/) を使用します。

   ```bash wrap lines
   mlr --csv filter '$last_login >= "2024-01-01T00:00:00Z"' file.csv > filtered.csv
   ```
