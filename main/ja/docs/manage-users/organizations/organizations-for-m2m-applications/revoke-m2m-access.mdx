title: M2Mアクセスを取り消す
---

M2Mアクセスを取り消す、つまりアプリケーションの organization へのアクセスを削除するには、そのクライアントグラントと organization の関連付けを削除する必要があります。この関連付けを削除すると、アプリケーションは、そのグラントで定義されたAPIに対して新しいトークンをリクエストする際に、その organization を使用できなくなります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  M2Mアクセスを取り消しても、いずれかの organization へのアクセスが許可されているアプリケーションには影響しません。詳しくは、[organization の動作を定義する](/docs/ja/manage-users/organizations/organizations-for-m2m-applications/configure-your-application-for-m2m-access#define-organization-behavior)をご覧ください。
</Callout>

<Tabs>
  <Tab title="Auth0 Dashboard">
    [Auth0 Dashboard](https://manage.auth0.com/) でアプリケーションのクライアントグラントと organization の関連付けを削除するには、次の手順に従います。

    1. **Organizations** に移動し、関連付けを削除する organization を選択します。
    2. **Machine-to-Machine Access** タブを選択します。
    3. アプリケーションを選択します。
    4. organization との関連付けを解除するAPIの横にあるチェックボックスをオフにします。
    5. **Save** をクリックします。

    <Frame>![](/docs/images/cdy7uua7fh8z/5MFv5NAoBDswloVweCVU1D/9057b94d5c5fd35e25afc7170ac645f3/Acme_Bot_-_Travel_Api_-_English.png)</Frame>
  </Tab>

  <Tab title="Management API">
    アプリケーションのクライアントグラントと organization の関連付けは、[Organizations からクライアントグラントの関連付けを解除する](https://auth0.com/docs/api/management/v2/organizations/delete-client-grants-by-grant-id)エンドポイントを使用して削除できます。

    ```bash lines
    curl -X DELETE --location "https://{yourDomain}/api/v2/organizations/{ORG_id}/client-grants/{GRANT_ID}" \
      --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' ;
    ```
  </Tab>
</Tabs>
