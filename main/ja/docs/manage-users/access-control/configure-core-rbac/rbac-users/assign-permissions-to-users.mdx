ユーザーには、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用して[権限](/ja/docs/manage-users/access-control/rbac)を割り当てることができます。割り当てた権限は、API Authorization Core の機能セットで利用できます。

<Warning>
  ユーザーに権限を直接追加すると、ロールベースのアクセス制御 (RBAC) の利点が損なわれるため、通常は推奨されません。
</Warning>

<div id="prerequisites">
  ## 前提条件
</div>

* ロールベースのアクセス制御 (RBAC) を正しく機能させるには、Dashboard または Management API を使用して、API で RBAC を有効にする必要があります。Authorization Core の機能は Authorization Extension とは異なります。比較については、[Authorization Core と Authorization Extension の比較](/ja/docs/manage-users/access-control/authorization-core-vs-authorization-extension)を参照してください。
* Dashboard で [API を設定します](/ja/docs/get-started/auth0-overview/set-up-apis)。
* 権限は事前定義された値から選択します。ロールの一覧が空の場合は、API に [権限を追加](/ja/docs/get-started/apis/add-api-permissions) していることを確認してください。

<div id="dashboard">
  ## Dashboard
</div>

ユーザーに権限を割り当てる方法は 2 つあります。**Users** リストからユーザーを選択して権限を割り当てるか、個別のユーザーの **User Details** (ユーザープロファイル) ページに移動し、**Permissions** タブで割り当てる権限を選択します。

<div id="assign-permission-in-user-list">
  ### ユーザー一覧から権限を割り当てる
</div>

1. [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) に移動します。
2. 権限を割り当てるユーザーの横にある `...` をクリックし、**Assign Permissions** を選択します。
3. 権限を割り当てる API を選択し、次にそのユーザーに割り当てる権限を選択して、**Add Permissions** をクリックします。

<div id="assign-permissions-in-user-profile">
  ### ユーザープロファイルで権限を割り当てる
</div>

ユーザー個別のプロフィールページから権限を割り当てることもできます。

1. [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) に移動し、表示したいユーザーの名前をクリックします。
2. **Permissions** タブをクリックし、**Assign Permissions** をクリックします。
3. 権限を割り当てる API を選択し、次にそのユーザーに割り当てる権限を選択して、**Add Permissions** をクリックします。

<div id="management-api">
  ## Management API
</div>

[Assign User Permissions endpoint](https://auth0.com/docs/api/management/v2#!/Users/post_permissions) に `POST` リクエストを送信します。`USER_ID`、`MGMT_API_ACCESS_TOKEN`、`API_IDENTIFIER`、`PERMISSION_NAME` の各プレースホルダー値は、それぞれ実際のユーザー ID、Management API の <Tooltip tip="アクセストークン: API へのアクセスに使用される、オペーク文字列または JWT の形式の認可資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip>、API 識別子、権限名に必ず置き換えてください。

| Value                   | Description                                                                                               |
| ----------------------- | --------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | 権限を割り当てる対象ユーザーの ID。                                                                                       |
| `MGMT_API_ACCESS_TOKEN` | scope `update:users` を持つ [Management API のアクセストークン](https://auth0.com/docs/api/management/v2/tokens)。 |
| `API_IDENTIFIER`        | 指定したユーザーに割り当てる権限 に関連付けられた API の識別子。                                                              |
| `PERMISSION_NAME`       | 指定したユーザーに割り当てる権限 の名前。                                                                            |
