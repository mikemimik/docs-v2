ユーザーには、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要な製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Management API: テナント、アプリケーション、ユーザー、その他のリソースをプログラムで管理するための Auth0 の管理 API。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用して [ロール](/ja/docs/manage-users/access-control/rbac) を割り当てることができます。割り当てたロールは、API Authorization Core の機能セットで利用できます。

<div id="prerequisites">
  ## 前提条件
</div>

* ロールベースのアクセス制御 (RBAC) を正しく機能させるには、Dashboard または Management API を使用して、API で RBAC を有効にする必要があります。Authorization Core の機能は Authorization Extension とは異なります。比較については、[Authorization Core と Authorization Extension の比較](/ja/docs/manage-users/access-control/authorization-core-vs-authorization-extension)をご覧ください。
* ロールはあらかじめ定義された値から選択します。ロールの一覧が空の場合は、まず[ロールを作成](/ja/docs/manage-users/access-control/configure-core-rbac/roles/create-roles)していることを確認してください。

<div id="dashboard">
  ## Dashboard
</div>

ユーザーにロールを割り当てる方法は 2 つあります。**Users** リストからユーザーを選択してロールを割り当てるか、各ユーザーの **User Details** (ユーザープロフィール) ページに移動し、**Roles** タブで割り当てるロールを選択します。

ロールが 50 個を超える場合、**Roles** タブに表示されるのは先頭の 50 個のみです。それ以外のロールは Management API で割り当てることができます。

<div id="assign-roles-in-user-list">
  ### ユーザー一覧でロールを割り当てる
</div>

1. [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) に移動します。
2. ロールを割り当てるユーザーの横にある `...` をクリックし、**Assign Roles** を選択します。
3. 割り当てるロールを選択し、**Assign** をクリックします。

<div id="assign-roles-in-user-profile">
  ### ユーザープロファイルでロールを割り当てる
</div>

各ユーザーのプロフィールページからロールを割り当てることもできます。

1. [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) に移動し、ユーザー名をクリックします。
2. **Roles** ビューを開き、**Assign Role** をクリックします。
3. 割り当てるロールを選択し、**Assign.** をクリックします。

<div id="management-api">
  ## Management API
</div>

[Assign User Roles エンドポイント](https://auth0.com/docs/api/management/v2#!/Users/post_user_roles) に `POST` リクエストを送信します。`USER_ID`、`MGMT_API_ACCESS_TOKEN`、`ROLE_ID` の各プレースホルダー値は、それぞれご使用のユーザー ID、Management API の <Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip>、およびロール ID に必ず置き換えてください。

  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/api/v2/users/USER_ID/roles' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "roles": [ "ROLE_ID", "ROLE_ID" ] }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users/USER_ID/roles");

  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("cache-control", "no-cache");
  request.AddParameter("application/json", "{ "roles": [ "ROLE_ID", "ROLE_ID" ] }", ParameterType.RequestBody);
  ```

  ```go Go
  	body, _ := ioutil.ReadAll(res.Body)
  ```

| 値                       | 説明                                                                                                                    |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | 更新するユーザーの ID。                                                                                                         |
| `MGMT_API_ACCESS_TOKEN` | スコープ `read:roles` および `update:users` を持つ [Management API のアクセストークン](https://auth0.com/docs/api/management/v2/tokens)。 |
| `ROLE_ID`               | 指定したユーザーに追加するロールの ID。                                                                                                 |
