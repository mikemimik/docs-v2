title: ロールベースのアクセス制御向けの Core Authorization 機能を設定する
---

Auth0 のコア Authorization 機能を使うと、API にロールベースのアクセス制御 (RBAC) を適用できます。

RBAC を正しく機能させるには、Auth0 Dashboard または <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用して、API でこれを有効にする必要があります。詳しくは、[API のロールベースのアクセス制御を有効にする](/ja/docs/get-started/apis/enable-role-based-access-control-for-apis) を参照してください。

Authorization Core の機能は Authorization Extension とは異なります。両者の比較については、[Authorization Core と Authorization Extension の比較](/ja/docs/manage-users/access-control/authorization-core-vs-authorization-extension) を参照してください。

コア機能を最も効率よく利用するには、次の手順を行ってください。

1. [Auth0 に API を登録する](/ja/docs/get-started/auth0-overview/set-up-apis)
2. [API の権限を定義する](/ja/docs/get-started/apis/add-api-permissions)
3. [ロールを作成する](/ja/docs/manage-users/access-control/configure-core-rbac/roles/create-roles)
4. [ユーザーにロールを割り当てる](/ja/docs/manage-users/access-control/configure-core-rbac/rbac-users/assign-roles-to-users)
5. 必要に応じて、[ユーザーに権限を割り当てる](/ja/docs/manage-users/access-control/configure-core-rbac/rbac-users/assign-permissions-to-users)。
