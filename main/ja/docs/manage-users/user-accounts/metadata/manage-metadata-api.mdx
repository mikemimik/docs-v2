<div id="create-user-metadata">
  ### ユーザーのメタデータを作成する
</div>

以下のプロファイル情報を持つユーザーを作成するには、

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing"
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

ユーザーを作成し、プロパティ値を設定するには、<Tooltip tip="Management API: 顧客が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> [`/post_users`](https://auth0.com/docs/api/management/v2#!/Users/post_users) エンドポイントに対して、次の `POST` 呼び出しを行います。

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/api/v2/users' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'content-type: application/json' \
    --data '{"email": "jane.doe@example.com", "user_metadata": {"hobby": "surfing"}, "app_metadata": {"plan": "full"}}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{"email": "jane.doe@example.com", "user_metadata": {"hobby": "surfing"}, "app_metadata": {"plan": "full"}}", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/users"

  	payload := strings.NewReader("{"email": "jane.doe@example.com", "user_metadata": {"hobby": "surfing"}, "app_metadata": {"plan": "full"}}")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("content-type", "application/json")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```


<div id="update-user-metadata">
  ### ユーザーのメタデータを更新する
</div>

Management API の [`/patch_users_by_id`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) エンドポイントに `PATCH` リクエストを送信することで、ユーザーのメタデータを更新できます。

次のメタデータでユーザーを作成したとします。

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing"
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

`user_metadata` を更新し、ユーザーの住所を第2階層のプロパティとして追加するには:

```json lines
{
    "addresses": {
        "home": "123 Main Street, Anytown, ST 12345"
    }
}
```

次の`PATCH`リクエストを送信します：

ユーザーのプロファイルは次のようになります。

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing",
        "addresses": {
            "home": "123 Main Street, Anytown, ST 12345"
        }
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

<Warning>
  プロパティの値を `null` に設定した `PATCH` リクエスト (例: `{user_metadata: {color: null}}`) を送信すると、Auth0 はそのプロパティと値をデータベースから**削除**します。さらに、空のオブジェクトでメタデータ自体にパッチを適用すると、メタデータ全体が完全に削除されます。
</Warning>

<div id="merge-user-metadata">
  ### ユーザーのメタデータをマージする
</div>

オブジェクトにマージされるのは、ルートレベルのプロパティのみです。下位レベルのプロパティはすべて置き換えられます。

たとえば、ユーザーの勤務先住所を追加の内部プロパティとして加えるには、`addresses` プロパティの内容をすべて含める必要があります。`addresses` オブジェクトはルートレベルのプロパティであるため、ユーザーを表す最終的な JSON オブジェクトにはマージされますが、その下位プロパティはマージされません。

```json lines
{
  "user_metadata": {
    "addresses": {
      "home": "123 Main Street, Anytown, ST 12345",
      "work": "100 Industrial Way, Anytown, ST 12345"
    }
  }
}
```

したがって、対応する API への `PATCH` リクエストは次のようになります:

```json lines
{
  "user_metadata": {}
```

<div id="app-metadata">
  ## アプリのメタデータ
</div>

空のオブジェクトでメタデータをパッチすると、メタデータは完全に削除されます。たとえば、このボディを送信すると、`app_metadata` 内のすべてが削除されます:

```json lines
{
  "app_metadata": {}
}
```

<div id="create-applications-with-client-metadata">
  ### クライアントメタデータを含めてアプリケーションを作成する
</div>

`POST /api/v2/` applications エンドポイントを使用して新しいアプリケーションを作成する際に、`clientMetadata` オブジェクトを含めることができます。

<div id="read-client-metadata">
  ### クライアントメタデータを取得する
</div>

クライアントメタデータは、`GET /api/v2/clients` および `GET /api/v2/client/{id}` エンドポイントのレスポンスに含まれます。

<div id="update-client-metadata">
  ### クライアントメタデータを更新する
</div>

クライアントメタデータは、[`PATCH /api/v2/clients/{id}`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) エンドポイントを使用して更新できます。`clientMetadata property` を持つアプリケーションオブジェクトを指定し、そのプロパティの値には、変更したいメタデータを含むオブジェクトを設定します。

**変更前のアプリケーション:**

```json lines
{
  ...
  "name": "myclient",
  "client_metadata": {
    "mycolor": "red",
    "myflavor": "grape"
  }
  ...
}
```

リクエスト: `PATCH /api/v2/client/myclientid123`、ボディ:

```json lines
{ "client_metadata": { "mycolor": "blue" } }
```

**変更後のアプリケーション:**

```json lines
{
  "name": "myclient",
  "client_metadata": {
    "mycolor": "blue",
    "myflavor": "grape"
  }
  ...
}
```

<div id="delete-client-metadata-properties-and-values">
  ### クライアントメタデータのプロパティと値を削除する
</div>

クライアントメタデータのキーは、上記の「Update app&#95;metadata」で説明されているように、`PATCH` リクエストを送信し、キーの値に `null` を指定することで削除できます。この挙動は、`PATCH` [/api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) エンドポイントにおける `user_metadata` および `app_metadata` プロパティの挙動と一致します。
