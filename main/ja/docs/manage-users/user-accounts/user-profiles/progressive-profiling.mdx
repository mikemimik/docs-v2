description: ユーザーがウェブサイトやアプリケーションを利用する中で、段階的プロファイリングによってどのように継続的に情報を収集できるのかを理解します。
title: 段階的プロファイリングの仕組みを理解する
---

段階的プロファイリングとは、ユーザーがウェブサイトやアプリケーションを利用する中で、ユーザーに関する追加情報を少しずつ収集していくプロセスです。詳しいユーザー情報は、ユーザーの行動に応じて必要なタイミングでのみ収集できます。

たとえば、サインアップ時にユーザーの名前、メールアドレス、パスワードを収集し、次回ログイン時に会社名や役職を尋ねることができます。[Auth0 フォーム](/ja/docs/customize/forms/configure-progressive-profile-form)を使用して、ユーザー情報を収集するフローを設計することもできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザーに追加情報を尋ねる際は、すでに把握している可能性のある情報を改めて尋ねないようにしてください。たとえば、ユーザーが OpenID Connect プロトコルを使用するソーシャル接続 (Google や Facebook など) を通じてサインアップした場合、アイデンティティプロバイダー (IdP) から、想像以上に多くのユーザー情報が返されることがあります。詳しくは、[OpenID Connect スコープ](/ja/docs/get-started/apis/scopes/openid-connect-scopes#standard-claims)をご覧ください。
</Callout>

段階的プロファイリングを実装すると、次のような形でユーザー体験を向上できます。

* 登録フォームを短くできる。
* 同じ質問を繰り返さずに済む。
* より関連性の高い情報を収集できる。
* コンバージョン率を高められる。

<div id="how-it-works">
  ## 仕組み
</div>

ユーザーが Auth0 経由で認証されるたびに、Auth0 はそのユーザーのユーザープロファイルを更新します。ユーザープロファイルの更新に使用されるデータは、さまざまなソースから取得できます。

* <Tooltip tip="IDプロバイダー (IdP): デジタルアイデンティティを保存・管理するサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=IdP">IdP</Tooltip> から提供される属性。
* [Auth0 Actions](/ja/docs/customize/actions) で動的に作成される属性。
* [FullContact](https://www.fullcontact.com/) や [Clearbit](https://clearbit.com/) などの API の呼び出しで返される属性。
* アプリケーションで管理される属性。

<div id="use-action-triggers">
  ## Action トリガーを使う
</div>

`post-login` Action トリガーを使用すると、ログインフロー中にユーザーの `user_metadata` フィールドと `app_metadata` フィールドを変更できます。これは、ユーザーのプロファイルにアプリケーション固有のデータを保存したり、特定の操作がそのユーザーに対して実行されたかどうかを記録したり、負荷の高い処理の結果をユーザープロファイルにキャッシュして、今後のログインで再利用できるようにしたりする場合に便利です。詳しくは、[post-login Action トリガーでユーザーメタデータを管理する](/ja/docs/manage-users/user-accounts/metadata/manage-user-metadata) をご覧ください。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

Auth0 の <Tooltip tip="Management API: 顧客が管理タスクを実行するための製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> の [Update a User](https://auth0.com/docs/api/management/v2/#!/Users/patch_users_by_id) エンドポイントを使用すると、認証済みユーザーのプロファイルデータ (`app_metadata` フィールドと `user_metadata` フィールドを含む) を更新できます。詳しくは、[Manage Metadata Using the Management API](/ja/docs/manage-users/user-accounts/metadata/manage-metadata-api) をご覧ください。

<div id="use-the-lock-library">
  ## Lock ライブラリを使用する
</div>

`additionalSignUpFields` オプションを使用すると、ユーザー向けのサインアップフォームにカスタムフィールドを追加できます。ユーザーがカスタムフィールドにデータを入力すると、Auth0 はその値をユーザーのプロファイルの `user_metadata` フィールドに保存します。詳しくは、[Lock Configuration Options](/ja/docs/libraries/lock/lock-configuration#additionalsignupfields-array-) を参照してください。
