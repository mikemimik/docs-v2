アップストリームの<Tooltip tip="アイデンティティプロバイダー（IdP）: デジタルIDを保存および管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=identity+provider">アイデンティティプロバイダー</Tooltip>の接続設定を更新すると、<Tooltip tip="アイデンティティプロバイダー（IdP）: デジタルIDを保存および管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>または<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip>を使用して、ユーザープロファイルのルート属性の更新をいつ許可するかを制御できます。

デフォルトでは、Auth0 以外のアイデンティティプロバイダー (Google、Facebook、X など) から提供されるユーザープロファイル属性は、ユーザーがログインするたびにアイデンティティプロバイダーから更新されるため、直接編集できません。

正規化されたユーザープロファイルで name、nickname、given&#95;name、family&#95;name、または picture のルート属性を編集できるようにするには、ユーザー属性がアイデンティティプロバイダーから更新されるのはユーザープロファイルの作成時のみになるように、Auth0 との接続の同期を設定する必要があります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **ログイン時にユーザーを同期** はパスワードレス接続には適用されず、Tenant UI または Management API から有効にすることもできません。
</Callout>

Management API を使用すると、[ルート属性を個別に編集](/ja/docs/manage-users/user-accounts/user-profiles/root-attributes/update-root-attributes-for-users)したり、[一括インポート](/ja/docs/manage-users/user-accounts/user-profiles/root-attributes/set-root-attributes-during-user-import)で設定したりできます。

<div id="use-the-dashboard">
  ## Auth0 Dashboard を使用する
</div>

1. [Auth0 Dashboard &gt; Authentication](https://manage.auth0.com/#/connections) に移動し、接続の種類 (Database、Social、Enterprise、またはパスワードレス) を選択します。
2. 接続名を選択して、その設定を表示します。
3. **Advanced** セクションで、**各ログイン時にユーザープロファイル属性を同期する** スイッチを希望の設定に切り替え、**Save** を選択します。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

この手順を完了する前に、現在の値を上書きしないよう、まず[接続の `options` オブジェクトの既存の値を取得](/ja/docs/authenticate/identity-providers/retrieve-connection-options)してください。これを行わないと、元のオブジェクトに含まれていないパラメーターは、更新後に失われます。

[Update a Connection エンドポイント](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id)に `PATCH` リクエストを送信します。現在の値を上書きしないよう、リクエストには元の options の値を必ず含めてください。また、`CONNECTION_ID`、`MGMT_API_ACCESS_TOKEN`、`ATTRIBUTE_UPDATE_VALUE` のプレースホルダーは、それぞれ使用している接続 ID、Management API の <Tooltip tip="Access Token: API へのアクセスに使用される、opaque な文字列または JWT 形式の Authorization credential。" cta="用語集を見る" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip>、および属性の更新値に置き換えてください。

| 値                        | 説明                                                                                                            |
| ------------------------ | ------------------------------------------------------------------------------------------------------------- |
| `CONNECTION_ID`          | ルート属性の更新を許可する接続の ID。                                                                                          |
| `MGMT_API_ACCESS_TOKEN`  | スコープ `update:connections` を持つ [Management API のアクセストークン](https://auth0.com/docs/api/management/v2/tokens)。    |
| `ATTRIBUTE_UPDATE_VALUE` | ユーザープロファイルのルート属性の更新をいつ許可するかを示します。有効な値は `on_first_login` と `on_each_login` です。新しい接続のデフォルトは `on_each_login` です。 |
