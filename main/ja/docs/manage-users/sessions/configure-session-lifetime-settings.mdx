セッションの**絶対有効期限**は、セッションの作成時に設定されます。絶対有効期限は、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> でセッション設定を構成して調整できます。

<div id="dashboard">
  ## Auth0 Dashboard
</div>

1. [Auth0 Dashboard &gt; Tenant Settings](https://manage.auth0.com/#/tenant) に移動し、**Advanced** ビューを選択します。
2. **Session Expiration** セクションまでスクロールし、**アイドルセッションの有効期間** と **Maximum Session Lifetime** を見つけて必要な設定を入力し、**Save** を選択します。

   <Frame>![Dashboard Tenant Settings Advanced Login Session Management](/docs/images/cdy7uua7fh8z/q2fGPzoUqCcXj7OxcHhjy/de15afcd26e8ea80937733034ffbd387/Screenshot_2024-11-06_at_11.02.16.png)</Frame>

| 設定                       | 説明                                                                                                                                                          |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| アイドルセッションの有効期間    | ユーザーが認可サーバーを操作しない状態が続いた場合に、セッションの有効期限が切れるまでの時間 (分単位) 。Essential または Professional では 4,320 分 (3 日) 、Enterprise プランでは 144,000 分 (100 日) を超えると、システムの上限が適用されます。 |
| Maximum Session Lifetime | ユーザーのアクティビティに関係なく、再度ログインが必要になるまでの時間 (分単位) 。Essential または Professional では 43,200 分 (30 日) 、Enterprise プランでは 525,600 分 (365 日) を超えると、システムの上限が適用されます。          |

<div id="management-api">
  ## Management API
</div>

[Tenant Settings エンドポイント](https://auth0.com/docs/api/management/v2#!/tenants/patch_settings) に `PATCH` リクエストを送信します。`MGMT_API_ACCESS_TOKEN`、`SESSION_LIFETIME`、`IDLE_SESSION_LIFETIME` のプレースホルダー値は、それぞれ Management API の <Tooltip tip="アクセストークン: API へのアクセスに使用される、opaque な文字列または JWT 形式の資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip>、セッションの有効期間の値、アイドルセッションの有効期間の値に必ず置き換えてください。

  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/tenants/settings' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "session_lifetime": SESSION_LIFETIME_VALUE, "idle_session_lifetime": IDLE_SESSION_LIFETIME_VALUE }'
  ```

| Value                         | Description                                                                                                                                                  |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MGMT_API_ACCESS_TOKEN`       | scope `update:tenant_settings` を持つ [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens)。                                   |
| `IDLE_SESSION_LIFETIME_VALUE` | ユーザーが認可サーバーとやり取りしない状態が続いた場合に、セッションが期限切れになるまでの時間 (時間単位) 。Essential または Professional では 72 時間 (3 日) 、Enterprise plan では 2,400 時間 (100 日) を超えると、システムの制限が優先されます。 |
| `SESSION_LIFETIME_VALUE`      | アクティビティの有無にかかわらず、ユーザーが再度ログインを求められるまでの時間 (時間単位) 。Essential または Professional では 720 時間 (30 日) 、Enterprise plan では 8,760 時間 (365 日) を超えると、システムの制限が優先されます。       |
