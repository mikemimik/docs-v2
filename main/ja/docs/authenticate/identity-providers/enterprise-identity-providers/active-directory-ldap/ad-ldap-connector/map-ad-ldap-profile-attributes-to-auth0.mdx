description: Connector Admin Console の Profile Mapper を使用して、AD/LDAP のプロファイル属性を Auth0 ユーザープロファイル属性にマッピングする方法を説明します。
title: AD/LDAP プロファイル属性を Auth0 ユーザープロファイルにマッピングする
---

AD/LDAP の属性から Auth0 ユーザープロファイルの属性へのマッピングは変更できます。

1. ブラウザーで `http://localhost:8357` にアクセスして、AD/LDAP Connector サーバー上の **Connector Admin Console** を起動します。
2. **Profile Mapper** をクリックして、ソース ディレクトリー サービス (`raw_data` 変数で表されます) の属性を、Auth0 ユーザープロファイルに設定するために返される変数にマッピングするコードエディター画面を開きます。
   関数の前半では、`profile` という変数を作成し、Auth0 User Profile の中核部分に対するマッピングを定義しています。追加の属性は、その下に次の形式で設定できます。
   `profile['department'] = raw_data['companydept'];`
   この例では、`department` は Auth0 ユーザープロファイル内の属性名で、`companydept` はソース ディレクトリー サービス (AD など) 内の属性名です。
3. LDAP 設定の変更が完了したら、**Save** をクリックします。
4. Connector によるテストの実行後、すべてのテストがグリーンになっていることを確認します。

| テスト   | 説明                                                              | トラブルシューティング                                      |
| ----- | --------------------------------------------------------------- | ------------------------------------------------ |
| テスト 1 | 指定した LDAP サーバーおよびポートへの TCP 接続の確立を試みます。                          | 接続を妨げている可能性がある基本的なネットワーク接続とファイアウォールの設定を確認してください。 |
| テスト 2 | 指定した LDAP サーバーおよびポートに対し、指定したユーザー名とパスワードを使用して LDAP バインドの実行を試みます。 | LDAP 接続文字列、検索パス、ユーザー名、パスワードを確認してください。            |
| テスト 3 | 指定したユーザー名の権限を確認するために、ディレクトリーに対して LDAP 検索の実行を試みます。               | 対象ディレクトリー内でそのユーザー名に付与されている権限を確認してください。           |
| テスト 4 | Auth0 サーバーへの接続の確立を試みます。                                         | 接続を妨げている可能性があるネットワーク接続とファイアウォールの設定を確認してください。     |
