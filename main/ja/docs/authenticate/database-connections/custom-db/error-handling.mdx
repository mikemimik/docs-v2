カスタムデータベースでエラーが発生することがあります。また、トラブルシューティングのために、カスタムデータベース接続のアクションスクリプトからエラーを返すこともできます。

<div id="custom-database-error-types">
  ## カスタムデータベースのエラーの種類
</div>

データベース接続では、次のエラーが返されることがあります。

| Error                                                                               | Login Script      | Description                                                                                                                                                                                                                                                                                                       |
| ----------------------------------------------------------------------------------- | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `new WrongUsernameOrPasswordError(`<br />`  <email or user_id>, <message>`<br />`)` | Login             | ユーザーの認証情報が無効な場合に発生します。引数を渡さない場合、または最初の引数が falsey の場合、エラーは `user_id` が空文字列の `fu` イベント (無効なユーザー名またはメールアドレス) としてログに記録されます。最初の引数が truthy の場合、エラーは `user_id` の値が `auth0\|<first_argument>` の `fp` イベント (ユーザーは存在するがパスワードが無効) としてログに記録されます。詳しくは、[ログイベントタイプコード](/ja/docs/deploy-monitor/logs/log-event-type-codes)を参照してください。 |
| `new ValidationError(`<br />`  <user_exists>, <message>`<br />`)`                   | Create            | データベース内にユーザーがすでに存在する場合に発生します。                                                                                                                                                                                                                                                                                     |
| `callback(<null>,<false>)`                                                          | Change Password   | ユーザーのパスワードが更新されなかった場合に発生します。                                                                                                                                                                                                                                                                                      |
| `callback(<null>)`                                                                  | Get User          | ユーザーが見つからない場合に発生します。                                                                                                                                                                                                                                                                                              |
| `new UnauthorizedError(<message>)`                                                  | All Login Scripts | データベースへの接続中に問題が発生した場合に発生します。カスタムデータベースからフロントエンドにエラーを返す際の推奨される方法です。                                                                                                                                                                                                                                                |
| `new Error(<message>)`                                                              | All Login Scripts | データベースへの接続中に問題が発生した場合に発生します。                                                                                                                                                                                                                                                                                      |
| `error code: auth0_idp_error;` `user already exists`                                | Create            | レガシー ID ストア、または Auth0 にすでに存在するユーザーを作成しようとした場合に、レガシー ID ストアで発生します。                                                                                                                                                                                                                                                 |
| `error code: auth0_idp_error;` `user already exists`                                | Delete            | カスタムデータベーススクリプトでユーザーを削除しようとした場合に、レガシー ID ストアで発生します。このエラーは、&quot;Error! There was a problem deleting `user@example.com`&quot; または &quot;Sandbox Error: Please implement the Delete script for this database connection at `https://manage.auth0.com/#/connections/database`&quot; と表示されることもあります。                   |

<div id="return-errors">
  ## エラーを返す
</div>

エラーを返すには、`callback()` 関数を呼び出し、第1引数に `error` を渡します。

```js lines
callback(error);
```

たとえば：

```js wrap lines
callback(new ValidationError('email-too-long', 'Email is too long.'));
```

<div id="return-errors-when-using-lock">
  ### Lock 使用時にエラーを返す
</div>

Lock を使用している場合は、辞書に追加することで[エラーメッセージをカスタマイズ](/ja/docs/customize/login-pages/classic-login/customize-lock-error-messages)できます。

<div id="troubleshoot-errors">
  ## エラーのトラブルシューティング
</div>

**Try** ボタンを使ってスクリプトをテストします。

期待した結果が得られない場合やエラーが発生した場合は、[Actions Real-time Logs](/ja/docs/customize/actions/actions-real-time-logs) を確認し、スクリプト内で `console.log()` 文を使用して、再度接続を試してください。`console.log()` の出力は Actions Real-time Logs ウィンドウに表示されます。

アクションスクリプトを使用してユーザーのインポート、作成、削除を行う際に、エラーが発生することがあります。エラーの原因がアクションスクリプトの使用にあることを確認するには、`console.logs` を使用してください。詳しくは、[Custom Database Action Script Templates](/ja/docs/authenticate/database-connections/custom-db/templates) を参照してください。
