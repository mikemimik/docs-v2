<Warning>
  テナントで Flexible Identifiers を有効化すると、本番環境に互換性のない変更が生じる可能性があります。発生しうる問題や制限事項の詳細については、[Flexible Identifiers and Attributes](/ja/docs/authenticate/database-connections/flexible-identifiers-and-attributes) を参照してください。
</Warning>

データベース接続の属性 (email、電話番号、またはユーザー名) は、Flexible Identifiers を構成する要素です。エンドユーザーは、これらの属性を使ってサインアップ、認証、パスワードのリセットを行います。各属性には固有の設定があります。

<div id="activate-attributes">
  ## 属性を有効化
</div>

<Tabs>
  <Tab title="新規接続" icon="database">
    1. [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) に移動します。
    2. **+Create DB Connection** を選択します。
    3. 接続名を入力し、属性を選択します。
    4. 接続の認証方法を選択します。
  </Tab>

  <Tab title="既存の接続" icon="database">
    1. [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) に移動します。
    2. 一覧から更新するデータベース接続を選択します。
    3. **Attributes** タブを選択し、**Activate** を選択します。
    4. 開発環境で設定をテスト済みであることの確認を求められたら、**Proceed** を選択します。
    5. **Attributes** 画面で、**Add Attribute** を選択するか、既存の属性の **Configure** を選択します。
  </Tab>
</Tabs>

新しい接続にはデフォルト値が設定されており、作成後に編集できます。

| 設定                                | 値            |
| --------------------------------- | ------------ |
| `\{attribute}` を識別子として使用          | `True`       |
| `\{attribute}` を使用したサインアップを許可     | 必須           |
| ユーザープロファイルで `\{attribute}` を必須にする | `True`       |
| ユーザー名の長さ                          | 最小: 1 最大: 15 |
| メールアドレス形式のユーザー名を許可                | `False`      |
| 電話番号形式のユーザー名を許可                   | `False`      |

<div id="configure-attributes">
  ## 属性を設定する
</div>

接続の **Attributes** タブを選択します。利用可能な属性から選び、**Configure** を選択します。属性の設定を更新したら、**Save** を選択します。

<div id="configure-or-remove-existing-attributes-on-a-connection">
  ### 接続の既存の属性を設定または削除する
</div>

すべての属性は、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で変更または削除できます。削除した属性は、ログイン、サインアップ、またはパスワードのリセットには使用できず、各接続では少なくとも 1 つの属性を有効なままにしておく必要があります。

属性を変更または削除するには、**Attributes** タブに移動して、**Remove** を選択します。

<div id="revert-a-connection">
  ### 接続を元の状態に戻す
</div>

接続を元の状態に戻すには、以下のように `attributes` パラメータを含まない `options` オブジェクトを指定し、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> の [Update a connection](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) エンドポイントを使用します。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * `options` パラメータを使用すると、`options` オブジェクト全体が上書きされます。不完全なデータやその他の問題を避けるため、接続の `options` オブジェクトの内容をコピーして PATCH 呼び出しに含め、現在のすべてのパラメータが含まれていることを確認してください。削除する必要があるのは `attributes` のみです。
  * テナントで `username` が属性として有効になっている場合、以下の操作を行うには `username` の設定を変更する必要があります。この方法を選択する場合は、ユーザー名の設定を再構成する必要があります。
</Callout>

```json lines
{
"options": {
       }
}
```
