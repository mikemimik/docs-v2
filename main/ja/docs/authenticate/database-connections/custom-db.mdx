カスタムデータベース接続は、主に認証を目的として独自の (レガシー) ID データストアへのアクセスを Auth0 に提供し (<Tooltip tip="Identity Provider (IdP): デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=identity+provider">ID プロバイダー</Tooltip>として機能) 、あわせてユーザーデータを Auth0 のデータストアへ移行したい場合に使用します。

Auth0 の [Extensibility](/ja/docs/customize/extensions) を使用すると、Identity and Access Management (IdAM) 向けのラストマイルソリューションを構築するためのカスタムロジックを追加できます。Auth0 では、カスタムデータベース接続とカスタムデータベース移行の両方に対して、[Actions](/ja/docs/customize/actions) と [scripts](/ja/docs/authenticate/database-connections/custom-db/templates) を通じて拡張性を提供しています。これらはいずれも、Auth0 プラットフォーム上の Auth0 テナントで動作する [Node.js](https://nodejs.org/en/) を使って実装されます。

Auth0 の拡張機能は、IAM パイプライン内のさまざまなポイントで実行されます。

* **Actions** は、Auth0 プラットフォーム内の特定のポイントで実行される、安全でテナント固有のバージョン管理された Node.js 関数です。Actions は、カスタムロジックによって Auth0 の機能をカスタマイズおよび拡張するために使用されます。詳しくは、[Understand how Auth0 Actions Work](/ja/docs/customize/actions/actions-overview) をご覧ください。

* **Custom database action scripts** は、既存のユーザー ID ストアと統合するために使用できるほか、レガシー ID ストアからの[自動ユーザー移行](/ja/docs/manage-users/user-migration)が必要な場合にも使用できます。各種 [templates](/ja/docs/authenticate/database-connections/custom-db/templates) が用意されています。

どのようなユースケースであっても、Auth0 の拡張性を使えば、IAM 運用を要件に合わせてきめ細かく調整できます。ただし、適切に使用しないと、不適切または意図しない利用につながり、後々問題を引き起こすおそれがあります。こうした事態を未然に防ぐため、Auth0 は設計担当者と実装担当者の双方に向けた[ベストプラクティス ガイダンス](/ja/docs/authenticate/database-connections/custom-db/custom-database-connections-scripts)を提供しています。すでに Auth0 の利用を始めている場合でも、少なくとも一度は全体に目を通すことをおすすめします。

<Card title="利用可否は Auth0 プランによって異なります">
  ご利用の Auth0 プランまたは個別契約によっては、この機能を利用できない場合があります。詳しくは、[Pricing](https://auth0.com/pricing) をご覧ください。
</Card>
