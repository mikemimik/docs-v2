---
description: ユーザーをIDプロバイダーから強制的にログアウトさせる方法を説明します。
title: IDプロバイダーからユーザーをログアウトさせる
---

一部のプロバイダーでは、ユーザーを<Tooltip tip="IDプロバイダー（IdP）: デジタルIDを保存および管理するサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+provider">アイデンティティプロバイダー</Tooltip>から強制的にログアウトさせることができます。Auth0 では通常、`/oidc/logout` エンドポイントへのリダイレクトに `federated` クエリ文字列パラメーターを追加することで、これを実現します。

これを行うには、ログアウト URL に `federated` クエリ文字列パラメーターを追加します。

`https://{yourDomain}/oidc/logout?federated`

<div id="federated-logout-support">
  ## フェデレーテッドログアウトへの対応
</div>

以下のアイデンティティプロバイダーは、フェデレーテッドログアウトに対応しています。

* Evernote
* Facebook
* Fitbit
* GitHub
* Google

  * Google Workspace
  * OAuth 2.0
* Microsoft

  * Active Directory Federation Services
  * Microsoft 365
  * Microsoft Entra ID
  * Windows Live
* Okta
* OpenID Connect (OIDC)
* Salesforce/Salesforce Sandbox
* X
* Yahoo
* Microsoft Viva Engage

<div id="clear-application-session">
  ## アプリケーションセッションをクリアする
</div>

Auth0 の Logout エンドポイントでは、Auth0 から、必要に応じてアイデンティティプロバイダーからもログアウトできます。ただし、アプリケーションからはログアウトされません。これについては、ご自身の側で実装する必要があります。ユーザーのセッションをクリアして、アプリケーションからログアウトさせる必要があります。

<div id="alternative-logout">
  ## 代替ログアウト
</div>

[代替ログアウトエンドポイント](https://auth0.com/docs/api/authentication#logout)にフェデレーテッドログアウトを追加するには、ログアウト URL に `federated` クエリ文字列パラメーターを追加します。

`https://{yourDomain}/v2/logout?federated`

<div id="alternative-logout-limitations">
  ### 代替ログアウトの制限事項
</div>

* `returnTo` パラメーターの値として指定された URL や、その URL に含まれるクエリ文字列またはハッシュ情報は検証されません。
* ソーシャルプロバイダーに対するフェデレーテッドログアウトの動作には一貫性がありません。`returnTo` パラメーターの扱いはプロバイダーごとに異なり、機能しない場合もあります。実際の動作を確認するには、ご利用のソーシャルプロバイダーの設定を確認してください。
* Google や Facebook などのソーシャルアイデンティティプロバイダーを使用している場合、ログアウトが正しく機能するよう、Auth0 Dashboard でそれらのプロバイダーの <Tooltip tip="Client ID: Auth0 から登録済みリソースに付与される識別値。" cta="用語集を見る" href="/docs/ja/glossary?term=Client+ID">Client ID</Tooltip> と Secret を設定する必要があります。
* Auth0 Enterprise ユーザーの場合、通常は SharePoint、いくつかの .NET アプリケーション、いくつかの Java アプリケーション、Zendesk など、複数のアプリケーションで <Tooltip tip="Single Sign-On (SSO): ユーザーが 1 つのアプリケーションにログインすると、そのユーザーを他のアプリケーションにも自動的にログインさせるサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=SSO">SSO</Tooltip> が有効になっています。この場合、ユーザーがサインアウトする際は、それらすべてのアプリケーションでも同様にサインアウトされる必要があることがよくあります。
