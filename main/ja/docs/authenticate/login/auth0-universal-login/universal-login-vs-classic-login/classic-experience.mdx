title: クラシックログイン
---

クラシックログインは、JavaScript を使ってカスタマイズできる Auth0 ホスト型のログイン体験です。認証プロセスをアプリケーションに直接組み込むよりも実装の複雑さを抑えられ、クロスオリジン認証に伴うリスクの回避にも役立ちます。

クラシックログインは、Auth0 の JavaScript ライブラリ ([Lock.js](/ja/docs/libraries/lock)、[auth0.js](/ja/docs/libraries/auth0js)、<Tooltip tip="多要素認証（MFA）: SMS で送信されるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> Widget、Password Reset) をベースに構築されています。<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> でクラシックログインページをカスタマイズする際、デフォルトテンプレートでも同じ JavaScript ライブラリが使われているため、標準のユーザーインターフェースからカスタム UI へ自然に移行できます。

クラシックログインページのデフォルトテンプレートを選択したら、ニーズに合わせて変更できます。また、Lock widget のさまざまな動作や外観もカスタマイズ可能です。Auth0.js テンプレートでは、カスタム UI を作成し、アプリケーションのスタイルに合わせて調整できるため、さらに柔軟に対応できます。

<Warning>
  現在、Auth0 の継続的な開発は Universal Login に注力しており、クラシックログインはすでに更新対象ではありません。特別なユースケースで Classic 体験が必要な場合を除き、Universal Login の実装を推奨します。

  ログイン体験の機能比較については、[Universal Login vs. Classic Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login) をご確認ください。
</Warning>
