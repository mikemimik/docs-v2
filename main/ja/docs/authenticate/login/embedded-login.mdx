title: 埋め込みログイン
---

埋め込みログインを使用すると、ユーザーはアプリケーションに直接ログインし、認証のために認証情報を Auth0 サーバーに送信できます。Auth0 では、埋め込みログインの使用を推奨していません。詳しくは、[Centralized Universal Login vs. Embedded Login](/ja/docs/authenticate/login/universal-vs-embedded-login) をご覧ください。

埋め込みログインを使用する場合は、[Cross-Origin Resource Sharing (CORS) 用にアプリケーションを設定](/ja/docs/get-started/applications/set-up-cors)する必要があります。また、[カスタムドメインを設定](/ja/docs/customize/custom-domains)することを推奨します。その後、アプリケーションに [Lock SDK](/ja/docs/libraries/lock) または [Auth0.js SDK](/ja/docs/libraries/auth0js) を実装するか、[Auth0 Authentication API](https://auth0.com/docs/api/authentication) を直接呼び出すことができます。

Auth0 は、複数のアプリケーションタイプにおいて、<Tooltip tip="Passwordless: 最初の要素としてパスワードに依存しない認証の形式。" cta="用語集を見る" href="/ja/docs/glossary?term=Passwordless">Passwordless</Tooltip> 接続による埋め込みログインをサポートしています。

* [ネイティブアプリケーションでの埋め込み Passwordless ログイン](/ja/docs/authenticate/passwordless/implement-login/embedded-login/native)
* [Regular Web Applications での埋め込み Passwordless ログイン](/ja/docs/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Single-Page Apps 向けの埋め込み Passwordless 認証](/ja/docs/authenticate/passwordless/implement-login/embedded-login/spa)
