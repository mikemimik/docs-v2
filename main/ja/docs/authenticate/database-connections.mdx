Auth0 では、識別子 (メールアドレス、ユーザー名、または電話番号) とパスワード、または[パスキー](/docs/ja/authenticate/database-connections/passkeys/configure-passkey-policy)を使用してユーザーを認証するためのデータベース接続を提供しています。これらの資格情報は、Auth0 のユーザーストアまたは独自のデータベースに安全に保存されます。

新しいデータベース接続の作成や既存の接続の管理は、[Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database)で行えます。

  ## Database オプション
</div>

<div id="use-auth0-user-store">
  ### Auth0 のユーザーストアを使用する
</div>

Auth0 では、既定でユーザーを保存するためのデータベースインフラストラクチャが提供されます。このシナリオでは、すべてのデータが Auth0 に保存されるため、認証プロセスで最高のパフォーマンスが得られます。

Auth0 がホストするデータベースは非常に安全です。パスワードが平文で保存またはログに記録されることはなく、**bcrypt** でハッシュ化されます。また、パスワードに対してさまざまなレベルのセキュリティ要件を適用することもできます。詳しくは、[Auth0データベース接続 の Password Strength](/docs/ja/authenticate/database-connections/password-strength) を参照してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  データベース接続では、Auth0 はユーザーごとおよび IP アドレスごとの繰り返しのログイン試行回数を制限しています。詳しくは、[データベースのログイン制限](/docs/ja/troubleshoot/customer-support/operational-policies/rate-limit-policy#database-login-limits) を参照してください。
</Callout>

ユーザーストアとして Auth0 のデータベースインフラストラクチャを使用する場合、ユーザーを Auth0 に移行するためのいくつかの方法があります。[自動移行](/docs/ja/manage-users/user-migration/configure-automatic-migration-from-your-database) 機能を使うと、各ユーザーがログインするたびに 1 人ずつ Auth0 のデータベースに追加されるため、すべてのユーザーに同時にパスワードのリセットを求める必要がありません。あるいは、Management API を使用してジョブを作成し、[ユーザーをインポートする](/docs/ja/manage-users/user-migration/bulk-user-imports) こともできます。詳しくは、移行の例を示した [ユーザー移行シナリオ](/docs/ja/manage-users/user-migration/user-migration-scenarios) を参照してください。

<div id="use-your-own-database">
  ### 独自のデータベースを使用する
</div>

既存のユーザーストアがある場合、またはユーザーの資格情報を自社サーバーに保存したい場合は、Auth0 で [カスタムデータベース](/docs/ja/authenticate/database-connections/custom-db) やリポジトリに接続し、それをアイデンティティプロバイダーとして利用できます。

<Frame>![](/docs/images/cdy7uua7fh8z/3SNKwx0KzkUwbodqUQbtvW/df720f8871b56dd7ec399a24aa621c36/Authentication>Database>use-my-own-database.png)</Frame>

このシナリオでは、ユーザーがログインを試行するたびに実行される、ユーザーを認証するためのログインスクリプトを用意します。必要に応じて、サインアップ、メールアドレスの確認、パスワードリセット、ユーザー削除のための [スクリプトを作成](/docs/ja/authenticate/database-connections/custom-db/templates) することもできます。

これらのスクリプトは Node.js のコードです。Auth0 では、ASP.NET Membership Provider、MongoDB、MySQL、PostgreSQL、SQL Server、Windows Azure SQL Database、Basic Auth でアクセスする Web サービスなど、一般的なデータベースの多くに対応した [テンプレート](/docs/ja/authenticate/database-connections/custom-db/templates) を提供しています。つまり、カスタムスクリプトを使えば、ほぼあらゆる種類のデータベースや Web サービスに接続できます。
