title: Oracle Eloqua Marketing Cloud を SAML サービスプロバイダーとして設定する
---

Oracle Eloqua Marketing Cloud には、以下の <Tooltip tip="Security Assertion Markup Language（SAML）: パスワードを使わずに 2 者間で認証情報をやり取りできる標準化されたプロトコル。" cta="用語集を見る" href="/ja/docs/glossary?term=SAML">SAML</Tooltip> 設定を使用します。

<div id="configure-saml-sso-in-auth0">
  ## Auth0 で SAML SSO を設定する
</div>

1. [Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications) に移動し、新しいアプリケーションを作成するか、更新するアプリケーション名をクリックします。
2. **アドオン** タブに移動し、**SAML2 Web App** トグルを有効にします。
3. **設定** タブで、**Application Callback URL** を `https://login.eloqua.com/auth/saml2/acs` に設定します。

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. 次のコードを **設定** テキストボックスに貼り付け、**Debug** をクリックします。

   ```json lines
   {
       "audience": "<Service Provider Entity Url from within Eloqua>",
        "recipient": "https://login.eloqua.com/auth/saml2/acs",
          "mappings": {
       "user_id": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier",
       "email": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
       "name": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name",
          },
        "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
        "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
       ],
        "destination": "https://login.eloqua.com/auth/saml2/acs"
         }
   ```

   `audience` は `http://foo` に設定されている点に注意してください。これは、Eloqua 側でアイデンティティプロバイダーの設定を完了したら変更されます。
5. タブの一番下までスクロールし、**Enable** をクリックします。
6. **使用状況** タブで **IDプロバイダーのメタデータ** を見つけて、**Download** をクリックし、メタデータファイルをダウンロードします。これは、Eloqua で Auth0 を IDプロバイダーとして設定する際に必要です。

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Usage Tab](/docs/images/cdy7uua7fh8z/3T90BDpyTXFUWDp1JkncBU/86bdba6bed8130b34b42b25c9bf2d232/dashboard-applications-applications-addons-saml2-web-app-usage.png)</Frame>

<div id="configure-saml-sso-in-oracle-eloqua">
  ## Oracle EloquaでSAML SSOを設定する
</div>

1. AdminとしてEloquaにログインし、**設定 &gt; Users &gt; シングルサインオン &gt; IDプロバイダー設定**をクリックします。
2. **Upload Identity Provider from Metadata**をクリックします。Auth0 Dashboardの**使用状況**タブからダウンロードした**IDプロバイダーのメタデータ**をアップロードします。
3. ダウンロードしたメタデータファイルを参照して選択します。
4. 新しく追加したIDプロバイダーを編集し、次の手順を実行します。

   1. **User Identity Mapping**を`Assertion contains the Email Address from the User object`に設定し、**Save**をクリックします。
   2. **Identity Provider Details**から**Service Provider Entity URL**をコピーします。
   3. ELOQUAのIdP設定で、これがデフォルトのIdPである場合は、このIdPをデフォルトとして設定します。

<div id="update-audience-restriction-in-auth0">
  ## Auth0 のaudience制限を更新する
</div>

Eloqua の<Tooltip tip="アイデンティティプロバイダー（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=identity+provider">アイデンティティプロバイダー</Tooltip>設定からコピーした **Service Provider Entity URL** を使用して、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で `audience` 制限を設定します。

1. [Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications) に移動し、更新するアプリケーションの名前をクリックします。
2. **アドオン** タブに移動し、**SAML2 Web App** のトグルを有効にします。
3. **設定** タブで、`audience` を Eloqua からコピーした値に更新します。

<div id="test-configuration">
  ## 設定をテストする
</div>

**Auth0でEloquaにログイン** は、これで有効になっているはずです。IdP-initiated login と SP-initiated login の両方で Eloqua にサインインできます。

* SP-initiated login の場合は、&lt;https://login.eloqua.com/auth/saml2&gt; にアクセスし、会社名を入力して、Auth0 を使用した SAML ログインプロセスを開始します。
* IdP-Initiated login の場合は、**Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション** の **アドオン** タブにある **SAML 2 Web App** の **使用状況** タブで定義されている **IDプロバイダーのログインURL** を使用します。
