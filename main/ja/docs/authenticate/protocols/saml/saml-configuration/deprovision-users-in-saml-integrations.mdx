description: SAML 連携のユーザーのプロビジョニング解除方法を学びます。
title: SAML 連携でユーザーのプロビジョニングを解除する
---

ユーザーのアプリケーションへのアクセスを削除する必要がある場合は、少なくとも <Tooltip tip="Identity Provider（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">アイデンティティプロバイダー</Tooltip> でそのユーザーのプロビジョニングを解除する必要があります。必要な手順はアイデンティティプロバイダーによって異なります。詳細については、ご利用のプロバイダーに確認してください。

ユーザーのアカウントがアイデンティティプロバイダーで削除または無効化されると、そのユーザーはログインできなくなります。

Auth0 がサービスプロバイダーである場合、またはアプリが Auth0 と連携している場合は、プロビジョニング解除済みのユーザーの Auth0 ユーザーアカウントも削除することを検討するとよいでしょう。Auth0 がアイデンティティプロバイダーでもサービスプロバイダーでも、[Auth0 Dashboard](https://manage.auth0.com/#/users) または <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用してユーザーを削除できます。
