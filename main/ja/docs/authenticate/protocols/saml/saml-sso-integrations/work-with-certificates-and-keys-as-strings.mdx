description: ファイルの文字列表現が必要なルールまたは Management API リクエストで、証明書やキーを扱う方法について説明します。
title: 証明書とキーを文字列として扱う
---

UNIX シェルでは、PEM 形式の証明書を `\n` (エスケープした改行) で連結するために、次のコマンドを使用できます。

```bash wrap lines
awk 'NF {sub(/\r/, ""); printf "%s\\n",$0;}' certificate.pem | pbcopy
```

これにより、値がクリップボードにコピーされ、必要な場所に貼り付けられるようになります。

<div id="more-information">
  ### 詳細情報
</div>

ルールや<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> v2 のリクエストで証明書やキーを扱う場合は、通常、そのファイルの文字列表現が必要になります。

テキストエディターで証明書ファイル (`cer`、`pem`) を開くと、次のように表示されます。

```text lines
-----BEGIN CERTIFICATE-----
MIICzDCCAbQCCQDH8GvxPIeH+DANBgkqhkiG9w0BAQsFADAoMQswCQYDVQQGEwJh
cjEZMBcGA1UEAwwQaHR0cHM6Ly9uaWNvLmNvbTAeFw0xOTA0MDgxODA3NTVaFw0y
//
// base64エンコードされた情報がさらに続く
//
nSWyabd+LiBGtLTMB+ZLbOIi3EioWPGw/nHOI8jzPrqhiCLuZCSQmiqrLQYNsc1W
-----END CERTIFICATE-----
```

`-----BEGIN CERTIFICATE-----` ヘッダーと `-----END CERTIFICATE-----` フッターの間の行には、base64 でエンコードされたバイナリ情報が含まれています。公開鍵と秘密キー (.key ファイル) も、ヘッダー／フッターが異なるだけで、見た目はよく似ています。

証明書／キーファイルを文字列表現にするには、ファイル内の実際の改行を `\n` (エスケープされた改行) に置き換え、全体を 1 行に連結する必要があります。したがって、上記のサンプルからは次のような文字列が得られます。

```text lines
"-----BEGIN CERTIFICATE-----
\nMIICzDCCAbQCCQDH8GvxPIeH+DANBgkqhkiG9w0BAQsFADAoMQswCQYDVQQGEwJh\n
[..all the other lines..]
-----END CERTIFICATE-----\n"
```
