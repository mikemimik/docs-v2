<Warning>
  この機能では委譲を使用します。デフォルトでは、2017 年 6 月 8 日時点で使用中のアドオンがないテナントでは、委譲は無効になっています。現在、委譲を必要とするアドオンを使用しているレガシーテナントは、この機能を引き続き利用できます。今後、委譲の機能が変更されたり、サービスから削除されたりする場合は、現在利用しているお客様に事前に通知し、移行のための十分な時間が提供されます。なお、委譲はカスタムドメインの使用をサポートしていないため、これに依存する機能はカスタムドメインと併用すると完全に動作しない可能性があります。
</Warning>

<div id="step-3-build-the-application">
  ## ステップ3 - アプリケーションを構築する
</div>

このステップでは、AngularJSフレームワークを使用して、静的ウェブサイトとして動作するように設定した AWS S3 バケットから配信するシングルページのサーバーレスアプリケーションを構築します。

<div id="step-1-set-up-your-sample-application">
  ### ステップ 1. サンプルアプリケーションをセットアップする
</div>

シンプルなスターターアプリとして、このチュートリアル用の[サンプルプロジェクトをダウンロード](https://github.com/auth0/auth0-aws/tree/master/examples/api-gateway/client)して始めてください。ログインすると、Auth0 の資格情報があらかじめ設定されます。

このテンプレートプロジェクトの内容を、このチュートリアルの残りの手順で使用する `pets` という名前のローカルフォルダにコピーします。このフォルダ内で、`auth0-variables.js` を更新し、Auth0 アプリケーションの `AUTH0_CLIENT_ID` と `AUTH0_CLIENT_ID` を設定してください (この情報は、該当するアプリケーションの[管理ダッシュボード](https://manage.auth0.com/#/applications)で確認できます) 。

<div id="aws-s3-bucket">
  #### AWS S3 バケット
</div>

[静的ウェブサイトとして機能するように設定された AWS S3 バケットを作成](http://docs.aws.amazon.com/gettingstarted/latest/swh/website-hosting-intro.html)してあることを確認してください。設定の際に、Web サイトのコンテンツとして使用するため、`pets` フォルダーの内容を S3 バケットにコピーします。

既存のバケットを使用している場合は、次のコマンドを使用して AWS CLI でファイルを移動できます。

```bash lines
aws s3 cp --recursive --acl "public-read" ./ s3://{yourBucket}/
```

先に進む前に、Username-Password-Authentication 接続 (または使用しているアプリケーションに関連付けられたデータベース接続) に、少なくとも 1 人のユーザーが関連付けられていることを確認してください。サンプルアプリの機能と AWS との連携を十分に試すには、そのユーザーを使って認証をテストし、アクセスできることを確認する必要があります。

最後に、アプリケーションの**設定**ページにある **Allowed Origins** フィールドに URL を指定して、Auth0 があなたの Web サイトからの認証を許可するようにしてください。Web サイトの URL は、次のようになります。

`http://your-bucket.s3-website-us-east-1.amazonaws.com`

URL がわからない場合は、S3 バケットの **Properties** タブで確認できます。

先に進む前に、アプリケーションへのログインをテストしてください。ブラウザーで `http://your-bucket-domain/index.html` を開きます。ログインすると、&quot;getPets not implemented&quot; と表示された alert ボックスがポップアップするはずです。

また、ペットを表示するページも表示されるはずです。

<Frame>![AWS API Gateway - project directory](/docs/images/cdy7uua7fh8z/5fYL14NE1L9WuzUyA0xTky/35e78ca14e6d0c5b36582b5b74a6fc55/aws-api-gateway-project.png)</Frame>

Auth0 の委譲機能を使うと、Auth0 の <Tooltip tip="ID Token: リソースへのアクセスではなく、クライアント自体を対象としたクレデンシャルです。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+token">ID トークン</Tooltip> に基づく AWS の <Tooltip tip="Access Token: API にアクセスするために使われる認可クレデンシャルで、不透明な文字列または JWT の形式を取ります。" cta="用語集を見る" href="/docs/ja/glossary?term=Access+Token">Access Token</Tooltip> を取得できます。内部的には、Auth0 が ID トークンを認証し、その後、設定した addon に基づいて <Tooltip tip="ID Token: リソースへのアクセスではなく、クライアント自体を対象としたクレデンシャルです。" cta="用語集を見る" href="/docs/ja/glossary?term=SAML">SAML</Tooltip> を使用します。

`auth.signin` で正常にサインインした後、ID トークンから AWS 委譲トークンを取得できるように、以下のように `pets/login/login.js` を更新します。なお、Social Connection を使わずにログインしたユーザーは、すべて管理者として扱っています。後で 2 つ目の role をコード化し、role の選択をより適切に強制する方法を紹介します。

  profile.isAdmin = !profile.identities[0].isSocial;
  store.set('profile', profile);
  store.set('token', token);

  // IDトークンから委譲トークンを取得する。
  var options = getOptionsForRole(profile.isAdmin, token);

<div id="modify-the-role-and-principal-strings">
  #### `role` と `principal` の文字列を変更する
</div>

`role` と `principal` の文字列 (提供されている関数内の `if` 文の最後の 2 行) を変更するには、[ルール](https://manage.auth0.com/#/rules) で適切な値を指定します。

```javascript lines
function (user, context, callback) {
  if (context.clientID === 'CLIENT_ID' &&
      context.protocol === 'delegation') {
    // AWS設定をセットする
    context.addonConfiguration = context.addonConfiguration || {};
    context.addonConfiguration.aws = context.addonConfiguration.aws || {};
    context.addonConfiguration.aws.principal = 'arn:aws:iam::[omitted]:saml-provider/auth0-provider';
    context.addonConfiguration.aws.role = 'arn:aws:iam::[omitted]:role/auth0-role';
  }

  callback(null, user, context);
}
```

必ず、`role` と `[principal]` の ARN の値を、ご利用の連携に合わせたものに更新してください。

更新したファイルを、ウェブサイト用の S3 バケットにコピーしてください。

必要に応じて、ブラウザーで `store.set('awstoken', delegation.Credentials);` にブレークポイントを設定できます。ログアウトして再度ログインしたら、ブレークポイントに到達した時点で `delegation.Credentials` を確認してください。AccessKeyId や SecretAccessKey など、見覚えのある値が表示されます。

```js lines
{
    AccessKeyId: "ASIAJB...BNQ",
    SecretAccessKey: "vS+b6...2Noav",
    SessionToken: "AQoDYBqsivOV...DdQW0gsKr8rgU=",
    Expiration: "2015-08-27T14:48:32.000Z"
}
```

これらの値が表示されない場合は、Auth0 アプリケーションの [Addons] タブで Amazon Web Services アドオンが有効になっていることを確認してください。

<div id="add-the-api-code-to-call-your-api">
  #### API を呼び出すコードを追加する
</div>

サービスを呼び出す API コードを追加するには、以前ダウンロードした apiGateway-js-sdk.zip の内容を `pets` ディレクトリにコピーします。内容には次のものが含まれているはずです。

* `apiClient.js`;
* `lib` フォルダー;

ダウンロードした内容を確認するには、[AWS API Gateway Tutorial Part 2](/docs/ja/customize/integrations/aws/aws-api-gateway-delegation/aws-api-gateway-delegation-2#deploy-the-api) を参照してください。

<div id="configure-the-getpets-method">
  #### getPets メソッドを設定する
</div>

`home` フォルダー内の `home.js` を開き、`getPets` の内容をペットデータを取得するメソッドに更新します (`us-east-1` 以外で実行している場合は、リージョンも必ず更新してください) :

```javascript lines
function getPets() {
    // これは未認証です
    var apigClient = apigClientFactory.newClient({
        region: 'us-east-1' // APIがデプロイされているリージョン
    });

    apigClient.petsGet({},{})
      .then(function(response) {
        console.log(response);
        $scope.pets = response.data;
        $scope.$apply();
      }).catch(function (response) {
        alert('pets get failed');
        showError(response);
    });
}
```

更新したコードを S3 バケットにコピーします。ページを更新すると、2 匹の動物が一覧表示されます (これらのペットを作成する、前述の API テストを実行していれば) 。

<div id="update-pets-with-the-aws-api-service">
  ### AWS API サービス を使って Pets を更新する
</div>

これで、API Gateway と連携する Auth0 アプリケーションが動作するようになったので、`pets` テーブルを更新するためのメソッドを追加します。

API 関数を使って `pets` を更新するように、`putPets` メソッドのロジックを変更します。この関数は、`pets` の追加と削除の両方で使用します。

```javascript lines
function putPets(updatedPets) {
    var body = {pets: updatedPets};

    var apigClient = apigClientFactory.newClient({
        region: 'us-east-1' // 使用しているリージョンに設定してください。
    });

    apigClient.petsPost({},body)
      .then(function(response) {
        console.log(response);
       }).catch(function (response) {
        alert('pets update failed');
        showError(response);
      });
}
```

更新したコードを S3 バケットにコピーし、メソッドをテストします。

1. ログアウトしてから、もう一度ログインします。
2. `Pet Type` と `Pet Price` に値を入力します。
3. **Save** をクリックしてデータを送信します。

左側に赤い **REMOVE** ボタンが付いた「`<Pet Type>` を `<Pet Price>` で販売中です」というメッセージが表示されるはずです。

セキュリティを強化するには、`putPets` メソッドの先頭に `getSecureApiClient` 関数を追加します。

```javascript lines
function putPets(updatedPets) {
     var apigClient = getSecureApiClient();
}
```

コードをお使いのS3バケットにコピーします。

用意されている `getSecureApiClient` 関数は、API への委譲を使って取得した AWS トークンをローカルストレージから取得し、アクセスキー、シークレット、およびセッショントークンを使用します。

```javascript lines
function getSecureApiClient() {
    var awstoken = store.get('awstoken');

    return apigClientFactory.newClient({
        accessKey: awstoken.AccessKeyId,
        secretKey: awstoken.SecretAccessKey,
        sessionToken: awstoken.SessionToken,
        region: 'us-east-1' // 使用しているリージョン。
    });
  }
```
