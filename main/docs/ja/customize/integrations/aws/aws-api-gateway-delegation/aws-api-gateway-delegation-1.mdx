<div id="step-1-set-up-the-amazon-api-gateway">
  ## ステップ 1 - Amazon API Gateway を設定する
</div>

このステップを完了すると、次の状態になります。

* AWS Lambda 関数を使用して、Amazon DynamoDB テーブルへのペットの保存と取得を行うサービスロジックを実行する Amazon API Gateway の設定が完了します。
* ペット一覧の取得と更新を行う、認証不要の REST サービスメソッドが 2 つ作成されます。

始める前に、[Node.js](https://nodejs.org/) をインストールしておいてください。

<div id="step-1-create-the-amazon-dynamodb-table">
  ### ステップ 1. Amazon DynamoDB テーブルを作成する
</div>

[Amazon DynamoDB Console](https://console.aws.amazon.com/dynamodb) で、**Create Table** をクリックします。

<Frame>![DynamoDB Console - Create Table](/docs/images/cdy7uua7fh8z/2ZPhi7Ddx4TznSXEEhUSDK/52a06412f7c5e6d0ef0d13c6ffe17611/dynamodb-create-table.png)</Frame>

テーブルに関連する項目を設定します。

* **Table name**: Pets
* **Primary key**: ユーザー名
* **Primary key type**: String
* **Use default settings**: オフ
* **Read capacity units**: 3
* **Write capacity units**: 3

<Frame>![DynamoDB Console - Configure Newly Created Table](/docs/images/cdy7uua7fh8z/6MYJV97Xa3DxajG8DYj2dx/e428ab2ce137176d7771c61a9c528f7c/configure-newly-created-table.png)</Frame>

指定した設定でテーブルを作成するには、**Create** をクリックします。

テーブルの作成中に、Table details セクションにある Amazon Resource Name (ARN) を控えておいてください。次のステップでテーブルの ARN が必要になります。

<Frame>![DynamoDB Console - Table Details - Amazon Resource Name](/docs/images/cdy7uua7fh8z/4ZFemddtUD8tRi0NUvzXJ2/38abd951974cad385ef7417f48adf359/table-arn.png)</Frame>

<div id="step-2-create-the-policy-that-grants-aws-lambda-functions-access-to-the-dynamodb-pets-table">
  ### ステップ 2. AWS Lambda 関数に DynamoDB Pets テーブルへのアクセスを許可するポリシーを作成する
</div>

[AWS IAM Console](https://console.aws.amazon.com/iam) に移動します。

左側のメニューで **Roles** をクリックし、次に **Create New Role** ボタンをクリックします。

<Frame>![IAM Console - Roles - Create New Roles](/docs/images/cdy7uua7fh8z/4mgCTXLpycHLA8RbYoEFim/5d9eb81be979355aa9cd10d476ead515/roles.png)</Frame>

ロール名に `APIGatewayLambdaExecRole` を入力し、**Next Step** をクリックします。

Role Type を選択します。AWS Service Roles で AWS Lambda を選択します。

<Frame>![IAM Console - Select the Role Type - Service Roles - AWS Lambda](/docs/images/cdy7uua7fh8z/326kKGbahlvIwVFTi61Gfw/026d1fc0c8c6e3a489ec4fb840f551da/select-role-type.png)</Frame>

Attach Policy 画面では、**Next Step** をクリックしてこの手順をスキップします。ここで、入力した内容を確認します。問題がなければ、**ロールの作成** をクリックします。完了すると、IAM ホームページにロールが表示されます。

先ほど作成した **APIGatewayLambdaExecRole** ロールを選択します。Inline Policies の下向き矢印をクリックし、**Click Here** リンクをクリックします。

Custom Policy を選択し、**Select** をクリックします。ポリシー名に `LogAndDynamoDBAccess` を入力し、以下のコードをポリシードキュメントとして追加します (先に DynamoDB テーブルの Amazon Resource Name (ARN) を更新してください) 。**Apply Policy** をクリックします。

```json lines
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AccessCloudwatchLogs",
      "Action": ["logs:*"],
      "Effect": "Allow",
      "Resource": "arn:aws:logs:*:*:*"
    },
    {
      "Sid": "PetsDynamoDBReadWrite",
                  "Effect": "Allow",
      "Action": [
                  "dynamodb:DeleteItem",
                  "dynamodb:GetItem",
                  "dynamodb:PutItem",
                  "dynamodb:UpdateItem"
                  ],
      "Resource": ["DYNAMODB_TABLE_ARN_VALUE_FROM_PREVIOUS_STEP"]
    }
   ]
}
```

<div id="step-2-create-the-aws-lambda-functions">
  ### ステップ 2. AWS Lambda 関数を作成する
</div>

次の 3 つのステップで、DynamoDB テーブル内のペット情報を取得・更新するための AWS Lambda 関数を作成します。

<div id="create-the-lambda-function-for-getpetinfo">
  #### GetPetInfo 用の Lambda 関数 を作成する
</div>

[AWS Lambda Console](https://console.aws.amazon.com/lambda) で、**Create a Lambda Function** を選択します (これまでに AWS Lambda 関数を作成したことがない場合は、**Get Started Now** をクリックします) 。

[Select blueprint] 画面で、**Blank Function** をクリックします。

次に、トリガーを設定するよう求められます。先に進むには **Next** をクリックします。現時点では設定する必要はありません。

最後に、function を設定するよう求められます。

該当するフィールドに次の情報を入力します。

* **Name**: `GetPetInfo`
* **Runtime**: Node.js 6.10

以下のコードを **Lambda function code** 領域に貼り付け、DynamoDB テーブルからペット情報を読み取ります。

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();

exports.handler = function(event, context) {
   var cb = function(err, data) {
      if(err) {
         console.log('error on GetPetsInfo: ',err);
         context.done('Unable to retrieve pet information', null);
      } else {
         if(data.Item && data.Item.pets) {
             context.done(null, data.Item.pets);
         } else {
              context.done(null, {});
         }
      }
   };

   dynamo.getItem({TableName:"Pets", Key:{username:"default"}}, cb);
};
```

Role では、**Choose an existing role** を選択します。次に、Existing Role として **APIGatewayLambdaExecRole** を選びます。その他の設定はすべて既定値のままにします。

**Next** をクリックして、入力した情報を確認します。問題がなければ、**Create function** をクリックします。

**Test** をクリックし、Input test event は既定のままにします (Hello World テンプレートが使用されます) 。テストが完了すると、Execution Result セクションに空の出力 (`{}`) が表示されます。テーブルは空です。

<div id="create-the-lambda-function-for-updatepetinfo">
  #### UpdatePetInfo 用の Lambda 関数 を作成する
</div>

`GetPetInfo` 関数の作成時と同じ手順を繰り返しますが、関数コードには代わりに以下を使用します。

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();
exports.handler = function(event, context) {
    var item = { username:"default",
                 pets: event.pets || {}
            };

    var cb = function(err, data) {
        if(err) {
            console.log(err);
            context.fail('unable to update pets at this time');
        } else {
            console.log(data);
                context.done(null, data);
        }
    };
    dynamo.putItem({TableName:"Pets", Item:item}, cb);
};
```

**Actions** ドロップダウンをクリックして **Configure sample event** を選択し、関数をテストします。サンプルデータとして以下を入力し、**Submit** をクリックします。

```json lines
{
  "pets": [{
    "id": 1,
    "type": "dog",
    "price": 249.99
  }]
}
```

空の戻り値 (`{}`) が表示されるはずです。

`GetPetInfo` Lambda 関数 に戻り、もう一度 **Test** をクリックします。今度はペットが 1 匹表示されるはずです。

<div id="create-the-third-lambda-function">
  #### 3つ目のLambda 関数 を作成する
</div>

Lambda 関数 をもう1つ追加で作成します。この関数自体は何もしませんが、後のセクションで説明するCORS用のOPTIONSメソッドで必要になります。

上記の手順に従って、`NoOp` という名前のLambda 関数 を作成します。関数のコードは次のとおりです。

```js lines
exports.handler = function(event, context) {
    context.succeed('');
}
```

この3つ目のLambda関数を作成する代わりに、API Gatewayで [OPTIONS メソッドを作成](#method-options)することもできます。

<div id="step-3-create-the-amazon-api-gateway-api">
  ### ステップ 3. Amazon API Gateway API を作成する
</div>

2 つのメソッドを持つ API を作成します。1 つはペット情報を取得する `GET` メソッド、もう 1 つはペット情報を送信する `POST` メソッドです。

<div id="method-get-pet-information">
  #### Method: GET Pet Information
</div>

[Amazon API Gateway Console](https://console.aws.amazon.com/apigateway) に移動し、**Create API** をクリックします。API を初めて作成する場合は、代わりに **Get Started** を促す画面が表示されます。

API を初めて作成する場合は、Example API を作成するよう求められます。ポップアップ通知を閉じるために **OK** をクリックし、**Example API** ではなく **New API** のラジオボタンを選択します。

API 名に `SecurePets` を入力し、**Create API** をクリックします。

`SecurePets` API の Resources タブに移動し、**Create Resource** をクリックします。

リソース名に `Pets` を入力し、再度 **Create Resource** をクリックします。

左側のペインで `/pets` を選択し、**CreateMethod** ボタンをクリックします。

ドロップダウンで GET を選択し、チェックマークボタンをクリックします。`GET` メソッドには以下の設定値を指定します。

* **Integration type**: Lambda 関数;
* **Lambda Region**: 使用している Region;
* **Lambda Function**: GetPetInfo.

**Save** をクリックし、続いて Lambda 関数 に権限を付与するためのポップアップが表示されたら **OK** をクリックします。

次に表示される Method Execution ウィンドウで、**Test** をクリックします。

レスポンスボディに 1 件のペット情報が返されます。

<div id="method-post-pet-information">
  #### Method: POST ペット情報
</div>

ペット情報を `POST` するための API の作成手順は、ペット情報を `GET` するための API を作成する場合とほぼ同じです。

左側のペインで `/pets` を選択し、**CreateMethod** をクリックします。

ドロップダウンで POST を選択し、チェックマークのボタンをクリックします。

Integration Type で Lambda 関数 を選択し、使用しているリージョンを選択して、Lambda 関数 には UpdatePetInfo を選択します。

**Save** をクリックし、Lambda 関数 に権限を付与するよう popup で求められたら、**OK** をクリックします。

**Test** をクリックし、request body に以下を貼り付けます。

```json lines
{"pets": [
    {"id": 1, "type": "dog", "price": 249.99},
    {"id": 2, "type": "cat", "price": 124.99}
  ]
}
```

空の戻り値 (`{}`) が表示されるはずです。

GET メソッドに戻り、もう一度 **Test** をクリックして、レスポンス本文にテーブル内のペットが 2 匹あることが示されるのを確認します。

```json lines
[
  {
    "id": 1,
    "price": 249.99,
    "type": "dog"
  },
  {
    "id": 2,
    "price": 124.99,
    "type": "cat"
  }
]
```

<div id="method-options">
  #### Method: OPTIONS
</div>

何も実行しないLambda 関数を作成する代わりに、API Gateway に `OPTIONS` メソッドを作成できます。

左側のペインで `/pets` を選択し、**CreateMethod** をクリックします。ドロップダウンで OPTIONS を選択し、チェックマーク ボタンをクリックします。Integration Type で モック を選択します。**Save** をクリックします。

Response Body は空のまま、**Test** をクリックします。`no data` を示す Response Body が返されるはずです。

この時点では、AWS Lambda 関数と Amazon API Gateway のメソッドは、セキュリティが設定されていない状態で定義されています。
