<Warning>
  Rules と Hooks のサポート終了日 (EOL) は **2026年11月18日** です。なお、**2023年10月16日** 以降に作成された新規テナントでは、これらはすでに利用できません。既存のテナントで有効な Hooks を使用している場合は、サポート終了日まで Hooks へのアクセスが維持されます。

  Auth0 を拡張するには、Actions の利用を強くお勧めします。Actions では、豊富な型情報、インラインドキュメント、公開 `npm` パッケージを利用できるほか、拡張性をさらに高める外部連携も利用できます。Actions の詳細については、[Understand How Auth0 Actions Work](/docs/ja/customize/actions/actions-overview) をご覧ください。

  移行に役立つよう、[Rules から Actions への移行](/docs/ja/customize/actions/migrate/migrate-from-rules-to-actions) および [Hooks から Actions への移行](/docs/ja/customize/actions/migrate/migrate-from-hooks-to-actions) のガイドを用意しています。さらに、機能比較、[Actions のデモ](https://www.youtube.com/watch?v=UesFSY1klrI)、および移行に役立つその他のリソースをまとめた専用ページ [Move to Actions](https://auth0.com/extensibility/movetoactions) もあります。

  Rules と Hooks の非推奨化について詳しくは、ブログ記事 [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/) をご覧ください。
</Warning>

<Warning>
  2026 年に Rules と Hooks の機能を廃止する予定のため、新しい Rules または Hooks を作成するのは Development 環境のみに限定し、Actions への移行をテストする目的でのみ行ってください。

  Rules を Actions に移行する方法については、[Migrate from Rules to Actions](/docs/ja/customize/actions/migrate/migrate-from-rules-to-actions) をご覧ください。Hooks を Actions に移行する方法については、[Migrate from Hooks to Actions](/docs/ja/customize/actions/migrate/migrate-from-hooks-to-actions) をご覧ください。
</Warning>

Hooks は Dashboard または <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を使用して更新できます。

また、[Deploy Command-Line Interface (CLI) tool](/docs/ja/deploy-monitor/deploy-cli-tool) を使用して Hooks をインポートおよびエクスポートすることもできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  必要に応じて、Hooks にシークレット (Twilio Keys やデータベース接続文字列など) を追加できます。
</Callout>

<div id="rename-hooks-using-the-dashboard">
  ## Dashboard を使用して Hook の名前を変更する
</div>

1. [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/) に移動し、名前を変更する Hook の横にある歯車アイコンをクリックします。
2. **Rename** をクリックします。
3. Hook の新しい名前を入力し、**Rename** をクリックします。

   <Frame>![Auth0 Rename Hooks](/docs/images/cdy7uua7fh8z/2Xp8Z3uYGoGZrF5EGXLPLQ/6c68c77f97133b2066219a68540b1e73/Screen_Shot_2021-05-19_at_12.08.00_PM.png)</Frame>

<div id="update-hook-scripts-using-the-dashboard">
  ## Dashboard を使用してHookスクリプトを更新する
</div>

1. [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/) に移動し、更新するHookの横にある鉛筆アイコンをクリックします。

   <Frame>![Auth0 Hooks でHookを選択](/docs/images/cdy7uua7fh8z/2y2R713TkAvQWIMZPMd1zh/bf0a657261f2d090492660ea5962e0a3/dashboard-hooks-list.png)</Frame>
2. Hook Editor でHookを更新し、ディスクアイコンをクリックして保存します。

   <Frame>![Auth0 Hook Editor](/docs/images/cdy7uua7fh8z/26u71QdXxZxg5zIH8SJQAg/e9bbd25b99d9b8fd44c281b6de83a5a3/dashboard-hooks-edit_menu.png)</Frame>

<div id="update-hooks-using-the-management-api">
  ## Management API を使用してフックを更新する
</div>

[Hook を更新するエンドポイント](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_hooks_by_id)に `PATCH` リクエストを送信します。`HOOK_ID` と `MGMT_API_ACCESS_TOKEN` のプレースホルダー値は、それぞれご自身の Hook ID と Management API の<Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を見る" href="/docs/ja/glossary?term=Access+Token">アクセストークン</Tooltip>に必ず置き換えてください。

<AuthCodeGroup>

  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/hooks/HOOK_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \

    --header 'content-type: application/json' \
    --data '{ "name": "HOOK_NAME", "script": "HOOK_SCRIPT", "enabled": "true" }'

| Value                   | Description                                                                                                                                                                    |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `HOOK_ID`               | 更新する Hook の ID。                                                                                                                                                                |
| `MGMT_API_ACCESS_TOKEN` | [scope](/docs/ja/glossary?term=scope) `update:hooks` を持つ Management API のアクセストークン。詳しくは、[Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens) を参照してください。 |
| `HOOK_NAME`             | 作成する Hook の名前。                                                                                                                                                                 |
| `HOOK_SCRIPT`           | Hook のコードを含むスクリプト。Dashboard で新しい Hook を作成する際に入力する内容と一致している必要があります。                                                                                                             |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  `enabled` プロパティは、そのルールが有効 (`true`) か無効 (`false`) かを表します。
</Callout>

<Card title="Hook 内から API を呼び出す際のレート制限への対応">
  Hook のスクリプト内から Auth0 API を呼び出す場合は、レート制限に対応する必要があります。そのためには、`X-RateLimit-Remaining` ヘッダーを確認し、返された値が 0 に近づいたら適切に対処してください。

  さらに、設定されたレート制限を超えて `429` HTTP Status Code (`Too Many Requests`) を受け取った場合に備えたロジックも追加してください。この場合、再試行が必要であれば、無限の再試行ループを避けるためにバックオフを設けるのが最適です。
</Card>
