<Warning>
  Rules と フック のサポート終了日 (EOL) は **2026年11月18日** です。また、**2023年10月16日** 以降に作成された新規テナントでは、すでに利用できません。アクティブな フック を使用している既存のテナントでは、サポート終了日まで フック 製品を引き続き利用できます。

  Auth0 を拡張するには、Actions の使用を強く推奨します。Actions では、豊富な型情報、インラインドキュメント、公開 `npm` パッケージを利用できるほか、外部連携にも接続でき、拡張性に関する全体的な利用体験を向上させることができます。Actions の詳細については、[Understand How Auth0 Actions Work](/docs/ja/customize/actions/actions-overview) をご覧ください。

  移行を支援するために、[Rules から Actions への移行](/docs/ja/customize/actions/migrate/migrate-from-rules-to-actions) と [Hooks から Actions への移行](/docs/ja/customize/actions/migrate/migrate-from-hooks-to-actions) のガイドを用意しています。また、機能比較、[Actions のデモ](https://www.youtube.com/watch?v=UesFSY1klrI)、そのほか移行に役立つリソースをまとめた専用ページ [Move to Actions](https://auth0.com/extensibility/movetoactions) もあります。

  Rules と フック の廃止に関する詳細については、ブログ記事 [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/) をご覧ください。
</Warning>

<Warning>
  2026年に Rules と フック の機能を削除する予定のため、新しい Rules または フック を作成する場合は、Development 環境でのみ、かつ Actions への移行をテストする目的に限って行ってください。

  Rules を Actions に移行する方法については、[Migrate from Rules to Actions](/docs/ja/customize/actions/migrate/migrate-from-rules-to-actions) をご覧ください。フック を Actions に移行する方法については、[Migrate from Hooks to Actions](/docs/ja/customize/actions/migrate/migrate-from-hooks-to-actions) をご覧ください。
</Warning>

Dashboard または <Tooltip tip="Management API: お客様が管理タスクを実行するための製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を使用して、フック を有効化または無効化できます。

<Warning>
  特定の拡張ポイントに対して複数の フック を作成できますが、各拡張ポイントで同時に有効化できる フック は 1 つだけです。その拡張ポイントに対して後から作成した フック は自動的に無効になるため、明示的に有効化する必要があります。有効化されている フック は、すべてのアプリケーションと API に対して実行されます。
</Warning>

<Warning>
  新しい フック を作成すると、Auth0 はその拡張ポイントに対して最初に作成した フック を自動的に有効化します。その拡張ポイントに対して後から作成した フック は自動的に無効になるため、明示的に有効化する必要があります。
</Warning>

<div id="use-the-dashboard">
  ## Dashboardを使用する
</div>

1. [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/) に移動し、Hook を有効化または無効化する拡張ポイントを探します。
2. 拡張ポイントの名前と説明のすぐ下にあるドロップダウンボックスをクリックします。

   <Frame>![Auth0 Hook selection](/docs/images/cdy7uua7fh8z/2LCBPLtrsjmmJYxCGfWquT/74b232fd96aa179c89f01e46d2e91ce3/dashboard-hooks-list.png)</Frame>
3. 有効化したい Hook を選択し、確認します。すべての Hook を無効化する場合は、`None` を選択します。

有効化されている Hook の名前の横には、緑色の点が表示されます。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

[Update a Hook endpoint](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_hooks_by_id) エンドポイントに `PATCH` リクエストを送信します。`HOOK_ID` と `MGMT_API_ACCESS_TOKEN` のプレースホルダー値は、それぞれご自身の Hook ID と Management API の <Tooltip tip="Access Token: API にアクセスするために使用される認可資格情報で、不透明な文字列または JWT の形式を取ります。" cta="用語集を見る" href="/docs/ja/glossary?term=Access+Token">アクセストークン</Tooltip> に必ず置き換えてください。

<AuthCodeGroup>
  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/hooks/HOOK_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "enabled": "true" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/hooks/HOOK_ID");
  var request = new RestRequest(Method.PATCH);
  request.AddHeader("content-type", "application/json");
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("cache-control", "no-cache");
  request.AddParameter("application/json", "{ "enabled": "true" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/hooks/HOOK_ID"

  	payload := strings.NewReader("{ "enabled": "true" }")

  	req, _ := http.NewRequest("PATCH", url, payload)

  	req.Header.Add("content-type", "application/json")
  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("cache-control", "no-cache")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

| 値                       | 説明                                                                                                                                                                            |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `HOOK_ID`               | 更新対象のフックの ID。                                                                                                                                                                 |
| `MGMT_API_ACCESS_TOKEN` | [スコープ](/docs/ja/glossary?term=scope) `create:hooks` を持つ Management API のアクセストークン。詳しくは、[Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens) を参照してください。 |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  `enabled` プロパティは、そのルールが有効 (`true`) か無効 (`false`) かを示します。
</Callout>
