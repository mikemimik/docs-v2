<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ログストリームは、監視と分析を目的として設計されています。ログストリームをアプリケーションのクリティカルパスで使用したり、リアルタイムの意思決定に利用したりすることは推奨していません。

  デフォルトでは、ログは実際に発生した順序どおりに表示されない場合があります。テナントログをストリーミングしていて時系列順での並べ替えが必要な場合は、ストリーム処理側でログの並べ替えを実装することを推奨します。
</Callout>

Auth0 のログストリーミングサービスを使用すると、テナントログをログ分析サービスにエクスポートできます。<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> では、コンテンツタイプと形式を選択し、payload をプレビューできます。ログストリーミングを使用すると、パスワード変更や新規登録といったログイベントに対して、独自のビジネスロジックで対応できます。Auth0 Dashboard と API を使用して、ストリームの一時停止、設定の表示と更新、ストリームの稼働状況の確認を行えます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ログストリーム を有効にすると、Customer は当該情報の Controller となり、どの情報をストリーミングするかや、Customer がそれをどのように使用するかを含め、Auth0 Platform からエクスポートされるデータについて責任を負います。
</Callout>

Auth0 Dashboard では、サポートされているサービスのいずれかを使用して新しいログストリームを作成できます。また、カスタムウェブフックを使用して、すでに利用しているサービスへのストリームを設定することもできます。

<div id="delivery-attempts">
  ## 配信の試行
</div>

Auth0 は、システム内で各ログがトリガーされるたびに、お客様のサーバーへ配信します。ログは少なくとも 1 回配信されることを保証します。

ログがトリガーされると:

1. お客様のサーバーへのログ配信を最大 3 回試行します。
2. 3 回目の試行でもお客様のサーバーに到達できなかった場合は、ログストリームの **稼働状況** ビューに表示されるエラーが記録されます。
3. 問題が解決するまで、エラーになった各ログについてこのプロセスを再開します。

7 日間連続でお客様のサーバーに到達できなかった場合、ストリームは自動的に一時停止されます。問題を解決した後、手動でストリームを再開する必要があります。

<div id="log-retention">
  ## ログの保持期間
</div>

Auth0 のログ保持期間は、[サブスクリプションレベル](/docs/ja/deploy-monitor/logs/log-data-retention)によって異なります。

サーバーで Auth0 から送信されたログを受信できない場合は、ストリームを削除してから、Starting Cursor フィールドを保持期間内の特定の日時に設定して再作成できます。

<div id="filtering-and-pii-obfuscation">
  ## フィルタリングとPIIの難読化
</div>

顧客は、自身のログについて、どの[フィルターカテゴリ](/docs/ja/deploy-monitor/logs/log-event-filters)をストリーミングし、どれを除外するかを選択できるため、外部の送信先には必要なデータのみを送信できます。さらに、ストリーミングされるログに含まれる個人を特定できる情報 (PII) をマスクまたはハッシュ化することもできます。

<div id="pii-obfuscation">
  ### PII の難読化
</div>

Auth0 では、テナントログの PII フィールドをサードパーティの destination にストリーミングする際に難読化する方法として、次の 2 つを提供しています。

* **マスキング:** フィールド値をアスタリスクに置き換えます
* **xxHash:** 高速な非暗号学的ハッシュアルゴリズム

ストリーミング時に、顧客は次のログデータとフィールド名を難読化できます。

| データ          | フィールド名                                                                      |
| ------------ | --------------------------------------------------------------------------- |
| `first_name` | `first_name`<br />`given_name`<br />`name`                                  |
| `last_name`  | `last_name`<br />`family_name`<br />`name`                                  |
| `email`      | `email`                                                                     |
| `phone`      | `phone`<br />`phone_number`<br />`old_phone_number`<br />`new_phone_number` |
| `address`    | `address`                                                                   |
| `username`   | `username`<br />`user_name`                                                 |
