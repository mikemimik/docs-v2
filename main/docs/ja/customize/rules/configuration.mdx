<Warning>
  ルールとフックの 提供終了 (EOL) は **2026年11月18日** です。また、**2023年10月16日** 以降に作成された新しいテナントでは、これらはすでに利用できません。アクティブなフックを使用している既存のテナントは、提供終了までフック製品へのアクセスを維持できます。

  Auth0 を拡張するには、Actions の利用を強くおすすめします。Actions では、豊富な型情報、インラインドキュメント、公開 `npm` パッケージを利用できるほか、拡張性をさらに高める外部連携にも接続できます。Actions でできることの詳細については、[Understand How Auth0 Actions Work](/docs/ja/customize/actions/actions-overview) をご覧ください。

  移行を支援するために、[ルールから Actions への移行](/docs/ja/customize/actions/migrate/migrate-from-rules-to-actions) と [フックから Actions への移行](/docs/ja/customize/actions/migrate/migrate-from-hooks-to-actions) に関するガイドを用意しています。さらに、機能比較、[Actions のデモ](https://www.youtube.com/watch?v=UesFSY1klrI)、および移行に役立つその他のリソースを紹介した専用の [Move to Actions](https://auth0.com/extensibility/movetoactions) ページもあります。

  ルールとフックの非推奨化について詳しくは、ブログ記事 [Preparing for Rules and Hooks 提供終了](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/) をお読みください。
</Warning>

グローバル `configuration` オブジェクトは、URL などのよく使う値を保存するためにルールで利用できます。認証情報や API キーなどの機密情報は、`configuration` オブジェクトに保存し、ルールのコードには含めないようにしてください。

<div id="configure-values">
  ## 値を設定する
</div>

Auth0 Dashboard の [ルール Settings](https://manage.auth0.com/#/rules/) で設定値を設定できます。

設定キーの値を編集または変更するには、既存の設定を削除して、更新した値に置き換えます。設定エリアを表示するには、少なくとも 1 つの Rule を作成しておく必要があります。作成していない場合は、代わりにルールのデモが表示されます。

<div id="use-the-configuration-object">
  ## `configuration` オブジェクトを使用する
</div>

設定した構成値には、ルールのコード内で `configuration` オブジェクトからキーを使ってアクセスできます。

```javascript lines
var MY_API_KEY = configuration.MY_API_KEY;
```

次の例は、新しいユーザーがサインアップしたときに Slack にメッセージを送信する Rule です。[Slack Webhook](https://api.slack.com/incoming-webhooks) は、キー `SLACK_HOOK_URL` に設定する `configuration` の値です。

```javascript lines
function (user, context, callback) {
  // ユーザーがすでにサインアップ済みか、リフレッシュトークンを使用している場合はスキップする
  if (context.stats.loginsCount > 1 || context.protocol === 'oauth2-refresh-token') {
    return callback(null, user, context);
  }

  // SlackのフックURLをhttps://slack.com/services/10525858050から取得する
  const SLACK_HOOK = configuration.SLACK_HOOK_URL;

  const slack = require('slack-notify')(SLACK_HOOK);
  const message = 'New User: ' + (user.name || user.email) + ' (' + user.email + ')';
  const channel = '#some_channel';

  slack.success({
    text: message,
    channel: channel
  });

  // Slack APIの呼び出し完了を待たずにすぐに返す（リクエストはサンドボックス上で処理が続行される）`
  callback(null, user, context);
}
```
