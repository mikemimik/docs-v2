テナントには、メールプロバイダーまたはご自身で運用しているメールサーバーの SMTP サーバー情報を設定できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  お使いのメールプロバイダーを当社が直接サポートしている場合は、より安全な認証プロトコルに対応しているため、SMTP サーバーの詳細ではなく、その連携を使用することをおすすめします。
</Callout>

<div id="smtp-server-requirements">
  ## SMTP サーバーの要件
</div>

SMTP サーバーは、次の要件を満たしている必要があります。

* LOGIN プロトコルによる [SMTP AUTH](https://en.wikipedia.org/wiki/SMTP_Authentication) をサポートしていること。
* [TLS 1.2](https://en.wikipedia.org/wiki/STARTTLS) 以降をサポートしていること。
* 公的な認証局 (CA) によって署名された証明書を使用していること。
* [Auth0 IP addresses](/docs/ja/secure/security-guidance/data-security/allowlist) からの接続を許可していること。

<div id="configure-tenants-with-an-smtp-server">
  ## SMTPサーバーを使用してテナントを設定する
</div>

Auth0 Dashboard または Management API を使用して、テナントに SMTP サーバーを設定できます。

<div id="using-the-auth0-dashboard">
  ### Auth0 Dashboard を使用する
</div>

Auth0 Dashboard を使用してテナントを SMTP サーバーで設定するには、次の手順に従います。

1. [Auth0 Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email-provider) に移動します。
2. **Use my own email provider** トグルを有効にします。
3. **SMTP** を選択します。
4. **From** のメールアドレスを入力し、続けて SMTP サーバーの **Host**、**Port**、**Username**、**Password** を入力します。

   利用可能であれば、現在の標準的な SMTP ポートであり TLS をサポートしているため、ポート 587 の使用を推奨します。プロバイダーによっては、古いポート 465 の使用が必要な場合もあります。元々の SMTP ポートであるポート 25 は、多くのプロバイダーや ISP でブロックされているため、使用は推奨されません。
5. **Save** をクリックします。

その後、**Send Test Email** ボタンをクリックしてテストメールを送信できます。数分待っても確認メールが届かない場合は、[Auth0 のログ](https://manage.auth0.com/#/logs) を確認して失敗が発生していないか確認してください。

<div id="using-the-management-api">
  ### Management API の使用
</div>

Management API を使用して SMTP サーバーでテナントを設定するには、`/api/v2/emails/provider` エンドポイントを使用します。

* 初めてメールプロバイダーを設定するには、[`POST` リクエストを `/api/v2/emails/provider` に送信します](/docs/ja/api/management/v2/emails/post-provider)。

* 既存のメールプロバイダーを更新するには、[`PATCH` リクエストを `/api/v2/emails/provider` に送信します](/docs/ja/api/management/v2/emails/patch-provider)。host、port、またはユーザー名を変更する場合は、[パスワードを指定する必要があります](/docs/ja/troubleshoot/product-lifecycle/deprecations-and-migrations#allow-omitting-password-on-smtp-email-provider-host-related-changes)。

<div id="additional-considerations">
  ## 追加の考慮事項
</div>

* LOGIN 以外の認証プロトコルのサポートが必要な場合は、[SMTP サーバーに接続する Action を作成](/docs/ja/customize/email/configure-a-custom-email-provider)できます。

* 2026 年 4 月に、Microsoft Exchange Online は [SMTH AUTH の基本認証を廃止する](https://techcommunity.microsoft.com/blog/exchange/exchange-online-to-retire-basic-auth-for-client-submission-smtp-auth/4114750)予定で、これにより `smtp.office365.com` と `smtp-legacy.office365.com` のエンドポイントが影響を受けます。

  [Microsoft Office 365 Exchange Online](/docs/ja/customize/email/smtp-email-providers/configure-365-exchange-as-smtp-email-provider) および [Azure Communication Services Email](/docs/ja/customize/email/smtp-email-providers/configure-azure-comm-service-as-smtp-email-provider) との連携を利用することをお勧めします。
