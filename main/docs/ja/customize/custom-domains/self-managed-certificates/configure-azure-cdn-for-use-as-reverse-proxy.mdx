title: リバースプロキシとして Azure CDN を設定する
---

<Card title="利用可否は Auth0 のプランによって異なります">
  この機能を利用できるかどうかは、Auth0 のプランまたは個別契約の内容によって異なります。詳しくは、[Pricing](https://auth0.com/pricing) をご覧ください。
</Card>

Azure CDN をリバースプロキシとして設定するには、Azure CDN Premium プランが必要です。

1. まだ設定していない場合は、[自己管理証明書を使用してカスタムドメインを設定する](/docs/ja/customize/custom-domains/self-managed-certificates) を実施します。後で必要になるため、**Origin Domain Name** と **cname-api-key** の値を控えておいてください。
2. [Azure Portal](https://portal.azure.com/) にログインします。
3. [新しい Azure CDN プロファイルとエンドポイントを作成](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint) します。CDN エンドポイントの設定では、Auth0 Dashboard の **Origin Domain Name** を指定してください。
4. [Azure CDN エンドポイントにカスタムドメインを追加](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup) します。
5. [Azure CDN のカスタムドメインに HTTPS を設定](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate) します。この手順では、ドメインの所有権を確認する必要があります。完了後、証明書がすべての CDN POP ロケーションにデプロイされるまで最大 6 時間かかることがあります。
6. Azure CDN Rules engine を使用して、Auth0 とカスタムドメイン間の通信を設定します。 (詳しくは、Microsoft ドキュメントの [Override HTTP behavior using the Azure CDN from Verizon Premium rules engine](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) を参照してください。) 次の設定で [新しい Azure CDN ルールを作成](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) します。

   | Setting          | Value                                         |
   | ---------------- | --------------------------------------------- |
   | Name/Description | Auth0 カスタムドメイン                                |
   | Type of requests | **Edge CName** オプションを選択し、一覧からカスタムドメイン名を選択します。 |
7. 次の **Features** を Azure CDN ルールに追加します。
   **azureedge.net** CNAME の使用を拒否するため、別の Azure CDN ルールを作成することをおすすめします。

   | Setting                      | Value                                                                          |
   | ---------------------------- | ------------------------------------------------------------------------------ |
   | Bypass Cache                 | **Enabled**                                                                    |
   | Modify Client Request Header | **Override** を選択し、名前に **cname-api-key**、値に Auth0 から提供された CNAME API Key を入力します。 |
8. Azure CDN ルールが承認されると、status は Pending XML から Active XML に変わります。この時点で、Azure CDN はルールと証明書の公開を開始します。Azure によるすべての変更の処理が完了すると、カスタムドメインを使用できるようになります。
