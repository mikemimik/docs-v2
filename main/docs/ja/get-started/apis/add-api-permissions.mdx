<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要な製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要な製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を使用して、API に権限を追加できます。

<Warning>
  デフォルトでは、どのアプリケーションのユーザーでも、ここで定義した任意の権限を要求できます。この動作を制限するには、[ルール](/docs/ja/customize/rules) を使用してアクセスポリシーを実装できます。
</Warning>

<div id="use-the-dashboard">
  ## Auth0 Dashboard を使用する
</div>

1. [Auth0 Dashboard &gt; アプリケーション &gt; APIs](https://manage.auth0.com/#/apis) に移動し、確認する API の名前をクリックします。

   <Frame>![Auth0 Dashboard アプリケーション APIs 一覧](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>
2. **Permissions** タブに移動し、追加する権限の名前と説明を入力します。予約済みの権限名は使用しないでください (「予約済みの名前」セクションを参照) 。

   <Frame>![Auth0 Dashboard API の権限を追加する API 権限を定義する画面](/docs/images/cdy7uua7fh8z/32Pb185OFs2mC6z2fmunEw/2708c52c2869d016066cc456dd00b6a2/dashboard-applications-apis-permissions.png)</Frame>
3. **Add** をクリックします。API と適切に連携するには、各アプリケーションで権限やスコープの更新が必要になる場合があることに注意してください。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  権限に空のオブジェクトを指定してパッチすると、権限は完全に削除されます。
</Callout>

[Update Resource Server エンドポイント](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id) に `PATCH` リクエストを送信します。`API_ID`、`MGMT_API_ACCESS_TOKEN`、`PERMISSION_NAME`、`PERMISSION_DESC` の各プレースホルダー値は、それぞれ実際の API ID、Management API の <Tooltip tip="Access Token: API へのアクセスに使用される、opaque な文字列または JWT 形式の Authorization 資格情報。" cta="用語集を見る" href="/docs/ja/glossary?term=Access+Token">Access Token</Tooltip>、権限名、権限の説明に置き換えてください。予約済みの権限名は使用しないようにしてください (予約済みの名前 セクションを参照) 。

<AuthCodeGroup>
  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/resource-servers/API_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "scopes": [ { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" }, { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" } ] }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/resource-servers/API_ID");
  var request = new RestRequest(Method.PATCH);
  request.AddHeader("content-type", "application/json");
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("cache-control", "no-cache");
  request.AddParameter("application/json", "{ "scopes": [ { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" }, { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" } ] }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"

  func main() {

  	url := "https://{yourDomain}/api/v2/resource-servers/API_ID"

  	payload := strings.NewReader("{ "scopes": [ { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" }, { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" } ] }")

  	req, _ := http.NewRequest("PATCH", url, payload)

  	req.Header.Add("content-type", "application/json")
  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("cache-control", "no-cache")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  スコープを追加または更新する場合、Management API には含めるすべてのスコープを渡す必要があります。既存のスコープのうち渡されなかったものは削除されます。
</Callout>

| 値                       | 説明                                                                                                                         |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `API_ID`                | 権限を削除する対象の API の ID。                                                                                                       |
| `MGMT_API_ACCESS_TOKEN` | スコープ `update:resource_servers` を持つ [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens)。 |
| `PERMISSION_NAME`       | 指定した API で保持したい権限の名前。                                                                                                      |
| `PERMISSION_DESC`       | 指定した API で保持したい権限のユーザー向け説明。                                                                                                |

<div id="reserved-names">
  ## 予約済みの名前
</div>

以下の権限名は予約済みであり、カスタム API の権限として設定することはできません。

* address
* created&#95;at
* email
* email&#95;verified
* family&#95;name
* given&#95;name
* identities
* name
* nickname
* offline&#95;access
* <Tooltip tip="OpenID: アプリケーションがログイン情報を収集・保存することなく、ユーザーの本人確認を行えるようにする認証のためのオープン標準です。" cta="用語集を見る" href="/docs/ja/glossary?term=openid">openid</Tooltip>
* phone
* picture
* profile
