<Warning>
  この機能は、Public Cloud Enterprise テナントでは早期アクセス、Private Cloud Enterprise テナントではベータ版として提供されています。この機能を使用すると、Okta の[Master Subscription Agreement](https://www.okta.com/legal/)に記載されている該当する Free Trial 条項に同意したものとみなされます。Auth0 のリリースステージの詳細については、[Product Release Stages](/docs/ja/troubleshoot/product-lifecycle/product-release-stages)を参照してください。
</Warning>

セキュリティポリシーを使用すると、[チームオーナー](/docs/ja/get-started/auth0-teams/team-member-management)は、インフラストラクチャシステムやアプリケーションへのアクセスに関して、組織の IT セキュリティポリシーに準拠した認証ルールを設定して適用できます。

<div id="available-enterprise-idp-connections">
  ## 利用可能なEnterprise IdP接続
</div>

Auth0 Teamsでは、チームメンバー向けにシングルサインオン (<Tooltip tip="Identity Provider（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=SSO">SSO</Tooltip>) を提供するため、<Tooltip tip="Identity Provider（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+provider">アイデンティティプロバイダー</Tooltip> (IdP) に接続できます。

<div id="add-sso-connection-beta">
  ### SSO 接続を追加する (ベータ)
</div>

Auth0 Teams Dashboard で SSO 接続を設定できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  SSO 接続を追加する前に、次の情報を用意する必要があります。

  * 使用する IdP プロバイダー (例: Okta、ADFS、Google Workspace)
  * IdP ドメイン
  * Auth0 Callback URL (`https://auth0.auth0.com/login/callback`)
  * Client ID、Client Secret、サインイン URL、署名証明書 (IdP によって異なります)
</Callout>

1. [**Security**](https://accounts.auth0.com/teams/#/security-policies) に移動します。
2. * **Add Connection (Beta)** を選択し、**Get Started** を選択します。
3. アイデンティティプロバイダーを選択し、**Next** を選択します。
4. 手順に従ってアプリケーションを作成し、**Next** を選択します。
5. 接続を設定し、**Create Connection** を選択します。
6. 内容を確認して、**Proceed** を選択します。
7. 手順に従ってユーザーとグループにアクセス権を付与し、**Next** を選択します。
8. **Test Connection** を選択して、接続が正しく設定されていることを確認します。
9. 準備ができたら、**Enable Connection** を選択します。

接続を有効にすると、**セキュリティポリシー** ページの **利用可能な Enterprise IdP 接続** セクションに表示されます。

<div id="configure-just-in-time-jit-provisioning">
  ### ジャストインタイム (JIT) プロビジョニングを設定する
</div>

JITプロビジョニングを使用すると、SSO 接続経由でログインしたチームメンバーのアカウントを Auth0 が自動的に作成します。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  JITプロビジョニングを設定するには、[Tenant Member Management](/docs/ja/get-started/auth0-teams/tenant-member-management) を有効にする必要があります。
</Callout>

1. [Security](https://accounts.auth0.com/teams/#/security-policies) に移動します。
2. **利用可能な Enterprise IdP 接続** セクションを見つけます。
3. 対象の接続で **JIT Membership** トグルを有効にします。

<div id="enforce-single-sign-on">
  ## シングルサインオンを必須にする
</div>

Auth0 Teams では、**利用可能な Enterprise IdP 接続**のいずれかを使用して Team メンバーがログインするよう必須にできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  SSO を必須にするには、SSO 接続経由でログインしており、**Team Owner** の role を持っている必要があります。

  SSO 接続経由でログインしていなくても、その接続が設定済みであれば、その接続で[自分自身を Auth0 Team に招待](/docs/ja/get-started/auth0-teams/team-member-management)できます。
</Callout>

<div id="use-jit-provisioning">
  ### JIT プロビジョニングを使用する
</div>

この方法を使うと、チームメンバーは SSO 接続を有効にした直後からログインでき、初回ログインに成功した時点で Auth0 がそのユーザーのアカウントを自動的に作成します。

1. [Tenant Member Management](/docs/ja/get-started/auth0-teams/tenant-member-management) を有効にします。
2. SSO 接続の **JIT Membership** トグルを有効にします。
3. Teams Dashboard の設定ページに移動し、チームの permalink 値を控えます。
4. すべてのチームメンバーに、Auth0 Teams Dashboard からログアウトし、SSO 接続経由でログインするよう案内します。チームメンバーが使用する URL は `https://accounts.auth0.com/teams/{team-permalink}` です。
5. Auth0 は各チームメンバーに対して、新しいアカウント (SSO 接続上) を自動的に作成します。
6. 各チームメンバーの新しいアカウントに、以前のアカウントと同じ [チームロール](/docs/ja/get-started/auth0-teams/team-member-management) を割り当てます。
7. (任意) 各チームメンバーの以前のアカウントを削除します。

<div id="manage-team-and-tenant-membership-manually">
  ### チームとテナントのメンバーシップを手動で管理する
</div>

この方法では、チームメンバーとテナントメンバーを別々に管理できます。

チームメンバーについては、[Teams Dashboard から新しい招待を送信し](/docs/ja/get-started/auth0-teams/team-member-management)、SSO 接続を使って招待を承諾するよう案内してください。

テナントメンバーについては、所属する各テナントごとに [Auth0 Dashboard から新しい招待を送信し](/docs/ja/get-started/manage-dashboard-access/add-dashboard-users)、SSO 接続を使って招待を承諾するよう案内してください。

招待を承諾すると、SSO を必須にしている、または Tenant Member Management を使用している Private Cloud Enterprise の顧客は、Private Cloud の<Tooltip tip="" cta="用語集を表示" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>で **Continue with Auth0 Teams** の login ボタンをクリックできます。

<div id="configure-home-realm-discovery-hrd">
  ### ホームレルム検出 (HRD) を設定する
</div>

HRD を有効にすると、Auth0 はチームメンバーが入力したメールアドレスのドメインを認識し、対応する SSO 接続にリダイレクトします。

<Warning>
  HRD を有効にする前に、チームメンバーの準備が整っていることを確認してください。

  Team Member Management と JIT プロビジョニングを使用している場合は、この新しい動作についてチームメンバーに知らせてください。

  チームのメンバーシップを手動で管理している場合は、すべてのチームメンバーが対応する SSO 接続にアカウントを持っており、ログイン方法を理解していることを確認してください。
</Warning>

1. Auth0 Support に[チケットを送信](https://support.auth0.com)します。
2. Auth0 Teams の SSO 接続で HRD を有効にしたいことを伝え、次の情報を共有します。

   * **Team Name** と **Team Permalink**
   * SSO 接続の名前
   * SSO 接続に関連付けられているドメイン
