<div id="endpoint">
  ## エンドポイント
</div>

`POST /oauth/token`

ユーザーがコールバック先にリダイレクトされたら、認可コードをアクセストークンや ID トークンに交換するために、`oauth/token` エンドポイントに `POST` リクエストを送信する必要があります。

<div id="remarks">
  ### 備考
</div>

`/oauth/token` エンドポイントを呼び出すには、次の要件を満たす必要があります。

* リクエストの Content-Type を `application/x-www-form-urlencoded` に設定する
* 渡すすべてのパラメータに `string` 型を使用する
* リクエストに、アプリケーション認証用の追加パラメータを含める (例: `client_secret`、または JSON Web Token Client Authentication の場合は `client_assertion` と `client_assertion_type`、あるいは Mutual TLS を使用する場合は `client-certificate` ヘッダーと `client-certificate-ca-verified` ヘッダーを渡す) 。

<div id="headers">
  ## ヘッダー
</div>

<ParamField header="DPoP" type="string">
  リクエスト用の DPoP 証明です。これは省略可能で、アプリケーションで Demonstrating Proof-of-Possession を使用している場合にのみ必要です。
</ParamField>

<div id="body-parameters">
  ## リクエストボディパラメータ
</div>

<ParamField body="grant_type" type="string" required>
  フローを指定します。有効な認可コードとともに `authorization_code` を使用してください。
</ParamField>

<ParamField body="code" type="string">
  最初の `/authorize` 呼び出しで取得した認可コードです。
</ParamField>

<ParamField body="client_id" type="string" required>
  アプリケーションの `client_id` です。必須です。
</ParamField>

<ParamField body="request_uri" type="string">
  `GET` `/oauth/par` エンドポイントで設定した場合にのみ必須です。
</ParamField>

<ParamField body="code_verifier" type="string">
  `code_challenge` の生成に使用する、暗号学的にランダムなキーです。`code_challenge` を指定した場合は使用を推奨します。
</ParamField>

<div id="response-schema">
  ## レスポンススキーマ
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    保護されたリソースへのアクセスに使用するアクセストークン。
  </ResponseField>

  <ResponseField name="refresh_token" type="string">
    新しいアクセストークンの取得に使用するトークン。
  </ResponseField>

  <ResponseField name="id_token" type="string">
    IDトークン。
  </ResponseField>

  <ResponseField name="token_type" type="string">
    トークンの種類。通常は `Bearer` です。
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    アクセストークンの有効期間 (秒) 。
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## レスポンスメッセージ
</div>

| ステータス | 説明               |
| ----- | ---------------- |
| 200   | トークンを含む正常なレスポンス。 |
