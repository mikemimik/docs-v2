<div id="endpoint">
  ## エンドポイント
</div>

`GET /wsfed/{clientId}`

このエンドポイントは、ログインを開始するための WS-Federation リクエストを受け付けます。

<div id="remarks">
  ### 備考
</div>

* `wtrealm` パラメーターは、以下のいずれかの形式である必要があります。
  * `urn:clientID` (例: `urn:${account.clientId}`) 
  * このパラメーターが urn で始まらない場合は、`client.clientAliases` 配列を使って検索されます。これは [/api/v2/clients](https://auth0.com/docs/api/management/v2/clients/get-clients) Management API でのみ設定できます。
* `whr` パラメーターは、`urn:CONNECTION_NAME` の形式で 接続 にマッピングされます。たとえば、`urn:google-oauth2` は Google でログインすることを示します。`whr` パラメーターが含まれていない場合、ユーザーは [Auth0 Login Page](https://auth0.com/docs/login_page) にリダイレクトされます。

<div id="path-parameters">
  ## パスパラメータ
</div>

<ParamField path="client-id" type="string" required>
  アプリケーションの `client-id`。
</ParamField>

<div id="query-parameters">
  ## クエリパラメーター
</div>

<ParamField query="wtrealm" type="string">
  `client-id` の代わりに使用できます。
</ParamField>

<ParamField query="whr" type="string">
  接続名 (ログインページをスキップするために使用) 。
</ParamField>

<ParamField query="wctx" type="string">
  アプリケーションの state。
</ParamField>

<ParamField query="wreply" type="string">
  コールバックURL。
</ParamField>
