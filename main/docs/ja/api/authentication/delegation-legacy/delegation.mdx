<div id="endpoint">
  ## エンドポイント
</div>

`POST /delegation`

<Note>
  デフォルトでは、この機能は、2017年6月8日時点でアドオンを使用していないテナントでは無効になっています。現在 委譲 を必要とするアドオンを使用している既存のテナントは、この機能を引き続き利用できます。今後、委譲 機能が変更されたりサービスから削除されたりする場合は、現在利用しているお客様に事前に通知し、移行のための十分な時間が提供されます。
</Note>

委譲 トークンは、アプリケーションが、呼び出し元プログラムと同じテナント内で Auth0 に登録・設定された Firebase や SAP などの Application Addon の API を呼び出す必要がある場合に、取得して使用できます。

既存のトークンがある場合、このエンドポイントは `target` アプリのシークレットで署名された新しいトークンを生成します。これは、ユーザーのIDをアプリケーションから API に渡すために使用されます。

<div id="remarks">
  ### 備考
</div>

* `profile` の scope 値は、エンドユーザーのデフォルトのプロフィール Claim (`name`、`family_name`、`given_name`、`middle_name`、`nickname`、`preferred_username`、`profile`、`picture`、`website`、`gender`、`birthdate`、`zoneinfo`、`locale`、`updated_at`) へのアクセスを要求します。

* `email` の scope 値は、`email` および `email_verified` Claim へのアクセスを要求します。

* [auth0.js](https://auth0.com/docs/libraries/auth0js) のバージョン 8 では、委譲は **サポートされていません**。ライブラリのバージョン 7 のサンプルについては、[auth0.js v7 reference guide](https://auth0.com/docs/libraries/auth0js/v7) を参照してください。

* この endpoint では、同じ IP アドレスから同じ `user_id` を使用するリクエストは、1 分あたり最大 10 件に制限されます。

* この endpoint は、rate limits に関する関連データを提供する 3 つの HTTP Response Header を返します。
  * `X-RateLimit-Limit`: 1 分あたりに許可されるリクエスト数。
  * `X-RateLimit-Remaining`: 利用可能なリクエスト数。新しいリクエストがあるたびに、この数は 1 減ります。1 分経過するごとにリクエスト枠が 1 件分戻るため、そのたびにこの数は 1 増えます。
  * `X-RateLimit-Reset`: rate limit (`X-RateLimit-Limit`) がリセットされるまでの残り時間。値は [UTC epoch seconds](https://en.wikipedia.org/wiki/Unix_time) です。
