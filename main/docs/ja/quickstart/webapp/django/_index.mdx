    Auth0を使用すると、アプリケーションに認証を追加し、ユーザープロファイル情報にアクセスできます。このガイドでは、[Authlib](https://authlib.org/) SDKを使用して、Auth0をPython [Django](https://www.djangoproject.com/)アプリケーションに統合する方法を説明します。

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションでは、
      開発中のプロジェクトで authentication をどのように機能させるかを設定します。

      ### アプリケーションを設定する

      対話型セレクターを使用して新しい Auth0 アプリケーションを作成するか、連携するプロジェクトに対応する既存のアプリケーションを選択します。Auth0 のすべてのアプリケーションには、英数字からなる一意の client ID
      が割り当てられており、アプリケーションのコードは SDK を通じて Auth0 API を呼び出す際にこれを使用します。

      この Quickstart で行った設定は、[Auth0 Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/) 内のアプリケーションに自動的に反映されます。今後はそこでアプリケーションを管理できます。

      設定全体を確認したい場合は、代わりにサンプルアプリケーションを参照することもできます。

      ### コールバック URL を設定する

      コールバック URL とは、ユーザーの認証完了後に Auth0 がリダイレクトする、アプリケーション内の URL のことです。これを設定しない場合、ユーザーはログイン後にアプリケーションへ戻されません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを
        `http://localhost:3000/callback` に設定してください。
      </Info>

      ### Logout URL を設定する

      ログアウト URL とは、ユーザーがログアウトした後に Auth0 がリダイレクトする、アプリケーション内の URL のことです。これを設定しない場合、ユーザーはアプリケーションからログアウトできず、エラーが表示されます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

      この連携では、Authlib などの依存ライブラリをいくつか追加します。プロジェクトの
      `requirements.txt` ファイルをプロジェクトのディレクトリに作成し、次の内容を記載します。

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      これらの依存関係を利用できるようにするには、シェルで次のコマンドを実行してください。

      ```text lines
      pip install -r requirements.txt
      ```

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      この例では、アプリケーションに 4 つのルート (`login`、`callback`、`logout`、`index`) を追加します。

      * `login` - アプリの訪問者が `/login` ルートにアクセスすると、Auth0 に移動して
        認証フローを開始します。
      * `callback` - ユーザーが Auth0 でのログインを完了すると、`/callback` ルートでアプリケーション
        に戻ります。このルートではユーザーのセッションを保存するため、再度アクセスしたときに
        もう一度ログインする必要がなくなります。
      * `logout` - `/logout` ルートは、ユーザーをアプリケーションからサインアウトさせます。このルートでは
        アプリ内のユーザーセッションを消去し、セッションが保持されないように Auth0 の logout endpoint にリダイレクトします。
        その後、アプリケーションはユーザーをホームルートにリダイレクトします。
      * `index` - ホームルートでは、認証済みユーザーの詳細をレンダリングするか、訪問者がサインインできるようにします。
