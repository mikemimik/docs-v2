Auth0 の Laravel SDK を使用すると、Laravel アプリケーションに認証、プロファイル管理、ルーティングのアクセス制御をすばやく追加できます。このガイドでは、新規または既存の Laravel 11 または 12 のアプリケーションに Auth0 を統合する方法を紹介します。アカウント用に設定されたサンプルを使ってこの Quickstart を進められるよう、あらかじめログインしておくことをおすすめします。

<div id="laravel-installation">
  ## Laravel のインストール
</div>

**まだ Laravel アプリケーションを用意していない場合**、新しいプロジェクト用の適切なディレクトリでシェルを開き、次のコマンドを実行します。

```bash lines
composer create-project laravel/laravel:^12 auth0-laravel-app
```

このガイドのコマンドはすべて、Laravelプロジェクトのルートディレクトリで実行することを前提としているため、新しいプロジェクトのディレクトリに `cd` しておいてください。

```bash lines
cd auth0-laravel-app
```

<div id="sdk-installation">
  ## SDK のインストール
</div>

[Auth0 Laravel SDK](https://github.com/auth0/laravel-auth0) をインストールするには、プロジェクトのディレクトリで次のコマンドを実行します。

```bash lines
composer require auth0/login:^7 --update-with-all-dependencies
```

次に、アプリケーション用のSDK設定ファイルを生成します：

```bash lines
php artisan vendor:publish --tag auth0
```

<div id="sdk-configuration">
  ## SDK の設定
</div>

プロジェクトのディレクトリで次のコマンドを実行して、[Auth0 CLI](https://github.com/auth0/auth0-cli) をダウンロードします。

```bash lines
curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
```

次に、Auth0 アカウントで CLI を認証し、画面が表示されたら &quot;as a user&quot; を選択します。

```bash lines
./auth0 login
```

次に、Auth0で新しいアプリケーションを作成します。

```bash lines
./auth0 apps create \
  --name "My Laravel Application" \
  --type "regular" \
  --auth-method "post" \
  --callbacks "http://localhost:8000/callback" \
  --logout-urls "http://localhost:8000" \
  --reveal-secrets \
  --no-input \
  --json > .auth0.app.json
```

新しいAPIも作成してください。

```bash lines
./auth0 apis create \
  --name "My Laravel Application's API" \
  --identifier "https://github.com/auth0/laravel-auth0" \
  --offline-access \
  --no-input \
  --json > .auth0.api.json
```

これにより、SDK を設定する 2 つのファイルがプロジェクトのディレクトリ内に生成されます。

これらのファイルには資格情報が含まれているため、機密情報として慎重に扱うことが重要です。これらをバージョン管理にコミットしないようにしてください。Git を使用している場合は、`.gitignore` ファイルに追加してください。

```bash lines
echo ".auth0.*.json" >> .gitignore
```

<div id="login-routes">
  ## ログインルート
</div>

SDK は、アプリケーションのユーザーが認証できるように、必要なすべてのルートを自動的に登録します。

| Route       | Purpose                |
| ----------- | ---------------------- |
| `/login`    | 認証フローを開始します。           |
| `/logout`   | ユーザーをログアウトします。         |
| `/callback` | Auth0 からのコールバックを処理します。 |

これらをより細かく制御したい場合や、アプリケーション内の既存のルートと競合する場合は、代わりに SDK のコントローラーを手動で登録することもできます。高度な連携については、[SDK の README](https://github.com/auth0/laravel-auth0)を参照してください。

<div id="access-control">
  ## アクセス制御
</div>

Laravel の認証機能では、各リクエストでユーザーをどのように認証するかを定義するために「ガード」を使用します。Auth0 SDK の認証ガードを使用すると、アプリケーションのルートへのアクセスを制限できます。

ルートへのアクセス前にユーザー認証を必須にするには、Laravel の `auth` ミドルウェアを使用できます:

```php lines
Route::get('/private', function () {
  return response('Welcome! You are logged in.');
})->middleware('auth');
```

これをLaravel の `can` ミドルウェアと組み合わせることで、認証済みユーザーに特定の[権限](https://auth0.com/docs/manage-users/access-control/rbac)を必須にすることもできます。

```php lines
Route::get('/scope', function () {
    return response('You have the `read:messages` permissions, and can therefore access this resource.');
})->middleware('auth')->can('read:messages');
```

<div id="user-information">
  ## ユーザー情報
</div>

認証済みユーザーに関する情報は、Laravel の `Auth` ファサードまたは `auth()` ヘルパー関数から利用できます。

たとえば、ユーザーの識別子とメールアドレスを取得するには、次のようにします。

```php lines
Route::get('/', function () {
  if (! auth()->check()) {
    return response('You are not logged in.');
  }

  $user = auth()->user();
  $name = $user->name ?? 'User';
  $email = $user->email ?? '';

  return response("Hello {$name}! Your email address is {$email}.");
});
```

<div id="user-management">
  ## User Management
</div>

[Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md) を使用して、ユーザー情報を更新できます。Management のすべてのエンドポイントには、SDK の `management()` メソッドからアクセスできます。

**Management API を呼び出す前に、アプリケーションが Management API と通信できるように設定する必要があります。** これは [Auth0 DashboardのAPIページ](https://manage.auth0.com/#/apis/) で `Auth0 Management API` を選択し、&#39;Machine to Machine Applications&#39; タブを開くことで行えます。Laravel アプリケーションを認可し、下向き矢印をクリックして、付与したいスコープを選択してください。

次の例では、ユーザーのメタデータを更新し、ランダムなお気に入りの色を設定します。そのため、`read:users` と `update:users` のスコープを付与してください。API エンドポイントの一覧と必要なスコープは、[Management API ドキュメント](https://auth0.com/docs/api/management/v2) で確認できます。

```php lines
use Auth0\Laravel\Facade\Auth0;

Route::get('/colors', function () {
  $endpoint = Auth0::management()->users();

  $colors = ['red', 'blue', 'green', 'black', 'white', 'yellow', 'purple', 'orange', 'pink', 'brown'];

  $endpoint->update(
    id: auth()->id(),
    body: [
        'user_metadata' => [
            'color' => $colors[random_int(0, count($colors) - 1)]
        ]
    ]
  );

  $metadata = $endpoint->get(auth()->id());
  $metadata = Auth0::json($metadata);

  $color = $metadata['user_metadata']['color'] ?? 'unknown';
  $name = auth()->user()->name;

  return response("Hello {$name}! Your favorite color is {$color}.");
})->middleware('auth');
```

SDK の Management API の全メソッドをまとめたクイックリファレンスガイドは、[こちら](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md)から参照できます。

<div id="run-the-application">
  ## アプリケーションを起動する
</div>

これで、Laravel アプリケーションを起動してリクエストを受け付ける準備が整いました。

```bash lines
php artisan serve
```

<div id="checkpoint">
  ### チェックポイント
</div>

ウェブブラウザーを開き、次のルートにアクセスしてみてください。

* `http://localhost:8000` にアクセスして、公開ルートを確認します。
* `http://localhost:8000/private` にアクセスして、認証を求められることを確認します。
* `http://localhost:8000` に再度アクセスして、認証済みの公開ルートを確認します。
* `http://localhost:8000/scope` にアクセスして、`read:messages` の[権限](https://auth0.com/docs/manage-users/access-control/rbac)があるか確認します。
* `http://localhost:8000/update` にアクセスして、ユーザーのプロファイルを更新します。
* `http://localhost:8000/logout` にアクセスして、ログアウトします。

問題が発生した場合は、次のことを試してください。

* `php artisan optimize:clear` を実行して、Laravel のキャッシュをクリアしてみてください。
* `.auth0.app.json` ファイルと `.auth0.api.json` ファイルが、プロジェクトのルートディレクトリにあることを確認してください。
* Laravel アプリケーションを Machine-to-Machine アプリケーションとして有効にし、[Auth0 Dashboard](https://manage.auth0.com/#/apis/) から `Auth0 Management API` に必要なスコープをすべて付与していることを確認してください。

問題が解決しませんか？ SDK の[ドキュメント](https://github.com/auth0/laravel-auth0)または[ドキュメントハブ](https://auth0.com/docs)を確認してください。また、[コミュニティ](https://community.auth0.com) にアクセスしてみるのもおすすめです。そこでは、私たちのチームや他のコミュニティメンバーが質問への回答を手伝ってくれます。
