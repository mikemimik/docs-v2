    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 は、PHP アプリに Auth0 の認証および認可を実装する作業を簡単にする [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワークリクエストを処理するために、[PSR-17](https://www.php-fig.org/psr/psr-17/) および [PSR-18](https://www.php-fig.org/psr/psr-18/) に準拠した HTTP ライブラリがインストールされている必要があります。利用可能なライブラリが
      ない場合は、ターミナルで次のコマンドを実行して、信頼できるライブラリをインストールできます。

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      次に、ターミナルで次のコマンドを実行して、Auth0 PHP SDK をインストールしてください。

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      SDK が正しく機能するように、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `strategy`: アプリケーションのユースケースに応じた SDK の動作を指定するための設定です。この
        場合は、これを定数

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
        に設定します。
      * domain: Auth0 テナントのドメインです。通常は、Auth0 Dashboard の アプリケーション の
        設定 にある Domain フィールドで確認できます。カスタムドメインを使用している場合は、代わりにこの
        値にカスタムドメインを設定してください。
      * clientId: この Quickstart の前の手順で設定した Auth0 アプリケーション の ID です。これは
        Auth0 Dashboard の アプリケーション の 設定 にある Client ID フィールドで確認できます。
      * clientSecret: この Quickstart の前の手順で作成した Auth0 アプリケーション のシークレットです。Client secret は
        Auth0 Dashboard の アプリケーション の 設定 にある Client Secret フィールドにあります。
      * audience: 上で登録した Auth0 API の識別子です。これは配列として指定する必要があります。

      次に、ベアラートークンを取得して処理できるよう、アプリケーションを拡張します。ベアラートークンは、クライアントがユーザーに代わってリクエストを送信する際にAPIへ渡されるアクセストークンです。アクセストークンは、アプリケーション内のルートへのアクセスを許可または拒否します。これはエンドポイント認可と呼ばれます。

      リクエストからアクセストークンを取得する最も簡単な方法は、PHP SDKの`getBearerToken()`
      メソッドを使用することです。このメソッドは、GETパラメータ、POSTボディ、リクエストヘッダー、その他のソースからトークンを取得します。この
      場合、PHP SDKは、GETリクエストの`token`パラメータ、またはHTTP
      `Authorization`ヘッダーから渡されたトークンを処理します。

      Auth0 application、Auth0 PHP SDK を設定し、アプリケーションがリクエストからベアラー
      トークンを取得できるようになったので、次のステップは、プロジェクトのエンドポイントの認可を設定することです。上で実装した
