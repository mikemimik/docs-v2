<div id="overview">
  ## 概要
</div>

次の両方に当てはまる場合:

* Rules を使用して、ユーザーのメールアドレスに基づいてスコープを割り当てている **かつ**
* アプリケーションで複数の接続を使用している

スコープが侵害される可能性があります。

<div id="how-this-works">
  ## 仕組み
</div>

Auth0 では、メールアドレスは接続ごとに一意である必要があります。ただし、アプリケーションごとに見ると、そのような制限はありません。

そのため、ユーザー A がある接続を使ってそのアプリケーションに登録し、ユーザー B が別の接続を使って同じメールアドレスでそのアプリケーションに登録することも可能です。

Rules でメールアドレスに基づいてユーザーにスコープを割り当てている場合、2 人目のユーザーは別人であるにもかかわらず、1 人目のユーザーと同じスコープを付与されてしまいます。

<div id="how-do-i-fix-this">
  ## これはどうすれば解決できますか？
</div>

最も簡単な対策は、ユーザーがサインアップ後、ログインを許可する前にメールアドレスを確認するよう必須にすることです。
