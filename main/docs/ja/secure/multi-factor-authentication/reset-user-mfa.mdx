title: ユーザーの多要素認証とリカバリーコードをリセットする
---

ユーザーがモバイルデバイスを紛失した場合は、リカバリーコードを使ってログインできます。リカバリーコードがない場合は、テナント管理者がそのユーザーの<Tooltip tip="多要素認証 (MFA): SMSで送信されるコードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/docs/ja/glossary?term=multi-factor+authentication">多要素認証</Tooltip> (MFA) をリセットする必要があります。この操作は、ユーザーのMFA登録を削除するのと同じです。ユーザーに関連付けられているMFA設定は削除されるため、次回ログイン時には新規ユーザーと同じようにMFAを設定できるようになります。エンドユーザーではなく管理者のMFAをリセットする場合は、[Auth0 Support](/docs/ja/troubleshoot/customer-support)にお問い合わせください。

<div id="use-the-dashboard">
  ## Auth0 Dashboard を使用する
</div>

1. [Auth0 Dashboard &gt; Users Management &gt; Users](https://manage.auth0.com/#/users) に移動します。
2. MFA をリセットするユーザーをクリックします。
3. 画面右上の **Actions** ボタンをクリックします。
4. ドロップダウンから **Reset Multi-factor** を選択します。ユーザーがすでに MFA に登録済みの場合、管理者には **User Details** ページの **Multi-Factor Authentication** タブの下部にある **Reset MFA** リンクも表示されます。どちらの方法でも同じように機能します。確認用のポップアップが表示されます。
5. ユーザーの MFA をリセットするには、**Yes, reset it** をクリックします。

次回ユーザーがログインすると、新規ユーザーと同様に MFA を設定する必要があります。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

管理者は、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を使用して、`DELETE` `/api/v2/users/{id}/authentication-methods` によりユーザーの MFA enrollment を削除することもできます。ユーザーに複数の enrollment がある場合は、それぞれに対してこの手順を繰り返す必要があります。

<div id="recovery-codes">
  ## リカバリーコード
</div>

ほとんどのMFA認証要素では、エンドユーザーにサインアップ時にリカバリーコードが提供されるため、これを控えて厳重に保管しておく必要があります。デバイスを手元で使えない場合や、何らかの理由で通常のMFA手順を一時的に利用できない場合、ユーザーはユーザー名とパスワードを入力した後、このコードを入力してログインできます。リカバリーコードが使用されると、その時点で新しいリカバリーコードが発行されます。

ユーザーがGuardianをアンインストールした後で再インストールした場合、リカバリーコードの入力を求められることがあります。リカバリーコードを紛失した場合は、Guardianバックアップの自動復元を無効にすることで、アプリを新規インストールできます。これを行うには、Guardianをアンインストールし、デバイス設定でバックアップの自動復元を一時的に無効にし (手順はデバイスによって異なります) 、その後アプリを再インストールする必要があります。次に、初回セットアップ時と同じように、アプリにMFAアカウントを追加する必要があります。ユーザーのデバイスで自動バックアップまたは自動復元が有効になっていない場合、アプリを再インストールしてもリカバリーコードの入力は求められず、初回設定時と同様にMFAアカウントを追加することになります。
