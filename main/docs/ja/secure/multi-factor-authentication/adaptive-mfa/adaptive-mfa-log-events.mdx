テナントログには、成功したログインイベントと失敗したログインイベントのエントリがあり、<Tooltip tip="Adaptive Multi-factor Authentication: ログイン試行の信頼度が低いと判断された場合にのみ、ユーザーに対してトリガーされる多要素認証 (MFA)。" cta="用語集を見る" href="/docs/ja/glossary?term=Adaptive+MFA">Adaptive MFA</Tooltip> のリスク評価スコアに関する情報も含まれます。これらは、ルールのコンテキストオブジェクトで使用できるエントリと同じです。Adaptive <Tooltip tip="Adaptive Multi-factor Authentication: ログイン試行の信頼度が低いと判断された場合にのみ、ユーザーに対してトリガーされる多要素認証 (MFA)。" cta="用語集を見る" href="/docs/ja/glossary?term=MFA">MFA</Tooltip> のエントリは、次の構造になっています。

```json lines expandable
{
  "date": "2020-06-26T15:12:43.654Z",
  "type": "s",

    "riskAssessment": {
      "confidence": "high",

        "UntrustedIP": {

    	},
    	"NewDevice": {
      	  "confidence": "medium",
      	  "code": "partial_match",

            "device": "unknown",

      	  "confidence": "high",
      	  "code": "minimal_travel_from_last_login"
    	}
      }

}
```

<div id="attack-protection-details">
  ## 攻撃対策の詳細
</div>

Auth0 は、ルールの実行情報に加えて、Adaptive MFA の <Tooltip tip="攻撃対策: Auth0 が攻撃を検出して緩和するために提供する機能で、総当たり攻撃対策、不審な IP スロットリング、漏えいパスワードの検知、ボット検出、Adaptive MFA が含まれます。" cta="用語集を表示" href="/docs/ja/glossary?term=attack+protection">攻撃対策</Tooltip> 情報もテナントログに含めます。ルールに表示されるものと同じデータ構造が、テナントログにも表示されます:

```json lines expandable
{
  "date": "2020-06-26T15:12:43.654Z",
  "type": "s",
  "details": {
	"anomalyDetection": {
  	  "confidence": "high",
  	  "assessments": {
    	    "UntrustedIP": {
      	    "confidence": "high",
      	    "reason": "ip not found",
      	    "code": "ip_not_found"
    	    },
    	    "NewDevice": {
      	    "confidence": "medium",
      	    "reason": "match useragent",
      	    "code": "match_useragent"
    	    },
    	    "ImpossibleTravel": {
      	    "confidence": "high",
      	    "reason": "minimal travel",
      	    "code": "minimal_travel_from_last_login"
    	    }
  	  }
	},
  },
  "description": "Successful login"
}
```
