Security Centerは、潜在的な攻撃の傾向を把握し、リアルタイムで迅速に対応できる可観測性ツールを提供します。Security Centerでは、顧客アイデンティティおよびアクセス管理 (CIAM) の異常検知メトリクスをリアルタイムで監視できるほか、同じ画面内で攻撃緩和機能を設定することもできます。

<div id="real-time-monitoring">
  ## リアルタイム監視
</div>

Security Center では、テナントのセキュリティ状況を概要で確認でき、Auth0 Management Dashboard 内から必要な対応を進められます。Security Center では、次のことができます。

* 総トラフィック数と脅威の総数を監視する
* 脅威の動向を把握する
* 脅威の動向に関連するアプリケーションを特定する
* ログインおよびサインアップのトラフィックを追跡する
* <Tooltip tip="Attack Protection: Auth0 が提供する、攻撃を検出して軽減するための機能です。これには、総当たり攻撃対策、不審な IP スロットリング、漏えいパスワードの検知、ボット検出、適応型多要素認証が含まれます。" cta="用語集を見る" href="/docs/ja/glossary?term=Attack+Protection">攻撃対策</Tooltip> と <Tooltip tip="Attack Protection: Auth0 が提供する、攻撃を検出して軽減するための機能です。これには、総当たり攻撃対策、不審な IP スロットリング、漏えいパスワードの検知、ボット検出、適応型多要素認証が含まれます。" cta="用語集を見る" href="/docs/ja/glossary?term=MFA">MFA</Tooltip> 機能によって検出された脅威を監視する

<div id="filtering-and-aggregation">
  ### フィルタリングと集計
</div>

Security Center では、必要に応じて利用可能なデータを絞り込めます。

次の項目に基づいてデータをフィルタリングできます。

* 期間 (直近14日間まで)
* アプリケーション
* 接続

選択した期間に応じて、データは1分ごと、1時間ごと、または1日ごとに自動集計されます。

<div id="threat-behavior-trends">
  ### 脅威動向のトレンド
</div>

Security Center では、次の脅威タイプについて脅威動向のトレンドを確認できます。

* **クレデンシャルスタッフィング**: ユーザーアカウントの侵害を目的として、資格情報を送信しようとする自動化された試行が関与しているとみられる行動パターン。
* **サインアップ攻撃**: 新しいユーザーアカウントの作成を目的とした、自動化された試行が関与しているとみられる行動パターン。
* **MFA回避**: ユーザーの多要素認証 (MFA) による保護を回避することを目的とした、自動化された試行が関与しているとみられる行動パターン。

ビューを使うと、脅威タイプごとにデータを絞り込み、脅威動向のトレンドに関連するアプリケーションを特定できます。

<div id="authentication-events">
  ### 認証イベント
</div>

Security Center では、ログイン試行やサインアップ試行を含む認証イベントを確認できます。

<div id="login-attempts">
  #### ログイン試行
</div>

ログイン試行には、過去7日間の成功および失敗したログイントランザクションが含まれます。

<div id="signup-attempts">
  #### サインアップ試行
</div>

サインアップ試行には、過去7日間の成功した Signup トランザクションと失敗した Signup トランザクションの両方が含まれます。

<div id="attack-protection-and-mfa-monitoring">
  ### 攻撃対策とMFAの監視
</div>

Security Center は、攻撃対策 と MFA 機能によって検出された現在の攻撃傾向を把握するのに役立ちます。また、これらの機能を有効にして設定することで、対策を講じることもできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  攻撃対策 機能の設定時に対策を選択したかどうかにかかわらず、既知の攻撃タイプやログ関連イベントの兆候となるパターンを特定します。
</Callout>

<Frame>![スクリーンショット: Security Center の Threat Protection タブに複数の折れ線グラフが表示されています。](/docs/images/cdy7uua7fh8z/Threat_Monitoring/Threat_Monitoring.png)</Frame>

<div id="bot-detection">
  #### ボット検出
</div>

<Tooltip tip="ボット検出: Auth0 がログインプロセス中に CAPTCHA を有効にして、ボットと疑われるトラフィックをブロックする攻撃対策の一種です。" cta="用語集を見る" href="/docs/ja/glossary?term=Bot+detection">ボット検出</Tooltip> は、request がボットによるものである可能性が高い場合を検出し、スクリプトによる攻撃を軽減します。ボット検出には、過去 7 日間に検出されたボットの数が含まれます。

この機能の詳細については、[ボット検出](/docs/ja/secure/attack-protection/bot-detection) を参照してください。

<div id="suspicious-ip-throttling">
  #### 不審な IP スロットリング
</div>

<Tooltip tip="不審な IP スロットリング: 単一の IP アドレスから多数のアカウントを狙う不審なログインからテナントを保護する攻撃対策の一種です。" cta="用語集を見る" href="/docs/ja/glossary?term=Suspicious+IP+throttling">不審な IP スロットリング</Tooltip> は、短時間に過剰な数のログインまたはサインアップを試行する IP アドレスからのトラフィックをブロックします。不審な IP スロットリングには、過去 7 日間にブロックされた不審な IP の数も含まれます。

この機能の詳細については、[不審な IP スロットリング](/docs/ja/secure/attack-protection/suspicious-ip-throttling) を参照してください。

<div id="brute-force-protection">
  #### 総当たり攻撃対策
</div>

<Tooltip tip="総当たり攻撃対策: 単一のIPアドレスから発生し、単一のユーザーアカウントを標的とする総当たり攻撃を防ぐ攻撃対策の一形態。" cta="用語集を見る" href="/docs/ja/glossary?term=Brute-force+protection">総当たり攻撃対策</Tooltip> は、単一のIPアドレスによる単一のユーザーアカウントへの攻撃を防ぎます。総当たり攻撃対策には、過去7日間にブロックされた総当たり攻撃の試行回数が含まれます。

この機能の詳細については、[Brute-Force Protection](/docs/ja/secure/attack-protection/brute-force-protection) をご覧ください。

<div id="breached-password-detection">
  #### 漏えいパスワードの検知
</div>

<Tooltip tip="漏えいパスワードの検知: ユーザー名/パスワードの組み合わせがサードパーティのウェブサイトまたはアプリで発生したデータ漏えいにより漏えいしている場合に、Auth0 がユーザーに通知する攻撃対策の一種です。" cta="用語集を見る" href="/docs/ja/glossary?term=Breached+password+detection">漏えいパスワードの検知</Tooltip> は、盗まれた資格情報を使ってサインアップやログインを試みる <Tooltip tip="漏えいパスワードの検知: ユーザー名/パスワードの組み合わせがサードパーティのウェブサイトまたはアプリで発生したデータ漏えいにより漏えいしている場合に、Auth0 がユーザーに通知する攻撃対策の一種です。" cta="用語集を見る" href="/docs/ja/glossary?term=bad+actors">悪意のある攻撃者</Tooltip> からアプリケーションを保護します。漏えいパスワードの検知には、過去7日間のログインフローとサインアップフローで検出された漏えい資格情報の数が含まれます。

この機能の詳細については、[漏えいパスワードの検知](/docs/ja/secure/attack-protection/breached-password-detection) を参照してください。

<div id="multi-factor-authentication">
  #### 多要素認証
</div>

多要素認証 (MFA) では、複数の方法でユーザー確認を行うことで、ユーザーを認証します。MFA には、検出された MFA チャレンジの数と、過去 7 日間に成功または失敗した MFA チャレンジの数が含まれます。

この機能の詳細については、[多要素認証](/docs/ja/secure/multi-factor-authentication)を参照してください。
