title: JSON Web Key Sets
---

JSON Web Key Set (JWKS) は、<Tooltip tip="JSON Web Token (JWT): 2者間でクレームを安全にやり取りするために使用される標準的な ID トークン形式（多くの場合、アクセストークン形式としても使用）。" cta="用語集を見る" href="/docs/ja/glossary?term=JSON+Web+Token+%28JWT%29">JSON Web Token (JWT)</Tooltip> を検証するための公開キーを含むキーセットです。これらの JWT は <Tooltip tip="Authorization Server: ユーザーを認証し、アプリケーションに認可データやトークンを発行するサーバーコンポーネント。" cta="用語集を見る" href="/docs/ja/glossary?term=Authorization+Server">Authorization Server</Tooltip> によって発行され、RS256 [signing algorithm](/docs/ja/get-started/applications/signing-algorithms) を使用して署名されます。

Auth0 でアプリケーションと API を作成する際、<Tooltip tip="JSON Web Token (JWT): 2者間でクレームを安全にやり取りするために使用される標準的な ID トークン形式（多くの場合、アクセストークン形式としても使用）。" cta="用語集を見る" href="/docs/ja/glossary?term=JWTs">JWTs</Tooltip> の署名でサポートされている algorithms は **RS256** と **HS256** の 2 つです。RS256 は非対称署名を生成するため、JWT の署名には秘密キーを使用し、署名の検証には別の公開キーを使用する必要があります。

Auth0 では、RS256 トークンの署名に使用される暗号キーを表すために、[JSON Web Key (JWK) specification](https://tools.ietf.org/html/rfc7517) を使用しています。この仕様では、2 つの高水準データ構造、**JSON Web Key (JWK)** と **JSON Web Key Set (JWKS)** を定義しています。以下は、この仕様における定義です。

| 項目                          | 説明                                                                    |
| --------------------------- | --------------------------------------------------------------------- |
| **JSON Web Key (JWK)**      | 暗号キーを表す JSON オブジェクト。オブジェクトのメンバーは、その値を含むキーのプロパティを表します。                 |
| **JSON Web Key Set (JWKS)** | JWK のセットを表す JSON オブジェクト。この JSON オブジェクトには、JWK の配列である `keys` メンバーが必須です。 |

Auth0 はテナントごとに JWKS エンドポイントを公開しており、その場所は `https://{yourDomain}/.well-known/jwks.json` です。このエンドポイントには、このテナント向けに Auth0 が発行するすべての JWT を検証するための JWK が含まれます。

<Warning>
  現在、Auth0 が一度に署名に使用する JWK は 1 つだけです。ただし、このエンドポイントには複数の JWK が含まれる可能性があることを前提にしておくことが重要です。たとえば、[アプリケーションの署名鍵をローテーションする](/docs/ja/get-started/tenant-settings/signing-keys/rotate-signing-keys) と、JWKS に複数のキーが含まれることがあります。
</Warning>
