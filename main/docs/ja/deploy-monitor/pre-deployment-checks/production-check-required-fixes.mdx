---
description: 本番環境へのデプロイ前に、Auth0 Application に対して必ず実施すべき修正
title: 'Production Readiness Checks: 重大な修正'
---

以下のチェックでは、本番環境へのデプロイを正常に行うために必要な要件を、すべて満たしているかどうかを確認します。

チェックの出力の詳細については、[Run Production Checks](/docs/ja/deploy-monitor/pre-deployment-checks/how-to-run-production-checks) を参照してください。

| チェック項目                                                                                                                                                            | 説明                                                                                                                                                                                                                                                                                      |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| すべてのActionsが[推奨されるバージョンのNodeJS](/docs/ja/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)で実行されている                                      | すべての[Actions](/docs/ja/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)を推奨されるバージョンのNodeJSに設定し、セキュリティパッチや新機能を活用してください。                                                                                                                                        |
| [Management API v2 の認証で ID Token を許可する](/docs/ja/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) が無効になっている                 | Management API の Users および Device Credentials の一部のエンドポイントで、認可に ID Token を使用する機能は非推奨化されています。アクセストークンへの移行が完了したら、[Management API v2 認証で ID Token を許可](https://manage.auth0.com/#/tenant/advanced) のトグルがオフになっていることを確認してください。この設定が表示されない場合は、この機能が非推奨化された後にテナントが作成されているため、デフォルトですでに無効になっています。 |
| allowLegacyRoGrantTypes                                                                                                                                           | tenant migration settings のレガシーな [oauth/ro endpoint](/docs/ja/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) は非推奨となっています。アプリケーションをサポート対象のエンドポイントを使用するように更新したら、このエンドポイントを必ずオフにしてください。                                                                         |
| allowOtherLegacyGrantTypes                                                                                                                                        | テナント移行設定にある従来の[oauth/access&#95;token エンドポイント](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token)は廃止予定です。アプリケーションを更新してサポート対象のエンドポイントを使用するようにしたら、このエンドポイントを必ずオフにしてください。                                                                          |
| allowLegacyTokenInfoEndpoint                                                                                                                                      | テナントの移行設定にある従来の[tokeninfo endpoint](/docs/ja/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation)は非推奨となる予定です。アプリケーションをサポート対象のエンドポイントを使用するよう更新したら、このエンドポイントを必ずオフにしてください。                                                                                           |
| allowLegacyDelegationGrantTypes                                                                                                                                   | テナントの移行設定にある従来の[delegation endpoint](/docs/ja/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation)は非推奨となる予定です。アプリケーションを更新してサポート対象のエンドポイントを使用するようにしたら、このエンドポイントは必ず無効にしてください。                                                                                         |
| [Allowed Callback URLs](/docs/ja/authenticate/login/redirect-users-after-login) に Localhost を含めないこと                                                                  | [アプリケーションの Allowed Callback URLs](https://manage.auth0.com/#/applications) に `localhost` や `127.0.0.1` などが含まれていないことを確認します。                                                                                                                                                              |
| [許可されたオリジン (CORS) ](/docs/ja/authenticate/login/cross-origin-authentication) は Localhost であってはならない                                                                   | ページの[Location URL](https://manage.auth0.com/#/applications)が`localhost`を指していないことを確認します。                                                                                                                                                                                                 |
| [許可された Web オリジン](/docs/ja/get-started/applications/application-settings) に `localhost` が含まれていないこと                                                                    | [Allowed Web Origins URLs](https://manage.auth0.com/#/applications) が `localhost` を指していないことを確認します。                                                                                                                                                                                      |
| [メールプロバイダー](/docs/ja/customize/email/smtp-email-providers) が設定済みであること                                                                                                | [カスタムメールプロバイダー](https://manage.auth0.com/#/emails/provider)が設定済みであることを確認します。                                                                                                                                                                                                            |
| Hooks は非推奨となるため、[Actions へ移行](/docs/ja/customize/actions/migrate/migrate-from-hooks-to-actions)する必要があります                                                             | Hooks は非推奨となる予定です。[Actions](/docs/ja/customize/actions/migrate/migrate-from-hooks-to-actions) の利用を開始し、移行を進めることを強くお勧めします。既存の Hooks は 2024 年 11 月 18 日まで引き続き有効です。                                                                                                                            |
| [多要素認証](/docs/ja/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) を設定しておく必要があります (前提条件: MFA が設定済みであること) | MFA で [Phone Message factor](https://manage.auth0.com/#/mfa) を使用している場合は、デフォルトの配信プロバイダーのレート制限を避けるため、Twilio またはカスタム配信プロバイダーを使用するよう設定してください。                                                                                                                                               |
| [Legacy User Profile](/docs/ja/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) を無効にする                                   | ID Token と `/userinfo` エンドポイントに完全なユーザープロファイルを含められる従来の認証フローは、非推奨になっています。新しい OIDC 準拠 API への移行が完了したら、[Legacy User Profile](https://manage.auth0.com/#/account/advanced) のトグルがオフになっていることを確認してください。                                                                                          |
| Rules は非推奨となるため、[Actions への移行が必要です](/docs/ja/customize/actions/migrate/migrate-from-rules-to-actions)                                                                | Rules は非推奨となる予定です。[Actions](/docs/ja/customize/actions/migrate/migrate-from-rules-to-actions) の利用を開始し、移行プロセスに着手することを強くお勧めします。既存の Rules は 2024 年 11 月 18 日まで引き続き有効です。                                                                                                                       |
| [ソーシャル接続](/docs/ja/authenticate/identity-providers/social-identity-providers/devkeys) で Auth0 Developer Keys を使用していないこと                                              | [Social Connections](https://manage.auth0.com/#/connections/social) でデフォルトの Auth0 開発者キーが使用されていないことを確認します。                                                                                                                                                                               |
| サポートメールが設定されていること                                                                                                                                                 | テナント設定で [Support Email](https://manage.auth0.com/#/account) が設定されていることを確認します。                                                                                                                                                                                                           |
| サポートURLが設定されていること                                                                                                                                                 | Tenant Settingsで[Support URL](https://manage.auth0.com/#/account)が設定されていることを確認します。                                                                                                                                                                                                      |
| テナント環境タグが設定されていること                                                                                                                                                | [テナント環境タグ](https://support.auth0.com/tenants/public)が、Production、Staging、またはDevelopmentに適切に設定されていることを確認します。このタグを変更すると、テナントのレート制限に影響します。                                                                                                                                                 |
| テナントで推奨デフォルトの[NodeJS バージョン](/docs/ja/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22)が使用されるよう設定されている                                    | セキュリティパッチや新機能を利用するため、テナントのNodeJSを[推奨されるバージョン](/docs/ja/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)に更新してください。                                                                                                                                            |
| テナントログインURI                                                                                                                                                       | Tenant Settingsで [Tenant Login URI](https://manage.auth0.com/#/tenant/advanced) が設定されていることを確認します。                                                                                                                                                                                       |
| ブランドメールテンプレートでカスタムドメインを使用する                                                                                                                                       | メールテンプレートで[カスタムドメイン](/docs/ja/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails)を使用するようにしてください。ユーザー向けのすべてのメールについて、カスタムドメインと自社のブランドを反映するようカスタマイズすることを強くお勧めします。                                                                                 |
