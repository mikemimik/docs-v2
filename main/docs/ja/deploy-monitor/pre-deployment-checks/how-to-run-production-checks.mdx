title: 本番環境準備チェックを実行する
---

本番環境をデプロイする前に、テナントの本番チェック結果を確認してください。

1. Auth0 Management Dashboard に移動し、アカウントにログインします。
2. ドロップダウンから、本番チェック を実行するテナントを選択します
3. 同じドロップダウンで **Run Readiness Check** をクリックして、インターフェースを開きます

<Frame>![](/docs/images/cdy7uua7fh8z/nHS8TrcMAsgUUWOv5gcNO/66d641136222fe6634b1d53f0a455ffd/Screenshot_2023-06-02_at_10.53.40_AM.jpeg)</Frame><Frame>![デプロイ前の Auth0 Support 本番チェック](/docs/images/cdy7uua7fh8z/4EjetzpVocGQKy6HYVTHMw/a4a4bfb85e8db8aaf393a7e97aeb8bb4/2025-01-28_15-33-08_copy.png)</Frame>

任意:

1. **Filters** をクリックし、このテナントに関連付けられている [アプリケーション](/docs/ja/get-started/applications) のうち、チェックを実行するものを 1 つ以上選択します。
2. **Apply** をクリックします。

テストが完了すると、画面が自動的に更新され、チェック結果が表示されます。

<div id="production-check-results">
  ## 本番環境チェックの結果
</div>

チェック結果の修正には、次の3つのカテゴリがあります。

* [重要な修正](/docs/ja/deploy-monitor/pre-deployment-checks/production-check-required-fixes)
* [重要度の低い修正](/docs/ja/deploy-monitor/pre-deployment-checks/production-check-recommended-fixes)
* [ベストプラクティス](/docs/ja/deploy-monitor/pre-deployment-checks/production-checks-best-practices)

チェック結果として表示される可能性があるのは、次のとおりです。

* 合格
* 失敗
* ベストプラクティス

各結果セットには、アプリケーションが合格したチェックの数と、確認が必要なチェックの数が Auth0 によって表示されます。

アプリケーションが1つ以上のチェックで**失敗**した場合は、次の内容が表示されます。

* チェック名
* チェックで確認される内容に関する情報
* アプリケーションがそのチェックに合格できるよう、必要な修正を行うための適切な場所への **Configure** リンク
* 対応するドキュメントへのハイパーリンク

アプリケーションが**合格**したチェックはすべて、結果セットの passed オブジェクトにまとめられます。チェック名やその内容を確認できるほか、関連ドキュメントを参照したり、必要に応じて変更を行える対応する設定画面へハイパーリンクから移動したりできます。
