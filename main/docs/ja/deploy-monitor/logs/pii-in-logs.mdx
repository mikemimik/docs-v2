description: Auth0がテナントログに書き込む可能性のある個人を特定できる情報（PII）について説明します。
title: Auth0ログにおける個人を特定できる情報
---

Auth0のテナントログ内の多くのログには、個人を特定できる情報 (PII) が含まれる場合があります。認証イベントによって生成されるログエントリには、ユーザーのPIIが含まれることがあります。また、Auth0の<Tooltip tip="Management API: お客様が管理作業を実行できるようにする製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip>やDashboardを使用してユーザーを追加した場合、Auth0はそのユーザーアカウントの詳細をログに記録します。

ログエントリ内の以下のフィールドには、PIIが含まれる場合があります。

* 氏名
* 電話番号 (<Tooltip tip="多要素認証（MFA）: SMSで送信されるコードなど、ユーザー名とパスワードに加えて別の要素を使用するユーザー認証プロセスです。" cta="用語集を見る" href="/docs/ja/glossary?term=multi-factor+authentication">多要素認証</Tooltip>に使用)
* メールアドレス
* IPアドレス (所在地が推測される可能性があります)
* ユーザーに対して定義した任意のカスタム情報

Auth0は、Auth0または任意の<Tooltip tip="Access Token: APIへのアクセスに使用される、JWTまたは不透明な文字列形式の認可資格情報です。" cta="用語集を見る" href="/docs/ja/glossary?term=access+tokens">access tokens</Tooltip>を<Tooltip tip="Access Token: APIへのアクセスに使用される、JWTまたは不透明な文字列形式の認可資格情報です。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+provider">identity provider</Tooltip>から取得した場合でも、それらがログに記録されることはありません。認可コードの交換が行われた場合、ログにはコードの一部のみが表示されます (例: `code: 31XXXXX`) 。
