[ロールベースのアクセス制御 (RBAC)](/docs/ja/manage-users/access-control/rbac) は、<Tooltip tip="Auth0 Dashboard: サービスを構成するための Auth0 の主要製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを構成するための Auth0 の主要製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を使用して有効にできます。これにより、API Authorization Core の機能セットが有効になります。

RBAC を有効にすると、<Tooltip tip="アクセストークン: API へのアクセスに使用される、opaque な文字列または JWT 形式の Authorization 資格情報です。" cta="用語集を見る" href="/docs/ja/glossary?term=access+token">アクセストークン</Tooltip> の `scope` claim には、権限がアクセストークンに含まれているかどうかにかかわらず、要求された権限とユーザーに割り当てられた権限の積集合が含まれます。RBAC を無効にすると、アプリケーションは API に定義された任意の権限を要求でき、`scope` claim には要求されたすべての権限が含まれます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  アクセストークンの scope を変更する [Actions](/docs/ja/customize/actions) を設定している場合、それらによって RBAC で設定された scope は上書きされます。
</Callout>

<div id="dashboard">
  ## Auth0 Dashboard
</div>

1. [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) に移動し、表示したい API の名前をクリックします。

   <Frame>![Dashboard Applications APIs List](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>
2. **RBAC Settings** までスクロールし、**Enable RBAC** トグルを有効にします。

   <Frame>![Auth0 Dashboard API Settings RBAC toggle](/docs/images/cdy7uua7fh8z/65tKb6aj0ktc2qXLUVlV3e/641adef615d6af9e5a3b588ff397af87/dashboard-apis-edit_view-settings_rbac-settings.png)</Frame>
3. ユーザーに割り当てられているすべての権限をアクセストークンの `permissions` クレームに含めるには、**Add Permissions in the Access Token** トグルを有効にして、**Save** をクリックします。アクセストークンに権限を含めると、権限を取得するための呼び出しを最小限に抑えられますが、その分トークンサイズは大きくなります。
   **Add Permissions in the Access Token** トグルを有効にすると、Auth0 は API に設定されている[アクセストークンプロファイル](/docs/ja/secure/tokens/access-tokens/access-token-profiles)に基づいて、トークンダイアレクトも更新します。

   * トークンダイアレクトが `access_token` の場合、Auth0 は `access_token_authz` に更新します。これは、`permissions` クレームを含む `access_token` プロファイルと同等です。
   * トークンダイアレクトが `rfc9068_profile` の場合、Auth0 は `rfc9068_profile_authz` に更新します。これは、`permissions` クレームを含む `rfc9068_profile` と同等です。

   使用可能なトークンダイアレクトの詳細については、[Token dialect options](#token-dialect-options) を参照してください。

<div id="management-api">
  ## Management API
</div>

Management API を使用して RBAC を有効にするには、[リソースサーバーを更新するエンドポイント](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id) に PATCH リクエストを送信します。PATCH リクエストで、`enforce_policies` を `true` に設定します：

<AuthCodeGroup>
  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/resource-servers/API_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "enforce_policies": "true", "token_dialect": "TOKEN_DIALECT" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/resource-servers/API_ID");
  var request = new RestRequest(Method.PATCH);
  request.AddHeader("content-type", "application/json");
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("cache-control", "no-cache");
  request.AddParameter("application/json", "{ "enforce_policies": "true", "token_dialect": "TOKEN_DIALECT" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/resource-servers/API_ID"

  	payload := strings.NewReader("{ "enforce_policies": "true", "token_dialect": "TOKEN_DIALECT" }")

  	req, _ := http.NewRequest("PATCH", url, payload)

  	req.Header.Add("content-type", "application/json")
  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("cache-control", "no-cache")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

`API_ID`、`MGMT_API_ACCESS_TOKEN`、`TOKEN_DIALECT` は、以下の表に示す対応する値に置き換えてください。

| パラメーター                  | 説明                                                                                                                             |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `API_ID`                | RBAC を有効にする対象 API の ID。                                                                                                        |
| `MGMT_API_ACCESS_TOKEN` | scope `update:resource_servers` が付与された [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens)。 |
| `TOKEN_DIALECT`         | 指定した API のアクセストークンのトークンダイアレクト。詳しくは、[Token dialect options](#token-dialect-options) を参照してください。                                  |

<div id="token-dialect-options">
  ### トークンダイアレクトのオプション
</div>

Auth0 は次のトークンダイアレクトをサポートしています。

| Value                   | Description                                                                                                                                                                                                                                                                                                                                           |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `access_token`          | Auth0 のデフォルトのトークンプロファイルは、[JSON Web トークン (JWT)](/docs/ja/secure/tokens/json-web-tokens) 形式のアクセストークンを生成します。アクセストークンの `scope` claim には、リクエストされた権限とユーザーに割り当てられた権限の積集合が含まれます。`permissions` claim は含まれません。詳しくは、[Access Token Profiles](/docs/ja/secure/tokens/access-tokens/access-token-profiles) を参照してください。                                                     |
| `access_token_authz`    | `permissions` claim を含む Auth0 のデフォルトのトークンプロファイル (`access_token`) です。アクセストークンの `scope` claim には、リクエストされた権限とユーザーに割り当てられた権限の積集合が含まれます。アクセストークンの `permissions` claim には、ユーザーに割り当てられたすべての権限が含まれます。これにより、権限を取得するための呼び出し回数を最小限に抑えられますが、トークンサイズは大きくなります。                                                                                                      |
| `rfc9068_profile`       | RFC 9068 トークンプロファイルは、[IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068) に準拠した JWT 形式のアクセストークンを生成します。アクセストークンの `scope` claim には、リクエストされた権限とユーザーに割り当てられた権限の積集合が含まれます。`permissions` claim は含まれません。詳しくは、[Access Token Profiles](/docs/ja/secure/tokens/access-tokens/access-token-profiles) を参照してください。 |
| `rfc9068_profile_authz` | `permissions` claim を含む RFC 9068 トークンプロファイル (`rfc9068_profile`) です。アクセストークンの `scope` claim には、リクエストされた権限とユーザーに割り当てられた権限の積集合が含まれます。アクセストークンの `permissions` claim には、ユーザーに割り当てられたすべての権限が含まれます。これにより、権限を取得するための呼び出し回数を最小限に抑えられますが、トークンサイズは大きくなります。                                                                                                       |
