Auth0&#39;s Dashboard または <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を使用して、[ロール](/docs/ja/manage-users/access-control/rbac)を作成できます。これらのロールは、API Authorization Core の機能セットで利用できます。

<div id="prerequisites">
  ## 前提条件
</div>

* ロールベースのアクセス制御 (RBAC) を正しく機能させるには、Dashboard または Management API を使用して、API で RBAC を有効にする必要があります。Authorization Core の機能は Authorization Extension とは異なります。比較については、[Authorization Core と Authorization Extension の比較](/docs/ja/manage-users/access-control/authorization-core-vs-authorization-extension)をご覧ください。
* <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で [API を設定](/docs/ja/get-started/auth0-overview/set-up-apis) します。
* 権限は事前定義された値から選択します。権限の一覧が空の場合は、API に[権限を追加](/docs/ja/get-started/apis/add-api-permissions)する必要があります。

<div id="dashboard">
  ## Dashboard
</div>

1. [Dashboard &gt; User Management &gt; Roles](https://manage.auth0.com/#/roles) に移動し、**Create Role** をクリックします。
2. ロール名と説明を入力し、**Create** をクリックします。

<div id="management-api">
  ## Management API
</div>

[Create Role endpoint](https://auth0.com/docs/api/management/v2#!/Roles/post_roles) に `POST` リクエストを送信します。`MGMT_API_ACCESS_TOKEN`、`ROLE_NAME`、`ROLE_DESC` のプレースホルダー値は、それぞれご自身の Management API の<Tooltip tip="Access Token: Authorization credential, in the form of an opaque string or JWT, used to access an API." cta="用語集を見る" href="/docs/ja/glossary?term=access+token">アクセストークン</Tooltip>、ロール名、ロールの説明に必ず置き換えてください。

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/api/v2/roles' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "name": "ROLE_NAME", "description": "ROLE_DESC" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/roles");
  var request = new RestRequest(Method.POST);
  request.AddHeader("content-type", "application/json");
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("cache-control", "no-cache");
  request.AddParameter("application/json", "{ "name": "ROLE_NAME", "description": "ROLE_DESC" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/roles"

  	payload := strings.NewReader("{ "name": "ROLE_NAME", "description": "ROLE_DESC" }")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("content-type", "application/json")
  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("cache-control", "no-cache")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

| **値**                   | **説明**                                                                                              |
| ----------------------- | --------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN` | `create:roles` スコープを持つ [Management API のアクセストークン](https://auth0.com/docs/api/management/v2/tokens)。 |
| `ROLE_NAME`             | 作成するロールの名前。                                                                                         |
| `ROLE_DESC`             | ロールのユーザー向け説明。                                                                                       |
