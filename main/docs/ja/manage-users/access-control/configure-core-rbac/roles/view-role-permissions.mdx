ロールに追加された[権限](/docs/ja/manage-users/access-control/rbac)は、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>または<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip>で確認できます。割り当てられた権限とロールは、API Authorization Core 機能セットで使用されます。

<div id="prerequisites">
  ## 前提条件
</div>

ロールベースのアクセス制御 (RBAC) を正しく機能させるには、Auth0 Dashboard または Management API のいずれかを使用して、対象のAPIでこれを有効にする必要があります。Authorization Core の機能は Authorization Extension とは異なります。比較については、[Authorization Core と Authorization Extension の比較](/docs/ja/manage-users/access-control/authorization-core-vs-authorization-extension)をご覧ください。

<div id="dashboard">
  ## Auth0 Dashboard
</div>

1. [Auth0 Dashboard &gt; User Management &gt; Roles](https://manage.auth0.com/#/roles) に移動し、表示したいロール名をクリックします。
2. **Permissions** ビューをクリックします。

各権限について、次の情報が表示されます。

| Column      | Description             |
| ----------- | ----------------------- |
| Name        | 権限定義に含まれる権限名。           |
| Description | 権限定義に含まれる権限の説明。         |
| API         | その権限が関連付けられている API の名前。 |

<div id="management-api">
  ## Management API
</div>

[Get Role Permissions endpoint](https://auth0.com/docs/api/management/v2#!/Roles/get_role_permission) に `GET` リクエストを送信します。プレースホルダー値 `ROLE_ID` と `MGMT_API_ACCESS_TOKEN` は、それぞれロール ID と Management API の <Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を見る" href="/docs/ja/glossary?term=access+token">アクセストークン</Tooltip> に必ず置き換えてください。

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/roles/ROLE_ID/permissions' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/roles/ROLE_ID/permissions");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/roles/ROLE_ID/permissions"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse<String> response = Unirest.get("https://{yourDomain}/api/v2/roles/ROLE_ID/permissions")
    .header("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
    .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'GET',
  ```

  ```php PHP
  $curl = curl_init();

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/api/v2/roles/ROLE_ID/permissions",
  ```

  ```python Python

  conn = http.client.HTTPSConnection("")

  headers = { 'authorization': "Bearer MGMT_API_ACCESS_TOKEN" }

  conn.request("GET", "/{yourDomain}/api/v2/roles/ROLE_ID/permissions", headers=headers)

  res = conn.getresponse()
  data = res.read()

  print(data.decode("utf-8"))
  ```

  ```ruby Ruby
  require 'uri'

  http.use_ssl = true

  response = http.request(request)
  puts response.read_body
  ```
</AuthCodeGroup>

| 値                       | 説明                                                                                                 |
| ----------------------- | -------------------------------------------------------------------------------------------------- |
| `ROLE_ID`               | permissions を取得するロールの ID。                                                                          |
| `MGMT_API_ACCESS_TOKEN` | スコープ `read:roles` を持つ [Management API 用アクセストークン](https://auth0.com/docs/api/management/v2/tokens)。 |
