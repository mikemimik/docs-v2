[organization](/docs/ja/manage-users/organizations/organizations-overview) を削除すると、すべてのユーザーのメンバーシップと削除された organization との関連付けは削除されますが、ユーザー自体は tenant から削除されません。

<Warning>
  いったん確定すると、この操作は元に戻せません。
</Warning>

organization は、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を表示" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を表示" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> のいずれかを使用して削除できます。

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Auth0 Dashboard から organization を削除するには、次の手順に従います。

1. [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) に移動し、削除する organization を選択します。
2. **Danger Zone** セクションまでスクロールし、**Delete** を選択して確認します。

<div id="management-api">
  ## Management API
</div>

`Delete Organization` endpoint に `DELETE` リクエストを送信します。`ORG_ID` と `MGMT_API_ACCESS_TOKEN` のプレースホルダー値は、それぞれ organization ID と Management API の <Tooltip tip="Access Token: API へのアクセスに使用される、不透明な文字列または JWT 形式の認証情報。" cta="用語集を見る" href="/docs/ja/glossary?term=Access+Token">Access Token</Tooltip> に必ず置き換えてください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Auth0 ドメインを確認する**

  Auth0 ドメインは、テナント名、リージョンのサブドメイン (テナントが US リージョンにあり、2020 年 6 月より前に作成されている場合を除く) 、および `.auth0.com` を組み合わせたものです。たとえば、テナント名が `travel0` の場合、Auth0 ドメイン名は `travel0.us.auth0.com` になります。 (テナントが US にあり、2020 年 6 月より前に作成されている場合、ドメイン名は `https://travel0.auth0.com` になります。)

  カスタムドメインを使用している場合は、ここにそのカスタムドメイン名を指定してください。
</Callout>

| Value                   | Description                                                                                                                             |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | 削除する organization の ID。                                                                                                                 |
| `MGMT_API_ACCESS_TOKEN` | scope `delete:organizations` を持つ [Access Token for the Management API](/docs/ja/secure/tokens/access-tokens/management-api-access-tokens)。 |

<div id="response-status-codes">
  #### レスポンスステータスコード
</div>

考えられるレスポンスステータスコードは次のとおりです。

| ステータスコード | エラーコード               | メッセージ                                                                                  | 原因                                                          |
| -------- | -------------------- | -------------------------------------------------------------------------------------- | ----------------------------------------------------------- |
| `204`    |                      | organization は削除されました。                                                                 |                                                             |
| `400`    | `invalid_uri`        | リクエスト URI が無効です。メッセージは原因によって異なります。                                                     | パスが無効です。                                                    |
| `401`    |                      | トークンが無効です。                                                                             |                                                             |
| `401`    |                      | JSON Web トークンのバリデーションで無効な署名を受信しました。                                                    |                                                             |
| `403`    | `insufficient_scope` | スコープが不足しています。次のいずれかが必要です: `delete:organizations`。                                      | 指定された bearer token のスコープでは許可されていないフィールドの読み取りまたは書き込みを試行しました。 |
| `404`    |                      | organization は存在しません。                                                                  |                                                             |
| `429`    |                      | リクエストが多すぎます。X-RateLimit-Limit、X-RateLimit-Remaining、X-RateLimit-Reset の各ヘッダーを確認してください。 |                                                             |
