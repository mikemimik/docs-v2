ユーザーを検索する際は、検索を絞り込むために [Lucene query syntax](https://lucene.apache.org/core/2_9_4/queryparsersyntax.html) を使ってクエリを作成できます。

クエリ文字列は、一連のタームと演算子に解析されます。

* タームには、`jane` や `smith` のような単一の単語を指定できます。
* タームには、二重引用符で囲んだフレーズ (`"green apple"`) も指定でき、その場合はフレーズ内のすべての単語が同じ順序で一致します。
* フィールド名のないタームは、[ユーザーメタデータ](/docs/ja/manage-users/user-accounts/metadata) フィールド内のテキストには一致しません。
* 複数のタームは、かっこでグループ化してサブクエリを作成できます。
* 正規化されたユーザーフィールド (`email`、`name`、`given_name`、`family_name`、`nickname`) の検索値では、大文字と小文字は区別されません。それ以外のすべてのフィールド (すべての `app_metadata` / `user_metadata` フィールドを含む) では、大文字と小文字が区別されます。
* 演算子 (`AND`、`OR`、`NOT`) は、すべての正規化されたユーザーフィールドと root フィールドで機能します。
* 演算子は常に大文字で記述する必要があります。

<div id="searchable-fields">
  ## 検索可能なフィールド
</div>

すべての[正規化されたユーザープロファイルのフィールド](/docs/ja/manage-users/user-accounts/user-profiles/normalized-user-profile-schema)と、以下のフィールドを使用してユーザーを検索できます。

| 検索フィールド           | データ型                       | 説明                                                                                                                                                      |
| ----------------- | -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `phone_number`    | text                       | ユーザーの電話番号です。SMS 接続のユーザーに対してのみ有効です。                                                                                                                      |
| `phone_verified`  | boolean                    | `true/false` の値で、ユーザーの電話番号が確認済みかどうかを示します。SMS 接続のユーザーに対してのみ有効です。                                                                                         |
| `logins_count`    | integer                    | ユーザーがログインした回数です。ユーザーがブロックされている状態でログインした場合、そのブロックされたセッションも `logins_count` に含まれ、`last_login` の値も更新されます。                                                   |
| `created_at`      | date time                  | ユーザープロファイルが最初に作成された日時のタイムスタンプです。                                                                                                                        |
| `updated_at`      | date time                  | ユーザーのプロファイルが最後に更新または変更された日時のタイムスタンプです。                                                                                                                  |
| `last_login`      | date time                  | ユーザーが最後にログインした日時のタイムスタンプです。このプロパティを `user` オブジェクトとともに [Rule](/docs/ja/customize/rules) 内から参照した場合、その値はそのルールをトリガーしたログインに対応します (ルールは実際のログイン後に実行されるため) 。       |
| `last_ip`         | text (有効な IP アドレス)         | ユーザーの最後のログインに関連付けられた IP アドレスです。                                                                                                                         |
| `blocked`         | boolean                    | `true` または `false` の値で、ユーザーがブロックされているかどうかを示します。注: `true` で返されるのは、Auth0 Dashboard と Management API でブロックされたユーザーのみであり、ブルートフォース異常検知によってブロックされたユーザーは返されません。 |
| `email.domain`    | text                       | ユーザーのメールアドレスのドメイン部分です。                                                                                                                                  |
| `organization_id` | text (有効な organization ID) | ユーザーがメンバーになっている organization です。                                                                                                                        |

メタデータフィールドでは、次の型を使用できます。

* boolean
* numeric: integer または double
* text
* objects: 別のオブジェクト内にネストされたスカラー値を検索するには、フィールドへのパスを使用します。たとえば、`app_metadata.subscription.plan:"gold"`
* arrays: 配列内のオブジェクトにネストされたフィールドを検索するには、フィールドへのパスを使用し、配列レベルは無視します。たとえば、`user_metadata.addresses.city:"Paris"`

空の配列、空のオブジェクト、または `null` 値を含むメタデータフィールドはインデックス化されないため、検索できません。

範囲検索およびワイルドカード検索は、`user_metadata` フィールドでは使用できません。

<div id="exact-match">
  ## 完全一致
</div>

完全一致で検索するには、二重引用符を使用します: `name:"jane smith"`。

たとえば、名前が `jane smith` のユーザーを検索するには、`q=name:"jane smith"` を使用します:

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3' \
    --header 'authorization: Bearer {yourMgmtApiAccessToken}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse<String> response = Unirest.get("https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3")
    .header("authorization", "Bearer {yourMgmtApiAccessToken}")
    .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'GET',
    url: 'https://{yourDomain}/api/v2/users',
    params: {q: 'name:"jane smith"', search_engine: 'v3'},
    headers: {authorization: 'Bearer {yourMgmtApiAccessToken}'}
  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP
  $curl = curl_init();

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_ENCODING => "",
    CURLOPT_MAXREDIRS => 10,
    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
    CURLOPT_CUSTOMREQUEST => "GET",
    CURLOPT_HTTPHEADER => [
      "authorization: Bearer {yourMgmtApiAccessToken}"
    ],
  ]);

  $response = curl_exec($curl);
  $err = curl_error($curl);

  curl_close($curl);

  if ($err) {
    echo "cURL Error #:" . $err;
  } else {
    echo $response;
  }
  ```

  ```python Python
  import http.client

  conn = http.client.HTTPSConnection("")

  headers = { 'authorization': "Bearer {yourMgmtApiAccessToken}" }

  conn.request("GET", "/{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3", headers=headers)

  res = conn.getresponse()
  data = res.read()

  print(data.decode("utf-8"))
  ```

  ```ruby Ruby
  require 'uri'
  require 'net/http'
  require 'openssl'

  url = URI("https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3")

  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Get.new(url)
  request["authorization"] = 'Bearer {yourMgmtApiAccessToken}'

  response = http.request(request)
  puts response.read_body
  ```
</AuthCodeGroup>

<div id="wildcards">
  ## ワイルドカード
</div>

ワイルドカード検索では、アスタリスク文字 (`*`) を使って0文字以上の文字を置き換え、タームを検索できます。ワイルドカード検索は `user_metadata` フィールドでは使用できません。

<div id="examples">
  #### 例
</div>

* `name:john*` は、名前が `john` で始まるすべてのユーザーを返します。
* `name:j*` は、名前が `j` で始まるすべてのユーザーを返します。
* `q=name:john*` は、名前が `john` で始まるすべてのユーザーを返します。
* 接尾辞一致では、リテラルは 3 文字以上である必要があります。たとえば、`name:*usa`` は使用できますが、`name:*sa&#96; は使用できません。

<div id="ranges">
  ## 範囲
</div>

ユーザー検索クエリでは範囲を指定できます。ユーザーのメタデータフィールドでは範囲検索は使用できません。

* 境界値を含む範囲には、角括弧を使用します: `[min TO max]`。
* 境界値を含まない範囲には、波括弧を使用します: `{min TO max}`。
* 波括弧と角括弧は、同じ範囲式内で組み合わせて使用できます: `logins_count:[100 TO 200}`。
* 範囲はワイルドカードと組み合わせて使用できます。たとえば、ログイン回数が 100 回を超えるすべてのユーザーを検索するには、`q=logins_count:{100 TO *]` を使用します。

<div id="searchable-profile-attribute-examples">
  ## 検索可能なプロファイル属性の例
</div>

Auth0 <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにするための製品。" cta="用語集を表示" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> でユーザーを検索する際は、`user_metadata` または `app_metadata` を使ってユーザーを絞り込めます。これには、`q` パラメータで Lucene Search Syntax を使用します。

Auth0 Management API の [ユーザー一覧またはユーザー検索](https://auth0.com/docs/api/management/v2#!/Users/get_users) エンドポイントは、1000 件の結果 (100 レコード × 10 ページ) までに制限されているため、絞り込みは関連性の高い結果を確実に返すうえで有効です。

以下は、ユーザープロファイルの `user_metadata` のサンプルです。

```json lines
{
  "favorite_color": "blue",
  "approved": false,
  "preferredLanguage": "en",
  "preferences": {
    "fontSize": 13
  },
  "addresses":{
    "city":["Paris","Seattle"]
  }
}
```

<div id="filter-metadata-attributes">
  ### メタデータ属性の絞り込み
</div>

`user_metadata` の値を返すには、属性で絞り込むフィルターを `q` クエリに追加します。

`user_metadata` の値は、プロファイルに対して直接クエリできます。

`q: _exists_:user_metadata.fav_color`

このクエリは、`user_metadata` に `fav_color` 属性があるすべてのユーザープロファイルを返します。

<div id="filter-metadata-nested-object-attributes-and-values">
  ### メタデータ内のネストされたオブジェクトの属性と値で絞り込む
</div>

`user_metadata` 内のネストされたオブジェクトも検索できます。

`q: _exists_:user_metadata.preferences.fontSize`

このクエリは、`user_metadata` に `preferences.fontSize` が設定されているすべてのユーザープロファイルを検索します。

別のオブジェクト内にあるネストされたオブジェクトの値を検索するには、以下のクエリを参照してください。

`q: user_metadata.preferences.fontSize:13`

このクエリは、`fontSize` 属性の値が `13` に一致するすべてのユーザープロファイルを返します。

<div id="filter-metadata-nested-array-values">
  ### メタデータ内のネストされた配列値を絞り込む
</div>

ネストされた配列内のフィールドを検索するには、以下のクエリを使用できます。

`q: user_metadata.addresses.city:"Seattle"`

これにより、`user_metadata` 内の `address.city` 属性の値が `Seattle` であるすべてのユーザープロファイルが返されます。
