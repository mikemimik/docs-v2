---
description: カスタムデータベースで自動ユーザー移行を有効にする方法を説明します。
title: データベースからの自動移行を設定する
---

[カスタムデータベースのアクションスクリプト](/docs/ja/authenticate/database-connections/custom-db/templates)を使用して、外部ユーザーストアから Auth0 にユーザーが自動的に移行されるよう、データベース接続を設定します。

<Warning>
  複数の移行方法 (たとえば、自動移行の後に一括ユーザーインポートを行うなど) を使用しようとすると、`DUPLICATED_USER` エラーが発生することがあります。このエラーは、ユーザーが Auth0 の内部ユーザーストアには存在している一方で、テナントには存在していないことを示します。このエラーを解消するには、Auth0 Management API の [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) エンドポイントを使用してそのユーザーを削除し、その後でもう一度インポートを試してください。
</Warning>

<div id="configure-database-connection-as-custom">
  ## データベース接続をカスタムとして設定
</div>

データベース接続を作成して、カスタムとして設定します。

1. [Auth0 Dashboard &gt; 認証 &gt; データベース](https://manage.auth0.com/#/connections/database) に移動し、対象のデータベース接続を選択します。
2. **Custom Database** ビューを選択し、**Use my own database** をオンにします。

   <Frame>![Auth0 Dashboard Authentication Database Connection Custom Database Settings Use Own Database Enabled](/docs/images/cdy7uua7fh8z/11HPAdVwJMmnWbzMVjHCJ8/f75ff47f613a41778f266643f29ab2b1/2025-02-27_17-25-36.png)</Frame>
3. **Settings** ビューを選択し、**Import Users to Auth0** をオンにして、**Save** を選択します。

<div id="configure-database-action-scripts">
  ## Database Action Scripts を設定する
</div>

**Custom Database**ビューを選択し、**Database Action Scripts**を探します。

<Frame>![Auth0 Dashboard 認証 データベース接続 Custom Database タブ Database Action Scripts](/docs/images/cdy7uua7fh8z/4LBIvvjaABo51It4eVCmX3/4ef14765c79cc5b7ecfa3f3155caa9eb/Screen_Shot_2021-05-18_at_8.54.55_PM.png)</Frame>

<div id="login">
  ### Login
</div>

[Login](/docs/ja/authenticate/database-connections/custom-db/templates/login) スクリプトは、Auth0 に存在しないユーザーがログインを試みるたびに実行されます。このスクリプトは、ユーザーにパスワードの再入力を求めることなく、そのユーザーが外部ユーザーストアに存在するかどうかを確認します。

<div id="get-user">
  ### Get User
</div>

[Get User](/docs/ja/authenticate/database-connections/custom-db/templates/get-user) スクリプトは、次のいずれかの状況で実行されます。

* ユーザーがサインアップを試みたとき。
* ユーザーがパスワード変更フローを開始したとき。
* ユーザーが Identifier First + Biometrics の使用時にログイン識別子を入力したとき。
* <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を表示" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> の [ユーザーの Email または Username を更新](https://auth0.com/docs/api/v2#!/Users/patch_users_by_id) または [ユーザー作成](https://auth0.com/docs/api/management/v2#!/Users/post_users) エンドポイントが呼び出されたとき。

まだ移行されていないユーザーがパスワード変更を確定し、正常にログインすると、そのユーザーのユーザープロファイルが新しいパスワードとともに Auth0 に作成されます。このユーザープロファイルには、Get User スクリプトで返されるすべてのデータが含まれます。以後、このユーザーのログインはすべて Auth0 で直接実行されます。

<div id="verify-user-migration-is-complete">
  ## ユーザー移行が完了していることを確認する
</div>

<Tooltip tip="サービスの設定に使用する Auth0 の主要なプロダクト: Auth0 Dashboard。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または Auth0 Management API を使用して、すべてのユーザーが Auth0 のユーザーストアに移行されたことを確認します。

1. [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) に移動し、ユーザー一覧を確認します。
2. Management API の [List or Search Users](https://auth0.com/docs/api/v2#!/Users/get_users) エンドポイントを使用します。

<div id="disconnect-external-user-store">
  ## 外部ユーザーストアの接続を解除する
</div>

Database Action Scripts を何もしない関数として再設定します。これにより、Auth0 がユーザーを認証するために外部ユーザーストアへアクセスしなくなります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Import Users to Auth0** オプションは、必ずオンのままにしてください。このオプションをオフにすると、Auth0 はローカルにインポートされたユーザーを使わず、認証やその他のユーザー関連処理にスクリプトのみを使用するようになります。
</Callout>

1. [Auth0 Dashboard &gt; 認証 &gt; Database](https://manage.auth0.com/#/connections/database) に移動し、データベース接続を選択します。

2. **Custom Database** ビューに切り替え、**Database Action Scripts** を見つけます。

3. **Login** スクリプトを更新します。

   ```javascript lines
   function login (email, password, callback) {
     return callback(null, null);
   }
   ```

4. **Get User** スクリプトを更新します。

   ```javascript lines
   function getByEmail (email, callback) {
     return callback(null, null);
   }
   ```

<div id="troubleshoot-user-migration-issues">
  ## ユーザー移行の問題のトラブルシューティング
</div>

自動移行で問題が発生した場合は、まず [カスタムデータベース接続とアクションスクリプトのベストプラクティス](/docs/ja/authenticate/database-connections/custom-db/custom-database-connections-scripts) を参照してください。

<div id="user-already-exists">
  ### ユーザーはすでに存在します
</div>

この問題が発生する一般的なケースは、次のとおりです。

* データベース接続からユーザーを削除した後、そのユーザーを再作成しようとする。
* ユーザーが外部ユーザーストアにすでに存在している状態で、データベース接続に新しいユーザーを作成しようとする。
* ユーザーの移行プロセスが途中で中断された。

ユーザー移行プロセスでは、Auth0 はまず内部ユーザーストアに不完全なユーザープロファイルを作成し、その後、データベース接続に完全なユーザープロファイルを作成します。何らかの問題が発生してこの完全なユーザープロファイルを作成できない場合、`The user already exists` エラーが発生することがあります。

通常、この問題は、データベース接続からユーザーを削除し、内部ユーザーストアからもユーザーを削除したうえで、移行プロセスを再試行することで解決できます。

1. [Actions リアルタイムログ](/docs/ja/customize/actions/actions-real-time-logs) で `console.log()` の出力を確認します。
2. Management API の [Delete a User](https://auth0.com/docs/api/management/v2#!/Users/delete_users_by_id) エンドポイントを使用してユーザーを削除します。
3. Management API の [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) エンドポイントを使用してユーザーを削除します。
4. ユーザーに、移行を再試行するため、ログインするかパスワードを変更するよう案内します。

<div id="missing-metadata">
  ### メタデータの欠落
</div>

移行プロセスが開始されるログインまたはパスワード変更フローの途中でユーザーの操作が中断されると、Auth0 が他のプロファイルデータとあわせてそのメタデータ (`user_metadata` または `app_metadata`) を移行できない場合があります。

この問題は、ユーザーのプロファイルにメタデータがないことを確認し、外部ユーザーストアから取得して Auth0 に保存する [Action](/docs/ja/customize/actions) を作成することで軽減できます。
