title: アプリケーションへのユーザーアクセスの管理
---

1 つの Auth0 テナントに関連付けられているすべてのユーザーは、そのテナント内のアプリケーション間で共有されます (そのため、各アプリケーションにアクセスできます) 。

* ユーザーを分離してアクセスを制限するには、特定のユーザーグループにアクセスさせたいアプリケーションを含む[追加のテナントを作成する](/docs/ja/get-started/auth0-overview/create-tenants/create-multiple-tenants)ことをおすすめします。

* 特定のアプリケーションへのユーザーのアクセスを制限するには、[ルール](/docs/ja/customize/rules)を使用できます。

  * ルール内では、ユーザーがログイン時に使用しているアプリケーションを確認するために、`context.clientName` と `context.clientID` の変数を使用できます。
  * 例については、[このルール](https://github.com/auth0/rules/blob/aeaf93bc058408e260192d0941a688963449d6be/src/rules/simple-user-whitelist-for-app.js)を参照してください。

* 新しい接続を設定し、特定のアプリケーションにのみアクセスを許可することで、ユーザーがアプリケーションを利用できないように制限することもできます。

  * [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social) に移動し、接続をクリックします。
  * **アプリケーション** タブで、アプリケーションを有効または無効にします。

* アプリケーションへのユーザーアクセスを無効にするには、Auth0 Dashboard で[ユーザーのブロックとブロック解除](/docs/ja/manage-users/user-accounts/block-and-unblock-users)を行うことができます。
