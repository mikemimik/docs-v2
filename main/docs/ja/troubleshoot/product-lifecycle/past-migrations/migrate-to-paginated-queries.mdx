description: 特定のManagement API v2エンドポイントのクエリで、ページネーションパラメーターを使用する方法について説明します。
title: Management API v2 エンドポイントのページネーション付きクエリへの移行
---

**2021年1月26日**以降 (またはPrivate Cloud version 2202以降) 、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> v2エンドポイントへのリクエストでは、Public Cloud のテナントに対して最大50件までしか返されません。さらに多くの項目を取得するには、`page` と `per_page` パラメーターを含める必要があります。**2020年7月21日** (またはPrivate Cloud version 2108) 以降、Auth0 はこの変更に備えられるよう、テナントログと移行用のトグルを表示します。

影響を受けるテナントは、次の条件を満たすものです。

* **2020年7月21日より前**に作成された Public Cloud テナント、またはバージョン2108の Private Cloud デプロイ
* 影響を受けるエンドポイントに対して、複数の結果を返す可能性があるクエリで `per_page` パラメーターを渡さずに、継続的に呼び出しを行っていること。

次のテナントでは、新しいページネーション動作がデフォルトとなり、2021年1月26日以降も現在と同じように動作し続けます。

* **2020年7月21日以降**に作成された Public Cloud テナント、またはバージョン2108以上の Private Cloud デプロイ
* 影響を受けるエンドポイントを使用していない
* 影響を受けるエンドポイントを使用し、`per_page` パラメーターを渡している、または常に単一の結果を返すクエリを実行している。

<div id="endpoints-affected">
  ## 影響を受けるエンドポイント
</div>

以下の Management API v2 エンドポイントの呼び出しが影響を受けます:

* [`GET /api/v2/clients`](https://auth0.com/docs/api/management/v2#!/Clients/get_clients)
* [`GET /api/v2/client_grants`](https://auth0.com/docs/api/management/v2#!/Clients/client_grants)
* [`GET /api/v2/grants`](https://auth0.com/docs/api/management/v2#!/Clients/grants)
* [`GET /api/v2/connections`](https://auth0.com/docs/api/management/v2#!/Clients/connections)
* [`GET /api/v2/device-credentials`](https://auth0.com/docs/api/management/v2#!/Clients/device_credentials)
* [`GET /api/v2/resource-servers`](https://auth0.com/docs/api/management/v2#!/Clients/resource_servers)
* [`GET /api/v2/rules`](https://auth0.com/docs/api/management/v2#!/Clients/rules)

ページネーションオプションを指定していないすべてのリクエストのうち、現在 2 件以上の項目を返しているものについては、クライアントとエンドポイントの組み合わせごとに 1 時間に 1 回、非推奨通知がテナントログに記録されます。

<div id="actions">
  ## Actions
</div>

1. `page` パラメーターと `per_page` パラメーターを指定して、影響を受けるすべてのエンドポイントへの呼び出しを置き換えます。

| パラメーター     | 型       | 説明                                           |
| ---------- | ------- | -------------------------------------------- |
| `page`     | Integer | 返される結果のページの index。最初のページは 0 です。              |
| `per_page` | Integer | 1 ページあたりの結果数。このパラメーターを送信しない場合、ページングは無効になります。 |

2. テナントログに非推奨通知が表示されなくなったことを確認します。リクエストが 50 件を超える項目を返していないか確認してください。`details.size_exceeded` フィールドを確認し、値が `true` かどうかを確認します。

   1. ページネーションオプションなしで 1 件を超える結果を返すすべての呼び出しを確認するには、次のログクエリを使用します。
      `type:depnote AND description:*Unpaginated*`
   2. ページネーションオプションなしで 50 件を超える結果を返すすべての呼び出しを確認するには、次のログクエリを使用します。
      `type:depnote AND description:*Unpaginated* AND details.size_exceeded:true`

   リクエストを行っているアプリケーションを特定するには、ログにそのリクエストで使用された `client_id` が含まれます。使用されているエンドポイントは、ログの `details.path` フィールドでも確認できます。
3. テナントで Management API のページネーションなしリクエストを無効にします。[Auth0 Dashboard &gt; Tenant Settings &gt; Advanced &gt; Migration](https://manage.auth0.com/#/tenant/advanced) に移動してください。これにより、migration window の終了後に想定される動作を再現でき、影響を受けるエンドポイントへの呼び出しは最大 50 件の結果を返すようになります。その日付までは、ページネーションなしリクエストをいつでも再度有効にできます。

<div id="update-extensions">
  ### 拡張機能を更新する
</div>

以前のバージョンの[Auth0 Extensions](/docs/ja/customize/extensions)やカスタム拡張機能は、ページネーション対応のクエリのみを実行するように、最新バージョンへ更新する必要がある場合があります。

1. テナントログで、ID が拡張機能の URL と一致するクライアントに関する非推奨通知を確認します。通知がある場合は、その拡張機能を更新する必要があります。
2. [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions)に移動し、**Installed Extensions**を選択して、表示されていれば拡張機能の **Update** リンクをクリックします。

<div id="update-deploy-cli-tool">
  ### Deploy CLI ツールを更新する
</div>

`auth0-deploy-cli` ツールを使用している場合は、**バージョン 5.3.0 以降**に更新する必要があります。詳細については、[Deploy CLI ツール](/docs/ja/deploy-monitor/deploy-cli-tool) を参照してください。
