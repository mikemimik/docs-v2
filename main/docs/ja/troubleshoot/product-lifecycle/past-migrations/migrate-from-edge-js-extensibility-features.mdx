2023年6月21日以降、Auth0 では .NET および C# の Node.js Extensibility 機能はサポートされなくなります。

<div id="affected-workflows">
  ## 影響を受けるワークフロー
</div>

`require(edge)` を含むコードがないか、Extensibility 機能を確認してください。

拡張コードを使用できる以下のフローまたは機能で、`edge.js` への参照がないか確認してください。

* [フック](/docs/ja/customize/hooks)
* [ルール](/docs/ja/customize/rules)
* [Custom Database Scripts](/docs/ja/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Custom Social Connections](/docs/ja/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## 推奨される更新
</div>

この機能について、公式に予定されている後継機能はありません。

移行方法の 1 つは、`edge` のロジックを自社のインフラストラクチャ上の別サービスに切り出すことです。インフラストラクチャ内にそのサービスを作成したら、拡張機能の関数を使って HTTP リクエストを送信し、必要な操作をトリガーできます。

たとえば、`edge` を使用する Custom Database Login スクリプトでは、レスポンスを取得するために `edge` を呼び出す関数を削除し、代わりに汎用の HTTP クライアントを使用します。

別の移行方法として、拡張機能を .NET/C# ではなく JavaScript で書き直すこともできます。

Actions の作成や Actions への移行の詳細については、[Auth0 のドキュメントを参照してください](/docs/ja/customize/actions)。
