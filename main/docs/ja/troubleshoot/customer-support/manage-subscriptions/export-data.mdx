description: Auth0 からデータをエクスポートする方法を説明します。
title: データのエクスポート
---

Auth0 テナント内のすべてのデータは常にお客様の管理下にあり、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> を通じていつでも利用できます。

<div id="user-passwords">
  ## ユーザーのパスワード
</div>

API 経由では、Auth0 でホストされるデータベースユーザーのパスワードハッシュと秘密鍵のみ、 (セキュリティ上の理由により) 利用できません。この情報は、[サポートチケット](https://support.auth0.com)を作成することでリクエストできます。詳しくは、[Export Password Hashes and MFA Secrets](/docs/ja/manage-users/user-migration/export-password-hashes-and-mfa-secrets)をお読みください。この操作は Free サブスクリプションティアでは利用できません。また、特定の日時を指定したエクスポートの依頼はお受けできず、保証もいたしかねます。

ユーザーのパスワードをお使いのデータベースに保存している場合は、ユーザーがログインするたびに Auth0 が問い合わせるカスタムデータベース接続を設定してください。この場合、ユーザーを Auth0 に段階的に移行することを選ばない限り、Auth0 がパスワードハッシュを保存することはありません。

<div id="auth0-components">
  ## Auth0 のコンポーネント
</div>

Auth0 は認証プロトコルとして主に <Tooltip tip="OpenID: アプリケーションがログイン情報を収集・保存することなく、ユーザーの本人確認を行えるようにする認証のオープン標準。" cta="用語集を表示" href="/docs/ja/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) を使用しているため、標準ライブラリを使ってアプリケーションとの連携を実装できるはずです。[SAML](/docs/ja/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider) を介して Auth0 と連携する場合も同様です。Auth0 のすべての SDK、ライブラリ、サンプルは、[GitHub で無償ソフトウェアとして公開されています](https://github.com/auth0/)。

<div id="social-identity-providers">
  ## ソーシャルIDプロバイダー
</div>

Auth0 を使用しない場合でも、ソーシャル <Tooltip tip="IDプロバイダー (IdP): デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+providers">アイデンティティプロバイダー</Tooltip> で同じ <Tooltip tip="OAuth 2.0: 認可のプロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を見る" href="/docs/ja/glossary?term=OAuth">OAuth</Tooltip> <Tooltip tip="OAuth 2.0: 認可のプロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を見る" href="/docs/ja/glossary?term=client+IDs">クライアントID</Tooltip> とシークレットを引き続き使用するなら、新たな同意ダイアログを表示しなくても、ユーザー情報へのアクセスを維持できます。

<div id="custom-code">
  ## カスタムコード
</div>

Auth0 のカスタムコード機能 (ルール、カスタムデータベーススクリプト、カスタム OAuth 接続など) は、いずれも Node.js のサンドボックスサービス上で実行されます。サンドボックスサービスで利用できるライブラリは、通常の Node.js コードで使えるよう、いずれも npm でも提供されています。
