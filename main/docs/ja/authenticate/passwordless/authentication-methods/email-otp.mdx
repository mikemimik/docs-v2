詳しくは、[パスワードレス認証用の Email または SMS を設定する](/docs/ja/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms)をご覧ください。

<div id="how-it-works">
  ## 仕組み
</div>

新規ユーザーが OTP を受け取り、初めてあなたのウェブサイトでその OTP を入力すると、Auth0 による認証前に、そのユーザーのユーザープロファイルが `email` 接続上に作成されます。

OTP の送信先メールアドレスが既存ユーザーのものと一致する場合、Auth0 はそのユーザーを認証します:

<div id="configure-the-connection">
  ## 接続を設定する
</div>

1. [Auth0 Dashboard &gt; Authentication &gt; パスワードレス](https://manage.auth0.com/#/connections/passwordless) に移動し、**Email** トグルを有効にします。
2. **Email** を選択して設定ウィンドウを開き、**設定**ビューに切り替えてから、メールの **From**、**件名**、**Message** テキストを入力します。

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Auth0 でカスタムメールテンプレートを送信するには、**From** の値を `auth0.com` ドメイン以外のメールアドレスに変更する必要があります。変更しない場合、Auth0 はデフォルトのメールテンプレートを送信します。
   </Callout>
3. **OTP Expiry** と **OTP Length** を設定します。

   * 発行された OTP (またはリンク) のうち、有効なのは最新のものだけです。新しいものが発行されると、それ以前のものはすべて無効になります。また、最新のものも一度使用すると無効になります。
   * OTP の入力に失敗できる回数は 3 回までです。3 回失敗すると、新しいコードをリクエストする必要があります。
   * 発行された OTP の有効期間は、デフォルトで 3 分です。
   * OTP の有効期限を延ばす場合は、OTP の長さも長くする必要があります。そうしないと、攻撃者が短いコードを推測するための猶予が大きくなります。
4. **Disable Sign Ups** を有効にするかどうかを決定します。この設定を有効にすると、既存ユーザーにのみパスワードレスアクセスを許可できますが、アプリケーションがユーザー列挙攻撃にさらされる可能性があります。詳しくは、[パスワードレス接続のベストプラクティス](/docs/ja/authenticate/passwordless/best-practices) を参照してください。
5. **Save** を選択します。

<div id="multi-language-support">
  ### 多言語サポート
</div>

**Message** エリアは複数の言語に対応しています。

言語を指定するには、[Auth0 Authentication API Get Code or Link endpoint](https://auth0.com/docs/api/authentication#get-code-or-link) を呼び出し、`x-request-language` ヘッダーの値を設定します。このヘッダーが設定されていない場合、言語はブラウザーによって自動的に設定される `accept-language` ヘッダーから取得されます。

<div id="message-syntax">
  ### Message の構文
</div>

**Message** 欄では Liquid 構文を使用できます。この構文をパラメーター値と組み合わせることで、メッセージの要素を動的に作成できます。

たとえば、`request_language` パラメーターを参照して、メッセージの言語を変更できます。

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ code }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ code }}
{% endif %}
```

メッセージテンプレートの定義時に使用できるパラメーターは次のとおりです。

| Parameter          | Description                                                                                       |
| ------------------ | ------------------------------------------------------------------------------------------------- |
| `code`             | 使用するパスワード。                                                                                        |
| `link`             | 生成されたサインインリンク。                                                                                    |
| `application.name` | ユーザーがサインアップに使用しているアプリケーションの名前。                                                                    |
| `request_language` | メッセージ内容に対してリクエストされた言語。                                                                            |
| `operation`        | API を介したユーザーのメールアドレス更新によってテンプレートがトリガーされたかどうかを示します。トリガーされた場合、値は `change_email`、それ以外の場合は `null` です。 |

ユーザーが [組織](/docs/ja/manage-users/organizations/organizations-overview) 経由でログインしている場合は、次のパラメーターも使用できます。

* `organization.id`
* `organization.display_name`
* `organization.name`
* `organization.metadata`
* `organization.branding.logo_url`
* `organization.branding.colors.primary`
* `organization.branding.colors.page_background`

これらのパラメーターとその値の詳細については、Auth0 の <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> の [Get organization](https://auth0.com/docs/api/management/v2#!/Organizations/get_organizations_by_id) エンドポイントを参照してください。

**アプリケーション**ビューに切り替え、Passwordless Email を使用するアプリケーションを有効にします。

<div id="email-providers">
  ## メールプロバイダー
</div>

Auth0 はデフォルトで独自の SMTP プロバイダーを使用してメールを送信します。Auth0 に組み込まれているメールプロバイダーはテスト専用であり、メールテンプレートのカスタマイズには対応していません。

[独自の SMTP メールプロバイダーを設定](/docs/ja/customize/email/smtp-email-providers)すると、メール送信の管理、監視、トラブルシューティングに加え、メールテンプレートのカスタマイズも行えます。

Auth0 では、次のメールプロバイダーをサポートしています。

* [Mandrill](/docs/ja/customize/email/smtp-email-providers/configure-mandrill-as-external-smtp-email-provider)
* [Amazon SES](/docs/ja/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider)
* [Azure Communication Services](/docs/ja/customize/email/smtp-email-providers/configure-azure-comm-service-as-smtp-email-provider)
* [SendGrid](/docs/ja/customize/email/smtp-email-providers/configure-sendgrid-as-external-smtp-email-provider)
* [SparkPost](/docs/ja/customize/email/smtp-email-providers/configure-sparkpost-as-external-smtp-email-provider)
* [Mailgun](/docs/ja/customize/email/smtp-email-providers/configure-mailgun-as-external-smtp-email-provider)
* [カスタム SMTP 外部メールプロバイダー](/docs/ja/customize/email/smtp-email-providers/configure-custom-external-smtp-email-provider)
