title: SSOCircle を SAML ID プロバイダーとして設定する
---

以下の手順で、SSOCircle を <Tooltip tip="Identity Provider（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+provider">ID プロバイダー</Tooltip>として設定します。

1. SSOCircle のメタデータを取得します。
2. Auth0 でエンタープライズ接続を設定します。
3. SSOCircle を ID プロバイダーとして設定します。
4. 接続をテストします。

<div id="prerequisite">
  ## 前提条件
</div>

[SSOCircle](https://idp.ssocircle.com/) のアカウントが必要です。

<div id="obtain-ssocircle-metadata">
  ## SSOCircle のメタデータを取得する
</div>

1. 連携に必要なメタデータを確認するには、[SSOCircle の IDP ページ](https://idp.ssocircle.com/)にアクセスします。XML ファイルが表示されます。
2. このページから、次の属性を保存します。

   1. `HTTP-Redirect` タイプの `SingleSignOnService` 属性の Location URL
   2. `HTTP-Redirect` タイプの `SingleLogoutService` 属性の Location URL
3. [SSOCircle CA 証明書をダウンロード](http://www.ssocircle.com/en/idp-tips-tricks/public-idp-configuration)して保存します。

<div id="configure-enterprise-connection-in-auth0">
  ## Auth0 でエンタープライズ接続を設定する
</div>

1. [Dashboard &gt; Authentication &gt; Enterprise &gt; SAMLP](https://manage.auth0.com/#/connections) に移動し、**プラス**アイコンをクリックして、新しい接続を作成するページを開きます。
2. この接続に必要な設定を行います。必須項目は次のとおりです。

| 設定         | 説明                                                                                                     | 例                                                                                                                                   |
| ---------- | ------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------- |
| 接続名        | Auth0 の接続名。                                                                                            | `myoktaconnection`                                                                                                                  |
| サインイン URL  | ユーザーのログインリクエストの送信先となる Okta の URL。<br /><br />これは、先ほど控えた **Identity Provider Single Sign-On URL** の値です。 | `https://my_okta_tenant_name.okta.com/app/` `my_okta_tenant_namemy_okta_saml_app_integration_name/` `dakflkbzevu5i5zBi939/sso/saml` |
| X509 署名証明書 | Okta テナントの公開鍵署名証明書。<br /><br />先ほどダウンロードした **X509 Certificate** をアップロードします。                            | `myOktaTenantSigningCertificate.pem`                                                                                                |

3. マッピングを設定する際は、以下の JSON を使用して SSOCircle の SAML 属性を適切にマッピングします。

   ```json lines
   {
         "email": "EmailAddress",
         "given_name": "FirstName",
         "family_name": "LastName"
       }
   ```

4. **Save** をクリックします。次の画面で、2 つの選択肢が表示されます。

   1. ドメイン管理者の場合は、**Continue** をクリックして、SAML ID プロバイダーの設定に関する追加手順を確認します。
   2. ドメイン管理者でない場合は、表示された URL をドメイン管理者に渡して、設定を完了してもらうことができます。

次の形式の URL を使って、Auth0 の<Tooltip tip="Security Assertion Markup Language (SAML): パスワードを使用せずに、2者間で認証情報を交換できるようにする標準化されたプロトコル。" cta="用語集を見る" href="/docs/ja/glossary?term=SAML">SAML</Tooltip>接続のメタデータにアクセスできます。
`https://{yourDomain}/samlp/metadata?connection={yourConnectionName}`。

次の手順では、このメタデータを SSOCircle に提供する必要があります。

<div id="configure-ssocircle-as-identity-provider">
  ## SSOCircle を ID プロバイダーとして設定する
</div>

1. [SSOCircle](http://ssocircle.com) アカウントにログインします。ユーザープロファイルが表示され、その左側にナビゲーションバーがあります。
2. **Manage Metadata** をクリックします。
3. **Add New Service Provider** を選択し、以下の情報を入力して新しいサービスプロバイダーを設定します。この場合、サービスプロバイダーは Auth0 です。

| Setting                               | Description                             |
| ------------------------------------- | --------------------------------------- |
| **FQDN of the ServiceProvider**       | `auth0.com`                             |
| **Attributes to send in assertion**   | `EmailAddress` にチェックを入れます               |
| **Insert your metadata information:** | Auth0 接続の設定後にダウンロードした XML メタデータを貼り付けます。 |

4. **Submit** をクリックします。

<div id="test-connection">
  ## 接続をテスト
</div>

<div id="create-auth0-application">
  ### Auth0 アプリケーションを作成
</div>

1. [Auth0 Dashboard](https://manage.auth0.com/#) の **アプリケーション** ページに移動し、**+ New Application** をクリックします。
2. 新しいアプリケーションの基本情報を入力します。アプリケーションの種類には **Regular Web Applications** を選択します。
3. **Create** をクリックして設定を完了し、アプリケーションの作成を開始します。アプリケーションの詳細ページに移動します。
4. **Settings** をクリックします。
5. **Allowed Callback URL** フィールドに、許可するコールバック URL の一覧を入力します。ここには、認証後にユーザーがリダイレクトされる URL を含めます。ここで入力する URL は、後の手順で作成する HTML コード内の **callback URL** と一致している必要があります。通常はアプリケーションの URL を入力しますが、この例では簡単にするため、ユーザーは Auth0 JWT Tool にリダイレクトされます。
6. **SAVE CHANGES** をクリックします。
7. **Settings** の先頭に戻り、**Connections** をクリックします。
8. **Enterprise** セクションまでスクロールします。先ほど作成した SAML 接続の行を見つけ、トグルをオンにして SAML 接続を有効にします。

<div id="test-enterprise-connection">
  ### エンタープライズ接続をテストする
</div>

1. 接続をテストするには、[Test Enterprise Connections](/docs/ja/authenticate/identity-providers/enterprise-identity-providers/test-enterprise-connections) の手順に従ってください。
2. この手順の途中で、ログインと同意を求められます。
3. また、「Your session has timed out」というメッセージが表示された場合は、そのメッセージの下にある **Return to Login page** リンクをクリックしてください。

アプリケーションへのログインが最初はうまくいかない場合は、再度テストする前にブラウザーの履歴と Cookie を削除してください。ブラウザーで HTML の最新バージョンが反映されていない可能性があります。

<Tooltip tip="Single Sign-On (SSO): ユーザーが1つのアプリケーションにログインした後、そのユーザーを他のアプリケーションにも自動的にログインさせるサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=SSO">SSO</Tooltip> のトラブルシューティングでは、通信の HTTP トレースを取得すると役立つことがよくあります。分析用にブラウザーの HTTP トラフィックを取得できるツールは数多くあります (ニーズに合ったものを見つけるには「HTTP Trace」で検索してください) 。HTTP トレーサーを用意したら、ログインの一連の流れを最初から最後まで取得し、トレースを分析して `GET` リクエストの流れを確認し、どこでエラーが発生しているかを調べてください。次のような流れが確認できるはずです。

* 元のサイトから IDP へのリダイレクト
* 認証情報の送信 (ログインを求められた場合)
* コールバック URL へのリダイレクト

ブラウザーで Cookie と JavaScript が有効になっていることを確認してください。

HTML で指定したコールバック URL が、Auth0 Application の **Settings** タブにある **Allowed Callback URLs** フィールドにも登録されていることを確認してください。
