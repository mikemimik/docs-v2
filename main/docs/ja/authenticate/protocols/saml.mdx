<Tooltip tip="Security Assertion Markup Language（SAML）: パスワードを使わずに 2 者間で認証情報をやり取りできる標準化されたプロトコル。" cta="用語集を見る" href="/docs/ja/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) プロトコルは、2 つの主体間でパスワードを使わずに認証と認可を行うための、オープン標準の XML ベースのフレームワークです。

* **サービスプロバイダー** (SP) は、ユーザー認証を <Tooltip tip="Security Assertion Markup Language（SAML）: パスワードを使わずに 2 者間で認証情報をやり取りできる標準化されたプロトコル。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+provider">identity provider</Tooltip> に委ね、その認証結果を信頼します。
* **Identity provider** (IdP) はユーザーを認証し、そのユーザーが認証済みであることを示す認証アサーションをサービスプロバイダーに提供します。

Auth0 は SAML プロトコルをサポートしており、IdP、SP、またはその両方として次の用途で利用できます。

* SAML2 Web アプリケーション
* SAML <Tooltip tip="Single Sign-On（SSO）: ユーザーが 1 つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインされるサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=SSO">SSO</Tooltip> 連携
* IdP 開始型 SSO
* Microsoft Active Directory Federation Services (ADFS)
* SAML リクエストの署名と暗号化

<div id="supported-saml-bindings-and-options">
  ## サポートされている SAML バインディングとオプション
</div>

Auth0 では、次の SAML バインディングをサポートしています。

* HTTP Redirect
* HTTP POST

Auth0 では、次の SAML オプションをサポートしています。

* Web Browser SSO Profile
* Single Logout Profile
* Name Identifier Management Profile
* Name Identifier Mapping Profile
