[Auth0 Universal Login](/docs/ja/authenticate/login/auth0-universal-login) は、<Tooltip tip="Authorization Server: ユーザーのアクセス範囲の定義に寄与する集中管理サーバーです。たとえば、認可サーバーはユーザーが利用できるデータ、タスク、機能を制御できます。" cta="用語集を見る" href="/docs/ja/glossary?term=authorization+server">認可サーバー</Tooltip>の中核機能であるログインフローを提供します。ユーザーがアプリケーションへのアクセスのために本人確認を行う必要がある場合は、そのユーザーを<Tooltip tip="Authorization Server: ユーザーのアクセス範囲の定義に寄与する集中管理サーバーです。たとえば、認可サーバーはユーザーが利用できるデータ、タスク、機能を制御できます。" cta="用語集を見る" href="/docs/ja/glossary?term=Universal+Login">Universal Login</Tooltip> にリダイレクトし、Auth0 に認証プロセスを処理させることができます。

Universal Login は、Auth0 の主要なホスト型ログインソリューションです。[Universal Login](/docs/ja/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) には使いやすいカスタマイズツールが備わっており、エンドユーザーによりシンプルでスピーディーな体験を提供します。一方、Auth0 は [Classic Login](/docs/ja/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) もサポートしています。これは、カスタマイズに JavaScript コントロールを使用するホスト型ログインエクスペリエンスです。

<Warning>
  現時点では、Auth0 の継続的な開発は Universal Login に注力されており、Classic Login はすでに更新されていません。特別な要件で Classic エクスペリエンスが必要な場合を除き、Universal Login の実装を推奨します。
</Warning>
