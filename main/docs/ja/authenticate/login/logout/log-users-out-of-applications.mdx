description: Auth0のlogout endpointを使用して、ユーザーをアプリケーションから強制的にログアウトさせる方法について説明します。
title: ユーザーをアプリケーションからログアウトさせる
---

エンタープライズユーザーは通常、複数のアプリケーション (例: SharePoint、いくつかの .NET アプリケーション、いくつかの Java アプリケーション、Zendesk) で <Tooltip tip="シングルサインオン（SSO）: ユーザーが1つのアプリケーションにログインすると、そのユーザーは他のアプリケーションにも自動的にログインされる仕組み。" cta="用語集を見る" href="/docs/ja/glossary?term=Single+Sign-on">シングルサインオン</Tooltip> (SSO) が有効になっています。この場合、ユーザーがサインアウトする際は、多くの場合、利用中のすべてのアプリケーションからもサインアウトさせる必要があります。

Auth0 の [RP-initiated logout endpoint](/docs/ja/authenticate/login/logout/log-users-out-of-auth0) は、次の2つのいずれかの方法で動作します。

* Auth0 のシングルサインオン (SSO) Cookie を無効にします。 (Cookie 自体はブラウザーに残ります。)
* ユーザーを <Tooltip tip="アイデンティティプロバイダー（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/docs/ja/glossary?term=identity+provider">アイデンティティプロバイダー</Tooltip> (IdP)  (ADFS や Google など) からサインアウトさせます。

ユーザーを logout endpoint にリダイレクトするだけでは、ユーザーが利用していたすべてのアプリケーションからサインアウトさせる必要があるケースには対応できません。この機能が必要な場合は、次の2つの方法のいずれかで対応する必要があります。

* ローカルセッションのタイムアウトを短く設定し、短い間隔で Auth0 にリダイレクトして再認証します。これは、クライアントから `checkSession` を呼び出すことで実行できます。このリダイレクトは非表示の iFrame で行われます。非表示の iFrame を使用する場合は、[レート制限](/docs/ja/troubleshoot/customer-support/operational-policies/rate-limit-policy) とサードパーティの [cookie](/docs/ja/manage-users/cookies) に関する問題に注意する必要があります。
* ログアウトが発生したときに、他のすべてのアプリケーションに通知する手段を用意し、これを完全にアプリケーションレベルで処理します。

代替の logout endpoint を使用している場合は、[API documentation](https://auth0.com/docs/api/authentication#logout) を確認してください。
