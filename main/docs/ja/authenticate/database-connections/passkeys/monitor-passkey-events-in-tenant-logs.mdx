description: Auth0 Dashboard または Management API でパスキー固有のイベントコードを使って絞り込むことで、Auth0 テナントのテナントログ内でパスキーの登録イベントと認証イベントを追跡できます。
title: テナントログでパスキーイベントを監視する
---

[テナントログ](/docs/ja/deploy-monitor/logs)で、データベース接続に関連付けられたパスキーのフローを監視できます。

<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ([Monitoring &gt; Logs](https://manage.auth0.com/#/logs)) でログイベントを確認するか、[Management API](https://auth0.com/docs/api/management/v2/logs/get-logs) を使用してログを取得できます。

| イベントコード                         | シナリオ                                | 注記                                                                                                                                                                |
| ------------------------------- | ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ss`                            | Signup 中に、ユーザーが主要な認証方法としてパスキーを登録した。 | このログイベントには、`passkey` というエントリー値を含む `details.authentication_methods` 配列フィールドが含まれます。<br /><br />ユーザーがパスワードで登録した場合、この配列フィールドには `pwd` というエントリー値が含まれます。                 |
| `s`                             | login 中に、ユーザーがパスキーで認証した。            | このログイベントには `details.prompts` 配列が含まれており、login 中にユーザーがたどった prompt の一覧が記録されます。<br /><br />ユーザーがパスキーを使用して認証した場合、この配列には次のフィールドを持つ要素が含まれます: `"performed_amr": ["phr"]`。 |
| `gd_enrollment_complete`        | 段階的またはローカルの登録フローが正常に完了した。           | ユーザーがパスキーを登録した場合、このログイベントには値 `passkey` を持つ `details.authenticator.type` フィールドが含まれます。                                                                              |
| `gd_webauthn_enrollment_failed` | 段階的またはローカルの登録フロー中にエラーが発生した。         | ユーザーがパスキーを登録しようとした場合、このログイベントには `passkey` というエントリー値を含む `details.authentication_methods array` フィールドが含まれます。                                                        |
