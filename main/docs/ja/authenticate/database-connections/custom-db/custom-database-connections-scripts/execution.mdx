カスタムデータベース接続タイプでは、アクションスクリプトを設定できます。アクションスクリプトには、Auth0 がレガシーアイデンティティストアと連携するために使用するカスタムコードが含まれます。アクションスクリプトは、あらかじめ定義された一連のパラメータを受け取る、名前付きの JavaScript 関数です。

<div id="webtask-containers">
  ## Webtask コンテナー
</div>

アクションスクリプトは、それぞれ独立した Webtask コンテナー内で実行され、実行時間の上限は約 20 秒です。関数の実行後、コンテナーは再利用されます。

コンテナーが再利用されると、アクションスクリプトで保留中の処理は終了します。その結果、エラーが返される可能性があり、`global` オブジェクトがリセットされることもあります。`global` オブジェクトの詳細については、[Custom Database アクションスクリプト環境のベストプラクティス](/docs/ja/authenticate/database-connections/custom-db/custom-database-connections-scripts/environment)を参照してください。

<div id="asynchronous-features">
  ## 非同期機能
</div>

[Promise](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise)オブジェクトや[async functions](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/async_function)を含むJavaScriptの非同期機能は、アクションスクリプトでサポートされています。

非同期機能を使用すると、アクションスクリプト内で非ブロッキング処理を実行できます。ただし、これらの処理がWebtask コンテナーの実行時間制限を超えないようにしてください。Webtask コンテナーが再利用されると、保留中の処理はすべて終了されるため、予期しない動作やエラーが発生する可能性があります。

アクションスクリプト内で外部サービスまたはAPIを呼び出す場合は、関数が適切な時間でタイムアウトするように設定し、外部サービスまたはAPIにアクセスできない場合は[エラーを返す](#error-handling)ようにしてください。

<div id="examples">
  ### 例
</div>

Auth0 では、アクションスクリプト内で [Promise](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) オブジェクトと [async functions](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/async_function) を使用できます。

<div id="promise-object">
  #### Promise オブジェクト
</div>

この例では、組み込みの JavaScript の `fetch` メソッドを使用します。これはネットワークからリソースを取得し、Promise オブジェクト を返します。記述には [promise chains](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Using_promises#chaining) を使用しています。

```javascript lines expandable
function login(userNameOrEmail, password, callback) {
	const hashedPassword = hash(password);
	const apiEndpoint = 'https://example.com/api/authenticate';
	const options = {
		method: 'POST',
		body: {
			email: userNameOrEmail,
			password: hashedPassword
		}
	};

	fetch(apiEndpoint, options) 
		.then((response) => {
			if (!response.ok) {
				return callback(new Error(`HTTP error! Status: ${response.status}`));
			}

			return response.json();
		})
		.then((response) => {
			if (response.err) {
				return callback(new Error(`Error authenticating user: ${err}`));
			}

			let profile = {
				email: response.profileData.email,
				username: response.profileData.username
			};

			return callback(null, profile);
		})
		.catch((err) => {
			return callback(new Error(`An error occurred: ${err}`));
		});
  }
```

<div id="async-function">
  #### Async 関数
</div>

この例では、組み込みのJavaScript `fetch` メソッドを使用しています。これはネットワークからリソースを取得し、その後 Promise オブジェクトを返すもので、[async functions](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/async_function) を使って記述されています。

```js lines expandable
async function login(userNameOrEmail, password, callback) {
	const hashedPassword = hash(password);
	const apiEndpoint = 'https://example.com/api/authenticate';
	const options = {
		method: 'POST',
		body: {
			email: userNameOrEmail,
			password: hashedPassword
		}
	};

	const response = await fetch(apiEndpoint, options);

	if (!response.ok) {
		return callback(new Error(`HTTP error! Status: ${response.status}`));
	}

	const result = response.json();

	if (result.err) {
		return callback(new Error(`Error authenticating user: ${err}`));
	}

	let profile = {
		email: response.profileData.email,
		username: response.profileData.username
	};

	return callback(null, profile);
}
```

<div id="callback-function">
  ## `callback` 関数
</div>

`callback` 関数は、アクションスクリプトの処理が完了したことを示すもので、必ず1回だけ呼び出す必要があります。アクションスクリプトは、`callback` 関数を呼び出したら、できれば `return` 文を明示的に使って、すぐに終了するようにしてください。

<Warning>
  `callback` 関数が複数回呼び出されると、予期しない結果やエラーが発生する可能性があります。

  `callback` 関数がまったく呼び出されない場合、アクションスクリプトの実行は停止したままとなり、Webtask コンテナが再利用される際にエラーが返されます。
</Warning>

<div id="asynchronous-processing">
  ### 非同期処理
</div>

アクションスクリプトで非同期処理を使用する場合は、すべての非同期操作が完了してから `callback` 関数を呼び出す必要があります。

<div id="parameters">
  ### パラメータ
</div>

`callback` 関数 がパラメータなしで呼び出された場合は、`null` が渡されたものとして実行されます。

<div id="size">
  ## サイズ
</div>

アクションスクリプトの実装全体のサイズは、100kB を超えてはなりません。このサイズ制限には、インポートした `npm` モジュールは含まれません。`npm` モジュールの詳細については、[Custom Database アクションスクリプト環境のベストプラクティス](/docs/ja/authenticate/database-connections/custom-db/custom-database-connections-scripts/environment)を参照してください。

スクリプトのサイズが大きいほど、Webtask プラットフォームで行われるパッケージ化と転送の処理により、レイテンシが大きくなります。サイズはシステムのパフォーマンスに影響します。

<div id="anonymous-functions">
  ## 無名関数
</div>

アクションスクリプトは無名関数として実装することもできますが、推奨されません。無名関数を使用すると、アクションスクリプトのデバッグや、例外発生時に生成されるコールスタックの解釈が難しくなります。無名関数の詳細については、[MDN Web Docs の IIFE](https://developer.mozilla.org/en-US/docs/Glossary/IIFE)を参照してください。

<div id="error-handling">
  ## エラー処理
</div>

エラー内容を説明するメッセージを含む `Error` オブジェクトを `callback` 関数に渡します。

```js lines
return callback(new Error('My custom error message'));
```

<div id="security">
  ## セキュリティ
</div>

<div id="database-interface-vs-api">
  ### データベースインターフェースと API
</div>

Auth0 とレガシーアイデンティティストア間のすべての通信が安全に保護されていることを確認してください。レガシーアイデンティティストアにまだ API が実装されていない場合は、実装を強く推奨します。

レガシーアイデンティティストアで API を利用できる場合は、Auth0 で [API を登録](/docs/ja/get-started/auth0-overview/set-up-apis)し、[Action を作成](/docs/ja/manage-users/access-control/sample-use-cases-actions-with-authorization#deny-access-to-anyone-calling-an-api)して、エンドユーザーからのアクセスを制限できます。

レガシーアイデンティティストアで API を利用できず、かつ実装も難しい場合でも、データベースと直接通信することは可能です。Auth0 からの受信トラフィックを許可するため、[Auth0 の IP アドレスをファイアウォールの allow list に追加](/docs/ja/secure/security-guidance/data-security/allowlist)してください。

<div id="identity-provider-tokens">
  ## アイデンティティプロバイダーのトークン
</div>

`user` オブジェクトが `access_token` と `refresh_token` プロパティを返す場合、Auth0 はそれらを他のユーザー情報とは異なる形で扱います。Auth0 はそれらを `user` オブジェクトの `identities` プロパティに格納します。

```json lines
{
	"email": "you@example.com",
	"updated_at": "2019-03-15T15:56:44.577Z",
	"user_id": "auth0|some_unique_id",
	"nickname": "a_nick_name",
	"identities": [ 
		{
			"user_id": "some_unique_id",
			"access_token": "e1b5.................92ba",
			"refresh_token": "a90c.................620b",
			"provider": "auth0", 
			"connection": "custom_db_name",
			"isSocial": false 
		}
  ], 
  "created_at": "2019-03-15T15:56:44.577Z",
  "last_ip": "192.168.1.1",
  "last_login": "2019-03-15T15:56:44.576Z",
  "logins_count": 3
}
```

これらのプロパティのいずれかを Auth0 の <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Management+API">Management API</Tooltip> で取得するには、[アクセストークンをリクエストする](/docs/ja/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production)際に `read:user_idp_tokens` スコープを含めてください。
