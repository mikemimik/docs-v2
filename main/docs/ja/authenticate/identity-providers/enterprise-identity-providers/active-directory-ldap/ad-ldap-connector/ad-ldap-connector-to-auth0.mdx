---
description: AD/LDAP Connector インスタンスを Auth0 の新しい接続に向ける方法を説明します。
title: AD/LDAP Connector の接続先を Auth0 の接続に変更する
---

たとえば、新しい Auth0 テナントに移行した場合や、Auth0 の接続名を変更した場合には、AD/LDAP Connector インスタンスの接続先を Auth0 の新しい接続に変更する必要があることがあります。Auth0 では接続名を変更できないため、新しい AD/ LDAP 接続を作成し、既存の Connector インスタンスの接続先をその接続に変更する必要があります。

1. Auth0 Dashboard で新しい AD/LDAP 接続を作成し、生成された **TICKET URL** をコピーします。カスタムドメインを使用している場合は、**TICKET URL** の `{yourDomain}` の部分を `identity.fabrikam.com` のようなカスタムドメインに置き換える必要があります。
2. AD/LDAP Connector ホスト上で Connector Admin アプリを開き、**Import / Export** タブから既存の設定をエクスポートします。これは、以降の手順で誤ってカスタム設定が上書きされた場合に備えるための予防措置です。Web ブラウザーがなく Connector Admin Web サイトにアクセスできないホストで Connector を実行している場合は、`config.json` ファイルをコピーしておくだけで構いません。
3. AD/LDAP Connector ホストで `config.json` ファイルを編集し、`PROVISIONING_TICKET` プロパティの値を手順 1 でコピーした **TICKET URL** に変更します。
4. ある Auth0 テナントから別のテナントに移行した場合は、`config.json` ファイル内の `urn:auth0:{oldAuth0TenantName}` という名前のプロパティを削除します。これを削除しなくても Connector は引き続き動作しますが、この古い設定データは不要です。
5. AD/LDAP Connector サービス (Windows では **Auth0 ADLDAP** サービス) を再起動します。
6. Connector のログ (Connector Admin ツールの **Troubleshooting** タブ、または `logs.log` ファイルを tail して確認) を見て、次のような最近のエントリがあることを確認します。`2016-03-10T22:47:32.970Z - debug: [2016-03-10 22:47:32] Loading settings from ticket: {yourTicketUrl}/info`
7. Auth0 Dashboard で、新しい AD/LDAP 接続が接続済みとして表示されていることを確認します (新しい接続の左側の点が赤ではなく緑になっています) 。
8. 新しい接続を使ってテスト認証を実行し、Connector のログにもアクティビティが表示されることを確認します。
