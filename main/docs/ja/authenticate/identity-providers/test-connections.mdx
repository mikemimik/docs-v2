通常、接続をテストするには、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/docs/ja/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> にログインする必要があります。

1. [Auth0 Dashboard](https://manage.auth0.com/#) に移動し、**Authentication** を選択してから、テストする接続の種類を選択します。
2. テストする接続を見つけ、**More Options** メニュー (**...**) から **Try Connection** を選択します。
3. アイデンティティプロバイダーを使ってログインします。
4. 結果が表示される **It Works!** ページが表示されるまで待ちます。

Auth0 はアプリケーションとして認証フローをシミュレートし、認証に成功した結果として得られる User Profile を表示します。

<div id="test-partner-connections">
  ## パートナー接続をテストする
</div>

他者が所有する接続をテストする際に、その相手先のテスト用認証情報を持っていない場合は、Auth0 Dashboard にログインした状態ではテストできません。これは、<Tooltip tip="Security Assertion Markup Language（SAML）: パスワードなしで 2 者間の認証情報の交換を可能にする標準化されたプロトコル。" cta="用語集を見る" href="/docs/ja/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language（SAML）: パスワードなしで 2 者間の認証情報の交換を可能にする標準化されたプロトコル。" cta="用語集を見る" href="/docs/ja/glossary?term=Identity+Providers">アイデンティティプロバイダー</Tooltip> (IdPs) や Active Directory などのエンタープライズ接続に接続する場合によくあります。では、どうすればよいのでしょうか。パートナーに新しい接続をテストしてもらいます。

1. テスト用アプリを登録します。

   * [Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications) に移動し、**Create Application** を選択します。名前は自由に付けられます (たとえば `Test App`) 。
   * 新しく作成したアプリの設定で、**Allowed Callback URLs** を `http://jwt.io` に設定します。
   * **Save Changes** を選択します。
2. ログイン用リンクをパートナーに送信します。

   <AuthCodeBlock children={codeExample} language="http" />

   次の 2 つのパラメーターを置き換えます。

   * `client_id`: ステップ 1 で作成したアプリケーションの Client ID。
   * `connection`: テストする接続の名前。
3. 接続をテストします。パートナーがこのリンクを開くと、設定されているアイデンティティプロバイダー (接続先) にリダイレクトされます。認証に成功すると、[JWT.io](http://jwt.io) に戻り、すべてのユーザープロパティがトークンからデコードされます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  このテスト用アプリは実際のアプリではありません。[JWT.io](http://jwt.io) は、URL フラグメントで送信されたトークンをデコードするテスト用サイトです。
</Callout>
