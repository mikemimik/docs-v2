シナリオは次のとおりです。ユーザーの認証後、その資格情報を保存し、有効な間はそれを使い続けたいとします。期限切れになった場合は、ログインページを再度表示しなくて済むよう、`refreshToken` を使って更新したいところです。ただし、これを自動で行うのではなく、ユーザーに指紋または顔で本人確認してもらう必要があります。

[Auth0.swift](https://github.com/auth0/Auth0.swift/) の [Credentials Manager](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) ユーティリティを使用して、ユーザーの資格情報の管理を効率化し、生体認証を行います。

<div id="getting-started">
  ## はじめに
</div>

まず、`Auth0` モジュールをインポートします。

`import Auth0`

<div id="credentials-manager">
  ### Credentials Manager
</div>

資格情報を取得する前に、iOS デバイスでサポートされている生体認証 (Face ID または Touch ID) を利用することもできます。

まず Credentials Manager を設定し、次に生体認証を有効にします。プロンプトに表示するタイトルを渡すこともできます。

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Face ID を使用する理由を表示するため、プロジェクトの `Info.plist` に [NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) 設定を追加することを強く推奨します。場合によっては、説明文を指定していない状態でユーザーが Face ID 認証を試みると、その試行が失敗することがあります。

```xml lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

<div id="login">
  ### Login
</div>

<Tooltip tip="Universal Login: ユーザーの本人確認を行うため、アプリケーションは Auth0 の認可サーバーでホストされている Universal Login にリダイレクトされます。" cta="用語集を見る" href="/docs/ja/glossary?term=Universal+Login">Universal Login</Tooltip> ページを表示し、認証に成功したら資格情報を Credentials Manager に渡します。

export const codeExample = `Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // エラーを処理する
        case .success(let credentials):
            // Credentials Manager を使って資格情報を安全に保存する
            credentialsManager.store(credentials: credentials)
        }
}`;

<div id="renew-user-credentials">
  ### ユーザー資格情報の更新
</div>

ユーザーの資格情報を更新する必要がある場合は、Credentials Manager の `credentials` メソッドを呼び出せます。

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // エラーを処理する
        // ログイン画面にフォールバックする
    }
    // 認証が成功したため、ユーザーのルーティングを続行する
}
```

更新時にはCredentials Managerが自動的に処理するため、新しい資格情報を手動で保存する必要はありません。
