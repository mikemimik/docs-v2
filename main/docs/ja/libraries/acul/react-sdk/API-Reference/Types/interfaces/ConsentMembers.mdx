ConsentMembers

````ts Example
export interface ConsentMembers extends BaseMembers {
  /**
   * Provides access to the specific properties and data of the 同意画面,
   * including the list of `scopes` being requested and the `hideScopes` flag.
   * @type {ScreenMembersOnConsent}
   */
  screen: ScreenMembersOnConsent;

  /**
   * リクエストされた権限を承認（グラント）するユーザーの決定を送信します。
   * このアクションは `action: "accept"` を付けて `/u/consent` エンドポイントにPOSTします。
   *
   * @param {CustomOptions} [payload] - 省略可能。リクエストと共に送信するカスタムキーと値のペアを含むオブジェクト。
   *                                    これらのパラメータはサーバーに送信されるフォームデータに含まれます。
   * @returns {Promise<void>} 承認アクションが正常に送信されたときに解決されるPromise。
   *                          成功した場合、Auth0は通常ユーザーをアプリケーションに戻すか、
   *                          認証フローの次のステップにリダイレクトします。
   * @throws {Error} フォームの送信に失敗した場合（例：ネットワークエラー、invalid state）にエラーをスローします。
   *                 サーバーサイドのエラー（"invalid_request" など）は、JavaScriptエラーとしてスローされるのではなく、
   *                 操作後に `this.transaction.errors` に反映されます。
   *
   * @example
   * ```typescript
   * // Assuming 'consentManager' is an instance of the Consent screen SDK class
   * try {
   *   await consentManager.accept();
   *   // If successful, the page will typically redirect.
   * } catch (error) {
   *   // Handle unexpected errors during the submission itself.
   *   console.error("Failed to submit consent acceptance:", error);
   * }
   * // After the await, check consentManager.transaction.errors for server-side validation issues.
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * Submits the user's decision to deny (reject) the 権限.
   * This action posts to the `/u/consent` endpoint with `action: "deny"`.
   *
   * @param {CustomOptions} [payload] - Optional. An object for any custom key-value pairs
   *                                    to be sent with the request. These parameters will be
   *                                    included in the form data submitted to the server.
   * @returns {Promise<void>} A promise that resolves when the deny action is successfully submitted.
   *                          On success, Auth0 typically redirects the user, potentially showing an
   *                          access denied message or returning an error to the application.
   * @throws {Error} Throws an error if the form submission fails (e.g., network issue, invalid state).
   *                 Server-side errors are reflected in `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // Assuming 'consentManager' is an instance of the Consent screen SDK class
   * try {
   *   await consentManager.deny({ reason: "user_declined" }); // Example custom option
   *   // If successful, the page will typically redirect.
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // After the await, check consentManager.transaction.errors for server-side validation issues.
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}
````

<ParamField body="accept" type="Promise<void>">
  要求された権限を受け入れる (許可する) というユーザーの決定を送信します。
  このアクションは、`action: "accept"` を指定して `/u/consent` エンドポイントに送信します。

  accept アクションが正常に送信されると解決される Promise。
  成功すると、Auth0 は通常、ユーザーをアプリケーションにリダイレクトするか、
  認証フローの次のステップに進めます。

  #### Throws

  フォームの送信に失敗した場合 (例: ネットワークの問題、invalid state) にエラーをスローします。
  サーバー側のエラー (&quot;invalid&#95;request&quot; など) は、JavaScript のエラーとしてスローされるのではなく、
  操作後に `this.transaction.errors` に反映されます。

  ```typescript Example
  // 'consentManager' が 同意画面 SDK クラスのインスタンスであると仮定します
  try {
    await consentManager.accept();
    // 成功した場合、通常はページがリダイレクトされます。
  } catch (error) {
    // 送信処理自体で発生した予期しないエラーを処理します。
    console.error("Failed to submit consent acceptance:", error);
  }
  // await の後で、サーバー側のバリデーションの問題について consentManager.transaction.errors を確認します。
  ```

<ParamField body="deny" type="Promise<void>">
  要求された権限を拒否するというユーザーの決定を送信します。
  このアクションは、`action: "deny"` を指定して `/u/consent` エンドポイントに送信します。

  deny アクションが正常に送信されると解決される Promise。
  成功すると、Auth0 は通常ユーザーをリダイレクトし、アクセス拒否メッセージを表示するか、
  アプリケーションにエラーを返す可能性があります。

  #### Throws

  フォームの送信に失敗した場合 (例: ネットワークの問題、invalid state) にエラーをスローします。
  サーバー側のエラーは `this.transaction.errors` に反映されます。

  ```typescript Example
  // 'consentManager' が 同意画面 SDK クラスのインスタンスであると仮定します
  try {
    await consentManager.deny({ reason: "user_declined" }); // カスタムオプションの例
    // 成功した場合、通常はページがリダイレクトされます。
  } catch (error) {
    console.error("Failed to submit consent denial:", error);
  }
  // await の後で、サーバー側のバリデーションの問題について consentManager.transaction.errors を確認します。
  ```
