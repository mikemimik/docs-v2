````ts Example
export interface MfaWebAuthnPlatformChallengeMembers extends BaseMembers {
  /**
   * 画面固有のプロパティとデータ。WebAuthnチャレンジオプションおよびデバイスを記憶する設定を含みます。
   * @type {ScreenMembersOnMfaWebAuthnPlatformChallenge}
   */
  screen: ScreenMembersOnMfaWebAuthnPlatformChallenge;

  /**
   * WebAuthnプラットフォーム認証器のチャレンジを開始します。
   * このメソッドは内部的に、`screen.publicKey` で提供されたチャレンジオプションを使用して
   * `navigator.credentials.get()` を呼び出します。
   * 成功した場合、`action: "default"` とともに結果の資格情報をAuth0に送信します。
   *
   * `navigator.credentials.get()` が失敗した場合（例：ユーザーによるキャンセル、タイムアウト、またはその他のWebAuthn APIエラー）、
   * エラーの詳細を指定して `reportBrowserError` を呼び出すことを推奨します。
   *
   * @param {VerifyPlatformAuthenticatorOptions} [options] - 検証プロセスのオプションパラメーター。
   * `rememberDevice`（`screen.showRememberDevice` が true の場合）やその他のカスタムオプションを指定できます。
   * @returns {Promise<void>} 検証の試行が送信されたときに解決されるPromise。
   *                          操作が成功した場合、通常はリダイレクトが発生します。
   * @throws {Error} `screen.publicKey` が存在しない場合、
   *                 `navigator.credentials.get()` が `getPasskeyCredentials` で処理されない予期しないエラーで失敗した場合、
   *                 またはAuth0へのフォーム送信が失敗した場合にエラーをスローします。
   *
   * @example
   * ```typescript
   * // 'sdk' は MfaWebAuthnPlatformChallenge のインスタンスと仮定します
   * try {
   *   const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
   *   await sdk.verify({ rememberDevice: remember });
   *   // 成功時、Auth0がリダイレクトを処理します。
   * } catch (error) {
   *   console.error("Platform authenticator verification failed:", error);
   *   // WebAuthn APIエラーの場合は報告します
   *   if (error.name && error.message) { // DOMException的なエラーの基本チェック
   *     await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
   *   }
   *   // ページが再読み込みされた場合、sdk.transaction.errors でサーバー側のバリデーションメッセージを確認してください。
   * }
   * ```
   */
  verify(options?: VerifyPlatformAuthenticatorOptions): Promise<void>;

  /**
   * 特定のWebAuthn APIエラー（`navigator.credentials.get()` からのもの）をAuth0に報告します。
   * このメソッドは、`verify()`（または手動の `navigator.credentials.get()` 呼び出し）がブラウザ側の
   * WebAuthnの問題（例：ユーザーによるキャンセル `NotAllowedError`、タイムアウト）により失敗した場合に使用します。
   * `action: "showError::{errorDetailsJsonString}"` と空の `response` とともにエラーの詳細を送信します。
   *
   * @param {ReportBrowserErrorOptions} options - WebAuthn API DOMException の `name` と `message` を持つ
   * `error` オブジェクト、およびその他のカスタムオプションを含みます。
   * @returns {Promise<void>} エラーレポートが送信されたときに解決されるPromise。
   * @throws {Error} フォームの送信が失敗した場合（例：ネットワークエラー、無効な状態）にエラーをスローします。
   *
   * @example
   * ```typescript
   * // 'sdk' は MfaWebAuthnPlatformChallenge のインスタンスと仮定します
   * // webAuthnDomException は失敗した navigator.credentials.get() 呼び出しからのエラーオブジェクトです。
   * try {
   *   await sdk.reportBrowserError({
   *     error: { name: webAuthnDomException.name, message: webAuthnDomException.message }
   *   });
   *   // Auth0はこのエラーを処理し、ページを再レンダリングするかリダイレクトする場合があります。
   * } catch (submitError) {
   *   console.error("Failed to report WebAuthn browser error:", submitError);
   * }
   * ```
   */
  reportBrowserError(options: ReportBrowserErrorOptions): Promise<void>;

  /**
   * ユーザーがWebAuthnプラットフォームチャレンジをオプトアウトし、別のMFA方式を選択できるようにします。
   * このアクションは `action: "pick-authenticator"` をAuth0に送信し、
   * ユーザーをMFA認証要素の選択画面に遷移させます。
   *
   * @param {TryAnotherMethodOptions} [options] - リクエストとともに送信されるオプションのカスタムパラメーター。
   * @returns {Promise<void>} 'pick-authenticator' アクションが送信されたときに解決されるPromise。
   * @throws {Error} フォームの送信が失敗した場合にエラーをスローします。
   *
   * @example
   * ```typescript
   * // 'sdk' は MfaWebAuthnPlatformChallenge のインスタンスと仮定します
   * try {
   *   await sdk.tryAnotherMethod();
   *   // 成功時、Auth0がMFA選択へのリダイレクトを処理します。
   * } catch (error) {
   *   console.error("Failed to switch MFA method:", error);
   * }
   * ```
   */
  tryAnotherMethod(options?: TryAnotherMethodOptions): Promise<void>;
}

<ParamField body="reportBrowserError" type="Promise<void>">
  特定の WebAuthn API エラー (`navigator.credentials.get()` で発生したもの) を Auth0 に報告します。
  このメソッドは、`verify()` (または手動の `navigator.credentials.get()` 呼び出し) が
  ブラウザー側の WebAuthn の問題 (例: ユーザーによるキャンセル `NotAllowedError`、タイムアウト) で失敗した場合に使用します。
  `action: "showError::{errorDetailsJsonString}"` と空の `response` を付けて、エラーの詳細を送信します。

  エラー報告が送信されると解決される Promise です。

  #### 例外

  フォームの送信に失敗した場合 (例: ネットワークエラー、無効な state) にエラーをスローします。

  ```typescript Example
  // 'sdk' は MfaWebAuthnPlatformChallenge のインスタンスであるとします
  // また、webAuthnDomException は失敗した navigator.credentials.get() 呼び出しで返されたエラーオブジェクトです。
  try {
    await sdk.reportBrowserError({
      error: { name: webAuthnDomException.name, message: webAuthnDomException.message }
    });
    // Auth0 はこのエラーを処理し、ページを再レンダリングするかリダイレクトする場合があります。
  } catch (submitError) {
    console.error("Failed to report WebAuthn browser error:", submitError);
  }
  ```

<ParamField body="verify" type="Promise<void>">
  WebAuthnプラットフォーム認証器のチャレンジを開始します。
  このメソッドは、`screen.publicKey` で提供されるチャレンジオプションを使用して、
  内部的に `navigator.credentials.get()` を呼び出します。
  成功すると、生成された資格情報を `action: "default"` とともに Auth0 に送信します。

  `navigator.credentials.get()` が失敗した場合 (たとえば、ユーザーによるキャンセル、タイムアウト、またはその他の WebAuthn API エラー) 、
  エラーの詳細を指定して `reportBrowserError` を呼び出すことを推奨します。

  検証の試行が送信されると完了する Promise です。
  通常、成功するとリダイレクトされます。

  #### Throws

  `screen.publicKey` が存在しない場合、
  `navigator.credentials.get()` が `getPasskeyCredentials` で処理されない予期しないエラーで失敗した場合、
  または Auth0 へのフォーム送信が失敗した場合に、エラーをスローします。

  ```typescript Example
  // 'sdk' が MfaWebAuthnPlatformChallenge のインスタンスであると仮定します
  try {
    const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
    await sdk.verify({ rememberDevice: remember });
    // 成功時は、Auth0 がリダイレクトを処理します。
  } catch (error) {
    console.error("Platform authenticator verification failed:", error);
    // WebAuthn API エラーであれば、それを報告します
    if (error.name && error.message) { // DOMException のようなエラーかどうかの基本的な確認
      await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
    }
    // ページが再読み込みされた場合は、サーバー側のバリデーションメッセージについて sdk.transaction.errors を確認します。
  }
  ```
