ローカルの開発環境は通常、初期状態では http (例: `http://localhost`) で動作します。このガイドでは、ローカルサーバーを https で実行すべきケースと、localhost で https を設定する方法を説明します。

<div id="when-to-use-a-local-https-server">
  ## ローカルの https サーバーを使用すべき場合
</div>

Web ブラウザーは http と https で挙動が異なるため、こうした違いを踏まえると、開発サーバーでも https 経由でページを配信する必要が生じることがあります。たとえば、次のようなケースです。

* ページが https 経由で配信されている場合、http URL からリソースを読み込もうとするリクエストはブロックされます。本番環境でのみ発生する不具合を防ぐため、あるいは機密情報を含む http リクエストを誤って送信しないようにするために、ローカルでもこの挙動が必要になることがあります。
* `Secure` 属性付きの Cookie や、[SameSite](https://auth0.com/blog/browser-behavior-changes-what-developers-need-to-know/) 属性が `None` に設定された Cookie は、安全でないチャネルでは送信されないため、認証やその他の機能に支障が出ます。

<div id="how-to-set-up-a-secure-local-server">
  ## 安全なローカルサーバーを設定する方法
</div>

基本的な考え方は、ローカル開発サーバーの前段で Web サーバーを動かし、そのサーバーで HTTPS リクエストを受け取り、HTTP 経由でアプリケーションの Web サーバーへ転送することです。これにより、アプリケーションへのリクエストをプロキシ経由で HTTPS で提供できます。これは HTTPS リバースプロキシ と呼ばれ、この HTTPS リバースプロキシの構築には既製の製品である [Caddy](https://caddyserver.com/v2) を使用できます。

手順は次のとおりです。

1. Caddy binary をダウンロードする
2. 通常どおり、アプリケーションの Web サーバーを HTTP で実行する
3. Caddy binary をリバースプロキシとして実行し、トラフィックをアプリケーションの Web サーバーに転送するよう設定する
4. HTTPS 経由でリバースプロキシの Web サーバーにアクセスする

<div id="step-1-download-the-caddy-binary">
  ### ステップ 1. Caddy binary をダウンロードする
</div>

https://caddyserver.com/download にアクセスし、ご利用のプラットフォームを選択して、ファイルをコンピューターにダウンロードします。

<div id="step-2-run-your-applications-web-server-over-http-as-normal">
  ### ステップ 2. 通常どおり、http でアプリケーションのWebサーバーを起動します
</div>

通常どおり、アプリケーションのローカル開発サーバーを起動します。たとえば、$ PORT=3000 npm start を実行します

<div id="step-3-run-the-caddy-binary-as-a-reverse-proxy-telling-it-to-forward-traffic-to-your-applications-web-server">
  ### Step 3. Caddy binaryをリバースプロキシとして実行し、トラフィックをアプリケーションのWebサーバーに転送するよう設定します
</div>

ダウンロードしたCaddy binaryの場所を確認し、次のコマンドを実行してリバースプロキシを作成し、ポート443 (httpsで使用されるデフォルトのポート) へのリクエストをアプリケーションのポート (この場合は3000) に転送します。

```bash wrap lines
/path/to/caddy reverse-proxy --from localhost:443 --to localhost:3000
```

<div id="step-4-now-visit-your-web-server-over-https">
  ### ステップ 4. 次に、https 経由で Web サーバーにアクセスします
</div>

https://localhost にアクセスすると、アプリケーションへのリクエストは https 経由で処理されます

<div id="running-your-application-behind-a-proxy">
  ## プロキシ環境でアプリケーションを実行する
</div>

アプリケーションをプロキシ環境で実行する場合は、多少の調整が必要になることがあります。

たとえば Express.js では、アプリ自体は正常に動作しても、`trust proxy` を設定していないと、クライアント IP アドレスとしてプロキシの IP アドレスが誤って登録されます。詳しくは &lt;https://expressjs.com/en/guide/behind-proxies.html&gt; を参照してください。
