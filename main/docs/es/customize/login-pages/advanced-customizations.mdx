**Advanced Customizations for Universal Login (ACUL)** amplía las capacidades de [Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience#universal-login-experience) para admitir múltiples marcas, configuraciones de seguridad complejas y la integración con numerosas plataformas de análisis.

<video autoPlay muted loop playsInline className="w-full aspect-video rounded-xl" src="/docs/images/videos/ACUL.mp4" />

Con ACUL, puedes escribir código en tu framework preferido, integrar tus plataformas de análisis y pruebas A/B existentes, y definir la experiencia de usuario. Los [ACUL SDKs](/docs/es/customize/login-pages/advanced-customizations/reference) sirven de conexión entre tu código personalizado y Universal Login.

<div id="benefits">
  ## Beneficios
</div>

* Amplía las capacidades de Universal Login de Auth0 para admitir varias marcas en un mismo tenant.
* Aprovecha las funcionalidades de Auth0, como [Claves de acceso](/docs/es/authenticate/database-connections/passkeys), [Bot Detection](/docs/es/secure/attack-protection/bot-detection) y [MFA](/docs/es/secure/multi-factor-authentication), sin tener que reconstruir tu lógica de autenticación.
* Integra analítica avanzada y pruebas A/B para optimizar la experiencia del usuario.
* Usa los ACUL SDKs para desarrollar con tu propia pila tecnológica. Puedes:
  * Usar cualquier biblioteca de CSS, JS o componentes de terceros
  * Integrar la plataforma de analítica que prefieras, como Heap, Mixpanel, entre otras
  * Desarrollar con el framework de frontend que prefieras, como Vue o React

<div id="use-cases">
  ## Casos de uso
</div>

Te conviene usar ACUL con tu implementación de Universal Login si:

* Necesitas aplicar directrices de marca específicas a tus flujos de autenticación.
* Necesitas ofrecer una experiencia multimarca.
* Necesitas realizar pruebas A/B a nivel de UI o análisis detallados.
* Necesitas ofrecer Universal Login en idiomas que Auth0 no admite actualmente.
* Necesitas añadir lógica compleja o personalizada a tus flujos de autenticación.

Para obtener más información, consulta [Casos de uso de ACUL](/docs/es/customize/login-pages/advanced-customizations/acul-use-cases).

<div id="how-it-works">
  ## Cómo funciona
</div>

<img className="block dark:hidden" src="/docs/images/cdy7uua7fh8z/ACUL/Final-Light-ACUL.png" alt="Cómo funciona ACUL" />

<img className="hidden dark:block" src="/docs/images/cdy7uua7fh8z/ACUL/Final-Dark-ACUL.png" alt="Cómo funciona ACUL" />

La **fase de compilación** comienza cuando creas tus pantallas de autenticación personalizadas en tu propio entorno.

Usa los [ACUL SDKs](/docs/es/customize/login-pages/advanced-customizations/reference) para enviar tu código personalizado y definir prompts y pantallas con tu stack de frontend preferido, como React o Angular. Cuando haces push de commits, tu pipeline de CI/CD ejecuta el proceso de compilación, compila tu aplicación en assets estáticos de JavaScript y CSS, y sube estos assets a tu almacenamiento en Private Cloud, que funciona como tu propia red de distribución de contenido (CDN).

El **tiempo de ejecución** comienza cuando tu aplicación redirige a los usuarios finales a tu página de inicio de sesión de Auth0.

Auth0 reconoce las pantallas de autenticación que configuraste para usar ACUL y entrega una host page mínima. El navegador del usuario carga esta página, que contiene etiquetas HTML que apuntan a las URL de tus assets en tu CDN. El navegador descarga tu JavaScript y CSS, verifica su integridad mediante los hashes de Subresource Integrity (SRI) y renderiza tu UI personalizada. A partir de ahí, el código de tu aplicación y el SDK de ACUL gestionan el resto de la interacción del usuario.

<div id="requirements">
  ## Requisitos
</div>

Para comenzar a usar Advanced Customizations for Universal Login, debes tener:

* Un tenant de desarrollo de Auth0 configurado con [Universal Login](/docs/es/authenticate/login/auth0-universal-login) y un [dominio personalizado](/docs/es/customize/custom-domains).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Te recomendamos crear un tenant para pruebas, desarrollo y producción con las etiquetas de entorno adecuadas.
</Callout>

* Una [aplicación de primera parte](/docs/es/get-started/auth0-overview/create-applications#create-applications) de Auth0
  * Una CDN con un pipeline de CI/CD integrado, como GitHub, para alojar recursos.
