Puedes configurar ACUL mediante Auth0 Dashboard, Auth0 Management API y herramientas de infraestructura como código (IaC). Cada método modifica el mismo objeto de configuración de renderización de cada pantalla.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para configurar ACUL, asegúrate de que tus archivos de configuración personalizados y recursos estén alojados en una CDN pública.
</Callout>

Para entender el objeto de configuración de renderización y sus propiedades, consulta la [referencia de la configuración de renderización de ACUL](/docs/es/customize/login-pages/advanced-customizations/configure/reference).

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Puedes usar el Dashboard para personalizar tus pantallas con ACUL:

A. Ve a [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/dashboard/#/universal-login/customizations-new)

1. Selecciona **Customize authentication screens**.
   2. Selecciona una <code>screen</code> para personalizarla.
   3. En **Settings &gt; Rendering mode**, selecciona **Advanced mode**.
   4. (Opcional) Selecciona **Use custom page template**.
   5. (Opcional) Selecciona **Advanced mode filters**.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/aculDashboard.png" alt="Página de ACUL del Auth0 Dashboard" />
</Frame>

6. Selecciona **Head Tags** para añadir tus propias etiquetas HTML `Head Tags`.
   7. (Opcional) Selecciona **Additional Data** para elegir qué datos adicionales se enviarán al navegador.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Additional Data** te permite añadir propiedades opcionales de datos de contexto. Por ejemplo: los datos de contexto opcionales de [`user`](/docs/es/libraries/acul/js-sdk/Screens/interfaces/UserMembers).
</Callout>

B. Selecciona **Save &amp; Publish**.

<div id="infrastructure-as-code-iac">
  ## Infraestructura como código (IaC)
</div>

Configura ACUL con herramientas de IaC, entre ellas:

* El comando [`auth0 acul`](https://auth0.github.io/auth0-cli/auth0_acul.html) de **Auth0 CLI**.
  * El recurso [`auth0_prompt_screen_renderer`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) de **Auth0 Terraform Provider**.
  * El recurso [`prompts`](https://github.com/auth0/auth0-deploy-cli/blob/master/docs/resource-specific-documentation.md#prompts) de **Auth0 Deploy CLI**.

<div id="auth0-management-api">
  ## Auth0 Management API
</div>

Utiliza los endpoints de Auth0 Management API para configurar las pantallas de Universal Login individualmente o para actualizar varias pantallas con una sola llamada a la API. Debes tener un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) en la solicitud.

**Configuración de pantallas individuales**

* El [endpoint `GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/es/api/management/v2/prompts/get-rendering) obtiene la configuración de una sola pantalla.

  * El [endpoint `PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/es/api/management/v2/prompts/patch-rendering) configura ACUL en una sola pantalla.

**Configuración masiva de pantallas**

* El [endpoint `GET /v2/prompts/rendering`](/docs/es/api/management/v2/prompts/get-all-rendering) obtiene las configuraciones de todas las pantallas.

  * El [endpoint `PATCH /v2/prompts/rendering`](/docs/es/api/management/v2/prompts/patch-bulk-rendering) actualiza varias pantallas en bloque.
