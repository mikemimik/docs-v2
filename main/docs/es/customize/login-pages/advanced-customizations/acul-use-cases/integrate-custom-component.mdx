<Card title="Antes de empezar">
  Necesitas:

  * Un tenant de desarrollo de Auth0 configurado con [Universal Login](/docs/es/authenticate/login/auth0-universal-login) y un [dominio personalizado](/docs/es/customize/custom-domains).
  * Una [aplicación de primera parte](/docs/es/get-started/auth0-overview/create-applications#create-applications) de Auth0.
  * Tener [Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first) habilitado en tu tenant de Auth0.
  * [Node.js](http://Node.js) V22+
  * La [herramienta Auth0 CLI](https://github.com/auth0/auth0-cli) [autenticada en tu tenant existente](https://github.com/auth0/auth0-cli?tab=readme-ov-file#authenticating-to-your-tenant).
  * Revisar la [guía Quickstart de ACUL](/docs/es/customize/login-pages/advanced-customizations/quickstart)
</Card>

Con ACUL, puedes usar la biblioteca de componentes que prefieras para personalizar las pantallas del prompt de Universal Login. En los siguientes ejemplos se utiliza [Shadcn](https://ui.shadcn.com/docs), una biblioteca de componentes reutilizables, y la pantalla `login-passwordless-email-code` de Auth0. En este ejemplo, reemplaza el campo de OTP predeterminado por el componente [InputOTP](https://ui.shadcn.com/docs/components/input-otp) de Shadcn.

1. Usa la herramienta Auth0 CLI para crear un proyecto de ACUL.

```bash
auth0 acul init <Your-App-Name>
```

Selecciona la pantalla `login-passwordless-email-code`

2. Ejecuta el servidor de desarrollo local de ACUL para editar y ver los cambios en tu pantalla.

```bash
auth0 acul dev
```

3. Inicializa Shadcn en la raíz de tu proyecto:

```bash
npx shadcn-ui@latest init
```

4. Sigue las indicaciones de la CLI para crear el archivo `components.json`, que contendrá la configuración de tu proyecto, y el archivo `src/lib/utils.ts`.

5. Agrega los archivos del componente en `src/components/ui/input-otp.tsx`:

```bash
npx shadcn-ui@latest add input-otp
```

6. Integra el componente:
   a. Navega hasta `src/screens/login-passwordless-email-code/components/IdentifierForm.tsx` y abre el archivo.
   b. Importa los componentes de InputOTP y reemplaza el campo de entrada existente. También debes gestionar el estado del código OTP y usar el hook correcto del SDK.

```bash
// En IdentifierForm.tsx
import { useState } from 'react';
import { useEmailOtpChallenge } from '@auth0/auth0-acul-react'; 
import {
  InputOTP,
  InputOTPGroup,
  InputOTPSlot,
} from '@/components/ui/input-otp'; // Importar desde ShadCN

// ... dentro de tu componente
const { submit } = useEmailOtpChallenge(); 
const [otp, setOtp] = useState('');

const handleSubmit = (e) => {
  e.preventDefault();
  submit({ code: otp }); // Llama al método submit con el código
};

return (
  <form onSubmit={handleSubmit}>
    {/* ... otros elementos de la interfaz ... */}
    <InputOTP maxLength={6} value={otp} onChange={setOtp}>
      <InputOTPGroup>
        <InputOTPSlot index={0} />
        <InputOTPSlot index={1} />
        <InputOTPSlot index={2} />
        <InputOTPSlot index={3} />
        <InputOTPSlot index={4} />
        <InputOTPSlot index={5} />
      </InputOTPGroup>
    </InputOTP>
    <Button type="submit">Verify Code</Button>
  </form>
);
```

7. Ejecuta la pantalla en local con el ACUL Context Inspector para ver tu nuevo componente:

```bash
auth0 acul dev -s  login-passwordless-email-code
```

8. Conecta tu entorno de desarrollo local a tu tenant de prueba para probar la nueva pantalla en un flujo de autenticación real:

```bash
auth0 acul dev --connected --screen login-passwordless-email-code
```

9. Sigue las indicaciones para compilar tus assets locales, iniciar el servidor local de desarrollo y actualizar la configuración de ACUL en tu tenant.

10. Prueba el flujo de autenticación passwordless:

```bash
auth0 test login
```
