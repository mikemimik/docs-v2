<Warning>
  En este momento, los esfuerzos de desarrollo activos de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. A menos que tu caso de uso específico requiera la experiencia Classic, se recomienda implementar Universal Login.
  Para comparar las funcionalidades de ambas experiencias de inicio de sesión, consulta [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).
</Warning>

Te recomendamos migrar la implementación de tu pantalla de [Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience#classic-login-experience) a [Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) con [Advanced Customizations for Universal Login (ACUL)](/docs/es/customize/login-pages/advanced-customizations#advanced-customizations-for-universal-login-acul).

<Card title="Antes de comenzar">
  Necesitas:

  * Un tenant de desarrollo de Auth0 configurado con [Universal Login](/docs/es/authenticate/login/auth0-universal-login) y un [custom domain](/docs/es/customize/custom-domains).
  * [Auth0 CLI](https://auth0.github.io/auth0-cli/) instalado y configurado con tu tenant de desarrollo.
  * Un repositorio de [GitHub](https://github.com/) para alojar tus pantallas de inicio de sesión personalizadas de ACUL.
  * Un inventario de los recursos de tu experiencia de Classic Login:
    * Componentes de UI.
    * Tokens de CSS o del tema y recursos de marca.
    * Reglas de validación (formato del correo electrónico, campos obligatorios).
    * Comportamiento de manejo de errores.
</Card>

<div id="initialize-an-acul-project">
  ## Inicializa un proyecto de ACUL
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si no has configurado tu tenant de desarrollo con Auth0 CLI, usa el comando [auth0 login](https://auth0.github.io/auth0-cli/auth0_login.html).
</Callout>

Ve a la ubicación donde quieras crear tu proyecto de ACUL y ejecuta el siguiente comando:

```bash
auth0 acul init acul-login --screens login-id,login-password
```

Auth0 CLI inicializa tu proyecto con las pantallas de autenticación `login-id` y `login-password` predefinidas.

Para obtener más información, lee [auth0 aclu en GitHub](https://auth0.github.io/auth0-cli/auth0_acul_init.html).

<div id="customize-acul-screens">
  ## Personaliza las pantallas de ACUL
</div>

Personaliza las pantallas `login-id` y `login-password` para ajustarlas a los estándares de tu organización.

Debes tener en cuenta lo siguiente:

* Reutiliza el CSS/tema y los componentes de React existentes.
  * Aplica la identidad visual (assets, tipografía, layout).
  * Agrega una validación básica del lado del cliente (correo electrónico obligatorio y con formato válido; contraseña obligatoria).
  * Estandariza la presentación de errores (de forma coherente y accesible).

Puedes desarrollar tus pantallas de ACUL localmente para personalizarlas con el comando:

```bash
auth0 acul dev
```

Para obtener más información, consulta [Flujo de trabajo de desarrollo de ACUL](/docs/es/customize/login-pages/advanced-customizations/development-workflow).

<div id="validate-the-acul-migration">
  ## Validar la migración de ACUL
</div>

Para validar de extremo a extremo la experiencia de inicio de sesión del usuario, puedes usar el comando [auth0 acul](https://auth0.github.io/auth0-cli/auth0_acul_dev.html):

```bash
auth0 acul dev --connected
```

Necesitas probar y asegurarte de lo siguiente:

* Que el flujo de autenticación use tu UI personalizada para las pantallas `login-id` y `login-password`, y que estas se muestren correctamente.
  * Que la contraseña incorrecta y otros mensajes de error del servidor se rendericen con claridad.
  * Que se redirija correctamente a los usuarios de vuelta a tu aplicación.
  * Que la validación del correo electrónico, si está habilitada, funcione como se espera.

Para obtener más información, consulta [las pruebas de integración con tenant en vivo de ACUL](/docs/es/customize/login-pages/advanced-customizations/development-workflow#live-tenant-integration-testing).

<div id="deploy-acul-screens-to-production-tenant">
  ## Desplegar pantallas de ACUL en el tenant de producción
</div>

El proyecto de pantallas de ACUL incluye una configuración de despliegue que puedes usar como punto de partida para desplegar tus pantallas en producción.

Los pasos de despliegue de ACUL incluyen:

* Generar los paquetes de recursos de tus pantallas.
  * Publicarlos en tu CDN.
  * Habilitar ACUL en tu tenant de producción de Auth0.

Para obtener más información, consulta el [workflow de despliegue de ACUL](/docs/es/customize/login-pages/advanced-customizations/deployment-workflow).
