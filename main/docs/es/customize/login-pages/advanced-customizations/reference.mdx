Auth0 ofrece varios SDKs para usar con ACUL.

<div id="acul-sdks">
  ## SDKs de ACUL
</div>

* [SDK de Auth0 ACUL React](/docs/es/libraries/acul/react-sdk)
  Proporciona hooks de React para simplificar la integración en aplicaciones de React.

  * [Auth0 ACUL JS SDK](/docs/es/libraries/acul/js-sdk)
    La biblioteca principal de JavaScript (escrita en TypeScript), adecuada para usarla con cualquier framework de frontend o con JavaScript puro.

<div id="acul-sample-applications">
  ## Aplicaciones de ejemplo de ACUL
</div>

Las [aplicaciones de ejemplo de ACUL](https://github.com/auth0-samples/auth0-acul-samples) incluyen un pipeline de compilación completo, flujos de trabajo de despliegue e implementaciones para las pantallas más comunes de Universal Login. Estas aplicaciones de ejemplo te ayudan a desarrollar tus pantallas. Los ejemplos están disponibles en React y usan los SDKs de ACUL React y JS.

Puedes usar el comando `auth0 acul init` para inicializar un proyecto base de React con los SDKs de ACUL React o JS. Para obtener más información, consulta [Auth0 CLI](https://github.com/auth0/auth0-cli) en GitHub.

<div id="acul-resources">
  ## Recursos de ACUL
</div>

Auth0 ofrece varios recursos para personalizar tu experiencia con Universal Login.

<div id="auth0-management-api">
  ### Auth0 Management API
</div>

Actualiza pantallas individuales de Universal Login o configura varias pantallas a la vez con la Management API. Para usar Auth0 Management API, debes tener un [access token](/docs/es/secure/tokens/access-tokens/management-api-access-tokens).

**Configuración individual de pantallas**

* El endpoint [`GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/es/api/management/v2/prompts/get-rendering) te permite recuperar la configuración de una sola pantalla.

  * El endpoint [`PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/es/api/management/v2/prompts/patch-rendering) te permite configurar ACUL en una sola pantalla.

**Configuración masiva de pantallas**

* El endpoint [`GET /v2/prompts/rendering`](/docs/es/api/management/v2/prompts/get-all-rendering) te permite recuperar la configuración de todas las pantallas.

  * El endpoint [`PATCH /v2/prompts/rendering`](/docs/es/api/management/v2/prompts/patch-bulk-rendering) te permite actualizar pantallas de forma masiva.

<div id="auth0-developer-tools">
  ### Herramientas de desarrollo de Auth0
</div>

Auth0 ofrece varias herramientas para simplificar la gestión del tenant, incluida la configuración de ACUL.

* **Auth0 CLI**<p />
  Usa los comandos `acul` para crear la estructura de proyectos, desarrollar localmente y desplegar tus configuraciones de pantalla. Para obtener más información, consulta [Auth0 CLI](https://github.com/auth0/auth0-cli) en GitHub.

  * **Auth0 Deploy CLI**<p />
    Gestiona toda la configuración de tu tenant como código, incluida la configuración de renderización de ACUL definida en archivos JSON. Para obtener más información, consulta [Auth0 Deploy CLI](https://github.com/auth0/auth0-deploy-cli) en GitHub.

  * **Auth0 Terraform provider**<p />
    Define las configuraciones de tus pantallas de ACUL. Para obtener más información, consulta [`promt_screen_renderer resource`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) en Terraform.
