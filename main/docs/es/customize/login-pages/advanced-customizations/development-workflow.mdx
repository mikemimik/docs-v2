Hay dos modos principales para desarrollar sus pantallas de ACUL: desarrollo local y pruebas de integración con tenant en vivo. 

<div id="local-development">
  ## Desarrollo local
</div>

Personaliza tus pantallas de ACUL localmente con el comando `auth0 acul dev` de [Auth0 CLI](https://github.com/auth0/auth0-cli/). Itera rápidamente en tus pantallas de ACUL sin demoras ni dependencias de backend con [Universal Login Context Inspector](https://github.com/auth0/ul-context-inspector).

Universal Login Context Inspector proporciona datos de contexto simulados para probar tus pantallas de autenticación con configuraciones específicas del tenant.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/UniversalLoginContextInspector.png" alt="Universal Login Context Inspector" />
</Frame>

* Selecciona **Screen** para ver las pantallas de ACUL configuradas disponibles con datos de contexto simulados predeterminados.
  * Selecciona **Data Source** para elegir datos de contexto simulados proporcionados desde **Auth0 CDN** o datos de contexto simulados del directorio `public/screens/{prompt}/{screen}` de tu proyecto de **Local Development**.
  * Selecciona el icono **Download JSON** para descargar los datos de contexto de la pantalla seleccionada. 

<div id="live-tenant-integration-testing">
  ## Pruebas de integración con tenant en vivo
</div>

La integración con tenant en vivo mediante el comando de Auth0 CLI `auth0 acul dev --connected` permite realizar pruebas de extremo a extremo de las UI de tus pantallas personalizadas y verificar la integración con tu tenant de Auth0.
Usa el modo `--connected` para las pruebas de integración cuando la UI esté prácticamente lista, ya que cada recarga reinicia el flujo de autenticación.

Al usar `auth0 acul dev --connected`

* Selecciona el directorio de tu proyecto de ACUL.
  * Selecciona `npm run build` para compilar el proyecto y configurar tu tenant de Auth0.
  * Selecciona el **puerto** para servir tus assets locales, `55444` de forma predeterminada, e inicia el servidor local en segundo plano.
  * Opcionalmente, permite que el modo `--connected` supervise continuamente los cambios en el directorio `/dist/assets` del proyecto.

Abre una terminal nueva y ejecuta `auth0 test login` para probar el flujo de login en tu tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En el modo `--connected`, el comando de Auth0 CLI supervisa los cambios en los archivos, vuelve a compilar los assets y actualiza automáticamente la configuración de tu tenant.
</Callout>

Para obtener más información, consulta [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_acul_dev.html) en GitHub.
