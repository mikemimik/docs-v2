Hay dos modos principales para desarrollar tus pantallas de ACUL: desarrollo local y pruebas de integración con un tenant real. 

<div id="local-development">
  ## Desarrollo local
</div>

Personaliza localmente tus pantallas de ACUL con el comando [Auth0 CLI](https://github.com/auth0/auth0-cli/) `auth0 acul dev`. Itera rápidamente en tus pantallas de ACUL sin demoras ni dependencias del backend con [Universal Login Context Inspector](https://github.com/auth0/ul-context-inspector).

Universal Login Context Inspector proporciona datos de contexto simulados para probar tus pantallas de autenticación con configuraciones específicas del tenant.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/UniversalLoginContextInspector.png" alt="Universal Login Context Inspector" />
</Frame>

* Selecciona **Screen** para ver las pantallas de ACUL configuradas que tienes disponibles con datos de contexto simulados predeterminados.
  * Selecciona **Data Source** para elegir datos de contexto simulados servidos desde **Auth0 CDN** o datos de contexto simulados del directorio `public/screens/{prompt}/{screen}` de tu proyecto de **Local Development**.
  * Selecciona el icono **Download JSON** para descargar los datos de contexto de la pantalla seleccionada. 

<div id="live-tenant-integration-testing">
  ## Pruebas de integración con tenant real
</div>

La integración con un tenant real mediante el comando `auth0 acul dev --connected` de Auth0 CLI permite realizar pruebas de extremo a extremo de las interfaces de usuario de tus pantallas personalizadas y verificar la integración con tu tenant de Auth0.
Usa el modo `--connected` para las pruebas de integración cuando la UI esté prácticamente completa, ya que cada recarga reinicia el flujo de autenticación.

Al usar `auth0 acul dev --connected`

* Selecciona el directorio de tu proyecto ACUL.
  * Selecciona `npm run build` para compilar el proyecto y configurar tu tenant de Auth0.
  * Selecciona el **puerto** desde el que se servirán tus assets locales; el valor predeterminado es `55444`, e inicia el servidor local en segundo plano.
  * De forma opcional, permite que el modo `--connected` supervise continuamente los cambios en el directorio `/dist/assets` del proyecto.

Abre una nueva terminal y ejecuta `auth0 test login` para probar el flujo de login en tu tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En el modo `--connected`, el comando de Auth0 CLI supervisa los cambios en los archivos, recompila los assets y actualiza automáticamente la configuración de tu tenant.
</Callout>

Para obtener más información, consulta [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_acul_dev.html) en GitHub.
