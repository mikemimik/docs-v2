title: Control de versiones de las páginas de Classic Login
---

<Warning>
  En este momento, los esfuerzos de desarrollo activo de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. Para personalizar tus páginas de inicio de sesión, Auth0 recomienda [Advanced Customizations for Universal Login](/docs/es/customize/login-pages/advanced-customizations).
</Warning>

Puedes usar software de control de versiones para gestionar el código fuente de tus páginas de Classic Login. Usar control de versiones para tu contenido puede ser especialmente útil si utilizas un flujo de CI/CD para la configuración de tu tenant y has personalizado el HTML de una o más páginas de Classic Login.

Para implementar el control de versiones, usa la integración de Auth0 Marketplace del método que prefieras:

* [GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline)
* [GitHub Actions](https://marketplace.auth0.com/integrations/github-actions)
* [Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline)
* [Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline)

El proceso general de despliegue requiere los siguientes pasos:

1. Crea una carpeta en tu repositorio de control de versiones con el nombre adecuado (`pages`).
2. Crea una página HTML (`login.html`, `password_reset.html`, `guardian_multifactor.html` o `error_page.html`) dentro de esa carpeta.
3. Crea un archivo JSON con el mismo nombre que tu página HTML para cada página alojada que quieras gestionar con control de versiones. Para habilitar la página, el archivo JSON debe contener lo siguiente:

`{
"enabled": true
}`

Por ejemplo:
`your-repo/pages/error_page.html
your-repo/pages/error_page.json`
