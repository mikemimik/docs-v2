<Warning>
  En este momento, los esfuerzos de desarrollo activo de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. Para personalizar tus páginas de login, Auth0 recomienda [Advanced Customizations for Universal Login](/docs/es/customize/login-pages/advanced-customizations).
</Warning>

Classic Login es una experiencia de login alojada por Auth0 que se basa en JavaScript para la [personalización avanzada](/docs/es/customize/login-pages/classic-login/customization-classic). Implementar Classic Login es menos complejo que integrar directamente el proceso de autenticación en tu aplicación y puede ayudarte a evitar los riesgos de la autenticación de origen cruzado.

De forma predeterminada, la página de Classic Login usa el widget de Lock para la autenticación de usuario. Sin embargo, también puedes personalizar plantillas para Lock en modo <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> o crear una interfaz personalizada con el SDK auth0.js.

Para personalizar las plantillas de tu página de Classic Login, primero debes habilitar la personalización avanzada. Para ello, sigue estos pasos:

1. En el Auth0 Dashboard, ve a [Branding &gt; Universal Login &gt; pestaña Login](https://manage.auth0.com/#/login_page).
2. Activa el interruptor **Customize Login Page**.
3. Encima del editor de código HTML, selecciona el menú Default Templates y elige la opción que quieras.

Las plantillas disponibles incluyen:

* Lock
* Lock (passwordless)
* Custom Login Form

Ambas plantillas de Lock te permiten personalizar el widget de Lock que se usa para la autenticación. El widget de Lock proporciona un conjunto estándar de comportamientos y una interfaz de usuario personalizable. Como alternativa, puedes usar la plantilla Custom Login Form para personalizar tu página de login con el [SDK de Auth0 para la Web](/docs/es/libraries/auth0js) o la [Authentication API](https://auth0.com/docs/api/authentication).

Los SDK de Auth0 son bibliotecas del lado del cliente que **no** ofrecen una interfaz de usuario, pero sí permiten una mayor personalización del comportamiento y la apariencia de la página de login. La Authentication API proporciona integración directa sin usar SDK de Auth0.

La plantilla que uses para personalizar tu página de login dependerá de las necesidades específicas de tu aplicación. En las secciones siguientes encontrarás una descripción general de cada opción: Lock, SDK de Auth0 o Authentication API.

<div id="using-the-lock-widget">
  ## Uso del widget de Lock
</div>

[Lock](/docs/es/libraries/lock) es un formulario de inicio de sesión que facilita a tus usuarios autenticarse con una conexión seleccionada. Lock gestiona automáticamente la mayoría de los detalles implicados en la creación y autenticación de usuarios.

Con Lock, implementarás una UI que:

* Es robusta y ofrece una excelente experiencia de usuario en cualquier dispositivo y resolución
* Tiene un diseño sencillo que encaja con la mayoría de los sitios web con solo unos pequeños ajustes en su color personalizado
* Se adapta a tu configuración y muestra los controles de formulario adecuados para cada conexión disponible, y solo los que están permitidos (como registro o restablecimiento de contraseña)
* Selecciona automáticamente la conexión correcta. Puedes especificar un comportamiento predeterminado para los casos ambiguos
* Recuerda la última conexión utilizada por un usuario determinado
* Se adapta automáticamente a la internacionalización
* Proporciona comprobación instantánea de la política de contraseñas durante el registro

Aunque no puedes modificar significativamente el comportamiento de Lock, sí puedes configurar varias opciones básicas para que Lock tenga un aspecto y un comportamiento diferentes.

Considera usar Lock si:

* Te gustan la estructura y la apariencia del widget.
* Prefieres una implementación simplificada de Classic Login con una UI responsiva lista para usar.
* Tu proceso incluye muchos de los casos de uso que Lock resuelve de forma nativa:

  * Inicios de sesión empresariales
  * Conexiones de base de datos con políticas de contraseñas
  * Registro de usuarios y restablecimiento de contraseña
  * Autenticación con proveedores sociales
  * Avatares

<div id="sample-login-customization-scripts">
  ### Ejemplos de scripts para personalizar el inicio de sesión
</div>

<div id="customize-the-application-logo">
  #### Personaliza el logo de la aplicación
</div>

El siguiente script de ejemplo personaliza el logo de cada aplicación, o bien puedes establecer un logo predeterminado. La resolución mínima recomendada es de 200 píxeles (ancho) por 200 píxeles (alto). Agrega la configuración `logouturl` al bloque `<scripts>`:

```javascript lines
var logourl = "https://example.com/defaultlogo1.png"; //establecer logo predeterminado
if(config.clientID === "HUXwC72R3qr9JJo9ImPsdzJbtY8aD5kS")
{
    logourl = "https://example.com/defaultlogo2.png";
}
theme: {
    logo: logourl,
    primaryColor: colors.primary ? colors.primary : 'green'
},
```

<div id="customize-the-signup-terms">
  #### Personaliza los términos del registro
</div>

El siguiente ejemplo personaliza los términos del registro de tu aplicación. Puedes añadir texto personalizado que se mostrará cuando los usuarios se registren.

```javascript lines
var languageDictionary;

languageDictionary = {
    signUpTerms: "I agree to the <a href='https://my-app-url.com/terms' target='_blank'>terms of service</a> and <a href='https://my-app-url.com/privacy' target='_blank'>privacy policy</a>."
};
```

<div id="using-the-auth0-sdk-for-web">
  ## Uso del SDK de Auth0 para la Web
</div>

Si los requisitos de tu aplicación no pueden cubrirse con el comportamiento estandarizado de Lock, o si tienes un proceso de autenticación personalizado complejo, necesitarás una interfaz de usuario personalizada. También puedes preferir esta opción si ya tienes una interfaz de usuario que deseas usar.

Con la [biblioteca de Auth0 para Web](/docs/es/libraries/auth0js), puedes personalizar el comportamiento y el flujo del proceso que se usa para activar el registro y la autenticación. Si lo prefieres, también puedes usar directamente la Authentication API, sin ningún tipo de envoltorio.

A diferencia de Lock, ninguna de estas opciones incluye una interfaz de usuario. Tendrás control total sobre la experiencia de usuario del registro y el flujo de autenticación, así como sobre aspectos de la UI como el layout, la apariencia, el branding, la internacionalización, la compatibilidad con RTL y más.

Considera implementar una interfaz de usuario personalizada junto con una biblioteca de Auth0 o la Authentication API para tu aplicación si:

* Tienes requisitos estrictos sobre la apariencia de la interfaz de usuario
* Tienes requisitos estrictos en cuanto al tamaño de los archivos: las bibliotecas de Auth0 son significativamente más pequeñas que Lock y, si en su lugar eliges trabajar directamente con la API, no añadirás peso adicional.
* Te sientes cómodo con HTML, CSS y JavaScript: crearás tu propia UI
* Solo necesitas gestionar la autenticación con nombre de usuario/contraseña y mediante proveedores sociales
* Tienes varias conexiones de base de datos o de Active Directory

También puedes ver ejemplos concretos del uso de Lock y de los SDKs de Auth0 para una amplia variedad de lenguajes de programación y plataformas en nuestros [Quickstarts](/docs/es/quickstart). Estas guías también pueden ayudarte a decidir cuál usar según las necesidades específicas de tu aplicación.
