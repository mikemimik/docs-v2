<Warning>
  En este momento, los esfuerzos de desarrollo activo de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. Para personalizar tus páginas de login, Auth0 recomienda [Advanced Customizations for Universal Login](/docs/es/customize/login-pages/advanced-customizations).
</Warning>

<Card title="¿Restablecer una contraseña?">
  En esta página se describe cómo personalizar una página de restablecimiento de contraseña de Classic Login.

  Si necesitas ayuda para restablecer una contraseña, consulta la siguiente documentación:

  * Para administradores del tenant que restablecen las contraseñas de otros usuarios: [Cambiar las contraseñas de los usuarios](/docs/es/authenticate/database-connections/password-change)
  * Para personas que restablecen sus propias contraseñas: [Restablecer la contraseña de la cuenta de Auth0](/docs/es/troubleshoot/customer-support/reset-account-passwords)
</Card>

Si usas Classic Login, la página de restablecimiento de contraseña ofrece a tus usuarios una forma de cambiar sus contraseñas si no pueden iniciar sesión. En la página de restablecimiento de contraseña, Auth0 gestiona toda la funcionalidad necesaria, incluida:

* El alojamiento de la propia página
* La redirección del usuario que quiere restablecer su contraseña cuando sea necesario
* Garantizar que la contraseña del usuario cumpla los requisitos que hayas definido
* Redirigir automáticamente al usuario después de restablecer su contraseña

Para crear una experiencia de marca uniforme en todo el login flow, puedes personalizar cómo se muestra a los usuarios el prompt de restablecimiento de contraseña. También puedes actualizar la página de restablecimiento de contraseña para mostrar cierta información personal, si corresponde.

<div id="enable-password-reset-page-customization">
  ## Habilitar la personalización de la página de restablecimiento de contraseña
</div>

La página de restablecimiento de contraseña está habilitada de forma predeterminada para todos los usuarios de Auth0 y no requiere personalización. Sin embargo, puedes habilitar la [personalización avanzada](/docs/es/customize/login-pages/classic-login/customization-classic) si quieres actualizar la imagen de marca y otros elementos de la página de restablecimiento de contraseña.

Para habilitar la personalización avanzada desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>:

1. Ve a [Branding &gt; Universal Login &gt; pestaña Restablecimiento de contraseña](https://manage.auth0.com/#/password_reset).
2. Activa el interruptor **Customize Password Reset Page**.

En el editor de código debajo del interruptor, ahora puedes modificar directamente la plantilla de página.

<div id="edit-password-reset-page">
  ## Editar la página de restablecimiento de contraseña
</div>

Después de habilitar la personalización de la página de restablecimiento de contraseña, puedes usar el editor de código integrado para modificar su HTML, dar estilo a la página con CSS y cambiar el JavaScript que se utiliza para recuperar y mostrar variables personalizadas. Asegúrate de seleccionar **Guardar cambios** debajo del editor de código para guardar los cambios.

<div id="display-custom-information-on-the-password-reset-page">
  ### Mostrar información personalizada en la página de restablecimiento de contraseña
</div>

Puede mostrar información personalizada en la página de restablecimiento de contraseña. Para ello, edite el código JavaScript integrado con el editor de la página de restablecimiento de contraseña:

```js lines expandable
new Auth0ChangePassword({
      container:         "change-password-widget-container",     // obligatorio
      email:             "{{email}}",                            // NO MODIFICAR
      csrf_token:        '{{csrf_token}}',                       // NO MODIFICAR
      ticket:            '{{ticket}}',                           // NO MODIFICAR
      password_policy:   '{{password_policy}}',                  // NO MODIFICAR
      theme: {
        icon: "{{tenant.picture_url | default: '//cdn.auth0.com/styleguide/1.0.0/img/badge.png'}}",
        primaryColor: "#ea5323"
      },
      dict: {
        // passwordPlaceholder: "your new password",
        // passwordConfirmationPlaceholder: "confirm your new password",
        // passwordConfirmationMatchError: "Please ensure the password and the confirmation are the same.",
        // passwordStrength: {
        //   containsAtLeast: "Contain at least %d of the following %d types of characters:",
        //   identicalChars: "No more than %d identical characters in a row (such as, \"%s\" not allowed)",
        //   nonEmpty: "Non-empty password required",
        //   numbers: "Numbers (such as 0-9)",
        //   lengthAtLeast: "At least %d characters in length",
        //   lowerCase: "Lower case letters (a-z)",
        //   shouldContain: "Should contain:",
        //   specialCharacters: "Special characters (such as !@#$%^&*)",
        //   upperCase: "Upper case letters (A-Z)"
        // },
        // successMessage: "Your password has been reset successfully.",
        // configurationError: "An error occurred. There appears to be a misconfiguration in the form.",
        // networkError: "The server cannot be reached, there is a problem with the network.",
        // timeoutError: "The server cannot be reached, please try again.",
        // serverError: "There was an error processing the password reset.",
        // headerText: "Enter a new password for<br />{email}",
        // title: "Change Password",
        // weakPasswordError: "Password is too weak."
        // passwordHistoryError: "Password has previously been used."
      }
    });
```

Por ejemplo, el siguiente fragmento de plantilla de ejemplo muestra la variable `tenant.picture_url`. Esta variable devuelve el valor de la URL del logotipo que se define en [Dashboard &gt; Configuración](https://manage.auth0.com/#/tenant).

```js lines
new Auth0ChangePassword({
      theme: {
        icon: "{{tenant.picture_url | default: '//cdn.auth0.com/styleguide/1.0.0/img/badge.png'}}",
      }
    });
```

Auth0 obtiene el logotipo de la URL y lo muestra en el prompt de restablecimiento de contraseña. Si Auth0 no puede resolver la URL, mostrará la imagen predeterminada.

<div id="custom-variables">
  #### Variables personalizadas
</div>

Puede usar las siguientes variables personalizadas para mostrar información personalizada en la página de restablecimiento de contraseña:

| Variable                      | Descripción                                                                                                                                                                                                         |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `email`                       | La dirección de correo electrónico del usuario que solicita el cambio de contraseña                                                                                                                                 |
| `ticket`                      | El ticket que representa la solicitud de restablecimiento de contraseña indicada                                                                                                                                    |
| `csrf_token`                  | Token utilizado para evitar ataques CSRF                                                                                                                                                                            |
| `tenant.name`                 | El nombre asociado con su tenant de Auth0                                                                                                                                                                           |
| `tenant.friendly_name`        | El nombre que se muestra para su tenant de Auth0                                                                                                                                                                    |
| `tenant.picture_url`          | La URL que dirige al logotipo que lo representa en Auth0                                                                                                                                                            |
| `tenant.support_email`        | La dirección de correo electrónico de soporte de su empresa que se muestra a sus usuarios de Auth0                                                                                                                  |
| `tenant.support_url`          | La URL de soporte de su empresa que se muestra a sus usuarios de Auth0                                                                                                                                              |
| `lang`                        | El idioma del usuario                                                                                                                                                                                               |
| `password_policy`             | La política de seguridad de la conexión activa. Puede ver cuál es usando `https://manage.auth0.com/#/connections/database/con_YOUR-CONNECTION-ID/security`. Asegúrese de proporcionar su ID de conexión en la URL.) |
| `password_complexity_options` | Objeto que contiene la configuración de los requisitos de complejidad de la contraseña                                                                                                                              |
| `min_length`                  | La longitud mínima requerida para las contraseñas recién creadas. Puede variar entre 1 y 128 caracteres                                                                                                             |

**Notas:**

* Puede establecer o comprobar los valores de las variables de su tenant en el área de Settings, en [Dashboard &gt; Settings](https://manage.auth0.com/#/tenant).
* No puede hacer que las personalizaciones sean condicionales en función del ID de cliente (`client_id`).

<div id="custom-error-messages">
  #### Mensajes de error personalizados
</div>

Si tu script de base de datos personalizada devuelve un error personalizado, puedes asociar una descripción del error mediante la propiedad `dict`:

```js lines
new Auth0ChangePassword({
  // ...otras opciones
  dict: {
    yourCustomError: 'This is a custom error message'
  }
});

function changePassword(email, newPassword, callback) {
  callback(new ValidationError('yourCustomError'));
}
```

<div id="update-password-reset-prompt">
  ## Actualizar el prompt de restablecimiento de contraseña
</div>

Si no habilita la personalización de la página de restablecimiento de contraseña, Auth0 se encargará de las actualizaciones necesarias del script, incluidos los cambios en el número de versión del prompt de restablecimiento de contraseña incluido.

**Cuando habilita la personalización de la página de restablecimiento de contraseña**, asume la responsabilidad de actualizar y mantener el script. Esto incluye actualizar el número de versión del prompt de restablecimiento de contraseña. Con la personalización habilitada, Auth0 no puede actualizar su script automáticamente sin correr el riesgo de interferir con los cambios que haya realizado.

La versión actual del prompt de restablecimiento de contraseña es la 1.5.5. Use este enlace para cargar la versión más reciente, incluidas las futuras actualizaciones de la versión menor:

`https://cdn.auth0.com/js/change-password-1.5.min.js`

<div id="revert-changes">
  ## Revertir cambios
</div>

Para revertir la página de restablecimiento de contraseña a un diseño anterior, puedes hacer lo siguiente:

* Revertir a la última plantilla guardada haciendo clic en **Reset to Last.**
* Revertir a la plantilla predeterminada de Auth0 haciendo clic en **Reset to Default**.
