<Warning>
  En este momento, los esfuerzos de desarrollo activo de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. Para personalizar tus páginas de inicio de sesión, Auth0 recomienda [Advanced Customizations for Universal Login](/docs/es/customize/login-pages/advanced-customizations).
</Warning>

Puedes personalizar los mensajes de error que se mostrarán en determinadas situaciones proporcionando la opción `languageDictionary`. Puedes encontrar una lista completa de los campos de `languageDictionary` disponibles para personalizar en el [archivo English Dictionary de Lock del repositorio de GitHub](https://github.com/auth0/lock/blob/master/src/i18n/en.js). A continuación, se muestra un ejemplo de algunos mensajes de error personalizados:

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample = `// Ejemplos de mensajes de error personalizados en la opción languageDictionary
var options = {
  languageDictionary: {
    error: {
      login: {
        "lock.invalid_email_password": "Mensaje personalizado sobre credenciales no válidas",
        "lock.network": "Mensaje personalizado que indica un error de red y sugiere al usuario revisar la conexión",
        "lock.unauthorized": "Mensaje personalizado sobre un error de permisos",
        "too_many_attempts": "Mensaje personalizado que indica que el usuario ha intentado iniciar sesión demasiadas veces sin éxito."
      },
      signUp: {
        "invalid_password": "Mensaje personalizado que indica que la contraseña no era válida",
        "user_exists": "Mensaje personalizado que indica que el usuario ya existe"
      }
    }
  }
};

// Inicializamos nuestro Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<div id="custom-errors-in-rules">
  ## Errores personalizados en Rules
</div>

Si devuelves códigos de error personalizados desde una [rule](/docs/es/customize/rules) o un script de base de datos personalizada, puedes manejar esos errores personalizados:

* En la URL de redirección de tu aplicación, leyendo los parámetros de query string `error` y `error_mesage`.
* Redirigiendo al usuario a tus páginas alojadas con un mensaje de error personalizado y mostrándolo con un [mensaje flash](/docs/es/libraries/lock/lock-api-reference).
