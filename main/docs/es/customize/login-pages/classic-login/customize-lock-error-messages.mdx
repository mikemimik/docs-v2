<Warning>
  En este momento, los esfuerzos de desarrollo activo de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. Para personalizar tus páginas de inicio de sesión, Auth0 recomienda [Advanced Customizations for Universal Login](/docs/es/customize/login-pages/advanced-customizations).
</Warning>

Puedes personalizar los mensajes de error que se mostrarán en determinadas situaciones al proporcionar la opción `languageDictionary`. Puedes encontrar una lista completa de los campos de `languageDictionary` disponibles para personalizar en el [archivo English Dictionary de Lock](https://github.com/auth0/lock/blob/master/src/i18n/en.js) del repositorio de GitHub. A continuación, se muestra un ejemplo de algunos mensajes de error personalizados:

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample = `// Ejemplos de mensajes de error personalizados en la opción languageDictionary
var options = {
  languageDictionary: {
    error: {
      login: {
        "lock.invalid_email_password": "Mensaje personalizado sobre credenciales no válidas",
        "lock.network": "Mensaje personalizado que indica un error de red y sugiere al usuario que compruebe la conexión",
        "lock.unauthorized": "Mensaje personalizado sobre un problema de permisos",
        "too_many_attempts": "Mensaje personalizado que indica que el usuario no ha podido iniciar sesión demasiadas veces."
      },
      signUp: {
        "invalid_password": "Mensaje personalizado que indica que la contraseña no era válida",
        "user_exists": "Mensaje personalizado que indica que ya existe un usuario"
      }
    }
  }
};

// Iniciamos Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<div id="custom-errors-in-rules">
  ## Errores personalizados en Rules
</div>

Si devuelves códigos de error personalizados desde una [rule](/docs/es/customize/rules) o un script de una base de datos personalizada, puedes manejar errores personalizados:

* En la URL de redirección de tu aplicación, leyendo los parámetros de la cadena de consulta `error` y `error_mesage`.
* Redirigiendo al usuario de vuelta a tus páginas alojadas con un mensaje de error personalizado y mostrando el mensaje con un [flash message](/docs/es/libraries/lock/lock-api-reference).
