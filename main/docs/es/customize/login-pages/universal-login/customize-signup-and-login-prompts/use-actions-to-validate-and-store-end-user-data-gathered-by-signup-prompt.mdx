---
description: Aprende a usar Auth0 Actions para validar y almacenar datos del usuario final.
title: Usa Actions para validar y almacenar datos del usuario final recopilados mediante las personalizaciones del prompt de registro
---

Puedes usar la [Personalización del prompt de registro](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts) junto con el trigger `pre-user-registration` para añadir a `user_metadata` los datos proporcionados por el usuario final (como su número de teléfono o su ubicación) desde el prompt de registro. Opcionalmente, puedes validar estos datos y mostrar un error de validación en el prompt.

<div id="prerequisites">
  ### Requisitos previos
</div>

* El tenant tiene un [Custom Domain](/docs/es/customize/custom-domains) verificado
* El tenant tiene una [Page Template](/docs/es/customize/login-pages/universal-login/customize-templates) configurada

<div id="add-a-field-to-the-signup-prompt">
  ### Añadir un campo al prompt de registro
</div>

Usa la [Management API](https://auth0.com/docs/api/management/v2/prompts/put-custom-text-by-language) para insertar un campo personalizado en un prompt mediante uno de los [Puntos de entrada](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts#-signup-prompt-entry-points). Este ejemplo añade el siguiente contenido en el punto de inserción `ulp-container-form-content-start`:

```html lines
<div class="ulp-field">
  <label for="first-name">First Name</label>
  <input type="text" name="ulp-first-name" id="first-name">
</div>
```

El resultado es una sección «First Name» en el prompt de registro:

<Frame>![](/docs/images/cdy7uua7fh8z/580LNGMxSkRlazJEdcVNpn/5fa98d4b414ac2133d1d6f4e327cea5a/Screenshot_2023-10-05_at_4.07.38_PM.png)</Frame>

<div id="create-an-action-in-the-pre-user-registration-trigger">
  ### Cree una Action en el trigger Pre-User Registration
</div>

Puede crear una Action personalizada de pre-user registration yendo a [**Actions &gt; Library &gt; Build Custom**](https://manage.auth0.com/dashboard/us/dev-6endizjt/actions/library)**.**

<Frame>![](/docs/images/cdy7uua7fh8z/52VSznphe2TyoJQnAn4nQg/ed54ca48dea191916a3173b648463fd8/Create_Action_-_EN.png)</Frame>

En el editor de código, actualice el controlador `onExecutePreUserRegistration`:

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const firstName = event.request.body['ulp-first-name'];
  api.user.setUserMetadata("firstName", firstName);
};
```

Opcionalmente, puede validar la entrada del usuario y enviar un error de validación llamando al método `api.validation.error` y, a continuación, implementar la Action:

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const firstName = event.request.body['ulp-first-name'];
  if(!firstName) {
    api.validation.error("invalid_payload", "Missing first name");
    return;
  }

  api.user.setUserMetadata("firstName", firstName);
};
```

<div id="add-the-action-to-the-flow">
  ### Agregar la Action al flujo
</div>

Ve a [**Actions &gt; Flows &gt; Registro previo de usuario &gt; Agregar Action &gt; Custom**](https://manage.auth0.com/dashboard/us/dev-6endizjt/actions/flows/pre-user-signup/), y luego arrastra y suelta tu nueva Action en el flujo de registro y selecciona **Aplicar**.

<Frame>![](/docs/images/cdy7uua7fh8z/5JmjwcdZCX7pDukNa90cgD/dae853b10cf4c41d2d3543a8e167aae7/3333.png)</Frame>

<div id="test-the-action">
  ### Prueba la Action
</div>

Regístrate para crear una cuenta en tu flujo de prueba y deja en blanco el campo **Nombre**. Verás un error al enviar el formulario:

<Frame>![](/docs/images/cdy7uua7fh8z/7qIWCsP8doFWPJGvMTNXJY/1225f9b3ff7ac01a7e9b59f03ca5e139/2024-01-31_16-10-10.png)</Frame>

Después de ingresar un nombre en el campo **Nombre**, podrás enviar el formulario correctamente:

<Frame>![](/docs/images/cdy7uua7fh8z/6TaK1VQyKGELMGmhuRIbVA/b453fe2eb97c95d92e4fbd68ce3cbd6a/2024-01-31_16-12-01.png)</Frame>

<div id="verify-that-the-data-was-saved-on-user_metadata">
  ### Verifica que los datos se hayan guardado en user_metadata
</div>

Ve a [**User Management &gt; Users**](https://manage.auth0.com/dashboard/us/dev-6endizjt/users) y confirma que los datos se hayan guardado en la pestaña Details:

<Frame>![](/docs/images/cdy7uua7fh8z/3x5lyrSJJgJQhbjsxoTmG6/033820476be7f67dc688da993fee7924/2024-01-31_16-13-35.png)</Frame>
