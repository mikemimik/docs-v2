El cambio flexible entre conexiones es una funcionalidad opcional que permite a los usuarios elegir su método de autenticación al iniciar sesión en una aplicación. Cuando se implementa, la pantalla de inicio de sesión de su aplicación les ofrece la opción de autenticarse con credenciales tradicionales de base de datos o con una conexión <Tooltip tip="" cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>. Los usuarios que seleccionan una [conexión passwordless](/docs/es/authenticate/passwordless) reciben una contraseña de un solo uso (OTP) por correo electrónico o SMS que pueden usar para iniciar sesión en su aplicación.

El cambio flexible entre conexiones usa [prompts personalizados de Universal Login](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts) para ofrecer a los usuarios una experiencia de autenticación con mayor autonomía.

<div id="before-you-begin">
  ## Antes de comenzar
</div>

Antes de implementar el cambio flexible entre conexiones, asegúrese de cumplir los siguientes requisitos:

* Use [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

  * Esta funcionalidad no está disponible para las páginas de inicio de sesión personalizadas.
* Configure un [dominio personalizado](/docs/es/customize/custom-domains).
* Habilite lo siguiente para su aplicación:

  * [autenticación Identifier First](/docs/es/authenticate/login/auth0-universal-login/identifier-first)
  * [conexión de base de datos](/docs/es/authenticate/database-connections)
  * [autenticación passwordless por correo electrónico o SMS](/docs/es/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms)

<div id="implement-flexible-connection-switching">
  ## Implementar cambio flexible entre conexiones
</div>

Para implementar esta funcionalidad, usa la [Auth0 Management API](https://auth0.com/docs/api/management/v2) para configurar partials personalizados del prompt de registro e inicio de sesión. Los partials son fragmentos de código personalizado que se insertan en un punto de entrada dentro de una pantalla de prompt, como la pantalla de inicio de sesión. Para obtener más información, consulta [Personalizar los prompt de registro e inicio de sesión](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts).

Para implementar el cambio flexible entre conexiones, configurarás partials personalizados de prompt con los siguientes parámetros:

| Parámetro    | Descripción                                                                                                                                                                                                                                                                                                                         | Ejemplo                                                 |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| `state`      | Renderiza el valor de `state` de la página actual, que es opaco y se usa con fines de seguridad.<br /><br />Para obtener más información sobre la pantalla actual, consulta [Personalizar las plantillas de página de Universal Login](/docs/es/customize/login-pages/universal-login/customize-templates#current-screen-information). | `<input type='hidden' name='state' value='{{state}}'>`  |
| `connection` | Nombre y tipo de la conexión.<br /><br />Para las conexiones passwordless, el valor es `email` o `sms`.                                                                                                                                                                                                                             | `<input type='hidden' name='connection' value='email'>` |

<Warning>
  En los ejemplos de código a continuación, asegúrate de reemplazar los marcadores de posición por los valores adecuados:

  * Reemplaza `{yourDomain}` por `yourdomain.auth0.com`.
  * Reemplaza `{mgmtApiToken}` por tu token de acceso.
</Warning>

<div id="configure-signup-prompt">
  ### Configurar prompt de registro
</div>

Puede configurar el prompt `signup-password` mediante el endpoint [Establecer partials para un prompt](https://auth0.com/docs/api/management/v2/prompts/put-partials):

<AuthCodeGroup>
  ```bash cURL
  curl --request PUT \
    --url 'https://{yourDomain}/api/v2/prompts/signup-password/partials' \
    --header 'authorization: Bearer {mgmtApiToken}' \
    --header 'content-type: application/json' \
    --data '{"signup-password":{"form-footer-start":"<form method=\"post\" data-form-secondary=\"true\"><input type=\"hidden\" name=\"state\" value=\"{{state}}\"> <input type=\"hidden\" name=\"connection\" value=\"email\"> <button type=\"submit\" id=\"switchConnectionButton\" style=\"background: #635dff; width: 100%; padding: 12px 16px; border: none; color: white;\" data-action-button-secondary=\"true\"> <span>Send a secure code by email</span> </button></form>"}}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/prompts/signup-password/partials");
  var request = new RestRequest(Method.PUT);
  request.AddHeader("authorization", "Bearer {mgmtApiToken}");
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{"signup-password":{"form-footer-start":"<form method=\"post\" data-form-secondary=\"true\"><input type=\"hidden\" name=\"state\" value=\"{{state}}\"> <input type=\"hidden\" name=\"connection\" value=\"email\"> <button type=\"submit\" id=\"switchConnectionButton\" style=\"background: #635dff; width: 100%; padding: 12px 16px; border: none; color: white;\" data-action-button-secondary=\"true\"> <span>Send a secure code by email</span> </button></form>"}}", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"

  func main() {

  	url := "https://{yourDomain}/api/v2/prompts/signup-password/partials"

  	fmt.Println(string(body))

  }
  ```

Como resultado, se agrega un **botón Enviar un código seguro por correo electrónico** a la pantalla `signup-password`. Cuando un usuario selecciona este botón, se envía un cuerpo de formulario que contiene el parámetro state del inicio de sesión y el nombre de la conexión deseada.
