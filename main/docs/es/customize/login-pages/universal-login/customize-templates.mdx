Puedes personalizar las páginas de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> proporcionando una plantilla de página creada con el [lenguaje de plantillas Liquid](https://shopify.github.io/liquid/basics/introduction/). Con las plantillas de página, puedes definir el contenido que se muestra alrededor de los prompts de Universal Login, como el cuadro de inicio de sesión o un <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">desafío MFA</Tooltip>. Como la misma plantilla de página se usa en todas las páginas del flujo de inicio de sesión, este método de personalización te permite implementar fácilmente una experiencia coherente y alineada con tu marca para los usuarios.

Para usar plantillas de página personalizadas, debes configurar un [dominio personalizado](/docs/es/customize/custom-domains) para tu tenant. Además, solo puedes actualizar las plantillas de página de Universal Login con la [Management API](https://auth0.com/docs/api/management/v2).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El término `prompt` se refiere a un paso específico del flujo de inicio de sesión, como la página de registro o un desafío MFA. Las variables y los ejemplos de código de esta página pueden usar el término `prompt` o el término `widget`. Aunque estos términos son sinónimos en el contexto de Universal Login, **no** son intercambiables dentro de tu código.  Para garantizar una personalización correcta, verifica que estés usando el término adecuado indicado para cualquier elemento que agregues a tu código.
</Callout>

<div id="page-template-requirements">
  ## Requisitos de la plantilla de página
</div>

Al crear una plantilla de página de Universal Login, debes incluir las siguientes etiquetas:

| Etiqueta       | Descripción                                                                                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth0:widget` | Contiene el HTML del prompt que se muestra en cada página del flujo de inicio de sesión, como la página de inicio de sesión o la página para Restablecer contraseña |
| `auth0:head`   | Contiene las etiquetas necesarias para renderizar el prompt                                                                                                         |

Para centrar el prompt en la página, agrega `class="_widget-auto-layout"` al elemento `<body>`. Puedes omitir este atributo para colocar el prompt manualmente según sea necesario.

<Warning>
  **Limitaciones de la plantilla de página:**

  * Los nombres de las clases CSS cambian cada vez que Auth0 genera el proyecto. El CSS personalizado que apunta a estas clases dejará de funcionar con cada nueva compilación.
  * La estructura HTML de las páginas de Universal Login puede cambiar. Evita personalizaciones que dependan de la estructura HTML para prevenir interrupciones.

  Para obtener más información, consulta [Personalización de CSS](#css-customization).
</Warning>

<div id="example-template">
  ##### Plantilla de ejemplo:
</div>

El siguiente ejemplo muestra la plantilla de página más sencilla de Universal Login que puedes crear con los tags requeridos:

```liquid lines
<!DOCTYPE html>
{% assign resolved_dir = dir | default: "auto" %}
<html lang="{{locale}}" dir="{{resolved_dir}}">
  <head>
    {%- auth0:head -%}
  </head>
  <body class="_widget-auto-layout">
    {%- auth0:widget -%}
  </body>
</html>
```

<Warning>
  Si usas Storybook para ver tu plantilla, ten en cuenta que la etiqueta `<script>` interfiere con la renderización, ya que no puede interpretar correctamente esas etiquetas.  Como alternativa, usa las comillas invertidas (&#96;) y el signo más (+) para inyectar correctamente la etiqueta `<script>` en el código de tu plantilla.

  **Ejemplo**:

  ``<scr`+`ipt>console.log("test");</scr`+`ipt>``
</Warning>

<div id="page-template-variables">
  ## Variables de plantilla de página
</div>

Las plantillas de página admiten diversas variables de contexto que influyen en cómo se renderiza una página.

Por ejemplo, puedes usar estas variables para:

* Renderizar contenido distinto según la aplicación asociada con el flujo de login. Por ejemplo, puedes gestionar dos marcas que requieran distintos diseños de página.
* Renderizar contenido distinto según el prompt específico. Por ejemplo, puede que quieras añadir información sobre lo que ofrece tu aplicación en la página de inicio de sesión, pero prefieras que el flujo de MFA solo muestre el prompt de desafío MFA.
* Añadir un pie de página con información de soporte para el usuario, como enlaces a tu página de soporte o información de contacto.

<div id="custom-query-parameters">
  ### Parámetros de consulta personalizados
</div>

<Warning>
  Auth0 recomienda que trates los parámetros `ext-` como no confiables, ya que estos valores pueden ser entradas de URL controladas por un atacante. Debes seguir los pasos recomendados para mitigar los riesgos de seguridad:

  * Usa salidas HTML escapadas en las plantillas
  * No interpoles valores `ext-` en bloques `<script>` ni en atributos de controladores de eventos
  * Valida que no usen esquemas de JavaScript o de datos antes de utilizar estos valores en URL

  Ten en cuenta que la lista de permitidos de caracteres no elimina por completo el riesgo de XSS en todos los contextos de renderización. Para obtener más información sobre las directrices y recomendaciones de seguridad, consulta [Security Guidance](/docs/es/secure/security-guidance).
</Warning>

También puedes usar parámetros de consulta en el contexto pasándolos al endpoint `/authorize` al iniciar la solicitud de autenticación. Estos parámetros de consulta personalizados deben tener el prefijo `ext-`.

El siguiente ejemplo usa los parámetros de consulta `ext-ga` y `ext-test` en la plantilla de la página de inicio de sesión:

```html lines
<!DOCTYPE html>
<html>
  <head>
    {%- auth0:head -%}
  </head>
  <body>
    {%- auth0:widget -%}
  </body>
  <pre style='background: wheat'>
    <b>Value of the ext-ga parameter:</b>{{ transaction.params.ext-ga }}
    <b>Value of the ext-test parameter:</b>{{ transaction.params.ext-test }}
  </pre>
</html>
```

Los parámetros de consulta personalizados tienen las siguientes limitaciones:

* Cada nombre de parámetro `ext-` debe ser único
* Una solicitud de autorización puede contener como máximo diez parámetros `ext-`
* El nombre del parámetro `ext-` debe comenzar con ext-, contener solo [a-zA-z0-9&#95;-] y tener un máximo de 28 caracteres, como en el siguiente ejemplo: `/^ext-[\w-]{1,28}$/`
* El valor del parámetro `ext-` debe contener solo [a-zA-Z0-9-.*~@+ /:&#95;] y tener un máximo de 255 caracteres, como en el siguiente ejemplo: `/^[-\w.*~@+ /:]{1,255}$/`

<div id="css-customization">
  ## Personalización de CSS
</div>

<Warning>
  **Limitaciones de la plantilla de página:**

  * Los nombres de las clases de CSS cambian cada vez que Auth0 compila el proyecto. El CSS personalizado que apunte a esas clases dejará de funcionar con cada nueva compilación.
  * La estructura HTML de las páginas de Universal Login puede cambiar. Evita las personalizaciones que dependan de la estructura HTML para prevenir interrupciones.
</Warning>

Hay algunas cosas que puedes personalizar con CSS:

* Puedes cambiar el tamaño del prompt de inicio de sesión incluyendo las variables de abajo dentro de etiquetas `<style>` en el elemento `<head>`.

  * Usa `--prompt-width` para ajustar el ancho del contenedor. Su valor predeterminado es 400px.
  * Para [formularios](/docs/es/customize/forms): usa `--form-max-width` para establecer el ancho máximo del formulario. Su valor predeterminado es 500px.

    * Para aplicar tu código solo a la página del formulario, incluye `{% if prompt.name == "custom-form" %} en el elemento <head>`.
* Puedes usar una fuente de Google importándola y sobrescribiendo la variable CSS `--font-family`.
* Puedes ocultar el logo del tenant agregando `class="_hide-prompt-logo"` en el elemento `<body>`.
* Puedes especificar un logo personalizado agregando `class="_use-custom-prompt-logo"` en el elemento `<body>`. Esto te permite, por ejemplo, cambiar el logo de la página de inicio de sesión según la aplicación:

```html lines expandable
<!DOCTYPE html>
<html lang="{{locale}}">

  <head>
    <title>Welcome to {{ application.name }} </title>
    {%- auth0:head -%}
```

<div id="using-the-auth0-cli">
  ## Uso de Auth0 CLI
</div>

Puedes usar la [Auth0 CLI](https://github.com/auth0/auth0-cli) para actualizar fácilmente las plantillas de página.

En la Auth0 CLI, ejecuta:

`auth0 universal-login customize`

La Auth0 CLI abrirá dos ventanas:

* Una ventana del navegador con un [Storybook](https://storybook.js.org/) que muestra la página de inicio de sesión con la plantilla de página aplicada:

<Frame>![Page Templates Storybook](/docs/images/cdy7uua7fh8z/1Ms4jj0pdMowoSZi54i72S/a2adcffa34a491ed7a587094884ad0f1/storybook.png)</Frame>

* El editor predeterminado, con el código de la plantilla de página:

<Frame>![undefined](/docs/images/cdy7uua7fh8z/3fTqpTmRRgVeLu5p8cioWa/e5a5c8a12dc93f5de95f12581ce0e5ff/vs-code.png)</Frame>

Ahora puedes cambiar el código de la plantilla de página y previsualizar los cambios en la ventana del navegador.

Una vez que cierres la ventana, se te preguntará si quieres guardar la plantilla. Si respondes **Sí**, la plantilla se subirá a tu tenant.

<div id="troubleshooting">
  ## Resolución de problemas
</div>

Si la plantilla no se está aplicando, asegúrate de acceder a `{customDomain}/authorize`. Si accedes a `{yourDomain}/authorize`, Auth0 no renderizará la plantilla de página.
