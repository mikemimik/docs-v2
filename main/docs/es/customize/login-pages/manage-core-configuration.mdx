La página de Core configuration te permite administrar la siguiente configuración para Universal Login y Classic Login:

* **Motor de inicio de sesión**: Cambia entre Universal y Classic Login.
* **Incrustación en iframe**: Permite que los flujos de inicio de sesión se carguen dentro de iframes en entornos de desarrollo.

<div id="settings">
  ## Configuración
</div>

<div id="universal-or-classic-login">
  ### Universal o Classic Login
</div>

Elige qué experiencia de login usa tu tenant. Cambiar esta configuración afecta a todas las applications que redirigen a tu Auth0 página de inicio de sesión. Para obtener más información, consulta [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<Steps>
  <Step title="Ve a Core configuration">
    Ve a **Branding &gt; Universal Login &gt; Pro-code customizations** y selecciona **Core configuration**.
  </Step>

  <Step title="Selecciona el tipo de login">
    En **Settings**, selecciona **Universal Login** o **Classic Login**.
  </Step>
</Steps>

<div id="iframe-embedding">
  ### Incrustación en iframe
</div>

Permite cargar Universal o Classic Login cuando la aplicación y el flujo de inicio de sesión se incrustan en un iframe, como ocurre con plataformas de UI generativa como Vercel. Cuando se habilita, la incrustación en iframe solo se aplica en entornos de desarrollo.

<Warning>
  Cuando la incrustación en iframe está habilitada, se desactivan los encabezados de seguridad que protegen contra el clickjacking.
</Warning>

<Steps>
  <Step title="Ve a Core configuration">
    Ve a **Branding &gt; Universal Login &gt; Pro-code customizations** y selecciona **Core configuration**.
  </Step>

  <Step title="Habilita la compatibilidad con iframe">
    En **Iframe settings**, activa **Enable support for generative UI platforms**.
  </Step>

  <Step title="Configura los orígenes permitidos">
    En **Allowed iframe URLs for Login**, introduce los orígenes permitidos.

    Para v0, introduce los siguientes orígenes:

    * `https://v0.app`
    * `https://*.v0.app`
    * `https://*.vusercontent.net`

    Cuando se crea una aplicación desde una plataforma de UI generativa como Vercel mediante una integración de Auth0 disponible, Auth0 precompleta los orígenes permitidos desde la plataforma. Verifica siempre los orígenes precompletados para asegurarte de que no falte ninguno. Para obtener más información, consulta [Integrate with Vercel](/docs/es/customize/integrations/integrate-with-vercel).
  </Step>
</Steps>
