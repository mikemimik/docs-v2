La página Core configuration te permite administrar los siguientes ajustes de Universal Login y Classic Login:

* **Motor de inicio de sesión**: Cambia entre Universal y Classic Login.
* **Incrustación en iframe**: Permite que los flujos de inicio de sesión se carguen dentro de iframes en entornos de desarrollo.

<div id="settings">
  ## Configuración
</div>

<div id="universal-or-classic-login">
  ### Universal Login o Classic Login
</div>

Elija qué experiencia de inicio de sesión usa su tenant. Cambiar esta configuración afecta a todas las aplicaciones que redirigen a su página de inicio de sesión de Auth0. Para obtener más información, consulte [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<Steps>
  <Step title="Vaya a Core configuration">
    Vaya a **Branding &gt; Universal Login &gt; Pro-code customizations** y seleccione **Core configuration**.
  </Step>

  <Step title="Seleccione el tipo de inicio de sesión">
    En **Settings**, seleccione **Universal Login** o **Classic Login**.
  </Step>
</Steps>

<div id="iframe-embedding">
  ### Incrustación en iframe
</div>

Habilite Universal Login o Classic Login para que se carguen cuando la aplicación y el flujo de inicio de sesión estén incrustados en un iframe, como sucede con plataformas de UI generativa como Vercel. Cuando esta opción está habilitada, la incrustación en iframe solo se aplica en entornos de desarrollo.

<Warning>
  Cuando la incrustación en iframe está habilitada, se desactivan los encabezados de seguridad que protegen contra el clickjacking.
</Warning>

<Steps>
  <Step title="Vaya a Core configuration">
    Vaya a **Branding &gt; Universal Login &gt; Pro-code customizations** y seleccione **Core configuration**.
  </Step>

  <Step title="Habilite la compatibilidad con iframe">
    En **Iframe settings**, active **Enable support for generative UI platforms**.
  </Step>

  <Step title="Configure los orígenes permitidos">
    En **Allowed iframe URLs for Login**, introduzca los orígenes permitidos.

    Para v0, introduzca los siguientes orígenes:

    * `https://v0.app`
    * `https://*.v0.app`
    * `https://*.vusercontent.net`

    Cuando se crea una aplicación desde una plataforma de UI generativa como Vercel mediante una integración disponible de Auth0, Auth0 rellena automáticamente los orígenes permitidos desde la plataforma. Verifique siempre los orígenes rellenados automáticamente para asegurarse de que estén completos. Para obtener más información, consulte [Integrate with Vercel](/docs/es/customize/integrations/integrate-with-vercel).
  </Step>
</Steps>
