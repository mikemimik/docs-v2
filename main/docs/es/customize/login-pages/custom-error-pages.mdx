Cuando se produce un error de autorización y tu URL de devolución de llamada es válida, el <Tooltip tip="Servidor de autorización: servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Authorization+Server">servidor de autorización</Tooltip> devuelve los parámetros de error y state correspondientes a tu URL de devolución de llamada. Si tu URL de devolución de llamada no es válida, tu aplicación mostrará la [página de error predeterminada de Auth0](/docs/es/authenticate/login/auth0-universal-login/error-pages).

Tu aplicación también puede mostrar la página de error predeterminada de Auth0 por razones distintas de una URL de devolución de llamada no válida, como las siguientes:

* Faltan parámetros obligatorios al llamar al [endpoint de inicio de sesión](https://auth0.com/docs/api/authentication#login) de Auth0 Authentication API.
* El usuario abre un enlace de Restablecimiento de contraseña caducado (cuando usa la experiencia de Classic Login).
* El usuario accede a una página de inicio de sesión guardada en favoritos y no se especifica una [Ruta de inicio de sesión predeterminada](/docs/es/authenticate/login/auth0-universal-login/configure-default-login-routes).

<div id="parameters">
  ## Parámetros
</div>

Si eliges configurar una página de error personalizada, el servidor de autorización devolverá parámetros añadidos a la URL como una cadena de consulta.

| Parámetro           | Descripción                                                                         |
| ------------------- | ----------------------------------------------------------------------------------- |
| `client_id`         | Identificador de la aplicación de Auth0.                                            |
| `connection`        | Conexión utilizada en el momento del error.                                         |
| `lang`              | Idioma configurado para su uso en el momento del error.                             |
| `error`             | Código del error.                                                                   |
| `error_description` | Descripción del error.                                                              |
| `tracking`          | Identificador utilizado por Auth0 para encontrar errores en los registros internos. |

Los parámetros presentados varían según el tipo de error y son específicos de la solicitud. Por ejemplo, si la solicitud que produjo el error no contenía un `client_id`, el servidor de autorización no devolverá el parámetro `client_id`.

<div id="display-a-custom-error-page">
  ## Mostrar una página de error personalizada
</div>

Si quieres mostrar una página de error personalizada, tienes dos opciones:

1. Redirigir a los usuarios a una página de error personalizada usando el Auth0 Dashboard o la Auth0 Management API.
2. Configurar Auth0 para que renderice una página de error personalizada por ti mediante la Management API.

<div id="redirect-users-to-a-custom-error-page-using-the-dashboard">
  ### Redirigir a los usuarios a una página de error personalizada desde el Dashboard
</div>

Usa el Dashboard para configurar Auth0 de modo que redirija a los usuarios a una página de error personalizada:

1. Ve a [Auth0 Dashboard &gt; Configuración del tenant](https://manage.auth0.com/#/tenant/).
2. Busca la sección **Páginas de error**.
3. Selecciona la opción **Redirigir a los usuarios a tu propia página de error**.
4. Introduce la URL de la página de error que quieres que vean tus usuarios y selecciona **Guardar**.

<div id="redirect-users-to-a-custom-error-page-using-the-management-api">
  ### Redirija a los usuarios a una página de error personalizada con la Management API
</div>

Use el endpoint [Update Tenant Settings](https://auth0.com/docs/api/management/v2/#!/Tenants/patch_settings) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Sustituya el placeholder value `{mgmtApiAccessToken}` por su <Tooltip tip="token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de la Management API y actualice el valor del campo `url` en el cuerpo JSON para que apunte a la ubicación de la página de error.

  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/tenants/settings' \
    --header 'authorization: Bearer {mgmtApiAccessToken}' \

  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/tenants/settings");

  request.AddParameter("application/json", "{"error_page": {"html": "", "show_log_link":false, "url": "http://www.example.com"}}", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```


  ```ruby Ruby
  require 'uri'
  require 'net/http'
  require 'openssl'

  url = URI("https://{yourDomain}/api/v2/tenants/settings")

  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Patch.new(url)
  request["authorization"] = 'Bearer {mgmtApiAccessToken}'
  request["content-type"] = 'application/json'
  request.body = "{"error_page": {"html": "", "show_log_link":false, "url": "http://www.example.com"}}"

  response = http.request(request)
  puts response.read_body
  ```

| Valor                   | Descripción                                                                                                                   |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN` | [token de acceso for the Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:tenant_settings`. |
| `show_log_link`         | Indica si se debe mostrar un enlace al error en los registros del tenant. Los valores válidos son `true` y `false`.           |
| `url`                   | Ubicación de la página de error personalizada a la que quieres redirigir.                                                     |

<div id="render-a-custom-error-page">
  ### Renderizar una página de error personalizada
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cuando los errores se muestran como parte del widget de Classic Login (por ejemplo, un enlace de restablecimiento de contraseña caducado), no se renderizará una página de error personalizada alojada por Auth0, aunque esté configurada.
</Callout>

Use el endpoint [Update Tenant Settings](https://auth0.com/docs/api/management/v2/#!/Tenants/patch_settings) de la Management API. Reemplace el valor de marcador `{mgmtApiAccessToken}` por su token de acceso de la Management API y actualice el valor del campo `html` en el cuerpo JSON con una cadena que contenga el HTML de su página.

Puede usar la sintaxis de Liquid para incluir las siguientes variables:

* `{client_id}`
* `{connection}`
* `{lang}`
* `{error}`
* `{error_description}`
* `{tracking}`
