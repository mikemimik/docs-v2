Cuando un experimento está activo y se asigna una variación, Experiment Center inyecta automáticamente el contexto del experimento en el contexto de renderización de la [plantilla de página](/docs/es/customize/login-pages/universal-login/customize-templates#customize-universal-login-page-templates). Las plantillas de página no requieren un paso de activación: el contexto del experimento se pasa a todas las plantillas de forma incondicional cuando un experimento está activo. Las plantillas reciben el contexto como una variable local llamada `experiment`.

<Warning>
  Durante la Beta, Experiment Center solo funciona en tenants de desarrollo. Los tenants de producción no son compatibles.
</Warning>

<div id="available-context-fields">
  ## Campos de contexto disponibles
</div>

Los siguientes campos están disponibles en la plantilla de página cuando hay un experimento activo:

| **Variable**               | **Tipo** | **Descripción**                                                                       |
| -------------------------- | -------- | ------------------------------------------------------------------------------------- |
| `experiment.experiment_id` | string   | El ID del experimento activo                                                          |
| `experiment.variation_id`  | string   | El ID de la variación asignada                                                        |
| `experiment.config`        | object   | Configuración combinada: parámetros de la línea base + sobrescrituras de la variación |
| `experiment.is_control`    | boolean  | Indica si esta es la variación de control                                             |

Cuando no hay ningún experimento activo, `experiment` es `null` (o no está presente). Comprueba siempre si es `null` antes de usarlo.

<div id="experimentconfig-structure">
  ### Estructura de `experiment.config`
</div>

`experiment.config` es un mapa de claves y valores en el que cada clave es el nombre de un parámetro y cada valor es un objeto con una propiedad `value`:

```
experiment.config.my_param.value  → el valor del parámetro para esta variación
```

Cada parámetro definido en el flag de funcionalidad tiene un valor en `config`. Experiment Center fusiona los parámetros de la línea base con las sobrescrituras de la variación antes de inyectarlos, por lo que nunca tendrás que buscar por tu cuenta el valor predeterminado.

<div id="read-a-parameter-in-a-template">
  ## Leer un parámetro en una plantilla
</div>

Las plantillas de página de Auth0 Universal Login utilizan EJS. En los ejemplos siguientes se usa la sintaxis de EJS.

**Sintaxis de EJS:**

```html
<%= experiment && experiment.config.heading_text.value %>
```

Con una comprobación de valor nulo:

```html
<% if (experiment) { %>
  <h1><%= experiment.config.heading_text.value %></h1>
<% } else { %>
  <h1>Create your account</h1>
<% } %>
```

<div id="customize-signup-and-login-with-partials">
  ## Personaliza Signup y Login con Partials
</div>

Cuando personalizas los prompts de Signup y Login con [partials](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts#manage-partials-programmatically), el contexto del experimento se pasa explícitamente como parte del partial.

En el partial, también está disponible la misma variable `experiment`:

```html
<!-- partials/signup-header.ejs -->
<header>
  <h1>
    <% if (experiment && experiment.config.heading_text) { %>
      <%= experiment.config.heading_text.value %>
    <% } else { %>
      Create your account
    <% } %>
  </h1>
</header>
```

<div id="example-signup-headline-copy-variant">
  ## Ejemplo: variante del texto del encabezado de registro
</div>

Este ejemplo muestra un flag de funcionalidad con un parámetro de tipo String que controla el encabezado de la página de registro. La variación de control usa &quot;Create your account&quot;; la variación de tratamiento usa &quot;Join in seconds.&quot;

**Parámetros del flag de funcionalidad:**

```json
{
  "signup_headline": {
    "type": "string",
    "value": "Create your account",
    "description": "Headline text on the signup screen"
  }
}
```

**Variación de control:** sin sobrescrituras (hereda `signup_headline: "Crea tu cuenta"`)

**Variación de tratamiento:**

```json
{
  "overrides": {
    "signup_headline": { "value": "Join in seconds" }
  }
}
```

**Código de la plantilla:**

```html
<!-- signup.ejs -->
<div class="signup-container">
  <h1>
    <% if (experiment) { %>
      <%= experiment.config.signup_headline.value %>
    <% } else { %>
      Create your account
    <% } %>
  </h1>

  <form action="/usernamepassword/login" method="POST">
    <!-- campos del formulario -->
  </form>
</div>
```

Los usuarios asignados a la variación de control ven &quot;Crea tu cuenta.&quot; Los usuarios del grupo de tratamiento ven &quot;Únete en segundos.&quot; Los usuarios sin ningún experimento activo también ven &quot;Crea tu cuenta&quot; (la rama `else`).

<div id="example-conditional-ui-element">
  ## Ejemplo: elemento condicional de la UI
</div>

Este ejemplo usa un parámetro booleano para mostrar u ocultar un elemento adicional de la UI:

```html
<!-- login.ejs -->
<% if (experiment && experiment.config.show_social_proof.value === true) { %>
  <div class="social-proof">
    <p>Join 2 million people who sign in with their passkey.</p>
  </div>
<% } %>

<form action="/usernamepassword/login" method="POST">
  <!-- campos del formulario -->
</form>
```

La comparación `=== true` (en lugar de comprobar si es truthy) evita que el elemento aparezca cuando `experiment` es `null` o cuando el parámetro no está presente.

<div id="safe-defaults-when-no-experiment-is-active">
  ## Valores predeterminados seguros cuando no hay ningún experimento activo
</div>

Cuando no hay ningún experimento activo, `experiment` es `null` en el contexto de la plantilla. Estructura tus plantillas para que siempre se puedan renderizar correctamente, incluso sin el contexto del experimento:

```html
<h1>
  <%= (experiment && experiment.config.page_title && experiment.config.page_title.value)
      || "Default page title" %>
</h1>
```

O usa bloques if/else explícitos (más legibles para condiciones complejas):

```html
<% if (experiment && experiment.config.page_title) { %>
  <h1><%= experiment.config.page_title.value %></h1>
<% } else { %>
  <h1>Default page title</h1>
<% } %>
```

Define de forma fija los valores de respaldo en tus plantillas. Coinciden con la línea base de la variación de control, por lo que tus plantillas se comportarán correctamente tanto cuando no haya ningún experimento en ejecución como cuando se asigne la variación de control.
