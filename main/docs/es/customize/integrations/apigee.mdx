description: Aprende a usar Auth0 para proteger el acceso a los proxies de API desarrollados y gestionados por Apigee Edge.
title: Protege Apigee con Auth0
---

Si utilizas Apigee Edge para desarrollar y gestionar las API de tu servicio backend, puedes usar Auth0 para proteger el acceso a tus proxies de API.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de comenzar, necesitarás:

1. Tener un proxy de API de Apigee Edge que deba protegerse. Para obtener más información, consulta el artículo [Get Started](https://docs.apigee.com/api-platform/get-started/get-started) de Apigee.
2. [Regístrate](https://auth0.com/signup) para obtener una cuenta de Auth0.

El proceso de creación de tu proxy de API queda fuera del alcance de este artículo. En su lugar, nos centraremos en proteger un proxy de API que ya tienes con Auth0.

<div id="create-a-custom-api">
  ## Crear una API personalizada
</div>

Primero, [registra tu proxy de API de Apigee Edge en el Dashboard](/docs/es/get-started/auth0-overview/set-up-apis). Auth0 necesita reconocer Apigee como una <Tooltip tip="Audience: identificador único de la audiencia de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o el de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip> para asegurarse de que todos los <Tooltip tip="Audience: identificador único de la audiencia de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o el de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=Access+Tokens">Tokens de acceso</Tooltip> emitidos tengan la audiencia correcta. El usuario se autentica con Auth0 a través de la aplicación, y la aplicación especifica este valor de audience para asegurarse de que el token de acceso tenga los scopes adecuados para la audiencia proporcionada.

Necesitarás hacer lo siguiente:

1. Proporcionar un nombre para tu API (por ejemplo, `apigee`).
2. Proporcionar un identificador para tu API: `urn:apigee:target:api`
3. Elegir un algoritmo de firma: `RS256` (predeterminado)

Cuando registras tu proxy de API de Apigee Edge, Auth0 también crea una aplicación de **máquina a máquina (M2M)** en tu nombre y la nombra para que coincida con la API que registraste. Puedes usar esta aplicación para hacer pruebas; se configura automáticamente para que quede autorizada a llamar a tu API.

<div id="note-variables-from-the-test-application">
  ## Anota las variables de la aplicación de prueba
</div>

Cambia a la aplicación de prueba que creaste al registrar tu API y toma nota de las variables que se configuraron durante el proceso de registro de tu API y la creación de la aplicación M2M asociada. Las necesitarás en los siguientes pasos de este tutorial.

1. Ve a [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/#/applications)], y haz clic en el nombre de tu aplicación de prueba.
2. Desplázate hacia abajo y toma nota de las siguientes variables:

   * audiencia de la API
   * dominio de Auth0
   * ID de cliente
   * URL(s) de devolución de llamada permitidas: las URL a las que se puede redirigir al usuario después de la autenticación. Puedes especificar varias URL separándolas por comas. (Esto suele hacerse para gestionar distintos entornos, cada uno con sus propias redirecciones.)

<div id="implement-the-client-credentials-flow">
  ## Implementa el flujo de credenciales de cliente
</div>

Ahora ya estás listo para implementar el flujo de credenciales de cliente y solicitar los tokens de acceso que puedes proporcionar a Apigee Edge. Ten en cuenta que, en este escenario, usarás el flujo de credenciales de cliente porque estás usando Apigee con las API de tus servicios de backend, lo que representa una aplicación de máquina a máquina (M2M); en otros casos, puede ser necesario usar flujos diferentes.

Para aprender a iniciar sesión y obtener un token de acceso que pueda usarse para llamar a Apigee Edge, consulta [Llamar a la API mediante el flujo de credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).

<div id="validate-the-access-token">
  ## Validar el token de acceso
</div>

Una vez que hayas solicitado y recibido un token de acceso de Auth0, podrás usarlo para llamar al proxy de API que configuraste con Edge.

Cuando [uses tu token de acceso](https://docs.apigee.com/api-platform/security/oauth/using-access-tokens.html), lo primero que hará Apigee Edge será [verificar el token](https://docs.apigee.com/api-platform/security/oauth/using-access-tokens.html#addingaverifyaccesstokenpolicy).

Los tokens de acceso de Auth0 son <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Tokens">JSON Web Tokens</Tooltip> (JWT), por lo que puedes aprovechar la política Verify JWT de Apigee Edge para hacerlo. Para obtener más información, consulta el artículo de Apigee [Verify JWT policy signed with the RS256 algorithm](https://docs.apigee.com/api-platform/reference/policies/verify-jwt-policy#verify-a-jwt-signed-with-the-rs256-algorithm).

Apigee Edge verificará el token antes de que ocurra cualquier otra cosa; si se rechaza el token, se detendrá todo el procesamiento y Edge devolverá un error al cliente.

<div id="testing">
  ## Pruebas
</div>

Para probar la implementación, realiza una solicitud HTTP POST a Apigee Edge con el token de acceso emitido por Auth0 incluido en el encabezado de la solicitud. Puedes ver cómo hacerlo en el artículo de Apigee [Calling the protected API](https://docs.apigee.com/api-platform/security/oauth/oauth-20-client-credentials-grant-type.html#callingtheprotectedapi).

Cuando recibas una respuesta exitosa, extrae el token y revísalo para verificar que incluya los claims de usuario obligatorios/solicitados.
