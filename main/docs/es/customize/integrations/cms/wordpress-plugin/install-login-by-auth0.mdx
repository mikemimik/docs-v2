<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para instalar o personalizar plugins, debe usar un sitio de WordPress.org alojado por su cuenta. En un sitio de WordPress.com no es posible instalar plugins. Para obtener más información sobre las diferencias, lea [WordPress.com and WordPress.org on wordpress.com](https://wordpress.com/support/com-vs-org/).
</Callout>

Este plugin se puede añadir a su sitio de WordPress desde la pantalla **Plugins** de wp-admin:

1. Inicie sesión en un sitio de WordPress existente como administrador.
2. Vaya a **Plugins &gt; Add New** en el menú de administración de la izquierda.
3. Busque &quot;Login by Auth0&quot;
4. En el plugin Login by Auth0, haga clic en **Install Now** y luego en **Activate**.

Para obtener más información sobre este proceso, lea [WordPress.com and WordPress.org on wordpress.com](https://wordpress.com/support/com-vs-org/).

En cuanto se active el plugin, se le redirigirá al inicio del Setup Wizard.

Si todavía no tiene una cuenta de Auth0, [r**egístrese gratis para crear una**](https://auth0.com/signup) antes de continuar.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no reemplazará sus formularios de inicio de sesión hasta que se hayan añadido un **Domain**, un **ID de cliente** y un **secreto de cliente** en la pestaña Basic de la página de configuración.
</Callout>

<div id="multisite-setup">
  ## Configuración multisitio
</div>

El plugin Login by Auth0 es compatible con las redes multisitio de WordPress. El plugin puede activarse en toda la red para proteger automáticamente los sitios de la red (una vez completada la configuración) o activarse solo en una parte de la red.

Hay varias formas de configurar una red de sitios en Auth0:

1. **Todos los sitios pueden compartir tanto una aplicación como una conexión de base de datos**

   1. Completa todos los pasos del Setup Wizard para el sitio principal.
   2. Configura manualmente todos los demás sitios con el **Domain**, el **ID de cliente** y el **secreto de cliente** del sitio principal en la pestaña Basic de la página de configuración de Auth0.
   3. Actualiza los **Allowed Callback URLs**, **Allowed Web Origins** y **Allowed Logout URLs** de la aplicación para incluir cada sitio (puedes usar comodines si tu red usa subdominios).
2. **Cada sitio puede tener su propia aplicación y compartir una conexión de base de datos**

   1. Completa todos los pasos del Setup Wizard para el sitio principal.
   2. Luego, [crea manualmente una aplicación para cada sitio](/docs/es/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0) y agrega cada una a la conexión de base de datos creada anteriormente.
   3. Agrega los valores de **Domain**, **ID de cliente** y **secreto de cliente** en la pestaña Basic de la página de configuración de Auth0 para cada sitio.
3. **Cada sitio puede tener su propia aplicación y su propia conexión de base de datos.** En este caso, completa todos los pasos del Setup Wizard para cada sitio.

Cada una de las opciones anteriores tiene sus ventajas y desventajas. La opción 1 implica la menor cantidad de entidades distintas que administrar en Auth0, pero si tu red tiene cientos de sitios y no usas subdominios, podrías encontrarte con limitaciones en la cantidad de URL de devolución de llamada. La opción 2 requerirá administrar muchas Applications distintas, pero te permitirá configurar de forma diferente la aplicación de cada sitio.

Como siempre, si tienes alguna pregunta sobre este proceso de configuración, [crea una publicación en nuestra Community y etiquétala como &quot;wordpress&quot;](https://community.auth0.com/tags/wordpress).

<div id="setup-wizard">
  ## Setup Wizard
</div>

El Setup Wizard intentará crear todos los componentes necesarios para usar Auth0 en su sitio de WordPress. Si ya tiene una aplicación o una conexión de base de datos que desea usar, consulte los pasos de configuración manual a continuación.

<div id="option-1-standard-setup">
  ### Opción 1: Setup estándar
</div>

Esto creará y configurará una aplicación y una conexión de base de datos para este sitio.

Primero, [genera un token de la Management API](https://auth0.com/docs/api/management/v2/get-access-tokens-for-test#get-access-tokens-manually). Una vez generado el token, toma nota del nombre de dominio usado en el campo **Identifier** de la pestaña **Settings**. Por ejemplo, si tu Identifier es `https://tenant-name.auth0.com/api/v2/`, entonces el dominio del tenant es `tenant-name.auth0.com`. Para obtener más información sobre los dominios de tenant, consulta [Crear tenants](/docs/es/get-started/auth0-overview/create-tenants).

De vuelta en el Setup Wizard del administrador de WordPress, haz clic en **Standard**. En la ventana modal que aparece, haz clic en **Start Standard Setup**.

Introduce el dominio del tenant y el token de API indicados arriba. Este token solo se usa durante el proceso de configuración y no se guardará en la base de datos.

Si la primera parte de la configuración se completa correctamente, verás la pantalla &quot;Configura tus conexiones sociales&quot;. Haz clic en **Next** para continuar el proceso de configuración migrando tu cuenta de administrador.

Este paso conecta tu usuario de WordPress con un usuario de Auth0 que te autoriza a iniciar sesión. Puedes elegir la misma contraseña que tu cuenta de administrador o una distinta, pero asegúrate de que cumpla con las políticas de fortaleza de la contraseña de la conexión de base de datos que se está usando. Para obtener más información, consulta [Fortaleza de la contraseña en Auth0 Database Connections: políticas de contraseña](/docs/es/authenticate/database-connections/password-strength#password-policieslicies).

El Setup Wizard debe ejecutarse hasta completarse para que tu sitio quede configurado correctamente. Si el Wizard falla por cualquier motivo antes de la pantalla &quot;configuración correcta&quot;, revisa el registro de errores del plugin en **wp-admin &gt; Auth0 &gt; Error Log** y los pasos a continuación para identificar la incidencia.

Para reiniciar el proceso por completo, elimina cualquier aplicación o conexión de base de datos que se haya creado en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. En WordPress, ve a **Auth0 &gt; Settings &gt; Basic**, elimina los campos Domain, <Tooltip tip="secreto de cliente: secreto que usa un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip> y <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>, y haz clic en **Save**. Luego, haz clic en **Setup Wizard** en el menú de administración para volver a iniciar el proceso.

Si aún no puedes instalarlo, [publica un hilo en nuestra Community](https://community.auth0.com/tags/wordpress) con los mensajes de error que ves en el Error Log, y con gusto te ayudaremos.

<div id="option-2-user-migration-setup">
  ### Opción 2: Configuración de la Migración de usuarios
</div>

Esto creará y configurará una aplicación y una conexión de base de datos, además de migrar los datos desde tu base de datos de WordPress. Esto requiere una conexión entrante desde los servidores de Auth0 y no se puede cambiar más adelante sin perder datos. Para obtener más información, consulta [Migración de usuarios en Login by Auth0 WordPress Plugin](/docs/es/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).

::: warning
Si tienes más de una conexión de base de datos personalizada en Auth0, deberás asegurarte de que los ID de usuario utilicen un espacio de nombres para evitar conflictos. Esto se hace automáticamente en los sitios que instalan la versión 3.11.0 o posterior. Si tus conexiones se están creando o se crearon con una versión anterior, consulta los pasos de troubleshooting en [Migración de usuarios en Login by Auth0 WordPress Plugin](/docs/es/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).
:::

Una vez completado el proceso de configuración, cierra sesión en tu sitio de WordPress e intenta volver a iniciarla con tus credenciales actuales de WordPress en el formulario de inicio de sesión de Auth0. Esto debería crear un usuario de Auth0 vinculado a tu cuenta de WordPress.

<div id="option-3-manual-setup">
  ### Opción 3: Configuración manual
</div>

Se omitirá la configuración automática y podrá crear y configurar su propia Aplicación y conexión de base de datos (consulte a continuación). Debe usar esta opción si quiere que este sitio utilice una Aplicación o conexión de base de datos existente.

<div id="option-4-import-setup">
  ### Opción 4: Importar configuración
</div>

El sitio también puede configurarse importando la configuración de otro sitio. Esto resulta útil si estás migrando entre entornos o si ya tienes un sitio de WordPress similar configurado.

<div id="setup-complete">
  ## Configuración completada
</div>

Cuando veas la pantalla &quot;Done&quot;, Auth0 estará habilitado y listo para aceptar inicios de sesión y, si está configurado, registros de usuarios.

Este es un buen momento para comprobar que lo básico funciona en tu sitio antes de cambiar cualquiera de los ajustes predeterminados:

1. Cierra sesión en WordPress y confirma que el formulario de Auth0 ahora aparece en `/wp-login.php`.
2. Inicia sesión con el usuario de Auth0 creado anteriormente.
3. Cierra sesión e intenta crear una cuenta con una dirección de correo electrónico diferente (si tienes activada la opción &quot;Anyone can register&quot; en la configuración general de WordPress).
4. Intenta iniciar sesión usando una conexión social (si las has activado).

Ahora estás listo para [configurar la aplicación](/docs/es/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0).

<div id="manual-setup">
  ## Configuración manual
</div>

El plugin puede configurarse con el Setup Wizard integrado (explicado más abajo) o manualmente, creando una aplicación y asignando conexiones. El proceso de configuración totalmente manual puede ser útil si tienes problemas con el wizard, si ya has pasado antes por el proceso de configuración o si quieres compartir una conexión de base de datos entre aplicaciones.

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#) y selecciona **Create**.
2. Asigna a tu aplicación un nombre descriptivo, selecciona **aplicaciones web clásicas** y, a continuación, selecciona **Create**.
3. Configura la aplicación para tu instalación de WordPress. Para obtener más información sobre cómo configurarla, consulta [Configure Login by Auth0](/docs/es/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0) y sigue todas las secciones hasta &quot;Actualizar la configuración de Auth0 en WordPress&quot;.

Una vez completados los pasos anteriores, tu sitio estará configurado y listo para usar.
