Aquí tienes algunas preguntas frecuentes sobre solución de problemas. Si los elementos a continuación no resuelven tu incidencia, considera las siguientes alternativas:

* Si estás configurando el plugin por primera vez o tienes problemas para que los usuarios inicien sesión, lee [Configurar Login by Auth0](/docs/es/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0).
* Si encontraste un error en el código del plugin, [envía una incidencia](https://github.com/auth0/wordpress/issues) o [crea una pull request](https://github.com/auth0/wordpress/pulls) en GitHub.
* Si tienes preguntas sobre cómo usar Auth0 o el plugin, [publica en nuestro sitio de Community](https://community.auth0.com/) o [crea una solicitud en el foro de soporte del plugin en WordPress](https://wordpress.org/support/plugin/auth0/).
* Para ver documentación adicional y encontrar más respuestas, [visita nuestro sitio de soporte](https://support.auth0.com/center/s/). Los clientes con un plan de Auth0 de pago pueden enviar un ticket de soporte para obtener una respuesta rápida.

<div id="im-seeing-the-error-message-invalid-state-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Invalid state&quot;, que me impide iniciar sesión
</div>

La validación de `state` es una funcionalidad de seguridad añadida en la [versión 3.6.0](https://github.com/auth0/wordpress/releases/tag/3.6.0). Una URL de devolución de llamada almacenada en caché suele provocar este mensaje de error (consulta la configuración de la aplicación para ver qué URL de devolución de llamada no deberían almacenarse en caché). Para obtener más información, lee [Solucionar errores &quot;Invalid State&quot; del WordPress Plugin](/docs/es/customize/integrations/cms/wordpress-plugin/troubleshoot-wordpress-plugin-invalid-state-errors).

<div id="im-having-an-issue-logging-in-or-changing-emailpassword-using-a-custom-database">
  ## Tengo problemas para iniciar sesión o cambiar el correo electrónico o la contraseña con una base de datos personalizada
</div>

Para conocer posibles soluciones, lee [Migración de usuarios en Login by Auth0 WordPress Plugin](/docs/es/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).

<div id="im-seeing-the-error-message-invalid-id-token-or-expired-id-token-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Token de identidad no válido&quot; o &quot;Token de identidad expirado&quot;, que me impide iniciar sesión
</div>

Esto suele deberse a que el servidor tiene la hora mal configurada. Si el mensaje de error incluye &quot;used too early,&quot; significa que la hora de su servidor está adelantada. Si indica que el token ha expirado, entonces la hora del servidor está demasiado atrasada. Es normal que haya una diferencia horaria entre dos servidores. Ejecute `echo date(DateTime::ISO8601)` en PHP en su servidor y compare ese valor, incluidos los segundos, con la hora UTC actual. Si la hora de su servidor difiere de la hora UTC en más de 60 segundos (el margen predeterminado), tendrá que configurar un margen mayor para compensar el desfase del reloj de su servidor. Puede pegar el siguiente código en el archivo `functions.php` de su tema o en cualquier otro lugar donde se ejecute después de que se cargue el plugin y antes de que se ejecute el hook de login:

```js wrap lines
add_filter( 'auth0_jwt_leeway', function( $default_leeway ) { return 120; } );
```

Esto daría un margen de 120 segundos. Puede que tengas que ajustarlo según el desfase horario de tu servidor.

<div id="i-see-the-error-message-this-account-does-not-have-an-email-associated-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Esta cuenta no tiene un correo electrónico asociado...&quot; que me impide iniciar sesión
</div>

Si recibes este error, asegúrate de solicitar una dirección de correo electrónico a cada proveedor en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en **Connections &gt; Social** (expande cada proveedor). Ten en cuenta que no todos los proveedores devuelven direcciones de correo electrónico de los usuarios (por ejemplo, X). Si esto ocurre, siempre puedes añadir una dirección de correo electrónico a cualquier usuario que haya iniciado sesión a través del Auth0 Dashboard (o de la API). Consulta **Users &gt; Edit**.

Para las conexiones que no proporcionan un indicador `email_verified` (algunas conexiones empresariales no lo incluyen), o para omitir esta validación en conexiones sociales específicas, añade la estrategia de esa conexión en el campo &quot;Skip Strategies&quot;. Este campo se encuentra debajo del interruptor **Require Verified Email**, al que puedes acceder desde **wp-admin** &gt; **Auth0** &gt; **Settings** &gt; **Advanced**.

**Este campo solo debe usarse si es necesario, ya que omite las medidas de seguridad recomendadas por Auth0.**

<div id="i-see-the-error-message-there-is-a-user-with-the-same-email-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Hay un usuario con el mismo correo electrónico&quot;, que me impide iniciar sesión
</div>

Esto significa que hay un usuario en WordPress con el mismo correo electrónico que se está usando para iniciar sesión, pero asociado a otro usuario de Auth0. Si estás probando el plugin o quieres asociar el usuario existente con el nuevo usuario de Auth0 en su lugar:

1. Inicia sesión como administrador
2. Ve a **wp-admin &gt; Users** y busca el correo electrónico que se está usando
3. Abre el perfil del usuario y desplázate hasta el final
4. Haz clic en **Eliminar datos de Auth0** y confirma

Si tienes 2 cuentas de usuario en Auth0 con la misma dirección de correo electrónico, este mensaje de error seguirá apareciendo. Te recomendamos que [vincules cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking/link-user-accounts).

<div id="i-see-the-error-message-failed-cross-origin-authentication-or-no-verifier-returned-from-client-in-my-browsers-console-logs-when-trying-to-log-in">
  ## Veo el mensaje de error &quot;Failed cross origin authentication&quot; o &quot;No verifier returned from client&quot; en los registros de la consola de mi navegador al intentar iniciar sesión
</div>

Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), selecciona tu aplicación de WordPress para ver su configuración y comprueba que **Allowed Callback URLs** y **Allowed Origins (CORS)** coincidan con tu sitio de WordPress.

Si usas un navegador basado en Chromium, asegúrate de no tener desactivadas las cookies de terceros. Para obtener más información, consulta [Cross-Origin Authentication](/docs/es/authenticate/login/cross-origin-authentication).

<div id="i-need-to-rerun-the-setup-wizard-but-i-dont-see-that-menu-option-anymore">
  ## Necesito volver a ejecutar el Setup Wizard, pero ya no veo esa opción en el menú.
</div>

Esto significa que el plugin ya está configurado con un Domain, <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip> y <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>. Ejecutar el Setup Wizard por segunda vez puede producir resultados impredecibles. Si estás configurando WordPress por primera vez y quieres empezar de nuevo antes de que se haya iniciado sesión:

1. Ve a **wp-admin** &gt; **Auth0** &gt; **Settings** &gt; **Basic**.
2. Elimina el Domain y el ID de cliente. Desplázate hacia abajo y haz clic en **Save Changes**.
3. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
4. Busca la aplicación que creó WordPress (su nombre debería ser el nombre de tu sitio de WordPress).
5. Selecciona la aplicación para ver su configuración. Desplázate hacia abajo y selecciona **Delete Application**.
6. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database).
7. Busca la conexión que creó WordPress (su nombre debería ser el nombre de tu sitio de WordPress con el prefijo &quot;DB-&quot;).
8. Selecciona la conexión para ver su configuración. Desplázate hacia abajo y selecciona **I Want To Delete This Connection.** Ten en cuenta que esto eliminará la conexión y todos los usuarios que se hayan creado en ella.
9. Vuelve a WordPress. Ahora verás la opción Setup Wizard en Auth0 dentro del menú de administración.

<div id="how-do-i-setup-passwordless-login">
  ## ¿Cómo configuro el inicio de sesión Passwordless?
</div>

El inicio de sesión con <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> es posible en cualquier sitio web habilitado con Auth0 mediante correo electrónico o SMS. Para que esto funcione en tu sitio de WordPress:

1. Activa **Passwordless Login** en la vista **Funcionalidades** de la configuración del plugin y guarda los cambios.
2. Ve a [Auth0 Dashboard &gt; Authentication &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless).

* Para usar correo electrónico, activa la conexión **Email** y modifica la configuración si lo deseas. Esto habilitará el inicio de sesión con código por correo electrónico (se enviará un código por correo electrónico a los usuarios, que luego deberán escribir en el formulario de inicio de sesión de tu sitio).
* Para usar un &quot;magic link&quot; (un enlace enviado por correo electrónico que iniciará sesión automáticamente a los usuarios), agrega `{passwordlessMethod: 'code'}` al campo &quot;Extra Settings&quot; en la pestaña **Advanced** de la configuración del plugin.
* Para usar el inicio de sesión por SMS, activa la conexión **SMS** y sigue los pasos para configurar una cuenta de desarrollador de Twilio (según el uso, esto puede requerir una cuenta de pago de Twilio).

El formulario de inicio de sesión de Auth0 seleccionará un método Passwordless según la conexión que actives arriba. Si tienes ambas conexiones activas, se usará el correo electrónico de forma predeterminada. En ese caso, desactiva la conexión de correo electrónico para mostrar SMS o agrega `sms` al campo **Connections** en la pestaña **Advanced** de la configuración del plugin.

<div id="i-have-two-accounts-for-the-same-user-in-wordpress">
  ## Tengo dos cuentas para un mismo usuario en WordPress
</div>

En algunas situaciones, puede ocurrir que un usuario tenga dos cuentas. WordPress te permite fusionar usuarios eliminando una de las cuentas y asignando el contenido de esa cuenta a otro usuario. Ve a wp-admin &gt; Users, selecciona la cuenta que quieres eliminar y, en el cuadro de diálogo de confirmación, selecciona otro usuario al que transferir el contenido.

<div id="my-configuration-is-wrong-and-i-cant-authenticate-using-auth0-is-there-another-way-to-access-the-plugin">
  ## Mi configuración es incorrecta y no puedo autenticarme con Auth0. ¿Hay alguna otra forma de acceder al plugin?
</div>

Se puede acceder al plugin con credenciales válidas de WordPress a través del inicio de sesión habitual de WordPress añadiendo `?wle` a la URL de inicio de sesión. Por ejemplo: `http://yourdomain.com/wp-login.php?wle`.

<div id="i-am-having-problems-when-a-user-logs-in-where-can-i-find-a-log-of-what-is-happening">
  ## Tengo problemas cuando un usuario inicia sesión. ¿Dónde puedo encontrar un registro de lo que está pasando?
</div>

El plugin incluye un registro de errores donde puedes ver qué ha ocurrido. Accede a él a través del subelemento **Error Log** del menú del plugin **Auth0**. Puedes encontrar información adicional en [Auth0 Dashboard &gt; Monitoreo &gt; Logs](https://manage.auth0.com/#/logs).

<div id="how-can-i-show-the-widget-or-shortcode-in-signup-mode-by-default">
  ## ¿Cómo puedo mostrar el widget o shortcode en modo de registro de forma predeterminada?
</div>

Puedes usar la opción `Extra configuration` del widget (o el atributo `extra_conf` en el shortcode) y agregar este JSON `{"mode":"signup" }` para forzar que el plugin se muestre en este modo.

<div id="when-using-a-plugin-to-force-the-login-the-user-is-not-logged-in">
  ## Al usar un plugin para forzar el inicio de sesión, el usuario no queda autenticado
</div>

Esto suele deberse a una página en caché después del inicio de sesión. Consulta con tu proveedor de hosting qué estrategias pueden ayudar a mitigarlo o intenta añadir un parámetro a la URL para invalidar la caché. Para obtener más información, [usa este gist en GitHub](https://gist.github.com/joshcanhelp/e3eb693749f0fe66aad097c3bbb3b415).

<div id="the-user-is-not-logged-in-when-using-the-force-login-plugin">
  ### El usuario no inicia sesión al usar el plugin &quot;Force Login&quot;
</div>

Esto se debe a que no se ha añadido la URL de devolución de llamada a la lista de permitidos. Prueba a añadir este código al filtro `my_forcelogin_whitelist`:

```javascript lines
function wp_auth0_forcelogin_whitelist( $whitelist ) {
  if ( ! empty( $_GET['auth0'] ) ) {
    $whitelist[] = site_url($_SERVER['REQUEST_URI']);
  }
  return $whitelist;
}
add_filter('v_forcelogin_whitelist', 'wp_auth0_forcelogin_whitelist', 10, 1);
```

<div id="how-can-i-redirect-the-users-to-a-specific-url-after-login">
  ## ¿Cómo puedo redirigir a los usuarios a una URL específica después de iniciar sesión?
</div>

Antes de ejecutarse, todos los redireccionamientos se validan con `wp_safe_redirect()`. Si intentas redirigir a un dominio distinto de tu dominio principal, agrega ese dominio a la validación mediante el filtro `allowed_redirect_hosts` del núcleo de WordPress. Para obtener más información, consulta [apply&#95;filters( &#39;allowed&#95;redirect&#95;hosts&#39;, string[] $hosts, string $host ) en wordpress.org](https://developer.wordpress.org/reference/hooks/allowed_redirect_hosts/).

<div id="on-the-login-page">
  ### En la página de inicio de sesión
</div>

Este plugin aprovecha las funcionalidades de WordPress para funcionar sin problemas con la configuración predeterminada. Para agregar una redirección, puedes añadir el parámetro de consulta `redirect_to` a la URL cuando dirijas al usuario a la página de inicio de sesión. El plugin redirigirá al usuario a esta URL después de iniciar sesión correctamente.

También puedes usar la opción **URL de redirección de inicio de sesión** en la página de Configuración del plugin de Auth0. Esta URL se usará para redirigir al usuario cuando no se proporcione el parámetro `redirect_to`.

<div id="using-the-widget">
  ### Uso del widget
</div>

El widget redirigirá automáticamente a la misma página en la que se encontraba el usuario antes de la autenticación. Puedes cambiar este comportamiento con la configuración `Redirect after login:`.

<div id="using-the-shortcode">
  ### Uso del shortcode
</div>

El shortcode redirigirá automáticamente a la misma página en la que estaba el usuario antes de la autenticación. Puedes cambiar este comportamiento con el atributo `redirect_to`.

<div id="how-can-i-migrate-my-wordpress-users-to-auth0">
  ## ¿Cómo puedo migrar mis usuarios de WordPress a Auth0?
</div>

La versión actual del plugin no ofrece una forma de migrar usuarios a Auth0 automáticamente, pero tienes varias opciones:

* El plugin expone dos endpoints para marcar tu conexión de base de datos personalizada en modo de **importación a Auth0**. Para obtener más información sobre estos endpoints, consulta [Configure Automatic Migration from Your Database](/docs/es/manage-users/user-migration/configure-automatic-migration-from-your-database).
* Exporta tus datos de usuario a un archivo JSON y súbelo para importarlos por lotes a Auth0. Al principio, tus usuarios tendrán que restablecer sus contraseñas al iniciar sesión con Auth0, ya que Auth0 no puede descifrar las contraseñas de WordPress durante la migración. Para aprender a generar el archivo JSON, consulta [Bulk User Imports](/docs/es/manage-users/user-migration/bulk-user-imports). Luego tendrás que subir el archivo mediante el endpoint [Import Users](https://auth0.com/docs/api/management/v2/jobs/post-users-imports) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.
* [Usa este script](https://gist.github.com/glena/b31716e3c8fe48927be2) y el endpoint XML-RPC de WordPress para configurar el flujo de migración mediante una conexión de base de datos personalizada. Para obtener más información sobre cómo configurar el flujo de migración, consulta [Configure Automatic Migration from Your Database](/docs/es/manage-users/user-migration/configure-automatic-migration-from-your-database). Para obtener más información sobre el endpoint XML-RPC de WordPress, consulta [XML-RPC Support on wordpress.org](https://codex.wordpress.org/XML-RPC_Support).

<div id="the-form_title-setting-is-ignored-when-i-set-up-the-dict-setting">
  ## La configuración form_title se ignora cuando configuro dict
</div>

Internamente, el plugin usa la configuración dict para cambiar el título del widget de Auth0. Cuando configuras el campo dict, este sustituye a form&#95;title.

Para cambiar form&#95;title en este caso, debes agregar el siguiente atributo al JSON de dict:

```json lines
{
    "signin": {
        "title": "The desired form title"
    }
}
```

<div id="how-can-i-modify-the-embedded-auth0-login-form">
  ## ¿Cómo puedo modificar el formulario de inicio de sesión integrado de Auth0?
</div>

Hay muchas opciones en la pestaña **Appearance** de la página de configuración del plugin que pueden cambiar la apariencia del formulario de inicio de sesión integrado en su sitio (página `wp-login/php`, shortcodes o widgets). Para obtener más información sobre estas opciones, lea [Configure Login by Auth0](/docs/es/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0). Esto no afectará al formulario de inicio de sesión de la página de <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> alojada en Auth0.

También hay un campo llamado &quot;Extra Settings&quot; en la pestaña **Advanced** que acepta un objeto JSON válido con todos los ajustes que quiera definir. Esto anulará cualquier cambio realizado en la pestaña **Appearance**. Para ver todas las opciones de configuración posibles, lea [Lock Configuration Options](/docs/es/libraries/lock/lock-configuration).

Las hojas de estilo externas y los archivos JavaScript deben cargarse en su tema usando el hook `wp_enqueue_scripts` para shortcodes/widgets y el hook `login_enqueue_scripts` para `wp-login.php`. Para obtener más información, lea [`do_action( 'wp_enqueue_scripts' )` en wordpress.org](https://developer.wordpress.org/reference/hooks/wp_enqueue_scripts/) y [`do_action( 'login_enqueue_scripts' )` en wordpress.org](https://developer.wordpress.org/reference/hooks/login_enqueue_scripts/).

<div id="the-session-expires-too-soon">
  ## La sesión expira demasiado pronto
</div>

El plugin de Auth0 no gestiona las sesiones, sino que utiliza la configuración de WordPress. De forma predeterminada, las sesiones de los usuarios se mantienen activas durante dos días. Puedes habilitar la opción `Remember users session` en la pestaña **Advanced** de la configuración del plugin para que las sesiones permanezcan activas hasta 14 horas.

<div id="how-do-i-implement-a-refresh-token">
  ## ¿Cómo implemento un token de actualización?
</div>

Hemos implementado parámetros adicionales en los métodos de inicio de sesión utilizados por el Plugin para permitir <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Tokens">tokens de actualización</Tooltip>. Usa el filtro `auth0_auth_scope` junto con la Action `auth0_user_login` para hacerlo.

<div id="profile-data-saved-in-wordpress-is-not-being-synced-to-the-auth0-user-account">
  ## Los datos del perfil guardados en WordPress no se sincronizan con la cuenta de usuario de Auth0.
</div>

Esta es una limitación actual del plugin, pero es algo que estamos evaluando para una versión futura. La única excepción es la contraseña del usuario. Si la contraseña se cambia en WordPress y cumple la política de seguridad establecida para la conexión de base de datos en uso, esa contraseña también se actualizará para el usuario de Auth0. En una versión futura, añadiremos un mensaje de error para detener el proceso si la contraseña no es lo bastante segura.

<div id="how-do-i-migrate-from-social-login-with-auth0-to-login-by-auth0">
  ## ¿Cómo migro de &quot;Social Login with Auth0&quot; a &quot;Login by Auth0&quot;?
</div>

Históricamente, Auth0 ha mantenido dos plugins de WordPress:

* [Login by Auth0](https://wordpress.org/plugins/auth0/)
* [Social Login with Auth0](https://wordpress.org/plugins/social-login-with-auth0/)

Estos dos plugins son prácticamente iguales, pero **Social Login with Auth** no recibirá más actualizaciones después de la versión 3.7.0 (publicada el 13 de agosto de 2018). Migrar de **Social Login with Auth** a **Login by Auth0** es sencillo y no provocará ninguna pérdida de datos en Auth0 ni en WordPress.

Pasar de **Social Login with Auth** a **Login by Auth0** actualizará el número de versión que ves, así que asegúrate de probar este cambio primero en un servidor de staging o de desarrollo (igual que harías al actualizar el plugin en wp-admin). Además, es posible que los inicios de sesión no funcionen durante el proceso de migración, así que asegúrate de usar un plugin de modo de mantenimiento o de completar la migración en horas de baja actividad.

La forma más sencilla de migrar es mediante (S)FTP:

1. Inicia sesión en el sitio de WordPress como administrador.
2. [Descargar Login by Auth0](https://downloads.wordpress.org/plugin/auth0.zip) y descomprímelo localmente.
3. Desactiva el plugin **Social Login with Auth0** desde la pantalla de WordPress admin &gt; Plugins.
4. Inicia sesión en el servidor al que quieres migrar y ve a `wp-content/plugins`.
5. Mueve la carpeta `social-login-with-auth0` fuera de la carpeta de plugins para hacer una copia de seguridad de su contenido.
6. Sube la nueva carpeta del plugin `auth0` al directorio de plugins.
7. Activa el nuevo plugin &quot;Login by Auth0&quot; desde la pantalla de WordPress **Admin** &gt; **Plugins**.

Si no puedes acceder al sitio mediante FTP, también puedes realizar el proceso directamente desde WordPress admin:

1. Inicia sesión en el sitio de WordPress como administrador.
2. Ve a **Auth0** &gt; **Import-Export Settings**.
3. Haz clic en **Export Settings** y luego en **Export**.
4. Desactiva el plugin **Social Login with Auth0** desde la pantalla de WordPress **Admin** &gt; **Plugins**.
5. Elimina el plugin **Social Login with Auth0** y confirma.
6. Ve a **Plugins** &gt; **Add New** y busca &quot;Auth0&quot;.
7. En el plugin **Login by Auth0** (asegúrate de comprobar el nombre), haz clic en **Install Now**.
8. Cuando termine, haz clic en **Activate**.
9. Revisa **Auth0** &gt; **Settings** y asegúrate de que la configuración anterior siga ahí. Si no es así:

   1. Ve a **Auth0** &gt; **Import-Export Settings**.
   2. Pega el JSON de configuración exportado anteriormente y haz clic en **Import**.
10. Elimina por completo el JSON exportado del archivo de configuración (contiene información confidencial).

Ahora todo debería funcionar como se espera con el nuevo plugin y las actualizaciones se reanudarán con normalidad.
