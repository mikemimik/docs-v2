Auth0 puede ayudar a simplificar drásticamente el proceso de autenticación de las aplicaciones de SharePoint. Auth0 gestionará un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> que luego podrá usar para llamar a las API de SharePoint.

No necesitará ninguna biblioteca especial. Puede usar cualquiera de los SDKs compatibles con Auth0.

<div id="step-1-register-your-application-in-auth0">
  ## Paso 1. Registra tu aplicación en Auth0
</div>

Solo tienes que registrar una nueva aplicación en Auth0 como lo harías normalmente: **Applications &gt; NEW**. Consulta cualquiera de los SDKs disponibles para ver instrucciones detalladas. Ten a mano el `client_id`, ya que lo necesitarás en el siguiente paso.

<div id="step-2-create-a-package-for-your-application">
  ## Paso 2. Crea un paquete para tu aplicación
</div>

Necesitas obtener un **<Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** y un **<Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>** para tu aplicación. Hay muchas formas de registrar tu aplicación según el uso que se le vaya a dar.

[Register SharePoint Add-ins (Microsoft Docs)](https://docs.microsoft.com/en-us/sharepoint/dev/sp-add-ins/register-sharepoint-add-ins) explica las distintas formas de registrar tu aplicación en SharePoint. En este paso del tutorial, se usará la forma más sencilla: el autorregistro en un tenant específico (el tuyo).

<div id="open-sharepoint-online">
  ### Abrir SharePoint Online
</div>

La URL del Dashboard es `https://{your Office365 tenant}.sharepoint.com/_layouts/15/appregnew.aspx`

<div id="complete-the-information-in-the-form">
  ### Completa la información del formulario:
</div>

Como Auth0 actúa de intermediario entre tu aplicación y la infraestructura de Office 365, debes usar esta URL para la aplicación:

**Dominio de la aplicación**:

export const codeExample1 = `{yourDomain}`;

<AuthCodeBlock children={codeExample1} language="http" />

**URI de redirección**:

export const codeExample2 = `https://{yourDomain}/login/callback?SP_APP_TOKEN&connection=CONNECTION&client_id={yourClientId}&redirect_uri={https://yourApp/callback}`;

<AuthCodeBlock children={codeExample2} language="http" />

* `connection` es simplemente el nombre que usarás en las conexiones de Auth0 (como &quot;sharepoint&quot;).
* `client_id` identifica tu aplicación en Auth0 (creada en el paso 1).
* `redirect_uri` es la ubicación de tu aplicación a la que llegarán finalmente tus usuarios cuando se complete todo el proceso. Si no la especificas, siempre será la URL de devolución de llamada de la aplicación definida en Auth0 (podría ser localhost)

<div id="package-the-app-and-upload-to-sharepoint">
  ### Empaqueta la aplicación y súbela a SharePoint:
</div>

Completa la información del manifiesto de tu aplicación en Visual Studio:

<Frame>![SharePoint - Visual Studio - Manifiesto de la aplicación](/docs/images/cdy7uua7fh8z/36dscLHlLHZjJx1rLgrQ7N/8d64ec7a51f4e053627ecb5dfd3c7707/90SEc.png)</Frame>

Ten en cuenta que la `Query string` será exactamente igual que la `URI de redirección` que completaste antes. Luego, haz clic con el botón derecho en el proyecto y selecciona `Publish`:

Crea un **Publishing Profile** (tendrás que introducir el mismo **ID de cliente** y **secreto de cliente** obtenidos en el Dashboard de SharePoint).

Haz clic en **Package** y sube el archivo resultante a SharePoint.

<div id="step-3-create-the-connection-in-auth0">
  ## Paso 3. Crear la conexión en Auth0
</div>

El último paso de la integración es agregar una conexión de SharePoint en Auth0.

Necesitarás:

* `Connection Name`. Es un nombre arbitrario. Debe coincidir con lo que ingresaste en el paso 2.
* `ID de cliente` &amp; `secreto de cliente`. También deben coincidir con lo que ingresaste en el paso 2.
* `Test SharePoint Site Url`. Esta es la URL del sitio de SharePoint que se usa para probar la conexión (por ejemplo, al pulsar el botón &#39;Try&#39; en el Dashboard). Esto nunca se usa en runtime, porque los usuarios siempre seguirán el enlace a tu sitio desde SharePoint.

Los usuarios instalarán tu aplicación desde Office Marketplace. Cuando hagan clic en el enlace, se les redirigirá a Auth0, que negociará el token de acceso por ti y, por último, los dirigirá a tu aplicación. Tu aplicación recibirá un `perfil de usuario` con un aspecto similar a este:

<Frame>![Office Marketplace - Redirect -  User Profile Example](/docs/images/cdy7uua7fh8z/4piqwmx1ONWMd3kjTFMq4I/714545f5cda3ab1d1dd5c620bc96b083/8Xp6x.png)</Frame>

Se incluirán las siguientes propiedades: `cacheKey`, `refresh_token`, `host` y `site`. Estas te permitirán llamar a las API de SharePoint (como las listas).
