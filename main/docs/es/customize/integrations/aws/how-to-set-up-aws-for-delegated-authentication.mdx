Puede configurar la autenticación delegada de Amazon Web Services (AWS) con Auth0. Para obtener más información, consulte [Configurar Amazon Web Services para el inicio de sesión único](/docs/es/customize/integrations/aws/configure-amazon-web-services-for-sso) y [Proteger los endpoints de AWS API Gateway mediante autorizadores personalizados](/docs/es/customize/integrations/aws/aws-api-gateway-custom-authorizers).

<div id="create-a-saml-provider-in-aws">
  ## Crear un proveedor SAML en AWS
</div>

1. Inicia sesión en AWS y ve a la [consola de IAM](https://console.aws.amazon.com/iam). En el menú de navegación de la izquierda, selecciona **Identity Providers**. Haz clic en **Create Provider**. Configura los siguientes parámetros:

   | Parámetro         | Descripción y valor de ejemplo                                                                                                                                                                  |
   | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | Provider Type     | El tipo de proveedor. Establécelo como [`SAML`](/docs/es/glossary?term=%60SAML%60)                                                                                                                 |
   | Provider Name     | Un nombre descriptivo para el proveedor, como `auth0SamlProvider`                                                                                                                               |
   | Metadata Document | Carga el archivo que contiene los metadatos de Auth0, que encontrarás en **Dashboard &gt; Applications &gt; Application Settings &gt; Advanced Settings &gt; Endpoints &gt; SAML Metadata URL** |
2. Haz clic en **Next Step**.
3. Verifica la configuración y haz clic en **Create** si todo es correcto.

<div id="create-a-role-for-your-saml-provider">
  ## Crea un rol para tu proveedor SAML
</div>

Para usar el proveedor, debes crear un rol de IAM que use el proveedor en la política de confianza del rol.

1. En la consola de IAM, ve a [Roles](https://console.aws.amazon.com/iam/home#/roles). Haz clic en **Create role**. Se te redirigirá a la página **Trust**. Selecciona **Saml 2.0 federation** en **Select type of trusted entity**. Introduce los siguientes valores:

   | Parámetro      | Valor                     |
   | -------------- | ------------------------- |
   | Proveedor SAML | El nombre de tu nuevo rol |
   | Atributo       | `SAML:iss`                |
   | Valor          | `urn:{yourDomain}`        |
2. Haz clic en **Next: Permissions** para continuar.
3. Tendrás que adjuntar políticas de permisos a tu nuevo rol. Adjuntarás una política personalizada. Para crearla, haz clic en **Create Policy**.
4. En el editor de **Create policy** que se abre, cambia a la pestaña **JSON**. Introduce una política personalizada.

   ```json lines
   {
     "Version": "2012-10-17",
     "Statement": [{
         "Effect": "Allow",
         "Action": [
           "*"
         ],
         "Resource": [
         "arn:aws:s3:::{yourBucket}/<%= '${saml:sub}' %>",
         "arn:aws:s3:::{yourBucket}/<%= '${saml:sub}' %>/*"]
     }]
   }
   ```

   Esto define los permisos que tendrán en AWS los usuarios a los que se les asigne este rol. Haz clic en **Review policy**.
5. Revisa la política que has creado. Asegúrate de indicar un **Name** para tu política y, opcionalmente, una **Description**.
6. Haz clic en **Create policy** cuando termines. Si todo sale bien, verás el siguiente mensaje que confirma la creación de tu nueva política: &quot;auth0SamlRolePolicy has been created&quot;.
7. Al volver al asistente de creación de roles (deberías estar en el paso **2 - Permissions**), busca la nueva política que acabas de crear y marca su casilla para adjuntarla a tu rol. Te recomendamos usar el filtro **Customer managed** para encontrarla.
8. Haz clic en **Next: Review** para continuar.
9. Revisa la información de tu rol, indica un **Role name** y, opcionalmente, una **Role description**. También verás la política que adjuntaste. Si todo parece correcto, haz clic en **Create role** para continuar.
10. Una vez creado, podrás encontrar tus roles en la página principal de **Roles**.

<div id="copy-the-arn-values">
  ## Copiar los valores de ARN
</div>

A continuación se indica dónde puede encontrar los valores de Provider y Role ARN.

<div id="provider-arn">
  ### Provider ARN
</div>

En la consola de IAM, ve a [Identity Providers](https://console.aws.amazon.com/iam/home#/providers). Selecciona el rol que te interese para abrir su página de resumen. Copia el valor de **Provider ARN**, que figura en primer lugar en **Summary**.

<div id="role-arn">
  ### Role ARN
</div>

En la consola de IAM, ve a [Roles](https://console.aws.amazon.com/iam/home#/roles). Selecciona el rol que te interesa para abrir su página Summary. Copia el valor de **Role ARN**, que aparece primero en **Summary**.
