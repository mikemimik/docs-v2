Puede configurar Amazon Web Services (AWS) para la autenticación delegada con Auth0. Para obtener más información, consulte [Configurar Amazon Web Services para el inicio de sesión único](/docs/es/customize/integrations/aws/configure-amazon-web-services-for-sso) y [Proteger los endpoints de AWS API Gateway mediante autorizadores personalizados](/docs/es/customize/integrations/aws/aws-api-gateway-custom-authorizers).

<div id="create-a-saml-provider-in-aws">
  ## Crear un proveedor SAML en AWS
</div>

1. Inicie sesión en AWS y vaya a la [consola de IAM](https://console.aws.amazon.com/iam). En el menú de navegación de la izquierda, seleccione **Proveedores de identidad**. Haga clic en **Crear proveedor**. Configure los siguientes parámetros:

   | Parámetro              | Descripción y valor de ejemplo                                                                                                                                                                                     |
   | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | Tipo de proveedor      | El tipo de proveedor. Establézcalo como [`SAML`](/docs/es/glossary?term=%60SAML%60)                                                                                                                                   |
   | Nombre del proveedor   | Un nombre descriptivo para el proveedor, como `auth0SamlProvider`                                                                                                                                                  |
   | Documento de metadatos | Cargue el archivo que contiene los metadatos de Auth0, que encontrará en **Dashboard &gt; Aplicaciones &gt; Configuración de la aplicación &gt; Configuración avanzada &gt; Endpoints &gt; URL de metadatos SAML** |
2. Haga clic en **Siguiente paso**.
3. Verifique la configuración y haga clic en **Crear** si todo es correcto.

<div id="create-a-role-for-your-saml-provider">
  ## Crea un rol para tu proveedor SAML
</div>

Para usar el proveedor, debes crear un rol de IAM con el proveedor en la política de confianza del rol.

1. En la consola de IAM, ve a [Roles](https://console.aws.amazon.com/iam/home#/roles). Haz clic en **Create role**. Se te redirigirá a la página **Trust**. Selecciona **Saml 2.0 federation** en **Select type of trusted entity**. Proporciona los siguientes valores:

   | Parámetro      | Valor                     |
   | -------------- | ------------------------- |
   | Proveedor SAML | El nombre de tu nuevo rol |
   | Attribute      | `SAML:iss`                |
   | Valor          | `urn:{yourDomain}`        |
2. Haz clic en **Next: Permissions** para continuar.
3. Tendrás que adjuntar políticas de permisos a tu nuevo rol. Adjuntarás una política personalizada. Para crearla, haz clic en **Create Policy**.
4. En el editor de **Create policy** que se abre, cambia a la pestaña **JSON**. Proporciona una política personalizada.

   ```json lines
   {
     "Version": "2012-10-17",
     "Statement": [{
         "Effect": "Allow",
         "Action": [
           "*"
         ],
         "Resource": [
         "arn:aws:s3:::{yourBucket}/<%= '${saml:sub}' %>",
         "arn:aws:s3:::{yourBucket}/<%= '${saml:sub}' %>/*"]
     }]
   }
   ```

   Esto define los permisos que tendrán en AWS los usuarios a los que se les asigne este rol. Haz clic en **Review policy**.
5. Revisa la política que has creado. Asegúrate de indicar un **Name** para tu política y, opcionalmente, una **Description**.
6. Haz clic en **Create policy** cuando termines. Si todo sale bien, verás el siguiente mensaje que confirma la creación de tu nueva política: &quot;auth0SamlRolePolicy has been created&quot;.
7. De vuelta en el asistente de creación de roles (deberías estar en el paso **2 - Permissions**), busca la nueva política que acabas de crear y marca su casilla para adjuntarla a tu rol. Te recomendamos usar el filtro **Customer managed** para encontrar tu política.
8. Haz clic en **Next: Review** para continuar.
9. Revisa la información sobre tu rol, indica un **Role name** y, opcionalmente, una **Role description**. También verás la política que adjuntaste. Si todo parece correcto, haz clic en **Create role** para continuar.
10. Una vez creado, podrás encontrar tus roles en la página principal de **Roles**.

<div id="copy-the-arn-values">
  ## Copia los valores de ARN
</div>

A continuación, se indica dónde encontrar los valores de Provider y Role ARN.

<div id="provider-arn">
  ### ARN del proveedor
</div>

En la consola de IAM, ve a [Proveedores de identidad](https://console.aws.amazon.com/iam/home#/providers). Selecciona el rol que te interese para abrir su página de resumen. Copia el valor de **ARN del proveedor**, que aparece primero en **Resumen**.

<div id="role-arn">
  ### Role ARN
</div>

En la consola de IAM, ve a [Roles](https://console.aws.amazon.com/iam/home#/roles). Selecciona el rol que te interese para abrir su página de resumen. Copia el valor de **Role ARN**, que aparece en primer lugar en **resumen**.
