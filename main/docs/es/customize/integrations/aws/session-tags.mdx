title: Usar Amazon Web Services Session Tags para el control de acceso basado en roles
---

Con Amazon Web Services (AWS) Session Tags, puede etiquetar recursos y asignar a los usuarios pares de clave/valor, lo que le permite implementar el control de acceso basado en roles (RBAC) para las API y los recursos de AWS.

En el ejemplo incluido en esta guía, etiquetaremos nuestros recursos de AWS con AWS Session Tags y, a continuación, crearemos una política para un rol de AWS IAM que permitirá a los usuarios con este rol y las etiquetas adecuadas realizar acciones específicas en nuestros recursos de AWS. Luego, crearemos una rule en Auth0 que asignará nuestro rol de AWS IAM y las AWS Session Tags correspondientes a un usuario de Auth0, y las transmitirá mediante aserciones de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> en el token. Este ejemplo se basa en el proporcionado en [Configurar Amazon Web Services para inicio de sesión único](/docs/es/customize/integrations/aws/configure-amazon-web-services-for-sso).

Para usar AWS Session Tags con las API y los recursos de AWS, debe:

1. Etiquetar las instancias de AWS.
2. Crear un rol especializado de AWS IAM.
3. Crear una rule de Auth0.
4. Probar la configuración.

<div id="prerequisites">
  ## Requisitos previos
</div>

* Necesitarás una [cuenta de Amazon Web Services (AWS)](https://portal.aws.amazon.com/billing/signup#/start) en la que tengas permisos de administrador.
* [Configura AWS para SSO](/docs/es/customize/integrations/aws/configure-amazon-web-services-for-sso)
* [Configura algunas instancias de VM de AWS](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/EC2_GetStarted.html). Para el ejemplo de esta guía, usamos tres instancias independientes.

<div id="tag-aws-instances">
  ### Etiqueta las instancias de AWS
</div>

Agrega etiquetas a tus recursos de AWS. Sigue las instrucciones de [Amazon Elastic Compute Cloud: Tag your Amazon EC2 resources](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/Using_Tags.html).

Ya deberías haber creado tres instancias. Agrega las siguientes etiquetas:

| Instancia | Etiquetas                                                                                        |
| --------- | ------------------------------------------------------------------------------------------------ |
| 1         | Clave: `CostCenter`, Valor: `marketing`.<br />Clave: `Project`, Valor: `website`.                |
| 2         | Clave: `CostCenter`, Valor: `engineering`.<br />Clave: `Project`, Valor: `management_dashboard`. |
| 3         | Clave: `CostCenter`, Valor: `marketing`.<br />Clave: `Project`, Valor: `community_site`.         |

<div id="create-a-specialized-aws-iam-role">
  ### Crear un rol especializado de AWS IAM
</div>

Crea un rol de IAM con el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> SAML de AWS que configuraste durante los requisitos previos. Sigue las instrucciones de [AWS Identity and Access Management User Guide: Creating a Role for SAML 2.0 Federation (Console)](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create_for-idp_saml.html).

Mientras configuras el rol, asegúrate de usar los siguientes parámetros:

| Parámetro     | Descripción y valor de ejemplo                                                                                                                                                                           |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SAML Provider | Nombre del proveedor de identidad que creaste en los requisitos previos, por ejemplo, `auth0SamlProvider`. Selecciona **Permitir acceso mediante programación y a la consola de administración de AWS**. |

Cuando se te pida **Adjuntar políticas de permisos**, crea una política con el siguiente JSON y asígnale el nombre `VirtualMachineAccessByCostCenter`.

```json lines
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "ec2:DescribeInstances"
            ],
            "Resource": "*"
        },
        {
            "Effect": "Allow",
            "Action": [
                "ec2:StartInstances",
                "ec2:StopInstances"
            ],
            "Resource": "*",
            "Condition": {
                "StringEquals": {
                    "ec2:ResourceTag/CostCenter": "<%= "${aws:PrincipalTag/CostCenter}" %>"
                }
            }
        }
    ]
}
```

Una vez creada la política, actualiza la lista de políticas del rol; después, filtra y selecciona la nueva política.

Al revisar tu configuración, asegúrate de usar los siguientes parámetros:

| Parámetro           | Descripción                                              |
| ------------------- | -------------------------------------------------------- |
| Nombre del rol      | Nombre descriptivo de tu rol, como `AccessByCostCenter`. |
| Descripción del rol | Descripción de la finalidad para la que se usa tu rol.   |

<div id="create-an-auth0-rule">
  ### Crear una Rule de Auth0
</div>

Para asignar el rol y las etiquetas de AWS a un usuario, tendrás que [crear una rule](/docs/es/customize/rules/create-rules) en Auth0. Luego, estos valores se incluirán en las aserciones SAML del token. Por ejemplo:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Sustituye el valor de la variable `awsAccount` por tu propio número de cuenta.
</Callout>

```javascript lines
function(user, context, callback) {
  var awsAccount = '013823792818';
  var rolePrefix = `arn:aws:iam::` + awsAccount; 
  var samlIdP = rolePrefix + `:saml-provider/auth0SamlProvider`;

  user.awsRole = rolePrefix + `:role/AccessByCostCenter,` + samlIdP;
  user.awsRoleSession = user.email;
  user.awsTagKeys = ['CostCenter', 'Project'];
  user.CostCenter = 'marketing';
  user.Project = 'website';

  context.samlConfiguration.mappings = {
    'https://aws.amazon.com/SAML/Attributes/Role': 'awsRole',
    'https://aws.amazon.com/SAML/Attributes/RoleSessionName': 'awsRoleSession',
    'https://aws.amazon.com/SAML/Attributes/PrincipalTag:CostCenter': 'CostCenter',
    'https://aws.amazon.com/SAML/Attributes/PrincipalTag:Project': 'Project'
  };

  callback(null, user, context);
}
```

<div id="test-configuration">
  ### Probar la configuración
</div>

Ahora debería poder iniciar sesión en la Consola de AWS con un usuario de Auth0 y probar su implementación.

Para iniciar sesión, necesitará el inicio de sesión de <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip> para la Consola de AWS. Para encontrarlo:

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione el nombre de la aplicación para verla.
2. En la pestaña **Addons**, habilite el add-on **SAML2 Web App**.
3. En la pestaña **Usage**, busque **Identity Provider Login URL**.
4. Vaya a la URL indicada.
5. Una vez que haya iniciado sesión, en **EC2**, seleccione **Instances**. Haga clic en una de las instancias etiquetadas con un `CostCenter` de `marketing` y luego en **Actions** &gt; **Instance State** &gt; **Stop**. Observe que la acción se completa correctamente.
6. Haga clic en la instancia etiquetada con un `CostCenter` de `engineering` y luego en **Actions** &gt; **Instance State** &gt; **Stop**. Observe que la acción falla y muestra un error.
