title: Integrar con Amazon Cognito
---

Amazon Cognito es una plataforma de backend como servicio que te permite centrarte en desarrollar la experiencia de usuario de tu aplicación. Para obtener más información y registrarte en los servicios, consulta [Amazon Cognito](http://aws.amazon.com/cognito/) y el proceso para crear un [proveedor de OpenID Connect (OIDC) mediante AWS CLI](https://docs.aws.amazon.com/cli/latest/reference/iam/create-open-id-connect-provider.html).

<div id="configure-amazon-web-services">
  ## Configurar Amazon Web Services
</div>

<div id="get-auth0-application-details">
  ### Obtén los detalles de la aplicación de Auth0
</div>

Primero, tendrás que obtener cierta información sobre tu aplicación en Auth0:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Selecciona tu aplicación y, a continuación, cambia a la vista **Settings**.
3. Anota el **Domain** y el **ID de cliente**.

<div id="create-new-openid-connect-oidc-provider">
  ### Crear un nuevo proveedor de OpenID Connect (OIDC)
</div>

En AWS, crea un nuevo <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> (IdP):

1. Abre la [IAM Console](https://console.aws.amazon.com), selecciona **Identity Providers** en la barra lateral izquierda y, a continuación, selecciona **Add Provider**.
2. Selecciona **OpenID Connect** como **Provider Type**.
3. En **Provider URL**:

   1. Introduce tu **Domain** en el campo **Provider URL**.
   2. Introduce tu **ID de cliente** en el campo **Audience**.
4. Selecciona **Add Provider**.
5. Ve al proveedor que acabas de crear y anota el **Provider ARN.**

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No es necesario configurar un IAM role después de crear el IdP. Si todavía no tienes uno, Cognito creará un IAM role predeterminado cuando crees un Identity Pool.
</Callout>

<div id="create-identity-pool">
  ### Crear Identity Pool
</div>

Crea un Identity Pool en AWS para permitir que Cognito use el proveedor de identidad de OIDC de Auth0 para la autenticación:

1. Inicia sesión en la [Consola de Cognito.](https://console.aws.amazon.com/cognito/home)
2. Selecciona **Identidades federadas**.
3. En **Nombre del Identity Pool**, especifica un nombre para el pool (por ejemplo, `Auth0`).
4. En **Proveedores de autenticación**, selecciona la pestaña **OpenID** y, a continuación, selecciona el nombre del proveedor que creaste anteriormente.
5. Selecciona **Crear pool** y, a continuación, selecciona **Permitir** para terminar de crear el nuevo Identity Pool.
6. En el Dashboard, selecciona **Editar Identity Pool** para ver el **ID del Identity Pool**.
7. Anota el **ARN** del rol de IAM que se creó automáticamente. Este valor se usará al enviar credenciales a Cognito.

<div id="configure-auth0">
  ## Configurar Auth0
</div>

Cognito usa la clave pública de firma de los [metadatos del proveedor de OpenID](https://subscription.auth0.com/.well-known/jwks.json) para validar la firma del <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT).

Confirma que tu aplicación de Auth0 esté configurada para usar el algoritmo de firma RS256:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), selecciona tu aplicación y luego cambia a la vista **Settings**.
2. Expande la sección **Advanced Settings** y luego cambia a la vista **OAuth**.
3. Busca el campo **JSON Web Token (JWT) Signature Algorithm** y selecciona **RS256** en el menú desplegable.
4. Selecciona **Save Changes**.

<div id="implementation">
  ## Implementación
</div>

Puedes usar el [Auth0 Lock SDK](/docs/es/libraries#lock) para integrar Auth0 en tu aplicación.

Una vez que el usuario inicie sesión con Auth0, el siguiente paso es enviar sus credenciales a Cognito. Para obtener más información, consulta [Open ID Connect providers (identity pools) on AWS Docs](http://docs.aws.amazon.com/cognito/latest/developerguide/open-id.html).

Cognito toma el <Tooltip tip="Token de identidad: credencial destinada al propio cliente, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> que recibe un usuario de Auth0 y lo usa para generar IDs únicos de Cognito. Cuando el usuario inicia sesión en Cognito mediante Auth0, puedes almacenar en Cognito información a la que solo él pueda acceder.

<div id="troubleshooting">
  ## Solución de problemas
</div>

<div id="invalid-login-token-error">
  ### Error de &quot;token de inicio de sesión no válido&quot;
</div>

Si recibes un error de &quot;token de inicio de sesión no válido&quot;, confirma que la aplicación de Auth0 esté configurada para usar el algoritmo de firma RS256:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), selecciona tu aplicación y luego cambia a la vista **Settings**.
2. Expande la sección **Advanced Settings** y luego cambia a la vista **OAuth**.
3. Busca el campo **JSON Web Token (JWT) Signature Algorithm** y selecciona **RS256** en el menú desplegable.
