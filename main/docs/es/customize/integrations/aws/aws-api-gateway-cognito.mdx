---
description: Cómo proteger el tutorial de API Gateway con Cognito en lugar de usar roles y políticas de IAM.
title: Proteger AWS API Gateway con Cognito
---

En lugar de usar roles y políticas de IAM para proteger la API, puede hacerlo mediante grupos de usuarios de Amazon Cognito.

Por favor, [cree los grupos de usuarios de Amazon Cognito correspondientes](http://docs.aws.amazon.com/cognito/latest/developerguide/setting-up-cognito-user-identity-pools.html) antes de comenzar este tutorial.

<div id="integrate-the-cognito-user-pool-with-the-api-gateway-api">
  ## Integra el grupo de usuarios de Cognito con la API de API Gateway
</div>

Ve a la [Amazon API Gateway Console](https://console.aws.amazon.com/apigateway). En la barra de navegación de la izquierda, selecciona la API de **SecurePets**.

Luego, selecciona **Authorizers** en la API de **SecurePets**.

En la columna **Authorizers**, cerca del centro de la pantalla, elige **Create** e indica que vas a crear un **autorizador de grupo de usuarios de Cognito**.

Para configurar tu autorizador:

1. Elige la **región de Cognito** en la que creaste tu grupo de usuarios.
2. Personaliza el campo **Authorizer name**, si lo deseas (se completará automáticamente con el nombre del grupo de usuarios que elijas, así que puedes dejarlo como está).
3. Personaliza el campo **Identity token source**. De forma predeterminada, este campo está configurado como `method.request.header.Authorization`, lo que establece como `Authorization` el nombre del encabezado de la solicitud entrante que contiene el token de identidad de quien llama a la API.
4. Si lo deseas, agrega una expresión regular al campo **App client ID regex** para validar los ID de cliente asociados al grupo de usuarios.

Cuando termines de configurar tu autorizador, haz clic en **Create** para integrar el grupo de usuarios con tu API.

<div id="enable-the-user-pool-authorizer-on-methods">
  ## Habilitar el autorizador del grupo de usuarios en los métodos
</div>

Para cada método en el que quieras que el grupo de usuarios actúe como autorizador, debes habilitarlo para ese método en particular.

Para habilitar el autorizador del grupo de usuarios en el método `GET`:

1. Después de seleccionar la API de SecurePets, selecciona el método `GET` que figura en `/pets`.
2. Haz clic en **Method Request**.
3. En Authorization Settings, haz clic en el **icono de edición** junto al campo Authorization.
4. Elige de la lista el autorizador de grupo de usuarios de Cognito correspondiente. Haz clic en el **icono de marca de verificación** para guardar la selección.

Repite este proceso con cualquier método adicional en el que quieras que el grupo de usuarios de Cognito actúe como autorizador (`GET` y `PURCHASE` para `/pets`, así como `POST` para `/purchase`).
