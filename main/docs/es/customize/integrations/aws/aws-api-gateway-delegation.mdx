<Warning>
  Esta funcionalidad usa delegation. De forma predeterminada, delegation está deshabilitada para los tenants que no tenían ningún add-on en uso al 8 de junio de 2017. Los tenants Legacy que actualmente usan un add-on que requiere delegation pueden seguir usando esta funcionalidad. Si la funcionalidad de delegation cambia o se elimina del servicio en algún momento, se notificará previamente a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar. Además, ten en cuenta que delegation no admite el uso de custom domains, por lo que cualquier funcionalidad que dependa de ella puede no funcionar completamente junto con un custom domain.
</Warning>

Con AWS, puedes crear potentes API y aplicaciones sin servidor altamente escalables mediante AWS Lambda, Amazon API Gateway y una aplicación JavaScript.

Una aplicación sin servidor ejecuta código personalizado como un servicio de cómputo sin necesidad de mantener un entorno operativo para alojar tu servicio. En su lugar, un servicio como [AWS Lambda](https://aws.amazon.com/lambda/) ejecuta tu código por ti.

Amazon API Gateway amplía las capacidades de AWS Lambda al agregar una capa de servicio delante de tus funciones Lambda para reforzar la seguridad, administrar las transformaciones de los mensajes de entrada y salida, y ofrecer funcionalidades como limitación y auditoría. Un enfoque sin servidor simplifica tus necesidades operativas, ya que aspectos como el escalado y la tolerancia a fallos pasan a ser responsabilidad del servicio de cómputo que ejecuta tu código.

Sin embargo, a menudo querrás vincular tus API a tus usuarios existentes, ya sea desde social providers como Twitter y Facebook, o dentro de tu propia organización desde Active Directory o una base de datos de clientes. Este tutorial muestra cómo autorizar el acceso a los métodos de tu Amazon API Gateway para tus usuarios existentes mediante Auth0 delegation para AWS y la integración con AWS Identity and Access Management (IAM).

A continuación, el tutorial te guiará por la configuración de Amazon API Gateway con funciones de AWS Lambda, la protección de esas funciones con roles de IAM de AWS y, después, el uso de Auth0 delegation para obtener un token para el AWS IAM role. Luego te mostrará cómo asignar diferentes permisos a distintas clases de usuarios, como usuarios internos de base de datos o usuarios sociales, y cómo transmitir la identidad usando un <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, también de token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT).
