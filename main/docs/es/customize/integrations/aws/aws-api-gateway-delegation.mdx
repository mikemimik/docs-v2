<Warning>
  Esta característica usa delegation. De forma predeterminada, delegation está deshabilitada para los tenants que no tuvieran un add-on en uso al 8 de junio de 2017. Los tenants Legacy que actualmente usan un add-on que requiere delegation pueden seguir usando esta característica. Si la funcionalidad de delegation cambia o se elimina del servicio en algún momento, se notificará con antelación a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar. Además, ten en cuenta que delegation no admite el uso de custom domains, por lo que cualquier característica que dependa de ella puede no funcionar por completo junto con un custom domain.
</Warning>

Con AWS, puedes crear API y aplicaciones potentes, sin servidor y altamente escalables mediante AWS Lambda, Amazon API Gateway y una aplicación JavaScript.

Una aplicación sin servidor ejecuta código personalizado como servicio de cómputo, sin necesidad de mantener un entorno operativo para alojar tu servicio. En su lugar, un servicio como [AWS Lambda](https://aws.amazon.com/lambda/) ejecuta tu código por ti.

Amazon API Gateway amplía las capacidades de AWS Lambda al añadir una capa de servicio delante de tus funciones Lambda para reforzar la seguridad, gestionar las transformaciones de los mensajes de entrada y salida, y proporcionar capacidades como limitación y auditoría. Un enfoque sin servidor simplifica tus necesidades operativas, ya que aspectos como el escalado y la tolerancia a fallos pasan a ser responsabilidad del servicio de cómputo que ejecuta tu código.

Sin embargo, a menudo querrás vincular tus API a tus usuarios actuales, ya sea desde proveedores sociales como Twitter y Facebook, o dentro de tu propia organización desde Active Directory o una base de datos de clientes. Este tutorial muestra cómo autorizar el acceso a tus métodos de Amazon API Gateway para tus usuarios actuales mediante Auth0 delegation para AWS y la integración con AWS Identity and Access Management (IAM).

A continuación, el tutorial te guía por la configuración de Amazon API Gateway mediante funciones de AWS Lambda, la protección de esas funciones con roles de AWS IAM y, después, el uso de Auth0 delegation para obtener un token para el rol de AWS IAM. Luego te mostrará cómo asignar distintos permisos a varios tipos de usuarios, como usuarios internos de base de datos o usuarios sociales, y cómo transferir la identidad usando un <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, también de Access Token) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT).
