<Warning>
  Esta funcionalidad usa delegación. De forma predeterminada, la delegación está deshabilitada para los tenants que no usaban un add-on al 8 de junio de 2017. Los tenants Legacy que actualmente usan un add-on que requiere delegación pueden seguir usando esta funcionalidad. Si la funcionalidad de delegación cambia o se elimina del servicio en algún momento, se notificará previamente a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar. Además, ten en cuenta que la delegación no admite el uso de dominios personalizados, por lo que cualquier funcionalidad que dependa de ella puede no funcionar por completo junto con un dominio personalizado.
</Warning>

Con AWS, puedes crear API y aplicaciones potentes, sin servidor y altamente escalables mediante AWS Lambda, Amazon API Gateway y una aplicación JavaScript.

Una aplicación sin servidor ejecuta código personalizado como servicio de cómputo sin necesidad de mantener un entorno operativo para alojar tu servicio. En su lugar, un servicio como [AWS Lambda](https://aws.amazon.com/lambda/) ejecuta tu código por ti.

Amazon API Gateway amplía las capacidades de AWS Lambda al agregar una capa de servicio delante de tus funciones Lambda para reforzar la seguridad, gestionar las transformaciones de los mensajes de entrada y salida, y ofrecer capacidades como limitación y auditoría. Un enfoque sin servidor simplifica tus necesidades operativas, ya que aspectos como el escalado y la tolerancia a fallos pasan a ser responsabilidad del servicio de cómputo que ejecuta tu código.

Sin embargo, muchas veces querrás vincular tus API con tus usuarios existentes, ya sea desde proveedores sociales como Twitter y Facebook, o dentro de tu propia organización desde Active Directory o una base de datos de clientes. Este tutorial muestra cómo autorizar el acceso a los métodos de tu Amazon API Gateway para tus usuarios existentes usando la delegación de Auth0 para AWS y la integración con AWS Identity and Access Management (IAM).

A continuación, el tutorial te guía por la configuración de Amazon API Gateway con funciones de AWS Lambda, la protección de esas funciones con roles de AWS IAM y, después, el uso de la delegación de Auth0 para obtener un token para el rol de AWS IAM. Luego te mostrará cómo asignar diferentes permisos a varias clases de usuarios, como usuarios internos de base de datos o usuarios sociales, y cómo transferir la identidad usando un <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, también de access token) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT).
