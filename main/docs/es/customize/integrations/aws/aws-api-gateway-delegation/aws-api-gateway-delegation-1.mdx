  Esta funcionalidad usa Delegation. De forma predeterminada, Delegation está deshabilitado para los tenants que no tenían ningún add-on en uso a fecha de 8 de junio de 2017. Los tenants Legacy que actualmente usan un add-on que requiere Delegation pueden seguir usando esta funcionalidad. Si la funcionalidad de Delegation cambia o se retira del servicio en algún momento, se notificará con antelación a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar. Además, ten en cuenta que Delegation no admite el uso de dominios personalizados, por lo que cualquier funcionalidad que dependa de ella puede no ser totalmente funcional junto con un dominio personalizado.
</Warning>

<div id="step-1-set-up-the-amazon-api-gateway">
  ## Paso 1 - Configura Amazon API Gateway
</div>

Después de completar este paso, habrás:

* Configurado Amazon API Gateway con funciones de AWS Lambda para ejecutar la lógica de tu servicio, que almacena y recupera mascotas de una tabla de Amazon DynamoDB;
* Creado dos métodos de servicio REST no autenticados para obtener y actualizar una lista de mascotas.

Antes de comenzar, asegúrate de tener instalado [Node.js](https://nodejs.org/).

<div id="step-1-create-the-amazon-dynamodb-table">
  ### Paso 1. Crear la tabla de Amazon DynamoDB
</div>

En la [consola de Amazon DynamoDB](https://console.aws.amazon.com/dynamodb), haz clic en **Create Table**.

<Frame>![Consola de DynamoDB - Create Table](/docs/images/cdy7uua7fh8z/2ZPhi7Ddx4TznSXEEhUSDK/52a06412f7c5e6d0ef0d13c6ffe17611/dynamodb-create-table.png)</Frame>

Configura las variables asociadas a la tabla:

* **Nombre de la tabla**: Pets
* **Clave principal**: username
* **Tipo de clave principal**: String
* **Usar la configuración predeterminada**: desmarcado
* **Unidades de capacidad de lectura**: 3
* **Unidades de capacidad de escritura**: 3

<Frame>![Consola de DynamoDB - Configurar la tabla recién creada](/docs/images/cdy7uua7fh8z/6MYJV97Xa3DxajG8DYj2dx/e428ab2ce137176d7771c61a9c528f7c/configure-newly-created-table.png)</Frame>

Haz clic en **Create** para crear la tabla con la configuración que proporcionaste.

Mientras se crea la tabla, toma nota del nombre de recurso de Amazon (ARN) en la sección de detalles de la tabla. Necesitarás el ARN de la tabla en el siguiente paso.

<Frame>![Consola de DynamoDB - Detalles de la tabla - nombre de recurso de Amazon](/docs/images/cdy7uua7fh8z/4ZFemddtUD8tRi0NUvzXJ2/38abd951974cad385ef7417f48adf359/table-arn.png)</Frame>

<div id="step-2-create-the-policy-that-grants-aws-lambda-functions-access-to-the-dynamodb-pets-table">
  ### Step 2. Cree la política que otorga a las funciones de AWS Lambda acceso a la tabla Pets de DynamoDB
</div>

Vaya a la [consola de AWS IAM](https://console.aws.amazon.com/iam).

Haga clic en **Roles** en el menú de la izquierda y, a continuación, en el botón **Create New Role**.

<Frame>![Consola de IAM - Roles - Crear nuevos roles](/docs/images/cdy7uua7fh8z/4mgCTXLpycHLA8RbYoEFim/5d9eb81be979355aa9cd10d476ead515/roles.png)</Frame>

Asigne al rol el nombre `APIGatewayLambdaExecRole` y haga clic en **Siguiente paso**.

Seleccione el tipo de rol. En AWS Service Roles, seleccione AWS Lambda.

<Frame>![Consola de IAM - Seleccionar el tipo de rol - Roles de servicio - AWS Lambda](/docs/images/cdy7uua7fh8z/326kKGbahlvIwVFTi61Gfw/026d1fc0c8c6e3a489ec4fb840f551da/select-role-type.png)</Frame>

En la pantalla Attach Policy, omita este paso haciendo clic en **Siguiente paso**. En este punto, revise la información que proporcionó. Si todo parece correcto, haga clic en **Create Role**. Cuando termine, debería ver el rol en la lista de la página principal de IAM.

Seleccione el rol que acaba de crear, **APIGatewayLambdaExecRole**. Haga clic en la flecha desplegable de Inline Policies y luego en el enlace **Click Here**.

Seleccione Custom Policy y, a continuación, haga clic en **Select**. Asigne a la política el nombre `LogAndDynamoDBAccess` y agregue el siguiente código como documento de la política (asegúrese primero de actualizar el nombre de recurso de Amazon (ARN) de su tabla de DynamoDB). Haga clic en **Apply Policy**.

```json lines
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AccessCloudwatchLogs",
      "Action": ["logs:*"],
      "Effect": "Allow",
      "Resource": "arn:aws:logs:*:*:*"
    },
    {
      "Sid": "PetsDynamoDBReadWrite",
                  "Effect": "Allow",
      "Action": [
                  "dynamodb:DeleteItem",
                  "dynamodb:GetItem",
                  "dynamodb:PutItem",
                  "dynamodb:UpdateItem"
                  ],
      "Resource": ["DYNAMODB_TABLE_ARN_VALUE_FROM_PREVIOUS_STEP"]
    }
   ]
}
```

<div id="step-2-create-the-aws-lambda-functions">
  ### Paso 2. Crear las funciones de AWS Lambda
</div>

Los tres pasos siguientes crean las funciones de AWS Lambda para consultar y actualizar la información de las mascotas en la tabla de DynamoDB.

<div id="create-the-lambda-function-for-getpetinfo">
  #### Crea la función Lambda para GetPetInfo
</div>

En la [consola de AWS Lambda](https://console.aws.amazon.com/lambda), selecciona **Create a Lambda Function** (si aún no has creado una función Lambda de AWS, haz clic en **Get Started Now**).

En la pantalla Select blueprint, haz clic en **Blank Function**.

A continuación, se te pedirá que configures los triggers. Haz clic en **Next** para continuar. No es necesario hacerlo en este momento.

Por último, se te pedirá que configures la función.

Completa los campos correspondientes con la siguiente información:

* **Name**: `GetPetInfo`
* **Runtime**: Node.js 6.10

Pega el siguiente código en el área **Lambda function code** para leer la información de la mascota desde la tabla de DynamoDB.

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();

exports.handler = function(event, context) {
   var cb = function(err, data) {
      if(err) {
         console.log('error on GetPetsInfo: ',err);
         context.done('Unable to retrieve pet information', null);
      } else {
         if(data.Item && data.Item.pets) {
             context.done(null, data.Item.pets);
         } else {
              context.done(null, {});
         }
      }
   };

   dynamo.getItem({TableName:"Pets", Key:{username:"default"}}, cb);
};
```

En **Role**, selecciona **Choose an existing role**. A continuación, elige **APIGatewayLambdaExecRole** como **Existing Role**. Deja todas las demás configuraciones con sus valores predeterminados.

Haz clic en **Next** para revisar la información que ingresaste. Si todo se ve correcto, haz clic en **Create function**.

Haz clic en **Test** y deja el evento de prueba de entrada con su valor predeterminado (que usa la plantilla Hello World). Cuando se complete la prueba, deberías ver una salida vacía (`{}`) en la sección **Execution Result**. La tabla está vacía.

<div id="create-the-lambda-function-for-updatepetinfo">
  #### Crea la función Lambda para UpdatePetInfo
</div>

Repite las instrucciones utilizadas para crear la función `GetPetInfo`, pero usa lo siguiente como código de la función:

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();
exports.handler = function(event, context) {
    var item = { username:"default",
                 pets: event.pets || {}
            };

    var cb = function(err, data) {
        if(err) {
            console.log(err);
            context.fail('unable to update pets at this time');
        } else {
            console.log(data);
                context.done(null, data);
        }
    };
    dynamo.putItem({TableName:"Pets", Item:item}, cb);
};
```

Pruebe la función haciendo clic en el menú desplegable Actions y seleccionando **Configurar evento de muestra**. Ingrese los siguientes datos de muestra y haga clic en **Enviar**:

```json lines
{
  "pets": [{
    "id": 1,
    "type": "dog",
    "price": 249.99
  }]
}
```

Deberías ver un resultado vacío (`{}`).

Vuelve a tu función Lambda `GetPetInfo` y haz clic en **Test** de nuevo. Ahora deberías ver una sola mascota.

<div id="create-the-third-lambda-function">
  #### Crear la tercera función Lambda
</div>

Creará una función Lambda adicional. Aunque esta función no hará nada, es necesaria para el método OPTIONS de CORS, como se describe en una sección posterior.

Siguiendo los pasos descritos anteriormente, cree una función Lambda llamada `NoOp`. El código de la función será el siguiente:

```js lines
exports.handler = function(event, context) {
    context.succeed('');
}
```

En lugar de crear esta tercera función Lambda, puedes optar por [crear un método OPTIONS](#method-options) en API Gateway.

<div id="step-3-create-the-amazon-api-gateway-api">
  ### Paso 3. Crear la API de Amazon API Gateway
</div>

Creará una API con dos métodos: uno para `GET` de la información de la mascota y otro para `POST` de la información de la mascota.

<div id="method-get-pet-information">
  #### Método: GET Pet Information
</div>

Ve a la [Amazon API Gateway Console](https://console.aws.amazon.com/apigateway) y haz clic en **Create API**. Si es la primera vez que creas una API, en su lugar verás una pantalla con la opción **Get Started**.

Si es la primera vez que creas una API, se te pedirá crear una API de ejemplo. Haz clic en **OK** para cerrar la notificación emergente y selecciona el botón de opción **New API** en lugar de **Example API**.

Asigna a la API el nombre `SecurePets` y haz clic en **Create API**.

Ve a la pestaña Resources de la API `SecurePets` y haz clic en la acción **Create Resource**.

Asigna al recurso el nombre `Pets` y haz clic de nuevo en **Create Resource**.

En el panel izquierdo, selecciona `/pets` y luego haz clic en el botón **CreateMethod**.

En el menú desplegable, selecciona GET y haz clic en el botón con la marca de verificación. Proporciona los siguientes valores de configuración para el método `GET`:

* **Integration type**: Lambda Function;
* **Lambda Region**: la Region en la que te encuentras;
* **Lambda Function**: GetPetInfo.

Haz clic en **Save** y luego en **OK** cuando se te pida, en la ventana emergente, conceder permisos a la función Lambda.

En la ventana Method Execution que aparece a continuación, haz clic en **Test**.

Deberías ver que se devuelve una única mascota en el cuerpo de la respuesta.

<div id="method-post-pet-information">
  #### Método: POST de información de mascotas
</div>

La creación de la API que se usa para `POST` de información de mascotas es similar a la de la API que se usa para `GET` de información de mascotas.

En el panel izquierdo, selecciona `/pets` y haz clic en **CreateMethod**.

En el menú desplegable, selecciona POST y haz clic en el botón con la marca de verificación.

Selecciona Lambda Function en Integration Type, selecciona la región en la que te encuentras y selecciona UpdatePetInfo como función Lambda.

Haz clic en **Save** y luego en **OK** cuando aparezca la ventana emergente para conceder permisos a la función Lambda.

Haz clic en **Test** y pega lo siguiente en el cuerpo de la solicitud:

```json lines
{"pets": [
    {"id": 1, "type": "dog", "price": 249.99},
    {"id": 2, "type": "cat", "price": 124.99}
  ]
}
```

Deberías ver un resultado vacío (`{}`).

Vuelve al método GET y haz clic en **Test** de nuevo para ver que el cuerpo de la respuesta indica que hay dos mascotas en la tabla:

```json lines
[
  {
    "id": 1,
    "price": 249.99,
    "type": "dog"
  },
  {
    "id": 2,
    "price": 124.99,
    "type": "cat"
  }
]
```

<div id="method-options">
  #### Método: OPTIONS
</div>

En lugar de crear una función Lambda que no haga nada, puedes crear un método `OPTIONS` en API Gateway.

En el panel izquierdo, selecciona `/pets` y haz clic en **CreateMethod**. En la lista desplegable, selecciona OPTIONS y haz clic en el botón con la marca de verificación. En Integration Type, selecciona Mock. Haz clic en **Save**.

Deja Response Body en blanco y haz clic en **Test**. Deberías recibir un Response Body que indique `no data`.

En este punto, las funciones Lambda de AWS y los métodos de Amazon API Gateway están definidos sin seguridad.
