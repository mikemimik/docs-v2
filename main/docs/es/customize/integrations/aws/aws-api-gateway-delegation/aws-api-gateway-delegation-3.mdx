<Warning>
  Esta funcionalidad usa delegación. Por defecto, la delegación está deshabilitada para los tenants que no tenían ningún add-on en uso a fecha del 8 de junio de 2017. Los tenants Legacy que actualmente usan un add-on que requiere delegación pueden seguir usando esta funcionalidad. Si la funcionalidad de delegación cambia o se elimina del servicio en algún momento, se avisará con antelación a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar. Además, tenga en cuenta que la delegación no admite el uso de dominios personalizados, por lo que cualquier funcionalidad que dependa de ella puede no funcionar por completo junto con un dominio personalizado.
</Warning>

<div id="step-3-build-the-application">
  ## Paso 3 - Crear la aplicación
</div>

En este paso, crearás una aplicación serverless de una sola página con el framework AngularJS, que se servirá desde un bucket de AWS S3 configurado para funcionar como un sitio web estático.

<div id="step-1-set-up-your-sample-application">
  ### Paso 1. Configura tu aplicación de ejemplo
</div>

Para empezar con una aplicación sencilla, [descarga un proyecto de ejemplo](https://github.com/auth0/auth0-aws/tree/master/examples/api-gateway/client) específico para este tutorial. Inicia sesión para que tus credenciales de Auth0 queden preconfiguradas.

Copia el contenido de este proyecto base en una carpeta local llamada `pets`, que usarás durante el resto de este tutorial. Dentro de esta carpeta, actualiza `auth0-variables.js` con el `AUTH0_CLIENT_ID` y el `AUTH0_CLIENT_ID` de tu aplicación de Auth0 (esta información está disponible en el [Dashboard de administración](https://manage.auth0.com/#/applications) de la aplicación en cuestión).

<div id="aws-s3-bucket">
  #### bucket de AWS S3
</div>

Asegúrate de haber [creado el bucket de AWS S3 configurado para que funcione como un sitio web estático](http://docs.aws.amazon.com/gettingstarted/latest/swh/website-hosting-intro.html). Durante el proceso de configuración, copia el contenido de la carpeta `pets` a tu bucket de S3 para proporcionar contenido para el sitio web.

Si estás usando un bucket existente, puedes mover los archivos con la AWS CLI mediante el siguiente comando.

```bash lines
aws s3 cp --recursive --acl "public-read" ./ s3://{yourBucket}/
```

Antes de continuar, asegúrate de tener al menos un usuario asociado a tu conexión Username-Password-Authentication (o a la conexión de base de datos asociada a la aplicación que estás usando). Para aprovechar al máximo la funcionalidad de tu aplicación de ejemplo y su integración con AWS, necesitarás ese usuario para probar la autenticación y obtener acceso.

Por último, asegúrate de que Auth0 permita la autenticación desde tu sitio web proporcionando la URL en el campo **Allowed Origins** de la página Settings de tu aplicación. La URL de tu sitio web debería verse más o menos así:

`http://your-bucket.s3-website-us-east-1.amazonaws.com`

Si no sabes cuál es tu URL, puedes encontrarla en la pestaña **Propiedades** de tu bucket de S3.

Antes de continuar, prueba a iniciar sesión en tu aplicación. Abre `http://your-bucket-domain/index.html` en tu navegador. Después de iniciar sesión, deberías ver un cuadro de alerta emergente con el mensaje &quot;getPets no implementado&quot;:

También deberías ver la página para ver las mascotas.

<div id="modify-the-role-and-principal-strings">
  #### Modifica las cadenas `role` y `principal`
</div>

Para modificar las cadenas `role` y `principal` (que corresponden a las dos últimas líneas de la sentencia `if` incluida en la función proporcionada), especifica los valores adecuados mediante [Rules](https://manage.auth0.com/#/rules):

```javascript lines
function (user, context, callback) {
  if (context.clientID === 'CLIENT_ID' &&
      context.protocol === 'delegation') {
    // establecer configuración de AWS
    context.addonConfiguration = context.addonConfiguration || {};
    context.addonConfiguration.aws = context.addonConfiguration.aws || {};
    context.addonConfiguration.aws.principal = 'arn:aws:iam::[omitted]:saml-provider/auth0-provider';
    context.addonConfiguration.aws.role = 'arn:aws:iam::[omitted]:role/auth0-role';
  }

  callback(null, user, context);
}
```

Asegúrese de actualizar los valores de ARN de `role` y `[principal]` con los correspondientes a su integración.

Copie los archivos actualizados en su bucket de S3 para el sitio web.

Opcionalmente, puede establecer un punto de interrupción en el navegador en `store.set('awstoken', delegation.Credentials);`. Cuando cierre sesión y vuelva a iniciarla, inspeccione `delegation.Credentials` al llegar al punto de interrupción. Verá valores conocidos, como AccessKeyId y SecretAccessKey:

```js lines
{
    AccessKeyId: "ASIAJB...BNQ",
    SecretAccessKey: "vS+b6...2Noav",
    SessionToken: "AQoDYBqsivOV...DdQW0gsKr8rgU=",
    Expiration: "2015-08-27T14:48:32.000Z"
}
```

Si no ves estos valores, asegúrate de que esté habilitado el addon de Amazon Web Services en la pestaña Addons de tu aplicación de Auth0.

<div id="configure-the-getpets-method">
  #### Configura el método `getPets`
</div>

Abre `home.js` en la carpeta `home` y actualiza el contenido de `getPets` con un método para obtener los datos de las mascotas (asegúrate de actualizar la región si no lo estás ejecutando en `us-east-1`):

```javascript lines
function getPets() {
    // esto no está autenticado
    var apigClient = apigClientFactory.newClient({
        region: 'us-east-1' // La región donde se desplegó la API
    });

    apigClient.petsGet({},{})
      .then(function(response) {
        console.log(response);
        $scope.pets = response.data;
        $scope.$apply();
      }).catch(function (response) {
        alert('pets get failed');
        showError(response);
    });
}
```

Copia el código actualizado en tu bucket de S3. Recarga la página para ver dos animales en la lista (si ejecutaste la prueba descrita anteriormente en tus API, que creó estas mascotas).

<div id="update-pets-with-the-aws-api-service">
  ### Actualizar Pets con AWS API Service
</div>

Ahora que tiene una aplicación de Auth0 en funcionamiento con API Gateway, agregará un método para actualizar la tabla `pets`.

Modifique la lógica del método `putPets` para actualizar las mascotas con su función de API. Esta función se usará tanto para agregar como para eliminar mascotas.

```javascript lines
function putPets(updatedPets) {
    var body = {pets: updatedPets};

    var apigClient = apigClientFactory.newClient({
        region: 'us-east-1' // establece esto en la región en la que estás trabajando.
    });

    apigClient.petsPost({},body)
      .then(function(response) {
        console.log(response);
       }).catch(function (response) {
        alert('pets update failed');
        showError(response);
      });
}
```

Copia el código actualizado en tu bucket de S3. Prueba el método:

1. Cierra sesión y vuelve a iniciarla.
2. Introduce valores para `Pet Type` y `Pet Price`.
3. Haz clic en **Guardar** para publicar tus datos.

Deberías ver un mensaje que diga: &quot;Tenemos un `<Pet Type>` en venta por `<Pet Price>`&quot;, con un botón rojo **ELIMINAR** a la izquierda.

Para añadir seguridad, agrega la función `getSecureApiClient` al inicio del método `putPets`:

```javascript lines
function putPets(updatedPets) {
     var apigClient = getSecureApiClient();
}
```

Copia el código en tu bucket de S3.

La función `getSecureApiClient` proporcionada recupera el AWS token de local storage obtenido mediante delegación a la API y usa la clave de acceso, la clave secreta y el token de sesión:

```javascript lines
function getSecureApiClient() {
    var awstoken = store.get('awstoken');

    return apigClientFactory.newClient({
        accessKey: awstoken.AccessKeyId,
        secretKey: awstoken.SecretAccessKey,
        sessionToken: awstoken.SessionToken,
        region: 'us-east-1' // La región desde la que estás trabajando.
    });
  }
```
