description: Detalles sobre la migración de conexiones de Office365 a Windows Azure AD.
title: Migrar conexiones de Office365 a Windows Azure AD
---

Office365 ha quedado obsoleto. Debe migrar sus conexiones de Office365 a conexiones de Windows Azure AD.

Desde el principio, admitimos la autenticación de usuarios con Office365. Office365 siempre ha usado Windows Azure AD entre bastidores, pero no había una buena interfaz para crear una &quot;aplicación&quot; en Windows Azure AD. Por eso, tenía que crearla en el Seller Dashboard. De ahora en adelante, Microsoft quiere que use Windows Azure AD, y ahora puede crear fácilmente un directorio asociado a su cuenta de Office365 y a la aplicación.

<div id="how-to-migrate-to-azure">
  ## Cómo migrar a Azure
</div>

1. Crea una suscripción de Windows Azure AD (gratuita)
2. Crea un directorio (que estará asociado a tu cuenta de Office365) y una aplicación. Para obtener más información, consulta [Conecta tu aplicación con Microsoft Azure Active Directory](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

Si usabas `user_id` en tu aplicación, ten en cuenta que cambiará de `office365|....some-guid....` a `waad|...email....`.
