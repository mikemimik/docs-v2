title: Introducción a la integración con Auth0
---

Auth0provides un servicio de autenticación y autorización basado en estándares abiertos como OIDC, OAuth2, <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> y otros. Pero, además, ofrecemos herramientas para que nuestros clientes puedan externalizar la centralización y gestión de usuarios, algunos componentes de la seguridad de las aplicaciones y la conexión de redes de aplicaciones a una única fuente de identidad.

Una parte importante de esto es permitir que partners, como usted, creen un ecosistema de integraciones que pueda utilizarse para mejorar y ampliar lo que ya ofrecemos. Lo hacemos proporcionando puntos de extensibilidad y mecanismos que permiten la personalización. Esto se traduce en más funcionalidad para los clientes y una menor carga de mantenimiento para ellos.

Casi el 90 % de nuestros clientes amplían la plataforma Auth0 para adaptarla a sus necesidades específicas. Al trabajar con usted para resolver problemas de identidad comunes y casos de uso frecuentes, podemos acelerar el tiempo que tardan nuestros clientes en implementar sus flujos de trabajo de identidad.

<div id="what-can-be-extended">
  ## ¿Qué se puede extender?
</div>

A continuación se muestra una lista de las distintas formas en que se puede personalizar Auth0. Algunos puntos de extensibilidad permiten instalar integraciones directamente, mientras que otros requieren una guía que el cliente puede seguir, y algunos puntos de extensibilidad requieren una cuenta de Auth0 de pago.

* **Single-Sign On en un servicio externo**

  * [SAML](/docs/es/customize/integrations/marketplace-partners/sso-integrations-for-partners)
  * OpenID Connect
  * WS-Fed
* [**Personalizar la UI de Auth0 de cara al usuario**](/docs/es/customize)

  * Cargar JavaScript y CSS personalizados en el formulario de inicio de sesión
  * Plantillas de correo electrónico personalizadas
* **Fuentes de identidad**

  * [OpenID Connect](/docs/es/authenticate/identity-providers/enterprise-identity-providers/oidc)
  * [OAuth2](/docs/es/customize/integrations/marketplace-partners/social-connections-for-partners)
  * [SAML](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml)
  * [Active Directory/LDAP](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap)
  * [Código personalizado que apunta a una fuente de identidad remota](/docs/es/authenticate/database-connections/custom-db)
* [**Código personalizado después de una autenticación exitosa**](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)

  * Llamar a una API externa para enriquecer el perfil del usuario, bloquear inicios de sesión, etc.
  * [Redirigir a un servicio externo](/docs/es/customize/integrations/marketplace-partners/redirect-actions-for-partners)
  * Enriquecer el perfil del usuario almacenado en Auth0
  * Proporcionar un segundo factor para la autenticación
  * [Autenticación escalonada](/docs/es/secure/multi-factor-authentication/step-up-authentication)
  * Activar MFA de Auth0
* **Código personalizado** [**antes**](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/pre-user-registration-trigger) **y** [**después de**](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/post-user-registration-trigger) **registrarse en la base de datos de Auth0**

  * Database Connections
  * Conexiones passwordless
* [**Enviar códigos MFA con un proveedor de SMS personalizado**](/docs/es/customize/actions/explore-triggers/mfa-notifications-trigger)
* **Integración con aplicaciones del cliente**

  * Integración mediante un SDK propio

* [**Proveedores de correo electrónico a través de SMTP**](/docs/es/customize/email/smtp-email-providers)
