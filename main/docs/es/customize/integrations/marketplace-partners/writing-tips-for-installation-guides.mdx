Si no puedes consultar una guía de estilo completa, revisa estos consejos para mejorar notablemente tu contenido.

<div id="use-active-voice">
  ## Usa la voz activa
</div>

Evita la voz pasiva (verbo `to be` + participio). Para que tu texto sea más dinámico, claro y, a menudo, más breve, procura usar sujetos y voz activa siempre que sea posible.

| Preferible                                                                                                                                                                                                                                                                                 | Desaconsejado                                                                                                                                                                                                                                                                               |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Una vez que agregues la configuración necesaria, Auth0 procesará todos los inicios de sesión de tu tenant con esta Rule. Antes de activar la integración en producción, asegúrate de haber configurado correctamente todos los componentes y de haberlo verificado en un tenant de prueba. | Una vez agregada la configuración necesaria, todos los inicios de sesión de tu tenant serán procesados por esta Rule. Asegúrate de que todos los componentes se hayan configurado correctamente y se hayan verificado en un tenant de prueba antes de activar la integración en producción. |

<div id="make-link-titles-scannable">
  ## Haz que los títulos de los enlaces sean fáciles de identificar de un vistazo
</div>

Cuando incluyas enlaces a otros recursos, asegúrate de que el enlace describa la acción o tarea del documento al que dirige. Un texto de enlace genérico dificulta la lectura rápida, ya que obliga a los lectores a leer todo el texto que rodea al enlace para identificar qué deben seleccionar.

| Recomendado                                                                                                                                                                              | Desaconsejado                                 |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| Puedes usar el Auth0 Dashboard o la Management API para [personalizar el texto de la experiencia de Universal Login](/docs/es/customize/login-pages/universal-login/customize-templates).   | Haz clic [aquí](/docs/es/get-started).           |
| Para saber cómo asignar roles a miembros de Organizations, lee [Agregar roles a miembros de la organización](/docs/es/manage-users/organizations/configure-organizations/add-member-roles). | Puedes ir a [este enlace](/docs/es/get-started). |

<div id="write-device-agnostic-directions">
  ## Escribe instrucciones independientes del dispositivo
</div>

Recuerda que las interfaces de usuario se muestran de manera distinta en diferentes dispositivos. Además, ten presente que los usuarios pueden estar usando dispositivos adaptativos para ver tu contenido.

| Preferido                           | Desaconsejado                                         |
| ----------------------------------- | ----------------------------------------------------- |
| Selecciona **Create**.              | Haz clic en **Create**.                               |
| Cambia a la vista **Applications**. | Cambia a la pestaña **Applications**.                 |
| Busca **Secreto de cliente**.       | Desplázate hacia arriba hasta **Secreto de cliente**. |

<div id="use-verbs-and-nounadjectives-properly">
  ## Usa correctamente los verbos y los sustantivos/adjetivos
</div>

Ten cuidado con las palabras que pueden usarse tanto como verbos como sustantivos o adjetivos. Las formas verbales suelen llevar un espacio entre las palabras, mientras que los sustantivos y adjetivos pueden escribirse como una sola palabra.

| Preferido                     | Desaconsejado                  |
| ----------------------------- | ------------------------------ |
| Log in to your account.       | Login to your account.         |
| Auth0 processes logins.       | Auth0 processes log ins.       |
| Set up your hardware.         | Setup your hardware.           |
| Describe your hardware setup. | Describe your hardware set up. |

<div id="use-sentence-case-for-headings">
  ## Usa mayúscula solo en la primera palabra de los encabezados
</div>

Escribe con mayúscula solo la primera palabra y cualquier nombre propio o nombre de producto de Auth0.

| Preferido                             | Desaconsejado                         |
| ------------------------------------- | ------------------------------------- |
| Agregar una dependencia               | Agregar una Dependencia               |
| Validar el inicio de sesión reanudado | Validar el Inicio de Sesión Reanudado |
| Crear una Action                      | Crear una action                      |

<div id="use-simple-tense-for-headings">
  ## Usa tiempos simples en los encabezados
</div>

Evita la verbosidad innecesaria usando tiempos simples en los encabezados.

| Preferido                                | Desaconsejado                                 |
| ---------------------------------------- | --------------------------------------------- |
| Asignar y cambiar usuarios               | Asignando y cambiando usuarios                |
| Conectar una base de datos personalizada | Cómo conectar una base de datos personalizada |

<div id="write-steps-concisely">
  ## Redacta los pasos de forma concisa
</div>

Al redactar el texto introductorio de los pasos, sé directo y conciso. Recuerda que los usuarios ya saben qué tienen que hacer con una lista numerada de pasos.

| Preferido             | Desaconsejado                                |
| --------------------- | -------------------------------------------- |
| Para instalar el SDK: | Sigue los pasos que aparecen a continuación: |

<div id="set-user-expectations-in-steps">
  ## Establece las expectativas de los usuarios en los pasos
</div>

Indica a los usuarios cuál será el resultado de su acción antes de describir lo que deben hacer.

| Preferido                                                   | Desaconsejado                                              |
| ----------------------------------------------------------- | ---------------------------------------------------------- |
| Para activar esta integración, selecciona **Save Changes**. | Selecciona **Save Changes** para activar esta integración. |

<div id="list-actions-within-steps-in-order">
  ## Enumera las acciones dentro de los pasos en orden
</div>

Dentro de cada paso, indica al usuario qué debe hacer y en qué orden.

| Recomendado                                                                | Desaconsejado                                                      |
| -------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| Localiza la sección **Integrations** y selecciona **Agregar integración**. | Selecciona **Agregar integración** en la sección **Integrations**. |

<div id="use-notes-and-warning-appropriately">
  ## Usa las notas y las advertencias correctamente
</div>

Las notas y las advertencias cumplen funciones distintas. Las notas deben incluir información general que conviene conocer, mientras que las advertencias contienen información que puede provocar fallos si no se sigue.

Si una información debe presentarse como advertencia, debes incluirla exactamente donde el usuario deba realizar la acción relacionada. Los usuarios no deberían tener que andar buscando las advertencias.
