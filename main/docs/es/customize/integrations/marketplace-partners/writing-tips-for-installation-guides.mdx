title: Consejos de redacción para las guías de instalación
---

Si no puedes consultar una guía de estilo completa, revisa estos consejos para mejorar notablemente tu contenido.

<div id="use-active-voice">
  ## Usa la voz activa
</div>

Evita la voz pasiva (verbo `to be` + participio pasado). Para que tu texto sea más dinámico, claro y, a menudo, más breve, intenta usar un sujeto claro y la voz activa siempre que sea posible.

| Preferido                                                                                                                                                                                                                                                                              | Desaconsejado                                                                                                                                                                                                                                                                                           |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Cuando agregues la configuración requerida, Auth0 procesará todos los inicios de sesión de tu tenant con esta Rule. Antes de activar la integración en producción, asegúrate de haber configurado correctamente todos los componentes y de haberlos verificado en un tenant de prueba. | Una vez que se haya agregado la configuración requerida, todos los inicios de sesión de tu tenant serán procesados por esta Rule. Asegúrate de que todos los componentes se hayan configurado correctamente y se hayan verificado en un tenant de prueba antes de activar la integración en producción. |

<div id="make-link-titles-scannable">
  ## Haz que los títulos de los enlaces sean fáciles de identificar
</div>

Cuando incluyas enlaces a otros recursos, asegúrate de que el enlace describa la acción o tarea del documento al que apunta. Un texto de enlace genérico dificulta identificar el contenido de un vistazo, porque obliga a los lectores a leer todo el texto que rodea al enlace para saber qué deben seleccionar.

| Recomendado                                                                                                                                                                                   | Desaconsejado                                 |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| Puedes usar el Auth0 Dashboard o la Management API para [personalizar el texto de la experiencia de Universal Login](/docs/es/customize/login-pages/universal-login/customize-templates).        | Haz clic [aquí](/docs/es/get-started).           |
| Para aprender a asignar roles a miembros de organizaciones, lee [Agregar roles a los miembros de la organización](/docs/es/manage-users/organizations/configure-organizations/add-member-roles). | Puedes ir a [este enlace](/docs/es/get-started). |

<div id="write-device-agnostic-directions">
  ## Redacta instrucciones que no dependan del dispositivo
</div>

Recuerda que las interfaces de usuario se muestran de forma diferente en cada dispositivo. Además, ten presente que los usuarios pueden estar utilizando dispositivos de asistencia para ver tu contenido.

| Preferido                              | Desaconsejado                                         |
| -------------------------------------- | ----------------------------------------------------- |
| Selecciona **Create**.                 | Haz clic en **Create**.                               |
| Cambia a la vista de **Applications**. | Cambia a la pestaña **Applications**.                 |
| Busca **Secreto de cliente**.          | Desplázate hacia arriba hasta **Secreto de cliente**. |

<div id="use-verbs-and-nounadjectives-properly">
  ## Usa correctamente los verbos y los sustantivos/adjetivos
</div>

Ten cuidado con las palabras que pueden usarse tanto como verbos como sustantivos o adjetivos. Las formas verbales suelen llevar un espacio entre palabras, mientras que los sustantivos y los adjetivos pueden escribirse en una sola palabra.

| Preferido                                 | Desaconsejado                  |
| ----------------------------------------- | ------------------------------ |
| Log in to your account.                   | Login to your account.         |
| Auth0 procesa inicios de sesión.          | Auth0 procesa log ins.         |
| Set up your hardware.                     | Setup your hardware.           |
| Describe la configuración de tu hardware. | Describe your hardware set up. |

<div id="use-sentence-case-for-headings">
  ## Usa mayúscula inicial en los encabezados
</div>

Escribe con mayúscula solo la primera palabra y cualquier otro nombre propio o nombre de producto de Auth0.

| Preferido                            | Desaconsejado                        |
| ------------------------------------ | ------------------------------------ |
| Agrega una dependencia               | Agrega una Dependencia               |
| Valida el inicio de sesión reanudado | Valida el Inicio de Sesión Reanudado |
| Crea una Action                      | Crea una action                      |

<div id="use-simple-tense-for-headings">
  ## Usa tiempos simples en los encabezados
</div>

Evita las palabras innecesarias usando tiempos simples en los encabezados.

| Preferido                               | No recomendado                                |
| --------------------------------------- | --------------------------------------------- |
| Asigna y cambia usuarios                | Asignando y cambiando usuarios                |
| Conecta una base de datos personalizada | Cómo conectar una base de datos personalizada |

<div id="write-steps-concisely">
  ## Redacta los pasos de forma concisa
</div>

Al redactar el texto introductorio de los pasos, sé directo y conciso. Recuerda que los usuarios ya saben qué deben hacer cuando ven una lista numerada de pasos.

| Preferido             | Desaconsejado                   |
| --------------------- | ------------------------------- |
| Para instalar el SDK: | Sigue los pasos a continuación: |

<div id="set-user-expectations-in-steps">
  ## Establece las expectativas de los usuarios en los pasos
</div>

Diles primero a los usuarios cuál será el resultado de lo que hagan antes de describir la acción que deben realizar.

| Preferido                                                   | Desaconsejado                                              |
| ----------------------------------------------------------- | ---------------------------------------------------------- |
| Para activar esta integración, selecciona **Save Changes**. | Selecciona **Save Changes** para activar esta integración. |

<div id="list-actions-within-steps-in-order">
  ## Enumera las acciones en cada paso en orden
</div>

En cada paso, indícale al usuario qué debe hacer y en qué orden.

| Recomendado                                                              | Desaconsejado                                                       |
| ------------------------------------------------------------------------ | ------------------------------------------------------------------- |
| Busca la sección **Integraciones** y selecciona **Agregar integración**. | Selecciona **Agregar integración** en la sección **Integraciones**. |

<div id="use-notes-and-warning-appropriately">
  ## Usa las notas y las advertencias correctamente
</div>

Las notas y las advertencias cumplen funciones distintas. Las notas deben incluir información general útil, mientras que las advertencias contienen información cuyo incumplimiento puede provocar fallos.

Si una información debe presentarse como advertencia, debes incluirla exactamente en el punto en que el usuario tenga que realizar la acción relacionada. Los usuarios no deberían tener que andar buscando las advertencias.
