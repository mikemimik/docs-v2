Si no puedes consultar una guía de estilo completa, revisa estos consejos para mejorar notablemente tu contenido.

<div id="use-active-voice">
  ## Usa la voz activa
</div>

Evita la voz pasiva (`to be` + verbo en pasado). Para que tu texto sea más dinámico, claro y, a menudo, más breve, intenta usar sujetos y la voz activa siempre que sea posible.

| Preferido                                                                                                                                                                                                                                                                                  | Desaconsejado                                                                                                                                                                                                                                                                               |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Una vez que agregues la configuración requerida, Auth0 procesará todos los inicios de sesión de tu tenant con esta Rule. Antes de activar la integración en producción, asegúrate de haber configurado correctamente todos los componentes y de haberlo verificado en un tenant de prueba. | Una vez agregada la configuración requerida, todos los inicios de sesión de tu tenant serán procesados por esta Rule. Asegúrate de que todos los componentes se hayan configurado correctamente y se hayan verificado en un tenant de prueba antes de activar la integración en producción. |

<div id="make-link-titles-scannable">
  ## Haz que los títulos de los enlaces se puedan revisar de un vistazo
</div>

Cuando enlaces a otros recursos, asegúrate de que el enlace describa la acción o la tarea del documento al que apunta. Un texto de enlace genérico dificulta revisar el contenido de un vistazo porque obliga a los lectores a leer todo el texto que rodea al enlace para identificar qué deben seleccionar.

| Preferido                                                                                                                                                                                 | Desaconsejado                                 |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| Puedes usar el Auth0 Dashboard o la Management API para [personalizar el texto de la experiencia de Universal Login](/docs/es/customize/login-pages/universal-login/customize-templates).    | Haz clic [aquí](/docs/es/get-started).           |
| Para saber cómo asignar roles a miembros de organizaciones, lee [Agregar roles a miembros de la organización](/docs/es/manage-users/organizations/configure-organizations/add-member-roles). | Puedes ir a [este enlace](/docs/es/get-started). |

<div id="write-device-agnostic-directions">
  ## Escriba instrucciones que no dependan del dispositivo
</div>

Recuerde que las interfaces de usuario se muestran de forma diferente en distintos dispositivos. Además, tenga en cuenta que los usuarios pueden estar usando dispositivos de asistencia para ver su contenido.

| Preferido                           | No recomendado                                        |
| ----------------------------------- | ----------------------------------------------------- |
| Seleccione **Create**.              | Haga clic en **Create**.                              |
| Cambie a la vista **Applications**. | Cambie a la pestaña **Applications**.                 |
| Ubique **Secreto de cliente**.      | Desplácese hacia arriba hasta **Secreto de cliente**. |

<div id="use-verbs-and-nounadjectives-properly">
  ## Usa correctamente los verbos y los sustantivos/adjetivos
</div>

Ten cuidado con las palabras que pueden usarse tanto como verbos como sustantivos o adjetivos. Las formas verbales suelen escribirse separadas, mientras que los sustantivos y los adjetivos pueden escribirse como una sola palabra.

| Preferido                                 | Desaconsejado                       |
| ----------------------------------------- | ----------------------------------- |
| Inicia sesión en tu cuenta.               | Inicio sesión en tu cuenta.         |
| Auth0 procesa los inicios de sesión.      | Auth0 procesa los inicia sesión.    |
| Configura tu hardware.                    | Configuración tu hardware.          |
| Describe la configuración de tu hardware. | Describe tu hardware configuración. |

<div id="use-sentence-case-for-headings">
  ## Usa mayúscula solo en la primera palabra en los encabezados
</div>

Escribe con mayúscula solo la primera palabra y cualquier nombre propio o nombre de producto de Auth0.

| Preferido                            | No recomendado                       |
| ------------------------------------ | ------------------------------------ |
| Agrega una dependencia               | Agrega una Dependencia               |
| Valida el inicio de sesión reanudado | Valida el Inicio de Sesión Reanudado |
| Crea una Action                      | Crea una action                      |

<div id="use-simple-tense-for-headings">
  ## Usa tiempos verbales simples en los encabezados
</div>

Evita rodeos innecesarios usando tiempos verbales simples en los encabezados.

| Preferido                               | Desaconsejado                                 |
| --------------------------------------- | --------------------------------------------- |
| Asigna y cambia usuarios                | Asignando y cambiando usuarios                |
| Conecta una base de datos personalizada | Cómo conectar una base de datos personalizada |

<div id="write-steps-concisely">
  ## Redacta los pasos de forma concisa
</div>

Al redactar el texto introductorio de una serie de pasos, sé directo y conciso. Recuerda que los usuarios ya saben qué deben hacer cuando ven una lista numerada de pasos.

| Preferido             | Desaconsejado                   |
| --------------------- | ------------------------------- |
| Para instalar el SDK: | Sigue los pasos a continuación: |

<div id="set-user-expectations-in-steps">
  ## Establece las expectativas del usuario en los pasos
</div>

Indica a los usuarios cuál será el resultado antes de describir la acción que deben realizar.

| Recomendado                                                 | Desaconsejado                                              |
| ----------------------------------------------------------- | ---------------------------------------------------------- |
| Para activar esta integración, selecciona **Save Changes**. | Selecciona **Save Changes** para activar esta integración. |

<div id="list-actions-within-steps-in-order">
  ## Enumera las acciones dentro de cada paso en orden
</div>

Dentro de cada paso, indica al usuario qué debe hacer y en qué orden.

| Recomendado                                                            | Desaconsejado                                                  |
| ---------------------------------------------------------------------- | -------------------------------------------------------------- |
| Localiza la sección **Integrations** y selecciona **Add Integration**. | Selecciona **Add Integration** en la sección **Integrations**. |

<div id="use-notes-and-warning-appropriately">
  ## Usa las notas y las advertencias de forma adecuada
</div>

Las notas y las advertencias cumplen funciones diferentes. Las notas deben incluir información general que sea útil conocer, mientras que las advertencias contienen información que puede provocar fallos si no se siguen.

Si una información debe presentarse como advertencia, debes incluirla justo donde el usuario necesite realizar la acción relacionada. Los usuarios no deberían tener que buscar las advertencias.
