Si no puedes consultar una guía de estilo completa, revisa estos consejos para mejorar significativamente tu contenido.

<div id="use-active-voice">
  ## Usa la voz activa
</div>

Evita la voz pasiva (`to be` + verbo en participio). Para que tu texto sea más dinámico, claro y, a menudo, más breve, intenta usar sujetos y voz activa siempre que sea posible.

| Preferido                                                                                                                                                                                                                                                                                       | Desaconsejado                                                                                                                                                                                                                                                                 |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Una vez que agregues la configuración requerida, Auth0 procesará todos los inicios de sesión de tu tenant mediante esta Rule. Antes de activar la integración en producción, asegúrate de haber configurado correctamente todos los componentes y de haberlo verificado en un tenant de prueba. | Una vez que se agregue la configuración requerida, esta Rule procesará todos los inicios de sesión de tu tenant. Asegúrate de que todos los componentes se hayan configurado correctamente y verificado en un tenant de prueba antes de activar la integración en producción. |

<div id="make-link-titles-scannable">
  ## Haz que los títulos de los enlaces sean fáciles de revisar
</div>

Cuando incluyas enlaces a otros recursos, asegúrate de que el enlace describa la acción o la tarea del documento al que apunta. El texto genérico en los enlaces dificulta la revisión rápida porque obliga a los lectores a leer todo el texto que rodea al enlace para identificar qué deben seleccionar.

| Preferido                                                                                                                                                                                     | Desaconsejado                                 |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| Puedes usar el Auth0 Dashboard o la Management API para [personalizar el texto de la experiencia de Universal Login](/docs/es/customize/login-pages/universal-login/customize-templates).        | Haz clic [aquí](/docs/es/get-started).           |
| Para aprender a asignar roles a miembros de organizaciones, lee [Agregar roles a los miembros de la organización](/docs/es/manage-users/organizations/configure-organizations/add-member-roles). | Puedes ir a [este enlace](/docs/es/get-started). |

<div id="write-device-agnostic-directions">
  ## Redacta instrucciones independientes del dispositivo
</div>

Recuerda que las interfaces de usuario se muestran de forma diferente en distintos dispositivos. Además, recuerda que los usuarios pueden estar usando dispositivos adaptativos para ver tu contenido.

| Recomendado                         | Desaconsejado                                         |
| ----------------------------------- | ----------------------------------------------------- |
| Selecciona **Create**.              | Haz clic en **Create**.                               |
| Cambia a la vista **Applications**. | Cambia a la pestaña **Applications**.                 |
| Ubica **secreto de cliente**.       | Desplázate hacia arriba hasta **secreto de cliente**. |

<div id="use-verbs-and-nounadjectives-properly">
  ## Usa correctamente verbos y sustantivos/adjetivos
</div>

Ten cuidado con las palabras que pueden usarse tanto como verbos como sustantivos o adjetivos. Las formas verbales suelen requerir un espacio entre palabras, mientras que los sustantivos y los adjetivos pueden escribirse en una sola palabra.

| Preferido                                 | Desaconsejado                     |
| ----------------------------------------- | --------------------------------- |
| Inicia sesión en tu cuenta.               | Inicio de sesión en tu cuenta.    |
| Auth0 procesa inicios de sesión.          | Auth0 procesa iniciar sesión.     |
| Configura tu hardware.                    | Configuración tu hardware.        |
| Describe la configuración de tu hardware. | Describe tu hardware configurado. |

<div id="use-sentence-case-for-headings">
  ## Usa mayúscula solo en la primera palabra de los encabezados
</div>

Pon en mayúscula solo la primera palabra y cualquier otro nombre propio o nombre de producto de Auth0.

| Recomendado                           | Desaconsejado                         |
| ------------------------------------- | ------------------------------------- |
| Agregar una dependencia               | Agregar una Dependencia               |
| Validar el inicio de sesión reanudado | Validar el Inicio de Sesión Reanudado |
| Crear una Action                      | Crear una action                      |

<div id="use-simple-tense-for-headings">
  ## Usa tiempos simples en los encabezados
</div>

Evita las palabras innecesarias usando tiempos simples en los encabezados.

| Preferido                               | Desaconsejado                                 |
| --------------------------------------- | --------------------------------------------- |
| Asigna y cambia usuarios                | Asignando y cambiando usuarios                |
| Conecta una base de datos personalizada | Cómo conectar una base de datos personalizada |

<div id="write-steps-concisely">
  ## Redacta los pasos de forma concisa
</div>

Al redactar el texto introductorio de los pasos, sé directo y conciso. Recuerda que los usuarios ya saben qué deben hacer cuando ven una lista numerada de pasos.

| Preferible            | Desaconsejado                                |
| --------------------- | -------------------------------------------- |
| Para instalar el SDK: | Sigue los pasos que aparecen a continuación: |

<div id="set-user-expectations-in-steps">
  ## Define las expectativas de los usuarios en los pasos
</div>

Indica a los usuarios cuál será el resultado de su acción antes de describir la acción que deben realizar.

| Preferido                                                   | Desaconsejado                                              |
| ----------------------------------------------------------- | ---------------------------------------------------------- |
| Para activar esta integración, selecciona **Save Changes**. | Selecciona **Save Changes** para activar esta integración. |

<div id="list-actions-within-steps-in-order">
  ## Enumera las acciones de cada paso en orden
</div>

En cada paso, indica al usuario qué debe hacer y en qué orden debe hacerlo.

| Recomendado                                                              | No recomendado                                                      |
| ------------------------------------------------------------------------ | ------------------------------------------------------------------- |
| Busca la sección **Integraciones** y selecciona **Agregar integración**. | Selecciona **Agregar integración** en la sección **Integraciones**. |

<div id="use-notes-and-warning-appropriately">
  ## Usa las notas y las advertencias correctamente
</div>

Las notas y las advertencias cumplen funciones diferentes. Las notas deben incluir información general que conviene conocer, mientras que las advertencias contienen información cuyo incumplimiento puede provocar fallos.

Si una información debe presentarse como advertencia, inclúyela exactamente en el punto en que el usuario deba realizar la acción correspondiente. Los usuarios no deberían tener que buscar las advertencias.
