Los partners como tú usan integraciones de Actions para crear soluciones sin código que integran sus productos en los flujos de trabajo de Auth0. integraciones de Actions permiten a los clientes de Auth0 implementar soluciones como la verificación de identidad o la gestión del consentimiento sin escribir código. Tú creas la integración de Actions; los clientes la arrastran y la sueltan en su flujo de trabajo.

Las integraciones de Actions son funciones autónomas que se ejecutan en puntos específicos de la plataforma de Auth0. Escritas en JavaScript, son de código cerrado; los clientes no pueden modificar el código.

<div id="create-and-publish-an-actions-integration">
  ## Crear y publicar una integración de Actions
</div>

Crear una integración de Actions implica planificación, desarrollo y pruebas. Antes de ponerla a disposición de los clientes en el [Auth0 Marketplace](https://marketplace.auth0.com/), Auth0 revisará la integración de Actions. Para obtener información sobre Auth0 Marketplace, consulta [Marketplace Partners](/docs/es/customize/integrations/marketplace-partners).

<div id="step-1-determine-your-use-case">
  ### Paso 1. Determina tu caso de uso
</div>

Identifica cuál será el caso de uso de tu integración de Actions. ¿Qué problema resolverá para los clientes? ¿Cómo lo resolverá? Para obtener más información, lee [Definir un caso de uso de integración](/docs/es/customize/integrations/marketplace-partners/defining-an-integration-use-case).

<div id="step-2-choose-a-flow">
  ### Paso 2. Elegir un flujo
</div>

Las integraciones de Actions constan de una sola [Action](/docs/es/customize/actions). Lo que una Action puede hacer depende de en qué parte del entorno de ejecución de Auth0 se ejecute: su [flujo](/docs/es/customize/actions/explore-triggers) de Action. Los flujos de Action disponibles son:

* **Login flow** se ejecuta de forma sincrónica después de que un usuario se registre o inicie sesión correctamente. Los eventos de login incluyen flujos interactivos (como username/password o inicio de sesión federado) y flujos no interactivos (como comprobaciones de sesión e intercambios de <Tooltip tip="token de actualización: token que se usa para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+token">token de actualización</Tooltip>). Para obtener más información sobre este flujo, lee [Login Flow](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).
* **Machine to Machine flow** se ejecuta de forma sincrónica cuando una aplicación solicita un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> mediante la Concesión de credenciales de cliente. Para obtener más información sobre este flujo, lee [Machine to Machine Flow](/docs/es/customize/actions/explore-triggers/machine-to-machine-trigger).
* **Pre User Registration flow** se ejecuta de forma sincrónica antes de que Auth0 agregue un usuario a una conexión de base de datos o <Tooltip tip="passwordless: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>. Este flujo no se ejecuta para inicios de sesión federados, como conexiones empresariales o sociales. Para obtener más información sobre este flujo, lee [Pre User Registration Flow](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/pre-user-registration-trigger).
* **Post User Registration flow** se ejecuta de forma asíncrona después de que Auth0 agregue un usuario a una conexión de base de datos o passwordless. Este flujo no se ejecuta para inicios de sesión federados, como conexiones empresariales o sociales. Para obtener más información sobre este flujo, lee [Post User Registration Flow](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/post-user-registration-trigger).
* **Post Change Password flow** se ejecuta de forma asíncrona después de que el cliente cambie la contraseña de un usuario de conexión de base de datos. Este flujo no se ejecuta para cambios de contraseña en otros tipos de conexiones. Para obtener más información sobre este flujo, lee [Post Change Password Flow](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-change-password-trigger).
* **Send Phone Message** **flow** envía un mensaje de voz o SMS como parte de un proveedor personalizado de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además de username y password, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>. Para obtener información sobre los proveedores de MFA personalizados, lee [Customize Multi-factor Authentication SMS and Voice Messages](/docs/es/customize/customize-sms-or-voice-messages). Para obtener más información sobre este flujo, lee [Send Phone Message Flow](/docs/es/customize/actions/explore-triggers/mfa-notifications-trigger).

Para obtener más información sobre los flujos de Action, lee [Explore Flows and Triggers](/docs/es/customize/actions/explore-triggers). Si necesitas redirigir a los usuarios a un endpoint durante el login, lee [Redirect Actions for Partners](/docs/es/customize/integrations/marketplace-partners/redirect-actions-for-partners).

<div id="step-3-build-an-actions-integration">
  ## Paso 3. Crear una integración de Actions
</div>

Una vez que hayas decidido qué flujo se ajusta al caso de uso de tu integración de Action, crea tu Action en un tenant de prueba. Para obtener instrucciones, lee [Write Your First Action](/docs/es/customize/actions/write-your-first-action). Esta Action será la base del código de tu integración de Action.

Además, para ayudarte a crear tu integración, puedes usar nuestras plantillas de integración de Action:

* [Plantilla de integración de Action de Post Login](https://github.com/auth0-marketplace/TEMPLATE-action-post-login)
* [Plantilla de integración de Action de máquina a máquina](https://github.com/auth0-marketplace/TEMPLATE-action-client-credentials)
* [Plantilla de integración de Action de Registro previo de usuario](https://github.com/auth0-marketplace/TEMPLATE-action-pre-user-registration)
* [Plantilla de integración de Action de Post User Registration](https://github.com/auth0-marketplace/TEMPLATE-action-post-user-registration)
* [Plantilla de integración de Action de Post Change Password](https://github.com/auth0-marketplace/TEMPLATE-action-post-password-change)
* [Plantilla de integración de Action de Enviar mensaje telefónico](https://github.com/auth0-marketplace/TEMPLATE-action-phone-message)

Para saber cómo hacer que tu integración de Actions sea segura y estable, lee [Pautas de codificación de Actions](/docs/es/customize/actions/action-coding-guidelines). Sigue también estas pautas específicas para las integraciones de Actions:

* Al crear tu Action en un tenant de prueba, usa secretos para todos los valores que un cliente deba proporcionar (como identificadores, nombres de tenant y umbrales). Cuando envías tu integración de Actions, Auth0 te pide que indiques qué campos deben estar cifrados en reposo (seguir siendo secretos) y cuáles pueden ser campos de texto editables.
* Usa un indicador de configuración para el modo de depuración/prueba del registro que ayude con el Setup o la resolución de problemas. Las integraciones de Actions rara vez deberían generar registros en todas las ejecuciones.
* Escribe los nombres de los secretos en `ALL_CAPS_SNAKE_CASE` para facilitar la revisión.
* Las integraciones de Actions son de código cerrado; los clientes no pueden ver ni modificar el código. Cualquier entrada específica del cliente necesaria para controlar la integración de Action debe provenir de secretos o de la configuración.

<div id="step-4-test-your-actions-integrations">
  ### Paso 4. Prueba tus integraciones de Actions
</div>

Como los revisores de Auth0 no tienen acceso a tu servicio, debes probar tu Action antes de enviarla para su revisión. Prueba manualmente tanto las ejecuciones correctas como los casos límite. Incluye estas comprobaciones:

* ¿La Action falla silenciosamente o bloquea la ejecución posterior?
* ¿Cómo funciona la Action si falla una solicitud HTTP?
* ¿Qué mensajes de error, si los hay, se muestran a los usuarios?
* ¿Qué información se registra?
* ¿Se puede eludir o suplantar la Action?

Durante el proceso de envío, Auth0 solicita cobertura de pruebas unitarias para el caso satisfactorio. Auth0 proporciona todas las herramientas y ejemplos necesarios para que esto sea lo más sencillo posible.

<div id="step-5-submit-your-actions-integration">
  ### Paso 5. Envía tu integración de Actions
</div>

Una vez que hayas escrito y probado exhaustivamente tu integración de Actions, envíala a Auth0 para su revisión: presenta una [solicitud de servicio de Auth0 Marketplace](https://autheco.atlassian.net/servicedesk/customer/portal/1/group/1/create/43). Tras una revisión inicial satisfactoria, la integración se evalúa para comprobar el cumplimiento de los requisitos de seguridad.

<div id="step-6-publish-your-actions-integration">
  ### Paso 6. Publica tu integración de Actions
</div>

Una vez que Auth0 complete las revisiones de tu integración de Actions, te notificaremos si la integración las ha superado y te informaremos de los plazos de publicación.
