title: Actions Integrations para Partners
---

Los partners como tú usan Actions Integrations para crear soluciones sin código que integran sus productos en los flujos de trabajo de Auth0. Actions Integrations permiten a los clientes de Auth0 implementar soluciones como la verificación de identidad o la gestión del consentimiento sin escribir código. Tú escribes la Actions Integration; los clientes la arrastran y la sueltan en su flujo de trabajo.

Actions Integrations son funciones autónomas que se ejecutan en puntos específicos de la plataforma Auth0. Escritas en JavaScript, son de código cerrado; los clientes no pueden modificar el código.

<div id="create-and-publish-an-actions-integration">
  ## Crear y publicar una Actions Integration
</div>

Crear una Actions Integration implica planificarla, desarrollarla y probarla. Antes de ponerla a disposición de los clientes en [Auth0 Marketplace](https://marketplace.auth0.com/), Auth0 revisa la Actions Integration. Para obtener más información sobre Auth0 Marketplace, consulta [Marketplace Partners](/docs/es/customize/integrations/marketplace-partners).

<div id="step-1-determine-your-use-case">
  ### Paso 1. Determina tu caso de uso
</div>

Identifica qué caso de uso cubrirá tu Actions Integration. ¿Qué problema resolverá para los clientes? ¿Cómo lo resolverá? Para obtener más información, lee [Definir un caso de uso de integración](/docs/es/customize/integrations/marketplace-partners/defining-an-integration-use-case).

<div id="step-2-choose-a-flow">
  ### Paso 2. Elegir un flujo
</div>

Las Actions Integrations constan de una sola [Action](/docs/es/customize/actions). Lo que una Action puede hacer está determinado por el lugar en el que se ejecuta dentro del entorno de ejecución de Auth0: su Action [flujo](/docs/es/customize/actions/explore-triggers). Los flujos de Action disponibles son:

* **flujo de Login** se ejecuta de forma sincrónica después de que un usuario se registre o inicie sesión correctamente. Los eventos de login incluyen flujos interactivos (como nombre de usuario/contraseña o inicio de sesión federado) y flujos no interactivos (como comprobaciones de sesión e intercambios de <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=refresh+token">token de actualización</Tooltip>). Para obtener más información sobre este flujo, lea [Flujo de Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).
* **flujo de máquina a máquina** se ejecuta de forma sincrónica cuando una aplicación solicita un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> mediante la Concesión de credenciales de cliente. Para obtener más información sobre este flujo, lea [Flujo de máquina a máquina](/docs/es/customize/actions/explore-triggers/machine-to-machine-trigger).
* **flujo de Registro previo de usuario** se ejecuta de forma sincrónica antes de que Auth0 agregue un usuario a una conexión de base de datos o <Tooltip tip="Passwordless: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>. Este flujo no se ejecuta para inicios de sesión federados, como conexiones empresariales o sociales. Para obtener más información sobre este flujo, lea [Flujo de Registro previo de usuario](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/pre-user-registration-trigger).
* **flujo de Registro posterior de usuario** se ejecuta de forma asincrónica después de que Auth0 agregue un usuario a una conexión de base de datos o passwordless. Este flujo no se ejecuta para inicios de sesión federados, como conexiones empresariales o sociales. Para obtener más información sobre este flujo, lea [Flujo de Registro posterior de usuario](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/post-user-registration-trigger).
* **flujo Posterior al cambio de contraseña** se ejecuta de forma asincrónica después de que el cliente cambie la contraseña de un usuario de una conexión de base de datos. Este flujo no se ejecuta para cambios de contraseña en otros tipos de conexiones. Para obtener más información sobre este flujo, lea [Flujo Posterior al cambio de contraseña](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-change-password-trigger).
* **Enviar mensaje telefónico** **flujo** envía un mensaje de voz o SMS como parte de un proveedor de <Tooltip tip="Multi-factor authentication (MFA): User authentication process that uses a factor in addition to username and password such as a code via SMS." cta="View Glossary" href="/docs/es/glossary?term=MFA">MFA</Tooltip> personalizado. Para obtener información sobre proveedores de MFA personalizados, lea [Personalizar mensajes SMS y de voz de autenticación multifactor](/docs/es/customize/customize-sms-or-voice-messages). Para obtener más información sobre este flujo, lea [Flujo de Enviar mensaje telefónico](/docs/es/customize/actions/explore-triggers/mfa-notifications-trigger).

Para obtener más información sobre los flujos de Action, lea [Explorar Flows y triggers](/docs/es/customize/actions/explore-triggers). Si necesita redirigir a los usuarios a un endpoint durante el login, lea [Redirect Actions para partners](/docs/es/customize/integrations/marketplace-partners/redirect-actions-for-partners).

<div id="step-3-build-an-actions-integration">
  ## Paso 3. Crea una Actions Integration
</div>

Una vez que hayas decidido qué flujo se ajusta al caso de uso de tu Actions Integration, crea tu Action en un tenant de pruebas. Para obtener instrucciones, consulta [Write Your First Action](/docs/es/customize/actions/write-your-first-action). Esta Action será la base del código de tu Actions Integration.

Además, para ayudarte a crear tu integración, puedes usar nuestras plantillas de integración de Action:

* [Plantilla de integración de Action de Post Login](https://github.com/auth0-marketplace/TEMPLATE-action-post-login)
* [Plantilla de integración de Action de máquina a máquina](https://github.com/auth0-marketplace/TEMPLATE-action-client-credentials)
* [Plantilla de integración de Action de Registro previo de usuario](https://github.com/auth0-marketplace/TEMPLATE-action-pre-user-registration)
* [Plantilla de integración de Action de Post User Registration](https://github.com/auth0-marketplace/TEMPLATE-action-post-user-registration)
* [Plantilla de integración de Action de Post Change Password](https://github.com/auth0-marketplace/TEMPLATE-action-post-password-change)
* [Plantilla de integración de Action de Enviar mensaje telefónico](https://github.com/auth0-marketplace/TEMPLATE-action-phone-message)

Para saber cómo hacer que tu Actions Integration sea segura y estable, consulta [Pautas de codificación de Actions](/docs/es/customize/actions/action-coding-guidelines). Sigue también estas pautas específicas para las Actions Integrations:

* Al crear tu Action en un tenant de prueba, usa secretos para todos los valores que un cliente necesite proporcionar (como identificadores, nombres de tenant y umbrales). Cuando envíes tu Actions Integration, Auth0 te pedirá que indiques qué campos deben cifrarse en reposo (seguir siendo secretos) y cuáles pueden ser campos de texto editables.
* Usa un indicador de configuración para el modo de depuración/prueba del logging que ayude con la configuración o la resolución de problemas. Las Actions Integrations rara vez deberían generar logs en todas las ejecuciones.
* Escribe los nombres de los secretos en `ALL_CAPS_SNAKE_CASE` para facilitar la revisión.
* Las Actions Integrations son de código cerrado; los clientes no pueden ver ni modificar el código. Cualquier dato específico del cliente necesario para controlar la Actions Integration debe provenir de secretos o de la configuración.

<div id="step-4-test-your-actions-integrations">
  ### Paso 4. Prueba tus Actions Integrations
</div>

Como los revisores de Auth0 no tienen acceso a tu servicio, debes probar tu Action antes de enviarla para su revisión. Haz pruebas manuales tanto de las ejecuciones correctas como de los casos límite. Incluye estas comprobaciones:

* ¿La Action falla silenciosamente o bloquea la ejecución posterior?
* ¿Cómo funciona la Action si falla una solicitud HTTP?
* ¿Qué mensajes de error, si corresponde, quedan expuestos a los usuarios?
* ¿Qué información se registra?
* ¿Se puede eludir o falsificar la Action?

Durante el proceso de envío, Auth0 solicita cobertura de pruebas unitarias para el caso exitoso. Auth0 proporciona todas las herramientas y ejemplos necesarios para que esto sea lo más sencillo posible.

<div id="step-5-submit-your-actions-integration">
  ### Paso 5. Envía tu Actions Integration
</div>

Una vez que hayas escrito y probado exhaustivamente tu Actions Integration, envíala a Auth0 para su revisión: presenta una [solicitud de servicio en Auth0 Marketplace](https://autheco.atlassian.net/servicedesk/customer/portal/1/group/1/create/43). Tras una revisión inicial satisfactoria, la integración se somete a una revisión de cumplimiento de seguridad.

<div id="step-6-publish-your-actions-integration">
  ### Paso 6. Publica tu Actions Integration
</div>

Una vez que Auth0 haya completado las revisiones de tu Actions Integration, te notificaremos si la integración las ha superado y sobre los plazos de publicación.
