<Card title="Antes de empezar">
  Necesitarás una cuenta de Microsoft con acceso al [Azure Portal](https://azure.microsoft.com/en-us).
</Card>

El servicio [Azure API Management](https://azure.microsoft.com/en-us/products/api-management/) te permite crear nuevas API o importar definiciones de API existentes y publicarlas para que las utilicen las audiencias autorizadas. Auth0 facilita la autorización de los usuarios de tu API (mediante los estándares de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>).

En este tutorial, aprenderás a usar Auth0 para autenticar y autorizar a los usuarios cuando accedan a la Basic Calculator API administrada por Azure API Management.

<div id="configure-auth0">
  ## Configurar Auth0
</div>

<div id="create-an-api">
  ### Crear una API
</div>

Debes crear una API en Auth0 para representar la API administrada por Azure API Management Service. Para obtener más información, consulta [Registrar APIs](/docs/es/get-started/auth0-overview/set-up-apis).

1. Ve a [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) y selecciona **Crear API**.
2. Configura los siguientes parámetros:

   | Campo                  | Valor              |
   | ---------------------- | ------------------ |
   | **Nombre**             | `Basic Calculator` |
   | **Identificador**      | `basic-calculator` |
   | **Algoritmo de firma** | `RS256`            |
3. Selecciona **Crear**.

<div id="create-a-regular-web-application">
  ### Crear una aplicación web clásica
</div>

Deberás crear una aplicación web clásica (RWA) en Auth0 para comunicarte con el OAuth 2.0 <Tooltip tip="Servidor de autorización: servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip> en Azure:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y selecciona el botón **Create Application**.
2. Introduce un **Name** para tu aplicación y selecciona **Aplicaciones web clásicas** como tipo de aplicación.

<div id="create-and-enable-a-connection">
  ### Crear y habilitar una conexión
</div>

Tendrás que crear una conexión en Auth0 y habilitarla para tu aplicación:

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y selecciona **Create DB Connection**.
2. Introduce un **Name** para tu conexión. Te sugerimos elegir un nombre que refleje el origen de los usuarios (por ejemplo, `Facebook` para una conexión que contiene usuarios que usan sus credenciales de Facebook, o `site-sign-ups` para una conexión de base de datos en la que los usuarios se registran en tu sitio).
3. Selecciona **Create**.
4. Ve a la página **Settings** de tu conexión.
5. Cambia a la vista **Applications**, donde verás una lista de todas las aplicaciones de tu tenant.
6. Activa la aplicación que creaste anteriormente para habilitarla en la conexión.

<div id="create-a-user">
  ### Crear un usuario
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si ya tienes una lista de usuarios, puedes importarlos o crear una [conexión de base de datos personalizada](/docs/es/authenticate/database-connections/custom-db).
</Callout>

Necesitarás tener al menos un usuario en tu conexión para probar la autenticación y la autorización. Si no tienes una lista existente de usuarios para la conexión, puedes crear uno manualmente:

1. Ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y selecciona **Crear usuario**.
2. Introduce un **Email** y una **Password**, y selecciona la conexión que creaste en el paso 3 en el campo **Connection**.
3. Selecciona **Create**.

<div id="configure-azure">
  ## Configurar Azure
</div>

<div id="create-azure-api-management-service">
  ### Crear Azure API Management Service
</div>

Deberás crear una instancia de Azure API Management Service en Azure para administrar tu API:

1. En la barra de navegación, selecciona **Crear un recurso**.
2. Una vez redirigido, selecciona **Web** &gt; **API Management**.
3. Proporciona los siguientes parámetros de configuración:

   | Parámetro                                | Descripción                                                                                                                                                                                                                                                                                                                                                                           |
   | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Nombre**                               | El nombre de tu servicio (que también se usará para crear la URL que necesitas para acceder al servicio)                                                                                                                                                                                                                                                                              |
   | **Suscripción**                          | El plan de suscripción de Azure que usarás con el servicio                                                                                                                                                                                                                                                                                                                            |
   | **Grupo de recursos**                    | La colección de [recursos](https://learn.microsoft.com/en-us/azure/azure-resource-manager/management/manage-resources-portal) que comparten un ciclo de vida, permisos y políticas. Puedes usar un grupo de recursos existente o crear uno nuevo (deberás proporcionar un nombre para el grupo si creas uno nuevo)                                                                    |
   | **Ubicación**                            | Elige la ubicación de tu instancia de API                                                                                                                                                                                                                                                                                                                                             |
   | **Nombre de la organización**            | El nombre de tu organización                                                                                                                                                                                                                                                                                                                                                          |
   | **Correo electrónico del administrador** | La dirección de correo electrónico de la persona que administrará esta instancia                                                                                                                                                                                                                                                                                                      |
   | **Nivel de precios**                     | El nivel de precios que desees, que determina la cantidad de llamadas que puedes hacer a tu API, así como el volumen máximo de transferencia de datos permitido. Debes optar por el [plan Developer](https://azure.microsoft.com/en-us/pricing/details/api-management/) o uno superior; el plan Consumption no ofrece la funcionalidad suficiente para que esta integración funcione. |
4. Selecciona **Crear** para iniciar el aprovisionamiento de tu servicio (esto puede tardar hasta 15 minutos en completarse).

<div id="import-the-sample-api">
  ### Importa la API de ejemplo
</div>

Para este tutorial, importaremos y usaremos la Basic Calculator API que proporciona Microsoft. Para obtener instrucciones detalladas, consulta [Importa y publica tu primera API](https://docs.microsoft.com/en-us/azure/api-management/import-and-publish#go-to-your-api-management-instance) de Microsoft.

Cuando termines, selecciona **Create** para importar tu API. Se te redirigirá a la página de resumen de tu API cuando se haya importado por completo.

<div id="configure-an-oauth-20-authorization-server">
  ### Configurar un servidor de autorización OAuth 2.0
</div>

Para usar Auth0 para proteger tu API de Azure, tendrás que registrar Auth0 como un servidor de autorización OAuth 2.0:

1. Busca el área **OAuth 2.0 + OpenID Connect** de tu instancia de API Management Service cerca de la barra de navegación. Selecciona **OAuth 2.0** y, a continuación, selecciona **Add**.
2. Ve a la pantalla de configuración **Add OAuth2 service** y selecciona el tipo de concesión de código de autorización.
3. Configura los siguientes parámetros:

   | Parameter                          | Description                                                                                                                                                                                                                   |
   | ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Display name**                   | Introduce un nombre descriptivo para tu servidor de autorización, como `Auth0`.                                                                                                                                               |
   | **Id**                             | Este campo debería completarse automáticamente según el nombre para mostrar que proporciones.                                                                                                                                 |
   | **Description**                    | Introduce una descripción para tu servidor de autorización, como `Auth0 API Authentication`.                                                                                                                                  |
   | **Client registration page URL**   | Introduce `https://placeholder.contoso.com` como marcador de posición.                                                                                                                                                        |
   | **Authorization code grant types** | Selecciona `Authorization code`.                                                                                                                                                                                              |
   | **Authorization endpoint URL**     | Consulta el [endpoint de descubrimiento de OIDC](https://\{yourDomain}/.well-known/openid-configuration) de tu tenant y añade el parámetro `audience`. Por ejemplo: `https://{yourDomain}/authorize?audience={API_AUDIENCE}`. |
   | **Authorization request method**   | El valor predeterminado es `GET`.                                                                                                                                                                                             |
   | **Token endpoint URL**             | Consulta el [endpoint de descubrimiento de OIDC](https://\{yourDomain}/.well-known/openid-configuration) de tu tenant.                                                                                                        |
   | **Client authentication methods**  | Selecciona `Basic`.                                                                                                                                                                                                           |
   | **Access Token sending method**    | Selecciona `encabezado Authorization`.                                                                                                                                                                                        |
   | **Default scope**                  | Especifica un [scope](/docs/es/glossary?term=scope) predeterminado, si es necesario.                                                                                                                                             |
4. En la sección **Client Credentials**, introduce el ID de cliente de tus aplicaciones de Auth0 en el campo **Client ID** y el secreto de cliente en el campo **Client secret**.
5. Cambia a la vista **Redirect URI** y copia el valor del URI del campo **Authorization code flow grant**.
6. Selecciona **Create**.

<div id="configure-azure-api-to-use-auth0">
  ### Configurar la API de Azure para usar Auth0
</div>

Después de crear tu servidor de autorización de OAuth 2.0 en Azure, tendrás que actualizar tu API de Azure para usar Auth0 para la autorización de usuarios:

1. En la sección **APIs** del menú de navegación, selecciona **APIs**.
2. Selecciona Basic Calculator API y ve a la vista **Design**.
3. Cambia a la vista **Settings**.
4. Ve a la sección **Security** y selecciona **OAuth 2.0** en **User Authorization**.
5. En el campo **servidor de autorización**, selecciona el servidor que configuraste en el paso anterior.
6. Haz clic en **Save**.

<div id="set-the-allowed-callback-url-in-auth0">
  ## Configura la Allowed Callback URL en Auth0
</div>

Después de que tu API de Azure se haya aprovisionado y configurado para usar Auth0 para autorizar usuarios, tendrás que actualizar tu aplicación de Auth0:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Selecciona tu aplicación y cambia a la vista **Settings**.
3. Introduce el Redirect URI que copiaste anteriormente en el campo **Allowed Callback URLs**.
4. Haz clic en **Save**.

<div id="test-the-integration">
  ## Probar la integración
</div>

En el Azure Portal, abra su instancia de API Management Service.

1. Seleccione **Developer Console** para abrir la interfaz para desarrolladores de sus API.
2. Vaya a **APIs** &gt; **Basic Calculator**. Se abrirá la página donde puede hacer una llamada `GET` para sumar dos números enteros.
3. Seleccione **Try It**. Se abrirá la página donde puede proporcionar los parámetros de la llamada.
4. Vaya a la sección **Authorization** y seleccione **código de autorización** (junto al campo **Auth0**).
5. Aparecerá una ventana emergente con el widget de inicio de sesión de Auth0 (si no aparece, asegúrese de que los bloqueadores de ventanas emergentes estén deshabilitados en su navegador). Introduzca las credenciales del usuario de Auth0 que creó en el Paso 4 e inicie sesión.
6. Si pudo iniciar sesión correctamente, aparecerá un mensaje con la fecha de expiración del token de acceso que puede usar para llamar a la API.
7. Vaya hasta la parte inferior y seleccione **Send**. Si la solicitud se realiza correctamente, verá un mensaje con la respuesta `HTTP 200` en la parte inferior de la página.

<div id="configure-a-jwt-validation-policy-for-access-tokens">
  ### Configurar una directiva de validación de JWT para los tokens de acceso
</div>

En el paso anterior, se le solicita al usuario que inicie sesión cuando intenta realizar una llamada desde la Developer Console. La Developer Console intenta obtener un <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> en nombre del usuario para incluirlo en la solicitud a la API. Todos los tokens de acceso se pasarán a la API mediante el encabezado `Authorization`.

Si desea validar el token de acceso incluido en cada solicitud, puede hacerlo mediante la [directiva Validate JWT](https://learn.microsoft.com/en-us/azure/api-management/validate-jwt-policy). Consulte la documentación de Microsoft sobre [cómo configurar una directiva de API Management](https://learn.microsoft.com/en-us/azure/api-management/set-edit-policies?tabs=form).
