<Card title="Antes de empezar">
  Necesitarás una cuenta de Microsoft con acceso al [Azure Portal](https://azure.microsoft.com/en-us).
</Card>

El servicio [Azure API Management](https://azure.microsoft.com/en-us/products/api-management/) te permite crear nuevas API o importar definiciones de API existentes y publicarlas para que las usen los públicos autorizados. Auth0 facilita la autorización de los usuarios de tu API (mediante los estándares de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>).

En este tutorial, aprenderás a usar Auth0 para autenticar y autorizar a los usuarios cuando accedan a una Basic Calculator API administrada por Azure API Management.

<div id="configure-auth0">
  ## Configura Auth0
</div>

<div id="create-an-api">
  ### Crear una API
</div>

Deberás crear una API en Auth0 para representar la API administrada por Azure API Management Service. Para obtener más información, consulta [Register APIs](/docs/es/get-started/auth0-overview/set-up-apis).

1. Ve a [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) y selecciona **Create API**.
2. Configura los siguientes parámetros:

   | Campo                  | Valor              |
   | ---------------------- | ------------------ |
   | **Nombre**             | `Basic Calculator` |
   | **Identificador**      | `basic-calculator` |
   | **Algoritmo de firma** | `RS256`            |
3. Selecciona **Create**.

<div id="create-a-regular-web-application">
  ### Crear una aplicación web clásica
</div>

Tendrás que crear una aplicación web clásica (RWA) en Auth0 para comunicarte con el servidor OAuth 2.0 <Tooltip tip="Servidor de autorización: servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funciones a las que un usuario tiene acceso." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip> en Azure:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y selecciona el botón **Create Application**.
2. Introduce un **Name** para tu aplicación y selecciona **aplicación web clásica** como tipo de aplicación.

<div id="create-and-enable-a-connection">
  ### Crear y habilitar una conexión
</div>

Tendrás que crear una conexión en Auth0 y habilitarla para tu aplicación:

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y selecciona **Create DB Connection**.
2. Introduce un **Name** para tu conexión. Te sugerimos elegir un nombre que refleje el origen de los usuarios (por ejemplo, `Facebook` para una conexión que contiene usuarios que usan sus credenciales de Facebook, o `site-sign-ups` para una conexión de base de datos en la que los usuarios se registran en tu sitio).
3. Selecciona **Create**.
4. Ve a la página **Settings** de tu conexión.
5. Cambia a la vista **Applications**, donde verás una lista de todas las aplicaciones de tu tenant.
6. Activa la aplicación que creaste anteriormente para habilitarla en la conexión.

<div id="create-a-user">
  ### Crear un usuario
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si ya tienes un conjunto de usuarios, puedes importarlos o crear una [conexión de base de datos personalizada](/docs/es/authenticate/database-connections/custom-db).
</Callout>

Necesitarás tener al menos un usuario en tu conexión para probar la autenticación y la autorización. Si no tienes un conjunto de usuarios existente para la conexión, puedes crear uno manualmente:

1. Ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y selecciona **Create User**.
2. Introduce un **Email** y una **Password**, y selecciona la conexión que creaste en el paso 3 en el campo **Connection**.
3. Selecciona **Create**.

<div id="configure-azure">
  ## Configurar Azure
</div>

<div id="create-azure-api-management-service">
  ### Crear Azure API Management Service
</div>

Deberás crear una instancia de API Management Service en Azure para administrar tu API:

1. En la barra de navegación, selecciona **Create a resource**.
2. Cuando se te redirija, selecciona **Web** &gt; **API Management**.
3. Introduce los siguientes valores de configuración:

   | Parámetro               | Descripción                                                                                                                                                                                                                                                                                                                                                                          |
   | ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Name**                | El nombre de tu servicio (que también se usará para crear la URL que necesitas para acceder al servicio)                                                                                                                                                                                                                                                                             |
   | **Subscription**        | El plan de suscripción de Azure que usarás con el servicio                                                                                                                                                                                                                                                                                                                           |
   | **Resource group**      | El conjunto de [recursos](https://learn.microsoft.com/en-us/azure/azure-resource-manager/management/manage-resources-portal) que comparten un ciclo de vida, permisos y políticas. Puedes usar un grupo de recursos existente o crear uno nuevo (si creas uno nuevo, deberás proporcionar un nombre para el grupo)                                                                   |
   | **Location**            | Elige la ubicación que dará servicio a tu instancia de API                                                                                                                                                                                                                                                                                                                           |
   | **Organization name**   | El nombre de tu organización                                                                                                                                                                                                                                                                                                                                                         |
   | **Administrator email** | La dirección de correo electrónico de la persona que administrará esta instancia                                                                                                                                                                                                                                                                                                     |
   | **Pricing tier**        | El nivel de precios que quieras, que determina la cantidad de llamadas que puedes hacer a tu API, así como la cantidad máxima de transferencia de datos permitida. Debes optar por el [plan Developer](https://azure.microsoft.com/en-us/pricing/details/api-management/) o uno superior; el plan Consumption no ofrece suficiente funcionalidad para que esta integración funcione. |
4. Selecciona **Create** para comenzar a aprovisionar tu servicio (esto puede tardar hasta 15 minutos en completarse).

<div id="import-the-sample-api">
  ### Importar la API de ejemplo
</div>

En este tutorial, importaremos y usaremos la Basic Calculator API proporcionada por Microsoft. Para obtener instrucciones detalladas, consulta [Importar y publicar tu primera API](https://docs.microsoft.com/en-us/azure/api-management/import-and-publish#go-to-your-api-management-instance) de Microsoft.

Cuando termines, selecciona **Create** para importar tu API. Se te redirigirá a la página de resumen de tu API cuando la importación se haya completado.

<div id="configure-an-oauth-20-authorization-server">
  ### Configurar un servidor de autorización OAuth 2.0
</div>

Para usar Auth0 para proteger tu API de Azure, tendrás que registrar Auth0 como un servidor de autorización OAuth 2.0:

1. Busca la sección **OAuth 2.0 + OpenID Connect** de tu instancia de API Management Service, cerca de la barra de navegación. Selecciona **OAuth 2.0** y, a continuación, **Add**.
2. Ve a la pantalla de configuración **Add OAuth2 service** y selecciona el tipo de concesión código de autorización.
3. Configura los siguientes parámetros:

   | Parámetro                          | Descripción                                                                                                                                                                                                           |
   | ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Display name**                   | Introduce un nombre descriptivo para tu servidor de autorización, como `Auth0`.                                                                                                                                       |
   | **Id**                             | Este campo debería completarse automáticamente según el nombre para mostrar que indiques.                                                                                                                             |
   | **Description**                    | Introduce una descripción para tu servidor de autorización, como `Auth0 API Authentication`.                                                                                                                          |
   | **Client registration page URL**   | Introduce `https://placeholder.contoso.com` como marcador de posición.                                                                                                                                                |
   | **Authorization code grant types** | Selecciona `Authorization code`.                                                                                                                                                                                      |
   | **Authorization endpoint URL**     | Consulta el [endpoint de OIDC Discovery](https://\{yourDomain}/.well-known/openid-configuration) de tu tenant y añade el parámetro `audience`. Por ejemplo: `https://{yourDomain}/authorize?audience={API_AUDIENCE}`. |
   | **Authorization request method**   | El valor predeterminado es `GET`.                                                                                                                                                                                     |
   | **Token endpoint URL**             | Consulta el [endpoint de OIDC Discovery](https://\{yourDomain}/.well-known/openid-configuration) de tu tenant.                                                                                                        |
   | **Client authentication methods**  | Selecciona `Basic`.                                                                                                                                                                                                   |
   | **Access Token sending method**    | Selecciona `Authorization header`.                                                                                                                                                                                    |
   | **Default scope**                  | Especifica un [scope](/docs/es/glossary?term=scope) predeterminado, si es necesario.                                                                                                                                     |
4. En la sección **Client Credentials**, introduce el ID de cliente de tus aplicaciones de Auth0 en el campo **Client ID** y el secreto de cliente en el campo **Client secret**.
5. Cambia a la vista **Redirect URI** y copia el valor del URI del campo **Authorization code flow grant**.
6. Selecciona **Create**.

<div id="configure-azure-api-to-use-auth0">
  ### Configura Azure API para usar Auth0
</div>

Después de crear tu servidor de autorización OAuth 2.0 en Azure, tendrás que actualizar tu Azure API para usar Auth0 para la autorización de usuarios:

1. En la sección **APIs** del menú de navegación, selecciona **APIs**.
2. Selecciona Basic Calculator API y ve a la vista **Design**.
3. Cambia a la vista **Settings**.
4. Ve a la sección **Security** y selecciona **OAuth 2.0** en **User Authorization**.
5. Selecciona el servidor que configuraste en el paso anterior en el campo **Authorization Server**.
6. Haz clic en **Save**.

<div id="set-the-allowed-callback-url-in-auth0">
  ## Configura la URL de devolución de llamada permitida en Auth0
</div>

Una vez aprovisionada y configurada tu API de Azure para usar Auth0 en la autorización de usuarios, tendrás que actualizar tu aplicación de Auth0:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Selecciona tu aplicación y ve a la vista **Settings**.
3. Pega la Redirect URI que copiaste anteriormente en el campo **Allowed Callback URLs**.
4. Haz clic en **Save**.

<div id="test-the-integration">
  ## Prueba la integración
</div>

En Azure Portal, abre tu instancia de API Management Service.

1. Selecciona **Developer Console** para abrir la parte de tus API orientada a desarrolladores.
2. Ve a **APIs** &gt; **Basic Calculator**. Se abrirá la página donde puedes hacer una llamada `GET` para sumar dos enteros.
3. Selecciona **Try It**. Se abrirá la página donde puedes proporcionar los parámetros de tu llamada.
4. Ve a la sección **Authorization** y selecciona **Código de autorización** (junto al campo **Auth0**).
5. Aparecerá una ventana emergente con el widget de inicio de sesión de Auth0 (si no aparece, asegúrate de que cualquier bloqueador de ventanas emergentes esté desactivado en tu navegador). Introduce las credenciales del usuario de Auth0 que creaste en el paso 4 e inicia sesión.
6. Si has iniciado sesión correctamente, aparecerá un mensaje con la fecha de vencimiento del token de acceso que puedes usar para llamar a tu API.
7. Ve hasta la parte inferior y selecciona **Send**. Si la solicitud se realiza correctamente, verás un mensaje con la respuesta `HTTP 200` en la parte inferior de la página.

<div id="configure-a-jwt-validation-policy-for-access-tokens">
  ### Configurar una política de validación de JWT para Access Tokens
</div>

En el paso anterior, se le pide al usuario que inicie sesión cuando intenta hacer una llamada desde la Developer Console. La Developer Console intenta obtener un <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> en nombre del usuario para incluirlo en la solicitud a la API. Todos los Access Tokens se enviarán a la API mediante el encabezado `Authorization`.

Si desea validar el Access Token incluido en cada solicitud, puede hacerlo mediante la [política Validate JWT](https://learn.microsoft.com/en-us/azure/api-management/validate-jwt-policy). Consulte la documentación de Microsoft sobre cómo [configurar una política de API Management](https://learn.microsoft.com/en-us/azure/api-management/set-edit-policies?tabs=form).
