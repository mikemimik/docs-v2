Cuando se anunció en Kickstarter, le dimos soporte de inmediato y esperamos largas semanas para conseguir una.

Por fin llegó y ya podemos escribir nuestro primer programa: obtener un token de Auth0 y llamar a una API.

Tessel apuesta por la compatibilidad total con Javascript. La mayoría de los módulos básicos de Node también funcionan, pero no todos. Consulta [la documentación de Tessel en Github](https://github.com/tessel/docs/blob/master/compatibility.md) para más detalles.

<div id="the-sample">
  ## El ejemplo
</div>

Este ejemplo es sencillo:

1. Llamamos al endpoint del propietario del recurso en Auth0 con las credenciales del dispositivo
2. Recibimos un token
3. Usamos el token para llamar a una API

<Frame>![Tessel - Diagrama de flujo de Tessel a Auth0](/docs/images/cdy7uua7fh8z/3iyOfO564gkbyQSsvTJE9c/a981ca1166d1384ba2e5e7dd72f5a255/2023-09-22_13-14-21.png)</Frame>

export const codeExample = `var http = require('https');
var tessel = require('tessel');

tessel.syncClock(function () {

  var device_id = 'tessel-01';
  var password = 'LA CONTRASEÑA DEL TESSEL';

  authenticate(device_id, password, function(e,token){

    if(e) return console.log("Error:" + e);

    getDeviceProfile(token.access_token, function(e, profile){
      console.log("Perfil del dispositivo:");
      console.log(profile);
    });
  });

  function getDeviceProfile(token, done){
    request('{yourDomain}',
          '/userinfo',
          'GET',
          {
          "Content-type": "application/json",
          "Authorization": "Bearer " + token
        },
        null,
        function(e,response){
          if(e) return done(e);
          done(null, JSON.parse(response));
        });
  }

  function authenticate(device_id, password, done)
  {
    request('{yourDomain}',
          '/oauth/ro',
          'POST',
          {
          "Content-type": "application/json",
        },
        JSON.stringify({
            client_id:   '{yourClientId}',
            username:    device_id,
            password:    password,
            connection:  'devices',
            grant_type:  "password",
            scope: 'openid'
          }),
          function(e,response){
            if(e) return done(e);
            done(null, JSON.parse(response));
        });
  }

  function request(host, path, method, headers, body, done){
    var options = {
      hostname: host,
      path: path,
      method: method,
      headers: headers
    };

    var req = http.request(options, function(res) {
      res.setEncoding('utf8');

      var response = "";

      res.on('data', function (chunk) {
        response += chunk;
      });

      res.on('end', function(){
        done(null, response);
        });
    });

    req.on('error', function(e) {
      done(e);
    });

    if( body ) req.write(body);
    req.end();
  }
});`;

Aspectos destacados del código:

1. Esto es compatible en un 99 % con Node (el único módulo específico del dispositivo es `tessel`, que solo usamos para asegurarnos de que todas las llamadas SSL se realicen con referencias de tiempo adecuadas.
2. La función `request` es un contenedor simple para las funciones del módulo `http`. El módulo `request` no funciona actualmente en Tessel.

El endpoint de <Tooltip tip="Resource Owner: Entidad (como un usuario o aplicación) capaz de otorgar acceso a un recurso protegido." cta="Ver glosario" href="/docs/es/glossary?term=Resource+Owner">Resource Owner</Tooltip> requiere credenciales (como nombre de usuario/contraseña), por lo que el repositorio de usuarios del backend conectado a Auth0 debe ser compatible con esto (como una conexión de base de datos o Active Directory).

<div id="tessel-setup">
  ## Configuración de Tessel
</div>

* Ejecuta `tessel update` para asegurarte de instalar el firmware más reciente con soporte para SSL.
* Obviamente, necesitarás conexión a Internet. Puedes configurar la conexión Wi‑Fi con el comando `tessel wifi`.
* Envía siempre las credenciales (como `username`/`password`) a través de redes seguras.
