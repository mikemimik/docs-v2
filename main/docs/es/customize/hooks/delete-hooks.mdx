<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Recomendamos encarecidamente que utilice Actions para extender Auth0. Con Actions, tiene acceso a información de tipos detallada, documentación integrada y paquetes públicos de `npm`, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lea [Descubra cómo funcionan las Auth0 Actions](/docs/es/customize/actions/actions-overview).

  Para ayudarle con la migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página específica de [Pasarse a Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarle durante el proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada de blog: [Preparación para el fin de vida útil de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debe crear nuevas Rules o Hooks solo en su entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar sus Rules a Actions, lea [Migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar sus Hooks a Actions, lea [Migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Cuando ya no necesite Hooks, puede eliminarlos usando el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Hooks](https://manage.auth0.com/#/) y haz clic en el icono de engranaje junto al hook que quieres eliminar.
2. Haz clic en **Delete** y luego en YES, DELETE HOOK.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Realiza una solicitud `DELETE` al [endpoint Delete a Hook](https://auth0.com/docs/api/management/v2/#!/Hooks/delete_hooks_by_id). Asegúrate de reemplazar los valores de marcador de posición `HOOK_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de tu hook y tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de la Management API, respectivamente.

<AuthCodeGroup>
  ```bash cURL
  curl --request DELETE \
    --url 'https://{yourDomain}/api/v2/hooks/HOOK_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/hooks/HOOK_ID");
  var request = new RestRequest(Method.DELETE);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"

  	req, _ := http.NewRequest("DELETE", url, nil)

  	res, _ := http.DefaultClient.Do(req)

  	body, _ := ioutil.ReadAll(res.Body)
  ```

| Valor                   | Descripción                                                                                                                                                                                                             |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `HOOK_ID`               | ID del Hook que se va a eliminar.                                                                                                                                                                                       |
| `MGMT_API_ACCESS_TOKEN` | Tokens de acceso de la Management API con el [scope](/docs/es/glossary?term=scope) `update:hooks`. Para obtener más información, consulta [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |
