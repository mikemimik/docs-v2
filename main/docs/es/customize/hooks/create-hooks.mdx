<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026** y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de su vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías para [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte durante el proceso de migración.

  Para obtener más información sobre la deprecación de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debes crear Rules o Hooks nuevos solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, consulta [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, consulta [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes crear varios hooks desde el Dashboard o con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

También puedes importar y exportar hooks con la herramienta de línea de comandos (CLI) de Auth0 Deploy.

<Warning>
  Aunque puedes crear varios hooks para un mismo punto de extensibilidad, cada punto de extensibilidad solo puede tener un hook habilitado a la vez. Todos los hooks adicionales que crees para ese punto de extensibilidad se deshabilitan automáticamente, por lo que debes habilitarlos de forma explícita. El hook habilitado se ejecutará para todas las aplicaciones y API.
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si lo deseas, puedes añadir secretos (como claves de Twilio o cadenas de conexión a bases de datos) a los hooks.
</Callout>

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/hooks) y haz clic en **+Create**.
2. Introduce un nombre descriptivo para el hook, selecciona el punto de extensibilidad en el que debe ejecutarse y haz clic en **Create**.
3. Busca la sección correspondiente al punto de extensibilidad que seleccionaste y haz clic en el icono de lápiz junto al hook que creaste.
4. Actualiza el hook con el Hook Editor y haz clic en el icono del disquete para guardar.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Haz una solicitud `POST` al [endpoint Create a Hook](https://auth0.com/docs/api/management/v2/#!/Hooks/post_hooks). Asegúrate de reemplazar los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `HOOK_NAME`, `HOOK_SCRIPT` y `EXTENSIBILITY_POINT_NAME` por tu <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de la Management API, el nombre del Hook, el script del Hook y el nombre del punto de extensibilidad, respectivamente.

| Valor                      | Descripción                                                                                                                                                                                                                                                                                                                                |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MGMT_API_ACCESS_TOKEN`    | Token de acceso para la Management API con el [scope](/docs/es/glossary?term=scope) `create:hooks`. Para obtener más información, consulta [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens).                                                                                                                   |
| `HOOK_NAME`                | Nombre del Hook que se va a crear.                                                                                                                                                                                                                                                                                                         |
| `HOOK_SCRIPT`              | Script que contiene el código del Hook. Debe coincidir con lo que introducirías si crearas un Hook nuevo desde el Dashboard.                                                                                                                                                                                                               |
| `EXTENSIBILITY_POINT_NAME` | Nombre del punto de extensibilidad en el que debe ejecutarse el Hook. Las opciones incluyen: `credentials-exchange`, `pre-user-registration`, `post-user-registration`, `post-change-password`. Para obtener más información sobre los puntos de extensibilidad, consulta [Extensibiity Points](/docs/es/customize/actions/explore-triggers). |

<Card title="Gestiona los rate limits al llamar a las API desde Hooks">
  Si llamas a las API de Auth0 desde el script de un Hook, tendrás que gestionar los rate limits. Para ello, revisa el encabezado `X-RateLimit-Remaining` y actúa en consecuencia cuando el valor devuelto se acerque a 0.

  Además, agrega lógica para gestionar los casos en los que superes los rate limits establecidos y recibas el código de estado HTTP `429` (`Too Many Requests`). En ese caso, si es necesario reintentar, lo mejor es aplicar una espera progresiva para evitar entrar en un bucle infinito de reintentos.
</Card>

Para obtener más información sobre los rate limits de Auth0 para Hooks, consulta Rate Limits.
