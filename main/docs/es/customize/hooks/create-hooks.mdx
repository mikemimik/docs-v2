<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026** y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Recomendamos encarecidamente que uses Actions para extender Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías para [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, solo debes crear nuevas Rules o Hooks en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, consulta [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, consulta [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes crear varios hooks con el Dashboard o <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

También puedes importar y exportar hooks con la herramienta Deploy Command-Line Interface (CLI).

<Warning>
  Aunque puedes crear varios hooks para un punto de extensibilidad determinado, cada punto de extensibilidad solo puede tener un hook habilitado a la vez. Los hooks que crees después para ese punto de extensibilidad se deshabilitarán automáticamente, por lo que deberás habilitarlos explícitamente. El hook habilitado se ejecutará para todas las aplicaciones y API.
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Opcionalmente, puedes agregar secrets (como claves de Twilio o cadenas de conexión de base de datos) a los hooks.
</Callout>

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Hooks](https://manage.auth0.com/#/hooks) y haz clic en **+Create**.
2. Introduce un nombre descriptivo para el hook, selecciona el punto de extensibilidad en el que debe ejecutarse y haz clic en **Create**.
3. Busca la sección del punto de extensibilidad que seleccionaste y haz clic en el icono del lápiz junto al hook que creaste.
4. Actualiza el hook en el Hook Editor y haz clic en el icono del disco para guardar.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Haz una solicitud `POST` al [endpoint Create a Hook](https://auth0.com/docs/api/management/v2/#!/Hooks/post_hooks). Asegúrate de reemplazar los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `HOOK_NAME`, `HOOK_SCRIPT` y `EXTENSIBILITY_POINT_NAME` por tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de la Management API, el nombre del Hook, el script del Hook y el nombre del punto de extensibilidad, respectivamente.

| Valor                      | Descripción                                                                                                                                                                                                                                                                                                                                     |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN`    | Token de acceso para la Management API con el [scope](/docs/es/glossary?term=scope) `create:hooks`. Para obtener más información, consulta [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens).                                                                                                                        |
| `HOOK_NAME`                | Nombre del Hook que se va a crear.                                                                                                                                                                                                                                                                                                              |
| `HOOK_SCRIPT`              | Script que contiene el código del Hook. Debe coincidir con lo que introducirías si crearas un Hook nuevo desde el Dashboard.                                                                                                                                                                                                                    |
| `EXTENSIBILITY_POINT_NAME` | Nombre del punto de extensibilidad en el que debe ejecutarse el Hook. Las opciones incluyen: `credentials-exchange`, `pre-user-registration`, `post-user-registration`, `post-change-password`. Para obtener más información sobre los puntos de extensibilidad, consulta [Puntos de extensibilidad](/docs/es/customize/actions/explore-triggers). |

<Card title="Gestiona los límites de frecuencia al llamar a APIs desde Hooks">
  Si llamas a las APIs de Auth0 desde el script de un Hook, tendrás que gestionar los límites de frecuencia. Para ello, revisa el header `X-RateLimit-Remaining` y actúa en consecuencia cuando el valor devuelto se acerque a 0.

  Además, agrega lógica para gestionar los casos en los que superes los límites de frecuencia establecidos y recibas el código de estado HTTP `429` (`Too Many Requests`). En este caso, si es necesario reintentar, lo mejor es aplicar una espera progresiva para evitar entrar en un bucle infinito de reintentos.
</Card>

Para obtener más información sobre la limitación de frecuencia de Auth0 para Hooks, consulta Límites de frecuencia.
