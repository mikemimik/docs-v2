<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lee [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías para [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página específica, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, solo debes crear nuevos Rules o Hooks en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, lee [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, lee [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes actualizar Hooks desde el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

También puedes importar y exportar hooks con la [herramienta Deploy Command-Line Interface (CLI)](/docs/es/deploy-monitor/deploy-cli-tool).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Opcionalmente, puedes agregar secrets (como claves de Twilio o cadenas de conexión de bases de datos) a los hooks.
</Callout>

<div id="rename-hooks-using-the-dashboard">
  ## Cambiar el nombre de los hooks desde el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Hooks](https://manage.auth0.com/#/) y haz clic en el icono de engranaje junto al hook que quieres cambiar de nombre.
2. Haz clic en **Rename**.
3. Escribe un nombre nuevo para el Hook y haz clic en **Rename**.

   <Frame>![Auth0 Cambiar el nombre de Hooks](/docs/images/cdy7uua7fh8z/2Xp8Z3uYGoGZrF5EGXLPLQ/6c68c77f97133b2066219a68540b1e73/Screen_Shot_2021-05-19_at_12.08.00_PM.png)</Frame>

<div id="update-hooks-using-the-management-api">
  ## Actualiza los hooks con la Management API
</div>

Haz una llamada `PATCH` al [endpoint para actualizar un Hook](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_hooks_by_id). Asegúrate de reemplazar los placeholder values `HOOK_ID` y `MGMT_API_ACCESS_TOKEN` por tu ID de Hook y tu <Tooltip tip="Access Token: Credencial de Authorization, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> de la Management API, respectivamente.

<AuthCodeGroup>
  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/hooks/HOOK_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "name": "HOOK_NAME", "script": "HOOK_SCRIPT", "enabled": "true" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/hooks/HOOK_ID");
  var request = new RestRequest(Method.PATCH);
  request.AddHeader("content-type", "application/json");
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("cache-control", "no-cache");
  request.AddParameter("application/json", "{ "name": "HOOK_NAME", "script": "HOOK_SCRIPT", "enabled": "true" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"

  	req, _ := http.NewRequest("PATCH", url, payload)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("cache-control", "no-cache")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

| Valor                   | Descripción                                                                                                                                                                                                               |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `HOOK_ID`               | ID del Hook que se va a actualizar.                                                                                                                                                                                       |
| `MGMT_API_ACCESS_TOKEN` | Tokens de acceso para la Management API con el [scope](/docs/es/glossary?term=scope) `update:hooks`. Para obtener más información, consulta [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |
| `HOOK_NAME`             | Nombre del Hook que se va a crear.                                                                                                                                                                                        |
| `HOOK_SCRIPT`           | Script que contiene el código del Hook. Debe coincidir con lo que introducirías si estuvieras creando un Hook nuevo desde el Dashboard.                                                                                   |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La propiedad `enabled` indica si la rule está habilitada (`true`) o deshabilitada (`false`).
</Callout>

<Card title="Gestiona los límites de frecuencia al llamar a APIs desde Hooks">
  Si llamas a las API de Auth0 desde el script de un Hook, tendrás que gestionar los límites de frecuencia. Para ello, revisa el header `X-RateLimit-Remaining` y actúa según corresponda cuando el valor devuelto se aproxime a 0.

  Además, agrega lógica para gestionar los casos en los que superes los límites de frecuencia proporcionados y recibas el código de estado HTTP `429` (`Too Many Requests`). En ese caso, si hace falta reintentar, lo mejor es aplicar una espera progresiva para evitar entrar en un bucle infinito de reintentos.
</Card>
