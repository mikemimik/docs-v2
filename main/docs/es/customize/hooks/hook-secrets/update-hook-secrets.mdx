<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los tenants nuevos creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos detallada, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lee [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías para [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparativas de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debes crear nuevas Rules o Hooks solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, lee [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, lee [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes actualizar los secretos de Hooks añadidos a un Hook determinado usando el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Los secretos de Hooks también se pueden importar y exportar con la herramienta Deploy Command-Line Interface (CLI). Para obtener más información, lee [Deploy CLI Tool](/docs/es/deploy-monitor/deploy-cli-tool).

<div id="update-hook-secrets-using-the-dashboard">
  ## Actualizar secretos de Hooks desde el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Hooks](https://manage.auth0.com/#/) y haz clic en el icono del lápiz junto al Hook que quieres editar.
2. En el editor del Hook, haz clic en el icono de la llave inglesa y luego en **Secrets**.
3. Haz clic en el icono del lápiz y el papel junto al valor del secreto de Hooks que quieres editar.
4. Haz los cambios necesarios en el nombre y/o el valor del secreto de Hooks seleccionado y haz clic en **Save**.

<div id="update-hook-secrets-using-the-management-api">
  ## Actualizar secretos de Hooks mediante la Management API
</div>

Realiza una llamada `PATCH` al [endpoint actualizar secretos de Hooks](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_secrets). Asegúrate de sustituir los valores de marcador de posición `HOOK_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de tu hook y tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> de la Management API, respectivamente.

| Valor                   | Descripción                                                                                                                                                                                                               |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN` | Tokens de acceso para la Management API con el [scope](/docs/es/glossary?term=scope) `update:hooks`. Para obtener más información, consulta [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |
| `HOOK_ID`               | ID del hook para el que deseas actualizar los secretos.                                                                                                                                                                   |
| `HOOK_SECRET_KEY`       | Nombre del secreto que deseas actualizar para el hook especificado. Este endpoint acepta un objeto de pares clave-valor.                                                                                                  |
| `HOOK_SECRET_VALUE`     | Valor del secreto que deseas actualizar para el hook especificado. Este endpoint acepta un objeto de pares clave-valor.                                                                                                   |

<Warning>
  Al recuperar secretos de Hooks configurados para un Hook específico, los valores contendrán el texto de marcador de posición `_VALUE_NOT_SHOWN`. Ten cuidado de no devolver este texto de marcador de posición durante una actualización, o se sobrescribirá el valor real del secreto.
</Warning>
