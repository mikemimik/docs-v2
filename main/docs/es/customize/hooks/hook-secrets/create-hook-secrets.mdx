<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para tenants nuevos creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Recomendamos encarecidamente usar Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos detallada, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y a [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparativas de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debes crear nuevas Rules o Hooks solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para aprender a migrar tus Rules a Actions, consulta [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para aprender a migrar tus Hooks a Actions, consulta [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes crear varios secretos de Hooks para cualquier [Hook](/docs/es/customize/hooks) usando el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Los secretos de Hooks también se pueden importar y exportar con la [Deploy Command-Line Interface (CLI) Tool](/docs/es/deploy-monitor/deploy-cli-tool).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Puedes crear hasta 20 secretos para un Hook determinado.
</Callout>

<div id="create-hook-secrets-using-the-dashboard">
  ## Crear secretos de Hooks desde el Dashboard
</div>

1. Ve a la página de [Hooks](https://manage.auth0.com/#/hooks) en el [Auth0 Dashboard](https://manage.auth0.com/#/) y haz clic en el icono del lápiz junto al Hook que quieres editar.
2. En el editor del Hook, haz clic en el icono de la llave inglesa y luego en **Secrets**.
3. Haz clic en **Add Secret**.
4. Introduce un nombre descriptivo y un valor para el secreto, y haz clic en **Save**.

<div id="create-hook-secrets-using-the-management-api">
  ## Crear secretos de Hooks con la Management API
</div>

Realiza una llamada `POST` al [endpoint Add Hook Secrets](https://auth0.com/docs/api/management/v2/#!/Hooks/post_secrets). Asegúrate de sustituir los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `HOOK_ID`, `HOOK_SECRET_KEY` y `HOOK_SECRET_VALUE` por tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o un JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de la Management API, el ID del Hook y los pares clave-valor del Hook, respectivamente.

| Valor                   | Descripción                                                                                                                                      |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens) con el [scope](/docs/es/glossary?term=scope) `create:hooks`. |
| `HOOK_ID`               | ID del hook al que desea agregar secrets.                                                                                                        |
| `HOOK_SECRET_KEY`       | Nombre del secret que desea agregar al hook especificado. Este endpoint acepta un objeto de pares clave-valor.                                   |
| `HOOK_SECRET_VALUE`     | Valor del secret que desea agregar al hook especificado. Este endpoint acepta un objeto de pares clave-valor.                                    |
