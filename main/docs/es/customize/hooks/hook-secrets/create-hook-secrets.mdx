<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprender cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparación para el fin de vida útil de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto retirar las funcionalidades de Rules y Hooks en 2026, debes crear nuevas Rules o Hooks solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para aprender a migrar tus Rules a Actions, consulta [Migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para aprender a migrar tus Hooks a Actions, consulta [Migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes crear varios secretos de Hooks para cualquier [Hook](/docs/es/customize/hooks) mediante el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Los secretos de Hooks también se pueden importar y exportar con la [Deploy Command-Line Interface (CLI) Tool](/docs/es/deploy-monitor/deploy-cli-tool).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Puedes crear hasta 20 secretos para un Hook determinado.
</Callout>

<div id="create-hook-secrets-using-the-dashboard">
  ## Crear secretos de Hooks desde el Dashboard
</div>

1. Vaya a la página [Hooks](https://manage.auth0.com/#/hooks) en el [Auth0 Dashboard](https://manage.auth0.com/#/) y haga clic en el icono de lápiz junto al Hook que desea editar.
2. En el editor del Hook, haga clic en el icono de llave inglesa y luego en **Secrets**.
3. Haga clic en **Add Secret**.
4. Introduzca un nombre descriptivo y un valor para su secreto y haga clic en **Save**.

<div id="create-hook-secrets-using-the-management-api">
  ## Crear secretos de Hooks con la Management API
</div>

Haz una llamada `POST` al [endpoint Add Hook Secrets](https://auth0.com/docs/api/management/v2/#!/Hooks/post_secrets). Asegúrate de reemplazar los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `HOOK_ID`, `HOOK_SECRET_KEY` y `HOOK_SECRET_VALUE` por tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> de la Management API, el ID del Hook y los pares clave-valor del Hook, respectivamente.

| Valor                   | Descripción                                                                                                                                      |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MGMT_API_ACCESS_TOKEN` | [token de acceso for the Management API](https://auth0.com/docs/api/management/v2/tokens) con el [scope](/docs/es/glossary?term=scope) `create:hooks`. |
| `HOOK_ID`               | ID del hook al que desea agregar secretos.                                                                                                       |
| `HOOK_SECRET_KEY`       | Nombre del secreto que desea agregar al hook especificado. Este endpoint acepta un objeto de pares clave-valor.                                  |
| `HOOK_SECRET_VALUE`     | Valor del secreto que desea agregar al hook especificado. Este endpoint acepta un objeto de pares clave-valor.                                   |
