<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026** y ya no están disponibles para tenants nuevos creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso a la funcionalidad Hooks hasta el fin de vida útil.

  Le recomendamos encarecidamente que use Actions para ampliar Auth0. Con Actions, tiene acceso a información de tipos más completa, documentación integrada y paquetes `npm` públicos, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulte [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para facilitar su migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y a [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarle en su proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulte nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debe crear nuevos Rules o Hooks solo en su entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar sus Rules a Actions, consulte [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar sus Hooks a Actions, consulte [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Hooks incorpora gestión integrada de secretos para almacenarlos de forma segura y ponerlos a disposición del código.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para acceder a un secreto de Hook configurado desde un hook, use `context.webtask.secrets.SECRET_NAME`.
</Callout>

</div>

Puede crear, actualizar, eliminar y ver secretos de Hooks desde el Dashboard o desde <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulte:

* [Crear secretos de Hooks](/docs/es/customize/hooks/hook-secrets/create-hook-secrets)
* [Actualizar secretos de Hooks](/docs/es/customize/hooks/hook-secrets/update-hook-secrets)
* [Eliminar secretos de Hooks](/docs/es/customize/hooks/hook-secrets/delete-hook-secrets)
* [Ver secretos de Hooks](/docs/es/customize/hooks/hook-secrets/view-hook-secrets)

Los secretos de Hooks también se pueden importar o exportar mediante la [herramienta Deploy Command-Line Interface (CLI).](/docs/es/deploy-monitor/deploy-cli-tool)
