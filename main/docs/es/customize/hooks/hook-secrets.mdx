<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para tenants nuevos creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos enriquecida, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprender cómo funcionan Auth0 Actions](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte durante el proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debes crear nuevos Rules o Hooks solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para aprender a migrar tus Rules a Actions, consulta [Migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para aprender a migrar tus Hooks a Actions, consulta [Migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

La funcionalidad Hooks incluye una gestión integrada de secretos para almacenarlos de forma segura y, al mismo tiempo, ponerlos a disposición del código de manera práctica.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para acceder a un Hook Secret configurado desde un hook, usa `context.webtask.secrets.SECRET_NAME`.
</Callout>

<div id="manage-hook-secrets">
  ## Administrar secretos de Hooks
</div>

Puedes crear, actualizar, eliminar y ver secretos de Hooks desde el Dashboard o desde <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulta:

* [Crear secretos de Hooks](/docs/es/customize/hooks/hook-secrets/create-hook-secrets)
* [Actualizar secretos de Hooks](/docs/es/customize/hooks/hook-secrets/update-hook-secrets)
* [Eliminar secretos de Hooks](/docs/es/customize/hooks/hook-secrets/delete-hook-secrets)
* [Ver secretos de Hooks](/docs/es/customize/hooks/hook-secrets/view-hook-secrets)

Los secretos de Hooks también se pueden importar o exportar con la herramienta [Deploy Command-Line Interface (CLI).](/docs/es/deploy-monitor/deploy-cli-tool)
