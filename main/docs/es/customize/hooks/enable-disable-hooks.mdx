<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y a [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en el proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, solo debes crear nuevas Rules o Hooks en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, consulta [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, consulta [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes habilitar o deshabilitar hooks desde el Dashboard o mediante <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Warning>
  Aunque puedes crear varios hooks para un punto de extensibilidad determinado, cada punto de extensibilidad solo puede tener un hook habilitado a la vez. Los hooks adicionales que crees para ese punto de extensibilidad se deshabilitarán automáticamente, por lo que deberás habilitarlos de forma explícita. El hook habilitado se ejecutará para todas las aplicaciones y API.
</Warning>

<Warning>
  Al crear nuevos hooks, Auth0 habilita automáticamente el primer Hook que crees para un punto de extensibilidad. Los hooks adicionales que crees para ese punto de extensibilidad se deshabilitarán automáticamente, por lo que deberás habilitarlos de forma explícita.
</Warning>

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Hooks](https://manage.auth0.com/#/) y localiza el punto de extensibilidad en el que quieres habilitar o deshabilitar un Hook.
2. Haz clic en el menú desplegable que está justo debajo del nombre y la descripción del punto de extensibilidad.

   <Frame>![Selección de Hook de Auth0](/docs/images/cdy7uua7fh8z/2LCBPLtrsjmmJYxCGfWquT/74b232fd96aa179c89f01e46d2e91ce3/dashboard-hooks-list.png)</Frame>
3. Selecciona el Hook que quieres habilitar y confirma. Si quieres deshabilitar todos los Hooks, selecciona `None`.

Aparecerá un punto verde junto al nombre de cada Hook habilitado.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Haz una solicitud `PATCH` al [endpoint Update a Hook](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_hooks_by_id). Asegúrate de reemplazar los valores de marcador de posición `HOOK_ID` y `MGMT_API_ACCESS_TOKEN` por tu Hook ID y el <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de la Management API, respectivamente.

<AuthCodeGroup>
  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/hooks/HOOK_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "enabled": "true" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/hooks/HOOK_ID");
  var request = new RestRequest(Method.PATCH);
  request.AddHeader("content-type", "application/json");
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("cache-control", "no-cache");
  request.AddParameter("application/json", "{ "enabled": "true" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/hooks/HOOK_ID"

  	payload := strings.NewReader("{ "enabled": "true" }")

  	req, _ := http.NewRequest("PATCH", url, payload)

  	req.Header.Add("content-type", "application/json")
  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("cache-control", "no-cache")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

| Valor                   | Descripción                                                                                                                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `HOOK_ID`               | ID del hook que se va a actualizar.                                                                                                                                                                                      |
| `MGMT_API_ACCESS_TOKEN` | token de acceso para la Management API con el [scope](/docs/es/glossary?term=scope) `create:hooks`. Para obtener más información, consulta [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La propiedad `enabled` indica si la rule está habilitada (`true`) o deshabilitada (`false`).
</Callout>
