<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026** y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso a la funcionalidad Hooks hasta su fin de vida útil.

  Recomendamos encarecidamente usar Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada, paquetes públicos de `npm` y la posibilidad de conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Entender cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarte durante la migración.

  Si quieres leer más sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debes crear nuevas Rules o Hooks solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, consulta [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, consulta [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Puedes habilitar o deshabilitar hooks desde el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Warning>
  Aunque puedes crear varios hooks para un mismo punto de extensibilidad, cada punto de extensibilidad solo puede tener un hook habilitado a la vez. Cualquier hook adicional que crees para ese punto de extensibilidad se deshabilitará automáticamente, por lo que tendrás que habilitarlo explícitamente. El hook habilitado se ejecutará para todas las aplicaciones y API.
</Warning>

<Warning>
  Al crear hooks nuevos, Auth0 habilita automáticamente el primer Hook que creas para un punto de extensibilidad. Cualquier hook adicional que crees para ese punto de extensibilidad se deshabilitará automáticamente, por lo que tendrás que habilitarlo explícitamente.
</Warning>

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Hooks](https://manage.auth0.com/#/) y localiza el punto de extensibilidad en el que quieres habilitar o deshabilitar un Hook.
2. Haz clic en el menú desplegable que está justo debajo del nombre y la descripción del punto de extensibilidad.

   <Frame>![Selección de un Hook de Auth0](/docs/images/cdy7uua7fh8z/2LCBPLtrsjmmJYxCGfWquT/74b232fd96aa179c89f01e46d2e91ce3/dashboard-hooks-list.png)</Frame>
3. Selecciona el Hook que quieres habilitar y confirma. Si quieres deshabilitar todos los Hooks, selecciona `None`.

Aparecerá un punto verde junto al nombre de cualquier Hook habilitado.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Realiza una solicitud `PATCH` al [endpoint Update a Hook](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_hooks_by_id). Asegúrate de reemplazar los valores de marcador de posición `HOOK_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de tu Hook y tu <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> de la Management API, respectivamente.

<AuthCodeGroup>
  ```bash cURL
  curl --request PATCH \
    --url 'https://{yourDomain}/api/v2/hooks/HOOK_ID' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "enabled": "true" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/hooks/HOOK_ID");

  request.AddHeader("cache-control", "no-cache");
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/hooks/HOOK_ID"

  	payload := strings.NewReader("{ "enabled": "true" }")

  	req, _ := http.NewRequest("PATCH", url, payload)

  	req.Header.Add("content-type", "application/json")
  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("cache-control", "no-cache")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

| Valor                   | Descripción                                                                                                                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `HOOK_ID`               | ID del hook que se va a actualizar.                                                                                                                                                                                      |
| `MGMT_API_ACCESS_TOKEN` | Token de acceso para la Management API con el [scope](/docs/es/glossary?term=scope) `create:hooks`. Para obtener más información, consulta [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La propiedad `enabled` indica si la rule está habilitada (`true`) o deshabilitada (`false`).
</Callout>
