Con los nodos Router, puedes agregar rules a formularios para crear saltos de lógica condicional que redirijan a los usuarios entre nodos. Por ejemplo, se puede definir una condición de rule para un atributo específico de `user_metadata` a fin de dirigir a un usuario a un step node o flow node específico.

* El nodo router agrega automáticamente una rule de caso predeterminado. Si no se cumple ninguna condición de rule, el recorrido continúa al nodo vinculado.
* Las rules se evalúan en el orden en que se colocan en el router. Cuando se cumple una condición de rule, el recorrido continúa al nodo vinculado y no se evalúa ninguna otra condición de rule.
* Puedes agregar hasta 10 condiciones de rule al nodo router.

<div id="add-a-router-rule">
  ## Agregar una rule de Router
</div>

Para agregar una condición de rule, sigue estos pasos:

1. En el editor de formularios, selecciona un nodo Router para abrir su configuración.
2. Selecciona **+ New Rule**.
3. Despliega la rule recién creada.
4. Para identificar la rule, agrega un **Alias**.
5. Agrega una variable con el icono **+**.
6. Selecciona el operador de rule para la variable.
7. Agrega el valor o la variable que debe cumplir la rule.
8. Vincula la nueva rule al paso o nodo de flujo correspondiente.
9. Selecciona **Publish** para guardar.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Te recomendamos definir un alias para la rule, de modo que puedas identificar fácilmente cada una en el nodo Router.
  * Puedes seleccionar entre varios operadores de rule para evaluar operandos. Por ejemplo, puedes configurar first&#95;name `is equal to` Jane.
  * Los operandos pueden ser valores estáticos o variables.
  * Una rule puede tener operadores `AND` y `OR` para evaluar lógica condicional adicional.
  * Después de agregar una rule, asegúrate de vincularla al nodo que quieras.
</Callout>

<div id="available-rule-operators">
  ## Operadores disponibles para rule
</div>

Los operadores de rule son:

* `is greater than`
* `is greater or equal than`
* `is less than`
* `is less than or equal`
* `is equal to`
* `is different to`
* `includes`
* `does not include`
* `starts with`
* `ends with`
* `exists`
* `does not exist`
* `is null`
* `is not null`
* `has not value`
* `is true`
* `is false`
