<Card title="Antes de empezar">
  1. Crea una [aplicación de máquina a máquina](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps) con los siguientes scopes habilitados:

  * `read:users`
  * `update:users`
  * `create:users`
  * `read:users_app_metadata`
  * `update:users_app_metadata`
  * `create:users_app_metadata`

  2. Agrega una [conexión de Vault](/docs/es/customize/forms/vault) usando las credenciales de la aplicación M2M.
</Card>

Formularios para Actions te permite crear pasos adicionales en tus flujos de registro o inicio de sesión y hacer que sean obligatorios antes de permitir que los usuarios accedan a tu aplicación.

<Frame>![Dashboard > Actions > Formularios > Caso de uso: formulario de registro personalizado](/docs/images/cdy7uua7fh8z/ZOQNi6SrEC9OREcBbdiJ3/d79f84fb61acd5473bdf932527e26d58/custom-signup-form.png)</Frame>

En las secciones siguientes se explica cómo crear formularios con pasos adicionales de registro usando nodos y flujos, junto con los pasos para agregar tu formulario a una Action de [Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="create-a-form-from-scratch">
  ## Crear un formulario desde cero
</div>

Sigue estos pasos para crear un formulario y recopilar información:

1. Abre el editor de formulario desde **Auth0 Dashboard &gt; Actions &gt; Formularios.**
2. Selecciona **Create Form** &gt; **Start from scratch**.

De manera predeterminada, un formulario nuevo contiene un nodo Start, un nodo Step y un nodo Ending screen.

<Frame>![Dashboard > Formularios > Use Cases > Form](/docs/images/cdy7uua7fh8z/2TZ9jSjgEcARFpULI6hz4R/a4319a1bd482f2ee5cc23100fdb2f42c/New-form-Forms-for-action-use-case.png)</Frame>

<div id="configure-the-step-node">
  ### Configurar el nodo Step
</div>

El nodo Step es la interfaz gráfica que ven los usuarios. Agrega [campos](/docs/es/customize/forms/nodes-and-components) al nodo Step para recopilar el nombre completo y el cargo del usuario siguiendo estos pasos:

1. Arrastra un campo de **Texto enriquecido** desde el **menú Components** al nodo **Step**.

   * **Texto enriquecido**: Introduce un mensaje personalizado.
   * Selecciona **Guardar**.
2. Arrastra un campo de **Texto** al nodo **Step**.

   * **ID**: Introduce `full_name`.
   * **Etiqueta**: Introduce `Full name`.
   * **Etiqueta**: Activa la casilla.
   * **Obligatorio**: Activa la casilla.
   * Selecciona **Guardar**.
3. Arrastra un campo de **Texto** al nodo **Step**.

   * **ID**: Introduce `job_title`.
   * **Etiqueta**: Introduce `Job title.`
   * **Etiqueta**: Activa la casilla.
   * **Obligatorio**: Activa la casilla.
   * Selecciona **Guardar**.
4. Selecciona **Publicar** para guardar.

<Frame>![Dashboard > Actions > Forms > Caso de uso Custom Signup Steps Step node](/docs/images/cdy7uua7fh8z/6Kcn0ai9fsVsqHwZpTGByG/37cbf4ffee63a7010926b15f06dd20ac/custom-signup-step-node.png)</Frame>

<div id="configure-the-flow-node">
  ### Configura el nodo Flow
</div>

Agrega un nodo **Flow** después del nodo **Step** para actualizar `user_metadata` y reanudar el flujo de autenticación siguiendo estos pasos:

1. Selecciona **Flow** en la parte inferior del editor de formularios.
2. Elimina el enlace existente entre los nodos **Step** y **Ending Screen**.
3. Selecciona el nuevo **Flow** &gt; **Haz clic para agregar un flujo** &gt; **Crea un flujo nuevo.**

   * Ingresa `Update user_metadata` en el campo **Name**.
   * Selecciona **Crear**.
   * Selecciona **Guardar**.
4. Vincula el nodo **Flow** con los nodos **Step** y **Ending Screen**, como se muestra en la imagen a continuación.
5. Selecciona **Publicar** para guardar.

<Frame>![Dashboard > Actions > formulario > Caso de uso Custom Signup Step flujo de Update User metadata](/docs/images/cdy7uua7fh8z/4nTdYdA2Hu5vK65smkdvJg/db3294ebc25a7c2636be96362a3ea56e/custom-signup-step-flow.png)</Frame>

Agrega al flujo una Action de Auth0 **Update user** siguiendo estos pasos:

1. Selecciona **Flow** **Update user&#95;metadata** &gt; **Editar flujo** para abrir el editor de flujos en una pestaña nueva.
2. Debajo de StartAction, selecciona el ícono **+** para agregar una Action **Update user**. Completa los siguientes campos y luego selecciona **Guardar** para continuar.

   * **Conexión**: Selecciona en el menú desplegable la conexión de Vault de tu aplicación M2M.

   * **ID de usuario**: Ingresa `{{context.user.user_id}}`.

   * **Cuerpo**: Copia y pega el siguiente código para actualizar `user_metadata` con las propiedades `full_name` y `job_title`.

     ```json lines
     {
       "user_metadata": {
         "full_name": "{{fields.full_name}}",
         "job_title": "{{fields.job_title}}"
       }
     }
     ```

   * Selecciona **Guardar**.
3. Selecciona **Publicar** para guardar.

<div id="retrieve-your-form-render-code">
  ## Obtén el código para renderizar tu formulario
</div>

Obtén el código para renderizar tu formulario y poder renderizarlo con una Action de Post Login personalizada siguiendo estos pasos:

* En el editor de formularios, selecciona **`<> Render`**.
* Selecciona **Copy**.

<Frame>![Dashboard > Actions > Formularios > Form > Embed](/docs/images/cdy7uua7fh8z/2EIUzc539sq9lwoTKbjoJv/496a864c0fc44a782c9cadca119dbfb0/embed.png)</Frame>

<div id="create-a-post-login-action">
  ## Crear una Action de Post Login
</div>

Crea una Action de Post Login para renderizar tu formulario siguiendo estos pasos:

1. Ve a **Auth0 Dashboard &gt; Actions &gt; Flows &gt; Login.**
2. Selecciona el icono **+** y luego **Build from scratch**:

   * **Name**: Ingresa `Render Additional Signup Form`.
   * **Trigger**: Selecciona `Login / Post Login`.
   * **Runtime**: Selecciona la versión recomendada.
3. Selecciona **Create**.

Para configurar la Action de Post Login:

1. Elimina el código existente del editor de código.
2. Pega el código para renderizar el formulario en el editor de código.
3. Edita el código para definir la lógica condicional que renderizará el formulario.

   ```javascript lines
   /**
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del login.
   */
   exports.onExecutePostLogin = async (event, api) => {
     const FORM_ID = 'REPLACE_WITH_YOUR_FORM_ID';

     if (
       !event.user.user_metadata.full_name &&
       !event.user.user_metadata.job_title
     ) {
       api.prompt.render(FORM_ID);
     }
   }

   exports.onContinuePostLogin = async (event, api) => { }
   ```

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     El código de ejemplo anterior renderizará el formulario cuando al usuario le falten las propiedades `full_name` y `job_title` de User Metadata.
   </Callout>
4. Selecciona **Deploy.**
5. Arrastra y suelta la Action **Render Additional Signup Form** en el flujo **Login**.

   <Frame>![Dashboard > Forms > Use Cases Custom Signup Steps Login Flow](/docs/images/cdy7uua7fh8z/1WBOicIOcH98hKj6Lq9HF2/4951e30a0cc08c7c5daa0116eaa17e11/Screenshot_2024-03-01_at_15.48.00.png)</Frame>
6. Selecciona **Apply**.

<div id="test-implementation">
  ### Probar la implementación
</div>

Prueba la implementación siguiendo estos pasos:

* Regístrate en una aplicación con un usuario nuevo.
* La Action de Post Login en el flujo Login renderizará el formulario y solicitará la información.
* Selecciona **<Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> &gt; User Management &gt; Users,** busca el usuario nuevo y verifica que sus atributos `full_name` y `job_title` contengan la información del paso adicional de registro.
