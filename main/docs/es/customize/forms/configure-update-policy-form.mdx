<Card title="Antes de empezar">
  1. Crea una [Machine-to-Machine Application](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps) con los siguientes scopes habilitados:

  * `read:users`
  * `update:users`
  * `create:users`
  * `read:users_app_metadata`
  * `update:users_app_metadata`
  * `create:users_app_metadata`

  2. Agrega una [Vault connection](/docs/es/customize/forms/vault) con las credenciales de la aplicación de máquina a máquina.
</Card>

formulario para Actions te permite crear flujos personalizados de políticas de privacidad y almacenar el consentimiento en el objeto `app_metadata`.

En las secciones siguientes se explica cómo crear un formulario para actualizar políticas usando nodos y flujos, junto con los pasos para agregar tu formulario a una Action de [Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="create-a-form-from-scratch">
  ## Crear un formulario desde cero
</div>

Sigue estos pasos para crear un formulario y recopilar información:

1. Abre el editor de formularios seleccionando **Auth0 Dashboard &gt; Actions &gt; formulario.**
2. Selecciona **Create Form** &gt; **Start from scratch**.

De forma predeterminada, un formulario nuevo contiene un nodo Start, un nodo Step y un nodo Ending screen.

<Frame>![Dashboard > formulario > Casos de uso > formulario](/docs/images/cdy7uua7fh8z/2TZ9jSjgEcARFpULI6hz4R/a4319a1bd482f2ee5cc23100fdb2f42c/New-form-Forms-for-action-use-case.png)</Frame>

<div id="configure-the-step-node">
  ### Configurar el nodo Step
</div>

El nodo Step es la interfaz gráfica que ven los usuarios. Agrega [campos](/docs/es/customize/forms/nodes-and-components) al nodo Step para recopilar el nuevo valor de aceptación del acuerdo siguiendo estos pasos:

1. Arrastra un campo **Rich text** desde el **menú Components** hasta el nodo **Step**.

   * **Rich text**: Ingresa una descripción de las actualizaciones de tus políticas de privacidad.
   * Selecciona **Save**.
2. Arrastra un campo **Legal** hasta el nodo **Step**.

   * **ID**: Ingresa `privacy_policies`.
   * **Required**: Habilita la casilla.
   * **Legal**: Ingresa `I agree to the Privacy Policy`.
   * Selecciona **Save**.
3. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > formulario > nodo Step del caso de uso de la política de privacidad](/docs/images/cdy7uua7fh8z/59zFFjQjAnjogkc2QGIqqJ/0ad66b1b636544122aa4e65968b76c2c/privacy-policy-step-node.png)</Frame>

<div id="configure-the-flow-node">
  ### Configura el nodo Flow
</div>

Agrega un nodo **Flow** después del nodo **Step** para actualizar `app_metadata` y reanudar el flujo de autenticación siguiendo estos pasos:

1. Selecciona **Flow** en la parte inferior del editor de formularios.
2. Elimina el vínculo existente entre los nodos **Step** y **Ending Screen**.
3. Selecciona el nuevo **Flow** &gt; **Click to add a flow** &gt; **Create a new flow.**

   * Ingresa `Update app_metadata` en el campo **Name**.
   * Selecciona **Create**.
   * Selecciona **Save**.
4. Vincula el nodo **Flow** con los nodos **Step** y **Ending Screen**, como se muestra a continuación.
5. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > formulario > Form](/docs/images/cdy7uua7fh8z/6YIkzm7SrpM9LW76AXuRzh/353c518fe82bbaa440836da1d83596d6/link-policies.png)</Frame>

Agrega una Action de Auth0 Update user al flujo siguiendo estos pasos:

1. Selecciona **Flow** **Update app&#95;metadata** &gt; **Edit flow** para abrir el editor de flujos en una pestaña nueva.
2. Debajo de StartAction, selecciona el ícono **+** para agregar una Action de **Update user**. Completa los siguientes campos y luego selecciona **Save** para continuar.

   * **Connection**: Selecciona en el menú desplegable la Vault connection de tu aplicación M2M.

   * **User ID**: Ingresa `{{context.user.user_id}}`.

   * **Body**: Copia y pega el siguiente código para actualizar app&#95;metadata con las propiedades privacy&#95;policies y privacy&#95;policies&#95;timestamp.

     ```text lines
     {
       "app_metadata": {
         "privacy_policies": true,
         "privacy_policies_timestamp": "{{ functions.toTimestamp() }}"
       }
     }
     ```

   * Selecciona **Save**.
3. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > formulario > User case privacy policy code](/docs/images/cdy7uua7fh8z/CvMNIwWrOUwaFW4X3G28p/1fb70ae57a20749d9ffaaba8dc0b7e26/update-user-privacy.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El flujo **Update app&#95;metadata** establece `privacy_policies` en `true` dentro del objeto app`_metadata` y también genera una marca de tiempo con una [función auxiliar.](/docs/es/customize/forms/variables)
</Callout>

<div id="retrieve-the-form-render-code">
  ## Obtén el código para renderizar el formulario
</div>

Obtén el código para renderizar tu formulario y mostrarlo visualmente con una Action de Post Login personalizada siguiendo estos pasos:

* En el editor de formularios, selecciona **`<> Render`**.
* Selecciona **Copiar**.

<Frame>![Dashboard > Actions > formulario > Form > Embed](/docs/images/cdy7uua7fh8z/2EIUzc539sq9lwoTKbjoJv/496a864c0fc44a782c9cadca119dbfb0/embed.png)</Frame>

<div id="create-a-post-login-action">
  ## Crear una Action de Post Login
</div>

Crea una Action de Post Login para renderizar tu formulario siguiendo estos pasos:

1. Ve a **Auth0 Dashboard &gt; Actions &gt; Flows &gt; Login.**
2. Selecciona el ícono **+** y luego **Build from scratch**:

   * **Name**: Ingresa `Render Update Policy Form`.
   * **Trigger**: Selecciona `Login / Post Login`.
   * **Runtime**: Selecciona la versión recomendada.
3. Selecciona **Create**.

Para configurar la Action de Post Login:

1. Elimina el código existente del editor de código.
2. Pega el código para renderizar el formulario en el editor de código.
3. Edita el código para definir la lógica condicional que renderizará el formulario.

   ```javascript lines
   /**
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que inicia sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
   */
   exports.onExecutePostLogin = async (event, api) => {
     const FORM_ID = 'REPLACE_WITH_YOUR_FORM_ID';

     if (event.user.app_metadata.privacy_policies !== true) {
       api.prompt.render(FORM_ID);
     }
   }

   exports.onContinuePostLogin = async (event, api) => { }
   ```

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     El código de ejemplo anterior renderizará el formulario cuando al usuario le falte la propiedad `privacy_policies` en app metadata.
   </Callout>
4. Selecciona **Deploy.**
5. Arrastra y suelta la Action **Render Update Policy Form** en el flujo **Login**.

   <Frame>![Dashboard > Forms > Use Case > Render update policy form login action](/docs/images/cdy7uua7fh8z/5TOwDgevZsBqd7yZynUYWU/41ca909dc3007aed4f40ff794a05efa5/postlogin.png)</Frame>
6. Selecciona **Apply.**

<div id="test-implementation">
  ### Probar la implementación
</div>

Pruebe la implementación siguiendo estos pasos:

1. Inicie sesión con un usuario existente cuyo atributo `app_metadata` `privacy_policies` no tenga ningún valor.
2. La Action de Post Login en el flujo de Login renderizará el formulario y solicitará su aceptación.
3. Seleccione **Auth0 Dashboard &gt; User Management &gt; Users,** busque al usuario y verifique que el atributo `app_metadata` `privacy_policies` contenga un valor `true` y una marca de tiempo.
