<Card title="Antes de empezar">
  1. Crea una [Machine-to-Machine Application](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps) con los siguientes scopes habilitados:

  * `read:users`
  * `update:users`
  * `create:users`
  * `read:users_app_metadata`
  * `update:users_app_metadata`
  * `create:users_app_metadata`

  2. Agrega una [conexión de Vault](/docs/es/customize/forms/vault) con las credenciales de la aplicación M2M.
</Card>

Forms for Actions te permite crear flujos personalizados de políticas de privacidad y almacenar el consentimiento en el objeto `app_metadata`.

<Frame>![Dashboard > Actions > Forms > Caso de uso de la política de privacidad](/docs/images/cdy7uua7fh8z/6i68dMvHDTijfRmSgN1x8Q/ad900e7d4c0fb34645921435472b8684/privavy-policy-form.png)</Frame>

En las secciones siguientes se explica cómo crear un formulario para actualizar políticas mediante nodos y flujos, junto con los pasos para agregarlo a una Action de [Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="create-a-form-from-scratch">
  ## Crear un formulario desde cero
</div>

Sigue estos pasos para crear un formulario y recopilar información:

1. Abre el editor de formularios seleccionando **Auth0 Dashboard &gt; Actions &gt; Forms.**
2. Selecciona **Create Form** &gt; **Start from scratch**.

De forma predeterminada, un formulario nuevo contiene un nodo Start, un nodo Step y un nodo Ending screen.

<Frame>![Dashboard > Forms > Use Cases > Form](/docs/images/cdy7uua7fh8z/2TZ9jSjgEcARFpULI6hz4R/a4319a1bd482f2ee5cc23100fdb2f42c/New-form-Forms-for-action-use-case.png)</Frame>

<div id="configure-the-step-node">
  ### Configura el nodo Step
</div>

El nodo Step es la interfaz gráfica que ven los usuarios. Agrega [Fields](/docs/es/customize/forms/nodes-and-components) al nodo Step para recopilar el nuevo valor de aceptación del acuerdo siguiendo estos pasos:

1. Arrastra un campo de **Rich text** desde el **menú Components** hasta el nodo **Step**.

   * **Rich text**: Introduce una descripción de las actualizaciones de tus políticas de privacidad.
   * Selecciona **Save**.
2. Arrastra un campo **Legal** hasta el nodo **Step**.

   * **ID**: Introduce `privacy_policies`.
   * **Required**: Marca la casilla.
   * **Legal**: Introduce `I agree to the Privacy Policy`.
   * Selecciona **Save**.
3. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > nodo Step del caso de uso de la política de privacidad](/docs/images/cdy7uua7fh8z/59zFFjQjAnjogkc2QGIqqJ/0ad66b1b636544122aa4e65968b76c2c/privacy-policy-step-node.png)</Frame>

<div id="configure-the-flow-node">
  ### Configura el nodo Flow
</div>

Agrega un nodo **Flow** después del nodo **Step** para actualizar `app_metadata` y reanudar el flujo de autenticación siguiendo estos pasos:

1. Selecciona **Flow** en la parte inferior del editor de formularios.
2. Elimina el enlace existente entre los nodos **Step** y **Ending Screen**.
3. Selecciona el nuevo **Flow** &gt; **Haz clic para agregar un flujo** &gt; **Crear un flujo nuevo**.

   * Ingresa `Update app_metadata` en el campo **Name**.
   * Selecciona **Create**.
   * Selecciona **Save**.
4. Vincula el nodo **Flow** a los nodos **Step** y **Ending Screen**, como se muestra a continuación.
5. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/6YIkzm7SrpM9LW76AXuRzh/353c518fe82bbaa440836da1d83596d6/link-policies.png)</Frame>

Agrega una acción Auth0 Update user al flujo siguiendo estos pasos:

1. Selecciona **Flow** **Update app&#95;metadata** &gt; **Edit flow** para abrir el editor de Flow en una pestaña nueva.
2. Debajo de StartAction, selecciona el ícono **+** para agregar una acción **Update user**. Completa los siguientes campos y luego selecciona **Save** para continuar.

   * **Connection**: En el menú desplegable, selecciona la conexión de Vault para tu aplicación M2M.

   * **User ID**: Ingresa `{{context.user.user_id}}`.

   * **Body**: Copia y pega el siguiente código para actualizar `app_metadata` con las propiedades `privacy_policies` y `privacy_policies_timestamp`.

     ```text lines
     {
       "app_metadata": {
         "privacy_policies": true,
         "privacy_policies_timestamp": "{{ functions.toTimestamp() }}"
       }
     }
     ```

   * Selecciona **Save**.
3. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > User case privacy policy code](/docs/images/cdy7uua7fh8z/CvMNIwWrOUwaFW4X3G28p/1fb70ae57a20749d9ffaaba8dc0b7e26/update-user-privacy.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El flujo **Update app&#95;metadata** agrega el valor `privacy_policies true` al objeto `app_metadata` y también genera una marca de tiempo con una [función auxiliar](/docs/es/customize/forms/variables).
</Callout>

<div id="retrieve-the-form-render-code">
  ## Obtén el código de renderizado del formulario
</div>

Obtén el código de renderizado de tu formulario para mostrarlo visualmente con una Action personalizada de Post Login siguiendo estos pasos:

* En el editor de formularios, selecciona **`<> Render`**.
* Selecciona **Copy**.

<Frame>![Dashboard > Actions > Forms > Form > Embed](/docs/images/cdy7uua7fh8z/2EIUzc539sq9lwoTKbjoJv/496a864c0fc44a782c9cadca119dbfb0/embed.png)</Frame>

<div id="create-a-post-login-action">
  ## Crear una Action Post Login
</div>

Crea una Action Post Login para mostrar tu formulario siguiendo estos pasos:

1. Ve a **Auth0 Dashboard &gt; Actions &gt; Flows &gt; Login.**
2. Selecciona el ícono **+** y luego **Build from scratch**:

   * **Name**: Ingresa `Render Update Policy Form`.
   * **Trigger**: Selecciona `Login / Post Login`.
   * **Runtime**: Selecciona la versión recomendada.
3. Selecciona **Create**.

Para configurar la Action Post Login:

1. Elimina el código existente del editor de código.
2. Pega el código de renderizado del formulario en el editor de código.
3. Edita el código para definir la lógica condicional que mostrará el formulario.

   ```javascript lines
   /**
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
   */
   exports.onExecutePostLogin = async (event, api) => {
     const FORM_ID = 'REPLACE_WITH_YOUR_FORM_ID';

     if (event.user.app_metadata.privacy_policies !== true) {
       api.prompt.render(FORM_ID);
     }
   }

   exports.onContinuePostLogin = async (event, api) => { }
   ```

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     El código de ejemplo anterior mostrará el formulario cuando al usuario le falte la propiedad `privacy_policies` en los metadatos de la aplicación.
   </Callout>
4. Selecciona **Deploy.**
5. Arrastra y suelta la Action **Render Update Policy Form** en el flujo **Login**.

   <Frame>![Dashboard > Formularios > Caso de uso > Action de login para mostrar el formulario de actualización de política](/docs/images/cdy7uua7fh8z/5TOwDgevZsBqd7yZynUYWU/41ca909dc3007aed4f40ff794a05efa5/postlogin.png)</Frame>
6. Selecciona **Apply.**

<div id="test-implementation">
  ### Probar la implementación
</div>

Prueba la implementación siguiendo estos pasos:

1. Inicia sesión con un usuario existente cuyo atributo `privacy_policies` de `app_metadata` no tenga ningún valor.
2. La Action de Post Login en el flujo de Login mostrará el formulario y solicitará su aceptación.
3. Selecciona **Auth0 Dashboard &gt; User Management &gt; Users,** localiza al usuario y verifica que el atributo `privacy_policies` de `app_metadata` contenga un valor `true` y una marca de tiempo.
