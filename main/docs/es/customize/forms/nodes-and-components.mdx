Formularios te permite agregar nodos y componentes, como campos, bloques y widgets, para que tus usuarios interactúen con el formulario. Luego, puedes hacer referencia en tus formularios y flujos a los datos introducidos en los componentes.

<div id="form-nodes">
  ## Nodos de formulario
</div>

Hay cinco tipos de nodos de formulario: Inicio, Paso, Flujo, Router y pantalla final. Estos nodos cumplen una función muy específica dentro de la estructura del formulario.

<Frame>![Dashboard >  Formularios > Editor de formularios](/docs/images/cdy7uua7fh8z/5V4GsHxxC2cjxx1aODCxPd/309850c72bbce224e7a8d3f5af34114c/form-editor-preview.png)</Frame>

<div id="step-node">
  ## nodo de paso
</div>

Los nodos de paso son la representación visual del formulario. Pueden contener cualquier número de componentes, como campos, bloques y widgets, en los que el usuario puede completar sus datos.

<Frame>![Dashboard > Foms > nodo de paso](/docs/images/cdy7uua7fh8z/2tbkcXkVon0BawCvYgnvIz/e2ce566320af18a6b08702f04f763126/form-step-node.png)</Frame>

<div id="field-components">
  ### Componentes de campo
</div>

Los campos son componentes de la interfaz de usuario en los que los usuarios ingresan su información en Formularios.

<Frame>![Dashboard > Formularios > Campos](/docs/images/cdy7uua7fh8z/6fuBdJcbDybzcVaiByUkoQ/2ad29c8dbbcc6ddf850f5efc004a75e0/forms-fields.png)</Frame>

<div id="field-settings">
  #### Configuración de los campos
</div>

Los campos tienen varios ajustes predeterminados que puedes configurar según tus necesidades.

Los ajustes predeterminados de los campos son:

| Configuración                  | Descripción                                  |
| ------------------------------ | -------------------------------------------- |
| ID                             | El id único                                  |
| Etiqueta                       | Mostrar u ocultar una etiqueta               |
| Obligatorio                    | El valor del campo es obligatorio            |
| Texto de ayuda                 | Mostrar un texto de ayuda                    |
| Texto del marcador de posición | Texto visible para el usuario                |
| Valor predeterminado           | Valor predeterminado visible para el usuario |
| Transitorio                    | Habilitar o deshabilitar la máscara de datos |

<div id="field-validation-rules">
  #### Reglas de validación de campos
</div>

| Configuración                 | Descripción                                     |
| ----------------------------- | ----------------------------------------------- |
| Longitud mín. / Longitud máx. | Requiere una longitud para el valor de entrada. |
| Valor mín. / Valor máx.       | Requiere un valor numérico.                     |

<div id="available-fields-in-forms">
  #### Campos disponibles en Formularios
</div>

Los campos disponibles, junto con su configuración específica y sus tipos de datos de salida, son:

<AccordionGroup>
  <Accordion title="Campo de texto">
    La entrada estándar permite al usuario introducir cualquier valor de texto.

    ##### Configuración del campo de texto

    La configuración del campo de texto es:

    | Configuración | Descripción                                   |
    | ------------- | --------------------------------------------- |
    | Multiline     | Habilita la introducción de texto multilínea. |

    ##### Valor de salida del campo de texto

    El tipo de dato del valor de salida del campo de texto es `string`.

    ```json lines
    {
      "text_field_id": "Auth0"
    }
    ```
  </Accordion>

  <Accordion title="Campo de correo electrónico">
    La entrada estándar permite que un usuario introduzca valores de correo electrónico como cadena.

    ##### Valor de salida del campo de correo electrónico

    El tipo de dato del valor de salida del campo de correo electrónico es `string`.

    ```json lines
    {
      "email_field_id": "username@domain.com"
    }
    ```
  </Accordion>

  <Accordion title="Campo de número de teléfono">
    La entrada estándar permite que un usuario introduzca valores de número de teléfono.

    ##### Configuración del campo de teléfono

    La configuración del campo de teléfono es:

    | Configuración                             | Descripción                                                                                                    |
    | ----------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
    | Selector de país                          | Activa o desactiva el selector de país. De forma predeterminada, muestra la ubicación del usuario según su IP. |
    | Texto del marcador de posición del filtro | Texto visible para el usuario en la ventana de búsqueda de países.                                             |

    ##### Valor de salida del campo de teléfono

    El tipo de datos del valor de salida del campo de teléfono es `string` o, si el selector de país está activado, un `object` .

    El valor de salida del campo de teléfono como `string`:

    ```json lines
    {
      "phone_field_id": "8005550175"
    }
    ```

    El valor del tipo de dato de salida del campo phone es un `object`:

    ```json lines
    {
      "phone_field_id": {
        "national_number": "8005550175",
        "national_format": "(800) 555-0175",
        "international_number": "+18005550175",
        "international_format": "+1 800-555-0175",
        "country_code_iso": "US",
        "country_code_number": "1"
      }
    }
    ```
  </Accordion>

  <Accordion title="Campo de selección">
    Campo estándar que permite al usuario seleccionar uno o varios valores.

    ##### Configuración del campo de selección

    La configuración del campo de selección es la siguiente:

    | Settings                       | Description                                                                   |
    | ------------------------------ | ----------------------------------------------------------------------------- |
    | Opción múltiple                | Habilita varias opciones de selección.                                        |
    | Otra opción                    | Permite a los usuarios ingresar su propia opción.                             |
    | Advanced &gt; Valores internos | Asigna un valor interno a cada opción de selección.                           |
    | Advanced &gt; Edición masiva   | Edita en bloque los valores de label y de las opciones de selección internas. |

    ##### Valor de salida del campo de selección

    El tipo de datos del valor de salida del campo de selección es `string` o, si la opción múltiple está habilitada, un `arreglo de strings` .

    El valor de salida del campo de selección como `string`:

    ```json lines
    {
      "choice_field_id": "Option A"
    }
    ```

    El valor del tipo de dato de salida del campo de selección es un `array of strings`:

    ```json lines
    {
      "choice_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Campo de tarjetas">
    Entrada estándar que permite a un usuario seleccionar uno o varios valores de imagen.

    ##### Configuración del campo de tarjetas

    Los ajustes del campo de tarjetas son:

    | Configuración                  | Descripción                               |
    | ------------------------------ | ----------------------------------------- |
    | Selección múltiple             | Habilita varias opciones de selección.    |
    | Ocultar etiquetas              | Oculta las etiquetas de las tarjetas.     |
    | Advanced &gt; Valores internos | Asigna un valor interno para cada opción. |

    ##### Valor de salida del campo de tarjetas

    El tipo de dato del valor de salida del campo de tarjetas es `string` o, si la selección múltiple está habilitada, un `arreglo de strings`.

    El valor de salida del campo de tarjetas con tipo de dato `string`:

    ```json lines
    {
      "card_field_id": "Option A"
    }
    ```

    El campo cards devuelve un valor del tipo de dato `array of strings`:

    ```json lines
    {
      "card_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Campo URL">
    La entrada estándar permite introducir un valor de URL.

    ##### Valor de salida del campo URL

    El tipo de dato del valor de salida del campo URL es `string`:

    ```json lines
    {
      "url_field_id": "https://auth0.com"
    }
    ```
  </Accordion>

  <Accordion title="Campo de contraseña">
    Campo de entrada estándar que permite al usuario introducir una contraseña o un valor secreto.

    ##### Configuración del campo de contraseña

    La configuración del campo de contraseña es la siguiente:

    | Settings                     | Descripción                                                                                                                                                                                                                                   |
    | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Requerir contraseña compleja | Las contraseñas deben contener una mayúscula, una minúscula, un símbolo y un número.                                                                                                                                                          |
    | Aplicar las directrices NIST | La contraseña debe tener al menos 8 caracteres y no puede proceder de filtraciones de datos anteriores, ni ser una palabra de diccionario, contener caracteres repetitivos o secuenciales, ni incluir palabras relacionadas con el contenido. |
    | Añadir medidor de fortaleza  | Debajo del campo de contraseña se muestra un medidor gráfico con una escala roja, amarilla y verde para indicar la fortaleza de la contraseña introducida.                                                                                    |
    | Hash                         | Seleccione un algoritmo para calcular el hash del valor introducido directamente en el navegador.                                                                                                                                             |

    ##### Valor de salida del campo de contraseña

    El tipo de dato del valor de salida del campo de contraseña es un `string` enmascarado:

    ```json lines
    {
      "password_field_id": "███"
    }
    ```
  </Accordion>

  <Accordion title="Campo de pago">
    Campo estándar que permite a un usuario introducir información de pago mediante un proveedor de pagos. Los pagos se procesan antes del nodo Ending.

    ##### Configuración del campo de pago

    La configuración del campo de pago es la siguiente:

    | Settings                  | Description                                                                                                                       |
    | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
    | Tipo de pago              | Seleccione pagos únicos (Charge) o recurrentes (Subscription).                                                                    |
    | Importe                   | Especifique el importe de un pago único (Charge).                                                                                 |
    | Moneda                    | Especifique la moneda de un pago único (Charge).                                                                                  |
    | Acción de suscripción     | Seleccione si desea crear o actualizar una suscripción existente.                                                                 |
    | Acción del cliente        | Seleccione si desea crear o actualizar un cliente existente.                                                                      |
    | Opciones de visualización | Muestre y edite las etiquetas de Número de tarjeta, Fecha de vencimiento y Código de seguridad, así como las marcas de confianza. |

    ##### Valor de salida del campo de pago

    El tipo de dato del valor de salida del campo de pago es un string o un objeto para los flujos posteriores al envío.

    El tipo de dato del valor de salida del campo de pago como `string`:

    ```json lines
    {
      "payment_field_id": "pm_1P19e..."
    }
    ```

    El tipo de dato del valor de salida del campo de pago de las suscripciones es un `object`:

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "price_ids": [
          "price_1ONHR..."
        ],
        "subscription_id": "sub_1P1A...",
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```

    El tipo de dato del valor de salida del campo de pago para charge como `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "amount": 100,
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```
  </Accordion>

  <Accordion title="Campo de aviso legal">


  <Accordion title="Campo de lista desplegable">


  <Accordion title="Campo de fecha y hora">

    | Formato | Seleccione el formato Fecha o Hora. |

<div id="blocks-components">
  ### Componentes de Blocks
</div>

Los Blocks son componentes de la interfaz de usuario que añaden funcionalidad a tus formularios, pero no recopilan información de los usuarios.

<Frame>![Dashboard > Forms > Components > Blocks](/docs/images/cdy7uua7fh8z/7IijftJnuwcevL4bdHiaKw/f0376ca75303bb58d0df87e6af75e347/forms-block.png)</Frame>

<div id="available-blocks-in-forms">
  #### Bloques disponibles en formularios
</div>

Los bloques disponibles son:

<AccordionGroup>
  <Accordion title="Bloque de botón Siguiente">
    Un botón que permite a los usuarios continuar al siguiente nodo del formulario.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      * Antes de continuar, las reglas de validación de campos se aplican y se validan tanto del lado del cliente como del lado del servidor. Si estas reglas fallan, se muestra un mensaje de error para que el usuario lo revise.
      * Si el siguiente nodo es un nodo de flujo, se muestra un indicador de carga hasta que el flujo se complete. Si el flujo falla o devuelve un mensaje de error personalizado, se muestra un mensaje de error.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Anterior">
    Un botón que permite a los usuarios volver al nodo de paso anterior.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un usuario selecciona el botón anterior, las reglas de validación de campos se vuelven a ejecutar aunque no se haya realizado ningún cambio en los datos introducidos antes de continuar.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Saltar">
    Un botón que permite a los usuarios omitir el paso actual y saltar directamente al nodo de destino.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Al omitir el paso actual, no se validará ni se recopilará ninguna información introducida en él.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Reenviar">
    Un componente que permite a los usuarios volver a ejecutar un flujo para generar y enviar un nuevo código OTP

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un usuario selecciona el botón de reenviar, el flujo asociado generará y enviará un nuevo código OTP.
    </Callout>

    | Configuración        | Descripción                                                                                                                                                                                                                                                                                                                                                   |
    | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Alineación del texto | Seleccione la alineación del texto: izquierda, centro o derecha                                                                                                                                                                                                                                                                                               |
    | Texto                | Texto predeterminado que se muestra cuando los usuarios seleccionan el botón de reenviar                                                                                                                                                                                                                                                                      |
    | Texto del botón      | Texto del botón                                                                                                                                                                                                                                                                                                                                               |
    | Texto de espera      | Texto que se muestra cuando los usuarios han seleccionado el botón de reenviar. Reemplaza la configuración `Text` y `Button text`. Use la variable `“{{remaining_seconds}}”` para mostrar dinámicamente la cantidad de segundos restantes hasta que el botón de envío vuelva a estar habilitado. Por ejemplo: “Reenviar en `{{remaining_seconds}}` segundos”. |
    | Flujo                | Flujo que se ejecuta después de que el usuario selecciona el botón de reenviar.                                                                                                                                                                                                                                                                               |
    | Intentos máximos     | Número máximo de veces que el usuario puede seleccionar el botón de reenviar                                                                                                                                                                                                                                                                                  |
    | Tiempo de espera     | El tiempo de espera entre intentos, en segundos.                                                                                                                                                                                                                                                                                                              |
  </Accordion>

  <Accordion title="Bloque de texto enriquecido">
    Un bloque de texto enriquecido para personalizar el nodo de paso con información adicional.
  </Accordion>

  <Accordion title="Bloque divisor">
    Una línea para dividir distintas secciones del nodo de paso. Puede contener un texto breve.
  </Accordion>

  <Accordion title="Bloque HTML">
    Un bloque HTML para crear una UI personalizada.
  </Accordion>

  <Accordion title="Bloque de imagen">
    Un bloque de imagen para personalizar el paso añadiendo imágenes.
  </Accordion>
</AccordionGroup>

<div id="available-widgets-in-forms">
  #### Widgets disponibles en formulario
</div>

El widget disponible es:

<AccordionGroup>
  <Accordion title="Widget de direcciones de Google">
    Un widget que permite a un usuario introducir y validar su dirección.

    ##### Configuración del widget de direcciones de Google

    La configuración del widget de direcciones de Google incluye:

    | Configuración | Descripción                                                               |

    ```json lines
    {
      "google_address_widget_id": {
        "type": "Feature",
        "geometry": {
          "type": "Point",
          "coordinates": [
            -73.9654415,
            40.8054491
          ]
        },
        "properties": {
          "geocoding": {
            "type": "house",
            "housenumber": "2880",
            "street": "Broadway",
            "city": "New York",
    ```

  </Accordion>

  <Accordion title="Google reCAPTCHA">
    Un widget que ayuda a proteger su sitio web frente al spam y el abuso al verificar que un usuario es una persona y no un bot.

    El widget admite:

    * Basado en puntuación (v3)
    * Desafío (v2)

      * Insignia invisible de reCAPTCHA

    Diferencias entre `v2` y `v3`:

    * `v2` requiere la interacción del usuario con desafíos visibles, como hacer clic en una casilla de verificación o resolver rompecabezas.
    * `v3` se ejecuta en segundo plano y proporciona una puntuación basada en el comportamiento del usuario, sin requerir interacción. Al usar esta versión, asegúrese de implementar lógica de negocio adicional para gestionar la puntuación y determinar si es necesaria alguna otra acción.

    ##### Configuración del widget de Google reCAPTCHA

    La configuración del widget de Google reCAPTCHA incluye:

    | Configuración   | Descripción                                                                                                                                                                                                                                                   |
    | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Clave del sitio | La clave pública del sitio que se utiliza para inicializar reCAPTCHA en su sitio web. Puede crearla en la [consola de Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) o en su proyecto de Google Cloud Platform.                             |
    | Clave secreta   | La clave secreta que se utiliza para comunicarse de forma segura con el servicio reCAPTCHA del lado del servidor. Puede crearla en la [consola de Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) o en su proyecto de Google Cloud Platform. |

    ##### Valor de salida de Google reCAPTCHA

    El tipo de dato del valor de salida del widget de Google reCAPTCHA es un `object`.

    Ejemplo de respuesta de `v2`:

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com"
      }
    }
    ```

    Ejemplo de respuesta de `v3`:

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com",
        "score": 0.9
      }
    }
    ```
  </Accordion>

  <Accordion title="Credenciales verificables">
    Un widget que permite a un usuario verificar su identidad mediante credenciales verificables almacenadas en su billetera digital.

    ##### Configuración del widget de credenciales verificables

    La configuración del widget de credenciales verificables incluye:

    | Configuración           | Descripción                                                                                                                                                                                                                              |
    | ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | URL                     | El valor de la URL que se utiliza para generar el código QR. Este valor se devuelve en la propiedad `engagement` de la solicitud de verificación.                                                                                        |
    | Texto del enlace        | Texto alternativo que se mostrará a los usuarios que no puedan escanear el código QR y prefieran abrir el enlace directamente en su dispositivo.                                                                                         |
    | Tamaño                  | El tamaño del código QR.                                                                                                                                                                                                                 |
    | Token público           | El token de acceso necesario para consumir el endpoint de polling. Asegúrese de que el token se genere solo con el scope `read:verification_request` para evitar exponer acceso a otros recursos.                                        |
    | ID de verificación      | El ID de verificación generado cuando inicia la solicitud de verificación.                                                                                                                                                               |
    | Tiempo máximo de espera | La cantidad máxima de tiempo que se debe esperar para que se complete el proceso de verificación. Una vez alcanzado el tiempo establecido, el polling se detendrá y se generará un error, independientemente del estado de verificación. |

    ##### Valor de salida de credenciales verificables

    El tipo de dato del valor de salida del widget de credenciales verificables es un `object`.

    ```json lines
    {
      "verifiable_credentials_widget_id": {
        "state": "honored",
        "reason": "...", // El campo reason contiene información adicional (si está disponible) sobre el estado de la solicitud de verificación. 
        "presentation": {} // La presentación contiene los claims proporcionados por la wallet en respuesta a la definición de presentación.
      }
    }
    ```
  </Accordion>
</AccordionGroup>

<div id="router-node">
  ## Nodo Router
</div>

Los nodos Router te permiten agregar reglas para crear saltos condicionales entre nodos.

De manera predeterminada, un nodo Router tiene una sola regla de paso directo llamada **Caso predeterminado** que se conecta con otros nodos. Puedes agregar reglas adicionales basadas en un conjunto de condiciones que una variable debe cumplir para conectarse después a otros nodos. Para obtener más información, consulta [Router.](/docs/es/customize/forms/routers)
