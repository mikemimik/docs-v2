formulario te permite añadir nodos y componentes, como campos, bloques y widgets, con los que tus usuarios pueden interactuar en el formulario. Después, puedes hacer referencia a los datos introducidos en los componentes dentro de tus formularios y flujos.

<div id="form-nodes">
  ## Nodos del formulario
</div>

Hay cinco tipos de nodos del formulario: Start, Paso, Flujo, Router y pantalla final. Estos nodos cumplen una función muy específica dentro de la estructura del formulario.

<Frame>![Dashboard >  Formularios > Editor de formularios](/docs/images/cdy7uua7fh8z/5V4GsHxxC2cjxx1aODCxPd/309850c72bbce224e7a8d3f5af34114c/form-editor-preview.png)</Frame>

<div id="step-node">
  ## Nodo de paso
</div>

Los nodos de paso son la representación visual del formulario. Pueden contener cualquier cantidad de componentes, como campos, bloques y widgets, en los que el usuario puede introducir sus datos.

<Frame>![Dashboard > Foms > Nodo de paso](/docs/images/cdy7uua7fh8z/2tbkcXkVon0BawCvYgnvIz/e2ce566320af18a6b08702f04f763126/form-step-node.png)</Frame>

<div id="field-components">
  ### Componentes de campo
</div>

Los campos son componentes de la UI en los que los usuarios introducen su información en Formularios.

<Frame>![Dashboard > Forms > Fields](/docs/images/cdy7uua7fh8z/6fuBdJcbDybzcVaiByUkoQ/2ad29c8dbbcc6ddf850f5efc004a75e0/forms-fields.png)</Frame>

<div id="field-settings">
  #### Configuración del campo
</div>

Los campos tienen varios ajustes predeterminados que puedes configurar según tus necesidades.

Los ajustes predeterminados de los campos son:

| Configuración                 | Descripción                                    |
| ----------------------------- | ---------------------------------------------- |
| ID                            | El identificador único                         |
| Etiqueta                      | Muestra u oculta una etiqueta                  |
| Obligatorio                   | El valor de entrada es obligatorio             |
| Texto de ayuda                | Muestra un texto de ayuda                      |
| Texto de marcador de posición | Texto visible para el usuario                  |
| Valor predeterminado          | Valor predeterminado visible para el usuario   |
| Transitorio                   | Activa o desactiva el enmascaramiento de datos |

<div id="field-validation-rules">
  #### Reglas de validación de campos
</div>

| Configuración                 | Descripción                                                     |
| ----------------------------- | --------------------------------------------------------------- |
| Longitud mín. / Longitud máx. | Requiere una longitud mínima o máxima para el valor de entrada. |
| Valor mín. / Valor máx.       | Requiere un valor numérico mínimo o máximo.                     |

<div id="available-fields-in-forms">
  #### Campos disponibles en Formularios
</div>

Los campos disponibles, junto con su configuración específica y sus tipos de salida de datos, son:

<AccordionGroup>
  <Accordion title="Campo de texto">
    La entrada estándar permite al usuario introducir cualquier valor de texto.

    ##### Configuración del campo de texto

    La configuración del campo de texto es:

    | Configuración | Descripción                                |
    | ------------- | ------------------------------------------ |
    | Multiline     | Permite introducir texto en varias líneas. |

    ##### Valor de salida del campo de texto

    El tipo de dato del valor de salida del campo de texto es `string`.

    ```json lines
    {
      "text_field_id": "Auth0"
    }
    ```
  </Accordion>

  <Accordion title="Campo de correo electrónico">
    La entrada estándar permite que un usuario introduzca valores de correo electrónico de tipo cadena.

    ##### Valor de salida del campo Correo electrónico

    El tipo de dato del valor de salida del campo Correo electrónico es un `string`.

    ```json lines
    {
      "email_field_id": "username@domain.com"
    }
    ```
  </Accordion>

  <Accordion title="Campo de teléfono">
    La entrada estándar permite al usuario introducir valores de número de teléfono.

    ##### Configuración del campo phone

    La configuración del campo phone es la siguiente:

    | Configuración                            | Descripción                                                                                                    |
    | ---------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
    | Selector de país                         | Activa o desactiva el selector de país. De forma predeterminada, muestra la ubicación del usuario según su IP. |
    | Texto de marcador de posición del filtro | Texto visible para el usuario en la ventana de búsqueda de país.                                               |

    ##### Valor de salida del campo phone

    El tipo de dato del valor de salida del campo phone es `string` o, si el selector de país está activado, un `object` .

    El valor de salida del campo phone como `string`:

    ```json lines
    {
      "phone_field_id": "8005550175"
    }
    ```

    El valor del tipo de dato de salida del campo phone es un `object`:

    ```json lines
    {
      "phone_field_id": {
        "national_number": "8005550175",
        "national_format": "(800) 555-0175",
        "international_number": "+18005550175",
        "international_format": "+1 800-555-0175",
        "country_code_iso": "US",
        "country_code_number": "1"
      }
    }
    ```
  </Accordion>

  <Accordion title="Campo de selección">
    Campo de entrada estándar que permite a un usuario seleccionar valores de una sola opción o de varias opciones.

    ##### Configuración del campo de selección

    La configuración del campo de selección es la siguiente:

    | Settings                       | Description                                                                            |
    | ------------------------------ | -------------------------------------------------------------------------------------- |
    | Selección múltiple             | Habilita varias opciones de selección.                                                 |
    | Otra opción                    | Permite a los usuarios introducir su propia opción.                                    |
    | Advanced &gt; Valores internos | Asigna un valor interno a cada opción de selección.                                    |
    | Advanced &gt; Edición masiva   | Permite editar en bloque el label y los valores internos de las opciones de selección. |

    ##### Valor de salida del campo de selección

    El tipo de dato del valor de salida del campo de selección es `string` o, si la selección múltiple está habilitada, `array of strings` .

    El valor de salida del campo de selección con el tipo de dato `string`:

    ```json lines
    {
      "choice_field_id": "Option A"
    }
    ```

    El valor del tipo de dato de salida del campo de selección es un  `array of strings`:

    ```json lines
    {
      "choice_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Campo de tarjetas">
    Entrada estándar que permite al usuario seleccionar uno o varios valores de imagen.

    ##### Configuración del campo de tarjetas

    La configuración del campo de tarjetas es la siguiente:

    | Configuración                  | Descripción                            |
    | ------------------------------ | -------------------------------------- |
    | Selección múltiple             | Habilita varias opciones de selección. |
    | Ocultar etiquetas              | Oculta las etiquetas de las tarjetas.  |
    | Avanzado &gt; Valores internos | Asigna un valor interno a cada opción. |

    ##### Valor de salida del campo de tarjetas

    El tipo de dato del valor de salida del campo de tarjetas es `string` o, si la selección múltiple está habilitada, un `arreglo de strings`.

    El valor de salida del campo de tarjetas como `string`:

    ```json lines
    {
      "card_field_id": "Option A"
    }
    ```

    El campo de tarjetas devuelve un valor de salida del tipo de dato `array of strings`:

    ```json lines
    {
      "card_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Campo de URL">
    La entrada estándar permite introducir un valor de URL.

    ##### Valor de salida del campo URL

    El tipo de dato del valor de salida del campo URL es un String:

    ```json lines
    {
      "url_field_id": "https://auth0.com"
    }
    ```
  </Accordion>

  <Accordion title="Campo de contraseña">
    Entrada estándar que permite a un usuario introducir una contraseña o un valor secreto.

    ##### Configuración del campo de contraseña

    La configuración del campo de contraseña es la siguiente:

    | Configuración                    | Descripción                                                                                                                                                                                                                                        |
    | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Requerir una contraseña compleja | Las contraseñas deben contener una mayúscula, una minúscula, un símbolo y un número.                                                                                                                                                               |
    | Aplicar las directrices del NIST | La contraseña no puede tener menos de 8 caracteres ni ser una contraseña obtenida de filtraciones de datos anteriores, una palabra del diccionario, una secuencia de caracteres repetidos o consecutivos, ni una palabra específica del contenido. |
    | Añadir medidor de seguridad      | Se muestra un medidor gráfico con una escala roja, amarilla y verde debajo del campo de contraseña para indicar la seguridad de la contraseña introducida.                                                                                         |
    | Hash                             | Selecciona un algoritmo para aplicar hash al valor introducido directamente en el navegador.                                                                                                                                                       |

    ##### Valor de salida del campo de contraseña

    El tipo de dato del valor de salida del campo de contraseña es un `string` enmascarado:

    ```json lines
    {
      "password_field_id": "███"
    }
    ```
  </Accordion>

  <Accordion title="Campo de pago">
    Campo estándar que permite a un usuario introducir información de pago con un proveedor de pagos. Los pagos se procesan antes del nodo Ending.

    ##### Configuración del campo de pago

    La configuración del campo de pago es la siguiente:

    | Configuración             | Descripción                                                                                                                       |
    | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
    | Tipo de pago              | Seleccione pagos únicos (Charge) o recurrentes (Subscription).                                                                    |
    | Importe                   | Especifique el importe de un pago único (Charge).                                                                                 |
    | Moneda                    | Especifique la moneda de un pago único (Charge).                                                                                  |
    | Acción de suscripción     | Seleccione si desea crear o actualizar una suscripción existente.                                                                 |
    | Acción del cliente        | Seleccione si desea crear o actualizar un cliente existente.                                                                      |
    | Opciones de visualización | Muestre y edite las etiquetas de Número de tarjeta, Fecha de vencimiento y Código de seguridad, así como los sellos de confianza. |

    ##### Valor de salida del campo de pago

    El tipo de dato del valor de salida del campo de pago para los flujos posteriores al envío es una cadena o un objeto.

    El tipo de dato del valor de salida del campo de pago como `string`:

    ```json lines
    {
      "payment_field_id": "pm_1P19e..."
    }
    ```

    El tipo de dato del valor de salida del campo de pago de las suscripciones como `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "price_ids": [
          "price_1ONHR..."
        ],
        "subscription_id": "sub_1P1A...",
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```

    El tipo de dato del valor de salida del campo de pago para charge es un `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "amount": 100,
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```
  </Accordion>

  <Accordion title="Campo de aviso legal">
    Entrada estándar que agrega texto legal y una casilla de verificación.

    ##### Valor de salida del campo legal

    El tipo de dato del valor de salida del campo legal es `boolean`.

    ```json lines
    {
      "legal_field_id": true
    }
    ```
  </Accordion>

  <Accordion title="Campo de lista desplegable">
    Entrada estándar que permite a un usuario seleccionar uno o varios valores de una lista desplegable.

    ##### Configuración del campo desplegable

    | Advanced &gt; Valores internos | Asignar un valor interno para cada opción.    |

    ##### Valor de salida del campo desplegable

    El tipo de dato del valor de salida del campo desplegable es `string` o, si la selección múltiple está habilitada, un `arreglo de strings` .

    El valor del tipo de dato de salida del campo desplegable como `string`:

    ```json lines
    {
      "dropdown_field_id": "Option A"

    ```

    El campo desplegable devuelve un valor con el tipo de dato `array of strings`:

    ```json lines
    {
      "dropdown_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Campo numérico">
    Entrada estándar que permite al usuario introducir valores numéricos.

  <Accordion title="Campo de fecha y hora">

    | Clave   | Descripción                         |
    | ------- | ----------------------------------- |
    | Formato | Seleccione el formato Fecha o Hora. |

    ##### Valor de salida del campo de fecha / hora

  <Accordion title="Campo booleano">

<div id="blocks-components">
  ### Componentes de bloques
</div>

Los bloques son componentes de la interfaz de usuario que añaden funcionalidad a tus formularios, pero no recopilan información del usuario.

<Frame>![Dashboard > Formularios > Componentes > Blocks](/docs/images/cdy7uua7fh8z/7IijftJnuwcevL4bdHiaKw/f0376ca75303bb58d0df87e6af75e347/forms-block.png)</Frame>

<div id="available-blocks-in-forms">
  #### Bloques disponibles en Formularios
</div>

Los bloques disponibles son:

<AccordionGroup>
  <Accordion title="Bloque de botón Siguiente">
    Un botón que permite a los usuarios continuar al siguiente nodo del formulario.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      * Antes de continuar, las reglas de validación de campos se aplican y validan en el cliente y en el servidor. Si estas reglas fallan, se muestra un mensaje de error para que el usuario lo revise.
      * Si el siguiente nodo es un nodo de flujo, se muestra un indicador de carga hasta que el flujo se complete. Si el flujo falla o devuelve un mensaje de error personalizado, se muestra un mensaje de error.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Anterior">
    Un botón que permite a los usuarios ir al nodo de paso anterior.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un usuario selecciona el botón anterior, las reglas de validación de campos se vuelven a ejecutar aunque no se haya realizado ningún cambio en los datos de entrada antes de continuar.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Saltar">
    Un botón que permite a los usuarios omitir el paso actual y saltar directamente al nodo de destino.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Al omitir el paso actual, no se validará ni se recopilará ninguna información completada en el paso actual.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Reenviar">
    Un componente que permite a los usuarios volver a ejecutar un flujo para generar y enviar un nuevo código OTP

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un usuario selecciona el botón de reenviar, el flujo asociado generará y enviará un nuevo código OTP.
    </Callout>

    | Configuración        | Descripción                                                                                                                                                                                                                                                                                                                                                 |
    | -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Alineación del texto | Seleccione la alineación del texto: izquierda, centro o derecha                                                                                                                                                                                                                                                                                             |
    | Texto                | Texto predeterminado que se muestra cuando los usuarios seleccionan el botón de reenviar                                                                                                                                                                                                                                                                    |
    | Texto del botón      | Texto del botón                                                                                                                                                                                                                                                                                                                                             |
    | Texto de espera      | Texto que se muestra cuando los usuarios han seleccionado el botón de reenviar. Reemplaza la configuración de `Text` y `Button text`. Use la variable `“{{remaining_seconds}}”` para mostrar dinámicamente la cantidad de segundos restantes hasta que el botón de envío vuelva a habilitarse. Por ejemplo: “Reenviar en `{{remaining_seconds}}` segundos”. |
    | Flujo                | Flujo que se ejecuta después de que el usuario selecciona el botón de reenviar.                                                                                                                                                                                                                                                                             |
    | Intentos máximos     | La cantidad máxima de veces que el usuario puede seleccionar el botón de reenviar                                                                                                                                                                                                                                                                           |
    | Tiempo de espera     | El tiempo de espera entre intentos, en segundos.                                                                                                                                                                                                                                                                                                            |
  </Accordion>

  <Accordion title="Bloque de texto enriquecido">
    Un bloque de texto enriquecido para personalizar el nodo de paso con información adicional.
  </Accordion>

  <Accordion title="Bloque divisor">
    Una línea para dividir diferentes secciones del nodo de paso. Puede contener un texto breve.
  </Accordion>

  <Accordion title="Bloque HTML">
    Un bloque HTML para crear una UI personalizada.
  </Accordion>

  <Accordion title="Bloque de imagen">
    Un bloque de imagen para personalizar el paso agregando imágenes.
  </Accordion>
</AccordionGroup>

<div id="available-widgets-in-forms">

El widget disponible es:

<AccordionGroup>
  <Accordion title="Widget de dirección de Google">
    Un widget que permite a un usuario introducir y validar su dirección.

    La configuración del widget de dirección de Google incluye:

    | Configuración | Descripción                                                               |

          "coordinates": [
            -73.9654415,
            40.8054491
          ]

        "properties": {
          "geocoding": {

  <Accordion title="Google reCAPTCHA">
    Un widget que ayuda a proteger tu sitio web del spam y el abuso al verificar que un usuario es una persona y no un bot.

    El widget admite:

    * Basado en puntuación (v3)
    * Desafío (v2)

      * Insignia invisible de reCAPTCHA

    Diferencias entre `v2` y `v3`:

    * `v2` requiere la interacción del usuario con desafíos visibles, como hacer clic en una casilla de verificación o resolver acertijos.
    * `v3` se ejecuta en segundo plano y proporciona una puntuación basada en el comportamiento del usuario, sin requerir interacción. Al usar esta versión, asegúrate de implementar lógica de negocio adicional para gestionar la puntuación y determinar si se necesita alguna acción adicional.

    ##### Configuración del widget de Google reCAPTCHA

    La configuración del widget de Google reCAPTCHA incluye:

    | Configuración   | Descripción                                                                                                                                                                                                                                                |
    | --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Clave del sitio | La clave pública del sitio que se usa para inicializar reCAPTCHA en tu sitio web. Puedes crearla en la [consola de Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) o en tu proyecto de Google Cloud Platform.                             |
    | Clave secreta   | La clave secreta que se usa para comunicarse de forma segura con el servicio reCAPTCHA del lado del servidor. Puedes crearla en la [consola de Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) o en tu proyecto de Google Cloud Platform. |

    ##### Valor de salida de Google reCAPTCHA

    El tipo de dato del valor de salida del widget de Google reCAPTCHA es un `object`.

    Ejemplo de respuesta de `v2`:

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com"
      }
    }
    ```

    Ejemplo de respuesta de `v3`:

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com",
        "score": 0.9
      }
    }
    ```
  </Accordion>

  <Accordion title="Credenciales verificables">
    Un widget que permite a un usuario verificar su identidad mediante credenciales verificables almacenadas en su billetera digital.

    ##### Configuración del widget de Credenciales verificables

    La configuración del widget de Credenciales verificables incluye:

    | Configuración           | Descripción                                                                                                                                                                                                                              |
    | ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | URL                     | El valor de la URL que se usa para generar el código QR. Este valor se devuelve en la propiedad `engagement` de la solicitud de verificación.                                                                                            |
    | Texto del enlace        | Texto alternativo que se mostrará para los usuarios que no puedan escanear el código QR y prefieran abrir el enlace directamente en su dispositivo.                                                                                      |
    | Tamaño                  | El tamaño del código QR.                                                                                                                                                                                                                 |
    | Token público           | El token de acceso necesario para consumir el endpoint de polling. Asegúrate de que el token se genere solo con el scope `read:verification_request` para evitar exponer acceso a otros recursos.                                        |
    | ID de verificación      | El ID de verificación generado cuando inicias la solicitud de verificación.                                                                                                                                                              |
    | Tiempo máximo de espera | La cantidad máxima de tiempo que se debe esperar para que se complete el proceso de verificación. Una vez alcanzado el tiempo establecido, el polling se detendrá y se generará un error, independientemente del estado de verificación. |

    ##### Valor de salida de Credenciales verificables

    El tipo de dato del valor de salida del widget de Credenciales verificables es un `object`.

    ```json lines
    {
      "verifiable_credentials_widget_id": {
        "state": "honored",
        "reason": "...", // El campo reason contiene información adicional (si está disponible) sobre el estado de la solicitud de verificación. 
        "presentation": {} // La presentación contiene los claims proporcionados por la billetera en respuesta a la definición de presentación.
      }
    }
    ```
  </Accordion>
</AccordionGroup>

<div id="router-node">
  ## Nodo Router
</div>

Los nodos Router te permiten agregar reglas para crear bifurcaciones de lógica condicional entre nodos.

De forma predeterminada, un nodo Router tiene una única regla de paso directo llamada **Caso predeterminado** que se conecta a otros nodos. Puedes agregar reglas adicionales en función de un conjunto de condiciones que debe cumplir una variable para conectarlas luego a otros nodos. Para obtener más información, consulta [Router.](/docs/es/customize/forms/routers)
