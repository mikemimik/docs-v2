---
description: Aprende a renderizar formularios con Actions.
title: Renderizar formularios con Actions
---

Una vez que crees un formulario, puedes renderizarlo con [Actions](/docs/es/customize/actions) mediante el método `api.prompt.render()`:

```javascript lines
// Ejemplo usando el trigger Post Login

exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id');
}

exports.onContinuePostLogin = async (event, api) => {
  // Agrega tu lógica después de completar el formulario
}
```

Reemplaza form&#95;id por el ID del Form. Puedes encontrar el ID en la URL del Form, por ejemplo:  `ap_pUMG...` o seleccionarlo en la pestaña **Render** del Form editor.
En el [editor de código de Actions](/docs/es/customize/actions/write-your-first-action#create-an-action), puedes definir la lógica de negocio para decidir cuándo y cómo renderizar el Form.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {

  // Solo renderizar el formulario si al usuario le falta el metadata company_name
  if (!event.user.user_metadata.company_name) {
    api.prompt.render(':form_id');
  }
}

exports.onContinuePostLogin = async (event, api) => {
  // Agrega tu lógica después de completar el formulario
}
```

Para obtener más información sobre estos objetos, consulta:

* [Objeto Event](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object): Obtén más información sobre los objetos Event y sus propiedades.
* [objeto de API](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object): Obtén más información sobre los objetos de API y sus métodos.

<div id="populate-values-for-existing-fields-and-hidden-fields-client-side">
  ## Asignar valores a campos existentes y campos ocultos (del lado del cliente)
</div>

Puedes asignar valores a campos existentes y campos ocultos usando la propiedad `fields` como segundo argumento del método `render`.

En el ejemplo siguiente, el valor `Jane` se asigna al campo `first_name`.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id', {
    fields: {
      first_name: 'Jane',
    }
  });
}

exports.onContinuePostLogin = async (event, api) => {
  // Agrega tu lógica después de completar el formulario
}
```

<Warning>
  No completes información sensible; los valores introducidos quedan expuestos en el navegador (en el cliente).
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Asegúrate de que el campo `ID` al que se hace referencia en el segundo argumento esté disponible en el cliente en tus [nodos y componentes](/docs/es/customize/forms/nodes-and-components); de lo contrario, el valor no se completará.
</Callout>

<div id="inject-custom-data-with-shared-variables-server-side">
  ## Inyecta datos personalizados con variables compartidas (en el servidor)
</div>

Puedes inyectar variables del lado del servidor usando la propiedad `vars` como segundo argumento en tu método `render`. Esto puede usarse para inyectar información sensible sin exponerla del lado del cliente.

En el ejemplo siguiente, el valor `123456789` asigna un valor a la variable `external_user_id`.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id', {
    vars: {
      external_user_id: '123456789',
    }
  });
}

exports.onContinuePostLogin = async (event, api) => {
  // Agrega tu lógica después de completar el formulario
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las Variables compartidas `{{vars.*}}` no tienen restricciones de tipo de dato.

  Las Variables compartidas `{{vars.*}}` siempre se enmascaran automáticamente.

  A diferencia de cuando se rellenan variables `{{fields.*}}`, la propiedad `{{vars.*}}` no necesita existir en tu formulario antes de inyectar valores.

  Puedes usar la propiedad `{{vars.*}}` para hacer referencia a las Variables compartidas en tus formularios y los Flows vinculados. Para obtener más información, consulta [Variables](/docs/es/customize/forms/variables).
</Callout>

<div id="fields-and-shared-variables-data-in-actions">
  ## Datos de campos y variables compartidas en Actions
</div>

Los datos de campos y variables compartidas recopilados en tus formularios están disponibles automáticamente en `event.prompt` en la función `resume` de la Action actual:

* La propiedad `id`, con el ID del prompt que estás renderizando.
* El objeto `fields`, que contiene todos los datos de tus campos y campos ocultos.
* El objeto `vars`, que contiene todos los datos de tus variables compartidas.

<Accordion title="Ejemplo del objeto event.prompt">
  ```json lines
  {
    "id": "ap_fuVuFiiQWN3mTEujWTy966",
    "fields": {
      "first_name": "Jane",
      "company_name": "Okta"
    },
    "vars": {
      "external_crm_uuid": "f8f32e6f-2329-49bd-bf21-fa8b0bea2652",
      "api_hostname": "api.example.com"
    }
  }
  ```
</Accordion>

En el siguiente ejemplo, `api.user.setUserMetadata` rellena el atributo `company_name` de `user_metadata` con la propiedad `event.prompt.fields.company_name` recopilada de tu formulario.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id');
}

exports.onContinuePostLogin = async (event, api) => {
  api.user.setUserMetadata('company_name', event.prompt.fields.company_name);
}
```

<div id="restrictions-and-limitations">
  ## Restricciones y limitaciones
</div>

* No puedes [redirigir](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/redirect-with-actions) a un usuario y renderizar un formulario en la misma Action. Si necesitas ambas opciones, considera usar distintas Actions.
* Solo puedes renderizar un formulario por Action. Si necesitas renderizar más de uno, debes renderizarlos en distintas Actions.
* El límite de tamaño de la propiedad `fields` es de 24 KB.
* El método `api.prompt.render()` está disponible en los siguientes triggers:

  * [Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object)
  * [post-challenge](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object)
