title: Variables y funciones auxiliares

Las variables te permiten acceder a los datos de formularios y Flows para crear lógica de negocio y automatizaciones personalizadas.

Algunas consideraciones básicas sobre el uso de variables son:

* Las variables van entre llaves `{{ variable }}`.
* Si una variable no existe o no se puede resolver, tiene el mismo efecto que una variable `undefined`.
* Puedes usar operadores de fusión nula para manejar variables `null` o `undefined`; por ejemplo: `{{fields.foo ?? fields.bar}}`.

<div id="available-variables">
  ## Variables disponibles
</div>

Puedes usar distintos tipos de variables para hacer referencia a los datos que recopilas de los clientes con formularios y Flows, y transformarlos.

| Variable                         | Sintaxis          | Dónde               | Descripción                                                                         |
| -------------------------------- | ----------------- | ------------------- | ----------------------------------------------------------------------------------- |
| Contexto                         | `{{context.*}}`   | formularios / Flows | Hace referencia a los datos de contexto de la transacción actual                    |
| Campos del formulario            | `{{fields.*}}`    | formularios / Flows | Hace referencia a los datos de los campos del formulario y de los campos ocultos    |
| Variables compartidas            | `{{vars.*}}`      | formularios / Flows | Hace referencia a los datos almacenados como variables compartidas                  |
| Salida de las acciones del flujo | `{{actions.*}}`   | Flows               | Hace referencia a los datos de la respuesta de salida de acciones previas del flujo |
| Funciones auxiliares             | `{{functions.*}}` | formularios / Flows | Funciones auxiliares para transformar datos                                         |

<div id="context-variables">
  ### Variables de contexto
</div>

Los formularios y los flujos heredan automáticamente las variables de contexto de la transacción actual. Puedes acceder a las siguientes variables de contexto:

* El objeto `user`, con acceso a las siguientes propiedades:

  * `user.user_id`
  * `user.username`
  * `user.name`
  * `user.given_name`
  * `user.family_name`
  * `user.nickname`
  * `user.email`
  * `user.email_verified`
  * `user.phone_number`
  * `user.phone_verified`
  * `user.picture`
  * `user.user_metadata`
  * `user.app_metadata`
  * `user.created_at`
  * `user.updated_at`
  * `user.last_password_reset`
  * `user.identities`
* El objeto `organization`, con acceso a las siguientes propiedades:

  * `organization.id`
  * `organization.name`
  * `organization.display_name`
  * `organization.metadata`
* El objeto `client`, con acceso a las siguientes propiedades:

  * `client.client_id`
  * `client.name`
* El objeto `tenant`, con acceso a la siguiente propiedad:

  * `tenant.name`
* El objeto `transaction`, con acceso a las siguientes propiedades:

  * `transaction.state`
  * `transaction.correlation_id`
* El objeto `custom_domain`, con acceso a las siguientes propiedades:

  * `custome_domain.domain`
  * `custome_domain.domain_metadata`

<div id="form-fields-variables">
  ### Variables de los campos del formulario
</div>

Las variables de los campos contienen los datos recopilados del usuario a través de los campos de entrada del formulario y los datos de los campos ocultos. Los datos de cada campo varían según su tipo. Por ejemplo: `{{fields.first_name}}` hace referencia al valor introducido en un campo con el ID `first_name`.

<Frame>![Dashboard > Actions > Formularios > Formulario](/docs/images/cdy7uua7fh8z/4D42A6ww4XvFxweoxh04Rd/016c2ed9d25bc2267ed3ed31bf8b2bf6/form-fields-variable.png)</Frame>

<div id="flow-output-actions-variables">
  ### Variables de salida de las acciones de Flow
</div>

Cuando se ejecuta una acción de Flow, se genera un objeto de salida cuya respuesta puede referenciarse en acciones posteriores. Por ejemplo: `{{actions.generate_otp.code}}` hace referencia al código de salida de la acción con ID `generate_otp`.

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/6jYtEdYqSGXm3DDv0mxQ0o/2d48012629745e07137bf6ba42f4d829/flow-output-variable.png)</Frame>

<div id="shared-variables">
  ### Variables compartidas
</div>

Puede almacenar variables para compartirlas con flujos posteriores o incluso con componentes del formulario dentro de la misma transacción. Las variables compartidas son efímeras y están asociadas a un recorrido específico del formulario. No se puede acceder a sus valores fuera de ese recorrido. Por ejemplo, `{{vars.external_id}}` hace referencia al valor de una variable compartida con el ID `external_id`.

<Frame>![Dashboard > Actions > Formularios > Flows](/docs/images/cdy7uua7fh8z/4571ExRTHL0KZKJKX9pNwj/abb631b8608a6356b0e5b3784cf8b02f/flow-shared-variable.png)</Frame>

<Warning>
  Variables compartidas:

  * son internas (del lado del servidor) y no se exponen al navegador (del lado del cliente).
  * pueden resolverse y exponerse al navegador si hace referencia a una variable compartida en el componente del formulario.
  * pueden contener información confidencial, así que tenga cuidado al acceder a una variable compartida para evitar cualquier exposición innecesaria o no intencionada.
</Warning>

<div id="helper-functions">
  ### Funciones auxiliares
</div>

Las funciones auxiliares te permiten transformar datos y realizar operaciones sencillas. Por ejemplo, `{{functions.toString(fields.privacy_policies)}}` transforma el valor del campo con ID `privacy_policies` mediante la función `toString()`.

<div id="variables-best-practices">
  ## Prácticas recomendadas para las variables
</div>

<div id="use-meaningful-ids">
  ### Usa IDs descriptivos
</div>

Cuando añades un campo de formulario o una acción de Flow, el ID se genera automáticamente.

Actualiza el ID con un valor más descriptivo para que puedas identificar la información que contiene. Por ejemplo, si creas un campo de formulario que almacena el nombre del usuario, actualiza el ID a `first_name`.

<div id="be-mindful-when-updating-existing-ids">
  ### Ten cuidado al actualizar IDs existentes
</div>

Si una variable ya está en uso y necesitas modificar el ID, identifica dónde lo has usado y actualiza el ID según corresponda para evitar valores indefinidos o vacíos.

<div id="check-flow-executions">
  ### Revisa las ejecuciones de Flow
</div>

Si ves un valor de variable inesperado, usa la vista [**Executions**](/docs/es/customize/forms/flows/flow-execution-and-debugger) para depurar la variable.

<div id="use-the-autocomplete-menu">
  ### Usa el menú de autocompletado
</div>

El menú de autocompletado te ayuda a localizar y referenciar correctamente las variables existentes.
