<Card title="Antes de comenzar">
  1. Crea una [aplicación de máquina a máquina](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps) con los siguientes scopes habilitados:

  * `read:users`
  * `update:users`
  * `create:users`
  * `read:users_app_metadata`
  * `update:users_app_metadata`
  * `create:users_app_metadata`

  2. Agrega una [conexión de Vault](/docs/es/customize/forms/vault) usando las credenciales de la aplicación M2M.
</Card>

formulario para Actions te permite crear flujos de perfil progresivo para decidir cuándo solicitar a los usuarios información faltante, como el nombre de la empresa y el cargo, y almacenarla como atributos de `user_metadata`.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/xUx9YmnHvnAqL19JvNtNP/60dae88f893d4ebe8e3523421e960596/Progressive-profle-overview.png)</Frame>

En las siguientes secciones se explica cómo puedes crear un formulario de perfil progresivo usando nodos y flujos, junto con los pasos para agregar tu formulario a una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="create-a-form-from-scratch">
  ## Crear un formulario desde cero
</div>

Sigue estos pasos para crear un formulario y recopilar información:

1. Abre el editor de formularios seleccionando **Auth0 Dashboard &gt; Actions &gt; Formularios.**
2. Selecciona **Crear formulario** &gt; **Empezar desde cero**.

De forma predeterminada, un formulario nuevo contiene los nodos Start, Step y Ending screen.

<Frame>![Dashboard > Formularios > Casos de uso > Formulario](/docs/images/cdy7uua7fh8z/2TZ9jSjgEcARFpULI6hz4R/a4319a1bd482f2ee5cc23100fdb2f42c/New-form-Forms-for-action-use-case.png)</Frame>

<div id="configure-the-step-node">
  ### Configurar el nodo Step
</div>

El nodo Step es la interfaz gráfica que ven los usuarios. Agregue [Fields](/docs/es/customize/forms/nodes-and-components) al nodo Step para recopilar el nombre de la empresa y el cargo del usuario siguiendo estos pasos:

1. Arrastre un campo **Rich text** desde el menú **Components** hasta el nodo **Step**.

   * **Rich text**: Introduzca un mensaje personalizado.

     * **Ejemplo**: `¡Complete su perfil! Necesitamos que complete su perfil para personalizar su experiencia.`
   * Seleccione **Save**.
2. Arrastre un campo **Text** al nodo **Step**.

   * **ID**: Introduzca `company_name`.
   * **Label**: Active la casilla.
   * **Label**: Introduzca `Company name`.
   * **Required**: Active la casilla.
   * Seleccione **Save**.
3. Arrastre un campo **Text** al nodo **Step**.

   1. **ID**: Introduzca `job_title`.
   2. **Label**: Introduzca `Job title`.
   3. **Label**: Active la casilla.
   4. **Required**: Active la casilla.
   5. Seleccione **Save**.
4. Seleccione **Publish** para guardar.

<Frame>![Dashboard > Actions > Formularios > Formulario > nodo Step](/docs/images/cdy7uua7fh8z/6H1lRzwKyYioQyQJ9MtIoM/aa248e92fc028a68a5b92ff361a11644/progressive-profile-step.png)</Frame>

<div id="configure-the-flow-node">
  ### Configurar el nodo de flujo
</div>

Agrega un nodo **Flow** después del nodo **Step** para actualizar `user_metadata` y reanudar el flujo de autenticación siguiendo estos pasos:

1. Selecciona **Flow** en la parte inferior del editor de formularios.
2. Elimina el enlace existente entre los nodos **Step** y **Ending Screen**.
3. Selecciona el nuevo **Flow** &gt; **Click to add a flow** &gt; **Create a new flow.**

   * Ingresa `Update user_metadata` en el campo **Name**.
   * Selecciona **Create**.
   * Selecciona **Save**.
4. Vincula el nodo **Flow** con los nodos **Step** y **Ending Screen**, como se muestra a continuación.
5. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/5C91DaWn4LGtJT9BF87ssu/095171d0c452a287c44bfde12fd6d13f/link-nodes.png)</Frame>

Agrega una Action de Auth0 **Update user** al flujo siguiendo estos pasos:

1. Selecciona el flujo **Update user&#95;metadata** y luego **Edit Flow** para abrir el editor de flujos en una pestaña nueva.
2. Debajo de la Action **Start**, selecciona el ícono **+** para agregar una Action **Update user**. Completa los siguientes campos y luego selecciona **Save** para continuar.

   * **Connection**: Selecciona en el menú desplegable la conexión de Vault de tu aplicación M2M.

   * **User ID**: Ingresa `{{context.user.user_id}}`.

   * **Body**: Copia y pega el siguiente código para actualizar `user_metadata` con las propiedades `job_title` y `company_name`.

     ```json lines
     {
        "user_metadata": {
           "job_title": "{{fields.job_title}}",
           "company_name": "{{fields.company_name}}"
        }
     }
     ```

   * Selecciona **Save**.
3. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/65Jx4W5oIZJyyvA8z4HqIx/0661b1a8ca6361c54fc4fef07ba59f0b/update-user.png)</Frame>

<div id="retrieve-the-form-render-code">
  ### Recupera el código para renderizar el formulario
</div>

Recupera el código para renderizar el formulario con una Action de Post Login personalizada siguiendo estos pasos:

* En el editor de formularios, selecciona **`<> Render`**.
* Selecciona **Copiar**.

<Frame>![Dashboard > Actions > Formularios > Formulario > Insertar](/docs/images/cdy7uua7fh8z/2EIUzc539sq9lwoTKbjoJv/496a864c0fc44a782c9cadca119dbfb0/embed.png)</Frame>

<div id="create-a-post-login-action">
  ## Crear una Action de Post Login
</div>

Cree una Action de Post Login para renderizar su formulario siguiendo estos pasos:

1. Vaya a **Auth0 Dashboard &gt; Actions &gt; Flows &gt; Login.**
2. Seleccione el icono **+** y luego **Build from scratch**:

   * **Name**: Introduzca `Render Progressive Profile Form`.
   * **trigger**: Seleccione `Login / Post Login`.
   * **Runtime**: Seleccione la versión recomendada.
3. Seleccione **Create**.

Para configurar la Action de Post Login:

1. Elimine el código existente del editor de código.
2. Pegue el código de inserción del formulario en el editor de código.
3. Edite el código para definir la lógica condicional que renderizará el formulario.

   ```javascript lines
   /**
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden utilizarse para cambiar el comportamiento del inicio de sesión.
   */
   exports.onExecutePostLogin = async (event, api) => {
     const FORM_ID = 'REPLACE_WITH_YOUR_FORM_ID';

     if (
       event.stats.logins_count > 2 &&
       !event.user.user_metadata.company_name &&
       !event.user.user_metadata.job_title
     ) {
       api.prompt.render(FORM_ID);
     }
   }

   exports.onContinuePostLogin = async (event, api) => { }
   ```

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     El código de ejemplo anterior renderizará el formulario cuando un usuario tenga más de 2 inicios de sesión y no tenga definidas las propiedades `company_name` y `job_title` en los metadatos del usuario.
   </Callout>
4. Seleccione **Deploy.**
5. Arrastre y suelte la Action **Render Progressive Profile Form** en el flujo **Login**.

   <Frame>![Dashboard > formulario > Casos de uso Action de Login de Perfil Progresivo](/docs/images/cdy7uua7fh8z/1QjjZdpTqTxx012UDN4xk9/99fba5ef4068a5688f1762eea7f42976/renderprogessiveprofile.png)</Frame>
6. Seleccione **Apply**.

<div id="test-implementation">
  ### Probar la implementación
</div>

Pruebe la implementación siguiendo estos pasos:

1. Inicie sesión con un usuario existente cuyo número de inicios de sesión sea superior a 2 y cuyos atributos `user_metadata` `company_name` y `job_title` no tengan valores.
2. La Action de Post Login del Login flow renderizará el formulario y solicitará esa información.
3. Seleccione **Auth0 Dashboard &gt; User Management &gt; Users,** ubique al usuario y verifique que los atributos `user_metadata` `company_name` y `job_title` contengan información.
