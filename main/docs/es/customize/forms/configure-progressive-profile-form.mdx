<Card title="Antes de empezar">
  1. Crea una [Aplicación de máquina a máquina](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps) con los siguientes scopes habilitados:

  * `read:users`
  * `update:users`
  * `create:users`
  * `read:users_app_metadata`
  * `update:users_app_metadata`
  * `create:users_app_metadata`

  2. Agrega una [conexión de Vault](/docs/es/customize/forms/vault) con las credenciales de la aplicación M2M.
</Card>

Forms for Actions te permite crear flujos de perfilado progresivo para decidir cuándo solicitar a los usuarios la información que falta, como el nombre de la empresa y el puesto, y almacenarla como atributos de `user_metadata`.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/xUx9YmnHvnAqL19JvNtNP/60dae88f893d4ebe8e3523421e960596/Progressive-profle-overview.png)</Frame>

En las siguientes secciones se describe cómo puedes crear un formulario de perfil progresivo con nodos y flujos, junto con los pasos para agregarlo a una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="create-a-form-from-scratch">
  ## Crear un formulario desde cero
</div>

Sigue estos pasos para crear un formulario para recopilar información:

1. Abre el editor de formularios seleccionando **Auth0 Dashboard &gt; Actions &gt; Forms.**
2. Selecciona **Create Form** &gt; **Start from scratch**.

De forma predeterminada, un formulario nuevo contiene un nodo Start, un nodo Step y un nodo de pantalla Ending.

<Frame>![Dashboard > Forms > Use Cases > Form](/docs/images/cdy7uua7fh8z/2TZ9jSjgEcARFpULI6hz4R/a4319a1bd482f2ee5cc23100fdb2f42c/New-form-Forms-for-action-use-case.png)</Frame>

<div id="configure-the-step-node">
  ### Configurar el nodo Step
</div>

El nodo Step es la interfaz gráfica visible para los usuarios. Agrega [Fields](/docs/es/customize/forms/nodes-and-components) al nodo Step para recopilar el nombre de la empresa y el cargo del usuario siguiendo estos pasos:

1. Arrastra un campo de **Rich text** desde el **Components menu** al nodo **Step**.

   * **Rich text**: Introduce un mensaje personalizado.

     * **Example**: `¡Completa tu perfil! Necesitamos que completes tu perfil para personalizar tu experiencia.`
   * Selecciona **Save**.
2. Arrastra un campo de **Text** al nodo **Step**.

   * **ID**: Introduce `company_name`.
   * **Label**: Activa la casilla.
   * **Label**: Introduce `Company name`.
   * **Required**: Activa la casilla.
   * Selecciona **Save**.
3. Arrastra un campo de **Text** al nodo **Step**.

   1. **ID**: Introduce `job_title`.
   2. **Label**: Introduce `Job title`.
   3. **Label**: Activa la casilla.
   4. **Required**: Activa la casilla.
   5. Selecciona **Save**.
4. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > Form > Step node](/docs/images/cdy7uua7fh8z/6H1lRzwKyYioQyQJ9MtIoM/aa248e92fc028a68a5b92ff361a11644/progressive-profile-step.png)</Frame>

<div id="configure-the-flow-node">
  ### Configura el nodo **Flow**
</div>

Agrega un nodo **Flow** después del nodo **Step** para actualizar `user_metadata` y reanudar el flujo de autenticación siguiendo estos pasos:

1. Selecciona **Flow** en la parte inferior del editor de formularios.
2. Elimina la conexión existente entre los nodos **Step** y **Ending Screen**.
3. Selecciona el nuevo **Flow** &gt; **Click to add a flow** &gt; **Create a new flow.**

   * Ingresa `Update user_metadata` en el campo **Name**.
   * Selecciona **Create**.
   * Selecciona **Save**.
4. Conecta el nodo **Flow** con los nodos **Step** y **Ending Screen**, como se muestra a continuación.
5. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/5C91DaWn4LGtJT9BF87ssu/095171d0c452a287c44bfde12fd6d13f/link-nodes.png)</Frame>

Agrega una acción **Update user** de Auth0 al flujo siguiendo estos pasos:

1. Selecciona el flujo **Update user&#95;metadata** y luego **Edit Flow** para abrir el editor de flujos en una pestaña nueva.
2. Debajo de la acción **Start**, selecciona el ícono **+** para agregar una acción **Update user**. Completa los siguientes campos y luego selecciona **Save** para continuar.

   * **Connection**: Selecciona en el menú desplegable la conexión de Vault de tu aplicación M2M.

   * **User ID**: Ingresa `{{context.user.user_id}}`.

   * **Body**: Copia y pega el siguiente código para actualizar `user_metadata` con las propiedades `job_title` y `company_name`.

     ```json lines
     {
        "user_metadata": {
           "job_title": "{{fields.job_title}}",
           "company_name": "{{fields.company_name}}"
        }
     }
     ```

   * Selecciona **Save**.
3. Selecciona **Publish** para guardar.

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/65Jx4W5oIZJyyvA8z4HqIx/0661b1a8ca6361c54fc4fef07ba59f0b/update-user.png)</Frame>

<div id="retrieve-the-form-render-code">
  ### Obtener el código de renderizado del formulario
</div>

Obtenga el código de renderizado del formulario para mostrarlo visualmente con una Action de Post Login personalizada siguiendo estos pasos:

* En el editor de formularios, seleccione **`<> Render`**.
* Seleccione **Copy**.

<div id="create-a-post-login-action">
  ## Crear una Action de Post Login
</div>

Crea una Action de Post Login para mostrar tu formulario siguiendo estos pasos:

1. Ve a **Auth0 Dashboard &gt; Actions &gt; Flows &gt; Login.**
2. Selecciona el icono **+** y luego **Build from scratch**:

   * **Name**: Introduce `Render Progressive Profile Form`.
   * **Trigger**: Selecciona `Login / Post Login`.
   * **Runtime**: Selecciona la versión recomendada.
3. Selecciona **Create**.

Para configurar la Action de Post Login:

1. Elimina el código existente del editor de código.
2. Pega el código de inserción del formulario en el editor de código.
3. Edita el código para definir la lógica condicional que mostrará el formulario.

   ```javascript lines
   /**
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
   */
   exports.onExecutePostLogin = async (event, api) => {
     const FORM_ID = 'REPLACE_WITH_YOUR_FORM_ID';

     if (
       event.stats.logins_count > 2 &&
       !event.user.user_metadata.company_name &&
       !event.user.user_metadata.job_title
     ) {
       api.prompt.render(FORM_ID);
     }
   }

   exports.onContinuePostLogin = async (event, api) => { }
   ```

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     El código de ejemplo anterior mostrará el formulario cuando un usuario tenga más de 2 inicios de sesión y no tenga las propiedades `company_name` y `job_title` en los metadatos del usuario.
   </Callout>
4. Selecciona **Deploy.**
5. Arrastra y suelta la Action **Render Progressive Profile Form** en el flujo de **Login**.

   <Frame>![Dashboard > Forms > Use Cases Progressive Profile Login Action](/docs/images/cdy7uua7fh8z/1QjjZdpTqTxx012UDN4xk9/99fba5ef4068a5688f1762eea7f42976/renderprogessiveprofile.png)</Frame>
6. Selecciona **Apply**.

<div id="test-implementation">
  ### Probar la implementación
</div>

Prueba la implementación siguiendo estos pasos:

1. Inicia sesión con un usuario existente cuyo número de inicios de sesión sea mayor que 2 y cuyos atributos `user_metadata` `company_name` y `job_title` no tengan ningún valor.
2. La acción posterior al inicio de sesión en el flujo de Login mostrará el formulario y solicitará esa información.
3. Selecciona **Auth0 Dashboard &gt; User Management &gt; Users,** localiza al usuario y verifica que los atributos `user_metadata` `company_name` y `job_title` contengan información.
