El Vault te permite almacenar de forma segura secretos o configuraciones comunes (por ejemplo: claves de API, <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip>, <Tooltip tip="Token de actualización: token que se usa para obtener nuevos tokens de acceso sin necesidad de que el usuario vuelva a autenticarse." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip>) para autenticar solicitudes con servicios externos o integraciones como conexiones de Vault.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Hay conexiones que requieren que el usuario proporcione credenciales (por ejemplo, con una aplicación de Auth0 de máquina a máquina (M2M), se genera un token de acceso con un tiempo de expiración y un token de actualización). La conexión almacena ambos tokens y genera un nuevo token de acceso antes de que expire.
  * Hay servicios externos que requieren autorizar scopes para generar credenciales de API. Si cambias los scopes, actualiza la conexión de esos servicios para evitar problemas de conexión.
  * Cuando se agrega una conexión, queda lista para usarse en tus flujos. Cualquier actualización de una conexión existente se propagará automáticamente.
</Callout>

<div id="create-and-modify-vault-connections">
  ## Crear y modificar conexiones de Vault
</div>

Crea y modifica conexiones de Vault desde la página de Vault o desde el editor de flujos cuando una Action requiera credenciales.

<div id="create-vault-connections-from-the-vault-page">
  ### Crear conexiones de Vault desde la página de Vault
</div>

Añade una conexión de Vault desde la página de Vault siguiendo estos pasos:

1. Abre el editor de formularios seleccionando **Auth0 Dashboard &gt; Actions &gt; Forms.**
2. En la esquina superior izquierda, selecciona el menú del tenant y, a continuación, **Vault** para abrir la página de Vault.

   <Frame>![Dashboard > Forms > Vault](/docs/images/cdy7uua7fh8z/1sPNPKO6nuEDCuiYGRWSur/d85dc616170eaf2d7d5f321750863ccf/vault-connection.png)</Frame>
3. Selecciona **Añadir conexión de Vault**.
4. Selecciona la Action que quieras y **Continuar**.
5. Añade un **Nombre de la conexión** de Vault y selecciona **Continuar**.
6. Introduce las credenciales.
   Si no tienes las credenciales específicas, puedes seleccionar **crear una conexión vacía** y añadir las credenciales más tarde. Las conexiones vacías tienen un icono naranja que indica que faltan credenciales.
7. Selecciona **Añadir conexión de Vault**.

<div id="create-vault-connections-from-the-flow-editor">
  ### Crear conexiones de Vault desde el editor de flujos
</div>

Añade una conexión de Vault desde una Action de flujo, por ejemplo, Twilio, siguiendo estos pasos:

1. Abre el editor de flujos seleccionando **Auth0 Dashboard &gt; Actions &gt; Forms &gt; Flows**
2. Selecciona un flujo y añade una Action de **Send SMS with Twilio**.
3. En la configuración de **conexión de Vault** de la Action, selecciona el icono **+**.

   <Frame>![Dashboard > Forms > Vault ](/docs/images/cdy7uua7fh8z/1AsaznhRKxm5LZxPlcbfUw/10e5a8af700aab909f5b03fcd266da7e/twilio-vault-connection.png)</Frame>
4. Añade un **Nombre de la conexión** de Vault y selecciona **Continuar**.
5. Introduce las credenciales.
   Si no tienes las credenciales específicas, puedes seleccionar **crear una conexión vacía** y añadirlas más tarde. Las conexiones vacías muestran un icono naranja que indica que faltan credenciales.

   <Frame>![Dashboard > Forms > Vault > configuración de conexión de Twilio](/docs/images/cdy7uua7fh8z/yobXanutPRCO3FWek8cOk/bc29e39fb8593cff35e2738234bfe6fe/twilio-vault-settings.png)</Frame>
6. Selecciona **Añadir conexión de Vault**.
