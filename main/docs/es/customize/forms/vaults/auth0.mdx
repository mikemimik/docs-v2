<div id="get-access-to-auth0-management-api">
  ## Obtén acceso a Auth0 Management API
</div>

[Crea una aplicación de máquina a máquina (M2M)](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps) autorizada para Auth0 Management API.

<Warning>
  Recuerda autorizar tu aplicación M2M para acceder a la Management API y seleccionar los scopes adecuados siguiendo el principio del mínimo privilegio:

  * Para Get User: `read:users`
  * Para Create User: `create:users`
  * Para Update User: `update:users`
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No hace falta autorización de Auth0 Management API para enviar notificaciones a través de los proveedores configurados en tu tenant de Auth0.
</Callout>

<div id="configure-your-vault-connection">
  ## Configura tu conexión de Vault
</div>

Para configurar una conexión de Vault de Auth0 para tus Auth0 Flow Actions, define los parámetros obligatorios en función de la aplicación M2M creada anteriormente.

<Frame>![Dashboard > Actions > formulario > Vault > Auth0](/docs/images/customize/forms/vaults/auth0-vault-connection.png)</Frame>

<div id="input-settings">
  #### configuración de entrada
</div>

| Parámetro           | Descripción                                                                   |
| ------------------- | ----------------------------------------------------------------------------- |
| Dominio del tenant  | Dominio del tenant de tu aplicación M2M.                                      |
| ID de cliente       | ID de cliente de tu aplicación M2M autorizada para Auth0 Management API.      |
| Secreto de cliente  | Secreto de cliente de tu aplicación M2M autorizada para Auth0 Management API. |
| Audience (opcional) | Audience de Auth0 Management API.                                             |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  De forma predeterminada, el Audience se basa en el dominio del tenant.
</Callout>
