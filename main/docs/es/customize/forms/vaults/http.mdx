<div id="configure-your-http-vault-connection-options">
  ## Configura las opciones de tu HTTP Vault Connection
</div>

HTTP Vault Connection se puede configurar con distintas opciones de autenticación para usarse luego en tus HTTP Request Flow Actions.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configura tu HTTP Vault Connection para Bearer Token
</div>

Para configurar una HTTP Vault Connection con autorización de **Bearer Token**, debes configurar los parámetros obligatorios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro    | Descripción                                                 |
| ------------ | ----------------------------------------------------------- |
| Bearer Token | Token estático utilizado para autorizar servicios externos. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configura tu HTTP Vault Connection para credenciales de cliente de OAuth
</div>

Para configurar una HTTP Vault Connection con autorización de **credenciales de cliente**, debes establecer los parámetros requeridos.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro            | Descripción                                                                          |
| -------------------- | ------------------------------------------------------------------------------------ |
| Endpoint del tenant  | Endpoint del tenant del servidor de autorización que emite los tokens de acceso.     |
| ID de cliente        | ID de cliente de tu aplicación de máquina a máquina autorizada para Custom API.      |
| Secreto de cliente   | Secreto de cliente de tu aplicación de máquina a máquina autorizada para Custom API. |
| Audiencia (opcional) | Audiencia de Custom API. Normalmente se usa para las Custom API definidas en Auth0.  |
| Recurso (opcional)   | Recurso de Custom API. Parámetro estándar con un significado similar al de Audience. |
| Scope (opcional)     | Lista de scope en formato String, separada por espacios.                             |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configura tu HTTP Vault Connection para API Key
</div>

Para configurar una HTTP Vault Connection con autorización de **API Key**, debes establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

### configuración de entrada

| Parámetro | Descripción                                                                                                     |
| --------- | --------------------------------------------------------------------------------------------------------------- |
| Nombre    | Nombre de la API Key que se añadirá como encabezado o parámetro de consulta.                                   |
| Valor     | Valor de la API Key que se añadirá como encabezado o parámetro de consulta.                                    |
| Agregar a | De forma predeterminada, es encabezado. Determina si la API Key se añadirá como encabezado o parámetro de consulta. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configura tu HTTP Vault Connection para usar autenticación **Basic Auth**
</div>

Para configurar una HTTP Vault Connection con **Basic Auth** como método de autorización, debes establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

### configuración de entrada

| Parameter         | Description                                                                                                                             |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Nombre de usuario | Nombre de usuario que se concatenará y codificará en base64, y luego se añadirá como parámetro del encabezado Authorization para Basic. |
| Contraseña        | Contraseña que se concatenará y codificará en base64, y luego se añadirá como parámetro del encabezado Authorization para Basic.        |

<Warning>
  La configuración de HTTP Vault Connection con Basic Auth está pensada para reemplazar la opción integrada actualmente en la Action HTTP Request de Flows.
</Warning>
