<div id="configure-your-http-vault-connection-options">
  ## Configura las opciones de tu conexión HTTP de Vault
</div>

conexión HTTP de Vault se puede configurar con distintas opciones de autenticación y luego usar en tus HTTP Request Flow Actions.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configura tu conexión HTTP de Vault para Bearer Token
</div>

Para configurar una conexión HTTP de Vault con autorización de **Bearer Token**, debes establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro    | Descripción                                                            |
| ------------ | ---------------------------------------------------------------------- |
| Bearer Token | Token estático utilizado para la autorización con el servicio externo. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configure su conexión HTTP de Vault para OAuth con credenciales de cliente
</div>

Para configurar una conexión HTTP de Vault con autorización de **credenciales de cliente**, debe establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro           | Descripción                                                                                          |
| ------------------- | ---------------------------------------------------------------------------------------------------- |
| Endpoint del tenant | Endpoint del tenant del servidor de autorización que emite los tokens de acceso.                     |
| ID de cliente       | ID de cliente de tu aplicación M2M autorizada para la API personalizada.                             |
| Secreto de cliente  | Secreto de cliente de tu aplicación M2M autorizada para la API personalizada.                        |
| Audience (Opcional) | Audience de la API personalizada. Normalmente se usa para las API personalizadas definidas en Auth0. |
| Recurso (Opcional)  | Recurso de la API personalizada. Parámetro estándar con un significado similar a Audience.           |
| Scope (Opcional)    | Lista de scopes en formato String, separada por espacios.                                            |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configura tu conexión HTTP de Vault para API Key
</div>

Para configurar una conexión HTTP de Vault con autorización mediante **API Key**, debes definir los parámetros necesarios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro | Descripción                                                                                                                        |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| Nombre    | Nombre de la API Key que se agregará como encabezado o parámetro de consulta.                                                      |
| Valor     | Valor de la API Key que se agregará como encabezado o parámetro de consulta.                                                       |
| Agregar a | De forma predeterminada, se agrega al encabezado. Determina si la API Key se debe agregar como encabezado o parámetro de consulta. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configura tu conexión HTTP de Vault para la autenticación Basic Auth
</div>

Para configurar una conexión HTTP de Vault con autorización **Basic Auth**, debes establecer los parámetros obligatorios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

### configuración de entrada

| Parameter         | Description                                                                                                                                  |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| Nombre de usuario | Nombre de usuario que se concatenará y codificará en base64 para luego añadirse como parámetro del encabezado Authorization para Basic Auth. |
| Contraseña        | Contraseña que se concatenará y codificará en base64 para luego añadirse como parámetro del encabezado Authorization para Basic Auth.        |

<Warning>
  La configuración conexión HTTP de Vault con Basic Auth está pensada para sustituir la opción integrada actualmente en la Action HTTP Request de Flows.
</Warning>
