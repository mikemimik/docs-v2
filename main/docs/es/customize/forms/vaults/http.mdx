<div id="configure-your-http-vault-connection-options">
  ## Configura las opciones de tu conexión HTTP de Vault
</div>

Puedes configurar conexión HTTP de Vault con diferentes opciones de autenticación y luego usarla en tus HTTP Request Flow Actions.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configura tu conexión HTTP de Vault para Bearer Token
</div>

Para configurar una conexión HTTP de Vault con autorización de **Bearer Token**, debes establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro    | Descripción                                                             |
| ------------ | ----------------------------------------------------------------------- |
| Bearer Token | Token estático utilizado para la autorización ante el servicio externo. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configura tu conexión HTTP de Vault para las credenciales de cliente de OAuth
</div>

Para configurar una conexión HTTP de Vault con autorización de **credenciales de cliente**, debes establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

### configuración de entrada

| Parameter           | Description                                                                                          |
| ------------------- | ---------------------------------------------------------------------------------------------------- |
| Endpoint del tenant | Endpoint del tenant del servidor de autorización que emite los tokens de acceso.                     |
| ID de cliente       | ID de cliente de su aplicación M2M autorizada para la API personalizada.                             |
| Secreto de cliente  | Secreto de cliente de su aplicación M2M autorizada para la API personalizada.                        |
| Audience (opcional) | Audience de la API personalizada. Normalmente se usa para las API personalizadas definidas en Auth0. |
| Recurso (opcional)  | Recurso de la API personalizada. Parámetro estándar con un significado similar al de Audience.       |
| Scope (opcional)    | Lista de scopes en formato String separada por espacios.                                             |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configura tu conexión HTTP de Vault para API Key
</div>

Para configurar una conexión HTTP de Vault con autorización de **API Key**, debes establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

### configuración de entrada

| Parámetro | Descripción                                                                                                              |
| --------- | ------------------------------------------------------------------------------------------------------------------------ |
| Nombre    | Nombre de la API Key que se añadirá como encabezado o parámetro de consulta.                                             |
| Valor     | Valor de la API Key que se añadirá como encabezado o parámetro de consulta.                                              |
| Agregar a | De manera predeterminada, es Encabezado. Determina si la API Key se debe añadir como encabezado o parámetro de consulta. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configura tu conexión HTTP de Vault para usar autenticación Basic Auth
</div>

Para configurar una conexión HTTP de Vault con autorización **Basic Auth**, debes establecer los parámetros requeridos.

<Frame>![Dashboard > Actions > formularios > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

### configuración de entrada

| Parámetro         | Descripción                                                                                                                                |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Nombre de usuario | Nombre de usuario que se concatenará y codificará en base64, y luego se añadirá como parámetro del encabezado Authorization de Basic Auth. |
| Contraseña        | Contraseña que se concatenará y codificará en base64, y luego se añadirá como parámetro del encabezado Authorization de Basic Auth.        |

<Warning>
  La configuración de HTTP Vault Connection con Basic Auth está pensada para reemplazar la opción integrada actualmente en la Action HTTP Request de Flows.
</Warning>
