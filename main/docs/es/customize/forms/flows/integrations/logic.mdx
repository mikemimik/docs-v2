Esta lista de acciones de lógica te permite agregar lógica condicional en formularios, almacenar variables compartidas, mapear valores y mostrar mensajes de error personalizados.

<div id="ifthen-condition">
  ## Condición if/then
</div>

Agrega una acción condicional con ramas True / False para personalizar el comportamiento del flujo en función de los datos de entrada o de las propiedades.

<Frame>![](/docs/images/cdy7uua7fh8z/4ehTUSu8W1LibUd1uj3x5s/00d76b15c68746ed74e44127804c9557/if-then-condition-action.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro | Descripción                                                                                                                                                                |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Condition | Acción condicional configurable que activa las ramas `True` / `False` en función del resultado. Puedes usar los operadores `AND` / `OR` para anidar distintas condiciones. |

<div id="output-object">
  ### Objeto de salida
</div>

| Propiedad | Tipo    | Descripción                                                   |
| --------- | ------- | ------------------------------------------------------------- |
| `result`  | Boolean | Devuelve `true` o `false` según el resultado de la condición. |

<div id="output-object-example">
  ### Ejemplo del objeto de salida
</div>

```json lines
{
  "result": true
}
```

<div id="store-shared-variable">
  ## Almacenar variable compartida
</div>

Almacena variables en un estado privado disponible en tu formulario y en todos sus flujos.

Puedes hacer referencia a estas variables compartidas mediante `{{vars.*}}`**.** Para obtener más información, consulta [Variables](/docs/es/customize/forms/variables).

<Frame>![](/docs/images/cdy7uua7fh8z/4MO9wvChhnCJOg9OAdWuEf/98182a2bd8b6cf4c4baebefb96bce121/stored-shared-variable.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro       | Descripción                                                                                                                                              |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Campos de datos | Las variables que se almacenarán en un estado privado para usarlas en el formulario y a lo largo del flujo. Use pares clave/valor para añadir variables. |

<div id="show-error-message">
  ## Mostrar mensaje de error
</div>

Muestra un mensaje de error personalizado al usuario.

<Warning>
  Esta acción del flujo finaliza el flujo. Las acciones insertadas después no se ejecutarán.
</Warning>

<Frame>![](/docs/images/cdy7uua7fh8z/1Kd4SyaBtAC9S8nfWFeGfh/6e42b16c0ab69dc6d026fb3601133fbe/show-error-message-flow-action.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro        | Descripción                                     |
| ---------------- | ----------------------------------------------- |
| Mensaje de error | El mensaje de error que se mostrará al usuario. |

<div id="input-value-mapping">
  ## Mapeo de valores de entrada
</div>

Asocia una variable de entrada con una salida en función de sus valores.

<Frame>![](/docs/images/cdy7uua7fh8z/4VtAr4vCSXtKlSaxtiXQhV/bcb07221f688483aa2010d80394ae630/input-value-mapping.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro                      | Descripción                                                                                                                         |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------- |
| Valor de entrada (obligatorio) | La variable de entrada.                                                                                                             |
| Tabla de mapeo                 | Asigna el valor de una variable de entrada a una salida.                                                                            |
| Valor de salida predeterminado | Establece un valor de salida predeterminado que se usará cuando no se haya establecido ningún valor de salida en la tabla de mapeo. |

<div id="output-object">
  ### Objeto de salida
</div>

| Propiedad | Tipo   | Descripción                               |
| --------- | ------ | ----------------------------------------- |
| `result`  | String | Devuelve el valor asociado a una entrada. |

<div id="output-object-example">
  ### Ejemplo del objeto de salida
</div>

```json lines
{
	"result": "Please enter a valid email address."
}
```
