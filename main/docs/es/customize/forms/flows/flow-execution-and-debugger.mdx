---
description: Describe cómo revisar las ejecuciones de flujos y depurar las ejecuciones de Formularios.
title: Ejecución y depurador
---

Cada vez que se ejecuta un flujo, Auth0 almacena un registro que puedes usar para depurar la ejecución. Esto resulta útil para entender por qué se produjo un error en la ejecución del flujo. Por ejemplo, un flujo puede fallar por la ausencia de propiedades, errores tipográficos, claves API incorrectas, etc.

<div id="review-flow-executions">
  ## Revisar ejecuciones de flujos
</div>

Para revisar una entrada de ejecución de un flujo, sigue estos pasos:

1. Ve a **Auth0 Dashboard &gt; Actions &gt; Formularios** y selecciona **Flows**.
2. Selecciona un flujo de la lista.
3. Selecciona **Executions**.

<Frame>![Dashboard > Actions > Formularios > Flows > Ejecución](/docs/images/cdy7uua7fh8z/448jfjHtgCxfIeDOakuHyd/fba64deda0cf920f399de2190e94bdc9/Flows-execution.png)</Frame>

Cada entrada de ejecución tiene cuatro propiedades:

* **Fecha de ejecución**: La fecha de ejecución del flujo.
* **ID de ejecución**: El ID único de la ejecución del flujo.
* **Estado**: La etapa actual de la ejecución del flujo. Hay cuatro posibles estados de ejecución:

  * **En ejecución**: La ejecución está en curso.
  * **Failed**: La ejecución falló debido a un error interno, como la falta de configuraciones requeridas en una acción de flujo.
  * **Completed**: La ejecución finalizó sin errores internos.
* **View Details**: Abre la ventana de depuración de la ejecución.

<Frame>![Dashboard > Actions > Formularios > Flows > Executions](/docs/images/cdy7uua7fh8z/1VqWEaVZg5lrXatXe9dbFB/dbcf7edca5aabdfc544f04cb18be4149/Flow-execution-details.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Completed y Failed son estados finales. Cuando una ejecución alcanza cualquiera de estos estados, el flujo se procesa por completo.
</Callout>

<div id="debug-flow-executions">
  ## Depurar ejecuciones de flujos
</div>

Para ayudar a depurar o diagnosticar un flujo, puedes revisar los datos de entrada, salida y error del flujo accediendo a **View Details** de la ejecución.

<Frame>![Dashboard > Actions > formularios > formulario](/docs/images/cdy7uua7fh8z/5aeniTfHXplDyZfwCKBcgK/a2098a476edde76da10a527b0a606e95/Flow-error.png)</Frame>

Para depurar un error que interrumpe la ejecución del flujo, sigue estos pasos:

1. Ve a **Auth0 Dashboard &gt; Actions &gt; formularios**
2. Ubica el flujo vinculado al nodo de paso.

   <Frame>![Dashboard > Actions > formularios > formulario](/docs/images/cdy7uua7fh8z/4LjULYyo0nhtNowm7J5okz/27ea3f8386bb5015d447c48facf6b90f/flow-error-linked.png)</Frame>
3. Selecciona **Edit Flow.**
4. Selecciona **Executions**.
5. Selecciona **View Details** de la ejecución **Failed**.

En el siguiente ejemplo, la vista **Input** de `Update user` muestra un valor nulo para la propiedad `user_id`.

<Frame>![Dashboard > Actions > formularios > Flow > ejecución](/docs/images/cdy7uua7fh8z/6QNYKDqG9AIOEzQkP6Fnnr/db885e146179a0490eddd5ad6cb518dc/Flow-execution-failed.png)</Frame>

La vista **Error** indica que la validación del esquema de la acción falló porque falta el `user_id` obligatorio. El flujo devuelve un error interno que interrumpe la ejecución del flujo.

<Frame>![Dashboard > Actions > formularios > Flow > ejecución > Error](/docs/images/cdy7uua7fh8z/1kaKihtJtv8VvNAyTkW4cC/6b34bcecc8642d7d97aaf2c8dec51ebc/flow-execution-error-view.png)</Frame>

Al revisar la acción **Update user**, se observa un error de sintaxis en el parámetro `User ID`; la sintaxis correcta es `{{context.user.user_id}}`. Una vez corregida la sintaxis, prueba el flujo para confirmarlo. Para obtener más información sobre la sintaxis de las variables, consulta [Variables](/docs/es/customize/forms/variables).

<Frame>![Dashboard > Actions > formularios > Flows](/docs/images/cdy7uua7fh8z/4YLRVNbzK5LjJEQjFXIJXM/5003833784ce5b0d9f63730ed46ff047/Action-typo-error.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si un campo está configurado como transitorio o si su valor está enmascarado con una [Helper function](/docs/es/customize/forms/variables#helper-functions), el valor no está disponible para la depuración.

  <Frame>![Dashboard > Actions > formularios > Flows > ejecución](/docs/images/cdy7uua7fh8z/4MVtjnVD50cG63ocN5a458/c0337a94d1f4c21f28d06e28ec59121e/masked-information.png)</Frame>
</Callout>
