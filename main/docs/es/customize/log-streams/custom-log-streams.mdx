description: Aprende a configurar un flujo de registro con un webhook personalizado para transmitir eventos de registro a la URL que elijas.
title: Crear flujos de registro personalizados con webhooks
---

Los webhooks permiten enviar eventos a un servidor web externo. Auth0 ofrece varias integraciones que envían automáticamente eventos a sistemas de gestión y análisis de registros de terceros. Sin embargo, si tu sistema de terceros no es compatible, puedes configurar una integración de webhook personalizada.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Solo puedes suscribir una URL del payload por configuración de webhook, pero puedes usar la misma URL para varios flujos. El payload se entregará en formato JSON lines. Tenlo en cuenta al consumir los registros en la configuración de tu webhook.
</Callout>

<div id="configure-api-endpoint">
  ## Configurar el endpoint de la API
</div>

Crearás una API que expone una sola ruta (como `/api/logs`) y acepta solicitudes `HTTP POST`.

<Warning>
  No se admiten certificados HTTP autofirmados.
</Warning>

<div id="configure-log-stream">
  ## Configurar flujo de logs
</div>

1. Ve a [Dashboard &gt; monitoreo &gt; Streams](https://manage.auth0.com/#/log-streams) y haz clic en **Crear flujo**.

   <Frame>![Dashboard Monitoring Streams](/docs/images/cdy7uua7fh8z/5ZY6sT68DabXNL9Dr4bhss/ff0f7306fcf9d64247a61a48b12a39a9/log_stream.png)</Frame>
2. Selecciona **Custom Webhook** e ingresa un nombre único para tu nuevo flujo.
3. Configura las siguientes opciones:

   <Frame>![Dashboard Monitoring Streams Custom Webhook Settings Tab](/docs/images/cdy7uua7fh8z/5vJVD09aqSF0xEZZFk6wh2/e6469041b2dd1e4f1b142961ca0a12ab/custom-webhook-settings.png)</Frame>

   | Configuración                       | Descripción                                                                                  |
   | ----------------------------------- | -------------------------------------------------------------------------------------------- |
   | **Nombre**                          | Un nombre para mostrar único que permita distinguir esta integración de otras integraciones. |
   | **URL del payload**                 | La URL a la que se envían los payloads de eventos como solicitudes HTTP POST.                |
   | **Token de autorización**           | (Opcional) El valor del encabezado Authorization de la solicitud.                            |
   | **Tipo de contenido**               | El tipo de medio del payload que se enviará al webhook.                                      |
   | **Formato del contenido**           | Recibe datos en formato JSON lines, arreglos u objetos.                                      |
   | **Filtrar por categoría de evento** | Lista de filtros del flujo de logs.                                                          |
   | **Cursor inicial**                  | (Opcional) El día y la hora específicos a partir de los cuales se iniciará el flujo.         |
4. Haz clic en **Guardar**.

<div id="test-configuration">
  ## Probar la configuración
</div>

1. Cambie a la vista **Health** y confirme que el **Estado del flujo** esté **Activo.**
2. Realice una acción que cree un evento de log en el tenant, como un inicio de sesión, un intercambio de tokens o una llamada a la **Auth0 Management API**.
3. Confirme que el evento se haya creado en los registros de su tenant.
4. Confirme que su API recibió una solicitud `HTTP POST` con los datos del evento.
5. Vaya a la vista **Health** y confirme que no se produjo ningún error.

<div id="troubleshoot-webhooks">
  ## Solucionar problemas con webhooks
</div>

Si tu webhook no funciona, puede ser difícil diagnosticar el problema y determinar cuál es la causa. Los webhooks son asíncronos, así que probarlos puede implicar activar el webhook, esperar y luego revisar la respuesta (si es que recibiste una).

Sin embargo, hay alternativas a ese proceso ineficiente que describimos antes. Aunque explicar en detalle cómo solucionar problemas de un webhook en particular queda fuera del alcance de este artículo, aquí tienes algunos pasos que puedes seguir para depurarlo:

1. Revisa [Auth0 Dashboard &gt; Monitoreo &gt; Registros](/docs/es/deploy-monitor/logs) para ver mensajes útiles.
2. Analiza las solicitudes que hace tu webhook con una herramienta como [Mockbin](http://mockbin.io/), [Beeceptor](https://beeceptor.com/) o [RequestBin](https://github.com/RonTt/Requestbin) (autoalojado).
3. Simula solicitudes con cURL o [Postman](https://www.getpostman.com/).
4. Prueba el código de tu webhook localmente con herramientas como [localtunnel](https://github.com/localtunnel/localtunnel) o [ngrok](https://ngrok.com/).
5. Usa una herramienta como [Runscope](https://www.runscope.com/) o [Assertible](https://assertible.com/) para observar todo el flujo de ejecución.
