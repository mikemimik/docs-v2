Puedes configurar un proveedor telefónico personalizado para mensajes SMS usando un [número de teléfono como identificador de usuario](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers). Un proveedor telefónico personalizado usa [Actions](/docs/es/customize/actions/actions-overview) para enviar los mensajes SMS al proveedor de telefonía que elijas al verificar números de teléfono y restablecer contraseñas.

Quizá quieras configurar un proveedor telefónico personalizado para los siguientes casos de uso:

* Usar distintos proveedores de telefonía según el contexto de la aplicación, lo que puede incluir diferentes regiones geográficas donde los precios de los mensajes pueden variar significativamente entre proveedores.
* Usar distintos proveedores de telefonía según el contexto de la organización, ya que algunos clientes B2B pueden requerir proveedores específicos.

Puedes configurar un proveedor telefónico personalizado con el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="configure-a-custom-phone-provider-with-the-auth0-dashboard-or-management-api">
  ## Configurar un proveedor telefónico personalizado con Auth0 Dashboard o Management API
</div>

Para configurar un proveedor telefónico personalizado, debe usar el trigger de Action `custom-phone-provider`, que no está disponible en el Actions Editor de Auth0. Para obtener más información, consulte [Action de ejemplo de custom-phone-provider](#example-custom-phone-provider-action).

<Tabs>
  <Tab title="Auth0 Dashboard">
    #### Agregar un proveedor telefónico personalizado con Auth0 Dashboard

    Para configurar un nuevo proveedor telefónico personalizado en Auth0 Dashboard:

    1. Vaya a [Branding &gt; Phone Provider](https://manage.auth0.com/dashboard/#/phone/templates/phone/provider). Esto le llevará a la página Phone Message Provider. De forma predeterminada, está seleccionado el proveedor telefónico de Twilio.
    2. Seleccione la opción de proveedor telefónico **Custom** en la sección Phone Provider.
    3. Seleccione el **método de entrega:**

       1. Texto: El código de verificación se enviará a sus usuarios por SMS.
       2. Voz: El código de verificación se enviará a sus usuarios mediante un mensaje de voz.
    4. En **Configuración del proveedor**, escriba su código de Actions para configurar un proveedor telefónico personalizado:

       1. Consulte la documentación de su proveedor para entender cómo enviar mensajes mediante su API.
       2. Agregue los secrets necesarios para autenticarse con la API.
       3. (Opcional) Puede editar la Action con la interfaz completa del editor de Actions haciendo clic en el botón **Edit in Expanded Editor**. Después de editar su Action en el editor de Actions, **Save** y **Deploy** la Action desde esa interfaz. Haga clic en el enlace **Back to Phone Provider** en la parte superior de la página del editor de Actions para volver a la página Phone Message Provider. Tendrá que volver a seleccionar la opción de proveedor telefónico **Custom** en la sección Phone Provider.
    5. Haga clic en el botón **Save** al final de la página.

    Al hacer clic en **Save**, la Action se guarda y se implementa automáticamente. Para probar la configuración del proveedor telefónico personalizado antes de usarla en un entorno de producción, haga clic en **Enviar mensaje de prueba**.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      No podrá hacer clic en **Enviar mensaje de prueba** para probar su proveedor telefónico personalizado hasta después de guardar la configuración de su proveedor telefónico personalizado.
    </Callout>

    #### Editar un proveedor telefónico personalizado con Auth0 Dashboard

    Para editar un proveedor telefónico personalizado existente en Auth0 Dashboard:

    1. Vaya a [Branding &gt; Phone Provider](https://manage.auth0.com/dashboard/#/phone/templates/phone/provider). Esto le llevará a la página Phone Message Provider.
    2. Modifique el código de la Action en la página Phone Message Provider según sea necesario.

       1. (Opcional) Puede editar la Action con la interfaz completa del editor de Actions haciendo clic en el botón **Edit in Expanded Editor**. Después de editar su Action en el editor de Actions, **Save** y **Deploy** la Action desde esa interfaz. Luego, haga clic en el enlace **Back to Phone Provider** en la parte superior de la página del editor de Actions.
    3. Haga clic en el botón **Save** al final de la página.

    Al hacer clic en **Save**, la Action se guarda y se implementa automáticamente. Para probar la configuración del proveedor telefónico personalizado antes de usarla en un entorno de producción, haga clic en **Enviar mensaje de prueba**.
  </Tab>

  <Tab title="Management API">
    #### Agregar un proveedor telefónico personalizado con Management API

    También puede configurar un proveedor telefónico personalizado con la Management API. Este proceso tiene varios pasos:

    1. Cree una [Action](https://auth0.com/docs/api/management/v2/actions/post-action) y establezca `supported_triggers.id` en `custom-phone-provider`.

       1. Consulte la documentación de su proveedor para entender cómo enviar mensajes mediante su API.
       2. Agregue los secrets necesarios para autenticarse con la API.
    2. [Implemente](https://auth0.com/docs/api/management/v2/actions/post-deploy-action) la Action.
    3. Establezca el [phone provider](https://auth0.com/docs/api/management/v2/branding/create-phone-provider) en `custom`.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      La Action que cree con el trigger `custom-phone-provider` no aparecerá en la biblioteca de Actions.
    </Callout>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si no hay ninguna Action implementada e intenta configurar un proveedor `custom`, Auth0 devolverá un error con `"errorCode":"custom_email_provider_deployed_action_not_found"`.
    </Callout>

    #### Editar un proveedor telefónico personalizado con la Management API

    También puede editar un proveedor telefónico personalizado con la Management API.

    1. (Opcional) Si no conoce el Action ID específico, puede [obtener la lista de actions](https://auth0.com/docs/api/management/v2/actions/get-actions) mediante la Management API. La respuesta incluirá un objeto Actions. La Action utilizada para el proveedor telefónico personalizado puede identificarse por `supported_triggers.id`, que debe establecerse en `custom-phone-provider`.
    2. [Actualice la Action](https://auth0.com/docs/api/management/v2/actions/patch-action) según sea necesario.
    3. [Implemente](https://auth0.com/docs/api/management/v2/actions/post-deploy-action) la Action.
  </Tab>
</Tabs>

Un ejemplo de un proveedor telefónico personalizado que puedes configurar es [Twilio Verify](https://www.twilio.com/docs/verify). Para obtener más información, consulta [Configurar un proveedor telefónico personalizado con Twilio Verify](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/configure-a-custom-phone-provider-with-twilio-verify).

<div id="example-custom-phone-provider-action">
  ## Ejemplo de la Action custom-phone-provider
</div>

El trigger `custom-phone-provider` es diferente del trigger `send-phone-message`, que no debe usarse para configurar un proveedor telefónico personalizado. Para obtener más información, consulta lo siguiente:

* [Triggers de Action: objeto de API de custom-phone-provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/action-triggers-custom-phone-provider-api-object)
* [Triggers de Action: objeto de evento de custom-phone-provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/actions-triggers-custom-phone-provider-event-object)

El siguiente ejemplo de código es una Action `custom-phone-provider` con la funcionalidad de una Action `send-phone-message`:

```javascript lines expandable
/**
* Controlador que se ejecuta al enviar una notificación telefónica*
* @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
* @param {CustomPhoneProviderAPI} api - Métodos y utilidades para modificar el comportamiento del envío de una notificación telefónica.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {

  if (event.notification.message_type.startsWith('otp')) {
    const body = {
      from: event.notification.from,
      action: event.notification.message_type === 'otp_verify' ? 'second-factor-authenticator' : 'enrollment',
      language: event.notification.locale,
      recipient: event.notification.recipient,
      message_type: event.notification.delivery_method,
      text: event.notification.as_text,
      code: event.notification.code,
    };

    const response = await fetch(event.secret.SERVICE_URL, {
      method: 'POST',
      headers: {
        authorization: `Bearer ${event.secrets.TOKEN}`,
      },
      body: JSON.stringify(body),
    });

    console.log(response);
  }

  return;
};
```
