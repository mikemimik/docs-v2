title: Configurar un proveedor telefónico personalizado con Terraform
---

Puede configurar un [proveedor telefónico personalizado](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) con Terraform Auth0 provider. Terraform Auth0 provider se usa para interactuar con la [Auth0 Management API](https://auth0.com/docs/api/management/v2) a fin de configurar un tenant de Auth0. Para obtener más información, consulte la documentación de Terraform sobre [Auth0 Provider](https://registry.terraform.io/providers/auth0/auth0/latest/docs).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Si usa Terraform para configurar un proveedor telefónico personalizado, elimine el proveedor [telefónico](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) personalizado existente de su tenant de Auth0.
  * Auth0 tiene un límite máximo de una Action para las Actions asociadas a los siguientes triggers: custom-phone-provider y custom-email-provider.
</Callout>

Los siguientes pasos describen cómo configurar un proveedor telefónico personalizado con Terraform:

1. Cree una [`auth0_action`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/phone_provider):

```terraform lines
resource "auth0_action" "custom_phone_provider" {
  name    = "Custom Phone Provider"
  runtime = "node22"
  deploy  = true
  code    = <<-EOT
    /**
    * Manejador que se ejecuta al enviar una notificación telefónica
    * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
    * @param {CustomPhoneProviderAPI} api - Métodos y utilidades para modificar el comportamiento del envío de una notificación telefónica.
    */
    exports.onExecuteCustomPhoneProvider = async (event, api) => {
      // El código va aquí
      return;
    };
  EOT
  supported_triggers {
    id      = "custom-phone-provider"
    version = "v1"
  }
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para vincular un `auth0_action` de Terraform a un `auth0_trigger_action`, establece `deploy = true` en `auth0_action`. Para obtener más información, consulta la documentación de Terraform sobre [auth0&#95;action](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/action).
</Callout>

2. Vincula el `auth0_action` al `auth0_trigger_action` de custom-phone-provider:

```terraform lines
resource "auth0_trigger_action" "custom_phone_provider" {
  trigger = "custom-phone-provider"
  actions {
    id           = auth0_action.custom_phone_provider.id
    display_name = auth0_action.custom_phone_provider.name
  }
  depends_on = [
    auth0_action.custom_phone_provider
  ]
}
```

3. Configura el recurso `auth0_phone_provider` para que use `auth0_action`:

```terraform lines
resource "auth0_phone_provider" "custom_phone_provider" {
  depends_on = [auth0_trigger_actions.custom_phone_provider] # Asegúrese de que la Action se cree primero con `custom-phone-provider` como supported_triggers
  name       = "custom"                         # Indica una implementación personalizada
  disabled   = false                            # Deshabilita el proveedor telefónico predeterminado
  configuration {
    delivery_methods = ["text", "voice"]
  }
  credentials {}
},
```

Para obtener más información, consulta la documentación de Terraform sobre [auth0&#95;phone&#95;provider](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/phone_provider).

<div id="troubleshoot">
  ## Solucionar problemas
</div>

**Escenario: el proveedor telefónico personalizado configurado con Terraform no se está aplicando, y el <Tooltip tip="Auth0 Dashboard: El principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> muestra una Action diferente.**

Es posible que tenga varias Actions de proveedor personalizado con `deploy: true`. Auth0 permite un máximo de una Action entre las Actions asociadas a los siguientes triggers: `custom-phone-provider` y `custom-email-provider`.

* Si se encuentra en este escenario, le recomendamos [listar sus Actions](https://auth0.com/docs/api/management/v2/actions/get-actions) para identificar y eliminar cualquier duplicado.
* Si se encuentra en este escenario después de administrar sus proveedores personalizados mediante varios métodos (Terraform, Auth0 Dashboard, <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> y/o Auth0 SDKs), es aconsejable restablecer la configuración del proveedor personalizado. Le recomendamos evitar el uso de varios métodos para administrar su proveedor personalizado.

Para restablecer el proveedor telefónico personalizado, vaya a [Auth0 Dashboard &gt; Branding &gt; Phone Provider](https://manage.auth0.com/#/phone/templates/phone/provider). Luego, seleccione el botón **Reset** en Provider Configuration para restaurar su proveedor personalizado a su configuración predeterminada.
