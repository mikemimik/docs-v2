<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de tipo string en la caché en la clave especificada.

Los valores almacenados en esta caché se limitan al trigger en el que
se establecen. Están sujetos a los [Límites de la caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta manera tendrán una vigencia de *hasta* los valores
`ttl` o `expires_at` especificados. Si no se especifica una vigencia, se usará de forma predeterminada
una vigencia de 15 minutos. La vigencia no puede superar la duración máxima
indicada en [Límites de la caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Esta caché está diseñada para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, aunque sigan dentro de la vigencia indicada.

<ResponseField name="key" type="string">
  La clave del registro que se almacenará.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se almacenará.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento de la caché.

  <Expandable title="propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora de expiración absoluta en milisegundos desde la época Unix.
      Aunque los registros en caché pueden eliminarse antes, nunca

      *Nota*: Este valor no debe proporcionarse si también se proporcionó un valor
      para `ttl`. Si se proporcionan ambas opciones, se usará
      la expiración más próxima de las dos.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada de caché en milisegundos.
      Aunque los valores en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor `ttl` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó un valor
      para `expires_at`. Si se proporcionan ambas opciones, se usará
      la expiración más próxima de las dos.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="apinotification">
  ## `api.notification`
</div>

Indica si el evento de notificación debe reintentarse o descartarse.
Consulta cada uno de estos métodos para obtener más información sobre su comportamiento real.
Si se realizan varias llamadas, solo se tendrá en cuenta la última.

<div id="apinotificationdropreason">
  ### `api.notification.drop(reason)`
</div>

Al llamarlo, el evento de notificación se considera fallido, sin posibilidad de recuperación:
Registraremos un error para este evento, pero no volveremos a enviarlo a la Action en el futuro.
Si necesitas que este evento de notificación se vuelva a intentar, considera llamar a retry en su lugar.

<ResponseField name="reason" type="string">
  este motivo formará parte de la entrada de registro, lo que te ayudará a analizar mejor el error. Ten en cuenta que este campo está limitado a 1024 caracteres y se truncará si supera esa longitud.
</ResponseField>

<div id="apinotificationretryreason">
  ### `api.notification.retry(reason)`
</div>

Al invocarse, el evento de notificación se considera fallido, pero recuperable:
Registraremos un error para este evento, pero lo reintentaremos hasta 5 veces en los próximos minutos.
Si considera que este evento de notificación no debe reintentarse, llame a drop en su lugar.

<ResponseField name="reason" type="string">
  este motivo formará parte de la entrada de registro, lo que le ayudará a analizar mejor el error. Tenga en cuenta que este campo está limitado a 1024 caracteres y se truncará si supera ese límite.
</ResponseField>
