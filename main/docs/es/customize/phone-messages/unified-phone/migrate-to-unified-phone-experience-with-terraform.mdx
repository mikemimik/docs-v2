---
title: Migrar a Unified Phone Experience con Terraform
description: Aprende a migrar de la configuración telefónica heredada de Guardian a Unified Phone Experience con Auth0 Terraform Provider.
---

Migra tu tenant de la configuración telefónica heredada de [Auth0 Guardian](/docs/es/secure/multi-factor-authentication/auth0-guardian) a [Unified Phone Experience (UPE)](/docs/es/customize/phone-messages/unified-phone/configure-unified-phone) con Auth0 [Terraform Provider](https://registry.terraform.io/providers/auth0/auth0/latest/docs).

Después de la migración, todas las [notificaciones telefónicas de autenticación multifactor (MFA)](/docs/es/customize/phone-messages/unified-phone/use-auth0s-unified-phone-experience-for-multi-factor-authentication) se envían a través de un único proveedor telefónico a nivel del tenant.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de comenzar la migración, necesitas:

* [Auth0 Terraform Provider v1.49.0](https://registry.terraform.io/providers/auth0/auth0/latest) o una versión posterior.
* Si quieres configurar un proveedor telefónico personalizado, crea y despliega una Auth0 Action [`custom-phone-provider`](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) para encargarse de la entrega de mensajes.

<div id="enable-the-phone-provider-resource-and-remove-legacy-configuration">
  ## Habilitar el recurso del proveedor telefónico y eliminar la configuración heredada
</div>

Para habilitar el recurso `auth0_phone_provider` de Terraform y limpiar la configuración heredada:

* [Establecer el indicador `phone_consolidated_experience` en `true`](#set-the-phone-consolidated-experience-flag-to-true)
* [Agregar el proveedor telefónico a nivel del tenant](#add-the-tenant-level-phone-provider)
* [Agregar el recurso `auth0_branding_phone_notification_template` (opcional)](#add-the-auth0-branding-phone-notification-template-resource-optional)
* [Limpiar el bloque `phone` del recurso `auth0_guardian`](#clean-up-the-auth0-guardian-resource-phone-block)
* [Planificar y aplicar](#plan-and-apply)

<div id="set-the-consolidated-experience-flag">
  ### Configura el indicador de experiencia consolidada
</div>

Para asegurarte de que tu tenant de Auth0 habilite UPE y dirija todas las notificaciones telefónicas de MFA a través del proveedor unificado, edita el recurso [`auth0_tenant`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/tenant) y establece el indicador `phone_consolidated_experience` en `true`.

```hcl wrap lines
resource "auth0_tenant" "main" {
    # ... otras configuraciones del tenant ...

    phone_consolidated_experience = true
}
```

<div id="add-the-tenant-level-phone-provider">
  ### Agrega el proveedor telefónico a nivel del tenant
</div>

Crea un proveedor telefónico a nivel del tenant con el recurso de Terraform [`auth0_phone_provider`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/phone_provider) para gestionar la entrega en todos los flujos basados en teléfono dentro de Unified Phone Experience.

<Tabs>
  <Tab title="Twilio">
    ```hcl wrap lines
    resource "auth0_phone_provider" "default" {
        name = "twilio"

        configuration {
            delivery_methods = ["text", "voice"]
            default_from     = "YOUR_DEFAULT_PHONE_NUMBER"
            sid              = var.twilio_sid
            mssid            = var.twilio_messaging_service_sid # Opcional
        }

        credentials {
            auth_token = var.twilio_auth_token
        }
    }
    ```

    Para enviar solo SMS, establece `delivery_methods` en `["text"]`.
  </Tab>

  <Tab title="Custom (Actions)">
    ```hcl wrap lines
    resource "auth0_phone_provider" "default" {
        name = "custom"

        configuration {
            delivery_methods = ["text"]
        }
    }
    ```

    Para un [proveedor telefónico personalizado](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/phone_provider), Auth0 espera que la Action [`custom-phone-provider`](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) se haya desplegado antes de ejecutar `terraform apply`. La Action se encarga de la entrega.
  </Tab>
</Tabs>

<div id="add-the-auth0_branding_phone_notification_template-resource-optional">
  ### Añade el recurso auth0_branding_phone_notification_template (opcional)
</div>

Si usas `enrollment_message` o `verification_message` en el bloque `phone` heredado de Guardian para personalizar el texto del mensaje del código de un solo uso (OTP), puedes replicar ese comportamiento con el recurso [`auth0_branding_phone_notification_template`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/branding_phone_notification_template).

```hcl wrap lines
resource "auth0_branding_phone_notification_template" "default" {
    # Consulta la documentación del Auth0 Terraform Provider para conocer las variables de plantilla disponibles.
}
```

<div id="clean-up-the-auth0-guardian-resource-phone-block">
  ### Limpia el bloque `phone` del recurso Auth0 Guardian
</div>

Elimina todos los atributos heredados específicos del proveedor del bloque `phone` de [`auth0_guardian`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/guardian).

Después de la limpieza, el bloque debe contener únicamente los atributos `enabled` y `message_types`, además de un bloque `depends_on` que haga referencia a los recursos `auth0_tenant.main` y `auth0_phone_provider.default`. Esto aplica la configuración de UPE y del proveedor de teléfono antes del bloque `phone` heredado de Guardian.

**Antes:**

```hcl wrap lines
resource "auth0_guardian" "default" {
    phone {
        enabled       = true
        provider      = "twilio"
        message_types = ["sms"]

        options {
            enrollment_message   = "Your enrollment code is {{code}}"
            verification_message = "Your verification code is {{code}}"
        }

        twilio_config {
            sid                   = var.twilio_sid
            auth_token            = var.twilio_auth_token
            from                  = "YOUR_DEFAULT_PHONE_NUMBER"
            messaging_service_sid = var.twilio_messaging_service_sid
        }
    }
}
```

**Después:**

```hcl wrap lines
resource "auth0_guardian" "default" {
    phone {
        enabled       = true
        message_types = ["sms"]
    }
    depends_on = [auth0_tenant.main, auth0_phone_provider.default]
}
```

<div id="plan-and-apply">
  ### Planificar y aplicar
</div>

Revisa el plan detenidamente antes de aplicar los cambios. Deberías ver:

* `auth0_phone_provider.default` creado
* `auth0_guardian.default` actualizado (atributos eliminados)
* `auth0_tenant.main` actualizado (`phone_consolidated_experience = true`)
* `auth0_branding_phone_notification_template.default` creado (si se agregó)

```bash wrap lines
terraform plan
terraform apply
```

<div id="troubleshooting">
  ## Solución de problemas
</div>

<div id="403-forbidden-errors-during-terraform-apply">
  ### Errores 403 Forbidden durante terraform apply
</div>

Este error suele significar que `phone_consolidated_experience` ya estaba configurado como `true` cuando Terraform intentó gestionar los atributos telefónicos heredados de Guardian. Asegúrate de que el bloque [`depends_on`](#clean-up-the-auth0-guardian-resource-phone-block) haga referencia a `auth0_tenant.main` y `auth0_phone_provider.default` para que Terraform los aplique en el orden correcto.

<div id="409-conflict-error-message-type-is-not-supported-by-the-configured-phone-provider">
  ### Error 409 Conflict: el tipo de mensaje no es compatible con el proveedor telefónico configurado
</div>

Este error significa que el valor `message_types` del recurso [`auth0_guardian`](#clean-up-the-auth0-guardian-resource-phone-block) no es compatible con el proveedor configurado. Asegúrate de que los `delivery_methods` del proveedor incluyan los `message_types` solicitados. Usa `message_types: ["sms"]` para la entrega `text` o `message_types: ["voice"]` para la entrega `voice`.

<div id="terraform-apply-error-provider-or-options-cannot-be-specified">
  ### Error de Terraform apply: no se pueden especificar `provider` ni `options`
</div>

El recurso heredado `auth0_guardian` todavía tiene un atributo `provider` o un bloque `options` mientras UPE está habilitado. Elimina `provider` y `options` de la [configuración](#clean-up-the-auth0-guardian-resource-phone-block) para que el bloque phone contenga solo las opciones `enabled` y `message_types`.

<div id="custom-phone-provider-not-delivering-messages">
  ### El proveedor telefónico personalizado no envía mensajes
</div>

Verifica que la Auth0 Action con el trigger [custom-phone-provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) exista y esté implementada en Auth0 antes de ejecutar `terraform apply`.

<div id="otp-messages-not-arriving-after-migration">
  ### Los mensajes de OTP no llegan después de la migración
</div>

Confirma que las `credentials` de `auth0_phone_provider` sean correctas y que `delivery_methods` incluya `"text"` para SMS. Si antes usabas un SID de Messaging Service, asegúrate de que `mssid` esté configurado en el nuevo proveedor.
