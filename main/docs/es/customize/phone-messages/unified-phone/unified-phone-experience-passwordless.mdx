<Card title="Antes de empezar">
  Para migrar a Unified Phone Experience, debes:

  * Tener habilitadas y configuradas en tu tenant las notificaciones por SMS, voz o SMS+voz de la Autenticación multifactor (MFA), o
  * Tener habilitado y configurado Passwordless SMS en tu tenant

  Si no has configurado ni MFA ni Passwordless, lee [Configura Unified Phone Experience de Auth0](/docs/es/customize/phone-messages/unified-phone/configure-unified-phone) para empezar desde el principio.

  Si tienes integraciones de MFA y Passwordless con distintos proveedores, debes usar un [Custom Phone Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider).

  Si usas Phone as ID, ya utilizas automáticamente Unified Phone Experience y no necesitas realizar ninguna acción adicional.
</Card>

Unified Phone Experience centraliza toda la configuración de tus proveedores telefónicos en un único lugar dentro de tu tenant. No tendrás que configurar un proveedor telefónico varias veces para distintos factores telefónicos de autenticación. Para obtener más información sobre los factores telefónicos de <Tooltip tip="Passwordless: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>, lee [Autenticación passwordless con SMS](/docs/es/authenticate/passwordless/authentication-methods/sms-otp).

Si ya tienes un proveedor telefónico configurado para Passwordless, puedes migrar tu configuración a Unified Phone Experience y aplicarla a nivel de tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no informa al usuario final de si un mensaje telefónico se ha enviado correctamente con el proveedor a nivel de tenant. Para solucionar problemas, revisa los [registros del tenant](/docs/es/deploy-monitor/logs).
</Callout>

<div id="update-your-phone-provider">
  ## Actualiza tu proveedor telefónico
</div>

Para usar Unified Phone Experience, debes:

* Verificar la configuración actual
* Configurar tu proveedor:

  * Configurar Twilio
  * Configurar un proveedor telefónico personalizado
* Actualizar los métodos de entrega

<div id="verify-current-settings">
  ### Verificar la configuración actual
</div>

1. Para verificar la configuración actual de Passwordless, ve a [**Auth0 Dashboard &gt; Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless).
2. Selecciona **Phone Message**.
3. Si puedes usar Unified Phone Experience, verás un banner que te indicará que configures un proveedor a nivel de tenant:

<Frame>![Dashboard > Authentication > Passwordless > SMS](/docs/images/cdy7uua7fh8z/7piF5GX9dZL0EXC8SRtMzE/41737a34205541cee46cdbf364401bbc/2025-02-12_15-56-26.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si debes usar la experiencia heredada para configurar tu proveedor telefónico y no aparece el selector para usar un proveedor a nivel de tenant, contacta al [equipo de soporte de Auth0](/docs/es/troubleshoot/customer-support).
</Callout>

<div id="configure-twilio-as-a-tenant-level-provider">
  ### Configura Twilio como proveedor a nivel de tenant
</div>

Si usas Twilio como proveedor telefónico para Passwordless, debes agregar el SID de Twilio y el AuthToken a Unified Phone Experience. Puedes copiar el SID de Twilio de tu configuración actual, pero tendrás que obtener el valor del AuthToken desde Twilio.

1. Ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona y configura tu proveedor:

   <Frame>![Branding > Phone Provider > Twilio](/docs/images/cdy7uua7fh8z/6TESTIdmhUL0LEitACadIK/fcdc34e18a6b66b47370f6d754c085b5/2025-02-21_12-49-53.png)</Frame>

   1. Elige Twilio como proveedor telefónico.
   2. Elige los métodos de entrega.
   3. Ingresa la configuración de Twilio.
3. **Guardar** los cambios.
4. Selecciona **Enviar mensaje de prueba** para probar el proveedor a nivel de tenant antes de usarlo en un entorno de producción.
5. Habilita un proveedor a nivel de tenant.

   1. Ve a [**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless).
   2. Selecciona **Usar proveedor de mensajería a nivel de tenant**

      <Warning>
        Si no aparece ningún interruptor en tu tenant, ya estás usando Unified Phone Experience.
      </Warning>
6. **Guardar** los cambios.

<div id="create-a-new-custom-phone-provider-action-for-passwordless">
  ### Crea una nueva Action custom-phone-provider para Passwordless
</div>

Unified Phone Experience no utiliza el [gateway de SMS personalizado](/docs/es/authenticate/passwordless/authentication-methods/use-sms-gateway-passwordless) que configuraste para Passwordless SMS. Puedes usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para configurar un proveedor telefónico personalizado con Auth0 Actions. Para obtener más información, consulta [Configure a Custom Phone Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider).

<div id="configure-passwordless-custom-phone-provider">
  ##### Configurar el custom-phone-provider de Passwordless
</div>

Para configurar la opción de teléfono de tu gateway de SMS personalizado mediante la Action del proveedor telefónico:

1. Ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona tu proveedor:

   1. Elige **Custom**.
   2. En **Provider Configuration**, agrega tu nueva Action `custom-phone-provider`, que reenvía la notificación de Passwordless a un gateway de SMS personalizado.
3. **Save** los cambios.
4. Selecciona **Enviar mensaje de prueba** para probar el proveedor a nivel de tenant antes de usarlo en un entorno de **Production**.
5. Ve a [**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless) para habilitar **Usar proveedor de mensajería a nivel de tenant**.
6. **Save** los cambios.

<div id="custom-sms-gateway-to-custom-phone-provider-example">
  ##### Ejemplo de gateway de SMS personalizado para custom-phone-provider
</div>

```javascript lines expandable
const request = require('<favorite_http_client>');
/**
* Manejador que se ejecuta al enviar una notificación telefónica
* 
@param
 {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
* 
@param
 {CustomPhoneProviderAPI} api - Métodos y utilidades para modificar el comportamiento del envío de una notificación telefónica.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {
  const notificationType = event.notification.message_type;
  switch (notificationType) {
    case '<otp_verify>': {
      const req = {
        url: event.secrets.GATEWAY_URL,
        method: 'POST',
        headers: {
          'User-Agent': 'Auth0'
        },
        body: {
          recipient: event.notification.recipient,
          body: event.notification.as_text, // o event.notification.as_voice
          sender: event.notification.from, // TODO: falta el remitente, hay que verificar si queremos agregarlo.
          req: event.request || {}, // si no está habilitado, llegará como undefined
        }
      }
      const response = await request(req);
      console.log(response.statusCode);
      return
    }
    default: {
      // manejar otras notificaciones de forma habitual
      return;
    }
  }
};
```

<div id="update-delivery-methods">
  ### Actualizar los métodos de entrega
</div>

| Factores telefónicos | Método de entrega actual a nivel de tenant | Método de entrega deseado a nivel de tenant | Acción del usuario                                                                                                                                                         |
| -------------------- | ------------------------------------------ | ------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SMS                  | SMS                                        | SMS + Voz                                   | Ninguna                                                                                                                                                                    |
| SMS                  | SMS                                        | Voz                                         | El usuario debe actualizar los factores telefónicos de Passwordless para que sean un subconjunto de los métodos de entrega habilitados para el proveedor a nivel de tenant |
| SMS                  | SMS + Voz                                  | Voz                                         | El usuario debe actualizar los factores telefónicos de Passwordless para que sean un subconjunto de los métodos de entrega habilitados para el proveedor a nivel de tenant |
| SMS                  | SMS + Voz                                  | SMS                                         | Ninguna                                                                                                                                                                    |

<div id="disable-phone-provider">
  ## Deshabilitar el proveedor telefónico
</div>

Si configuras Unified Phone Experience y luego la deshabilitas, las integraciones existentes con tu proveedor telefónico, incluida Passwordless, podrían dejar de funcionar.
