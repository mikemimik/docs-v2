<Card title="Antes de empezar">
  Si usas [autenticación Passwordless SMS](/docs/es/authenticate/passwordless/authentication-methods/use-sms-gateway-passwordless), [notificaciones de SMS y voz de MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) o Phone as ID, necesitas configurar un proveedor telefónico con Unified Phone Experience.

  Si necesitas distintos proveedores para MFA, Passwordless y/o Phone as ID, debes configurar un [proveedor telefónico personalizado](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) usando Auth0 Actions.
</Card>

Unified Phone Experience centraliza todas las configuraciones de proveedores telefónicos en un solo lugar dentro de tu tenant. Esta centralización elimina la necesidad de configurar un proveedor telefónico varias veces para distintos factores de autenticación por teléfono.

Si quieres probar las capacidades de mensajería telefónica, necesitas configurar un proveedor telefónico a nivel del tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no proporciona información al usuario final sobre si un mensaje telefónico se envió correctamente con el proveedor a nivel del tenant. Para solucionar problemas, revisa los [logs del tenant](/docs/es/deploy-monitor/logs).
</Callout>

<div id="configure-mfa">
  ## Configurar MFA
</div>

Puedes usar Unified Phone Experience con los métodos de entrega y factores SMS, voz y SMS + voz. Para obtener más información sobre la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor adicional además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> con Auth0, consulta [Multi-Factor Authentication](/docs/es/secure/multi-factor-authentication).

1. En el Auth0 Dashboard, ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Activa el interruptor Phone Message Provider.

   <Frame>![Auth0 Dashboard > Branding > Phone Provider > Enable Phone Message Provider](/docs/images/cdy7uua7fh8z/1k93jvdxDq2FZLiRdvr5aR/d917431cd0265943c5118174a8673118/2025-02-24_16-43-14.png)</Frame>
3. Selecciona tu proveedor de mensajería telefónica.

   1. Para usar Twilio como proveedor telefónico, sigue [Configure Twilio as a Phone Messaging Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-twilio-as-a-phone-messaging-provider).
   2. Para usar un proveedor personalizado, sigue [Configure a Custom Phone Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) para usar Auth0 Actions.
4. Selecciona tus métodos de entrega.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Los factores de MFA deseados deben ser un subconjunto de los métodos de entrega que configures a nivel del tenant.
   </Callout>
5. **Save** los cambios.
6. Cuando termines, ve a [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
7. Selecciona **Phone Message**.
8. Elige los factores de autenticación.
9. **Save** los cambios.

<div id="configure-passwordless">
  ## Configurar Passwordless
</div>

Puede usar Unified Phone Experience con una conexión SMS de <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>. Para obtener más información sobre la autenticación passwordless con Auth0, consulte [Conexiones Passwordless](/docs/es/authenticate/passwordless).

1. En el Auth0 Dashboard, vaya a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Active el toggle Phone Message Provider.

   <Frame>![Auth0 Dashboard > Branding > Phone Provider > Enable Phone Message Provider](/docs/images/cdy7uua7fh8z/1k93jvdxDq2FZLiRdvr5aR/d917431cd0265943c5118174a8673118/2025-02-24_16-43-14.png)</Frame>
3. Seleccione su proveedor de mensajería telefónica.

   1. Para usar Twilio como proveedor telefónico, siga [Configure Twilio as a Phone Messaging Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-twilio-as-a-phone-messaging-provider).
   2. Para usar un proveedor personalizado, siga [Configure a Custom Phone Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) para usar Auth0 Actions.
4. Seleccione el método de entrega.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Si no selecciona **Text** como método de entrega configurado, la integración con Passwordless no funcionará.
   </Callout>
5. **Save** los cambios.
6. Cuando termine, vaya a [**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless).
7. **Save** los cambios.
8. Si usa Twilio:

   1. Vaya a [**Applications &gt; Applications**](https://manage.auth0.com/#/applications).
   2. Seleccione las aplicaciones que quiere usar con su conexión SMS de Twilio para Passwordless.
   3. En **Connections**, habilite la opción SMS (Twilio).
   4. Vuelva a [**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless).
   5. En la pestaña **Applications**, habilite todas las aplicaciones que quiera usar para Passwordless SMS.
   6. **Save** los cambios.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si tiene un caso de uso que requiere la experiencia Legacy de [MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) y [Passwordless](/docs/es/authenticate/passwordless/authentication-methods/use-sms-gateway-passwordless), póngase en contacto con el [equipo de soporte de Auth0](/docs/es/troubleshoot/customer-support).
</Callout>

<div id="test-your-phone-provider">
  ### Pruebe su proveedor telefónico
</div>

Pruebe el proveedor a nivel de tenant antes de usarlo en un entorno de producción:

1. Vaya a [**Branding &gt; proveedor telefónico**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Seleccione **Enviar mensaje de prueba** para probar el proveedor a nivel de tenant antes de usarlo en un entorno de producción.

<div id="configure-phone-templates">
  ## Configurar plantillas de teléfono
</div>

El proveedor telefónico a nivel del tenant usa [la sintaxis Liquid](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) para insertar variables de contexto en el contenido de tus mensajes telefónicos. Para configurar tus plantillas de teléfono, sigue las instrucciones de [Personalizar las plantillas de teléfono](/docs/es/customize/phone-messages/phone-templates/customize).

<div id="disable-phone-provider">
  ## Desactivar el proveedor telefónico
</div>

Si configuras Unified Phone Experience y luego la desactivas, las integraciones existentes con tu proveedor telefónico podrían dejar de funcionar, incluido tu factor telefónico de MFA.
