<Card title="Antes de empezar">
  Para migrar a Unified Phone Experience, debes:

  * Tener habilitadas y configuradas en tu tenant las notificaciones por SMS, voz o SMS+voz de autenticación multifactor (MFA), o
  * Tener Passwordless SMS habilitado y configurado en tu tenant

  Si no has configurado ni MFA ni Passwordless, lee [Configurar Unified Phone Experience de Auth0](/docs/es/customize/phone-messages/unified-phone/configure-unified-phone) para empezar desde cero.

  Si tienes integraciones de MFA y Passwordless con distintos proveedores, debes usar un [Custom Phone Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider).

  Si utilizas Phone as ID, ya estás usando automáticamente Unified Phone Experience de Auth0 y no necesitas realizar ninguna acción adicional.
</Card>

Unified Phone Experience centraliza toda la configuración de tus proveedores telefónicos en un solo lugar dentro de tu tenant. No necesitas configurar un proveedor telefónico varias veces para distintos factores telefónicos de autenticación. Para obtener más información sobre los factores telefónicos para <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de username and password, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>, consulta [Notificaciones de SMS y voz para MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa).

Si ya tienes un proveedor telefónico configurado para [las notificaciones de SMS y voz para Multi-Factor Authentication (MFA)](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa), puedes migrar tu configuración a Unified Phone Experience y aplicarla a nivel del tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no proporciona confirmación al usuario final sobre si un mensaje telefónico se ha enviado correctamente con el proveedor a nivel de tenant. Para solucionar problemas, revisa los [registros del tenant](/docs/es/deploy-monitor/logs).
</Callout>

<div id="update-your-phone-provider">
  ## Actualiza tu proveedor telefónico
</div>

Para usar Unified Phone Experience, debes:

* Verificar la configuración actual
* Configurar tu proveedor:

  * Configurar Twilio
  * Configurar un proveedor telefónico personalizado
* Actualizar tus factores telefónicos y métodos de entrega
* Actualizar los endpoints de <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>
* Actualizar los triggers de Auth0 Actions

<div id="configure-twilio-as-a-tenant-level-provider">
  ### Configurar Twilio como proveedor a nivel de tenant
</div>

Si usas Twilio como proveedor telefónico para la autenticación multifactor, debes agregar el SID de Twilio y el AuthToken a Unified Phone Experience. Puedes copiar el SID de Twilio de tu configuración actual, pero tendrás que obtener el valor del AuthToken en Twilio.

1. Ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona y configura tu proveedor:

   <Frame>![Branding > Phone Provider > Twilio](/docs/images/cdy7uua7fh8z/6TESTIdmhUL0LEitACadIK/fcdc34e18a6b66b47370f6d754c085b5/2025-02-21_12-49-53.png)</Frame>

   1. Elige Twilio como proveedor telefónico.
   2. Elige tus métodos de entrega.
   3. Ingresa la configuración de Twilio.

      <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
        Los factores de MFA deseados deben ser un subconjunto de los métodos de entrega que configures a nivel del tenant.
      </Callout>
3. **Guardar** tus cambios.
4. Selecciona **Enviar mensaje de prueba** para probar el proveedor a nivel de tenant antes de usarlo en un entorno de producción.
5. Activa un proveedor a nivel de tenant.

   1. Para MFA, ve a [**Security &gt; Multi-factor Auth &gt; Phone Message**.](https://manage.auth0.com/#/security/mfa)
   2. Selecciona **Usar proveedor de mensajería a nivel de tenant**.

      <Warning>
        Si no hay ningún interruptor en tu tenant, ya estás usando Unified Phone Experience.
      </Warning>
6. **Guardar** tus cambios.

<div id="configure-a-custom-provider-with-auth0-actions">
  ### Configurar un proveedor personalizado con Auth0 Actions
</div>

Unified Phone Experience usa el trigger de Actions `custom-phone-provider` para las notificaciones por teléfono, en lugar del trigger de Actions `send-phone-message` para las notificaciones personalizadas por teléfono.

No puedes migrar directamente tu trigger de Actions `send-phone-message` al trigger de Actions `custom-phone-provider` porque:

* No existe una correspondencia 1:1 entre los tipos de mensaje y los tipos de notificación.
* Migrar los secrets y las dependencias asociados con el trigger de Actions `send-phone-message` podría generar problemas de seguridad o afectar la experiencia del tenant.
* Actions puede admitir más de un trigger, y migrarlas podría resultar difícil. Por ejemplo, Actions puede admitir tanto los triggers `send-phone-message` como `post-login`.
* Los triggers de Actions `send-phone-message` y `custom-phone-provider` admiten distintos tipos de eventos:

  * `send-phone-message` admite los tipos de mensaje SMS y voz. Para obtener más información, consulta [Triggers de Action: objeto Event de send-phone-message](/docs/es/customize/actions/explore-triggers/mfa-notifications-trigger/send-phone-message-event-object).
  * `custom-phone-provider` admite los tipos de notificación `otp_verify`, `otp_enroll`, `blocked_account`, `change_password` y `password_breach`. Para obtener más información, consulta [Triggers de Actions: objeto Event de custom-phone-provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/actions-triggers-custom-phone-provider-event-object).
* Solo puedes configurar una Action por proveedor de telefonía personalizado. Si ya has configurado una Action para un proveedor de telefonía personalizado, **debes** deshabilitarla antes de asociarla con una nueva Action.

Como resultado, debes convertir tu `send-phone-message` existente en un nuevo `custom-phone-provider` usando las nuevas variables contextuales:

Revise el ejemplo del proveedor telefónico personalizado:

```javascript lines expandable
const { fetch } = require('undici');
/**
* Controlador que se ejecuta al enviar una notificación telefónica
* @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
* @param {CustomPhoneProviderAPI} api - Métodos y utilidades para modificar el comportamiento del envío de una notificación telefónica.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {

  if (event.notification.message_type.startsWith('otp')) {
    const body = {
      from: event.notification.from,
      action: event.notification.message_type === 'otp_verify' ? 'second-factor-authenticator' : 'enrollment',
      language: event.notification.locale,
      recipient: event.notification.recipient,
      message_type: event.notification.delivery_method,
      text: event.notification.as_text,
      code: event.notification.code,
    };

    const response = await fetch(event.secrets['MY_ENDPOINT'], {
      method: 'POST',
      headers: {
        authorization: `Bearer ${event.secrets['MY_SECRET']}`,
      },
      body: JSON.stringify(body),
    });

    console.log(response.status); // 200
    console.log(await response.text()); // ¡cuerpo de la respuesta de la solicitud!
  }

  return;
};
```

<div id="configure-mfa-custom-phone-provider">
  #### Configura MFA custom-phone-provider
</div>

Para continuar y configurar tu trigger de MFA `custom-phone-provider`:

1. Ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona tu proveedor:

   1. Elige **Custom**.

6. **Guarda** los cambios.

<div id="disable-phone-provider">
  ## Deshabilitar el proveedor telefónico
