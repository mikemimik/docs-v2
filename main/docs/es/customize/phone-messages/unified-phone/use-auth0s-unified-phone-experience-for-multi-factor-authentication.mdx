<Card title="Antes de comenzar">
  Para migrar a Unified Phone Experience, debes:

  * Tener habilitadas y configuradas en tu tenant las notificaciones SMS, de voz o SMS+voz de MFA, o
  * Tener Passwordless SMS habilitado y configurado en tu tenant

  Si no has configurado ni MFA ni Passwordless, consulta [Configurar Unified Phone Experience de Auth0](/docs/es/customize/phone-messages/unified-phone/configure-unified-phone) para empezar desde el principio.

  Si tienes integraciones de MFA y de Passwordless con distintos proveedores, debes usar un [Custom Phone Provider](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider).

  Si utilizas Phone as ID, ya estás usando automáticamente Unified Phone Experience de Auth0 y no necesitas realizar ninguna acción adicional.
</Card>

Unified Phone Experience centraliza toda la configuración de tu proveedor telefónico en un solo lugar dentro de tu tenant. No necesitas configurar un proveedor telefónico varias veces para distintos factores telefónicos de autenticación. Para obtener más información sobre los factor telefónico para <Tooltip tip="Multi-factor authentication (MFA): Proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>, consulta [Notificaciones SMS y de voz para MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa).

Si ya tienes configurado un proveedor telefónico para [las notificaciones SMS y de voz de la autenticación multifactor (MFA)](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa), puedes migrar tu configuración a Unified Phone Experience y aplicarla a nivel del tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no informa al usuario final si un mensaje telefónico se envió correctamente con el proveedor a nivel de tenant. Para solucionar problemas, revisa los [logs del tenant](/docs/es/deploy-monitor/logs).
</Callout>

<div id="update-your-phone-provider">
  ## Actualiza tu proveedor de telefonía
</div>

Para usar Unified Phone Experience, debes:

* Verificar la configuración actual
* Configurar tu proveedor:

  * Configurar Twilio
  * Configurar un proveedor de telefonía personalizado
* Actualizar tus factores telefónicos y métodos de entrega
* Actualizar los endpoints de <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>
* Actualizar los triggers de Auth0 Actions

</div>

1. Para verificar la configuración actual de MFA, vaya a [**Auth0 Dashboard &gt; Security &gt; Multi-factor Auth**](https://manage.auth0.com/#/security/mfa).
2. Seleccione **Phone Message**.
3. Los nuevos tenants de Auth0 pueden usar Unified Phone Experience; un banner le indicará que configure un proveedor telefónico del tenant:

   <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/mfa-phone-message-upe.png)</Frame>

* Los tenants existentes anteriores a marzo de 2025 pueden configurar SMS and Voice notifications para Multi-Factor Authentication (MFA) junto con Unified Phone Experience.

  <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/mfa-phone-message-no-upe.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recomienda usar Unified Phone Experience. Si debe usar la experiencia de [SMS and Voice notifications for Multi-Factor Authentication (MFA)](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) para configurar su proveedor telefónico y no aparece el interruptor **Use  Tenant-Level Messaging Provider**, póngase en contacto con el [equipo de soporte de Auth0](/docs/es/troubleshoot/customer-support).

4. Para verificar la configuración de su tenant, vaya a [**Dashboard &gt; Configuración del tenant**](https://manage.auth0.com/#/tenant/advanced).
5. Seleccione **Advanced**.
6. Busque **Unified Phone Experience**.
   <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/tenant-settings-upe.png)</Frame>
7. Active el interruptor para habilitarlo, si es necesario.

<div id="configure-twilio-as-a-tenant-level-provider">
  ### Configurar Twilio como proveedor a nivel de tenant
</div>

Si usas Twilio como proveedor telefónico para la autenticación multifactor, debes agregar el SID de Twilio y el AuthToken a Unified Phone Experience. Puedes copiar el SID de Twilio de tu configuración actual, pero tendrás que obtener el valor del AuthToken de Twilio.

1. Ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona y configura tu proveedor:

   <Frame>![Branding > Phone Provider > Twilio](/docs/images/cdy7uua7fh8z/6TESTIdmhUL0LEitACadIK/fcdc34e18a6b66b47370f6d754c085b5/2025-02-21_12-49-53.png)</Frame>

   1. Elige Twilio como proveedor telefónico.
   2. Elige tus métodos de entrega.
   3. Ingresa la configuración de Twilio.

      <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
        Los factores de MFA deseados deben ser un subconjunto de los métodos de entrega que configures a nivel del tenant.
      </Callout>
3. **Guardar** los cambios.
4. Selecciona **enviar mensaje de prueba** para probar el proveedor a nivel de tenant antes de usarlo en un entorno de Production.
5. Habilita un proveedor a nivel de tenant.

   1. Para MFA, ve a [**Security &gt; Multi-factor Auth &gt; Phone Message**.](https://manage.auth0.com/#/security/mfa)
   2. Selecciona **Usar proveedor de mensajería a nivel de tenant**.

      <Warning>
        Si no aparece ningún control en tu tenant, ya estás usando Unified Phone Experience.
      </Warning>
6. **Guardar** los cambios.

<div id="configure-a-custom-provider-with-auth0-actions">
  ### Configurar un proveedor personalizado con Auth0 Actions
</div>

Unified Phone Experience usa el trigger de Actions `custom-phone-provider` para las notificaciones telefónicas, en lugar del trigger de Actions `send-phone-message` para las notificaciones telefónicas personalizadas.

No puede migrar directamente su trigger de Actions `send-phone-message` al trigger de Actions `custom-phone-provider` porque:

* No existe una correspondencia 1:1 entre los tipos de mensaje y los tipos de notificación.
* Migrar los secrets y las dependencias asociadas con el trigger de Actions `send-phone-message` podría generar problemas de seguridad o afectar la experiencia del tenant.
* Las Actions pueden admitir más de un trigger, lo que puede dificultar la migración. Por ejemplo, las Actions pueden admitir tanto los triggers `send-phone-message` como `post-login`.
* Los triggers de Actions `send-phone-message` y `custom-phone-provider` admiten distintos tipos de evento:

  * `send-phone-message` admite los tipos de mensaje SMS y voz. Para obtener más información, consulte [Action triggers: send-phone-message Event Object](/docs/es/customize/actions/explore-triggers/mfa-notifications-trigger/send-phone-message-event-object).
  * `custom-phone-provider` admite los tipos de notificación `otp_verify`, `otp_enroll`, `blocked_account`, `change_password` y `password_breach`. Para obtener más información, consulte [Actions triggers: custom-phone-provider Event Object](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/actions-triggers-custom-phone-provider-event-object).
* Solo puede establecer una Action por proveedor telefónico personalizado. Si ya estableció una Action para un proveedor telefónico personalizado, **debe** deshabilitarla antes de asociarla con una nueva Action.

Como resultado, debe convertir su `send-phone-message` existente en un nuevo `custom-phone-provider` usando las nuevas variables contextuales:

Revise el ejemplo del proveedor de teléfono personalizado:

```javascript lines expandable
const { fetch } = require('undici');
/**
* Controlador que se ejecuta al enviar una notificación telefónica
* @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
* @param {CustomPhoneProviderAPI} api - Métodos y utilidades para modificar el comportamiento del envío de una notificación telefónica.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {

  if (event.notification.message_type.startsWith('otp')) {
    const body = {
      from: event.notification.from,
      action: event.notification.message_type === 'otp_verify' ? 'second-factor-authenticator' : 'enrollment',
      language: event.notification.locale,
      recipient: event.notification.recipient,
      message_type: event.notification.delivery_method,
      text: event.notification.as_text,
      code: event.notification.code,
    };

    const response = await fetch(event.secrets['MY_ENDPOINT'], {
      method: 'POST',
      headers: {
        authorization: `Bearer ${event.secrets['MY_SECRET']}`,
      },
      body: JSON.stringify(body),
    });

    console.log(response.status); // 200
    console.log(await response.text()); // ¡cuerpo de la respuesta de la solicitud!
  }

  return;
};
```

<div id="configure-mfa-custom-phone-provider">
  #### Configurar MFA custom-phone-provider
</div>

Para continuar y configurar el trigger de MFA `custom-phone-provider`:

1. Ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona tu proveedor:

   1. Elige **Custom**.
   2. En **Configuración del proveedor**, agrega el código traducido de tu Action `send-phone-message` en el editor de código integrado.
3. **Guardar** los cambios.
4. Selecciona **enviar mensaje de prueba** para probar el proveedor a nivel de tenant antes de usarlo en un entorno de Production.
5. Vuelve a [**Security &gt; Multi-factor Auth &gt; Phone Message**](https://manage.auth0.com/#/security/mfa) para habilitar **usar proveedor de mensajería a nivel de tenant**.
6. **Guardar** los cambios.
