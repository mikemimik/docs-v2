Las siguientes secciones ofrecen información sobre cómo probar los Event Streams y gestionar los eventos.

<div id="test-the-event-stream">
  ## Prueba el Event Stream
</div>

Después de crear y habilitar un Event Stream, puedes probar el nuevo flujo. Actualmente, Auth0 no permite enviar eventos a Event Streams deshabilitados.

Para probar tu Event Stream, primero crea un usuario para tu aplicación ejecutando lo siguiente con las credenciales que prefieras:

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    auth0 users create # sigue las indicaciones para crear un usuario de prueba
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -X POST https://$TENANT/api/v2/users \
      -d '{
            "email": "nick-testing-21@example.com",
            "password": "Password123!",
            "connection": "Username-Password-Authentication"
          }'
    ```
  </Tab>
</Tabs>

Una vez creado el usuario, debería activarse el evento `user.created` en tu Event Stream y recibirse una respuesta. En este ejemplo, se usan webhooks para gestionar el Event Stream.

Puedes listar los errores de entrega de eventos de un Event Stream específico ejecutando:

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    auth0 api get event-streams/$EVENT_STREAM_ID/deliveries
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -H "Authorization: Bearer $TOKEN" \
      "https://$TENANT/api/v2/event-streams/${EVENT_STREAM_ID}/deliveries" | jq .
    ```
  </Tab>
</Tabs>

Si se devuelve un arreglo vacío de errores de entrega, tu Event Stream funciona correctamente.

<div id="observability">
  ## Observabilidad
</div>

Para garantizar la confiabilidad de los Event Streams, implemente un proceso de monitoreo que verifique periódicamente si hay errores de entrega y genere alertas cuando sea necesario.

Los errores de entrega pueden deberse a lo siguiente:

* Event Streams deshabilitados.
* Event Streams no disponibles.
* Errores de conexión.

<div id="polling-deliveries-endpoint">
  ### Endpoint de Deliveries para polling
</div>

Consulta el endpoint de Deliveries cada **5 minutos** para comprobar si hay eventos fallidos. Usa la siguiente llamada a la API para obtener las entregas de eventos recientes:

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    auth0 api get event-streams/$EVENT_STREAM_ID/deliveries
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -H "Authorization: Bearer $TOKEN" \
      "https://$TENANT/api/v2/event-streams/${EVENT_STREAM_ID}/deliveries" | jq .
    ```
  </Tab>
</Tabs>

Si se detectan fallos:

* Registra los detalles del evento fallido.
* Envía una alerta al equipo o sistema correspondiente para revisar y diagnosticar el problema.

<div id="recovery">
  ## Recuperación
</div>

La entrega de eventos se reintentará automáticamente. Para obtener más información, consulta [Prácticas recomendadas](/docs/es/customize/events/event-testing-observability-and-failure-recovery#batch-redelivery-of-failed-events).

Si se producen errores, usa la Redelivery API para volver a intentar la entrega de eventos.

<div id="redeliver-a-single-failed-event">
  ### Reentregar un solo evento fallido
</div>

Para volver a intentar la entrega de un evento fallido específico, usa:

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    EVENT_ID=<your event id>
    auth0 api POST event-streams/$EVENT_STREAM_ID/redeliver/$EVENT_ID --data '{}'
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -X POST -H "Authorization: Bearer $TOKEN" \
      "https://<your-domain>/api/v2/event-streams/$EVENT_STREAM_ID/redeliver/$EVENT_ID"
    ```
  </Tab>
</Tabs>

<div id="batch-redelivery-of-failed-events">
  ### Reenvío en lote de eventos fallidos
</div>

Para reintentar todos los eventos fallidos del Event Stream:

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash wrap lines
    auth0 api POST event-streams/$EVENT_STREAM_ID/redeliver --data '{}'
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -X POST -H "Authorization: Bearer $TOKEN" \
      "https://<your-tenant>.auth0.com/api/v2/event-streams/<event_stream_id>/redeliver"
    ```
  </Tab>
</Tabs>

<div id="auto-disable-mechanism">
  ### Mecanismo de deshabilitación automática
</div>

Auth0 deshabilita automáticamente un Event Stream si se producen demasiados fallos.

| Condición               | Medida adoptada                                     |
| ----------------------- | --------------------------------------------------- |
| 500 fallos consecutivos | El Event Stream se deshabilita automáticamente. |
| 5000 fallos en total    | El Event Stream se deshabilita automáticamente. |

* Si un Event Stream se deshabilita automáticamente, **debe** usar la Redelivery API para procesar los eventos fallidos.
* El Event Stream **no puede** volver a habilitarse manualmente hasta que el número total de fallos sea inferior a 5000.
* Una vez que se reduzca el número de fallos, puede volver a habilitar el flujo mediante la API.
