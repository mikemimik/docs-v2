Al suscribirte a eventos y enviarlos a un destino de tu elección mediante flujos de eventos, puedes habilitar varios casos de uso relacionados, entre ellos:

* Enviar correos electrónicos a nuevos clientes para darles la bienvenida o pedirles que verifiquen su dirección de correo electrónico.
* Monitorear los cambios en el ciclo de vida de los usuarios para poder actualizar los sistemas de CRM (gestión de relaciones con clientes) o de facturación.

Puedes crear un flujo de eventos con AWS EventBridge o webhooks. En las secciones siguientes se describe el proceso de Setup para ambas opciones.

<div id="access-management-api-optional">
  ## API de gestión del acceso (opcional)
</div>

Los flujos de eventos pueden configurarse desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip> o con la [Auth0 Management API](https://auth0.com/docs/api/management/v2). Si usa la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, antes de poder configurar un flujo de eventos, debe crear una aplicación de máquina a máquina (M2M) y autenticarse con un token de la Management API. Para obtener más información, consulte [Tokens de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens).

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione **Create Application**.
2. Introduzca un nombre descriptivo para su aplicación y elija **Machine to Machine Applications**. Luego, seleccione **Create**.
3. Seleccione la API a la que desea llamar desde su aplicación. En este caso, use la **Auth0 Management API**.
4. Elija los permisos que desea incluir en el token de acceso de su aplicación y, a continuación, seleccione **Authorize**. Para realizar pruebas, seleccione:

   * `read:event_streams`
   * `create:event_streams`
   * `update:event_streams`
   * `delete:event_streams`
   * `read:event_deliveries`
   * `update:event_deliveries`
   * `create:users`
5. Vaya a la pestaña **Settings** para obtener su **Client ID**, **Client Secret** y **Domain**.
6. Consulte [Obtener tokens de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) para obtener y almacenar su token de acceso.

<div id="aws-eventbridge">
  ## AWS EventBridge
</div>

La siguiente información describe cómo puede crear y habilitar un flujo de eventos con AWS EventBridge.

<div id="eventbridge-prerequisites">
  ### Requisitos previos de EventBridge
</div>

Para usar AWS EventBridge con flujos de eventos, necesitarás lo siguiente:

* Cuenta de AWS

  * Tu cuenta de AWS debe tener permisos para usar EventBridge. Si no tienes una, regístrate en https://aws.amazon.com/eventbridge/.
* Permisos de AWS IAM
* Bus de eventos de AWS EventBridge
* ID y región de la cuenta de AWS

<div id="create-an-event-stream-eventbridge">
  ### Crear un flujo de eventos (EventBridge)
</div>

<Tabs>
  <Tab title="Management API">
    Los flujos de eventos te permiten capturar cambios en tiempo real dentro de tu tenant de Auth0 y enviarlos a un sistema externo para que los procese.

    Antes de configurar un flujo de eventos, debes identificar los [tipos de eventos](/docs/es/customize/events/event-types) que quieres monitorizar. Luego, usarás el ID y la región de tu cuenta de AWS para configurar tu flujo de eventos, como se muestra a continuación.

    Este ejemplo utiliza Auth0 CLI para crear un flujo de eventos que se suscribe al evento `user.created`, que se activa cada vez que se registra un usuario nuevo en tu tenant.

    ```bash wrap lines
    auth0 events create --name ng-demo-eventbridge --type eventbridge --subscriptions "user.created" --configuration '{"aws_account_id":"<your-aws-account-id>","aws_region":"<your-aws-region>"}'
    ```

    Si la llamada se realiza correctamente, devuelve el siguiente JSON con el `id` de tu flujo de eventos. Los flujos de eventos nuevos están habilitados de forma predeterminada.

    ```json lines
    {
      "id": "est_8of6RXoM1997qikH7NS11h",
      "status": "enabled",
      "name": "ng-demo-eventbridge",
      "subscriptions": [
        {
          "event_type": "user.created"
        }
      ],
      "created_at": "2025-01-29T18:08:43.440Z",
      "updated_at": "2025-01-29T18:08:43.440Z",
      "destination": {
        "type": "eventbridge",
        "configuration": {
          "aws_account_id": "<your-aws-account-id>",
          "aws_region": "<your-aws-region>",
          "aws_partner_event_source": "default"
        }
      }
    }
    ```
  </Tab>

  <Tab title="Dashboard">
    Para crear un flujo de eventos en el Auth0 Dashboard con AWS EventBridge:

    1. Ve a **Auth0 Dashboard** &gt; **Event Streams (Early)**.
    2. Selecciona **+Create Event Stream**.
    3. En la lista de tipos de flujo, selecciona **AWS EventBridge**. Esto abrirá el formulario de configuración de tu nuevo flujo de EventBridge.
    4. **Configure Stream Details:** En el formulario de configuración, deberás proporcionar la siguiente información:

       * **Stream Name:**

         Introduce un nombre descriptivo para tu flujo de eventos. Esto te ayudará a identificarlo dentro del Auth0 Dashboard.
       * **AWS Account ID:**

         Introduce el ID de 12 dígitos de la cuenta de AWS a la que quieres que se envíen los eventos de Auth0.
       * **AWS Region:**

         Selecciona la región de AWS específica donde se encuentra tu bus de eventos de EventBridge.
    5. **Select Event Types:** En la sección **Select Event Types**, elige los tipos de eventos de Auth0 específicos que quieres incluir en este flujo. Puedes seleccionar varios tipos de eventos según tus necesidades (p. ej., `user.created`, `user.updated`, `user.deleted`).
    6. **Save Changes:** Una vez que hayas configurado el nombre del flujo, los datos de AWS y seleccionado los tipos de eventos que deseas, haz clic en el botón **Save Changes**.

    Tu nuevo flujo de eventos ya está creado y Auth0 puede comenzar a publicar los tipos de eventos seleccionados en el bus de eventos de AWS EventBridge especificado. Puedes supervisar el estado y administrar tu flujo de eventos desde la página Event Streams (Early) en el Auth0 Dashboard.
  </Tab>

  <Tab title="Terraform">
    <Note>
      Antes de empezar con Terraform, necesitas:

      * Instalar la CLI de Terraform

    ### Configuración de archivos

    Para aprovisionar tu flujo de eventos con Terraform, necesitas un directorio de proyecto bien estructurado. A continuación, se muestra un ejemplo de configuración si lo estás configurando por primera vez:

    ```bash

    auth0-event-streams/
    ├── main.tf             # Core configuration: define los providers y el recurso 'auth0_event_stream'.
    ├── variables.tf        # Declaración de variables: define las entradas como 'aws_account_id' y 'aws_region'.
    └── terraform.tfvars    # Valores de variables: contiene los valores de configuración reales (p. ej., "123456789012").

    ```

    ### Definir variables

    En tu archivo `variables.tf`, declara las variables de entrada necesarias para la configuración de tu flujo:

    ```bash
    variable "aws_account_id" {
      description = "The 12-digit AWS Account ID where the EventBridge event bus resides."
      type        = string
    }

    variable "aws_region" {
      description = "The AWS region (e.g., us-east-1) for the EventBridge event bus."
      type        = string
    }
    ```

    ### Establece los valores de las variables

    En tu archivo `terraform.tfvars`, proporciona los valores reales de tu entorno específico:

    ```bash
    aws_account_id = "<your-aws-account-id>" # p. ej., "123456789012"
    aws_region     = "<your-aws-region>"     # p. ej., "us-east-1"
    ```

    ### Establece variables de entorno

    En tu terminal, configura las tres variables de entorno obligatorias usando las credenciales que copiaste en Access Management API (opcional) más arriba.

    ```bash
    export AUTH0_DOMAIN="your-tenant-name.auth0.com"
    export AUTH0_CLIENT_ID="your_m2m_client_id"
    export AUTH0_CLIENT_SECRET="your_m2m_client_secret"
    ```

    ## Crear un flujo de eventos

    Este ejemplo usa el recurso `auth0_event_stream` para crear un flujo de eventos que se suscribe al evento `user.created`, el cual se activa cada vez que se registra un usuario nuevo en tu tenant. Usa el siguiente ejemplo en tu archivo `main.tf`:

    ```hcl
    terraform {
      required_providers {
        auth0 = {
          source  = "auth0/auth0"
          version = ">= 1.0.0" 
        }
      }
    }

    provider "auth0" {}

    resource "auth0_event_stream" "eventbridge_stream" {
      name             = "ng-demo-eventbridge-tf"
      destination_type = "eventbridge" # Obligatorio
      subscriptions    = ["user.created"]

      eventbridge_configuration {
        aws_account_id = var.aws_account_id
        aws_region     = var.aws_region
      }
    }
    ```

    Si la llamada se realiza correctamente, devuelve el siguiente JSON con el `id` de tu flujo de eventos. Los nuevos flujos de eventos se habilitan de forma predeterminada.

    Una vez que hayas creado y guardado los tres archivos anteriores, estarás listo para crear tu flujo.

    1. Inicializa el proveedor

    ```bash
    terraform init
    ```

    2. Revisa los cambios:

    ```bash
    terraform plan
    ```

    3. Crea el flujo:

    ```bash
    terraform apply
    ```

</Tabs>

<div id="webhooks">
  ## Webhooks
</div>

Como alternativa a AWS EventBridge, puedes usar webhooks para habilitar los flujos de eventos.

Para empezar, primero configura un controlador de webhook para recibir notificaciones en tiempo real cuando se produzca un evento específico. Luego, podrás crear tu flujo de eventos.

Puedes crear un controlador de webhook básico siguiendo las instrucciones que aparecen a continuación, o usar un servicio existente como:

* Vercel
* Inngest

Si decides usar un servicio existente, puedes ir a [Crear un flujo de eventos (webhook)](#create-an-event-stream-webhooks-). De lo contrario, sigue las instrucciones que aparecen a continuación para crear tu propio controlador de webhook básico.

<div id="webhook-prerequisites">
  ### Requisitos previos del webhook
</div>

Asegúrate de tener instalados los siguientes elementos para poder escribir correctamente tu controlador de webhook:

* `node.js`
* `jq`
* `npm`
* `ngrok`

<div id="write-the-webhook-handler">
  ### Escribe el controlador de webhook
</div>

1. Instala `express` en la carpeta `node_modules` y añádelo a las dependencias de tu `package.json`.

2. Instala `dotenv` en tu directorio raíz para usar un archivo `.env` en el que almacenar variables de entorno.

3. Crea un archivo `webhook.js` para recibir el evento `user.created` y almacenarlo en una base de datos.

   ```javascript lines expandable
   const express = require('express');
   const app = express();

   // Middleware de Authorization
   app.use((req, res, next) => {
     const token = req.headers["authorization"];
     if (token !== `Bearer ${API_TOKEN}`) {
       return res.status(401).json({ error: "Unauthorized" });
     }
     next();
   });

   // Endpoint de webhook
   app.post("/webhook", async (req, res) => {
     console.log("Webhook received:", JSON.stringify(req.body, null, 2));

     const eventData = req.body;
     const { id, type, time, data } = eventData;
     const user = data.object;

     try {
       switch (type) {
         case "user.created":
           await handleUserCreated(user, time);
           break;
         case "user.updated":
           await handleUserUpdated(user, time);
           break;
         case "user.deleted":
           await handleUserDeleted(user, time);
           break;
         default:
           await handleDefaultEvent(id, type, time, data);
       }

       console.log(`Webhook event of type '${type}' committed to the database.`);
       res.sendStatus(204);
     } catch (err) {
       console.error("Error processing webhook:", err);
       res.status(500).json({ error: "Internal server error" });
     }
   });

   // Función específica para manejar el evento de creación de usuario
   // En este ejemplo, nos aseguramos de que los usuarios también se creen en nuestra propia base de datos
   async function handleUserCreated(user, time) {
     const { user_id, email, name, nickname, created_at, updated_at } = user;

     const query = `
       INSERT INTO users (user_id, email, name, nickname, created_at, updated_at, raw_user, last_event_processed)
       VALUES ($1, $2, $3, $4, $5, $6, $7, $8)
     `;
     const values = [
       user_id,
       email,
       name,
       nickname,
       created_at,
       updated_at,
       user,
       time,
     ];

     try {
       await getPool().query(query, values);
     } catch (err) {
       if (err.code === "23505") {
         console.error(`Duplicate user_id=${user_id}, skipping insert.`);
       } else {
         console.error(`Database error while creating user_id=${user_id}:`, err);
         throw err;
       }
     }
   }
   ```

4. En la raíz de tu proyecto, crea un archivo `.env` y agrega tu token de API con lo siguiente:

   ```bash lines
   API_TOKEN=`openssl rand -hex 32`
   echo "API_TOKEN=$API_TOKEN" > .env
   ```

5. Inicia el servidor:

   ```bash lines
   node webhook.js
   ```

6. Para probar el webhook, expón tu controlador de webhook con una herramienta como ngrok:

   ```bash lines
   ngrok http 3000
   ```

   Esto proporciona una URL pública para tu controlador de webhook local, por ejemplo:

   ```bash lines
   http://localhost:3000
   ```

    ## Crear un flujo de eventos

    Este ejemplo utiliza el recurso `auth0_event_stream` para crear un flujo de eventos que se suscribe al evento `user.created`, que se activa cada vez que se registra un usuario nuevo en tu tenant. Usa el siguiente ejemplo en tu archivo `main.tf`:

<div id="create-an-event-stream-actions">
  ### Crear un flujo de eventos (Actions)
</div>

<Tabs>
  <Tab title="Management API">
    Los flujos de eventos le permiten capturar cambios en tiempo real dentro de su tenant de Auth0 y enviarlos a un sistema externo para su procesamiento.

    Antes de configurar un flujo de eventos, debe identificar los [tipos de eventos](/docs/es/customize/events/event-types) que desea supervisar. En este ejemplo, se usa Auth0 CLI para crear un flujo de eventos que se suscribe al evento `user.created`, que se activa cada vez que su tenant registra un nuevo usuario.

    ```bash wrap lines
    auth0 event-streams create \
      --name actions-1 \
      --type action \

      --configuration '{"action_id":"385db79f-28a4-43d9-909d-6f5511dd6632"}'
    ```

    Si la llamada se realiza correctamente, devuelve el siguiente JSON con el `id` de su flujo de eventos. Los nuevos flujos de eventos están habilitados de forma predeterminada.

    ```json lines
    {
        "configuration": {
          "action_id": "act_xyz789...",
          "status": "enabled"
      }
    }
    ```
  </Tab>

  <Tab title="Dashboard">
    Para crear un flujo de eventos en el Auth0 Dashboard con Auth0 Actions:

    1. Vaya a **Auth0 Dashboard** &gt; **Event Streams (Early)**.

    2. Seleccione **+Create Event Stream**.                .

    3. En la lista de tipos de flujo, seleccione **+Auth0 Actions**. Esto abre el formulario de configuración de su nuevo flujo de Auth0 Actions.

    4. **Configure Stream Details:** En el formulario de configuración, debe proporcionar la siguiente información:

       * **Stream Name:**

         Introduzca un nombre descriptivo para su flujo de eventos. Esto le ayudará a identificarlo dentro del Auth0 Dashboard.

    5. **Select Event Types:** En la sección **Select Event Types**, elija los tipos de eventos de Auth0 específicos que desea incluir en este flujo. Puede seleccionar varios tipos de eventos según sus necesidades (por ejemplo, `Created`, `Updated`, `Deleted`).

    6. En el **Action Editor**, escriba el código de su controlador. El sistema requiere la función `onExecuteEventStream`. Puede usar el objeto `api` para gestionar estados de éxito o error.

    ```javascript
    /**
     * Controlador que se llamará durante la ejecución de un Event Stream.
     * @param {Event} event - Detalles sobre el Cloud Event.
     * @param {EventStreamAPI} api - Interfaz cuyos métodos pueden usarse para gestionar el evento.
     */
    exports.onExecuteEventStream = async (event, api) => {
      const eventType = event.message.type;
      const eventData = event.message.data;

      console.log(`Received event: ${eventType}`);

      // Lógica personalizada: p. ej., enviar datos a una API externa
      // await axios.post('https://my-crm.com/ingest', eventData);

      // MANEJO DE FALLOS:
      // Para activar un reintento, lance un error o use api.message.fail()
      if (!eventData) {
         // api.message.fail('No data received');
         throw new Error('No data received');
      }

      // ÉXITO:
      // Si la función termina sin errores, el evento se marca como entregado.
    };
    ```
