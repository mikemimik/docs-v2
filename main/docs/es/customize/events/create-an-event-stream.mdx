Al suscribirte a eventos y enviarlos al destino que elijas mediante flujos de eventos, puedes facilitar varios casos de uso relacionados, entre ellos:

* Enviar correos electrónicos a clientes nuevos para darles la bienvenida o pedirles que verifiquen su dirección de correo electrónico.
* Monitorear cambios en el ciclo de vida del usuario para que puedas actualizar sistemas de CRM (gestión de relaciones con clientes) o de facturación.

Puedes crear un flujo de eventos con AWS EventBridge o webhooks. Las secciones a continuación describen el proceso de Setup para ambas opciones.

<div id="access-management-api-optional">
  ## Acceso a la Management API (opcional)
</div>

Los flujos de eventos pueden configurarse desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip> o mediante la [Auth0 Management API](https://auth0.com/docs/api/management/v2). Si utiliza la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, antes de poder configurar un flujo de eventos, debe crear una aplicación de máquina a máquina (M2M) y autenticarse con un token de la Management API. Para obtener más información, consulte [Tokens de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens).

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione **Create Application**.
2. Introduzca un nombre descriptivo para su aplicación y elija **Machine to Machine Applications**. Luego, seleccione **Create**.
3. Seleccione la API que desea usar desde su aplicación. En este caso, use **Auth0 Management API**.
4. Elija los permisos que desea que se emitan como parte del token de acceso de su aplicación y, luego, seleccione **Authorize**. Para fines de prueba, seleccione:

   * `read:event_streams`
   * `create:event_streams`
   * `update:event_streams`
   * `delete:event_streams`
   * `read:event_deliveries`
   * `update:event_deliveries`
   * `create:users`
5. Vaya a la pestaña **Settings** para obtener su **Client ID**, **Client Secret** y **Domain**.
6. Consulte [Obtener tokens de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) para obtener y almacenar su token de acceso.

<div id="aws-eventbridge">
  ## AWS EventBridge
</div>

A continuación, se explica cómo crear y habilitar un flujo de eventos con AWS EventBridge.

<div id="eventbridge-prerequisites">
  ### Requisitos previos para EventBridge
</div>

Para usar AWS EventBridge para flujos de eventos, necesitará lo siguiente:

* Cuenta de AWS

  * Su cuenta de AWS debe tener permisos para usar EventBridge. Si no tiene una cuenta, regístrese en https://aws.amazon.com/eventbridge/.
* Permisos de AWS IAM
* Bus de eventos de AWS EventBridge
* ID y región de la cuenta de AWS

<div id="create-an-event-stream-eventbridge">
  ### Crear un flujo de eventos (EventBridge)
</div>

<Tabs>
  <Tab title="Management API">
    Los flujos de eventos le permiten capturar cambios en tiempo real en su tenant de Auth0 y enviarlos a un sistema externo para su procesamiento.

    Antes de configurar un flujo de eventos, debe identificar los [tipos de eventos](/docs/es/customize/events/event-types) que desea supervisar. Luego, usará el ID de su cuenta de AWS y la región para configurar su flujo de eventos, tal como se muestra a continuación.

    Este ejemplo usa Auth0 CLI para crear un flujo de eventos que se suscribe al evento `user.created`, que se desencadena cada vez que se registra un nuevo usuario en su tenant.

    ```bash wrap lines
    auth0 events create --name ng-demo-eventbridge --type eventbridge --subscriptions "user.created" --configuration '{"aws_account_id":"<your-aws-account-id>","aws_region":"<your-aws-region>"}'
    ```

    Si la llamada se realiza correctamente, devuelve el siguiente JSON con el `id` de tu flujo de eventos. Los nuevos flujos de eventos están habilitados de forma predeterminada.

    ```json lines
    {
      "id": "est_8of6RXoM1997qikH7NS11h",
      "status": "enabled",
      "name": "ng-demo-eventbridge",
      "subscriptions": [
        {
          "event_type": "user.created"
        }
      ],
      "created_at": "2025-01-29T18:08:43.440Z",
      "updated_at": "2025-01-29T18:08:43.440Z",
      "destination": {
        "type": "eventbridge",
        "configuration": {
          "aws_account_id": "<your-aws-account-id>",
          "aws_region": "<your-aws-region>",
          "aws_partner_event_source": "default"
        }
      }
    }
    ```
  </Tab>

  <Tab title="Dashboard">
    Para crear un flujo de eventos en el Auth0 Dashboard mediante AWS EventBridge:

    1. Vaya a **Auth0 Dashboard** &gt; **Event Streams (Early)**.
    2. Seleccione **+Create Event Stream**.
    3. En la lista de tipos de flujo, seleccione **AWS EventBridge**. Esto abrirá el formulario de configuración de su nuevo flujo de EventBridge.
    4. **Configure Stream Details:** En el formulario de configuración, deberá proporcionar la siguiente información:

       * **Stream Name:**

         Introduzca un nombre descriptivo para su flujo de eventos. Esto le ayudará a identificarlo en el Auth0 Dashboard.
       * **AWS Account ID:**

         Introduzca el ID de la cuenta de AWS de 12 dígitos a la que desea enviar los eventos de Auth0.
       * **AWS Region:**

         Seleccione la AWS Region específica donde se encuentra su bus de eventos de EventBridge.
    5. **Select Event Types:** En la sección **Select Event Types**, elija los tipos de eventos de Auth0 específicos que desea incluir en este flujo. Puede seleccionar varios tipos de eventos según sus necesidades (por ejemplo, `user.created`, `user.updated`, `user.deleted`).
    6. **Save Changes:** Una vez que haya configurado el nombre del flujo, los detalles de AWS y seleccionado los tipos de eventos deseados, haga clic en el botón **Save Changes**.

    Su nuevo flujo de eventos ya está creado y Auth0 puede comenzar a publicar los tipos de eventos seleccionados en el bus de eventos de AWS EventBridge especificado. Puede supervisar el estado y administrar su flujo de eventos desde la página Event Streams (Early) en el Auth0 Dashboard.

    ## Crear un flujo de eventos

    Este ejemplo usa el recurso `auth0_event_stream` para crear un flujo de eventos que se suscribe al evento `user.created`, el cual se activa cada vez que se registra un usuario nuevo en tu tenant. Usa el siguiente ejemplo en tu archivo `main.tf`:

    ```hcl
    terraform {
      required_providers {
        auth0 = {
          source  = "auth0/auth0"
          version = ">= 1.0.0" 
        }
      }
    }

    provider "auth0" {}

    resource "auth0_event_stream" "eventbridge_stream" {
      name             = "ng-demo-eventbridge-tf"
      destination_type = "eventbridge" # Obligatorio
      subscriptions    = ["user.created"]

      eventbridge_configuration {
        aws_account_id = var.aws_account_id
        aws_region     = var.aws_region
      }
    }
    ```

    Si la llamada se realiza correctamente, devuelve el siguiente JSON con el `id` de tu flujo de eventos. Los nuevos flujos de eventos se habilitan de forma predeterminada.

    Después de crear y guardar los tres archivos anteriores, ya estás listo para crear tu flujo.

    1. Inicializa el Provider

    ```bash
    terraform init
    ```

    2. Revisa los cambios:

    ```bash
    terraform plan
    ```

    3. Crea el flujo:

    ```bash
    terraform apply
    ```

<div id="webhooks">
  ## Webhooks
</div>

Como alternativa a AWS EventBridge, puedes usar webhooks para habilitar flujos de eventos.

Para empezar, primero configura un controlador de webhook para recibir notificaciones en tiempo real cuando se produzca un evento específico. Luego, podrás crear tu flujo de eventos.

Puedes crear un controlador de webhook básico siguiendo las instrucciones a continuación, o usar un servicio existente como:

* Vercel
* Inngest

Si decides usar un servicio existente, puedes continuar con [Crear un flujo de eventos (webhook)](#create-an-event-stream-webhooks-). De lo contrario, sigue las instrucciones a continuación para crear tu propio controlador de webhook básico.

<div id="webhook-prerequisites">
  ### Requisitos previos del webhook
</div>

Asegúrate de tener instalados los siguientes elementos para poder escribir correctamente tu controlador de webhook:

* `node.js`
* `jq`
* `npm`
* `ngrok`

<div id="write-the-webhook-handler">
  ### Escribe el controlador de webhook
</div>

1. Instala `express` en la carpeta `node_modules` y agrégalo a las dependencias de tu `package.json`.

2. Instala `dotenv` en tu directorio raíz para usar un archivo `.env` donde almacenar variables de entorno.

3. Crea un archivo `webhook.js` para recibir el evento `user.created` y almacenarlo en una base de datos.

   ```javascript lines expandable
   const express = require('express');
   const app = express();

   // Middleware de Authorization
   app.use((req, res, next) => {
     const token = req.headers["authorization"];
     if (token !== `Bearer ${API_TOKEN}`) {
       return res.status(401).json({ error: "Unauthorized" });
     }
     next();
   });

   // Endpoint de webhook
   app.post("/webhook", async (req, res) => {
     console.log("Webhook received:", JSON.stringify(req.body, null, 2));

     const eventData = req.body;
     const { id, type, time, data } = eventData;
     const user = data.object;

     try {
       switch (type) {
         case "user.created":
           await handleUserCreated(user, time);
           break;
         case "user.updated":
           await handleUserUpdated(user, time);
           break;
         case "user.deleted":
           await handleUserDeleted(user, time);
           break;
         default:
           await handleDefaultEvent(id, type, time, data);
       }

       console.log(`Webhook event of type '${type}' committed to the database.`);
       res.sendStatus(204);
     } catch (err) {
       console.error("Error processing webhook:", err);
       res.status(500).json({ error: "Internal server error" });
     }
   });

   // Función específica para procesar el evento de creación de usuario
   // En este ejemplo, nos aseguramos de que los usuarios también se creen en nuestra propia base de datos
   async function handleUserCreated(user, time) {
     const { user_id, email, name, nickname, created_at, updated_at } = user;

     const query = `
       INSERT INTO users (user_id, email, name, nickname, created_at, updated_at, raw_user, last_event_processed)
       VALUES ($1, $2, $3, $4, $5, $6, $7, $8)
     `;
     const values = [
       user_id,
       email,
       name,
       nickname,
       created_at,
       updated_at,
       user,
       time,
     ];

     try {
       await getPool().query(query, values);
     } catch (err) {
       if (err.code === "23505") {
         console.error(`Duplicate user_id=${user_id}, skipping insert.`);
       } else {
         console.error(`Database error while creating user_id=${user_id}:`, err);
         throw err;
       }
     }
   }
   ```

4. En la raíz de tu proyecto, crea un archivo `.env` y agrega tu token de API con lo siguiente:

   ```bash lines
   API_TOKEN=`openssl rand -hex 32`
   echo "API_TOKEN=$API_TOKEN" > .env
   ```

5. Inicia tu servidor:

   ```bash lines
   node webhook.js
   ```

6. Para probar el webhook, expón tu controlador de webhook con una herramienta como ngrok:

   ```bash lines
   ngrok http 3000
   ```

   Esto proporciona una URL pública para tu controlador de webhook local, por ejemplo:

   ```bash lines
   http://localhost:3000
   ```

<div id="create-an-event-stream-actions">
  ### Crear un flujo de eventos (Actions)
</div>

<Tabs>
  <Tab title="Management API">
    Los flujos de eventos te permiten capturar cambios en tiempo real dentro de tu tenant de Auth0 y enviarlos a un sistema externo para su procesamiento.

    Antes de configurar un flujo de eventos, debes identificar los [tipos de eventos](/docs/es/customize/events/event-types) que quieres supervisar. En este ejemplo, se usa Auth0 CLI para crear un flujo de eventos que se suscribe al evento `user.created`, que se activa cada vez que tu tenant registra un usuario nuevo.

    ```bash wrap lines
    auth0 event-streams create \
      --name actions-1 \
      --type action \
      --subscriptions "user.created,user.updated,user.deleted" \
      --configuration '{"action_id":"385db79f-28a4-43d9-909d-6f5511dd6632"}'
    ```

    Si la llamada se realiza correctamente, devuelve el siguiente JSON con el `id` de tu flujo de eventos. Los flujos de eventos nuevos se habilitan de forma predeterminada.

    ```json lines
    {
        "configuration": {
          "action_id": "act_xyz789...",
          "status": "enabled"
      }
    }
    ```
  </Tab>

  <Tab title="Dashboard">
    Para crear un flujo de eventos en el Auth0 Dashboard con Auth0 Actions:

    1. Ve a **Auth0 Dashboard** &gt; **Event Streams (Early)**.

    2. Selecciona **+Create Event Stream**.                .

    3. En la lista de tipos de flujo, selecciona **+Auth0 Actions**. Esto abre el formulario de configuración de tu nuevo flujo de Auth0 Actions.

    4. **Configure Stream Details:** En el formulario de configuración, debes proporcionar la siguiente información:

       * **Stream Name:**

         Introduce un nombre descriptivo para tu flujo de eventos. Esto te ayudará a identificarlo dentro del Auth0 Dashboard.

    5. **Select Event Types:** En la sección **Select Event Types**, elige los tipos de eventos de Auth0 que quieres incluir en este flujo. Puedes seleccionar varios tipos de eventos según tus necesidades (por ejemplo, `Created`, `Updated`, `Deleted`).

    6. En el **Action Editor**, escribe el código de tu controlador. El sistema requiere la función `onExecuteEventStream`. Puedes usar el objeto `api` para gestionar los estados de éxito o error.

    ```javascript
    /**
     * Controlador que se llamará durante la ejecución de un Event Stream.
     * @param {Event} event - Detalles sobre el Cloud Event.
     * @param {EventStreamAPI} api - Interfaz cuyos métodos se pueden usar para manejar el evento.
     */
    exports.onExecuteEventStream = async (event, api) => {
      const eventType = event.message.type;
      const eventData = event.message.data;

      console.log(`Received event: ${eventType}`);

      // Lógica personalizada: p. ej., enviar datos a una API externa
      // await axios.post('https://my-crm.com/ingest', eventData);

      // MANEJO DE FALLOS:
      // Para activar un reintento, lanza un error o usa api.message.fail()
      if (!eventData) {
         // api.message.fail('No data received');
         throw new Error('No data received');
      }

      // ÉXITO:
      // Si la función termina sin error, el evento se marca como entregado.
    };
    ```

    7. **Save Changes:** Una vez que hayas configurado el nombre del flujo y el controlador de la Action, el sistema crea automáticamente la Action, la vincula a un flujo y la habilita.

    Tu nuevo flujo de eventos ya está creado y Auth0 puede empezar a publicar los tipos de eventos seleccionados en la Auth0 Action especificada. Puedes supervisar el estado y gestionar tu flujo de eventos desde la página Event Streams (Early) en el Auth0 Dashboard.
