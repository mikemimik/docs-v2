---
description: Limitaciones de las actividades específicas de migración al pasar de Rules y Hooks a Actions
title: Limitaciones de migración de Actions
---

Para aprovechar las funciones que ofrece Auth0 Actions, debe migrar su código actual desde Rules, Hooks o versiones anteriores de Actions. Sin embargo, debe tener en cuenta las limitaciones actuales al migrar entre las opciones heredadas de extensibilidad y Actions.

<div id="general-differences">
  ## Diferencias generales
</div>

* Actions solo puede devolver códigos de error `access denied` y no admite devolver un código de error `unauthorized`.
* Los claims estándar de un `accessToken` no se pueden eliminar con una Action.

<div id="rule-specific-differences">
  ## Diferencias específicas de Rule
</div>

* El objeto event de las Actions de Post Login no proporciona actualmente una alternativa a `context.connectionOptions`, disponible en Rules.
* El objeto event de las Actions de Post Login no proporciona actualmente una alternativa para acceder a ningún token del IdP de origen para usuarios federados mediante el objeto `user.identities`, disponible en Rules.
* Los claims estándar de un `idToken` no pueden eliminarse mediante una Action.
* Actions solo admiten modificar el usuario principal de las cuentas vinculadas.
