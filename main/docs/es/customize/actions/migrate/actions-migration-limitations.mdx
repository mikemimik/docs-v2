---
description: Limitaciones de las actividades específicas de migración al pasar de Rules y Hooks a Actions
title: Limitaciones de la migración a Actions
---

Para aprovechar las funciones que ofrece Auth0 Actions, debe migrar su código actual desde Rules, Hooks o versiones anteriores de Actions. Sin embargo, debe tener en cuenta las limitaciones actuales al migrar entre las soluciones heredadas de extensibilidad y Actions.

<div id="general-differences">
  ## Diferencias generales
</div>

* Actions solo puede devolver códigos de error `access denied` y no admite la devolución de un código de error `unauthorized`.
* Los claims estándar de un `accessToken` no se pueden eliminar con una Action.

<div id="rule-specific-differences">
  ## Diferencias específicas de Rule
</div>

* El objeto de evento en las Actions de Post Login no ofrece actualmente una alternativa a `context.connectionOptions`, que sí está disponible en Rules.
* El objeto de evento en las Actions de Post Login no ofrece actualmente una alternativa para acceder, a través del objeto `user.identities`, a ningún token del IdP de origen para usuarios federados, como sí ocurre en Rules.
* Los claims estándar de un `idToken` no se pueden eliminar con una Action.
* Las Actions solo permiten modificar al usuario principal entre cuentas vinculadas.
