---
description: Limitaciones de las actividades específicas de migración al migrar de Rules y Hooks a Actions
title: Limitaciones de migración de Actions
---

Para aprovechar las funcionalidades que ofrece Auth0 Actions, debes migrar tu código actual desde Rules, Hooks o versiones anteriores de Actions. Sin embargo, debes tener en cuenta las limitaciones actuales al migrar entre las soluciones de extensibilidad heredadas y Actions.

<div id="general-differences">
  ## Diferencias generales
</div>

* Actions solo puede devolver códigos de error `access denied` y no admite devolver el código de error `unauthorized`.
* Las claims estándar de un `accessToken` no se pueden eliminar con una Action.

<div id="rule-specific-differences">
  ## Diferencias específicas de Rule
</div>

* El objeto de evento en las Actions de Post Login actualmente no ofrece una alternativa a `context.connectionOptions`, disponible en Rules.
* El objeto de evento en las Actions de Post Login actualmente no ofrece una alternativa para acceder, a través del objeto `user.identities`, a ningún token del IdP de origen para usuarios federados, como sí está disponible en Rules.
* Los claims estándar de un `idToken` no se pueden eliminar con una Action.
* Actions solo permite modificar el usuario principal entre cuentas vinculadas.
