---
description: Limitaciones de las actividades específicas de migración al migrar de Rules y Hooks a Actions
title: Limitaciones de migración a Actions
---

Para aprovechar las funcionalidades que ofrece Auth0 Actions, debe migrar su código actual desde Rules, Hooks o versiones anteriores de Actions. Sin embargo, debe tener en cuenta las limitaciones actuales al migrar entre las soluciones de extensibilidad heredadas y Actions.

<div id="general-differences">
  ## Diferencias generales
</div>

* Actions solo pueden devolver códigos de error `access denied` y no admiten devolver códigos de error `unauthorized`.
* No se pueden eliminar los claims estándar de un `accessToken` mediante una Action.

<div id="rule-specific-differences">
  ## Diferencias específicas de Rule
</div>

* El objeto de evento en las Actions de Post Login no proporciona actualmente una alternativa a `context.connectionOptions`, disponible en Rules.
* El objeto de evento en las Actions de Post Login no proporciona actualmente una alternativa para acceder, a través del objeto `user.identities`, a ningún token del IdP de origen para usuarios federados, como sí está disponible en Rules.
* Los claims estándar de un `idToken` no se pueden eliminar usando una Action.
* Actions solo admiten modificar al usuario principal entre cuentas vinculadas.
