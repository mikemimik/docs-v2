title: Plantillas de Actions
---

La galería de plantillas de Actions ofrece una variedad de plantillas base para crear una Action. Estas plantillas proporcionan código de ejemplo para diversos triggers y casos de uso, como <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+Authentication">Autenticación multifactor</Tooltip>, control de acceso y enriquecimiento del perfil.

Las plantillas proporcionan código inicial que puedes modificar y usar como base dentro de un trigger. Una vez que selecciones una plantilla para tu caso de uso específico, podrás personalizar el código para adaptarlo a las necesidades concretas de tu aplicación.

Para ver una guía paso a paso sobre cómo usar las plantillas de Actions, consulta [Escribe tu primera Action](/docs/es/customize/actions/write-your-first-action).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las plantillas de Actions también están disponibles en: https://github.com/auth0/opensource-marketplace.
</Callout>

<div id="best-practices">
  ## Prácticas recomendadas
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las Action Templates están diseñadas únicamente con fines educativos.

  Estas plantillas consisten en fragmentos de código incompletos y no deben utilizarse en un entorno de producción bajo ninguna circunstancia. Los desarrolladores son responsables de desarrollar de forma segura y completar la plantilla de Actions antes de desplegarla en un entorno de producción.
</Callout>

* Asegúrese de que las plantillas estén adaptadas a su entorno específico. Dado que el código está incompleto, debe personalizar y completar los ejemplos antes de importarlos a un entorno de producción.
* No despliegue una plantilla sin editarla y completarla previamente.
* Las plantillas de Actions suelen contener tags en los comentarios, que no deben modificarse ni eliminarse.
* Utilice siempre la versión más reciente de cualquier paquete de `npm`.
* Las plantillas de Actions están orientadas, de forma predeterminada, a las versiones más recientes de Node.js. Después de crear una plantilla, puede modificarla para admitir versiones adicionales de Node.js según sea necesario.

Revise también todas las [Directrices de codificación de Actions](/docs/es/customize/actions/action-coding-guidelines), ya que se aplican aquí.
