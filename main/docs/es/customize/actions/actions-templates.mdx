title: Plantillas para Actions
---

La galería de plantillas de Actions ofrece una variedad de plantillas de inicio para crear una Action. Estas plantillas proporcionan código de ejemplo para diversos triggers y casos de uso, como <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuarios que utiliza un factor adicional al nombre de usuario y la contraseña, como un código enviado por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+Authentication">Autenticación multifactor</Tooltip>, control de acceso y enriquecimiento de perfiles.

Las plantillas proporcionan código base que puedes modificar y usar como componente básico dentro de un trigger. Una vez que selecciones una plantilla para tu caso de uso específico, podrás personalizar el código para adaptarlo a las necesidades particulares de tu aplicación.

Para obtener una guía paso a paso sobre cómo usar las plantillas de Actions, consulta [Escribe tu primera Action](/docs/es/customize/actions/write-your-first-action).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las plantillas de Actions también están disponibles en: https://github.com/auth0/opensource-marketplace.
</Callout>

<div id="best-practices">
  ## Buenas prácticas
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las plantillas de Actions están diseñadas únicamente con fines educativos.

  Estas plantillas consisten en fragmentos de código incompletos y no deben usarse en un entorno de producción bajo ninguna circunstancia. Los desarrolladores son responsables de desarrollar y completar de forma segura la plantilla de Action antes de implementarla en un entorno de producción.
</Callout>

* Asegúrese de que las plantillas se adapten a su entorno específico. Como el código está incompleto, debe personalizar y completar los ejemplos antes de importarlos a un entorno de producción.
* No implemente una plantilla sin editarla y completarla previamente.
* Las plantillas de Actions suelen contener etiquetas en los comentarios que no deben modificarse ni eliminarse.
* Utilice siempre la versión más reciente de cualquier paquete `npm`.
* De forma predeterminada, las plantillas de Actions están orientadas a las versiones más recientes de Node.js. Después de crear una plantilla, puede modificarla para admitir versiones adicionales de Node.js según sea necesario.

Revise todas las [directrices de codificación de Actions](/docs/es/customize/actions/action-coding-guidelines), ya que también se aplican aquí.
