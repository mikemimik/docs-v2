title: Plantillas para Actions
---

La galería de plantillas de Actions ofrece una variedad de plantillas de inicio para crear una Action. Estas plantillas proporcionan código de ejemplo para distintos triggers y casos de uso, como la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+Authentication">Autenticación multifactor</Tooltip>, el control de acceso y el enriquecimiento del perfil.

Las plantillas proporcionan código base que puedes modificar y usar como punto de partida dentro de un trigger. Una vez que selecciones una plantilla para tu caso de uso específico, podrás personalizar el código para adaptarlo a las necesidades concretas de tu aplicación.

Para obtener una guía paso a paso sobre cómo usar las plantillas de Actions, consulta [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las plantillas de Actions también están disponibles en: https://github.com/auth0/opensource-marketplace.
</Callout>

<div id="best-practices">
  ## Prácticas recomendadas
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las plantillas de Actions están diseñadas únicamente con fines educativos.

  Estas plantillas consisten en fragmentos de código incompletos y no deben utilizarse en un entorno de producción bajo ninguna circunstancia. Los desarrolladores son responsables de completar de forma segura la plantilla de Actions antes de implementarla en un entorno de producción.
</Callout>

* Asegúrese de que las plantillas estén adaptadas a su entorno específico. Como el código está incompleto, debe personalizar y completar los ejemplos antes de importarlos en un entorno de producción.
* No implemente una plantilla sin editarla y completarla antes.
* Las plantillas de Actions suelen contener tags en los comentarios, que no deben modificarse ni eliminarse.
* Utilice siempre la versión más reciente de cualquier paquete de `npm`.
* Las plantillas de Actions están dirigidas de forma predeterminada a las versiones más recientes de Node.js. Después de crear una plantilla, puede modificarla para que admita versiones adicionales de Node.js según sea necesario.

Revise también todas las [Actions Coding Guidelines](/docs/es/customize/actions/action-coding-guidelines), ya que también se aplican aquí.
