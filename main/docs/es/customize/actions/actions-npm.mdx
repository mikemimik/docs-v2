El [**`@auth0/actions`** paquete de npm](https://www.npmjs.com/package/@auth0/actions) es la **biblioteca oficial de Actions** e incluye las **definiciones de TypeScript de Auth0 Actions**. Esto te permite programar y probar las Actions de tu proyecto en editores e IDE externos.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El [**`@auth0/actions`** repositorio público](https://github.com/auth0/auth0-actions) te da acceso al código del paquete de npm.
</Callout>

<div id="benefits">
  ### Beneficios
</div>

Esta biblioteca ayuda con los siguientes casos de uso:

* **Asistencia en IDE / editor de código**: Al hacer referencia a esta biblioteca, los IDE y los editores de código pueden ayudar a los desarrolladores con el **autocompletado**, las **definiciones de objetos y funciones** y la **detección de errores**.
* **Desarrollo con TypeScript**: Aunque las Actions todavía se escriben y ejecutan con Node.js CommonJS, esta biblioteca permite desarrollar Actions en proyectos externos con **TypeScript**, que luego pueden compilarse y desplegarse como CommonJS en el tenant de Auth0.
* **Mejoras en las pruebas unitarias**: Al permitir el desarrollo con TypeScript en proyectos externos, esta biblioteca permite a los desarrolladores seguir las prácticas recomendadas y **mejorar sus pruebas unitarias** a partir de las definiciones de TypeScript.
* **Generación de Actions con IA**: Esta biblioteca nos acerca un paso más a que la IA pueda generar ejemplos de Actions más precisos.

<div id="installation">
  #### Instalación
</div>

Usa uno de los siguientes gestores de paquetes para instalar el paquete como **dependencia de desarrollo**:

> El paquete debe usarse como dependencia de desarrollo para complementar tus herramientas de desarrollo.

* **NPM**: `npm install @auth0/actions --save-dev`
* **Yarn**: `yarn add @auth0/actions --dev`
* **Pnpm**: `pnpm add @auth0/actions --save-dev`

│   └───v1

    └───v2

La sentencia `import` debe basarse en cada **nombre del trigger** y **número de versión**, teniendo en cuenta la estructura anterior de la biblioteca.

**Sigue este patrón**: `@auth0/actions/[trigger_name]/[trigger_version]`

**Por ejemplo**: `@auth0/actions/post-login/v3`

Usa una de las siguientes alternativas para importar las definiciones de TypeScript en tus Actions, según tu tecnología:

Usa esta alternativa cuando quieras IntelliSense sin cambiar la estructura actual de tu código JavaScript:

<Tabs>
  <Tab title="JSDoc @import">
    Usa esta alternativa cuando quieras IntelliSense sin cambiar la estructura actual de tu código JavaScript:

    ```javascript
    /** @import {Event, PostLoginAPI} from "@auth0/actions/post-login/v3" */

    /**
    * Manejador que se llamará durante la ejecución de un flujo Post Login.
    *
    * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
    * @param {PostLoginAPI} api - Interfaz cuyos métodos se pueden usar para cambiar el comportamiento del inicio de sesión.

    exports.onExecutePostLogin = async (event, api) => {
      // Tu código
    }
    ```
  </Tab>

  <Tab title="JSDoc @param">
    Usa esta alternativa para contar con seguridad de tipos en archivos JavaScript mediante sentencias `import` en comentarios JSDoc:

    ```javascript

    * Manejador que se llamará durante la ejecución de un flujo Post Login.

    exports.onExecutePostLogin = async (event, api) => {
      // Tu código
    };
    ```
  </Tab>

  <Tab title="TypeScript import">
    Usa esta alternativa cuando desarrolles con TypeScript para obtener comprobación de tipos completa y sintaxis moderna:

    ```javascript
    import type { Event, PostLoginAPI } from '@auth0/actions/post-login/v3';

    /**
    * Manejador que se llamará durante la ejecución de un flujo Post Login.
    *
    * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
    * @param {PostLoginAPI} api - Interfaz cuyos métodos se pueden usar para cambiar el comportamiento del inicio de sesión.

    exports.onExecutePostLogin = async (event: Event, api: PostLoginAPI) => {
      // Tu código
    };
    ```
  </Tab>
</Tabs>

<Warning>
  Cuando uses TypeScript, debes compilar tu código a JavaScript antes de desplegarlo en Auth0. El runtime de Auth0 Actions solo ejecuta JavaScript. Usa el compilador de TypeScript (`tsc`) para transpilar tus archivos `.ts` a archivos `.js` antes de poder desplegarlos. También debes incluir comentarios JSDoc para habilitar IntelliSense en el Dashboard.
</Warning>

<div id="post-login-access-control-and-id-token-custom-claims">
  ### Control de acceso en Post Login y claims personalizados del token de identidad
</div>

La siguiente Action se ejecuta durante el flujo de Post Login. Comprueba si el usuario tiene roles asignados y llama a `api.access.deny()` si no encuentra ninguno. Si hay roles, establece el claim personalizado en el token de identidad.

La sentencia `import` declara la disponibilidad de tipos externos para tu código. Esto permite que el editor conozca la estructura de los objetos `event` y `api`.

<Tabs>
  <Tab title="JavaScript">
    ```javascript
    /** @import {Event, PostLoginAPI} from "@auth0/actions/post-login/v3" */

    const CUSTOM_CLAIM_NAMESPACE = 'https://example.com';

    /**
    * Manejador que se llamará durante la ejecución de un flujo Post Login.
    *
    * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
    * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
    */
    exports.onExecutePostLogin = async (event, api) => {
      const roles = event.authorization?.roles;

      if (roles === undefined || roles.length === 0) {
        api.access.deny('Restricted');
        return;
      }

      api.idToken.setCustomClaim(`${CUSTOM_CLAIM_NAMESPACE}/roles`, roles);
    }
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    import type { Event, PostLoginAPI } from '@auth0/actions/post-login/v3';

    const CUSTOM_CLAIM_NAMESPACE = 'https://example.com';

    /**
    * Manejador que se llamará durante la ejecución de un flujo Post Login.
    *
    * @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
    * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
    */
    exports.onExecutePostLogin = async (event: Event, api: PostLoginAPI) => {
      const roles = event.authorization?.roles;

      if (roles === undefined || roles.length === 0) {
        api.access.deny('Restricted');
        return;
      }

      api.idToken.setCustomClaim(`${CUSTOM_CLAIM_NAMESPACE}/roles`, roles);
    };
    ```
  </Tab>
</Tabs>
