description: Aprende a usar los logs en tiempo real de Actions de Auth0.
title: logs en tiempo real de Actions
---

La funcionalidad de logs en tiempo real de Actions muestra en tiempo real todos los logs del código personalizado de tu cuenta. Esto incluye toda la salida de `console.log` y las excepciones. Estos logs resultan útiles al depurar tus Actions, script de Custom Database y Custom Social Connections.

<Warning>
  Al usar los logs en tiempo real de Actions, ten cuidado de no registrar información sensible. Por ejemplo, los parámetros que reciben las funciones como entrada para Actions, script de Custom Database o Custom Social Connections pueden contener información sensible explícita o implícita. Registrar objetos completos puede provocar la divulgación de información sensible en los logs en tiempo real.

  Auth0 desaconseja firmemente estas prácticas. **Te recomendamos que prestes atención a los objetos que registras, te asegures de que no se registre información sensible y evites registrar explícitamente cualquier tipo de información sensible.**
</Warning>

<div id="how-to-use">
  ## Cómo usarlo
</div>

Para ver los logs en tiempo real de tu código personalizado:

1. Ve a [Auth0 Dashboard &gt; monitoreo &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs).
2. Espera a que el flujo se conecte. Los intentos de conexión se realizan automáticamente.
3. Una vez conectado, las entradas de registro aparecen en el panel principal a medida que se generan.

<div id="features">
  ## Funcionalidades
</div>

El panel de logs en tiempo real de Actions te permite supervisar la actividad del sistema en tiempo real. Esto proporciona información en vivo sobre las Actions ejecutadas a través del Dashboard, lo que facilita identificar problemas, confirmar operaciones correctas o simplemente observar el comportamiento del sistema. A continuación, encontrarás una lista detallada de la funcionalidad disponible actualmente en esta funcionalidad.

<div id="live-log-streaming">
  ### Transmisión de logs en tiempo real
</div>

Las entradas de registro se transmiten en tiempo real a medida que las generan Actions. Cada vez que se ejecuta una Action, su correspondiente entrada de registro aparece automáticamente en el panel de registros, sin necesidad de actualizar la página.

<div id="connection-status">
  ### Estado de la conexión
</div>

En la parte superior del panel se muestra el estado actual de la conexión entre tu navegador y el flujo de logs. Esto te ayuda a confirmar si se están recibiendo logs en ese momento. Los estados más comunes incluyen:

| Estado                         | Descripción                                           |
| ------------------------------ | ----------------------------------------------------- |
| Conectando al flujo de logs... | Intentando establecer una conexión.                   |
| CONECTADO                      | Se ha conectado correctamente y está recibiendo logs. |
| DESCONECTADO                   | No hay conexión con el flujo en este momento.         |

<div id="search-functionality">
  ### Función de búsqueda
</div>

Usa la barra de búsqueda en la parte superior del panel para filtrar las entradas de registro. Puedes buscar por:

* Palabras clave o frases
* Códigos de error específicos
* Mensajes exactos de los registros

Esto es especialmente útil al revisar grandes volúmenes de registros para encontrar una Action o una incidencia específica.

<div id="options-menu">
  ### Menú de opciones (...)
</div>

Haz clic en el menú **Opciones** de la esquina superior derecha del panel de registros para acceder a controles adicionales.

* **Hora local:** Alterna la visualización de la marca de tiempo de las entradas de registro entre la hora predeterminada del servidor (a menudo, UTC) y la hora local del navegador del usuario.
* **Descargar:** Te permite descargar los registros que se muestran actualmente como un archivo JSON para analizarlos sin conexión, compartirlos o archivarlos.
* **Borrar registros:** Borra todas las entradas de registro visibles en la ventana de visualización. Esto no elimina los registros del almacenamiento del sistema, solo de tu vista actual.

<div id="scroll-to-bottom">
  ### Ir al final
</div>

Cuando llegan nuevas entradas y quedan fuera de la vista, aparece un botón de **Ir al final** en el panel de registros. Al hacer clic en él, saltarás a la entrada de registro más reciente, para que siempre estés al día con la actividad más reciente.

<div id="debug-an-action">
  ## Depurar una Action
</div>

El siguiente ejemplo describe cómo crear una Action genérica de Hello World, ejecutarla y usar los logs en tiempo real de Actions para ver los resultados.

1. Siga las instrucciones de [Write Your First Action](/docs/es/customize/actions/write-your-first-action) para escribir una Action de Post Login usando el siguiente ejemplo de código:

   ```javascript lines
   /**
   * Manejador que se llamará durante la ejecución de un flujo de Post Login.
   *
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que inicia sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
   */
   exports.onExecutePostLogin = async (event, api) => {
     console.log('Hello World!');
   };

   /**
   * Manejador que se invocará cuando esta Action reanude su ejecución después de una redirección externa. Si su
   * función onExecutePostLogin no realiza una redirección, esta función puede ignorarse sin problema.
   *
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que inicia sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
   */
   // exports.onContinuePostLogin = async (event, api) => {
   // };
   ```

2. Abra una nueva pestaña y vaya a [Auth0 Dashboard &gt; monitoreo &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs)

3. Una vez que el flujo esté conectado, vuelva al editor de Action y ejecute una prueba.

4. Los resultados deberían aparecer en tiempo real en los logs en tiempo real de Actions.

<div id="supported-statements">
  ### Sentencias compatibles
</div>

La siguiente tabla muestra las sentencias de consola de JavaScript compatibles con la funcionalidad de registro en tiempo real:

| Sentencia         | Descripción                                                                                     | Caso de uso                                                                   |
| ----------------- | ----------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| `console.log()`   | Registro de información de uso general.                                                         | Mensajes informativos básicos, valores de variables.                          |
| `console.info()`  | Mensajes informativos que destacan el progreso o el estado de la aplicación.                    | Informar sobre operaciones correctas e hitos de la aplicación.                |
| `console.warn()`  | Indica posibles problemas o situaciones inesperadas que no son errores críticos.                | Señalar problemas no críticos y funcionalidades obsoletas.                    |
| `console.error()` | Informa sobre errores que se han producido en la aplicación.                                    | Indicar fallos, excepciones y problemas críticos.                             |
| `console.trace()` | Muestra un rastreo de pila, con la secuencia de llamadas a funciones que lleva al punto actual. | Depurar flujos de llamadas complejos y comprender el origen de los problemas. |
