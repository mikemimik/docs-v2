title: Logs en tiempo real de Actions
---

La funcionalidad de logs en tiempo real de Actions muestra en tiempo real todos los logs del código personalizado de tu cuenta. Esto incluye toda la salida de `console.log` y las excepciones. Estos logs son útiles para depurar tus Actions, scripts de Custom Database y Custom Social Connections.

<Warning>
  Al usar los logs en tiempo real de Actions, ten cuidado al registrar información confidencial. Por ejemplo, los parámetros que reciben como entrada las funciones de Actions, Custom Database Scripts o Custom Social Connections pueden contener información confidencial explícita o implícita. Registrar objetos completos puede provocar la divulgación de información confidencial en los logs en tiempo real.

  Auth0 desaconseja firmemente estas prácticas. **Te recomendamos que tengas en cuenta los objetos que registras, te asegures de que no se registre información confidencial y evites registrar explícitamente cualquier tipo de información confidencial.**
</Warning>

<div id="how-to-use">
  ## Cómo usarlo
</div>

Para ver los logs en tiempo real de tu código personalizado:

1. Ve a [Auth0 Dashboard &gt; monitoreo &gt; registro de Actions](https://manage.auth0.com/#/tenant/action-logs).
2. Espera a que el flujo se conecte. Los intentos de conexión se realizan automáticamente.
3. Una vez conectado, las entradas de registro aparecen en el panel principal a medida que se producen.

<div id="features">
  ## Funcionalidades
</div>

El panel de logs en tiempo real de Actions te permite supervisar la actividad del sistema a medida que sucede. Esto proporciona información en vivo sobre las Actions ejecutadas a través del Dashboard, lo que facilita identificar incidencias, confirmar operaciones correctas o simplemente observar el comportamiento del sistema en tiempo real. A continuación, encontrarás una lista detallada de la funcionalidad disponible actualmente en esta funcionalidad.

<div id="live-log-streaming">
  ### Transmisión de logs en tiempo real
</div>

Las entradas de registro se transmiten en tiempo real a medida que las generan las Actions. A medida que se ejecuta cada Action, su correspondiente entrada de registro aparece automáticamente en el panel de registros, sin necesidad de actualizar la página.

<div id="connection-status">
  ### Estado de la conexión
</div>

La parte superior del panel muestra el estado actual de la conexión entre tu navegador y el flujo de logs. Esto te permite confirmar si los logs se están recibiendo activamente. Los estados más comunes son:

| Estado                             | Descripción                                 |
| ---------------------------------- | ------------------------------------------- |
| Conectando con el flujo de logs... | Intentando establecer la conexión.          |
| CONECTADO                          | Conectado correctamente y recibiendo logs.  |
| DESCONECTADO                       | No está conectado al flujo en este momento. |

<div id="search-functionality">
  ### Función de búsqueda
</div>

Usa la barra de búsqueda en la parte superior del panel para filtrar las entradas de registro. Puedes buscar por:

* Palabras clave o frases
* Códigos de error específicos
* Mensajes de registro exactos

Esto resulta especialmente útil al revisar grandes volúmenes de registros para encontrar una Action o una incidencia específica.

<div id="options-menu">
  ### Menú de Opciones (...)
</div>

Haz clic en el menú de **Opciones** en la esquina superior derecha del panel de registros para acceder a controles adicionales.

* **Hora local:** Alterna la visualización de la marca de tiempo de las entradas de registro entre la hora predeterminada del servidor (a menudo, UTC) y la hora local del navegador del usuario.
* **Descargar:** Te permite descargar los registros que se muestran actualmente como un archivo JSON para analizarlos sin conexión, compartirlos o archivarlos.
* **Borrar registros:** Borra todas las entradas de registro visibles actualmente en la ventana de visualización. Esto no elimina los registros del almacenamiento del sistema, solo de la vista actual.

<div id="scroll-to-bottom">
  ### Ir al final
</div>

Cuando llegan nuevas entradas y quedan fuera de la vista, aparece un botón **Ir al final** en el panel de registros. Al hacer clic en él, irás a la entrada de registro más reciente, para que siempre estés al día con la última actividad.

<div id="debug-an-action">
  ## Depurar una Action
</div>

El siguiente ejemplo describe cómo crear una Action genérica de Hello World, ejecutarla y usar los logs en tiempo real de Actions para ver los resultados.

1. Sigue las instrucciones de [Escribe tu primera Action](/docs/es/customize/actions/write-your-first-action) para crear una Action de Post Login con el siguiente ejemplo de código:

   ```javascript lines
   /**
   * Manejador que se llamará durante la ejecución de un flujo de Post Login.
   *
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que inicia sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
   */
   exports.onExecutePostLogin = async (event, api) => {
     console.log('Hello World!');
   };

   /**
   * Manejador que se invocará cuando esta Action se reanude después de una redirección externa. Si tu
   * función onExecutePostLogin no realiza una redirección, esta función se puede ignorar sin problema.
   *
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que inicia sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento del inicio de sesión.
   */
   // exports.onContinuePostLogin = async (event, api) => {
   // };
   ```

2. Abre una pestaña nueva y ve a [Auth0 Dashboard &gt; monitoreo &gt; registro de Actions](https://manage.auth0.com/#/tenant/action-logs)

3. Una vez que el flujo esté conectado, vuelve a tu editor de Action y ejecuta una prueba.

4. Los resultados deberían aparecer en tiempo real en los logs en tiempo real de Actions.

<div id="supported-statements">
  ### Sentencias compatibles
</div>

La siguiente tabla muestra las sentencias de consola de JavaScript compatibles con la funcionalidad de registro en tiempo real:

| Sentencia         | Descripción                                                                                           | Caso de uso                                                                  |
| ----------------- | ----------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| `console.log()`   | Registro general de información.                                                                      | Mensajes informativos básicos, valores de variables.                         |
| `console.info()`  | Mensajes informativos que resaltan el progreso o el estado de la aplicación.                          | Informar sobre operaciones exitosas, hitos de la aplicación.                 |
| `console.warn()`  | Indica posibles problemas o situaciones inesperadas que no son errores críticos.                      | Señalar problemas no críticos, funcionalidades obsoletas.                    |
| `console.error()` | Informa sobre errores que se han producido en la aplicación.                                          | Indicar fallos, excepciones y problemas críticos.                            |
| `console.trace()` | Muestra un seguimiento de pila que indica la secuencia de llamadas a funciones hasta el punto actual. | Depurar flujos de llamadas complejos, comprender el origen de los problemas. |
