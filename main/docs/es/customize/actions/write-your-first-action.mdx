Esta guía te mostrará cómo crear e implementar tu primera Action con el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Al finalizar, deberías comprender la funcionalidad básica que usarás con todas las Actions y estar familiarizado con el modelo de programación. Lo que aprenderás constituye la base para escribir cualquier Action, independientemente de su propósito o del flujo.

<div id="the-goal-notify-slack-on-user-login">
  ## El objetivo: notificar a Slack cuando un usuario inicia sesión
</div>

Va a crear una Action que envía un mensaje a un canal de Slack cuando un usuario inicia sesión. Aunque este tutorial usa el trigger Post Login, la información que se proporciona aquí es útil para triggers de cualquier tipo.

Para realizar esta tarea, hará lo siguiente:

* [Crear una Action](#create-an-action)

<div id="prerequisites">
  ## Requisitos previos
</div>

Como esta Action enviará mensajes a un canal de Slack, debes [crear un webhook entrante para un espacio de trabajo de Slack](https://api.slack.com/messaging/webhooks). Una vez que tengas la URL del webhook de Slack, podrás continuar con esta guía.

<div id="create-an-action">
  ## Crear una Action
</div>

Para que una Action funcione en un flujo específico, primero debes crearla y luego agregarla a un flujo.

1. Ve a [Auth0 Dashboard &gt; Actions &gt; biblioteca,](https://manage.auth0.com/#/actions/library) y luego selecciona **Create Action** &gt; **Build from scratch**.
2. Ingresa un **Name** y selecciona el trigger **Login / Post Login**, ya que agregarás una Action al Login flow. Luego, selecciona **Create.**

Aparecerá el Actions Code Editor:

<Frame>![Actions Code Editor](/docs/images/cdy7uua7fh8z/12Q8OEuDI39eWuWsrVVwN0/ff7dd385d33aee704e1cb603f43c679b/2025-02-28_13-27-14.png)</Frame>

<div id="create-an-action-from-a-template">
  ## Crear una Action a partir de una plantilla
</div>

La galería de plantillas de Actions ofrece varias plantillas de inicio para ayudarte a crear una Action. Para crear una Action a partir de una plantilla:

1. Ve a [Auth0 Dashboard &gt; Actions &gt; biblioteca](https://manage.auth0.com/#/actions/library) y selecciona **Create Action.**
2. Selecciona **Choose from template**.
3. Selecciona la plantilla que quieras usar para crear una Action.
4. Ahora verás una vista previa de solo lectura del código de la plantilla. Para continuar, selecciona **Use this template**.
5. Introduce un nombre y selecciona **Create**.

<div id="add-a-secret">
  ## Agregar un Secret
</div>

Cada Action puede contener valores Secret, aptos para almacenar información confidencial, como <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Tokens">Tokens de acceso</Tooltip> y claves API. Guardemos la URL del webhook de Slack como un Secret.

1. Selecciona el icono de llave en la barra lateral izquierda del editor de código
2. Selecciona **Agregar Secret**.
3. Asigna al Secret el siguiente nombre: `SLACK_WEBHOOK_URL`.
4. Pega la URL del webhook proporcionada por Slack y selecciona **Crear**.

Verás que el Secret se agregó a la Action y podrás usar el nuevo Secret mediante el autocompletado inteligente escribiendo `event.secrets`.

<Frame>![Actions Code Editor - Tu Secret se agregó en la sección Secrets.](/docs/images/cdy7uua7fh8z/4FyFtKRSIYuigxE9U8TkEJ/9598b5909694631ca673f748abf2268a/Screen_Shot_2022-02-09_at_3.24.05_PM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Una vez creado un Secret, su valor no volverá a mostrarse. Auth0 cifra todos los Secrets y los almacena de forma segura.
</Callout>

<div id="add-a-dependency">
  ## Agrega una dependencia
</div>

Usarás el [paquete de `npm` `@slack/webhook`](https://www.npmjs.com/package/@slack/webhook) para enviar un mensaje a Slack más fácilmente. Puedes usar casi cualquier paquete público de `npm` en una Action, siempre que pueda instalarse sin depender de [add-ons nativos](https://www.npmjs.com/package/node-gyp).

Para agregar la dependencia:

1. Selecciona el icono de Dependency (icono de cubo) en la barra lateral.
2. En **Name**, ingresa `@slack/webhook`.

   De forma predeterminada, tu Action usará la versión más reciente de la dependencia disponible cuando la agregues.
3. Selecciona **Crear** y la dependencia debería agregarse a la Action:

   <Frame>![Actions Code Editor: Se agregó tu dependencia.](/docs/images/cdy7uua7fh8z/Jmzvgydo204pBLikzuDJy/1d9a614b4c92f27afbbdd2b2af3b3c0b/Screen_Shot_2022-02-23_at_11.28.41_AM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cuando guardes esta Action, se resolverá la versión más reciente de tu dependencia y se reemplazará por un número de versión específico para evitar que futuras actualizaciones del paquete afecten a tu Action.
</Callout>

<div id="save-the-draft">
  ## Guardar borrador
</div>

Selecciona **Guardar borrador**. Tu Action se guarda, pero todavía no se ejecutará como parte de ningún flujo de usuario. Con Actions, puedes crear, editar y probar Actions antes de que afecten al tráfico de tu tenant de Auth0. Cuando estés conforme con el comportamiento de la Action, más adelante implementarás la Action y la agregarás a un Flow.

<div id="add-custom-logic">
  ## Agregar lógica personalizada
</div>

Ahora que la Action se ha configurado con un Secret y una dependencia, ¡vamos a escribir algo de código!

Todas las Actions tienen un objeto de evento que contiene información contextual de solo lectura relevante para el trigger asociado. Si empiezas a escribir `event.` en el Editor, deberían aparecer todas las propiedades disponibles del evento. Para obtener más información sobre qué datos están disponibles en cada trigger, consulta [Explorar Flows y triggers](/docs/es/customize/actions/explore-triggers).

Usemos nuestra Action para notificar a Slack. Agrega este código a tu Action:

```javascript lines
const { IncomingWebhook } = require('@slack/webhook');

exports.onExecutePostLogin = async (event, api) => {
    const url = event.secrets.SLACK_WEBHOOK_URL;
    const webhook = new IncomingWebhook(url);

    // Enviar la notificación
    await webhook.send({ text: "Logging In..." });
};
```

Selecciona **Guardar borrador** para guardar tu Action.

<div id="test-the-action">
  ## Prueba la Action
</div>

Antes de ejecutar esta Action en tu tenant con tráfico real de usuarios, pruébala para asegurarte de que se comporte como esperas.

1. Selecciona el icono de Test (triángulo) en la barra lateral. En la sección Payload, verás datos JSON editables relevantes para el trigger que seleccionaste cuando creaste la Action.

   <Frame>![Actions Code Editor: panel de Test Runner](/docs/images/cdy7uua7fh8z/4MSu8EfpMLNJpcYtrwP7Jf/66480a0434c89eadb809012bca890037/Screen_Shot_2022-02-10_at_10.01.10_AM.png)</Frame>
2. Selecciona el botón **Run** y deberías ver un mensaje en tu canal de Slack.

<div id="deploy-the-action">
  ## Despliega la Action
</div>

Ahora que hemos comprobado que la Action funciona como se espera, es momento de desplegarla.

Selecciona **Deploy**. Al desplegar una Action, se toma una instantánea de la Action en ese momento y se registra como una **versión de Action**.

Selecciona **Version History** y deberías ver la Action registrada como versión 1:

<Frame>![Actions: Historial de versiones](/docs/images/cdy7uua7fh8z/53sn70cXoxUHibc3Bmk7Og/193af512ac2b58f3d60ab36ca5499d90/Screen_Shot_2022-02-10_at_10.02.47_AM.png)</Frame>

Ahora puedes actualizar tu Action y guardarla sin afectar la versión 1 de tu Action. Auth0 solo ejecutará la versión actualmente desplegada de una Action, así que, hasta que vuelvas a desplegar una Action, Auth0 ejecutará la versión 1.

<div id="attach-the-action-to-a-flow">
  ## Asocia la Action a un flujo
</div>

El paso final para configurar una nueva Action para que se ejecute como parte del tráfico de tu tenant es asociarla a un flujo.

1. Ve a [Auth0 Dashboard &gt; Actions &gt; Flows](https://manage.auth0.com/#/actions/flows) para ver una lista de los Flows disponibles:
2. Selecciona el [Login flow](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) y arrastra `my-slack-action` al flujo.

   <Frame>![Actions Flows: Login Flow](/docs/images/cdy7uua7fh8z/6hU7sKWtGzSx9Z8dhsXbvp/0253fb4e9eac1133cc9626385622c54e/Login_Action_-_English.png)</Frame>
3. Selecciona **Aplicar**.

La Action ya se está ejecutando con tráfico real de usuarios dentro de tu tenant. Si quieres un entorno de pruebas más sólido, considera [configurar varios entornos de Auth0](/docs/es/get-started/auth0-overview/create-tenants/set-up-multiple-environments).

<div id="observe-actions-in-tenant-logs">
  ## Observa Actions en el registro del tenant
</div>

Ahora que tu Action se ejecuta como parte del flujo de Login, Auth0 captura información sobre cada ejecución de la Action en el registro del tenant. Para verlo, ve a [Auth0 Dashboard &gt; Monitoreo &gt; Logs](https://manage.auth0.com/#/logs) y selecciona un evento de **Inicio de sesión exitoso**.

Deberías ver una vista de **Detalles de la Action** que contiene información sobre cualquier Action que se haya ejecutado como parte de ese flujo.

<Frame>![Registro del tenant: vista de Detalles de la Action](/docs/images/cdy7uua7fh8z/61k82wmsXSb9M3wOExM4jH/51fae3a36a8ecddeec08bc45da468e2b/Action_logs_-_English.png)</Frame>
