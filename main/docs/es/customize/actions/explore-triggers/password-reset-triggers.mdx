title: Triggers de Restablecimiento de contraseña

Los triggers de Restablecimiento de contraseña se ejecutan durante el proceso de restablecimiento de contraseña cuando un usuario completa el primer desafío, normalmente un enlace enviado al [correo electrónico del usuario](/docs/es/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa/enroll-and-challenge-email-authenticators), pero antes de que se establezca una nueva contraseña. Puede usar esto para solicitar al usuario un factor adicional de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) o para redirigirlo a un sitio externo, como un verificador de terceros.

Después de la verificación, los usuarios pueden introducir la nueva contraseña de su cuenta.

<Frame>![Para usar el flujo de Restablecimiento de contraseña, vaya a Dashboard > Actions > Flows ](/docs/images/cdy7uua7fh8z/1Pugm9fFYCmdhh7QO5Klzm/7f7eefbf810fd4ebeb22d6393151ffa2/2023-08-14_15-54-03.png)</Frame>

Las Actions de este flujo son bloqueantes (sincrónicas), lo que significa que se ejecutan como parte del proceso de un trigger y evitan que el resto del pipeline de Auth0 se ejecute hasta que la Action se complete.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para que estas Actions se ejecuten correctamente, debe tener Universal Login habilitado. Estas Actions no pueden activarse cuando se usa Classic Login.
</Callout>

<div id="triggers">
  ## Triggers
</div>

<div id="postchallenge">
  ### PostChallenge
</div>

El trigger `post-challenge` es una función que se ejecuta después de que un usuario completa el primer desafío del restablecimiento de contraseña, normalmente un Magic Link por correo electrónico. Puedes crear hasta cuatro Actions en tu tenant que utilicen el trigger `post-challenge`.

<div id="references">
  #### Referencias
</div>

* [Objeto de evento](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-event-object): Proporciona información contextual sobre un único usuario que inicia sesión a través de Auth0.
* [Objeto de API](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object): Proporciona métodos para cambiar el comportamiento del flujo.

<div id="limitations">
  ## Limitaciones
</div>

Los triggers de Restablecimiento de contraseña no admiten [conexiones de Active Directory/LDAP](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap).

<div id="common-use-cases">
  ## Casos de uso comunes
</div>

<div id="secure-password-reset-with-additional-mfa-factors">
  ### Restablecimiento seguro de contraseña con factores de MFA adicionales
</div>

Una Action `password-reset` / `post-challenge` de Actions puede generar un desafío MFA después de que el usuario complete el primer desafío. Por ejemplo, puedes generar un desafío basado en WebAuthn como segundo factor si tu tenant tiene WebAuthn habilitado como factor.

```javascript lines
/**@type {PasswordResetPostChallengeAction}**/
module.exports.onExecutePostChallenge = async (event, api) => {
  const enrolledFactors = event.user.enrolledFactors.map((x) => ({
    type: x.type
  }));
  api.authentication.challengeWith({ type: 'webauthn-roaming' }, { additionalFactors: enrolledFactors });
};
```

<div id="redirect-users-to-a-third-party-application">
  ### Redirigir a los usuarios a una aplicación de terceros
</div>

Además de un desafío MFA, también puedes probar a añadir una redirección en la Action personalizada, por ejemplo, a un verificador de terceros o a un evaluador de riesgos.

```javascript lines
/** @type {PasswordResetPostChallengeAction}
 * Esta Action de ejemplo redirige al usuario a una aplicación de ejemplo
 * y luego continúa la Action después de la redirección para desafiar
 * al usuario con un factor de MFA
 */

module.exports.onExecutePostChallenge = async (event, api) => {
  // Enviar al usuario a https://my-app.example.com
  api.redirect.sendUserTo('https://my-app.example.com');
};

module.exports.onContinuePostChallenge = async (event, api) => {
  const enrolledFactors = event.user.enrolledFactors.map((x) => ({
    type: x.type
  }));

  // Desafiar al usuario con Email OTP O con otro factor registrado
  api.authentication.challengeWith({ type: 'email' }, { additionalFactors: enrolledFactors });

  // Ejemplo de cómo desafiar al usuario con múltiples opciones
  // en este caso Email OTP O SMS OTP
  // api.authentication.challengeWithAny([{ type: 'email' }, { type: 'sms' }]);
};
```

El pipeline de Actions no está activo mientras Auth0 redirige al usuario. Una vez que el usuario continúa el proceso de inicio de sesión de Auth0, el pipeline de Actions se reanuda. Las Actions que se ejecutaron antes de la redirección no se ejecutan de nuevo. Para obtener más información, consulta [Redirect with Actions](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/redirect-with-actions).
