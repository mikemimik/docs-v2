<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="object">
  Información sobre la transacción de intercambio de credenciales.

  <Expandable title="propiedades de la transacción" defaultOpen>
    <ResponseField name="correlation_id" type="string" post={["optional"]}>
      El ID de correlación se puede proporcionar en la solicitud de autenticación inicial cuando la aplicación redirige a Universal Login. Puedes usar este valor para correlacionar registros y solicitudes de tu código de Action con el flujo del usuario.
    </ResponseField>

    <ResponseField name="requested_scopes" type="array of strings">
      Los scopes especificados (si los hay) al solicitar el token de acceso.
    </ResponseField>

    <ResponseField name="target_scopes" type="array of strings" post={["optional"]}>
      &#91;Acceso anticipado&#93; El conjunto activo de scopes de destino para el token de acceso. Se inicializa a partir de las autorizaciones de cliente y se actualiza de inmediato mediante los métodos de scopes de destino de api.transaction en la Action actual y en las posteriores. Después de que se completan todas las Actions, estos scopes se intersectan con la autorización de cliente. Los scopes que no estén presentes en la autorización se excluyen silenciosamente del token de acceso final.
    </ResponseField>
  </Expandable>
</ResponseField>
