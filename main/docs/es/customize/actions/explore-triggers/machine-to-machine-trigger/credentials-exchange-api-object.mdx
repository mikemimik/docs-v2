El objeto de API del trigger credentials-exchange de Actions incluye:

<div id="apiaccess">
  ## `api.access`
</div>

Controla la disponibilidad del token de acceso.

<div id="apiaccessdenycode-reason">
  ### `api.access.deny(code, reason)`
</div>

Marca el intercambio de tokens actual como denegado.

<ResponseField name="code" type="string">
  El código de error específico del protocolo que justifica el rechazo del inicio de sesión.

  Valores permitidos: `invalid_scope`, `invalid_request`, `server_error`
</ResponseField>

<ResponseField name="reason" type="string" post={["optional"]}>
  Una explicación en lenguaje claro de por qué se rechaza la concesión del token de acceso.
</ResponseField>

<div id="apiaccesstoken">
  ## `api.accessToken`
</div>

Solicita cambios en el token de acceso que se está emitiendo.

<div id="apiaccesstokensetcustomclaimkey-value">
  ### `api.accessToken.setCustomClaim(key, value)`
</div>

Establece un custom claim en el token de acceso que se emitirá.

<ResponseField name="key" type="string">
  Nombre del claim (ten en cuenta que puede que deba ser una URL completa).
</ResponseField>

<ResponseField name="value" type="unknown">
  El valor del claim.
</ResponseField>

<div id="apicache">
  ## `api.cache`
</div>

Modifica la caché.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina el registro que describe un valor en caché asociado a la
clave proporcionada, si existe.

<ResponseField name="key" type="string">
  La clave del registro de caché que se eliminará.
</ResponseField>

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en la caché en la clave proporcionada,
si existe. Si se encuentra un registro, el valor almacenado en la caché se puede encontrar
en la propiedad `value` del objeto devuelto.

<ResponseField name="key" type="string">
  La clave del registro almacenado en la caché.
</ResponseField>

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de cadena en la caché en la clave especificada.

Los valores almacenados en esta caché están limitados al trigger en el que
se establecen. Están sujetos a los [Límites de la caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta manera tendrán una vigencia de *hasta* los valores
especificados en `ttl` o `expires_at`. Si no se especifica ninguna vigencia, se usará una
vigencia predeterminada de 15 minutos. La vigencia no puede superar la duración máxima
indicada en [Límites de la caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Esta caché está diseñada para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores aunque aún estén dentro de la vigencia indicada.

<ResponseField name="key" type="string">
  La clave del registro que se almacenará.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se almacenará.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento de la caché.

  <Expandable title="Propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora absoluta de vencimiento en milisegundos desde la época unix.
      Aunque los registros en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `expires_at` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se
      proporcionó un valor para `ttl`. Si se proporcionan ambas opciones, se
      usará el vencimiento más temprano de los dos.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada de caché en milisegundos.
      Aunque los valores en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `ttl` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se
      proporcionó un valor para `expires_at`. Si se proporcionan ambas opciones, se
      usará el vencimiento más temprano de los dos.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="apitransaction">
  ## `api.transaction`
</div>

[acceso anticipado] Realiza cambios en la transacción.

<div id="apitransactionaddtargetscopescope">
  ### `api.transaction.addTargetScope(scope)`
</div>

[acceso anticipado] Añade un scope al conjunto de scopes de destino. Los scopes añadidos se intersectan con la
autorización de cliente después de que se completen todas las Actions. Los scopes que no estén presentes en la concesión
se omiten sin notificación del token de acceso final.

<ResponseField name="scope" type="string">
  El scope que se va a añadir.
</ResponseField>

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.addTargetScope('read:reports');
};
```

<div id="apitransactionremovetargetscopescope">
  ### `api.transaction.removeTargetScope(scope)`
</div>

[acceso anticipado] Elimina un scope del conjunto de scopes de destino.

<ResponseField name="scope" type="string">
  El scope que se eliminará.
</ResponseField>

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.removeTargetScope('admin:full');
};
```

<div id="apitransactionsettargetscopesscopes">
  ### `api.transaction.setTargetScopes(scopes)`
</div>

[Acceso anticipado] Reemplaza por completo el conjunto de scopes de destino. Los nuevos scopes se intersectan con
la autorización de cliente una vez que se completan todas las Actions. Los scopes que no estén presentes en la
concesión se eliminan del token de acceso final sin notificación.

<ResponseField name="scopes" type="arreglo de cadenas">
  El nuevo conjunto de scopes de destino.
</ResponseField>

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.setTargetScopes(['read:users', 'write:users']);
};
```

<div id="apitransactioncleartargetscopes">
  ### `api.transaction.clearTargetScopes()`
</div>

[Acceso anticipado] Elimina todos los scopes del conjunto de scopes de destino.

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.clearTargetScopes();
};
```
