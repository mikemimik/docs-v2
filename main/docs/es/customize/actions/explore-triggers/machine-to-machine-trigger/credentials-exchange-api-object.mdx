El objeto de API del trigger credentials-exchange de Actions incluye:

<div id="apiaccess">
  ## `api.access`
</div>

Controla si el token de acceso está disponible.

<div id="apiaccessdenycode-reason">
  ### `api.access.deny(code, reason)`
</div>

Marca el intercambio de tokens actual como denegado.

<ResponseField name="code" type="string">
  El código de error específico del protocolo que justifica el rechazo del inicio de sesión.

  Valores permitidos: `invalid_scope`, `invalid_request`, `server_error`
</ResponseField>

<ResponseField name="reason" type="string" post={["optional"]}>
  Una explicación en lenguaje claro para justificar el rechazo de la concesión del token de acceso.
</ResponseField>

<div id="apiaccesstoken">
  ## `api.accessToken`
</div>

Solicita cambios en el token de acceso que se está emitiendo.

<div id="apiaccesstokensetcustomclaimkey-value">
  ### `api.accessToken.setCustomClaim(key, value)`
</div>

Establece una claim personalizada en el token de acceso que se emitirá.

<ResponseField name="key" type="string">
  Nombre de la claim (ten en cuenta que puede ser necesario usar una URL completa).
</ResponseField>

<ResponseField name="value" type="unknown">
  El valor de la claim.
</ResponseField>

<div id="apicache">
  ## `api.cache`
</div>

Modifica la caché.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina, si existe, un registro que describe un valor en caché
asociado con la clave proporcionada.

<ResponseField name="key" type="string">
  La clave del registro de caché que se eliminará.
</ResponseField>

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en la caché en la clave proporcionada,
si existe. Si se encuentra un registro, el valor almacenado en la caché puede encontrarse
en la propiedad `value` del objeto devuelto.

<ResponseField name="key" type="string">
  La clave del registro almacenado en la caché.
</ResponseField>

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de texto en el caché con la clave especificada.

Los valores almacenados en este caché se limitan al trigger en el que
se establecen. Están sujetos a los [Límites del caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta forma tendrán una vigencia de *hasta* los valores
de `ttl` o `expires_at` especificados. Si no se especifica ninguna vigencia, se usará una
vigencia predeterminada de 15 minutos. La vigencia no puede superar la
duración máxima indicada en [Límites del caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Este caché está diseñado para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si todavía están dentro de la vigencia indicada.

<ResponseField name="key" type="string">
  La clave del registro que se almacenará.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se almacenará.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento del caché.

  <Expandable title="propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora de vencimiento absoluta, en milisegundos desde el unix epoch.
      Aunque los registros almacenados en caché pueden eliminarse antes, nunca
      permanecerán más allá del `expires_at` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó
      un valor para `ttl`. Si se proporcionan ambas opciones, se
      usará el vencimiento más próximo de los dos.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada de caché, en milisegundos.
      Aunque los valores almacenados en caché pueden eliminarse antes, nunca
      permanecerán más allá del `ttl` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó
      un valor para `expires_at`. Si se proporcionan ambas opciones, se
      usará el vencimiento más próximo de los dos.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="apitransaction">
  ## `api.transaction`
</div>

[Acceso anticipado] Realiza cambios en la transacción.

<div id="apitransactionaddtargetscopescope">
  ### `api.transaction.addTargetScope(scope)`
</div>

[Acceso anticipado] Agrega un scope al conjunto de scopes objetivo. Los scopes agregados se cruzan con la
autorización de cliente una vez que se completan todas las Actions. Los scopes que no estén presentes en la concesión
se omiten silenciosamente del token de acceso final.

<ResponseField name="scope" type="string">
  El scope que se va a agregar.
</ResponseField>

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.addTargetScope('read:reports');
};
```

<div id="apitransactionremovetargetscopescope">
  ### `api.transaction.removeTargetScope(scope)`
</div>

[acceso anticipado] Elimina un scope del conjunto de scopes de destino.

<ResponseField name="scope" type="string">
  El scope que se debe eliminar.
</ResponseField>

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.removeTargetScope('admin:full');
};
```

<div id="apitransactionsettargetscopesscopes">
  ### `api.transaction.setTargetScopes(scopes)`
</div>

[Acceso anticipado] Reemplaza por completo el conjunto de scopes de destino. Los nuevos scopes se cruzan con
la concesión de cliente una vez que se completan todas las Actions. Los scopes que no estén presentes en la
concesión se omiten silenciosamente del token de acceso final.

<ResponseField name="scopes" type="array of strings">
  El nuevo conjunto de scopes de destino.
</ResponseField>

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.setTargetScopes(['read:users', 'write:users']);
};
```

<div id="apitransactioncleartargetscopes">
  ### `api.transaction.clearTargetScopes()`
</div>

[acceso anticipado] Elimina todos los scopes del conjunto de scopes de destino.

```js
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.transaction.clearTargetScopes();
};
```
