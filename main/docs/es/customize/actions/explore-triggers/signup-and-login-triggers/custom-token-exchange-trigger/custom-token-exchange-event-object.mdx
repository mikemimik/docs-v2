<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="dictionary">
  Detalles sobre la transacción de intercambio de token personalizado actual.

  <Expandable title="propiedades de la transacción" defaultOpen>
    <ResponseField name="actor_token" type="string" post={["optional"]}>
      El token de actor incluido en la solicitud de intercambio de tokens.
    </ResponseField>

    <ResponseField name="actor_token_type" type="string" post={["optional"]}>
      El tipo del token de actor proporcionado en la solicitud de intercambio de tokens.
    </ResponseField>

    <ResponseField name="actor_token_user" type="dictionary" post={["optional"]}>
      El usuario representado por el token de actor. Solo estará presente si el actor&#95;token&#95;type es urn:ietf:params:oauth:token-type:id&#95;token y el token de actor proporcionado en la solicitud de intercambio de tokens es un token de identidad válido generado por Auth0.

      <Expandable title="propiedades de actor_token_user" defaultOpen>
        <ResponseField name="app_metadata" type="dictionary">
          Campos personalizados que almacenan información sobre un usuario y que influye en su acceso, como el plan de soporte, los roles de seguridad o los grupos de control de acceso.
        </ResponseField>

        <ResponseField name="created_at" type="string">
          Marca de tiempo que indica cuándo se creó el perfil del usuario por primera vez.
        </ResponseField>

        <ResponseField name="correo electrónico" type="string" post={["optional"]}>
          (única) Dirección de correo electrónico del usuario.
        </ResponseField>

        <ResponseField name="email_verified" type="boolean">
          Indica si el usuario ha verificado su dirección de correo electrónico.
        </ResponseField>

        <ResponseField name="apellido" type="string" post={["optional"]}>
          Apellido del usuario.
        </ResponseField>

        <ResponseField name="nombre" type="string" post={["optional"]}>
          Nombre de pila del usuario.
        </ResponseField>

        <ResponseField name="ultimo_restablecimiento_de_contraseña" type="string" post={["optional"]}>
          Marca temporal que indica la última vez que se restableció o cambió la contraseña del usuario. Al crear el usuario, este campo no existe. Esta propiedad solo está disponible para las conexiones de base de datos.
        </ResponseField>

        <ResponseField name="nombre" type="string" post={["optional"]}>
          Nombre completo del usuario.
        </ResponseField>

        <ResponseField name="apodo" type="string" post={["optional"]}>
          El alias del usuario.
        </ResponseField>

        <ResponseField name="phone_number" type="string" post={["optional"]}>
          Número de teléfono del usuario.
        </ResponseField>

        <ResponseField name="phone_verified" type="boolean" post={["optional"]}>
          Indica si el usuario ha verificado su número de teléfono.
        </ResponseField>

        <ResponseField name="foto" type="string" post={["optional"]}>
          URL que apunta a la [foto de perfil del usuario](https://auth0.com/docs/users/change-user-picture).
        </ResponseField>

        <ResponseField name="updated_at" type="string">
          Marca de tiempo que indica cuándo se actualizó o modificó por última vez el perfil del usuario.
        </ResponseField>

        <ResponseField name="user_id" type="string">
          (unique) Identificador único del usuario.
        </ResponseField>

        <ResponseField name="user_metadata" type="dictionary">
          Campos personalizados que almacenan información sobre un usuario y que no afectan a aquello a lo que puede o no puede acceder, como la dirección del trabajo, la dirección de casa o las preferencias del usuario.
        </ResponseField>

        <ResponseField name="nombre de usuario" type="string" post={["optional"]}>
          (único) nombre de usuario del usuario.
        </ResponseField>

        <ResponseField name="enrolledFactors" type="array of objects" post={["optional"]}>
          Un arreglo de factores de autenticación que el usuario ya ha registrado.

          <Expandable title="Propiedades de enrolledFactors">
            <ResponseField name="type" type="string">
              El tipo de factor de autenticación, como `push-notification`, `phone`, `email`, `otp`, `webauthn-roaming` y `webauthn-platform`.
            </ResponseField>

            <ResponseField name="options" type="dictionary" post={["optional"]}>
              Opciones adicionales que describen esta instancia del factor registrado.
            </ResponseField>
          </Expandable>
        </ResponseField>

        <ResponseField name="multifactor" type="array of strings" post={["optional"]}>
          Lista de proveedores de autenticación multifactor (MFA) con los que el usuario está inscrito. Este arreglo se actualiza cuando el usuario se inscribe en MFA y cuando un administrador restablece las inscripciones de MFA de un usuario.
        </ResponseField>

        <ResponseField name="identidades" type="array of objects">
          Contiene información obtenida del proveedor de identidad con el que el usuario se autenticó originalmente. Los usuarios también pueden vincular su perfil a múltiples proveedores de identidad; esas identidades también aparecerán en este arreglo. El contenido de cada objeto de proveedor de identidad varía según el proveedor.

          <Expandable title="propiedades del elemento identities">
            <ResponseField name="connection" type="string" post={["optional"]}>
              Nombre de la conexión de Auth0 utilizada para autenticar al usuario.
            </ResponseField>

            <ResponseField name="isSocial" type="boolean" post={["optional"]}>
              Indica si la conexión es de tipo social.
            </ResponseField>

            <ResponseField name="profileData" type="dictionary" post={["optional"]}>
              Información del usuario asociada con la conexión. Cuando los perfiles están vinculados, se rellena con la información de usuario asociada a las cuentas secundarias.
            </ResponseField>

            <ResponseField name="provider" type="string" post={["optional"]}>
              Nombre de la entidad que autentica al usuario, como Facebook, Google, SAML o su propio proveedor.
            </ResponseField>

            <ResponseField name="user_id" type="string" post={["optional"]}>
              Identificador único del usuario en esta conexión/proveedor.
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="scopes_solicitados" type="array of strings">
      Los scopes solicitados (si los hay) incluidos en la solicitud de intercambio de tokens.
    </ResponseField>

    <ResponseField name="tipo_de_token_solicitado" type="string">
      El tipo de token que generará Auth0. Por ejemplo: urn:ietf:params:oauth:token-type:access&#95;token.
    </ResponseField>

    <ResponseField name="subject_token" type="string">
      El token de sujeto proporcionado en la solicitud de intercambio de tokens.
    </ResponseField>

    <ResponseField name="subject_token_type" type="string">
      El subject&#95;token&#95;type incluido en la solicitud de intercambio de tokens.
    </ResponseField>
  </Expandable>
</ResponseField>
