<div id="eventsecurity_context">
  ## `event.security_context`
</div>

<ResponseField name="event.security_context" type="object" post={["optional"]}>
  Un objeto que contiene firmas de huella digital. Esto solo estará disponible si el cliente usa cloudflare. La huella digital JA3/JA4 puede ser nula o estar vacía en algunos casos. El caso más común es en las peticiones HTTP, porque JA3 y JA4 se calculan en TLS. También puede estar vacía porque el Worker envía peticiones dentro de la misma zona o a una zona que no pasa por proxy (o a un tercero).

  <Expandable title="propiedades de security_context" defaultOpen>
    <ResponseField name="ja3" type="string" post={["optional"]}>
      Firma de huella digital JA3. Esto solo estará disponible si el cliente usa una conexión TLS.
    </ResponseField>

    <ResponseField name="ja4" type="string" post={["optional"]}>
      Firma de huella digital JA4. Esto solo estará disponible si el cliente usa una conexión TLS.
    </ResponseField>
  </Expandable>
</ResponseField>
