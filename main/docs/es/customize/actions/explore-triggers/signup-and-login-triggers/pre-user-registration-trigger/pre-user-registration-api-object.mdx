El objeto de API del trigger de Action de Registro previo de usuario incluye:

<div id="apiaccess">
  ## `api.access`
</div>

Modifica el acceso del usuario que está iniciando sesión, por ejemplo, rechazando su intento de inicio de sesión.

<div id="apiaccessdenyreason-usermessage">
  ### `api.access.deny(reason, userMessage)`
</div>

Impide que el usuario pueda registrarse. El flujo de registro se detendrá inmediatamente al
completarse esta Action y no se ejecutará ninguna Action adicional.

<ResponseField name="reason" type="string">
  Un motivo interno que describe por qué se deniega este intento de registro. Este valor
  aparecerá en el registro del tenant.
</ResponseField>

<ResponseField name="userMessage" type="string">
  Una explicación en lenguaje claro de por qué se rechaza el intento de registro. Puede mostrarse
  directamente en las interfaces de usuario final.
</ResponseField>

<div id="apiuser">
  ## `api.user`
</div>

Modifica los metadatos del usuario que inicia sesión.

<div id="apiusersetappmetadatakey-value">
  ### `api.user.setAppMetadata(key, value)`
</div>

Establece metadatos específicos de la aplicación para el usuario que está iniciando sesión.

Nota: Este método no debe usarse en devoluciones de llamada. Al invocar este método, los metadatos no se actualizarán de inmediato.
Puedes llamarlo varias veces a lo largo de varias Actions del mismo flujo, y el motor agregará los
cambios y actualizará los metadatos de una sola vez antes de que finalice el flujo.

<ResponseField name="key" type="string">
  La propiedad de metadato que se establecerá.
</ResponseField>

<ResponseField name="value" type="unknown">
  El valor de la propiedad de metadato. Puede establecerse en `null` para eliminar la
  propiedad de metadato.
</ResponseField>

<div id="apiusersetusermetadatakey-value">
  ### `api.user.setUserMetadata(key, value)`
</div>

Establece metadatos generales para el usuario que está iniciando sesión.

Nota: Este método no debe usarse en devoluciones de llamada. Invocar este método no actualizará los metadatos de inmediato.
Puedes llamar a este método varias veces a lo largo de varias Actions del mismo flujo, y el motor agrupará los
cambios y actualizará los metadatos de una sola vez antes de que se complete el flujo.

<ResponseField name="key" type="string">
  La propiedad de metadatos que se establecerá.
</ResponseField>

<ResponseField name="value" type="unknown">
  El valor de la propiedad de metadatos. Se puede establecer en `null` para eliminar la
  propiedad de metadatos.
</ResponseField>

<div id="apicache">
  ## `api.cache`
</div>

Almacena y recupera datos que se mantienen entre ejecuciones.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina, si existe, un registro que describe un valor almacenado en caché en la
clave proporcionada.

<ResponseField name="key" type="string">
  La clave del registro de caché que se va a eliminar.
</ResponseField>

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en caché en la clave proporcionada,
si existe. Si se encuentra un registro, el valor almacenado en caché se encuentra
en la propiedad `value` del objeto devuelto.

<ResponseField name="key" type="string">
  La clave del registro almacenado en la caché.
</ResponseField>

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de cadena en la caché con la clave especificada.

Los valores almacenados en esta caché están limitados al trigger en el que
se establecen. Están sujetos a los [Límites de caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta manera tendrán una vigencia de *hasta* los valores
de `ttl` o `expires_at` especificados. Si no se especifica ninguna vigencia, se usará de forma predeterminada
una vigencia de 15 minutos. La vigencia no puede superar la duración máxima
indicada en [Límites de caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Esta caché está diseñada para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si todavía están dentro de la vigencia indicada.

<ResponseField name="key" type="string">
  La clave del registro que se va a almacenar.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se va a almacenar.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento de la caché.

  <Expandable title="propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora de vencimiento absoluta en milisegundos desde la época Unix.
      Aunque los registros almacenados en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `expires_at` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó
      un valor para `ttl`. Si se proporcionan ambas opciones, se usará
      el vencimiento más próximo de las dos.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada de caché en milisegundos.
      Aunque los valores almacenados en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `ttl` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó
      un valor para `expires_at`. Si se proporcionan ambas opciones, se usará
      el vencimiento más próximo de las dos.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="apivalidation">
  ## `api.validation`
</div>

Modifica el acceso del usuario que está iniciando sesión, por ejemplo, rechazando el intento de inicio de sesión debido a errores de validación.

<div id="apivalidationerrorerrorcode-errormessage">
  ### `api.validation.error(errorCode, errorMessage)`
</div>

Impide que el usuario se registre. El flujo de registro se detendrá inmediatamente al completarse esta
Action y no se ejecutará ninguna Action más.

<ResponseField name="errorCode" type="string">
  Un código de error definido por el cliente que describe por qué se deniega este intento de registro. Este valor
  aparecerá en los registros del tenant.
</ResponseField>

<ResponseField name="errorMessage" type="string">
  Una explicación definida por el cliente para rechazar el intento de registro. Puede mostrarse
  directamente en las interfaces de usuario final.
</ResponseField>
