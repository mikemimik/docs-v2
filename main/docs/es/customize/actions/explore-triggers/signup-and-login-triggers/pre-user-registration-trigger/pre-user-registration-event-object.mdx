<div id="eventsecurity_context">
  ## `event.security_context`
</div>

<ResponseField name="event.security_context" type="object" post={["optional"]}>
  Un objeto que contiene firmas de huella digital. Solo estará disponible si el cliente usa Cloudflare. La huella digital JA3/JA4 puede ser nula o estar vacía en algunos casos. El caso más común es el de las solicitudes HTTP, porque JA3 y JA4 se calculan en TLS. También puede estar vacía porque el Worker envía solicitudes dentro de la misma zona o a una zona que no tiene proxy (o a un tercero).

  <Expandable title="Propiedades de security_context" defaultOpen>
    <ResponseField name="ja3" type="string" post={["optional"]}>
      Firma de huella digital JA3. Solo estará disponible si el cliente usa una conexión TLS.
    </ResponseField>

    <ResponseField name="ja4" type="string" post={["optional"]}>
      Firma de huella digital JA4. Solo estará disponible si el cliente usa una conexión TLS.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="dictionary" post={["optional"]}>
  Detalles sobre la transacción actual.

  <Expandable title="propiedades de la transacción" defaultOpen>
    <ResponseField name="acr_values" type="array of strings">
      Cualquier `acr_values` proporcionado en la solicitud de autenticación original.
    </ResponseField>

    <ResponseField name="locale" type="string">
      La configuración regional que se usará para esta transacción, determinada al comparar los idiomas solicitados por el navegador con la configuración de idioma del tenant.
    </ResponseField>

    <ResponseField name="login_hint" type="string" post={["optional"]}>
      Indicación para el servidor de autorización sobre el identificador de inicio de sesión que el usuario final podría usar para iniciar sesión (si fuera necesario).
    </ResponseField>

    <ResponseField name="prompt" type="array of strings" post={["optional"]}>
      Lista de instrucciones que indica si se puede solicitar al usuario que vuelva a autenticarse y otorgue consentimiento.
    </ResponseField>

    <ResponseField name="protocol" type="string" post={["optional"]}>
      Valores posibles:

      * `oidc-basic-profile`
      * `oidc-ciba`
      * `oidc-ciba-web-link`
      * `oidc-implicit-profile`
      * `oauth2-device-code`
      * `oauth2-resource-owner`
      * `oauth2-resource-owner-jwt-bearer`
      * `oauth2-password`
      * `oauth2-webauthn`
      * `oauth2-access-token`
      * `oauth2-refresh-token`
      * `oauth2-token-exchange`
      * `oidc-hybrid-profile`
      * `samlp`
      * `wsfed`
      * `wstrust-usernamemixed`
    </ResponseField>

    <ResponseField name="redirect_uri" type="string" post={["optional"]}>
      La URL a la que Auth0 redirigirá el navegador después de que se complete la transacción.
    </ResponseField>

    <ResponseField name="requested_scopes" type="array of strings">
      Los scopes solicitados (si los hay) al iniciar este flujo de autenticación.
    </ResponseField>

    <ResponseField name="response_mode" type="string" post={["optional"]}>
      Informa al servidor de autorización del mecanismo que se usará para devolver parámetros desde el endpoint de autorización.

      Valores permitidos: `query`, `fragment`, `form_post`, `web_message`
    </ResponseField>

    <ResponseField name="response_type" type="array of strings" post={["optional"]}>
      Indica el tipo de credencial que Auth0 devolverá.
    </ResponseField>

    <ResponseField name="state" type="string" post={["optional"]}>
      Una cadena alfanumérica arbitraria y opaca que tu aplicación agrega a la solicitud inicial y que Auth0 incluye al redirigir de vuelta a tu aplicación.
    </ResponseField>

    <ResponseField name="ui_locales" type="array of strings">
      Los `ui_locales` proporcionados en la solicitud de autenticación original.
    </ResponseField>

    <ResponseField name="correlation_id" type="string" post={["optional"]}>
      El ID de correlación puede proporcionarse en la solicitud de autenticación inicial cuando la aplicación redirige a Universal Login. Puedes usar este valor para correlacionar logs y solicitudes de tu código de Action con el flujo del usuario.
    </ResponseField>
  </Expandable>
</ResponseField>
