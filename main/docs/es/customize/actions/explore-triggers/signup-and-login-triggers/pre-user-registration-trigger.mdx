El trigger de Registro previo de usuario se ejecuta antes de que se agregue un usuario a una conexión Database o de <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>.

<Frame>![Diagrama que muestra el flujo de Registro previo de usuario de Actions.](/docs/images/cdy7uua7fh8z/2KtUcZhaBcT12GxjhBJZFG/9633b4454ac1c06c0deed6c97e70fe7d/pre-user-registration-flow.png)</Frame>

Las Actions de este flujo son bloqueantes (sincrónicas), lo que significa que se ejecutan como parte del proceso de un trigger y evitan que el resto del pipeline de Auth0 se ejecute hasta que la Action se haya completado.

<div id="triggers">
  ## Triggers
</div>

<div id="pre-user-registration">
  ### Registro previo de usuario
</div>

El trigger `pre-user-registration` se ejecuta cuando un usuario intenta registrarse a través de una conexión Database o passwordless. Este trigger puede usarse para añadir metadatos al perfil del usuario antes de que se cree o para denegar el registro con lógica personalizada.

<Warning>
  Actualmente, no puedes usar Actions de `pre-user-registration` para añadir metadatos a usuarios passwordless.
</Warning>

<div id="common-use-cases">
  ## Casos de uso habituales
</div>

<div id="deny-registration-by-location">
  ### Denegar el registro según la ubicación
</div>

Se puede usar una Action de Registro previo de usuario para impedir que un usuario se registre.

```javascript lines
/**
 * @param {Event} event - Details about registration event.
 * @param {PreUserRegistrationAPI} api
 */
exports.onExecutePreUserRegistration = async (event, api) => {
  if (event.request.geoip.continentCode === "NA") {

    // localizar el mensaje de error 
    const LOCALIZED_MESSAGES = {
      en: 'You are not allowed to register.',
      es: 'No tienes permitido registrarte.'
    };

    const userMessage = LOCALIZED_MESSAGES[event.request.language] || LOCALIZED_MESSAGES['en'];
    api.access.deny('no_signups_from_north_america', userMessage);
  }
};
```

<div id="set-metadata-in-the-user-profile">
  ### Establecer metadatos en el perfil del usuario
</div>

Se puede usar una Action de Registro previo de usuario para añadir metadatos al perfil del usuario antes de que se cree.

<Warning>
  Actualmente no puedes usar Actions de `pre-user-registration` para añadir metadatos a usuarios passwordless.
</Warning>

```javascript lines
/**
 * @param {Event} event - Detalles sobre el evento de registro.
 * @param {PreUserRegistrationAPI} api
 */
exports.onExecutePreUserRegistration = async (event, api) => {
  api.user.setUserMetadata("screen_name", "username");  
};
```

<div id="store-a-user-id-from-another-system-in-the-user-profile">
  ### Almacenar un ID de usuario de otro sistema en el perfil de usuario
</div>

Se puede usar una Action de Registro previo de usuario para guardar en el perfil de usuario un ID de usuario de otro sistema.

```javascript lines expandable
const axios = require('axios');

const REQUEST_TIMEOUT = 2000; // Ejemplo de tiempo de espera

/**
* Controlador que se invocará durante la ejecución de un flujo PreUserRegistration.
*
* @param {Event} event - Detalles sobre el contexto y el usuario que intenta registrarse.
* @param {PreUserRegistrationAPI} api - Interfaz cuyos métodos pueden usarse para modificar el comportamiento del registro.
*/
exports.onExecutePreUserRegistration = async (event, api) => {
  try {
    // Establece un Secret USER_SERVICE_URL = 'https://yourservice.com'
    const remoteUser = await axios.get(event.secrets.USER_SERVICE_URL, {
      timeout: REQUEST_TIMEOUT,
      params: {
        email: event.user.email 
      }
    });

    if (remoteUser) {
      api.user.setAppMetadata('my-api-user-id', remoteUser.id); 
    }
  } catch (err) {
    api.validation.error('custom_error', 'Custom Error');
  }
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar una biblioteca de `npm` como `axios`, debes agregar la biblioteca a la Action como dependencia. Para obtener más información, consulta la sección &quot;Agregar una dependencia&quot; de [Write Your First Action](/docs/es/customize/actions/write-your-first-action).
</Callout>

<div id="deny-access-to-specific-ja3ja4-fingerprints">
  ### Denegar acceso a huellas digitales JA3/JA4 específicas
</div>

El objeto `event.security_context` contiene los valores de las huellas digitales JA3/JA4 de la transacción actual.

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const clientJa4 = event?.security_context?.ja4;
  console.log('[ACTION]', {clientJa4});
  const badFingerprints = ['t13d1517h2_8daaf6152771_b6f405a00624','t13d1516h2_8daaf6152771_d8a2da3f94cd'];
  if (clientJa4 && badFingerprints.includes(clientJa4)){
    api.access.deny('suspicious_tls_fingerprint', 'Your TLS fingerprint has been flagged as suspicious');
  }
};
```
