El objeto de API del trigger de Actions Posterior al cambio de contraseña incluye:

<div id="apicache">
  ## `api.cache`
</div>

Almacena y recupera datos que se conservan entre ejecuciones.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina el registro correspondiente a un valor en caché en la
clave proporcionada, si existe.

<ResponseField name="key" type="string">
  La clave del registro de caché que se va a eliminar.
</ResponseField>

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en caché en la clave indicada,
si existe. Si se encuentra un registro, el valor almacenado en caché se encuentra
en la propiedad `value` del objeto devuelto.

<ResponseField name="key" type="string">
  La clave del registro almacenado en la caché.
</ResponseField>

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de cadena en el caché con la clave especificada.

Los valores almacenados en este caché quedan limitados al trigger en el que
se establecen. Están sujetos a los [Límites del caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta forma tendrán una vigencia de *hasta* los valores
de `ttl` o `expires_at` especificados. Si no se especifica ninguna vigencia, se usará una
vigencia predeterminada de 15 minutos. La vigencia no puede superar la duración máxima
indicada en [Límites del caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Este caché está diseñado para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si aún están dentro de la vigencia indicada.

<ResponseField name="key" type="string">
  La clave del registro que se va a almacenar.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se va a almacenar.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento del caché.

  <Expandable title="propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora de vencimiento absoluta en milisegundos desde la época Unix.
      Aunque los registros almacenados en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `expires_at` proporcionado.

      *Nota*: No se debe proporcionar este valor si también se proporcionó un valor
      para `ttl`. Si se proporcionan ambas opciones, se usará
      el vencimiento más cercano.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada de caché en milisegundos.
      Aunque los valores almacenados en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `ttl` proporcionado.

      *Nota*: No se debe proporcionar este valor si también se proporcionó un valor
      para `expires_at`. Si se proporcionan ambas opciones, se usará
      el vencimiento más cercano.
    </ResponseField>
  </Expandable>
</ResponseField>
