El objeto de API del trigger Posterior al cambio de contraseña de Actions incluye:

<div id="apicache">
  ## `api.cache`
</div>

Almacena y recupera datos que se mantienen entre ejecuciones.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina el registro correspondiente a un valor almacenado en la caché en la clave proporcionada, si existe.

Devuelve un objeto `CacheWriteResult` con `type: "success"` si se eliminó un valor de la caché. Si la operación falla, devuelve `type: "error"`. En caso de error, el objeto devuelto tendrá una
propiedad `code` que indica la naturaleza del fallo.

| Parámetro | Descripción                                             |
| --------- | ------------------------------------------------------- |
| `key`     | *String*. La clave del registro almacenado en la caché. |

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en la
`key` proporcionada, si existe. Si se encuentra un registro, el valor almacenado en caché
se puede encontrar en la propiedad `value` del objeto devuelto.

Devuelve un registro de la caché si se encuentra un elemento en la caché para la
`key` proporcionada. Los registros de la caché son objetos con una
propiedad `value` que contiene el valor almacenado en caché, así como una
propiedad `expires_at` que indica la fecha de expiración máxima del
registro en milisegundos desde la época Unix.

**Importante:**
Esta caché está diseñada para datos efímeros de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si todavía se encuentran dentro de su
tiempo de vida.

| Parámetro | Descripción                                             |
| --------- | ------------------------------------------------------- |
| `key`     | *String*. La clave del registro almacenado en la caché. |

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, [options])`
</div>

Almacena o actualiza un valor de tipo cadena en el caché en la clave especificada.

Los valores almacenados en este caché se aplican al trigger en el que se establecen.
Están sujetos a los
[Límites del caché de Actions](/docs/es/customize/actions/limitations).

Los valores almacenados de esta forma tienen tiempos de vida de hasta los valores
`ttl` o `expires_at` especificados. Si no se especifica
ningún tiempo de vida, se usará un tiempo de vida predeterminado de 15 minutos. Los tiempos de vida no pueden
superar la duración máxima indicada en
[Límites del caché de Actions](/docs/es/customize/actions/limitations).

| Parámetro            | Descripción                                                                                                                                                                                                                                                                                                                                                                                        |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `key`                | *String*. La clave del registro almacenado en el caché.                                                                                                                                                                                                                                                                                                                                            |
| `value`              | *String*. El valor del registro que se almacenará.                                                                                                                                                                                                                                                                                                                                                 |
| `options`            | *Optional object*. Opciones para ajustar el comportamiento del caché.                                                                                                                                                                                                                                                                                                                              |
| `options.expires_at` | *Optional number*. La hora de vencimiento absoluta, en milisegundos desde la época unix. Aunque los registros en caché pueden eliminarse antes, nunca permanecerán más allá del valor `expires_at` proporcionado.<br />*Nota:* No debe proporcionarse este valor si también se proporcionó un valor para `ttl`. Si se proporcionan ambas opciones, se usará la expiración más temprana de las dos. |
| `options.ttl`        | *Optional number*. El valor de tiempo de vida de esta entrada de caché en milisegundos. Aunque los valores en caché pueden eliminarse antes, nunca permanecerán más allá del valor `ttl` proporcionado.<br />*Nota:* No debe proporcionarse este valor si también se proporcionó un valor para `expires_at`. Si se proporcionan ambas opciones, se usará la expiración más temprana de las dos.    |
