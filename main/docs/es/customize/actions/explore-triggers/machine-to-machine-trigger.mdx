El trigger de máquina a máquina se ejecuta cuando se emite un <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> mediante el [Client Credentials Flow](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow).

<Frame>![Diagrama que muestra el flujo de máquina a máquina de Actions y cuándo se ejecutan los triggers dentro de este.](/docs/images/cdy7uua7fh8z/1JPl54LFWCUh5StuglZS2o/41f89372526574c3b8cdac4d5ba38072/Machine_to_Machine_Flow.png)</Frame>

Las Actions de este flujo son bloqueantes (sincrónicas), lo que significa que se ejecutan como parte del proceso de un trigger e impiden que el resto del pipeline de Auth0 se ejecute hasta que la Action se complete.

  ## Triggers
</div>

<div id="m2m-client-credentials">
  ### M2M / Credenciales del cliente
</div>

El `trigger` `credentials-exchange` es una función que se ejecuta antes de devolver el token de acceso.

<div id="common-use-cases">
  ## Casos de uso habituales
</div>

<div id="access-control">
  ### Control de acceso
</div>

Se puede usar una Action de intercambio de credenciales para denegar un token de acceso basándose en una lógica personalizada.

```javascript lines
/**
 * @param {Event} event - Detalles sobre la solicitud de concesión de credenciales de cliente.
 * @param {CredentialsExchangeAPI} api - Interfaz cuyos métodos pueden usarse para cambiar el comportamiento de la concesión de credenciales de cliente.
 */
exports.onExecuteCredentialsExchange = async (event, api) => {
  if (event.request.geoip.continentCode === "NA") {
    api.access.deny('invalid_request', "Access from North America is not allowed.");
  }
};
```

<div id="add-custom-claims-to-the-access-token">
  ### Agregar claims personalizadas al token de acceso
</div>

Se puede usar una Action de intercambio de credenciales para agregar claims personalizadas a un token de acceso.

```javascript lines
/**
 * @param {Event} event - Detalles sobre la solicitud de Concesión de credenciales de cliente.
 * @param {CredentialsExchangeAPI} api - Interfaz cuyos métodos pueden usarse para modificar el comportamiento de la Concesión de credenciales de cliente.
 */
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.accessToken.setCustomClaim("https://my-api.exampleco.com/request-ip", event.request.ip);  
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Recomendamos encarecidamente usar un claim personalizado con un espacio de nombres en forma de URI. Para obtener más información sobre los claims personalizados con y sin espacio de nombres, consulta [Crear claims personalizados](/docs/es/secure/tokens/json-web-tokens/create-custom-claims).
</Callout>
