El objeto de API del trigger send-phone-message de Actions incluye:

<div id="apicache">
  ## `api.cache`
</div>

Almacena y recupera datos que se conservan entre ejecuciones.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina el registro que describe un valor almacenado en caché en la clave proporcionada, si existe.

Devuelve un objeto `CacheWriteResult` con `type: "success"` si se eliminó un valor de la caché. Si la operación falla, devuelve `type: "error"`. En caso de error, el objeto devuelto tendrá una propiedad
`code` que indica la naturaleza del fallo.

| Parámetro | Descripción                                             |
| --------- | ------------------------------------------------------- |
| `key`     | *String*. La clave del registro almacenado en la caché. |

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe el valor en caché asociado a la
`key` proporcionada, si existe. Si se encuentra un registro, el valor en caché
se puede encontrar en la propiedad `value` del objeto devuelto.

Devuelve un registro de caché si se encuentra un elemento en la caché para la
`key` proporcionada. Los registros de caché son objetos con una
propiedad `value` que contiene el valor en caché, así como una
propiedad `expires_at` que indica la hora máxima de expiración del
registro en milisegundos desde la época Unix.

**Importante:**
Esta caché está diseñada para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si todavía se encuentran dentro de su
tiempo de vida.

| Parámetro | Descripción                                             |
| --------- | ------------------------------------------------------- |
| `key`     | *String*. La clave del registro almacenado en la caché. |

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, [options])`
</div>

Almacena o actualiza un valor de tipo cadena en la caché en la clave especificada.

Los valores almacenados en esta caché están limitados al trigger en el que se establecen.
Están sujetos a los
[Límites de caché de Actions](/docs/es/customize/actions/limitations).

Los valores almacenados de esta forma tendrán tiempos de vida de hasta los valores
`ttl` o `expires_at` especificados. Si no se especifica ningún tiempo de vida,
se usará un tiempo de vida predeterminado de 15 minutos. Los tiempos de vida no pueden
superar la duración máxima indicada en
[Límites de caché de Actions](/docs/es/customize/actions/limitations).

| Parámetro            | Descripción                                                                                                                                                                                                                                                                                                                                                                                              |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `key`                | *String*. La clave del registro almacenado en la caché.                                                                                                                                                                                                                                                                                                                                                  |
| `value`              | *String*. El valor del registro que se almacenará.                                                                                                                                                                                                                                                                                                                                                       |
| `options`            | *Objeto opcional*. Opciones para ajustar el comportamiento de la caché.                                                                                                                                                                                                                                                                                                                                  |
| `options.expires_at` | *Número opcional*. La hora de vencimiento absoluta en milisegundos desde la época de Unix. Aunque los registros en caché pueden eliminarse antes, nunca permanecerán más allá del valor de `expires_at` proporcionado.<br />*Nota:* No se debe proporcionar este valor si también se proporcionó un valor para `ttl`. Si se proporcionan ambas opciones, se usará el vencimiento más cercano de las dos. |
| `options.ttl`        | *Número opcional*. El valor de tiempo de vida de esta entrada de caché en milisegundos. Aunque los valores en caché pueden eliminarse antes, nunca permanecerán más allá del valor de `ttl` proporcionado.<br />*Nota:* No se debe proporcionar este valor si también se proporcionó un valor para `expires_at`. Si se proporcionan ambas opciones, se usará el vencimiento más cercano de las dos.      |
