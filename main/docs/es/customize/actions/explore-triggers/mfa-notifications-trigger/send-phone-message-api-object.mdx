<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de cadena en el caché con la clave especificada.

Los valores almacenados en este caché quedan limitados al trigger en el que
se establecen. Están sujetos a los [Límites del caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta manera tendrán tiempos de vida de *hasta* los valores
de `ttl` o `expires_at` especificados. Si no se especifica ningún tiempo de vida, se usará un
tiempo de vida predeterminado de 15 minutos. Los tiempos de vida no pueden superar la duración máxima
indicada en [Límites del caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Este caché está diseñado para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si aún se encuentran dentro del tiempo de vida indicado.

<ResponseField name="key" type="string">
  La clave del registro que se almacenará.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se almacenará.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento del caché.

  <Expandable title="Propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>

      *Nota*: No debe proporcionarse este valor si también se proporcionó un valor
      para `expires_at`. Si se proporcionan ambas opciones, se utilizará

    </ResponseField>
  </Expandable>
</ResponseField>
