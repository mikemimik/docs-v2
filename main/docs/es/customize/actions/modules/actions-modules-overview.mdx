Los Módulos de Actions te permiten crear, gestionar y compartir código reutilizable entre diferentes Actions dentro de tu tenant de Auth0.

Anteriormente, cada Action funcionaba de forma independiente, lo que dificultaba compartir funciones comunes entre ellas sin usar dependencias externas, como módulos de NPM. Con Módulos de Actions, ahora es posible:

* Crear módulos personalizados con funciones alojadas en Auth0.
* Usar las funciones definidas en cada módulo personalizado en una o varias Actions.
* Cada módulo personalizado puede definir secretos independientes y añadir dependencias de módulos de NPM.

Para obtener más información sobre cómo crear Actions con Módulos de Actions, consulta [Escribe tu primer módulo de Action](/docs/es/customize/actions/modules/write-your-first-action-module).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los Módulos de Actions no están vinculados a un trigger ni a un tipo de Action en particular; por lo tanto, las funciones definidas en ellos se pueden reutilizar en Actions vinculadas a diferentes triggers.
</Callout>

Para obtener más información sobre las limitaciones de Módulos de Actions, consulta [Limitaciones de Actions](/docs/es/customize/actions/limitations).

<div id="performance">
  ## Rendimiento
</div>

Al adoptar Módulos de Actions de acuerdo con las [Coding Guidelines](/docs/es/customize/actions/action-coding-guidelines), puedes mejorar el rendimiento de ejecución compartiendo y reutilizando objetos cuya creación de instancias es costosa.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Sin embargo, la mejora no está garantizada, ya que depende del caso de uso concreto, la implementación y la carga y frecuencia de las ejecuciones.
</Callout>
