title: Gestionar dependencias
---

Actions te permite usar paquetes del [`npm` registry](https://www.npmjs.com/). Puedes instalar y gestionar las dependencias de tus Actions con el Actions Code Editor del <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o con la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Auth0 Management API</Tooltip> de Auth0.

<div id="add-a-dependency-using-the-actions-code-editor">
  ## Agregar una dependencia con el Actions Code Editor
</div>

Para agregar una dependencia de un paquete del registro público de `npm`:

1. Vaya a [Auth0 Dashboard &gt; Actions &gt; Library](https://manage.auth0.com/#/actions/library) y seleccione su Action.
2. Ubique el Actions Code Editor y seleccione Modules (icono de cubo) en la barra lateral.
3. Seleccione **Add Module** e introduzca el nombre del módulo. Para usar la versión más reciente disponible del paquete, puede dejar en blanco el campo de versión.
4. En su código de Actions, use `require` para importar el módulo. Por ejemplo, si agregó el paquete Axios, añadiría la siguiente línea al principio de su código en el Actions Code Editor:

   ```javascript lines
   const axios = require('axios');
   ```

<div id="add-a-dependency-using-the-auth0-management-api">
  ## Agregar una dependencia con la Auth0 Management API
</div>

Como alternativa, puedes agregar una dependencia con el [endpoint Create Action](https://auth0.com/docs/api/management/v2#!/Actions/post_action) de la Auth0 Management API. Al llamar al endpoint, especifica las dependencias en el payload:

```bash lines
curl -H "Authorization: Bearer {managementApiToken}" \
     -X POST  -H "Content-Type: application/json" \
     -d '{ \
            "name":"my-action", \
            "supported_triggers":[{"id":"post-login","version":"v2"}], \
            "code":"module.exports = () => {}",\
            "dependencies": [{\
                "name":"lodash",\
                "version":"4.17.21"\
            }],\
            "runtime":"node14"\
        }' \
    https://{yourTenant}.com/api/v2/actions/actions
```

<div id="security-considerations">
  ## Consideraciones de seguridad
</div>

Los paquetes de terceros pueden contener vulnerabilidades y código malicioso, por lo que recomendamos auditar los paquetes antes de usarlos en tus Actions. También te recomendamos supervisar continuamente los paquetes para detectar vulnerabilidades mediante herramientas de análisis de composición de software.

<div id="limitations-for-dependencies-in-actions">
  ## Limitaciones de las dependencias en Actions
</div>

<div id="no-native-module-support">
  ### No hay compatibilidad con módulos nativos
</div>

Los módulos nativos no se pueden usar en Actions. Al ejecutar una Action que dependa de un módulo nativo, la Action generará un error con un mensaje que contiene `Compilation failed: Cannot find module`.

Para comprobar si un paquete de `npm` es nativo, ve al repositorio público de Github del paquete y revisa en la barra lateral los lenguajes que usa el código. Si entre los lenguajes se incluyen C/C++, es probable que el paquete sea nativo. Algunos ejemplos populares son `bcrypt` y `sqlite3`.

<div id="no-private-module-or-private-registry-support">
  ### No se admiten módulos ni registros privados
</div>

Los módulos privados de `npm` y los registros privados de `npm` no se pueden usar con Actions. Actions usa el [registro público de `npm`](https://registry.npmjs.org).
