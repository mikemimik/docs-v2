Actions te permite usar paquetes del [`npm` registry](https://www.npmjs.com/). Puedes instalar y administrar dependencias para tus Actions con el Actions Code Editor en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Auth0 Management API</Tooltip>.&quot;

<div id="add-a-dependency-using-the-actions-code-editor">
  ## Agregar una dependencia usando el Actions Code Editor
</div>

Para agregar una dependencia de un paquete del registro público de `npm`:

1. Ve a [Auth0 Dashboard &gt; Actions &gt; Library](https://manage.auth0.com/#/actions/library) y selecciona tu Action.
2. Busca el Actions Code Editor y selecciona Modules (icono de cubo) en la barra lateral.
3. Selecciona **Add Module** y escribe el nombre del módulo. Para usar la versión más reciente disponible del paquete, puedes dejar en blanco el campo de versión.
4. En tu código de Actions, carga el módulo con `require`. Por ejemplo, si agregaste el paquete Axios, añadirías la siguiente línea al principio de tu código en el Actions Code Editor:

   ```javascript lines
   const axios = require('axios');
   ```

<div id="add-a-dependency-using-the-auth0-management-api">
  ## Agregar una dependencia mediante Auth0 Management API
</div>

Como alternativa, puedes agregar una dependencia mediante el [endpoint Create Action](https://auth0.com/docs/api/management/v2#!/Actions/post_action) de Auth0 Management API. Al llamar al endpoint, especifica las dependencias en el payload:

```bash lines
curl -H "Authorization: Bearer {managementApiToken}" \
     -X POST  -H "Content-Type: application/json" \
     -d '{ \
            "name":"my-action", \
            "supported_triggers":[{"id":"post-login","version":"v2"}], \
            "code":"module.exports = () => {}",\
            "dependencies": [{\
                "name":"lodash",\
                "version":"4.17.21"\
            }],\
            "runtime":"node14"\
        }' \
    https://{yourTenant}.com/api/v2/actions/actions
```

<div id="security-considerations">
  ## Consideraciones de seguridad
</div>

Los paquetes de terceros pueden contener vulnerabilidades y código malicioso, por lo que recomendamos auditar los paquetes antes de usarlos en tus Actions. También recomendamos supervisarlos continuamente en busca de vulnerabilidades mediante herramientas de análisis de composición de software.

<div id="limitations-for-dependencies-in-actions">
  ## Limitaciones de las dependencias en Actions
</div>

<div id="no-native-module-support">
  ### No se admiten módulos nativos
</div>

Los módulos nativos no se pueden usar en Actions. Al ejecutar una Action que depende de un módulo nativo, la Action mostrará un error con un mensaje que contiene `Compilation failed: Cannot find module`.

Para comprobar si un paquete de `npm` es nativo, ve al repositorio público de Github del paquete de `npm` y revisa en la barra lateral los lenguajes que usa el código. Si entre los lenguajes se incluyen C/C++, es probable que el paquete sea nativo. Algunos ejemplos populares son `bcrypt` y `sqlite3`.

<div id="no-private-module-or-private-registry-support">
  ### Sin compatibilidad con módulos ni registros privados
</div>

Los módulos privados de `npm` y los registros privados de `npm` no pueden usarse con Actions. Actions usa el [registro público de `npm`](https://registry.npmjs.org).
