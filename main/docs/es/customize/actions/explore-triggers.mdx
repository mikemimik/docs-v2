---
description: Aprende sobre los triggers, que son eventos que invocan automáticamente una Action cuando se produce una operación específica, como el inicio de sesión de un usuario, en runtime.
sidebarTitle: Información general
title: Explorar triggers
---

Los triggers son puntos del proceso de Auth0 en los que se pueden agregar Actions para la personalización necesaria. En un trigger se pueden colocar varias Actions, que se ejecutan en secuencia. A excepción de Custom Token Exchange, los triggers pueden ser síncronos (con bloqueo) o asíncronos (sin bloqueo). Cada trigger tiene objetos y API específicos. Estos pueden variar según tu conexión (como una conexión <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>).

Para explorar los objetos y las API de referencia de Actions específicas de un trigger, selecciona el trigger a continuación y ten en cuenta la definición de Auth0 de una [conexión passwordless](/docs/es/authenticate/passwordless).

<div id="signup-and-login">
  #### Registro e inicio de sesión
</div>

Cuando se agrega un usuario a una conexión de base de datos o passwordless (registro), o como parte de la transacción de authentication de un usuario (inicio de sesión).

| Trigger                  | Descripción                                                                            |
| ------------------------ | -------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se activa antes de que se cree un usuario.                                             |
| `post-user-registration` | Se activa de forma asíncrona después de que se crea un usuario.                        |
| `Post Login`             | Se activa después de que se autentica al usuario, pero antes de que se emita un token. |

<div id="mfa-notifications">
  #### Notificaciones de MFA
</div>

Al usar SMS como factor para la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> (MFA) o al configurar un proveedor personalizado.

| Trigger              | Descripción                                                                                             |
| -------------------- | ------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se activa al usar un proveedor personalizado para enviar los mensajes para la inscripción y el desafío. |

<div id="password-reset">
  #### Restablecimiento de contraseña
</div>

Cuando se inicia una solicitud de restablecimiento de contraseña.

| Trigger                         | Descripción                                                                                         |
| ------------------------------- | --------------------------------------------------------------------------------------------------- |
| `post-change-password`          | Se activa después de que se cambie una contraseña.                                                  |
| `password-reset-post-challenge` | Se activa después de completar el primer desafío y antes de que se restablezca la contraseña. |

<div id="machine-to-machine">
  #### De máquina a máquina
</div>

Cuando una aplicación, en lugar de un usuario, se autentica y se autoriza para usar tus APIs.

| Trigger                | Descripción                                            |
| ---------------------- | ------------------------------------------------------ |
| `credentials-exchange` | Se activa antes de que se devuelva un token de acceso. |

<div id="custom-token-exchange">
  ### Custom Token Exchange
</div>

Cuando se recibe una solicitud de Custom Token Exchange para iniciar la sesión de un usuario.

| Trigger                 | Descripción                                                                  |
| ----------------------- | ---------------------------------------------------------------------------- |
| `custom-token-exchange` | Se activa en el primer paso de la transacción, antes del trigger Post Login. |
