---
description: Aprende sobre los triggers, que son eventos que invocan automáticamente una Action cuando en runtime se produce una operación específica, como el inicio de sesión de un usuario.
sidebarTitle: Información general
title: Explora los triggers
---

Los triggers son puntos del proceso de Auth0 en los que se pueden agregar Actions para realizar la personalización necesaria. En un trigger se pueden colocar varias Actions, que se ejecutan en secuencia. Con la excepción de Custom Token Exchange, los triggers pueden ser síncronos (con bloqueo) o asíncronos (sin bloqueo). Cada trigger tiene objetos y API específicos. Estos pueden variar según tu conexión (como una conexión <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>).

Para explorar los objetos y las API de referencia de las Actions específicas de un trigger, selecciona el trigger a continuación, teniendo en cuenta la definición de Auth0 de una [conexión passwordless](/docs/es/authenticate/passwordless).

<div id="signup-and-login">
  #### Registro e inicio de sesión
</div>

Cuando se agrega un usuario a una conexión de base de datos o passwordless (registro), o como parte de la transacción de autenticación de un usuario (inicio de sesión).

| Trigger                  | Descripción                                                                            |
| ------------------------ | -------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se activa antes de que se cree el usuario.                                             |
| `post-user-registration` | Se activa de forma asíncrona después de que se crea el usuario.                        |
| `post-login`             | Se activa después de que se autentica al usuario, pero antes de que se emita un token. |

<div id="mfa-notifications">
  #### Notificaciones de MFA
</div>

Al usar SMS como factor para la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> (MFA) o al configurar un proveedor personalizado.

| Trigger              | Descripción                                                                                      |
| -------------------- | ------------------------------------------------------------------------------------------------ |
| `send-phone-message` | Se activa al usar un proveedor personalizado para enviar los mensajes de registro y del desafío. |

<div id="password-reset">
  #### Restablecimiento de contraseña
</div>

Cuando se inicia una solicitud de restablecimiento de contraseña.

| Trigger                         | Descripción                                                                                  |
| ------------------------------- | -------------------------------------------------------------------------------------------- |
| `post-change-password`          | Se activa después de que se cambia la contraseña.                                            |
| `password-reset-post-challenge` | Se activa después de completar la primera verificación y antes de restablecer la contraseña. |

<div id="machine-to-machine">
  #### Machine to Machine
</div>

Cuando una aplicación, en lugar de un usuario, se autentica y autoriza para usar tus API.

| Trigger                | Descripción                                            |
| ---------------------- | ------------------------------------------------------ |
| `credentials-exchange` | Se activa antes de que se devuelva un token de acceso. |

<div id="custom-token-exchange">
  ### Custom Token Exchange
</div>

Cuando se recibe una solicitud de Custom Token Exchange para iniciar sesión de un usuario.

| Trigger                 | Descripción                                                                     |
| ----------------------- | ------------------------------------------------------------------------------- |
| `custom-token-exchange` | Se activa en el primer paso de la transacción, antes del trigger de Post Login. |
