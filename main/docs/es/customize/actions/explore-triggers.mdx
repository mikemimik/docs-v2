---
description: Conozca los Triggers, que son eventos que invocan automáticamente una Action cuando, en tiempo de ejecución, se produce una operación específica, como el inicio de sesión de un usuario.
sidebarTitle: Descripción general
title: Explorar Triggers
---

Los Triggers son puntos del proceso de Auth0 en los que se pueden agregar Actions para realizar las personalizaciones necesarias. Se pueden colocar varias Actions en un Trigger, que se ejecutan en secuencia. Con la excepción de Custom Token Exchange, los Triggers pueden ser sincrónicos (bloqueantes) o asincrónicos (no bloqueantes). Cada Trigger tiene Objetos y API específicos. Estos pueden variar según su conexión (como una conexión <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>).

Para explorar los Objetos y las API de referencia de Actions específicas para un Trigger, seleccione el Trigger a continuación, teniendo en cuenta la definición de Auth0 de una [conexión passwordless](/docs/es/authenticate/passwordless).

<div id="signup-and-login">
  #### Registro e inicio de sesión
</div>

Cuando se añade un usuario a una conexión de base de datos o passwordless (registro), o como parte del proceso de autenticación de un usuario (inicio de sesión).

| Trigger                | Descripción                                                                               |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se activa antes de que se cree un usuario.                                                |
| `post-user-registration` | Se activa de forma asíncrona después de que se cree un usuario.                           |
| `post-login`             | Se activa después de que se autentique a un usuario, pero antes de que se emita un token. |

<div id="mfa-notifications">
  #### Notificaciones de MFA
</div>

Al usar SMS como factor de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">autenticación multifactor</Tooltip> (MFA) o al configurar un proveedor personalizado.

| Trigger            | Descripción                                                                                                      |
| -------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se activa al usar un proveedor personalizado para enviar los mensajes del proceso de registro y de verificación. |

<div id="password-reset">
  #### Restablecimiento de contraseña
</div>

Cuando se inicia una solicitud de restablecimiento de contraseña.

| Trigger                       | Descripción                                                                                  |
| ------------------------------- | -------------------------------------------------------------------------------------------- |
| `post-change-password`          | Se activa después de cambiar una contraseña.                                                 |
| `password-reset-post-challenge` | Se activa después de completar la primera verificación y antes de restablecer la contraseña. |

<div id="machine-to-machine">
  #### De máquina a máquina
</div>

Cuando una aplicación, en lugar de un usuario, se autentica y queda autorizada para usar tus API.

| Trigger                | Descripción                                            |
| ---------------------- | ------------------------------------------------------ |
| `credentials-exchange` | Se activa antes de que se devuelva un token de acceso. |

<div id="custom-token-exchange">
  ### Custom Token Exchange
</div>

Cuando se recibe una solicitud de Custom Token Exchange para iniciar la sesión de un usuario.

| Trigger                 | Descripción                                                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------ |
| `custom-token-exchange` | Se activa en el primer paso de la transacción, antes del Trigger posterior al inicio de sesión. |
