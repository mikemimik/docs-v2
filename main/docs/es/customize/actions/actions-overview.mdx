Actions son funciones seguras, específicas del tenant y [versionadas](/docs/es/customize/actions/manage-versions), escritas en Node.js, que se ejecutan en determinados puntos de la plataforma Auth0. Actions se utilizan para personalizar y ampliar las capacidades de Auth0 con lógica personalizada.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 Actions utiliza el [modelo de programación basado en promesas](https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Async_JS/Promises) de JavaScript moderno, que sirve de base para las funciones asíncronas en Actions.
</Callout>

Los procesos que se pueden ampliar de esta forma se denominan triggers. Las agrupaciones de triggers representan el flujo lógico por el que se desplaza la información durante un punto concreto del recorrido en Auth0. Se pueden añadir varias Actions a un trigger, y cada Action se ejecuta en el orden en que se agregó. Algunos triggers se ejecutan de forma síncrona, bloqueando el flujo en el que intervienen, y otros se ejecutan de forma asíncrona, como se indica en la siguiente tabla.

Actions son una pieza fundamental de nuestro producto global de extensibilidad en Auth0. Con Actions, puedes añadir lógica personalizada esencial a tus flujos de inicio de sesión e identidad según tus necesidades. Actions también te permiten conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Por ejemplo, puedes añadir una Action a tu trigger de inicio de sesión para verificar credenciales, como una licencia o un pasaporte, mediante un Marketplace Partner especializado en verificación de identidad.

<div id="what-can-you-do-with-actions">
  ## ¿Qué puedes hacer con Actions?
</div>

Lo que una Action puede hacer depende de dónde se ejecute dentro del entorno de ejecución de Auth0.

<AccordionGroup>
  <Accordion title="Registro e inicio de sesión: Cuando se agrega un usuario a una conexión de base de datos o sin contraseña.">
    | trigger                  | Descripción                                                                                             |
    | ------------------------ | ------------------------------------------------------------------------------------------------------- |
    | `pre-user-registration`  | Se activa antes de que se cree un usuario. No se ejecuta para conexiones sociales.                      |
    | `post-user-registration` | Se activa de forma asíncrona después de que se crea un usuario. No se ejecuta para conexiones sociales. |
    | `post-login`             | Se activa después de que se autentica a un usuario, pero antes de que se emita el token.                |
  </Accordion>

  <Accordion title="Notificaciones de MFA: Al usar SMS como factor para la autenticación multifactor (MFA) o para configurar un proveedor personalizado.">
    | trigger              | Descripción                                                                                                                                                   |
    | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `send-phone-message` | Se activa al usar un proveedor personalizado para enviar mensajes durante los procesos de inscripción y verificación. No se ejecuta para conexiones sociales. |
  </Accordion>

  <Accordion title="Restablecimiento de contraseña: Cuando se inicia una solicitud de restablecimiento de contraseña.">
    | trigger                         | Descripción                                                                                               |
    | ------------------------------- | --------------------------------------------------------------------------------------------------------- |
    | `post-change-password`          | Se activa después de que se cambia una contraseña. No se ejecuta para conexiones sociales.                |
    | `password-reset-post-challenge` | Se activa después de que se completa la primera verificación y antes de que se restablezca la contraseña. |
  </Accordion>

  <Accordion title="Máquina a máquina: Cuando se autentica y autoriza a una aplicación, en lugar de a un usuario, para usar tus API.">
    | trigger                | Descripción                                            |
    | ---------------------- | ------------------------------------------------------ |
    | `credentials-exchange` | Se activa antes de que se devuelva un token de acceso. |
  </Accordion>
</AccordionGroup>

<div id="key-benefits-of-actions">
  ## Principales beneficios de Actions
</div>

<div id="improved-developer-experience">
  ### Experiencia mejorada para desarrolladores
</div>

* Al editar una Action en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, tendrás acceso a información de tipos enriquecida y documentación integrada sobre lo que es posible en cada trigger, lo que facilita descubrir qué capacidades admite cada Trigger.
* Una Action se puede editar y probar sin afectar a la versión que actualmente gestiona el tráfico de producción.
* Si se detecta un problema en una Action, se puede revertir a una versión anterior.

<div id="reuse-custom-functions">
  ### Reutiliza funciones personalizadas
</div>

* Crea, administra y comparte funciones entre Actions con [Actions Modules](/docs/es/customize/actions/modules/actions-modules-overview).

<div id="access-to-npm-packages">
  ### Acceso a paquetes de npm
</div>

* Puedes usar casi todos los [paquetes públicos de `npm`](https://www.npmjs.com/) en Actions.

<div id="observability">
  ### Observabilidad
</div>

* Cuando Auth0 ejecuta Actions, captura métricas clave sobre ellas y las vincula a los [Logs de Auth0](/docs/es/deploy-monitor/logs).
* Para depurar, usa la función [Logs en tiempo real de Actions](/docs/es/customize/actions/actions-real-time-logs).

<div id="multiple-actions-on-every-trigger">
  ### Varias acciones en cada trigger
</div>

* Cada trigger de Action admite varias Actions independientes.
