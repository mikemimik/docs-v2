Actions son funciones seguras, específicas del tenant y [versionadas](/docs/es/customize/actions/manage-versions), escritas en Node.js, que se ejecutan en determinados puntos de la plataforma Auth0. Actions se usan para personalizar y ampliar las capacidades de Auth0 con lógica personalizada.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 Actions utiliza el [modelo de programación basado en promesas](https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Async_JS/Promises) de JavaScript moderno, que sirve de base para las funciones asíncronas en Actions.
</Callout>

Los procesos que se pueden ampliar de esta manera se llaman triggers. Las agrupaciones de triggers representan el pipeline lógico por el que se mueve la información en un punto determinado del recorrido en Auth0. Se pueden agregar varias Actions a un trigger, y cada Action se ejecuta en el orden en que se colocó. Algunos triggers se ejecutan de forma sincrónica, bloqueando el flujo en el que participan, y otros se ejecutan de forma asíncrona, como se indica en la tabla siguiente.

Actions son una pieza fundamental de nuestro producto global de extensibilidad en Auth0. Con Actions, puedes añadir lógica personalizada esencial a tus flujos de login e identidad según tus necesidades. Actions también te permiten conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Por ejemplo, puedes agregar una Action a tu trigger de login para verificar credenciales como una licencia o un pasaporte mediante un partner de Marketplace especializado en verificación de identidad.

<div id="what-can-you-do-with-actions">
  ## ¿Qué puedes hacer con Actions?
</div>

Lo que puede hacer una Action depende de dónde se ejecute dentro del entorno de runtime de Auth0.

<AccordionGroup>
  <Accordion title="Registro e inicio de sesión: cuando se agrega un usuario a una conexión de base de datos o passwordless.">
    | Trigger                  | Descripción                                                                                             |
    | ------------------------ | ------------------------------------------------------------------------------------------------------- |
    | `pre-user-registration`  | Se activa antes de que se cree un usuario. No se ejecuta para conexiones sociales.                      |
    | `post-user-registration` | Se activa de forma asíncrona después de que se crea un usuario. No se ejecuta para conexiones sociales. |
    | `post-login`             | Se activa después de que un usuario se autentica, pero antes de que se emita el token.                  |
  </Accordion>

  <Accordion title="Notificaciones de MFA: al usar SMS como factor para la autenticación multifactor (MFA) o para configurar un proveedor personalizado.">
    | Trigger              | Descripción                                                                                                                             |
    | -------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
    | `send-phone-message` | Se activa al usar un proveedor personalizado para enviar los mensajes de inscripción y desafío. No se ejecuta para conexiones sociales. |
  </Accordion>

  <Accordion title="Restablecimiento de contraseña: cuando se inicia una solicitud de restablecimiento de contraseña.">
    | Trigger                         | Descripción                                                                                         |
    | ------------------------------- | --------------------------------------------------------------------------------------------------- |
    | `post-change-password`          | Se activa después de que se cambia una contraseña. No se ejecuta para conexiones sociales.          |
    | `password-reset-post-challenge` | Se activa después de que se completa el primer desafío y antes de que se restablezca la contraseña. |
  </Accordion>

  <Accordion title="Machine to Machine: cuando una aplicación, en lugar de un usuario, se autentica y autoriza para usar tus APIs.">
    | Trigger                | Descripción                                            |
    | ---------------------- | ------------------------------------------------------ |
    | `credentials-exchange` | Se activa antes de que se devuelva un token de acceso. |
  </Accordion>
</AccordionGroup>

<div id="key-benefits-of-actions">
  ## Principales ventajas de Actions
</div>

<div id="improved-developer-experience">
  ### Experiencia mejorada para desarrolladores
</div>

* Al editar una Action en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, tendrá acceso a información de tipos detallada y documentación integrada sobre lo que es posible dentro de cada trigger, lo que facilita descubrir qué capacidades ofrece cada trigger.
* Una Action puede editarse y probarse sin afectar a la versión que actualmente gestiona el tráfico de producción.
* Si se encuentra una incidencia en una Action, puede revertirse a una versión anterior.

<div id="reuse-custom-functions">
  ### Reutiliza funciones personalizadas
</div>

* Crea, gestiona y comparte funciones entre Actions con [Módulos de Actions](/docs/es/customize/actions/modules/actions-modules-overview).

<div id="access-to-npm-packages">
  ### Acceso a paquetes de npm
</div>

* Puedes usar casi todos los [paquetes públicos de `npm`](https://www.npmjs.com/) en Actions.

<div id="observability">
  ### Observabilidad
</div>

* Cuando Auth0 ejecuta Actions, captura métricas clave sobre su ejecución y las vincula a los [registros de Auth0](/docs/es/deploy-monitor/logs).
* Al depurar, usa la función [logs en tiempo real de Actions](/docs/es/customize/actions/actions-real-time-logs).

<div id="multiple-actions-on-every-trigger">
  ### Varias Actions en cada trigger
</div>

* Cada trigger de Action admite varias Actions independientes.
