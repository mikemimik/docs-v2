title: Limitaciones de Actions
---

Existen las siguientes limitaciones al usar Actions:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para obtener más información sobre los límites de entidades, consulte: [Límites de entidades de Actions](/docs/es/troubleshoot/customer-support/operational-policies/entity-limit-policy#actions).
</Callout>

<div id="actions">
  ## Actions
</div>

* Cada Action no debe superar los 100 kB. Cuanto mayor sea el tamaño, más latencia se introduce, lo que puede afectar el rendimiento de su sistema. Este límite de tamaño no incluye ningún módulo de `npm` al que se haga referencia como parte de cualquier instrucción `require`.

<div id="actions-modules">
  ## Módulos de Actions
</div>

* Cada módulo de Action puede tener dependencias de módulos de NPM, pero no de otros Módulos de Actions.
* **Los Módulos de Actions no tienen su propio runtime.** Se ejecutan en el runtime de la Action que los incluye.

<Warning>
  **Cuando compilas un módulo de Action, Auth0 lo compila con el runtime del builder. Sin embargo, cuando el módulo se ejecuta, usa el runtime de la Action que lo importa.**

  *Ejemplo:*

  *Si tu módulo de Action usa llamadas a la API de Node.js que quedaron obsoletas en Node.js 22, las Actions que usan el runtime de Node.js 18 funcionarán bien, pero las Actions que usan el runtime de Node.js 22 fallarán.*
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Escribe Módulos de Actions con APIs de Node.js que sean compatibles con todas las versiones de runtime que planeas admitir.
</Callout>

<div id="account-linking-setprimaryuser">
  ## Vinculación de cuentas (setPrimaryUser)
</div>

* `primary_user_id` tiene un límite de 128 caracteres
* Solo se puede llamar a `setPrimaryUser` una vez por transacción
* Cualquier `userMetadata` establecida en la misma Action que `setPrimaryUser` se descarta y se perderá. Las Actions posteriores dentro de la misma transacción conservarán `userMetadata` en el nuevo usuario principal.
* `setPrimaryUser` no puede usarse en la misma transacción en la que una Rule establece `context.primaryUser.`

<div id="cached-data">
  ## Datos en caché
</div>

* Los elementos en caché se conservan durante un máximo de 24 horas.
* Se pueden almacenar en caché un máximo de 20 entradas por trigger.
* Las claves de caché tienen un tamaño máximo de 64 bytes y los valores, un tamaño máximo de 4 kB.
* El tamaño acumulado de las claves almacenadas en caché y sus valores no debe superar los 8 kB.
* La caché debería estar disponible de forma fiable para todas las Actions del mismo trigger durante una sola ejecución; sin embargo, no se garantiza en ejecuciones posteriores (por ejemplo, en un flujo distinto, en el inicio de sesión de otro usuario o cuando un usuario regresa de una Action de redirección).
* Las Actions que realizan una ejecución que devuelve el control (como una redirección) pueden hacer que las Actions posteriores se programen en una instancia distinta con un estado de caché diferente. Los datos en caché pueden ser incoherentes de una Action a la siguiente, incluso si se trata de la misma ejecución.

<div id="executions">
  ## Executions
</div>

* Cada ejecución de un trigger debe completarse en 20 segundos o menos; de lo contrario, el procesamiento terminará con un error. Limitar las solicitudes HTTP es la mejor manera de mantenerse dentro de este límite de tiempo.
* Cada ejecución de un trigger debe completarse en 20 segundos o menos; de lo contrario, el procesamiento terminará con un error. Para mantenerse dentro de este límite de tiempo, es necesario limitar los procesos de larga duración, como las solicitudes HTTP salientes sin un timeout. Una Action que [redirige a los usuarios a una página externa](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/redirect-with-actions) tiene un timeout independiente antes del Redirect y otro después.
* Se crea un nuevo objeto `event.request` cada vez que un trigger de Action se suspende y luego se reanuda (por ejemplo, debido a un Redirect o a una verificación de <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>).

<div id="logs">
  ## Registros
</div>

* Se pueden almacenar de forma persistente hasta 256 caracteres de la salida de `console.log()` por cada Action.
* Los registros de ejecución se conservan durante 10 días.

<div id="programming-languages">
  ## Lenguajes de programación
</div>

* No se admite TypeScript en Actions. Los archivos fuente deben escribirse en JavaScript antes de implementarse.

<div id="secrets">
  ## Secrets
</div>

* Cada clave de un Secret puede tener una longitud máxima de 128 caracteres.
* Cada valor de un Secret puede tener una longitud máxima de 4096 caracteres.

<div id="saml-attributes">
  ## Atributos SAML
</div>

* Actions puede cambiar o agregar un máximo de 100 atributos <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>.
* Los nombres de los atributos SAML tienen un tamaño máximo de 1 kB.
* Los valores SAML tienen un tamaño máximo de 2 kB.
* El tamaño total de las aserciones SAML tiene un máximo de 10 kB.

<div id="saml-configuration">
  ## Configuración de SAML
</div>

* `audience` tiene un tamaño máximo de 2kB
* `recipient` tiene un tamaño máximo de 2kB
* `destination` tiene un tamaño máximo de 2kB
* `nameIdentifierFormat` tiene un tamaño máximo de 0.5kB
* `nameIdentifierProbes` tiene un máximo de 10 comprobaciones, con un tamaño máximo de 0.5kB cada una
* `authnContextClassRef` tiene un tamaño máximo de 0.5kB
* `signingCert` tiene un tamaño máximo de 4kB
* `encryptionCert` tiene un tamaño máximo de 4kB
* `encryptionPublicKey` tiene un tamaño máximo de 4kB
* `cert` tiene un tamaño máximo de 4kB
* `key` tiene un tamaño máximo de 4kB

<div id="service-requests">
  ## Solicitudes de servicio
</div>

* Las llamadas realizadas a la <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 y las actualizaciones de User Metadata están sujetas a [límites de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy/management-api-endpoint-rate-limits).
* Las llamadas HTTP a servicios externos se originan desde [estas direcciones IP](/docs/es/secure/security-guidance/data-security/allowlist).

<div id="transaction-metadata">
  ## Metadatos de la transacción
</div>

* Está disponible solo en [Actions de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).
* No se conserva una vez que finaliza un trigger de autenticación.
* No se puede acceder a ellos fuera de las Actions de la misma transacción.
* Las claves están limitadas a 64 caracteres.
* Los valores están limitados a 8 KB.
* Los valores solo admiten los tipos `string`, `number` y `boolean`.
* Tiene un tamaño total máximo de metadatos de 16 KB dentro de la misma transacción.
* No aceptará como valores válidos números que no superen una comprobación de seguridad. Los valores enteros no seguros deben serializarse de forma segura por el desarrollador. Para obtener más información, consulta [safe integers](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/isSafeInteger).

<div id="upstream-tokens-from-external-idps">
  ## Tokens de proveedores de identidad externos
</div>

* Recuperación de tokens de <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip> externos del arreglo Identities

<div id="user-metadata-and-app-metadata">
  ## Metadatos de usuario y metadatos de la aplicación
</div>

* Cada sesión puede tener un máximo de 32 kB para la persistencia de metadatos de usuario y 32 kB para la persistencia de metadatos de la aplicación.
