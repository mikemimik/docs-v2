<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos detallada, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprende cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y a [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

El objeto `context` almacena información contextual sobre la transacción de autenticación actual, como la dirección IP, la aplicación o la ubicación del usuario.

Si cambias el contenido del token usando el objeto `context` dentro de una rule, tus cambios estarán disponibles en los tokens después de que todas las rules terminen de ejecutarse. Si tu aplicación también requiere <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multifactor+authentication">autenticación multifactor</Tooltip> o consentimiento del usuario, se le pedirá al usuario que complete ese paso antes de que los cambios en el token estén disponibles.

| `context.request`            | Un objeto que contiene información útil de la solicitud. Este valor también puede establecerse en `undefined`. Tiene las siguientes propiedades:<ul> <li>`userAgent`: el agente de usuario de la aplicación que intenta iniciar sesión.</li> <li>`ip`: la dirección IP de origen del usuario que intenta iniciar sesión. Exponemos direcciones IPv6 en nuestros endpoints públicos (por ejemplo, travel0.us.auth0.com). Si una solicitud llega desde una máquina compatible con IPv6, esto contendrá una dirección IPv6. Si manipula direcciones IP manualmente, le sugerimos usar la [biblioteca ipaddr.js@1.9.0](https://www.npmjs.com/package/ipaddr.js/v/1.9.0).</li> <li>`hostname`: el hostname que se está utilizando para el flujo de autenticación.</li> <li>`query`: un objeto que contiene los parámetros de la cadena de consulta de la transacción de inicio de sesión enviada por la aplicación.</li> <li>`body`: el cuerpo de la solicitud POST en las transacciones de inicio de sesión utilizadas con los protocolos `oauth2-resource-owner`, `oauth2-resource-owner-jwt-bearer` o `wstrust-usernamemixed`.</li> <li>`geoip`: un objeto que contiene información geográfica de la IP. Tiene las siguientes propiedades:<ul> <li>`country_code`: un código de dos caracteres para el país asociado a la dirección IP.</li> <li>`country_code3`: un código de tres caracteres para el país asociado a la dirección IP.</li> <li>`country_name`: el nombre del país asociado a la dirección IP.</li> <li>`city_name`: el nombre de la ciudad o localidad asociado a la dirección IP.</li> <li>`latitude`: la latitud asociada a la dirección IP.</li> <li>`longitude`: la longitud asociada a la dirección IP.</li> <li>`time_zone`: la zona horaria asociada a la dirección IP.</li> <li>`continent_code`: un código de dos caracteres para el continente asociado a la dirección IP.</li> <li>`subdivision_code`: el código ISO 3166-2 de la subdivisión o región de nivel superior.</li> <li>`subdivision_name`: el nombre en inglés de esta subdivisión o región.</li></ul></li></ul> |
| `context.primaryUser`        | El ID de usuario único de la cuenta principal del usuario. Se utiliza para [vincular cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking#automatic-account-linking) de distintos proveedores de identidad.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `context.authentication`     | Un objeto que contiene información relacionada con la transacción de autenticación con las siguientes propiedades:<br /><br />`methods`: un arreglo de objetos que contiene los métodos de autenticación que un usuario ha completado durante su sesión. Por ejemplo, un usuario que ha completado una autenticación basada en contraseña seguida de MFA puede tener los siguientes métodos:<br />`json lines [ { "name": "pwd", "timestamp": 1434454643024 }, { "name": "mfa", "timestamp": 1534454643881 } ] `<br />Los objetos de método contienen las siguientes propiedades: <ul> <li> `name`: una cadena que representa el nombre del método de autenticación que se ha completado. Puede ser uno de los siguientes valores (en el futuro podrían admitirse valores adicionales):<br /><ul> <li>`federated`: se utilizó una conexión social o empresarial para autenticar al usuario</li> <li>`pwd`: se utilizó una conexión de base de datos para autenticar al usuario</li> <li>`sms`: se utilizó una conexión SMS de [Passwordless](/docs/es/glossary?term=Passwordless) para autenticar al usuario</li> <li>`email`: se utilizó una conexión Passwordless Email para autenticar al usuario</li> <li>`mfa`: el usuario completó una autenticación multifactor</li> </ul> </li> <li>`timestamp`: un entero que indica la hora, en segundos, en la que tuvo lugar el método de autenticación en tiempo Unix epoch</li> </ul> Puedes ver un ejemplo de caso de uso de la propiedad `context.authentication.methods` en la [Rule Require MFA once per session](https://github.com/auth0/rules/blob/master/src/rules/require-mfa-once-per-session.js).                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `context.authorization`      | Un objeto que contiene información relacionada con la transacción de autorización con las siguientes propiedades:<ul> <li>`roles`: un arreglo de cadenas que contiene los nombres de los [roles](/docs/es/glossary?term=roles) asignados a un usuario. Puedes ver un caso de uso de ejemplo que utiliza la propiedad `context.authorization.roles` para agregar roles a los tokens en [Casos de uso de ejemplo: Rules con autorización](/docs/es/manage-users/access-control/sample-use-cases-rules-with-authorization#add-user-roles-to-tokens).</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |

Incluye las siguientes propiedades:<ul> <li>`id`: String que contiene el ID de la organización con la que el usuario inicia sesión.</li><li>`name`: String que contiene el nombre de la organización (tal como se define en el Auth0 Dashboard).</li><li>`metadata`: Dictionary de pares clave/valor de tipo string que contiene otras propiedades de la organización.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
