<Warning>
  La fecha de End of Life (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Le recomendamos encarecidamente que use Actions para ampliar Auth0. Con Actions, tiene acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lea [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarle con su migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarle en su proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

El objeto `context` almacena información contextual sobre la transacción de autenticación actual, como la dirección IP, la aplicación o la ubicación del usuario.

Si cambia el contenido del token usando el objeto `context` dentro de una rule, sus cambios estarán disponibles en los tokens después de que todas las reglas terminen de ejecutarse. Si su aplicación también requiere <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multifactor+authentication">autenticación multifactor</Tooltip> o consentimiento del usuario, se le solicitará antes de que los cambios en el token estén disponibles.

| `context.request`            | Un objeto que contiene información útil sobre la solicitud. También se puede establecer como `undefined`. Tiene las siguientes propiedades:<ul> <li>`userAgent`: el agente de usuario de la aplicación que está intentando iniciar sesión.</li> <li>`ip`: la dirección IP de origen del usuario que está intentando iniciar sesión. Exponemos direcciones IPv6 en nuestros endpoints públicos (por ejemplo, travel0.us.auth0.com). Si una solicitud llega desde una máquina compatible con IPv6, este campo contendrá una dirección IPv6. Tiene las siguientes propiedades:<ul> <li>`country_code`: un código de dos caracteres del país asociado a la dirección IP.</li> <li>`country_code3`: un código de tres caracteres del país asociado a la dirección IP.</li> <li>`country_name`: el nombre del país asociado a la dirección IP.</li> <li>`city_name`: el nombre de la ciudad o localidad asociado a la dirección IP.</li> <li>`latitude`: la latitud asociada a la dirección IP.</li> <li>`longitude`: la longitud asociada a la dirección IP.</li> <li>`time_zone`: la zona horaria asociada a la dirección IP.</li> <li>`continent_code`: un código de dos caracteres del continente asociado a la dirección IP.</li> <li>`subdivision_code`: el código ISO 3166-2 de la subdivisión/región de nivel superior.</li> <li>`subdivision_name`: el nombre en inglés de esta subdivisión/región.</li></ul></li></ul> |
| `context.primaryUser`        | El ID de usuario único de la cuenta principal del usuario. Se utiliza para [vincular cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking#automatic-account-linking) de distintos proveedores de identidad.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

Por ejemplo, un usuario que ha completado una autenticación basada en contraseña seguida de MFA puede tener los siguientes métodos:<br />`json lines [ { "name": "pwd", "timestamp": 1434454643024 }, { "name": "mfa", "timestamp": 1534454643881 } ] `<br />Los objetos de método contendrán las siguientes propiedades: <ul> <li> `name`: una cadena que representa el nombre del método de autenticación que se ha completado. Puede ser uno de los siguientes valores (en el futuro podrían admitirse valores adicionales):<br /><ul> <li>`federated`: se utilizó una conexión social o empresarial para autenticar al usuario</li> <li>`pwd`: se utilizó una conexión de base de datos para autenticar al usuario</li> <li>`sms`: se utilizó una conexión SMS de [Passwordless](/docs/es/glossary?term=Passwordless) para autenticar al usuario</li> <li>`email`: se utilizó una conexión Passwordless Email para autenticar al usuario</li> <li>`mfa`: el usuario completó una autenticación multifactor</li> </ul> </li> <li>`timestamp`: un entero que indica el momento, en segundos, en que tuvo lugar el método de autenticación en tiempo Unix Epoch</li> </ul> Puede ver un caso de uso de ejemplo de la propiedad `context.authentication.methods` en la [Rule Require MFA once per session](https://github.com/auth0/rules/blob/master/src/rules/require-mfa-once-per-session.js).                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

Puedes ver un caso de uso de ejemplo en el que se usa la propiedad `context.authorization.roles` para agregar roles a tokens en [Casos de uso de ejemplo: Rules con Authorization](/docs/es/manage-users/access-control/sample-use-cases-rules-with-authorization#add-user-roles-to-tokens).</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
