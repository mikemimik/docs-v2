<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Le recomendamos encarecidamente que use Actions para ampliar Auth0. Con Actions, tiene acceso a información de tipos detallada, documentación integrada y paquetes `npm` públicos, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lea [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarle con la migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparativas de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarle en su proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

El objeto `context` almacena información contextual sobre la transacción de autenticación actual, como la dirección IP, la aplicación o la ubicación del usuario.

Si cambia el contenido del token usando el objeto `context` dentro de una rule, los cambios estarán disponibles en los tokens después de que todas las reglas terminen de ejecutarse. Si su aplicación también requiere <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multifactor+authentication">autenticación multifactor</Tooltip> o consentimiento del usuario, se le pedirá que complete ese paso antes de que los cambios en el token estén disponibles.

| `context.request`            | Un objeto que contiene información útil de la solicitud. También puede establecerse como `undefined`. Tiene las siguientes propiedades:<ul> <li>`userAgent`: el user-agent de la aplicación que intenta iniciar sesión.</li> <li>`ip`: la dirección IP de origen del usuario que intenta iniciar sesión. Tiene las siguientes propiedades:<ul> <li>`country_code`: un código de dos caracteres correspondiente al país asociado con la dirección IP.</li> <li>`country_code3`: un código de tres caracteres correspondiente al país asociado con la dirección IP.</li> <li>`country_name`: el nombre del país asociado con la dirección IP.</li> <li>`city_name`: el nombre de la ciudad o localidad asociada con la dirección IP.</li> <li>`latitude`: la latitud asociada con la dirección IP.</li> <li>`longitude`: la longitud asociada con la dirección IP.</li> <li>`time_zone`: la zona horaria asociada con la dirección IP.</li> <li>`continent_code`: un código de dos caracteres correspondiente al continente asociado con la dirección IP.</li> <li>`subdivision_code`: el código ISO 3166-2 de la subdivisión o región de nivel superior.</li> <li>`subdivision_name`: el nombre en inglés de esta subdivisión o región.</li></ul></li></ul> |

Puede ser uno de los siguientes valores (es posible que en el futuro se admitan valores adicionales):<br /><ul> <li>`federated`: se usó una conexión social o empresarial para autenticar al usuario</li> <li>`pwd`: se usó una conexión de base de datos para autenticar al usuario</li> <li>`sms`: se usó una conexión SMS de [Passwordless](/docs/es/glossary?term=Passwordless) para autenticar al usuario</li> <li>`email`: se usó una conexión Passwordless Email para autenticar al usuario</li> <li>`mfa`: el usuario completó una autenticación multifactor</li> </ul> </li> <li>`timestamp`: un entero que indica la hora, en segundos, en que se produjo el método de autenticación, en tiempo de época Unix</li> </ul> Puedes ver un caso de uso de ejemplo de la propiedad `context.authentication.methods` en la [Require MFA once per session Rule](https://github.com/auth0/rules/blob/master/src/rules/require-mfa-once-per-session.js).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
