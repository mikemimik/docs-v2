En Rules puedes almacenar recursos costosos usando el objeto `global`. El objeto `global` se conserva entre ejecuciones individuales, por lo que puedes reutilizar lo que almacenes allí en lugar de crearlo cada vez que se ejecuta una Rule. Sin embargo, el entorno de Rules puede reciclarse en cualquier momento, por lo que tu código **debe** comprobar siempre que `global` contiene lo que esperas.

Este ejemplo muestra cómo usar el objeto `global` para conservar una conexión de MongoDB:

```javascript lines
//Si el objeto db está disponible, úsalo.
if (global.db){
  return query(global.db, callback);
}

//Si no, obtén la db (mongodb en este caso)
mongo('mongodb://user:pass@mymongoserver.com/my-db',  function (db){
  global.db = db;
  return query(db, callback);
});

//Realiza el trabajo real
function query(db, cb){
  // Haz algo con db
  ...
  return cb(null, user, context);
}
```

Para obtener más información sobre el uso de MongoDB, consulta [el manual de MongoDB](https://docs.mongodb.com/manual/).
