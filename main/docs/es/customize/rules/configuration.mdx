<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes `npm` públicos, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con tu migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarte en tu proceso de migración.

  Si quieres obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

El objeto de configuración `configuration` global está disponible en Rules para almacenar valores de uso común, como URL. La información confidencial, como las credenciales o las claves de API, debe almacenarse mediante el objeto de configuración `configuration` y mantenerse fuera del código de tus Rules.

<div id="configure-values">
  ## Configurar valores
</div>

Puedes establecer valores de configuración en la [configuración de Rules](https://manage.auth0.com/#/rules/) del Dashboard.

Para editar o cambiar el valor de una clave de configuración, elimina la configuración existente y sustitúyela por el valor actualizado. Debes haber creado al menos una Rule para ver el área de configuración; de lo contrario, se mostrará la demo de Rules.

<div id="use-the-configuration-object">
  ## Usa el objeto de configuración
</div>

Puedes acceder a cualquier valor de configuración que establezcas mediante una clave a través del objeto `configuration` en tu código de Rules.

```javascript lines
var MY_API_KEY = configuration.MY_API_KEY;
```

El siguiente ejemplo es una Rule para enviar un mensaje de Slack cuando un usuario nuevo se registra. El [Slack Webhook](https://api.slack.com/incoming-webhooks) es un valor de `configuration` establecido con la clave `SLACK_HOOK_URL`.

```javascript lines
function (user, context, callback) {
  // omitir si el usuario ya se registró o está usando un token de actualización
  if (context.stats.loginsCount > 1 || context.protocol === 'oauth2-refresh-token') {
    return callback(null, user, context);
  }

  // obtén la URL del hook de Slack en: https://slack.com/services/10525858050
  const SLACK_HOOK = configuration.SLACK_HOOK_URL;

  const slack = require('slack-notify')(SLACK_HOOK);
  const message = 'New User: ' + (user.name || user.email) + ' (' + user.email + ')';
  const channel = '#some_channel';

  slack.success({
    text: message,
    channel: channel
  });

  // no esperes a que finalice la llamada a la API de Slack, regresa de inmediato (la solicitud continuará en el sandbox)`
  callback(null, user, context);
}
```
