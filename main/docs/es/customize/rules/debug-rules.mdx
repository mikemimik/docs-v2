<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para tenants nuevos creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para extender Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprende cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada de blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Exponemos direcciones IPv6 en nuestros endpoints públicos (por ejemplo, `travel0.us.auth0.com`). Si una solicitud llega desde una máquina compatible con IPv6, la propiedad `context.request.ip` contendrá una dirección IPv6. Si realizas una manipulación manual de direcciones IP, te sugerimos usar la [biblioteca ipaddr.js@1.9.0](https://www.npmjs.com/package/ipaddr.js/v/1.9.0).
</Warning>

Para depurar cualquier [Rule de Auth0](/docs/es/customize/rules) que hayas creado, puedes usar `console.log` dentro del código de tu Rule. Puedes ver la salida de `console.log` con **Save and Try**, consultando los logs disponibles en [logs en tiempo real de Actions](/docs/es/customize/actions/actions-real-time-logs) o, en el caso de clientes Legacy, usando [Debug Rule CLI](#debug-rule-cli).

<div id="try-this-rule">
  ## Prueba esta Rule
</div>

En el [Rules Editor](https://manage.auth0.com/#/rules/create), el botón **Save and Try** te permite ejecutar una Rule de forma aislada con objetos **user** y **context** simulados. Al hacer clic en **TRY**, la Rule se ejecutará con esos dos objetos como entrada y mostrará cualquier salida de `console.log`.

La funcionalidad **Try this rule script** se ejecuta fuera del contexto de un cliente específico. Al usar esta funcionalidad, puedes encontrarte con problemas si tu Rule depende de datos que se proporcionarían al invocarla desde una aplicación real.

<div id="actions-real-time-logs">
  ## logs en tiempo real de Actions
</div>

Los logs en tiempo real de Actions muestran en tiempo real todos los logs del código personalizado de tu cuenta, incluidos todos los resultados de `console.log` y las excepciones. Para obtener más información, consulta [logs en tiempo real de Actions](/docs/es/customize/actions/actions-real-time-logs).
