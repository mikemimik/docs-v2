<Warning>
  La fecha de fin de vida (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida.

  Recomendamos encarecidamente que uses Actions para ampliar las capacidades de Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lee [Comprende cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

  Para ayudarte con tu migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la deprecación de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

El objeto `user` almacena información sobre el usuario que ha iniciado sesión, devuelta por el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>. Se genera cuando un usuario se autentica y antes de que se ejecuten las Rules. Debido al orden de los eventos cuando un usuario se autentica, los cambios realizados en el perfil de un usuario desde una Rule solo estarán disponibles en el objeto de usuario actual si también guardas los cambios en el objeto `user` desde esa misma Rule.

Para obtener más información sobre el flujo de la transacción de autenticación, lee la sección &quot;Cómo funcionan las Rules&quot; en [Crear Rules](/docs/es/customize/rules/create-rules).

<Warning>
  Usa nombres únicos para las propiedades raíz del objeto de usuario y las propiedades dentro de `app_metadata`. Como las propiedades dentro de `app_metadata` se fusionan con el objeto de usuario, las propiedades raíz que comparten nombres con propiedades de `app_metadata` pueden ver sus valores sobrescritos al ejecutar Rules.
</Warning>
