<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Recomendamos encarecidamente usar Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprender cómo funcionan Auth0 Actions](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías para [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica, [Pasarse a Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarte durante la migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Prepararse para el fin de vida útil de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

El objeto `user` almacena información sobre el usuario que inició sesión, devuelta por el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>. Se genera cuando un usuario se autentica y antes de que se ejecuten las Rules. Debido al orden de los eventos durante la autenticación de un usuario, los cambios realizados en el perfil de un usuario desde una Rule solo estarán disponibles en el objeto `user` actual si también guardas esos cambios en el objeto `user` desde esa misma Rule.

Para obtener más información sobre el flujo de transacciones de autenticación, consulta la sección &quot;Cómo funcionan las rules&quot; en [Crear Rules](/docs/es/customize/rules/create-rules).

<Warning>
  Usa nombres únicos para las propiedades raíz del objeto `user` y las propiedades dentro de `app_metadata`. Como las propiedades de `app_metadata` se combinan con el objeto `user`, las propiedades raíz que comparten nombre con propiedades de `app_metadata` pueden ver sus valores sobrescritos al ejecutar Rules.
</Warning>
