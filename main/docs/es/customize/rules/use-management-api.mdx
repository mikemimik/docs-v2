<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para extender Auth0. Con Actions, tienes acceso a información de tipos detallada, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia de extensibilidad en general. Para obtener más información sobre lo que ofrece Actions, consulta [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Move to Actions](https://auth0.com/platform/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

Desde cualquier [Rule de Auth0](/docs/es/customize/rules) que escribas, puedes actualizar el `app_metadata` o `user_metadata` de un usuario mediante el objeto `auth0`, que es una instancia de `ManagementClient` con restricciones especiales (definida en la biblioteca cliente de Node.js [node-auth0](https://github.com/auth0/node-auth0)) y proporciona acceso limitado a la [Auth0 Management API](https://auth0.com/docs/api/management/v2).

Para acceder a endpoints adicionales de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> desde Rules, debes usar otra versión de la biblioteca.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El token de acceso para la Management API, disponible a través de `auth0.accessToken`, está limitado a los scopes `read:users` y `update:users`. Si necesitas un conjunto más amplio de scopes, puedes solicitar un token mediante el Client Credentials Flow. Consulta [Obtener tokens de acceso de la Management API para producción](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).
</Callout>

<div id="access-a-newer-version-of-the-library">
  ## Accede a una versión más reciente de la biblioteca
</div>

Puedes cargar una versión más reciente de la biblioteca cliente de Node.js de Auth0 indicando la versión específica de la biblioteca. Para obtener información actualizada sobre las versiones, consulta el [repositorio de Node de Auth0](https://github.com/auth0/node-auth0) en Github.

En este ejemplo, cargamos la versión `2.9.1` de la biblioteca, luego consultamos la lista de usuarios y los mostramos en la consola (para inspeccionarlos con [logs en tiempo real de Actions](/docs/es/customize/actions/actions-real-time-logs)).

<Warning>
  [Buscar usuarios](/docs/es/manage-users/user-search/user-search-best-practices) desde Rules puede afectar al rendimiento de tus inicios de sesión; te recomendamos no hacerlo.
</Warning>

```javascript lines
function (user, context, callback) {
  var ManagementClient = require('auth0@2.9.1').ManagementClient;
  var management = new ManagementClient({
    token: auth0.accessToken,
    domain: auth0.domain
  });

  management.getUsers(function (err, users) {
    console.log(users);
    callback(null, user, context);
  });
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para ver una lista filtrada de las bibliotecas disponibles que pueden configurarse como obligatorias, consulta las [versiones de biblioteca disponibles](https://auth0-extensions.github.io/canirequire/#auth0).
</Callout>
