  Para obtener más información sobre la obsolescencia de Rules y Hooks, lee nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).

Puedes crear tus propias rules para satisfacer tus necesidades específicas de funcionalidad. Puedes modificar una plantilla de rule existente o empezar desde cero con uno de nuestros ejemplos. Auth0 ofrece varias rules y plantillas de rule preexistentes para ayudarte a alcanzar tus objetivos. Para ver una lista, visita nuestro[repositorio de reglas en GitHub](https://github.com/auth0/rules).

<div id="how-rules-work">
  ## Cómo funcionan las reglas
</div>

Las reglas son funciones de JavaScript que se ejecutan cuando un usuario se autentica en tu aplicación. Se ejecutan una vez completado el proceso de autenticación, y puedes usarlas para personalizar y ampliar las capacidades de Auth0. Por motivos de seguridad, el código de tus reglas se ejecuta de forma aislada del código de otros tenants de Auth0 en un sandbox. Las reglas también se ejecutan durante el flujo de actualización de tokens. Para obtener más información, consulta [Tokens de actualización](/docs/es/secure/tokens/refresh-tokens).

En Auth0, el flujo de la transacción de autenticación funciona de la siguiente manera cuando usas reglas:

<Frame>![Diagrama de las reglas en el flujo de autenticación](/docs/images/cdy7uua7fh8z/2gtBtkPChIyguA24x6enx2/ffbb8e21e86920ef9914f6fc126dc1df/flow.png)</Frame>

1. Una aplicación inicia una solicitud de autenticación a Auth0.
2. Auth0 envía la solicitud a un proveedor de identidad a través de una conexión configurada.
3. El usuario se autentica correctamente.
4. El token de ID y/o el token de acceso pasa por el pipeline de reglas y luego se envía a la aplicación.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Realiza una llamada `POST` al [endpoint Create Rule](https://auth0.com/docs/api/management/v2#!/Rules/post_rules). Asegúrate de reemplazar los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `RULE_NAME`, `RULE_SCRIPT`, `RULE_ORDER` y `RULE_ENABLED` por tu <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip>, el nombre de la Rule, el script de la Rule, el número de orden de la Rule y el valor que indica si la Rule está habilitada, respectivamente.

| Valor                     | Descripción                                                                                                                                                                                                                                                  |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MGMT_API_ACCESS_TOKEN`   | Token de acceso para la Management API con el `scope` `create:rules`.                                                                                                                                                                                        |
| `RULE_NAME`               | Nombre de la regla que quieres crear. El nombre de la regla solo puede contener caracteres alfanuméricos, espacios y guiones; no puede empezar ni terminar con espacios ni guiones.                                                                            |
| `RULE_SCRIPT`             | Script que contiene el código de la regla. Debe coincidir con lo que introducirías al crear una nueva regla desde el Dashboard.                                                                                                                                |
| `RULE_ORDER` (opcional)   | Entero que representa el orden en que debe ejecutarse la regla con respecto a otras reglas. Las reglas con números más bajos se ejecutan antes que las reglas con números más altos. Si no se proporciona ningún número de orden, la regla se ejecutará al final. |
| `RULE_ENABLED` (opcional) | Booleano que indica si la regla está habilitada (`true`) o deshabilitada (`false`).                                                                                                                                                                           |

<Warning>
  Exponemos direcciones IPv6 en nuestros endpoints públicos (por ejemplo, `travel0.us.auth0.com`). Si una request llega desde una máquina compatible con IPv6, la propiedad `context.request.ip` contendrá una dirección IPv6. Si manipulas direcciones IP manualmente, te recomendamos usar la [biblioteca ipaddr.js@1.9.0](https://www.npmjs.com/package/ipaddr.js/v/1.9.0).
</Warning>

<div id="manage-rate-limits">
  ## Administrar límites de frecuencia
</div>

En las reglas que llaman a las API de Auth0, siempre debes controlar la limitación de frecuencia verificando el encabezado `X-RateLimit-Remaining` y actuando de forma adecuada cuando el valor devuelto se acerque a 0. También debes añadir lógica para gestionar los casos en los que superes los límites de frecuencia establecidos y recibas el código de estado HTTP 429 (Too Many Requests); en ese caso, si necesitas reintentar, lo mejor es aplicar un back-off para evitar entrar en un bucle infinito de reintentos. Para obtener más información sobre los límites de frecuencia, consulta la [Política de límite de tasa para las API de Auth0](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy).

<div id="available-modules">
  ## Módulos disponibles
</div>

Las reglas se ejecutan en un sandbox de JavaScript configurado para una versión específica de Node.js.

El sandbox es compatible con todas las versiones del lenguaje JavaScript (y la sintaxis asociada) correspondientes a la versión de Node.js configurada, así como con una gran cantidad de módulos de Node.js. Para obtener una lista de los módulos compatibles con el sandbox, consulta [Can I require: Auth0 Extensibility](https://auth0-extensions.github.io/canirequire/).
