<div id="language-selection">
  ### Selección de idioma
</div>

Auth0 selecciona el idioma para renderizar las páginas en función de:

* Los idiomas compatibles con Auth0, que se enumeran arriba
* La lista de idiomas configurados en la sección [**Configuración del tenant**](https://manage.auth0.com/#/tenant) de Auth0 Dashboard, donde puedes seleccionar los idiomas que admite tu tenant y elegir uno predeterminado. De forma predeterminada, la lista solo tiene seleccionado el inglés, pero puedes elegir los que necesites.
* El valor del parámetro `ui_locales` enviado al [endpoint de Authorization Request](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest), que puede usarse para limitar la lista de idiomas de una aplicación o sesión. Puedes proporcionar una lista de locales separada por espacios. El primer locale de la lista debe coincidir con un locale habilitado en tu tenant para que se refleje en la UI.
* El encabezado HTTP `Accept-Language` enviado por el navegador; Auth0 renderiza las páginas en este idioma si está permitido por la configuración anterior. Si no, las páginas se renderizan en el idioma predeterminado.

Usa la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 para hacer una llamada `PATCH` al [endpoint de Tenants](https://auth0.com/docs/api/management/v2/tenants/patch-settings) con el siguiente cuerpo, reemplazando `he` por el código de idioma que quieras agregar:

```json lines
{
  "enabled_locales": [
    "en","he"
  ]
}
```

Después de añadirlo, especifica el idioma en el parámetro de consulta `ui_locales` de la solicitud de inicio de sesión.

También puedes especificar los idiomas habilitados para el tenant mediante la Management API usando el [endpoint Update Tenants](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings). El idioma predeterminado es el primero de la lista.

<div id="right-to-left-languages">
  ### Idiomas de derecha a izquierda
</div>

<Warning>
  La compatibilidad con idiomas de derecha a izquierda está disponible en acceso anticipado. Para solicitar esta funcionalidad, [contacta con el equipo de soporte de Auth0](https://support.auth0.com/). Al usar esta funcionalidad, aceptas los términos aplicables de la prueba gratuita del [Master Subscription Agreement](https://www.okta.com/legal/) de Okta. Para obtener más información sobre el ciclo de lanzamiento de productos de Auth0, consulta [las fases de lanzamiento del producto](/docs/es/troubleshoot/product-lifecycle/product-release-stages).
</Warning>

La compatibilidad con idiomas de derecha a izquierda (RtL) permite que tu tenant muestre texto en árabe (estándar, egipcio y saudí), persa, hebreo y urdu. Los tenants que usan texto RtL deben cumplir con [WCAG 2.2 AA](/docs/es/authenticate/login/auth0-universal-login#accessibility) y, si tu tenant usa plantillas de página, debes actualizar la etiqueta HTML para incluir el elemento `dir`.

```liquid lines
{% assign resolved_dir = dir | default: "auto" %}
<html lang="{{locale}}" dir="{{resolved_dir}}">
```

<div id="limitations">
  ### Limitaciones
</div>

* El parámetro `ui_locales` solo puede usarse en flujos de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip>, ya que no está disponible con Security Assertion Markup Language <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=SAML">(SAML)</Tooltip> ni con WS-Federation.
* El parámetro `ui_locales` no se reenvía a los proveedores de identidad de origen <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdPs">IdPs</Tooltip>. Para obtener más información sobre cómo pasar parámetros a los IdPs, consulta [Pasar parámetros a los proveedores de identidad](/docs/es/authenticate/identity-providers/pass-parameters-to-idps).
* No es posible localizar los scopes en la página de consentimiento.

<div id="known-issues">
  ### Problemas conocidos
</div>

* La página de Universal Login renderiza el atributo HTML `lang` para el código de idioma `fr-FR` como `fr`.
* La página de Universal Login renderiza el atributo HTML `lang` para el código de idioma `pt-PT` como `pt`.

<div id="classic-login-localization">
  ## Localización de Classic Login
</div>

En la experiencia de Classic Login, la localización utiliza los widgets de JavaScript de Auth0 para [login](/docs/es/customize/internationalization-and-localization/lock-internationalization), la [página de restablecimiento de contraseña](/docs/es/customize/login-pages/classic-login/customize-password-reset-page) y las [políticas de contraseña](/docs/es/customize/internationalization-and-localization/password-options-translation).

De forma predeterminada, la [página de autenticación multifactor (MFA)](/docs/es/secure/multi-factor-authentication/customize-mfa/customize-mfa-classic-login) utiliza el widget de Auth0 <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>, que no se puede localizar. Puede crear versiones localizadas con [guardian.js](https://github.com/auth0/auth0-guardian.js).

No es posible localizar la página de [consentimiento](/docs/es/customize/login-pages/customize-consent-prompts).
