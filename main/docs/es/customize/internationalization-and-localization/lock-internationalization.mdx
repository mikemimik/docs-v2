<div id="set-language-option">
  ## Establecer la opción de idioma
</div>

Para usar los siguientes ejemplos, primero tendrás que incluir Lock en tu página:

`<script src="https://cdn.auth0.com/js/lock/11.14/lock.min.js"></script>`

Luego, define tu objeto `options` e incluye la opción `language`. La opción `language` debe ser un String que coincida con el nombre del archivo correspondiente en el directorio `i18n` [dentro de Lock](https://github.com/auth0/lock/tree/master/src/i18n). Después, instancia Lock.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample1 = `// Selecciona un idioma compatible
var options = {
  language: 'es'
};

// Inicializamos Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample1} language="javascript" />

Los datos de traducción de Lock provienen de archivos de idioma que contienen pares clave-valor con distintas traducciones. En algunos idiomas, es posible que falten ciertos valores; en ese caso, verás una advertencia: `language does not have property <missing prop>`. Te animamos a enviar un [pull request](https://github.com/auth0/lock/tree/master/src/i18n) para agregar esos valores faltantes. Como alternativa, puedes definirlos en las `options` de Lock (consulta a continuación).

<div id="replace-dictionary-terms">
  ## Reemplazar términos del diccionario
</div>

También puedes personalizar elementos específicos de tu propio diccionario mediante la opción `languageDictionary`. Esto resulta útil si usas uno de los idiomas compatibles, pero quieres cambiar la redacción de algunos elementos concretos. Por ejemplo, podrías reformular el `title` o cambiar cómo se muestran otras etiquetas al usuario, dejando intacto el resto del texto del widget.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample2 = `// Personaliza algunos atributos de languageDictionary
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Iniciar sesión"
  },
};

// Inicializamos Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample2} language="javascript" />

Para ver un ejemplo de los nombres de propiedad `languageDictionary` disponibles y de cómo estructurar un archivo `language`, consulta el [archivo de diccionario en inglés de Lock](https://github.com/auth0/lock/blob/master/src/i18n/en.js).

La opción `languageBaseUrl`, que acepta un valor de tipo String (una URL), sobrescribe la URL de origen del idioma para las traducciones proporcionadas por Auth0. De forma predeterminada, utiliza la URL de la CDN de Auth0 `https://cdn.auth0.com`, ya que allí se almacenan las traducciones de idioma proporcionadas. Si proporcionas otro valor, puedes usar tu propia fuente para las traducciones de idioma según sea necesario para tus applications. La fuente del idioma debe ser un archivo Javascript.
