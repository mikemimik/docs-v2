El **Settings Query Hook** te permite personalizar la apariencia y la experiencia de la extensión de Administración delegada.

<div id="hook-contract">
  ## Contrato del Hook
</div>

* **ctx**: Objeto `context`.

  * **request.user**: Usuario que ha iniciado sesión.
  * **locale**: `locale` (como se infiere de la URL) -- `https://{yourTenant}.us.webtask.io/auth0-delegated-admin/en/users` establecerá **locale** en `en`.
* **callback(error, settings)**: Devolución de llamada mediante la cual puedes devolver un error y un objeto de configuración.

## Ejemplo de uso

```javascript lines
function(ctx, callback) {
  var department = ctx.request.user.app_metadata && ctx.request.user.app_metadata.department;

  return callback(null, {
    // Solo estas conexiones deben ser visibles en el selector de conexiones. Si solo hay una conexión disponible, el selector de conexiones no se mostrará en la interfaz de usuario.
    connections: [ 'Username-Password-Authentication', 'My-Custom-DB' ],
    // El diccionario permite sobrescribir el título del Dashboard y la etiqueta "Memberships" en el diálogo Crear usuario.
    dict: {
      title: department ? department + ' User Management' : 'User Management Dashboard',
      memberships: 'Departments',
      menuName: ctx.request.user.name
    },
    // La opción CSS permite inyectar un archivo CSS personalizado según el contexto del usuario actual (p. ej.: un CSS diferente para cada cliente)
    css: (department && department !== 'IT') && 'https://cdn.jsdelivr.net/gh/auth0-extensions/auth0-delegated-administration-extension/docs/theme/fabrikam.css',
    // Esta opción permite restringir la creación de nuevos usuarios
    canCreateUser: (department === 'IT')
  });
}
```

<div id="properties">
  ### Propiedades
</div>

* **connections**: Lista de las conexiones en las que este administrador puede crear y editar usuarios.
* **dict**: Diccionario que permite sobrescribir el título del Dashboard y la etiqueta **Memberships** en el diálogo Crear usuario.

  * **dict.title**: Título que se mostrará en la parte superior de la UI.
  * **dict.memberships**: Etiqueta que se establecerá para los campos de membresía.
  * **dict.menuName**: Nombre que se establecerá para el menú desplegable de la esquina superior derecha.
  * **dict.logoutUrl**: URL alternativa para la opción de menú de cierre de sesión.
* **userFields**: Arreglo de campos de usuario (consulte &quot;campos personalizados&quot; a continuación).
* **css**: URL de tipo String para importar CSS.
* **altcss**: URL de tipo String para importar un segundo conjunto de CSS. Puede usarla para especificar, por ejemplo, CSS de accesibilidad para fuentes más grandes. Se mostrará al usuario una opción de menú para activar o desactivar este conjunto de CSS.
* **languageDictionary**: URL de tipo String o Dictionary Object (consulte &quot;localización&quot; a continuación).
* **suppressRawData**: Establézcalo en **true** para omitir las páginas que muestran Raw JSON
* **errorTranslator**: Función que traduce mensajes de error en función de la localización. Ejemplo: `(function (error, languageDictionary) { return languageDictionary.customErrors[error] || error; }).toString()`
* **canCreateUser**: Indicador Boolean. Si se establece en `false`, elimina el botón `Crear usuario` y prohíbe crear nuevos usuarios; `true` de forma predeterminada.

<div id="custom-fields">
  ## Campos personalizados
</div>

A partir de la versión 3.0 de la extensión Delegated Admin, puedes definir campos personalizados y especificar sus valores. Los campos personalizados pueden almacenarse en los campos de **metadata del usuario** y **app metadata** a los que se puede acceder durante los procesos de creación o actualización de usuarios.

También puedes personalizar campos existentes definidos por Auth0, como el correo electrónico, el username, el nombre y la conexión.

Para usar campos personalizados, debes:

* Agregar tu lista de **userFields** al Settings Query Hook
* Implementar un Write Hook. Los campos personalizados requieren el uso del Write Hook para actualizar correctamente `user_metadata` y `app_metadata`. Debes actualizar el objeto de usuario que se pasa a la función de devolución de llamada con `user_metadata` y `app_metadata` del contexto (objeto `ctx`) proporcionado al Hook.

Para obtener más información sobre los Write Hooks, consulta [Administración delegada: Write Hook](/docs/es/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-write-hook).

Esquema de ejemplo para **userFields**:

```js lines expandable
userFields: [
    {
        "property": string, // requerido
        "label": string,
        "sortProperty": string,
        "display": true || function.toString(),
        "search": false || {
            "display": true || function.toString()
            "listOrder": 1,
            "listSize": string(###%), // p. ej. 15%
            "filter": boolean,
            "sort": boolean
        },
        "edit": false || {
            "display": true || function.toString()
            "type": "text || select || password || hidden",
            "component": "InputText || Input Combo || InputMultiCombo || InputSelectCombo",
            "options": Array(string) || Array ({ "value": string, "label": string }),
            "disabled": true || false,
            "validationFunction": function.toString()
        },
        "create": false || {
            "display": true || function.toString()
            "type": "text || select || password || hidden",
            "component": "InputText || Input Combo || InputMultiCombo || InputSelectCombo",
            "options": Array(string) || Array ({ "value": string, "label": string }),
            "disabled": true || false,
            "validationFunction": function.toString()
        }
    },
    ...
]
```

* **property** (**obligatorio**): Nombre de la propiedad del objeto **ctx.payload** para el Write Hook. En el Write Hook, `"property": "app_metadata.dbId"` establece `ctx.payload.app_metadata.dbId`.
* **label**: label que se usará al añadir una etiqueta al campo en la página de información del usuario, la página de creación, la página de edición del perfil o la página de búsqueda.
* **sortProperty**: Si en la tabla de búsqueda se ordena por un campo distinto de este, use este campo. Se permite la notación por puntos.
* **display**: true || false || stringified =&gt; Este es el valor de visualización predeterminado. Si no se redefine en search, edit o create, se usará este valor.

  * si `true`, simplemente devolverá `user.<property>`.
  * Por defecto: si `false`, este valor no se mostrará en ninguna página (a menos que se sobrescriba en search, edit o Create).
  * si se trata de una función convertida en cadena, ejecuta la función para obtener el valor que se mostrará. Ejemplo: `(function display(user, value, languageDictionary) { return moment(value).fromNow(); }).toString()`
* **search**: false || object =&gt; Indica cómo se comportará este campo en la página de búsqueda.

  * Valor predeterminado: si es `false`, no aparecerá en la tabla de búsqueda.
  * **search.display**: Sobrescribe el valor de visualización predeterminado.
  * **search.listOrder**: Especifica el orden de las columnas en la tabla de resultados de búsqueda.
  * **search.listSize**: Especifica el ancho predeterminado de la columna.
  * **search.filter**: Especifica si se permite buscar este campo en el menú desplegable de búsqueda. El valor predeterminado es false.
  * **search.sort**: Especifica si esta columna se puede ordenar. Usa sortProperty si quieres ordenar por un campo distinto de property. El valor predeterminado es false.
* **edit**: false || objeto =&gt; Describe si el campo se muestra en los cuadros de diálogo de edición. Si no es un campo predeterminado y se define como un objeto, aparecerá en la página `Change Profile`, en el menú desplegable User Actions de la página de usuario.

  * Predeterminado: si es `false`, no se mostrará en ninguna página de edición/actualización.
  * **edit.display**: Sobrescribe el valor de visualización predeterminado.
  * **edit.required**: Establézcalo en `true` para que falle si no tiene un valor. El valor predeterminado es `false`.
  * **edit.type** **required**: text || select || password
  * **edit.component**: InputText || Input Combo || InputMultiCombo || InputSelectCombo

    * **InputText** (predeterminado): Cuadro de texto simple.
    * **InputCombo**: Menú desplegable con búsqueda, con un solo valor.
    * **InputMultiCombo**: Menú desplegable con búsqueda que permite varios valores.
    * **InputSelectCombo**: Menú desplegable de selección de opciones.
  * **edit.options**: Si el componente es InputCombo, InputMultiCombo o InputSelectCombo, se deben especificar los valores de las opciones.

    * **Array(string)**: Arreglo de valores (los campos `label` y `value` se establecerán con el mismo valor).
    * **Array(&#123; &quot;value&quot;: string, &quot;label&quot;: string &#125;)**: Permite establecer valores distintos para `value` y `label`. Esto hará que el valor en el Write Hook tenga ese mismo valor, pero puede reducirse para dejar solo `value` en el Write Hook.
    * La validación del lado del servidor garantizará que cualquier valor especificado para este campo aparezca en el arreglo de opciones.
  * **edit.disabled**: `true` si el componente debe ser de solo lectura; el valor predeterminado es `false`.
  * **edit.validateFunction**: Función convertida en String para validación. Tenga en cuenta que esta función de validación se ejecutará tanto del lado del servidor como del lado del cliente. Ejemplo: `(function validate(value, values, context, languageDictionary) { if (value...) return 'something went wrong'; return false; }).toString()`.
* **create**: false || objeto =&gt; Indica si el campo aparece en el cuadro de diálogo de creación.

  * Predeterminado: si es `false`, no aparecerá en la página de creación.
  * **create.placeholder**: Proporciona un texto de marcador de posición para mostrar cuando la entrada esté vacía.
  * **create.required**: Establécelo en true para que falle si no tiene un valor. El valor predeterminado es false.
  * **create.type** **required**: text || select || password
  * **create.component**: InputText || Input Combo || InputMultiCombo || InputSelectCombo

    * **InputText** (predeterminado): Cuadro de texto. Es el predeterminado para los tipos text y password.
    * **InputCombo**: Menú desplegable con búsqueda, con un solo valor.
    * **InputMultiCombo**: Menú desplegable con búsqueda que permite varios valores.
    * **InputSelectCombo**: Menú desplegable de selección de opciones.
  * **create.options**: Si el componente es uno de InputCombo, InputMultiCombo o InputSelectCombo, se deben especificar los valores de las opciones.

    * **Array(string)**: Arreglo simple de valores; label y value se establecerán con el mismo valor.
    * **Array(&#123; &quot;value&quot;: string, &quot;label&quot;: string &#125;)**: Permite establecer valores independientes para value y label. Como resultado, el valor en el Write Hook tendrá ese mismo valor, aunque puede reducirse para dejar solo el valor en el Write Hook.
    * La validación del lado del servidor garantizará que cualquier valor especificado para este campo esté en el arreglo options.
  * **create.disabled**: true si el componente debe ser de solo lectura; el valor predeterminado es false.
  * **create.validateFunction**: Función convertida en String para comprobar la validación.

    * Ejemplo: `(function validate(value, values, context, languageDictionary) { if (value...) return 'something went wrong'; return false; }).toString()`
    * Esta función de validación se ejecutará tanto del lado del servidor como del lado del cliente.

<div id="predefined-fields">
  ## Campos predefinidos
</div>

Hay un conjunto de campos predefinidos que se pueden buscar para el comportamiento predeterminado.

Puedes anular el comportamiento predeterminado añadiendo el campo como userField y luego anulando el comportamiento que quieras cambiar. Esto suele hacerse para ocultar un campo estableciendo display en false.

<div id="search-fields">
  ### Campos de búsqueda
</div>

* **name**: Campo generado a partir de otros campos: función de visualización predeterminada: `(function(user, value) { return (value || user.nickname || user.email || user.user_id); }).toString()`
* **email**: Dirección de correo electrónico o N/A
* **last&#95;login&#95;relative**: Hora del último inicio de sesión
* **logins&#95;count**: Número de inicios de sesión
* **connection**: conexión de base de datos

<div id="user-info-fields">
  ### Campos de información del usuario
</div>

* **user&#95;id**: ID de usuario
* **name**: Nombre del usuario
* **username**: Nombre de usuario
* **email**: Correo electrónico del usuario
* **identity.connection**: Valor de la conexión
* **isBlocked**: Si el usuario está bloqueado o no
* **blocked&#95;for**: Si el usuario tiene bloqueos por <Tooltip tip="Protección contra ataques: funcionalidades que Auth0 proporciona para detectar y mitigar ataques, incluida la protección contra ataques de fuerza bruta, la limitación de IP sospechosas, la detección de contraseñas vulneradas, la detección de bots y la autenticación multifactor adaptativa." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip>
* **last&#95;ip**: Última IP que usó el usuario para iniciar sesión
* **logins&#95;count**: Número de veces que el usuario ha iniciado sesión
* **currentMemberships**: Lista de membresías de este usuario
* **created&#95;at**: Fecha y hora en que se creó el usuario
* **updated&#95;at**: Fecha y hora en que se actualizó el usuario
* **last&#95;login**: Fecha y hora del último inicio de sesión del usuario

<div id="create-and-edit-user-fields">
  ### Crear y editar campos de usuario
</div>

* **connection**: Base de datos del usuario
* **password**: Nueva contraseña
* **repeatPassword**: Confirmación de la contraseña del usuario
* **email**: Correo electrónico del usuario
* **username**: Nombre de usuario

<div id="localization">
  ## Localización
</div>

A partir de la versión 3.0 de la extensión Delegated Admin, puede proporcionar un diccionario de idiomas para usarlo con la localización. El diccionario de idiomas se usa solo para el contenido estático de la página; para el contenido a nivel de campo, debe usar las etiquetas de **userFields**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La localización está pensada para quienes trabajan con funciones no administrativas al gestionar usuarios. Actualmente, Auth0 no admite la localización en ninguna de las páginas de configuración.
</Callout>

Para especificar la locale, puede usar la ruta. Por ejemplo: `https://{yourTenant}.us.webtask.io/auth0-delegated-admin/en/users` establecerá `context.locale` en `en` en la consulta de configuración.

El **languageDictionary** se establece como parte de la consulta de configuración, lo que le permite:

* Definir explícitamente un **languageDictionary**
* Proporcionar una URL para obtener el contenido del parámetro **languageDictionary**

Para obtener más información, puede consultar el [archivo del diccionario de idiomas de la extensión Delegated Administration](https://github.com/auth0-extensions/auth0-delegated-administration-extension/blob/master/tests/utils/en.json).
