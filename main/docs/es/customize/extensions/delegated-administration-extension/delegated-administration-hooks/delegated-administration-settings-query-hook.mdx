El **Settings Query Hook** te permite personalizar el aspecto y la experiencia de uso de la extensión Delegated Administration.

<div id="hook-contract">
  ## Contrato del Hook
</div>

* **ctx**: Objeto `context`.

  * **request.user**: Usuario que ha iniciado sesión.
  * **locale**: locale (como se infiere de la URL) -- `https://{yourTenant}.us.webtask.io/auth0-delegated-admin/en/users` establecerá **locale** en `en`.
* **callback(error, settings)**: Devolución de llamada en la que puedes devolver un error y un objeto de configuración.

## Ejemplo de uso

```javascript lines
function(ctx, callback) {
  var department = ctx.request.user.app_metadata && ctx.request.user.app_metadata.department;

  return callback(null, {
    // Solo estas conexiones deben ser visibles en el selector de conexiones. Si solo hay una conexión disponible, el selector de conexiones no se mostrará en la interfaz de usuario.
    connections: [ 'Username-Password-Authentication', 'My-Custom-DB' ],
    // El diccionario permite sobrescribir el título del Dashboard y la etiqueta "Memberships" en el diálogo Crear usuario.
    dict: {
      title: department ? department + ' User Management' : 'User Management Dashboard',
      memberships: 'Departments',
      menuName: ctx.request.user.name
    },
    // La opción CSS permite inyectar un archivo CSS personalizado según el contexto del usuario actual (p. ej.: un CSS diferente para cada cliente)
    css: (department && department !== 'IT') && 'https://cdn.jsdelivr.net/gh/auth0-extensions/auth0-delegated-administration-extension/docs/theme/fabrikam.css',
    // Esta opción permite restringir la creación de nuevos usuarios
    canCreateUser: (department === 'IT')
  });
}
```

<div id="properties">
  ### Propiedades
</div>

* **connections**: Lista de conexiones en las que este administrador puede crear y editar usuarios.
* **dict**: Diccionario que le permite sobrescribir el título del Dashboard y la etiqueta **Memberships** en el cuadro de diálogo Crear usuario.

  * **dict.title**: Título que se mostrará en la parte superior de la UI.
  * **dict.memberships**: Etiqueta que se establecerá para los campos de membresía.
  * **dict.menuName**: Nombre que se establecerá para el menú desplegable de la esquina superior derecha.
  * **dict.logoutUrl**: URL alternativa para la opción de menú de cierre de sesión.
* **userFields**: Arreglo de campos de usuario (consulte &quot;Campos personalizados&quot; a continuación).
* **css**: URL de tipo String para importar CSS.
* **altcss**: URL de tipo String para importar un segundo conjunto de CSS. Puede usarla para especificar, por ejemplo, CSS de accesibilidad para fuentes más grandes. Se mostrará al usuario una opción de menú que le permitirá activar o desactivar este conjunto de CSS.
* **languageDictionary**: URL de tipo String u objeto Dictionary (consulte &quot;Localización&quot; a continuación).
* **suppressRawData**: Establézcalo en **true** para omitir las páginas que muestran JSON sin formato
* **errorTranslator**: Función que traduce mensajes de error según la localización. Ejemplo: `(function (error, languageDictionary) { return languageDictionary.customErrors[error] || error; }).toString()`
* **canCreateUser**: Indicador de tipo Boolean. Si se establece en `false`, elimina el botón `Crear usuario` y prohíbe crear nuevos usuarios; `true` de forma predeterminada.

<div id="custom-fields">
  ## Campos personalizados
</div>

A partir de la versión 3.0 de la extensión Delegated Admin, puedes definir campos personalizados y especificar sus valores. Los campos personalizados pueden almacenarse en los campos de **metadatos del usuario** y **metadatos de la aplicación** a los que se puede acceder durante los procesos de creación o actualización de usuarios.

También puedes personalizar campos existentes definidos por Auth0, como correo electrónico, username, nombre y conexión.

Para usar campos personalizados, debes:

* Añadir tu lista de **userFields** al Settings Query Hook
* Implementar un Write Hook. Los campos personalizados requieren el uso del Write Hook para actualizar correctamente `user_metadata` y `app_metadata`. Debes actualizar el objeto de usuario que se pasa a la función de devolución de llamada con `user_metadata` y `app_metadata` del contexto (objeto `ctx`) proporcionado al Hook.

Para obtener más información sobre los Write Hooks, consulta [Administración delegada: Write Hook](/docs/es/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-write-hook).

Esquema de ejemplo para **userFields**:

```js lines expandable
userFields: [
    {
        "property": string, // requerido
        "label": string,
        "sortProperty": string,
        "display": true || function.toString(),
        "search": false || {
            "display": true || function.toString()
            "listOrder": 1,
            "listSize": string(###%), // p. ej. 15%
            "filter": boolean,
            "sort": boolean
        },
        "edit": false || {
            "display": true || function.toString()
            "type": "text || select || password || hidden",
            "component": "InputText || Input Combo || InputMultiCombo || InputSelectCombo",
            "options": Array(string) || Array ({ "value": string, "label": string }),
            "disabled": true || false,
            "validationFunction": function.toString()
        },
        "create": false || {
            "display": true || function.toString()
            "type": "text || select || password || hidden",
            "component": "InputText || Input Combo || InputMultiCombo || InputSelectCombo",
            "options": Array(string) || Array ({ "value": string, "label": string }),
            "disabled": true || false,
            "validationFunction": function.toString()
        }
    },
    ...
]
```

* **property** (**required**): Nombre de la propiedad del objeto **ctx.payload** para el Write Hook. En el Write Hook, `"property": "app_metadata.dbId"` establece `ctx.payload.app_metadata.dbId`.
* **label**: Label que se usará al añadir una etiqueta al campo en la página de información del usuario, la página de creación, la página de edición del perfil o la página de búsqueda.
* **sortProperty**: Si para la tabla de búsqueda se ordena por un campo distinto a este, usa este campo. Se permite la notación con puntos.
* **display**: true || false || stringified =&gt; Este es el valor de visualización predeterminado. Si no se redefine en search, edit o create, se usará este valor.

  * si es `true`, simplemente devolverá `user.<property>`.
  * Por defecto: si es `false`, este valor no se mostrará en ninguna página (a menos que se sobrescriba en búsqueda, edición o Create).
  * si es una función convertida en cadena, ejecuta la función para obtener el valor que se mostrará. Ejemplo: `(function display(user, value, languageDictionary) { return moment(value).fromNow(); }).toString()`
* **search**: false || objeto =&gt; Indica cómo se comportará este campo en la página de búsqueda.

  * Valor predeterminado: si es `false`, no aparecerá en la tabla de búsqueda.
  * **search.display**: Sobrescribe el valor de visualización predeterminado.
  * **search.listOrder**: Especifica el orden de las columnas en la tabla de resultados de búsqueda.
  * **search.listSize**: Especifica el ancho predeterminado de la columna.
  * **search.filter**: Especifica si se permite buscar este campo en el menú desplegable de búsqueda. El valor predeterminado es `false`.
  * **search.sort**: Especifica si esta columna se puede ordenar. Usa sortProperty si quieres ordenar por un campo distinto de property. El valor predeterminado es `false`.
* **edit**: false || objeto =&gt; Indica si el campo aparece en los cuadros de diálogo de edición. Si no es un campo predeterminado y se configura como un objeto, aparecerá en la página `Change Profile`, en el menú desplegable User Actions de la página del usuario.

  * Predeterminado: si es `false`, no aparecerá en ninguna página de edición/actualización.
  * **edit.display**: Sobrescribe el valor de visualización predeterminado.
  * **edit.required**: Establécelo en true para que falle si no tiene ningún valor. El valor predeterminado es false.
  * **edit.type** **required**: text || select || password
  * **edit.component**: InputText || Input Combo || InputMultiCombo || InputSelectCombo

    * **InputText** (predeterminado): Cuadro de texto simple.
    * **InputCombo**: Menú desplegable con búsqueda, con un solo valor.
    * **InputMultiCombo**: Menú desplegable con búsqueda que permite varios valores.
    * **InputSelectCombo**: Menú desplegable para seleccionar opciones.
  * **edit.options**: Si component es uno de InputCombo, InputMultiCombo o InputSelectCombo, se deben especificar los valores de las opciones.

    * **Array(string)**: Arreglo de valores (los campos label y value se establecerán con el mismo valor).
    * **Array(&#123; &quot;value&quot;: string, &quot;label&quot;: string &#125;)**: Te permite establecer valores distintos para value y label. Como resultado, el valor en el Write Hook tendrá ese mismo valor, aunque puede reducirse para dejar solo el value en el Write Hook.
    * La validación del lado del servidor garantizará que cualquier valor especificado para este field aparezca en el arreglo options.
  * **edit.disabled**: `true` si el componente debe ser de solo lectura; el valor predeterminado es false.
  * **edit.validateFunction**: Función serializada como String para la validación. Ten en cuenta que esta función de validación se ejecutará tanto del lado del servidor como del lado del cliente. Ejemplo: `(function validate(value, values, context, languageDictionary) { if (value...) return 'something went wrong'; return false; }).toString()`.
* **create**: false || object =&gt; Indica si el campo se muestra en el diálogo de creación.

  * Predeterminado: si es `false`, no aparecerá en la página de creación.
  * **create.placeholder**: Proporciona un texto de ejemplo para mostrar cuando el campo de entrada esté vacío.
  * **create.required**: Establécelo en true para que falle si no tiene ningún valor. El valor predeterminado es false.
  * **create.type** **required**: text || select || password
  * **create.component**: InputText || Input Combo || InputMultiCombo || InputSelectCombo

    * **InputText** (predeterminado): Cuadro de texto. Predeterminado para los tipos text y password.
    * **InputCombo**: Menú desplegable con búsqueda, con un solo valor.
    * **InputMultiCombo**: Menú desplegable con búsqueda que permite varios valores.
    * **InputSelectCombo**: Menú desplegable para seleccionar opciones.
  * **create.options**: Si el componente es InputCombo, InputMultiCombo o InputSelectCombo, se deben especificar los valores de las opciones.

    * **Arreglo(string)**: Arreglo simple de valores; label y value se establecerán con el mismo valor.
    * **Arreglo(&#123; &quot;value&quot;: string, &quot;label&quot;: string &#125;)**: Te permite establecer valores distintos para value y label. Esto hará que el valor en el Write Hook tenga ese mismo valor, aunque puede reducirse a solo el value en el Write Hook.
    * La validación del lado del servidor garantizará que cualquier valor especificado para este campo esté en el arreglo de options.
  * **create.disabled**: true si el componente debe ser de solo lectura; el valor predeterminado es false.
  * **create.validateFunction**: Función convertida a String para comprobar la validación.

    * Ejemplo: `(function validate(value, values, context, languageDictionary) { if (value...) return 'something went wrong'; return false; }).toString()`
    * Esta función de validación se ejecutará tanto del lado del servidor como del lado del cliente.

<div id="predefined-fields">
  ## Campos predefinidos
</div>

Hay un conjunto de campos predefinidos en los que se puede buscar para el comportamiento predeterminado.

Puedes modificar el comportamiento predeterminado añadiendo el campo como un userField y luego ajustando el comportamiento que quieras cambiar. Esto suele hacerse para ocultar un campo estableciendo display en false.

<div id="search-fields">
  ### Campos de búsqueda
</div>

* **name**: Campo generado a partir de otros campos: función de visualización predeterminada: `(function(user, value) { return (value || user.nickname || user.email || user.user_id); }).toString()`
* **email**: Dirección de correo electrónico o N/A
* **last&#95;login&#95;relative**: Hora del último inicio de sesión
* **logins&#95;count**: Número de inicios de sesión
* **connection**: conexión de base de datos

<div id="user-info-fields">
  ### Campos de información del usuario
</div>

* **user&#95;id**: ID de usuario
* **name**: Nombre del usuario
* **username**: Nombre de usuario
* **email**: Correo electrónico del usuario
* **identity.connection**: Valor de la conexión
* **isBlocked**: Si el usuario está bloqueado o no
* **blocked&#95;for**: Si el usuario tiene bloqueos por <Tooltip tip="Protección contra ataques: Funcionalidades que Auth0 proporciona para detectar y mitigar ataques, incluida la protección contra ataques de fuerza bruta, la limitación de IP sospechosas, la detección de contraseñas vulneradas, Bot Detection y la autenticación multifactor adaptativa." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip>
* **last&#95;ip**: Última IP que usó el usuario para iniciar sesión
* **logins&#95;count**: Número de veces que el usuario ha iniciado sesión
* **currentMemberships**: Lista de membresías de este usuario
* **created&#95;at**: Fecha y hora en que se creó el usuario
* **updated&#95;at**: Fecha y hora en que se actualizó el usuario
* **last&#95;login**: Fecha y hora en que el usuario inició sesión por última vez

<div id="create-and-edit-user-fields">
  ### Campos para crear y editar usuarios
</div>

* **connection**: conexión del usuario
* **password**: Nueva contraseña
* **repeatPassword**: Confirmación de la contraseña del usuario
* **email**: Correo electrónico del usuario
* **username**: Nombre de usuario

<div id="localization">
  ## Localización
</div>

A partir de la versión 3.0 de la extensión Delegated Admin, puedes proporcionar un diccionario de idiomas para usarlo con la localización. El diccionario de idiomas se usa solo para el contenido estático de la página; para el contenido a nivel de campo, debes usar las etiquetas de **userFields**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La localización está pensada para quienes trabajan con funciones no administrativas al gestionar usuarios. Actualmente, Auth0 no admite la localización en ninguna de las páginas de configuración.
</Callout>

Para especificar el locale, puedes usar la ruta. Por ejemplo: `https://{yourTenant}.us.webtask.io/auth0-delegated-admin/en/users` establecerá `context.locale` en `en` en la consulta de configuración.

El **languageDictionary** se configura como parte de la consulta de configuración, lo que te permite:

* Definir explícitamente un **languageDictionary**
* Proporcionar una URL para obtener el contenido del parámetro **languageDictionary**

Para obtener más información, puedes consultar el [archivo del diccionario de idiomas de la extensión Delegated Administration](https://github.com/auth0-extensions/auth0-delegated-administration-extension/blob/master/tests/utils/en.json).
