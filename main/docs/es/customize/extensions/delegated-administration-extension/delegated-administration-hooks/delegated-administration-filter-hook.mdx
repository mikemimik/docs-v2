---
description: Aprende a usar el Filter Hook con la extensión de Administración delegada.
title: 'Administración delegada: Filter Hook'
---

De forma predeterminada, los usuarios con el rol **Delegated Admin - User** ven todos los usuarios asociados a la cuenta de Auth0. Sin embargo, puedes filtrar los datos que ven los usuarios mediante el **Filter Hook**.

<div id="hook-contract">
  ## Contrato del Hook
</div>

* **ctx**: Objeto de contexto.
* **callback(error, query)**: Devolución de llamada a la que puede devolver un error o la consulta de Lucene usada para filtrar usuarios. La extensión enviará esta consulta al endpoint [**Get Users**](https://auth0.com/docs/api/management/v2#!/Users/get_users) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Para obtener más información sobre la consulta de Lucene, consulte [User Search Query Syntax](/docs/es/manage-users/user-search/user-search-query-syntax).

<div id="sample-use">
  ### Ejemplo de uso
</div>

Si Kelly administra el departamento de Finanzas, solo debería ver a los usuarios que también forman parte del departamento de Finanzas. Filtraremos los usuarios según el departamento del usuario actual (que, en este caso, son el departamento de Finanzas y Kelly, respectivamente).

```javascript lines
function(ctx, callback) {
  // Obtener el departamento de los metadatos del usuario actual.
  var department = ctx.request.user.app_metadata && ctx.request.user.app_metadata.department;
  if (!department || !department.length) {
    return callback(new Error('The current user is not part of any department.'));
  }

  // El departamento de IT puede ver todos los usuarios.
  if (department === 'IT') {
    return callback();
  }

  // Devolver la consulta de Lucene.
  return callback(null, 'app_metadata.department:"' + department + '"');
}
```

<div id="search-engine-override">
  ### Reemplazo del motor de búsqueda
</div>

Puedes reemplazar el motor de búsqueda predeterminado indicando tu elección en la respuesta.

```javascript lines
// Retorna la consulta de lucene.
return callback(null, { query: 'app_metadata.department:"' + department + '"', searchEngine: 'v2' });
```

<div id="notes">
  ## Notas
</div>

No uses comillas simples, comillas dobles ni ningún otro carácter especial (como **+** o **-**) en los términos por los que quieras filtrar. Esto puede causar problemas con la consulta de Lucene.

Si usas varios operadores como OR, NOT o AND, encierra esos parámetros de búsqueda entre paréntesis [`()`] para delimitar qué operador se aplica en cada caso.

Si no configuras este Hook, la búsqueda devuelve **todos los usuarios**.
