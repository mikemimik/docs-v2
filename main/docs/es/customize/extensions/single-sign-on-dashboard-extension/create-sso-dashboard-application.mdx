Usa la extensión <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (Single Sign-On Dashboard) para gestionar el inicio de sesión único de tus usuarios en varias aplicaciones empresariales. Para obtener más información, [consulta esta extensión en GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).

<div id="create-auth0-application">
  ## Crear una aplicación de Auth0
</div>

Antes de [agregar la extensión SSO Dashboard](/docs/es/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension), debes crear una aplicación en Auth0:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y selecciona **+Create Application**.
2. Introduce un nombre descriptivo para tu aplicación (por ejemplo, SSO Dashboard), selecciona **Single-Page Web Application** en **Application Type** y luego selecciona **Create**.
3. En la pestaña **Settings**, configura las **Allowed Callback URLs** según la región de tu tenant:

   1. Para **Admins**: `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/admins/login`
   2. Para **Users**: `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/login`

      | Región | Dominio de la región |
      | ------ | -------------------- |
      | US-1   | us.webtask.io        |
      | US-3   | us.webtask.run       |
      | US-4   | us.webtask.run       |
      | US-5   | us.webtask.run       |
      | EU     | eu.webtask.io        |
      | EU-2   | eu.webtask.run       |
      | AU     | au.webtask.io        |
      | JP-1   | jp.webtask.run       |
      | UK-1   | uk.webtask.run       |
      | CA-1   | ca.webtask.run       |
4. Selecciona y copia el valor de **Client ID**.
5. Desplázate hasta la parte inferior de la página y selecciona **Advanced Settings**.
6. Selecciona la pestaña **OAuth** y luego pega el valor de **Client ID** en el campo **Allowed APPs / APIs**.
7. Establece **JsonWebToken Signature Algorithm** en **RS256**.
8. Selecciona **Save Changes**.

De forma predeterminada, todos los tipos de conexión están habilitados para que los usuarios puedan iniciar sesión en el SSO Dashboard. Si quieres cambiar esto, ve a la pestaña **Connections** de la aplicación.
