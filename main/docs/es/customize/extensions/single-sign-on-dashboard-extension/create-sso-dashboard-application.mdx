title: Crear una aplicación de SSO Dashboard para inicio de sesión único (SSO)
---

Use la extensión SSO Dashboard de <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO) para administrar el inicio de sesión único de sus usuarios en múltiples aplicaciones empresariales. Para obtener más información, [consulte esta extensión en GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).

<div id="create-auth0-application">
  ## Crear aplicación de Auth0
</div>

Antes de [agregar la extensión SSO Dashboard](/docs/es/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension), debes crear una aplicación en Auth0:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y selecciona **+Create Application**.
2. Ingresa un nombre descriptivo para tu aplicación (por ejemplo, SSO Dashboard), selecciona **Single-Page Web Application** en **Application Type** y luego selecciona **Create**.
3. En la pestaña **Settings**, configura las **Allowed Callback URLs** según la región de tu tenant:

   1. Para **Admins**: `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/admins/login`
   2. Para **Users**: `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/login`

      | Region | Region Domain  |
      | ------ | -------------- |
      | US-1   | us.webtask.io  |
      | US-3   | us.webtask.run |
      | US-4   | us.webtask.run |
      | US-5   | us.webtask.run |
      | EU     | eu.webtask.io  |
      | EU-2   | eu.webtask.run |
      | AU     | au.webtask.io  |
      | JP-1   | jp.webtask.run |
      | UK-1   | uk.webtask.run |
      | CA-1   | ca.webtask.run |
4. Selecciona y copia el valor de **Client ID**.
5. Ve al final de la página y selecciona **Advanced Settings**.
6. Selecciona la pestaña **OAuth** y pega el valor de **Client ID** en el campo **Allowed APPs / APIs**.
7. Configura **JsonWebToken Signature Algorithm** como **RS256**.
8. Selecciona **Save Changes**.

De forma predeterminada, todos los tipos de conexión están habilitados para que los usuarios puedan iniciar sesión en el SSO Dashboard. Si quieres cambiar esto, ve a la pestaña **Connections** de la aplicación.
