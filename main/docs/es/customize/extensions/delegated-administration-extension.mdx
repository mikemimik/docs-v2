<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recomienda que uses la configuración de Tenant Members para definir las capacidades de uso del tenant para otros usuarios y administradores de tu organización. Para obtener más información, consulta [Dashboard Access](/docs/es/get-started/manage-dashboard-access).
</Callout>

La Delegated Administration Extension (DAE) te permite otorgar permisos administrativos a un grupo selecto de personas sin darles acceso a ninguna otra área.

<div id="configure-the-dae">
  ## Configurar el DAE
</div>

Para configurar el DAE, debes:

1. [Registrar la aplicación en Auth0](#register-the-application-with-auth0)
2. [Crear una conexión de base de datos](#create-a-database-connection)
3. [Deshabilitar todas las demás conexiones de la aplicación de Auth0](#disable-all-other-connections-for-the-auth0-application)
4. [Crear usuarios para la conexión de base de datos](#create-user-for-the-database-connection)
5. [Asignar roles a los usuarios](#assign-roles-to-users)
6. [Instalar y configurar la extensión](#install-and-configure-the-extension)
7. [Usar la extensión](#use-the-extension)

<div id="register-the-application-with-auth0">
  ### Registrar la aplicación en Auth0
</div>

Cree la aplicación que la Delegated Administration Extension pondrá a disposición de quienes deban tener privilegios administrativos para la página Users. Para ello, [cree una aplicación de administración delegada](/docs/es/customize/extensions/delegated-administration-extension/create-delegated-admin-applications) en Auth0.

Cuando termine, tome nota del **<Tooltip tip="ID de cliente: valor de identificación que Auth0 asigna a su recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** de la aplicación.

<div id="create-a-database-connection">
  ### Crear una conexión de base de datos
</div>

En este ejemplo, una conexión de base de datos servirá como origen de los usuarios que tendrán acceso al área de Users. Para configurarlo, [cree una conexión de base de datos](/docs/es/get-started/applications/set-up-database-connections).

Mientras configura su conexión:

* Use un nombre de conexión adecuado, como `HelpDesk`.
* Active el interruptor **Disable Sign Ups**. Por motivos de seguridad, esto garantiza que incluso los usuarios que tengan el enlace de la conexión de base de datos no puedan registrarse por sí mismos.

<div id="disable-all-other-connections-for-the-auth0-application">
  ### Deshabilita todas las demás conexiones de la aplicación de Auth0
</div>

De forma predeterminada, Auth0 habilita todas las conexiones asociadas a tu tenant cuando creas una nueva aplicación. Para este ejemplo, deshabilita todas las conexiones salvo la conexión de base de datos que acabas de crear. Esto ayuda a mantener la seguridad de la aplicación, ya que nadie podrá agregarse mediante una de tus conexiones existentes.

Para configurar esto, [actualiza las conexiones de la aplicación](/docs/es/get-started/applications/update-application-connections).

<div id="create-users-for-the-database-connection">
  ### Crear usuarios para la conexión de base de datos
</div>

Para continuar, debes [crear al menos un usuario](/docs/es/manage-users/user-accounts/create-users) y asociarlo a tu conexión.

<div id="assign-roles-to-users">
  ### Asignar roles a los usuarios
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Aunque la [Delegated Administration Extension (DAE)](/docs/es/customize/extensions/delegated-administration-extension) y el conjunto de funcionalidades de [Authorization Core](/docs/es/manage-users/access-control/rbac) son funcionalidades completamente independientes, puedes usar el conjunto de funcionalidades de Authorization Core para crear y administrar roles para la DAE mediante Actions. Para saber cómo hacerlo, consulta [Sample Use Cases: Actions with Authorization](/docs/es/manage-users/access-control/sample-use-cases-actions-with-authorization).
</Callout>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cuando inicies sesión como usuario con Organizations y la Delegated Administration Extension (DAE), tus roles de usuario no estarán disponibles. Solo estarán disponibles los roles de miembro de tu organización dentro de `event.authorization.roles.` Para agregar roles a los miembros de una organización, consulta [Add Roles to Organization Members](/docs/es/manage-users/organizations/configure-organizations/add-member-roles).
</Callout>

Auth0 concede acceso a la Delegated Administration Extension (DAE) a los usuarios asociados a tu conexión en función de sus roles. Estos son los roles específicos de la DAE:

| Este rol...                       | Concede permiso para...                                                                                                                       |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `Delegated Admin - User`          | Buscar usuarios, crear usuarios, abrir usuarios y ejecutar acciones sobre ellos (como eliminar o bloquear).                                   |
| `Delegated Admin - Administrator` | Hacer todo lo que puede hacer Delegated Admin - User, además de ver todos los registros del tenant y configurar Hooks.                        |
| `Delegated Admin - Auditor`       | Buscar usuarios y ver la información del usuario, pero sin realizar cambios. Los botones basados en acciones no están visibles para este rol. |
| `Delegated Admin - Operator`      | Acceder a la administración de usuarios y a los registros, pero no a la sección de configuración de la extensión.                             |

Al trabajar con roles, te recomendamos que uses el conjunto de funcionalidades de [Authorization Core](/docs/es/manage-users/access-control/rbac):

1. [Crear roles de DAE](/docs/es/manage-users/access-control/configure-core-rbac/roles/create-roles). Los nombres de los roles que crees deben coincidir con los nombres de los roles de DAE predefinidos anteriores.
2. [Asignar roles de DAE a un usuario manualmente](/docs/es/manage-users/access-control/configure-core-rbac/rbac-users/assign-roles-to-users).
3. Agrega los roles del usuario al espacio de nombres de la DAE en el token de ID usando [Actions](/docs/es/customize/actions)**:**

   ```text lines
   exports.onExecutePostLogin = async (event, api) => {
     const namespace = `https://{yourTenant}/auth0-delegated-admin`;
     if (event.client.client_id === 'CLIENT_ID' && event.authorization) {
       api.idToken.setCustomClaim(namespace, { "roles": event.authorization.roles });
     }
   };
   ```

Recuerda reemplazar el marcador de posición `CLIENT_ID` por el **ID de cliente** de tu aplicación de administración delegada y `{yourTenant}` por el nombre de tu tenant. Por ejemplo, si el nombre de tu tenant es &quot;tenant&#95;name&#95;example&quot;, el espacio de nombres sería: `https://tenant_name_example/auth0-delegated-admin`

Para obtener más información sobre cómo crear Actions, consulta [Write Your First Action](/docs/es/customize/actions/write-your-first-action)

<Warning>
  Auth0 devuelve la información del perfil en un formato de claim estructurado, tal como lo define la [especificación OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Esto significa que los claims personalizados agregados a los tokens de ID o a los access tokens deben [ajustarse a las directrices y restricciones](/docs/es/secure/tokens/json-web-tokens/create-custom-claims) para evitar posibles colisiones.
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El uso de Authorization Core definirá roles en el objeto `context.authorization`.

  Si decides no usar Authorization Core, debes definir los roles de la DAE en uno de los siguientes campos del perfil del usuario:

  * `user.app_metadata.roles`
  * `user.app_metadata.authorization.roles`
</Callout>

<div id="install-and-configure-the-extension">
  ## Instalar y configurar la extensión
</div>

Ahora que hemos creado y configurado una aplicación, una conexión y nuestro usuario, podemos [instalar y configurar la extensión Delegated Admin](/docs/es/customize/extensions/delegated-administration-extension/install-delegated-admin-extension).

<div id="use-the-extension">
  ## Usa la extensión
</div>

Una vez instalada, ya puedes usar la extensión Delegated Admin.

Ve a la extensión con el enlace de inicio de sesión correspondiente a tu región y al runtime de extensibilidad de tu tenant.

| Ubicación   | Nombre | Enlace de inicio de sesión                                  |
| ----------- | ------ | ----------------------------------------------------------- |
| Australia   | AU     | `https://{yourTenant}.au.webtask.io/auth0-delegated-admin`  |
| Europa      | EU     | `https://{yourTenant}.eu.webtask.io/auth0-delegated-admin`  |
| Europa      | EU-2   | `https://{yourTenant}.eu.webtask.run/auth0-delegated-admin` |
| Japón       | JP-1   | `https://{yourTenant}.jp.webtask.run/auth0-delegated-admin` |
| Reino Unido | UK     | `https://{yourTenant}.uk.webtask.run/auth0-delegated-admin` |
| EE. UU.     | US-1   | `https://{yourTenant}.us.webtask.io/auth0-delegated-admin`  |
| EE. UU.     | US-3   | `https://{yourTenant}.us.webtask.run/auth0-delegated-admin` |

Se abrirá una nueva pestaña en la que se mostrará el prompt de inicio de sesión.

Como en este ejemplo desactivamos el registro para la conexión de base de datos mientras la configurábamos, la pantalla de inicio de sesión no muestra la opción **Sign Up**.
Una vez que proporciones credenciales válidas, Auth0 te dirigirá a tu página personalizada del **Dashboard de administración delegada**, que mostrará el **Title** que proporcionaste en la parte superior de la página y, si proporcionaste un archivo CSS personalizado, tu diseño.

<Card title="Tiempo de espera de la sesión de Administración delegada">
  De forma predeterminada, el tiempo de expiración del token es de 10 horas. Sin embargo, por motivos de seguridad, al usar Administración delegada, Auth0 no guarda ningún token en cookies ni en `sessionStorage`. Debes iniciar una nueva sesión cada vez que recargues la página.
</Card>
