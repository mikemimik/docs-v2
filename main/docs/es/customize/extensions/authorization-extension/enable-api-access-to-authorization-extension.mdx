<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 ofrece dos maneras de implementar el control de acceso basado en roles (RBAC), que puede utilizar en lugar del sistema interno de control de acceso de su API o en combinación con él:

  * [Authorization Core](/docs/es/manage-users/access-control/rbac)
  * [Authorization Extension](/docs/es/customize/extensions/authorization-extension)

  El conjunto de funcionalidades de Authorization Core ofrece la misma funcionalidad que Authorization Extension, mejora el rendimiento y la escalabilidad, y proporciona un sistema RBAC más flexible que Authorization Extension.

  Actualmente, ambas implementan las funcionalidades clave de RBAC y le permiten restringir los `scopes` personalizados definidos para una API a aquellos que se hayan asignado al usuario como permisos.
</Callout>

Una vez configurada y lista, su extensión debería contener usuarios, así como grupos, roles y permisos. Puede automatizar el aprovisionamiento y consultar en tiempo real el contexto de autorización de sus usuarios si habilita el acceso mediante API a su extensión.

<div id="enable-api-access">
  ## Habilitar el acceso a la API
</div>

1. Ve a [Auth0 Dashboard &gt; Extensions &gt; Auth0 Authorization](https://manage.auth0.com/#/extensions).
2. Para ir a la sección **API**, selecciona el nombre de tu tenant de Auth0 en la esquina superior derecha del **Authorization Dashboard** y, a continuación, selecciona **API**.

   <Frame>![Dashboard - Extensions - Authorization Extensions Dashboard - Seleccionar API](/docs/images/cdy7uua7fh8z/6cF78dwvbK7Sc0JbuxJF85/fbd895fb6f7338559628ea71b3003f52/click-configuration_select-api.png)</Frame>
3. En la página **Settings**, habilita el interruptor **API Access**.

   <Frame>![Dashboard - Extensions - Authorization Extensions Dashboard - Habilitar API Access](/docs/images/cdy7uua7fh8z/5toxhsSik9EWMnQoK5wTjH/fc40d80a32461e9216f5c69a89e031ec/enable-api-access.png)</Frame>
4. Una vez habilitado **API Access**, podrás ver o controlar (dentro de la extensión) algunos de los parámetros de los tokens emitidos por la API. Puedes controlar el **tiempo hasta el vencimiento** del token, así como ver el **audience**, el **issuer** y la **URL** para acceder a la API.

   <Frame>![Dashboard - Extensions - Authorization Extensions Dashboard - API Access habilitado](/docs/images/cdy7uua7fh8z/3WfHXaEyi8FvD8pTDlYQpt/cbe92c5b5f4baf9c35a8c686ff4f511d/api-access-enabled.png)</Frame>

<div id="access-extension-api">
  ## Acceder a la API de la extensión
</div>

Al habilitar el acceso a la API de la extensión, Auth0 creó automáticamente una API para que la uses en el [Auth0 Dashboard](https://manage.auth0.com/#/apis). Para acceder a la API, tendrás que crear una aplicación de máquina a máquina, que es la entidad que interactúa directamente con la API.

<div id="create-applications">
  ### Crear aplicaciones
</div>

1. Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione **Create aplicación**. Asigne un nombre a su nueva aplicación y elija el tipo **aplicación de máquina a máquina**. Seleccione **Create** para continuar.
   Se le redirigirá a la página **Quick Start** de la aplicación, donde podrá personalizar la documentación interactiva según la API con la que usará la aplicación. Seleccione la API que Auth0 creó para su extensión (debería llamarse **auth0-authorization-extension-api** o algo similar).
   Como es la primera vez que trabaja con la API y la aplicación juntas, verá un mensaje que dice: &quot;This application is not authorized for this API.&quot; Para autorizar la aplicación para usarla con la API, seleccione **Navigate to the API and Authorize**.

   <Frame>![Dashboard - Authorization Extensions - Client Quick Start](/docs/images/cdy7uua7fh8z/UbMM1C6a1GjMIhNlk8cCD/462b9dcd29e5d7f5bbd261490e6a46b0/Dashboard_-_Authorization_Extensions_-_Client_Quick_Start.png)</Frame>
2. Verá una lista de aplicaciones de máquina a máquina que puede usar con su API. Active el switch junto a la aplicación que acaba de crear para autorizarla.

   <Frame>![Dashboard - Authorization Extension - Clients for API](/docs/images/cdy7uua7fh8z/4WFJ3BhCf2kZjbRWhQ0sCp/38aeb0ca9360fd75bd7d54f897ac1735/Dashboard_-_Authorization_Extension_-_Clients_for_API.png)</Frame>

   Una vez que haya autorizado la aplicación, verá el **Grant ID**. También puede seleccionar los **Scopes** que se concederán a la aplicación. Los scopes que conceda dependen de los endpoints a los que quiera acceder.
3. Si realiza algún cambio en los scopes, seleccione **Update** para guardar.

   <Frame>![Dashboard - Authorization Extension Dashboard - Client Scopes](/docs/images/cdy7uua7fh8z/2RvMFmLusnUOmBp1uXPh1/a78add868b094d37cec63365f745f196/extensions-auth-api-scopes.png)</Frame>

<div id="get-access-token">
  ### Obtener un token de acceso
</div>

Para acceder a la API, deberás [solicitar y obtener el token adecuado](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).

<div id="call-the-api">
  ### Llama a la API
</div>

Puedes llamar a la API mediante:

* Una solicitud HTML
* Un comando cURL

También puedes encontrar información detallada sobre los endpoints, así como ejemplos de cómo llamar a cada endpoint con los tres métodos anteriores, en [Authorization Extension API Explorer](https://auth0.com/docs/api/authorization-extension).

También puedes consultar el **API Explorer**, que documenta todo lo que puedes hacer desde la línea de comandos una vez que hayas habilitado el acceso a tu extensión mediante la API.

Selecciona la página **Explorer** para ver la documentación de la API.

<Frame>![Authorization Extension API Explorer](/docs/images/cdy7uua7fh8z/5mwKE5muQHYbXKqYZvWOwR/aa0ecec3e5cb803a105d857f24c8b6b7/api-explorer.png)</Frame>
