<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Le recomendamos encarecidamente que use Actions para extender Auth0. Con Actions, tiene acceso a información de tipos detallada, documentación integrada y paquetes `npm` públicos, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lea [Comprender cómo funcionan Auth0 Actions](/docs/es/customize/actions/actions-overview).

  Para ayudarle con la migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y a [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarle en su proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada de blog: [Prepararse para el fin de vida útil de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 proporciona dos formas de implementar control de acceso basado en roles (RBAC), que puede usar en lugar del sistema interno de control de acceso de su API o en combinación con este:

  * [Authorization Core](/docs/es/manage-users/access-control/rbac)
  * [Authorization Extension](/docs/es/customize/extensions/authorization-extension)

  El conjunto de funcionalidades de Authorization Core equivale a la funcionalidad de Authorization Extension, mejora el rendimiento y la escalabilidad, y proporciona un sistema RBAC más flexible que Authorization Extension.

  Actualmente, ambos implementan las funciones clave de RBAC y le permiten restringir los scopes personalizados definidos para una API a los que se hayan asignado al usuario como permisos.
</Callout>

Uno de los principales cambios entre las versiones 1 y 2 de Authorization Extension es la eliminación de la sección **Applications**. Esta sección se eliminó para simplificar su complejidad inherente, por ejemplo, cuando se utilizaba para definir una política de control de acceso. El enfoque recomendado para esos casos de uso es usar [reglas](/docs/es/customize/rules).

<div id="upgrade-extension-version">
  ## Actualizar la versión de la extensión
</div>

Para actualizar la Authorization Extension, ve a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y selecciona la vista **Installed Extensions**.

En la fila de la Authorization Extension, verás un enlace para iniciar el proceso de actualización a la versión más reciente.
