<Warning>
  La fecha de End of Life (EOL) de Rules y Hooks será el **18 de noviembre de 2026** y ya no están disponibles para tenants nuevos creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de vida útil.

  Recomendamos encarecidamente usar Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos enriquecida, documentación integrada y paquetes `npm` públicos, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte durante el proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 ofrece dos formas de implementar control de acceso basado en roles (RBAC), que puedes usar en lugar del sistema interno de control de acceso de tu API o en combinación con este:

  * [Authorization Core](/docs/es/manage-users/access-control/rbac)
  * [Authorization Extension](/docs/es/customize/extensions/authorization-extension)

  El conjunto de funcionalidades de Authorization Core ofrece la misma funcionalidad que Authorization Extension, mejora el rendimiento y la escalabilidad, y proporciona un sistema RBAC más flexible que Authorization Extension.

  Actualmente, ambos implementan las funcionalidades clave de RBAC y te permiten restringir los scopes personalizados definidos para una API a aquellos que se le han asignado al usuario como permisos.
</Callout>

Uno de los principales cambios entre las versiones 1 y 2 de Authorization Extension es la eliminación de la sección **Applications**. Esta sección se eliminó para reducir su complejidad inherente, por ejemplo, cuando se usaba para definir una policy de control de acceso. El enfoque recomendado para esos casos de uso es usar [reglas](/docs/es/customize/rules).

<div id="upgrade-extension-version">
  ## Actualizar la versión de la extensión
</div>

Para actualizar la Authorization Extension, ve a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y selecciona la vista **Installed Extensions**.

En la fila de la Authorization Extension, verás un enlace para iniciar el proceso de actualización a la versión más reciente.
