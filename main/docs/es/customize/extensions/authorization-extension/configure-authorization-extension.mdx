<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 ofrece dos formas de implementar el control de acceso basado en roles (RBAC), que puedes usar en lugar del sistema interno de control de acceso de tu API o en combinación con él:

  * [Authorization Core](/docs/es/manage-users/access-control/rbac)
  * [Authorization Extension](/docs/es/customize/extensions/authorization-extension)

  El conjunto de funcionalidades de Authorization Core equivale a la funcionalidad de Authorization Extension, mejora el rendimiento y la escalabilidad, y ofrece un sistema de RBAC más flexible que Authorization Extension.

  Actualmente, ambos implementan las funciones clave de RBAC y te permiten restringir los scopes personalizados definidos para una API a aquellos que se hayan asignado al usuario como permisos.
</Callout>

Antes de que la extensión pueda aplicar tu lógica de autorización, debes configurar cómo se comportará durante la transacción de inicio de sesión. Los ajustes de configuración se recogerán en una [rule](/docs/es/customize/rules) que se ejecuta durante el runtime.

<div id="configure-extension">
  ## Configurar la extensión
</div>

1. Ve a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y selecciona **Auth0 Authorization**.
2. Elige **Configuración** en la esquina superior derecha del **Authorization Dashboard**.

   <Frame>![Dashboard - Authorization Extension Dashboard - Configuración](/docs/images/cdy7uua7fh8z/r8Ic6OYMcVfXDvIKePLrb/ed9c75536d20c12149e8b8e62a8edc44/click-configuration.png)</Frame>
3. Esto te lleva a la sección **Configuración de la Rule** de la página **Configuración**. Todos los cambios que hagas en las secciones bajo **Contenido del token**, como los relacionados con grupos, roles y permisos, se reflejarán en la Rule que exportes al completar este paso.

   <Frame>![Configuración de Authorization Extension](/docs/images/cdy7uua7fh8z/4ydHJBdKGs4XVLnRst66A3/1f61ad2e087252c6c3aa10add13d99cd/configuration0auth.png)</Frame>

<div id="apikey">
  ### ApiKey
</div>

La rule usa la ApiKey para comunicarse con la API de Authorization Extension y obtener la policy. La ApiKey se almacena como una configuración de la rule y se creará automáticamente cuando la rule se publique. Para rotar la ApiKey, haz clic en el botón **Rotate**; cuando se rote la ApiKey, la configuración de la rule se actualizará automáticamente.

<div id="add-authorization-information-to-the-token-issued">
  ## Agregue información de autorización al token emitido
</div>

Puede almacenar datos de autorización, como grupos, roles o permisos, en el token emitido por Auth0. Su aplicación puede luego usar esta información inspeccionando el token y realizar las acciones correspondientes según el contexto de autorización actual del usuario.

Para agregar información de grupos, roles y/o permisos al token emitido, active el control deslizante junto a la opción que quiera incluir.

<Warning>
  Almacenar demasiados datos en el token puede causar problemas de rendimiento o incluso impedir su emisión. Asegúrese de almacenar solo lo necesario. Si necesita tener disponible una gran cantidad de datos de usuario, considere usar persistencia en lugar de agregarlos al token.
</Warning>

<div id="merge-authorization-data-from-idp">
  ### Fusionar datos de autorización desde el IdP
</div>

Es posible que tenga usuarios que reciban grupos, roles o permisos del <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> (IdP) que utiliza, como Active Directory. Si desea fusionar estos elementos (para conservarlos) con los definidos en la Authorization Extension, asegúrese de habilitar las opciones de **Passthrough** correspondientes. Active el interruptor junto a cada combinación que quiera habilitar.

<div id="store-authorization-information-in-user-profiles">
  ### Almacenar información de autorización en los perfiles de usuario
</div>

Si tu contexto de autorización es amplio (por ejemplo, el usuario podría pertenecer a muchos grupos o tener muchos permisos asignados), puede resultarte útil almacenar parte del contenido de autorización en los perfiles de los usuarios. Esto te permite guardar menos información en el token, por lo que es menos probable que surjan problemas de rendimiento o incluso problemas con la emisión de tokens. **Persistencia** es el proceso mediante el cual almacenas información sobre grupos, roles y permisos en los perfiles de los usuarios.

Los datos se almacenarán en el campo `app_metadata` del usuario, y luego podrás usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> o el [Dashboard](https://manage.auth0.com/#/users) para recuperar esta información después de que el usuario haya iniciado sesión.

<Frame>![Authorization Extension - Configuration - Persistence](/docs/images/cdy7uua7fh8z/5kfgcLhwwZkQ2Oh5uxaGxF/de5cf05429d1d1730a0165f4c61ac629/persistence.png)</Frame>

<div id="save-changes-to-rule">
  ## Guardar cambios en la rule
</div>

Una vez que hayas configurado tu rule, haz clic en **Publish Rule**. Esto crea una rule para tu tenant que se ejecuta después de cada inicio de sesión.

<div id="view-rule">
  ### Ver la rule
</div>

Si quieres ver la rule que has creado, puedes hacerlo en [Auth0 Dashboard &gt; pipeline de autenticación &gt; Rules](https://manage.auth0.com/#/rules).

Puedes abrir la rule para ver su configuración exacta.
