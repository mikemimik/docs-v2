<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [Authorization Core](/docs/es/manage-users/access-control/rbac) y [Authorization Extension](/docs/es/customize/extensions/authorization-extension) son funcionalidades totalmente independientes. Para gestionar grupos, roles o permisos, debes usar la funcionalidad en la que se crearon originalmente.
</Callout>

Authorization Extension permite gestionar la autorización de usuarios mediante grupos, roles y permisos. Puedes definir el comportamiento esperado durante el proceso de inicio de sesión, y los ajustes de configuración se guardarán en una regla que se ejecuta durante el runtime. Para obtener más información, consulta [Auth0 Reglas](/docs/es/customize/rules).

Con Authorization Extension, puedes almacenar datos de autorización, como grupos, roles o permisos, en el token emitido por Auth0. Tu aplicación puede usar esta información inspeccionando el token y realizar las acciones adecuadas en función del contexto de autorización actual del usuario.

Con Authorization Extension, los roles y permisos se establecen por aplicación. Si necesitas los mismos roles o permisos en otra aplicación, tendrás que crearlos por separado. En cambio, [Authorization Core](/docs/es/manage-users/access-control/rbac) ofrece mucha más flexibilidad con roles y permisos.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de poder usar la extensión, tendrás que:

1. [Instalar](/docs/es/customize/extensions/authorization-extension/install-authorization-extension) la extensión.
2. [Configurar](/docs/es/customize/extensions/authorization-extension/configure-authorization-extension) cómo se comportará la extensión durante el proceso de inicio de sesión.
3. [Configurar](/docs/es/customize/extensions/authorization-extension/set-up-authorization-extension-users) usuarios, grupos, roles y permisos.

<div id="manage-data">
  ## Gestionar datos
</div>

Puedes mover datos fácilmente a la extensión o extraerlos de ella mediante un archivo JSON. Para obtener más información, lee [Importar y exportar datos de Authorization Extension](/docs/es/customize/extensions/authorization-extension/import-and-export-authorization-extension-data).

<div id="add-functionality">
  ## Agregar funcionalidad
</div>

Una vez que la extensión esté en funcionamiento, puedes agregarle funcionalidades adicionales.

* [Habilitar el acceso a la API de la extensión](/docs/es/customize/extensions/authorization-extension/enable-api-access-to-authorization-extension): Aprende cómo automatizar el aprovisionamiento y consultar en tiempo real el contexto de autorización de tus usuarios mediante la API de la extensión.
* [Explorar los endpoints de la API de Authorization Extension](https://auth0.com/docs/api/authorization-extension): Aprende sobre los endpoints de la API de Authorization Extension y cómo usarlos.
* [Usar datos de Authorization Extension en reglas](/docs/es/customize/extensions/authorization-extension/use-rules-with-the-authorization-extension): Aprende cómo usar reglas para configurar lógica adicional en tus inicios de sesión.

<div id="upgrade-from-previous-versions">
  ## Actualización desde versiones anteriores
</div>

<div id="upgrade-from-version-26-or-later">
  ### Actualización desde la versión 2.6 o posteriores
</div>

Las actualizaciones desde la versión 2.6 o posteriores no presentan cambios incompatibles y no requieren ninguna acción especial.

<div id="upgrade-from-versions-before-26">
  ### Actualización desde versiones anteriores a la 2.6
</div>

Authorization Extension 2.6 incluye cambios incompatibles derivados de modificaciones en la lógica de almacenamiento y gestión de la API Key; por ello, al actualizar debe realizar pasos adicionales, como se detalla a continuación. Si no completa estos pasos, se producirá un error `InvalidApiKey` o `You are not allowed to access this application` al ejecutarse la rule. Para obtener más información, consulte el [registro de cambios de GitHub](https://github.com/auth0/auth0-authorization-extension/blob/master/CHANGELOG.md).

1. Vaya a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y seleccione la vista **Installed Extensions**.
2. Busque **Auth0 Authorization**, seleccione **Upgrade** y confirme. Espere a que se complete la actualización.

<div id="rotate-the-extensions-api-key">
  #### Rotar la API Key de la extensión
</div>

1. Selecciona **Auth0 Authorization** para abrir la extensión.
2. En el menú desplegable de la esquina superior derecha del Dashboard de la extensión, selecciona **Configuración**.
3. Ubica la sección **API Key** y selecciona **Rotar**.

<div id="republish-the-extensions-rule">
  #### Republicar la rule de la extensión
</div>

Selecciona **Publicar Rule**.

<div id="delete-the-old-extension-rule-if-it-exists">
  #### Elimina la rule antigua de la extensión, si existe
</div>

1. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Rules](https://manage.auth0.com/#/rules).
2. Ubica la rule `auth0-authz`. Si no existe, ya terminaste; de lo contrario, continúa con los siguientes pasos.
3. Ubica la rule `auth0-authorization-extension` y arrástrala hasta la posición debajo de la rule `auth0-authz`.
4. Comprueba que la rule `auth0-authz`:

   * Fue creada por la Authorization Extension y no se ha modificado manualmente
   * No cambiará el flujo de autorización de forma que conceda acceso o privilegios a usuarios no deseados si se elimina
5. Si se cumplen las condiciones anteriores, usa el interruptor para deshabilitar la rule `auth0-authz`. Después de verificar que todo funciona correctamente, puedes decidir si dejar la rule deshabilitada o eliminarla por completo.
