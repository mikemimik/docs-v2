<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las funcionalidades de [Authorization Core](/docs/es/manage-users/access-control/rbac) y [Authorization Extension](/docs/es/customize/extensions/authorization-extension) son completamente independientes. Para administrar grupos, roles o permisos, debes usar la funcionalidad en la que se crearon originalmente.
</Callout>

Authorization Extension ofrece soporte para la autorización de usuarios mediante grupos, roles y permisos. Puedes definir el comportamiento esperado durante el proceso de inicio de sesión, y tu configuración se reflejará en una rule que se ejecuta durante el runtime. Para obtener más información, consulta [Auth0 Rules](/docs/es/customize/rules).

Con Authorization Extension, puedes almacenar datos de autorización, como grupos, roles o permisos, en el token emitido por Auth0. Tu aplicación puede usar esta información inspeccionando el token y realizar las acciones adecuadas según el contexto de autorización actual del usuario.

Con Authorization Extension, los roles y permisos se establecen por aplicación. Si necesitas los mismos roles o permisos en otra aplicación, tendrás que crearlos por separado. En cambio, [Authorization Core](/docs/es/manage-users/access-control/rbac) ofrece mucha más flexibilidad con roles y permisos.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de poder usar la extensión, tendrás que:

1. [Instalar](/docs/es/customize/extensions/authorization-extension/install-authorization-extension) la extensión.
2. [Configurar](/docs/es/customize/extensions/authorization-extension/configure-authorization-extension) cómo se comportará la extensión durante el proceso de inicio de sesión.
3. [Configurar](/docs/es/customize/extensions/authorization-extension/set-up-authorization-extension-users) usuarios, grupos, roles y permisos.

<div id="manage-data">
  ## Administrar datos
</div>

Puedes mover datos fácilmente a la extensión o desde ella mediante un archivo JSON. Para obtener más información, consulta [Importar y exportar datos de Authorization Extension](/docs/es/customize/extensions/authorization-extension/import-and-export-authorization-extension-data).

<div id="add-functionality">
  ## Añadir funcionalidad
</div>

Una vez que la extensión esté en funcionamiento, puedes añadirle más funcionalidad.

* [Habilitar el acceso a la API de la extensión](/docs/es/customize/extensions/authorization-extension/enable-api-access-to-authorization-extension): Aprende a automatizar el aprovisionamiento y consultar en tiempo real el contexto de autorización de tus usuarios mediante la API de la extensión.
* [Explorar los endpoints de la API de Authorization Extension](https://auth0.com/docs/api/authorization-extension): Obtén más información sobre los endpoints de la API de Authorization Extension y cómo utilizarlos.
* [Usar datos de Authorization Extension en rules](/docs/es/customize/extensions/authorization-extension/use-rules-with-the-authorization-extension): Aprende a usar rules para configurar lógica adicional en tus inicios de sesión.

<div id="upgrade-from-previous-versions">
  ## Actualizar desde versiones anteriores
</div>

<div id="upgrade-from-version-26-or-later">
  ### Actualizar desde la versión 2.6 o posterior
</div>

Las actualizaciones desde la versión 2.6 o posterior no presentan cambios incompatibles ni requieren ninguna acción especial.

<div id="upgrade-from-versions-before-26">
  ### Actualización desde versiones anteriores a la 2.6
</div>

Authorization Extension 2.6 contiene cambios incompatibles debido a una modificación en la lógica para almacenar y gestionar la API Key; por ello, al actualizar, debe realizar los pasos adicionales que se detallan a continuación. Si no completa estos pasos, se producirá un error `InvalidApiKey` o `You are not allowed to access this application` al ejecutar la rule. Para obtener más información, consulte el [registro de cambios de GitHub](https://github.com/auth0/auth0-authorization-extension/blob/master/CHANGELOG.md).

1. Vaya a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y seleccione la vista **Installed Extensions**.
2. Busque **Auth0 Authorization**, seleccione **Upgrade** y confirme. Espere a que se complete la actualización.

<div id="rotate-the-extensions-api-key">
  #### Rota la API key de la extensión
</div>

1. Selecciona **Auth0 Authorization** para abrir la extensión.
2. En el menú desplegable de la esquina superior derecha del Dashboard de la extensión, selecciona **Configuración**.
3. Busca la sección **API Key** y selecciona **Rotar**.

<div id="republish-the-extensions-rule">
  #### Vuelve a publicar la rule de la extensión
</div>

Selecciona **Publish Rule**.

<div id="delete-the-old-extension-rule-if-it-exists">
  #### Elimine la rule antigua de la extensión, si existe
</div>

1. Vaya a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Rules](https://manage.auth0.com/#/rules).
2. Localice la rule `auth0-authz`. Si no existe, ya ha terminado; de lo contrario, continúe con los siguientes pasos.
3. Localice la rule `auth0-authorization-extension` y arrástrela a la posición debajo de la rule `auth0-authz`.
4. Compruebe que la rule `auth0-authz`:

   * Fue creada por la Authorization Extension y no se ha modificado manualmente
   * No cambiará el flujo de autorización de forma que otorgue acceso o privilegios a usuarios no deseados si se elimina
5. Si se cumplen las condiciones anteriores, use el interruptor para deshabilitar la rule `auth0-authz`. Después de verificar que todo funcione correctamente, puede decidir si deja la rule deshabilitada o la elimina por completo.
