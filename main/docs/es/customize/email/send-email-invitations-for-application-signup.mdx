Una forma de gestionar los registros en una aplicación de Auth0 es mediante invitaciones de usuarios.

En un workflow típico de invitación de usuario, un administrador de la aplicación crea una nueva cuenta de usuario y luego envía por correo electrónico una invitación a un usuario con un enlace de activación. Cuando el usuario sigue el enlace, establece una contraseña para la cuenta y luego puede iniciar sesión.

Un sistema de invitación de usuario puede ser útil en casos de uso como restringir los registros o crear cuentas en bloque.

<div id="implementing-user-invitations-for-application-sign-up-on-auth0">
  ## Implementación de invitaciones de usuario para el registro en una aplicación con Auth0
</div>

En Auth0, puedes implementar invitaciones de usuario mediante el flujo de cambio de contraseña.

* Al personalizar el estilo de la página de restablecimiento de contraseña de Universal Login o Classic Login para que también admita una experiencia de invitación de usuario, puedes usar un enlace de cambio de contraseña como invitación a una nueva cuenta de usuario.

* Del mismo modo, al personalizar la plantilla de correo electrónico de cambio de contraseña, puedes usar el flujo de cambio de contraseña de Auth0 para generar el enlace de cambio de contraseña y enviar el correo de invitación con el [proveedor SMTP configurado](/docs/es/customize/email/smtp-email-providers) de tu tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si ya cuentas con una solución de correo electrónico externa y no quieres configurar un proveedor SMTP en tu tenant, puedes [usar la Management API para generar el ticket de cambio de contraseña](https://auth0.com/docs/api/management/v2/tickets/post-password-change) y enviar tú mismo por correo electrónico el enlace del ticket al usuario, en lugar de usar la plantilla de correo electrónico y el flujo de Auth0.
</Callout>

Esta implementación se basa en una propiedad de metadatos del usuario (llamada `user.app_metadata.needsInvitation` en estas instrucciones) que indica que el usuario tiene una invitación pendiente: cuando es `true`, el usuario recibe la experiencia de invitación de usuario; de lo contrario, recibe la experiencia habitual de restablecimiento de contraseña.

<div id="1-customize-the-password-change-email-template">
  ### 1. Personaliza la plantilla de correo electrónico de cambio de contraseña
</div>

Primero, [personaliza la plantilla de correo electrónico de cambio de contraseña (enlace)](/docs/es/customize/email/email-templates/customize-email-templates) para gestionar tanto las invitaciones de usuarios como los flujos de cambio de contraseña.

[Usa Liquid](/docs/es/customize/email/email-templates/supported-liquid-syntax) para comprobar si está establecida la propiedad de metadatos del usuario correspondiente a una invitación pendiente y enviar el mensaje adecuado. Por ejemplo:

```liquid lines
{% if user.app_metadata.needsInvitation %}
    // Contenido del correo electrónico de invitación de usuario aquí
{% else %}
    // Contenido del correo electrónico de cambio de contraseña aquí
{% endif %}
```

Además, añade un parámetro de consulta al enlace de restablecimiento de contraseña en el cuerpo del mensaje para indicar qué experiencia debe mostrarse al usuario en la página de restablecimiento de contraseña. Por ejemplo, puedes añadir `type=invite` o `type=reset`.

<div id="2-style-the-password-reset-page">
  ### 2. Dar estilo a la página de restablecimiento de contraseña
</div>

Según el parámetro de consulta que configuraste en la plantilla, da estilo a la página de restablecimiento de contraseña para que admita tanto la invitación de usuarios como los flujos de cambio de contraseña en función de ese parámetro.

Para obtener instrucciones sobre Universal Login, lee [Customize Universal Login](/docs/es/customize/login-pages/universal-login). Para obtener instrucciones sobre Classic Login, lee [Customize Password Reset Page](/docs/es/customize/login-pages/classic-login/customize-password-reset-page).

<div id="3-create-a-post-login-action-to-unset-the-invitation-property">
  ### 3. Cree una Action de Post Login para desactivar la propiedad de invitación
</div>

Una vez que el usuario establece su contraseña e inicia sesión por primera vez, ya no necesita una invitación y, en el futuro, debería pasar por el flujo de cambio de contraseña.

Para gestionar esto, [cree una Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) que establezca `user.app_metadata.needsInvitation` en `false`:

```javascript
exports.onExecutePostLogin = async (event, api) => {
    if (event.user.app_metadata.needsInvitation) {
        api.user.setAppMetadata("needsInvitation", false);
    }
};
```

<div id="4-send-user-invitations">
  ### 4. Enviar invitaciones de usuario
</div>

Para enviar una invitación de usuario, [cree un nuevo usuario de base de datos de Auth0 desde el Dashboard](/docs/es/authenticate/database-connections) o el [endpoint Crear un usuario](https://auth0.com/docs/api/management/v2/#!/Users/post_users) de la Management API y:

* Establezca el parámetro `user.email_verified` en `false`.

* Establezca `user.app_metadata.needsInvitation` en `true`.

Luego, llame al [endpoint de cambiar la contraseña](https://auth0.com/docs/api/authentication/change-password/change-password) de la Authentication API para enviar el correo electrónico de invitación al usuario.

<div id="learn-more">
  ## Más información
</div>

Para controlar adónde se redirige al usuario después de restablecer su contraseña:

* Para Universal Login, [personaliza la ruta de inicio de sesión del flujo de restablecimiento de contraseña](/docs/es/authenticate/login/auth0-universal-login/configure-default-login-routes#complete-password-reset-flow).

* Para Classic Login, configura el campo **Redirect To** en la plantilla de correo electrónico.

Para obtener más información sobre la configuración del correo electrónico, consulta:

* [Personalizar plantillas de correo electrónico](/docs/es/customize/email/email-templates)

* [Configurar proveedores externos de correo electrónico SMTP](/docs/es/customize/email/smtp-email-providers)
