<div id="prerequisites">
  ## Prerrequisitos
</div>

* Debe tener una cuenta de [Microsoft Azure](https://azure.microsoft.com/en-us).

* Debe configurar su proveedor para permitir conexiones entrantes desde [direcciones IP de Auth0 específicas](/docs/es/secure/security-guidance/data-security/allowlist).

<div id="configure-azure-communication-services">
  ## Configurar Azure Communication Services
</div>

Primero, crea un recurso en Microsoft Azure:

1. Inicia sesión en tu cuenta de [Microsoft Azure](https://azure.microsoft.com/en-us).
2. Escribe &quot;Communication Services&quot; en la barra de búsqueda.
3. Crea un recurso de [Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource) si aún no tienes uno.
4. Selecciona **Create**.
5. En la configuración de **Tools**, selecciona **Keys**. Obtén tu [Connection string de Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource).
6. En **Primary Key**, copia el valor de **Connection string**.
   Una cadena de conexión válida sigue el formato `endpoint=COMMUNICATION_SERVICES_URL;accesskey=ACCESS_KEY`.

A continuación, configura tu proveedor de correo electrónico en Auth0:

1. Ve a [Dashboard &gt; Branding &gt; proveedor de correo electrónico](https://manage.auth0.com/#/branding/email_provider).

2. Selecciona **Use my own email provider**.

3. Selecciona **Azure Communication Services**.

4. Introduce una dirección de correo electrónico en **From**.

   <Warning>
     Azure Communications Services solo admite una dirección de correo electrónico simple (`user@example.com`) en el campo **From**.
   </Warning>

5. Agrega la **Connection string** de Azure Communication Services.

6. Selecciona **Save**.

<div id="send-a-test-email">
  ## Enviar un correo electrónico de prueba
</div>

Envía un correo electrónico de prueba con el botón **Enviar correo electrónico de prueba**. Si la configuración es correcta, recibirás un correo electrónico de confirmación. Si no recibes ningún correo electrónico después de unos minutos, revisa tus [registros de Auth0](https://manage.auth0.com/#/logs) para ver los códigos de error.

Una vez completada la configuración, la página [Monitor de métricas](https://learn.microsoft.com/en-us/azure/communication-services/concepts/metrics#chat-api-request-metric-operations) de Communication Services muestra información sobre la entrega de todos los correos electrónicos enviados a tus usuarios.
