El objeto de API del trigger custom-email-provider de Actions incluye:

<div id="apicache">
  ## `api.cache`
</div>

Modifica la caché.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina, si existe, un registro que describe un valor en
caché en la clave proporcionada.

<ResponseField name="key" type="string">
  La clave del registro de caché que se va a eliminar.
</ResponseField>

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en la caché en la clave proporcionada,
si existe. Si se encuentra un registro, el valor almacenado en la caché se encuentra
en la propiedad `value` del objeto devuelto.

<ResponseField name="key" type="string">
  La clave del registro almacenado en la caché.
</ResponseField>

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de cadena en el caché con la clave especificada.

Los valores almacenados en este caché quedan limitados al trigger en el que
se establecen. Están sujetos a [Actions Cache Limits](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta forma tendrán una vigencia de *hasta* los
valores `ttl` o `expires_at` especificados. Si no se especifica ninguna vigencia, se usará una vigencia predeterminada de
15 minutos. La vigencia no puede superar la duración máxima
indicada en [Actions Cache Limits](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Este caché está diseñado para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si aún se encuentran dentro de la vigencia indicada.

<ResponseField name="key" type="string">
  La clave del registro que se almacenará.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se almacenará.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento del caché.

  <Expandable title="propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora de vencimiento absoluta en milisegundos desde la época Unix.
      Aunque los registros en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor `expires_at` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó un valor
      para `ttl`. Si se proporcionan ambas opciones, se usará
      el vencimiento más temprano de las dos.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada del caché en milisegundos.
      Aunque los valores en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor `ttl` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó un valor
      para `expires_at`. Si se proporcionan ambas opciones, se usará
      el vencimiento más temprano de las dos.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="apinotification">
  ## `api.notification`
</div>

Indica si el evento de notificación debe reintentarse o descartarse.
Consulta cada uno de estos métodos para obtener más información sobre el comportamiento real.
Si se realizan varias llamadas, solo se tiene en cuenta la última.

<div id="apinotificationdropreason">
  ### `api.notification.drop(reason)`
</div>

Al llamarse, el evento de notificación se considera fallido y sin posibilidad de recuperación:
Registraremos un error para este evento, pero no volveremos a enviarlo a la Action en el futuro.
Si necesitas que este evento de notificación se vuelva a intentar, considera llamar a retry en su lugar.

<ResponseField name="reason" type="string">
  este motivo formará parte de la entrada de registro; esto te ayudará a analizar el error más a fondo. Ten en cuenta que este campo está limitado a 1024 caracteres y se truncará si supera ese límite.
</ResponseField>

<div id="apinotificationretryreason">
  ### `api.notification.retry(reason)`
</div>

Cuando se llama, el evento de notificación se considera fallido, pero recuperable:
Registraremos un error para este evento, pero volveremos a intentarlo hasta 5 veces en los próximos minutos.
Si considera que este evento de notificación no debe volver a intentarse, llame a drop en su lugar.

<ResponseField name="reason" type="string">
  este motivo formará parte de la entrada de registro, lo que le ayudará a analizar mejor el error. Tenga en cuenta que este campo está limitado a 1024 caracteres y se truncará si lo supera.
</ResponseField>
