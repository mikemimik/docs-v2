Puedes configurar tu tenant con los datos del servidor SMTP de un proveedor de correo electrónico o de un servidor de correo que administres.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si admitimos directamente tu proveedor de correo electrónico, te recomendamos usar esa integración (en lugar de los datos del servidor SMTP), ya que admite protocolos de autenticación más seguros.
</Callout>

<div id="smtp-server-requirements">
  ## Requisitos del servidor SMTP
</div>

El servidor SMTP debe:

* Admitir [SMTP AUTH](https://en.wikipedia.org/wiki/SMTP_Authentication) con el protocolo LOGIN.
* Admitir [TLS 1.2](https://en.wikipedia.org/wiki/STARTTLS) o una versión superior.
* Tener un certificado firmado por una autoridad de certificación (CA) pública.
* Permitir conexiones entrantes desde las [direcciones IP de Auth0](/docs/es/secure/security-guidance/data-security/allowlist).

<div id="configure-tenants-with-an-smtp-server">
  ## Configurar tenants con un servidor SMTP
</div>

Puedes configurar tu tenant con un servidor SMTP desde el Auth0 Dashboard o mediante la Management API.

<div id="using-the-auth0-dashboard">
  ### Uso de Auth0 Dashboard
</div>

Para configurar su tenant con un servidor SMTP mediante Auth0 Dashboard:

1. Vaya a [Auth0 Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email-provider).
2. Active el toggle **Use my own email provider**.
3. Seleccione **SMTP**.
4. Introduzca una dirección de correo electrónico en **From** y, a continuación, los valores de **Host**, **Port**, **Username** y **Password** de su servidor SMTP.

   Recomendamos el puerto 587 si está disponible, ya que es el puerto SMTP predeterminado actual y admite TLS. Algunos providers pueden requerir el uso del puerto 465, que es más antiguo. No recomendamos el puerto 25, el puerto SMTP original, porque muchos providers e ISP lo bloquean.
5. Haga clic en **Save**.

Luego, puede enviar un correo electrónico de prueba haciendo clic en el botón **Send Test Email**. Si no recibe un correo electrónico de confirmación después de unos minutos, revise sus [Auth0 logs](https://manage.auth0.com/#/logs) para ver si hubo algún fallo.

<div id="using-the-management-api">
  ### Uso de la Management API
</div>

Para configurar su tenant con un servidor SMTP mediante la Management API, use el endpoint `/api/v2/emails/provider`.

* Para configurar un proveedor de correo electrónico por primera vez, envíe una [solicitud `POST` a `/api/v2/emails/provider`](/docs/es/api/management/v2/emails/post-provider).

* Para actualizar un proveedor de correo electrónico existente, envíe una [solicitud `PATCH` a `/api/v2/emails/provider`](/docs/es/api/management/v2/emails/patch-provider). Al cambiar el host, el puerto o el nombre de usuario, [debe proporcionar una contraseña](/docs/es/troubleshoot/product-lifecycle/deprecations-and-migrations#allow-omitting-password-on-smtp-email-provider-host-related-changes).

<div id="additional-considerations">
  ## Consideraciones adicionales
</div>

* Si necesitas soporte para protocolos de autenticación distintos de LOGIN, puedes [crear una Action para conectarte a tu servidor SMTP](/docs/es/customize/email/configure-a-custom-email-provider).

* En abril de 2026, Microsoft Exchange Online planea [dejar de admitir la autenticación básica para SMTH AUTH](https://techcommunity.microsoft.com/blog/exchange/exchange-online-to-retire-basic-auth-for-client-submission-smtp-auth/4114750), lo que afecta a los endpoints `smtp.office365.com` y `smtp-legacy.office365.com`.

  Recomendamos usar nuestras integraciones para [Microsoft Office 365 Exchange Online](/docs/es/customize/email/smtp-email-providers/configure-365-exchange-as-smtp-email-provider) y [Azure Communication Services Email](/docs/es/customize/email/smtp-email-providers/configure-azure-comm-service-as-smtp-email-provider).
