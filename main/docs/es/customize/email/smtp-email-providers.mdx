Los flujos de trabajo por correo electrónico de Auth0 gestionan cuándo se envían los correos electrónicos y qué contienen. Para entregar el correo saliente, necesita un servidor o servicio SMTP de un proveedor externo de correo electrónico.

Puede configurar un único proveedor de correo electrónico para gestionar todos los correos electrónicos que se envían desde su tenant. Ofrecemos tres opciones para esta configuración: integración con proveedores de correo electrónico de terceros compatibles, conexiones directas a servidores SMTP o una configuración personalizada con Actions.

<div id="supported-email-provider-integrations">
  ## Integraciones compatibles de proveedores de correo electrónico
</div>

Nuestras integraciones de proveedores de correo electrónico le permiten configurar su tenant agregando credenciales de API de un proveedor de terceros compatible.

Ofrecemos los siguientes proveedores:

* [Amazon SES](/docs/es/customize/email/smtp-email-providers/amazon-ses)
* [Azure Communication Services](/docs/es/customize/email/smtp-email-providers/azure-communication-services)
* [Mandrill](/docs/es/customize/email/smtp-email-providers/mandrill)
* [Microsoft 365 Exchange Online](/docs/es/customize/email/smtp-email-providers/microsoft-365-exchange-online)
* [Resend](/docs/es/customize/email/smtp-email-providers/resend)
* [SendGrid](/docs/es/customize/email/smtp-email-providers/sendgrid)
* [SparkPost](/docs/es/customize/email/smtp-email-providers/sparkpost)
* [Mailgun](/docs/es/customize/email/smtp-email-providers/mailgun)

<div id="custom-configuration-with-actions">
  ## Configuración personalizada con Actions
</div>

Puede crear una <Tooltip tip="Action: Una función de Node.js que se ejecuta en puntos definidos durante el runtime de Auth0." cta="Ver glosario" href="/docs/es/glossary?term=MFA">Action</Tooltip> para conectarse a cualquier proveedor externo de correo electrónico. Esto le permite aplicar lógica personalizada a su caso de uso y tener un mayor control sobre el proceso de entrega de correos electrónicos.

Para obtener más información, consulte [Configuración de Custom Email Provider con Actions](/docs/es/customize/email/smtp-email-providers/custom).

<div id="smtp-server-configuration">
  ## Configuración del servidor SMTP
</div>

Puedes configurar tu tenant añadiendo directamente los datos del servidor SMTP. Esta es una buena opción si:

* No admitimos la integración con el proveedor de correo electrónico que hayas elegido y no necesitas ni quieres escribir una Action para la configuración.

* Quieres configurar un servidor SMTP simulado (como [Debug Mail](https://debugmail.io/), [FakeSMTP](https://nilhcem.github.io/FakeSMTP/), o [smtp4dev](https://github.com/rnwood/smtp4dev)) para hacer pruebas.

No permitimos ciertos dominios falsos que se suelen usar durante las pruebas. Usa direcciones de correo electrónico reales para evitar interrupciones o errores.

Para ver las instrucciones, lee [Configurar un proveedor de correo electrónico con datos del servidor SMTP](/docs/es/customize/email/smtp-email-providers/smtp-server).

<div id="the-built-in-email-provider">
  ## El proveedor de correo electrónico integrado
</div>

El proveedor de correo electrónico integrado está disponible de forma predeterminada y no requiere configuración. Sin embargo, no admite su uso en producción y tiene las siguientes limitaciones:

* No puede usar plantillas de correo electrónico personalizadas.

* No puede personalizar la **dirección del remitente**. Todos los correos electrónicos se envían desde `no-reply@auth0user.net`.

* Está limitado a enviar 10 correos electrónicos por minuto, independientemente del tipo de correo electrónico.

* Si sus correos electrónicos tienen una tasa de rebote alta, podemos reducir o bloquear temporalmente la capacidad de enviar correos electrónicos desde su tenant.

Para eliminar estas limitaciones y usar correos electrónicos en producción, configure un proveedor externo de correo electrónico SMTP.
