description: Configure un proveedor externo de correo electrónico SMTP para enviar correos electrónicos en producción y usar plantillas de correo electrónico para personalizar su apariencia y contenido.

Los flujos de trabajo por correo electrónico de Auth0 gestionan cuándo se envían los correos y qué contenido incluyen. Para enviar el correo saliente, necesita un servidor o servicio SMTP de un proveedor externo de correo electrónico.

Puede configurar un único proveedor de correo electrónico para gestionar todos los correos enviados desde su tenant. Ofrecemos tres opciones para esta configuración: integración con proveedores de correo electrónico de terceros compatibles, conexiones directas a servidores SMTP o una configuración personalizada con Actions.

<div id="supported-email-provider-integrations">
  ## Integraciones compatibles de proveedores de correo electrónico
</div>

Nuestras integraciones de proveedores de correo electrónico te permiten configurar tu tenant agregando credenciales de API de un proveedor externo compatible.

Admitimos los siguientes proveedores:

* [Amazon SES](/docs/es/customize/email/smtp-email-providers/amazon-ses)
* [Azure Communication Services](/docs/es/customize/email/smtp-email-providers/azure-communication-services)
* [Mandrill](/docs/es/customize/email/smtp-email-providers/mandrill)
* [Microsoft 365 Exchange Online](/docs/es/customize/email/smtp-email-providers/microsoft-365-exchange-online)
* [Resend](/docs/es/customize/email/smtp-email-providers/resend)
* [SendGrid](/docs/es/customize/email/smtp-email-providers/sendgrid)
* [SparkPost](/docs/es/customize/email/smtp-email-providers/sparkpost)
* [Mailgun](/docs/es/customize/email/smtp-email-providers/mailgun)

<div id="custom-configuration-with-actions">
  ## Configuración personalizada con Actions
</div>

Puede crear una <Tooltip tip="Action: Una función de Node.js que se ejecuta en puntos definidos durante el runtime de Auth0." cta="Ver glosario" href="/docs/es/glossary?term=MFA">Action</Tooltip> para conectarse a cualquier proveedor externo de correo electrónico. Esto le permite aplicar lógica personalizada a su caso de uso y tener un mayor control sobre el proceso de entrega de correos electrónicos.

Para obtener más información, lea [Configuración de proveedor de correo electrónico personalizado con Actions](/docs/es/customize/email/smtp-email-providers/custom).

<div id="smtp-server-configuration">
  ## Configuración del servidor SMTP
</div>

Puede configurar su tenant añadiendo directamente los datos del servidor SMTP. Esta es una buena opción si:

* No admitimos la integración con el proveedor de correo electrónico que haya elegido y no necesita ni quiere escribir una Action para la configuración.

* Quiere configurar un servidor SMTP de prueba (como [Debug Mail](https://debugmail.io/), [FakeSMTP](https://nilhcem.github.io/FakeSMTP/), o [smtp4dev](https://github.com/rnwood/smtp4dev)) para hacer pruebas.

No permitimos ciertos dominios falsos que suelen usarse durante las pruebas. Use direcciones de correo electrónico reales para evitar interrupciones o errores.

Para obtener instrucciones, lea [Configurar un proveedor de correo electrónico con datos del servidor SMTP](/docs/es/customize/email/smtp-email-providers/smtp-server).

<div id="the-built-in-email-provider">
  ## El proveedor de correo electrónico integrado
</div>

El proveedor de correo electrónico integrado está disponible de forma predeterminada y no requiere configuración. Sin embargo, no se puede usar en Production y tiene las siguientes limitaciones:

* No puedes usar plantillas de correo electrónico personalizadas.

* No puedes personalizar la **dirección del remitente**. Todos los correos electrónicos se envían desde `no-reply@auth0user.net`.

* Solo puedes enviar 10 correos electrónicos por minuto, independientemente del tipo de correo electrónico.

* Si tus correos electrónicos tienen una tasa de rebote alta, podemos reducir o bloquear temporalmente la capacidad de enviar correos electrónicos desde tu tenant.

Para eliminar estas limitaciones y usar correos electrónicos en Production, configura un proveedor externo de correo electrónico SMTP.
