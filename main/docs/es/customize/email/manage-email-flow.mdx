La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> tiene varios endpoints relacionados con el correo electrónico que puedes usar para complementar los flujos de correo electrónico estándar. Por ejemplo, puedes:

* Enviar algunos tipos de correos electrónicos (como correos de inscripción en MFA) cuando lo necesites

* Crear tickets, que son URL generadas para acciones de flujos de correo electrónico (como el restablecimiento de contraseña), cuando lo necesites

* Personalizar el comportamiento de los tickets, por ejemplo, estableciendo distintas URL de **Return To** para los correos de verificación o configurando tickets de restablecimiento de contraseña para verificar la dirección de correo electrónico del usuario.

<div id="verification-email-endpoints">
  ## Endpoints del correo de verificación
</div>

<div id="password-change-endpoints">
  ## Endpoints de cambio de contraseña
</div>

Fuera del flujo de trabajo del enlace de cambio de contraseña, puedes usar el endpoint [Crear un ticket de cambio de contraseña](https://auth0.com/docs/api/management/v2/tickets/post-password-change) para generar tickets de cambio de contraseña.

La plantilla del enlace de cambio de contraseña tiene un campo **Redirect To** que admite variables, como el ID del usuario. Si necesitas un control más preciso, puedes enviarle al usuario por correo electrónico un ticket de cambio de contraseña que generes con este endpoint y establecer la `result_url` según tus propios criterios.

Este endpoint también te permite personalizar si se debe verificar el correo electrónico del usuario o incluir la dirección de correo electrónico como parte de la `return_url`.

<div id="mfa-enrollment-endpoints">
  ## Endpoints de inscripción en MFA
</div>

Fuera del flujo de inscripción en MFA, puedes usar el endpoint [Crear un ticket de inscripción en autenticación multifactor](https://auth0.com/docs/api/management/v2/guardian/post-ticket) para generar tickets de inscripción en MFA y, opcionalmente, enviar un correo electrónico con el ticket creado.

Al usar Universal Login, este endpoint también te permite especificar con qué factor debe inscribirse el usuario y si un usuario que ya está inscrito en MFA puede inscribirse con factores adicionales.
