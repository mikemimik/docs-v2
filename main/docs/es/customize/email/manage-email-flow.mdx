La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> tiene varios endpoints relacionados con el correo electrónico que puedes usar para ampliar los flujos de trabajo estándar de correo electrónico. Por ejemplo, puedes:

* Enviar algunos tipos de correos electrónicos (como los correos de inscripción en MFA) cuando lo necesites

* Crear tickets, que son URL generadas para acciones de flujos de trabajo de correo electrónico (como restablecimientos de contraseña), cuando lo necesites

* Personalizar el comportamiento de los tickets, por ejemplo, estableciendo distintas URL de **Return To** para los correos electrónicos de verificación o configurando tickets de restablecimiento de contraseña para verificar la dirección de correo electrónico del usuario.

<div id="verification-email-endpoints">
  ## Endpoints de verificación por correo electrónico
</div>

Fuera del flujo de verificación del correo electrónico, puedes usar el [endpoint para enviar un correo de verificación de la dirección de correo electrónico](https://auth0.com/docs/api/management/v2/jobs/post_verification_email) para enviar estos correos cuando lo necesites.

<div id="password-change-endpoints">
  ## Endpoints de cambio de contraseña
</div>

Además del flujo del enlace para cambiar la contraseña, puedes usar el endpoint [Create a password change ticket](https://auth0.com/docs/api/management/v2/tickets/post-password-change) para generar tickets de cambio de contraseña.

La plantilla del enlace de cambio de contraseña tiene un campo **Redirect To** que admite variables, como el ID del usuario. Si necesitas un control más preciso, puedes enviar al usuario por correo electrónico un ticket de restablecimiento de contraseña generado con este endpoint y definir la `result_url` según tus propios criterios.

Este endpoint también te permite personalizar si se debe verificar el correo electrónico del usuario o incluir la dirección de correo electrónico como parte de la `return_url`.

<div id="mfa-enrollment-endpoints">
  ## Endpoints de inscripción en MFA
</div>

Fuera del flujo de inscripción en MFA, puedes usar el endpoint [Create a multi-factor authentication enrollment ticket](https://auth0.com/docs/api/management/v2/guardian/post-ticket) para generar tickets de inscripción en MFA y, opcionalmente, enviar un correo electrónico con el ticket creado.

Al usar Universal Login, este endpoint también te permite especificar con qué factor debe registrarse el usuario y si un usuario que ya está inscrito en MFA puede registrarse con factores adicionales.
