---
title: Plantillas de correo electrónico
description: Habilita y personaliza los correos electrónicos enviados por los flujos de trabajo de Auth0 mediante plantillas de correo electrónico.
---

Cada correo electrónico que se envía como parte de los flujos de trabajo de Auth0 tiene una plantilla de correo electrónico correspondiente que determina la dirección del remitente, el asunto, el cuerpo del mensaje y más.

Después de [configurar un proveedor de correo electrónico SMTP externo](/docs/es/customize/email/smtp-email-providers), puedes modificar las plantillas de correo electrónico para personalizar tus correos.

Los campos de contenido de las plantillas de correo electrónico son compatibles con [Liquid](https://liquidjs.com/index.html), un lenguaje de plantillas de código abierto que amplía la funcionalidad de HTML. Liquid te permite:

* Rellenar variables de forma dinámica con datos, como reemplazar `{{ user.name }}` por el nombre del usuario
* Implementar lógica, como instrucciones `if`/`else`
* Modificar datos con filtros para manipular cadenas y arreglos, usar operadores matemáticos y más

Puedes habilitar, deshabilitar y personalizar las plantillas de correo electrónico en la [página de plantillas de correo electrónico del Dashboard](https://manage.auth0.com/#/templates).

| Correo electrónico de cuenta bloqueada                   | El acceso a una cuenta desde una dirección IP específica es bloqueado por la [protección contra ataques de fuerza bruta](/docs/es/secure/attack-protection/brute-force-protection).                                                                             | Un enlace para volver a habilitar el origen de la solicitud.                                 |
| Correo electrónico con código para cambiar la contraseña | Un usuario solicita un cambio de contraseña                                                                                                                                                                                                                  | Un enlace a la página de restablecimiento de contraseña y un código para usar en esa página. |
| Correo electrónico con enlace para cambiar la contraseña | Un usuario solicita un cambio de contraseña                                                                                                                                                                                                                  | Un enlace a la página de restablecimiento de contraseña.                                     |
| Correo electrónico de inscripción en MFA                 | Un administrador envía una invitación de inscripción en MFA de [Auth0 Guardian](/docs/es/secure/multi-factor-authentication/auth0-guardian).                                                                                                                    | Un enlace que muestra la experiencia de inscripción en MFA.                                  |
| Correo electrónico con código de verificación de MFA     | Un usuario que usa un verificador de correo electrónico de MFA necesita un código de verificación de MFA                                                                                                                                                     | Un código de verificación de MFA                                                             |
| Correo electrónico de alerta de filtración de contraseña | Auth0 detecta que el usuario está intentando acceder a la aplicación con una contraseña que ha sido filtrada por un tercero                                                                                                                                  | Contenido informativo                                                                        |
| Correo electrónico de Passwordless OTP                   | Un usuario inicia sesión con autenticación por correo electrónico <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> |                                                                                              |
| Correo electrónico de invitación de usuario              | Se invita a un usuario a una organización o aplicación                                                                                                                                                                                                       | Un enlace para unirse a la organización                                                      |
| Correo electrónico con código de verificación            | Ha habilitado la verificación basada en código y un usuario se registra o inicia sesión por primera vez, o ha habilitado la política de MFA adaptativo y hay una transacción de baja confianza cuya titularidad de la cuenta debe verificarse                | Un código que el usuario debe introducir durante la autenticación                            |
| Correo electrónico con enlace de verificación            | Un usuario se registra o inicia sesión por primera vez                                                                                                                                                                                                       | Un enlace para verificar el correo electrónico del usuario                                   |
| Correo electrónico de bienvenida                         | El usuario verifica su dirección de correo electrónico (si esta plantilla está habilitada), o se registra o inicia sesión por primera vez (si está deshabilitada)                                                                                            | Contenido informativo                                                                        |

<div id="email-template-limits">
  ## Límites de las plantillas de correo electrónico
</div>

* La personalización de plantillas de correo electrónico no está disponible cuando se usa el proveedor de correo electrónico integrado. Para usar plantillas de correo electrónico, [configure un proveedor de correo electrónico SMTP externo](/docs/es/customize/email/smtp-email-providers).

* Cada tipo de correo electrónico tiene exactamente una plantilla. Puede usar Liquid para implementar condicionales y otra lógica para casos de uso, como correos electrónicos multilingües.

* Las plantillas de correo electrónico no admiten correos electrónicos en texto plano. Todos los correos electrónicos deben estar en HTML.

* En la plantilla de correo electrónico de inscripción en MFA, la variable `connection.name` no está disponible.

* En la plantilla de correo electrónico de Passwordless OTP, solo está disponible la propiedad `email` del objeto `user`.

* A diferencia de otras plantillas de correo electrónico, la plantilla de correo electrónico de Passwordless OTP se configura en la [página de configuración de correo electrónico de Passwordless Connections](https://manage.auth0.com/#/connections/passwordless), en lugar de en la página plantilla de correo electrónico.

* Por motivos de seguridad, la **dirección del remitente** en las plantillas de correo electrónico no puede contener `@auth0.com`. Si introduce una dirección de correo electrónico `@auth0.com` en las direcciones del remitente, su tenant solo enviará correos electrónicos predeterminados e ignorará cualquier personalización. Los usuarios de Private Cloud pueden configurar una DenyList de dominios similar.
