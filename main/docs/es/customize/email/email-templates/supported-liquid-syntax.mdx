Para obtener una visión general de la sintaxis de Liquid, lee la [introducción a LiquidJS](https://liquidjs.com/tutorials/intro-to-liquid.html). Incluye un breve resumen de los dos tipos de marcado de Liquid:

* **Salidas**, que generan HTML.

  La sintaxis de las salidas consiste en una variable o un valor, y en una lista opcional de filtros, entre `{{` y `}}`. Por ejemplo, `{{ user.name }}`.

* **Etiquetas**, que controlan la renderización de la plantilla e implementan lógica.

  La sintaxis de las etiquetas consiste en un nombre de etiqueta y argumentos opcionales entre `{%` y `%}`. Por ejemplo, `{% if user_metadata.lang == 'it' %} Ciao! {% endif %}`.

En esta página se enumeran las etiquetas, los filtros y las variables admitidos en las plantillas de correo electrónico.

<div id="tags">
  ## Etiquetas
</div>

Todas las plantillas de correo electrónico son compatibles con las siguientes etiquetas de Liquid:

| Categoría        | Etiquetas                               |
| ---------------- | --------------------------------------- |
| Iteración        | `for`, `cycle`, `tablerow`              |
| Flujo de control | `if`, `unless`, `elsif`, `else`, `case` |
| Variable         | `assign`, `increment`, `decrement`      |
| Archivo          | `include`, `layout`                     |
| Lenguaje         | `raw`, `comment`                        |

Puedes consultar descripciones y ejemplos de las etiquetas de Liquid en la [página de etiquetas de LiquidJS](https://liquidjs.com/tags/overview.html).

<div id="debug-tag">
  ### etiqueta debug
</div>

Para facilitar la depuración y el desarrollo, nuestra etiqueta personalizada `{% debug %}` muestra un resumen de las variables de plantilla disponibles para tu plantilla en el momento en que se renderizó.

<div id="filters">
  ## Filtros
</div>

Todas las plantillas de correo electrónico admiten los siguientes filtros de Liquid:

| Categoría   | Filtros                                                                                                                                                                                            |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Matemáticas | `plus`, `minus`, `modulo`, `times`, `floor`, `ceil`, `round`, `divided_by`, `abs`, `at_least`, `at_most`                                                                                           |
| String      | `append`, `prepend`, `capitalize`, `upcase`, `downcase`, `strip`, `lstrip`, `rstrip`, `strip_newlines`, `split`, `replace`, `replace_first`, `remove`, `remove_first`, `truncate`, `truncatewords` |
| HTML/URI    | `escape`, `escape_once`, `url_encode`, `url_decode`, `strip_html`, `newline_to_br`                                                                                                                 |
| Arreglo     | `slice`, `map`, `sort`, `sort_natural`, `first`, `last`, `join`, `reverse`, `size`                                                                                                                 |
| Fecha       | `date`                                                                                                                                                                                             |
| Varios      | `default`                                                                                                                                                                                          |

Puedes consultar descripciones y ejemplos de los filtros de Liquid en la [página de filtros de LiquidJS](https://liquidjs.com/filters/overview.html).

<div id="general-variables">
  ## Variables generales
</div>

Las siguientes variables están disponibles en los campos **dirección del remitente**, **Asunto** y **Mensaje** de todas las plantillas (a menos que se indique lo contrario).

Para usar propiedades no confiables en las plantillas de correo electrónico, escápelas con el filtro `escape` (por ejemplo, `{{ user.name | escape }}`).

* El objeto `application`, con acceso a las propiedades estándar del cliente, como `application.name` y `application.client_metadata`

* `request_language`, selección de idioma para los usuarios

* `connection.name`

<Warning>
  - En la plantilla de correo electrónico de inscripción en MFA, la variable `connection.name` no está disponible.
</Warning>

* Información relacionada con el tenant definida en [Configuración del tenant](https://manage.auth0.com/#/tenant):
  * `tenant`, que es el nombre original del tenant
  * `friendly_name`
  * `support_email`
  * `support_url`

* Las siguientes variables de `user`:
  * `user.email`
  * `user.email_verified`
  * `user.picture` (**no confiable**)
  * `user.nickname` (**no confiable**)
  * `user.given_name` (**no confiable**)
  * `user.family_name` (**no confiable**)
  * `user.name` (**no confiable**)
  * `user.app_metadata`, atributos del usuario que pueden afectar su funcionalidad principal, como el funcionamiento de una aplicación o a qué puede acceder el usuario, incluidos su plan de soporte, roles de seguridad y grupos de control de acceso
  * `user.user_metadata` (**no confiable** si contiene información proporcionada por el usuario), atributos del usuario que no afectan su funcionalidad principal, incluidas sus preferencias

<Warning>
  En la plantilla de correo electrónico de Passwordless OTP, solo está disponible la propiedad `email` del objeto `user`.
</Warning>

* Cuando un usuario inicia sesión a través de una [Organization](/docs/es/manage-users/organizations/organizations-overview), las siguientes variables adicionales están disponibles:
  * `organization.id`
  * `organization.display_name`
  * `organization.name`
  * `organization.metadata`
  * `organization.branding.logo_url`
  * `organization.branding.colors.primary`
  * `organization.branding.colors.page_background`

* Variables para múltiples <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domains">dominios personalizados</Tooltip>:

  * `custom_domain.domain`, el dominio personalizado que se está usando (por ejemplo, `login.example.com`)
  * `custom_domain.domain_metadata`, los campos de metadatos del dominio personalizado (pares clave-valor) que puede usar para personalizar el contenido del correo electrónico según el dominio

Al usar [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains), Auth0 usa automáticamente el dominio personalizado adecuado para los enlaces de correo electrónico. Puede usar estas variables para personalizar el contenido del correo electrónico según qué dominio personalizado originó el correo.

**Ejemplo**: Personalice el branding del correo electrónico según los metadatos del dominio personalizado:

```liquid
{% if custom_domain.domain_metadata.brand == "BrandA" %}
  <h1 style="color: #0000FF;">Welcome to Brand A</h1>
{% elsif custom_domain.domain_metadata.brand == "BrandB" %}
  <h1 style="color: #FF0000;">Welcome to Brand B</h1>
{% else %}
  <h1>Welcome</h1>
{% endif %}
```

**Ejemplo**: Incluye información de soporte específica para el dominio:

```liquid
{% if custom_domain.domain_metadata.support_email %}
  <p>Need help? Contact us at {{ custom_domain.domain_metadata.support_email }}</p>
{% else %}
  <p>Need help? Contact us at {{ support_email }}</p>
{% endif %}
```

Para usar dominios personalizados en los correos, asegúrate de haber:

1. Configurado y verificado tus [dominios personalizados](/docs/es/customize/custom-domains)
2. Habilitado **Use Custom Domain in Emails** en **Dashboard** &gt; **Branding** &gt; **Custom Domains**
3. Establecido un [dominio personalizado predeterminado](/docs/es/customize/custom-domains/multiple-custom-domains/default-domain) si usas múltiples dominios personalizados

<div id="redirect-to-variables">
  ### Variables de Redirect To
</div>

En las plantillas de correo electrónico con un campo de URL **Redirect To**, solo se admiten las tres variables siguientes:

* `application.name` (o su sinónimo `client.name`)
* `application.clientID`
* `application.callback_domain` (o su sinónimo `client.callback_domain`), que contiene el origen de la primera URL de la lista **URL de devolución de llamada permitida** de la aplicación. Se trata de un origen y, por lo tanto, incluye el protocolo (como `https://`) además del dominio.

  El dominio de devolución de llamada permite redirigir a los usuarios a una ruta de la aplicación que activó la Action mediante una sintaxis como `{{ application.callback_domain }}/result_page`.

<div id="template-specific-variables">
  ## Variables específicas de la plantilla
</div>

Además de las variables comunes de Liquid disponibles en todas las plantillas de correo electrónico, las siguientes plantillas de correo electrónico cuentan con variables adicionales.

<div id="blocked-account-email-template-variables">
  ### Variables de la plantilla de correo electrónico de bloqueo de cuenta
</div>

* `user.source_ip`
* `user.city`
* `user.country`
* `url`, el enlace para desbloquear la cuenta

<div id="change-password-code-email-template-variables">
  ### Variables de la plantilla de correo electrónico para cambiar la contraseña (código)
</div>

* `code`, el código para cambiar la contraseña

<div id="change-password-link-email-template-variables">
  ### Variables de la plantilla de correo electrónico de cambiar la contraseña (enlace)
</div>

* `url`, el enlace para restablecer la contraseña

<div id="mfa-enrollment-email-template-variables">
  ### Variables de la plantilla de correo electrónico de inscripción en MFA
</div>

* `link`, el enlace de inscripción en MFA.

<div id="passwordless-otp-email-template-variables">
  ### Variables de la plantilla de correo electrónico de Passwordless OTP
</div>

* `send`, con un valor de `link`, `link_ios`, `link_android` o `code`, según el tipo de correo electrónico passwordless solicitado
* `code`, con el código de un solo uso para acceder a la aplicación
* `link`, con el enlace en el que el usuario puede hacer clic para acceder a la aplicación (solo para correos electrónicos passwordless de tipo enlace)
* `operation`, con un valor de `change_email` para una operación passwordless de cambio de correo electrónico

<div id="user-invitation-email-template-variables">
  ### Variables de la plantilla de correo electrónico de invitación de usuarios
</div>

* `inviter.name`
* `roles.id`
* `roles.name`
* `roles.description`

<div id="verification-email-code-template-variables">
  ### Variables de plantilla del correo electrónico de verificación (código)
</div>

* `code`, el código de verificación

<div id="verification-email-link-template-variables">
  ### Variables de plantilla del correo electrónico de verificación (enlace)
</div>

* `url`, el enlace de verificación
