Events ofrece a los clientes de Auth0 un método basado en API para sincronizar, correlacionar u orquestar los cambios que se producen en Auth0 o en <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> (IdPs) de terceros hacia aplicaciones externas o servicios de terceros.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Al habilitar Event Streams, el cliente pasa a ser el responsable del tratamiento de dicha información y, por tanto, de los datos exportados desde la plataforma Auth0, incluida la información que se transmite y el uso que se le da.
</Callout>

<div id="use-cases">
  ## Casos de uso
</div>

Events son notificaciones en tiempo real sobre acciones específicas —al principio, cambios en un usuario, y con el tiempo también de otro tipo— que ocurren dentro de tu tenant de Auth0. Puedes crear event streams que escuchen estos eventos para orquestar cambios asíncronos (es decir, cambios completados en una entidad o un flujo) en Auth0 y reenviarlos a uno de varios destinos para su procesamiento externo. Events admite diversos casos de uso, entre ellos:

* **[Sincronizar datos entre sistemas](/docs/es/customize/events/sync-data-across-systems)** — Mantén sincronizados con los cambios de identidad en Auth0 una base de datos externa, un almacén de datos o un índice de búsqueda.
* **[Correlacionar cambios de identidad con sistemas externos](/docs/es/customize/events/correlate-identity-changes)** — Reacciona a los cambios de identidad y propaga las actualizaciones a plataformas de CRM, servicios de facturación o herramientas de licencias SaaS.
* **[Orquestar flujos de trabajo empresariales](/docs/es/customize/events/orchestrate-business-workflows)** — Distribuye los cambios de identidad a varios sistemas de destino, como un CRM y un canal de notificación, en un solo flujo de trabajo.
