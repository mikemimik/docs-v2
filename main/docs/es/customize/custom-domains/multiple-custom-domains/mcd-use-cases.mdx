La funcionalidad Multiple <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o de marca." cta="Ver glosario" href="/docs/es/glossary?term=Custom+Domains">dominios personalizados</Tooltip> (MCD) de Auth0 ofrece una gran flexibilidad, ya que permite a las organizaciones gestionar distintas necesidades de marca, requisitos regionales e integraciones de partners, todo desde un único tenant de Auth0. Esto simplifica la gestión de identidades y, al mismo tiempo, ofrece a sus usuarios una experiencia fluida y de marca.

<div id="single-brand-multiple-top-level-domains-tlds">
  ### Una sola marca, múltiples dominios de nivel superior (TLD)
</div>

Algunas empresas, como las aerolíneas, necesitan distintos nombres de dominio de nivel superior (por ejemplo, `.com`, `.co.uk`, `.com.au`) incluso con una única base de usuarios. Esto suele responder a normativas jurisdiccionales o a las preferencias de los clientes. Con MCD, puedes crear fácilmente esta configuración en Auth0 y operar con un solo tenant y un repositorio de usuarios unificado, a la vez que ofreces múltiples dominios de nivel superior. Por ejemplo, tu sitio web principal podría ser `exampleco.com`, pero también puedes operar `exampleco.co.uk` y `exampleco.com.au`.

<div id="multiple-brands-single-user-base">
  ### Múltiples marcas, una sola base de usuarios
</div>

Es común que las empresas tengan varias marcas, cada una con su propio dominio, y aun así se beneficien de un único repositorio de usuarios y otras funcionalidades unificadas dentro de un solo tenant de Auth0. En el sector minorista, por ejemplo, una empresa podría gestionar distintas marcas que ofrecen muebles de alta gama para el hogar (`brand1-home-furniture.com`) y muebles de oficina (`brand2-office-furniture.com`). MCD permite que estas marcas mantengan nombres de dominio independientes mientras comparten un repositorio de usuarios centralizado y la configuración de Auth0.

<div id="white-labeled-b2b-saas">
  ### SaaS B2B de marca blanca
</div>

Muchos proveedores de SaaS ofrecen sus servicios con un modelo de marca blanca. Imagina job-saas.com, una plataforma para gestionar candidaturas de empleo. Las empresas cliente pueden ofrecer los servicios de `job-saas.com` desde un subdominio propio y personalizar su aspecto y experiencia, como `jobs.client-a.com` y `jobs.client-b.com`. Con MCD, `job-saas.com` puede mantener toda su configuración de Auth0 en un único tenant y, aun así, ofrecer distintas organizaciones y dominios personalizados directamente a sus clientes.
