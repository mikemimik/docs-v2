La funcionalidad Auth0 Multiple <Tooltip tip="Custom Domain: Dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=Custom+Domains">dominios personalizados</Tooltip> (MCD) ofrece una gran flexibilidad, ya que permite a las organizaciones gestionar distintas identidades de marca, requisitos regionales e integraciones con socios, todo desde un único tenant de Auth0. Esto simplifica la gestión de identidades y, al mismo tiempo, brinda a sus usuarios una experiencia fluida y alineada con la marca.

<div id="single-brand-multiple-top-level-domains-tlds">
  ### Una sola marca, varios dominios de nivel superior (TLD)
</div>

Algunas empresas, como las aerolíneas, necesitan distintos nombres de dominio de nivel superior (por ejemplo: `.com`, `.co.uk`, `.com.au`) aunque compartan una única base de usuarios. Esto suele deberse a normativas jurisdiccionales o a las preferencias de los clientes. Con MCD, puedes crear fácilmente esta configuración en Auth0, gestionando un único tenant con un repositorio de usuarios unificado y, al mismo tiempo, ofreciendo varios dominios de nivel superior. Por ejemplo, tu sitio web principal podría ser `exampleco.com`, pero también puedes operar `exampleco.co.uk` y `exampleco.com.au`.

<div id="multiple-brands-single-user-base">
  ### Varias marcas, una sola base de usuarios
</div>

Es habitual que las empresas tengan varias marcas, cada una con su propio dominio, y aun así se beneficien de un único repositorio de usuarios y de otras funcionalidades unificadas de un solo tenant de Auth0. En el sector minorista, por ejemplo, una empresa podría gestionar distintas marcas que ofrecen muebles de alta gama para el hogar (`brand1-home-furniture.com`) y muebles de oficina (`brand2-office-furniture.com`). MCD permite que estas marcas mantengan nombres de dominio independientes mientras comparten un repositorio de usuarios centralizado y la misma configuración de Auth0.

<div id="white-labeled-b2b-saas">
  ### SaaS B2B de marca blanca
</div>

Muchos proveedores de SaaS ofrecen sus servicios bajo un modelo de marca blanca. Imagina job-saas.com, una plataforma para gestionar solicitudes de empleo. Las empresas cliente pueden ofrecer los servicios de `job-saas.com` desde un subdominio propio y personalizar su apariencia y estilo, como `jobs.client-a.com` y `jobs.client-b.com`. Con MCD, `job-saas.com` puede mantener toda su configuración de Auth0 en un solo tenant y, aun así, ofrecer diferentes organizaciones y dominios personalizados directamente a sus clientes.
