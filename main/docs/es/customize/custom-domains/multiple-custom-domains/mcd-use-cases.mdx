La funcionalidad Auth0 Multiple <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=Custom+Domains">dominios personalizados</Tooltip> (MCD) ofrece una gran flexibilidad, ya que permite a las organizaciones gestionar distintas necesidades de marca, requisitos regionales e integraciones de partners, todo desde un único tenant de Auth0. Esto simplifica la gestión de identidades y, al mismo tiempo, brinda a sus usuarios una experiencia fluida y alineada con la marca.

<div id="single-brand-multiple-top-level-domains-tlds">
  ### Una sola marca, varios dominios de nivel superior (TLD)
</div>

Algunas empresas, como las aerolíneas, necesitan distintos nombres de dominio de nivel superior (por ejemplo, `.com`, `.co.uk`, `.com.au`) incluso con una sola base de usuarios. Esto suele deberse a normativas de distintas jurisdicciones o a las preferencias de los clientes. Con MCD, puedes crear fácilmente esta configuración en Auth0, usando un solo tenant con un repositorio de usuarios unificado y ofreciendo varios dominios de nivel superior. Por ejemplo, tu sitio web principal podría ser `exampleco.com`, pero también puedes operar `exampleco.co.uk` y `exampleco.com.au`.

<div id="multiple-brands-single-user-base">
  ### Múltiples marcas, una sola base de usuarios
</div>

Es habitual que las empresas tengan varias marcas, cada una con su propio dominio, y aun así se beneficien de un único repositorio de usuarios y otras funcionalidades unificadas de un único tenant de Auth0. En el comercio minorista, por ejemplo, una empresa podría gestionar distintas marcas que ofrecen muebles para el hogar de alta gama (`brand1-home-furniture.com`) y muebles de oficina (`brand2-office-furniture.com`). MCD permite que estas marcas mantengan nombres de dominio independientes mientras comparten un repositorio de usuarios centralizado y la configuración de Auth0.

<div id="white-labeled-b2b-saas">
  ### SaaS B2B de marca blanca
</div>

Muchos proveedores de SaaS ofrecen sus servicios bajo un modelo de marca blanca. Imagine job-saas.com, una plataforma para gestionar candidaturas de empleo. Las empresas cliente pueden alojar los servicios de `job-saas.com` en un subdominio propio y personalizar su aspecto y funcionamiento, como `jobs.client-a.com` y `jobs.client-b.com`. Con MCD, `job-saas.com` puede mantener toda su configuración de Auth0 en un solo tenant y, aun así, ofrecer directamente a sus clientes diferentes organizaciones y dominios personalizados.
