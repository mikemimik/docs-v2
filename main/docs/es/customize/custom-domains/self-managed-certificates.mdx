<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Debes tener una suscripción Enterprise para gestionar certificados en tu dominio personalizado. Para obtener más información, consulta [Auth0 Pricing](https://auth0.com/pricing) y [Login](/docs/es/authenticate/login).
</Callout>

Si eliges gestionar los certificados de tus <Tooltip tip="Custom Domain: Dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domains">dominios personalizados</Tooltip> sin la ayuda de Auth0, debes saber cómo administrar tu propio proxy y tus certificados para conexiones entrantes que requieren varios registros DNS en el dominio. Debes comprar o proporcionar los certificados y encargarte tú mismo de las renovaciones. También necesitarás un proxy inverso donde se instalará el certificado. Una vez verificado el dominio, aceptaremos tráfico desde el proxy.

Elige esta opción para:

* Tener más control sobre tus certificados (por ejemplo, elegir tu propia CA o la fecha de vencimiento del certificado).
* Habilitar monitoreo adicional de tus llamadas a la API de Auth0.

Para configurar tu dominio personalizado mediante certificados autogestionados, debes proporcionar tu nombre de dominio a Auth0, verificar que eres propietario de ese dominio y configurar el proxy inverso. Una vez configurado tu dominio personalizado, tendrás que ajustar tus funcionalidades de Auth0 para empezar a usarlo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Tu proxy hacia Auth0 debe usar TLS (SSL) versión 1.2 o posterior.
</Callout>

<div id="provide-your-domain-name-to-auth0">
  ## Indica tu nombre de dominio a Auth0
</div>

1. Ve a [Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) o [Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/custom_domains). Introduce tu dominio personalizado en el campo correspondiente y selecciona **certificados autogestionados** en el menú desplegable.
2. Haz clic en **Add Domain**.
   Solo puedes agregar un dominio por tenant, aunque el botón **Add Domain** siga apareciendo después de agregarlo.

<div id="verify-ownership">
  ## Verificar la titularidad
</div>

Antes de poder usar el dominio con Auth0, tendrás que verificar que te pertenece.

1. Ve a [Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/branding/custom_domains).

   <Frame>![Auth0 Branding Custom Domains Certificado autogestionado Verificar dominio](/docs/images/cdy7uua7fh8z/4XhEmQwKfoqydOldfHnnud/c63a58c72f79b4d19cd417f70ba262cc/Custom_Domains_-_Self_-_English.png)</Frame>

2. Añade la información de verificación TXT al registro DNS de tu dominio.
   Estos pasos pueden variar según tu proveedor de dominios:

   1. Mantén abierta en tu navegador la página del dominio personalizado de Auth0 para poder copiar los valores.
   2. Inicia sesión en tu servicio de administración de dominios.
   3. Crea un nuevo registro y guárdalo con esta configuración:

      | Parámetro          | Valor                                                               |
      | ------------------ | ------------------------------------------------------------------- |
      | Tipo de registro   | TXT                                                                 |
      | Nombre             | Copia y pega el valor de **TXT Record** de tu dominio desde Auth0.  |
      | Time to Live (TTL) | Usa el valor predeterminado.                                        |
      | Valor              | Copia y pega el valor de **TXT Content** de tu dominio desde Auth0. |

3. Haz clic en **Verify** para continuar.

   Auth0 puede tardar unos minutos en verificar tu dominio, según tu configuración de DNS.

   Si Auth0 verifica correctamente tu nombre de dominio, verás una ventana de confirmación.
   Guarda la información proporcionada en esta ventana, específicamente el valor de `cname-api-key`, ya que esta es la **única** vez que verás este valor.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Por motivos de seguridad, este valor se almacena como un hash, por lo que no se puede recuperar. Si no capturas `cname-api-key` en este momento, tendrás que volver a crear los dominios personalizados.
   </Callout>

   El proceso de verificación habrá finalizado y, en un plazo de 1 a 2 minutos, tu dominio personalizado debería estar listo para usarse. Si no puedes completar el proceso de verificación en un plazo de tres días, repite estos pasos.

<div id="configure-reverse-proxy">
  ## Configurar el proxy inverso
</div>

El servidor de proxy inverso recupera recursos de uno o más servidores en nombre de tu aplicación. Luego, esos recursos se devuelven a la aplicación como si se hubieran originado en el propio servidor proxy.

Puedes usar un servicio como [Cloudflare](/docs/es/customize/custom-domains/self-managed-certificates/configure-cloudflare-for-use-as-reverse-proxy), [Azure CDN](/docs/es/customize/custom-domains/self-managed-certificates/configure-azure-cdn-for-use-as-reverse-proxy), [Google Cloud Platform](/docs/es/customize/custom-domains/self-managed-certificates/configure-gcp-as-reverse-proxy) o [AWS Cloudfront](/docs/es/customize/custom-domains/self-managed-certificates/configure-aws-cloudfront-for-use-as-reverse-proxy) y configurar los ajustes de tu dominio personalizado. Añadirás el nuevo valor CNAME a tu DNS para que tu dominio personalizado apunte al nombre de dominio del servidor de proxy inverso para la distribución.

1. Después de crear la configuración del proxy inverso en tu servicio, ve a la pestaña [Auth0 Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/branding/custom_domains).
2. Agrega un nuevo registro CNAME a tu DNS para que tu dominio personalizado apunte al nombre de dominio del servicio para tu distribución. Puedes encontrarlo buscando el **Distribution ID** en la configuración de tu servidor de proxy inverso.
   Una vez agregado, el registro CNAME debe estar presente en todo momento para evitar problemas durante la renovación del certificado.
3. La forma de configurar el servidor proxy variará según el servicio que uses. Es probable que debas configurar los siguientes tipos de ajustes:

* [Distribution](#distribution-settings)
* [hostname de origen](#origin-hostname-settings)
* [Encabezados HTTP](#http-headers)
* [Comportamiento predeterminado de la caché](#default-cache-behavior-settings)

<div id="distribution-settings">
  ### Configuración de Distribution
</div>

| Configuración                   | Valor                                                                                                                                               |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Origin Domain Name              | Establece este valor en el **Origin Domain Name** obtenido del Auth0 Dashboard durante el proceso de Setup de Custom Domains.                       |
| Origin ID                       | Una descripción del origen. Este valor te permite distinguir entre varios orígenes dentro de la misma Distribution y, por lo tanto, debe ser único. |
| Origin Protocol Policy          | Establécelo en `HTTPS Only`.                                                                                                                        |
| Alternate Domain Names (CNAMEs) | Establece este valor en tu dominio personalizado (el mismo que configuraste en el Auth0 Dashboard).                                                 |

<div id="origin-hostname-settings">
  ### Configuración del hostname de origen
</div>

| Configuración                | Valor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| hostname de origen           | Ingrese `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.us.auth0.com` y asegúrese de reemplazar `<CUSTOM_DOMAIN_ID>` por el valor de Origin Domain Name (ID del dominio personalizado) que recibió de Auth0 al configurar el nuevo dominio personalizado. Si sus tenants no están en la región de EE. UU., use una de las siguientes opciones: <ul><li>EU: `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.eu.auth0.com`</li><li>AU: `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.au.auth0.com`</li><li>JP: `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.jp.auth0.com`</li></ul> Por ejemplo, si su CUSTOM&#95;DOMAIN&#95;ID es:<br /><br />`cd_TXIdNgQ07HrAFVmz`<br /><br />Para un tenant de EE. UU., el hostname de origen debe ser:<br /><br />`US: {yourTenant}-cd-txIdngq07hrafvmz.edge.tenants.us.auth0.com` |
| Server Name Indication (SNI) | Use el nombre que proporcionó para el hostname de origen.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |

Para obtener más información sobre cómo consultar los detalles de su dominio personalizado, vea [Get custom domain configurations](https://auth0.com/docs/api/management/v2/custom-domains/get-custom-domains).

<div id="http-headers">
  ### Encabezados HTTP
</div>

| Nombre del encabezado | Valor                                                                                                                                                                                                                                                                                                                                                                |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Host`                | El [hostname de origen](#origin-hostname-settings).                                                                                                                                                                                                                                                                                                                  |
| `cname-api-key`       | El valor de la API Key de CNAME que se te proporcionó justo después de que verificaras la propiedad de tu nombre de dominio con Auth0. Debes establecer este encabezado para que Auth0 acepte tráfico desde tu proxy.                                                                                                                                                |
| `X-Forwarded-For`     | La dirección IP del usuario final. No especifiques varias direcciones IP. <br /><br /> Puedes especificar otro encabezado para la dirección IP del usuario mediante el endpoint [Configurar un nuevo dominio personalizado](/docs/es/api/management/v2/custom-domains/post-custom-domains) de la Management API para establecer la propiedad `custom_client_ip_header`. |

Auth0 usa la dirección IP del usuario final para funciones como la detección de anomalías, la limitación de frecuencia y MFA. Si tu proxy no reenvía la dirección IP del usuario final, Auth0 considera la dirección IP de tu proxy como la del cliente, lo que puede hacer que estas funciones se comporten de forma incorrecta.

<div id="default-cache-behavior-settings">
  ### Configuración predeterminada del comportamiento de caché
</div>

| Configuración                                              | Valor                                                                                                                           |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Política de protocolo del visor                            | Seleccione **Redirect HTTP to HTTPS**.                                                                                          |
| Métodos HTTP permitidos                                    | Seleccione **GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE**.                                                                    |
| Caché basada en los encabezados de solicitud seleccionados | Seleccione **Allow list**.                                                                                                      |
| Encabezados de la lista de permitidos                      | Se deben permitir los siguientes encabezados: `User-Agent`, `Origin`, `Referer`, `Authorization`, `Accept` y `Accept-Language`. |
| Reenviar cookies                                           | Seleccione **All**.                                                                                                             |
| Reenvío y almacenamiento en caché de la cadena de consulta | Seleccione **Forward all, cache based on all**.                                                                                 |

<div id="additional-steps-for-specific-auth0-features">
  ## Pasos adicionales para funcionalidades específicas de Auth0
</div>

Hay pasos de configuración adicionales que debes completar según las funcionalidades de Auth0 que uses. Para obtener más información, lee [Configurar funcionalidades para usar dominios personalizados](/docs/es/customize/custom-domains/configure-features-to-use-custom-domains).

Cloudflare utiliza una funcionalidad llamada [aplanamiento de CNAME](https://developers.cloudflare.com/dns/cname-flattening/), que afecta a la verificación de Auth0 y a la renovación de certificados debido a cómo gestiona los registros DNS. Recomendamos desactivar el aplanamiento de CNAME, a menos que sea absolutamente necesario.

Como esta funcionalidad te da control sobre tus propios certificados, debes diseñar estrategias de mitigación (como un firewall para aplicaciones web) para bloquear solicitudes en caso de ataque. Para obtener más información, lee [nuestro blog](https://auth0.com/blog/the-future-of-bot-protection-smarter-attacks-demand-a-layered-approach/) sobre el futuro de la protección contra bots.
