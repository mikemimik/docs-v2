title: Configurar Azure CDN como proxy inverso
---

<Card title="La disponibilidad varía según el plan de Auth0">
  La disponibilidad de esta funcionalidad depende de tu plan de Auth0 o de tu acuerdo personalizado. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Card>

Para configurar Azure CDN como proxy inverso, se requiere un plan Azure CDN Premium.

1. [Configura dominio personalizado con Self-Managed Certificates](/docs/es/customize/custom-domains/self-managed-certificates) si aún no lo has hecho. Toma nota de los valores de **Origin Domain Name** y **cname-api-key**, ya que los necesitarás más adelante.
2. Inicia sesión en [Azure Portal](https://portal.azure.com/).
3. [Crea un nuevo perfil y endpoint de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint). En la configuración del endpoint de CDN, proporciona el **Origin Domain Name** de Auth0 Dashboard.
4. [Agrega tu dominio personalizado a tu endpoint de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup).
5. [Configura HTTPS para tu dominio personalizado de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate). Este proceso requiere verificar la propiedad del dominio. Una vez completado, el certificado puede tardar hasta 6 horas en implementarse en todas las ubicaciones POP de la CDN.
6. Configura la comunicación del dominio personalizado con Auth0 mediante el motor de Rules de Azure CDN. (Para obtener más información, consulta [Override HTTP behavior using the Azure CDN from Verizon Premium rules engine](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) en la documentación de Microsoft). [Crea una nueva Rule de Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) con la siguiente configuración:

   | Configuración    | Valor                                                                                                   |
   | ---------------- | ------------------------------------------------------------------------------------------------------- |
   | Name/Description | Auth0 Custom Domain                                                                                     |
   | Type of requests | Selecciona la opción **Edge CName** y, a continuación, selecciona tu dominio personalizado de la lista. |
7. Agrega las siguientes **funcionalidades** a tu Rule de Azure CDN:
   Recomendamos crear otra Rule de Azure CDN para denegar el uso del CNAME **azureedge.net**.

   | Configuración                | Valor                                                                                                                                |
   | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
   | Bypass Cache                 | **Enabled**                                                                                                                          |
   | Modify Client Request Header | Selecciona **Override**, introduce **cname-api-key** como nombre e introduce la API Key de CNAME proporcionada por Auth0 como valor. |
8. Una vez aprobada la Rule de Azure CDN, el estado cambiará de Pending XML a Active XML. En ese momento, Azure CDN publicará las reglas y los certificados. Cuando Azure termine de procesar todos los cambios, podrás usar tu dominio personalizado.
