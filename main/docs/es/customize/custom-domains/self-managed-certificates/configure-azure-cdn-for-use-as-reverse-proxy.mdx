title: Configurar Azure CDN como proxy inverso
---

<Card title="La disponibilidad varía según el plan de Auth0">
  La disponibilidad de esta funcionalidad depende de tu plan de Auth0 o de tu acuerdo personalizado. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Card>

Para configurar Azure CDN como proxy inverso, se requiere un plan Premium de Azure CDN.

1. [Configura dominios personalizados con certificados autogestionados](/docs/es/customize/custom-domains/self-managed-certificates) si aún no lo has hecho. Toma nota de los valores de **Origin Domain Name** y **cname-api-key**, ya que los necesitarás más adelante.
2. Inicia sesión en el [Azure Portal](https://portal.azure.com/).
3. [Crea un nuevo perfil y endpoint de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint). En la configuración del endpoint de CDN, proporciona el valor de **Origin Domain Name** desde Auth0 Dashboard.
4. [Agrega tu dominio personalizado a tu endpoint de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup).
5. [Configura HTTPS para tu dominio personalizado de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate). Este proceso requiere que verifiques la propiedad del dominio. Una vez completado, la implementación del certificado en todas las ubicaciones POP de la CDN puede tardar hasta 6 horas.
6. Configura la comunicación del dominio personalizado con Auth0 mediante el motor de reglas de Azure CDN. (Para obtener más información, consulta [Override HTTP behavior using the Azure CDN from Verizon Premium rules engine](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) en la documentación de Microsoft). [Crea una nueva regla de Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) con la siguiente configuración:

   | Configuración       | Valor                                                                                                  |
   | ------------------- | ------------------------------------------------------------------------------------------------------ |
   | Nombre/Descripción  | Auth0 Custom Domain                                                                                    |
   | Tipo de solicitudes | Selecciona la opción **Edge CName** y luego selecciona tu nombre de dominio personalizado de la lista. |
7. Agrega las siguientes **Features** a tu regla de Azure CDN:
   Te recomendamos crear otra regla de Azure CDN para denegar el uso del CNAME **azureedge.net**.

   | Configuración                                    | Valor                                                                                                                         |
   | ------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
   | Omitir caché                                     | **Enabled**                                                                                                                   |
   | Modificar el encabezado de solicitud del cliente | Selecciona **Override**, ingresa **cname-api-key** como nombre e ingresa la CNAME API Key proporcionada por Auth0 como valor. |
8. Una vez que se apruebe la regla de Azure CDN, el estado cambiará de Pending XML a Active XML. En ese momento, Azure CDN publicará las reglas y los certificados. Cuando Azure termine de procesar todos los cambios, podrás usar tu dominio personalizado.
