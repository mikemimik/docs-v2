<Card title="La disponibilidad varía según el plan de Auth0">
  Tanto tu implementación específica de inicio de sesión como tu plan de Auth0 o acuerdo personalizado influyen en si esta función está disponible. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Card>

Para configurar un producto de Akamai como proxy inverso, completa lo siguiente:

1. Verifica la propiedad de tu dominio personalizado. Para obtener más información, consulta [Verificar la propiedad](/docs/es/customize/custom-domains/auth0-managed-certificates#verify-ownership).
2. Ve a [Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) para recopilar la distribución, el encabezado personalizado de origen, el hostname de origen y la configuración predeterminada del comportamiento de caché. Luego, usa esta información para actualizar la configuración de tu dominio personalizado en Akamai para que apunte al dominio del servidor del proxy inverso. Para obtener más información, consulta [Configurar proxy inverso](/docs/es/customize/custom-domains/self-managed-certificates#configure-reverse-proxy).
3. Actualiza el encabezado de IP del cliente del dominio personalizado usando el endpoint PATCH de la Management API  `/api/v2/custom-domains/{id}` como se muestra a continuación:

   ```json lines
   { 
   "custom_client_ip_header": "true-client-ip" 
   }
   ```

   Para obtener más información, consulta [Actualizar la configuración del dominio personalizado](https://auth0.com/docs/api/management/v2#!/Custom_Domains/patch_custom_domains_by_id).

Para obtener más detalles sobre cómo configurar Akamai como proxy inverso con la información anterior, consulta la documentación de Akamai o ponte en contacto con soporte.
