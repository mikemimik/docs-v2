<Card title="Antes de comenzar">
  El perímetro de red de Auth0 requiere que se configure la Server Name Indication (SNI) en todas las solicitudes. La mayoría de los clientes establecen SNI de forma predeterminada; si su cliente web no lo hace, consulte la documentación de su cliente web para saber cómo configurar manualmente una SNI.
</Card>

El perímetro de red de Auth0 dispone de un conjunto seguro de combinaciones permitidas de versiones SSL/TLS y conjuntos de cifrado. Al conectarse a los servicios de Auth0 mediante un proxy inverso con certificados autogestionados, **debe** usar una versión de TLS y un conjunto de cifrado compatibles. Durante el handshake de TLS, el servidor y el cliente especifican la versión de TLS y el conjunto de cifrado. Si no utiliza una versión compatible, podría producirse un fallo.

<div id="supported-versions">
  ## Versiones compatibles
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si usas certificados autogestionados en tu dominio personalizado, deben ser compatibles con alguna de las versiones de TLS y los cifrados que se indican a continuación. Por motivos de seguridad, un protocolo o cifrado puede dejar de ser compatible sin previo aviso.
</Callout>

Auth0 requiere el uso de TLS 1.2 o 1.3 con los cifrados compatibles.

<div id="tls-13-supported-ciphers">
  #### Cifrados compatibles con TLS 1.3
</div>

TLS&#95;AES&#95;128&#95;GCM&#95;SHA256
TLS&#95;AES&#95;256&#95;GCM&#95;SHA384
TLS&#95;CHACHA20&#95;POLY1305&#95;SHA256

  #### Cifrados compatibles con TLS 1.2
</div>

TLS&#95;ECDHE&#95;ECDSA&#95;WITH&#95;CHACHA20&#95;POLY1305&#95;SHA256

<div id="tls-parameters">
  #### Parámetros de TLS
</div>
