title: Auth0 Hooks
---

<Warning>
  La fecha de End of Life (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para los nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos enriquecida, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprender cómo funcionan Auth0 Actions](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página dedicada de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra publicación del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como planeamos eliminar las funciones de Rules y Hooks en 2026, debes crear nuevas Rules o Hooks solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para aprender a migrar tus Rules a Actions, consulta [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para aprender a migrar tus Hooks a Actions, consulta [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si estás detrás de un firewall, para que esta funcionalidad funcione correctamente, puede que necesites agregar las direcciones IP de Auth0 adecuadas a la lista de permitidos. Para obtener más información, consulta [Direcciones IP de Auth0 para listas de permitidos](/docs/es/secure/security-guidance/data-security/allowlist).
</Callout>

Hooks son funciones seguras y autocontenidas que te permiten personalizar el comportamiento de Auth0 cuando se ejecutan en puntos de extensibilidad seleccionados de la plataforma Auth0. Auth0 invoca hooks durante el runtime para ejecutar tu código personalizado de Node.js.

La posibilidad de usar hooks con conexiones varía según el punto de extensibilidad. Los hooks que pueden usarse con conexiones solo funcionan con conexiones de base de datos y <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip> y están sujetos a la [Política de límite de tasa](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy) de Auth0. Para obtener más información, consulta [Database Connections](/docs/es/authenticate/database-connections) y [conexiones passwordless](/docs/es/authenticate/passwordless).
