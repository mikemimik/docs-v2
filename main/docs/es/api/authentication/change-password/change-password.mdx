<div id="endpoint">
  ## Endpoint
</div>

`POST /dbconnections/change_password`

Envía un correo electrónico para cambiar la contraseña a la dirección de correo electrónico y la `conexión` proporcionadas por el usuario.

Opcionalmente, puedes proporcionar un ID de organización para admitir variables específicas de la organización en las [plantillas de correo electrónico personalizadas](https://auth0.com/docs/customize/email/email-templates) e incluir los parámetros `organization_id` y `organization_name` en la URL de **Redirect To**.

**Nota:** Este endpoint solo funciona con conexiones de base de datos.

<div id="remarks">
  ### Observaciones
</div>

* Cuando el usuario haga clic en el enlace para cambiar la contraseña, se le redirigirá a una página donde se le pedirá una nueva contraseña.
* Este endpoint devolverá tres encabezados de respuesta HTTP que proporcionan datos relevantes sobre sus límites de velocidad:
  * `X-RateLimit-Limit`: Número de solicitudes permitidas por minuto.
  * `X-RateLimit-Remaining`: Número de solicitudes disponibles. Cada nueva solicitud reduce este número en 1. Por cada minuto que transcurre, se vuelve a añadir una solicitud, por lo que este número aumenta en 1 cada vez.
  * `X-RateLimit-Reset`: Tiempo restante hasta que se restablezca el límite de velocidad (`X-RateLimit-Limit`). El valor está en [segundos de época UTC](https://en.wikipedia.org/wiki/Unix_time).

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu aplicación cliente.
</ParamField>

<ParamField body="email" type="string" required>
  La dirección de correo electrónico del usuario.
</ParamField>

<ParamField body="connection" type="string" required>
  El nombre de la conexión de base de datos configurada para tu aplicación cliente.
</ParamField>

<ParamField body="organization" type="string">
  El `organization_id` de la organización asociada al usuario.
</ParamField>

<div id="response-schema">
  ## Esquema de la respuesta
</div>

<ResponseSchema type="string">
  El cuerpo de la respuesta 200 es una cadena de texto sin formato.

  <br />

  Ejemplo: `"Te acabamos de enviar un correo electrónico para cambiar la contraseña."`
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                      |
| ------ | ---------------------------------------------------------------- |
| 200    | Se ha enviado un correo electrónico para cambiar la contraseña. |
| 400    | Solicitud incorrecta: faltan parámetros o no son válidos.           |
| 404    | Usuario no encontrado.                                              |
