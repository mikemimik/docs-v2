<div id="endpoint">
  ## Endpoint
</div>

`POST /tokeninfo`

<Note>
  Este endpoint forma parte del pipeline de autenticación heredada y se deshabilitará para quienes utilicen nuestro pipeline más reciente, compatible con OIDC. Recomendamos usar en su lugar el [endpoint /userinfo](#get-user-info). Para obtener más información sobre el pipeline de autenticación más reciente, consulta [Introducing OIDC Conformant Authentication](/api-auth/intro).
</Note>

Este endpoint valida un JSON Web Token (JWT) (firma y expiración) y devuelve la información del usuario asociada a la propiedad `sub` del token, que contiene el ID de usuario.

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint devuelve tres cabeceras de respuesta HTTP que proporcionan información relevante sobre sus límites de frecuencia:
  * `X-RateLimit-Limit`: Número de solicitudes permitidas por minuto.
  * `X-RateLimit-Remaining`: Número de solicitudes disponibles. Cada nueva solicitud reduce este número en 1. Por cada minuto que transcurre, se repone una solicitud, por lo que este número aumenta en 1 cada vez.
  * `X-RateLimit-Reset`: Tiempo restante hasta que se restablezca el límite de frecuencia (`X-RateLimit-Limit`). El valor se expresa en [segundos de época UTC](https://en.wikipedia.org/wiki/Unix_time).

  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="email_verified" type="boolean">
    Indica si la dirección de correo electrónico del usuario está verificada.
  </ResponseField>

  <ResponseField name="email" type="string">
    La dirección de correo electrónico del usuario.
  </ResponseField>

  <ResponseField name="clientID" type="string">
    El ID de cliente asociado al token.
  </ResponseField>

  <ResponseField name="updated_at" type="string">
    Fecha y hora de la última actualización de la información del usuario.
  </ResponseField>

  <ResponseField name="name" type="string">
    El nombre completo del usuario.
  </ResponseField>

  <ResponseField name="picture" type="string">
    URL de la foto de perfil del usuario.
  </ResponseField>

  <ResponseField name="user_id" type="string">
    El identificador único del usuario.
  </ResponseField>

  <ResponseField name="nickname" type="string">
    El apodo del usuario.
  </ResponseField>

  <ResponseField name="provider" type="string">
    El proveedor de identidad del usuario.
  </ResponseField>

  <ResponseField name="connection" type="string">
    La conexión utilizada para autenticar al usuario.
  </ResponseField>

  <ResponseField name="isSocial" type="boolean">
    Indica si el usuario se autenticó mediante un proveedor social.
  </ResponseField>

  <ResponseField name="created_at" type="string">
    Fecha y hora de creación del usuario.
  </ResponseField>

  <ResponseField name="global_client_id" type="string">
    El ID de cliente global.
  </ResponseField>
</ResponseSchema>
