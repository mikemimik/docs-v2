<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/ro`

<Note>
  Este endpoint forma parte del pipeline de autenticación heredada y ha sido sustituido por la [Concesión de contraseña](#resource-owner-password). Para obtener más información sobre el pipeline de autenticación más reciente, consulta [Introducing OIDC Conformant Authentication](/api-auth/intro).
</Note>

Usa este endpoint para la autenticación basada en API (activa). Con las credenciales del usuario y la `conexión` especificada, autentica al usuario con el proveedor y devuelve un JSON con el token de acceso y el token de ID.

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo funciona con conexiones de base de datos, conexiones passwordless, Active Directory/LDAP, Windows Azure AD y ADFS.

* La principal diferencia entre la autenticación pasiva y la activa es que la primera tiene lugar en el navegador a través de la [Auth0 Login Page](http://manage.auth0.com/login), mientras que la segunda puede invocarse desde cualquier lugar (un script, de servidor a servidor, etc.).

* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si usa la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js/v7).

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu aplicación.
</ParamField>

<ParamField body="username" type="string" required>
  Nombre de usuario/correo electrónico del usuario que va a iniciar sesión.
</ParamField>

<ParamField body="password" type="string" required>
  Contraseña del usuario que va a iniciar sesión.
</ParamField>

<ParamField body="connection" type="string" required>
  El nombre de la conexión que se usará para iniciar sesión.
</ParamField>

<ParamField body="scope" type="string">
  Configúralo como `openid` para obtener también un token de ID.
</ParamField>

<ParamField body="grant_type" type="string" required>
  Configúralo como `password` o `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="device" type="string">
  Obligatorio cuando `grant_type` es `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="id_token" type="string">
  Obligatorio cuando `grant_type` es `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema statusCode="200">
  <ResponseField name="id_token" type="string">
    El token de ID.
  </ResponseField>

  <ResponseField name="access_token" type="string">
    El token de acceso.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    El tipo de token. Normalmente, `Bearer`.
  </ResponseField>
</ResponseSchema>

<ResponseSchema statusCode="400">
  <ResponseField name="error" type="string">
    Código de error.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Descripción del error.
  </ResponseField>
</ResponseSchema>
