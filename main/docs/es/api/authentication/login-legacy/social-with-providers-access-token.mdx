<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/access_token`

<Note>
  Este endpoint forma parte del pipeline de autenticación heredada. En su lugar, le recomendamos abrir el navegador para realizar la autenticación social, tal como [recomiendan Google y Facebook](https://developers.googleblog.com/2016/08/modernizing-oauth-interactions-in-native-apps.html). Para obtener más información sobre el pipeline de autenticación más reciente, consulte [Introducing OIDC Conformant Authentication](/api-auth/intro).

  Esta funcionalidad está deshabilitada de forma predeterminada para los tenants nuevos desde el 8 de junio de 2017. Consulte [Tipos de concesión de aplicaciones](/applications/concepts/application-grant-types) para obtener más información.
</Note>

Dado el token de acceso del proveedor social y la `conexión`, este endpoint autenticará al usuario con el proveedor y devolverá un JSON con el token de acceso y, opcionalmente, un token de ID. Este endpoint solo funciona con Facebook, Google, Twitter y Weibo.

<div id="remarks">
  ### Observaciones
</div>

* El valor de `scope` `profile` solicita acceso a las claims predeterminadas del perfil del usuario final, que son: `name`, `family_name`, `given_name`, `middle_name`, `nickname`, `preferred_username`, `profile`, `picture`, `website`, `gender`, `birthdate`, `zoneinfo`, `locale` y `updated_at`.

* El valor de `scope` `email` solicita acceso a las claims `email` y `email_verified`.

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu aplicación.
</ParamField>

<ParamField body="access_token" type="string" required>
  El token de acceso del proveedor social.
</ParamField>

<ParamField body="connection" type="string" required>
  El nombre de una conexión configurada en tu aplicación.
</ParamField>

<ParamField body="scope" type="string">
  Usa `openid` para obtener un token de ID, o `openid profile email` para incluir información del usuario.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema statusCode="200">
  <ResponseField name="id_token" type="string">
    El token de ID.
  </ResponseField>

  <ResponseField name="access_token" type="string">
    El token de acceso.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    El tipo de token. Normalmente, `Bearer`.
  </ResponseField>
</ResponseSchema>

<ResponseSchema statusCode="400 / 401 / 403">
  <ResponseField name="error" type="string">
    Código de error.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Descripción del error.
  </ResponseField>
</ResponseSchema>
