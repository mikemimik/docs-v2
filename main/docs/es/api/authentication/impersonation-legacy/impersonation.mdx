<div id="endpoint">
  ## Endpoint
</div>

`POST /users/{user_id}/impersonate`

Usa este endpoint para obtener una URL de suplantación con la que iniciar sesión como otro usuario. Es útil para la resolución de problemas.

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo puede usarse con credenciales de **Global Client**.
* Para distinguir entre inicios de sesión reales e inicios de sesión por suplantación, el perfil del usuario suplantado contendrá las propiedades adicionales `impersonated` e `impersonator`. Por ejemplo: `"impersonated": true, "impersonator": {"user_id": "auth0|...", "email": "admin@example.com"}`.
* Para una aplicación web tradicional, debes configurar `additionalParameters`: establece `response_type` en `code`, `callback_url` como la URL de devolución de llamada a la que Auth0 redirigirá con el código de autorización, y `scope` como los claims de JWT que quieres incluir en el JWT.
