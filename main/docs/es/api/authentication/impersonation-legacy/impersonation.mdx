<div id="endpoint">
  ## Endpoint
</div>

`POST /users/{user_id}/impersonate`

Usa este endpoint para obtener una URL de suplantación con la que iniciar sesión como otro usuario. Resulta útil para solucionar problemas.

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo se puede usar con credenciales de **Global Client**.
* Para distinguir entre inicios de sesión reales e inicios de sesión por suplantación, el perfil del usuario suplantado incluirá propiedades adicionales de `impersonated` e `impersonator`. Por ejemplo: `"impersonated": true, "impersonator": {"user_id": "auth0|...", "email": "admin@example.com"}`.
* En una aplicación web tradicional, debe configurar `additionalParameters`: establezca `response_type` en `code`, `callback_url` en la URL de devolución de llamada a la que Auth0 redirigirá con el código de autorización, y `scope` en los claims de JWT que desea incluir en el JWT.
