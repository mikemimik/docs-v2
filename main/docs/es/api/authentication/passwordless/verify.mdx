title: "Verify"

---

<div id="endpoint">
  ## Endpoint
</div>

`POST /passwordless/verify`

<Note>
  Esta funcionalidad está deshabilitada de forma predeterminada para los tenants nuevos desde el 8 de junio de 2017. Consulta [Tipo de concesión de aplicaciones](/applications/concepts/application-grant-types) para obtener más información.
</Note>

Una vez que tengas un código de verificación, usa este endpoint para iniciar la sesión del usuario con su número de teléfono/correo electrónico y el código de verificación. Esta es una autenticación activa, por lo que el usuario debe introducir el código en tu aplicación.

<div id="remarks">
  ### Observaciones
</div>

* El valor de `scope` `profile` solicita acceso a los claims predeterminados del perfil del usuario final, que son: `name`, `family_name`, `given_name`, `middle_name`, `nickname`, `preferred_username`, `profile`, `picture`, `website`, `gender`, `birthdate`, `zoneinfo`, `locale` y `updated_at`.
* El valor de `scope` `email` solicita acceso a los claims `email` y `email_verified`.
* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si usa la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js).

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="grant_type" type="string" required>
  Tipo de concesión; debe ser `password`.
</ParamField>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu aplicación.
</ParamField>

<ParamField body="connection" type="string" required>
  Usa `sms` o `email` (debe coincidir con [POST /passwordless/start](#get-code-or-link)).

  Valores permitidos: `email`, `sms`
</ParamField>

<ParamField body="username" type="string" required>
  El número de teléfono del usuario si `connection=sms`, o su correo electrónico si `connection=email`.
</ParamField>

<ParamField body="password" type="string" required>
  El código de verificación del usuario.
</ParamField>

<ParamField body="scope" type="string">
  Usa `openid` para obtener un token de ID, o `openid profile email` para incluir información del perfil del usuario en el token de ID.
</ParamField>

<ParamField body="redirect_uri" type="string" required>
  URL de devolución de llamada registrada en las Allowed Callback URLs de tu aplicación.
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                       |
| ------ | ------------------------------------------------- |
| 200    | Usuario autenticado correctamente.                |
| 400    | Solicitud incorrecta. Parámetros no válidos.      |
| 401    | No autorizado. Credenciales no válidas.           |
| 403    | Prohibido. Se requiere autenticación del cliente. |
| 500    | Error interno del servidor.                       |
