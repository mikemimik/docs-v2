<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/token`

Verifica la autenticación multifactor (MFA) mediante una contraseña de un solo uso (OTP). Para verificar la MFA con una OTP, solicita al usuario el código OTP y luego realiza una solicitud al endpoint /oauth/token. La solicitud debe incluir el código OTP, el mfa&#95;token que recibiste (del error mfa&#95;required) y el grant&#95;type configurado como http://auth0.com/oauth/grant-type/mfa-otp. La respuesta es la misma que para los tipos de concesión password o http://auth0.com/oauth/grant-type/password-realm. Obtén más información en [Asociar autenticadores OTP](/mfa/guides/mfa-api/otp).

<div id="headers">
  ## Encabezados
</div>

<ParamField header="DPoP" type="string">
  Una prueba DPoP para la solicitud. Es opcional y solo se requiere si tu aplicación utiliza Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="grant_type" type="string" required>
  Indica el flujo que estás usando. Para OTP MFA, usa `http://auth0.com/oauth/grant-type/mfa-otp`.
</ParamField>

<ParamField body="client_id" type="string">
  El Client ID de tu aplicación.
</ParamField>

<ParamField body="client_secret" type="string">
  El Client Secret de tu aplicación.
</ParamField>

<ParamField body="mfa_token" type="string" required>
  El `mfa_token` que recibiste en el error `mfa_required`.
</ParamField>

<ParamField body="otp" type="string" required>
  El código OTP proporcionado por el usuario.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT que contiene una aserción firmada con las credenciales de tu aplicación.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  El valor es `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`.
</ParamField>

<div id="response-schema">
  ## Esquema de la respuesta
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    El token de acceso de la sesión autenticada.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    El tipo de token emitido.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La vigencia del token de acceso en segundos.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                                       |
| ------ | --------------------------------------------------------------------------------- |
| 200    | Verificación del OTP exitosa.                                                     |
| 400    | Solicitud no válida, por ejemplo, por falta de parámetros o por un OTP no válido. |
