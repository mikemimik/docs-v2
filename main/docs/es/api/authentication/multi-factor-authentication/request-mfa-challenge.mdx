<div id="endpoint">
  ## Endpoint
</div>

`POST /mfa/challenge`

Los endpoints de la API de autenticación multifactor (MFA) le permiten exigir MFA cuando los usuarios interactúan con [los endpoints de token](#get-token), así como registrar y administrar los autenticadores de los usuarios.

Primero, solicite un desafío según los tipos de desafío compatibles con la aplicación y el usuario. Si sabe que se admite una contraseña de un solo uso (OTP), puede omitir la solicitud del desafío.

A continuación, verifique la autenticación multifactor mediante el endpoint `/oauth/token` y el tipo de desafío especificado: una contraseña de un solo uso (OTP), un código de recuperación o un desafío fuera de banda (OOB).

Para obtener más información, lea:

* [Autenticación multifactor y contraseña del propietario del recurso](https://auth0.com/docs/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa)
* [API de autenticación multifactor](https://auth0.com/docs/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api)
* [Autenticación multifactor en Auth0](https://auth0.com/docs/secure/multi-factor-authentication)

<div id="challenge-request">
  ## Solicitud de desafío
</div>

Solicita un desafío para la autenticación multifactor (MFA) según los tipos de desafío que admiten la aplicación y el usuario.

El `challenge_type` indica cómo recibirá el usuario el desafío y cómo demostrará que lo posee. Los tipos de desafío compatibles incluyen:

* `otp`: para contraseña de un solo uso (OTP)
* `oob`: para mensajes por SMS/voz o fuera de banda (OOB)

Si el usuario admite OTP y no quieres solicitar un factor distinto, puedes omitir la solicitud de desafío y [verificar la autenticación multifactor con una contraseña de un solo uso](#verify-with-one-time-password-otp-).

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint no admite el registro; el usuario debe estar registrado con el método preferido antes de solicitar un desafío.
* Auth0 elige el tipo de desafío en función de los tipos compatibles de la aplicación y de los tipos con los que el usuario está registrado.
* Se devuelve un error `unsupported_challenge_type` si su aplicación no admite ninguno de los tipos de desafío con los que el usuario está registrado.
* Se devuelve un error `unsupported_challenge_type` si el usuario no está registrado.
* Si el usuario no está registrado, recibirá un error `association_required`, lo que indica que el usuario debe registrarse para usar MFA. Lea [Agregar un autenticador](/multi-factor-authentication/add-an-authenticator) a continuación para saber cómo proceder.

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="mfa_token" type="string" required>
  El token recibido del error mfa&#95;required.
</ParamField>

<ParamField body="client_id" type="string" required>
  El ID de cliente de tu aplicación.
</ParamField>

<ParamField body="client_secret" type="string">
  El secreto de cliente de tu aplicación.
</ParamField>

<ParamField body="challenge_type" type="string">
  Una lista de tipos de desafío que acepta tu aplicación, separados por espacios en blanco.

  Valores permitidos: `oob`, `otp`
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT que contiene una assertion firmada con las credenciales de tu aplicación.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  El valor es urn:ietf:params:oauth:client-assertion-type:jwt-bearer.
</ParamField>

<ParamField body="authenticator_id" type="string">
  El ID del autenticador al que se enviará el desafío.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema statusCode="200">
  <ResponseField name="challenge_type" type="string">
    El tipo de desafío que emite este autenticador. Valores posibles: `otp`, `oob`.
  </ResponseField>

  <ResponseField name="oob_code" type="string">
    El código del desafío fuera de banda. Solo aparece cuando `challenge_type` es `oob`.
  </ResponseField>

  <ResponseField name="binding_method" type="string">
    El método utilizado para el Binding. Solo aparece cuando corresponde. Valores posibles: `prompt`.
  </ResponseField>
</ResponseSchema>

<ResponseSchema statusCode="400">
  <ResponseField name="error" type="string">
    Código de error.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Descripción detallada del error.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                                                 |
| ------ | ------------------------------------------------------------------------------------------- |
| 200    | Solicitud de desafío realizada correctamente                                                |
| 400    | Solicitud no válida, por ejemplo, un tipo de desafío no admitido o la falta de inscripción. |
