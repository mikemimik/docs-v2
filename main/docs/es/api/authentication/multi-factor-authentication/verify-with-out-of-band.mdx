<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/token`

Verifica la autenticación multifactor (MFA) mediante un desafío fuera de banda (OOB), ya sea por notificación push, SMS o voz. Para verificar la MFA mediante un desafío OOB, tu aplicación debe hacer una solicitud a `/oauth/token` con `grant_type=http://auth0.com/oauth/grant-type/mfa-oob`. Incluye el `oob_code` que recibiste en la respuesta del desafío, así como el `mfa_token` que recibiste como parte del error `mfa_required`.

La respuesta a esta solicitud depende del estado de la verificación del desafío subyacente:

* Si el desafío fue aceptado y verificado, será la misma que para los tipos de concesión `password` o `http://auth0.com/oauth/grant-type/password-realm`.
* Si el desafío fue rechazado, obtendrás un error `invalid_grant`, lo que significa que el usuario rechazó el desafío. En este punto debes dejar de consultar, ya que esta respuesta es definitiva.
* Si la verificación del desafío sigue pendiente (es decir, no fue aceptado ni rechazado), obtendrás un error `authorization_pending`, lo que significa que debes reintentar la misma solicitud unos segundos después. Si haces solicitudes con demasiada frecuencia, obtendrás un error `slow_down`.

Cuando la respuesta del desafío incluye un `binding_method: prompt`, tu aplicación debe pedirle al usuario el `binding_code` y enviarlo como parte de la solicitud. El `binding_code` suele ser un número de 6 dígitos (similar a un OTP) incluido en el desafío. No se necesita ningún `binding_code` si la respuesta del desafío no incluyó un `binding_method`. En este caso, la respuesta será inmediata; recibirás un `invalid_grant` o un `access_token` como respuesta.

<div id="headers">
  ## Encabezados
</div>

<ParamField header="DPoP" type="string">
  Una prueba DPoP de la solicitud. Es opcional y solo se requiere si tu aplicación usa Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="grant_type" type="string" required>
  Indica el flujo que estás usando. Para OTP MFA, usa `http://auth0.com/oauth/grant-type/mfa-oob`.

  Valores permitidos: `http://auth0.com/oauth/grant-type/mfa-oob`
</ParamField>

<ParamField body="client_id" type="string" required>
  El ID de cliente de tu aplicación.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT que contiene una aserción firmada con las credenciales de tu aplicación.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  El valor es `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`.

  Valores permitidos: `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`
</ParamField>

<ParamField body="client_secret" type="string">
  El secreto de cliente de tu aplicación.
</ParamField>

<ParamField body="mfa_token" type="string" required>
  El `mfa_token` que recibiste en el error `mfa_required`.
</ParamField>

<ParamField body="oob_code" type="string" required>
  El código OOB recibido en la solicitud de desafío.
</ParamField>

<ParamField body="binding_code" type="string">
  Código que se usa para vincular el canal secundario con el canal principal que estás usando para autenticarte.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    El token de acceso devuelto tras una verificación exitosa.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    El tipo de token emitido.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La duración del token de acceso en segundos.
  </ResponseField>

  <ResponseField name="error" type="string">
    Código de error devuelto cuando la verificación está pendiente o ha fallado.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Descripción del error.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                 |
| ------ | ----------------------------------------------------------- |
| 200    | Respuesta satisfactoria para la verificación OOB.           |
| 400    | Solicitud incorrecta por parámetros faltantes o no válidos. |
| 401    | No autorizado; `mfa_token` u `oob_code` no válidos.         |
