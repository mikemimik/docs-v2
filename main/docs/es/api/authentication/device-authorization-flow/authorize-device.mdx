<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/device/code`

Este flujo está diseñado para que los dispositivos con limitación de entrada accedan a una API. Usa este endpoint para obtener un código de dispositivo que permita al usuario autorizarlo.

<div id="request-example">
  ## Ejemplo de solicitud
</div>

```http
POST https://{yourDomain}/oauth/device/code
Content-Type: application/x-www-form-urlencoded

client_id=${account.clientId}&scope=SCOPE&audience=API_IDENTIFIER
```

<div id="response-values">
  ### Valores de respuesta
</div>

| Valor                       | Descripción                                                                   |
| :-------------------------- | :---------------------------------------------------------------------------- |
| `device_code`               | El código único del dispositivo.                                              |
| `user_code`                 | El código que el usuario debe introducir para autorizar el dispositivo.       |
| `verification_uri`          | La URL que el usuario debe visitar para autorizar el dispositivo.             |
| `verification_uri_complete` | La URL completa, incluido el código de usuario, para acceder más fácilmente.  |
| `expires_in`                | El tiempo de validez de los códigos de dispositivo y de usuario, en segundos. |
| `interval`                  | El intervalo de consulta, en segundos, para solicitar un token.               |

<div id="remarks">
  ### Notas
</div>

* Incluye `offline_access` en el `scope` para obtener un Refresh Token.
* Usa el código de dispositivo devuelto para solicitar un token de acceso al endpoint de token.

<div id="token-request-example">
  ### Ejemplo de solicitud de token
</div>

```http
POST https://{yourDomain}/oauth/token
Content-Type: application/x-www-form-urlencoded

client_id=${account.clientId}&device_code=YOUR_DEVICE_CODE&grant_type=urn:ietf:params:oauth:grant-type:device_code
```

<div id="responses">
  ### Respuestas
</div>

<div id="200">
  #### 200
</div>

Una solicitud exitosa devuelve un token de acceso.

```json
HTTP/1.1 200 OK
Content-Type: application/json
{
   "access_token": "eyJz93a...k4laUWw",
   "id_token": "eyJ...0NE",
   "refresh_token": "eyJ...MoQ",
   "expires_in": 86400,
   "token_type": "Bearer"
}
```

<div id="error-responses">
  ### Respuestas de error
</div>

```json
HTTP/1.1 403 Forbidden
Content-Type: application/json
{ "error": "authorization_pending", "error_description": "User has yet to authorize device code." }
```

```json
HTTP/1.1 429 Too Many Requests
Content-Type: application/json
{ "error": "slow_down", "error_description": "You are polling faster than the specified interval of 5 seconds." }
```

```json
HTTP/1.1 403 Forbidden
Content-Type: application/json
{ "error": "access_denied", "error_description": "User cancelled the confirmation prompt." }
```

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="client_id" type="string" required>
  El ID de tu aplicación.
</ParamField>

<ParamField body="scope" type="string">
  Los permisos que quieres solicitar para la autorización.
</ParamField>

<ParamField body="audience" type="string">
  El identificador único de la API de destino a la que quieres acceder.
</ParamField>

<ParamField body="resource" type="string">
  El identificador de la API de destino (servidor de recursos) a la que quieres acceder. Debe coincidir con un identificador de API registrado en tu tenant de Auth0. Se utiliza como alternativa a `audience` cuando el [Perfil de compatibilidad del parámetro Resource](https://auth0.com/docs/get-started/tenant-settings#settings-advanced) del tenant está configurado como `compatibility`.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="device_code" type="string">
    El código único del dispositivo. Cuando el usuario accede a `verification_uri` desde un dispositivo con navegador, este código se vincula a su sesión.
  </ResponseField>

  <ResponseField name="user_code" type="string">
    El código que el usuario debe introducir en `verification_uri`.
  </ResponseField>

  <ResponseField name="verification_uri" type="string">
    La URL que el usuario debe visitar para autorizar el dispositivo.
  </ResponseField>

  <ResponseField name="verification_uri_complete" type="string">
    La URL completa que el usuario debe visitar, incluido el `user_code`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    El tiempo de validez en segundos de `device_code` y `user_code`.
  </ResponseField>

  <ResponseField name="interval" type="integer">
    El intervalo en segundos con el que la aplicación debe consultar si hay un token disponible.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                         |
| ------ | --------------------------------------------------- |
| 200    | Devuelve los códigos del dispositivo y del usuario. |
