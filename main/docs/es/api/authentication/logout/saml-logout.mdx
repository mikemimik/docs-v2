title: "Cierre de sesión SAML"
description: "Cerrar la sesión de un usuario en un tenant de Auth0 configurado como proveedor de identidad SAML."
---

<div id="endpoint">
  ## Endpoint
</div>

`POST /samlp/{client_id}/logout`

Utiliza este endpoint para cerrar la sesión de un usuario en un tenant de Auth0 configurado como proveedor de identidad SAML (IdP). El comportamiento del cierre de sesión depende de la configuración del complemento SAML2 Web App de la aplicación en el tenant de Auth0 que actúa como IdP de SAML. Para obtener más información, consulta [Cerrar la sesión de los usuarios en proveedores de identidad SAML](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-saml-idps#configure-slo-when-auth0-is-the-saml-idp).

<div id="remarks">
  ### Observaciones
</div>

* El cuerpo de la solicitud POST debe contener un mensaje SAML `<LogoutRequest>` válido. Para obtener más información, consulte [Assertions and Protocols for the OASIS Security Assertion Markup Language (SAML) V2.0 on Oasis](https://docs.oasis-open.org/security/saml/v2.0/saml-core-2.0-os.pdf).

<div id="path-parameters">
  ## Parámetros de ruta
</div>

<ParamField path="client_id" type="string" required>
  ID de cliente de tu aplicación, configurada con el complemento [complemento SAML2 Web App](https://auth0.com/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon).
</ParamField>

  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                      |
| ------ | ---------------------------------------------------------------- |
| 200    | Cierre de sesión realizado correctamente                         |
| 400    | Solicitud incorrecta, posiblemente por un mensaje SAML no válido |
