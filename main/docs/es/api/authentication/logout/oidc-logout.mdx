title: "Cierre de sesión de OIDC"
description: "Cierra la sesión de un usuario mediante el endpoint de cierre de sesión compatible con OIDC y, opcionalmente, redirígelo a una URI de redirección posterior al cierre de sesión."
---

<div id="endpoint">
  ## Endpoint
</div>

`GET /oidc/logout`

Usa este endpoint para cerrar la sesión de un usuario. Si quieres redirigirlo a una URL específica después del cierre de sesión, establece esa URL en el parámetro `post_logout_redirect_uri`. La URL debe estar incluida en la lista `Allowed Logout URLs` correspondiente:

* Si se incluye el parámetro `id_token_hint`:
  * Cuando se incluye el parámetro `client_id`, el servidor usa la URL del claim `aud` en `id_token_hint` para seleccionar cuál de las `Allowed Logout URLs` usar de la aplicación especificada por `client_id`.
  * Cuando NO se incluye el parámetro `client_id`, el servidor usa la URL del claim `aud` en `id_token_hint` para seleccionar cuál de las `Allowed Logout URLs` usar en el nivel del tenant.
* Si no se incluye el parámetro `id_token_hint`:
  * Si se incluye el parámetro `client_id`, la URL `post_logout_redirect_uri` debe figurar en el conjunto `Allowed Logout URLs` a nivel de la aplicación.
  * Si NO se incluye el parámetro `client_id`, la URL `post_logout_redirect_uri` debe figurar en el conjunto `Allowed Logout URLs` a nivel de tenant.
  * Si se incluye el parámetro `client_id` y NO se establece la URL `post_logout_redirect_uri`, el servidor redirige al usuario a la primera URL configurada en `Allowed Logout URLs` en Auth0 Dashboard.

Para obtener más información, consulta [Log Users Out of Auth0 with OIDC Endpoint](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-auth0).

<div id="remarks">
  ### Observaciones
</div>

* Cerrar la sesión del usuario en su proveedor de identidad social no es una práctica habitual, así que ten en cuenta la experiencia del usuario antes de usar el parámetro de cadena de consulta `federated` con proveedores de identidad social.
* Si proporcionas tanto `id_token_hint` como `logout_hint`, el valor de `logout_hint` debe coincidir con el claim `sid` de `id_token_hint`.
* Si proporcionas tanto `id_token_hint` como `client_id`, el valor de `client_id` debe coincidir con el claim `aud` de `id_token_hint`.
* Si no se proporciona `id_token_hint`, se solicitará el consentimiento del usuario, a menos que se proporcione un `logout_hint` que coincida con el ID de sesión del usuario.
* El método HTTP `POST` también se admite para esta solicitud. Al usar `POST`, los parámetros de la solicitud deben proporcionarse en el cuerpo de la solicitud como parámetros de formulario, en lugar de en la cadena de consulta. El parámetro `federated` requiere un valor de `true` o `false`.
* Esto cumple con la [Especificación de cierre de sesión iniciada por RP de OIDC](https://openid.net/specs/openid-connect-rpinitiated-1_0.html).

<div id="query-parameters">
  ## Parámetros de consulta
</div>

<ParamField query="id_token_hint" type="string">
  &#91;Recomendado&#93; Token de ID emitido previamente para el usuario. Se utiliza para indicar qué usuario debe cerrar sesión.
</ParamField>

<ParamField query="logout_hint" type="string">
  Valor `sid` (ID de sesión) opcional para indicar qué usuario debe cerrar sesión. Debe proporcionarse cuando `id_token_hint` no esté disponible.
</ParamField>

<ParamField query="post_logout_redirect_uri" type="string">
  URL a la que se redirige al usuario después del cierre de sesión.
</ParamField>

<ParamField query="client_id" type="string">
  El `client_id` de tu aplicación.
</ParamField>

<ParamField query="federated" type="string">
  Añade este parámetro de consulta para cerrar la sesión del usuario en su proveedor de identidad: `https://YOUR_DOMAIN/oidc/logout?federated`.
</ParamField>

<ParamField query="state" type="string">
  Un valor opaco que la aplicación añade a la solicitud inicial y que el servidor de autorización incluye al redirigir de vuelta a la `post_logout_redirect_uri`.
</ParamField>

<ParamField query="ui_locales" type="string">
  Lista de locales delimitada por espacios que se usa para restringir la lista de idiomas de la solicitud. El primer locale de la lista debe coincidir con el locale habilitado en tu tenant.
</ParamField>
