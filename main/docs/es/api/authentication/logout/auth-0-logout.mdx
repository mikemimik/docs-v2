---
title: "Cierre de sesión de Auth0"
description: "Cerrar la sesión de un usuario y, opcionalmente, redirigirlo a una URL especificada después de cerrar sesión."
---

<div id="endpoint">
  ## Endpoint
</div>

`GET /v2/logout`

Usa este endpoint para cerrar la sesión de un usuario. Si quieres redirigir al usuario a una URL específica después del cierre de sesión, establece esa URL en el parámetro `returnTo`. La URL debe incluirse en la lista correspondiente de `Allowed Logout URLs`:

* Si se incluye el parámetro `client_id`, la URL de `returnTo` debe figurar en el conjunto de `Allowed Logout URLs` a nivel de la aplicación. Para obtener más información, consulta [Cerrar la sesión de los usuarios de las aplicaciones](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-applications).
* Si NO se incluye el parámetro `client_id`, la URL de `returnTo` debe figurar en el conjunto de `Allowed Logout URLs` a nivel del tenant. Para obtener más información, consulta [Cerrar la sesión de los usuarios de Auth0](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-auth0).
* Si se incluye el parámetro `client_id` y NO se establece la URL de `returnTo`, el servidor redirige al usuario a la primera URL de `Allowed Logout URLs` configurada en el Dashboard. Para obtener más información, consulta [Cerrar la sesión de los usuarios de las aplicaciones](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-applications).

<div id="remarks">
  ### Observaciones
</div>

* Cerrar la sesión del usuario en su proveedor de identidad no es una práctica habitual, así que piensa en la experiencia del usuario antes de usar el parámetro de la cadena de consulta `federated`.
* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si estás usando la versión 7 de auth0.js, consulta esta [guía de referencia](https://auth0.com/docs/libraries/auth0js).

<div id="query-parameters">
  ## Parámetros de consulta
</div>

<ParamField query="client_id" type="string">
  El `client_id` de su aplicación.
</ParamField>

<ParamField query="returnTo" type="string" required>
  URL para redirigir al usuario después del cierre de sesión.
</ParamField>

<ParamField query="federated" type="string">
  Añada este parámetro de consulta a la URL de cierre de sesión para cerrar la sesión del usuario en su proveedor de identidad.
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                             |
| ------ | ----------------------------------------------------------------------- |
| 200    | Se cerró la sesión correctamente                                        |
| 400    | Solicitud incorrecta, posiblemente por falta de parámetros obligatorios |
