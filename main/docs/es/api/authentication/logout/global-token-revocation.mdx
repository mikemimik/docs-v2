---
title: "Revocación global de tokens"
description: "Revoca las cookies de sesión y los tokens de actualización de un usuario en todas las aplicaciones a través del Universal Logout de Okta."
---

<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/global-token-revocation/connection/{connection_name}`

Usa este endpoint con [Okta Workforce Identity Cloud Universal Logout](https://developer.okta.com/docs/guides/oin-universal-logout-overview/) para cerrar la sesión de los usuarios en tus aplicaciones. Una solicitud a este endpoint revoca las cookies de sesión y los tokens de actualización, pero no los tokens de acceso. Para obtener más información, consulta [Universal Logout](https://auth0.com/docs/authenticate/login/logout/universal-logout). La solicitud debe autenticarse antes de revocar las sesiones de los usuarios. Consulta [Autenticación del endpoint](https://developer.okta.com/docs/guides/oin-universal-logout-overview/#endpoint-authentication).

<div id="path-parameters">
  ## Parámetros de la ruta
</div>

<ParamField path="connection_name" type="string" required>
  El nombre de la conexión cuya sesión de usuario quieres revocar.
</ParamField>

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="subject" type="object">
  <Expandable title="propiedades">
    <ParamField body="format" type="string" required />

    <ParamField body="iss" type="string" required>
      La URL del emisor (normalmente, la URL de tu servidor de autorización).
    </ParamField>

    <ParamField body="sub" type="string" required>
      El identificador del sujeto (normalmente, el identificador de usuario).
    </ParamField>
  </Expandable>
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                                                                  |
| ------ | ------------------------------------------------------------------------------------------------------------ |
| 200    | Las cookies de sesión y los tokens de actualización se revocaron correctamente.                              |
| 400    | Solicitud incorrecta, posiblemente debido a un formato de `subject` no válido o a la falta de autenticación. |
| 401    | No autorizado; la autenticación falló.                                                                       |
| 500    | Error interno del servidor.                                                                                  |
