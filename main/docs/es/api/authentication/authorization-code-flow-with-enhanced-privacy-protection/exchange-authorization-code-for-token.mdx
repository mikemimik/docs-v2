<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/token`

Cuando se redirija a los usuarios de vuelta a tu callback, debes hacer una llamada `POST` al endpoint `oauth/token` para intercambiar un código de autorización por un token de acceso y/o un token de ID.

<div id="remarks">
  ### Observaciones
</div>

Para hacer una llamada al endpoint `/oauth/token`, debes:

* Establecer el tipo de contenido de la solicitud en `application/x-www-form-urlencoded`
* Usar `strings` para todos los parámetros enviados
* Incluir en la solicitud un parámetro adicional para la autenticación de la aplicación (por ejemplo, `client_secret`; o `client_assertion` y `client_assertion_type` para JSON Web Token Client Authentication; o bien pasar los encabezados `client-certificate` y `client-certificate-ca-verified` cuando se usa Mutual TLS).

<div id="headers">
  ## Encabezados
</div>

<ParamField header="DPoP" type="string">
  Una prueba DPoP para la solicitud. Es opcional y solo se requiere si su aplicación utiliza Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="grant_type" type="string" required>
  Indica el flujo; usa `authorization_code` con un código de autorización válido.
</ParamField>

<ParamField body="code" type="string">
  El código de autorización de la llamada inicial a `/authorize`.
</ParamField>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu aplicación. Obligatorio.
</ParamField>

<ParamField body="request_uri" type="string">
  Solo es obligatorio si se estableció en el endpoint `GET` `/oauth/par`.
</ParamField>

<ParamField body="code_verifier" type="string">
  Clave aleatoria criptográficamente segura que se usa para generar el `code_challenge`. Se recomienda si se proporcionó `code_challenge`.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    El token de acceso para acceder a recursos protegidos.
  </ResponseField>

  <ResponseField name="refresh_token" type="string">
    Token que se utiliza para obtener nuevos tokens de acceso.
  </ResponseField>

  <ResponseField name="id_token" type="string">
    El token de ID.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    El tipo de token. Por lo general, `Bearer`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La duración del token de acceso en segundos.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                |
| ------ | ------------------------------------------ |
| 200    | Respuesta exitosa que contiene los tokens. |
