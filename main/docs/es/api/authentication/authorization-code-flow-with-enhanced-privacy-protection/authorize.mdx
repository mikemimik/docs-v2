---
title: "Autorizar"
description: "Redirige al usuario al endpoint /authorize tras una solicitud PAR correcta para completar el flujo de código de autorización."
---

<div id="endpoint">
  ## Endpoint
</div>

`GET /authorize`

Después de llamar al endpoint `/oauth/par`, redirige al usuario final al endpoint `/authorize` con una llamada `GET`.

<Note>
  El endpoint `/authorize` responderá según los parámetros enviados al endpoint `/oauth/par`. Si solicitas un `response_type`, deberías recibir un código de autorización para usarlo en el endpoint `/oauth/token`.
</Note>

<div id="query-parameters">
  ## Parámetros de consulta
</div>

<ParamField query="client_id" type="string" required>
  El `client_id` de tu aplicación. Obligatorio.
</ParamField>

<ParamField query="request_uri" type="string" required>
  El valor de `request_uri` que se recibió del endpoint `/oauth/par`. Obligatorio.
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                            |
| ------ | -------------------------------------- |
| 302    | Redirige al Redirect URI especificado. |
