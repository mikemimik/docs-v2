<div id="endpoint">
  ## Endpoint
</div>

`GET /authorize`

Usa el repositorio de usuarios de Auth0 o tu propia base de datos para almacenar y gestionar credenciales de nombre de usuario y contraseña. Si tienes tu propia base de datos de usuarios, puedes usarla como proveedor de identidad en Auth0 para autenticar a los usuarios. Cuando haces una llamada `GET` al endpoint `/authorize` para autenticación basada en navegador (pasiva), se devuelve una redirección `302` a la [Auth0 Login Page](http://manage.auth0.com/login), donde se mostrará el Login Widget para que el usuario pueda iniciar sesión con correo electrónico y contraseña.

<Note>
  La autenticación pasiva se realiza a través del navegador y se inicia desde la [Auth0 Login Page](http://manage.auth0.com/login), donde el usuario introduce sus credenciales. No requiere código personalizado ni procesamiento en el backend, como ocurre con la autenticación activa.
</Note>

<div id="remarks">
  ### Observaciones
</div>

* Si `response_type=token`, después de que el usuario se autentique, se lo redirigirá a la `URL de devolución de llamada` de su aplicación, con el Access Token y el token de ID en `location.hash`. Esto se utiliza en aplicaciones web de página única y también en SDKs móviles nativos.
* La principal diferencia entre la autenticación pasiva y la autenticación activa es que la primera ocurre en el navegador a través de la [Auth0 Login Page](http://manage.auth0.com/login), mientras que la segunda puede invocarse desde cualquier lugar (un script, de servidor a servidor, etc.).
* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si usa la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js/v7).

<div id="query-parameters">
  ## Parámetros de consulta
</div>

<ParamField query="response_type" type="string" required>
  Especifica el tipo de token. Use `code` para los flujos del lado del servidor y `token` para los flujos del lado de la aplicación.

  Valores permitidos: `code`, `token`
</ParamField>

<ParamField query="client_id" type="string" required>
  El `client_id` de su aplicación.
</ParamField>

<ParamField query="connection" type="string">
  El nombre de la conexión configurada para su aplicación. Si es null, redirigirá a la [Auth0 Login Page](http://manage.auth0.com/login) y mostrará el Login Widget usando la primera conexión de base de datos.
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  La URL a la que Auth0 redirigirá el navegador una vez que el usuario haya concedido la autorización. Especifique el `redirect_uri` en [Settings de la aplicación](https://manage.auth0.com/dashboard/#/applications).
</ParamField>

<ParamField query="scope" type="string">
  Scopes de OIDC y scopes de API personalizadas. Por ejemplo: `openid read:timesheets`. Incluya `offline_access` para obtener un token de actualización.
</ParamField>

<ParamField query="state" type="string">
  Un valor opaco que la aplicación agrega a la solicitud inicial y que el servidor de autorización incluye al redirigir de vuelta a la aplicación. La aplicación debe usar este valor para evitar ataques CSRF.
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                 |
| ------ | ------------------------------------------- |
| 302    | Redirige a la página de inicio de sesión    |
| 400    | Solicitud incorrecta: parámetros no válidos |
| 500    | Error interno del servidor                  |
