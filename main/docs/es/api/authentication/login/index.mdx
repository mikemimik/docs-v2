<div id="endpoint">
  ## Endpoint
</div>

`GET /authorize`

<div id="social">
  ## Social
</div>

Puedes conectar tu servicio de Auth0 a un proveedor de identidad social y permitir que tus usuarios inicien sesión en tu aplicación con Facebook, Google, Apple u otros proveedores compatibles. Para obtener más información sobre los proveedores compatibles, visita [Marketplace](https://marketplace.auth0.com/features/social-connections).

Para autenticar usuarios con un proveedor social, haz una llamada `GET` al endpoint `/authorize`. Esto devolverá una redirección `302` al proveedor social especificado en el parámetro `connection`.

<Note>
  Las conexiones sociales solo admiten autenticación basada en navegador (pasiva), ya que la mayoría de los proveedores sociales no permiten introducir un nombre de usuario y una contraseña en aplicaciones que no les pertenecen. Por lo tanto, se redirigirá al usuario a la página de inicio de sesión del proveedor.
</Note>

<div id="remarks">
  ### Observaciones
</div>

* Si `response_type=token`, después de que el usuario se autentique en el proveedor, se lo redirigirá a la `URL de devolución de llamada` de su aplicación, pasando el token de acceso y el token de ID en `location.hash`. Esto se usa en las aplicaciones de página única y también en los SDK móviles nativos.

* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si usa la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js).

<div id="query-parameters">
  ## Parámetros de consulta
</div>

<ParamField query="response_type" type="string" required>
  Especifica el tipo de token (`code` o `token`)

  Valores permitidos: `code`, `token`
</ParamField>

<ParamField query="client_id" type="string" required>
  El client&#95;id de tu aplicación
</ParamField>

<ParamField query="connection" type="string">
  El nombre de un proveedor de identidad social configurado para tu aplicación
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  La URL a la que Auth0 redirigirá tras la autorización
</ParamField>

<ParamField query="state" type="string">
  Valor opaco para evitar ataques CSRF
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta

| ------ | ----------------------------------------- |
| 302    | Redirige al proveedor de identidad social |
