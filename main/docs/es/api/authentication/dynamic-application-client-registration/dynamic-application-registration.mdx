<div id="endpoint">
  ## Endpoint
</div>

`POST /oidc/register`

Con un nombre y la URL de devolución de llamada necesaria, puedes registrar dinámicamente un cliente en Auth0. No se necesita ningún token para esta solicitud.

<div id="body-parameters">
  ## Parámetros del Body
</div>

<ParamField body="client_name" type="string">
  El nombre del cliente dinámico que se creará.
</ParamField>

<ParamField body="grant_types" type="array">
  Lista de tipos de concesión que el cliente puede usar. Si no se proporciona, se aplicará de manera predeterminada el tipo de concesión `authorization_code` para las <a href="https://auth0.com/docs/get-started/applications/third-party-applications">aplicaciones de terceros</a> con controles de seguridad reforzados.
</ParamField>

<ParamField body="redirect_uris" type="array" required>
  Un arreglo de URL que Auth0 considerará válidas para redirigir al final de un flujo de autenticación.
</ParamField>

<ParamField body="token_endpoint_auth_method" type="string">
  Método utilizado para la autenticación en el endpoint de token.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="client_name" type="string">
    El nombre del cliente recién registrado.
  </ResponseField>

  <ResponseField name="client_id" type="string">
    El identificador único del cliente.
  </ResponseField>

  <ResponseField name="client_secret" type="string">
    El secreto de cliente.
  </ResponseField>

  <ResponseField name="client_secret_expires_at" type="integer">
    La fecha de expiración del secreto de cliente.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                           |
| ------ | ----------------------------------------------------- |
| 200    | Respuesta satisfactoria con los detalles del cliente. |
