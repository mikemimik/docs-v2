<div id="endpoint">
  ## Endpoint
</div>

`POST /oidc/register`

Con un nombre y la URL de devolución de llamada requerida, puedes registrar un cliente dinámicamente en Auth0. Esta solicitud no requiere ningún token.

<div id="body-parameters">
  ## Parámetros del Body
</div>

<ParamField body="client_name" type="string">
  El nombre del cliente dinámico que se va a crear.
</ParamField>

<ParamField body="grant_types" type="array">
  Lista de tipos de concesión que el cliente puede usar. Si no se proporciona, el tipo de concesión `authorization_code` se aplicará de forma predeterminada a las <a href="https://auth0.com/docs/get-started/applications/third-party-applications">aplicaciones de terceros</a> con controles de seguridad reforzados.
</ParamField>

<ParamField body="redirect_uris" type="array" required>
  Un arreglo de URL que Auth0 considerará válidas al final de un flujo de autenticación.
</ParamField>

<ParamField body="token_endpoint_auth_method" type="string">
  Método utilizado para la autenticación en el endpoint de token.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="client_name" type="string">
    El nombre del cliente recién registrado.
  </ResponseField>

  <ResponseField name="client_id" type="string">
    El identificador único del cliente.
  </ResponseField>

  <ResponseField name="client_secret" type="string">
    El secreto de cliente.
  </ResponseField>

  <ResponseField name="client_secret_expires_at" type="integer">
    El momento de expiración del secreto de cliente.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                     |
| ------ | ----------------------------------------------- |
| 200    | Respuesta exitosa con los detalles del cliente. |
