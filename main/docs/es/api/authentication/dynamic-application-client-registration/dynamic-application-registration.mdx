<div id="endpoint">
  ## Endpoint
</div>

`POST /oidc/register`

Con un nombre y la URL de devolución de llamada necesaria, puedes registrar dinámicamente un cliente en Auth0. No se necesita ningún token para esta solicitud.

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="client_name" type="string">
  El nombre del cliente dinámico que se va a crear.
</ParamField>

<ParamField body="grant_types" type="array">
  Lista de tipos de concesión que el cliente puede usar. Si no se proporciona, se aplicará de forma predeterminada el tipo de concesión `authorization_code` a las <a href="https://auth0.com/docs/get-started/applications/third-party-applications">Aplicaciones de terceros</a> con controles de seguridad reforzados.
</ParamField>

<ParamField body="redirect_uris" type="array" required>
  Una matriz de URL que Auth0 considerará válidas al final de un flujo de autenticación.
</ParamField>

<ParamField body="token_endpoint_auth_method" type="string">
  Método utilizado para la autenticación en el endpoint de token.
</ParamField>

<div id="response-schema">
  ## Esquema de la respuesta
</div>

<ResponseSchema>
  <ResponseField name="client_name" type="string">
    El nombre del cliente recién registrado.
  </ResponseField>

  <ResponseField name="client_id" type="string">
    El identificador único del cliente.
  </ResponseField>

  <ResponseField name="client_secret" type="string">
    El secreto del cliente.
  </ResponseField>

  <ResponseField name="client_secret_expires_at" type="integer">
    La fecha y hora de vencimiento del secreto del cliente.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                   |
| ------ | --------------------------------------------- |
| 200    | Respuesta correcta con los datos del cliente. |
