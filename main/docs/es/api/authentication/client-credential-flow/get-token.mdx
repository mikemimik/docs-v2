<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/token`

Esta es la concesión de OAuth 2.0 que usan los procesos del servidor para acceder a una API. Usa este endpoint para solicitar directamente un token de acceso mediante las credenciales de la aplicación (un Client ID y un Client Secret).

<div id="responses">
  ### Respuestas
</div>

<div id="200">
  #### 200
</div>

Una respuesta exitosa devolverá un token de acceso.

```json
{
  "access_token":"eyJz93a...k4laUWw",
  "token_type":"Bearer",
  "expires_in":86400
}
```

<div id="headers">
  ## Encabezados
</div>

<ParamField header="DPoP" type="string">
  Una prueba DPoP para la solicitud. Es opcional y solo se requiere si su aplicación utiliza Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="grant_type" type="string" required>
  Indica el flujo que estás usando. Para Client Credentials, usa `client_credentials`.

  Valores permitidos: `client_credentials`
</ParamField>

<ParamField body="client_id" type="string" required>
  El Client ID de tu aplicación.
</ParamField>

<ParamField body="client_secret" type="string" required>
  El Client Secret de tu aplicación.
</ParamField>

<ParamField body="audience" type="string" required>
  El identificador único de la API de destino a la que quieres acceder.
</ParamField>

<ParamField body="resource" type="string">
  El identificador de la API de destino (servidor de recursos) a la que quieres acceder. Debe coincidir con un identificador de API registrado en tu tenant de Auth0. Se usa como alternativa a `audience` cuando el [Resource Parameter Compatibility Profile](https://auth0.com/docs/get-started/tenant-settings#settings-advanced) del tenant está configurado como `compatibility`.
</ParamField>

<ParamField body="organization" type="string">
  &#91;Recomendado&#93; La organización o el identificador con el que quieres que se asocie la solicitud. Para obtener más información, consulta [Machine-to-Machine Access for Organizations](https://auth0.com/docs/manage-users/organizations/organizations-for-m2m-applications).
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    El token de acceso.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    El tipo de token. Por lo general, `Bearer`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La vigencia del token de acceso en segundos.
  </ResponseField>
</ResponseSchema>
