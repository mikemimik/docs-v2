<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/ro`

Con las credenciales del usuario, este endpoint autenticará al usuario ante el proveedor y devolverá un objeto JSON con el token de acceso y un token de ID.

<Note>
  Este endpoint forma parte del pipeline de autenticación heredada y ha sido reemplazado por la [Concesión de contraseña](#resource-owner-password). Para obtener más información sobre el pipeline de autenticación más reciente, consulta [Presentación de la autenticación compatible con OIDC](/api-auth/intro).
</Note>

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo funciona con conexiones de base de datos, conexiones passwordless, Active Directory/LDAP, Windows Azure AD y ADFS.
* El scope `profile` solicita acceso a las claims predeterminadas del perfil del usuario final, que incluyen `name`, `family_name`, `given_name` y otras.
* El scope `email` solicita acceso a las claims `email` y `email_verified`.
