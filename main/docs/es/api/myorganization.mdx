</Warning>

My Organization API de Auth0 proporciona una interfaz segura con ámbito de organización que permite a tus clientes empresariales administrar sus propias organizaciones dentro de tu tenant de Auth0. Esta API sirve como base técnica para la administración delegada integrada y las integraciones API-first.

La documentación de My Organization API sigue el [schema OpenAPI v3.1 de My Organization API](https://auth0.com/docs/oas/myorganization/myorganization-api-oas.json). Ten en cuenta que la compatibilidad con el schema OpenAPI v3.1 se encuentra actualmente en Beta.

<Tip>
  <p class="uppercase font-bold">Uso del dominio de Auth0 frente al dominio personalizado</p>

  My Organization API admite el uso de tu dominio canónico de Auth0 o de tu dominio personalizado, pero debes usar siempre el mismo durante todo el proceso, incluido lo siguiente:

  * Obtener un token de acceso
  * Establecer el valor de audience
  * Llamar al endpoint de My Organization API

  Para obtener más información, consulta [Custom Domains](https://auth0.com/docs/customize/custom-domains).
</Tip>

<div id="activate-the-my-organization-api-in-auth0-dashboard">
  ## Activa My Organization API en Auth0 Dashboard
</div>

1. Ve a **[Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis)**.
2. Busca el banner de My Organization API.
3. Selecciona **Activate**.
4. La API aparece en tu lista de Applications &gt; API como My Organization API.

<img src="/docs/images/api/myorganization/My_Organization_API_-_Activate.png" />

Una vez que actives My Organization API:

* La API está deshabilitada para todas las aplicaciones cliente de forma predeterminada.
* Debes otorgar acceso a las aplicaciones y a los roles mediante [autorizaciones de cliente](/docs/es/get-started/applications/application-access-to-apis-client-grants) o políticas de [RBAC](/docs/es/manage-users/access-control/configure-core-rbac/enable-role-based-access-control-for-apis).
* Tus clientes empresariales pueden recuperar los detalles de la organización o configurar proveedores de identidad (IdPs) en nombre de sus propias organizaciones.

De forma predeterminada, My Organization API se activa con las siguientes políticas de acceso de la aplicación a la API:

`require_client_grant` para flujos de usuario
`deny-all` para flujos de cliente (máquina a máquina)

Para que una aplicación acceda a My Organization API en nombre del usuario, debes crear explícitamente una autorización de cliente para esa aplicación, lo que te permite definir los scopes máximos que la aplicación puede solicitar. Como alternativa, puedes cambiar la política de los flujos de acceso de usuario a `allow_all`, lo que permite que cualquier aplicación de tu tenant solicite cualquier scope de My Organization API.

Como My Organization API expone información y operaciones sensibles, Auth0 no recomienda usar `allow_all` para los flujos de acceso de usuario. Debes seguir el principio de mínimo privilegio con My Organization API para garantizar que las aplicaciones solo obtengan acceso a lo que realmente necesitan, minimizando así los posibles riesgos de seguridad.

Los permisos finales otorgados a la aplicación estarán determinados por la intersección entre los scopes permitidos por la política de acceso de la aplicación a la API, los permisos de Role-Based Access Control (RBAC) asignados al usuario final y cualquier consentimiento otorgado por el usuario (si corresponde).

Para obtener más información sobre cómo gestionar las políticas de acceso de la aplicación a la API y sus autorizaciones de cliente asociadas, consulta [Application Access to APIs: Client Grants](https://auth0.com/docs/get-started/applications/application-access-to-apis-client-grants).

<div id="client-application-attributes">
  ## Atributos de la aplicación cliente
</div>

[Crea una aplicación](/docs/es/get-started/auth0-overview/create-applications) en Auth0 para usarla con My Organization API. Una vez creada, ve a **[Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis)** y autoriza My Organization API, incluidos los scopes que quieres que la aplicación pueda realizar.

La aplicación cliente debe proporcionar un objeto de configuración específico (`my_organization_configuration`) que contenga las siguientes propiedades:

| **Propiedad**                                                | **Descripción**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `my_organization_configuration`                              | **Object.** La aplicación debe proporcionar este objeto con la configuración que debe seguir My Organization API. Si la aplicación no define este objeto, My Organization API devolverá un error y rechazará la solicitud.                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `my_organization_configuration.connection_profile_id`        | **Connection Profile ID.** ID del [perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile) usado con la aplicación al utilizar My Organization API. Si no se proporciona, las funciones de My Organization API que requieren un perfil de conexión no funcionarán. Este ID debe hacer referencia a un perfil de conexión válido en el mismo tenant.                                                                                                                                                                                                                                                                                |
| `my_organization_configuration.user_attribute_profile_id`    | **User Attribute Profile ID**. ID del [perfil de atributos de usuario](/docs/es/authenticate/enterprise-connections/user-attribute-profile) usado con la aplicación al utilizar My Organization API. Si no se proporciona, las funciones de My Organization API que requieren un perfil de atributos de usuario no funcionarán. Este ID debe hacer referencia a un perfil de atributos de usuario válido en el mismo tenant.                                                                                                                                                                                                                                                            |
| `my_organization_configuration.allowed_strategies`           | **Array of strings.** Cada string es único y hace referencia a una strategy compatible. Las strategies compatibles —los valores del enum— son las siguientes: `pingfederate`, `ad`, `adfs`, `waad`, `google-apps`, `okta`, `oidc` y `samlp`.                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `my_organization_configuration.connection_deletion_behavior` | **Enum (allow, allow&#95;if&#95;empty).** Describe cómo se comporta My Organization API cuando un usuario final intenta eliminar una conexión a través de My Organization API desde esta aplicación. Los valores y la descripción del enum son los siguientes: <p /> 1. `allow`: Si el usuario tiene el scope correcto, puede eliminar la conexión, lo que provoca que se eliminen todos los usuarios que provienen de esa conexión. <p /> 2.`allow_if_empty`: Si el usuario tiene el scope correcto, solo puede eliminar la conexión si no hay usuarios en ella. Si hay usuarios, My Organization API devolverá un error y no procederá con la eliminación. |

<div id="get-an-access-token">
  ## Obtener un token de acceso
</div>

Puedes obtener un token de acceso para My Organization API de la misma manera que obtendrías un token de acceso para una de tus propias API.

<Note>
  <p class="uppercase font-bold">Operaciones sensibles</p>

  Si vas a permitir que My Organization API realice operaciones sensibles (como registrar un método de autenticación), te recomendamos encarecidamente que uses la [autenticación escalonada](https://auth0.com/docs/secure/multi-factor-authentication/step-up-authentication) para aplicar políticas de seguridad adicionales mediante la [autenticación multifactor (MFA)](https://auth0.com/docs/secure/multi-factor-authentication).
</Note>

<div id="examples">
  ## Ejemplos
</div>

<div id="authorization-code-flow-example">
  ### Ejemplo de flujo de código de autorización
</div>

Usa el [flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow) para aplicaciones web confidenciales con un secreto de cliente.

```bash
curl --request POST \
--url 'https://YOUR_DOMAIN/oauth/token' \
--header 'content-type: application/x-www-form-urlencoded' \
--data 'grant_type=authorization_code' \
--data 'client_id=YOUR_CLIENT_ID' \
--data 'client_secret=YOUR_CLIENT_SECRET' \
--data 'code=AUTH_CODE' \
--data 'redirect_uri=https://yourapp/callback' \
--data 'audience=https://YOUR_DOMAIN/my-org/'
```

**Respuesta de ejemplo**

```json
{
  "access_token": "eyJz93a...k4laUWw",
  "token_type": "Bearer",
  "expires_in": 900,
  "scope": "read:my_org:details update:my_org:identity_providers"
}
```

<div id="authorization-code-flow-with-pkce-example">
  ### Ejemplo de flujo de código de autorización con PKCE
</div>

Utiliza el [flujo de código de autorización con PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) para aplicaciones públicas sin secreto de cliente, aplicaciones de página única, aplicaciones móviles o nativas y herramientas de CLI.

```bash
curl --request POST \
--url 'https://YOUR_DOMAIN/oauth/token' \
--header 'content-type: application/x-www-form-urlencoded' \
--data 'grant_type=authorization_code' \
--data 'client_id=YOUR_CLIENT_ID' \
--data 'code=AUTH_CODE' \
--data 'code_verifier=CODE_VERIFIER' \
--data 'redirect_uri=https://yourapp/callback' \
--data 'audience=https://YOUR_DOMAIN/my-org/'
```

<div id="profiles">
  ## Perfiles
</div>

La My Organization API utiliza [Perfiles de conexión](/docs/es/authenticate/enterprise-connections/connection-profile) y [Perfiles de atributos de usuario](/docs/es/authenticate/enterprise-connections/user-attribute-profile) para definir la estructura, las restricciones y las reglas de las configuraciones creadas por clientes externos.

<div id="connection-profile-cp">
  ### Perfil de conexión (CP)
</div>

El perfil de conexión permite a los desarrolladores de Auth0 especificar cómo deben configurarse los ajustes privados de una conexión de Auth0 cuando la crean terceros. Para obtener más información sobre cómo funciona el perfil de conexión, su mapeo de atributos, sus sobrescrituras, ver ejemplos y aprender a configurar uno, consulta [Perfiles de conexión](/docs/es/authenticate/enterprise-connections/connection-profile).

<div id="user-attribute-profile-uap">
  ### Perfil de atributos de usuario (UAP)
</div>

El Perfil de atributos de usuario (UAP) proporciona una manera coherente de definir, gestionar y mapear atributos de usuario entre protocolos como SCIM, SAML y OIDC. Para obtener más información sobre cómo funciona el UAP, sus mapeos de atributos y sobrescrituras, ver ejemplos y cómo configurar uno, consulta [Perfiles de atributos de usuario](/docs/es/authenticate/enterprise-connections/user-attribute-profile).

<div id="rate-limits">
  ## Límites de frecuencia
</div>

Los límites de frecuencia se aplican en función de tu nivel de servicio:

| **Nivel**               | **Lectura (RPS)** | **Escritura (RPS)** |
| ----------------------- | ----------------- | ------------------- |
| **Free**                | 4                 | 2                   |
| **Public Self-Service** | 8                 | 4                   |
| **Public Enterprise**   | 40                | 20                  |
| **Private Basic**       | 40                | 20                  |
| **Private Performance** | 160               | 80                  |

<div id="per-organization-rate-limits">
  ### Límites de frecuencia por organización
</div>

Además de los límites de frecuencia del nivel de servicio, My Organization API también aplica límites de frecuencia por Organización. Estos límites están diseñados para garantizar una asignación justa de los recursos y evitar que una sola Organización afecte al rendimiento general de tu tenant. Al aplicar estos límites, mitigamos el efecto del “vecino ruidoso”, para garantizar que un aumento repentino de la actividad de una Organización no consuma recursos compartidos ni afecte a otra dentro del mismo entorno. A cada Organización se le asigna una cantidad específica de solicitudes por segundo (RPS), tanto para operaciones de lectura como de escritura.

| **Nivel**               | **Lectura por organización (RPS)** | **Escritura por organización (RPS)** |
| ----------------------- | ---------------------------------- | ------------------------------------ |
| **Free**                | 4                                  | 2                                    |
| **Public Self-Service** | 4                                  | 2                                    |
| **Public Enterprise**   | 8                                  | 4                                    |
| **Private Basic**       | 8                                  | 4                                    |
| **Private Performance** | 16                                 | 8                                    |
