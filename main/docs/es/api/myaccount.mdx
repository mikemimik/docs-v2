<Badge>Versión: 1.0 (Actual)</Badge>

My Account API de Auth0 proporciona un conjunto específico de endpoints para que los usuarios administren la información de su propia cuenta. Los clientes pueden usar estas API para crear experiencias de autoservicio en sus aplicaciones o añadir progresivamente detalles a una cuenta de usuario.

My Account API funciona en el contexto del usuario que ha iniciado sesión y puede usarse directamente en aplicaciones orientadas al usuario.

<Tip>
  <p class="uppercase font-bold">Uso del dominio de Auth0 frente al dominio personalizado</p>

  My Account API admite el uso de tu dominio canónico de Auth0 o de tu dominio personalizado, pero debes usar siempre el mismo durante todo el proceso, lo que incluye:

  * Obtener un token de acceso
  * Establecer el valor de audience
  * Llamar al endpoint de My Account API

  Para obtener más información, consulta [Custom Domains](https://auth0.com/docs/customize/custom-domains).
</Tip>

<div id="activate-the-my-account-api">
  ## Activar la My Account API
</div>

Puedes activar la My Account API para tu tenant en el Auth0 Dashboard:

1. Ve a **Authentication &gt; APIs**.
2. Busca el banner de My Account API.
3. Selecciona **Activate**.

<img src="/docs/images/api/myaccount/My_Account_API_-_Activate.png" />

De forma predeterminada, la My Account API se crea con las siguientes políticas de acceso a la API para aplicaciones:

`require_client_grant` para flujos de usuario

`deny_all` para flujos de cliente (máquina a máquina)

Para que una aplicación acceda a la My Account API en nombre del usuario, debes crear explícitamente una autorización de cliente para esa aplicación, lo que te permite definir los scopes máximos que la aplicación puede solicitar. Como alternativa, puedes cambiar la política de los flujos de acceso de usuario a `allow_all`, lo que permite que cualquier aplicación de tu tenant solicite cualquier scope de la My Account API.

Como la My Account API expone información y operaciones sensibles, Auth0 no recomienda usar `allow_all` para los flujos de acceso de usuario. Debes seguir el principio de privilegio mínimo con la My Account API para asegurarte de que las aplicaciones solo obtengan acceso a lo que realmente necesitan, minimizando así los posibles riesgos de seguridad.

Los permisos finales concedidos a la aplicación estarán determinados por la intersección de los scopes permitidos por la política de acceso a la API para aplicaciones, los permisos de control de acceso basado en roles (RBAC) asignados al usuario final y cualquier consentimiento otorgado por el usuario (si corresponde).

<Note>
  <p class="uppercase font-bold">Flujos compatibles</p>

  No puedes actualizar la política de acceso de cliente para la My Account API, lo que significa que no puedes acceder a la My Account API mediante el Client Credentials Flow.
</Note>

Para obtener más información sobre cómo administrar las políticas de acceso a la API para aplicaciones y sus autorizaciones de cliente asociadas, consulta [Application Access to APIs: Client Grants](https://auth0.com/docs/get-started/applications/application-access-to-apis-client-grants).

<div id="get-an-access-token">
  ## Obtén un token de acceso
</div>

Puedes obtener un token de acceso para la My Account API de la misma manera que obtendrías un token de acceso para una de tus propias API.

<Note>
  <p class="uppercase font-bold">Operaciones sensibles</p>

  Si vas a permitir que la My Account API realice operaciones sensibles (como registrar un método de autenticación), te recomendamos encarecidamente que uses [autenticación escalonada](https://auth0.com/docs/secure/multi-factor-authentication/step-up-authentication) para aplicar políticas de seguridad adicionales mediante [autenticación multifactor (MFA)](https://auth0.com/docs/secure/multi-factor-authentication).
</Note>

Si usas Universal Login, lee los siguientes artículos:

* [Llama a tu API con el flujo de código de autorización](https://auth0.com/docs/get-started/authentication-and-authorization-flow/authorization-code-flow/call-your-api-using-the-authorization-code-flow)
* [Llama a tu API con el flujo de código de autorización con PKCE](https://auth0.com/docs/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce/call-your-api-using-the-authorization-code-flow-with-pkce)

Si usas inicio de sesión integrado, lee los siguientes artículos:

* [Llama a tu API con el flujo de contraseña del propietario del recurso](https://auth0.com/docs/get-started/authentication-and-authorization-flow/resource-owner-password-flow/call-your-api-using-resource-owner-password-flow)
* [Flujo de inicio de sesión con la API Native Passkeys](https://auth0.com/docs/native-passkeys-api#login-flow)

<div id="examples">
  ## Ejemplos
</div>

<div id="universal-login-with-authorization-code-flow">
  ### Universal Login con el flujo de código de autorización
</div>

**Paso 1: Solicitar código de autorización**

```bash lines
curl --request GET \
  --url 'https://{yourDomain}/authorize?response_type=code&client_id={yourClientId}&redirect_uri=%7ByourRedirectUri%7D&scope=create%3Ame%3Aauthentication_methods&offline_access=&audience=https%3A%2F%2F{yourDomain}%2Fme%2F'
```

**Paso 2: Intercambiar el código por un token de acceso**

```bash lines
curl --request POST \
  --url 'https://{yourDomain}/oauth/token' \
  --header 'content-type: application/json' \
  --data '{"grant_type": "authorization_code","client_id": "{yourClientId}","client_secret": "{yourClientId}","code": "{yourAuthorizationCode}","redirect_uri": "{yourRedirectUri}","audience": "{yourAudience}","scope": "create:me:authentication_methods","offline_access": ""}'
```

<div id="embedded-login-with-native-passkeys">
  ### Inicio de sesión integrado con claves de acceso nativas
</div>

**Paso 1: Solicitar el desafío de inicio de sesión**

```bash lines
curl --request POST \
  --url 'https://{yourDomain}/passkey/challenge' \
  --header 'content-type: application/json' \
  --data '{"client_id": "{yourDomain}"}'
```

**Paso 2: Autenticar al usuario existente**

```bash lines
curl --request POST \
  --url 'https://{yourDomain}/oauth/token' \
  --header 'content-type: application/json' \
  --data '{  "grant_type": "urn:okta:params:oauth:grant-type:webauthn",  "client_id": "{yourClientId}",  "scope": "create:me:authentication_methods offline_access",  "audience": "https://{yourDomain}/me/"  "auth_session": "{sessionIdFromTheFirstRequest}",  "authn_response": "{authenticatorResponse}"}'
```

<div id="rate-limits">
  ## Límites de frecuencia
</div>

Durante el acceso anticipado, My Account API tiene un límite de 25 solicitudes por segundo por tenant.
