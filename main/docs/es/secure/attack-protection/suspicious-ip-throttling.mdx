<Tooltip tip="Suspicious IP Throttling: forma de protección contra ataques que protege tu tenant frente a inicios de sesión sospechosos dirigidos a demasiadas cuentas desde una sola dirección IP." cta="Ver glosario" href="/docs/es/glossary?term=Suspicious+IP+Throttling">Suspicious IP Throttling</Tooltip> bloquea el tráfico de cualquier dirección IP que intente realizar rápidamente demasiados inicios de sesión o registros. Esto ayuda a proteger tus aplicaciones frente a ataques de alta frecuencia dirigidos a múltiples cuentas.

Suspicious IP Throttling está habilitado de forma predeterminada cuando creas tu tenant de Auth0. Cuando Auth0 detecta una gran cantidad de intentos de registro o intentos de inicio de sesión fallidos desde una dirección IP, responde a los intentos posteriores con el [código de estado HTTP 429 Too Many Requests](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/429) hasta que esa dirección IP deje de estar limitada.

Para obtener más información sobre los intentos de registro y los intentos de inicio de sesión fallidos registrados en tu tenant, consulta [Log Type Codes](/docs/es/deploy-monitor/logs/log-event-type-codes).

<div id="configure-suspicious-ip-throttling">
  ## Configurar Suspicious IP Throttling
</div>

Puedes configurar Suspicious IP Throttling en el [Auth0 Dashboard](https://manage.auth0.com/#) o con la [Auth0 Management API](https://auth0.com/docs/api/management/v2/attack-protection/patch-suspicious-ip-throttling).

<Warning>
  Auth0 recomienda encarecidamente no desactivar Suspicious IP Throttling.
</Warning>

1. Ve a [Dashboard &gt; Security &gt; protección contra ataques](https://manage.auth0.com/#/security/attack-protection) y selecciona **Suspicious IP Throttling**.
2. Selecciona el interruptor en la parte superior de la página para activar o desactivar Suspicious IP Throttling.

<div id="allow-trusted-ip-addresses-exceed-throttling-limits">
  ## Permita que las direcciones IP de confianza superen los límites de limitación
</div>

Puede eximir hasta 100 direcciones IP individuales y/o rangos CIDR (IPv4 o IPv6) de Suspicious IP Throttling agregándolos a la **IP AllowList**. Auth0 no bloquea ni alerta a los administradores del tenant cuando estas direcciones IP superan los límites de limitación.

1. Vaya a [Dashboard &gt; Security &gt; protección contra ataques](https://manage.auth0.com/#/security/attack-protection) y seleccione **Suspicious IP Throttling**.
2. En el campo **IP AllowList**, introduzca las direcciones IP y/o los rangos CIDR a los que desea permitir intentos ilimitados de login y signup. Separe varias direcciones IP y/o rangos CIDR con comas.

<div id="configure-response">
  ## Configurar la respuesta
</div>

De forma predeterminada, Auth0 envía correos electrónicos a los administradores del tenant cuando una dirección IP se marca como sospechosa.

Puedes configurar cómo responde Auth0 ante intentos de login o registro de alta velocidad.

<Warning>
  Habilitar las funciones de protección contra ataques sin tener activada ninguna configuración de respuesta activa el modo de monitoreo, que solo registra los eventos relacionados en el log del tenant. Para obtener más información, consulta [Ver eventos de log de Attack Protection](/docs/es/secure/attack-protection/view-attack-protection-events).
</Warning>

1. Ve a [Dashboard &gt; Seguridad &gt; Protección contra ataques](https://manage.auth0.com/#/security/attack-protection) y selecciona **Suspicious IP Throttling.**
2. Busca la sección **Respuesta**.
3. En la sección **Configuración de bloqueo**, habilita **Limitar el tráfico de alta velocidad dirigido a demasiadas cuentas** para aplicar limitación al tráfico procedente de direcciones IP que superen el umbral de login o registro.
4. En la sección **Notificaciones**, habilita **Enviar notificación al administrador de la cuenta** para enviar automáticamente un correo electrónico a los administradores del tenant cuando una dirección IP supere el umbral de login o registro.
5. Selecciona **Guardar**.

<div id="configure-throttling-limits-and-rates">
  ## Configurar los límites y las tasas de limitación
</div>

Puedes personalizar cómo Auth0 limita las direcciones IP sospechosas. Puedes cambiar lo siguiente:

* La cantidad máxima de intentos fallidos de login y signup permitidos desde una dirección IP determinada.
* La tasa a la que las direcciones IP sujetas a limitación permiten nuevos intentos de login y signup.

Para obtener más información sobre la cantidad máxima de intentos, la tasa de limitación y cómo funcionan, consulta [Cómo funciona Suspicious IP Throttling](#how-suspicious-ip-throttling-works).

1. Ve a [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection) y selecciona **Suspicious IP Throttling.**

   <Frame>![Personalización de Suspicious IP Throttling en el Dashboard de Auth0](/docs/images/cdy7uua7fh8z/5yLYmDn0FvDB061WgTH2fk/0f3dce552a41aa5756a2eca788944965/Screen_Shot_2021-09-29_at_10.19.32_AM.png)</Frame>
2. Busca la sección **Detection**.
3. Selecciona **Custom** para **Suspicious IP Thresholds.**
4. Configura los ajustes de **Login Threshold**:

   * **Maximum Attempts**: Introduce la cantidad de intentos fallidos de login que una sola dirección IP puede realizar en un día antes de que Auth0 bloquee el siguiente intento.
   * **Throttling Rate**: Introduce la tasa a la que se conceden nuevos tokens de login.
5. Configura los ajustes de **Signup Threshold**:

   * **Maximum Attempts**: Introduce la cantidad de intentos de signup que una sola dirección IP puede realizar en un minuto antes de que Auth0 bloquee el siguiente intento.
   * **Throttling Rate**: Introduce la tasa a la que se conceden nuevos tokens de signup.
6. Selecciona **Save**.

<div id="how-suspicious-ip-throttling-works">
  ### Cómo funciona Suspicious IP Throttling
</div>

Auth0 contabiliza y permite por separado los intentos de inicio de sesión y de registro. Las direcciones IP suspendidas para seguir realizando intentos de inicio de sesión aún pueden intentar registrarse. Las direcciones IP suspendidas para seguir realizando intentos de registro aún pueden intentar iniciar sesión.

<div id="login-attempts">
  #### Intentos de inicio de sesión
</div>

Auth0 aplica limitación a cualquier dirección IP que intente iniciar sesión y falle demasiadas veces en un día. La tasa de limitación de los intentos de inicio de sesión determina cuántos intentos de inicio de sesión permite Auth0 a una dirección IP, distribuidos de manera uniforme a lo largo de 24 horas. Por ejemplo, una tasa de limitación de 100 significa que Auth0 permite un nuevo intento aproximadamente cada 15 minutos.

<div id="signup-attempts">
  #### Intentos de registro
</div>

Auth0 aplica una limitación a cualquier dirección IP que intente realizar demasiados intentos de registro en un minuto. Por ejemplo, si una dirección IP realiza 50 intentos de registro, Auth0 bloquea los intentos siguientes. A diferencia de los inicios de sesión, los intentos de registro no tienen que ser fallidos.

La tasa de limitación de registros determina cuántos intentos de registro concede Auth0 a una dirección IP de manera uniforme a lo largo de 24 horas. Por ejemplo, una tasa de limitación de 72.000 significa que Auth0 concede un nuevo intento aproximadamente cada segundo.

<div id="malformed-requests-and-schema-validation-errors">
  #### Solicitudes malformadas y errores de validación del esquema
</div>

Las solicitudes de inicio de sesión y registro que no siguen la estructura o el formato adecuados no se consideran intentos válidos y, por lo tanto, no cuentan para el umbral configurado.

Para obtener más información sobre las solicitudes de inicio de sesión y registro, consulta la [Auth0 Authentication API](https://auth0.com/docs/api/authentication).

<div id="special-cases">
  ## Casos especiales
</div>

Como la limitación para IP sospechosas depende de la dirección IP del usuario, los siguientes casos de uso requieren configuración adicional:

* **Uso de la concesión de contraseña de <Tooltip tip="Resource Owner: Entidad (como un usuario o una aplicación) capaz de otorgar acceso a un recurso protegido." cta="Ver glosario" href="/docs/es/glossary?term=Resource+Owner">Resource Owner</Tooltip> desde el backend de una aplicación**: Al usar esta llamada no se obtiene la dirección IP del usuario; sin embargo, para que la limitación para IP sospechosas funcione correctamente, puedes configurar tu aplicación para enviar la dirección IP del usuario como parte de la solicitud. Consulta [Avoid Common Issues with Resource Owner Password Flow and Attack Protection: Send the user&#39;s IP address from your server](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
* **Autenticar a un gran número de usuarios desde la misma dirección IP**: Los usuarios que están detrás de un proxy tienen más probabilidades de alcanzar los límites establecidos y activar la limitación. Puedes evitar activar la limitación por error configurando una lista de permitidos para la IP y el rango CIDR del proxy. Consulta [Avoid Common Issues with Resource Owner Password Flow and Attack Protection: Configure your Application to trust the IP address](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
