description: Aprende a habilitar y configurar Bot Detection al usar páginas de inicio de sesión personalizadas creadas con la biblioteca auth0.js.
title: Agregar Bot Detection a páginas de inicio de sesión personalizadas
---

Si creas una página de inicio de sesión personalizada con la biblioteca auth0.js, puedes habilitar <Tooltip tip="Bot Detection: Forma de protección contra ataques en la que Auth0 bloquea el tráfico sospechoso de bots habilitando un CAPTCHA durante el proceso de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> para renderizar un paso de CAPTCHA en situaciones en las que Auth0 determina que una solicitud es de alto riesgo.

El código de tu formulario de inicio de sesión personalizado debe contemplar los casos en los que se le pida al usuario que complete un paso de CAPTCHA. Si no contemplas este caso, tu aplicación puede generar un error.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Bot Detection solo es compatible con páginas de inicio de sesión personalizadas alojadas por Auth0. Es posible que debas ponerte en contacto con el representante de tu cuenta para habilitar esta función en tu tenant.
</Callout>

<div id="use-custom-login-page-template">
  ## Usa una plantilla de página de inicio de sesión personalizada
</div>

Auth0 proporciona una plantilla para usar con código y gestionar inicios de sesión de alto riesgo.

1. Ve a [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings) y selecciona **Classic**.
2. Haz clic en la pestaña **Login** y habilita el interruptor **Customize Login Page** si todavía no está habilitado.
3. Localiza el menú desplegable **Default Templates** y selecciona **Custom Login Form**.

   <Frame>![Dashboard Branding Universal Login Classic pestaña Login Custom Login Form](/docs/images/cdy7uua7fh8z/4m3WA0sKMoR0C1KVnVmZ1G/b311bdbc6c48b4910eac49cc8f1b9ba8/2025-02-26_15-17-18.png)</Frame>
4. Usa la plantilla proporcionada para empezar a personalizar tu página de inicio de sesión.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Recomendamos usar software de control de versiones para gestionar el código fuente de tus páginas de inicio de sesión personalizadas. Para obtener más información, consulta [Classic Login Page Version Control](/docs/es/customize/login-pages/classic-login/version-control).
   </Callout>
5. Si no usas software de control de versiones, puedes reemplazar la plantilla por tu código fuente directamente en el Dashboard.
6. Selecciona **Preview** para ver tu nuevo formulario.
7. Selecciona **Save Changes**.

<div id="customize-the-login-form">
  ## Personaliza el formulario de login
</div>

Si quieres usar Bot Detection, debes usar la versión `9.28` o posterior de la biblioteca `auth0.js`.

`<script src="https://cdn.auth0.com/js/auth0/9.28/auth0.min.js"></script>`

1. Agrega un elemento para renderizar el CAPTCHA debajo del campo de contraseña y encima de los botones de registro e inicio de sesión. Por ejemplo: `<div class="captcha-container"></div>`

2. Inicializa los componentes `loginCaptcha` y `signupCaptcha` después del constructor de `WebAuth`.

   ```javascript lines
   var webAuth = new auth0.WebAuth(params);

   var loginCaptcha = webAuth.renderCaptcha(
       document.querySelector('.captcha-container'),
       null,
       (error, payload) => {
           if (payload) {
               triggerCaptcha = payload.triggerCaptcha;
           }
       }
   );

   var signupCaptcha = webAuth.renderSignupCaptcha(
       document.querySelector('.captcha-container'),
       null,
       (error, payload) => {
           if (payload) {
               triggerCaptcha = payload.triggerCaptcha;
           }
       }
   );
   ```

3. Cuando llames al método `login`, asigna a la propiedad `captcha` el valor `loginCaptcha.getValue()`:

   ```text lines
   webAuth.login({
       realm: connection,
       username: username,
       password: password,
       captcha: loginCaptcha.getValue()
   }, function(err) {
       displayError(err);
       //...
   });
   ```

   Para obtener más información sobre el parámetro de la función callback (`cb`) del método `login`, consulta la [documentación de WebAuth en auth0.js](https://auth0.github.io/auth0.js/global.html#login).

4. Cuando llames al método `signupAndLogin`, asigna a la propiedad `captcha` el valor `signupCaptcha.getValue()`:

   ```text lines
   webAuth.redirect.signupAndLogin({
       connection: databaseConnection,
       email: email,
       password: password,
       captcha: signupCaptcha.getValue()
   }, function(err) {
       displayError(err);
       //...
   });
   ```

   Para obtener más información sobre el parámetro de la función callback (`cb`) del método `signupAndLogin`, consulta la [documentación de WebAuth en auth0.js](https://auth0.github.io/auth0.js/global.html#login).

5. Vuelve a cargar los componentes `loginCaptcha` y `signupCaptcha` en tu lógica genérica de manejo de errores.

   ```javascript lines
   function displayError(err) {
     loginCaptcha.reload();
     signupCaptcha.reload();

     var errorMessage = document.getElementById('error-message');
     errorMessage.innerHTML = err.description;
     errorMessage.style.display = 'block';
   }
   ```

<div id="configure-captcha-templates">
  ### Configurar plantillas de CAPTCHA
</div>

Cuando llamas a los métodos `renderCaptcha` y `renderSignupCaptcha`, puedes configurar la plantilla para cada [proveedor de CAPTCHA compatible](/docs/es/secure/attack-protection/bot-detection/configure-captcha) mediante el parámetro `options`.

La propiedad `templates` del parámetro `options` admite las siguientes propiedades:

| Nombre                 | Descripción                                                                                                                                       |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth0`                | Función de plantilla que recibe el desafío y devuelve una cadena.                                                                                 |
| `recaptcha_v2`         | Función de plantilla que recibe el desafío y devuelve una cadena.                                                                                 |
| `recaptcha_enterprise` | Función de plantilla que recibe el desafío y devuelve una cadena.                                                                                 |
| `hcaptcha`             | Función de plantilla que recibe el desafío y devuelve una cadena.                                                                                 |
| `friendly_captcha`     | Función de plantilla que recibe el desafío y devuelve una cadena.                                                                                 |
| `arkose`               | Función de plantilla que recibe el desafío y devuelve una cadena.                                                                                 |
| `auth0_v2`             | Función de plantilla que recibe el desafío y devuelve una cadena.                                                                                 |
| `error`                | Función de plantilla que devuelve un mensaje de error personalizado cuando no se puede obtener el desafío; recibe el error como primer argumento. |

Para obtener más información sobre las funciones de plantilla predeterminadas de cada proveedor, consulta [auth0.js/src/web-auth/captcha.js en GitHub](https://github.com/auth0/auth0.js/blob/a3ddc0905a5da33aa190a9098467576976b95ec8/src/web-auth/captcha.js#L28).

<div id="support-passwordless-flows">
  ## Admite flujos passwordless
</div>

Si quieres admitir Bot Detection para flujos <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>, debes usar la versión `9.24` o superior de la biblioteca auth0.js.

`<script src="https://cdn.auth0.com/js/auth0/9.24/auth0.min.js"></script>`

1. Agrega un elemento para renderizar el CAPTCHA encima del botón de envío. Si también admites un inicio de sesión con nombre de usuario y contraseña, debes crear un elemento independiente para el CAPTCHA de passwordless. Por ejemplo:
   `<div class="passwordless-captcha-container"></div>`

2. Inicializa el componente CAPTCHA para los flujos passwordless después del constructor de WebAuth.

   ```javascript lines
   var passwordlessCaptcha = webAuth.renderPasswordlessCaptcha(
     document.querySelector('.passwordless-captcha-container')
   );
   ```

3. Agrega la propiedad captcha a la llamada de Passwordless y vuelve a cargar el componente CAPTCHA cuando se produzcan errores.

   ```text lines
   webAuth.passwordlessStart({
     connection: 'email',
     send: 'code',
     email: 'foo@bar.com',
     captcha: passwordlessCaptcha.getValue()
   }, function (err,res) {
     if (err) {
       passwordlessCaptcha.reload();
       // gestionar errores
     }
     // continuar
   });
   ```
