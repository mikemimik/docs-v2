Auth0 admite varias integraciones con proveedores de CAPTCHA de terceros para [Bot Detection](/docs/es/secure/attack-protection/bot-detection).

<div id="google-recaptcha-enterprise-on-google-cloud-platform-gcp">
  ## Google reCAPTCHA Enterprise en Google Cloud Platform (GCP)
</div>

Puede usar Google reCAPTCHA Enterprise como servicio de <Tooltip tip="Bot Detection: Forma de protección contra ataques en la que Auth0 bloquea el tráfico de bots sospechoso al habilitar un CAPTCHA durante el proceso de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> para el tráfico que supere 1 millón de evaluaciones al mes.

Para obtener información sobre cómo configurar reCAPTCHA Enterprise en GCP, consulte [Set up reCAPTCHA Enterprise ... en la documentación de Google](https://cloud.google.com/recaptcha-enterprise/docs/authenticate-apis).

<div id="locate-your-gcp-project-id">
  ### Encuentra el ID de tu proyecto de GCP
</div>

1. Ve a tu [Google Cloud Dashboard](https://console.cloud.google.com/).
2. Selecciona tu proyecto en el menú desplegable para ver el **ID del proyecto** correspondiente.
3. Anota el **ID del proyecto**, ya que lo necesitarás más adelante.

<div id="enable-recaptcha-enterprise">
  ### Habilitar reCAPTCHA Enterprise
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe reCAPTCHA Enterprise API en la barra de búsqueda y luego selecciónala entre los resultados de Marketplace.
2. Selecciona **Habilitar** para habilitar la API.

<div id="create-a-site-key">
  ### Crear una clave del sitio
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe reCAPTCHA Enterprise en la barra de búsqueda y, a continuación, selecciónalo en los resultados de Marketplace.
2. Selecciona **Create Key**.
3. En **Domain List**, agrega el dominio de tu tenant de Auth0.
4. Activa el interruptor de **Disable domain verification**.
5. Activa el interruptor de **Use checkbox challenge** para habilitar la casilla de verificación &quot;No soy un robot&quot; de Google.
6. En **Challenge Security**, selecciona un nivel de dificultad para el desafío.
7. Selecciona **Create Key**.
8. Anota la **Site Key**, ya que la necesitarás más adelante.

<div id="create-an-api-key">
  ### Crear una API key
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe Credentials en la barra de búsqueda y luego selecciónalo entre los resultados.
2. Selecciona **Create Credentials &gt; API Key**. Anota la **API Key**, ya que la necesitarás más adelante.
3. En la ventana emergente, selecciona **Edit Key**.
4. En **Application Restrictions**, selecciona **IP addresses (web servers, cron jobs, etc.)** y luego ingresa las [Direcciones IP de Auth0 para listas de permitidos](/docs/es/secure/security-guidance/data-security/allowlist).
5. En **API Restrictions**, selecciona **Restrict Key**, elige **reCAPTCHA Enterprise API** en la lista desplegable y haz clic en OK.
6. Selecciona **Save**.

<div id="hcaptcha">
  ## hCaptcha
</div>

El widget de hCaptcha puede proteger tus aplicaciones contra bots, spam y otras formas de abuso automatizado.

Para obtener más información sobre hCaptcha y cómo conseguir los datos de configuración, lee [Cambiar de reCAPTCHA a hCaptcha en la documentación de hCaptcha](https://docs.hcaptcha.com/switch).

<div id="friendly-captcha">
  ## Friendly Captcha
</div>

Friendly Captcha es un sistema para evitar el spam en tu sitio web. Puedes añadir el widget de Friendly Captcha a tu aplicación web para combatir el spam, con un impacto mínimo en la experiencia del usuario.

Para obtener más información sobre Friendly Captcha y cómo conseguir los detalles de configuración, consulta [Instalación en la documentación de Friendly Captcha](https://docs.friendlycaptcha.com/#/installation?id=installation).

<div id="arkose">
  ## Arkose
</div>

Arkose Labs ofrece una solución de detección de bots para mitigar ataques realizados tanto por personas como por bots. Para obtener más información sobre cómo integrar Arkose sin complicaciones como tu proveedor de CAPTCHA, consulta [Arkose Bot Manager](https://www.arkoselabs.com/arkose-bot-manager/).

<Warning>
  Debes actualizar a auth0.js v9.23+ para usar Arkose como proveedor de CAPTCHA.
</Warning>

<div id="configure-callback-action-with-auth0js">
  ### Configurar la acción de devolución de llamada con Auth0.js
</div>

Al usar Arkose como proveedor de CAPTCHA, el método `renderCaptcha` del cliente WebAuth de Auth0.js te permite pasar una función de devolución de llamada. El segundo parámetro de la devolución de llamada será un objeto payload que contiene la función `triggerCaptcha`. La función `triggerCaptcha` se puede usar para configurar cómo tu aplicación maneja y renderiza el prompt de CAPTCHA.

Para obtener más información sobre cómo configurar el método `renderCaptcha`, consulta [renderCaptcha](https://auth0.github.io/auth0.js/WebAuth.html#renderCaptcha) en la documentación de Auth0.js.
