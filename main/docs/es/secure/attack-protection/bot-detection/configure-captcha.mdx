Auth0 admite varias integraciones con proveedores de CAPTCHA de terceros para [Bot Detection](/docs/es/secure/attack-protection/bot-detection).

<div id="google-recaptcha-enterprise-on-google-cloud-platform-gcp">
  ## Google reCAPTCHA Enterprise en Google Cloud Platform (GCP)
</div>

Puedes usar Google reCAPTCHA Enterprise como servicio de <Tooltip tip="Bot Detection: Función de protección contra ataques en la que Auth0 bloquea el tráfico sospechoso de bots al habilitar un CAPTCHA durante el proceso de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> para el tráfico que supere 1 millón de evaluaciones al mes.

Para saber cómo configurar reCAPTCHA Enterprise en GCP, consulta [Set up reCAPTCHA Enterprise ... on Google Documentation](https://cloud.google.com/recaptcha-enterprise/docs/authenticate-apis).

<div id="locate-your-gcp-project-id">
  ### Busca el ID de tu proyecto de GCP
</div>

1. Ve a tu [Google Cloud Dashboard](https://console.cloud.google.com/).
2. Selecciona tu proyecto en el menú desplegable para ver el **ID del proyecto** correspondiente.
3. Anota el **ID del proyecto**, ya que lo necesitarás más adelante.

<div id="enable-recaptcha-enterprise">
  ### Habilitar reCAPTCHA Enterprise
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe reCAPTCHA Enterprise API en la barra de búsqueda y selecciónala en los resultados de Marketplace.
2. Selecciona **Enable** para habilitar la API.

<div id="create-a-site-key">
  ### Crea una clave de sitio
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe reCAPTCHA Enterprise en la barra de búsqueda y, a continuación, selecciónalo en los resultados de Marketplace.
2. Selecciona **Create Key**.
3. En **Domain List**, agrega el dominio de tu tenant de Auth0.
4. Activa el toggle de **Disable domain verification**.
5. Activa el toggle de **Use checkbox challenge** para habilitar la casilla de verificación &quot;No soy un robot&quot; de Google.
6. En **Challenge Security**, selecciona un nivel de dificultad para el desafío.
7. Selecciona **Create Key**.
8. Toma nota de la **Site Key**, ya que la necesitarás más adelante.

<div id="create-an-api-key">
  ### Crear una clave de API
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe Credentials en la barra de búsqueda y luego selecciónalo en los resultados.
2. Selecciona **Crear credenciales &gt; Clave de API**. Toma nota de la **clave de API**, ya que la necesitarás más adelante.
3. En la ventana emergente, selecciona **Editar clave**.
4. En **Restricciones de la aplicación**, selecciona **Direcciones IP (servidores web, tareas cron, etc.)** y luego introduce las [direcciones IP de Auth0 para listas de permitidos](/docs/es/secure/security-guidance/data-security/allowlist).
5. En **Restricciones de la API**, selecciona **Restringir clave**, elige **reCAPTCHA Enterprise API** en la lista desplegable y haz clic en Aceptar.
6. Selecciona **Guardar**.

<div id="hcaptcha">
  ## hCaptcha
</div>

El widget de hCaptcha puede proteger sus aplicaciones contra bots, spam y otras formas de abuso automatizado.

Para obtener más información sobre hCaptcha y sobre cómo conseguir los datos de configuración, consulte [Cambiar de reCAPTCHA a hCaptcha en la documentación de hCaptcha](https://docs.hcaptcha.com/switch).

<div id="friendly-captcha">
  ## Friendly Captcha
</div>

Friendly Captcha es un sistema para evitar el spam en tu sitio web. Puedes añadir el widget de Friendly Captcha a tu aplicación web para combatir el spam, con un impacto mínimo en la experiencia del usuario.

Para obtener más información sobre Friendly Captcha y cómo conseguir los datos de configuración, consulta [Instalación en la documentación de Friendly Captcha](https://docs.friendlycaptcha.com/#/installation?id=installation).

<div id="arkose">
  ## Arkose
</div>

Arkose Labs ofrece una solución de detección de bots para mitigar ataques llevados a cabo tanto por personas como por bots. Para obtener más información sobre cómo integrar Arkose sin problemas como proveedor de CAPTCHA, consulte [Arkose Bot Manager](https://www.arkoselabs.com/arkose-bot-manager/).

<Warning>
  Debe actualizar a Auth0.js v9.23+ para usar Arkose como proveedor de CAPTCHA.
</Warning>

<div id="configure-callback-action-with-auth0js">
  ### Configurar la acción de callback con Auth0.js
</div>

Al usar Arkose como proveedor de CAPTCHA, el método `renderCaptcha` del cliente WebAuth de Auth0.js te permite pasar una función de callback. El segundo parámetro de esa función será un objeto payload que contiene la función `triggerCaptcha`. La función `triggerCaptcha` puede usarse para configurar cómo tu aplicación maneja y renderiza el prompt de CAPTCHA.

Para obtener más información sobre cómo configurar el método `renderCaptcha`, consulta [renderCaptcha](https://auth0.github.io/auth0.js/WebAuth.html#renderCaptcha) en la documentación de Auth0.js.
