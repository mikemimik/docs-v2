title: Agregar Bot Detection a los flujos passwordless
---

Si usa flujos de login y registro <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>, puede habilitar <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> para renderizar un paso de CAPTCHA en situaciones en las que Auth0 determina que una solicitud es de alto riesgo.

<Warning>
  Si usa el [SDK de Lock](/docs/es/libraries/lock) con la experiencia de Classic Login, debe actualizar a la versión 11.35.0 o superior para usar Bot Detection en flujos passwordless.

  Si usa el [SDK de Auth0.js](/docs/es/libraries/auth0js), debe actualizar a la versión 9.20.0 o superior para usar Bot Detection en flujos passwordless.
</Warning>

<div id="configure-bot-detection-for-passwordless-flows">
  ## Configurar Bot Detection para flujos passwordless
</div>

Puedes usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para configurar el comportamiento de la funcionalidad CAPTCHA.

1. En el [Auth0 Dashboard](https://manage.auth0.com/dashboard), ve a [Security &gt; Attack Protection &gt; Bot Detection](https://manage.auth0.com/dashboard/#/security/attack-protection/bot-detection).
2. Busca la sección **Response**.
3. En la configuración **Enforce CAPTCHA for passwordless flows**, selecciona cuándo exigir CAPTCHA:

   * **Never**: los usuarios no tienen que completar un CAPTCHA para iniciar sesión.
   * **When Risky**: los usuarios tienen que completar un CAPTCHA si el inicio de sesión es de alto riesgo.
   * **Always**: los usuarios siempre tienen que completar un CAPTCHA para iniciar sesión.
