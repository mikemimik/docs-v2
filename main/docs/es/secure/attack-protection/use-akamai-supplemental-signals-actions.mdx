<Card title="Antes de empezar">
  Para usar Akamai Supplmental Signals en Actions, debes:

  * [configurar Akamai como proxy inverso](/docs/es/customize/custom-domains/self-managed-certificates)
  * [configurar Akamai para enviar Supplemental Signals](/docs/es/secure/attack-protection/configure-akamai-supplemental-signals)
</Card>

Una vez que hayas configurado Akamai para usar Supplemental Signals, podrás usar los datos proporcionados en esas señales en [Auth0 Actions](/docs/es/customize/actions).

<div id="supplemental-signal-object-schemas">
  ## Esquemas de objetos de Supplemental Signal
</div>

Los objetos `akamaiBot` y `akamaiUserRisk` contienen varias propiedades que puedes usar para personalizar tu flujo de autenticación.

<ResponseField name="akamaiBot" type="object">
  <Expandable>
    <ResponseField name="action" type="string">
      La acción del resultado de Akamai Bot Manager.

      Ejemplo: `Monitor`
    </ResponseField>

    <ResponseField name="botCategory" type="string[]">
      La categoría de bot del resultado de Akamai Bot Manager.

      Ejemplo: `["Web Search Engine Bots"]`
    </ResponseField>

    <ResponseField name="botScore" type="number">
      La puntuación de bot del resultado de Akamai Bot Manager.

      Ejemplo: `90`
    </ResponseField>

    <ResponseField name="botScoreResponseSegment" type="string">
      El segmento de respuesta de la puntuación de bot del resultado de Akamai Bot Manager.

      Ejemplo: `aggressive`
    </ResponseField>

    <ResponseField name="botnetId" type="string">
      El ID de la botnet del resultado de Akamai Bot Manager.

      Ejemplo: `googlebot`
    </ResponseField>

    <ResponseField name="type" type="string">
      El tipo del resultado de Akamai Bot Manager.

      Ejemplo: `Akamai-Categorized Bot`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="akamaiUserRisk" type="object">
  <Expandable>
    <ResponseField name="action" type="string">
      La acción de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `monitor`
    </ResponseField>

    <ResponseField name="allow" type="number">
      El estado permitido de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `0`
    </ResponseField>

    <ResponseField name="emailDomain" type="string">
      El dominio de correo electrónico del usuario.

      Ejemplo: `example.com`
    </ResponseField>

    <ResponseField name="general" type="object">
      El riesgo general de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `{ aci: “0”, db: “Chrome 85”, di: “0fc91b5ec42f5a471c16a85e3e388ca57697c1a9”, do: “Mac OX X 10” }`
    </ResponseField>

    <ResponseField name="ouid" type="string">
      El OUID del usuario.

      Ejemplo: `m534264`
    </ResponseField>

    <ResponseField name="requestid" type="string">
      El ID de solicitud del usuario.

      Ejemplo: `19e22e`
    </ResponseField>

    <ResponseField name="risk" type="object">
      El riesgo de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `{ ugp: “ie/M”, unp: “432/H” }`
    </ResponseField>

    <ResponseField name="score" type="number">
      La puntuación de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `0`
    </ResponseField>

    <ResponseField name="status" type="number">
      El estado de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `4`
    </ResponseField>

    <ResponseField name="trust" type="object">
      El nivel de confianza de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `{ udbp: "Chrome85", udfp: "25ba44ec3b391ba4ce5fbbd2979635e254775werwe", udop: "Mac OS X 10", ugp: "FR", unp: "12322", utp: "weekday_3" }`
    </ResponseField>

    <ResponseField name="username" type="string">
      El nombre de usuario del usuario.

      Ejemplo: `testuser@example.com`
    </ResponseField>

    <ResponseField name="uuid" type="string">
      El UUID de la evaluación de riesgo del usuario de Akamai.

      Ejemplo: `86b37525-8047-4a3c-8d7a-23e99666da05`
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="use-cases">
  ## Casos de uso
</div>

<AccordionGroup>
  <Accordion title="Revocar una sesión en función de los resultados de Akamai Account Protector">
    Aquí tienes un ejemplo de cómo podrías revocar una sesión en función de la propiedad `akamaiUserRisk.score`:

    ```javascript
    exports.onExecutePostLogin = async (event, api) => {
      const userRiskHeader = event.authentication?.riskAssessment?.supplemental?.akamai?.akamaiUserRisk;
      if (userRiskHeader?.score && userRiskHeader?.score >= 90) {
            console.log('User is deemed high risk.');
            //Esto revocará las cookies de sesión para denegar el login.
            api.session.revoke('Session revoked, User risk score is greater than 90.');
        }
    };

    ```

    El uso del método `api.session.revoke` (en lugar del método `api.access.deny`) garantiza que, si el usuario actualiza la aplicación, las Supplmental Signals de Akamai se envíen con la solicitud de autenticación y se active el flujo de Action Post Login.
  </Accordion>

  <Accordion title="Solicitar autenticación multifactor (MFA) en función de los resultados de Akamai Bot Manager">
    Aquí tienes un ejemplo de cómo podrías exigir MFA en función de la propiedad `akamaiBot.score`.

    #### Exigir MFA

    Esta Action realiza dos tareas:

    1. **Actualizar [app metadata](/docs/es/manage-users/user-accounts/metadata/metadata-fields-data)**: Si la propiedad `score` supera un valor especificado, registra que se requiere MFA para la sesión.
    2. **Requerir MFA**: Si la propiedad `score` supera un valor especificado o si hay un registro en `app metadata` que indica que se requiere MFA para la sesión, exige MFA.

    ```javascript
    exports.onExecutePostLogin = async (event, api) => {
      const userRiskHeader = event.authentication?.riskAssessment?.supplemental?.akamai?.akamaiUserRisk;

      if (userRiskHeader?.score && userRiskHeader?.score >= 90) {
        console.log(`Setting app metadata for session id: ${event.session?.id}`);
        api.user.setAppMetadata(`mfa_required_${event.session?.id}`, true);
      }

      if (userRiskHeader?.score && userRiskHeader?.score >= 90 ||
          event.user.app_metadata[`mfa_required_${event.session?.id}`]) {
            console.log(`Requiring MFA FOR Session id: ${event.session?.id}`);
            api.multifactor.enable('any', {allowRememberBrowser: false});
      }
    };

    ```

    #### Limpiar app metadata

    Esta Action elimina de `app metadata` la información de MFA específica de la sesión después de que el usuario complete MFA correctamente.

    ```javascript
    exports.onExecutePostLogin = async (event, api) => {
      const mfaMethod = event.authentication?.methods.find((method) => {
        return method.name === 'mfa';
      });

      if (mfaMethod) {
        console.log(`Removing MFA requirement for session id: ${event.session?.id}`);
        api.user.setAppMetadata(`mfa_required_${event.session?.id}`, undefined);
      }
    };
    ```
  </Accordion>
</AccordionGroup>
