  Para usar una de las integraciones compatibles con proveedores de CAPTCHA de terceros, necesitas los detalles de configuración del proveedor. Para obtener más información, consulta [Configurar integraciones con proveedores de CAPTCHA de terceros](/docs/es/secure/attack-protection/bot-detection/configure-captcha).
</Card>

<Tooltip tip="Bot Detection: Forma de protección contra ataques en la que Auth0 bloquea el tráfico sospechoso de bots al habilitar un CAPTCHA durante el proceso de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> mitiga los ataques automatizados al detectar cuándo es probable que una solicitud provenga de un bot. A este tipo de ataques a veces se los denomina ataques de credential stuffing o ataques de validación de listas. Bot Detection ofrece protección frente a ciertos ataques y añade muy poca fricción para los usuarios legítimos. Auth Challenge es nuestra respuesta predeterminada para la detección de bots y proporciona verificación de usuario sin CAPTCHA.

Para obtener más información, consulta [Credential Stuffing Attacks: What Are They and How to Combat Them](https://auth0.com/resources/whitepapers/credential-stuffing-attacks).

Auth0 utiliza una gran cantidad de datos y modelos estadísticos para identificar patrones que indiquen cuándo es probable que ráfagas de tráfico de inicio de sesión, registro o Restablecimiento de contraseña provengan de un bot o script. Los usuarios que intentan iniciar sesión, crear cuentas o restablecer contraseñas desde direcciones IP con una alta probabilidad de formar parte de un ataque de credential stuffing deben completar un paso de verificación adicional. Estos mecanismos detectan el tráfico relacionado con estos ataques sin añadir fricción innecesaria para los usuarios legítimos.

<div id="configure-bot-detection">
  ## Configurar Bot Detection
</div>

Cuando está habilitado, puedes personalizar las preferencias de Bot Detection, como el modelo de detección y las acciones de respuesta, lo que se aplica a todas las conexiones de tu tenant.

Si no configuras acciones de **Respuesta** con Bot Detection habilitado, Bot Detection funciona en modo de [monitoreo](/docs/es/secure/attack-protection#monitoring). El modo de monitoreo registra eventos relacionados, incluidos los detalles de la evaluación de riesgo, en los logs de tu tenant para que puedas revisarlos. Para obtener más información, consulta [View Attack Protection Log Events](/docs/es/secure/attack-protection/view-attack-protection-events).

Puedes habilitar los logs del tenant para Risk Assessment en el [Auth0 Dashboard](https://manage.auth0.com/#).

1. Ve a [Dashboard &gt; Seguridad &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection) y selecciona **Bot Detection**.
2. En la sección **Detección**, habilita el interruptor.

   <Frame>![Sección Detección de la pantalla Attack Protection](/docs/images/cdy7uua7fh8z/3q0HlAm0m0aLilEOAhQ81h/d8626e6e7572a1eb026421d17799d2f0/Screenshot_2024-02-01_at_10.43.58_AM.png)</Frame>

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Si no ves el interruptor para habilitar los logs del tenant para Risk Assessment, es posible que necesites [actualizar tu plan](https://manage.auth0.com/#/tenant/billing/subscription).
   </Callout>
3. En la sección **Respuesta**, elige una respuesta de detección de bots.

   <Frame>![Dashboard - Attack Protection - Bot Detection](/docs/images/cdy7uua7fh8z/3vuVvc01mfCPdc70AU9Oq1/0d5b1519818211ba49c17f5c9e1d1887/2024-05-06_15-47-06.png)</Frame>

   Al usar Auth Challenge, el interruptor **Fail open** está deshabilitado de forma predeterminada.

   Si habilitas el interruptor **Fail Open**:

   * Cuando no se puede acceder al servicio de Bot Detection, Auth0 prioriza el proceso de autenticación y continúa sin un desafío CAPTCHA. Los usuarios finales no ven ningún mensaje de error y aun así pueden iniciar sesión o registrarse.
   * Cuando se puede acceder al servicio de Bot Detection, el proceso de autenticación continúa con un desafío CAPTCHA.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Auth Challenge es la respuesta predeterminada de detección de bots y ofrece una experiencia web sin CAPTCHA, con mayor privacidad y una mejor experiencia de usuario.

     Como se indica en el Dashboard, esta experiencia de login requiere Javascript; si tu experiencia de login debe funcionar sin Javascript, selecciona Simple CAPTCHA.
   </Callout>
4. Selecciona cuándo quieres exigir CAPTCHA para los flujos con contraseña, los flujos passwordless y los flujos de restablecimiento de contraseña.

   * **Never**: Nunca exijas a tus usuarios completar un CAPTCHA para iniciar sesión.
   * **When Risky**: Exige a tus usuarios completar un CAPTCHA solo si el login coincide con la configuración de **Bot Detection Level**.
   * **Always**: Exige siempre a tus usuarios completar un CAPTCHA para iniciar sesión.
5. Si eliges **When Risky** o **Always**, el campo **CAPTCHA Providers** aparecerá en la sección **Respuesta**. Selecciona **Auth Challenge** (proporcionado por Auth0), **Simple CAPTCHA** (proporcionado por Auth0) o una de las integraciones compatibles con proveedores de terceros (requiere configuración y registro externos).

   * Si eliges **Auth Challenge** o **Simple CAPTCHA**, ya terminaste. Si tu experiencia de login no admite JavaScript, debes seleccionar **Simple CAPTCHA**.
   * Si eliges una de nuestras integraciones con proveedores de terceros, ingresa los detalles de configuración del proveedor. Para obtener más información, consulta [Configurar integraciones con proveedores de CAPTCHA de terceros](/docs/es/secure/attack-protection/bot-detection/configure-captcha).
   * Si eliges **Simple CAPTCHA,** la imagen CAPTCHA no es legible para los lectores de pantalla.
6. Si eliges **When Risky**, el campo **Bot Detection Level** aparecerá en la sección **Respuesta**. Selecciona el nivel de seguridad que mejor se adapte a tu caso de uso. Para obtener más información, consulta [Configurar Bot Detection Level](/docs/es/secure/attack-protection/bot-detection#configure-bot-detection-level).
7. Selecciona **Guardar**.

<div id="configure-bot-detection-level">
  ## Configurar Bot Detection Level
</div>

Configura la opción **Bot Detection Level** para adaptarla a tu tolerancia al riesgo y a las necesidades de tu negocio.

Hay tres opciones para elegir:

1. **Low**: Activa CAPTCHA cuando hay una alta probabilidad de actividad de bots, lo que ofrece una experiencia relativamente fluida para los usuarios reales.
2. **Medium**: Predeterminado. Activa CAPTCHA cuando hay una probabilidad moderada de actividad de bots, lo que ofrece un equilibrio entre seguridad y experiencia para los usuarios reales.
3. **High**: Activa CAPTCHA cuando hay una pequeña probabilidad de actividad de bots, lo que ofrece más seguridad, pero también puede generar más fricción para los usuarios reales.

<div id="allow-trusted-ip-addresses-to-bypass-bot-detection">
  ## Permita que las direcciones IP de confianza eludan Bot Detection
</div>

Puede añadir hasta 100 direcciones IP individuales o rangos CIDR (IPv4 o IPv6) al campo **IP AllowList** para que eludan Bot Detection. Auth0 no bloquea ni envía alertas para las direcciones IP o los rangos CIDR de esta lista.

1. Vaya a [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection) y seleccione **Bot Detection**.
2. En el campo **IP AllowList**, introduzca las direcciones IP y/o los rangos CIDR que quiere que eludan Bot Detection. Separe varias direcciones o rangos con comas.

<div id="configure-signup-detection-model-for-custom-login-page-and-classic-login-experience">
  ## Configurar el modelo de detección de registro para la página de inicio de sesión personalizada y la experiencia de Classic Login
</div>

Auth0 proporciona un modelo de aprendizaje automático para la detección de registros, distinto del modelo de inicio de sesión, que aborda diferentes tipos de ataques mediante el análisis de señales únicas.

Este modelo permite que CAPTCHA funcione de forma diferente en los flujos de registro y de inicio de sesión para ofrecer una protección más sólida contra los ataques de registro y ajustarse a los avances más recientes en seguridad.

<Warning>
  Asegúrate de que todas las aplicaciones que usan [Auth0.js](https://github.com/auth0/auth0.js/) y/o [Lock](https://github.com/auth0/lock) estén usando la versión más reciente de la biblioteca antes de habilitar este modelo:

  * **Auth0.js**: 9.28.0+
  * **Lock**: 13.0+

  Si habilitas este modelo mientras usas una biblioteca desactualizada, tu aplicación puede presentar errores.
</Warning>

Puedes configurar el modelo de detección que Bot Detection usa para los flujos de registro si usas una [página de inicio de sesión personalizada](/docs/es/customize/login-pages) o la [experiencia de Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Ve a [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection) y selecciona **Bot Detection**.
2. Busca la sección **Detection Models**.
3. Habilita el interruptor de **Signup detection models for custom and classic login pages**.

<div id="restrictions-and-limitations">
  ## Restricciones y limitaciones
</div>

<div id="flow-limitations">
  ### Limitaciones de los flujos
</div>

Bot Detection funciona para aplicaciones web y móviles que usan [Auth0 Universal Login](/docs/es/authenticate/login/auth0-universal-login). En las aplicaciones que no usan <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>, los niveles de soporte son limitados, en particular en los flujos que no admiten un desafío CAPTCHA o reCAPTCHA.

Asegúrese de que todas sus experiencias de inicio de sesión sean compatibles antes de habilitar Bot Detection; de lo contrario, podría introducir errores en su aplicación.

| Flujo                                                                                                                                        | Limitación                                                                                                                                                                                                                                                                                |
| -------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Universal Login                                                                                                                              | Compatible de forma predeterminada.                                                                                                                                                                                                                                                       |
| Classic Login (sin personalizaciones)                                                                                                        | Compatible de forma predeterminada.                                                                                                                                                                                                                                                       |
| Classic Login (página de inicio de sesión personalizada que usa la plantilla **Lock**)                                                       | Compatible si usa la versión 12.4.0 o posterior del SDK `lock.js`.                                                                                                                                                                                                                        |
| Classic Login (página de inicio de sesión personalizada que usa la plantilla **Custom Login Form**)                                          | Compatible si usa la versión 9.24 o posterior del SDK `auth0.js` y adapta su código para manejar un desafío CAPTCHA o reCAPTCHA.                                                                                                                                                          |
| Aplicaciones nativas                                                                                                                         | Compatible si usa uno de los siguientes SDKs:<ul><li>`Auth0.swift` versión 1.28.0+</li><li>`Auth0.Android` versión 1.25.0+</li><li>`Lock.Swift` versión 2.19.0+</li><li>`Lock.Android` versión 2.22.0+</li></ul>                                                                          |
| Aplicaciones web normales o aplicaciones nativas que usan el flujo de contraseña del propietario del recurso                                 | Compatibilidad limitada. Una respuesta de Bot Detection, como CAPTCHA, requiere un flujo interactivo y, por lo tanto, no es compatible. Si el SDK devuelve el error `requires_verification`, debe activar un flujo de inicio de sesión web para que el usuario complete la autenticación. |
| Flujos no alojados por Auth0 que usan los SDK `lock.js` o `auth0.js` y realizan autenticación de origen cruzado (endpoint `co/authenticate`) | No compatible.                                                                                                                                                                                                                                                                            |

<div id="connection-type-limitations">
  ### Limitaciones del tipo de conexión
</div>

Según los tipos de conexiones que utilice, Bot Detection tiene las siguientes limitaciones.

| Tipo de conexión            | Limitación                                                                                                                             |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| Database                    | Compatible si el inicio de sesión usa un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |
| base de datos personalizada | Compatible si el inicio de sesión usa un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |
| Active Directory/LDAP       | Compatible si el inicio de sesión usa un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |
| empresarial                 | No compatible.                                                                                                                         |
| inicio de sesión social     | No compatible.                                                                                                                         |
| Passwordless                | Compatible si el inicio de sesión usa un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |

<div id="custom-login-page-support">
  ### Compatibilidad con la página de inicio de sesión personalizada
</div>

Si creas una página de inicio de sesión personalizada con Auth0.js, puedes habilitar Bot Detection para renderizar un paso de CAPTCHA en los casos en que Auth0 determine que una solicitud de inicio de sesión es de alto riesgo.

El código de tu formulario de inicio de sesión personalizado debe manejar los casos en que se le pida al usuario completar un paso de CAPTCHA. Para obtener más información, consulta [Agregar Bot Detection a páginas de inicio de sesión personalizadas](/docs/es/secure/attack-protection/bot-detection/bot-detection-custom-login-pages).

<div id="native-application-support">
  ### Compatibilidad con aplicaciones nativas
</div>

Si desarrollas aplicaciones nativas con el SDK de Auth0 para el flujo de inicio de sesión, puedes habilitar Bot Detection para renderizar un paso de CAPTCHA cuando Auth0 determine que una solicitud de inicio de sesión es de alto riesgo.

El código de tu formulario de inicio de sesión personalizado debe contemplar los casos en que se le pida al usuario completar un paso de CAPTCHA. Para obtener más información, lee [Agregar Bot Detection a aplicaciones nativas](/docs/es/secure/attack-protection/bot-detection/bot-detection-native-apps).
