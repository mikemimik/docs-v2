<Card title="Antes de comenzar">
  Para usar una de las integraciones compatibles con proveedores de CAPTCHA de terceros, necesitas los datos de configuración del proveedor. Para obtener más información, lee [Configurar integraciones con proveedores de CAPTCHA de terceros](/docs/es/secure/attack-protection/bot-detection/configure-captcha).
</Card>

<Tooltip tip="Bot Detection: forma de protección contra ataques en la que Auth0 bloquea el tráfico de bots sospechoso al habilitar un CAPTCHA durante el proceso de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> mitiga los ataques automatizados al detectar cuándo es probable que una solicitud provenga de un bot. A este tipo de ataques a veces se los denomina ataques de credential stuffing o ataques de validación de listas. Bot Detection ofrece protección frente a determinados ataques y añade muy poca fricción para los usuarios legítimos. Auth Challenge es nuestra respuesta predeterminada de detección de bots y proporciona verificación de usuario sin CAPTCHA.

Para obtener más información, lee [Credential Stuffing Attacks: What Are They and How to Combat Them](https://auth0.com/resources/whitepapers/credential-stuffing-attacks).

Auth0 utiliza una gran cantidad de datos y modelos estadísticos para identificar patrones que indican cuándo es probable que ráfagas de tráfico de inicio de sesión, registro o restablecimiento de contraseña provengan de un bot o script. Los usuarios que intentan iniciar sesión, crear cuentas o restablecer contraseñas desde direcciones IP con una alta probabilidad de formar parte de un ataque de credential stuffing deben completar un paso de verificación adicional. Los triggers detectan el tráfico relacionado con estos ataques sin añadir fricción innecesaria para los usuarios legítimos.

<div id="configure-bot-detection">
  ## Configurar Bot Detection
</div>

Cuando está habilitado, puedes personalizar las preferencias de Bot Detection, como el modelo de detección y las acciones de respuesta, que se aplican a todas las conexiones de tu tenant.

Si no configuras acciones de **Response** con Bot Detection habilitado, Bot Detection funciona en modo [monitoreo](/docs/es/secure/attack-protection#monitoring). El modo de monitoreo registra eventos relacionados, incluidos los detalles de la evaluación de riesgos, en los logs de tu tenant para que puedas revisarlos. Para obtener más información, consulta [View Attack Protection Log Events](/docs/es/secure/attack-protection/view-attack-protection-events).

Puedes habilitar los logs del tenant para Risk Assessment en el [Auth0 Dashboard](https://manage.auth0.com/#).

1. Ve a [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection) y selecciona **Bot Detection**.
2. En la sección **Detection**, habilita el toggle.

   <Frame>![Sección Detection de la pantalla Attack Protection](/docs/images/cdy7uua7fh8z/3q0HlAm0m0aLilEOAhQ81h/d8626e6e7572a1eb026421d17799d2f0/Screenshot_2024-02-01_at_10.43.58_AM.png)</Frame>

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Si no ves el toggle para habilitar los logs del tenant para Risk Assessment, es posible que tengas que [actualizar tu plan](https://manage.auth0.com/#/tenant/billing/subscription).
   </Callout>
3. En la sección **Response**, elige una respuesta de detección de bots.

   <Frame>![Dashboard - Attack Protection - Bot Detection](/docs/images/cdy7uua7fh8z/3vuVvc01mfCPdc70AU9Oq1/0d5b1519818211ba49c17f5c9e1d1887/2024-05-06_15-47-06.png)</Frame>

   Al usar Auth Challenge, el toggle **Fail open** está deshabilitado de forma predeterminada.

   Si habilitas el toggle **Fail Open**:

   * Cuando el servicio de Bot Detection no está disponible, Auth0 prioriza el proceso de authentication y continúa sin un desafío CAPTCHA. Los usuarios finales no ven ningún mensaje de error y aun así pueden iniciar sesión o registrarse.
   * Cuando el servicio de Bot Detection está disponible, el proceso de authentication continúa con un desafío CAPTCHA.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Auth Challenge es la respuesta predeterminada de detección de bots y ofrece una experiencia web sin CAPTCHA, con mayor privacidad y una mejor experiencia de usuario.

     Como se indica en el Dashboard, esta experiencia de login requiere Javascript; si tu experiencia de login debe funcionar sin Javascript, selecciona Simple CAPTCHA.
   </Callout>
4. Selecciona cuándo quieres exigir CAPTCHA para flujos con contraseña, flujos passwordless y flujos de restablecimiento de contraseña.

   * **Never**: Nunca exijas a tus usuarios que completen un CAPTCHA para iniciar sesión.
   * **When Risky**: Exige a tus usuarios que completen un CAPTCHA solo si el login coincide con la configuración de **Bot Detection Level**.
   * **Always**: Exige siempre a tus usuarios que completen un CAPTCHA para iniciar sesión.
5. Si eliges **When Risky** o **Always**, el campo **CAPTCHA Providers** aparecerá en la sección **Response**. Selecciona **Auth Challenge** (proporcionado por Auth0), **Simple CAPTCHA** (proporcionado por Auth0) o una de las integraciones compatibles de proveedores externos (requiere configuración y registro externos).

   * Si eliges **Auth Challenge** o **Simple CAPTCHA**, ya habrás terminado. Si tu experiencia de login no admite JavaScript, debes seleccionar **Simple CAPTCHA**.
   * Si eliges una de nuestras integraciones de proveedores externos, introduce los detalles de configuración del proveedor. Para obtener más información, consulta [Configurar integraciones con proveedores de CAPTCHA de terceros](/docs/es/secure/attack-protection/bot-detection/configure-captcha).
   * Si eliges **Simple CAPTCHA,** la imagen CAPTCHA no es legible para lectores de pantalla.
6. Si eliges **When Risky**, el campo **Bot Detection Level** aparecerá en la sección **Response**. Selecciona el nivel de seguridad que mejor se adapte a tu caso de uso. Para obtener más información, consulta [Configurar Bot Detection Level](/docs/es/secure/attack-protection/bot-detection#configure-bot-detection-level).
7. Selecciona **Save**.

<div id="configure-bot-detection-level">
  ## Configurar Bot Detection Level
</div>

Configura la opción **Bot Detection Level** para ajustarla a tu tolerancia al riesgo y a las necesidades de tu negocio.

Hay tres opciones para elegir:

1. **Low**: Activa CAPTCHA cuando hay una alta probabilidad de actividad de bots, lo que ofrece una experiencia relativamente fluida para los usuarios reales.
2. **Medium**: Predeterminado. Activa CAPTCHA cuando hay una probabilidad moderada de actividad de bots, lo que ofrece un equilibrio entre seguridad y experiencia para los usuarios reales.
3. **High**: Activa CAPTCHA cuando hay una baja probabilidad de actividad de bots, lo que ofrece más seguridad, pero también puede generar más fricción para los usuarios reales.

<div id="allow-trusted-ip-addresses-to-bypass-bot-detection">
  ## Permita que las direcciones IP de confianza omitan Bot Detection
</div>

Puede permitir que hasta 100 direcciones IP individuales y/o rangos CIDR (IPv4 o IPv6) omitan Bot Detection agregándolos al campo **IP AllowList**. Auth0 no aplica bloqueos ni envía alertas para las direcciones IP o los rangos CIDR de esta lista.

1. Vaya a [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection) y seleccione **Bot Detection**.
2. En el campo **IP AllowList**, introduzca las direcciones IP y/o los rangos CIDR que desea que omitan Bot Detection. Separe varias direcciones o rangos con comas.

<div id="configure-signup-detection-model-for-custom-login-page-and-classic-login-experience">
  ## Configurar el modelo de detección de registros para la página de inicio de sesión personalizada y la experiencia Classic Login
</div>

Auth0 ofrece un modelo de aprendizaje automático para detectar registros, distinto del modelo de inicio de sesión, que aborda diferentes tipos de ataques mediante el análisis de señales específicas.

Este modelo permite que CAPTCHA funcione de forma distinta en los flujos de registro y de inicio de sesión para ofrecer una protección más sólida frente a los ataques de registro y mantenerse alineado con los avances más recientes en seguridad.

<Warning>
  Asegúrate de que todas las aplicaciones que usan [Auth0.js](https://github.com/auth0/auth0.js/) y/o [Lock](https://github.com/auth0/lock) estén usando las versiones más recientes de la biblioteca antes de habilitar este modelo:

  * **Auth0.js**: 9.28.0+
  * **Lock**: 13.0+

  Si habilitas este modelo mientras usas una biblioteca desactualizada, tu aplicación puede presentar errores.
</Warning>

Puedes configurar el modelo de detección que Bot Detection usa para los flujos de registro cuando utilizas una [página de inicio de sesión personalizada](/docs/es/customize/login-pages) o la [experiencia Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Ve a [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection) y selecciona **Bot Detection**.
2. Ubica la sección **Detection Models**.
3. Activa el toggle de **Signup detection models for custom and classic login pages**.

<div id="restrictions-and-limitations">
  ## Restricciones y limitaciones
</div>

<div id="flow-limitations">
  ### Limitaciones de los flujos
</div>

Bot Detection funciona para aplicaciones web y móviles que usan [Auth0 Universal Login](/docs/es/authenticate/login/auth0-universal-login). Para las aplicaciones que no usan <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>, los niveles de soporte son limitados, en particular en flujos que no pueden admitir un desafío CAPTCHA o reCAPTCHA.

Asegúrate de que todas tus experiencias de inicio de sesión sean compatibles antes de habilitar Bot Detection; de lo contrario, podrías introducir errores en tu aplicación.

| Flujo                                                                                                                                        | Limitación                                                                                                                                                                                                                                                                                                  |
| -------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Universal Login                                                                                                                              | Compatible de forma predeterminada.                                                                                                                                                                                                                                                                         |
| Classic Login (sin personalizaciones)                                                                                                        | Compatible de forma predeterminada.                                                                                                                                                                                                                                                                         |
| Classic Login (página de inicio de sesión personalizada con la plantilla de **Lock**)                                                        | Compatible si usas la versión 12.4.0 o superior del SDK `lock.js`.                                                                                                                                                                                                                                          |
| Classic Login (página de inicio de sesión personalizada con la plantilla de **Custom Login Form**)                                           | Compatible si usas la versión 9.24 o superior del SDK `auth0.js` y adaptas tu código para manejar un desafío CAPTCHA o reCAPTCHA.                                                                                                                                                                           |
| Aplicaciones nativas                                                                                                                         | Compatible si usas uno de los siguientes SDKs:<ul><li>`Auth0.swift` versión 1.28.0+</li><li>`Auth0.Android` versión 1.25.0+</li><li>`Lock.Swift` versión 2.19.0+</li><li>`Lock.Android` versión 2.22.0+</li></ul>                                                                                           |
| Aplicaciones web regulares o aplicaciones nativas que usan el flujo de contraseña del propietario del recurso                                | Compatible de forma limitada. La respuesta de Bot Detection, como un CAPTCHA, requiere un flujo interactivo y, por lo tanto, no es compatible. Si el SDK devuelve el error `requires_verification`, debes activar un flujo de inicio de sesión basado en web para que el usuario complete la autenticación. |
| Flujos no alojados por Auth0 que usan los SDK `lock.js` o `auth0.js` y realizan autenticación de origen cruzado (endpoint `co/authenticate`) | No compatible.                                                                                                                                                                                                                                                                                              |

<div id="connection-type-limitations">
  ### Limitaciones de los tipos de conexión
</div>

Según los tipos de conexión que utilices, Bot Detection tiene las siguientes limitaciones.

| Tipo de conexión            | Limitación                                                                                                                     |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Database                    | Se admite si el login utiliza un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |
| base de datos personalizada | Se admite si el login utiliza un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |
| Active Directory/LDAP       | Se admite si el login utiliza un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |
| Empresarial                 | No se admite.                                                                                                                  |
| Inicio de sesión social     | No se admite.                                                                                                                  |
| Passwordless                | Se admite si el login utiliza un flujo de inicio de sesión compatible, como se describe en la tabla de limitaciones del flujo. |

<div id="custom-login-page-support">
  ### Soporte para la página de inicio de sesión personalizada
</div>

Si creas una página de inicio de sesión personalizada con Auth0.js, puedes habilitar Bot Detection para renderizar un paso de CAPTCHA en los casos en que Auth0 determine que una solicitud de inicio de sesión es de alto riesgo.

El código de tu formulario de inicio de sesión personalizado debe contemplar los casos en que se le pida al usuario completar un paso de CAPTCHA. Para obtener más información, consulta [Agregar Bot Detection a páginas de inicio de sesión personalizadas](/docs/es/secure/attack-protection/bot-detection/bot-detection-custom-login-pages).

<div id="native-application-support">
  ### Compatibilidad con aplicaciones nativas
</div>

Si desarrollas aplicaciones nativas con un SDK de Auth0 para el flujo de inicio de sesión, puedes habilitar Bot Detection para renderizar un paso de CAPTCHA en los casos en que Auth0 determine que una solicitud de inicio de sesión es de alto riesgo.

El código de tu formulario de inicio de sesión personalizado debe contemplar los casos en que se le pida al usuario completar un paso de CAPTCHA. Para obtener más información, lee [Agregar Bot Detection a aplicaciones nativas](/docs/es/secure/attack-protection/bot-detection/bot-detection-native-apps).
