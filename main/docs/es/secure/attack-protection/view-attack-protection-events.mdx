---
title: Ver eventos de log de protección contra ataques
description: Describe cómo usar los datos de log del tráfico del tenant para ver eventos de protección contra ataques.
---

Los logs de tu tenant contienen datos útiles que puedes usar para crear gráficos y analizar el perfil del tráfico que pasa por tu tenant. Esto resulta útil al evaluar la actividad de <Tooltip tip="Protección contra ataques: funcionalidades que Auth0 proporciona para detectar y mitigar ataques, incluidas la protección contra ataques de fuerza bruta, Suspicious IP Throttling, la detección de contraseñas vulneradas, la detección de bots y la autenticación multifactor adaptativa." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip>. Por ejemplo, puedes buscar los siguientes eventos para determinar si estás bajo ataque:

* Ráfagas anómalas de tráfico hacia el flujo de inicio de sesión que provocan errores (como errores por nombre de usuario o contraseña incorrectos).
* Ráfagas anómalas de tráfico procedentes de ubicaciones de IP no esperadas.

Estos eventos tienden a ocurrir sin que cambie mucho la tasa de inicios de sesión exitosos.

Puedes usar el campo `event` de los datos de log de tu tenant para ver datos del tráfico del tenant. Recomendamos crear un histograma diario de eventos fallidos de los siguientes tipos:

| Event Code | Evento                                                       |
| ---------- | ------------------------------------------------------------ |
| `f`        | Inicio de sesión fallido                                     |
| `fcoa`     | Autenticación de origen cruzado fallida                      |
| `feccft`   | Intercambio fallido                                          |
| `fepft`    | Intercambio fallido                                          |
| `fsa`      | Autenticación silenciosa fallida                             |
| `fu`       | Inicio de sesión fallido (email/nombre de usuario no válido) |
| `pla`      | Evaluación de pre-login                                      |
| `sepft`    | Intercambio exitoso                                          |

Estos eventos fallidos dependen del flujo que hayas configurado con Auth0.

El siguiente ejemplo muestra un ataque de credential stuffing el 13/02, con un gran aumento de eventos de tipo `fu`, que indica un nombre de usuario no válido (típico de un ataque de credential stuffing).

<div id="rate-of-errors-in-login-flow">
  ## Tasa de errores en el flujo de inicio de sesión
</div>

Busque un pico o una cantidad anómala de errores por nombre de usuario o contraseña incorrectos. Por ejemplo: ¿espera &gt;30,000 errores por hora?

| Código de evento | Evento                                                                   |
| ---------------- | ------------------------------------------------------------------------ |
| `s`              | Inicio de sesión correcto                                                |
| `fu`             | Inicio de sesión fallido, correo electrónico/nombre de usuario no válido |
| `fp`             | Inicio de sesión fallido, contraseña incorrecta                          |

Este es un ejemplo de cómo podrían verse los datos.

<div id="rate-of-attack-protection-events">
  ## Tasa de eventos de protección contra ataques
</div>

Busque un volumen de tráfico anormalmente alto en eventos de protección contra ataques, como <Tooltip tip="Detección de contraseñas vulneradas: Forma de protección contra ataques en la que Auth0 notifica a sus usuarios si usan una combinación de nombre de usuario y contraseña que se ha visto comprometida en una filtración de datos en un sitio web o una aplicación de terceros." cta="Ver glosario" href="/docs/es/glossary?term=breached+password+detection">detección de contraseñas vulneradas</Tooltip> o ataques de fuerza bruta en varias cuentas.

| Código de evento  | Evento                                                     |
| ----------------- | ---------------------------------------------------------- |
| `limit_mu`        | Dirección IP bloqueada                                     |
| `limit_wc`        | Cuenta bloqueada                                           |
| `pwd_leak`        | Contraseña vulnerada durante el inicio de sesión           |
| `signup_pwd_leak` | Contraseña vulnerada durante el registro                   |
| `reset_pwd_leak`  | Contraseña vulnerada durante la recuperación de contraseña |

<div id="number-of-ips-producing-errors-and-their-locations">
  ## Número de IPs que generan errores y sus ubicaciones
</div>

Busca un número elevado de IPs de ubicaciones inesperadas. Por ejemplo: ¿Esperas recibir tráfico de 10.000 IPs de Rusia todos los días? Revisa los datos de la dirección `ip` junto con el tráfico del evento `fu` para determinar de dónde procede el tráfico fallido.

Los datos de geolocalización de IP no están disponibles en los logs del tenant, salvo que puedas enriquecerlos con datos de otra fuente.
