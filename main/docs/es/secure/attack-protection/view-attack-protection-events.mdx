---
title: Ver eventos de log de protección contra ataques
description: Describe cómo usar los datos de log de tráfico del tenant para ver eventos de protección contra ataques.
---

Los registros del tenant contienen datos útiles que puedes usar para crear gráficos y analizar el patrón del tráfico que pasa por tu tenant. Esto resulta útil al evaluar la actividad de <Tooltip tip="Protección contra ataques: Funcionalidades que Auth0 proporciona para detectar y mitigar ataques, incluida la protección contra ataques de fuerza bruta, la limitación de IP sospechosas, la detección de contraseñas vulneradas, la detección de bots y la autenticación multifactor adaptativa." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip>. Por ejemplo, puedes buscar los siguientes eventos para determinar si estás bajo ataque:

* Picos anormales de tráfico hacia el flujo de inicio de sesión que generan errores (como errores por nombre de usuario o contraseña incorrectos).
* Picos anormales de tráfico procedente de ubicaciones geográficas de IP no esperadas.

Estos eventos suelen ocurrir sin que haya grandes cambios en la tasa de inicios de sesión exitosos.

Puedes usar el campo `event` de los datos de registro de tu tenant para ver los datos de tráfico del tenant. Recomendamos crear un histograma diario de eventos de fallo de los siguientes tipos:

| Código de evento | Evento                                                                    |
| ---------------- | ------------------------------------------------------------------------- |
| `f`              | Inicio de sesión fallido                                                  |
| `fcoa`           | Autenticación de origen cruzado fallida                                   |
| `feccft`         | Intercambio fallido                                                       |
| `fepft`          | Intercambio fallido                                                       |
| `fsa`            | Autenticación silenciosa fallida                                          |
| `fu`             | Inicio de sesión fallido (correo electrónico/nombre de usuario no válido) |
| `pla`            | Evaluación de pre-login                                                   |
| `sepft`          | Intercambio exitoso                                                       |

Estos eventos de fallo dependen del flujo que hayas configurado con Auth0.

El siguiente ejemplo muestra un ataque de credential stuffing el 13/02, con un gran aumento de eventos de tipo `fu`, que indica un nombre de usuario no válido (típico de un ataque de credential stuffing).

<div id="rate-of-errors-in-login-flow">
  ## Tasa de errores en el flujo de inicio de sesión
</div>

Busque un aumento repentino o una cantidad inusual de errores por nombre de usuario o contraseña incorrectos. Por ejemplo: ¿espera &gt;30,000 errores por hora?

| Código de evento | Evento                                                                    |
| ---------------- | ------------------------------------------------------------------------- |
| `s`              | Inicio de sesión correcto                                                 |
| `fu`             | Inicio de sesión fallido, correo electrónico/nombre de usuario no válidos |
| `fp`             | Inicio de sesión fallido, contraseña incorrecta                           |

<div id="rate-of-attack-protection-events">
  ## Frecuencia de eventos de protección contra ataques
</div>

Busque un volumen de tráfico anormalmente alto en eventos de protección contra ataques, como la <Tooltip tip="Detección de contraseñas vulnerada: forma de protección contra ataques en la que Auth0 notifica a sus usuarios si usan una combinación de nombre de usuario/contraseña que se ha visto comprometida en una filtración de datos en un sitio web o aplicación de terceros." cta="Ver glosario" href="/docs/es/glossary?term=breached+password+detection">detección de contraseñas vulnerada</Tooltip> o ataques de fuerza bruta dirigidos a varias cuentas.

| Código de evento  | Evento                                                     |
| ----------------- | ---------------------------------------------------------- |
| `limit_mu`        | Dirección IP bloqueada                                     |
| `limit_wc`        | Cuenta bloqueada                                           |
| `pwd_leak`        | Contraseña vulnerada durante el inicio de sesión           |
| `signup_pwd_leak` | Contraseña vulnerada durante el registro                   |
| `reset_pwd_leak`  | Contraseña vulnerada durante la recuperación de contraseña |

<div id="number-of-ips-producing-errors-and-their-locations">
  ## Número de IPs que generan errores y sus ubicaciones
</div>

Busca un número elevado de IPs procedentes de lugares que no tengan sentido. Por ejemplo: ¿Esperas recibir cada día tráfico de 10.000 IPs de Rusia? Observa los datos de las direcciones `ip` junto con el tráfico de eventos `fu` para determinar de dónde proviene el tráfico asociado a fallos.

Los datos de geolocalización de IP no están disponibles en los registros del tenant, a menos que puedas enriquecerlos desde otra fuente.
