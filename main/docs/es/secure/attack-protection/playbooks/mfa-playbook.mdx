Los atacantes pueden aprovecharse de las alertas de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) y usarlas indebidamente para acceder a sus sistemas. A continuación, se describen algunos vectores de ataque habituales contra MFA y se ofrece orientación sobre cómo investigarlos.

<div id="find-log-events-of-interest">
  ### Buscar eventos de log de interés
</div>

Los siguientes tipos de eventos de log son relevantes para investigar un ataque de MFA. Se encuentran en los [registros del tenant de Auth0](/docs/es/deploy-monitor/logs/log-event-type-codes).

| Tipo de evento de log                                                           | Descripción                                                                                                                                                                                                                                               |
| ------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `gd_auth_failed`                                                                | La autenticación multifactor falló. Esto podría deberse a una falla del sistema o a que el usuario introdujo un código incorrecto al usar SMS/voz/Email/TOTP como factor de MFA. Las fallas frecuentes indican un ataque o una mala configuración de MFA. |
| `gd_auth_fail_email_verification`                                               | Una alta frecuencia de eventos de log de fallo en la verificación del correo electrónico puede indicar actividad maliciosa o una mala configuración del tenant.                                                                                           |
| `gd_auth_rejected`, `gd_send_pn` y `gd_send_pn_failure`                         | Los eventos push frecuentes y los eventos push sin respuesta pueden indicar ataques de fatiga de MFA (T1621).                                                                                                                                             |
| `gd_otp_rate_limit_exceed`                                                      | Demasiadas fallas de MFA en un período corto pueden indicar ataques automatizados.                                                                                                                                                                        |
| `gd_recovery_failed`                                                            | Las fallas repetidas en la recuperación de MFA pueden indicar intentos de atacantes de eludir o reemplazar factores de autenticación adicionales.                                                                                                         |
| `gd_send_sms`, `gd_send_sms_failure`, `gd_send_voice` y `gd_send_voice_failure` | Una alta frecuencia de estos eventos indica ataques de SMS pumping o fraude telefónico. También puede indicar intentos de eludir SMS/voz como factor.                                                                                                     |
| `gd_unenroll`                                                                   | La desinscripción masiva de dispositivos de MFA puede indicar campañas exitosas de toma de control de cuentas.                                                                                                                                            |

<div id="mitigation-strategies">
  ## Estrategias de mitigación
</div>

Las siguientes son respuestas de ejemplo ante ataques contra MFA:

* Migre a opciones de MFA más seguras sustituyendo la MFA basada en SMS/voz por [OTP](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa) o [Webauthn](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa) para mitigar ataques de SMS pumping o fraude telefónico.
* Refuerce la seguridad del proveedor de SMS/voz implementando protección contra el fraude, como [Preventing Fraud in Verify](https://www.twilio.com/docs/verify/preventing-toll-fraud) de Twilio, al usar MFA por SMS/voz.
* Evite la fatiga de MFA aplicando límites de frecuencia a las notificaciones push.
