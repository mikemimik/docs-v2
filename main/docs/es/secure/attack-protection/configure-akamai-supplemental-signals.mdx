<Card title="Antes de comenzar">
  Para configurar Akamai para que envíe Supplemental Signals a tu tenant de Auth0, debes [configurar Akamai como proxy inverso](/docs/es/customize/custom-domains/self-managed-certificates).
</Card>

Si usas Akamai Bot Manager y/o Akamai Account Protector, puedes [configurar tu proxy inverso en Akamai](/docs/es/customize/custom-domains/self-managed-certificates/configure-akamai-for-use-as-reverse-proxy) para enviar los resultados a Auth0 y [usar los resultados en Actions](/docs/es/secure/attack-protection/use-akamai-supplemental-signals-actions).

<div id="supported-login-flows">
  ## Flujos de inicio de sesión compatibles
</div>

Auth0 admite actualmente los siguientes flujos de inicio de sesión para Supplemental Signals:

* Universal Login: Identifier + Password
  * [Universal Login: Identifier First](/docs/es/authenticate/login/auth0-universal-login/identifier-first)
  * [Universal Login: Registro en conexiones de base de datos](/docs/es/authenticate/database-connections)
  * Universal Login: restablecimiento de contraseña, incluido Auth0 Organizations
  * [Resource Owner Password Grant (ROPG)](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow)

<div id="step-1-register-an-api-in-akamai">
  # Paso 1: Registrar una API en Akamai
</div>

Para que Akamai sepa cuándo los intentos de inicio de sesión se realizan correctamente o fallan, tendrás que registrar una API en Akamai y luego definir un endpoint para cada flujo de inicio de sesión de Auth0 que quieras admitir.

Para obtener más información, consulta [Register an API on Akamai TechDocs](https://techdocs.akamai.com/api-definitions/docs/register-api) y [Add API resources on Akamai TechDocs](https://techdocs.akamai.com/api-definitions/docs/add-api-resources).

<div id="step-2-forward-bot-results-from-akamai-to-auth0">
  # Paso 2: Reenviar los resultados de bots de Akamai a Auth0
</div>

Para configurar Akamai para que reenvíe los resultados de bots a Auth0, consulta [Forward Bot Results to Origin on Akamai TechDocs](https://techdocs.akamai.com/content-protector/docs/forward-bot-results-to-origin).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Es posible que el encabezado `akamai-user-risk` no esté presente en todas las solicitudes de inicio de sesión, ya que Akamai solo lo envía cuando crea una puntuación de riesgo de usuario.
</Callout>

<div id="step-3-enable-processing-of-akamai-headers-in-auth0">
  # Paso 3: Habilita el procesamiento de encabezado de Akamai en Auth0
</div>

Puedes configurar Auth0 para que acepte los Supplemental Signals enviados desde Akamai en el Auth0 Dashboard o con la Auth0 Management API.

<Tabs>
  <Tab title="Dashboard">
    Para habilitar Supplemental Signals en el Dashboard:

    1. Ve a **[Auth0 Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/dashboard/#/security/attack-protection)**.
    2. Selecciona **Supplemental Signals**.
    3. Activa el interruptor de **Enable encabezado de Akamai**.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si no ves la opción de Supplemental Signals en Auth0 Dashboard, debes solicitar el complemento de Attack Protection. Para obtener más información, ponte en contacto con [Ventas de Auth0](https://auth0.com/get-started?place=header\&type=button\&text=talk%20to%20sales).
    </Callout>
  </Tab>

  <Tab title="Management API">
    Para habilitar Supplemental Signals con la Management API:

    1. [Obtén un token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con el scope `update:attack_protection`.
    2. Llama al endpoint de la Management API [Actualizar la configuración de Supplemental Signals para un tenant](/docs/es/api/management/v2/supplemental-signals/patch-supplemental-signals) con el siguiente cuerpo:

    ```json
    {
      "akamai_enabled": true
    }
    ```
  </Tab>
</Tabs>

<div id="step-4-test-and-verify-configuration">
  # Paso 4: Probar y verificar la configuración
</div>

Pruebe la configuración llamando a cada endpoint de los flujos de inicio de sesión que haya configurado.

Si ha configurado todo correctamente:

* Los encabezados `akamai-user-risk` y `akamai-bot` estarán presentes en los registros del tenant para los eventos relacionados.
* Puede usar las siguientes propiedades en el objeto `event` de la Action de Post Login:
  * `authentication.riskAssessment.supplemental.akamai.akamaiBot`
  * `authentication.riskAssessment.supplemental.akamai.akamaiUserRisk`
