title: Estructura del token de identidad
---

<Tooltip tip="Token de identidad: credencial destinada al propio cliente, y no al acceso a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">Tokens de identidad</Tooltip> siguen el estándar [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens), lo que significa que su estructura básica se ajusta a la [estructura típica de JWT](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure) y que contienen [claims de JWT](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims) estándar sobre el propio token.

Además de lo que exige <Tooltip tip="Token de identidad: credencial destinada al propio cliente, y no al acceso a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>, los tokens de identidad también contienen claims sobre el usuario autenticado, predefinidas por el protocolo <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de login." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC), y por ello se conocen como claims estándar de OIDC. Algunas claims estándar de OIDC incluyen:

* `name`
* `nickname`
* `picture`
* `email`
* `email_verified`

Para consultar la lista completa de claims estándar de OIDC, consulta [la especificación de OIDC: Claims estándar](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). También puedes crear [claims personalizadas](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims), que son claims que defines, controlas y agregas a un token mediante Auth0 Actions.

Puedes controlar qué claims de OIDC se incluyen en el token de identidad que consume tu aplicación incluyendo [Scopes de OpenID Connect](/docs/es/get-started/apis/scopes/openid-connect-scopes) específicos en un parámetro cuando solicitas tokens al autenticar a los usuarios. Para aprender a solicitar un token de identidad, consulta [Get ID Tokens](/docs/es/secure/tokens/id-tokens/get-id-tokens).

<div id="sample-id-token">
  ## Ejemplo de token de ID
</div>

Este ejemplo muestra el contenido de un token de identidad. Observa que el valor de <Tooltip tip="Audiencia: Identificador único de la audiencia de un token emitido. Se denomina aud en un token, y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip> (ubicado en el claim `aud`) corresponde al identificador de la aplicación, lo que significa que solo esta aplicación específica debe usar el token. Para obtener más información sobre los claims de este ejemplo, consulta [Claims de JSON Web Token](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims).

```json lines
{
  "iss": "http://my-domain.auth0.com",
  "sub": "auth0|123456",
  "aud": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "name": "Jane Doe",
  "given_name": "Jane",
  "family_name": "Doe",
  "gender": "female",
  "birthdate": "0000-10-31",
  "email": "janedoe@example.com",
  "picture": "http://example.com/janedoe/me.jpg"
}
```
