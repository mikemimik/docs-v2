title: Estructura del token de identidad
---

<Tooltip tip="token de identidad: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="View Glossary" href="/docs/es/glossary?term=ID+tokens">Los tokens de identidad</Tooltip> siguen el estándar [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens), lo que significa que su estructura básica se ajusta a la típica [Estructura JWT](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure) y contienen [Claims JWT](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims) estándar sobre el propio token.

Más allá de lo que exige el <Tooltip tip="token de identidad: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="View Glossary" href="/docs/es/glossary?term=JWT">JWT</Tooltip>, los tokens de identidad también contienen claims sobre el usuario autenticado, predefinidos por el protocolo <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="View Glossary" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC), conocidos como claims estándar de OIDC. Algunos claims estándar de OIDC son:

* `name`
* `nickname`
* `picture`
* `email`
* `email_verified`

Para ver la lista completa de claims estándar de OIDC, consulte [Especificación OIDC: Claims estándar](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). También puede crear [custom claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims), que son claims que usted define, controla y agrega a un token mediante Auth0 Actions.

Usted controla qué claims de OIDC se incluyen en el token de identidad que consume su aplicación incluyendo [Scopes de OpenID Connect](/docs/es/get-started/apis/scopes/openid-connect-scopes) específicos en un parámetro al solicitar tokens durante la autenticación de usuarios. Para aprender cómo solicitar un token de identidad, consulte [Get ID Tokens](/docs/es/secure/tokens/id-tokens/get-id-tokens).

<div id="sample-id-token">
  ## Ejemplo de token de ID
</div>

Este ejemplo muestra el contenido de un token de identidad. Ten en cuenta que el valor de <Tooltip tip="Audiencia: Identificador único de la audiencia de un token emitido. Se llama aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip> (ubicado en el claim `aud`) se establece en el identificador de la aplicación, lo que significa que solo esta aplicación específica debe usar el token. Para obtener más información sobre los claims de este ejemplo, consulta [JSON Web Token Claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims).

```json lines
{
  "iss": "http://my-domain.auth0.com",
  "sub": "auth0|123456",
  "aud": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "name": "Jane Doe",
  "given_name": "Jane",
  "family_name": "Doe",
  "gender": "female",
  "birthdate": "0000-10-31",
  "email": "janedoe@example.com",
  "picture": "http://example.com/janedoe/me.jpg"
}
```
