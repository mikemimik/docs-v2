title: Obtener tokens de identidad
---

Para obtener un <Tooltip tip="token de identidad: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip>, debes solicitarlo al autenticar a los usuarios. Auth0 facilita que tu aplicación autentique usuarios con:

* [Quickstarts](/docs/es/quickstarts): La forma más sencilla de implementar la autenticación, ya que te muestran cómo usar [Universal Login](/docs/es/authenticate/login/auth0-universal-login), el widget de Lock y los SDKs de Auth0 específicos para cada lenguaje y framework. Nuestra [documentación de Lock](/docs/es/libraries/lock) y la [documentación de Auth0.js](/docs/es/libraries/auth0js) ofrecen información detallada sobre cómo obtener un token de identidad después de la autenticación.
* [Authentication API](https://auth0.com/docs/api/authentication): Si prefieres implementarlo por tu cuenta, puedes llamar directamente a nuestra API. Primero, debes saber [qué flujo usar](/docs/es/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use) antes de seguir el tutorial del flujo correspondiente.

<div id="control-id-token-contents">
  ## Controla el contenido del token de identidad
</div>

Puedes controlar qué claims sobre el usuario autenticado se incluyen en el token de identidad que consume tu aplicación incluyendo [Scopes de OpenID Connect](/docs/es/get-started/apis/scopes/openid-connect-scopes) específicos en el parámetro `scope` al solicitar tokens para autenticar a los usuarios.

También puedes crear [claims personalizadas](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims), es decir, claims que defines, controlas y agregas a un token mediante una rule. Al igual que con otros <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, del token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWTs">JWT</Tooltip>, debes [validar un token de identidad](/docs/es/secure/tokens/id-tokens/validate-id-tokens) antes de asumir que su contenido es confiable.

<div id="renew-id-tokens">
  ## Renovar tokens de identidad
</div>

De forma predeterminada, un token de identidad es válido durante 36000 segundos (10 horas). Si hay alguna preocupación de seguridad, puedes [acortar el tiempo antes de que el token expire](/docs/es/secure/tokens/id-tokens/update-id-token-lifetime), pero recuerda que uno de los objetivos de este token es mejorar el rendimiento almacenando en caché la información del usuario.

Una vez que un token de identidad haya expirado, puede que quieras renovar tu token de identidad. Para renovar el token de identidad, puedes volver a autenticar al usuario con Auth0 o usar un [token de actualización](/docs/es/secure/tokens/refresh-tokens).
