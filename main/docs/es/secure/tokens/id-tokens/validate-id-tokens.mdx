description: Describe cómo validar un token de ID.
title: Validar tokens de ID
---

Un [token de ID](/docs/es/secure/tokens/id-tokens), que contiene atributos del perfil del usuario, lo consume una aplicación y normalmente se usa para mostrar la interfaz de usuario. Auth0 emite todos los <Tooltip tip="Token de ID: credencial destinada al propio cliente, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de ID</Tooltip> en formato [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens).

Si alguna de estas comprobaciones falla, el token se considera no válido y la solicitud debe rechazarse.

1. [Valida el JWT](/docs/es/secure/tokens/json-web-tokens/validate-json-web-tokens).
2. Comprueba los claims estándar adicionales. Si ya realizaste la validación estándar del JWT, ya habrás decodificado el [payload del JWT](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure) y revisado sus claims estándar. Los claims adicionales que deben verificarse para los tokens de ID incluyen:

   * **Audience del token** (`aud`, string): El valor de audience del token debe coincidir con el ID de cliente de la aplicación, tal como se define en **Client ID** dentro de [Settings de la aplicación](https://manage.auth0.com/#/applications).
   * **Nonce** (`nonce`, string): Se recomienda pasar un nonce en la solicitud de token (obligatorio para el flujo implícito) para ayudar a prevenir ataques de repetición. El valor nonce del token debe coincidir exactamente con el nonce original enviado en la solicitud. Consulta [Mitigate Replay Attacks](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow) para obtener más información.
