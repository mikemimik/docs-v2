description: Describe cómo validar un token de identidad.
title: Validar tokens de identidad
---

Un [token de identidad](/docs/es/secure/tokens/id-tokens), que contiene atributos del perfil de usuario, lo consume una aplicación y suele usarse para mostrar la interfaz de usuario. Auth0 emite todos los <Tooltip tip="Token de ID: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de identidad</Tooltip> en formato [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens).

Si alguna de estas comprobaciones falla, el token se considera no válido y la solicitud debe rechazarse.

1. [Valida el JWT](/docs/es/secure/tokens/json-web-tokens/validate-json-web-tokens).
2. Comprueba claims estándar adicionales. Si has realizado la validación estándar del JWT, ya has decodificado el [payload del JWT](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure) y has revisado sus claims estándar. Entre las claims adicionales que debes verificar en los tokens de identidad se incluyen:

   * **Audiencia del token** (`aud`, string): El valor de `audience` del token debe coincidir con el ID de cliente de la aplicación, tal como se define en la [Configuración de la aplicación](https://manage.auth0.com/#/applications), en el campo **ID de cliente**.
   * **Nonce** (`nonce`, string): Se recomienda incluir un nonce en la solicitud de token (es obligatorio para el flujo implícito) para ayudar a prevenir ataques de repetición. El valor de nonce del token debe coincidir exactamente con el nonce original enviado en la solicitud. Consulta [Mitigar ataques de repetición](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow) para obtener más información.
