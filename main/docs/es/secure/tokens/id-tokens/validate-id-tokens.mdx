title: Validar tokens de identidad
---

Un [token de identidad](/docs/es/secure/tokens/id-tokens), que contiene atributos del perfil del usuario, lo consume una aplicación y normalmente se usa para mostrar la interfaz de usuario. Auth0 emite todos los <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de identidad</Tooltip> en formato [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens).

Si alguna de estas comprobaciones falla, el token se considera no válido y la solicitud debe rechazarse.

1. [Valide el JWT](/docs/es/secure/tokens/json-web-tokens/validate-json-web-tokens).
2. Compruebe otros claims estándar. Si ya realizó la validación estándar del JWT, ya decodificó el [payload del JWT](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure) y revisó sus claims estándar. Entre los claims adicionales que deben verificarse en los tokens de identidad se incluyen:

   * **Audience del token** (`aud`, string): El valor de audience del token debe coincidir con el ID de cliente de la aplicación, tal como se define en la [configuración de la aplicación](https://manage.auth0.com/#/applications) en el campo **Client ID**.
   * **Nonce** (`nonce`, string): Se recomienda enviar un nonce en la solicitud de token (es obligatorio para el flujo implícito) para ayudar a prevenir ataques de repetición. El valor de nonce del token debe coincidir exactamente con el nonce original enviado en la solicitud. Consulte [Mitigate Replay Attacks](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow) para obtener más información.
