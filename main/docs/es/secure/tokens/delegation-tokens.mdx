<Warning>
  De forma predeterminada, la delegación está deshabilitada para los tenants que no usaban ningún add-on al 8 de junio de 2017. Los tenants Legacy que actualmente usan un add-on que requiere delegación pueden seguir usando esta funcionalidad. Si la funcionalidad de delegación cambia o se elimina del servicio en algún momento, se notificará con antelación a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar.
</Warning>

Se debe obtener y usar un token de delegación cuando una aplicación necesita llamar a la API de un add-on de una aplicación, como Firebase o SAP, registrado y configurado en Auth0, dentro del mismo tenant que el programa que realiza la llamada.

A partir de un token existente, este endpoint generará un nuevo token firmado con el secreto de la aplicación `target`. Esto se usa para transferir la identidad del usuario desde la aplicación hasta una API.

El tipo de token de delegación varía según el proveedor. Por ejemplo, si se emite para Azure Blob Storage, será una SAS (Shared Access Signature). Si es para el add-on de Firebase, será un <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de Access Token) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>.

El <Tooltip tip="ID token: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> de un usuario autenticado se puede usar con el endpoint `/delegation` para solicitar un token de delegación para un destino concreto. El destino puede ser un add-on de una aplicación configurado en Auth0. Esto puede hacerse con los add-ons que no sean add-ons de <Tooltip tip="Security Assertion Markup Language (SAML): protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> o <Tooltip tip="Security Assertion Markup Language (SAML): protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=WS-Fed">WS-Fed</Tooltip>. El add-on debe configurarse en Auth0 con secrets obtenidos del servicio del add-on, como Firebase. Las instrucciones para configurar los secrets están disponibles en la página de configuración de cada add-on. Los secrets se usan para firmar el token de delegación, de modo que la API del add-on pueda validar el token y confiar en él.

El endpoint de delegación te permite establecer varios parámetros que determinan el contenido del token de delegación, incluidos `target`, `scope`, la API a la que se llamará (`api_type`) y un área adicional de formato libre para parámetros adicionales.

<div id="validity-period-and-termination">
  ## Período de validez y revocación
</div>

El período de validez y la posibilidad de revocar un token de delegación varían según cada add-on. Para obtener más información, consulte la documentación del proveedor de la API de cualquier add-on.

<div id="use-delegation-tokens-with-public-applications">
  ## Usa tokens de delegación con aplicaciones públicas
</div>

Hay una advertencia importante que debes tener en cuenta al usar el endpoint de delegación con aplicaciones públicas.

Si llamas al endpoint `/token` desde una aplicación pública, el token de identidad se firmará obligatoriamente con `RS256`, incluso si el algoritmo de firma JsonWebToken en la configuración de la aplicación está establecido como `HS256`.

Si después llamas al endpoint de delegación con ese token de identidad, fallará si el algoritmo de firma JsonWebToken de la aplicación estaba configurado como `HS256`. Esto se debe a que delegación realiza la validación según la configuración de la aplicación, pero el token de identidad se emitió con un algoritmo distinto debido al cambio forzado de algoritmo.

Por lo tanto, si tienes previsto usar delegación con una aplicación pública, es importante que configures el algoritmo de firma JsonWebToken de tu aplicación como `RS256`.
