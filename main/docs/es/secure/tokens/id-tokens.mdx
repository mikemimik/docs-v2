<Tooltip tip="Token de identidad: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">Tokens de identidad</Tooltip> se usan en la autenticación basada en tokens para almacenar en caché la información del perfil del usuario y proporcionársela a una aplicación cliente, lo que mejora el rendimiento y la experiencia. La aplicación recibe un token de identidad después de que un usuario se autentica correctamente; luego lo consume y extrae de él la información del usuario, que puede usar para personalizar su experiencia.

Por ejemplo, supongamos que tienes una aplicación web tradicional que [registras en Auth0](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps) y configuras para permitir que los usuarios inicien sesión con Google. Una vez que un usuario inicia sesión, usa el token de identidad para recopilar información como el nombre y la dirección de correo electrónico, que luego puedes usar para generar y enviar automáticamente un correo electrónico de bienvenida personalizado.

Los tokens de identidad nunca deben usarse para obtener acceso directo a las API ni para tomar decisiones de autorización.

<div id="id-token-security">
  ## Seguridad del token de identidad
</div>

Al igual que con otros [JWTs](/docs/es/secure/tokens/json-web-tokens), debe seguir las [prácticas recomendadas para el uso de tokens](/docs/es/secure/tokens/token-best-practices) al usar y almacenar tokens de identidad.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Asegúrese de [validar los tokens de identidad](/docs/es/secure/tokens/id-tokens/validate-id-tokens) antes de utilizar la información que contienen. Puede usar una [biblioteca](https://jwt.io/#libraries-io) para facilitar esta tarea.
</Callout>

Proteger las aplicaciones que realizan llamadas a la API plantea sus propios desafíos. Debe asegurarse de que los tokens y otros datos sensibles no sean vulnerables a ataques de cross-site scripting (XSS) y de que JavaScript malicioso no pueda leerlos.

<div id="id-token-lifetime">
  ## Duración del token de identidad
</div>

De forma predeterminada, un token de identidad es válido durante 36000 segundos (10 horas). Si hay preocupaciones de seguridad, puedes acortar el tiempo antes de que el token caduque, teniendo en cuenta que uno de los propósitos del token es mejorar la experiencia del usuario al almacenar en caché su información. Consulta [Update ID Token Lifetime](/docs/es/secure/tokens/id-tokens/update-id-token-lifetime) para obtener más detalles.
