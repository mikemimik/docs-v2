<Tooltip tip="Token de identidad: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">Tokens de identidad</Tooltip> se usan en la autenticación basada en tokens para almacenar en caché información del perfil del usuario y proporcionársela a una aplicación cliente, lo que mejora el rendimiento y la experiencia. La aplicación recibe un token de identidad después de que el usuario se autentica correctamente; luego lo utiliza y extrae de él información del usuario, que puede usar para personalizar su experiencia.

Por ejemplo, supongamos que tienes una aplicación web tradicional que [registras en Auth0](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps) y configuras para permitir que los usuarios inicien sesión con Google. Cuando un usuario inicia sesión, usa el token de identidad para recopilar información como su nombre y dirección de correo electrónico, que luego puedes usar para generar y enviar automáticamente un correo electrónico de bienvenida personalizado.

Los tokens de identidad nunca deben usarse para obtener acceso directo a las API ni para tomar decisiones de autorización.

<div id="id-token-security">
  ## Seguridad del token de identidad
</div>

Al igual que con otros [JWT](/docs/es/secure/tokens/json-web-tokens), debes seguir las [prácticas recomendadas para el uso de tokens](/docs/es/secure/tokens/token-best-practices) al usar y almacenar tokens de ID.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Asegúrate de [validar tokens de ID](/docs/es/secure/tokens/id-tokens/validate-id-tokens) antes de usar la información que contienen. Puedes usar una [biblioteca](https://jwt.io/#libraries-io) para ayudarte con esta tarea.
</Callout>

Proteger las aplicaciones que realizan llamadas a la API plantea sus propios desafíos. Debes asegurarte de que los tokens y otros datos sensibles no sean vulnerables a ataques de cross-site scripting (XSS) y de que JavaScript malicioso no pueda leerlos.

<div id="id-token-lifetime">
  ## Duración del token de identidad
</div>

Un token de identidad es válido de forma predeterminada durante 36000 segundos (10 horas). Si hay consideraciones de seguridad, puedes acortar el tiempo antes de que expire, teniendo en cuenta que uno de sus propósitos es mejorar la experiencia del usuario mediante el almacenamiento en caché de su información. Consulta [Actualizar la duración del token de ID](/docs/es/secure/tokens/id-tokens/update-id-token-lifetime) para obtener más información.
