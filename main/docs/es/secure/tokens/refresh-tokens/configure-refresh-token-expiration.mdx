<Tooltip tip="Token de actualización: token utilizado para obtener un nuevo token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+tokens">
  Tokens de actualización
</Tooltip>

pueden ser objeto de uso indebido si se filtran, ya que pueden usarse para obtener nuevos

<Tooltip tip="Token de actualización: token utilizado para obtener un nuevo token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">
  tokens de acceso
</Tooltip>

. Para mitigar este riesgo, Auth0 recomienda usar la detección automática de reutilización y la

<Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token+Rotation">
  rotación del Refresh Token
</Tooltip>

. La rotación del Refresh Token emite un token de actualización que expira tras un período predefinido. Una vez que expira, el usuario recibe un nuevo token de actualización de la misma familia, es decir, tokens de actualización que comparten un ID de familia,
o un nuevo par de token de acceso/token de actualización. Para obtener más información, consulta [rotación del Refresh Token](/docs/es/secure/tokens/refresh-tokens/refresh-token-rotation).

Puedes habilitar y configurar dos ajustes de duración del token de actualización, la expiración máxima y la expiración por inactividad, mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0. Puedes usar una combinación de períodos de expiración máxima y por inactividad para lograr un equilibrio entre seguridad y experiencia de usuario que se adapte a las necesidades de tu empresa.

* **Duración máxima**: Establece la duración de un token de actualización o de una familia de tokens de actualización, tras la cual el usuario debe volver a autenticarse antes de que se le emita un nuevo token de acceso. Si deshabilitas esta configuración, la duración máxima será indefinida.
* **Duración por inactividad**: Establece la duración por inactividad de los tokens de actualización emitidos para que expiren si el usuario no está activo en tu aplicación durante un período determinado.

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Dashboard &gt; Applications](https://manage.auth0.com/#/applications).
2. Selecciona la aplicación que quieres configurar.
3. Ve a la pestaña **Settings**.
4. En **expiración del token de actualización**, habilita **Establecer duración inactiva del token de actualización**. Al habilitarla, el token de actualización expirará según la duración inactiva del token de actualización; después de ese tiempo, ya no podrá usarse. Si la rotación está habilitada, debes establecer una duración de expiración.

   <Frame>![Dashboard Applications Applications pestaña Settings expiración del token de actualización](/docs/images/cdy7uua7fh8z/2EnpIEc4kBxjnS3T67xPzC/22fdb95e2a2f31cee6f01e02d0a709c2/Refresh_Token_Expiration_-_English.png)</Frame>
5. Introduce **Duración inactiva del token de actualización** en segundos. El token de actualización expira después del intervalo especificado y ya no puede usarse para obtener un nuevo token de acceso. Cuando la rotación está habilitada, la duración inactiva del token de actualización también se aplica a la posibilidad de obtener nuevos tokens.

   | Duración       | Valor                        |
   | -------------- | ---------------------------- |
   | Predeterminado | 2,592,000 segundos (30 días) |
   | Mínimo         | 1 segundo                    |
   | Máximo         | 31,557,600 segundos (1 año)  |

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     * La **Duración inactiva del token de actualización** no puede superar la **Duración máxima del token de actualización**.
     * El cálculo de un año equivale a 365.25 días para tener en cuenta los años bisiestos.
   </Callout>
6. Habilita **Establecer duración máxima del token de actualización**. Al habilitarla, el token de actualización expirará según la duración máxima del token de actualización que especifiques; después de ese tiempo, ya no podrá usarse.
7. Introduce **Duración máxima del token de actualización** en segundos. Si el token de actualización no se intercambia dentro del intervalo especificado, expira y ya no puede usarse para obtener un nuevo token de acceso. El período de expiración de la duración inactiva del token de actualización se renueva cada vez que el token de actualización se intercambia por un nuevo token de acceso dentro de ese intervalo.

   | Duración | Valor                       |
   | -------- | --------------------------- |
   | Mínimo   | 1 segundo                   |
   | Máximo   | 31,557,600 segundos (1 año) |
8. Selecciona **Save Changes**.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Puedes configurar los valores de duración máxima y de inactividad en el payload del endpoint de la Management API [/api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id). Aquí tienes un ejemplo que establece la duración de expiración de un token de actualización sin rotación:

```json lines
PATCH /api/v2/clients/{id}
{
  "refresh_token": {
      "rotation_type": "non-rotating",
      "expiration_type": "expiring",
      "token_lifetime": 2592000,
      "infinite_token_lifetime": false,
      "idle_token_lifetime": 604800,
      "infinite_idle_token_lifetime": false
  }
}
```

<div id="support-and-limitations">
  ## Soporte y limitaciones
</div>

* La **duración máxima del token de actualización** se determina en el momento de la emisión y permanece fija durante toda su vigencia, incluso si se actualiza la duración máxima del token de actualización de la aplicación. Sin embargo, puede anular explícitamente la duración máxima del token de actualización en el momento de la emisión mediante [sesiones con Actions](/docs/es/manage-users/sessions/manage-sessions-actions), incluso durante intercambios exitosos de token de actualización.
* La **duración inactiva del token de actualización** refleja la configuración actual de la aplicación y se actualiza con cada intercambio exitoso de token de actualización.
* Si configura la Duración máxima del token de actualización para una aplicación, cualquier token de actualización sin expiración emitido por esa aplicación pasa a ser un token de actualización con expiración después de un intercambio exitoso de token de actualización.
* Una vez que un token de actualización sin expiración se convierte en un token de actualización con expiración, no puede volver a convertirse en un token de actualización sin expiración.
* Puede optar por usar las capacidades de expiración del token de actualización; no necesita hacer nada. Los tokens de actualización existentes no se ven afectados.
* La expiración del token de actualización funciona con los siguientes flujos:

  * [Flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow)
  * [Flujo de código de autorización con Proof Key for Code Exchange](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce)
  * [Flujo de autorización de dispositivos](/docs/es/get-started/authentication-and-authorization-flow/device-authorization-flow)
  * [Flujo de contraseña del propietario del recurso](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow)
* Todos los SDKs de Auth0 admiten la expiración del token de actualización.
* La funcionalidad de expiración del token de actualización cumple con las [recomendaciones de seguridad de OAuth 2.0 BCP](https://www.ietf.org/archive/id/draft-ietf-oauth-security-topics-16.html#section-4.13.2).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El OAuth BCP establece que los tokens de actualización emitidos para aplicaciones basadas en navegador deben tener expiración y aplicar restricciones del remitente o rotar los tokens con cada solicitud. Por lo tanto, las SPA usarán la rotación de forma predeterminada y **no admitirán** tokens de actualización sin expiración.
</Callout>
