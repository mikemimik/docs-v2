Configure la <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+token">rotación del token de actualización</Tooltip> para cada aplicación mediante el Dashboard o el [Auth0 SPA SDK](/docs/es/libraries/auth0-single-page-app-sdk). Cuando la <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+token+rotation">rotación del Refresh Token</Tooltip> está habilitada, la transición para el usuario se realiza sin interrupciones. La aplicación usa el token de actualización anterior, no rotatorio y aún vigente, y lo intercambia por un token de actualización rotatorio.

Los escenarios de migración permiten la revocación automática de tokens al migrar de un token de actualización no rotatorio a uno rotatorio y viceversa.

* **Cuando la rotación del Refresh Token está habilitada**:
  Al intercambiar un token de actualización no rotatorio, se emite un nuevo token de actualización rotatorio y se eliminan todos los tokens no rotatorios emitidos para el mismo `client_id`, <Tooltip tip="Servidor de recursos: servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/docs/es/glossary?term=resource+server">servidor de recursos</Tooltip>, usuario y tenant.
* **Cuando la rotación del Refresh Token está deshabilitada**:
  Al intercambiar un token de actualización rotatorio, se emite un token de actualización no rotatorio y se revoca la familia de tokens de actualización rotatorios emitida para el mismo `client_id`, servidor de recursos, usuario y tenant.

<div id="configure-in-the-dashboard">
  ## Configurar en el Dashboard
</div>

1. Ve a [Dashboard &gt; Applications](https://manage.auth0.com/#/applications).
2. Selecciona la aplicación que quieres configurar.
3. Ve a la pestaña **Settings**.
4. En **rotación del Refresh Token**, habilita **Allow rotación del Refresh Token**.

   <Frame>![Dashboard Applications Applications Settings Tab Refresh Token Rotation](/docs/images/cdy7uua7fh8z/Rk6i8MUVWZG34P7K6aurb/9457a6ea7ca5942522778c7ed7925023/Screenshot_2024-11-05_at_18.14.22.png)</Frame>
5. Introduce **Rotation Overlap Period** (en segundos) para el token de actualización, a fin de contemplar el margen de tiempo entre la solicitud y la respuesta antes de activar la detección automática de reutilización.
   Este período de superposición ayuda a evitar problemas de concurrencia al intercambiar el token de actualización rotativo varias veces dentro de un período determinado. Durante la ventana de tolerancia, las funcionalidades de detección de compromiso no se aplican y se emite un nuevo token de actualización rotativo. Solo se puede reutilizar el token anterior; si se intercambia el penúltimo, se activará la detección de compromiso.
6. Selecciona **Save Changes**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  rotación del Refresh Token solo es compatible con aplicaciones conformes con OIDC que tengan habilitado el tipo de concesión de token de actualización.
</Callout>

<div id="configure-with-the-auth0-spa-sdk">
  ## Configura con el Auth0 SPA SDK
</div>

Puedes usar el Auth0 SPA SDK para habilitar la rotación del token de actualización. Debes habilitar el acceso sin conexión y solicitar el scope `offline_access` en el SDK del cliente.

1. Instala la versión más reciente del SDK `auth0-spa-js`:
   `npm install @auth0/auth0-spa-js`

2. Habilita la funcionalidad en el SDK estableciendo `useRefreshTokens: true` para empezar a enviar el scope `offline_access`.

   {/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

   <AuthCodeBlock children={codeExample} language="javascript" />

3. Configura los ajustes de rotación del token de actualización. Por ejemplo:

   ```http lines
   PATCH /api/v2/clients/{client_id}
   {
     "refresh_token": {
       "rotation_type": "rotating",
       "expiration_type": "expiring",
       "token_lifetime": "2592000",
       "leeway": 3
     }
   }
   ```

   | Atributo          | Descripción                                                                                                                                                                                                                                                                                                                              |
   | ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | `rotation_type`   | Cadena de texto: &quot;rotating&quot; o &quot;non-rotating&quot;                                                                                                                                                                                                                                                                         |
   | `expiration_type` | Cadena de texto: &quot;expiring&quot; o &quot;non-expiring&quot;                                                                                                                                                                                                                                                                         |
   | `token_lifetime`  | El período predeterminado de expiración del token de actualización, cuando la **rotación del token de actualización** está habilitada, es de 30 días (2.592.000 segundos). Puedes configurar hasta 1 año (31.557.600 segundos). **La vigencia no se amplía cuando se rotan los tokens.**                                                          |
   | `leeway`          | Permite usar el mismo token de actualización dentro de un período determinado para contemplar posibles problemas de concurrencia de red que, de otro modo, invalidarían el token si el cliente intentara volver a usar ese mismo token de actualización. De forma predeterminada, `leeway` está deshabilitado. Se configura en segundos. |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La rotación del token de actualización solo es compatible con aplicaciones conformes con OIDC que tienen habilitado el tipo de concesión de token de actualización.
</Callout>

<div id="automatic-reuse-detection">
  ## Detección automática de reutilización
</div>

Si se utiliza un token previamente invalidado, se revocará de inmediato todo el conjunto de tokens de actualización emitidos desde la emisión de ese token invalidado, junto con la concesión, lo que obligará al usuario a volver a autenticarse.
