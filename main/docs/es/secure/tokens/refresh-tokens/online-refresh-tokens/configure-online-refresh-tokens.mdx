<div id="configure-orts">
  ## Configurar ORTs
</div>

Online Refresh Tokens (ORTs) deben habilitarse a nivel de la [API (Resource Server)](/docs/es/get-started/apis/api-settings) desde el Auth0 Dashboard o mediante la Management API.

<div id="configure-using-the-dashboard">
  ### Configurar mediante el Dashboard
</div>

1. Ve a [Dashboard &gt; Applications &gt; API](https://manage.auth0.com/#/apis).

2. Selecciona la API que quieres configurar.

3. En la pestaña **Settings**, habilita la opción **Allow Online Access**.

   <Frame>![Dashboard Applications APIs Settings Allow Online Access](/docs/images/refresh-tokens/allow-online-access.png)</Frame>

4. Selecciona **Save**.

Para revisar y configurar los valores de tiempo de espera por inactividad y duración máxima absoluta de las sesiones, consulta la configuración de [Expiración de sesión del tenant](https://manage.auth0.com/#/tenant/advanced). Para obtener más información, consulta [Configurar la duración de la sesión](/docs/es/manage-users/sessions/configure-session-lifetime).

<div id="configure-using-the-management-api">
  ### Configurar mediante la Management API
</div>

Para habilitar los ORT, haz una solicitud `PATCH` al endpoint [Update Resource Server](/docs/es/api/management/v2/resource-servers/patch-resource-servers-by-id):

```bash
curl --request PATCH \
  --url 'https://<YOUR_DOMAIN>/api/v2/resource-servers/<YOUR_RESOURCE_SERVER_ID>' \
  --header 'authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
  --header 'content-type: application/json' \
  --data '{"allow_online_access": true}'
```

Para verificar la configuración, haz una solicitud `GET` al servidor de recursos:

```bash
curl --request GET \
  --url 'https://<YOUR_DOMAIN>/api/v2/resource-servers/<YOUR_RESOURCE_SERVER_ID>' \
  --header 'authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>'
```

La respuesta incluye la propiedad `allow_online_access`:

```json
{
  "id": "resource-server-id",
  "name": "My API",
  "identifier": "https://my-api.example.com",
  "allow_online_access": true,
  ...
}
```

<div id="get-an-ort">
  ## Obtener un ORT
</div>

Para obtener un ORT, incluye el `scope` [online&#95;access](/docs/es/get-started/apis/scopes) en tu solicitud de autorización mediante el [flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow) o el [flujo de código de autorización con PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

```bash
https://<YOUR_DOMAIN>/authorize \
    audience=<YOUR_API_AUDIENCE> \
    scope=openid profile online_access \
    response_type=code \
    client_id=<YOUR_CLIENT_ID> \
    redirect_uri=<https://YOUR_APP/callback> \
    state=<OPAQUE_VALUE>
```

Una vez que el usuario se autentique, intercambia el código de autorización por tokens:

```bash
curl --request POST \
  --url 'https://<YOUR_DOMAIN>/oauth/token' \
  --header 'content-type: application/x-www-form-urlencoded' \
  --data grant_type=authorization_code \
  --data 'client_id=<YOUR_CLIENT_ID>' \
  --data 'code=<YOUR_AUTHORIZATION_CODE>' \
  --data 'redirect_uri=<https://YOUR_APP/callback>' \
  --data 'code_verifier=<YOUR_CODE_VERIFIER>'
```

La respuesta del token incluye el ORT:

```json
{
  "access_token": "eyJ...",
  "id_token": "eyJ...",
  "refresh_token": "ORT...",
  "token_type": "Bearer",
  "expires_in": 86400
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los ORT llevan el prefijo `ORT` para distinguirlos de los tokens de actualización. Sin embargo, debes tratar el token como opaco y no depender de su estructura interna.
</Callout>

<div id="use-an-ort">
  ## Usa un ORT
</div>

Intercambia el ORT por un nuevo [token de acceso](/docs/es/secure/tokens/access-tokens) cuando el token de acceso actual caduque o esté a punto de caducar:

```bash
curl --request POST \
  --url 'https://<YOUR_DOMAIN>/oauth/token' \
  --header 'content-type: application/x-www-form-urlencoded' \
  --data grant_type=refresh_token \
  --data 'client_id=<YOUR_CLIENT_ID>' \
  --data 'refresh_token=<YOUR_ONLINE_REFRESH_TOKEN>'
```

La respuesta contiene un nuevo token de acceso.
Si tu solicitud incluye el scope `openid`, la respuesta también incluye un nuevo [token de identidad](/docs/es/secure/tokens/id-tokens):

```json
{
  "access_token": "eyJ...",
  "id_token": "eyJ...",
  "token_type": "Bearer",
  "expires_in": 86400
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no emite un nuevo `refresh_token` y, como los ORT no se rotan, sigues usando el mismo ORT en los intercambios posteriores.
</Callout>

<div id="session-extension-behavior">
  ### Comportamiento de la extensión de la sesión
</div>

Cada intercambio de tokens exitoso:

* **Restablece el tiempo de espera por inactividad de la sesión:** El tiempo de espera por inactividad vuelve a su duración completa.
* **No extiende la duración máxima absoluta:** El vencimiento absoluto de la sesión no cambia.
* **Mantiene el SSO:** Otras aplicaciones pueden obtener tokens mediante SSO mientras la sesión siga activa.

<div id="revoke-an-ort">
  ## Revocar un ORT
</div>

Cuando revocas un ORT, se cierra toda la sesión de Auth0, no solo el token. Esto invalida todos los ORT vinculados a esa sesión y pone fin al SSO del usuario.

```bash
curl --request POST \
  --url 'https://<YOUR_DOMAIN>/oauth/revoke' \
  --header 'content-type: application/x-www-form-urlencoded' \
  --data 'client_id=<YOUR_CLIENT_ID>' \
  --data 'token=<YOUR_ONLINE_REFRESH_TOKEN>'
```

<div id="use-orts-with-actions">
  ## Usa ORTs con Actions
</div>

Puedes usar ORTs con Auth0 [Actions](/docs/es/customize/actions/actions-overview) y el [trigger Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

Con Actions, puedes:

* Usar el objeto `event.refresh_token` para determinar si el token es un ORT.
* Acceder a datos específicos de la sesión mediante el objeto `event.session` para tomar decisiones según el estado actual de la sesión

```javascript
exports.onExecutePostLogin = async (event, api) => {
  // Verificar si el token es un token de actualización en línea
  
  if (event.refresh_token?.access == 'online') {
    // El token es un ORT, puedes referenciar event.session y api.session
    console.log('Exchanging Online Refresh Token bound to Session ID: ', event.session?.id);
    // Obtener los metadatos de la sesión y agregarlos a los tokens
    // Suponiendo que los metadatos de la sesión se almacenaron previamente
    const importantInformation = event.session?.metadata?.importantInformation;
    api.accessToken.setCustomClaim('info', importantInformation);
    api.idToken.setCustomClaim('info', importantInformation);
  }
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 revoca la sesión completa del usuario en lugar de solo el token cuando se usa el método `api.refreshToken.revoke()` con un ORT.
</Callout>
