Puedes revocar <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> si se ven comprometidos. Auth0 gestiona la revocación de tokens como si el token hubiera quedado potencialmente expuesto a actores maliciosos.

Además, puedes considerar habilitar la [rotación del Refresh Token](/docs/es/secure/tokens/refresh-tokens/refresh-token-rotation) para que, cada vez que un cliente intercambie un token de actualización para obtener un nuevo token de acceso, también reciba un nuevo token de actualización.

<div id="choose-whether-token-revocation-deletes-grants">
  ## Elige si la revocación de tokens elimina las concesiones
</div>

Una concesión proporciona a una aplicación acceso a un recurso en otra entidad sin exponer las credenciales del usuario. Los tokens se emiten en el contexto de una concesión.

Cuando se revoca una concesión, también se revocan todos los tokens emitidos en el contexto de esa concesión. Sin embargo, cuando se revoca un token, no necesariamente hay que revocar la concesión.

Puedes elegir si la revocación de tokens elimina la concesión asociada en la configuración de tu tenant:

1. Ve a [Dashboard &gt; Configuración del tenant &gt; Advanced](https://manage.auth0.com/#/tenant/advanced).

2. En la sección **Settings**, habilita o deshabilita el toggle **Refresh Token Revocation Deletes Grant**.

   * Habilita el toggle para eliminar la concesión subyacente cuando revoques el token de actualización.

   Cada solicitud de revocación invalida todos los demás tokens basados en la misma concesión de autorización, lo que revoca todos los tokens de actualización emitidos para el mismo usuario, aplicación y audience.

   * Deshabilita el toggle para conservar la concesión subyacente cuando revoques el token de actualización.

   Cuando se desvincula un device, solo se revoca el token de actualización asociado, y la concesión permanece intacta.

Esta funcionalidad está deshabilitada de forma predeterminada para todos los tenants creados después del 13 de enero de 2021.

<div id="revoke-a-refresh-token">
  ## Revocar un token de actualización
</div>

Puedes revocar un token de actualización de las siguientes formas:

* [Desde el Auth0 Dashboard](#using-the-auth0-dashboard)
* [Mediante la Authentication API](#using-the-authenticaton-api)
* [Mediante la Management API](#using-the-management-api)

<div id="using-the-auth0-dashboard">
  ### Uso de Auth0 Dashboard
</div>

Puede usar el Auth0 Dashboard para revocar el acceso autorizado de un usuario a la aplicación que emitió el token. Esto invalida el token de actualización, lo que en la práctica es equivalente a revocar el propio token.

1. Vaya a [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y seleccione el nombre del usuario que quiere ver.
2. Seleccione la pestaña **Authorized Applications**. Esta página muestra todas las aplicaciones a las que el usuario ha autorizado el acceso.
3. Para revocar el acceso del usuario a una aplicación autorizada y, por lo tanto, invalidar el token de actualización, haga clic en **Revoke**.

<div id="using-the-authentication-api">
  ### Uso de la Authentication API
</div>

Puedes usar el [endpoint Revoke a refresh token](https://auth0.com/docs/api/authentication/revoke-refresh-token/revoke-refresh-token) de la Authentication API para revocar un token de actualización.

El endpoint permite el acceso sin el <Tooltip tip="Secreto de cliente: Secret usado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip> en aplicaciones que no pueden mantenerlo seguro (como las aplicaciones nativas). La propia aplicación debe tener su [método de autenticación](/docs/es/get-started/applications/credentials) (`tokenEndpointAuthMethod`) configurado como none.

Cuando haces una solicitud, la API primero valida las credenciales de la aplicación y luego verifica si el token se emitió para la aplicación que realiza la solicitud de revocación. Si esta validación falla, la solicitud se rechaza y se informa del error a la aplicación.

A continuación, la API invalida el token. La invalidación se produce de inmediato y el token no puede volver a usarse después de la revocación. Cada solicitud de revocación invalida todos los tokens que se han emitido para la misma concesión de autorización.

Para obtener más información, incluidos los parámetros del cuerpo, los esquemas de respuesta y ejemplos de código, consulta la documentación de la Authentication API del [endpoint Revoke a refresh token](https://auth0.com/docs/api/authentication/revoke-refresh-token/revoke-refresh-token).

<div id="using-the-management-api-device-credentials-endpoints">
  ### Uso de los endpoints de credenciales de dispositivo de la Management API
</div>

Para revocar un token de actualización mediante los endpoints de credenciales de dispositivo de Auth0 Management API:

1. Obtén el `id` del token de actualización que quieres revocar mediante el [endpoint de recuperación de credenciales de dispositivo](https://auth0.com/docs/api/management/v2/device-credentials/get-device-credentials).

   Especifica `type=refresh_token` y un `user_id` con un token de acceso que incluya el `scope` `read:device_credentials`. Para acotar los resultados, también puedes especificar el `client_id` asociado al token (si lo conoces).

2. Revoca el token de actualización mediante el [endpoint para eliminar una credencial de dispositivo](https://auth0.com/docs/api/management/v2/device-credentials/delete-device-credentials-by-id). Especifica un token de acceso que incluya el `scope` `delete:device_credentials` y el ID del token de actualización.

<div id="using-the-management-api-refresh-token-endpoints">
  ### Uso de los endpoints de token de actualización de la Management API
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Actualmente, los endpoints de token de actualización de Auth0 Management API solo están disponibles para clientes con planes Enterprise. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Callout>

Los endpoints de token de actualización de la Management API sustituyen a los endpoints del recurso `/v2/device-credentials`, con propiedades ampliadas y operaciones de revocación masiva.

Puedes ver o revocar un token de actualización específico con los siguientes endpoints:

* Consulta la información de un token de actualización por ID con el [endpoint Obtener un token de actualización](https://auth0.com/docs/api/management/v2/refresh-tokens/get-refresh-token)
* Revoca un token de actualización por ID con el [endpoint Eliminar un token de actualización](https://auth0.com/docs/api/management/v2/refresh-tokens/delete-refresh-token)
* Enumera todos los tokens de actualización de un usuario con el [endpoint Obtener los tokens de actualización de un usuario](https://auth0.com/docs/api/management/v2/users/get-refresh-tokens-for-user)
* Revoca todos los tokens de actualización de un usuario con el [endpoint Eliminar los tokens de actualización de un usuario](https://auth0.com/docs/api/management/v2/users/delete-refresh-tokens-for-user)

<div id="considerations-and-limitations">
  ## Consideraciones y limitaciones
</div>

* Con el [Device Authorization Flow](/docs/es/get-started/authentication-and-authorization-flow/device-authorization-flow), la única forma de forzar que un device se vuelva a autorizar es revocar el token de actualización asignado al device. No se obligará al device a volver a autorizarse hasta que caduque el token de acceso actual y la aplicación intente usar el token de actualización revocado. Para obtener más información, consulta [Desvincular devices de los usuarios](/docs/es/manage-users/user-accounts/unlink-devices-from-users).

* Al usar [rotación del Refresh Token](/docs/es/secure/tokens/refresh-tokens/refresh-token-rotation), si se usa un token invalidado previamente, se revocará de inmediato todo el conjunto de tokens de actualización emitidos desde la emisión de ese token invalidado, lo que requerirá que el usuario final vuelva a autenticarse.

* Las operaciones de revocación (`DELETE`) de tokens de actualización se ejecutan de forma asíncrona y tienen consistencia eventual.

* Los tokens de actualización emitidos a partir del 21-09-2023 (22-02-2024 para tenants en la región US-3) contienen la propiedad de ID de sesión (`session_id`) con el valor correspondiente. Los tokens de actualización emitidos antes de esta fecha contienen esta propiedad con un valor `null`.

Para obtener más información sobre las limitaciones de los tokens de actualización, consulta [Limitaciones de los tokens de actualización](/docs/es/secure/tokens/refresh-tokens).
