<Warning>
  Los metadatos del token de actualización están actualmente en acceso anticipado y solo disponibles para clientes empresariales. Al usar esta funcionalidad, aceptas los términos aplicables de la prueba gratuita del [Master Subscription Agreement](https://www.okta.com/legal) de Okta. Para obtener más información sobre el ciclo de lanzamiento de productos de Auth0, consulta [Etapas de lanzamiento del producto](https://auth0.com/docs/troubleshoot/product-lifecycle/product-release-stages).
</Warning>

Para configurar los metadatos del token de actualización, puedes usar una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) de Auth0 y la [Management API](https://auth0.com/docs/api/management/v2).

<div id="auth0-post-login-actions">
  ## Actions de Post Login de Auth0
</div>

Puedes gestionar operaciones CRUD sobre los metadatos del token de actualización con los objetos `api.refreshToken` mediante una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger). Esto te permite gestionar los metadatos del token de actualización según la lógica específica del usuario o del contexto.

### Recupera los metadatos existentes del token de actualización

Usa el objeto `event.refresh_token.metadata` para consultar los metadatos del token de actualización:

```javascript lines
const deviceName = event.refresh_token?.metadata?.deviceName;
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El objeto `event.refresh_token.metadata` incluye metadatos establecidos en:

  * Actions anteriores dentro del mismo flujo
  * Transacciones previas si el token de actualización se reutilizó durante un intercambio de token de actualización
</Callout>

<div id="add-or-update-existing-metadata">
  ### Agrega o actualiza los metadatos existentes
</div>

Usa el método `api.refreshToken.setMetadata()` para establecer o actualizar los metadatos del token de actualización:

```javascript lines
api.refreshToken.setMetadata("deviceName", "Auth0's iPhone");
```

Los cambios están disponibles de inmediato en el objeto `event.refresh_token` en las Actions siguientes.

<div id="delete-refresh-token-metadata">
  ### Eliminar los metadatos del token de actualización
</div>

Usa los siguientes métodos de `api.refreshToken` para eliminar los metadatos del token de actualización:

* `api.refreshToken.deleteMetadata("key")` elimina los metadatos especificados del token de actualización

* `api.refreshToken.evictMetadata()` elimina todos los metadatos del token de actualización

Para obtener más información sobre estos objetos, consulta:

* [objeto Event](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object): Obtén más información sobre el objeto Event del token de actualización y sus propiedades.
* [objeto de API](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object): Obtén más información sobre el objeto de API del token de actualización y sus métodos.

<div id="auth0-management-api">
  ## Auth0 Management API
</div>

Puedes gestionar las solicitudes CRUD (crear, reemplazar, actualizar y eliminar) de los metadatos del token de actualización mediante la Management API:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las llamadas al endpoint `/api/v2/refresh-tokens/{id}` requieren un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens) con el `scope` `update:refresh_tokens`.
</Callout>

<div id="retrieve-existing-refresh-token-metadata">
  ### Recuperar los metadatos de un token de actualización existente
</div>

Haz una solicitud `GET` al endpoint [`/api/v2/refresh-tokens/{id}`](https://auth0.com/docs/api/management/v2/refresh-tokens/get-refresh-token):

```bash lines
GET /api/v2/refresh-tokens/{id}
```

<div id="add-or-update-existing-refresh-token-metadata">
  ### Agregar o actualizar los metadatos de un token de actualización existente
</div>

Realiza una solicitud `PATCH` al endpoint [`/api/v2/refresh-tokens/{id}`](https://auth0.com/docs/api/management/v2/refresh-tokens/patch-refresh-token):

```bash lines
PATCH /api/v2/refresh-tokens/{id}
Content-Type: application/json

{
  "refresh_token_metadata": {
    "my_metadata": "my new metadata"
  }
}
```

<div id="delete-refresh-token-metadata">
  ### Eliminar los metadatos del token de actualización
</div>

Haz una petición `PATCH` al endpoint [`/api/v2/refresh-tokens/{id}`](https://auth0.com/docs/api/management/v2/refresh-tokens/patch-refresh-token) con un objeto de metadatos vacío:

```bash lines
PATCH /api/v2/refresh-tokens/{id}
Content-Type: application/json

{
  "refresh_token_metadata": {}
}
```

<div id="use-case-store-and-consume-organization-context">
  ## Caso de uso: Almacenar y consumir el contexto de la organización
</div>

Puedes usar los metadatos del token de actualización para almacenar el contexto de la organización durante la autenticación inicial y usarlo más adelante durante los intercambios de token de actualización. Esto resulta útil para sistemas de destino, como los flujos de auditoría, análisis y revocación.

<div id="set-metadata-at-initial-authentication">
  ### Establece metadatos en la autenticación inicial
</div>

Durante la autenticación inicial, establece el contexto de la organización en los metadatos del token de actualización:

```javascript lines
/**
 * Action de Post Login
 * Agrega el contexto de organización a los metadatos del token de actualización durante la autenticación inicial.
 * Estos metadatos estarán disponibles en los intercambios de token de actualización posteriores.
 */
exports.onExecutePostLogin = async (event, api) => {
  // Continuar solo si la transacción tiene como destino una organización
  if (!event.organization) return;

  // Mantener los valores cortos y solo de tipo cadena (los metadatos del token de actualización requieren cadenas de texto)
  const orgId = String(event.organization.id || "");
  const orgSlug = String(event.organization.name || "");
  const orgDisplay = String(event.organization.display_name || orgSlug);

  // Establecer metadatos: se almacenarán junto con el token de actualización
  api.refreshToken.setMetadata("org_id", orgId);
  api.refreshToken.setMetadata("org_slug", orgSlug);
  api.refreshToken.setMetadata("org_name", orgDisplay);
};
```

<div id="consume-metadata-during-refresh-token-exchange">
  ### Usar metadata durante el intercambio de token de actualización
</div>

Durante un intercambio de token de actualización, el objeto `event.refresh_token` está disponible y puedes leer la metadata almacenada anteriormente:

```javascript lines
/**
 * Action de Post Login
 * Consume el contexto de organización desde los metadatos del token de actualización durante el intercambio de tokens.
 * Úsalo para aplicar políticas o agregar claims según el contexto de autenticación original.
 */
exports.onExecutePostLogin = async (event, api) => {
  // Verifica si se trata de un intercambio de token de actualización (event.refresh_token existe)
  if (!event.refresh_token) return;

  // Lee el contexto de organización almacenado durante la autenticación inicial
  const orgId = event.refresh_token.metadata?.org_id;
  const orgSlug = event.refresh_token.metadata?.org_slug;

  if (orgId) {
    // Usa el contexto de organización para lógica condicional
    console.log(`Refresh token exchange for organization: ${orgSlug}`);
    
    // Ejemplo: Agrega custom claims según el contexto de organización original
    api.accessToken.setCustomClaim("org_id", orgId);
  }
};
```

<div id="retrieve-via-management-api">
  ### Recuperar con Management API
</div>

También puedes consultar los metadatos del token de actualización mediante el endpoint [/api/v2/refresh-tokens/{id}](https://auth0.com/docs/api/management/v2/refresh-tokens/get-refresh-token):

```bash lines
GET /api/v2/refresh-tokens/{id}
```

Respuesta de ejemplo:

```json lines
{
  "refresh_token_metadata": {
    "org_id": "org_abc123",
    "org_slug": "acme",
    "org_name": "Acme Corp"
  }
}
```

<div id="use-case-track-and-validate-device-information">
  ## Caso de uso: Registrar y validar la información del dispositivo
</div>

Puede usar los metadatos del token de actualización para capturar información del dispositivo durante la autenticación inicial y validarla durante los intercambios posteriores de token de actualización por motivos de seguridad.

<div id="set-device-information-at-initial-authentication">
  ### Configurar la información del dispositivo en la autenticación inicial
</div>

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  // Solo establecer información del dispositivo si NO es un intercambio de token de actualización
  // (es decir, esta es la autenticación inicial)
  if (event.refresh_token) return;

  // Almacenar información del dispositivo en los metadatos del token de actualización
  api.refreshToken.setMetadata("initial_ip", event.request?.ip || "unknown");
  api.refreshToken.setMetadata("initial_asn", event.request?.asn?.value || "unknown");
  api.refreshToken.setMetadata("initial_country", event.request?.geoip?.country_code || "unknown");
  api.refreshToken.setMetadata("initial_user_agent", event.request?.user_agent || "unknown");
};
```

<div id="validate-device-information-during-refresh-token-exchange">
  ### Validar la información del dispositivo durante el intercambio de token de actualización
</div>

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  // Solo validar si se trata de un intercambio de token de actualización
  if (!event.refresh_token) return;

  // Leer la información original del dispositivo
  const initialCountry = event.refresh_token.metadata?.initial_country;
  const currentCountry = event.request?.geoip?.country_code;

  // Ejemplo: Detectar cambio de país y tomar acción
  if (initialCountry && currentCountry && initialCountry !== currentCountry) {
    console.log(`Country changed from ${initialCountry} to ${currentCountry}`);
    
    // Opción 1: Revocar el token de actualización por actividad sospechosa
    // api.refreshToken.revoke("Suspicious country change detected");
    
    // Opción 2: Actualizar los metadatos para registrar el cambio
    api.refreshToken.setMetadata("country_changed", "true");
    api.refreshToken.setMetadata("last_country", currentCountry);
  }
};
```

<div id="error-handling">
  ## Manejo de errores
</div>

Puede revisar los eventos de log de los metadatos del token de actualización en [Dashboard &gt; monitoreo &gt; Logs](https://manage.auth0.com/#/logs) o recuperar registros mediante el endpoint de [logs de Management API](/docs/es/api/management/v2/logs/get-logs).

* Si se produce un error al agregar o actualizar los metadatos del token de actualización con Actions, la transacción de autenticación falla y se devuelve un error a la URL de devolución de llamada.

Se registra un código de evento de error `f` con su error correspondiente:

```json lines
{
  "error": "access_denied",
  "error_description": "Failed to set refresh token metadata: Invalid metadata: Metadata keys may only include letters, numbers, underscores, or hyphens",
  "state": "my-custom-state"
}
```

* Si se produce un error al gestionar los metadatos del token de actualización mediante la Auth0 Management API, la API responde con un error `HTTP status: 400` y su mensaje correspondiente:

```json lines
{
  "statusCode": 400,
  "message": "Metadata must not exceed 25 entries. Each key and value must be ≤ 255 characters."
}
```
