Auth0 usa <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, formato de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token+%28JWT%29">JSON Web Token (JWT)</Tooltip> para la transmisión segura de datos, la autenticación y la autorización. Los tokens deben analizarse y validarse en aplicaciones web tradicionales, nativas y de página única para asegurarse de que el token no esté comprometido y de que la firma sea auténtica. Los tokens deben verificarse para reducir los riesgos de seguridad si, por ejemplo, el token ha sido manipulado, usado de forma indebida o ha caducado. La validación de <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, formato de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> comprueba la estructura, los claims y la firma para minimizar el riesgo.

La firma del token JWT se genera con un <Tooltip tip="Algoritmo de firma: algoritmo usado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=Signing+Algorithm">algoritmo de firma</Tooltip>. Aunque los tokens pueden usar varios <Tooltip tip="Algoritmo de firma: algoritmo usado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=signing+algorithms">algoritmos de firma</Tooltip>, Auth0 admite RS256, cifrado RSA con función hash SHA-256, o HS256, código de autenticación de mensajes HMAC (MAC) con SHA-256. Para obtener más información sobre el algoritmo recomendado por Auth0, lee [Algoritmos de firma](/docs/es/get-started/applications/signing-algorithms).

Al validar un JWT, por lo general se analizan, o se decodifican, el valor hash actual y el valor hash original, y luego se comparan para verificar que la firma del token sea auténtica. Todos nuestros [quickstarts de backend API](/docs/es/quickstart/backend) usan SDK que realizan por ti la validación y el análisis de JWT.

<div id="parse-and-validate">
  ## Analizar y validar
</div>

Si no usa uno de nuestros SDKs que realizan la validación y el análisis de JWT automáticamente, puede analizar y validar un JWT de la siguiente manera:

* Usar cualquier middleware existente para su framework web.
* Elegir una biblioteca de terceros de JWT.io.
* Implementar manualmente las comprobaciones descritas en [specification RFC 7519 &gt; 7.2 Validating a JWT](https://tools.ietf.org/html/rfc7519#section-7.2).

Le recomendamos encarecidamente que use middleware o alguna de las bibliotecas de terceros de código abierto ya disponibles para analizar y validar JWT. En [JWT.io](https://jwt.io/#libraries-io), puede encontrar bibliotecas para varias plataformas y lenguajes, como .NET, Python, Java, Ruby, Objective-C, Swift y PHP.

<div id="middleware">
  ## Middleware
</div>

Muchos frameworks web, como [ASP.NET Core](/docs/es/quickstart/backend/aspnet-core-webapi), incluyen middleware de JWT que se encarga de validar los JWT. Por lo general, esta es la mejor opción, ya que el middleware se integra bien con los mecanismos generales de authentication del framework.

<div id="third-party-libraries">
  ## Bibliotecas de terceros
</div>

Si eliges una biblioteca de terceros, asegúrate de que sea compatible con el algoritmo de firma que seleccionaste al registrar tu aplicación o API con Auth0. Además, ten en cuenta que no todas las bibliotecas validan todos los claims de JWT. En JWT.io, puedes ver qué validaciones admite cada biblioteca (busca las marcas de verificación verdes).

La mayoría de las bibliotecas de terceros implementan un método para verificar un JWT e incluyen varios argumentos que te permiten personalizar la verificación. Por ejemplo, si usas Node.js y la [biblioteca node-jsonwebtoken](https://github.com/auth0/node-jsonwebtoken), llamarías al método [jwt.verify()](https://github.com/auth0/node-jsonwebtoken#jwtverifytoken-secretorpublickey-options-callback). Este método admite un argumento `algorithms` para personalizar los algoritmos permitidos (asegúrate de deshabilitar `none`), un argumento `secretOrPublicKey` que debes completar con el secreto o la clave pública RSA (según el algoritmo de firma seleccionado), y otros argumentos de entrada que te permiten personalizar la validación de claims. Si el análisis falla, la biblioteca devuelve un [error JsonWebTokenError](https://github.com/auth0/node-jsonwebtoken#jsonwebtokenerror) con el mensaje `jwt malformed`, tras lo cual **debes** rechazar la solicitud asociada.

Recomendaciones generales para usar bibliotecas de terceros:

* Para obtener claims de un JWT, usa el método `verify()` para validar los claims y la firma. Evita usar el método `decode()` para validar un token, especialmente si proviene de un <Tooltip tip="Cliente público: Cliente (aplicación) que no puede almacenar credenciales de forma segura. Algunos ejemplos son una aplicación nativa de escritorio o móvil y una aplicación web del lado del cliente basada en JavaScript (como una aplicación web de página única (SPA))." cta="Ver glosario" href="/docs/es/glossary?term=public+client">cliente público</Tooltip>.
* Sigue atentamente todas las instrucciones sobre cómo usar la biblioteca elegida. La biblioteca podría depender de valores o configuraciones predeterminados que podrían generar riesgos de seguridad.

<div id="manually-implement-checks">
  ## Implementa comprobaciones manuales
</div>

Desaconsejamos validar JWT manualmente, ya que es fácil implementarlo de forma incorrecta y pasar por alto detalles importantes que podrían dar lugar a vulnerabilidades de seguridad graves. La mayoría de las bibliotecas de JWT se encargan de la validación por ti. Visita JWT.io para encontrar una biblioteca de JWT para tu plataforma y lenguaje de programación.

Para obtener instrucciones sobre cómo validar manualmente un JWT, consulta [RFC 7519](https://tools.ietf.org/html/rfc7519#section-7.2). Todos los JWT emitidos por Auth0 tienen una JSON Web Signature (JWS), lo que significa que están firmados y no cifrados.

<div id="verify-rs256-signed-tokens">
  ## Verificar tokens firmados con RS256
</div>

Para verificar visualmente tokens firmados con RS256:

1. Ve a [Dashboard &gt; Applications](https://manage.auth0.com/#/applications).
2. Ve a la vista **Settings** y abre **Advanced Settings**.
3. Ve a la vista **Certificates**, localiza el campo **Signed Certificate** y copia la **clave pública**.
4. Ve al sitio web [JWT.io](https://jwt.io/), localiza el menú desplegable **Algorithm** y selecciona **RS256**.
5. Localiza la sección **Verify Signature** y pega la clave pública que copiaste anteriormente en el campo que comienza con `-----BEGIN PUBLIC KEY-----`, sustituyendo su contenido.

Para verificar la firma de un token de una de tus aplicaciones:

Te recomendamos obtener la clave pública del JWKS de tu tenant aquí:
`https://{yourDomain}/.well-known/jwks.json`
