Cada propiedad de la clave está definida por la especificación JWK [RFC 7517 Section 4](https://tools.ietf.org/html/rfc7517#section-4) o, en el caso de las propiedades específicas de algoritmos, por [RFC 7518](https://tools.ietf.org/html/rfc7518).

| Nombre de la propiedad | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `alg`                  | El algoritmo criptográfico específico que se utiliza con la clave. Este es un parámetro opcional. De forma predeterminada, Auth0 incluye en el JSON Web Key Set (JWKS) el algoritmo de firma definido a nivel del tenant, que luego se publica. Para permitir que las claves se usen con varios algoritmos en lugar de uno solo, es decir, RS256, desactive **Include Signing Algorithms in JSON Web Key Set** en Configuración avanzada del tenant dentro del Dashboard. Esto elimina el parámetro `alg` y requiere que los consumidores del JWKS interpreten los algoritmos de firma según corresponda. |
| `kty`                  | La familia de algoritmos criptográficos que se utiliza con la clave.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `use`                  | El uso previsto de la clave; `sig` representa la firma.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `x5c`                  | La cadena de certificados x.509. La primera entrada del arreglo es el certificado que debe usarse para verificar el token; los demás certificados pueden usarse para verificar este primer certificado.                                                                                                                                                                                                                                                                                                                                                                                                   |
| `n`                    | El módulo de la [clave pública RSA](https://tools.ietf.org/html/rfc7518#page-30).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `e`                    | El exponente de la [clave pública RSA](https://tools.ietf.org/html/rfc7518#page-30).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `kid`                  | El identificador único de la clave.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `x5t`                  | La huella digital del certificado x.509 (huella digital SHA-1).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 solo admite HMAC y RSA; actualmente no admite criptografía de curva elíptica (ECDSA).
</Callout>
