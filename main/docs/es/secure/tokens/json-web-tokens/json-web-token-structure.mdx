Todos los <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, del token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWTs">JWT</Tooltip> emitidos por Auth0 tienen [JSON Web Signatures (JWS)](https://tools.ietf.org/html/rfc7515), lo que significa que están firmados en lugar de cifrados. Un JWS representa contenido protegido con <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, del token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=digital+signatures">firmas digitales</Tooltip> o códigos de autenticación de mensajes (MAC) mediante estructuras de datos basadas en JSON.

Un JWT bien formado consta de tres cadenas codificadas en Base64url concatenadas, separadas por puntos (`.`):

* **encabezado JOSE**: contiene metadatos sobre el tipo de token y los algoritmos criptográficos usados para proteger su contenido.
* **payload de JWS** (conjunto de [claims](https://tools.ietf.org/html/rfc7519#section-4)): contiene afirmaciones de seguridad verificables, como la identidad del usuario y los permisos que tiene asignados.
* **firma JWS**: se usa para validar que el token es confiable y no ha sido manipulado. Cuando uses un JWT, **debes** [comprobar su firma](/docs/es/secure/tokens/json-web-tokens/validate-json-web-tokens) antes de almacenarlo y usarlo.

<div id="jose-header">
  ## Encabezado JOSE
</div>

Objeto JSON que contiene los parámetros que describen las operaciones y los parámetros criptográficos utilizados. El encabezado JOSE (JSON Object Signing and Encryption) se compone de un conjunto de parámetros de encabezado que normalmente consisten en un par nombre/valor: el algoritmo de firma utilizado (p. ej., HMAC SHA256 o RSA) y el tipo de JWT.

```json lines
{
      "alg": "HS256",
      "typ": "JWT"
    }
```

<div id="jws-payload">
  ## Payload de JWS
</div>

El payload contiene información sobre la entidad (normalmente, el usuario) y atributos adicionales de la entidad, llamados claims. En este ejemplo, nuestra entidad es un usuario.

```json lines
{
      "sub": "1234567890",
      "name": "John Doe",
      "admin": true
    }
```

Al trabajar con claims de JWT, debe tener en cuenta los distintos tipos de claim y las reglas de nomenclatura.

<div id="jws-signature">
  ## Firma JWS
</div>

La firma se utiliza para verificar que quien envía el JWT es quien dice ser y para garantizar que el mensaje no se haya modificado durante el proceso.

Para crear la firma, se toman el `header` y el `payload` codificados en Base64, junto con un secreto, y se firman con el algoritmo especificado en el `header`.

Por ejemplo, si estás creando una firma para un token con el algoritmo HMAC SHA256, harías lo siguiente:

```js lines
HMACSHA256(
      base64UrlEncode(header) + "." +
      base64UrlEncode(payload),
      secret)
```
