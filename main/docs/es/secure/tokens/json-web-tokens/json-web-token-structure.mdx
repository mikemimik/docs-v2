Todos los <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, con frecuencia, formato de access token) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWTs">JWT</Tooltip> emitidos por Auth0 tienen [JSON Web Signatures (JWSs)](https://tools.ietf.org/html/rfc7515), lo que significa que están firmados en lugar de cifrados. Un JWS representa contenido protegido con <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, con frecuencia, formato de access token) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=digital+signatures">firmas digitales</Tooltip> o códigos de autenticación de mensajes (MAC) mediante estructuras de datos basadas en JSON.

Un JWT bien formado consta de tres cadenas codificadas en Base64url concatenadas, separadas por puntos (`.`):

* **encabezado JOSE**: contiene metadatos sobre el tipo de token y los algoritmos criptográficos utilizados para proteger su contenido.
* **JWS payload** (conjunto de [claims](https://tools.ietf.org/html/rfc7519#section-4)): contiene afirmaciones de seguridad verificables, como la identidad del usuario y los permisos que tiene asignados.
* **JWS signature**: se utiliza para validar que el token es confiable y que no ha sido manipulado. Cuando uses un JWT, **debes** [comprobar su firma](/docs/es/secure/tokens/json-web-tokens/validate-json-web-tokens) antes de almacenarlo y usarlo.

<div id="jose-header">
  ## Encabezado JOSE
</div>

Objeto JSON que contiene los parámetros que describen las operaciones y los parámetros criptográficos empleados. El Encabezado JOSE (JSON Object Signing and Encryption) está formado por un conjunto de parámetros de encabezado que normalmente consisten en pares de nombre/valor: el algoritmo de firma utilizado (p. ej., HMAC SHA256 o RSA) y el tipo de JWT.

```json lines
{
      "alg": "HS256",
      "typ": "JWT"
    }
```

<div id="jws-payload">
  ## JWS payload
</div>

El payload contiene información sobre la entidad (normalmente, el usuario) y atributos adicionales de la entidad, denominados claims. En este ejemplo, nuestra entidad es un usuario.

```json lines
{
      "sub": "1234567890",
      "name": "John Doe",
      "admin": true
    }
```

Al trabajar con los claims de JWT, debes tener en cuenta los distintos tipos de claims y las reglas de nomenclatura.

<div id="jws-signature">
  ## Firma JWS
</div>

La firma se utiliza para verificar que el emisor del JWT es quien dice ser y para garantizar que el mensaje no se haya modificado en el proceso.

Para crear la firma, se toman el encabezado y el payload codificados en Base64, junto con un secreto, y se firma todo con el algoritmo especificado en el encabezado.

Por ejemplo, si estás creando una firma para un token con el algoritmo HMAC SHA256, harías lo siguiente:

```js lines
HMACSHA256(
      base64UrlEncode(header) + "." +
      base64UrlEncode(payload),
      secret)
```
