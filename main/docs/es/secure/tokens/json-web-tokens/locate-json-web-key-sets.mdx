---
description: Describe cómo usar las claves web JSON (JWK) descubiertas mediante el endpoint de JSON Web Key Set (JWKS) para verificar la firma de un JWT.
title: Localizar conjuntos de claves web JSON
---

Usa este endpoint de Discovery para configurar tu aplicación o API de modo que localice automáticamente el endpoint de [JSON Web Key Set (JWKS)](/docs/es/secure/tokens/json-web-tokens/json-web-key-sets) (`jwks_uri`), que contiene el JWKS que se usa para firmar todos los <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de access token) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Tokens">JSON Web Tokens</Tooltip> (JWT) emitidos por Auth0 y firmados con el <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de access token) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=signing+algorithm">algoritmo de firma</Tooltip> RS256. El endpoint está disponible en:

`https://{yourDomain}/.well-known/openid-configuration`.

Al [validar un JWT](/docs/es/secure/tokens/json-web-tokens/validate-json-web-tokens) con un JWKS, tendrás que:

1. Recuperar el JWKS desde el endpoint de Discovery de Auth0 y filtrar las posibles claves de firma (por ejemplo, cualquier clave a la que le falte una clave pública o una propiedad `kid`).
2. Obtener la propiedad `kid` del encabezado del JWT decodificado.
3. Buscar en el JWKS filtrado la clave cuya propiedad `kid` coincida.
4. Crear un certificado con la propiedad `x5c` correspondiente de tu JWKS.
5. Usar el certificado para verificar la firma del JWT.

Para ver un ejemplo que usa JWKS para verificar la firma de un JWT, consulta [Navigating RS256 and JWKS](https://auth0.com/blog/navigating-rs256-and-jwks/) (usa Node.js) o nuestros [Quickstarts de backend/API](/docs/es/quickstart/backend).

Para obtener más información sobre la estructura de un JWT, consulta [Estructura de JSON Web Token](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure).

Es una buena práctica asumir que en tu JWKS puede haber varias claves de firma. Esto puede parecer innecesario, ya que el endpoint JWKS de Auth0 normalmente contiene una sola clave de firma; sin embargo, pueden encontrarse varias claves en el JWKS cuando se rotan los certificados de firma.

Te recomendamos almacenar en caché tus claves de firma para mejorar el rendimiento de la aplicación y evitar alcanzar los [límites de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy), pero debes asegurarte de que, si falla la decodificación de un token, invalides la caché y recuperes nuevas claves de firma antes de volver a intentarlo **solo una** vez.
