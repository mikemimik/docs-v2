---
description: Describe cómo usar las claves web JSON (JWK) descubiertas mediante el endpoint JSON Web Key Set (JWKS) para verificar la firma de un JWT.
title: Localizar conjuntos de claves web JSON
---

Usa este endpoint de descubrimiento para configurar tu aplicación o API de modo que localice automáticamente el endpoint [JSON Web Key Set (JWKS)](/docs/es/secure/tokens/json-web-tokens/json-web-key-sets) (`jwks_uri`), que contiene el JWKS usado para firmar todos los <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, formato de Access Token) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Tokens">JSON Web Tokens</Tooltip> (JWT) emitidos por Auth0 y firmados con el algoritmo de firma <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, formato de Access Token) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=signing+algorithm">RS256</Tooltip>. El endpoint se encuentra en:

`https://{yourDomain}/.well-known/openid-configuration`.

Al [validar un JWT](/docs/es/secure/tokens/json-web-tokens/validate-json-web-tokens) con un JWKS, necesitarás:

1. Recuperar el JWKS desde el endpoint de descubrimiento de Auth0 y filtrar posibles claves de firma (por ejemplo, cualquier clave a la que le falte una clave pública o que tenga una propiedad `kid`).
2. Obtener la propiedad `kid` del encabezado del JWT decodificado.
3. Buscar en tu JWKS filtrado la clave cuya propiedad `kid` coincida.
4. Crear un certificado con la propiedad `x5c` correspondiente de tu JWKS.
5. Usar el certificado para verificar la firma del JWT.

Para ver un ejemplo que usa JWKS para verificar la firma de un JWT, consulta [Navigating RS256 and JWKS](https://auth0.com/blog/navigating-rs256-and-jwks/) (usa Node.js) o revisa nuestros [Quickstarts de backend/API](/docs/es/quickstart/backend).

Para obtener más información sobre la estructura de un JWT, consulta [Estructura de JSON Web Token](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure).

Es una buena práctica asumir que puede haber varias claves de firma en tu JWKS. Esto puede parecer innecesario, ya que el endpoint JWKS de Auth0 normalmente contiene una sola clave de firma; sin embargo, es posible encontrar varias claves en el JWKS al rotar certificados de firma.

Te recomendamos almacenar en caché tus claves de firma para mejorar el rendimiento de la aplicación y evitar alcanzar los [límites de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy), pero debes asegurarte de que, si falla la decodificación de un token, invalides la caché y recuperes nuevas claves de firma antes de volver a intentarlo **solo una** vez más.
