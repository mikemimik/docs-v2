<div id="management-api-endpoints">
  ## Endpoints de la Management API
</div>

La <Tooltip tip="Management API: Un producto para permitir a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 proporciona endpoints para gestionar los tokens de actualización de forma individual o en conjunto.

Estos endpoints complementan la [revocación alternativa de tokens de actualización](/docs/es/secure/tokens/refresh-tokens/revoke-refresh-tokens) y sustituyen a los endpoints del recurso `device-credentials` al añadir propiedades ampliadas y operaciones de revocación masiva.

<div id="refresh-token-resource">
  ### Recurso de token de actualización
</div>

Puedes consultar o revocar un token de actualización específico mediante los siguientes endpoints:

| Nombre                                                       | URL                                       | Scope(s) requeridos     |
| ------------------------------------------------------------ | ----------------------------------------- | ----------------------- |
| Consultar la información de un token de actualización por ID | `GET /api/v2/refresh-tokens/{tokenId}`    | `read:refresh_tokens`   |
| Revocar un token de actualización por ID                     | `DELETE /api/v2/refresh-tokens/{tokenId}` | `delete:refresh_tokens` |

<div id="user-resource">
  ### Recurso de usuario
</div>

Puede listar o revocar todos los tokens de actualización de un usuario concreto con los siguientes endpoints:

| Nombre                                                  | URL                                            | Scopes requeridos       |
| ------------------------------------------------------- | ---------------------------------------------- | ----------------------- |
| Listar todos los tokens de actualización de un usuario  | `GET /api/v2/users/{userId}/refresh-tokens`    | `read:refresh_tokens`   |
| Revocar todos los tokens de actualización de un usuario | `DELETE /api/v2/users/{userId}/refresh-tokens` | `delete:refresh_tokens` |

<div id="refresh-token-properties">
  ## Propiedades del token de actualización
</div>

Los <Tooltip tip="Token endpoint: Endpoint del servidor de autorización que se utiliza para solicitar tokens mediante programación." cta="Ver glosario" href="/docs/es/glossary?term=token+endpoints">token endpoints</Tooltip> de actualización devuelven información relevante sobre el token y su historial.

| Field                                   | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Refresh Token ID                        | Un identificador único y específico del tenant del token de actualización en la API. No es el token en sí, que es un secreto que se intercambia en el token endpoint de la Authentication API.<br /><br />Los tokens de actualización pueden seguir siendo válidos más allá de una sesión. El token de actualización conserva esta información independientemente del ciclo de vida de la sesión.                                                                                                                                                                                                                                       |
| Session ID                              | El ID de sesión es un identificador persistente de la sesión en el tenant de Auth0. Ten en cuenta que el ID de sesión corresponde al claim `sid` ya presente en los tokens de ID y los tokens de cierre de sesión, y puede usarse para establecer referencias cruzadas entre estas entidades.<br /><br />El token de actualización contiene referencias a la sesión que se utilizó para crear el token. El ID de sesión puede usarse para recuperar información adicional con el endpoint de la Management API [inspeccionar una sesión específica por ID](/docs/es/manage-users/sessions/manage-user-sessions-with-auth0-management-api). |
| Relevant Time                           | Información sobre la creación y el vencimiento del token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Client and Resource Servers Information | Información resumida sobre las aplicaciones y las API asociadas a un token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

<div id="limitations">
  ## Limitaciones
</div>

* Las operaciones de revocación de tokens de actualización (`DELETE`) se ejecutan de forma asíncrona y presentan consistencia eventual.
* Los tokens de actualización emitidos a partir del 21-09-2023 (22-02-2024 para tenants de la región US-3) incluyen la propiedad de ID de sesión (`session_id`) con el valor correspondiente. Los tokens de actualización emitidos antes de esa fecha incluyen esta propiedad con un valor `null`.
