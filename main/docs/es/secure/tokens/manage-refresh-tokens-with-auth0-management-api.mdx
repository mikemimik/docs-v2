<div id="management-api-endpoints">
  ## Endpoints de la Auth0 Management API
</div>

La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Auth0 Management API</Tooltip> proporciona endpoints para administrar tokens de actualización de forma individual o como un conjunto.

Estos endpoints complementan la [revocación alternativa de tokens de actualización](/docs/es/secure/tokens/refresh-tokens/revoke-refresh-tokens) y sustituyen a los endpoints del recurso `device-credentials` al ofrecer propiedades ampliadas y operaciones de revocación masiva.

<div id="refresh-token-resource">
  ### Recurso de token de actualización
</div>

Puedes consultar o revocar un token de actualización específico con los siguientes endpoints:

| Nombre                                                       | URL                                       | Scopes requeridos       |
| ------------------------------------------------------------ | ----------------------------------------- | ----------------------- |
| Consultar la información de un token de actualización por ID | `GET /api/v2/refresh-tokens/{tokenId}`    | `read:refresh_tokens`   |
| Revocar un token de actualización por ID                     | `DELETE /api/v2/refresh-tokens/{tokenId}` | `delete:refresh_tokens` |

<div id="user-resource">
  ### Recurso de usuario
</div>

Puedes listar o revocar todos los tokens de actualización de un usuario determinado con los siguientes endpoints:

| Nombre                                                  | URL                                            | scopes requeridos       |
| ------------------------------------------------------- | ---------------------------------------------- | ----------------------- |
| Listar todos los tokens de actualización de un usuario  | `GET /api/v2/users/{userId}/refresh-tokens`    | `read:refresh_tokens`   |
| Revocar todos los tokens de actualización de un usuario | `DELETE /api/v2/users/{userId}/refresh-tokens` | `delete:refresh_tokens` |

<div id="refresh-token-properties">
  ## Propiedades del token de actualización
</div>

Los <Tooltip tip="Endpoint de token: endpoint del servidor de autorización que se utiliza para solicitar tokens de forma programática." cta="Ver glosario" href="/docs/es/glossary?term=token+endpoints">endpoints de token</Tooltip> de actualización devuelven información relevante sobre el token y su historial.

| Campo                                                   | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID del token de actualización                           | Un identificador único y específico del tenant para el token de actualización en la API. No es el token en sí, que es un secreto que se intercambia en el endpoint de token de la Authentication API.<br /><br />Los tokens de actualización pueden durar más que una sesión. El token de actualización conserva esta información independientemente del ciclo de vida de la sesión.                                                                                                                                                                                                                                                     |
| ID de sesión                                            | El ID de sesión es un identificador persistente de la sesión en el tenant de Auth0. Ten en cuenta que el ID de sesión corresponde al claim `sid` ya presente en los tokens de ID y los tokens de cierre de sesión, y puede usarse para hacer referencias cruzadas entre estas entidades.<br /><br />El token de actualización contiene referencias a la sesión que se usó para crear el token. El ID de sesión puede usarse para recuperar información adicional mediante el endpoint de la Management API para [inspeccionar una sesión específica por ID](/docs/es/manage-users/sessions/manage-user-sessions-with-auth0-management-api). |
| Tiempo relevante                                        | Información sobre la creación y el vencimiento del token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Información del cliente y de los servidores de recursos | Información resumida sobre las aplicaciones y las API vinculadas a un token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

<div id="limitations">
  ## Limitaciones
</div>

* Las operaciones de revocación de tokens de actualización (`DELETE`) se ejecutan de forma asíncrona y son eventualmente consistentes.
* Los tokens de actualización emitidos el 21-09-2023 o con posterioridad (22-02-2024 para tenants de la región US-3) contienen la propiedad de ID de sesión (`session_id`) con el valor correspondiente. Los tokens de actualización emitidos antes de esta fecha contienen esta propiedad con un valor `null`.
