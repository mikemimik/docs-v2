---
description: Obtén información sobre los scopes y endpoints disponibles para los tokens de acceso de la Management API para aplicaciones de página única (SPA).
title: Obtener tokens de acceso de la Management API para aplicaciones de página única
---

En ciertos casos, puede que quieras usar la [Management API](https://auth0.com/docs/api/management/v2#!) de Auth0 para administrar tus aplicaciones y API en lugar del Dashboard de Auth0.

Para llamar a cualquier endpoint de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, debes autenticarte con un [token de acceso](/docs/es/secure/tokens/access-tokens) especializado llamado token de la Management API. Los tokens de la Management API son [JSON Web Tokens (JWT)](/docs/es/secure/tokens/json-web-tokens) que contienen permisos específicos concedidos (también conocidos como scopes) para los endpoints de la Management API que quieres llamar.

<div id="limitations">
  ## Limitaciones
</div>

Dado que las aplicaciones de página única (SPA) son <Tooltip tip="Cliente público: Cliente (aplicación) que no puede almacenar credenciales de forma segura. Algunos ejemplos son una aplicación nativa de escritorio o móvil y una aplicación web del lado del cliente basada en JavaScript (como una aplicación web de página única (SPA))." cta="Ver glosario" href="/docs/es/glossary?term=public+clients">clientes públicos</Tooltip> y no pueden almacenar de forma segura información confidencial (como un **<Tooltip tip="Secreto de cliente: Secret utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>**), deben obtener tokens de la Management API desde el frontend, a diferencia de otros [tipos de aplicación](/docs/es/get-started/applications). Esto significa que los tokens de la Management API para las SPA tienen ciertas limitaciones. En concreto, se emiten en el contexto del usuario que ha iniciado sesión en Auth0, lo que limita las actualizaciones únicamente a los datos del usuario autenticado. Aunque esto restringe el uso de la Management API, aún puede utilizarse para realizar acciones relacionadas con la actualización del perfil del usuario que ha iniciado sesión.

<Warning>
  Auth0 no recomienda exponer en el frontend tokens de la Management API que permitan a los usuarios cambiar la metadata del usuario. Esto puede permitir que los usuarios manipulen su propia metadata de una forma que podría perjudicar el funcionamiento de las aplicaciones. También permite que un cliente realice un ataque DoS contra la Management API de otra persona simplemente saturándola hasta alcanzar los límites de frecuencia.
</Warning>

<div id="available-scopes-and-endpoints">
  ## Scopes y endpoints disponibles
</div>

Con un token de la Management API emitido para una SPA, puedes acceder a los siguientes scopes (y, por lo tanto, endpoints).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No es posible cambiar la contraseña a través del endpoint [PATCH /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) con un token de la Management API emitido para una SPA.
</Callout>

| Scope del usuario actual                 | Endpoint                                                                                                                                                                                                                                                 |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `read:current_user`                      | [GET /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/get_users_by_id)<br />[GET /api/v2/users/{id}/enrollments](https://auth0.com/docs/api/management/v2#!/Users/get_enrollments)                                                   |
| `update:current_user_identities`         | [POST/api/v2/users/{id}/identities](https://auth0.com/docs/api/management/v2#!/Users/post_identities)<br />[DELETE /api/v2/users/{id}/identities/{provider}/{user_id}](https://auth0.com/docs/api/management/v2#!/Users/delete_user_identity_by_user_id) |
| `update:current_user_metadata`           | [PATCH /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id)                                                                                                                                                           |
| `create:current_user_metadata`           | [PATCH /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id)                                                                                                                                                           |
| `create:current_user_device_credentials` | [POST /api/v2/device-credentials](https://auth0.com/docs/api/management/v2#!/Device_Credentials/post_device_credentials)                                                                                                                                 |
| `delete:current_user_device_credentials` | [DELETE /api/v2/device-credentials/{id}](https://auth0.com/docs/api/management/v2#!/Device_Credentials/delete_device_credentials_by_id)                                                                                                                  |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los scopes y endpoints anteriores están sujetos a [límites de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy).
</Callout>

<div id="use-management-api-token-to-call-management-api-from-a-spa">
  ## Usa un token de la Management API para llamar a la Management API desde una SPA
</div>

Puedes obtener un token de la Management API desde una SPA (usando el <Tooltip tip="Audience: Identificador único del audience de un token emitido. Se llama aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip> de la Management API para generarlo) y usarlo para llamar a la Management API y obtener el perfil completo del usuario que ha iniciado sesión.

1. [Obtén un token de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).

   1. Autentica al usuario redirigiéndolo al endpoint de autorización, que es adonde se redirige a los usuarios cuando inician sesión o se registran.
   2. Cuando recibas el token de la Management API, estará en [formato JSON Web Token](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure).
   3. Decodifícalo y revisa su contenido.
2. Llama a la Management API para obtener el perfil del usuario que ha iniciado sesión desde el endpoint [Get User by ID](https://auth0.com/docs/api/management/v2#!/Users/get_users_by_id).

   1. Para llamar al endpoint, incluye el token codificado de la Management API que obtuviste en el encabezado `Authorization` de la solicitud.
   2. Asegúrate de reemplazar los valores de marcador de posición `USER_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de usuario del usuario que ha iniciado sesión (el valor `sub` del token de la Management API decodificado) y el token de acceso de la Management API, respectivamente.
