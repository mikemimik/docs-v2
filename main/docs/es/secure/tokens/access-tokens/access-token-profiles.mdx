---

Los perfiles de <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+token">token de acceso</Tooltip> definen el formato y los claims de los tokens de acceso emitidos para una API. Auth0 admite los siguientes perfiles de tokens de acceso, también conocidos como dialectos del token:

* Auth0 token profile, o el perfil predeterminado de token de acceso
* RFC 9068 token profile, o el perfil de token de acceso que sigue el [IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068)

| Perfil de token        | Descripción                                                                                                                                                                                       | Dialecto del token                                                                                                                                                                                       |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Auth0 token profile    | El perfil de token predeterminado. Emite tokens de acceso en formato [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens).                                                                 | El Auth0 token profile está asociado a dos dialectos del token:<ul><li>`access_token`</li><li>`access_token_authz`, o el perfil `access_token` con el claim `permissions` incluido</li></ul>             |
| RFC 9068 token profile | Emite tokens de acceso en formato JSON Web Token (JWT), siguiendo el [estándar del IETF](https://datatracker.ietf.org/doc/html/rfc9068) para codificar Access Tokens de OAuth 2.0 en formato JWT. | El RFC 9068 token profile está asociado a dos dialectos del token:<ul><li>`rfc9068_profile`</li><li>`rfc9068_profile_authz`, o el perfil `rfc9068_profile` con el claim `permissions` incluido</li></ul> |

Aunque ambos perfiles de tokens de acceso emiten <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWTs">JWT</Tooltip>, los JWT tienen [formatos de token diferentes](#token-profile-differences). Ambos perfiles de tokens de acceso permiten habilitar [Role-Based Access Control (RBAC)](/docs/es/get-started/apis/enable-role-based-access-control-for-apis) y agregar el claim `permissions` al token de acceso.

Para configurar el perfil de token de acceso para una API, consulte [Configure Access Token Profile](/docs/es/get-started/apis/configure-access-token-profile).

  ### Token de muestra del perfil de Auth0
</div>

```json lines
{
  "iss": "https://my-domain.auth0.com/",
  "sub": "auth0|123456",
  "aud": [
    "https://example.com/health-api",
    "https://my-domain.auth0.com/userinfo"
  ],
  "azp": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "scope": "openid profile read:patients read:admin",
  "my_custom_claim": "my_custom_value"
}
```


<div id="token-profile-differences">
  ## Diferencias entre perfiles de token
</div>

El perfil de Auth0 y el perfil RFC 9068 emiten JWT con formatos de token diferentes. Las principales diferencias son:

* El perfil RFC 9068 incorpora el claim `jti`, que proporciona un identificador único para el JWT.
* El perfil de Auth0 usa el claim `azp` para representar el <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>, mientras que el perfil RFC 9068 usa el claim `client_id`.
* El perfil RFC 9068 no usa el claim `gty`, que es un claim específico de Auth0 que representa el flujo de autenticación.

  ### Cabecera
</div>

| Datos  | perfil RFC 9068                       | Perfil de Auth0                          |
| ------ | ---------------------------------------- | ---------------------------------------- |
| `type` | `at+jwt`                                 | `JWT`                                    |
| `alg`  | Algoritmo de firma, por ejemplo, `RS256` | Algoritmo de firma, por ejemplo, `RS256` |
