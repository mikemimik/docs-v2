<Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o un JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+token">Token de acceso</Tooltip> profiles definen el formato y los claims de los tokens de acceso emitidos para una API. Auth0 admite los siguientes perfiles de tokens de acceso, también conocidos como dialectos del token:

* Auth0 token profile, o el perfil de token de acceso predeterminado
* RFC 9068 token profile, o el perfil de token de acceso que sigue el [IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068)

| Perfil de token        | Descripción                                                                                                                                                                                       | Dialecto del token                                                                                                                                                                                      |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Auth0 token profile    | Es el perfil de token predeterminado. Emite tokens de acceso con formato de [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens).                                                          | Auth0 token profile está asociado con dos dialectos del token:<ul><li>`access_token`</li><li>`access_token_authz`, o el perfil `access_token` con el claim `permissions` incluido</li></ul>             |
| RFC 9068 token profile | Emite tokens de acceso con formato JSON Web Token (JWT), siguiendo el [estándar de la IETF](https://datatracker.ietf.org/doc/html/rfc9068) para codificar OAuth 2.0 Access Tokens en formato JWT. | RFC 9068 token profile está asociado con dos dialectos del token:<ul><li>`rfc9068_profile`</li><li>`rfc9068_profile_authz`, o el perfil `rfc9068_profile` con el claim `permissions` incluido</li></ul> |

Aunque ambos perfiles de tokens de acceso emiten <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, también del token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWTs">JWT</Tooltip>, los JWT tienen [formatos de token diferentes](#token-profile-differences). Ambos perfiles de tokens de acceso pueden habilitar [Role-Based Access Control (RBAC)](/docs/es/get-started/apis/enable-role-based-access-control-for-apis) y agregar el claim permissions al token de acceso.

Para configurar el perfil de token de acceso de una API, consulta [Configurar el perfil de token de acceso](/docs/es/get-started/apis/configure-access-token-profile).

<div id="auth0-profile-sample-token">
  ### Token de ejemplo del perfil de Auth0
</div>

```json lines
{
  "iss": "https://my-domain.auth0.com/",
  "sub": "auth0|123456",
  "aud": [
    "https://example.com/health-api",
    "https://my-domain.auth0.com/userinfo"
  ],
  "azp": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "scope": "openid profile read:patients read:admin",
  "my_custom_claim": "my_custom_value"
}
```

  ### Token de ejemplo del perfil RFC 9068
</div>

```json lines

  "sub": "auth0|123456",
  "aud": [
    "https://example.com/health-api",
    "https://my-domain.auth0.com/userinfo"

  "scope": "openid profile read:patients read:admin",
  "my_custom_claim": "my_custom_value"
}
```

<div id="token-profile-differences">
  ## Diferencias entre perfiles de token
</div>

El perfil de Auth0 y el perfil RFC 9068 emiten JWT con formatos de token diferentes. Las principales diferencias son:

* El perfil RFC 9068 incorpora el claim `jti`, que proporciona un identificador único para el JWT.
* El perfil de Auth0 usa el claim `azp` para representar el <Tooltip tip="ID de cliente: valor de identificación asignado a su recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>, mientras que el perfil RFC 9068 usa el claim `client_id`.
* El perfil RFC 9068 no usa el claim `gty`, un claim específico de Auth0 que representa el flujo de autenticación.

  ### Encabezado
</div>

| Datos  | perfil RFC 9068                       | Perfil de Auth0                          |
| ------ | ---------------------------------------- | ---------------------------------------- |
| `type` | `at+jwt`                                 | `JWT`                                    |
| `alg`  | Algoritmo de firma, por ejemplo, `RS256` | Algoritmo de firma, por ejemplo, `RS256` |
