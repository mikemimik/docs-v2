Para acceder a tu API, debes solicitar un [token de acceso](/docs/es/secure/tokens/access-tokens) al autenticar a un usuario.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Estas herramientas de Auth0 te ayudan a adaptar tu aplicación para autenticar usuarios:

  * Los [Quickstarts](/docs/es/quickstarts) son la forma más sencilla de implementar la autenticación. Te muestran cómo usar [Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login) y los SDKs de Auth0 específicos para cada lenguaje y framework.
  * La [Auth0 Authentication API](https://auth0.com/docs/api/authentication) sirve de referencia para quienes prefieren escribir código por su cuenta. Primero, identifica [qué flujo debes usar](/docs/es/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use). Luego, sigue las instrucciones para implementar ese flujo.
</Callout>

Para solicitar un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>, haz una solicitud POST a la [URL del token](https://auth0.com/docs/api/authentication#client-credentials-flow).

  ```bash cURL
    --header 'content-type: application/x-www-form-urlencoded' \

    --data client_secret={yourClientSecret} \
    --data audience=YOUR_API_IDENTIFIER

  ```

  ```go Go
  }
  ```

<div id="parameters">
  ##### Parámetros
</div>

| Nombre del parámetro | Descripción                                                                                                                                                                                                                                                                                                                    |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `grant_type`         | Configura este valor como &quot;client&#95;credentials&quot;.                                                                                                                                                                                                                                                                  |
| `client_id`          | El ID de cliente de tu aplicación. Puedes encontrar este valor en la [pestaña Settings de la aplicación](https://manage.auth0.com/#/applications).                                                                                                                                                                             |
| `client_secret`      | El secreto de cliente de tu aplicación. Puedes encontrar este valor en la [pestaña Settings de la aplicación](https://manage.auth0.com/#/applications). Para obtener más información sobre los métodos de autenticación de aplicaciones disponibles, consulta [Application Credentials](/docs/es/secure/application-credentials). |
| `audience`           | La audiencia del token, es decir, tu API. Puedes encontrarla en el campo **Identifier** de la [pestaña Settings de tu API](https://manage.auth0.com/#/apis).                                                                                                                                                                   |
| `organization`       | Opcional. El nombre o identificador de la organización a la que quieres asociar la solicitud. Para obtener más información, consulta [Machine-to-Machine Access for Organizations](/docs/es/manage-users/organizations/organizations-for-m2m-applications).                                                                       |

<div id="response">
  #### Respuesta
</div>

Recibes una respuesta `HTTP 200` con un payload que contiene los valores `access_token`, `token_type` y `expires_in`:

```json lines
{
  "access_token":"eyJz93a...k4laUWw",
  "token_type":"Bearer",
  "expires_in":86400
}
```

<Warning>
  Valida tus tokens antes de guardarlos. Para saber cómo hacerlo, lee [Validar tokens de ID](/docs/es/secure/tokens/id-tokens/validate-id-tokens) y [Validar tokens de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).
</Warning>

<div id="control-access-token-audience">
  ## Controlar la audiencia del token de acceso
</div>

Cuando un usuario se autentica, solicitas un token de acceso e incluyes la <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip> de destino y el scope de acceso en tu request. La aplicación usa el endpoint `/authorize` para solicitar acceso. La aplicación solicita este acceso y el usuario lo concede durante la autenticación.

Puedes configurar tu tenant para incluir siempre una audiencia predeterminada.

| Uso del token               | Formato | Audiencia solicitada                                                                                             | Scope solicitado |
| --------------------------- | ------- | ---------------------------------------------------------------------------------------------------------------- | ---------------- |
| endpoint `/userinfo`        | Opaco   | nombre del tenant (`\{yourDomain}`), sin valor para el parámetro `audience`, no se envía el parámetro `audience` | `openid`         |
| Auth0 Management API        | JWT     | identificador de Management API v2 (`https://{tenant}.auth0.com/api/v2/`)                                        |                  |
| Tu propia API personalizada | JWT     | El identificador de API de tu API personalizada registrada en el Auth0 Dashboard                                 |                  |

Solo en un caso específico los tokens de acceso pueden tener varias audiencias de destino. Esto requiere que el <Tooltip tip="Signing Algorithm: Algoritmo usado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=signing+algorithm">algoritmo de firma</Tooltip> de tu API personalizada esté configurado como **RS256**. Para obtener más información, consulta [Token Best Practices](/docs/es/secure/tokens/token-best-practices).

<div id="multiple-audiences">
  ### Múltiples audiencias
</div>

Si especificas como audiencia el identificador de tu API personalizada y el scope `openid`, el claim `aud` del token de acceso resultante será un arreglo en vez de una cadena, y el token de acceso será válido tanto para tu API personalizada como para el endpoint `/userinfo`. Tus tokens de acceso solo pueden tener dos o más audiencias si usas una única API personalizada y el endpoint `/userinfo` de Auth0.

<div id="custom-domains-and-the-auth0-management-api">
  ### Dominios personalizados y la Auth0 Management API
</div>

Auth0 emite tokens con una claim de emisor (`iss)` del dominio que hayas usado al solicitar el token. Los usuarios de [dominios personalizados](/docs/es/customize/custom-domains) pueden usar su <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip> o su dominio de Auth0.

Por ejemplo, supongamos que tienes un dominio personalizado, `https://login.northwind.com`. Si solicitas un token de acceso desde `https://login.northwind.com/authorize`, la claim `iss` de tu token será `https://login.northwind.com/`. Sin embargo, si solicitas un token de acceso desde `https://northwind.auth0.com/authorize`, la claim `iss` de tu token será `https://northwind.auth0.com/`.

Si solicitas un token de acceso desde tu dominio personalizado con la audiencia de destino de la Auth0 <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, entonces **debes** llamar a la Auth0 Management API desde tu dominio personalizado. De lo contrario, tu token de acceso se considerará no válido.

<div id="renew-access-tokens">
  ## Renovar tokens de acceso
</div>

De forma predeterminada, un token de acceso para una API personalizada es válido durante 86400 segundos (24 horas). Puedes [reducir el tiempo antes de que el token caduque](/docs/es/secure/tokens/access-tokens/update-access-token-lifetime).

Una vez que el token de acceso haya caducado, puedes renovarlo. Para ello, vuelve a autenticar al usuario con Auth0 o usa un [token de actualización](/docs/es/secure/tokens/refresh-tokens).
