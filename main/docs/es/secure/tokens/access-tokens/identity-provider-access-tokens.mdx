title: Tokens de acceso de proveedores de identidad
---

<Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+providers">Proveedores de identidad</Tooltip> emiten <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> de terceros después de que los usuarios se autentican con ese proveedor. Puede usar estos tokens de acceso para llamar a la API del proveedor externo que los emitió. Por ejemplo, puede usar un token de acceso emitido después de que un usuario se autentique con Facebook para llamar a la Facebook Graph API.

Para usuarios individuales, los tokens del proveedor de identidad están disponibles en el arreglo `identities` del objeto `user`, en el elemento correspondiente a esa conexión. Para acceder de forma segura a los tokens de un usuario específico, necesita un [token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) que incluya el scope `read:user_idp_tokens`. Luego, puede hacer una llamada HTTP GET al [endpoint Get a User](https://auth0.com/docs/api/management/v2/#!/Users/get_users_by_id) para recuperar los tokens.

Para usuarios administradores de conexiones empresariales configuradas mediante un flujo de consentimiento de <Tooltip tip="OAuth 2.0: marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>, los tokens del proveedor de identidad están disponibles en el objeto `connection`. Para acceder de forma segura a los tokens de una conexión, necesita un [token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) que incluya el scope `read:connections`. Luego, puede hacer una llamada HTTP GET al [endpoint obtener una conexión](https://auth0.com/docs/api/management/v2/connections/get-connections-by-id) para recuperar los tokens.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El contenido de los tokens de acceso de terceros variará según el proveedor de identidad que los emita. Dado que los tokens son creados y administrados por un tercero (como Facebook, GitHub, etc.), el período de validez de los tokens de terceros también variará según el emisor. Si cree que estos tokens se han visto comprometidos, debe revocarlos o restablecerlos con el tercero que los emitió.
</Callout>

<div id="renew-third-party-tokens">
  ## Renovar tokens de terceros
</div>

No existe una forma estándar de renovar los tokens de acceso del proveedor de identidad a través de Auth0. El mecanismo para renovar los tokens de acceso del proveedor de identidad varía según el proveedor. En ciertos proveedores de identidad, Auth0 puede almacenar un [token de actualización](/docs/es/secure/tokens/refresh-tokens), que puedes usar para obtener un nuevo token de acceso del proveedor de identidad. Aquí tienes una lista de algunos de esos proveedores de identidad:

* BitBucket
* Google OAuth 2.0 (pasa el parámetro `access_type=offline`, así como el parámetro `connection_scope` con los scopes necesarios, al llamar al endpoint `/authorize` de Auth0)
* Cualquier otro proveedor de identidad OAuth 2.0
* SharePoint
* Azure AD

Puedes obtener <Tooltip tip="Token de actualización: Token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> de la misma manera que obtienes tokens de acceso. Revisa las pautas anteriores para obtener más información.

<div id="validate-third-party-tokens">
  ## Validar tokens de terceros
</div>

En general, si has recibido un token de acceso de un proveedor de identidad, no necesitas validarlo. Puedes pasarlo al emisor, y el emisor se encarga del resto.
