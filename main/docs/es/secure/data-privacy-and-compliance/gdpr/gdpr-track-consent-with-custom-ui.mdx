En este tutorial veremos cómo puedes usar auth0.js o las API de Auth0 para solicitar información de consentimiento y guardar esa información en los metadatos del usuario. Para obtener más información, lee [Comprende cómo funcionan los metadatos en los perfiles de usuario](/docs/es/manage-users/user-accounts/metadata).

Si, en cambio, quieres hacer seguimiento del consentimiento con Lock, consulta [RGPD: seguimiento del consentimiento con Lock](/docs/es/secure/data-privacy-and-compliance/gdpr/gdpr-track-consent-with-lock).

<div id="overview">
  ## Información general
</div>

Recopilaremos la información de consentimiento en varios escenarios y la guardaremos en los metadatos del usuario.

En todos los escenarios se guardarán las siguientes propiedades en los metadatos del usuario:

* `consentGiven` (true/false) muestra si el usuario ha dado su consentimiento (true) o no (false)
* `consentTimestamp` (marca de tiempo Unix) indica cuándo el usuario otorgó su consentimiento

Por ejemplo:

```json lines
{
  "consentGiven": "true"
  "consentTimestamp": "1525101183"
}
```

Veremos cuatro implementaciones diferentes para esto:

1. una que muestra un indicador, funciona con conexiones de base de datos y usa la biblioteca auth0.js para crear el usuario (la usan las Aplicaciones de página única). Para obtener más información, consulta la [Referencia de Auth0.js](/docs/es/libraries/auth0js).
2. una que muestra un indicador, funciona con conexiones de base de datos y usa la Authentication API para crear el usuario (la usan las Regular Web Apps)
3. una que muestra un indicador, funciona con conexiones sociales y usa la Management API para actualizar la información del usuario (la usan tanto las SPA como las Regular Web Apps)
4. una que redirige a otra página donde se pueden revisar los Términos y condiciones y/o la política de privacidad, y otorgar el consentimiento (la usan tanto las SPA como las Regular Web Apps)

<div id="option-1-use-auth0js">
  ## Opción 1: Usar auth0.js
</div>

En esta sección, usaremos una aplicación de página única sencilla y personalizaremos el widget de login para añadir una casilla que los usuarios puedan usar para proporcionar información de consentimiento. En lugar de crear una aplicación desde cero, [usaremos el ejemplo de Quickstart de JavaScript de Auth0](/docs/es/quickstart/spa/vanillajs). También usaremos las páginas de <Tooltip tip="Universal Login: su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> de Auth0 para implementar Universal Login, en lugar de integrar el login en nuestra aplicación. Para obtener más información sobre Universal Login, consulte [Auth0 Universal Login](/docs/es/authenticate/login/auth0-universal-login). Para obtener más información sobre las diferencias entre Universal Login y embedded login, consulte [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

Esto funciona **solo** para conexiones de base de datos (usaremos la infraestructura de Auth0 en lugar de configurar nuestra propia base de datos).

1. Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y cree una nueva aplicación. Elija `Single Web Page Applications` como tipo. Vaya a **Settings** y configure **Allowed Callback URLs** como `http://localhost:3000`.

   Este campo contiene el conjunto de URL a las que Auth0 puede redirigir a los usuarios después de autenticarse. Nuestra aplicación de ejemplo se ejecutará en `http://localhost:3000`, por lo que configuramos este valor.

2. Copie los valores de **Client Id** y **Domain**. Los necesitará en breve.

3. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y cree una nueva conexión. Haga clic en **Create DB Connection**, asigne un nombre a la nueva conexión y haga clic en **Save**. Vaya a la pestaña **Applications** de la conexión y asegúrese de que su aplicación recién creada esté habilitada.

4. [Descargue el ejemplo de JavaScript SPA](/docs/es/quickstart/spa/vanillajs).

5. [Configure los valores de Client ID y Domain](https://github.com/auth0-samples/auth0-javascript-samples/tree/master/01-Login#set-the-client-id-and-domain).

6. Vaya a [Auth0 Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings). En la pestaña **Login**, habilite el interruptor.

7. En el menú desplegable **Default Templates**, asegúrese de que esté seleccionada la opción `Custom Login Form`. El código ya aparece rellenado previamente.

8. Establezca el valor de la variable `databaseConnection` con el nombre de la conexión de base de datos que usa su aplicación.

   ```javascript lines
   //código reducido por simplicidad
   	var databaseConnection = 'test-db';
   	//código reducido por simplicidad
   ```

9. Para añadir un campo para los metadatos `consentGiven`, agregue una casilla de verificación al formulario. En nuestro ejemplo, configuraremos la casilla para que esté marcada por defecto y deshabilitada, de modo que el usuario no pueda desmarcarla. Puede ajustarlo según las necesidades de su negocio.

   ```javascript lines
   //código reducido por simplicidad
       <div class="form-group">
         <label for="name">Consiento el procesamiento de datos</label>
         <input
           type="checkbox"
           id="userConsent"
           checked disabled>
       </div>
       //código reducido por simplicidad
   ```

10. Edite la función de registro para establecer los metadatos. Tenga en cuenta que establecemos el valor de los metadatos como una cadena con el valor `true`, y no como un valor booleano, y usamos `toString` para convertir el número en una cadena. Esto se debe a una restricción del endpoint [**Signup**](https://auth0.com/docs/api/authentication#signup) de la Authentication API, que solo acepta cadenas como valores.

    ```text lines
    //código reducido por simplicidad
        webAuth.redirect.signupAndLogin({
          connection: databaseConnection,
          email: email,
          password: password,
          user_metadata: { consentGiven: 'true', consentTimestamp: Date.now().toString() }
        }, function(err) {
          if (err) displayError(err);
        });
        //código reducido por simplicidad
    ```

11. Para ver cómo se verá el widget de login, haga clic en la pestaña **Preview**.

<Frame>![Dashboard Branding Universal Login pestaña Classic Login Custom Login Form](/docs/images/cdy7uua7fh8z/4m3WA0sKMoR0C1KVnVmZ1G/b311bdbc6c48b4910eac49cc8f1b9ba8/2025-02-26_15-17-18.png)</Frame>

1. Para probar esta configuración, ejecuta la aplicación y ve a `http://localhost:3000`. Registra un usuario nuevo. Luego ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y busca a ese usuario. Ve a **User Details** y desplázate hacia abajo hasta la sección **Metadata**. En el área de texto **user&#95;metadata**, deberías ver el metadato `consentGiven` establecido en `true`.

<div id="option-3-use-the-api-social">
  ## Opción 3: Usar la API (social)
</div>

Si usas conexiones sociales, no puedes usar la Authentication API para crear el usuario en Auth0, ya que ese endpoint solo funciona con conexiones de base de datos.

En su lugar, debes permitir que el usuario se registre con el proveedor social (lo que creará un registro de usuario en Auth0) y luego usar la Management API para actualizar la información del usuario.

Antes de llamar a la Management API, necesitas obtener un token válido. Para obtener más información, lee [Get Management API Access Tokens for Production](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).

El artículo enlazado usa el Client Credentials Flow para obtener un token, pero no puedes usarlo desde una aplicación que se ejecuta en el navegador. En su lugar, puedes usar el flujo implícito. Para obtener más información sobre el Client Credentials Flow, lee [Client Credentials Flow](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow). Para obtener más información sobre el Implicit Flow, lee [Implicit Flow](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post).

Configura el parámetro de solicitud **<Tooltip tip="Audience: Identificador único de la audience de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip>** como `https://YOUR_DOMAIN/api/v2/` y el parámetro **scope** como el scope `create:current_user_metadata`. Puedes usar el <Tooltip tip="Access Token: Credencial de Authorization, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> que recibirás en la respuesta para llamar al endpoint [**Update User**](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) de la Management API.

Una vez que tengas un token válido, usa el siguiente fragmento para actualizar los metadatos del usuario.

<div id="option-4-redirect-to-another-page">
  ## Opción 4: Redirigir a otra página
</div>

Si quieres mostrarle más información al usuario, al registrarse puedes redirigirlo a otra página donde le solicites consentimiento y cualquier información adicional, y luego volver a redirigirlo para finalizar la transacción de autenticación. Esto puede hacerse con rules de redirección. Esa misma rule puede usarse para guardar la información de consentimiento en los metadatos del usuario, de modo que puedas hacer seguimiento de esta información y no volver a pedirle consentimiento en el siguiente inicio de sesión. Para obtener más información, consulta [Redirect Users from Within Rules](/docs/es/customize/rules/redirect-users).

Tendrás que alojar este formulario, y la URL del formulario debe ser accesible públicamente. Más adelante en este tutorial, tendrás que proporcionar a Auth0 la URL en la que se puede acceder al formulario.

1. Agrega la rule de redirección. Ve a [Auth0 Dashboard &gt; Auth Pipeline &gt; Rules](https://manage.auth0.com/#/rules) y haz clic en **Create Rule**. En **Rules Templates**, selecciona **empty rule**. Cambia el nombre de la rule predeterminada de `empty rule` a algo descriptivo (por ejemplo, `Redirect to consent form`).

2. Agrega el siguiente código JavaScript al editor de scripts y **Save** los cambios.

   ```js lines
   exports.onExecutePostLogin = async (event, api) => {
       const { consentGiven } = event.user.user_metadata || {};

       // redirigir al formulario de consentimiento si el usuario todavía no ha dado su consentimiento
       if (!consentGiven && api.redirect.canRedirect()) {
         const options = {
           query: {
             auth0_domain: `${event.tenant.id}.auth0.com`,
           },
         };
         api.redirect.sendUserTo(event.secrets.CONSENT_FORM_URL, options);
       }
   };

   // si el usuario hace clic en 'I agree' en el formulario de consentimiento, guárdalo en su perfil para que no se le vuelva a pedir
   exports.onContinuePostLogin = async (event, api) => {
     if (event.request.body.confirm === "yes") {
       api.user.setUserMetadata("consentGiven", true);
       api.user.setUserMetadata("consentTimestamp", Date.now());
       return;
     } else {
       return api.access.deny("User did not consent");
     }
   };
   ```

3. Vuelve a [Auth0 Dashboard &gt; Auth0 Pipeline &gt; Rules](https://manage.auth0.com/#/rules) y desplázate hasta la parte inferior de la página, donde está la sección **Settings**. Crea un par clave/valor de la siguiente manera:

   1. **Key**: `CONSENT_FORM_URL`
   2. **Value**: `your-consent-form-url.com`

Asegúrate de proporcionar la URL de acceso público donde se encuentra tu formulario de consentimiento.

Al configurar la redirección a tu formulario de consentimiento para usarla en un entorno de producción, asegúrate de revisar [Trusted Callback URLs](https://github.com/auth0/rules/tree/master/redirect-rules/simple#trusted-callback-urls) y [Data Integrity](https://github.com/auth0/rules/tree/master/redirect-rules/simple#data-integrity) en relación con las consideraciones de seguridad.

Si necesitas una solicitud de consentimiento especializada, por ejemplo, consentimiento parental, tendrás que crear tu propio formulario de consentimiento personalizado. Ten en cuenta que las leyes varían según el país.

Hemos terminado con la parte de configuración. ¡Vamos a probar!

<div id="test-the-configuration">
  ## Prueba la configuración
</div>

1. Ejecuta la aplicación y ve a `https://localhost:3000`.
2. Completa el registro con un usuario nuevo. Se te redirigirá al formulario de consentimiento.
3. Marca el indicador **I agree** y haz clic en **Submit**.
4. Ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y busca tu nuevo usuario.
5. Ve a **User Details** y desplázate hacia abajo hasta la sección **Metadata**.
6. En el área de texto **user&#95;metadata**, deberías ver el metadato `consentGiven` establecido en `true` y `consentTimestamp` establecido en la marca de tiempo Unix del momento en que el usuario dio su consentimiento.

¡Eso es todo, ya terminaste!
