<div id="overview">
  ## Información general
</div>

Capturaremos la información de consentimiento en varios escenarios y la guardaremos en los metadatos del usuario.

En todos los escenarios se guardarán las siguientes propiedades en los metadatos del usuario:

* `consentGiven` (true/false) muestra si el usuario ha dado su consentimiento (true) o no (false)
* `consentTimestamp` (marca de tiempo Unix) indica cuándo el usuario dio su consentimiento

Por ejemplo:

```json lines
{
  "consentGiven": "true"
  "consentTimestamp": "1525101183"
}
```

Veremos cuatro implementaciones diferentes para esto:

1. una que muestra un indicador, funciona con conexiones de base de datos y usa la biblioteca auth0.js para crear el usuario (usada por Aplicaciones de página única). Para obtener más información, consulta la [Referencia de Auth0.js](/docs/es/libraries/auth0js).
2. una que muestra un indicador, funciona con conexiones de base de datos y usa la Authentication API para crear el usuario (usada por Regular Web Apps)
3. una que muestra un indicador, funciona con conexiones sociales y usa la Management API para actualizar la información del usuario (usada tanto por SPAs como por Regular Web Apps)
4. una que redirige a otra página donde se pueden consultar los Términos y Condiciones y/o la política de privacidad, y otorgar el consentimiento correspondiente (usada tanto por SPAs como por Regular Web Apps)

<div id="option-1-use-auth0js">
  ## Opción 1: Usar auth0.js
</div>

En esta sección, usaremos una aplicación de página única sencilla y personalizaremos el widget de login para agregar una casilla que los usuarios puedan usar para proporcionar información de consentimiento. En lugar de crear una aplicación desde cero, [usaremos el ejemplo de JavaScript Quickstart de Auth0](/docs/es/quickstart/spa/vanillajs). También usaremos las páginas de <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> de Auth0 para implementar Universal Login, en lugar de integrar el login en nuestra aplicación. Para obtener más información sobre Universal Login, lea [Auth0 Universal Login](/docs/es/authenticate/login/auth0-universal-login). Para obtener más información sobre las diferencias entre Universal Login y embedded login, lea [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

Esto funciona **solo** para conexiones de base de datos (usaremos la infraestructura de Auth0, en lugar de configurar nuestra propia base de datos).

1. Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y cree una nueva aplicación. Elija `Single Web Page Applications` como tipo. Vaya a **Settings** y establezca **Allowed Callback URLs** en `http://localhost:3000`.

   Este campo contiene el conjunto de URL a las que Auth0 puede redirigir a los usuarios después de autenticarse. Nuestra aplicación de ejemplo se ejecutará en `http://localhost:3000`, por lo que establecemos este valor.

2. Copie los valores de **Client Id** y **Domain**. Los necesitará dentro de un momento.

3. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y cree una nueva conexión. Haga clic en **Create DB Connection**, asigne un nombre a la nueva conexión y haga clic en **Save**. Vaya a la pestaña **Applications** de la conexión y asegúrese de que su aplicación recién creada esté habilitada.

4. [Descargue el ejemplo de JavaScript SPA](/docs/es/quickstart/spa/vanillajs).

5. [Establezca los valores de Client ID y Domain](https://github.com/auth0-samples/auth0-javascript-samples/tree/master/01-Login#set-the-client-id-and-domain).

6. Vaya a [Auth0 Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings). En la pestaña **Login**, active el interruptor.

7. En el menú desplegable **Default Templates**, asegúrese de que esté seleccionado `Custom Login Form`. El código aparecerá precargado.

8. Establezca el valor de la variable `databaseConnection` con el nombre de la conexión de base de datos que usa su aplicación.

   ```javascript lines
   //código reducido por simplicidad
   	var databaseConnection = 'test-db';
   	//código reducido por simplicidad
   ```

9. Para agregar un campo para los metadatos `consentGiven`, añada una casilla de verificación al formulario. En este ejemplo, configuraremos la casilla para que esté seleccionada de forma predeterminada y deshabilitada, de modo que el usuario no pueda desmarcarla. Puede ajustar esto según las necesidades de su negocio.

   ```javascript lines
   //código reducido por simplicidad
       <div class="form-group">
         <label for="name">Doy mi consentimiento para el procesamiento de datos</label>
         <input
           type="checkbox"
           id="userConsent"
           checked disabled>
       </div>
       //código reducido por simplicidad
   ```

10. Edite la función de registro para establecer los metadatos. Tenga en cuenta que establecemos el valor de los metadatos en una cadena con el valor `true`, y no en un valor booleano, y que usamos `toString` para convertir el número en una cadena. Esto se debe a una restricción del endpoint [**Signup**](https://auth0.com/docs/api/authentication#signup) de Authentication API, que solo acepta cadenas como valores.

    ```text lines
    //código reducido por simplicidad
        webAuth.redirect.signupAndLogin({
          connection: databaseConnection,
          email: email,
          password: password,
          user_metadata: { consentGiven: 'true', consentTimestamp: Date.now().toString() }
        }, function(err) {
          if (err) displayError(err);
        });
        //código reducido por simplicidad
    ```

11. Para ver cómo se verá el widget de login, haga clic en la pestaña **Preview**.

1. Para probar esta configuración, ejecuta la aplicación y ve a `http://localhost:3000`. Regístrate con un usuario nuevo. Luego ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y busca tu nuevo usuario. Ve a **User Details** y desplázate hacia abajo hasta la sección **Metadata**. En el área de texto **user&#95;metadata**, deberías ver el metadato `consentGiven` establecido en `true`.

<div id="option-3-use-the-api-social">
  ## Opción 3: Usa la API (social)
</div>

Si usas conexiones sociales, no puedes usar la Authentication API para crear el usuario en Auth0, ya que ese endpoint solo funciona con conexiones de base de datos.

Lo que debes hacer en su lugar es permitir que el usuario se registre con el proveedor social (lo que creará un registro de usuario en Auth0) y luego usar la Management API para actualizar la información del usuario.

Antes de llamar a la Management API, debes obtener un token válido. Para obtener más información, consulta [Get Management API Access Tokens for Production](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).

El artículo enlazado usa el Client Credentials Flow para obtener un token, pero no puedes usarlo desde una aplicación que se ejecuta en el navegador. En su lugar, puedes usar el flujo implícito. Para obtener más información sobre el Client Credentials Flow, consulta [Client Credentials Flow](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow). Para obtener más información sobre el flujo implícito, consulta [Implicit Flow](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post).

Configura el parámetro de solicitud **<Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip>** como `https://YOUR_DOMAIN/api/v2/` y el parámetro **scope** como el scope `create:current_user_metadata`. Puedes usar el <Tooltip tip="Token de acceso: Credencial de acceso, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> que recibirás en la respuesta para llamar al endpoint [**Update User** de la Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).

Una vez que tengas un token válido, usa el siguiente fragmento para actualizar los metadatos del usuario.

<div id="option-4-redirect-to-another-page">
  ## Opción 4: Redirigir a otra página
</div>

Si quieres mostrarle más información al usuario, al registrarse puedes redirigirlo a otra página para solicitar su consentimiento y cualquier dato adicional, y después redirigirlo de vuelta para completar la transacción de autenticación. Esto puede hacerse con redirect rules. Esa misma rule también puede usarse para guardar la información de consentimiento en los metadatos del usuario, de modo que puedas hacer seguimiento de ella y no volver a solicitar el consentimiento en el próximo inicio de sesión. Para obtener más información, consulta [Redirect Users from Within Rules](/docs/es/customize/rules/redirect-users).

Tendrás que alojar este formulario, y la URL del formulario debe ser accesible públicamente. Más adelante en este tutorial, tendrás que proporcionar a Auth0 la URL en la que se puede acceder al formulario.

1. Agrega la rule de redirección. Ve a [Auth0 Dashboard &gt; pipeline de autenticación &gt; Rules](https://manage.auth0.com/#/rules) y haz clic en **Create Rule**. En **Rules Templates**, selecciona **empty rule**. Cambia el nombre predeterminado de la rule de `empty rule` a algo descriptivo (por ejemplo, `Redirect to consent form`).

2. Agrega el siguiente código Javascript al editor de script y **Save** los cambios.

   ```js lines
   exports.onExecutePostLogin = async (event, api) => {
       const { consentGiven } = event.user.user_metadata || {};

       // redirigir al formulario de consentimiento si el usuario aún no ha dado su consentimiento
       if (!consentGiven && api.redirect.canRedirect()) {
         const options = {
           query: {
             auth0_domain: `${event.tenant.id}.auth0.com`,
           },
         };
         api.redirect.sendUserTo(event.secrets.CONSENT_FORM_URL, options);
       }
   };

   // si el usuario hace clic en 'I agree' en el formulario de consentimiento, guardarlo en su perfil para que no se le vuelva a pedir
   exports.onContinuePostLogin = async (event, api) => {
     if (event.request.body.confirm === "yes") {
       api.user.setUserMetadata("consentGiven", true);
       api.user.setUserMetadata("consentTimestamp", Date.now());
       return;
     } else {
       return api.access.deny("User did not consent");
     }
   };
   ```

3. Vuelve a [Auth0 Dashboard &gt; Auth0 Pipeline &gt; Rules](https://manage.auth0.com/#/rules) y desplázate hasta la parte inferior de la página, donde se encuentra la sección **Settings**. Crea un par clave/valor de la siguiente manera:

   1. **Key**: `CONSENT_FORM_URL`
   2. **Value**: `your-consent-form-url.com`

Asegúrate de proporcionar la URL de acceso público donde se encuentre tu formulario de consentimiento.

Al configurar la redirección a tu formulario de consentimiento para usarlo en un entorno de Production, asegúrate de revisar [Trusted Callback URLs](https://github.com/auth0/rules/tree/master/redirect-rules/simple#trusted-callback-urls) y [Data Integrity](https://github.com/auth0/rules/tree/master/redirect-rules/simple#data-integrity) en relación con cuestiones de seguridad.

Si necesitas una solicitud de consentimiento especializada, por ejemplo, consentimiento parental, tendrás que crear tu propio formulario de consentimiento personalizado. Ten en cuenta que las leyes varían según el país.

Hemos terminado con la parte de configuración. ¡Probémosla!

<div id="test-the-configuration">
  ## Prueba la configuración
</div>

1. Ejecuta la aplicación y ve a `https://localhost:3000`.
2. Regístrate con un usuario nuevo. Se te redirigirá al formulario de consentimiento.
3. Marca el indicador **I agree** y haz clic en **Submit**.
4. Ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y busca tu nuevo usuario.
5. Ve a **User Details** y desplázate hacia abajo hasta la sección **Metadata**.
6. En el área de texto **user&#95;metadata**, deberías ver que el metadato `consentGiven` está establecido en `true` y que `consentTimestamp` está establecido en la marca de tiempo Unix del momento en que el usuario dio su consentimiento.

¡Eso es todo, ya terminaste!
