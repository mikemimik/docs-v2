En este tutorial, veremos cómo puedes usar Lock para solicitar información de consentimiento y luego guardar esos datos en los metadatos del usuario. Para obtener más información, lee [Comprende cómo funcionan los metadatos en los perfiles de usuario](/docs/es/manage-users/user-accounts/metadata).

Si prefieres hacer el seguimiento del consentimiento mediante una interfaz personalizada, consulta [RGPD: Seguimiento del consentimiento con una interfaz personalizada](/docs/es/secure/data-privacy-and-compliance/gdpr/gdpr-track-consent-with-custom-ui).

<div id="overview">
  ## Información general
</div>

Configuraremos una aplicación de página única de JavaScript sencilla y una conexión de base de datos (usaremos la infraestructura de Auth0 en lugar de configurar nuestra propia base de datos).

En lugar de crear una aplicación desde cero, [usaremos el ejemplo de Quickstart de JavaScript de Auth0](/docs/es/quickstart/spa/vanillajs). También usaremos las páginas de <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> de Auth0 para implementar una experiencia de Universal Login, en lugar de integrar el inicio de sesión en nuestra aplicación. Para obtener más información sobre Universal Login, consulte [Auth0 Universal Login](/docs/es/authenticate/login/auth0-universal-login). Para obtener más información sobre las diferencias entre Universal Login y embedded login, consulte [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

Capturaremos información de consentimiento en varios escenarios y la guardaremos en los metadatos del usuario.

Todos los escenarios guardarán las siguientes propiedades en los metadatos del usuario:

* una propiedad `consentGiven`, con valores true/false, indica si el usuario ha dado su consentimiento (true) o no (false)
* una propiedad `consentTimestamp`, que contiene la marca de tiempo Unix de cuándo el usuario dio su consentimiento

Por ejemplo:

```json lines
{
  "consentGiven": "true"
  "consentTimestamp": "1525101183"
}
```

Veremos tres implementaciones distintas para esto:

* una que muestra enlaces a otras páginas donde se puede consultar la información sobre los Términos y Condiciones y/o la política de privacidad
* una que añade campos personalizados al widget de registro y funciona con conexiones de base de datos
* una que redirige a otra página donde el usuario puede dar su consentimiento y funciona con conexiones sociales

<div id="configure-the-application">
  ## Configura la aplicación
</div>

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y crea una nueva aplicación. Elige `Single Web Page Applications` como tipo.
2. Ve a **Settings** y configura las **Allowed Callback URLs** como `http://localhost:3000`.

   Este campo contiene el conjunto de URL a las que Auth0 puede redirigir a los usuarios después de autenticarse. Nuestra aplicación de ejemplo se ejecutará en `http://localhost:3000`, por lo que configuramos este valor.
3. Copia los valores de **ID de cliente** y **Domain**. Los necesitarás en breve.
4. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y crea una nueva conexión. Haz clic en **Create DB Connection**, asigna un nombre a la nueva conexión y haz clic en **Save**. También puedes habilitar una conexión social en [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social) (vamos a habilitar el login con Google para este tutorial).
5. Ve a la pestaña **Applications** de la conexión y asegúrate de que la aplicación que acabas de crear esté habilitada.
6. [Descarga el ejemplo de JavaScript SPA](/docs/es/quickstart/spa/vanillajs).

<div id="option-1-display-terms-conditions-link">
  ## Opción 1: Mostrar el enlace de Términos y condiciones
</div>

En esta sección, personalizaremos el widget de inicio de sesión para agregar un indicador que los usuarios deben marcar para poder registrarse. La etiqueta del indicador incluirá enlaces a páginas que muestren los Términos y condiciones y la política de privacidad.

Esto funciona tanto para conexiones de base de datos como para inicios de sesión sociales.

1. Ve a [Auth0 Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_page).
2. Selecciona la vista **Login** y activa el interruptor **Customize Login Page**.
3. Busca el menú desplegable **Default Templates** y selecciona `Lock`. El bloque de código se completará automáticamente.
4. Para agregar un campo para los metadatos de `consentGiven`, usa la opción `mustAcceptTerms`. Para incluir enlaces a tus páginas de Términos y condiciones y/o política de privacidad, usa la opción `languageDictionary`. Para obtener más información, consulta [Lock Configuration Options](/docs/es/libraries/lock/lock-configuration). El siguiente ejemplo muestra el texto `I agree to the terms of service and privacy policy` (incluidos los enlaces a ambas páginas) junto al indicador:

   ```javascript lines
   //código reducido por simplicidad
       var lock = new Auth0Lock(config.clientID, config.auth0Domain, {
         auth: {
           //código reducido por simplicidad
         },
         languageDictionary: {
           signUpTerms: "I agree to the <a href='https://my-app-url.com/terms' target='_blank'>terms of service</a> and <a href='https://my-app-url.com/privacy' target='_blank'>privacy policy</a>."
         },
         mustAcceptTerms: true,
         //código reducido por simplicidad
       });
   ```

   Para ver cómo quedará, selecciona la vista **Preview** y, cuando se cargue Lock, selecciona **Sign Up**.
5. Este indicador obliga a los usuarios a aceptar los términos antes de poder registrarse, pero no guarda ningún metadato. Para guardar la selección del usuario en la propiedad de metadatos `consentGiven`, [crea una nueva Action](/docs/es/customize/actions/write-your-first-action). Introduce un **Name** descriptivo para tu Action (por ejemplo, `Set consent flag upon signup`), selecciona el trigger `Login / Post Login` porque vas a agregar la Action al Login Flow y, a continuación, selecciona **Create**.
6. La siguiente pantalla es el Actions Code Editor. Copia en él el siguiente código Javascript y luego selecciona **Save Draft** para guardar los cambios:

   ```text lines
   exports.onExecutePostLogin = async (event, api) => {
     const { consentGiven } = event.user.user_metadata || {};

     // salir anticipadamente si el usuario ya se registró
     if ( consentGiven ) {
       return;
     }

     // primer login/registro
     api.user.setUserMetadata("consentGiven", true);
     api.user.setUserMetadata("consentTimestamp", Date.now());
     return;
   }
   ```

   Este código establece el metadato `consentGiven` en `true` si todavía no está establecido (lo que significa que es el primer login después del registro).
7. En la barra lateral del Actions Code Editor, selecciona Test (icono de reproducción) y luego selecciona **Run** para [probar tu código](/docs/es/customize/actions/test-actions).
8. Cuando estés listo para que la Action entre en funcionamiento, selecciona **Deploy**.

Por último, agrega la Action que creaste al [Login Flow](https://manage.auth0.com/#/actions/flows/login/). Para obtener más información sobre cómo adjuntar Actions a Flows, consulta la sección &quot;Attach the Action to a flow&quot; de [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

<div id="option-2-add-custom-fields-for-database-connections">
  ## Opción 2: Agregar campos personalizados para conexiones de base de datos
</div>

En esta sección, personalizaremos el widget de inicio de sesión para agregar un indicador que los usuarios marcarán si aceptan el procesamiento de su información.

Esto solo funciona para conexiones de base de datos (si usa inicios de sesión sociales, consulte el siguiente párrafo).

1. Vaya a [Auth0 Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_page).

2. Seleccione la vista **Login** y active el interruptor **Customize Login Page**.

3. Busque el menú desplegable **Default Templates** y seleccione `Lock`. El bloque de código se rellenará automáticamente.

4. Para agregar un campo para los metadatos `consentGiven`, use la opción `additionalSignUpFields`. Para obtener más información, consulte [Lock Configuration Options](/docs/es/libraries/lock/lock-configuration). El siguiente ejemplo establece el tipo como `checkbox` (para que tengamos un indicador), el texto como `I consent to data processing` y el valor predeterminado como `checked`.

   ```javascript lines
   //código reducido por simplicidad
       var lock = new Auth0Lock(config.clientID, config.auth0Domain, {
         auth: {
           //código reducido por simplicidad
         },
         additionalSignUpFields: [{
           type: "checkbox",
           name: "consentGiven",
           prefill: "true",
           placeholder: "I consent to data processing"
         }],
         //código reducido por simplicidad
       });
   ```

5. Para ver cómo se verá, seleccione la vista **Preview** y, cuando Lock se cargue, seleccione **Sign Up**.

Tenga en cuenta que, en esta opción, solo establecemos el indicador y no la marca de tiempo. Mostrar la hora actual en el widget de inicio de sesión no es lo más adecuado, por eso no agregamos un campo adicional de registro. Lo que debe hacer es establecer la marca de tiempo en segundo plano, con una rule que verifique el valor de `consentGiven` y establezca los metadatos adicionales `consentTimestamp` con la marca de tiempo actual.

<div id="option-3-redirect-to-another-page">
  ## Opción 3: Redirigir a otra página
</div>

Si usas inicios de sesión sociales, agregar campos personalizados no es una opción, pero puedes redirigir al usuario a otra página para solicitar su consentimiento y cualquier información adicional, y luego redirigirlo de regreso para completar la transacción de autenticación. Esto se puede hacer con Redirect Actions. Para obtener más información, consulta [Redirect with Actions](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/redirect-with-actions). Usaremos esta misma Action para guardar la información de consentimiento en los metadatos del usuario, de modo que podamos hacer seguimiento de esta información y no volver a solicitar el consentimiento en el próximo login.

Para simplificar, usaremos un [formulario de consentimiento de ejemplo](https://github.com/auth0/rules/blob/master/redirect-rules/simple/webtask.js#L31). Tendrás que alojar este formulario, y la URL del formulario debe ser de acceso público. En el Paso 2, deberás proporcionar a Auth0 la URL en la que se puede acceder al formulario. Si necesitas una solicitud de consentimiento especializada (por ejemplo, consentimiento parental), debes crear tu propio formulario de consentimiento personalizado. Ten en cuenta que las leyes varían según el país.

1. [Crea una nueva Action](/docs/es/customize/actions/write-your-first-action). Introduce un **Nombre** descriptivo para tu Action (por ejemplo, `Redirect to consent form`), selecciona el trigger `Login / Post Login` porque agregarás la Action al Login flow y, a continuación, selecciona **Create**.

2. Ubica el Actions Code Editor y selecciona el icono **Secrets** (llave) en la barra lateral. Agrega la URL del formulario de consentimiento como un Secret creando un par clave/valor:

   * **Key**: `CONSENT_FORM_URL`
   * **Value**: `your-consent-form-url.com` (Asegúrate de proporcionar la URL de acceso público donde se encuentra tu formulario de consentimiento).

3. Copia el siguiente código JavaScript en el Actions Code Editor y selecciona **Save Draft** para guardar los cambios:

   ```text lines
   exports.onExecutePostLogin = async (event, api) => {
       const { consentGiven } = event.user.user_metadata || {};

       // redirige al formulario de consentimiento si el usuario todavía no ha dado su consentimiento
       if (!consentGiven && api.redirect.canRedirect()) {
         const options = {
           query: {
             auth0_domain: `${event.tenant.id}.auth0.com`,
           },
         };
         api.redirect.sendUserTo(event.secrets.CONSENT_FORM_URL, options);
       }
   };

   // si el usuario hace clic en 'I agree' en el formulario de consentimiento, guárdalo en su perfil para que no se le vuelva a pedir
   exports.onContinuePostLogin = async (event, api) => {
     if (event.request.body.confirm === "yes") {
       api.user.setUserMetadata("consentGiven", true);
       api.user.setUserMetadata("consentTimestamp", Date.now());
       return;
     } else {
       return api.access.deny("User did not consent");
     }
   };
   ```

4. En la barra lateral del Actions Code Editor, selecciona Test (icono de reproducción) y luego selecciona **Run** para [probar tu código](/docs/es/customize/actions/test-actions).

5. Cuando estés listo para que la Action entre en funcionamiento, selecciona **Deploy**.

Por último, agrega la Action que creaste al [Login Flow](https://manage.auth0.com/#/actions/flows/login/). Para aprender a adjuntar Actions a Flows, lee la sección &quot;Attach the Action to a flow&quot; en [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

Al configurar la redirección a tu formulario de consentimiento para usarla en un entorno de producción, asegúrate de revisar [Trusted Callback URLs](https://github.com/auth0/rules/tree/master/redirect-rules/simple#trusted-callback-urls) y [Data Integrity](https://github.com/auth0/rules/tree/master/redirect-rules/simple#data-integrity) en relación con cuestiones de seguridad.

Hemos terminado con la parte de configuración; ¡vamos a probar!

<div id="test-the-configuration">
  ## Prueba la configuración
</div>

1. Ve a la carpeta donde descargaste la aplicación y ejecútala.

   ```text lines
   npm install
       npm run
   ```

2. Ve a `http://localhost:3000`. Haz clic en **Login**. Cuando se muestre Lock, haz clic en **Sign Up**.
   La login page se mostrará de forma predeterminada en `YOUR_DOMAIN/login`. Para saber cómo usar tu propio dominio, consulta [Custom Domains](/docs/es/customize/custom-domains).

3. Si seguiste la primera opción de implementación, deberías ver el indicador para aceptar los Términos de servicio y la política de privacidad. Ten en cuenta que el botón **Sign Up** permanece deshabilitado hasta que marques el indicador. Sigue los enlaces para comprobar que funcionan. Introduce un correo electrónico y una contraseña, acepta los términos y haz clic en **Sign Up**. Como alternativa, si usas una conexión social, acepta los términos y elige **Sign Up with Google**.

4. Si seguiste la segunda opción de implementación, deberías ver el nuevo campo personalizado que agregamos. Introduce un correo electrónico y una contraseña y deja marcado el indicador `I consent to data processing`. Haz clic en **Sign Up**.

5. Si seguiste la tercera opción de implementación, elige **Sign Up with Google**. Se te redirigirá al formulario de consentimiento. Marca el indicador **I agree** y haz clic en **Submit**.

   Si no marcas el indicador **I agree** antes de hacer clic en **Submit**, verás un error emergente `Unauthorized. Check the console for details.`. En la consola verás este JSON:

   ```json lines
   {
         error: "unauthorized", 
         errorDescription: "User did not consent!", 
         state: "q0GjMwzZN_q5r8XPHvfakkMYcYM2q1N3"
       }
   ```

   Ten en cuenta que el usuario se crea, pero no podrá iniciar sesión. Si lo intenta, se le volverá a pedir que otorgue su consentimiento.

6. Ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y busca al nuevo usuario.

7. Ve a **User Details** y desplázate hacia abajo hasta la sección **Metadata**. En el área de texto **user&#95;metadata** deberías ver lo siguiente:

   ```json lines
   {
         "consentGiven": "true"
         "consentTimestamp": "1525101183"
       }
   ```

¡Eso es todo! ¡Ya terminaste!
