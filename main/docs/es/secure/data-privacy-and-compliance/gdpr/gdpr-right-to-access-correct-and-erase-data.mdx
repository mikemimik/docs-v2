title: 'RGPD: derecho de acceso, rectificación y supresión de datos'
---

De acuerdo con los artículos 15, 16, 17 y 19 del RGPD, los usuarios tienen derecho a obtener una copia de los datos personales que usted está tratando, a solicitar su rectificación si son inexactos y a pedir que se eliminen.

Con Auth0, puede acceder, editar y eliminar la información del usuario manualmente, mediante el Dashboard, o mediante programación, usando la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El contenido de estos documentos no pretende constituir asesoramiento jurídico ni debe considerarse un sustituto de asistencia legal. La responsabilidad final de entender y cumplir el RGPD recae en usted, aunque Auth0 le ayudará a cumplir sus requisitos siempre que sea posible.
</Callout>

<div id="manual-process">
  ## Proceso manual
</div>

Puedes ver, editar y eliminar la información de los usuarios en [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users). Haz clic en un usuario para ver su información. La información que puedes cambiar es:

| Campo              | Cómo editar                                                                                                                                                                                       |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Email**          | Haz clic en Edit. Introduce el nuevo correo electrónico.                                                                                                                                          |
| **Email verified** | Haz clic en **Edit** en el campo **Εmail**. Haz clic en el enlace **Set email as verified**.                                                                                                      |
| **Metadata**       | Tanto los objetos `app_metadata` como `user_metadata` se pueden editar desde esta pantalla. Edita el JSON en la sección **Metadata** y guarda los cambios.                                        |
| **Blocked**        | No se puede editar directamente. Haz clic en **Actions &gt; Block User** en la esquina superior derecha de esta pantalla. Para desbloquear al usuario, haz clic en **Actions &gt; Unblock User**. |
| **Email**          | No se puede editar directamente. Haz clic en **Actions &gt; Change Email** en la esquina superior derecha de esta pantalla.                                                                       |
| **Password**       | No se puede editar directamente. Haz clic en **Actions &gt; Change Password** en la esquina superior derecha de esta pantalla.                                                                    |

Para eliminar un usuario, haz clic en **Actions &gt; Delete User**.

<div id="programmatic-process">
  ## Proceso programático
</div>

Puedes recuperar, editar y eliminar información del usuario mediante nuestra API. Elige un endpoint que se ajuste a tus necesidades:

* Recuperar un usuario usando el ID como criterio de búsqueda
* Recuperar un usuario usando el correo electrónico como criterio de búsqueda
* Exportar todos los usuarios a un archivo mediante una tarea de larga duración
* Actualizar un usuario. Ten en cuenta que no todos los campos se pueden editar (consulta la siguiente sección). Recuerda que:

  * Las propiedades del nuevo objeto reemplazarán a las anteriores. Los campos `user_metadata` y `app_metadata` son una excepción a esta regla. Estas propiedades se fusionan en lugar de reemplazarse, aunque la fusión solo se produce en el primer nivel.
  * Si estás actualizando `email_verified`, `phone_verified`, `username` o `password`, debes establecer el parámetro `connection`.
  * Si estás actualizando `email` o `phone_number`, debes establecer los parámetros `connection` y `client_id`.
* Eliminar un usuario según el ID

Para llamar a cualquiera de los endpoints de la API, necesitarás un <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">access token</Tooltip> válido. Este token debe tener los permisos requeridos para cada endpoint. Cada endpoint en Management API Explorer tiene una sección de scopes que enumera los scopes que debe contener el access token para poder acceder a él. Una vez que sepas a qué endpoint quieres acceder y tengas un access token válido, estarás listo para enviar tu solicitud.

<div id="editable-data">
  ## Datos editables
</div>

La siguiente información del usuario puede actualizarse mediante la API:

* blocked
* email&#95;verified
* email
* verify&#95;email
* password
* phone&#95;number
* phone&#95;verified
* verify&#95;password
* user&#95;metadata
* app&#95;metadata
* username

La siguiente información del usuario **no** se puede editar:

* given&#95;name
* family&#95;name
* name
* nickname
* picture

<div id="searchable-fields">
  ## Campos de búsqueda
</div>

Puede buscar usuarios por lo siguiente:

* Todos los campos del perfil de usuario normalizado
* La información del perfil en el objeto `user_metadata`:

  * name
  * nickname
  * given&#95;name
  * family&#95;name

Usted es responsable de asegurarse de que el cliente se elimine o de que los datos se actualicen en cualquier otra base de datos a la que Auth0 no esté conectado.
