title: 'RGPD: Derecho de acceso, rectificación y supresión de datos'
---

De conformidad con los artículos 15, 16, 17 y 19 del RGPD, los usuarios tienen derecho a obtener una copia de los datos personales que usted procesa, solicitar su rectificación si son inexactos y pedirle que suprima sus datos personales.

Con Auth0, puede acceder a la información del usuario, editarla y eliminarla manualmente mediante el Dashboard o mediante programación con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El contenido de estos documentos no pretende constituir asesoramiento jurídico ni debe considerarse un sustituto de asistencia legal. La responsabilidad final de comprender y cumplir el RGPD recae en usted, aunque Auth0 le ayudará a cumplir sus requisitos siempre que sea posible.
</Callout>

<div id="manual-process">
  ## Proceso manual
</div>

Puede ver, editar y eliminar la información del usuario en [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users). Seleccione un usuario para ver su información. La información que puede cambiar es:

| Campo                | Cómo editar                                                                                                                                                                                         |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Email**            | Haga clic en Edit. Configure el nuevo correo electrónico.                                                                                                                                           |
| **Email verificado** | Haga clic en **Edit** en el campo **Εmail**. Haga clic en el enlace **Set email as verified**.                                                                                                      |
| **Metadata**         | Tanto los objetos `app_metadata` como `user_metadata` se pueden editar desde esta pantalla. Edite el JSON en la sección **Metadata** y guarde los cambios.                                          |
| **Bloqueado**        | No se puede editar directamente. Haga clic en **Actions &gt; Block User** en la esquina superior derecha de esta pantalla. Para desbloquear al usuario, haga clic en **Actions &gt; Unblock User**. |
| **Email**            | No se puede editar directamente. Haga clic en **Actions &gt; Change Email** en la esquina superior derecha de esta pantalla.                                                                        |
| **Password**         | No se puede editar directamente. Haga clic en **Actions &gt; Change Password** en la esquina superior derecha de esta pantalla.                                                                     |

Para eliminar un usuario, haga clic en **Actions &gt; Delete User**.

<div id="programmatic-process">
  ## Proceso programático
</div>

Puede recuperar, editar y eliminar información de usuarios mediante nuestra API. Elija un endpoint que se ajuste a sus necesidades:

* Recuperar un usuario usando el ID como criterio de búsqueda
* Recuperar un usuario usando el Email como criterio de búsqueda
* Exportar todos los usuarios a un archivo mediante una tarea de larga duración
* Actualizar un usuario. Tenga en cuenta que no todos los campos se pueden editar (consulte la siguiente sección). Tenga presente que:

  * Las propiedades del nuevo objeto reemplazarán a las anteriores. Los campos `user_metadata` y `app_metadata` son una excepción a esta regla. Estas propiedades se fusionan en lugar de reemplazarse, aunque la fusión solo se realiza en el primer nivel.
  * Si está actualizando `email_verified`, `phone_verified`, `username` o `password`, debe establecer el parámetro `connection`.
  * Si está actualizando `email` o `phone_number`, debe establecer los parámetros `connection` y `client_id`.
* Eliminar un usuario según el ID

Para llamar a cualquiera de los endpoints de la API, necesitará un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> válido. Este token debe tener los permisos requeridos para cada endpoint. Cada endpoint en el Management API Explorer tiene una sección de scopes que enumera los scopes que debe contener el token de acceso para poder acceder a él. Una vez que sepa a qué endpoint desea acceder y tenga un token de acceso válido, estará listo para enviar su solicitud.

<div id="editable-data">
  ## Datos editables
</div>

La siguiente información del usuario se puede actualizar mediante la API:

* blocked
* email&#95;verified
* email
* verify&#95;email
* password
* phone&#95;number
* phone&#95;verified
* verify&#95;password
* user&#95;metadata
* app&#95;metadata
* username

La siguiente información del usuario **no** se puede editar:

* given&#95;name
* family&#95;name
* name
* nickname
* picture

<div id="searchable-fields">
  ## Campos de búsqueda
</div>

Puede buscar usuarios por lo siguiente:

* Todos los campos del perfil de usuario normalizado
* La información del perfil en el objeto `user_metadata`:

  * name
  * nickname
  * given&#95;name
  * family&#95;name

Es su responsabilidad asegurarse de que el cliente se elimine o de que los datos se actualicen en cualquier otra base de datos a la que Auth0 no esté conectado.
