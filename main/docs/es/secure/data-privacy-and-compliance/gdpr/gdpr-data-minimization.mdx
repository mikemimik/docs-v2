---
description: Describe cómo puede minimizar los datos personales que recopila para su tratamiento y garantizar su seguridad.
title: 'RGPD: Minimización de datos'
---

De acuerdo con el artículo 5 del RGPD, los datos personales que recopile deben limitarse a lo necesario para su tratamiento y conservarse solo durante el tiempo imprescindible. Debe garantizarse una seguridad adecuada durante el tratamiento de los datos, incluida la protección frente al tratamiento no autorizado o ilícito y frente a la pérdida, destrucción o daño accidentales.

Existen varias funcionalidades de Auth0 que pueden ayudarle a lograr estos objetivos, como la vinculación de cuentas, el cifrado del perfil de usuario, entre otras.

<div id="restrict-user-profile-information">
  ## Restringir la información del perfil de usuario
</div>

Para limitar la cantidad de información personal en el perfil de usuario de Auth0, puedes:

* Minimizar (o evitar) el almacenamiento de información personal en la sección de metadatos del perfil de usuario
* Si usas directorios empresariales, configúralos para que devuelvan solo la información mínima necesaria
* Si usas proveedores sociales, configúralos para que devuelvan solo la información mínima necesaria
* Añadir a la lista de exclusión los atributos de usuario que no quieras guardar en las bases de datos de Auth0

<div id="encrypt-user-profile-information">
  ## Cifrar la información del perfil de usuario
</div>

Puedes cifrar la información del usuario antes de guardarla en el perfil de usuario. Puedes usar cualquier mecanismo de cifrado que prefieras antes de almacenar datos en los campos de metadatos. Cuando un usuario introduzca información confidencial, llama al endpoint Update a User.

<div id="use-account-linking">
  ## Usar la vinculación de cuentas
</div>

Cada vez que un usuario usa una conexión para iniciar sesión en tu aplicación, se crea un perfil de usuario si aún no existe. Ten en cuenta que esto ocurre por conexión.

Para entenderlo mejor, considera el siguiente escenario. Tu aplicación ofrece tres opciones distintas de registro:

* registrarse con correo electrónico y contraseña
* iniciar sesión con Google
* iniciar sesión con Facebook

Si un usuario se registra con Google, se creará un perfil de usuario en Auth0. Si ese mismo usuario, cuando regresa, no recuerda con qué se registró y elige iniciar sesión con Facebook, Auth0 creará otro perfil de usuario para él. Así que ahora tienes dos perfiles para el mismo usuario.

Puedes solucionar esto con la vinculación de cuentas. Puedes vincular varias cuentas a un solo perfil de usuario, independientemente del tipo de conexión (por ejemplo, usuario/contraseña, social o <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>).

Hay dos formas de implementar esto:

* **Vinculación de cuentas iniciada por el usuario**: tu aplicación debe proporcionar la interfaz de usuario para que un usuario autenticado pueda vincular sus cuentas manualmente.
* **Vinculación de cuentas sugerida**: en este caso, igualmente configuras una rule que vinculará cuentas con la misma dirección de correo electrónico verificada. Sin embargo, en lugar de completar la vinculación automáticamente, tu aplicación primero le pedirá al usuario que vincule sus identidades.

<div id="export-logs">
  ## Exportar registros
</div>

Puedes exportar los registros de Auth0 y almacenarlos por tu cuenta o enviarlos automáticamente a un servicio externo de registros en [Auth0 Marketplace](https://marketplace.auth0.com/features/log-streaming). Esta funcionalidad puede ayudarte a cumplir los requisitos de retención de datos, así como los de análisis de registros.

<div id="export-logs-with-the-api">
  ### Exportar registros con la API
</div>

Puedes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para exportar registros y almacenarlos por tu cuenta. Hay dos endpoints disponibles, y cada uno proporciona información ligeramente diferente.

<div id="search-all-logs">
  #### Buscar en todos los registros
</div>

El endpoint Search Log Events recupera entradas de registro que coinciden con los criterios de búsqueda que proporcionaste. Si no proporcionas ningún criterio de búsqueda, obtendrás una lista de todas las entradas disponibles.

Puedes indicar criterios de búsqueda con el parámetro **q** y recuperar campos específicos con el parámetro **fields**.

Para acceder a la API, necesitas un token de Management APIv2.

Esta solicitud de ejemplo recupera todos los registros de inicios de sesión correctos (el acrónimo del evento para un login correcto es `s`). La lista de campos que recuperaremos por entrada de registro es: **date**, **description**, **client&#95;id** y **log&#95;id**.

<div id="get-a-single-log-entry">
  #### Obtener una entrada de registro
</div>

El endpoint Get a Log Event by ID recupera la entrada de registro asociada al ID indicado.

Esta solicitud de ejemplo recupera una entrada de registro con el ID `90020180129170850881585554625888895190928456277777449010`.

<div id="export-to-an-external-service">
  ### Exportar a un servicio externo
</div>

Puede instalar y configurar una de nuestras [soluciones de transmisión de registros](https://marketplace.auth0.com/features/log-streaming) en Auth0 Marketplace para exportar registros automáticamente a otro proveedor, como Sumo Logic o Loggly. Para ver la lista de proveedores disponibles y los pasos detallados para configurar cada uno, consulte Exportar registros de Auth0 a un servicio externo.

<div id="keep-sensitive-information-from-logs">
  ## Mantén la información sensible fuera de los registros
</div>

Debes minimizar cualquier información sensible contenida en las URL que pueda quedar capturada en los archivos de registro de Auth0. Por ejemplo, considera usar `health-site` o algo parecido como nombre de dominio en lugar de `cancer-treatments`.

También debes asegurarte de hacer lo siguiente:

* Analiza qué datos recopilas en el registro y a través de las redes sociales, y si realmente son necesarios para la finalidad de tu servicio.
* Configura los <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> empresariales para controlar qué datos se devuelven a Auth0.
* Especifica qué datos quieres recopilar del proveedor social y negocia con él cualquier condición específica sobre el inicio de sesión social y el uso de los datos que obtendrá sobre el inicio de sesión de tus usuarios.
