---
description: Describe cómo puede minimizar los datos personales que recopila para su tratamiento y garantizar su seguridad.
title: 'RGPD: Minimización de datos'
---

De acuerdo con el artículo 5 del RGPD, los datos personales que recopile deben limitarse a lo necesario para su tratamiento y conservarse solo durante el tiempo necesario. Debe garantizarse una seguridad adecuada durante el tratamiento de los datos, incluida la protección contra el tratamiento no autorizado o ilícito, así como contra la pérdida, destrucción o daños accidentales.

Auth0 ofrece varias funcionalidades que pueden ayudarle a lograr estos objetivos, como la vinculación de cuentas, el cifrado del perfil de usuario y otras más.

</Callout>

<div id="restrict-user-profile-information">
  ## Restringir la información del perfil de usuario
</div>

Para limitar la cantidad de información personal en el perfil de usuario de Auth0, puedes:

* Minimizar (o evitar) guardar información personal en la sección de metadatos del perfil de usuario
* Si usas directorios empresariales, configúralos para que devuelvan solo la información mínima necesaria
* Si usas proveedores sociales, configúralos para que devuelvan solo la información mínima necesaria
* Añadir a la lista de exclusión los atributos de usuario que no quieras guardar en las bases de datos de Auth0

<div id="encrypt-user-profile-information">
  ## Cifra la información del perfil de usuario
</div>

Puedes cifrar la información del usuario antes de guardarla en el perfil de usuario. Puedes usar cualquier mecanismo de cifrado que prefieras antes de almacenar datos en los campos de metadatos. Cuando un usuario establezca información confidencial, llama al endpoint Update a User.

<div id="use-account-linking">
  ## Usar la vinculación de cuentas
</div>

Cada vez que un usuario usa una conexión para iniciar sesión en tu aplicación, se crea un perfil de usuario si aún no existe. Ten en cuenta que esto ocurre por conexión.

Para entenderlo mejor, considera el siguiente escenario. Tu aplicación ofrece tres opciones distintas de registro:

* registrarse con correo electrónico/contraseña
* iniciar sesión con Google
* iniciar sesión con Facebook

Si un usuario se registra con Google, se creará un perfil de usuario en Auth0. Si ese mismo usuario, cuando vuelve, no recuerda con qué se registró y elige iniciar sesión con Facebook, Auth0 creará otro perfil de usuario para él. Así, ahora tendrás dos perfiles para el mismo usuario.

Puedes solucionar esto con la vinculación de cuentas. Puedes vincular varias cuentas a un único perfil de usuario, independientemente del tipo de conexión (por ejemplo, usuario/contraseña, social o <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>).

Hay dos formas de implementar esto:

* **Vinculación de cuentas iniciada por el usuario**: tu aplicación debe proporcionar la UI para que un usuario autenticado pueda vincular sus cuentas manualmente.
* **Vinculación de cuentas sugerida**: en este caso, sigues configurando una rule que vinculará cuentas con la misma dirección de correo electrónico verificada. Sin embargo, en lugar de completar la vinculación automáticamente, tu aplicación primero le pedirá al usuario que vincule sus identidades.

<div id="export-logs">
  ## Exportar logs
</div>

Puedes exportar los logs de Auth0 y almacenarlos tú mismo o enviarlos automáticamente a un servicio externo de logs en [Auth0 Marketplace](https://marketplace.auth0.com/features/log-streaming). Esta funcionalidad puede ayudarte a cumplir los requisitos de retención de datos, así como los de análisis de logs.

<div id="export-logs-with-the-api">
  ### Exportar logs con la API
</div>

Puedes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para exportar logs y almacenarlos por tu cuenta. Hay dos endpoints disponibles, y cada uno proporciona información ligeramente diferente.

<div id="search-all-logs">
  #### Buscar todos los logs
</div>

El endpoint Search Log Events recupera las entradas de registro que coinciden con los criterios de búsqueda que proporcionaste. Si no proporcionas ningún criterio de búsqueda, obtendrás una lista de todas las entradas disponibles.

Puedes indicar criterios de búsqueda mediante el parámetro **q** y recuperar campos específicos con el parámetro **fields**.

Para acceder a la API, necesitas un token de Management APIv2.

Esta solicitud de ejemplo recupera todos los logs de inicios de sesión correctos (el acrónimo del evento para un inicio de sesión correcto es `s`). La lista de campos que recuperaremos por cada entrada de registro es: **date**, **description**, **client&#95;id** y **log&#95;id**.

<div id="get-a-single-log-entry">
  #### Obtener una sola entrada de registro
</div>

El endpoint Get a Log Event by ID recupera la entrada de registro asociada al ID proporcionado.

Esta solicitud de ejemplo recupera una sola entrada de registro con el ID `90020180129170850881585554625888895190928456277777449010`.

<div id="export-to-an-external-service">
  ### Exportar a un servicio externo
</div>

Puedes instalar y configurar una de nuestras [soluciones de transmisión de logs](https://marketplace.auth0.com/features/log-streaming) en Auth0 Marketplace para exportar automáticamente los logs a otro proveedor, como Sumo Logic o Loggly. Para consultar la lista de proveedores disponibles y los pasos detallados para configurar cada uno, consulta Exportar los logs de Auth0 a un servicio externo.

<div id="keep-sensitive-information-from-logs">
  ## Mantén la información sensible fuera de los logs
</div>

Debes minimizar cualquier información sensible contenida en las URL que pueda quedar capturada en los archivos de log de Auth0. Por ejemplo, considera usar `health-site` o algo similar como nombre de dominio en lugar de `cancer-treatments`.

También debes asegurarte de hacer lo siguiente:

* Analiza qué estás recopilando durante el registro y a través de proveedores sociales, y si eso es necesario para la finalidad de tu servicio.
* Configura los <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> empresariales para controlar qué datos se devuelven a Auth0.
* Especifica qué datos quieres recopilar del proveedor social y negocia con él cualquier condición particular sobre el inicio de sesión social y el uso de los datos que recibirán sobre el inicio de sesión de tus usuarios.
