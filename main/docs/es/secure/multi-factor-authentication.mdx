<Card title="Información general">
  La autenticación multifactor (MFA) es otro método para proteger tu aplicación y las identidades de tus usuarios. MFA añade una capa de seguridad durante el inicio de sesión que requiere que los usuarios proporcionen más de una credencial para demostrar su identidad digital. Los factores pueden ser:

  * Algo que eres, como un dato biométrico
  * Algo que sabes, como una contraseña
  * Algo que tienes, como un dispositivo

  Lee estas preguntas y respuestas para ver si usar MFA con tu instancia de Auth0 es la opción adecuada para ti.
</Card>

<div id="what-is-multi-factor-authentication">
  ## ¿Qué es la autenticación multifactor?
</div>

<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+authentication">Autenticación multifactor</Tooltip> (MFA) es un método de verificación de identidad que requiere más de un tipo de verificación del usuario. Evita que <Tooltip tip="Actores maliciosos: Entidad (una persona o grupo) que representa una amenaza para la empresa o el entorno con la intención de causar daño." cta="Ver glosario" href="/docs/es/glossary?term=bad+actors">actores maliciosos</Tooltip> accedan a una cuenta incluso si han obtenido el nombre de usuario y la contraseña.

<div id="why-use-multi-factor-authentication">
  ## ¿Por qué usar la autenticación multifactor?
</div>

La autenticación multifactor (MFA) reduce la probabilidad de muchos tipos de ciberataques. Es habitual que terceros roben nombres de usuario y contraseñas o lancen ataques automatizados contra las cuentas de usuario. Un factor de MFA adicional, como una huella dactilar o un código de un solo uso, dificulta este tipo de ataques.

<div id="how-does-multi-factor-authentication-work">
  ## ¿Cómo funciona la autenticación multifactor?
</div>

La MFA funciona exigiendo información de verificación adicional (conocida como factores). Los usuarios no pueden iniciar sesión solo con su nombre de usuario y contraseña. Deben proporcionar una verificación adicional de identidad, como reconocimiento facial o notificaciones por mensaje de texto.

<div id="mfa-factors">
  ## Factores de MFA
</div>

<Warning>
  Los factores de MFA dependen de la disponibilidad del plan; algunos solo están disponibles en los planes Professional y Enterprise. Para obtener más información, consulta [Precios de Auth0](https://auth0.com/pricing).
</Warning>

Auth0 admite diversos factores de MFA, entre ellos:

* Notificaciones push
* Notificaciones por SMS
* Notificaciones de voz
* Contraseñas de un solo uso
* WebAuthn con claves de seguridad
* WebAuthn con biometría del dispositivo
* Notificaciones por correo electrónico
* Cisco Duo Security
* Códigos de recuperación

Para obtener más información, consulta [Factores de autenticación multifactor](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors).
