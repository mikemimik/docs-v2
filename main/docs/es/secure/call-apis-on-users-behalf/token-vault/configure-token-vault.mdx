description: Aprende a configurar Token Vault.
title: Configurar Token Vault
---

Una vez que un usuario se autentica con un [proveedor externo compatible](/docs/es/secure/tokens/token-vault#supported-external-providers) y autoriza la conexión, tu aplicación puede acceder a Token Vault para intercambiar un token de Auth0 por el token de acceso de un proveedor externo.

Para configurar Token Vault, debes hacer lo siguiente:

1. [Configurar Connected Accounts para Token Vault](/docs/es/secure/tokens/token-vault/configure-token-vault#configure-connected-accounts-for-token-vault) para una conexión social o empresarial compatible.
2. [Configurar tu aplicación](#configure-application) con el tipo de concesión de Token Vault.
3. Configurar el intercambio de tokens para tu aplicación:
   * [Intercambio de token de actualización](#configure-refresh-token-exchange)
   * [Intercambio de token de acceso](#configure-access-token-exchange)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si anteriormente configuraste tu política de MFA como **Always** en el Auth0 Dashboard, debes cambiarla a **Never** para obtener un token de acceso desde Token Vault. De lo contrario, recibirás un error. Para obtener más información sobre las distintas políticas de MFA, consulta [Habilitar MFA en el Auth0 Dashboard](/docs/es/secure/multi-factor-authentication/enable-mfa#enable-mfa-in-the-auth0-dashboard).

  Si necesitas activar desafíos MFA para flujos interactivos, habilita **Customize MFA Factors using Actions** al [configurar MFA para tu tenant](/docs/es/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login#prepare-your-tenant). Después, puedes usar una Action para activar un desafío MFA según la propiedad `event.transaction.protocol`. Para obtener más información, consulta [Personalizar la selección de MFA para Universal Login](/docs/es/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
</Callout>

<div id="configure-connected-accounts-for-token-vault">
  ## Configurar Connected Accounts para Token Vault
</div>

Connected Accounts para Token Vault gestiona un perfil de usuario de Auth0 unificado, vinculado a varias cuentas externas. Luego, su aplicación recupera las credenciales almacenadas en Token Vault para interactuar con las API externas en nombre del usuario.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si su aplicación usa [Organizations](/docs/es/manage-users/organizations), configure la conexión y, luego, autentique al usuario con la organización de destino antes de iniciar el flujo de Connected Accounts. La cuenta conectada y los tokens almacenados siguen vinculados al perfil de Auth0 del usuario.
</Callout>

Puede configurar Connected Accounts para conexiones sociales y empresariales compatibles. Para obtener más información, consulte [Configurar Connected Accounts](/docs/es/secure/tokens/token-vault/connected-accounts-for-token-vault#configure-connected-accounts).

<div id="configure-application">
  ## Configurar aplicación
</div>

Configura tu aplicación con el tipo de concesión de Token Vault mediante el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Solo ciertos tipos de clientes pueden usar el tipo de concesión de Token Vault:

1. El cliente debe ser un cliente first-party; es decir, la propiedad `is_first_party` debe ser `true`.
2. El cliente debe ser un cliente confidencial con un mecanismo de autenticación válido; es decir, la propiedad `token_endpoint_auth_method` no debe establecerse en `none`.
3. El cliente debe ser compatible con OIDC; es decir, `oidc_conformant` debe ser `true`.

<Tabs>
  <Tab title="Auth0 Dashboard">
    1) Ve a **Applications &gt; Applications**.
    2) Selecciona la aplicación que quieres configurar.
    3) En **Advanced Settings &gt; Grant Types**, selecciona el tipo de concesión **Token Vault**.
    4) Selecciona **Save Changes**.

    <Frame>![](/docs/images/token-vault/configure_token_vault_grant_type.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Para habilitar Token Vault para una aplicación, realiza una llamada `PATCH` al endpoint [actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para agregar el tipo de concesión `urn:auth0:params:oauth:grant-type:token-exchange:federated-connection-access-token` al objeto JSON del cliente:

    ```bash lines
    curl --location --request PATCH 'https://{yourDomain}/api/v2/clients/{clientId}' \
      --header 'Content-Type: application/json' \
      --header 'Authorization: Bearer <YOUR_MANAGEMENT_API_ACCESS_TOKEN>' \
      --data '{
        "grant_types": [
          "authorization_code",
          "refresh_token",
          "urn:auth0:params:oauth:grant-type:token-exchange:federated-connection-access-token"
        ]
      }'
    ```
  </Tab>
</Tabs>

<div id="configure-token-exchange">
  ## Configurar intercambio de tokens
</div>

Para llamar a las API de un proveedor externo, tu aplicación debe intercambiar un token válido de Auth0 por un token de acceso del proveedor externo desde Token Vault. El tipo de token de Auth0 que se usa en el intercambio depende de tu tipo de cliente y de tu caso de uso. Para obtener más información, consulta [Intercambios de tokens compatibles](/docs/es/secure/tokens/token-vault#supported-token-exchanges).

<div id="configure-refresh-token-exchange">
  ### Configurar el intercambio de token de actualización
</div>

Para usar el [intercambio de token de actualización con Token Vault](/docs/es/secure/tokens/token-vault/refresh-token-exchange-with-token-vault), debes configurar tu aplicación con los siguientes tipos de concesión:

* código de autorización: Permite que tu aplicación realice el inicio de sesión inicial del usuario, en el que intercambia un código de autorización temporal por un token de acceso de Auth0, un token de actualización y un token de identidad.
* Token de actualización: Permite que tu aplicación use un token de actualización de Auth0 de larga duración para solicitar un nuevo token de acceso de Auth0 sin necesidad de que el usuario vuelva a iniciar sesión.
* Token Vault: Permite que tu aplicación intercambie un token de actualización de Auth0 por el token de acceso de un proveedor externo almacenado en Token Vault.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para configurar tu aplicación para el intercambio de token de actualización:

    * Ve a **Applications &gt; Applications**.
    * Selecciona la aplicación que quieres configurar.
    * En **Advanced Settings &gt; Grant Types**, selecciona los tipos de concesión **Refresh Token**, **código de autorización** y **Token Vault**.
    * Selecciona **Save Changes**.
  </Tab>

  <Tab title="Management API">
    Para configurar tu aplicación para el intercambio de token de actualización, haz una llamada `PATCH` al endpoint [actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para agregar los tipos de concesión `refresh_token`, `authorization_code,` y `urn:auth0:params:oauth:grant-type:token-exchange:federated-connection-access-token` al objeto JSON del cliente:

    ```bash lines
    curl --location --request PATCH 'https://{yourDomain}/api/v2/clients/{clientId}' \
      --header 'Content-Type: application/json' \
      --header 'Authorization: Bearer <YOUR_MANAGEMENT_API_ACCESS_TOKEN>' \
      --data '{
        "grant_types": [
          "authorization_code",
          "refresh_token",
          "urn:auth0:params:oauth:grant-type:token-exchange:federated-connection-access-token"
        ]
      }'
    ```
  </Tab>
</Tabs>

<div id="configure-access-token-exchange">
  ### Configurar el intercambio de token de acceso
</div>

Para usar el [intercambio de token de acceso con Token Vault](/docs/es/secure/tokens/token-vault/access-token-exchange-with-token-vault), debes hacer lo siguiente:

* [Configurar tu SPA](#configure-your-spa) con el tipo de concesión `authorization_code`.
* [Crear una API de backend](#create-backend-api) para que la SPA pueda solicitar un Auth0 token de acceso al especificarla como audience.
* [Crear un cliente de API personalizada](#create-custom-api-client) que esté vinculado a la API de backend con el tipo de concesión de Token Vault habilitado.

<div id="configure-your-spa">
  #### Configure su SPA
</div>

Configure su SPA con el tipo de concesión `authorization_code`. Esto permite que la SPA solicite al servidor de autorización de Auth0 un token de acceso de Auth0 con alcance para la API de backend.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para configurar su SPA con el tipo de concesión `authorization_code`:

    * Vaya a **Applications &gt; Applications**.
    * Seleccione la aplicación que desea configurar.
    * En **Advanced Settings &gt; Grant Types**, seleccione el tipo de concesión **código de autorización**.
    * Seleccione **Save Changes**.
  </Tab>

  <Tab title="Management API">
    Para configurar su SPA, haga una llamada `PATCH` al endpoint [actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para agregar el tipo de concesión `authorization_code` al objeto JSON del cliente:

    ```bash lines
    curl --request PATCH 'https://{yourDomain}/api/v2/clients/{clientId}' \
      --header 'Content-Type: application/json' \
      --header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
      --data '{
        "grant_types": [
          "authorization_code"
        ]
      }'
    ```
  </Tab>
</Tabs>

<div id="create-backend-api">
  #### Crear API de backend
</div>

Cree una API de backend con un identificador único y los scopes deseados para realizar el intercambio de tokens de acceso con el servidor de autorización de Auth0.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para crear una API de backend en el Auth0 Dashboard:

    * Vaya a **Applications &gt; APIs** y haga clic en **Create API**.
    * Para crear su API, siga las instrucciones de [Registrar APIs](/docs/es/get-started/auth0-overview/set-up-apis). **Nota:** Una vez que establezca un identificador para su API, no podrá cambiarlo.
    * Haga clic en **Create**.
    * Una vez que haya creado su API, debe agregar scopes a la API. Vaya a la pestaña **Permissions**. En **Add a Permission**, agregue los scopes.
  </Tab>

  <Tab title="Management API">
    Para crear una API de backend con la Management API, haga una solicitud `POST` al endpoint `/resource-servers`:

    ```bash lines 
    curl --request POST 'https://{yourDomain}/api/v2/resource-servers' \
      --header 'Content-Type: application/json' \
      --header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
      --data '{
        "name": "My API Resource Server",
        "identifier": "https://my-api.example.com",
        "scopes": [
          {
            "value": "read:calendar",
            "description": "Read calendar events"
          },
          {
            "value": "write:calendar",
            "description": "Write calendar events"
          }
        ]
      }'
    ```
  </Tab>
</Tabs>

<div id="create-custom-api-client">
  #### Crear cliente de API personalizada
</div>

Para el intercambio de token de acceso, debes crear un cliente de API personalizada vinculado a la API de backend. La SPA podrá solicitar un token de acceso para la API de backend especificándola como `audience` en la solicitud de autorización al servidor de autorización de Auth0. El cliente de API personalizada tiene el mismo identificador que tu API de backend y tiene habilitado el tipo de concesión de Token Vault.

Cuando la API de backend realiza el intercambio de token de acceso, se autentica enviando las credenciales del cliente de API personalizada al servidor de autorización de Auth0, lo que demuestra que es la misma entidad que se registró en el Auth0 Dashboard.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para crear un cliente de API personalizada en el Auth0 Dashboard:

    * Ve a **Applications &gt; APIs** y selecciona tu API de backend.
    * Selecciona **Add Application** e introduce un nombre para la aplicación.
    * Haz clic en **Add**. Una vez que la aplicación se haya creado correctamente, haz clic en **Configure Application** y desplázate hasta **Application Properties**. El **Application Type** es un cliente de API personalizada.
    * En **Advanced Settings &gt; Grant Types**, el tipo de concesión **Token Vault** ya debería estar habilitado para el cliente de API personalizada.

    <Frame>![](/docs/images/token-vault/create_custom_api_client.png)</Frame>
  </Tab>

  <Tab title="Management API">
    El siguiente ejemplo de código crea un cliente de API personalizada con el mismo identificador que tu API de backend y agrega el tipo de concesión de Token Vault:

    ```bash lines
    curl --request POST 'https://{yourDomain}/api/v2/clients' \
      --header 'Content-Type: application/json' \
      --header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
      --data '{
        "name": "Custom API Client",
        "app_type": "resource_server",
        "resource_server_identifier": "https://my-api.example.com",
        "grant_types": [
          "urn:auth0:params:oauth:grant-type:token-exchange:federated-connection-access-token"
        ]
      }'
    ```

    | Parameter                    | Description                                                                                                                                                                                                         |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `name`                       | Nombre de tu cliente de API personalizada.                                                                                                                                                                          |
    | `app_type`                   | El tipo de aplicación de tu cliente de API personalizada. Para registrar el cliente como resource server, configúralo como `resource_server`.                                                                       |
    | `resource_server_identifier` | El identificador único de tu cliente de API personalizada. Configúralo con el `audience` de tu API de backend; es decir, `https://my-api.example.com.`                                                              |
    | `grant_types`                | Los tipos de concesión habilitados para tu cliente de API personalizada. Configúralo con el tipo de concesión de Token Vault: `urn:auth0:params:oauth:grant-type:token-exchange:federated-connection-access-token`. |
  </Tab>
</Tabs>

Una vez que hayas creado correctamente el cliente de API personalizada, el usuario será redirigido a este en lugar de a la SPA después de iniciar sesión.
