title: Token Vault

Token Vault simplifica la forma en que tus aplicaciones acceden a APIs externas en nombre del usuario. Cuando te integras con Token Vault, obtienes una forma segura de gestionar el acceso de la aplicación a una amplia variedad de servicios externos y sus APIs, como Google, GitHub y Microsoft.

Cuando un usuario se conecta con un [proveedor externo compatible](#supported-external-providers) y autoriza el acceso mediante scopes de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip>, Auth0 añade automáticamente esa cuenta conectada al perfil del usuario. Una cuenta conectada permite que las aplicaciones accedan a APIs externas mediante un perfil de usuario de Auth0 unificado. Para obtener más información, consulta [Connected Accounts for Token Vault](/docs/es/secure/tokens/token-vault/connected-accounts-for-token-vault).

Auth0 almacena los tokens de acceso y los tokens de actualización de cada cuenta conectada en Token Vault. Para recuperar estas credenciales almacenadas en Token Vault, tu aplicación realiza un [intercambio de tokens seguro](#supported-token-exchanges). Este intercambio de tokens permite que tu aplicación obtenga los tokens necesarios para llamar a una API externa, lo que evita que tengas que desarrollar y mantener integraciones personalizadas con cada proveedor.

  ## Proveedores externos compatibles
</div>

Token Vault es compatible con proveedores externos populares, como:

</div>

* Google
* Microsoft
* Box
* Slack
* GitHub
* conexión social personalizada

<div id="enterprise">
  ### Empresarial

* Google Workspace
* Microsoft Azure AD (Entra ID)
* <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect

Para ver la lista completa de proveedores externos compatibles, consulta [Auth0 Integrations](https://auth0.com/ai/docs/integrations/overview).

<div id="common-use-cases">
  ## Casos de uso comunes
</div>

Los casos de uso comunes de Token Vault incluyen:

* Un agente de IA que se ejecuta como una aplicación web hace llamadas a APIs externas para realizar tareas en nombre del usuario, como programar una reunión en Google Calendar.
* Un servicio interno o de backend puede acceder a Token Vault para intercambiar un token de acceso de Auth0 por un token de acceso de un proveedor externo y llamar a APIs externas.

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un usuario se conecta con un proveedor externo compatible y autoriza la conexión:

* Auth0 obtiene tokens de acceso y de actualización mediante scopes de OAuth 2.0, y el usuario aprueba explícitamente los permisos solicitados.
* Auth0 almacena de forma segura los tokens de cada cuenta conectada en Token Vault. Como cada cuenta conectada está vinculada al perfil del usuario, la aplicación puede acceder a APIs y servicios externos sin necesidad de que el usuario vuelva a autorizar la conexión.
* La aplicación llama a Auth0 para intercambiar un token válido de Auth0 del usuario por un token de acceso de un proveedor externo emitido para ese usuario. Para obtener más información, consulta [Intercambios de tokens compatibles](#supported-token-exchanges).
* Con el token de acceso del proveedor externo, tu aplicación puede llamar a APIs externas en nombre del usuario.

<div id="use-with-organizations">
  ## Uso con Organizations
</div>

Si tu aplicación usa [Organizations](/docs/es/manage-users/organizations), primero autentica al usuario con la organización de destino y luego inicia el flujo de Connected Accounts para el proveedor externo que quieras usar.

Cuando usas Organizations con Token Vault:

* Organizations definen el contexto de inicio de sesión, identidad de marca y política de acceso para la sesión.
* Connected Accounts sigue vinculando el proveedor externo con el perfil de usuario individual de Auth0.
* Cada miembro de la organización conecta y autoriza su propia cuenta externa; Token Vault no crea una cuenta de organización compartida.

<div id="supported-token-exchanges">
  ## Intercambios de tokens compatibles
</div>

Para llamar a las APIs de un proveedor externo, tu aplicación debe intercambiar un token válido de Auth0 por un token de acceso de un proveedor externo en Token Vault. El tipo de token de Auth0 que se usa en el intercambio depende del tipo de aplicación cliente y del caso de uso.

Las aplicaciones pueden acceder a Token Vault mediante los siguientes intercambios de tokens:

| Intercambio de tokens                                                                                                                 | Descripción                                                                                                         | Tipo de aplicación cliente                                                                                                                                                                  |
| ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Intercambio de token de actualización](/docs/es/secure/tokens/token-vault/refresh-token-exchange-with-token-vault)                      | Intercambia un token de actualización de Auth0 por un token de acceso de un proveedor externo.                      | Aplicaciones que necesitan mantener la sesión de un usuario y acceder a APIs externas cuando el usuario no está usando activamente la aplicación, como aplicaciones web, móviles y nativas. |
| [Intercambio de token de acceso](/docs/es/secure/tokens/token-vault/access-token-exchange-with-token-vault)                              | Intercambia un token de acceso de Auth0 por un token de acceso de un proveedor externo.                             | APIs o microservicios que necesitan intercambiar tokens de acceso que han recibido de otros servicios o aplicaciones, como una aplicación de página única (SPA).                            |
| [Intercambio de tokens de workers con privilegios](/docs/es/secure/tokens/token-vault/privileged-worker-token-exchange-with-token-vault) | Intercambia un token JWT bearer firmado por un token de acceso o un token de actualización de un proveedor externo. | Aplicaciones de backend o workers de servicio en flujos de servicio a servicio (máquina a máquina).                                                                                         |
