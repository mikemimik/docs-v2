Revisar los registros para evaluar el impacto de un ataque es un paso fundamental en tu plan de respuesta a incidentes. En esta página verás cómo acceder a los registros en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, así como algunos ejemplos de consultas de búsqueda en los registros para encontrar indicios de un ataque y revisar la actividad de la cuenta.

<div id="check-auth0-logs">
  ## Consulta los logs de Auth0
</div>

1. [Inicia sesión en el Auth0 Dashboard](https://manage.auth0.com/#)
2. La página Logs se encuentra en **monitoreo** en el menú de la izquierda.
3. En la página Logs, verás una barra de búsqueda junto con una opción de filtro y un selector de fecha.

<Frame>![Dashboard monitoreo logs](/docs/images/cdy7uua7fh8z/2l9LfKEjrprVmHfWt80VX0/4e48bc6ad3a8b591014e23fdb7353405/2025-01-28_15-07-43.png)</Frame>

Selecciona un evento de log de la lista para ver un **Summary** del evento junto con más **Details**, incluido el Raw JSON.

<div id="log-structure">
  ### Estructura del registro
</div>

Cada evento de log tiene los siguientes campos:

| Campo           | Descripción                                                                        |
| --------------- | ---------------------------------------------------------------------------------- |
| `date`          | Marca de tiempo de cuándo ocurrió este evento.                                     |
| `log_id`        | El ID del evento de log                                                            |
| `type`          | El tipo de evento de log                                                           |
| `description`   | La descripción del evento.                                                         |
| `connection`    | El nombre de la conexión relacionada con el evento.                                |
| `connection_id` | El ID de la conexión relacionada con el evento.                                    |
| `client_id`     | El ID de cliente relacionado con el evento                                         |
| `client_name`   | El nombre del cliente relacionado con el evento.                                   |
| `ip`            | La dirección IP desde la que se originó la solicitud que provocó el evento de log. |
| `user_agent`    | El agente de usuario relacionado con el evento.                                    |
| `details`       | Un objeto que contiene información adicional sobre este evento de log.             |
| `user_id`       | El ID de usuario relacionado con el evento.                                        |
| `user_name`     | El nombre de usuario relacionado con el evento.                                    |
| `strategy`      | La estrategia de conexión relacionada con el evento.                               |
| `strategy_type` | El tipo de estrategia de conexión relacionado con el evento.                       |

<div id="indicators-of-an-attack">
  ## Indicadores de un ataque
</div>

Detectar un ataque en una fase temprana puede ser difícil, pero aquí tiene algunos aspectos que puede buscar en sus logs, junto con consultas de búsqueda de ejemplo:

* Un gran número de inicios de sesión fallidos con nombres de usuario no válidos o intentos de inicio de sesión de usuarios inexistentes.

  * `type:"fu"`
  * `description:"missing username parameter"`
  * `description:"Wrong email or password"`
* Un gran número de cuentas que alcanzan el límite de intentos de inicio de sesión fallidos.

  * `type:"limit_wc"`
* Un gran número de intentos de inicio de sesión con una contraseña filtrada.

  * `type:"pwd_leak"`

Durante su investigación, tome nota de las direcciones IP, las aplicaciones a las que se dirigen los ataques y las conexiones o los <Tooltip tip="Identity Provider (IdP): Servicio que almacena y administra identidades digitales." cta="View Glossary" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> utilizados.

<div id="identify-compromised-user-accounts">
  ## Identificar cuentas de usuario comprometidas
</div>

Para identificar cuentas de usuario que podrían haber sido comprometidas, puede buscar:

* Eventos de inicio de sesión correctos desde una dirección IP sospechosa:

  * `type:"s" AND ip:"99.xxx.xxx.xxx"`

<div id="check-compromised-user-account-activity">
  ## Revisa la actividad de una cuenta de usuario comprometida
</div>

Después de identificar una cuenta de usuario comprometida, te convendrá revisar la actividad de la cuenta:

* Busca otros eventos de log con el mismo `user_id`: `user_id:"auth0|ABC123"`
* Revisa los campos `client_name` o `client_id` del evento de log para ver a qué aplicaciones se accedió. Toma nota de cuándo se produjo el acceso.
* Revisa si hubo acceso administrativo o cambios en la configuración de Auth0
* Busca llamadas recientes a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>: `type:"sapi"`

<div id="delete-or-block-users-from-the-dashboard">
  ## Elimina o bloquea usuarios desde el Dashboard
</div>

1. Ve a [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
2. Busca al usuario que quieres eliminar o bloquear.
3. Haz clic en el botón &quot;**...**&quot; situado en el extremo derecho del usuario.
4. Selecciona **Block** o **Delete** y confirma.
