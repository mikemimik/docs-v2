title: Direcciones IP de Auth0 para listas de direcciones permitidas
---

Si tienes código personalizado que se ejecuta en Auth0 y realiza llamadas a un servicio dentro de tu red, o si has configurado un proveedor SMTP local en Auth0, es posible que necesites configurar tu firewall para permitir tráfico entrante desde Auth0.

Entre las funcionalidades que pueden requerir que permitas tráfico entrante desde Auth0 se incluyen:

* [Actions](/docs/es/customize/actions)
* [Rules](/docs/es/customize/rules)
* [Hooks](/docs/es/customize/hooks)
* [Custom Database Action Scripts](/docs/es/authenticate/database-connections/custom-db/templates)
* [Log Streams](/docs/es/customize/log-streams)
* [Proveedores externos de correo electrónico SMTP](/docs/es/customize/email/smtp-email-providers)

<div id="outbound-calls">
  ## Llamadas salientes
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las direcciones IP pueden cambiar. Si se produce un cambio en una dirección IP, Auth0 envía notificaciones varios meses antes de que entre en vigor. Las listas están actualizadas en el momento de la redacción.
</Callout>

Cuando Auth0 realiza llamadas salientes, las direcciones IP son estáticas. Auth0 traduce las direcciones IP internas a una de las opciones que se muestran al comunicarse mediante NAT.

<div id="public-cloud">
  ### Public Cloud
</div>

Para los tenants de Public Cloud, las direcciones IP que debe permitir en su firewall varían según la región del tenant.

<div id="machine-readable-list">
  #### Lista en formato legible por máquina
</div>

Puedes acceder a una lista en formato legible por máquina de los rangos CIDR de IP de Auth0 CDN en: https://cdn.auth0.com/ip-ranges.json.

<div id="schema">
  #### Esquema
</div>

La lista en formato legible por máquina contiene las siguientes propiedades:

| **Propiedad**     | **Tipo de dato**    | **Descripción**                                                                                                                                                                                                                                                                                                                                                                           |
| :---------------- | :------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `last_updated_at` | timestamp (ISO8601) | Cuándo se actualizó la lista por última vez.                                                                                                                                                                                                                                                                                                                                              |
| `regions`         | objeto              | Colección de [regiones de tenants de Auth0](/docs/es/get-started/auth0-overview/create-tenants#region-locality-and-sub-locality). Cada objeto de región de tenant contiene la siguiente propiedad: * `ipv4_cidrs`: Arreglo de strings. Lista de rangos CIDR IPv4 para la región indicada.                                                                                                    |
| `changelog`       | arreglo de objetos  | Registro histórico de todas las adiciones y eliminaciones de rangos CIDR IP. Cada objeto contiene las siguientes propiedades: * `date`: String. Marca de tiempo en formato YYYY-MM-DD. * `region`: String. Región de Auth0 afectada. * `action`: String. Acción realizada. Los valores posibles son: `add`, `remove`. * `ipv4_cidrs`: Arreglo de strings. Lista de rangos CIDR afectados. |

<div id="detecting-changes">
  #### Detección de cambios
</div>

Puedes detectar cambios en la lista en formato legible por máquina mediante programación, ya sea a partir de la propiedad last&#95;updated&#95;at o de la respuesta de Auth0 a una solicitud HTTP condicional. Auth0 actualiza esta lista antes de que haya cualquier impacto funcional en los tenants existentes. Programa una comprobación semanal automatizada para supervisar los cambios y mantener una configuración de firewall adecuada.

Si envías una solicitud HTTP condicional, puedes usar el encabezado If-None-Match o If-Modified-Since. Si la lista ha cambiado, Auth0 devuelve una respuesta HTTP 200 (OK) con la lista actualizada en el cuerpo. En caso contrario, Auth0 devuelve una respuesta 304 (Not Modified) sin cuerpo.
Para obtener más información sobre las solicitudes HTTP condicionales, consulta [RFC 9110: Semántica HTTP en RFC Editor](https://www.rfc-editor.org/rfc/rfc9110.html#name-conditional-requests).

<div id="example-of-conditional-http-request-if-none-match-header">
  #### Ejemplo de una solicitud HTTP condicional (encabezado If-None-Match)
</div>

```curl
# La primera solicitud.
$ curl https://cdn.auth0.com/ip-ranges.json
HTTP/2 200
last-modified: Tue, 22 Apr 2025 07:50:08 GMT
etag: "1652cbe64a22099d62424fbec1022da5"
...

# El contenido no ha cambiado, la respuesta es `HTTP 304`
$ curl https://cdn.auth0.com/ip-ranges.json \
       -H "If-None-Match: 1652cbe64a22099d62424fbec1022da5" 
HTTP/2 304
last-modified: Tue, 22 Apr 2025 07:50:08 GMT
etag: "1652cbe64a22099d62424fbec1022da5"
...

# El contenido ha cambiado, la respuesta es `HTTP 200` con nuevo `etag`:
$ curl https://cdn.auth0.com/ip-ranges.json \
       -H "If-None-Match: 1652cbe64a22099d62424fbec1022da5" 
HTTP/2 200
last-modified: Mon, 07 Jul 2025 01:04:18 GMT
etag: "8e58c97d39a67706c644cbe3b5c682d3"
...
```

<div id="private-cloud">
  ### Private Cloud
</div>

Para los tenants de Private Cloud, las direcciones IP que debe permitir en su firewall son exclusivas del entorno del tenant. Auth0 puede recibir las direcciones IP privadas de su tenant si habilita funcionalidades como Tenant Logs, <Tooltip tip="Suspicious IP Throttling: mecanismo de protección contra ataques que protege su tenant frente a inicios de sesión sospechosos dirigidos a demasiadas cuentas desde una sola dirección IP." cta="Ver glosario" href="/docs/es/glossary?term=Suspicious+IP+throttling">Suspicious IP throttling</Tooltip>, Custom Databases y Actions que dependen de ellas.

Estas direcciones IP se conocen como **Primary Egress IPs** y figuran en los datos de configuración del entorno disponibles en el [Auth0 Support Center](https://support.auth0.com/tenants/private).

<div id="inbound-calls">
  ## Llamadas entrantes
</div>

Las direcciones IP relacionadas con las llamadas entrantes a Auth0 pueden variar debido a la falta de direcciones IP fijas en los equilibradores de carga. En este caso, las reglas del firewall deben basarse en el nombre del servicio (por ejemplo: `<YOUR_TENANT>.<YOUR_REGION>.auth0.com`).

Si su suscripción de Auth0 le permite configurar un <Tooltip tip="Dominio personalizado: Dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip> autogestionado, puede configurarlo para que tenga una dirección IP estática. Los dominios personalizados autogestionados le dan control sobre el punto de entrada de la red y le permiten asegurarse de que la dirección IP sea fija. Para obtener información sobre los planes de suscripción, consulte [Precios de Auth0](https://auth0.com/pricing).
