**Publicado**: 31 de marzo de 2020

**Números CVE**: CVE-2020-5391, CVE-2020-5392, CVE-2020-6753, CVE-2020-7948, CVE-2020-7947

**Reconocimiento**: Muhamad Visat

<div id="overview">
  ## Información general
</div>

Auth0 ha lanzado una nueva versión mayor del [plugin de WordPress para Auth0](https://github.com/auth0/wp-auth0) para solucionar varias vulnerabilidades.

Le recomendamos revisar los siguientes avisos de seguridad y actualizar a la nueva versión mayor:

* Falta de controles CSRF para el campo de dominio en el plugin de WordPress para Auth0: [CVE-2020-5391](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-5391)
* XSS almacenado en el plugin de WordPress para Auth0 (página Settings): [CVE-2020-5392](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-5392)
* XSS almacenado en el plugin de WordPress para Auth0 (varias páginas): [CVE-2020-6753](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-6753)
* Vulnerabilidades de inyección CSV en el plugin de WordPress para Auth0: [CVE-2020-7947](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-7947)
* Referencia directa insegura a objetos en el plugin de WordPress para Auth0: [CVE-2020-7948](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-7948)

<div id="am-i-affected">
  ## ¿Me afecta?
</div>

Los clientes que utilen cualquier versión del plugin de WordPress para Auth0 3.11.3 o anterior pueden verse afectados.

<div id="how-to-fix-that">
  ## ¿Cómo corregirlo?
</div>

Los clientes que usan [plugin de WordPress para Auth0](https://github.com/auth0/wp-auth0) deben actualizar a la versión 4.0.0 o posterior.

<div id="will-this-update-impact-my-users">
  ## ¿Esta actualización afectará a mis usuarios?
</div>

Las [notas de la versión](https://github.com/auth0/wp-auth0/blob/master/CHANGELOG.md) ofrecen información más detallada sobre los cambios realizados, y las [instrucciones de migración](https://github.com/auth0/wp-auth0/blob/master/MIGRATE-v3-TO-v4.md) ofrecen más información sobre el proceso de actualización.
