**Publicado**: 31 de marzo de 2020

**Números de CVE**: CVE-2020-5391, CVE-2020-5392, CVE-2020-6753, CVE-2020-7948, CVE-2020-7947

**Crédito**: Muhamad Visat

<div id="overview">
  ## Resumen
</div>

Auth0 ha publicado una nueva versión principal del [plugin de WordPress para Auth0](https://github.com/auth0/wp-auth0) para corregir varias vulnerabilidades.

Le recomendamos revisar los siguientes avisos de seguridad y actualizar a la nueva versión principal:

* Faltan controles CSRF para el campo de dominio en el plugin de Auth0 para WP: [CVE-2020-5391](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-5391)
* XSS almacenado en el plugin de Auth0 para WP (página de configuración): [CVE-2020-5392](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-5392)
* XSS almacenado en el plugin de Auth0 para WP (varias páginas): [CVE-2020-6753](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-6753)
* Vulnerabilidades de inyección CSV en el plugin de Auth0 para WP: [CVE-2020-7947](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-7947)
* Referencia directa insegura a objetos en el plugin de Auth0 para WP: [CVE-2020-7948](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-7948)

<div id="am-i-affected">
  ## ¿Me afecta?
</div>

Los clientes que utilicen la versión 3.11.3 o cualquier versión anterior del plugin de WordPress para Auth0 pueden verse afectados.

<div id="how-to-fix-that">
  ## ¿Cómo se soluciona?
</div>

Los clientes que usan [plugin de WordPress para Auth0](https://github.com/auth0/wp-auth0) deben actualizar a la versión 4.0.0 o superior.

<div id="will-this-update-impact-my-users">
  ## ¿Esta actualización afectará a mis usuarios?
</div>

Las [notas de la versión](https://github.com/auth0/wp-auth0/blob/master/CHANGELOG.md) ofrecen información más detallada sobre los cambios realizados, y las [instrucciones de migración](https://github.com/auth0/wp-auth0/blob/master/MIGRATE-v3-TO-v4.md) brindan más detalles sobre el proceso de actualización.
