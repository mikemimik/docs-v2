**Publicado**: 21 de diciembre de 2022

**Números de CVE**: CVE-2022-23539, CVE-2022-23541, CVE-2022-23540

<div id="overview">
  ### Información general
</div>

Auth0 ha lanzado una nueva versión mayor de la biblioteca `jsonwebtoken` para corregir cuatro vulnerabilidades.

Le recomendamos revisar los siguientes avisos de seguridad y actualizar a la nueva versión mayor:

* Un tipo de clave sin restricciones podría dar lugar al uso de claves heredadas: [CVE-2022-23539](https://github.com/auth0/node-jsonwebtoken/security/advisories/GHSA-8cf7-32gw-wr33)
* Una implementación insegura de la función de recuperación de claves podría dar lugar a tokens públicos/privados falsificables de RSA a HMAC: [CVE-2022-23541](https://github.com/auth0/node-jsonwebtoken/security/advisories/GHSA-hjrf-2m68-5959)
* Un algoritmo predeterminado inseguro en <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, con frecuencia, de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=jwt">jwt</Tooltip>.verify() podría provocar la elusión de la validación de la firma: [CVE-2022-23540](https://github.com/auth0/node-jsonwebtoken/security/advisories/GHSA-qwph-4952-7xr6)

<div id="am-i-affected">
  ### ¿Me afecta?
</div>

Podrías verte afectado si usas `jsonwebtoken` en una versión &lt;= 8.5.1, según la configuración. Consulta los avisos de seguridad individuales para obtener más detalles.

<div id="how-to-fix-that">
  ### ¿Cómo solucionarlo?
</div>

Si estás usando `jsonwebtoken`, actualiza a la versión 9.0.0 o superior. Puede que necesites alguna configuración adicional. Consulta los avisos de seguridad correspondientes para obtener más detalles.

<div id="will-this-update-impact-my-users">
  ### ¿Esta actualización afectará a mis usuarios?
</div>

La actualización a la versión 9.0.0 puede afectar a tus usuarios según tu configuración y las necesidades de tu aplicación. Consulta los avisos de seguridad individuales para obtener más detalles.
