Cuando administras la ACL del tenant (ACL) con la [Management API](/docs/es/api/management/v2), los siguientes parámetros de las reglas definen su comportamiento.

<div id="signals">
  ## Señales
</div>

<ParamField path="asns" type="string[]">
  Lista de números de sistema autónomo (ASN)
</ParamField>

<ParamField path="connecting_ipv4_cidrs" type="string[]">
  Lista de direcciones IPv4 o rangos CIDR de la infraestructura que se conecta directamente al edge de Auth0. Úsala para compararla con la IP de tu proxy inverso o VPN, en lugar de con la IP del usuario final.
</ParamField>

<ParamField path="connecting_ipv6_cidrs" type="string[]">
  Lista de direcciones IPv6 o rangos CIDR de la infraestructura que se conecta directamente al edge de Auth0. Úsala para compararla con la IP de tu proxy inverso o VPN, en lugar de con la IP del usuario final.
</ParamField>

<ParamField path="hostnames" type="string[]">
  Lista de `hostname` relevantes para el tenant. Se compara con la Indicación de Nombre del Servidor (SNI) de la solicitud. Los valores admitidos incluyen:

  , `hostname` canónicos (por ejemplo, `t1.us.auth0.com`

  * dominios personalizados administrados por Auth0 (por ejemplo, `auth.acmetest.org`
  * dominios personalizados autogestionados (por ejemplo, `cd-t1…edge.tenants.us.auth0.com`

  Puedes usar esta señal para deshabilitar o restringir el acceso a tu dominio canónico y, al mismo tiempo, permitir tráfico a través de dominios personalizados.
</ParamField>

<ParamField path="ipv4_cidrs" type="string[]">
  Lista de direcciones IPv4 o rangos CIDR.
</ParamField>

<ParamField path="ipv6_cidrs" type="string[]">
  Lista de direcciones IPv6 o rangos CIDR.
</ParamField>

<ParamField path="geo_country_codes" type="string[]">
  Lista de códigos de país ISO 3166-1 alfa-2.
</ParamField>

<ParamField path="geo_subdivision_codes" type="string[]">
  Lista de códigos de subdivisión ISO 3166-2.
</ParamField>

<ParamField path="ja3_fingerprints" type="string[]">
  Lista de huellas digitales TLS JA3.
</ParamField>

<ParamField path="ja4_fingerprints" type="string[]">
  Lista de huellas digitales TLS JA4.
</ParamField>

<ParamField path="user_agents" type="string[]">
  Lista de dispositivos o navegadores del cliente.
</ParamField>

<div id="conditions">
  ## Condiciones
</div>

<ParamField path="match" type="object">
  Devuelve Successful si la señal proporcionada es equivalente a cualquiera de los valores proporcionados.
</ParamField>

<ParamField path="not_match" type="object">
  Devuelve Successful si la señal proporcionada no es equivalente a ninguno de los valores proporcionados.
</ParamField>

<div id="actions">
  ## Actions
</div>

<ParamField path="allow" type="boolean">
  Permite que el tráfico pase sin cambios.
</ParamField>

<ParamField path="block" type="boolean">
  Impide que el tráfico acceda a los scopes especificados.
</ParamField>

<ParamField path="redirect" type="boolean">
  Redirige el tráfico a la ubicación proporcionada.
</ParamField>

<ParamField path="redirect_uri" type="string">
  URI a la que se redirige el tráfico.
</ParamField>

<ParamField path="log" type="boolean">
  Modo de monitoreo. No se realiza ninguna acción, pero los resultados se incluyen en el evento de log de la ACL del tenant.
</ParamField>

<div id="scopes">
  ## Scopes
</div>

<ParamField path="tenant">
  Aplica la ACL del tenant a los scopes `management` y `authentication`.
</ParamField>

<ParamField path="management">
  Aplica la ACL del tenant a las solicitudes enviadas a `{yourDomain}/api/v2/\*}` y `{yourDomain}/scim/\*}`.
</ParamField>

<ParamField path="dcr">
  Controla el acceso al punto de conexión de registro dinámico de clientes `/oidc/register`.
</ParamField>

<ParamField path="authentication">
  Aplica la ACL del tenant a las solicitudes enviadas a cualquier ruta no cubierta por el scope `management`.
</ParamField>
