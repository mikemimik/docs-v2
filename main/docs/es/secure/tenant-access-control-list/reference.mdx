---

Cuando administras la ACL del tenant con la [Management API](/docs/es/api/management/v2), los siguientes parámetros de las reglas definen su comportamiento.

<div id="signals">
  ## Señales
</div>

<ParamField path="asns" type="string[]">
  Lista de números de sistema autónomo (ASN)
</ParamField>

<ParamField path="connecting_ipv4_cidrs" type="string[]">
  Lista de direcciones IPv4 o rangos CIDR de la infraestructura que se conecta directamente al edge de Auth0. Úselo para compararlo con la IP de su proxy inverso o VPN, en lugar de la IP del usuario final.
</ParamField>

<ParamField path="connecting_ipv6_cidrs" type="string[]">
  Lista de direcciones IPv6 o rangos CIDR de la infraestructura que se conecta directamente al edge de Auth0. Úselo para compararlo con la IP de su proxy inverso o VPN, en lugar de la IP del usuario final.
</ParamField>

<ParamField path="hostnames" type="string[]">
  Lista de nombres de host relevantes para el tenant. Esto se compara con la Server Name Indication (SNI) de la solicitud. Los valores compatibles incluyen:

  , nombres de host canónicos (por ejemplo, `t1.us.auth0.com`

  * dominios personalizados administrados por Auth0 (por ejemplo, `auth.acmetest.org`
  * dominios personalizados autogestionados (por ejemplo, `cd-t1…edge.tenants.us.auth0.com`

  Puede usar esta señal para deshabilitar o restringir el acceso a su dominio canónico mientras permite el tráfico a través de dominios personalizados.
</ParamField>

<ParamField path="ipv4_cidrs" type="string[]">
  Lista de direcciones IPv4 o rangos CIDR.
</ParamField>

<ParamField path="ipv6_cidrs" type="string[]">
  Lista de direcciones IPv6 o rangos CIDR.
</ParamField>

<ParamField path="geo_country_codes" type="string[]">
  Lista de códigos de país ISO 3166-1 alfa-2.
</ParamField>

<ParamField path="geo_subdivision_codes" type="string[]">
  Lista de códigos de subdivisión ISO 3166-2.
</ParamField>

<ParamField path="ja3_fingerprints" type="string[]">
  Lista de huellas digitales TLS JA3.
</ParamField>

<ParamField path="ja4_fingerprints" type="string[]">
  Lista de huellas digitales TLS JA4.
</ParamField>

<ParamField path="user_agents" type="string[]">
  Lista de dispositivos cliente o navegadores.
</ParamField>

<div id="conditions">
  ## Condiciones
</div>

<ParamField path="match" type="object">
  Se considera satisfactorio si la señal proporcionada y cualquiera de los valores proporcionados son equivalentes.
</ParamField>

<ParamField path="not_match" type="object">
  Se considera satisfactorio si la señal proporcionada y ninguno de los valores proporcionados son equivalentes.
</ParamField>

<div id="actions">
  ## Actions
</div>

<ParamField path="allow" type="boolean">
  Permite que el tráfico pase sin verse afectado.
</ParamField>

<ParamField path="block" type="boolean">
  Impide que el tráfico acceda a los scopes especificados.
</ParamField>

<ParamField path="redirect" type="boolean">
  Redirige el tráfico a la ubicación proporcionada.
</ParamField>

<ParamField path="redirect_uri" type="string">
  URI a la que se redirige el tráfico.
</ParamField>

<ParamField path="log" type="boolean">
  Modo de monitoreo. No se realiza ninguna acción, pero los resultados se incluyen en el evento de log de la ACL del tenant.
</ParamField>

<div id="scopes">
  ## Scopes
</div>

<ParamField path="tenant">
  Aplica la ACL del tenant a los scopes `management` y `authentication`.
</ParamField>

<ParamField path="management">
  Aplica la ACL del tenant a las solicitudes enviadas a `{yourDomain}/api/v2/\*}` y `{yourDomain}/scim/\*}`.
</ParamField>

<ParamField path="dcr">
  Controla el acceso al endpoint de registro dinámico de clientes `/oidc/register`.
</ParamField>

<ParamField path="authentication">
  Aplica la ACL del tenant a las solicitudes enviadas a cualquier endpoint no cubierto por el scope `management`.
</ParamField>
