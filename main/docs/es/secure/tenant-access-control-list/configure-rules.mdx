---
title: Cómo configurar las reglas de la ACL del tenant
sidebarTitle: Configurar reglas
description: Defina la señal, la condición, la acción, el scope y la prioridad de una regla para establecer su comportamiento. Use la Management API o los SDKs, CLI o Terraform Provider de Auth0 para configurar y administrar la ACL del tenant.
---

Una regla de la ACL del tenant se compone de los siguientes elementos:

* **Signal**: La señal es un dato identificador que proporciona la solicitud entrante, como la dirección IP, la geolocalización o el agente de usuario.

* **Condition**: La condición es la combinación de un operador (como `match`) y un conjunto de valores (como una lista de direcciones IP).

* **Action**: La acción es la directiva que ejecuta su regla si se cumplen los criterios, por ejemplo, permitir, bloquear o redirigir.

* **Scope**: El scope indica para qué conjuntos de endpoints se aplica la regla, incluidos la Authentication API, la Management API o todo su tenant.

* **Priority**: La prioridad define el orden en que se ejecuta la regla en relación con las demás reglas.

Para obtener información detallada sobre estas propiedades de las reglas de la ACL del tenant y cómo usarlas, consulte la [Referencia de parámetros del endpoint de la Management API](./reference).

<Note>
  La ACL del tenant usa una [lógica uniforme de evaluación de reglas](./rule-evaluation) para determinar qué acción, si corresponde, debe aplicarse a una solicitud determinada.

  Asigne cuidadosamente las prioridades para crear políticas granulares de control de acceso para su caso de uso.
</Note>

<div id="create-and-delete-rules">
  ## Crear y eliminar reglas
</div>

Puede crear y eliminar reglas de ACL del tenant mediante los siguientes endpoints de la Management API. Necesita un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los scopes indicados.

| Acción             | Endpoint                                                                                              | Scope requerido       |
| ------------------ | ----------------------------------------------------------------------------------------------------- | --------------------- |
| Crear una regla    | [Crear lista de control de acceso](/docs/es/api/management/v2/network-acls/post-network-acls)            | `create:network_acls` |
| Eliminar una regla | [Eliminar lista de control de acceso](/docs/es/api/management/v2/network-acls/delete-network-acls-by-id) | `delete:network_acls` |

<div id="modify-rules">
  ## Modificar reglas
</div>

Puede modificar las reglas de ACL del tenant mediante los siguientes endpoints de la Management API. Necesita un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los scopes indicados.

| Acción                 | Endpoint                                                                                                                 | Scope requerido       |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------ | --------------------- |
| Actualizar una regla   | [Actualización parcial de una lista de control de acceso](/docs/es/api/management/v2/network-acls/patch-network-acls-by-id) | `update:network_acls` |
| Sobrescribir una regla | [Actualizar lista de control de acceso](/docs/es/api/management/v2/network-acls/put-network-acls-by-id)                     | `update:network_acls` |

<div id="enable-or-disable-rules">
  ### Habilitar o deshabilitar reglas
</div>

<Tabs>
  <Tab title="Management API">
    Puedes habilitar o deshabilitar una regla con el endpoint [Actualizar lista de control de acceso](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) de Management API.

    Establece la propiedad `active` del objeto `rule` en `true` o `false`, respectivamente.
  </Tab>

  <Tab title="Dashboard">
    Para habilitar o deshabilitar una regla desde el Dashboard, ve a [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control).

    En la lista de reglas existentes, puedes habilitar o deshabilitar reglas específicas con el interruptor que aparece junto a cada una.

    ![Habilitar o deshabilitar una regla de ACL del tenant en el Auth0 Dashboard](/docs/images/cdy7uua7fh8z/29b92e102142d3cb20e69a40e292f6b7/Screenshot_2025-09-03_at_12.34.05.png)

    La columna **Status** muestra el estado de la regla (**Active** o **Disabled**).
  </Tab>
</Tabs>

<div id="view-rules">
  ## Ver reglas
</div>

<Tabs>
  <Tab title="Management API">
    Puede ver las reglas existentes de la ACL del tenant mediante los siguientes endpoints de la Management API. Necesita un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los scopes indicados.

    | Acción               | Endpoint                                                                                                                                    | Scope requerido     |
    | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
    | Ver una regla        | [Obtener una entrada específica de la lista de control de acceso de un tenant](/docs/es/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls` |
    | Ver todas las reglas | [Obtener todas las entradas de la lista de control de acceso de un tenant](/docs/es/api/management/v2/network-acls/get-network-acls)           | `read:network_acls` |
  </Tab>

  <Tab title="Dashboard">
    Vaya a [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) para ver la lista de sus reglas existentes.

    | Nombre de la columna | Descripción                                                                                  | Campo relacionado de la respuesta de la API |
    | -------------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------- |
    | Prioridad            | Número ordinal que indica cuándo se evalúa la regla                                          | `priority`                                  |
    | Nombre               | Descripción del propósito o la funcionalidad de la regla                                     | `description`                               |
    | Action               | La [Action](/docs/es/secure/tenant-access-control-list/reference#actions) que realiza la regla. | `action`                                    |
    | Estado               | Valor que indica si la regla está activa o deshabilitada.                                    | `active`                                    |
    | Última actualización | Fecha y hora de la última actualización de la regla.                                         | `updated_at`                                |

    ![Ver la lista de reglas de la ACL del tenant en el Auth0 Dashboard](/docs/images/cdy7uua7fh8z/4093f9a39aba48bdf8d579c134dc4da5/Screenshot_2025-09-03_at_12.33.47.png)
  </Tab>
</Tabs>
