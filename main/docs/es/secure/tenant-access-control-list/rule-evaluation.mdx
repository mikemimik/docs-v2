---
title: Evaluación de rules para la ACL del tenant
sidebarTitle: Evaluación de rules
description: Aprende cómo el motor determinista de la ACL del tenant evalúa las coincidencias de rules y su ejecución.
---

La ACL del tenant usa un motor determinista para evaluar las solicitudes entrantes. Para determinar si debe realizarse una Action (permitir, bloquear o redirigir), el motor sigue estos pasos de evaluación:

1. Orden de evaluación (prioridad)

2. Coincidencia de condiciones (señal)

3. Terminación tras la coincidencia, con excepción del modo de monitoreo

<div id="evaluation-order">
  ## Orden de evaluación
</div>

La ACL del tenant evalúa las rules en orden ascendente según su número de prioridad. La evaluación comienza con el número más bajo, prioridad 0, y continúa de forma secuencial.

Si dos rules pudieran coincidir con una misma solicitud, se activará primero la que tenga el número de prioridad más bajo.

<div id="condition-matching">
  ## Coincidencia de condiciones
</div>

Para que una sola rule active su Action, debe cumplir la lógica de coincidencia. Esta lógica determina cómo el motor trata múltiples valores y distintos tipos de señal:

| Tipo de lógica        | Comportamiento | Descripción                                                                                                                                                                                                                                             | Ejemplo                                                                           |
| --------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
| Coincidencia de señal | OR             | Dentro de una señal específica, una solicitud debe coincidir con al menos una entrada de la lista para cumplir ese requisito de señal.                                                                                                                  | Si `ipv4_cidrs` contiene varios CIDR de IP, al menos uno de ellos debe coincidir. |
| Múltiples señales     | AND            | Si una rule contiene varios tipos de señal diferentes, la solicitud debe coincidir con todos los tipos de señal proporcionados. Como excepción, si una rule contiene tanto `ipv4_cidrs` como `ipv6_cidrs`, solo es necesario que coincida uno de ellos. | Si una rule tiene tanto `ipv4_cidrs` como `asn`, ambos deben coincidir.           |

Los tipos de señal que no se proporcionen en una rule se ignoran y no afectan la lógica de evaluación de la rule.

<div id="match-termination">
  ## Finalización de coincidencia
</div>

La ACL del tenant aplica la primera coincidencia. Una vez que se cumplen por completo las condiciones de una rule, el comportamiento depende de si la rule está en [modo de monitoreo](./enable-monitoring-mode) o no.

| Comportamiento | Modo de monitoreo deshabilitado                                                                             | Modo de monitoreo habilitado                                            |
| -------------- | ----------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| **Registro**   | Se genera un evento de log `acls_summary`.                                                                  | Se genera un evento de log `acls_summary`.                              |
| **Ejecución**  | La Action definida de la rule se ejecuta de inmediato.                                                     | La Action definida de la rule no se ejecuta.                           |
| **Evaluación** | La evaluación finaliza. No se evalúan rules posteriores de la ACL del tenant ni rules de menor prioridad. | La evaluación continúa con la siguiente rule de la lista de prioridad. |
