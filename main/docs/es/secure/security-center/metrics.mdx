---

Security Center usa eventos de log del tenant para identificar patrones que suelen indicar tipos de ataque conocidos. Clasificamos los patrones de eventos de log del tenant en estas categorías: amenazas de credential stuffing, amenazas de ataques al registro y amenazas de evasión de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>.

<Warning>
  La clasificación de los códigos de tipo de evento puede cambiar. Evita implementar soluciones que dependan de las definiciones actuales de los códigos de eventos de log.
</Warning>

<div id="credential-stuffing">
  ## Credential stuffing
</div>

Identificamos amenazas de credential stuffing en el plazo de una hora con los siguientes códigos de evento:

| Código de evento | Evento                                                                                       |
| ---------------- | -------------------------------------------------------------------------------------------- |
| `f`              | Inicio de sesión fallido del usuario                                                         |
| `fu`             | Inicio de sesión fallido del usuario debido a un nombre de usuario no válido                 |
| `fp`             | Inicio de sesión fallido del usuario debido a una contraseña no válida                       |
| `pwd_leak`       | Intento de inicio de sesión con una contraseña filtrada                                      |
| `limit_wc`       | IP bloqueada por &gt;10 intentos de inicio de sesión fallidos contra una sola cuenta         |
| `limit_sul`      | Usuario bloqueado por &gt;20 inicios de sesión por minuto desde la misma dirección IP        |
| `limit_mu`       | IP bloqueada por &gt;100 intentos de inicio de sesión fallidos o &gt;50 intentos de registro |

<div id="signup-attack">
  ## Ataque al registro
</div>

Identificamos amenazas de ataque al registro en el plazo de una hora con los siguientes códigos de evento:

| Código de evento | Evento           |
| ---------------- | ---------------- |
| `fs`             | Registro fallido |

<div id="mfa-bypass">
  ## Evasión de MFA
</div>

Identificamos amenazas de evasión de MFA en el plazo de una hora con los siguientes códigos de evento:

| Código de evento                | Evento                                  |
| ------------------------------- | --------------------------------------- |
| `gd_send_email`                 | Correo electrónico enviado              |
| `gd_send_pn`                    | Notificación push enviada               |
| `gd_send_sms`                   | SMS enviado                             |
| `gd_send_voice`                 | Llamada de voz enviada                  |
| `gd_auth_failed`                | Error en la autenticación OTP           |
| `gd_auth_rejected`              | Autenticación OTP rechazada             |
| `gd_otp_rate_limit_exceed`      | Demasiados errores de autenticación OTP |
| `gd_recovery_failed`            | Error de recuperación                   |
| `gd_recovery_rate_limit_exceed` | Demasiados errores de recuperación      |
| `gd_webauthn_challenge_failed`. | Error del navegador de WebAuthn         |
