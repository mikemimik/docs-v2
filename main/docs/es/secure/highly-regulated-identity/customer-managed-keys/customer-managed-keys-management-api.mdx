title: Configurar Customer Managed Keys con la Management API
---

Auth0 protege los secrets y los datos de tu tenant con una clave raíz del entorno, ubicada en la parte superior de la jerarquía de claves de cifrado de sobre.  La clave raíz del entorno y la clave raíz proporcionada por el cliente se almacenan en el Hardware Security Module (HSM) del correspondiente proveedor de servicios de Auth0 Cloud,  AWS o Azure.

<div id="bring-your-own-key">
  ## Bring Your Own Key
</div>

Con Bring Your Own Key, los usuarios con el [Key Management Editor role](/docs/es/get-started/manage-dashboard-access/add-dashboard-users) pueden usar la [Auth0 Management API](https://auth0.com/docs/api/management/v2) para sustituir la clave raíz del entorno predeterminada por su propia clave raíz proporcionada por el cliente. Los clientes pueden cargar de forma segura su propia clave raíz, que contiene su propio material criptográfico, para:

* Cumplir requisitos personalizados de generación y procedencia de claves para la clave raíz del entorno
* Cumplir requisitos específicos de instalación o ciclo de vida de las claves para la clave raíz del entorno
* Mantener una copia de la clave raíz proporcionada por el cliente en sus propias instalaciones.

<Warning>
  Al importar su propia clave raíz proporcionada por el cliente con Bring Your Own Key, autoriza implícitamente que Auth0 deje de gestionar el ciclo de vida de la clave raíz proporcionada por el cliente, excepto su eliminación.
</Warning>

Puede crear y gestionar su clave raíz proporcionada por el cliente con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, siguiendo estos pasos:

1. Llame al endpoint [Create the new encryption key](https://auth0.com/docs/api/management/v2/keys/post-encryption) para iniciar el proceso de Bring Your Own Key y obtener el `kid` (identificador de clave) de la clave raíz proporcionada por el cliente preactivada sin material criptográfico.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Este endpoint devuelve el valor `kid`, que es el identificador único de la clave.
   </Callout>
2. Llame al endpoint [Create the public wrapping key](https://auth0.com/docs/api/management/v2/keys/post-encryption-wrapping-key) para crear y devolver una clave pública de envoltura.
3. En su entorno, genere su propio material de clave criptográfica y envuélvalo (cifrándolo) con la clave pública de envoltura para crear una Wrapped Encryption Key (la clave raíz proporcionada por el cliente).
4. Llame al endpoint [Import the encryption key](https://auth0.com/docs/api/management/v2/keys/post-encryption-key) para importar su clave raíz proporcionada por el cliente a Auth0.

<div id="cryptographic-material-requirements">
  ## Requisitos del material criptográfico
</div>

Usa tu sistema de gestión de claves para envolver tu propio material criptográfico con la clave pública de envoltura y crear la Wrapped Encryption Key. Usa esta configuración para los parámetros del algoritmo [CKM&#95;RSA&#95;AES&#95;KEY&#95;WRAP](https://docs.oasis-open.org/pkcs11/pkcs11-curr/v2.40/cos01/pkcs11-curr-v2.40-cos01.html#_Toc408226894) según tu proveedor de servicios de Auth0 Cloud (AWS o Azure):

<div id="auth0-on-aws-cloud">
  ### Auth0 en la nube de AWS
</div>

* Longitud de la clave pública de envoltura: 3072 bits
* Algoritmo: CKG&#95;MGF1&#95;SHA256
* Longitud de la clave AES temporal para CKM&#95;AES&#95;KEY&#95;WRAP&#95;PAD: 256 bits
* Tipo de clave raíz proporcionada por el cliente: clave simétrica AES de 256 bits

<div id="auth0-on-azure-cloud">
  ### Auth0 en la nube de Azure
</div>

* Longitud de la clave pública de envoltura: 2048 bits
* Algoritmo: CKG&#95;MGF1&#95;SHA-1
* Longitud de la clave AES temporal para CKM&#95;AES&#95;KEY&#95;WRAP&#95;PAD: 256 bits
* Tipo de clave raíz proporcionada por el cliente: clave privada RSA de 2048 bits
* Codificación de la clave privada: PKCS #8 - ASN.1 DER

<div id="permissions">
  ## Permisos
</div>

Utilice los siguientes permisos para otorgar acceso a los endpoints de claves de cifrado.

| Permisos                 | Descripciones                                                                                                                                                                                                |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `read:encryption_keys`   | Los usuarios pueden buscar y recuperar claves en el tenant de Auth0. Esto incluye consultar los endpoints `/api/v2/keys/encryption`, `/api/v2/keys/encryption/{kid}`.                                        |
| `create:encryption_keys` | Los usuarios pueden crear su propia clave raíz proporcionada por el cliente y su clave pública de envoltura mediante los endpoints `/api/v2/keys/encryption` y `/api/v2/keys/encryption/{kid}/wrapping-key`. |
| `delete:encryption_keys` | Los usuarios pueden eliminar su propia clave raíz proporcionada por el cliente mediante el endpoint `/api/v2/keys/encryption/{kid}`.                                                                         |
| `update:encryption_keys` | Los usuarios pueden importar su propia clave raíz proporcionada por el cliente mediante el endpoint `/api/v2/keys/encryption/{kid}`.                                                                         |

<div id="endpoints">
  ## Endpoints
</div>

* Usa el endpoint [Get all encryption keys](https://auth0.com/docs/api/management/v2/keys/get-encryption-keys) para obtener información sobre la clave raíz del entorno actual o la clave raíz proporcionada por el cliente existentes, así como sobre la clave maestra del tenant.
* Usa el endpoint [Get the encryption key by its key id](https://auth0.com/docs/api/management/v2/keys/get-encryption-key) para obtener información sobre la clave especificada.
* Usa el endpoint [Create the new encryption key](https://auth0.com/docs/api/management/v2/keys/post-encryption) para crear un marcador de posición preactivado para una clave raíz proporcionada por el cliente.
* Usa el endpoint [Create the public wrapping key](https://auth0.com/docs/api/management/v2/keys/post-encryption-wrapping-key) para obtener una clave pública de envoltura con la que envolver tu material criptográfico.
* Usa el endpoint [Import the encryption key](https://auth0.com/docs/api/management/v2/keys/post-encryption-key) para importar el material de una nueva clave raíz proporcionada por el cliente.
* Usa el endpoint [Delete the encryption key by its key id](https://auth0.com/docs/api/management/v2/keys/delete-encryption-key) para eliminar una clave raíz proporcionada por el cliente existente y dejar de usar Bring Your Own Keys. La clave raíz del entorno se utiliza para volver a cifrar la jerarquía de claves. Esta operación no afecta negativamente a los datos existentes ni a los flujos de autenticación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si la funcionalidad Customer Managed Keys no está habilitada en tu tenant, al llamar a los endpoints de API se devolverá el siguiente error: `This feature is not enabled for this tenant`.
</Callout>
