title: Configurar Customer Managed Keys con la Management API
---

Auth0 protege los secretos y los datos de tu tenant mediante una Auth0 Environment Root Key, que se encuentra en la parte superior de la jerarquía de claves de cifrado de sobre.  La Auth0 Environment Root Key y la clave raíz proporcionada por el cliente se almacenan en el Hardware Security Module (HSM) del Auth0 Cloud Service Provider correspondiente,  AWS o Azure.

<div id="bring-your-own-key">
  ## Bring Your Own Key
</div>

Con Bring Your Own Key, los usuarios con el [Key Management Editor role](/docs/es/get-started/manage-dashboard-access/add-dashboard-users) pueden usar la [Auth0 Management API](https://auth0.com/docs/api/management/v2) para reemplazar la Auth0 Environment Root Key predeterminada por su propia clave raíz proporcionada por el cliente. Los clientes pueden cargar de forma segura su propia clave raíz, que contiene su propio material criptográfico, para:

* Cumplir requisitos personalizados de generación y procedencia de claves para la clave raíz del entorno
* Cumplir requisitos específicos de instalación o de vida útil de las claves para la clave raíz del entorno
* Mantener una copia de la clave raíz proporcionada por el cliente en sus propias instalaciones.

<Warning>
  Al importar su propia clave raíz proporcionada por el cliente con Bring Your Own Key, autoriza implícitamente a Auth0 a dejar de gestionar el ciclo de vida de la clave raíz proporcionada por el cliente, salvo su eliminación.
</Warning>

Puede crear y gestionar su clave raíz proporcionada por el cliente mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, siguiendo estos pasos:

1. Llame al endpoint [Create the new encryption key](https://auth0.com/docs/api/management/v2/keys/post-encryption) para iniciar el proceso de Bring Your Own Key y obtener el `kid` (identificador de clave) de la clave raíz proporcionada por el cliente, preactivada pero sin material criptográfico.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Este endpoint devuelve el valor `kid`, que es el identificador único de la clave.
   </Callout>
2. Llame al endpoint [Create the public wrapping key](https://auth0.com/docs/api/management/v2/keys/post-encryption-wrapping-key) para crear y devolver una clave pública de envoltura.
3. En su entorno, genere su propio material de clave criptográfica y envuélvalo (cifrándolo) con la clave pública de envoltura para crear una clave de cifrado envuelta (la clave raíz proporcionada por el cliente).
4. Llame al endpoint [Import the encryption key](https://auth0.com/docs/api/management/v2/keys/post-encryption-key) para importar su clave raíz proporcionada por el cliente a Auth0.

<div id="cryptographic-material-requirements">
  ## Requisitos del material criptográfico
</div>

Usa tu sistema de gestión de claves para envolver tu propio material criptográfico con la clave pública de envoltura y crear la clave de cifrado envuelta. Usa esta configuración para los parámetros del algoritmo [CKM&#95;RSA&#95;AES&#95;KEY&#95;WRAP](https://docs.oasis-open.org/pkcs11/pkcs11-curr/v2.40/cos01/pkcs11-curr-v2.40-cos01.html#_Toc408226894) según tu Auth0 Cloud Service Provider (AWS o Azure):

<div id="auth0-on-aws-cloud">
  ### Auth0 en AWS
</div>

* Longitud de la clave pública de envoltura: 3072 bits
* Algoritmo: CKG&#95;MGF1&#95;SHA256
* Longitud de la clave AES temporal para CKM&#95;AES&#95;KEY&#95;WRAP&#95;PAD: 256 bits
* Tipo de clave raíz proporcionada por el cliente: clave simétrica AES de 256 bits

<div id="auth0-on-azure-cloud">
  ### Auth0 en la nube de Azure
</div>

* Longitud de la clave pública de envoltura: 2048 bits
* Algoritmo: CKG&#95;MGF1&#95;SHA-1
* Longitud de la clave AES temporal para CKM&#95;AES&#95;KEY&#95;WRAP&#95;PAD: 256 bits
* Tipo de clave raíz proporcionada por el cliente: clave privada RSA de 2048 bits
* Codificación de la clave privada: PKCS #8 - ASN.1 DER

<div id="permissions">
  ## Permisos
</div>

Usa los siguientes permisos para proporcionar acceso a los endpoints de claves de cifrado.

| Permisos                 | Descripciones                                                                                                                                                                                                |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `read:encryption_keys`   | Los usuarios pueden buscar y recuperar claves en el tenant de Auth0. Esto incluye consultar los endpoints `/api/v2/keys/encryption` y `/api/v2/keys/encryption/{kid}`.                                       |
| `create:encryption_keys` | Los usuarios pueden crear su propia clave raíz proporcionada por el cliente y su clave pública de envoltura mediante los endpoints `/api/v2/keys/encryption` y `/api/v2/keys/encryption/{kid}/wrapping-key`. |
| `delete:encryption_keys` | Los usuarios pueden eliminar su propia clave raíz proporcionada por el cliente mediante el endpoint `/api/v2/keys/encryption/{kid}`.                                                                         |
| `update:encryption_keys` | Los usuarios pueden importar su propia clave raíz proporcionada por el cliente mediante el endpoint `/api/v2/keys/encryption/{kid}`.                                                                         |

<div id="endpoints">
  ## Endpoints
</div>

* Usa el endpoint [Obtener todas las claves de cifrado](https://auth0.com/docs/api/management/v2/keys/get-encryption-keys) para obtener información sobre la actual clave raíz del entorno o la clave raíz proporcionada por el cliente, así como sobre la clave maestra del tenant.
* Usa el endpoint [Obtener la clave de cifrado por su ID de clave](https://auth0.com/docs/api/management/v2/keys/get-encryption-key) para obtener información sobre la clave especificada.
* Usa el endpoint [Crear la nueva clave de cifrado](https://auth0.com/docs/api/management/v2/keys/post-encryption) para crear un marcador de posición preactivado para una clave raíz proporcionada por el cliente.
* Usa el endpoint [Crear la clave pública de envoltura](https://auth0.com/docs/api/management/v2/keys/post-encryption-wrapping-key) para obtener una clave pública de envoltura con la que envolver tu material criptográfico.
* Usa el endpoint [Importar la clave de cifrado](https://auth0.com/docs/api/management/v2/keys/post-encryption-key) para importar nuevo material de clave raíz proporcionada por el cliente.
* Usa el endpoint [Eliminar la clave de cifrado por su ID de clave](https://auth0.com/docs/api/management/v2/keys/delete-encryption-key) para eliminar una clave raíz proporcionada por el cliente existente y dejar de usar Bring Your Own Keys. La clave raíz del entorno de Auth0 se utiliza para volver a cifrar la jerarquía de claves. Esta operación no afecta negativamente a los datos existentes ni a los flujos de autenticación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si la funcionalidad Customer Managed Keys no está habilitada en tu tenant, las llamadas a los endpoints de la API devolverán el error: `This feature is not enabled for this tenant`.
</Callout>
