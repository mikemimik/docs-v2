description: Describe cómo configurar notificaciones de OTP basadas en tiempo para MFA.
title: Configurar notificaciones de OTP para MFA
---

Auth0 admite contraseñas de un solo uso (OTP) como factores de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>. Para que los usuarios puedan usar OTP, primero debes [habilitarlas como factor de MFA](/docs/es/secure/multi-factor-authentication/enable-mfa) en tu tenant de Auth0.

Una vez habilitadas para tu aplicación, los usuarios pueden registrar OTP al acceder a tu aplicación. Para hacerlo, deben instalar una aplicación de autenticación en su dispositivo, como:

* Auth0 Guardian ([Google Play](https://play.google.com/store/apps/details?id=com.auth0.guardian) / [App Store](https://itunes.apple.com/us/app/auth0-guardian/id1093447833))
* Authy ([Google Play](https://play.google.com/store/apps/details?id=com.authy.authy) / [App Store](https://itunes.apple.com/us/app/authy/id494168017))
* Google Authenticator ([Google Play](https://play.google.com/store/apps/details?id=com.google.android.apps.authenticator2) / [App Store](https://itunes.apple.com/us/app/google-authenticator/id388497605))
* Microsoft Authenticator ([Google Play](https://play.google.com/store/apps/details?id=com.azure.authenticator) / [App Store](https://itunes.apple.com/us/app/microsoft-authenticator/id983156458))

<div id="user-otp-workflow">
  ## Flujo de OTP del usuario
</div>

Cuando un usuario intenta registrarse a través de un <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>, se le pide que escanee un código QR para registrarse en OTP.

<Frame>![Ejemplo de configuración del código QR de OTP de Auth0 Guardian](/docs/images/cdy7uua7fh8z/2J6vWaonPbCvpuBUu9b8q1/b4ec4e66fc86021e5e31d000acaf18e2/QR_Code_-_EN_-_Small.png)</Frame>

Si se detecta que el dispositivo del usuario es móvil, Universal Login omite la pantalla del código QR y, en su lugar, muestra directamente un código de registro en el prompt.

<Frame>![La pantalla temporal de registro en OTP, tal como se muestra en un dispositivo móvil. ](/docs/images/cdy7uua7fh8z/5QvQ9FLYnVZXINFAeLkIr9/04df9be004169ac06b8d9fee456dc599/OTP_prompt_-_English.png)</Frame>

Una vez que el usuario se haya registrado en OTP, puede usar OTP para iniciar sesión en tu aplicación. Para hacerlo, primero obtiene un código de un solo uso desde su aplicación de autenticación preferida.

<Frame>![Ejemplo de código OTP de Auth0 Guardian en Google](/docs/images/cdy7uua7fh8z/1vVY2NLQwsOwdHIPU4Qj3B/026255e1549c3ec5bdc790280590d210/google-auth-screenshot.png)</Frame>

Luego, introduce el código de un solo uso en el prompt de inicio de sesión.

<Frame>![Ejemplo del prompt de inicio de sesión de OTP de Auth0 Guardian](/docs/images/cdy7uua7fh8z/4dntQCYqgx5QyLZKW7quxR/20cab4bcde8270914d2c2979810cd5f5/OTP_challenge_-_EN_-_Small.png)</Frame>
