---
description: Describe cómo configurar el correo electrónico como un factor de MFA para los usuarios que no tienen disponible su factor principal.
title: Configurar notificaciones por correo electrónico para MFA
---

Usa el correo electrónico como un factor de <Tooltip tip="autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) para ofrecer a los usuarios una forma de realizar MFA cuando no tienen disponible su factor principal (por ejemplo, no tienen su dispositivo para recibir un SMS o una notificación push).

<Card title="La disponibilidad varía según la implementación de inicio de sesión">
  La implementación de inicio de sesión que uses determina si esta funcionalidad está disponible. Para obtener más información, consulta [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).
</Card>

El correo electrónico no es un verdadero MFA, porque no representa un factor distinto de la contraseña. No representa **algo que tengo** ni **algo que soy**, sino simplemente **algo que sé** (la contraseña del correo electrónico). Además, es más débil que otros factores, ya que su seguridad depende de la del propio correo electrónico (por ejemplo, si está cifrado de extremo a extremo).

Los usuarios no necesitan registrarse explícitamente en MFA por correo electrónico. Pueden usarlo cuando tienen una dirección de correo electrónico verificada. Esto ocurre cuando:

* Completan el flujo de verificación de correo electrónico, que actualiza el campo `email_verified` mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.
* Inician sesión con una conexión que proporciona direcciones de correo electrónico verificadas (como Google).

Solo puedes habilitar el correo electrónico como un factor de MFA si ya hay otro factor habilitado.

Una vez habilitado Email MFA, se pedirá a los usuarios que completen la MFA con otro factor habilitado. Si seleccionan **Try another method** y luego eligen **Email**, recibirán un correo electrónico con un código de 6 dígitos que deberán introducir para completar el flujo de autenticación.

<div id="configure-email-notifications">
  ## Configurar notificaciones por correo electrónico
</div>

Puede registrar explícitamente el correo electrónico para MFA mediante la [API de MFA](/docs/es/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa/enroll-and-challenge-email-authenticators). Si los usuarios tienen un correo electrónico verificado y una o más direcciones de correo registradas explícitamente, pueden elegir qué correo electrónico quieren usar para completar MFA al iniciar sesión con <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>.

1. Vaya a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/multifactor-auth) y active el selector **Email**.
   Solo podrá activarlo si hay otro factor habilitado.
2. Auth0 proporciona un proveedor de correo electrónico de prueba, pero solo permite enviar una cantidad limitada de correos electrónicos, por lo que debería configurar su propio proveedor de correo electrónico. Para obtener más información, consulte [Configure External SMTP Email Providers](/docs/es/customize/email/smtp-email-providers).
