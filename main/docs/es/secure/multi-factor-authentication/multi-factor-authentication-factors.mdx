Auth0 admite varias opciones o factores para proteger el acceso a las cuentas de usuario con <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código enviado por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA).

Para seleccionar los factores de MFA que quiere usar en su tenant, vaya a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Debe habilitar al menos un factor en su tenant para usar MFA. Los factores disponibles dependen de su plan de suscripción. Para obtener más información, consulte [Auth0 Pricing](https://auth0.com/pricing/).

<div id="factors">
  ## Factores
</div>

Auth0 admite los siguientes factores para implementar MFA.

<div id="push-notifications">
  ### Notificaciones push
</div>

Envía a los usuarios notificaciones push a sus dispositivos registrados previamente, normalmente un teléfono móvil o una tableta, desde los que pueden permitir o denegar de inmediato el acceso a la cuenta con solo pulsar un botón. El factor push se ofrece con la aplicación móvil **Auth0 Guardian**, disponible tanto para iOS como para Android. Para obtener más información, consulta [Auth0 Guardian](/docs/es/secure/multi-factor-authentication/auth0-guardian).

Si no quieres que tus clientes tengan que descargar una aplicación independiente, Auth0 también proporciona un **Guardian SDK** para crear un flujo de segundo factor en tu aplicación móvil actual. Para obtener más información, consulta [Install Guardian SDK](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/install-guardian-sdk).

<div id="sms-notifications">
  ### Notificaciones por SMS
</div>

Envía a los usuarios un código de un solo uso por SMS, que luego se les pedirá que introduzcan antes de poder completar la autenticación. Para obtener más información, consulta [Configurar notificaciones de SMS y voz para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)

<div id="voice-notifications">
  ### Notificaciones de voz
</div>

Envía un código de un solo uso a los usuarios mediante una llamada de voz para que luego lo introduzcan antes de completar la autenticación. Para obtener más información, consulta [Configurar notificaciones SMS y de voz para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)

<div id="one-time-passwords">
  ### Contraseñas de un solo uso
</div>

Las contraseñas de un solo uso te permiten usar una aplicación autenticadora en el dispositivo personal del usuario, como Google Authenticator, para generar una contraseña de un solo uso que cambia con el tiempo y que puede introducirse como segundo factor para validar la cuenta. Para obtener más información, consulta [Configurar notificaciones de OTP para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)

<div id="webauthn-with-security-keys">
  ### WebAuthn con claves de seguridad
</div>

Permite a los usuarios usar MFA con claves de seguridad compatibles con FIDO (p. ej., [Yubikey](https://www.yubico.com/), [Google Titan](https://cloud.google.com/titan-security-key)). Para obtener más información, consulta [Configurar WebAuthn con claves de seguridad para MFA.](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)

<div id="webauthn-with-device-biometrics">
  ### WebAuthn con biometría del dispositivo
</div>

Permite a los usuarios realizar MFA con autenticadores de plataforma, como la TouchBar del MacBook, Windows Hello, Touch/FaceId de iOS y el reconocimiento facial o de huella en Android. Para obtener más información, consulta [Configurar WebAuthn con biometría del dispositivo para MFA.](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)

<div id="email-notifications">
  ### Notificaciones por correo electrónico
</div>

Permita que los usuarios usen MFA con contraseñas de un solo uso enviadas por correo electrónico si ya tienen inscritos [factores de autenticación independientes](/docs/es/secure/multi-factor-authentication/enable-mfa#independent-factors) adicionales. Para obtener más información, consulte [Configurar notificaciones por correo electrónico para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)

<div id="cisco-duo-security">
  ### Cisco Duo Security
</div>

Cisco Duo es un Provider polifacético y solo puede usarse si es el único factor disponible para el usuario. Usa tu cuenta de Duo para gestionar la MFA con Auth0. Para obtener más información, consulta [Configurar Cisco Duo Security para MFA.](/docs/es/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="recovery-codes">
  ### Códigos de recuperación
</div>

Los códigos de recuperación son códigos únicos que permiten a los usuarios recuperar el acceso a su cuenta si no pueden acceder al dispositivo o la cuenta que usaron para registrar MFA. Para obtener más información, consulta [Configurar códigos de recuperación para MFA.](/docs/es/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="policies">
  ## Políticas
</div>

Las políticas determinan cuándo se le pedirá a un usuario que complete pasos adicionales para demostrar que es el propietario de una cuenta determinada. Use las políticas para definir su propio nivel de riesgo aceptable. Puede elegir entre **Nunca**, **Usar <Tooltip tip="MFA adaptativo: autenticación multifactor (MFA) que solo se activa para los usuarios cuando se determina que un intento de inicio de sesión es de baja confianza." cta="Ver glosario" href="/docs/es/glossary?term=Adaptive+MFA">MFA adaptativo</Tooltip>** y **Siempre**.

Puede lograr configuraciones de autenticación multifactor más precisas (como por aplicación, por usuario, etc.) con Auth0 Actions. Para obtener más información, lea [Auth0 Actions](/docs/es/customize/actions).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las Rules que afectan al MFA tienen prioridad sobre la configuración de políticas en el Dashboard.
</Callout>

Para obtener más información, lea [Políticas de autenticación definidas en OpenID.](http://openid.net/specs/openid-provider-authentication-policy-extension-1_0.html#rfc.section.4)

<div id="use-cases">
  ## Casos de uso
</div>

Hay diferentes maneras de gestionar la MFA según tu entorno:

* B2B: Tus clientes gestionan los factores de MFA de sus usuarios.
* B2C: Los usuarios finales gestionan sus propios factores de MFA a través de una página de **My MFA Settings**.
* B2E: Tú gestionas los factores de MFA de tus usuarios.

Para conocer los endpoints de la API que puedes usar para crear una interfaz de usuario para gestionar factores de MFA, consulta [Administrar factores del autenticador mediante la API de MFA.](/docs/es/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authenticator-factors-mfa-api)

Las aplicaciones que permiten acceder a distintos tipos de recursos pueden exigir que los usuarios se autentiquen con un mecanismo más sólido para acceder a recursos sensibles. Consulta [Autenticación escalonada](/docs/es/secure/multi-factor-authentication/step-up-authentication) para más detalles.

Para crear una Action, ve a [Dashboard &gt; Actions &gt; Flows](https://manage.auth0.com/#/actions/flows) y define las condiciones que activarán desafíos de autenticación. Usa Actions para forzar la MFA a usuarios de ciertas aplicaciones o a usuarios con metadatos de usuario o rangos de IP específicos, entre otros triggers.

Añade MFA contextual, que te permite definir condiciones arbitrarias que activarán desafíos de autenticación adicionales para tus clientes con el fin de aumentar la seguridad; por ejemplo, la ubicación geográfica (geofencing), la dirección o el tipo de red utilizada (filtrado de IP), la hora del día, el día de la semana o un cambio en la ubicación o el dispositivo que se usa para iniciar sesión.
