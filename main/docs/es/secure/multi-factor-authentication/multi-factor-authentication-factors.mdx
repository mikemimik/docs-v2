Auth0 admite varias opciones o factores para proteger el acceso a la cuenta de usuario con <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor adicional además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA).

Para seleccionar los factores de MFA que usarás en tu tenant, ve a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Debes habilitar al menos un factor en tu tenant para usar MFA. Los factores disponibles dependen de tu plan de suscripción. Para obtener más información, consulta [Auth0 Pricing](https://auth0.com/pricing/).

<div id="factors">
  ## Factores
</div>

Auth0 admite los siguientes factores para implementar MFA.

<div id="push-notifications">
  ### Notificaciones push
</div>

Envíe a los usuarios notificaciones push a sus dispositivos registrados previamente, normalmente un teléfono móvil o una tableta, desde los que pueden aprobar o denegar de inmediato el acceso a la cuenta con solo pulsar un botón. El factor push está disponible con la aplicación móvil **Auth0 Guardian**, disponible tanto para iOS como para Android. Para obtener más información, lea [Auth0 Guardian](/docs/es/secure/multi-factor-authentication/auth0-guardian).

Si no quiere que sus clientes tengan que descargar una aplicación aparte, Auth0 también proporciona un **Guardian SDK** para crear un flujo de trabajo de segundo factor en su aplicación móvil existente. Para obtener más información, lea [Install Guardian SDK](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/install-guardian-sdk).

<div id="sms-notifications">
  ### Notificaciones SMS
</div>

Envía a los usuarios un código de un solo uso por SMS que deberán introducir antes de completar la autenticación. Para obtener más información, consulta [Configurar las notificaciones de SMS y voz para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)

<div id="voice-notifications">
  ### Notificaciones de voz
</div>

Envía un código de un solo uso a los usuarios mediante una llamada de voz, que luego deberán introducir antes de poder completar la autenticación. Para obtener más información, consulta [Configurar notificaciones de SMS y voz para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)

<div id="one-time-passwords">
  ### Contraseñas de un solo uso
</div>

Las contraseñas de un solo uso permiten usar una aplicación autenticadora en el dispositivo personal del usuario, como Google Authenticator, para generar una contraseña de un solo uso que cambia con el tiempo y que puede introducirse como segundo factor para verificar una cuenta. Para obtener más información, consulta [Configurar notificaciones de OTP para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)

<div id="webauthn-with-security-keys">
  ### WebAuthn con claves de seguridad
</div>

Permita que los usuarios usen MFA con claves de seguridad compatibles con FIDO (p. ej., [Yubikey](https://www.yubico.com/), [Google Titan](https://cloud.google.com/titan-security-key)). Para obtener más información, lea [Configurar WebAuthn con claves de seguridad para MFA.](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)

<div id="webauthn-with-device-biometrics">
  ### WebAuthn con biometría del dispositivo
</div>

Permita a los usuarios realizar MFA con autenticadores de plataforma, como Touch Bar de MacBook, Windows Hello, Touch ID/Face ID de iOS y el reconocimiento facial o de huellas dactilares de Android. Para obtener más información, lea [Configurar WebAuthn con biometría del dispositivo para MFA.](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)

<div id="email-notifications">
  ### Notificaciones por email
</div>

Permita a los usuarios realizar MFA con contraseñas de un solo uso enviadas por email si tienen configurados [factores de autenticación independientes adicionales](/docs/es/secure/multi-factor-authentication/enable-mfa#independent-factors). Para obtener más información, consulte [Configurar las notificaciones por email para MFA.](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)

<div id="cisco-duo-security">
  ### Cisco Duo Security
</div>

Cisco Duo es un proveedor multifunción y solo se puede usar si es el único factor disponible para el usuario. Usa tu cuenta de Duo para gestionar MFA con Auth0. Para obtener más información, consulta [Configurar Cisco Duo Security para MFA.](/docs/es/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="recovery-codes">
  ### Códigos de recuperación
</div>

Los códigos de recuperación son códigos únicos que permiten a los usuarios recuperar el acceso a su cuenta si pierden acceso al dispositivo o a la cuenta que usaron para el registro en MFA. Para obtener más información, lee [Configurar códigos de recuperación para MFA.](/docs/es/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="policies">
  ## Políticas
</div>

Las políticas determinan cuándo se le pedirá a un usuario que complete pasos adicionales para demostrar que es el propietario de una cuenta determinada. Usa las políticas para definir tu propio nivel de riesgo aceptable. Puedes elegir entre **Nunca**, **Usar <Tooltip tip="MFA adaptativo: autenticación multifactor (MFA) que solo se activa para los usuarios cuando se determina que un intento de inicio de sesión es de baja confianza." cta="Ver glosario" href="/docs/es/glossary?term=Adaptive+MFA">MFA adaptativo</Tooltip>** y **Siempre**.

Puedes lograr configuraciones de autenticación multifactor más precisas (como por aplicación, por usuario, etc.) mediante Auth0 Actions. Para obtener más información, consulta [Auth0 Actions](/docs/es/customize/actions).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las Rules que afectan al MFA tienen prioridad sobre la configuración de políticas en el Dashboard.
</Callout>

Para obtener más información, consulta [Defined Authentication Policies en OpenID.](http://openid.net/specs/openid-provider-authentication-policy-extension-1_0.html#rfc.section.4)

<div id="use-cases">
  ## Casos de uso
</div>

Hay distintas formas de gestionar MFA según el entorno:

* B2B: Sus clientes gestionan los factores de MFA de sus usuarios.
* B2C: Los usuarios finales gestionan sus propios factores de MFA a través de una página de **My MFA Settings**.
* B2E: Usted gestiona los factores de MFA de sus usuarios.

Para conocer los endpoints de la API que puede usar para crear una interfaz de usuario para gestionar factores de MFA, consulte [Manage Authenticator Factors Using the MFA API.](/docs/es/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authenticator-factors-mfa-api)

Las aplicaciones que dan acceso a distintos tipos de recursos pueden requerir que los usuarios se autentiquen con un mecanismo de autenticación más sólido para acceder a recursos sensibles. Consulte [Autenticación escalonada](/docs/es/secure/multi-factor-authentication/step-up-authentication) para obtener más información.

Para crear una Action, vaya a [Dashboard &gt; Actions &gt; Flows](https://manage.auth0.com/#/actions/flows) y defina condiciones para activar desafíos de autenticación. Use Actions para forzar MFA para usuarios de determinadas aplicaciones o para usuarios con metadatos de usuario o rangos de IP específicos, entre otros triggers.

Añada MFA contextual, que le permite definir condiciones arbitrarias que activarán desafíos de autenticación adicionales para sus clientes con el fin de aumentar la seguridad; por ejemplo, la ubicación geográfica (geofencing), la dirección o el tipo de red utilizados (filtrado de IP), la hora del día, el día de la semana o un cambio en la ubicación o el device que se usa para iniciar sesión.
