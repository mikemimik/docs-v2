Auth0 proporciona un flujo integrado de inscripción y autenticación de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> con [Universal Login](/docs/es/authenticate/login/auth0-universal-login). Sin embargo, si quieres crear tu propia interfaz de usuario, puedes usar la [API de MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api) para hacerlo.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de poder usar las APIs de MFA, debes habilitar el tipo de concesión MFA para tu aplicación. Ve a [Auth0 Dashboard &gt; Applications &gt; Advanced Settings &gt; Grant Types](https://manage.auth0.com/#/applications) y selecciona **MFA**.

* [Configurar el teléfono como factor](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) en el Dashboard o mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Guardian/put_factors_by_name).

<div id="enroll-with-sms-or-voice">
  ## Inscribirse con SMS o voz
</div>

<div id="get-mfa-token">
  ### Obtener un token de MFA
</div>

Según cuándo inicies la inscripción, puedes obtener un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> para usar la API de MFA de distintas maneras:

* Si realizas la inscripción durante la autenticación, consulta [Autenticar con la concesión de contraseña del propietario del recurso y MFA](/docs/es/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa).
* Si quieres permitir que el usuario inscriba un factor en cualquier momento, consulta [Administrar la inscripción de factores de MFA](/docs/es/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authenticator-factors-mfa-api).

<div id="enroll-authenticator">
  ### Inscribir autenticador
</div>

Realiza una solicitud `POST` al endpoint MFA Associate para inscribir el autenticador del usuario. El bearer token que requiere este endpoint es el MFA token obtenido en el paso anterior.

Para inscribirse con SMS o voz, el usuario se inscribe con un número de teléfono que puede usarse para un challenge por SMS o voz. Especifica los siguientes parámetros para llamar al endpoint. El parámetro `oob_channels` indica cómo quieres enviar el código al usuario (SMS o voz).

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/mfa/associate' \
    --header 'authorization: Bearer {mfaToken}' \
    --header 'content-type: application/json' \
    --data '{ "authenticator_types": ["oob"], "oob_channels": ["sms"], "phone_number": "+11...9" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/mfa/associate");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer {mfaToken}");
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{ "authenticator_types": ["oob"], "oob_channels": ["sms"], "phone_number": "+11...9" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  	payload := strings.NewReader("{ "authenticator_types": ["oob"], "oob_channels": ["sms"], "phone_number": "+11...9" }")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("authorization", "Bearer {mfaToken}")
  	req.Header.Add("content-type", "application/json")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

</AuthCodeGroup>

| Parámetro              | Valor                                                                                                   |
| ---------------------- | ------------------------------------------------------------------------------------------------------- |
| `authentication_types` | `[oob]`                                                                                                 |
| `oob_channels`         | `[sms]` o `[voice]`                                                                                     |
| `phone_number`         | `+11...9`, el número de teléfono en [formato compatible con E.164](https://en.wikipedia.org/wiki/e.164) |

Si todo sale bien, recibirás una respuesta como esta:

```json lines
{
  "authenticator_type": "oob",
  "binding_method": "prompt",
  "recovery_codes": [ "N3BGPZZWJ85JLCNPZBDW6QXC" ],
  "oob_channel": "sms",
  "oob_code": "ata6daXAiOi..."
}
`
```

Si obtienes un error `User is already enrolled`, el usuario ya tiene registrado un factor de MFA. Antes de asociar otro factor con el usuario, debes solicitarle un desafío con el factor existente.

Si esta es la primera vez que el usuario asocia un autenticador, notarás que la respuesta incluye `recovery_codes`. Los códigos de recuperación se usan para acceder a la cuenta del usuario en caso de que pierda acceso a la cuenta o al dispositivo utilizado para la autenticación de segundo factor. Estos códigos solo se pueden usar una vez, y se generan otros nuevos cuando es necesario.

<div id="confirm-sms-or-voice-enrollment">
  ### Confirmar el registro por SMS o voz
</div>

Los usuarios deberían recibir un mensaje con un código de 6 dígitos que deben introducir en la aplicación.

Para completar el registro, haz una solicitud `POST` al <Tooltip tip="Endpoint de token: endpoint del servidor de autorización que se usa para solicitar tokens mediante programación." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip> <Tooltip tip="OAuth 2.0: marco de autorización que define protocolos y flujos de autorización." cta="Ver glosario" href="/docs/es/glossary?term=Token+endpoint">endpoint de token</Tooltip>. Debes incluir el `oob_code` devuelto en la respuesta anterior y el `binding_code` con el valor recibido en el mensaje.

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/oauth/token' \
    --header 'authorization: Bearer {mfaToken}' \
    --header 'content-type: application/x-www-form-urlencoded' \
    --data grant_type=http://auth0.com/oauth/grant-type/mfa-oob \
    --data 'client_id={yourClientId}' \
    --data 'client_secret={yourClientSecret}' \
    --data 'mfa_token={mfaToken}' \
    --data 'oob_code={oobCode}' \
    --data 'binding_code={userOtpCode}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/oauth/token");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer {mfaToken}");
  request.AddHeader("content-type", "application/x-www-form-urlencoded");
  request.AddParameter("application/x-www-form-urlencoded", "grant_type=http%3A%2F%2Fauth0.com%2Foauth%2Fgrant-type%2Fmfa-oob&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D&mfa_token=%7BmfaToken%7D&oob_code=%7BoobCode%7D&binding_code=%7BuserOtpCode%7D", ParameterType.RequestBody);
  ```


<div id="challenge-user-with-otp">
  ### Solicitar al usuario un OTP
</div>

  ```bash cURL

    --url 'https://{yourDomain}/mfa/challenge' \
    --header 'content-type: application/json' \
    --data '{ "client_id": "{yourClientId}",  "client_secret": "{yourClientSecret}", "challenge_type": "oob", "authenticator_id": "sms|dev_NU1Ofuw3Cw0XCt5x", "mfa_token": "{mfaToken}" }'
  ```


<div id="customize-mfa">
  ## Personalizar MFA
</div>

<Warning>
  La MFA personalizable con los flujos de Resource Owner Password Grant, Embedded o token de actualización está en acceso anticipado. Al usar esta funcionalidad, aceptas los términos aplicables de la prueba gratuita del [Master Subscription Agreement](https://www.okta.com/legal/) de Okta. Para obtener más información sobre las etapas de lanzamiento de Auth0, consulta [Product Release Stages](/docs/es/troubleshoot/product-lifecycle/product-release-stages). Para participar en el acceso anticipado, ponte en contacto con el [equipo de soporte de Auth0](https://support.auth0.com/).
</Warning>

La MFA personalizable permite a los usuarios inscribirse y desafiar con los factores compatibles con su aplicación que elijan.

Durante la autenticación en el endpoint `oauth/token`, la respuesta devuelve el error `mfa_required`, que incluye el `mfa_token` para usar la API de MFA y el parámetro `mfa_requirements` con una lista de autenticadores:

```json
{
  "error": "mfa_required",
  "error_description": "Multifactor authentication required",
  "mfa_token": "Fe26...Ha",
  "mfa_requirements": {
    "challenge": [
      { "type": "otp" },
      { "type": "push-notification" },
      { "type": "phone" },
      { "type": "recovery-code" }
      { "type": "email"} //solo puede funcionar con challenge
    ]
  }
}
```

Usa el `mfa_token` para llamar al endpoint [`mfa/authenticator`](/docs/es/api/authentication/muti-factor-authentication/list-authenticators), obtener una lista de todos los factores que el usuario tiene inscritos y hacerla coincidir con el mismo tipo que admite tu aplicación. También debes obtener el `authenticator_type` correspondiente para emitir desafíos:

```json
[
  {
    "type": "recovery-code",
    "id": "recovery-code|dev_qpOkGUOxBpw6R16t",
    "authenticator_type": "recovery-code",
    "active": true
  },
  {
    "type": "otp",
    "id": "totp|dev_6NWz8awwC8brh2dN",
    "authenticator_type": "otp",
    "active": true
  }
]
```

Exige el desafío MFA llamando al endpoint [`request/mfa/challenge`](/docs/es/api/authentication/muti-factor-authentication/request-mfa-challenge).

Personaliza aún más tu flujo MFA con Auth0 Actions. Para obtener más información, consulta [Actions Triggers: post-challenge - objeto de API](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object).
