title: Importar inscripciones de autenticadores de MFA de usuarios
---

Puedes importar las inscripciones de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> de un usuario con [migración automática](/docs/es/manage-users/user-migration/configure-automatic-migration-from-your-database) e [importaciones masivas de usuarios](/docs/es/manage-users/user-migration/bulk-user-imports). Los tipos de inscripción compatibles son:

* Email: para verificación por correo electrónico.
* Phone: para verificación por SMS.
* OTP: para contraseñas de un solo uso (OTP) utilizadas con aplicaciones de autenticación, como Google Authenticator.

Importar inscripciones de MFA ofrece una experiencia de usuario fluida, ya que los usuarios no tendrán que volver a inscribirse después de la migración.

La experiencia de Classic Login no admite la selección de factores para usuarios con varios factores. Si planeas importar usuarios con varios factores registrados, considera usar la experiencia de [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

<div id="schema">
  ## Esquema
</div>

El esquema se aplica a los factores de MFA en los dos flujos de trabajo siguientes.

```json lines expandable
{
    "type": "array",
    "items": {
        "type": "object",
        "properties": {
            "totp": {
                "type": "object",
                "properties": {
                "secret": {
                    "type": "string",
                        "pattern": "^[A-Z2-7]+$",
                        "description": "The OTP secret is used for MFA authentication with Google Authenticator type apps. It must be supplied in un-padded Base32 encoding, such as: JBTWY3DPEHPK3PNP"
                    },
                },
                "additionalProperties": false,
                "required": ["secret"],
            },
            "phone": {
                "type": "object",
                "properties": {
                "value": {
                    "type": "string",
                    "pattern": "^\\+[0-9]{1,15}$",
                    "description": "The phone number for SMS or Voice MFA. The phone number should include a country code and begin with +, such as: +12125550001"
                },
                },
                "additionalProperties": false,
                "required": ["value"],
            },
            "email": {
                "type": "object",
                "properties": {
                    "value": {
                        "type": "string",
                        "format": "email",
                        "description": "The email address for MFA"
                    },
                },
                "additionalProperties": false,
                "required": ["value"],
            },
        },
        "maxProperties": 1,
        "additionalProperties": false,
    },
    "minItems": 1,
    "maxItems": 10
}
```

<div id="automatic-migration">
  ## Migración automática
</div>

Las inscripciones de MFA también se pueden importar durante una [migración automática](/docs/es/authenticate/database-connections/custom-db/overview-custom-db-connections#automatic-migration-scenario). Para ello, proporciona cualquier inscripción existente en el campo `mfa_factors` del usuario que se pasa a la devolución de llamada al final de tu [script de inicio de sesión](/docs/es/authenticate/database-connections/custom-db/templates/login) de base de datos personalizada.

Cualquier error aparecerá en los logs de tu tenant como inicios de sesión fallidos y se podrá distinguir de otros errores por su descripción: `Unable to import MFA factors`. Por ejemplo:

```json lines
{
  "_id": "5e9df3b29ebabe00571c04a7",
  "date": "2020-04-20T19:10:42.916Z",
  "type": "fu",
  "description": "Unable to import MFA factors.",
  "connection": "Username-Password-Authentication",
  "connection_id": "con_mMkvaycgzgCS0p0z",
  "client_id": "aCbTAJNi5HbsjPJtRpSP6BIoLPOrSj2Cgg",
  "client_name": "All Applications",
  "ip": "10.12.13.1",
  "client_ip": null,
  "user_agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_6) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/80.0.3987.149 Safari/537.36",
  "details": {
    "error": {
      "message": "Unable to import MFA factors."
    }
  },
  "user_name": "test@test.io",
  "strategy": "auth0",
  "strategy_type": "database"
}
```

<div id="bulk-user-import">
  ## Importación masiva de usuarios
</div>

1. Prepara un archivo `users.json`. Consulta [importaciones masivas de usuarios](/docs/es/manage-users/user-migration/bulk-user-imports) para obtener más información.
2. Incluye las inscripciones de MFA existentes de cada usuario.
3. Inicia una importación masiva de usuarios.
4. Actualiza los factores de cualquier usuario existente habilitando la opción `upsert` en la solicitud inicial.
5. Cuando se complete la tarea de importación, revisa la respuesta por si hubiera errores. Si no se pudo importar alguno de los factores de MFA de los usuarios, verás errores como:

Al usar la opción `upsert`, cualquier actualización de usuarios existentes no relacionada con MFA se habrá aplicado al perfil del usuario. Por ejemplo, el siguiente resumen de errores muestra que el atributo `foto` del usuario se estableció correctamente en `http://example.org/jdoe.png`; sin embargo, no pudimos importar los factores de MFA proporcionados. En casos como este, puedes volver a intentar con seguridad la importación de los usuarios que fallaron.

```json lines expandable
[
  {
    "user": {
      "email": "antoinette@contoso.com",
      "picture": "http://example.org/jdoe.png",
      "mfa_factors": [
        {
          "totp": {
            "secret": "2PRXZWZAYYDAWCD"
          }
        },
        {
          "phone": {
            "value": "+15551112233"
          }
        },
        {
          "email": {
            "value": "antoinette@antoinette.biz"
          }
        }
      ]
    },
    "errors": [
      {
        "code": "MFA_FACTORS_FAILED",
        "message": "Unable to import factors"
      }
    ]
  }
]
```

<div id="recovery-codes">
  ## Códigos de recuperación
</div>

Auth0 no ofrece ninguna forma de importar códigos de recuperación. Cuando se importan los factores de MFA del usuario, este no tendrá un código de recuperación.

Para proporcionar a los usuarios un código de recuperación, puedes comprobar si ya tienen uno registrado y, si no, usar el [endpoint de Regeneración de código de recuperación](https://auth0.com/docs/api/management/v2#!/Users/post_recovery_code_regeneration) para generar uno nuevo.
