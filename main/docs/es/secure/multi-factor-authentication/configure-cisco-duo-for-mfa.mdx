Cisco Duo es un proveedor de autenticación con múltiples funciones y solo puede usarse en tu tenant de Auth0 si todos los demás factores están deshabilitados. Según tu configuración, tu cuenta de Duo puede admitir notificaciones push, SMS, OTP, llamadas telefónicas y más.

Tampoco puedes habilitar Duo si hay otros factores habilitados. Duo solo está disponible para los usuarios cuando es el único factor habilitado.

La aplicación le pedirá al usuario el segundo factor con Duo y mostrará las opciones que hayas habilitado en tu cuenta de Duo.

Tus usuarios pueden descargar Duo desde [Google Play](https://play.google.com/store/apps/details?id=com.duosecurity.duomobile) o la [App Store](https://itunes.apple.com/us/app/duo-mobile/id422663827?mt=8) para usarlo como segundo factor.

<div id="configure-duo">
  ## Configurar Duo
</div>

Para configurar Duo Security, debes proporcionar las credenciales de Duo a tu aplicación en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Ve a [Dashboard &gt; Security &gt; Multi-factor Auth &gt; Duo Security](https://manage.auth0.com/#/security/mfa/duo) y actívalo.
2. Introduce la información en los campos para vincular tu cuenta de Duo con Auth0.
3. Selecciona **Guardar**.

<div id="use-actions-to-enable-duo">
  ## Usa Actions para habilitar Duo
</div>

Para habilitar Duo dentro de una Action, pasa `duo` como valor del parámetro `provider` al habilitar la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>.

```js lines
exports.onExecutePostLogin = async (event, api) => {
  api.multifactor.enable('duo', { allowRememberBrowser: false });
};
```

Duo no ofrece una opción para el comportamiento de &quot;Recuérdame&quot;. La sesión de MFA de 30 días está configurada de forma fija para recordar al usuario después del inicio de sesión inicial.

Para obligar a sus usuarios a iniciar sesión con Duo cada vez, cree una rule con `allowRememberBrowser: false`.

<div id="current-limitations">
  ## Limitaciones actuales
</div>

* No puedes usar [Auth0 MFA Enrollment Tickets](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/create-custom-enrollment-tickets) para registrar usuarios con Duo. Da de alta a esos usuarios directamente en Duo.
* Si usas <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>, debes habilitar Duo en una Action con `provider` establecido en `duo`, como se describió anteriormente. Puedes usar de forma condicional Duo o el proveedor integrado de Auth0 para aplicaciones específicas.
