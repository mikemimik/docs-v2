title: Personalizar MFA para Classic Login
---

Puedes habilitar la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> en la sección [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Puedes personalizar las páginas de MFA con opciones básicas de marca en el <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Para una personalización más avanzada, puedes proporcionar tu propio HTML para la página de MFA. Al usar tu propio HTML, Classic Login utiliza el Auth0 MFA Widget con las siguientes limitaciones:

* No admite MFA por correo electrónico.
* Si los usuarios tienen más de un factor configurado, no pueden seleccionar cuál usar. El widget de MFA les pedirá que inicien sesión con el factor más seguro.
* No utiliza las funcionalidades de localización de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>. Para obtener más información, consulta [Universal Login Internationalization](/docs/es/customize/internationalization-and-localization/universal-login-internationalization).

<div id="customize-html-for-the-mfa-page">
  ## Personaliza el HTML de la página de MFA
</div>

Para personalizar la página de MFA, ve a la pestaña [Dashboard &gt; Branding &gt; Universal Login &gt; Multi-factor Authentication](https://manage.auth0.com/#/mfa_page) y activa el interruptor **Customize MFA Page**. Después, puedes usar el editor de texto para modificar el HTML, aplicar estilos a la página con CSS y cambiar el JavaScript que se usa para recuperar variables personalizadas. Cuando termines, asegúrate de hacer clic en **Save Changes**.

Las páginas de MFA funcionan sin necesidad de personalización, y Auth0 actualizará los scripts incluidos cuando sea necesario. Sin embargo, una vez que actives la personalización, serás responsable de actualizar y mantener el script (incluido el cambio de números de versión, como el de MFA Widget), ya que Auth0 ya no podrá actualizarlo automáticamente.

Para obtener más información sobre cómo sobrescribir el texto en muchas áreas del proceso de MFA para Classic Login, consulta [diccionario de idiomas del tema de MFA](/docs/es/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary). Para obtener más información sobre las opciones de tema de MFA Widget, consulta [Opciones de tema del MFA Widget](/docs/es/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options).

<div id="reset-page">
  ## Restablecer la página
</div>

Si quieres revertir a un diseño anterior, tienes dos opciones:

* Revertir a la última plantilla guardada haciendo clic en **Reset to Last**;
* Revertir a la plantilla predeterminada de Auth0 haciendo clic en **Reset to Default**.

<div id="render-enrollment-invitations">
  ## Renderizar invitaciones de inscripción
</div>

Hay dos escenarios posibles en los que se renderiza la página. Si se ha dirigido a un usuario a esta página específicamente para completar la inscripción (por ejemplo, desde un correo electrónico con un enlace de inscripción), la propiedad `ticket` estará disponible. De lo contrario, la propiedad `requestToken` estará disponible.

<div id="html-and-liquid-syntax">
  ## Sintaxis de HTML y Liquid
</div>

La página alojada utiliza la sintaxis de Liquid para las plantillas. Para obtener más información, consulta [Liquid for Designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) en GitHub. Los siguientes parámetros están disponibles para ayudar con la renderización de la página:

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

La mayoría de los parámetros que se usan en el MFA Widget deben pasarse a Guardian, como se muestra en la plantilla predeterminada incluida en el área de personalización. Si necesitas un mayor nivel de personalización, puedes usar [auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example).

Para obtener más información, consulta [Customize MFA User Pages](/docs/es/secure/multi-factor-authentication/customize-mfa).
