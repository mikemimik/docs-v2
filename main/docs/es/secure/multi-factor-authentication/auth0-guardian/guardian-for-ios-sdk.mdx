title: SDK de iOS Guardian.swift
---

[Guardian.swift](https://github.com/auth0/Guardian.swift) te permite integrar el servicio Guardian multifactor de Auth0 en tu propia aplicación de iOS, convirtiéndola en el segundo factor. Tus usuarios disfrutarán de todas las ventajas de nuestra <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> sin fricciones directamente desde tu aplicación. Para obtener más información, consulta [Primeros pasos con Apple Push Notification Service](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa#configure-push-notifications-for-apple-using-apn-).

<div id="requirements">
  ## Requisitos
</div>

* Se requieren iOS 10+ y Swift 4.1 para usar Guardian.
* Para usar este SDK, debes configurar el servicio Guardian de tu tenant con tus propias credenciales de notificaciones push; de lo contrario, no recibirás ninguna notificación push. Para obtener más información, consulta [Configurar notificaciones push para MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa).

<div id="install-guardian-ios-sdk">
  ## Instala el SDK de Guardian para iOS
</div>

<div id="cocoapods">
  ### CocoaPods
</div>

Guardian.swift está disponible a través de [CocoaPods](http://cocoapods.org). Para instalarlo, agrega la siguiente línea a tu archivo Podfile:

```bash lines
pod 'Guardian', '~> 1.1.0'
```

<div id="carthage">
  ### Carthage
</div>

Añade esta línea a tu Cartfile:

```bash lines
github "auth0/Guardian.swift" ~> 1.1.0
```

<div id="enable-guardian-push-notifications">
  ## Habilita las notificaciones push de Guardian
</div>

1. Ve a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian).
2. Activa **Push Notification** para habilitarla.
3. [Configura las notificaciones push](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa#configure-push-notifications-for-apple-using-apn-).

<div id="usage">
  ## Uso
</div>

`Guardian` es el componente principal del SDK. Para usar el SDK, importa la biblioteca:

```swift lines
import Guardian
```

Define el dominio de tu tenant. O usa el <Tooltip tip="Custom Domain: Dominio de terceros con un nombre específico o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip> si configuraste uno para tu tenant:

```swift lines
let domain = "<tenant>.<region>.auth0.com"
```

<div id="enroll">
  ### Inscribir
</div>

Una inscripción es un vínculo entre el segundo factor y una cuenta de Auth0. Cuando una cuenta está inscrita, deberás proporcionar el segundo factor necesario para verificar la identidad. Si tu aplicación aún no usa notificaciones push o no estás familiarizado con ellas, consulta [Apple Push Notification Service Overview](https://developer.apple.com/go/?id=push-notifications) para obtener más información.

Para una inscripción necesitas la siguiente información, además del dominio de tu tenant:

| Variable           | Descripción                                                                                                                              |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **Enrollment URI** | Valor codificado en el código QR escaneado desde Guardian Web Widget o en tu ticket de inscripción enviado por correo electrónico o SMS. |
| **APNS Token**     | Token de APNS de Apple para el device. Debe ser una cadena que contenga los 64 bytes (en formato hexadecimal).                           |
| **Key Pair**       | Un par de claves RSA (privada/pública) que se usa para acreditar tu identidad ante Auth0 Guardian.                                       |

Después de obtener la información, puedes inscribir tu device:

```swift lines
Guardian
        .enroll(forDomain: "{yourTenantDomain}",
                usingUri: "{enrollmentUri}",
                notificationToken: "{apnsToken}",
                signingKey: signingKey,
                verificationKey: verificationKey
                )
        .start { result in
            switch result {
            case .success(let enrolledDevice):
                // éxito, tenemos los datos del device de enrollment disponibles
            case .failure(let cause):
                // algo falló, revisa cause para ver qué salió mal
            }
        }
```

Si todo sale bien, obtendrás la información de inscripción, que debe almacenarse de forma segura en tu aplicación. Esta información incluye el identificador de inscripción y el token de la API de Guardian asociado a tu dispositivo para actualizar o eliminar tu inscripción.

<div id="signing-and-verification-keys">
  #### Claves de firma y verificación
</div>

Guardian.swift proporciona una clase auxiliar para generar una clave de firma:

```swift lines
let signingKey = try DataRSAPrivateKey.new()
```

Esta clave solo existe en memoria, pero puedes obtener su representación como `Data` y almacenarla de forma segura, por ejemplo, en una SQLiteDB cifrada:

```javascript lines
// Almacenar datos
let data = signingKey.data
// realizar el almacenamiento

// Cargar desde el almacenamiento
let loadedKey = try DataRSAPrivateKey(data: data)
```

Pero si solo quieres guardarlo en el Keychain de iOS:

```swift wrap lines
let signingKey = try KeychainRSAPrivateKey.new(with: "com.myapp.mytag")
```

El ejemplo anterior crea una clave y la almacena automáticamente con la etiqueta indicada. Si quieres recuperarla, puedes usar la etiqueta:

```swift wrap lines
let signingKey = try KeychainRSAPrivateKey(tag: "com.myapp.mytag")
```

Para la clave de verificación, basta con obtenerla de cualquier `SigningKey`, por ejemplo:

```swift lines
let verificationKey = try signingKey.verificationKey()
```

<div id="allow-login-requests">
  ### Aprobar solicitudes de inicio de sesión
</div>

Una vez que tengas la inscripción configurada, recibirás una notificación push cada vez que el usuario tenga que validar su identidad con MFA. Guardian proporciona un método para interpretar los datos recibidos de APNs y devolver una instancia de `Notification` lista para usar.

```swift lines
if let notification = Guardian.notification(from: notificationPayload) {
    // hemos recibido una notificación push de Guardian
}
```

Una vez que tengas la instancia de notificación, puedes aprobar fácilmente la solicitud de autenticación con el método `allow`. También necesitarás cierta información del dispositivo registrado que obtuviste anteriormente. En caso de que tengas más de una inscripción, tendrás que encontrar la que tenga el mismo `id` que la notificación (la propiedad `enrollmentId`).

Cuando tengas la información, el parámetro `device` puede ser cualquier elemento que implemente el protocolo `AuthenticatedDevice`:

```swift lines
struct Authenticator: Guardian.AuthenticationDevice {
    let signingKey: SigningKey
    let localIdentifier: String
}
```

El identificador local es el ID local del dispositivo; de forma predeterminada, al registrarlo se usa `UIDevice.current.identifierForVendor`. Luego, simplemente llama a:

```swift lines
Guardian
        .authentication(forDomain: "{yourTenantDomain}", device: device)
        .allow(notification: notification)
        .start { result in
            switch result {
            case .success:
                // la solicitud de autenticación fue permitida exitosamente
            case .failure(let cause):
                // algo falló, revisa cause para ver qué salió mal
            }
        }
```

<div id="reject-login-requests">
  ### Rechazar solicitudes de inicio de sesión
</div>

Para rechazar una solicitud de autenticación, debes llamar a `reject`. También puedes enviar un motivo de rechazo opcional. El motivo de rechazo aparecerá en los registros de Guardian.

```swift lines
Guardian
        .authentication(forDomain: "{yourTenantDomain}", device: device)
        .reject(notification: notification)
        // o reject(notification: notification, withReason: "hacked")
        .start { result in
            switch result {
            case .success:
                // la solicitud de autenticación fue rechazada exitosamente
            case .failure(let cause):
                // algo falló, revisa cause para ver qué salió mal
            }
        }
```

<div id="unenroll">
  ### Cancelar la inscripción
</div>

Si quieres eliminar una inscripción, por ejemplo, si quieres desactivar MFA, puedes hacer la siguiente solicitud:

```swift lines
Guardian
        .api(forDomain: "{yourTenantDomain}")
        .device(forEnrollmentId: "{userEnrollmentId}", token: "{enrollmentDeviceToken}")
        .delete()
        .start { result in
            switch result {
            case .success:
                // éxito, la inscripción fue eliminada
            case .failure(let cause):
                // algo falló, revisa cause para ver qué salió mal
            }
        }
```

<div id="set-up-mobile-only-otp-enrollment">
  ### Configurar la inscripción de OTP solo para dispositivos móviles
</div>

Puede habilitar las contraseñas de un solo uso (OTP) como factor de MFA mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Esta opción no requiere un QR code y permite que los usuarios se inscriban manualmente.

Para invitar a un usuario a inscribirse, vaya a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y seleccione un usuario. Luego, abra la pestaña Details y use la sección Multi-Factor Authentication para enviar una invitación de inscripción.

<div id="connect-a-resource">
  #### Conectar un recurso
</div>

Puedes conectar un recurso mediante el Auth0 Dashboard o el SDK de Guardian.

<div id="use-auth0-dashboard">
  ##### Usa Auth0 Dashboard
</div>

1. Accede al prompt de inicio de sesión de Auth0 y copia el código de un solo uso proporcionado o una clave similar codificada en base32 obtenida de otra fuente. En el siguiente paso, introducirás este código en una aplicación de autenticación.

   <Frame>![Un ejemplo de prompt de inicio de sesión que muestra un código de un solo uso](/docs/images/cdy7uua7fh8z/1yoqiIuERVTwCU8yfx6IM8/047513dfe1d40a22ce811b131d5ea289/OTP_Challenge_2_-_English.png)</Frame>
2. Agrega el código que copiaste a una aplicación de autenticación, como Guardian.

<div id="use-the-sdk">
  ##### Usa el SDK
</div>

1. Importa la biblioteca Guardian.

   ```swift lines
   import Guardian
   ```

2. Crea un generador de códigos.

   ```swift lines
   let codeGenerator = try Guardian.totp(

      base32Secret: enrollmentCode,  // código de inscripción introducido por el usuario

      algorithm: .sha1			// Algoritmo utilizado por TOTP

   )
   ```

3. Obtén el código generado.

   ```swift lines
   codeGenerator.code()
   ```

<div id="enter-one-time-code">
  #### Introduce el código de un solo uso
</div>

En el prompt de inicio de sesión de Auth0, introduce el código que generaste en el paso anterior.

<Frame>![Un ejemplo de prompt de inicio de sesión que muestra un código de un solo uso](/docs/images/cdy7uua7fh8z/1yoqiIuERVTwCU8yfx6IM8/047513dfe1d40a22ce811b131d5ea289/OTP_Challenge_2_-_English.png)</Frame>

Después de seleccionar Continuar, se mostrará un mensaje indicando que tu aplicación se ha agregado como factor de autenticación para ese usuario.
