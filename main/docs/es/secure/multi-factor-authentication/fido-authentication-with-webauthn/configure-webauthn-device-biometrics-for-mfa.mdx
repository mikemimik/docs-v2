<Warning>
  Si necesitas configurar un dominio personalizado, hazlo **antes** de implementar WebAuthn en entornos de producción. Si configuras o cambias un dominio personalizado, los usuarios inscritos previamente no podrán autenticarse.

  Puedes usar la configuración de [Relying Party](#configure-relying-party) para especificar el dominio que se usa para autenticar a los usuarios.

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

Habilita **WebAuthn with Device Biometrics** en [Dashboard &gt; Security &gt; Multifactor Auth](https://manage.auth0.com/#/security/mfa). Tendrás que habilitar otro factor, ya que este no puede ser el único factor habilitado.

<div id="configure-relying-party">
  ## Configurar Relying Party
</div>

WebAuthn hace que el phishing sea imposible al vincular las credenciales al origen del navegador. Los usuarios tampoco pueden usar WebAuthn en un sitio en el que no se registraron.

Vincular las credenciales al origen significa que, si configuras un <Tooltip tip="Custom Domain: Dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip> o lo cambias, los usuarios inscritos antes del cambio no podrán autenticarse.

WebAuthn define un [atributo de ID de Relying Party](https://www.w3.org/TR/webauthn-2/#relying-party-identifier), que te permite especificar el dominio que se usa para autenticar a los usuarios. Puedes establecerlo en cualquier sufijo de dominio registrable del origen del navegador. Por ejemplo, si el dominio personalizado es `login.example.com`, puedes configurar el ID de <Tooltip tip="Relying Party: Entidad (como un servicio o aplicación) que depende de un proveedor de identidad de terceros para autenticar a un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Relying+Party">Relying Party</Tooltip> como `example.com`. Esto permite a los usuarios autenticarse en cualquier dominio `example.com` con sus credenciales de WebAuthn.

Auth0 te permite especificar el ID de Relying Party solo si tienes configurado un dominio personalizado. Si el dominio personalizado cambia, debes actualizar el ID de Relying Party.

<div id="device-recognition">
  ## Reconocimiento de dispositivos
</div>

Cuando habilitas WebAuthn with Device Biometrics, Auth0 intentará registrar progresivamente todos los dispositivos de un usuario final compatibles con WebAuthn. Los navegadores con Javascript deshabilitado o sin compatibilidad con un autenticador de plataforma de WebAuthn no tendrán la opción de registrarse ni de autenticarse con Device Biometrics.

Como no existe una forma determinista de saber si un dispositivo específico se registró o no sin solicitar al usuario una verificación de WebAuthn, Auth0 se basa en el agente de usuario para decidir qué hacer. El comportamiento depende del sistema operativo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En el escenario más habitual, en el que los usuarios siempre usan el mismo navegador y tienen un dispositivo móvil y una computadora portátil o de escritorio, los matices de comportamiento que se describen a continuación no afectarán la experiencia del usuario.
</Callout>

<div id="windows-and-ios-145">
  ### Windows e iOS 14.5+
</div>

En Windows e iOS 14.5+, el autenticador de plataforma WebAuthn se registra a **nivel del sistema operativo**. Los usuarios pueden registrarse con un navegador e iniciar sesión desde cualquier navegador. Si Auth0 detecta que un usuario tiene un dispositivo registrado, se le ofrecerá la opción de autenticarse con Face ID / Touch ID / Windows Hello. Si se registró con ese mismo dispositivo, podrá autenticarse. De lo contrario, fallará y tendrá que usar otro método de autenticación.

<div id="macos">
  ### macOS
</div>

En Mac, el autenticador de plataforma WebAuthn se registra a nivel del sistema del **navegador**. Se pedirá a los usuarios que se registren en WebAuthn en cada navegador que utilicen. Si Auth0 detecta que el usuario se registró desde Chrome en un Mac, tendrá la opción de autenticarse con Touch ID cuando inicie sesión desde Chrome en un Mac. Si se registró desde el mismo Mac, podrá autenticarse. Si no, fallará y tendrá que usar otro método de autenticación. Si intenta registrarse desde Safari en el mismo Mac, se le pedirá que complete la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> con el otro método de autenticación y luego **se le pedirá que se registre** con Touch ID.

<div id="android">
  ### Android
</div>

En Android, **solo Chrome** admite autenticadores de plataforma WebAuthn. Si Auth0 detecta que los usuarios tienen registrado un dispositivo Android, verán la opción de autenticarse con el reconocimiento de huella dactilar/facial de Android. Si registraron ese mismo dispositivo Android, podrán autenticarse. De lo contrario, fallará y tendrán que usar otro método de autenticación.

<div id="device-support">
  ## Compatibilidad de dispositivos
</div>

El usuario debe tener activado otro registro de MFA antes de usar la biometría del dispositivo.

Para usar las claves de biometría del dispositivo, el navegador debe tener Javascript habilitado y ser compatible con los autenticadores de plataforma WebAuthn. Si no se cumplen esas condiciones, Auth0 no ofrecerá la opción de registrar ni autenticar el dispositivo. Auth0 solicitará al usuario otro factor.

Las versiones más recientes de los navegadores y sistemas operativos más populares ofrecen compatibilidad con WebAuthn con claves de seguridad. Para más detalles, consulta la [sección de compatibilidad de navegadores en webauthn.me](https://webauthn.me/browser-support).

<div id="limitations">
  ## Limitaciones
</div>

* No es posible registrar WebAuthn with Device Biometrics fuera del prompt de inscripción progresiva.
* Al usar la [API de MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api), puedes listar y eliminar registros de WebAuthn, pero no registrarlos.
* Los usuarios solo pueden registrar un dispositivo por tipo con WebAuthn with Device Biometrics (un teléfono, una tableta, una laptop o computadora de escritorio). Si un usuario quiere registrar otro dispositivo del mismo tipo, primero debe anularse el registro del dispositivo anterior.

  ## Webauthn.me
