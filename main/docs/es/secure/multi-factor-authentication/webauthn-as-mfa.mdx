---
description: Breve descripción de cómo funciona WebAuthn como factor para MFA
title: WebAuthn como autenticación multifactor
---

Cuando los usuarios se autentican con WebAuthn, usan algo que **tienen** como factor de autenticación: una clave de seguridad o un dispositivo.

Tanto las Security Keys como Device Biometrics admiten la **verificación del usuario**, que exige que los usuarios aporten algo que **saben** (un PIN o un código de acceso) y algo que **son** (como rasgos biométricos).

Al usar Device Biometrics, la verificación del usuario siempre se realiza. Para realizar la verificación con Security Keys, [debes configurar Auth0](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa) para exigir un PIN. Después, se pedirá a los usuarios que introduzcan un PIN, que solo se almacena en la clave de seguridad, para completar la autenticación. Cuando se realiza la verificación del usuario, los usuarios pueden iniciar sesión con WebAuthn como único método de autenticación para lograr la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>.

Al usar WebAuthn para la autenticación junto con la verificación del usuario, no solo sustituyes la contraseña por algo mucho más sencillo de usar, sino que también eliminas la necesidad de añadir otro paso de autenticación cuando se requiere MFA.
