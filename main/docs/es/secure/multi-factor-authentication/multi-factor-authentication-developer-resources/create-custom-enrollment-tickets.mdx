Si quieres personalizar y gestionar las inscripciones de tus usuarios en la [Autenticación multifactor (MFA)](/docs/es/secure/multi-factor-authentication), puedes usar tickets de inscripción personalizados. Los tickets de inscripción personalizados de Auth0 generan un enlace de un solo uso para que tus usuarios se inscriban en <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> con los [factores](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors) que especifiques.

<div id="use-cases">
  ### Casos de uso
</div>

Usa tickets de inscripción personalizados para:

* Incorporar nuevos usuarios
* Configurar los ajustes de los usuarios
* Especificar qué factores quieres que los clientes elijan durante la inscripción
* Permitir que los usuarios inscriban más de un factor

<div id="configure-custom-enrollment-tickets">
  ### Configurar tickets de inscripción personalizados
</div>

Haz una solicitud POST al endpoint [`/guardian/post_ticket`](https://auth0.com/docs/api/management/v2#!/Guardian/post_ticket) de <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

</AuthCodeGroup>

| Parámetro                    | Obligatorio | Descripción                                                                                                                                                                            |
| ---------------------------- | ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `allow_multiple_enrollments` | Opcional    | Permite que un usuario ya inscrito se inscriba con factores adicionales, excepto correo electrónico.                                                                                   |
| `factor`                     | Opcional    | Especifica con qué factor debe inscribirse el usuario, excepto correo electrónico. Si se usa con `allow_multiple_enrollments`, puede especificar más de un factor para la inscripción. |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Estos parámetros solo se pueden usar con [Universal Login](/docs/es/authenticate/login/auth0-universal-login) y no se pueden usar con Classic Login ni con páginas de MFA personalizadas.
</Callout>

Management API devuelve un ticket de inscripción que contiene un `ticket_id` y una `ticket_url`. El `ticket_id` es solo para uso interno. Entrega la `ticket_url` al usuario con el parámetro `send_email` para iniciar el proceso de inscripción. Los tickets caducan después de 5 días y son de un solo uso. Un usuario que recibe un ticket solo podrá inscribirse una vez.

Respuesta de ejemplo:

```json lines
{
  "ticket_id": "gten_8b2f5e90d2c848dc9230d34f",
  "ticket_url": "https://guardianadnrdoifcmtest0.eu.auth0.com/guardian/enroll?ticket=gten_8b2f5e90d2c848dc9230d34f"
}
```

Si habilitas [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains), debes incluir el encabezado HTTP `auth0-custom-domain`. Para obtener más información, consulta [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains#customize-email-handling-using-the-management-api).

<div id="custom-enrollment-tickets-with-classic-login">
  ### Tickets de inscripción personalizados con Classic Login
</div>

Si usas la experiencia [Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) y necesitas personalizar el aspecto de la página cuando el usuario accede a `ticket_url`, puedes editar la página de MFA.

1. Ve a [Dashboard &gt; Branding &gt; Universal Login &gt; Opciones avanzadas &gt; Autenticación multifactor.](https://manage.auth0.com/#/mfa_page)
2. Si **Personalizar página de MFA** no está seleccionado, activa esta opción.
3. Actualiza la variable `ticket`.
   Ejemplo:

```liquid lines
{% if ticket %}
<h4 class="message">Welcome, {{ userData.email }}, enroll your device below</h4>
{% else %}
<h4 class="message">Welcome back, {{ userData.email }}, authenticate below</h4>
{% endif %}
```
