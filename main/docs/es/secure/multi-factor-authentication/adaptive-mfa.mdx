es una política de

flexible y extensible que puede ayudarte a proteger tu tenant frente a

<div id="how-it-works">
  ## Cómo funciona
</div>

Durante una transacción de inicio de sesión, MFA adaptativo calcula una puntuación de confianza general a partir del análisis de tres evaluaciones de riesgo:

| Evaluador                    | Señal de riesgo                                                                                                                                               | Cómo se calcula                                                                                                                                                                                                                                                                                             |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `NewDevice`                  | El usuario intenta iniciar sesión desde un dispositivo que no se ha utilizado para acceder a la cuenta en los últimos 30 días.                                | El agente de usuario y las cookies del navegador permiten identificar un dispositivo. En el momento del inicio de sesión, la información del dispositivo se compara con la lista de dispositivos de la cuenta.                                                                                              |
| `ImpossibleTravel`           | El usuario intenta iniciar sesión desde una ubicación geográfica que, en comparación con el último inicio de sesión, indica una situación de viaje imposible. | Se calcula la distancia entre la última ubicación válida y la ubicación del intento de inicio de sesión; la diferencia de tiempo entre ambos inicios de sesión se utiliza para calcular una velocidad de desplazamiento hipotética. Esa velocidad se compara con una velocidad de desplazamiento razonable. |
| `UntrustedIP`                | El usuario intenta iniciar sesión desde una dirección IP que se sabe que está asociada con comportamientos sospechosos.                                       | Auth0 usa inteligencia obtenida de eventos de tráfico para determinar la probabilidad de que la dirección IP haya sido utilizada por actores maliciosos para perpetrar ataques de alta velocidad.                                                                                                           |
| Puntuación de riesgo general | Una combinación de los 3 factores anteriores.                                                                                                                 | Auth0 usa las 3 puntuaciones para asignar una puntuación general. Usa [Actions](/docs/es/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa) si quieres implementar tu propia lógica de negocio.                                                                                           |

Cuando MFA adaptativo determina que la puntuación de confianza general es baja (es decir, que la transacción de inicio de sesión es de alto riesgo), exige que el usuario verifique su identidad con MFA. Si el usuario no está inscrito en MFA, debe completar una verificación adicional antes de poder hacerlo.

MFA adaptativo incluye un flujo de seguridad integral que garantiza la autenticidad de los usuarios:

<Frame>![Diagrama del flujo de autenticación multifactor adaptativa de Auth0 Login](/docs/images/cdy7uua7fh8z/6NpeGW8jtQIma1jTVydW7U/d0f80c28226abfe10863f34d30f1a0be/Adaptive_MFA_Flow.png)</Frame>

MFA adaptativo ignora cualquier sesión de MFA existente (por ejemplo, si un usuario seleccionó **Recordar este navegador** durante un flujo de MFA anterior) y no permite que los usuarios eludan los desafíos MFA.

<div id="authorization-flows">
  ### Flujos de autorización
</div>

MFA adaptativo es compatible con todos los flujos de autenticación y <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) especificada en el marco de OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flows">flujos de autorización</Tooltip> que comienzan con el usuario final. Para obtener más información sobre los distintos flujos y protocolos, consulta [Flujos de autenticación y autorización](/docs/es/get-started/authentication-and-authorization-flow) y [Protocolos](/docs/es/authenticate/protocols).

| Protocolo     | Flujo                                                     | Compatible     |
| ------------- | --------------------------------------------------------- | -------------- |
| OIDC/OAuth2   | Flujo de código de autorización                           | Compatible     |
| OIDC/OAuth2   | flujo de código de autorización con PKCE                  | Compatible     |
| OIDC/OAuth2   | flujo implícito con Form Post                             | Compatible     |
| OIDC/OAuth2   | Flujo híbrido                                             | Compatible     |
| OIDC/OAuth2   | Credenciales de cliente                                   | No compatible  |
| OIDC/OAuth2   | Autorización de dispositivo                               | No compatible  |
| OIDC/OAuth2   | Contraseña del propietario del recurso (ROP)              | No compatible  |
| OIDC/OAuth2   | Custom Token Exchange                                     | No compatible  |
| SAML          | Iniciado por el proveedor de servicios (iniciado por SP)  | Compatible     |
| SAML          | Iniciado por el proveedor de identidad (iniciado por IdP) | No compatible* |
| WS-Federation | N/A                                                       | Compatible     |
| AD/LDAP       | N/A                                                       | Compatible     |

*MFA adaptativo no es compatible con los flujos de SAML iniciados por <Tooltip tip="Security Assertion Markup Language (SAML): protocolo estandarizado que permite que dos partes intercambien información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML): protocolo estandarizado que permite que dos partes intercambien información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip>, pero puedes simular este flujo con aplicaciones OIDC. Para obtener más información, consulta [Configurar el inicio de sesión SAML iniciado por IdP en aplicaciones OIDC](/docs/es/authenticate/protocols/saml/saml-sso-integrations/configure-idp-initiated-saml-sign-on-to-oidc-apps).

<div id="social-connections">
  ### Conexiones sociales
</div>

El MFA adaptativo es totalmente compatible con los tipos de conexión social en los que hay una dirección de correo electrónico disponible para cada usuario.

El MFA adaptativo requiere una dirección de correo electrónico para completar el paso del desafío por correo electrónico que se produce cuando un usuario no está inscrito en MFA. Si no hay una dirección de correo electrónico disponible, el MFA adaptativo no puede realizar el desafío por correo electrónico y la transacción se bloqueará. Este caso no introduce ningún riesgo de seguridad, pero sí limita la funcionalidad.

Si configuraste una conexión social y esperas que haya una dirección de correo electrónico disponible, pero no la hay, verifica tu configuración y confirma que se estén solicitando los `scopes`, `claims` y permisos correctos. Para obtener más información sobre las conexiones sociales compatibles y cómo instalarlas, consulta [Social Connections en Auth0 Marketplace](https://marketplace.auth0.com/features/social-connections).
