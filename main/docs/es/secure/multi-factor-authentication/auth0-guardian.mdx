Auth0 Guardian es una aplicación móvil para dispositivos iOS y Android que permite a los usuarios completar la [autenticación multifactor (MFA)](/docs/es/secure/multi-factor-authentication) mediante notificaciones push o contraseñas temporales de un solo uso.

Auth0 Guardian puede enviar notificaciones push a los dispositivos registrados de los usuarios (normalmente teléfonos móviles o tabletas) o generar contraseñas de un solo uso directamente en la aplicación. Después, los usuarios pueden responder rápidamente a estas notificaciones push o recuperar una contraseña de un solo uso para completar su inicio de sesión.

Los usuarios pueden descargar la aplicación Auth0 Guardian desde la [Apple App Store](https://apps.apple.com/us/app/auth0-guardian/id1093447833) o la [Google Play Store](https://play.google.com/store/apps/details?id=com.auth0.guardian). Como alternativa, puedes integrar las capacidades de Auth0 Guardian en tu propia aplicación personalizada mediante el Guardian SDK.

<div id="push-notifications">
  ## Notificaciones push
</div>

Para usar las notificaciones push, los usuarios deben tener instalada en su dispositivo la aplicación Auth0 Guardian o una aplicación personalizada creada con el Guardian SDK. Cuando un usuario intenta autenticarse, se envían notificaciones push a la aplicación instalada. El usuario debe responder a la notificación para completar su inicio de sesión, lo que demuestra que conoce sus credenciales de acceso y que además posee el dispositivo configurado para la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>.

Las notificaciones push de Auth0 se pueden implementar con AWS Simple Notification Service (SNS) o con uno o más de los siguientes servicios directos a proveedores para configurar integraciones específicas de cada proveedor:

* Firebase Cloud Messaging (FCM)

  * Para obtener más información sobre cómo modificar el contenido de las notificaciones push, consulta la [documentación de FCM](https://firebase.google.com/docs/cloud-messaging/concept-options).
* Apple Push Notification (APN)

  * Para obtener más información sobre cómo modificar el contenido de las notificaciones push, consulta la [documentación de Apple](https://developer.apple.com/documentation/usernotifications/modifying-content-in-newly-delivered-notifications).

<div id="enroll-in-push-notifications">
  ### Inscribirse en notificaciones push
</div>

Cuando usa Auth0 Guardian para configurar notificaciones push, se les pide a los usuarios que descarguen la aplicación móvil la primera vez que se registran o inician sesión en su aplicación. Si usa el Guardian SDK para implementar notificaciones push en una aplicación personalizada, no se les pedirá que descarguen ninguna aplicación durante la inscripción.

Puede habilitar las notificaciones push desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> en [Security &gt; Multi-factor Auth &gt; Push Notification using Auth0 Guardian](https://manage.auth0.com/#/security/mfa/guardian).

<Frame>![Auth0 Dashboard > Security > Multi-factor Auth > Push Notification using Auth0 Guardian](/docs/images/cdy7uua7fh8z/fwjSKFNhsvObzdPDiA79N/65850980f88029cb23eef5eaef4f9b20/Auth0_Dashboard_-_Push_Notification_screen.png)</Frame>

La primera vez que un usuario se registra o inicia sesión en su aplicación, [Universal Login](/docs/es/authenticate/login/auth0-universal-login) muestra un código QR que puede usar para registrar la aplicación Auth0 Guardian o una aplicación personalizada con Guardian SDK como factor de autenticación secundario. El usuario dispone de poco tiempo para escanear este código con la aplicación indicada y completar la inscripción.

Después de inscribirse, el usuario podrá usar notificaciones push como factor de autenticación. Cada vez que intente iniciar sesión en su aplicación, recibirá una notificación push en su dispositivo a través de la aplicación Auth0 Guardian o de una aplicación personalizada con Guardian SDK. El usuario debe aprobar esta solicitud para iniciar sesión correctamente en su aplicación.

Para saber cómo restablecer MFA para los usuarios que han perdido sus dispositivos o códigos de recuperación, consulte [Reset Multi-Factor Authentication and Recovery Codes](/docs/es/secure/multi-factor-authentication/reset-user-mfa).

<div id="temporary-one-time-passwords">
  ## Contraseñas temporales de un solo uso
</div>

La aplicación Auth0 Guardian y el Guardian SDK también admiten el uso de contraseñas temporales de un solo uso (OTP) como factores de autenticación secundarios. Tanto la aplicación como el SDK pueden generar OTP temporales que los usuarios pueden usar para completar los <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> desafíos MFA.

De forma predeterminada, la aplicación Auth0 Guardian muestra las aplicaciones registradas del usuario. Después de seleccionar una aplicación, se muestra el OTP correspondiente. Cada OTP de 6 dígitos tiene una validez de 30 segundos. Los OTP que están por vencer se muestran en rojo. Al vencer, la aplicación genera inmediatamente un nuevo OTP. Para evitar errores de autenticación, los usuarios no deben usar ningún OTP en rojo

En las aplicaciones personalizadas con Guardian SDK, la apariencia de los OTP puede variar.

<div id="use-temporary-one-time-passwords">
  ### Usar contraseñas temporales de un solo uso
</div>

Puedes usar las OTP de dos maneras principales:

* Como opción de respaldo para las notificaciones push en caso de que la aplicación Auth0 Guardian u otra aplicación personalizada no reciba una solicitud push.
* Como desafío MFA, si habilitas el [factor One-Time Password](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors) en tu tenant.

<AccordionGroup>
  <Accordion title="Método de respaldo: usar OTP como opción de respaldo para las notificaciones push">
    Si un usuario no recibe una notificación push a través de Auth0 Guardian u otra aplicación personalizada, puede usar una OTP temporal para completar su desafío MFA.

    **Ejemplo de flujo de usuario**:

    1. Un usuario intenta iniciar sesión en tu aplicación. Llega al prompt MFA de Universal Login, pero no recibe una notificación push.
    2. En el prompt MFA, el usuario selecciona el botón **Manually Enter Code**. Debe seleccionar esta opción para autenticarse con OTP.

       <Frame>![Ejemplo de un prompt MFA que muestra el botón Manually Enter Code](/docs/images/cdy7uua7fh8z/7reEOLI5CWxbpRzgZY7gxg/58da830927a33545d9a576974ec3ba6e/MFA_-_Push_-_English.png)</Frame>
    3. El usuario abre Auth0 Guardian o la aplicación personalizada con Guardian SDK y selecciona la aplicación a la que intenta acceder.
    4. El usuario copia la OTP que aparece en la parte inferior de la página Detalles de la cuenta.
    5. El usuario introduce la OTP en el prompt MFA para completar el desafío.
  </Accordion>

  <Accordion title="Desafío MFA: usar OTP como desafío MFA">
    Para permitir que los usuarios se autentiquen con OTP, debes habilitar **One-Time Password** como [factor MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors) para tu tenant. Una vez habilitado, los usuarios pueden completar prompts MFA con OTP al iniciar sesión mediante Universal Login. Para habilitar factores MFA, ve a [Auth0 Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).

    Los usuarios pueden inscribirse en OTP como factor MFA cuando se registran por primera vez o inician sesión en tu aplicación. Según el tipo de dispositivo que utilicen, [Universal Login](/docs/es/authenticate/login/auth0-universal-login) muestra una de las siguientes opciones:

    * **Para dispositivos no móviles**: Universal Login muestra un código QR que el usuario puede escanear para continuar con la inscripción.

    <Frame>![Ejemplo de un prompt de inscripción en OTP que muestra el enlace Trouble Scanning?](/docs/images/cdy7uua7fh8z/3kbyOBiLOpJiY8vI0kUqwg/b10f58a3b3ace74de613e28517726069/OTP_Challenge_3_-_English.png)</Frame>

    * **Para dispositivos móviles**: Universal Login omite el código QR y, en su lugar, muestra directamente el código de inscripción.

    <Frame>![Pantalla de inscripción de contraseña temporal de un solo uso tal como se muestra en un dispositivo móvil.](/docs/images/cdy7uua7fh8z/5QvQ9FLYnVZXINFAeLkIr9/04df9be004169ac06b8d9fee456dc599/OTP_prompt_-_English.png)</Frame>

    Con cualquiera de estos métodos, los usuarios pueden seguir los pasos en pantalla para registrar Auth0 Guardian o una aplicación personalizada con Guardian SDK como factor de autenticación secundario. Normalmente, este proceso implica que el usuario agregue tu aplicación a la app correspondiente y obtenga la OTP generada para esa aplicación. Luego, el usuario introduce la OTP en el prompt de Universal Login para finalizar el proceso de inscripción.

    La próxima vez que el usuario intente iniciar sesión en tu aplicación después de completar la inscripción, podrá autenticarse introduciendo una OTP desde Auth0 Guardian o desde su aplicación personalizada con Guardian SDK en el desafío.
  </Accordion>
</AccordionGroup>

<div id="user-security-settings">
  ### Configuración de seguridad del usuario
</div>

En la aplicación Auth0 Guardian, los usuarios pueden habilitar códigos de acceso y biometría como capas de seguridad de la aplicación en iOS y Android. Si un usuario habilita una o más de estas opciones, debe completar estas verificaciones antes de poder responder a las notificaciones push u obtener contraseñas de un solo uso.

Para habilitar estas opciones en dispositivos iOS o Android, los usuarios pueden seguir los pasos que se indican a continuación.

<Tabs>
  <Tab title="iOS">
    Para habilitar la configuración de seguridad de la aplicación Auth0 Guardian en iOS:

    1. En la aplicación, selecciona el **icono de engranaje** para abrir el menú Settings.
    2. Selecciona **Código de acceso** para habilitar la protección con código de acceso.
    3. Define y confirma un código de acceso de 6 dígitos.

    La aplicación ahora está protegida por el código de acceso. El usuario debe ingresar este código de acceso antes de poder responder a las notificaciones push u obtener OTP.

    Después de habilitar la protección con código de acceso, el usuario puede configurar la biometría del dispositivo como medida de seguridad alternativa. Para habilitar la biometría del dispositivo:

    1. En la aplicación, selecciona el **icono de engranaje** para abrir el menú Settings.
    2. Habilita la biometría del dispositivo seleccionando una de las opciones disponibles, como **Face ID** o **Touch ID**.
  </Tab>

  <Tab title="Android">
    Para habilitar la configuración de seguridad de la aplicación Auth0 Guardian en Android:

    1. En la aplicación, selecciona el **menú de tres puntos** para abrir el menú Settings.
    2. Habilita el switch de la opción **Solicitar siempre la passphrase**.
    3. Ingresa la passphrase que se configuró durante la configuración de la aplicación Auth0 Guardian.

    La aplicación ahora está protegida por la passphrase de Auth0 Guardian. El usuario debe ingresar esta passphrase antes de poder responder a las notificaciones push u obtener OTP.

    Después de habilitar la protección con passphrase, el usuario puede configurar la biometría del dispositivo como medida de seguridad alternativa. Para habilitar la biometría del dispositivo:

    1. En la aplicación, selecciona el **menú de tres puntos** para abrir el menú Settings.
    2. Habilita el switch de la opción **Usar biometría para desbloquear**.
  </Tab>
</Tabs>

En la sección Language del menú Settings, los usuarios pueden seleccionar su idioma preferido. De forma predeterminada, la app usa el mismo idioma que el sistema del dispositivo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

<div id="guardian-app-themes">
  ### Temas de la aplicación Auth0 Guardian
</div>

La aplicación Auth0 Guardian, tanto para iOS como para Android, admite los temas de modo claro y oscuro.

En la sección Theme del menú Settings, los usuarios pueden seleccionar las siguientes opciones:

* **System**: Usa el tema predeterminado del sistema
* **Light**: Activa el tema de modo claro
* **Dark**: Activa el tema de modo oscuro

<div id="customization-options-for-protected-applications">
  ### Opciones de personalización para aplicaciones protegidas
</div>

Para que a los usuarios les resulte más fácil distinguir sus aplicaciones protegidas en la aplicación Auth0 Guardian, cada aplicación puede personalizarse con un nombre, un color de acento y un icono diferentes.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El nombre predeterminado asignado a tu aplicación en la aplicación Auth0 Guardian depende de tu [nombre del tenant](/docs/es/get-started/tenant-settings/find-your-tenant-name-or-tenant-id) o del Friendly Name configurado en la [configuración de tu tenant](/docs/es/get-started/tenant-settings). Del mismo modo, el logotipo predeterminado proviene de la URL del logotipo indicada en la configuración de tu tenant.

  Para editar la configuración de tu tenant, ve a [Auth0 Dashboard &gt; Settings &gt; General](https://manage.auth0.com/dashboard/#/tenant/general).
</Callout>

Para personalizar una aplicación protegida:

1. Selecciona una aplicación de la lista.

   * **Consejo**: También puedes deslizar el dedo hacia la izquierda sobre la aplicación para mostrar las opciones Edit y Delete.
2. En la página Detalles de la cuenta, elige **Edit**.
3. Actualiza el nombre, el color y el icono de la aplicación según sea necesario.
4. Selecciona **Save**.

<div id="guardian-sdks">
  ## SDKs de Guardian
</div>

Puedes [instalar el Guardian SDK](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/install-guardian-sdk) (disponible para [iOS](/docs/es/secure/multi-factor-authentication/auth0-guardian/guardian-for-ios-sdk) y [Android](/docs/es/secure/multi-factor-authentication/auth0-guardian/guardian-for-android-sdk)) para crear tu propia aplicación de autenticación multifactor con control total sobre el branding y la apariencia. Con el Guardian SDK, puedes crear tus propias aplicaciones móviles personalizadas que funcionen como Guardian o integrar algunas funcionalidades de Guardian, como recibir notificaciones push en tus aplicaciones móviles existentes. Un caso típico podría ser una aplicación bancaria. Puedes usar el Guardian SDK en tu aplicación móvil existente para recibir y confirmar notificaciones push cuando alguien realiza una transacción en un cajero automático.

<div id="use-actions-to-enable-multi-factor-with-auth0-guardian">
  ## Usa Actions para habilitar la autenticación multifactor con Auth0 Guardian
</div>

Para habilitar Auth0 Guardian en una Action, pasa `guardian` como valor del parámetro `provider` al habilitar la autenticación multifactor:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  api.multifactor.enable('guardian', { allowRememberBrowser: false });
};
```

Para obligar a sus usuarios a iniciar sesión con Auth0 Guardian cada vez que lo hagan, cree la Action con `allowRememberBrowser: false`.

<div id="multi-factor-with-auth0-guardian-and-authorization-extension">
  ## Autenticación multifactor con Auth0 Guardian y Authorization Extension
</div>

Esta plantilla ofrece un ejemplo y un punto de partida para activar la autenticación multifactor con Auth0 Guardian mediante notificaciones push cuando se cumple una condición.

En el primer inicio de sesión, el usuario puede registrar el dispositivo. Consulta Enable Multi-Factor Authentication para obtener más información sobre M

```js lines
exports.onExecutePostLogin = async (event, api) => {
const groups = event.user.app_metadata.authorization.groups;
const GROUPS_WITH_MFA = {
// Agregar aquí los grupos que necesitan MFA
// Ejemplo
admins: true
};

const needsMFA = !!groups.find(function (group) {
return GROUPS_WITH_MFA[group];
});

if (needsMFA) {
// opcional, el valor predeterminado es true. Establecer en false para forzar la autenticación de Guardian en cada inicio de sesión.
// Ver https://auth0.com/docs/secure/multi-factor-authentication/customize-mfa#change-frequency-of-mfa-prompts para más detalles
api.multifactor.enable('guardian', { allowRememberBrowser: false });
}

};
```
