description: Describe los eventos de log del tenant que capturan actividades de MFA adaptativo en Auth0.
title: Eventos de log de MFA adaptativo
---

El registro del tenant contiene entradas de eventos de inicio de sesión exitosos y fallidos, incluida información relacionada con las puntuaciones de evaluación de riesgo de <Tooltip tip="MFA adaptativo: autenticación multifactor (MFA) que solo se activa cuando se determina que un intento de inicio de sesión es de baja confianza." cta="Ver glosario" href="/docs/es/glossary?term=Adaptive+MFA">MFA adaptativo</Tooltip>. Estas son las mismas entradas que pueden usarse en objetos `context` de rule. Las entradas de <Tooltip tip="MFA adaptativo: autenticación multifactor (MFA) que solo se activa cuando se determina que un intento de inicio de sesión es de baja confianza." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> adaptativo tienen la siguiente estructura:

```json lines expandable
{
  "date": "2020-06-26T15:12:43.654Z",
  "type": "s",
  "details": {
    "riskAssessment": {
      "confidence": "high",
      "assessments": {
        "UntrustedIP": {
          "confidence": "high",
          "code": "not_found_on_deny_list"
    	},
    	"NewDevice": {
      	  "confidence": "medium",
      	  "code": "partial_match",
          "details": {
            "device": "unknown",
            "useragent": "known",
          }
    	},
    	"ImpossibleTravel": {
      	  "confidence": "high",
      	  "code": "minimal_travel_from_last_login"
    	}
      }
    }
  },
  "description": "Successful login"
}
```

<div id="attack-protection-details">
  ## Detalles de la protección contra ataques
</div>

Además de proporcionar información sobre la ejecución de Rules, Auth0 incluye información de <Tooltip tip="Protección contra ataques: funcionalidades que Auth0 proporciona para detectar y mitigar ataques, incluida la protección contra ataques de fuerza bruta, Suspicious IP Throttling, la detección de contraseñas vulneradas, la detección de bots y MFA adaptativo." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip> en el registro del tenant. La misma estructura de datos que aparece en Rules aparece en el registro del tenant:

```json lines expandable
{
  "date": "2020-06-26T15:12:43.654Z",
  "type": "s",
  "details": {
	"anomalyDetection": {
  	  "confidence": "high",
  	  "assessments": {
    	    "UntrustedIP": {
      	    "confidence": "high",
      	    "reason": "ip not found",
      	    "code": "ip_not_found"
    	    },
    	    "NewDevice": {
      	    "confidence": "medium",
      	    "reason": "match useragent",
      	    "code": "match_useragent"
    	    },
    	    "ImpossibleTravel": {
      	    "confidence": "high",
      	    "reason": "minimal travel",
      	    "code": "minimal_travel_from_last_login"
    	    }
  	  }
	},
  },
  "description": "Successful login"
}
```
