---
description: Aprenda a restablecer el MFA de un usuario en caso de que pierda su dispositivo móvil y no tenga un código de recuperación.
title: Restablecer la autenticación multifactor y los códigos de recuperación de un usuario
---

Si un usuario ha perdido su dispositivo móvil, puede usar su código de recuperación para iniciar sesión. Si no tiene un código de recuperación, necesitará que el administrador del tenant restablezca su <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de username y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA). Esta acción equivale a eliminar el registro de MFA del usuario. Se eliminará la configuración de MFA asociada al usuario, lo que le permitirá configurar el MFA como si fuera un usuario nuevo en su próximo intento de inicio de sesión. Para restablecer el MFA de un administrador en lugar del MFA de un usuario final, comuníquese con el [equipo de soporte de Auth0](/docs/es/troubleshoot/customer-support).

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Dashboard &gt; Gestión de usuarios &gt; Users](https://manage.auth0.com/#/users).
2. Haz clic en el usuario cuyo MFA quieres restablecer.
3. Haz clic en el botón **Actions** en la esquina superior derecha de la pantalla.
4. Selecciona **Restablecer multifactor** en el menú desplegable. Los usuarios con el rol **Admin** también verán un enlace **Restablecer MFA** en la parte inferior de la pestaña **Autenticación multifactor** de la página **User Details** si el usuario ya tiene MFA configurado. Ambos métodos funcionan de la misma manera. Aparecerá una ventana emergente para confirmar tu decisión.
5. Haz clic en **Sí, restablecerlo** para restablecer el MFA del usuario.

La próxima vez que el usuario inicie sesión, tendrá que configurar su MFA igual que un usuario nuevo.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Como administrador, también puedes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para eliminar el registro en MFA de un usuario mediante `DELETE` `/api/v2/users/{id}/authentication-methods`. Si el usuario tiene más de un registro, tendrás que repetir el proceso para cada uno.

<div id="recovery-codes">
  ## Códigos de recuperación
</div>

Con la mayoría de los factores de MFA, el usuario final recibe un código de recuperación al registrarse, que debe anotar y mantener en secreto. Si no tiene su device o, por cualquier otro motivo, temporalmente no puede usar su proceso habitual de MFA, puede iniciar sesión introduciendo este código después de su nombre de usuario y contraseña. Si se utiliza un código de recuperación, en ese momento se proporcionará uno nuevo.

Si un usuario desinstala Guardian y más tarde vuelve a instalarlo, es posible que se le pida que introduzca su código de recuperación. Si ha perdido el código de recuperación, puede realizar una nueva instalación de la aplicación desactivando la restauración automática de su copia de seguridad de Guardian. Para ello, deberá desinstalar Guardian, desactivar temporalmente la restauración automática de copias de seguridad en la configuración de su dispositivo (los pasos pueden variar según el dispositivo) y, a continuación, volver a instalar la aplicación. Después, tendrá que añadir sus cuentas de MFA a la aplicación como si fuera una configuración inicial. Si las copias de seguridad automáticas o la restauración automática no están habilitadas en el dispositivo del usuario, al reinstalar la aplicación no se solicitará un código de recuperación y el usuario tendrá que añadir sus cuentas de MFA como en una configuración inicial.
