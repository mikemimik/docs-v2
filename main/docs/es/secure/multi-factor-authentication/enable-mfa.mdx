Puede habilitar la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) para su tenant en el <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

Cuando habilite MFA, seleccione los factores (como las notificaciones push o los mensajes telefónicos) que desea poner a disposición de los usuarios y defina una política para determinar cuándo deben autenticarse.

<div id="select-factors">
  ## Seleccionar factores
</div>

Auth0 ofrece varios factores que puedes habilitar para que tus usuarios completen la autenticación.

<div id="independent-factors">
  ### Factores independientes
</div>

Debe habilitar y configurar al menos un factor independiente para exigir MFA a los usuarios de su tenant. Si lo desea, puede habilitar varios factores independientes simultáneamente.

Los factores independientes disponibles son:

* [WebAuthn con FIDO Security Keys](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)
* [Contraseña de un solo uso (OTP)](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)
* [Push Notification using Auth0 Guardian](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa)
* [Phone Message](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)
* [Cisco Duo Security](/docs/es/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="dependent-factors">
  ### Factores dependientes
</div>

Puede habilitar uno o varios factores dependientes después de haber habilitado y configurado al menos un factor independiente para los usuarios de su tenant.

Los factores dependientes disponibles son:

* [WebAuthn con biometría FIDO](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)
* [Email](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)
* [Código de recuperación](/docs/es/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="enable-mfa-in-the-auth0-dashboard">
  ## Habilitar MFA en Auth0 Dashboard
</div>

1. Ve a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
2. En la sección **Factors**, habilita y configura los factores que quieras poner a disposición de los usuarios.
3. En la sección **Define policies**, selecciona una política para determinar cuándo se les solicitará MFA a los usuarios.

   * **Never**: no se requiere MFA para ningún inicio de sesión.
   * **Use MFA adaptativo**: se requiere MFA según la evaluación de riesgo de Auth0. Para obtener más información, consulta [Enable MFA adaptativo](/docs/es/secure/multi-factor-authentication/adaptive-mfa).
   * **Always**: se requiere MFA para todos los inicios de sesión.
4. Si eliges la política **Never** o **Always**, aparece la sección **MFA Risk Assessors**. De forma predeterminada, la opción **Enable MFA adaptativo Risk Assessment** está deshabilitada. Activa el interruptor para evaluar y registrar el riesgo de todas las transacciones de inicio de sesión en los registros de tu tenant.
5. En la sección **Additional Settings**, configura las siguientes opciones según sea necesario:

   * **Show Multi-factor Authentication options:** Activa este interruptor para permitir que los usuarios seleccionen factores de autenticación durante la inscripción. Cuando está desactivado, Auth0 presenta automáticamente a los usuarios los factores de autenticación más seguros durante la inscripción.

     <Frame>![Vista del usuario final de Show Multi-factor Authentication options](/docs/images/cdy7uua7fh8z/3Mo3EXUk3ZFK4hZKck0yOO/37268f33f70d1e09032f0db317ac11d3/MFA_Options_-_English.png)</Frame>
   * **Customize MFA Factors using Actions**: Activa este interruptor para personalizar tus MFA flows con [Actions](/docs/es/customize/actions) de `Post Login`. Esto te permite solicitar a los usuarios un factor específico o una secuencia de factores, así como usar información contextual para crear experiencias más personalizadas. Para obtener más información, consulta [Customize MFA Selection for Universal Login](/docs/es/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
6. Haz clic en **Save**.

<div id="customize-mfa-flow">
  ## Personaliza el flujo de MFA
</div>

Puedes usar Auth0 Actions para personalizar el flujo de MFA de los usuarios de tu tenant. Por ejemplo, puedes crear una Action para exigir MFA en circunstancias específicas o para forzar el uso de un factor determinado.

Para obtener más información, consulta [Personaliza el MFA adaptativo](/docs/es/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).
