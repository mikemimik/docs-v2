Puedes habilitar la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) para tu tenant en el <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

Al habilitar MFA, selecciona los factores (como las notificaciones push o los mensajes telefónicos) que quieras poner a disposición de los usuarios y define una política para determinar cuándo deben autenticarse.

<div id="select-factors">
  ## Selecciona factores
</div>

Auth0 admite varios factores que puedes habilitar para que tus usuarios completen la autenticación.

<div id="independent-factors">
  ### Factores independientes
</div>

Debe habilitar y configurar al menos un factor independiente para exigir MFA a los usuarios de su tenant. Si lo desea, puede habilitar varios factores independientes al mismo tiempo.

Los factores independientes disponibles son:

* [WebAuthn con FIDO Security Keys](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)
* [Contraseña de un solo uso (OTP)](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)
* [Push Notification using Auth0 Guardian](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa)
* [Phone Message](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)
* [Cisco Duo Security](/docs/es/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="dependent-factors">
  ### Factores dependientes
</div>

Puedes habilitar uno o más factores dependientes después de haber habilitado y configurado al menos un factor independiente para los usuarios de tu tenant.

Los factores dependientes disponibles son:

* [WebAuthn con biometría FIDO](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)
* [Email](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)
* [Código de recuperación](/docs/es/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="enable-mfa-in-the-auth0-dashboard">
  ## Habilita MFA en el Auth0 Dashboard
</div>

1. Ve a [Dashboard &gt; Seguridad &gt; Autenticación multifactor](https://manage.auth0.com/#/security/mfa).
2. En la sección **Factores**, habilita y configura los factores que quieras poner a disposición de los usuarios.
3. En la sección **Definir políticas**, selecciona una política para determinar cuándo se les solicitará MFA a los usuarios.

   * **Nunca**: no se requiere MFA para ningún inicio de sesión.
   * **Usar MFA adaptativo**: se requiere MFA en función de la evaluación de riesgo de Auth0. Para obtener más información, consulta [Habilitar MFA adaptativo](/docs/es/secure/multi-factor-authentication/adaptive-mfa).
   * **Siempre**: se requiere MFA para todos los inicios de sesión.
4. Si eliges la política **Nunca** o **Siempre**, aparecerá la sección **Evaluadores de riesgo de MFA**. De forma predeterminada, la configuración **Habilitar la evaluación de riesgo de MFA adaptativo** está deshabilitada. Activa el interruptor para evaluar y registrar el riesgo de todas las transacciones de inicio de sesión en los registros de tu tenant.
5. En la sección **Configuración adicional**, configura las siguientes opciones según sea necesario:

   * **Mostrar opciones de Multi-factor Authentication:** Activa este interruptor para permitir que los usuarios seleccionen factores de autenticación durante la inscripción. Cuando está deshabilitado, Auth0 presenta automáticamente a los usuarios los factores de autenticación más seguros durante la inscripción.

     <Frame>![Vista del usuario final de Show Multi-factor Authentication options](/docs/images/cdy7uua7fh8z/3Mo3EXUk3ZFK4hZKck0yOO/37268f33f70d1e09032f0db317ac11d3/MFA_Options_-_English.png)</Frame>
   * **Personalizar factores de MFA con Actions**: Activa este interruptor para personalizar tus flujos de MFA usando las [Actions](/docs/es/customize/actions) de `post-login`. Esto te permite solicitar a los usuarios un factor específico o una secuencia de factores, así como usar información contextual para crear experiencias más personalizadas. Para obtener más información, consulta [Personalizar la selección de MFA para Universal Login](/docs/es/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
6. Haz clic en **Guardar**.

<div id="customize-mfa-flow">
  ## Personaliza el flujo de MFA
</div>

Puedes usar Auth0 Actions para personalizar el flujo de MFA de los usuarios de tu tenant. Por ejemplo, puedes crear una Action para exigir MFA en circunstancias específicas o para forzar el uso de un factor concreto.

Para obtener más información, consulta [MFA adaptativo](/docs/es/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).
