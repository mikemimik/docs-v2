title: Cookies de Authentication API
---

Auth0 Authentication API utiliza un conjunto de cookies HTTP para habilitar las funciones de [inicio de sesión único (SSO)](/docs/es/authenticate/single-sign-on), [autenticación multifactor (MFA)](/docs/es/secure/multi-factor-authentication) y [protección contra ataques](/docs/es/secure/attack-protection). En la siguiente tabla se muestran algunas de las cookies de las que depende Authentication API y se describe su propósito:

| **Cookie**          | **Funcionalidad**         | **Propósito**                                                                                                   |
| ------------------- | ------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `auth0`             | Inicio de sesión único    | Se utiliza para implementar la [capa de sesión de Auth0](/docs/es/manage-users/sessions/session-layers).           |
| `auth0_compat`      | Inicio de sesión único    | Cookie de respaldo para el inicio de sesión único en navegadores que no admiten el atributo `sameSite=None`.    |
| `auth0-mf`          | Autenticación multifactor | Se utiliza para establecer el nivel de confianza de un `device` determinado.                                    |
| `auth0-mf_compat`   | Autenticación multifactor | Cookie de respaldo para la autenticación multifactor en navegadores que no admiten el atributo `sameSite=None`. |
| `a0_users:sess`     | Classic Login             | Se utiliza para la protección contra CSRF en los flujos de Classic Login.                                       |
| `a0_users:sess.sig` | Classic Login             | Se utiliza para la protección contra CSRF en los flujos de Classic Login.                                       |
| `did`               | Protección contra ataques | Identificación del dispositivo para la protección contra ataques.                                               |
| `did_compat`        | Protección contra ataques | Cookie de respaldo para la detección de anomalías en navegadores que no admiten el atributo `sameSite=None`.    |

<Warning>
  Auth0 no admite escenarios en los que las cookies de Authentication indicadas se modifiquen de cualquier forma, incluida la adición, modificación o eliminación de atributos de las cookies, ya sea mediante navegadores no estándar, complementos del navegador o proxies HTTP.
</Warning>

<div id="cookies-and-custom-domains">
  ## Cookies y dominios personalizados
</div>

Si usas [dominios personalizados](/docs/es/customize/custom-domains), las cookies de la Authentication API se envían al nombre de host personalizado, o CNAME, que configuraste en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. El atributo de dominio de cada cookie, que especifica para qué dominio es válida la cookie, se define en el encabezado de solicitud de la cookie y debe coincidir con el atributo de dominio.

Si no se especifica ningún dominio, el atributo de dominio toma de forma predeterminada el host de la solicitud. Si utilizas la especificación [HTTP State Management Mechanism](https://datatracker.ietf.org/doc/html/rfc2109#section-2) del IETF para establecer cookies en el dominio principal, la cookie se compartirá con todos sus subdominios.

Por ejemplo, configuras tu CNAME como `login.example_domain.com`, un subdominio de `example_domain.com`. Alojas otras aplicaciones bajo el dominio principal, como `app1.example_domain.com` y `app2.example_domain.com`. Cuando los usuarios visitan `login.example_domain.com`, las cookies de `app1.example_domain.com` y `app2.example_domain.com` pueden enviarse junto con las solicitudes a la Authentication API de Auth0.

Para proteger nuestra plataforma, y dado que estas cookies pueden alcanzar un tamaño considerable y compartirse con otros subdominios, Auth0 puede rechazar solicitudes con encabezados excesivamente grandes (varios kilobytes). Las aplicaciones deben diseñarse de modo que no envíen cookies excesivamente grandes a la Auth0 Authentication API. Para obtener más información sobre el comportamiento de las cookies con <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domains">dominios personalizados</Tooltip>, consulta [Sending Cookies to the Origin Server](https://datatracker.ietf.org/doc/html/rfc2109#section-4.3.4).
