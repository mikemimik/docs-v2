  Connected Accounts permite vincular un único perfil de usuario de Auth0 con varias cuentas externas. Cuando habilitas Connected Accounts para un proveedor externo compatible, Auth0 agrega automáticamente la cuenta de ese proveedor al perfil del usuario después de que este inicie sesión correctamente. Para obtener más información, consulta [Connected Accounts for Token Vault](/docs/es/secure/tokens/token-vault/connected-accounts-for-token-vault).

Auth0 admite la vinculación de cuentas de usuario de varios <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip>. Esto permite que un usuario se autentique con cualquiera de sus cuentas y que tu aplicación lo siga reconociendo y asociando al mismo perfil de usuario.

<Card title="La disponibilidad varía según el plan de Auth0">
  Tanto la implementación concreta del inicio de sesión como tu plan de Auth0 o acuerdo personalizado influyen en si esta funcionalidad está disponible. Para obtener más información, consulta [Pricing](https://auth0.com/pricing).
</Card>

De forma predeterminada, Auth0 trata todas las identidades como independientes. Por ejemplo, si un usuario inicia sesión primero en la base de datos de Auth0 y luego a través de Google o Facebook, Auth0 interpretaría esos dos intentos como dos usuarios distintos. Puedes implementar una funcionalidad para permitir que un usuario vincule cuentas de forma explícita. En este escenario, el usuario iniciaría sesión con un proveedor inicial, quizá Google. Tu aplicación proporcionaría un enlace o botón para que pueda vincular otra cuenta con la primera. El usuario haría clic en ese enlace o botón, y tu aplicación realizaría una llamada para que, cuando el usuario inicie sesión con el segundo proveedor, la segunda cuenta quede vinculada a la primera.

<div id="advantages-of-linking-accounts">
  ## Ventajas de vincular cuentas
</div>

* Permite a los usuarios iniciar sesión con cualquier proveedor de identidad sin crear un perfil independiente para cada uno
* Permite a los usuarios registrados usar un nuevo inicio de sesión social o <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip> y seguir usando su perfil existente
* Permite a los usuarios que se registraron mediante un inicio de sesión passwordless vincularlo a una cuenta con un perfil más completo
* Permite que tus aplicaciones recuperen datos del perfil del usuario almacenados en varias conexiones

<div id="precautions">
  ## Precauciones
</div>

Vincular cuentas de forma insegura puede permitir que personas malintencionadas accedan a cuentas de usuario legítimas. Tenga en cuenta lo siguiente:

Tanto en la vinculación manual como en la automática de cuentas, su tenant debe solicitar autenticación para **ambas** cuentas antes de que se produzca la vinculación.

Además, **toda** vinculación manual de cuentas debe solicitar al usuario que introduzca sus credenciales.

<div id="how-it-works">
  ## Cómo funciona
</div>

El proceso de vinculación de cuentas combina dos perfiles de usuario existentes en uno solo. Al vincular cuentas, se deben especificar una **cuenta principal** y una **cuenta secundaria**.

En el ejemplo siguiente puedes ver cómo quedará el perfil vinculado resultante para las cuentas principal y secundaria de muestra.

Ten en cuenta que:

* El `user_id` y todas las demás propiedades principales del perfil siguen siendo los de la identidad principal
* La primera identidad del arreglo `user.identities` es la identidad principal
* La cuenta secundaria ahora queda integrada en el arreglo `user.identities` del perfil principal
* Los atributos de la cuenta secundaria se colocan dentro del campo `profileData` de la identidad correspondiente dentro del arreglo
* El `user_metadata` y el `app_metadata` de la cuenta principal no cambian
* El `user_metadata` y el `app_metadata` de la cuenta secundaria se descartan
* No hay una combinación automática de perfiles de usuario con identidades asociadas
* La cuenta secundaria se elimina de la lista de usuarios
* Si eliminas la cuenta principal, la cuenta secundaria también se elimina

<div id="metadata-merge">
  ### Fusión de metadatos
</div>

Los metadatos no se fusionan automáticamente durante la vinculación de cuentas. Si quieres fusionarlos, tienes que hacerlo manualmente mediante el endpoint [`/users/patch_users_by_id`](https://auth0.com/docs/api/v2#!/Users/patch_users_by_id) de la API de administración.

El [SDK de Auth0 para Node.js APIv2](https://github.com/auth0/node-auth0/) también está disponible.

Para obtener más información, consulta [Cómo funcionan los metadatos en los perfiles de usuario](/docs/es/manage-users/user-accounts/metadata)

<div id="scenarios">
  ## Escenarios
</div>

Hay dos formas diferentes de implementar la vinculación de cuentas:

* Vinculación de cuentas iniciada por el usuario: permite que tus usuarios vinculen sus cuentas desde una pantalla de administración de tu aplicación.
* Vinculación de cuentas sugerida: identifica cuentas con la misma dirección de correo electrónico y pide al usuario en tu aplicación que las vincule.

<div id="user-initiated-account-linking">
  ### Vinculación de cuentas iniciada por el usuario
</div>

Normalmente, la vinculación de cuentas la inicia un usuario autenticado. Tu aplicación debe proporcionar la interfaz de usuario, por ejemplo, un botón de **Vincular cuentas** en la página del perfil del usuario.

Para obtener más información sobre cómo implementar la vinculación de cuentas iniciada por el usuario en una Single Page Application, consulta [Vinculación de cuentas del lado del cliente](/docs/es/manage-users/user-accounts/user-account-linking/user-initiated-account-linking-client-side-implementation). Para obtener más información sobre cómo implementar la vinculación de cuentas iniciada por el usuario en una Regular Web Application, consulta [Vinculación de cuentas del lado del servidor](/docs/es/manage-users/user-accounts/user-account-linking/suggested-account-linking-server-side-implementation).

<div id="suggested-account-linking">
  ### vinculación de cuentas sugerida
</div>

Puedes detectar cuentas con el mismo correo electrónico y pedir a los usuarios que las vinculen. Por ejemplo, un usuario puede crear una cuenta de Google con `user@gmail.com` y luego iniciar sesión con Facebook con una cuenta asociada al mismo correo electrónico.

Si eso ocurre, puedes mostrar a los usuarios la lista de cuentas disponibles para que las vinculen, autenticándose primero con la cuenta a la que la van a vincular.

También puedes usar la [extensión Account Link](/docs/es/customize/extensions/account-link-extension) para conseguir el mismo resultado.

<Warning>
  Anteriormente, en algunos casos, se podían usar tokens de ID para vincular y desvincular cuentas de usuario. Esta funcionalidad está quedando obsoleta. Tendrás que usar tokens de acceso en todos los casos.
</Warning>
