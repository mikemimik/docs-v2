<div id="auth0-configuration">
  ## Configuración de Auth0
</div>

1. Crea una cuenta de Auth0 y ve al [Dashboard](https://manage.auth0.com/#).
2. Ve a [Dashboard &gt; Applications](https://manage.auth0.com/#/applications) y haz clic en **+ Create Application**. Elige la opción `Single-Page Application` y ve a **Settings**. Toma nota del **ID de cliente** y del **Domain**.
3. Agrega la `URL de devolución de llamada` tanto en **Allowed Callback URLs** como en **Allowed Origins (CORS)**. Usa la URL de tu página de Unbounce. Por ejemplo: `http://unbouncepages.com/changeit`.
4. Ve a [Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social) y habilita los proveedores sociales que quieras admitir.

<div id="unbounce-configuration">
  ## Configuración de Unbounce
</div>

1. Crea un nuevo elemento de la interfaz, como un botón, que inicie el login con el proveedor. Anota el ID del elemento en **Propiedades &gt; Metadatos del elemento**.
2. Agrega un nuevo JavaScript a tu página de destino de Unbounce, selecciona `Before Body End Tag` en `Placement` y agrega este código. Asegúrate también de marcar jQuery como dependencia.

export const codeExample = `<script src="https://cdn.auth0.com/js/auth0/9.11/auth0.min.js"></script>
<script type="application/javascript">
  var webAuth = new auth0.WebAuth({
    domain:       '{yourDomain}',
    clientID:     '{yourClientId}',
    audience: 'https://{yourClientId}/userinfo'
    redirectUri:  '{yourUnbouncePageUrl}', // p. ej., http://unbouncepages.com/changeit
    scope: 'openid profile email',
    responseType: 'token id_token',
  });
</script>`;

Debes usar el <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip> y el dominio de la aplicación que acabas de configurar.

A continuación, necesitas una forma de pasar a Unbounce la información que proviene de los proveedores sociales:

1. Crea un Form y agrega `Hidden fields` para cada campo. Por ejemplo, los campos `name` y `email`.
2. Vuelve al editor de JavaScript de Unbounce.
3. Agrega un controlador de clic a cada botón para activar la autenticación social.

   1. Reemplaza el ID del botón que anotaste antes y el [nombre de la conexión](/docs/es/authenticate/identity-providers/locate-the-connection-id). Por ejemplo, para Google usarías `google-oauth2` y para LinkedIn, `linkedin`.
   2. Asegúrate de reemplazar correctamente los ID. En lugar de `#name` y `#email`, debes poner el ID de los campos del formulario correspondientes (puedes verlos mientras editas el formulario, en `Field Name and ID`).

```js lines
$('#{buttonId}').bind('click', function() { 
  webAuth.authorize({
    connection: '{yourConnectionName}'
  });
});

// Una vez completada la autenticación, el método parseHash analiza el fragmento hash de la URL para extraer el resultado de una respuesta de autenticación de Auth0.

webAuth.parseHash({ hash: window.location.hash }, function(err, authResult) { 
  if (err) { 
    return console.log(err); 
  }

  if (authResult != null && authResult.accessToken != null) {
    webAuth.client.userInfo(authResult.accessToken, function(err, user) {
      $('#name').val(user.name); 
      $('#email').val(user.email); 
    }); 
  } 

});
```

Ahora podrás ver la información proporcionada por el <Tooltip tip="proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip> en la sección `Leads` de tu panel de administración de Unbounce una vez que el usuario envíe el formulario.
