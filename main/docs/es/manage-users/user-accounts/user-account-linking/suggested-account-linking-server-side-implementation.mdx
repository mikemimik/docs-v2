Auth0 permite vincular cuentas de usuario de distintos <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip>. Puedes usar código del lado del servidor para vincular cuentas en una aplicación web tradicional, involucrando al usuario y pidiéndole permiso antes de continuar. Tu código autenticará a los usuarios y buscará e identificará usuarios por sus direcciones de correo electrónico. Luego, tu aplicación le pedirá al usuario que vincule sus cuentas autenticándose con las credenciales de la cuenta de destino y, posteriormente, vinculará las cuentas.

Puedes encontrar el código fuente completo de esta aplicación de ejemplo [en GitHub](https://github.com/auth0-samples/auth0-link-accounts-sample/tree/master/RegularWebApp).

1. Haz que el usuario inicie sesión en tu aplicación.<br />

   El usuario se autentica en su aplicación mediante [Universal Login](/docs/es/authenticate/login/auth0-universal-login). Para obtener más información, consulte el [Quickstart para aplicación web tradicional](/docs/es/quickstart/webapp), donde se solicita un token para la audiencia de Auth0 Management API (audience=`https://{yourDomain}/api/v2/`).

2. Busca usuarios con la misma dirección de correo electrónico.<br />

   Puedes obtener el perfil del usuario y la lista de usuarios con la misma dirección de correo electrónico verificada.

   ```javascript lines expandable
   router.get("/", async (req, res) => {
     const { sub, email_verified } = req.openid.user;
     //obtener el perfil de usuario que contiene las propiedades user_metadata y app_metadata
     try {
       let getUsersWithSameVerifiedEmail = [];
       const getUserProfile = auth0Client.getUser(sub);
       if (email_verified)
         // la vinculación de cuentas solo se ofrece con correo electrónico verificado
         getUsersWithSameVerifiedEmail = auth0Client.getUsersWithSameVerifiedEmail(
           req.openid.user
         );

       const [user, suggestedUsers] = await Promise.all([
         getUserProfile,
         getUsersWithSameVerifiedEmail,
       ]);

       const flashError = clear(req);
       res.render("user", {
         user,
         suggestedUsers,
         wrongAccountError: flashError && flashError === Errors.WrongAccount,
       });
     } catch (err) {
       debug("GET /user[s] failed: %o", err);
       res.render("error", err);
     }
   });
   ```

   Para obtener una lista de todos los registros de usuarios con la misma dirección de correo electrónico, su aplicación llama al [endpoint Get Users By Email de Auth0 Management API](https://auth0.com/docs/api/v2#!/users-by-email/get_users_by_email) con un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el `scope` `read:users`.

   ```javascript lines
   const request = require('request');
   class Auth0Client {
   ...
   async getUsersWithSameVerifiedEmail({ sub, email }) {
     return await this.request({
       url: `${process.env.ISSUER_BASE_URL}/api/v2/users-by-email?email=${email}`,
     });
   }
   ```

3. Pide al usuario que vincule cuentas.

   1. Si Auth0 devuelve uno o más registros con direcciones de correo electrónico que coinciden, el usuario verá la lista junto con el siguiente mensaje para que vincule las cuentas.
   2. Si el usuario quiere vincular una cuenta determinada, puede hacer clic en **Vincular** junto a la cuenta correspondiente.

      <Frame>![Aplicación de ejemplo con página de vinculación de cuentas del lado del servidor](/docs/images/cdy7uua7fh8z/7unQjUtxTZUc4tRFOxhaP3/050fb853d2b3be37795ce88b8a60ffed/account-linking-webapp-small__1_.png)</Frame>

4. Cuando el usuario haga clic en **Vincular**, tu aplicación le pedirá que se autentique con la cuenta de destino y luego realizará la vinculación de cuentas.

   <Warning>
     Para conservar y fusionar los `user_metadata` de la cuenta secundaria, debes recuperarlos y fusionarlos con los metadatos de la cuenta principal antes de llamar al endpoint de la API. Una vez vinculadas las cuentas, se descartan los metadatos de la cuenta secundaria.

     Cuando inicias la vinculación de cuentas, puedes seleccionar qué identidad se usará como cuenta principal y cuál como secundaria. Esta elección dependerá del conjunto de atributos que quieras conservar en el perfil principal.
   </Warning>

   El siguiente fragmento de código muestra cómo verificar y combinar metadatos:

   ```javascript lines expandable
   async function accountLink(req, res, next) {
   const {
     linking: { targetUserId },
   } = req.appSession;
   const { sub: authenticatedTargetUserId } = req.openidTokens.claims();
   if (authenticatedTargetUserId !== targetUserId) {
     debug(
       "Skipping account linking as the authenticated user(%s)  is different than target linking user (%s)",
       authenticatedTargetUserId,
       targetUserId
     );
     set(req, Errors.WrongAccount);
     return next();
   }

   debug(
     "User %s succesfully authenticated. Account linking with %s... ",
     authenticatedTargetUserId,
     targetUserId
   );
   const { id_token: targetIdToken } = req.openidTokens;
   const { sub: primaryUserId } = req.appSession.claims;

   try {
     await mergeMetadata(primaryUserId, authenticatedTargetUserId);
     await auth0Client.linkAccounts(primaryUserId, targetIdToken);
     debug("Accounts linked.");
   } catch (err) {
     debug("Linking failed %o", err);
   } finally {
     next();
   }
   }
   ```

5. Tu aplicación llama al [endpoint Link a User Account de la Auth0 Management API](https://auth0.com/docs/api/v2#!/Users/post_identities) usando un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:users`.

<div id="metadata-merge-example">
  ## Ejemplo de fusión de metadata
</div>

El siguiente ejemplo muestra explícitamente cómo la `user_metadata` y la `app_metadata` de la cuenta secundaria se fusionan en la cuenta principal mediante el [SDK de Auth0 para API V2 de Node.js](https://github.com/auth0/node-auth0).

```javascript lines expandable
/*
 * Combina recursivamente user_metadata y app_metadata del usuario secundario en el usuario primario.
 * Los datos del usuario primario tienen precedencia.
 * Los campos de tipo arreglo se unen.
 */
async function mergeMetadata(primaryUserId, secondaryUserId) {
  // cargar ambos usuarios con sus metadatos.
  const [primaryUser, secondaryUser] = await Promise.all(
    [primaryUserId, secondaryUserId].map((uid) => auth0Client.getUser(uid))
  );

  const customizerCallback = function (objectValue, sourceValue) {
    if (_.isArray(objectValue)) {
      return sourceValue.concat(objectValue);
    }
  };
  const mergedUserMetadata = _.merge(
    {},
    secondaryUser.user_metadata,
    primaryUser.user_metadata,
    customizerCallback
  );
  const mergedAppMetadata = _.merge(
    {},
    secondaryUser.app_metadata,
    primaryUser.app_metadata,
    customizerCallback
  );
  await auth0Client.updateUser(primaryUserId, {
    user_metadata: mergedUserMetadata,
    app_metadata: mergedAppMetadata,
  });
}
```
