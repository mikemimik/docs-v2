Auth0 normaliza las propiedades de perfil comunes en el perfil de usuario; esto incluye los campos `name`, `picture` y otros. El campo `picture` se rellena con la foto de perfil del proveedor social o con la imagen de Gravatar asociada a la dirección de correo electrónico del usuario.

De forma predeterminada, todos los usuarios de base de datos tendrán una imagen de marcador de posición con sus iniciales. Cuando autenticas al usuario, se hace referencia a este campo `picture` como `user.picture`.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

El atributo `user.picture` no se puede editar directamente cuando lo proporcionan <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0, como Google, Facebook o X. Para editar este atributo, debes configurar la sincronización de tu conexión con Auth0 para que los atributos del usuario se actualicen desde el proveedor de identidad solo al crear el perfil del usuario. Para obtener más información, consulta [Configure Identity Provider Connection for User Profile Updates](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0). Los atributos raíz estarán disponibles para editarlos individualmente o mediante importación masiva con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulta [Bulk User Imports](/docs/es/manage-users/user-migration/bulk-user-imports).

Como alternativa, puedes usar metadatos para almacenar el atributo `picture` de los usuarios. Por ejemplo, si tu aplicación ofrece una forma de subir fotos de perfil, una vez cargada la foto, puedes establecer la URL en `user.user_metadata.picture`:

<div id="use-actions">
  ## Usar Actions
</div>

Para asegurarnos de que la foto de `user_metadata` se incluya en el <Tooltip tip="token de ID: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip>, tendremos que [crear una nueva Action](/docs/es/customize/actions/write-your-first-action) para comprobar si el atributo `event.user.user_metadata.picture` está presente y, de ser así, sustituir el atributo `user.picture` por ese valor. Esto garantiza que la foto de `user_metadata` se incluya en el claim `picture` del token de ID.

1. Ve a [Auth0 Dashboard &gt; Actions &gt; Library](https://manage.auth0.com/#/actions/library) y selecciona **Build Custom**.

2. Introduce un **Name** descriptivo para tu Action (por ejemplo, `Change user pictures`), selecciona el trigger `Login / Post Login` porque vas a añadir la Action al flujo de Login y, a continuación, selecciona **Create**.

3. Localiza el Actions Code Editor, copia en él el siguiente código JavaScript y selecciona **Save Draft** para guardar los cambios:

   ```js lines
   exports.onExecutePostLogin = async (event, api) => {
     const { picture } = event.user.user_metadata;
     if (picture) {
       // Return the persisted user_metadata.picture in the ID token
       api.idToken.setCustomClaim("picture", picture)
     }
   };
   ```

4. En la barra lateral del Actions Code Editor, selecciona Test (icono de reproducción) y luego selecciona **Run** para [probar tu código](/docs/es/customize/actions/test-actions).

5. Cuando tengas todo listo para activar la Action, selecciona **Deploy**.

Por último, añade la Action que creaste al [Login Flow](https://manage.auth0.com/#/actions/flows/login/). Para aprender a asociar Actions a Flows, consulta la sección &quot;Attach the Action to a flow&quot; de [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

<div id="change-default-picture-for-all-users">
  ## Cambiar la foto predeterminada para todos los usuarios
</div>

Para cambiar la foto predeterminada de todos los usuarios que no hayan configurado una foto de perfil, puedes usar una Action. Por ejemplo:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  const DEFAULT_PROFILE_IMAGE = '{yourDefaultImageUrl}';
  api.idToken.setCustomClaim("picture", {defaultProfileImage});
};
```

En esta Action, la foto personalizada se incorpora al token de ID y sobrescribe cualquier propiedad `picture` que pueda provenir del inicio de sesión con un proveedor de identidad externo, como Google.

<div id="limitations">
  ## Limitaciones
</div>

El Auth0 data store tiene capacidad limitada y, para evitar que los datos de la aplicación superen esos límites, le recomendamos usar una base de datos externa para almacenar las fotos de los usuarios. Esto le permite mantener reducido su Auth0 data store y usar una base de datos externa más eficiente para guardar los datos adicionales. Para obtener más información, consulte [Almacenamiento de datos de usuario](/docs/es/secure/security-guidance/data-security/user-data-storage).
