---
description: Conozca los atributos raíz disponibles para el perfil de usuario normalizado de Auth0 y los métodos para actualizarlos.
title: Atributos raíz del perfil de usuario
---

El perfil de usuario normalizado de Auth0 contiene varios atributos raíz (atributos almacenados en el primer nivel, o nivel raíz, del objeto), algunos de los cuales se pueden actualizar. Para obtener más información sobre la estructura del perfil de usuario y sus atributos, consulte [Estructura del perfil de usuario.](/docs/es/manage-users/user-accounts/user-profiles/user-profile-structure)

Los métodos para actualizar los atributos raíz varían según el tipo de conexión.

<div id="auth0-as-the-identity-provider">
  ## Auth0 como proveedor de identidad
</div>

Cuando Auth0 es el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP), los suscriptores pueden:

* Establecer atributos raíz [cuando un usuario se registra](https://auth0.com/docs/api/management/guides/users/set-root-attributes-user-signup) (a través de la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> o mediante el registro público)
  o [durante la importación](https://auth0.com/docs/api/management/guides/users/set-root-attributes-user-import)
* [Actualizar los atributos raíz de forma individual](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) mediante la Management API

Auth0 es el IdP para los siguientes tipos de conexión:

* [Conexiones de base de datos estándar](/docs/es/authenticate/database-connections)
* [Conexiones de base de datos personalizadas](/docs/es/authenticate/database-connections/custom-db) con modo de importación
* [Conexiones passwordless](/docs/es/authenticate/passwordless)

<div id="upstream-identity-providers">
  ## Proveedores de identidad de origen
</div>

Cuando se usa un IdP de origen (como Google o Facebook), los clientes tienen dos opciones:

* El IdP de origen establece los atributos raíz cuando se crea a los usuarios por primera vez y luego
  los actualiza automáticamente en cada inicio de sesión posterior. Este es el comportamiento predeterminado.
* El IdP de origen establece los atributos raíz solo al crear al usuario y no
  los actualiza en los inicios de sesión posteriores, lo que permite a los clientes [actualizar los atributos raíz individualmente](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) mediante la Management API. Para habilitar esto, deberá [configurar la sincronización de su conexión con Auth0](https://auth0.com/docs/api/management/guides/connections/configure-connection-sync).

Los proveedores de identidad de origen admiten los siguientes tipos de conexión:

* [conexiones sociales](/docs/es/authenticate/identity-providers#social)
* [conexiones empresariales](/docs/es/authenticate/identity-providers#enterprise)
* [conexiones de identidad legal](/docs/es/authenticate/identity-providers#legal-identities)
