description: Comprende cómo Auth0 normaliza las propiedades comunes del usuario en el perfil de usuario.
title: Perfiles de usuario normalizados
---

El perfil de usuario normalizado es una forma propia de Auth0 de estandarizar y almacenar información relacionada con el usuario. Dado que cada <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP) proporciona un conjunto distinto de información sobre un usuario, Auth0 normaliza las propiedades comunes del perfil en una representación del usuario independiente del protocolo al almacenar claims relacionados con el usuario. Por ejemplo, `family_name` en el perfil de usuario normalizado contiene datos que un IdP puede devolver como `surname` o `last_name`.

Por ello, los claims de Auth0 incluidos en el perfil de usuario normalizado difieren del conjunto estándar de claims que pueden devolverse en los <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Tokens">tokens de ID</Tooltip> desde el [endpoint `oauth/token` de Authentication API](https://auth0.com/docs/api/authentication#get-token) o en la respuesta del endpoint [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (ambos siguen los requisitos detallados en la especificación de OIDC).

Para conocer los atributos incluidos en el perfil de usuario normalizado y entender cómo puedes trabajar con ellos, lee [Estructura del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/user-profile-structure).
