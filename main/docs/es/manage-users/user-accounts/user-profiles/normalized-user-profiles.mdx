description: Comprende cómo Auth0 normaliza las propiedades comunes de los usuarios en el perfil de usuario.
title: Perfiles de usuario normalizados
---

El perfil de usuario normalizado es una forma propia de Auth0 de estandarizar y almacenar información relacionada con el usuario. Como cada <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> (IdP) proporciona un conjunto distinto de información sobre un usuario, Auth0 normaliza las propiedades comunes del perfil en una representación del usuario independiente del protocolo al almacenar las reclamaciones relacionadas con el usuario. Por ejemplo, `family_name` en el perfil de usuario normalizado contiene datos que un IdP puede devolver como `surname` o `last_name`.

Por ello, las reclamaciones de Auth0 incluidas en el perfil de usuario normalizado difieren del conjunto estándar de reclamaciones que pueden devolverse en los <Tooltip tip="ID Token: credencial destinada al propio cliente, en lugar de al acceso a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Tokens">ID Tokens</Tooltip> desde el [endpoint `oauth/token` de la API de autenticación](https://auth0.com/docs/api/authentication#get-token) o en la respuesta del endpoint [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (ambos siguen los requisitos detallados en la especificación OIDC).

Para obtener más información sobre los atributos incluidos en el perfil de usuario normalizado y entender cómo puedes trabajar con ellos, consulta [Estructura del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/user-profile-structure).
