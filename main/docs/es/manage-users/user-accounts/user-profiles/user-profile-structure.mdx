El perfil de usuario normalizado de Auth0 consta de varios componentes:

* **Details**: Objeto principal del perfil de usuario, que contiene información básica, como el nombre, el correo electrónico y la marca de tiempo del último login del usuario, en atributos predefinidos. Este objeto también puede contener información de la conexión de origen del usuario. La mayoría de los atributos del usuario son atributos raíz (atributos almacenados en el primer nivel, o nivel raíz, del objeto `user`), y algunos de ellos se pueden editar.
* **Metadata**: Dos subobjetos que funcionan como almacenamiento secundario para guardar información adicional del usuario en atributos personalizables: `user_metadata` y `app_metadata`. Para obtener más información, incluido cuándo usar `app_metadata` y `user_metadata`, consulta [Cómo funcionan los metadatos en los perfiles de usuario](/docs/es/manage-users/user-accounts/metadata).

<div id="user-profile-attributes">
  ## Atributos del perfil de usuario
</div>

Los siguientes atributos están disponibles en el perfil de usuario. Muchos son atributos raíz (atributos almacenados en el primer nivel, o nivel raíz, del objeto `user`), y algunos se pueden actualizar, importar y exportar, como se indica a continuación.

Si hay campos de usuario que no deberían almacenarse en Auth0 por motivos de privacidad, puede agregar a la Deny List los atributos que no desea que se conserven en las bases de datos de Auth0. Para obtener más información, consulte [Agregar atributos de usuario a la Deny List](/docs/es/secure/security-guidance/data-security/denylist).

De forma predeterminada, los atributos del perfil de usuario proporcionados por <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente, ya que se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Para obtener más información sobre los atributos de los proveedores de identidad, consulte [Configurar la conexión del proveedor de identidad para las actualizaciones del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Para poder editar los atributos raíz `name`, `nickname`, `given_name`, `family_name` o `picture` en el perfil de usuario normalizado, debe [configurar la sincronización de su conexión con Auth0](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo cuando se cree el perfil de usuario. Estos atributos raíz estarán disponibles para editarse individualmente o mediante importaciones masivas

  ### Validación del perfil de usuario
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Todos los caracteres se convierten a minúsculas al guardarse en el perfil de usuario.

  Esto se aplica a todas las conexiones de base de datos de Auth0. Cuando Auth0 DB no es la fuente autorizada (por ejemplo, una base de datos personalizada con el modo de importación desactivado), así como en otros tipos de conexiones sociales o empresariales en las que Auth0 no es la fuente autorizada, el correo electrónico se conserva tal como está y no se convierte a minúsculas.

Hay otros tres campos que técnicamente no forman parte del perfil de usuario, pero pueden ser de interés al importar usuarios:

* `password_hash` (text): Contraseña con hash para la conexión del usuario. Cuando se crean usuarios, Auth0 usa [bcrypt](https://auth0.com/blog/hashing-in-action-understanding-bcrypt/) para proteger la contraseña. Importar contraseñas con hash compatibles permite que los usuarios conserven sus contraseñas, lo que proporciona una experiencia más fluida. Las contraseñas compatibles deben estar hasheadas con bcrypt {"$2a$"} o {"$2b$"} y tener 10 saltRounds. Ten en cuenta que solo puedes proporcionar este campo cuando el usuario se importa por primera vez; no puedes actualizarlo después.
* `custom_password_hash` (object): Contraseña con hash para la conexión del usuario cuando se crea con un algoritmo alternativo. Se usa en lugar del campo `password_hash`. Durante el proceso de importación masiva, puedes actualizar `custom_password_hash` si el usuario no inició sesión con el `custom_password_hash` importado inicialmente.
