title: Administrar Users usando la Management API
---

Además de usar el Dashboard, puedes recuperar, crear, actualizar o eliminar usuarios mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Users/get_users). Si quieres llamar directamente a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, primero tendrás que generar el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> adecuado. Para obtener más información, lee [Tokens de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens). Como alternativa, puedes usar un SDK para implementar la funcionalidad que necesitas para llamar a la Management API desde tu aplicación. Para consultar una lista de los SDKs disponibles, lee [la sección de SDKs de nuestra matriz de soporte](/docs/es/troubleshoot/customer-support/product-support-matrix).

Para aprender a administrar el acceso al <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> de los miembros de tu equipo, lee [Administrar el acceso al Dashboard](/docs/es/get-started/manage-dashboard-access).

<div id="permissions">
  ## Permisos
</div>

Usa los siguientes permisos para obtener un token de acceso para las aplicaciones de máquina a máquina. Para obtener más información, consulta [Registrar aplicaciones de máquina a máquina.](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps)

| Permisos                    | Descripción                                                                                                                                                                                                                           |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `read:users`                | Las aplicaciones pueden buscar y recuperar cualquier registro de usuario almacenado en un tenant de Auth0. Esto incluye la lectura a través de los endpoints `/api/v2/users`, `/api/v2/users-by-email` y `/api/v2/jobs/users-export`. |
| `create:users`              | Las aplicaciones pueden crear usuarios en cualquier conexión de base de datos o passwordless de un tenant de Auth0. Esto incluye la creación a través de los endpoints `/api/v2/users` y `/api/v2/jobs/users-import`.                 |
| `update:users`              | Las aplicaciones pueden actualizar cualquier usuario almacenado en un tenant de Auth0. Esto incluye la actualización a través del endpoint `/api/v2/users`.                                                                           |
| `update:users_app_metadata` | Las aplicaciones solo pueden actualizar el atributo `user.app_metadata` de cualquier usuario almacenado en un tenant de Auth0. Esto incluye la actualización a través de `/api/v2/users endpoint`.                                    |
| `delete:users`              | Las aplicaciones pueden eliminar cualquier usuario almacenado en un tenant de Auth0. Esto incluye la eliminación a través del endpoint `/api/v2/users`.                                                                               |

<div id="limitations">
  ## Limitaciones
</div>

De forma predeterminada, los atributos del perfil de usuario proporcionados por <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente porque se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Para obtener más información sobre los atributos de los proveedores de identidad, consulta [Configure Identity Provider Connection for User Profile Updates](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Para poder editar los atributos raíz `name`, `nickname`, `given_name`, `family_name` o `picture` del perfil de usuario normalizado, debes [configurar la sincronización de tu conexión con Auth0](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo cuando se cree el perfil de usuario. Estos atributos raíz estarán disponibles para editarlos individualmente o mediante importaciones masivas.

Puedes establecer contraseñas mediante los endpoints `create` o `update`, pero, por motivos de seguridad, las contraseñas no se pueden ver con los comandos `get` o `list user`. En el lado derecho de API Explorer se muestran indicaciones sobre los atributos del perfil de usuario que se pueden ver o modificar en cada solicitud.

<div id="endpoints">
  ## Endpoints
</div>

* Usa el endpoint [`/users`](https://auth0.com/docs/api/management/v2/users/get-users) para [obtener información sobre todos los usuarios](https://auth0.com/docs/api/management/v2/users/get-users). También puedes incluir criterios de búsqueda para encontrar usuarios específicos.
* Usa [`/user_id`](https://auth0.com/docs/api/management/v2/users/get-users-by-id) para [obtener información sobre un usuario](https://auth0.com/docs/api/management/v2/users/get-users-by-id) en función del `user_id`. El `user_id` es un identificador interno que consta de un nombre de conexión y un identificador único del usuario. El `user_id` es distinto del <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip>.
* El endpoint [`/userinfo`](https://auth0.com/docs/api/authentication/reference#get-user-info) recibe como entrada el [token de acceso de Auth0 y devuelve información del perfil de usuario](https://auth0.com/docs/api/authentication/reference#get-user-info). Este endpoint incluirá los resultados de cualquier regla que pueda haber alterado el perfil de usuario durante la transacción de autenticación, pero el perfil de usuario resultante no se filtrará según ningún [Scoping](/docs/es/get-started/apis/scopes/api-scopes).
* El endpoint [`/tokeninfo`](https://auth0.com/docs/api/authentication/reference#get-token-info) recibe como entrada el [token de identidad de Auth0 y devuelve información del perfil de usuario](https://auth0.com/docs/api/authentication/reference#get-token-info). Este endpoint devolverá un resultado que no incluye los resultados de ninguna regla que altere el perfil de usuario.
