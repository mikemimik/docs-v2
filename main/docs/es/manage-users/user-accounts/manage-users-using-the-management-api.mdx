description: Aprende a crear usuarios y a ver sus detalles de perfil mediante la Management API.
title: Gestionar usuarios mediante la Management API
---

Además de usar el Dashboard, puedes recuperar, crear, actualizar o eliminar usuarios mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Users/get_users). Si quieres llamar directamente a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, primero tendrás que generar el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> adecuado. Para obtener más información, lee [Tokens de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens). Como alternativa, puedes usar un SDK para implementar la funcionalidad que necesitas para llamar a la Management API desde tu aplicación. Para consultar la lista de SDK disponibles, lee [la sección de SDKs de nuestra matriz de soporte](/docs/es/troubleshoot/customer-support/product-support-matrix).

Para aprender a administrar el acceso de los miembros de tu equipo al <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, lee [Administrar el acceso al Dashboard](/docs/es/get-started/manage-dashboard-access).

<div id="permissions">
  ## Permisos
</div>

Usa los siguientes permisos para obtener un token de acceso para las aplicaciones de máquina a máquina. Para obtener más información, consulta [Registrar aplicaciones de máquina a máquina.](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps)

| Permisos                    | Descripción                                                                                                                                                                                                               |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `read:users`                | Las aplicaciones pueden buscar y recuperar cualquier usuario almacenado en un tenant de Auth0. Esto incluye la lectura a través de los endpoints `/api/v2/users`, `/api/v2/users-by-email` y `/api/v2/jobs/users-export`. |
| `create:users`              | Las aplicaciones pueden crear usuarios en cualquier conexión de base de datos o passwordless de un tenant de Auth0. Esto incluye la creación a través de los endpoints `/api/v2/users` y `/api/v2/jobs/users-import`.     |
| `update:users`              | Las aplicaciones pueden actualizar cualquier usuario almacenado en un tenant de Auth0. Esto incluye la actualización a través del endpoint `/api/v2/users`.                                                               |
| `update:users_app_metadata` | Las aplicaciones pueden actualizar únicamente el atributo `user.app_metadata` de cualquier usuario almacenado en un tenant de Auth0. Esto incluye la actualización a través del endpoint `/api/v2/users endpoint`.        |
| `delete:users`              | Las aplicaciones pueden eliminar cualquier usuario almacenado en un tenant de Auth0. Esto incluye la eliminación a través del endpoint `/api/v2/users`.                                                                   |

<div id="limitations">
  ## Limitaciones
</div>

De forma predeterminada, los atributos del perfil de usuario proporcionados por <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente, ya que se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Para obtener más información sobre los atributos de los proveedores de identidad, consulta [Configurar la conexión del proveedor de identidad para actualizaciones del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Para poder editar los atributos raíz `name`, `nickname`, `given_name`, `family_name` o `picture` en el perfil de usuario normalizado, debes [configurar la sincronización de tu conexión con Auth0](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo cuando se cree el perfil de usuario. Estos atributos raíz estarán disponibles para editarse individualmente o mediante importaciones masivas

Puedes establecer contraseñas mediante los endpoints `create` o `update`, pero, por motivos de seguridad, no se pueden ver con los comandos `get` o `list user`. El lado derecho del API Explorer proporciona indicaciones sobre los atributos del perfil de usuario que se pueden ver o modificar en una llamada determinada.

<div id="endpoints">
  ## Endpoints
</div>

* Usa el endpoint [`/users`](https://auth0.com/docs/api/management/v2/users/get-users) para [obtener información sobre todos los usuarios](https://auth0.com/docs/api/management/v2/users/get-users). También puedes incluir criterios de búsqueda para encontrar usuarios específicos.
* Usa [`/user_id`](https://auth0.com/docs/api/management/v2/users/get-users-by-id) para [obtener información sobre un usuario](https://auth0.com/docs/api/management/v2/users/get-users-by-id) en función del `user_id`. El `user_id` es un identificador interno que consta de un nombre de conexión y un identificador único para el usuario. El `user_id` es distinto del <Tooltip tip="token de ID: Credencial destinada al propio cliente, y no a acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip>.
* El endpoint [`/userinfo`](https://auth0.com/docs/api/authentication/reference#get-user-info) toma como entrada el [token de acceso de Auth0 y devuelve información del perfil del usuario](https://auth0.com/docs/api/authentication/reference#get-user-info). Este endpoint incluirá los resultados de cualquier regla que haya podido modificar el perfil del usuario durante la transacción de autenticación, pero el perfil del usuario resultante no se filtrará por ningún [scope](/docs/es/get-started/apis/scopes/api-scopes).
* El endpoint [`/tokeninfo`](https://auth0.com/docs/api/authentication/reference#get-token-info) toma como entrada el [token de ID de Auth0 y devuelve información del perfil del usuario](https://auth0.com/docs/api/authentication/reference#get-token-info). Este endpoint devolverá un resultado que no incluye los resultados de ninguna regla que modifique el perfil del usuario.
