---
description: Cómo ver los usuarios y los detalles de su perfil.
title: Ver User Details
---

La página [Users](https://manage.auth0.com/#/users) muestra los usuarios asociados a tus aplicaciones. Para abrir un usuario concreto, haz clic en la foto de perfil o en el nombre del usuario en la columna &quot;Name&quot;. Se abrirá la página User Details y se mostrará la información de ese usuario.

La página User Details tiene enlaces a cinco pestañas:

1. Details
2. Devices
3. History
4. Locations
5. Raw JSON

<div id="user-details-details-tab">
  ## User Details: Details Tab
</div>

La pestaña Details contiene tres secciones que ofrecen una visión general de la información del perfil del usuario:

* **User Identity** proporciona detalles del usuario de un vistazo, incluido el identificador que utilizó, las Connections asociadas y los permisos de acceso.
* **Metadata** muestra la información de `app_metadata` y `user_metadata`. Puede editar estos valores.
* **<Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Atributos del proveedor de identidad</Tooltip>** muestra la información recuperada del proveedor de autenticación. Tenga en cuenta que los atributos del proveedor de identidad son de solo lectura.
* **Claves de acceso** muestra las [claves de acceso](/docs/es/authenticate/database-connections/passkeys) creadas por el usuario.
* **<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip>** muestra los factores de [MFA](/docs/es/secure/multi-factor-authentication) que el usuario tiene registrados.
  Si habilita [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains), dispone de la opción **Domain** para seleccionar el dominio de notificación de su tenant.

<div id="user-details-devices-tab">
  ## User Details: pestaña Devices
</div>

La pestaña Devices muestra los dispositivos desde los que el usuario ha solicitado autenticación. Solicitar autorización en un dispositivo vincula ese dispositivo a la cuenta del usuario.

Los datos de login del usuario están asociados al <Tooltip tip="token de actualización: token que se utiliza para obtener un Access Token renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token">token de actualización</Tooltip> asignado a ese dispositivo. Para revocar el token de actualización, haga clic en **Unlink** junto al dispositivo.

<div id="user-details-history-tab">
  ## User Details: pestaña History
</div>

La pestaña History muestra un registro de la actividad de la cuenta del usuario de los últimos 2 días.

Los registros incluyen información sobre:

* Los eventos que se han producido
* Cuándo se produjeron los eventos
* Las aplicaciones asociadas con los eventos
* El proveedor de identidad utilizado para la autenticación
* Las direcciones IP de origen de los eventos
* Dónde se originaron los eventos

<div id="user-details-locations-tab">
  ## User Details: pestaña Locations
</div>

La pestaña Locations muestra un mapa con marcadores que indican la ubicación o ubicaciones del usuario cuando inició sesión en las aplicaciones.

<div id="user-details-raw-json-tab">
  ## User Details: pestaña Raw JSON
</div>

La pestaña Raw JSON muestra toda la información contenida en el perfil del usuario en formato JSON, para que puedas consultar rápidamente toda la información disponible sobre el usuario.
