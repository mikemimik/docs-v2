---
description: Cómo ver a los usuarios y los detalles de su perfil.
title: Ver User Details
---

La página [Users](https://manage.auth0.com/#/users) muestra la lista de usuarios asociados a tus aplicaciones. Para abrir un usuario en particular, haz clic en la foto de perfil o en el nombre del usuario en la columna &quot;Name&quot;. Se abrirá la página User Details y mostrará la información de ese usuario.

La página User Details tiene enlaces a cinco pestañas:

1. Details
2. Devices
3. historial
4. Locations
5. Raw JSON

<div id="user-details-details-tab">
  ## User Details: pestaña Details
</div>

La pestaña Details contiene tres secciones que ofrecen una visión general de alto nivel de la información del perfil del usuario:

* **Identidad del usuario** proporciona detalles del usuario de un vistazo, incluido el identificador que utilizó, las Connections asociadas y los derechos de acceso.
* **Metadatos** muestra la información de `app_metadata` y `user_metadata`. Puedes editar estos valores.
* **<Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Atributos del proveedor de identidad</Tooltip>** muestra la información recuperada del proveedor de autenticación. Ten en cuenta que los atributos del proveedor de identidad son de solo lectura.
* **Claves de acceso** muestra las [claves de acceso](/docs/es/authenticate/database-connections/passkeys) creadas por el usuario.
* **<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip>** muestra los factores de [MFA](/docs/es/secure/multi-factor-authentication) en los que se ha inscrito el usuario.
  Si habilitas [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains), tienes la opción **Domain** para seleccionar el dominio de notificación de tu tenant.

<div id="user-details-devices-tab">
  ## User Details: pestaña Devices
</div>

La pestaña Devices muestra los dispositivos desde los que el usuario ha solicitado autenticación. Al solicitar autorización en un dispositivo, este queda vinculado a la cuenta del usuario.

Los datos de inicio de sesión del usuario están asociados al <Tooltip tip="token de actualización: Token utilizado para obtener un nuevo token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token">token de actualización</Tooltip> asignado a ese dispositivo. Para revocar el token de actualización, haz clic en **Desvincular** junto al dispositivo.

<div id="user-details-history-tab">
  ## User Details: Pestaña Historial
</div>

La pestaña Historial muestra un registro de la actividad de la cuenta del usuario de los últimos 2 días.

Los registros incluyen información sobre:

* Los eventos que han ocurrido
* Cuándo ocurrieron los eventos
* Las aplicaciones asociadas a los eventos
* El proveedor de identidad utilizado para la autenticación
* Las direcciones IP de origen de los eventos
* Dónde se originaron los eventos

<div id="user-details-locations-tab">
  ## User Details: Locations Tab
</div>

La pestaña Locations muestra un mapa con marcadores que indican las ubicaciones del usuario al iniciar sesión en las aplicaciones.

<div id="user-details-raw-json-tab">
  ## User Details: pestaña Raw JSON
</div>

La pestaña Raw JSON muestra toda la información del perfil del usuario en formato JSON para que puedas consultar rápidamente toda la información disponible sobre ese usuario.
