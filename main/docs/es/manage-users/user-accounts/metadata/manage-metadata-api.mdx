<div id="create-user-metadata">
  ### Crear metadatos del usuario
</div>

Para crear un usuario con los siguientes datos del perfil:

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing"
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

Haz la siguiente llamada `POST` al <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> endpoint [`/post_users`](https://auth0.com/docs/api/management/v2#!/Users/post_users) para crear el usuario y establecer los valores de las propiedades:

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/api/v2/users' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'content-type: application/json' \
    --data '{"email": "jane.doe@example.com", "user_metadata": {"hobby": "surfing"}, "app_metadata": {"plan": "full"}}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{"email": "jane.doe@example.com", "user_metadata": {"hobby": "surfing"}, "app_metadata": {"plan": "full"}}", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("content-type", "application/json")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

<div id="update-user-metadata">
  ### Actualizar los metadatos del usuario
</div>

Puedes actualizar los metadatos de un usuario mediante una llamada `PATCH` al endpoint [`/patch_users_by_id`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) de la Management API.

Suponiendo que hayas creado un usuario con los siguientes valores de metadatos:

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing"
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

Para actualizar `user_metadata` y añadir la dirección de domicilio del usuario como una propiedad de segundo nivel:

```json lines
{
    "addresses": {
        "home": "123 Main Street, Anytown, ST 12345"
    }
}
```

El perfil del usuario ahora se verá así:

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing",
```

<Warning>
  Cuando envías una llamada `PATCH` en la que estableces el valor de una propiedad en `null` (por ejemplo, `{user_metadata: {color: null}}`), Auth0 **elimina** la propiedad/el valor de la base de datos. Además, si aplicas un parche a los metadatos con un objeto vacío, los metadatos se eliminan por completo.
</Warning>

<div id="merge-user-metadata">
  ### Combinar los metadatos del usuario
</div>

Solo se combinan en el objeto las propiedades del nivel raíz. Todas las propiedades de niveles inferiores se reemplazarán.

Por ejemplo, para añadir la dirección de trabajo de un usuario como una subpropiedad adicional, tendrías que incluir todo el contenido de la propiedad `addresses`. Como el objeto `addresses` es una propiedad de nivel raíz, se combinará en el objeto JSON final que representa al usuario, pero sus subpropiedades no.

```json lines
{
  "user_metadata": {
    "addresses": {
      "home": "123 Main Street, Anytown, ST 12345",
      "work": "100 Industrial Way, Anytown, ST 12345"
    }
  }
}
```

Por lo tanto, la llamada `PATCH` correspondiente a la API sería:

<div id="app-metadata">
  ## Metadatos de la aplicación
</div>

Al aplicar un parche a los metadatos con un objeto vacío, los metadatos se eliminan por completo. Por ejemplo, al enviar este cuerpo se elimina todo lo que hay en `app_metadata`:

```json lines
{
  "app_metadata": {}
```

Se puede incluir un objeto `clientMetadata` al crear una aplicación nueva mediante el endpoint de aplicaciones `POST /api/v2/`.

<div id="read-client-metadata">

</div>

Los metadatos del cliente se incluyen en la respuesta de los endpoints `GET /api/v2/clients` y `GET /api/v2/client/{id}`.

<div id="update-client-metadata">
  ### Actualizar los metadatos del cliente
</div>

Los metadatos del cliente se pueden actualizar mediante el endpoint [`PATCH /api/v2/clients/{id}`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id), proporcionando un objeto de aplicación con la propiedad `clientMetadata`, cuyo valor consiste en un objeto que contiene los metadatos que quieras cambiar.

**Aplicación antes:**

```json lines
{
  ...
  "name": "myclient",
  "client_metadata": {
    "mycolor": "red",
    "myflavor": "grape"
  }
  ...
}
```

Solicitud: `PATCH /api/v2/client/myclientid123` con el cuerpo:

```json lines
{ "client_metadata": { "mycolor": "blue" } }
```

**Aplicación después:**

```json lines
{
  "name": "myclient",
  "client_metadata": {
    "mycolor": "blue",
    "myflavor": "grape"
  }
  ...
}
```

<div id="delete-client-metadata-properties-and-values">
  ### Eliminar propiedades y valores de los metadatos del cliente
</div>

Las claves de los metadatos del cliente se pueden eliminar mediante una solicitud PATCH, como se describe en &quot;Update app&#95;metadata&quot; más arriba, pero proporcionando `null` como valor de la clave. Este comportamiento coincide con el de las propiedades `user_metadata` y `app_metadata` en el endpoint `PATCH` [/api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).
