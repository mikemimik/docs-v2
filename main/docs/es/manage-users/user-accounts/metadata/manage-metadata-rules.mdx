<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Te recomendamos encarecidamente que uses Actions para extender Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprende cómo funcionan Auth0 Actions](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página específica de [Migrar a Actions](https://auth0.com/extensibility/movetoactions) que destaca comparativas de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte durante el proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Cómo prepararse para el fin de vida útil de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

Puedes leer, actualizar y eliminar metadatos con [Auth0 Rules](/docs/es/customize/rules). En las siguientes secciones, nos referiremos a este ejemplo, en el que el usuario y su información están representados por el siguiente fragmento JSON:

```json lines
{
  "user_id": "jdoe",
  "email": "john.doe@example.com",
  "app_metadata": {
    "roles": [ "writer" ]
  },
  "user_metadata": {
    "preferences": {
      "color": "blue"
    }
  }
}
```

<div id="read-metadata">
  ## Leer metadatos
</div>

Puede leer metadatos usando Rules con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. También puede buscar información relacionada con el perfil en `user_metadata`, como:

* `name`
* `nickname`
* `given_name`
* `family_name`

De forma predeterminada, los atributos del perfil de usuario proporcionados por <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente porque se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Para obtener más información sobre los atributos de los proveedores de identidad, consulte [Configurar la conexión del proveedor de identidad para las actualizaciones del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Para poder editar los atributos raíz `name`, `nickname`, `given_name`, `family_name` o `picture` del perfil de usuario normalizado, debe [configurar la sincronización de su conexión con Auth0](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo cuando se cree el perfil de usuario. Estos atributos raíz estarán disponibles para editarlos individualmente o mediante importaciones masivas.

Como ejemplo, suponga que los siguientes metadatos están almacenados para un usuario con la dirección de correo electrónico `jane.doe@example.com`:

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing"
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

Usando los metadatos de ejemplo anteriores, puedes consultar elementos específicos del conjunto de datos en [Auth0 Rules](/docs/es/customize/rules) o mediante una llamada a la [Management API](/docs/es/manage-users/user-accounts/metadata), de la siguiente manera:

```js lines
console.log(user.email); // "jane.doe@example.com"
console.log(user.user_metadata.hobby); // "surfing"
console.log(user.app_metadata.plan); // "full"
```

Cualquier fragmento de JSON válido puede usarse como metadato, pero ten en cuenta que `user.app_metadata` es `Undefined` de manera predeterminada.

Para leer los metadatos disponibles, tendrás que acceder a la propiedad correcta del usuario.

<div id="read-app-metadata">
  ### Leer metadatos de la aplicación
</div>

Puedes tomar una decisión en función de los roles del usuario:

```javascript lines
function(user, context, callback){
  user.app_metadata = user.app_metadata || {};
  if (user.app_metadata.roles.indexOf('writer')){
    // código a ejecutar
  }
  ...
}
```

<div id="read-user-metadata">
  ### Leer los metadatos del usuario
</div>

Puedes basar tus decisiones en preferencias específicas, como la preferencia de color:

```javascript lines
function(user, context, callback){
  user.user_metadata = user.user_metadata || {};
  if (user.user_metadata.preferences.color === 'black'){
    // código a ejecutar
  }
  ...
}
```

<div id="read-application-metadata-clientmetadata">
  ### Leer los metadatos de la aplicación (clientMetadata)
</div>

Los metadatos de la aplicación (`clientMetadata`) son una propiedad opcional de nivel superior del objeto `context`. Las aplicaciones existentes no tendrán ningún valor en esta propiedad.

```javascript lines
function(user, context, callback){
  context.clientMetadata = context.clientMetadata || {};
  if (context.clientMetadata.usersuppliedkey1 === 'black'){
    // este código no se ejecutaría para el usuario
  }
  ...
}
```

<div id="update-metadata">
  ## Actualizar metadatos
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Usa Rules para asignar los atributos SAML que Auth0 recibe del IdP a `user_metadata` o `app_metadata`.
</Callout>

<div id="update-app-metadata">
  ### Actualizar los metadatos de la aplicación
</div>

Para agregar un rol de administrador al usuario:

```javascript lines
function(user, context, callback){
  user.app_metadata = user.app_metadata || {};
  // actualizar el app_metadata que formará parte de la respuesta
  user.app_metadata.roles = user.app_metadata.roles || [];
  user.app_metadata.roles.push('administrator');

  // guardar la actualización de app_metadata
  auth0.users.updateAppMetadata(user.user_id, user.app_metadata)
    .then(function(){
      callback(null, user, context);
    })
    .catch(function(err){
      callback(err);
    });
}
```

Esto da lugar a la siguiente representación JSON de los detalles del perfil del usuario:

```json lines
{
  "user_id": "jdoe",
  "email": "john.doe@example.com",
  "app_metadata": {
    "roles": [ "writer", "administrator" ]
  },
  "user_metadata": {
    "preferences": {
      "color": "blue"
    }
  }
}
```

<div id="update-user-metadata">
  ### Actualizar los metadatos del usuario
</div>

Para añadir la preferencia `fontSize` del usuario al perfil del usuario:

```javascript lines
function(user, context, callback){
  user.user_metadata = user.user_metadata || {};
  // actualizar el user_metadata que formará parte de la respuesta
  user.user_metadata.preferences = user.user_metadata.preferences || {};
  user.user_metadata.preferences.fontSize = 12;

  // conservar la actualización de user_metadata
  auth0.users.updateUserMetadata(user.user_id, user.user_metadata)
    .then(function(){
      callback(null, user, context);
    })
    .catch(function(err){
      callback(err);
    });
}
```

Esto da lugar a la siguiente representación JSON de los detalles del perfil del usuario:

```json lines
{
  "user_id": "jdoe",
  "email": "john.doe@example.com",
  "app_metadata": {
    "roles": [ "writer" ]
  },
  "user_metadata": {
    "preferences": {
      "color": "blue",
      "fontSize": 12
    }
  }
}
```

<div id="update-app-and-user-metadata-simultaneously">
  ### Actualizar simultáneamente los metadatos de la aplicación y del usuario
</div>

Para reducir el tiempo de procesamiento de la rule, puede actualizar `app_metadata` y `user_metadata` en la misma rule:

```javascript lines expandable
function(user, context, callback){

  var q = require('q');

  user.app_metadata = user.app_metadata || {};
  user.user_metadata = user.user_metadata || {};
  // actualizar el user_metadata que formará parte de la respuesta
  user.user_metadata.preferences = user.user_metadata.preferences || {};
  user.user_metadata.preferences.fontSize = 12;

  // actualizar el app_metadata que formará parte de la respuesta
  user.app_metadata.roles = user.app_metadata.roles || [];
  user.app_metadata.roles.push('admin');

  // guardar la actualización de app_metadata
  var appMetadataPromise  = auth0.users.updateAppMetadata(user.user_id, user.app_metadata);

  // guardar la actualización de user_metadata
  var userMetadataPromise = auth0.users.updateUserMetadata(user.user_id, user.user_metadata);

  // usar la biblioteca q para esperar a que todas las promesas se completen
  q.all([userMetadataPromise, appMetadataPromise])
    .then(function(){
      callback(null, user, context);
    })
    .catch(function(err){
      callback(err);
    });
}
```

Esto da lugar a la siguiente representación JSON de los detalles del perfil del usuario:

```json lines
{
  "user_id": "jdoe",
  "email": "john.doe@example.com",
  "app_metadata": {
    "roles": [ "writer", "admin" ]
  },
  "user_metadata": {
    "preferences": {
      "color": "blue",
      "fontSize": 12
    }
  }
}
```

<div id="delete-metadata">
  ## Eliminar metadatos
</div>

<div id="delete-app-metadata-properties-and-values">
  ### Eliminar propiedades y valores de los metadatos de la aplicación
</div>

Para eliminar una propiedad, establece su valor en `null`.

<div id="delete-users-roles-example">
  #### Ejemplo de cómo eliminar los roles de un usuario
</div>

Para eliminar los roles de un usuario, use la siguiente `rule` de ejemplo:

```javascript lines
function(user, context, callback){
  user.app_metadata = user.app_metadata || {};
  // actualizar el app_metadata que formará parte de la respuesta
  user.app_metadata.roles = null;

  // persistir la actualización del app_metadata
  auth0.users.updateAppMetadata(user.user_id, user.app_metadata)
    .then(function(){
      callback(null, user, context);
    })
    .catch(function(err){
      callback(err);
    });
}
```

Esto da como resultado la siguiente representación JSON del perfil del usuario:

```json lines
{
  "user_id": "jdoe",
  "email": "john.doe@example.com",
  "app_metadata": { },
  "user_metadata": {
    "preferences": {
      "color": "blue"
    }
  }
}
```

<div id="delete-single-property-value-example">
  #### Ejemplo para eliminar un único valor de una propiedad
</div>

Para eliminar un único valor de una propiedad, quite ese valor concreto. Por ejemplo, para quitar el rol `writer` del perfil de usuario:

```javascript lines
function(user, context, callback){
  user.app_metadata = user.app_metadata || {};
  user.app_metadata.roles = user.app_metadata.roles || [];

  var index = user.app_metadata.roles.indexOf('writer');

  if (index !== -1){
    // actualizar el app_metadata que formará parte de la respuesta
    user.app_metadata.roles.splice(index, 1);
  }

  // persistir la actualización de app_metadata
  auth0.users.updateAppMetadata(user.user_id, user.app_metadata)
    .then(function(){
      callback(null, user, context);
    })
    .catch(function(err){
      callback(err);
    });
}
```

Esto da como resultado la siguiente representación JSON del perfil del usuario:

```json lines
{
  "user_id": "google-oauth2|1234",
  "email": "john.doe@gmail.com",
  "app_metadata": {
    "roles": [ ]
  },
  "user_metadata": {
    "preferences": {
      "color": "blue"
    }
  }
}
```

Ten en cuenta que la propiedad `roles` sigue existiendo, pero no contiene ningún valor.

<div id="delete-user-metadata-properties-and-values">
  ### Eliminar propiedades y valores de los metadatos del usuario
</div>

Para eliminar la preferencia de color del usuario:

```javascript lines
function(user, context, callback){
  user.user_metadata = user.user_metadata || {};
  // actualizar el user_metadata que formará parte de la respuesta
  user.user_metadata.preferences = user.user_metadata.preferences || {};
  delete user.user_metadata.preferences.color;

  // guardar la actualización del user_metadata
  auth0.users.updateUserMetadata(user.user_id, user.user_metadata)
    .then(function(){
      callback(null, user, context);
    })
    .catch(function(err){
      callback(err);
  });
}
```

El resultado es la siguiente representación JSON de los detalles del perfil de usuario:

```json lines
{
  "user_id": "jdoe",
  "email": "john.doe@example.com",
  "app_metadata": {
    "roles": [ "writer" ]
  },
  "user_metadata": {
    "preferences": { }
  }
}
```
