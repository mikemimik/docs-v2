sidebarTitle: Metadatos
title: Comprende cómo funcionan los metadatos en los perfiles de usuario
---

Auth0 proporciona un sistema completo para almacenar metadatos en el perfil de usuario de Auth0. Puedes usar los metadatos para realizar las siguientes acciones:

* Almacenar datos específicos de la aplicación en el perfil de usuario.
* Registrar si se han realizado o no operaciones específicas para un usuario.
* Almacenar en caché los resultados de operaciones costosas en el perfil de usuario para poder reutilizarlos en futuros inicios de sesión.
* Almacenar información que no proviene de un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> o que sobrescribe la que proporciona un proveedor de identidad.

Los metadatos pueden modificarse como parte del flujo de inicio de sesión de un usuario.

Puedes configurar la sincronización de la conexión para que el proveedor de identidad actualice los atributos raíz del usuario solo cuando se crea el perfil de usuario. Después, puedes editar los atributos raíz de forma individual o mediante una importación masiva. Para obtener más información, consulta [Configurar la conexión del proveedor de identidad para actualizaciones de perfiles de usuario](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

<div id="metadata-types">
  ## Tipos de metadatos
</div>

Auth0 usa tres tipos de metadatos para almacenar tipos concretos de información.

| Tipo de metadatos                | Nombre del campo                                                                                                                   | Descripción                                                                                                                                                                                                                                                                             |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Información del usuario**      | `user_metadata`                                                                                                                    | Almacena atributos del usuario, como preferencias, que no afectan a la funcionalidad principal del usuario. Estos datos **pueden** ser editados por usuarios que hayan iniciado sesión si creas un formulario con la Management API y **no** deben usarse como almacén seguro de datos. |
| **Información de acceso**        | `app_metadata`                                                                                                                     | Almacena información como permisos, el plan de Auth0 e IDs externos que pueden afectar el acceso del usuario a las funcionalidades. Estos datos **no pueden** ser editados por los usuarios y existen restricciones sobre lo que se puede almacenar en este campo.                      |
| **Información de la aplicación** | `client_metadata` en el objeto `Client`, `context.clientMetadata` en Rules y `event.client.metadata` en las Actions de Post Login. | Almacena información sobre una aplicación (o *cliente* en la terminología de OIDC OAuth2). Por ejemplo, la URL de la página de inicio de la aplicación (cualquier valor que Auth0 no establezca en la configuración de la aplicación).                                                  |

<div id="manage-metadata">
  ## Administrar metadatos
</div>

Puedes crear y actualizar metadatos con Rules, Authentication API, la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y la biblioteca Lock.

En los scripts de Custom DB y en el script `fetchUserProfile` de las conexiones sociales personalizadas, puedes crear y actualizar metadatos mediante el objeto `metadata`.

<div id="use-actions">
  ### Usa Actions
</div>

[Actions](/docs/es/customize/actions) son funciones seguras, propias del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma Auth0. Las Actions se utilizan para personalizar y ampliar las capacidades de Auth0 con lógica personalizada. También pueden usarse para [enriquecer el perfil de usuario](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

Por ejemplo, puedes crear una Action de `Post Login` que use claims personalizados para copiar propiedades de `user_metadata` a los <Tooltip tip="token de ID: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de ID</Tooltip>. Después, puedes recuperar el `user_metadata` de los usuarios a través del endpoint [Get User Info](https://auth0.com/docs/api/authentication#get-user-info) de la Authentication API.

Para obtener más información, consulta [Administra los metadatos del usuario con el trigger de Action de Post Login](/docs/es/manage-users/user-accounts/metadata/manage-user-metadata).

<div id="use-the-management-api">
  ### Usa la Management API
</div>

Un usuario puede solicitar un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> con los scopes adecuados y usar los siguientes endpoints de la [Management API](https://auth0.com/docs/api/management/v2) para ver, crear o actualizar `user_metadata`, o eliminar una configuración de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>.

| Tarea      | Endpoint                                                                                                                              | Scope                          |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ |
| Ver        | [`GET /api/v2/users/{id}`](https://auth0.com/docs/api/management/v2#!/Users/get_users_by_id)                                          | `read:current_user`            |
| Crear      | [`PATCH /api/v2/users/{id}`](https://auth0.com/docs/api/management/v2#!/Users/post_users)                                             | `create:current_user_metadata` |
| Actualizar | [`PATCH /api/v2/users/{id}`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id)                                      | `update:current_user_metadata` |
| Eliminar   | [`DELETE /api/v2/users/{id}/multifactor/{provider}`](https://auth0.com/docs/api/management/v2#!/Users/delete_multifactor_by_provider) | `update:users`                 |

Se requiere un token de acceso para llamar a la Auth0 Management API. Para obtener más información, consulta [Access Tokens for the Management API](https://auth0.com/docs/api/management/v2/tokens) y [Get Management API Tokens for SPAs](https://auth0.com/docs/api/management/v2/get-access-tokens-for-spas).

<div id="use-the-dashboard">
  ### Usa el Dashboard
</div>

Usa Auth0 Dashboard para configurar los metadatos de la aplicación, que contienen pares de clave/valor. Para obtener más información, consulta [Configurar los metadatos de la aplicación](/docs/es/get-started/applications/configure-application-metadata).

<div id="use-the-lock-library">
  ### Usa la biblioteca Lock
</div>

Usa la [biblioteca Lock](/docs/es/libraries/lock) para definir, agregar, leer y actualizar `user_metadata.` Lee las propiedades de `user_metadata` de la misma forma en que leerías cualquier otra propiedad del perfil de usuario. Por ejemplo, el siguiente fragmento de código recupera el valor asociado a `user_metadata.hobby` y lo asigna a un elemento de la página:

```javascript lines
// Usa el accessToken obtenido durante la autenticación para llamar a getUserInfo
lock.getUserInfo(accessToken, function(error, profile) {
  if (!error) {
    document.getElementById('hobby').textContent = profile.user_metadata.hobby;
  }
});
```

Puedes usar `additionalSignUpFields` para añadir campos personalizados a los formularios de registro de usuarios. Cuando un usuario introduce datos en un campo personalizado, Auth0 almacena los valores introducidos en el `user_metadata` del usuario. Para obtener más información sobre cómo añadir `user_metadata` durante el registro, consulta [Additional Signup Fields](/docs/es/libraries/lock/lock-configuration#additionalsignupfields).

<div id="custom-database-connections-and-metadata">
  ## Conexiones de base de datos personalizadas y metadatos
</div>

Si tienes una [conexión de base de datos personalizada](/docs/es/authenticate/database-connections#using-your-own-user-store), puedes usar el endpoint [`/dbconnections/signup`](https://auth0.com/docs/api/authentication?shell#signup) de Authentication API para establecer `user_metadata` de un usuario. Para obtener más información sobre cómo trabajar con metadatos durante un proceso de registro personalizado, consulta [Custom Signup](/docs/es/libraries/custom-signup).

Cuando estableces el campo `user_metadata` mediante el endpoint `/dbconnections/signup` de Authentication API, el límite es de 10 campos de tipo String y 500 caracteres como máximo.

<div id="custom-emails-and-metadata">
  ## Correos electrónicos personalizados y metadatos
</div>

Usa metadatos para almacenar la información que quieras utilizar para personalizar los correos electrónicos de Auth0. Por ejemplo, usa `user_metadata.lang` si quieres que el usuario pueda cambiar el valor del campo y, después, utiliza esa información para personalizar el idioma de un correo electrónico. Para obtener más información, consulta [Personalizar plantillas de correo electrónico](/docs/es/customize/email/email-templates).
