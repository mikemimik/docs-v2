title: Prácticas recomendadas para User Search
---

Estas son algunas prácticas recomendadas para la búsqueda de usuarios:

* Necesita un token para realizar solicitudes a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, lea [Management API Access Tokens](/docs/es/secure/tokens/access-tokens/management-api-access-tokens).
* Para realizar solicitudes de búsqueda de usuarios, se requiere el scope `read:users`.
* Aproveche la búsqueda no determinista: si su aplicación no requiere un orden coherente de los resultados de búsqueda y sus consultas son lo suficientemente cortas como para no depender de la paginación, establezca `primary_order=false` para mejorar el rendimiento de la consulta.
* Para obtener los resultados de búsqueda más recientes, use un endpoint **inmediatamente consistente** durante los procesos de autenticación, como [Get Users by ID](/docs/es/manage-users/user-search/retrieve-users-with-get-users-by-id-endpoint) y [Get Users by Email](/docs/es/manage-users/user-search/retrieve-users-with-get-users-by-email-endpoint). Las búsquedas que usan estos endpoints reflejan los resultados de todas las operaciones de escritura correctas, incluidas las que ocurrieron poco antes de su solicitud.
* Use un esquema well-known para metadata:

  * Use tipos de datos coherentes para las propiedades.
  * Evite los nombres de propiedad dinámicos.
  * Evite esquemas grandes y estructuras profundas.
  * Evite almacenar datos que no necesita para fines de autenticación y autorización.
* Las consultas de búsqueda superan el tiempo de espera (código de estado HTTP 503) si no se completan en dos segundos o menos. Las consultas que tardan más indican que se trata de una consulta costosa o que la consulta contiene un error que impide que se complete rápidamente.
* Las consultas de búsqueda con atributos definidos por el usuario en `app_metadata` y `user_metadata` pueden superar el tiempo de espera. Le recomendamos no usar [List or search users](https://auth0.com/docs/api/management/v2/users/get-users) para operaciones críticas.
* No use un criterio de búsqueda que devuelva un conjunto de datos grande (más de 1000 resultados).
* No use consultas de existencia (por ejemplo, &quot;deme todos los usuarios con una propiedad independientemente de su valor&quot;).
* No consulte repetidamente las API de búsqueda.
* No realice solicitudes de búsqueda de usuarios dentro de puntos de extensión del flujo de inicio de sesión, como Rules o Actions de `Post Login`.
* No use campos de metadata grandes (intente mantener los campos de metadata en 2 KB o menos).
* El uso de wildcards en las búsquedas puede afectar al rendimiento. En algunos casos, las búsquedas con wildcards en conjuntos de datos grandes pueden provocar errores por tiempo de espera. También recomendamos evitar los wildcards al principio del término de búsqueda, mientras que usarlos como sufijos ofrece un mejor rendimiento.
* Escape el carácter de espacio para mejorar el rendimiento (p. ej., `q=name:John Doe` debe escribirse como `q=name:John\ Doe`).
