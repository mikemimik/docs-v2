---

Aquí tienes algunas prácticas recomendadas para la búsqueda de usuarios:

* Necesitas un token para hacer solicitudes a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulta [Management API Access Tokens](/docs/es/secure/tokens/access-tokens/management-api-access-tokens).
* Para realizar solicitudes de búsqueda de usuarios, se requiere el scope `read:users`.
* Aprovecha la búsqueda no determinista: si tu aplicación no requiere un orden consistente de los resultados de búsqueda y tus consultas son lo suficientemente cortas como para no depender de la paginación, establece `primary_order=false` para mejorar el rendimiento de la consulta.
* Para obtener los resultados de búsqueda más recientes, usa un endpoint **inmediatamente consistente** durante los procesos de autenticación, como [Get Users by ID](/docs/es/manage-users/user-search/retrieve-users-with-get-users-by-id-endpoint) y [Get Users by Email](/docs/es/manage-users/user-search/retrieve-users-with-get-users-by-email-endpoint). Las búsquedas que usan estos endpoints reflejan los resultados de todas las operaciones de escritura exitosas, incluidas las que ocurrieron poco antes de tu solicitud.
* Usa un schema conocido para los metadatos:

  * Usa tipos de datos consistentes para las propiedades.
  * Evita nombres de propiedades dinámicos.
  * Evita schemas grandes y estructuras profundas.
  * Evita almacenar datos que no necesites para fines de autenticación y autorización.
* Las consultas de búsqueda agotan el tiempo de espera (código de estado HTTP 503) si no se completan en dos segundos o menos. Las consultas que tardan más indican que es una consulta costosa o que contiene un error que impide que se complete rápidamente.
* Las consultas de búsqueda con atributos definidos por el usuario en `app_metadata` y `user_metadata` pueden agotar el tiempo de espera. Te recomendamos que no uses [List or search users](https://auth0.com/docs/api/management/v2/users/get-users) para operaciones críticas.
* No uses un criterio de búsqueda que devuelva un conjunto de datos grande (más de 1000 resultados).
* No uses consultas de existencia (por ejemplo, &quot;dame todos los usuarios con una propiedad independientemente de su valor&quot;).
* No consultes repetidamente las API de búsqueda.
* No realices solicitudes de búsqueda de usuarios dentro de puntos de extensión del flujo de login, como Rules o Actions de `post-login`.
* No uses campos de metadatos grandes (intenta mantenerlos en 2 KB o menos).
* El uso de wildcard en las búsquedas puede afectar el rendimiento. En algunos casos, las búsquedas con wildcard en conjuntos de datos grandes pueden provocar errores por tiempo de espera. También recomendamos evitar wildcards al principio del término de búsqueda, mientras que usarlos como sufijos ofrece un mejor rendimiento.
* Escapa el carácter de espacio para mejorar el rendimiento (p. ej., `q=name:John Doe` debería escribirse como `q=name:John\ Doe`).
