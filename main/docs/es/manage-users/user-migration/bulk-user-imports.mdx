Puedes importar datos de usuario de forma masiva a Auth0 mediante el endpoint [Create Import Users Job](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports). Las importaciones masivas son útiles para migrar usuarios desde una base de datos o un servicio existente a Auth0.

<Warning>
  Si intentas usar más de un método de migración (por ejemplo, migración automática y luego importación masiva de usuarios), puede que te encuentres con un error `DUPLICATED_USER`. Este error indica que el usuario existe en el repositorio interno de usuarios de Auth0, pero no en tu tenant. Para corregir este error, elimina al usuario mediante el endpoint [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de la Auth0 Management API y vuelve a intentar la importación.
</Warning>

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de iniciar la tarea de importación de usuarios:

* Configure una conexión de base de datos en la que importar los usuarios y habilítela para al menos una aplicación.
* Si va a importar contraseñas, asegúrese de que tengan hash con uno de los algoritmos compatibles. Los usuarios cuyas contraseñas tengan hash con algoritmos no compatibles tendrán que restablecer su contraseña la primera vez que inicien sesión después de la importación masiva.
* Si va a importar inscripciones de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>, asegúrese de que sean de un tipo compatible: `email`, `phone` o `totp`.
* [Obtenga un token de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) para las solicitudes al endpoint de tareas.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si utiliza un archivo de exportación de un tenant de Auth0, debe convertir el archivo exportado de `ndjson` a [JSON](/docs/es/manage-users/user-migration/bulk-user-exports#json-compatible-format). Para conservar los mismos ID de usuario, debe eliminar el `prefijo auth0|` de todos los ID de usuario importados.

  El proceso de importación agrega automáticamente el `prefijo auth0|` a los ID de usuario importados. Si no elimina el prefijo `auth0|` antes de importar, los ID de usuario quedarán como `auth0|auth0|...`
</Callout>

<div id="create-users-json-file">
  ## Crear archivo JSON de usuarios
</div>

Cree un archivo JSON con los datos de usuario que desea importar a Auth0. La forma de exportar los datos de usuario a un archivo JSON variará según la base de datos de usuarios que utilice actualmente. El endpoint de <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> espera recibir secciones del archivo JSON. Por lo tanto, en lugar de usar `fs.readFileSync`, requiere `fs.createReadStream`. El endpoint espera un flujo de lectura canalizado en lugar del archivo JSON completo.

Para obtener más información sobre el esquema del archivo JSON y ver ejemplos, lea [Esquema y ejemplos para la importación masiva de usuarios de base de datos](/docs/es/manage-users/user-migration/bulk-user-import-database-schema-and-examples).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El límite de tamaño de archivo para una importación masiva es de 500 KB. Deberá iniciar varias importaciones si sus datos superan ese tamaño.
</Callout>

<div id="request-bulk-user-import">
  ## Solicitar una importación masiva de usuarios
</div>

Para iniciar una tarea de importación masiva de usuarios, envía una solicitud `POST` al endpoint [Create Import Users Job](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports). Asegúrate de reemplazar los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `USERS_IMPORT_FILE.json`, `CONNECTION_ID` y `EXTERNAL_ID` por tu <Tooltip tip="Access Token: credencial de Authorization, en forma de cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> de Management API, el archivo JSON de usuarios, el [ID de conexión](/docs/es/authenticate/identity-providers/locate-the-connection-id) de la conexión de base de datos y el ID externo, respectivamente.

| Parameter               | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `users`                 | Archivo en formato JSON que contiene los usuarios que se van a importar.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| `connection_id`         | ID de la conexión en la que se insertarán los usuarios. Puede recuperar el ID mediante el endpoint [GET /api/v2/connections](https://auth0.com/docs/api/management/v2#!/Connections/get_connections).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `upsert`                | Valor Boolean; `false` de forma predeterminada. Cuando se establece en `false`, los usuarios existentes que coincidan por dirección de correo electrónico, ID de usuario, phone o username generarán un error. Cuando se establece en `true`, los usuarios existentes que coincidan por dirección de correo electrónico se actualizarán, pero solo con atributos que permiten upsert. Para ver una lista de los campos del perfil de usuario que pueden actualizarse mediante upsert durante la importación, consulte [Estructura del perfil de usuario: Atributos del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/user-profile-structure#user-profile-attributes). Nota: Proporcionar una entrada de usuario duplicada en el archivo de importación provocará un error. En este caso, Auth0 no realizará una inserción seguida de una actualización. |
| `external_id`           | Cadena opcional definida por el usuario que puede usarse para correlacionar múltiples tareas. Se devuelve como parte de la respuesta de status de la tarea.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| `send_completion_email` | Valor Boolean; `true` de forma predeterminada. Cuando se establece en `true`, envía un correo electrónico de finalización a todos los propietarios del tenant cuando finaliza la tarea de importación. Si *no* desea que se envíen correos electrónicos, debe establecer explícitamente este parámetro en `false`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

Si la solicitud se realiza correctamente, recibirá una respuesta similar a la siguiente:

```json lines
{
  "status": "pending",
  "type": "users_import",
  "created_at": "",
  "id": "job_abc123",
  "connection_id": "CONNECTION_ID",
  "upsert": false,
  "external_id": "EXTERNAL_ID",
  "send_completion_email": true
}
```

La entidad devuelta representa la tarea de importación.

Cuando la tarea de importación de usuarios finalice y `send_completion_email` se haya establecido en `true`, los administradores del tenant recibirán un correo electrónico para notificarles si la tarea falló o se completó correctamente. Un correo electrónico sobre una tarea fallida podría notificar a los administradores que no se pudo analizar el archivo JSON de usuarios durante la importación.

<div id="concurrent-import-jobs">
  ### Tareas de importación simultáneas
</div>

El endpoint [Create Import Users Job](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports) tiene un límite de dos tareas de importación simultáneas. Si se solicitan tareas adicionales mientras hay dos pendientes, se devuelve una respuesta `429 Too Many Requests`:

```json lines
{
  "statusCode": 429,
  "error": "Too Many Requests",
  "message": "There are 2 active import users jobs, please wait until some of them are finished and try again
}
```

**Pendiente**

```json lines
{
  "status": "pending",
  "type": "users_import",
  "created_at": "",
  "id": "job_abc123",
  "connection_id": "CONNECTION_ID",
  "external_id": "EXTERNAL_ID"
```

**Completada**

Si una tarea se completa, la respuesta del estado de la tarea incluirá los totales de registros exitosos, fallidos, insertados y actualizados.

```json lines
{
  "status": "completed",
  "type": "users_import",
  "created_at": "",
  "id": "job_abc123",
  "connection_id": "CONNECTION_ID",
  "external_id": "EXTERNAL_ID",

    "failed": 0,
    "updated": 0,
    "inserted": 1,
    "total": 1
  }
}
```

**Failed**

Si se produce un error en la tarea, su estado será Failed. Sin embargo, ten en cuenta que la información de usuario no válida, como una dirección de correo electrónico no válida, no hará que falle toda la tarea.

```json lines
{
  "status": "failed",
  "type": "users_import",
  "created_at": "",
  "id": "job_abc123",
  "connection_id": "CONNECTION_ID",
  "external_id": "EXTERNAL_ID",
}
```

Para obtener más información sobre las entradas fallidas, consulta **Recuperar entradas fallidas** más abajo.

<div id="job-timeouts">
  ## Tiempos de espera de las tareas
</div>

Todas las tareas de importación de usuarios caducan después de **dos (2) horas**. Si tu tarea no se completa dentro de ese plazo, se marca como fallida.

Además, todos los datos relacionados con tu tarea se eliminan automáticamente después de 24 horas y ya no se podrá acceder a ellos. Por ello, **recomendamos encarecidamente almacenar los resultados de la tarea mediante el mecanismo de almacenamiento que elijas**.

<div id="retrieve-failed-entries">

<div id="best-practices-for-large-scale-migrations">
  ## Prácticas recomendadas para migraciones a gran escala
</div>

Al importar grandes volúmenes de usuarios (que requieren 10 o más tareas), sigue estas recomendaciones para garantizar una migración exitosa:

<div id="file-size-and-chunking">
  ### Tamaño de archivo y división en bloques
</div>

Divide los registros de usuario en bloques pequeños de menos de 400 KB por archivo. Esto permite procesar aproximadamente 1.000 usuarios por tarea con menos de 10 campos de metadatos por usuario, sin acercarse al límite de tamaño de archivo de 500 KB.

<div id="job-scheduling-and-concurrency">
  ### Programación de tareas y concurrencia
</div>

Dado que Auth0 restringe las importaciones de usuarios a dos tareas simultáneas por tenant, usa un framework de programación de tareas (como [Bull](https://github.com/OptimalBits/bull) o [Agenda](https://github.com/agenda/agenda)) para:

* Programar las tareas de importación de forma sistemática
* Aplicar el límite de concurrencia requerido
* Hacer un seguimiento del progreso de la importación para poder retomarla donde la dejaste si se interrumpe
* Mantener el historial de tareas y sus estados actuales

<div id="error-handling-strategy">
  ### Estrategia de manejo de errores
</div>

* No interrumpa el proceso de importación por fallos aislados de usuarios. Importe todos los usuarios y gestione los problemas después.
* Implemente una tarea de &quot;finalización&quot; que revise todas las tareas generadas y recopile en un nuevo archivo los registros que hayan fallado.
* Corrija los errores de los registros fallidos recopilados e impórtelos en una nueva tarea.
* Incluya una estrategia de reintento para manejar cortes en la conexión de red y fallos temporales.

<div id="upsert-mode-considerations">
  ### Consideraciones sobre el modo upsert
</div>

Tenga cuidado al habilitar el modo `upsert`:

* Las importaciones con upsert son más lentas que las importaciones normales.
* El modo upsert no permite fusionar `user_metadata` ni `app_metadata`. Los metadatos existentes se sobrescribirán por completo con los nuevos valores.
* Use el modo upsert solo cuando necesite actualizar usuarios existentes y comprenda estas limitaciones.

<div id="data-retention">
  ### Retención de datos
</div>

Recuerda que todos los datos relacionados con las tareas se eliminan automáticamente a las 24 horas. Guarda los resultados de las tareas y los detalles de los errores con tu propio mecanismo de almacenamiento para su seguimiento y análisis a largo plazo.
