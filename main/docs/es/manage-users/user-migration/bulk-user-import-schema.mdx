El archivo de usuarios debe contener un arreglo con la información de los usuarios en formato JSON.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El límite de tamaño de archivo para una importación masiva es de 500 KB. Tendrás que iniciar varias importaciones si tus datos superan este límite.
</Callout>

* `__tenant`
* `_id`
* `blocked`
* `clientID`
* `created_at`
* `email_verified`
* `email`
* `globalClientID`
* `global_client_id`
* `identities`
* `lastIP`
* `lastLogin`
* `loginsCount`
* `metadata`
* `multifactor_last_modified`
* `multifactor`
* `updated_at`
* `user_id`

<div id="custom-password-hash">
  ## Hash de contraseña personalizado
</div>

El objeto `user.custom_password_hash` puede usarse en lugar de la propiedad `user.password_hash` cuando el hash de la contraseña del usuario se creó con un algoritmo distinto. Ten en cuenta que este campo y `password_hash` son mutuamente excluyentes.

El objeto `user.custom_password_hash` tiene las siguientes propiedades:

| Propiedad           | Tipo    | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ------------------- | ------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `algorithm`         | string  | El algoritmo usado para generar el hash de la contraseña. Debe ser uno de los siguientes: <ul><li>`argon2`</li><li>`bcrypt`</li><li>`hmac`</li><li>`ldap`</li><li>`md4`</li><li>`md5`</li><li>`sha1`</li><li>`sha256`</li><li>`sha512`</li><li>`pbkdf2`</li><li>`scrypt`</li></ul>                                                                                                                                                                                                            |
| `hash`              | object  |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `hash.value`        | string  | El hash de la contraseña.                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `hash.encoding`     | string  | La codificación del hash proporcionado. Debe ser una de las siguientes: <ul><li>`base64`</li><li>`hex`</li><li>`utf8`</li></ul>Se admiten variantes hexadecimales en mayúsculas y minúsculas, así como base64 codificado para URL.                                                                                                                                                                                                                                                            |
| `hash.digest`       | string  | El algoritmo usado para generar el hash HMAC. Debe ser uno de los siguientes: <ul><li>`md4`</li><li>`md5`</li><li>`ripemd160`</li><li>`sha1`</li><li>`sha224`</li><li>`sha256`</li><li>`sha384`</li><li>`sha512`</li><li>`whirlpool`</li></ul>                                                                                                                                                                                                                                                |
| `hash.key`          | object  | La clave usada para generar el hash HMAC.                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `hash.key.value`    | string  | El valor de la clave.                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `hash.key.encoding` | string  | La codificación de la clave. Debe ser una de las siguientes: <ul><li>`base64`</li><li>`hex`</li><li>`utf8`</li></ul>De forma predeterminada, `hash.key.encoding` es `utf8`.                                                                                                                                                                                                                                                                                                                   |
| `salt`              | object  |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `salt.value`        | string  | El valor de la sal usada para generar el hash.                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| `salt.encoding`     | string  | La codificación de la sal proporcionada. Debe ser una de las siguientes: <ul><li>`base64`</li><li>`hex`</li><li>`utf8`</li></ul> Se admiten variantes hexadecimales en mayúsculas y minúsculas, así como base64 codificado para URL. De forma predeterminada, `salt.encoding` es `utf8`.                                                                                                                                                                                                      |
| `salt.position`     | string  | La posición de la sal cuando se calculó el hash. De forma predeterminada, `salt.position` es `prefix`.                                                                                                                                                                                                                                                                                                                                                                                        |
| `password.encoding` | string  | La codificación de la contraseña usada para generar el hash. Debe ser una de las siguientes: <ul><li>`ascii`</li><li>`utf8`</li><li>`utf16le`</li><li>`ucs2`</li><li>`latin1`</li><li>`binary`</li></ul> Al iniciar sesión, la contraseña proporcionada por el usuario se transcodificará desde `password.encoding` antes de comprobarse con el hash proporcionado. Por ejemplo, si tu hash se generó a partir de una cadena codificada en `ucs2`, deberías establecer: ` "encoding": "ucs2"` |
| `keylen`            | integer | Longitud de clave deseada, en bytes, para el hash scrypt. Debe ser un entero mayor que cero.<br />Este parámetro es obligatorio cuando `algorithm` está establecido en `scrypt`.                                                                                                                                                                                                                                                                                                              |
| `cost`              | integer | Parámetro de costo de CPU/memoria usado para el hash scrypt. Debe ser una potencia de dos mayor que uno. De forma predeterminada, `cost` es 16384.<br />Este parámetro solo se usa cuando `algorithm` está establecido en `scrypt`.                                                                                                                                                                                                                                                           |
| `blockSize`         | integer | Parámetro de tamaño de bloque usado para el hash scrypt. Debe ser un entero positivo. De forma predeterminada, `blockSize` es 8.<br />Este parámetro solo se usa cuando `algorithm` está establecido en `scrypt`.                                                                                                                                                                                                                                                                             |
| `parallelization`   | integer | Parámetro de paralelización usado para el hash scrypt. Debe ser un entero positivo. De forma predeterminada, `parallelization` es 1.<br />Este parámetro solo se usa cuando `algorithm` está establecido en `scrypt`.                                                                                                                                                                                                                                                                         |

<div id="update-custom-password-hash">
  ### Actualizar el hash de contraseña personalizado
</div>

Durante el proceso de importación masiva, puedes actualizar `custom_password_hash` si el usuario no inició sesión con el `custom_password_hash` importado inicialmente. Por ejemplo, puedes enviar el siguiente JSON dos veces al endpoint [`/api/v2/jobs/users-imports`](https://auth0.com/docs/api/management/v2?_ga=2.169020323.346193905.1601302257-746691936.1587131255#!/Jobs/post_users_imports) con valores diferentes para `custom_password_hash`. En el segundo envío, establece el indicador `upsert` en `true`.

```json lines
[
    {
    	"user_id": "2000",
        "email": "examplecouser20@gmail.com",
        "given_name": "ExampleCo User",
        "name" : "ExampleCoUser20",
        "custom_password_hash": {
            "algorithm": "bcrypt",
            "hash": {
                "value": "$2a$10$aHF7mbpWT6tZ7PJVtwtjNelaKbszikcYBCB2jibvbFcGFmOsu/s4K"
            }
        }
    }
]
```

Puedes usar el [Bcrypt Password Generator](https://www.browserling.com/tools/bcrypt) de browserling.com para generar hashes de contraseña con bcrypt.

<div id="supported-hash-algorithms">
  ### Algoritmos de hash compatibles
</div>

Auth0 actualmente admite la importación de contraseñas de usuario con hash generado mediante:

* [Argon2](https://github.com/p-h-c/phc-winner-argon2)
* [bcrypt](https://auth0.com/blog/hashing-in-action-understanding-bcrypt/)
* [LDAP](https://tools.ietf.org/html/rfc2307#section-5.3) (`RFC-2307 "userPassword"`)
* [HMAC](https://tools.ietf.org/html/rfc2104)
* [MD4](https://tools.ietf.org/html/rfc1320)
* [MD5](https://tools.ietf.org/html/rfc1321)
* [SHA1](https://tools.ietf.org/html/rfc3174)
* [SHA256 y SHA512](https://tools.ietf.org/html/rfc4634)
* [PBKDF2](https://tools.ietf.org/html/rfc2898#section-5.2)
* [scrypt](https://datatracker.ietf.org/doc/rfc7914/)

Tenga en cuenta las siguientes secciones al proporcionar un `custom_password_hash`.

<div id="argon2">
  #### Argon2
</div>

Cuando `algorithm` se establece en `argon2`:

* `hash.encoding` debe ser `utf8`.
* `hash.salt` no se permite.
* `hash.value` debe estar en formato de cadena PHC, como se especifica en [P-H-C / phc-string-format](https://github.com/P-H-C/phc-string-format/blob/master/phc-sf-spec.md) en GitHub. También debe cumplir los requisitos especificados en [Auth0 / magic](https://github.com/auth0/magic#magicpasswordhash--magicverifypassword) en GitHub.
* `hash.value` debe incluir la `sal` codificada en base64 (como se especifica en la documentación de `PHC`).

<div id="bcrypt">
  #### bcrypt
</div>

Cuando `algorithm` se establece en `bcrypt`:

* `hash.encoding` debe ser `utf8`.
* `hash.salt` se permite junto con la codificación y la posición de sal.
* `hash.value` debe incluir uno de estos prefijos:

  * `$2a$`
  * `$2b$`
  * `$2y$`

  Otros prefijos, como `$2$`, `$sha1$` y `$2x$`, no son compatibles por el momento.

Por ejemplo, esto se generó a partir de la cadena `hello` con un parámetro de costo de 10:

`$2b$10$nFguVi9LsCAcvTZFKQlRKeLVydo8ETv483lkNsSFI/Wl1Rz1Ypo1K`

El algoritmo `bcrypt` procesa un máximo de 72 bytes de entrada al calcular hashes de contraseña o realizar comparaciones, y la longitud de `salt.value` se contabiliza dentro del límite de entrada de 72 bytes. Cualquier entrada que supere el límite de 72 bytes se trunca; por ejemplo, si la sal ocupa 10 bytes, la longitud máxima de la contraseña para generar el hash o para la comparación es de 62 bytes.

Las contraseñas que superen este límite reducido se truncan, lo que puede debilitar la fortaleza de la contraseña o provocar colisiones de hash. Valida siempre la longitud de las contraseñas antes de generar el hash.

<div id="hmac">
  #### HMAC
</div>

Cuando `algorithm` se configura como `hmac`:

* `hash.encoding` debe ser `hex` o `base64`.
* `hash.digest` es obligatorio y debe ser uno de los siguientes:

  * `md4`
  * `md5`
  * `ripemd160`
  * `sha1`
  * `sha224`
  * `sha256`
  * `sha384`
  * `sha512`
  * `whirlpool`
* `hash.key.value` es obligatorio.
* `hash.key.encoding` debe ser `base64`, `hex` o `utf8`.

<div id="ldap">
  #### LDAP
</div>

Cuando `algorithm` se establece en `ldap`:

* `hash.encoding` debe ser `utf8`.
* `salt` no está permitido.
* `hash.value` debe ajustarse al formato descrito en la [sección 5.3 del RFC-2307](https://tools.ietf.org/html/rfc2307#section-5.3) en IETF Datatracker.
* El esquema debe ser uno de `md5|smd5|sha*|ssha*`; consulta [aquí](https://www.openldap.org/faq/data/cache/347.html) para obtener más información.
* Ten en cuenta que el esquema [crypt](https://www.openldap.org/faq/data/cache/344.html) **no es compatible** debido a un comportamiento que depende del sistema o de la implementación. Para obtener más información, consulta [Open LDAP Admin Guide - 14.4.2. CRYPT password storage scheme](https://www.openldap.org/doc/admin24/guide.html#CRYPT%20password%20storage%20scheme).

<div id="md-or-sha">
  #### MD o SHA
</div>

Cuando `algorithm` está configurado como `md4`, `md5`, `sha1`, `sha256` o `sha512`:

* `hash.encoding` debe ser `hex` o `base64`.

<div id="pbkdf2">
  #### PBKDF2
</div>

Cuando `algorithm` se establece en `pbkdf2`:

* `hash.encoding` debe ser `utf8`.
* No se permite `hash.salt`.
* `hash.value` debe estar en formato de cadena PHC, tal como se especifica en [P-H-C / phc-string-format](https://github.com/P-H-C/phc-string-format/blob/master/phc-sf-spec.md) en GitHub.
* `hash.value` debe incluir la sal codificada en B64 (base64, omitiendo los caracteres de relleno `=`, tal como se especifica en la documentación de `PHC`).
* `hash.value` debería incluir los parámetros `i` (iteraciones) y `l` (keylen). Si se omiten estos parámetros, se usarán por defecto `i=100000` y `l=64`.
* El `id` debe tener el formato `pbkdf2-<digest>` (`pbkdf2-sha512`, `pbkdf2-md5`, etc.). Los resúmenes admitidos son:

  * `RSA-MD4`
  * `RSA-MD5`
  * `RSA-MDC2`
  * `RSA-RIPEMD160`

<div id="scrypt">
  #### scrypt
</div>

Cuando `algorithm` se configura como `scrypt`:

* `hash.encoding` debe ser `hex` o `base64`.
* El parámetro `keylen` es obligatorio.
* Se puede especificar el parámetro `cost`; si no se hace, el valor predeterminado será 16384.
* Se puede especificar el parámetro `blockSize`; si no se hace, el valor predeterminado será 8.
* Se puede especificar el parámetro `parallelization`; si no se hace, el valor predeterminado será 1.

<div id="mfa-factors">
  ### Factores de MFA
</div>

El arreglo `user.mfa_factors` contiene las inscripciones en <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación del usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> del usuario. Para obtener más información, consulta [Autenticación multifactor en Auth0](/docs/es/secure/multi-factor-authentication). Importar estas inscripciones evita que los usuarios tengan que volver a inscribirse en MFA después de ser importados. Los tipos de inscripción admitidos son:

| Propiedad     | Tipo   | Descripción                                                                                                                                                                                                                            |
| ------------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `email`       | object |                                                                                                                                                                                                                                        |
| `email.value` | string | La dirección de correo electrónico para MFA.                                                                                                                                                                                           |
| `phone`       | object |                                                                                                                                                                                                                                        |
| `phone.value` | string | El número de teléfono para MFA por SMS. Debe incluir un código de país y comenzar con `+`, por ejemplo: `"+12125550001"`                                                                                                               |
| `totp`        | object |                                                                                                                                                                                                                                        |
| `totp.secret` | string | El secreto OTP para la autenticación MFA con aplicaciones autenticadoras (Google Authenticator, Microsoft Authenticator, Authy, 1Password, LastPass). Debe estar en codificación Base32 sin relleno; por ejemplo: `"JBTWY3DPEHPK3PNP"` |
