<Card title="Antes de empezar">
  Para usar esta funcionalidad, debes contar con el rol **Editor - Users** o **Admin**. Para obtener más información sobre los roles del Dashboard, consulta [Dashboard Access by Role](/docs/es/get-started/manage-dashboard-access/feature-access-by-role).
</Card>

Puedes importar y exportar datos de usuario de forma masiva en Auth0 desde el Dashboard. Las importaciones y exportaciones masivas son útiles para migrar usuarios desde una base de datos existente a Auth0 o desde Auth0 a otro servicio. Todas las tareas iniciadas desde el Dashboard se registran y se rastrean mediante IDs de tarea.

Para ver una lista de los campos del perfil de usuario que se pueden importar y exportar, consulta [Perfiles de usuario](/docs/es/manage-users/user-accounts/user-profiles).

<div id="import-users">
  ### Importar usuarios
</div>

Para importar usuarios a tu tenant:

1. En el Auth0 Dashboard, ve a [**User Management &gt; Users**](https://manage.auth0.com/#/users). Si no hay usuarios en tu tenant, elige **Import Users**. Si ya hay usuarios, elige **Import/Export Users &gt; Import Users**.
2. Carga un archivo JSON (tamaño máximo: 500 KB) con el formato indicado en las [especificaciones disponibles aquí](/docs/es/manage-users/user-migration/bulk-user-import-database-schema-and-examples).
3. Selecciona la conexión de base de datos en la que quieres importar los usuarios. Ten en cuenta que solo se admiten conexiones de base de datos y que la base de datos personalizada que elijas debe tener habilitado el modo de importación para al menos una aplicación.

   1. (Opcional) Selecciona **Actualizar usuarios preexistentes en la conexión** para actualizar los usuarios existentes con identificadores coincidentes
   2. (Opcional) Selecciona **Send completion email to all tenant owners**.
4. Elige **Import Users** para enviar la tarea. Envía solo una tarea por cada importación.

Para supervisar las tareas que enviaste, ve a **View Jobs** en la pantalla Import/Export Users y elige **More info** en cualquier tarea. Los estados de las tareas incluyen **Job creation failed**, **Job created, user import failed** y **Job created, user import succeeded**.

<div id="export-users">
  ### Exportar usuarios
</div>

<Frame>![](/docs/images/cdy7uua7fh8z/2koXwmAu99VfxJIq49yGGq/394be27431b66cfd09866ef91b0e0cd0/exportusers.png)</Frame>

Para exportar usuarios desde tu tenant:

1. En el Dashboard, ve a **User Management &gt; Users,** y luego selecciona **Import/Export Users &gt; Export Users**.
2. Selecciona la conexión de base de datos de la que quieres exportar usuarios y elige JSON o CSV como formato de exportación.
3. Especifica qué campos exportar:

   1. Selecciona **Add default fields** para incluir los atributos de usuario estándar.
   2. Actualiza los nombres de los campos de tu archivo de salida.
   3. Agrega atributos de usuario personalizados según sea necesario.
4. (Opcional) Selecciona **Export as sample** para ver una vista previa de la estructura con hasta 10 usuarios.
5. Selecciona **Export Users** para enviar la tarea.

Para supervisar y descargar tareas de exportación, ve a **View jobs** en la pantalla **Import/Export Users**. Selecciona **Check Status** para supervisar el progreso de la tarea, **Download export** para descargar el resultado o **More info** para revisar errores o metadatos de la tarea.

<div id="update-user-profiles">
  ### Actualizar perfiles de usuario
</div>

La [Auth0 Management API](https://auth0.com/docs/api/management/v2/users/patch-users-by-id) permite actualizar en bloque los perfiles de usuario en `user_metadata` y `app_metadata`.

Este proceso puede provocar la pérdida de datos; cree siempre una copia de seguridad de su base de datos de usuarios en caso de que se produzca algún daño en los datos. Además, este proceso no permite bloquear a varios usuarios a la vez, y cada usuario debe bloquearse individualmente en el <Tooltip tip="" cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o con la [Auth0 Management API](https://auth0.com/docs/api/management/v2/users/patch-users-by-id).

Para actualizar perfiles de usuario de forma masiva:

1. Mediante el proceso de exportación anterior, cree un archivo de exportación completo que incluya todos los atributos de sus usuarios.
2. Elija el formato JSON para la exportación.
3. Cuando la exportación se complete, descargue el archivo que contiene sus usuarios.

###

<div id="update-your-configuration">
  ### Actualiza tu configuración
</div>

<Warning>
  No cambies la dirección de correo electrónico del usuario, ya que es la clave de búsqueda de los usuarios en la base de datos.
</Warning>

Haz una copia del archivo JSON que descargaste en el paso anterior y luego ábrela en un editor de texto. Tu esquema de usuario debería ser similar al siguiente:

```json lines expandable
[
   {
      "Id":"auth0|626737a9813c000069daf462",
      "Nickname":"testuser1",
      "Name":"testuser1@auth0.com",
      "email":"testuser1@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-04-26T00:07:05.098Z",
      "Updated At":"2022-07-06T14:54:19.214Z"
   },
   {
      "Id":"auth0|62d1a1853bdcf11ff5dca672",
      "Nickname":"testuser2",
      "Name":"testuser2@auth0.com",
      "email":"testuser2@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-07-15T17:19:01.531Z",
      "Updated At":"2022-07-15T17:19:01.532Z"
   },
   {
      "Id":"auth0|62d1a1a524f32a4185c9ce23",
      "Nickname":"testuser3",
      "Name":"testuser3@auth0.com",
      "email":"testuser3@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-07-15T17:19:33.484Z",
      "Updated At":"2022-07-15T17:19:33.484Z"
   }
]
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El Id debe tener un `connection|id` único. Si el mismo Id de usuario existe en varias conexiones, la actualización fallará.
</Callout>

Para agregar un atributo, crea un nuevo campo en cada esquema de usuario y actualízalo con un valor. En el ejemplo siguiente, el atributo **Apellido** tiene el valor **Smith**, con una coma después del valor del atributo nuevo.

También puedes agregar el atributo `email_verified` a varios usuarios y establecer su valor en `true`.

```json lines
{
   "Id":"auth0|626737a9813c000069daf462",
   "Nickname":"testuser1",
   "Family Name":"Smith",
   "Name":"testuser1@auth0.com",
   "email":"testuser1@auth0.com",
   "email_verified":true,
   "Connection":"Username-Password-Authentication",
   "Created At":"2022-04-26T00:07:05.098Z",
   "Updated At":"2022-07-06T14:54:19.214Z"
}
```

Guarda los cambios en tu archivo de usuarios.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este proceso no permite bloquear a varios usuarios. Debes bloquearlos individualmente en el Auth0 Dashboard o con la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).
</Callout>

Para importar datos actualizados del perfil de usuario, sube tu archivo de perfil de usuario actualizado y actualiza varios usuarios en una sola tarea. **Actualizar usuarios preexistentes en la conexión** debe estar en True/seleccionada para que se actualicen los usuarios existentes.
