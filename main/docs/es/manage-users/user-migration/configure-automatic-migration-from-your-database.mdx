description: Aprende a habilitar la migración automática de usuarios con tu base de datos personalizada.
title: Configura la migración automática desde tu base de datos
---

Configura tu conexión de base de datos para migrar automáticamente a tus usuarios desde un repositorio de usuarios externo a Auth0 con [scripts de acción de base de datos personalizados](/docs/es/authenticate/database-connections/custom-db/templates).

<Warning>
  Si intentas usar más de un método de migración (por ejemplo, migración automática y luego importación masiva de usuarios), es posible que te encuentres con un error `DUPLICATED_USER`. Este error indica que el usuario existe en el repositorio de usuarios interno de Auth0, pero no en tu tenant. Para corregir este error, elimina al usuario con el endpoint [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de la Auth0 Management API y luego vuelve a intentar la importación.
</Warning>

<div id="configure-database-connection-as-custom">
  ## Configurar la conexión de base de datos como personalizada
</div>

Cree una conexión de base de datos y luego configúrela como personalizada.

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y seleccione la base de datos que desea ver.
2. Seleccione la vista **Custom Database** y habilite **Use my own database**.

   <Frame>![Auth0 Dashboard Authentication Database Connection Custom Database Settings Use Own Database Enabled](/docs/images/cdy7uua7fh8z/11HPAdVwJMmnWbzMVjHCJ8/f75ff47f613a41778f266643f29ab2b1/2025-02-27_17-25-36.png)</Frame>
3. Seleccione la vista **Settings**, habilite **Import Users to Auth0** y seleccione **Save**.

<div id="configure-database-action-scripts">
  ## Configurar scripts de acción de base de datos
</div>

Selecciona la vista **Custom Database** y luego ubica **Scripts de acción de base de datos**.

<Frame>![Dashboard Autenticación conexión de base de datos pestaña Custom Database Scripts de acción de base de datos](/docs/images/cdy7uua7fh8z/4LBIvvjaABo51It4eVCmX3/4ef14765c79cc5b7ecfa3f3155caa9eb/Screen_Shot_2021-05-18_at_8.54.55_PM.png)</Frame>

<div id="login">
  ### Login
</div>

El script [Login](/docs/es/authenticate/database-connections/custom-db/templates/login) se ejecuta cada vez que un usuario que no se encuentra en Auth0 intenta iniciar sesión. Verifica que el usuario exista en el repositorio de usuarios externo sin volver a pedirle la contraseña.

<div id="get-user">
  ### Get User
</div>

El script [Get User](/docs/es/authenticate/database-connections/custom-db/templates/get-user) se ejecuta en cualquiera de los siguientes casos:

* Un usuario inicia un proceso de registro.
* Un usuario inicia un flujo para cambiar la contraseña.
* Un usuario introduce su identificador de inicio de sesión al usar Identifier First + Biometrics.
* Se invoca el endpoint [Update a User&#39;s Email or Username](https://auth0.com/docs/api/v2#!/Users/patch_users_by_id) o [Crear usuario](https://auth0.com/docs/api/management/v2#!/Users/post_users) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Si un usuario que no ha sido migrado confirma el cambio de contraseña e inicia sesión correctamente, su perfil de usuario se creará en Auth0 con la nueva contraseña. Este perfil de usuario contiene todos los datos devueltos por el script Get User. Todos los inicios de sesión posteriores de este usuario se realizarán directamente en Auth0.

<div id="verify-user-migration-is-complete">
  ## Verifica que la migración de usuarios se haya completado
</div>

Verifica que todos los usuarios se hayan migrado al repositorio de usuarios de Auth0 mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la Auth0 Management API.

1. Ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y revisa la lista de usuarios.
2. Usa el endpoint [List or Search Users](https://auth0.com/docs/api/v2#!/Users/get_users) de la Management API.

<div id="disconnect-external-user-store">
  ## Desconectar el repositorio de usuarios externo
</div>

Vuelve a configurar los scripts de acción de base de datos como funciones que no realizan ninguna operación. Esto evitará que Auth0 acceda a tu repositorio de usuarios externo para autenticar a los usuarios.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Asegúrate de dejar activada la opción **Import Users to Auth0**. Si desactivas esta opción, Auth0 solo usará los scripts para autenticar y realizar otras acciones de usuario, en lugar de usar los usuarios importados localmente.
</Callout>

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y, a continuación, selecciona tu conexión de base de datos.

2. Cambia a la vista **Custom Database** y localiza **Scripts de acción de base de datos.**

3. Actualiza el script **Login**.

   ```javascript lines
   function login (email, password, callback) {
     return callback(null, null);
   }
   ```

4. Actualiza el script **Get User**.

   ```javascript lines
   function getByEmail (email, callback) {
     return callback(null, null);
   }
   ```

<div id="troubleshoot-user-migration-issues">
  ## Solucionar problemas de migración de usuarios
</div>

Si tienes algún problema con la migración automática, primero consulta [Prácticas recomendadas para conexiones de base de datos personalizadas y scripts de acción](/docs/es/authenticate/database-connections/custom-db/custom-database-connections-scripts).

<div id="user-already-exists">
  ### El usuario ya existe
</div>

Estos son los casos más comunes en los que puedes encontrarte con este problema:

* Eliminas un usuario de tu conexión de base de datos y luego intentas volver a crearlo.
* Intentas crear un usuario nuevo en tu conexión de base de datos cuando el usuario ya existe en tu repositorio externo de usuarios.
* Se interrumpió el proceso de migración de usuarios.

Durante el proceso de migración de usuarios, Auth0 primero crea un perfil de usuario parcial en un repositorio interno de usuarios y luego crea un perfil de usuario completo en tu conexión de base de datos. Si se produce algún problema que impida crear ese perfil de usuario completo, es posible que aparezca el error `The user already exists`.

Normalmente, puedes resolver este problema eliminando al usuario de la conexión de base de datos, eliminándolo del repositorio interno de usuarios y volviendo a intentar el proceso de migración.

1. Revisa las sentencias `console.log()` en los [logs en tiempo real de Actions](/docs/es/customize/actions/actions-real-time-logs).
2. Elimina al usuario con el endpoint [Delete a User](https://auth0.com/docs/api/management/v2#!/Users/delete_users_by_id) de la Management API.
3. Elimina al usuario con el endpoint [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de la Management API.
4. Indica al usuario que inicie sesión o cambie su contraseña para volver a intentar la migración.

<div id="missing-metadata">
  ### Metadatos ausentes
</div>

Si la sesión de un usuario se interrumpe durante el inicio de sesión o el flujo de cambio de contraseña que inicia el proceso de migración, es posible que Auth0 no pueda transferir sus metadatos (`user_metadata` o `app_metadata`) junto con el resto de los datos de su perfil.

Puede mitigar este problema creando una [Action](/docs/es/customize/actions) que verifique si faltan metadatos en el perfil del usuario, los recupere del almacén de usuarios externo y luego los guarde en Auth0.
