Puede usar el endpoint `POST /api/v2/jobs/users-exports` para crear una tarea que exporte todos los usuarios asociados a una [conexión](/docs/es/authenticate/identity-providers) o todos los usuarios del tenant.

Para obtener una lista de los campos del perfil de usuario que se pueden exportar, consulte [Estructura del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/user-profile-structure).

Cuando cree la tarea, deberá proporcionar:

* El [ID de la conexión](/docs/es/authenticate/identity-providers/locate-the-connection-id) desde la que quiere exportar usuarios (opcional)
* El formato del archivo de exportación (CSV o compatible con JSON)
* El número máximo de registros de usuario que se exportarán (opcional; si se omite, se exportarán todos los registros)
* Los campos relacionados con el usuario (como el ID de usuario o el nombre) que quiere incluir en la exportación

También necesitará un [token de acceso de la Management API](https://auth0.com/docs/api/management/v2/tokens).

<div id="create-a-request-body">
  ## Crear el cuerpo de la solicitud
</div>

De forma opcional, busca el `connection_id` y el nombre de dominio de tu tenant de Auth0 en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Crea un nuevo archivo de texto con el cuerpo de la solicitud que aparece a continuación:

```json lines
{
   "connection_id":"connection_id",
   "format":"csv",
   "limit":20,
   "fields":[
      {
         "name":"user_id"
      },
      {
         "name":"email"
      },
      {
         "name":"user_metadata.country"
      }
   ]
}
```

Actualiza el `connection_id` con el ID de tu conexión de base de datos o elimínalo para exportar todos los usuarios del tenant.

<div id="include-user-metadata-in-exported-csv">
  ## Incluir metadatos de usuario en el CSV exportado
</div>

Si exporta datos de usuario en formato CSV y quiere incluir metadatos, especifique cada campo de metadatos que desee exportar. Puede exportar hasta 30 campos.

<Warning>
  No puede exportar la totalidad de `app_metadata` ni de `user_metadata` en CSV. Debe especificar explícitamente los campos de los objetos de metadatos.

  Para exportar `app_metadata` o `user_metadata` como objetos individuales, use el formato compatible con JSON e incluya el campo deseado en el parámetro `fields` del cuerpo de la solicitud. Por ejemplo:

  `{"name":"app_metadata"}`

  Como solo puede exportar hasta 30 campos, se recomienda usar el formato JSON si los datos de usuario tienen muchos campos.
</Warning>

Por ejemplo, para metadatos con esta estructura:

```json lines
{
  "consent": {
      "given": true,
      "date": "01/23/2019",
      "text_details": "{yourURL}"
  }
}
```

La solicitud de exportación (para los tres campos) sería así:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En los archivos CSV de exportación de usuarios, aplicamos escape a los tipos de datos string de conformidad con los estándares de OWASP para mitigar la inyección en CSV:

  * A los caracteres de comillas dobles se les antepone otro carácter de comillas dobles.
  * A cada string se le antepone un carácter de comilla simple.
  * Cada string va entre comillas dobles.

  Esto no se aplica a las fechas generadas por Auth0 en formato ISO 8601.
</Callout>

<div id="json-compatible-format">
  ### Formato compatible con JSON
</div>

Si exportas los datos en un formato compatible con JSON, solo necesitas indicar la propiedad raíz; no hace falta nombrar cada propiedad interna por separado, ya que se incluirán automáticamente.

Los archivos de exportación de Auth0 usan el formato [NDJSON](https://github.com/ndjson/ndjson-spec) debido a su gran tamaño, mientras que la función de importación espera un archivo JSON.

Antes de poder importar usuarios mediante una exportación generada por Auth0, tendrás que convertir el archivo de NDJSON a JSON usando la biblioteca que prefieras (como [jq](https://stedolan.github.io/jq/)).

En este caso, para el mismo ejemplo que usamos antes, la petición tendría este aspecto:

<div id="check-export-status">
  ## Consultar el estado de la exportación
</div>

Una vez que hayas creado tu tarea para exportar usuarios, puedes consultar su estado mediante el [endpoint Get a Job](https://auth0.com/docs/api/management/v2#!/Jobs/get_jobs_by_id).

Proporciona el ID de la tarea (que recibiste en la respuesta al crearla). Si usas la solicitud de ejemplo que aparece a continuación, reemplaza el marcador de posición `{yourJobId}` por el valor del ID.

Scopes requeridos: `create:users`, `read:users`, `create:passwords_checking_job`

<div id="find-export-data">
  ## Encontrar datos exportados
</div>

Puedes acceder a tus archivos exportados mediante la URL proporcionada como valor del parámetro **location**. El nombre de tu tenant también es el nombre de tu archivo. Por ejemplo, si el nombre de tu tenant es `auth0docs`, tu archivo será `auth0docs.csv` o `auth0docs.json`. Al acceder a la URL, la descarga del archivo comenzará automáticamente.

El enlace de descarga es válido durante 60 segundos. Si ese plazo vence, tendrás 24 horas para volver a usarlo antes de que la tarea caduque.

<div id="job-cleanup">
  ## Limpieza de tareas
</div>

Todos los datos relacionados con sus tareas se eliminan automáticamente después de 24 horas y ya no se puede acceder a ellos. Por ello, **recomendamos encarecidamente guardar los resultados de la tarea mediante el mecanismo de almacenamiento que prefiera**.

<div id="filter-example">
  ## Ejemplo de filtro
</div>

Puede que quieras filtrar el archivo `.csv` exportado para obtener un subconjunto concreto de tus datos, como la fecha en que los usuarios iniciaron sesión por última vez en tu aplicación.

1. Haz una llamada `POST` al endpoint [Create export user&#39;s job](https://auth0.com/docs/api/management/v2/jobs/post-users-exports) de Management API para exportar usuarios a un archivo `.csv`:

   1. Crea una nueva tarea:

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/users-exports' \
      -H 'Content-Type: application/json' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN' \
      -d '{"format":"csv"}'
      ```

   2. Comprueba el estado:

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/JOB_ID' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN'
      ```

2. Ve al archivo en la carpeta de descargas de tu equipo local.

3. Usa una herramienta de línea de comandos para añadir un filtro. En este ejemplo, usamos [Miller](https://miller.readthedocs.io/en/latest/installing-miller/):

   ```bash wrap lines
   mlr --csv filter '$last_login >= "2024-01-01T00:00:00Z"' file.csv > filtered.csv
   ```
