---
description: Comprenda el concepto de las políticas de autorización y cómo se aplican en Auth0.
title: Políticas de autorización
---

Entre bastidores, la autorización basada en roles utiliza una política de autorización preconfigurada que contiene condiciones que permiten al código evaluar si se debe permitir que un usuario acceda a una API protegida.

La política de autorización determina:

* cómo definir y organizar los usuarios o roles a los que afecta la política
* qué lógica y condiciones se aplican a la política y si su resultado permite o deniega el acceso

Al utilizar las funciones principales de autorización de Auth0 y el [control de acceso basado en roles (RBAC)](/docs/es/manage-users/access-control/rbac), la política incluye evaluar los roles y permisos asignados a los usuarios. Para usar estas funcionalidades, debe [habilitar el control de acceso basado en roles para las API](/docs/es/get-started/apis/enable-role-based-access-control-for-apis).

Puede personalizar aún más la política de autorización mediante [Rules](/docs/es/customize/rules). Para obtener más información, consulte [Rules para políticas de autorización](/docs/es/manage-users/access-control/rules-for-authorization-policies).
