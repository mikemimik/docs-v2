description: Conozca las diferencias entre la versión principal de RBAC de Auth0 y Authorization Extension.
title: Authorization Core vs. Authorization Extension
---

<Warning>
  Tenemos previsto declarar obsoletos Rules y Hooks en 2024 y, a partir de ese momento, las Extensions basadas en Rules y Hooks dejarán de recibir soporte. Le recomendamos encarecidamente migrar a Actions para extender Auth0. Con Actions, tiene acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad.

  Para obtener más información sobre lo que ofrece Actions, lea [Comprenda cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview). También contamos con una página dedicada de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demo de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarle en su proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada de blog: [Preparándose para el fin de vida útil de Rules y Hooks.](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/)
</Warning>

Auth0 ofrece actualmente dos formas de implementar [control de acceso basado en roles (RBAC)](/docs/es/manage-users/access-control/rbac): nuestra implementación Core y nuestra Authorization Extension. Nuestra implementación Core mejora el rendimiento y la escalabilidad.

Recomendamos usar Authorization Core para la mayoría de las implementaciones. Si desea representar equipos, clientes empresariales o partners en una aplicación B2B o SaaS, le recomendamos representarlos como [Organizations](/docs/es/manage-users/organizations/organizations-overview) y usar Authorization Core. Authorization Extension no es compatible con Organizations.

Para ayudarle a decidir qué funcionalidad es la adecuada para su implementación, le presentamos las diferencias entre ambas:

| Funcionalidad                                                                                                        | Authorization Core                                                                                           | Authorization Extension                                                                                                                                                        |
| -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Rendimiento y escalabilidad mejorados                                                                                | Sí - Lea [política de límite de entidad](/docs/es/troubleshoot/customer-support/operational-policies/entity-limit-policy) | No - Limitado a 500 KB de datos (1000 grupos, 3000 usuarios, donde cada usuario es miembro de 3 grupos; o 20 grupos, 7000 usuarios, donde cada usuario es miembro de 3 grupos) |
| Crear/editar/eliminar roles                                                                                          | Sí                                                                                                           | Sí                                                                                                                                                                             |
| Los roles pueden contener permisos de una o varias API                                                               | Sí                                                                                                           | No                                                                                                                                                                             |
| Se pueden asignar usuarios y roles a grupos                                                                          | No                                                                                                           | Sí                                                                                                                                                                             |
| Los roles están asociados a aplicaciones específicas                                                                 | No                                                                                                           | Sí                                                                                                                                                                             |
| Crear/editar/eliminar usuarios                                                                                       | Sí                                                                                                           | Sí                                                                                                                                                                             |
| Buscar usuarios por usuario, correo electrónico, conexión                                                            | Sí                                                                                                           | Sí                                                                                                                                                                             |
| Buscar usuarios por proveedor de identidad, número de inicios de sesión, último inicio de sesión, número de teléfono | Sí                                                                                                           | No                                                                                                                                                                             |
| Buscar usuarios mediante la sintaxis de Lucene                                                                       | Sí                                                                                                           | No                                                                                                                                                                             |
| Importación/exportación de usuarios mediante JSON                                                                    | Por el momento, no                                                                                           | Sí                                                                                                                                                                             |
| Crear políticas de autorización personalizadas                                                                       | Sí                                                                                                           | No                                                                                                                                                                             |
