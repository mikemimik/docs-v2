title: Authorization Core vs. Authorization Extension
---

<Warning>
  Tenemos previsto dejar obsoletos Rules y Hooks en 2024 y, en ese momento, las Extensions basadas en Rules y Hooks dejarán de recibir soporte. Le recomendamos encarecidamente que migre a Actions para extender Auth0. Con Actions, tiene acceso a información de tipos detallada, documentación integrada y paquetes públicos de `npm`, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad.

  Para obtener más información sobre lo que ofrece Actions, lea [Comprenda cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview). También tenemos una página dedicada de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparativas de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarle en su proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada del blog: [Preparándose para el fin de vida útil de Rules y Hooks.](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/)
</Warning>

Auth0 ofrece actualmente dos formas de implementar el [control de acceso basado en roles (RBAC)](/docs/es/manage-users/access-control/rbac): nuestra implementación principal y nuestra Authorization Extension. Nuestra implementación principal mejora el rendimiento y la escalabilidad.

Recomendamos usar Authorization Core para la mayoría de las implementaciones. Si busca representar equipos, clientes empresariales o partners en una aplicación B2B o SaaS, le recomendamos representarlos como [Organizations](/docs/es/manage-users/organizations/organizations-overview) y usar Authorization Core. Authorization Extension no es compatible con Organizations.

Para ayudarle a decidir qué funcionalidad es la adecuada para su implementación, presentamos las diferencias entre ambas:

| Funcionalidad                                                                                                     | Authorization Core                                                                                                       | Authorization Extension                                                                                                                                                       |
| ----------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Rendimiento y escalabilidad mejorados                                                                             | Sí: lea [Política de límites de entidades](/docs/es/troubleshoot/customer-support/operational-policies/entity-limit-policy) | No: limitado a 500 KB de datos (1000 grupos, 3000 usuarios, donde cada usuario es miembro de 3 grupos; o 20 grupos, 7000 usuarios, donde cada usuario es miembro de 3 grupos) |
| Crear/editar/eliminar Roles                                                                                       | Sí                                                                                                                       | Sí                                                                                                                                                                            |
| Los Roles pueden contener permisos de una o más API                                                               | Sí                                                                                                                       | No                                                                                                                                                                            |
| Se pueden asignar usuarios y Roles a Groups                                                                       | No                                                                                                                       | Sí                                                                                                                                                                            |
| Los Roles están vinculados a applications específicas                                                             | No                                                                                                                       | Sí                                                                                                                                                                            |
| Crear/editar/eliminar Users                                                                                       | Sí                                                                                                                       | Sí                                                                                                                                                                            |
| Buscar Users por usuario, correo electrónico, conexión                                                            | Sí                                                                                                                       | Sí                                                                                                                                                                            |
| Buscar Users por proveedor de identidad, número de inicios de sesión, último inicio de sesión, número de teléfono | Sí                                                                                                                       | No                                                                                                                                                                            |
| Buscar Users usando sintaxis de Lucene                                                                            | Sí                                                                                                                       | No                                                                                                                                                                            |
| Importación/exportación de usuarios mediante JSON                                                                 | Actualmente no                                                                                                           | Sí                                                                                                                                                                            |
| Crear políticas de autorización personalizadas                                                                    | Sí                                                                                                                       | No                                                                                                                                                                            |
