Las [Actions](/docs/es/customize/actions) de Auth0 te permiten modificar o complementar el resultado de la decisión tomada por una [política de autorización](/docs/es/manage-users/access-control/authorization-policies) preconfigurada, para que puedas gestionar casos más complejos de los que es posible solo con el [control de acceso basado en roles (RBAC)](/docs/es/manage-users/access-control/rbac). Según el orden en que se ejecuten, las Actions pueden cambiar el resultado de una decisión de autorización antes de que se agreguen permisos al <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip>. También pueden permitirte personalizar el contenido de tus tokens.

<div id="allow-access-only-on-weekdays-for-a-specific-application">
  ## Permitir acceso solo entre semana para una aplicación específica
</div>

Digamos que tiene una aplicación y quiere asegurarse de que solo se pueda acceder a ella entre semana. [Cree una nueva Action](/docs/es/customize/actions/write-your-first-action) y seleccione el trigger `Login / Post Login`, porque va a añadir la Action al flujo de Login. Copie el siguiente código en el Actions Code Editor:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.client.name === "APP_NAME") {
    const d = new Date().getDay();

    if (d === 0 || d === 6) {
      api.access.deny("This app is only available during the week.");
    }
  }
}
```

Por último, agrega la Action que creaste al [flujo de Login](https://manage.auth0.com/#/actions/flows/login/). Para saber cómo adjuntar Actions a Flows, consulta la sección &quot;Adjuntar la Action a un flujo&quot; de [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

Si un usuario intenta acceder a la aplicación durante el fin de semana, se le denegará el acceso, aunque se autentique y tenga los privilegios adecuados.

<div id="allow-access-only-to-users-who-are-inside-the-corporate-network">
  ## Permite el acceso solo a los usuarios que estén dentro de la red corporativa
</div>

Supongamos que quieres permitir el acceso a una aplicación, pero solo a los usuarios que accedan a ella desde dentro de tu red corporativa. [Crea una nueva Action](/docs/es/customize/actions/write-your-first-action) y selecciona el trigger `Login / Post Login`, ya que añadirás la Action al flujo de Login. Copia el siguiente código en el Actions Code Editor:

```js lines
const ipaddr = require("ipaddr.js");

exports.onExecutePostLogin = async (event, api) => {
  const corpNetwork = "192.168.1.134/26";
  const currentIp = ipaddr.parse(event.request.ip);

  if (!currentIp.match(ipaddr.parseCIDR(corpNetwork))) {
    api.access.deny("This app is only available from inside the corporate network.");
  };
};
```

Por último, agrega la Action que creaste al [flujo de Login](https://manage.auth0.com/#/actions/flows/login/). Para aprender a asociar Actions a Flows, consulta la sección &quot;Adjuntar la Action a un flujo&quot; en [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

Si el usuario está fuera de la red corporativa, se le denegará el acceso aunque se autentique correctamente y tenga los privilegios adecuados.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar una `biblioteca` de `npm` como `ipaddr.js`, debes agregar la biblioteca como dependencia de la Action. Para obtener más información, consulta la sección &quot;Agregar una dependencia&quot; en [Write Your First Action](/docs/es/customize/actions/write-your-first-action).
</Callout>

<div id="deny-access-to-anyone-calling-an-api">
  ## Deniega el acceso a cualquiera que llame a una API
</div>

Supongamos que quieres denegar el acceso a todos los usuarios que llaman a una API. Esto significa que debes denegar el acceso en función del valor de `identifier` de tu API, que puedes encontrar en el campo **API <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (Client ID) para un token de ID o de una API (API Identifier) para un Access Token." cta="Ver glosario" href="/docs/es/glossary?term=Audience">Audiencia</Tooltip>** de tu API en [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis). [Crea una nueva Action](/docs/es/customize/actions/write-your-first-action) y selecciona el trigger `Login / Post Login`, ya que añadirás la Action al flujo de Login. Copia el siguiente código en el Actions Code Editor:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  // En Actions, una API se denomina Resource Server.
  const { identifier } = event.resource_server || {};
  if (identifier === "https://api.example.com") {
    api.access.deny("end_users_not_allowed");
  }
}
```

Por último, agrega la Action que creaste al [flujo de Login](https://manage.auth0.com/#/actions/flows/login/). Para aprender a adjuntar Actions a Flows, consulta la sección &quot;Adjuntar la Action a un flujo&quot; de [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

En este caso, el valor de `identifier` de la API es `https://api.example.com`, por lo que esta es la audiencia que rechazaremos.

<div id="add-user-roles-to-tokens">
  ## Agregar roles de usuario a los tokens
</div>

Para agregar roles de usuario a los tokens emitidos por Auth0, usa el objeto `event.authorization` junto con los métodos `api.idToken.setCustomClaim` y `api.accessToken.setCustomClaim`. [Crea una nueva Action](/docs/es/customize/actions/write-your-first-action) y selecciona el trigger `Login / Post Login`, ya que vas a añadir la Action al flujo de Login. Copia el siguiente código en el Actions Code Editor:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  const namespace = 'https://my-app.example.com';
  if (event.authorization) {
    api.idToken.setCustomClaim(`${namespace}/roles`, event.authorization.roles);
    api.accessToken.setCustomClaim(`${namespace}/roles`, event.authorization.roles);
  }
}
```

Por último, añade la Action que creaste al [flujo de Login](https://manage.auth0.com/#/actions/flows/login/). Para saber cómo vincular Actions a Flows, lee la sección &quot;Adjuntar la Action a un flujo&quot; de [Write Your First Action](/docs/es/customize/actions/write-your-first-action).

Recuerda:

* El <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> que se devuelve a la aplicación solicitante se genera y firma al final del procesamiento del trigger. No se puede acceder al JWT final ya firmado desde una Action.

<div id="deny-access-to-specific-ja3ja4-fingerprints">
  ## Denegar el acceso a huellas digitales JA3/JA4 específicas
</div>

El objeto `event.security_context` contiene los valores de las huellas digitales JA3/JA4 de la transacción actual.

[Crea una nueva Action](/docs/es/customize/actions/write-your-first-action) y selecciona el trigger `Login / Post Login`, ya que agregarás la Action al flujo de Login. Copia el siguiente código en el Actions Code Editor:

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const clientJa4 = event?.security_context?.ja4;
  console.log('[ACTION]', {clientJa4});
  const badFingerprints = ['t13d1517h2_8daaf6152771_b6f405a00624','t13d1516h2_8daaf6152771_d8a2da3f94cd'];
  if (clientJa4 && badFingerprints.includes(clientJa4)){
    api.access.deny('suspicious_tls_fingerprint', 'Your TLS fingerprint has been flagged as suspicious');
  }
};
```

Por último, agrega la Action que creaste al [flujo de Login](https://manage.auth0.com/#/actions/flows/login/). Para aprender a adjuntar Actions a Flows, lee la sección &quot;Adjuntar la Action a un flujo&quot; en [Escribe tu primera Action](/docs/es/customize/actions/write-your-first-action).
