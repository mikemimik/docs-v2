Auth0 te permite asignar y quitar [roles](/docs/es/manage-users/access-control/rbac) a grupos empresariales aprovisionados mediante <Tooltip tip="System for Cross-domain Identity Management (SCIM): Un protocolo estandarizado para el aprovisionamiento y la gestión de identidades de usuario y grupos entre sistemas." cta="Ver glosario" href="/docs/es/glossary?term=SCIM">SCIM</Tooltip> con Auth0 Dashboard o Management API. Cuando se asignan roles a un grupo, todos sus miembros heredan esos roles al iniciar sesión. Puedes gestionar los roles de grupo en dos ámbitos:

* **Grupos de organización** — grupos empresariales dentro de una [organización](/docs/es/manage-users/organizations). Requiere que [membresía automática](/docs/es/manage-users/organizations/configure-organizations/grant-just-in-time-membership) esté habilitado en la conexión empresarial para que la herencia de roles surta efecto.
* **Grupos empresariales** — grupos empresariales con alcance en tu tenant de Auth0.

<div id="assign-roles">
  ## Asignar roles
</div>

<div id="organization-groups">
  ### Grupos de organización
</div>

Asigna roles a los grupos empresariales aprovisionados mediante SCIM dentro de una organización. Los grupos empresariales se sincronizan desde un proveedor de identidad empresarial (como Okta o Microsoft Entra ID) a través de SCIM. Cuando un miembro de la organización inicia sesión, hereda los roles asignados a los grupos empresariales a los que pertenece.

La conexión empresarial de la organización debe tener la [membresía automática habilitada](/docs/es/manage-users/organizations/configure-organizations/grant-just-in-time-membership) para que la herencia de roles de grupo surta efecto. Para obtener más información sobre cómo se aprovisionan los grupos empresariales, consulta [Configurar SCIM entrante](/docs/es/authenticate/protocols/scim/configure-inbound-scim#use-groups-with-organizational-roles).

<Tabs>
  <Tab title="Dashboard">
    1. Ve a [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización.
    2. Selecciona la pestaña **Groups**.
    3. Selecciona el grupo al que quieres asignar roles.
    4. Selecciona **Assign Roles**.
    5. Busca y selecciona los roles que quieres asignar y, después, selecciona **Assign**.

    Los roles asignados aparecen en los detalles del grupo.
  </Tab>

  <Tab title="Management API">
    Haz una llamada `POST` al endpoint [Assign Organization Group Roles](/docs/es/api/management/v2). Necesitas la siguiente información:

    | Parámetro                    | Descripción                                                                                                                                            |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | `org_id`                     | ID de la organización.                                                                                                                                 |
    | `group_id`                   | ID del grupo empresarial.                                                                                                                               |
    | `role_id`                    | ID del rol de la organización que se va a asignar. Repítelo en el arreglo para asignar varios roles a la vez.                                          |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organization_group_roles`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/{ORG_ID}/groups/{GROUP_ID}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Si la solicitud se realiza correctamente, devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Grupos empresariales
</div>

Asigna roles a los Grupos empresariales aprovisionados mediante SCIM. Cuando se asigna un rol a un grupo, todos sus miembros heredan el rol asignado cuando inician sesión.

Para obtener información sobre cómo se aprovisionan los Grupos empresariales desde un proveedor de identidad empresarial, consulta [Configurar SCIM entrante](/docs/es/authenticate/protocols/scim/configure-inbound-scim).

<Tabs>
  <Tab title="Dashboard">
    1. Ve a [Dashboard &gt; User Management &gt; Grupos empresariales](https://manage.auth0.com/#/groups).
    2. Selecciona el grupo al que quieres asignar roles.
    3. Selecciona la pestaña **Roles** y, luego, **Assign Roles**.
    4. Busca y selecciona los roles que quieras asignar y, después, selecciona **Assign**.

    Los roles asignados aparecen en la pestaña **Roles** del grupo.
  </Tab>

  <Tab title="Management API">
    Haz una llamada `POST` al endpoint [Assign Roles to a Group](/docs/es/api/management/v2). Necesitas la siguiente información:

    | Parámetro                    | Descripción                                                                                                                          |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
    | `group_id`                   | ID del Grupo empresarial.                                                                                                            |
    | `role_id`                    | ID del rol que se va a asignar. Repítelo en el arreglo para asignar varios roles a la vez.                                           |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:groups`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/{group_id}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Una solicitud correcta devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="remove-roles">
  ## Eliminar roles
</div>

<div id="organization-groups">
  ### Grupos de organización
</div>

Elimine roles de grupos empresariales aprovisionados mediante SCIM dentro de una organización. Una vez que se elimina un rol, los miembros de la organización de ese grupo dejarán de heredarlo cuando inicien sesión.

La conexión empresarial de la organización debe tener la [membresía automática habilitada](/docs/es/manage-users/organizations/configure-organizations/grant-just-in-time-membership) para que la herencia de roles del grupo surta efecto.

<Tabs>
  <Tab title="Dashboard">
    1. Vaya a [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y seleccione la organización.
    2. Seleccione la pestaña **Groups**.
    3. Seleccione el grupo que desea actualizar.
    4. Busque el rol que desea eliminar y, a continuación, seleccione el icono de la papelera junto a él.
    5. Confirme la eliminación.

    El rol ya no aparece en los detalles del grupo.
  </Tab>

  <Tab title="Management API">
    Realice una llamada `DELETE` al endpoint [Delete Organization Group Roles](/docs/es/api/management/v2). Necesita la siguiente información:

    | Parámetro                    | Descripción                                                                                                                                            |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | `org_id`                     | ID de la organización.                                                                                                                                 |
    | `group_id`                   | ID del grupo empresarial.                                                                                                                              |
    | `role_id`                    | ID del rol de la organización que se va a eliminar. Repítalo en el arreglo para eliminar varios roles a la vez.                                        |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `delete:organization_group_roles`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/ORG_ID/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Una solicitud correcta devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Grupos empresariales
</div>

Elimina roles de los grupos empresariales aprovisionados con SCIM. Una vez que se elimina un rol, los miembros del grupo dejarán de heredarlo cuando inicien sesión.

<Tabs>
  <Tab title="Dashboard">
    1. Ve a [Dashboard &gt; User Management &gt; grupo empresarial](https://manage.auth0.com/#/groups).
    2. Selecciona el grupo que quieres actualizar.
    3. Selecciona la pestaña **Roles**.
    4. Ubica el rol que quieres eliminar y luego selecciona el icono de la papelera que aparece junto a él.
    5. Confirma la eliminación.

    El rol ya no aparece en la pestaña **Roles** del grupo.
  </Tab>

  <Tab title="Management API">
    Haz una solicitud `DELETE` al endpoint [Remove Roles from a Group](/docs/es/api/management/v2). Necesitas la siguiente información:

    | Parámetro                    | Descripción                                                                                                                          |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
    | `group_id`                   | ID del grupo empresarial.                                                                                                             |
    | `role_id`                    | ID del rol que se va a eliminar. Repítelo en el arreglo para eliminar varios roles a la vez.                                         |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:groups`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Una solicitud correcta devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>
