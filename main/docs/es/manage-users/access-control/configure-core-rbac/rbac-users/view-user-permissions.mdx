Puede consultar los [permisos](/docs/es/manage-users/access-control/rbac) asignados a un usuario desde el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Esta tarea también se puede realizar mediante la <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Los permisos asignados se usan con el conjunto de funcionalidades Authorization Core de la API.

<div id="prerequisites">
  ## Requisitos previos
</div>

Para que el control de acceso basado en roles (RBAC) funcione correctamente, debes habilitarlo para tu API desde el Dashboard o mediante la Management API. La funcionalidad de Authorization Core es diferente de la Authorization Extension. Para compararlas, lee [Authorization Core vs. Authorization Extension](/docs/es/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Dashboard
</div>

1. Ve a [Dashboard &gt; Administrar Users &gt; Users](https://manage.auth0.com/#/users) y haz clic en el nombre del usuario que quieres ver.
2. Haz clic en la vista **Permissions**.

   <Frame>![Auth0 Dashboard: detalle del usuario con la lista de permisos asignados a este usuario](/docs/images/cdy7uua7fh8z/4vShfxoUcSpCDnM5FnAbsR/de85942623430e3cedc097addbaa2f9d/User_Details_-_Permissions_-_English.png)</Frame>

Se muestra la siguiente información para cada permiso:

| Columna     | Descripción                                                                  |
| ----------- | ---------------------------------------------------------------------------- |
| Nombre      | Nombre del permiso según su definición.                                      |
| Descripción | Descripción del permiso según su definición.                                 |
| API         | Nombre de la API a la que pertenece el permiso.                              |
| Asignación  | Indica si el permiso se asignó directamente al usuario o a través de un rol. |

<div id="management-api">
  ## Management API
</div>

Haz una llamada `GET` al [endpoint Get User Permissions](https://auth0.com/docs/api/management/v2#!/Users/get_permissions). Asegúrate de reemplazar los valores de marcador de posición `USER_ID` y `MGMT_API_ACCESS_TOKEN` por tu ID de usuario y tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de Management API, respectivamente.

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/users/USER_ID/permissions' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users/USER_ID/permissions");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/users/USER_ID/permissions"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  }
  ```

</AuthCodeGroup>

| Valor                   | Descripción                                                                                                       |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | El ID del usuario cuyos permisos quieres obtener.                                                                 |
| `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `read:users`. |
