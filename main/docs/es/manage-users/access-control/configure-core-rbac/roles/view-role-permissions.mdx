Puedes ver los [permisos](/docs/es/manage-users/access-control/rbac) asignados a un rol en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o mediante la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Los permisos y roles asignados se utilizan con el conjunto de funcionalidades Authorization Core de la API.

<div id="prerequisites">
  ## Requisitos previos
</div>

Para que el control de acceso basado en roles (RBAC) funcione correctamente, debes habilitarlo en tu API mediante el Dashboard o la Management API. La funcionalidad de Authorization Core es diferente de la de Authorization Extension. Para compararlas, lee [Authorization Core vs. Authorization Extension](/docs/es/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Dashboard
</div>

1. Ve a [Dashboard &gt; User Management &gt; Roles](https://manage.auth0.com/#/roles) y haz clic en el nombre del rol para verlo.
2. Haz clic en la vista **Permissions**.

Se muestra la siguiente información para cada permiso:

| Columna     | Descripción                                              |
| ----------- | -------------------------------------------------------- |
| Nombre      | Nombre del permiso según la definición del permiso.      |
| Descripción | Descripción del permiso según la definición del permiso. |
| API         | Nombre de la API a la que está asociado el permiso.      |

<div id="management-api">
  ## Management API
</div>

Haz una llamada `GET` al [endpoint Get Role Permissions](https://auth0.com/docs/api/management/v2#!/Roles/get_role_permission). Asegúrate de reemplazar los valores de marcador de posición `ROLE_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de tu rol y tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de Management API, respectivamente.

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'

| Valor                   | Descripción                                                                                                       |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `ROLE_ID`               | El ID del rol para el que quieres obtener permisos.                                                               |
| `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `read:roles`. |
