Puede agregar permisos a los [roles](/docs/es/manage-users/access-control/rbac) mediante el <Tooltip tip="Auth0 Dashboard: la interfaz principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: la interfaz principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Los roles y los permisos asociados pueden usarse con el conjunto de funcionalidades Authorization Core de la API.

<div id="prerequisites">
  ## Requisitos previos
</div>

* Para que el control de acceso basado en roles (RBAC) funcione correctamente, debes habilitarlo para tu API mediante el Dashboard o la Management API. La funcionalidad de Authorization Core es distinta de la de Authorization Extension. Para ver una comparación, lee [Authorization Core vs. Authorization Extension](/docs/es/manage-users/access-control/authorization-core-vs-authorization-extension).
* Los roles se seleccionan de valores predefinidos. Si tu lista de roles está vacía, debes [crear un rol](/docs/es/manage-users/access-control/configure-core-rbac/roles/create-roles).
* [Configura una API](/docs/es/get-started/auth0-overview/set-up-apis) en el Auth0 Dashboard.
* Los permisos se seleccionan de valores predefinidos. Si tu lista de permisos está vacía, debes [agregar permisos](/docs/es/get-started/apis/add-api-permissions) a tu API.

<div id="dashboard">
  ## Dashboard
</div>

1. Ve a [Dashboard &gt; User Management &gt; Roles](https://manage.auth0.com/#/roles) y haz clic en el nombre del rol para verlo.
2. Haz clic en la pestaña **Permissions** y luego en **agregar permisos**.
3. Selecciona la API a la que quieres asignar permisos, luego selecciona los permisos que quieres agregar al rol y haz clic en **agregar permisos**.

<div id="management-api">
  ## Management API
</div>

Haz una solicitud `POST` al [endpoint Add Role Permissions](https://auth0.com/docs/api/management/v2#!/Roles/post_role_permission_assignment). Asegúrate de reemplazar los valores de marcador de posición `ROLE_ID`, `MGMT_API_ACCESS_TOKEN`, `API_IDENTIFIER` y `PERMISSION_NAME` por el ID de tu rol, el <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">access token</Tooltip> de Management API, el identificador de la API (<Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip>) y el nombre del permiso o de los permisos, respectivamente.

| Valor                   | Descripción                                                                                                                                                  |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ROLE_ID`               | El ID del rol al que quieres agregar permisos.                                                                                                               |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para la Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:roles`.                                       |
| `API_IDENTIFIER`        | Este es el identificador de la API asociada con los permisos que quieres agregar al rol especificado, también conocido como audience. No es el ID de la API. |
| `PERMISSION_NAME`       | Nombre de los permisos que quieres agregar al rol especificado.                                                                                              |
