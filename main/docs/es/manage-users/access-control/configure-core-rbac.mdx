title: Configurar las funcionalidades de Authorization Core para el control de acceso basado en roles
---

Las funcionalidades principales de Authorization de Auth0 permiten aplicar control de acceso basado en roles (RBAC) a tus API.

Para que RBAC funcione correctamente, debes habilitarlo para tu API mediante el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulta [Habilitar el control de acceso basado en roles para las API](/docs/es/get-started/apis/enable-role-based-access-control-for-apis).

La funcionalidad de Authorization Core es diferente de la Authorization Extension. Para consultar una comparación entre ambos productos, lee [Authorization Core vs. Authorization Extension](/docs/es/manage-users/access-control/authorization-core-vs-authorization-extension).

Para usar la funcionalidad principal de la forma más eficiente, debes hacer lo siguiente:

1. [Registrar una API con Auth0](/docs/es/get-started/auth0-overview/set-up-apis)
2. [Definir permisos para la API](/docs/es/get-started/apis/add-api-permissions)
3. [Crear roles](/docs/es/manage-users/access-control/configure-core-rbac/roles/create-roles)
4. [Asignar roles a los usuarios](/docs/es/manage-users/access-control/configure-core-rbac/rbac-users/assign-roles-to-users)
5. [Asignar permisos a los usuarios](/docs/es/manage-users/access-control/configure-core-rbac/rbac-users/assign-permissions-to-users), si es necesario.
