Auth0 usa [sesiones](/docs/es/manage-users/sessions) para mantener el estado de autenticación de un usuario entre solicitudes.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los endpoints de gestión de sesiones de Auth0 Management API están disponibles para los clientes con planes Enterprise. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Callout>

<div id="management-api-endpoints">
  ## Endpoints de la Management API
</div>

La [Management API](https://auth0.com/docs/api/management/v2) proporciona acceso out-of-band al funcionamiento interno de las sesiones de usuario en la [Auth0 Session Layer](/docs/es/manage-users/sessions/session-layers), así como métodos de eliminación para forzar la finalización de la sesión.

<div id="session-resource">
  ### Recurso de sesión
</div>

Puedes ver o eliminar una sesión específica con los siguientes endpoints:

| Nombre                                    | URL                                   | Scope(s) requerido(s) |
| ----------------------------------------- | ------------------------------------- | --------------------- |
| Inspeccionar una sesión específica por ID | `GET /api/v2/sessions/{sessionId}`    | `read:sessions`       |
| Eliminar una sesión específica por ID     | `DELETE /api/v2/sessions/{sessionId}` | `delete:sessions`     |

<div id="user-resource">
  ### Recurso del usuario
</div>

Puedes listar o eliminar todas las sesiones de un usuario determinado con los siguientes endpoints:

| Nombre                                            | URL                                      | scope(s) requerido(s) |
| ------------------------------------------------- | ---------------------------------------- | --------------------- |
| Listar los detalles de las sesiones de un usuario | `GET /api/v2/users/{userId}/sessions`    | `read:sessions`       |
| Eliminar todas las sesiones del usuario           | `DELETE /api/v2/users/{userId}/sessions` | `delete:sessions`     |

<div id="session-properties">
  ## Propiedades de la sesión
</div>

Los endpoints de sesión devuelven información relevante sobre la sesión y su historial.

| Campo                          | Descripción                                                                                                                                                                                                                                                                    |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ID de sesión                   | El ID de sesión es un identificador persistente de la sesión en el tenant de Auth0.<br /><br />Ten en cuenta que el ID de sesión corresponde al claim `sid`, ya presente en los tokens de ID y los tokens de cierre de sesión, y puede usarse para relacionar estas entidades. |
| Información temporal relevante | Información sobre la creación de la sesión, el momento de autenticación y la expiración.                                                                                                                                                                                       |
| Información del dispositivo    | La propiedad device recoge detalles relacionados con el agente de usuario (por ejemplo, el navegador) utilizado en las interacciones con esta sesión de Auth0.                                                                                                                 |
| Información de autenticación   | Contiene información resumida sobre los métodos utilizados para autenticarse en esta sesión.                                                                                                                                                                                   |

Para obtener información detallada sobre estos campos, consulta la [documentación de la Management API](https://auth0.com/docs/api/management/v2).

<div id="oidc-back-channel-logout-initiators">
  ## Iniciadores de cierre de sesión OIDC por canal secundario
</div>

Los eventos de eliminación de sesiones se vinculan al [cierre de sesión OIDC por canal secundario](/docs/es/authenticate/login/logout/back-channel-logout) mediante el iniciador `session-deleted`. Para obtener más información, consulta [Iniciadores de cierre de sesión OIDC por canal secundario](/docs/es/authenticate/login/logout/back-channel-logout/oidc-back-channel-logout-initiators).

<div id="sessions-and-refresh-tokens">
  ## Sesiones y tokens de actualización
</div>

Las sesiones y los <Tooltip tip="Token de actualización: Token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> funcionan conjuntamente para reducir la fricción en la autenticación de usuario y, al mismo tiempo, optimizar la seguridad. Para obtener más información, lee [Prácticas recomendadas para la gestión de sesiones de aplicaciones en el blog de Auth0](https://auth0.com/blog/application-session-management-best-practices).

Los tokens de actualización pueden seguir activos después de que una sesión haya caducado o se haya eliminado, o después de que el usuario cierre sesión. Puedes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para [gestionar los tokens de actualización](/docs/es/secure/tokens/manage-refresh-tokens-with-auth0-management-api) de forma independiente o junto con las sesiones.

<div id="limitations">
  ## Limitaciones
</div>

Las operaciones de Delete session se ejecutan de forma asíncrona y presentan consistencia eventual.
