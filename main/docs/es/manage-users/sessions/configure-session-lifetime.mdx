title: Configurar la duración de la sesión
---

Para configurar la duración de una sesión, puedes usar el [Dashboard](/docs/es/get-started/auth0-overview/dashboard), la [Management API](https://auth0.com/docs/api/management/v2) o una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para configurar la duración de la sesión en Auth0 Dashboard:

1. Ve a **Dashboard &gt; Configuración del tenant** y selecciona la vista **Advanced**.

2. En **expiración de la sesión**, puedes configurar:

| Política de sesión                                  | Descripción                                                                                              |
| --------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| **duración de la sesión inactiva (persistente)**    | Tiempo máximo (en minutos) de inactividad antes de que expire una sesión persistente.                    |
| **duración de la sesión inactiva (no persistente)** | Tiempo máximo (en minutos) de inactividad antes de que expire una sesión no persistente.                 |
| **duración máxima de la sesión (persistente)**      | Tiempo máximo (en minutos) que puede durar una sesión persistente, incluso si el usuario está activo.    |
| **duración máxima de la sesión (no persistente)**   | Tiempo máximo (en minutos) que puede durar una sesión no persistente, incluso si el usuario está activo. |

<div id="auth0-management-api">
  ## Auth0 Management API
</div>

Para configurar la duración de la sesión mediante Auth0 Management API:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las llamadas al endpoint `/api/v2/tenants/settings` requieren un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens) con el scope `update:tenant_settings`.
</Callout>

Realiza una solicitud `PATCH` al endpoint [/api/v2/tenants/settings](/docs/es/api/management/v2/tenants/patch-settings):

```bash cURL
curl --request PATCH \
  --url 'https://<your-domain>/api/v2/tenants/settings' \
  --header 'Authorization: Bearer MGMT_API_ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{\
    "session_lifetime": SESSION_LIFETIME_VALUE,\
    "idle_session_lifetime": IDLE_SESSION_LIFETIME_VALUE,\
    "ephemeral_session_lifetime": EPHEMERAL_SESSION_LIFETIME_VALUE,\
    "idle_ephemeral_session_lifetime": EPHEMERAL_IDLE_SESSION_LIFETIME_VALUE\
  }'
```

| Parámetro                         | Descripción                                                                 |
| --------------------------------- | --------------------------------------------------------------------------- |
| `session_lifetime`                | Duración máxima (en horas) para el tiempo de espera absoluto.               |
| `idle_session_lifetime`           | Duración máxima (en horas) antes de que una sesión caduque por inactividad. |
| `ephemeral_session_lifetime`      | Duración máxima (en horas) para el tiempo de espera absoluto.               |
| `idle_ephemeral_session_lifetime` | Duración máxima (en horas) antes de que una sesión caduque por inactividad. |

<div id="auth0-post-login-actions">
  ## Auth0 Actions de Post Login
</div>

Puedes configurar dinámicamente el comportamiento de la sesión con métodos de `api.session` mediante una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger). Esto te permite anular la configuración predeterminada de sesión del tenant en cada login, según la lógica específica del usuario o del contexto.

Los casos de uso incluyen:

* Reducir los tiempos de espera para los inicios de sesión de alto riesgo

* Ampliar los tiempos de espera para usuarios u organizaciones de confianza

* Ajustar la persistencia de las cookies según el tipo de aplicación

Los métodos de `api.session` disponibles para configurar la duración de la sesión son:

* `api.session.setExpiresAt`

* `api.session.setIdleExpiresAt`

* `api.session.setCookieMode`

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los métodos de `api.session` solo se aplican a la sesión actual y deben llamarse durante la transacción de login. Estos métodos requieren que la propiedad `event.session.id` del evento esté disponible.
</Callout>

Para obtener más información, consulta [Sessions with Actions](/docs/es/manage-users/sessions/manage-sessions-actions).
