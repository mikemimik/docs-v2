La **expiración absoluta** de una sesión se define al crearla. Puedes ajustarla configurando la sesión desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="dashboard">
  ## Dashboard
</div>

1. Ve a [Dashboard &gt; Configuración del tenant](https://manage.auth0.com/#/tenant) y selecciona la vista **Advanced**.
2. Desplázate hasta la sección **Expiración de la sesión**, localiza **Duración de la sesión inactiva** y **Duración máxima de la sesión**, introduce la configuración deseada y selecciona **Save**.

   <Frame>![Dashboard Configuración del tenant Advanced Login Gestión de sesiones](/docs/images/cdy7uua7fh8z/q2fGPzoUqCcXj7OxcHhjy/de15afcd26e8ea80937733034ffbd387/Screenshot_2024-11-06_at_11.02.16.png)</Frame>

| Configuración                  | Descripción                                                                                                                                                                                                                                                                                        |
| ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Duración de la sesión inactiva | Periodo (en minutos) tras el cual la sesión de un usuario expirará si no ha interactuado con el servidor de autorización. Se aplicarán los límites del sistema si supera los 4.320 minutos (3 días) para Essential o Professional, o los 144.000 minutos (100 días) para los planes Enterprise.    |
| Duración máxima de la sesión   | Periodo (en minutos) tras el cual se requerirá que un usuario vuelva a iniciar sesión, independientemente de su actividad. Se aplicarán los límites del sistema si supera los 43.200 minutos (30 días) para Essential o Professional, o los 525.600 minutos (365 días) para los planes Enterprise. |

<div id="management-api">
  ## Management API
</div>

Haz una solicitud `PATCH` al [endpoint de Configuración del tenant](https://auth0.com/docs/api/management/v2#!/tenants/patch_settings). Asegúrate de reemplazar los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `SESSION_LIFETIME` e `IDLE_SESSION_LIFETIME` por tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de Management API, el valor de duración de la sesión y el valor de duración de la sesión inactiva, respectivamente.

| Valor                         | Descripción                                                                                                                                                                                                                                                                               |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN`       | [token de acceso para la Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:tenant_settings`.                                                                                                                                                          |
| `IDLE_SESSION_LIFETIME_VALUE` | Período (en horas) tras el cual la sesión de un usuario caducará si no ha interactuado con el servidor de autorización. Si supera las 72 horas (3 días) para Essential o Professional, o las 2.400 horas (100 días) para planes empresariales, se aplicarán los límites del sistema.      |
| `SESSION_LIFETIME_VALUE`      | Período (en horas) tras el cual se le pedirá a un usuario que vuelva a iniciar sesión, independientemente de su actividad. Si supera las 720 horas (30 días) para Essential o Professional, o las 8.760 horas (365 días) para planes empresariales, se aplicarán los límites del sistema. |
