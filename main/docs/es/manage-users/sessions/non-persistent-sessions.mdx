---
description: Describe las sesiones no persistentes disponibles con Auth0.
title: Sesiones no persistentes
---

En las sesiones persistentes, los usuarios pueden cerrar el navegador y conservar la sesión al volver a abrirlo en el mismo dispositivo. Para mejorar la experiencia del usuario y ofrecer un acceso más seguro, las sesiones no persistentes invalidan una <Tooltip tip="Cookie de sesión: Entidad que, cuando está presente, permite que el usuario sea considerado autenticado." cta="Ver glosario" href="/docs/es/glossary?term=session+cookie">cookie de sesión</Tooltip> al cerrar el navegador.

<Warning>
  La autenticación con Biometrics como tipo de primer factor no funcionará con sesiones no persistentes.
</Warning>

<div id="browser-limitations">
  ## Limitaciones del navegador
</div>

En algunos casos, las sesiones no persistentes **no pueden** imponerse mediante la configuración del tenant. Algunos ejemplos son:

* El usuario tiene habilitada en el navegador una opción de restauración de sesión; al restaurarla, también se restaura la cookie de sesión.
* El usuario cierra una pestaña, pero no la ventana del navegador; la cookie de sesión no se borra hasta que la sesión finaliza según la expiración por inactividad o la expiración absoluta.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En estos casos, las sesiones no persistentes **no** aportan seguridad adicional a la sesión.
</Callout>

<div id="sessions">
  ## Sesiones
</div>

Las sesiones constan de una cookie en el navegador y de un registro de sesión en el servidor. En las sesiones no persistentes, las cookies no se conservan y se establece un tiempo de espera del tenant, para que los usuarios no tengan que cerrar sesión manualmente en un dispositivo.

La duración de la sesión determina cuánto tiempo debe mantener el sistema la sesión de inicio de sesión y se configura por tenant. La Expiración absoluta de una sesión se define cuando se crea la sesión. Puede habilitar la configuración no persistente para las sesiones de usuario o permitir que la sesión continúe incluso si se cierra el navegador. Puede ajustar la Expiración absoluta y la configuración no persistente en el Dashboard o en la <Tooltip tip="Management API: Un producto para permitir que los clientes realicen tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, lea [Configurar los ajustes de duración de la sesión](/docs/es/manage-users/sessions/configure-session-lifetime-settings).

<div id="cookies">
  ## Cookies
</div>

La plataforma Auth0 utiliza tres tipos de cookies: de sesión, <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> y device. Todas ellas se vuelven no persistentes una vez configurado tu tenant de Auth0. El tiempo de espera del tenant desencadena la expiración de la sesión.

Cuando las sesiones no persistentes están habilitadas, las siguientes cookies se emitirán como cookies de sesión al interactuar con el <Tooltip tip="Servidor de autorización: Servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>:

* `auth0` / `auth0_compat`
* `auth0-mf` / `auth0-mf_compat`
* `did` / `did_compat`

Para obtener más información sobre las cookies, consulta [Authentication API Cookies](/docs/es/manage-users/cookies/authentication-api-cookies).

<div id="tenant-configuration">
  ## Configuración del tenant
</div>

Puedes configurar tu tenant para gestionar las sesiones de una de estas dos maneras:

* **Persistente**: las cookies de sesión se mantienen aunque se cierre el navegador.
* **No persistente**: las cookies de sesión no se mantienen. El navegador conserva las cookies de sesión hasta que finaliza la sesión de navegación. Para obtener más información, consulta [HTTP State Management Mechanism on IETF Datatracker](https://datatracker.ietf.org/doc/rfc2109/).

Para configurar estos ajustes en el Dashboard:

1. Ve a [Dashboard &gt; Configuración del tenant](https://manage.auth0.com/#/tenant) y selecciona la vista **Advanced**.
2. Desplázate hasta la sección **Expiración de la sesión**, busca **Duración de la sesión inactiva** y **Duración máxima de la sesión**, introduce la configuración que quieras y selecciona **Guardar.**

   | Configuración                  | Descripción                                                                                                                                                                                                                                                                                        |
   | ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | Duración de la sesión inactiva | Período (en minutos) tras el cual la sesión de un usuario expirará si no ha interactuado con el servidor de autorización. Se aplicarán los límites del sistema si supera los 4.320 minutos (3 días) para Essential o Professional, o los 144.000 minutos (100 días) para planes empresariales.     |
   | Duración máxima de la sesión   | Período (en minutos) tras el cual se le pedirá al usuario que vuelva a iniciar sesión, independientemente de su actividad. Se aplicarán los límites del sistema si supera los 43.200 minutos (30 días) para Developer o Developer Pro, o los 525.600 minutos (365 días) para planes empresariales. |

<div id="management-api">
  ## Management API
</div>

Puede utilizar la Management API para habilitar sesiones no persistentes, así como para establecer valores para la duración de la sesión, la duración de la sesión inactiva y las propiedades de las cookies de sesión. Para obtener más información, consulte el [endpoint Tenant Update de la Management API](https://auth0.com/docs/api/management/v2/tenants/patch-settings).
