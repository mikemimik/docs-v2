Para configurar los metadatos de la sesión, puedes usar una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) de Auth0 y la [Management API](https://auth0.com/docs/api/management/v2). También puedes incluirlos en el token de [OpenID Connect Back-Channel Logout](/docs/es/authenticate/login/logout/back-channel-logout).

<Warning>
  Los metadatos de la sesión de Auth0 no son un almacén de datos seguro y no deben utilizarse para almacenar información confidencial. Esto incluye secretos y PII de alto riesgo, como números de la seguridad social o de tarjetas de crédito. Se recomienda encarecidamente a los clientes de Auth0 que evalúen los datos almacenados en los metadatos y guarden solo lo necesario para fines de gestión de identidades y accesos. Para obtener más información, consulta [Cumplimiento del RGPD de Auth0](/docs/es/secure/data-privacy-and-compliance/gdpr).
</Warning>

## Auth0 Management API

Puede gestionar las solicitudes CRUD (crear, reemplazar, actualizar y eliminar) de los metadatos de la sesión mediante la Management API:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las llamadas al endpoint `/api/v2/sessions/{id}` requieren un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens) con el scope `update:session`.
</Callout>

### Recupera los metadatos de la sesión existentes

Haz una solicitud `GET` al endpoint [`/api/v2/sessions/{id}`](https://auth0.com/docs/api/management/v2/sessions/get-session):

```bash lines
GET /api/v2/sessions/{id}
```

<div id="add-or-update-existing-session-metadata">
  ### Añadir o actualizar los metadatos existentes de la sesión
</div>

Haz una solicitud `PATCH` al endpoint [`/api/v2/sessions/{id}`](https://auth0.com/docs/api/management/v2/sessions/patch-sessions-by-id):

```bash lines
PATCH /api/v2/sessions/{id}
Content-Type: application/json

{
  "session_metadata": {
    "my_metadata": "my new metadata"
  }
}
```

<div id="delete-session-metadata">
  ### Eliminar metadatos de la sesión
</div>

Haz una solicitud `PATCH` al endpoint [`/api/v2/sessions/{id}`](https://auth0.com/docs/api/management/v2/sessions/patch-sessions-by-id):

```bash lines
PATCH /api/v2/sessions/{id}
Content-Type: application/json

{
  "session_metadata": {}
}
```

<div id="auth0-post-login-actions">
  ## Actions de Auth0 para Post Login
</div>

Puedes realizar operaciones CRUD sobre los metadatos de la sesión mediante los objetos `api.session` con una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger). Esto te permite gestionar los metadatos de la sesión según una lógica específica del usuario o del contexto.

<div id="retrieve-existing-session-metadata">
  ### Recuperar los metadatos existentes de la sesión
</div>

Usa el objeto `event.session.metadata?.deviceName` para leer el metadato `deviceName`:

```javascript lines
const device = event.session.metadata?.deviceName;
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El objeto `event.session.metadata` incluye metadatos configurados en:
  *Actions anteriores dentro del mismo flujo
  *Transacciones anteriores si se reutilizó la sesión
</Callout>

<div id="add-or-update-existing-metadata">
  ### Agregar o actualizar metadatos existentes
</div>

Usa el método `api.session.setMetadata()` para actualizar los metadatos de la sesión:

```javascript lines
api.session.setMetadata("deviceName", "Auth0's iPhone");
```

Los cambios están disponibles inmediatamente en el objeto `event.session` en las Actions posteriores.

<div id="delete-session-metadata">
  ### Eliminar metadatos de la sesión
</div>

Usa los siguientes objetos `api.session` para eliminar metadatos de la sesión:

* `api.session.deleteMetadata("key")` elimina los metadatos de la sesión especificados

* `api.session.evictMetadata()` elimina todos los metadatos de la sesión

Para obtener más información sobre estos objetos, consulta:

* [objeto de evento](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object): Obtén más información sobre el objeto de evento del token de actualización y sus propiedades.
* [objeto de API](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object): Obtén más información sobre el objeto de API del token de actualización y sus métodos.

<div id="oidc-back-channel-logout">
  ## cierre de sesión OIDC por canal secundario
</div>

Puede configurar el `logout_token` para que incluya metadatos de la sesión desde el [Dashboard](/docs/es/get-started/auth0-overview/dashboard) o la [Management API](https://auth0.com/docs/api/management/v2).

<div id="auth0-dashboard">
  ### Auth0 Dashboard
</div>

Para configurar el token de cierre de sesión OIDC por canal secundario con metadatos de la sesión:

1. Vaya a [Dashboard &gt; Applications](https://manage.auth0.com/#/applications) y seleccione su aplicación.

2. Seleccione la pestaña **Settings**.

3. En **OpenID Connect Back-Channel Logout &gt; Back-Channel Logout URL**, agregue el URI de cierre de sesión de la aplicación que recibirá los `logout_tokens`.

4. Establezca **Back-Channel Logout Initiators** en una de estas opciones:

   * **Selected initiators only** o

   * **All supported initiators**

5. Active **Include Session Metadata**.

6. Seleccione **Save Changes**.

Una vez configurado, el `logout_token` incluirá todos los metadatos de la sesión almacenados.

<div id="auth0-management-api">
  ### Auth0 Management API
</div>

Puede usar el endpoint [`/api/v2/clients/{id}`](https://auth0.com/docs/api/management/v2/sessions/patch-sessions-by-id) para actualizar su aplicación a fin de incluir metadatos de la sesión en el `logout_token`.

Haga una solicitud `PATCH` al endpoint [`/api/v2/sessions/{id}`](https://auth0.com/docs/api/management/v2/sessions/patch-sessions-by-id):

```json lines
"oidc_backchannel_logout": {
  "backchannel_logout_initiators": {
    "mode": "all"
  },
  "backchannel_logout_urls": [
    "https://httpdump.app/inspect/9bccf574-e55f-4b2e-9822-f37372588fc1"
  ],
  "backchannel_logout_session_metadata": {
    "include": true
  }
}
```

<div id="error-handling">
  ## Manejo de errores
</div>

Puedes revisar los eventos de log de los metadatos de la sesión en [Dashboard &gt; monitoreo &gt; Logs](https://manage.auth0.com/#/logs) o recuperar los logs mediante el endpoint de [logs de Management API](/docs/es/api/management/v2/logs/get-logs).

* Si se produce un error al agregar o actualizar los metadatos de la sesión con Actions, la transacción de autenticación falla y se devuelve un error a la URL de devolución de llamada.

Se registra un código de evento de fallo `f` con el error correspondiente:

```json lines
{
  "error": "access_denied",
  "error_description": "Failed to set session metadata: Invalid metadata: Metadata keys may only include letters, numbers, underscores, or hyphens",
  "state": "my-custom-state"
}
```

* Si se produce un fallo al gestionar los metadatos de la sesión con la Auth0 Management API, la API responde con un error `HTTP status: 400` y su mensaje correspondiente:

```json lines
{
  "statusCode": 400,
  "message": "Metadata must not exceed 25 entries. Each key and value must be ≤ 255 characters."
}
```
