<Card title="Antes de empezar">
  Asegúrate de tener:

  * Un tenant de Auth0 con [Custom Domain](/docs/es/customize/custom-domains) configurado para tu aplicación

  * [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_actions.html) instalado y autenticado.

  * Una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)

  * Una sesión del tenant de Auth0 que use la política de sesiones [no persistentes](/docs/es/manage-users/sessions/session-lifecycle)

  * (Opcional) Configura los valores de tiempo de espera por inactividad y de tiempo de espera absoluto de la sesión
</Card>

La experiencia **Keep Me Signed In** ofrece a los usuarios la opción de mantener la sesión iniciada en un dispositivo de confianza.

<Frame>![Imagen de login](/docs/images/cdy7uua7fh8z/sessions-use-case/keep-me-signed-in-login.png)</Frame>

* Si se selecciona la casilla **Keep Me Signed In**, se establece una sesión persistente que se mantiene aunque se reinicie el navegador.

* Si no se selecciona la casilla **Keep Me Signed In**, se establece una sesión no persistente que vence al cerrar el navegador.

Al usar [sesiones no persistentes (efímeras)](/docs/es/manage-users/sessions/session-lifecycle) con [Actions](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger), el servidor gestiona esta lógica de forma segura sin exponer tokens de actualización al front-end.

Con prompts personalizados, Auth0 CLI y sesiones efímeras mediante Actions, puedes implementar un flujo de &quot;Keep Me Signed In&quot; seguro y moderno, sin tokens de actualización, sin soluciones improvisadas del lado del cliente para la sesión y con control total desde el backend.

Esta solución es ideal para clientes que priorizan:

* Gestión de sesiones del navegador con enfoque de confianza cero

* Persistencia mínima de la sesión

* Sobrescrituras para dispositivos de confianza

<div id="1-configure-a-universal-login-page-and-add-a-custom-prompt">
  ## 1. Configura una página de Universal Login y agrega un prompt personalizado
</div>

Para recopilar la preferencia del usuario con respecto a **Keep Me Signed In**, debes personalizar [Universal Login](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts) configurando una plantilla de página y agregando un prompt personalizado con Auth0 CLI.

<div id="optional-enable-the-universal-login-page-template">
  ### Opcional: habilita la plantilla de la página de Universal Login
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este paso solo es necesario si tu tenant no tiene configurada una plantilla para Universal Login.
</Callout>

Usa el siguiente comando para crear o actualizar la plantilla de la página:

```bash lines
auth0 universal-login templates update
```

El editor de personalización abre la plantilla de la página de inicio de sesión. Puedes usar cualquier estructura HTML que se adapte a tu marca.

Aquí tienes una plantilla mínima de ejemplo con un fondo y un widget integrado:

```html lines
<!DOCTYPE html>
<html lang="{{locale}}">
  <head>
    {%- auth0:head -%}
    <style>
      body {
        background-image: url("https://images.unsplash.com/photo-1643916861364-02e63ce3e52f");
        background-size: cover;
        background-position: center;
      }
      .prompt-wrapper {
        display: flex;
        align-items: center;
        justify-content: center;
        height: 100%;
        background-color: rgba(0, 0, 0, 0.3);
      }
    </style>
    <title>{{ prompt.screen.texts.pageTitle }}</title>
  </head>
  <body class="_widget-auto-layout">
    {% if prompt.name == "login" or prompt.name == "signup" %}
      <div class="prompt-wrapper">
        {%- auth0:widget -%}
      </div>
    {% else %}
      {%- auth0:widget -%}
    {% endif %}
  </body>
</html>
```

Guarda y cierra el editor; Auth0 CLI te pedirá que confirmes:

```bash lines
Do you want to save the template? Yes

▸ Successfully updated the template for the New Universal Login Experience!
```

<div id="2-add-a-keep-me-signed-in-checkbox-using-auth0-cli">
  ## 2. Agrega una casilla de verificación «Keep Me Signed In» con Auth0 CLI
</div>

Para agregar una casilla de verificación en la pantalla de Universal Login, sigue estos pasos:

<div id="launch-the-customization-editor">
  ### Abre el editor de personalización
</div>

1. Abre el editor de personalización del prompt login-id o del prompt de inicio de sesión, según la configuración de tu tenant:

```bash lines
auth0 universal-login customize -p login-id
```

2. Selecciona **standard** como modo de renderización

```bash lines
Rendering Mode: standard
```

El editor de personalización se abre en una dirección local:

```bash lines
http://localhost:52649
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si no sabe con certeza si su tenant usa `login` o `login-id`, consulte la [personalización del prompt](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts).
</Callout>

<div id="configure-the-checkbox">
  ### Configura la casilla de verificación
</div>

Abre la opción **Partials** en el editor de personalización:

1. Configura **Prompt** como: `login-id` (o `login`)

2. Configura **Screen** como: `login-id` (o `login`)

3. Configura **Partial** como: `form-content-end`

<Frame>![Partials de Auth0 CLI](/docs/images/cdy7uua7fh8z/sessions-use-case/partials.png)</Frame>

Agrega el siguiente fragmento de HTML:

```html lines
<div class="ulp-field">
  <input type="checkbox" name="ulp-remember-me" id="remember-me">
  <label for="remember-me">¿Mantener sesión iniciada en este dispositivo?</label>
</div>
```

Esto renderiza la casilla de verificación debajo del formulario de inicio de sesión.

<div id="deploy-your-changes">
  ### Despliega tus cambios
</div>

Selecciona **Desplegar cambios** en la esquina superior derecha para guardar y aplicar la personalización.

<div id="checkpoint">
  ## Checkpoint
</div>

Inicia sesión con Universal Login.
La pantalla de inicio de sesión incluye la casilla **Keep Me Signed** In.

<div id="3-create-a-post-login-action-using-auth0-cli">
  ## 3. Cree una Action de Post Login con Auth0 CLI
</div>

Cree un nuevo archivo llamado `set-session-persistence.js` para usarlo con Auth0 CLI y agregue el siguiente código:

```javascript lines
exports.onExecutePostLogin = async (event, api) => {

  // Aplicar el cambio solo cuando el parámetro esté disponible
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode('persistent');
  }
};
```

Esta Action hace que el cookie de sesión se conserve al reiniciar el navegador cuando el usuario marca la casilla. Para ampliar también la duración de la sesión en dispositivos de confianza, consulta [Set session duration for trusted devices](#set-session-duration-for-trusted-devices) más abajo.

Usa Auth0 CLI para crear o actualizar tu Action de Post Login con el archivo `set-session-persistence.js`:

```bash lines
auth0 actions create --name kmsi --trigger post-login --code "$(cat set-session-persistence.js)"
```

Auth0 CLI devuelve el ID de la Action:

```bash lines

NAME           kmsi
TYPE           post-login
STATUS         pending
DEPLOYED       ✗
LAST DEPLOYED

CREATED        0 seconds ago

                   if (event.request.body['ulp-remember-me']) {
                     api.session.setCookieMode("persistent");
                   } else {
                     api.session.setCookieMode("non-persistent");
                   }
                 };
```

Implementa tu Action con su `id`:

```bash lines
auth0 actions deploy <id>
```

Ve a [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post Login](https://manage.auth0.com/#/actions/triggers/post-login) para añadir la nueva Action al flujo actual y selecciona **Apply**.

<div id="set-session-duration-for-trusted-devices">
  ### Establecer la duración de la sesión para dispositivos de confianza
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [`setCookieMode()`](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object#api-session-setcookiemode-mode) solo controla cómo se envía la cookie de sesión al navegador. Una cookie persistente se mantiene incluso si se reinicia el navegador, mientras que una cookie no persistente (de sesión) se elimina al cerrar el navegador. No cambia el tiempo de expiración de la sesión en el backend. Si quieres que los dispositivos de confianza tengan una sesión de mayor duración, debes establecer explícitamente la expiración mediante `api.session.setExpiresAt()` o `api.session.setIdleExpiresAt()`.
</Callout>

Para que los dispositivos de confianza tengan una sesión de mayor duración, usa `api.session.setExpiresAt()` o `api.session.setIdleExpiresAt()` junto con `setCookieMode()`:

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode("persistent");

    const now = Date.now();
    api.session.setExpiresAt(now + 7 * 24 * 60 * 60 * 1000); // 7 días
  } else {
    api.session.setCookieMode("nonpersistent");
  }
};
```

<div id="4-test-implementation">
  ## 4. Prueba de la implementación
</div>

Para verificar el comportamiento de la sesión, puedes hacer lo siguiente:

Inicia sesión con Universal Login y selecciona la casilla **Keep Me Signed In**.

* En los registros del tenant, busca (tipos de evento `s` o `ssa`)

  * `cookie: { mode: "non-persistent" }`

  * `cookie: { mode: "persistent" }`

* Busca `event.session.cookie.mode` en tus Actions
