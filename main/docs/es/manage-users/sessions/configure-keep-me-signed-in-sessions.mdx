<Card title="Antes de empezar">
  Asegúrate de tener:

  * Un [Custom Domain](/docs/es/customize/custom-domains) configurado en tu tenant de Auth0 para tu aplicación

  * [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_actions.html) instalado y autenticado.

  * Una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)

  * Una sesión en el tenant de Auth0 que use la política de sesiones [no persistentes](/docs/es/manage-users/sessions/session-lifecycle)

  * (Opcional) Configurar los valores de tiempo de inactividad y de tiempo máximo de sesión
</Card>

La experiencia de **Keep Me Signed In** ofrece a los usuarios la opción de seguir con la sesión iniciada en un dispositivo de confianza.

<Frame>![Imagen de login](/docs/images/cdy7uua7fh8z/sessions-use-case/keep-me-signed-in-login.png)</Frame>

* Si se selecciona la casilla **Keep Me Signed In**, se establece una sesión persistente que se mantiene incluso después de reiniciar el navegador.

* Si no se selecciona la casilla **Keep Me Signed In**, se establece una sesión no persistente que vence al cerrar el navegador.

Al usar [sesiones no persistentes (efímeras)](/docs/es/manage-users/sessions/session-lifecycle) con [Actions](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger), el servidor gestiona esta lógica de forma segura sin exponer tokens de actualización al frontend.

Con prompts personalizados, Auth0 CLI y sesiones efímeras mediante Actions, puedes implementar un flujo de &quot;Keep Me Signed In&quot; seguro y moderno, sin tokens de actualización, sin apaños de sesión del lado del cliente y con control total desde el backend.

Esta solución es ideal para clientes que priorizan:

* Gestión de sesiones del navegador con un enfoque de confianza cero

* Persistencia mínima de la sesión

* Sobrescrituras para dispositivos de confianza

<div id="1-configure-a-universal-login-page-and-add-a-custom-prompt">
  ## 1. Configura una página de Universal Login y añade un prompt personalizado
</div>

Para recopilar la preferencia del usuario sobre **Keep Me Signed In**, debes personalizar [Universal Login](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts) configurando una plantilla de página y añadiendo un prompt personalizado con Auth0 CLI.

<div id="optional-enable-the-universal-login-page-template">
  ### Opcional: Activa la plantilla de página de Universal Login
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este paso solo es necesario si tu tenant no tiene configurada una plantilla de Universal Login.
</Callout>

Usa el siguiente comando para crear o actualizar la plantilla de página:

```bash lines
auth0 universal-login templates update
```

El editor de personalización abre la plantilla de la página de inicio de sesión. Puedes usar cualquier estructura HTML que se ajuste a tu marca.

Aquí tienes una plantilla de ejemplo mínima con un fondo y un widget dentro de un contenedor:

```html lines
<!DOCTYPE html>
<html lang="{{locale}}">
  <head>
    {%- auth0:head -%}
    <style>
      body {
        background-image: url("https://images.unsplash.com/photo-1643916861364-02e63ce3e52f");
        background-size: cover;
        background-position: center;
      }
      .prompt-wrapper {
        display: flex;
        align-items: center;
        justify-content: center;
        height: 100%;
        background-color: rgba(0, 0, 0, 0.3);
      }
    </style>
    <title>{{ prompt.screen.texts.pageTitle }}</title>
  </head>
  <body class="_widget-auto-layout">
    {% if prompt.name == "login" or prompt.name == "signup" %}
      <div class="prompt-wrapper">
        {%- auth0:widget -%}
      </div>
    {% else %}
      {%- auth0:widget -%}
    {% endif %}
  </body>
</html>
```

Guarda y cierra el editor; Auth0 CLI te pedirá que confirmes:

```bash lines
Do you want to save the template? Yes

▸ Successfully updated the template for the New Universal Login Experience!
```

<div id="2-add-a-keep-me-signed-in-checkbox-using-auth0-cli">
  ## 2. Añade una casilla de verificación Keep Me Signed In con Auth0 CLI
</div>

Para añadir una casilla de verificación en la pantalla de Universal Login, sigue estos pasos:

<div id="launch-the-customization-editor">
  ### Abra el editor de personalización
</div>

1. Abra el editor de personalización para `login-id` o para el prompt de inicio de sesión, según la configuración de su tenant:

```bash lines
auth0 universal-login customize -p login-id
```

2. Selecciona **standard** como modo de renderización

```bash lines
Rendering Mode: standard
```

El editor de personalización se abre en una dirección local:

```bash lines
http://localhost:52649
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si no sabe con certeza si su tenant usa `login` o `login-id`, consulte la [personalización de prompts](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts) de Auth0.
</Callout>

<div id="configure-the-checkbox">
  ### Configurar la casilla de verificación
</div>

Abre la opción **Partials** en el editor de personalización:

1. Configura **Prompt** como: `login-id` (o `login`)

2. Configura **Screen** como: `login-id` (o `login`)

3. Configura **Partial** como: `form-content-end`

<Frame>![partials de Auth0 CLI](/docs/images/cdy7uua7fh8z/sessions-use-case/partials.png)</Frame>

Agrega el siguiente fragmento de HTML:

```html lines
<div class="ulp-field">
  <input type="checkbox" name="ulp-remember-me" id="remember-me">
  <label for="remember-me">¿Mantener la sesión iniciada en este dispositivo?</label>
</div>
```

Esto renderiza la casilla de verificación debajo del formulario de inicio de sesión.

<div id="deploy-your-changes">
  ### Implementa tus cambios
</div>

Selecciona **Implementar cambios** en la esquina superior derecha para guardar y aplicar la personalización.

<div id="3-create-a-post-login-action-using-auth0-cli">
  ## 3. Crear una Action de Post Login con Auth0 CLI
</div>

Crea un archivo nuevo llamado `set-session-persistence.js` para usarlo con Auth0 CLI y agrega el siguiente código:

```javascript lines
exports.onExecutePostLogin = async (event, api) => {

  // Aplicar el cambio solo cuando el parámetro esté disponible
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode('persistent');
  }
};
```

Esta Action hace que la cookie de sesión se conserve aunque se reinicie el navegador cuando el usuario marca la casilla. Para ampliar también la duración de la sesión en dispositivos de confianza, consulta [Establecer la duración de la sesión para dispositivos de confianza](#set-session-duration-for-trusted-devices) más abajo.

Usa Auth0 CLI para crear o actualizar tu Action de Post Login con el archivo `set-session-persistence.js`:

```bash lines
auth0 actions create --name kmsi --trigger post-login --code "$(cat set-session-persistence.js)"
```

Auth0 CLI devuelve el ID de la Action:

```bash lines

NAME           kmsi
TYPE           post-login
STATUS         pending
DEPLOYED       ✗
LAST DEPLOYED

CREATED        0 seconds ago

                   if (event.request.body['ulp-remember-me']) {
                     api.session.setCookieMode("persistent");
                   } else {
                     api.session.setCookieMode("non-persistent");
                   }
                 };
```

Implementa tu Action con su `id`:

```bash lines
auth0 actions deploy <id>
```

Vaya a [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post Login](https://manage.auth0.com/#/actions/triggers/post-login) para añadir la nueva Action al flujo actual y seleccione **Apply**.

<div id="set-session-duration-for-trusted-devices">
  ### Establecer la duración de la sesión para dispositivos de confianza
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [`setCookieMode()`](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object#api-session-setcookiemode-mode) solo controla cómo se envía la cookie de sesión al navegador. Una cookie persistente se mantiene después de reiniciar el navegador, mientras que una cookie no persistente (de sesión) se elimina al cerrar el navegador. No cambia el tiempo de expiración de la sesión del backend. Si quiere que los dispositivos de confianza tengan una sesión más larga, debe establecer explícitamente la expiración con `api.session.setExpiresAt()` o `api.session.setIdleExpiresAt()`.
</Callout>

Para que los dispositivos de confianza tengan una sesión más larga, use `api.session.setExpiresAt()` o `api.session.setIdleExpiresAt()` junto con `setCookieMode()`:

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode("persistent");

    const now = Date.now();
    api.session.setExpiresAt(now + 7 * 24 * 60 * 60 * 1000); // 7 días
  } else {
    api.session.setCookieMode("nonpersistent");
  }
};
```

<div id="4-test-implementation">
  ## 4. Prueba de la implementación
</div>

Para verificar el comportamiento de la sesión, puedes hacer lo siguiente:

Inicia sesión con Universal Login y selecciona la casilla **Keep Me Signed In**.

* En los logs del tenant, busca (tipos de evento `s` o `ssa`)

  * `cookie: { mode: "non-persistent" }`

  * `cookie: { mode: "persistent" }`

* Busca `event.session.cookie.mode` en tus Actions
