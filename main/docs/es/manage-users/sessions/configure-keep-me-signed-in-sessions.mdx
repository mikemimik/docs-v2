<Card title="Antes de comenzar">
  Asegúrate de tener:

  * Un [Custom Domain](/docs/es/customize/custom-domains) configurado en el tenant de Auth0 de tu aplicación

  * [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_actions.html) instalado y autenticado.

  * Una [Action de Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)

  * Una sesión del tenant de Auth0 que use la política de sesiones [no persistentes](/docs/es/manage-users/sessions/session-lifecycle)

  * (Opcional) Configura los valores de tiempo de espera por inactividad y de tiempo de espera absoluto de la sesión
</Card>

La experiencia **Keep Me Signed In** ofrece a los usuarios la opción de mantener la sesión iniciada en un dispositivo de confianza.

<Frame>![Imagen de inicio de sesión](/docs/images/cdy7uua7fh8z/sessions-use-case/keep-me-signed-in-login.png)</Frame>

* Si la casilla **Keep Me Signed In** está seleccionada, establece una sesión persistente que se mantiene incluso al reiniciar el navegador.

* Si la casilla **Keep Me Signed In** no está seleccionada, establece una sesión no persistente que vence cuando se cierra el navegador.

Al usar [sesiones no persistentes (efímeras)](/docs/es/manage-users/sessions/session-lifecycle) con [Actions](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger), el servidor gestiona esta lógica de forma segura sin exponer tokens de actualización al frontend.

Con prompts personalizados, Auth0 CLI y sesiones efímeras mediante Actions, puedes implementar un flujo de &quot;Keep Me Signed In&quot; seguro y moderno, sin tokens de actualización, sin soluciones improvisadas del lado del cliente para la sesión y con control total desde el backend.

Esta solución es ideal para clientes que priorizan:

* La gestión de sesiones del navegador con enfoque de confianza cero

* Una persistencia mínima de la sesión

* Excepciones para dispositivos de confianza

<div id="1-configure-a-universal-login-page-and-add-a-custom-prompt">
  ## 1. Configura una página de Universal Login y añade un prompt personalizado
</div>

Para recopilar la preferencia del usuario de **Keep Me Signed In**, debes personalizar [Universal Login](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts) configurando una plantilla de página y añadiendo un prompt personalizado con Auth0 CLI.

<div id="optional-enable-the-universal-login-page-template">
  ### Opcional: Habilitar la plantilla de página de Universal Login
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este paso solo es necesario si tu tenant no tiene configurada una plantilla de Universal Login.
</Callout>

Usa el siguiente comando para crear o actualizar la plantilla de página:

```bash lines
auth0 universal-login templates update
```

El editor de personalización abre la plantilla de la página de inicio de sesión. Puedes usar cualquier estructura HTML que se adapte a tu marca.

Aquí tienes una plantilla de ejemplo mínima con un fondo y un widget dentro de un contenedor:

```html lines
<!DOCTYPE html>
<html lang="{{locale}}">
  <head>
    {%- auth0:head -%}
    <style>
      body {
        background-image: url("https://images.unsplash.com/photo-1643916861364-02e63ce3e52f");
        background-size: cover;
        background-position: center;
      }
      .prompt-wrapper {
        display: flex;
        align-items: center;
        justify-content: center;
        height: 100%;
        background-color: rgba(0, 0, 0, 0.3);
      }
    </style>
    <title>{{ prompt.screen.texts.pageTitle }}</title>
  </head>
  <body class="_widget-auto-layout">
    {% if prompt.name == "login" or prompt.name == "signup" %}
      <div class="prompt-wrapper">
        {%- auth0:widget -%}
      </div>
    {% else %}
      {%- auth0:widget -%}
    {% endif %}
  </body>
</html>
```

Guarda y cierra el editor; Auth0 CLI te pedirá que confirmes:

```bash lines
Do you want to save the template? Yes

▸ Successfully updated the template for the New Universal Login Experience!
```

<div id="2-add-a-keep-me-signed-in-checkbox-using-auth0-cli">
  ## 2. Añade una casilla Keep Me Signed In con Auth0 CLI
</div>

Para añadir una casilla en la pantalla de Universal Login, sigue estos pasos:

<div id="launch-the-customization-editor">
  ### Abre el editor de personalización
</div>

1. Abre el editor de personalización para `login-id` o el prompt de inicio de sesión, según la configuración de tu tenant:

```bash lines
auth0 universal-login customize -p login-id
```

2. Selecciona **standard** como modo de renderización

```bash lines
Rendering Mode: standard
```

El editor de personalización se abre en una dirección local:

```bash lines
http://localhost:52649
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si no está seguro de si su tenant usa `login` o `login-id`, consulte la [personalización del prompt de Auth0](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts).
</Callout>

<div id="configure-the-checkbox">
  ### Configura la casilla de verificación
</div>

Abre la opción **Partials** en el editor de personalización:

1. Establece **Prompt** en: `login-id` (o `login`)

2. Establece **Screen** en: `login-id` (o `login`)

3. Establece **Partial** en: `form-content-end`

<Frame>![partials de Auth0 CLI](/docs/images/cdy7uua7fh8z/sessions-use-case/partials.png)</Frame>

Agrega el siguiente fragmento de HTML:

```html lines
<div class="ulp-field">
  <input type="checkbox" name="ulp-remember-me" id="remember-me">
  <label for="remember-me">Stay signed in on this device?</label>
</div>
```

Esto renderiza la casilla debajo del formulario de inicio de sesión.

<div id="deploy-your-changes">
  ### Implementa tus cambios
</div>

Selecciona **Implementar cambios** en la esquina superior derecha para guardar y aplicar la personalización.

<div id="checkpoint">
  ## Checkpoint
</div>

Inicia sesión con Universal Login.
La pantalla de inicio de sesión incluye una casilla **Keep Me Signed In**.

<div id="3-create-a-post-login-action-using-auth0-cli">
  ## 3. Crear una Action de Post Login con Auth0 CLI
</div>

Cree un nuevo archivo llamado `set-session-persistence.js` para usarlo con Auth0 CLI y agregue el siguiente código:

```javascript lines
exports.onExecutePostLogin = async (event, api) => {

  // Aplicar el cambio solo cuando el parámetro esté disponible
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode('persistent');
  }
};
```

Esta Action hace que la cookie de sesión se mantenga al reiniciar el navegador si el usuario marca la casilla. Para extender también la duración de la sesión en dispositivos de confianza, consulta [Establecer la duración de la sesión para dispositivos de confianza](#set-session-duration-for-trusted-devices) más abajo.

Usa Auth0 CLI para crear o actualizar tu Action de Post Login con el archivo `set-session-persistence.js`:

```bash lines
auth0 actions create --name kmsi --trigger post-login --code "$(cat set-session-persistence.js)"
```

Auth0 CLI devuelve el ID de la Action:

```bash lines

NAME           kmsi
TYPE           post-login
STATUS         pending
DEPLOYED       ✗
LAST DEPLOYED

CREATED        0 seconds ago

                   if (event.request.body['ulp-remember-me']) {
                     api.session.setCookieMode("persistent");
                   } else {
                     api.session.setCookieMode("non-persistent");
                   }
                 };
```

Implementa tu Action con su `id`:

```bash lines
auth0 actions deploy <id>
```

Ve a [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post Login](https://manage.auth0.com/#/actions/triggers/post-login) para añadir la nueva Action al flujo actual y selecciona **Aplicar**.

<div id="set-session-duration-for-trusted-devices">
  ### Configurar la duración de la sesión para dispositivos de confianza
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [`setCookieMode()`](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object#api-session-setcookiemode-mode) solo controla cómo se envía la cookie de sesión al navegador. Una cookie persistente se mantiene aunque se reinicie el navegador, mientras que una cookie no persistente (de sesión) se elimina al cerrar el navegador. No cambia el tiempo de expiración de la sesión del backend. Si quieres que los dispositivos de confianza tengan una sesión más larga, debes establecer explícitamente la expiración con `api.session.setExpiresAt()` o `api.session.setIdleExpiresAt()`.
</Callout>

Para dar a los dispositivos de confianza una sesión más larga, usa `api.session.setExpiresAt()` o `api.session.setIdleExpiresAt()` junto con `setCookieMode()`:

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode("persistent");

    const now = Date.now();
    api.session.setExpiresAt(now + 7 * 24 * 60 * 60 * 1000); // 7 días
  } else {
    api.session.setCookieMode("nonpersistent");
  }
};
```

<div id="4-test-implementation">
  ## 4. Probar la implementación
</div>

Para verificar el comportamiento de la sesión, puedes hacer lo siguiente:

Inicia sesión con Universal Login y selecciona la casilla **Mantener la sesión** iniciada.

* En los registros del tenant, busca (tipos de evento `s` o `ssa`)

  * `cookie: { mode: "non-persistent" }`

  * `cookie: { mode: "persistent" }`

* Busca `event.session.cookie.mode` en tus Actions
