Puedes ampliar las capacidades de Auth0 con metadato de [organización](/docs/es/manage-users/organizations/organizations-overview) y Actions, o usar nuestras API y nuestros SDKs para crear paneles de administración de organizaciones para tus usuarios.

<Card title="La disponibilidad varía según el plan de Auth0">
  Tu plan de Auth0 o tu acuerdo personalizado determina si esta funcionalidad está disponible. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Card>

<div id="extensibility">
  ## Extensibilidad
</div>

Organizations es compatible con nuestros puntos de extensibilidad, por lo que puedes definir propiedades en los metadatos de la organización y exponer esos datos a [Actions](/docs/es/customize/actions). Esto te permite personalizar funcionalidades para clientes o aplicaciones concretos; por ejemplo, puedes ejecutar lógica personalizada en Actions para determinados clientes en función de su plan de suscripción, almacenando esa información en los metadatos de la organización.

<div id="actions-event-object">
  ### Objeto de evento de Actions
</div>

El objeto de evento de Action almacena información contextual sobre la transacción de autenticación actual, como la dirección IP, la aplicación o la ubicación del usuario.

Si cambias el contenido del token usando el objeto `event` dentro de una Action, tus cambios estarán disponibles en los tokens cuando todas las Actions hayan terminado de ejecutarse.

<div id="sdks">
  ## SDKs
</div>

Para permitir que los miembros gestionen sus organizaciones por sí mismos, puedes asignarles roles y usar nuestra API y los SDKs para crear paneles en tus productos. Los administradores pueden configurar <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO), invitar usuarios a organizaciones, asignar miembros a organizaciones, asignar roles a los miembros, etc.

A continuación, se describen algunas tareas de ejemplo que quizá quieras realizar con organizaciones usando los SDKs.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los ejemplos siguientes hacen referencia al claim `org_id`, disponible de forma predeterminada en los tokens de ID y de acceso. Sin embargo, puedes configurar tu tenant para que también admita el uso de nombres de organización en la Authentication API. Esto da como resultado tokens que contienen tanto el claim `org_id` como el claim `org_name`. Si está presente, valida el claim `org_name` además de `org_id` para asegurarte de que los valores recibidos correspondan a una entidad de confianza.

  En general, usar IDs de organización es el método preferido para validar tokens. Sin embargo, los nombres de organización pueden usarse si resultan más adecuados para tu caso de uso. Para comprender las posibles implicaciones de usar nombres de organización para validar tokens, consulta [Use Organization Names in Authentication API](/docs/es/manage-users/organizations/configure-organizations/use-org-name-authentication-api).
</Callout>

<div id="i-want-users-to-log-in-to-a-specified-organization">
  ### Quiero que los usuarios inicien sesión en una organización específica
</div>

Al definir un nuevo cliente, pasa el ID de la organización en el parámetro `organization`. Luego, en la devolución de llamada, asegúrate de que la organización devuelta en el <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> sea la misma que se envió en la solicitud a `/authorize`, validando el `claim` `org_id` de la misma manera en que se validan otros claims, como `exp` y `nonce`.

<div id="from-my-application-i-want-to-get-the-organization-to-which-the-authenticated-user-logged-in">
  ### Desde mi aplicación, quiero obtener la organización con la que inició sesión el usuario autenticado
</div>

Si el usuario se autenticó mediante una organización, el ID de la organización aparecerá en el claim `org_id` del token de identidad. Con el Auth0 SPA SDK, esto se puede obtener de la siguiente manera:

`const { org_id } = await client.getIdTokenClaims();`

<div id="from-my-api-i-want-to-get-the-organization-with-which-the-access-token-was-issued">
  ### En mi API, quiero obtener la organización con la que se emitió el token de acceso
</div>

Si el usuario se autenticó usando una organización y se especificó una <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (Client ID) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip>, el <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (Client ID) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> será un <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> y contendrá el claim `org_id` con el ID de la organización en la que el usuario inició sesión.

Esto se puede validar junto con los demás claims en el backend, como en el siguiente ejemplo para Ruby:

```ruby lines
class JsonWebToken
  def self.verify(token)
    decoded = JWT.decode(token, nil,
               true, # Verificar la firma de este token
               algorithms: 'RS256',
               iss: 'https://YOUR_DOMAIN/',
               verify_iss: true,
               aud: Rails.application.secrets.auth0_api_audience,
               verify_aud: true) do |header|
      jwks_hash[header['kid']]
    end

    // Obtener el valor del ID de organización del token decodificado
    org = decoded[0]['org_id']
  end
end
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 se compone del nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`).

  Si usas dominios personalizados, este debería ser tu dominio personalizado.
</Callout>
