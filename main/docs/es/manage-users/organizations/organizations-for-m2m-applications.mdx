  Esta funcionalidad está disponible en los planes B2B Professional, Enterprise y Enterprise Premium. Para obtener más información, consulta [Precios](https://auth0.com/pricing).

El [flujo de credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow) lo usa una aplicación para obtener <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> para sí misma, en lugar de para un usuario. Esto resulta útil en casos de uso de máquina a máquina, como bots, CLI, procesos de backend o demonios, o cualquier aplicación que consuma API en su propio nombre. Aunque los usuarios finales no intervienen, el flujo de credenciales de cliente debe seguir limitándose por organización para garantizar que solo las aplicaciones autorizadas puedan acceder a los datos de cada organización.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para limitar cada solicitud a una organización determinada, incluye el claim `org_id` correspondiente en los tokens de máquina a máquina. Los servidores de la API deben tener en cuenta este claim y aplicarlo al autorizar el acceso a los recursos y datos de la API. Para obtener más información, consulta [Trabajar con tokens y Organizations](/docs/es/manage-users/organizations/using-tokens).
</Callout>

<div id="open-up-apis-to-third-party-applications">
  ## Abrir las API a aplicaciones de terceros
</div>

Con Auth0 Organizations y la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, Travel0 ofrece un portal de autoservicio donde los clientes pueden crear y gestionar su propia organización. Travel0 quiere facilitar que los clientes creen bots para ayudar a los usuarios finales a encontrar y comprar aventuras. Por ello, como parte de este portal, Travel0 permite a los clientes registrar sus aplicaciones (p. ej., los bots) para consumir la Travel0 API en su propio nombre mediante acceso máquina a máquina. En este caso de uso, el acceso entre organizaciones debe controlarse correctamente para que las aplicaciones que pertenecen a una organización solo puedan acceder a los datos de esa organización a través de la Travel0 API.

**Acceso de máquina a máquina para Organizations te permite configurar el acceso mediante credenciales de cliente para cada API asociándolo a una organización específica**. En el siguiente diagrama, las aplicaciones de nuestra organización de ejemplo, org&#95;X, solo pueden acceder a la Travel0 API dentro del ámbito de org&#95;X. También puedes configurar el acceso de una sola aplicación a varias organizaciones en los casos en que se usan agregadores. Con M2M Access for Organizations, controlas qué aplicaciones pueden acceder a una organización específica mediante acceso máquina a máquina para cada API.

<div id="isolation-of-organizations-for-internal-applications">
  ## Aislamiento de las organizaciones para aplicaciones internas
</div>

Travel0 también tiene algunos procesos internos y herramientas de CLI que necesitan acceder a la Travel0 API mediante el flujo de credenciales de cliente. Para aplicar una estrategia unificada de control de acceso en la API, Travel0 quiere que las solicitudes de sus propias aplicaciones se limiten a una organización específica para garantizar que, en cada caso, solo se acceda a los datos correctos.

<Frame>![](/docs/images/cdy7uua7fh8z/5bMEPlmTLGgBn8fq5Fp59S/fe874efcfba549114ad062bd7e3d874d/image7.png)</Frame>

Machine-to-Machine Access to Organizations le permite configurar una aplicación para que acceda a cualquier organización en cada API de su tenant mediante el flujo de credenciales de cliente.
