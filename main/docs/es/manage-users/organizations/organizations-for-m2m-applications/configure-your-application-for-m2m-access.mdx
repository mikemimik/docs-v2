Después de crear la aplicación, configúrala para el acceso de máquina a máquina siguiendo estos pasos:

1. [Definir el comportamiento de la organización](#define-organization-behavior) para cada API a la que necesites acceder.
2. En los casos en que la aplicación deba acceder solo a organizaciones específicas, es decir, no a todas, [autoriza el acceso de máquina a máquina](/docs/es/manage-users/organizations/organizations-for-m2m-applications/authorize-m2m-access) para cada API a la que necesites acceder en esas organizaciones específicas.

<div id="define-organization-behavior">
  ## Definir el comportamiento de la organización
</div>

Puede configurar cómo su aplicación utiliza Organizations durante el flujo de credenciales de cliente para cada API; por ejemplo, si la aplicación debe usar una organización o si puede acceder a cualquier organización o solo a aquellas asociadas de forma explícita.

La siguiente tabla explica los campos para definir el comportamiento de la organización para el acceso M2M:

<div id="define-organization-behavior-table">
  ### Tabla de comportamiento de la organización
</div>

| Campo                                                         | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                | Correspondencia con la API                                                                                                                                                                                                                                  |
| ------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Compatibilidad con organizaciones                             | Determina cómo esta aplicación puede usar organizaciones al acceder mediante el flujo de credenciales de cliente.<br /><br />Las opciones incluyen: <ul> <li>**None:** La aplicación *no debe* usar organizaciones. Este es el valor predeterminado.</li> <li>**Optional:** La aplicación *puede decidir* si usa una organización.</li> <li>**Required:** La aplicación *debe* usar una organización.</li> </ul>                                                                                                           | Compatibilidad con organizaciones se corresponde con `organization_usage`.<br /><br />Opciones: <ul> <li>**None** se corresponde con `deny`</li> <li>**Optional** se corresponde con `allow`</li> <li>**Required** se corresponde con `required`</li> </ul> |
| Permitir acceso de máquina a máquina a cualquier organización | Determina si esta aplicación puede acceder a cualquier organización o si queda limitada a un conjunto de organizaciones asociadas explícitamente cuando usa el flujo de credenciales de cliente. De forma predeterminada, este comportamiento no está permitido.<br /><br />**Advertencia:** Habilitar el acceso de máquina a máquina a cualquier organización permite que la aplicación acceda a cualquier organización sin estar asociada explícitamente. Solo debe habilitarse para aplicaciones internas de confianza. | Permitir acceso de máquina a máquina a cualquier organización se corresponde con `allow_any_organization`.<br /><br />Opciones: <ul> <li>Marcado se corresponde con `true`</li> <li>Desmarcado se corresponde con `false`</li> </ul>                        |

<div id="define-organization-behavior-for-an-application">
  ### Definir el comportamiento de la organización para una aplicación
</div>

Para definir el comportamiento de la organización para una aplicación, usa el [Auth0 Dashboard](https://manage.auth0.com/) o la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para una aplicación de máquina a máquina, define el comportamiento de la organización desde el Auth0 Dashboard:

    1. Ve a **Auth0 Dashboard &gt; Applications** y selecciona la aplicación para la que quieres configurar Organizations.
    2. Cambia a la pestaña **APIs** y expande los detalles de `client_grant` de la API a la que quieres configurar el acceso.
    3. Configura los ajustes correspondientes como se explica en la [tabla Definir el comportamiento de la organización](#define-organization-behavior-table).
    4. Selecciona **Save.**

    Como alternativa, si no es una aplicación de máquina a máquina:

    1. Ve a **Auth0 Dashboard &gt; APIs** y selecciona la API para la que quieres configurar el acceso a Organizations.
    2. Cambia a la pestaña **Machine To Machine Applications**. Muestra todas las aplicaciones, no solo las aplicaciones de máquina a máquina. Busca tu aplicación.
    3. Para autorizar el acceso, activa el interruptor de tu aplicación. Luego, haz clic en tu aplicación para expandir los detalles de configuración.
    4. Configura los ajustes correspondientes como se explica en la [tabla Definir el comportamiento de la organización](#define-organization-behavior-table).
    5. Selecciona **Save**.
  </Tab>

  <Tab title="Management API">
    También puedes definir el comportamiento de la organización para una aplicación con la Management API. El objeto `client_grant` controla el acceso de tu aplicación a una API, por lo que debes configurar el `client_grant` para cada API a la que tu aplicación necesite acceder. Para configurar el `client_grant` para el acceso de máquina a máquina, usa los endpoints [Create client grant](https://auth0.com/docs/api/management/v2/client-grants/post-client-grants) o [Update client grant](https://auth0.com/docs/api/management/v2/client-grants/patch-client-grants-by-id), y la [tabla Definir el comportamiento de la organización](#define-organization-behavior-table) para actualizar los ajustes correspondientes.

<div id="set-default-organization">
  ## Establecer la organización predeterminada
</div>

Algunos clientes no admiten campos no estándar en el [flujo de credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow). Por lo tanto, no pueden enviar el parámetro `organization` requerido al endpoint `/oauth/token`. En estos casos, puedes establecer una organización predeterminada que se aplique automáticamente a cualquier solicitud de credenciales de cliente de la aplicación cuando se especifique `none` y la API requiera soporte para organizaciones.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La organización predeterminada se aplica cuando `organization_usage=require` para la API (es decir, la audience de destino).
</Callout>

La siguiente tabla explica los campos para configurar una organización predeterminada para el acceso de máquina a máquina:

<div id="set-default-organization-table">
  ### Tabla para establecer la organización predeterminada
</div>

| Campo                                                        | Descripción                                                                                                     | API Mapping                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Organización predeterminada para acceso de máquina a máquina | Define qué organización se aplica a las solicitudes de credenciales de cliente que no incluyen una organización | Organización predeterminada para acceso de máquina a máquina corresponde al objeto default&#95;organization con las siguientes propiedades: <ul> <li>`organization_id`: El ID de la organización que se debe usar</li> <li> `flows`: El flujo de autenticación que se debe usar para la organización predeterminada.<br /><br />Esta configuración está oculta en la UI y su valor predeterminado es `["client_credentials"]` </li> </ul> |

<div id="set-default-organization-for-an-application">
  ### Establecer la organización predeterminada para una aplicación
</div>

Para establecer la organización predeterminada para una aplicación, usa el [Auth0 Dashboard](https://manage.auth0.com/) o la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para habilitar una organización predeterminada para una aplicación desde el Auth0 Dashboard:

    1. Ve a **Auth0 Dashboard &gt; Applications** y selecciona la aplicación que quieres configurar.
    2. Selecciona la pestaña **Organizations**.
    3. Configura los ajustes correspondientes como se explica en la [tabla Establecer la organización predeterminada](#set-default-organization-table).
    4. Haz clic en **Save.**
