title: Revocar acceso de máquina a máquina
---

Para revocar el acceso de máquina a máquina, o quitarle a tu aplicación el acceso a una organización, debes eliminar la asociación entre sus autorizaciones de cliente y la organización. Una vez eliminada esta asociación, la aplicación ya no podrá usar la organización al solicitar nuevos tokens para la API definida en la concesión.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Revocar el acceso de máquina a máquina no afecta a las aplicaciones a las que se les haya concedido acceso a alguna organización. Para obtener más información, consulta [Definir el comportamiento de la organización](/docs/es/manage-users/organizations/organizations-for-m2m-applications/configure-your-application-for-m2m-access#define-organization-behavior).
</Callout>

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para quitar la asociación entre la autorización de cliente de una aplicación y una organización a través del [Auth0 Dashboard](https://manage.auth0.com/):

    1. Ve a **Organizations** y elige la organización de la que deseas quitar la asociación.
    2. Selecciona la pestaña **Machine-to-Machine Access**.
    3. Elige una aplicación.
    4. Desmarca la casilla junto a las API que deseas desvincular de la organización.
    5. Haz clic en **Save**.

    <Frame>![](/docs/images/cdy7uua7fh8z/5MFv5NAoBDswloVweCVU1D/9057b94d5c5fd35e25afc7170ac645f3/Acme_Bot_-_Travel_Api_-_English.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Las asociaciones entre las autorizaciones de cliente de una aplicación y las organizaciones se pueden eliminar mediante el endpoint [Disassociate client grants from organizations](https://auth0.com/docs/api/management/v2/organizations/delete-client-grants-by-grant-id):

    ```bash lines
    curl -X DELETE --location "https://{yourDomain}/api/v2/organizations/{ORG_id}/client-grants/{GRANT_ID}" \
      --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' ;
    ```
  </Tab>
</Tabs>
