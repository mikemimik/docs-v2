[Auth0 Organizations](/docs/es/manage-users/organizations) permite a los responsables de productos B2B o aplicaciones SaaS crear arquitecturas multi-tenant, almacenar correctamente los tokens de identificación y minimizar la fricción en el inicio de sesión del usuario final.

<div id="configure-the-login-experience-of-your-application">
  ## Configura la experiencia de inicio de sesión de tu aplicación
</div>

Tu aplicación puede configurarse en la pestaña **Login Experience** para admitir tres tipos de usuarios:

1. Individuals
2. Business Users
3. Both

<Frame>![](/docs/images/cdy7uua7fh8z/Rz0i0zgIY7Yugx8M3D7pw/4cf50e68d32dc931ca18caae8cf48146/image3.png)</Frame>

Es probable que las aplicaciones diseñadas específicamente para consumidores —por ejemplo, Netflix o Spotify— no necesiten la gestión de organizaciones. Al elegir **Individuals**, los usuarios inician sesión directamente en la aplicación y no se proporciona contexto de organización.

Las aplicaciones B2B o SaaS —por ejemplo, Slack o Jira— suelen funcionar mejor con **Business Users**, de modo que los usuarios finales solo puedan acceder a tu aplicación en el contexto de una Auth0 Organization. Los usuarios que pertenecen a varias Organizations se redirigen al Organization Picker después del login flow, que muestra las primeras 20 organizaciones a las que se unieron.

<Frame>![](/docs/images/cdy7uua7fh8z/3b6s5amCEjFyRTKWkTrFZI/ae9cd3bcbd55d299d7bdddfb8747ea0b/image4.png)</Frame>

Elige **Both** si tu end user puede tener tanto una cuenta personal como una cuenta empresarial en tu aplicación. Por ejemplo, Github suele almacenar repositorios de código tanto personales como profesionales.

Puedes configurar el tipo de usuario de tu aplicación a través del <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> (como se describió anteriormente) o de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. En concreto, usa el parámetro `organization_usage` del endpoint [actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para establecer el tipo de usuario adecuado. Para obtener más información sobre ambos métodos, consulta [Define Organization Behavior](/docs/es/manage-users/organizations/configure-organizations/define-organization-behavior).

<div id="configure-the-login-flow-for-your-application">
  ## Configura el flujo de inicio de sesión para tu aplicación
</div>

Después de seleccionar **Business Users** o **Both**, puedes personalizar aún más la experiencia de tus usuarios al iniciar sesión en tu aplicación. La mayoría de las organizaciones deberían elegir **Prompt for Credentials** y luego habilitar [Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first). Si ya sabes en qué organización intenta iniciar sesión un usuario, la opción **No Prompt**, junto con [Custom Development with Organizations](/docs/es/manage-users/organizations/custom-development), permite que tu aplicación mantenga un flujo de inicio de sesión de marca y personalizado.

<Frame>![](/docs/images/cdy7uua7fh8z/3e18zLZ8dPeXBohvBLpKAU/8c8f9710b9fc71c681b6e777d90887c9/Login_Flow_-_Prompt_-_Enflish.png)</Frame>

Los administradores pueden ajustar aún más la experiencia del usuario final habilitando el interruptor **Prompt for Organization**, que exige a los usuarios identificar la organización en la que van a iniciar sesión mediante el nombre de la organización o el correo electrónico de la organización.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Incluso si seleccionas Prompt for Credentials, puedes seguir redirigiendo a los usuarios para [iniciar sesión con el prompt de inicio de sesión de una organización específica](/docs/es/manage-users/organizations/custom-development#i-want-users-to-log-in-to-a-specified-organization) cuando sea necesario.
</Callout>

Puedes configurar el flujo de inicio de sesión de tu aplicación desde el Auth0 Dashboard (como se describió anteriormente) o mediante la Management API. En concreto, usa el parámetro `organization_require_behavior` del endpoint [Actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para establecer el flujo adecuado. Para obtener más información sobre ambos métodos, consulta [Define Organization Behavior](/docs/es/manage-users/organizations/configure-organizations/define-organization-behavior).

<div id="identifier-first-authentication-with-prompt-for-credentials">
  ## Autenticación Identifier First con prompt de credenciales
</div>

Si su aplicación empresarial usa [federación empresarial](/docs/es/authenticate/enterprise-connections), puede activar [Identifier First Authentication with Home Realm Discovery](/docs/es/authenticate/login/auth0-universal-login/identifier-first) en su perfil de autenticación. Una vez habilitado, Home Realm Discovery detecta direcciones de correo electrónico de un dominio conocido y las redirige automáticamente al inicio de sesión de Workforce correspondiente.

<Frame>![](/docs/images/cdy7uua7fh8z/1vyoeNqhRmqP3iYP58X59Y/eaa1556a41b5f4d9a3211f877d284b36/image2.png)</Frame>

En este flujo, se puede usar exactamente una [conexión de base de datos](/docs/es/authenticate/database-connections/custom-db) de Auth0 como respaldo cuando el dominio del correo electrónico de un usuario no coincide con el dominio del <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> (IdP) de ninguna conexión empresarial. A los usuarios se les muestra el prompt de inicio de sesión de su aplicación en lugar del prompt de inicio de sesión de una organización, y [las conexiones habilitadas para la aplicación](/docs/es/get-started/applications/update-application-connections) están visibles para el usuario.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si su tenant de Auth0 tiene configurado un [Default Directory](https://manage.auth0.com/#/tenant/general) en la configuración, Auth0 lo usa como conexión para este flujo de inicio de sesión, incluso si Default Directory no está habilitado en la aplicación. Default Directory tiene prioridad sobre las conexiones habilitadas de la aplicación.

  Este comportamiento se aplica solo al flujo **Prompt for Credentials**. Para el flujo Prompt for Organization o los escenarios sin prompt, debe habilitar Default Directory en la aplicación para que Auth0 lo seleccione.
</Callout>

Después de que un usuario proporciona una dirección de correo electrónico, Auth0 la coteja con las conexiones empresariales habilitadas para esta aplicación y todas las conexiones empresariales habilitadas para Organizations. Si encuentra una coincidencia, el usuario es redirigido para autenticarse con el IdP asociado. Si no se encuentra ninguna coincidencia, se muestra un campo de contraseña.

<div id="identifier-first-authentication-with-prompt-for-organization">
  ## Identifier First Authentication con Prompt for Organization
</div>

<Frame>![](/docs/images/cdy7uua7fh8z/Rz0i0zgIY7Yugx8M3D7pw/IdentifierFirst_promptswithorgs.png)</Frame>

Cuando un usuario inicia la autenticación, Auth0 primero determina la organización (si corresponde) y luego identifica el proveedor de identidad (IdP) o la conexión correctos para la autenticación.

El usuario comienza ingresando su dirección de correo electrónico (por ejemplo, `alice@rockymountainadventures.com`) o el nombre de su organización (por ejemplo, `Rocky Mountain High Adventures`).

El contenido de la pantalla del prompt depende de la configuración **Login Experience &gt; Type of Users** de la aplicación:

* **Business Users** muestra solo el prompt de organización y los usuarios deben iniciar sesión con una organización.

* **Both** muestra tanto el prompt de organización como el de cuenta personal, y los usuarios pueden iniciar sesión con cualquiera de los dos.

<div id="organization-domain-discovery-optional">
  ### Organization Domain Discovery (opcional)
</div>

Auth0 admite Organization Domain Discovery, que puede usarse de forma opcional para detectar la organización del usuario antes de que se ejecute Identifier First Authentication.

Cuando está configurado, Organization Domain Discovery ayuda a Auth0 a detectar automáticamente la organización de un usuario o a acotar las opciones de organización cuando los usuarios introducen su dirección de correo electrónico. Organization Domain Discovery no restringe quién puede registrarse o iniciar sesión.

En conjunto, Identifier First Authentication y Organization Domain Discovery agilizan el flujo de inicio de sesión empresarial y, al mismo tiempo, mantienen la claridad y el control en entornos con múltiples organizaciones.

Si un tenant tiene [configurados dominios de organización verificados](./configure-organizations/create-org-domains) y no se proporciona ningún parámetro `organization`, Auth0 intenta determinar automáticamente la organización del usuario en función de los dominios verificados (no pendientes):

* Si coincide exactamente una organización, Auth0 la selecciona automáticamente y continúa el flujo de Identifier First Authentication en el contexto de esa organización.

* Si varias organizaciones comparten el mismo dominio o nombre (por ejemplo, si tanto AdventureZ como Granite Outpost configuraron `travelco.com`), Auth0 muestra un selector de organización para que el usuario pueda elegir. Una vez hecha la elección, Auth0 continúa el flujo de Identifier First Authentication.

* Si no se encuentra ninguna coincidencia, Auth0 continúa con el proceso estándar de Identifier First Authentication.

<div id="identifier-first-authentication">
  ## Identifier First Authentication
</div>

El objetivo de Identifier First Authentication es determinar a qué conexión o proveedor de identidad (IdP) debe dirigirse el usuario.

Una vez que se determina la organización (si existe), Auth0 aplica Identifier First Authentication con Home Realm Discovery:

* Si se proporciona un parámetro `connection`, Auth0 lo usa para dirigir al usuario a esa conexión específica.
  * De lo contrario, Auth0 examina todas las conexiones empresariales habilitadas tanto para la aplicación como para la organización (si corresponde).

Si se encuentra un IdP que coincida con el dominio de correo electrónico del usuario, Auth0 redirige al usuario a ese IdP para la autenticación.

Si no se encuentra ninguna coincidencia, o si el dominio de correo electrónico del usuario no corresponde a ningún IdP habilitado, Auth0 recurre a la conexión de base de datos de Auth0 (si hay una habilitada para la aplicación). En este caso, los usuarios ven el prompt de inicio de sesión de su aplicación en lugar del prompt de inicio de sesión de una organización, junto con todas las conexiones habilitadas para la aplicación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Hay casos de uso, como varias configuraciones de base de datos asignadas a diferentes Organizations, en los que Auth0 no puede determinar con qué IdP está asociado el correo electrónico de un usuario. En estos casos, seleccione Prompt for Organization como prompt de inicio de sesión o envíe el parámetro `organization` a Auth0.
</Callout>

Puede usar la Management API para configurar Identifier First Authentication. En concreto, use el parámetro `identifier_first` del endpoint [Update prompts settings](/docs/es/api/management/v2/prompts/patch-prompts).

<div id="auto-membership">
  ## Membresía automática
</div>

En lugar de invitar o asignar usuarios directamente a una organización, quizá quieras permitir que cualquier usuario que pueda autenticarse con un IdP federado obtenga acceso a una organización. Para estos casos, Auth0 recomienda la configuración [Auto-Membership](/docs/es/manage-users/organizations/configure-organizations/grant-just-in-time-membership).

La membresía automática suele activarse [dirigiendo al usuario para que inicie sesión mediante el prompt de inicio de sesión de la organización](/docs/es/manage-users/organizations/custom-development), que puede pasar los parámetros de conexión y organización en su nombre. Si no se puede determinar la organización deseada de un usuario antes del inicio de sesión, el flujo Prompt for Credentials concede membresía a la única organización que tenga configurada la membresía automática.

Sin embargo, puede haber casos en los que no puedas determinar la organización deseada de un usuario antes de enviarlo a iniciar sesión. En ese caso, puedes usar el flujo Prompt for Credentials mencionado anteriormente, pero ten en cuenta que al usuario solo se le concederá membresía en la organización si una, y solo una, organización tiene esta conexión configurada como conexión habilitada para la organización con la membresía automática activada.

Puedes usar la Management API para configurar la membresía automática. En concreto, usa el parámetro `assign_membership_on_login` del endpoint [Modify an organization&#39;s connection](/docs/es/api/management/v2/organizations/patch-enabled-connections-by-connection-id).
