Para empezar a usar la funcionalidad [Organizations](/docs/es/manage-users/organizations/organizations-overview), debes crear y configurar tu organización.

<Card title="La disponibilidad varía según el plan de Auth0">
  Tu plan de Auth0 o tu acuerdo personalizado determinan si esta funcionalidad está disponible. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Card>

<div id="create-an-organization">
  ## Crear una organización
</div>

Cuando [creas una organización](/docs/es/manage-users/organizations/configure-organizations/create-organizations), la configuración que definas se utiliza para personalizar la página de inicio de sesión y las plantillas de correo electrónico que ven los usuarios finales cuando se autentican en el contexto de la organización.

Al usar Organizations de forma predeterminada, esta configuración reemplazará la de las páginas de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>. Para obtener más información sobre cómo seguir personalizando la página de inicio de sesión o las plantillas de correo electrónico, consulta [Personalizar prompts y plantillas de correo electrónico](#customize-prompts-and-email-templates).

<div id="define-organization-behavior">
  ## Definir el comportamiento de la organización
</div>

Puede que conozca aplicaciones como Heroku que muestran comportamientos diferentes según si un usuario inicia sesión con una cuenta personal o selecciona una organización asociada a su cuenta empresarial. Auth0 le permite definir un comportamiento de la organización similar a nivel de la aplicación.

Para cada aplicación que cree en Auth0, puede decidir si los usuarios finales deben iniciar sesión directamente o si deben autenticarse en el contexto de una organización. En el caso de las aplicaciones que requieren que los usuarios inicien sesión a través de una organización, también puede especificar qué sucede si no se proporciona ninguna al flujo de inicio de sesión.

También puede actualizar la **Allowed Callback URLs** y los **Allowed Origins (CORs)** de su aplicación en la [configuración de la aplicación](/docs/es/get-started/applications/application-settings) para incluir [marcadores de posición para subdominios](/docs/es/get-started/applications/wildcards-for-subdomains) de organización.

<div id="customize-prompts-and-email-templates">
  ## Personaliza los prompts y las plantillas de correo electrónico
</div>

Al usar Organizations con los prompts predeterminados de Universal Login, la configuración de marca que estableces al [crear organizaciones](/docs/es/manage-users/organizations/configure-organizations/create-organizations) sustituye la configuración de marca de las páginas de Universal Login y de las plantillas de correo electrónico. Si quieres modificar aún más las páginas de Universal Login y los correos electrónicos que recibe el usuario final, puedes personalizar las plantillas de página y de correo electrónico.

<div id="page-templates">
  ### Plantillas de página
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para acceder a la personalización de las plantillas de página, debes usar dominios personalizados, una funcionalidad incluida en cualquier plan de pago.
</Callout>

Para modificar la página de Universal Login, personaliza una plantilla de página con el [lenguaje de plantillas Liquid](https://shopify.github.io/liquid/) y variables de plantilla, y luego aplícala mediante la API de Universal Login Page Templates. Para obtener más información, consulta [Universal Login Page Templates](/docs/es/customize/login-pages/universal-login/customize-templates).

Como se usa la misma plantilla para todas las páginas, puedes crear páginas de inicio de sesión coherentes con un esfuerzo mínimo.

La plantilla más sencilla que puedes escribir es:

```html lines
<!DOCTYPE html>
<html>
  <head>
    {%- auth0:head -%}
  </head>
  <body>
    {%- auth0:widget -%}
  </body>
</html>
```

Las siguientes etiquetas deben estar presentes en la plantilla:

* `auth0:widget`: Contiene el HTML que da estructura al widget que se muestra en cada tipo de página (por ejemplo, Login, Restablecer contraseña).
* **`auth0:head`**: Contiene las etiquetas necesarias para renderizar el widget.

Para centrar el widget en la página, reemplaza la etiqueta `<body>` por `<body class="_widget-auto-layout">`.

<div id="email-templates">
  ### Plantillas de correo electrónico
</div>

Para modificar los correos electrónicos que recibe el usuario final, personaliza una plantilla de correo electrónico con [Liquid template language](https://shopify.github.io/liquid/) y variables de plantilla. Para obtener más información, consulta [Personalizar plantillas de correo electrónico](/docs/es/customize/email/email-templates).

Las siguientes plantillas de correo electrónico pueden personalizarse aún más para Organizations:

* **Welcome**: Lo recibe el usuario final una vez que verifica su dirección de correo electrónico o, si la verificación de correo electrónico está deshabilitada, cuando se registra (o inicia sesión por primera vez).
* **Password Change**: Lo recibe el usuario final cuando solicita un cambio de contraseña. Contiene un enlace que lo redirige a la página de Restablecimiento de contraseña.
* **Invite User**: Lo recibe el usuario final cuando se le invita a una organización. Contiene un enlace que lo redirige a la ruta predeterminada de inicio de sesión que hayas configurado. Para obtener más información, consulta [Invite Organization Members](/docs/es/manage-users/organizations/configure-organizations/invite-members).

<div id="template-variables">
  ### Variables de plantilla
</div>

Las plantillas de página y de correo electrónico pueden acceder a un conjunto de variables de contexto que puedes usar para afectar la forma en que se renderiza la plantilla.

Para ver las listas de variables disponibles, consulta [Universal Login Page Templates](/docs/es/customize/login-pages/universal-login/customize-templates) para las variables de la plantilla de página y [Personalizar plantillas de correo electrónico](/docs/es/customize/email/email-templates) para las variables comunes de las plantillas de correo electrónico.

Hay variables adicionales disponibles para la plantilla de página cuando un usuario inicia sesión a través de una organización:

* `organization.id`
* `organization.display_name`
* `organization.name`
* `organization.metadata`
* `organization.branding.logo_url`
* `organization.branding.colors.primary`
* `organization.branding.colors.page_background`

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los parámetros de la organización solo se exponen en una plantilla de correo electrónico cuando el correo asociado se activa en el contexto de una organización. Por ejemplo, el correo electrónico de invitación de usuario siempre se activa en el contexto de una organización, por lo que los parámetros de la organización siempre se exponen en su plantilla. Sin embargo, los correos electrónicos de bienvenida pueden activarse o no en el contexto de una organización, por lo que los parámetros de la organización no siempre estarán disponibles en su plantilla.
</Callout>

<div id="configure-organization-domains">
  ## Configurar dominios de organización
</div>

<Warning>
  Organization Domains for Discovery se encuentra actualmente en acceso anticipado. Al usar esta funcionalidad, aceptas los términos aplicables de la prueba gratuita del [Master Subscription Agreement de Okta](https://www.okta.com/legal). Para obtener más información sobre las etapas de lanzamiento, consulta [Etapas de lanzamiento del producto](/docs/es/troubleshoot/product-lifecycle/product-release-stages).
</Warning>

Durante el pre-login, Auth0 puede detectar las Organizations de un usuario mediante el nombre de la organización o el dominio de correo electrónico que introduzca. Para asociar una organización con un dominio, puedes crear dominios de organización. Auth0 luego los redirige automáticamente al proveedor de identidad (IdP) correcto.

<Note>
  Nota: Los usuarios aún deben introducir su dirección de correo electrónico completa o el nombre de la organización durante el inicio de sesión. Auth0 usa esa información para determinar la organización asociada.
</Note>

<div id="add-a-domain-to-an-organization">
  ### Agregar un dominio a una organización
</div>

Los administradores del tenant pueden agregar y administrar dominios en la sección Domains de una organización en el Auth0 Dashboard o a través de la Management API.

Para agregar un dominio en el Auth0 Dashboard:

1. Vaya a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations).
2. Seleccione el nombre de la organización. Seleccione **Domains**.
3. Elija **Add Domain**.
4. Ingrese el dominio (por ejemplo, `travel0.com`)
5. Una vez verificado, el dominio aparecerá como Verified y se usará para la detección de organizaciones en pre-login.

<Note>
  Solo se usan los dominios verificados durante la detección de dominios. Los dominios pendientes o no verificados no activarán el enrutamiento.
</Note>

Para obtener más información sobre cómo la detección de dominios afecta el comportamiento de inicio de sesión, consulte [Login Flows for Organizations](https://auth0.com/docs/manage-users/organizations/login-flows-for-organizations).

<div id="configure-organization-membership">
  ## Configurar la membresía de la organización
</div>

Cada organización tiene sus propios miembros asociados; es decir, los usuarios que pueden acceder a sus aplicaciones iniciando sesión a través de la página de inicio de sesión configurada para la organización.

Las aplicaciones SaaS y B2B suelen ofrecerse a equipos, en lugar de a usuarios individuales aislados. En este contexto, un equipo podría ser una empresa entera, un pequeño grupo de empleados (por ejemplo, el departamento de marketing) o incluso un grupo temporal de usuarios organizados en torno a un objetivo (por ejemplo, un grupo de vecinos que se han registrado para colaborar con una organización benéfica y compiten como equipo para recaudar la mayor cantidad de dinero para una causa).

Organizations en Auth0 es lo suficientemente flexible como para admitir todos estos casos de uso, pero requiere cierta planificación al crear una aplicación compatible con equipos. Una de las primeras cosas que debe considerar es cómo gestionar la membresía del equipo, lo que puede lograr de las siguientes maneras:

* [Invitar usuarios por correo electrónico](/docs/es/manage-users/organizations/configure-organizations/invite-members)
* [Otorgar membresía Just-In-Time](/docs/es/manage-users/organizations/configure-organizations/grant-just-in-time-membership) a los usuarios que inicien sesión mediante una conexión habilitada
* Gestionar directamente la membresía a través de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> o el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> [asignando miembros](/docs/es/manage-users/organizations/configure-organizations/assign-members) o [eliminando miembros](/docs/es/manage-users/organizations/configure-organizations/remove-members)

Una vez definida la membresía, puede [obtener los miembros de las organizaciones](/docs/es/manage-users/organizations/configure-organizations/retrieve-members) o [obtener la membresía de organización de un usuario](/docs/es/manage-users/organizations/configure-organizations/retrieve-user-membership).

<div id="assign-roles-to-members">
  ## Asignar roles a los miembros
</div>

A cada miembro de la organización se le puede asignar uno o más roles, que se aplican cuando los usuarios inician sesión a través de la organización. En concreto, puedes [agregar roles a los miembros de la organización](/docs/es/manage-users/organizations/configure-organizations/add-member-roles), [eliminar roles de los miembros de la organización](/docs/es/manage-users/organizations/configure-organizations/remove-member-roles) o [obtener los roles de los miembros de una organización](/docs/es/manage-users/organizations/configure-organizations/retrieve-member-roles). Puedes usar los roles y sus permisos asociados para controlar el acceso a las [aplicaciones que crees](/docs/es/manage-users/organizations/custom-development). Para obtener más información sobre los roles, consulta [control de acceso basado en roles](/docs/es/manage-users/access-control/rbac).
