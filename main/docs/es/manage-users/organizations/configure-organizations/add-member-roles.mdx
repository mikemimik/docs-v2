A cada miembro de la [organización](/docs/es/manage-users/organizations/organizations-overview) se le puede asignar uno o más roles, que se aplican cuando los usuarios inician sesión a través de la organización. Para obtener más información sobre los roles y su comportamiento, consulta [Control de acceso basado en roles](/docs/es/manage-users/access-control/rbac).

Puedes agregar roles a miembros de Organizations mediante el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Para habilitar un rol para un miembro de la organización, primero debes haber [creado el rol](/docs/es/manage-users/access-control/configure-core-rbac/roles/create-roles) en tu tenant.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para agregar roles a un miembro de una organización desde el Auth0 Dashboard:

1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización cuya membresía quieres configurar.
2. Selecciona la vista **Members** y luego el nombre del miembro al que quieras agregarle un rol.
3. Selecciona **Assign** rol.
4. Introduce el nombre del rol o los roles que quieras asignarle al miembro y selecciona **Add rol(s) to organization**.

<div id="management-api">
  ## Management API
</div>

Para agregar roles a un miembro de la organización mediante la Management API:
Haz una llamada `POST` al endpoint `Create Organization Member Roles`. Asegúrate de reemplazar los valores de marcador de posición `ORG_ID`, `MGMT_API_ACCESS_TOKEN`, `USER_ID` y `ROLE_ID` por tu ID de organización, el <Tooltip tip="Access Token: credencial de Authorization, en forma de una cadena opaque o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> de la Management API, el user ID y el ID del rol, respectivamente.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 es el nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020), más `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, el nombre de tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`).

  Si usas dominios personalizados, este debe ser tu dominio personalizado.
</Callout>

| Valor                   | Descripción                                                                                                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ORG_ID`                | ID de la organización a cuyo miembro quieres agregar roles.                                                                                            |
| `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organization_member_roles`. |
| `USER_ID`               | ID del usuario al que quieres agregar los roles especificados.                                                                                         |
| `ROLE_ID`               | ID del rol que quieres agregar al usuario especificado para la organización especificada. Máximo de 100 roles por usuario.                             |

<div id="response-status-codes">
  ##### Códigos de estado de respuesta
</div>

Los posibles códigos de estado de respuesta son los siguientes:

| Código de estado | Código de error        | Mensaje                                                                                                         | Causa                                                                                           |
| ---------------- | ---------------------- | --------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `204`            |                        | Roles asociados correctamente al usuario.                                                                       |                                                                                                 |
| `400`            | `invalid_body`         | Cuerpo de la solicitud no válido. El mensaje variará según la causa.                                            | El payload de la solicitud no es válido.                                                        |
| `400`            | `invalid_query_string` | Cadena de consulta de la solicitud no válida. El mensaje variará según la causa.                                | La cadena de consulta no es válida.                                                             |
| `401`            |                        | Token no válido.                                                                                                |                                                                                                 |
| `401`            |                        | Se recibió una firma no válida para la validación de JSON Web Token.                                            |                                                                                                 |
| `401`            |                        | El cliente no es global.                                                                                        |                                                                                                 |
| `403`            | `insufficient_scope`   | Scope insuficiente; se esperaba cualquiera de los siguientes: `create:organization_member_roles`.               | Se intentó leer o escribir un campo no permitido con los scopes del bearer token proporcionado. |
| `429`            |                        | Demasiadas solicitudes. Compruebe los encabezados X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                 |
