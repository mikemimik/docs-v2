Puedes habilitar conexiones específicas para cada [Organización](/docs/es/manage-users/organizations/organizations-overview) a fin de ofrecer a los usuarios distintas opciones de inicio de sesión. Después de habilitar una conexión, esta se agrega al prompt de inicio de sesión de la organización, y los usuarios pueden autenticarse con esa conexión para acceder a tus aplicaciones.

Para habilitar una conexión para una organización, la conexión **debe** existir ya en tu tenant. Entre las conexiones compatibles se incluyen las [conexiones de base de datos](/docs/es/authenticate/database-connections), las [conexiones sociales](/docs/es/authenticate/identity-providers/social-identity-providers) y las [conexiones empresariales](/docs/es/authenticate/identity-providers/enterprise-identity-providers).

<div id="organization-properties">
  ## Propiedades de la organización
</div>

Al usar Organizations, algunas conexiones tienen propiedades adicionales que puedes configurar:

| **Propiedad**                                        | **Tipos de conexión**            | **Descripción**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ---------------------------------------------------- | -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Nombre de la conexión de la organización             | Todos los tipos de conexión      | Este nombre lo proporciona el administrador de la organización mediante la [My Organization API](/docs/es/manage-users/my-organization-api). Si no se completa, especifica el nombre que quieres que vean los administradores de la organización al consultar esta conexión.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Nivel de acceso del administrador de la organización | Todos los tipos de conexión      | Esta propiedad determina qué nivel de acceso tienen los administradores de la organización para esta conexión. Los valores disponibles son: <ul><li>**None**: Gestionado por el administrador del tenant; no es visible ni editable mediante la My Organization API.</li> <li>**Solo lectura**: Gestionado por el administrador del tenant; visible mediante la My Organization API.</li> <li>**Limitado**: Los administradores de la organización pueden editar configuraciones seleccionadas no sensibles mediante la My Organization API.</li> <li>**Completo**: Los administradores de la organización pueden editar toda la configuración permitida para esta conexión, con sujeción a las restricciones definidas por el [perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile).</li></ul>                                                                                          |
| membresía al autenticarse                            | Todos los tipos de conexión      | Cuando está habilitada, esta propiedad asigna automáticamente membresía a una organización a los usuarios finales la primera vez que se autentican con la conexión. La membresía al autenticarse es útil en situaciones en las que puede asumirse que todos los usuarios que pueden autenticarse con una conexión específica son miembros de una organización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| registro en la organización                          | Solo conexiones de base de datos | Esta propiedad determina si los usuarios finales pueden acceder a un enlace de registro en el prompt de inicio de sesión que les permite obtener membresía en una organización. Para usar el registro en la organización, también debes habilitar la membresía al autenticarse. El registro en la organización es útil en situaciones en las que los usuarios necesitan acceso de autoservicio para unirse a Organizations, como: <ul><li>Cuando se usan Organizations para modelar poblaciones de usuarios con membresía abierta, como espacios de trabajo.</li> <li>En casos de uso business-to-business-to-consumer (B2B2C) en los que los usuarios pueden registrarse libremente para crear cuentas.</li></ul>                                                                                                                                                                                                    |
| Mostrar la conexión como un botón                    | Solo conexiones empresariales    | Opcional. Esta propiedad determina si una conexión específica se muestra como opción en el prompt de inicio de sesión de la organización. Los usuarios finales también pueden autenticarse usando el perfil de autenticación [Identifier First with Home Realm Discovery](/docs/es/authenticate/login/auth0-universal-login/identifier-first) en combinación con el flujo de inicio de sesión de la organización [Prompt for Credentials](/docs/es/manage-users/organizations/login-flows-for-organizations#configure-the-login-flow-for-your-application). Esto solo funciona si **no** proporcionas un `org_id` con la solicitud de autorización.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Si esta opción está deshabilitada, los usuarios finales igualmente pueden autenticarse mediante la conexión enviando el parámetro `connection` directamente en la solicitud de autorización. </Callout> |

<div id="configure-organization-connections">
  ## Configurar conexiones de Organizations
</div>

Puede configurar las conexiones de las Organizations mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    #### Auth0 Dashboard

    Para habilitar una conexión en Auth0 Dashboard:

    1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización que quieres configurar.
    2. Selecciona la pestaña **Connections** y luego **Add Connections**.
    3. Elige la conexión que quieres habilitar y selecciona **Add Connection**.
    4. En la sección Organization Connection, configura ambas propiedades necesarias para usar una conexión mediante My Organization API:

    * Busca **Organization Connection Name**. Si no está completo, especifica el nombre que quieres que los administradores de la organización vean al consultar esta conexión.
    * Busca **Organization Admin Access Level** y, si no está completo, elige el nivel de acceso que quieres que los administradores de la organización tengan para esta conexión mediante My Organization API.

    5. En la sección Authentication, busca **Membership On Authentication** y elige si quieres habilitar o deshabilitar la membresía automática. Cuando está habilitada, la membresía automática agrega automáticamente como miembros de la organización a todos los usuarios que inician sesión con la conexión.
    6. **Solo para conexiones de base de datos**: En la sección Organization Signup, elige si quieres habilitar o deshabilitar el registro de autoservicio. Cuando está habilitado, los usuarios pueden acceder a un enlace de registro en el prompt de inicio de sesión para crear su cuenta y obtener automáticamente membresía en la organización.

       * Para habilitar esta propiedad, primero debes habilitar **Membership on Authentication**.
       * **Nota**: Organization Signup tiene prioridad sobre la configuración de registro de la conexión de base de datos. Cuando Organization Signup está habilitado, los usuarios pueden seguir registrándose en una organización o aceptar una invitación a una organización, aunque el registro esté deshabilitado en la conexión de base de datos.
    7. **Solo para conexiones empresariales**: En la sección Connection button, habilita opcionalmente la propiedad **Display connection as a button** para mostrar la conexión como una opción en el prompt de inicio de sesión de la organización.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         Si todas las conexiones habilitadas dentro de la organización son conexiones empresariales y todas las conexiones están ocultas, Auth0 devuelve un error que dice `Message: no connections enabled for the organization are visible` cuando los usuarios acceden a la aplicación.
       </Callout>
    8. Selecciona **Save**.
  </Tab>

  <Tab title="Management API">
    #### Management API

    Para habilitar una conexión mediante la Management API:

    Realiza una llamada `POST` al endpoint `Create Organization Connections`. Asegúrate de reemplazar los siguientes valores de marcador de posición con la información correspondiente:

    * Reemplaza `{orgId}` por tu Organization ID.
    * Reemplaza `{mgmtApiAccessToken}` por tu token de acceso de la Management API.
    * Reemplaza `{connectionId}` por un ID de conexión específico.
    * Reemplaza `{assignMembershipOption}` por `true` o `false` según tu selección de membresía al autenticarse.
    * Reemplaza `{isEnabled}` por `true` o `false` según quieras habilitar la conexión o no.
    * **Solo para Database Connections**: Reemplaza `{isSignupEnabled}` por `true` o `false` según tu selección de registro.
    * **Solo para conexiones empresariales**: Reemplaza `{showAsButtonOption}` por `true` o `false` según tu selección de botón de conexión.
    * **Para usar My Organization API**: Reemplaza `{organizationAccessLevel}` y `{organizationConnectionName}` por los valores correspondientes.

    | ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `{orgId}`                      | ID de la organización para la que desea habilitar una conexión.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
    | `{mgmtApiAccessToken}`         | [Token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organization_connections`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
    | `{connectionId}`               | ID de la conexión que desea habilitar para la organización especificada.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
    | `{assignMembershipOption}`     | Indica si desea que los usuarios que inicien sesión con esta conexión obtengan automáticamente la pertenencia a la organización. Cuando se establece en `true`, los usuarios obtendrán automáticamente la pertenencia. Cuando se establece en `false`, no la obtendrán automáticamente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
    | `{isEnabled}`                  | Habilita o deshabilita una conexión para una Organización.     | `{isSignupEnabled}`            | Determina si los usuarios pueden acceder a un enlace de registro de autoservicio en el prompt de inicio de sesión. Cuando se establece en `true`, el enlace de registro se muestra en el prompt. Cuando se establece en `false`, el enlace permanece oculto. **Nota**: Para habilitar esta opción, también debes establecer `{assignMembershipOption}` en `true`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
    | `{showAsButtonOption}`         | Indica si desea que una conexión empresarial específica se muestre como opción en el prompt de inicio de sesión de la organización. Cuando se establece en `true`, la conexión se muestra como un botón en el prompt. Cuando se establece en `false`, la conexión se oculta en el prompt.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
    | `{organizationAccessLevel}`    | Determina qué tipo de acceso tendrá un administrador de la organización al usar My Organization API. Cuando se establece en `none`, la conexión será administrada por el Tenant Admin y no será visible ni editable a través de My Organization API. Cuando se establece en `read-only`, la conexión será administrada por el Tenant Admin y será visible a través de My Organization API, pero no se podrá editar. Cuando se establece en `limited`, los administradores de la organización pueden editar determinadas configuraciones no sensibles a través de My Organization API. Cuando se establece en `full`, los administradores de la organización pueden editar todas las configuraciones permitidas para esta conexión, sujetas a las restricciones definidas por el [perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile). |
    | `{organizationConnectionName}` | Este nombre lo proporciona el Admin de la organización a través de My Organization API. Si no se ha indicado, especifique el nombre que desea que vean los Admins de la organización al consultar esta conexión.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

    ##### Códigos de estado de respuesta

    Los posibles status codes de respuesta son los siguientes:

    | **Código de estado** | **Código de error**  | **Mensaje**                                                                                                  | **Causa**                                                                                              |
    | -------------------- | -------------------- | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------ |
    | `201`                |                      | Conexión añadida correctamente a una Organización.                                                           |                                                                                                        |
    | `401`                |                      | Token no válido.                                                                                             |                                                                                                        |
    | `401`                |                      | Se recibió una firma no válida durante la validación del JSON Web Token.                                     |                                                                                                        |
    | `401`                |                      | El cliente no es global.                                                                                     |                                                                                                        |
    | `403`                | `insufficient_scope` | Scope insuficiente; se esperaba alguno de los siguientes: `create:organizations_connections`.                | Se intentó leer/escribir un field que no está permitido con los scopes del bearer token proporcionado. |
    | `429`                |                      | Demasiadas solicitudes. Revise los encabezados X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                        |
