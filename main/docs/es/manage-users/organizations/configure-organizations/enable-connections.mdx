Puedes habilitar conexiones específicas para cada [organización](/docs/es/manage-users/organizations/organizations-overview) para ofrecer a los usuarios distintas opciones de inicio de sesión. Después de habilitar una conexión, esta se agrega al prompt de inicio de sesión de la organización, y los usuarios pueden autenticarse mediante esa conexión para acceder a tus aplicaciones.

Para habilitar una conexión para una organización, la conexión **debe** existir previamente en tu tenant. Las conexiones compatibles incluyen [conexiones de base de datos](/docs/es/authenticate/database-connections), [conexiones sociales](/docs/es/authenticate/identity-providers/social-identity-providers) y [conexiones empresariales](/docs/es/authenticate/identity-providers/enterprise-identity-providers).

<div id="organization-properties">
  ## Propiedades de la organización
</div>

Al usar organizaciones, algunas conexiones tienen propiedades adicionales que puedes configurar:

| **Propiedad**                                        | **Tipos de conexión**            | **Descripción**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------------------------------------------- | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Nombre de la conexión de la organización             | Todos los tipos de conexión      | Este nombre lo proporciona el administrador de la organización mediante la [My Organization API](/docs/es/manage-users/my-organization-api). Si no se completa, especifica el nombre que quieres que los administradores de la organización vean al consultar esta conexión.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Nivel de acceso del administrador de la organización | Todos los tipos de conexión      | Esta propiedad determina qué nivel de acceso tienen los administradores de la organización para esta conexión. Los valores disponibles son: <ul><li>**None**: Lo administra el administrador del tenant; no es visible ni editable mediante la My Organization API.</li> <li>**Solo lectura**: Lo administra el administrador del tenant; es visible mediante la My Organization API.</li> <li>**Limitado**: Los administradores de la organización pueden editar configuraciones seleccionadas no sensibles mediante la My Organization API.</li> <li>**Completo**: Los administradores de la organización pueden editar toda la configuración permitida para esta conexión, con sujeción a las restricciones definidas por el [perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile).</li></ul>                                                                                 |
| Membresía al autenticarse                            | Todos los tipos de conexión      | Cuando está habilitada, esta propiedad asigna automáticamente la membresía de la organización a los usuarios finales la primera vez que se autentican con la conexión. La membresía al autenticarse resulta útil en escenarios en los que puede asumirse que todos los usuarios con capacidad para autenticarse con una conexión específica son miembros de una organización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Registro en la organización                          | Solo conexiones de base de datos | Esta propiedad determina si los usuarios finales pueden acceder a un enlace de registro en el prompt de inicio de sesión que les permite obtener membresía en una organización. Para usar el registro en la organización, también debes habilitar la membresía al autenticarse. El registro en la organización resulta útil en escenarios en los que los usuarios necesitan acceso de autoservicio para unirse a organizaciones, como: <ul><li>Cuando las organizaciones se usan para modelar poblaciones de usuarios con membresía abierta, como los espacios de trabajo.</li> <li>En casos de uso business-to-business-to-consumer (B2B2C) en los que los usuarios pueden registrarse libremente para obtener cuentas.</li></ul>                                                                                                                                                                                                  |
| Mostrar la conexión como un botón                    | Solo conexiones empresariales    | Opcional. Esta propiedad determina si una conexión específica se muestra como opción en el prompt de inicio de sesión de la organización. Los usuarios finales también pueden autenticarse usando el perfil de autenticación [Identifier First with Home Realm Discovery](/docs/es/authenticate/login/auth0-universal-login/identifier-first) en combinación con el flujo de inicio de sesión de la organización [Prompt for Credentials](/docs/es/manage-users/organizations/login-flows-for-organizations#configure-the-login-flow-for-your-application). Esto solo funciona si **no** proporcionas un `org_id` con la solicitud de autorización.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Si esta opción está deshabilitada, los usuarios finales aún pueden autenticarse mediante la conexión enviando directamente el parámetro de conexión en la solicitud de autorización. </Callout> |

<div id="configure-organization-connections">
  ## Configurar conexiones de organización
</div>

Puede configurar las conexiones de las organizaciones mediante el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    #### Auth0 Dashboard

    Para habilitar una conexión en Auth0 Dashboard:

    1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización que quieres configurar.
    2. Selecciona la pestaña **Connections** y luego **Add Connections**.
    3. Elige la conexión que quieres habilitar y selecciona **Add Connection**.
    4. En la sección Organization Connection, configura las dos propiedades necesarias para usar una conexión mediante My Organization API:

    * Busca **Organization Connection Name**. Si no está completado, especifica el nombre que quieres que los administradores de la organización vean al consultar esta conexión.
    * Busca **Organization Admin Access Level** y, si no está completado, elige el nivel de acceso que quieres que los administradores de la organización tengan para esta conexión mediante My Organization API.

    5. En la sección Authentication, busca **Membership On Authentication** y elige si quieres habilitar o deshabilitar la membresía automática. Cuando está habilitada, la membresía automática agrega automáticamente como miembros de la organización a todos los usuarios que inician sesión con la conexión.
    6. **Solo para conexiones de base de datos**: En la sección Organization Signup, elige si quieres habilitar o deshabilitar los registros de autoservicio. Cuando está habilitado, los usuarios pueden acceder a un enlace de registro en el prompt de inicio de sesión para crear su cuenta y obtener automáticamente membresía en la organización.

       * Para habilitar esta propiedad, primero debes habilitar **Membership On Authentication**.
       * **Nota**: Organization Signup tiene prioridad sobre la configuración de registro de la conexión de base de datos. Cuando Organization Signup está habilitado, los usuarios pueden seguir registrándose en una organización o aceptar una invitación a una organización, incluso si los registros están deshabilitados en la conexión de base de datos.
    7. **Solo para conexiones empresariales**: En la sección Connection button, habilita opcionalmente la propiedad **Display connection as a button** para mostrar la conexión como una opción en el prompt de inicio de sesión de la organización.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         Si todas las conexiones habilitadas dentro de la organización son conexiones empresariales y todas las conexiones están ocultas, Auth0 devuelve un error que dice `Message: no connections enabled for the organization are visible` cuando los usuarios acceden a la aplicación.
       </Callout>
    8. Selecciona **Save**.
  </Tab>

  <Tab title="Management API">
    #### Management API

    Para habilitar una conexión mediante la Management API:

    Realiza una llamada `POST` al endpoint `Create Organization Connections`. Asegúrate de reemplazar los siguientes valores de marcador de posición con la información correspondiente:

    * Reemplaza `{orgId}` por tu ID de la organización.
    * Reemplaza `{mgmtApiAccessToken}` por tu token de acceso de la Management API.
    * Reemplaza `{connectionId}` por un ID de conexión específico.
    * Reemplaza `{assignMembershipOption}` por `true` o `false`, según tu selección de Membresía en la autenticación.
    * Reemplaza `{isEnabled}` por `true` o `false`, según si quieres habilitar la conexión o no.
    * **Solo para conexiones de base de datos**: Reemplaza `{isSignupEnabled}` por `true` o `false`, según tu selección de registro.
    * **Solo para conexiones empresariales**: Reemplaza `{showAsButtonOption}` por `true` o `false`, según la opción que hayas seleccionado para el botón de conexión.
    * **Para usar My Organization API**: Reemplaza `{organizationAccessLevel}` y `{organizationConnectionName}` por los valores adecuados.

      **Encuentra tu dominio de Auth0**

      Tu dominio de Auth0 se compone del nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`).

      Si usas [dominios personalizados](/docs/es/customize/custom-domains), este debería ser tu dominio personalizado.

    | `{orgId}`                      | ID de la organización para la que desea habilitar una conexión.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |

Cuando se establece en `true`, el enlace de registro se muestra en el prompt. Cuando se establece en `false`, el enlace permanece oculto. **Nota**: Para habilitar esta opción, también debes configurar `{assignMembershipOption}` como `true`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
    | `{showAsButtonOption}`         | Indica si quiere que una conexión empresarial específica aparezca como opción en el prompt de inicio de sesión de la organización. Cuando se establece en `true`, la conexión aparece como un botón en el prompt. Cuando se establece en `false`, la conexión se oculta en el prompt.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
    | `{organizationAccessLevel}`    | Determina qué tipo de acceso tendrá un administrador de la organización al usar la My Organization API. Cuando se establece en `none`, la conexión será administrada por el administrador del tenant y no será visible ni editable a través de la My Organization API. Cuando se establece en `read-only`, la conexión será administrada por el administrador del tenant y será visible a través de la My Organization API, pero no se podrá editar. Cuando se establece en `limited`, los administradores de la organización pueden editar determinadas configuraciones no sensibles a través de la My Organization API. Cuando se establece en `full`, los administradores de la organización pueden editar todas las configuraciones permitidas para esta conexión, con sujeción a las restricciones definidas por el [Perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile). |
    | `{organizationConnectionName}` | Este nombre lo proporciona el Admin de la organización a través de My Organization API. Si no se completa, especifique el nombre que le gustaría que los Admins de la organización vieran al consultar esta conexión.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

    ##### Códigos de estado de respuesta

    Los posibles status codes de respuesta son los siguientes:

    | **Código de estado** | **Código de error**  | **Mensaje**                                                                                              | **Causa**                                                                                       |
    | -------------------- | -------------------- | -------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
    | `201`                |                      | Conexión agregada correctamente a una organización.                                                      |                                                                                                 |
    | `401`                |                      | Token no válido.                                                                                         |                                                                                                 |
    | `401`                |                      | Se recibió una firma no válida durante la validación del JSON Web Token.                                 |                                                                                                 |
    | `401`                |                      | El cliente no es global.                                                                                 |                                                                                                 |
    | `403`                | `insufficient_scope` | Scope insuficiente; se esperaba alguno de: `create:organizations_connections`.                           | Se intentó leer o escribir un field no permitido con los scopes del bearer token proporcionado. |
    | `429`                |                      | Demasiadas solicitudes. Revise los headers X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                 |
