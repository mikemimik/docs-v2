Al trabajar con [organizations](/docs/es/manage-users/organizations/organizations-overview) de forma programática, es posible que necesites recuperar una lista de organizaciones de las que un usuario es miembro.

<div id="management-api">
  ## Management API
</div>

<Warning>
  Se pueden mostrar hasta 1000 miembros de la organización con el Auth0 Dashboard o la Management API, aunque puede haber más.
</Warning>

Haz una llamada `GET` al endpoint `Get User Organizations`. Asegúrate de reemplazar los valores de marcador de posición `USER_ID` y `MGMT_API_ACCESS_TOKEN` por el ID del usuario y tu <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip>, respectivamente.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 se compone del nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, el nombre de tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`.)

  Si usas dominios personalizados, este debe ser el nombre de tu dominio personalizado.
</Callout>

| Valor | Descripción |
| `USER_ID` | ID del usuario del que quieres obtener la membresía de la organización. |
| `MGMT_API_ACCESS_TOKEN` | [token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `read:organizations`. |

<div id="response-status-codes">
  ### Códigos de estado de respuesta
</div>

Los posibles códigos de estado de respuesta son los siguientes:

| Código de estado | Código de error        | Mensaje                                                                                                        | Causa                                                                                           |
| ---------------- | ---------------------- | -------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `200`            |                        | Organizations obtenidas correctamente.                                                                         |                                                                                                 |
| `400`            | `invalid_uri`          | `invalid_request_uri`                                                                                          | La ruta no es válida.                                                                           |
| `400`            | `invalid_query_string` | La cadena de consulta de la solicitud no es válida. El mensaje variará según la causa.                         | La cadena de consulta no es válida.                                                             |
| `401`            |                        | Token no válido.                                                                                               |                                                                                                 |
| `401`            |                        | Se recibió una firma no válida para la validación de JSON Web Token.                                           |                                                                                                 |
| `403`            | `insufficient_scope`   | Permisos insuficientes; se esperaba alguno de los siguientes: `read:organizations`.                            | Se intentó leer o escribir un campo no permitido con los scopes del bearer token proporcionado. |
| `404`            |                        | Usuario no encontrado                                                                                          |                                                                                                 |
| `429`            |                        | Demasiadas solicitudes. Consulta los encabezados X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                 |
