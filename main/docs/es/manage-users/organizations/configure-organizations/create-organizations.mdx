<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para crear una organización desde el Auth0 Dashboard:

1. Vaya a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations).
2. Seleccione **Crear organización**.
3. Ingrese la información básica de su organización y seleccione **Agregar organización**:

   | **Campo**               | **Descripción**                                                                                                                                                                                                                                                                                                                                                           |
   | ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Nombre**              | Nombre de la organización que desea crear. Este es el nombre que un usuario final escribiría en el prompt de pre-login para identificar a través de qué organización quiere iniciar sesión. Identificador lógico único. Puede contener letras minúsculas, números, guiones bajos (`_`) y guiones (`-`). Puede comenzar con un número. Debe tener entre 1 y 50 caracteres. |
   | **Nombre para mostrar** | Nombre descriptivo que se mostrará.                                                                                                                                                                                                                                                                                                                                       |
4. Busque la sección **Branding** y personalice su organización; luego, seleccione **Guardar cambios**:

   | **Campo**                       | **Descripción**                                                                                          |
   | ------------------------------- | -------------------------------------------------------------------------------------------------------- |
   | **Logo de la organización**     | Logo que se mostrará. La resolución mínima recomendada es de 200 píxeles (ancho) por 200 píxeles (alto). |
   | **Color primario**              | Color de los elementos principales.                                                                      |
   | **Color de fondo de la página** | Color del fondo.                                                                                         |
5. Busque la sección **Metadata** y agregue a la organización los pares clave/valor de metadatos necesarios; luego, seleccione **Agregar**.

<div id="management-api">
  ## Management API
</div>

Para crear una organización a través de la Management API:

Realiza una solicitud `POST` al endpoint `Create Organizations`. Asegúrate de reemplazar los valores de marcador de posición por los valores correspondientes de tu tenant. Para obtener más detalles, consulta la tabla de parámetros a continuación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 es el nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`).

  Si usas dominios personalizados, este debe ser tu dominio personalizado.
</Callout>

| **Valor**                  | **Descripción**                                                                                                                                                                                                                                                                                                                                                             |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN`    | [Token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organizations`.                                                                                                                                                                                                                               |
| `ORG_NAME`                 | Nombre de la organización que quieres crear. Este es el nombre que un usuario final escribiría en el prompt de pre-login para identificar a través de qué organización quiere iniciar sesión. Identificador lógico único. Puede contener letras minúsculas, números, guiones bajos (`_`) y guiones (`-`). Puede comenzar con un número. Debe tener entre 1 y 50 caracteres. |
| `ORG_DISPLAY_NAME`         | Opcional. Nombre descriptivo de la organización que se puede mostrar en el flujo de inicio de sesión y en las plantillas de correo electrónico.                                                                                                                                                                                                                             |
| `ORG_LOGO`                 | Opcional. URL del logotipo de la organización.                                                                                                                                                                                                                                                                                                                              |
| `ORG_PRIMARY_COLOR`        | Opcional. Código de color HEX para los elementos principales.                                                                                                                                                                                                                                                                                                               |
| `ORG_BACKGROUND_COLOR`     | Opcional. Código de color HEX para el fondo.                                                                                                                                                                                                                                                                                                                                |
| `KEY`/`VALUE`              | Opcional. Pares clave/valor de tipo String que representan los metadatos de la organización. Máximo de 255 caracteres cada uno. Máximo de 25 pares de metadatos.                                                                                                                                                                                                            |
| `CONNECTION_ID`            | Opcional. ID de la conexión que quieres habilitar para la organización especificada. Las conexiones habilitadas se muestran en el prompt de inicio de sesión de la organización, para que los usuarios puedan acceder a tu aplicación o aplicaciones a través de ellas.                                                                                                     |
| `ASSIGN_MEMBERSHIP_OPTION` | Opcional. Indica si quieres que los usuarios que inicien sesión con la conexión habilitada obtengan automáticamente la membresía en la organización especificada. Cuando se establece en `true`, los usuarios obtendrán la membresía automáticamente. Cuando se establece en `false`, no la obtendrán automáticamente.                                                      |

<div id="response-status-codes">
  ### Códigos de estado de respuesta
</div>

Los posibles códigos de estado de respuesta son los siguientes:

| **Código de estado** | **Código de error**     | **Mensaje**                                                                                                  | **Causa**                                                                                       |
| -------------------- | ----------------------- | ------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------- |
| `201`                |                         | La organización se creó correctamente.                                                                       |                                                                                                 |
| `400`                | `invalid_body`          | Cuerpo de la solicitud no válido. El mensaje variará según la causa.                                         | El payload de la solicitud no es válido.                                                        |
| `400`                | `invalid_query_string`  | Query string de la solicitud no válido. El mensaje variará según la causa.                                   | El query string no es válido.                                                                   |
| `401`                |                         | Token no válido.                                                                                             |                                                                                                 |
| `401`                |                         | Se recibió una firma no válida para la validación de JSON Web Token.                                         |                                                                                                 |
| `401`                |                         | El cliente no es global.                                                                                     |                                                                                                 |
| `403`                | `insufficient_scope`    | Scope insuficiente; se esperaba alguno de los siguientes: `create:organizations`.                            | Se intentó leer o escribir un campo no permitido con los scopes del bearer token proporcionado. |
| `409`                | `organization_conflict` | Ya existe una organización con el mismo nombre.                                                              | Ya existe una organización con el mismo nombre.                                                 |
| `429`                |                         | Demasiadas solicitudes. Revise los encabezados X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                 |
