Cuando representas tu aplicación en Auth0, puedes especificar qué tipos de usuarios debe admitir. Algunas aplicaciones permiten que las personas inicien sesión con cuentas personales, mientras que otras están pensadas para miembros de [organizaciones](/docs/es/manage-users/organizations/organizations-overview). Algunas deben admitir ambos casos. Esto se conoce como comportamiento de la organización y se puede establecer para cada aplicación que conectes a Auth0.

Por ejemplo, tu aplicación podría tener:

* Una página de destino de marketing genérica con un botón de **Iniciar sesión** que lleve a tus usuarios al flujo de inicio de sesión de Auth0 sin una organización.
* Una URL independiente para cada uno de tus clientes B2B (p. ej., los usuarios de Acme van a `acme.yourcompany.com`) que redirija a los usuarios a Auth0 con una organización, para que vean el botón de Login de <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente su sesión en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip> de Acme.

Puedes definir el comportamiento de la organización para permitir cualquiera de estos escenarios. Además, puedes configurarlo de modo que, si tu aplicación requiere que se proporcione una organización, pero el usuario es enviado accidentalmente a Auth0 sin una, vea un prompt que le permita introducir el nombre de su organización.

Puedes definir el comportamiento de la organización usando el <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para definir el comportamiento de la organización desde el Auth0 Dashboard:

1. Ve a [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/#/applications) y selecciona la aplicación para la que quieres configurar Organizations.
2. Cambia a la vista Login Experience y configura los ajustes correspondientes:

| Campo                      | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   | Mapeo de API                                                                                                                                                                                                                                                                  |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Type of Users**          | Determina qué categoría de usuarios puede iniciar sesión en tu aplicación.<br /><br />Las opciones incluyen: <ul> <li> **Individuals**: Los usuarios pueden registrarse con una cuenta personal e iniciar sesión directamente en tu aplicación. Los usuarios individuales no pueden iniciar sesión mediante una organización. </li> <li> **Business Users**: Los usuarios deben ser miembros de una organización para poder iniciar sesión. Si seleccionas esta opción, debes proporcionar una organización cuando redirijas a los usuarios al endpoint `/authorize` o configurar tu Login Flow como Prompt for Organization. </li> <li> **Both**: Los usuarios pueden iniciar sesión como miembros de una organización o registrarse con una cuenta personal. </li> </ul>                                                                                    | **Type of Users** se mapea a `organization_usage`<br /><br />Opciones: <ul> <li>**Individuals** se mapea a `deny`</li> <li>**Business Users** se mapea a `require`</li> <li>**Both** se mapea a `allow`</li> </ul>                                                            |
| **Login Flow**             | Determina el prompt de inicio de sesión inicial que se muestra a los usuarios cuando acceden a tu aplicación. Solo puedes configurar este campo si Type of Users está establecido en Business Users o Both.<br /><br />Las opciones incluyen: <ul> <li> **Prompt for Credentials**: Primero se pide a los usuarios que proporcionen sus credenciales de inicio de sesión. Después de iniciar sesión, pueden seleccionar su organización. </li> <li> **Prompt for Organization**: Primero se pide a los usuarios que seleccionen su organización. Después, pueden proporcionar sus credenciales para iniciar sesión. </li> <li> **No Prompt**: Auth0 no determina qué prompt de inicio de sesión se muestra a los usuarios. En su lugar, tu aplicación es responsable de enviar a Auth0 los parámetros necesarios para mostrar el prompt adecuado. </li> </ul> | **Login Flow** se mapea a `organization_require_behavior`<br /><br />Opciones: <ul> <li>**Prompt for Credentials** se mapea a `post_login_prompt`</li> <li>**Prompt for Organization** se mapea a `pre_login_prompt`</li> <li>**No Prompt** se mapea a `no_prompt`</li> </ul> |
| **Organization Discovery** | Define el método que se utiliza para identificar la organización del usuario antes de la autenticación. Solo está disponible con el Login Flow Prompt for Organization.<br /><br />Las opciones incluyen: <ul> <li> **Prompt for Organization Email**: Primero se pide a los usuarios que proporcionen la dirección de correo electrónico asociada a su organización. </li> <li> **Prompt for Organization Name**: Primero se pide a los usuarios que proporcionen el nombre de su organización. </li> <li> **Prompt for Organization Email and Name**: Primero se pide a los usuarios que proporcionen el nombre de su organización o el correo electrónico asociado a su organización. </li> </ul>                                                                                                                                                          | <ul> <li> **Prompt for Emails**: se mapea a `email`. </li> <li> **Prompt for Organization**: se mapea a `organization_name`. </li> <li> **Prompt for Organization Email and Name**: se mapea a `email` y `organization_name`. </li> </ul>                                     |

3. Selecciona **Save changes**.

<div id="management-api">
  ## Management API
</div>

Haz una llamada `PATCH` al [endpoint actualizar un cliente](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id). Asegúrate de reemplazar los valores de marcador de posición `client_id`, `mgmt_api_access_token`, `organization_usage` y `organization_require_behavior` por tu <Tooltip tip="ID de cliente: Valor de identificación asignado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>, el <Tooltip tip="ID de cliente: Valor de identificación asignado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de la Management API, la opción de uso de la organización y la opción de comportamiento de la organización, respectivamente.

| Valor                            | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CLIENT_ID`                      | ID de la aplicación a la que desea agregar el comportamiento de la organización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| `MGMT_API_ACCESS_TOKEN`          | [Access Tokens para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:clients`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `ORGANIZATION_USAGE`             | Indica si su aplicación puede permitir que los usuarios inicien sesión en una organización. Las opciones incluyen: <ul> <li>`deny`: (Predeterminado) Los usuarios no pueden iniciar sesión mediante una organización.</li> <li>`allow`: Los usuarios pueden iniciar sesión con o sin una organización. Cuando se selecciona esta opción, debe proporcionar una organización al redirigir a los usuarios al endpoint `/authorize`.</li> <li>`require`: Los usuarios deben iniciar sesión mediante una organización. Cuando se selecciona esta opción, debe proporcionar una organización al redirigir a los usuarios al endpoint `/authorize` o establecer `organization_require_behavior` en `pre_login_prompt` para permitir que los usuarios elijan una organización antes de iniciar sesión.</li> </ul> |
| `ORGANIZATION_REQUIRE_BEHAVIOR`  | Determina el Login Flow que se presenta a los usuarios que acceden a su aplicación. Solo se aplica cuando `organization_usage` se establece en `require` o `allow`.<br /><br />Las opciones incluyen: <ul> <li>`no_prompt`: (Predeterminado) No se muestra ningún prompt. Las solicitudes sin un parámetro de organización válido se rechazan.</li> <li>`pre_login_prompt`: Se solicita a los usuarios que seleccionen una organización antes de poder iniciar sesión. Solo puede usar esta opción si `organization_usage` se establece en `require`.</li> <li>`post_login_prompt`: Se solicita a los usuarios que inicien sesión con sus credenciales. Después de iniciar sesión, se les solicita que seleccionen su organización.</li> </ul>                                              |
| `ORGANIZATION_DISCOVERY_METHODS` | Determina el método para identificar la organización de un usuario antes de la autenticación. Solo se aplica cuando `organization_require_behavior` es `pre_login_prompt` y `organization_usage` se establece en `require` o `allow`. <ul> <li> `email`: (Predeterminado) Se solicita a los usuarios que introduzcan la dirección de correo electrónico asociada a su organización. </li> <li> `organization_name`: Se solicita a los usuarios que introduzcan el nombre asociado a su organización. </li> <li> `Organization name` y `email`: Se solicita a los usuarios la dirección de correo electrónico o el nombre asociado a su organización. </li> </ul>                                                                                                                                           |
