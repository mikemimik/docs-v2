Al trabajar con [organizaciones](/docs/es/manage-users/organizations/organizations-overview) mediante programación, es posible que necesite obtener una lista de roles asignados a un miembro de una organización.

Puede ver los roles asociados a un miembro de una organización mediante el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización cuya membresía quieres ver.
2. Selecciona la vista **Members** y luego el usuario cuyos roles quieres ver.

<div id="management-api">
  ## Management API
</div>

Haz una llamada `GET` al endpoint `Get Organization Member Roles`. Asegúrate de reemplazar los valores de marcador de posición `ORG_ID`, `USER_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de tu organización, el ID de usuario del miembro y el <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de Management API, respectivamente.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Si tu dominio de Auth0 está formado por el nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, el nombre de tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`.)

  Si usas custom domains, este debe ser tu dominio personalizado.
</Callout>

| Valor                   | Descripción                                                                                                                                             |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de la organización de la que desea obtener los roles de un miembro.                                                                                  |
| `USER_ID`               | ID de usuario del miembro del que desea obtener los roles.                                                                                              |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `read:organization_member_roles`. |

<div id="response-status-codes">
  #### Códigos de estado de respuesta
</div>

Los posibles códigos de estado de respuesta son los siguientes:

| Código de estado | Código de error        | Mensaje                                                                                                     | Causa                                                                                           |
| ---------------- | ---------------------- | ----------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `200`            |                        | Roles recuperados correctamente.                                                                            |                                                                                                 |
| `400`            | `invalid_query_string` | El query string de la solicitud no es válido. El mensaje variará según la causa.                            | El query string no es válido.                                                                   |
| `401`            |                        | Token no válido.                                                                                            |                                                                                                 |
| `401`            |                        | Se recibió una firma no válida para la validación del JSON Web Token.                                       |                                                                                                 |
| `401`            |                        | El cliente no es global.                                                                                    |                                                                                                 |
| `403`            | `insufficient_scope`   | Scope insuficiente; se esperaba cualquiera de los siguientes: `read:organization_member_roles`.             | Se intentó leer o escribir un campo no permitido con los scopes del bearer token proporcionado. |
| `429`            |                        | Demasiadas solicitudes. Verifique los headers X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                 |
