Al trabajar con [organizaciones](/docs/es/manage-users/organizations/organizations-overview) de forma programática, es posible que necesite obtener una lista de miembros de una organización.

Puede ver los miembros asociados a una organización mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización cuya membresía quieras consultar.
2. Selecciona la vista **Members**.

<div id="management-api">
  ## Management API
</div>

<Warning>
  Se pueden mostrar hasta 1000 miembros de la organización en el Auth0 Dashboard o mediante la Management API, aunque puede haber más.
</Warning>

Haz una llamada `GET` al endpoint `Get Organization Members`. Asegúrate de reemplazar los valores de marcador de posición `ORG_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de tu organización y el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de la Management API, respectivamente.

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 se compone del nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, el dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu dominio sería `https://travel0.auth0.com`.)

  Si usas dominios personalizados, este debe ser tu dominio personalizado.
</Callout>

| Valor                   | Descripción                                                                                                                                        |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de la organización para la que quieres obtener los miembros.                                                                                    |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `read:organization_members`. |

<div id="response-status-codes">
  ### Códigos de estado de la respuesta
</div>

Los posibles códigos de estado de la respuesta son los siguientes:

| Código de estado | Código de error      | Mensaje                                                                                                     | Causa                                                                                           |
| ---------------- | -------------------- | ----------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `200`            |                      | Miembros recuperados correctamente.                                                                         |                                                                                                 |
| `400`            | `invalid_paging`     | La página solicitada supera el máximo permitido de 1000 registros.                                          | La API está limitada a devolver un máximo de 1000 registros.                                    |
| `400`            | `invalid_body`       | Cuerpo de la solicitud no válido. El mensaje variará según la causa.                                        | El payload de la solicitud no es válido.                                                        |
| `401`            |                      | Token no válido.                                                                                            |                                                                                                 |
| `401`            |                      | Se recibió una firma no válida durante la validación del JSON Web Token.                                    |                                                                                                 |
| `401`            |                      | El cliente no es global.                                                                                    |                                                                                                 |
| `403`            | `insufficient_scope` | Scope insuficiente; se esperaba alguno de los siguientes: `read:organization_members`.                      | Se intentó leer o escribir un campo no permitido con los scopes del bearer token proporcionado. |
| `429`            |                      | Demasiadas solicitudes. Compruebe los headers X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                 |
