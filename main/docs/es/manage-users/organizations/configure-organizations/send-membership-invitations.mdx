title: Enviar invitaciones de membresía para Organizations
---

Puedes enviar a los usuarios invitaciones de membresía a una [organización](/docs/es/manage-users/organizations/organizations-overview) mediante el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para invitar a miembros mediante Auth0 Dashboard:

1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización para la que quieres configurar la membresía.
2. Selecciona la vista **Invitations** y, a continuación, **Invite Members**.
3. Selecciona la aplicación desde la que quieres invitar al miembro, introduce la dirección de correo electrónico del usuario que quieres invitar a la organización y selecciona **Send Invite(s)**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Opcionalmente, puedes especificar la conexión con la que quieres que los usuarios acepten la invitación, así como los roles que quieres asignar al miembro de la organización cuando la acepte.
   </Callout>

Si habilitas [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains), puedes seleccionar **Domain** en la sección **Send Invite(s)** para elegir el dominio de notificación de tu tenant.

<div id="management-api">
  ## Management API
</div>

<Warning>
  Si expones la funcionalidad de invitación desde un panel de administración que has personalizado para tu aplicación, usa una [aplicación confidencial](/docs/es/get-started/applications/confidential-and-public-applications) para validar que el usuario autenticado pertenece a una organización antes de invitar a nuevos miembros, o bien restringe la posibilidad de invitar a nuevos miembros a los usuarios a los que se les haya asignado un rol específico.
</Warning>

Para invitar a miembros mediante la Management API:
Realiza una llamada `POST` al endpoint Create Organization Invitations. Asegúrate de reemplazar los valores de marcador de posición `ORG_ID`, `MGMT_API_ACCESS_TOKEN`, `NAME_OF_USER`, `EMAIL_ADDRESS`, `CLIENT_ID`, `CONNECTION_ID`, `EXP_TIME`, `ROLE_ID` y `SEND_INVITATION_EMAIL_OPTION` por el ID de tu organización, el <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> de la Management API, el nombre del usuario invitado, la dirección de correo electrónico del usuario invitado, el <Tooltip tip="ID de cliente: Valor de identificación asignado a tu recurso registrado de Auth0." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>, el ID de conexión, el tiempo de expiración y los ID de rol, respectivamente.

```json lines
{
  "method": "POST",
  "url": "https://{yourDomain}/api/v2/organizations/ORG_ID/invitations",
"headers": [
  { "name": "Content-Type", "value": "application/json" },
  { "name": "Authorization", "value": "Bearer MGMT_API_ACCESS_TOKEN" },
  { "name": "Cache-Control", "value": "no-cache" }
  ],
  "postData": {
  "mimeType": "application/json",
  "text" : "{ \"inviter\": { \"name\": \"NAME_OF_USER\"}, \"invitee\": { \"email\": \"EMAIL_ADDRESS\" }, \"client_id\": \"CLIENT_ID\", \"connection_id\": \"CONNECTION_ID\", \"ttl_sec\": \"EXP_TIME\", \"roles\": [ \"ROLE_ID\", \"ROLE_ID\", \"ROLE_ID\" ], \"send_invitation_email\": \"SEND_EMAIL_INVITATION_OPTION\" }"
  }
}
```

Si habilitas [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains), debes incluir el encabezado HTTP `auth0-custom-domain`. Para obtener más información, consulta [múltiples dominios personalizados](/docs/es/customize/custom-domains/multiple-custom-domains#customize-email-handling-using-the-management-api).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 se compone del nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`.)

  Si usas dominios personalizados, este debe ser el nombre de tu dominio personalizado.
</Callout>

| Value                          | Description                                                                                                                                                                                                                                   |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                       | ID de la organización a la que quieres asignar la membresía.                                                                                                                                                                                  |
| `MGMT_API_ACCESS_TOKEN`        | [Access Token for the Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organization_invitations`.                                                                                         |
| `NAME_OF_USER`                 | Nombre del usuario al que quieres enviar la invitación. Máximo de 300 caracteres.                                                                                                                                                             |
| `EMAIL_ADDRESS`                | Dirección de correo electrónico a la que se debe enviar la invitación.                                                                                                                                                                        |
| `CLIENT_ID`                    | ID de la aplicación en la que el usuario invitado debe autenticarse.                                                                                                                                                                          |
| `CONNECTION_ID`                | ID de la conexión mediante la cual debe autenticarse el miembro invitado.                                                                                                                                                                     |
| `EXP_TIME`                     | Número de segundos antes de que caduque la invitación. Si no se especifica o se establece en 0, el valor predeterminado es 604800 segundos (7 días). Máximo de 2592000 segundos (30 días).                                                    |
| `ROLE_ID`                      | ID del rol o de los roles que quieres asignar al usuario invitado para la organización especificada. Máximo de 50 roles por miembro.                                                                                                          |
| `SEND_INVITATION_EMAIL_OPTION` | Indica si Auth0 debe enviar el correo electrónico. Los valores son `true` o `false`. Cuando se establece en `false`, Auth0 generará una URL de invitación que podrás enviar a los usuarios mediante tu propio servicio de correo electrónico. |
