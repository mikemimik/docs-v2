Puedes quitar roles de los miembros de la [organización](/docs/es/manage-users/organizations/organizations-overview) mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para eliminar un rol de un miembro de una organización desde el Auth0 Dashboard:

1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización para la que quieres configurar la membresía.
2. Selecciona la vista **Members** y haz clic en el nombre del miembro al que quieres quitarle un rol.
3. Busca el rol que quieres eliminar, haz clic en el icono de la papelera y confirma.

<div id="management-api">
  ## Management API
</div>

Para eliminar roles de un miembro de una organización mediante la Management API:
Haga una llamada `DELETE` al endpoint `Delete Organization Member Roles`. Asegúrese de reemplazar los valores de marcador de posición `ORG_ID`, `MGMT_API_ACCESS_TOKEN`, `USER_ID` y `ROLE_ID` por su ID de organización, el Management API <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip>, el ID de usuario y el ID del rol, respectivamente.

| Valor                   | Descripción                                                                                                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ORG_ID`                | ID de la organización de cuyo miembro quieres eliminar roles.                                                                                          |
| `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `delete:organization_member_roles`. |
| `USER_ID`               | ID del usuario al que quieres quitarle los roles especificados.                                                                                        |
| `ROLE_ID`               | ID del rol que quieres quitarle al usuario especificado para la organización especificada.                                                             |

<div id="response-status-codes">
  ### Códigos de estado de la respuesta
</div>

Los posibles códigos de estado de la respuesta son los siguientes:

| Código de estado | Código de error        | Mensaje                                                                                                  | Causa                                                                                                    |
| ---------------- | ---------------------- | -------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| `204`            |                        | Roles eliminados correctamente del miembro de la organización.                                           |                                                                                                          |
| `400`            | `invalid_body`         | Cuerpo de la solicitud no válido. El mensaje variará según la causa.                                     | El payload de la solicitud no es válido.                                                                 |
| `400`            | `invalid_query_string` | Cadena de consulta de la solicitud no válida. El mensaje variará según la causa.                         | La cadena de consulta no es válida.                                                                      |
| `401`            |                        | Token no válido.                                                                                         |                                                                                                          |
| `401`            |                        | Se recibió una firma no válida para la validación de JSON Web Token.                                     |                                                                                                          |
| `401`            |                        | El cliente no es global.                                                                                 |                                                                                                          |
| `403`            | `insufficient_scope`   | Scope insuficiente; se esperaba cualquiera de los siguientes: `delete:organization_member_roles`.        | Se intentó leer o escribir un field que no está permitido con los scopes del bearer token proporcionado. |
| `429`            |                        | Demasiadas solicitudes. Revisa los headers X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                          |
