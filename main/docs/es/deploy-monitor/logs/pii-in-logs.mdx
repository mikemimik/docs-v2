description: Describe la posible información de identificación personal (PII) que Auth0 escribe en los registros del tenant.
title: Información de identificación personal en los registros de Auth0
---

Muchos de los registros del tenant de Auth0 pueden contener información de identificación personal (PII). Los eventos de autenticación generan entradas de registro que pueden incluir la PII de los usuarios. Y si usas la Auth0 <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> o el Dashboard para añadir usuarios, Auth0 registra los detalles de la cuenta del usuario.

Estos campos en las entradas de registro pueden contener PII:

* Nombre
* Número de teléfono (usado para la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de username y password, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>)
* Dirección de correo electrónico
* Dirección IP (que puede revelar la ubicación)
* Cualquier información de usuario personalizada que definas

Auth0 no registra nunca <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaque o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> de Auth0 ni de ningún <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaque o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>. Cuando se realizan intercambios de código de autorización, los registros muestran solo una parte del código (por ejemplo: `code: 31XXXXX`).
