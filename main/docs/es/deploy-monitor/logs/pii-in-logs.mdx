---
description: Describe la posible información de identificación personal (PII) que Auth0 escribe en los registros del tenant.
title: Información de identificación personal en los registros de Auth0
---

Muchos registros del tenant de Auth0 pueden contener información de identificación personal (PII). Los eventos de autenticación generan entradas de registro que pueden incluir PII de los usuarios. Y si usas la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> o el Dashboard para agregar usuarios, Auth0 registra los datos de la cuenta de usuario.

Estos campos de las entradas de registro pueden contener PII:

* Nombre
* Número de teléfono (utilizado para la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>)
* Dirección de correo electrónico
* Dirección IP (que puede revelar la ubicación)
* Cualquier información de usuario personalizada que definas

Auth0 no registra en ningún momento <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> de Auth0 ni de ningún <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>. Cuando se producen intercambios de código de autorización, los registros solo muestran una parte del código (por ejemplo: `code: 31XXXXX`).
