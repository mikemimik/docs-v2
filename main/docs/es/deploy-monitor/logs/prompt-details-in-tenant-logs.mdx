---
description: Describe los atributos y valores de los prompts de usuario en los registros del tenant de Auth0.
title: Detalles del prompt en los registros del tenant
---

Los registros del tenant de Auth0 proporcionan detalles de transacciones relacionadas con acciones de autenticación de usuario. Los registros del tenant capturan información de los procesos de inicio de sesión, inicio de sesión y verificación para su revisión. Puede revisar cómo se autenticó el usuario, qué credenciales proporcionó, la duración de la transacción y el estado de la conexión.

A continuación, se muestran los valores predeterminados capturados en los registros del tenant:

| Atributo      | Descripción                                                                     |
| ------------- | ------------------------------------------------------------------------------- |
| `completedAt` | Hora de finalización de la etapa, incluido el tiempo de interacción del usuario |
| `elapsedTime` | Diferencia entre `completedAt` y `initiatedAt`                                  |
| `flow`        | Varía según la experiencia de inicio de sesión (es decir, Universal o Classic)             |
| `initiatedAt` | Hora de inicio de la etapa                                                      |
| `name`        | Nombre de la etapa                                                              |

<div id="login">
  ## inicio de sesión
</div>

La etapa de inicio de sesión proporciona los siguientes valores:

| Atributo    | Descripción                                                     |
| ----------- | --------------------------------------------------------------- |
| `flow`      | Puede tomar dos valores, según se use Universal o Classic Login |
| `user_id`   | ID de Auth0 del usuario autenticado en esta etapa               |
| `user_name` | Nombre del usuario autenticado en esta etapa                    |

Campos adicionales con la siguiente información:

| Atributo        | Descripción                                                    |
| --------------- | -------------------------------------------------------------- |
| `connection`    | Conexión utilizada para autenticar al usuario                  |
| `connection_id` | ID de la conexión utilizada para autenticar al usuario         |
| `strategy`      | Estrategia de la conexión utilizada para autenticar al usuario |
| `identity`      | Identidad autenticada del usuario                              |

Valores para eventos adicionales en el flujo de inicio de sesión:

| Nombre                         | Experiencia de Universal Login | Descripción                                                                           |
| ------------------------------ | ------------------------------ | ------------------------------------------------------------------------------------- |
| `federated-authenticate`       | Universal                      | Conexión federada utilizada para la autenticación, ya sea social, legal o empresarial |
| `lock-password-authenticate`   | Classic                        | Usuario autenticado con un nombre de usuario y contraseña                             |
| `oauth1-authenticate`          | Classic                        | Usuario autenticado con una federación OAuth                                          |
| `oauth2-authenticate`          | Classic                        | Usuario autenticado con una federación OAuth2                                         |
| `oidc-authenticate`            | Classic                        | Usuario autenticado con una federación OIDC                                           |
| `prompt-authenticate`          | Universal                      | Usuario autenticado con un nombre de usuario y contraseña                             |
| `prompt-authenticate-password` | Universal                      | Usuario autenticado con nombre de usuario y contraseña en el flujo `idfirst`          |
| `prompt-signup`                | Universal                      | Usuario registrado con un nombre de usuario y contraseña                              |
| `prompt-signup-password`       | Universal                      | Usuario registrado con nombre de usuario y contraseña mediante el flujo `idfirst`     |
| `saml-authenticate`            | Classic                        | Usuario autenticado con una federación SAML                                           |
| `wsfed-authenticate`           | Classic                        | Usuario autenticado con una federación WS-FED                                         |

<div id="consent">
  ## consentimiento
</div>

La etapa de consentimiento proporciona los siguientes valores:

| Atributo    | Descripción                                                                                  |
| ----------- | -------------------------------------------------------------------------------------------- |
| `grantInfo` | El `{"{myApiIdentifier}"}` o identificador de la API para la que el usuario concedió permiso |
| `audience`  | Audiencia del token                                                                          |
| `id`        | Concesión del usuario guardada en la base de datos                                           |
| `scope`     | Scope aprobado concedido por el usuario                                                      |

<div id="login-email-verification">
  ## login-email-verification
</div>

El prompt de verificación del correo electrónico proporciona los siguientes valores:

| Atributo                    | Descripción                                                              |
| --------------------------- | ------------------------------------------------------------------------ |
| ` login-email-verification` | Se incluye si se mostró el prompt de verificación del correo electrónico |

<div id="redirect">
  ## redirect
</div>

La etapa redirect proporciona los siguientes valores:

| Atributo | Descripción                          |
| -------- | ------------------------------------ |
| `URL`    | URL a la que se redirigió al usuario |

<div id="mfa">
  ## mfa
</div>

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+authentication">Autenticación multifactor</Tooltip> proporciona los siguientes valores:

| Atributo        | Descripción                                                                                                                |
| --------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `flow`          | Puede tomar dos valores, según se use con la experiencia Classic (`mfa`) o con la experiencia Universal (`universal-mfa`). |
| `performed_acr` | `acr` que se incluirá en el `id_token`.                                                                                    |
| `performed_amr` | `amr` que se incluirá en el `id_token`.                                                                                    |
| `provider`      | Proveedor utilizado para realizar MFA.                                                                                     |
