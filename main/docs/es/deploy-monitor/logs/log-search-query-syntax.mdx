title: Sintaxis de consultas de búsqueda de registros
---

Al buscar logs, puedes crear consultas mediante un subconjunto de la [sintaxis de consulta de Lucene](https://lucene.apache.org/core/2_9_4/queryparsersyntax.html) para refinar la búsqueda.

La cadena de consulta se analiza en una serie de términos y operadores:

* Un término puede ser una sola palabra, como `jane` o `smith`.
* Un término puede ser una frase entre comillas dobles (`"customer log"`), que coincidirá con todas las palabras de la frase en el mismo orden.
* Un término sin nombre de campo solo coincidirá con estos campos seleccionados.
* Se pueden agrupar varios términos entre paréntesis para formar subconsultas.
* Todos los campos de búsqueda distinguen entre mayúsculas y minúsculas.
* Los operadores (`AND`, `OR`, `NOT`) funcionan en todos los campos de búsqueda.

<div id="searchable-fields">
  ## Campos de búsqueda
</div>

Los siguientes campos admiten búsquedas y distinguen entre mayúsculas y minúsculas:

| Field             | Description                                                                          |
| ----------------- | ------------------------------------------------------------------------------------ |
| `client_id`       | ID del cliente relacionado con el evento.                                            |
| `client_name`     | Nombre del cliente relacionado con el evento.                                        |
| `connection`      | Nombre de la conexión relacionado con el evento.                                     |
| `connection_id`   | ID de la conexión relacionado con el evento.                                         |
| `description`     | Descripción del evento.                                                              |
| `date`            | Fecha en que ocurrió el evento, en formato YYYY-MM-DD.                               |
| `hostname`        | Nombre de host utilizado para el flujo de autenticación.                             |
| `ip`              | Dirección IP desde la que se originó la solicitud que generó la entrada de registro. |
| `log_id`          | ID del registro del evento.                                                          |
| `organization_id` | ID de la organización relacionada con el evento.                                     |
| `user_id`         | ID del usuario relacionado con el evento.                                            |
| `user_name`       | Nombre del usuario relacionado con el evento.                                        |
| `user_agent`      | Agente de usuario relacionado con el evento.                                         |
| `strategy`        | Estrategia de conexión relacionada con el evento.                                    |
| `strategy_type`   | Tipo de estrategia de conexión relacionada con el evento.                            |
| `type`            | Tipo de evento.                                                                      |

<div id="fields-searchable-against-bare-terms">
  ## Campos en los que se pueden buscar términos sin campo
</div>

Si se introduce un término de búsqueda sin nombre de campo, solo se buscará en los siguientes campos:

* `client_name`
* `connection`
* `description`
* `ip`
* `log_id`
* `type`
* `user_name`

<div id="exact-matching">
  ## Coincidencia exacta
</div>

Para encontrar coincidencias exactas, usa comillas dobles: `description:"Username invalid"`.

Por ejemplo, para encontrar logs con la descripción `Username invalid`, usa `q=description:"Username invalid"`.

<div id="wildcards">
  ## Comodines
</div>

Se pueden realizar búsquedas con comodines en términos usando el carácter de asterisco (`*`) para reemplazar cero o más caracteres: `user_name:john*`. Pueden usarse para coincidencias por prefijo, por ejemplo `user_name:j*`. Para otros usos de los comodines (por ejemplo, coincidencias por sufijo), los literales deben tener 3 caracteres o más. Por ejemplo, `name:*usa` está permitido, pero `name:*sa` no.

El signo de interrogación (`?`) no es compatible.

Por ejemplo, para encontrar todos los logs de los usuarios cuyos nombres de usuario empiezan por `john`, usa `q=user_name:john*`.

<div id="ranges">
  ## Rangos
</div>

Puedes usar rangos en tus consultas de búsqueda de logs. Para los rangos inclusivos, usa corchetes: `[min TO max]`, y para los rangos exclusivos, usa llaves: `{min TO max}`.

Las llaves y los corchetes se pueden combinar en la misma expresión de rango. También puedes usar comodines dentro de los rangos.

Por ejemplo, para encontrar todos los logs desde el 18 de diciembre de 2018 hasta la actualidad, usa `q=date:[2018-12-18 TO *]`.

Si quieres buscar logs desde el inicio de tu período de retención hasta el 19 de diciembre de 2018, sin incluir esa fecha, usa `q=date:[* TO 2018-12-19}`.

<div id="example-queries">
  ## Consultas de ejemplo
</div>

A continuación, se muestran algunos ejemplos para ilustrar los tipos de consultas que puedes realizar con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

| Caso de uso                                                                                                                            | Consulta                                            |
| -------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| Buscar todos los logs con conexiones que contengan &quot;Pass&quot;                                                                    | `connection:*pass*`                                 |
| Buscar todos los logs de usuarios con un nombre de usuario que contenga &quot;fred&quot;                                               | `user_name:*fred*`                                  |
| Buscar todos los logs con ID de usuario que coincidan exactamente con &quot;123&quot;                                                  | `user_id:"123"`                                     |
| Buscar todos los logs con un tipo que empiece por &quot;s&quot;                                                                        | `type:s*`                                           |
| Buscar nombres de usuario que empiecen por &quot;jane&quot; y terminen por &quot;smith&quot;                                           | `user_name:jane*smith`                              |
| Buscar todos los logs de diciembre de 2018                                                                                             | `date:[2018-12 TO 2019-01-01}`                      |
| Buscar todos los logs a partir del 10 de diciembre de 2018                                                                             | `date:[2018-12-10 TO *]`                            |
| Buscar todos los logs desde la 1:00 a. m. del 1 de enero de 2019 hasta el 1 de enero de 2019 a las 12:23:45, sin incluir esa hora    | `date:[2019-01-01T01:00:00 TO 2019-01-01T12:23:45}` |

<div id="limitations">
  ## Limitaciones
</div>

* Si recibes el error `414 Request-URI Too Large`, significa que la cadena de consulta supera la longitud admitida. En ese caso, acota tu búsqueda.
* Los campos de registro no se tokenizan, por lo que `description:rule` no coincidirá con una descripción cuyo valor sea `Create a rule` ni `Update a rule`. En su lugar, usa `description:*rule`. Consulta los comodines y la coincidencia exacta.
* La extensión de campo `.raw` no es compatible. Los campos coinciden con el valor completo proporcionado y no se tokenizan.

<div id="pagination">
  ## Paginación
</div>

Al llamar a los endpoints `GET /api/v2/logs` o `GET /api/v2/users/{user_id}/logs` con el parámetro `include_totals`, el resultado es un objeto JSON que contiene un resumen de los resultados y los logs solicitados. El objeto JSON tiene un aspecto similar al siguiente:

```json lines
{
  "length": 5,
  "limit": 5,
  "logs": [...],
  "start": 0,
  "total": 5
}
```

Al buscar `logs`, el campo **totals** indica cuántos `logs` se devuelven en esa página (de forma similar a lo que devuelve el campo **length**).
