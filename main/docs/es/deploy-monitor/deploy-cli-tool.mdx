<Card title="Antes de empezar">
  Configura un [tenant de Auth0](https://manage.auth0.com/).
</Card>

Auth0 Deploy CLI es una herramienta que te ayuda a administrar la configuración de tu tenant de Auth0. Se integra en tus flujos de trabajo de desarrollo como una CLI independiente o como un módulo de Node.js.

Tipos de recursos compatibles: `actions`, `branding`, `client grants`, `clients (applications)`, `connections`, `custom domains`, `email templates`, `emails`, `grants`, `guardian`, `hook secrets`, `log streams`, `migrations`, `organizations`, `pages`, `prompts`, `resource servers (APIs)`, `roles`, `tenant settings`, `themes`.

<div id="highlights">
  ## Aspectos destacados
</div>

* **Orientado a múltiples entornos:** Diseñado para ayudarte a probar las integraciones de Auth0 de tus aplicaciones desde la rama de funcionalidad hasta producción.
* **Reemplazo de palabras clave:** Configuraciones de recursos compartidas entre todos los entornos con reemplazo dinámico de palabras clave.
* **Versátil:** Intégralo en tus flujos de trabajo de CI/CD, ya sea como CLI o como módulo de Node.js.

<div id="get-started">
  ## Primeros pasos
</div>

Esta guía le ayudará a tener una implementación funcional de la herramienta Deploy CLI usada como una CLI independiente. Hay tres pasos principales antes de poder ejecutar Deploy CLI:

1. [Crear una aplicación de Auth0 dedicada](#create-a-dedicated-auth0-application)
2. [Configurar Deploy CLI](#configure-the-deploy-cli)
3. [Ejecutar Deploy CLI](#call-the-deploy-cli)

<Warning>
  Esta herramienta puede tener efectos destructivos en su tenant de Auth0. Se recomienda familiarizarse con la [configuración AUTH0&#95;ALLOW&#95;DELETE](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli#auth0_allow_delete) y hacer pruebas en tenants de desarrollo antes de usarla en producción.
</Warning>

<div id="install-the-deploy-cli">
  ### Instala Deploy CLI
</div>

Para ejecutarlo como una herramienta de línea de comandos independiente:

`npm install -g auth0-deploy-cli`

<div id="create-a-dedicated-auth0-application">
  ### Crear una aplicación de Auth0 dedicada
</div>

Para que Deploy CLI pueda llamar a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, se debe crear una aplicación de Auth0 dedicada para hacer llamadas en nombre de la herramienta:

1. Ve a **Auth0 Dashboard &gt; Applications &gt; Applications**.
2. Selecciona **+ Create Application**.

   1. Asígnale un nombre descriptivo (como “Deploy CLI”).
   2. Selecciona **Machine to Machine Applications** como **Application Type**.
   3. Selecciona **Create**.
3. Cambia a la vista **Authorize Machine to Machine Application**:

   1. Selecciona **Auth0 Management API**.
   2. Selecciona los permisos adecuados para los recursos que quieres administrar. Consulta la sección [Client Scopes](#client-scopes) para obtener más información.
   3. Selecciona **Authorize**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El cliente propio de Deploy CLI no se puede configurar para [evitar cambios potencialmente destructivos](/docs/es/deploy-monitor/deploy-cli-tool/resource-specific-documentation#client-grants).
</Callout>

<div id="client-scopes">
  #### Scopes del cliente
</div>

A la aplicación designada se le deben conceder scopes para permitir que Deploy CLI ejecute operaciones de Management.

Se respeta el principio de mínimo privilegio, por lo que funcionará dentro del conjunto de permisos concedidos. Deberá seleccionar `read:clients`, pero se recomienda seleccionar los permisos `read:*`, `create:*` y `update:*` para todos los tipos de recursos dentro del ámbito de Management. Para habilitar eliminaciones, necesita los scopes `delete:*`.

<div id="configure-the-deploy-cli">
  ### Configurar Deploy CLI
</div>

Puedes configurar Deploy CLI de dos maneras:

1. Usa el archivo de configuración (`config.json`).
2. Define variables de entorno.

La elección de una opción o de ambas dependerá de tu caso de uso y de tus preferencias. Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

Para este ejemplo, define las siguientes variables de entorno:

* `AUTH0_DOMAIN`
* `AUTH0_CLIENT_ID`
* `AUTH0_CLIENT_SECRET`

Estos valores se encuentran en la pestaña **Settings** de la aplicación de Auth0 dedicada que creaste anteriormente.

<div id="call-the-deploy-cli">
  ### Ejecutar Deploy CLI
</div>

El comando `export` de Deploy CLI se ejecuta con el siguiente comando:

`a0deploy export --format=yaml --output_folder=local`

Una vez completado el proceso, revisa los archivos de configuración de recursos generados en el directorio local. Luego, ejecuta el comando `import` para subir la configuración desde tu máquina local a tu tenant de Auth0:

`a0deploy import --config_file=config.json --input_file local/tenant.yaml`

Para ver una lista completa de indicadores y opciones, consulta [Use as CLI](/docs/es/deploy-monitor/deploy-cli-tool/use-as-a-cli).
