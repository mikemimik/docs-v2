<div id="use-automated-deployment-for-actions">
  ## Usa el despliegue automatizado para Actions
</div>

Desarrollar una Action dentro del editor de <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> es una excelente manera de implementar y probar mientras aún estás en la etapa de desarrollo. Sin embargo, cuando llega el momento de desplegarla en entornos de pruebas automatizadas o de producción, se necesita un mecanismo más automatizado; copiar y pegar código entre tenants de Auth0 no es una solución adecuada.

<div id="set-up-version-control-for-actions">
  ### Configura el control de versiones para Actions
</div>

Los cambios realizados en una Action en el Auth0 Dashboard se guardan como borradores hasta que se despliegan. Después de desplegarla, se crea una versión, pero debes vincular la Action a un trigger para activarla. Recomendamos usar control de versiones para facilitar la gestión de cambios y la revisión de código. [Actions admite el control de versiones](docs/customize/actions/manage-versions) de forma nativa en el Dashboard.

Auth0 ofrece métodos de despliegue automatizado para Actions entre entornos de tenant de Auth0. Las siguientes integraciones de Auth0 te permiten actualizar Actions desde el sistema de control de versiones, ya sea de forma manual o automática (por ejemplo, cuando se detecta un cambio en el sistema de control de versiones): [GitHub Actions](https://marketplace.auth0.com/integrations/github-actions), [GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline), [Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline) y [Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline).

<div id="use-deploy-cli-tool-for-action-import-and-export">
  ### Usa la herramienta Deploy CLI para importar y exportar Actions
</div>

La herramienta Deploy CLI de Auth0 automatiza el despliegue de Actions entre tenants de Auth0. Funciona con archivos YAML locales o basados en directorios junto con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 y permite exportar Actions desde un tenant de Auth0, así como importarlas a un tenant de Auth0. Además, la herramienta proporciona control programático sobre el orden y la configuración del entorno de una Action como parte de la automatización del despliegue. Para obtener más información, consulta [Deploy CLI Tool](/docs/es/deploy-monitor/deploy-cli-tool).

Como práctica recomendada, usa Auth0 Deploy CLI en casi todos los casos que impliquen despliegues en entornos de prueba o de producción. Aunque las integraciones de Marketplace proporcionan detección automatizada de cambios, Deploy CLI permite un control preciso sobre qué se despliega, cuándo y cómo.

<div id="set-up-test-tenants">
  ## Configura tenants de prueba
</div>

Te recomendamos configurar un tenant de prueba independiente en Auth0 para probar de forma segura cualquier Action antes de implementarla en producción. Para obtener más información, consulta [Configurar múltiples entornos](/docs/es/get-started/auth0-overview/create-tenants/set-up-multiple-environments).
