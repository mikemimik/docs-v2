Ve a la página [Auth0 Public Cloud Status](https://status.auth0.com) para consultar la disponibilidad del servicio de la versión de Auth0 en Public Cloud.

Selecciona una **Región** en el menú desplegable para ver el estado de los servicios principales (como autenticación de usuario y autenticación máquina a máquina) y los servicios de soporte (como <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor adicional además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>, <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, logs del tenant y CDN).

Para obtener más información sobre las regiones disponibles, consulta la tabla [Región, localidad y sublocalidad](/docs/es/get-started/auth0-overview/create-tenants#region-locality-and-sub-locality) en [Crear tenants](/docs/es/get-started/auth0-overview/create-tenants).

<div id="subscribe-to-public-cloud-status-updates">
  ## Suscribirse a las actualizaciones de estado de Public Cloud
</div>

Seleccione una **Región** y, a continuación, seleccione **Subscribe** para recibir actualizaciones. Puede elegir entre dos opciones para consultar el estado:

1. Siga a [@auth0status](https://x.com/auth0status) en X para obtener las últimas actualizaciones de estado.
2. Suscríbase al feed Atom o RSS de Auth0 para recibir actualizaciones de estado que afecten a su tenant. Sustituya `{YOUR_TENANT_DOMAIN}` por el **Domain** de su tenant.

Para encontrar el **Domain** de su tenant, vaya a [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/dashboard/#/applications), seleccione una aplicación y, a continuación, vaya a **Settings**. El **Domain** de su tenant aparece en **Basic Information**.

<div id="check-auth0-status-for-private-cloud">
  ## Consultar el estado de Auth0 para Private Cloud
</div>

Ve a [Auth0 Public Cloud Status](https://status.auth0.com) y, a continuación, selecciona **Private Cloud Login**.

Una vez que te hayas autenticado, podrás ver el estado de cualquier entorno que no sea de desarrollo (por ejemplo, `travel0-prod-1`) si está disponible en Auth0 Converged Platform.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para acceder al estado de tu entorno de Private Cloud, debes ser miembro del tenant de soporte de Public Cloud asociado. Para obtener más información, consulta [Abrir y administrar tickets de soporte](/docs/es/troubleshoot/customer-support/open-and-manage-support-tickets) y [Administrar el acceso al Dashboard](/docs/es/get-started/manage-dashboard-access).
</Callout>

Para volver a la página de Auth0 Public Cloud Status, selecciona **Auth0 Public Cloud Status** en la navegación superior derecha.

<div id="subscribe-to-private-cloud-status-updates">
  ## Suscribirse a las actualizaciones de estado de Private Cloud
</div>

Seleccione un entorno y, a continuación, seleccione **Subscribe**.

Suscríbase al feed Atom o RSS de Auth0 copiando la URL generada en el cuadro &#39;Subscribe&#39; para recibir actualizaciones de estado que afecten a su entorno.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La URL generada del feed ATOM / RSS es única para cada entorno de Private Cloud y contiene tokens que pueden rotarse [poniéndose en contacto con Soporte](https://support.auth0.com/).
</Callout>
