Ve a la página [Auth0 Public Cloud Status](https://status.auth0.com) para consultar la disponibilidad del servicio de la versión de Public Cloud de Auth0.

Selecciona una **región** en el menú desplegable para ver el estado de los Core Services (como la autenticación de usuarios y la autenticación de máquina a máquina) y los Supporting Services (como <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>, <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, los registros del tenant y la CDN).

Para obtener más información sobre las regiones disponibles, consulta la tabla [región, locality, and sub-locality](/docs/es/get-started/auth0-overview/create-tenants#region-locality-and-sub-locality) en [Create Tenants](/docs/es/get-started/auth0-overview/create-tenants).

<div id="subscribe-to-public-cloud-status-updates">
  ## Suscribirse a las actualizaciones de estado de Public Cloud
</div>

Selecciona una **Región** y, después, selecciona **Subscribe** para recibir actualizaciones. Puedes elegir entre dos opciones para consultar el estado:

1. Sigue a [@auth0status](https://x.com/auth0status) en X para obtener las actualizaciones de estado más recientes.
2. Suscríbete al feed Atom o RSS de Auth0 para recibir actualizaciones de estado que afecten a tu tenant. Sustituye `{YOUR_TENANT_DOMAIN}` por el **Domain** de tu tenant.

Para encontrar el **Domain** de tu tenant, ve a [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/dashboard/#/applications), selecciona una aplicación y, luego, ve a **Settings**. El **Domain** de tu tenant aparece en **Basic Information**.

<div id="check-auth0-status-for-private-cloud">
  ## Consulta el estado de Auth0 para Private Cloud
</div>

Ve a [Auth0 Public Cloud Status](https://status.auth0.com) y selecciona **Private Cloud Login**.

Una vez que hayas iniciado sesión, podrás ver el estado de cualquier entorno que no sea de desarrollo (por ejemplo, `travel0-prod-1`), si está disponible en la Auth0 Converged Platform.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para acceder al estado de tu entorno de Private Cloud, debes ser miembro del tenant de soporte de Public Cloud asociado. Para obtener más información, consulta [Open and Manage Support Tickets](/docs/es/troubleshoot/customer-support/open-and-manage-support-tickets) y [Manage Dashboard Access](/docs/es/get-started/manage-dashboard-access).
</Callout>

Para volver a la página de Auth0 Public Cloud Status, selecciona **Auth0 Public Cloud Status** en la navegación superior derecha.

<div id="subscribe-to-private-cloud-status-updates">
  ## Suscribirse a las actualizaciones de estado de Private Cloud
</div>

Selecciona un Environment y, a continuación, **Subscribe**.

Suscríbete al feed Atom o RSS de Auth0 copiando la URL generada en el cuadro &#39;Subscribe&#39; para recibir actualizaciones de estado sobre tu entorno.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La URL generada del feed ATOM / RSS es única para cada entorno de Private Cloud y contiene tokens que pueden rotarse [poniéndote en contacto con Support](https://support.auth0.com/).
</Callout>
