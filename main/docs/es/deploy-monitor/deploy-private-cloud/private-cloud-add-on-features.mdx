La opción de despliegue de Private Cloud incluye dos funcionalidades complementarias: Geo-failover y PCI Compliance. Si tiene alguna pregunta sobre estas funcionalidades, póngase en contacto con [Ventas de Auth0](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales).

<div id="private-cloud-geo-failover">
  ## Private Cloud Geo-Failover
</div>

Geo-failover es una configuración de conmutación por error activo-pasivo que permite a los clientes especificar una región secundaria de conmutación por error para usarla en caso de una caída regional grave. En general, se puede seleccionar como región de conmutación por error cualquier región del mismo proveedor de nube con una latencia inferior a 100 ms. Sin embargo, en algunos casos, los proveedores de nube prefieren regiones de conmutación por error específicas. El equipo de incorporación de Auth0 trabajará con usted para seleccionar las mejores regiones para su caso de uso.

Además, se requiere una tercera región de &quot;desempate&quot; para uno de nuestros almacenes de datos. Esta tercera región nunca recibe tráfico de producción, pero es necesaria para elegir a un nuevo líder en caso de una conmutación por error.

Si se detecta un fallo regional, Auth0 activa Geo-failover en nombre del cliente. Todo el proceso queda automatizado una vez activado. Cuando se recibe la confirmación de que la región ya no presenta problemas, Auth0 revertirá a la región principal.

  ## PCI Compliance
</div>

PCI Compliance está disponible como complemento para los clientes que lo necesiten.
