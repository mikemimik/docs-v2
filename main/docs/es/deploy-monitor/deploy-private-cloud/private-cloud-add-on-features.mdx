La opción de despliegue de Private Cloud incluye dos funcionalidades complementarias: Geo-failover y cumplimiento de PCI. Si tienes alguna pregunta sobre estas funcionalidades, [contacta con Ventas de Auth0](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales).

<div id="private-cloud-geo-failover">
  ## Private Cloud Geo-Failover
</div>

Geo-failover es una configuración de conmutación por error activo-pasivo que permite a los clientes especificar una región secundaria de failover para utilizarla en caso de una interrupción regional grave. En general, puede seleccionarse como región de failover cualquier región del mismo proveedor de nube que tenga una latencia de hasta 100 ms. Sin embargo, en algunos casos, los proveedores de nube prefieren regiones de failover específicas. El equipo de incorporación de Auth0 trabajará con usted para seleccionar las regiones más adecuadas para su caso de uso.

Además, se requiere una tercera región de &quot;desempate&quot; para uno de nuestros almacenes de datos. Esta tercera región nunca recibe tráfico real, pero es necesaria para elegir un nuevo líder en caso de failover.

Si se detecta un fallo regional, Auth0 activa Geo-failover en nombre del cliente. Todo el proceso queda automatizado una vez activado. Cuando se confirma que la región ya no presenta problemas, Auth0 revertirá el servicio a la región principal.
