Los paquetes de Private Cloud son servicios gestionados que puedes usar si:

* No puedes usar un servicio Public Cloud multi-tenant en tu organización
* Tienes requisitos geográficos de implementación para los que la opción de Public Cloud no es viable
* Necesitas un nivel garantizado de peticiones por segundo (RPS) o una capacidad superior a la disponible en Public Cloud

Private Cloud de Auth0 puede implementarse de las siguientes maneras:

| Opción                                                                                         | Descripción                                                                                                                                                      |
| ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [**Private Cloud on AWS**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-aws)     | Un servicio de nube dedicado y gestionado que se ejecuta en AWS y ofrece aislamiento, mayor rendimiento, instancias de desarrollo, add-ons de Geo-HA y mucho más |
| [**Private Cloud on Azure**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-azure) | Un servicio de nube dedicado que se ejecuta en Azure y ofrece aislamiento, mayor rendimiento, instancias de desarrollo, add-ons y mucho más                      |

También ofrecemos [funcionalidades add-on](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-add-on-features) que pueden complementar tu implementación de Auth0 Private Cloud.

Si tienes requisitos de soporte específicos o necesitas más información sobre los Professional Services que ofrecemos, [ponte en contacto con Ventas de Auth0](https://auth0.com/contact-us).
