Los paquetes de Private Cloud son servicios administrados que puedes utilizar si:

* No puedes usar un servicio de Public Cloud multi-tenant en tu organización
* Tienes requisitos de implementación geográfica para los que la opción de Public Cloud no es factible
* Necesitas un nivel garantizado de solicitudes por segundo (RPS) o una capacidad superior a la disponible en Public Cloud

La nube privada de Auth0 puede implementarse de las siguientes maneras:

| Opción                                                                                         | Descripción                                                                                                                                                                  |
| ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [**Private Cloud en AWS**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-aws)     | Un servicio de nube dedicado y administrado que se ejecuta en AWS y proporciona aislamiento, mayor rendimiento, instancias de desarrollo, complementos de Geo-HA y mucho más |
| [**Private Cloud on Azure**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-azure) | Un servicio de nube dedicado que se ejecuta en Azure y proporciona aislamiento, mayor rendimiento, instancias de desarrollo, complementos y mucho más                        |

También ofrecemos [funcionalidades complementarias](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-add-on-features) que pueden complementar tu implementación de Auth0 Private Cloud.

Si tienes requisitos de soporte específicos o necesitas más información sobre los Servicios Profesionales que ofrecemos, [contacta con Ventas de Auth0](https://auth0.com/contact-us).
