Los paquetes de Private Cloud son servicios administrados que puedes usar si:

* No puedes usar un servicio de Public Cloud multi-tenant en tu organización
* Tienes requisitos de despliegue geográfico para los que la opción de Public Cloud no es viable
* Requieres un nivel garantizado de solicitudes por segundo (RPS) o una capacidad mayor que la disponible en Public Cloud

La Private Cloud de Auth0 puede implementarse de las siguientes formas:

| Opción                                                                                         | Descripción                                                                                                                                                     |
| ---------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [**Private Cloud on AWS**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-aws)     | Un servicio en la nube dedicado y administrado que se ejecuta en AWS y ofrece aislamiento, mayor rendimiento, instancias de desarrollo, add-ons de Geo-HA y más |
| [**Private Cloud on Azure**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-azure) | Un servicio en la nube dedicado que se ejecuta en Azure y ofrece aislamiento, mayor rendimiento, instancias de desarrollo, add-ons y más                        |

También ofrecemos [funcionalidades add-on](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-add-on-features) que pueden complementar tu despliegue de Private Cloud de Auth0.

Si tienes requisitos específicos de soporte o necesitas más información sobre los Servicios Profesionales que ofrecemos, [contacta con Ventas de Auth0](https://auth0.com/contact-us).
