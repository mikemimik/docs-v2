Los paquetes de Private Cloud son servicios administrados que puede usar si:

* No puede usar un servicio de Public Cloud multi-tenant en su organización
* Tiene requisitos geográficos de implementación para los que la opción de Public Cloud no resulta viable
* Requiere un nivel garantizado de solicitudes por segundo (RPS) o una capacidad superior a la disponible en Public Cloud

La Private Cloud de Auth0 se puede implementar de las siguientes maneras:

| Opción                                                                                         | Descripción                                                                                                                                                    |
| ---------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [**Private Cloud on AWS**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-aws)     | Un servicio de nube dedicado y administrado que se ejecuta en AWS y ofrece aislamiento, mayor rendimiento, instancias de desarrollo, complementos Geo-HA y más |
| [**Private Cloud on Azure**](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-azure) | Un servicio de nube dedicado que se ejecuta en Azure y ofrece aislamiento, mayor rendimiento, instancias de desarrollo, complementos y más                     |

También ofrecemos [funcionalidades adicionales](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-add-on-features) que pueden complementar su implementación de Private Cloud de Auth0.

Si tiene requisitos específicos de soporte o necesita más información sobre los Servicios Profesionales que ofrecemos, [póngase en contacto con Ventas de Auth0](https://auth0.com/contact-us).
