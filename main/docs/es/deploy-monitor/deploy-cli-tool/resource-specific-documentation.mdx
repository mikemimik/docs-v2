En general, los archivos de configuración de recursos de Deploy CLI se corresponden estrechamente con los esquemas de payload de la [Auth0 Management API](https://auth0.com/docs/api/management/v2), pero hay algunos matices importantes que debes tener en cuenta.

<div id="client-grants">
  ## Autorizaciones de cliente
</div>

La propia autorización de cliente de Deploy CLI no se exporta intencionalmente ni puede configurarse por sí sola. Esto se hace para evitar cambios incompatibles; de lo contrario, la herramienta podría revocar el acceso o incluso bloquearse en medio de una importación. En un contexto multi-tenant y multi-environment, se espera que los nuevos tenants ya cuenten con un cliente designado para Deploy CLI, como se menciona en [Getting Started](/docs/es/deploy-monitor/deploy-cli-tool).

<div id="prompts">
  ## Prompts
</div>

Los prompts de texto personalizado multilingüe siguen una jerarquía específica. Dentro de la propiedad del recurso `prompts` en el nivel raíz hay una propiedad propia, `customText`, que se utiliza para agrupar las traducciones de texto personalizado con otras opciones de configuración de los prompts. Debajo de `customText` se encuentra el código de idioma de dos caracteres. En tercer lugar está el ID del prompt, seguido del ID de pantalla y, por último, del ID de texto.

<div id="hierarchy">
  ### Jerarquía
</div>

```yaml lines
prompts:
  customText:
    <LANGUAGE>: # código de idioma de dos caracteres
      <PROMPT_ID>: # ID del prompt
        <SCREEN_ID>: # ID de pantalla del prompt
          <TEXT_ID>: 'Some text'
```

<div id="example">
  ### Ejemplo
</div>

```yaml lines
prompts:
  identifier_first: true
  universal_login_experience: classic
  customText:
    en:
      login:
        login:
          description: Descripción de inicio de sesión en inglés
          buttonText: Texto del botón
      mfa:
        mfa-detect-browser-capabilities:
          pickAuthenticatorText: 'Probar otro método'
          reloadButtonText: 'Recargar'
          noJSErrorTitle: 'JavaScript requerido'
        mfa-login-options:
          pageTitle: 'Iniciar sesión en ${clientName}'
          authenticatorNamesSMS: 'SMS'
```

<div id="databases">
  ## Bases de datos
</div>

Al gestionar conexiones de base de datos, los valores de `options.customScripts` apuntan a archivos específicos de Javascript en relación con la ruta de la carpeta de salida. Por lo demás, el payload coincide en gran medida con el de la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Connections/post_connections).

### Ejemplo en YAML

Estructura de carpetas en modo YAML:

```text lines
./databases/
    /Username-Password-Authentication
        /change_password.js   
        /create.js   
        /delete.js   
        /get_user.js   
        /login.js   
        /verify.js   
./tenant.yaml
```

 Contenido del archivo `tenant.yaml`:

```yaml lines
databases:
  - name: Username-Password-Authentication
    # ...
    options:
      # ...
      customScripts:
        change_password: ./databases/Username-Password-Authentication/change_password.js
        create: ./databases/Username-Password-Authentication/create.js
        delete: ./databases/Username-Password-Authentication/delete.js
        get_user: ./databases/Username-Password-Authentication/get_user.js
        login: ./databases/Username-Password-Authentication/login.js
        verify: ./databases/Username-Password-Authentication/verify.js
```

### Ejemplo de directorio

Estructura de carpetas en el modo directorio:

```text lines
./database-connections/
    ./Username-Password-Authentication/
        ./change_password.js
        ./create.js
        ./database.json
        ./delete.js
        ./get_user.js
        ./login.js
        ./verify.js
```

Contenido de `database.json`:

```json lines
{
  "options": {
    "customScripts": {
      "change_password": "./change_password.js",
      "create": "./create.js",
      "delete": "./delete.js",
      "get_user": "./get_user.js",
      "login": "./login.js",
      "verify": "./verify.js"
    }
  }
}
```

<div id="universal-login">
  ## Universal Login
</div>

<div id="pages">
  ### Páginas
</div>

Al sobrescribir el <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> con HTML personalizado, el contenido de error, inicio de sesión, <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> y restablecimiento de contraseña se organiza en páginas HTML específicas.

<div id="yaml-example">
  #### Ejemplo en YAML
</div>

Estructura de carpetas en modo YAML:

```text lines
./pages/
    /error_page.html
    /guardian_multifactor.html
    /login.html
    /password_reset.html
./tenant.yaml
```

Contenido de `tenant.yaml`:

```yaml lines
pages:
  - name: error_page
    html: ./pages/error_page.html
    show_log_link: false
    url: https://mycompany.org/error
  - name: guardian_multifactor
    enabled: true
    html: ./pages/guardian_multifactor.html
  - name: login
    enabled: false
    html: ./pages/login.html
  - name: password_reset
    enabled: true
    html: ./pages/password_reset.html
```

<div id="directory-example">
  #### Ejemplo de directorio
</div>

Estructura de carpetas en el modo directorio:

```text lines
./pages/
    ./error_page.html
    ./error_page.json
    ./guardian_multifactor.html
    ./guardian_multifactor.json
    ./login.html
    ./login.json
    ./password_reset.html
    ./password_reset.json
```

Contenido de `login.json`:

```json lines
{
  "name": "login",
  "enabled": false,
  "html": "./login.html"
}
```

Contenido de `error_page.json`:

```json lines
{
  "html": "./error_page.html",
  "show_log_link": false,
  "url": "https://mycompany.org/error",
  "name": "error_page"
}
```

Contenido de `guardian_multifactor.json`:

```json lines
{
  "enabled": true,
  "html": "./guardian_multifactor.html",
  "name": "guardian_multifactor"
}
```

Contenido de `password_reset.json`:

```json lines
{
  "enabled": true,
  "html": "./password_reset.html",
  "name": "password_reset"
}
```
