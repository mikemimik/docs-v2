Se admiten tres métodos para autenticar Auth0 Deploy CLI con tu tenant.

<div id="client-credentials">
  ## Credenciales de cliente
</div>

La autenticación con [credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow) requiere que proporciones directamente a la aplicación un par de <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip> y <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>. Esta opción es sencilla y ofrece la forma más rápida de configurar la herramienta.

Para configurar la autenticación con credenciales de cliente, establece los siguientes campos en tu [archivo de configuración](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                 | Valor                                         |
| --------------------- | --------------------------------------------- |
| `AUTH0_CLIENT_ID`     | ID de cliente de tu aplicación de Auth0.      |
| `AUTH0_CLIENT_SECRET` | secreto de cliente de tu aplicación de Auth0. |

<div id="private-key-jwt">
  ## Clave privada JWT
</div>

La autenticación de [Private key JWT](/docs/es/get-started/applications/configure-private-key-jwt) requiere que configures una clave privada para la aplicación y una clave pública para el tenant remoto. Esta opción es ideal si no quieres almacenar credenciales en Auth0.

Para configurar la autenticación con clave privada <Tooltip tip="JSON Web Token (JWT): formato estándar de token de identidad (y, a menudo, también de token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>, establece los siguientes campos en tu [archivo de configuración](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                            | Valor                                                          |
| -------------------------------- | -------------------------------------------------------------- |
| `AUTH0_CLIENT_SIGNING_KEY_PATH`  | Ruta relativa al certificado de la clave de firma de JWT.      |
| `AUTH0_CLIENT_SIGNING_ALGORITHM` | Opcional. Algoritmo de firma de JWT usado para el certificado. |

<div id="access-token">
  ## Token de acceso
</div>

La autenticación con [token de acceso](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) requiere que proporciones directamente a la aplicación un <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de Auth0. Esta opción traslada más responsabilidad a los desarrolladores, pero puede permitir flujos de trabajo flexibles y específicos cuando sea necesario.

Para configurar la autenticación con token de acceso, pasa un token de acceso de la Management API de Auth0 mediante la `AUTH0_ACCESS_TOKEN` [variable de entorno](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).
