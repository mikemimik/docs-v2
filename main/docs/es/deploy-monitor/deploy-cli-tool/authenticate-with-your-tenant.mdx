Se admiten tres métodos para autenticar Auth0 Deploy CLI con tu tenant.

<div id="client-credentials">
  ## Credenciales de cliente
</div>

La autenticación mediante [credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow) requiere que proporciones directamente a la aplicación un par de <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip> y <Tooltip tip="Secreto de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>. Esta opción es sencilla y permite configurar la herramienta de la forma más rápida.

Para configurar la autenticación mediante credenciales de cliente, establece los siguientes campos en tu [archivo de configuración](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                 | Valor                                         |
| --------------------- | --------------------------------------------- |
| `AUTH0_CLIENT_ID`     | ID de cliente de tu aplicación de Auth0.      |
| `AUTH0_CLIENT_SECRET` | secreto de cliente de tu aplicación de Auth0. |

<div id="private-key-jwt">
  ## Clave privada JWT
</div>

La [autenticación con clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) requiere que configures una clave privada para la aplicación y una clave pública para el tenant remoto. Esta opción es ideal si no quieres almacenar credenciales en Auth0.

Para configurar la autenticación con clave privada <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>, establece los siguientes campos en tu [archivo de configuración](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                            | Valor                                                               |
| -------------------------------- | ------------------------------------------------------------------- |
| `AUTH0_CLIENT_SIGNING_KEY_PATH`  | Ruta relativa al certificado de la clave de firma de JWT.           |
| `AUTH0_CLIENT_SIGNING_ALGORITHM` | Opcional. Algoritmo de firma de JWT que se usa para el certificado. |

<div id="access-token">
  ## Token de acceso
</div>

La autenticación con [token de acceso](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) requiere que proporcione un <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de Auth0 directamente a la aplicación. Esta opción traslada más responsabilidad a los desarrolladores, pero puede permitir flujos de trabajo flexibles y específicos cuando sea necesario.

Para configurar la autenticación con token de acceso, pase un token de acceso de Auth0 para la Management API a través de la [variable de entorno](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) `AUTH0_ACCESS_TOKEN`.
