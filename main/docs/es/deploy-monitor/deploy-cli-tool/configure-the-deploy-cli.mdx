---
description: Aprende a configurar Auth0 Deploy CLI para adaptar la herramienta a tus necesidades.
title: Configurar Deploy CLI
---

Configurar Deploy CLI es esencial para establecer las credenciales de Auth0, así como para adaptar el comportamiento de la herramienta a tus necesidades específicas. Hay dos formas de configurar Deploy CLI:

1. Usar el archivo de configuración (`config.json`).
2. Establecer variables de entorno.

<div id="configuration-file">
  ## Archivo de configuración
</div>

Se puede usar un archivo JSON independiente para configurar Deploy CLI. Este archivo suele estar en el directorio raíz de tu proyecto y debe llamarse `config.json`.

### Ejemplo

```json lines
{
  "AUTH0_DOMAIN": "<YOUR_TENANT_DOMAIN>",
  "AUTH0_CLIENT_ID": "{yourClientId}",
  "AUTH0_ALLOW_DELETE": false
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No se recomienda dejar las credenciales codificadas de forma fija, ya que existe el riesgo de que se expongan secretos si este archivo llegara a confirmarse en un sistema público de control de versiones. Para mitigar este riesgo, pase las credenciales como [variables de entorno](#environment-variables), como práctica recomendada.
</Callout>

<div id="environment-variables">
  ## Variables de entorno
</div>

De forma predeterminada, Deploy CLI lee variables de entorno, lo que permite pasar credenciales y otras configuraciones a la herramienta sin necesidad de incluirlas en el archivo `config.json`. Las variables de entorno pueden usarse para complementar el archivo `config.json` o para reemplazarlo por completo, según las necesidades del proyecto.

Los valores de configuración no primitivos, como `AUTH0_KEYWORD_REPLACE_MAPPINGS` y `AUTH0_EXCLUDED`, también pueden pasarse mediante variables de entorno, siempre que estén serializados correctamente como JSON.

Para deshabilitar el uso de variables de entorno en los comandos `import` o `export`, pasa el argumento `--env=false`.

<div id="examples">
  ### Ejemplos
</div>

<div id="deploying-configuration-for-yaml-formats-without-a-configjson-file">
  #### Despliegue de la configuración para formatos YAML sin archivo config.json
</div>

```bash lines
$ export AUTH0_DOMAIN={yourDomain}
$ export AUTH0_CLIENT_ID={yourClientId}
$ export AUTH0_CLIENT_SECRET={yourClientSecret}
$ a0deploy import --input_file=local/tenant.yaml
```

<div id="disable-environment-variable-ingestion">
  #### Desactivar la lectura de variables de entorno
</div>

```bash wrap lines
$ a0deploy export -c=config.json --format=yaml --output_folder=local --env=false
```

<div id="non-primitive-configuration-values">
  #### Valores de configuración no primitivos
</div>

```bash lines
$ export AUTH0_EXCLUDED='["actions","organizations"]'
$ export AUTH0_KEYWORD_REPLACE_MAPPINGS='{"ENVIRONMENT":"dev"}'
$ a0deploy export -c=config.json --format=yaml --output_folder=local
```

<div id="available-configuration-properties">
  ## Propiedades de configuración disponibles
</div>

<div id="auth0_domain">
  ### AUTH0_DOMAIN
</div>

String. El dominio del tenant de Auth0 de destino.

<div id="auth0_client_id">
  ### AUTH0_CLIENT_ID
</div>

String. El ID de la aplicación de Auth0 especificada que se utiliza para realizar solicitudes a la API.

<div id="auth0_client_secret">
  ### AUTH0_CLIENT_SECRET
</div>

String. El secreto de la aplicación de Auth0 especificada que se usa para realizar solicitudes a la API.

<div id="auth0_access_token">
  ### AUTH0_ACCESS_TOKEN
</div>

String. <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de corta duración para la <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de la aplicación de Auth0 especificada. Se puede usar en sustitución de la combinación de <Tooltip tip="ID de cliente: valor de identificación dado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip> y <Tooltip tip="ID de cliente: valor de identificación dado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>.

<div id="auth0_client_signing_key_path">
  ### AUTH0_CLIENT_SIGNING_KEY_PATH
</div>

String. La ruta de la clave privada que utiliza el cliente para la autenticación mediante <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">clave privada JWT</Tooltip> con clave privada. Ruta relativa al directorio de trabajo. Ten en cuenta también `AUTH0_CLIENT_SIGNING_ALGORITHM` para especificar el <Tooltip tip="Signing Algorithm: Algoritmo utilizado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=signing+algorithm">algoritmo de firma</Tooltip>.

<div id="auth0_client_signing_algorithm">
  ### AUTH0_CLIENT_SIGNING_ALGORITHM
</div>

String. Especifica los algoritmos de firma de JWT que usa el cliente para la autenticación con clave privada JWT. Solo se usa en combinación con `AUTH0_CLIENT_SIGNING_KEY_PATH`.

Valores aceptados: `RS256`, `RS384`, `PS256`.

<div id="auth0_allow_delete">
  ### AUTH0_ALLOW_DELETE
</div>

booleano. Si se habilita, la herramienta podrá eliminar recursos. Valor predeterminado: `false`.

<div id="auth0_excluded">
  ### AUTH0_EXCLUDED
</div>

Arreglo de cadenas. Excluye tipos de recursos completos de la gestión, de forma bidireccional. Para obtener más información, consulta [Excluir recursos de la gestión](/docs/es/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

No puede usarse simultáneamente con [AUTH0&#95;INCLUDED&#95;ONLY](#auth0_included_only).

Valores posibles: `actions`, `attackProtection`, `branding`, `clientGrants`, `clients`, `connections`, `customDomains`, `databases`, `emailProvider`, `emailTemplates`, `guardianFactorProviders`, `guardianFactorTemplates`, `guardianFactors`, `guardianPhoneFactorMessageTypes`, `guardianPhoneFactorSelectedProvider`, `guardianPolicies`, `logStreams`, `migrations`, `organizations`, `pages`, `prompts`, `resourceServers`, `roles`, `tenant`, `triggers`.

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "AUTH0_EXCLUDED": ["organizations", "connections", "hooks"]
}
```

<div id="auth0_included_only">
  ### AUTH0_INCLUDED_ONLY
</div>

Arreglo de cadenas. Especifica qué tipos de recursos se deben administrar de forma bidireccional. Para obtener más información, consulta [Excluir recursos de la gestión](/docs/es/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

No puede usarse simultáneamente con [AUTH0&#95;EXCLUDED](#auth0_excluded).

Valores posibles: `actions`, `attackProtection`, `branding`, `clientGrants`, `clients`, `connections`, `customDomains`, `databases`, `emailProvider`, `emailTemplates`, `guardianFactorProviders`, `guardianFactorTemplates`, `guardianFactors`, `guardianPhoneFactorMessageTypes`, `guardianPhoneFactorSelectedProvider`, `guardianPolicies`, `logStreams`, `migrations`, `organizations`, `pages`, `prompts`, `resourceServers`, `roles`, `tenant`, `triggers`.

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "AUTH0_INCLUDED_ONLY": ["clients", "connections", "tenant", "branding"]
}
```

<div id="auth0_keyword_replace_mappings">
  ### AUTH0_KEYWORD_REPLACE_MAPPINGS
</div>

Mapeo de palabras clave específicas para facilitar el reemplazo dinámico. Para obtener más información, consulta [Reemplazo de palabras clave](/docs/es/deploy-monitor/deploy-cli-tool/keyword-replacement).

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "ENVIRONMENT": "DEV",
  "ALLOWED_ORIGINS": ["https://dev.test-site.com", "localhost"]
}
```

<div id="auth0_preserve_keywords">
  ### AUTH0_PRESERVE_KEYWORDS
</div>

Booleano. Cuando está habilitado, se intentará preservar los marcadores de reemplazo de palabras clave en los archivos de recursos locales durante la exportación. De lo contrario, los valores remotos sobrescribirán esos marcadores de palabras clave colocados manualmente.

Esta configuración requiere la presencia de archivos de configuración locales y de mapeos de reemplazo de palabras clave definidos mediante la propiedad de configuración `AUTH0_KEYWORD_REPLACE_MAPPINGS`.

Consulte también: [Preservar palabras clave al exportar](/docs/es/deploy-monitor/deploy-cli-tool/keyword-replacement#preserve-keywords-on-export).

<div id="auth0_export_identifiers">
  ### AUTH0_EXPORT_IDENTIFIERS
</div>

Booleano. Cuando está habilitado, devuelve los identificadores de todos los recursos. Puede resultar útil en ciertos casos de depuración o registro dentro de un contexto de tenant único. Predeterminado: `false`.

<div id="excluded_props">
  ### EXCLUDED_PROPS
</div>

Permite excluir de la gestión las propiedades no deseadas.

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "connections": ["options.twilio_token"]
}
```

<div id="auth0_audience">
  ### AUTH0_AUDIENCE
</div>

String. Separe este valor del valor de <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Llamado aud en un token, su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip> al obtener un token de acceso para la Management API. Útil cuando los endpoints predeterminados de la Management API no están expuestos públicamente.

<div id="auth0_excluded_rules">
  ### AUTH0_EXCLUDED_RULES
</div>

Arreglo de cadenas. Excluye de la gestión las reglas específicas por ID. Nota: Esta configuración puede entrar en obsolescencia en el futuro. Para obtener más información, consulta [Excluir recursos de la gestión](/docs/es/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_clients">
  ### AUTH0_EXCLUDED_CLIENTS
</div>

arreglo de cadenas. Excluye la gestión de clientes específicos por ID. Nota: Esta configuración puede quedar sujeta a obsolescencia en el futuro. Para obtener más información, consulta [Excluir recursos de la gestión](/docs/es/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_databases">
  ### AUTH0_EXCLUDED_DATABASES
</div>

Arreglo de cadenas. Excluye de la gestión bases de datos específicas por ID. Nota: Esta configuración puede quedar obsoleta en el futuro. Para obtener más información, lee [Excluir recursos de la gestión](/docs/es/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_connections">
  ### AUTH0_EXCLUDED_CONNECTIONS
</div>

Arreglo de cadenas. Excluye de la gestión conexiones específicas por su ID. Nota: Esta configuración podría entrar en obsolescencia en el futuro. Para obtener más información, consulta [Excluir recursos de la gestión](/docs/es/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_resource_servers">
  ### AUTH0_EXCLUDED_RESOURCE_SERVERS
</div>

Arreglo de cadenas. Excluye de la gestión <Tooltip tip="Servidor de recursos: Servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/docs/es/glossary?term=resource+servers">servidores de recursos</Tooltip> específicos mediante su ID. Nota: Esta configuración puede estar sujeta a obsolescencia en el futuro. Para obtener más información, lee [Excluir recursos de la gestión](/docs/es/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).
