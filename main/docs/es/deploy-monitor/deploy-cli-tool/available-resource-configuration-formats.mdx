description: Obtenga información sobre los formatos de configuración disponibles al usar Auth0 Deploy CLI.
title: Formatos de configuración de recursos disponibles
---

El estado de los recursos de Auth0 se representa en dos formatos de archivo de configuración diferentes: YAML y directorio (JSON). Al usar el comando `export` de Deploy CLI, se le pedirá que elija entre uno u otro.

<div id="yaml">
  ## YAML
</div>

El formato YAML se presenta principalmente como un archivo plano `tenant.yaml`, junto con archivos de código complementarios para recursos como Actions y plantillas de correo electrónico. Tener un único archivo facilita el seguimiento de los cambios a lo largo del tiempo en el control de versiones. Además, ese único archivo elimina parte de la ambigüedad en los nombres de directorios y archivos, que puede no resultar evidente de inmediato.

<div id="directory-json">
  ## Directorio (JSON)
</div>

El formato de directorio separa los tipos de recursos en directorios distintos, y cada recurso individual se almacena en un archivo JSON específico. Este formato facilita una separación conceptual más clara entre cada tipo de recurso, así como entre los propios recursos individuales. Además, Deploy CLI refleja fielmente las estructuras de datos definidas en la [Auth0 Management API](https://auth0.com/docs/api/management/v2), por lo que consultar los ejemplos de JSON en la documentación puede resultar útil al usar este formato.

<div id="how-to-choose">
  ## Cómo elegir
</div>

La decisión sobre qué formato usar debería basarse principalmente en tus preferencias. Ambos formatos son soluciones viables que cumplen la misma función, aunque tienen fortalezas y debilidades ligeramente distintas, descritas anteriormente. Asegúrate de evaluar cada uno en función de tu contexto. Es importante tener en cuenta que esta elección no es permanente, y que cambiar de uno a otro con el comando `import` es una opción a tu disposición.
