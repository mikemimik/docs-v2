description: Obtén información sobre los formatos de configuración disponibles al usar Auth0 Deploy CLI.
title: Formatos de configuración de recursos disponibles
---

El estado de los recursos de Auth0 se representa en dos formatos de archivo de configuración: YAML y directorio (JSON). Al usar el comando `export` de Deploy CLI, se te pedirá que elijas entre uno y otro.

<div id="yaml">
  ## YAML
</div>

El formato YAML se presenta principalmente como un archivo plano `tenant.yaml`, con archivos de código complementarios para recursos como Actions y plantillas de correo electrónico. Contar con un único archivo facilita el seguimiento de los cambios a lo largo del tiempo en el control de versiones. Además, usar un solo archivo elimina parte de la ambigüedad en los nombres de directorios y archivos, que puede no resultar evidente a primera vista.

<div id="directory-json">
  ## Directorio (JSON)
</div>

El formato de directorio separa los tipos de recursos en directorios independientes, y cada recurso se almacena en su propio archivo JSON. Este formato facilita una separación conceptual más clara entre cada tipo de recurso, así como entre los propios recursos individuales. Además, Deploy CLI refleja fielmente las estructuras de datos definidas en la [Auth0 Management API](https://auth0.com/docs/api/management/v2), por lo que consultar los ejemplos de JSON en la documentación puede resultar útil al usar este formato.

<div id="how-to-choose">
  ## Cómo elegir
</div>

La decisión sobre qué formato usar debe basarse principalmente en las preferencias. Ambos formatos son soluciones viables para lograr lo mismo, aunque presentan fortalezas y debilidades ligeramente distintas, como se describió anteriormente. Asegúrate de evaluar cada uno en función de tu contexto. Es importante tener en cuenta que esta elección no es permanente, y que cambiar de uno a otro con el comando `import` es una opción a tu alcance.
