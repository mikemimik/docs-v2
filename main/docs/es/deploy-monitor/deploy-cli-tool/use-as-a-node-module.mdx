title: Usar como módulo de Node.js
---

Deploy CLI no solo puede usarse como un CLI independiente, sino también como un módulo de Node.js. Esto te permite administrar recursos de Auth0 desde scripts de Node.js más flexibles.

Para ver una lista completa de opciones de configuración, lee [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="dump-function">
  ## función dump
</div>

Descarga la configuración del tenant de Auth0 a la máquina local.

<div id="argument-parameters">
  ### Parámetros de los argumentos
</div>

<div id="format">
  #### format
</div>

Opciones: `yaml` o `directory`. Determina el formato de archivo de los archivos exportados de configuración de recursos. Para obtener más información, consulta [Formatos de configuración de recursos disponibles](/docs/es/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="output_folder">
  #### output_folder
</div>

Ruta. Especifica el directorio de destino donde se escribirán los archivos de configuración.

<div id="config">
  #### config
</div>

Objeto. Configura el comportamiento de esta utilidad. Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="config_file">
  #### config_file
</div>

Ruta. Especifica el archivo de configuración definido por el usuario (`config.json`). Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export_ids">
  #### export_ids
</div>

Boolean. Cuando está habilitado, exporta los campos de identificador de cada recurso. Valor predeterminado: `false`.

<div id="env">
  #### env
</div>

Boolean. Indica si la herramienta debe leer variables de entorno o no. Valor predeterminado: `false`.

<div id="proxy_url">
  #### proxy_url
</div>

Una URL para encaminar solicitudes a través de un proxy. Solo configúrala si estás detrás de un proxy.

<div id="example">
  ### Ejemplo
</div>

```jsx lines
import { dump } from 'auth0-deploy-cli';

dump({
  output_folder: './local',
  format: 'yaml',
  config: {
    AUTH0_DOMAIN: '<YOUR_AUTH0_TENANT_DOMAIN>',
    AUTH0_CLIENT_ID: '<YOUR_AUTH0_CLIENT_ID>',
    AUTH0_CLIENT_SECRET: '<YOUR_AUTH0_CLIENT_SECRET>',
  },
})
  .then(() => {
    console.log('Exportación de configuración de Auth0 exitosa');
  })
  .catch((err) => {
    console.log('Error durante la exportación de configuración de Auth0:', err);
  });
```

<div id="deploy-function">
  ## función deploy
</div>

Aplica configuraciones desde la máquina local al tenant de Auth0.

<div id="argument-parameters">
  ### Parámetros de los argumentos
</div>

<div id="input_file">
  #### input_file
</div>

Ruta. Especifica la ubicación de los archivos de configuración de recursos. En los formatos YAML, apuntará al archivo `tenant.yaml`. En los formatos de directorio, apuntará al directorio de configuración de recursos.

<div id="config">
  #### config
</div>

Objeto. Configura el comportamiento de la utilidad. Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="config_file">
  #### config_file
</div>

Ruta. Especifica el archivo de configuración definido por el usuario (`config.json`). Para más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export_ids">
  #### export_ids
</div>

Boolean. Si está habilitado, exporta los campos de identificador de cada recurso. Valor predeterminado: `false`.

<div id="env">
  #### env
</div>

Boolean. Indica si la herramienta debe leer las variables de entorno o no. Predeterminado: `false`.

<div id="proxy_url">
  #### proxy_url
</div>

Una URL para enrutar las solicitudes a través de un proxy. Configúrala solo si estás detrás de un proxy.

<div id="example">
  ### Ejemplo
</div>

```jsx lines
import { deploy } from 'auth0-deploy-cli';

deploy({
  input_file: './local/tenant.yaml',
  config: {
    AUTH0_DOMAIN: '<YOUR_AUTH0_TENANT_DOMAIN>',
    AUTH0_CLIENT_ID: '<YOUR_AUTH0_CLIENT_ID>',
    AUTH0_CLIENT_SECRET: '<YOUR_AUTH0_CLIENT_SECRET>',
  },
})
  .then(() => {
    console.log('Configuración de Auth0 aplicada al tenant correctamente');
  })
  .catch((err) => {
    console.log('Error al aplicar la configuración al tenant de Auth0:', err);
  });
```
