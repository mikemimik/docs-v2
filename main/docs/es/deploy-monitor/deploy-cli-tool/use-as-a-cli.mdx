title: Usar como CLI
---

Deploy CLI puede usarse como una utilidad de línea de comandos independiente. Esto ofrece una forma sencilla de gestionar la configuración de tu tenant de Auth0 en flujos de trabajo de CI/CD.

Para ver una lista completa de opciones de configuración, lee [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export-command">
  ## comando export
</div>

Obtiene la configuración del tenant de Auth0 y la guarda en la máquina local.

<div id="output_folder-o">
  ### `--output_folder`, `-o`
</div>

Ruta. Indica el directorio de destino donde se escribirán los archivos de configuración.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Ruta. Especifica el archivo de configuración definido por el usuario (`config.json`). Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="format-f">
  ### `--format`, `-f`
</div>

Opciones: `yaml` o `directory`. Determina el formato de los archivos de configuración de recursos exportados. Para obtener más información, consulta [Formatos de configuración de recursos disponibles](/docs/es/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="export_ids-e">
  ### `--export_ids`, `-e`
</div>

Booleano. Si se habilita, exporta los campos de identificación de cada recurso. Valor predeterminado: `false`.

<div id="env">
  ### `--env`
</div>

Booleano. Indica si la herramienta debe usar variables de entorno o no. Predeterminado: `true`.

<div id="debug">
  ### `--debug`
</div>

Booleano. Activa un registro de errores más detallado; útil para la resolución de problemas. Predeterminado: `false`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Una URL para canalizar las solicitudes a través de un proxy. Configúrala solo si estás detrás de un proxy.

<div id="examples">
  ### Ejemplos
</div>

<div id="fetching-auth0-tenant-configuration-in-the-yaml-format">
  #### Obtener la configuración del tenant de Auth0 en formato YAML
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local`

<div id="fetching-auth0-tenant-configuration-in-directory-json-format">
  #### Recuperación de la configuración del tenant de Auth0 en formato directory (JSON)
</div>

`a0deploy export -c=config.json --format=directory --output_folder=local`

<div id="fetching-auth0-tenant-configurations-with-ids-of-all-assets">
  #### Obtención de las configuraciones del tenant de Auth0 con los IDs de todos los assets
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local --export_ids=true`

<div id="import-command">
  ## comando import
</div>

Aplica configuraciones desde la máquina local al tenant de Auth0.

<div id="input_file-i">
  ### `--input_file`, `-i`
</div>

Ruta. Especifica la ubicación de los archivos de configuración de recursos. En los formatos YAML, apuntará al archivo `tenant.yaml`; en los formatos de directory, al directory de configuración de recursos.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Ruta. Especifica el archivo de configuración definido por el usuario (`config.json`). Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="env">
  ### `--env`
</div>

Booleano. Indica si la herramienta debe cargar las variables de entorno o no. Predeterminado: `true`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Una URL para enrutar solicitudes a través de un proxy. Configúrala solo si estás detrás de un proxy.

<div id="debug">
  ### `--debug`
</div>

Booleano. Activa un registro de errores más detallado; útil para la resolución de problemas. Valor predeterminado: `false`.

<div id="examples">
  ### Ejemplos
</div>

<div id="deploying-configuration-for-yaml-formats">
  #### Despliegue de la configuración para formatos YAML
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml`

<div id="deploying-configuration-for-directory-format">
  #### Despliegue de la configuración en formato de directory
</div>

`a0deploy import -c=config.json --input_file=local`

<div id="deploying-configuration-with-environment-variables-ignored">
  #### Despliegue de la configuración ignorando las variables de entorno
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml --env=false`
