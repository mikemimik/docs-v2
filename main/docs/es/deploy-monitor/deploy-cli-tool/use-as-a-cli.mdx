---
description: Aprende a usar el Deploy CLI de Auth0 como una utilidad independiente de línea de comandos.
title: Usar como CLI
---

Deploy CLI se puede usar como una utilidad independiente de línea de comandos. Esto proporciona una forma sencilla de gestionar la configuración de tu tenant de Auth0 en flujos de trabajo de CI/CD.

Para obtener una lista completa de las opciones de configuración, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export-command">
  ## comando export
</div>

Recupera la configuración del tenant de Auth0 en la máquina local.

<div id="output_folder-o">
  ### `--output_folder`, `-o`
</div>

Ruta. Especifica el directorio de destino donde se escribirán los archivos de configuración.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Ruta. Especifica el archivo de configuración definido por el usuario (`config.json`). Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="format-f">
  ### `--format`, `-f`
</div>

Opciones: `yaml` o `directory`. Determina el formato de los archivos de configuración de recursos exportados. Para obtener más información, consulta [Formatos de configuración de recursos disponibles](/docs/es/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="export_ids-e">
  ### `--export_ids`, `-e`
</div>

Booleano. Si se habilita, exporta los campos identificadores de cada recurso. Predeterminado: `false`.

<div id="env">
  ### `--env`
</div>

booleano. Indica si la herramienta debe tomar las variables de entorno o no. Predeterminado: `true`.

<div id="debug">
  ### `--debug`
</div>

Booleano. Habilita un registro de errores más detallado; útil para la resolución de problemas. Valor predeterminado: `false`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Una URL para enviar las solicitudes a través de un proxy. Configura esto solo si te conectas a través de un proxy.

<div id="examples">
  ### Ejemplos
</div>

<div id="fetching-auth0-tenant-configuration-in-the-yaml-format">
  #### Obtener la configuración del tenant de Auth0 en formato YAML
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local`

<div id="fetching-auth0-tenant-configuration-in-directory-json-format">
  #### Obtención de la configuración del tenant de Auth0 en formato de directorio (JSON)
</div>

`a0deploy export -c=config.json --format=directory --output_folder=local`

<div id="fetching-auth0-tenant-configurations-with-ids-of-all-assets">
  #### Recuperación de las configuraciones del tenant de Auth0 con los ID de todos los recursos
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local --export_ids=true`

<div id="import-command">
  ## comando import
</div>

Aplica configuraciones desde la máquina local al tenant de Auth0.

<div id="input_file-i">
  ### `--input_file`, `-i`
</div>

Ruta. Especifica la ubicación de los archivos de configuración de recursos. En los formatos YAML, apuntará al archivo `tenant.yaml`; en los formatos de directorio, apuntará al directorio de configuración de recursos.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Ruta. Especifica el archivo de configuración definido por el usuario (`config.json`). Para obtener más información, consulta [Configurar Deploy CLI](/docs/es/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="env">
  ### `--env`
</div>

booleano. Indica si la herramienta debe leer las variables de entorno o no. Valor predeterminado: `true`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Una URL para enviar solicitudes a través de un proxy. Solo configúrala si estás detrás de un proxy.

<div id="debug">
  ### `--debug`
</div>

Booleano. Habilita un registro de errores más detallado; útil para diagnosticar problemas. Valor predeterminado: `false`.

<div id="examples">
  ### Ejemplos
</div>

<div id="deploying-configuration-for-yaml-formats">
  #### Despliegue de la configuración en formato YAML
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml`

<div id="deploying-configuration-for-directory-format">
  #### Despliegue de la configuración en formato de directorio
</div>

`a0deploy import -c=config.json --input_file=local`

<div id="deploying-configuration-with-environment-variables-ignored">
  #### Despliegue de la configuración ignorando las variables de entorno
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml --env=false`
