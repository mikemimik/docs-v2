title: Reemplazo de palabras clave
---

Auth0 Deploy CLI admite el reemplazo dinámico de palabras clave con valores específicos del entorno. Esto permite un flujo de trabajo multi-tenant escalable en el que todos los tenants comparten los mismos archivos de configuración de recursos, pero inyectan valores ligeramente distintos.

Para usar el reemplazo de palabras clave, la propiedad de configuración `AUTH0_KEYWORD_REPLACE_MAPPINGS` debe contener los mapeos adecuados. Luego, en los archivos de configuración de recursos, las palabras clave pueden inyectarse de una de estas dos formas:

1. `@@EXAMPLE_KEY@@`: Usar los símbolos `@` hace que la herramienta aplique `JSON.stringify` a tu valor antes de reemplazarlo. Así, si tu valor es una cadena, la herramienta añadirá comillas; si tu valor es un arreglo o un objeto, la herramienta añadirá llaves.
2. `##EXAMPLE_KEY##`: Usar el símbolo `#` hace que la herramienta realice un reemplazo literal; no añadirá comillas ni llaves.

## Ejemplo de config.json

```json lines
{
  "AUTH0_DOMAIN": "test-tenant.us.auth0.com",
  "AUTH0_CLIENT_ID": "FOO",
  "AUTH0_CLIENT_SECRET": "BAR",
  "AUTH0_KEYWORD_REPLACE_MAPPINGS": {
    "ENVIRONMENT": "dev",
    "ALLOWED_LOGOUT_URLS": ["https://dev-test-site.com/logout", "localhost:3000/logout"],
    "ALLOWED_ORIGINS": ["https://dev-test-site.com", "localhost:3000"]
  }
}
```

## Ejemplo de tenant.yaml

```yaml lines
tenant:
  friendly_name: "##ENVIRONMENT## tenant"
  allowed_logout_urls: @@ALLOWED_LOGOUT_URLS@@
  enabled_locales:
    - en
clients:
  - name: Test App
    allowed_origins: @@ALLOWED_ORIGINS@@
    allowed_logout_urls: @@ALLOWED_LOGOUT_URLS@@
```

  "allowed_logout_urls": "@@ALLOWED_LOGOUT_URLS@@"

<div id="array-concatenation">
  ## Concatenación de arreglos
</div>

Puede haber situaciones en las que quiera concatenar valores a un arreglo estático mediante el reemplazo de palabras clave. No existe una sintaxis especial para este caso; sin embargo, es posible lograrlo escapando las comillas dobles en una sola cadena que contenga los valores adecuados e inyectándola con la sintaxis de palabra clave `##`. Esta técnica funciona tanto para los [formatos YAML y de directorio](/docs/es/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="example-configjson">
  ### Ejemplo de config.json
</div>

```json lines
{
  "AUTH0_KEYWORD_REPLACE_MAPPINGS": {
    "GLOBAL_WEB_ORIGINS": "\"http://local.me:8080\", \"http://localhost\", \"http://localhost:3000\""
  }
}
```

<div id="example-tenantyaml">
  ### Ejemplo de tenant.yaml
</div>

```yaml lines
clients:
  - name: Test App
    web_origins: [ "http://production-app.com", "https://production-app.com", ##GLOBAL_WEB_ORIGINS## ]
```

<div id="preserve-keywords-on-export">
  ## Preservar palabras clave al exportar
</div>

En general, Deploy CLI funciona mejor cuando se usa en un flujo de trabajo unidireccional desde los entornos de nivel inferior (p. ej., dev, test) hasta los entornos de producción. Sin embargo, a veces será necesario exportar la configuración desde un entorno de nivel superior a su directorio de configuración local. De forma predeterminada, los valores remotos sobrescribirán los valores locales, **lo que eliminará sus marcadores de palabras clave**. No obstante, la preservación del reemplazo de palabras clave puede habilitarse mediante la propiedad de configuración booleana `AUTH0_PRESERVE_KEYWORDS`. Al habilitar esta configuración, Deploy CLI intentará preservar los marcadores de palabras clave definidos en sus archivos de configuración locales durante la exportación.

La funcionalidad de preservación de palabras clave intentará conservar la mayor cantidad posible de palabras clave y, al mismo tiempo, mantener la precisión de sus archivos de configuración de recursos. En la mayoría de los casos, funcionará sin ninguna intervención del usuario. Sin embargo, existen algunas limitaciones importantes:

* En el caso de un campo de configuración con reemplazo de palabras clave cuyos valores difieran entre el entorno local y el remoto, siempre se priorizará el valor de la configuración local. Esto hará que **se borre cualquier cambio realizado directamente en el entorno remoto** si existe un marcador de reemplazo de palabras clave en cualquier parte del valor de ese campo en el archivo de definición del recurso; no existe una conciliación &quot;inteligente&quot;.
* Los arreglos sin identificadores específicos no pueden preservarse. Ej.: `["http://site.com/logout", "localhost:3000/logout", "##LOGOUT_URL##"]`. Esto se debe a que el orden de estos valores no es determinista. Como alternativa, para preservar estos valores, se recomienda usar la sintaxis de reemplazo de palabras clave `@@ARRAY_REPLACE@@` con el valor completo.

Para obtener más información sobre la historia y los desafíos técnicos de la preservación de palabras clave, consulte [RFC: Keyword Preservation During Export](https://github.com/auth0/auth0-deploy-cli/issues/688).
