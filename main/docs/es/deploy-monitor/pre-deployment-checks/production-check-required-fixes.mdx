---
description: Correcciones que debes realizar en tu Auth0 Application antes del despliegue en producción
title: 'Verificación de preparación para producción: correcciones críticas'
---

Las siguientes verificaciones comprueban si has cumplido todos los requisitos para desplegar correctamente en Producción.

Para obtener más información sobre el resultado de tus verificaciones, consulta [Ejecutar Production Checks](/docs/es/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Comprobación                                                                                                                                                                                               | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Todas las Actions usan una [versión recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)                                                            | Establezca todas las [Actions](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) en una versión recomendada de NodeJS para aprovechar los parches de seguridad y las nuevas funcionalidades.                                                                                                                                                                                                                                                                                                                                               |
| [Permitir tokens de ID para la autenticación en Management API v2](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) está desactivado                        | Las capacidades para usar tokens de ID para autorizar algunos de los endpoints de Users y Device Credentials de Management API se están marcando como obsoletas. Después de completar la migración a tokens de acceso, asegúrate de que el interruptor [Permitir tokens de ID para la autenticación de Management API v2](https://manage.auth0.com/#/tenant/advanced) esté desactivado. Si no puedes ver esta configuración, entonces tu tenant se creó después de que esta funcionalidad se marcara como obsoleta, por lo que ya está deshabilitada de forma predeterminada. |
| allowLegacyRoGrantTypes                                                                                                                                                                                    | El [endpoint oauth/ro heredado](/docs/es/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) de la configuración de Migración del tenant se está marcando como obsoleto. Después de actualizar tus aplicaciones para usar endpoints admitidos, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                                                                   |
| allowOtherLegacyGrantTypes                                                                                                                                                                                 | El [endpoint oauth/access&#95;token heredado](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) en la configuración de migración del tenant va a quedar obsoleto. Después de actualizar tus aplicaciones para que usen endpoints compatibles, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                                                 |
| allowLegacyTokenInfoEndpoint                                                                                                                                                                               | El [endpoint tokeninfo](/docs/es/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) heredado de la configuración de migración del tenant está quedando obsoleto. Después de actualizar tus aplicaciones para usar endpoints compatibles, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                                                                       |
| allowLegacyDelegationGrantTypes                                                                                                                                                                            | El [endpoint de delegación](/docs/es/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation) heredado de la configuración de migración del tenant se está retirando. Después de actualizar sus aplicaciones para usar endpoints admitidos, asegúrese de desactivar este endpoint.                                                                                                                                                                                                                                                                            |
| [Allowed Callback URLs](/docs/es/authenticate/login/redirect-users-after-login) no deben incluir Localhost                                                                                                    | Valida que las [URL de devolución de llamada permitidas de la aplicación](https://manage.auth0.com/#/applications) no apunten a `localhost`, `127.0.0.1` ni similares.                                                                                                                                                                                                                                                                                                                                                                                                        |
| [Orígenes permitidos (CORS)](/docs/es/authenticate/login/cross-origin-authentication) no deben ser Localhost                                                                                                  | Valida que la [URL de la página](https://manage.auth0.com/#/applications) no apunte a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| [Orígenes web permitidos](/docs/es/get-started/applications/application-settings) no deben incluir `localhost`                                                                                                | Verifica que las [URL de Orígenes web permitidos](https://manage.auth0.com/#/applications) no apunten a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| [Proveedor de correo electrónico](/docs/es/customize/email/smtp-email-providers) debe estar configurado                                                                                                       | Verifica que se haya configurado el [proveedor de correo electrónico personalizado](https://manage.auth0.com/#/emails/provider).                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Hooks quedarán obsoletos y deben [migrarse a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions)                                                                                       | Hooks van a quedar obsoletos. Le recomendamos encarecidamente que empiece a usar [Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions) y que inicie el proceso de migración. Los Hooks existentes seguirán activos hasta el 18 de noviembre de 2024.                                                                                                                                                                                                                                                                                                       |
| [Autenticación multifactor](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) debe estar configurada (Dependencia: MFA está configurado) | Si utiliza MFA con el [factor Phone Message](https://manage.auth0.com/#/mfa), configúrelo con Twilio o con un proveedor de entrega personalizado para evitar los límites de frecuencia del proveedor de entrega predeterminado.                                                                                                                                                                                                                                                                                                                                               |
| [Perfil de usuario heredado](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) debe estar desactivado                                                     | Los flujos de autenticación heredados que permiten que los tokens de ID y el endpoint `/userinfo` incluyan el perfil completo del usuario están quedando obsoletos. Después de completar la migración a las nuevas API conformes con OIDC, asegúrese de que el interruptor [Legacy User Profile](https://manage.auth0.com/#/account/advanced) esté desactivado.                                                                                                                                                                                                               |
| Rules quedará obsoleto y debe [migrarse a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions)                                                                                          | Rules están quedando obsoletas. Recomendamos encarecidamente que empiece a usar [Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) e inicie el proceso de migración. Las Rules existentes seguirán activas hasta el 18 de noviembre de 2024.                                                                                                                                                                                                                                                                                                            |
| [Conexiones sociales](/docs/es/authenticate/identity-providers/social-identity-providers/devkeys) no deben usar las claves de desarrollador de Auth0                                                          | Verifica que las [Conexiones sociales](https://manage.auth0.com/#/connections/social) no estén usando las claves de desarrollador predeterminadas de Auth0.                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Se debe configurar Support Email                                                                                                                                                                           | Asegura que el [Correo electrónico de soporte](https://manage.auth0.com/#/account) esté configurado en Configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Support URL está configurada                                                                                                                                                                               | Asegura que [Support URL](https://manage.auth0.com/#/account) esté configurado en Configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Se debe configurar la etiqueta de entorno del tenant                                                                                                                                                       | Comprueba que la [etiqueta de entorno del tenant](https://support.auth0.com/tenants/public) esté configurada correctamente como Production, Staging o Development. Los cambios en esta etiqueta afectan al límite de frecuencia del tenant.                                                                                                                                                                                                                                                                                                                                   |
| El tenant está configurado para usar la [versión predeterminada recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22)                                 | Actualiza tu tenant a una [versión recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) para beneficiarte de los parches de seguridad y las nuevas funcionalidades.                                                                                                                                                                                                                                                                                                                                                    |
| URI de inicio de sesión del tenant                                                                                                                                                                         | Garantiza que [Tenant Login URI](https://manage.auth0.com/#/tenant/advanced) esté configurado en la Configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| Usar un dominio personalizado en las plantillas de correo electrónico con tu marca                                                                                                                         | Asegúrese de que las plantillas de sus correos electrónicos usen un [dominio personalizado](/docs/es/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails). Recomendamos encarecidamente personalizar todos los correos electrónicos dirigidos a los usuarios para que usen su dominio personalizado y la imagen de marca de la empresa.                                                                                                                                                                                               |
