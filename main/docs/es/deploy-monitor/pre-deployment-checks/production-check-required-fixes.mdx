---
description: Correcciones que debes realizar en tu Auth0 Application antes del despliegue en producción
title: 'Verificaciones de preparación para producción: correcciones críticas'
---

Las siguientes verificaciones comprueban si has completado todos los requisitos para realizar un despliegue correcto en Production.

Para obtener más información sobre el resultado de tus verificaciones, consulta [Ejecutar Production Checks](/docs/es/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Verificación                                                                                                                                                                                               | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Todas las Actions usan una [versión recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)                                                            | Actualiza todas las [Actions](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) a una versión recomendada de NodeJS para aprovechar los parches de seguridad y las nuevas funciones.                                                                                                                                                                                                                                                                                                        |
| [Permitir tokens de ID para la autenticación de Management API v2](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) está desactivada                        | Se está retirando el soporte para usar token de ID para autorizar algunos de los endpoints de Users y Device Credentials de la Management API. Después de completar la migración a tokens de acceso, asegúrate de que el toggle [Allow ID Tokens for Management API v2 Authentication](https://manage.auth0.com/#/tenant/advanced) esté desactivado. Si no ves esta configuración, significa que tu tenant se creó después de que esta funcionalidad quedara obsoleta, por lo que ya está desactivada de forma predeterminada. |
| allowLegacyRoGrantTypes                                                                                                                                                                                    | El [endpoint oauth/ro](/docs/es/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) heredado de la configuración de Migración del tenant está quedando obsoleto. Después de actualizar tus aplicaciones para usar endpoints compatibles, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                          |
| allowOtherLegacyGrantTypes                                                                                                                                                                                 | El [endpoint oauth/access&#95;token heredado](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) de la configuración de migración del tenant quedará obsoleto. Después de actualizar sus aplicaciones para usar endpoints compatibles, asegúrese de desactivar este endpoint.                                                                                                                                                                                                          |
| allowLegacyTokenInfoEndpoint                                                                                                                                                                               | El [endpoint tokeninfo](/docs/es/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) heredado de la configuración de migración del tenant quedará obsoleto. Después de actualizar tus aplicaciones para usar endpoints compatibles, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                              |
| allowLegacyDelegationGrantTypes                                                                                                                                                                            | El [endpoint de delegación](/docs/es/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation) heredado en la configuración de migración del tenant está obsoleto. Después de actualizar tus aplicaciones para que usen endpoints admitidos, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                             |
| [Allowed Callback URLs](/docs/es/authenticate/login/redirect-users-after-login) no deben incluir Localhost                                                                                                    | Valida que las [URL de devolución de llamada permitidas de la aplicación](https://manage.auth0.com/#/applications) no apunten a `localhost`, `127.0.0.1` y demás.                                                                                                                                                                                                                                                                                                                                                              |
| [Orígenes permitidos (CORS)](/docs/es/authenticate/login/cross-origin-authentication) no deben incluir Localhost                                                                                              | Valida que la [URL de la ubicación](https://manage.auth0.com/#/applications) de la página no apunte a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                             |
| [Orígenes web permitidos](/docs/es/get-started/applications/application-settings) no deben incluir `localhost`                                                                                                | Valida que las [URL de Allowed Web Origins](https://manage.auth0.com/#/applications) no apunten a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                 |
| [Proveedor de correo electrónico](/docs/es/customize/email/smtp-email-providers) debe estar configurado                                                                                                       | Verifica que el [proveedor de correo electrónico personalizado](https://manage.auth0.com/#/emails/provider) esté configurado.                                                                                                                                                                                                                                                                                                                                                                                                  |
| Hooks están en desuso y deben [migrarse a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions)                                                                                          | Hooks están quedando obsoletos. Le recomendamos encarecidamente que empiece a usar [Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions) y que inicie el proceso de migración. Los Hooks existentes seguirán activos hasta el 18 de noviembre de 2024.                                                                                                                                                                                                                                                      |
| [Autenticación multifactor](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) debe estar configurada (Dependencia: MFA está configurado) | Si usa MFA con el [factor Phone Message](https://manage.auth0.com/#/mfa), configúrelo con Twilio o con un proveedor de entrega personalizado para evitar los límites de frecuencia del proveedor de entrega predeterminado.                                                                                                                                                                                                                                                                                                    |
| [Perfil de usuario heredado](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) debe estar deshabilitado                                                   | Los flujos de autenticación heredados que permiten que los tokens de ID y el endpoint `/userinfo` incluyan el perfil de usuario completo se están declarando obsoletos. Después de completar la migración a las nuevas API conformes con OIDC, asegúrate de que la opción [Legacy User Profile](https://manage.auth0.com/#/account/advanced) esté desactivada.                                                                                                                                                                 |
| Rules quedará obsoleto y debe [migrarse a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions)                                                                                          | Rules quedará obsoleto. Recomendamos encarecidamente que empiece a usar [Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y comience el proceso de migración. Las Rules existentes seguirán activas hasta el 18 de noviembre de 2024.                                                                                                                                                                                                                                                                   |
| [Conexiones sociales](/docs/es/authenticate/identity-providers/social-identity-providers/devkeys) no deben usar claves de desarrollador de Auth0                                                              | Verifica que las [Conexiones sociales](https://manage.auth0.com/#/connections/social) no usen las claves de desarrollador predeterminadas de Auth0.                                                                                                                                                                                                                                                                                                                                                                            |
| Debe configurarse Support Email                                                                                                                                                                            | Asegura que el [Correo electrónico de soporte](https://manage.auth0.com/#/account) esté configurado en Configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                                               |
| Support URL está configurada                                                                                                                                                                               | Garantiza que la [Support URL](https://manage.auth0.com/#/account) esté configurada en Configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                                                               |
| Debe configurarse la etiqueta de entorno del tenant                                                                                                                                                        | Garantiza que la [etiqueta de entorno del tenant](https://support.auth0.com/tenants/public) esté configurada adecuadamente como Production, Staging o Development. Los cambios en esta etiqueta afectan al límite de frecuencia de su tenant.                                                                                                                                                                                                                                                                                  |
| El tenant está configurado para usar la [versión predeterminada recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22)                                 | Actualiza tu tenant a una [versión recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) para aprovechar los parches de seguridad y las nuevas funciones.                                                                                                                                                                                                                                                                                                                |
| URI de inicio de sesión del tenant                                                                                                                                                                         | Garantiza que la [URI de inicio de sesión del tenant](https://manage.auth0.com/#/tenant/advanced) esté configurada en la Configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                             |
| Usa un dominio personalizado en las plantillas de correo electrónico con tu marca                                                                                                                          | Asegúrese de que las plantillas de sus correos electrónicos utilicen un [dominio personalizado](/docs/es/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails). Recomendamos encarecidamente personalizar todos los correos electrónicos dirigidos a los usuarios para que utilicen su dominio personalizado y la imagen de marca de su empresa.                                                                                                                                        |
