---
description: Recomendaciones sobre cómo puede mejorar su aplicación de Auth0 antes del despliegue en producción
title: 'Verificaciones de preparación para producción: correcciones no críticas'
---

Las siguientes verificaciones comprueban si ha completado todas las recomendaciones (opcionales) para un despliegue exitoso en Production.

Para obtener más información sobre el resultado de estas verificaciones, consulte [Ejecutar las Production Checks](/docs/es/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Verificación                                                                                                               | Descripción                                                                                                                                                                                                                                                                                                                                                           |
| -------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Configurar la transmisión de logs                                                                                          | Asegúrese de transmitir las entradas de registro del tenant a una ubicación o proveedor de almacenamiento externo. Recomendamos que [configure un servicio de transmisión de logs](/docs/es/customize/log-streams) para conservar los registros durante más de 30 días y facilitar un mejor análisis y la generación de alertas.                                         |
| Se debe configurar un [dominio personalizado](/docs/es/customize/custom-domains)                                              | Use [dominios personalizados con Universal Login](https://manage.auth0.com/#/tenant/custom_domains) para ofrecer a sus usuarios finales la experiencia más fluida y segura posible.                                                                                                                                                                                   |
| Se debe configurar una [página de error personalizada](/docs/es/customize/login-pages/custom-error-pages)                     | [Configure una página de error personalizada](https://manage.auth0.com/#/account) con los detalles específicos de su aplicación y la imagen de marca corporativa.                                                                                                                                                                                                     |
| Habilitar Universal Login                                                                                                  | Asegúrese de estar usando la versión más reciente de [Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login). Recomendamos actualizar su página de inicio de sesión de la experiencia Classic a Universal Login para beneficiarse de una mejor accesibilidad y rendimiento, personalización sin código y nuevas funciones. |
| Se deben configurar las [plantillas de correo electrónico](/docs/es/customize/email/manage-email-flow)                        | [Configure plantillas de correo electrónico personalizadas](https://manage.auth0.com/#/emails) con los detalles específicos de su aplicación y la imagen de marca corporativa.                                                                                                                                                                                        |
| Se debe configurar la [autenticación multifactor](/docs/es/secure/multi-factor-authentication)                                | Se recomienda considerar la autenticación multifactor como una práctica segura para autenticar a sus usuarios.                                                                                                                                                                                                                                                        |
| La [MFA para administradores del tenant](/docs/es/get-started/manage-dashboard-access) está habilitada                        | [Habilite la autenticación multifactor](https://manage.auth0.com/#/account/admins) para los administradores del tenant.                                                                                                                                                                                                                                               |
| Se debe definir la [Redirect Logout URL](/docs/es/authenticate/login/logout#set-the-allowed-logout-urls-at-the-account-level) | Revise las [Allowed Redirect Logout URLs](https://manage.auth0.com/#/account/advanced) de su aplicación.                                                                                                                                                                                                                                                              |
| Las aplicaciones deben usar RS256 en lugar de HS256 para el algoritmo de firma de JSON Web Token (JWT)                     | Configure el [algoritmo de firma](/docs/es/get-started/apis#signing-algorithms) de JSON Web Token (JWT) como RS256 en lugar de HS256 para cada aplicación.                                                                                                                                                                                                               |
| Establecer [Application Login URI](/docs/es/get-started/applications/application-settings#application-uris)                   | Indique un [Application Login URI](/docs/es/get-started/applications/application-settings#application-uris) para todas las aplicaciones interactivas.                                                                                                                                                                                                                    |
| Establecer [Tenant Allowed Logout URL](/docs/es/get-started/applications/application-settings#application-uris)               | Cree al menos una ruta de cierre de sesión de acceso público y [actualice la configuración de su tenant](/docs/es/get-started/applications/application-settings#application-uris) para que apunte a ella.                                                                                                                                                                |
| Establecer [Tenant Login URI](/docs/es/get-started/tenant-settings#login-and-logout)                                          | Publique una ruta de inicio de sesión predeterminada y [actualice la configuración de su tenant](/docs/es/get-started/tenant-settings#login-and-logout) para que apunte a ella.                                                                                                                                                                                          |
