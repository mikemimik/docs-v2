description: Aprenda a probar su implementación de Auth0 antes de desplegarla en entornos de producción.
title: Ejecute pruebas previas al despliegue
---

Debe ejecutar pruebas unitarias y de integración antes de desplegar Auth0 en una aplicación o servicio en producción. Realizar pruebas contra las API de Auth0 puede hacer que se aplique un [límite de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy) a su cuenta, por lo que recomendamos crear versiones simuladas de las API de Auth0 durante las pruebas. Según su entorno de desarrollo, sus herramientas de prueba también pueden ofrecer funcionalidad de simulación de API. También hay numerosas herramientas disponibles para simular API, como [MockServer](https://www.mock-server.com/) o [JSON Server](https://github.com/typicode/json-server), que puede usar para crear rápidamente API ficticias para las pruebas.

Si tiene una suscripción Enterprise, puede solicitar [pruebas de carga](/docs/es/troubleshoot/customer-support/operational-policies/load-testing-policy) en Auth0.

<div id="performance-tests">
  ## Pruebas de rendimiento
</div>

Si tienes problemas al realizar pruebas de rendimiento, utiliza los siguientes métodos para solucionarlos.

<div id="auth0-dashboard">
  ### Auth0 Dashboard
</div>

La sección [Monitoreo &gt; Logs](https://manage.auth0.com/#/logs) del Dashboard almacena datos sobre:

* Las acciones que los administradores realizan en el Dashboard
* Las autenticaciones realizadas por tus usuarios

También hay [extensiones](/docs/es/customize/extensions) que puedes usar para el registro de eventos, incluida la exportación de logs a herramientas de terceros y la recopilación de información sobre el uso de código personalizado en tu cuenta.

<div id="third-party-test-tools">
  ### Herramientas de prueba de terceros
</div>

Hay varias herramientas de prueba de terceros que puedes usar para hacer pruebas de rendimiento en API RESTful. Estas son algunas opciones que podrías considerar (ten en cuenta que Auth0 no recomienda ningún producto ni herramienta en particular):

* [Apache JMeter](https://jmeter.apache.org/)
* [Artillery](https://www.artillery.io/)
* [Micro Focus LoadRunner](https://www.radview.com/)
* [Loader](https://loader.io/)
* [RadView Webload](https://www.radview.com/)
* [SmartBear LoadUI](https://smartbear.com/)
* [Vegeta](https://github.com/tsenart/vegeta)
* [Wrk](https://github.com/wg/wrk)

Estas herramientas generan logs de actividad que te ayudan a identificar cualquier problema. Si necesitas ayuda para interpretar tu log o identificar la posible incidencia, ponte en contacto con el equipo de soporte.

<div id="http-archive-files">
  ### Archivos HTTP Archive
</div>

Si detecta un problema que puede reproducir, le recomendamos que [cree un archivo HTTP Archive (HAR)](https://support.auth0.com/center/s/article/generate-and-analyze-har-files-to-troubleshoot-auth0-issues) y se lo envíe a nuestro equipo de soporte para obtener ayuda adicional.
