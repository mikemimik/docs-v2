title: Ejecuta pruebas previas al despliegue
---

Debes ejecutar pruebas unitarias y de integración antes de desplegar Auth0 en una aplicación o servicio activos. Realizar pruebas con las API de Auth0 puede hacer que se aplique un [límite de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy) a tu cuenta, por lo que te recomendamos crear API simuladas de Auth0 durante las pruebas. Según tu entorno de desarrollo, es posible que tus herramientas de prueba también ofrezcan funciones de simulación de API. Además, hay numerosas herramientas disponibles para simular API, como [MockServer](https://www.mock-server.com/) o [JSON Server](https://github.com/typicode/json-server), que puedes usar para crear rápidamente API falsas para las pruebas.

Si tienes una suscripción Enterprise, puedes solicitar [pruebas de carga](/docs/es/troubleshoot/customer-support/operational-policies/load-testing-policy) con Auth0.

<div id="performance-tests">
  ## Pruebas de rendimiento
</div>

Si encuentras problemas al realizar pruebas de rendimiento, utiliza los siguientes métodos para solucionarlos.

<div id="auth0-dashboard">
  ### Auth0 Dashboard
</div>

La sección [Monitoreo &gt; Logs](https://manage.auth0.com/#/logs) del Dashboard almacena datos sobre:

* Acciones realizadas en el Dashboard por administradores
* Autenticaciones de tus usuarios

También hay [extensiones](/docs/es/customize/extensions) que puedes usar para fines de registro, incluida la exportación de logs a herramientas de terceros y la recopilación de información sobre el uso de código personalizado en tu cuenta.

<div id="third-party-test-tools">
  ### Herramientas de prueba de terceros
</div>

Hay varias herramientas de prueba de terceros que puede usar para realizar pruebas de rendimiento en API RESTful. Estas son algunas opciones que podría considerar (tenga en cuenta que Auth0 no respalda ningún producto ni herramienta en particular):

* [Apache JMeter](https://jmeter.apache.org/)
* [Artillery](https://www.artillery.io/)
* [Micro Focus LoadRunner](https://www.radview.com/)
* [Loader](https://loader.io/)
* [RadView Webload](https://www.radview.com/)
* [SmartBear LoadUI](https://smartbear.com/)
* [Vegeta](https://github.com/tsenart/vegeta)
* [Wrk](https://github.com/wg/wrk)

Estas herramientas proporcionan registros de actividad que le ayudan a identificar cualquier problema. Si necesita ayuda para interpretar su registro o identificar la posible incidencia, póngase en contacto con Soporte.

<div id="http-archive-files">
  ### Archivos HAR
</div>

Si detecta un problema que puede reproducir, le recomendamos que [cree un archivo HAR (HTTP Archive)](https://support.auth0.com/center/s/article/generate-and-analyze-har-files-to-troubleshoot-auth0-issues) y lo envíe a nuestro equipo de soporte para recibir asistencia adicional.
