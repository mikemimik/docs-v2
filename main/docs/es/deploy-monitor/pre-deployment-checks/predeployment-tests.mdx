description: Aprende cómo probar tu implementación de Auth0 antes de desplegarla en entornos de producción.
title: Ejecuta pruebas previas al despliegue
---

Debes ejecutar pruebas unitarias y de integración antes de desplegar Auth0 en una aplicación o servicio en producción. Realizar pruebas con las API de Auth0 puede hacer que tu cuenta quede [limitada por tasa](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy), por lo que recomendamos crear API simuladas de Auth0 durante las pruebas. Según tu entorno de desarrollo, es posible que tus herramientas de prueba también ofrezcan funciones de simulación de API. También hay numerosas herramientas disponibles para simular API, como [MockServer](https://www.mock-server.com/) o [JSON Server](https://github.com/typicode/json-server), que puedes usar para crear rápidamente API falsas para las pruebas.

Si tienes una suscripción Enterprise, puedes solicitar [pruebas de carga](/docs/es/troubleshoot/customer-support/operational-policies/load-testing-policy) en Auth0.

<div id="performance-tests">
  ## Pruebas de rendimiento
</div>

Si tienes problemas al realizar pruebas de rendimiento, utiliza los siguientes métodos para solucionarlos.

<div id="auth0-dashboard">
  ### Auth0 Dashboard
</div>

La sección [Monitoring &gt; Logs](https://manage.auth0.com/#/logs) del Dashboard almacena datos sobre:

* Las acciones realizadas por los administradores en el Dashboard
* Las autenticaciones realizadas por tus usuarios

También hay [extensiones](/docs/es/customize/extensions) que puedes usar con fines de registro, como exportar logs a herramientas de terceros y recopilar información sobre el uso de código personalizado en tu cuenta.

<div id="third-party-test-tools">
  ### Herramientas de prueba de terceros
</div>

Hay varias herramientas de prueba de terceros que puedes usar para realizar pruebas de rendimiento en API RESTful. Estas son algunas opciones que podrías considerar (ten en cuenta que Auth0 no respalda ningún producto ni herramienta en particular):

* [Apache JMeter](https://jmeter.apache.org/)
* [Artillery](https://www.artillery.io/)
* [Micro Focus LoadRunner](https://www.radview.com/)
* [Loader](https://loader.io/)
* [RadView Webload](https://www.radview.com/)
* [SmartBear LoadUI](https://smartbear.com/)
* [Vegeta](https://github.com/tsenart/vegeta)
* [Wrk](https://github.com/wg/wrk)

Estas herramientas proporcionan logs de actividad que te ayudan a identificar cualquier problema. Si necesitas ayuda para interpretar tus logs o identificar el posible problema, ponte en contacto con el equipo de Soporte.

<div id="http-archive-files">
  ### Archivos HTTP
</div>

Si detectas un problema que puedes reproducir, te recomendamos que [generes un archivo HTTP (HAR)](https://support.auth0.com/center/s/article/generate-and-analyze-har-files-to-troubleshoot-auth0-issues) y lo envíes a nuestro equipo de Soporte para obtener ayuda adicional.
