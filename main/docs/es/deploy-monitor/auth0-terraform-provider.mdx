title: Auth0 Terraform Provider
---

Deploy CLI no es la única herramienta disponible para administrar la configuración de su tenant de Auth0; también existe un [Terraform Provider con soporte oficial](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) es una herramienta de terceros para representar como código las configuraciones de sus recursos en la nube. Cuenta con un framework de complementos consolidado que admite una amplia variedad de proveedores de nube, incluido Auth0.

Auth0 Terraform Provider también está disponible para [OpenTofu](https://opentofu.org/), la herramienta de código abierto compatible con Terraform. Puede encontrar el proveedor de Auth0 en [OpenTofu Registry](https://search.opentofu.org/provider/auth0/auth0/latest).

Tanto Deploy CLI como Terraform Provider existen para ayudarle a administrar las configuraciones de su tenant de Auth0, pero cada uno tiene su propio conjunto de ventajas y desventajas.

Puede que le convenga considerar Auth0 Terraform Provider si:

* Sus flujos de trabajo de desarrollo ya utilizan Terraform u OpenTofu
* Sus necesidades de administración del tenant son granulares o solo se refieren a unos pocos recursos específicos

Puede que no le convenga considerar Auth0 Terraform Provider si:

* Su flujo de trabajo de desarrollo no usa Terraform ni OpenTofu, lo que exige una configuración inicial adicional
* Sus flujos de trabajo de desarrollo se centran principalmente en administrar varios tenants en bloque
* Su tenant tiene muchos recursos existentes, lo que puede requerir un esfuerzo considerable para “importarlos”
