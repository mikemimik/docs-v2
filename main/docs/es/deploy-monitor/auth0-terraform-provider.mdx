title: Auth0 Terraform Provider
---

El Deploy CLI no es la única herramienta disponible para gestionar la configuración de su tenant de Auth0; también existe un [Terraform Provider con soporte oficial](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) es una herramienta de terceros para representar como código la configuración de sus recursos en la nube. Cuenta con un framework de complementos consolidado que admite una amplia variedad de proveedores de nube, incluido Auth0.

El Auth0 Terraform Provider también está disponible para [OpenTofu](https://opentofu.org/), la herramienta de código abierto compatible con Terraform. Puede encontrar el proveedor de Auth0 en el [OpenTofu Registry](https://search.opentofu.org/provider/auth0/auth0/latest).

Tanto el Deploy CLI como el Terraform Provider existen para ayudarle a gestionar las configuraciones de su tenant de Auth0, pero cada uno tiene su propio conjunto de ventajas y desventajas.

Puede que le convenga considerar el Auth0 Terraform Provider si:

* Sus flujos de trabajo de desarrollo ya utilizan Terraform u OpenTofu
* Sus necesidades de gestión del tenant son detalladas o solo se refieren a unos pocos recursos específicos

Puede que no le convenga considerar el Auth0 Terraform Provider si:

* Su flujo de trabajo de desarrollo no usa Terraform ni OpenTofu, lo que requiere una Setup adicional al principio
* Sus flujos de trabajo de desarrollo se centran principalmente en gestionar sus tenants de forma masiva
* Su tenant tiene muchos recursos existentes, lo que puede requerir un esfuerzo considerable para “importarlos”
