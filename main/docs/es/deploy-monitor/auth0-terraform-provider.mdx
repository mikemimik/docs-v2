title: Auth0 Terraform Provider
---

Deploy CLI no es la única herramienta disponible para gestionar la configuración de tu tenant de Auth0; también existe un [Terraform Provider con soporte oficial](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) es una herramienta de terceros para representar como código las configuraciones de tus recursos en la nube. Cuenta con un framework de complementos consolidado que admite una amplia variedad de proveedores de nube, incluido Auth0.

Auth0 Terraform Provider también está disponible para [OpenTofu](https://opentofu.org/), la herramienta de código abierto compatible con Terraform. Puedes encontrar el proveedor de Auth0 en el [OpenTofu Registry](https://search.opentofu.org/provider/auth0/auth0/latest).

Tanto Deploy CLI como Terraform Provider existen para ayudarte a gestionar las configuraciones de tu tenant de Auth0, pero cada uno tiene sus propias ventajas y desventajas.

Quizá te convenga considerar Auth0 Terraform Provider si:

* Tus flujos de trabajo de desarrollo ya usan Terraform u OpenTofu
* Tus necesidades de gestión del tenant son granulares o solo afectan a unos pocos recursos específicos

Quizá no te convenga considerar Auth0 Terraform Provider si:

* Tu flujo de trabajo de desarrollo no usa Terraform ni OpenTofu, por lo que requiere una configuración inicial adicional
* Tus flujos de trabajo de desarrollo se centran principalmente en gestionar tus tenants de forma masiva
* Tu tenant tiene muchos recursos existentes, lo que puede requerir un esfuerzo considerable para “importarlos”
