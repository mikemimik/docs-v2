Para usar las API de <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> integrado en aplicaciones web clásicas, asegúrate de habilitar la concesión **Passwordless OTP** en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), dentro de la configuración de tu aplicación, en **Advanced Settings** &gt; **Grant Types**.

La autenticación passwordless para aplicaciones web clásicas consta de dos pasos:

1. Captura el identificador de usuario en tu aplicación (el correo electrónico o número de teléfono del usuario) y llama al endpoint `/passwordless/start` para iniciar el flujo passwordless. El usuario recibirá un correo electrónico, un SMS con un código de un solo uso o un Magic Link.
2. Si no enviaste un Magic Link, pide al usuario el código de un solo uso y llama al endpoint `/oauth/token` para obtener tokens de autenticación.

Ten en cuenta que, si usas Magic Links, no necesitas llamar a `/oauth/token`. El usuario hará clic en el Magic Link y será redirigido a la URL de devolución de llamada de la aplicación.

A continuación, mostramos algunos fragmentos de código que pueden usarse para llamar a estos endpoints de la API en distintos escenarios.

**Enviar un código de un solo uso por correo electrónico**

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/passwordless/start' \
    --header 'content-type: application/json' \
    --data '{"client_id": "{yourClientId}", "client_secret": "{yourClientSecret}", "connection": "email", "email": "{userEmail}","send": "code"}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/passwordless/start");
  var request = new RestRequest(Method.POST);
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{"client_id": "{yourClientId}", "client_secret": "{yourClientSecret}", "connection": "email", "email": "{userEmail}","send": "code"}", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/passwordless/start"

  	payload := strings.NewReader("{"client_id": "{yourClientId}", "client_secret": "{yourClientSecret}", "connection": "email", "email": "{userEmail}","send": "code"}")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("content-type", "application/json")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse<String> response = Unirest.post("https://{yourDomain}/passwordless/start")
    .header("content-type", "application/json")
    .body("{"client_id": "{yourClientId}", "client_secret": "{yourClientSecret}", "connection": "email", "email": "{userEmail}","send": "code"}")
    .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'POST',
    url: 'https://{yourDomain}/passwordless/start',
    headers: {'content-type': 'application/json'},
    data: {
      client_id: '{yourClientId}',
      client_secret: '{yourClientSecret}',
      connection: 'email',
      email: '{userEmail}',
      send: 'code'
    }
  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP
  $curl = curl_init();

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/passwordless/start",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_ENCODING => "",
    CURLOPT_MAXREDIRS => 10,
    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
    CURLOPT_CUSTOMREQUEST => "POST",
    CURLOPT_POSTFIELDS => "{"client_id": "{yourClientId}", "client_secret": "{yourClientSecret}", "connection": "email", "email": "{userEmail}","send": "code"}",
    CURLOPT_HTTPHEADER => [
      "content-type: application/json"
    ],
  ]);

  $response = curl_exec($curl);
  $err = curl_error($curl);

  curl_close($curl);

  if ($err) {
    echo "cURL Error #:" . $err;
  } else {
    echo $response;
  }
  ```

  ```python Python
  import http.client

  conn = http.client.HTTPSConnection("")

  payload = "{"client_id": "{yourClientId}", "client_secret": "{yourClientSecret}", "connection": "email", "email": "{userEmail}","send": "code"}"

  headers = { 'content-type': "application/json" }

  conn.request("POST", "/{yourDomain}/passwordless/start", payload, headers)

  res = conn.getresponse()
  data = res.read()

  print(data.decode("utf-8"))
  ```

  ```ruby Ruby
  require 'uri'
  require 'net/http'
  require 'openssl'

  url = URI("https://{yourDomain}/passwordless/start")

  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Post.new(url)
  request["content-type"] = 'application/json'
  request.body = "{"client_id": "{yourClientId}", "client_secret": "{yourClientSecret}", "connection": "email", "email": "{userEmail}","send": "code"}"

  response = http.request(request)
  puts response.read_body
  ```

Cuando envías un Magic Link, no necesitas llamar a una API para autenticar al usuario. Los usuarios harán clic en el enlace y serán redirigidos a la URL de devolución de llamada.

<div id="customize-mfa">
  ## Personalizar MFA
</div>

<ReleaseStageNotice feature="MFA personalizable con los flujos de concesión de contraseña del propietario del recurso, integrado o token de actualización" stage="ea" terms="true" contact="equipo de soporte de Auth0" />

Personaliza <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> con flujos integrados. Usa la API de MFA para permitir que los usuarios registren factores de su elección y respondan a desafíos con ellos, siempre que tu aplicación los admita.

Al usar [Lock for Web](/docs/es/libraries/lock#2-authenticating-and-getting-user-info), el endpoint `oauth/token` devuelve el error `mfa_required` e incluye el `mfa_token` que necesitas para usar la API de MFA y el parámetro `mfa_requirements` con una lista de autenticadores que tu aplicación admite actualmente:

```json lines
{
  "error": "mfa_required",
  "error_description": "Multifactor authentication required",
  "mfa_token": "Fe26...Ha",
  "mfa_requirements": {
    "challenge": [
      { "type": "otp" },
      { "type": "push-notification" },
      { "type": "phone" },
      { "type": "recovery-code" }
      { "type": "email"} //solo puede funcionar con challenge
    ]
  }
}
```

Usa `mfa_token` para llamar al endpoint [`mfa/authenticator`](https://auth0.com/docs/api/authentication/muti-factor-authentication/list-authenticators) a fin de listar todos los factores que el usuario tiene registrados y encontrar el mismo `type` que admite tu aplicación. También necesitas obtener el `authenticator_type` correspondiente para emitir desafíos:

```json lines
[
  {
    "type": "recovery-code",
    "id": "recovery-code|dev_qpOkGUOxBpw6R16t",
    "authenticator_type": "recovery-code",
    "active": true
  },
  {
    "type": "otp",
    "id": "totp|dev_6NWz8awwC8brh2dN",
    "authenticator_type": "otp",
    "active": true
  }
]
```

Proceda a aplicar el desafío MFA llamando al endpoint [`request/mfa/challenge`](https://auth0.com/docs/api/authentication/muti-factor-authentication/request-mfa-challenge).

Personalice aún más su flujo MFA con Auth0 Actions. Para obtener más información, consulte [Actions Triggers: post-challenge - objeto de API](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object).
