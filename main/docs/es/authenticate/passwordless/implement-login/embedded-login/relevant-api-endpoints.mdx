<Tooltip tip="Passwordless: Forma de autenticación que no utiliza una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> Las API se pueden usar en dos situaciones:

* Al implementar <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> y quieres personalizar la página de inicio de sesión con auth0.js para interactuar con Auth0.
* Cuando quieres incorporar el flujo de inicio de sesión en tu aplicación.

Para obtener más información sobre cómo implementar Passwordless con Universal Login y Embedded login en distintos escenarios, consulta [Autenticación passwordless con Universal Login](/docs/es/authenticate/passwordless/implement-login/universal-login) o [Autenticación passwordless con Embedded Login](/docs/es/authenticate/passwordless/implement-login/embedded-login).

<div id="passwordless-endpoints">
  ## Endpoints de Passwordless
</div>

<div id="post-passwordlessstart">
  ### POST /passwordless/start
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Debes habilitar la concesión **Passwordless OTP** en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), en la configuración de tu aplicación, dentro de **Advanced Settings** &gt; **Grant Types**.
</Callout>

El endpoint [POST /passwordless/start](https://auth0.com/docs/api/authentication#get-code-or-link) puede invocarse para iniciar el proceso de autenticación Passwordless tanto en Classic Login como en Embedded Login.

Según los parámetros proporcionados al endpoint, Auth0 inicia el proceso de verificación del usuario enviando una de las siguientes opciones:

* Un código de un solo uso por correo electrónico o SMS
* Un enlace de un solo uso por correo electrónico

La llamada a la API debe tener la siguiente estructura:

export const codeExample1 = `POST https://{yourDomain}/passwordless/start
Content-Type: application/json
{
  "client_id": "{yourClientID}",
  "client_secret": "{yourClientSecret}", // Para aplicación web clásica
  "connection": "email|sms",
  "email": "{email}", // establecer para connection=email
  "phone_number": "{phoneNumber}", // establecer para connection=sms
  "send": "link|code", // si se deja como null, el valor predeterminado es link
  "authParams": { // cualquier parámetro de autenticación que desees agregar
    "scope": "openid",     // se usa al solicitar un Magic Link
    "state": "{yourState}"  // se usa al solicitar un Magic Link o desde la página de inicio de sesión personalizada
  }
}`;

<AuthCodeBlock children={codeExample1} language="json" />

Si usas un enlace mágico, los usuarios recibirán un enlace generado por la Authentication API. Los usuarios seleccionarán el enlace, lo que activará una llamada a &#123;`yourAuth0Tenant}.auth0.com/passwordless/verify-redirect`. Auth0 redirigirá al usuario a la aplicación y el usuario iniciará sesión.

Si usas un código, tu aplicación tendrá que solicitarlo y luego deberás usar el endpoint `/oauth/token` o el método `passwordlessLogin` del SDK de Auth0.js para intercambiar ese código por tokens de autenticación.

<div id="post-oauthtoken">
  ### POST /oauth/token
</div>

Si estás implementando passwordless para Native Applications o aplicaciones web clásicas, debes usar `/oauth/token` para intercambiar el código OTP por tokens de autenticación. No puedes usar este endpoint desde aplicaciones de página única.

Para ello, primero debes habilitar la concesión **Passwordless OTP** para tu aplicación en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), dentro de la configuración de tu aplicación, en **Advanced Settings** &gt; **Grant Types**.

El usuario recibirá el código OTP y tu aplicación Native o Web le mostrará un prompt para que lo introduzca. Cuando el usuario introduzca el código, puedes completar el flujo de autenticación llamando al endpoint `/oauth/token` con los siguientes parámetros:

export const codeExample2 = `POST https://{yourAuth0Domain}/oauth/token
Content-Type: application/json
{
  "grant_type" : "http://auth0.com/oauth/grant-type/passwordless/otp",
  "client_id": "{yourAuth0ClientID}",
  "client_secret": "{yourClientSecret}", // solo para aplicaciones web; las aplicaciones nativas no tienen secreto de cliente
  "username":"<dirección de correo electrónico>", // o "<número de teléfono>"
  "otp": "CODE",
  "realm": "email", // o "sms" 
  "audience" : "your-api-audience", // por si necesitas un token de acceso para una API específica
  "scope": "openid profile email" // los scopes que necesites
}`;

<AuthCodeBlock children={codeExample2} language="json" />

Si todo va bien, Auth0 devolverá una respuesta similar a la siguiente:

```json lines
HTTP/1.1 200 OK
Content-Type: application/json
{
"access_token":"eyJz93a...k4laUWw",
"refresh_token":"GEbRxBN...edjnXbL",
"id_token":"eyJ0XAi...4faeEoQ",
"token_type":"Bearer",
"expires_in":86400
}
```

Puedes decodificar el <Tooltip tip="Token de ID: credencial destinada al cliente en sí, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">Token de ID</Tooltip> para obtener información sobre el usuario, o usar el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> para llamar a tu API con normalidad.

<div id="using-auth0js">
  ## Uso de Auth0.js
</div>

Al implementar la autenticación passwordless en aplicaciones de página única o en una página personalizada de Universal Login, debes [usar Auth0.js y el método `passwordlessLogin` incluido](/docs/es/libraries/auth0js). La implementación es compleja, por lo que te recomendamos usar la biblioteca en lugar de llamar directamente a las API.

<div id="rate-limiting-in-passwordless-endpoints">
  ## Límites de frecuencia en endpoints passwordless
</div>

Los límites de frecuencia de Auth0 y las funciones de <Tooltip tip="Protección contra ataques: Funciones que Auth0 ofrece para detectar y mitigar ataques, incluida la protección contra ataques de fuerza bruta, Suspicious IP Throttling, la detección de contraseñas vulneradas, Bot Detection y la autenticación multifactor adaptativa." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip> solo tienen en cuenta la IP de la máquina que realiza la llamada a la API. Cuando la llamada a la API se hace desde un servidor backend, normalmente querrás que Auth0 tenga en cuenta la IP del usuario final, no la del servidor.

Auth0 permite especificar un encabezado `auth0-forwarded-for` en las llamadas a la API, pero solo se tiene en cuenta cuando:

* la llamada a la API se realiza para una aplicación confidencial.
* la llamada a la API incluye el <Tooltip tip="Secreto de cliente: Secret que usa un cliente (aplicación) para autenticarse ante el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>.
* el interruptor **Confiar en el encabezado IP del <Tooltip tip="Endpoint de token: Endpoint del servidor de autorización que se usa para solicitar tokens mediante programación." cta="Ver glosario" href="/docs/es/glossary?term=Token+Endpoint">endpoint de token</Tooltip>** está activado.

Para obtener una explicación completa, lee [Evita problemas comunes con el flujo de contraseña del propietario del recurso y la protección contra ataques](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).

<div id="customize-mfa">
  ## Personaliza MFA
</div>

<ReleaseStageNotice feature="MFA personalizable con los flujos de contraseña del propietario del recurso, Embedded o token de actualización" stage="ea" terms="true" contact="equipo de soporte de Auth0" />

Personaliza <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> con la API de Passwordless. Cuando tu aplicación llama al endpoint `/oauth/token` para solicitar un token de acceso, el <Tooltip tip="Servidor de autorización: servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip> devuelve un error `mfa_required` que proporciona:

* El `mfa_token` que necesitas para llamar a la API de MFA para el registro y los desafíos.

* El parámetro `mfa_requirements`, que proporciona el `type` del factor que tu aplicación admite para los desafíos.

```json lines
{
  "error": "mfa_required",
  "error_description": "Multifactor authentication required",
  "mfa_token": "Fe26...Ha",
  "mfa_requirements": {
    "challenge": [
      { "type": "otp" },
      { "type": "push-notification" },
      { "type": "phone" },
      { "type": "recovery-code" }
    ]
  }
}
```

Usa el `mfa_token` para llamar al endpoint [`mfa/authenticator`](https://auth0.com/docs/api/authentication/muti-factor-authentication/list-authenticators), listar todos los factores que el usuario ha registrado y hacer coincidir el mismo `type` que admite tu aplicación. También necesitas obtener el `authenticator_type` correspondiente para emitir desafíos:

```json lines
[
  {
    "type": "recovery-code",
    "id": "recovery-code|dev_qpOkGUOxBpw6R16t",
    "authenticator_type": "recovery-code",
    "active": true
  },
  {
    "type": "otp",
    "id": "totp|dev_6NWz8awwC8brh2dN",
    "authenticator_type": "otp",
    "active": true
  }
]
```

Impón el desafío MFA llamando al endpoint [`request/mfa/challenge`](https://auth0.com/docs/api/authentication/muti-factor-authentication/request-mfa-challenge).

Personaliza aún más tu flujo de MFA con Auth0 Actions. Para obtener más información, consulta [Actions Triggers: post-challenge - objeto de API](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object).
