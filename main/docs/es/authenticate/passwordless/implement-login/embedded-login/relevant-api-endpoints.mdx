<Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> APIs pueden usarse en dos escenarios:

* Al implementar <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> y quieres personalizar la página de inicio de sesión con auth0.js para interactuar con Auth0.
* Cuando quieres integrar el flujo de inicio de sesión en tu aplicación.

Para obtener más información sobre cómo implementar Passwordless para Universal Login y Embedded Login en distintos escenarios, consulta [Autenticación Passwordless con Universal Login](/docs/es/authenticate/passwordless/implement-login/universal-login) o [Autenticación Passwordless con Embedded Login](/docs/es/authenticate/passwordless/implement-login/embedded-login).

<div id="passwordless-endpoints">
  ## Endpoints de Passwordless
</div>

<div id="post-passwordlessstart">
  ### POST /passwordless/start
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Debes habilitar la concesión **Passwordless OTP** en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), en la configuración de tu aplicación, dentro de **Advanced Settings** &gt; **Grant Types**.
</Callout>

Se puede llamar al endpoint [POST /passwordless/start](https://auth0.com/docs/api/authentication#get-code-or-link) para iniciar el proceso de autenticación Passwordless tanto en Classic Login como en Embedded Login.

Según los parámetros proporcionados al endpoint, Auth0 inicia el proceso de verificación del usuario enviando una de las siguientes opciones:

* Un código de un solo uso por correo electrónico o SMS
* Un enlace de un solo uso por correo electrónico

La llamada a la API debe tener la siguiente estructura:

export const codeExample1 = `POST https://{yourDomain}/passwordless/start
Content-Type: application/json
{
  "client_id": "{yourClientID}",
  "client_secret": "{yourClientSecret}", // Para aplicación web clásica
  "connection": "email|sms",
  "email": "{email}", //configúralo para connection=email
  "phone_number": "{phoneNumber}", //configúralo para connection=sms
  "send": "link|code", //si se deja como null, el valor predeterminado es link
  "authParams": { // cualquier parámetro de autenticación que quieras añadir
    "scope": "openid",     // se usa al solicitar un Magic Link
    "state": "{yourState}"  // se usa al solicitar un Magic Link o desde la página de inicio de sesión personalizada
  }
}`;

<AuthCodeBlock children={codeExample1} language="json" />

Si utiliza un Magic Link, los usuarios recibirán un enlace generado por la Authentication API. Los usuarios harán clic en el enlace, lo que activará una llamada a &#123;`yourAuth0Tenant}.auth0.com/passwordless/verify-redirect`. Auth0 redirigirá al usuario a la aplicación y este iniciará sesión.

Si utiliza un código, su aplicación deberá solicitar ese código y, a continuación, debería usar el endpoint `/oauth/token` o el método `passwordlessLogin` del SDK de auth0.js para intercambiar ese código por tokens de autenticación.

<div id="post-oauthtoken">
  ### POST /oauth/token
</div>

Si está implementando passwordless para Native Applications o aplicaciones web clásicas, debe usar `/oauth/token` para intercambiar el código OTP por tokens de autenticación. No puede usar este endpoint desde aplicaciones web de página única.

Para ello, primero debe habilitar la concesión **Passwordless OTP** para su aplicación en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), dentro de la configuración de su aplicación, en **Advanced Settings** &gt; **Grant Types**.

El usuario recibirá el código OTP y su aplicación nativa o web le pedirá que lo introduzca. Cuando el usuario ingrese el código, podrá completar el flujo de autenticación llamando al endpoint `/oauth/token` con los siguientes parámetros:

export const codeExample2 = `POST https://{yourAuth0Domain}/oauth/token
Content-Type: application/json
{
  "grant_type" : "http://auth0.com/oauth/grant-type/passwordless/otp",
  "client_id": "{yourAuth0ClientID}",
  "client_secret": "{yourClientSecret}", // solo para aplicaciones web; las aplicaciones nativas no tienen secreto de cliente
  "username":"<correo electrónico>", // o "<número de teléfono>"
  "otp": "CÓDIGO",
  "realm": "email", // o "sms" 
  "audience" : "your-api-audience", // si necesitas un token de acceso para una API específica
  "scope": "openid profile email" // los scopes que necesites
}`;

<AuthCodeBlock children={codeExample2} language="json" />

Si todo sale bien, Auth0 devolverá una respuesta similar a la siguiente:

```json lines
HTTP/1.1 200 OK
Content-Type: application/json
{
"access_token":"eyJz93a...k4laUWw",
"refresh_token":"GEbRxBN...edjnXbL",
"id_token":"eyJ0XAi...4faeEoQ",
"token_type":"Bearer",
"expires_in":86400
}
```

Luego, puedes decodificar el <Tooltip tip="Token de ID: credencial destinada al propio cliente, no para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> para obtener información sobre el usuario, o usar el <Tooltip tip="Token de acceso: credencial de autorización en forma de cadena opaca o JWT que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> para llamar a tu API con normalidad.

<div id="using-auth0js">
  ## Uso de Auth0.js
</div>

Al implementar la autenticación Passwordless en aplicaciones web de página única o en una página personalizada de Universal Login, debes [usar Auth0.js y el método passwordlessLogin incluido](/docs/es/libraries/auth0js). La implementación es compleja, por lo que te recomendamos usar la biblioteca en lugar de llamar a las API directamente.

<div id="rate-limiting-in-passwordless-endpoints">
  ## Límites de frecuencia en endpoints passwordless
</div>

Los límites de frecuencia de Auth0 y las funcionalidades de <Tooltip tip="Protección contra ataques: Funcionalidades que Auth0 proporciona para detectar y mitigar ataques, incluida la protección contra ataques de fuerza bruta, la limitación de IP sospechosas, la detección de contraseñas vulneradas, la detección de bots y la autenticación multifactor adaptativa." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip> solo tienen en cuenta la IP de la máquina que hace la llamada a la API. Cuando la llamada a la API se realiza desde un servidor backend, normalmente querrás que Auth0 tenga en cuenta la IP del usuario final, no la del servidor.

Auth0 permite especificar un encabezado `auth0-forwarded-for` en las llamadas a la API, pero solo se tiene en cuenta cuando:

* la llamada a la API se hace para una aplicación confidencial.
* la llamada a la API incluye el <Tooltip tip="Secreto de cliente: Secreto utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>.
* el interruptor **Confiar en el encabezado IP del <Tooltip tip="Endpoint de token: Endpoint en el servidor de autorización que se utiliza para solicitar tokens mediante programación." cta="Ver glosario" href="/docs/es/glossary?term=Token+Endpoint">endpoint de token</Tooltip>** está activado.

Para obtener una explicación completa, lee [Evita problemas comunes con el flujo de contraseña del propietario del recurso y la protección contra ataques](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).

<div id="customize-mfa">
  ## Personaliza MFA
</div>

<ReleaseStageNotice feature="MFA personalizable con los flujos de contraseña del propietario del recurso, Embedded o token de actualización" stage="ea" terms="true" contact="equipo de soporte de Auth0" />

Personaliza <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor adicional además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> con la API de Passwordless. Cuando tu aplicación llama al endpoint `/oauth/token` para solicitar un token de acceso, el <Tooltip tip="Servidor de autorización: Servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip> devuelve un error `mfa_required`, que proporciona:

* El `mfa_token` que necesitas para llamar a la API de MFA para el registro y los desafíos.

* El parámetro `mfa_requirements`, que indica el `type` de factor que tu aplicación admite para los desafíos.

```json lines
{
  "error": "mfa_required",
  "error_description": "Multifactor authentication required",
  "mfa_token": "Fe26...Ha",
  "mfa_requirements": {
    "challenge": [
      { "type": "otp" },
      { "type": "push-notification" },
      { "type": "phone" },
      { "type": "recovery-code" }
    ]
  }
}
```

Usa el `mfa_token` para llamar al endpoint [`mfa/authenticator`](https://auth0.com/docs/api/authentication/muti-factor-authentication/list-authenticators), listar todos los factores en los que el usuario se ha inscrito e identificar el mismo `type` que admite tu aplicación. También debes obtener el `authenticator_type` correspondiente para emitir desafíos:

```json lines
[
  {
    "type": "recovery-code",
    "id": "recovery-code|dev_qpOkGUOxBpw6R16t",
    "authenticator_type": "recovery-code",
    "active": true
  },
  {
    "type": "otp",
    "id": "totp|dev_6NWz8awwC8brh2dN",
    "authenticator_type": "otp",
    "active": true
  }
]
```

Exige el desafío MFA llamando al endpoint [`request/mfa/challenge`](https://auth0.com/docs/api/authentication/muti-factor-authentication/request-mfa-challenge).

Personaliza aún más tu flujo MFA con Auth0 Actions. Para obtener más información, consulta [Actions Triggers: post-challenge - objeto de API](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object).
