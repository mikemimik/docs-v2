La experiencia de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> admite [conexiones passwordless](/docs/es/authenticate/passwordless), que permiten a los usuarios proporcionar un número de teléfono o una dirección de correo electrónico y luego recibir una contraseña de un solo uso (OTP) para completar la autenticación.

<div id="enable-passwordless-for-universal-login">
  ## Habilita Passwordless en Universal Login
</div>

Para configurar Universal Login para que admita conexiones passwordless de <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>, debes establecer el Authentication Profile para que use un flujo de login compatible y actualizar tu aplicación para especificar la conexión durante la autenticación.

<div id="enable-identifier-first">
  ### Habilitar Identifier First
</div>

En el <Tooltip tip="Auth0 Dashboard: la interfaz principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, ve a [Authentication &gt; Authentication Profile.](https://manage.auth0.com/?/authentication-profiles)

Selecciona **Identifier First** o **Identifier First + Biometrics**.

<div id="update-your-application">
  ### Actualiza tu aplicación
</div>

Según el tipo de conexiones que tengas habilitadas en tu aplicación, puede que necesites actualizarla para especificar una conexión Passwordless durante el inicio de sesión:

| conexión de base de datos | conexión Passwordless      | ¿Es necesario especificarla? | Experiencia de usuario                                                                                            |
| ------------------------- | -------------------------- | ---------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Sí                        | SMS y/o correo electrónico | Sí                           | La conexión Passwordless especificada se muestra durante el inicio de sesión.                                     |
| No                        | SMS y correo electrónico   | No                           | Se muestra la conexión Passwordless que se creó primero.                                                          |
| No                        | SMS o correo electrónico   | No                           | Se muestra la conexión Passwordless durante el inicio de sesión (junto con cualquier conexión social habilitada). |

Para especificar una conexión Passwordless durante el inicio de sesión, debes pasar el parámetro `connection` y su valor (`sms` o `email`) al [Login endpoint de Auth0 Authentication API](https://auth0.com/docs/api/authentication#login):

`https://mytenant.us.auth0.com/authorize?client_id={id}&``connection={sms|email}``&scope=…&response_type=code&response_mode=query&state=…&redirect_uri=http%3A%2F%2Flocalhost%3A3000&code_challenge=…`

Si usas uno de nuestros SDKs, puedes especificar una conexión Passwordless durante la inicialización. Por ejemplo:

```js lines
auth0 = await createAuth0Client({
     domain: config.domain,
     client_id: config.clientId,
     connection: "email"
  });

auth0 = await createAuth0Client({
     domain: config.domain,
     client_id: config.clientId,
     connection: "sms"
  });
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si utiliza Passwordless con Universal Login y [inicio de sesión único](/docs/es/authenticate/single-sign-on), los parámetros de conexión `sms` y `email` no utilizan la sesión existente de Auth0, por lo que se le pedirá al usuario que inicie sesión.
</Callout>
