Puedes configurar una <Tooltip tip="Passwordless: forma de autenticación que no utiliza una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">conexión passwordless</Tooltip> para enviar un Magic Link a un usuario por correo electrónico y completar la autenticación.

Para obtener más información, consulta [Configurar Email o SMS para la autenticación passwordless](/docs/es/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un usuario abre un Magic Link, Auth0 inicia su sesión directamente. Es similar a que reciba un correo electrónico con una contraseña de un solo uso (OTP), vuelva a tu aplicación e introduzca el OTP, pero sin tener que realizar esos pasos.

<div id="limitations">
  ### Limitaciones
</div>

Magic Links no son compatibles con <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>.

Al usar Magic Links con Classic Login, tanto la solicitud inicial como su respuesta deben realizarse en el mismo navegador; de lo contrario, la transacción fallará. Esto es especialmente relevante para los usuarios de iOS, que no pueden cambiar su navegador web predeterminado.

Por ejemplo, el usuario podría hacer la solicitud inicial desde el navegador Chrome, pero cuando abre el Magic Link en su correo electrónico, iOS lo abre automáticamente en Safari (el navegador predeterminado). Si esto sucede, la transacción fallará.

<div id="configure-the-connection">
  ## Configurar la conexión
</div>

1. En el Auth0 Dashboard, ve a [Authentication &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless) y activa la opción **Email**.
2. Selecciona **Email** para abrir la ventana de configuración, cambia a la vista **Settings** e introduce el texto de **From**, **Subject** y **Message** del correo electrónico.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Debes cambiar el valor de **From** por una dirección de correo electrónico que no use el dominio `auth0.com` para que Auth0 envíe tus plantillas de correo electrónico personalizadas. Si no lo haces, Auth0 enviará la plantilla de correo electrónico predeterminada.
   </Callout>
3. Introduce los **Parámetros de autenticación** que quieras incluir en el enlace de inicio de sesión generado.
4. Decide si quieres **Disable Sign Ups**. Si habilitas esta configuración, puedes permitir el acceso passwordless solo a usuarios existentes, pero podrías exponer tu aplicación al riesgo de ataques de enumeración de usuarios. Para obtener más información, consulta [Prácticas recomendadas de las conexiones passwordless](/docs/es/authenticate/passwordless/best-practices).
5. Selecciona **Save**.

<div id="multi-language-support">
  ### Compatibilidad con varios idiomas
</div>

El área de **Message** admite varios idiomas.

Para especificar un idioma, llama al [endpoint Get Code or Link de la Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) y establece el valor de la cabecera `x-request-language`. Si no se define esta cabecera, el idioma se extrae de la cabecera `accept-language`, que el navegador establece automáticamente.

<div id="message-syntax">
  ### Sintaxis de Message
</div>

El área de **Message** admite sintaxis Liquid. Puedes usar esta sintaxis, combinada con valores de parámetros, para construir mediante programación elementos del mensaje.

Por ejemplo, puedes hacer referencia al parámetro `request_language` para cambiar el idioma del mensaje:

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ code }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ code }}
{% endif %}
```

Los siguientes parámetros están disponibles al definir la plantilla del mensaje:

| Parámetro          | Descripción                                                                                                                                                                                 |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `code`             | La contraseña que se debe usar.                                                                                                                                                             |
| `link`             | El enlace de inicio de sesión generado.                                                                                                                                                     |
| `application.name` | El nombre de la aplicación con la que el usuario se está registrando.                                                                                                                       |
| `request_language` | El idioma solicitado para el contenido del mensaje.                                                                                                                                         |
| `operation`        | Indica si la plantilla se activó a raíz de una actualización del correo electrónico de un usuario a través de la API. Si se activa, el valor es `change_email`; de lo contrario, es `null`. |

<div id="update-classic-login-settings">
  ## Actualizar la configuración de Classic Login
</div>

Si usas Classic Login para tu aplicación, debes actualizar la plantilla de página para implementar Magic Links.

1. En el Auth0 Dashboard, ve a [**Branding &gt; Universal Login &gt; Advanced Options**](https://manage.auth0.com/#/login_settings).

2. Cambia a la vista **Login**, busca el menú desplegable **Default Templates** y selecciona **Lock (passwordless).**

3. Luego, actualiza la sección de código `Auth0LockPasswordless` con un parámetro nuevo. Agrega `passwordlessMethod` a tu plantilla.

   ```bash lines
   passwordlessMethod: 'link',
   ```

4. Selecciona **Save**.

Para obtener más información sobre las opciones de configuración de embedded login, consulta [Autenticación passwordless con embedded login](/docs/es/authenticate/passwordless/implement-login/embedded-login).

<div id="enable-applications">
  ## Habilitar aplicaciones
</div>

Ve a la vista de **Applications** y habilita las aplicaciones con las que quieras usar Passwordless Email.

<div id="email-providers">
  ## Proveedores de correo electrónico
</div>

Auth0 envía correos electrónicos desde su propio proveedor SMTP de forma predeterminada. El proveedor de correo electrónico integrado de Auth0 está diseñado únicamente para pruebas y no admite la personalización de las plantillas de correo electrónico.

[Configure su propio proveedor SMTP de correo electrónico](/docs/es/customize/email/smtp-email-providers) para gestionar, supervisar y solucionar problemas de sus comunicaciones por correo electrónico, así como personalizar las plantillas de correo electrónico.

Auth0 admite los siguientes proveedores de correo electrónico:

* [Mandrill](/docs/es/customize/email/smtp-email-providers/configure-mandrill-as-external-smtp-email-provider)
* [Amazon SES](/docs/es/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider)
* [SendGrid](/docs/es/customize/email/smtp-email-providers/configure-sendgrid-as-external-smtp-email-provider)
* [SparkPost](/docs/es/customize/email/smtp-email-providers/configure-sparkpost-as-external-smtp-email-provider)
* [Mailgun](/docs/es/customize/email/smtp-email-providers/configure-mailgun-as-external-smtp-email-provider)
* [Proveedor externo SMTP de correo electrónico personalizado](/docs/es/customize/email/smtp-email-providers/configure-custom-external-smtp-email-provider)
