Puedes configurar una <Tooltip tip="Passwordless: Forma de autenticación que no utiliza una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> conexión para enviarle un Magic Link a un usuario por correo electrónico y completar la autenticación.

Para obtener más información, consulta [Configurar Email o SMS para la autenticación passwordless](/docs/es/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un usuario abre un Magic Link, Auth0 inicia su sesión directamente. Es similar a recibir un correo electrónico con una contraseña de un solo uso (OTP), volver a tu aplicación e introducir el OTP, pero sin tener que realizar realmente esos pasos.

<div id="limitations">
  ### Limitaciones
</div>

Magic Links no es compatible con <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>.

Al usar Magic Links con Classic Login, tanto la solicitud inicial como la respuesta deben producirse en el mismo navegador; de lo contrario, la transacción fallará. Esto es especialmente relevante para los usuarios de iOS, que no pueden cambiar su navegador web predeterminado.

Por ejemplo, el usuario puede hacer la solicitud inicial desde Chrome, pero cuando abre el Magic Link desde su correo electrónico, iOS lo abre automáticamente en Safari (el navegador predeterminado). Si esto sucede, la transacción fallará.

Luego, el usuario hace clic en el botón o en el enlace del correo electrónico e inicia sesión automáticamente en tu aplicación.

<div id="configure-the-connection">
  ## Configura la conexión
</div>

1. En el Auth0 Dashboard, ve a [Authentication &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless) y, a continuación, habilita el interruptor **Email**.
2. Selecciona **Email** para abrir la ventana de configuración, cambia a la vista **Settings** e introduce los valores de **From**, **Subject** y **Message** del correo electrónico.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Debes cambiar el valor de **From** por una dirección de correo electrónico que no use el dominio `auth0.com` para que Auth0 pueda enviar tus plantillas de correo electrónico personalizadas. Si no lo haces, Auth0 enviará la plantilla de correo electrónico predeterminada.
   </Callout>
3. Introduce los **Authentication Parameters** que quieras incluir en el enlace de inicio de sesión generado.
4. Decide si quieres **Disable Sign Ups**. Si habilitas esta configuración, puedes permitir acceso passwordless solo a usuarios existentes, pero podrías exponer tu aplicación al riesgo de ataques de enumeración de usuarios. Para obtener más información, consulta [Prácticas recomendadas para conexiones passwordless](/docs/es/authenticate/passwordless/best-practices).
5. Selecciona **Save**.

<div id="multi-language-support">
  ### Soporte para varios idiomas
</div>

El área de **Message** admite varios idiomas.

Para especificar un idioma, llama al [endpoint Get Code or Link de Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) y establece el valor del encabezado `x-request-language`. Cuando no se establece este encabezado, el idioma se extrae del encabezado `accept-language`, que el navegador configura automáticamente.

<div id="message-syntax">
  ### Sintaxis de Message
</div>

El área de **Message** acepta sintaxis Liquid. Puedes usar esta sintaxis, combinada con valores de parámetros, para generar de forma programática elementos del mensaje.

Por ejemplo, puedes hacer referencia al parámetro `request_language` para cambiar el idioma del mensaje:

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ code }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ code }}
{% endif %}
```

Los siguientes parámetros están disponibles al definir la plantilla del mensaje:

| Parameter          | Description                                                                                                                                                                                    |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `code`             | La contraseña que se debe usar.                                                                                                                                                                |
| `link`             | El enlace de inicio de sesión generado.                                                                                                                                                        |
| `application.name` | El nombre de la aplicación con la que el usuario se registra.                                                                                                                                  |
| `request_language` | El idioma solicitado para el contenido del mensaje.                                                                                                                                            |
| `operation`        | Indica si la plantilla se ha activado por una actualización del correo electrónico de un usuario a través de la API. Cuando se activa, el valor es `change_email`; de lo contrario, es `null`. |

<div id="update-classic-login-settings">
  ## Actualizar la configuración de Classic Login
</div>

Si usas Classic Login para tu aplicación, debes actualizar la plantilla de página para implementar Magic Links.

1. En el Auth0 Dashboard, ve a [**Branding &gt; Universal Login &gt; Advanced Options**](https://manage.auth0.com/#/login_settings).

2. Cambia a la vista **Login**, busca el menú desplegable **Default Templates** y selecciona **Lock (passwordless).**

3. A continuación, actualiza la sección de código `Auth0LockPasswordless` con un parámetro nuevo. Agrega `passwordlessMethod` a tu plantilla.

   ```bash lines
   passwordlessMethod: 'link',
   ```

4. Selecciona **Save**.

Para obtener más información sobre las opciones de configuración de Embedded Login, consulta [Autenticación passwordless con Embedded Login](/docs/es/authenticate/passwordless/implement-login/embedded-login).

<div id="enable-applications">
  ## Habilitar aplicaciones
</div>

Cambia a la vista de **Aplicaciones** y habilita las aplicaciones para las que quieras usar Passwordless Email.

<div id="email-providers">
  ## Proveedores de correo electrónico
</div>

Auth0 envía correos electrónicos a través de su propio proveedor SMTP de forma predeterminada. El proveedor de correo electrónico integrado de Auth0 está diseñado únicamente para pruebas y no admite la personalización de las plantillas de correo electrónico.

[Configure su propio proveedor SMTP de correo electrónico](/docs/es/customize/email/smtp-email-providers) para administrar, supervisar y solucionar problemas de sus comunicaciones por correo electrónico, así como personalizar las plantillas de correo electrónico.

Auth0 admite los siguientes proveedores de correo electrónico:

* [Mandrill](/docs/es/customize/email/smtp-email-providers/configure-mandrill-as-external-smtp-email-provider)
* [Amazon SES](/docs/es/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider)
* [SendGrid](/docs/es/customize/email/smtp-email-providers/configure-sendgrid-as-external-smtp-email-provider)
* [SparkPost](/docs/es/customize/email/smtp-email-providers/configure-sparkpost-as-external-smtp-email-provider)
* [Mailgun](/docs/es/customize/email/smtp-email-providers/configure-mailgun-as-external-smtp-email-provider)
* [Proveedor externo de correo electrónico SMTP personalizado](/docs/es/customize/email/smtp-email-providers/configure-custom-external-smtp-email-provider)
