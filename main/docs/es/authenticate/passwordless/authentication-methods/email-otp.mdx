Puede configurar una <Tooltip tip="Passwordless: forma de autenticación que no usa una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> conexión para enviar una contraseña de un solo uso (OTP) a un usuario por correo electrónico y completar la autenticación.

Para obtener más información, lea [Configurar Email o SMS para la autenticación Passwordless](/docs/es/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un usuario nuevo recibe un OTP y lo introduce por primera vez en tu sitio web, su perfil de usuario se crea en la conexión `email` antes de que Auth0 lo autentique.

Si la dirección de correo electrónico a la que se envió el OTP coincide con la de un usuario existente, Auth0 autentica al usuario:

<div id="configure-the-connection">
  ## Configura la conexión
</div>

1. Ve a [Auth0 Dashboard &gt; Autenticación &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless) y activa el interruptor **Email**.
2. Selecciona **Email** para abrir la ventana de configuración, cambia a la vista **Settings** y, a continuación, introduce el texto de **From**, **Subject** y **Message** de tu correo electrónico.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Debes cambiar el valor de **From** por una dirección de correo electrónico que no use el dominio `auth0.com` para que Auth0 envíe tus plantillas de correo electrónico personalizadas. Si no lo haces, Auth0 enviará la plantilla de correo electrónico predeterminada.
   </Callout>
3. Configura **OTP Expiry** y **OTP Length.**

   * Solo se aceptará el último OTP (o enlace) emitido. En cuanto se emita el más reciente, los demás quedarán invalidados. Una vez utilizado, el más reciente también se invalidará.
   * Solo se permiten tres intentos fallidos de introducir el OTP. Después de eso, tendrás que solicitar un código nuevo.
   * El OTP emitido será válido (de forma predeterminada) durante tres minutos antes de caducar.
   * Si decides ampliar el tiempo de validez del OTP, también deberías aumentar su longitud. De lo contrario, un atacante tendrá una ventana de tiempo mayor para intentar adivinar un código corto.
4. Decide si quieres **Disable Sign Ups**. Si habilitas esta configuración, puedes permitir acceso passwordless solo a los usuarios existentes, pero esto puede exponer tu aplicación al riesgo de ataques de enumeración de usuarios. Para obtener más información, consulta [Prácticas recomendadas para conexiones passwordless](/docs/es/authenticate/passwordless/best-practices).
5. Selecciona **Save**.

<div id="multi-language-support">
  ### Compatibilidad con varios idiomas
</div>

El área de **Message** admite varios idiomas.

Para especificar un idioma, llame al [endpoint Get Code or Link de la Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) y establezca el valor del encabezado `x-request-language`. Si no se establece este encabezado, el idioma se extrae del encabezado `accept-language`, que el navegador configura automáticamente.

<div id="message-syntax">
  ### Sintaxis de Message
</div>

El área de **Message** admite sintaxis Liquid. Puedes usar esta sintaxis, combinada con valores de parámetros, para construir mediante programación elementos del mensaje.

Por ejemplo, puedes hacer referencia al parámetro `request_language` para cambiar el idioma del mensaje:

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ code }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ code }}
{% endif %}
```

Los siguientes parámetros están disponibles al definir la plantilla del mensaje:

| Parameter          | Description                                                                                                                                                                               |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `code`             | La contraseña que se va a usar.                                                                                                                                                           |
| `link`             | El enlace de inicio de sesión generado.                                                                                                                                                   |
| `application.name` | El nombre de la aplicación con la que el usuario se está registrando.                                                                                                                     |
| `request_language` | El idioma solicitado para el contenido del mensaje.                                                                                                                                       |
| `operation`        | Indica si la plantilla se activó por una actualización del correo electrónico de un usuario a través de la API. Cuando se activa, el valor es `change_email`; de lo contrario, es `null`. |

Si el usuario ha iniciado sesión a través de una [organización](/docs/es/manage-users/organizations/organizations-overview), los siguientes parámetros también están disponibles:

* `organization.id`
* `organization.display_name`
* `organization.name`
* `organization.metadata`
* `organization.branding.logo_url`
* `organization.branding.colors.primary`
* `organization.branding.colors.page_background`

Para obtener más información sobre estos parámetros y sus valores, consulta el endpoint [Get organization](https://auth0.com/docs/api/management/v2#!/Organizations/get_organizations_by_id) de Auth0 <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Cambia a la vista de **Applications** y habilita las aplicaciones para las que quieras usar Passwordless Email.

<div id="email-providers">
  ## Proveedores de correo electrónico
</div>

Auth0 envía correos electrónicos desde su propio proveedor SMTP de forma predeterminada. El proveedor de correo electrónico integrado de Auth0 está diseñado únicamente para pruebas y no permite personalizar las plantillas de correo electrónico.

[Configure su propio proveedor SMTP de correo electrónico](/docs/es/customize/email/smtp-email-providers) para administrar, supervisar y solucionar problemas relacionados con sus comunicaciones por correo electrónico, así como personalizar las plantillas de correo electrónico.

Auth0 admite los siguientes proveedores de correo electrónico:

* [Mandrill](/docs/es/customize/email/smtp-email-providers/configure-mandrill-as-external-smtp-email-provider)
* [Amazon SES](/docs/es/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider)
* [Azure Communication Services](/docs/es/customize/email/smtp-email-providers/configure-azure-comm-service-as-smtp-email-provider)
* [SendGrid](/docs/es/customize/email/smtp-email-providers/configure-sendgrid-as-external-smtp-email-provider)
* [SparkPost](/docs/es/customize/email/smtp-email-providers/configure-sparkpost-as-external-smtp-email-provider)
* [Mailgun](/docs/es/customize/email/smtp-email-providers/configure-mailgun-as-external-smtp-email-provider)
* [Proveedor SMTP externo personalizado de correo electrónico](/docs/es/customize/email/smtp-email-providers/configure-custom-external-smtp-email-provider)
