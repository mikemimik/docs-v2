<Warning>
  Esta guía corresponde a la implementación heredada de SMS personalizado para conexión passwordless y solo se aplica a tenants de Auth0 creados antes de marzo de 2025.

  Los tenants nuevos deben seguir las [instrucciones de passwordless authentication con proveedor telefónico SMS](/docs/es/authenticate/passwordless/authentication-methods/sms-otp#configure-the-phone-provider).
</Warning>

Esta guía te mostrará cómo usar un gateway de SMS personalizado para enviar tus códigos de un solo uso.

De forma predeterminada, las [conexión passwordless de Passwordless SMS](/docs/es/authenticate/passwordless/authentication-methods/sms-otp) usan [Twilio](https://www.twilio.com) para enviar códigos de un solo uso. Sin embargo, si tienes un gateway de SMS personalizado, puedes modificar tu conexión para usar ese en su lugar.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no admite la autenticación básica por SMS.
</Callout>

1. Configura una conexión passwordless por SMS. Para saber cómo hacerlo, consulta la sección Implement Passwordless en [Connections](/docs/es/authenticate/passwordless).
2. [Obtén un token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens). Lo necesitarás para hacer llamadas a la Management API y actualizar tu conexión passwordless.
3. Usa el endpoint [GET Connections](https://auth0.com/docs/api/management/v2#!/Connections/get_connections) para recuperar información sobre las conexiones asociadas a tu tenant. En concreto, necesitas obtener el ID de tu conexión de Passwordless SMS para poder usarlo en una llamada posterior a la API que actualiza la propia conexión.
   Asegúrate de reemplazar `ACCESS_TOKEN` por el token que obtuviste en el paso 1 antes de realizar la siguiente llamada a la Management API:

La respuesta del endpoint será un arreglo de objetos. Cada objeto representa una conexión asociada a tu tenant.
4. Identifica el ID de tu conexión. Puedes encontrar el ID asociado a tu conexión Passwordless revisando el arreglo de objetos que obtuviste del [endpoint GET Connections](https://auth0.com/docs/api/management/v2#!/Connections/get_connections) en el paso 2.
Para encontrar el objeto específico de tu conexión Passwordless, puedes buscar la propiedad `"name": "sms"`. Ten en cuenta que la conexión muestra actualmente la información de Twilio que proporcionaste durante el proceso de Setup.


5. Actualiza la conexión. Puedes hacerlo haciendo una llamada PATCH al [endpoint actualizar una conexión](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id). En concreto, actualizarás el objeto `options` de la conexión para proporcionar información sobre el gateway de SMS.

   <Warning>
     Debes enviar el objeto `options` completo en cada llamada; de lo contrario, sobrescribirás los datos existentes que no se incluyan en llamadas posteriores.
   </Warning>

   Realiza los siguientes cambios:

   * Elimina los parámetros `twilio_sid` y `twilio_token`
   * Agrega el parámetro `provider` y asígnale el valor `sms_gateway`
   * Agrega el parámetro `gateway_url` y asígnale la URL de tu gateway de SMS. Auth0 debe poder acceder a esta URL para usar tu gateway y enviar mensajes en tu nombre.

   Tu payload tendrá un aspecto similar a este:

   ```json lines
   {
       "options": {
         "strategy": "sms",
         "provider": "sms_gateway",
         "gateway_url": "{urlOfYourGateway}",
         "from": "+1 234 567",
         "template": "Your verification code is: @@password@@",
         "brute_force_protection": true,
         "forward_req_info": "true",
         "disable_signup": false,
         "name": "sms",
         "syntax": "md_with_macros",
         "totp": {
           "time_step": 300,
           "length": 6
         }
       },
       "is_domain_connection": false,
   }
   ```

<div id="authenticated-requests">
  ## Solicitudes autenticadas
</div>

Si tu gateway de SMS acepta solicitudes autenticadas mediante tokens, puedes agregar lo siguiente a tu objeto `options`:

```json lines
"gateway_authentication": {
    "method": "bearer",
    "subject": "urn:Auth0",
    "audience": "urn:MySmsGateway",
    "secret": "MySecretToSignTheToken",
    "secret_base64_encoded": false
}
```

Cuando incluyes `gateway_authentication` en tu objeto `options`, Auth0 agrega un [JSON Web Token](/docs/es/secure/tokens/json-web-tokens) al encabezado `Authorization` cada vez que envía solicitudes a tu gateway de SMS. El token contiene los valores `gateway_authentication.subject` y `gateway_authentication.audience`, y está firmado con `gateway_authentication.secret`.

Si tu secreto está codificado en base64url, establece `secret_base64_encoded` en `true`.

Una vez que hayas actualizado tu conexión, Auth0 enviará lo siguiente a tu gateway de SMS cada vez que un usuario se registre o inicie sesión con tu conexión <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>.

```json lines
{
  "recipient": "+1 399 999",
  "body": "Your verification code is: 12345",
  "sender": "+1 234 567"
}
```

Si estableces la propiedad `forward_req_info` en `true` dentro del objeto `options`, el gateway también recibirá información de la solicitud HTTP que inició el proceso de Passwordless. Esto incluye la dirección IP del cliente que llama a `/passwordless/start` y su User Agent.

```json lines
{
  "recipient": "+1 399 999",
  "body": "Your verification code is: 12345",
  "sender": "+1 234 567",
  "req" : { 
      "ip" : "167.56.227.117",
      "user-agent" : "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_3) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/72.0.3626.109 Safari/537.36"
       }
}
```

<div id="error-handling">
  ## Gestión de errores
</div>

Auth0 solo tendrá en cuenta el código HTTP que devuelva el gateway de SMS; ignorará el resto de la respuesta (p. ej., el cuerpo de la respuesta y el tipo de respuesta).

Si el gateway de SMS devuelve un código HTTP distinto de 200, el endpoint `/passwordless/start` devolverá un código HTTP 400 y una respuesta similar a la siguiente:

```json lines
{
 "error":"sms_provider_error",
 "error_description":"Unexpected response while calling the SMS gateway: <HTTP Code Returned by the SMS Gateway>"}
}
```

Si el gateway de SMS devuelve HTTP 401, el valor de `error_description` será **La autenticación falló al llamar al gateway de SMS: 401**. (Ten en cuenta que la redacción de la descripción del error puede cambiar en cualquier momento).

Auth0 establece un tiempo de espera de 30 segundos para las llamadas HTTP a gateways de SMS personalizados. Si el gateway de SMS no responde dentro de ese plazo, el endpoint `/passwordless/start` también devolverá un código HTTP 400. La respuesta tendrá el formato que se muestra arriba y el campo `error_description` será **Se agotó el tiempo de espera al llamar al gateway de SMS: &lt;Timeout Code&gt;**. (De nuevo, la redacción de la descripción del error puede cambiar en cualquier momento).
