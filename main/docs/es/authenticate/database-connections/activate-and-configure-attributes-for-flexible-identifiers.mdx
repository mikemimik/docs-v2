<Warning>
  Habilitar Identificadores flexibles en tu tenant puede introducir cambios incompatibles en tu entorno de producción. Para obtener más información sobre los posibles problemas y las limitaciones, consulta [Identificadores flexibles y atributos](/docs/es/authenticate/database-connections/flexible-identifiers-and-attributes).
</Warning>

Un atributo de las conexiones de base de datos, como el correo electrónico, el número de teléfono o el nombre de usuario, forma parte de los Identificadores flexibles. Los usuarios finales usan un atributo para registrarse, autenticarse y restablecer sus contraseñas. Cada atributo tiene una configuración única.

<div id="activate-attributes">
  ## Activar atributos
</div>

<Tabs>
  <Tab title="Nueva conexión" icon="database">
    1. Ve a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Selecciona **+Crear conexión de base de datos**.
    3. Introduce un nombre para la conexión y elige los atributos.
    4. Elige el método de autenticación de la conexión.
  </Tab>

  <Tab title="Conexión existente" icon="database">
    1. Ve a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. En la lista, selecciona la conexión de base de datos que quieres actualizar.
    3. Selecciona la pestaña **Atributos** y elige **Activar**.
    4. Cuando se te pida confirmar que has probado tu configuración en un entorno de desarrollo, selecciona **Continuar**.
    5. En la pantalla Atributos, selecciona **Agregar atributo** o **Configurar** un atributo existente.
  </Tab>
</Tabs>

A una conexión nueva se le asignan valores predeterminados, que pueden editarse después de crearla.

| Configuración                                                 | Valor            |
| ------------------------------------------------------------- | ---------------- |
| Usar `\{attribute}` como identificador                        | `True`           |
| Permitir registro con `\{attribute}`                          | Obligatorio      |
| Requerir `\{attribute}` en el perfil del usuario              | `True`           |
| Longitud del nombre de usuario                                | Mín.: 1 Máx.: 15 |
| Permitir nombres de usuario con formato de correo electrónico | `False`          |
| Permitir nombres de usuario con formato de número de teléfono | `False`          |

<div id="configure-attributes">
  ## Configurar atributos
</div>

Selecciona la pestaña **Atributos** en la conexión. Elige uno de los Atributos disponibles y selecciona **Configurar**. Cuando hayas actualizado la configuración de los atributos, selecciona **Guardar**.

<div id="configure-or-remove-existing-attributes-on-a-connection">
  ### Configurar o quitar atributos existentes en una conexión
</div>

Todos los atributos se pueden cambiar o eliminar desde tu <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Los usuarios no podrán iniciar sesión, registrarse ni restablecer contraseñas con el atributo eliminado, y en cada conexión debe permanecer activo al menos un atributo.

Para cambiar o quitar un atributo, ve a la pestaña **Atributos** y luego elige **Quitar**.

<div id="revert-a-connection">
  ### Revertir una conexión
</div>

Para revertir una conexión a su estado original, usa el endpoint <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> [actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) con un objeto `options` que no incluya el parámetro `attributes`, como se muestra a continuación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Si usas el parámetro `options`, se sobrescribirá todo el objeto `options`. Para evitar datos parciales u otros problemas, asegúrate de que estén presentes todos los parámetros actuales; para ello, copia el contenido del objeto `options` de la conexión e inclúyelo en la llamada PATCH. El único parámetro que debes eliminar es `attributes`.
  * Si tu tenant tiene `username` habilitado como atributo, para realizar la siguiente acción deberás modificar la configuración de `username`. Si eliges esta opción, tendrás que volver a configurar los ajustes de `username`.
</Callout>

```json lines
{
"options": {
       }
}
```
