<Warning>
  Habilitar Flexible Identifiers en tu tenant puede introducir cambios incompatibles en tu entorno de producción. Para obtener más información sobre los posibles problemas y las limitaciones, consulta [Flexible Identifiers and Attributes](/docs/es/authenticate/database-connections/flexible-identifiers-and-attributes).
</Warning>

Un atributo de las conexiones de base de datos, como el correo electrónico, el número de teléfono o el nombre de usuario, forma parte de Flexible Identifiers. Los usuarios finales se registran, se autentican y restablecen contraseñas con un atributo. Cada atributo tiene una configuración única.

<div id="activate-attributes">
  ## Activar atributos
</div>

<Tabs>
  <Tab title="Nueva conexión" icon="database">
    1. Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Seleccione **+Create DB Connection**.
    3. Introduzca un nombre para la conexión y elija los atributos.
    4. Elija el método de autenticación de la conexión.
  </Tab>

  <Tab title="Conexión existente" icon="database">
    1. Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Seleccione en la lista la conexión de base de datos que desea actualizar.
    3. Seleccione la pestaña **Atributos** y elija **Activar**.
    4. Cuando se le pida confirmar que ha probado la configuración en un entorno de desarrollo, seleccione **Continuar**.
    5. En la pantalla **Atributos**, seleccione **Agregar atributo** o **Configurar** un atributo existente.
  </Tab>
</Tabs>

A una conexión nueva se le asignan valores predeterminados, que pueden editarse después de crear la conexión.

| Configuración                                                 | Valor            |
| ------------------------------------------------------------- | ---------------- |
| Usar `\{attribute}` como identificador                        | `True`           |
| Permitir el registro con `\{attribute}`                       | Obligatorio      |
| Requerir `\{attribute}` en el perfil del usuario              | `True`           |
| Longitud del nombre de usuario                                | Mín.: 1 Máx.: 15 |
| Permitir nombres de usuario con formato de correo electrónico | `False`          |
| Permitir nombres de usuario con formato de número de teléfono | `False`          |

<div id="configure-attributes">
  ## Configurar atributos
</div>

Selecciona la pestaña **Atributos** en la conexión. Elige uno de los atributos disponibles y selecciona **Configurar**. Una vez que hayas actualizado la configuración de los atributos, selecciona **Guardar**.

<div id="configure-or-remove-existing-attributes-on-a-connection">
  ### Configura o elimina atributos existentes en una conexión
</div>

Todos los atributos pueden modificarse o eliminarse desde tu <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Los usuarios no podrán iniciar sesión, registrarse ni restablecer la contraseña con el atributo eliminado, y debe permanecer activo al menos un atributo en cada conexión.

Para cambiar o eliminar un atributo, ve a la pestaña **Atributos** y luego elige **Eliminar**.

<div id="revert-a-connection">
  ### Revertir una conexión
</div>

Para revertir una conexión a su estado original, usa el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> endpoint [actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) con un objeto `options` que no incluya el parámetro `attributes`, como se muestra a continuación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Si usas el parámetro `options`, se sobrescribirá todo el objeto `options`. Para evitar datos parciales u otros problemas, asegúrate de que todos los parámetros actuales estén incluidos; para ello, copia el contenido del objeto `options` de la conexión e inclúyelo en la llamada PATCH. El único parámetro que debes eliminar es `attributes`.
  * Si tu tenant tiene `username` habilitado como atributo, para realizar la siguiente acción tendrás que modificar la configuración de `username`. Si eliges esta opción, luego tendrás que volver a configurar `username`.
</Callout>

```json lines
{
"options": {
       }
}
```
