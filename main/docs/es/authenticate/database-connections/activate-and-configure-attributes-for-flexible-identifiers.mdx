<Warning>
  Habilitar Flexible Identifiers en tu tenant puede introducir cambios incompatibles en tu entorno de producción. Para obtener más información sobre los posibles problemas y limitaciones, lee [Identificadores flexibles y atributos](/docs/es/authenticate/database-connections/flexible-identifiers-and-attributes).
</Warning>

En las conexiones de base de datos, un atributo, como el correo electrónico, el número de teléfono o el nombre de usuario, forma parte de Flexible Identifiers. Los usuarios finales se registran, se autentican y restablecen sus contraseñas con un atributo. Cada atributo tiene una configuración única.

<div id="activate-attributes">
  ## Activar atributos
</div>

<Tabs>
  <Tab title="Nueva conexión" icon="database">
    1. Ve a [**Auth0 Dashboard &gt; Autenticación &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Selecciona **+Crear conexión de base de datos**.
    3. Introduce un nombre para la conexión y elige los atributos.
    4. Elige el método de autenticación de la conexión.
  </Tab>

  <Tab title="Conexión existente" icon="database">
    1. Ve a [**Auth0 Dashboard &gt; Autenticación &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Selecciona en la lista la conexión de base de datos que quieras actualizar.
    3. Selecciona la pestaña **Atributos** y elige **Activar**.
    4. Cuando se te pida confirmar que has probado la configuración en un entorno de desarrollo, selecciona **Continuar**.
    5. En la pantalla Atributos, selecciona **Añadir atributo** o **configurar** un atributo existente.
  </Tab>
</Tabs>

A una conexión nueva se le asignan valores predeterminados, que pueden editarse después de crearla.

| Configuración                                                 | Valor            |
| ------------------------------------------------------------- | ---------------- |
| Usar `\{attribute}` como identificador                        | `True`           |
| Permitir el registro con `\{attribute}`                       | Obligatorio      |
| Requerir `\{attribute}` en el perfil de usuario               | `True`           |
| Longitud del nombre de usuario                                | Mín.: 1 Máx.: 15 |
| Permitir nombres de usuario con formato de correo electrónico | `False`          |
| Permitir nombres de usuario con formato de número de teléfono | `False`          |

<div id="configure-attributes">
  ## Configurar atributos
</div>

Selecciona la pestaña **Atributos** en la conexión. Elige uno de los atributos disponibles y selecciona **Configurar**. Una vez que hayas actualizado la configuración de los atributos, selecciona **Guardar**.

<div id="configure-or-remove-existing-attributes-on-a-connection">
  ### Configure o elimine los atributos existentes de una conexión
</div>

Todos los atributos se pueden cambiar o eliminar desde su <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Los usuarios no pueden iniciar sesión, registrarse ni restablecer la contraseña con el atributo eliminado, y debe permanecer activo al menos un atributo en cada conexión.

Para cambiar o eliminar un atributo, vaya a la pestaña **Atributos** y, a continuación, seleccione **Eliminar**.

<div id="revert-a-connection">
  ### Revertir una conexión
</div>

Para revertir una conexión a su estado original, usa el endpoint [actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> con un objeto `options` que no incluya el parámetro `attributes`, como se muestra a continuación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Si usas el parámetro `options`, se sobrescribirá todo el objeto `options`. Para evitar datos parciales u otros problemas, asegúrate de incluir todos los parámetros actuales: copia el contenido del objeto `options` de la conexión e inclúyelo en la llamada PATCH. El único parámetro que debes eliminar es `attributes`.
  * Si tu tenant tiene `username` habilitado como atributo, para realizar la siguiente acción tendrás que modificar la configuración de `username`. Si eliges esta opción, tendrás que volver a configurar `username`.
</Callout>

```json lines
{
"options": {
       }
}
```
