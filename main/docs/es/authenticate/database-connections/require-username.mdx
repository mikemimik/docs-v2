title: Agregar un nombre de usuario para Database Connections
---

En las Database Connections, puedes hacer que tus usuarios inicien sesión con un nombre de usuario en lugar de su dirección de correo electrónico.

<div id="require-username">
  ## Requerir nombre de usuario
</div>

1. Ve a [Auth0 Dashboard &gt; Autenticación &gt; Database](https://manage.auth0.com/#/connections/database) y selecciona el nombre de la conexión que quieres editar.
2. Busca **Requires Username** y usa el toggle para activar o desactivar la obligatoriedad del nombre de usuario.

   <Frame>![Activar o desactivar la configuración para requerir nombre de usuario](/docs/images/cdy7uua7fh8z/1MlOrFmcSCOCRYKRxWz6xT/7a87dac16a31cbb6e608489c5843d6c7/2025-02-25_12-02-23.png)</Frame>

Para ver cómo afectará esto a la pantalla de inicio de sesión, selecciona la vista **Try Connection**. Ten en cuenta que, una vez activado **Requires Username**, los usuarios nuevos deberán introducir un nombre de usuario y su dirección de correo electrónico para completar el registro.

<Frame>![Formulario de inicio de sesión con nombre de usuario](/docs/images/cdy7uua7fh8z/AiB37UZqyBPb8h55PZtzm/73e6fd0033f63f865c40f189db1620ef/2025-01-29_15-22-11.png)</Frame>

Después, los usuarios podrán iniciar sesión con su nombre de usuario o con su dirección de correo electrónico. Los usuarios que se registraron mientras **Requires Username** estaba desactivado no tendrán almacenado el campo Username y deberán iniciar sesión con su dirección de correo electrónico.

<div id="username-length">
  ### Longitud del nombre de usuario
</div>

De forma predeterminada, los nombres de usuario pueden tener entre 1 y 15 caracteres. Sin embargo, con el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> v2, puede modificar la longitud mínima y máxima permitidas (hasta un máximo de 128 caracteres).

<div id="allowed-characters">
  ### Caracteres permitidos
</div>

El campo `username` admite los siguientes caracteres:

* Caracteres alfanuméricos (sin acentos, convertidos automáticamente a minúsculas);
* El carácter arroba (@) (pero no se permiten direcciones de correo electrónico);
* El carácter acento circunflejo (^);
* El carácter signo de dólar ($);
* El carácter punto (.);
* El carácter signo de exclamación (!);
* El carácter acento grave (&#96;);
* El carácter guion (-);
* El carácter almohadilla (#);
* El carácter signo más (+);
* El carácter comilla simple (&#39;);
* El carácter tilde (~);
* El carácter guion bajo (&#95;);

No se permite ningún otro carácter o símbolo, y Auth0 no valida ni sanea las entradas de la base de datos personalizada.
