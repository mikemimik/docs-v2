<ReleaseStageNotice feature="Política flexible de contraseñas" stage="ea" terms="true" />

La Política flexible de contraseñas reemplaza las [políticas de contraseñas heredadas](/docs/es/authenticate/database-connections/password-options) para ofrecer mayor granularidad y más opciones de configuración.

| Política                   | Comportamiento Legacy                                                                                                               | Comportamiento de la Política flexible de contraseñas                                                                                                                                  |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Fortaleza de la contraseña | Elija entre cinco niveles predefinidos de complejidad.<br /><br />Trunca sin avisar las contraseñas que superan la longitud máxima. | Personalice de forma independiente todos los requisitos de complejidad.<br /><br />Elija qué hacer con las contraseñas que superen la longitud máxima (truncarlas o generar un error). |
| Historial de contraseñas   | Configure cuántas contraseñas anteriores no puede reutilizar un usuario, hasta un máximo de 24.                                     | Conserva la misma funcionalidad.                                                                                                                                                       |
| Diccionario de contraseñas | Usa un diccionario integrado de 10.000 palabras comunes.                                                                            | Elija entre dos diccionarios integrados de 10.000 o 100.000 palabras comunes.                                                                                                          |
| Bloquear datos personales  | Bloquea un conjunto fijo de campos de datos de usuario.                                                                             | Personalice por completo qué campos bloquear, hasta un máximo de 12.                                                                                                                   |

En el esquema de configuración de Management API para las conexiones de base de datos, la Política flexible de contraseñas también reemplaza los objetos y atributos `options` de la política de contraseñas heredada por un único objeto configurable: `options.password_options`.

<div id="prerequisites">
  ## Requisitos previos
</div>

Para usar la política de contraseñas flexible, debes cumplir los siguientes requisitos:

* Debes usar una conexión de base de datos con el repositorio de usuarios de Auth0 (`"strategy": "auth0"`).

* El tenant debe usar Universal Login.

* El tenant no debe tener configurada una pantalla personalizada de restablecimiento de contraseña.

* Si usas la Management API, tu token de acceso debe tener los scopes `read:connections` y `update:connections`. Sin ellos, respectivamente, no podrás recuperar ni modificar la configuración de la conexión de base de datos.

<div id="enable-the-flexible-password-policy">
  ## Habilitar la política flexible de contraseñas
</div>

Puede habilitar la política flexible de contraseñas desde el Auth0 Dashboard o la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para habilitar la Política flexible de contraseñas desde el Auth0 Dashboard:

    1. Ve a [Auth0 Dashboard &gt; Autenticación &gt; Database](https://manage.auth0.com/#/connections/database) y selecciona el nombre de la conexión que quieres editar.

    2. Selecciona la pestaña **Métodos de autenticación**. Luego, en la sección **Contraseña**, selecciona **Configurar** para abrir el panel **Contraseña**.

    3. En el banner de Política flexible de contraseñas de la parte superior, selecciona **Activar** y, después, **Confirmar**.

    Esto convierte las políticas de contraseña existentes de tu conexión de base de datos de la configuración heredada a la configuración de Política flexible de contraseñas y te devuelve a la pestaña **Métodos de autenticación**.

    El mismo panel **Métodos de autenticación &gt; Contraseña &gt; Configurar** ahora muestra las opciones de configuración de la Política flexible de contraseñas en tres secciones: **Política**, **Composición** y **Seguridad**.

    <AccordionGroup>
      <Accordion title="Configuración de la política">
        La sección **Política** contiene la siguiente configuración:

        * **Contraseña para el login**:  Permite o bloquea que los usuarios inicien sesión con una contraseña.

        * **Contraseña en el registro**:  Permite o bloquea que los usuarios se registren con una contraseña.

        * **Autoservicio para cambiar la contraseña**:  Permite o bloquea que los usuarios cambien su propia contraseña.

        * **Admitir usuarios sin contraseña**:  Determina si los usuarios pueden crearse sin contraseña mediante las API de Management y Authentication.
      </Accordion>

      <Accordion title="Configuración de composición">
        La sección **Composición** contiene la siguiente configuración:

        * **Longitud mínima de la contraseña**: Introduce la longitud mínima de la contraseña.

        * **Reglas de composición adicionales**

          * **Opciones de contraseña obligatorias**: Al menos una letra mayúscula, al menos una letra minúscula, al menos un número, al menos un carácter especial y/o al menos 3 de 4 de los tipos de caracteres anteriores.

          * **Bloquear tres o más caracteres secuenciales**, como ABC o 321

          * **Bloquear tres o más caracteres idénticos seguidos**, como 000

          * **Se superó la longitud máxima de la contraseña**: Para contraseñas de más de 72 bytes, elige entre **Truncar contraseña** (permitir contraseñas más largas, pero cifrar solo los primeros 72 bytes) o **Mostrar error** (rechazar contraseñas de más de 72 bytes).
      </Accordion>

      <Accordion title="Configuración de seguridad">
        La sección **Seguridad** contiene la siguiente configuración:

        * **Historial de contraseñas**: Evita la reutilización de contraseñas.

          * **Tamaño del historial de contraseñas**: Elige cuántas contraseñas anteriores no pueden reutilizar los usuarios.

        * **Diccionario de contraseñas**: Evita que los usuarios usen contraseñas del diccionario especificado o de tus entradas adicionales del diccionario.

          * **Diccionario predeterminado**: Elige entre los diccionarios de [10,000 palabras comunes](https://github.com/danielmiessler/SecLists/blob/master/Passwords/Common-Credentials/10k-most-common.txt) o [100,000 palabras comunes](https://github.com/danielmiessler/SecLists/blob/master/Passwords/Common-Credentials/100k-most-used-passwords-NCSC.txt).

          * **Entradas adicionales del diccionario**: Introduce contraseñas prohibidas adicionales, una por línea.

        * **Bloquear datos del perfil en las contraseñas**: Evita que los usuarios establezcan contraseñas que contengan datos del perfil del usuario.

          * **Selecciona los campos del perfil que se bloquearán**: Elige entre ocho campos predeterminados comunes para bloquear.

          * **Campos adicionales del perfil para bloquear**: Introduce campos adicionales del perfil para bloquear.
      </Accordion>
    </AccordionGroup>

    Después de modificar esta configuración, haz clic en **Guardar**.
  </Tab>

  <Tab title="Management API">
    En la configuración de tu conexión de base de datos, el objeto `options` contiene la configuración de la política de contraseñas. Flexible Password Policy reemplaza los objetos y atributos de contraseña heredados de `options` por un único objeto (`options.password_options`) con una configuración más granular.

    Para habilitar Flexible Password Policy con la Management API:

    <Steps>
      <Step title="Obtén la configuración actual de la conexión de base de datos">
        Primero, obtén la configuración actual de tu conexión de base de datos mediante el [endpoint obtener una conexión](https://auth0.com/docs/api/management/v2/connections/get-connections-by-id).

        Guarda la configuración completa de la respuesta. Puedes revisar el esquema del cuerpo de la respuesta en la documentación del endpoint.
      </Step>

      <Step title="Reemplaza la configuración heredada por la configuración de Flexible Password Policy">
        En el objeto `options` de la configuración, elimina los siguientes objetos y valores de contraseña heredados y, a continuación, agrega un único objeto `password_options` con la configuración de Flexible Password Policy que desees.

        | atributo heredado de `options` | atributo de `options` de Flexible Password Policy |
        | ------------------------------ | ------------------------------------------------- |
        | `password_complexity_options`  | `password_options.complexity`                     |
        | `password_history`             | `password_options.history`                        |
        | `password_no_personal_info`    | `password_options.profile_data`                   |
        | `password_dictionary`          | `password_options.dictionary`                     |
        | `passwordPolicy`               | Eliminado                                         |

        Por ejemplo, las dos configuraciones de contraseña de `options` siguientes son equivalentes:

        <Columns cols={2}>
          <div>
            ```json Ejemplo de política de contraseñas heredada
            "password_history": {
                "enable": true,
                "size": 5
            },
            "password_dictionary": {
                "enable": true,
                "dictionary": [
                    "badPassword",
                    "reallyBadPassword"
                ]
            }
            ```
          </div>

          <div>
            ```json Ejemplo de Flexible Password Policy
            "password_options": {
                "history": {
                    "active": true,
                    "size": 5
                },
                "dictionary": {
                    "active": true,
                    "default": "en_10k",
                    "custom": [
                        "badPassword",
                        "reallyBadPassword"
                    ]
                }
            }
            ```
          </div>
        </Columns>

        Puedes consultar el esquema completo del objeto `password_options` en la referencia de la API (por ejemplo, en los parámetros del cuerpo del [endpoint actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id)).
      </Step>

      <Step title="Opcionalmente, actualiza otras configuraciones de contraseña">
        Los demás atributos de contraseña fuera del objeto `password_options` permanecen sin cambios, pero se muestran en la configuración de Flexible Password Policy del Auth0 Dashboard:

        | atributo de `options` de la Management API        | configuración del Auth0 Dashboard                   |
        | ------------------------------------------------- | --------------------------------------------------- |
        | `authentication_methods.password.enabled`         | Policy &gt; Contraseña para iniciar sesión          |
        | `authentication_methods.password.signup_behavior` | Policy &gt; Contraseña al registrarse               |
        | `authentication_methods.password.api_behavior`    | Policy &gt; Admitir usuarios sin contraseña         |
        | `disable_self_service_change_password`            | Policy &gt; Autoservicio para cambiar la contraseña |

        Si lo deseas, puedes actualizar estos atributos para modificar la configuración.
      </Step>

      <Step title="Actualiza la conexión de base de datos">
        Actualiza tu conexión de base de datos con el [endpoint actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id).

        <Warning>
          Este endpoint sobrescribe la configuración existente con la configuración que envíes, así que incluye el objeto completo modificado en tu solicitud `PATCH` para conservar los demás ajustes de tu conexión de base de datos.
        </Warning>
      </Step>
    </Steps>

    Puedes verificar la actualización obteniendo de nuevo la configuración de tu conexión de base de datos con el [endpoint obtener una conexión](https://auth0.com/docs/api/management/v2/connections/get-connections-by-id). Si la respuesta contiene el objeto `options.password_options`, Flexible Password Policy está habilitada para esa conexión de base de datos.
  </Tab>
</Tabs>

<div id="revert-to-the-legacy-password-policies">
  ## Revertir a las políticas de contraseñas heredadas
</div>

Para revertir las políticas de contraseñas a la versión heredada, debes usar la Management API.

Sigue el mismo proceso indicado arriba para actualizar la conexión de base de datos con sus `options` originales y elimina por completo el objeto `password_options`.
