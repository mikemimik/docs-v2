<ReleaseStageNotice feature="Autenticación passwordless en Database Connections" stage="ea" contact="support" terms="true" />

Auth0 te permite configurar la autenticación passwordless por correo electrónico y teléfono directamente en una [conexión de base de datos](/docs/es/authenticate/database-connections). En lugar de crear una conexión independiente para la autenticación con contraseña de un solo uso (OTP), puedes ofrecer inicio de sesión passwordless directamente desde tu conexión de base de datos. Esto reduce la complejidad de la implementación y simplifica la experiencia de inicio de sesión para los usuarios finales.

<Callout icon="triangle-exclamation" color="#F59E0B" iconType="regular">
  Passwordless en conexiones de base de datos no es compatible con Classic Login.
</Callout>

<div id="how-it-works">
  ## Cómo funciona
</div>

Auth0 utiliza un enfoque de [autenticación Identifier-First](/docs/es/authenticate/login/auth0-universal-login/identifier-first) en el que los usuarios se verifican mediante autenticación OTP:

1. Un usuario final introduce un identificador para iniciar sesión con [Universal Login](/docs/es/authenticate/login/auth0-universal-login).
2. Auth0 busca los métodos de autenticación de cada identificador configurado en tu conexión de base de datos.
3. Al usuario se le muestra la opción más adecuada según `default_method` y los métodos de autenticación disponibles (por ejemplo, recibir un OTP por correo electrónico o por teléfono, o introducir una contraseña).
4. Si OTP es el método de autenticación elegido, Auth0 envía un código al correo electrónico o al teléfono del usuario.
5. El usuario introduce el código y queda autenticado. Si has configurado claves de acceso, se le pedirá al usuario que realice la inscripción progresiva de claves de acceso.

<Card title="Antes de empezar">
  * Configura tu tenant para usar el [perfil de autenticación Identifier-First](https://auth0.com/docs/authenticate/login/auth0-universal-login/identifier-first).
  * Si tienes previsto usar OTP por teléfono (SMS o voz), debes habilitar la opción [Unified Phone Experience](/docs/es/customize/phone-messages/unified-phone/configure-unified-phone) en [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
  * Los [atributos](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers) deben estar habilitados en la conexión de base de datos.
  * Si tienes previsto usar conexiones passwordless existentes (Legacy) junto con la autenticación passwordless en conexiones de base de datos, y no quieres configurar tus conexiones passwordless existentes para usar Unified Phone Provider:
    1. Ve a [**Auth0 Dashboard &gt; Authentication &gt; Passwordless**](https://manage.auth0.com/#/connections/passwordless).
    2. Selecciona **Configure** junto a SMS.
    3. Asegúrate de que la opción **Use Tenant-Level Messaging Provider** esté deshabilitada.
</Card>

<div id="configure-attributes-identifiers">
  ## Configurar atributos (identificadores)
</div>

Al configurar la autenticación passwordless con Email y teléfono en conexiones de base de datos, primero debes determinar qué atributo(s) quieres que los usuarios finales proporcionen durante el registro y el inicio de sesión.

Revisa el tipo de identificador y el método de autenticación correspondiente:

| Identificador     | Método(s) de autenticación                    |
| ----------------- | --------------------------------------------- |
| Email             | Contraseña, Email OTP, Clave de acceso        |
| Teléfono          | Contraseña, OTP por teléfono, Clave de acceso |
| Nombre de usuario | Contraseña                                    |

Para los identificadores de Email y teléfono, si la contraseña **no** está habilitada, debes tener habilitada la verificación por OTP para el registro. Puedes configurar los atributos de Email y teléfono como opcionales en el registro, para que los usuarios puedan registrarse solo con un Email o solo con un número de teléfono.

Cuando un usuario se autentica mediante Email OTP, `email_verified` se establece automáticamente en `true` en su perfil. Cuando un usuario se autentica mediante OTP por teléfono, `phone_verified` se establece automáticamente en `true`.

<Warning>
  Si quieres que los usuarios finales se autentiquen con claves de acceso, **debes** tener configurado un método de autenticación alternativo, como Email o teléfono. La autenticación solo con clave de acceso no es compatible.
</Warning>

Para obtener más información, consulta [Activar y configurar atributos para Flexible Identifiers](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers).

<div id="create-a-new-database-connection">
  ## Crear una nueva conexión de base de datos
</div>

Si no tiene una conexión de base de datos existente, cree una con Auth0 Dashboard o la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard" icon="browser">
    1. Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database). Seleccione **Create DB Connection** para crear la conexión.
    2. Introduzca un nombre único para su conexión.
    3. Elija uno o varios atributos para que los usuarios finales inicien sesión o se registren.
    4. Seleccione sus métodos de autenticación. Podrá configurar estos métodos con más detalle una vez que haya creado la conexión.
    5. Active **Disable Sign Ups** si no quiere que los usuarios se registren mediante endpoints públicos.
    6. Active **Promote Connection to Domain Level** si quiere usar esta conexión con aplicaciones de terceros.
    7. Seleccione **Create**.

    Para crear una conexión verdaderamente passwordless, complete estos pasos adicionales:

    1. En su nueva conexión, seleccione la pestaña **Attributes**.
    2. Para desactivar Username como identificador, seleccione **Configure** y desactive **Use Username as Identifier**.
    3. Para configurar los identificadores Email y Phone, seleccione **Configure**.

    * Para los atributos de Email, seleccione **One-Time Password (OTP)** en Verification Methods y active **Verify email on sign up** para una configuración verdaderamente passwordless. Esto garantiza que `email_verified` se establezca automáticamente para que siempre se solicite un OTP a los usuarios al iniciar sesión y al registrarse.

    4. Seleccione **Save**.

    5. Configure los ajustes de los métodos de autenticación que correspondan a los identificadores elegidos. No puede desactivar password a menos que tenga configurados `phone_otp` y/o `email_otp`.

    6. En la configuración de Password, seleccione **Policy** y elija **Block** para:

       * Password on Login
       * Password on Signup
       * Self-service change password (se actualiza automáticamente)

    7. Active **Support users without a password**.
       <Warning>
         Puede que reciba un error si no activa **Support users without a password**.
       </Warning>

    8. Seleccione **Save**. En el prompt, seleccione **Continue** para confirmar que entiende que los usuarios existentes pueden verse afectados.

    9. Vaya a la pestaña **Applications** y habilite la conexión para su aplicación o API.
  </Tab>

  <Tab title="Management API" icon="code">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Para usar la Management API, necesita un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).
    </Callout>

    Llame al endpoint [Create a Connection](https://auth0.com/docs/api/management/v2/connections/post-connections). Use la opción `default_method` para establecer el primer método de autenticación para cada identificador.

    ```bash expandable lines
    curl --location 'https://YOUR_AUTH0_TENANT/api/v2/connections' \
    --header 'Authorization: Bearer YOUR_MANAGEMENT_API_TOKEN' \
    --header 'Accept: application/json, text/plain, */*' \
    --header 'Content-Type: application/json' \
    --data '{
      "name": "YOUR_CONNECTION_NAME",
      "strategy": "auth0",
      "is_domain_connection": false,
      "options": {
        "disable_signup": false,
        "attributes": {
          "email": {
            "identifier": {
              "active": true,
              "default_method": "email_otp"
            },
            "profile_required": true,
            "signup": {
              "status": "required",
              "verification": {
                "active": true
              }
            },
            "verification_method": "otp",
            "unique": true
          },
          "phone_number": {
            "identifier": {
              "active": true,
              "default_method": "password"
            },
            "profile_required": true,
            "signup": {
              "status": "required",
              "verification": {
                "active": true
              }
            }
          },
          "username": {
            "identifier": {
              "active": true
            },
            "profile_required": true,
            "signup": {
              "status": "required"
            },
            "validation": {
              "min_length": 1,
              "max_length": 15,
              "allowed_types": {
                "email": false,
                "phone_number": false
              }
            }
          }
        },
        "authentication_methods": {
          "password": {
            "enabled": true
          },
          "passkey": {
            "enabled": false
          },
          "email_otp": {
            "enabled": true
          },
          "phone_otp": {
            "enabled": true
          }
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="update-existing-connections">
  ## Actualizar conexiones existentes
</div>

Si tiene conexiones de base de datos existentes, actualice la configuración de passwordless en el Auth0 Dashboard o en la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard" icon="browser">
    1. **Abra la conexión**: Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) y seleccione la conexión que desea actualizar.
    2. **Activar Attributes**: En la pestaña **Attributes**, seleccione **Activate** para habilitar la nueva configuración de atributos.
    3. **Agregar los atributos Email y Phone**: Seleccione **+ Add Attributes** y agregue Email y Phone Number si todavía no están presentes.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         En una conexión OTP verdaderamente passwordless, no se admite el identificador `username`.
       </Callout>
    4. **Habilitar métodos de autenticación con OTP**: En la pestaña **Authentication Methods**:
       * Configure Phone para **Allow** Phone OTP y guarde los cambios.
       * Configure Email para **Allow** Email OTP y guarde los cambios.
    5. **Bloquear la autenticación con contraseña**: En la configuración de Password, elija **Policy** y seleccione **Block** para:

       * Password on Login
       * Password on Signup
       * Self-service change password

       Active **Support users without a password**.

       <Warning>
         Es posible que reciba un error si no activa **Support users without a password**.
       </Warning>
    6. **Save**: Seleccione **Save**.
  </Tab>

  <Tab title="Management API" icon="code">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Para usar la Management API, necesita un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/get-management-api-access-tokens-for-production).
    </Callout>

    Llame al endpoint [actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id). Agregue la opción `default_method` para establecer el primer método de autenticación. Para los atributos Email, establezca `password` o `email_otp`. Para los atributos Phone, establezca `password` o `phone_otp`.

    ```bash
    curl -L -X PATCH "https://YOUR_AUTH0_TENANT/api/v2/connections/<connection_id>" \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer YOUR_MANAGEMENT_API_TOKEN" \
    -d '{
      "options": {
        "passwordPolicy": "none",
        "authentication_methods": {
          "password": { "enabled": true },
          "passkey": { "enabled": true },
          "email_otp": { "enabled": true },
          "phone_otp": { "enabled": true }
        },
        "attributes": {
          "email": {
            "signup": {
              "status": "required",
              "verification": { "active": true }
            },
            "unique": true,
            "identifier": {
              "active": true,
              "default_method": "email_otp"
            },
            "profile_required": true,
            "verification_method": "otp"
          },
          "phone_number": {
            "signup": {
              "status": "required",
              "verification": { "active": true }
            },
            "identifier": {
              "active": true,
              "default_method": "phone_otp"
            },
            "profile_required": true
          }
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="use-auth0-actions">
  ## Usa Auth0 Actions
</div>

Para profundizar en los factores de autenticación passwordless en las conexiones de base de datos, configura atributos mediante Auth0 Actions.

<div id="post-login-trigger">
  ### trigger Post Login
</div>

El trigger `post-login` se activa después de que un usuario se autentica, pero antes de que el servidor de autorización devuelva un token. El objeto `event.authentication` en [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post Login](https://manage.auth0.com/#/actions/flows) expone los siguientes métodos:

| Método                     | Parámetro | Descripción                                                                     |
| -------------------------- | --------- | ------------------------------------------------------------------------------- |
| Email OTP                  | `email`   | Email OTP usado para autenticar al usuario con el primer factor.                |
| OTP por teléfono con texto | `sms`     | OTP por teléfono (SMS) usado para autenticar al usuario con el primer factor.   |
| OTP por teléfono con voz   | `tel`     | OTP por teléfono con voz usado para autenticar al usuario con el primer factor. |
| Password                   | `pwd`     | Password usada para autenticar al usuario con el primer factor.                 |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  `email_verified` y `phone_verified` se establecen automáticamente en el perfil del usuario cuando este se autentica mediante Email OTP u OTP por teléfono. Si antes usabas una Action Post Login para establecer manualmente estos indicadores, puedes eliminar esa solución temporal.
</Callout>

Usa `event.authentication` para:

* Detectar qué factor passwordless completó el usuario (`email`, `sms`, `tel` o `pwd`)
* Añadir claims personalizadas a los tokens según el método de autenticación
* Ejecutar lógica de forma condicional según cómo se autenticó el usuario

<div id="example">
  #### Ejemplo
</div>

El siguiente ejemplo lee `event.authentication.methods` para detectar qué factor passwordless completó el usuario y lo añade como un claim personalizado en el token de identidad.

```javascript
exports.onExecutePostLogin = async (event, api) => {
  if (!event.authentication?.methods || event.authentication.methods.length === 0) {
    return;
  }

  if (event.connection.name !== 'YOUR_AUTH0_CONNECTION') {
    return;
  }

  const firstFactor = event.authentication.methods[0];

  if (firstFactor.name === 'email') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'email_otp');
  } else if (firstFactor.name === 'sms' || firstFactor.name === 'tel') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'phone_otp');
  } else if (firstFactor.name === 'pwd') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'password');
  }
};
```

<div id="post-challenge-trigger">
  ### trigger posterior al desafío
</div>

El trigger `post-challenge` se activa después de que los usuarios completan un desafío, como un restablecimiento de contraseña, la validación del número de teléfono o MFA. El objeto `event.authentication` en [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; password-reset-post-challenge](https://manage.auth0.com/#/actions/flows) expone los siguientes atributos:

| Atributo | Parámetro      | Descripción                                                                 |
| -------- | -------------- | --------------------------------------------------------------------------- |
| Email    | `email`        | Restablecimiento de contraseña con OTP por correo electrónico o Magic Link. |
| Teléfono | `phone_number` | Restablecimiento de contraseña con OTP por teléfono.                        |

<div id="example">
  #### Ejemplo
</div>

El siguiente ejemplo lee `event.authentication.methods` para detectar qué factor passwordless completó la verificación y añade un claim personalizado al token de identidad.

```javascript
exports.onExecutePostChallenge = async (event, api) => {
  if (!event.authentication?.methods || event.authentication.methods.length === 0) {
    return;
  }

  if (event.connection.name !== 'YOUR_AUTH0_CONNECTION') {
    return;
  }

  const firstFactor = event.authentication.methods[0];

  if (firstFactor.name === 'email') {
    api.idToken.setCustomClaim('https://your-app.com/challenge_method', 'email_otp');
  } else if (firstFactor.name === 'phone_number') {
    api.idToken.setCustomClaim('https://your-app.com/challenge_method', 'phone_otp');
  }
};
```

<div id="benefits">
  ## Beneficios
</div>

* **Implementación simplificada:** Menos conexiones que configurar y mantener. No es necesario vincular cuentas, salvo que se utilicen conexiones sociales/federadas junto con conexiones de base de datos.
* **Mejor experiencia de usuario**: Permite combinar OTP por correo electrónico y por teléfono con contraseñas, clave de acceso e inicio de sesión social/federado, todo desde la misma experiencia centrada en el identificador en Universal Login.
* **Flujos de registro flexibles**: Configure los atributos de correo electrónico y teléfono como opcionales en el registro, para que los usuarios puedan registrarse solo con un correo electrónico o solo con un número de teléfono; ideal para experiencias centradas en dispositivos móviles o únicamente en correo electrónico.
* **OTP por voz**: OTP por voz se incluye como primer factor cuando se configura en la experiencia telefónica unificada.

<div id="limitations">
  ## Limitaciones
</div>

* Disponible solo para flujos basados en Universal Login; aún no es compatible con la autenticación mediante API.
* Passwordless para conexiones de base de datos no es compatible con Classic Login.
* Passwordless para conexiones de base de datos no es compatible con Implicit Signup &amp; Login.
* En las conexiones passwordless heredadas, no había diferencias en la experiencia de usuario entre el registro y el inicio de sesión. Las conexiones de base de datos distinguen entre la experiencia de registro y la de inicio de sesión, lo que requiere un registro y un inicio de sesión *explícitos*.
  * Si un usuario sin una identidad de Auth0 entra en el flujo de inicio de sesión, el sistema no lo registrará automáticamente. El usuario recibirá un error después de validar el OTP.
  * Si un usuario con una identidad de Auth0 entra en el flujo de registro, recibirá un error después de validar el OTP.
