<ReleaseStageNotice feature="Autenticación passwordless en Database Connections" stage="ea" contact="support" terms="true" />

Auth0 te permite configurar la autenticación passwordless por correo electrónico y teléfono directamente en una [conexión de base de datos](/docs/es/authenticate/database-connections). En lugar de crear una conexión independiente para la autenticación con contraseña de un solo uso (OTP), puedes ofrecer inicio de sesión passwordless directamente desde tu conexión de base de datos. Esto reduce la complejidad de la implementación y simplifica la experiencia de inicio de sesión para los usuarios finales.

<Callout icon="triangle-exclamation" color="#F59E0B" iconType="regular">
  Passwordless en conexiones de base de datos no es compatible con Classic Login.
</Callout>

<div id="how-it-works">
  ## Cómo funciona
</div>

Auth0 usa un enfoque de [Identifier-First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first) en el que los usuarios se verifican con autenticación con OTP:

1. Un usuario final introduce un identificador para iniciar sesión con [Universal Login de Auth0](/docs/es/authenticate/login/auth0-universal-login).
2. Auth0 busca los método de autenticación de cada identificador configurado en tu conexión de base de datos.
3. Al usuario se le muestra la opción más adecuada según `default_method` y los métodos de autenticación disponibles (por ejemplo, recibir un OTP por email o por teléfono, o introducir una contraseña).
4. Si OTP es el método de autenticación seleccionado, Auth0 envía un código al email o al teléfono del usuario.
5. El usuario introduce el código y queda autenticado. Si has configurado claves de acceso, se le pedirá que complete la inscripción progresiva de clave de acceso.

<Card title="Antes de empezar">
  * Configura tu tenant para usar el [Identifier-First Authentication Profile](https://auth0.com/docs/authenticate/login/auth0-universal-login/identifier-first).
  * Si planeas usar OTP por teléfono (SMS o voz), debes habilitar la configuración de [Unified Phone Experience](/docs/es/customize/phone-messages/unified-phone/configure-unified-phone) en [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
  * [Atributos](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers) debe estar habilitado en la conexión de base de datos.
  * Si planeas usar conexiones passwordless existentes (legacy) junto con autenticación passwordless en conexiones de base de datos, y no quieres configurar tus conexiones passwordless existentes para que usen el proveedor de mensajería unificado:
    1. Ve a [**Auth0 Dashboard &gt; Authentication &gt; Passwordless**](https://manage.auth0.com/#/connections/passwordless).
    2. Selecciona **configurar** junto a SMS.
    3. Asegúrate de que la configuración **Usar proveedor de mensajería a nivel de tenant** esté deshabilitada.
</Card>

<div id="configure-attributes-identifiers">
  ## Configurar atributos (identificadores)
</div>

Al configurar la autenticación passwordless con Email y teléfono en conexiones de base de datos, primero debes determinar qué atributo o atributos quieres que los usuarios finales proporcionen durante el registro y el inicio de sesión.

Revisa el tipo de identificador y el método de autenticación correspondiente:

| Identificador     | Método(s) de autenticación                    |
| ----------------- | --------------------------------------------- |
| Email             | Contraseña, Email OTP, Clave de acceso        |
| Teléfono          | Contraseña, OTP por teléfono, Clave de acceso |
| Nombre de usuario | Contraseña                                    |

Para los identificadores de Email y teléfono, si la contraseña **no** está habilitada, debes tener habilitada la verificación por OTP para el registro. Puedes configurar los atributos de Email y teléfono como opcionales durante el registro, para que los usuarios puedan registrarse solo con un Email o solo con un número de teléfono.

Cuando un usuario se autentica mediante Email OTP, `email_verified` se establece automáticamente en `true` en su perfil. Cuando un usuario se autentica mediante OTP por teléfono, `phone_verified` se establece automáticamente en `true`.

<Warning>
  Si quieres que los usuarios finales se autentiquen con claves de acceso, **debes** tener configurado un método de autenticación alternativo, como Email o teléfono. La autenticación solo con claves de acceso no es compatible.
</Warning>

Para obtener más información, consulta [Activate and Configure Attributes for Flexible Identifiers](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers).

<div id="create-a-new-database-connection">
  ## Crear una nueva conexión de base de datos
</div>

Si no tiene una conexión de base de datos existente, cree una mediante el Auth0 Dashboard o la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard" icon="browser">
    1. Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database). Seleccione **Create DB Connection** para crear la conexión.
    2. Introduzca un nombre único para su conexión.
    3. Elija uno o varios atributos para que los usuarios finales inicien sesión o se registren.
    4. Seleccione sus método(s) de autenticación. Podrá configurar estos métodos con más detalle una vez que haya creado la conexión.
    5. Active **Disable Sign Ups** si no quiere que los usuarios se registren mediante endpoints públicos.
    6. Active **Promote Connection to Domain Level** si quiere usar esta conexión con aplicaciones de terceros.
    7. Seleccione **Create**.

    Para una conexión passwordless real, complete los siguientes pasos adicionales:

    1. En su nueva conexión, seleccione la pestaña **Atributos**.
    2. Para deshabilitar Username como identificador, seleccione **Configure** y desactive **Use Username as Identifier**.
    3. Para configurar los identificadores Email y Phone, seleccione **Configure**.

    * Para los atributos de Email, seleccione **One-Time Password (OTP)** en Verification Methods y habilite **Verify email on sign up** para una configuración passwordless real. Esto garantiza que `email_verified` se establezca automáticamente, de modo que siempre se solicite OTP a los usuarios al iniciar sesión y al registrarse.

    4. Seleccione **Save**.

    5. Configure los ajustes de los métodos de autenticación que correspondan a los identificadores elegidos. No puede deshabilitar password a menos que tenga configurados `phone_otp` y/o `email_otp`.

    6. En la configuración de Password, seleccione **Policy** y elija **Block** para:

       * Password on Login
       * Password on Signup
       * Self-service change password (se actualiza automáticamente)

    7. Active **Support users without a password**.
       <Warning>
         Puede que reciba un error si no activa **Support users without a password**.
       </Warning>

    8. Seleccione **Save**. En el prompt, seleccione **Continue** para confirmar que entiende que los usuarios existentes pueden verse afectados.

    9. Vaya a la pestaña **Applications** y habilite la conexión para su aplicación o API.
  </Tab>

  <Tab title="Management API" icon="code">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Para usar la Management API, necesita un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).
    </Callout>

    Llame al endpoint [Create a Connection](https://auth0.com/docs/api/management/v2/connections/post-connections). Use la opción `default_method` para establecer el primer método de autenticación para cada identificador.

    ```bash expandable lines
    curl --location 'https://YOUR_AUTH0_TENANT/api/v2/connections' \
    --header 'Authorization: Bearer YOUR_MANAGEMENT_API_TOKEN' \
    --header 'Accept: application/json, text/plain, */*' \
    --header 'Content-Type: application/json' \
    --data '{
      "name": "YOUR_CONNECTION_NAME",
      "strategy": "auth0",
      "is_domain_connection": false,
      "options": {
        "disable_signup": false,
        "attributes": {
          "email": {
            "identifier": {
              "active": true,
              "default_method": "email_otp"
            },
            "profile_required": true,
            "signup": {
              "status": "required",
              "verification": {
                "active": true
              }
            },
            "verification_method": "otp",
            "unique": true
          },
          "phone_number": {
            "identifier": {
              "active": true,
              "default_method": "password"
            },
            "profile_required": true,
            "signup": {
              "status": "required",
              "verification": {
                "active": true
              }
            }
          },
          "username": {
            "identifier": {
              "active": true
            },
            "profile_required": true,
            "signup": {
              "status": "required"
            },
            "validation": {
              "min_length": 1,
              "max_length": 15,
              "allowed_types": {
                "email": false,
                "phone_number": false
              }
            }
          }
        },
        "authentication_methods": {
          "password": {
            "enabled": true
          },
          "passkey": {
            "enabled": false
          },
          "email_otp": {
            "enabled": true
          },
          "phone_otp": {
            "enabled": true
          }
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="update-existing-connections">
  ## Actualizar conexiones existentes
</div>

Si ya tiene conexiones de base de datos, actualice la configuración de passwordless en el Auth0 Dashboard o la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard" icon="browser">
    1. **Abra la conexión**: Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) y seleccione la conexión que desea actualizar.
    2. **Active atributos**: En la pestaña **Atributos**, seleccione **Activar** para habilitar la nueva configuración de atributos.
    3. **Agregue los atributos Email y Phone**: Elija **+ Agregar atributos** y agregue Email y Número de teléfono si aún no están presentes.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         En una conexión passwordless (OTP) propiamente dicha, el identificador de username no es compatible.
       </Callout>
    4. **Habilite los métodos de autenticación OTP**: En la pestaña **Métodos de autenticación**:
       * Configure Teléfono para **Permitir** OTP por teléfono y guarde los cambios.
       * Configure Email para **Permitir** Email OTP y guarde los cambios.
    5. **Bloquee la autenticación con contraseña**: En la configuración de Password, elija **Policy** y seleccione **Block** para:

       * Password en Login
       * Password en Signup
       * cambiar la contraseña de autoservicio

       Active **Permitir usuarios sin contraseña**.

       <Warning>
         Puede que reciba un error si no activa **Permitir usuarios sin contraseña**.
       </Warning>
    6. **Guardar**: Seleccione **Guardar**.
  </Tab>

  <Tab title="Management API" icon="code">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Para usar la Management API, necesita un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/get-management-api-access-tokens-for-production).
    </Callout>

    Llame al endpoint [actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id). Agregue la opción `default_method` para establecer el método de autenticación inicial. Para los atributos de Email, establezca `password` o `email_otp`. Para los atributos de Teléfono, establezca `password` o `phone_otp`.

    ```bash
    curl -L -X PATCH "https://YOUR_AUTH0_TENANT/api/v2/connections/<connection_id>" \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer YOUR_MANAGEMENT_API_TOKEN" \
    -d '{
      "options": {
        "passwordPolicy": "none",
        "authentication_methods": {
          "password": { "enabled": true },
          "passkey": { "enabled": true },
          "email_otp": { "enabled": true },
          "phone_otp": { "enabled": true }
        },
        "attributes": {
          "email": {
            "signup": {
              "status": "required",
              "verification": { "active": true }
            },
            "unique": true,
            "identifier": {
              "active": true,
              "default_method": "email_otp"
            },
            "profile_required": true,
            "verification_method": "otp"
          },
          "phone_number": {
            "signup": {
              "status": "required",
              "verification": { "active": true }
            },
            "identifier": {
              "active": true,
              "default_method": "phone_otp"
            },
            "profile_required": true
          }
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="use-auth0-actions">
  ## Usa Auth0 Actions
</div>

Para conocer mejor los factores de autenticación passwordless en las conexiones de base de datos, configura atributos con Auth0 Actions.

<div id="post-login-trigger">
  ### trigger de Post Login
</div>

El trigger `post-login` se ejecuta después de que un usuario se autentica, pero antes de que el servidor de autorización devuelva un token. El objeto `event.authentication` en [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post Login](https://manage.auth0.com/#/actions/flows) expone los siguientes métodos:

| Método                     | Parámetro | Descripción                                                                   |
| -------------------------- | --------- | ----------------------------------------------------------------------------- |
| Email OTP                  | `email`   | Email OTP usado para autenticar al usuario como primer factor.                |
| OTP por teléfono con texto | `sms`     | OTP por teléfono (SMS) usado para autenticar al usuario como primer factor.   |
| OTP por teléfono con voz   | `tel`     | OTP por teléfono con voz usado para autenticar al usuario como primer factor. |
| Password                   | `pwd`     | Password usado para autenticar al usuario como primer factor.                 |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  `email_verified` y `phone_verified` se establecen automáticamente en el perfil del usuario cuando este se autentica mediante Email OTP u OTP por teléfono. Si antes usabas una Action de Post Login para establecer manualmente estos indicadores, puedes eliminar esta solución temporal.
</Callout>

Usa `event.authentication` para:

* Detectar qué factor passwordless completó el usuario (`email`, `sms`, `tel` o `pwd`)
* Agregar claims personalizadas a los tokens según el método de autenticación
* Ejecutar lógica de forma condicional según cómo se autenticó el usuario

<div id="example">
  #### Ejemplo
</div>

El siguiente ejemplo lee `event.authentication.methods` para detectar qué factor passwordless completó el usuario y añadirlo como custom claim al token de identidad.

```javascript
exports.onExecutePostLogin = async (event, api) => {
  if (!event.authentication?.methods || event.authentication.methods.length === 0) {
    return;
  }

  if (event.connection.name !== 'YOUR_AUTH0_CONNECTION') {
    return;
  }

  const firstFactor = event.authentication.methods[0];

  if (firstFactor.name === 'email') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'email_otp');
  } else if (firstFactor.name === 'sms' || firstFactor.name === 'tel') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'phone_otp');
  } else if (firstFactor.name === 'pwd') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'password');
  }
};
```

<div id="post-challenge-trigger">
  ### Trigger post-challenge
</div>

El trigger `post-challenge` se activa después de que los usuarios completan una verificación, como un restablecimiento de contraseña, la validación del número de teléfono o MFA. El objeto `event.authentication` en [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; password-reset-post-challenge](https://manage.auth0.com/#/actions/flows) expone los siguientes atributos:

| Atributo           | Parámetro      | Descripción                                                                 |
| ------------------ | -------------- | --------------------------------------------------------------------------- |
| Correo electrónico | `email`        | Restablecimiento de contraseña con OTP por correo electrónico o Magic Link. |
| Teléfono           | `phone_number` | Restablecimiento de contraseña con OTP por teléfono.                        |

<div id="example">
  #### Ejemplo
</div>

El siguiente ejemplo lee `event.authentication.methods` para detectar qué factor passwordless completó la verificación y lo añade como un claim personalizado en el token de identidad.

```javascript
exports.onExecutePostChallenge = async (event, api) => {
  if (!event.authentication?.methods || event.authentication.methods.length === 0) {
    return;
  }

  if (event.connection.name !== 'YOUR_AUTH0_CONNECTION') {
    return;
  }

  const firstFactor = event.authentication.methods[0];

  if (firstFactor.name === 'email') {
    api.idToken.setCustomClaim('https://your-app.com/challenge_method', 'email_otp');
  } else if (firstFactor.name === 'phone_number') {
    api.idToken.setCustomClaim('https://your-app.com/challenge_method', 'phone_otp');
  }
};
```

<div id="benefits">
  ## Beneficios
</div>

* **Implementación simplificada:** Menos conexiones que configurar y mantener. No es necesario vincular cuentas, salvo que se usen conexiones sociales/federadas junto con conexiones de base de datos.
* **Mejora de la experiencia de usuario**: Ofrece combinaciones de OTP por correo electrónico y teléfono con contraseñas, claves de acceso e inicio de sesión social/federado, todo desde la misma experiencia de identificador primero en Universal Login.
* **Flujos de registro flexibles**: Configura los atributos de correo electrónico y teléfono como opcionales en el registro, lo que permite a los usuarios registrarse solo con un correo electrónico o solo con un número de teléfono, ideal para experiencias centradas en dispositivos móviles o solo por correo electrónico.
* **OTP por voz**: El OTP por voz se incluye como primer factor cuando se configura en Unified Phone Experience.

<div id="limitations">
  ## Limitaciones
</div>

* Disponible solo para flujos basados en Universal Login; todavía no es compatible con la autenticación basada en API.
* passwordless para conexiones de base de datos no es compatible con Classic Login.
* passwordless para conexiones de base de datos no es compatible con Implicit Signup &amp; Login.
* En las conexiones passwordless heredadas, no había diferencias en la experiencia de usuario entre el registro y el inicio de sesión. Las conexiones de base de datos distinguen entre la experiencia de registro y la de inicio de sesión, lo que requiere un registro y un inicio de sesión *explícitos*.
  * Si un usuario sin una identidad de Auth0 entra en el flujo de inicio de sesión, el sistema no lo registrará automáticamente. El usuario recibe un error después de validar el OTP.
  * Si un usuario con una identidad de Auth0 entra en el flujo de registro, recibe un error después de validar el OTP.
