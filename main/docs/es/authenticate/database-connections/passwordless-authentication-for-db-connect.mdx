<ReleaseStageNotice feature="Autenticación passwordless en Database Connections" stage="ea" contact="support" terms="true" />

Auth0 te permite configurar la autenticación passwordless por correo electrónico y teléfono directamente en una [conexión de base de datos](/docs/es/authenticate/database-connections). En lugar de crear una conexión independiente para la autenticación con código de un solo uso (OTP), puedes ofrecer inicio de sesión passwordless directamente desde tu conexión de base de datos. Esto reduce la complejidad de la implementación y simplifica la experiencia de inicio de sesión para los usuarios finales.

<Callout icon="triangle-exclamation" color="#F59E0B" iconType="regular">
  Passwordless para conexiones de base de datos no es compatible con Classic Login.
</Callout>

<div id="how-it-works">
  ## Cómo funciona
</div>

Auth0 usa un enfoque de [autenticación Identifier-First](/docs/es/authenticate/login/auth0-universal-login/identifier-first) en el que los usuarios se verifican mediante autenticación OTP:

1. Un usuario final introduce un identificador para iniciar sesión con [Universal Login de Auth0](/docs/es/authenticate/login/auth0-universal-login).
2. Auth0 busca los métodos de autenticación para cada identificador configurado en tu conexión de base de datos.
3. Se le presenta al usuario la opción más adecuada según `default_method` y los métodos de autenticación disponibles (por ejemplo, recibir un OTP por correo electrónico o por teléfono, o introducir una contraseña).
4. Si OTP es el método de autenticación seleccionado, Auth0 envía un código al correo electrónico o al teléfono del usuario.
5. El usuario introduce el código y queda autenticado. Si has configurado claves de acceso, se le pedirá que realice la inscripción progresiva de una clave de acceso.

<Card title="Antes de empezar">
  * Configura tu tenant para usar el [perfil de autenticación Identifier-First](https://auth0.com/docs/authenticate/login/auth0-universal-login/identifier-first).
  * Si planeas usar OTP por teléfono (SMS o voz), debes habilitar la configuración [Unified Phone Experience](/docs/es/customize/phone-messages/unified-phone/configure-unified-phone) en [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
  * [Atributos](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers) debe estar habilitado en la conexión de base de datos.
  * Si planeas usar conexiones passwordless existentes (Legacy) junto con autenticación passwordless en conexiones de base de datos, y no quieres configurar tus conexiones passwordless existentes para usar Unified Phone Provider:
    1. Ve a [**Auth0 Dashboard &gt; Authentication &gt; Passwordless**](https://manage.auth0.com/#/connections/passwordless).
    2. Selecciona **configurar** en SMS.
    3. Asegúrate de que la configuración **Usar proveedor de mensajería a nivel de tenant** esté deshabilitada.
</Card>

<div id="configure-attributes-identifiers">
  ## Configurar atributos (identificadores)
</div>

Al configurar la autenticación passwordless con Email y teléfono en conexiones de base de datos, primero debes determinar qué atributo(s) quieres que los usuarios finales proporcionen durante el registro y el inicio de sesión.

Revisa el tipo de identificador y el método de autenticación correspondiente:

| Identificador     | Método(s) de autenticación                    |
| ----------------- | --------------------------------------------- |
| Email             | Contraseña, Email OTP, Clave de acceso        |
| Teléfono          | Contraseña, OTP por teléfono, Clave de acceso |
| Nombre de usuario | Contraseña                                    |

Para los identificadores de Email y teléfono, si la contraseña **no** está habilitada, debes tener habilitada la verificación por OTP para el registro. Puedes configurar los atributos de Email y teléfono como opcionales durante el registro, para que los usuarios puedan registrarse solo con un Email o solo con un número de teléfono.

Cuando un usuario se autentica mediante Email OTP, `email_verified` se establece automáticamente en `true` en su perfil. Cuando un usuario se autentica mediante OTP por teléfono, `phone_verified` se establece automáticamente en `true`.

<Warning>
  Si quieres que los usuarios finales se autentiquen con claves de acceso, **debes** tener configurado un método de autenticación alternativo, como Email o teléfono. La autenticación únicamente con claves de acceso no es compatible.
</Warning>

Para obtener más información, consulta [Activate and Configure Attributes for Flexible Identifiers](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers).

<div id="create-a-new-database-connection">
  ## Crear una nueva conexión de base de datos
</div>

Si no tienes una conexión de base de datos existente, crea una con Auth0 Dashboard o la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard" icon="browser">
    1. Ve a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database). Selecciona **Create DB Connection** para crear la conexión.
    2. Introduce un nombre único para tu conexión.
    3. Elige uno o varios atributos para que los usuarios finales inicien sesión o se registren.
    4. Selecciona tus métodos de autenticación. Podrás configurarlos con más detalle una vez que hayas creado la conexión.
    5. Activa **Disable Sign Ups** si no quieres que los usuarios se registren mediante endpoints públicos.
    6. Activa **Promote Connection to Domain Level** si quieres usar esta conexión con aplicaciones de terceros.
    7. Selecciona **Create**.

    Para una conexión realmente passwordless, completa estos pasos adicionales:

    1. En tu nueva conexión, selecciona la pestaña **Atributos**.
    2. Para desactivar Username como identificador, selecciona **Configure** y desactiva **Use Username as Identifier**.
    3. Para configurar los identificadores Email y Phone, selecciona **Configure**.

    * Para los atributos de Email, selecciona **One-Time Password (OTP)** en Verification Methods y activa **Verify email on sign up** para una configuración realmente passwordless. Esto garantiza que `email_verified` se establezca automáticamente, de modo que siempre se solicite OTP a los usuarios al iniciar sesión y registrarse.

    4. Selecciona **Save**.

    5. Configura los ajustes de los métodos de autenticación que correspondan a los identificadores elegidos. No puedes desactivar password a menos que tengas configurados `phone_otp` y/o `email_otp`.

    6. En la configuración de Password, selecciona **Policy** y elige **Block** para:

       * Password en Login
       * Password en Signup
       * Self-service cambiar la contraseña (se actualiza automáticamente)

    7. Activa **Support users without a password**.
       <Warning>
         Es posible que recibas un error si no activas **Support users without a password**.
       </Warning>

    8. Selecciona **Save**. En el prompt, selecciona **Continue** para confirmar que entiendes que los usuarios existentes pueden verse afectados.

    9. Ve a la pestaña **Applications** y habilita la conexión para tu aplicación o API.
  </Tab>

  <Tab title="Management API" icon="code">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Para usar la Management API, necesitas un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).
    </Callout>

    Llama al endpoint [Create a Connection](https://auth0.com/docs/api/management/v2/connections/post-connections). Usa la opción `default_method` para establecer el primer método de autenticación para cada identificador.

    ```bash expandable lines
    curl --location 'https://YOUR_AUTH0_TENANT/api/v2/connections' \
    --header 'Authorization: Bearer YOUR_MANAGEMENT_API_TOKEN' \
    --header 'Accept: application/json, text/plain, */*' \
    --header 'Content-Type: application/json' \
    --data '{
      "name": "YOUR_CONNECTION_NAME",
      "strategy": "auth0",
      "is_domain_connection": false,
      "options": {
        "disable_signup": false,
        "attributes": {
          "email": {
            "identifier": {
              "active": true,
              "default_method": "email_otp"
            },
            "profile_required": true,
            "signup": {
              "status": "required",
              "verification": {
                "active": true
              }
            },
            "verification_method": "otp",
            "unique": true
          },
          "phone_number": {
            "identifier": {
              "active": true,
              "default_method": "password"
            },
            "profile_required": true,
            "signup": {
              "status": "required",
              "verification": {
                "active": true
              }
            }
          },
          "username": {
            "identifier": {
              "active": true
            },
            "profile_required": true,
            "signup": {
              "status": "required"
            },
            "validation": {
              "min_length": 1,
              "max_length": 15,
              "allowed_types": {
                "email": false,
                "phone_number": false
              }
            }
          }
        },
        "authentication_methods": {
          "password": {
            "enabled": true
          },
          "passkey": {
            "enabled": false
          },
          "email_otp": {
            "enabled": true
          },
          "phone_otp": {
            "enabled": true
          }
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="update-existing-connections">
  ## Actualizar conexiones existentes
</div>

Si ya tienes conexiones de base de datos, actualiza la configuración de passwordless en el Auth0 Dashboard o en la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard" icon="browser">
    1. **Abre la conexión**: Ve a [**Auth0 Dashboard &gt; Autenticación &gt; Base de datos**](https://manage.auth0.com/#/connections/database) y selecciona la conexión que quieres actualizar.
    2. **Activar atributos**: En la pestaña **Atributos**, selecciona **Activar** para habilitar la nueva configuración de atributos.
    3. **Agregar atributos de Email y Phone**: Elige **+ Agregar atributos** y agrega Email y Número de teléfono si todavía no están presentes.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         En una conexión passwordless (OTP) real, no se admite el identificador de nombre de usuario.
       </Callout>
    4. **Habilitar métodos de autenticación OTP**: En la pestaña **Métodos de autenticación**:
       * Configura Phone para **Permitir** OTP por teléfono y guarda los cambios.
       * Configura Email para **Permitir** Email OTP y guarda los cambios.
    5. **Bloquear la autenticación con contraseña**: En la configuración de Password, elige **Policy** y selecciona **Block** para:

       * Password on Login
       * Password on Signup
       * Self-service change password

       Activa **Support users without a password**.

       <Warning>
         Es posible que recibas un error si no activas **Support users without a password**.
       </Warning>
    6. **Guardar**: Selecciona **Guardar**.
  </Tab>

  <Tab title="Management API" icon="code">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Para usar la Management API, necesitas un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/get-management-api-access-tokens-for-production).
    </Callout>

    Llama al endpoint [actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id). Agrega la opción `default_method` para establecer el método de autenticación inicial. Para los atributos de Email, establece `password` o `email_otp`. Para los atributos de Phone, establece `password` o `phone_otp`.

    ```bash
    curl -L -X PATCH "https://YOUR_AUTH0_TENANT/api/v2/connections/<connection_id>" \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer YOUR_MANAGEMENT_API_TOKEN" \
    -d '{
      "options": {
        "passwordPolicy": "none",
        "authentication_methods": {
          "password": { "enabled": true },
          "passkey": { "enabled": true },
          "email_otp": { "enabled": true },
          "phone_otp": { "enabled": true }
        },
        "attributes": {
          "email": {
            "signup": {
              "status": "required",
              "verification": { "active": true }
            },
            "unique": true,
            "identifier": {
              "active": true,
              "default_method": "email_otp"
            },
            "profile_required": true,
            "verification_method": "otp"
          },
          "phone_number": {
            "signup": {
              "status": "required",
              "verification": { "active": true }
            },
            "identifier": {
              "active": true,
              "default_method": "phone_otp"
            },
            "profile_required": true
          }
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="use-auth0-actions">
  ## Usa Auth0 Actions
</div>

Para conocer mejor los factores de autenticación passwordless de las conexiones de base de datos, configura los atributos con Auth0 Actions.

<div id="post-login-trigger">
  ### Trigger de Post Login
</div>

El trigger `post-login` se activa después de que un usuario se haya autenticado, pero antes de que el servidor de autorización devuelva un token. El objeto `event.authentication` en [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post Login](https://manage.auth0.com/#/actions/flows) expone los siguientes métodos:

| Método                   | Parámetro | Descripción                                                                   |
| ------------------------ | --------- | ----------------------------------------------------------------------------- |
| Email OTP                | `email`   | Email OTP usado para autenticar al usuario como primer factor.                |
| OTP por teléfono con SMS | `sms`     | OTP por teléfono (SMS) usado para autenticar al usuario como primer factor.   |
| OTP por teléfono con voz | `tel`     | OTP por teléfono con voz usado para autenticar al usuario como primer factor. |
| Password                 | `pwd`     | Password usado para autenticar al usuario como primer factor.                 |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  `email_verified` y `phone_verified` se establecen automáticamente en el perfil del usuario cuando un usuario se autentica mediante Email OTP u OTP por teléfono. Si anteriormente usabas una Action de Post Login para establecer manualmente estos indicadores, puedes eliminar esa solución alternativa.
</Callout>

Usa `event.authentication` para:

* Detectar qué factor passwordless completó el usuario (`email`, `sms`, `tel` o `pwd`)
* Agregar claims personalizadas a los tokens según el método de autenticación
* Ejecutar lógica de forma condicional según cómo se autenticó el usuario

<div id="example">
  #### Ejemplo
</div>

El siguiente ejemplo lee `event.authentication.methods` para detectar qué factor de passwordless completó el usuario y lo agrega como una custom claim en el token de identidad.

```javascript
exports.onExecutePostLogin = async (event, api) => {
  if (!event.authentication?.methods || event.authentication.methods.length === 0) {
    return;
  }

  if (event.connection.name !== 'YOUR_AUTH0_CONNECTION') {
    return;
  }

  const firstFactor = event.authentication.methods[0];

  if (firstFactor.name === 'email') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'email_otp');
  } else if (firstFactor.name === 'sms' || firstFactor.name === 'tel') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'phone_otp');
  } else if (firstFactor.name === 'pwd') {
    api.idToken.setCustomClaim('https://your-app.com/auth_method', 'password');
  }
};
```

<div id="post-challenge-trigger">
  ### trigger posterior al desafío
</div>

El trigger `post-challenge` se ejecuta después de que los usuarios completan un desafío, como un restablecimiento de contraseña, la validación del número de teléfono o MFA. El objeto `event.authentication` en [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; password-reset-post-challenge](https://manage.auth0.com/#/actions/flows) expone los siguientes atributos:

| Atributo | Parámetro      | Descripción                                                |
| -------- | -------------- | ---------------------------------------------------------- |
| Email    | `email`        | Restablecimiento de contraseña con Email OTP o Magic Link. |
| Teléfono | `phone_number` | Restablecimiento de contraseña con OTP por teléfono.       |

<div id="example">
  #### Ejemplo
</div>

El siguiente ejemplo lee `event.authentication.methods` para detectar qué factor passwordless completó la verificación y lo añade como una custom claim en el token de identidad.

```javascript
exports.onExecutePostChallenge = async (event, api) => {
  if (!event.authentication?.methods || event.authentication.methods.length === 0) {
    return;
  }

  if (event.connection.name !== 'YOUR_AUTH0_CONNECTION') {
    return;
  }

  const firstFactor = event.authentication.methods[0];

  if (firstFactor.name === 'email') {
    api.idToken.setCustomClaim('https://your-app.com/challenge_method', 'email_otp');
  } else if (firstFactor.name === 'phone_number') {
    api.idToken.setCustomClaim('https://your-app.com/challenge_method', 'phone_otp');
  }
};
```

<div id="benefits">
  ## Beneficios
</div>

* **Implementación simplificada:** Menos conexiones que configurar y mantener. No es necesario vincular cuentas, salvo que se utilicen conexiones sociales/federadas junto con conexiones de base de datos.
* **Mejor experiencia de usuario**: Ofrece combinaciones de OTP por correo electrónico y por teléfono con contraseñas, claves de acceso e inicio de sesión social/federado, todo dentro de la misma experiencia de Universal Login basada primero en el identificador.
* **Flujos de registro flexibles**: Configura los atributos de correo electrónico y teléfono como opcionales durante el registro, para que los usuarios puedan registrarse solo con un correo electrónico o solo con un número de teléfono; ideal para experiencias centradas en dispositivos móviles o solo en correo electrónico.
* **OTP por voz**: El OTP por voz se incluye como primer factor cuando se configura en Unified Phone Experience.

<div id="limitations">
  ## Limitaciones
</div>

* Disponible solo para flujos basados en Universal Login; aún no es compatible con la autenticación basada en API.
* Passwordless para conexiones de base de datos no es compatible con Classic Login.
* Passwordless para conexiones de base de datos no admite Implicit Signup &amp; Login.
* En las conexiones passwordless heredadas, no había diferencias en la experiencia del usuario entre el registro y el inicio de sesión. Las conexiones de base de datos distinguen entre la experiencia de registro y la de inicio de sesión, lo que requiere un registro y un inicio de sesión *explícitos*.
  * Si un usuario sin una identidad de Auth0 entra en el flujo de inicio de sesión, el sistema no lo registrará automáticamente. El usuario recibirá un error después de validar el OTP.
  * Si un usuario con una identidad de Auth0 entra en el flujo de registro, recibirá un error después de validar el OTP.
