<Card title="Información general">
  Conceptos clave

  * Inicia un restablecimiento de contraseña con Auth0 Dashboard o la Management API.
</Card>

En este tema se describen distintas formas de restablecer la contraseña de un usuario en tu base de datos. Solo puedes cambiar contraseñas de usuarios de tus [conexiones de base de datos](/docs/es/authenticate/database-connections). Los usuarios que inician sesión con conexiones [sociales](/docs/es/authenticate/identity-providers/social-identity-providers) o [empresariales](/docs/es/authenticate/identity-providers/enterprise-identity-providers) deben restablecer sus contraseñas con el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> (como Google o Facebook), y las siguientes instrucciones solo funcionan si se conoce la dirección de correo electrónico del usuario.

Hay dos métodos básicos para cambiar la contraseña de un usuario:

* [Iniciar un flujo interactivo de restablecimiento de contraseña](#trigger-an-interactive-password-reset-flow) que envía al usuario un enlace por correo electrónico. El enlace abre la página de restablecimiento de contraseña de Auth0, donde el usuario puede introducir una contraseña nueva.
* [Establecer directamente la nueva contraseña](#directly-set-the-new-password) con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 o desde el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Al restablecer la contraseña de un usuario, su sesión caduca.
</Callout>

<Card title="¿No es lo que buscas?">
  * Para configurar la página personalizada de Restablecimiento de contraseña, consulta [Personalizar la página de Restablecimiento de contraseña](/docs/es/customize/login-pages/classic-login/customize-password-reset-page).
  * Para implementar un comportamiento personalizado después de un cambio de contraseña correcto, consulta [Actions Triggers: Posterior al cambio de contraseña](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-change-password-trigger).
  * Para restablecer la contraseña de tu cuenta personal de usuario de Auth0, consulta [Restablecer contraseñas de cuentas](/docs/es/troubleshoot/customer-support/reset-account-passwords).
</Card>

<div id="trigger-an-interactive-password-reset-flow">
  ## Inicia un flujo interactivo de restablecimiento de contraseña
</div>

Hay dos formas de iniciar un flujo interactivo de restablecimiento de contraseña, según tu caso de uso: a través de la página de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad del usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> o de la Authentication API.

<div id="universal-login-page">
  ### Página de Universal Login
</div>

Si su aplicación usa Universal Login, el usuario puede usar el widget de Lock en la pantalla de inicio de sesión para solicitar un correo electrónico de restablecimiento de contraseña. Con Universal Login, el usuario puede hacer clic en el enlace **¿No recuerda su contraseña?** y luego introducir su dirección de correo electrónico. Esto envía una solicitud POST a Auth0 que desencadena el proceso de restablecimiento de contraseña. El usuario [recibe un correo electrónico de restablecimiento de contraseña](#password-reset-email).

<div id="authentication-api">
  ### Authentication API
</div>

Si tu aplicación usa un flujo interactivo de restablecimiento de contraseña a través de la Authentication API, haz una llamada `POST`. En el campo `email`, proporciona la dirección de correo electrónico del usuario que necesita cambiar su contraseña. Si la llamada se realiza correctamente, el usuario [recibe un correo electrónico de restablecimiento de contraseña](#password-reset-email).

Si llamas a la API desde el navegador, asegúrate de que la URL de origen esté permitida:

Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications/\{yourClientId}/settings) y agrega la URL a la lista de **Allowed Origins (CORS)**.

Si tu conexión es una base de datos personalizada, comprueba si el usuario existe en la base de datos antes de invocar la Authentication API para `changePassword`.

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/dbconnections/change_password' \
    --header 'content-type: application/json' \
    --data '{"client_id": "{yourClientId}","email": "","connection": "Username-Password-Authentication"}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/dbconnections/change_password");
  var request = new RestRequest(Method.POST);
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{"client_id": "{yourClientId}","email": "","connection": "Username-Password-Authentication"}", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/dbconnections/change_password"

  	payload := strings.NewReader("{"client_id": "{yourClientId}","email": "","connection": "Username-Password-Authentication"}")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("content-type", "application/json")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse<String> response = Unirest.post("https://{yourDomain}/dbconnections/change_password")
    .header("content-type", "application/json")
    .body("{"client_id": "{yourClientId}","email": "","connection": "Username-Password-Authentication"}")
    .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'POST',
    url: 'https://{yourDomain}/dbconnections/change_password',
    headers: {'content-type': 'application/json'},
    data: {
      client_id: '{yourClientId}',
      email: '',
      connection: 'Username-Password-Authentication'
    }
  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP
  $curl = curl_init();

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/dbconnections/change_password",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_ENCODING => "",
    CURLOPT_MAXREDIRS => 10,
    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
    CURLOPT_CUSTOMREQUEST => "POST",
    CURLOPT_POSTFIELDS => "{"client_id": "{yourClientId}","email": "","connection": "Username-Password-Authentication"}",
    CURLOPT_HTTPHEADER => [
      "content-type: application/json"
    ],
  ]);

  $response = curl_exec($curl);
  $err = curl_error($curl);

  curl_close($curl);

  if ($err) {
    echo "cURL Error #:" . $err;
  } else {
    echo $response;
  }
  ```

  ```python Python
  import http.client

  conn = http.client.HTTPSConnection("")

  payload = "{"client_id": "{yourClientId}","email": "","connection": "Username-Password-Authentication"}"

  headers = { 'content-type': "application/json" }

  conn.request("POST", "/{yourDomain}/dbconnections/change_password", payload, headers)

  res = conn.getresponse()
  data = res.read()

  print(data.decode("utf-8"))
  ```

  ```ruby Ruby
  require 'uri'
  require 'net/http'
  require 'openssl'

  url = URI("https://{yourDomain}/dbconnections/change_password")

  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Post.new(url)
  request["content-type"] = 'application/json'
  request.body = "{"client_id": "{yourClientId}","email": "","connection": "Username-Password-Authentication"}"

  response = http.request(request)
  puts response.read_body
  ```

<div id="password-reset-email">
  ### Correo electrónico de restablecimiento de contraseña
</div>

Independientemente de cómo se haya iniciado el proceso de restablecimiento de contraseña, el usuario recibe un correo electrónico con un enlace para restablecer su contraseña.

<Frame>![Correo electrónico de restablecimiento de contraseña](/docs/images/cdy7uua7fh8z/5IBhcrCJ7XXI6OgQOnHMJa/33cdafbfe7a63a48cfaa2e58a12a2494/password-reset-email.png)</Frame>

Al hacer clic en el enlace, se lleva al usuario a la [página de restablecimiento de contraseña](/docs/es/customize/login-pages/classic-login/customize-password-reset-page).

Después de enviar la nueva contraseña, el usuario ve una confirmación de que ya puede iniciar sesión con sus nuevas credenciales.

Notas sobre los restablecimientos de contraseña:

* El enlace para restablecer la contraseña incluido en el correo electrónico solo es válido para un único uso.
* Si el usuario recibe varios correos electrónicos de restablecimiento de contraseña, solo será válido el enlace del correo más reciente.
* El campo **URL Lifetime** determina durante cuánto tiempo es válido el enlace. Desde el Auth0 Dashboard, puedes [personalizar el correo electrónico para cambiar la contraseña](/docs/es/customize/email/email-templates) y [modificar la duración del enlace](https://auth0.com/docs/api/authentication/reference#change-password).
* Puedes ampliar el flujo de restablecimiento de contraseña para incluir otro factor con Auth0 Actions. Para obtener más información, consulta [Flujo de restablecimiento de contraseña](/docs/es/customize/actions/explore-triggers/password-reset-triggers).

Con [Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience), puedes configurar una URL a la que redirigir a los usuarios después de completar el restablecimiento de contraseña. La URL recibe un indicador de éxito y un mensaje. Para obtener más información, consulta la sección &quot;Configuring the Redirect-To URL&quot; de [Customize Email Templates](/docs/es/customize/email/email-templates).

[Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) redirige al usuario a la [ruta predeterminada de inicio de sesión](/docs/es/authenticate/login/auth0-universal-login/configure-default-login-routes) cuando el proceso se completa correctamente, y gestiona los casos de error como parte del flujo de Universal Login. Esta experiencia ignora la URL de redirección de la plantilla de correo electrónico.

<Card title="Generar tickets de restablecimiento de contraseña">
  La Management API proporciona el endpoint [Create a Password Change Ticket](https://auth0.com/docs/api/management/v2#!/Tickets/post_password_change), que genera una URL como la del correo electrónico de restablecimiento de contraseña. Puedes usar la URL generada cuando el correo electrónico no sea un método de entrega adecuado. Ten en cuenta que, en el flujo predeterminado, la entrega de correos electrónicos verifica la identidad del usuario. (Un impostor no tendría acceso a la bandeja de entrada del correo electrónico). Si usas la URL del ticket, tu aplicación será responsable de verificar la identidad del usuario de alguna otra forma.
</Card>

<div id="directly-set-the-new-password">
  ## Establecer directamente la nueva contraseña
</div>

Para establecer directamente una nueva contraseña para el usuario sin enviar un correo electrónico de restablecimiento de contraseña, usa la [Management API](#use-the-management-api) o el [Auth0 Dashboard](#manually-set-users-passwords-using-the-auth0-dashboard).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los usuarios no reciben ninguna notificación cuando se les cambia la contraseña.
</Callout>

<div id="use-the-management-api">
  ### Usa la Management API
</div>

Si quieres implementar tu propio flujo de restablecimiento de contraseña, puedes cambiar directamente la contraseña de un usuario enviando una solicitud desde el servidor a la Management API: haz una llamada `PATCH` al [endpoint Update a User](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si usas la Management API para establecer o actualizar la contraseña de un usuario mediante el [endpoint Update a User](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id), se aplicará la política de seguridad de contraseñas que hayas configurado en Auth0 Dashboard.
</Callout>

<div id="manually-set-users-passwords-using-the-auth0-dashboard">
  ### Cambiar manualmente la contraseña de los usuarios desde el Auth0 Dashboard
</div>

Cualquier persona con privilegios administrativos en tu tenant de Auth0 puede cambiar manualmente la contraseña de un usuario en [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).

1. Selecciona el nombre del usuario cuya contraseña quieres cambiar.
2. Busca la **Danger Zone** en la parte inferior de la página.
3. En el recuadro rojo **Change Password**, selecciona **Change**.

   <Frame>![Cambiar la contraseña manualmente](/docs/images/cdy7uua7fh8z/5mrEG3UtlZW47AnTikwIGv/94d186b748a53c13145388fa281af8c4/dashboard-users-edit_view-details_danger-zone__1_.png)</Frame>
4. Introduce la nueva contraseña y selecciona **Save**.
