Si utilizas la estructura de base de datos alojada por Auth0 para almacenar tus usuarios, puedes probar la conexión autenticando a los usuarios en el mismo tenant o en uno distinto.

<div id="test-with-import-users-to-auth0-enabled">
  ## Probar con Import Users to Auth0 habilitado
</div>

Cuando la importación de usuarios está habilitada, los usuarios se migran gradualmente desde su repositorio externo de usuarios al repositorio de usuarios de Auth0. Durante este proceso, la primera vez que inicien sesión correctamente, los usuarios se autenticarán en el repositorio externo. Todos los intentos de inicio de sesión posteriores se realizarán en el repositorio de usuarios de Auth0. Para obtener más información, consulte las migraciones automáticas en [Import and Export Users](/docs/es/manage-users/user-migration).

<div id="create-a-test-tenant-and-application">
  ### Crear un tenant y una aplicación de prueba
</div>

1. Usa el Auth0 Dashboard para [crear un nuevo tenant](/docs/es/get-started/auth0-overview/create-tenants).
2. Ve a [Applications &gt; Applications](https://manage.auth0.com/#/applications/applications) y crea una aplicación de [Machine-to-Machine](https://manage.auth0.com/#/applications/applications/create).

   <Warning>
     Debes anotar el **ID de cliente** y el **secreto de cliente** al crear los scripts de base de datos personalizados. También debes anotar el **ID de cliente** para habilitar scopes en Management API.
   </Warning>
3. Habilita tanto la concesión **Password** como la **Client Credential** para esta aplicación (ambas concesiones deben estar habilitadas para esta prueba).

   <Frame>![Auth0 Dashboard > Applications > Advanced Settings](/docs/images/cdy7uua7fh8z/4caAveuhYurCDb5F8mdImm/38496efe6f670d828542b02a56e5ca64/Grant_Types_-_English.png)</Frame>
4. Para autorizar tu aplicación, ve a [Applications &gt; APIs](https://manage.auth0.com/#/apis).
5. Selecciona **Management API**.
6. En la pestaña **Machine-to-Machine Applications**, usa el toggle para autorizar tu aplicación de prueba.
7. Selecciona el menú desplegable para habilitar los siguientes scopes de Auth0 Management API:

   * `read:users`
   * `update:users`
   * `delete:users`
   * `create:users`
   * `read:users_app_metadata`
   * `update:users_app_metadata`
   * `create:users_app_metadata`

     <Frame>![Auth0 Dashboard > APIs > Management API >  M2M Tab > App Permissions](/docs/images/cdy7uua7fh8z/2vW8zD1iIObuqb0i59yw77/c43bafa05b271a4343af51cb5c8ce1ce/2024-03-14_10-28-19.png)</Frame>

<div id="create-test-database-connections">
  ### Crear conexiones de base de datos de prueba
</div>

Después de crear un tenant y una aplicación en el Dashboard, crea una conexión de base de datos de origen y una conexión de base de datos de destino.

1. Ve a [Authentication &gt; Database](https://manage.auth0.com/#/connections/database) para crear una nueva conexión de base de datos que será la de origen.
2. Asigna un nombre a tu conexión de origen de prueba, habilita **Requires Username** y selecciona **Create**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Puedes crear usuarios para usarlos en tu prueba en [User Management &gt; Users](https://manage.auth0.com/#/users). Después de seleccionar **Create** **Users** y completar los campos necesarios, selecciona la base de datos de origen en el campo **Connection**.
   </Callout>
3. Crea una segunda base de datos como destino con la misma configuración del paso 2.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Puedes establecer **Password Policy** en **Non-empty password required** tanto en la base de datos de destino como en la de origen para usar contraseñas simples en tus pruebas.
   </Callout>
4. En tu base de datos de destino, cambia a la vista **Custom Database** y activa **Use my own database**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Si quieres hacer pruebas con Organizations usando tus bases de datos personalizadas, habilita **Context object in database scripts**.
   </Callout>
5. Cambia a la vista **Settings** y habilita **Import Users to Auth0**.
6. Cambia a la vista **Custom Database** y luego busca la sección **Database settings**. Agrega la siguiente información de tu base de datos de origen creada en el paso 1:

| Key               | Value                                                                     |
| ----------------- | ------------------------------------------------------------------------- |
| `client_id`       | ID de cliente de la aplicación que creaste.                               |
| `client_secret`   | secreto de cliente de la aplicación que creaste.                          |
| `auth0_domain`    | El nombre de tu tenant en el dominio de Auth0: `yourTenant.us.auth0.com`. |
| `source_database` | Nombre de la conexión de origen.                                          |

7. Actualiza los scripts de acción de base de datos [Login](/docs/es/authenticate/database-connections/custom-db/test-custom-database-connections#login) y [Get User](/docs/es/authenticate/database-connections/custom-db/test-custom-database-connections#get-user) en tu base de datos de destino. Para obtener más información sobre las prácticas recomendadas para usar scripts de acción de base de datos, consulta [Custom Database Connection and Action Scripts Best Practices](/docs/es/authenticate/database-connections/custom-db/custom-database-connections-scripts).
8. Selecciona **Save and Try** en cada script. Debes supervisar la salida de `console.log` en los logs en tiempo real de Actions. Para obtener más información, consulta [Actions Real-time Logs](/docs/es/customize/actions/actions-real-time-logs).
9. Selecciona **Try Connection** para probar la conexión en tiempo real.

<div id="test-without-import-users-enabled">
  ## Probar sin tener habilitado Import Users to Auth0
</div>

1. Repite los pasos de [Crear un tenant y una aplicación de prueba](/docs/es/authenticate/database-connections/custom-db/test-custom-database-connections#create-a-test-tenant-and-application).
2. Crea una conexión de base de datos de prueba.
3. Asegúrate de que **Import Users to Auth0** esté deshabilitado en la configuración de la base de datos de origen. Los usuarios se autenticarán en el repositorio de usuarios externo durante cada intento de inicio de sesión.
4. Actualiza todos los scripts de las acciones de base de datos con los ejemplos que aparecen a continuación.

<div id="get-user-script">
  ### Script de Get User
</div>

El script [Get User](/docs/es/authenticate/database-connections/custom-db/templates/get-user) implementa una función ejecutable que determina el estado actual de un usuario.

Cuando **Import Users to Auth0** está habilitado, el script Get User se ejecuta cuando un usuario intenta registrarse para comprobar si el usuario ya existe en el repositorio externo de usuarios.

El script Get User también se ejecuta cuando un usuario intenta:

* cambiar el correo electrónico de un usuario (script [cambiar correo electrónico](/docs/es/authenticate/database-connections/custom-db/templates/change-email))
* iniciar sesión (script [Login](/docs/es/authenticate/database-connections/custom-db/templates/login))
* cambiar la contraseña de un usuario (script [Change Password](/docs/es/authenticate/database-connections/custom-db/templates/change-password))

Cuando **Import Users to Auth0** está deshabilitado, el script Get User se ejecuta cuando un usuario intenta registrarse para comprobar si el usuario ya existe en el repositorio externo de usuarios. Si un usuario ya existe en el repositorio externo de usuarios, no se ejecutará el script Create.

El script Get User también se ejecuta cuando un usuario intenta:

* crear un usuario (script [Create](/docs/es/authenticate/database-connections/custom-db/templates/create))
* cambiar el correo electrónico de un usuario (script [cambiar correo electrónico](/docs/es/authenticate/database-connections/custom-db/templates/change-email))
* cambiar la contraseña de un usuario (script [Change Password](/docs/es/authenticate/database-connections/custom-db/templates/change-password))

<div id="example">
  #### Ejemplo
</div>

```js lines expandable
async function getUser(user, context, callback) {
    log(`Script started.`);
    log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
    let accessToken = await getAccessToken();
    accessToken = accessToken.access_token;
    if (!accessToken) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);
    log(`The Access Token is available. Searching for user "${user}" in "${configuration.source_database}"`);
    user = user.toLowerCase();
    const searchQuery = encodeURI(`identities.connection:"${configuration.source_database}"+AND+(email:${user} OR username:${user})`);
    var options = {
        method: `GET`,
        url: `https://${configuration.auth0_domain}/api/v2/users?q=${searchQuery}`,
        headers: {
            Authorization: `Bearer ${accessToken}`,
        }
    };
    request(options, function (error, response) {
        if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
        let search_results = JSON.parse(response.body);
        let profile = null;
        if (search_results.length > 0) {
            log(`A user "${user}" is FOUND in "${configuration.source_database}" database.`);
            profile = {
                user_id: search_results[0].user_id.toString(),
                nickname: search_results[0].nickname,
                username: search_results[0].username,
                email: search_results[0].email
            };
        } else {
            log(`A user "${user}" is NOT FOUND in "${configuration.source_database}" database.`);
        }
        log(`Script completed!`);
        return callback(null, profile);
    });
    /* -- OBTENER TOKEN DE ACCESO VÍA CREDENCIALES DE CLIENTE -- */
    async function getAccessToken() {
        var options = {
            method: `POST`,
            url: `https://${configuration.auth0_domain}/oauth/token`,
            headers: {
                "Content-Type": `application/x-www-form-urlencoded`,
            },
            form: {
                grant_type: `client_credentials`,
                client_id: configuration.client_id,
                client_secret: configuration.client_secret,
                audience: `https://${configuration.auth0_domain}/api/v2/`
            },
            json: true
        };
        return new Promise(function (resolve) {
            request(options, function (error, response) {
                resolve(error || response.body);
            });
        });
    }
    /* -- REGISTRO -- */
    function log(message, error = false) {
        const script_name = `GET USER`;
        const error_label = error ? `(ERROR)` : ``;
        const return_message = `${script_name}: ${error_label} ${message}`;
        console.log(return_message);
        if (error) return callback(new Error(return_message));
    }
}
```

<div id="login-script">
  ### Script de inicio de sesión
</div>

El script [Login](/docs/es/authenticate/database-connections/custom-db/templates/login) implementa una función ejecutable que autentica a un usuario cuando un usuario inicia sesión. Si el usuario existe en la base de datos de destino (Auth0), lo autentica mediante ese registro. De lo contrario, autentica al usuario mediante su registro en la base de datos de origen (externa).

<div id="example">
  #### Ejemplo
</div>

```javascript lines expandable
function login(usernameOrEmail, password, context, callback) {
  log(`Script started.`);
  const jwt = require('jsonwebtoken');
  const options = {
    method: `POST`,
    url: `https://${configuration.auth0_domain}/oauth/token`,
    headers: { "Content-Type": `application/x-www-form-urlencoded` },
    json: true,
    form: {
      grant_type: `http://auth0.com/oauth/grant-type/password-realm`,
      client_id: configuration.client_id,
      client_secret: configuration.client_secret,
      username: usernameOrEmail,
      password: password,
      realm: `${configuration.source_database}`
    }
  };
  request(options, function (error, response, body) {
    log(`Attempting to authenticate a user "${usernameOrEmail}" against "${configuration.source_database}" database in "${configuration.auth0_domain}" tenant.`);
    if (error) return log(`Cannot connect to "${configuration.auth0_domain}" database.`, true);
    if (response.statusCode !== 200) {
      console.log(`LOGIN: (ERROR) ${response.body.error_description}`);
      return callback(new WrongUsernameOrPasswordError(usernameOrEmail, `LOGIN: (ERROR) ${response.body.error_description}`));
    }
    log(`Successfuly authenticated user "${usernameOrEmail}" against "${configuration.source_database}" database in "${configuration.auth0_domain}" tenant.`);
    const decoded_id_token = jwt.decode(body.id_token);
    const profile = {
      user_id: decoded_id_token.sub,
      nickname: decoded_id_token.nickname,
      username: decoded_id_token.username,
      email: decoded_id_token.email
    };
    log(`Script completed.`);
    return callback(null, profile);
  });
  /* -- REGISTRO -- */
  function log(message, error = false) {
    const script_name = `LOGIN`;
    const error_label = error ? `(ERROR)` : ``;
    const return_message = `${script_name}: ${error_label} ${message}`;
    console.log(return_message);
    if (error) return callback(new Error(return_message));
  }
}
```

<div id="create-script">
  ### Script Create
</div>

El [script Create](/docs/es/authenticate/database-connections/custom-db/templates/create) implementa una función ejecutable que crea el registro de usuario correspondiente en la base de datos externa cuando un usuario se registra mediante <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>, o se crea en el <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0.

<div id="example">
  #### Ejemplo
</div>

```js lines expandable
async function create(user, context, callback) {
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
  let accessToken = await getAccessToken();
  if (!accessToken.access_token) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);
  accessToken = accessToken.access_token;
  log(`The Access Token is available. Attempting to create a user "${user.email}" in "${configuration.source_database}"`);
  const options = {
    method: `POST`,
    url: `https://${configuration.auth0_domain}/api/v2/users`,
    headers: {
      Authorization: `Bearer ${accessToken}`,
      "Content-Type": `application/x-www-form-urlencoded`
    },
    form: {
      connection: configuration.source_database,
      email: user.email,
      password: user.password,
      username: user.username
    },
    json: true
  };
  request(options, function (error, response) {
    if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
    switch (response.statusCode) {
      case 201:
        log(`The user "${user.email}" is successfuly created in "${configuration.source_database}" database.`);
        return callback(null);
      case 409:
        return callback(new ValidationError(`user_exists`, `The user already exists in "${configuration.source_database}" database.`));
      default:
        return log(`Failed to create a user "${user.email}" in "${configuration.source_database}" database. Error: "${response.statusCode}, ${response.body.message}"`, true);
    }
  });
  /* -- OBTENER TOKEN DE ACCESO VÍA CREDENCIALES DE CLIENTE -- */
  async function getAccessToken() {
    var options = {
      method: `POST`,
      url: `https://${configuration.auth0_domain}/oauth/token`,
      headers: {
        "Content-Type": `application/x-www-form-urlencoded`,
      },
      form: {
        grant_type: `client_credentials`,
        client_id: configuration.client_id,
        client_secret: configuration.client_secret,
        audience: `https://${configuration.auth0_domain}/api/v2/`
      },
      json: true
    };
    return new Promise(function (resolve) {
      request(options, function (error, response) {
        resolve(error || response.body);
      });
    });
  }
  /* -- REGISTRO -- */
  function log(message, error = false) {
    const script_name = `CREATE`;
    const error_label = error ? `(ERROR)` : ``;
    const return_message = `${script_name}: ${error_label} ${message}`;
    console.log(return_message);
    if (error) return callback(new Error(return_message));
  }
}
```

<div id="verify-script">
  ### Script Verify
</div>

El [script Verify](/docs/es/authenticate/database-connections/custom-db/templates/verify) implementa una función ejecutable que marca el estado de verificación de la dirección de correo electrónico de un usuario en la base de datos externa cuando el usuario hace clic en el enlace del correo de verificación enviado por Auth0.

<div id="example">
  #### Ejemplo
</div>

```js lines expandable
async function verify(user, context, callback) {
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
  let accessToken = await getAccessToken();
  if (!accessToken.access_token) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);

    headers: {
      Authorization: `Bearer ${accessToken}`,
    }
  };
  request(options, function (error, response) {
    if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
    let search_results = JSON.parse(response.body);
    if (search_results.length > 0) {
      log(`A user "${user}" is found in "${configuration.source_database}" database.`);
```

<div id="change-password-script">
  ### Script Change Password
</div>

El [script Change Password](/docs/es/authenticate/database-connections/custom-db/templates/change-password) implementa una función ejecutable que actualiza la contraseña de un usuario en la base de datos externa cuando un flujo de cambio de contraseña se inicia desde el Auth0 Dashboard o la Auth0 Management API.

<div id="example">
  #### Ejemplo
</div>

```js lines expandable
async function changePassword(user, newPassword, context, callback) {
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
  let accessToken = await getAccessToken();
  if (!accessToken.access_token) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);
  accessToken = accessToken.access_token;
  log(`The Access Token is available. Searching for user "${user}" in "${configuration.source_database}" database.`);
  user = user.toLowerCase();
  const searchQuery = encodeURI(`identities.connection:"${configuration.source_database}"+AND+(email:${user} OR username:${user})`);
  var options = {
    method: `GET`,
    url: `https://${configuration.auth0_domain}/api/v2/users?q=${searchQuery}`,
    headers: {
      Authorization: `Bearer ${accessToken}`,
    }
  };
  request(options, function (error, response) {
    if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
    let search_results = JSON.parse(response.body);
    if (search_results.length > 0) {
      log(`A user "${user}" is found in "${configuration.source_database}" database.`);
      const user_id = search_results[0].user_id.toString();
      log(`Attempting to change password for user "${user_id}" in "${configuration.source_database}" database.`);
      const options = {
        method: `PATCH`,
        url: `https://${configuration.auth0_domain}/api/v2/users/${user_id}`,
        headers: {
          Authorization: `Bearer ${accessToken}`,
          "Content-Type": `application/x-www-form-urlencoded`
        },
        form: {
          password: newPassword,
        },
        json: true
      };
      request(options, function (error, response) {
        if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
        switch (response.statusCode) {
          case 200:

  });

  async function getAccessToken() {

      url: `https://${configuration.auth0_domain}/oauth/token`,
```

<div id="change-email-script">
  ### Script de cambiar correo electrónico
</div>

El script [cambiar correo electrónico](/docs/es/authenticate/database-connections/custom-db/templates/change-email) implementa una función ejecutable que actualiza la dirección de correo electrónico de un usuario cuando este cambia su dirección de correo electrónico o el estado de verificación de esta.

Este script no está disponible en el Auth0 Dashboard. Debe llamar al endpoint de Management API [actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) y proporcionar la propiedad `options.customScripts.change_email`.

<div id="example">
  #### Ejemplo
</div>

```js lines expandable
async function changeEmail(user, newEmail, verified, callback) {
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
  let accessToken = await getAccessToken();
  if (!accessToken.access_token) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);
  accessToken = accessToken.access_token;
  log(`The Access Token is available. Searching for user "${user}" in "${configuration.source_database}" database.`);
  user = user.toLowerCase();
  const searchQuery = encodeURI(`identities.connection:"${configuration.source_database}"+AND+(email:${user} OR username:${user})`);
  var options = {
    method: `GET`,
    url: `https://${configuration.auth0_domain}/api/v2/users?q=${searchQuery}`,
    headers: {
      Authorization: `Bearer ${accessToken}`,
    }
  };
  request(options, function (error, response) {
    if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
    let search_results = JSON.parse(response.body);
    if (search_results.length > 0) {
      log(`A user "${user}" is found in "${configuration.source_database}" database.`);
      const user_id = search_results[0].user_id.toString();
      log(`Attempting to change email / verified status for user "${user_id}" in "${configuration.source_database}" database.`);
      const options = {
        method: `PATCH`,
        url: `https://${configuration.auth0_domain}/api/v2/users/${user_id}`,
        headers: {
          Authorization: `Bearer ${accessToken}`,
          "Content-Type": `application/x-www-form-urlencoded`
        },
        form: {
          email: newEmail,
          email_verified: verified || false
        },
        json: true
      };
      request(options, function (error, response) {
        if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
        switch (response.statusCode) {
          case 200:
            log(`The user "${user}" email / verified status successfully changed in "${configuration.source_database}" database.`);
            return callback(null, true);
          default:
            return log(`Failed to change email / verified status for "${user}"  in "${configuration.source_database}" database. Error: "${response.statusCode}, ${response.body.message}"`, true);
        }
      });
    } else {
      log(`A user "${user}" is not found in "${configuration.source_database}" database. Unable to change email / verified status.`, true);
    }
  });
  /* -- OBTENER ACCESS TOKEN VIA CREDENCIALES DE CLIENTE -- */
  async function getAccessToken() {
    var options = {
      method: `POST`,
      url: `https://${configuration.auth0_domain}/oauth/token`,
      headers: {
        "Content-Type": `application/x-www-form-urlencoded`,
      },
      form: {
        grant_type: `client_credentials`,
        client_id: configuration.client_id,
        client_secret: configuration.client_secret,
        audience: `https://${configuration.auth0_domain}/api/v2/`
      },
      json: true
    };
    return new Promise(function (resolve) {
      request(options, function (error, response) {
        resolve(error || response.body);
      });
    });
  }
  /* -- REGISTRO -- */
  function log(message, error = false) {
    const script_name = `CHANGE EMAIL`;
    const error_label = error ? `(ERROR)` : ``;
    const return_message = `${script_name}: ${error_label} ${message}`;
    console.log(return_message);
    if (error) return callback(new Error(return_message));
  }
}
```
