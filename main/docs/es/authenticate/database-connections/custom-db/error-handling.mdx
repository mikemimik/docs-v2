Es posible que recibas errores de bases de datos personalizadas. También puedes devolver errores desde los scripts de acción de tu conexión de base de datos personalizada para solucionar problemas.

<div id="custom-database-error-types">
  ## Tipos de errores de base de datos personalizada
</div>

Las conexiones de base de datos pueden devolver los siguientes errores:

| Error                                                                               | Script de inicio de sesión            | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ----------------------------------------------------------------------------------- | ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `new WrongUsernameOrPasswordError(`<br />`  <email or user_id>, <message>`<br />`)` | Inicio de sesión                      | Se produce cuando las credenciales del usuario no son válidas. Si no se pasa ningún argumento o si el primer argumento tiene un valor falso, el error se registrará como un evento `fu` (nombre de usuario/correo electrónico no válido) con una cadena vacía como `user_id`. Si se proporciona un primer argumento con un valor verdadero, el error se registrará como un evento `fp` (el usuario existe, pero la contraseña no es válida) con un valor de `user_id` de `auth0\|<first_argument>`. Para obtener más información, consulta [Log Event Type Codes](/docs/es/deploy-monitor/logs/log-event-type-codes). |
| `new ValidationError(`<br />`  <user_exists>, <message>`<br />`)`                   | Crear                                 | Se produce cuando un usuario ya existe en tu base de datos.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `callback(<null>,<false>)`                                                          | Cambiar contraseña                    | Se produce cuando la contraseña del usuario no se actualizó.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `callback(<null>)`                                                                  | Obtener usuario                       | Se produce cuando no se encuentra al usuario.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `new UnauthorizedError(<message>)`                                                  | Todos los scripts de inicio de sesión | Se produce cuando algo sale mal al intentar acceder a la base de datos. Es la forma preferida de comunicar errores desde bases de datos personalizadas al front-end.                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `new Error(<message>)`                                                              | Todos los scripts de inicio de sesión | Se produce cuando algo sale mal al intentar acceder a la base de datos.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| `error code: auth0_idp_error;` `user already exists`                                | Crear                                 | Se produce en el almacén de identidades heredado cuando intentas crear un usuario que ya existe en tu almacén de identidades heredado o en Auth0.                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `error code: auth0_idp_error;` `user already exists`                                | Eliminar                              | Se produce en el almacén de identidades heredado cuando intentas eliminar un usuario con scripts de base de datos personalizada. El error también puede aparecer como &quot;Error! There was a problem deleting `user@example.com`&quot; o &quot;Sandbox Error: Please implement the Delete script for this database connection at `https://manage.auth0.com/#/connections/database`&quot;.                                                                                                                                                                                                                       |

<div id="return-errors">
  ## Devolver errores
</div>

Para devolver un error, llama a la función `callback()` y pasa `error` como primer argumento:

```js lines
callback(error);
```

Por ejemplo:

```js wrap lines
callback(new ValidationError('email-too-long', 'Email is too long.'));
```

<div id="return-errors-when-using-lock">
  ### Devolver errores al usar Lock
</div>

Si usas Lock, puedes [personalizar los mensajes de error](/docs/es/customize/login-pages/classic-login/customize-lock-error-messages) añadiéndolos al diccionario.

<div id="troubleshoot-errors">
  ## Solucionar errores
</div>

Prueba el script con el botón **Try**.

Si no obtienes el resultado esperado o recibes un error, usa [Actions Real-time Logs](/docs/es/customize/actions/actions-real-time-logs), añade sentencias `console.log()` a tu script y vuelve a probar la conexión. La salida de `console.log()` se muestra en la ventana de Actions Real-time Logs.

Es posible que se produzcan errores al usar scripts de acción para importar, crear o eliminar usuarios. Usa `console.logs` para comprobar si el error se debe al uso de scripts de acción. Para obtener más información, consulta [Custom Database Action Script Templates](/docs/es/authenticate/database-connections/custom-db/templates).
