Es posible que reciba errores de Custom Databases. También puede devolver errores desde los scripts de acción de su conexión de base de datos personalizada para solucionar problemas.

<div id="custom-database-error-types">
  ## Tipos de errores de Custom Database
</div>

Las conexiones de base de datos pueden devolver los siguientes errores:

| Error                                                                               | script de inicio de sesión            | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ----------------------------------------------------------------------------------- | ------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `new WrongUsernameOrPasswordError(`<br />`  <email or user_id>, <message>`<br />`)` | Login                                 | Ocurre cuando las credenciales del usuario no son válidas. Si no se pasa ningún argumento o el primer argumento es falsey, el error se registrará como un evento `fu` (nombre de usuario/correo electrónico no válido) con una cadena vacía para `user_id`. Si se proporciona un primer argumento truthy, el error se registrará como un evento `fp` (el usuario existe, pero la contraseña no es válida) con un valor de `user_id` de `auth0\|<first_argument>`. Para obtener más información, consulta [Log Event Type Codes](/docs/es/deploy-monitor/logs/log-event-type-codes). |
| `new ValidationError(`<br />`  <user_exists>, <message>`<br />`)`                   | Create                                | Ocurre cuando un usuario ya existe en tu base de datos.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `callback(<null>,<false>)`                                                          | Change Password                       | Ocurre cuando la contraseña del usuario no se actualizó.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `callback(<null>)`                                                                  | Get User                              | Ocurre cuando no se encuentra al usuario.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `new UnauthorizedError(<message>)`                                                  | Todos los scripts de inicio de sesión | Ocurre cuando algo salió mal al intentar acceder a la base de datos. Es la forma preferida de comunicar errores desde las Custom Databases al front-end.                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `new Error(<message>)`                                                              | Todos los scripts de inicio de sesión | Ocurre cuando algo salió mal al intentar acceder a la base de datos.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| `error code: auth0_idp_error;` `user already exists`                                | Create                                | Ocurre en un almacén de identidades heredado cuando intentas crear un usuario que ya existe en tu almacén de identidades heredado o en Auth0.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `error code: auth0_idp_error;` `user already exists`                                | Delete                                | Ocurre en un almacén de identidades heredado cuando intentas eliminar un usuario con Custom Database Scripts. El error también puede mostrarse como &quot;Error! There was a problem deleting `user@example.com`&quot; o &quot;Sandbox Error: Please implement the Delete script for this database connection at `https://manage.auth0.com/#/connections/database`&quot;.                                                                                                                                                                                                        |

<div id="return-errors">
  ## Devolver errores
</div>

Para devolver un error, llama a la función `callback()` y pasa `error` como primer parámetro:

```js lines
callback(error);
```

Por ejemplo:

```js wrap lines
callback(new ValidationError('email-too-long', 'Email is too long.'));
```

<div id="return-errors-when-using-lock">
  ### Devolver errores al usar Lock
</div>

Si usas Lock, puedes [personalizar los mensajes de error](/docs/es/customize/login-pages/classic-login/customize-lock-error-messages) añadiéndolos al diccionario.

<div id="troubleshoot-errors">
  ## Solucionar errores
</div>

Prueba el script con el botón **Try**.

Si no obtienes el resultado esperado o recibes un error, usa [logs en tiempo real de Actions](/docs/es/customize/actions/actions-real-time-logs), añade instrucciones `console.log()` en tu script y vuelve a probar la conexión. La salida de `console.log()` se muestra en la ventana de logs en tiempo real de Actions.

Es posible que se produzcan errores al usar scripts de acción para importar, crear o eliminar usuarios. Usa `console.logs` para verificar que el error se debe al uso de scripts de acción. Para obtener más información, consulta [Custom Database Action Script Templates](/docs/es/authenticate/database-connections/custom-db/templates).
