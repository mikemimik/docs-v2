<div id="access-legacy-identity-storage-via-custom-api">
  ## Accede al almacenamiento de identidad heredado mediante una API personalizada
</div>

Te recomendamos implementar una API para aplicar el principio de mínimo privilegio a tu almacenamiento de identidad heredado, en lugar de simplemente abrir un acceso general a través de internet.

Proteger el almacenamiento de identidad heredado frente al acceso general es una práctica recomendada. Exponer directamente a internet una base de datos de identidad heredada, por ejemplo, puede resultar extremadamente problemático: las interfaces de bases de datos para SQL y tecnologías similares ofrecen una funcionalidad excesivamente amplia, lo que vulnera el principio de mínimo privilegio en términos de seguridad.

La alternativa es crear una API (personalizada) sencilla, protegida mediante el uso de un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>, a la que pueda llamar cada script de acción. Esta actuaría como interfaz para el almacén heredado de identidades. Después, puede usarse el flujo de concesión de credenciales de cliente para obtener un token de acceso desde un script, que posteriormente puede almacenarse en caché para reutilizarse dentro del objeto `global` y así mejorar el rendimiento. La API puede, a su vez, ofrecer un número limitado de endpoints protegidos que realicen únicamente la funcionalidad de gestión de identidad heredada necesaria (p. ej., leer un usuario, cambiar la contraseña).

De forma predeterminada, Auth0 te dará un token para cualquier API si te autenticas correctamente e incluyes la <Tooltip tip="Audience: identificador único de la audiencia de un token emitido. Llamado aud en un token, su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip> adecuada. Restringir el acceso a la API del almacén heredado de identidades limitando la asignación de tokens de acceso mediante el uso de una rule evitará el uso no autorizado y mitigará varios escenarios de ataque, como aquellos en los que se intercepta la redirección a `/authorize` y se añade la audiencia de la API.

<Warning>
  Restringir el acceso a la API mediante Rule mitigará escenarios de ataque, como aquellos en los que se intercepta la redirección a `/authorize` y se añade la audiencia de la API, y garantizará que solo se conceda acceso mediante credenciales de cliente específicas.
</Warning>

<div id="access-to-legacy-identity-storage">
  ## Acceso al almacenamiento de identidad heredado
</div>

Tanto si administras el acceso al almacenamiento de identidad heredado mediante una API personalizada como si usas la interfaz nativa proporcionada, debes restringir el acceso a la lista de direcciones IP asociadas a tu tenant de Auth0. Para ver la lista de direcciones IP, consulta [Direcciones IP de Auth0 para listas de permitidos](/docs/es/secure/security-guidance/data-security/allowlist). Añadir URL a la lista de permitidos limita el acceso al almacén heredado de identidades y garantiza que solo se permitan los scripts de acciones de la base de datos personalizada definidos en Auth0.

<Warning>
  La lista de permitidos de direcciones IP de Auth0 se comparte entre todos los tenants de Auth0 definidos en una región. Nunca uses la lista de permitidos como único método para proteger el acceso a tu almacén heredado de identidades; hacerlo podría generar posibles vulnerabilidades de seguridad que permitan acceso no autorizado a tus usuarios.
</Warning>
