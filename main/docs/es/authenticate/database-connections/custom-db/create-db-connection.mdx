Si tienes tu propia base de datos de usuarios, puedes usarla como <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> en Auth0 para autenticar usuarios. Sigue el proceso para crear la conexión de base de datos personalizada, crear scripts de acción de base de datos y añadir parámetros de configuración.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Asegúrate de que tu base de datos tenga los campos adecuados para almacenar [atributos del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/user-profile-structure#user-profile-attributes), como `id`, `nickname`, `email` y `password`. Para conocer el esquema del perfil de usuario de Auth0, consulta [Perfiles de usuario normalizados](/docs/es/manage-users/user-accounts/user-profiles/normalized-user-profiles). Para obtener más información sobre cómo actualizar perfiles de usuario, consulta [Actualizar perfiles de usuario usando tu base de datos](/docs/es/manage-users/user-accounts/user-profiles/update-user-profiles-using-your-database).
</Callout>

Auth0 te permite crear conexiones y scripts para la mayoría de las bases de datos más utilizadas, incluidas:

* ASP.NET Identity
* MongoDB
* MySQL
* PostgreSQL
* SQLServer
* Windows Azure SQL Database
* Servicios web a los que se accede mediante Basic Auth

Puedes conectarte a cualquier tipo de base de datos o servicio web con un script personalizado correctamente configurado. Ten en cuenta que la sanitización de las entradas debe realizarse del lado del cliente; Auth0 no sanitiza ni valida ninguna combinación de nombre de usuario/contraseña enviada por una base de datos personalizada. Los scripts de Custom Database están sujetos a la [Política de límite de tasa](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy) de Auth0.

<Card title="Firewall de red">
  Si estás detrás de un firewall, es posible que esta funcionalidad requiera que añadas las direcciones IP correspondientes de Auth0 a la lista de permitidos para que funcione correctamente.
</Card>

<div id="create-the-connection-in-the-auth0-dashboard">
  ## Cree la conexión en el Auth0 Dashboard
</div>

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y seleccione **Create DB Connection**.
2. Configure los ajustes de la conexión y seleccione **Create**:

| Parámetro                                             | Definición                                                                                                                                                                   |
| ----------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Name**                                              | El nombre de la conexión. El nombre debe comenzar y terminar con un carácter alfanumérico, contener solo caracteres alfanuméricos y guiones, y no superar los 35 caracteres. |
| **Choose one or more attributes as user identifiers** | Elija los [atributos identificadores](/docs/es/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers) de la conexión.                    |
| **Choose one or more authentication methods**         | Elija un método de autenticación para verificar a los usuarios.                                                                                                              |
| **Use my own database**                               | Active esta opción si tiene una base de datos que desea conectar con Auth0.                                                                                                  |
| **Disable Sign Ups**                                  | Evita el registro de nuevos usuarios desde endpoints públicos.                                                                                                               |
| **Promote Connection to Domain Level**                | Promueve esta conexión al nivel de dominio para usarla con aplicaciones de terceros.                                                                                         |

Una vez que Auth0 cree la conexión, tendrá las siguientes vistas (además de la vista **Settings**):

* Attributes
  * Métodos de autenticación
  * Custom Database
  * Applications

3. Seleccione la vista **Custom Database** y active el interruptor **Use my own database**.

   <Frame>![Habilitar la opción Use My Own Database en Custom Database](/docs/images/cdy7uua7fh8z/3kgHDpBFdVWNq9XOfhsTXI/2efa43e44793b8dd69b9f9e8b54bf752/2025-02-25_10-20-20.png)</Frame>

<div id="create-database-action-scripts">
  ## Crear scripts de acción de base de datos
</div>

Al activar el interruptor **Use my own database**, se habilita el área de **scripts de acción de base de datos**, donde crearás scripts para configurar cómo funciona la autenticación al usar tu base de datos. Puedes escribir tus propios scripts de acción de base de datos o seleccionar una plantilla del menú desplegable **Templates** y modificarla según sea necesario.

**Debes** configurar un script de `login`; los scripts adicionales para funciones de usuario son opcionales.

Los scripts de acción de base de datos disponibles son:

| Nombre                     | Descripción                                                                                                                    | Parámetros             |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------ | ---------------------- |
| **Login**<br />Obligatorio | Se ejecuta cada vez que un usuario intenta iniciar sesión.                                                                     | `email`, `password`    |
| **Create**                 | Se ejecuta cuando un usuario se registra.                                                                                      | `user`                 |
| **Verify**                 | Se ejecuta después de que un usuario sigue el enlace de verificación.                                                          | `email`                |
| **Change Password**        | Se ejecuta cuando un usuario hace clic en el enlace de confirmación después de solicitar el restablecimiento de la contraseña. | `email`, `newPassword` |
| **Get User**               | Recupera el perfil de un usuario de tu base de datos sin autenticarlo.                                                         | `email`                |
| **Delete**                 | Se ejecuta cuando se elimina un usuario mediante la API o el Auth0 Dashboard.                                                  | `id`                   |

Para obtener más información sobre los scripts, consulta [Plantillas de scripts de acción de base de datos personalizadas](/docs/es/authenticate/database-connections/custom-db/templates) y [Prácticas recomendadas para la ejecución de scripts de acción de base de datos personalizados](/docs/es/authenticate/database-connections/custom-db/custom-database-connections-scripts/execution).

<div id="create-a-login-script">
  ### Crear un script de inicio de sesión
</div>

<Card title="Evita colisiones de ID de usuario con múltiples bases de datos">
  Auth0 utilizará la propiedad `id` (o, alternativamente, `user_id`) del perfil de usuario devuelto para identificar al usuario.

  Si utilizas varias conexiones de base de datos personalizadas, el valor de `id` debe ser único en todas ellas para evitar colisiones de ID de usuario. Te recomendamos anteponer al valor de `id` el nombre de la conexión (omitiendo cualquier espacio en blanco). Para obtener más información sobre los ID de usuario, consulta [Identify Users](/docs/es/manage-users/user-accounts/identify-users).
</Card>

Los siguientes pasos usan como ejemplo un script de inicio de sesión para una base de datos MySQL

1. Después de activar el switch **Use my own database**, se habilita el área de **scripts de acción de base de datos**. Asegúrate de estar en la pestaña **Login**.
2. Usa el menú desplegable **Templates** para seleccionar la plantilla de script de base de datos MySQL.

   ```javascript lines expandable
   function login(email, password, callback) {
     var bcrypt = require('bcrypt');
     var mysql = require('mysql');
     var connection = mysql.createConnection({
       host: 'localhost',
       user: 'me',
       password: 'secret',
       database: 'mydb'
     });
     connection.connect();
     var query = "SELECT id, nickname, email, password " +
       "FROM users WHERE email = ?";
     connection.query(query, [email], function (err, results) {
       if (err) return callback(err);
       if (results.length === 0) return callback(new WrongUsernameOrPasswordError(email));
       var user = results[0];
       bcrypt.compare(password, user.password, function (err, isValid) {
         if (err) {
           callback(err);
         } else if (!isValid) {
           callback(new WrongUsernameOrPasswordError(email));
         } else {
           callback(null, {
             // Este prefijo (sustitúyelo por el nombre de tu propia BD personalizada)
             // garantiza la unicidad entre distintas BD personalizadas si existe la
             // posibilidad de colisiones (por ejemplo, si el ID de usuario es una dirección de email o un entero)
             id: 'MyConnection1|' + user.id.toString(),
             nickname: user.nickname,
             email: user.email
           });
         }
       });
     });
   }
   ```

   El script anterior se conecta a una base de datos MySQL y ejecuta una consulta para recuperar el primer usuario cuyo `email == user.email`.
   Luego, con el método `bcrypt.compare`, valida que las contraseñas coincidan y, si es así, devuelve un objeto con la información del perfil del usuario, incluidos `id`, `nickname` y `email`.
   Este script asume que tienes una tabla `users` con estas columnas. El `id` que devuelve el script de inicio de sesión se usa para construir el atributo de ID de usuario del perfil del usuario.
3. Selecciona **Save and Try**.
4. En la ventana modal emergente, introduce tu email y contraseña, y haz clic en **Try**. Se mostrará un menú desplegable para seleccionar entre una lista de runtimes disponibles, con tu runtime actual en la parte superior.

   <Frame>![Ventana modal para probar el script de login.](/docs/images/cdy7uua7fh8z/8uBGZhHYZwOcO19ljJNNp/ccd69d84fdbdff90dc779a6b1ea8a6fd/db-connection-try-login-script.png)</Frame>

Auth0 no usa las plantillas de script hasta que selecciones **Save** o **Try**. Esto es así incluso si solo modificas un script y no has hecho cambios en ningún otro. Debes seleccionar **Save** al menos una vez para que todos los scripts queden guardados.

<div id="custom-database-support-for-organizations">
  ## Compatibilidad de bases de datos personalizadas con Organizations
</div>

Si usa conexiones de base de datos personalizadas junto con Organizations, puede usar el parámetro `context` en la firma del script para que los detalles de la organización, como `id`, `name` y `metadata`, estén disponibles para sus scripts de acción de base de datos personalizada. Los datos de la organización se pasan en el objeto `context` cuando la acción asociada ocurre en el contexto de una organización; por ejemplo, cuando un usuario se autentica en el prompt de inicio de sesión de una organización, se pasa el script de acción de inicio de sesión. Para obtener más información, consulte [Database Connections](/docs/es/authenticate/database-connections).

<div id="enable-context-object">
  ### Habilitar el objeto `context`
</div>

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/dashboard/us/#/connections/database/).
2. Seleccione su conexión de base de datos.
3. En la pestaña **Custom Database**, busque **Context objects in database scripts** y seleccione **Enable**.

   <Warning>
     Esta funcionalidad no se puede desactivar una vez habilitada.
   </Warning>
4. En el prompt de confirmación, seleccione **Confirm**.

<div id="use-organization-context">
  ### Usar el contexto de la organización
</div>

Una vez habilitado el objeto `context` en el Dashboard, puedes agregar un objeto `context` a tus scripts de acción de base de datos personalizados, insertándolo directamente antes del parámetro de devolución de llamada. Cuando se activan eventos con contexto de organización, los datos correspondientes estarán disponibles para tus scripts de acción de base de datos personalizados con el siguiente formato:

<div id="add-configuration-parameters">
  ## Agregar parámetros de configuración
</div>

Puedes almacenar parámetros, como las credenciales necesarias para conectarte a tu base de datos, en la sección **Configuración**, debajo del editor de scripts. Estarán disponibles para todos tus scripts, y podrás acceder a los valores de esos parámetros mediante el objeto `configuration` en tus scripts de acción de base de datos (es decir, `configuration.MYSQL_PASSWORD`).

Usa los parámetros añadidos en tus scripts para configurar la conexión. Por ejemplo, podrías agregar lo siguiente al script de inicio de sesión de MySQL:

```javascript lines
function login (username, password, callback) {
  var mysql = require('mysql');
  var connection = mysql.createConnection({
    host     : configuration.MYSQL_HOST,
    user     : 'me',
    password : configuration.MYSQL_PASSWORD,
    database : 'mydb'
  });
}
```
