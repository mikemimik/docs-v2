La extensibilidad permite a los administradores agregar inicio de sesión personalizado en Auth0 como un mecanismo para desarrollar soluciones de última milla para la administración de identidades y accesos (IdAM). Cada tipo de extensibilidad usa Node.js y se ejecuta en la plataforma de Auth0 dentro de un tenant de Auth0.

La extensibilidad de Auth0 se presenta de varias formas:

* [Actions](/docs/es/customize/actions): Actions son funciones seguras, específicas del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma de Auth0. Usa Actions para personalizar y ampliar las capacidades de Auth0 con inicio de sesión personalizado.
* [Rules](/docs/es/customize/rules): Se ejecutan cuando se generan artefactos de autenticidad del usuario. Por ejemplo:

  * token de ID en OpenID Connect (OIDC)
  * token de acceso en OAuth 2.0
  * una aserción en SAML
* [Hooks](/docs/es/customize/hooks): Proporcionan extensibilidad adicional cuando hay un intercambio de artefactos no relacionados con el usuario y cuando se crean identidades de usuario, como en el registro previo y posterior del usuario.

A continuación, se incluyen algunas prácticas recomendadas para usar los productos de extensibilidad de Auth0:

* Los scripts para las conexiones de base de datos personalizadas y las migraciones se usan para integrarse con un identity store de usuario existente o cuando se requiere la migración automática de usuarios desde un legacy identity store o un almacén independiente. Para obtener más información sobre la configuración, consulta [Create Custom Database Connections.](/docs/es/authenticate/database-connections/custom-db/create-db-connection)
* Para evitar duplicados de usuarios, establece un `user_id` coherente en cada user profile que se devuelva. Además, asegúrate de no devolver la misma dirección de correo electrónico para dos usuarios distintos en el script `get_user` o `login`, y de que ambos scripts devuelvan el mismo usuario.
* Para mantener la compatibilidad con versiones anteriores, `app_metadata` se llama `metadata` en los scripts de Custom DB. Dado que la transferencia de datos puede interrumpirse, usa una Action para obtener los metadatos que deberían estar en `app_metadata` o `user_metadata` si faltan.
* Asegúrate de restringir el acceso al <Tooltip tip="Audience: Identificador único de la audience de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip> con una [Action](/docs/es/customize/actions).

La extensibilidad de Auth0 es una capacidad integral y sofisticada para adaptar las operaciones de IdAM, pero también puede exponer tu base de datos a usos indebidos o no previstos. Los documentos de esta sección destacan prácticas recomendadas tanto para diseñadores como para implementadores; te recomendamos leerlos completos al menos una vez, incluso si ya has comenzado tu recorrido con Auth0.
