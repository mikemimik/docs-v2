La extensibilidad permite a los administradores agregar inicio de sesión personalizado en Auth0 como mecanismo para desarrollar soluciones de última milla para la gestión de identidades y accesos (IdAM). Cada tipo de extensibilidad usa Node.js y se ejecuta en la plataforma Auth0 dentro de un tenant de Auth0.

La extensibilidad de Auth0 se presenta de varias formas:

* [Actions](/docs/es/customize/actions): Actions son funciones seguras, específicas del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma Auth0. Use Actions para personalizar y ampliar las capacidades de Auth0 con inicio de sesión personalizado.
* [Rules](/docs/es/customize/rules): Se ejecutan cuando se generan artefactos relacionados con la autenticación del usuario. Por ejemplo:

  * token de ID en OpenID Connect (OIDC)
  * token de acceso en OAuth 2.0
  * una aserción en SAML
* [Hooks](/docs/es/customize/hooks): Proporcionan extensibilidad adicional cuando hay intercambio de artefactos no relacionados con el usuario y cuando se crean identidades de usuario, como en el registro previo y posterior del usuario.

A continuación se muestran algunas prácticas recomendadas para usar los productos de extensibilidad de Auth0:

* Los scripts tanto para las conexiones de Custom Database personalizadas como para las migraciones se usan para integrarse con un repositorio de identidades de usuario existente o cuando se requiere la migración automática de usuarios desde un repositorio de identidades independiente o heredado. Para obtener más información sobre la configuración, consulte [Create Custom Database Connections.](/docs/es/authenticate/database-connections/custom-db/create-db-connection)
* Para evitar la duplicación de usuarios, establezca un `user_id` coherente en cada perfil de usuario devuelto. Además, asegúrese de no devolver la misma dirección de correo electrónico para dos usuarios diferentes en el script `get_user` o `login`, y de que ambos scripts devuelvan el mismo usuario.
* Para mantener la compatibilidad con versiones anteriores, `app_metadata` se llama `metadata` en los scripts de Custom DB. Dado que la transferencia de datos puede interrumpirse, use una Action para obtener los metadatos que deberían estar en `app_metadata` o `user_metadata` si faltan.
* Asegúrese de restringir el acceso a la <Tooltip tip="Audiencia: Identificador único de la audiencia de un token emitido. Llamado aud en un token, su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip> con una [Action](/docs/es/customize/actions).

La extensibilidad de Auth0 es una capacidad amplia y sofisticada para adaptar las operaciones de IdAM, pero también puede exponer su base de datos a usos inadecuados o no deseados. Los documentos de esta sección destacan prácticas recomendadas tanto para diseñadores como para quienes implementan estas soluciones; recomendamos leerlos completos al menos una vez, incluso si ya ha comenzado su recorrido con Auth0.
