---
title: Conexiones de base de datos personalizadas
description: Obtén información sobre cómo autenticar usuarios usando tu base de datos como proveedor de identidad.
validatedOn: 2026-02-25
---

Usa una conexión de base de datos personalizada cuando quieras darle a Auth0 acceso a tu propio repositorio independiente de datos de identidad (heredado), principalmente para la autenticación (cumpliendo el papel de un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>) y para migrar datos de usuario al almacén de datos de Auth0.

La [extensibilidad](/docs/es/customize/extensions) de Auth0 te permite agregar lógica personalizada para crear soluciones a medida para la gestión de identidades y accesos (IdAM). Auth0 ofrece extensibilidad mediante [Actions](/docs/es/customize/actions) y [scripts](/docs/es/authenticate/database-connections/custom-db/templates), tanto para conexiones de base de datos personalizadas como para la migración de bases de datos personalizadas. Cada una se implementa con [Node.js](https://nodejs.org/en/) y se ejecuta en la plataforma Auth0 dentro de un tenant de Auth0.

La extensibilidad de Auth0 se ejecuta en distintos puntos del pipeline de IAM:

* **Actions** son funciones seguras, específicas del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma Auth0. Actions se usan para personalizar y ampliar las capacidades de Auth0 con lógica personalizada. Para obtener más información, consulta [Cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

* **Los scripts de acción de base de datos personalizados** pueden usarse para integrarse con un repositorio de identidades de usuario existente, o cuando se requiere la [migración automática de usuarios](/docs/es/manage-users/user-migration) desde un repositorio de identidades heredado. Se proporcionan varias [plantillas](/docs/es/authenticate/database-connections/custom-db/templates).

Sea cual sea el caso de uso, la extensibilidad de Auth0 te permite adaptar las operaciones de IAM a tus requisitos exactos. Sin embargo, si no se usa correctamente, puede dar lugar a usos inadecuados o no deseados, lo que a la larga puede causar problemas. Para ayudar a prevenirlos, Auth0 ofrece [guía de prácticas recomendadas](/docs/es/authenticate/database-connections/custom-db/custom-database-connections-scripts) tanto para diseñadores como para implementadores, y recomendamos leerla completa al menos una vez, incluso si ya has comenzado tu recorrido con Auth0.
