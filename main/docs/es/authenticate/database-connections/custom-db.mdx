---
title: Conexiones de base de datos personalizadas
description: Obtén información sobre cómo autenticar usuarios usando tu base de datos como proveedor de identidad.
validatedOn: 2026-02-25
---

Usa una conexión de base de datos personalizada cuando quieras darle a Auth0 acceso a tu propio almacén de datos de identidad independiente (heredado), principalmente para la autenticación (cumpliendo la función de un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>) y para migrar datos de usuario al almacén de datos de Auth0.

La [extensibilidad](/docs/es/customize/extensions) de Auth0 te permite agregar lógica personalizada para crear soluciones a medida para la gestión de identidades y accesos (IdAM). Auth0 ofrece extensibilidad mediante [Actions](/docs/es/customize/actions) y [scripts](/docs/es/authenticate/database-connections/custom-db/templates), tanto para conexiones de base de datos personalizadas como para la migración de usuarios con base de datos personalizada. Ambos se implementan con [Node.js](https://nodejs.org/en/), que se ejecuta en la plataforma de Auth0 dentro de un tenant de Auth0.

La extensibilidad de Auth0 se ejecuta en distintos puntos del pipeline de IAM:

* **Actions** son funciones seguras, específicas del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma de Auth0. Se usan para personalizar y ampliar las capacidades de Auth0 con lógica personalizada. Para obtener más información, consulta [Understand how Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

* **Scripts de acción para bases de datos personalizadas** se pueden usar para integrarse con un almacén de identidades de usuario existente o cuando se requiere la [migración automática de usuarios](/docs/es/manage-users/user-migration) desde un almacén de identidades heredado. Se proporcionan varias [plantillas](/docs/es/authenticate/database-connections/custom-db/templates).

Sea cual sea el caso de uso, la extensibilidad de Auth0 te permite adaptar las operaciones de IAM exactamente a tus necesidades. Sin embargo, si no se usa correctamente, puede dar lugar a usos indebidos o no intencionados que más adelante generen situaciones problemáticas. Para ayudar a prevenirlo, Auth0 ofrece [guía de prácticas recomendadas](/docs/es/authenticate/database-connections/custom-db/custom-database-connections-scripts) tanto para quienes diseñan como para quienes implementan, y recomendamos leerla completa al menos una vez, incluso si ya comenzaste a trabajar con Auth0.
