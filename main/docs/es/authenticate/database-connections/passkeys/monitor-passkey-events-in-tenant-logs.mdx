title: Monitorear eventos de claves de acceso en los logs del tenant
---

Puede monitorear los flujos de claves de acceso asociados con sus conexiones de base de datos a través de los [logs del tenant](/docs/es/deploy-monitor/logs).

Puede ver los eventos de log en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ([Monitoreo &gt; Logs](https://manage.auth0.com/#/logs)) o recuperar logs mediante la [Management API](https://auth0.com/docs/api/management/v2/logs/get-logs).

| Código de evento                | Escenario                                                                                               | Notas                                                                                                                                                                                                                                                                                                    |
| ------------------------------- | ------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ss`                            | Durante el registro, un usuario registró una clave de acceso como su método principal de autenticación. | Este evento de log contiene el campo de arreglo `details.authentication_methods` con el valor de entrada `passkey`.<br /><br />Si un usuario se registró con una contraseña, este campo de arreglo contiene el valor de entrada `pwd`.                                                                   |
| `s`                             | Durante el inicio de sesión, un usuario se autenticó con una clave de acceso.                           | Este evento de log contiene el arreglo `details.prompts`, que enumera el conjunto de prompts por los que pasó un usuario durante el inicio de sesión.<br /><br />Si un usuario se autenticó con una clave de acceso, el arreglo contiene un elemento con el siguiente campo: `"performed_amr": ["phr"]`. |
| `gd_enrollment_complete`        | Se completó correctamente un flujo de inscripción progresiva o local.                                   | Este evento de log contiene el campo `details.authenticator.type` con el valor `passkey` si un usuario inscribió una clave de acceso.                                                                                                                                                                    |
| `gd_webauthn_enrollment_failed` | Se produjo un error durante un flujo de inscripción progresiva o local.                                 | Este evento de log contiene el campo de arreglo `details.authentication_methods` con el valor de entrada `passkey` si un usuario intentó inscribir una clave de acceso.                                                                                                                                  |
