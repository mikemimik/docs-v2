---
title: Integraciones de inicio de sesión único de SAML
description: Describe las opciones de integración de Security Assertion Markup Language (SAML) para el inicio de sesión único (SSO).
---

Cuando implementa <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=single+sign-on">inicio de sesión único</Tooltip> (SSO), es importante tener en cuenta lo siguiente:

* Qué sistema(s) servirán como fuente de referencia para la información del perfil de usuario si surge un conflicto entre dos o más fuentes.
* Qué atributos del perfil de usuario necesita cada aplicación.
* Cómo se distribuirá la información del perfil de usuario a los sistemas que la necesiten.

Por lo general, una federación SAML se configura mediante SSO iniciado por el proveedor de servicios. El proveedor de servicios devuelve una redirección del navegador para que el usuario se autentique mediante el <Tooltip tip="Identity Provider (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip>. Después de la autenticación, el navegador redirige al usuario de vuelta al proveedor de servicios con una aserción SAML que contiene información sobre el estado de la autenticación. Esto se usa habitualmente en escenarios dirigidos al consumidor.

También puede configurar el IdP para que inicie el SSO en lugar del proveedor de servicios. En este escenario, el usuario accede a una URL en el IdP y se le solicita que se autentique; luego se lo redirige al proveedor de servicios con una aserción SAML. Esto se usa habitualmente en escenarios empresariales.

Tenemos muchas [integraciones de SSO que usan Auth0 como IdP](/docs/es/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider). También ofrecemos [integraciones de SSO para otros proveedores de identidad](/docs/es/authenticate/protocols/saml/saml-sso-integrations/configure-auth0-saml-service-provider).
