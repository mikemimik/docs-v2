Puedes personalizar tus aserciones <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>, así como los parámetros de los protocolos SAML y WS-Federation.

<div id="auth0-as-identity-provider">
  ## Auth0 como proveedor de identidad
</div>

Personaliza las aserciones SAML cuando Auth0 actúa como <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> configurando el addon en el Dashboard o con Rules.

<div id="use-the-dashboard">
  ### Usa el Dashboard
</div>

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications/\{yourClientId}/addons) y selecciona el nombre de la aplicación que quieras ver.
2. Selecciona la pestaña **Addons**.
3. Activa el control **SAML2 Web App** para ver la configuración y las opciones.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. En la pestaña **Settings**, puedes realizar varios tipos de personalizaciones, como:

   * Especificar un audience distinto del issuer predeterminado de la solicitud SAML.
   * Especificar un destinatario.
   * Mapear atributos del perfil a declaraciones de atributos específicas.
   * Cambiar el algoritmo de firma o de resumen.
   * Especificar si debe firmarse solo la aserción o toda la respuesta.

<div id="use-actions">
  ### Usar Actions
</div>

Puedes usar Actions para añadir personalizaciones más amplias o dinámicas a la respuesta SAML. Las personalizaciones realizadas en Actions prevalecen sobre las que se hacen desde la vista **Application Addons** en el Dashboard.

El objeto `api.samlResponse` se usa para sobrescribir los atributos SAML predeterminados o añadir atributos nuevos.

<div id="example-changing-the-saml-token-lifetime-and-use-upn-as-nameid">
  #### Ejemplo: cambiar la duración del token de acceso de SAML y usar UPN como NameID
</div>

```js lines
exports.onExecutePostLogin = async (event, api) => {
	api.samlResponse.setLifetimeInSeconds(36000);
	if (event.user.upn) {
		api.samlResponse.setAttribute('http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier', 'upn');
	}
};
```

<div id="example-include-user_metadata-attributes-in-an-assertion">
  #### Ejemplo: Incluir los atributos de `user_metadata` en una aserción
</div>

```js lines
exports.onExecutePostLogin = (event, api) => {
	event.user.user_metadata = event.user.user_metadata || {};
	event.user.user_metadata.color = 'purple';
	api.samlResponse.setAttribute('http://schemas.xmlsoap.org/ws/2005/05/identity/claims/color', event.user.user_metadata.color);
};
```

<div id="saml-assertion-attributes">
  ## Atributos de las aserciones SAML
</div>

A continuación se muestra una lista de atributos de personalización para las aserciones SAML.

| Atributo                         | Tipo     | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| -------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `audience`                       | string   | Audiencia de la aserción SAML. El valor predeterminado es issuer en SAMLRequest.                                                                                                                                                                                                                                                                                                                                                                                                        |
| `recipient`                      | string   | Destinatario de la aserción SAML (`SubjectConfirmationData`). El valor predeterminado es `AssertionConsumerUrl` en `SAMLRequest` o la URL de devolución de llamada si no se envió ninguna `SAMLRequest`.                                                                                                                                                                                                                                                                                |
| `issuer`                         | string   | Identificador único del proveedor de identidad SAML en formato de URL.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `mappings`                       | objeto   | Mapeos entre el perfil de Auth0 y los atributos de salida de la aserción SAML. El mapeo predeterminado se muestra arriba.                                                                                                                                                                                                                                                                                                                                                               |
| `createUpnClaim`                 | booleano | Indica si se debe crear un claim UPN. El valor predeterminado es `true`.                                                                                                                                                                                                                                                                                                                                                                                                                |
| `passthroughClaimsWithNoMapping` | booleano | Si es `true` (predeterminado), por cada claim que no se asigne al perfil común, Auth0 lo incluye en la aserción de salida. Si es `false`, esos claims no se asignarán.                                                                                                                                                                                                                                                                                                                  |
| `mapUnknownClaimsAsIs`           | booleano | Si `passthroughClaimsWithNoMapping` es `true` y esto es `false` (predeterminado), para cada claim que no esté mapeado al perfil común, Auth0 agrega el prefijo `http://schema.auth0.com`. Si es `true`, dejará pasar el claim tal cual.                                                                                                                                                                                                                                                 |
| `mapIdentities`                  | booleano | Si es `true` (predeterminado), agrega más información al token, como el proveedor (Google, ADFS, AD, etc.) y el token de acceso, si está disponible.                                                                                                                                                                                                                                                                                                                                    |
| `signatureAlgorithm`             | cadena   | Algoritmo de firma utilizado para firmar la SAML assertion o la respuesta. El valor predeterminado es `rsa-sha1`.                                                                                                                                                                                                                                                                                                                                                                       |
| `digestAlgorithm`                | cadena   | Algoritmo de hash para calcular el hash de la SAML assertion o la respuesta. El valor predeterminado es `sha1`.                                                                                                                                                                                                                                                                                                                                                                         |
| `destination`                    | objeto   | Destino de la respuesta SAML. Si no se especifica, será la `AssertionConsumerUrl` de la `SAMLRequest` o la URL de devolución de llamada si no hubo `SAMLRequest`.                                                                                                                                                                                                                                                                                                                       |
| `lifetimeInSeconds`              | entero   | Caducidad del token. El valor predeterminado es de `3600` segundos (1 hora).                                                                                                                                                                                                                                                                                                                                                                                                            |
| `signResponse`                   | booleano | Indica si se debe firmar o no la respuesta SAML. De forma predeterminada, se firmará la SAML assertion, pero no la respuesta SAML. Si es `true`, se firmará la SAML Response en lugar de la SAML assertion.                                                                                                                                                                                                                                                                             |
| `nameIdentifierFormat`           | string   | El valor predeterminado es `urn:oasis:names:tc:SAML:1.1:nameid-format:unspecified`.                                                                                                                                                                                                                                                                                                                                                                                                     |
| `nameIdentifierProbes`           | arreglo  | Auth0 probará cada uno de los atributos de este arreglo en orden. Si alguno de ellos tiene un valor, usará ese para el `Subject/NameID`. El orden es: `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier (asignado desde user_id)`, `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress (asignado desde email)`, `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name` (asignado desde name).                                                 |
| `authnContextClassRef`           | string   | El valor predeterminado es `urn:oasis:names:tc:SAML:2.0:ac:classes:unspecified`.                                                                                                                                                                                                                                                                                                                                                                                                        |
| `typedAttributes`                | boolean  | El valor predeterminado es `true`. Cuando se establece en `true`, inferimos el `xs:type` del elemento. Los tipos son `xs:string`, `xs:boolean`, `xs:double `y `xs:anyType`. Cuando se establece en `false`, todos los `xs:type` pasan a ser `xs:anyType`.                                                                                                                                                                                                                               |
| `includeAttributeNameFormat`     | boolean  | El valor predeterminado es `true`. Cuando se establece en `true`, inferimos el `NameFormat` en función del nombre del atributo. Los valores de `NameFormat` son `urn:oasis:names:tc:SAML:2.0:attrname-format:uri`, `urn:oasis:names:tc:SAML:2.0:attrname-format:basic` y `urn:oasis:names:tc:SAML:2.0:attrname-format:unspecified`. Si se establece en `false`, el `NameFormat` del atributo no se establece en la aserción.                                                            |
| `logout`                         | object   | Controla el cierre de sesión de SAML. Puede contener dos propiedades:`callback` (string), que contiene la URL del servicio de cierre de sesión único del proveedor de servicios (aplicación cliente), a la que Auth0 enviará las solicitudes y respuestas de cierre de sesión, y `slo_enabled`(boolean), que controla si Auth0 debe notificar a los proveedores de servicios la finalización de la sesión. El valor predeterminado es`true` (notificar a los proveedores de servicios). |
| `binding`                        | string   | Indica opcionalmente el enlace de protocolo utilizado para las respuestas de cierre de sesión de SAML. De forma predeterminada, Auth0 usa `HTTP-POST`, pero puedes cambiar a `HTTP-Redirect` configurando `"binding"` como `"urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect"`.                                                                                                                                                                                                      |
| `signingCert`                    | string   | Permite indicar opcionalmente el certificado de clave pública utilizado para validar solicitudes SAML. Si se configura, las solicitudes SAML deberán estar firmadas. Un valor de ejemplo sería `"-----BEGIN CERTIFICATE-----\nMIIC8jCCAdqgAwIBAgIJObB6jmhG0QIEMA0GCSqGSIb3DQEBBQUAMCAxHjAcBgNV\n[..all the other lines..]-----END CERTIFICATE-----\n"`.                                                                                                                                 |
