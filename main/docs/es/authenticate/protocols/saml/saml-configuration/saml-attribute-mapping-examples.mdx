---
description: Proporciona ejemplos de cómo asignar atributos SAML cuando Auth0 es el proveedor de identidad.
title: Asignar atributos SAML con Auth0 como IdP/complemento SAML
---

Cuando Auth0 es el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip>, puedes asignar atributos de usuario mediante el complemento SAML2 de Auth0. Pueden producirse errores si los atributos están mal configurados. Por ejemplo, un usuario introduce correctamente nombre de usuario y contraseña, pero no puede iniciar sesión en la aplicación aunque los logs del <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> muestren eventos de inicio de sesión exitosos. O bien, a tu aplicación le falta información del usuario, como el nombre o el correo electrónico.

<div id="use-cases">
  ## Casos de uso
</div>

El perfil de usuario que aparece a continuación sirve de ejemplo para los siguientes escenarios.

```javascript lines
//EJEMPLO DE PERFIL DE USUARIO IdP
{
   "created_at": "2021-06-21T13:26:08.579Z",
   "email": "testuser@example.com",
...
   "fav_genre": "fiction",
   "user_metadata": {
       "fav_streaming_service": "hulu"
   }
...
}
```

<div id="no-mappings-object">
  #### Sin objeto de mapeos
</div>

Al usar el complemento SAML2, se genera de forma predeterminada un objeto de mapeos vacío.

En este ejemplo, `fav_genre` y `user_metadata.fav_streaming_service` no están definidos, pero se pueden personalizar y mapear a la <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> Response generada por Auth0.

En el siguiente ejemplo, `"fav_genre": "fiction"` se mapea al atributo  [http://schemas.auth0.com/fav&#95;genre](http://schemas.auth0.com/fav_food) en la SAML Response con el valor `fiction`, mientras que `"user_metadata": {"fav_streaming_service": "hulu"}` no aparece en absoluto en la SAML Response.

SAML Response resultante que envía el IdP:

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/fav_genre" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
...
       </saml:AttributeStatement>
   </saml:Assertion>
</samlp:Response>
```

<div id="standard-mappings-example">
  #### Ejemplo de mapeos estándar
</div>

En el ejemplo anterior, no personalizar el objeto de mapeos dio lugar a un atributo [http://schemas.auth0.com/fav&#95;genre](http://schemas.auth0.com/fav_food) en la SAML Response con el valor `"fiction"`.

A continuación, mapea los atributos en el objeto de mapeos de la configuración del complemento SAML2 para tenerlo en cuenta.

Después de hacerlo, observa que el valor `"fiction"` sigue siendo el mismo en la SAML Response, pero el nombre del atributo en la SAML Response ha cambiado de [http://schemas.auth0.com/fav&#95;fiction](http://schemas.auth0.com/fav_food) predeterminado a [http://schemas.auth0.com/books](http://schemas.auth0.com/dinner).

Configuración del objeto de mapeos del complemento SAML2:

```json lines
"mappings": {
   "fav_genre": "http://schemas.auth0.com/books"
 }
```

Este mapeo produce la siguiente respuesta:

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/books" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
...
       </saml:AttributeStatement>
   </saml:Assertion>
</samlp:Response>
```

<div id="mapping-the-same-value-to-multiple-attributes">
  #### Mapear el mismo valor a varios atributos
</div>

Puede haber casos en los que sea necesario mapear la SAML Response a varios atributos con el mismo valor.

En este caso, existe la opción de mapear el mismo valor del perfil de usuario a varios atributos en la SAML Response.

Cómo configurar el objeto de mapeos del complemento SAML2:

```json lines
"mappings": {
   "fav_genre": [
     "http://schemas.auth0.com/movies",
     "http://schemas.auth0.com/books",
     "http://schemas.auth0.com/television"
   ]
 }
```

Este mapeo da lugar a la siguiente respuesta:

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/movies" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
           <saml:Attribute Name="http://schemas.auth0.com/books" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
           <saml:Attribute Name="http://schemas.auth0.com/television" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
...
       </saml:AttributeStatement>
   </saml:Assertion>
</samlp:Response>
```
