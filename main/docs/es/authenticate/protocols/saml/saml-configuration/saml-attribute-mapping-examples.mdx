---
description: Ofrece ejemplos de cómo mapear atributos SAML cuando Auth0 es el proveedor de identidad.
title: Mapear atributos SAML con Auth0 como IdP/complemento SAML2
---

Cuando Auth0 es el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip>, puedes mapear los atributos del usuario mediante el complemento SAML2 de Auth0. Pueden producirse errores si los atributos están mal configurados. Por ejemplo, un usuario introduce correctamente el nombre de usuario y la contraseña, pero no consigue iniciar sesión en la aplicación, aunque los registros del <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> muestran eventos de inicio de sesión correctos. O bien, a tu aplicación le falta información del usuario, como el nombre o el correo electrónico.

<div id="use-cases">
  ## Casos de uso
</div>

El siguiente perfil de usuario sirve de ejemplo para los escenarios que se muestran a continuación.

```javascript lines
//EJEMPLO de perfil de usuario de IdP
{
   "created_at": "2021-06-21T13:26:08.579Z",
   "email": "testuser@example.com",
...
   "fav_genre": "fiction",
   "user_metadata": {
       "fav_streaming_service": "hulu"
   }
...
}
```

<div id="no-mappings-object">
  #### Sin objeto `mappings`
</div>

Al usar el complemento SAML2, se genera de forma predeterminada un objeto `mappings` vacío.

En este ejemplo, `fav_genre` y `user_metadata.fav_streaming_service` no están definidos, pero se pueden personalizar y mapear a la <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> Respuesta SAML generada por Auth0.

En el ejemplo siguiente, `"fav_genre": "fiction"` se mapea al atributo  [http://schemas.auth0.com/fav&#95;genre](http://schemas.auth0.com/fav_food) en la Respuesta SAML con el valor `fiction`, mientras que `"user_metadata": {"fav_streaming_service": "hulu"}` no aparece en absoluto en la respuesta SAML.

Respuesta SAML resultante que envía el IdP:

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/fav_genre" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
...
       </saml:AttributeStatement>
   </saml:Assertion>
</samlp:Response>
```

<div id="standard-mappings-example">
  #### Ejemplo de mapeos estándar
</div>

En el ejemplo anterior, no personalizar el objeto `mappings` dio como resultado un atributo [http://schemas.auth0.com/fav&#95;genre](http://schemas.auth0.com/fav_food) en la Respuesta SAML con el valor `"fiction"`.

A continuación, mapea los atributos en el objeto de mapeos de la configuración del complemento SAML2 para reflejarlo.

Después de hacerlo, observa cómo el valor `"fiction"` sigue siendo el mismo en la Respuesta SAML, pero el nombre del atributo en la Respuesta SAML ha cambiado del valor predeterminado [http://schemas.auth0.com/fav&#95;fiction](http://schemas.auth0.com/fav_food) a [http://schemas.auth0.com/books](http://schemas.auth0.com/dinner).

Configuración del objeto de mapeos del complemento SAML2:

```json lines
"mappings": {
   "fav_genre": "http://schemas.auth0.com/books"
 }
```

Este mapeo da como resultado la siguiente respuesta:

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/books" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
...
       </saml:AttributeStatement>
   </saml:Assertion>
</samlp:Response>
```

<div id="mapping-the-same-value-to-multiple-attributes">
  #### Mapear el mismo valor a varios atributos
</div>

Puede haber casos en los que sea necesario mapear la Respuesta SAML a varios atributos con el mismo valor.

En este caso, existe la opción de mapear el mismo valor del perfil del usuario a varios atributos de la Respuesta SAML.

Cómo configurar el objeto `mappings` del complemento SAML2:

```json lines
"mappings": {
   "fav_genre": [
     "http://schemas.auth0.com/movies",
     "http://schemas.auth0.com/books",
     "http://schemas.auth0.com/television"
   ]
 }
```

Este mapeo da lugar a la siguiente respuesta:

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/movies" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
           <saml:Attribute Name="http://schemas.auth0.com/books" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
           <saml:Attribute Name="http://schemas.auth0.com/television" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
```
