<div id="configure-okta-saml-app-integration">
  ## Configurar la integración de aplicación SAML de Okta
</div>

Puedes crear una integración de aplicación SAML en la Okta Developer Console.

<div id="create-app-integration">
  ### Crear integración de aplicación
</div>

1. Inicia sesión en tu [Consola de administración de Okta](https://login.okta.com).
2. Ve a **Create App Integration** y elige **SAML 2.0** entre las opciones.
3. Configura los siguientes parámetros:

| Configuración               | Descripción                                                           | Ejemplo                                                                         |
| --------------------------- | --------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| Single Sign-On URL          | URL de devolución de llamada de inicio de sesión del tenant de Auth0. | `https://{yourAuth0Domain}/login/callback?connection={yourAuth0ConnectionName}` |
| Audience URI (SP Entity ID) | Valor de audience de la conexión de Auth0.                            | `urn:auth0:{yourAuth0TenantName}:{yourAuth0ConnectionName}`                     |

<Warning>
  El valor del nombre de la conexión (`{yourAuth0ConnectionName}`) que uses para configurar tu integración de aplicación SAML de Okta debe coincidir con el nombre de la conexión SAML que crees en Auth0.
</Warning>

4. Selecciona **Next** y, a continuación, **Finish** para completar la configuración de la integración de aplicación de Okta.

<div id="record-sso-url-and-download-certificate">
  ### Anota la URL de SSO y descarga el certificado
</div>

El flujo de inicio de sesión ahora se redirige a la página **Sign On** de la aplicación recién creada.

1. Selecciona **View SAML Setup Instructions**.
2. Anota la **Identity Provider Single Sign-On URL**.
3. Descarga el **certificado X.509** en formato PEM o CER.
4. Ve a **Assignments** y luego asigna un usuario a la aplicación de Okta.

<div id="configure-saml-connection-in-auth0">
  ## Configurar una conexión SAML en Auth0
</div>

Puede crear una conexión empresarial SAML en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Inicie sesión en el [Auth0 Dashboard](https://manage.auth0.com/#).
2. Vaya a [Autenticación &gt; empresarial](https://manage.auth0.com/#/connections/enterprise/samlp).
3. Seleccione **Crear** (botón **+**) junto a **SAML**.
4. Configure los siguientes ajustes:

| Configuración             | Descripción                                                                                                                                                                          | Ejemplo                                                                                                                             |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------- |
| Nombre de la conexión     | Nombre de la conexión de Auth0.                                                                                                                                                      | `myoktaconnection`                                                                                                                  |
| URL de inicio de sesión   | URL de Okta a la que se envían las solicitudes de inicio de sesión de los usuarios.<br /><br />Este es el valor de **Identity Provider Single Sign-On URL** que anotó anteriormente. | `https://my_okta_tenant_name.okta.com/app/` `my_okta_tenant_namemy_okta_saml_app_integration_name/` `dakflkbzevu5i5zBi939/sso/saml` |
| Certificado de firma X509 | Certificado de firma de clave pública del tenant de Okta.<br /><br />Cargue el **X509 Certificate** que descargó anteriormente.                                                      | `myOktaTenantSigningCertificate.pem`                                                                                                |

5. Seleccione **Crear**.

<div id="enable-saml-enterprise-connection-in-auth0">
  ## Habilita la conexión empresarial SAML en Auth0
</div>

Puedes habilitar tu conexión empresarial SAML en el Auth0 Dashboard.

<div id="enable-saml-enterprise-connection-when-using-organizations">
  ### Habilitar la conexión empresarial SAML al usar Organizations
</div>

Si usas Organizations:

1. Inicia sesión en el [Auth0 Dashboard](https://manage.auth0.com/#).
2. Ve a [Organizations](https://manage.auth0.com/#/organizations) y selecciona tu organización.
3. Cambia a la vista **Connections**.
4. Selecciona **Enable Connections**.
5. Selecciona la conexión SAML que creaste anteriormente y, a continuación, selecciona **Enable Connection**.

<div id="enable-saml-enterprise-connection-when-not-using-organizations">
  ### Habilita la conexión empresarial SAML cuando no uses Organizations
</div>

Si no usas Organizations:

1. Inicia sesión en el [Auth0 Dashboard](https://manage.auth0.com/#).
2. Ve a [Autenticación &gt; Empresarial &gt; SAML](https://manage.auth0.com/#/connections/enterprise/samlp) y selecciona la conexión SAML que creaste anteriormente.
3. Cambia a la vista de **Applications** y habilita la conexión para la(s) aplicación(es) que elijas.

<div id="test-connection">
  ## Probar la conexión
</div>

Puede probar su conexión en el Auth0 Dashboard.

1. Inicie sesión en el [Auth0 Dashboard](https://manage.auth0.com/#).
2. Vaya a [Autenticación &gt; Empresarial &gt; SAML](https://manage.auth0.com/#/connections/enterprise/samlp).
3. Busque su conexión en la lista.
4. Seleccione **More Actions** (botón **...**) y, a continuación, seleccione **Try**.

   * Si su conexión está configurada correctamente, verá la pantalla **It works!**.
   * Si no es así, verá un mensaje de error con detalles sobre lo que salió mal.

<Warning>
  El botón **Try** funciona solo para los usuarios que hayan iniciado sesión en el Auth0 Dashboard. No puede enviárselo a un usuario anónimo, como un cliente.

  Si no tiene un usuario de Okta, debe crear uno o [**configurar SSO iniciado por el IdP**](/docs/es/authenticate/protocols/saml/saml-sso-integrations/identity-provider-initiated-single-sign-on).
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Es posible que el usuario vea el dashboard de Okta después de autenticarse mediante un flujo de inicio de sesión iniciado por el Proveedor de servicios. Si integra su aplicación con Auth0 mediante el protocolo OIDC, Auth0 toma el valor del parámetro `state` y lo pasa a Okta mediante el parámetro SAML `RelayState`. Asegúrese de establecer el parámetro `state` con un valor que Okta pueda usar.
</Callout>

<div id="configure-global-token-revocation">
  ## Configurar Global Token Revocation
</div>

Este tipo de conexión admite un endpoint de Global Token Revocation, que permite a un proveedor de identidad compatible revocar las sesiones de usuario de Auth0, revocar <Tooltip tip="Refresh Token: token utilizado para obtener un Access Token renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> y activar el cierre de sesión por back-channel en las aplicaciones que usan un back-channel seguro.

Esta funcionalidad puede utilizarse con Universal Logout en Okta Workforce Identity.

Para obtener más información e instrucciones de configuración, consulta [Universal Logout](/docs/es/authenticate/login/logout/universal-logout).
