<div id="prerequisite">
  ## Requisito previo
</div>

Debe tener una cuenta en [SiteMinder](https://www.siteminder.com/login/).

<div id="configure-siteminder-settings">
  ## Configurar los ajustes de SiteMinder
</div>

Al configurar SiteMinder, usará los valores predeterminados para la mayoría de las opciones. También necesitará los siguientes valores relacionados con Auth0 en los pasos de configuración que aparecen a continuación:

* **EntityID:** `urn:auth0:{yourTenant}`
* **URL del Servicio de Consumo de Aserciones:** `https://{yourDomain}/login/callback`
* **URL de cierre de sesión:** `https://{yourDomain}/logout`
* binding **HTTP-Redirect** para la solicitud SAML
* binding **HTTP-POST** para la respuesta SAML

1. Inicie sesión en [SiteMinder](https://www.siteminder.com/login/).
2. Abra el **cuadro de diálogo del proveedor de servicios SAML**.
3. Proporcione un nombre adecuado para este proveedor de servicios.
4. Defina el `NameIdentifier`. Hay muchas formas de generarlo para los usuarios que se autentican con SiteMinder. Normalmente, este valor se asigna a una de las propiedades del usuario en el directorio de usuarios como `uid`.
5. Configure las propiedades generales de SAML del proveedor de servicios:

| Configuración          | Descripción              |
| ---------------------- | ------------------------ |
| **ID de SP**           | `urn:auth0:{yourTenant}` |
| **Versión de SAML**    | 2.0                      |
| **Tiempo de desfase:** | 30 segundos              |

6. Configure la **URL del Servicio de Consumo de Aserciones**, que es la ubicación a la que SiteMinder enviará de vuelta el token SAML mediante POST. Este proveedor de servicios ({yourTenant}) solo admite el binding `HTTP-POST` para las respuestas SAML. Use:
   `https://{yourDomain}/login/callback`
7. Marque la casilla **HTTP-Post**.
8. Configure propiedades de usuario adicionales para enviarlas en el token. Agregue cualquier otra propiedad que desee compartir sobre el usuario autenticado a este proveedor de servicios. Los valores habituales son: `name`, `lastname`, `email address`, etc. Este proveedor de servicios utilizará el `NameIdentifier` definido anteriormente como identificador único del usuario. Estos atributos se tratarán como información de referencia.
9. Introduzca la **URL de ubicación de SLO:** `https://{yourDomain}/logout`
10. (Opcional) El proveedor de servicios admite el cifrado de aserciones.
    Para usar esta opción, haga lo siguiente para descargar el certificado de clave pública del proveedor de servicios.

    1. En el Auth0 Dashboard, haga clic en **Connections** y luego en **Enterprise.**
    2. Haga clic en el proveedor de identidad SAMLP.
    3. Haga clic en el icono de configuración (lápiz).
    4. En la ventana que aparece, la séptima (última) viñeta le ofrece enlaces para descargar el certificado en formato .pem o .cer.
    5. Descargue el certificado que desee y agréguelo al **Policy Server Keystore** de SiteMinder.
