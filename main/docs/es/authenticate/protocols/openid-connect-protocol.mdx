<div id="what-is-openid-connect-oidc">
  ## ¿Qué es OpenID Connect (OIDC)?
</div>

<Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">
  OpenID
</Tooltip>

Connect (OIDC) es una capa de identidad basada en el

<Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">
  OAuth 2.0
</Tooltip>

framework. Permite que las aplicaciones de terceros verifiquen la identidad del usuario final y obtengan información básica del perfil del usuario. OIDC utiliza

<Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+web+tokens">
  JSON web tokens
</Tooltip>

(JWT), que puedes obtener mediante flujos conformes con las especificaciones de OAuth 2.0. Consulta nuestro [Manual de OIDC](https://auth0.com/resources/ebooks/the-openid-connect-handbook) para obtener más información.

<div id="openid-vs-oauth2">
  ## OpenID frente a OAuth2
</div>

Mientras que OAuth 2.0 se centra en el acceso a recursos y su uso compartido, OIDC se centra en la autenticación de usuario. Su propósito es ofrecerte un único inicio de sesión para varios sitios. Cada vez que necesites iniciar sesión en un sitio web con OIDC, se te redirige a tu sitio de OpenID, donde inicias sesión, y luego vuelves al sitio web. Por ejemplo, si elegiste iniciar sesión en Auth0 con tu cuenta de Google, entonces usaste OIDC. Una vez que te autenticas correctamente con Google y autorizas a Auth0 a acceder a tu información, Google envía a Auth0 información sobre el usuario y la autenticación realizada. Esta información se devuelve en un JWT. Recibirás un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> y, si se solicita, un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip>.

<div id="openid-and-jwts">
  ## OpenID y los JWT
</div>

Los JWT contienen [claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims), que son declaraciones (como el nombre o la dirección de correo electrónico) sobre una entidad (normalmente, el usuario), además de metadatos adicionales. La [especificación de OpenID Connect](https://openid.net/specs/openid-connect-core-1_0.html) define un conjunto de [claims estándar](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Este conjunto incluye el nombre, el correo electrónico, el género, la fecha de nacimiento, entre otros. Sin embargo, si quieres recopilar información sobre un usuario y actualmente no existe ningún claim estándar que refleje adecuadamente ese dato, puedes crear custom claims y añadirlos a tus tokens.
