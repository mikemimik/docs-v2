---
title: System for Cross-domain Identity Management (SCIM)
description: Describe cómo usar el esquema System for Cross-domain Identity Management (SCIM) en la gestión de identidades
---

SCIM (System for Cross-domain Identity Management) es un conjunto de protocolos de nivel de aplicación para gestionar y comunicar de forma segura datos de usuario en múltiples dominios. Los clientes SCIM pueden integrarse para gestionar operaciones CRUD (create, replace, update, delete), aplicar consultas y filtros, y crear grupos de usuarios dentro de su organización. SCIM le permite automatizar el ciclo de vida de los usuarios y mantener cuentas de usuario en distintas plataformas.

Para consultar la especificación de SCIM, lea [System for Cross-domain Identity Management: Core Schema](https://datatracker.ietf.org/doc/rfc7643/).

<div id="scim-with-auth0">
  ## SCIM con Auth0
</div>

Auth0 ofrece un directorio extensible y flexible diseñado para respaldar casos de uso de CIAM, y se centra en simplificar la gestión de identidades para aplicaciones de consumo directo y de software como servicio.

En un tenant de Auth0, puedes usar directorios aislados mediante conexiones empresariales, sociales o de base de datos. Para obtener más información sobre las conexiones disponibles, consulta [Proveedores de identidad](/docs/es/authenticate/identity-providers) o [Database Connections](/docs/es/authenticate/database-connections).

Auth0 admite SCIM entrante para <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Providers">proveedores de identidad</Tooltip> empresariales seleccionados. Para obtener más información, consulta [Configurar SCIM entrante](/docs/es/authenticate/protocols/scim/configure-inbound-scim).
