---
title: System for Cross-domain Identity Management (SCIM)
description: Describe el uso del esquema de System for Cross-domain Identity Management (SCIM) en la gestión de identidades
---

SCIM (System for Cross-domain Identity Management) es un conjunto de protocolos de nivel de aplicación para gestionar e intercambiar de forma segura datos de usuario entre múltiples dominios. Los clientes SCIM pueden integrarse para gestionar operaciones CRUD (create, replace, update, delete), aplicar consultas y filtros, y crear grupos de usuarios dentro de su organización. SCIM le permite automatizar el ciclo de vida de los usuarios y mantener cuentas de usuario en distintas plataformas.

Para consultar la especificación de SCIM, vea [System for Cross-domain Identity Management: Core Schema](https://datatracker.ietf.org/doc/rfc7643/).

<div id="scim-with-auth0">
  ## SCIM con Auth0
</div>

Auth0 proporciona un directorio extensible y flexible diseñado para admitir casos de uso de CIAM, y se centra en simplificar la gestión de identidades para aplicaciones orientadas al consumidor y de software como servicio.

En un tenant de Auth0, puede usar directorios aislados mediante conexiones empresariales, sociales o de base de datos. Para obtener más información sobre las conexiones disponibles, lea [Proveedores de identidad](/docs/es/authenticate/identity-providers) o [Database Connections](/docs/es/authenticate/database-connections).

Auth0 admite SCIM entrante para determinados <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Providers">proveedores de identidad</Tooltip> empresariales. Para obtener más información, lea [configurar SCIM entrante](/docs/es/authenticate/protocols/scim/configure-inbound-scim).
