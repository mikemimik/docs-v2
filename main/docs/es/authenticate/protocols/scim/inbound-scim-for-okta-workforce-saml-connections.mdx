<Card title="Antes de comenzar">
  Crea una [conexión SAML](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml) en tu tenant de Auth0 y [conéctala a la aplicación SAML personalizada correspondiente](https://help.okta.com/en-us/content/topics/apps/apps_app_integration_wizard_saml.htm) en tu Dashboard de Okta Workforce Identity.
</Card>

Esta sección describe cómo configurar una integración de aplicación personalizada de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> y SCIM en un tenant de Okta Workforce Identity, que puede usarse para aprovisionar usuarios en tu endpoint SCIM de Auth0.

<div id="configure-scim-settings-in-auth0">
  ## Configura los ajustes de SCIM en Auth0
</div>

1. Abre el [Auth0 Dashboard](https://manage.auth0.com/#/) y ve a **Autenticación &gt; Empresarial &gt; SAML &gt; [Conexión] &gt; Aprovisionamiento.**

2. Desactiva **sincronizar atributos del perfil de usuario en cada inicio de sesión** a menos que quieras [sincronizar atributos adicionales al iniciar sesión](/docs/es/authenticate/protocols/scim/configure-inbound-scim).

3. En la misma sección, activa **Sincronizar perfiles de usuario mediante SCIM**.

4. En la pestaña **Mapeo**, asegúrate de que la opción **Atributo de SCIM que contiene el ID de usuario** esté configurada como **userName**.

5. Revisa los **Mapeos adicionales** para asegurarte de que los atributos SCIM ampliados estén mapeados a los atributos de Auth0 que prefieras; consulta [mapeo de atributos](/docs/es/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para obtener más información. Si recibes números de teléfono y direcciones físicas, ten en cuenta que Okta Workforce los envía como `phoneNumbers[primary eq true].value` y `addresses[primary eq true].*` de forma predeterminada.

6. Mapea el atributo SCIM `userName` al atributo `email` de Auth0 para que Okta pueda [encontrar y vincular usuarios existentes](https://developer.okta.com/docs/reference/scim/scim-20/) si esta conexión ya se ha implementado en producción usando la dirección de correo electrónico como ID de inicio de sesión y no se ha establecido ningún atributo `username` de Auth0 en los perfiles de usuario existentes. En **Mapeos adicionales**, busca el siguiente par de atributos:

   ```json lines
   {
       "scim": "userName",
       "auth0": "username"
     }
   ```

   Reemplázalo por el que aparece a continuación y luego elimina el mapeo de `emails[primary eq true].value`.

   ```json lines
   {
       "scim": "userName",
       "auth0": "email"
     }
   ```

7. Selecciona **Save Changes.**

<div id="retrieve-scim-endpoint-url-and-token">
  ## Obtén la URL del endpoint de SCIM y el token
</div>

1. En el Auth0 Dashboard, ve a la pestaña **Setup** de SCIM y copia la **URL del endpoint de SCIM,** luego pégala en un lugar seguro.
2. Selecciona **Generate New Token** y establece una fecha de vencimiento opcional para el token. Si lo deseas, también puedes seleccionar los scopes que quieres otorgar a Okta Workforce; los scopes predeterminados que usa Okta Workforce son `get:users`, `post:users` y `put:users`. También puedes seleccionar `get:groups`, `post:groups`, `put:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.

<div id="configure-scim-in-okta-workforce-for-saml-apps">
  ## Configurar SCIM en Okta Workforce para aplicaciones SAML
</div>

Hay dos formas de configurar el aprovisionamiento SCIM en Okta. Elige el método que coincida con tu tipo de conexión y los requisitos de tamaño del grupo:

| Método                           | Descripción                                                                                                                                                                                                                                           | Consideraciones                                                                                                               |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **Aplicación SAML**              | Usa la pestaña **Aprovisionamiento** existente para las aplicaciones SAML en Okta. Este método no admite `PATCH` para grupos, lo que limita la cantidad máxima de miembros del grupo a los que se pueden enviar en una sola solicitud `POST` o `PUT`. | Adecuado para aprovisionar usuarios. No admite aprovisionar grupos con más de 10,000 miembros.                                |
| **Aplicación de plantilla SCIM** | Usa una aplicación de plantilla SCIM 2.0 en Okta que admite el método SCIM `PATCH` con grupos. Es necesaria para aprovisionar grupos de más de 10,000 miembros.                                                                                       | Funciona junto con las integraciones de SSO de SAML u OIDC existentes. Admite aprovisionar grupos con más de 10,000 miembros. |

<div id="okta-saml-application">
  ### Aplicación SAML de Okta
</div>

1. Ve a tu aplicación SAML de Okta Workforce Identity, selecciona la pestaña **General** y luego haz clic en **Edit** en **App Settings.**

2. En la sección **aprovisionamiento**, selecciona **SCIM** y luego **Save**.

3. En la pestaña **General**, confirma también que [Federation Broker Mode](https://help.okta.com/en-us/content/topics/apps/apps-fbm-main.htm) esté desactivado.

4. Selecciona la pestaña **aprovisionamiento**, luego ve a la pestaña **Integration** y selecciona **Edit**.

5. Introduce el valor de **SCIM Endpoint URL** que copiaste antes en la sección **SCIM connector base URL**.

6. En **Unique identifier field for users**, introduce **userName**.

7. En **Supported provisioning actions**, selecciona `Push New Users`, `Push Profile Updates` y, opcionalmente, `Push Groups`. Elige **HTTP Header** como **Authentication Mode.**

8. Pega el SCIM token en el campo **Authorization** y luego elige **Test Connection Configuration** si quieres probar la nueva conexión. Selecciona **Save**.

   <Frame>![Configuración de Inbound SCIM WIC](/docs/images/cdy7uua7fh8z/4kEFIeW8qbRqGb46KYuXJH/067213e99fbca35baab9bb104d1047f3/supersaas.jpg)</Frame>

9. Ve a **aprovisionamiento &gt; Settings &gt; To App** y elige **Edit**; después, habilita las operaciones **Create Users, Update User Attributes** y **Deactivate users**. Selecciona **Save**.

   <Frame>![](/docs/images/cdy7uua7fh8z/4RXROJ4amD4TpY3ozDg6wV/03c11b7ea8354a033b64d2df4f6360be/supersaasprovisioning.jpg)</Frame>

10. En la sección **mapeo de atributos**, usa el botón **X** para **eliminar** las siguientes líneas:

| Atributo                             | Valor                                                                    |
| ------------------------------------ | ------------------------------------------------------------------------ |
| Tipo de correo electrónico principal | `(user.email != null && user.email != '') ? 'work' : ‘'`                 |
| Tipo de teléfono principal           | `(user.primaryPhone != null && user.primaryPhone != '') ? 'work' : ‘'`   |
| Tipo de dirección                    | `(user.streetAddress != null && user.streetAddress != '') ? 'work' : ‘'` |

Usa la sección **mapeo de atributos** para configurar cualquier atributo SCIM adicional que quieras que Okta envíe a tu endpoint de SCIM. Si agregas atributos personalizados, deben incluir una propiedad de espacio de nombres externo válida de SCIM 2.0. Para obtener más información sobre los espacios de nombres externos, consulta [la sección de ayuda de Okta](https://support.okta.com/help/s/article/Receiving-Invalid-externalNamespace-task-error-when-assigning-a-user-to-a-SCIMenabled-application).

11. Usa la pestaña **Assignments** para asignar usuarios o grupos a tu aplicación. Al asignar un grupo, se aprovisiona a sus usuarios en tu tenant de Auth0.

12. Para enviar grupos, selecciona la pestaña **Push Groups**, busca y selecciona el grupo que quieres enviar y luego selecciona **Save**. Para obtener más información, consulta [la documentación de Okta sobre App assignments y Group Push](https://help.okta.com/en-us/content/topics/users-groups-profiles/usgp-group-push-main.htm).

<div id="okta-scim-template-application">
  ### Aplicación de plantilla SCIM de Okta
</div>

1. En el portal de Okta, selecciona **Applications &gt; Browse App Catalog**, busca **(OAuth Bearer Token) Governance with SCIM 2.0** y selecciona **Add Integration**. Define una **Application label**, marca **Do not display application icon to users**, desmarca **Browser plugin auto-submit** y luego selecciona **Next** y **Done**.

2. En la página de configuración de la integración, selecciona la pestaña **aprovisionamiento** y luego **Configure API Integration**. Marca **Enable API Integration**. Copia la **URL del endpoint de SCIM** de la pestaña **Setup** del Auth0 Dashboard y pégala en el campo **SCIM 2.0 Base URL**, quitando la barra diagonal final si la hay. En el Auth0 Dashboard, selecciona **Generate New Token &gt; Generate Token &gt; Copy and Close** y luego pega el token en el campo **OAuth Bearer Token** de Okta. Opcionalmente, selecciona **Test API Credentials** y luego **Save**.

3. Junto a **aprovisionamiento To App**, selecciona **Edit**. Habilita solo estas opciones: **Create Users**, **Update User Attributes** y **Deactivate Users**. Desmarca **Set password when creating new users** y luego selecciona **Save**.

4. En **mapeo de atributos**, usa el botón **X** para eliminar estas líneas, ya que pueden causar incidencias durante las operaciones `PUT`:

   | Atributo                             | Valor                                                                    |
   | ------------------------------------ | ------------------------------------------------------------------------ |
   | Tipo de correo electrónico principal | `(user.email != null && user.email != '') ? 'work' : ''`                 |
   | Tipo de teléfono principal           | `(user.primaryPhone != null && user.primaryPhone != '') ? 'work' : ''`   |
   | Tipo de dirección                    | `(user.streetAddress != null && user.streetAddress != '') ? 'work' : ''` |

   Opcionalmente, configura atributos SCIM adicionales para enviarlos desde Okta a tu endpoint de SCIM.

5. Usa la pestaña **Assignments** para asignar usuarios o grupos a tu aplicación. Al asignar un grupo, sus usuarios se aprovisionan en tu tenant de Auth0.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Los mismos grupos y usuarios deben estar asignados tanto a la instancia de la aplicación de SSO como a la instancia de la aplicación SCIM en Okta.
   </Callout>

6. Para enviar grupos, selecciona la pestaña **Push Groups**, busca y selecciona el grupo que quieres enviar y luego selecciona **Save**. Esto aprovisiona el objeto de grupo completo en Auth0. Para obtener más información sobre la relación entre Group Push y las asignaciones, consulta la [documentación de Okta sobre asignaciones de aplicaciones y Group Push](https://help.okta.com/en-us/content/topics/users-groups-profiles/usgp-group-push-main.htm).
