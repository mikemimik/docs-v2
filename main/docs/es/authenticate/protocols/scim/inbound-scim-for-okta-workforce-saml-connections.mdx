<Card title="Antes de empezar">
  Crea una [conexión SAML](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml) en tu tenant de Auth0 y [conéctala con la aplicación SAML personalizada correspondiente](https://help.okta.com/en-us/content/topics/apps/apps_app_integration_wizard_saml.htm) en tu Dashboard de Okta Workforce Identity.
</Card>

Esta sección describe cómo configurar una integración de aplicación <Tooltip tip="Security Assertion Markup Language (SAML): Standardized protocol allowing two parties to exchange authentication information without a password." cta="View Glossary" href="/docs/es/glossary?term=SAML">SAML</Tooltip> y SCIM en un tenant de Okta Workforce Identity, que puede usarse para aprovisionar usuarios en tu endpoint SCIM de Auth0.

<div id="configure-scim-settings-in-auth0">
  ## Configurar la configuración de SCIM en Auth0
</div>

1. Abre el [Auth0 Dashboard](https://manage.auth0.com/#/) y ve a **Autenticación &gt; Empresarial &gt; SAML &gt; [Conexión] &gt; Aprovisionamiento.**

2. Desactiva **sincronizar atributos del perfil de usuario en cada inicio de sesión** a menos que quieras [sincronizar atributos adicionales al iniciar sesión](/docs/es/authenticate/protocols/scim/configure-inbound-scim).

3. En la misma sección, activa **Sincronizar perfiles de usuario con SCIM**.

4. En la pestaña **Mapeo**, asegúrate de que la opción **atributo SCIM que contiene el ID de usuario** esté configurada como **userName**.

5. Revisa los **Mapeos adicionales** para asegurarte de que los atributos SCIM ampliados estén mapeados a los atributos de Auth0 que prefieras; consulta [mapeo de atributos](/docs/es/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para más información. Si recibes números de teléfono y direcciones físicas, ten en cuenta que Okta Workforce los envía como `phoneNumbers[primary eq true].value` y `addresses[primary eq true].*` de forma predeterminada.

6. Mapea el atributo SCIM `userName` al atributo `email` de Auth0 para que Okta pueda [encontrar y asociar usuarios existentes](https://developer.okta.com/docs/reference/scim/scim-20/) si esta conexión ya se ha desplegado en producción usando la dirección de correo electrónico como ID de inicio de sesión y no hay ningún atributo `username` de Auth0 configurado en los perfiles de usuario existentes. En **Mapeos adicionales**, busca el siguiente par de atributos:

   ```json lines
   {
       "scim": "userName",
       "auth0": "username"
     }
   ```

   Sustitúyelo por el siguiente y luego elimina el mapeo a `emails[primary eq true].value`.

   ```json lines
   {
       "scim": "userName",
       "auth0": "email"
     }
   ```

7. Selecciona **Guardar cambios.**

<div id="retrieve-scim-endpoint-url-and-token">
  ## Recuperar la URL del endpoint de SCIM y el token
</div>

1. En el Auth0 Dashboard, ve a la pestaña **Setup** de SCIM y copia la **URL del endpoint de SCIM**; luego, pégala en un lugar seguro.
2. Selecciona **Generate New Token** y establece una fecha de expiración opcional para el token. Opcionalmente, puedes seleccionar los scopes que quieras conceder a Okta Workforce; los scopes predeterminados que usa Okta Workforce son `get:users`, `post:users` y `put:users`. También puedes seleccionar `get:groups`, `post:groups`, `put:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.

<div id="configure-scim-in-okta-workforce-for-saml-apps">
  ## Configurar SCIM en Okta Workforce para aplicaciones SAML
</div>

Hay dos maneras de configurar el aprovisionamiento SCIM en Okta. Elige el método que se ajuste a tu tipo de conexión y a los requisitos de tamaño del grupo:

| Método                           | Descripción                                                                                                                                                                                                                                      | Consideraciones                                                                                                               |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| **Aplicación SAML**              | Usa la pestaña existente de **Aprovisionamiento** para aplicaciones SAML en Okta. Este método no admite `PATCH` para grupos, lo que limita el número máximo de miembros del grupo a lo que se puede enviar en una sola solicitud `POST` o `PUT`. | Adecuado para aprovisionar usuarios. No admite aprovisionar grupos con más de 10,000 miembros.                                |
| **Aplicación de plantilla SCIM** | Usa una aplicación de plantilla SCIM 2.0 en Okta que admite el método `PATCH` de SCIM con grupos. Es obligatoria para aprovisionar grupos de más de 10,000 miembros.                                                                             | Funciona junto con las integraciones de SSO de SAML u OIDC existentes. Admite aprovisionar grupos con más de 10,000 miembros. |

<div id="okta-saml-application">
  ### Aplicación SAML de Okta
</div>

1. Ve a tu aplicación SAML de Okta Workforce Identity, selecciona la pestaña **General** y luego elige **Edit** en **App Settings.**

2. En la sección **Provisioning**, selecciona **SCIM** y luego **Save**.

3. En la pestaña **General**, confirma también que [Federation Broker Mode](https://help.okta.com/en-us/content/topics/apps/apps-fbm-main.htm) esté desactivado.

4. Selecciona la pestaña **Provisioning**, luego ve a la pestaña **Integration** y selecciona **Edit**.

5. Introduce el valor de **URL del endpoint de SCIM** que copiaste antes en la sección **SCIM connector base URL**.

6. En **Unique identifier field for users**, introduce **userName**.

7. En **Supported provisioning actions**, selecciona `Push New Users`, `Push Profile Updates` y, opcionalmente, `Push Groups`. Elige **HTTP Header** como **Authentication Mode.**

8. Pega el token de SCIM en el campo **Authorization** y, si quieres probar la nueva conexión, selecciona **Test Connection Configuration**. Luego selecciona **Save**.

   <Frame>![Configuración de Inbound SCIM WIC](/docs/images/cdy7uua7fh8z/4kEFIeW8qbRqGb46KYuXJH/067213e99fbca35baab9bb104d1047f3/supersaas.jpg)</Frame>

9. Ve a **Provisioning &gt; Settings &gt; To App** y elige **Edit**; luego habilita las operaciones **Create Users, Update User Attributes,** y **Deactivate users**. Selecciona **Save**.

   <Frame>![](/docs/images/cdy7uua7fh8z/4RXROJ4amD4TpY3ozDg6wV/03c11b7ea8354a033b64d2df4f6360be/supersaasprovisioning.jpg)</Frame>

10. En la sección **mapeo de atributos**, usa el botón **X** para **eliminar** las siguientes líneas:

| Atributo                             | Valor                                                                    |
| ------------------------------------ | ------------------------------------------------------------------------ |
| Tipo de correo electrónico principal | `(user.email != null && user.email != '') ? 'work' : ‘'`                 |
| Tipo de teléfono principal           | `(user.primaryPhone != null && user.primaryPhone != '') ? 'work' : ‘'`   |
| Tipo de dirección                    | `(user.streetAddress != null && user.streetAddress != '') ? 'work' : ‘'` |

Usa la sección **mapeo de atributos** para configurar cualquier atributo SCIM adicional que quieras que Okta envíe a tu endpoint de SCIM. Si añades atributos personalizados, deben incluir una propiedad `external namespace` válida de SCIM 2.0. Para obtener más información sobre los `external namespaces`, consulta [la sección de ayuda de Okta](https://support.okta.com/help/s/article/Receiving-Invalid-externalNamespace-task-error-when-assigning-a-user-to-a-SCIMenabled-application).

11. Usa la pestaña **Assignments** para asignar usuarios o grupos a tu aplicación. Al asignar un grupo, se aprovisiona a sus usuarios en tu tenant de Auth0.

12. Para enviar grupos, selecciona la pestaña **Push Groups**, busca y selecciona el grupo que quieres enviar y luego selecciona **Save**. Para obtener más información, consulta [la documentación de Okta sobre asignaciones de aplicaciones y Group Push](https://help.okta.com/en-us/content/topics/users-groups-profiles/usgp-group-push-main.htm).

<div id="okta-scim-template-application">
  ### Aplicación de plantilla SCIM de Okta
</div>

1. En el portal de Okta, selecciona **Applications &gt; Browse App Catalog**, busca **(OAuth Bearer Token) Governance with SCIM 2.0** y selecciona **Add Integration**. Define una **Application label**, marca **Do not display application icon to users**, desmarca **Browser plugin auto-submit** y luego selecciona **Next** y **Done**.

2. En la página de configuración de la integración, selecciona la pestaña **Provisioning** y, después, **Configure API Integration**. Marca **Enable API Integration**. Copia la **URL del endpoint de SCIM** desde la pestaña **Setup** del **Auth0 Dashboard** y pégala en el campo **SCIM 2.0 Base URL**, quitando cualquier barra diagonal final. En el **Auth0 Dashboard**, selecciona **Generate New Token &gt; Generate Token &gt; Copy and Close** y pega el token en el campo **OAuth Bearer Token** de Okta. Si quieres, selecciona **Test API Credentials** y luego **Save**.

3. Junto a **Provisioning To App**, selecciona **Edit**. Habilita solo estas opciones: **Create Users**, **Update User Attributes** y **Deactivate Users**. Desmarca **Set password when creating new users** y luego selecciona **Save**.

4. En **Mapeo de atributos**, usa el botón **X** para eliminar estas líneas, ya que pueden causar problemas durante las operaciones `PUT`:

   | Atributo                             | Valor                                                                    |
   | ------------------------------------ | ------------------------------------------------------------------------ |
   | Tipo de correo electrónico principal | `(user.email != null && user.email != '') ? 'work' : ''`                 |
   | Tipo de teléfono principal           | `(user.primaryPhone != null && user.primaryPhone != '') ? 'work' : ''`   |
   | Tipo de dirección                    | `(user.streetAddress != null && user.streetAddress != '') ? 'work' : ''` |

   Si quieres, configura atributos SCIM adicionales para enviarlos desde Okta a tu endpoint de SCIM.

5. Usa la pestaña **Assignments** para asignar usuarios o grupos a tu aplicación. Al asignar un grupo, se aprovisiona a sus usuarios en tu tenant de Auth0.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Los mismos grupos y usuarios deben asignarse tanto a la instancia de la aplicación de SSO como a la instancia de la aplicación de SCIM en Okta.
   </Callout>

6. Para enviar grupos, selecciona la pestaña **Push Groups**, busca y selecciona el grupo que quieres enviar y luego selecciona **Save**. Esto aprovisiona el objeto de grupo completo en Auth0. Para obtener más información sobre la relación entre Group Push y las asignaciones, consulta la [documentación de Okta sobre App assignments y Group Push](https://help.okta.com/en-us/content/topics/users-groups-profiles/usgp-group-push-main.htm).
