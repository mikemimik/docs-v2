<Card title="Antes de empezar">
  Para usar esta funcionalidad, tu plan de Auth0 o tu acuerdo personalizado debe incluir Enterprise Connections. Para obtener más información, consulta la [página de precios de Auth0](https://auth0.com/pricing).

  Para probar una conexión SCIM entrante, debes instalar [Postman](https://www.postman.com/downloads/) en tu equipo.
</Card>

Para aprovisionar usuarios en el endpoint SCIM de Auth0, debes configurar una aplicación empresarial no incluida en la galería en Microsoft Entra ID (anteriormente Azure Active Directory).

<div id="configure-scim-settings-in-auth0">
  ## Configurar la configuración de SCIM en Auth0
</div>

1. Desde el [<Tooltip tip="..." cta="Ver glosario" href="/docs/es/glossary?term=Auth0%20Dashboard">Auth0 Dashboard</Tooltip>](https://manage.auth0.com/#/), ve a **Autenticación &gt; Empresarial &gt; SAML &gt; [YOUR&#95;AUTH0&#95;CONNECTION] &gt; aprovisionamiento**.
2. Activa **Sincronizar perfiles de usuario mediante SCIM**. Esto abrirá una nueva sección en la misma página con configuración adicional.
3. Verifica la siguiente configuración:
   A. En la pestaña **Mapeo**, asegúrate de que el campo **Atributo SCIM que contiene el ID de usuario** esté configurado como **userName**.
   B. En la sección **Mapeos adicionales**, verifica los siguientes valores:
   ```json lines
   {
       "scim": "emails[type eq \"work\"].value",
       "auth0": "email"
   },
   ```
   También puedes consultar el [mapeo de atributos](/docs/es/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) y asegurarte de que los atributos SCIM extendidos se asignen a los atributos de Auth0 que prefieras.
4. Si es necesario, selecciona **Guardar cambios**.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Obtén la URL del endpoint de SCIM y el token
</div>

Para configurar SCIM en un tenant de Microsoft Entra ID, necesitas una URL del endpoint de SCIM y un token de Auth0. Puedes obtener estos valores desde el Auth0 Dashboard:

1. En el [Auth0 Dashboard](https://manage.auth0.com/#/), ve a **Autenticación &gt; Empresarial &gt; SAML &gt; [tu-conexión] &gt; aprovisionamiento**.
2. En la sección **Sincronizar perfiles de usuario mediante SCIM &gt; Setup**, copia la **SCIM Endpoint URL**.
3. En la misma sección, en **Bearer Token**, selecciona **Generate New Token**. Elige la fecha de vencimiento del token y los scopes, y luego selecciona **Generate Token**. De forma predeterminada, se requieren los scopes que usa Azure AD: `get:users`, `post:users`, `patch:users` y `delete:users`. De forma opcional, puedes seleccionar `get:groups`, `post:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.
4. Copia el bearer token. Por seguridad, esta es la única vez que el token será visible.

También puedes obtener estos valores de forma programática mediante los endpoints de SCIM de la Management API, como [`/v2/connections/{id}/scim-configuration/tokens`](https://auth0.com/docs/api/management/v2/connections/get-scim-tokens).

Para conocer las prácticas recomendadas, lee las [directrices de implementación para configurar SCIM entrante](/docs/es/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines).

<div id="configure-scim-in-entra-id-for-saml-apps">
  ## Configurar SCIM en Entra ID para aplicaciones SAML
</div>

Si tu aplicación SAML aún no está registrada, sigue [Configure IdP-Initiated SAML Login with Azure as the IdP](https://support.auth0.com/center/s/article/Configure-IdP-Initiated-SAML-Login-with-Azure-as-the-IdP) para registrar una aplicación empresarial personalizada que no está en la galería en un tenant de Entra ID.

Luego, para configurar SCIM:

1. En el centro de administración de Microsoft Entra, ve a **Entra ID &gt; Aplicaciones empresariales &gt; [YOUR&#95;APPLICATION]**.
2. Continúa en **Administrar &gt; Propiedades** y establece **Se requiere asignación** en **Sí**.
3. Ve a **Administrar &gt; Users and Groups** y asigna los usuarios de Azure AD que quieras aprovisionar. Cuando asignas un <Tooltip tip="..." cta="View Glossary" href="/docs/es/glossary?term=Group">grupo</Tooltip>, se aprovisionan los usuarios de ese grupo.
4. Ve a **Administrar &gt; aprovisionamiento** y selecciona **Comenzar**.
5. Establece el **Modo de aprovisionamiento** en **Automático**.
6. Expande la sección **Credenciales de administrador** y configura los siguientes parámetros:
   * Establece el campo **URL del tenant** con la URL del endpoint de SCIM que guardaste en el paso anterior y añade el parámetro de consulta `?aadOptscim062020` al final de la URL. Esta es la [solución alternativa recomendada por Microsoft para un problema conocido de SCIM 2.0](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/application-provisioning-config-problem-scim-compatibility).
   * Establece **Secret Token** con el valor del token que guardaste en el paso anterior y, a continuación, selecciona **Guardar**.
7. Expande la sección **Asignaciones**.
8. Para aprovisionar grupos de forma opcional, confirma que **Provision Entra ID Groups** esté establecido en **Enabled**. De lo contrario, establécelo en **Disabled**.
9. Luego, selecciona **Provision Azure Active Directory Users** para abrir la pantalla **Mapeo de atributos**.
   <Frame>![Mapeo de atributos de SAML Azure](/docs/images/cdy7uua7fh8z/6JnGm4IeBglKiHNoVPX5vs/ddc17baeca48f02a6309b57ab87f845e/attributemap2.jpg)</Frame>
10. En la fila de la tabla de mapeo de atributos que contiene `emails[type eq "work"].value` y `mail`, selecciona **Editar** para ir a la pantalla **Editar atributo** y configurar los siguientes parámetros:
    * Establece **Hacer coincidir objetos con este atributo** en **Sí**.
    * Establece **Prioridad de coincidencia** en **2**.
11. Selecciona **Aceptar** para guardar la configuración. Esto cierra la pantalla **Editar atributo**. Luego, en la pantalla **Mapeo de atributos**, selecciona **Guardar**.

<div id="testing">
  ## Pruebas
</div>

1. En la pantalla **Información general** de la aplicación empresarial, ve a **Manage &gt; aprovisionamiento &gt; Provision on Demand** para probar la conexión SCIM.
2. Ve a **Select a user or group** e introduce el nombre de un usuario o grupo que hayas asignado a la aplicación; después, selecciona el elemento y elige **Provision**. Esto crea el usuario o grupo en el tenant de Auth0.
3. Establece **estado de aprovisionamiento** en **On** para [aprovisionar todos los usuarios y grupos asignados](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status).
