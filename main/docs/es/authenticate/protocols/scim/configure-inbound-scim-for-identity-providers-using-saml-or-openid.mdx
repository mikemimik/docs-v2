<div id="saml-identity-providers">
  #### Proveedores de identidad SAML
</div>

Cualquier <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> que use un [tipo de conexión SAML](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml) debe admitir el aprovisionamiento saliente de usuarios mediante un cliente SCIM 2.0, o utilizarse junto con un servicio de aprovisionamiento externo que proporcione aprovisionamiento saliente de usuarios. Revisa la [documentación oficial de SCIM](https://scim.cloud/#Implementations2) para obtener una lista de implementaciones conocidas de clientes SCIM 2.0.

Para obtener instrucciones sobre cómo configurar SCIM para proveedores de identidad SAML específicos, lee [Inbound SCIM para Okta Workforce SAML Connections](/docs/es/authenticate/protocols/scim/inbound-scim-for-okta-workforce-saml-connections) y [Inbound SCIM para Azure AD SAML Connections](/docs/es/authenticate/protocols/scim/inbound-scim-for-azure-ad-saml-connections)

<div id="openid-connect-identity-providers">
  #### Proveedores de identidad de OpenID Connect
</div>

Cualquier proveedor de identidad de <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect que use una [conexión de OpenID Connect](/docs/es/authenticate/identity-providers/enterprise-identity-providers/oidc) debe admitir el aprovisionamiento saliente de usuarios mediante un cliente SCIM 2.0 o utilizarse junto con un servicio de aprovisionamiento externo que lo proporcione. Consulta la [documentación oficial de SCIM](https://scim.cloud/#Implementations2) para ver una lista de implementaciones conocidas de clientes SCIM 2.0.

Para habilitar una gestión integral del ciclo de vida de los usuarios, tu proveedor de identidad de OpenID Connect debe estar configurado para enviar el mismo valor tanto en el atributo `sub` del [token de identidad](https://openid.net/specs/openid-connect-core-1_0.html#IDToken) como en el atributo SCIM `externalId`. Estos valores suelen corresponder al identificador único del usuario en el sistema del proveedor de identidad, que Auth0 utiliza para identificar al usuario durante el inicio de sesión.

Para obtener instrucciones sobre cómo configurar SCIM para proveedores de identidad específicos de OpenID Connect, consulta [Inbound SCIM para conexiones de Okta Workforce](/docs/es/authenticate/protocols/scim/inbound-scim-for-okta-workforce-connections), [Inbound SCIM para nuevas conexiones de Azure AD](/docs/es/authenticate/protocols/scim/inbound-scim-for-new-azure-ad-connections) y [Inbound SCIM para conexiones antiguas de Azure AD](/docs/es/authenticate/protocols/scim/inbound-scim-for-older-azure-ad-connections).
