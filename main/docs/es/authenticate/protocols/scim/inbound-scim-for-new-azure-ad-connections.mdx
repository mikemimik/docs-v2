<Card title="Antes de empezar">
  Asegúrate de que tu tenant de Auth0 esté conectado a Azure AD mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

  Esta integración requerirá registrar dos aplicaciones en Azure AD: la integración de OpenID Connect y la integración de SCIM. Para agilizar este proceso de configuración para tus clientes, considera [publicar tu aplicación en la galería de aplicaciones de Azure Active Directory](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/v2-howto-app-gallery-listing#submit-your-application).
</Card>

Auth0 puede integrarse con Microsoft Azure Active Directory (ahora conocido como Microsoft Entra ID) mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2), que utiliza el protocolo <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) para la autenticación de usuario. Estas instrucciones son para conexiones nuevas de Azure AD; para conexiones más antiguas que usan identificadores de usuario por pares (`sub)`, consulta [Conexiones antiguas](/docs/es/authenticate/protocols/scim/inbound-scim-for-older-azure-ad-connections).

<div id="configure-scim-settings-in-auth0">
  ## Configura los ajustes de SCIM en Auth0
</div>

1. Abre el [Auth0 Dashboard](https://manage.auth0.com/#/), ve a **Authentication &gt; empresarial &gt; Microsoft Azure AD &gt; [your-connection] &gt; Settings.**
2. Asegúrate de que **User ID Attribute Type** esté configurado como **User Object Identifier (oid)** y de que el selector **Use Common Endpoint** esté en **Disabled**.
3. Selecciona la pestaña **Aprovisionamiento** y desactiva **sincronizar atributos del perfil de usuario en cada inicio de sesión**, a menos que quieras [sincronizar atributos adicionales al iniciar sesión](/docs/es/authenticate/protocols/scim/configure-inbound-scim).
4. En la misma sección, habilita **Sync user profiles using SCIM**.
5. En la pestaña **Mapping**, asegúrate de que la opción **SCIM attribute containing the User ID** esté configurada como **externalId**.
6. Revisa los **Additional Mappings** para asegurarte de que los atributos SCIM extendidos se asignen a los atributos de Auth0 que prefieras. Consulta [mapeo de atributos](/docs/es/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para obtener más información.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Recuperar la URL del endpoint de SCIM y el token
</div>

En esta sección se utiliza el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, pero estos pasos también se pueden realizar con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Consulta la sección [Directrices de implementación](/docs/es/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines) para conocer las prácticas recomendadas.

1. En el [Auth0 Dashboard](https://manage.auth0.com/#/), ve a la pestaña **Setup**, copia la **URL del endpoint de SCIM** y pégala en un lugar seguro.
2. Genera un token de SCIM haciendo clic en **Generate New Token** y, si lo deseas, establece una fecha de vencimiento para el token.
3. Selecciona los scopes que quieras permitir. Los scopes predeterminados que requiere Azure AD son `get:users`, `post:users`, `patch:users` y `delete:users`. De forma opcional, puedes seleccionar `get:groups`, `post:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.

<div id="configure-scim-in-azure-ad-for-oidc-apps">
  ## Configurar SCIM en Azure AD para aplicaciones OIDC
</div>

1. Confirme que ya se haya registrado una aplicación de OpenID Connect para gestionar la autenticación de usuario en la sección **Microsoft Entra ID &gt; App registrations** del [Azure portal](https://portal.azure.com).
2. Confirme que su aplicación de OpenID Connect tenga **Assignment Required** establecido en **Yes** en la sección **Microsoft Entra ID &gt; Enterprise applications &gt; [your-oidc-app] &gt; Manage &gt; Properties** y que tenga usuarios asignados en la pestaña **Users and Groups**.
3. A continuación, registre una nueva aplicación **Non-gallery** en el [Azure portal](https://portal.azure.com) navegando a **Microsoft Entra ID &gt; Enterprise applications &gt; New application &gt; Create your own application,** introduciendo un nombre para la aplicación y seleccionando **Create**.
4. Vaya a la pestaña **Users and Groups** y asigne los mismos usuarios y grupos de Azure AD que están asignados a la aplicación registrada de OpenID Connect.
5. Seleccione la pestaña **aprovisionamiento**, luego **Get started**, y elija **Automatic** como **Provisioning Mode.**
6. Seleccione **Admin Credentials** y, a continuación, introduzca el valor de **URL del endpoint de SCIM** que guardó antes como **Tenant URL**. Al final de la URL, agregue el parámetro de consulta `?aadOptscim062020` para corregir las [incidencias conocidas de Azure AD descritas aquí.](https://learn.microsoft.com/en-us/azure/active-directory/app-provisioning/application-provisioning-config-problem-scim-compatibility)
7. Pegue el valor del token en el campo **Secret Token** y seleccione **Save.**
8. Vaya a **Mappings**.
9. Para aprovisionar grupos de forma opcional, confirme que **Provision Entra ID Groups** esté establecido en **Enabled**. De lo contrario, establézcalo en **Disabled**.
10. Seleccione **Provision Microsoft Entra ID Users,** luego vaya a **Attribute Mappings** y edite los atributos de la línea que contiene `externalId` y `mailNickname`.
11. En la pantalla **Edit Attribute**, cambie **Source attribute** a **objectId,** y luego elija **OK**.
12. Vuelva a **Attribute Mappings** y seleccione la línea que contiene `emails[type eq "work"].value and mail`.
13. En la pantalla **Edit Attribute**, cambie **Match objects using this attribute** a **Yes**, luego establezca **Matching precedence** en **2** y elija **OK.** La pantalla de mapeo de atributos se verá así mientras continúa usando la sección **Attribute Mappings** para configurar atributos SCIM adicionales:

<Frame>![Mapeo de atributos de SAML Azure](/docs/images/cdy7uua7fh8z/6JnGm4IeBglKiHNoVPX5vs/ddc17baeca48f02a6309b57ab87f845e/attributemap2.jpg)</Frame>

**Save** el mapeo de atributos y luego seleccione **X** en la esquina superior derecha para volver a la pantalla **aprovisionamiento**.

<div id="testing">
  ## Pruebas
</div>

1. En la pantalla **Información general** de la aplicación empresarial, ve a **Manage &gt; aprovisionamiento &gt; Provision on Demand** para probar la conexión SCIM.
2. Ve a **Select a user or group** e introduce el nombre de un usuario o de un grupo que hayas asignado a la aplicación; luego, selecciona la entrada y elige **Provision**. Esto crea el usuario o grupo en el tenant de Auth0.
3. Establece **Provisioning Status** en **On** para [aprovisionar todos los usuarios y grupos asignados](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status).
