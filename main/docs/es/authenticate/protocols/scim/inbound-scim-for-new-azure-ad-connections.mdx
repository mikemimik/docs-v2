<Card title="Antes de empezar">
  Asegúrate de que tu tenant de Auth0 esté conectado a Azure AD mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

  Esta integración requiere registrar dos aplicaciones en Azure AD: la integración de OpenID Connect y la integración de SCIM. Para agilizar este proceso de configuración para tus clientes, considera [publicar tu aplicación en la galería de aplicaciones de Azure Active Directory](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/v2-howto-app-gallery-listing#submit-your-application).
</Card>

Auth0 puede integrarse con Microsoft Azure Active Directory (ahora conocido como Microsoft Entra ID) mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2), que usa el protocolo <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) para la autenticación de usuario. Estas instrucciones son para conexiones nuevas de Azure AD; para conexiones más antiguas que usan identificadores de usuario pairwise (`sub)`, consulta [Conexiones antiguas](/docs/es/authenticate/protocols/scim/inbound-scim-for-older-azure-ad-connections).

<div id="configure-scim-settings-in-auth0">
  ## Configurar la configuración de SCIM en Auth0
</div>

1. Abra el [Auth0 Dashboard](https://manage.auth0.com/#/), vaya a **Authentication &gt; empresarial &gt; Microsoft Azure AD &gt; [su-conexión] &gt; Settings.**
2. Asegúrese de que **User ID Attribute Type** esté configurado como **User Object Identifier (oid)** y de que el interruptor **Use Common Endpoint** esté **Disabled**.
3. Seleccione la pestaña **aprovisionamiento** y desactive **sincronizar atributos del perfil de usuario en cada inicio de sesión**, a menos que quiera [sincronizar atributos adicionales al iniciar sesión](/docs/es/authenticate/protocols/scim/configure-inbound-scim).
4. En la misma sección, habilite **Sync user profiles using SCIM**.
5. En la pestaña **Mapping**, asegúrese de que la opción **SCIM attribute containing the User ID** esté configurada como **externalId**.
6. Revise **Additional Mappings** para asegurarse de que los atributos SCIM extendidos estén mapeados a los atributos de Auth0 que prefiera. Consulte [mapeo de atributos](/docs/es/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para más detalles.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Obtener la URL del endpoint de SCIM y el token
</div>

En esta sección se utiliza el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, pero estos pasos también se pueden realizar con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Consulta la sección [Deployment Guidelines](/docs/es/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines) para conocer las prácticas recomendadas.

1. En el [Auth0 Dashboard](https://manage.auth0.com/#/), ve a la pestaña **Setup**, copia la **SCIM Endpoint URL** y pégala en un lugar seguro.
2. Genera un token de SCIM haciendo clic en **Generate New Token** y, si lo deseas, establece una fecha de expiración para el token.
3. Selecciona los scopes que quieres permitir. Los scopes predeterminados que requiere Azure AD son `get:users`, `post:users`, `patch:users` y `delete:users`. De forma opcional, puedes seleccionar `get:groups`, `post:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.

<div id="configure-scim-in-azure-ad-for-oidc-apps">
  ## Configurar SCIM en Azure AD para aplicaciones OIDC
</div>

1. Confirma que ya se haya registrado una aplicación OpenID Connect para encargarse de la autenticación de usuario en la sección **Microsoft Entra ID &gt; App registrations** del [Azure portal](https://portal.azure.com).
2. Confirma que tu aplicación OpenID Connect tenga **Assignment Required** establecido en **Yes** en la sección **Microsoft Entra ID &gt; Enterprise applications &gt; [your-oidc-app] &gt; Manage &gt; Properties** y que tenga usuarios asignados en la pestaña **Users and Groups**.
3. A continuación, registra una nueva aplicación **Non-gallery** en el [Azure portal](https://portal.azure.com). Para ello, ve a **Microsoft Entra ID &gt; Enterprise applications &gt; New application &gt; Create your own application**, escribe un nombre para la aplicación y selecciona **Create**.
4. Ve a la pestaña **Users and Groups** y asigna los mismos usuarios y grupos de Azure AD que están asignados a la aplicación OpenID Connect registrada.
5. Selecciona la pestaña **aprovisionamiento**, haz clic en **Get started** y elige **Automatic** como **aprovisionamiento Mode.**
6. Selecciona **Admin Credentials** y, a continuación, introduce el valor de **URL del endpoint de SCIM** que guardaste antes como **Tenant URL**. Al final de la URL, agrega el parámetro de consulta `?aadOptscim062020` para corregir [las incidencias conocidas de Azure AD que se describen aquí.](https://learn.microsoft.com/en-us/azure/active-directory/app-provisioning/application-provisioning-config-problem-scim-compatibility)
7. Pega el valor del token en el campo **Secret Token** y selecciona **Save.**
8. Ve a **Mappings**.
9. Para aprovisionar grupos de forma opcional, confirma que **Provision Entra ID Groups** esté establecido en **Enabled**. De lo contrario, establécelo en **Disabled**.
10. Selecciona **Provision Microsoft Entra ID Users** y luego ve a **Attribute Mappings** para editar los atributos de la línea que contiene `externalId` y `mailNickname`.
11. En la pantalla **Edit Attribute**, cambia **Source attribute** a **objectId** y luego selecciona **OK**.
12. Vuelve a **Attribute Mappings** y selecciona la línea que contiene `emails[type eq "work"].value and mail`.
13. En la pantalla **Edit Attribute**, cambia **Match objects using this attribute** a **Yes**, luego establece **Matching precedence** en **2** y selecciona **OK.** La pantalla de mapeo de atributos se verá así mientras sigues usando la sección **Attribute Mappings** para configurar atributos SCIM adicionales:

<Frame>![Mapeo de atributos de SAML Azure](/docs/images/cdy7uua7fh8z/6JnGm4IeBglKiHNoVPX5vs/ddc17baeca48f02a6309b57ab87f845e/attributemap2.jpg)</Frame>

**Save** el mapeo de atributos y, a continuación, selecciona **X** en la esquina superior derecha para volver a la pantalla **aprovisionamiento**.

<div id="testing">
  ## Pruebas
</div>

1. En la pantalla **Información general** de la aplicación empresarial, ve a **Administrar &gt; Aprovisionamiento &gt; Aprovisionar a petición** para probar la conexión SCIM.
2. Ve a **Seleccionar un usuario o grupo** e introduce el nombre de un usuario o grupo que hayas asignado a la aplicación. Luego, selecciona el elemento y elige **Aprovisionar**. Esto crea el usuario o el grupo en el tenant de Auth0.
3. Establece **Estado de aprovisionamiento** en **Activado** para [aprovisionar a todos los usuarios y grupos asignados](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status).
