<Card title="Antes de empezar">
  Asegúrate de que tu tenant de Auth0 esté conectado a Azure AD mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

  Esta integración requiere registrar dos aplicaciones en Azure AD: la integración de OpenID Connect y la integración de SCIM. Para agilizar este proceso de configuración para tus clientes, considera [publicar tu aplicación en la galería de aplicaciones de Azure Active Directory](https://learn.microsoft.com/en-us/azure/active-directory/develop/v2-howto-app-gallery-listing).
</Card>

Auth0 puede integrarse con Microsoft Azure Active Directory (ahora conocido como Microsoft Entra ID) mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2), que usa el protocolo <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) para la autenticación de usuario. En esta sección se describe cómo configurar una aplicación empresarial fuera de la galería para administrar cuentas de usuario en modo de solo actualización y eliminación, lo que permite usar SCIM con conexiones antiguas de Azure AD en las que están presentes los [identificadores pairwise](https://learn.microsoft.com/en-us/entra/identity-platform/id-token-claims-reference#use-claims-to-reliably-identify-a-user).

Para conexiones nuevas, consulta [Nuevas conexiones](/docs/es/authenticate/protocols/scim/inbound-scim-for-new-azure-ad-connections).

<div id="configure-scim-settings-in-auth0">
  ### Configurar la configuración de SCIM en Auth0
</div>

1. En el [Auth0 Dashboard](https://manage.auth0.com/#/), vaya a **Authentication &gt; Enterprise &gt; Microsoft Azure AD &gt; [your-connection] &gt; Settings.**
2. Asegúrese de que **User ID Attribute Type** esté configurado como **Pairwise Subject Identifier (sub)** y de que el interruptor **Use Common Endpoint** esté **Disabled**.
3. Seleccione la pestaña **Provisioning** y habilite **Sync user profile attributes at each login** y **Sync user profiles using SCIM**.
4. En la pestaña **Mapping**, asegúrese de que la opción **SCIM attribute containing the User ID** esté configurada como **externalId**.
5. Revise los **Additional Mappings** para asegurarse de que los atributos SCIM extendidos estén mapeados a los atributos de Auth0 que prefiera. Consulte [mapeo de atributos](/docs/es/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para obtener más información.

<div id="retrieve-scim-endpoint-url-and-token">
  ### Obtener la URL del endpoint de SCIM y el token
</div>

Esta sección usa el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, pero también puedes completar estos pasos con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Consulta la sección [Directrices de implementación](/docs/es/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines) para conocer las prácticas recomendadas.

1. En el Auth0 Dashboard, ve a la pestaña **Setup** de SCIM y copia la **SCIM Endpoint URL**. Toma nota de esta URL, ya que tendrás que proporcionarla en un paso posterior.
2. Genera un token de SCIM haciendo clic en **Generate New Token** y, si lo deseas, establece una fecha de vencimiento para el token.
3. Selecciona los siguientes scopes: `get:users`, `patch:users` y `delete:users`.

<div id="configure-scim-in-azure-ad-for-oidc-apps">
  ### Configurar SCIM en Azure AD para aplicaciones OIDC
</div>

1. Confirme que ya se haya registrado una aplicación de OpenID Connect para su aplicación en la sección **Microsoft Entra ID &gt; App registrations** del [portal de Azure](https://portal.azure.com).
2. A continuación, registre una nueva aplicación **Non-gallery** en el [portal de Azure](https://portal.azure.com). Para ello, vaya a **Microsoft Entra ID &gt; Enterprise applications &gt; New application &gt; Create your own application,** introduzca un nombre para la aplicación y seleccione **Create**.
3. Vaya a la pestaña **Users and Groups** y asigne los usuarios y grupos de Azure AD que actualmente tienen acceso.
4. Seleccione la pestaña **Provisioning**, luego **Get started**, y elija **Automatic** como **Provisioning Mode.**
5. Seleccione **Admin Credentials** y, a continuación, introduzca el valor de **SCIM Endpoint URL** que guardó anteriormente como **Tenant URL**. Al final de la URL, agregue el parámetro de consulta `?aadOptscim062020` para corregir [problemas conocidos de Azure AD](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/application-provisioning-config-problem-scim-compatibility).
6. Pegue el valor del token en el campo **Secret Token** y seleccione **Save.**
7. Vaya a **Mappings** y seleccione **Provision Azure Active Directory Users,** luego busque **Provision Azure Active Directory Users** y desmarque **Create** en **Target Object Actions.**

   <Frame>![Mapeo de Azure AD para Inbound SCIM ](/docs/images/cdy7uua7fh8z/3ndwqpnyZvhWYSn2nvM2Qj/19e568c435d90aa8a294140f482b2257/azureadmappings.jpg)</Frame>
8. Vaya a **Attribute Mappings** y edite los atributos de la línea que contiene `externalId` y `mailNickname`
9. En la pantalla **Edit Attribute**, cambie **Source attribute** a **objectId,** y luego seleccione **OK**.
10. Vuelva a **Attribute Mappings** y seleccione la línea que contiene `emails[type eq "work"].value` y `mail`
11. En la pantalla **Edit Attribute**, cambie **Match objects using this attribute** a **Yes,** luego establezca **Matching precedence** en **2** y seleccione **OK.**

    <Frame>![Editar atributo en Azure AD para Inbound SCIM ](/docs/images/cdy7uua7fh8z/7EbSKlTnXlpQdqw6R5blBJ/01999edb569825e07004e0c2eb11508b/editattribute.jpg)</Frame>
12. Seleccione **Save** para guardar el mapeo de atributos. Aparecerá la siguiente vista:

    <Frame>![Mapeo de atributos en Azure para Inbound SCIM](/docs/images/cdy7uua7fh8z/2xCmuXkW7GGF8n3C84jOwI/3e70b7ab835d8dd73b36fdffbc3f5eb6/Attributemapping.jpg)</Frame>
13. Seleccione **X** en la esquina superior derecha para volver a la pantalla **Provisioning**.

<div id="testing">
  #### Pruebas
</div>

1. En la pantalla de **Información general** de la aplicación empresarial, seleccione **Manage &gt; Provisioning** y luego **Provision on Demand** para probar la conexión SCIM.
2. Vaya a **Select a user or group** y escriba el nombre de un usuario que asignó a la aplicación; después, selecciónelo y elija **Provision**. Si el usuario no existe en el tenant de Auth0, se mostrará un error. Si el usuario existe en el tenant de Auth0, aparecerá un mensaje que confirma que se actualizó.
3. Active el aprovisionamiento siguiendo las [instrucciones de Microsoft](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status) para establecer **Provisioning Status** en **On**.
