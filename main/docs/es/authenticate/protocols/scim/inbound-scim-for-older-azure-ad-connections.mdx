<Card title="Antes de empezar">
  Asegúrese de que su tenant de Auth0 esté conectado a Azure AD mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

  Esta integración requiere registrar dos aplicaciones en Azure AD: la integración de OpenID Connect y la integración de SCIM. Para agilizar este proceso de configuración para sus clientes, considere [publicar su aplicación en la galería de aplicaciones de Azure Active Directory](https://learn.microsoft.com/en-us/azure/active-directory/develop/v2-howto-app-gallery-listing).
</Card>

Auth0 puede integrarse con Microsoft Azure Active Directory (ahora conocido como Microsoft Entra ID) mediante el [tipo de conexión Microsoft Azure AD](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2), que utiliza el protocolo <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) para la autenticación de usuario. En esta sección se describe cómo configurar una aplicación empresarial fuera de la galería para administrar cuentas de usuario en modo de solo actualización y eliminación, lo que permite usar SCIM con conexiones antiguas de Azure AD en las que hay [identificadores por pares](https://learn.microsoft.com/en-us/entra/identity-platform/id-token-claims-reference#use-claims-to-reliably-identify-a-user).

Para las conexiones nuevas, lea [Conexiones nuevas](/docs/es/authenticate/protocols/scim/inbound-scim-for-new-azure-ad-connections).

<div id="configure-scim-settings-in-auth0">
  ### Configura los ajustes de SCIM en Auth0
</div>

1. En tu [Auth0 Dashboard](https://manage.auth0.com/#/), ve a **Authentication &gt; Enterprise &gt; Microsoft Azure AD &gt; [tu-conexión] &gt; Settings.**
2. Asegúrate de que **User ID Attribute Type** esté configurado como **Pairwise Subject Identifier (sub)** y de que el interruptor **Use Common Endpoint** esté **Disabled**.
3. Selecciona la pestaña **aprovisionamiento** y habilita **Sincronizar atributos del perfil de usuario en cada inicio de sesión** y **Sync user profiles using SCIM**.
4. En la pestaña **Mapping**, asegúrate de que la opción **SCIM attribute containing the User ID** esté configurada como **externalId**.
5. Revisa **Additional Mappings** para asegurarte de que los atributos SCIM extendidos estén mapeados a los atributos de Auth0 que prefieras. Consulta [mapeo de atributos](/docs/es/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para obtener más detalles.

<div id="retrieve-scim-endpoint-url-and-token">
  ### Recuperar la URL del endpoint de SCIM y el token
</div>

Esta sección usa el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, pero también puedes completar estos pasos con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Consulta la sección [Deployment Guidelines](/docs/es/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines) para conocer las prácticas recomendadas.

1. En el Auth0 Dashboard, ve a la pestaña **Setup** de SCIM y copia la **SCIM Endpoint URL**. Toma nota de esta URL, ya que tendrás que proporcionarla más adelante.
2. Genera un token de SCIM haciendo clic en **Generate New Token** y, si lo deseas, establece una fecha de vencimiento para el token.
3. Selecciona los siguientes scopes: `get:users`, `patch:users` y `delete:users`.

<div id="configure-scim-in-azure-ad-for-oidc-apps">
  ### Configurar SCIM en Azure AD para aplicaciones OIDC
</div>

1. Confirma que ya se haya registrado una aplicación OpenID Connect para tu aplicación en la sección **Microsoft Entra ID &gt; Registros de aplicaciones** del [portal de Azure](https://portal.azure.com).
2. A continuación, registra una nueva aplicación **No incluida en la galería** en el [portal de Azure](https://portal.azure.com). Para ello, ve a **Microsoft Entra ID &gt; Aplicaciones empresariales &gt; Nueva aplicación &gt; Crea tu propia aplicación,** introduce un nombre para la aplicación y selecciona **Crear**.
3. Ve a la pestaña **Usuarios y grupos** y asigna los usuarios y grupos de Azure AD que actualmente tienen acceso.
4. Selecciona la pestaña **Aprovisionamiento**, luego **Comenzar**, y elige **Automático** como **Modo de aprovisionamiento.**
5. Selecciona **Credenciales de administrador** y, a continuación, introduce como **URL del tenant** el valor de **SCIM Endpoint URL** que guardaste antes. Al final de la URL, agrega el parámetro de consulta `?aadOptscim062020` para corregir [incidencias conocidas de Azure AD](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/application-provisioning-config-problem-scim-compatibility).
6. Pega el valor del token en el campo **Secret Token** y selecciona **Guardar.**
7. Ve a **Mapeos** y selecciona **Aprovisionar usuarios de Azure Active Directory**; luego, busca **Aprovisionar usuarios de Azure Active Directory** y desmarca **Crear** en **Acciones de objeto de destino.**

   <Frame>![Mapeos de Azure AD para Inbound SCIM ](/docs/images/cdy7uua7fh8z/3ndwqpnyZvhWYSn2nvM2Qj/19e568c435d90aa8a294140f482b2257/azureadmappings.jpg)</Frame>
8. Ve a **Mapeo de atributos** y edita los atributos de la línea que contiene `externalId` y `mailNickname`
9. En la pantalla **Editar atributo**, cambia **Atributo de origen** a **objectId** y luego elige **Aceptar**.
10. Vuelve a **Mapeo de atributos** y selecciona la línea que contiene `emails[type eq "work"].value` y `mail`
11. En la pantalla **Editar atributo**, cambia **Hacer coincidir objetos usando este atributo** a **Sí**, luego establece **Precedencia de coincidencia** en **2** y elige **Aceptar.**

    <Frame>![Editar atributo en Azure AD para Inbound SCIM ](/docs/images/cdy7uua7fh8z/7EbSKlTnXlpQdqw6R5blBJ/01999edb569825e07004e0c2eb11508b/editattribute.jpg)</Frame>
12. Elige **Guardar** para guardar el mapeo de atributos. Aparecerá la siguiente vista:

    <Frame>![Mapeo de atributos en Azure para Inbound SCIM](/docs/images/cdy7uua7fh8z/2xCmuXkW7GGF8n3C84jOwI/3e70b7ab835d8dd73b36fdffbc3f5eb6/Attributemapping.jpg)</Frame>
13. Selecciona **X** en la esquina superior derecha para volver a la pantalla de **Aprovisionamiento**.

<div id="testing">
  #### Pruebas
</div>

1. En la pantalla **Información general** de la aplicación empresarial, selecciona **Administrar &gt; Aprovisionamiento** y luego **Aprovisionar a petición** para probar la conexión SCIM.
2. Ve a **Seleccionar un usuario o grupo** y escribe el nombre de un usuario que hayas asignado a la aplicación; después, selecciona el usuario y elige **Aprovisionar**. Si el usuario no está presente en el tenant de Auth0, recibirás un error. Si el usuario está presente en el tenant de Auth0, aparecerá un mensaje que confirma que el usuario se ha actualizado.
3. Activa el aprovisionamiento siguiendo [las instrucciones de Microsoft](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status) para establecer el **Estado de aprovisionamiento** en **Activado**.
