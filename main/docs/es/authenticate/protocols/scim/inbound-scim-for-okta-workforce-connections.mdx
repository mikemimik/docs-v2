<Card title="Antes de empezar">
  Habilita Okta Workforce Identity como proveedor de identidad siguiendo las instrucciones de la [página de ayuda de Identity Providers](/docs/es/authenticate/identity-providers/enterprise-identity-providers/okta).

  Esta integración requiere registrar dos aplicaciones en Okta Workforce: la integración de OpenID Connect y la integración de SCIM. Los mismos usuarios y grupos deben estar asignados a ambas. Para eliminar este requisito y agilizar el proceso de configuración para tus clientes, [envía tu aplicación a la Okta Integration Network](https://developer.okta.com/docs/guides/submit-app-overview/).
</Card>

En esta sección se describe cómo configurar una integración de aplicación personalizada de <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect y SCIM en un tenant de Okta Workforce Identity, que puede usarse para aprovisionar usuarios en tu endpoint SCIM de Auth0.

Para obtener información sobre cómo configurar una integración de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> y SCIM con Okta Workforce Identity, consulta [Inbound SCIM para conexiones SAML de Okta Workforce](/docs/es/authenticate/protocols/scim/inbound-scim-for-okta-workforce-saml-connections) en su lugar.

<div id="configure-scim-settings-in-auth0">
  ## Configurar la configuración de SCIM en Auth0
</div>

1. Abre el [Auth0 Dashboard](https://manage.auth0.com/#/) y ve a **Authentication &gt; Enterprise &gt; Okta Workforce &gt; [your-connection] &gt; Provisioning.**

2. Desactiva **Sincronizar atributos del perfil de usuario en cada inicio de sesión**, a menos que quieras [sincronizar atributos adicionales al iniciar sesión](/docs/es/authenticate/protocols/scim/configure-inbound-scim).

3. En la misma sección, activa **Sincronizar perfiles de usuario con SCIM**.

4. En la pestaña **Mapeo**, asegúrate de que el **atributo SCIM que contiene el ID de usuario** esté configurado como **externalId**.

También puedes revisar **Mapeos adicionales** para asegurarte de que los atributos SCIM ampliados estén mapeados a los atributos de Auth0 que prefieras.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Obtener la URL del endpoint de SCIM y el token
</div>

1. En el Auth0 Dashboard, ve a la pestaña **Setup** de SCIM y copia la **URL del endpoint de SCIM**; luego, pégala en un lugar seguro.

2. Selecciona **Generate New Token** y establece una fecha de vencimiento opcional para el token. También puedes seleccionar los scopes que quieras conceder a Okta Workforce; de forma predeterminada, se requieren los scopes que usa Okta Workforce: `get:users`, `post:users` y `put:users`. También puedes seleccionar `get:groups`, `post:groups`, `put:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.

<div id="configure-scim-in-okta-workforce-for-oidc-apps">
  ## Configurar SCIM en Okta Workforce para aplicaciones OIDC
</div>

1. Confirma que ya se haya [registrado](https://help.okta.com/en-us/content/topics/apps/apps_app_integration_wizard_oidc.htm) una aplicación de OpenID Connect en el tenant de Okta Workforce para la autenticación de usuario basada en OIDC.

2. Confirma que tu aplicación de OpenID Connect [tenga desactivado Federation Broker Mode](https://help.okta.com/en-us/content/topics/apps/apps-fbm-main.htm).

3. Registra una segunda aplicación en el portal de Okta seleccionando **Applications &gt; Browse App Catalog**, busca **(OAuth Bearer Token) Governance with SCIM 2.0** y selecciona **Add Integration**. Define una **etiqueta de la aplicación**, marca **Do not display application icon to users**, desmarca **Browser plugin auto-submit** y luego selecciona **Next** y **Done**.

4. En la página de configuración de la integración, selecciona la pestaña **Provisioning** y luego **Configure API Integration**. Marca **Enable API Integration**. Copia la **URL del endpoint de SCIM** desde la pestaña **Setup** del Auth0 Dashboard y pégala en el campo **SCIM 2.0 Base URL**, eliminando cualquier barra diagonal invertida al final. En el Auth0 Dashboard, selecciona **Generate New Token &gt; Generate Token &gt; Copy and Close** y luego pega el token en el campo **OAuth Bearer Token** de Okta. Si lo deseas, selecciona **Test API Credentials** y luego **Save**.

5. Junto a **Provisioning To App**, selecciona **Edit**. Habilita solo estas opciones: **Create Users**, **Update User Attributes** y **Deactivate Users**. Desmarca **Set password when creating new users** y luego selecciona **Save**.

6. En **Mapeo de atributos**, usa el botón **X** para eliminar estas líneas, ya que pueden causar problemas durante las operaciones `PUT`:

   | Atributo                             | Valor                                                                    |
   | ------------------------------------ | ------------------------------------------------------------------------ |
   | Tipo de correo electrónico principal | `(user.email != null && user.email != '') ? 'work' : ''`                 |
   | Tipo de teléfono principal           | `(user.primaryPhone != null && user.primaryPhone != '') ? 'work' : ''`   |
   | Tipo de dirección                    | `(user.streetAddress != null && user.streetAddress != '') ? 'work' : ''` |

   Si lo deseas, configura atributos SCIM adicionales para enviarlos desde Okta a tu endpoint de SCIM.

7. Usa la pestaña **Assignments** para asignar usuarios y/o grupos a tu aplicación. Al asignar un grupo, sus usuarios se aprovisionan en tu tenant de Auth0.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Los mismos grupos y usuarios deben estar asignados tanto a la instancia de la aplicación SSO como a la instancia de la aplicación SCIM en Okta.
   </Callout>

8. Para enviar grupos, selecciona la pestaña **Push Groups**, busca y selecciona el grupo que quieres enviar y luego selecciona **Save**. Esto aprovisiona el objeto de grupo completo en Auth0. Para obtener más información sobre la relación entre Group Push y las asignaciones, consulta [la documentación de Okta sobre App assignments y Group Push](https://help.okta.com/en-us/content/topics/users-groups-profiles/usgp-group-push-main.htm).
