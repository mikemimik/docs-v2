El protocolo <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin necesidad de contraseña." cta="Ver glosario" href="/docs/es/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) es un marco abierto basado en XML para la autenticación y autorización entre dos entidades sin necesidad de contraseña:

* El **proveedor de servicios** (SP) acuerda confiar en el <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin necesidad de contraseña." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> para autenticar a los usuarios.
* El **proveedor de identidad** (IdP) autentica a los usuarios y proporciona a los proveedores de servicios una aserción de autenticación que indica que un usuario ya ha sido autenticado.

Auth0 admite el protocolo SAML y puede actuar como IdP, SP o ambos, incluidos:

* aplicaciones web SAML2
* integraciones SSO de SAML <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip>
* SSO iniciado por IdP
* Microsoft Active Directory Federation Services (ADFS)
* firma y cifrado de solicitudes SAML

<div id="supported-saml-bindings-and-options">
  ## Bindings y opciones de SAML compatibles
</div>

Auth0 admite los siguientes bindings de SAML:

* HTTP Redirect
* HTTP POST

Auth0 admite las siguientes opciones de SAML:

* Perfil de SSO con navegador web
* Perfil de cierre de sesión único
* Perfil de gestión del identificador de nombre
* Perfil de asignación del identificador de nombre
