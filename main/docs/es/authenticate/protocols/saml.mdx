El protocolo <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite que dos partes intercambien información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) es un marco abierto basado en XML para la autenticación y autorización entre dos entidades sin contraseña:

* El **proveedor de servicios** (SP) confía en el <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite que dos partes intercambien información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> para autenticar a los usuarios.
* El **proveedor de identidad** (IdP) autentica a los usuarios y proporciona a los proveedores de servicios una aserción de autenticación que indica que un usuario ha sido autenticado.

Auth0 admite el protocolo SAML y puede actuar como IdP, como SP o como ambos, lo que incluye:

* aplicaciones web SAML2
* integraciones de <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">inicio de sesión único</Tooltip> de SAML
* inicio de sesión único iniciado por IdP
* Microsoft Active Directory Federation Services (ADFS)
* firma y cifrado de solicitudes SAML

<div id="supported-saml-bindings-and-options">
  ## Bindings y opciones de SAML compatibles
</div>

Auth0 admite los siguientes bindings de SAML:

* HTTP Redirect
* HTTP POST

Auth0 admite las siguientes opciones de SAML:

* Perfil de SSO de navegador web
* Perfil de cierre de sesión único
* Perfil de administración del Identificador de nombre
* Perfil de mapeo del Identificador de nombre
