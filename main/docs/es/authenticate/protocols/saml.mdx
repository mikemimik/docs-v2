El protocolo <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) es un marco de trabajo abierto basado en XML para la autenticación y autorización entre dos entidades sin necesidad de una contraseña:

* **proveedor de servicios** (SP) acepta confiar en el <Tooltip tip="Proveedor de identidad: servicio externo de identidad que autentica a los usuarios y proporciona información de identidad a otra aplicación o plataforma." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> para autenticar a los usuarios.
* **Proveedor de identidad** (IdP) autentica a los usuarios y proporciona a los proveedores de servicios una aserción de autenticación que indica que un usuario ha sido autenticado.

Auth0 es compatible con el protocolo SAML y puede actuar como IdP, como SP o como ambos, incluidos:

* aplicaciones web SAML2
* integraciones de SAML <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip>
* SSO iniciado por IdP
* Microsoft Active Directory Federation Services (ADFS)
* firma y cifrado de solicitudes SAML

<div id="supported-saml-bindings-and-options">
  ## Bindings y opciones de SAML admitidos
</div>

Auth0 admite los siguientes bindings de SAML:

* HTTP Redirect
* HTTP POST

Auth0 admite las siguientes opciones de SAML:

* Perfil de SSO del navegador web
* Perfil de cierre de sesión único
* Perfil de gestión del Identificador de nombre
* Perfil de mapeo del Identificador de nombre
