Auth0 ofrece conexiones empresariales para autenticar a los usuarios con un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> (IdP) externo y federado, como Azure AD, Google Workspace, PingFederate, entre otros.

<Card title="La disponibilidad varía según el plan de Auth0">
  Tu plan de Auth0 o tu acuerdo personalizado afectan la disponibilidad de esta funcionalidad. Para obtener más información, consulta [la página de precios de Auth0](https://auth0.com/pricing).
</Card>

<div id="create-an-enterprise-connection">
  ## Crear una conexión empresarial
</div>

Auth0 admite muchos proveedores de identidad de forma nativa. Para obtener más información, consulta [Proveedores de identidad empresariales](/docs/es/authenticate/identity-providers/enterprise-identity-providers).

<div id="openid-connect-oidc-protocol">
  ### Protocolo OpenID Connect (OIDC)
</div>

Las conexiones empresariales que usan <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect u Okta Workforce como proveedor de identidad pueden admitir lo siguiente:

* [Proof Key for Code Exchange (PKCE)](https://www.oauth.com/oauth2-servers/pkce/)

* Claims de atributos y mapeo del <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>

* Integración con UserInfo

Puede implementar estas funcionalidades en las conexiones de [OpenID Connect](/docs/es/authenticate/identity-providers/enterprise-identity-providers/oidc) o [Okta Workforce](/docs/es/authenticate/identity-providers/enterprise-identity-providers/okta). Para obtener más información, consulte [Configure PKCE and Claim Mapping for OIDC Connections](/docs/es/authenticate/identity-providers/enterprise-identity-providers/configure-pkce-claim-mapping-for-oidc).

<div id="view-enterprise-connections">
  ## Ver conexiones empresariales
</div>

<Tabs>
  <Tab title="Dashboard">
    Ve a [**Auth0 Dashboard &gt; Autenticación &gt; empresarial**](https://manage.auth0.com/#/connections/enterprise) para ver todos los tipos de conexión empresarial disponibles. Selecciona un tipo de conexión (por ejemplo, SAML) para comprobar si hay conexiones configuradas de ese tipo.

    También puedes seleccionar una conexión configurada y consultar la pestaña **Applications** para ver si está habilitada para alguna aplicación.
  </Tab>

  <Tab title="Management API">
    Llama al endpoint [Get all Connections de Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Connections/get_connections) para obtener información sobre tus conexiones. Incluye el parámetro `strategy` para filtrar por tipo de conexión.

    Los valores de `strategy` para el tipo de conexión empresarial son:

    * `ad` (Active Directory/LDAP)
    * `adfs` (ADFS)
    * `auth0-adldap`
    * `google-apps` (Google Workspace)
    * `ip`
    * `office365`
    * `oidc` (OpenID Connect)
    * `pingfederate` (PingFederate)
    * `samlp` (SAML)
    * `sharepoint`
    * `waad` (Microsoft Azure AD)
  </Tab>
</Tabs>

<div id="what-is-an-active-enterprise-connection">
  ## ¿Qué es una conexión empresarial activa?
</div>

Una conexión empresarial se considera activa si, durante el mes en curso, cumple ambas condiciones:

* Haber sido habilitada para una aplicación.
* Haber tenido actividad de usuarios (por ejemplo, inicio de sesión, registro o intercambio de tokens).

Si una conexión empresarial nunca fue habilitada para ninguna aplicación, o fue habilitada pero no tuvo actividad de usuarios durante el mes en curso, no se considera activa.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La conexión empresarial de Okta Workforce no cuenta para el límite de conexiones empresariales activas.
</Callout>
