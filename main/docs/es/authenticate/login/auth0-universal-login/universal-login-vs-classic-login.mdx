---
title: Universal Login vs. Classic Login
description: Compara las funcionalidades disponibles en las experiencias de Universal Login y Classic Login
---

[Auth0 Universal Login](/docs/es/authenticate/login/auth0-universal-login) proporciona la funcionalidad esencial de un <Tooltip tip="Servidor de autorización: servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, su servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>: el flujo de inicio de sesión. Cuando un usuario necesita demostrar su identidad para acceder a su aplicación, puede redirigirlo a <Tooltip tip="Servidor de autorización: servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, su servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> y dejar que Auth0 gestione el proceso de autenticación.

Universal Login es la principal solución de inicio de sesión alojado de Auth0. [Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) ofrece herramientas de personalización fáciles de usar y brinda una experiencia más simple y rápida a los usuarios finales. Como alternativa, Auth0 también ofrece [Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience), una experiencia de inicio de sesión alojado que utiliza controles de JavaScript para la personalización.

<Warning>
  En este momento, los esfuerzos de desarrollo activo de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. A menos que su caso de uso específico requiera la experiencia Classic, se recomienda implementar Universal Login.
</Warning>

Para ayudarle a elegir la mejor solución para sus necesidades, la siguiente tabla compara las funcionalidades de ambas experiencias de inicio de sesión.

| **Solución No-Code**                                                                    | Sí, mediante el [editor sin código](/docs/es/customize/login-pages/universal-login/customize-themes)                                                                                     | No                                                                                                                                                                               |
| **Autenticación passwordless**                                                          | Sí                                                                                                                                                                                    | Sí, con [SMS o correo electrónico](/docs/es/authenticate/login/auth0-universal-login/passwordless-login)                                                                            |

| **Cumplimiento de las Web Content Accessibility Guidelines (WCAG)**                     | Sí                                                                                                                                                                                    | No                                                                                                                                                                               |

| **MFA por correo electrónico**                                                          | [Sí](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)                                                              | No                                                                                                                                                                               |
| **MFA por voz**                                                                         | [Sí](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)                                                              | No                                                                                                                                                                               |
