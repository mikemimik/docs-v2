title: Configurar WebAuthn con biometría del dispositivo para la autenticación passwordless
---

Puedes configurar <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> para que los usuarios puedan autenticarse mediante [WebAuthn](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn) con biometría del dispositivo en lugar de usar una contraseña.

WebAuthn con biometría del dispositivo es el factor de autenticación más seguro y fácil de usar disponible actualmente, y reduce considerablemente la fricción en el inicio de sesión sin comprometer la seguridad.

<Warning>
  Debes usar esta función con una conexión de base de datos de Auth0. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) para elegir una conexión de base de datos. Para obtener más información, consulta [Database Connections](/docs/es/authenticate/database-connections).
</Warning>

<div id="prerequisite">
  ## Requisito previo
</div>

Para habilitar <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> con WebAuthn biometría del dispositivo, debes:

1. Asegúrate de que la experiencia de Universal Login esté habilitada y de que el HTML de la página de login no esté personalizado en [Dashboard &gt; Universal Login](https://manage.auth0.com/#/login_settings).
2. Selecciona **Identifier First + Biometrics** en [Dashboard &gt; Authentication Profile](https://manage.auth0.com/#/authentication-profiles). Esto [habilitará automáticamente WebAuthn con biometría del dispositivo](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa) en la sección de autenticación multifactor si aún no está habilitado.
3. Si usas una [conexión de base de datos personalizada](/docs/es/authenticate/database-connections/custom-db), asegúrate de que **Import Mode** esté establecido en **On.** Si no es así, puedes ejecutar el script [getUser](/docs/es/authenticate/database-connections/custom-db/templates/get-user) para conseguir el mismo resultado.

<div id="how-does-it-work">
  ## Cómo funciona
</div>

A los usuarios que se autentican con username/correo electrónico y Password, y tienen un dispositivo [compatible con WebAuthn con biometría del dispositivo](https://webauthn.me/browser-support), se les ofrece la opción de registrar su dispositivo:

<Frame>![Ejemplo de configuración de un inicio de sesión con Face ID para un dominio específico mediante WebAuthn](/docs/images/cdy7uua7fh8z/1JfsIYEo3O7xmTAxLRwNSs/2f2ba478ff32b0aa86f4f01cd6c0cf3b/2023-01-31_16-34-09.png)</Frame>

Después de habilitar esta función, ofrecemos varias opciones a los usuarios en el cuadro de diálogo Iniciar sesión más rápido en este dispositivo. Los usuarios pueden elegir registrar su dispositivo u omitir el registro para reducir la cantidad de veces que se les solicita hacerlo:

| Opción               | Descripción                                                                                                                         |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `Continuar`          | Solicita al usuario el registro del factor biométrico                                                                               |
| `Remind me later`    | Omite el registro y vuelve a solicitarlo dentro de dos semanas                                                                      |
| `Not on this device` | No vuelve a solicitar el registro durante 1 año o mientras las cookies de Auth0 permanezcan almacenadas en el navegador del usuario |

<Frame>![Inicio de sesión: biometría de WebAuthn; iniciar sesión más rápido en este dispositivo](/docs/images/cdy7uua7fh8z/1w3jwsf0ihbTD0BNudRXN9/6ee65674ffa41e5e9537129bb3f98e79/2023-01-31_16-34-09.png)</Frame>

Si los clientes deciden registrar su dispositivo, la próxima vez que se autentiquen desde ese dispositivo tendrán la opción de usar la biometría del dispositivo o una contraseña:

<Frame>![Ejemplo de uso de huella digital o reconocimiento facial para iniciar sesión en un dominio](/docs/images/cdy7uua7fh8z/4DkewyodXBQ3gncybz7KPI/873d6a3eafb644ee605daa209006d1b3/Docs_Login.png)</Frame>

A esta función la llamamos &#39;inscripción progresiva&#39;, y está diseñada para que la transición a la autenticación basada en WebAuthn sea lo más sencilla posible tanto para los desarrolladores como para los usuarios.

<div id="multi-factor-authentication">
  ## Autenticación multifactor
</div>

WebAuthn con biometría del dispositivo permite evitar que se requiera otro método de autenticación para realizar la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además del username y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>. **WebAuthn con biometría del dispositivo combina dos factores en uno**: algo que tienes (el dispositivo) y algo que eres (biometría) o algo que sabes (el código de acceso).

Esto tiene varias consecuencias:

* Cuando habilitas MFA en el Dashboard, Auth0 no solicitará MFA si los usuarios se autenticaron con WebAuthn w/Biometrics como primer factor.
* Cuando MFA está habilitado y los usuarios crean una cuenta nueva, harán lo siguiente:

  * Crear un usuario con username/contraseña.
  * Inscribirse en MFA con un método de autenticación no biométrico, para que puedan completar MFA en cualquier dispositivo.
  * Opcionalmente, inscribirse en biometría del dispositivo.

La próxima vez que inicien sesión, podrán hacerlo con contraseña + otro método de autenticación o con biometría del dispositivo.

* Cuando los usuarios se autentican con WebAuthn Biometrics como su único método de autenticación, el valor `amr` en el <Tooltip tip="Token de ID: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> se establecerá en `mfa`.
* Si quieres habilitar MFA desde nuestra plataforma de extensibilidad, podrás tener en cuenta cómo se autenticaron los usuarios para decidir si se les debe solicitar MFA o no. La Rule de abajo solo ejecutará MFA si el usuario no se autenticó con el método de autenticación `webauthn-platform`:

```javascript javascript lines
function (user, context, callback) {
  let authMethods = context.authentication.methods;

  const amr = authMethods.find((method) => method.name === 'webauthn-platform');

  if (!amr) {
    context.multifactor = {
      provider: 'any',
      allowRememberBrowser: false
    };
  }
  return callback(null, user, context);
}
```

Esta Action de Post Login surtirá el mismo efecto:

```javascript javascript lines
exports.onExecutePostLogin = async (event, api) => {
  let authMethods = event.authentication.methods;

  let amr = authMethods.find((method) => method.name === 'webauthn-platform');

   if (!amr) {
     api.multifactor.enable('any');
  }
};
```

<div id="device-recognition">
  ## Reconocimiento de dispositivos
</div>

Auth0 usará las reglas para determinar si el dispositivo ya está registrado o no, y solicitará al usuario que lo registre. Para obtener más información, consulta [Reconocimiento de dispositivos](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa#device-recognition) en el artículo [Configurar WebAuthn con biometría del dispositivo para MFA](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa).

Para evitar ataques de enumeración de usuarios, Auth0 solo pedirá biometría a los usuarios como primer factor si inician sesión desde un dispositivo conocido. De lo contrario, tendrán que iniciar sesión con la contraseña.

Por ejemplo:

* Un usuario inicia sesión desde Chrome en Windows y está registrado en Windows Hello. Como parte de la información de registro, Auth0 sabe que el usuario se registró desde un dispositivo con Windows y almacena un &quot;dispositivo conocido&quot; para reconocer el agente de usuario.
* La próxima vez que el usuario inicie sesión desde Chrome, se le pedirá que use Windows Hello en lugar de una contraseña.
* Si más adelante el usuario inicia sesión desde Firefox en Windows, como la cookie de &quot;dispositivo conocido&quot; no estará presente, tendrá que iniciar sesión con su contraseña. Como ya está registrado en Windows Hello, no se le pedirá que vuelva a registrarse.
* La próxima vez que el usuario inicie sesión desde Firefox, se le pedirá que use Windows Hello.

Esto no permitirá que los atacantes sepan si los usuarios tienen una cuenta o no, ni si usaron biometría del dispositivo con WebAuthn para autenticarse en el sitio.
