title: Configurar correo electrónico o SMS para la autenticación Passwordless
---

Auth0 te permite configurar <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> para que los usuarios puedan autenticarse con un Magic Link o una contraseña de un solo uso (OTP) por correo electrónico, o con un OTP por SMS.

Para habilitar la autenticación <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> mediante SMS o correo electrónico, debes definir una [conexión Passwordless](/docs/es/authenticate/passwordless) para tu aplicación.

Una vez creada la conexión, debes configurar Universal Login para usarla. Tienes tres opciones:

<div id="universal-login">
  ## Universal Login
</div>

Este método utiliza la experiencia de Universal Login, que admite de forma nativa conexiones passwordless. Para obtener más información, consulta [Passwordless con Universal Login](/docs/es/authenticate/passwordless/passwordless-with-universal-login).

<div id="universal-login-lock-passwordless">
  ## Universal Login + Lock (passwordless)
</div>

Este método usa una página de inicio de sesión personalizada con la plantilla **Lock (passwordless)** y autentica al usuario con el [SDK de Lock for Web](/docs/es/libraries/lock):

1. Ve a [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings) y haz clic en la pestaña **Login**.
2. Activa el interruptor de **página de inicio de sesión personalizada** y selecciona la plantilla **Lock (passwordless)**. La plantilla HTML se actualizará con código que usa el widget Lock con opciones de personalización de Passwordless.
3. Personaliza la plantilla y haz clic en **Guardar cambios**.

Puedes usar HTML y CSS para personalizar el formulario de inicio de sesión y obtener una vista previa de los cambios en el Dashboard. Para obtener más información sobre cómo personalizar la plantilla **Lock (passwordless)**, consulta la sección Passwordless en [SDK de Lock for Web](/docs/es/libraries/lock).

<div id="universal-login-custom-ui-auth0js">
  ## Universal Login + UI personalizada + Auth0.js
</div>

Este método utiliza una página de inicio de sesión personalizada con la plantilla **Custom Login Form** y autentica a los usuarios con el [SDK de Auth0.js](/docs/es/libraries/auth0js). Para obtener más información, consulta [Autenticación passwordless con Universal Login](/docs/es/authenticate/passwordless/implement-login/universal-login).
