description: Describe la información que se muestra en la página de error predeterminada de Auth0 y los parámetros utilizados para los eventos de error de autorización.
title: Página de error predeterminada de Auth0
---

Si tu URL de devolución de llamada es válida, cuando se produzca un error de autorización, el <Tooltip tip="Servidor de autorización: servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Authorization+Server">servidor de autorización</Tooltip> devolverá a tu URL de devolución de llamada los parámetros de error y state correspondientes. En los casos en que tu URL de devolución de llamada no sea válida, tu aplicación mostrará la página de error genérica predeterminada de Auth0. Como alternativa, puedes personalizar las páginas de error. Para obtener más información, consulta [Custom Error Pages](/docs/es/customize/login-pages/custom-error-pages).

Para encontrar el nombre predeterminado de la página de error genérica, consulta [Usa el control de versiones para gestionar tus páginas de Universal Login](/docs/es/customize/login-pages/classic-login/version-control).

<div id="display">
  ## Visualización
</div>

La página de error genérica muestra la siguiente información si ha configurado los campos correspondientes en la [Configuración del tenant](https://manage.auth0.com/#/tenant/).

| Campo                         | Descripción                                                         |
| ----------------------------- | ------------------------------------------------------------------- |
| Nombre descriptivo            | Nombre de su empresa                                                |
| URL del logotipo              | URL del logotipo de su empresa                                      |
| Correo electrónico de soporte | Dirección de correo electrónico del equipo de soporte de su empresa |
| URL de soporte                | URL de la página de soporte de su empresa                           |

<div id="parameters">
  ## Parámetros
</div>

El servidor de autorización devuelve parámetros añadidos a la URL en la cadena de consulta.

| Parámetro           | Descripción                                                                       |
| ------------------- | --------------------------------------------------------------------------------- |
| `client_id`         | Identificador de la aplicación de Auth0.                                          |
| `connection`        | Conexión utilizada en el momento del error.                                       |
| `lang`              | Idioma configurado para su uso en el momento del error.                           |
| `error`             | Código del error.                                                                 |
| `error_description` | Descripción del error.                                                            |
| `tracking`          | Identificador que Auth0 utiliza para encontrar errores en los registros internos. |

Los parámetros que se muestran varían según el tipo de error y son específicos de la solicitud. Por ejemplo, si la solicitud no contenía un `client_id` y eso produjo un error, la página de error no devolverá ningún <Tooltip tip="ID de cliente: Valor de identificación otorgado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>.
