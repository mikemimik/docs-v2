title: Página de error predeterminada de Auth0
---

Si su URL de devolución de llamada es válida, cuando se produce un error de autorización, el <Tooltip tip="Servidor de autorización: servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, su servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Authorization+Server">servidor de autorización</Tooltip> devuelve los parámetros de error y de state correspondientes a su URL de devolución de llamada. Si su URL de devolución de llamada no es válida, su aplicación mostrará la página de error genérica predeterminada de Auth0. También puede personalizar las páginas de error. Para obtener más información, consulte [Páginas de error personalizadas](/docs/es/customize/login-pages/custom-error-pages).

Para encontrar el nombre predeterminado de la página de error genérica, consulte [Use el control de versiones para administrar sus páginas de Universal Login](/docs/es/customize/login-pages/classic-login/version-control).

<div id="display">
  ## Visualización
</div>

La página de error genérica muestra la siguiente información si has configurado los campos correspondientes en la [Configuración del tenant](https://manage.auth0.com/#/tenant/).

| Campo                         | Descripción                                                         |
| ----------------------------- | ------------------------------------------------------------------- |
| Nombre descriptivo            | Nombre de tu empresa                                                |
| URL del logotipo              | URL del logotipo de tu empresa                                      |
| Correo electrónico de soporte | Dirección de correo electrónico del equipo de soporte de tu empresa |
| URL de soporte                | URL de la página de soporte de tu empresa                           |

<div id="parameters">
  ## Parámetros
</div>

El servidor de autorización devuelve parámetros añadidos a la URL como una cadena de consulta.

| Parámetro           | Descripción                                                                   |
| ------------------- | ----------------------------------------------------------------------------- |
| `client_id`         | Identificador de la aplicación de Auth0.                                      |
| `connection`        | Conexión utilizada en el momento del error.                                   |
| `lang`              | Idioma configurado para usarse en el momento del error.                       |
| `error`             | Código del error.                                                             |
| `error_description` | Descripción del error.                                                        |
| `tracking`          | Identificador que Auth0 usa para encontrar errores en los registros internos. |

Los parámetros que se muestran varían según el tipo de error y son específicos de la solicitud. Por ejemplo, si la solicitud no contenía un `client_id` y produjo un error, la página de error no devolverá ningún <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>.
