<Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">
  Universal Login
</Tooltip>

es la implementación de Auth0 del flujo de autenticación, que es la funcionalidad principal de un

<Tooltip tip="Authorization Server: Servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Authorization+Server">
  servidor de autorización
</Tooltip>

. Cada vez que un usuario necesita demostrar su identidad, tus aplicaciones redirigen a Universal Login y Auth0 garantiza la identidad del usuario. Puedes habilitar la autenticación

<Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">
  Passwordless
</Tooltip>

al configurar Universal Login en el

<Tooltip tip="Auth0 Dashboard: El principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">
  Auth0 Dashboard
</Tooltip>

.

Primero, determina si quieres que los usuarios usen SMS, correo electrónico o WebAuthn con biometría del dispositivo como método de autenticación Passwordless. Actualmente, no es posible admitir los tres métodos de autenticación en un solo flujo de inicio de sesión.

<div id="passwordless-with-email-or-sms">
  ## Passwordless con correo electrónico o SMS
</div>

Puedes configurar Auth0 para que los usuarios se autentiquen con un Magic Link o un código de un solo uso (OTP) enviado a su correo electrónico, o con un OTP enviado a su número de teléfono. Para obtener más información, consulta [Configurar correo electrónico o SMS para la autenticación passwordless](/docs/es/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="passwordless-with-webauthn-with-device-biometrics">
  ## passwordless con WebAuthn with Device Biometrics
</div>

Al usar WebAuthn with Device Biometrics, los usuarios podrán autenticarse con su contraseña o con cualquiera de sus dispositivos compatibles con WebAuthn with Device Biometrics.

Cada vez que inicien sesión desde un dispositivo nuevo, Auth0 les ofrecerá la opción de usar biometría en lugar de una contraseña. Para obtener más información, consulta [passwordless con WebAuthn with Device Biometrics](/docs/es/authenticate/login/auth0-universal-login/passwordless-login/webauthn-device-biometrics).
