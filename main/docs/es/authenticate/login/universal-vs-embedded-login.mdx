---
title: Inicio de sesión alojado vs. inicio de sesión integrado
description: Describe las diferencias entre el inicio de sesión alojado (Universal Login) y el inicio de sesión integrado.
---

Cuando diseñes la experiencia de inicio de sesión para tu aplicación, tendrás que decidir si quieres que sea alojado ([Universal Login](/docs/es/authenticate/login/auth0-universal-login)) o integrado.

<div id="hosted-login">
  ## Inicio de sesión alojado
</div>

En un flujo de inicio de sesión alojado, la aplicación redirige al usuario al <Tooltip tip="Servidor de autorización: Servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, su servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip> para autenticarse, y luego el servidor de autorización lo devuelve a la aplicación.

<div id="universal-login-flow">
  ### Flujo de Universal Login
</div>

En la experiencia típica de autenticación de usuario con <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>:

1. Tu aplicación inicia una solicitud de autenticación al invocar el método de login de un SDK de Auth0 o al llamar directamente al [endpoint de Login de Auth0 Authentication API](https://auth0.com/docs/api/authentication/login/).
2. Tu aplicación redirige al usuario al servidor de autorización de Auth0 (tu tenant de Auth0).
3. Auth0 comprueba si existe una cookie de sesión en la [capa de sesión de Auth0](/docs/es/manage-users/sessions/session-layers).

   * Si no se encuentra una cookie de sesión, Auth0 redirige al usuario a Universal Login para autenticarse y crea una cookie de sesión asociada.
   * Si se encuentra una cookie de sesión, Auth0 la utiliza para autenticar al usuario.
4. Auth0 devuelve al usuario a tu aplicación con un [token de identidad](/docs/es/secure/tokens/id-tokens) y un [token de acceso](/docs/es/secure/tokens/access-tokens).

<div id="embedded-login">
  ## Inicio de sesión integrado
</div>

En un flujo de inicio de sesión integrado, el usuario se autentica directamente en tu aplicación. La aplicación muestra el formulario de inicio de sesión, recopila las credenciales del usuario, envía esos datos al proveedor de autenticación y luego recibe una respuesta con el <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> y el <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> del usuario.

En las aplicaciones web, el inicio de sesión integrado usa [autenticación de origen cruzado](/docs/es/authenticate/login/cross-origin-authentication), a menos que [configures un dominio personalizado](/docs/es/customize/custom-domains) para tu tenant de Auth0. La autenticación de origen cruzado usa cookies de terceros para permitir transacciones de autenticación seguras entre distintos orígenes.

<div id="feature-comparison">
  ## Comparación de funcionalidades
</div>

| Funcionalidad                                                   | Alojado                                                                                                                                                                                                                                                                                                                                       | Integrado                                                                                                                                                                                                                                                                                                                                                                                               |
| --------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[inicio de sesión único](/docs/es/authenticate/single-sign-on)** | Compatibilidad completa con Universal Login mediante el uso de [cookies de sesión](/docs/es/manage-users/sessions) en el servidor de autorización de Auth0 (tu tenant de Auth0).                                                                                                                                                                 | Compatibilidad limitada. Las aplicaciones web que usan las bibliotecas [Lock](/docs/es/libraries/lock/lock-api-reference#checksession-) o [Auth0.js](/docs/es/libraries/auth0js#using-checksession-to-acquire-new-tokens) pueden compartir sesiones. Las aplicaciones nativas pueden compartir sesiones con aplicaciones web mediante [SSO de nativo a web](/docs/es/authenticate/single-sign-on/native-to-web). |
| **[Personalización](/docs/es/customize)**                          | Universal Login te permite personalizar fácilmente muchas partes de la experiencia (incluidas la tematización, las plantillas de página, los elementos de texto y los prompts).<br /><br />Se admite la personalización completa mediante [Advanced Customizations for Universal Login.](/docs/es/customize/login-pages/advanced-customizations) | El mayor grado de personalización, ya que controlas por completo la UI/UX de tu aplicación.                                                                                                                                                                                                                                                                                                             |
| **Gestión de funcionalidades**                                  | Las funcionalidades se pueden gestionar de forma centralizada desde el Auth0 Dashboard o mediante la Auth0 Management API.<br /><br />Por ejemplo, cuando habilitas o deshabilitas MFA en tu Dashboard, el cambio se reflejará inmediatamente en los siguientes inicios de sesión de los usuarios.                                            | Las funcionalidades deben gestionarse individualmente para cada aplicación.<br /><br />Por ejemplo, si quisieras implementar MFA para tu aplicación web y tu aplicación nativa, tendrías que actualizar y publicar nuevas versiones de ambas.                                                                                                                                                           |
| **Experiencia de usuario**                                      | Durante la autenticación, los usuarios son redirigidos entre tu aplicación y el servidor de autorización de Auth0.<br /><br />En el caso de las aplicaciones nativas, esto requiere implementar enlaces universales o enlaces profundos.                                                                                                      | Los usuarios permanecen en tu aplicación durante la autenticación.                                                                                                                                                                                                                                                                                                                                      |
| **Seguridad y mantenimiento**                                   | Auth0 supervisa las tendencias de seguridad y actualiza Universal Login en consecuencia.                                                                                                                                                                                                                                                      | Eres responsable de seguir las prácticas recomendadas de seguridad e implementar adecuadamente las tecnologías proporcionadas.                                                                                                                                                                                                                                                                          |

<div id="best-practice">
  ## Práctica recomendada
</div>

Según [RFC 8252: OAuth 2.0 for Native Apps on IETF](https://www.rfc-editor.org/rfc/rfc8252.txt), las aplicaciones nativas solo deben usar agentes de usuario externos (como el navegador) para los flujos de autenticación.
