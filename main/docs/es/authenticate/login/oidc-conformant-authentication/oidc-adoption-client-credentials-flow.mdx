---
description: Aprende cómo el flujo compatible con OIDC afecta al flujo de credenciales de cliente.
title: Flujo de credenciales de cliente con OIDC
---

El flujo compatible con OIDC permite usar el [flujo de credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow), que permite a las aplicaciones autenticarse en su propio nombre (en lugar de hacerlo en nombre de un usuario) para obtener acceso a una API de forma programática y segura.

Este intercambio no existe en el pipeline heredado; en su lugar, se utiliza el [flujo de contraseña del propietario del recurso](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow) para simularlo mediante la creación de un **usuario de servicio**. Desaconsejamos firmemente este enfoque y recomendamos usar el flujo de credenciales de cliente, que permite definir permisos granulares para cada aplicación.

Para aprender a ejecutar un flujo de credenciales de cliente, consulta [Llamar a una API mediante el flujo de credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).
