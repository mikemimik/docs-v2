---
description: Aprende cómo el flujo compatible con OIDC afecta a los tokens que se utilizan para proteger las API, incluidos los scopes y los claims.
title: Tokens de acceso con OIDC
---

Dado que las aplicaciones y las API (recursos) se definen como entidades independientes de Auth0 en el flujo compatible con OIDC, puedes obtener <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> para tus API. En consecuencia, todas las API deben protegerse con tokens de acceso en lugar de <Tooltip tip="Token de ID: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de ID</Tooltip>. Para obtener más información, consulta [Tokens de acceso](/docs/es/secure/tokens/access-tokens) e [ID Tokens](/docs/es/secure/tokens/id-tokens).

<div id="claims">
  ## Claims
</div>

El flujo compatible con OIDC estandariza los claims que puedes añadir a los tokens de ID y de acceso.

<div id="standard-claims">
  ### Claims estándar
</div>

La especificación de OIDC define un [](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) conjunto estándar de claims sobre los usuarios, como el perfil y el correo electrónico, que pueden devolverse en tokens de ID o en la respuesta del endpoint `/userinfo`.

Para obtener más información sobre la especificación de claims estándar de OIDC, lee la [Sección 5.1 Claims estándar](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) de la [Especificación de OpenID](https://openid.net).

<div id="custom-claims">
  ### Claims personalizados
</div>

Con el flujo compatible con OIDC, aún se pueden agregar claims personalizados al token de ID o a los tokens de acceso, pero deben ajustarse a un formato con espacio de nombres para evitar posibles colisiones con los claims estándar de OIDC.

Para saber cómo agregar un claim personalizado en el flujo compatible con OIDC, consulta [Crear claims personalizados con espacio de nombres](/docs/es/secure/tokens/json-web-tokens/create-custom-claims).

<div id="scopes">
  ## Scopes
</div>

En el flujo compatible con OIDC, puedes configurar tus aplicaciones en Auth0 para usar scopes y solicitar que:

* Los claims estándar de OIDC, como `profile` y `email`, se incluyan en el token de identidad (si el usuario da su consentimiento para proporcionar esta información a la aplicación).
* Los permisos admitidos por la API a la que quieren acceder se incluyan en el access token. Por ejemplo, puedes definir el <Tooltip tip="Audience: Identificador único del audience de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un Access Token." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip> y los scopes requeridos de tu custom API, lo que te permitirá segmentar el acceso a distintas operaciones dentro de tu API.

Para obtener más información, consulta [Scopes de OpenID Connect](/docs/es/get-started/apis/scopes/openid-connect-scopes).
