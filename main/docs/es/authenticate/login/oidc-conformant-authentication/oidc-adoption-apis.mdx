---
description: Aprende cómo el flujo compatible con OIDC afecta tu uso de APIs externas y los tokens que se utilizan para protegerlas.
title: APIs externas con OIDC
---

Con el flujo compatible con OIDC, debes definir las aplicaciones y las APIs (recursos) como entidades separadas en Auth0. Entre las ventajas se incluyen:

* Integración con APIs más sencilla: las APIs ya no están vinculadas a las aplicaciones que las llaman.
* Casos de uso de integración [máquina a máquina (M2M)](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow): las aplicaciones pueden autenticarse como ellas mismas (en lugar de actuar en nombre de un usuario) para obtener de forma programática y segura un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip>.

<div id="architecture">
  ## Arquitectura
</div>

Entre las soluciones sugeridas compatibles con OIDC para distintos escenarios se incluyen:

* **Varias aplicaciones que consumen una API con un solo <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>**: Representa cada aplicación con una sola aplicación de Auth0, y cada una podrá interactuar con la API de la que depende.
* **Intercambiar tokens obtenidos por una aplicación por tokens para otra aplicación mediante delegación**: Usa una solución con varias aplicaciones, en la que cada aplicación se autentique contra el mismo <Tooltip tip="Servidor de recursos: servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/docs/es/glossary?term=resource+server">servidor de recursos</Tooltip>.
* **Las aplicaciones no dependen de API externas; solo se usan para autenticar usuarios**: No necesitas definir una API, siempre que los <Tooltip tip="Token de ID: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de ID</Tooltip>:

  * Los procese únicamente la aplicación.
  * No se envíen a ningún servicio externo.
