---
description: Descubre cómo el flujo compatible con OIDC afecta al uso de API externas y de los tokens utilizados para protegerlas.
title: API externas con OIDC
---

Con el flujo compatible con OIDC, debes definir las aplicaciones y las API (recursos) como entidades separadas en Auth0. Entre las ventajas se incluyen:

* Integración de API más sencilla: las API ya no están vinculadas a las aplicaciones que las llaman.
* Escenarios de integración de [máquina a máquina (M2M)](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow): las aplicaciones pueden autenticarse como ellas mismas (en lugar de actuar en nombre de un usuario) para obtener de forma programática y segura un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip>.

<div id="architecture">
  ## Arquitectura
</div>

Las soluciones sugeridas compatibles con OIDC para distintos escenarios incluyen:

* **Varias aplicaciones que llaman a una API con un único <Tooltip tip="ID de cliente: Valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>**: Representa cada aplicación con una sola aplicación de Auth0, y cada una podrá interactuar con la API de la que depende.
* **Intercambiar tokens obtenidos por una aplicación por tokens para otra aplicación con Delegation**: Usa una solución con varias aplicaciones, en la que cada aplicación se autentique en el mismo <Tooltip tip="Resource Server: Servidor que aloja recursos protegidos. Los servidores de recursos aceptan solicitudes de recursos protegidos y responden a ellas." cta="Ver glosario" href="/docs/es/glossary?term=resource+server">servidor de recursos</Tooltip>.
* **Las aplicaciones no dependen de API externas; solo se usan para autenticar a los usuarios**: No necesitas definir una API siempre que los <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de ID</Tooltip> se:

  * Procesen solo en la aplicación.
  * No se envíen a ningún servicio externo.
