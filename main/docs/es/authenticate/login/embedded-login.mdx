title: Inicio de sesión integrado
---

El inicio de sesión integrado permite a sus usuarios iniciar sesión directamente en su aplicación y transmitir sus credenciales al servidor de Auth0 para autenticarse. No recomendamos usar el inicio de sesión integrado. Para obtener más información, consulte [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

Si decide usar el inicio de sesión integrado, debe [configurar su aplicación para Cross-Origin Resource Sharing](/docs/es/get-started/applications/set-up-cors) y debería [configurar un dominio personalizado](/docs/es/customize/custom-domains). Luego, puede implementar el [SDK de Lock](/docs/es/libraries/lock) o el [SDK de Auth0.js](/docs/es/libraries/auth0js) en su aplicación, o llamar directamente a la [Auth0 Authentication API](https://auth0.com/docs/api/authentication).

Auth0 admite el inicio de sesión integrado con conexiones de <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Sin contraseña</Tooltip> para varios tipos de aplicaciones:

* [Inicio de sesión integrado sin contraseña en aplicaciones nativas](/docs/es/authenticate/passwordless/implement-login/embedded-login/native)
* [Inicio de sesión integrado sin contraseña en aplicaciones web tradicionales](/docs/es/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Autenticación integrada sin contraseña para aplicaciones de una sola página](/docs/es/authenticate/passwordless/implement-login/embedded-login/spa)
