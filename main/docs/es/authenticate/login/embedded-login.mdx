title: Embedded Login
---

Embedded Login permite a tus usuarios iniciar sesión directamente en tu aplicación y transmitir sus credenciales al servidor de Auth0 para autenticarse. No recomendamos usar Embedded Login. Para obtener más información, consulta [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

Si decides usar Embedded Login, debes [configurar tu aplicación para Cross-Origin Resource Sharing](/docs/es/get-started/applications/set-up-cors) y te recomendamos [configurar un dominio personalizado](/docs/es/customize/custom-domains). Después, puedes implementar el [SDK de Lock](/docs/es/libraries/lock) o el [SDK de auth0.js](/docs/es/libraries/auth0js) en tu aplicación, o llamar directamente a la [Auth0 Authentication API](https://auth0.com/docs/api/authentication).

Auth0 admite Embedded Login con conexiones <Tooltip tip="Passwordless: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> para varios tipos de aplicación:

* [Inicio de sesión Passwordless integrado en Native Applications](/docs/es/authenticate/passwordless/implement-login/embedded-login/native)
* [Inicio de sesión Passwordless integrado en aplicaciones web clásicas](/docs/es/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Autenticación Passwordless integrada para aplicaciones de página única](/docs/es/authenticate/passwordless/implement-login/embedded-login/spa)
