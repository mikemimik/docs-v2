title: Embedded Login
---

Embedded Login permite a sus usuarios iniciar sesión directamente en su aplicación y transmitir sus credenciales al servidor de Auth0 para la autenticación. No recomendamos usar Embedded Login. Para obtener más información, consulte [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

Si decide usar Embedded Login, debe [configurar su aplicación para Cross-Origin Resource Sharing](/docs/es/get-started/applications/set-up-cors) y debería [configurar un dominio personalizado](/docs/es/customize/custom-domains). Luego puede implementar el [SDK de Lock](/docs/es/libraries/lock) o el [SDK de Auth0.js](/docs/es/libraries/auth0js) en su aplicación, o llamar directamente a la [Auth0 Authentication API](https://auth0.com/docs/api/authentication).

Auth0 admite Embedded Login con conexiones <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> para varios tipos de aplicación:

* [Embedded Passwordless Login en aplicaciones nativas](/docs/es/authenticate/passwordless/implement-login/embedded-login/native)
* [Embedded Passwordless Login en aplicaciones web clásicas](/docs/es/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Autenticación Passwordless embebida para aplicaciones de página única](/docs/es/authenticate/passwordless/implement-login/embedded-login/spa)
