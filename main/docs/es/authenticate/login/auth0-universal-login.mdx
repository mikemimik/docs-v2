Auth0 <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> proporciona la funcionalidad esencial de un <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>: el flujo de inicio de sesión. Cuando un usuario necesita demostrar su identidad para acceder a tu aplicación, puedes redirigirlo a Universal Login y dejar que Auth0 gestione el proceso de autenticación.

Con Universal Login, no tienes que realizar ningún trabajo de integración para admitir distintos métodos de autenticación. Puedes empezar con un flujo de autenticación sencillo y luego añadir funcionalidades adicionales, como inicio de sesión passwordless, inicio de sesión social o <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA), para crear una experiencia de inicio de sesión segura y fácil de usar.

La configuración de Universal Login es dinámica y no requiere cambios a nivel de aplicación, ya que las páginas web alojadas en el <Tooltip tip="Servidor de autenticación: Servidor que confirma o niega la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authentication+server">servidor de autenticación</Tooltip> centralizado de Auth0 controlan por completo el flujo de inicio de sesión. Además, no necesitas actualizar manualmente tu código para que las aplicaciones se beneficien de las mejoras que Auth0 realiza en Universal Login.

Desde el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, puedes personalizar la apariencia y el comportamiento de las páginas de inicio de sesión para crear una experiencia coherente y alineada con tu marca. Para casos de uso avanzados, también puedes modificar el código de cada página individualmente.

Para obtener más información sobre las diferencias entre Universal Login y Embedded Login, consulta [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

<div id="configure-universal-login">
  ## Configurar Universal Login
</div>

En tu Auth0 Dashboard, puedes elegir qué experiencia usar para las páginas de inicio de sesión predeterminadas en [Branding &gt; Universal Login &gt; Advanced Options](https://manage.auth0.com/#/universal-login/customizations-new).

Auth0 ofrece dos experiencias de inicio de sesión alojadas:

* **Universal Login** ofrece una experiencia más ágil para los usuarios y no requiere usar JavaScript para su personalización.
* **Classic Login** usa controles de JavaScript para cada página del flujo de inicio de sesión.

En este momento, los esfuerzos de desarrollo activo de Auth0 están centrados en Universal Login, y Classic Login ya no recibe actualizaciones. A menos que tu caso de uso específico requiera la experiencia Classic, se recomienda implementar Universal Login.

Para obtener más información sobre las diferencias entre estas dos experiencias, consulta [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<div id="accessibility">
  ### Accesibilidad
</div>

Auth0 emplea los siguientes estándares para mejorar el acceso a los flujos de Universal Login para las personas que usan tecnologías de asistencia:

* [Pautas de Accesibilidad para el Contenido Web (WCAG) versión 2.2 AA](https://www.w3.org/WAI/standards-guidelines/wcag/)
* [EN 301 549](https://www.etsi.org/human-factors-accessibility/en-301-549-v3-the-harmonized-european-standard-for-ict-accessibility)

Auth0 Universal Login incorpora estos estándares de forma predeterminada. A continuación se indican las áreas que podrían afectar a las personalizaciones de los clientes:

* Los errores de validación se muestran en línea en lugar de en tooltips, para que sean accesibles con lectores de pantalla.
* Los colores predeterminados de los componentes de la interfaz están alineados con las pautas WCAG sobre contraste de color.
* Los formularios usan autocorrección.
* Los atributos [ARIA](https://www.w3.org/WAI/standards-guidelines/aria/) y otros atributos de los componentes de la interfaz permiten navegar fácilmente con lectores de pantalla.
* Las direcciones de correo electrónico se validan del lado del cliente y siguen las mismas reglas de formato que la validación del lado del servidor.
* Las etiquetas HTML son accesibles y están asociadas a sus respectivos campos de entrada.
* Los requisitos de complejidad de la contraseña y los mensajes de error en línea son anunciados por los lectores de pantalla.
* Todas las pantallas del <Tooltip tip="Flujo de autorización: concesión de autorización (o flujo de trabajo) especificada en el framework OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip> tienen un título de página claro y distintivo.
* Los campos obligatorios incluyen un indicador visual en la etiqueta.

<div id="customize-login-page">
  ### Personaliza la página de inicio de sesión
</div>

Después de habilitar Universal Login, puedes personalizar tu página de inicio de sesión directamente desde el Auth0 Dashboard. En concreto, puedes personalizar las plantillas de página y los temas, el prompt de registro e inicio de sesión y otros elementos de texto. Para obtener más información, consulta Personaliza Universal Login.

Si decides usar Classic Login, puedes personalizar las opciones básicas de identidad visual desde el Auth0 Dashboard. Después, puedes configurar personalizaciones avanzadas con el SDK que se usa para crear tu flujo de inicio de sesión. Para obtener más información, consulta [Customize Classic Login](/docs/es/customize/login-pages/classic-login).

<div id="implement-universal-login">
  ## Implement Universal Login
</div>

Después de habilitar Universal Login o Classic Login en tu tenant, sigue estos pasos para implementar esta experiencia:

1. [Registra tu aplicación](/docs/es/get-started/auth0-overview/create-applications) en Auth0.
2. Configura tu aplicación para llamar al [endpoint de Login](https://auth0.com/docs/api/authentication#login) de la Authentication API, activar el flujo de inicio de sesión y gestionar la respuesta. Puedes configurar esto directamente o usar uno de los [SDKs](/docs/es/libraries) de Auth0.
3. Crea una conexión de [Database](/docs/es/get-started/applications/set-up-database-connections), [empresarial](/docs/es/authenticate/enterprise-connections) o [Social](/docs/es/authenticate/identity-providers/social-identity-providers) y habilítala para tu aplicación.

<div id="navigate-to-the-login-page">
  ### Ve a la página de inicio de sesión
</div>

Puedes llamar directamente al [endpoint de Login](https://auth0.com/docs/api/authentication#login) desde cualquier navegador:

export const codeExample = `https://{yourDomain}/authorize?
  response_type=code|token&
  client_id={yourClientId}&
  connection={connectionName}&
  redirect_uri=https://{yourApp}/callback&
  state={yourState}`;

<AuthCodeBlock children={codeExample} language="bash" filename="cURL" />

La solicitud debe incluir los siguientes valores:

* `response_type` (ya sea `code` o `token`)
* `client_id`
* `redirect_uri`
* `state`

  * Para entender mejor por qué este valor es obligatorio, consulta [Prevenir ataques y redirigir a los usuarios con los parámetros de estado de OAuth 2.0](/docs/es/secure/attack-protection/state-parameters).

Opcionalmente, puedes incluir el parámetro `connection` para solicitar que los usuarios se autentiquen con la conexión especificada.

<div id="correlation-id">
  ### ID de correlación
</div>

<ReleaseStageNotice feature="ID de correlación" stage="ea" terms="true" contact="equipo de soporte de Auth0 o su gerente técnico de cuenta" />

Universal Login admite eventos correlacionados en todos los flujos de autenticación. Una vez que implemente Universal Login, su aplicación genera un ID único que se pasa como `correlation_id` al endpoint `/authorize`. El ID único se mantiene a lo largo de los siguientes eventos de autenticación:

* Registro
* Inicio de sesión
* Inscripción y desafíos de autenticación multifactor (MFA)
* Restablecimiento de contraseña

Una vez que se pasa `correlation_id`, puede rastrear los eventos de autenticación en [Registros de Auth0](/docs/es/deploy-monitor/logs).

Para obtener más información, consulte [ID de correlación](/docs/es/authenticate/login/configure-correlation-id).
