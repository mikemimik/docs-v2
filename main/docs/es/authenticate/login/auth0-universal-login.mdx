Auth0 <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> proporciona la funcionalidad esencial de un <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>: el flujo de inicio de sesión. Cuando un usuario necesita demostrar su identidad para acceder a tu aplicación, puedes redirigirlo a Universal Login y dejar que Auth0 gestione el proceso de autenticación.

Con Universal Login, no tienes que realizar ninguna integración para admitir distintos métodos de autenticación. Puedes empezar con un flujo de autenticación sencillo y luego añadir funcionalidades adicionales, como inicio de sesión passwordless, inicio de sesión con redes sociales o <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA), para personalizar una experiencia de inicio de sesión segura y fácil de usar.

La configuración de Universal Login es dinámica y no requiere ningún cambio a nivel de aplicación, ya que las páginas web alojadas en el <Tooltip tip="Servidor de autenticación: Servidor que confirma o deniega la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authentication+server">servidor de autenticación</Tooltip> centralizado de Auth0 controlan por completo el flujo de inicio de sesión. Además, no necesitas actualizar manualmente tu código para que las aplicaciones se beneficien de las mejoras que Auth0 incorpora a Universal Login.

Desde el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, puedes personalizar la apariencia y el comportamiento de las páginas de inicio de sesión para crear una experiencia coherente con tu marca. Para casos de uso avanzados, también puedes cambiar el código de cada página de forma individual.

Para obtener más información sobre las diferencias entre Universal Login y embedded login, consulta [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

<div id="configure-universal-login">
  ## Configurar Universal Login
</div>

En tu Auth0 Dashboard, puedes elegir qué experiencia usar para las páginas de inicio de sesión predeterminadas en [Branding &gt; Universal Login &gt; Advanced Options](https://manage.auth0.com/#/universal-login/customizations-new).

Auth0 ofrece dos experiencias de inicio de sesión alojado:

* **Universal Login** ofrece una experiencia más ágil para los usuarios y no requiere usar JavaScript para personalizarla.
* **Classic Login** utiliza controles de JavaScript para cada página del flujo de inicio de sesión.

En este momento, los esfuerzos de desarrollo activo de Auth0 están centrados en Universal Login, y Classic Login ya no recibe actualizaciones. A menos que tu caso de uso específico requiera la experiencia Classic, se recomienda implementar Universal Login.

Para obtener más información sobre las diferencias entre estas dos experiencias, consulta [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<div id="accessibility">
  ### Accesibilidad
</div>

Auth0 emplea los siguientes estándares para mejorar el acceso a los flujos de Universal Login para las personas que utilizan tecnologías de asistencia:

* [Pautas de Accesibilidad para el Contenido Web (WCAG), versión 2.2 AA](https://www.w3.org/WAI/standards-guidelines/wcag/)
* [EN 301 549](https://www.etsi.org/human-factors-accessibility/en-301-549-v3-the-harmonized-european-standard-for-ict-accessibility)

Auth0 Universal Login aplica estos estándares de forma predeterminada. A continuación, se indican algunas áreas que podrían afectar a las personalizaciones del cliente:

* Los errores de validación se muestran en línea en lugar de en descripciones emergentes, para que sean accesibles a los lectores de pantalla.
* Los colores predeterminados de los componentes de la interfaz se ajustan a las pautas de WCAG sobre contraste de color.
* Los formularios usan autocorrección.
* Los atributos [ARIA](https://www.w3.org/WAI/standards-guidelines/aria/) y otros atributos de los componentes de la interfaz permiten una navegación sencilla con lectores de pantalla.
* Las direcciones de correo electrónico se validan del lado del cliente y siguen las mismas reglas de formato que la validación del lado del servidor.
* Las etiquetas HTML son accesibles y están asociadas a sus respectivos campos de entrada.
* Los requisitos de complejidad de la contraseña y los mensajes de error en línea son anunciados por los lectores de pantalla.
* Todas las pantallas del <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) especificada en el marco de OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip> tienen un título de página claro y distintivo.
* Los campos obligatorios incluyen un indicador visual en la etiqueta.

<div id="customize-login-page">
  ### Personalizar la página de inicio de sesión
</div>

Una vez que habilites Universal Login, puedes personalizar tu página de inicio de sesión directamente desde el Auth0 Dashboard. En concreto, puedes personalizar las plantillas de página y los temas, los prompts de registro e inicio de sesión y otros elementos de texto. Para obtener más información, consulta Personalizar Universal Login.

Si decides implementar Classic Login, puedes personalizar las opciones básicas de identidad visual desde el Auth0 Dashboard. Después, puedes configurar personalizaciones avanzadas con el SDK que uses para crear tu flujo de inicio de sesión. Para obtener más información, consulta [Personalizar Classic Login](/docs/es/customize/login-pages/classic-login).

<div id="implement-universal-login">
  ## Implementa Universal Login
</div>

Después de habilitar Universal Login o Classic Login en tu tenant, sigue estos pasos para implementar la experiencia:

1. [Registra tu aplicación](/docs/es/get-started/auth0-overview/create-applications) en Auth0.
2. Configura tu aplicación para llamar al [endpoint de login](https://auth0.com/docs/api/authentication#login) de la Authentication API, activar el flujo de inicio de sesión y gestionar la respuesta. Puedes configurarlo directamente o usar uno de los [SDKs](/docs/es/libraries) de Auth0.
3. Crea una conexión de [Database](/docs/es/get-started/applications/set-up-database-connections), [Enterprise](/docs/es/authenticate/enterprise-connections) o [Social](/docs/es/authenticate/identity-providers/social-identity-providers) y habilítala para tu aplicación.

<div id="navigate-to-the-login-page">
  ### Ve a la página de inicio de sesión
</div>

Puedes invocar directamente el [endpoint de Login](https://auth0.com/docs/api/authentication#login) desde cualquier navegador:

export const codeExample = `https://{yourDomain}/authorize?
  response_type=code|token&
  client_id={yourClientId}&
  connection={connectionName}&
  redirect_uri=https://{yourApp}/callback&
  state={yourState}`;

<AuthCodeBlock children={codeExample} language="bash" filename="cURL" />

La solicitud debe incluir los siguientes valores:

* `response_type` (ya sea `code` o `token`)
* `client_id`
* `redirect_uri`
* `state`

  * Para entender mejor por qué se requiere este valor, consulta [Prevenir ataques y redirigir a los usuarios con los parámetros state de OAuth 2.0](/docs/es/secure/attack-protection/state-parameters).

Opcionalmente, puedes incluir el parámetro `connection` para pedir a los usuarios que se autentiquen con la conexión especificada.

<div id="correlation-id">
  ### ID de correlación
</div>

<ReleaseStageNotice feature="ID de correlación" stage="ea" terms="true" contact="equipo de soporte de Auth0 o su gerente técnico de cuenta" />

Universal Login admite eventos correlacionados en todos los flujos de autenticación. Una vez que implemente Universal Login, su aplicación genera un ID único que pasa como `correlation_id` al endpoint `/authorize`. Ese ID único se mantiene a lo largo de los siguientes eventos de autenticación:

* Registro
* Inicio de sesión
* Registro y desafíos de Autenticación multifactor (MFA)
* Restablecimiento de contraseña

Una vez que se pasa `correlation_id`, puede rastrear los eventos de autenticación en [registros de Auth0](/docs/es/deploy-monitor/logs).

Para obtener más información, consulte [ID de correlación](/docs/es/authenticate/login/configure-correlation-id).
