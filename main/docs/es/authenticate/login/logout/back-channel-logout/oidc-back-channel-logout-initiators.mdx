Los iniciadores del cierre de sesión OIDC por canal secundario te permiten cerrar de forma remota la sesión de los usuarios en sus aplicaciones en función de eventos de finalización de sesión. Los iniciadores del cierre de sesión OIDC por canal secundario funcionan en todos los protocolos; por ejemplo, una solicitud de cierre de sesión iniciada por un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> (iniciada por el IdP) de <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>; y no se ven afectados por las restricciones de cookies de terceros.

Esta funcionalidad es una extensión de la especificación estándar de back-channel de OIDC. Puedes configurarla para iniciar una solicitud de cierre de sesión OIDC por canal secundario para eventos de finalización de sesión específicos, como un cambio de contraseña o el vencimiento de la sesión, o para todos los eventos de finalización de sesión.

Los administradores pueden habilitar esta funcionalidad para aplicaciones específicas con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0.

<div id="how-oidc-back-channel-logout-initiators-work">
  ## Cómo funcionan los iniciadores de cierre de sesión OIDC por canal secundario
</div>

Los iniciadores asocian una respuesta de cierre de sesión OIDC por canal secundario con un evento de finalización de sesión. Capturan el evento y lo usan para activar un token de cierre de sesión OIDC en todas las aplicaciones asociadas con esa sesión.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los iniciadores no controlan la gestión de sesiones en tu tenant, incluidos los eventos de finalización de sesión.
</Callout>

<div id="configure-oidc-back-channel-logout-initiators">
  ## Configurar los iniciadores del cierre de sesión OIDC por canal secundario
</div>

Puede configurar los iniciadores del cierre de sesión OIDC por canal secundario con Auth0 Management API.

<div id="management-api">
  ### Management API
</div>

Puede configurar los iniciadores del cierre de sesión OIDC por canal secundario de una aplicación mediante la Management API usando el endpoint [Actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id).

1. Obtenga un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:clients`.
2. Llame al endpoint [Actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) con los datos de configuración adecuados en el payload. Por ejemplo, para cerrar la sesión de una aplicación después de un evento de cambio de contraseña, proporcione lo siguiente:

   {/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample1 = `   PATCH /api/v2/clients/{yourClientId}
{
  ...
  "oidc_logout": {
    "backchannel_logout_urls": ["https://example.com/cb"]
    "backchannel_logout_initiators": {
      "mode":"custom",
      "selected_initiators": ["rp-logout", "idp-logout", "password-changed"]
    }
  }
  ...
}
`;

<div id="properties">
  #### Propiedades
</div>

El objeto `backchannel_logout_initiators` admite las siguientes propiedades:

| Propiedad             | Tipo    | ¿Obligatorio?                     | Descripción                                         | Valores admitidos                                                                                                                  |
| --------------------- | ------- | --------------------------------- | --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `mode`                | string  | Obligatorio                       | Método de configuración para habilitar iniciadores. | `custom`, `all`                                                                                                                    |
| `selected_initiators` | arreglo | Obligatorio si `mode` es `custom` | Lista de iniciadores que se habilitarán.            | `rp-logout`, `idp-logout`, `password-changed`, `session-expired`, `session-revoked`, `account-deleted`, `email-identifier-changed` |

<div id="mode-property">
  ##### propiedad `mode`
</div>

La propiedad `mode` determina el método de configuración para habilitar los iniciadores.

De forma predeterminada, se establece en `custom`, lo que le permite especificar qué iniciadores quiere habilitar. Si quiere que su aplicación cierre la sesión cada vez que finalice la sesión del IdP, establézcala en `all`.

La propiedad `mode` admite los siguientes valores:

| Valor    | Descripción                                                                   |
| -------- | ----------------------------------------------------------------------------- |
| `custom` | Habilita solo los iniciadores enumerados en el arreglo `selected_initiators`. |
| `all`    | Habilita automáticamente todos los iniciadores actuales y futuros.            |

<div id="selected_initiators-property">
  ##### Propiedad `selected_initiators`
</div>

La propiedad `selected_initiators` contiene la lista de initiators que se habilitarán para la aplicación especificada.

La propiedad `selected_initiators` admite los siguientes valores:

| Valor                      | Descripción                                                                      |
| -------------------------- | -------------------------------------------------------------------------------- |
| `rp-logout`                | La request fue iniciada por una parte confiable (RP).                            |
| `idp-logout`               | La request fue iniciada por un proveedor de identidad externo (IdP).             |
| `password-changed`         | La request fue iniciada por un cambio de contraseña.                             |
| `session-expired`          | La request fue iniciada por el vencimiento de la sesión.                         |
| `session-revoked`          | La request fue iniciada por la eliminación de la sesión.                         |
| `account-deleted`          | La request fue iniciada por la eliminación de una cuenta.                        |
| `email-identifier-changed` | La request fue iniciada por un cambio en el identificador de correo electrónico. |

<div id="selected-initiators-only">
  #### Solo los iniciadores seleccionados
</div>

De forma predeterminada, solo se suscribirá a los iniciadores obligatorios (`rp-logout` y `idp-logout`). Cualquier iniciador adicional, incluidos los que se agreguen en el futuro, primero debe seleccionarse para que pueda iniciar un cierre de sesión en su aplicación.

Seleccione esta opción si desea que su aplicación cierre sesión solo para los iniciadores que seleccione.

<div id="all-supported-initiators">
  #### Todos los iniciadores compatibles
</div>

Todos los iniciadores compatibles, incluidos los que se añadan en el futuro, quedarán suscritos de forma predeterminada.

Selecciona esta opción si quieres que se cierre la sesión de tu aplicación cada vez que finalice la sesión del IdP.
