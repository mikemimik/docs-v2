description: Describe cómo cerrar la sesión de los usuarios en un proveedor de identidad SAML externo.
title: cerrar la sesión de los usuarios en proveedores de identidad SAML
---

El cierre de sesión único (SLO) es una funcionalidad que permite a un usuario cerrar varias sesiones de autenticación mediante una sola acción de cierre de sesión.

Auth0 admite SLO cuando [conecta su aplicación a un proveedor de identidad SAML (IdP)](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml) y ofrece compatibilidad limitada con SLO cuando [configura Auth0 como un IdP de SAML](/docs/es/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

<div id="configure-slo-when-application-is-connected-to-a-saml-idp">
  ## Configurar SLO cuando la aplicación está conectada a un IdP de SAML
</div>

Si quieres que Auth0 [cierre la sesión del usuario en su proveedor de identidad](/docs/es/authenticate/login/logout/log-users-out-of-idps), incluye el parámetro `federated` cuando llames al [endpoint de Logout de Auth0 Authentication API](https://auth0.com/docs/api/authentication#logout).

Es posible que debas configurar ajustes adicionales para la conexión de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> para asegurarte de que Auth0 envíe la solicitud de cierre de sesión al endpoint de cierre de sesión del <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip> de SAML:

1. Ve a [Auth0 Dashboard &gt; Autenticación &gt; Empresarial &gt; SAML](https://manage.auth0.com/#/connections/enterprise/samlp) y selecciona tu conexión.
2. Activa **Enable Sign Out**.
3. Introduce la URL de Logout de SAML proporcionada por el IdP de SAML en el campo **Sign Out URL**. Si dejas este campo en blanco, Auth0 usará de forma predeterminada la URL del campo **Sign In URL**.
4. Confirma que el valor del campo **enlace de protocolo** coincida con el enlace de protocolo que espera el IdP de SAML.

<div id="configure-slo-when-auth0-is-the-saml-idp">
  ## Configurar SLO cuando Auth0 es el IdP de SAML
</div>

Cuando Auth0 es el IdP de SAML, hay dos escenarios de cierre de sesión que debe tener en cuenta:

* Iniciado por la aplicación
* Iniciado por el IdP

En ambos escenarios, debe [configurar el addon SAML2 Web App](/docs/es/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) para indicar adónde enviar las respuestas de cierre de sesión:

1. Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione su aplicación.
2. Vaya a la pestaña **Addons** y seleccione **SAML2 Web App**.
3. Vaya a la pestaña **Settings**.
4. Busque el cuadro **Settings** y agregue (o actualice) el objeto `logout` con las siguientes propiedades:

   * `callback`: Introduzca la URL de SLO de la aplicación.
   * `slo_enabled`: Introduzca `true` si desea habilitar el SLO iniciado por el IdP, en cuyo caso Auth0 envía una solicitud SAML de cierre de sesión a la aplicación cuando el usuario cierra sesión en Auth0 SAML IdP. De lo contrario, introduzca `false`.

     ```bash cURL lines
     {
       "logout": {
         "callback": "https://myapp.exampleco.com/saml/logout",
         "slo_enabled": true
       }
     }
     ```

De forma predeterminada, Auth0 envía respuestas SAML de cierre de sesión con el enlace de protocolo `HTTP-POST`. Si su aplicación espera el enlace de protocolo `HTTP-Redirect`, agregue (o actualice) la propiedad `binding` de la siguiente manera:

```json JSON lines
{
  "binding": "urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect"
}
```

<div id="application-initiated-logout">
  ### Cierre de sesión iniciado por la aplicación
</div>

Cuando el usuario inicia el cierre de sesión desde una aplicación, esta debe enviar una solicitud al endpoint de [SAML Logout](https://auth0.com/docs/api/authentication#saml-logout) de la Authentication API para activar el SLO.

Si tu aplicación no admite SLO, pero sí una URL de redirección para enviar al usuario después del cierre de sesión, establece la URL de redirección como el endpoint de [OIDC Logout](https://auth0.com/docs/api/authentication#oidc-logout) de la Authentication API (o el endpoint de [Auth0 Logout](https://auth0.com/docs/api/authentication#auth0-logout) de la Authentication API si usas el logout heredado). Auth0 no notificará a otros participantes de la sesión que se inició un cierre de sesión, pero eliminaremos la [sesión de Auth0](/docs/es/manage-users/sessions/session-layers) del usuario.

<div id="idp-initiated-logout">
  ### Cierre de sesión iniciado por el IdP
</div>

Cuando un usuario inicia el cierre de sesión desde un Auth0 SAML IdP, Auth0 envía una respuesta de cierre de sesión de SAML a la URL de la propiedad `logout.callback` en Settings del addon SAML2 Web App. Solo se puede especificar una única URL para esta propiedad.

Debido a esta restricción, Auth0 no admite de forma nativa el caso en que un usuario deba cerrar sesión en varias aplicaciones conectadas a su Auth0 SAML IdP.

Puede lograr una funcionalidad similar a SLO configurando sus aplicaciones para que comprueben periódicamente el estado de la sesión de Auth0 del usuario y finalicen la sesión de la aplicación en consecuencia. Para obtener más información, consulte [Límites de duración de la sesión](/docs/es/manage-users/sessions/session-lifetime-limits).
