description: Describe cómo cerrar la sesión de los usuarios en un proveedor de identidad SAML externo.
title: Cerrar la sesión de los usuarios en proveedores de identidad SAML
---

El cierre de sesión único (SLO) es una funcionalidad que permite a un usuario finalizar varias sesiones de autenticación mediante una sola acción de cierre de sesión.

Auth0 admite SLO cuando [conectas tu aplicación a un proveedor de identidad SAML (IdP)](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml) y admite un SLO limitado cuando [configuras Auth0 como IdP de SAML](/docs/es/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

<div id="configure-slo-when-application-is-connected-to-a-saml-idp">
  ## Configura SLO cuando la aplicación esté conectada a un IdP de SAML
</div>

Si quieres que Auth0 [cierre la sesión de un usuario en su proveedor de identidad](/docs/es/authenticate/login/logout/log-users-out-of-idps), incluye el parámetro `federated` cuando llames al [endpoint de cierre de sesión de Auth0 Authentication API](https://auth0.com/docs/api/authentication#logout).

Es posible que tengas que configurar ajustes adicionales en la conexión <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> para garantizar que Auth0 envíe la solicitud de cierre de sesión al endpoint de cierre de sesión del <Tooltip tip="Identity Provider (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip> de SAML:

1. Ve a [Auth0 Dashboard &gt; Autenticación &gt; Empresarial &gt; SAML](https://manage.auth0.com/#/connections/enterprise/samlp) y selecciona tu conexión.
2. Activa **Enable Sign Out**.
3. Introduce la URL de cierre de sesión de SAML proporcionada por el IdP de SAML en el campo **Sign Out URL**. Si dejas este campo en blanco, Auth0 usará de forma predeterminada la URL del campo **Sign In URL**.
4. Confirma que el valor del campo **Protocol Binding** coincida con el enlace de protocolo que espera el IdP de SAML.

<div id="configure-slo-when-auth0-is-the-saml-idp">
  ## Configurar SLO cuando Auth0 es el IdP de SAML
</div>

Cuando Auth0 es el IdP de SAML, hay dos escenarios de cierre de sesión que debes tener en cuenta:

* Iniciado por la aplicación
* Iniciado por el IdP

En ambos escenarios, debes [configurar el add-on SAML2 Web App](/docs/es/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) para indicar dónde enviar las respuestas de cierre de sesión:

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y selecciona tu aplicación.
2. Ve a la pestaña **Addons** y selecciona **SAML2 Web App**.
3. Ve a la pestaña **Settings**.
4. Busca el cuadro **Settings** y agrega (o actualiza) el objeto `logout` con las siguientes propiedades:

   * `callback`: Ingresa la URL de SLO de la aplicación.
   * `slo_enabled`: Ingresa `true` si quieres habilitar el SLO iniciado por el IdP, en el que Auth0 envía una solicitud de cierre de sesión SAML a la aplicación cuando el usuario cierra sesión en el IdP de SAML de Auth0. De lo contrario, ingresa `false`.

     ```bash cURL lines
     {
       "logout": {
         "callback": "https://myapp.exampleco.com/saml/logout",
         "slo_enabled": true
       }
     }
     ```

De forma predeterminada, Auth0 envía respuestas de cierre de sesión SAML con el enlace de protocolo `HTTP-POST`. Si tu aplicación espera el enlace de protocolo `HTTP-Redirect`, agrega (o actualiza) la propiedad `binding` de la siguiente manera:

```json JSON lines
{
  "binding": "urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect"
}
```

<div id="application-initiated-logout">
  ### Cierre de sesión iniciado por la aplicación
</div>

Cuando el usuario inicia el cierre de sesión desde una aplicación, esta debe enviar una solicitud al endpoint de SAML Logout de la Authentication API [SAML Logout endpoint](https://auth0.com/docs/api/authentication#saml-logout) para activar el SLO.

Si su aplicación no admite SLO, pero sí una URL de redirección para enviar al usuario después del cierre de sesión, configure la URL de redirección al endpoint de OIDC Logout de la Authentication API [OIDC Logout endpoint](https://auth0.com/docs/api/authentication#oidc-logout) (o al endpoint de Auth0 Logout de la Authentication API [Auth0 Logout endpoint](https://auth0.com/docs/api/authentication#auth0-logout) si usa el cierre de sesión heredado). Auth0 no notificará a otros participantes de la sesión que se inició un cierre de sesión, pero eliminará la [sesión de Auth0](/docs/es/manage-users/sessions/session-layers) del usuario.

<div id="idp-initiated-logout">
  ### Cierre de sesión iniciado por el IdP
</div>

Cuando un usuario inicia el cierre de sesión desde un Auth0 IdP de SAML, Auth0 envía una respuesta de cierre de sesión de SAML a la URL de la propiedad `logout.callback` dentro de Settings del addon SAML2 Web App. Solo puede especificar una única URL para esta propiedad.

Debido a esta limitación, Auth0 no admite de forma nativa el caso en que un usuario deba cerrar sesión en múltiples aplicaciones conectadas a su Auth0 IdP de SAML.

Puede lograr una funcionalidad similar a SLO configurando sus aplicaciones para que comprueben periódicamente el estado de la sesión de Auth0 del usuario y finalicen la sesión de la aplicación en consecuencia. Para obtener más información, consulte [Límites de duración de la sesión](/docs/es/manage-users/sessions/session-lifetime-limits).
