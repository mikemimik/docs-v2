<Card title="Requisitos previos">
  Para usar `correlation_id`, debe tener:

  * **Auth0 Universal Login configurado**: El ID de correlación está disponible en los flujos de Universal Login.
  * **Una aplicación capaz de generar identificadores únicos**: Su aplicación debe poder generar UUID o ID de transacción. (Ejemplo: `txn_12345_xyz`, ID de sesión o ID de pedido)

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    Si `correlation_id` no está disponible en sus solicitudes de autorización, es posible que esta funcionalidad no esté habilitada en su tenant.
  </Callout>
</Card>

Auth0 ofrece la posibilidad de rastrear transacciones de autenticación con un ID de correlación. Esta capacidad de seguimiento puede reducir el tiempo de respuesta del equipo de soporte y permitirle rastrear eventos en varios sistemas posteriores a los eventos de autenticación.

El parámetro `correlation_id` agrega un identificador único generado por su aplicación a la URL de autorización. Este ID se registra automáticamente, lo que le permite rastrear y filtrar eventos en [registro del tenant de Auth0](/docs/es/deploy-monitor/logs).

El ID de correlación se mantiene en los siguientes eventos:

* Registro
* Inicio de sesión
* Inscripción y desafíos de autenticación multifactor (MFA)
* Restablecimiento de contraseña

Puede usar el ID de correlación con Universal Login, Universal Login Page Templates, plantillas de correo electrónico, Plantilla SMS, página de error personalizada, Flows y Auth0 Actions.

<div id="configure-correlation-id">
  ## Configurar el ID de correlación
</div>

Los valores de `correlation_id` tienen las siguientes limitaciones:

* **Caracteres permitidos**: caracteres alfanuméricos y caracteres especiales que coincidan con `/^[-\w.*~@+/:]{1,64}$/`.
* **Longitud máxima**: 64 caracteres
* **No debe** contener información de identificación personal (PII)

Pasa el parámetro `correlation_id` en el objeto `authorizationParams` para añadir el ID único a la URL de autorización, como en el siguiente ejemplo:

```javascript wrap lines
const { loginWithRedirect } = useAuth0();
loginWithRedirect({
  authorizationParams: {
    // Agrega "&correlation_id=..." a la URL para rastreo
    correlation_id: "YOUR_CORRELATION_ID"
  }
});
```

<div id="universal-login">
  ## Universal Login
</div>

Todos los flujos de Universal Login admiten `correlation_id`. Una vez que pases el valor del ID de correlación al endpoint `/authorize`, los eventos de autenticación generan logs del tenant con el ID de correlación, que puedes usar para hacer seguimiento. Usa el SDK de Management API para recuperar y filtrar eventos recientes a fin de aislar una transacción específica.

La siguiente llamada de ejemplo usa el SDK de Management API para recuperar los eventos de autenticación más recientes. Luego, los filtra localmente para aislar un evento específico:

```javascript wrap lines
// 1. Llamar a la Management API (sintaxis de SDK v5)
const { data: logs } = await management.logs.list({
  per_page: 100,
  sort: 'date:-1'
});
// 2. Filtrar el arreglo en el cliente para encontrar tu ID
const transactionEvents = logs.filter(log =>
  log.references?.correlation_id === "YOUR_CORRELATION_ID"
);
console.log(`Found ${transactionEvents.length} events for this transaction.`);
```

<div id="login-page-template">
  ### Plantilla de página de inicio de sesión
</div>

Si personalizas tu experiencia de inicio de sesión con [Universal Login Page Templates](/docs/es/customize/login-pages/universal-login/customize-templates), agrega `correlation_id` a tu plantilla para rastrear eventos de autenticación en inicios de sesión personalizados.

El siguiente ejemplo muestra `correlationId: "{{correlation_id}}"` agregado a `{%- auth0:widget -%}`.

```html wrap lines
<!-- Ejemplo: Inyección del ID en un script del lado del cliente -->
<!-- Nota: {{ }} es sintaxis de plantilla Liquid evaluada en el servidor por Auth0, no son marcadores de posición de documentación -->
<script>
  window.addEventListener("load", function () {
    const loginContext = {
      application: "{{application.name}}",
      // Accede al correlation_id directamente a través de la sintaxis de Liquid
      correlationId: "{{correlation_id}}"
    };

    console.log("Tracking Context:", loginContext);
    // Ahora puedes pasar loginContext.correlationId a tu herramienta de análisis
  });
</script>
```

<div id="email-templates">
  ## Plantillas de correo electrónico
</div>

Puedes incluir `correlation_id` en [plantillas de correo electrónico](/docs/es/customize/email/email-templates/customize-email-templates) para ayudar a rastrear eventos de autenticación que activan notificaciones por correo electrónico, como restablecimientos de contraseña o correos electrónicos de verificación.

El siguiente ejemplo muestra `{{correlation_id}}` agregado al cuerpo de una plantilla de correo electrónico.

```liquid wrap lines
<p>If you did not initiate this request, please ignore this email.</p>
<p>Tracking reference: {{correlation_id}}</p>
```

<div id="sms-templates">
  ## Plantillas SMS
</div>

Puedes incluir `correlation_id` en [Plantillas SMS](/docs/es/customize/phone-messages/phone-templates) para rastrear eventos de autenticación que desencadenan notificaciones por SMS, como los desafíos MFA.

El siguiente ejemplo muestra `{{correlation_id}}` añadido a una plantilla SMS.

```liquid wrap lines
Your verification code is: {{code}}. Ref: {{correlation_id}}
```

<div id="custom-error-pages">
  ## Páginas de error personalizadas
</div>

Puedes incluir `correlation_id` en [Páginas de error personalizadas](/docs/es/customize/login-pages/custom-error-pages) para mostrar la referencia de seguimiento directamente en las páginas de error, lo que facilita que los usuarios reporten incidencias y que tu equipo de soporte rastree la transacción fallida.

En el siguiente ejemplo, `{{correlation_id}}` se añade a una plantilla personalizada de página de error.

```liquid wrap lines
<h1>{{error | escape}}: {{error_description | escape}}</h1>
<p>Si necesita asistencia, indique este ID: {{correlation_id}}</p>
```

<div id="auth0-actions">
  ## Auth0 Actions
</div>

Con Auth0 Actions [triggers de registro e inicio de sesión](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers), usa el objeto `event` para registrar eventos en servicios de terceros o pasar los eventos a API de destino.

Los siguientes objetos de evento admiten `correlation_id`:

* [Registro previo de usuario](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/pre-user-registration-trigger/pre-user-registration-event-object)
* [Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object)
* [Post-challenge](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-event-object)
* [Posterior al cambio de contraseña](/docs/es/customize/actions/explore-triggers/password-reset-triggers/post-change-password-trigger/post-change-password-event-object)
* [Proveedor de correo electrónico personalizado](/docs/es/customize/email/smtp-email-providers/custom/action-triggers-custom-email-provider-event-object)
* [Proveedor telefónico personalizado](/docs/es/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/actions-triggers-custom-phone-provider-event-object)
* [Notificación MFA send-phone-message](/docs/es/customize/actions/explore-triggers/mfa-notifications-trigger/send-phone-message-event-object)
* [Intercambio de credenciales](/docs/es/customize/actions/explore-triggers/machine-to-machine-trigger/credentials-exchange-event-object)

La Action de Post Login de ejemplo muestra cómo extraer `correlation_id` del objeto `event`.

```javascript wrap lines
exports.onExecutePostLogin = async (event, api) => {
  console.log('PostLogin Action Start');
  const correlation_id = event.transaction?.correlation_id;
  // Verificar si existe correlation_id
  if (correlation_id) {
    console.log(`Correlation ID found: ${correlation_id}`);
  } else {
    console.log('No correlation_id found in transaction');
  }
  console.log('PostLogin Action End');
};
```

<div id="forms">
  ## Formularios
</div>

Puede usar el ID de correlación en Formularios y Flows. Con Formularios, puede personalizar el registro y el inicio de sesión con lógica personalizada creada en Flows. Para usar el ID de correlación en Formularios y Flows, use el objeto `context` en el editor de Flows para agregar la variable de eventos correlacionados, `{{context.transaction.correlation_id}}`, a su lógica. Para obtener más información, consulte [Variables y funciones auxiliares](/docs/es/customize/forms/variables).
