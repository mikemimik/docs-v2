---

**Okta Access Gateway (OAG)** es una solución diseñada para extender la identidad moderna a aplicaciones locales heredadas y proteger tu nube híbrida. Ahora, puedes configurar fácilmente Okta Access Gateway para usar Auth0 como <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> para autenticar a los usuarios y contar con <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO) al acceder a aplicaciones locales. Para obtener más información sobre la configuración, consulta el [sitio web de Okta Access Gateway](https://help.okta.com/oag/en-us/content/topics/access-gateway/ag-main.htm).

Una vez configurado correctamente, puedes ver tus aplicaciones de OAG como **integraciones de SSO** en el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.
