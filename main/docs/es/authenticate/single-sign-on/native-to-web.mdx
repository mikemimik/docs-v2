---

La funcionalidad de Auth0 de nativo a web <Tooltip tip="inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO) ofrece una experiencia de usuario fluida al trasladar a los usuarios autenticados desde su [aplicación nativa](/docs/es/authenticate/login/native-login) a su aplicación web.

Si su aplicación web depende de WebViews integradas o navegadores externos para ofrecer funcionalidades ampliadas o avanzadas, sus usuarios pueden moverse entre entornos nativos y web dentro de la misma sesión autenticada.

Al vincular los Session Transfer Tokens al dispositivo específico mediante la dirección IP o el ASN, se mantienen la continuidad y la seguridad de la sesión, lo que garantiza que el contexto de autenticación siga siendo seguro durante toda la transición.

<div id="how-it-works">
  ## Cómo funciona
</div>

<Frame>![Flujo de trabajo de SSO de nativo a web](/docs/images/cdy7uua7fh8z/4gFE2RP5ZCEcukiNF3Gpae/f957034f63c36d5a11f6a64d988c00e9/sequence-diagram.png)</Frame>

1. Un usuario inicia sesión en una aplicación nativa.
2. Auth0 autentica al usuario y devuelve un token de acceso, un token de actualización y un token de ID.
3. La aplicación nativa llama al endpoint [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow-with-pkce/get-token-pkce) de Auth0 para intercambiar un token de actualización por un Session Transfer Token. El Session Transfer Token se vincula a una dirección IP o ASN específicos.
4. Auth0 devuelve el Session Transfer Token para autenticarse en una aplicación web.
5. Auth0 autoriza el Session Transfer Token como parte de la cookie o como parámetro de URL, y luego devuelve un [código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow).
6. La aplicación web intercambia el código de autorización por tokens de acceso o de actualización en el endpoint [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow/get-token).
7. La aplicación web inicializa una sesión para el usuario.

Aprende a [configurar e Implement SSO de nativo a web](/docs/es/authenticate/single-sign-on/native-to-web/configure-implement-native-to-web)

<div id="limitations">
  ## Limitaciones
</div>

* Una vez habilitado SSO de nativo a web en una aplicación, el parámetro `session_transfer_token` solo funciona para SSO de nativo a web.
* Los <Tooltip tip="Token de actualización: token que se usa para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Tokens">tokens de actualización</Tooltip> generados en una transacción anterior de Session Transfer Token no generan nuevos Session Transfer Tokens.
