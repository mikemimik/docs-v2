La funcionalidad de <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO) SSO de nativo a web de Auth0 ofrece una experiencia fluida al transferir a los usuarios autenticados de tu [aplicación nativa](/docs/es/authenticate/login/native-login) a tu aplicación web.

Si tu aplicación web depende de WebViews integradas o de navegadores externos para ofrecer funcionalidades ampliadas o avanzadas, tus usuarios pueden pasar de entornos nativos a web dentro de la misma sesión autenticada.

Al vincular los tokens de transferencia de sesión al dispositivo específico mediante la dirección IP o el ASN, se mantienen la continuidad y la seguridad de la sesión, y se garantiza que el contexto de autenticación siga siendo seguro durante toda la transición.

<div id="how-it-works">
  ## Cómo funciona
</div>

<Frame>![Flujo de trabajo de SSO de nativo a web](/docs/images/cdy7uua7fh8z/4gFE2RP5ZCEcukiNF3Gpae/f957034f63c36d5a11f6a64d988c00e9/sequence-diagram.png)</Frame>

1. Un usuario inicia sesión en tu aplicación nativa.
2. Auth0 autentica al usuario y devuelve un token de acceso, un token de actualización y un token de ID.
3. Tu aplicación nativa llama al endpoint [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow-with-pkce/get-token-pkce) de Auth0 para intercambiar un token de actualización por un Session Transfer Token. El Session Transfer Token está vinculado a una dirección IP o un ASN específicos.
4. Auth0 devuelve el Session Transfer Token para autenticarse en una aplicación web.
5. Auth0 autoriza el Session Transfer Token en la cookie o como parámetro de URL, y luego devuelve un [código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow).
6. La aplicación web intercambia el código de autorización por tokens de acceso o de actualización en el endpoint [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow/get-token).
7. La aplicación web inicia una sesión para el usuario.

Aprende a [configurar e implementar SSO de nativo a web](/docs/es/authenticate/single-sign-on/native-to-web/configure-implement-native-to-web)

<div id="limitations">
  ## Limitaciones
</div>

* Una vez habilitado SSO de nativo a web en una aplicación, el parámetro `session_transfer_token` solo funciona para SSO de nativo a web.
* Los <Tooltip tip="Token de actualización: token utilizado para obtener un nuevo token de acceso sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Tokens">tokens de actualización</Tooltip> generados a partir de una transacción previa de Session Transfer Token no generan nuevos Session Transfer Tokens.
