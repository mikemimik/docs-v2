Al implementar <Tooltip tip="inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente su sesión en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SS0) y configurar sus opciones, le resultarán útiles los siguientes endpoints de API.

<div id="authentication-api-endpoints">
  ## Endpoints de la Authentication API
</div>

<div id="post-logincallback">
  ### POST /login/callback
</div>

Para flujos de SSO iniciados por un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP), el endpoint [POST /login/callback](https://auth0.com/docs/api/authentication#idp-initiated-single-sign-on-sso-flow) puede aceptar una solicitud de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> de inicio de sesión del proveedor de identidad.

<div id="management-api">
  ## Management API
</div>

La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> cuenta con varios endpoints diseñados para ayudarte a administrar el SSO.

Para llamar a cualquiera de los endpoints de la Management API, necesitarás [obtener un token de acceso](https://auth0.com/docs/api/management/v2/tokens).

<div id="clients">
  ### Clients
</div>

Todas sus aplicaciones cliente/integraciones de SSO contienen información relevante para su implementación de SSO. Puede recuperar o cambiar esta información llamando a uno de los endpoints de Clients.

Los parámetros de interés de cada cliente incluyen:

* `sso`: indicador aplicable solo a clientes creados mediante integraciones de SSO. Si `true`, Auth0 gestiona el SSO; si `false`, el IdP gestiona el SSO.
* `sso_disabled`: indicador que se usa para habilitar o deshabilitar el SSO. Si `true`, el SSO está deshabilitado; si `false`, el SSO está habilitado. Esta opción solo se puede configurar mediante la Management API.
* `app_type`: tipo de aplicación. Si el cliente se creó mediante una de las integraciones de SSO integradas de Auth0, aparecerá el nombre de la integración de SSO (por ejemplo, `box` o `concur` en lugar de `native` o `spa`)

<div id="get-all-clients">
  #### Obtener todos los clientes
</div>

El endpoint [GET /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/get_clients) permite obtener información sobre las aplicaciones cliente que has configurado para tu tenant.

<div id="create-a-client">
  #### Crear un cliente
</div>

El endpoint [POST /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/post_clients) puede utilizarse para crear una nueva aplicación cliente.

<div id="get-a-client">
  #### Obtener un cliente
</div>

El endpoint [GET /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/get_clients_by_id) se puede usar para obtener información sobre un cliente específico que has configurado para tu tenant.

<div id="update-a-client">
  #### Actualizar un cliente
</div>

El endpoint [PATCH /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id) puede usarse para actualizar un cliente específico, incluidos los parámetros relacionados con SSO.

<div id="tenants">
  ### Tenants
</div>

Auth0 te permite controlar los siguientes parámetros a nivel del tenant que pueden afectar tu implementación de SSO:

* `session_lifetime`: Tiempo durante el que la sesión de Auth0 del usuario seguirá siendo válida.
* `idle_session_lifetime`: Tiempo que puede transcurrir antes de que el usuario deba volver a iniciar sesión por inactividad.

<div id="get-tenant-settings">
  ### Obtener la configuración del tenant
</div>

El endpoint [GET /api/v2/tenants/settings](https://auth0.com/docs/api/management/v2#!/Tenants/get_settings) devuelve la configuración de tu tenant.

<div id="update-tenant-settings">
  ### Actualizar la configuración del tenant
</div>

El endpoint [POST /api/v2/connections](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings) te permite actualizar la configuración del tenant.
