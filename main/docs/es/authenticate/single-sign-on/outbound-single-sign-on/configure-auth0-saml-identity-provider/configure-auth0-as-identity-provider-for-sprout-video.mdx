---
description: Describe cómo configurar Auth0 para que actúe como proveedor de identidad para Sprout Video.
title: Configurar Sprout Video como proveedor de servicios SAML
---

Usa la siguiente configuración de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> para Sprout Video.

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y crea una aplicación nueva o haz clic en el nombre de una aplicación existente para actualizarla.

2. Ve a la pestaña **Addons** y activa el interruptor **SAML2 Web App**.

3. En la pestaña **Settings**, establece la **URL de devolución de llamada de la aplicación** que puedes obtener en la configuración de Sprout Video. Normalmente será algo como: `https://app.sproutvideo.com/auth/sso/consumption`.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>

4. Pega el siguiente código en el cuadro de texto de **Settings** y haz clic en **Debug**.

   ```json lines
   {
        "audience":  "https://{yourSproutVideoAccount}.vids.io",
        "mappings": {  
          "user_id":     "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier",
          "email":       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
        },
        "createUpnClaim":       false,
        "passthroughClaimsWithNoMapping": false,
        "mapUnknownClaimsAsIs": false,
        "mapIdentities":        false,
        "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
        "nameIdentifierProbes": [
          "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
        ],
       }
   ```

5. Desplázate hasta la parte inferior de la página y haz clic en **Enable**.

6. En la pestaña **Usage**, localiza **Identity Provider Metadata** y haz clic en **Descargar** para descargar el archivo de metadatos. Lo necesitarás cuando configures Auth0 como proveedor de identidad.
