Puedes configurar Auth0 como <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> mediante el complemento SAML2 Web App para [GitHub Enterprise Server](https://help.github.com/en/enterprise/2.16/admin/user-management/using-saml) (es decir, tu instancia privada de GitHub). Si buscas instrucciones para configurar Auth0 como proveedor de identidad para GitHub Enterprise Cloud (github.com), consulta [Configurar Auth0 como proveedor de identidad para GitHub Enterprise Cloud](/docs/es/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider/configure-saml2-web-app-addon-for-github-enterprise-cloud).

<div id="configure-saml-sso-on-auth0">
  ## Configurar SAML SSO en Auth0
</div>

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y crea una nueva aplicación. Asígnale un nombre descriptivo, como **GitHub**. Puedes elegir cualquier tipo de aplicación, pero se recomienda **Aplicación web tradicional**.
2. Ve a la pestaña [Addons](https://manage.auth0.com/#/applications/\{yourClientId}/addons) y habilita el toggle **SAML2 Web App**.
3. En la pestaña **Settings**, establece la **URL de devolución de llamada de la aplicación** en: `https://github.com/orgs/{yourGitHubOrgName}/saml/consume`.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. Pega el siguiente código en el cuadro de texto de **Settings** y haz clic en **Debug**.

   ```json lines
   {
         "audience": "{yourGitHubServerUrl}",
         "mappings": {
       "user_id": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier",
       "email": "emails",
       "name": "full_name"
         },
         "passthroughClaimsWithNoMapping": false,
         "mapIdentities": false,
         "signatureAlgorithm": "rsa-sha256",
         "digestAlgorithm": "sha256",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier"
         ]
       }
   ```

   Reemplaza `{yourGitHubOrgName}` por el nombre de la organización de GitHub que corresponda a tu suscripción de GitHub. Por ejemplo, `https://github.yourname.com`.

   1. El mapeo anterior enviará `user_id` como **Identificador de nombre** a GitHub. Usa esta opción si habilitas más de una conexión para la aplicación de GitHub, ya que así se garantiza la unicidad (cada usuario tendrá un ID diferente).
   2. Si usas una sola conexión, quizá prefieras usar `nickname` u otro identificador único como un **Identificador de nombre** más fácil de reconocer, pero asegúrate de que la propiedad que elijas sea única.
5. Desplázate hasta la parte inferior de la pestaña y haz clic en **Enable**.
6. En la pestaña **Usage**, busca **Identity Provider Metadata** y haz clic en **Descargar** para descargar el archivo de metadatos. Lo necesitarás cuando configures Auth0 como proveedor de identidad.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Usage Tab](/docs/images/cdy7uua7fh8z/3T90BDpyTXFUWDp1JkncBU/86bdba6bed8130b34b42b25c9bf2d232/dashboard-applications-applications-addons-saml2-web-app-usage.png)</Frame>

<div id="configure-saml-sso-on-github">
  ## Configurar SAML SSO en GitHub
</div>

Sigue las instrucciones de [Using SAML on Enterprise Server - GitHub Help](https://help.github.com/en/enterprise/admin/user-management/using-saml#configuring-saml-settings) para completar la configuración en GitHub Enterprise Cloud. Necesitarás la siguiente información:

| Campo                   | Valor                                                                                                                                                      |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| URL de inicio de sesión | `https://{yourDomain}/samlp/{CLIENT_ID}`<br />`{'{CLIENT_ID}'}` será el `client_id` de la aplicación de GitHub que acabas de crear en Auth0.               |
| Emisor                  | `urn:auth0:{yourTenant}`                                                                                                                                   |
| Certificado público     | Descárgalo desde `https://{yourDomain}/pem`. Abre el archivo descargado con un editor de texto, copia su contenido y pégalo en el área de texto de GitHub. |
| Método de firma         | `RSA256-SHA256`                                                                                                                                            |
| Método de resumen       | `SHA256`                                                                                                                                                   |
