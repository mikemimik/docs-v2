Puedes configurar Auth0 para que actúe como <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> para [GitHub Enterprise Cloud](https://help.github.com/en/articles/about-authentication-with-saml-single-sign-on) (github.com). GitHub requiere una suscripción de nivel empresarial para habilitar el uso de un IdP externo de <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>.

* Para obtener más información sobre <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip> en GitHub, consulta [About authentication with SAML single sign-on](https://help.github.com/en/articles/about-authentication-with-saml-single-sign-on) en https://docs.github.com.
* Para aprender a configurar Auth0 como proveedor de identidad para GitHub Enterprise Server (instancia privada), consulta [Configure Auth0 as Identity Provider for GitHub Enterprise Server](/docs/es/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider/configure-saml2-web-app-addon-for-github-enterprise-server).

<div id="configure-saml-sso-on-auth0">
  ## Configura SAML SSO en Auth0
</div>

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y crea una nueva aplicación. Dale un nombre significativo, como **GitHub**. Puedes elegir cualquier tipo de aplicación, pero se recomienda **Aplicación web tradicional**.
2. Ve a la pestaña [Addons](https://manage.auth0.com/#/applications/\{yourClientId}/addons) y activa el interruptor **SAML2 Web App**.
3. En la pestaña **Settings**, establece la **URL de devolución de llamada de la aplicación** en: `https://github.com/orgs/{YOUR_GITHUB_ORG_NAME}/saml/consume`.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. Pega el siguiente código en el cuadro de texto de **Settings** y haz clic en **Debug**.

   ```json lines
   {
         "audience": "https://github.com/orgs/{yourGitHubOrgName}",
         "mappings": {
       "user_id": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier",
       "email": "emails",
       "name": "full_name"
         },
         "passthroughClaimsWithNoMapping": false,
         "mapIdentities": false,
         "signatureAlgorithm": "rsa-sha256",
         "digestAlgorithm": "sha256",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier"
         ]
       }
   ```

   1. Reemplaza `{YOUR_GITHUB_ORG_NAME}` por el nombre de la organización de GitHub correspondiente a tu suscripción. El mapeo enviará `user_id` como **Identificador de nombre** a GitHub. Esta es una buena opción si habilitas más de una conexión para la aplicación de GitHub, ya que garantiza la unicidad (cada usuario tendrá un ID diferente).
   2. Si usas una sola conexión, quizá prefieras usar `nickname` u otro identificador único como **Identificador de nombre** más fácil de identificar (pero asegúrate de que la propiedad que elijas sea única). Por ejemplo, si quieres usar `nickname` como **Identificador de nombre**, la sección `mappings` anterior se verá así:
5. Desplázate hasta la parte inferior de la pestaña y haz clic en **Enable**.
6. En la pestaña **Usage**, busca **Identity Provider Metadata** y haz clic en **Descargar** para descargar el archivo de metadatos. Lo necesitarás cuando configures Auth0 como proveedor de identidad.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Usage Tab](/docs/images/cdy7uua7fh8z/3T90BDpyTXFUWDp1JkncBU/86bdba6bed8130b34b42b25c9bf2d232/dashboard-applications-applications-addons-saml2-web-app-usage.png)</Frame>

<div id="configure-saml-sso-on-github">
  ## Configurar SAML SSO en GitHub
</div>

1. Sigue las instrucciones de GitHub para [habilitar y probar el inicio de sesión único de SAML (SSO) para tu organización](https://help.github.com/en/articles/enabling-and-testing-saml-single-sign-on-for-your-organization) y completar la configuración en GitHub Enterprise Cloud.
2. Necesitarás la siguiente información:

| Campo                   | Valor                                                                                                                                                   |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| URL de inicio de sesión | `https://{yourDomain}/samlp/{CLIENT_ID}`<br />`{'{CLIENT_ID}'}` será el `client_id` de la aplicación de GitHub que acabas de crear en Auth0.            |
| Emisor                  | `urn:auth0:{yourTenant}`                                                                                                                                |
| Certificado público     | Descárgalo de `https://{yourDomain}/pem`. Abre el archivo descargado con un editor de texto, copia su contenido y pégalo en el área de texto de GitHub. |
| Método de firma         | `RSA256-SHA256`                                                                                                                                         |
| Método de resumen       | `SHA256`                                                                                                                                                |

3. Como se recomienda en la documentación de GitHub, antes de habilitar SAML SSO para tu organización, haz clic en **Probar la configuración de SAML** para asegurarte de que la información que has introducido es correcta.

Al configurar el inicio de sesión único de SAML (SSO) para tu organización, puedes probar tu implementación sin afectar a los miembros de tu organización dejando sin marcar **Requerir autenticación SAML SSO** **para todos los miembros de la organización**.
