---
description: Aprende a configurar Auth0 como proveedor de identidad mediante el addon SAML2 Web App para Freshdesk.
title: Configurar Freshdesk como proveedor de servicios SAML
---

Usa la siguiente configuración de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin usar una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> para Freshdesk.

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y crea una aplicación nueva o haz clic en el nombre de una aplicación para actualizarla.

2. Ve a la pestaña **Addons** y activa el interruptor **SAML2 Web App**.

3. En la pestaña **Settings**, establece la **URL de devolución de llamada de la aplicación** en: `https://{FD Domain}.freshdesk.com/login/saml`.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>

   1. Puedes encontrarla en:
      **Freshworks Dashboard &gt; Security &gt; SAML SSO &gt; Assertion Consumer Service(ACS) URL**
   2. Puedes encontrar la audience en:
      **Freshworks Dashboard &gt; Security &gt; SAML SSO &gt; Service Provider(SP) Entity ID URL**

4. Pega el siguiente código en el cuadro de texto **Settings** y haz clic en **Debug**.

   ```json lines
   {
         "audience": "https://{FD Domain}.freshworks.com/sp/SAML/{xxxxxxxxxxxxxxxxxxx}/metadata",
         "mappings": {
       "email": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
       "given_name": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/givenname",
       "family_name": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/surname"
         },
         "createUpnClaim": false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities": false,
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
         ]
       }
   ```

5. Desplázate hasta la parte inferior de la página y haz clic en **Enable**.

6. En la pestaña **Usage**, localiza **Identity Provider Metadata** y haz clic en **Download** para descargar el archivo de metadatos. Lo necesitarás cuando configures Auth0 como proveedor de identidad.
