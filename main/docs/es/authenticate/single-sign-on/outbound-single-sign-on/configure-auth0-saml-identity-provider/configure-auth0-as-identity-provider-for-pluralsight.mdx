description: Aprenda a configurar Auth0 como proveedor de identidad mediante el complemento SAML2 Web App para Pluralsight.
title: Configurar Pluralsight como proveedor de servicios SAML
---

Use la siguiente configuración de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> para Pluralsight.

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y cree una aplicación nueva o haga clic en el nombre de una aplicación para editarla.

2. Vaya a la pestaña **Addons** y active el interruptor de **SAML2 Web App**.

3. En la pestaña **Settings**, establezca la **URL de devolución de llamada de la aplicación** en: `https://prod-pf.pluralsight.com/sp/ACS.saml2`.

   <Frame>![Dashboard Applications Applications pestaña Addons SAML2 Web App pestaña Settings](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>

4. Pegue el siguiente código en el cuadro de texto **Settings** y haga clic en **Debug**.

   ```json lines
   {
         "audience":  "www.pluralsight.com",
         "mappings": {
       "user_id":     "id",
       "email":       "email",
       "given_name":  "firstName",
       "family_name": "lastName"
         },
         "createUpnClaim":       false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities":        false,
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
         ]
       }
   ```

5. Desplácese hasta la parte inferior de la página y haga clic en **Enable**.

6. En la pestaña **Usage**, localice **Identity Provider Metadata** y haga clic en **Download** para descargar el archivo de metadatos. Lo necesitará cuando configure Auth0 como proveedor de identidad.

   <Frame>![Dashboard Applications Applications pestaña Addons SAML2 Web App pestaña Usage](/docs/images/cdy7uua7fh8z/3T90BDpyTXFUWDp1JkncBU/86bdba6bed8130b34b42b25c9bf2d232/dashboard-applications-applications-addons-saml2-web-app-usage.png)</Frame>
