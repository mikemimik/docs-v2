Cuando una WebView o un navegador inicia una llamada al endpoint `/authorize`, Auth0 determina si hay una sesión activa y luego reutiliza la sesión existente o tiene en cuenta el `session_transfer_token` proporcionado. Para evitar riesgos de inyección de sesión, Auth0 utiliza una evaluación segura y predefinida para determinar si el `session_transfer_token` es válido. Para obtener más información, consulta [Configure and Implement Native to Web SSO](/docs/es/authenticate/single-sign-on/native-to-web/configure-implement-native-to-web).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  SSO de nativo a web no modifica la autenticación estándar de [inicio de sesión único](/docs/es/authenticate/single-sign-on) de Auth0.
</Callout>

**Los flujos específicos de <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip> de nativo a web pueden dar lugar a los siguientes comportamientos:**

1. El usuario inicia sesión cuando se envía un `session_transfer_token` válido y no existe una sesión previa de Auth0.
2. El usuario inicia sesión cuando se envía un `session_transfer_token` válido y se encuentra una sesión previa de Auth0 para el mismo usuario.
3. Se solicita al usuario que inicie sesión cuando se encuentra una sesión previa de Auth0 y el `session_transfer_token` pertenece a otro usuario. Además, la sesión previa de Auth0 se revoca.
4. Se solicita al usuario que inicie sesión cuando se encuentra una sesión previa de Auth0 y el `session_transfer_token` no es válido.

<div id="sessions-and-refresh-token-revocation">
  ## Sesiones y revocación de tokens de actualización
</div>

Se utiliza un [`session_transfer_token`](/docs/es/authenticate/single-sign-on/native-to-web/configure-implement-native-to-web#create-and-manage-session-transfer-tokens) para iniciar una sesión segura en una WebView o en un navegador y autenticar al usuario de forma segura sin que tenga que iniciar sesión. Estas sesiones web también pueden emitir sus propios <Tooltip tip="Refresh Token: Token que se usa para obtener un nuevo token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip>.

SSO de nativo a web aplica un conjunto de reglas de revocación para garantizar un comportamiento uniforme y seguro cuando se revocan sesiones y tokens de actualización:

* Cuando se revoca un token de actualización, también se revocan los tokens de actualización y las sesiones asociadas si `enforce_cascade_revocation` está habilitado en la aplicación nativa.
* Cuando se revoca una sesión web, también se revocan los tokens de actualización asociados si `enforce_online_refresh_tokens` está habilitado en la aplicación web
* No se permite el anidamiento de SSO de nativo a web. Una sesión web creada con un `session_transfer_token` no puede generar otro `session_transfer_token`.
