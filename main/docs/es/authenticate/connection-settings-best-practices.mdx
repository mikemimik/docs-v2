title: Prácticas recomendadas para la configuración de conexiones
---

Aquí tienes algunas prácticas recomendadas para configurar conexiones. Antes de hacerlo, tómate un momento para revisar qué son las conexiones y [conocer los conceptos básicos de la autenticación](/docs/es/authenticate) para tu tipo de aplicación.

<div id="use-your-credentials-for-social-connections">
  ## Usa tus credenciales para las conexiones sociales
</div>

Auth0 proporciona credenciales predeterminadas para las conexiones sociales para que puedas empezar. Debes sustituir estas credenciales temporales por las tuyas para evitar restricciones.

<div id="review-requested-data">
  ## Revisa los datos solicitados
</div>

Debes revisar los datos que solicitas de cada conexión social. Los usuarios deben dar su consentimiento para los datos solicitados. Solicitar demasiados datos innecesarios puede hacer que los usuarios rechacen la solicitud de autorización por motivos de privacidad.

<div id="set-password-policy-for-database-connections">
  ## Configurar la política de contraseñas para las conexiones de base de datos
</div>

Configura la política de contraseñas para tus conexiones de base de datos de Auth0, de modo que los usuarios creados tengan contraseñas seguras. Puedes configurar la política de contraseñas en la configuración de las conexiones de base de datos en el Dashboard o mediante la Auth0 <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

La política de contraseñas se aplica a los restablecimientos de contraseña realizados tanto en la página de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> como con la Auth0 Management API.

<div id="disable-user-signup-if-its-not-appropriate-for-each-database-connection">
  ## Deshabilita el registro de usuarios si no corresponde para cada conexión de base de datos
</div>

En la configuración de tu conexión de base de datos, especifica si debe habilitarse el registro de usuarios de autoservicio. Si lo habilitaste durante el desarrollo, revisa si deberías deshabilitarlo en los tenants de producción.

Habilita esta configuración en los tenants de producción solo si permites que los usuarios finales se registren por autoservicio. Si no se permite el registro de autoservicio, deshabilita la funcionalidad. Luego, puedes agregar usuarios a la conexión de base de datos con la Auth0 Management API.

Al usar la funcionalidad Organizations, puedes habilitar el registro de autoservicio para una conexión de base de datos determinada en el contexto de una organización específica. Cuando está habilitado, los usuarios pueden acceder a un enlace de registro en el prompt de inicio de sesión y obtener membresía en una organización específica al crear su cuenta. Para obtener más información, consulta [Habilitar conexiones de organización](/docs/es/manage-users/organizations/configure-organizations/enable-connections).

<div id="review-applications-enabled-for-each-connection">
  ## Revisa las aplicaciones habilitadas para cada conexión
</div>

Para cada conexión, revisa la lista de aplicaciones permitidas. Asegúrate de que no existan rutas de autenticación no deseadas para acceder a una aplicación. De forma predeterminada, las aplicaciones nuevas pueden tener habilitadas todas las conexiones de tu tenant, lo que quizá no sea apropiado.

<div id="use-rsa-sha256-for-saml-connections">
  ## Usa RSA-SHA256 para las conexiones SAML
</div>

Configura cualquier conexión <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin necesidad de una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> para que firme las solicitudes y use RSA-SHA256 como algoritmo de firma. Esto garantiza que el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> SAML remoto pueda validar si las solicitudes de autenticación proceden o no de una aplicación legítima.
