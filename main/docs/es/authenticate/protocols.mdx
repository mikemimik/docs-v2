title: Protocolos de autenticación y autorización en Auth0
sidebarTitle: Protocolos

---

Existe un conjunto de especificaciones y protocolos abiertos que definen cómo diseñar un sistema de autenticación y autorización. Definen cómo debes gestionar la identidad, transferir datos personales de forma segura y decidir quién puede acceder a las aplicaciones y los datos.

Los estándares del sector de la identidad que usamos en Auth0 son:

* Open Authorization (<Tooltip tip="OAuth 2.0: marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip>) 1: el estándar original para la delegación de acceso. Se utiliza para que un usuario conceda a sitios web acceso a su información en otros sitios web o aplicaciones, pero sin darles sus credenciales.
* [Open Authorization (OAuth) 2](/docs/es/authenticate/protocols/oauth): un estándar de autorización que permite a un usuario conceder acceso limitado a sus recursos en un sitio a otro sitio, sin tener que exponer sus credenciales. Usas este estándar cada vez que inicias sesión en un sitio con tu cuenta de Google y se te pregunta si aceptas compartir tu dirección de correo electrónico y tu lista de contactos con ese sitio.
* [OpenID Connect (OIDC)](/docs/es/authenticate/protocols/openid-connect-protocol): una capa de identidad que se basa en OAuth 2 y permite verificar fácilmente la identidad del usuario, así como obtener información básica del perfil del <Tooltip tip="Identity Provider (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>.
* [Lightweight Directory Access Protocol (LDAP)](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector): un protocolo de aplicación que se utiliza para acceder a servicios distribuidos de información de directorio y mantenerlos a través de una red IP. La función de LDAP es permitir el acceso a un directorio existente, como Active Directory (AD). Auth0 usa un conector que instalas en tu red para integrarse con AD/LDAP.
* [JSON Web Tokens (JWTs)](/docs/es/secure/tokens/json-web-tokens): un estándar abierto que define una forma compacta y autocontenida de transmitir información de manera segura entre partes como un objeto JSON.
* [Security Assertion Markup Language (SAML)](/docs/es/authenticate/protocols/saml/saml-configuration): un formato de datos estándar abierto basado en XML que permite a las empresas comunicar información de autenticación y autorización de usuarios a compañías partner y aplicaciones empresariales que sus empleados puedan usar.
* [System for Cross-domain Identity Management (SCIM)](/docs/es/authenticate/protocols/scim): un conjunto de protocolos de nivel de aplicación para gestionar y comunicar de forma segura datos de usuario a través de múltiples dominios. Los clientes SCIM pueden integrarse para gestionar operaciones CRUD (create, replace, update, delete), aplicar consultas y filtros, y crear grupos de usuarios dentro de tu organización.
* [WS-Federation (WS-Fed)](/docs/es/authenticate/protocols/ws-fed-protocol): un estándar desarrollado por Microsoft y ampliamente utilizado en sus aplicaciones. Define la forma en que los <Tooltip tip="Security Token: artefacto firmado digitalmente que se usa para demostrar que el usuario fue autenticado correctamente." cta="Ver glosario" href="/docs/es/glossary?term=security+tokens">tokens de seguridad</Tooltip> pueden transportarse entre distintas entidades para intercambiar información de identidad y autorización.
