sidebarTitle: Protocolos

---

Existe un conjunto de especificaciones y protocolos abiertos que definen cómo diseñar un sistema de autenticación y autorización. Establecen cómo se debe gestionar la identidad, transferir datos personales de forma segura y decidir quién puede acceder a las aplicaciones y a los datos.

Los estándares del sector de la identidad que usamos en Auth0 son:

* Open Authorization (<Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip>) 1: el estándar original para la delegación de acceso. Se utiliza como una forma de que un usuario conceda a sitios web acceso a su información en otros sitios web o aplicaciones, pero sin proporcionarles las credenciales.
* [Open Authorization (OAuth) 2](/docs/es/authenticate/protocols/oauth): un estándar de autorización que permite a un usuario conceder acceso limitado a sus recursos de un sitio a otro, sin tener que exponer sus credenciales. Utiliza este estándar cada vez que inicia sesión en un sitio con su cuenta de Google y se le pregunta si acepta compartir su dirección de correo electrónico y su lista de contactos con ese sitio.
* [OpenID Connect (OIDC)](/docs/es/authenticate/protocols/openid-connect-protocol): una capa de identidad que se basa en OAuth 2 y permite verificar fácilmente la identidad del usuario, así como obtener información básica del perfil del <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>.
* [Lightweight Directory Access Protocol (LDAP)](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector): un protocolo de aplicación utilizado para acceder a servicios distribuidos de información de directorios y mantenerlos a través de una red IP. La función de LDAP es permitir el acceso a un directorio existente, como Active Directory (AD). Auth0 utiliza un conector que usted instala en su red para integrarse con AD/LDAP.
* [JSON Web Tokens (JWTs)](/docs/es/secure/tokens/json-web-tokens): un estándar abierto que define una forma compacta y autocontenida de transmitir información de forma segura entre partes como un objeto JSON.
* [Security Assertion Markup Language (SAML)](/docs/es/authenticate/protocols/saml/saml-configuration): un formato de datos abierto basado en XML que permite a las empresas comunicar información de autenticación y autorización de usuarios a empresas partner y aplicaciones empresariales que sus empleados puedan utilizar.
* [System for Cross-domain Identity Management (SCIM)](/docs/es/authenticate/protocols/scim): un conjunto de protocolos de nivel de aplicación para gestionar y comunicar de forma segura datos de usuario en múltiples dominios. Los clientes SCIM pueden integrarse para gestionar operaciones CRUD (create, replace, update, delete), aplicar consultas y filtros, y crear grupos de usuarios dentro de su organización.
* [WS-Federation (WS-Fed)](/docs/es/authenticate/protocols/ws-fed-protocol): un estándar desarrollado por Microsoft y ampliamente utilizado en sus aplicaciones. Define la forma en que los <Tooltip tip="Token de seguridad: Artefacto firmado digitalmente que se utiliza para demostrar que el usuario se autenticó correctamente." cta="Ver glosario" href="/docs/es/glossary?term=security+tokens">tokens de seguridad</Tooltip> pueden transportarse entre distintas entidades para intercambiar información de identidad y autorización.
