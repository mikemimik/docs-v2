En general, para probar una conexión, debe iniciar sesión en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Vaya a [Auth0 Dashboard](https://manage.auth0.com/#), seleccione **Authentication** y luego el tipo de conexión que desea probar.
2. Busque la conexión que desea probar y seleccione **Try Connection** en el menú **More Options** (**...**).
3. Inicie sesión con el proveedor de identidad.
4. Espere a que aparezca la página **¡Funciona!**, que muestra el resultado.

Auth0 simula el flujo de autenticación como si fuera una aplicación y muestra el perfil de usuario obtenido tras una autenticación correcta.

<div id="test-partner-connections">
  ## Probar conexiones de partners
</div>

Si está probando una conexión que pertenece a otra persona y no tiene credenciales de prueba con ella, no es posible tener sesión iniciada en el Auth0 Dashboard. Esto es habitual al conectarse a conexiones empresariales, como <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Providers">proveedores de identidad</Tooltip> (IdPs) o Active Directory. ¿La solución? Sus partners pueden probar la nueva conexión.

1. Registre una aplicación de prueba.

   * Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione **Create Application**. Puede darle cualquier nombre (por ejemplo, `Test App`).
   * En la configuración de la aplicación recién creada, establezca **Allowed Callback URLs** en `http://jwt.io`.
   * Seleccione **Save Changes**.
2. Envíe a su partner el enlace para iniciar sesión.

   <AuthCodeBlock children={codeExample} language="http" />

   Reemplace estos dos parámetros:

   * `client_id`: ID de cliente de la aplicación creada en el Paso 1.
   * `connection`: nombre de la conexión que desea probar.
3. Pruebe la conexión. Cuando su partner siga el enlace, se le redirigirá al proveedor de identidad configurado (la conexión). Después de autenticarse correctamente, volverá a [JWT.io](http://jwt.io), donde se decodificarán del token todas las propiedades del usuario.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La aplicación de prueba no es una aplicación real. [JWT.io](http://jwt.io) es un sitio web de pruebas que decodifica tokens enviados en un fragmento de URL.
</Callout>
