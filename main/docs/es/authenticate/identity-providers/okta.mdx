Okta Workforce Enterprise connection es una integración optimizada con soporte oficial y el método preferido para implementar Okta como <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP) en Auth0.

Esta integración permite a tus clientes gestionar con Okta el acceso de sus empleados a tu aplicación.

Además, si usas Auth0 para la gestión de identidades de clientes y Okta internamente para la gestión de identidades de empleados, esta integración es una forma eficaz de gestionar ambos entornos de identidad.

Si Okta Workforce Enterprise connection no es compatible con tu caso de uso, puedes [configurar Okta como un IdP de SAML](/docs/es/authenticate/protocols/saml/saml-sso-integrations/configure-auth0-saml-service-provider/configure-okta-as-saml-identity-provider) o [configurar un servidor de autorización personalizado](https://developer.okta.com/docs/concepts/auth-servers/#custom-authorization-server) según tus necesidades.

<div id="create-okta-oidc-app-integration">
  ## Crear integración de aplicación OIDC de Okta
</div>

Para saber cómo crear una integración de aplicación OIDC de Okta, consulta [Create OIDC app integrations on Okta Help Center](https://help.okta.com/en-us/Content/Topics/Apps/Apps_App_Integration_Wizard_OIDC.htm).

Usa la siguiente configuración al configurar tu integración de aplicación OIDC de Okta:

1. Selecciona **OIDC** como **Sign-in method**.
2. Selecciona **Web application** como **Application type** y configura los siguientes parámetros:

| Campo                 | Descripción                                                                 |
| --------------------- | --------------------------------------------------------------------------- |
| Name                  | El nombre de tu aplicación.                                                 |
| Sign-in Redirect URIs | `https://{YOUR_AUTH0_TENANT}.{YOUR_TENANT_REGION}.auth0.com/login/callback` |
| Trusted Origins       | `https://{yourDomain}`                                                      |

<Card title="Encuentra tu nombre de dominio de Auth0 para las redirecciones">
  Si tu nombre de dominio de Auth0 no aparece arriba y no estás usando nuestra funcionalidad de [dominios personalizados](/docs/es/customize/custom-domains), tu nombre de dominio se compone del nombre de tu tenant, tu subdominio regional y `auth0.com`, separados por el símbolo de punto (`.`).

  Por ejemplo, si el nombre de tu tenant es `exampleco-enterprises` y tu tenant está en la región de EE. UU., tu nombre de dominio de Auth0 sería `exampleco-enterprises.us.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.us.auth0.com/login/callback`.

  Sin embargo, si tu tenant está en la región de EE. UU. y se creó antes de junio de 2020, tu nombre de dominio de Auth0 sería `exampleco-enterprises.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.auth0.com/login/callback`.

  Si estás usando [dominios personalizados](/docs/es/customize/custom-domains), tu **Redirect URI** sería `https://<YOUR CUSTOM DOMAIN>/login/callback`.
</Card>

3. Anota el **ID de cliente** y el **secreto de cliente** que Okta genera para tu integración de aplicación.

<div id="add-test-user-to-okta-app-integration">
  ## Agregar un usuario de prueba a la integración de aplicación de Okta
</div>

Cree un usuario de prueba en su directorio de Okta para probar su integración de aplicación:

1. En el Dashboard de administración de Okta, vaya a **Directory &gt; People**.
2. Seleccione **Add Person**.
3. Introduzca los datos del usuario de prueba, incluida una contraseña.
4. Guarde el usuario de prueba.
5. En **Directory**, seleccione el usuario nuevo
6. Vaya a la pestaña **Applications** del usuario y elija **Assign Applications**.
7. Seleccione el nombre de la aplicación que creó anteriormente.

<div id="create-okta-workforce-enterprise-connection-in-auth0">
  ## Crear una Okta Workforce Enterprise connection en Auth0
</div>

Asegúrate de tener a mano el **<Tooltip tip="ID de cliente: valor de identificación proporcionado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** y el **<Tooltip tip="ID de cliente: valor de identificación proporcionado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>** de tu integración de aplicación Okta OIDC:

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), localiza **Okta Workforce** y selecciona el botón **+**.

   <Frame>![Dashboard - Conexiones - empresarial](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Introduce los datos de tu conexión y, a continuación, selecciona **Create:**

| Campo                  | Descripción                                                                                                                                                                                                        |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Connection name**    | Identificador lógico de tu conexión; debe ser único en tu tenant. Una vez establecido, este nombre no se puede cambiar.                                                                                            |
| **Okta Domain**        | El nombre de dominio de Okta para tu organización.                                                                                                                                                                 |
| **ID de cliente**      | Identificador único de tu aplicación de Okta registrada. Introduce el valor guardado del **ID de cliente** de la aplicación que acabas de registrar en la Consola de administración de Okta.                       |
| **secreto de cliente** | Cadena que se usa para obtener acceso a tu aplicación de Okta registrada. Introduce el valor guardado del **secreto de cliente** de la aplicación que acabas de registrar en la Consola de administración de Okta. |

3. En la vista **Aprovisionamiento**, configura cómo se crean y actualizan los perfiles de usuario en Auth0.

| Campo                                                                    | Descripción                                                                                                                                                                                                                        |
| ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **sincronizar atributos del perfil de usuario en cada inicio de sesión** | Cuando está habilitado, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada inicio de sesión, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0. |
| **Sincronizar perfiles de usuario mediante SCIM**                        | Cuando está habilitado, Auth0 permite sincronizar los datos del perfil de usuario mediante SCIM. Para más información, consulta [configurar SCIM entrante](/docs/es/authenticate/protocols/scim/configure-inbound-scim).              |

4. Cambia a la vista **Experiencia de inicio de sesión** y configura cómo iniciarán sesión tus usuarios con esta conexión:

| Campo                              | Descripción                                                                                                                                                                                                                                                    |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Home Realm Discovery**           | Compara el dominio de correo electrónico de un usuario con los dominios proporcionados del proveedor de identidad. Para más información, consulta [Configure Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first) |
| **Display connection button**      | Esta opción muestra las siguientes opciones para personalizar el botón de conexión de tu aplicación.                                                                                                                                                           |
| **Button display name** (Opcional) | Texto que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón muestra: &quot;Continuar con &#123;Button display name&#125;&quot;.                                                                          |
| **Button logo URL** (Opcional)     | URL de la imagen que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón de inicio de sesión de Universal Login muestra la imagen como un cuadrado de 20 px por 20 px.                                     |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los campos opcionales solo están disponibles con Universal Login. Los clientes que usan Classic Login no verán el botón Add, Button display name ni Button logo URL.
</Callout>

<div id="configure-global-token-revocation">
  ## Configurar la revocación global de tokens
</div>

Este tipo de conexión admite un endpoint de revocación global de tokens, que permite a un proveedor de identidad compatible revocar las sesiones de usuario de Auth0, revocar <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión nuevamente." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> y activar el cierre de sesión por back-channel en las aplicaciones que usan un back-channel seguro.

Esta funcionalidad se puede usar con Universal Logout de Okta.

Para obtener más información e instrucciones de configuración, consulta [Universal Logout](/docs/es/authenticate/login/logout/universal-logout).
