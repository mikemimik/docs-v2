title: Conecta tu aplicación a Active Directory con LDAP
---

Auth0 se integra con Active Directory (AD) mediante Lightweight Directory Access Protocol (LDAP) a través de un **Conector de Active Directory/LDAP** que instalas en tu red.

El **Conector de AD/LDAP** (1) es un puente entre tu **Active Directory/LDAP** (2) y el **servicio de Auth0** (3). Este puente es necesario porque AD/LDAP normalmente está restringido a tu red interna, y Auth0 es un servicio en la nube que funciona en un entorno completamente distinto.

<Frame>![Diagrama general del Conector de AD/LDAP](/docs/images/cdy7uua7fh8z/2HT4cRvUDzA2OdEPlfgLDV/be32ffe82562dc8f07fdd2097f14d881/ldap-connect.png)</Frame>

Para [alta disponibilidad y balanceo de carga](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-high-availability), puedes instalar varias instancias del conector. Todas las conexiones son salientes desde el conector hacia el servidor de Auth0, por lo que, en general, no es necesario hacer cambios en tu firewall.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de comenzar:

* [Registra tu aplicación en Auth0](/docs/es/get-started/auth0-overview/create-applications).

  * Selecciona un **Tipo de aplicación** adecuado.
  * Agrega una **URL de devolución de llamada permitida**: `{https://yourApp/callback}`.
  * Asegúrate de que los [**Tipos de concesión**](/docs/es/get-started/applications/update-grant-types) de tu aplicación incluyan los flujos adecuados.

<div id="steps">
  ## Pasos
</div>

Para conectar tu aplicación a Active Directory/LDAP, debes:

1. [Crear una conexión empresarial en Auth0](#create-an-enterprise-connection-in-auth0) y descargar el instalador
2. [Instalar el conector en tu red](#install-the-connector-on-your-network)
3. [Habilitar la conexión empresarial para tu aplicación en Auth0](#enable-the-enterprise-connection-for-your-auth0-application)
4. [Probar la conexión](#test-the-connection)

<div id="create-an-enterprise-connection-in-auth0">
  ## Crear una conexión empresarial en Auth0
</div>

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; empresarial](https://manage.auth0.com/#/connections/enterprise), localiza **Active Directory / LDAP** y selecciona su `+`.

   <Frame>![Dashboard - Conexiones - empresarial](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Introduce los datos de tu conexión y selecciona **Create**:

| Campo                                                                    | Descripción                                                                                                                                                                                                                                                                                                               |
| ------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Nombre de la conexión**                                                | Identificador lógico de tu conexión; debe ser único para tu tenant. Una vez establecido, este nombre no se puede cambiar.                                                                                                                                                                                                 |
| **Nombre para mostrar** (opcional)                                       | Texto que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón de inicio de sesión de Universal Login muestra: &quot;Continuar con &#123;Nombre para mostrar&#125;&quot;.                                                                                              |
| **URL del logotipo** (opcional)                                          | URL de la imagen que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón de inicio de sesión de Universal Login muestra la imagen como un cuadrado de 20 px por 20 px.                                                                                                |
| **Dominios del IdP** (opcional)                                          | Lista separada por comas de dominios de correo electrónico válidos que podrán iniciar sesión con esta conexión. Solo es necesario si usas el widget de inicio de sesión Lock.                                                                                                                                             |
| **Deshabilitar caché**                                                   | Cuando está habilitado, desactiva la caché.                                                                                                                                                                                                                                                                               |
| **Usar autenticación con certificado SSL del cliente**                   | Cuando está habilitado, usa autenticación con certificado SSL del cliente.                                                                                                                                                                                                                                                |
| **Usar autenticación integrada de Windows (Kerberos)**                   | Cuando está habilitado, se te pedirá que introduzcas un rango de direcciones IP. Cuando los usuarios inicien sesión desde estas direcciones IP, se usará Kerberos; de lo contrario, se solicitarán el username y la contraseña de AD/LDAP. Normalmente, el rango de IP introducido corresponde a direcciones de intranet. |
| **Sincronizar atributos del perfil de usuario en cada inicio de sesión** | Cuando está habilitado, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada inicio de sesión, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0.                                                                                        |

<Frame>![Introducir los detalles de la conexión AD / LDAP](/docs/images/cdy7uua7fh8z/2IR80sIwNNyPlDMxIjWv8e/b6748243f90ce0e7d771a39cef24cb0f/LDAP_Connection__1_.png)</Frame>
3. Descarga el instalador proporcionado y anota la **URL del ticket de aprovisionamiento**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Disponemos de distintas versiones del Conector para instalar en Windows o Linux.
</Callout>

<div id="install-the-connector-on-your-network">
  ## Instala el Conector en tu red
</div>

Configura el [Conector de AD/LDAP](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector) siguiendo las instrucciones correspondientes a tu plataforma:

* [Instala el Conector de AD/LDAP en Windows](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/install-configure-ad-ldap-connector)
* [Instala el Conector de AD/LDAP en plataformas que no son de Microsoft](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/install-configure-ad-ldap-connector)

<div id="enable-the-enterprise-connection-for-your-auth0-application">
  ## Habilita la conexión empresarial para tu aplicación de Auth0
</div>

Para usar tu nueva conexión de AD, primero debes [habilitar la conexión](/docs/es/authenticate/identity-providers/enterprise-identity-providers/enable-enterprise-connections) en tus Aplicaciones de Auth0.

  ## Probar la conexión
</div>
