<ReleaseStageNotice feature="Google Workplace Directory Sync para grupos" stage="ea" terms="true" />

Al habilitar Directory Sync en tu [conexión empresarial de Google Workspace](/docs/es/authenticate/identity-providers/enterprise-identity-providers/google-apps), puedes sincronizar los perfiles de usuario, las estructuras de grupos y la pertenencia a grupos desde Google Workspace a Auth0.

Puedes sincronizar automáticamente o de forma manual:

* La sincronización automática se ejecuta cada 30 minutos después de que finalice la sincronización anterior.

* La sincronización manual se ejecuta cuando la activas.

<div id="enable-directory-sync">
  ## Habilitar Directory Sync
</div>

Puede habilitar Directory Sync mediante el Auth0 Dashboard o la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    <Steps>
      <Step title="Requisitos previos">
        Antes de comenzar, debes tener:

        * [Una conexión empresarial de Google Workspace en Auth0](./google-apps)

        * Privilegios de administrador en la organización de Google Workspace.
      </Step>

      <Step title="Habilita la API de directorio de administrador para tu conexión empresarial">
        Para habilitar Directory Sync, el token de acceso de Google de tu conexión empresarial de Google Workspace debe tener los scopes adecuados para acceder a las API de Google.

        En la pestaña **Settings** de una conexión empresarial de Google Workspace existente (o al crear una conexión nueva), en la sección **Identity Provider API**, selecciona **Use Admin Directory API** y elige:

        * **Users scopes** para agregar scopes solo para acceder a usuarios.

        * **Users and Groups scopes** para agregar scopes para acceder tanto a usuarios como a grupos.

        <Tip>
          Si usas Directory Sync, también te recomendamos desactivar **Sync User Profile Attributes at Login** en esta sección para evitar actualizaciones en conflicto entre varios métodos de sincronización.
        </Tip>

        Haz clic en **Save Changes**.
      </Step>

      <Step title="Verifica el consentimiento del administrador de Google">
        Desde [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), abre tu conexión de Google Workspace. En la pestaña **Setup**, haz una de estas dos cosas:

        * Sigue el enlace **Continuar** si tienes permisos de administrador para configurar los ajustes de Google Workspace y usar las API de administrador de Google, o bien

        * Proporciona la URL indicada a tu administrador para que pueda ajustar la configuración requerida
      </Step>

      <Step title="Habilita Directory Sync">
        En la pestaña **Aprovisionamiento** de tu conexión, activa **Provision Users Using Directory Sync** y elige tus opciones de configuración:

        * En **Resources**, dentro de **Sync**, elige si quieres sincronizar **Users** o **Users and Groups**.

          Si sincronizas tanto usuarios como grupos, la sección se expande para mostrar qué grupos se están sincronizando (**Syncing all groups** de forma predeterminada). Para personalizar los grupos sincronizados, consulta el siguiente paso.

        * En **Schedule**, marca opcionalmente **Enable Automatic Synchronization** para sincronizar automáticamente cada 30 minutos.

          Puedes iniciar una sincronización manual seleccionando **Synchronize now**.

        * En **Attribute Mapping**, puedes personalizar el mapeo de atributos de Google a los atributos del perfil de usuario de Auth0.
      </Step>

      <Step title="Personaliza los grupos sincronizados (opcional)">
        Cuando habilitas Directory Sync para sincronizar tanto usuarios como grupos desde Google Workspace, todos los grupos se sincronizan de forma predeterminada. Puedes personalizar qué grupos sincronizas cargando un archivo JSON con los ID de los grupos que quieras sincronizar.

        Primero, crea el archivo JSON. Puedes [obtener una lista de todos los ID de grupo con la Google Workspace Directory API](https://developers.google.com/workspace/admin/directory/reference/rest/v1/groups/list). Crea el archivo con el siguiente formato y sustituye los valores de ejemplo por los ID de grupo reales:

        ```json
        {
            "groups": [
                {
                    "id": "example-id-1"
                },
                {
                    "id": "example-id-2"
                },
                {
                    "id": "example-id-3"
                }
            ]
        }
        ```

        A continuación, carga el archivo en Auth0:

        1. En la pestaña **Aprovisionamiento** de tu conexión, debajo de la opción activada **Provision Users Using Directory Sync**, busca la sección **Resources** y selecciona el botón **Select Groups...**.

        2. En la ventana **Select Groups** que se abre, selecciona **Pick specific groups** para mostrar la sección **Groups JSON file**.

        3. Selecciona **+ Choose file** y carga el archivo JSON.

        4. Cuando el archivo se haya cargado, selecciona **Select Groups** en la parte inferior de la ventana.

        La sección **Resources** muestra **Syncing specific groups**. Para actualizar qué grupos sincronizas, vuelve a la misma ventana **Select Groups** y carga un archivo JSON nuevo o elige **Sync all**.

        Si detectamos que se elimina un ID de grupo sincronizado en Google Workspace Directory, eliminamos automáticamente ese ID de grupo de la configuración para las siguientes tareas de sincronización.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Management API">
    <Steps>
      <Step title="Requisitos previos">
        Antes de comenzar, debes tener lo siguiente:

        * [Una conexión empresarial de Google Workspace en Auth0](./google-apps)

        * Privilegios de administrador en la organización de Google Workspace.

        * Un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens#management-api-access-tokens) con los siguientes scopes:
          * `create:directory_provisionings`
          * `read:directory_provisionings`
          * `update:directory_provisionings`
          * `delete:directory_provisionings`

        * El ID de tu conexión de Google Workspace (`CONNECTION_ID`).
      </Step>

      <Step title="Habilita la API de directorio de administración para tu conexión empresarial">
        Al crear una nueva conexión empresarial de Google Workspace con el [endpoint Create a connection](https://auth0.com/docs/api/management/v2/connections/post-connections) (`POST /v2/connections`), o al modificar una conexión empresarial existente con el [endpoint Update a connection](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) (`PATCH /v2/connections/{id}`), configura los siguientes parámetros del cuerpo de la solicitud:

        * `option.api_enable_users` en `true` para agregar scopes de acceso a usuarios.
        * `option.api_enable_groups` en `true` para agregar también scopes de acceso a grupos. El acceso a grupos requiere acceso a usuarios.

        <Tip>
          Si usas Directory Sync, te recomendamos desactivar la sincronización de perfiles de usuario durante el login (estableciendo `options.set_user_root_attributes` en `never_on_login`) para evitar actualizaciones en conflicto entre varios métodos de sincronización.
        </Tip>
      </Step>

      <Step title="Verifica el consentimiento del administrador de Google">
        En [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), abre tu conexión de Google Workspace. En la pestaña **Setup**, haz una de estas dos cosas:

        * Sigue el enlace **Continuar** si tienes permisos de administrador para configurar los ajustes de Google Workspace y usar las Admin APIs de Google, o

        * Proporciona la URL indicada a tu administrador para que pueda ajustar la configuración requerida
      </Step>

      <Step title="Habilita Directory Sync">
        Habilita Directory Sync con el [endpoint Create a directory provisioning configuration](https://auth0.com/docs/api/management/v2/connections/post-directory-provisioning) (`POST /v2/connections/{id}/directory-provisioning`), o actualiza una configuración existente de Directory Sync con el [endpoint Patch a directory provisioning configuration](https://auth0.com/docs/api/management/v2/connections/patch-directory-provisioning) (`PATCH /v2/connections/{id}/directory-provisioning`), y usa los siguientes parámetros del cuerpo de la solicitud:

        * `mapping` para definir el mapeo de los atributos de Google a los atributos del perfil de usuario de Auth0.

        * `synchronize_automatically` en `true` para sincronizar automáticamente cada 30 minutos, o en `false` para desactivar la sincronización automática.

        * `synchronize_groups` en uno de estos valores:
          * `all` para sincronizar todos los grupos, además de los usuarios
          * `selected` para sincronizar solo los grupos que especifiques
          * `off` para sincronizar solo usuarios

        Si eliges sincronizar solo grupos seleccionados, también debes especificar qué grupos se sincronizarán.

        Primero, [obtén una lista de IDs de grupos con la Google Workspace Directory API](https://developers.google.com/workspace/admin/directory/reference/rest/v1/groups/list). Luego, usa el [endpoint Configure groups to synchronize](https://auth0.com/docs/api/management/v2/connections/put-synchronized-groups) de la Management API (`PUT /v2/connections/{id}/directory-provisioning/synchronized-groups`) con el parámetro del cuerpo de la solicitud que contiene los IDs de los grupos que quieres sincronizar, con el siguiente formato:

        ```json
        "groups" [
            {
                "id": "example-id-string",
            },
            {
                "id": "example-id-string-2",
            },
            {
                "id": "example-id-string-3",
            }
        ]
        ```

        Para activar una sincronización manual, usa el [endpoint Create a directory provisioning configuration](https://auth0.com/docs/api/management/v2/connections/post-synchronizations).
      </Step>
    </Steps>
  </Tab>
</Tabs>

<div id="monitor-directory-sync-activity">
  ## Supervisa la actividad de Directory Sync
</div>

Puedes supervisar la actividad de sincronización en [el registro del tenant de Auth0](/docs/es/deploy-monitor/logs), en los tipos de registro **Directory Sync Started** y **Directory Sync Completed** (códigos de evento `directory_sync_started` y `directory_sync_completed`).

<div id="limits">
  ## Límites
</div>

* Si se sincroniza manualmente dentro de los 30 minutos posteriores a la última sincronización completada, se devuelve un error 400. Espere al menos 30 minutos antes de volver a sincronizar.

* Al usar la [vinculación de cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking), la cuenta de usuario aprovisionada por la sincronización debe configurarse como la cuenta de usuario principal. No recomendamos vincular cuentas de usuario empresariales con cuentas de usuario sociales o personales.

* Los nombres de grupo en Google Workspace Directory deben ser únicos. Aunque Google permite nombres duplicados sin distinguir entre mayúsculas y minúsculas, Auth0 exige nombres para mostrar únicos. Si tiene varios grupos con el mismo nombre en Google Workspace Directory, los grupos duplicados no se sincronizan y la sincronización se marca como fallida.

* Como Google solo informa que los usuarios se han eliminado dentro de los 20 días posteriores a su eliminación, si elimina un usuario en Google Workspace Directory y no sincroniza con Auth0 dentro de esos 20 días, no detectaremos que el usuario ha sido eliminado. Recomendamos la sincronización automática para evitar este problema.
