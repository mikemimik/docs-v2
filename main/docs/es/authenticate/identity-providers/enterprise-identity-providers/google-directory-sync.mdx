Activar Directory Sync en tu [conexión empresarial de Google Workspace](/docs/es/authenticate/identity-providers/enterprise-identity-providers/google-apps) te permite sincronizar los perfiles de usuario, las estructuras de grupos y la pertenencia a los grupos de Google Workspace en Auth0.

Puedes sincronizar automáticamente o manualmente:

* La sincronización automática se realiza cada 30 minutos después de que finaliza la sincronización anterior.

* La sincronización manual se realiza cuando la inicias.

<div id="enable-directory-sync">
  ## Habilitar Directory Sync
</div>

Puede habilitar Directory Sync desde el Auth0 Dashboard o mediante la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    <Steps>
      <Step title="Requisitos previos">
        Antes de comenzar, debe contar con lo siguiente:

        * [Una conexión empresarial de Google Workspace en Auth0](./google-apps)

        * Privilegios de administrador en la organización de Google Workspace.
      </Step>

      <Step title="Habilite la API de Admin Directory para su conexión empresarial">
        Para habilitar Directory Sync, el access token de Google de su conexión empresarial de Google Workspace debe tener los scopes adecuados para acceder a las API de Google.

        En la pestaña **Settings** de una conexión empresarial de Google Workspace existente (o al crear una conexión nueva), en la sección **Identity Provider API**, seleccione **Use Admin Directory API** y elija una de estas opciones:

        * **Users scopes** para agregar scopes solo para acceder a los usuarios.

        * **Users and Groups scopes** para agregar scopes para acceder tanto a usuarios como a grupos.

        <Tip>
          Al usar Directory Sync, también le recomendamos deshabilitar **Sync User Profile Attributes at Login** en esta sección para evitar actualizaciones en conflicto entre varios métodos de sincronización.
        </Tip>

        Haga clic en **Save Changes**.
      </Step>

      <Step title="Verifique el consentimiento del administrador de Google">
        En [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), abra su conexión de Google Workspace. En la pestaña **Setup**, haga una de las siguientes acciones:

        * Siga el enlace **Continuar** si tiene permisos de administrador para configurar los ajustes de Google Workspace a fin de usar las API de administración de Google, o bien

        * Proporcione la URL indicada a su administrador para que pueda ajustar la configuración requerida
      </Step>

      <Step title="Habilite Directory Sync">
        En la pestaña **Aprovisionamiento** de su conexión, active **Provision Users Using Directory Sync** y elija sus opciones de configuración:

        * En **Resources**, en **Sync**, elija si desea sincronizar **Users** o **Users and Groups**.

          Si sincroniza tanto usuarios como grupos, la sección se expande para mostrar qué grupos se están sincronizando (**Syncing all groups** de forma predeterminada). Para personalizar los grupos sincronizados, consulte el siguiente paso.

        * En **Schedule**, opcionalmente marque **Enable Automatic Synchronization** para sincronizar automáticamente cada 30 minutos.

          Puede iniciar una sincronización manual seleccionando **Synchronize now**.

        * En **Attribute Mapping**, puede personalizar el mapeo de atributos de Google a los atributos del perfil de usuario de Auth0.
      </Step>

      <Step title="Personalice los grupos sincronizados (opcional)">
        Cuando habilita Directory Sync para sincronizar tanto usuarios como grupos desde Google Workspace, todos los grupos se sincronizan de forma predeterminada. Puede personalizar qué grupos sincroniza cargando un archivo JSON con los ID de los grupos que desea sincronizar.

        Primero, cree el archivo JSON. Puede [obtener una lista de todos los ID de grupo con la Google Workspace Directory API](https://developers.google.com/workspace/admin/directory/reference/rest/v1/groups/list). Cree el archivo con el siguiente formato y sustituya los valores de ejemplo por los ID de grupo reales:

        ```json
        {
            "groups": [
                {
                    "id": "example-id-1"
                },
                {
                    "id": "example-id-2"
                },
                {
                    "id": "example-id-3"
                }
            ]
        }
        ```

        A continuación, cargue el archivo en Auth0:

        1. En la pestaña **Aprovisionamiento** de su conexión, en la opción activada **Provision Users Using Directory Sync**, busque la sección **Resources** y seleccione el botón **Select Groups...**.

        2. En la ventana **Select Groups** que se abre, seleccione **Pick specific groups** para mostrar la sección **Groups JSON file**.

        3. Seleccione **+ Choose file** y cargue el archivo JSON.

        4. Después de cargar el archivo, seleccione **Select Groups** en la parte inferior de la ventana.

        La sección **Resources** muestra **Syncing specific groups**. Para actualizar qué grupos sincroniza, vuelva a la misma ventana **Select Groups** y cargue un archivo JSON nuevo o elija **Sync all**.

        Si detectamos que se elimina un ID de grupo sincronizado en Google Workspace Directory, eliminamos automáticamente ese ID de grupo de la configuración para las tareas de sincronización posteriores.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Management API">
    <Steps>
      <Step title="Requisitos previos">
        Antes de comenzar, debes tener lo siguiente:

        * [Una conexión empresarial de Google Workspace en Auth0](./google-apps)

        * Privilegios de administrador en la organización de Google Workspace.

        * Un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens#management-api-access-tokens) con los siguientes scopes:
          * `create:directory_provisionings`
          * `read:directory_provisionings`
          * `update:directory_provisionings`
          * `delete:directory_provisionings`

        * El ID de tu conexión de Google Workspace (`CONNECTION_ID`).
      </Step>

      <Step title="Habilita la API del directorio de administración para tu conexión empresarial">
        Al crear una nueva conexión empresarial de Google Workspace mediante el [endpoint Create a connection](https://auth0.com/docs/api/management/v2/connections/post-connections) (`POST /v2/connections`), o al modificar una conexión empresarial existente mediante el [endpoint actualizar una conexión](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) (`PATCH /v2/connections/{id}`), establece los siguientes parámetros del cuerpo de la solicitud:

        * `option.api_enable_users` en `true` para agregar scopes de acceso a usuarios.
        * `option.api_enable_groups` en `true` para agregar también scopes de acceso a grupos. El acceso a grupos requiere acceso a usuarios.

        <Tip>
          Al usar Directory Sync, te recomendamos desactivar la sincronización de perfiles de usuario al iniciar sesión (estableciendo `options.set_user_root_attributes` en `never_on_login`) para evitar actualizaciones en conflicto procedentes de varios métodos de sincronización.
        </Tip>
      </Step>

      <Step title="Verifica el consentimiento del administrador de Google">
        En [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), abre tu conexión de Google Workspace. En la pestaña **Setup**, haz una de estas dos cosas:

        * Sigue el enlace **Continuar** si tienes permisos de administrador para configurar los ajustes de Google Workspace y usar las API de administración de Google, o

        * Proporciona la URL indicada a tu administrador para que pueda ajustar la configuración requerida
      </Step>

      <Step title="Habilita Directory Sync">
        Habilita Directory Sync con el [endpoint Create a directory provisioning configuration](https://auth0.com/docs/api/management/v2/connections/post-directory-provisioning) (`POST /v2/connections/{id}/directory-provisioning`), o actualiza una configuración existente de Directory Sync con el [endpoint Patch a directory provisioning configuration](https://auth0.com/docs/api/management/v2/connections/patch-directory-provisioning) (`PATCH /v2/connections/{id}/directory-provisioning`), y usa los siguientes parámetros del cuerpo de la solicitud:

        * `mapping` para definir el mapeo de atributos de Google a los atributos del perfil de usuario de Auth0.

        * `synchronize_automatically` establecido en `true` para sincronizar automáticamente cada 30 minutos, o en `false` para desactivar la sincronización automática.

        * `synchronize_groups` establecido en uno de los siguientes valores:
          * `all` para sincronizar todos los grupos además de los usuarios
          * `selected` para sincronizar solo los grupos que especifiques
          * `off` para sincronizar solo usuarios

        Si eliges sincronizar solo determinados grupos, también debes especificar qué grupos se sincronizarán.

        Primero, [obtén una lista de IDs de grupo mediante la Google Workspace Directory API](https://developers.google.com/workspace/admin/directory/reference/rest/v1/groups/list). Luego, usa el [endpoint Configure groups to synchronize](https://auth0.com/docs/api/management/v2/connections/put-synchronized-groups) de la Management API (`PUT /v2/connections/{id}/directory-provisioning/synchronized-groups`) con el parámetro del cuerpo de la solicitud que contiene los IDs de los grupos que deseas sincronizar, con el siguiente formato:

        ```json
        "groups" [
            {
                "id": "example-id-string",
            },
            {
                "id": "example-id-string-2",
            },
            {
                "id": "example-id-string-3",
            }
        ]
        ```

        Para iniciar una sincronización manual, usa el [endpoint Create a directory provisioning configuration](https://auth0.com/docs/api/management/v2/connections/post-synchronizations).
      </Step>
    </Steps>
  </Tab>
</Tabs>

<div id="monitor-directory-sync-activity">
  ## Supervisar la actividad de Directory Sync
</div>

Puedes supervisar la actividad de sincronización en [los registros del tenant de Auth0](/docs/es/deploy-monitor/logs), en los tipos de registro **Directory Sync Started** y **Directory Sync Completed** (códigos de evento `directory_sync_started` y `directory_sync_completed`).

<div id="limits">
  ## Límites
</div>

* Si sincroniza manualmente dentro de los 30 minutos posteriores a la última sincronización completada, se devuelve un error 400. Espere al menos 30 minutos antes de volver a sincronizar.

* Al usar la [vinculación de cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking), la cuenta de usuario aprovisionada por la sincronización debe configurarse como la cuenta de usuario principal. No recomendamos vincular cuentas de usuario empresariales con cuentas de usuario sociales y personales.

* Los nombres de grupo en Google Workspace Directory deben ser únicos. Aunque Google permite nombres duplicados sin distinguir entre mayúsculas y minúsculas, Auth0 exige que los nombres para mostrar sean únicos. Si tiene varios grupos con el mismo nombre en Google Workspace Directory, los grupos duplicados no se sincronizan y la sincronización se marca como fallida.

* Como Google solo informa que los usuarios fueron eliminados dentro de los 20 días posteriores a su eliminación, si elimina un usuario en Google Workspace Directory y no sincroniza con Auth0 dentro de esos 20 días, no detectaremos que el usuario fue eliminado. Recomendamos la sincronización automática para evitar esta incidencia.
