description: Aprende a crear una conexión empresarial entre un PingFederate Server y Auth0.
title: Conecta tu PingFederate Server a Auth0
---

Auth0 te permite crear conexiones de [PingFederate Server](https://documentation.pingidentity.com/pingfederate/pf84/#gettingStartedGuide/concept/gettingStarted.html).

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de comenzar:

* [Registra tu aplicación en Auth0](/docs/es/get-started/auth0-overview/create-applications).

  * Selecciona un **Tipo de aplicación** adecuado.
  * Agrega **`{https://yourApp/callback}`** como **URL de devolución de llamada permitida**.
  * Asegúrate de que los [Tipos de concesión](/docs/es/get-started/applications/update-grant-types) de tu aplicación incluyan los flujos adecuados.

<div id="steps">
  ## Pasos
</div>

Siempre que su servidor esté configurado de la forma estándar, para conectar su PingFederate Server con Auth0 debe:

1. [Obtener el certificado de firma del IdP](#get-the-signing-certificate-from-the-idp) y [convertirlo a Base64](#convert-signing-certificate-to-base64).
2. [Crear una conexión empresarial en Auth0](#create-an-enterprise-connection-in-auth0).
3. [Habilitar la conexión empresarial para su aplicación de Auth0](#enable-the-enterprise-connection-for-your-auth0-application).
4. [Probar la conexión](#test-the-connection).

<Warning>
  Si su servidor requiere configuración adicional (como mapeo de atributos), debe [crear una nueva conexión empresarial SAML](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml) en su lugar.
</Warning>

<div id="get-the-signing-certificate-from-the-idp">
  ## Obtén el certificado de firma del IdP
</div>

Con PingFederate Server, Auth0 actúa como el proveedor de servicios, por lo que deberás obtener un certificado de firma X.509 del <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=IdP">IdP</Tooltip> (en formato PEM o CER); más adelante, lo cargarás en Auth0. Los métodos para obtener este certificado pueden variar, así que consulta la [documentación de PingFederate](https://documentation.pingidentity.com/pingfederate/pf84/index.shtml#concept_digitalSignatureSettings.html) para obtener instrucciones sobre cómo administrar los certificados de tu servidor.

<div id="convert-signing-certificate-to-base64">
  ### Convertir el X.509 Signing Certificate a Base64
</div>

Antes de cargar el X.509 Signing Certificate en Auth0, debes convertir el archivo a Base64. Para ello, usa una [herramienta en línea sencilla](https://www.base64decode.org/) o ejecuta el siguiente comando en Bash: `cat signing-cert.crt | base64`.

<div id="assertion-encryption">
  ## Cifrado de aserciones
</div>

Si sus aserciones SAML están cifradas, debe [configurar valores adicionales](/docs/es/authenticate/protocols/saml/saml-sso-integrations/algorithm-profiles) para su conexión a fin de indicarle a Auth0 cómo realizar el descifrado.

<div id="create-an-enterprise-connection-in-auth0">
  ## Crear una conexión empresarial en Auth0
</div>

A continuación, si su servidor está configurado de la manera estándar, deberá crear y configurar una conexión empresarial de PingFederate en Auth0 y cargar su certificado de firma X.509. Esta tarea puede realizarse desde Auth0 Dashboard.

<Warning>
  Si su servidor requiere una configuración adicional (como mapeo de atributos), entonces debe [crear una nueva conexión empresarial SAML](/docs/es/authenticate/identity-providers/enterprise-identity-providers/saml) en su lugar.
</Warning>

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), busque **Ping Federate** y seleccione su `+`.

   <Frame>![Dashboard - Connections - Enterprise](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Introduzca los datos de su conexión y seleccione **Create:**

| Campo                                          | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Connection name**                            | Identificador lógico de su conexión; debe ser único en su tenant. Una vez establecido, este nombre no se puede cambiar.                                                                                                                                                                                                                                                                                                                                                                                      |
| **PingFederate Server URL**                    | URL de su PingFederate Server.                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **X.509 Signing Certificate**                  | Clave pública del PingFederate Server (codificada en PEM o CER) que recuperó anteriormente del IdP durante este proceso.                                                                                                                                                                                                                                                                                                                                                                                     |
| **Sign Request**                               | Cuando está habilitado, se firmará la solicitud de autenticación SAML. (Asegúrese de descargar y proporcionar al servidor PingFederate el [certificado de su tenant](https://\{yourDomain}/pem).) <Callout icon="file-lines" color="#0EA5E9" iconType="regular">Si sus aserciones SAML están cifradas, debe [establecer valores adicionales](/docs/es/authenticate/protocols/saml/saml-sso-integrations/algorithm-profiles) para su conexión a fin de indicarle a Auth0 cómo gestionar el descifrado.</Callout> |
| **Sign Request Algorithm**                     | Algoritmo que Auth0 utilizará para firmar las aserciones SAML. Asegúrese de que coincida con la configuración de su PingFederate Server.                                                                                                                                                                                                                                                                                                                                                                     |
| **Sign Request Digest Algorithm**              | Algoritmo que Auth0 utilizará para el resumen de la solicitud firmada. Asegúrese de que coincida con la configuración de su PingFederate Server.                                                                                                                                                                                                                                                                                                                                                             |
| **Sync user profile attributes at each login** | Cuando está habilitado, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada inicio de sesión, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0.                                                                                                                                                                                                                                                                           |

<Frame>![Configure Ping Federate Settings](/docs/images/cdy7uua7fh8z/6gBIIx0DC8363S1NNGBCqA/5e58d3fbde75661eb170d8bd0cbcda61/PingFederate.png)</Frame>
3. En la vista **Login Experience**, configure cómo inician sesión los usuarios con esta conexión.

| Campo                              | Descripción                                                                                                                                                                                                                                                              |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Home Realm Discovery**           | Compara el dominio del correo electrónico de un usuario con los dominios del proveedor de identidad proporcionados. Para obtener más información, consulte [Configurar Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first) |
| **Display connection button**      | Esta opción muestra las siguientes opciones para personalizar el botón de conexión de su aplicación.                                                                                                                                                                     |
| **Button display name** (Optional) | Texto utilizado para personalizar el botón de inicio de sesión de Universal Login. Cuando se establece, el botón muestra: &quot;Continuar con &#123;Button display name&#125;&quot;.                                                                                     |
| **Button logo URL** (Optional)     | URL de la imagen utilizada para personalizar el botón de inicio de sesión de Universal Login. Cuando se establece, el botón de inicio de sesión de Universal Login muestra la imagen como un cuadrado de 20 px por 20 px.                                                |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los campos opcionales están disponibles solo con Universal Login. Los clientes que usan Classic Login no verán el botón Add, Button display name ni Button logo URL.
</Callout>

<div id="enable-the-enterprise-connection-for-your-auth0-application">
  ## Habilita la conexión empresarial para tu aplicación de Auth0
</div>

Para usar tu nueva conexión empresarial de PingFederate, primero debes [habilitar la conexión](/docs/es/authenticate/identity-providers/enterprise-identity-providers/enable-enterprise-connections) en tus aplicaciones de Auth0.

  ## Prueba la conexión
</div>
