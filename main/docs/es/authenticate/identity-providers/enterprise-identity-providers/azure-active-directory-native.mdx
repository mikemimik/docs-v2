title: Conecte su aplicación nativa a Microsoft Azure Active Directory mediante Resource
---

Además de los flujos de WS-Federation y <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect, también es posible usar el flujo de [Resource Owner](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow) con Azure AD. Este flujo le permite capturar y validar las credenciales de un usuario (correo electrónico y contraseña) en lugar de mostrar la página de inicio de sesión de Azure AD. Por motivos de seguridad y de <Tooltip tip="inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente su sesión en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SSO), este no es el enfoque recomendado; aun así, el flujo de <Tooltip tip="Resource Owner: Entidad (como un usuario o una aplicación) capaz de conceder acceso a un recurso protegido." cta="Ver glosario" href="/docs/es/glossary?term=Resource+Owner">Resource Owner</Tooltip> puede ser útil en escenarios de aplicaciones móviles nativas o para procesar autenticaciones por lotes con Azure AD.

Esta configuración requiere dos aplicaciones: una aplicación web y/o una Web API, y una aplicación nativa. Desde el punto de vista de Azure AD, los usuarios se autenticarán mediante la aplicación nativa para obtener acceso a la aplicación web y/o la Web API.

<Frame>![Diagrama general de las aplicaciones de Azure AD que usan el flujo de Resource Owner](/docs/images/cdy7uua7fh8z/6Uy9Q35rn0d9rankvlJuBT/6a5fa5175db76fd1cab35b79f0a213ea/azure-ad-native-app.png)</Frame>

<div id="prerequisites">
  ## Requisitos previos
</div>

[Registra tu aplicación con Auth0](/docs/es/get-started/auth0-overview/create-applications):

1. Selecciona **Native** en **Tipo de aplicación**.
2. Agrega una **Allowed Callback URL**. El formato de la URL de devolución de llamada variará según tu plataforma. Para obtener más información sobre el formato correspondiente a tu plataforma, consulta nuestros [Quickstarts nativos](/docs/es/quickstart/native).
3. Asegúrate de que los [tipos de concesión](/docs/es/get-started/applications/update-grant-types) de tu aplicación incluyan los flujos adecuados.

<div id="steps">
  ## Pasos
</div>

Para conectar tu aplicación mediante el flujo de Resource Owner, debes:

1. [Configurar tus aplicaciones en el portal de Microsoft Azure](#set-up-your-applications-in-the-microsoft-azure-portal).
2. [Crear una conexión empresarial en Auth0](#create-an-enterprise-connection-in-auth0).
3. [Habilitar la conexión empresarial para tu aplicación de Auth0](#enable-the-enterprise-connection-for-your-auth0-application).
4. [Probar la conexión](#test-the-connection).

<Card title="Cuenta de Microsoft Azure">
  Antes de continuar, necesitarás una cuenta válida de Microsoft Azure y ser administrador global de **tu propio** directorio de Azure AD en Microsoft.

  Si no tienes una cuenta de Microsoft Azure, puedes [registrarte gratis](https://azure.microsoft.com/en-us/free); después, si es necesario, configura un directorio de Azure AD siguiendo la guía de Microsoft [Quickstart: Create a new tenant in Azure Active Directory - Create a new tenant for your organization](https://docs.microsoft.com/en-us/azure/active-directory/fundamentals/active-directory-access-create-new-tenant#create-a-new-tenant-for-your-organization).

  Como alternativa, si tienes una cuenta de Office 365, puedes usar la instancia de Azure AD asociada a esa cuenta en lugar de crear una nueva. Para acceder a la instancia de Azure AD de tu cuenta de Office 365:

  1. [Inicia sesión en Office 365](https://portal.office.com) y ve al [Centro de administración de Office 365](https://portal.office.com/adminportal/home#/homepage).
  2. En el menú de la izquierda, abre **Centros de administración** y haz clic en **Azure AD**.
</Card>

  ## Configura tus aplicaciones en el portal de Microsoft Azure

<div id="register-a-new-web-application">
  ### Registrar una nueva aplicación web
</div>

Para obtener información sobre cómo registrar tu aplicación en Azure AD, consulta [Quickstart: Register an application with the Microsoft identity platform on Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-register-app).

<Warning>
  Si tienes más de un directorio de Azure AD, asegúrate de estar en el directorio correcto al registrar tu aplicación.
</Warning>

Mientras configuras tu aplicación, asegúrate de usar la siguiente configuración:

* Si quieres permitir usuarios de organizaciones externas (como otros directorios de Azure AD), selecciona **Accounts in any organizational directory (Any Azure AD directory - Multitenant)** al configurar **Supported account types**.
* Cuando se te pida configurar un **Redirect URI**, selecciona **Web** e introduce tu URL de devolución de llamada: `https://{yourDomain}/login/callback`.

<Card title="Encuentra tu nombre de dominio de Auth0 para las redirecciones">
  Si tu nombre de dominio de Auth0 no aparece arriba y no estás usando nuestra funcionalidad de [dominios personalizados](/docs/es/customize/custom-domains), tu nombre de dominio es una concatenación del nombre de tu tenant, tu subdominio regional y `auth0.com`, separados por un punto (`.`).

  Por ejemplo, si el nombre de tu tenant es `exampleco-enterprises` y tu tenant está en la región de EE. UU., tu nombre de dominio de Auth0 sería `exampleco-enterprises.us.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.us.auth0.com/login/callback`.

  Sin embargo, si tu tenant está en la región de EE. UU. y se creó antes de junio de 2020, tu nombre de dominio de Auth0 sería `exampleco-enterprises.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.auth0.com/login/callback`.

  Si estás usando [dominios personalizados](/docs/es/customize/custom-domains), tu **Redirect URI** sería `https://<YOUR CUSTOM DOMAIN>/login/callback`.
</Card>

Durante este proceso, anota el **ID de la aplicación (cliente)** que Microsoft genera para tu aplicación (puedes encontrarlo en la pantalla **Información general** de la aplicación), ya que lo necesitarás más adelante en el tutorial.

<div id="configure-your-web-application-to-expose-an-api">
  ### Configure su aplicación web para exponer una API
</div>

Para obtener información sobre cómo configurar su aplicación web para exponer una API con Azure AD, consulte [Quickstart: Configure an application to expose web APIs on Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-configure-app-expose-web-apis).

Mientras configura la aplicación, asegúrese de usar la siguiente configuración:

* Cuando se le pida establecer un **nombre de scope**, introduzca `API.Access`.

Durante este proceso, anote el **Application ID URI** que Microsoft genera para su aplicación, ya que lo necesitará más adelante en el tutorial.

<div id="register-a-new-native-application">
  ### Registrar una nueva aplicación nativa
</div>

Para obtener información sobre cómo registrar una aplicación nativa en Azure, consulta [Quickstart: Register an application with the Microsoft identity platform on Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-register-app).

<Warning>
  Si tienes más de un directorio de Azure AD, asegúrate de estar en el directorio correcto al registrar tu aplicación.
</Warning>

Mientras configuras tu aplicación, asegúrate de usar la siguiente configuración:

* Si quieres permitir usuarios de organizaciones externas (como otros directorios de Azure AD), selecciona **Accounts in any organizational directory (Any Azure AD directory - Multitenant)** al configurar **Supported account types**.
* Cuando se te pida configurar un **Redirect URI**, selecciona `Public client/native (mobile & desktop)` e introduce tu URL de devolución de llamada. El formato de tu URL de devolución de llamada variará según la plataforma. Para obtener más información sobre el formato correspondiente a tu plataforma, consulta [Quickstarts nativos](/docs/es/quickstart/native).

Durante este proceso, anota el **ID de la aplicación (cliente)** que Microsoft genera para tu aplicación (puedes encontrarlo en la pantalla **Información general** de la aplicación), ya que lo necesitarás más adelante en el tutorial.

<div id="create-a-client-secret-for-your-native-application">
  ### Crea un secreto de cliente para tu aplicación nativa
</div>

Para saber cómo crear un <Tooltip tip="Client Secret: Secret usado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>, consulta [Quickstart: Configure a client application to access web APIs - Add Credentials to your web application on Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-configure-app-access-web-apis#add-credentials-to-your-web-application).

Genera un **secreto de cliente** y anota su valor. Lo necesitarás más adelante en el tutorial.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si configuras un secreto con vencimiento, asegúrate de **anotar la fecha de vencimiento**; tendrás que renovar la clave antes de ese día para evitar una interrupción del servicio.
</Callout>

<div id="add-permissions-for-your-native-application">
  ### Agrega permisos para tu aplicación nativa
</div>

Para saber cómo agregar permisos para tu aplicación **Native**, consulta [Quickstart: Configurar una aplicación cliente para acceder a API web - Agregar permisos para acceder a API web en Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-configure-app-access-web-apis#add-permissions-to-access-web-apis).

Auth0 requiere que habilites los permisos mínimos necesarios (para Microsoft Graph API y para la aplicación web que configuraste para exponer una API) para que esta configuración funcione correctamente. Para obtener más información sobre los permisos de Microsoft Graph API, consulta [Referencia de permisos de Microsoft Graph en Microsoft Docs](https://learn.microsoft.com/en-us/graph/permissions-reference).

Mientras configuras tus permisos, asegúrate de usar la siguiente configuración para Microsoft Graph API:

* Cuando se te pida un tipo de permiso, elige **Delegated permissions**.

  * En **User**, selecciona **User.Read** para que tu aplicación pueda iniciar sesión a los usuarios y leer el perfil del usuario que haya iniciado sesión.
  * En **Directory**, selecciona **Directory.Read.All** para que tu aplicación pueda leer datos del directorio en nombre del usuario que haya iniciado sesión.

Para tu aplicación web que configuraste para exponer una API, asegúrate de usar la siguiente configuración:

* Cuando se te pida un tipo de permiso, elige **Delegated permissions**.

  * En **API**, selecciona **API.Access** para que tu aplicación pueda acceder a tu API en nombre del usuario.

<div id="configure-the-connection-in-auth0">
  ## Configure la conexión en Auth0
</div>

Después de crear ambas aplicaciones en Azure AD, puede configurar la conexión de Auth0:

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), busque **Microsoft Azure AD** y seleccione el botón de agregar (**+**).

   <Frame>![Dashboard - Conexiones - empresarial](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Ingrese los detalles de su conexión y seleccione **Create**:

| Campo                                                                    | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Nombre de la conexión**                                                | Identificador lógico de su conexión; debe ser único para su tenant. Una vez establecido, este nombre no se puede cambiar.                                                                                                                                                                                                                                                                                                                                                                   |
| **Dominio de Microsoft Azure AD**                                        | Su nombre de dominio de Azure AD. Puede encontrarlo en la página de Información general de su directorio de Azure AD en el Microsoft Azure portal.                                                                                                                                                                                                                                                                                                                                          |
| **ID de cliente**                                                        | Identificador único de su aplicación registrada en Azure AD. Ingrese el valor guardado de **ID de la aplicación (cliente)** de la aplicación Native que registró en Azure AD.                                                                                                                                                                                                                                                                                                                     |
| **Secreto de cliente**                                                   | Cadena utilizada para obtener acceso a su aplicación registrada en Azure AD. Ingrese el valor guardado de **secreto de cliente** de la aplicación nativa que registró en Azure AD.                                                                                                                                                                                                                                                                                                               |
| **Use common endpoint** (opcional)                                       | Cuando está habilitada, su aplicación aceptará dinámicamente usuarios de nuevos directorios. Normalmente se habilita si seleccionó una opción multitenant para **Supported account types** en la aplicación que registró en Azure AD. Cuando está habilitada, Auth0 redirigirá a los usuarios al endpoint común de inicio de sesión de Azure, y Azure realizará *Home Realm Discovery* en función del dominio de la dirección de correo electrónico del usuario.                            |
| **API de identidad**                                                     | API que Auth0 utiliza para interactuar con los endpoints de Azure AD. Consulte las diferencias de comportamiento en la documentación de Microsoft [Why update to Microsoft identity platform (v2.0)](https://docs.microsoft.com/en-us/azure/active-directory/develop/azure-ad-endpoint-comparison). Seleccione `Azure Active Directory (v1)` y, en **App ID URI**, ingrese el valor guardado de **Application ID URI** que se creó cuando configuró su aplicación web para exponer una API. |
| **Atributos**                                                            | Atributos básicos del usuario que inició sesión a los que su aplicación puede acceder. Indica cuánta información desea almacenar en el perfil de usuario de Auth0.                                                                                                                                                                                                                                                                                                                          |
| **Atributos extendidos** (opcional)                                      | Atributos extendidos del usuario que inició sesión a los que su aplicación puede acceder.                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Auth0 APIs** (opcional)                                                | Cuando se selecciona, indica que necesita poder realizar llamadas a la API de usuarios de Azure AD.                                                                                                                                                                                                                                                                                                                                                                                         |
| **Sincronizar atributos del perfil de usuario en cada inicio de sesión** | Cuando está habilitada, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada inicio de sesión, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0.                                                                                                                                                                                                                                                          |
| **Verificación de correo electrónico**                                   | Elija cómo Auth0 establece el campo `email_verified` en el perfil de usuario. Para obtener más información, consulte [Email Verification for Azure AD and ADFS](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azuread-adfs-email-verification).                                                                                                                                                                                                                       |

<div id="group-memberships-and-advanced-profile-information">
  ## Membresías de grupos e información avanzada del perfil
</div>

En este flujo nativo, Auth0 recibirá un <Tooltip tip="Access Token: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> de Azure AD que se ha emitido para tu aplicación web de Azure AD. Como resultado, funcionalidades como cargar las membresías de grupos y la información avanzada del perfil dejarán de funcionar, ya que el Access Token emitido por Azure AD ya no se puede usar para consultar la Graph API de Azure AD y obtener esta información adicional.

Sin embargo, si dependes de las membresías de grupos y de la información avanzada del perfil, puedes cambiar tu configuración para ajustarla a tus necesidades:

1. En Azure, configura tu aplicación nativa con permisos adicionales para Microsoft Graph API:

   * Cuando se te pida el tipo de permiso, elige **Delegated permissions**.

     * En **Directory**, selecciona **Directory.AccessAsUser.All** para que tu aplicación pueda acceder al directorio como el usuario que ha iniciado sesión.
2. En Auth0, modifica tu conexión empresarial de Azure AD:

   * En **API de identidad**, selecciona `Azure Active Directory (v1)` y, en **App ID URI**, introduce el URI de la Azure AD Graph API:

     ```text lines
     https://graph.windows.net
     ```
