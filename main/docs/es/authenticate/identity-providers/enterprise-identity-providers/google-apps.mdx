Conectar tu aplicación de Auth0 con Google Workspace permite a tus usuarios iniciar sesión con sus credenciales de Google. Para ello, primero debes registrar la aplicación en Google y luego configurar una conexión empresarial de Google Workspace en Auth0.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de comenzar, necesitas:

* [Haber registrado tu aplicación en Auth0](/docs/es/get-started/auth0-overview/create-applications).

  * Selecciona un **Tipo de aplicación** adecuado.
  * Agrega una **URL de devolución de llamada permitida** de `{https://yourApp/callback}`.
  * Asegúrate de que los [Tipos de concesión](/docs/es/get-started/applications/update-grant-types) de tu aplicación incluyan los flujos adecuados.

* Una cuenta válida de Google Workspace y tu propia organización de Google Workspace.

* Privilegios de administrador en la organización de Google Workspace.

<div id="register-your-application-with-google">
  ## Registra tu aplicación en Google
</div>

Para permitir que los usuarios inicien sesión con Google Workspace, debes registrar tu aplicación en la consola de Google Cloud.

<Warning>
  Si tu aplicación solicita scopes sensibles de OAuth, puede estar [sujeta a revisión por parte de Google](https://developers.google.com/apps-script/guides/client-verification).
</Warning>

Sigue la documentación de Google sobre [cómo configurar OAuth 2.0](https://support.google.com/googleapi/answer/6158849) y usa la siguiente configuración:

* En la **pantalla de consentimiento de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip>**, en **Dominios autorizados**, agrega `auth0.com`.

* Cuando te pidan seleccionar un tipo de aplicación, elige **Aplicación web** y establece los siguientes parámetros:

  | Campo                              | Descripción                           |
  | ---------------------------------- | ------------------------------------- |
  | Nombre                             | El nombre de tu aplicación.           |
  | Orígenes autorizados de Javascript | `https://{yourDomain}`                |
  | URI de redirección autorizados     | `https://{yourDomain}/login/callback` |

  A menos que estés usando un [dominio personalizado](/docs/es/customize/custom-domains), el nombre de dominio de tu tenant es `<TENANT NAME>.<REGIONAL SUBDOMAIN>.auth0.com`. Por ejemplo, el nombre de dominio de un tenant de la región de EE. UU. llamado `example-name` es `example-name.us.auth0.com`). Los tenants creados antes de junio de 2020 omiten el subdominio regional.

* Si planeas conectarte a dominios empresariales de Google Workspace, debes [habilitar la **Admin SDK API**](https://support.google.com/googleapi/answer/6158841).

Guarda el **<Tooltip tip="ID de cliente: Valor de identificación dado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** y el **<Tooltip tip="ID de cliente: Valor de identificación dado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>** que Google genera para tu aplicación. Necesitarás estos valores para crear una conexión empresarial de Auth0 en el siguiente paso.

<div id="configure-a-google-workspace-enterprise-connection-in-auth0">
  ## Configurar una conexión empresarial de Google Workspace en Auth0
</div>

<Steps>
  <Step title="Crear una conexión empresarial de Google Workspace">
    En [**Auth0 Dashboard &gt; Autenticación &gt; Empresarial**](https://manage.auth0.com/#/connections/enterprise), busca **Google Workspace** y selecciona **Create**.

    <Frame>![Dashboard - Conexiones - Empresarial](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
  </Step>

  <Step title="Elige la configuración de tu conexión">
    En la página de creación de una nueva conexión empresarial de Google Workspace, configura estos ajustes:

    | Field                                  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
    | -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | **Purpose**                            | El uso de la conexión: inicio de sesión, cuentas conectadas o ambos. Para obtener más información, consulta [Autenticación de usuarios frente a cuentas conectadas](/docs/es/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).                                                                                                                                                                                                                                                                                                         |
    | **Connection name**                    | Un identificador lógico para tu conexión. Debe ser único en tu tenant y no se puede cambiar.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
    | **Google Workspace Domain**            | El nombre de dominio de Google Workspace de tu organización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
    | **Client ID**                          | Un identificador único para tu aplicación de Google registrada. Introduce el ID de cliente de la aplicación que registraste en la consola de Google Cloud.                                                                                                                                                                                                                                                                                                                                                                                                                                 |
    | **Client Secret**                      | Una cadena que se utiliza para acceder a tu aplicación de Google registrada. Introduce el secreto de cliente de la aplicación que registraste en la consola de Google Cloud.                                                                                                                                                                                                                                                                                                                                                                                                               |
    | **Attributes**                         | Los atributos del usuario que ha iniciado sesión a los que tu aplicación puede acceder y que Auth0 almacena en el perfil de usuario: <ul><li>**Basic Profile**: `email`, indicador `email verified`</li> <li>**Extended Profile**: nombre, URL del perfil público, foto, género, fecha de nacimiento, país, idioma y zona horaria</li></ul>                                                                                                                                                                                                                                                |
    | **Extended Attributes** (optional)     | Atributos adicionales del usuario que ha iniciado sesión a los que tu aplicación puede acceder y que Auth0 almacena en el perfil de usuario: <ul><li>**Groups**: las listas de distribución del usuario, ya sea en formato simplificado (nombres de grupo) o en formato ampliado (objetos de grupo con `name`, `id` y `email`)</li> <li>**Is Domain Administrator**: si el usuario es administrador del dominio</li> <li>**Is Account Suspended**: si la cuenta del usuario está suspendida</li> <li>**Agreed to Terms**: si el usuario ha aceptado las condiciones del servicio</li></ul> |
    | **Use Admin Directory API** (optional) | Al habilitar esta configuración, la conexión podrá hacer llamadas a la API de Google Directory. Elige los scopes **Users** o **Users and Groups**, según sea necesario.                                                                                                                                                                                                                                                                                                                                                                                                                    |
    | **Auth0 User ID** (optional)           | De forma predeterminada, el `user_id` de Auth0 se asigna a `email`. Al habilitar **Use ID instead of Email for Auth0 User ID**, `user_id` pasará a asignarse a `id`. Solo puedes establecer esto para conexiones nuevas y no podrás cambiarlo una vez configurado.                                                                                                                                                                                                                                                                                                                         |

    Cuando termines de elegir la configuración, haz clic en **Create**.
  </Step>

  <Step title="Configura cómo los usuarios inician sesión">
    En la pestaña **Login Experience**, configure cómo los usuarios inician sesión con esta conexión:

    | Field                    | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
    | ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Home Realm Discovery** | Compara el dominio del correo electrónico del usuario con los **dominios del proveedor de identidad** proporcionados. Para obtener más información, consulte [Configure Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first)                                                                                                                                                                                                     |
    | **Connection button**    | Seleccione **Display connection as a button** para habilitar un botón para esta conexión en la página de inicio de sesión de la aplicación. Al hacerlo, podrá personalizar dos opciones: <ul><li>**Button display name** (opcional): el texto que se muestra en el botón después de &quot;Continue with&quot;.</li><li>**Button logo URL** (opcional): la URL de una imagen que se mostrará como un cuadrado de 20 px por 20 px a la izquierda del texto del botón.</li></ul> |

    Después de introducir la configuración, haga clic en **Save**.
  </Step>

  <Step title="Verificar el consentimiento del administrador de Google">
    En la pestaña **Setup**, haga una de las siguientes acciones:

    * Siga el enlace **Continue** si tiene permisos de administrador para configurar Google Workspace y usar las API de administración de Google, o

    * Proporcione la URL indicada a su administrador para que ajuste la configuración necesaria

    <Warning>
      Si se elimina un administrador de Google Workspace, cualquier conexión empresarial de Google Workspace que haya autorizado deberá volver a ser autorizada por otro administrador de Google Workspace para evitar errores de inicio de sesión. Para volver a autorizarla, pida a un administrador actual que use el enlace de la pestaña **Setup** de la conexión empresarial de Google Workspace.
    </Warning>
  </Step>

  <Step title="Habilitar y probar la conexión">
    [Habilita la conexión](/docs/es/authenticate/identity-providers/enterprise-identity-providers/enable-enterprise-connections) para tus aplicaciones de Auth0 y luego [prueba la conexión](/docs/es/authenticate/identity-providers/enterprise-identity-providers/test-enterprise-connections) para comprobar que funciona.
  </Step>
</Steps>

<div id="customize-or-automate-user-profile-synchronization">
  ### Personaliza o automatiza la sincronización del perfil del usuario
</div>

En la pestaña **Provisioning** de la conexión, puedes configurar cómo se actualizan los perfiles de usuario:

* Habilita **Sync User Profile Attributes at Login** para permitir que Google sincronice los perfiles de usuario cuando el usuario inicie sesión.

  Una vez habilitada, puedes elegir la **Sync Frequency**. De forma predeterminada, Auth0 actualiza los perfiles de usuario en cada inicio de sesión. También puedes optar por sincronizarlos solo en el primer inicio de sesión.

* Habilita **Provision Users and Groups Using Directory Sync** para sincronizar automáticamente usuarios y grupos de Google Workspace con Auth0. Para obtener más información, consulta [Sync Google Workspace Users to Auth0 with Google Directory Sync](/docs/es/authenticate/identity-providers/enterprise-identity-providers/google-directory-sync).

<div id="get-access-and-refresh-tokens">
  ### Obtener tokens de acceso y de actualización
</div>

Después de que un usuario se autentica, Google emite un token de acceso. Puedes [usar este token de acceso](/docs/es/secure/tokens/access-tokens/identity-provider-access-tokens) para llamar a las APIs de Google.

Para solicitar además un <Tooltip tip="token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token">token de actualización</Tooltip> de Google, agrega `access_type=offline&approval_prompt=force` a la solicitud de autorización.

En Google Workspace, Auth0 almacena los tokens de acceso y de actualización en el objeto `user` para los usuarios individuales y en el objeto `connection` para los administradores del espacio de trabajo. Puedes acceder a ellos mediante el [endpoint obtener una conexión de la Management API](https://auth0.com/docs/api/management/v2/connections/get-connections-by-id).

<div id="retrieve-additional-user-and-group-details">
  ### Recuperar detalles adicionales de usuarios y grupos
</div>

Para recuperar detalles adicionales de usuarios y grupos, además de los que Auth0 obtiene de forma predeterminada, puedes usar la API Directory Admin de Google para obtener las propiedades ampliadas de los grupos de un usuario.

1. En la pestaña **Settings** de tu conexión empresarial, en la sección **General**, en **Extended Attributes**, habilita **Groups**.

2. En la pestaña **Setup**, completa el consentimiento del administrador mediante el enlace proporcionado.

3. Recupera el token de acceso de administrador almacenado en el objeto `connection` mediante el endpoint [obtener una conexión](https://auth0.com/docs/api/management/v2/connections/get-connections-by-id) de la Management API.

4. Usa el token de acceso para llamar al [endpoint Manage Groups de Google](https://developers.google.com/admin-sdk/directory/v1/guides/manage-groups#get_all_member_groups). Por ejemplo:

   ```bash wrap
   curl -H "Authorization: OAuth {admin_access_token}" \
       https://admin.googleapis.com/admin/directory/v1/groups\?userKey={user_key}
   ```

<div id="validate-authentication-with-actions">
  ### Validar la autenticación con Actions
</div>

Comprobar que los eventos de autenticación provienen de usuarios legítimos que pertenecen a su organización de Google puede ayudar a evitar el acceso no autorizado a sus aplicaciones por parte de usuarios que ya no forman parte de ella.

Para verificar las autenticaciones con Google, puede [usar una Action Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) para validar que el claim `idp_tenant_domain` asociado al usuario coincida con la organización esperada. Por ejemplo:

```js Example post-login Action code to validate Google authentications
exports.onExecutePostLogin = async (event, api) => {
    if (event.connection.strategy === 'google-oauth2') {
        // Bloquear cuentas sociales (que no pertenecen a la organización)
        if (!event.user.idp_tenant_domain) {
            api.access.deny('Social accounts are not allowed to log in');
        }

        // Solo permitir cuentas de organizaciones en la lista de permitidas
        if (event.user.idp_tenant_domain != 'example.com') {
            api.access.deny('Your Google Organization is not allowed to log in');
        }
    }
};
```

Solo puedes verificar `idp_tenant_domain` para usuarios que se autentican con los siguientes tipos de conexión:

* Google Social
* Google Workspace
* Google OIDC

<div id="limits">
  ## Límites
</div>

* Si ya tienes una [conexión social de Google](https://marketplace.auth0.com/integrations/google-social-connection) para tu aplicación y creas una nueva conexión empresarial de Google Workspace para el mismo dominio, los usuarios asociados a la conexión social iniciarán sesión con la conexión empresarial (aunque no la habilites).

* La conexión empresarial de Google Workspace permite que los perfiles de usuario de Auth0 tengan hasta 200 grupos. Si un perfil de usuario tiene más de 200 grupos, es posible que no se muestren. Si utilizas grupos de Google para la autorización, debes proteger el espacio de trabajo para evitar que usuarios no autorizados modifiquen los grupos.

* El botón de conexión no está disponible para Classic Login. Para habilitarlo y personalizarlo, usa Universal Login.
