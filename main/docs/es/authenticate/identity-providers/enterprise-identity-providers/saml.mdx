Auth0 te permite crear conexiones de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> (IdP).

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de comenzar:

* [Registra tu aplicación en Auth0](/docs/es/get-started/auth0-overview/create-applications).

  * Selecciona un **Tipo de aplicación** adecuado.
  * Agrega una **Allowed Callback URL** de **`{https://yourApp/callback}`**.
  * Asegúrate de que los [Tipos de concesión](/docs/es/get-started/applications/update-grant-types) de tu aplicación incluyan los flujos adecuados.
* Decide el nombre de esta conexión empresarial

  * La URL de retorno (también llamada URL del Servicio de Consumo de Aserciones) pasará a ser: `https://{yourDomain}/login/callback?connection={yourConnectionName}`
  * El ID de entidad pasará a ser: `urn:auth0:{yourTenant}:{yourConnectionName}`

<div id="steps">
  ## Pasos
</div>

Para conectar su aplicación a un proveedor de identidad SAML, debe:

1. Introducir la URL de retorno y el ID de entidad en el IdP (para saber cómo hacerlo, consulte [la configuración de un proveedor de identidad SAML](/docs/es/authenticate/protocols/saml/saml-identity-provider-configuration-settings)).
2. [Obtener el certificado de firma del IdP](#get-the-signing-certificate-from-the-idp) y [convertirlo a Base64](#convert-signing-certificate-to-base64).
3. [Crear una conexión empresarial en Auth0](#create-an-enterprise-connection-in-auth0).
4. [Habilitar la conexión empresarial para su aplicación de Auth0](#enable-the-enterprise-connection-for-your-auth0-application).
5. [Configurar los mapeos](#set-up-mappings) (innecesario en la mayoría de los casos).
6. [Probar la conexión](#test-the-connection).

<div id="get-the-signing-certificate-from-the-idp">
  ## Obtener el certificado de firma del IdP
</div>

Con SAML Login, Auth0 actúa como proveedor de servicios, por lo que deberá obtener un certificado de firma X.509 del IdP de SAML (en formato PEM o CER); más adelante, lo cargará en Auth0. Los métodos para obtener este certificado varían, así que consulte la documentación de su IdP si necesita ayuda adicional.

<div id="convert-signing-certificate-to-base64">
  ### Convertir el certificado de firma a Base64
</div>

Puedes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> o el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para cargar el certificado de firma X.509. Si usas la Management API, debes convertir el archivo a Base64. Para ello, usa una [herramienta en línea simple](https://www.base64decode.org/) o ejecuta el siguiente comando en Bash: `cat signing-cert.crt | base64`.

<div id="assertion-encryption">
  ## Cifrado de aserciones
</div>

Si las aserciones SAML están cifradas, debes [configurar valores adicionales](/docs/es/authenticate/protocols/saml/saml-sso-integrations/algorithm-profiles) en tu conexión para indicarle a Auth0 cómo realizar el descifrado.

<div id="create-an-enterprise-connection-in-auth0">
  ## Crear una conexión empresarial en Auth0
</div>

A continuación, deberás crear y configurar una conexión empresarial SAML en Auth0 y subir tu certificado de firma X.509. Esta tarea puede realizarse desde el Dashboard de Auth0 o con la Management API.

<div id="create-an-enterprise-connection-using-the-dashboard">
  ### Crear una conexión empresarial con el Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), busca **SAML** y selecciona su `+`.

   <Frame>![Dashboard - Connections - Enterprise](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Introduce los datos de tu conexión y selecciona **Create:**

| Campo                             | Descripción                                                                                                                                                                                                                |
| --------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Connection name**               | Identificador lógico de tu conexión; debe ser único en tu tenant y debe ser el mismo nombre que se usa al configurar la URL de retorno y el ID de entidad en el IdP. Una vez establecido, este nombre no se puede cambiar. |
| **Sign In URL**                   | URL de inicio de sesión único de SAML.                                                                                                                                                                                     |
| **X.509 Signing Certificate**     | Certificado de firma (codificado en PEM o CER) que obtuviste anteriormente del IdP durante este proceso.                                                                                                                   |
| **Enable Sign Out**               | Cuando está habilitado, se puede configurar una Sign Out URL específica. De lo contrario, se usa la Sign In URL de forma predeterminada.                                                                                   |
| **Sign Out URL** (opcional)       | URL de cierre de sesión único de SAML.                                                                                                                                                                                     |
| **User ID Attribute** (opcional)  | Atributo del token SAML que se asignará a la propiedad `user_id` en Auth0.                                                                                                                                                 |
| **Debug Mode**                    | Cuando está habilitado, se generarán registros más detallados durante el proceso de autenticación.                                                                                                                         |
| **Sign Request**                  | Cuando está habilitado, la solicitud de autenticación SAML se firmará. (Asegúrate de descargar y proporcionar el certificado correspondiente para que el IdP de SAML pueda validar la firma de las assertions).            |
| **Sign Request Algorithm**        | Algoritmo que Auth0 usará para firmar las assertions de SAML.                                                                                                                                                              |
| **Sign Request Digest Algorithm** | Algoritmo que Auth0 usará para el resumen de la solicitud firmada.                                                                                                                                                         |
| **Protocol Binding**              | Enlace HTTP compatible con el IdP.                                                                                                                                                                                         |
| **Request Template** (opcional)   | Plantilla que da formato a la solicitud SAML.                                                                                                                                                                              |

<Frame>![Configure SAML Settings](/docs/images/cdy7uua7fh8z/7hvlp8kjva9uFzm5nwsBTQ/4c9f4d01438a3dab6cfb19a5d61d3f13/SAML_Connection_2.png)</Frame>
3. En la vista **Provisioning**, configura cómo se crean y actualizan los perfiles de usuario en Auth0.

| Campo                                                                    | Descripción                                                                                                                                                                                                                        |
| ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **sincronizar atributos del perfil de usuario en cada inicio de sesión** | Cuando está habilitado, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada inicio de sesión, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0. |
| **Sync user profiles using SCIM**                                        | Cuando está habilitado, Auth0 permite sincronizar los datos del perfil de usuario mediante SCIM. Para más información, consulta [configurar SCIM entrante](/docs/es/authenticate/protocols/scim/configure-inbound-scim).              |

4. En la vista **Login Experience**, configura cómo inician sesión los usuarios con esta conexión.

| Campo                              | Descripción                                                                                                                                                                                                                                                     |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Home Realm Discovery**           | Compara el dominio del correo electrónico de un usuario con los dominios del proveedor de identidad proporcionados. Para más información, consulta [Configure Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first) |
| **Display connection button**      | Esta opción muestra las siguientes opciones para personalizar el botón de conexión de tu aplicación.                                                                                                                                                            |
| **Button display name** (Optional) | Texto que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón muestra: &quot;Continuar con &#123;Button display name&#125;&quot;.                                                                           |
| **Button logo URL** (Optional)     | URL de la imagen que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón de inicio de sesión de Universal Login muestra la imagen como un cuadrado de 20 px por 20 px.                                      |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los campos opcionales solo están disponibles con Universal Login. Los clientes que usan Classic Login no verán el botón Add, Button display name ni Button logo URL.
</Callout>

5. Si tienes los permisos administrativos adecuados para completar la integración, haz clic en **Continuar** para conocer los parámetros personalizados necesarios para configurar tu IdP. De lo contrario, proporciona la URL indicada a tu administrador para que pueda ajustar la configuración necesaria.

<div id="create-an-enterprise-connection-using-the-management-api">
  ### Crear una conexión empresarial con la Management API
</div>

También puede usar la [Management API](https://auth0.com/docs/api/management/v2) para crear su conexión SAML. Al hacerlo, puede optar por especificar manualmente cada campo de configuración de SAML o especificar un documento de metadatos SAML que contenga los valores de configuración.

<div id="create-a-connection-using-specified-values">
  #### Crear una conexión con valores especificados
</div>

Haz una llamada `POST` al [endpoint Create a connection](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id). Asegúrate de sustituir los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `CONNECTION_NAME`, `SIGN_IN_ENDPOINT_URL`, `SIGN_OUT_ENDPOINT_URL` y `BASE64_SIGNING_CERT` por tu <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> de la Management API, el nombre de la conexión, la URL de inicio de sesión, la URL de cierre de sesión y el certificado de firma codificado en Base64 (en formato PEM o CER), respectivamente.

<div id="create-a-connection-using-saml-metadata">
  #### Crear una conexión mediante metadatos SAML
</div>

En lugar de especificar cada campo de la configuración de SAML, puedes indicar un documento de metadatos SAML que contenga esos valores de configuración. Al especificar un documento de metadatos SAML, puedes proporcionar el contenido XML del documento (`metadataXml`) o la URL del documento (`metadataUrl`). Si proporcionas la URL, el contenido se descarga una sola vez; la conexión no se reconfigurará automáticamente si el contenido de esa URL cambia en el futuro.

<div id="provide-metadata-document-content">
  ##### Proporcione el contenido del documento de metadatos
</div>

Use la opción `metadataXml` para proporcionar el contenido del documento:

<div id="refresh-existing-connection-information-with-metadata-url">
  ##### Actualizar la información de una conexión existente con la URL de metadatos
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este proceso solo funcionará si la conexión se creó manualmente con `metadataUrl`.
</Callout>

Si tienes una implementación B2B y te federas con Auth0 mediante tu propio proveedor de identidad SAML, es posible que necesites actualizar la información de la conexión almacenada en Auth0, por ejemplo, cambios en el certificado de firma, en la URL del endpoint o nuevos campos de aserción. Auth0 hace esto automáticamente para las conexiones ADFS, pero no para las conexiones SAML.

Puedes crear un proceso por lotes (tarea cron) para realizar una actualización periódica. El proceso puede ejecutarse cada pocas semanas y hacer una llamada PATCH al endpoint `/api/v2/connections/CONNECTION_ID`, pasando un body que contenga `{options: {metadataUrl: '$URL'}}`, donde `$URL` es la misma URL de metadatos con la que creaste la conexión. Usa la URL de metadatos para crear una nueva conexión temporal y luego compara las propiedades de la conexión antigua y la nueva. Si hay alguna diferencia, actualiza la nueva conexión y después elimina la conexión temporal.

1. Crea una conexión SAML con `options.metadataUrl`. El objeto de conexión se rellenará con la información de los metadatos.
2. Actualiza el contenido de metadatos en la URL.
3. Envía un PATCH al endpoint `/api/v2/connections/CONNECTION_ID` con `{options: {metadataUrl: '$URL'}}`. Ahora el objeto de conexión se actualiza con el nuevo contenido de los metadatos.

<Warning>
  Si usas el parámetro `options`, sobrescribirás todo el objeto `options`. Asegúrate de que todos los parámetros estén presentes.
</Warning>

<div id="specify-a-custom-entity-id">
  ## Especifica un ID de entidad personalizado
</div>

Para especificar un ID de entidad personalizado, usa la Management API para reemplazar el valor predeterminado `urn:auth0:YOUR_TENANT:YOUR_CONNECTION_NAME`. Establece la propiedad `connection.options.entityID` cuando se cree la conexión por primera vez o al actualizar una conexión existente.

El siguiente ejemplo de JSON puede usarse para crear una nueva conexión SAML con la URL de metadatos del IdP de SAML y, al mismo tiempo, especificar un ID de entidad personalizado. El ID de entidad sigue siendo único, ya que se crea a partir del nombre de la conexión.

```json lines
{
  "strategy": "samlp", 
  "name": "{yourConnectionName}", 
  "options": { 
    "metadataUrl": "https://saml-idp/samlp/metadata/uarlU13n63e0feZNJxOCNZ1To3a9H7jX",
    "entityId": "urn:your-custom-sp-name:{yourConnectionName}"
  }
}
```
