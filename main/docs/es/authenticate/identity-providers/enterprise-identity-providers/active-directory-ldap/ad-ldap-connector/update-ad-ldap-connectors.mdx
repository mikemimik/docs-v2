---
description: Actualiza un Conector AD/LDAP de Auth0 instalado a la versión más reciente en Windows o Linux mientras conservas la configuración existente y el vínculo con el tenant.
title: Actualizar conectores AD/LDAP
---

Si hay varias instancias del Conector AD/LDAP en tu entorno, Auth0 recomienda realizar los pasos que se indican a continuación para cada instancia, una a la vez, de modo que solo una instancia esté inactiva en cada momento.

<div id="admin-console-windows">
  ## Consola de administración (Windows)
</div>

1. Abra la Admin Console en `http://localhost:8357/`. Si la Admin Console tiene acceso saliente a Internet, comprobará si hay una nueva versión disponible y lo mostrará en la parte superior de la página:

   <Frame>![Pantalla de actualización de la Admin Console de los conectores AD/LDAP](/docs/images/cdy7uua7fh8z/6Yp2uRnEU3RMeCqgHa6Ztl/b9c04ab83b9fd725910291b01ac9d05d/connector-update-available.png)</Frame>
2. Vaya a la pestaña **Update** para actualizar la instalación de la máquina actual a la versión más reciente. La actualización tardará unos 2 minutos y los registros del actualizador se mostrarán en la Admin Console.

<div id="updater-script-windows">
  ## Script de actualización (Windows)
</div>

El script de actualización actualizará Conector AD/LDAP desde la línea de comandos siguiendo estos pasos:

1. Verifique si hay una actualización disponible.
2. Haga una copia de seguridad de la configuración existente, los certificados y profileMapper.js.
3. Desinstale Conector AD/LDAP.
4. Descargue la actualización.
5. Instale Conector AD/LDAP.
6. Restaure la configuración existente, los certificados y profileMapper.js.
7. Inicie el servicio de Windows.
8. Para ejecutar el script de actualización, ejecute la siguiente instrucción en la línea de comandos:
   `@powershell -NoProfile -ExecutionPolicy Bypass -Command "iex ((new-object net.webclient).DownloadString('https://cdn.auth0.com/connector/windows/update-connector.ps1'))"`

El script de actualización usa comandos específicos de PowerShell que solo están disponibles en Windows PowerShell 3.0 y versiones posteriores. Si lo ejecuta en Windows 2008 o Windows 2008 R2, es posible que primero deba actualizar su versión de PowerShell.

Para obtener más información, consulte [Descargar Window Management Framework](https://www.microsoft.com/en-us/download/details.aspx?id=34595).

<div id="manual-update-windowslinux">
  ## Actualización manual (Windows/Linux)
</div>

1. Verifique la versión que tiene instalada. Coloque el cursor sobre el indicador de estado del Conector en la consola. La información sobre herramientas mostrará el estado actual y la versión instalada.
2. Descargue la versión más reciente del [instalador](https://cdn.auth0.com/connector/windows/adldap-5.0.10.msi?_ga=2.58973869.1675886805.1589821069-1123834949.1586212750).
   Para otras plataformas, use la [versión](https://github.com/auth0/ad-ldap-connector/releases/tag/v5.0.10) del repositorio de GitHub.
3. Haga una copia de seguridad de su configuración actual desde la Admin Console o manualmente.

   1. En la Admin Console, haga clic en el botón **Descargar** para generar un archivo .zip que contiene el archivo `config.json`, la carpeta `certs` y el archivo `lib\\profileMapper.js`.
   2. Para hacerlo manualmente, haga una copia de seguridad de los mismos archivos indicados arriba desde la carpeta `%Program Files(x86)%\Auth0\AD LDAP Connector\`. Esta ruta corresponde a Windows. En otras plataformas, es posible que los archivos estén en otra ubicación, pero contendrán los mismos recursos.
4. Ejecute el instalador y siga las instrucciones.
5. Cierre el cuadro de diálogo de configuración sin cambiar nada.
6. Restaure los archivos desde la Admin Console o manualmente.

   1. En la Admin Console, cargue el archivo .zip en la pestaña **Importar / Exportar**.
   2. Para hacerlo manualmente, copie los archivos de los que hizo la copia de seguridad anteriormente.
7. Reinicie el servicio **Auth0 AD LDAP** desde la consola de servicios.
