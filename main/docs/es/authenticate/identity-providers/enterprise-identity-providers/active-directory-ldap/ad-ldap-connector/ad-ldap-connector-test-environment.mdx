Puede configurar un entorno de prueba creando una instalación mínima de un controlador de dominio de AD en una máquina virtual (VM) desplegada en la nube. Puede ejecutar su VM en cualquier plataforma en la nube. Este ejemplo muestra cómo configurarla en Microsoft Azure.

1. Cree una máquina virtual en la consola de [Azure Management](https://azure.microsoft.com/en-us/features/azure-portal/).
2. Haga clic en la pestaña **ENDPOINTS** de la nueva VM y tome nota del **PUBLIC PORT** del endpoint de **Remote Desktop**.
3. Abra el cliente Microsoft Remote Desktop (Windows o Mac) o el cliente que prefiera (como [rdesktop](http://www.rdesktop.org/) para sistemas Linux). Cree una nueva conexión a su VM.
4. Abra la conexión e ignore cualquier advertencia de certificado que muestre el cliente de Remote Desktop. Si se le solicita buscar PC, dispositivos y contenido en la red local, elija **No**.
5. Haga clic en el icono de PowerShell en la barra de tareas de Windows para abrir el **PowerShell Command Prompt**. Instale ADDS.
6. Promueva el servidor a controlador de dominio para administrar un FQDN de `mycompany.local`.
7. Cuando se le solicite **SafeModeAdministratorPassword**, introduzca la contraseña de administrador que usó al crear la VM. Presione Entrar para continuar. El script de promoción se ejecutará y la VM se reiniciará automáticamente.
8. Agregue grupos y usuarios de prueba. Inicie sesión en la VM con el cliente de Remote Desktop y abra el PowerShell Command Prompt. Ejecute el siguiente script:

   ```text lines
   > New-ADGroup -Name "Accounting" -GroupScope "DomainLocal"
   > New-ADGroup -Name "IT" -GroupScope "DomainLocal"

   > New-ADUser -GivenName Bob -Surname Johnson -Name "Bob Johnson" -SamAccountName bob.johnson -Enabled $True -AccountPassword (ConvertTo-SecureString "Pass@word1!" -AsPlainText -force) -PasswordNeverExpires $True
   > New-ADUser -GivenName Mary -Surname Smith -Name "Mary Smith" -SamAccountName mary.smith -Enabled $True -AccountPassword (ConvertTo-SecureString "Pass@word1!" -AsPlainText -force) -PasswordNeverExpires $True

   > Add-ADGroupMember -Identity Accounting -Members "bob.johnson", "mary.smith"
   > Add-ADGroupMember -Identity IT -Members "mary.smith"
   ```

<div id="install-and-configure-adldap-connector">
  ## Instala y configura Conector AD/LDAP
</div>

1. Ve a [Auth0 Dashboard &gt; Autenticación &gt; empresarial](https://manage.auth0.com/#/connections/enterprise) y crea una nueva conexión de **Active Directory/LDAP** con el nombre `auth0-test-ad`. Asegúrate de copiar la **URL del ticket** que se genera al final de esas instrucciones.
2. En la VM, deshabilita la **Configuración de seguridad mejorada de Internet Explorer**.
3. Abre **Internet Explorer** con la **URL del ticket** que guardaste en el paso 1.
4. Sigue las instrucciones del navegador para descargar, instalar y configurar el **Conector**. Cuando se te solicite la cuenta de servicio de LDAP, usa la cuenta de administrador que creaste para la VM: **Nombre de usuario**: `mycompany\ad-admin,` **Con**traseña: (la misma que antes).
5. Cuando termines de configurar e instalar el **Conector**, reinicia el servidor.
6. Vuelve a iniciar sesión en la VM con Remote Desktop.
7. Abre el sitio de configuración del Conector y ve a `http://localhost:8357/`.
8. Comprueba que el **Conector** pueda encontrar un usuario:

   1. Haz clic en la pestaña **Search**.
   2. En &quot;Find User by Login&quot;, escribe `mary.smith`.

<div id="test-authentication-from-auth0">
  ## Probar la autenticación con Auth0
</div>

Para asegurarnos de que todo funcione con tu cuenta de Auth0, vamos a configurar tu **Default App** en Auth0 para que use tu nueva conexión **Active Directory / LDAP** y a usar el endpoint `/authorize` para iniciar un flujo de autenticación.

1. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).

2. Haz clic en el icono de **Settings** de tu **Default App**.

3. Agrega `http://jwt.io` a la lista de **Allowed Callback URLs** de la aplicación.

4. Haz clic en la pestaña **Connections**.

5. En **Empresarial**, habilita la conexión **Active Directory / LDAP** `auth0-test-ad`.

6. Prueba el flujo de autenticación abriendo el siguiente enlace en tu navegador:

   <AuthCodeBlock children={codeExample} language="http" />

7. Inicia sesión con uno de los usuarios de prueba que se crearon en el directorio:

   * Nombre de usuario: `mary.smith` o `bob.johnson`
   * Contraseña: `Pass@word1!`

8. Si todo funciona correctamente, se te redirigirá al sitio web de JWT.io para ver el contenido del JWT resultante.
