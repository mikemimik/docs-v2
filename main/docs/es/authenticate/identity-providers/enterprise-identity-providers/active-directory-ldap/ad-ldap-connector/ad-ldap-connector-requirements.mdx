---
description: Enumera todos los requisitos previos para instalar y configurar el Conector.
title: Requisitos del sistema del Conector AD/LDAP
---

Por lo general, el Conector AD/LDAP debe instalarlo un administrador de sistemas o un ingeniero de operaciones, y no un desarrollador, ya que suele requerir acceso a recursos de Production. A continuación, se incluye una lista de comprobación de aspectos que deben tener en cuenta antes de la instalación propiamente dicha:

<div id="host-servers">
  ## Servidores host
</div>

El Conector puede instalarse en un servidor existente, incluso en un Controlador de Dominio. Sin embargo, lo más habitual es instalarlo en máquinas virtuales destinadas exclusivamente al Conector. En cualquier caso, el servidor host debe contar con las siguientes especificaciones/configuraciones de hardware y software:

<div id="hardware-requirements">
  ### Requisitos de hardware
</div>

* **Arquitectura**: x86 o x86-64
* **Núcleos de CPU**: mín. 1; se recomiendan 2
* **Almacenamiento**: 500MB de espacio libre en disco
* **Sistema operativo**: El Conector puede ejecutarse en Windows o Linux. Windows es necesario si se va a utilizar la autenticación Kerberos.
* **RAM**: mín. 2GB

<div id="windows-version">
  ### Versión de Windows
</div>

Recomendamos usar una versión compatible de Windows Server, como Windows Server 2016 o Windows Server 2019. El Conector también puede ejecutarse en Windows Server 2012 R2.

<div id="time-synchronization">
  ### Sincronización horaria
</div>

Es muy importante que el reloj del servidor host del Conector se sincronice automáticamente con un servidor NTP. De lo contrario, el Conector no podrá iniciarse y mostrará un error de desfase horario.

<div id="outbound-connectivity">
  ## Conectividad saliente
</div>

El servidor host requiere conectividad de red saliente con los siguientes servicios:

<div id="auth0">
  ### Auth0
</div>

El conector debe instalarse en un servidor con conectividad saliente hacia el servicio de Auth0 en: `https://{yourDomain}` por el puerto 443.

El conector puede instalarse y configurarse detrás de un servidor proxy, pero no lo recomendamos. Habilite un proxy a través de la variable de entorno o de configuración `HTTP_PROXY`.

<div id="ldap">
  ### LDAP
</div>

El Conector debe instalarse en un servidor con acceso al servidor LDAP a través del puerto 389 para ldap o del 636 para ldaps. Antes de instalar el Conector, debe conocer la cadena de conexión de LDAP y el DN base necesarios para conectarse a su directorio LDAP.

<div id="inbound-connectivity">
  ## Conectividad entrante
</div>

No es necesario habilitar la conectividad entrante al Conector, salvo que esté habilitada la autenticación de Kerberos o mediante certificados. En esos casos, los servidores en los que esté instalado el Conector deben ser accesibles desde los navegadores de sus usuarios a través del puerto 443. Si hay instalada más de una instancia del conector, debe usar un equilibrador de carga para dirigir el tráfico a uno u otro conector.

Para obtener más información, consulte [Configurar la autenticación del Conector AD/LDAP con Kerberos](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-with-kerberos) o [Configurar la autenticación del Conector AD/LDAP con certificados de cliente](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-client-certificates).

<div id="service-account">
  ## Cuenta de servicio
</div>

El Conector se ejecutará con una cuenta de servicio que debe ser un usuario del dominio y que, como mínimo, debe tener acceso de lectura al directorio. Necesitarás el nombre de usuario y la contraseña de esta cuenta durante la instalación.

<div id="one-connector-per-auth0-tenant">
  ## Un Conector por cada tenant de Auth0
</div>

Si establece varios tenants de Auth0, por ejemplo para aislar los entornos de desarrollo y de producción, deberá configurar una conexión AD/LDAP en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y un Conector AD/LDAP para cada tenant de Auth0 que necesite esta forma de autenticación. Un Conector está vinculado a una conexión específica dentro de un tenant de Auth0.

Es posible tener varios Conectores dentro de un tenant de Auth0 si tiene varios directorios AD/LDAP contra los que se autenticarán los usuarios. Por ejemplo, para admitir distintos departamentos o clientes, cada uno con su propio directorio. Además, varios Conectores pueden apuntar al mismo directorio AD o LDAP, pero un Conector solo puede ser utilizado por una Conexión de Auth0 dentro de un mismo tenant de Auth0.

<div id="high-availability">
  ## Alta disponibilidad
</div>

El Conector puede instalarse en varios servidores host para garantizar alta disponibilidad y redundancia (la mayoría de las organizaciones instala dos), en caso de que uno de los servidores deje de estar disponible. Cada servidor tendrá los mismos requisitos indicados anteriormente. No se requiere ningún equilibrador de carga, ya que el propio servidor de Auth0 se encarga de ello, a menos que habilite Kerberos o autenticación basada en certificados de cliente.

Para obtener más información, lea [Implementar conectores AD/LDAP en entornos de alta disponibilidad.](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-high-availability)
