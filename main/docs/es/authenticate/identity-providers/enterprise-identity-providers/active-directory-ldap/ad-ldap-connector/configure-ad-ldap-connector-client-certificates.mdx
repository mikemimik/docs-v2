El Conector AD/LDAP también permite a los usuarios autenticarse con un certificado instalado en su equipo o dispositivo.

<div id="enable-client-certificates">
  ## Habilitar certificados de cliente
</div>

1. Ve a [Auth0 Dashboard &gt; Autenticación &gt; Empresarial &gt; Active Directory/LDAP](https://manage.auth0.com/#/connections/enterprise/ad) y selecciona la conexión que quieras configurar.
2. Activa la opción **Use client SSL certificate authentication** en la configuración.
3. Introduce los rangos de direcciones IP en el campo **IP Ranges**. Solo se pedirá a los usuarios que provengan de los rangos de IP indicados que se autentiquen con certificados de cliente. A los usuarios que provengan de otros rangos de IP se les pedirá que inicien sesión con el formulario de nombre de usuario y contraseña.

<div id="configure-certificates">
  ## Configurar certificados
</div>

Una vez configurada la conexión AD/LDAP en Auth0, deberá configurar los certificados en el Conector AD/LDAP. Para admitir certificados de cliente, se requiere lo siguiente:

* Un certificado SSL para la URL pública, ya que la interacción entre el usuario final y el Conector deberá realizarse a través de HTTPS.
* Uno o más certificados de CA.
* Un certificado de cliente firmado por la CA para cada usuario que necesite autenticarse con certificados de cliente.

1. Antes de cargar certificados en el Conector AD/LDAP, convierta los certificados X.509 a Base64. Use Base64 o Certutil en Windows Server. Para obtener más información, consulte [Base64 Decode](https://www.base64decode.org/) en Base64decode o [Certutil.exe](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/certutil#BKMK_encode) en la documentación de Microsoft.
2. Cargue los certificados SSL y de CA en el Conector AD/LDAP:

   <Frame>![Pantalla de Setup para configurar la autenticación del Conector AD/LDAP con certificados de cliente](/docs/images/cdy7uua7fh8z/2zRux90ZHATIBFWyHlodue/c426e5a475ff528a467db6409980de12/connector-client-cert-config.png)</Frame>
3. Para hacer pruebas, genere una CA autofirmada y certificados de cliente con **makecert.exe** en Windows, que forma parte del SDK de Windows:

   ```bash lines
   SET ClientCertificateName=jon
       SET RootCertificateName=FabrikamRootCA
       "C:\Program Files (x86)\Microsoft SDKs\Windows\v7.1A\Bin\makecert.exe" -sky exchange -r -n "CN=%RootCertificateName%" -pe -a sha1 -len 2048 -ss My "%RootCertificateName%.cer"
       "C:\Program Files (x86)\Microsoft SDKs\Windows\v7.1A\Bin\makecert.exe" -n "CN=%ClientCertificateName%" -pe -sky exchange -m 96 -ss My -in "%RootCertificateName%" -is my -a sha1
   ```

   Es importante que el subject del certificado de cliente tenga el formato `CN=AD_USERNAME`, por ejemplo, `CN=jon`.

En una aplicación, cuando un usuario inicia el flujo de inicio de sesión con una conexión AD/LDAP:

```js lines
auth.signin({
       popup: true,
       connection: 'FabrikamAD',
       scope: 'openid name email'
     }, onLoginSuccess, onLoginFailed);
```

Si la dirección IP del usuario está dentro del rango de IP configurado, se le pedirá que se autentique con un certificado de cliente:
