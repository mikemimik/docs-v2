<div id="prerequisites">
  ## Requisitos previos
</div>

* Asegúrese de que su sistema cumpla con los [requisitos del sistema](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-connector-requirements).
* Asegúrese de haber instalado ya [Git](https://git-scm.com/downloads), [Node.js](https://nodejs.org) y [npm](http://blog.npmjs.org/post/85484771375/how-to-install-npm).
* En la mayoría de las plataformas, necesitará privilegios de root y/o de administrador.

<div id="install-the-connector-for-windows">
  ## Instalar el Conector para Windows
</div>

<div id="download-and-run-the-installer">
  ### Descarga y ejecuta el instalador
</div>

Ve a [Auth0 Dashboard &gt; Authentication &gt; empresarial &gt; Active Directory/LDAP](https://manage.auth0.com/#/connections/enterprise/ad) y selecciona la conexión que quieras configurar.

En la pestaña **Setup**, selecciona el botón **Install For Windows** para descargar el instalador más reciente, que es un archivo MSI.

Ejecuta el instalador y sigue las instrucciones para instalar el **Conector AD/LDAP** como servicio de Windows.

<Frame>![Servicios del Conector AD/LDAP](/docs/images/cdy7uua7fh8z/MPMfLmJsXoPAzpPQhGAv5/25fb39ac7cef7a125110b8fa9f2dff01/adldap-connector-services.png)</Frame>

Una vez completada la instalación, verás en el navegador una pantalla que apunta a localhost:

<Frame>![Ticket de administrador del Conector AD/LDAP](/docs/images/cdy7uua7fh8z/3biSuxJYQ4swMGJkOZtC3R/cab77cb763ad3b9ff47710bc406c527b/adldap-connector-admin-ticket.png)</Frame>

Introduce la **URL DEL TICKET** que se te proporcionó cuando aprovisionaste la conexión. La **URL DEL TICKET** identifica de forma única este conector en Auth0. El Conector la usará para comunicarse con el servidor de Auth0 y completar automáticamente la configuración.

<div id="link-to-ldap">
  ### Vincular con LDAP
</div>

Introduce la configuración de LDAP:

<Frame>![adldap-connector-admin-settings](/docs/images/cdy7uua7fh8z/2mHaknek0qyBxGmsanb30L/600b0259ed6db3fe9c526aa9f57a76bd/adldap-connector-admin-settings.png)</Frame>

| Configuración               | Descripción                                                                                                                                                                                                                                                                                                                                                          |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Cadena de conexión LDAP** | Por ejemplo, `ldap://ldap.internal.contoso.com`. Este es el protocolo + el nombre de dominio o la dirección IP de tu servidor LDAP. Tu servidor LDAP es el controlador de dominio local donde está instalado Active Directory. El protocolo puede ser `ldap` o `ldaps`. Si necesitas usar `ldaps`, asegúrate de que el certificado sea válido en el servidor actual. |
| **DN base**                 | Por ejemplo, `dc=contoso,dc=com`. Este es el contenedor base para todas las consultas realizadas por el Conector.                                                                                                                                                                                                                                                    |
| **Nombre de usuario**       | Por ejemplo, `cn=svcauth0,dc=services,dc=contoso,dc=com`. El nombre distintivo completo de un usuario para realizar consultas.                                                                                                                                                                                                                                       |
| **Contraseña**              | La contraseña del usuario.                                                                                                                                                                                                                                                                                                                                           |

Una vez que envíes la información anterior, el Conector realizará una serie de pruebas para validar la información que introdujiste. Los resultados se muestran en **Registro de configuración** en la pantalla de la consola.

<Frame>![adldap-connector-admin-settings-ok](/docs/images/cdy7uua7fh8z/7kezId8mhvyYDbuovvi9FM/c2f3b2188fdfebc460458daf7247c124/adldap-connector-admin-settings-ok.png)</Frame>

Asegúrate de que todas las pruebas aparezcan en verde.

| Prueba   | Descripción                                                                                                                            | Solución de problemas                                                                                    |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| Prueba 1 | Intenta establecer una conexión TCP con el servidor LDAP y el puerto especificado.                                                     | Comprueba la conectividad básica de red y la configuración del firewall que podría impedir esa conexión. |
| Prueba 2 | Intenta realizar un enlace LDAP en el servidor LDAP y el puerto especificados con el nombre de usuario y la contraseña proporcionados. | Comprueba la cadena de conexión LDAP, la ruta de búsqueda, el nombre de usuario y la contraseña.         |
| Prueba 3 | Intenta realizar una búsqueda LDAP en el directorio para comprobar los privilegios del nombre de usuario especificado.                 | Comprueba los privilegios del nombre de usuario en el directorio de destino.                             |
| Prueba 4 | Intenta establecer una conexión con el servidor de Auth0.                                                                              | Comprueba la conectividad de red y la configuración del firewall que podría impedir esa conexión.        |

<div id="install-the-connector-for-other-platforms">
  ## Instala el Conector para otras plataformas
</div>

1. Descarga e instala la versión más reciente del paquete **[Auth0 Active Directory/LDAP Connector](https://github.com/auth0/ad-ldap-connector/releases)** en `/tmp`. Para obtener más información, consulta el [repositorio del Conector AD/LDAP](https://github.com/auth0/ad-ldap-connector).

2. Inicia el servidor.
   `node server.js`Cuando se te pida el número de ticket, introduce la URL completa del ticket desde la pestaña **Settings** de la pantalla **Setup AD/LDAP connector** en el Dashboard de Management de Auth0.

3. Se te pedirá que edites el archivo de configuración `config.json` con los datos de tu conexión LDAP y de autenticación:

   <AuthCodeBlock children={codeExample} language="json" lines />

4. Ejecuta `node server.js` una vez más para iniciar el Conector. Ten en cuenta que la línea `LDAP_BIND_PASSWORD` de `config.json` cambia a `LDAP_BIND_CREDENTIALS` en este punto.

5. Una vez que el Conector esté en ejecución, tendrás que ejecutarlo como daemon (si aún no has elegido una herramienta, puedes considerar [systemd](https://www.freedesktop.org/wiki/Software/systemd/)). Por ejemplo, para usar systemd con Ubuntu Xenial, el archivo `/lib/systemd/system/auth0-adldap.service` podría contener lo siguiente:

   ```bash
   [Unit]
   Description=Auth0 AD LDAP Agent
   After=network.target

   [Service]
   Type=simple
   Restart=always
   User=ubuntu
   WorkingDirectory=/opt/auth0-adldap
   ExecStart=/usr/bin/node server.js
   ```

6. Ejecuta `node admin/server.js` para acceder a la interfaz de administración. La interfaz de administración estará en ejecución y disponible en `http://localhost:8357`.

Si recibes un mensaje de `Invalid Ticket` al configurar el Conector por primera vez, la causa más probable es un problema de red (por ejemplo, que el Conector se esté ejecutando detrás de un proxy).
