<Card title="Información general">
  Conceptos clave

  * Sigue los pasos para registrar tu aplicación en Azure AD y conectarla a tu instancia de Auth0.
  * Prueba la conexión antes de pasar tu configuración a producción.
  * Revisa los consejos de resolución de problemas.
</Card>

<Warning>
  **Importante:** Atributos extendidos en las conexiones de Azure Active Directory (v1) Como Microsoft está retirando Azure AD Graph API, las conexiones que usan la API de identidad de Azure Active Directory (v1) ya no pueden habilitar por primera vez Obtener perfil extendido, Obtener grupos de usuarios ni Obtener grupos anidados (`options.ext_profile`, `options.ext_groups`, `options.ext_nested_groups`). En las conexiones v1 existentes, estas opciones solo se mantienen si ya estaban habilitadas, pero dejarán de funcionar por completo una vez que se complete el retiro de Microsoft Graph API en 2025. Los atributos extendidos siguen siendo compatibles con las conexiones que usan la API de identidad de Microsoft Identity Platform (v2), que se basa en Microsoft Graph y requiere los Delegated permissions `User.Read` y `Directory.Read.All`.
</Warning>

Puedes integrarte con Microsoft Azure Active Directory (AD) si quieres permitir que los usuarios:

* De tu empresa usen tu aplicación desde un Azure AD controlado por ti o por tu organización.
* De otras empresas usen tu aplicación desde sus Azure AD. (Te recomendamos configurar los directorios externos como conexiones independientes).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los claims que devuelve la conexión empresarial de Azure AD son estáticos; los claims personalizados u opcionales no aparecerán en los perfiles de usuario. Si necesitas incluir claims personalizados u opcionales en los perfiles de usuario, usa una conexión SAML u OIDC en su lugar.
</Callout>

<div id="register-your-app-with-azure-ad">
  ### Registra tu aplicación con Azure AD
</div>

Para registrar tu aplicación con Azure AD, consulta la [Quickstart: Register an application with the Microsoft identity platform](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-register-app) de Microsoft.

<Warning>
  Si tienes más de un directorio de Azure AD, asegúrate de estar en el directorio correcto al registrar la aplicación que quieres usar con Auth0.
</Warning>

Durante el registro, configura las siguientes opciones:

| Opción                      | Configuración                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Supported account types** | Para habilitar el aprovisionamiento de usuarios basado en SCIM desde este directorio, selecciona **Accounts in this organizational directory only (Single tenant)**. Para permitir el acceso a usuarios de organizaciones externas (como otros directorios de Azure AD), elige la opción multitenant adecuada. Las opciones multitenant incluyen la siguiente: **Accounts in any organizational directory (Any Azure AD directory - Multitenant)**. |
| **Redirect URI**            | Selecciona **Web** como tipo de Redirect URI e introduce tu URL de devolución de llamada: `https://{yourDomain}/login/callback`.                                                                                                                                                                                                                                                                                                                    |

<Card title="Encuentra tu nombre de dominio de Auth0 para las redirecciones">
  Si tu nombre de dominio de Auth0 no aparece arriba y no estás usando nuestra funcionalidad de [dominios personalizados](/docs/es/customize/custom-domains), tu nombre de dominio se compone del nombre de tu tenant, tu subdominio regional y `auth0.com`, separados por el símbolo de punto (`.`).

  Por ejemplo, si el nombre de tu tenant es `exampleco-enterprises` y tu tenant está en la región de EE. UU., tu nombre de dominio de Auth0 sería `exampleco-enterprises.us.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.us.auth0.com/login/callback`.

  Sin embargo, si tu tenant está en la región de EE. UU. y se creó antes de junio de 2020, tu nombre de dominio de Auth0 sería `exampleco-enterprises.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.auth0.com/login/callback`.

  Si estás usando [dominios personalizados](/docs/es/customize/custom-domains), tu **Redirect URI** sería `https://<YOUR CUSTOM DOMAIN>/login/callback`.
</Card>

Durante este proceso, Microsoft genera un **ID. de aplicación (cliente)** para tu aplicación; puedes encontrarlo en la pantalla **Información general** de la aplicación. **Toma nota de este valor.**

<div id="create-a-client-secret">
  #### Crear un secreto de cliente
</div>

Para crear un <Tooltip tip="secreto de cliente: secreto que usa un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>, consulta [Agregar y administrar credenciales de aplicaciones en Microsoft Entra ID - Agregar un secreto de cliente](https://learn.microsoft.com/en-us/entra/identity-platform/how-to-add-credentials?source=recommendations\&tabs=client-secret).

Una vez generado, **anota este valor**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si configuras un secreto con vencimiento, asegúrate de **anotar la fecha de vencimiento**; tendrás que renovar la clave antes de esa fecha para evitar una interrupción del servicio.
</Callout>

<div id="add-permissions">
  #### Agregar permisos
</div>

Para agregar permisos, consulta la guía de Microsoft [Quickstart: Configurar una aplicación cliente para acceder a Web API: agregar permisos para acceder a Web API](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-configure-app-access-web-apis#add-permissions-to-access-web-apis).

Mientras configuras los permisos, ten en cuenta lo siguiente:

Si quieres habilitar atributos extendidos (como **Perfil extendido** o **Grupos de seguridad**), tendrás que configurar los siguientes permisos para **Microsoft Graph API**.

| Delegated permissions                 | Descripción                                                                                                                        |
| ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **Users &gt; User.Read**              | Para que tu aplicación pueda iniciar sesión en nombre de los usuarios y leer los perfiles de los usuarios que han iniciado sesión. |
| **Directory &gt; Directory.Read.All** | Para que tu aplicación pueda leer datos del directorio en nombre del usuario que ha iniciado sesión.                               |

<div id="create-an-enterprise-connection-in-auth0">
  ### Crear una conexión empresarial en Auth0
</div>

Cree y configure una conexión empresarial de Azure AD en Auth0. Asegúrese de tener el **ID de aplicación (cliente)** y el **secreto de cliente** generados al configurar su aplicación en el portal de Microsoft Azure.

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), busque **Microsoft Azure AD** y seleccione el signo `+`.

   <Frame>![Dashboard - Connections - Enterprise](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Introduzca los datos de la conexión y seleccione **Create**:

| Campo                              | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Connection name**                | Identificador lógico de la conexión; debe ser único dentro de su tenant. Una vez establecido, este nombre no se puede cambiar.                                                                                                                                                                                                                                                                                                                            |
| **Microsoft Azure AD Domain**      | El nombre de dominio de su Azure AD. Puede encontrarlo en la página de información general de su directorio de Azure AD en el portal de Microsoft Azure.                                                                                                                                                                                                                                                                                                  |
| **Client ID**                      | Identificador único de su aplicación registrada en Azure AD. Introduzca el valor guardado del **ID de aplicación (cliente)** de la aplicación que acaba de registrar en Azure AD.                                                                                                                                                                                                                                                                         |
| **Client Secret**                  | Cadena que se utiliza para obtener acceso a su aplicación registrada en Azure AD. Introduzca el valor guardado del **secreto de cliente** de la aplicación que acaba de registrar en Azure AD.                                                                                                                                                                                                                                                            |
| **User ID Attribute Type**         | Atributo del token de ID que se asignará a la propiedad `user_id` en Auth0. Las opciones corresponden a los dos tipos de ID compatibles descritos en la documentación de Microsoft: [ID token claims reference - Microsoft identity platform](https://learn.microsoft.com/en-us/entra/identity-platform/id-token-claims-reference#use-claims-to-reliably-identify-a-user).                                                                                |
| **Use common endpoint**            | (Opcional) Cuando está habilitado, su aplicación aceptará dinámicamente usuarios de nuevos directorios. Normalmente se habilita si seleccionó una opción multi-tenant para los tipos de cuenta admitidos de la aplicación que acaba de registrar en Azure AD. Auth0 redirige a los usuarios al endpoint de inicio de sesión común de Azure y Azure realiza Home Realm Discovery en función del dominio de la dirección de correo electrónico del usuario. |
| **Identity API**                   | API de identidad que utiliza Auth0 para interactuar con los endpoints de Azure AD. Consulte las diferencias de comportamiento en la documentación de Microsoft: [Why update to Microsoft identity platform (v2.0)](https://docs.microsoft.com/en-us/azure/active-directory/develop/azure-ad-endpoint-comparison).                                                                                                                                         |
| **Attributes**                     | Atributos básicos del usuario que ha iniciado sesión a los que su aplicación puede acceder. Indica cuánta información desea almacenar en el perfil de usuario de Auth0.                                                                                                                                                                                                                                                                                   |
| **Extended Attributes** (opcional) | Atributos extendidos del usuario que ha iniciado sesión a los que su aplicación puede acceder.                                                                                                                                                                                                                                                                                                                                                            |
| **Auth0 APIs** (opcional)          | Cuando se selecciona, indica que necesitamos la capacidad de realizar llamadas a la API de Azure AD, lo que nos permite buscar usuarios en Azure AD Graph aunque nunca hayan iniciado sesión en Auth0.                                                                                                                                                                                                                                                    |
| **Email Verification**             | Elija cómo Auth0 establece el campo `email_verified` en el perfil del usuario. Para obtener más información, lea [Email Verification for Azure AD and ADFS](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azuread-adfs-email-verification).                                                                                                                                                                                         |
| **Purpose**                        | Habilite la conexión para authentication, Connected Accounts for Token Vault o ambas opciones. Para obtener más información, lea [User authentication vs Connected Accounts](/docs/es/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).                                                                                                                                                               |

3. En la vista **Provisioning**, puede configurar cómo se crean y actualizan los perfiles de usuario en Auth0.

| Campo                                          | Descripción                                                                                                                                                                                                                        |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Sync user profile attributes at each login** | Cuando está habilitado, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada inicio de sesión, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0. |
| **Sync user profiles using SCIM**              | Cuando está habilitado, Auth0 permite sincronizar los datos del perfil de usuario mediante SCIM. Para obtener más información, lea [Configure Inbound SCIM](/docs/es/authenticate/protocols/scim/configure-inbound-scim).             |

4. En la vista **Login Experience**, puede configurar cómo los usuarios inician sesión con esta conexión.

| Campo                                        | Descripción                                                                                                                                                                                                                                                           |
| -------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Home Realm Discovery**                     | Compara el dominio de correo electrónico del usuario con los dominios proporcionados del proveedor de identidad. Para obtener más información, consulta [Configurar Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first) |
| **Mostrar botón de conexión**                | Esta opción muestra las siguientes opciones para personalizar el botón de conexión de tu aplicación.                                                                                                                                                                  |
| **Nombre para mostrar del botón** (Opcional) | Texto que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón muestra: &quot;Continuar con &#123;Nombre para mostrar del botón&#125;&quot;.                                                                       |
| **URL del logotipo del botón** (Opcional)    | URL de la imagen que se usa para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón de inicio de sesión de Universal Login muestra la imagen como un cuadrado de 20 px por 20 px.                                            |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los campos opcionales solo están disponibles con Universal Login. Los clientes que usan Classic Login no verán el botón Add, Button display name ni Button logo URL.
</Callout>

5. Si tienes los permisos administrativos adecuados de Azure AD para otorgar el consentimiento a la aplicación y permitir que los usuarios inicien sesión, haz clic en **Continuar**.
   Se te pedirá que [inicies sesión en tu cuenta de Azure AD](https://docs.microsoft.com/en-us/azure/active-directory/develop/v2-permissions-and-consent#requesting-consent-for-an-entire-tenant) y otorgues tu consentimiento. De lo contrario, proporciona la URL indicada a tu administrador para que pueda otorgarlo.

<div id="troubleshooting">
  ## Resolución de problemas
</div>

Aquí tienes algunos consejos de resolución de problemas:

**Registré mi aplicación en Azure AD, pero cuando vuelvo a los registros de aplicaciones de Azure Active Directory, no veo mi aplicación.**

Es posible que hayas registrado tu aplicación por error en el directorio de Azure AD equivocado (o incluso que no hubieras creado ningún directorio de Azure AD antes de registrarla). Lo más sencillo suele ser volver a registrar la aplicación en Azure AD. Asegúrate de estar en el directorio correcto al registrarla. Si necesitas crear un directorio de Azure AD, sigue la guía de Microsoft [Quickstart: Create a new tenant in Azure Active Directory - Crear un nuevo tenant para tu organización](https://docs.microsoft.com/en-us/azure/active-directory/fundamentals/active-directory-access-create-new-tenant#create-a-new-tenant-for-your-organization).

**Recibo el siguiente mensaje de error: &quot;Access cannot be granted to this service because the service listing is not properly configured by the publisher&quot;.**

Para resolverlo, prueba a cambiar los **Supported account types** de tu aplicación registrada en Azure AD. Asegúrate de haber elegido una opción multitenant adecuada en la configuración de autenticación de la aplicación de Azure AD. Las opciones multitenant incluyen la siguiente: **Accounts in any organizational directory (Any Azure AD directory - Multitenant)**.

**Cuando los usuarios intentan iniciar sesión, recibimos el siguiente mensaje de error: &quot;invalid&#95;request; failed to obtain <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>&quot;.**

La causa más probable de este error es que el **secreto de cliente** de Azure AD no es válido o ha vencido. Para resolverlo, genera un nuevo **secreto de cliente** para tu aplicación en Azure AD y luego actualiza el **secreto de cliente** en la conexión empresarial configurada con Auth0.

<div id="signing-key-rollover-in-azure-ad">
  ## Rotación de la clave de firma en Azure AD
</div>

El <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> utiliza las claves de firma para firmar el token de autenticación que emite, y la aplicación cliente (Auth0, en este caso) para validar la autenticidad del token generado.

Por motivos de seguridad, la clave de firma de Azure AD [rota periódicamente](https://docs.microsoft.com/en-us/azure/active-directory/develop/active-directory-signing-key-rollover). Si esto ocurre, **no es necesario que realice ninguna acción**. Auth0 usará la nueva clave automáticamente.

<div id="remove-unverified-label">
  ## Eliminar la etiqueta «no verificado»
</div>

Si usas un <Tooltip tip="Dominio personalizado: Dominio de terceros con un nombre especializado o de vanidad." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip>, la solicitud de consentimiento de la aplicación para el inicio de sesión de Azure AD puede mostrar tu dominio con la etiqueta «no verificado». Para eliminar la etiqueta «no verificado»:

1. Verifica el dominio de la aplicación de Auth0: [Agrega tu dominio personalizado en el portal de Azure Active Directory](https://docs.microsoft.com/en-us/azure/active-directory/fundamentals/add-custom-domain#add-your-custom-domain-name-to-azure-ad).
2. Asigna el dominio verificado a la aplicación de Auth0: [Cómo configurar el dominio del publicador de una aplicación](https://docs.microsoft.com/en-us/azure/active-directory/develop/howto-configure-publisher-domain#configure-publisher-domain-using-the-azure-portal).
