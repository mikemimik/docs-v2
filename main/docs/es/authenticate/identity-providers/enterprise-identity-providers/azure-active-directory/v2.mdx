<Card title="Información general">
  Conceptos clave

  * Sigue los pasos para registrar tu aplicación en Azure AD y conectarla a tu instancia de Auth0.
  * Prueba la conexión antes de pasar la configuración a producción.
  * Revisa los consejos de solución de problemas.
</Card>

<Warning>
  **Importante:** atributos extendidos en conexiones de Azure Active Directory (v1) Dado que Microsoft está descontinuando y retirando Azure AD Graph API, las conexiones que usan la API de identidad de Azure Active Directory (v1) ya no pueden habilitar por primera vez Get extended profile, Get user groups ni Get nested groups (`options.ext_profile`, `options.ext_groups`, `options.ext_nested_groups`). En las conexiones v1 existentes, estas opciones solo se mantienen si ya estaban habilitadas, pero dejarán de funcionar por completo una vez que se complete el retiro de Microsoft Graph API en 2025. Los atributos extendidos siguen siendo compatibles con las conexiones que usan la API de identidad de Microsoft Identity Platform (v2), que depende de Microsoft Graph y requiere los Delegated permissions `User.Read` y `Directory.Read.All`.
</Warning>

Puedes integrarte con Microsoft Azure Active Directory (AD) si quieres permitir que los usuarios:

* De tu empresa usen tu aplicación desde un Azure AD controlado por ti o por tu organización.
* De Azure AD de otras empresas usen tu aplicación. (Te recomendamos configurar los directorios externos como conexiones distintas).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los claims que devuelve la conexión empresarial de Azure AD son estáticos; los claims personalizados u opcionales no aparecerán en los perfiles de usuario. Si necesitas incluir claims personalizados u opcionales en los perfiles de usuario, usa una conexión SAML u OIDC en su lugar.
</Callout>

<div id="register-your-app-with-azure-ad">
  ### Registra tu aplicación con Azure AD
</div>

Para registrar tu aplicación con Azure AD, consulta la guía de Microsoft [Quickstart: Register an application with the Microsoft identity platform](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-register-app).

<Warning>
  Si tienes más de un directorio de Azure AD, asegúrate de estar en el directorio correcto al registrar la aplicación que quieres usar con Auth0.
</Warning>

Durante el registro, configura las siguientes opciones:

| Option                      | Setting                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Supported account types** | Para habilitar el aprovisionamiento de usuarios basado en SCIM desde este directorio, selecciona **Accounts in this organizational directory only (Single tenant)**. Para permitir usuarios de organizaciones externas (como otros directorios de Azure AD), elige la opción multitenant adecuada. Las opciones multitenant incluyen la siguiente: **Accounts in any organizational directory (Any Azure AD directory - Multitenant)**. |
| **Redirect URI**            | Selecciona **Web** como tipo de Redirect URI e introduce tu URL de devolución de llamada: `https://{yourDomain}/login/callback`.                                                                                                                                                                                                                                                                                                        |

<Card title="Encuentra tu nombre de dominio de Auth0 para las redirecciones">
  Si tu nombre de dominio de Auth0 no aparece arriba y no estás usando nuestra funcionalidad de [dominios personalizados](/docs/es/customize/custom-domains), tu nombre de dominio es una concatenación del nombre de tu tenant, tu subdominio regional y `auth0.com`, separados por un punto (`.`).

  Por ejemplo, si el nombre de tu tenant es `exampleco-enterprises` y tu tenant está en la región de EE. UU., tu nombre de dominio de Auth0 sería `exampleco-enterprises.us.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.us.auth0.com/login/callback`.

  Sin embargo, si tu tenant está en la región de EE. UU. y se creó antes de junio de 2020, tu nombre de dominio de Auth0 sería `exampleco-enterprises.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.auth0.com/login/callback`.

  Si estás usando [dominios personalizados](/docs/es/customize/custom-domains), tu **Redirect URI** sería `https://<YOUR CUSTOM DOMAIN>/login/callback`.
</Card>

Durante este proceso, Microsoft genera un **ID de aplicación (cliente)** para tu aplicación; puedes encontrarlo en la pantalla **Información general** de la aplicación. **Toma nota de este valor.**

<div id="create-a-client-secret">
  #### Crear un secreto de cliente
</div>

Para crear un <Tooltip tip="Secret de cliente: Secret utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo el cliente y el servidor de autorización deben conocerlo, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>, consulta la documentación de Microsoft: [Microsoft&#39;s Add and manage application credentials in Microsoft Entra ID - Add a Client Secret](https://learn.microsoft.com/en-us/entra/identity-platform/how-to-add-credentials?source=recommendations\&tabs=client-secret).

Una vez generado, **guarda este valor**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si configuras un secreto con expiración, asegúrate de **anotar la fecha de expiración**; tendrás que renovar la clave antes de ese día para evitar una interrupción del servicio.
</Callout>

<div id="add-permissions">
  #### Agregar permisos
</div>

Para agregar permisos, consulta el [Quickstart: Configurar una aplicación cliente para acceder a API web - Agregar permisos para acceder a API web](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-configure-app-access-web-apis#add-permissions-to-access-web-apis) de Microsoft.

Al configurar los permisos, ten en cuenta lo siguiente:

Si quieres habilitar atributos extendidos (como **Perfil extendido** o **Grupos de seguridad**), tendrás que configurar los siguientes permisos en **Microsoft Graph API**.

| Delegated Permissions                 | Descripción                                                                                                                          |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **Users &gt; User.Read**              | Para que tu aplicación pueda permitir que los usuarios inicien sesión y leer los perfiles de los usuarios que hayan iniciado sesión. |
| **Directory &gt; Directory.Read.All** | Para que tu aplicación pueda leer datos del directorio en nombre del usuario que ha iniciado sesión.                                 |

<div id="create-an-enterprise-connection-in-auth0">
  ### Crear una conexión empresarial en Auth0
</div>

Cree y configure una conexión empresarial de Azure AD en Auth0. Asegúrese de tener el **ID de la aplicación (cliente)** y el **secreto de cliente** generados al configurar la aplicación en el portal de Microsoft Azure.

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), busque **Microsoft Azure AD** y seleccione su `+`.

   <Frame>![Dashboard - Connections - Enterprise](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Introduzca los detalles de la conexión y seleccione **Create**:

| Field                              | Description                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Connection name**                | Identificador lógico de su conexión; debe ser único para su tenant. Una vez establecido, este nombre no se puede cambiar.                                                                                                                                                                                                                                                                                                                    |
| **Microsoft Azure AD Domain**      | Su nombre de dominio de Azure AD. Puede encontrarlo en la página de información general de su directorio de Azure AD en el portal de Microsoft Azure.                                                                                                                                                                                                                                                                                        |
| **Client ID**                      | Identificador único de su aplicación registrada en Azure AD. Introduzca el valor guardado del **ID de la aplicación (cliente)** de la aplicación que acaba de registrar en Azure AD.                                                                                                                                                                                                                                                         |
| **Client Secret**                  | Cadena que se utiliza para obtener acceso a su aplicación registrada en Azure AD. Introduzca el valor guardado del **secreto de cliente** de la aplicación que acaba de registrar en Azure AD.                                                                                                                                                                                                                                               |
| **User ID Attribute Type**         | Atributo del token de ID que se asignará a la propiedad user&#95;id en Auth0. Las opciones corresponden a los dos tipos de ID admitidos descritos en la documentación de Microsoft: [ID token claims reference - Microsoft identity platform](https://learn.microsoft.com/en-us/entra/identity-platform/id-token-claims-reference#use-claims-to-reliably-identify-a-user).                                                                   |
| **Use common endpoint**            | (Opcional) Cuando está habilitado, su aplicación aceptará dinámicamente usuarios de nuevos directorios. Normalmente se habilita si seleccionó una opción multi-tenant para los tipos de cuenta admitidos de la aplicación que acaba de registrar en Azure AD. Auth0 redirige a los usuarios al Login endpoint común de Azure, y Azure realiza Home Realm Discovery en función del dominio de la dirección de correo electrónico del usuario. |
| **API de identidad**               | API que usa Auth0 para interactuar con los endpoints de Azure AD. Obtenga más información sobre las diferencias de comportamiento en la documentación de Microsoft [Why update to Microsoft identity platform (v2.0)](https://docs.microsoft.com/en-us/azure/active-directory/develop/azure-ad-endpoint-comparison).                                                                                                                         |
| **Attributes**                     | Atributos básicos del usuario que inició sesión a los que su aplicación puede acceder. Indica cuánta información desea almacenar en el perfil de usuario de Auth0.                                                                                                                                                                                                                                                                           |
| **Extended Attributes** (optional) | Atributos extendidos del usuario que inició sesión a los que su aplicación puede acceder.                                                                                                                                                                                                                                                                                                                                                    |
| **Auth0 APIs** (optional)          | Cuando se selecciona, indica que necesitamos poder realizar llamadas a la API de Azure AD, lo que nos permite buscar usuarios en Azure AD Graph aunque nunca hayan iniciado sesión en Auth0.                                                                                                                                                                                                                                                 |
| **Email Verification**             | Elija cómo Auth0 establece el campo `email_verified` en el perfil de usuario. Para obtener más información, lea [Email Verification for Azure AD and ADFS](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azuread-adfs-email-verification).                                                                                                                                                                             |
| **Purpose**                        | Habilite la conexión para autenticación, Connected Accounts for Token Vault o ambas. Para obtener más información, lea [User authentication vs Connected Accounts](/docs/es/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).                                                                                                                                                            |

3. En la vista de **Provisioning**, puede configurar cómo se crean y actualizan los perfiles de usuario en Auth0.

| Field                                          | Description                                                                                                                                                                                                                        |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Sync user profile attributes at each login** | Cuando está habilitado, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada inicio de sesión, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0. |
| **Sync user profiles using SCIM**              | Cuando está habilitado, Auth0 permite sincronizar los datos del perfil de usuario mediante SCIM. Para obtener más información, lea [Configure Inbound SCIM](/docs/es/authenticate/protocols/scim/configure-inbound-scim).             |

4. En la vista **Login Experience**, puede configurar cómo los usuarios inician sesión con esta conexión.

| Campo                              | Descripción                                                                                                                                                                                                                                                                |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Home Realm Discovery**           | Compara el dominio del correo electrónico de un usuario con los dominios proporcionados del proveedor de identidad. Para obtener más información, consulta [Configurar la autenticación Identifier First](/docs/es/authenticate/login/auth0-universal-login/identifier-first) |
| **Mostrar botón de conexión**      | Esta opción muestra las siguientes opciones para personalizar el botón de conexión de tu aplicación.                                                                                                                                                                       |
| **Button display name** (Opcional) | Texto que se utiliza para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón muestra: &quot;Continuar con &#123;Button display name&#125;&quot;.                                                                                  |
| **Button logo URL** (Opcional)     | URL de la imagen que se utiliza para personalizar el botón de inicio de sesión de Universal Login. Cuando se configura, el botón de inicio de sesión de Universal Login muestra la imagen como un cuadrado de 20 px por 20 px.                                             |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los campos opcionales solo están disponibles con Universal Login. Los clientes que usan Classic Login no verán el botón Add, Button display name ni Button logo URL.
</Callout>

5. Si tienes los permisos administrativos adecuados en Azure AD para otorgar consentimiento a la aplicación y permitir que los usuarios inicien sesión, haz clic en **Continuar**.
   Se te pedirá que [inicies sesión en tu cuenta de Azure AD](https://docs.microsoft.com/en-us/azure/active-directory/develop/v2-permissions-and-consent#requesting-consent-for-an-entire-tenant) y otorgues tu consentimiento. De lo contrario, proporciona la URL indicada a tu administrador para que pueda otorgarlo.

<div id="troubleshooting">
  ## Solución de problemas
</div>

Aquí tienes algunos consejos para la solución de problemas:

**Registré mi aplicación en Azure AD, pero cuando vuelvo a App registrations de Azure Active Directory, no puedo ver mi aplicación.**

Es posible que hayas registrado tu aplicación por error en el directorio de Azure AD equivocado (o que ni siquiera hayas creado un directorio de Azure AD antes de registrarla). Lo más sencillo probablemente sea volver a registrar tu aplicación en Azure AD. Asegúrate de estar en el directorio correcto cuando registres la aplicación. Si necesitas crear un directorio de Azure AD, sigue la guía de Microsoft [Quickstart: Create a new tenant in Azure Active Directory - Create a new tenant for your organization](https://docs.microsoft.com/en-us/azure/active-directory/fundamentals/active-directory-access-create-new-tenant#create-a-new-tenant-for-your-organization).

**Recibo el siguiente mensaje de error: &quot;Access cannot be granted to this service because the service listing is not properly configured by the publisher&quot;.**

Para resolverlo, intenta cambiar los **Supported account types** de tu aplicación registrada de Azure AD. Asegúrate de haber elegido una opción multitenant adecuada en la configuración de Authentication de la aplicación de Azure AD. Las opciones multitenant incluyen la siguiente: **Accounts in any organizational directory (Any Azure AD directory - Multitenant)**.

**Cuando los usuarios intentan iniciar sesión, recibimos el siguiente mensaje de error: &quot;invalid&#95;request; failed to obtain <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaque o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>&quot;.**

La causa más probable de este error es que el **secreto de cliente** de Azure AD no es válido o ha vencido. Para resolverlo, genera un nuevo **secreto de cliente** para tu aplicación en Azure AD y luego actualiza el **secreto de cliente** en la conexión empresarial configurada con Auth0.

<div id="signing-key-rollover-in-azure-ad">
  ## Rotación de la clave de firma en Azure AD
</div>

Las claves de firma las utiliza el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> para firmar el token de autenticación que emite, y la aplicación cliente (Auth0, en este caso) para validar la autenticidad del token generado.

Por motivos de seguridad, la clave de firma de Azure AD [rota periódicamente](https://docs.microsoft.com/en-us/azure/active-directory/develop/active-directory-signing-key-rollover). Si esto ocurre, **no necesitas hacer nada**. Auth0 usará la nueva clave automáticamente.

<div id="remove-unverified-label">
  ## Eliminar la etiqueta &quot;no verificado&quot;
</div>

Si usas un <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip>, la solicitud de consentimiento de la aplicación para el inicio de sesión en Azure AD puede mostrar tu dominio como &quot;no verificado&quot;. Para eliminar la etiqueta &quot;no verificado&quot;:

1. Verifica el dominio de la aplicación de Auth0: [Agrega tu dominio personalizado mediante el portal de Azure Active Directory](https://docs.microsoft.com/en-us/azure/active-directory/fundamentals/add-custom-domain#add-your-custom-domain-name-to-azure-ad).
2. Asigna el dominio verificado a la aplicación de Auth0: [Cómo: configurar el dominio del publicador de una aplicación](https://docs.microsoft.com/en-us/azure/active-directory/develop/howto-configure-publisher-domain#configure-publisher-domain-using-the-azure-portal).
