<div id="set-up-your-app-in-the-openid-connect-identity-provider">
  ## Configura tu aplicación en el proveedor de identidad de OpenID Connect
</div>

Para permitir que los usuarios inicien sesión con un proveedor de identidad OIDC, debes registrar tu aplicación en el IdP. El proceso varía según el proveedor de identidad OIDC, por lo que tendrás que seguir la documentación de tu IdP para completar esta tarea.

En general, debes asegurarte de ingresar en algún momento tu URL de devolución de llamada: `https://{YOUR_AUTH0_DOMAIN}/login/callback`.

<Card title="Encuentra tu nombre de dominio de Auth0 para las redirecciones">
  Si tu nombre de dominio de Auth0 no aparece arriba y no estás usando nuestra función de [dominios personalizados](/docs/es/customize/custom-domains), tu nombre de dominio se compone del nombre de tu tenant, tu subdominio regional y `auth0.com`, separados por el símbolo de punto (`.`).

  Por ejemplo, si el nombre de tu tenant es `exampleco-enterprises` y tu tenant está en la región de EE. UU., tu nombre de dominio de Auth0 sería `exampleco-enterprises.us.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.us.auth0.com/login/callback`.

  Sin embargo, si tu tenant está en la región de EE. UU. y se creó antes de junio de 2020, tu nombre de dominio de Auth0 sería `exampleco-enterprises.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.auth0.com/login/callback`.

  Si estás usando [dominios personalizados](/docs/es/customize/custom-domains), tu **Redirect URI** sería `https://<YOUR CUSTOM DOMAIN>/login/callback`.
</Card>

Durante este proceso, tu proveedor de identidad OIDC generará un identificador único para la API registrada, normalmente llamado **<Tooltip tip="ID de cliente: Valor de identificación asignado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** o **ID de la aplicación**. Toma nota de este valor; lo necesitarás más adelante.

<div id="create-an-enterprise-connection-in-auth0">
  ## Crear una conexión empresarial en Auth0
</div>

Antes de configurar una conexión empresarial de OIDC en Auth0, asegúrate de tener el **ID de la aplicación (cliente)** y el **<Tooltip tip="Secreto de cliente: secreto que usa un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=Client+secret">secreto de cliente</Tooltip>** que se generan al configurar tu aplicación en el proveedor de OIDC.

<div id="create-an-enterprise-connection-using-the-auth0-dashboard">
  ### Crear una conexión empresarial con Auth0 Dashboard
</div>

<Warning>
  Para que pueda configurarse a través de Auth0 Dashboard, el proveedor de identidad (IdP) de OpenID Connect (OIDC) debe admitir [OIDC Discovery](https://openid.net/specs/openid-connect-discovery-1_0.html). De lo contrario, puede [configurar la conexión mediante la Management API](#configure-the-connection-using-the-management-api).

<div id="create-an-enterprise-connection-using-the-management-api">
  ### Crear una conexión empresarial con la Management API
</div>

Estos ejemplos le mostrarán distintas formas de crear la conexión con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0. Puede configurar la conexión proporcionando un URI de metadatos o definiendo explícitamente las URL de OIDC. Para obtener más información, consulte [Proveedores de identidad](/docs/es/authenticate/identity-providers).

<div id="use-front-channel-with-discovery-endpoint">
  #### Usar front-channel con el endpoint de descubrimiento
</div>

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/api/v2/connections' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --data '{ "strategy": "oidc", "name": "CONNECTION_NAME", "options": { "type": "front_channel", "discovery_url": "https://IDP_DOMAIN/.well-known/openid-configuration", "client_id" : "IDP_CLIENT_ID",  "scopes": "openid profile" } }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/connections");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddParameter("undefined", "{ "strategy": "oidc", "name": "CONNECTION_NAME", "options": { "type": "front_channel", "discovery_url": "https://IDP_DOMAIN/.well-known/openid-configuration", "client_id" : "IDP_CLIENT_ID",  "scopes": "openid profile" } }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/connections"

  	payload := strings.NewReader("{ "strategy": "oidc", "name": "CONNECTION_NAME", "options": { "type": "front_channel", "discovery_url": "https://IDP_DOMAIN/.well-known/openid-configuration", "client_id" : "IDP_CLIENT_ID",  "scopes": "openid profile" } }")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse<String> response = Unirest.post("https://{yourDomain}/api/v2/connections")
    .header("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
    .body("{ "strategy": "oidc", "name": "CONNECTION_NAME", "options": { "type": "front_channel", "discovery_url": "https://IDP_DOMAIN/.well-known/openid-configuration", "client_id" : "IDP_CLIENT_ID",  "scopes": "openid profile" } }")

  ```


<div id="federate-with-auth0">
  ## Federarse con Auth0
</div>

La conexión empresarial de <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect es útil para federarse con otro tenant de Auth0. Introduce la URL de tu tenant de Auth0 (por ejemplo, `https://<tenant>.us.auth0.com`) en el campo **Issuer** e introduce el ID de cliente de cualquier aplicación del tenant al que quieras federarte en el campo **ID de cliente**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los tenants nuevos incluirán `us` como parte de la URL. Los tenants creados antes de que se añadieran los dominios regionales seguirán funcionando. Por ejemplo, `https://{YOUR ACCOUNT}.auth0.com`.
</Callout>

<div id="configure-global-token-revocation">
  ## Configurar la revocación global de tokens
</div>

Este tipo de conexión admite un endpoint de revocación global de tokens, que permite a un proveedor de identidad compatible revocar las sesiones de usuario de Auth0, revocar <Tooltip tip="Token de actualización: token usado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> y activar el logout por back-channel en las aplicaciones que usan un back-channel seguro.

Esta función puede usarse con Universal Logout en Okta Workforce Identity.

Para obtener más información e instrucciones de configuración, consulta [Universal Logout](/docs/es/authenticate/login/logout/universal-logout).
