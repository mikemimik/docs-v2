Auth0 admite inicio de sesión con GitHub, lo que permite a tus usuarios autenticarse con sus cuentas de GitHub mediante <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>. Añadir GitHub como opción de Social login ofrece una experiencia de inicio de sesión rápida y familiar para desarrolladores y usuarios técnicos.

Para configurar esta conexión, primero registras una aplicación OAuth en GitHub y después configuras la conexión en el Auth0 Dashboard con las credenciales que proporciona GitHub.

<Card title="Antes de empezar">
  Antes de configurar GitHub como una conexión social, necesitas:

  1. Una [cuenta de Auth0](https://auth0.com/signup). Si no tienes una, puedes registrarte gratis.
  2. Una [cuenta de GitHub](https://github.com/signup).
  3. Una aplicación registrada en el [Auth0 Dashboard](https://manage.auth0.com/#/applications).
</Card>

<div id="configure-github">
  ## Configurar GitHub
</div>

Cree una aplicación OAuth en GitHub para generar las credenciales que Auth0 necesita para establecer la conexión.

<Steps>
  <Step title="Abrir GitHub Developer Settings">
    Inicie sesión en su cuenta de GitHub. Seleccione su foto de perfil en la esquina superior derecha y luego **Settings**. En la barra lateral izquierda, seleccione **Developer settings** y, a continuación, **OAuth Apps**.
  </Step>

  <Step title="Registrar una nueva aplicación OAuth">
    Seleccione **New OAuth App**. Si aún no ha creado ninguna aplicación, seleccione **Register a new application**.

    Complete los siguientes campos:

    | **Campo**                      | **Valor**                                                                                        |
    | ------------------------------ | ------------------------------------------------------------------------------------------------ |
    | **Application name**           | Un nombre que sus usuarios reconozcan y en el que confíen, por ejemplo `My App (Auth0)`          |
    | **Homepage URL**               | La URL completa de la página principal de su aplicación, por ejemplo `https://myapp.example.com` |
    | **Application description**    | (Opcional) Una descripción que se muestra a todos los usuarios de su aplicación                  |
    | **Authorization callback URL** | `https://YOUR_AUTH0_DOMAIN/login/callback`                                                       |

    Reemplace `YOUR_AUTH0_DOMAIN` por el dominio de su tenant de Auth0. Puede encontrar este valor en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), en la pestaña **Settings**.

    Puede dejar **Enable Device Flow** sin marcar, a menos que su aplicación requiera autorización basada en dispositivos.

    Seleccione **Register application**.
  </Step>

  <Step title="Copiar el ID de cliente y el secreto de cliente">
    Después de registrarla, GitHub muestra la página de detalles de la aplicación. Copie el valor de **Client ID**.

    Seleccione **Generate a new client secret** para crear un secreto. Copie el valor generado de inmediato.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      GitHub muestra el secreto de cliente solo una vez. Guárdelo en un lugar seguro antes de salir de la página. Si lo pierde, deberá generar uno nuevo.
    </Callout>
  </Step>
</Steps>

<div id="configure-auth0">
  ## Configurar Auth0
</div>

Añade la conexión de GitHub a tu tenant de Auth0 e introduce las credenciales de la sección anterior.

<Steps>
  <Step title="Crear una conexión de GitHub">
    Ve a [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/social) y selecciona **Create Connection**. Selecciona **GitHub** de la lista de proveedores.
  </Step>

  <Step title="Selecciona el propósito de la conexión">
    En **Purpose**, especifica cómo se usará esta conexión: para inicio de sesión, Connected Accounts o ambos.
  </Step>

  <Step title="Introduce tus credenciales de GitHub">
    En **General**, introduce el **ID de cliente** y el **secreto de cliente** que copiaste de GitHub.
  </Step>

  <Step title="Selecciona los permisos">
    En **Permissions**, selecciona los permisos que requiere tu aplicación. La mayoría de las aplicaciones necesitan **Email address** para el correo electrónico del usuario y **Read user** para la información del perfil.
  </Step>

  <Step title="Guarda y habilita la conexión">
    Selecciona **Create** para guardar la conexión.

    Selecciona la pestaña **Applications** y habilita la conexión para las aplicaciones que deban ofrecer inicio de sesión con GitHub.
  </Step>
</Steps>

<div id="test-the-connection">
  ## Pruebe la conexión
</div>

Verifique que la conexión de GitHub funcione antes de implementarla en producción.

1. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/social).
2. Seleccione la conexión de GitHub de la lista.
3. Seleccione **Probar conexión**.
4. Autentíquese con su cuenta de GitHub cuando se le pida.
5. Confirme que Auth0 devuelve datos del perfil del usuario, como su nombre de usuario de GitHub y su dirección de correo electrónico.
