title: Gestionar los permisos de autorización rechazados
---

Cuando tus usuarios autorizan tu aplicación, algunos proveedores (como Facebook) permiten que el usuario seleccione los atributos que desea compartir.

De forma predeterminada, esta selección solo se realiza cuando el usuario autoriza la aplicación por primera vez. Si tu usuario decide no permitir ciertos atributos (como su correo electrónico) que tu aplicación requiere, el usuario no podrá acceder a tu aplicación.

En esos casos, habrá que volver a mostrar un prompt al usuario para que otorgue permiso para los atributos requeridos y pueda iniciar sesión.

<div id="re-prompt-for-permissions">
  ## Solicitar de nuevo los permisos
</div>

Al establecer el parámetro **prompt=consent** cuando llames al endpoint [/authorize](https://auth0.com/docs/api/authentication/reference#social) de la [API de autorización](https://auth0.com/docs/api/authentication), se le volverá a pedir al usuario que conceda permisos a tu aplicación.

Este parámetro también puede establecerse con Lock como un [Parámetro de autenticación](/docs/es/libraries/lock/lock-authentication-parameters) con **prompt: &#39;consent&#39;**.

También puedes establecerlo con [Auth0.js](https://github.com/auth0/auth0.js) usando **prompt: &#39;consent&#39;**.
