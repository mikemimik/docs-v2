<Card title="Antes de empezar">
  Antes de empezar a configurar tu aplicación para usar TikTok como conexión social, necesitas lo siguiente:

  1. Una [cuenta de desarrollador de TikTok](https://developer.tiktok.com/).
  2. Una cuenta de Auth0 con una aplicación que use Auth0 para autenticar a tus usuarios.
  3. Una URL de tu página de Términos de servicio para el [proceso de revisión de aplicaciones de TikTok](https://developers.tiktok.com/doc/getting-started-faq/).
  4. [Node.js](https://nodejs.org/en/) y [ngrok](https://ngrok.com/) instalados en tu entorno local.
</Card>

<div id="configure-tiktok">
  ## Configurar TikTok
</div>

Puedes usar TikTok como inicio de sesión social en tu aplicación. Accede a tu cuenta de desarrollador de TikTok y configúrala siguiendo estos pasos:

1. En TikTok developer, selecciona **Manage apps**.
2. Selecciona **Connect an app**.
3. En la sección **configuración**, agrega un icono, el nombre de la aplicación y una descripción.
4. En **Platforms**, elige tu tipo de aplicación:

   1. Para aplicaciones web, agrega una URL válida.
   2. Para Android, agrega el nombre del paquete, la URL de Play Store y la(s) firma(s) de la aplicación.
   3. Para iOS, agrega la URL de App Store y el Bundle ID.
5. En el menú Product, selecciona **Add Product**.
6. Selecciona **Login Kit**.
7. Luego, selecciona la **TikTok API**.
8. En la sección Product, agrega la URL de tu página de Términos de servicio, la URL de tu página de Política de privacidad y tu dominio de redirección a Login Kit. El dominio de redirección es tu dominio de Auth0, que encontrarás en [Dashboard &gt; Applications &gt; Applications](http://manage.auth0.com/#/applications), en la pestaña **Settings**. Por ejemplo: `dev-test-1.us.auth0.com`.
9. Selecciona **Save Changes**. Luego, selecciona **Submit for review**.
10. Espera hasta que el estado de tu aplicación cambie de `Staging` a `Production`. TikTok puede tardar varias horas en revisar tu aplicación y actualizar el estado.

<div id="configure-auth0">
  ## Configurar Auth0
</div>

Debes crear una conexión personalizada para asociar tu instancia de TikTok con Auth0.

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Social](http://manage.auth0.com/#/social).
2. Elige **Create Connection**.
3. Desplázate hasta el final de la lista y elige **Create Custom**.
4. Introduce lo siguiente para crear una **nueva conexión social personalizada**:

   1. Nombre: TikTok
   2. URL de autorización: la URL de autorización de TikTok `https://www.tiktok.com/auth/authorize/`
   3. URL del token: con el tiempo, esta será tu proxy. Usa una URL provisional: `https://example.com`
   4. Scope: `user.info.basic`
   5. ID de cliente: clave de cliente asignada por TikTok
   6. Secreto de cliente: secreto de cliente asignado por TikTok
   7. Purpose: habilita la conexión para Authentication, Connected Accounts for Token Vault o ambas. Para obtener más información, consulta [autenticación de usuario vs. Connected Accounts](/docs/es/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).
5. Configura el [Fetch User Profile Script](/docs/es/authenticate/identity-providers/social-identity-providers/oauth2#fetch-user-profile-script) para obtener información del perfil desde el [endpoint user&#95;info de TikTok](https://developers.tiktok.com/doc/tiktok-api-v2-get-user-info/). Asigna los atributos al perfil de usuario normalizado de Auth0.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Auth0 solo requiere el atributo `user_id`, que corresponde al `union_id` de TikTok.
   </Callout>

```javascript lines
function fetchUserProfile(accessToken, context, cb) {
  const axios = require('axios@0.22.0');
  const userInfoEndpoint = 'https://open.tiktokapis.com/v2/user/info?fields=union_id';
  const headers = { 'Authorization': 'Bearer ' + accessToken };

  axios
    .get(userInfoEndpoint, { headers })
    .then(res => {
      if (res.status !== 200) {
        return cb(new Error(res.data));
      }

      const profile = {
        user_id: res.data.user.union_id,
      };

      cb(null, profile);
    })
    .catch(err => cb(err));
}
```

6. Haz clic en **Crear**.

7. Dirígete a la aplicación que quieras usar con la conexión de TikTok en [Dashboard &gt; Applications &gt; Applications](http://manage.auth0.com/#/applications).

8. En la pestaña **Conexiones**, activa la opción de TikTok.

<div id="pass-customs-parameters-to-tiktok-with-the-management-api">
  ## Pasa parámetros personalizados a TikTok con la Management API
</div>

Como TikTok usa un parámetro `client_key` en lugar de `client_id`, debes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para pasar el parámetro `client_key` durante la autenticación.

Para usar la Management API, debes generar un <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>.

1. Ve al [Dashboard &gt; Applications &gt; APIs](http://manage.auth0.com/#/apis) de Auth0 y selecciona la **Auth0 Management API**.
2. Selecciona la pestaña **API Explorer**.
3. Selecciona **Create &amp; Authorize Test Application.**
4. Copia el token proporcionado.
5. Ve al [Auth0 Management API Explorer](https://auth0.com/docs/api/management/v2). Puede que necesites abrir una ventana de incógnito.
6. Selecciona **Set API Token** en la esquina superior izquierda.

   <Frame>![Captura de pantalla de Set API Token para TikTok](/docs/images/cdy7uua7fh8z/6994KjTIdvDNrye8ghN4og/a6d1eb83bce4a5b38f8ebce7188ee2ca/image2.png)</Frame>
7. Pega el token y selecciona **Set Token**.

Ahora deberías poder configurar tu tenant de Auth0 con la Management API.

<div id="configure-the-client_key-field">
  ### Configurar el campo `client_key`
</div>

1. Usa el método [obtener una conexión](https://auth0.com/docs/api/management/v2#!/Connections/get_connections_by_id) para obtener los valores del objeto `options`. A continuación se muestra un ejemplo de objeto de respuesta:

```json lines
{
  "options": {
    "client_id": "",
    "client_secret": "",
    "scope": "user.info.basic"
  }
}
```

2. Añade el objeto `upstream_params` con el campo `client_key`:

```json lines
{
  "options": {
    "client_id": "",
    "client_secret": "",
    "scope": "user.info.basic",
    "upstream_params": { 
      "client_key": { "value": "<Client Key from TikTok>" } 
    }
  }
}
```

3. Usa el método [actualizar una conexión](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id) usando el objeto `options` como cuerpo de la solicitud. Auth0 enviará el parámetro `client_key=<value>` al [endpoint de autorización de TikTok](https://developers.tiktok.com/doc/login-kit-web/).

<div id="access-token-request">
  ## Solicitud de token de acceso
</div>

No puedes enviar parámetros personalizados en una solicitud al endpoint [`/token`](https://developers.tiktok.com/doc/login-kit-manage-user-access-tokens/) de Authentication API para obtener un token de acceso o un <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip>. Debes enviar la solicitud mediante un proxy al <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=token+endpoint">endpoint de token</Tooltip> y añadir el parámetro `client_key` de forma programática mediante endpoints proxy en tu entorno.

<div id="deploy-tiktok-integration-proxy">
  ### Desplegar el proxy de integración de TikTok
</div>

1. Usa el código de ejemplo del repositorio de [GitHub](https://github.com/tylerkeesling/tiktok-access-token-proxy) y sigue las instrucciones del README para instalar las dependencias e iniciar el servidor de desarrollo.

   1. El servidor de ejemplo tiene una ruta `POST`, `/proxy/token`. El servidor debería estar ejecutándose en `http://localhost:3333`.
2. Copia el endpoint del proxy para usarlo en tu configuración de TikTok Developer. El endpoint del proxy debería ser similar a `https://405a-104-129-13b-250.ngrok.io/proxy/token`.
3. En TikTok Developer, vuelve a la configuración de la conexión social. Actualiza la URL del token que configuraste como `https://example.com` e introduce la URL del proxy.

Una vez guardada la configuración, tus usuarios deberían poder iniciar sesión con TikTok.
