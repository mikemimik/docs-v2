Puede agregar funcionalidad a su aplicación nativa para permitir que sus usuarios se autentiquen con Facebook de forma nativa, dentro de la aplicación. Esto no requiere redirección a través de un navegador web y permitirá que las aplicaciones móviles cumplan con la [Facebook Developer Policy](https://developers.facebook.com/policy/), que exige que las aplicaciones móviles usen el SDK de Facebook para [Android](https://developers.facebook.com/docs/android) o [iOS](https://developers.facebook.com/docs/ios) para autenticarse.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Al integrar los SDKs de Facebook, sus aplicaciones compartirán datos con Facebook. Asegúrese de comprender qué datos se comparten y de reflejarlos correctamente en la política de privacidad de su aplicación. Auth0 no tiene control sobre qué datos se compartirán con Facebook a través del SDK.

  Consulte la página de [Facebook RGPD](https://www.facebook.com/business/m/one-sheeters/gdpr-developer-faqs) para obtener más información sobre los datos recopilados por el SDK de Facebook y Facebook Login.
</Callout>

<div id="how-it-works">
  ## Cómo funciona
</div>

El flujo de inicio de sesión nativo de Facebook funciona de la siguiente manera:

* **Paso 1**: La aplicación autentica a un usuario mediante el SDK de Facebook y obtiene un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip>.
* **Paso 2**: La aplicación usa ese token de acceso para solicitar un [Facebook Session Info Access Token](https://developers.facebook.com/docs/facebook-login/access-tokens/session-info-access-token) especial.
* **Paso 3**: Usa el SDK de Facebook para recuperar el perfil del usuario.
* **Paso 4**: La aplicación puede usar entonces el Facebook Session Info Access Token para autenticarse con Auth0.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de configurar el inicio de sesión nativo de Facebook para su aplicación nativa mediante Auth0, debe:

1. [Configurar Facebook como una conexión de Auth0](https://marketplace.auth0.com/integrations/facebook-social-connection)
2. [Usar el SDK de Facebook correspondiente en su aplicación](https://developers.facebook.com/docs/apis-and-sdks/)
3. Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y cree una aplicación en Auth0 (si aún no lo ha hecho).
4. Al final de la página de configuración, seleccione **Show Advanced Settings** y luego la vista **Device Settings**. En **Native Social Login**, active el interruptor **Enable Sign In with Facebook**.

<div id="implementation">
  ## Implementación
</div>

El proceso para autenticar el perfil de un usuario con el inicio de sesión nativo de Facebook consta de cuatro pasos desde la perspectiva de tu aplicación:

<div id="step-1">
  ### Paso 1
</div>

La aplicación autentica al usuario a través del SDK de Facebook. Obtendrá un token de acceso de Facebook.

<div id="step-2">
  ## Paso 2
</div>

La aplicación utiliza el token de acceso para solicitar un [Facebook Session Info Access Token](https://developers.facebook.com/docs/facebook-login/access-tokens/session-info-access-token).

Esta solicitud será similar a la siguiente:

```json wrap lines
GET https://graph.facebook.com/v5.0/oauth/access_token?grant_type=fb_attenuate_token&client_id=457704041391802&fb_exchange_token=<facebook_access_token>
```

y la respuesta:

```json lines
{
    "access_token": "XAAGgR4b...1lHWNCpqrAhcpoAZDZD",
    "token_type": "bearer",
    "expires_in": 5183924
}
```

<div id="step-3">
  ### Paso 3
</div>

La aplicación debe recuperar el perfil del usuario desde Facebook mediante el SDK de Facebook, lo que dará lugar a una solicitud similar a la siguiente:

```json wrap lines
GET https://graph.facebook.com/v5.0/<facebook user id>?access_token=<facebook access token>&fields=email,name
```

<div id="step-4">
  ### Paso 4
</div>

La aplicación puede usar entonces el Facebook Session Info Access Token y el perfil de usuario de Facebook para autenticarse en Auth0 llamando al endpoint `/oauth/token` de Auth0 mediante el flujo de intercambio de tokens con el tipo de token `facebook-session-access-token`. Si todo sale bien, Auth0 devolverá una respuesta normal del intercambio, además del perfil de usuario. El perfil de usuario debe ser un objeto JSON codificado como una cadena.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample = `POST https://{yourDomain}/oauth/token

grant_type: 'urn:ietf:params:oauth:grant-type:token-exchange'
subject_token_type: 'http://auth0.com/oauth/token-type/facebook-info-session-access-token'
audience: 'your-api'
scope: 'read:appointments openid profile email email_verified'
subject_token: 'XAAGgR4b...1lHWNCpqrUHZAEtUuZAhcpoAZDZD'
client_id: '{yourClientId}'
user_profile: '{"email":"john@example.com", "name":"John Doe"}'`;

<AuthCodeBlock children={codeExample} language="json" />

y la respuesta de Auth0:

```json lines
{
    "access_token": "eyJ0eXA..yXQaPLVXg",
    "id_token": "eyJ0.tFE5HPipdOsA",
    "scope": "openid profile email read:appointments",
    "expires_in": 86400,
    "token_type": "Bearer"
}
```

<div id="user-profile-and-email-validation">
  ## Perfil de usuario y validación del correo electrónico
</div>

En el ejemplo anterior, fue necesario recuperar el perfil de usuario de Facebook e incluirlo en la llamada a `/oauth/token`. Esto se debe a que el Facebook Session Info Access Token no puede usarse para recuperar directamente el perfil, y el token de acceso de Facebook no puede enviarse directamente al servidor debido a las [AppStore Review Guidelines de Apple](https://developer.apple.com/app-store/review/guidelines). Por lo tanto, debe recuperarse en el cliente y enviarse a Auth0 de este modo.

Dado que Auth0 no puede garantizar que el perfil de usuario sea el mismo que devolvió Facebook, establecerá el campo `email_verified` en `false`.

<div id="logout">
  ## Cierre de sesión
</div>

Dado que la implementación del inicio de sesión nativo no utiliza los flujos estándar basados en navegador, los responsables de la aplicación también deben asegurarse de realizar correctamente el cierre de sesión. Cuando una aplicación necesite cerrar la sesión, también debe [revocar el token de actualización de Auth0](https://auth0.com/docs/api/authentication#revoke-refresh-token).
