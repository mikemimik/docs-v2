<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  MFA no es compatible con las transacciones de Custom Token Exchange en las que se establece un actor mediante `api.authentication.setActor()`. Si se requiere MFA (mediante una política o una Action de Post Login) y la Action de Custom Token Exchange establece un actor, la transacción falla con un error `400`: `MFA is not supported using actor_token with the requested token exchange profile.`
</Callout>

Para agregar protección contra el robo de tokens u otros riesgos de seguridad, puedes agregar autenticación multifactor a una solicitud de Custom Token Exchange usando uno de los siguientes métodos:

* [Habilita y configura tu política de MFA ](/docs/es/secure/multi-factor-authentication/enable-mfa) para tu tenant
* Usa `api.multifactor.enable()` en una Action de Post Login

Cuando agregas MFA, el servidor de autorización de Auth0 rechaza la llamada inicial de Custom Token Exchange al endpoint `/token` con un error `mfa_required`. Puedes usar ese error en la [API de MFA de Auth0](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api) para solicitar y verificar un factor a fin de obtener los tokens de acceso, ID y de actualización de Auth0 solicitados.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Custom Token Exchange no admite `api.authentication.challengeWith()` ni `api.authentication.enrollWith()`. Si usas esos métodos con tu Action de Post Login, la transacción fallará con un error no recuperable. Asegúrate de no usar esos métodos cuando `event.transaction.protocol==oauth2-token-exchange`, según el valor de `subject_token_type`.
</Callout>

Cuando una transacción de Custom Token Exchange está asociada a una organización, `api.multifactor.enable()` y las políticas de MFA tampoco son compatibles.

Para obtener más información sobre cómo usar la concesión de MFA, lee [Autenticar usando el flujo de contraseña del propietario del recurso con MFA](/docs/es/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa), ya que Custom Token Exchange sigue el mismo modelo. También puedes consultar un ejemplo detallado en [Caso de uso: Realizar MFA durante Custom Token Exchange](/docs/es/authenticate/custom-token-exchange/cte-example-use-cases#use-case%3A-perform-mfa-during-custom-token-exchange).
