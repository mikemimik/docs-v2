SDK de Auth0 para aplicaciones de página única es una nueva biblioteca de JavaScript para implementar la autenticación y la autorización en aplicaciones de una sola página (SPA) con Auth0. Proporciona una API de alto nivel y se encarga de muchos de los detalles para que puedas proteger las SPA siguiendo las prácticas recomendadas y escribiendo menos código.

Auth0 SPA SDK gestiona los detalles de la concesión y del protocolo, la expiración y renovación de tokens, así como el almacenamiento de tokens y el almacenamiento en caché. Internamente, implementa [Universal Login](/docs/es/authenticate/login/auth0-universal-login) y el [flujo de concesión Authorization Code con PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

La [biblioteca](https://github.com/auth0/auth0-spa-js) y la [documentación de la API](https://auth0.github.io/auth0-spa-js/) están alojadas en GitHub.

Si encuentras algún problema o error al usar el nuevo SDK de JavaScript, [consulta las preguntas frecuentes](https://github.com/auth0/auth0-spa-js/blob/master/FAQ.md) para comprobar si tu problema aparece allí.

<div id="installation">
  ## Instalación
</div>

Tienes varias opciones para usar el Auth0 SPA SDK en tu proyecto:

* Desde el CDN: `<script src="https://cdn.auth0.com/js/auth0-spa-js/2.0/auth0-spa-js.production.js"></script>`. Para más información, consulta las [FAQ](https://github.com/auth0/auth0-spa-js/blob/main/FAQ.md#how-to-use-from-a-cdn).
* Con [npm](https://npmjs.org): `npm install @auth0/auth0-spa-js`
* Con [yarn](https://yarnpkg.com): `yarn add @auth0/auth0-spa-js`

<div id="create-the-client">
  ### Crear el cliente
</div>

Primero, tendrás que crear una nueva instancia del objeto cliente `Auth0Client`. Crea la instancia de `Auth0Client` antes de renderizar o inicializar tu aplicación. Puedes hacerlo con el método async/await o con promesas. Solo debes crear una instancia del cliente.

Usar `createAuth0Client` hace un par de cosas automáticamente:

* Crea una instancia de `Auth0Client`.
* Llama a `getTokenSilently` para renovar la sesión del usuario.
* Suprime todos los errores de `getTokenSilently`, excepto `login_required`.

<div id="use-asyncawait">
  #### Usar async/await
</div>

export const codeExample1 = `import { createAuth0Client } from '@auth0/auth0-spa-js';

const auth0 = await createAuth0Client({
  domain: '{yourDomain}',
  clientId: '{yourClientId}'
});`;

<AuthCodeBlock children={codeExample1} language="jsx" />

#### Usa promesas

export const codeExample2 = `createAuth0Client({
  domain: '{yourDomain}',
  clientId: '{yourClientId}'
}).then(auth0 => {
  //...
});`;

<AuthCodeBlock children={codeExample2} language="jsx" />

También puedes crear el cliente directamente con el constructor `Auth0Client`. Esto puede resultar útil si quieres:

* Omitir la llamada a `getTokenSilently` durante la inicialización.
* Gestionar errores de forma personalizada.
* Inicializar el SDK de forma síncrona.

export const codeExample3 = `import { Auth0Client } from '@auth0/auth0-spa-js';

const auth0 = new Auth0Client({
  domain: '{yourDomain}',
  clientId: '{yourClientId}'
});`;

<div id="login-and-get-user-info">
  ### Iniciar sesión y obtener información del usuario
</div>

A continuación, crea un botón en el que los usuarios puedan hacer clic para empezar el proceso de inicio de sesión:

`<button id="login">Click to Login</button>`

Escucha los eventos de clic del botón que creaste. Cuando se produzca el evento, usa el método de login que prefieras para autenticar al usuario (`loginWithRedirect()` en este ejemplo). Después de autenticar al usuario, puedes recuperar su perfil con el método `getUser()`.

#### Usa async/await

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.loginWithRedirect({
    authorizationParams: {
      redirect_uri: 'http://localhost:3000/'
    }
  });
  //sesión iniciada. puedes obtener el perfil de usuario así:
  const user = await auth0.getUser();
  console.log(user);
});
```

#### Usa promesas

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.loginWithRedirect({
    authorizationParams: {
      redirect_uri: 'http://localhost:3000/'
    }
  }).then(token => {
    //sesión iniciada. puedes obtener el perfil de usuario así:
    auth0.getUser().then(user => {
      console.log(user);
    });
  });
});
```

<div id="call-an-api">
  ### Llamar a una API
</div>

Para llamar a tu API, primero obtén el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> del usuario. Luego, usa el token de acceso en tu solicitud. En este ejemplo, se utiliza el método `getTokenSilently` para recuperar el token de acceso:

`<button id="callApi">Llamar a una API</button>`

#### Usa async/await

```jsx lines
document.getElementById('callApi').addEventListener('click', async () => {
  const accessToken = await auth0.getTokenSilently();
  const result = await fetch('https://exampleco.com/api', {
    method: 'GET',
    headers: {
      Authorization: 'Bearer ' + accessToken
    }
  });
  const data = await result.json();
  console.log(data);
});
```

<div id="use-promises">
  #### Usar promesas
</div>

```jsx lines
document.getElementById('callApi').addEventListener('click', () => {
  auth0
    .getTokenSilently()
    .then(accessToken =>
      fetch('https://exampleco.com/api', {
        method: 'GET',
        headers: {
          Authorization: 'Bearer ' + accessToken
        }
      })
    )
    .then(result => result.json())
    .then(data => {
      console.log(data);
    });
});
```

Añade un botón para que los usuarios puedan cerrar sesión:

`<button id="logout">Logout</button>`

```jsx lines
$('#logout').click(async () => {
  auth0.logout({
    logoutParams: {

    }
  });

```

<div id="change-storage-options">
  ### Cambiar las opciones de almacenamiento
</div>

De manera predeterminada, Auth0 SPA SDK almacena los tokens en memoria. Sin embargo, esto no ofrece persistencia entre recargas de página y pestañas del navegador. En su lugar, puedes optar por almacenar los tokens en el almacenamiento local estableciendo la propiedad `cacheLocation` en `localstorage` al inicializar el SDK. Esto puede ayudar a mitigar algunos de los efectos de las tecnologías de privacidad del navegador que impiden el acceso a la cookie de Auth0 <Tooltip tip="Cookie de sesión: entidad que, cuando está presente, permite considerar autenticado al usuario." cta="Ver glosario" href="/docs/es/glossary?term=session+cookie">cookie de sesión</Tooltip> al almacenar tokens de acceso durante más tiempo.

<Warning>
  Almacenar tokens en el almacenamiento local del navegador ofrece persistencia entre recargas de página y pestañas del navegador. Sin embargo, si un atacante logra ejecutar JavaScript en la SPA mediante un ataque de cross-site scripting (XSS), puede recuperar los tokens almacenados en el almacenamiento local. Una vulnerabilidad que desemboque en un ataque XSS exitoso puede estar en el código fuente de la SPA o en cualquier código JavaScript de terceros (como Bootstrap, jQuery o Google Analytics) incluido en la SPA.

  Más información sobre el [almacenamiento de tokens](/docs/es/secure/tokens/token-best-practices).
</Warning>

export const codeExample4 = `const auth0 = await createAuth0Client({
  domain: '{yourDomain}',
  clientId: '{yourClientId}',
  cacheLocation: 'localstorage'
});`;

<div id="use-rotating-refresh-tokens">
  ### Usa tokens de actualización con rotación
</div>

El Auth0 SPA SDK se puede configurar para usar [tokens de actualización con rotación](/docs/es/secure/tokens/refresh-tokens/refresh-token-rotation) para obtener nuevos tokens de acceso de forma silenciosa. Se pueden usar para eludir la tecnología de privacidad del navegador que impide el acceso a la cookie de sesión de Auth0 al autenticarse de forma silenciosa, además de ofrecer detección de reutilización integrada.

Configura el SDK para ello estableciendo `useRefreshTokens` en `true` durante la inicialización:

export const codeExample5 = `const auth0 = await createAuth0Client({
  domain: '{yourDomain}',
  clientId: '{yourClientId}',
  useRefreshTokens: true
});

// Solicita un nuevo token de acceso con un token de actualización
const token = await auth0.getTokenSilently();`;

<AuthCodeBlock children={codeExample5} language="jsx" />

<Tooltip tip="Token de actualización: token usado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Tokens">Tokens de actualización</Tooltip> también deberán [configurarse para su tenant](/docs/es/secure/tokens/refresh-tokens/configure-refresh-token-rotation) antes de poder usarse en su SPA.

Una vez configurados, el SDK solicitará el scope `offline_access` durante el paso de autorización. Además, `getTokenSilently` llamará directamente al endpoint `/oauth/token` para intercambiar tokens de actualización por tokens de acceso.
El SDK respetará la configuración de almacenamiento al guardar los tokens de actualización. Si el SDK se ha configurado con el mecanismo predeterminado de almacenamiento en memoria, los tokens de actualización se perderán al actualizar la página.

<div id="usage">
  ## Uso
</div>

A continuación, se muestran ejemplos de uso de varios métodos del SDK. Ten en cuenta que en estos ejemplos se utiliza jQuery.

<div id="login-with-redirect">
  ### Inicio de sesión con redirección
</div>

Redirige al endpoint `/authorize` de Auth0 e inicia el flujo de [Universal Login](/docs/es/authenticate/login/auth0-universal-login):

```jsx lines
$('#loginRedirect').click(async () => {
  await auth0.loginWithRedirect({
    authorizationParams: {
      redirect_uri: 'http://localhost:3000/'
    }
  });
});
```

<div id="login-with-popup">
  ### Inicio de sesión con popup
</div>

Usa un popup para iniciar sesión a través de la página de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>:

```jsx lines
$('#loginPopup').click(async () => {
  await auth0.loginWithPopup();
});
```

Si el usuario tarda más de los 60 segundos del timeout predeterminado en completar el flujo de autenticación, la autenticación se interrumpirá y tendrás que capturar el error en tu código para hacer una de estas dos cosas:

Sugerir al usuario que lo intente de nuevo y cerrar manualmente el popup con `error.popup.close`:

```jsx lines
$('#loginPopup').click(async () => {
  try {
    await auth0.loginWithPopup();
  } catch {error}
  if (error instanceof auth0.PopupTimeoutError) {
    // lógica personalizada para informar al usuario que reintente
    error.popup.close();
  }
});
```

O bien, crea una opción `popup` personalizada en el objeto `options`:

```jsx lines
$('#loginPopup').click(async () => {
  const popup = window.open(
    '',
    'auth0:authorize:popup',
    'left=100,top=100,width=400,height=600,resizable'
  );
  try {
    await auth0.loginWithPopup({ popup });
  } catch {error}
  if (error instanceof auth0.PopupTimeoutError) {
    // lógica personalizada para informar al usuario que reintente
    error.popup.close();
  }
});
```

<div id="login-with-redirect-callback">
  ### Inicio de sesión con devolución de llamada tras la redirección
</div>

Cuando el navegador vuelva a tu SPA tras ser redirigido desde Auth0, se debe llamar a `handleRedirectCallback` para completar el flujo de inicio de sesión:

```jsx lines
$('#loginRedirectCallback').click(async () => {
  await auth0.handleRedirectCallback();
});
```

<div id="get-access-token-with-no-interaction">
  ### Obtener un token de acceso sin interacción
</div>

Obtén un nuevo token de acceso de forma silenciosa, ya sea usando un iframe oculto y `prompt=none` o un token de actualización rotativo. Los tokens de actualización se usan cuando `useRefreshTokens` se establece en `true` al configurar el SDK.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Obtener un token de acceso de forma silenciosa sin usar tokens de actualización no funcionará en navegadores que bloquean las cookies de terceros, como Safari y Brave. Para obtener más información sobre esta solución alternativa con dominio personalizado, consulta [Troubleshoot Renew Tokens When Using Safari](https://support.auth0.com/center/s/article/troubleshoot-auth0-token-renewal-issues-in-safari-with-itp-enabled).
</Callout>

Si se usan el almacenamiento en memoria (el valor predeterminado) y los tokens de actualización, los tokens nuevos se obtienen mediante un web worker en los navegadores compatibles:

```jsx lines
$('#getToken').click(async () => {
  const token = await auth0.getTokenSilently();
});
```

El método `getTokenSilently()` requiere que tengas habilitada la opción **Permitir omitir el consentimiento del usuario** en [la configuración de la API en el Dashboard](https://manage.auth0.com/#/apis). Además, [no se puede omitir el consentimiento del usuario en &#39;localhost&#39;](/docs/es/get-started/applications/third-party-applications/user-consent-and-third-party-applications).

<div id="get-access-token-with-popup">
  ### Obtener un token de acceso con popup
</div>

Los tokens de acceso también se pueden obtener mediante una ventana popup. A diferencia de `getTokenSilently`, este método para obtener un token de acceso funcionará en navegadores en los que las cookies de terceros estén bloqueadas de forma predeterminada:

```jsx lines
$('#getTokenPopup').click(async () => {
  const token = await auth0.getTokenWithPopup({
    authorizationParams: {
      audience: 'https://mydomain/api/',
      scope: 'read:rules'
    }
  });
});
```

<div id="get-access-token-for-a-different-audience">
  ### Obtener un token de acceso para una audiencia diferente
</div>

Se pueden pasar opciones a `getTokenSilently` para obtener un token de acceso con una <Tooltip tip="Audiencia: identificador único de la audiencia de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip> y un scope distintos de los que se solicitaron cuando el usuario se autenticó.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Esto solo funciona cuando no se usan tokens de actualización (`useRefreshTokens: false`), ya que un token de actualización está vinculado a la audiencia y al scope específicos que se solicitaron cuando el usuario se autenticó.
</Callout>

```jsx lines
$('#getToken_audience').click(async () => {
  const differentAudienceOptions = {
    authorizationParams: {
      audience: 'https://mydomain/another-api/',
      scope: 'read:rules',
      redirect_uri: 'http://localhost:3000/callback.html'
    }
  };
  const token = await auth0.getTokenSilently(differentAudienceOptions);
});
```

Puedes obtener los datos del perfil del usuario autenticado llamando al método `getUser`:

```jsx lines
$('#getUser').click(async () => {
```

<div id="get-id-token-claims">
  ### Obtener las claims del token de ID
</div>

Puedes obtener las claims del <Tooltip tip="Token de ID: credencial destinada al propio cliente, y no para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> del usuario autenticado llamando al método `getIdTokenClaims`:

```jsx lines
$('#getIdTokenClaims').click(async () => {
  const claims = await auth0.getIdTokenClaims();
  // si necesitas el id_token sin procesar, puedes acceder a él
  // mediante la propiedad __raw
  const id_token = claims.__raw;
});
```

<div id="logout-with-no-client-id">
  ### Cierre de sesión sin ID de cliente
</div>

Puede iniciar un cierre de sesión sin especificar ningún <Tooltip tip="ID de cliente: valor de identificación asignado a su recurso registrado en Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip> llamando al método `logout` e incluyendo `clientId: null`:

```jsx lines
$('#logoutNoClientId').click(async () => {

    logoutParams: {
      returnTo: 'http://localhost:3000/'
    }
  });
});
```
