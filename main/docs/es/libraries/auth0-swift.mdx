  ## Primeros pasos
</div>

<div id="requirements">
  ### Requisitos
</div>

* iOS 13.0+ / macOS 11.0+ / tvOS 13.0+ / watchOS 7.0+
* Xcode 14.x
* Swift 5.7+

<Warning>
  Consulta la [Política de soporte](https://github.com/auth0/Auth0.swift/tree/master#support-policy) para saber en qué casos dejar de admitir versiones de Xcode, Swift y las plataformas no se considerará un **cambio incompatible**.
</Warning>

<div id="installation">
  ### Instalación
</div>

<div id="swift-package-manager">
  #### Swift Package Manager
</div>

Abre la siguiente opción del menú en Xcode:

**File &gt; Add Packages...**

En el cuadro de búsqueda **Search or Enter Package URL**, introduce esta URL:

```http lines
https://github.com/auth0/Auth0.swift
```

A continuación, selecciona la regla de dependencia y pulsa **Agregar paquete**.

<div id="cocoapods">
  #### Cocoapods
</div>

Añade la siguiente línea a tu `Podfile`:

```bash lines
pod 'Auth0', '~> 2.0'
```

Luego, ejecuta `pod install`.

<div id="carthage">
  #### Carthage
</div>

Añade la siguiente línea a tu `Cartfile`:

```bash lines
github "auth0/Auth0.swift" ~> 2.0
```

Luego, ejecuta `carthage bootstrap --use-xcframeworks`.

<div id="configure-the-sdk">
  ### Configura el SDK
</div>

Ve al [Auth0 Dashboard](https://manage.auth0.com/#/applications/) y crea una nueva aplicación **Native**.

Auth0.swift necesita el **<Tooltip tip="ID de cliente: valor de identificación que Auth0 asigna a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** y **dominio** de la aplicación de Auth0 para comunicarse con Auth0. Puedes encontrar estos datos en la página Settings de tu aplicación de Auth0. Si usas un [dominio personalizado](/docs/es/customize/custom-domains), usa el valor de tu <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip> en lugar del que aparece en la página Settings.

<div id="configure-client-id-and-domain-with-a-plist">
  #### Configura el ID de cliente y el dominio con un archivo `plist`
</div>

Crea un archivo `plist` llamado `Auth0.plist` en el paquete de tu aplicación con el siguiente contenido:

```xml lines
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>ClientId</key>
    <string>{yourAuth0ClientId}</string>
    <key>Domain</key>
    <string>{yourAuth0Domain}</string>
</dict>
</plist>
```

<div id="configure-client-id-and-domain-programmatically">
  #### Configurar el ID de cliente y el dominio mediante programación
</div>

<div id="for-web-auth">
  ##### Para Web Auth
</div>

```swift lines
Auth0
    .webAuth(clientId: "{yourAuth0ClientID}", domain: "{yourAuth0Domain}")
    // ...
```

<div id="for-the-authentication-api-client">
  ##### Para el cliente de la Authentication API
</div>

```swift lines
Auth0
    .authentication(clientId: "{yourAuth0ClientID}", domain: "{yourAuth0Domain}")
    // ...
```

<div id="for-the-management-api-client-users">
  ##### Para el cliente de la Management API (Users)
</div>

```swift lines
Auth0
    .users(token: credentials.accessToken, domain: "{yourAuth0Domain}")
    // ...
```

<div id="configure-web-auth-ios-macos">
  ### Configurar Web Auth (iOS / macOS)
</div>

<div id="configure-callback-url-and-logout-url">
  #### Configurar la URL de devolución de llamada y la URL de cierre de sesión
</div>

La URL de devolución de llamada y la URL de cierre de sesión son las URL que Auth0 usa para redirigir de vuelta a tu aplicación. Auth0 usa la URL de devolución de llamada después de autenticar al usuario, y la URL de cierre de sesión después de eliminar la <Tooltip tip="Cookie de sesión: entidad que, cuando está presente, permite considerar autenticado al usuario." cta="Ver glosario" href="/docs/es/glossary?term=session+cookie">cookie de sesión</Tooltip>.

Como la URL de devolución de llamada y la URL de cierre de sesión pueden manipularse, deberás agregar tus URL a los campos **Allowed Callback URLs** y **Allowed Logout URLs** en la página Settings de tu aplicación de Auth0. Esto permitirá que Auth0 reconozca estas URL como válidas. Si la URL de devolución de llamada y la URL de cierre de sesión no están configuradas, los usuarios no podrán iniciar ni cerrar sesión en la aplicación y recibirán un error.

Ve a la página Settings de tu [aplicación de Auth0](https://manage.auth0.com/#/applications/) y agrega la URL correspondiente a **Allowed Callback URLs** y **Allowed Logout URLs**, según la plataforma de tu aplicación. Si usas un [dominio personalizado](/docs/es/customize/custom-domains), reemplaza `{yourAuth0Domain}` por el valor de tu dominio personalizado en lugar del valor de la página Settings.

<div id="ios">
  ##### iOS
</div>

```swift wrap lines
{yourBundleIdentifier}://{yourAuth0Domain}/ios/{yourBundleIdentifier}/callback
```

<div id="macos">
  ##### macOS
</div>

```swift wrap lines
{yourBundleIdentifier}://{yourAuth0Domain}/macos/{yourBundleIdentifier}/callback
```

Por ejemplo, si el identificador del paquete de tu app para iOS fuera `com.example.MyApp` y tu dominio de Auth0 fuera `example.us.auth0.com`, este valor sería:

```text wrap lines
com.example.MyApp://example.us.auth0.com/ios/com.example.MyApp/callback
```

<Warning>
  Asegúrate de que la configuración [**Método de autenticación del endpoint de token**](/docs/es/get-started/applications/confidential-and-public-applications/view-application-type) esté configurada como `None`.
</Warning>

<div id="configure-custom-url-scheme">
  #### Configurar un esquema de URL personalizado
</div>

En Xcode, ve a la pestaña **Info** de la configuración del target de tu app. En la sección **URL Types**, haz clic en el botón **＋** para añadir una nueva entrada. Allí, introduce `auth0` en el campo **Identifier** y `$(PRODUCT_BUNDLE_IDENTIFIER)` en el campo **URL Schemes**.

Esto registra tu identificador del paquete como un esquema de URL personalizado, para que la URL de devolución de llamada y la URL de cierre de sesión puedan redirigir a tu app.

<div id="web-auth-login-ios-macos">
  ### Inicio de sesión con Web Auth (iOS / macOS)
</div>

Importa el módulo `Auth0` en el archivo donde quieras mostrar la página de inicio de sesión.

```swift lines
import Auth0
```

Luego, muestra la página de [Universal Login](/docs/es/authenticate/login/auth0-universal-login) al pulsar tu botón de **Iniciar sesión**.

```swift lines
Auth0
    .webAuth()
    .start { result in
        switch result {
        case .success(let credentials):
            print("Obtained credentials: \(credentials)")
        case .failure(let error):
            print("Failed with: \(error)")
        }
    }
```

#### Con async/await

```swift lines
do {
    let credentials = try await Auth0.webAuth().start()
    print("Obtained credentials: \(credentials)")
} catch {
    print("Failed with: \(error)")
}
```

<div id="using-combine">
  #### Uso de Combine
</div>

```swift lines
Auth0
    .webAuth()
    .start()
    .sink(receiveCompletion: { completion in
        if case .failure(let error) = completion {
            print("Failed with: \(error)")
        }
    }, receiveValue: { credentials in
        print("Obtained credentials: \(credentials)")
    })
    .store(in: &cancellables)
```

<div id="web-auth-logout-ios-macos">
  ### Cierre de sesión con Web Auth (iOS / macOS)
</div>

Para cerrar la sesión del usuario, debes borrar la <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> cookie de sesión y, después, eliminar las credenciales del usuario de tu aplicación.

Llama al método `clearSession()` en la acción de tu botón de **Cierre de sesión**. Una vez borrada la cookie de sesión, [elimina las credenciales del usuario](https://github.com/auth0/Auth0.swift/blob/master/EXAMPLES.md#clear-stored-credentials).

```swift lines
Auth0
    .webAuth()
    .clearSession { result in
        switch result {
        case .success:
            print("Session cookie cleared")
            // Eliminar credenciales
        case .failure(let error):
            print("Failed with: \(error)")
        }
    }
```

<div id="using-asyncawait">
  #### Uso de async/await
</div>

```swift lines
do {
    try await Auth0.webAuth().clearSession()
    print("Session cookie cleared")
    // Eliminar credenciales
} catch {
    print("Failed with: \(error)")
}
```

<div id="using-combine">
  #### Uso de Combine
</div>

```swift lines
Auth0
    .webAuth()
    .clearSession()
    .sink(receiveCompletion: { completion in
        switch completion {
        case .finished:
            print("Session cookie cleared")
            // Eliminar credenciales
        case .failure(let error):
            print("Failed with: \(error)")
        }
    }, receiveValue: {})
    .store(in: &cancellables)
```
