Lock puede funcionar en dos modos diferentes. El modo predeterminado es **modo de redirección**. En este modo, se redirige al usuario para que se autentique y luego regrese a la aplicación. En el segundo modo, **modo popup**, una ventana emergente permite al usuario autenticarse con el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> sin salir de la aplicación.

<div id="redirect-mode">
  ## Modo de redirección
</div>

Cuando haces clic en el botón del IdP (por ejemplo, Facebook) en modo de redirección, se te redirige momentáneamente a Facebook. El modo de redirección es el predeterminado en Lock y el recomendado para casi todos los casos de uso. Una vez que inicias sesión correctamente (en Facebook, en este ejemplo), Facebook te redirigirá de vuelta a tu aplicación (a través de Auth0). La mayoría de los ejemplos o muestras de la documentación de referencia emplean el modo de redirección.

<div id="popup-mode">
  ## Modo popup
</div>

Si, después de hacer clic en el botón del IdP (Facebook, por ejemplo), se abre un popup (una nueva pestaña o ventana), significa que estás usando el modo popup. En ese popup, verás la página de Facebook. Una vez que inicies sesión correctamente en Facebook, el popup se cerrará y tu aplicación web reconocerá que el usuario se ha autenticado. La aplicación web **nunca se redirige a ninguna otra página**.

Implementar Lock con el modo popup vuelve a ser un cambio sencillo de la opción `redirect` con respecto a su valor predeterminado.

export const codeExample = `var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  {
    auth: {
      redirect: false
    }
  }
);`;

<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor adicional además de username and password, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+authentication">Autenticación multifactor</Tooltip> (MFA) no es compatible cuando Lock está en modo popup e integrado en tu aplicación. Además, el modo popup no funciona con <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>.

Algunas funcionalidades de Auth0, como el [inicio de sesión único (SSO)](/docs/es/authenticate/single-sign-on) entre varias aplicaciones, dependen de que se redirija a los usuarios a Auth0 para establecer una cookie en `'{yourDomain}'`.

Al usar el modo popup, se mostrará una ventana emergente para establecer esta cookie. Si los prompts no son necesarios, esta ventana emergente aparecerá en blanco y dentro de un iframe oculto para minimizar las interrupciones. Esto se debe a que las solicitudes de origen cruzado enviadas desde tu aplicación a Auth0 no pueden establecer cookies.

Si no quieres mostrar una ventana emergente y no necesitas <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip> entre varias aplicaciones, puedes establecer `sso: false` al usar Lock o auth0.js.

```js lines
var options = {
  auth: {
    sso: false
  }
}
```
