Puedes personalizar la apariencia de tu widget de Lock de varias maneras. La forma más segura y recomendada de hacerlo es mediante las opciones de JavaScript proporcionadas.

<div id="javascript-options">
  ## Opciones de JavaScript
</div>

Puedes configurar diversas personalizaciones para tu Lock mediante el parámetro `options` al instanciarlo. Algunas te permiten personalizar la UI. Las opciones de personalización de la UI aún están en desarrollo, y esperamos ir añadiendo más con el tiempo.

Primero, definirás el objeto `options`, que contendrá las opciones que quieras personalizar. Después, tendrás que incluir ese objeto `options` como tercer parámetro al instanciar Lock; a continuación encontrarás más información al respecto.

<div id="theming-options">
  ### Opciones de tematización
</div>

Actualmente hay un par de opciones de tematización disponibles, agrupadas bajo el espacio de nombres de la propiedad `theme`.

#### logo {String}

<Frame>![Personalización de Lock UI - Logo](/docs/images/cdy7uua7fh8z/FdOAKKy71rTbFjIlZEL8D/2eef96008095d85388daaa461707901d/lock-theme-logo.png)</Frame>

El valor de `logo` es la URL de una imagen que se mostrará en el header de Lock y, de forma predeterminada, será el logo de Auth0. La resolución mínima recomendada es de 200 píxeles (ancho) por 200 píxeles (alto).

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png'
  }
};
```

#### primaryColor {String}

<Frame>![Personalización de la UI de Lock: color primario](/docs/images/cdy7uua7fh8z/1qQjeTmXJyIziv400VPaOn/48dd3cb8a72a6e88b2634b4b8a8bed63/lock-theme-primarycolor.png)</Frame>

La propiedad `primaryColor` define el color primario de Lock; a partir de él se calcularán todos los colores utilizados en el widget. Esta opción resulta útil cuando se proporciona un `logo` personalizado, ya que garantiza que todos los colores combinen bien con la paleta de colores del `logo`. El valor predeterminado es `#ea5323`.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png',
    primaryColor: '#31324F'
  }
};
```

<div id="authbuttons-object">
  #### authButtons `{Object}`
</div>

Permite personalizar los botones de Lock. Debe indicarse por nombre cada conexión personalizada cuyo botón quieras personalizar, cada una con su propio conjunto de parámetros. Los parámetros que se pueden personalizar se indican a continuación:

* **displayName** &#123;String&#125;: El nombre que se mostrará en lugar del nombre de la conexión al crear el título del botón, por ejemplo, `LOGIN WITH MYCONNECTION` para el inicio de sesión).
* **primaryColor** &#123;String&#125;: El color de fondo del botón. El valor predeterminado es `#eb5424`.
* **foregroundColor** &#123;String&#125;: El color del texto del botón. El valor predeterminado es `#FFFFFF`.
* **icon** &#123;String&#125;: La URL del icono de esta conexión. Por ejemplo: `http://site.com/logo.png`.

```js lines
var options = {
  theme: {
    authButtons: {
      "testConnection": {
        displayName: "Test Conn",
        primaryColor: "#b7b7b7",
        foregroundColor: "#000000",
        icon: "http://example.com/icon.png"
      },
      "testConnection2": {
        primaryColor: "#000000",
        foregroundColor: "#ffffff",
      }
    }
  }
};
```

<div id="customizing-text">
  ### Personalización del texto
</div>

La opción `languageDictionary` permite personalizar todo el texto que se muestra en Lock. El valor predeterminado es {}. A continuación, puedes ver un ejemplo.

```js lines
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Log me in"
  },
};
```

<Frame>![Personalización del texto en la interfaz de usuario de Lock mediante la opción languageDictionary](/docs/images/cdy7uua7fh8z/iNvBLxxmea2tiWBhzwKX5/db0d7f86d0f7f95f70e5edcd34ab736f/lock-languagedictionary.png)</Frame>

Para ver la lista completa de los elementos que se pueden personalizar con `languageDictionary`, consulta la [Especificación del diccionario de idiomas en inglés](https://github.com/auth0/lock/blob/master/src/i18n/en.js) del repositorio.

<div id="instantiating-lock">
  ### Crear una instancia de Lock
</div>

Por último, deberás crear una instancia de Lock con el objeto `options` que definiste con tus opciones personalizadas.

`var lock = new Auth0Lock('{yourClientId}', '{yourDomain}', options);`

<div id="overriding-css">
  ## Sobrescribir CSS
</div>

No se recomienda personalizar tu Lock sobrescribiendo su CSS. El problema es que, con las nuevas versiones de Lock, algunos estilos pueden cambiar, lo que puede generar problemas no deseados si estás sobrescribiendo el CSS. Además, es fácil pasar por alto el uso de estilos en otras partes y, aunque el cambio se vea bien en una vista, puede que no ocurra lo mismo en otra.

Si aun así quieres sobrescribir el CSS para personalizar aún más tu Lock, te recomendamos usar una versión de parche específica de Lock en lugar de una versión mayor o menor. Así limitarás la cantidad de resultados inesperados que pueden producirse al modificar los estilos y después publicarse otro parche que pueda provocar un comportamiento inesperado en tu UI debido a esos cambios. Esto puede hacerse asegurándote de especificar esa versión de parche (`x.y.z`) al incluir Lock o al descargarlo.

Además, por supuesto, te recomendamos probar tus cambios de CSS exhaustivamente para asegurarte de que la experiencia sea la que pretendes ofrecer a tus clientes.
