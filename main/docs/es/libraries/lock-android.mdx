Lock for Android puede integrarse en sus aplicaciones Android nativas para ofrecer una forma atractiva de permitir que sus usuarios inicien sesión y se registren en su aplicación. Ofrece compatibilidad con <Tooltip tip="Identity Provider (IdP): Service that stores and manages digital identities." cta="View Glossary" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> sociales como Facebook, Google o X, así como con proveedores empresariales como Active Directory.

Consulte el [repositorio de Lock.Android](https://github.com/auth0/Lock.Android) en GitHub.

<div id="requirements">
  ## Requisitos
</div>

Para usar la UI de Lock o tu propia UI mediante la [biblioteca Auth0.Android](https://github.com/auth0/Auth0.Android), se requiere como mínimo el nivel de API 21+ de Android y Java 8 o una versión posterior. También necesitarás una aplicación de Auth0 de tipo &quot;Native&quot;.

Esto es lo que necesitas en `build.gradle` para generar bytecode de Java 8 con los plugins de Android y Kotlin, respectivamente.

```kotlin lines
android {
    compileOptions {
        sourceCompatibility JavaVersion.VERSION_1_8
        targetCompatibility JavaVersion.VERSION_1_8
    }

    kotlinOptions {
        jvmTarget = '1.8'
    }
}
```

<div id="installation">
  ## Instalación
</div>

Lock está disponible en [Maven Central](https://central.sonatype.com/). Para empezar a usar Lock, agrega estas líneas al archivo de dependencias `build.gradle`:

`implementation 'com.auth0.android:lock:3.+'`

Puedes consultar la versión más reciente en el [Readme](https://github.com/auth0/Lock.Android#install) del repositorio o en [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock).

Después de agregar la dependencia de Gradle, recuerda sincronizar tu proyecto con los archivos de Gradle.

<div id="dashboard-settings">
  ## Configuración del Dashboard
</div>

Debes completar algunos ajustes en tu [Auth0 Dashboard](https://manage.auth0.com/#) antes de empezar.

<div id="callback-url">
  ### URL de devolución de llamada
</div>

Ve a tu <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y abre la configuración de la aplicación. Agrega la siguiente URL a **Allowed Callback URLs** de la aplicación:

`https://{yourDomain}/android/{yourAppPackageName}/callback`

Reemplaza `{yourAppPackageName}` por el nombre real del paquete de tu aplicación, que encontrarás en el archivo `app/build.gradle` como el valor de `applicationId`.

<div id="keystores-and-key-hashes">
  ### Keystores y hashes de clave
</div>

Las aplicaciones de Android deben firmarse antes de poder instalarse en un dispositivo. Para ello, la primera vez que se ejecuta, el IDE Android Studio genera un &quot;Android Debug Keystore&quot; predeterminado que usará para firmar las versiones de desarrollo. Es probable que este [Keystore](https://developer.android.com/studio/publish/app-signing) sea diferente del de tu compilación de producción, ya que se usará para identificarte como desarrollador.

Al usar la funcionalidad Web Authentication (es decir, conexiones sociales), Lock se configurará de forma predeterminada para intentar usar Android App Links. Esto requiere una configuración adicional en el Dashboard de la aplicación de Auth0. Usa nuestra [guía de Android Keystores and Key Hashes](/docs/es/libraries/auth0-android/android-development-keystores-hashes) para completar este paso.

  ## Implementación de Lock (Social, Database, empresarial)
</div>

Las siguientes instrucciones explican cómo implementar Classic Lock for Android. Si lo que buscas específicamente es implementar <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> Lock for Android, consulta [Lock.Android: Passwordless](/docs/es/libraries/lock-android/lock-android-passwordless).

<div id="configuring-the-sdk">
  ### Configuración del SDK
</div>

En tu archivo `app/build.gradle`, agrega los [Manifest Placeholders](https://developer.android.com/build/manage-manifests) para las propiedades Auth0 Domain y Auth0 Scheme, que la biblioteca utilizará internamente para registrar un intent-filter que capture el resultado de la autenticación.

```kotlin lines
plugins {
    id "com.android.application"
    id "kotlin-android"
}

android {
    compileSdkVersion 30
    defaultConfig {
        applicationId "com.auth0.samples"
        minSdkVersion 21
        targetSdkVersion 30
        // ...

        // ---> Añade la siguiente línea
        manifestPlaceholders = [auth0Domain: "@string/com_auth0_domain", auth0Scheme: "https"]
        // <---
    }
}
```

Es recomendable añadir estos valores al archivo `strings.xml` como recursos de cadena a los que luego se pueda hacer referencia desde el código. Esta guía seguirá esa práctica.

export const codeExample = `<resources>
    <string name="com_auth0_client_id">{yourClientId}</string>
    <string name="com_auth0_domain">{yourDomain}</string>
</resources>`;

<div id="sdk-usage">
  ### Uso del SDK
</div>

En la actividad en la que vayas a invocar Lock, crea una instancia de `Auth0` con la información de tu aplicación. La forma más sencilla de crearla es pasarle un `Context` de Android. Esto usará los valores definidos previamente en el archivo `strings.xml`. Para que esto funcione, los recursos de cadena deben definirse con las mismas claves indicadas arriba.

```text lines
val account = Auth0(context)
```

Declara una implementación de `AuthenticationCallback` que gestionará los eventos de autenticación de usuario. El objeto `Credentials` que se devuelve cuando la autenticación se realiza correctamente contendrá los tokens que consumirá tu aplicación o API. Consulta [Tokens](/docs/es/secure/tokens) para obtener más información.

```kotlin lines
private val callback = object : AuthenticationCallback() {
    override fun onAuthentication(credentials: Credentials) {
        // Autenticado
    }

    override fun onError(error: AuthenticationException) {
        // Ocurrió una excepción
    }
}
```

Prepara una nueva instancia de Lock usando la clase Builder para configurarla. Proporciona los datos de la cuenta y la implementación de la devolución de llamada declarada anteriormente. Aquí puedes configurar valores como <Tooltip tip="Audience: identificador único del audience de un token emitido. En un token, se denomina aud; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un Access Token." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip>, scope y las conexiones disponibles.

Cuando hayas terminado, crea la instancia de Lock. Esta instancia está pensada para reutilizarse y debe desecharse cuando ya no sea necesaria. Un buen lugar para hacerlo es en el método `onDestroy` de tu Activity.

```kotlin lines expandable
// Esta actividad mostrará Lock
class MyActivity : AppCompatActivity() {

    private lateinit var lock: Lock

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val account = Auth0(this)
        // Instanciar Lock una vez
        lock = Lock.newBuilder(account, callback)
            // Personalizar Lock
            .build(this)
    }

    override fun onDestroy() {
        super.onDestroy()
        // ¡Importante! Liberar Lock y sus recursos
        lock.onDestroy(this)
    }

    private val callback = object : AuthenticationCallback() {
        override fun onAuthentication(credentials: Credentials) {
            // Autenticado
        }

        override fun onError(error: AuthenticationException) {
            // Se produjo una excepción
        }
    }
}
```

Por último, inicia el widget `Lock` desde la actividad.

```kotlin lines
startActivity(lock.newIntent(this))
```

¡Eso es todo! Lock se encargará del resto.

<div id="android-app-links-custom-scheme">
  #### Android App Links - Esquema personalizado
</div>

El esquema de URI de devolución de llamada que se usa en este artículo y que Lock utiliza de forma predeterminada es `https`. Esto funciona mejor en Android Marshmallow (API 23) o versiones posteriores si usas [Android App Links](https://developer.android.com/training/app-links), pero en versiones anteriores de Android puede aparecer el cuadro de diálogo del selector de intents, que le pide al usuario que elija entre tu aplicación o el navegador para resolver el intent. Esto se conoce como el &quot;cuadro de diálogo de desambiguación&quot;. Puedes cambiar este comportamiento usando un esquema personalizado único para que el sistema operativo abra el enlace directamente con tu aplicación.

1. Actualiza el valor de Manifest Placeholder `auth0Scheme` en el archivo `app/build.gradle` o directamente en la definición de Intent Filter del archivo `AndroidManifest.xml`, cambiando el esquema actual por el nuevo.
2. Actualiza las &quot;Allowed Callback URLs&quot; en la configuración de la aplicación de tu Auth0 Dashboard para que coincidan con las URL que empiezan por el nuevo esquema.
3. Llama a `withScheme()` al configurar Lock con el builder, pasando el esquema que quieres usar.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El valor del esquema debe estar completamente en minúsculas. Si no es así, se registrará un mensaje de advertencia y la autenticación nunca se completará.
</Callout>
