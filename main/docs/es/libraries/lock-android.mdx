Lock for Android puede integrarse en tus aplicaciones nativas de Android para ofrecer una forma atractiva de permitir que tus usuarios inicien sesión y se registren en tu aplicación. Ofrece compatibilidad con <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> sociales como Facebook, Google o X, así como con proveedores empresariales como Active Directory.

Consulta el [repositorio de Lock.Android](https://github.com/auth0/Lock.Android) en GitHub.

<div id="requirements">
  ## Requisitos
</div>

Para usar la UI de Lock o tu propia UI mediante la [biblioteca Auth0.Android](https://github.com/auth0/Auth0.Android), el nivel mínimo de API de Android requerido es 21+ y la versión de Java debe ser 8 o superior. También necesitarás una aplicación de Auth0 de tipo &quot;Native&quot;.

Esto es lo que necesitas en `build.gradle` para generar bytecode de Java 8 con los plugins de Android y Kotlin, respectivamente.

```kotlin lines
android {
    compileOptions {
        sourceCompatibility JavaVersion.VERSION_1_8
        targetCompatibility JavaVersion.VERSION_1_8
    }

    kotlinOptions {
        jvmTarget = '1.8'
    }
}
```

<div id="installation">
  ## Instalación
</div>

Lock está disponible en [Maven Central](https://central.sonatype.com/). Para empezar a usar Lock, agrega estas líneas al archivo de dependencias `build.gradle`:

`implementation 'com.auth0.android:lock:3.+'`

Puedes consultar la versión más reciente en el [Readme](https://github.com/auth0/Lock.Android#install) del repositorio o en [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock).

Después de agregar la dependencia de Gradle, recuerda sincronizar tu proyecto con los archivos de Gradle.

<div id="dashboard-settings">
  ## Configuración del Dashboard
</div>

Debes completar algunos ajustes en tu [Auth0 Dashboard](https://manage.auth0.com/#) antes de empezar.

<div id="callback-url">
  ### URL de devolución de llamada
</div>

Ve a tu <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y ve a la configuración de la aplicación. Agrega la siguiente URL a **Allowed Callback URLs** de la aplicación:

`https://{yourDomain}/android/{yourAppPackageName}/callback`

Reemplaza `{yourAppPackageName}` por el nombre real del paquete de tu aplicación, disponible en tu archivo `app/build.gradle` como el valor de `applicationId`.

<div id="keystores-and-key-hashes">
  ### Keystores y hashes de clave
</div>

Las aplicaciones de Android deben estar firmadas antes de poder instalarse en un dispositivo. Para ello, el IDE Android Studio genera, la primera vez que se ejecuta, un &quot;Android Debug Keystore&quot; predeterminado que usará para firmar las compilaciones de desarrollo. Es probable que este [Keystore](https://developer.android.com/studio/publish/app-signing) sea diferente del de tu compilación de producción, ya que se utilizará para identificarte como desarrollador.

Al usar la funcionalidad Web Authentication (es decir, conexiones sociales), Lock se configurará de forma predeterminada para intentar usar Android App Links. Esto requiere una configuración adicional en el Dashboard de la aplicación de Auth0. Usa nuestra [guía sobre Keystores de Android y hashes de clave](/docs/es/libraries/auth0-android/android-development-keystores-hashes) para completar este paso.

<div id="configuring-the-sdk">
  ### Configuración del SDK
</div>

En tu archivo `app/build.gradle`, agrega los [Manifest Placeholders](https://developer.android.com/build/manage-manifests) de las propiedades Auth0 Domain y Auth0 Scheme, que la biblioteca utilizará internamente para registrar un intent-filter que capture el resultado de la autenticación.

```kotlin lines
plugins {
    id "com.android.application"
    id "kotlin-android"
}

android {
    compileSdkVersion 30
    defaultConfig {
        applicationId "com.auth0.samples"
        minSdkVersion 21
        targetSdkVersion 30
        // ...

        // ---> Añade la siguiente línea
        manifestPlaceholders = [auth0Domain: "@string/com_auth0_domain", auth0Scheme: "https"]
        // <---
    }
}
```

Es una buena práctica agregar estos valores al archivo `strings.xml` como recursos de cadena, a los que luego se podrá hacer referencia desde el código. En esta guía seguiremos esa práctica.

export const codeExample = `<resources>
    <string name="com_auth0_client_id">{yourClientId}</string>
    <string name="com_auth0_domain">{yourDomain}</string>
</resources>`;

<div id="sdk-usage">
  ### Uso del SDK
</div>

En la actividad desde la que vayas a invocar Lock, crea una instancia de `Auth0` con la información de tu aplicación. La forma más sencilla de hacerlo es pasarle un `Context` de Android. Esto usará los valores definidos previamente en el archivo `strings.xml`. Para que esto funcione, los recursos de cadena deben definirse con las mismas claves indicadas arriba.

```text lines
val account = Auth0(context)
```

Defina una implementación de `AuthenticationCallback` que gestione los eventos de autenticación de usuario. El objeto `Credentials` que se devuelve cuando la autenticación se completa correctamente contendrá los tokens que consumirá su aplicación o API. Consulte [Tokens](/docs/es/secure/tokens) para obtener más información.

```kotlin lines
private val callback = object : AuthenticationCallback() {
    override fun onAuthentication(credentials: Credentials) {
        // Autenticado
    }

    override fun onError(error: AuthenticationException) {
        // Ocurrió una excepción
    }
}
```

Prepare una nueva instancia de Lock con la clase Builder para configurarla. Proporcione los detalles de la cuenta y la implementación de devolución de llamada declarada anteriormente. Aquí puede configurar valores como <Tooltip tip="Audiencia: identificador único de la audiencia de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip>, scope y las conexiones disponibles.

Cuando termine, cree la instancia de Lock. Esta instancia está pensada para reutilizarse y debe liberarse cuando ya no se necesite. Un buen lugar para hacerlo es en el método `onDestroy` de su actividad.

```kotlin lines expandable
// Esta actividad mostrará Lock
class MyActivity : AppCompatActivity() {

    private lateinit var lock: Lock

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val account = Auth0(this)
        // Instanciar Lock una vez
        lock = Lock.newBuilder(account, callback)
            // Personalizar Lock
            .build(this)
    }

    override fun onDestroy() {
        super.onDestroy()
        // ¡Importante! Liberar Lock y sus recursos
        lock.onDestroy(this)
    }

    private val callback = object : AuthenticationCallback() {
        override fun onAuthentication(credentials: Credentials) {
            // Autenticado
        }

        override fun onError(error: AuthenticationException) {
            // Se produjo una excepción
        }
    }
}
```

Por último, inicia el widget `Lock` desde tu actividad.

```kotlin lines
startActivity(lock.newIntent(this))
```

¡Eso es todo! Lock se encargará del resto.

<div id="android-app-links-custom-scheme">
  #### Android App Links - Esquema personalizado
</div>

El esquema de la URI de devolución de llamada que se usa en este artículo y que Lock usa de forma predeterminada es `https`. Esto funciona mejor en Android Marshmallow (API 23) o versiones posteriores si usas [Android App Links](https://developer.android.com/training/app-links), pero en versiones anteriores de Android puede aparecer el cuadro de diálogo de selección de intents, que le pide al usuario que elija entre tu aplicación y el navegador para resolver el intent. Esto se conoce como el &quot;cuadro de diálogo de desambiguación&quot;. Puedes cambiar este comportamiento usando un esquema personalizado único para que el SO abra el enlace directamente con tu aplicación.

1. Actualiza el valor de Manifest Placeholder `auth0Scheme` en el archivo `app/build.gradle` o directamente en la definición de Intent Filter del archivo `AndroidManifest.xml`, cambiando el esquema existente por el nuevo.
2. Actualiza las &quot;Allowed Callback URLs&quot; en la configuración de tu aplicación en el Auth0 Dashboard para que coincidan con las URL que comienzan con el nuevo esquema.
3. Llama a `withScheme()` al configurar Lock con el builder y pasa el esquema que quieras usar.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El valor del esquema debe estar completamente en minúsculas. Si no es así, se registrará un mensaje de advertencia y la autenticación nunca se completará.
</Callout>
