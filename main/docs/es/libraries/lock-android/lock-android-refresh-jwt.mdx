Cuando la autenticación se realiza con el `scope` `offline_access` incluido, se devuelve un <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token">token de actualización</Tooltip> con las credenciales. Este valor puede usarse para solicitar un nuevo <Tooltip tip="Token de acceso: token utilizado para acceder a un recurso protegido durante un período limitado de tiempo." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> y evitar volver a pedirle al usuario sus credenciales.

Los tokens deben almacenarse de forma segura después de una autenticación correcta. Ten en cuenta que los tokens de actualización **nunca expiran**. Para solicitar un nuevo token, usa `AuthenticationAPIClient` de A`uth0.Android`.

<div id="using-refresh-token">
  ## Uso del token de actualización
</div>

```kotlin lines
val refreshToken: String = // Recuperar el token de actualización del almacenamiento seguro
val account = Auth0(this)

val client = AuthenticationAPIClient(account)
client.renewAuth(refreshToken)
  .start(object: Callback<Credentials, AuthenticationException> {
  override fun onFailure(exception: AuthenticationException) {
       // Error
   }

   override fun onSuccess(credentials: Credentials) {
       // Usar las credenciales
   }
})
```
