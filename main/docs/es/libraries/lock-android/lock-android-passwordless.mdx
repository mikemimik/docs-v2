Lock <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> autentica a los usuarios enviándoles un correo electrónico o un SMS con un código de un solo uso que deben introducir y confirmar para poder iniciar sesión, de forma similar a como lo hace WhatsApp. En este artículo se explica cómo enviar un CÓDIGO con la biblioteca `Lock.Android`.

Puedes lograr un resultado similar [enviando un enlace en el que el usuario pueda hacer clic](/docs/es/libraries/lock-android/lock-android-passwordless-with-magic-link) para completar automáticamente la autenticación passwordless, pero requiere algunos pasos de configuración adicionales.

Para poder autenticar al usuario, tu aplicación debe tener habilitada y configurada la conexión de Email/SMS en tu [Auth0 Dashboard](https://manage.auth0.com/#/connections/passwordless).

  ## Implementación de Passwordless con código

<div id="configuring-the-sdk">
  ### Configuración del SDK
</div>

En tu archivo `app/build.gradle`, agrega los [marcadores de posición del manifiesto](https://developer.android.com/studio/build/manifest-build-variables.html) para las propiedades Auth0 Domain y Auth0 Scheme, que la biblioteca usará internamente para registrar un intent-filter encargado de capturar el resultado de la autenticación.

```kotlin lines
plugins {
    id "com.android.application"
    id "kotlin-android"
}

android {
    compileSdkVersion 30
    defaultConfig {
        applicationId "com.auth0.samples"
        minSdkVersion 21
        targetSdkVersion 30
        // ...

        // ---> Añade la siguiente línea
        manifestPlaceholders = [auth0Domain: "@string/com_auth0_domain", auth0Scheme: "https"]
        // <---
    }
}
```

Es recomendable añadir estos valores al archivo `strings.xml` como recursos de cadena, para poder consultarlos después desde el código.

export const codeExample = `<resources>
    <string name="com_auth0_client_id">{yourClientId}</string>
    <string name="com_auth0_domain">{yourDomain}</string>
</resources>`;

<div id="sdk-usage">
  ### Uso del SDK
</div>

En la actividad donde planeas invocar Lock, crea una instancia de `Auth0` con la información de tu aplicación. La forma más sencilla de hacerlo es pasar un `Context` de Android. Esto usará los valores definidos previamente en el archivo `strings.xml`. Para que funcione, los recursos de tipo cadena deben definirse con las mismas claves que se indicaron antes.

```kotlin lines
val account = Auth0(context)
```

Define una implementación de `AuthenticationCallback` que gestione eventos de autenticación de usuario. El objeto `Credentials` que se devuelve cuando la autenticación se completa correctamente contendrá los tokens que tu aplicación o API acabará utilizando. Consulta [Tokens](/docs/es/secure/tokens) para obtener más información.

```kotlin lines
private val callback = object : AuthenticationCallback() {
    override fun onAuthentication(credentials: Credentials) {
        // Autenticado
    }

    override fun onError(error: AuthenticationException) {
        // Ocurrió una excepción
    }
}
```

Prepara una nueva instancia de Lock con la clase Builder para configurarla. Proporciona los detalles de la cuenta y la implementación de devolución de llamada declarada anteriormente. Aquí puedes configurar valores como <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip>, scope y las conexiones disponibles, entre otros. Cuando termines, construye la instancia de Lock. Esta instancia está pensada para reutilizarse y debe liberarse cuando deje de ser necesaria. Un buen lugar para hacerlo es en el método `onDestroy` de tu actividad.

En el siguiente ejemplo se llama al método `useCode()` para hacer que Lock envíe un **CÓDIGO** al correo electrónico o al número de teléfono del usuario.

```kotlin lines expandable
// Esta actividad mostrará Passwordless Lock
class MyActivity : AppCompatActivity() {

    private lateinit var lock: PasswordlessLock

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val account = Auth0(this)
        // Instanciar Lock una sola vez
        lock = PasswordlessLock.newBuilder(account, callback)
            .useCode()
            // Personalizar Lock
            .build(this)
    }

    override fun onDestroy() {
        super.onDestroy()
        // ¡Importante! Liberar Lock y sus recursos
        lock.onDestroy(this)
    }

    private val callback = object : AuthenticationCallback() {
        override fun onAuthentication(credentials: Credentials) {
            // Autenticado
        }

        override fun onError(error: AuthenticationException) {
            // Se produjo una excepción
        }
    }
}
```

Por último, inicia el widget `PasswordlessLock` desde tu actividad.

```kotlin lines
startActivity(lock.newIntent(this))
```

Según las conexiones passwordless que estén habilitadas, Lock enviará el **CÓDIGO** por Email o SMS. La conexión &#39;email&#39; se selecciona primero si está disponible. Luego, el usuario debe introducir el **CÓDIGO** en el paso de confirmación. Si el valor coincide con el que espera el servidor, la autenticación se completará correctamente.
