Lock <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> autentica a los usuarios enviándoles un correo electrónico o SMS con un código de un solo uso que el usuario debe introducir y confirmar para poder iniciar sesión, de forma similar a como funciona WhatsApp. En este artículo se explicará cómo enviar un CÓDIGO usando la biblioteca `Lock.Android`.

Puedes lograr un resultado similar [enviando un enlace en el que el usuario puede hacer clic](/docs/es/libraries/lock-android/lock-android-passwordless-with-magic-link) para completar automáticamente la autenticación passwordless, pero requiere algunos pasos de configuración adicionales.

Para poder autenticar al usuario, tu aplicación debe tener habilitada y configurada la conexión de correo electrónico/SMS en tu [Auth0 Dashboard](https://manage.auth0.com/#/connections/passwordless).

  ## Implementación de Passwordless con código

<div id="configuring-the-sdk">
  ### Configuración del SDK
</div>

En el archivo `app/build.gradle`, agrega los [marcadores de posición del manifiesto](https://developer.android.com/studio/build/manifest-build-variables.html) para las propiedades Dominio de Auth0 y Esquema de Auth0, que la biblioteca usará internamente para registrar un filtro de intents que capture el resultado de la autenticación.

```kotlin lines
plugins {
    id "com.android.application"
    id "kotlin-android"
}

android {
    compileSdkVersion 30
    defaultConfig {
        applicationId "com.auth0.samples"
        minSdkVersion 21
        targetSdkVersion 30
        // ...

        // ---> Añade la siguiente línea
        manifestPlaceholders = [auth0Domain: "@string/com_auth0_domain", auth0Scheme: "https"]
        // <---
    }
}
```

Es una buena práctica agregar estos valores al archivo `strings.xml` como recursos de cadena a los que luego se pueda hacer referencia desde el código.

export const codeExample = `<resources>
    <string name="com_auth0_client_id">{yourClientId}</string>
    <string name="com_auth0_domain">{yourDomain}</string>
</resources>`;

<div id="sdk-usage">
  ### Uso del SDK
</div>

En la Activity donde planeas invocar Lock, crea una instancia de `Auth0` con la información de tu aplicación. La forma más sencilla de crearla es pasar un `Context` de Android. Esto usará los valores definidos previamente en el archivo `strings.xml`. Para que esto funcione, los recursos de cadena deben definirse con las mismas claves que las indicadas anteriormente.

```kotlin lines
val account = Auth0(context)
```

Declare una implementación de `AuthenticationCallback` que gestione los eventos de autenticación del usuario. El objeto `Credentials` que se devuelve cuando la autenticación se realiza correctamente contendrá los tokens que consumirá su aplicación o API. Consulte [Tokens](/docs/es/secure/tokens) para obtener más información.

```kotlin lines
private val callback = object : AuthenticationCallback() {
    override fun onAuthentication(credentials: Credentials) {
        // Autenticado
    }

    override fun onError(error: AuthenticationException) {
        // Ocurrió una excepción
    }
}
```

Prepare una nueva instancia de Lock con la clase Builder para configurarla. Proporcione los datos de la cuenta y la implementación de la devolución de llamada declarada anteriormente. Aquí puede configurar valores como <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token, y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un Access Token." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip>, scope y las conexiones disponibles, entre otros. Cuando haya terminado, cree la instancia de Lock. Esta instancia está pensada para reutilizarse y debe liberarse cuando ya no sea necesaria. Un buen lugar para hacerlo es en el método `onDestroy` de su Activity.

El ejemplo siguiente llama al método `useCode()` para hacer que Lock envíe un **CÓDIGO** al correo electrónico o al número de teléfono del usuario.

```kotlin lines expandable
// Esta actividad mostrará Passwordless Lock
class MyActivity : AppCompatActivity() {

    private lateinit var lock: PasswordlessLock

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val account = Auth0(this)
        // Instanciar Lock una sola vez
        lock = PasswordlessLock.newBuilder(account, callback)
            .useCode()
            // Personalizar Lock
            .build(this)
    }

    override fun onDestroy() {
        super.onDestroy()
        // ¡Importante! Liberar Lock y sus recursos
        lock.onDestroy(this)
    }

    private val callback = object : AuthenticationCallback() {
        override fun onAuthentication(credentials: Credentials) {
            // Autenticado
        }

        override fun onError(error: AuthenticationException) {
            // Se produjo una excepción
        }
    }
}
```

Por último, inicia el widget `PasswordlessLock` en tu actividad.

```kotlin lines
startActivity(lock.newIntent(this))
```

Dependiendo de las conexiones passwordless que estén habilitadas, Lock enviará el **CÓDIGO** por correo electrónico o SMS. La conexión &#39;email&#39; se selecciona primero si está disponible. Luego, el usuario debe introducir el **CÓDIGO** en el paso de confirmación. Si el valor coincide con el que espera el servidor, la autenticación se realizará correctamente.
