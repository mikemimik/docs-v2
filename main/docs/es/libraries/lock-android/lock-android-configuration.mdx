---

Puede usar las opciones que aparecen a continuación para configurar las funciones y el diseño de Lock for Android.

Las opciones de configuración se añaden al builder de Lock con el siguiente formato:

```kotlin lines
val lock = Lock.newBuilder(account, callback)
    // Opciones de configuración
    .closable(true)
    .allowSignUp(false)
    .setPrivacyURL('http://example.com/privacy')
    .setTermsURL('http://example.com/terms')
    // Fin de las opciones de configuración
    .build(this)
```

<div id="general-options">
  ## Opciones generales
</div>

* **closable(boolean)**: Define si se puede cerrar LockActivity. De forma predeterminada, no se puede cerrar.
* **allowedConnections(List)**: Filtra las conexiones permitidas de la lista configurada en el Dashboard. De forma predeterminada, si este valor está vacío, estarán disponibles todas las conexiones definidas en el Dashboard.
* **hideMainScreenTitle(boolean)**: Si se establece en `true`, el encabezado de la pantalla principal no mostrará el título.

<div id="authentication-options">
  ## Opciones de autenticación
</div>

* **withAuthenticationParameters(Map&lt;String, Object&gt;)**: Define parámetros de autenticación adicionales que se enviarán en cada llamada de inicio de sesión y registro. El `scope` predeterminado que se usa en las llamadas de autenticación es `openid`. Si quieres especificar otro, usa `withAuthenticationParameters` y añade un valor distinto para la clave `scope`.
* **withScope(String)**: Cambia el `scope` solicitado al realizar una solicitud de autenticación.
* **withAudience(String)**: Cambia la <Tooltip tip="Audience: Identificador único de la audience de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (API Identifier) para un Access Token." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip> solicitada al realizar una solicitud de autenticación.

<div id="database-options">
  ## Opciones de base de datos
</div>

* **withUsernameStyle(int)**: Define si debe solicitar solo correo electrónico, solo nombre de usuario o ambos. Los valores aceptados son `USERNAME` y `EMAIL`. De forma predeterminada, respetará la configuración del Dashboard para el parámetro `requires_username`.
* **loginAfterSignUp(boolean)**: Indica si, después de un evento `SignUp`, el usuario debe iniciar sesión automáticamente. El valor predeterminado es `true`.
* **initialScreen(int)**: Permite personalizar qué formulario aparecerá primero al iniciar Lock. Los valores aceptados son `LOG_IN`, `SIGN_UP` y `FORGOT_PASSWORD`. De forma predeterminada, `LOG_IN` es la pantalla inicial.
* **allowSignUp(boolean)**: Muestra el formulario de registro si hay una conexión de base de datos configurada y está permitida desde el Dashboard. El valor predeterminado es `true`.
* **allowLogIn(boolean)**: Muestra el formulario de inicio de sesión si hay una conexión de base de datos configurada. El valor predeterminado es `true`.
* **allowForgotPassword(boolean)**: Muestra el formulario de restablecimiento de contraseña si hay una conexión de base de datos configurada y está permitido desde el Dashboard. El valor predeterminado es `true`.
* **allowShowPassword(boolean)**: Muestra un botón para alternar la visibilidad del contenido de un campo de contraseña. El valor predeterminado es `true`.
* **setDefaultDatabaseConnection(String)**: Define cuál será la conexión de base de datos predeterminada. Esto es útil si tu aplicación tiene muchas conexiones de base de datos configuradas.
* **enableEnterpriseWebAuthenticationFor(List)**: Filtra las conexiones empresariales que iniciarán un flujo de autenticación web a través del navegador al hacer clic en ellas. Solo se permiten las estrategias &#39;ad&#39;, &#39;adfs&#39; o &#39;waad&#39;.
* **withSignUpFields(List)**: Muestra una segunda pantalla con campos adicionales para que el usuario los complete después de haber rellenado el nombre de usuario/correo electrónico y la contraseña en la pantalla de registro. Los valores enviados de esta manera pueden almacenarse en el perfil del usuario usando un atributo raíz o el atributo `user_metadata`. Para más información, consulta [Lock Android: Custom Fields at Signup](/docs/es/libraries/lock-android/lock-android-custom-fields-at-signup).
* **setPrivacyURL(String)**: Permite personalizar la URL de la política de privacidad. El valor predeterminado es `https://auth0.com/privacy`.
* **setTermsURL(String)**: Permite personalizar la URL de los Términos de servicio. El valor predeterminado es `https://auth0.com/terms`.
* **setSupportURL(String)**: Permite establecer una URL de soporte que se mostrará en caso de que se produzca un error no recuperable en Lock.
* **setMustAcceptTerms(boolean)**: Obliga al usuario a aceptar los términos y la política antes de registrarse. El valor predeterminado es `false`.
* **useLabeledSubmitButton(boolean)**: Si se establece en `true`, mostrará una etiqueta con el modo actual (registro/inicio de sesión) en el botón de envío en lugar de un icono. El valor predeterminado es `true`. Si la opción `hideMainScreenTitle` se establece en true, esta configuración se ignorará y se usará una etiqueta de todos modos.

<div id="oauth-options">
  ## Opciones de OAuth
</div>

* **withAuthStyle(String, int)**: Personaliza la apariencia de una conexión determinada (por nombre) con un estilo específico. Consulta [este documento sobre conexiones OAuth personalizadas](/docs/es/libraries/lock-android/lock-android-custom-theming) para obtener más información.
* **withConnectionScope(String, String...)**: Permite especificar `scopes` adicionales para un nombre de conexión determinado, que se solicitarán junto con los definidos en la configuración de la conexión en el [Auth0 Dashboard](https://manage.auth0.com/#). Los `scopes` no se validan de ninguna forma y deben ser reconocidos por el proveedor de autenticación indicado. Para ver una lista, consulta el [Auth0 Dashboard](https://manage.auth0.com/#) en la configuración de la conexión en cuestión.
* **withScheme(String)**: Permite cambiar el esquema del `redirect_uri` enviado en la llamada a `authorize`. De forma predeterminada, el esquema es `https`. Si se cambia esta configuración, también se deben actualizar el placeholder del manifiesto para `auth0Scheme` y las &quot;Allowed Callbacks URLs&quot; en el Dashboard de la aplicación de Auth0.

<div id="passwordless-options">
  ## Opciones de Passwordless
</div>

* **useCode()**: Envía un código en lugar de un enlace por correo electrónico/SMS para la autenticación <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>.
* **useLink()**: Envía un enlace en lugar de un código por correo electrónico/SMS para la autenticación Passwordless.
* **rememberLastLogin(boolean)**: Indica si se guardará el correo electrónico o el teléfono usados en la última autenticación exitosa para iniciar sesión automáticamente la próxima vez que se solicite una autenticación Passwordless.
