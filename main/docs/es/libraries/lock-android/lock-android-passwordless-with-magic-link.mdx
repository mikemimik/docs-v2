Para evitar pedirle al usuario que introduzca el código de un solo uso enviado para la autenticación <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip> en aplicaciones Android, incorporamos la posibilidad de enviar un enlace que el usuario puede tocar para iniciar sesión sin tener que introducir nada manualmente.

Estos enlaces incluyen el mismo **código** que se usaría en el flujo passwordless tradicional, pero, con la configuración correcta, se abrirán directamente en tu aplicación.

<div id="auth0-dashboard-configuration">
  ## Configuración de Auth0 Dashboard
</div>

Ve a la [configuración de la aplicación](https://manage.auth0.com/#/applications/\{yourClientId}/settings) y haz clic en **Mostrar configuración avanzada** al final de la página. Después, en la pestaña &quot;Configuración del dispositivo&quot;, tendrás que proporcionar tanto el nombre del paquete de la aplicación de Android como el hash de la clave del certificado.

* **Nombre del paquete de la aplicación**: Es el nombre del paquete, tal como se declara en el manifiesto de la aplicación. También está disponible en el archivo `app/build.gradle` como el atributo `applicationId`. Si ambos valores no coinciden, usa el de `applicationId`. Un ejemplo sería `com.example.android.myapp`
* **Hashes de claves**: Son las huellas SHA256 de los certificados de firma de nuestra aplicación de Android. Puedes incluir varias separándolas con comas. Aquí puedes especificar tanto las huellas del keystore de release como las del de debug. En la sección siguiente se explica cómo obtenerlas. Un ejemplo sería `DE:1A:5B:75:27:AA:48:D5:A6:72:2F:76:43:95:9B:79:C6:86:1A:5B:75:27:AA:48:D5:A6:73:FE`.

Después de establecer los valores, asegúrate de hacer clic en **Guardar cambios**.

<div id="getting-your-signing-certificates-fingerprint">
  ### Obtener la huella digital de tus certificados de firma
</div>

Usa el siguiente comando para generar la huella digital con la CLI `keytool` de Java:

`keytool -list -v -keystore my-release-key.keystore`

o bien obtén la clave de depuración por defecto:

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

El valor que requiere el Dashboard es el que aparece como &quot;SHA256&quot;. Puedes leer más sobre los almacenes de claves en este [artículo oficial](https://developer.android.com/studio/publish/app-signing).

<div id="enable-the-email-connection">
  ### Habilita la conexión de correo electrónico
</div>

Passwordless con Magic Link solo funciona con la conexión passwordless de tipo &quot;email&quot;. Ve a Dashboard &gt; [Autenticación](https://manage.auth0.com/#/applications/authentication) y, en las conexiones Passwordless, haz clic en **Email**. Se abrirá una ventana emergente y la plantilla de correo electrónico HTML + Liquid quedará habilitada para editarse. Verifica que el cuerpo contenga una condición como esta:

```liquid lines
{% if send == 'link' or send == 'link_ios' or send == 'link_android' %}
Your verification link is: {{ link }}
{% elsif send == 'code' %}
Your verification code is: {{ code }}
{% endif %}
```

<div id="configuring-the-sdk">
  ## Configuración del SDK
</div>

Ahora que la aplicación de Auth0 está configurada, sigue las instrucciones y configura PasswordlessLock con `Lock.Android`, tal como se muestra en la [documentación de passwordless](/docs/es/libraries/lock-android/lock-android-passwordless).

<div id="sdk-usage">
  ### Uso del SDK
</div>

Lock Passwordless autentica a los usuarios enviándoles un correo electrónico con un código de un solo uso que, en este caso, será un ENLACE en lugar de un código. Si has seguido la guía de Passwordless Classic, solo tendrás que eliminar la llamada a `useCode()` y sustituirla por `useLink()`.

Por último, abre el widget `PasswordlessLock` desde tu Activity.

```kotlin lines
startActivity(lock.newIntent(this))
```

Después de solicitar el LINK, la siguiente pantalla indicará que, para iniciar sesión, el usuario debe pulsarlo. Si esto no es posible, el usuario aún puede introducir el código que aparecerá después de hacer clic en el enlace del correo recibido.

<div id="optional-use-android-app-links">
  ## Opcional: Usar Android App Links
</div>

La biblioteca Lock está lista para usarse con App Links. Esta es una funcionalidad disponible en Android 6.0 (nivel de API 23) y versiones posteriores que permite que una app se establezca como el controlador predeterminado de un tipo determinado de enlace, sin mostrar el cuadro de diálogo de desambiguación que le pregunta al usuario si quiere usar el navegador o tu app para abrir el enlace.

Esto funciona siempre que el usuario no haya elegido ya una app predeterminada para gestionar ese patrón de URI en la configuración del dispositivo Android.

La gestión automática de enlaces requiere la cooperación entre tu app y el sitio web de Auth0. La app debe declarar la asociación con el sitio web y solicitar que el sistema la verifique. A su vez, el sitio web debe proporcionar esa verificación publicando un archivo de [Digital Asset Links](https://developers.google.com/digital-asset-links/).

Auth0 generará automáticamente el archivo de [Digital Asset Links](https://developers.google.com/digital-asset-links/) cuando configures nombre del paquete de la aplicación y hash de clave, como se mostró antes. Si has seguido todos los pasos de este artículo, el archivo debería estar disponible y se verificará la próxima vez que se instale tu aplicación de Android.

Puedes encontrar más información sobre App Links en la [documentación de Android](https://developer.android.com/training/app-links/index.html).
