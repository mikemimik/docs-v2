Para evitar pedir al usuario que introduzca la contraseña de un solo uso enviada para la autenticación <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip> en aplicaciones Android, incorporamos la posibilidad de enviar un enlace que el usuario puede tocar para iniciar sesión sin tener que introducir nada manualmente.

Estos enlaces incluyen el mismo **código** que se usaría en el flujo tradicional de passwordless, pero con la configuración correcta, se abrirán directamente en tu aplicación.

<div id="auth0-dashboard-configuration">
  ## Configuración del Auth0 Dashboard
</div>

Ve a [Application Settings](https://manage.auth0.com/#/applications/\{yourClientId}/settings) y haz clic en **Show Advanced Settings** al final de la página. Luego, en la pestaña &quot;Device Settings&quot;, deberás proporcionar tanto el **App Package Name** de la aplicación de Android como el **hash de clave** del certificado.

* **App Package Name**: Este es el nombre del paquete, tal como se declara en el manifiesto de la aplicación. También está disponible en el archivo `app/build.gradle` como el atributo `applicationId`. Si ambos valores no coinciden, usa el de `applicationId`. Un ejemplo sería `com.example.android.myapp`
* **Hashes de clave**: Son las huellas SHA256 de los certificados de firma de nuestra aplicación de Android. Puedes incluir varias separándolas con comas. Aquí puedes especificar tanto las huellas del keystore de producción como las del de depuración. En la sección siguiente se explica cómo obtenerlas. Un ejemplo sería `DE:1A:5B:75:27:AA:48:D5:A6:72:2F:76:43:95:9B:79:C6:86:1A:5B:75:27:AA:48:D5:A6:73:FE`.

Después de establecer los valores, asegúrate de hacer clic en **Save Changes**.

<div id="getting-your-signing-certificates-fingerprint">
  ### Cómo obtener la huella digital de tus certificados de firma
</div>

Usa el siguiente comando para generar la huella digital con la CLI de Java `keytool`:

`keytool -list -v -keystore my-release-key.keystore`

o bien usa la clave de depuración predeterminada:

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

El valor que requiere el Dashboard es el que aparece como &quot;SHA256&quot;. Puedes leer más sobre los keystores en este [artículo oficial](https://developer.android.com/studio/publish/app-signing).

<div id="enable-the-email-connection">
  ### Habilita la conexión de correo electrónico
</div>

Passwordless con Magic Link solo funciona con la conexión passwordless de tipo &quot;email&quot;. Ve a Dashboard &gt; [Autenticación](https://manage.auth0.com/#/applications/authentication) en la sección de conexiones passwordless y haz clic en **Email**. Se abrirá una ventana emergente y la plantilla de correo electrónico en HTML + Liquid se podrá editar. Verifica que el cuerpo del mensaje contenga una condición como esta:

```liquid lines
{% if send == 'link' or send == 'link_ios' or send == 'link_android' %}
Your verification link is: {{ link }}
{% elsif send == 'code' %}
Your verification code is: {{ code }}
{% endif %}
```

<div id="configuring-the-sdk">
  ## Configuración del SDK
</div>

Ahora que la aplicación de Auth0 está configurada, sigue las instrucciones y configura PasswordlessLock con `Lock.Android`, como se muestra en la [documentación de passwordless](/docs/es/libraries/lock-android/lock-android-passwordless).

<div id="sdk-usage">
  ### Uso del SDK
</div>

Passwordless de Lock autentica a los usuarios enviándoles un correo electrónico con una contraseña de un solo uso que, en este caso, será un ENLACE en lugar de un código. Si has seguido la guía de Passwordless Classic, solo tendrás que eliminar la llamada a `useCode()` y reemplazarla por `useLink()`.

Por último, abre el widget `PasswordlessLock` desde tu Activity.

```kotlin lines
startActivity(lock.newIntent(this))
```

Después de solicitar el ENLACE, la siguiente pantalla indicará que, para iniciar sesión, el usuario debe pulsarlo. Si esto no es posible, el usuario también puede introducir el código que aparecerá después de hacer clic en el enlace del correo electrónico recibido.

<div id="optional-use-android-app-links">
  ## Opcional: Usar Android App Links
</div>

La biblioteca Lock está lista para usarse con App Links. Esta funcionalidad está disponible en Android 6.0 (nivel de API 23) y versiones posteriores, y permite que una aplicación se establezca como la aplicación predeterminada para un tipo específico de enlace, sin mostrar el cuadro de diálogo de desambiguación que le pregunta al usuario si quiere usar el navegador o tu aplicación para abrirlo.

Esto funciona siempre que el usuario no haya elegido ya una aplicación predeterminada para gestionar ese patrón de URI en la configuración del dispositivo Android.

La gestión automática de enlaces requiere la colaboración de tu aplicación y del sitio web de Auth0. La aplicación debe declarar la asociación con el sitio web y solicitar que el sistema la verifique. A su vez, el sitio web debe proporcionar esa verificación publicando un archivo de [Digital Asset Links](https://developers.google.com/digital-asset-links/).

Auth0 generará automáticamente el archivo de [Digital Asset Links](https://developers.google.com/digital-asset-links/) cuando configures App Package Name y hash de clave, como se mostró anteriormente. Si has seguido todos los pasos de este artículo, el archivo debería estar disponible y se verificará la próxima vez que se instale tu aplicación para Android.

Puedes encontrar más información sobre App Links en la [documentación de Android](https://developer.android.com/training/app-links/index.html).
