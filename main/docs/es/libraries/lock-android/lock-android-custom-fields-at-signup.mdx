---
description: Agregar campos adicionales al registro con Lock.Android
title: 'Lock.Android: Campos personalizados en el registro'
---

**Lock.Android** le permite especificar campos adicionales que el usuario debe completar antes de crear una nueva cuenta. Los campos adicionales se mostrarán en una segunda pantalla después de que el usuario complete los campos básicos (correo electrónico, username y contraseña).

<div id="create-the-custom-fields">
  ## Crea los campos personalizados
</div>

Crea un nuevo objeto `CustomField` pasando estos 4 parámetros obligatorios.

1. Icono: El `int` que apunta al recurso que quieres usar como icono (haz que sea pequeño).
2. Tipo: El `FieldType` que se usará en este campo. El tipo define la disposición del teclado y, a veces, la validación de la entrada.
3. Clave: El `String` que identifica este valor en el JSON de resultado. ¡No debe repetirse! Si se repiten claves de campo, el segundo campo se eliminará de la lista.
4. Texto de ayuda: El `@StringRes` del texto que se mostrará como ayuda en el campo.

Además, puedes especificar dónde se almacenará el campo en el perfil del usuario. Consulta la sección [Almacenamiento](#storage) más abajo para obtener más detalles.

También puedes crear campos que no se mostrarán al usuario, pero que pueden usarse para incluir metadatos adicionales de forma dinámica después del registro.

```kotlin lines
// Campo normal
val nameField = CustomField(R.drawable.ic_name, FieldType.TYPE_NAME, "first_name", R.string.hint_first_name, CustomField.Storage.PROFILE_ROOT)

// Campo oculto
val androidField = HiddenField("android_version", Build.VERSION.SDK_INT.toString(), Storage.USER_METADATA)

val customFields = listOf(nameField, androidField)
```

Repite los pasos anteriores tantas veces como campos necesites.

<div id="use-the-custom-fields">
  ## Usa los campos personalizados
</div>

Pasa la lista de campos personalizados a la instancia de Lock mientras la configuras, usando el método `withSignUpFields()`.

```kotlin lines
val lock = Lock.newBuilder(auth0, callback)
              .withSignUpFields(customFields)
              //...
              .build(this)
```

Si ha habilitado el registro en el Dashboard de la aplicación, se pedirá a los usuarios que completen los campos restantes después de completar los campos básicos (correo electrónico/username, contraseña) y hacer clic en **Submit**. El usuario debe completar todos los campos personalizados antes de poder finalizar el registro.

Durante el registro, los campos adicionales se adjuntan al atributo `user_metadata` o se establecen directamente en la raíz del perfil de usuario, según la opción de almacenamiento elegida. Puede acceder a estas propiedades consultando el perfil del usuario en cualquier momento, incluso desde el Dashboard en la sección Users.

<div id="field-types">
  ## Tipos de campo
</div>

Cada campo personalizado solo puede tener un `FieldType` asociado.

* TYPE&#95;NAME
* TYPE&#95;NUMBER
* TYPE&#95;PHONE&#95;NUMBER
* TYPE&#95;EMAIL

<div id="storage">
  ## Almacenamiento
</div>

Cada campo personalizado solo puede tener un `Storage` asociado. Puedes optar por almacenarlo en el nivel raíz, en un atributo raíz del perfil, o dentro del atributo `user_metadata`. Para especificar la ubicación de almacenamiento, usa el constructor de cinco parámetros y pasa el parámetro `Storage` que prefieras. De forma predeterminada, los campos se almacenarán dentro del atributo `user_metadata`.

Opciones disponibles:

* PROFILE&#95;ROOT
* USER&#95;METADATA (predeterminado)

Para que los campos se guarden en el nivel raíz del perfil del usuario, sus claves deben coincidir con las que se enumeran en la [documentación del endpoint](https://auth0.com/docs/api/authentication#signup).
