La versión 2 del Auth0 Android SDK incluye muchas mejoras para integrar Auth0 en tu aplicación Android. **Esta actualización incluye cambios incompatibles.** Revisa esta guía para entender los cambios necesarios al migrar a la v2.

<div id="requirements-changes">
  ## Cambios en los requisitos
</div>

La v2 requiere la versión 21 o posterior de la API de Android y Java 8+.

Esto es lo que necesitas en `build.gradle` para generar bytecode de Java 8 para Android y para los plugins de Kotlin, respectivamente:

```kotlin lines
android {
  compileOptions {
      sourceCompatibility JavaVersion.VERSION_1_8
      targetCompatibility JavaVersion.VERSION_1_8
  }

  kotlinOptions {
      jvmTarget = '1.8'
  }
}
```

<div id="open-id-connect-only">
  ## Solo OpenID Connect
</div>

v2 solo admite aplicaciones conformes con OIDC. Al usar este SDK para autenticar usuarios, solo se utilizarán el pipeline actual de autenticación de Auth0 y sus endpoints. Hemos eliminado el método `setOIDCConformant(boolean value)` de la clase `Auth0`.

Obtén más información sobre el [protocolo OpenID Connect](/docs/es/authenticate/protocols/openid-connect-protocol).

<div id="authorization-code-with-pkce">
  ## Código de autorización con PKCE
</div>

v2 solo admite el [flujo de código de autorización con Proof Key for Code Exchange (PKCE)](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce). Eliminamos el método `withResponseType(@ResponseType int type)` de `WebAuthProvider.Builder`.

<div id="removal-of-webview-support">
  ## Eliminación de la compatibilidad con WebView
</div>

Eliminamos la capacidad obsoleta de autenticarse mediante un componente `WebView`. En su lugar, siempre se utilizarán aplicaciones de navegador externas para reforzar la seguridad y la confianza del usuario.

<div id="removal-of-legacy-authentication-api-support">
  ## Eliminación de la compatibilidad con la Legacy Authentication API
</div>

Eliminamos métodos y clases específicos para realizar llamadas a cualquiera de las [Authentication APIs](https://auth0.com/docs/api/authentication) clasificadas como Legacy. Consulta la lista completa en la sección **Clases e interfaces eliminadas** a continuación.

<div id="request-interfaces-changes">
  ## Cambios en las interfaces de solicitud
</div>

El paquete `com.auth0.android.request` define las interfaces principales para crear y ejecutar solicitudes HTTP. Históricamente, una incidencia común ha sido la imposibilidad de añadir encabezados personalizados y parámetros de solicitud a todos los tipos de solicitud. Hemos refactorizado las interfaces de solicitud para permitir que cualquier solicitud se pueda personalizar con encabezados adicionales y parámetros de solicitud. Ten en cuenta que estos parámetros ahora deben ser de tipo `String`.

La interfaz principal `Request` ahora especifica estos métodos:

* `public fun addParameters(parameters: Map<String, String>): Request<T, U>`
* `public fun addParameter(name: String, value: String): Request<T, U>`
* `public fun addHeader(name: String, value: String): Request<T, U>`

Eliminamos estas interfaces:

* `ParameterizableRequest`
* `AuthRequest`

Como resultado, `AuthenticationAPIClient` incluye muchos cambios en los tipos de retorno de sus métodos. Estos son algunos ejemplos:

* Los métodos que devolvían `ParameterizableRequest`, `TokenRequest` o `DatabaseConnectionRequest` ahora devuelven un `Request`

* Los métodos que devolvían `AuthRequest` ahora devuelven `AuthenticationRequest`

Si estás usando directamente el tipo de retorno de cualquiera de estos métodos, debes cambiar tu código para que espere el tipo indicado arriba. Si estás encadenando una llamada a `start` o `execute` para realizar la solicitud, no se requiere ningún cambio.

La interfaz `AuthenticationRequest` ya no tiene el método `setAccessToken("{ACCESS-TOKEN}")`. Este método se usaba para establecer el token en la solicitud enviada al endpoint heredado [/oauth/access&#95;token](https://auth0.com/docs/api/authentication#social-with-provider-s-access-token) de Authentication API, deshabilitado desde junio de 2017. Si necesitas enviar un parámetro con este nombre, usa `addParameter("access_token", "{ACCESS-TOKEN}")`.

Todas las clases que implementaban `ParameterizableRequest` o `AuthRequest` se han actualizado para adaptarse a estos cambios.

Consulta la lista completa en la sección **Clases e interfaces eliminadas** a continuación.

<div id="networking-client-customization">
  ## Personalización del cliente de red
</div>

v2 mejora la capacidad de personalizar cómo esta biblioteca realiza solicitudes HTTP. Una nueva interfaz, `NetworkingClient`, define el contrato para ejecutar solicitudes HTTP. Puedes configurar la clase `Auth0` con una instancia de `NetworkingClient`. Esto te permite proporcionar tu propia implementación y tener un control total sobre el cliente de red. Además, el cliente de red predeterminado que proporciona esta biblioteca admite varios puntos de personalización de ejemplo para facilitar la configuración de personalizaciones habituales.

<div id="timeout-configuration">
  ### Configuración del timeout
</div>

export const codeExample1 = `// Antes
val account = Auth0("{yourClientId}", "{yourDomain}")

account.connectTimeoutInSeconds = 30
account.readTimeoutInSeconds = 30

val authAPI = AuthenticationAPIClient(account)

// Después
val netClient = DefaultClient(
    connectTimeout = 30,
    readTimeout = 30
)

val account = Auth0("{yourClientId}", "{yourDomain}")
account.networkingClient = netClient`;

<div id="detailed-change-listing">
  ## Listado detallado de cambios
</div>

<div id="files-that-changed-their-package">
  ### Archivos cuyo paquete cambió
</div>

* Movimos la clase `com.auth0.android.authentication.request.ProfileRequest` a `com.auth0.android.request.ProfileRequest`.
* Movimos la clase `com.auth0.android.authentication.request.SignUpRequest` a `com.auth0.android.request.SignUpRequest`.

<div id="internal-package">
  ### Paquete interno
</div>

La versión anterior de esta biblioteca exponía algunas clases en el paquete `com.auth0.android.request.internal`. Nunca se pensó que formaran parte de la API pública de la biblioteca; la documentación de Java desaconsejaba su uso directo. Como la base de código se migró a Kotlin, ahora aprovechamos el modificador `internal` y lo usamos cuando es necesario. Estas clases pueden aparecer como clases `public` en proyectos de Java, pero no las uses. Siguen sin formar parte de la API pública de esta biblioteca.

No daremos soporte para estas clases. Las cambiaremos según sea necesario y sin previo aviso.

<div id="classes-and-interfaces-removed">
  ### Clases e interfaces eliminadas
</div>

* Se ha eliminado la clase `com.auth0.android.util.Base64`. Usa `android.util.Base64` en su lugar.
* Se ha eliminado la interfaz `com.auth0.android.request.ParameterizableRequest`. Hemos trasladado la capacidad de agregar encabezados y parámetros de solicitud a la interfaz `com.auth0.android.request.Request`.
* Se ha eliminado la interfaz `com.auth0.android.request.AuthRequest`. Usa en su lugar la interfaz `com.auth0.android.request.AuthenticationRequest`.
* Se ha eliminado la clase `com.auth0.android.provider.WebAuthActivity`. Para la authentication siempre se usarán aplicaciones de navegador externas.
* Se ha eliminado la clase `com.auth0.android.result.Delegation`. Se usaba como resultado de la solicitud al endpoint Legacy de Authentication API, deshabilitado desde junio de 2017.
* Se ha eliminado la clase `com.auth0.android.authentication.request.DelegationRequest`. Se usaba para representar la solicitud al endpoint Legacy de Authentication API, deshabilitado desde junio de 2017.
* La clase `com.auth0.android.util.Telemetry` se ha renombrado como `com.auth0.android.util.Auth0UserAgent`.
* Se ha eliminado la clase `com.auth0.android.request.AuthorizableRequest`. Puedes lograr el mismo resultado con `addHeader("Authorization", "Bearer {TOKEN_VALUE}")`.
* Se ha eliminado la clase `com.auth0.android.authentication.request.TokenRequest`. Hemos trasladado la capacidad de establecer un code verifier y cualquier encabezado o parámetro de solicitud a la interfaz `com.auth0.android.request.Request`.
* Se ha eliminado la clase `com.auth0.android.authentication.request.DatabaseConnectionRequest`. Hemos trasladado la capacidad de establecer cualquier encabezado o parámetro de solicitud a la interfaz `com.auth0.android.request.Request`.
* Se ha eliminado la clase `com.auth0.android.provider.VoidCallback`. Hemos reemplazado la posibilidad de usar una devolución de llamada que no recibe argumentos por `Callback<Void, AuthenticationException>`.
* Se ha eliminado la interfaz `com.auth0.android.request.ErrorBuilder`.
* Se ha eliminado la clase `com.auth0.android.RequestBodyBuildException`.
* Se ha eliminado la clase `com.auth0.android.util.CheckHelper`.
* Se ha eliminado la interfaz `com.auth0.android.util.JsonRequired`.
* Se ha eliminado la clase `com.auth0.android.util.JsonRequiredTypeAdapterFactory`.

<div id="constants-removed">
  ### Constantes eliminadas
</div>

* Se eliminó `ParameterBuilder.GRANT_TYPE_JWT`.
* Se eliminó `ParameterBuilder.ID_TOKEN_KEY`.
* Se eliminó `ParameterBuilder.DEVICE_KEY`.
* Se eliminó `ParameterBuilder.ACCESS_TOKEN_KEY`.
* Se eliminaron `ResponseType.CODE`, `ResponseType.ID_TOKEN` y `ResponseType.ACCESS_TOKEN`.

<div id="classes-and-interfaces-changed">
  ### Clases e interfaces modificadas
</div>

* `SignupRequest` ahora implementa `AuthenticationRequest` en lugar de `AuthRequest`, que ya se eliminó
* `AuthorizableRequest` ahora extiende `Request` en lugar de `ParameterizableRequest`, que ya se eliminó
* `BaseCallback` se ha marcado como obsoleta; usa `Callback` en su lugar.

<div id="constructors-changed">
  ### Constructores modificados
</div>

* Ya no puedes crear `AuthenticationAPIClient` a partir de un `Context`. Usa `AuthenticationAPIClient(auth0: Auth0)` en su lugar. Puedes crear una instancia de `Auth0` usando un `Context`.
* Ya no puedes crear `UsersAPIClient` a partir de un `Context`. Usa `UsersAPIClient(auth0: Auth0, token: String)` en su lugar. Puedes crear una instancia de `Auth0` usando un `Context`.
* `SignupRequest` ahora requiere que el segundo parámetro sea un `AuthenticationRequest`.
* `ProfileRequest` ahora requiere un `AuthenticationRequest` y un `Request<UserProfile, AuthenticationException>`.
* Ya no puedes crear `Credentials` con un valor de &quot;expires in&quot;. En su lugar, indica la fecha de expiración o &quot;expires at&quot;. Normalmente no crearás `Credentials` por tu cuenta.

<div id="methods-removed-or-changed">
  ### Métodos eliminados o modificados
</div>

<div id="auth0-methods-removed">
  #### Métodos de Auth0 eliminados
</div>

* Se han eliminado `setOIDCConformant(boolean enabled)` e `isOIDCConformant()`. El SDK ahora solo es compatible con aplicaciones conformes con OIDC.
* Se ha eliminado `doNotSendTelemetry()`. No tiene reemplazo.
* Se han eliminado `setWriteTimeoutInSeconds(seconds)` y `getWriteTimeoutInSeconds(seconds)`. No tienen reemplazo.
* Se han eliminado `setReadTimeoutInSeconds(seconds)` y `getReadTimeoutInSeconds(seconds)`. Puedes personalizar el `timeout` de lectura directamente en la clase `DefaultClient`. Usa `DefaultClient(readTimeout = 123)`.
* Se han eliminado `setConnectTimeoutInSeconds(seconds)` y `getConnectTimeoutInSeconds(seconds)`. Puedes personalizar el `timeout` de conexión directamente en la clase `DefaultClient`. Usa `DefaultClient(connectTimeout = 123)`.
* Se han eliminado `setLoggingEnabled(boolean enabled)` e `isLoggingEnabled()`. Puedes habilitar el logger del tráfico de red directamente en la clase `DefaultClient`. Usa `DefaultClient(enableLogging = true)`. Úsalo solo para depuración y nunca lo habilites en entornos de producción.
* Se han eliminado `setTLS12Enforced()` e `isTLS12Enforced()`. El SDK ahora es compatible con TLS moderno de forma predeterminada.

<div id="authenticationapiclient-methods-removed-or-changed">
  #### Métodos de AuthenticationAPIClient eliminados o modificados
</div>

* Se eliminó `setUserAgent(String)`. Puede configurar los encabezados que se enviarán directamente en cada instancia de `Request` con el método `addHeader("{NAME}", "{VALUE}")` o de forma global en la instancia del cliente de red mediante el parámetro del constructor `DefaultClient(defaultHeaders = mapOf())`.

Los métodos y clases específicos para llamar a cualquier [Authentication API](https://auth0.com/docs/api/authentication) catalogada como Legacy se eliminaron en la v2. Se eliminaron estos métodos:

* `delegation()`, `delegationWithIdToken("{ID-TOKEN}")`, `delegationWithRefreshToken("{REFRESH-TOKEN}")` y `delegationWithIdToken("{ID-TOKEN}", "{API-TYPE}}")`. Se eliminó la compatibilidad con los endpoints de Legacy Authentication API.
* `loginWithOAuthAccessToken("{TOKEN}", {CONNECTION})`. Para determinados proveedores sociales, puede usar `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")` en su lugar.
* `tokenInfo("{ID-TOKEN}")`. Use `userInfo("{ACCESS-TOKEN}")` en su lugar.

Los métodos que devolvían un `ParameterizableRequest` ahora devuelven un `Request`:

* `userInfo("{ACCESS-TOKEN}")`
* `revokeToken("{REFRESH-TOKEN}")`
* `renewAuth("{REFRESH-TOKEN}")`
* `passwordlessWithEmail("{EMAIL}", PasswordlessType, "{CONNECTION}")`
* `passwordlessWithSMS("{PHONE-NUNBER}", PasswordlessType, "{CONNECTION}")`
* `fetchJsonWebKeys()`

Los métodos que devolvían un `AuthRequest` ahora devuelven un `AuthenticationRequest`:

* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}", "{REALM-OR-CONNECTION}")`
* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}")`
* `loginWithOTP("{MFA-TOKEN}", "{OTP}")`
* `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")`
* `loginWithPhoneNumber("{PHONE-NUMBER}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`
* `loginWithEmail("{EMAIL}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`

Los métodos que devolvían un `DatabaseConnectionRequest` ahora devuelven un `Request`:

* `createUser("{EMAIL}", "{PASSWORD}", "{USERNAME}", "{CONNECTION}")`
* `resetPassword("{EMAIL}","{CONNECTION}")`

Los métodos que devolvían un `TokenRequest` ahora devuelven un `Request`:

* `token("{AUTHORIZATION-CODE}", "{CODE-VERIFIER}", "{REDIRECT-URI}")`

<div id="authenticationrequest-methods-removed">
  #### Métodos de AuthenticationRequest eliminados
</div>

* Se ha eliminado `addAuthenticationParameters(parameters)`. Usa `addParameters(parameters)` en su lugar.
* Se ha eliminado `setDevice("{DEVICE}")`. Usa `addParameter("device", "{VALUE}")` en su lugar.

<div id="webauthproviderbuilder-methods-removed">
  #### Métodos eliminados de WebAuthProvider.Builder
</div>

* `useCodeGrant(boolean useCodeGrant)`. No hay reemplazo; en v2 solo se admite el flujo Code + PKCE.
* `useBrowser(boolean useBrowser)`. No hay reemplazo; esta biblioteca ya no admite la autenticación con WebView.
* `useFullscreen(boolean useFullscreen)`. No hay reemplazo; esta biblioteca ya no admite la autenticación con WebView.
* `withResponseType(@ResponseType int type)`. No hay reemplazo; en v2 solo se admite el flujo Code + PKCE.
* Se eliminó `start(activity: Activity, callback: AuthCallback, requestCode: Int)`. Use `start(activity: Activity, callback: Callback<Credentials, AuthenticationException>)` en su lugar.

<div id="webauthproviderlogoutbuilder-methods-removed">
  #### Métodos eliminados de WebAuthProvider.LogoutBuilder
</div>

* `start(context: Context, callback: VoidCallback)`. Usa `start(context: Context, callback: Callback<Void, AuthenticationException>)` en su lugar.

<div id="webauthprovider-methods-removed">
  #### Métodos de `WebAuthProvider` eliminados
</div>

* Se eliminó `init(account: Auth0)`. Usa `login(account: Auth0)` en su lugar.
* Se eliminó `init(context: Context)`. Usa `login(account: Auth0)` en su lugar.
* Se eliminó `resume(requestCode: Int, resultCode: Int, intent: Intent)`. Usa `resume(intent: Intent)` en su lugar.
* Se eliminó `init(account: Auth0)`. Usa `login(account: Auth0)` en su lugar.

<div id="parameterbuilder-methods-removed">
  #### Se eliminaron métodos de ParameterBuilder
</div>

* Se eliminó `setDevice("{DEVICE}")`. En su lugar, usa `set("device", "{VALUE}")`.

<div id="usersapiclient-methods-changed">
  #### Métodos de UsersAPIClient modificados
</div>

* Se ha eliminado `setUserAgent(String)`. Puedes establecer los encabezados que se enviarán directamente en cada instancia de `Request` mediante el método `addHeader("{NAME}", "{VALUE}")` o de forma global en la instancia del cliente de red a través del parámetro del constructor `DefaultClient(defaultHeaders = mapOf())`.

Los métodos que devolvían un `ParameterizableRequest` ahora devuelven un `Request`:

* `link("{PRIMARY-USER-ID}", "{SECONDARY-TOKEN}")`
* `unlink("{PRIMARY-USER-ID}", "{SECONDARY-USER-ID}", "{SECONDARY-PROVIDER}")`
* `updateMetadata("{USER-ID}", userMetadata)`
* `getProfile("{USER-ID}")`

<div id="requestfactory-methods-removed-or-changed">
  #### Métodos de `RequestFactory` eliminados o modificados
</div>

La clase `RequestFactory` contiene métodos para facilitar la realización de solicitudes HTTP, incluida la serialización y deserialización del cuerpo JSON de la solicitud y la respuesta. Como forma parte del paquete `com.auth0.android.request.internal`, no está pensada para uso público, pero, al ser un tipo público, a continuación se resume lo que cambió.

* Todos los métodos de solicitud se han refactorizado para desacoplarlos de una biblioteca específica de solicitudes HTTP (por ejemplo, `OkHttp`)
* Todos los métodos de solicitud ahora devuelven una `Request`.
* Todos los métodos de solicitud ahora están en minúsculas (por ejemplo, `POST()` -&gt; `post()`).
* El método `authenticationPOST` se eliminó sin sustitución, ya que todas las instancias de `Request` pueden parametrizarse con los encabezados que sean necesarios.

<div id="profilerequest-methods-changed">
  #### Métodos de ProfileRequest modificados
</div>

* El método `addParameters` ahora requiere que el valor sea un Map de String a String, en lugar de uno de String a Object (`addParameters(mapOf("key" to "val"))`)

<div id="signuprequest-methods-changed">
  #### Métodos de SignUpRequest modificados
</div>

Los métodos que establecen parámetros ahora requieren que el valor sea un Map de String a String, en lugar de String a Object:

* `addAuthenticationParameters(mapOf("key" to "val"))`
* `addSignupParameters(mapOf("key" to "val"))`
* `addParameters(mapOf("key" to "val"))`

Además, se eliminó `setDevice("device")`. Usa `addParameter("device", "{VALUE}")` en su lugar.
