La versión 2 del Auth0 Android SDK incluye numerosas mejoras para integrar Auth0 en tu aplicación de Android. **Esta actualización introduce cambios incompatibles.** Consulta esta guía para conocer los cambios necesarios al migrar a la versión 2.

<div id="requirements-changes">
  ## Cambios en los requisitos
</div>

La v2 requiere Android API 21 o posterior y Java 8+.

Esto es lo que necesitas en `build.gradle` para generar bytecode de Java 8 en Android y en los plugins de Kotlin, respectivamente:

```kotlin lines
android {
  compileOptions {
      sourceCompatibility JavaVersion.VERSION_1_8
      targetCompatibility JavaVersion.VERSION_1_8
  }

  kotlinOptions {
      jvmTarget = '1.8'
  }
}
```

<div id="open-id-connect-only">
  ## Solo OpenID Connect
</div>

v2 solo admite aplicaciones compatibles con OIDC. Al usar este SDK para autenticar usuarios, solo se utilizarán el pipeline y los endpoints actuales de autenticación de Auth0. Hemos eliminado el método `setOIDCConformant(boolean value)` de la clase `Auth0`.

Obtén más información sobre el [protocolo OpenID Connect](/docs/es/authenticate/protocols/openid-connect-protocol).

<div id="authorization-code-with-pkce">
  ## Código de autorización con PKCE
</div>

v2 solo es compatible con el [flujo de código de autorización con PKCE (Proof Key for Code Exchange)](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce). Eliminamos el método `withResponseType(@ResponseType int type)` de `WebAuthProvider.Builder`.

<div id="removal-of-webview-support">
  ## Eliminación de la compatibilidad con WebView
</div>

Eliminamos la función obsoleta de autenticarse mediante un componente `WebView`. En su lugar, siempre se usarán aplicaciones de navegador externas para reforzar la seguridad y la confianza del usuario.

<div id="removal-of-legacy-authentication-api-support">
  ## Eliminación del soporte para las API de autenticación heredadas
</div>

Eliminamos métodos y clases específicos para llamar a cualquiera de las [authentication APIs](https://auth0.com/docs/api/authentication) clasificadas como Legacy. Consulta la lista completa en la sección **Clases e interfaces eliminadas** a continuación.

<div id="request-interfaces-changes">
  ## Cambios en las interfaces de solicitud
</div>

El paquete `com.auth0.android.request` define las interfaces principales para crear y ejecutar solicitudes HTTP. Históricamente, una incidencia común ha sido la imposibilidad de agregar encabezados personalizados y parámetros de solicitud a todos los tipos de solicitud. Hemos refactorizado las interfaces de solicitud para que cualquier solicitud pueda personalizarse con encabezados adicionales y parámetros de solicitud. Ten en cuenta que estos parámetros ahora deben ser de tipo `String`.

La interfaz principal `Request` ahora especifica estos métodos:

* `public fun addParameters(parameters: Map<String, String>): Request<T, U>`
* `public fun addParameter(name: String, value: String): Request<T, U>`
* `public fun addHeader(name: String, value: String): Request<T, U>`

Eliminamos estas interfaces:

* `ParameterizableRequest`
* `AuthRequest`

Como resultado, `AuthenticationAPIClient` incluye muchos cambios en el tipo de retorno de sus métodos. Estos son algunos ejemplos:

* Todos los métodos que devolvían `ParameterizableRequest`, `TokenRequest` o `DatabaseConnectionRequest` ahora devuelven un `Request`

* Todo método que devolvía `AuthRequest` ahora devuelve `AuthenticationRequest`

Si estás usando directamente el tipo de retorno de cualquiera de estos métodos, debes cambiar tu código para que espere el tipo indicado arriba. Si estás encadenando una llamada a `start` o `execute` para realizar la solicitud, no hace falta hacer cambios.

La interfaz `AuthenticationRequest` ya no incluye el método `setAccessToken("{ACCESS-TOKEN}")`. Este método se usaba para establecer el token en la solicitud enviada al legacy endpoint [/oauth/access&#95;token](https://auth0.com/docs/api/authentication#social-with-provider-s-access-token) de Authentication API, deshabilitado desde junio de 2017. Si necesitas enviar un parámetro con este nombre, usa `addParameter("access_token", "{ACCESS-TOKEN}")`.

Todas las clases que implementaban `ParameterizableRequest` o `AuthRequest` se han actualizado para adaptarse a estos cambios.

Consulta la lista completa en la sección **Clases e interfaces eliminadas** a continuación.

<div id="networking-client-customization">
  ## Personalización del cliente de red
</div>

La versión 2 mejora la capacidad de personalizar la forma en que esta biblioteca realiza solicitudes HTTP. Una nueva interfaz, `NetworkingClient`, define el contrato para ejecutar solicitudes HTTP. Puedes configurar la clase `Auth0` con una instancia de `NetworkingClient`. Esto te permite proporcionar tu propia implementación y tener un control total sobre el cliente de red. Además, el cliente de red predeterminado que proporciona esta biblioteca admite varios ejemplos de personalización para facilitar la configuración de los puntos de personalización más comunes.

<div id="timeout-configuration">
  ### Configuración del timeout
</div>

export const codeExample1 = `// Antes
val account = Auth0("{yourClientId}", "{yourDomain}")

account.connectTimeoutInSeconds = 30
account.readTimeoutInSeconds = 30

val authAPI = AuthenticationAPIClient(account)

// Después
val netClient = DefaultClient(
    connectTimeout = 30,
    readTimeout = 30
)

val account = Auth0("{yourClientId}", "{yourDomain}")
account.networkingClient = netClient`;

<div id="detailed-change-listing">
  ## Listado detallado de cambios
</div>

<div id="files-that-changed-their-package">
  ### Archivos cuyo paquete cambió
</div>

* Movimos la clase `com.auth0.android.authentication.request.ProfileRequest` a `com.auth0.android.request.ProfileRequest`.
* Movimos la clase `com.auth0.android.authentication.request.SignUpRequest` a `com.auth0.android.request.SignUpRequest`.

<div id="internal-package">
  ### Paquete interno
</div>

La versión anterior de esta biblioteca exponía algunas clases en el paquete `com.auth0.android.request.internal` . Nunca se pretendió que formaran parte de la API pública de la biblioteca; la documentación de Java desaconsejaba su uso directo. Como la base de código se había migrado a Kotlin, ahora aprovechamos el modificador `internal` y lo usamos cuando es necesario. Estas clases pueden aparecer como clases `public` en proyectos Java, pero no las uses. Siguen sin formar parte de la API pública de esta biblioteca.

No proporcionaremos soporte para estas clases. Las modificaremos según sea necesario y sin previo aviso.

<div id="classes-and-interfaces-removed">
  ### Clases e interfaces eliminadas
</div>

* Se ha eliminado la clase `com.auth0.android.util.Base64`. Utilice `android.util.Base64` en su lugar.
* Se ha eliminado la interfaz `com.auth0.android.request.ParameterizableRequest`. Hemos trasladado la capacidad de agregar encabezados y parámetros de la solicitud a la interfaz `com.auth0.android.request.Request`.
* Se ha eliminado la interfaz `com.auth0.android.request.AuthRequest`. Utilice en su lugar la interfaz `com.auth0.android.request.AuthenticationRequest`.
* Se ha eliminado la clase `com.auth0.android.provider.WebAuthActivity`. Siempre se usarán aplicaciones de navegador externas para la autenticación.
* Se ha eliminado la clase `com.auth0.android.result.Delegation`. Se usaba como resultado de la solicitud al endpoint heredado de Authentication API, deshabilitado desde junio de 2017.
* Se ha eliminado la clase `com.auth0.android.authentication.request.DelegationRequest`. Se usaba para representar la solicitud al endpoint heredado de Authentication API, deshabilitado desde junio de 2017.
* La clase `com.auth0.android.util.Telemetry` ha pasado a llamarse `com.auth0.android.util.Auth0UserAgent`.
* Se ha eliminado la clase `com.auth0.android.request.AuthorizableRequest`. Puede lograr el mismo resultado usando `addHeader("Authorization", "Bearer {TOKEN_VALUE}")`.
* Se ha eliminado la clase `com.auth0.android.authentication.request.TokenRequest`. Hemos trasladado la capacidad de establecer un Code Verifier y cualquier encabezado y parámetro de la solicitud a la interfaz `com.auth0.android.request.Request`.
* Se ha eliminado la clase `com.auth0.android.authentication.request.DatabaseConnectionRequest`. Hemos trasladado la capacidad de establecer cualquier encabezado y parámetro de la solicitud a la interfaz `com.auth0.android.request.Request`.
* Se ha eliminado la clase `com.auth0.android.provider.VoidCallback`. Hemos sustituido la posibilidad de usar una devolución de llamada que no acepta argumentos por `Callback<Void, AuthenticationException>`.
* Se ha eliminado la interfaz `com.auth0.android.request.ErrorBuilder`.
* Se ha eliminado la clase `com.auth0.android.RequestBodyBuildException`.
* Se ha eliminado la clase `com.auth0.android.util.CheckHelper`.
* Se ha eliminado la interfaz `com.auth0.android.util.JsonRequired`.
* Se ha eliminado la clase `com.auth0.android.util.JsonRequiredTypeAdapterFactory`.

<div id="constants-removed">
  ### Constantes eliminadas
</div>

* Se eliminó `ParameterBuilder.GRANT_TYPE_JWT`.
* Se eliminó `ParameterBuilder.ID_TOKEN_KEY`.
* Se eliminó `ParameterBuilder.DEVICE_KEY`.
* Se eliminó `ParameterBuilder.ACCESS_TOKEN_KEY`.
* Se eliminaron `ResponseType.CODE`, `ResponseType.ID_TOKEN` y `ResponseType.ACCESS_TOKEN`.

<div id="classes-and-interfaces-changed">
  ### Clases e interfaces modificadas
</div>

* `SignupRequest` ahora implementa `AuthenticationRequest` en lugar de `AuthRequest`, que ya se ha eliminado
* `AuthorizableRequest` ahora extiende `Request` en lugar de `ParameterizableRequest`, que ya se ha eliminado
* `BaseCallback` se ha marcado como obsoleto; usa `Callback` en su lugar.

<div id="constructors-changed">
  ### Constructores modificados
</div>

* Ya no puedes crear `AuthenticationAPIClient` a partir de un `Context`. Usa `AuthenticationAPIClient(auth0: Auth0)` en su lugar. Puedes crear una instancia de `Auth0` con un `Context`.
* Ya no puedes crear `UsersAPIClient` a partir de un `Context`. Usa `UsersAPIClient(auth0: Auth0, token: String)` en su lugar. Puedes crear una instancia de `Auth0` con un `Context`.
* `SignupRequest` ahora requiere que el segundo parámetro sea un `AuthenticationRequest`.
* `ProfileRequest` ahora requiere un `AuthenticationRequest` y un `Request<UserProfile, AuthenticationException>`.
* Ya no puedes crear `Credentials` con un valor de &quot;expires in&quot;. En su lugar, indica la fecha de expiración o &quot;expires at&quot;. Normalmente no crearás `Credentials` por tu cuenta.

<div id="methods-removed-or-changed">
  ### Métodos eliminados o modificados
</div>

<div id="auth0-methods-removed">
  #### Métodos de Auth0 eliminados
</div>

* Se han eliminado `setOIDCConformant(boolean enabled)` e `isOIDCConformant()`. El SDK ahora solo admite aplicaciones compatibles con OIDC.
* Se ha eliminado `doNotSendTelemetry()`. No tiene reemplazo.
* Se han eliminado `setWriteTimeoutInSeconds(seconds)` y `getWriteTimeoutInSeconds(seconds)`. No tienen reemplazo.
* Se han eliminado `setReadTimeoutInSeconds(seconds)` y `getReadTimeoutInSeconds(seconds)`. Puedes personalizar el tiempo de espera de lectura directamente en la clase `DefaultClient`. Usa `DefaultClient(readTimeout = 123)`.
* Se han eliminado `setConnectTimeoutInSeconds(seconds)` y `getConnectTimeoutInSeconds(seconds)`. Puedes personalizar el tiempo de espera de conexión directamente en la clase `DefaultClient`. Usa `DefaultClient(connectTimeout = 123)`.
* Se han eliminado `setLoggingEnabled(boolean enabled)` e `isLoggingEnabled()`. Puedes habilitar el registrador del tráfico de red directamente en la clase `DefaultClient`. Usa `DefaultClient(enableLogging = true)`. Úsalo solo para depuración y nunca lo habilites en entornos de producción.
* Se han eliminado `setTLS12Enforced()` e `isTLS12Enforced()`. El SDK ahora admite TLS moderno de forma predeterminada.

<div id="authenticationapiclient-methods-removed-or-changed">
  #### Métodos de `AuthenticationAPIClient` eliminados o modificados
</div>

* Se eliminó `setUserAgent(String)`. Puedes configurar los encabezados que se enviarán directamente en cada instancia de `Request` con el método `addHeader("{NAME}", "{VALUE}")` o de forma global en la instancia del cliente de red mediante el parámetro del constructor `DefaultClient(defaultHeaders = mapOf())`.

Los métodos y clases específicos para llamar a cualquier [Authentication APIs](https://auth0.com/docs/api/authentication) categorizada como Legacy se eliminaron en la v2. Se eliminaron estos métodos:

* `delegation()`, `delegationWithIdToken("{ID-TOKEN}")`, `delegationWithRefreshToken("{REFRESH-TOKEN}")` y `delegationWithIdToken("{ID-TOKEN}", "{API-TYPE}}")`. Se eliminó la compatibilidad con los endpoints de Legacy Authentication API.
* `loginWithOAuthAccessToken("{TOKEN}", {CONNECTION})`. Para algunos proveedores sociales, puedes usar `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")` en su lugar.
* `tokenInfo("{ID-TOKEN}")`. Usa `userInfo("{ACCESS-TOKEN}")` en su lugar.

Los métodos que devolvían un `ParameterizableRequest` ahora devuelven un `Request`:

* `userInfo("{ACCESS-TOKEN}")`
* `revokeToken("{REFRESH-TOKEN}")`
* `renewAuth("{REFRESH-TOKEN}")`
* `passwordlessWithEmail("{EMAIL}", PasswordlessType, "{CONNECTION}")`
* `passwordlessWithSMS("{PHONE-NUNBER}", PasswordlessType, "{CONNECTION}")`
* `fetchJsonWebKeys()`

Los métodos que devolvían un `AuthRequest` ahora devuelven un `AuthenticationRequest`:

* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}", "{REALM-OR-CONNECTION}")`
* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}")`
* `loginWithOTP("{MFA-TOKEN}", "{OTP}")`
* `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")`
* `loginWithPhoneNumber("{PHONE-NUMBER}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`
* `loginWithEmail("{EMAIL}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`

Los métodos que devolvían un `DatabaseConnectionRequest` ahora devuelven un `Request`:

* `createUser("{EMAIL}", "{PASSWORD}", "{USERNAME}", "{CONNECTION}")`
* `resetPassword("{EMAIL}","{CONNECTION}")`

Los métodos que devolvían un `TokenRequest` ahora devuelven un `Request`:

* `token("{AUTHORIZATION-CODE}", "{CODE-VERIFIER}", "{REDIRECT-URI}")`

<div id="authenticationrequest-methods-removed">
  #### Se eliminaron métodos de `AuthenticationRequest`
</div>

* Se eliminó `addAuthenticationParameters(parameters)`. Usa `addParameters(parameters)` en su lugar.
* `setDevice("{DEVICE}").` Usa `addParameter("device", "{VALUE}")` en su lugar.

<div id="webauthproviderbuilder-methods-removed">
  #### Métodos eliminados de WebAuthProvider.Builder
</div>

* `useCodeGrant(boolean useCodeGrant)`. No hay reemplazo; en v2 solo se admite el flujo Code + PKCE.
* `useBrowser(boolean useBrowser)`. No hay reemplazo; esta biblioteca ya no admite la autenticación con WebView.
* `useFullscreen(boolean useFullscreen)`. No hay reemplazo; esta biblioteca ya no admite la autenticación con WebView.
* `withResponseType(@ResponseType int type)`. No hay reemplazo; en v2 solo se admite el flujo Code + PKCE.
* Se eliminó `start(activity: Activity, callback: AuthCallback, requestCode: Int)`. Usa `start(activity: Activity, callback: Callback<Credentials, AuthenticationException>)` en su lugar.

<div id="webauthproviderlogoutbuilder-methods-removed">
  #### Métodos eliminados de WebAuthProvider.LogoutBuilder
</div>

* `start(context: Context, callback: VoidCallback)`. Utiliza `start(context: Context, callback: Callback<Void, AuthenticationException>)` en su lugar.

<div id="webauthprovider-methods-removed">
  #### Métodos eliminados de WebAuthProvider
</div>

* Se eliminó `init(account: Auth0)`. Usa `login(account: Auth0)` en su lugar.
* Se eliminó `init(context: Context)`. Usa `login(account: Auth0)` en su lugar.
* Se eliminó `resume(requestCode: Int, resultCode: Int, intent: Intent)`. Usa `resume(intent: Intent)` en su lugar.
* Se eliminó `init(account: Auth0)`. Usa `login(account: Auth0)` en su lugar.

<div id="parameterbuilder-methods-removed">
  #### Métodos de ParameterBuilder eliminados
</div>

* Se ha eliminado `setDevice("{DEVICE}")`. Usa `set("device", "{VALUE}")` en su lugar.

<div id="usersapiclient-methods-changed">
  #### Métodos de UsersAPIClient modificados
</div>

* Se ha eliminado `setUserAgent(String)`. Puede establecer los encabezados que se enviarán directamente en cada instancia de `Request` mediante el método `addHeader("{NAME}", "{VALUE}")` o de forma global en la instancia del cliente de red a través del parámetro del constructor `DefaultClient(defaultHeaders = mapOf())`.

Los métodos que devolvían un `ParameterizableRequest` ahora devuelven un `Request`:

* `link("{PRIMARY-USER-ID}", "{SECONDARY-TOKEN}")`
* `unlink("{PRIMARY-USER-ID}", "{SECONDARY-USER-ID}", "{SECONDARY-PROVIDER}")`
* `updateMetadata("{USER-ID}", userMetadata)`
* `getProfile("{USER-ID}")`

<div id="requestfactory-methods-removed-or-changed">
  #### Métodos de `RequestFactory` eliminados o modificados
</div>

La clase `RequestFactory` contiene métodos que facilitan la realización de solicitudes HTTP, incluida la serialización y deserialización del cuerpo de la solicitud JSON y de la respuesta. Como forma parte del paquete `com.auth0.android.request.internal`, no está pensada para uso público, pero, al ser un tipo público, a continuación se resumen los cambios.

* Todos los métodos de solicitud se han refactorizado para desacoplarse de una biblioteca de solicitudes HTTP concreta (por ejemplo, `OkHttp`)
* Todos los métodos de solicitud ahora devuelven un `Request`.
* Todos los métodos de solicitud ahora están en minúsculas (por ejemplo, `POST()` -&gt; `post()`).
* El método `authenticationPOST` se eliminó sin reemplazo, ya que todas las instancias de `Request` pueden parametrizarse con los encabezados necesarios.

<div id="profilerequest-methods-changed">
  #### Métodos de ProfileRequest modificados
</div>

* El método `addParameters` ahora requiere que el valor sea un Map de String a String, en lugar de String a Object (`addParameters(mapOf("key" to "val"))`)

<div id="signuprequest-methods-changed">
  #### Métodos de SignUpRequest modificados
</div>

Los métodos que establecen parámetros ahora requieren que el valor sea un `Map` de `String` a `String`, en lugar de `String` a `Object`:

* `addAuthenticationParameters(mapOf("key" to "val"))`
* `addSignupParameters(mapOf("key" to "val"))`
* `addParameters(mapOf("key" to "val"))`

Además, se eliminó `setDevice("device")`. Usa `addParameter("device", "{VALUE}")` en su lugar.
