---

<Warning>
  La autenticación passwordless desde aplicaciones nativas está deshabilitada de forma predeterminada para los tenants nuevos a partir del 8 de junio de 2017. Se recomienda a los usuarios usar Universal Login y usar Web Authentication en su lugar. Si aun así quieres continuar, primero tendrás que habilitar el tipo de concesión de MFA en tu Dashboard. Consulta [Tipos de concesión de aplicaciones](/docs/es/get-started/applications/application-grant-types) para obtener más información.
</Warning>

<Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> puede hacerse por correo electrónico o SMS, ya sea enviando al usuario un código o un enlace que lo contiene. Todos los métodos de autenticación Passwordless requieren dos pasos: solicitar el código e introducirlo para su verificación.

<div id="set-up-the-android-sdk">
  ## Configura el SDK de Auth0 para Android
</div>

Primero, [configura el SDK de Auth0 para Android](/docs/es/libraries/auth0-android) para poder usar los métodos de Passwordless que se indican a continuación.

<div id="configure-auth0-and-the-android-sdk">
  ## Configura Auth0 y el SDK de Auth0 para Android
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar la API de Passwordless, debes usar la versión 1.20 o superior del SDK de Auth0 para Android.
</Callout>

Para usar la API de Passwordless desde un cliente nativo, habilita la concesión Passwordless OTP para tu aplicación en **Dashboard &gt; Applications &gt; (YOUR APPLICATION) &gt; Settings &gt; Advanced Settings &gt; Grant Types**.

<div id="request-the-code">
  ## Solicitar el código
</div>

Este ejemplo solicita el código llamando a `passwordlessWithEmail` con el correo electrónico del usuario, `PasswordlessType.CODE` y el nombre de la conexión como parámetros. Si la operación se realiza correctamente, quizá quieras avisar al usuario de que su código está en camino y, tal vez, dirigirlo a la pantalla donde podrá introducirlo.

```kotlin lines
authentication
    .passwordlessWithEmail("username@domain.com", PasswordlessType.CODE, "my-passwordless-connection")
    .start(object: Callback<Void?, AuthenticationException>() {
        override fun onSuccess(result: Void?) {
            // ¡Código enviado!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    })
```

Puedes usar el método `passwordlessWithSms` para enviar el código por SMS.

<div id="input-the-code">
  ## Ingresa el código
</div>

Una vez que el usuario tenga un código, podrá ingresarlo. Llama al método `loginWithEmail` y pásale el correo electrónico del usuario, el código que recibió y el nombre de la conexión correspondiente. Si la operación se realiza correctamente, recibirás un objeto `Credentials` en la respuesta.

```kotlin lines
authentication
    .loginWithEmail("username@domain.com", "123456", "my-passwordless-connection")
    .start(object: Callback<Credentials, AuthenticationException>() {
        override fun onSuccess(result: Credentials) {
            // ¡Sesión iniciada!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    })
```

Puedes usar el método `loginWithSms` para enviar el código recibido por SMS y autenticar al usuario.

El scope que se utiliza de forma predeterminada es `openid profile email`.
