title: Autenticación passwordless en Auth0.Android
---

<Warning>
  La autenticación passwordless desde aplicaciones nativas está deshabilitada de forma predeterminada para los tenants nuevos desde el 8 de junio de 2017. Se recomienda usar Universal Login y Web Authentication en su lugar. Si aun así desea continuar, primero deberá habilitar el tipo de concesión de MFA en su Dashboard. Consulte [Tipo de concesión de aplicaciones](/docs/es/get-started/applications/application-grant-types) para obtener más información.
</Warning>

<Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> se puede realizar por correo electrónico o SMS, ya sea enviando al usuario un código o un enlace que contenga un código. Todos los métodos de autenticación Passwordless requieren dos pasos: solicitar el código e introducirlo para su verificación.

<div id="set-up-the-android-sdk">
  ## Configura el Android SDK
</div>

Primero, [configura el Android SDK](/docs/es/libraries/auth0-android) para poder usar los métodos de Passwordless que se indican a continuación.

<div id="configure-auth0-and-the-android-sdk">
  ## Configura Auth0 y el Android SDK
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar la API Passwordless, debes configurar Auth0 Android SDK con la versión 1.20 o posterior.
</Callout>

Para usar la API Passwordless desde un cliente nativo, habilita la concesión Passwordless OTP para tu aplicación en **Dashboard &gt; Applications &gt; (YOUR APPLICATION) &gt; Settings &gt; Advanced Settings &gt; Grant Types**.

<div id="request-the-code">
  ## Solicitar el código
</div>

En este ejemplo, se solicita el código llamando a `passwordlessWithEmail` con el correo electrónico del usuario, `PasswordlessType.CODE` y el nombre de la conexión como parámetros. Si todo sale bien, quizá quieras avisar al usuario de que recibirá el código en breve y, tal vez, redirigirlo a la pantalla donde deberá introducirlo.

```kotlin lines
authentication
    .passwordlessWithEmail("username@domain.com", PasswordlessType.CODE, "my-passwordless-connection")
    .start(object: Callback<Void?, AuthenticationException>() {
        override fun onSuccess(result: Void?) {
            // ¡Código enviado!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    })
```

Puedes usar el método `passwordlessWithSms` para enviar el código por SMS.

<div id="input-the-code">
  ## Introduce el código
</div>

Una vez que el usuario haya recibido el código, puede introducirlo. Llama al método `loginWithEmail` y pásale el correo electrónico del usuario, el código que recibió y el nombre de la conexión correspondiente. Si la operación se completa correctamente, recibirás un objeto `Credentials` en la respuesta.

```kotlin lines
authentication
    .loginWithEmail("username@domain.com", "123456", "my-passwordless-connection")
    .start(object: Callback<Credentials, AuthenticationException>() {
        override fun onSuccess(result: Credentials) {
            // ¡Sesión iniciada!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    })
```

Puede usar el método `loginWithSms` para enviar el código recibido por SMS y autenticar al usuario.

El scope que se usa de forma predeterminada es `openid profile email`.
