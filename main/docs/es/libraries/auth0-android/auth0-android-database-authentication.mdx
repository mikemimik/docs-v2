<Warning>
  La autenticación con usuario/correo electrónico y contraseña desde aplicaciones nativas está deshabilitada de forma predeterminada para los tenants nuevos desde el 8 de junio de 2017. Se recomienda usar Universal Login y realizar Web Authentication en su lugar. Si aun así quieres continuar, primero tendrás que habilitar el tipo de concesión Password en tu Dashboard. Consulta [Tipo de concesión de aplicaciones](/docs/es/get-started/applications/application-grant-types) para obtener más información.
</Warning>

<div id="log-in-with-a-database-connection">
  ## Iniciar sesión con una conexión de base de datos
</div>

Para iniciar sesión con una conexión de base de datos, llama a `login` con el **correo electrónico**, la **contraseña** y la **conexión** con la que quieras autenticarte. La respuesta es un objeto `Credentials`.

```kotlin lines
authentication
    .login("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(payload: Credentials) {
            // ¡Sesión iniciada!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    })
```

El scope predeterminado es `openid profile email`.

<div id="sign-up-with-a-database-connection">
  ## Registrarse con una conexión de base de datos
</div>

Para registrarse con una conexión de base de datos, llama al método `signUp` y pasa el correo electrónico, la contraseña y el nombre de la conexión.

```kotlin lines
authentication
    .signUp("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(result: Credentials) {
            // ¡Registrado y con sesión iniciada!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    });
```
