<Warning>
  La autenticación con Username/Email y Password desde aplicaciones nativas está deshabilitada de forma predeterminada para los tenants nuevos desde el 8 de junio de 2017. Se recomienda a los usuarios usar Universal Login y utilizar Web Authentication en su lugar. Si aun así desea continuar, primero deberá habilitar el tipo de concesión Password en su Dashboard. Consulte [Tipo de concesión de aplicaciones](/docs/es/get-started/applications/application-grant-types) para obtener más información.
</Warning>

<div id="log-in-with-a-database-connection">
  ## Inicia sesión con una conexión de base de datos
</div>

Para iniciar sesión con una conexión de base de datos, llama a `login` con el **correo electrónico**, la **contraseña** y la **conexión** con la que quieres autenticarte. La respuesta es un objeto `Credentials`.

```kotlin lines
authentication
    .login("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(payload: Credentials) {
            // ¡Sesión iniciada!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    })
```

El scope predeterminado es `openid profile email`.

<div id="sign-up-with-a-database-connection">
  ## Registrarse con una conexión de base de datos
</div>

Para registrarse con una conexión de base de datos, llame al método `signUp` y pásеле el correo electrónico, la contraseña y el nombre de la conexión del usuario.

```kotlin lines
authentication
    .signUp("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(result: Credentials) {
            // ¡Registrado e iniciado sesión!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    });
```
