El SDK de Auth0-PHP se integra en tus aplicaciones PHP y ofrece un inicio de sesión y registro de usuarios sencillos. Es compatible con <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> sociales como Facebook, Google o X, así como con proveedores empresariales como Active Directory. El SDK ofrece métodos prácticos para acceder a los endpoints de Authentication and Management de Auth0.

El repositorio de Auth0-PHP es de código abierto y está [alojado en GitHub](https://github.com/auth0/auth0-PHP). Agradecemos todas las contribuciones, incluidos los informes de errores, las propuestas de mejora y las pull requests.

<div id="requirements">
  ## Requisitos
</div>

* PHP 7.4+ (se recomienda 8.0+)
* [Composer](https://getcomposer.org/doc/00-intro.md)

<div id="installation">
  ## Instalación
</div>

Para instalar el SDK de Auth0 para PHP, necesitas [Composer](https://getcomposer.org/doc/00-intro.md#installation-linux-unix-macos), la herramienta estándar de gestión de dependencias para PHP. Composer te permite declarar las bibliotecas de las que depende tu proyecto y las instala por ti. Asegúrate de que Composer esté instalado y sea accesible desde tu shell antes de continuar.

A continuación, ejecuta el siguiente comando de shell en el directorio de tu proyecto para instalar el SDK:

```php lines
composer require auth0/auth0-php
```

Esto creará una subcarpeta `vendor` dentro de tu proyecto y descargará todas las dependencias necesarias para usar el SDK de PHP. También creará un archivo `vendor/autoload.php`, necesario para que el SDK funcione con tu aplicación, que importaremos más adelante.

<div id="getting-started">
  ## Primeros pasos
</div>

Para usar las API de Authentication and Management de Auth0, necesitarás una cuenta gratuita de Auth0 y una aplicación:

1. Ve a [auth0.com/signup](https://auth0.com/signup) y crea tu cuenta.
2. Una vez en el Dashboard, ve a **Applications** y luego a **Create Application**.
3. Dale un nombre a tu aplicación, selecciona **Regular Web Application** y luego **Create**
4. Haz clic en la pestaña **Settings** para ver las credenciales necesarias que se usan a continuación. Para obtener más información, consulta [Application Settings](/docs/es/get-started/applications/application-settings).

<div id="configure-the-sdk">
  ## Configura el SDK
</div>

Debes usar [variables de entorno](https://secure.php.net/manual/en/reserved.variables.environment.php) para almacenar y cargar las credenciales sensibles de Auth0. Así evitas tener que incluirlas directamente en el código de tu aplicación. Vamos a crear un archivo `.env` en la raíz del directorio de nuestro proyecto para almacenar las credenciales de nuestra aplicación:

La forma más sencilla de usar variables de entorno en tu proyecto es con una biblioteca como [PHP Dotenv](https://github.com/josegonzalez/php-dotenv) junto con un archivo `.env` local. Crea un archivo `.env` (asegúrate de que no sea accesible públicamente y de que esté excluido del control de versiones) y agrega los siguientes valores:

export const codeExample = `# La URL del dominio de nuestro tenant de Auth0.
# Si usamos un Custom Domain, asegúrate de establecer aquí ese valor.
AUTH0_DOMAIN='https://{yourDomain}'

# El ID de cliente de nuestra aplicación de Auth0.
AUTH0_CLIENT_ID='{yourClientId}'

# El secreto de cliente de nuestra aplicación de Auth0.
AUTH0_CLIENT_SECRET='{yourClientSecret}'

# Un valor secreto largo que usaremos para cifrar las cookies de sesión. Puede generarse con \`openssl rand -hex 32\` desde nuestro shell.
AUTH0_COOKIE_SECRET='VER COMENTARIO ANTERIOR'

# La URL base de nuestra aplicación.
AUTH0_BASE_URL='http://127.0.0.1:3000'`;

<AuthCodeBlock children={codeExample} language="php" />

Nunca debes hacer commit de este archivo en el control de versiones ni compartirlo de forma insegura. Su contenido debe manejarse con cuidado y tratarse como una contraseña.
Como PHP no puede leer de forma nativa el archivo `.env`, tendrás que instalar una biblioteca de PHP para poder hacerlo. Para esta documentación usaremos `vlucas/phpdotenv`, pero cualquier biblioteca &quot;dotenv&quot; que prefieras funcionará. Desde el directorio de nuestro proyecto, ejecuta el siguiente comando en el shell para instalar la biblioteca:

```php lines
composer require vlucas/phpdotenv
```

<div id="initialize-the-sdk">
  ## Inicializa el SDK
</div>

Ya estamos listos para configurar e inicializar una instancia del SDK en nuestra nueva aplicación PHP. Empecemos por crear el archivo fuente de PHP con el que trabajaremos en esta demostración, `index.php`, y usemos el siguiente fragmento para empezar:

```php lines
<?php

// Importar el Autoloader de Composer para hacer accesibles las clases del SDK:
require 'vendor/autoload.php';

// Cargar las variables de entorno desde el archivo .env:
(Dotenv\Dotenv::createImmutable(__DIR__))->load();

// Ahora instanciar la clase Auth0 con nuestra configuración:
$auth0 = new \Auth0\SDK\Auth0([
    'domain' => $_ENV['AUTH0_DOMAIN'],
    'clientId' => $_ENV['AUTH0_CLIENT_ID'],
    'clientSecret' => $_ENV['AUTH0_CLIENT_SECRET'],
    'cookieSecret' => $_ENV['AUTH0_COOKIE_SECRET']
]);
```
