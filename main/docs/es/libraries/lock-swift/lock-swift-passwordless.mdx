Lock <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> admite la autenticación passwordless mediante conexiones de correo electrónico y SMS. Para usar la autenticación Passwordless, necesitas Lock.Swift versión 2.14.0 o superior.

Para mostrar Lock, añade el siguiente fragmento a tu `UIViewController`.

```swift lines
Lock
    .passwordless()
    .withOptions {
         $0.oidcConformant = true
    }
    // withConnections, withOptions, withStyle, etc.
    .onAuth { credentials in
      // Guardar el objeto Credentials
    }
    .present(from: self)
```

Passwordless solo puede usarse con una única conexión y dará prioridad a las conexiones de correo electrónico frente a las de SMS.

<div id="passwordless-method">
  ### Método passwordless
</div>

Al usar Passwordless de Lock, `passwordlessMethod` usa por defecto `.code`, que envía al usuario un código de un solo uso para iniciar sesión. Si quieres usar [Universal Links](/docs/es/get-started/applications/enable-universal-links-support-in-apple-xcode), puedes agregar lo siguiente:

```swift lines
.withOptions {
    $0.passwordlessMethod = .magicLink
}
```

<div id="activity-callback">
  ### Devolución de llamada de la Activity
</div>

Si estás usando Passwordless de Lock y has especificado la opción `.magicLink` para enviar al usuario un universal link, tendrás que añadir lo siguiente a tu `AppDelegate.swift`:

```swift lines
func application(_ application: UIApplication, continue userActivity: NSUserActivity, restorationHandler: @escaping ([Any]?) -> Void) -> Bool {
    return Lock.continueAuth(using: userActivity)
}
```
