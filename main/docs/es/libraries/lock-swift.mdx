Esta guía de referencia te mostrará cómo implementar la interfaz de usuario de Lock y te dará información detallada sobre cómo configurar y personalizar Lock para usarlo como interfaz para tus necesidades de autenticación. Sin embargo, si quieres aprender a hacer más con Auth0 y Swift, como guardar, usar y renovar <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Tokens">Access Tokens</Tooltip>, obtener información del perfil del usuario y mucho más, consulta el [SDK de Auth0.swift](/docs/es/libraries/auth0-swift). O bien, echa un vistazo al [Quickstart de Swift](/docs/es/quickstart/native/ios-swift) para ver ejemplos completos y las opciones disponibles, tanto para usar Lock como interfaz como para usar una interfaz personalizada.

Consulta el [repositorio de Lock.swift](https://github.com/auth0/Lock.swift) en GitHub.

<div id="cocoapods">

Si usas [Cocoapods](https://cocoapods.org), añade esta línea a tu `Podfile`:

Después, ejecuta `pod install`.

Para más información sobre el uso de Carthage, consulta [su documentación oficial](https://github.com/Carthage/Carthage#if-youre-building-for-ios-tvos-or-watchos).

<div id="integrate-with-your-application">
  ### Intégralo con tu aplicación
</div>

Hay que notificar a Lock cuando se le pide a la aplicación que abra una URL. Puedes hacerlo en el archivo `AppDelegate`.

```swift lines
func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey : Any] = [:]) -> Bool {
  return Lock.resumeAuth(url, options: options)
}
```

<div id="auth0-credentials">

</div>

<div id="implementation-of-lock-classic">
  ## Implementación de Lock Classic
</div>

Lock Classic gestiona la autenticación mediante conexiones Database, Social y Enterprise.

<div id="oidc-conformant-mode">
  ### Modo compatible con OIDC
</div>

Se recomienda encarecidamente usar este SDK en modo compatible con OIDC. Cuando este modo está habilitado, obliga al SDK a usar el pipeline de autenticación actual de Auth0 y evita que acceda a endpoints heredados. De forma predeterminada, es `false`

```swift lines
.withOptions {
    $0.oidcConformant = true
}
```

Para mostrar Lock, agrega el siguiente fragmento a tu UIViewController.

```swift lines
Lock
    .classic()
    // withConnections, withOptions, withStyle, y así sucesivamente
    .withOptions {
      $0.oidcConformant = true
      $0.scope = "openid profile"
    }
    .onAuth { credentials in
      // Guardemos el valor de credentials.accessToken
    }
    .present(from: self)
```

<div id="use-auth0swift-library-to-access-user-profile">
  ## Usa la biblioteca Auth0.Swift para acceder al perfil del usuario
</div>

Para acceder a la información del perfil del usuario, necesitarás usar la biblioteca `Auth0.Swift`:

```swift lines
Auth0
   .authentication()
   .userInfo(withAccessToken: accessToken)
   .start { result in
       switch result {
       case .success(let profile):
           print("User Profile: \(profile)")
       case .failure(let error):
           print("Failed with \(error)")
       }
   }
```

Consulta la [documentación de la biblioteca Auth0.Swift](/docs/es/libraries/auth0-swift) para obtener más información sobre cómo usarla.

<div id="specify-connections">
  ## Especificar conexiones
</div>

Lock cargará automáticamente las conexiones configuradas para su aplicación. Si desea modificar el comportamiento predeterminado, puede especificar manualmente qué conexiones debe mostrar a los usuarios como opciones de authentication. Para ello, llame al método y proporcione un closure con el que pueda especificar la(s) conexión(es).

Añadir una conexión de base de datos:

```swift lines
.withConnections {
    connections.database(name: "Username-Password-Authentication", requiresUsername: true)
}
```

Añadir varias conexiones sociales:

```swift lines
.withConnections {
    connections.social(name: "facebook", style: .Facebook)
    connections.social(name: "google-oauth2", style: .Google)
}
```

<div id="configuration-options">
  ## Opciones de configuración
</div>

Hay muchas opciones para configurar el comportamiento de Lock. A continuación se muestra un ejemplo de Lock configurado para que se pueda cerrar, para limitarlo únicamente a nombres de usuario (y no a direcciones de correo electrónico) y para mostrar solo las pantallas de Login y Restablecer contraseña.

```swift lines
Lock
  .classic()
  .withOptions {
    $0.closable = true
    $0.usernameStyle = [.Username]
    $0.allow = [.Login, .ResetPassword]
  }
```

Puedes consultar el conjunto completo de opciones de configuración del comportamiento para cambiar cómo funciona Lock en tu aplicación en la [Guía de configuración](/docs/es/libraries/lock-swift/lock-swift-configuration-options).

<div id="password-manager-support">
  ## Soporte para gestores de contraseñas
</div>

De forma predeterminada, la compatibilidad con [1Password](https://1password.com/) está habilitada para las conexiones de base de datos. Aun así, para que la opción aparezca en las pantallas de inicio de sesión y registro, el usuario deberá tener instalada la aplicación 1Password. Puede deshabilitar la compatibilidad con 1Password mediante la propiedad enabled de passwordManager.

```swift lines
.withOptions {
    $0.passwordManager.enabled = false
}
```

De forma predeterminada, `appIdentifier` se establecerá con el identificador del paquete de la app y `displayName`, con el nombre para mostrar de la app. Puedes personalizarlos de la siguiente manera:

```swift lines
.withOptions {
    $0.passwordManager.appIdentifier = "www.myapp.com"
    $0.passwordManager.displayName = "My App"
}
```

Deberás añadir lo siguiente al archivo `info.plist` de tu aplicación:

```xml lines
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>org-appextension-feature-password-management</string>
</array>
```
