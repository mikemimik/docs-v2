A continuación se muestra una lista de incidencias que pueden surgir al usar la biblioteca PHP de Auth0 y cómo solucionarlas.

**Recibo una excepción &quot;Invalid State&quot; al intentar iniciar sesión.**

La [validación de state](/docs/es/secure/attack-protection/state-parameters) se añadió en la versión 5.1.0 para mejorar la seguridad. De forma predeterminada, usa almacenamiento de sesión y se ejecuta automáticamente si está utilizando una combinación de `Auth0::login()` y cualquier método que llame a `Auth0::exchange()` en su devolución de llamada.

Si sus usuarios encuentran este error:- Asegúrese de que su aplicación no esté invocando accidentalmente `Auth0::login()` más de una vez, ya que esto podría invalidar el state almacenado en el dispositivo del usuario final.- Asegúrese de que el usuario final esté usando un navegador moderno en su dispositivo y no esté bloqueando cookies.

**Recibo** **`curl error 60: SSL certificate problem: self-signed certificate in certificate chain`** **en Windows**

Este es un problema común en las versiones más recientes de PHP en **Windows OS** (está relacionado con una incompatibilidad entre Windows y la base de datos de CA de OpenSSL).

1. Descargue esta base de datos de CA `https://curl.haxx.se/ca/cacert.pem` en `c:/cacert.pem`.
2. Edite su php.ini y agregue `openssl.cafile=c:/cacert.pem`. (Debe apuntar al archivo que descargó).

**Mi host no permite usar Composer**

El SDK de PHP requiere Composer para gestionar las dependencias (bibliotecas externas de PHP). Si Composer no puede instalarse globalmente en su host, aún puede instalarlo localmente para ejecutarlo desde su cuenta de shell de usuario. Puede encontrar instrucciones al respecto en el sitio web de Composer: &lt;https://getcomposer.org/doc/00-intro.md#locally&gt;
