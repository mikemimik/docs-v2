A continuación, se muestra una lista de problemas que podrías encontrar al usar la biblioteca PHP de Auth0 y cómo solucionarlos.

**Recibo una excepción &quot;Invalid State&quot; al intentar iniciar sesión.**

La [validación de state](/docs/es/secure/attack-protection/state-parameters) se agregó en la versión 5.1.0 para mejorar la seguridad. De forma predeterminada, usa almacenamiento de sesión y se ejecutará automáticamente si usas una combinación de `Auth0::login()` y cualquier método que llame a `Auth0::exchange()` en tu devolución de llamada.

Si tus usuarios se encuentran con este error:

* Asegúrate de que tu aplicación no esté invocando accidentalmente `Auth0::login()` más de una vez, ya que esto podría invalidar el state almacenado en el dispositivo del usuario final.
* Verifica que el usuario final esté usando un navegador moderno en su dispositivo y que no esté bloqueando cookies.

**Recibo** **`curl error 60: SSL certificate problem: self-signed certificate in certificate chain`** **en Windows**

Este es un problema común en las versiones más recientes de PHP en **Windows OS** (está relacionado con una incompatibilidad entre Windows y la base de datos de CA de OpenSSL).

1. Descarga esta base de datos de CA `https://curl.haxx.se/ca/cacert.pem` en `c:/cacert.pem`.
2. Edita tu php.ini y agrega `openssl.cafile=c:/cacert.pem`. (Debe apuntar al archivo que descargaste).

**Mi host no permite usar Composer**

El SDK de PHP requiere Composer para administrar dependencias (bibliotecas externas de PHP). Si no se permite instalar Composer de forma global en tu host, igualmente puedes instalarlo localmente para ejecutarlo desde tu cuenta de usuario en el shell. Puedes encontrar instrucciones en el sitio web de Composer: &lt;https://getcomposer.org/doc/00-intro.md#locally&gt;
