El SDK de Auth0-PHP agrupa tres clases principales: `Auth0\SDK\Auth0`, `Auth0\SDK\API\Authentication` y `Auth0\SDK\API\Management`, cada una de las cuales ofrece interfaces para distintas funcionalidades en las API de Auth0. Si estás creando una aplicación web con estado que necesita realizar un seguimiento de las sesiones de los usuarios, la clase base `Auth0` será con la que más trabajarás. Proporciona métodos para gestionar tareas habituales de autenticación y manejo de sesiones, como iniciar y cerrar sesión, recuperar credenciales de usuario, comprobar si hay una sesión disponible y gestionar la devolución de llamada. Estas tareas se explican a continuación.

Para obtener más información sobre estas capacidades y otras más, consulta la [página de documentación de la Authentication API](/docs/es/libraries/auth0-php/using-the-authentication-api-with-auth0-php).

<div id="logging-in">
  ### Inicio de sesión
</div>

El proceso de inicio de sesión predeterminado en el SDK de PHP usa una [concesión de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/call-your-api-using-the-authorization-code-flow) combinada con la página de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>. En resumen, este proceso es así:

1. Un usuario que solicita acceso es redirigido a la página de Universal Login.
2. El usuario se autentica mediante una de las [muchas conexiones posibles](/docs/es/authenticate/identity-providers): social (Google, X, Facebook), base de datos (correo electrónico y contraseña), passwordless (correo electrónico, SMS) o empresarial (ActiveDirectory, ADFS, Office 365).
3. Si todo sale bien, el usuario es redirigido o devuelto mediante POST a la URL de devolución de llamada de tu aplicación con los valores `code` y `state`; si no, con un `error` y `error_description`.
4. Si la autenticación fue correcta, se valida el valor de `state`.
5. Si `state` es válido, el valor de `code` se intercambia con Auth0 por un token de ID y/o un token de acceso.
6. La identidad incluida en el token de ID puede usarse para crear una cuenta, iniciar una sesión específica de la aplicación o mantenerse como la sesión del usuario.

Auth0-PHP gestiona automáticamente la mayoría de estos pasos. Tu aplicación tendrá que:

1. Llamar a `Auth0\SDK\Auth0::login()` cuando los usuarios necesiten iniciar sesión (por ejemplo: al hacer clic en un enlace, al visitar contenido restringido, etc.)
2. Llamar a `Auth0\SDK\Auth0::exchange()` cuando los usuarios sean redirigidos a tu URL de devolución de llamada.
3. Llamar a `Auth0\SDK\Auth0::getCredentials()` cuando necesites comprobar si un usuario ha iniciado sesión y recuperar su información.

Una implementación sencilla de estos pasos se ve así:

```php lines
// 👆 Continuamos desde la guía de "introducción" enlazada en "Requisitos previos" arriba. Añade esto al archivo index.php que creaste allí.

// getExchangeParameters() se puede usar en tu URL de devolución de llamada para verificar que todos los parámetros necesarios estén presentes para el intercambio de código posterior a la autenticación.
if ($auth0->getExchangeParameters()) {
    // Si están presentes, debemos realizar el intercambio de código.
    $auth0->exchange();
}

// Verificar si el usuario ya inició sesión
$session = $auth0->getCredentials();

if ($session === null) {
    // ¡El usuario no ha iniciado sesión!
    // Redirigir a la página de Universal Login para la autenticación.
    header("Location: " . $auth0->login());
    exit;
}

// 🎉 En este punto tenemos una sesión de usuario autenticada accesible desde $session; ¡la lógica de tu aplicación puede continuar desde aquí!
echo "Authenticated!";
```

Por último, tendrás que añadir la URL de tu aplicación al campo &quot;Allowed Callback URLs&quot; de tu aplicación de Auth0 en la página de configuración. Después de eso, al cargar tu página con script debería ocurrir lo siguiente:

1. Redirigirte inmediatamente a una página de login de Auth0 para tu tenant.
2. Después de iniciar sesión correctamente con cualquier conexión, redirigirte de vuelta a tu aplicación.
3. Mostrar una página sencilla con el texto &#39;¡Autenticado!&#39;.

<div id="profile">
  ### Perfil
</div>

Ahora que hemos autenticado a un usuario, podemos trabajar con los datos persistentes de su sesión para hacer cosas como mostrar perfiles de usuario.

```php lines
// 👆 Continuamos desde el código anterior. Añade esto al archivo index.php.

printf(
    '<h1>Hi %s!</h1>
    <p><Frame><img width="100" src="/%s"></Frame></p>
    <p><strong>Last update:</strong> %s</p>
    <p><strong>Contact:</strong> %s %s</p>
    <p><a href="/docs/logout.php">Logout</a></p>',
    isset($session->user['nickname']) ? strip_tags($session->user['nickname']) : '[unknown]',
    isset($session->user['picture']) ? filter_var($session->user['picture'], FILTER_SANITIZE_URL) : 'https://gravatar.com/avatar/',
    isset($session->user['updated_at']) ? date('j/m/Y', strtotime($session->user['updated_at'])) : '[unknown]',
    isset($session->user['email']) ? filter_var($session->user['email'], FILTER_SANITIZE_EMAIL) : '[unknown]',
    ! empty($session->user['email_verified']) ? '✓' : '✗'
);
```

<div id="logout">
  ### Cierre de sesión
</div>

Además de iniciar sesión, también queremos que los usuarios puedan cerrar sesión. Cuando los usuarios cierran sesión, deben invalidar su sesión en la aplicación. Para este SDK, eso significa eliminar los datos persistentes del usuario y del token:

```php lines
// Cerrar sesión de la aplicación.
header("Location: $auth0->logout());
```

Si usas <Tooltip tip="inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SSO) y también quieres finalizar su sesión de Auth0, consulta la [sección sobre cierre de sesión de SSO aquí](/docs/es/libraries/auth0-php/using-the-authentication-api-with-auth0-php). Puedes encontrar más información sobre el cierre de sesión, en general, [aquí](/docs/es/authenticate/login/logout).
