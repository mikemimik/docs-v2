El SDK de Auth0 para PHP proporciona una clase `Auth0\SDK\Token` que se utiliza para procesar <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, del access token) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Tokens">JSON Web Tokens</Tooltip> (JWT). Te permite decodificar, validar y verificar tokens para que tu aplicación los use. Puedes encontrar más información sobre los JWT y sobre cómo crearlos y decodificarlos en [jwt.io](https://jwt.io/).

La clase puede procesar tokens HS256 y RS256. Ambos tipos requieren configurar en el SDK el algoritmo y las audiencias válidas antes de procesarlos. Los tokens HS256 requieren que se configure el <Tooltip tip="Client Secret: Secret que usa un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>. Los tokens RS256 requieren un issuer autorizado, que se utiliza para obtener un archivo JWKs durante el proceso de decodificación. ([Más información sobre algoritmos de firma aquí](https://auth0.com/blog/navigating-rs256-and-jwks/).)

<div id="example-usage">
  ## Ejemplo de uso
</div>

A continuación se muestra un ejemplo de un pequeño procesador de JSON Web Token basado en URL, basado en la clase `Token` del SDK.

```php lines expandable
<?php

// Importar el Autoloader de Composer para que las clases del SDK sean accesibles:
require 'vendor/autoload.php';

// Cargar las variables de entorno desde el archivo .env:
(Dotenv\Dotenv::createImmutable(__DIR__))->load();

$token = filter_var($_GET['token'] ?? null, FILTER_UNSAFE_RAW, FILTER_NULL_ON_FAILURE);
$algorithm = filter_var($_GET['algorithm'] ?? 'HS256', FILTER_UNSAFE_RAW, FILTER_NULL_ON_FAILURE);

if ($token === null) {
    die('No `token` request parameter.');
}

if (! in_array($algorithm, ['HS256', 'RS256'])) {
    die('Invalid `algorithm` supplied.');
}

// El SDK de Auth0 incluye una utilidad de procesamiento de tokens que usaremos para esto:
$token = new \Auth0\SDK\Token([
    'domain' => $env['AUTH0_DOMAIN'],
    'clientId' => $env['AUTH0_CLIENT_ID'],
    'clientSecret' => $env['AUTH0_CLIENT_SECRET'],
    'tokenAlgorithm' => $algorithm
], $token, \Auth0\SDK\Token::TYPE_ID_TOKEN);

// Verificar el token: (Esto lanzará una \Auth0\SDK\Exception\InvalidTokenException si la verificación falla.)
$token->verify();

// Validar los claims del token: (Esto lanzará una \Auth0\SDK\Exception\InvalidTokenException si la validación falla.)
$token->validate();

echo '<pre>';
print_r($token->toArray(), true);
echo '</pre>';
```

Tanto `verify()` como `validate()` ofrecen varios argumentos de opciones que pueden usarse para personalizar su comportamiento, como validar claims de <Tooltip tip="Nonce: Número arbitrario emitido una sola vez en un protocolo de autenticación para detectar y prevenir ataques de repetición." cta="Ver glosario" href="/docs/es/glossary?term=nonce">nonce</Tooltip>, restringir el tiempo máximo transcurrido desde el `auth_time` de un token, la tolerancia de reloj `leeway` para las comprobaciones de tiempo, entre otros. Estos métodos están completamente comentados, por lo que puedes revisar estas opciones tanto en el código fuente como en el IDE que prefieras.
