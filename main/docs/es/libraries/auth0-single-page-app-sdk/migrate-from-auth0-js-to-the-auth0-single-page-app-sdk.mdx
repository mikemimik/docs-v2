En este artículo, verás cómo migrar tu aplicación web de página única (SPA) de [auth0.js](/docs/es/libraries/auth0js) a [auth0-spa-js](/docs/es/libraries/auth0-single-page-app-sdk). A continuación, se muestran varios escenarios con auth0.js y su código equivalente en auth0-spa-js.

<div id="functionality-that-cannot-be-migrated">
  ## Funcionalidad que no se puede migrar
</div>

No toda la funcionalidad de auth0.js se puede migrar directamente a auth0-spa-js. Entre los escenarios que no se pueden migrar directamente se incluyen:

* [login](https://auth0.github.io/auth0.js/WebAuth.html#login) integrado con nombre de usuario y contraseña, así como [login passwordless](https://auth0.github.io/auth0.js/WebAuth.html#passwordlessLogin) integrado
* [registro de usuarios](https://auth0.github.io/auth0.js/WebAuth.html#signup)
* [obtener un perfil de usuario desde el endpoint /userinfo](https://auth0.github.io/auth0.js/Authentication.html#userInfo)
* [solicitar un correo electrónico para cambiar la contraseña del usuario](https://auth0.github.io/auth0.js/WebAuth.html#changePassword)
* [vincular usuarios con la Management API](https://auth0.github.io/auth0.js/Management.html#linkUser)
* [obtener un usuario con la Management API](https://auth0.github.io/auth0.js/Management.html#getUser)
* [actualizar los atributos del usuario con la Management API](https://auth0.github.io/auth0.js/Management.html#patchUserAttributes)
* [actualizar los metadatos del usuario con la Management API](https://auth0.github.io/auth0.js/Management.html#patchUserMetadata)
* También hay algunas opciones configurables en Auth0.js que no tienen equivalente en auth0-spa-js. Un ejemplo de ello es `responseType`. Hay una razón por la que no existe un mapeo directo 1:1 para cada opción. En este caso, `responseType` es innecesario, porque el SDK está pensado únicamente para su uso en SPA y, por lo tanto, no hace falta cambiar el tipo de respuesta.

<div id="authentication-parameters-are-not-modified-anymore">
  ## Los parámetros de autenticación ya no se modifican
</div>

Auth0.js convierte internamente los parámetros personalizados que configuras de `camelCase` a `snake_case`. Por ejemplo, si configuras `deviceType: 'offline'`, auth0.js en realidad envía `device_type: 'offline'` al servidor.

Al usar auth0-spa-js, los parámetros personalizados **no se convierten** de `camelCase` a `snake_case`. Enviará al <Tooltip tip="Servidor de autorización: servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip> cualquier parámetro que envíes.

<div id="get-the-user-information">
  ## Obtener la información del usuario
</div>

<div id="auth0js">
  ### auth0.js
</div>

La función `userInfo()` llama al [endpoint `/userinfo`](https://auth0.com/docs/api/authentication#user-profile) y devuelve el perfil del usuario.

* [userInfo()](https://auth0.github.io/auth0.js/Authentication.html#userInfo)

```jsx lines
window.addEventListener('load', () => {
  auth0.client.userInfo(accessToken, function(err, user) {
    console.log(user)
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

A diferencia de auth0.js, Auth0 SPA SDK no llama al [`endpoint` `/userinfo`](https://auth0.com/docs/api/authentication#user-profile) para obtener el perfil de usuario. En su lugar, `Auth0Client.getUser()` devuelve la información del usuario a partir del `id_token` decodificado.

* [Auth0Client.getUser()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getUser)

```jsx lines
window.addEventListener('load', async () => {
  const user = await auth0.getUser();
  console.log(user);
});
```

<div id="refresh-tokens">
  ## Tokens de actualización
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({}, function(err, authResult) {
    // Tokens de autenticación o error
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Auth0 SPA SDK gestiona por ti la renovación del <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip>. Cada vez que llames a `getTokenSilently`, obtendrás un token de acceso válido o un error si no hay una sesión en Auth0.

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently();
});
```

<div id="get-a-token-for-a-different-audience-or-with-more-scopes">
  ## Obtén un token para otra audience o con scopes adicionales
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  }, function(err, authResult) {
    // Tokens de autenticación o error
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Auth0 SPA SDK se encarga de renovar el token de acceso por ti. Cada vez que llames a `getTokenSilently`, obtendrás un token de acceso válido o un error si no hay ninguna sesión en Auth0.

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)
* [GetTokenSilentlyOptions](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```

Usa [getTokenWithPopup](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenWithPopup) para abrir un popup y permitir que el usuario dé su consentimiento para la nueva API:

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenWithPopup({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```
