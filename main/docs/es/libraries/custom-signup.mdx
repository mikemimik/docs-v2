title: Registro personalizado con Lock o la API de Auth0.
---

Puedes personalizar el formulario de registro de usuarios con campos adicionales, además del correo electrónico y la contraseña, al usar Lock o la API de Auth0.

Hay muchos factores que debes considerar antes de elegir [Lock vs. Custom UI](/docs/es/customize/login-pages/classic-login/customize-with-lock-sdk). Por ejemplo, con Lock puedes redirigir a otra página para recopilar datos o usar perfilado progresivo. Con la API de Auth0, puedes capturar campos personalizados y almacenarlos en una base de datos. Hay ciertas limitaciones de personalización que debes tener en cuenta al elegir el método que mejor se adapte a tu caso. Algunas personalizaciones típicas incluyen agregar un nombre de usuario y verificar la fortaleza de la contraseña.

<Card title="Se recomienda Universal Login">
  Auth0 ofrece una opción de [Universal Login](/docs/es/authenticate/login/auth0-universal-login) que puedes usar en lugar de diseñar tus propias páginas personalizadas de inicio de sesión o registro, o de usar las que vienen integradas en cualquiera de las bibliotecas de Auth0. Si quieres ofrecer opciones de registro e inicio de sesión y solo necesitas personalizar el nombre de la aplicación, el logotipo y el color de fondo, Universal Login podría ser una opción más fácil de implementar.
</Card>

<div id="using-lock">
  ## Uso de Lock
</div>

Lock admite el [registro con campos personalizados](/docs/es/libraries/lock/lock-configuration).

La opción `additionalSignUpFields` de Lock solo funciona con registros mediante Database. En el caso de los registros que usan <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> sociales, no es posible recopilar estos campos de la misma manera con Lock, pero hay otras dos opciones para permitir registros con IdP sociales en Lock sin dejar de recopilar campos personalizados adicionales.

<div id="redirect-to-another-page">
  ### Redirigir a otra página
</div>

Una forma de usar el registro con proveedores sociales en Lock y recopilar campos personalizados consiste en usar [redirect rules](/docs/es/customize/rules/redirect-users) para redirigir a los usuarios a otra página, donde se les solicite información adicional, y luego devolverlos para completar la transacción de autenticación.

<div id="progressive-profiling">
  ### Elaboración progresiva de perfiles
</div>

Otra forma de recopilar datos de campos personalizados al registrar usuarios con proveedores sociales es mediante la [elaboración progresiva de perfiles](/docs/es/manage-users/user-accounts/user-profiles/progressive-profiling), que te permite ir completando poco a poco los datos del perfil del usuario con el tiempo. Al registrarse, recopilas solo la información mínima imprescindible, pero cuando el usuario vuelve a interactuar con tu aplicación, recopilas una pequeña cantidad de datos (quizá una sola pregunta) cada vez hasta completar su perfil. Esto te permite obtener la información que necesitas con menos fricción, ya que el objetivo de usar un IdP social para el registro es hacer que el proceso resulte más sencillo y fluido para el usuario.

<div id="using-the-api">
  ## Uso de la API
</div>

<div id="create-a-signup-form-to-capture-custom-fields">
  ### Crear un formulario de registro para recopilar campos personalizados
</div>

`name` es un atributo del perfil de usuario y `color` es un campo personalizado.

Actualmente no hay forma de validar los campos personalizados proporcionados por el usuario durante el registro. La validación debe realizarse desde una [Rule](/docs/es/customize/rules) al iniciar sesión, o con lógica personalizada **del lado del servidor** en su aplicación.

<div id="send-the-form-data">
  ### Enviar los datos del formulario
</div>

Envía una solicitud POST al endpoint [/dbconnections/signup](https://auth0.com/docs/api/authentication/reference#signup) de Auth0.

Deberás enviar:

* El `client_id` de tu aplicación
* El `email` y la `password` del usuario que se va a registrar
* El nombre de la `conexión` de base de datos donde se almacenarán los datos del usuario
* Cualquier atributo del perfil de usuario que quieras actualizar, incluidos `given_name`, `family_name`, `name`, `nickname` y `picture`.
* Cualquier campo personalizado como parte de `user_metadata`

<div id="custom-fields-limitations">
  ## Limitaciones de los campos personalizados
</div>

Cuando tus usuarios completan el registro, los campos personalizados se envían como parte de `user_metadata`. Las limitaciones de este campo son:

* `user_metadata` no debe contener más de 10 campos
* `user_metadata.field` debe ser una cadena
* `user_metadata.field.value.length` debe tener menos de 500 caracteres
* `user_metadata.field.length` debe tener menos de 100 caracteres
* El límite de tamaño actual de `user_metadata` es de **16 MB**

<div id="redirect-mode">
  ## Modo de redirección
</div>

Después de iniciar sesión correctamente, Auth0 redirigirá al usuario a la URL de devolución de llamada configurada con un <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, del token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> (`id_token`) en la cadena de consulta.

Luego, el servidor deberá llamar a APIv2 para agregar los campos personalizados necesarios al perfil del usuario.

<div id="add-username-to-the-signup-form">
  ## Agrega username al formulario de registro
</div>

Una personalización habitual del registro es agregar un username.

Para habilitar esta funcionalidad, activa la opción **Requires Username** en la sección [Connections &gt; Database](https://manage.auth0.com/#/connections/database/) del Dashboard, en la pestaña **Settings** de la conexión que quieres editar.

Captura el campo `username` en tu formulario personalizado y agrega `username` al cuerpo de la solicitud.

<div id="optional-verify-password-strength">
  ## Opcional: Verificar la fortaleza de la contraseña
</div>

Las políticas de contraseñas para las conexiones de base de datos se pueden configurar en el Dashboard. Para obtener más información, consulta: [Fortaleza de la contraseña en Auth0 Database Connections](/docs/es/authenticate/database-connections/password-strength).

Si es necesario para implementar formularios de registro personalizados, las políticas de contraseñas configuradas, junto con otra información de la conexión, se pueden recuperar desde la [Management v2 API](https://auth0.com/docs/api/management/v2#!/Connections/get_connections_by_id). El resultado se puede analizar en el cliente y contendrá información sobre la política de contraseñas actual (o las políticas) configurada en el Dashboard para esa conexión.
