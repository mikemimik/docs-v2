title: Registro personalizado con Lock o la Auth0 API.
---

Puedes personalizar el formulario de registro de usuario con más campos, además del correo electrónico y la contraseña, al usar Lock o la Auth0 API.

Hay muchos factores que debes considerar antes de elegir [Lock vs. interfaz de usuario personalizada](/docs/es/customize/login-pages/classic-login/customize-with-lock-sdk). Por ejemplo, con Lock puedes redirigir a otra página para recopilar datos o usar perfilado progresivo. Al usar la Auth0 API, puedes capturar campos personalizados y almacenarlos en una base de datos. Hay ciertas limitaciones de personalización que debes tener en cuenta al elegir el método que mejor se adapte a tus necesidades. Algunas personalizaciones habituales incluyen agregar un username y comprobar la fortaleza de la contraseña.

<Card title="Se recomienda Universal Login">
  Auth0 ofrece la opción de [Universal Login](/docs/es/authenticate/login/auth0-universal-login), que puedes usar en lugar de diseñar tus propias páginas personalizadas de inicio de sesión o registro, o de usar las que están integradas en cualquiera de las bibliotecas de Auth0. Si quieres ofrecer opciones de registro e inicio de sesión y solo necesitas personalizar el nombre de la aplicación, el logotipo y el color de fondo, Universal Login podría ser una opción más fácil de implementar.
</Card>

<div id="using-lock">
  ## Uso de Lock
</div>

Lock permite el [registro con campos personalizados](/docs/es/libraries/lock/lock-configuration).

La opción `additionalSignUpFields` de Lock solo funciona con registros en Database. En el caso de los registros que usan <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> sociales, no es posible recopilar estos campos de la misma manera con Lock, pero existen otras dos opciones para permitir registros con IdP sociales en Lock y seguir recopilando campos personalizados adicionales.

<div id="redirect-to-another-page">
  ### Redirigir a otra página
</div>

Una forma de usar el registro con proveedores sociales en Lock y recopilar campos personalizados es usar [Rules de redirección](/docs/es/customize/rules/redirect-users) para enviar al usuario a otra página, pedirle información adicional y luego redirigirlo de vuelta para completar la transacción de autenticación.

<div id="progressive-profiling">
  ### Perfilado progresivo
</div>

Otra forma de recopilar datos de campos personalizados cuando los usuarios se registran con proveedores sociales es mediante el [perfilado progresivo](/docs/es/manage-users/user-accounts/user-profiles/progressive-profiling), que te permite ir completando gradualmente los datos del perfil de usuario. Recopilas la información mínima indispensable durante el registro, pero cuando el usuario vuelve a interactuar con tu aplicación más adelante, recopilas una pequeña cantidad de datos (quizá una sola pregunta) cada vez hasta completar su perfil. Esto te permite recopilar la información deseada con menos fricción, ya que el objetivo de usar un IdP social para el registro es hacer que la experiencia sea más sencilla y fluida para el usuario.

<div id="using-the-api">
  ## Uso de la API
</div>

<div id="create-a-signup-form-to-capture-custom-fields">
  ### Crear un formulario de registro para recopilar campos personalizados
</div>

`name` es un atributo del perfil de usuario y `color` es un campo personalizado.

Actualmente no existe ninguna forma de validar los campos personalizados ingresados por el usuario durante el registro. La validación debe hacerse desde una [Rule](/docs/es/customize/rules) de Auth0 al iniciar sesión, o con lógica personalizada **en el servidor** en su aplicación.

<div id="send-the-form-data">
  ### Envía los datos del formulario
</div>

Envía una solicitud POST al endpoint [/dbconnections/signup](https://auth0.com/docs/api/authentication/reference#signup) de Auth0.

Deberás enviar:

* El `client_id` de tu aplicación
* El `email` y la `password` del usuario que se va a registrar
* El nombre de la `conexión` de base de datos donde se almacenarán los datos del usuario
* Cualquier atributo del perfil del usuario que quieras actualizar, incluidos `given_name`, `family_name`, `name`, `nickname` y `picture`.
* Cualquier campo personalizado como parte de `user_metadata`

<div id="custom-fields-limitations">
  ## Limitaciones de los campos personalizados
</div>

Cuando tus usuarios se registran, los campos personalizados se envían como parte de `user_metadata`. Las limitaciones de este campo son las siguientes:

* `user_metadata` no debe contener más de 10 campos
* `user_metadata.field` debe ser una cadena de texto
* `user_metadata.field.value.length` debe tener menos de 500 caracteres
* `user_metadata.field.length` debe tener menos de 100 caracteres
* El límite de tamaño actual de `user_metadata` es de **16 MB**

<div id="redirect-mode">
  ## Modo de redirección
</div>

Después de un inicio de sesión correcto, Auth0 redirigirá al usuario a la URL de devolución de llamada configurada con un <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, del token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> (`id_token`) en la cadena de consulta.

Luego, el servidor deberá llamar a la API v2 para agregar los campos personalizados necesarios al perfil del usuario.

<div id="add-username-to-the-signup-form">
  ## Agrega un username al formulario de registro
</div>

Una personalización habitual del registro es agregar un username.

Para habilitar esta función, activa la opción **Requires Username** en la sección [Connections &gt; Database](https://manage.auth0.com/#/connections/database/) del Dashboard, en la pestaña **Settings** de la conexión que quieres editar.

Captura el campo `username` en tu formulario personalizado y agrega `username` al cuerpo de la solicitud.

<div id="optional-verify-password-strength">
  ## Opcional: Verificar la fortaleza de la contraseña
</div>

Las políticas de contraseña de las conexiones de base de datos se pueden configurar en el Dashboard. Para obtener más información, consulta: [Fortaleza de la contraseña en Auth0 Database Connections](/docs/es/authenticate/database-connections/password-strength).

Si es necesario para la implementación de formularios de registro personalizados, las políticas de contraseña configuradas, junto con otra información de la conexión, se pueden recuperar desde la [Management v2 API](https://auth0.com/docs/api/management/v2#!/Connections/get_connections_by_id). El resultado puede analizarse en el cliente y contendrá información sobre la política de contraseña actual (o las políticas) configurada en el Dashboard para esa conexión.
