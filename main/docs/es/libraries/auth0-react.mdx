El SDK de Auth0 para React (auth0-react.js) es una biblioteca de JavaScript para implementar autenticación y autorización en aplicaciones React con Auth0. Proporciona un hook personalizado de React y otros componentes de orden superior para que puedas proteger aplicaciones React siguiendo las prácticas recomendadas y escribiendo menos código.

El SDK de Auth0 para React se encarga de los detalles de la concesión y del protocolo, la expiración y renovación de tokens, así como del almacenamiento de tokens y el almacenamiento en caché. Internamente, implementa [Universal Login](/docs/es/authenticate/login/auth0-universal-login) y [Authorization Code Grant Flow with PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

La biblioteca está [alojada en GitHub](https://github.com/auth0/auth0-react), donde puedes [obtener más información sobre la API](https://auth0.github.io/auth0-react/).

<div id="installation">
  ## Instalación
</div>

Tienes varias opciones para usar auth0-react.js en tu proyecto.

* Desde [npm](https://npmjs.org/):

  `npm install @auth0/auth0-react`
* Desde [yarn](https://yarnpkg.com/):
  `yarn add @auth0/auth0-react`

<div id="getting-started">
  ## Primeros pasos
</div>

Primero, tendrás que envolver tu aplicación en un único componente `Auth0Provider`. Así, se proporcionará el contexto de React a los componentes que estén dentro de tu aplicación.

export const codeExample = `import React from 'react';
    import ReactDOM from 'react-dom';
    import { Auth0Provider } from '@auth0/auth0-react';
    import App from './App';
    ReactDOM.render(
      <Auth0Provider
        domain="{yourDomain}"
        clientId="{yourClientId}"
        authorizationParams={{
          redirect_uri: window.location.origin
        }}
    >
      <App />
    </Auth0Provider>,
    document.getElementById('app')
 );`;

<div id="isloading-and-error">
  ## isLoading y error
</div>

Espera a que se inicialice el SDK y gestiona cualquier error con los estados `isLoading` y `error`.

```js lines
import React from 'react';
import { useAuth0 } from '@auth0/auth0-react';
function Wrapper({ children }) {
  const {
    isLoading,
    error,
  } = useAuth0();
  if (isLoading) {
    return <div>Loading...</div>;
  }
  if (error) {
    return <div>Oops... {error.message}</div>;
  }
  return <>{children}</>;
}
export default Wrapper;
```

<div id="login">
  ## Inicio de sesión
</div>

Usa `loginWithRedirect` o `loginWithPopup` para que tus usuarios inicien sesión.

```js lines
import React from 'react';
import { useAuth0 } from '@auth0/auth0-react';

function LoginButton() {
  const {
    isAuthenticated,
    loginWithRedirect,
  } = useAuth0();

  return !isAuthenticated && (
    <button onClick={loginWithRedirect}>Log in</button>
  );
}

export default LoginButton;
```

<div id="logout">
  ## Cierre de sesión
</div>

Usa `logout` para cerrar la sesión de tus usuarios. Asegúrate de que `returnTo` esté especificado en &quot;Allowed Logout URLs&quot; del <Tooltip tip="Auth0 Dashboard: la interfaz principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

```js lines
import React from 'react';
import { useAuth0 } from '@auth0/auth0-react';

function LogoutButton() {
  const {
    isAuthenticated,
    logout,
  } = useAuth0();

  return isAuthenticated && (
    <button onClick={() => {
      logout({ 
        logoutParams: {
          returnTo: window.location.origin
        }
      });
    }}>Log out</button>
  );
}

export default LogoutButton;
```

<div id="user">
  ## Usuario
</div>

Accede a la información del perfil de usuario con el valor `user`.

```js lines
import React from 'react';
import { useAuth0 } from '@auth0/auth0-react';

function Profile() {
  const { user } = useAuth0();

  return <div>Hello {user.name}</div>;
}

export default Profile;
```

<div id="use-with-a-class-component">
  ## Uso con un componente de clase
</div>

Usa el componente de orden superior `withAuth0` para añadir la propiedad `auth0` a los componentes de clase en lugar de usar el hook.

```jsx lines
import React, { Component } from 'react';
import { withAuth0 } from '@auth0/auth0-react';

class Profile extends Component {
  render() {
    const { user } = this.props.auth0;
    return <div>Hello {user.name}</div>;
  }
}

export default withAuth0(Profile);
```

<div id="protect-a-route">
  ## Protege una ruta
</div>

Protege un componente de ruta con el componente de orden superior `withAuthenticationRequired`. Si se accede a esta ruta sin autenticarse, el usuario será redirigido a la página de inicio de sesión y volverá a esta página después de iniciar sesión.

```js lines
import React from 'react';
import { withAuthenticationRequired } from '@auth0/auth0-react';

const PrivateRoute = () => (<div>Private</div>);

export default withAuthenticationRequired(PrivateRoute, {
  // Muestra un mensaje mientras el usuario espera ser redirigido a la página de inicio de sesión.
  onRedirecting: () => (<div>Redirecting you to the login page...</div>)
});
```

**Nota** Si estás usando un router personalizado, deberás proporcionar a `Auth0Provider` un método `onRedirectCallback` personalizado para ejecutar la acción que devuelve al usuario a la página protegida. Consulta ejemplos de [react-router](https://github.com/auth0/auth0-react/blob/master/EXAMPLES.md#1-protecting-a-route-in-a-react-router-dom-app), [Gatsby](https://github.com/auth0/auth0-react/blob/master/EXAMPLES.md#2-protecting-a-route-in-a-gatsby-app) y [Next.js](https://github.com/auth0/auth0-react/blob/master/EXAMPLES.md#3-protecting-a-route-in-a-nextjs-app-in-spa-mode).

<div id="call-an-api">
  ## Llamar a una API
</div>

Para llamar a una API protegida con un <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip>, asegúrate de especificar la `audience` y el `scope` de tu [token de acceso](/docs/es/secure/tokens/access-tokens/get-access-tokens), ya sea en `Auth0Provider` o en `getAccessTokenSilently`. Luego, úsalo para llamar a una API protegida incluyéndolo en el encabezado `Authorization` de tu solicitud.

```jsx lines expandable
import React, { useEffect, useState } from 'react';
import { useAuth0 } from '@auth0/auth0-react';

const Posts = () => {
  const { getAccessTokenSilently } = useAuth0();
  const [posts, setPosts] = useState(null);

  useEffect(() => {
    (async () => {
      try {
        const token = await getAccessTokenSilently({
          authorizationParams: {
            audience: 'https://api.example.com/', // Valor del campo Identifier de la API a la que se llama.
            scope: 'read:posts', // Scope existente para la API a la que se llama. Puedes crearlo mediante la Auth0 Management API o desde el Auth0 Dashboard, en la vista Permissions de tu API.
          }
        });
        const response = await fetch('https://api.example.com/posts', {
          headers: {
            Authorization: `Bearer ${token}`,
          },
        });
        setPosts(await response.json());
      } catch (e) {
        console.error(e);
      }
    })();
  }, [getAccessTokenSilently]);

  if (!posts) {
    return <div>Loading...</div>;
  }

  return (
    <ul>
      {posts.map((post, index) => {
        return <li key={index}>{post}</li>;
      })}
    </ul>
  );
};

export default Posts;
```
