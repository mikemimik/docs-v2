Cuando se realiza una autenticación con el scope `offline_access` incluido, se devolverá un <Tooltip tip="Token de actualización: token que se usa para obtener un nuevo token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token">token de actualización</Tooltip> que puede usarse para solicitar un nuevo token sin volver a pedir credenciales.

<div id="credentials-manager">
  ## Credentials Manager
</div>

[Auth0.swift](https://github.com/auth0/Auth0.swift) proporciona una clase de utilidad para simplificar el proceso de almacenar y renovar credenciales. Puedes acceder a las propiedades `accessToken` o `idToken` desde la instancia de [Credentials](https://github.com/auth0/Auth0.swift/blob/master/Auth0/Credentials.swift). Este es el método preferido para gestionar las credenciales del usuario.

Primero, importa el módulo `Auth0`:

`import Auth0`

A continuación, muestra la página de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>:

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())

Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .start { result in
        switch result {
        case .success(let credentials):
            // Pasar las credenciales al Credentials Manager
            credentialsManager.store(credentials: credentials)
        case .failure(let error):
            // Manejar el error
        }
}
```

<Warning>
  Los elementos del Keychain no se eliminan después de desinstalar tu app. Te recomendamos borrar siempre todos los elementos del Keychain de tu app cuando se inicie por primera vez.
</Warning>

<div id="credentials-check">
  ### Comprobación de credenciales
</div>

Puede ser útil hacer una comprobación rápida al iniciar la aplicación para asegurarse de que hay credenciales renovables almacenadas en el gestor. De lo contrario, se puede redirigir al usuario para que se autentique.

```swift lines
guard credentialsManager.canRenew() else {
    // Mostrar página de inicio de sesión
}
```

<div id="retrieving-user-credentials">
  ### Recuperar las credenciales del usuario
</div>

Puedes recuperar las credenciales del usuario de la siguiente manera:

```swift lines
credentialsManager.credentials { result in 
    switch result {
    case .success(let credentials):
        // Credenciales válidas; puedes acceder a las propiedades del token como
        // `idToken`, `accessToken`
    case .failure(let error):
        // Manejar el error, mostrar la página de inicio de sesión
    }
}
```

Renovar las credenciales de un usuario funciona exactamente igual si el token ha expirado. Credentials Manager renovará automáticamente las credenciales, almacenará las credenciales renovadas en el Keychain y luego devolverá un `Result` que contendrá las credenciales o un error.

<div id="alternative-method-simplekeychain">
  ## Método alternativo: SimpleKeychain
</div>

Esta sección está pensada para desarrolladores que prefieren no usar Credentials Manager. Incluimos la utilidad SimpleKeychain —un wrapper ligero sobre el sistema Keychain— que puede usarse para almacenar los tokens de forma segura.

Primero, importa el módulo `SimpleKeychain`:

`import SimpleKeychain`

A continuación, crea una instancia y almacena los tokens que necesites. En este caso, guardarás `access_token` y `refresh_token` en el Keychain después de una autenticación correcta.

```swift lines
let keychain = SimpleKeychain(service: "Auth0")

Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .start { result in
        switch result {
        case .success(let credentials):
            guard let refreshToken = credentials.refreshToken else { 
                // Manejar el error 
                return
            }
            // Almacenar los tokens
            do {
                try keychain.set(credentials.accessToken, forKey: "access_token")
                try keychain.set(refreshToken, forKey: "refresh_token")
            } catch {
                // Manejar el error
            }
            // En este punto, puede redirigir al flujo principal de su aplicación
        case .failure(let error):
            // Manejar el error
        }
}
```

Una vez que las tengas almacenadas, puedes solicitar una nueva instancia de [Credentials](https://github.com/auth0/Auth0.swift/blob/master/Auth0/Credentials.swift) en cualquier momento.

<div id="renewing-user-credentials">
  ### Renovación de credenciales de usuario
</div>

```swift lines expandable
let keychain = SimpleKeychain(service: "Auth0")

Auth0
    .authentication()
    .renew(withRefreshToken: refreshToken)
    .start { result in
        switch(result) {
        case .success(let credentials):
            // Si tienes la rotación del Refresh Token habilitada, obtienes un 
            // nuevo token de actualización
            // De lo contrario, solo obtienes un nuevo token de acceso
            guard let refreshToken = credentials.refreshToken else { 
                // Gestionar el error 
                return
            }
            // Guardar los nuevos tokens
            do {
                try keychain.set(credentials.accessToken, forKey: "access_token")
                try keychain.set(refreshToken, forKey: "refresh_token")
            } catch {
                // Gestionar el error
            }
        case .failure(let error):
            // Gestionar el error
        }
}
```
