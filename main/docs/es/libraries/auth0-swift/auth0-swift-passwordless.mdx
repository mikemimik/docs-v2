<Tooltip tip="Passwordless: Forma de autenticación que no utiliza una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> permite a los usuarios iniciar sesión solo con una dirección de correo electrónico o un número de teléfono, lo que reduce la fricción de tener que recordar una contraseña. La autenticación passwordless puede realizarse por correo electrónico o por SMS, ya sea enviando al usuario un código o un enlace que lo contiene.

Para usar la autenticación Passwordless, necesitas Auth0.Swift versión `1.20.0` o superior.

<div id="how-passwordless-works">
  ## Cómo funciona Passwordless
</div>

Passwordless requiere dos pasos:

1. Solicitar el código
2. Introducir el código

Al usar enlaces, ocurre lo mismo, pero de una forma ligeramente distinta, porque el usuario no tiene que introducir el código por sí mismo. El código está incluido en la URL.

<div id="step-1-request-the-code">
  ### Paso 1: Solicita el código
</div>

En este ejemplo, el código se solicita llamando a `startPasswordless` con el correo electrónico del usuario y el tipo de conexión. El parámetro `type` será `Code` de forma predeterminada. Si la operación se completa correctamente, probablemente mostrarás un aviso al usuario indicándole que su código está por llegar y, quizás, lo redirigirás a una vista para introducirlo.

```swift lines
Auth0
   .authentication()
   .startPasswordless(email: "support@auth0.com")
   .start { result in
       switch result {
       case .success:
           print("Sent OTP to support@auth0.com!")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="step-2-input-the-code">
  ### Paso 2: Introduce el código
</div>

Una vez que el usuario tenga el código, puede introducirlo. Llama al método `login` y pásale el correo electrónico del usuario, el código que recibió y el nombre de la conexión correspondiente. Si la operación se completa correctamente, recibirás un objeto Credentials en la respuesta.

```swift lines
Auth0
   .authentication()
   .login(
       email: "support@auth0.com", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

Si usaste SMS, la llamada sería similar a este ejemplo:

```swift lines
Auth0
   .authentication()
   .login(
       phoneNumber: "+4591131761367", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="passwordless-parameters">
  ## Parámetros de Passwordless
</div>

La autenticación passwordless se puede iniciar con distintos parámetros.

Por ejemplo:

`.startPasswordless(email: String, type: String, connection: String)`

o

`.startPasswordless(phoneNumber: String, type: String, connection: String)`

| **Parámetro** | **Obligatorio** | **Descripción**                                                                                                                                                                          |
| ------------- | --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `email`       | obligatorio     | (String) Se requiere `email` o `phoneNumber` (no ambos), según cuál se vaya a utilizar.                                                                                                  |
| `phoneNumber` | obligatorio     | (String) Se requiere `email` o `phoneNumber` (no ambos), según cuál se vaya a utilizar.                                                                                                  |
| `type`        | opcional        | (String) El tipo de transacción de Passwordless que se va a usar: `.Code` o `.iOSLink`. El valor predeterminado es `.Code`.                                                              |
| `connection`  | opcional        | (String) El nombre de la conexión que se va a usar para la autenticación passwordless. El valor predeterminado es `sms` para la sobrecarga de SMS o `email` para la sobrecarga de email. |
