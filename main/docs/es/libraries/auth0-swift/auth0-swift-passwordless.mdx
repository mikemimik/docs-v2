La autenticación <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> permite a los usuarios iniciar sesión usando solo una dirección de correo electrónico o un número de teléfono, lo que reduce la fricción de tener que recordar una contraseña. La autenticación passwordless puede realizarse por correo electrónico o por SMS, ya sea enviando al usuario un código o un enlace que lo contiene.

Para usar la autenticación Passwordless, necesitas Auth0.Swift versión `1.20.0` o superior.

<div id="how-passwordless-works">
  ## Cómo funciona Passwordless
</div>

Passwordless requiere dos pasos:

1. Solicitar el código
2. Introducir el código

Cuando se usan enlaces, ocurre lo mismo, pero de una forma ligeramente distinta, ya que el usuario no tiene que introducir el código por sí mismo. El código está incluido en la URL.

<div id="step-1-request-the-code">
  ### Paso 1: Solicitar el código
</div>

En este ejemplo, para solicitar el código se llama a `startPasswordless` con el correo electrónico del usuario y el tipo de conexión. El parámetro `type` usará `Code` de forma predeterminada. Si todo va bien, probablemente le mostrarás al usuario un aviso indicándole que su código está en camino y quizá lo redirijas a una vista para introducirlo.

```swift lines
Auth0
   .authentication()
   .startPasswordless(email: "support@auth0.com")
   .start { result in
       switch result {
       case .success:
           print("Sent OTP to support@auth0.com!")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="step-2-input-the-code">
  ### Paso 2: Introduce el código
</div>

Una vez que el usuario tenga un código, podrá introducirlo. Llama al método `login` y pásale el correo electrónico del usuario, el código que recibió y el nombre de la conexión correspondiente. Si la operación se realiza correctamente, recibirás un objeto Credentials en la respuesta.

```swift lines
Auth0
   .authentication()
   .login(
       email: "support@auth0.com", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

Si utilizaste SMS, la llamada sería similar a este ejemplo:

```swift lines
Auth0
   .authentication()
   .login(
       phoneNumber: "+4591131761367", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="passwordless-parameters">
  ## Parámetros de Passwordless
</div>

La autenticación passwordless puede iniciarse con distintos parámetros.

Por ejemplo:

`.startPasswordless(email: String, type: String, connection: String)`

o

`.startPasswordless(phoneNumber: String, type: String, connection: String)`

| **Parámetro** | **Obligatorio** | **Descripción**                                                                                                                                                                                     |
| ------------- | --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `email`       | obligatorio     | (String) Se requiere `email` o `phoneNumber` (no ambos), según cuál se vaya a utilizar.                                                                                                             |
| `phoneNumber` | obligatorio     | (String) Se requiere `email` o `phoneNumber` (no ambos), según cuál se vaya a utilizar.                                                                                                             |
| `type`        | opcional        | (String) El tipo de transacción passwordless que se utilizará, ya sea `.Code` o `.iOSLink`. El valor predeterminado es `.Code`.                                                                     |
| `connection`  | opcional        | (String) El nombre de la conexión que se utilizará para la autenticación de Passwordless. El valor predeterminado es `sms` para la variante de SMS o `email` para la variante de correo electrónico |
