La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> proporciona [Vinculación de cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking), lo que te permite vincular y desvincular cuentas de usuario distintas de diferentes proveedores y asociarlas a un único perfil. También te permite actualizar los metadatos del usuario y otra información del perfil.

<div id="link-users">
  ## Vincular usuarios
</div>

Vincular cuentas de usuario permite que un usuario se autentique con cualquiera de sus cuentas y, sin importar cuál use, siga accediendo al mismo perfil al iniciar sesión. Auth0 trata todas estas cuentas como perfiles separados de forma predeterminada, así que, si quieres que las cuentas de un usuario estén vinculadas, esta es la forma de hacerlo.

El método `link` acepta dos parámetros: el ID de usuario del perfil principal y el <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o un JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> del perfil secundario (el token obtenido tras iniciar sesión con esta identidad). El ID de usuario en cuestión es el identificador único de esta cuenta de usuario. Si el ID tiene el formato `facebook|1234567890`, el ID requerido es la parte que aparece después de la barra vertical delimitadora (en este caso, `1234567890`).

```swift lines
Auth0
   .users(token: "user-scoped access token")
   .link(userId, withOtherUserToken: "another user token")
   .start { result in
      switch result {
      case .success(let userInfo):
        print("User: \(userInfo)")
      case .failure(let error):
        print(error)
      }
   }
```

<div id="unlink-users">
  ## Desvincular usuarios
</div>

Desvincular usuarios es un proceso similar al de vincularlos. El método `unlink` recibe tres parámetros: el ID de usuario del perfil secundario, el proveedor del perfil secundario (el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> de la conexión) y el ID de usuario del perfil principal.
En esencia, los parámetros se leen así: &quot;Desvincula este **usuario secundario** (con este **proveedor**) de este **usuario primario**&quot;.

```swift lines
Auth0
   .users(token: "user-scoped access token")
   .unlink(identityId: identifier, provider: provider, fromUserId:userId)
   .start { result in
      switch result {
      case .success(let userInfo):
        print("User: \(userInfo)")
      case .failure(let error):
        print(error)
      }
   }
```

Cuando se vinculan cuentas, los metadatos del perfil de la cuenta secundaria no se fusionan con los del perfil de la cuenta principal. De igual manera, al desvincular cuentas, el perfil de la cuenta secundaria no conserva los metadatos del perfil de la cuenta principal.

<div id="retrieve-user-metadata">
  ## Recuperar los metadatos del usuario
</div>

```swift lines
Auth0
    .users(token: "user-scoped access token")
    .get(userId, fields: ["user_metadata"], include: true)
    .start { result in
        switch result {
        case .success(let userInfo):
            print("User: \(userInfo)")
        case .failure(let error):
            print(error)
        }
    }
```

<div id="update-user-metadata">
  ## Actualizar los metadatos del usuario
</div>

Al actualizar los metadatos del usuario, crearás un objeto `userMetadata` y luego llamarás al método `patch`, al que le pasarás el ID de usuario y el objeto `userMetadata`. Los valores de este objeto sobrescribirán los valores existentes que tengan la misma clave, o agregarán otros nuevos si todavía no existen en los metadatos del usuario.

```swift lines
Auth0
    .users(token: "user-scoped access token")
    .patch("user identifier", userMetadata: ["first_name": "John", "last_name": "Doe"])
    .start { result in
        switch result {
        case .success(let userInfo):
          print("User: \(userInfo)")
        case .failure(let error):
          print(error)
        }
    }
```
