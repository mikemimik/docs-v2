Los entornos de desarrollo local suelen ejecutarse en http (es decir, `http://localhost`) de forma predeterminada. Esta guía explica cuándo conviene ejecutar un servidor local con https y cómo configurar https en localhost.

<div id="when-to-use-a-local-https-server">
  ## Cuándo usar un servidor local https
</div>

Los navegadores web se comportan de forma diferente en http y en https; para tener en cuenta estas diferencias, puede que necesites que tu servidor de desarrollo sirva páginas a través de https. Por ejemplo:

* Cuando tu página se sirve a través de https, se bloquearán las solicitudes para cargar recursos desde una URL http. Puede que te convenga este comportamiento en local para evitar errores que solo se producirían en el entorno de Production o para no hacer por error solicitudes http que contengan información confidencial.
* Las cookies que usan el atributo Secure o tienen el atributo [SameSite](https://auth0.com/blog/browser-behavior-changes-what-developers-need-to-know/) configurado como None no se enviarán por canales inseguros, lo que interrumpirá la autenticación y otras funciones.

<div id="how-to-set-up-a-secure-local-server">
  ## Cómo configurar un servidor local seguro
</div>

La idea es ejecutar un servidor web delante de tu servidor de desarrollo local para que acepte peticiones https y las reenvíe al servidor web de tu aplicación a través de http, de modo que las peticiones de tu aplicación se sirvan por https mediante proxy. Esto se llama proxy inverso https y podemos usar un producto listo para usar llamado [Caddy](https://caddyserver.com/v2) para crear uno.

El proceso para hacerlo es:

1. Descargar el binario de Caddy
2. Ejecutar el servidor web de tu aplicación sobre http, como de costumbre
3. Ejecutar el binario de Caddy como proxy inverso, indicándole que reenvíe el tráfico al servidor web de tu aplicación
4. Acceder al servidor web del proxy inverso a través de https

<div id="step-1-download-the-caddy-binary">
  ### Paso 1. Descarga el binario de Caddy
</div>

Visita https://caddyserver.com/download, selecciona tu plataforma y descarga el archivo en tu computadora.

<div id="step-2-run-your-applications-web-server-over-http-as-normal">
  ### Paso 2. Ejecuta el servidor web de tu aplicación por HTTP como de costumbre
</div>

Ejecuta el servidor de desarrollo local de tu aplicación como de costumbre; por ejemplo, $ PORT=3000 npm start

<div id="step-3-run-the-caddy-binary-as-a-reverse-proxy-telling-it-to-forward-traffic-to-your-applications-web-server">
  ### Paso 3. Ejecuta el binario de Caddy como proxy inverso para redirigir el tráfico al servidor web de tu aplicación
</div>

Busca dónde descargaste el binario de Caddy y ejecuta el siguiente comando para crear un proxy inverso y redirigir las solicitudes del puerto 443 (el puerto predeterminado que se usa para https) al puerto de tu aplicación (en este caso, 3000):

```bash wrap lines
/path/to/caddy reverse-proxy --from localhost:443 --to localhost:3000
```

<div id="step-4-now-visit-your-web-server-over-https">
  ### Paso 4. Ahora visita tu servidor web a través de HTTPS
</div>

Visita https://localhost y servirá las solicitudes de tu aplicación a través de HTTPS

<div id="running-your-application-behind-a-proxy">
  ## Ejecutar tu aplicación detrás de un proxy
</div>

Puede que tengas que hacer algunos pequeños ajustes en tu aplicación cuando se ejecute detrás de un proxy.

Por ejemplo, en Express.js, aunque la aplicación no dejará de ejecutarse, registrará incorrectamente la dirección IP del proxy como la dirección IP del cliente, a menos que se configure `trust proxy`; consulta &lt;https://expressjs.com/en/guide/behind-proxies.html&gt;
