<ParamField body="useMfaPolling(options?: MfaPollingOptions)" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPollingResult">MfaPollingResult</a></span>}>
  Hook de React para gestionar el polling de MFA push (p. ej., esperar la aprobación de una push notification)
  en una pantalla de Advanced Customization of Universal Login (ACUL) de Auth0.

  Este hook configura y controla un bucle de polling de larga duración que consulta repetidamente
  el endpoint de MFA push challenge hasta que ocurre una de las siguientes situaciones:

  * El challenge es **aprobado o rechazado** por el usuario, lo que activa `options.onCompleted`.
  * Se produce un **error** (error de red o respuesta distinta de 200/429), lo que activa `options.onError`.
  * El **componente se desmonta** o se llama a `stopPolling()`, lo que cancela el polling.

  ### Funcionalidades clave

  * `isRunning` es **reactivo**: se actualiza automáticamente si el bucle de polling
    se detiene internamente o se cancela.
  * Usa una **única instancia estable de polling** (`useRef`) para evitar
    llamadas de red duplicadas y reinicios no deseados durante los re-renderizados de React.
  * **Limpieza automática** al desmontarse: sin temporizadores huérfanos ni solicitudes XHR pendientes.

  ## Parámetros

  <ParamField body="options" type={<span><a href='/docs/es/libraries/acul/react-sdk/API-Reference/Types/type-aliases/MfaPollingOptions'>MfaPollingOptions</a></span>}>
    Especifica el intervalo de polling, la devolución de llamada de éxito (`onCompleted`) y el controlador de errores opcional (`onError`).
  </ParamField>

  ## Devuelve

  [`MfaPollingResult`](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPollingResult), que contiene:

  * `isRunning` — `true` mientras el polling está activo.
  * `startPolling()` — inicia o reanuda el polling.
  * `stopPolling()` — detiene el polling inmediatamente.

  ## Pantallas compatibles

  * `mfa-push-challenge-push`
  * `reset-password-mfa-push-challenge-push`
  * `mfa-push-enrollment-qr`

  ```tsx Example
  import { useMfaPolling } from '@auth0/auth0-acul-react/mfa-push-challenge-push';

  export function MfaPushStatus() {
    const { isRunning, startPolling, stopPolling } = useMfaPolling({
      intervalMs: 5000,
      onCompleted: () => console.log('Push approved!/denied'),
      onError: (error) => console.error('Polling error:', error)
    });

    return (
      <div>
        <button onClick={startPolling} disabled={isRunning}>
          {isRunning ? 'Waiting for approval…' : 'Start MFA Polling'}
        </button>
        {isRunning && <button onClick={stopPolling}>Cancel</button>}
      </div>
    );
  }
  ```

  ## Observaciones

  * La devolución de llamada `onError` recibe un objeto [ULError](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error)
    con `status` y `responseText` que describen la respuesta del servidor.
  * Las respuestas internas de limitación de tasa (`429`) se gestionan automáticamente:
    el polling espera a que finalice la ventana de restablecimiento antes de reintentarlo.
  * Llamar repetidamente a `startPolling()` mientras está en ejecución es seguro e idempotente.
</ParamField>
