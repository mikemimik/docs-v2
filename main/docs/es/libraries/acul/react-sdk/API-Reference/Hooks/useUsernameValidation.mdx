```tsx
useUsernameValidation(

  options?: { includeInErrors?: boolean },
): UsernameValidationResult
```

<ParamField body="useUsernameValidation" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  Hook de React para validar un username según la política de username actual de Auth0.

  Este hook comprueba el username proporcionado con todas las reglas de validación configuradas
  y devuelve un resultado estructurado que indica si las cumple.
  Opcionalmente, puede enviar errores de validación al administrador global de errores para que
  los componentes de UI que observan el campo `username` puedan mostrarlos o reaccionar a ellos
  automáticamente.

  ### Funcionalidades clave

  * **Validación según la política** — comprueba el username con las reglas de la política de username de Auth0 configuradas en el tenant.
  * **Integración con el administrador de errores** — opcionalmente muestra automáticamente los fallos de validación en los componentes de error del formulario.
  * **Recálculo optimizado** — solo vuelve a calcular cuando cambian `username` o `options.includeInErrors`.

  ## Parámetros

  <ParamField body="username" type="string">
    La cadena de username que se va a validar.
  </ParamField>

  <ParamField body="options" type="{ includeInErrors?: boolean }">
    Configuración opcional para el hook.

    <Expandable title="propiedades">
      <ParamField body="includeInErrors" type="boolean">
        Cuando es `true`, los errores de validación se almacenan en el administrador global de errores bajo el campo `username`. El valor predeterminado es `false`.
      </ParamField>
    </Expandable>
  </ParamField>

  ## Devuelve

  [`UsernameValidationResult`](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult)

  Un objeto [UsernameValidationResult](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult) con:

  * `isValid` — `true` si el username cumple todas las reglas configuradas.
  * `errors` — un arreglo de errores de validación por regla con `code`, `message` e `isValid`.

  ## Pantallas compatibles

  * `signup`
  * `signup-id`

  ```tsx Example
  import { useUsernameValidation } from "@auth0/auth0-acul-react/signup";

  export function UsernameField() {
    const { isValid, errors } = useUsernameValidation(username, { includeInErrors: true });

    return (
      <div>
        <input
          value={username}
          onChange={e => setUsername(e.target.value)}
          aria-invalid={!isValid}
        />

        {!isValid && (
          <ul>
            {errors.map(err => (
              <li key={err.code}>{err.message}</li>
            ))}
          </ul>
        )}
      </div>
    );
  }
  ```

  ## Observaciones

  * Cuando `includeInErrors` está habilitado, el hook actualiza automáticamente los errores en el almacén de errores, que puede consumir el hook `useErrors`.
  * El hook solo vuelve a calcular cuando cambian `username` o `options.includeInErrors`.
  * Llama a `useUsernameValidation` en el nivel superior de tu componente; no lo llames de forma condicional ni dentro de controladores de eventos.
  * La ruta de importación debe coincidir con la pantalla; usa `signup` o `signup-id` según corresponda.
</ParamField>
