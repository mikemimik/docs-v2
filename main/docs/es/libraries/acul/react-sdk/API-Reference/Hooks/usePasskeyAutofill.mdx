<ParamField body="usePasskeyAutofill()" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsePasskeyAutofillResult">UsePasskeyAutofillResult</a></span>}>
  Hook de React que habilita la funcionalidad de UI condicional del navegador (autocompletado de clave de acceso) para los
  campos identificadores de inicio de sesión en las pantallas de Advanced Customization of Universal Login (ACUL) de Auth0.

  Este hook inicializa la API Conditional Mediation del navegador, que muestra las
  claves de acceso almacenadas en el dispositivo en el menú desplegable de autocompletado del campo de nombre de usuario sin requerir
  ninguna interacción adicional por parte del usuario.

  * El hook se registra **una sola vez por ciclo de vida de la página**; es seguro llamarlo varias veces.
  * **Se degrada de forma progresiva** en navegadores no compatibles sin bloquear la interacción del usuario.
  * Vincular el `inputRef` devuelto a un campo de entrada garantiza que el atributo
    `autocomplete="username webauthn"` correcto se aplique automáticamente.
  * Si tu campo de entrada ya declara `autocomplete="username webauthn"` en el marcado,
    **vincular la ref es opcional**; el hook se sigue registrando correctamente.

  ### Características clave

  * **Conditional Mediation automática** — inicializa el autocompletado de clave de acceso sin
    necesidad de llamadas manuales a la API del navegador.
  * **Diseño con ref opcional** — funciona tanto si vinculas `inputRef` al campo de entrada como si no.
  * **Alternativa sin fricción** — los entornos no compatibles se gestionan silenciosamente,
    preservando el flujo de inicio de sesión normal.

  ## Parámetros

  None.

  ## Devuelve

  [`UsePasskeyAutofillResult`](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsePasskeyAutofillResult)

  Objeto que contiene:

  * `inputRef` *(optional)* — una ref de React para adjuntarla al campo de entrada de nombre de usuario. Garantiza
    que el atributo `autocomplete` correcto se establezca al vincularla.

  ## Pantallas compatibles

  * `login-id`

  ```tsx Ejemplo
  import { usePasskeyAutofill } from '@auth0/auth0-acul-react/login-id';

  export function LoginIdentifierInput() {
    const { inputRef } = usePasskeyAutofill();

    return (
      <input
        ref={inputRef}
        type="text"
        name="username"
        placeholder="Email or username"
      />
    );
  }
  ```

  ## Observaciones

  * Si `inputRef` no está vinculado a un campo de entrada, asegúrate de que el elemento declare explícitamente
    `autocomplete="username webauthn"` para que el navegador pueda asociarlo con el autocompletado de clave de acceso.
  * El hook no tiene efecto en entornos donde la API Conditional Mediation no está disponible
    (por ejemplo, navegadores antiguos o contextos que no usan HTTPS); no se genera ningún error.
  * Llama a `usePasskeyAutofill` una vez en el nivel superior de tu componente; no lo llames
    de forma condicional ni dentro de controladores de eventos.
