<div id="methods">
  ## Métodos
</div>

* [useAuth0Themes](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes): Hook de React para obtener las opciones del tema actual con la configuración consolidada del contexto de branding.
* [useChangeLanguage](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage): Hook de React que devuelve una función para cambiar el idioma mostrado en la pantalla actual de ACUL.
* [useCurrentScreen](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen): Hook de React que obtiene el contexto y el estado de la pantalla actual.
* [useErrors](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useErrors): Hook de React para leer y gestionar errores en ACUL (Advanced Customization of Universal Login). Incluye todos los errores de validación y del servidor. Agrupa los errores en tres tipos: - `server` — errores devueltos por Auth0 o por tu propio backend. - `client` — errores de validación del lado del cliente (p. ej., entradas de formulario no válidas). - `developer` — errores causados por una integración incorrecta o un uso indebido del SDK.
* [useLoginIdentifiers](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useLoginIdentifiers): Devuelve una lista de tipos de identificador activos (como `'email'`, `'phone'` o `'username'`) que se usan actualmente en el flujo de autenticación o en el proceso de inicio de sesión.
* [useMfaPolling](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useMfaPolling): Hook de React para gestionar el polling de MFA push (p. ej., esperar la aprobación de una notificación push) en una pantalla de Auth0 Advanced Customization of Universal Login (ACUL).
* [useResend](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useResend): Este hook de React gestiona las acciones de «reenviar» (p. ej., reenviar un código de verificación) en las pantallas de ACUL.
* [useSignupIdentifiers](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useSignupIdentifiers): Devuelve una lista de identificadores habilitados (email, phone o username), cada uno con su estado `required`, según los identificadores de registro de la pantalla actual.
* [usePasswordValidation](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/usePasswordValidation): Este hook de React valida una contraseña según la política actual de contraseñas de Auth0 y devuelve un resultado estructurado que indica si la contraseña cumple cada regla.
* [useUsernameValidation](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useUsernameValidation): Hook de React para validar un nombre de usuario según la política actual de nombres de usuario de Auth0.
