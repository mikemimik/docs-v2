<ParamField body="useErrors(options?: UseErrorOptions)" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult">UseErrorsResult</a></span>}>
  Hook de React para consultar y gestionar errores en ACUL (Advanced Customization of Universal Login).
  Agrupa los errores en tres tipos:

  * `auth0` — errores devueltos por Auth0 o por tu propio backend.
  * `validation` — errores de validación del lado del cliente (p. ej., datos de formulario no válidos).
  * `configuration` — errores provocados por una integración incorrecta o por un uso indebido del SDK.

  ### Funcionalidades clave

  * **Almacén de errores unificado** — reúne los errores de auth0, validation y configuration en un único hook.
  * **Filtrado flexible** — consulta errores por tipo, campo o una combinación de ambos.
  * **Controles de descarte** — elimina errores individuales o borra todos de una sola vez.

  ## Parámetros

  <ParamField body="options" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorOptions">UseErrorOptions</a></span>}>
    Configuración opcional para el hook.

    <Expandable title="propiedades">
      <ParamField body="includeDevErrors" type="boolean">
        Cuando es `true`, incluye en la lista detalles de errores específicos del desarrollo. El valor predeterminado es `false`.
      </ParamField>
    </Expandable>
  </ParamField>

  ## Devuelve

  [`UseErrorsResult`](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult)

  Un objeto [`UseErrorsResult`](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult) que contiene:

  * `errors` — la lista completa de errores de tipo [`ErrorsResult`](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ErrorsResult), con funciones auxiliares:
    * `errors.byType(type, filter?)` — filtra por tipo de error y, opcionalmente, por field.
    * `errors.byField(field, filter?)` — filtra por field y, opcionalmente, por tipo.
  * `hasError` — `true` si actualmente hay algún error.
  * `dismiss(id)` — elimina un error específico por su ID.
  * `dismissAll()` — borra todos los errores registrados.

  ## Pantallas compatibles

  * El hook `useErrors` está disponible en todas las pantallas de ACUL.

  ```tsx Example
  import { useErrors } from "@auth0/auth0-acul-react";

  export function SignupForm() {
    const { errors, hasError, dismiss, dismissAll } = useErrors();

    return (
      <div>
        {hasError && (
          <div className="mb-4">
            {errors.byType("auth0").map(err => (
              <div key={err.id} className="text-red-600">
                {err.message}
                <button onClick={() => dismiss(err.id)}>Dismiss</button>
              </div>
            ))}
          </div>
        )}

        <button onClick={dismissAll}>Clear All Errors</button>
      </div>
    );
  }
  ```

  Además de renderizar mensajes, puedes filtrar por campo o tipo:

  ```ts
  console.log(errors.byType('validation')); // todos los errores de validación
  console.log(errors.byType('validation', { field: 'username' })); // errores de validación para el field 'username'
  console.log(errors.byField('username')); // todos los errores para el field 'username'
  console.log(errors.byField('username', { type: 'auth0' })); // errores de auth0 para el field 'username'
  ```

  ## Observaciones

  * `useErrors` no está asociado a una pantalla concreta — impórtalo directamente desde `@auth0/auth0-acul-react`.
  * Llama a `useErrors` en el nivel superior de tu componente; no lo llames de forma condicional ni dentro de controladores de eventos.
  * Usa `includeDevErrors: true` durante el desarrollo para mostrar errores por uso indebido del SDK; desactívalo en producción.
