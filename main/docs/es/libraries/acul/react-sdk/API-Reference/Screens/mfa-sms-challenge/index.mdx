La pantalla `mfa-sms-challenge` pide al usuario que introduzca un código de un solo uso enviado a su teléfono por SMS para completar la verificación MFA.

<div id="import">
  ## Importación
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite **importación parcial** e **importación desde la raíz** para cada pantalla.

* La importación parcial te permite incluir solo el código que necesitas para tu caso de uso.
* La importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo que resulta útil si quieres una compilación unificada para gestionar todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useMfaSmsChallenge } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useMfaSmsChallenge,
  // Hooks de contexto
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks de utilidad
  useChangeLanguage,
  useResend,
  // Métodos
  continueMfaSmsChallenge,
  getACall,
  pickSms,
  resendCode,
  tryAnotherMethod,
} from "@auth0/auth0-acul-react/mfa-sms-challenge";

function MfaSmsChallengeScreen() {
  const { continueMfaSmsChallenge } = useMfaSmsChallenge();
  return (
    <button onClick={() => continueMfaSmsChallenge({ code: '123456' })}>
      Verify Code
    </button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `mfa-sms-challenge`. Impórtalos desde `@auth0/auth0-acul-react/mfa-sms-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona la configuración de identidad visual, como el logotipo, los colores y los ajustes del tema que se muestran en la pantalla `mfa-sms-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `mfa-sms-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la Organización del usuario si el flujo de MFA tiene alcance de Organización. Devuelve `null` cuando no hay contexto de Organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>Sin contexto de organización</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual en el flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnMfaSmsChallenge">ScreenMembersOnMfaSmsChallenge</a></span>}>
  Este hook contiene detalles específicos de la pantalla `mfa-sms-challenge`, incluida su configuración y contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y los metadatos asociados.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Este hook proporciona datos específicos de la transacción para la pantalla `mfa-sms-challenge`, como el estado actual del flujo de MFA.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembersOnMfaSmsChallenge">UntrustedDataMembersOnMfaSmsChallenge</a></span>}>
  Este hook gestiona los datos no confiables que se pasan a la pantalla, como valores rellenados previamente a partir de parámetros de URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook proporciona información del usuario activo, incluido `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useMfaSmsChallenge" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaSmsChallengeMembers">MfaSmsChallengeMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `mfa-sms-challenge`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="continueMfaSmsChallenge" type="Promise<void>">
  Este método envía el código de un solo uso por SMS para completar el desafío MFA.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function VerifyCodeForm() {
    const { continueMfaSmsChallenge } = useMfaSmsChallenge();
    return (
      <button onClick={() => continueMfaSmsChallenge({ code: '123456' })}>
        Verificar código
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [MfaSmsChallengeOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaSmsChallengeOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      El código de un solo uso enviado al teléfono del usuario por SMS.
    </ParamField>

    <ParamField body="rememberDevice" type="boolean">
      Si se debe recordar el dispositivo y omitir MFA en futuros inicios de sesión.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getACall" type="Promise<void>">
  Este método cambia el desafío MFA a una llamada de voz en lugar de SMS.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function GetACallButton() {
    const { getACall } = useMfaSmsChallenge();
    return (
      <button onClick={() => getACall()}>
        Recibir una llamada en su lugar
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="pickSms" type="Promise<void>">
  Este método navega a la pantalla de selección del número de teléfono para SMS.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function PickSmsButton() {
    const { pickSms } = useMfaSmsChallenge();
    return (
      <button onClick={() => pickSms()}>
        Usar un número de teléfono diferente
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="resendCode" type="Promise<void>">
  Este método reenvía el código de un solo uso por SMS al teléfono del usuario.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function ResendCodeButton() {
    const { resendCode } = useMfaSmsChallenge();
    return (
      <button onClick={() => resendCode()}>
        Reenviar código
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Este método navega a la pantalla de selección del método MFA para que el usuario pueda elegir un factor de autenticación diferente.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useMfaSmsChallenge();
    return (
      <button onClick={() => tryAnotherMethod()}>
        Probar otro método
      </button>
    );
  }
  ```
</ParamField>

<div id="commonutility-hooks">
  ## Hooks comunes y de utilidad

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useResend">useResend</a>} type="Hooks">
  Este hook gestiona el estado y la lógica del tiempo de espera para el reenvío en pantallas con funcionalidad para reenviar códigos.

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Este hook devuelve una función para cambiar el idioma que se muestra en la pantalla actual de ACUL.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Este hook obtiene el contexto y el estado de la pantalla actual.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Este hook lee y gestiona los errores del servidor, del cliente y del desarrollador en la pantalla.
</ParamField>
