La pantalla `email-otp-challenge` se muestra cuando un usuario necesita verificar su identidad introduciendo un código de un solo uso enviado a su dirección de correo electrónico. Permite al usuario enviar el código o solicitar que se reenvíe.

<div id="import">
  ## Import
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite **importación parcial** e **importación desde la raíz** para cada pantalla.

* La importación parcial te permite incluir solo el código que necesitas para tu caso de uso.
* La importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo que resulta útil cuando quieres una compilación unificada que gestione todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useEmailOTPChallenge } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useEmailOTPChallenge,
  // Hooks de contexto
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  useResend,
  // Hooks de utilidad
  useChangeLanguage,
  // Métodos
  submitCode,
  resendCode,
} from '@auth0/auth0-acul-react/email-otp-challenge';

function EmailOtpChallengeScreen() {
  const { submitCode } = useEmailOTPChallenge();
  return (
    <button onClick={() => submitCode({ code: '123456' })}>Submit</button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks con alcance de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `email-otp-challenge`. Impórtalos desde `@auth0/auth0-acul-react/email-otp-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona la configuración de Branding, como el logotipo, los colores y los ajustes del tema que se muestran en la pantalla `email-otp-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/email-otp-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `email-otp-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/email-otp-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la organización del usuario si el desafío OTP tiene alcance de organización. Devuelve `null` cuando no hay contexto de organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/email-otp-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No Organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual en el flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/email-otp-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnEmailOTPChallenge">ScreenMembersOnEmailOTPChallenge</a></span>}>
  Este hook contiene detalles específicos de la pantalla `email-otp-challenge`, incluida su configuración y contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/email-otp-challenge';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y los metadatos asociados.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/email-otp-challenge';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Este hook proporciona datos específicos de la transacción para la pantalla `email-otp-challenge`, como las conexiones activas y el estado actual del flujo.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/email-otp-challenge';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Este hook gestiona datos no confiables que se pasan a la pantalla, como parámetros rellenados previamente desde el query string de la URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/email-otp-challenge';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook proporciona información del usuario activo, incluidos `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/email-otp-challenge';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useEmailOTPChallenge" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/EmailOTPChallengeMembers">EmailOTPChallengeMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `email-otp-challenge`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="resendCode" type="Promise<void>">
  Este método vuelve a enviar el código de un solo uso a la dirección de correo electrónico del usuario.

  ```jsx Example
  import { useEmailOTPChallenge } from '@auth0/auth0-acul-react/email-otp-challenge';

  function ResendButton() {
    const { resendCode } = useEmailOTPChallenge();
    return (
      <button onClick={() => resendCode()}>
        Resend Code
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="submitCode" type="Promise<void>">
  Este método envía el código de un solo uso introducido por el usuario.

  ```jsx Example
  import { useEmailOTPChallenge } from '@auth0/auth0-acul-react/email-otp-challenge';

  function SubmitButton() {
    const { submitCode } = useEmailOTPChallenge();
    return (
      <button onClick={() => submitCode({ code: '123456' })}>
        Continue
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [OtpCodeOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OtpCodeOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      El código OTP introducido por el usuario.
    </ParamField>
  </Expandable>
</ParamField>

<div id="commonutility-hooks">

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">

</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useResend">useResend</a>} type="Hooks">
  Este hook gestiona el estado de reenvío y la lógica del tiempo de espera para el código de Email OTP.
</ParamField>
