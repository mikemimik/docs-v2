---
title: "LoginId"
description: "Describe los hooks y métodos disponibles para personalizar la pantalla `login-id` de Universal Login."
---

La pantalla `login-id` recopila el identificador del usuario como primer paso de un flujo de autenticación en el que primero se solicita el identificador. Según tu tenant, este identificador puede ser un correo electrónico, un número de teléfono o un nombre de usuario. También admite el inicio de sesión con clave de acceso y proveedores de identidad federados.

<div id="import">
  ## Importación
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite **importación parcial** e **importación desde la raíz** para cada pantalla.

* La importación parcial te permite incluir solo el código que necesitas para tu caso de uso específico.
* La importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo que resulta útil cuando quieres una única compilación que gestione todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useLoginId } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useLoginId,
  // Hooks de contexto
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks de utilidad
  useLoginIdentifiers,
  useChangeLanguage,
  usePasskeyAutofill,
  // Métodos
  login,
  federatedLogin,
  passkeyLogin,
  pickCountryCode,
} from '@auth0/auth0-acul-react/login-id';

function LoginIdForm() {
  const { login } = useLoginId();
  return (
    <button onClick={() => login({ username: 'user@example.com' })}>
      Continue
    </button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks con alcance a nivel de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `login-id`. Impórtalos desde `@auth0/auth0-acul-react/login-id`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona la configuración de branding, como el logotipo, los colores y los ajustes del tema que se muestran en la pantalla `login-id`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/login-id';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `login-id`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/login-id';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la organización del usuario si el inicio de sesión está limitado a una organización. Devuelve `null` cuando no hay contexto de organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/login-id';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual dentro del flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/login-id';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnLoginId">ScreenMembersOnLoginId</a></span>}>
  Este hook contiene detalles específicos de la pantalla `login-id`, incluida su configuración y su contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/login-id';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y la metadata asociada.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/login-id';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembersOnLoginId">TransactionMembersOnLoginId</a></span>}>
  Este hook proporciona datos específicos de la transacción para la pantalla `login-id`, como conexiones activas, métodos de inicio de sesión disponibles y el estado actual del flujo.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/login-id';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Este hook maneja datos no confiables que se pasan a la pantalla, como un identificador completado previamente a partir de parámetros de URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/login-id';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook proporciona información del usuario activo, incluidos `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/login-id';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useLoginId" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/LoginIdMembers">LoginIdMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `login-id`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="federatedLogin" type="Promise<void>">
  Este método inicia la autenticación a través de un proveedor de identidad federado, como Google o GitHub.

  ```jsx Example
  import { useLoginId } from '@auth0/auth0-acul-react/login-id';

  function SocialButton() {
    const { federatedLogin } = useLoginId();
    return (
      <button onClick={() => federatedLogin({ connection: 'google-oauth2' })}>
        Sign in with Google
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [FederatedLoginPayloadOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/FederatedLoginPayloadOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      El nombre de la conexión social que se usará (p. ej., `google-oauth2`, `github`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="login" type="Promise<void>">
  Este método envía el identificador del usuario para pasar al siguiente paso del flujo de autenticación.

  ```jsx Example
  import { useLoginId } from '@auth0/auth0-acul-react/login-id';

  function LoginIdForm() {
    const { login } = useLoginId();
    return (
      <button onClick={() => login({ username: 'user@example.com' })}>
        Continue
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LoginOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/LoginOptions).
    </ParamField>

    <ParamField body="username" type="string" required>
      El identificador (correo electrónico, número de teléfono o username) con el que se autenticará.
    </ParamField>

    <ParamField body="captcha?" type="string">
      Valor de captcha obligatorio cuando la detección de bots está habilitada en el tenant.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="passkeyLogin" type="Promise<void>">
  Este método inicia la autenticación passwordless mediante una clave de acceso.

  ```jsx Example
  import { useLoginId } from '@auth0/auth0-acul-react/login-id';

  function PasskeyButton() {
    const { passkeyLogin } = useLoginId();
    return (
      <button onClick={() => passkeyLogin()}>
        Sign in with Passkey
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [CustomOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/CustomOptions).
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="pickCountryCode" type="Promise<void>">
  Este método dirige al selector de código de país para los flujos de inicio de sesión por teléfono.

  ```jsx Example
  import { useLoginId } from '@auth0/auth0-acul-react/login-id';

  function PhoneLogin() {
    const { pickCountryCode } = useLoginId();
    return (
      <button onClick={() => pickCountryCode()}>
        Select Country Code
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [CustomOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/CustomOptions).
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<div id="commonutility-hooks">
  ## Hooks comunes y de utilidad
</div>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">
  Este hook obtiene las opciones del tema actual con la configuración simplificada del contexto de branding.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Este hook devuelve una función para cambiar el idioma que se muestra en la pantalla actual de ACUL.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Este hook obtiene el contexto y el estado de la pantalla actual.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Este hook lee y gestiona los errores del servidor, del cliente y del desarrollador en la pantalla.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useLoginIdentifiers">useLoginIdentifiers</a>} type="Hooks">
  Este hook devuelve una lista de los tipos de identificador activos (`email`, `phone`, `username`) en el flujo actual.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/usePasskeyAutofill">usePasskeyAutofill</a>} type="Hooks">
  Este hook habilita el autocompletado con clave de acceso en un campo de entrada para ofrecer una experiencia de inicio de sesión passwordless fluida.
</ParamField>
