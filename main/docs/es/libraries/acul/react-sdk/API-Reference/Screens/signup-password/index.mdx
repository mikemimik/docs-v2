title: "SignupPassword"
description: "Describe todos los hooks y métodos disponibles para personalizar la pantalla `signup-password` de Universal Login."
---

La pantalla `signup-password` recopila la contraseña del usuario para completar la creación de la cuenta. Es el segundo paso de un flujo de registro de varios pasos, después de recopilar el identificador. También admite el registro federado mediante conexiones sociales o empresariales.

<div id="import">
  ## Importación
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite la **importación parcial** y la **importación desde la raíz** para cada pantalla.

* La importación parcial te permite incluir solo el código que necesitas para tu caso de uso específico.
* La importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo que resulta útil cuando quieres una compilación única que gestione todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useSignupPassword } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useSignupPassword,
  // Hooks de contexto
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks de utilidad
  useChangeLanguage,
  usePasswordValidation,
  // Métodos
  signup,
  federatedSignup,
  switchConnection,
} from '@auth0/auth0-acul-react/signup-password';

function SignupPasswordForm() {
  const { signup } = useSignupPassword();
  return (
    <button onClick={() => signup({ password: 'secret' })}>
      Create Account
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexto
</div>

Hooks con alcance de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `signup-password`. Impórtalos desde `@auth0/auth0-acul-react/signup-password`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona configuraciones de branding, como el logotipo, los colores y los ajustes del tema que se muestran en la pantalla `signup-password`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/signup-password';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `signup-password`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/signup-password';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la Organización del usuario si el registro está limitado a la Organización. Devuelve `null` cuando no hay contexto de la organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/signup-password';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual del flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/signup-password';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnSignupPassword">ScreenMembersOnSignupPassword</a></span>}>
  Este hook contiene detalles específicos de la pantalla `signup-password`, incluida su configuración y su contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/signup-password';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y los metadatos asociados.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/signup-password';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembersOnSignupPassword">TransactionMembersOnSignupPassword</a></span>}>
  Este hook proporciona datos específicos de la transacción para la pantalla `signup-password`, como las conexiones activas y el estado actual del flujo.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/signup-password';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Este hook gestiona datos no confiables que se pasan a la pantalla, como un correo electrónico o nombre de usuario rellenado previamente a partir de parámetros de URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/signup-password';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook proporciona información del usuario activo, incluidos `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/signup-password';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useSignupPassword" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/SignupPasswordMembers">SignupPasswordMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `signup-password`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="federatedSignup" type="Promise<void>">
  Este método inicia el registro mediante un proveedor de identidad federado, como Google o GitHub.

  ```jsx Example
  import { useSignupPassword } from '@auth0/auth0-acul-react/signup-password';

  function SocialButton() {
    const { federatedSignup } = useSignupPassword();
    return (
      <button onClick={() => federatedSignup({ connection: 'google-oauth2' })}>
        Sign up with Google
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [FederatedSignupPasswordPayloadOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/FederatedSignupPasswordPayloadOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      El nombre de la conexión social o empresarial que se va a usar (p. ej., `google-oauth2`, `github`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="signup" type="Promise<void>">
  Este método envía la contraseña del usuario para completar la creación de la cuenta.

  ```jsx Example
  import { useSignupPassword } from '@auth0/auth0-acul-react/signup-password';

  function SignupPasswordForm() {
    const { signup } = useSignupPassword();
    return (
      <button onClick={() => signup({ password: 'secret' })}>
        Create Account
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [SignupPasswordOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/SignupPasswordOptions).
    </ParamField>

    <ParamField body="password" type="string" required>
      La contraseña de la nueva cuenta.
    </ParamField>

    <ParamField body="email" type="string">
      La dirección de correo electrónico del usuario.
    </ParamField>

    <ParamField body="username" type="string">
      El nombre de usuario de la nueva cuenta, si la conexión lo requiere.
    </ParamField>

    <ParamField body="phoneNumber" type="string">
      El número de teléfono del usuario, si la conexión lo requiere.
    </ParamField>

    <ParamField body="captcha?" type="string">
      Valor de captcha requerido cuando la detección de bots está habilitada en el tenant.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="switchConnection" type="Promise<void>">
  Este método cambia la conexión activa durante el paso signup-password, lo que permite al usuario autenticarse mediante una conexión distinta.

  ```jsx Example
  import { useSignupPassword } from '@auth0/auth0-acul-react/signup-password';

  function SwitchConnection() {
    const { switchConnection } = useSignupPassword();
    return (
      <button onClick={() => switchConnection({ connection: 'Username-Password-Authentication' })}>
        Use a different method
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [SwitchConnectionOptionsSignupPassword](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/SwitchConnectionOptionsSignupPassword).
    </ParamField>

    <ParamField body="connection" type="string" required>
      El nombre de la conexión a la que se cambiará. Usa &#39;email&#39; o &#39;sms&#39; para passwordless, o el nombre de una conexión de base de datos para la autenticación basada en contraseña.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<div id="commonutility-hooks">
  ## Hooks comunes y de utilidad
</div>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">
  Este hook obtiene las opciones del tema actual con la configuración plana a partir del contexto de branding.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Este hook devuelve una función para cambiar el idioma que se muestra en la pantalla actual de ACUL.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Este hook obtiene el contexto y el estado de la pantalla actual.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Este hook lee y gestiona los errores del servidor, del cliente y de desarrollo en la pantalla.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/usePasswordValidation">usePasswordValidation</a>} type="Hooks">
  Este hook valida la contraseña del usuario según los requisitos de la política de contraseñas del tenant.
</ParamField>
