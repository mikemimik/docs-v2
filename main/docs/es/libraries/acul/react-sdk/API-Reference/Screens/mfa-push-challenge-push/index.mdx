La pantalla `mfa-push-challenge-push` pide al usuario que apruebe una notificación push de MFA enviada a su dispositivo móvil.

<div id="import">
  ## Importación
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite **importación parcial** e **importación desde la raíz** para cada pantalla.

* Usar la importación parcial te permite incluir solo el código que necesitas para tu caso de uso.
* Usar la importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo cual es útil cuando quieres una compilación unificada para gestionar todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useMfaPushChallengePush } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useMfaPushChallengePush,
  // Hooks de contexto
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  useMfaPolling,
  // Hooks de utilidad
  useChangeLanguage,
  // Métodos
  continueMethod,
  enterCodeManually,
  resendPushNotification,
  tryAnotherMethod,
} from "@auth0/auth0-acul-react/mfa-push-challenge-push";

function MfaPushChallengePushScreen() {
  const { continueMethod } = useMfaPushChallengePush();
  return (
    <button onClick={() => continueMethod({})}>
      I've Approved
    </button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks con alcance de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `mfa-push-challenge-push`. Impórtalos desde `@auth0/auth0-acul-react/mfa-push-challenge-push`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona configuraciones de branding, como el logotipo, los colores y la configuración del tema que se muestran en la pantalla `mfa-push-challenge-push`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `mfa-push-challenge-push`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la organización del usuario si el flujo de MFA tiene alcance de organización. Devuelve `null` cuando no hay contexto de organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>Sin contexto de organización</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual en el flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnMfaPushChallengePush">ScreenMembersOnMfaPushChallengePush</a></span>}>
  Este hook contiene detalles específicos de la pantalla `mfa-push-challenge-push`, incluida su configuración y contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y los metadatos asociados.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Este hook proporciona datos específicos de la transacción para la pantalla `mfa-push-challenge-push`, como el estado actual del flujo de MFA.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembersOnMfaPushChallengePush">UntrustedDataMembersOnMfaPushChallengePush</a></span>}>
  Este hook gestiona los datos no confiables que se pasan a la pantalla, como los valores rellenados previamente a partir de parámetros de URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook proporciona información del usuario activo, incluidos `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/mfa-push-challenge-push';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useMfaPushChallengePush" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPushChallengePushMembers">MfaPushChallengePushMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `mfa-push-challenge-push`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="continueMethod" type="Promise<void>">
  Este método completa el desafío push después de que el usuario apruebe la notificación en su dispositivo móvil.

  ```jsx Example
  import { useMfaPushChallengePush } from '@auth0/auth0-acul-react/mfa-push-challenge-push';

  function ContinueButton() {
    const { continueMethod } = useMfaPushChallengePush();
    return (
      <button onClick={() => continueMethod({ rememberDevice: true })}>
        I've Approved
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [WithRememberOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/WithRememberOptions).
    </ParamField>

    <ParamField body="rememberDevice?" type="boolean">
      Si es `true`, se recordará el dispositivo y no se volverá a requerir MFA en él.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="enterCodeManually" type="Promise<void>">
  Este método lleva a la pantalla de introducción manual de código para que el usuario pueda ingresar un código OTP en lugar de usar la notificación push.

  ```jsx Example
  import { useMfaPushChallengePush } from '@auth0/auth0-acul-react/mfa-push-challenge-push';

  function EnterCodeButton() {
    const { enterCodeManually } = useMfaPushChallengePush();
    return (
      <button onClick={() => enterCodeManually()}>
        Enter Code Manually
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="resendPushNotification" type="Promise<void>">
  Este método reenvía la notificación push al dispositivo móvil del usuario.

  ```jsx Example
  import { useMfaPushChallengePush } from '@auth0/auth0-acul-react/mfa-push-challenge-push';

  function ResendButton() {
    const { resendPushNotification } = useMfaPushChallengePush();
    return (
      <button onClick={() => resendPushNotification({})}>
        Resend Notification
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [WithRememberOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/WithRememberOptions).
    </ParamField>

    <ParamField body="rememberDevice?" type="boolean">
      Si es `true`, se recordará el dispositivo y no se volverá a requerir MFA en él.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Este método lleva a la pantalla de selección del método MFA para que el usuario pueda elegir un factor de autenticación distinto.

  ```jsx Example
  import { useMfaPushChallengePush } from '@auth0/auth0-acul-react/mfa-push-challenge-push';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useMfaPushChallengePush();
    return (
      <button onClick={() => tryAnotherMethod()}>
        Try Another Method
      </button>
    );
  }
  ```
</ParamField>

<div id="commonutility-hooks">
  ## Hooks comunes/de utilidad
</div>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useMfaPolling">useMfaPolling</a>} type="Hooks">
  Este hook gestiona el estado de polling de las pantallas que esperan una respuesta asíncrona a una notificación push.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">
  Este hook obtiene las opciones del tema actual con la configuración simplificada del contexto de branding.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Este hook devuelve una función para cambiar el idioma de visualización en la pantalla actual de ACUL.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Este hook obtiene el contexto y el estado de la pantalla actual.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Este hook lee y gestiona los errores del servidor, del cliente y del desarrollador en la pantalla.
</ParamField>
