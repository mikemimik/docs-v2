La pantalla `device-code-confirmation` se muestra durante el Device Authorization Flow. Muestra al usuario un código para confirmar y le solicita que apruebe o rechace la solicitud de activación del dispositivo.

<div id="import">
  ## Importación
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite la **importación parcial** y la **importación desde la raíz** para cada pantalla.

* La importación parcial te permite incluir solo el código que necesitas para tu caso de uso.
* La importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo que resulta útil si quieres una sola compilación capaz de gestionar todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useDeviceCodeConfirmation } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useDeviceCodeConfirmation,
  // Hooks de contexto
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks de utilidad
  useChangeLanguage,
  // Métodos
  confirm,
  cancel,
} from '@auth0/auth0-acul-react/device-code-confirmation';

function DeviceCodeConfirmationScreen() {
  const { confirm, cancel } = useDeviceCodeConfirmation();
  return (
    <>
      <button onClick={() => confirm()}>Confirm</button>
      <button onClick={() => cancel()}>Cancel</button>
    </>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks con alcance de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `device-code-confirmation`. Impórtalos desde `@auth0/auth0-acul-react/device-code-confirmation`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona configuraciones de branding, como el logotipo, los colores y la configuración del tema que se muestran en la pantalla `device-code-confirmation`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/device-code-confirmation';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `device-code-confirmation`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/device-code-confirmation';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la organización del usuario si la confirmación del dispositivo tiene alcance de organización. Devuelve `null` cuando no hay contexto de organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/device-code-confirmation';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No hay contexto de organización</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual en el flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/device-code-confirmation';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnDeviceCodeConfirmation">ScreenMembersOnDeviceCodeConfirmation</a></span>}>
  Este hook contiene detalles específicos de la pantalla `device-code-confirmation`, incluida su configuración y contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/device-code-confirmation';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y los metadatos asociados.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/device-code-confirmation';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Este hook proporciona datos específicos de la transacción para la pantalla `device-code-confirmation`, como las conexiones activas y el estado actual del flujo.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/device-code-confirmation';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Este hook administra datos no confiables que se pasan a la pantalla, como un código de dispositivo rellenado previamente a partir de parámetros de URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/device-code-confirmation';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook ofrece detalles del usuario activo, incluidos `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/device-code-confirmation';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useDeviceCodeConfirmation" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/DeviceCodeConfirmationMembers">DeviceCodeConfirmationMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `device-code-confirmation`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="cancel" type="Promise<void>">
  Este método cancela el flujo de código de dispositivo y rechaza la solicitud de activación.

  ```jsx Example
  import { useDeviceCodeConfirmation } from '@auth0/auth0-acul-react/device-code-confirmation';

  function CancelButton() {
    const { cancel } = useDeviceCodeConfirmation();
    return (
      <button onClick={() => cancel()}>
        Cancel
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="confirm" type="Promise<void>">
  Este método confirma el código del dispositivo y aprueba la solicitud de activación.

  ```jsx Example
  import { useDeviceCodeConfirmation } from '@auth0/auth0-acul-react/device-code-confirmation';

  function ConfirmButton() {
    const { confirm } = useDeviceCodeConfirmation();
    return (
      <button onClick={() => confirm()}>
        Confirm
      </button>
    );
  }
  ```
</ParamField>
