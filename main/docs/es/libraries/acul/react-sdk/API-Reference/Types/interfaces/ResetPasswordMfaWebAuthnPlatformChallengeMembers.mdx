<ParamField body="continueWithPasskey" type="Promise<void>">
  Inicia el desafío del autenticador de plataforma WebAuthn llamando a `navigator.credentials.get()`
  con las opciones del desafío proporcionadas en `screen.publicKey`.
  Si se completa correctamente, envía la `assertion` de credencial resultante a Auth0 con `action: "default"`.

  Una promesa que se resuelve cuando se envía el intento de verificación.
  Normalmente, una operación correcta da lugar a una redirección.

  #### Lanza

  Lanza un error si falta `screen.publicKey`,
  si `navigator.credentials.get()` falla (por ejemplo, si el usuario cancela con `NotAllowedError`),
  o si falla el envío del formulario a Auth0. Si `navigator.credentials.get()` falla
  con una `DOMException`, se recomienda capturar ese error y llamar a `reportBrowserError`.

  ```typescript Example
  // Suponiendo que 'sdk' es una instancia de ResetPasswordMfaWebAuthnPlatformChallenge
  try {
    await sdk.continueWithPasskey({
      rememberDevice: true // si el usuario marcó la casilla y sdk.screen.showRememberDevice es true
    });
    // Si tiene éxito, Auth0 se encarga de la redirección.
  } catch (error) {
    console.error("Platform authenticator verification failed:", error);
    // Si es un error de la API de WebAuthn (DOMException), infórmalo
    if (error instanceof DOMException && error.name && error.message) {
      await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
    }
    // Revisa sdk.transaction.errors para ver los mensajes de validación del servidor si la página se recarga.
  }
  ```

<ParamField body="reportBrowserError" type="Promise<void>">
  Notifica un error del navegador ocurrido durante la llamada a la API `navigator.credentials.get()`.
  Esto se usa para informar a Auth0 sobre problemas como la cancelación del usuario, el tiempo de espera o hardware no compatible.
  Envía los detalles del error con `action: "showError::{errorDetailsJsonString}"`.

  Una promesa que se resuelve cuando se envía el informe del error.

  #### Lanza

  Si falla el envío del formulario (por ejemplo, por un error de red).

  ```typescript Example
  // Suponiendo que 'sdk' es una instancia de ResetPasswordMfaWebAuthnPlatformChallenge
  // y que 'webAuthnError' es una DOMException de una llamada fallida a navigator.credentials.get().
  try {
    await sdk.reportBrowserError({
      error: { name: webAuthnError.name, message: webAuthnError.message }
    });
    // Auth0 puede renderizar de nuevo la página o redirigir según el error.
  } catch (submitError) {
    console.error("Failed to report WebAuthn browser error:", submitError);
  }
  ```

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Permite al usuario omitir el desafío de plataforma de WebAuthn y seleccionar otro método MFA.
  Esta acción envía `action: "pick-authenticator"` a Auth0, lo que debería llevar
  al usuario a una pantalla de selección de factor MFA para el flujo de restablecimiento de contraseña.

  Una promesa que se resuelve cuando se envía la acción &#39;pick-authenticator&#39;.

  #### Lanza

  Si falla el envío del formulario.

  ```typescript Example
  // Suponiendo que 'sdk' es una instancia de ResetPasswordMfaWebAuthnPlatformChallenge
  try {
    await sdk.tryAnotherMethod();
    // Si se realiza correctamente, Auth0 gestiona la redirección a la selección de MFA.
  } catch (error) {
    console.error("Failed to switch MFA method:", error);
  }
  ```
