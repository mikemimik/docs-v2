<ParamField body="continue" type="Promise<void>">
  Envía el código de verificación al teléfono del usuario mediante el método especificado (SMS o voz).
  Esta acción corresponde a cuando el usuario hace clic en el botón &quot;Enviar código por SMS&quot; o &quot;Enviar código por llamada de voz&quot;.
  Activa el backend de Auth0 para iniciar el proceso de entrega del código.

  Una promesa que se resuelve cuando la acción se envía correctamente al servidor.

  #### Throws

  Lanza un error si el envío falla (por ejemplo, por un problema de red o un estado no válido).

  ```typescript Example
  import ResetPasswordMfaPhoneChallenge from '@auth0/auth0-acul-js/reset-password-mfa-phone-challenge';

  const screen = new ResetPasswordMfaPhoneChallenge();

  // Enviar código por SMS
  await screen.continue({ type: 'sms' });

  // Enviar código por llamada de voz
  await screen.continue({ type: 'voice' });
  ```

  <Expandable title="Parámetros">
    <ParamField body="payload" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaPhoneChallengeContinueOptions">ResetPasswordMfaPhoneChallengeContinueOptions</a></span>}>
      Las opciones que especifican el tipo de entrega (`sms` o `voice`) y cualquier dato personalizado.

      **Propiedades**

      <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">
        El método de entrega seleccionado por el usuario para recibir el código de verificación.

        * `sms`: Envía el código por mensaje de texto.
        * `voice`: Envía el código mediante una llamada de voz.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Lleva al usuario a la pantalla donde puede seleccionar un método de MFA alternativo.
  Esta acción corresponde a cuando el usuario hace clic en el botón &quot;Try Another Method&quot;.

  Una promesa que se resuelve cuando la acción se envía correctamente al servidor.

  #### Throws

  Lanza un error si el envío falla.

  ```typescript Example
  import ResetPasswordMfaPhoneChallenge from '@auth0/auth0-acul-js/reset-password-mfa-phone-challenge';

  const screen = new ResetPasswordMfaPhoneChallenge();

  // Si al usuario se le mostró inicialmente la opción de SMS:
  await screen.tryAnotherMethod({ type: 'sms' });

  // Si al usuario se le mostró inicialmente la opción de voz:
  await screen.tryAnotherMethod({ type: 'voice' });
  ```

  <Expandable title="Parámetros">
    <ParamField body="payload" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions">ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions</a></span>}>
      Las opciones que especifican el tipo de desafío actual (`sms` o `voice`) y cualquier dato personalizado. El campo `type` es necesario para indicar al backend de qué flujo se está yendo el usuario.

      **Propiedades**

      <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">
        El método de entrega que se mostró o seleccionó inicialmente en la pantalla actual.
        La API endpoint lo requiere para procesar correctamente la acción `pick-authenticator`.

        * `sms`: Indica que la pantalla actual era para un desafío por SMS.
        * `voice`: Indica que la pantalla actual era para un desafío por voz.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>
