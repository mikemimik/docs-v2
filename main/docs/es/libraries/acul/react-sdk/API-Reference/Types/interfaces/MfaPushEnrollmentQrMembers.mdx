      <ParamField body="MfaPollingOptions" type="object">
        Opciones de configuración para gestionar un proceso de polling de notificaciones push de MFA.

        Use estas opciones al iniciar una sesión de polling para controlar con qué frecuencia
        se consulta el endpoint del desafío y cómo se manejan los casos de éxito o error.

        <Expandable title="propiedades">
          <ParamField body="intervalMs?" type="number">
            Intervalo opcional, en **milisegundos**, entre solicitudes de polling consecutivas.

            * Si se omite, se usa el intervalo predeterminado interno del SDK (normalmente, 5000 ms).
            * Elija un valor que equilibre la capacidad de respuesta con las consideraciones sobre límites de tasa.
          </ParamField>

          <ParamField body="onCompleted()?" type="void">
            Función de callback opcional que se ejecuta una vez que se aprueba correctamente
            el desafío push de MFA y finaliza el polling.

            * Se llama exactamente **una vez**, tras lo cual el polling se detiene automáticamente.
            * Útil para avanzar en el flujo de login (p. ej., llamando a `continue()`).
          </ParamField>

          <ParamField body="onError()?" type="void">
            Función de callback opcional que se invoca si se produce un error durante el polling.

            Recibe un objeto [ULError](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) que contiene:

            * `status` — el código de estado HTTP de la solicitud fallida

            * `responseText` — el cuerpo de la respuesta sin procesar, si está disponible

            * Se llama una vez por cada evento de error; el polling puede reintentarse según el tipo de error.

            * Úselo para mostrar mensajes de error o activar lógica personalizada de reintento o cancelación.
