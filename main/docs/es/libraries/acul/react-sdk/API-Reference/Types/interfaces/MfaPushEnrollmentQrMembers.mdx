      <ParamField body="MfaPollingOptions" type="object">
        Opciones de configuración para gestionar un proceso de sondeo de notificaciones push de MFA.

        Proporcione estas opciones al iniciar una sesión de sondeo para controlar la frecuencia
        con la que se consulta el endpoint del desafío y cómo se gestionan los casos de éxito o error.

        <Expandable title="propiedades">
          <ParamField body="intervalMs?" type="number">
            Intervalo opcional, en **milisegundos**, entre solicitudes de sondeo consecutivas.

            * Si se omite, se utiliza el intervalo predeterminado interno del SDK (normalmente, 5000 ms).
            * Elija un valor que equilibre la capacidad de respuesta con las limitaciones de frecuencia.
          </ParamField>

          <ParamField body="onCompleted()?" type="void">
            Función callback opcional que se ejecuta cuando el desafío push de MFA se aprueba correctamente
            y el sondeo finaliza.

            * Se llama exactamente **una vez**, tras lo cual el sondeo se detiene automáticamente.
            * Resulta útil para avanzar en el flujo de inicio de sesión (por ejemplo, llamando a `continue()`).
          </ParamField>

          <ParamField body="onError()?" type="void">
            Función callback opcional que se invoca si se produce un error durante el sondeo.

            Recibe un objeto [ULError](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) que contiene:

            * `status` — el código de estado HTTP de la solicitud fallida

            * `responseText` — el cuerpo de la respuesta sin procesar, si está disponible

            * Se llama una vez por cada evento de error; el sondeo puede reintentarse según el tipo de error.

            * Úsela para mostrar mensajes de error o activar lógica personalizada de reintento o cancelación.
