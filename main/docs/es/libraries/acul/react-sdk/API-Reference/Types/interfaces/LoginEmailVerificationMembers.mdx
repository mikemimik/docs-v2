LoginEmailVerificationMembers
description Define los miembros (propiedades y métodos) disponibles para interactuar con la pantalla de verificación del correo electrónico durante el inicio de sesión.
Esta pantalla es una parte fundamental de los procesos de verificación del correo electrónico, normalmente durante el inicio de sesión, en los que el usuario debe demostrar
que controla una dirección de correo electrónico proporcionando un código de un solo uso. El SDK facilita el envío de este código
o la solicitud de uno nuevo.

El `universal_login_context` para esta pantalla (`window.universal_login_context`) contendrá:

* `client`: Información sobre la aplicación de Auth0.
* `organization` (opcional): Detalles si la autenticación es para una organización específica.
* `prompt`: Contexto del prompt de autenticación actual (por ejemplo, &#39;login&#39;).
* `screen`: Textos de la UI e información general de la pantalla. No se definen campos `data` específicos de esta pantalla para `login-email-verification` más allá de los estándar.
* `transaction`: Detalles de la transacción en curso, incluido el estado y cualquier error de intentos anteriores (por ejemplo, &quot;invalid-code&quot;).

````ts Example
export interface LoginEmailVerificationMembers extends BaseMembers {
  /**
   * Envía el código de verificación de correo electrónico introducido por el usuario.
   * Esta acción corresponde al usuario introduciendo el código que recibió por correo electrónico y
   * haciendo clic en el botón "Continuar" o "Verificar". El SDK enviará entonces este código mediante POST
   * al endpoint `/u/login-email-verification` de Auth0.
   *
   * Si el código es válido, Auth0 normalmente redirigirá al usuario al siguiente paso en el
   * flujo de autenticación. Si el código no es válido, ha expirado o se produce otro error,
   * Auth0 generalmente volverá a renderizar la pantalla login-email-verification, y el
   * arreglo `transaction.errors` en el contexto del SDK se actualizará con detalles
   * sobre el fallo (p. ej., código de error `invalid-code`).
   *
   * @param {ContinueWithCodeOptions} payload - Un objeto que contiene la cadena `code` introducida por el usuario.
   *                                            También puede incluir cualquier `CustomOptions` para la extensibilidad.
   * @returns {Promise<void>} Una promesa que se resuelve cuando se inicia el envío del formulario.
   *                          No devuelve datos directamente al resolverse, ya que una redirección o
   *                          un re-renderizado de la página es el resultado habitual.
   * @throws {Error} Lanza un error si `payload.code` no se proporciona o no es una cadena de texto,
   *                 o si el `FormHandler` encuentra un problema irrecuperable durante el envío (p. ej., error de red).
   *                 Los errores de validación de Auth0 (como un código inválido) no se lanzan como errores de JavaScript,
   *                 sino que se reflejan en `this.transaction.errors` tras la operación.
   *
   * @example
   * ```typescript
   * // Se asume que 'manager' es una instancia de LoginEmailVerification
   * const userInputCode = "123456";
   * try {
   *   await manager.continueWithCode({ code: userInputCode });
   *   // Si tiene éxito, la página redirige. No se requiere ninguna acción adicional del lado del cliente aquí.
   * } catch (error) {
   *   // Este catch es para errores inesperados, no para errores de validación de Auth0.
   *   console.error("Failed to submit the verification code:", error);
   * }
   * // Tras el await, comprueba siempre manager.transaction.errors para detectar problemas de validación del lado del servidor.
   * if (manager.transaction.errors && manager.transaction.errors.length > 0) {
   *   manager.transaction.errors.forEach(err => {
   *     if (err.code === 'invalid-code') {
   *       // Mostrar "El código que introdujiste no es válido" al usuario.
   *     }
   *   });
   * }
   * ```
   */
  continueWithCode(payload: ContinueWithCodeOptions): Promise<void>;

  /**
   * Solicita que se envíe un nuevo código de verificación a la dirección de correo electrónico del usuario.
   * Esta acción se invoca normalmente cuando el usuario hace clic en el botón "Reenviar código", quizás porque
   * no recibió el primer correo, el código expiró o sospecha que hay algún problema.
   * El SDK enviará mediante POST al endpoint `/u/login-email-verification` de Auth0 con una acción que indica
   * una solicitud de reenvío.
   *
   * Tras el envío exitoso de esta solicitud, Auth0 intenta enviar un nuevo correo electrónico.
   * La página puede volver a renderizarse. Si hay problemas (p. ej., demasiados intentos de reenvío para el mismo correo,
   * identificados por el código de error `too-many-emails`), `transaction.errors` se actualizará.
   *
   * @param {ResendCodeOptions} [payload] - Opcional. Un objeto para `CustomOptions` si se necesita enviar
   *                                        algún parámetro adicional con la solicitud.
   * @returns {Promise<void>} Una promesa que se resuelve cuando se inicia la solicitud de reenvío.
   *                          Al igual que `continueWithCode`, esto generalmente resulta en un re-renderizado de la página o una actualización de estado,
   *                          en lugar de datos directos en la resolución de la promesa.
   * @throws {Error} Lanza un error si el `FormHandler` encuentra un problema irrecuperable (p. ej., error de red).
   *                 Los errores del lado del servidor (como los límites de frecuencia) se reflejan en `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // Se asume que 'manager' es una instancia de LoginEmailVerification
   * try {
   *   await manager.resendCode();
   *   // Opcionalmente, actualiza la UI para informar al usuario de que se ha enviado un nuevo código.
   *   alert("A new verification code has been dispatched to your email.");
   * } catch (error) {
   *   console.error("Failed to request a new code:", error);
   * }
   * // Tras el await, comprueba manager.transaction.errors para detectar problemas del lado del servidor.
   * if (manager.transaction.errors && manager.transaction.errors.length > 0) {
   *   manager.transaction.errors.forEach(err => {
   *     if (err.code === 'too-many-emails') {
   *       // Mostrar "Has solicitado demasiados correos. Por favor, espera unos minutos."
   *     }
   *   });
   * }
   * ```
   */
  resendCode(payload?: ResendCodeOptions): Promise<void>;

  /**
   * Obtiene la funcionalidad de reenvío con gestión del tiempo de espera para esta pantalla
   * @param options Opciones de configuración para la funcionalidad de reenvío
   */
  resendManager(options?: StartResendOptions): ResendControl;
}
````

<ParamField body="continueWithCode" type="Promise<void>">
  Envía el código de verificación por correo electrónico introducido por el usuario.
  Esta acción corresponde a cuando el usuario introduce el código que recibió por correo electrónico y
  hace clic en un botón &quot;Continuar&quot; o &quot;Verificar&quot;. Después, el SDK enviará este código mediante POST
  al endpoint `/u/login-email-verification` de Auth0.

  Si el código es válido, Auth0 normalmente redirigirá al usuario al siguiente paso del
  flujo de autenticación. Si el código no es válido, ha caducado o se produce otro error,
  Auth0 normalmente volverá a renderizar la pantalla login-email-verification, y el
  arreglo `transaction.errors` en el contexto del SDK se actualizará con detalles
  sobre el error (por ejemplo, el código de error `invalid-code`).

  Una promesa que se resuelve cuando se inicia el envío del formulario.
  No devuelve datos directamente al resolverse, ya que una redirección o un
  nuevo renderizado de la página es el resultado habitual.

  #### Lanza

  Lanza un error si no se proporciona `payload.code` o si no es una cadena,
  o si `FormHandler` encuentra un problema irrecuperable durante el envío (por ejemplo, un error de red).
  Los errores de validación de Auth0 (como un código no válido) no se lanzan como errores de JavaScript,
  sino que se reflejan en `this.transaction.errors` después de la operación.

  ```typescript Example
  // Suponiendo que 'manager' es una instancia de LoginEmailVerification
  const userInputCode = "123456";
  try {
    await manager.continueWithCode({ code: userInputCode });
    // Si la operación se realiza correctamente, la página redirige. No se necesita ninguna otra acción del lado del cliente aquí.
  } catch (error) {
    // Este catch es para errores inesperados, no para errores de validación de Auth0.
    console.error("Error al enviar el código de verificación:", error);
  }
  // Después del await, comprueba siempre manager.transaction.errors para detectar problemas de validación del lado del servidor.
  if (manager.transaction.errors && manager.transaction.errors.length > 0) {
    manager.transaction.errors.forEach(err => {
      if (err.code === 'invalid-code') {
        // Muestra "El código que introdujiste no es válido" al usuario.
      }
    });
  }
  ```

<ParamField body="resendCode" type="Promise<void>">
  Solicita que se envíe un nuevo código de verificación a la dirección de correo electrónico del usuario.
  Esta acción suele invocarse cuando el usuario hace clic en un botón &quot;Reenviar código&quot;, quizá porque
  no recibió el primer correo, el código caducó o sospecha que hay alguna incidencia.
  El SDK hará un POST al endpoint `/u/login-email-verification` de Auth0 con una acción que indica
  una solicitud de reenvío.

  Cuando esta solicitud se envía correctamente, Auth0 intenta enviar un nuevo correo electrónico.
  La página puede volver a renderizarse. Si hay problemas (p. ej., demasiados intentos de reenvío para el mismo correo electrónico,
  identificados por el código de error `too-many-emails`), se actualizará `transaction.errors`.

  Una promesa que se resuelve cuando se inicia la solicitud de reenvío.
  Al igual que `continueWithCode`, esto normalmente da lugar a un nuevo renderizado de la página o a una actualización del estado,
  en lugar de devolver datos directos en la resolución de la promesa.

  #### Throws

  Lanza un error si `FormHandler` encuentra un problema irrecuperable (p. ej., un error de red).
  Los errores del lado del servidor (como los límites de frecuencia) se reflejan en `this.transaction.errors`.

  ```typescript Example
  // Suponiendo que 'manager' es una instancia de LoginEmailVerification
  try {
    await manager.resendCode();
    // Opcionalmente, actualiza la UI para informar al usuario de que se ha enviado un nuevo código.
    alert("A new verification code has been dispatched to your email.");
  } catch (error) {
    console.error("Failed to request a new code:", error);
  }
  // Después del await, comprueba manager.transaction.errors para ver si hay problemas del servidor.
  if (manager.transaction.errors && manager.transaction.errors.length > 0) {
    manager.transaction.errors.forEach(err => {
      if (err.code === 'too-many-emails') {
        // Muestra "Has solicitado demasiados correos electrónicos. Espera unos minutos."
      }
    });
  }
  ```
