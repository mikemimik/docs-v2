---
title: "ErrorItem"
---

Representa una entrada de error individual en el almacén de errores de ACUL. [`useErrors`](/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useErrors) lo devuelve como parte del objeto [`UseErrorsResult`](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult). Extiende la interfaz base `Error` con un `id`, `type` y un `label` opcional.

```typescript
interface ErrorItem {
  id: string;
  code: string;
  message: string;
  type?: ErrorType;
  field?: string;
  label?: string;
  rules?: PasswordComplexityRule[];
}
```

<div id="properties">
  ## Propiedades
</div>

<ParamField body="id" type="string" required>
  Un identificador único de esta instancia de error. Se usa para apuntar a un error específico con `dismiss(id)`.
</ParamField>

<ParamField body="code" type="string" required>
  Un código de error legible por máquina que identifica el tipo de error. Ejemplo: `"invalid_password"`, `"username_exists"`.
</ParamField>

<ParamField body="message" type="string" required>
  Una descripción del error legible para las personas, adecuada para mostrarse en la UI.
</ParamField>

<ParamField body="type" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/type-aliases/ErrorType">ErrorType</a>}>
  Clasifica el error como `"auth0"`, `"validation"` o `"configuration"`. Se usa para filtrar con `errors.byType()`.
</ParamField>

<ParamField body="field" type="string">
  El campo del form asociado al error, si corresponde. Ejemplo: `"username"`, `"email"`. Se usa para filtrar con `errors.byField()`.
</ParamField>

<ParamField body="label" type="string">
  Una label de visualización opcional para el error, que se usa cuando se necesita un nombre de campo más claro junto al mensaje de error.
</ParamField>

<ParamField body="rules" type={<span><a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PasswordComplexityRule">PasswordComplexityRule</a>[]</span>}>
  Reglas de complejidad de contraseña asociadas a este error cuando el error está relacionado con la validación de contraseña.
</ParamField>
