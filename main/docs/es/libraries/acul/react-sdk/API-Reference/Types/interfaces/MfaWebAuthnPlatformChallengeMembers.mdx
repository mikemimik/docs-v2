MfaWebAuthnPlatformChallengeMembers

````ts Example
export interface MfaWebAuthnPlatformChallengeMembers extends BaseMembers {
  /**
   * Propiedades y datos específicos de la pantalla, incluidas las opciones del desafío de WebAuthn y la preferencia de recordar dispositivo.
   * @type {ScreenMembersOnMfaWebAuthnPlatformChallenge}
   */
  screen: ScreenMembersOnMfaWebAuthnPlatformChallenge;

  /**
   * Inicia el desafío del autenticador de plataforma WebAuthn.
   * Este método llama internamente a `navigator.credentials.get()` usando las opciones del desafío
   * proporcionadas en `screen.publicKey`.
   * Si tiene éxito, envía la credencial resultante a Auth0 con `action: "default"`.
   *
   * Si `navigator.credentials.get()` falla (por ejemplo, por cancelación del usuario, timeout u otros errores de la API de WebAuthn),
   * se recomienda llamar a `reportBrowserError` con los detalles del error.
   *
   * @param {VerifyPlatformAuthenticatorOptions} [options] - Parámetros opcionales para el proceso de verificación,
   * como `rememberDevice` (si `screen.showRememberDevice` es true) y otras opciones personalizadas.
   * @returns {Promise<void>} Una promesa que se resuelve cuando se envía el intento de verificación.
   *                          Una operación exitosa generalmente resulta en una redirección.
   * @throws {Error} Lanza un error si `screen.publicKey` no está presente,
   *                 si `navigator.credentials.get()` falla con un error inesperado no gestionado por `getPasskeyCredentials`,
   *                 o si el envío del formulario a Auth0 falla.
   *
   * @example
   * ```typescript
   * // Suponiendo que 'sdk' es una instancia de MfaWebAuthnPlatformChallenge
   * try {
   *   const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
   *   await sdk.verify({ rememberDevice: remember });
   *   // En caso de éxito, Auth0 gestiona la redirección.
   * } catch (error) {
   *   console.error("Platform authenticator verification failed:", error);
   *   // Si es un error de la API de WebAuthn, repórtalo
   *   if (error.name && error.message) { // Verificación básica de error similar a DOMException
   *     await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
   *   }
   *   // Consulta sdk.transaction.errors para mensajes de validación del lado del servidor si la página se recarga.
   * }
   * ```
   */
  verify(options?: VerifyPlatformAuthenticatorOptions): Promise<void>;

  /**
   * Notifica a Auth0 sobre un error específico de la API de WebAuthn (proveniente de `navigator.credentials.get()`).
   * Este método debe usarse cuando `verify()` (o una llamada manual a `navigator.credentials.get()`) falla debido a
   * un problema de WebAuthn en el navegador (por ejemplo, cancelación del usuario `NotAllowedError`, timeout).
   * Envía los detalles del error con `action: "showError::{errorDetailsJsonString}"` y una `response` vacía.
   *
   * @param {ReportBrowserErrorOptions} options - Contiene el objeto `error` (con `name` y `message`
   * del DOMException de la API de WebAuthn) y cualquier otra opción personalizada.
   * @returns {Promise<void>} Una promesa que se resuelve cuando se envía el reporte del error.
   * @throws {Error} Lanza un error si el envío del formulario falla (por ejemplo, error de red, estado inválido).
   *
   * @example
   * ```typescript
   * // Suponiendo que 'sdk' es una instancia de MfaWebAuthnPlatformChallenge
   * // Y webAuthnDomException es un objeto de error de una llamada fallida a navigator.credentials.get().
   * try {
   *   await sdk.reportBrowserError({
   *     error: { name: webAuthnDomException.name, message: webAuthnDomException.message }
   *   });
   *   // Auth0 procesará este error y puede volver a renderizar la página o redirigir.
   * } catch (submitError) {
   *   console.error("Failed to report WebAuthn browser error:", submitError);
   * }
   * ```
   */
  reportBrowserError(options: ReportBrowserErrorOptions): Promise<void>;

  /**
   * Permite al usuario omitir el desafío de plataforma WebAuthn y seleccionar un método MFA diferente.
   * Esta acción envía `action: "pick-authenticator"` a Auth0, lo que debería llevar
   * al usuario a la pantalla de selección de factor MFA.
   *
   * @param {TryAnotherMethodOptions} [options] - Parámetros personalizados opcionales que se enviarán con la solicitud.
   * @returns {Promise<void>} Una promesa que se resuelve cuando se envía la acción 'pick-authenticator'.
   * @throws {Error} Lanza un error si el envío del formulario falla.
   *
   * @example
   * ```typescript
   * // Suponiendo que 'sdk' es una instancia de MfaWebAuthnPlatformChallenge
   * try {
   *   await sdk.tryAnotherMethod();
   *   // En caso de éxito, Auth0 gestiona la redirección a la selección de MFA.
   * } catch (error) {
   *   console.error("Failed to switch MFA method:", error);
   * }
   * ```
   */
  tryAnotherMethod(options?: TryAnotherMethodOptions): Promise<void>;
}

<ParamField body="verify" type="Promise<void>">
  Inicia el desafío del autenticador de plataforma de WebAuthn.
  Este método llama internamente a `navigator.credentials.get()` usando las opciones del desafío
  proporcionadas en `screen.publicKey`.
  Si se realiza correctamente, envía la credencial resultante a Auth0 con `action: "default"`.

  Si `navigator.credentials.get()` falla (p. ej., cancelación del usuario, timeout u otros errores de la API de WebAuthn),
  se recomienda llamar a `reportBrowserError` con los detalles del error.

  Una Promise que se resuelve cuando se envía el intento de verificación.
  Una operación correcta normalmente da lugar a una redirección.

  #### Lanza

  Lanza un error si falta `screen.publicKey`,
  si `navigator.credentials.get()` falla con un error inesperado no controlado por `getPasskeyCredentials`,
  o si falla el envío del formulario a Auth0.

  ```typescript Example
  // Suponiendo que 'sdk' es una instancia de MfaWebAuthnPlatformChallenge
  try {
    const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
    await sdk.verify({ rememberDevice: remember });
    // Si tiene éxito, Auth0 se encarga de la redirección.
  } catch (error) {
    console.error("Platform authenticator verification failed:", error);
    // Si es un error de la API de WebAuthn, repórtalo
    if (error.name && error.message) { // Comprobación básica para un error similar a DOMException
      await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
    }
    // Revisa sdk.transaction.errors para ver mensajes de validación del servidor si la página se recarga.
  }
  ```
