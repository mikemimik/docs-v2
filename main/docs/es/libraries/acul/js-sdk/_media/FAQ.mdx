<div id="where-can-i-see-errors-related-to-my-screen-and-how-to-handle-them">
  ## Dónde puedo ver los errores relacionados con mi pantalla y cómo gestionarlos
</div>

El SDK ofrece varias formas de detectar y ver errores durante el proceso:

1. **Método `getErrors` para errores específicos de la pantalla**
   Se puede usar una función getter de nivel superior, `getErrors`, para obtener errores de autenticación detallados cuando falla la validación del backend.
   ```javascript
      import Login from "@auth0/auth0-acul-js/login";

      const loginIdManager = new Login();

      const errors = loginIdManager.getErrors();
      if (errors) {
      console.error("Login failed:", errors);
   }
   ```
2. **Errores específicos de la pantalla**
   Cada pantalla incluye una propiedad `transaction.errors`, que devuelve una lista de errores específicos de esa pantalla. Estos errores se generan después de la validación final de la solicitud, lo que te ayuda a identificar los problemas que requieren atención.

Al usar estos métodos, puedes seguir y resolver fácilmente los errores a lo largo del flujo del SDK.

<div id="can-i-use-this-sdk-from-a-cdn">
  ## ¿Puedo usar este SDK desde una CDN?
</div>

Actualmente, este SDK solo puede instalarse a través del npm registry y no puede usarse directamente desde una CDN.

Para instalar el SDK, ejecuta el siguiente comando:

```sh
npm install @auth0/auth0-acul-js
```

Una vez instalado, puedes importarlo y usarlo en tus proyectos de JavaScript, TypeScript o ReactJS.

<div id="can-i-build-a-reactjs-application-with-this-sdk">
  ## ¿Puedo crear una aplicación de ReactJS con este SDK?
</div>

Sí, este SDK puede integrarse fácilmente en una aplicación de ReactJS.

Aunque el SDK está escrito en Vanilla JavaScript, es totalmente compatible con todos los frameworks basados en JavaScript, incluidos React, Angular, Vue y otros. Puedes usar el SDK sin problemas en tus componentes o hooks de React.
