<ParamField body="MfaPollingOptions" type="object">
  Opciones de configuración para gestionar un proceso de polling de notificaciones push de MFA.

  Proporcione estas opciones al iniciar una sesión de polling para controlar la frecuencia con la que
  se consulta el endpoint de challenge y cómo se gestionan los casos de éxito o error.

  ```ts Example
  export type MfaPollingOptions = {
    /**
     * Intervalo opcional, en **milisegundos**, entre solicitudes de polling consecutivas.
     *
     * - Si se omite, se usa el intervalo predeterminado interno del SDK (normalmente 5000 ms).
     * - Elija un valor que equilibre la capacidad de respuesta con las consideraciones de límites de tasa.
     *
     * @defaultValue Específico de la implementación (normalmente 5000 ms)
     */
    intervalMs?: number;

    /**
     * Devolución de llamada opcional que se ejecuta una vez que el MFA push challenge se aprueba
     * correctamente y el polling finaliza.
     *
     * - Se llama exactamente **una vez**, después de lo cual el polling se detiene automáticamente.
     * - Útil para avanzar en el flujo de login (por ejemplo, llamando a `continue()`).
     */
    onCompleted?: () => void;

    /**
     * Devolución de llamada opcional que se invoca si se produce un error durante el polling.
     *
     * Recibe un objeto {@link ULError} que contiene:
     * - `status` — el código de estado HTTP de la solicitud fallida
     * - `responseText` — el cuerpo de la respuesta sin procesar, si está disponible
     *
     * - Se llama una vez por cada evento de error; el polling puede reintentarse según el tipo de error.
     * - Úselo para mostrar mensajes de error o activar lógica personalizada de reintento/aborto.
     */
    onError?: (error: ULError) => void;
  };
  ```
