<ParamField body="MfaPollingOptions" type="object">
  Opciones de configuración para gestionar un proceso de consulta periódica de notificaciones push de MFA.

  Proporcione estas opciones al iniciar una sesión de consulta periódica para controlar con qué frecuencia
  se consulta el endpoint del challenge y cómo se gestionan los casos de éxito o error.

  ```ts Example
  export type MfaPollingOptions = {
    /**
     * Intervalo opcional, en **milisegundos**, entre solicitudes consecutivas de consulta periódica.
     *
     * - Si se omite, se usa el intervalo predeterminado interno del SDK (normalmente 5000 ms).
     * - Elija un valor que equilibre la capacidad de respuesta con las limitaciones de tasa.
     *
     * @defaultValue Específico de la implementación (normalmente 5000 ms)
     */
    intervalMs?: number;

    /**
     * Devolución de llamada opcional que se ejecuta una vez que el MFA push challenge
     * se aprueba correctamente y finaliza la consulta periódica.
     *
     * - Se llama exactamente **una vez**, tras lo cual la consulta periódica se detiene automáticamente.
     * - Útil para avanzar en el flujo de inicio de sesión (por ejemplo, llamando a `continue()`).
     */
    onCompleted?: () => void;

    /**
     * Devolución de llamada opcional que se invoca si se produce un error durante la consulta periódica.
     *
     * Recibe un objeto {@link ULError} que contiene:
     * - `status` — el código de estado HTTP de la solicitud fallida
     * - `responseText` — el cuerpo de la respuesta sin procesar, si está disponible
     *
     * - Se llama una vez por cada evento de error; la consulta periódica puede reintentarse según el tipo de error.
     * - Úsela para mostrar mensajes de error o activar lógica personalizada de reintento o cancelación.
     */
    onError?: (error: ULError) => void;
  };
  ```

      * Úsela para mostrar mensajes de error o activar lógica personalizada de reintento o cancelación.
