CustomizedConsentMembers

````ts Example
export interface CustomizedConsentMembers extends BaseMembers {
  /**
   * Proporciona acceso a las propiedades y datos específicos de la pantalla consentimiento personalizado,
   * incluida la lista de `scopes` y `authorizationDetails` que se están solicitando.
   * @type {ScreenMembersOnCustomizedConsent}
   */
  screen: ScreenMembersOnCustomizedConsent;

  /**
   * Envía la decisión del usuario de aceptar (conceder) los permisos solicitados y los detalles de autorización.
   * Esta action realiza una solicitud POST al endpoint `/u/customized-consent` con `action: "accept"`.
   * El estado de la transacción se incluye automáticamente.
   *
   * @param {CustomOptions} [payload] - Opcional. Un objeto para cualquier par clave-valor personalizado
   *                                    que se enviará con la request. Estos parámetros se incluirán
   *                                    en los datos del formulario enviados al servidor.
   * @returns {Promise<void>} Una promesa que se resuelve cuando la action de aceptación se envía correctamente.
   *                          En caso de éxito, Auth0 normalmente redirige al usuario de vuelta a la application
   *                          o al siguiente paso en el flow de autenticación.
   * @throws {Error} Lanza un error si el envío del formulario falla (p. ej., problema de red, estado inválido).
   *                 Los errores del lado del servidor (como "invalid_request") se reflejarán en `this.transaction.errors`
   *                 tras la operación, en lugar de lanzarse como errores de JavaScript.
   *
   * @example
   * ```typescript
   * // Se asume que 'customizedConsentManager' es una instancia de la clase CustomizedConsent SDK
   * try {
   *   await customizedConsentManager.accept();
   *   // En caso de éxito, la página normalmente redirigirá.
   * } catch (error) {
   *   // Manejar errores inesperados durante el envío.
   *   console.error("Failed to submit consent acceptance:", error);
   * }
   * // Tras el await, verificar customizedConsentManager.transaction.errors para problemas de validación del servidor.
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * Envía la decisión del usuario de denegar (rechazar) los permisos solicitados y los detalles de autorización.
   * Esta action realiza una solicitud POST al endpoint `/u/customized-consent` con `action: "deny"`.
   * El estado de la transacción se incluye automáticamente.
   *
   * @param {CustomOptions} [payload] - Opcional. Un objeto para cualquier par clave-valor personalizado
   *                                    que se enviará con la request. Estos parámetros se incluirán
   *                                    en los datos del formulario enviados al servidor.
   * @returns {Promise<void>} Una promesa que se resuelve cuando la action de denegación se envía correctamente.
   *                          En caso de éxito, Auth0 normalmente redirige al usuario, mostrando potencialmente un
   *                          mensaje de acceso denegado o devolviendo un error a la application.
   * @throws {Error} Lanza un error si el envío del formulario falla (p. ej., problema de red, estado inválido).
   *                 Los errores del lado del servidor se reflejan en `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // Se asume que 'customizedConsentManager' es una instancia de la clase CustomizedConsent SDK
   * try {
   *   await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // Ejemplo de opción personalizada
   *   // En caso de éxito, la página normalmente redirigirá.
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // Tras el await, verificar customizedConsentManager.transaction.errors para problemas de validación del servidor.
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}

<ParamField body="accept" type="Promise<void>">
  Envía la decisión del usuario de aceptar (conceder) los permisos solicitados y los detalles de autorización.
  Esta acción envía una solicitud al endpoint `/u/customized-consent` con `action: "accept"`.
  El estado de la transacción se incluye automáticamente.

  Una promesa que se resuelve cuando la acción de aceptación se envía correctamente.
  Si la operación se completa con éxito, Auth0 normalmente redirige al usuario de vuelta a la aplicación
  o al siguiente paso del flujo de autenticación.

  #### Lanza

  Lanza un error si falla el envío del formulario (p. ej., por un problema de red o un estado no válido).
  Los errores del lado del servidor (como &quot;invalid&#95;request&quot;) se reflejarán en `this.transaction.errors`
  después de la operación, en lugar de lanzarse como errores de JavaScript.

  ```typescript Example
  // Suponiendo que 'customizedConsentManager' es una instancia de la clase SDK CustomizedConsent
  try {
    await customizedConsentManager.accept();
    // Si tiene éxito, la página normalmente redirigirá.
  } catch (error) {
    // Gestiona errores inesperados durante el propio envío.
    console.error("Failed to submit consent acceptance:", error);
  }
  // Después del await, revisa customizedConsentManager.transaction.errors para detectar problemas de validación del lado del servidor.
  ```
