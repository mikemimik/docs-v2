export interface MfaWebAuthnChangeKeyNicknameMembers extends BaseMembers {

   * Esta acción corresponde al usuario ingresando un nuevo apodo y haciendo clic en el botón "Save" o "Continuar".

   * Si el nuevo apodo es válido y la actualización es exitosa, Auth0 normalmente redirigirá al usuario
   * a la siguiente pantalla correspondiente (por ejemplo, de vuelta a la gestión de factores de MFA o a una confirmación de éxito).

   *                 o si el `FormHandler` encuentra un problema irrecuperable durante el envío (por ejemplo, un error de red).
   *                 Los errores de validación de Auth0 (como un apodo no válido) no se lanzan como errores de JavaScript,
   *                 sino que se reflejan en `this.transaction.errors` tras la operación.
   *
   * @example
   * ```typescript

   * const newNickname = "My Favorite YubiKey"; // Value from user input
   * try {
   *   await sdk.continueWithNewNickname({ nickname: newNickname });
   *   // Si tiene éxito, la página redirige.
   * } catch (error) {
   *   // Este catch es para errores inesperados durante la llamada al SDK.
   *   console.error("Failed to submit the new nickname:", error);
   * }
   * // Después del await, verificar siempre sdk.transaction.errors para detectar problemas de validación del lado del servidor.
   * if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
   *   sdk.transaction.errors.forEach(err => {
   *     if (err.field === 'nickname') {

   *       // p. ej., "Name is too short", "Name is required"
   *     }
   *   });
   * }
   * ```
   */
  continueWithNewNickname(payload: ContinueOptions): Promise<void>;

<ParamField body="continueWithNewNickname" type="Promise<void>">
  Envía el nuevo apodo de la clave de seguridad de WebAuthn.
  Esta acción corresponde a cuando el usuario introduce un nuevo apodo y hace clic en el botón &quot;Guardar&quot; o &quot;Continuar&quot;.
  El SDK enviará por POST este nuevo apodo al endpoint `/u/mfa-webauthn-change-key-nickname` de Auth0.

  Si el nuevo apodo es válido y la actualización se realiza correctamente, Auth0 normalmente redirigirá al usuario
  a la siguiente pantalla correspondiente (p. ej., de vuelta a la gestión de factores de MFA o a una confirmación de éxito).
  Si el apodo no es válido (p. ej., está vacío, es demasiado largo o demasiado corto), Auth0 volverá a renderizar la
  pantalla &#39;mfa-webauthn-change-key-nickname&#39;, y el arreglo `transaction.errors` en el
  contexto del SDK se actualizará con detalles sobre el error de validación.

  Una promesa que se resuelve cuando se inicia el envío del formulario.
  No devuelve datos directamente al resolverse, ya que el resultado habitual
  es una redirección o un nuevo renderizado de la página.

  #### Genera

  Genera un error si no se proporciona `payload.nickname` o si no es una cadena,
  o si el `FormHandler` encuentra un problema irrecuperable durante el envío (p. ej., un error de red).
  Los errores de validación de Auth0 (como un apodo no válido) no se generan como errores de JavaScript,
  sino que se reflejan en `this.transaction.errors` después de la operación.

  ```typescript Example
  // Suponiendo que 'sdk' es una instancia de MfaWebAuthnChangeKeyNickname
  const newNickname = "My Favorite YubiKey"; // Valor introducido por el usuario
  try {
    await sdk.continueWithNewNickname({ nickname: newNickname });
    // Si la operación es correcta, la página redirige.
  } catch (error) {
    // Este catch es para errores inesperados durante la propia llamada del SDK.
    console.error("Failed to submit the new nickname:", error);
  }
  // Después del await, comprueba siempre sdk.transaction.errors para detectar problemas de validación del lado del servidor.
  if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
    sdk.transaction.errors.forEach(err => {
      if (err.field === 'nickname') {
        // Muestra err.message relacionado con el campo de entrada del apodo.
        // p. ej., "Name is too short", "Name is required"
      }
    });
  }
  ```
