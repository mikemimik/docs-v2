<ParamField body="continueWithNewNickname" type="Promise<void>">
  Envía el nuevo nickname de la clave de seguridad de WebAuthn.
  Esta acción corresponde a cuando el usuario introduce un nuevo nickname y hace clic en un botón &quot;Guardar&quot; o &quot;Continuar&quot;.
  El SDK enviará este nuevo nickname al endpoint `/u/mfa-webauthn-change-key-nickname` de Auth0 mediante una solicitud POST.

  Si el nuevo nickname es válido y la actualización se realiza correctamente, Auth0 normalmente redirigirá al usuario
  a la siguiente pantalla correspondiente (por ejemplo, de vuelta a la gestión de factores de MFA o a una confirmación de éxito).
  Si el nickname no es válido (por ejemplo, si está vacío, es demasiado largo o demasiado corto), Auth0 volverá a realizar la renderización de la
  pantalla &#39;mfa-webauthn-change-key-nickname&#39;, y el arreglo `transaction.errors` en el
  contexto del SDK se actualizará con detalles sobre el error de validación.

  Una promesa que se resuelve cuando se inicia el envío del formulario.
  No devuelve datos directamente al resolverse, ya que el resultado habitual
  es una redirección o una nueva renderización de la página.

  #### Throws

  Lanza un error si no se proporciona `payload.nickname` o si no es una string,
  o si el `FormHandler` encuentra un problema irrecuperable durante el envío (por ejemplo, un error de red).
  Los errores de validación de Auth0 (como un nickname no válido) no se lanzan como errores de JavaScript,
  sino que se reflejan en `this.transaction.errors` después de la operación.

  ```typescript Example
  // Suponiendo que 'sdk' es una instancia de MfaWebAuthnChangeKeyNickname
  const newNickname = "My Favorite YubiKey"; // Valor introducido por el usuario
  try {
    await sdk.continueWithNewNickname({ nickname: newNickname });
    // Si se realiza correctamente, la página redirige.
  } catch (error) {
    // Este catch es para errores inesperados durante la propia llamada del SDK.
    console.error("Failed to submit the new nickname:", error);
  }
  // Después del await, comprueba siempre sdk.transaction.errors para detectar problemas de validación del lado del servidor.
  if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
    sdk.transaction.errors.forEach(err => {
      if (err.field === 'nickname') {
        // Muestra err.message relacionado con el campo de entrada del nickname.
        // p. ej., "Name is too short", "Name is required"
      }
    });
  }
  ```
