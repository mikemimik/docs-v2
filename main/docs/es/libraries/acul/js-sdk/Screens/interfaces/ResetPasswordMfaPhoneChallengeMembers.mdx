<ParamField body="continue" type="Promise<void>">
  Envía el código de verificación al teléfono del usuario mediante el método especificado (SMS o voz).
  Esta acción corresponde a cuando el usuario hace clic en el botón &quot;Enviar código por SMS&quot; o &quot;Enviar código por llamada de voz&quot;.
  Activa el Auth0 backend para iniciar el proceso de entrega del código.

  Una promesa que se resuelve cuando la acción se envía correctamente al servidor.

  #### Throws

  Lanza un error si el envío falla (por ejemplo, por un problema de red o un `state` no válido).

  ```typescript Example
  import ResetPasswordMfaPhoneChallenge from '@auth0/auth0-acul-js/reset-password-mfa-phone-challenge';

  const screen = new ResetPasswordMfaPhoneChallenge();

  // Enviar código por SMS
  await screen.continue({ type: 'sms' });

  // Enviar código por llamada de voz
  await screen.continue({ type: 'voice' });
  ```

  <Expandable title="Parámetros">
    <ParamField body="payload" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/ResetPasswordMfaPhoneChallengeContinueOptions">ResetPasswordMfaPhoneChallengeContinueOptions</a></span>}>
      Las opciones que especifican el método de entrega (`sms` o `voice`) y cualquier dato personalizado.

      **Propiedades**

      <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">
        El método de entrega que el usuario seleccionó para recibir el código de verificación.

        * `sms`: Envía el código por mensaje de texto.
        * `voice`: Envía el código mediante una llamada de voz.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Lleva al usuario a la pantalla donde puede seleccionar un método de MFA alternativo.
  Esta acción corresponde a cuando el usuario hace clic en un botón &quot;Probar otro método&quot;.

  Una promesa que se resuelve cuando la acción se envía correctamente al servidor.

  #### Throws

  Lanza un error si el envío falla.

  ```typescript Example
  import ResetPasswordMfaPhoneChallenge from '@auth0/auth0-acul-js/reset-password-mfa-phone-challenge';

  const screen = new ResetPasswordMfaPhoneChallenge();

  // Si inicialmente se le mostró al usuario la opción de SMS:
  await screen.tryAnotherMethod({ type: 'sms' });

  // Si inicialmente se le mostró al usuario la opción de voz:
  await screen.tryAnotherMethod({ type: 'voice' });
  ```

    <ParamField body="payload" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions">ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions</a></span>}>
      Las opciones que especifican el tipo de desafío actual (`sms` o `voice`) y cualquier dato personalizado. El campo `type` es necesario para indicar al backend de qué flujo se está apartando el usuario.

      **Propiedades**

      <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">

        Esto es obligatorio para que el endpoint de la API procese correctamente la acción `pick-authenticator`.

        * `sms`: Indica que la pantalla actual corresponde al desafío de SMS.
        * `voice`: Indica que la pantalla actual corresponde al desafío de voz.
