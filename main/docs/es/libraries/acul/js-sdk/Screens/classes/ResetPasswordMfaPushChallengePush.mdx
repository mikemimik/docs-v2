La clase ResetPasswordMfaPushChallengePush implementa la funcionalidad de la pantalla `reset-password-mfa-push-challenge-push`. Esta pantalla se muestra cuando se ha enviado una notificación push al dispositivo del usuario como parte del paso de MFA durante un flujo de restablecimiento de contraseña.

<ParamField body="pollingManager" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Este método inicia y administra la consulta periódica del desafío MFA push. Comprueba repetidamente si la notificación push se ha aprobado en el intervalo especificado e invoca la devolución de llamada `onCompleted` cuando detecta la aprobación. Las respuestas HTTP 429 de límite de frecuencia se gestionan internamente: la consulta periódica espera a que el límite de frecuencia se restablezca antes de volver a intentarlo.

  El [`MfaPushPollingControl`](/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) devuelto expone `startPolling()`, `stopPolling()` e `isRunning()` para administrar el ciclo de vida de la consulta periódica.

  ```typescript Example
  import ResetPasswordMfaPushChallengePush from '@auth0/auth0-acul-js/reset-password-mfa-push-challenge-push';
  const resetPasswordMfaPushChallengePushManager = new ResetPasswordMfaPushChallengePush();
  const control = resetPasswordMfaPushChallengePushManager.pollingManager({
    intervalMs: 5000,
    onCompleted: () => resetPasswordMfaPushChallengePushManager.continue(),
    onError: (error) => console.error('Polling error:', error),
  });

  control.startPolling();

  // Cancelar la consulta periódica si es necesario
  control.stopPolling();
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [MfaPollingOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      El intervalo en milisegundos entre intentos de consulta periódica.
    </ParamField>

    <ParamField body="onCompleted?" type="() => void">
      Devolución de llamada que se invoca cuando se aprueba el desafío push y la consulta periódica se detiene automáticamente.
    </ParamField>

    <ParamField body="onError?" type="(error: Error) => void">
      Devolución de llamada que se invoca si se produce un error durante la consulta periódica.
    </ParamField>
  </Expandable>
</ParamField>
