---
title: "LoginId"
description: "Describe todas las propiedades y métodos disponibles para personalizar la pantalla `login-id` de Universal Login."
---

La clase LoginId implementa la funcionalidad de la pantalla `login-id`. Esta pantalla recopila el identificador y la contraseña del usuario. Según tu tenant, este identificador puede ser un correo electrónico, un número de teléfono o un nombre de usuario.

<div id="constructors">
  ## Constructores
</div>

Cree una instancia del administrador de la pantalla LoginId:

```typescript Example
import LoginId from "@auth0/auth0-acul-js/login-id";
const loginIdManager = new LoginId();
loginIdManager.getLoginIdentifiers();
```

<div id="methods">
  ## Métodos
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma de visualización de la página de Universal Login.

  ```typescript Example
  import LoginId from "@auth0/auth0-acul-js/login-id";
  const loginIdManager = new LoginId();
  loginIdManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de configuración regional del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Cuando se establece en `'session'`, el idioma seleccionado se conserva durante toda la sesión.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedLogin" type="Promise<void>">
  Este método redirige al usuario al proveedor de identidad (IdP) social o empresarial para autenticarse.

  ```typescript Example
  import LoginId from "@auth0/auth0-acul-js/login-id";
  const loginIdManager = new LoginId();
  loginIdManager.federatedLogin({
    connection: "google-oauth2"
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [FederatedLoginPayloadOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/FederatedLoginPayloadOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      El nombre de la conexión social que se debe usar.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Este método recupera del contexto el arreglo de errores de la transacción, o un arreglo vacío si no existe ninguno.
</ParamField>

<ParamField body="getLoginIdentifiers" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/type-aliases/ScreenIdentifierType">ScreenIdentifierType</a>[] | null</span>}>
  Este método obtiene los tipos de identificador activos para la pantalla `login-id`.

  ```typescript Example
  import LoginId from "@auth0/auth0-acul-js/login-id";
  const loginIdManager = new LoginId();
  loginIdManager.getLoginIdentifiers();
  ```
</ParamField>

<ParamField body="login" type="Promise<void>">
  Este método solicita al usuario que proporcione su nombre de usuario.

  ```typescript Example
  import LoginId from "@auth0/auth0-acul-js/login-id";
  const loginIdManager = new LoginId();
  loginIdManager.login({
    username: "testUser"
  });
  ```

    <ParamField body="captcha?" type="string">

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).

  </Expandable>
</ParamField>

<ParamField body="passkeyLogin" type="Promise<void>">

  import LoginId from "@auth0/auth0-acul-js/login-id";

  loginIdManager.passkeyLogin();

  loginIdManager.pickCountryCode();

<ParamField body="registerPasskeyAutofill" type="Promise<void>">
  Este método inicializa una solicitud pasiva de credencial de WebAuthn mediante `navigator.credentials.get()` con `mediation: conditional`. Cuando es compatible, permite que el navegador muestre las claves de acceso almacenadas directamente en el menú desplegable de autocompletado del campo de nombre de usuario.

  ```typescript Example
  import LoginId from '@auth0/auth0-acul-js/login-id';

  async function initializeLogin() {
    const loginId = new LoginId();
    await loginId.registerPasskeyAutofill('username');
  }

  initializeLogin().catch(console.error);
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="inputId?" type="string">
      ID opcional del elemento `<input>` de nombre de usuario (sin `#`). Ejemplo: `"username"`. Si se omite, el desarrollador debe asegurarse manualmente de que el elemento de entrada tenga configurados los atributos `autocomplete` correctos.
    </ParamField>
  </Expandable>
</ParamField>
