La clase DeviceCodeActivationDenied implementa la funcionalidad de la pantalla `device-code-activation-denied`. Esta pantalla se muestra cuando se ha denegado la activación del código de dispositivo.

<div id="methods">
  ## Métodos
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma que se muestra en la página de Universal Login.

  ```typescript Example
  import DeviceCodeActivationDenied from '@auth0/auth0-acul-js/device-code-activation-denied';
  const deviceCodeActivationDeniedManager = new DeviceCodeActivationDenied();
  deviceCodeActivationDeniedManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de configuración regional del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Cuando se establece en `'session'`, el idioma seleccionado se mantiene durante la sesión.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Este método devuelve el arreglo de errores de la transacción del contexto, o un arreglo vacío si no existe ninguno.
</ParamField>
