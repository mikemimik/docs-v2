---
title: "SignupId"
description: "Describe todas las propiedades y métodos disponibles para personalizar la pantalla `signup-id` de Universal Login."
---

La clase SignupId implementa la funcionalidad de la pantalla `signup-id`. Esta pantalla recopila el identificador del usuario.

  <img style={{maxHeight:"400px"}} src="/docs/images/cdy7uua7fh8z/5lzGYj0k4VcxvPCBYH37SO/ebd291cc0dbd673a70f93530f724bf65/Signup_ID.png" alt="Signup ID" />
</Frame>

<div id="constructors">
  ## Constructores
</div>

Cree una instancia del administrador de la pantalla SignupId:

```typescript Example
import SignupId from '@auth0/auth0-acul-js/signup-id';
const signupIdManager = new SignupId();
signupIdManager.signup({
  email: 'test@example.com',
});
```

<div id="methods">
  ## Métodos
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma que se muestra en la página de Universal Login.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de locale del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Cuando se establece en `'session'`, el idioma seleccionado se conserva durante toda la sesión.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedSignup" type="Promise<void>">
  Este método gestiona el registro mediante distintos proveedores de identidad social. Por ejemplo: Google, Facebook, etc.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.federatedSignup({
    connection: 'google-oauth2',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [FederatedSignupOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      El nombre de la conexión social que se usará.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Este método recupera del contexto el arreglo de errores de la transacción, o un arreglo vacío si no existe ninguno.
</ParamField>

<ParamField body="getSignupIdentifiers" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Identifier">Identifier</a>[] | null</span>}>
  Este método devuelve la lista de identificadores habilitados para el formulario `signup-id` y marca cada uno como obligatorio u opcional según la configuración de la transacción. Un arreglo de objetos identificadores (por ejemplo: `email`, `phone`, `username`).

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  const identifiers = signupIdManager.getSignupIdentifiers();
  // [{ type: 'email', required: true }, { type: 'username', required: false }]
  ```
</ParamField>

<ParamField body="pickCountryCode" type="Promise<void>">
  Este método redirige al usuario a la lista de selección de código de país, donde puede actualizar el prefijo de su número de teléfono.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.pickCountryCode();
  ```
</ParamField>

<ParamField body="signup" type="Promise<void>">
  Este método gestiona la configuración relacionada con `signup-id`. Permite registrar usuarios nuevos mediante distintos identificadores.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.signup({
    email: 'test@example.com',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [SignupOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/SignupOptions).
    </ParamField>

    <ParamField body="email?" type="string">
      El correo electrónico del usuario.
    </ParamField>

    <ParamField body="username?" type="string">
      El nombre de usuario.
    </ParamField>

    <ParamField body="phone?" type="string">
      El número de teléfono del usuario.
    </ParamField>

    <ParamField body="captcha?" type="string">
      El código captcha o la respuesta del proveedor de CAPTCHA. Esta propiedad es obligatoria si tu tenant de Auth0 tiene [Bot Detection](/docs/es/secure/attack-protection/bot-detection) habilitado.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validateUsername" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  Este método valida un nombre de usuario según la política de nombre de usuario actual definida en el contexto de la transacción. Devuelve un objeto que indica si el nombre de usuario es válido y por qué.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  const result = signupIdManager.validateUsername('myusername');
  // result => { valid: true, errors: [] }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="username" type="string" required>
      La cadena de nombre de usuario que se va a validar.
    </ParamField>
  </Expandable>
</ParamField>
