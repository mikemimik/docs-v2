La clase MfaSmsChallenge implementa la funcionalidad de la pantalla `mfa-sms-challenge`. Esta pantalla se muestra cuando el usuario debe introducir un código SMS para verificar su identidad.

<div id="constructors">
  ## Constructores
</div>

Crea una instancia del gestor de la pantalla MFA SMS Challenge:

```typescript Example
import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

const mfaSmsChallenge = new MfaSmsChallenge();
await mfaSmsChallenge.continueMfaSmsChallenge({
  code: '123456',
  rememberDevice: true,
});
```

<div id="methods">
  ## Métodos
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma que se muestra en la página de Universal Login.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  mfaSmsChallenge.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de configuración regional del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Cuando se establece en `'session'`, el idioma seleccionado se mantiene durante toda la sesión.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="continueMfaSmsChallenge" type="Promise<void>">
  Este método envía el desafío de MFA por SMS con el código proporcionado.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.continueMfaSmsChallenge({
    code: '123456',
    rememberDevice: true,
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [MfaSmsChallengeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaSmsChallengeOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      El código SMS introducido por el usuario.
    </ParamField>

    <ParamField body="rememberDevice?" type="boolean">
      Cuando se establece en `true`, registra este dispositivo como de confianza para que no se le solicite MFA al usuario en los siguientes inicios de sesión desde el mismo dispositivo.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getACall" type="Promise<void>">
  Este método cambia el método de verificación a una llamada de voz.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.getACall();
  ```
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Este método recupera el arreglo de errores de la transacción desde el contexto, o un arreglo vacío si no existe ninguno.
</ParamField>

<ParamField body="pickSms" type="Promise<void>">
  Este método navega a la pantalla donde el usuario puede elegir un número de SMS diferente.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.pickSms();
  ```
</ParamField>

<ParamField body="resendCode" type="Promise<void>">
  Este método reenvía el código de verificación por SMS al usuario.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.resendCode();
  ```
</ParamField>

<ParamField body="resendManager" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/ResendControl">ResendControl</a></span>}>
  Este método proporciona la funcionalidad de reenvío con un tiempo de espera configurable y gestión del estado.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  const { startResend } = mfaSmsChallenge.resendManager({
    timeoutSeconds: 15,
    onStatusChange: (remainingSeconds, isDisabled) => {
      console.log(`Resend available in ${remainingSeconds}s, disabled: ${isDisabled}`);
    },
    onTimeout: () => {
      console.log('Resend is now available');
    },
  });

  startResend();
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [StartResendOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/StartResendOptions).
    </ParamField>

    <ParamField body="onStatusChange?" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/type-aliases/OnStatusChangeCallback">OnStatusChangeCallback</a></span>}>
      Devolución de llamada que se invoca cada vez que cambia el estado del reenvío y recibe los segundos restantes e indica si el reenvío está deshabilitado en ese momento.
    </ParamField>

    <ParamField body="onTimeout?" type="() => void">
      Devolución de llamada que se invoca cuando vence el tiempo de espera del reenvío y la opción de reenvío vuelve a estar disponible.
    </ParamField>

    <ParamField body="timeoutSeconds?" type="number">
      La cantidad de segundos que deben transcurrir antes de que la opción de reenvío esté disponible.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Este método permite al usuario probar con otro método de MFA.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.tryAnotherMethod();
  ```
</ParamField>
