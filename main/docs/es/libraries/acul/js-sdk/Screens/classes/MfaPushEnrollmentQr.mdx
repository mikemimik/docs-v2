<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma que se muestra en la página de Universal Login.

  ```typescript Example
  import MfaPushEnrollmentQr from '@auth0/auth0-acul-js/mfa-push-enrollment-qr';
  const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();
  mfaPushEnrollmentQr.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de locale del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Cuando se establece en `'session'`, el idioma seleccionado se mantiene durante toda la sesión.
    </ParamField>
  </Expandable>

<ParamField body="pollingManager" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Este método inicia y gestiona el polling de una inscripción en MFA push.

  Crea una sesión de polling que consulta repetidamente el endpoint de inscripción en MFA push en el intervalo especificado hasta que la inscripción se apruebe o se produzca un error. Cuando se cumple la condición de aprobación, se invoca la devolución de llamada [onCompleted](/docs/es/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) proporcionada y el polling se detiene automáticamente.

  Usa el [MfaPushPollingControl](/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) devuelto para iniciar, detener o consultar el estado del proceso de polling en cualquier momento.

  Devuelve una instancia de [MfaPushPollingControl](/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) que expone:

  * `startPolling()` — Inicia o reanuda el polling.
  * `stopPolling()` — Cancela el polling inmediatamente.
  * `isRunning()` — Indica si el polling está activo en ese momento.

  ```typescript Example
  import MfaPushEnrollmentQr from '@auth0/auth0-acul-js/mfa-push-enrollment-qr';

  const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();

  // Inicia el polling cada 5 segundos hasta que se apruebe la inscripción push
  const control = mfaPushEnrollmentQr.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushEnrollmentQr.continue(),
    onError: (error) => console.error('Polling error:', error),
  });

  // Más adelante, cancela el polling si es necesario
  control.stopPolling();
  ```

  **Observaciones**

  * Las respuestas HTTP 429 (límite de frecuencia) se gestionan internamente: el polling espera automáticamente hasta que el límite de frecuencia se restablezca antes de volver a intentarlo.
  * Puedes llamar a `startPolling()` varias veces sin problema; no tendrá efecto si ya se está ejecutando.

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [MfaPollingOptions](/docs/es/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalo opcional, en milisegundos, entre solicitudes de polling consecutivas. Si se omite, se usa el intervalo interno predeterminado del SDK (normalmente 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Devolución de llamada opcional que se ejecuta cuando la inscripción en MFA push se aprueba correctamente y el polling finaliza. Se llama exactamente una vez, tras lo cual el polling se detiene automáticamente.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Devolución de llamada opcional que se invoca si se produce un error durante el polling. Recibe un objeto `ULError` que contiene `status` (código de estado HTTP) y `responseText` (cuerpo de la respuesta sin procesar).
    </ParamField>
  </Expandable>
</ParamField>
