---
title: "Signup"
description: "Describe todas las propiedades y los métodos disponibles para personalizar la pantalla `signup` de Universal Login."
---

La clase Signup implementa la funcionalidad de la pantalla `signup`. Esta pantalla solicita el identificador y la contraseña del usuario. Según tu tenant, este identificador puede ser un correo electrónico, un número de teléfono o un nombre de usuario.

<div id="constructors">
  ## Constructores
</div>

Crea una instancia del administrador de la pantalla de Signup:

```typescript Example
import Signup from '@auth0/auth0-acul-js/signup';
const signupManager = new Signup();
signupManager.signup({
  email: 'test@example.com',
  password: 'P@$wOrd123!',
});
```

<div id="methods">
  ## Métodos
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma que se muestra en la página de Universal Login.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de locale del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Cuando se establece en `'session'`, el idioma seleccionado se mantiene durante toda la sesión.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedSignup" type="Promise<void>">
  Este método gestiona el envío del formulario de registro social. Es similar al método [signup](#signup), pero se usa para registros sociales.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.federatedSignup({
    connection: 'google-oauth2',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [FederatedSignupPayloadOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupPayloadOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      El nombre de la conexión social que se va a usar.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Este método recupera el arreglo de errores de la transacción desde el contexto, o un arreglo vacío si no existe ninguno. Devuelve un arreglo de objetos de error del contexto de la transacción.
</ParamField>

<ParamField body="getSignupIdentifiers" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Identifier">Identifier</a>[] | null</span>}>
  Este método devuelve la lista de identificadores habilitados para el formulario de registro y marca cada uno como obligatorio u opcional según la configuración de la transacción. Un arreglo de objetos identificadores (por ejemplo, correo electrónico, teléfono, nombre de usuario).

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const identifiers = signupManager.getSignupIdentifiers();
  // [{ type: 'email', required: true }, { type: 'username', required: false }]
  ```
</ParamField>

<ParamField body="pickCountryCode" type="Promise<void>">
  Este método redirige al usuario a la lista de selección de código de país, donde puede actualizar el prefijo del código de país de su número de teléfono.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.pickCountryCode();
  ```
</ParamField>

<ParamField body="signup" type="Promise<void>">
  Este método gestiona el envío del formulario de registro.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.signup({
    email: 'test@example.com',
    password: 'P@$wOrd123!',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [SignupPayloadOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/SignupPayloadOptions).
    </ParamField>

    <ParamField body="email?" type="string">
      El correo electrónico del usuario.
    </ParamField>

    <ParamField body="username?" type="string">
      El identificador del usuario.
    </ParamField>

    <ParamField body="phoneNumber?" type="string">
      El número de teléfono del usuario.
    </ParamField>

    <ParamField body="password?" type="string">
      La contraseña del usuario.
    </ParamField>

    <ParamField body="captcha?" type="string">
      El código captcha o la respuesta del proveedor de captcha. Esta propiedad es obligatoria si tu tenant de Auth0 tiene [Bot Detection](/docs/es/secure/attack-protection/bot-detection) habilitado.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validatePassword" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/PasswordValidationResult">PasswordValidationResult</a></span>}>
  Este método devuelve un objeto de tipo [PasswordValidationResult](/docs/es/libraries/acul/js-sdk/Screens/interfaces/PasswordValidationResult) que indica si la contraseña es válida y el motivo.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const result = signupManager.validatePassword('P@$wOrd123!');
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="password" type="string" required>
      La contraseña del usuario.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validateUsername" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  Este método valida un nombre de usuario según la política de nombres de usuario actual definida en el contexto de la transacción. Devuelve un objeto que indica si el nombre de usuario es válido y el motivo.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const result = signupManager.validateUsername('myusername');
  // result => { valid: true, errors: [] }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="username" type="string" required>
      El nombre de usuario que se va a validar.
    </ParamField>
  </Expandable>
</ParamField>
