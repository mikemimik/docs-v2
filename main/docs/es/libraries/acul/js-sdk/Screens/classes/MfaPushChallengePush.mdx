La clase MfaPushChallengePush implementa la funcionalidad de la pantalla `mfa-push-challenge-push`. Esta pantalla se muestra cuando un usuario necesita confirmar una notificación push durante la autenticación MFA.

    language: 'fr',

<ParamField body="pollingManager" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Este método inicia y administra la consulta de un desafío push de MFA.

  Crea una sesión de consulta que comprueba repetidamente el endpoint del desafío push de MFA en el intervalo especificado hasta que el desafío se apruebe o se produzca un error. Cuando se cumple la condición de aprobación, se invoca la devolución de llamada [onCompleted](/docs/es/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) proporcionada y la consulta se detiene automáticamente.

  Usa el [MfaPushPollingControl](/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) devuelto para iniciar, detener o comprobar el estado del proceso de consulta en cualquier momento.

  Devuelve una instancia de [MfaPushPollingControl](/docs/es/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) que expone:

  * `startPolling()` — Inicia o reanuda la consulta.
  * `stopPolling()` — Cancela la consulta de inmediato.
  * `isRunning()` — Indica si la consulta está activa en ese momento.

  ```typescript Example
  import MfaPushChallengePush from '@auth0/auth0-acul-js/mfa-push-challenge-push';

  const mfaPushChallengePush = new MfaPushChallengePush();

  // Inicia la consulta cada 5 segundos hasta que se apruebe el desafío push
  const control = mfaPushChallengePush.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushChallengePush.continue(),
    onError: (error) => console.error('Error de consulta:', error),
  });

  // Más adelante, cancela la consulta si es necesario
  control.stopPolling();
  ```

  **Observaciones**

  * Las respuestas HTTP 429 (límite de frecuencia) se gestionan internamente: la consulta espera automáticamente hasta que se restablezca el límite de frecuencia antes de volver a intentarlo.
  * Se puede llamar a `startPolling()` varias veces sin problema; no tiene efecto si ya está en ejecución.

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [MfaPollingOptions](/docs/es/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalo opcional, en milisegundos, entre solicitudes de consulta consecutivas. Si se omite, se usa el intervalo predeterminado interno del SDK (normalmente 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Devolución de llamada opcional que se ejecuta cuando el desafío push de MFA se aprueba correctamente y la consulta finaliza. Se llama exactamente una vez, tras lo cual la consulta se detiene automáticamente.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Devolución de llamada opcional que se invoca si se produce un error durante la consulta. Recibe un objeto `ULError` que contiene `status` (código de estado HTTP) y `responseText` (cuerpo de la respuesta sin procesar).
    </ParamField>
  </Expandable>
</ParamField>
