title: Migra a la confirmación del usuario final para URI de devolución de llamada no verificables
validatedOn: 2026-02-10
---

Auth0 recomienda que todas las aplicaciones nativas que usan el [flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) hagan la transición a devoluciones de llamada basadas en HTTPS mediante [Android App Links](https://developer.android.com/training/app-links) y [Apple Universal Links](https://developer.apple.com/documentation/xcode/allowing-apps-and-websites-to-link-to-your-content). Esto mejora la seguridad y mitiga el riesgo de suplantación de aplicaciones y de ataques de phishing.

Para entender cómo esto previene los ataques, revisa [Measures Against Application Impersonation](/docs/es/secure/security-guidance/measures-against-app-impersonation#prompt-customization).

Los tenants creados antes del 15 de octubre de 2025 mantendrán el comportamiento anterior como predeterminado hasta el 28 de abril de 2026. Después de la fecha límite de octubre, es posible que los tenants creados recientemente muestren por defecto el nuevo prompt de confirmación de inicio de sesión, con algunas excepciones según el calendario de implementación de cada entorno.

Los tenants que opten explícitamente por excluirse omitirán este prompt de forma indefinida. Esto seguirá siendo así incluso después del 28 de abril de 2026, cuando el servicio adopte el prompt de confirmación como comportamiento predeterminado y elimine el toggle de migración &quot;Unconfirmed Login with Non-Verifiable Callback URI Redirects&quot;.

<div id="how-are-you-affected">
  ## ¿Cómo le afecta?
</div>

En el caso de las aplicaciones cliente que ya especifican, o tienen previsto especificar, un esquema de URI personalizado o una URI de loopback de devolución de llamada, es posible que se exija a los usuarios finales que confirmen explícitamente el inicio de sesión interactuando con el nuevo prompt de confirmación de inicio de sesión. Sus usuarios finales podrían percibir este cambio como un deterioro de la experiencia de usuario.

Además, las solicitudes de autenticación que incluyan `prompt=none` se rechazarán cuando las aplicaciones usen URI de devolución de llamada no verificables y estén configuradas para usar el nuevo prompt de confirmación de inicio de sesión.

<div id="migration-tasks">
  ## Tareas de migración
</div>

**Auth0 recomienda encarecidamente migrar a devoluciones de llamada basadas en HTTPS** mediante Android App Links y Apple Universal Links siempre que sea posible para todas las aplicaciones nativas que utilicen el flujo de código de autorización.

Además, en los tenants en los que el comportamiento predeterminado cambie después del 28 de abril de 2026, debe seleccionar explícitamente el comportamiento que necesita para las solicitudes de autenticación que usen esquemas de URI personalizados o devoluciones de llamada de URI de loopback antes de que cambie el valor predeterminado del sistema.

<div id="review-whether-your-applications-are-using-non-verifiable-callback-uris">
  ### Revisa si tus aplicaciones usan Non-Verifiable Callback URIs
</div>

En los tenants en los que el toggle de migración Unconfirmed Login with Non-Verifiable Callback URI Redirects está disponible y habilitado, las solicitudes de autenticación que especifican un esquema de URI personalizado o una URI de loopback **generarán un registro del tenant con un aviso de obsolescencia** a menos que hayas establecido explícitamente la siguiente opción a nivel de la aplicación o del tenant:

`skip_non_verifiable_callback_uri_confirmation_prompt`

Estos registros del tenant contienen el identificador del cliente de la aplicación que realiza la solicitud. Puedes supervisar estos registros del tenant desde el Auth0 Dashboard con la siguiente consulta:

```bash
type:depnotetype:depnote AND description:Unconfirmed\ Login\ with\ Non-Verifiable\ Callback\ URI\ Redirects*
```

<div id="opt-in-to-new-login-confirmation-prompt">
  ### Habilita explícitamente el nuevo prompt de confirmación de inicio de sesión
</div>

Para habilitar explícitamente de antemano el nuevo prompt de confirmación de inicio de sesión y reforzar la seguridad de los flujos de autenticación que usan esquemas de URI personalizados o URI de loopback, completa los siguientes pasos en tu Auth0 Dashboard:

1. Ve a [**Auth0 Dashboard &gt; Configuración del tenant &gt; Advanced**](https://manage.auth0.com/#/tenant/advanced).
2. En la sección **Migración**, desactiva el selector **Unconfirmed Login with Non-Verifiable Callback URI Redirects**.

<Frame>![Auth0 Dashboard > Configuración del tenant > Advanced > selector desactivado](/docs/images/cdy7uua7fh8z/unconfirmed-login-uri.png)</Frame>

<div id="opt-out-of-new-login-confirmation-prompt">
  ### No usar el nuevo prompt de confirmación de inicio de sesión
</div>

Si, después de evaluar las consideraciones de seguridad, decide no usar el nuevo prompt de confirmación de inicio de sesión, puede configurar aplicaciones específicas o todo el tenant para no adoptar este nuevo comportamiento. Puede hacerlo desde su Auth0 Dashboard.

La configuración a nivel de aplicación tiene prioridad sobre la configuración a nivel del tenant. Asegúrese de configurar primero los ajustes específicos de la aplicación antes de cambiar la configuración a nivel del tenant para evitar cambios de comportamiento no deseados. Por ejemplo, puede que quiera omitir la **Non-Verifiable Callback URI End-User Confirmation** en algunas aplicaciones específicas y, de forma predeterminada, mostrarla en otras aplicaciones, o viceversa.

Para no usarlo en aplicaciones específicas:

1. Vaya a [Auth0 Dashboard &gt; Applications &gt; Settings &gt; Advanced Settings &gt; OAuth](https://manage.auth0.com/applications/settings).
2. Localice y desactive el toggle **Non-Verifiable Callback URI End-User Confirmation** y seleccione **Save**. Es posible que deba seleccionar la opción **Override the tenant setting** para poder administrar esta configuración de forma permanente.

<Frame>![Auth0 Dashboard > Applications > Settings > Advanced](/docs/images/cdy7uua7fh8z/custom-uri-override.png)</Frame>

Para no usarlo en todo el tenant:

1. Vaya a [Auth0 Dashboard &gt; Configuración del tenant &gt; Advanced](https://manage.auth0.com/tenant/advanced).
2. Localice y desactive el toggle **Non-Verifiable Callback URI End-User Confirmation** en la sección **Login and Logout** y seleccione **Save**. Es posible que deba seleccionar **Turn on** para poder administrar esta configuración de forma permanente.

<Frame>![Auth0 Dashboard > Configuración del tenant > Advanced](/docs/images/cdy7uua7fh8z/migrate-uri-setting.png)</Frame>

También puede configurar el comportamiento requerido del tenant mediante Auth0 Management API. En concreto, puede realizar la configuración en dos niveles:

* **Configuración a nivel del tenant**: Puede administrar el comportamiento del prompt de confirmación configurando la propiedad `skip_non_verifiable_callback_uri_confirmation_prompt` a través del endpoint [Update Tenant Settings](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings).​
* **Configuración a nivel de aplicación**: Para sobrescribir la configuración a nivel del tenant en aplicaciones específicas, configure la misma propiedad `skip_non_verifiable_callback_uri_confirmation_prompt` a través del endpoint [Update Client](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id).

Para obtener información adicional y orientación sobre cómo configurar sus aplicaciones, consulte [Measures Against Application Impersonation](/docs/es/secure/security-guidance/measures-against-app-impersonation).
