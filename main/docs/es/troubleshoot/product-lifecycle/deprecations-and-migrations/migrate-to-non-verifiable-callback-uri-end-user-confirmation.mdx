description: Descubre cómo puedes mejorar la seguridad y mitigar los riesgos de la suplantación de aplicaciones con devoluciones de llamada basadas en HTTPS.
title: Migrar a Non-Verifiable Callback URI End-User Confirmation
validatedOn: 2026-02-10
---

Auth0 recomienda que todas las aplicaciones nativas que usan el [flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) migren a devoluciones de llamada basadas en HTTPS mediante [Android App Links](https://developer.android.com/training/app-links) y [Apple Universal Links](https://developer.apple.com/documentation/xcode/allowing-apps-and-websites-to-link-to-your-content). Esto mejora la seguridad y reduce el riesgo de suplantación de aplicaciones y de ataques de phishing.

Para entender cómo esto ayuda a prevenir ataques, consulta [Measures Against Application Impersonation](/docs/es/secure/security-guidance/measures-against-app-impersonation#prompt-customization).

Los tenants creados antes del 15 de octubre de 2025 mantienen el comportamiento anterior como opción predeterminada hasta el 28 de abril de 2026. Después de la fecha límite de octubre, los tenants creados a partir de entonces pueden mostrar de forma predeterminada el nuevo prompt de confirmación de inicio de sesión, con algunas excepciones según el calendario de deployment de cada entorno.

Los tenants que decidan excluirse explícitamente seguirán omitiendo este prompt de forma indefinida. Esto seguirá siendo así incluso después del 28 de abril de 2026, cuando el servicio adopte el prompt de confirmación como comportamiento predeterminado y elimine el toggle de migración &quot;Unconfirmed Login with Non-Verifiable Callback URI Redirects&quot;.

<div id="how-are-you-affected">
  ## ¿Cómo le afecta?
</div>

En el caso de las aplicaciones cliente que ya especifican o tienen previsto especificar un esquema de URI personalizado o una devolución de llamada a un URI de loopback, es posible que se pida a los usuarios finales que confirmen explícitamente el inicio de sesión interactuando con el nuevo prompt de confirmación de inicio de sesión. Sus usuarios finales podrían percibir este cambio como un deterioro de la experiencia del usuario.

Además, las solicitudes de autenticación que incluyan `prompt=none` se rechazarán cuando las aplicaciones usen URI de devolución de llamada no verificables y estén configuradas para usar el nuevo prompt de confirmación de inicio de sesión.

<div id="migration-tasks">
  ## Tareas de migración
</div>

**Auth0 recomienda encarecidamente pasar a devoluciones de llamada basadas en HTTPS** mediante Android App Links y Apple Universal Links siempre que sea posible para todas las aplicaciones nativas que usan el flujo de código de autorización.

Además, en los tenants donde el comportamiento predeterminado cambie después del 28 de abril de 2026, debe seleccionar explícitamente el comportamiento que necesita para las solicitudes de autenticación que usan esquemas de URI personalizados o devoluciones de llamada a URI de loopback antes de que cambie el valor predeterminado del sistema.

<div id="review-whether-your-applications-are-using-non-verifiable-callback-uris">
  ### Revise si sus aplicaciones usan URI de devolución de llamada no verificables
</div>

En los tenants en los que el interruptor de migración Unconfirmed Login with Non-Verifiable Callback URI Redirects esté disponible y habilitado, las solicitudes de autenticación que especifiquen un esquema de URI personalizado o una URI de loopback **generarán en el registro del tenant un aviso de obsolescencia** a menos que haya establecido explícitamente la siguiente opción a nivel de la aplicación o del tenant:

`skip_non_verifiable_callback_uri_confirmation_prompt`

Estos registros del tenant contienen el identificador de cliente de la aplicación que realiza la solicitud. Puede supervisar estos registros del tenant desde el Auth0 Dashboard con la siguiente consulta:

```bash
type:depnotetype:depnote AND description:Unconfirmed\ Login\ with\ Non-Verifiable\ Callback\ URI\ Redirects*
```

<div id="opt-in-to-new-login-confirmation-prompt">
  ### Habilita explícitamente el nuevo prompt de confirmación de inicio de sesión
</div>

Para habilitar explícitamente de antemano el nuevo prompt de confirmación de inicio de sesión y reforzar la seguridad de los flujos de autenticación que usan esquemas URI personalizados o URI de loopback, sigue estos pasos en tu Auth0 Dashboard:

1. Ve a [**Auth0 Dashboard &gt; Configuración del tenant &gt; Avanzado**](https://manage.auth0.com/#/tenant/advanced).
2. En la sección **Migración**, desactiva el toggle **Unconfirmed Login with Non-Verifiable Callback URI Redirects**.

<Frame>![Auth0 Dashboard > Configuración del tenant > Avanzado > toggle desactivado](/docs/images/cdy7uua7fh8z/unconfirmed-login-uri.png)</Frame>

<div id="opt-out-of-new-login-confirmation-prompt">
  ### No usar el nuevo prompt de confirmación de inicio de sesión
</div>

Si, después de evaluar las consideraciones de seguridad, decide no usar el nuevo prompt de confirmación de inicio de sesión, puede configurar aplicaciones específicas o todo el tenant para excluirlos de este nuevo comportamiento. Puede hacerlo desde el Auth0 Dashboard.

La configuración a nivel de aplicación tiene prioridad sobre la configuración a nivel de tenant. Asegúrese de configurar primero las opciones específicas de la aplicación antes de cambiar la configuración a nivel de tenant para evitar cambios de comportamiento no deseados. Por ejemplo, puede que quiera omitir la **Non-Verifiable Callback URI End-User Confirmation** para algunas aplicaciones concretas y, de forma predeterminada, mostrarla para otras, o viceversa.

Para excluir aplicaciones específicas:

1. Vaya a [Auth0 Dashboard &gt; Applications &gt; Settings &gt; Advanced Settings &gt; OAuth](https://manage.auth0.com/applications/settings).
2. Busque y desactive el toggle **Non-Verifiable Callback URI End-User Confirmation** y seleccione **Save**. Es posible que tenga que seleccionar la opción **Override the tenant setting** para poder administrar esta configuración de forma permanente.

<Frame>![Auth0 Dashboard > Applications > Settings > Advanced](/docs/images/cdy7uua7fh8z/custom-uri-override.png)</Frame>

Para excluir todo el tenant:

1. Vaya a [Auth0 Dashboard &gt; Configuración del tenant &gt; Advanced](https://manage.auth0.com/tenant/advanced).
2. Busque y desactive el toggle **Non-Verifiable Callback URI End-User Confirmation** dentro de la sección **Login and Logout** y seleccione **Save**. Es posible que tenga que seleccionar **Turn on** para poder administrar esta configuración de forma permanente.

<Frame>![Auth0 Dashboard > Tenant Settings > Advanced](/docs/images/cdy7uua7fh8z/migrate-uri-setting.png)</Frame>

También puede configurar el comportamiento requerido del tenant mediante la Auth0 Management API. En concreto, puede realizar la configuración en dos niveles:

* **Configuración a nivel de tenant**: Puede administrar el comportamiento del prompt de confirmación estableciendo la propiedad `skip_non_verifiable_callback_uri_confirmation_prompt` mediante el endpoint [Update Tenant Settings](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings).​
* **Configuración a nivel de aplicación**: Para reemplazar la configuración a nivel de tenant en aplicaciones específicas, establezca la misma propiedad `skip_non_verifiable_callback_uri_confirmation_prompt` mediante el endpoint [Update Client](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id).

Para obtener más información y orientación sobre cómo configurar sus aplicaciones, consulte [Measures Against Application Impersonation](/docs/es/secure/security-guidance/measures-against-app-impersonation).
