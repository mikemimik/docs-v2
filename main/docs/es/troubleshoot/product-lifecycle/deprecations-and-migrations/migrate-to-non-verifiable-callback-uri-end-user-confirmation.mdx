title: Migra a Non-Verifiable Callback URI End-User Confirmation
validatedOn: 2026-02-10
---

Auth0 recomienda que todas las aplicaciones nativas que usan el [flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) hagan la transición a devoluciones de llamada basadas en HTTPS mediante [Android App Links](https://developer.android.com/training/app-links) y [Apple Universal Links](https://developer.apple.com/documentation/xcode/allowing-apps-and-websites-to-link-to-your-content). Esto refuerza la seguridad y mitiga el riesgo de suplantación de aplicaciones y de ataques de phishing.

Para entender cómo esto previene los ataques, consulta [Measures Against Application Impersonation](/docs/es/secure/security-guidance/measures-against-app-impersonation#prompt-customization).

Los tenants creados antes del 15 de octubre de 2025 mantendrán el comportamiento anterior como predeterminado hasta el 28 de abril de 2026. Después de la fecha límite de octubre, los tenants creados recientemente pueden mostrar de forma predeterminada el nuevo prompt de confirmación de inicio de sesión, con algunas excepciones según el calendario de deployment de cada entorno.

Los tenants que decidan excluirse explícitamente omitirán este prompt de forma indefinida. Esto seguirá siendo así incluso después del 28 de abril de 2026, cuando el servicio adopte el prompt de confirmación como comportamiento predeterminado y elimine el toggle de migración &quot;Unconfirmed Login with Non-Verifiable Callback URI Redirects&quot;.

<div id="how-are-you-affected">
  ## ¿Cómo le afecta?
</div>

Para las aplicaciones cliente que ya especifican, o prevén especificar, un esquema de URI personalizado o una URI de devolución de llamada de loopback, es posible que los usuarios finales tengan que confirmar explícitamente el inicio de sesión al interactuar con el nuevo prompt de confirmación de inicio de sesión. Sus usuarios finales podrían percibir este cambio como una peor experiencia de usuario.

Además, se rechazarán las solicitudes de autenticación que incluyan `prompt=none` cuando las aplicaciones usen URI de devolución de llamada no verificables y estén configuradas para usar el nuevo prompt de confirmación de inicio de sesión.

<div id="migration-tasks">
  ## Tareas de migración
</div>

**Auth0 recomienda encarecidamente adoptar URI de devolución de llamada basados en HTTPS** mediante Android App Links y Apple Universal Links siempre que sea posible para todas las aplicaciones nativas que utilicen el flujo de código de autorización.

Además, en los tenants donde el comportamiento predeterminado cambie después del 28 de abril de 2026, debe seleccionar explícitamente el comportamiento que necesita para las solicitudes de autenticación que usen esquemas de URI personalizados o URI de devolución de llamada de loopback antes de que cambie el valor predeterminado del sistema.

<div id="review-whether-your-applications-are-using-non-verifiable-callback-uris">
  ### Revise si sus aplicaciones están usando URI de devolución de llamada no verificables
</div>

En los tenants para los que el toggle de migración Unconfirmed Login with Non-Verifiable Callback URI Redirects está disponible y está habilitado, las solicitudes de autenticación que especifiquen un esquema URI personalizado o una URI de loopback **generarán un registro del tenant con un aviso de obsolescencia** a menos que haya configurado explícitamente la siguiente opción a nivel de la aplicación o del tenant:

`skip_non_verifiable_callback_uri_confirmation_prompt`

Estos registros del tenant contienen el identificador del cliente de la aplicación que realiza la solicitud. Puede supervisar estos registros del tenant desde el Auth0 Dashboard con la siguiente consulta:

```bash
type:depnotetype:depnote AND description:Unconfirmed\ Login\ with\ Non-Verifiable\ Callback\ URI\ Redirects*
```

<div id="opt-in-to-new-login-confirmation-prompt">
  ### Habilitar explícitamente el nuevo prompt de confirmación de inicio de sesión
</div>

Para habilitar explícitamente por adelantado el nuevo prompt de confirmación de inicio de sesión y reforzar la seguridad de los flujos de autenticación que usan esquemas de URI personalizados o URI de loopback, completa los siguientes pasos en tu Auth0 Dashboard:

1. Ve a [**Auth0 Dashboard &gt; Configuración del tenant &gt; Advanced**](https://manage.auth0.com/#/tenant/advanced).
2. En la sección **Migración**, desactiva el toggle **Unconfirmed Login with Non-Verifiable Callback URI Redirects**.

<Frame>![Auth0 Dashboard > Configuración del tenant > Advanced > toggle desactivado](/docs/images/cdy7uua7fh8z/unconfirmed-login-uri.png)</Frame>

<div id="opt-out-of-new-login-confirmation-prompt">
  ### No usar el nuevo prompt de confirmación de inicio de sesión
</div>

Si, después de evaluar las consideraciones de seguridad, decide no usar el nuevo prompt de confirmación de inicio de sesión, puede configurar aplicaciones específicas o todo el tenant para no adoptar este nuevo comportamiento. Puede hacerlo desde el Auth0 Dashboard.

La configuración a nivel de aplicación tiene prioridad sobre la configuración a nivel del tenant. Asegúrese de configurar primero los ajustes específicos de cada aplicación antes de cambiar la configuración a nivel del tenant para evitar cambios de comportamiento no deseados. Por ejemplo, puede que quiera omitir **Non-Verifiable Callback URI End-User Confirmation** para algunas aplicaciones específicas y, de forma predeterminada, mostrarla para otras aplicaciones, o viceversa.

Para no usarlo en aplicaciones específicas:

1. Vaya a [Auth0 Dashboard &gt; Applications &gt; Settings &gt; Advanced Settings &gt; OAuth](https://manage.auth0.com/applications/settings).
2. Localice y desactive el toggle **Non-Verifiable Callback URI End-User Confirmation** y seleccione **Save**. Es posible que deba seleccionar la opción **Override the tenant setting** para poder administrar esta configuración de forma permanente.

<Frame>![Auth0 Dashboard > Applications > Settings > Advanced](/docs/images/cdy7uua7fh8z/custom-uri-override.png)</Frame>

Para no usarlo en todo el tenant:

1. Vaya a [Auth0 Dashboard &gt; Configuración del tenant &gt; Advanced](https://manage.auth0.com/tenant/advanced).
2. Localice y desactive el toggle **Non-Verifiable Callback URI End-User Confirmation** en la sección **Login and Logout** y seleccione **Save**. Es posible que deba seleccionar **Turn on** para poder administrar esta configuración de forma permanente.

<Frame>![Auth0 Dashboard > Configuración del tenant > Advanced](/docs/images/cdy7uua7fh8z/migrate-uri-setting.png)</Frame>

También puede configurar el comportamiento requerido del tenant mediante la Auth0 Management API. En concreto, puede realizar la configuración en dos niveles:

* **Configuración a nivel del tenant**: Puede administrar el comportamiento del prompt de confirmación estableciendo la propiedad `skip_non_verifiable_callback_uri_confirmation_prompt` mediante el endpoint [Update Configuración del tenant](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings).​
* **Configuración a nivel de aplicación**: Para sobrescribir la configuración a nivel del tenant en aplicaciones específicas, establezca la misma propiedad `skip_non_verifiable_callback_uri_confirmation_prompt` mediante el endpoint [Update Client](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id).

Para obtener más información y orientación sobre cómo configurar sus aplicaciones, consulte [Measures Against Application Impersonation](/docs/es/secure/security-guidance/measures-against-app-impersonation).
