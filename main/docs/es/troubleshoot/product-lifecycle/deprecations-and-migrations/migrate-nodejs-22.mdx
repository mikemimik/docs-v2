---
description: Describe los elementos de Auth0 Actions afectados por Node.js 18
title: Migrar de Node 18 a Node 22
---

A partir del 15 de enero de 2025, el runtime de extensibilidad de Node 22 está disponible de forma general (GA) en toda nuestra oferta de extensibilidad. Esto incluye Actions, Rules, Hooks, scripts de base de datos y conexiones sociales personalizadas. Te recomendamos encarecidamente que actualices a Node 22 lo antes posible para seguir las mejores prácticas de seguridad del código.

<Warning>
  El código a nivel de módulo de tus Actions ahora se invoca una vez por instancia de tu Action y puede reutilizarse según sea necesario en múltiples ejecuciones.
</Warning>

<div id="general-considerations">
  ### Consideraciones generales
</div>

* Para consultar los módulos disponibles en el nuevo runtime, puedes visitar [Can I require: Auth0 Extensibility](https://auth0-extensions.github.io/canirequire/).
* La directiva `use npm` ya no está disponible en Node 22.
* Se ha deshabilitado el acceso de escritura al sistema de archivos.

<div id="marketplace-integrations">
  ### Integraciones del Marketplace
</div>

<div id="social-connections-integrations">
  #### Integraciones de Social Connections
</div>

Usa la [Management API](https://auth0.com/docs/api/management/v2) para obtener una lista completa de las conexiones sociales afectadas por un cambio de versión del runtime de Node. En particular, todas las conexiones sociales potencialmente afectadas, ya sea creadas explícitamente como una conexión social personalizada o añadidas inicialmente a través del Marketplace, tienen el atributo `strategy` con un valor de `oauth1` o `oauth2`.

Después, puedes recorrer de forma paginada todas las conexiones sociales personalizadas existentes en un tenant determinado mediante el endpoint [GET all connections](https://auth0.com/docs/api/management/v2#!/Connections/get_connections). Por ejemplo, las siguientes opciones de consulta devuelven los nombres y los identificadores de hasta 100 conexiones sociales personalizadas:

```http wrap lines
/api/v2/connections?strategy=oauth1&strategy=oauth2&include_totals=true&fields=name&per_page=100
```

<Note>
  El <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> no permite actualizar los scripts de las conexiones sociales personalizadas añadidas a través de Marketplace. Si es necesario cambiar un script para que sea compatible con Node 22, debe usar la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.
</Note>

<div id="migration-tasks">
  ## Tareas de migración
</div>

<div id="create-new-custom-actions">
  ### Crear nuevas Actions personalizadas
</div>

Para crear una nueva Action personalizada con Node 22 desde el Auth0 Dashboard:

1. Ve a [Auth0 Dashboard &gt; Actions &gt; Biblioteca](https://manage.auth0.com/#/dashboard/actions/library).
2. Selecciona **Create Action &gt; Build from scratch**.
3. En el campo **Runtime***, selecciona **Node 22 (Recommended)**.
4. Escribe tus Actions personalizadas en Node 22, pruébalas e implántalas cuando estén listas.

<div id="upgrade-existing-custom-actions">
  ### Actualizar las Actions personalizadas existentes
</div>

Puede actualizar individualmente las Actions personalizadas existentes creadas en Node 12, 16 o 18 a Node 22 y [revertir a la versión anterior](/docs/es/customize/actions/manage-versions#revert-actions) usando el runtime anterior. Para actualizar las Actions a Node 22, cree y despliegue una nueva versión de la implementación existente con los cambios necesarios y configúrela para usar Node 22 como runtime.

<div id="choose-node-22-for-other-extensibility-products">
  ### Elegir Node 22 para otras funcionalidades de extensibilidad
</div>

El runtime que se usa para las funcionalidades de extensibilidad restantes (que no son Actions) se define globalmente a nivel de la configuración avanzada del tenant. Cambiar esta configuración afecta a la siguiente funcionalidad al mismo tiempo:

* Rules
* Hooks
* scripts de Custom DB
* scripts de conexiones sociales personalizadas

Para cambiar la configuración del runtime de extensibilidad del tenant en el Auth0 Dashboard:

1. Ve a **Dashboard &gt; Settings &gt; Advanced**.
2. Desplázate hasta la sección **Extensibilidad**.
3. En **Runtime**, selecciona **Node 22**.
4. En **Legacy Extensibility**, selecciona la versión de Runtime, si corresponde.

Dado que esta es una configuración global que afecta simultáneamente a varias funcionalidades de extensibilidad, te recomendamos realizar este paso primero en tu tenant de desarrollo, completar las pruebas de todas las funcionalidades de extensibilidad que correspondan y pasar a tu tenant de producción solo cuando no detectes problemas en desarrollo.

En el caso específico de los scripts de Custom DB, puedes [seguir los pasos que se explican en esta página](/docs/es/authenticate/database-connections/custom-db/custom-database-connections-scripts/environment) para verificar individualmente un script con una versión específica del runtime antes de proceder a cambiar la versión global del runtime.
