title: Migra a los scopes de opciones de conexión de Management API
---

​Auth0 está incorporando dos nuevos [scopes](/docs/es/get-started/apis/scopes) de Management API para las opciones de conexión: `read:connections_options` y `update:connections_options`.

Además, estamos dejando obsoleta la posibilidad de acceder al campo `options` o actualizarlo sin los scopes adecuados, lo que da lugar a los siguientes cambios no retrocompatibles:

* Las respuestas de los endpoints [Get All Connections](/docs/es/api/management/v2/connections/get-connections) y [Get Connection by Id](/docs/es/api/management/v2/connections/get-connections-by-id) no incluyen el campo `options` a menos que quien realiza la solicitud tenga el scope `read:connections_options`.
* Las solicitudes al endpoint [Update a Connection](/docs/es/api/management/v2/connections/patch-connections-by-id) fallan si el cuerpo de la solicitud contiene el campo `options` y quien realiza la solicitud no tiene el scope `update:connections_options`. Las respuestas exitosas no incluyen el campo `options` a menos que quien realiza la solicitud tenga el scope `read:connections_options`.

Para seguir usando el campo `options`, debes actualizar tus concesiones de aplicación para incluir los nuevos scopes. Sigue los pasos a continuación para identificar si tu tenant y tus integraciones se ven afectados, migrar a los scopes específicos para opciones y, por último, desactivar el comportamiento obsoleto.

<div id="identify-affected-tenants">
  ## Identifique los tenants afectados
</div>

Use Auth0 Dashboard para confirmar si un tenant puede acceder a las opciones de conexión sin los nuevos scopes y si requiere migración.

1. Vaya a [Auth0 Dashboard &gt; Configuración del tenant &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Desplácese hacia abajo hasta la sección **Migración**.
3. Busque el toggle **Allow Connections Management** without Options Scopes:
   A. Si el toggle está **activado**, su tenant aún puede acceder a las opciones de conexión sin los nuevos scopes, y debe migrar antes de la fecha límite.
   B. Si el toggle no aparece o está **desactivado**, no es necesario realizar ninguna otra acción.

<div id="identify-affected-integrations">
  ## Identifica las integraciones afectadas
</div>

Revisa los registros del tenant `depnote` para determinar qué aplicaciones e integraciones usan los endpoints afectados. La siguiente consulta de ejemplo muestra cómo consultar los registros del tenant asociados al aviso de obsolescencia correspondiente a esta obsolescencia:

```
type:depnote AND description:Allow\ Connections\ Management\ without\ Options\ Scopes*
```

Para obtener más información, consulta [sintaxis de las consultas de búsqueda en logs](docs/deploy-monitor/logs/log-search-query-syntax).

Una vez que tengas los resultados de la consulta, inspecciona el campo `client_id` para identificar cada aplicación que podría verse afectada. Debido a la naturaleza de los cambios, la existencia de un aviso de obsolescencia para una aplicación cliente concreta no significa necesariamente que la aplicación se vea afectada.

​Si el campo `client_id` tiene un valor vacío, esto significa que la solicitud al endpoint afectado provino de una extensión. Varias extensiones de Auth0 pueden generar registros del tenant con avisos de obsolescencia que son falsos positivos. Una vez que completes los pasos específicos de cada extensión que se indican en la sección Update extensions, puedes ignorar los avisos de obsolescencia con un identificador de cliente vacío.

<div id="update-application-grants">
  ## Actualizar las concesiones de aplicación
</div>

Para las aplicaciones que recuperan o actualizan el campo `options` mediante los endpoints afectados, actualiza sus concesiones de la Management API para incluir los scopes necesarios.

1. Ve a [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/dashboard/#/apis).
2. Selecciona **Auth0 Management API** y luego la pestaña **Machine to Machine Applications**.
3. Junto a la aplicación que requiere actualización, selecciona el icono de flecha hacia abajo para expandir la sección.
4. En la sección Permissions, busca `“connections_options”`. Habilita uno o ambos scopes nuevos. Luego, selecciona **Update** para guardar los cambios.
   <Frame>![Actualizar permisos de la Management API](/docs/images/cdy7uua7fh8z/update-connections-permissions.png)</Frame>

<div id="update-extensions">
  ## Actualizar extensiones
</div>

Actualiza la extensión [Dashboard de administración delegada](/docs/es/customize/extensions/delegated-administration-extension/install-delegated-admin-extension) a la versión más reciente. Esta actualización es necesaria porque la extensión requiere acceso al campo de opciones de conexión para funcionar correctamente. Si usas esta extensión, asegúrate de mantenerla actualizada para que siga funcionando correctamente.

<div id="opt-out-finalize-migration">
  ## Exclusión voluntaria (finalizar la migración)
</div>

Una vez que tus integraciones dejen de depender del comportamiento obsoleto, te recomendamos configurar tu tenant para dejar explícitamente de permitir la administración de conexiones sin scopes de opciones. Esto te permite elegir exactamente cuándo tu tenant adopta el nuevo comportamiento, lo que te da más control sobre tu proceso de migración.

1. Ve a [Auth0 Dashboard &gt; Configuración del tenant &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. En la sección Migración, desactiva la opción **Allow Connections Management without Options Scopes**.

Si surge alguna incidencia, puedes volver a activar el mismo interruptor para restaurar temporalmente el comportamiento anterior mientras solucionas el problema.
