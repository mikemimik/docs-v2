title: Migra a scopes de opciones de conexión de Management API
---

​Auth0 está incorporando dos nuevos [scopes](/docs/es/get-started/apis/scopes) de Management API para las opciones de conexión: `read:connections_options` y `update:connections_options`.

Además, estamos dejando obsoleta la capacidad de acceder al campo `options` o actualizarlo sin los scopes adecuados, lo que da lugar a los siguientes cambios incompatibles:

* Las respuestas de los endpoints [Get All Connections](/docs/es/api/management/v2/connections/get-connections) y [Get Connection by Id](/docs/es/api/management/v2/connections/get-connections-by-id) no incluyen el campo `options`, a menos que quien realiza la llamada tenga el scope `read:connections_options`.
* Las solicitudes al endpoint [actualizar una conexión](/docs/es/api/management/v2/connections/patch-connections-by-id) fallan si el cuerpo de la solicitud contiene el campo `options` y quien realiza la llamada no tiene el scope `update:connections_options`. Las respuestas exitosas no incluyen el campo `options`, a menos que quien realiza la llamada tenga el scope `read:connections_options`.

Para seguir usando el campo `options`, debes actualizar tus concesiones de aplicación para incluir los nuevos scopes. Sigue los pasos a continuación para identificar si tu tenant y tus integraciones se ven afectados, migrar a los scopes específicos para las opciones y, por último, desactivar este comportamiento obsoleto.

<div id="identify-affected-tenants">
  ## Identifica los tenants afectados
</div>

Usa Auth0 Dashboard para confirmar si un tenant puede acceder a las opciones de conexión sin los nuevos scopes y si requiere migración.

1. Ve a [Auth0 Dashboard &gt; Configuración del tenant &gt; Avanzado](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Desplázate hacia abajo hasta la sección **Migraciones**.
3. Busca el interruptor **Allow Connections Management** without Options Scopes:
   A. Si el interruptor está **activado**, tu tenant todavía puede acceder a las opciones de conexión sin los nuevos scopes y debes migrarlo antes de la fecha límite.
   B. Si el interruptor no aparece o está **desactivado**, no se requiere ninguna acción adicional.

<div id="identify-affected-integrations">
  ## Identifica las integraciones afectadas
</div>

Revisa los logs `depnote` del tenant para determinar qué aplicaciones e integraciones usan los endpoints afectados. La siguiente consulta de ejemplo muestra cómo consultar los logs del tenant relacionados con esta obsolescencia:

```
type:depnote AND description:Allow\ Connections\ Management\ without\ Options\ Scopes*
```

Para obtener más información, lee [sintaxis de las consultas de búsqueda en logs](docs/deploy-monitor/logs/log-search-query-syntax).

Una vez que tengas los resultados de la consulta, inspecciona el campo `client_id` para identificar cada aplicación que pueda estar potencialmente afectada. Debido a la naturaleza de los cambios, la presencia de un aviso de obsolescencia para una aplicación cliente específica no significa necesariamente que la aplicación esté afectada.

​Si el campo `client_id` tiene un valor vacío, esto significa que la solicitud al endpoint afectado se originó en una extensión. Varias extensiones de Auth0 pueden generar avisos de obsolescencia en los logs del tenant que sean falsos positivos. Una vez que completes los pasos específicos de la extensión que se enumeran en la sección Actualizar extensiones, puedes ignorar los avisos de obsolescencia con un identificador de cliente vacío.

<div id="update-application-grants">
  ## Actualizar concesiones de aplicación
</div>

Si alguna aplicación recupera o actualiza el campo `options` mediante los endpoints afectados, actualiza sus concesiones para la Management API para incluir los scopes necesarios.

1. Ve a [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/dashboard/#/apis).
2. Selecciona **Auth0 Management API** y, después, la pestaña **Machine to Machine Applications**.
3. Junto a la aplicación que necesita actualizarse, selecciona el icono de flecha hacia abajo para expandir la sección.
4. En la sección **Permissions**, busca `“connections_options”`. Habilita uno o ambos scopes nuevos. Luego, selecciona **Update** para guardar los cambios.
   <Frame>![Actualizar permisos de la Management API](/docs/images/cdy7uua7fh8z/update-connections-permissions.png)</Frame>

<div id="update-extensions">
  ## Actualizar extensiones
</div>

Actualiza a la versión más reciente de la extensión [Dashboard de Administración delegada](/docs/es/customize/extensions/delegated-administration-extension/install-delegated-admin-extension). Esta actualización es necesaria porque la extensión necesita acceso al campo de opciones de la conexión para funcionar correctamente. Si usas esta extensión, asegúrate de actualizarla para mantener su funcionamiento.

<div id="opt-out-finalize-migration">
  ## Desactivación (finalizar la migración)
</div>

Una vez que tus integraciones dejen de depender del comportamiento obsoleto, te recomendamos configurar tu tenant para desactivar explícitamente la gestión de conexiones sin scopes de `options`. Esto te permite elegir exactamente cuándo tu tenant adopta el nuevo comportamiento, lo que te da más control sobre tu proceso de migración.

1. Ve a [Auth0 Dashboard &gt; Configuración del tenant &gt; Avanzado](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. En la sección Migración, desactiva la opción **Allow Connections Management without Options Scopes**.

Si surge algún problema, puedes revertir esa misma opción para restaurar temporalmente el comportamiento anterior mientras lo solucionas.
