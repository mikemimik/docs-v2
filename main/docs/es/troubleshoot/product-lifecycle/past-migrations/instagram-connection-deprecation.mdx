---
description: Describe los cambios derivados de la obsolescencia de la Authentication API de Instagram.
title: Obsolescencia de la conexión de Instagram
---

A partir del 31 de marzo de 2020, Facebook desactivó las API heredadas de Instagram en favor de un nuevo conjunto de API:

* [Instagram Graph API](https://developers.facebook.com/docs/instagram-api), diseñada para cuentas profesionales de Instagram, no para la autenticación de usuarios finales.
* [Instagram Basic Display API](https://developers.facebook.com/docs/instagram-basic-display-api), una API de OAuth2 que le permite otorgar acceso a los datos básicos de su cuenta de Instagram a una aplicación de terceros.

Facebook exige que las aplicaciones implementen Facebook Login para la autenticación. Facebook no aprobará aplicaciones que utilicen la Instagram Basic Display API para la autenticación.

<div id="actions">
  ## Actions
</div>

Para que tus usuarios actuales puedan seguir accediendo a tu aplicación, tendrás que pedirles a los usuarios que se autentican con Instagram que lo hagan de otra forma y usar Account Linking para vincular la nueva identidad con la anterior.

Un ejemplo de flujo sería:

* El usuario se autentica con Instagram.
* La aplicación le informa al usuario que ya no podrá autenticarse con Instagram y que deberá hacerlo de otra manera.
* La aplicación muestra las opciones de autenticación disponibles para el usuario, por ejemplo:

  * Facebook
  * Nombre de usuario y contraseña
* Después de que el usuario se autentique de otra manera, vinculas las cuentas con Account Linking.

Aunque podríamos sustituir nuestra implementación actual y usar el [flujo de OAuth de Instagram Basic Display](https://developers.facebook.com/docs/instagram-basic-display-api/guides/getting-access-tokens-and-permissions), Facebook no lo aceptaría según sus políticas. Tendrías que crear una aplicación de Instagram en Facebook y, en esa aplicación, aparece una notificación que dice:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Ten en cuenta que Basic Display no es una herramienta de autenticación. Los datos que devuelve la API no pueden usarse para autenticar a los usuarios de tu aplicación ni para iniciar sesión en ella. Si tu aplicación usa datos de la API para autenticar usuarios, será rechazada durante la revisión de la aplicación. Si necesitas una solución de autenticación, usa Facebook Login en su lugar.
</Callout>

Esto significa que, incluso si Auth0 implementara este flujo, tu aplicación de Instagram no sería aprobada por Facebook.

Si necesitas acceder a datos de Instagram, tendrás que autenticar al usuario de otra forma (por ejemplo, con Facebook Login o con nombre de usuario/contraseña) e implementar el flujo de Instagram <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip> en tu aplicación.
