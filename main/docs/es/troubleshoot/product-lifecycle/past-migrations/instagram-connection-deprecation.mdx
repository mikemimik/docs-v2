---
description: Describe los cambios que se producen porque Instagram ha dejado obsoleta su Authentication API.
title: Obsolescencia de la conexión de Instagram
---

A partir del 31 de marzo de 2020, Facebook desactivó las API heredadas de Instagram y las sustituyó por un nuevo conjunto de API:

* [Instagram Graph API](https://developers.facebook.com/docs/instagram-api), diseñada para cuentas profesionales de Instagram, no para la autenticación de usuarios finales.
* [Instagram Basic Display API](https://developers.facebook.com/docs/instagram-basic-display-api), una API de OAuth2 que le permite conceder a una aplicación de terceros acceso a los datos básicos de su cuenta de Instagram.

Facebook exige que las aplicaciones implementen Facebook Login para la autenticación. Facebook no aprobará aplicaciones que usen Instagram Basic Display API para fines de autenticación.

<div id="actions">
  ## Actions
</div>

Para que tus usuarios actuales puedan seguir accediendo a tu aplicación, tendrás que pedirles a los usuarios que se autentican con Instagram que lo hagan de otra forma y usar Account Linking para vincular la nueva identidad con la anterior.

Un flujo de ejemplo sería:

* El usuario se autentica con Instagram.
* La aplicación le informa al usuario de que ya no podrá autenticarse con Instagram y de que deberá hacerlo de otra forma.
* La aplicación muestra las opciones de autenticación disponibles para el usuario, por ejemplo:

  * Facebook
  * Nombre de usuario y contraseña
* Después de que el usuario se autentique de otra forma, vinculas las cuentas con Account Linking.

Aunque podríamos sustituir nuestra implementación actual y usar el [flujo de OAuth de Instagram Basic Display](https://developers.facebook.com/docs/instagram-basic-display-api/guides/getting-access-tokens-and-permissions), Facebook no lo aceptaría según sus políticas. Tendrías que crear una aplicación de Instagram en Facebook y, en esa aplicación, aparece una notificación que dice:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Ten en cuenta que Basic Display no es una herramienta de autenticación. Los datos devueltos por la API no se pueden usar para autenticar a los usuarios de tu aplicación ni para iniciar sesión en ella. Si tu aplicación usa datos de la API para autenticar usuarios, será rechazada durante la revisión de la aplicación. Si necesitas una solución de autenticación, usa Facebook Login en su lugar.
</Callout>

Esto significa que, aunque Auth0 implementara este flujo, Facebook no aprobaría tu aplicación de Instagram.

Si necesitas acceder a datos de Instagram, tendrás que autenticar al usuario de otra forma (por ejemplo, con Facebook Login o nombre de usuario y contraseña) e implementar en tu aplicación el flujo de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip> de Instagram.
