---
description: Describe los cambios derivados de la obsolescencia de la Authentication API de Instagram.
title: Obsolescencia de la conexión de Instagram
---

A partir del 31 de marzo de 2020, Facebook desactivó las API heredadas de Instagram a favor de un nuevo conjunto de API:

* [Instagram Graph API](https://developers.facebook.com/docs/instagram-api), diseñada para cuentas profesionales de Instagram, no para la autenticación de usuarios finales.
* [Instagram Basic Display API](https://developers.facebook.com/docs/instagram-basic-display-api), una API de OAuth2 que le permite conceder acceso a los datos básicos de su cuenta de Instagram a una aplicación de terceros.

Facebook exige que las aplicaciones implementen Facebook Login para la autenticación. Facebook no aprobará aplicaciones que usen la Instagram Basic Display API para la autenticación.

<div id="actions">
  ## Actions
</div>

Para que sus usuarios actuales puedan seguir accediendo a su aplicación, deberá pedirles a quienes se autentican con Instagram que lo hagan de otra forma y usar Account Linking para vincular la nueva identidad con la anterior.

Un flujo de ejemplo sería:

* El usuario se autentica con Instagram.
* La aplicación le indica al usuario que ya no podrá autenticarse con Instagram y que deberá hacerlo de otra forma.
* La aplicación muestra las opciones de autenticación disponibles para el usuario, por ejemplo:

  * Facebook
  * Nombre de usuario y contraseña
* Después de que el usuario se autentique de otra forma, usted vincula las cuentas mediante Account Linking.

Aunque podríamos reemplazar nuestra implementación actual y usar el [flujo de OAuth de Instagram Basic Display](https://developers.facebook.com/docs/instagram-basic-display-api/guides/getting-access-tokens-and-permissions), Facebook no lo aceptaría de acuerdo con sus políticas. Tendría que crear una aplicación de Instagram en Facebook y, en ella, aparece una notificación que dice:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Tenga en cuenta que Basic Display no es una herramienta de autenticación. Los datos devueltos por la API no pueden usarse para autenticar a los usuarios de su aplicación ni para iniciar sesión en ella. Si su aplicación usa datos de la API para autenticar usuarios, será rechazada durante la revisión de la aplicación. Si necesita una solución de autenticación, use Facebook Login en su lugar.
</Callout>

Esto significa que, aunque Auth0 implementara este flujo, Facebook no aprobaría su aplicación de Instagram.

Si necesita acceder a datos de Instagram, deberá autenticar a su usuario de otra manera (por ejemplo, con Facebook Login o nombre de usuario/contraseña) e implementar el flujo de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip> de Instagram en su aplicación.
