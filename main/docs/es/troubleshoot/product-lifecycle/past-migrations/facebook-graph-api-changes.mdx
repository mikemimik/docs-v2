title: Cambios en Facebook Graph API
---

A partir del 1 de agosto de 2018, Facebook cambió los permisos y campos de Facebook Graph API que pueden solicitarse. Auth0 ha actualizado las Conexiones de Facebook para reflejar estos cambios y ha modificado la interfaz de conexión para que resulte más clara. Consulta [Facebook Login Changelog: Recent Changes to Facebook Login](https://developers.facebook.com/docs/facebook-login/changelog#2018-07-02) para conocer todos los detalles y las fechas clave.

<div id="features-affected">
  ## Funcionalidades afectadas
</div>

Es posible que esta actualización no requiera cambios en tu código ni en tu configuración, pero tu aplicación podría recibir datos de perfil adicionales si los permisos existentes lo permiten. Ten en cuenta lo siguiente:

* Si tu conexión de Facebook está configurada para solicitar uno de los permisos eliminados, tu <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> no los incluirá en el scope.
* Si tu aplicación de Facebook está marcada como &quot;development&quot;, es posible que temporalmente sigas viendo un error al probar la conexión.
* Si agregas nuevos permisos a la conexión, se pedirá el consentimiento de los usuarios finales la próxima vez que inicien sesión. Consulta la documentación de Facebook para saber cómo gestionar los casos de usuarios que no tienen un permiso específico.

<div id="facebook-login-permissions">
  ### Permisos de Facebook Login
</div>

Tu aplicación solicita los [permisos de Facebook Login](https://developers.facebook.com/docs/facebook-login/permissions) cuando un usuario inicia sesión con Facebook. Si es la primera vez que el usuario inicia sesión o si los permisos han cambiado, se le mostrará una ventana de consentimiento en Facebook con los nuevos permisos solicitados. Una vez concedidos esos permisos, tu aplicación podrá actuar en nombre de ese usuario con un token de acceso de Facebook.

La interfaz de conexión de Facebook se ha actualizado para mostrar tanto el nombre común como el nombre interno de todos los permisos que se muestran. Esto facilita encontrar los permisos que necesitas y hacerlos corresponder con cualquier código que puedas estar ejecutando con esos nombres de permisos.

<div id="permissions-moved-to-deprecated">
  #### Permisos trasladados a Obsoleto
</div>

Los siguientes permisos se trasladaron a la sección **Obsoleto** y no deben usarse con la versión más reciente de la Graph API:

* publish&#95;actions
* user&#95;managed&#95;groups

<div id="facebook-graph-api-fields">
  ### Campos de la Facebook Graph API
</div>

La [Facebook Graph API](https://developers.facebook.com/docs/graph-api/reference/v3.2/user) se usa después de que un usuario inicia sesión para recuperar los datos del perfil del usuario de Auth0. Los permisos de datos de usuario solicitados determinan qué información se recupera de la Graph API. Los campos que se devuelven dependen de los permisos solicitados y de que esos campos existan en el perfil de Facebook del usuario.

Este cambio actualizó la Graph API de la versión 2.8 a la 3.2 y solicitará los siguientes campos de datos de usuario al iniciar sesión:

* address (added)
* age&#95;range
* birthday
* context
* cover
* currency (added)
* devices
* email
* favorite&#95;athletes
* favorite&#95;teams
* first&#95;name
* gender
* hometown
* id
* inspirational&#95;people
* install&#95;type (added)
* installed
* is&#95;verified
* languages
* last&#95;name
* link
* locale
* location
* meeting&#95;for (added)
* middle&#95;name
* name
* name&#95;format
* picture
* public&#95;key (added)
* quotes
* security&#95;settings (added)
* short&#95;name (added)
* significant&#95;other
