---
description: Describe los cambios causados por la eliminación por parte de Facebook del acceso al campo Social Context de su perfil.
title: Obsolescencia del campo Social Context de Facebook
---

A partir del **30 de julio de 2020**, las conexiones de Facebook que soliciten el campo `context` fallarán, por lo que Auth0 dejará de solicitarlo para todas las conexiones.

El 30 de abril, [Facebook dejó obsoleto](https://developers.facebook.com/docs/graph-api/changelog/4-30-2019-endpoint-deprecations) el uso del campo **Social Context** para las nuevas aplicaciones. Auth0 siguió solicitando ese campo de forma predeterminada para las conexiones de Facebook creadas antes del 30 de abril de 2019. Puede asegurarse de que el campo no se solicite antes del 30 de julio desmarcando el campo **Social context** en la sección User Data de la conexión:

<Frame>![Obsolescencia del campo Context de Facebook](/docs/images/cdy7uua7fh8z/4oi3foRuwYYOveDlq2vJM/a57cd954ceeaf6d7722796be7c743b76/facebook-context.png)</Frame>

Una vez que desmarque **Social context**, los datos del perfil no incluirán el campo `context`. El campo tiene el siguiente contenido:

```text lines
"context": {
      "mutual_likes": {"data": [],"summary": {"total_count": 0}},
      "id": "dXNlcl9...UZD"
    }
```

<div id="actions">
  ## Actions
</div>

Si no estás usando el campo `context` del perfil de Facebook que Auth0 devuelve en tu aplicación, tu aplicación seguirá funcionando sin cambios. De lo contrario, tendrás que ajustar el código de tu aplicación para que no dependa de él.

Para asegurarte de que tu aplicación no se vea afectada el 30 de julio, te recomendamos desmarcar el campo **Social context** en las propiedades de la conexión de Facebook.
