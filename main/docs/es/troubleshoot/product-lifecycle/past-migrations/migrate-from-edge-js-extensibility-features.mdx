---

A partir del 21 de junio de 2023, Auth0 dejará de ofrecer soporte para las funcionalidades de extensibilidad de Node.js para .NET y C#.

<div id="affected-workflows">
  ## Flujos de trabajo afectados
</div>

Revise sus funcionalidades de extensibilidad para detectar código que contenga `require(edge)`.

Revise los siguientes flujos o funcionalidades que permiten código de extensibilidad en busca de referencias a `edge.js`:

* [Hooks](/docs/es/customize/hooks)
* [Rules](/docs/es/customize/rules)
* [Custom Database Scripts](/docs/es/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Custom Social Connections](/docs/es/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Actualizaciones recomendadas
</div>

No hay un reemplazo oficial previsto para esta funcionalidad.

Una opción de migración es sustituir la lógica de `edge` por un servicio independiente dentro de tu propia infraestructura. Después de crear el servicio en tu infraestructura, puedes usar una función de extensibilidad para hacer una solicitud HTTP que active la operación necesaria.

Por ejemplo, en un script de Login de Custom Database que usa `edge`, elimina la función que llama a `edge` y usa un cliente HTTP genérico para capturar la respuesta.

Otra opción de migración es reescribir la funcionalidad de extensibilidad en JavaScript en lugar de .NET/C#.

Para obtener más información sobre cómo crear Actions o migrar a Actions, [consulta la documentación de Auth0](/docs/es/customize/actions).
