---
description: Describe los cambios en la API de Yahoo que afectan a la estructura del perfil de usuario.
title: Cambios en la API de Yahoo
---

Yahoo cambió la API que las aplicaciones deben usar para obtener el perfil de usuario, pasando de su [Social Directory API](https://developer.yahoo.com/oauth/social-directory-eol/) a un endpoint de Yahoo `/userinfo`. Este cambio implica que la estructura del perfil de usuario de los usuarios de Yahoo en Auth0 cambiará.

Anteriormente, Auth0 cargaba todos los datos del perfil que devolvía Yahoo y agregaba estos campos adicionales asignados desde el perfil de Yahoo.

Yahoo dejó de devolver `url`, `profileUrl`, `isConnected` y otros campos enumerados en la [documentación de Yahoo](https://developer.yahoo.com/oauth/social-directory-eol/) (consulta &quot;List Of Attributes Deprecated in Social Directory Profile Api&quot;). Esos otros campos tampoco formarán parte del perfil.

El endpoint de Yahoo `/userinfo` devuelve distintos campos según los permisos de API que configures en la definición de la [Yahoo Application](https://developer.yahoo.com/apps/). Yahoo te permite conceder uno de estos cuatro permisos en la sección de permisos **Profile (Social Directory)**:

* Leer Público Básico
* Read Public Extended
* Leer y escribir Público
* Leer y escribir Público y Privado

<div id="actions">
  ## Actions
</div>

Al configurar la conexión de Yahoo en su <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, seleccione el atributo que corresponda a los permisos que concedió en la configuración de Yahoo. Si elige un atributo que no coincide con lo que especificó en Yahoo, la transacción de inicio de sesión fallará.

Si no selecciona ningún permiso en la configuración de la conexión de Auth0, Auth0 solicitará de forma predeterminada el scope `openid`, que devuelve los campos del perfil que corresponden al permiso de API que especificó en la Yahoo Application. Por ejemplo, si su aplicación de Yahoo está configurada con el permiso **Read Public Extended**, Yahoo devuelve los siguientes campos:

* `sub, name`
* `given_name`
* `family_name`
* `locale`
* `email`
* `email_verified`
* `birthdate`
* `profile_images`
* `picture`
* `preferred_username`
* `phone_number`
* `nickname`

Si está usando la conexión de Yahoo para autenticar a los usuarios y obtener su información básica, su aplicación seguirá funcionando sin cambios.

Si su aplicación accede a campos del perfil de usuario que ya no están disponibles, deberá habilitar el **Atributo de conexión** correcto en el Auth0 Dashboard y ajustar el código de su aplicación para usar los nombres de campo adecuados.
