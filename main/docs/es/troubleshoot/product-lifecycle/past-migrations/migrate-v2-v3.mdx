title: Migrar de Search v2 a v3
---

User Search v2 alcanzó su fin de vida útil el **30 de junio de 2019**. Recomendamos encarecidamente migrar la funcionalidad de búsqueda de usuarios al [motor de búsqueda v3](/docs/es/manage-users/user-search) (`search_engine=v3`) lo antes posible.

<div id="migration-considerations">
  ## Consideraciones sobre la migración
</div>

Antes de comenzar la migración, hay algunas cosas que debe saber:

* Para asegurarse de que sus consultas usen el motor de búsqueda v3 antes de que la v2 deje de estar disponible, debe actualizar todas sus llamadas al endpoint `GET /api/v2/users` para incluir el parámetro `search_engine=v3`. Esto le permitirá ver si es necesario actualizar alguna consulta, para que no experimente tiempo de inactividad cuando la v2 deje de estar disponible.
* Si realiza operaciones de búsqueda de usuarios a través de cualquiera de los [SDKs afectados](#impacted-sdks), también debe pasar el parámetro `search_engine=v3`, como se indicó anteriormente.
* Los valores de búsqueda de los campos de usuario normalizados (`email`, `name`, `given_name`, `family_name` y `nickname`) no distinguen entre mayúsculas y minúsculas. Todos los demás campos (incluidos todos los campos de `app_metadata`/`user_metadata`) sí distinguen entre mayúsculas y minúsculas.
* La v3 limita a 1000 la cantidad de usuarios que puede recuperar. Si está alcanzando este límite, le recomendamos redefinir su consulta de búsqueda para obtener resultados más granulares. Si necesita una lista de más de 1000 usuarios en un momento dado, le recomendamos usar el endpoint de API de [Export Job](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_exports) o [User Import / Export Extension](/docs/es/manage-users/user-migration/user-import-export-extension).
* Las búsquedas por rango y con comodines no están disponibles en los campos `app_metadata`/`user_metadata`.
* Los campos de usuario no se tokenizan como en la v2, por lo que `user_id:auth0` no coincidirá con un `user_id` cuyo valor sea `auth0|12345`; en su lugar, use `user_id:auth0*`.
* Los comodines pueden usarse para coincidencias por prefijo, por ejemplo `name:j*`. Para otros usos de comodines (p. ej., coincidencias por sufijo), los literales deben tener 3 caracteres o más. Por ejemplo, `name:*usa` está permitido, pero `name:*sa` no.
* La extensión de campo `.raw` ya no es compatible y debe eliminarse. En la v3, los campos coinciden con el valor completo proporcionado y no se tokenizan como ocurría en la v2 sin el sufijo `.raw`.
* El campo `connection` no es compatible con la v3. En su lugar, debe usar su alias `identities.connection`.

<div id="queries-to-migrate">
  ## Consultas para migrar
</div>

| Caso de uso                                              | v2                                | v3                                        |
| -------------------------------------------------------- | --------------------------------- | ----------------------------------------- |
| Buscar por fecha                                         | `updated_at:>=2018-01-15`         | `updated_at:[2018-01-15 TO *]`            |
| Buscar por fecha                                         | `updated_at:>2018-01-15`          | `<br/>updated_at:\{2018-01-15 TO *]<br/>` |
| Buscar por fecha                                         | `updated_at:<=2018-01-15`         | `updated_at:[* TO 2018-01-15]`            |
| Buscar por fecha                                         | `updated_at:<2018-01-15`          | `updated_at:[* TO 2018-01-15}`            |
| Buscar por fecha                                         | `last_login:<=2017-12`            | `last_login:[* TO 2017-12]`               |
| Coincidencia exacta de String                            | `name.raw:"john richard doe"`     | `name:"john richard doe"`                 |
| La frase incluye una palabra                             | `name:"richard"`, `name:richard`  | `name:*richard*`                          |
| La frase incluye una palabra (con menos de 3 caracteres) | `name:*ri`,`name:*a`, `name:*ab*` | *(no compatible)*                         |

<div id="impacted-sdks">
  ## SDKs afectados
</div>

Los siguientes SDKs utilizan el motor User Search. Si usa alguno de ellos, asegúrese de estar usando las versiones que se indican a continuación (o una versión posterior) y de pasar el parámetro `search_engine=v3` al realizar operaciones de búsqueda de usuarios.

| SDK                                                   | Versión con soporte para v3 | Métodos afectados                              | Consideraciones                                                     |
| ----------------------------------------------------- | --------------------------- | ---------------------------------------------- | ------------------------------------------------------------------- |
| [Auth0 Java](https://github.com/auth0/auth0-java)     | 1.8.0                       | com.auth0.client.mgmt.UsersEntity.list         | Proporcione un `UserFilter` con `withSearchEngine("v3")`            |
| [Auth0 Python](https://github.com/auth0/auth0-python) | 3.0.0                       | management.Users.list                          | Proporcione el parámetro `search_engine='v3'`                       |
| [Auth0 Node](https://github.com/auth0/node-auth0)     | 2.0.0                       | UsersManager.getAll, ManagementClient.getUsers | Proporcione el parámetro `search_engine:'v3'`                       |
| [Auth0 .NET](https://github.com/auth0/auth0.net)      | 3.0.0 o 4.0.0               | Auth0.ManagementApi.IUsersClient.GetAllAsync   | Proporcione un objeto `GetUsersRequest` con `SearchEngine` = `"v3"` |
| [Auth0 PHP](https://github.com/auth0/auth0-php)       | 5.2.0                       | Auth0.SDK.API.Management.Users.getAll          | Proporcione el parámetro `'search_engine' => 'v3'`                  |
| [Auth0 Ruby](https://github.com/auth0/ruby-auth0)     | 4.5.0                       | Auth0.Api.V2.Users.users                       | Proporcione el parámetro `search_engine: 'v3'`                      |

| Extensión                                                                                | Versiones compatibles con v3 | Consideraciones                                                                                                                                                                                                                                                                |

| [Authorization Extension](/docs/es/customize/extensions/authorization-extension)            | 2.5.0+                       | Si estás usando una versión anterior, debes actualizar manualmente la extensión desde la página de [Extensions](https://manage.auth0.com/#/extensions).                                                                                                                        |
| [Administración delegada](/docs/es/customize/extensions/delegated-administration-extension) | 3.1+                         | Si estás usando una versión anterior, debes actualizar manualmente la extensión desde la página de [Extensions](https://manage.auth0.com/#/extensions). La opción de configuración `SEARCH_ENGINE` ya no existe en la versión 3.1, porque solo está disponible User Search v3. |

<div id="leverage-your-tenant-logs-to-find-usage-of-user-search-v2">
  ## Aprovecha los registros de tu tenant para identificar el uso de User Search v2
</div>

Puedes aprovechar los [registros](/docs/es/deploy-monitor/logs) en el [Dashboard](https://manage.auth0.com/#/logs) para encontrar llamadas al endpoint `/api/v2/users` que usan el motor User Search v2, incluidas las realizadas por SDKs. Esos registros te ayudarán a identificar en qué partes de tus aplicaciones podrían requerirse cambios de código.

Usa la siguiente consulta para recuperar todos los registros relacionados con User Search v2: `type:w AND description:*search_engine*`. Los registros proporcionarán información adicional en el campo description, en los siguientes casos:

* Consultas que podrían producir resultados diferentes en v3
* Consultas con sintaxis incompatible con v3
* Consultas que no cumplen los requisitos de paginación de v3

Si no se especifican detalles adicionales en las entradas de registro, es probable que tus consultas sean compatibles con v3. Aun así, nuestra recomendación sigue siendo que pruebes las consultas antes de implementar tus cambios en producción.

Ten en cuenta que solo se generará un registro del mismo tipo en un período de 60 minutos. Esto significa que, aunque hagas varias llamadas al endpoint de User Search, solo verás un registro de cada tipo por hora.
