title: Migrar de Search v2 a v3
---

User Search v2 alcanzó su fin de vida útil el **30 de junio de 2019**. Recomendamos encarecidamente migrar la funcionalidad de búsqueda de usuarios para usar [motor de búsqueda v3](/docs/es/manage-users/user-search) (`search_engine=v3`) lo antes posible.

<div id="migration-considerations">
  ## Consideraciones sobre la migración
</div>

Antes de empezar la migración, hay algunas cosas que debe tener en cuenta:

* Para garantizar que sus consultas utilicen el motor de búsqueda v3 antes de que v2 deje de estar disponible, debe actualizar todas sus llamadas al endpoint `GET /api/v2/users` para incluir el parámetro `search_engine=v3`. Esto le permitirá comprobar si es necesario actualizar alguna consulta, para no experimentar tiempo de inactividad cuando v2 deje de estar disponible.
* Si realiza operaciones de búsqueda de usuarios mediante alguno de los [SDKs afectados](#impacted-sdks), también debe incluir el parámetro `search_engine=v3` como se indicó anteriormente.
* Los valores de búsqueda de los campos de usuario normalizados (`email`, `name`, `given_name`, `family_name` y `nickname`) no distinguen entre mayúsculas y minúsculas. Todos los demás campos (incluidos todos los campos de `app_metadata`/`user_metadata`) sí distinguen entre mayúsculas y minúsculas.
* v3 limita a 1000 la cantidad de usuarios que puede recuperar. Si está alcanzando este límite, le recomendamos redefinir su consulta de búsqueda para obtener resultados más granulares. Si necesita una lista de más de 1000 usuarios en un momento dado, le recomendamos que utilice el endpoint de la API [Export Job](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_exports) o la [User Import / Export Extension](/docs/es/manage-users/user-migration/user-import-export-extension).
* Las búsquedas por rango y mediante comodines no están disponibles en los campos `app_metadata`/`user_metadata`.
* Los campos de usuario no se tokenizan como en v2, por lo que `user_id:auth0` no coincidirá con un `user_id` cuyo valor sea `auth0|12345`; en su lugar, use `user_id:auth0*`.
* Los comodines pueden usarse para coincidencias por prefijo, por ejemplo, `name:j*`. Para otros usos de los comodines (p. ej., coincidencia por sufijo), los literales deben tener 3 caracteres o más. Por ejemplo, `name:*usa` está permitido, pero `name:*sa` no.
* La extensión de campo `.raw` ya no es compatible y debe eliminarse. En v3, los campos coinciden con el valor completo proporcionado y no se tokenizan como ocurría en v2 sin el sufijo `.raw`.
* El campo `connection` no es compatible en v3. En su lugar, debe usar su alias `identities.connection`.

<div id="queries-to-migrate">
  ## Consultas para migrar
</div>

| Caso de uso                                              | v2                                | v3                                        |
| -------------------------------------------------------- | --------------------------------- | ----------------------------------------- |
| Búsqueda por fecha                                       | `updated_at:>=2018-01-15`         | `updated_at:[2018-01-15 TO *]`            |
| Búsqueda por fecha                                       | `updated_at:>2018-01-15`          | `<br/>updated_at:\{2018-01-15 TO *]<br/>` |
| Búsqueda por fecha                                       | `updated_at:<=2018-01-15`         | `updated_at:[* TO 2018-01-15]`            |
| Búsqueda por fecha                                       | `updated_at:<2018-01-15`          | `updated_at:[* TO 2018-01-15}`            |
| Búsqueda por fecha                                       | `last_login:<=2017-12`            | `last_login:[* TO 2017-12]`               |
| Coincidencia exacta de String                            | `name.raw:"john richard doe"`     | `name:"john richard doe"`                 |
| La frase incluye una palabra                             | `name:"richard"`, `name:richard`  | `name:*richard*`                          |
| La frase incluye una palabra (con menos de 3 caracteres) | `name:*ri`,`name:*a`, `name:*ab*` | *(no es compatible)*                      |

<div id="impacted-sdks">
  ## SDKs afectados
</div>

Los siguientes SDKs utilizan el motor User Search. Si los utiliza, asegúrese de usar las versiones que se indican a continuación (o una posterior) y de pasar el parámetro `search_engine=v3` al realizar operaciones de búsqueda de usuarios.

| SDK                                                   | Versión con soporte para v3 | Métodos afectados                              | Consideraciones                                                     |
| ----------------------------------------------------- | --------------------------- | ---------------------------------------------- | ------------------------------------------------------------------- |
| [Auth0 Java](https://github.com/auth0/auth0-java)     | 1.8.0                       | com.auth0.client.mgmt.UsersEntity.list         | Proporcione un `UserFilter` con `withSearchEngine("v3")`            |
| [Auth0 Python](https://github.com/auth0/auth0-python) | 3.0.0                       | management.Users.list                          | Proporcione el parámetro `search_engine='v3'`                       |
| [Auth0 Node](https://github.com/auth0/node-auth0)     | 2.0.0                       | UsersManager.getAll, ManagementClient.getUsers | Proporcione el parámetro `search_engine:'v3'`                       |
| [Auth0 .NET](https://github.com/auth0/auth0.net)      | 3.0.0 o 4.0.0               | Auth0.ManagementApi.IUsersClient.GetAllAsync   | Proporcione un objeto `GetUsersRequest` con `SearchEngine` = `"v3"` |
| [Auth0 PHP](https://github.com/auth0/auth0-php)       | 5.2.0                       | Auth0.SDK.API.Management.Users.getAll          | Proporcione el parámetro `'search_engine' => 'v3'`                  |
| [Auth0 Ruby](https://github.com/auth0/ruby-auth0)     | 4.5.0                       | Auth0.Api.V2.Users.users                       | Proporcione el parámetro `search_engine: 'v3'`                      |

<div id="impacted-extensions">
  ## Extensiones afectadas
</div>

Las siguientes extensiones usan el motor User Search. Si las tiene instaladas, asegúrese de usar las versiones que se indican a continuación (o una posterior).

| Extensión                                                                                | Compatibilidad de versiones con v3 | Consideraciones                                                                                                                                                                                                                                                      |
| ---------------------------------------------------------------------------------------- | ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Authorization Extension](/docs/es/customize/extensions/authorization-extension)            | 2.5.0+                             | Si usa una versión anterior, debe actualizar manualmente la extensión desde la página de [Extensions](https://manage.auth0.com/#/extensions).                                                                                                                        |
| [Administración delegada](/docs/es/customize/extensions/delegated-administration-extension) | 3.1+                               | Si usa una versión anterior, debe actualizar manualmente la extensión desde la página de [Extensions](https://manage.auth0.com/#/extensions). La opción de configuración `SEARCH_ENGINE` ya no existe en la versión 3.1, porque solo está disponible User Search v3. |

<div id="leverage-your-tenant-logs-to-find-usage-of-user-search-v2">
  ## Usa los registros de tu tenant para identificar el uso de User Search v2
</div>

Puedes usar los [registros](/docs/es/deploy-monitor/logs) en el [Dashboard](https://manage.auth0.com/#/logs) para encontrar llamadas al endpoint `/api/v2/users` que usan el motor User Search v2, incluidas las llamadas realizadas por SDKs. Estos registros te ayudarán a identificar en qué partes de tus aplicaciones podrían necesitarse cambios en el código.

Usa la siguiente consulta para recuperar todos los registros relacionados con User Search v2: `type:w AND description:*search_engine*`. Los registros proporcionarán información adicional en el campo description en los siguientes casos:

* Consultas que podrían producir resultados distintos en v3
* Consultas con una sintaxis incompatible con v3
* Consultas que no cumplen los requisitos de paginación de v3

Si no se especifican detalles adicionales en las entradas de registro, es probable que tus consultas sean compatibles con v3. Aun así, te recomendamos probarlas antes de desplegar los cambios en producción.

Ten en cuenta que solo se generará un registro del mismo tipo en un período de 60 minutos. Esto significa que, aunque hagas varias llamadas al endpoint de User Search, solo verás un registro de cada tipo por hora.
