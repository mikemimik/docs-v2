title: Migra desde las funcionalidades de extensibilidad de oracledb
---

A partir del 21 de junio de 2023, Auth0 dejará de admitir las conexiones a bases de datos Oracle desde Node.js en las funcionalidades de extensibilidad. Anteriormente, Auth0 admitía las conexiones a bases de datos Oracle a través del módulo `oracledb` de Node.js.

<div id="affected-workflows">
  ## Flujos de trabajo afectados
</div>

Revisa tus funcionalidades de extensibilidad para identificar código que contenga `require(oracledb)` o `require('oracledb@4.2.0')`.

Revisa los siguientes flujos o funcionalidades que permiten código de extensibilidad en busca de referencias a [oracledb](https://www.npmjs.com/package/oracledb):

* [Hooks](/docs/es/customize/hooks)
* [Rules](/docs/es/customize/rules)
* [Custom Database Scripts](/docs/es/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Custom Social Connections](/docs/es/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Actualizaciones recomendadas
</div>

No hay un reemplazo oficial previsto para esta funcionalidad. Auth0 recomienda trasladar la lógica de `oracledb` a un servicio independiente dentro de tu propia infraestructura. Después de crear ese servicio en tu infraestructura, puedes usar una función de extensibilidad para realizar una solicitud HTTP que active la lógica de la base de datos Oracle.

Para obtener más información sobre cómo implementar el módulo `oracledb` en tu propia infraestructura, consulta la [guía Quickstart de OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Por ejemplo, en un script de inicio de sesión de Custom Database que usa `oracledb`, reemplaza la función que llama a `oracledb` por un cliente HTTP genérico para capturar la respuesta.

Para obtener más información sobre cómo crear Actions o migrar a Actions, [consulta la documentación de Auth0](/docs/es/customize/actions).
