---
description: Describe las funcionalidades de Auth0 afectadas por la migración de Node.js v12 a Node.js v16 y ofrece recomendaciones para la migración.
title: Migrar a Node.js 16
---

El 30 de abril de 2022, [Node.js v12 dejó de contar con soporte a largo plazo (LTS)](https://github.com/nodejs/Release#release-schedule), lo que significa que el equipo de desarrollo de Node.js ya no incorpora correcciones críticas de seguridad a esta versión. Esto podría exponer su código de extensibilidad a vulnerabilidades de seguridad. Por lo tanto, Auth0 está migrando de Node 12 a Node 16.

<div id="recommendations">
  ## Recomendaciones
</div>

Aunque la actualización a Node 16 no introducirá ningún cambio incompatible en la biblioteca estándar de Node.js (Rules y Custom Database Action Scripts se ven afectados; consulte la sección [Cambios incompatibles: solo Rules y Custom Database Action Scripts](#breaking-changes-rules-only)), recomendamos a los clientes que usan Node 12 mantenerse al día con las versiones de Node con soporte activo a largo plazo (LTS) por motivos de seguridad y cumplimiento normativo. Los clientes que aún usan Node 8 ya no cumplen con los requisitos de seguridad y deben migrar a Node 16 para eliminar riesgos de seguridad. Eliminamos el runtime de Node 8 el 22 de febrero de 2022 para los tenants de Public Cloud, y en la versión de Private Cloud de abril de 2022. Después de esas fechas, los tenants que sigan configurados con Node 8 corren el riesgo de sufrir una interrupción del servicio.

Actions está diseñado para admitir varias versiones de Node de forma simultánea y actualmente admite tanto Node 12 como Node 16. Además, las Actions Integrations No-Code seguirán un control de versiones estándar del sector de tipo &quot;siempre actualizado&quot;, con versiones menores actualizadas automáticamente; el código subyacente se mantendrá siempre actualizado y en la versión más reciente de Node.js.

| Clientes que usan                       | Deben                                                                                                                                                                                                                                     | Tarea                                                                         |
| --------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| Actions Node 12                         | Actualizar cualquier versión existente de una Action personalizada a una nueva versión, lo que la actualizará a Node 16.                                                                                                                  | Actualice todas las versiones de Actions personalizadas lo antes posible.     |
| Legacy Node 12 (Rules/Hooks/Extensions) | Empezar a migrar a Actions Node 16.                                                                                                                                                                                                       | [Migrar a Actions Node 16](/docs/es/customize/actions/migrate) lo antes posible. |
| Legacy Node 8                           | Migrar inmediatamente a Actions siempre que sea posible. En el caso de Rules y Hooks que no puedan migrarse de inmediato a Actions, debe, como mínimo, actualizar su código a Node 12 y actualizar la Configuración del tenant a Node 12. | [Migrar a Actions Node 16](/docs/es/customize/actions/migrate) inmediatamente.   |

<div id="tasks">
  ## Tareas
</div>

<div id="upgrade-actions">
  ### Actualizar Actions
</div>

Las Actions existentes basadas en Node 12 pueden actualizarse a Node 16 y, si es necesario, revertirse después a una versión anterior en Node 12. Actualiza las Actions a Node 16 [creando e implementando una nueva versión](/docs/es/customize/actions/manage-versions) configurada para usar Node 16 como runtime.

<div id="migrate-rules-and-hooks-to-actions">
  ### Migra Rules y Hooks a Actions
</div>

Siempre que sea posible, migra Rules y Hooks a Actions. Para determinar cuáles Rules y Hooks pueden migrarse, lee [Actions Limitations](/docs/es/customize/actions/limitations). Para saber cómo migrar tus Rules y Hooks a Actions, consulta nuestras guías en [Migrate to Actions](/docs/es/customize/actions/migrate).

<div id="enable-node-16-runtime">
  ### Habilitar el runtime de Node 16
</div>

Si usa algún producto Legacy de extensibilidad, como Rules, Hooks o Extensions, debe habilitar el runtime de Node 16 a nivel del tenant. Le recomendamos que primero cambie su tenant de desarrollo al runtime de Node 16, pruebe la configuración y luego cambie su tenant de producción.

1. Habilite Node 16 en su tenant de desarrollo desde el panel de **Extensibilidad** de la página [Configuración avanzada del tenant](https://manage.auth0.com/#/tenant/advanced) del Dashboard. Elija **Node 16** en el menú desplegable **Runtime**.
2. Haga clic en **Save**.
3. Compruebe que no tenga los elementos que se indican a continuación o que haya completado los pasos de migración para cada uno.
4. Pruebe la configuración. Cuando termine de probar, repita los pasos 1 y 2 anteriores con su tenant de producción para habilitar Node 16 en producción.

<div id="breaking-changes-rules-and-custom-database-action-scripts-only">
  ### Cambios incompatibles: solo para Rules y Custom Database Action Scripts
</div>

Las Rules se ejecutan en un sandbox de JavaScript. El sandbox admite el lenguaje ECMAScript 6 y una gran cantidad de módulos de Node.js (versión 16+). Si usa alguna de estas propiedades globales dentro de Rules o Custom Database Action Scripts, [busque una alternativa en nuestra lista de módulos compatibles con el sandbox](https://auth0-extensions.github.io/canirequire/) e impórtela explícitamente con `require`.

* `_`
* `async`
* `Auth0`
* `azure_storage`
* `bcrypt`
* `crypto`
* `couchbase`
* `cql`
* `ip`
* `jwt`
* `Knex`
* `mongo`
* `mysql`
* `mysql_pool`
* `ObjectID`
* `pbkdf2`
* `pg`
* `postgres`
* `Pubnub`
* `q`
* `querystring`
* `request`
* `sqlserver`
* `uuid`
* `xml2js`
* `xmldom`
* `xpath`
* `xtend`
