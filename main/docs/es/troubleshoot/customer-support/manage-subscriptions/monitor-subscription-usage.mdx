Tu plan de suscripción de Auth0 determina los límites de usuarios y las funciones que tienes a tu disposición.

<div id="quota-utilization">
  ## Uso de la cuota
</div>

Puedes consultar la información de uso de la suscripción en Auth0 Support Center visitando [Reports &gt; Quota Utilization](https://support.auth0.com/reports/quota). La información de uso de la suscripción está disponible con los siguientes detalles:

* Consumo de usuarios

  * Active Users
* Consumo de funcionalidades

  * Conexiones empresariales
  * Autenticación de máquina a máquina
  * Autenticación multifactor

<Frame>![Informes que muestran el uso de las funcionalidades de la suscripción](/docs/images/cdy7uua7fh8z/7g0PDAKl5D8YEf4vC5FjSI/188182aabbeb7e0af77b1bc9d292cfad/Screenshot_2023-02-07_at_5.35.40_PM.jpeg)</Frame>

Cada tarjeta de resumen principal muestra el uso del mes actual en relación con el límite disponible de la suscripción. También permite profundizar (haciendo clic en la tarjeta) para ver más detalles, como el desglose del uso por Auth0 Private Instance (si corresponde) y por tenant, de hasta los últimos 12 meses.

También hay una opción para descargar un archivo CSV y exportar estos datos, si es necesario.

<Frame>![](/docs/images/cdy7uua7fh8z/7bBqDuycQWsnUdd9hlXN10/040a04d458bff06e8392c1432e3308ce/Screenshot_2023-02-07_at_6.07.01_PM.jpeg)</Frame>

<div id="active-users">
  ### Active Users
</div>

Puedes usar esto para obtener un recuento de usuarios únicos con un evento activo (login o signup) en un mes natural determinado. También puedes profundizar para obtener un desglose de estos usuarios activos por tenant y Private Auth0 Instance (si corresponde) de hasta los últimos 12 meses.

<div id="enterprise-connections">
  ### Conexiones empresariales
</div>

Una [conexión empresarial](/docs/es/authenticate/enterprise-connections) se considera activa si, durante el mes actual, cumple ambas condiciones:

* Haber sido habilitada para una aplicación.
* Haber tenido actividad de usuarios (por ejemplo, inicio de sesión, registro o intercambio de tokens).

Si una conexión empresarial nunca se habilitó para ninguna aplicación, o se habilitó pero no tuvo actividad de usuarios durante el mes actual, no se considera activa.

Puede usar este informe para obtener un recuento de conexiones empresariales activas para un mes calendario determinado, con un desglose de las conexiones activas por Private Auth0 Instance (si corresponde), así como el recuento de usuarios activos por conexión empresarial activa y tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La conexión empresarial de Okta Workforce no se contabiliza para el límite de conexiones empresariales activas.
</Callout>

<div id="machine-to-machine-auth">
  ### Autenticación de máquina a máquina
</div>

Este informe proporciona un recuento de los <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> emitidos para el [Client Credentials Flow](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow), también denominadas llamadas de autenticación de API, por mes calendario, excluyendo cualquier token emitido para la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0.

Puede consultar el recuento de llamadas de autenticación por Private Auth0 Instance (si corresponde) y tenant.

En el caso de los tenants con la opción de despliegue Auth0 Public Cloud, el informe también proporciona un desglose de las llamadas de autenticación por aplicación del tenant durante los últimos 7 días.

<div id="multi-factor-auth">
  ### Autenticación multifactor
</div>

Hay dos informes de uso de la Autenticación multifactor (<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>) que muestran el recuento de eventos de MFA (inscripciones o verificaciones) por mes natural.

Los recuentos de MFA empresarial corresponden a eventos de MFA relacionados con push notifications, WebAuthn y códigos de un solo uso generados por SMS, Email o Phone. Los recuentos de MFA Pro corresponden a eventos de MFA relacionados con aplicaciones de códigos de un solo uso, como Google Authentication o Cisco DUO.

Ambos tipos de informes proporcionan un desglose de los recuentos de MFA por Auth0 Private Instance* (si corresponde) y Tenant.

*El uso de MFA de Auth0 Private Instance solo está disponible desde el 1 de enero de 2023.

***

<div id="tenant-usage">
  ## Uso del tenant
</div>

Puedes ver los recuentos totales y de Active Users por mes calendario en Auth0 Support Center para un tenant específico; para ello, visita [Reports &gt; Usage](https://support.auth0.com/reports/usage) y selecciona una suscripción (y un tenant, en el caso de suscripciones de autoservicio). Esto proporciona el siguiente desglose:

* Active Users: recuento de Active Users por mes y tipo de usuario (Social, Contraseña, <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>, empresarial), junto con un recuento Total Active.
* Active Users por tenant: recuento de Active Users por mes, tenant y tipo de usuario (Social, Contraseña, Passwordless, empresarial), junto con un recuento Total Active.
* Usuarios totales por tenant

<Frame>![Support Center -> Reports -> Usage](/docs/images/cdy7uua7fh8z/2RmWp1ZIWwV2JCPJcftll5/acd9c3c71b8dbd9cca12a44588e015af/Screen_Shot_2022-10-27_at_1.21.35_PM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los recuentos de Total Active o Total Users pueden ser mayores que la suma de los usuarios de cada tipo individual cuando no se determina el tipo de usuario. Esto puede ocurrir, entre otros casos, cuando la vinculación de cuentas está activada.
</Callout>
