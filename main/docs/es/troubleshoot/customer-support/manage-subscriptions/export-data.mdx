description: Describe cómo exportar datos de Auth0.
title: Exportar datos
---

Todos los datos de tu tenant de Auth0 están siempre bajo tu control y disponibles a través de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> en cualquier momento.

<div id="user-passwords">
  ## Contraseñas de usuario
</div>

La única información que no está disponible a través de la API (por motivos de seguridad) son los hashes de contraseña de tus usuarios de base de datos alojados en Auth0 y las claves privadas. Aun así, puedes solicitar esta información abriendo un [ticket de soporte](https://support.auth0.com). Para obtener más información, consulta [Exportar hashes de contraseña y Secrets de MFA](/docs/es/manage-users/user-migration/export-password-hashes-and-mfa-secrets). Esta operación no está disponible en nuestro nivel de suscripción Free, y no podemos aceptar ni garantizar solicitudes de exportación para una fecha y hora específicas.

Si almacenas contraseñas de usuario en tu base de datos, configura una conexión de base de datos personalizada que Auth0 consultará cada vez que un usuario inicie sesión. En este caso, Auth0 nunca almacenará hashes de contraseña, a menos que decidas migrar usuarios progresivamente a Auth0.

<div id="auth0-components">
  ## Componentes de Auth0
</div>

Auth0 utiliza principalmente <Tooltip tip="OpenID: Estándar abierto para la autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) como protocolo de autenticación, por lo que deberías poder implementar una integración para tu aplicación con bibliotecas estándar. Lo mismo ocurre al integrar Auth0 mediante [SAML](/docs/es/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider). Todos los SDKs, las bibliotecas y los ejemplos de Auth0 [se publican en GitHub como software libre](https://github.com/auth0/).

<div id="social-identity-providers">
  ## Proveedores de identidad social
</div>

Si decides no usar Auth0, pero quieres seguir utilizando los mismos <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip> <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=client+IDs">ID de cliente</Tooltip> y secretos para tus <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> social, seguirás teniendo acceso a la información del usuario sin necesidad de mostrar nuevos cuadros de diálogo de consentimiento.

<div id="custom-code">
  ## Código personalizado
</div>

Todas las funcionalidades de código personalizado de Auth0 (reglas, scripts personalizados de base de datos y conexiones personalizadas de OAuth, entre otras) se ejecutan en un servicio sandbox de Node.js. Todas las bibliotecas disponibles en el servicio sandbox también están disponibles en npm para usarlas con código estándar de Node.js.
