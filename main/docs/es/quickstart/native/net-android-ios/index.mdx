
  Tu asistente de IA creará automáticamente tu aplicación de Auth0, recuperará las credenciales, instalará el SDK Auth0 OidcClient, configurará la configuración específica de cada plataforma e implementará los flujos de inicio y cierre de sesión. <div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Crear un nuevo proyecto de .NET" stepNumber={1}>
    Cree un nuevo proyecto de .NET MAUI o .NET Android/iOS para este Quickstart.

    <Tabs>
      <Tab title=".NET MAUI">
        **En Visual Studio 2022 o una versión posterior:**

        1. **Archivo** → **Nuevo** → **Proyecto**
        2. Seleccione la plantilla **.NET MAUI App**
        3. Configure su proyecto:
           * **Nombre del proyecto**: `Auth0MauiSample`
           * **Ubicación**: Elija la ubicación que prefiera
           * **Framework**: **.NET 8.0 o una versión posterior**
        4. Haga clic en **Crear**
      </Tab>

      <Tab title=".NET Android">
        **En Visual Studio 2022 o una versión posterior:**

        1. **Archivo** → **Nuevo** → **Proyecto**
        2. Seleccione la plantilla **Android App (.NET)**
        3. Configure su proyecto:
           * **Nombre del proyecto**: `Auth0AndroidSample`
           * **Framework**: **.NET 8.0 o una versión posterior**
        4. Haga clic en **Crear**
      </Tab>

      <Tab title=".NET iOS">
        **En Visual Studio 2022 (Mac) o Visual Studio for Mac:**

        1. **Archivo** → **Nuevo** → **Proyecto**
        2. Seleccione la plantilla **iOS App (.NET)**
        3. Configure su proyecto:
           * **Nombre del proyecto**: `Auth0iOSSample`
           * **Framework**: **.NET 8.0 o una versión posterior**
        4. Haga clic en **Crear**
      </Tab>
    </Tabs>

    <Info>
      Este Quickstart se centra en .NET Android e iOS, la nueva generación de Xamarin.Android y Xamarin.iOS. Si aún usa Xamarin, puede seguir esta guía, ya que la integración es idéntica y los SDKs son compatibles.
    </Info>
  </Step>

  <Step title="Instalar el SDK de Auth0" stepNumber={2}>
    Agrega el SDK Auth0 OIDC Client a tu proyecto.

    <Tabs>
      <Tab title="Consola del administrador de paquetes">
        Abre la Consola del administrador de paquetes (**View** → **Other Windows** → **Package Manager Console**) e instala el paquete adecuado:

        **Para .NET Android:**

        ```bash Package Manager Console
        Install-Package Auth0.OidcClient.AndroidX
        ```

        **Para .NET iOS:**

        ```bash Package Manager Console
        Install-Package Auth0.OidcClient.iOS
        ```

        **Para .NET MAUI (ambas plataformas):**

        ```bash Package Manager Console
        Install-Package Auth0.OidcClient.MAUI
        ```
      </Tab>

      <Tab title="Visual Studio for Mac">
        1. En el **Solution Pad**, haz Ctrl+clic (o clic derecho) en la carpeta **Packages**
        2. Selecciona **Add Packages...**
        3. Busca:
           * `Auth0.OidcClient.AndroidX` para Android
           * `Auth0.OidcClient.iOS` para iOS
           * `Auth0.OidcClient.MAUI` para proyectos MAUI
        4. Selecciona el paquete y haz clic en **Add Package**
      </Tab>

      <Tab title=".NET CLI">
        Ejecuta el siguiente comando en el directorio de tu proyecto:

        **Para .NET Android:**

        ```bash Terminal
        dotnet add package Auth0.OidcClient.AndroidX
        ```

        **Para .NET iOS:**

        ```bash Terminal
        dotnet add package Auth0.OidcClient.iOS
        ```

        **Para .NET MAUI:**

        ```bash Terminal
        dotnet add package Auth0.OidcClient.MAUI
        ```
      </Tab>
    </Tabs>

    <Tip>
      El SDK Auth0 OIDC Client se encarga de todos los detalles de los protocolos OAuth 2.0 y OIDC, y proporciona una API sencilla para la autenticación.
    </Tip>
  </Step>

  <Step title="Configura tu Auth0 App" stepNumber={3}>
    Crea una nueva aplicación en tu tenant de Auth0 y configúrala para dispositivos móviles.

    1. Ve al [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. Haz clic en **Applications** → **Applications** → **Create Application**
    3. Introduce un nombre para tu aplicación, selecciona **Native** como tipo de aplicación y haz clic en **Create**
    4. Ve a la pestaña **Settings** en la página de detalles de la aplicación
    5. Toma nota de tu **Domain** y **ID de cliente**; los necesitarás en el siguiente paso

    **Configura las URL de devolución de llamada:**

    En la pestaña **Settings**, agrega las siguientes URL:

    **Allowed Callback URLs:**

    <Tabs>
      <Tab title="Android">
        ```
        YOUR_ANDROID_PACKAGE_NAME://{yourDomain}/android/YOUR_ANDROID_PACKAGE_NAME/callback
        ```

        Reemplaza:

        * `YOUR_ANDROID_PACKAGE_NAME` por el nombre del paquete de tu aplicación (p. ej., `com.mycompany.myapp`)
        * `{yourDomain}` por tu dominio de Auth0 (p. ej., `dev-abc123.us.auth0.com`)

        Ejemplo: `com.mycompany.myapp://dev-abc123.us.auth0.com/android/com.mycompany.myapp/callback`
      </Tab>

      <Tab title="iOS">
        ```
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback
        ```

        Reemplaza:

        * `YOUR_BUNDLE_IDENTIFIER` por el identificador del paquete de tu aplicación (p. ej., `com.mycompany.myapp`)
        * `{yourDomain}` por tu dominio de Auth0 (p. ej., `dev-abc123.us.auth0.com`)

        Ejemplo: `com.mycompany.myapp://dev-abc123.us.auth0.com/ios/com.mycompany.myapp/callback`
      </Tab>
    </Tabs>

    **Allowed Logout URLs:**

    Usa las mismas URL que para tus URL de devolución de llamada.

    <Warning>
      Asegúrate de que las URL de devolución de llamada y de cierre de sesión estén en **minúsculas**. Si las URL no coinciden, la autenticación fallará.
    </Warning>

    <Info>
      Las **Allowed Callback URLs** son esenciales para la seguridad: garantizan que los usuarios regresen de forma segura a tu aplicación después de la autenticación. Sin una URL coincidente, el proceso de inicio de sesión fallará.

      Las **Allowed Logout URLs** ofrecen una experiencia fluida cuando los usuarios cierran sesión, ya que los redirigen de vuelta a tu aplicación en lugar de dejarlos en una página de Auth0.
    </Info>
  </Step>

  <Step title="Inicializar el cliente de Auth0" stepNumber={4}>
    Crea una instancia de `Auth0Client` para comunicarte con Auth0.

    <Tabs>
      <Tab title="Android - MainActivity">
        ```cs MainActivity.cs lines
        using Auth0.OidcClient;
        using Android.App;
        using Android.Content;

        [Activity(Label = "Auth0Sample", MainLauncher = true, Icon = "@drawable/icon",
            LaunchMode = LaunchMode.SingleTask)]
        [IntentFilter(
            new[] { Intent.ActionView },
            Categories = new[] { Intent.CategoryDefault, Intent.CategoryBrowsable },
            DataScheme = "YOUR_ANDROID_PACKAGE_NAME",
            DataHost = "{yourDomain}",
            DataPathPrefix = "/android/YOUR_ANDROID_PACKAGE_NAME/callback")]
        public class MainActivity : Activity
        {
            private Auth0Client auth0Client;

            protected override void OnCreate(Bundle savedInstanceState)
            {
                base.OnCreate(savedInstanceState);

                // Inicializar el cliente de Auth0
                auth0Client = new Auth0Client(new Auth0ClientOptions
                {
                    Domain = "{yourDomain}",
                    ClientId = "{yourClientId}"
                }, this);
            }

            protected override async void OnNewIntent(Intent intent)
            {
                base.OnNewIntent(intent);
                Auth0.OidcClient.ActivityMediator.Instance.Send(intent.DataString);
            }
        }
        ```

        <Warning>
          Reemplaza `YOUR_ANDROID_PACKAGE_NAME`, `{yourDomain}` y `{yourClientId}` por tus valores reales. Asegúrate de que todo el texto de `DataScheme`, `DataHost` y `DataPathPrefix` esté en **minúsculas**.
        </Warning>

        <Info>
          `IntentFilter` registra tu aplicación para gestionar la URL de devolución de llamada. `LaunchMode.SingleTask` garantiza que Android no cree una nueva instancia de la actividad cuando se invoque la devolución de llamada.
        </Info>
      </Tab>

      <Tab title="iOS - AppDelegate">
        Primero, registra el esquema de URL en tu `Info.plist`:

        ```xml Info.plist lines
        <key>CFBundleURLTypes</key>
        <array>
            <dict>
                <key>CFBundleTypeRole</key>
                <string>None</string>
                <key>CFBundleURLName</key>
                <string>Auth0</string>
                <key>CFBundleURLSchemes</key>
                <array>
                    <string>YOUR_BUNDLE_IDENTIFIER</string>
                </array>
            </dict>
        </array>
        ```

        Luego, configura tu `AppDelegate.cs`:

        ```cs AppDelegate.cs lines
        using Auth0.OidcClient;
        using Foundation;
        using UIKit;

        [Register("AppDelegate")]
        public class AppDelegate : UIApplicationDelegate
        {
            private Auth0Client auth0Client;

            public override bool FinishedLaunching(UIApplication application, NSDictionary launchOptions)
            {
                // Inicializar el cliente de Auth0
                auth0Client = new Auth0Client(new Auth0ClientOptions
                {
                    Domain = "{yourDomain}",
                    ClientId = "{yourClientId}"
                });

                return true;
            }

            public override bool OpenUrl(UIApplication application, NSUrl url,
                string sourceApplication, NSObject annotation)
            {
                ActivityMediator.Instance.Send(url.AbsoluteString);
                return true;
            }
        }
        ```

        <Warning>
          Reemplaza `YOUR_BUNDLE_IDENTIFIER`, `{yourDomain}` y `{yourClientId}` por tus valores reales.
        </Warning>
      </Tab>
    </Tabs>

    <Tip>
      Guarda tu dominio de Auth0 y tu ID de cliente en un archivo de configuración o en los ajustes de la aplicación, en lugar de incluirlos directamente en el código, para facilitar el mantenimiento.
    </Tip>
  </Step>

  <Step title="Implement Login y cierre de sesión" stepNumber={5}>
    Añade métodos para gestionar la autenticación de usuario.

    **Implement Login:**

    ```cs Authentication.cs lines
    public async Task LoginAsync()
    {
        var loginResult = await auth0Client.LoginAsync();

        if (!loginResult.IsError)
        {
            // Autenticación exitosa
            var accessToken = loginResult.AccessToken;
            var idToken = loginResult.IdentityToken;
            var user = loginResult.User;

            // Guardar credenciales y actualizar la interfaz
            Console.WriteLine($"Logged in as: {user.FindFirst("name")?.Value}");
        }
        else
        {
            // Gestionar error de autenticación
            Console.WriteLine($"Login error: {loginResult.Error}");
        }
    }
    ```

    **Cierre de sesión de Implement:**

    ```cs Authentication.cs lines
    public async Task LogoutAsync()
    {
        var logoutResult = await auth0Client.LogoutAsync();

        if (logoutResult == BrowserResultType.Success)
        {
            // Borrar credenciales almacenadas
            // Actualizar la interfaz al estado de sesión cerrada
            Console.WriteLine("Logged out successfully");
        }
    }
    ```

    <Info>
      El método `LoginAsync()` abre el navegador del sistema (o Chrome Custom Tabs en Android) para mostrar la página de Universal Login de Auth0. Tras la autenticación, se redirige al usuario de vuelta a tu aplicación mediante la URL de devolución de llamada.
    </Info>

    <Tip>
      Agrega estos métodos a tu MainActivity (Android) o a un ViewController (iOS) y llámalos cuando los usuarios toquen los botones de inicio/cierre de sesión.
    </Tip>
  </Step>

  <Step title="Inicia tu aplicación" stepNumber={6}>
    Compila y ejecuta tu aplicación.

    <Tabs>
      <Tab title="Visual Studio (Windows)">
        **Para Android:**

        1. Selecciona un emulador de Android o un dispositivo conectado en el menú desplegable de dispositivos
        2. Presiona **F5** o haz clic en el botón **Run**
        3. La aplicación se compilará, se desplegará y se iniciará

        **Para iOS (requiere un host de compilación Mac):**

        1. Conéctate a tu host de compilación Mac
        2. Selecciona un simulador o dispositivo iOS en el menú desplegable de dispositivos
        3. Presiona **F5** o haz clic en el botón **Run**
      </Tab>

      <Tab title="Visual Studio for Mac">
        1. Selecciona tu plataforma de destino (Android/iOS) en el menú desplegable de dispositivos
        2. Selecciona un emulador/simulador o un dispositivo conectado
        3. Presiona **⌘+Return** o haz clic en el botón **Run**
        4. La aplicación se compilará y se iniciará
      </Tab>

      <Tab title=".NET CLI">
        **Para Android:**

        ```bash Terminal
        dotnet build -f net8.0-android
        dotnet run -f net8.0-android
        ```

        **Para iOS:**

        ```bash Terminal
        dotnet build -f net8.0-ios
        dotnet run -f net8.0-ios
        ```
      </Tab>
    </Tabs>

    **Flujo esperado:**

    1. La aplicación se inicia con el botón Login
    2. Toca **Log In** → Se abre el navegador/Chrome Custom Tab → Completa la autenticación
    3. Se redirige de vuelta a tu aplicación automáticamente
    4. El usuario queda autenticado correctamente

    <Tip>
      En la primera ejecución, es posible que iOS te pida confirmar la apertura del navegador para la autenticación. Esto es normal y esperado.
    </Tip>
  </Step>
</Steps>

<div id="access-user-information">
  ## Accede a la información del usuario
</div>

Después de autenticarte correctamente, puedes acceder a la información del usuario desde el resultado del inicio de sesión.

<div id="authentication-result">
  ### Resultado de la autenticación
</div>

El método `LoginAsync()` devuelve un objeto `LoginResult` que contiene:

```cs UserInfo.cs lines
var loginResult = await auth0Client.LoginAsync();

if (!loginResult.IsError)
{
    // Tokens de acceso
    var accessToken = loginResult.AccessToken;
    var idToken = loginResult.IdentityToken;
    var refreshToken = loginResult.RefreshToken;

    // Acceder a los claims del usuario
    var user = loginResult.User;
    var name = user.FindFirst("name")?.Value;
    var email = user.FindFirst("email")?.Value;
    var picture = user.FindFirst("picture")?.Value;

    Console.WriteLine($"Name: {name}");
    Console.WriteLine($"Email: {email}");
}
```

<div id="iterate-through-all-claims">
  ### Recorrer todos los claims
</div>

Para ver toda la información disponible del usuario:

```cs UserClaims.cs lines
if (!loginResult.IsError)
{
    foreach (var claim in loginResult.User.Claims)
    {
        Console.WriteLine($"{claim.Type}: {claim.Value}");
    }
}
```

<Info>
  Los claims que se devuelven dependen de los scopes solicitados. Para obtener más información, consulta [Uso de scopes](https://auth0.github.io/auth0-oidc-client-net/documentation/advanced-scenarios/scopes.html) en la documentación de Auth0 OIDC Client.
</Info>

<div id="request-custom-scopes">
  ### Solicitar scopes personalizados
</div>

Para solicitar información adicional del usuario, especifique los scopes al crear el Auth0Client:

```cs CustomScopes.cs lines
var auth0Client = new Auth0Client(new Auth0ClientOptions
{
    Domain = "{yourDomain}",
    ClientId = "{yourClientId}",
    Scope = "openid profile email offline_access read:posts"
});
```

<div id="troubleshooting-advanced">
  ## Solución de problemas y opciones avanzadas
</div>

<Accordion title="Problemas comunes y soluciones">
  ### El navegador no vuelve a la aplicación

  **Soluciones:**

  1. Verifica que las URL de devolución de llamada en Auth0 Dashboard coincidan exactamente con el nombre del paquete o el identificador del paquete de tu aplicación
  2. Asegúrate de que las URL de devolución de llamada estén en **minúsculas**
  3. Comprueba que `DataScheme`, `DataHost` y `DataPathPrefix` (Android), o el esquema de URL (iOS), coincidan con tu configuración
  4. Limpia y recompila tu proyecto

  ### La autenticación falla con el error &quot;Invalid Callback URL&quot;

  **Solución:**

  * Vuelve a comprobar que la URL de devolución de llamada en Auth0 Dashboard coincida con este formato:
    * Android: `packagename://yourdomain/android/packagename/callback`
    * iOS: `bundleidentifier://yourdomain/ios/bundleidentifier/callback`
  * Asegúrate de que la URL esté en minúsculas
  * Verifica que el Domain de tu código coincida con el Domain en Auth0 Dashboard

  ### LoginAsync() se queda bloqueado o nunca termina

  **Soluciones:**

  * Asegúrate de que el filtro de Intent (Android) o el esquema de URL (iOS) estén configurados correctamente
  * Comprueba que `OnNewIntent()` (Android) o `OpenUrl()` (iOS) llamen a `ActivityMediator`
  * Verifica que tu aplicación pueda abrir el navegador del sistema
  * Comprueba la conectividad de red

  ### Error: &quot;Default App must use Token Endpoint Authentication Method &#39;None&#39;&quot;

  **Solución:**

  1. Ve a Application Settings de tu aplicación de Auth0 en el Dashboard
  2. Desplázate hasta **Application Properties**
  3. Establece **Application Type** como **Native**
  4. Establece **Token Endpoint Authentication Method** como **None**
  5. Haz clic en **Save Changes**

  ### iOS: el navegador no se abre

  **Soluciones:**

  * Verifica que `Info.plist` contenga la configuración correcta del esquema de URL
  * Comprueba que `OpenUrl()` esté implementado en `AppDelegate`
  * Asegúrate de que el objetivo de implementación de iOS sea compatible con la versión de tu SDK de Auth0
</Accordion>

<Accordion title="Consideraciones para producción">
  ### Prácticas recomendadas de seguridad

  * **Almacenamiento seguro de tokens**: Usa almacenamiento seguro específico de la plataforma (Android Keystore, iOS Keychain) para almacenar tokens
  * **Actualización de tokens**: Implementa el manejo del token de actualización para mantener las sesiones de usuario
  * **Fijación de certificados**: Considera la fijación de certificados para añadir seguridad adicional a la API
  * **ProGuard/ofuscación de código**: Agrega reglas adecuadas si usas ofuscación de código en Android

  ### Requisitos de la App Store

  * **Política de privacidad**: Asegúrate de que tu aplicación tenga una política de privacidad que describa el uso de Auth0
  * **Manejo de datos de usuario**: Sigue las directrices de la plataforma para manejar los datos de autenticación de usuario
  * **Enlaces profundos**: Prueba exhaustivamente el manejo de la URL de devolución de llamada en distintos escenarios
  * **Requisitos de red**: Maneja correctamente los casos sin conexión

  ### Optimización del rendimiento

  * **Usa Auth0Client en caché**: Crea una sola instancia y reutilízala en toda tu aplicación
  * **Carga diferida**: Inicializa Auth0Client solo cuando sea necesario
  * **Actualización en segundo plano**: Implementa la actualización de tokens en segundo plano para sesiones de larga duración
</Accordion>

<Accordion title="Configuración avanzada">
  ### Scopes personalizados y audiencia

  Solicita scopes específicos y establece una audiencia para tu API:

  ```cs AdvancedAuth.cs expandable lines
  var auth0Client = new Auth0Client(new Auth0ClientOptions
  {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}",
      Scope = "openid profile email offline_access read:posts write:posts",
      Audience = "https://myapi.example.com"
  });

  var loginResult = await auth0Client.LoginAsync();
  ```

  ### Parámetros adicionales

  Pasa parámetros adicionales a la solicitud de autorización:

  ```cs ExtraParams.cs expandable lines
  var extraParameters = new Dictionary<string, string>
  {
      { "prompt", "login" },
      { "ui_locales", "es" },
      { "custom_param", "value" }
  };

  var loginResult = await auth0Client.LoginAsync(extraParameters);
  ```

  ### Tokens de actualización

  Usa tokens de actualización para obtener nuevos tokens de acceso sin que el usuario tenga que interactuar:

  ```cs RefreshToken.cs expandable lines
  var refreshResult = await auth0Client.RefreshTokenAsync(loginResult.RefreshToken);

  if (!refreshResult.IsError)
  {
      var newAccessToken = refreshResult.AccessToken;
      var newIdToken = refreshResult.IdentityToken;
      // Almacena los nuevos tokens
  }
  ```

  <Info>
    Para recibir un token de actualización, incluye el scope `offline_access` en tu solicitud de autenticación.
  </Info>

  ### Configuración del navegador específica de la plataforma

  **Android - Usa Chrome Custom Tabs con colores personalizados:**

  ```cs AndroidBrowser.cs expandable lines
  var auth0Client = new Auth0Client(new Auth0ClientOptions
  {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}",
      Browser = new AndroidBrowser
      {
          ToolbarColor = Android.Graphics.Color.ParseColor("#FF6B35")
      }
  }, this);
  ```

  **iOS - Usa SFSafariViewController con presentación personalizada:**

  ```cs iOSBrowser.cs expandable lines
  var auth0Client = new Auth0Client(new Auth0ClientOptions
  {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}",
      Browser = new ASWebAuthenticationSessionBrowser
      {
          PrefersEphemeralWebBrowserSession = false
      }
  });
  ```
</Accordion>
