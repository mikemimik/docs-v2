<Recipe>
  <Content>
    Auth0 te permite agregar autenticación a casi cualquier tipo de aplicación de forma rápida. Esta guía muestra cómo
    integrar Auth0, añadir autenticación y mostrar información del perfil de usuario en cualquier aplicación .NET Android e iOS
    mediante los SDK de Auth0 para [Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/) e [iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS).

    <Info>
      Esta guía de inicio rápido se centra en .NET Android y iOS, ya que son la nueva generación de
      `Xamarin.Android` y `Xamarin.iOS`. Si todavía usas
      `Xamarin.Android` y `Xamarin.iOS`, puedes seguir esta guía, ya que la integración es
      idéntica y los SDKs son compatibles.
    </Info>

    Para usar esta guía de inicio rápido, necesitarás:

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación en tu proyecto.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva &quot;aplicación nativa&quot; o selecciona una aplicación existente que
      represente el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente alfanumérico único
      que el código de tu aplicación usará para llamar a las API de Auth0 mediante el SDK.

      Cualquier configuración que realices con este Quickstart se actualizará automáticamente para tu aplicación en el [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones en el futuro.

      Si prefieres explorar una configuración completa, puedes ver una aplicación de ejemplo.

      ### Configurar URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en una de las siguientes URL según
        tu plataforma:

        * **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en una de las siguientes URL según
        tu plataforma:

        * **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      Por último, asegúrate de que el **tipo de aplicación** de tu aplicación esté configurado como **Native** en la [Configuración de la
      aplicación](https://manage.auth0.com/#/applications/).
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 ofrece un SDK para [Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/) y [iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS) para simplificar el proceso de integrar la
      autenticación de Auth0 en aplicaciones .NET para Android e iOS.

      Use el Administrador de paquetes NuGet (Tools -&gt; Library Package Manager -&gt; Package Manager Console) para instalar el
      paquete `Auth0.OidcClient.AndroidX` o `Auth0.OidcClient.iOS`, según si está
      desarrollando una aplicación para Android o iOS.

      Como alternativa, puede usar la consola del Administrador de paquetes NuGet (`Install-Package`) o la
      CLI de `dotnet` (`dotnet add`).

      ```
      Install-Package Auth0.OidcClient.AndroidX
      Install-Package Auth0.OidcClient.iOS
      ```

      ```
      dotnet add Auth0.OidcClient.AndroidX
      dotnet add Auth0.OidcClient.iOS
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Para integrar Auth0 en tu aplicación, crea una instancia de la clase `Auth0Client`, pasándole
      una instancia de `Auth0ClientOptions` que contenga tu dominio de Auth0 y tu ID de cliente.

      ```csharp
      using Auth0.OidcClient;
      var client = new Auth0Client(new Auth0ClientOptions {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}"
      }, this);
      ```

      De forma predeterminada, el SDK usará Chrome Custom Tabs para Android y ASWebAuthenticationSession para iOS.

      <Note>
        ##### Punto de verificación

        Tu `Auth0Client` ahora debería estar correctamente instanciado. Ejecuta tu aplicación para verificar lo siguiente:

        * `Auth0Client` esté correctamente instanciado en la `Activity`(Android) o
          `UIViewController`(iOS).
        * Tu aplicación no esté generando ningún error relacionado con Auth0.
      </Note>
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Después de que un usuario se autentique correctamente, se le redirigirá a la URL de devolución de llamada que configuraste antes en este
      Quickstart.

      Para gestionar la devolución de llamada en dispositivos Android, debes registrar un intent que maneje esta URL de devolución de llamada. Una forma sencilla
      de hacerlo es registrar el intent en la misma actividad desde la que llamaste al método LoginAsync para
      inicializar el flujo de autenticación.

      Asegúrate de reemplazar `YOUR_ANDROID_PACKAGE_NAME` en el ejemplo de código por el Package Name real de tu
      aplicación, como `com.mycompany.myapplication`, y de que todo el texto de
      `DataScheme`, `DataHost` y `DataPathPrefix` esté en minúsculas. Además, configura
      `LaunchMode = LaunchMode.SingleTask` para la Activity; de lo contrario, el sistema creará una nueva instancia
      de la actividad cada vez que se invoque la URL de devolución de llamada.

      Además, debes gestionar el intent en el evento `OnNewIntent` de tu clase `Activity`.
      Debes notificar a Auth0 OIDC Client que finalice el flujo de autenticación llamando al método `Send`
      del singleton `ActivityMediator` y pasando la URL recibida.

      ```charp
      protected override void OnNewIntent(Intent intent)
      {

            base.OnNewIntent(intent);

            ActivityMediator.Instance.Send(intent.DataString);

      }
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Después de que un usuario se autentique correctamente, se le redirigirá a la URL de devolución de llamada que configuraste antes en este
      Quickstart.

      Para gestionar la devolución de llamada en dispositivos iOS:

      1. Abre el archivo `Info.plist` de tu aplicación en Visual Studio y ve a la pestaña &#42;&#42;Advanced &#42;&#42;.
      2. En **URL Types**, haz clic en el botón &#42;&#42;Add URL Type &#42;&#42;.
      3. Establece &#42;&#42;Identifier &#42;&#42;como Auth0, &#42;&#42;URL Schemes &#42;&#42;igual que el
         `identificador del paquete` de tu aplicación, y &#42;&#42;Role &#42;&#42;como `None`.

      Este es un ejemplo de la representación XML de tu archivo `info.plist` después de haber agregado el
      Type:

      ```xml
      <key>CFBundleURLTypes</key>
      <array>
      <dict>
      <key>CFBundleTypeRole</key>
      <string>None</string>
      <key>CFBundleURLName</key>
      <string>Auth0</string>
      <key>CFBundleURLSchemes</key>
      <array>
      <string>YOUR_BUNDLE_IDENTIFIER</string>
      </array>
      </dict>
      </array>
      ```

      Además, debes gestionar la URL de devolución de llamada en el evento `OpenUrl` de tu
      clase `AppDelegate`. Debes notificar a Auth0 OIDC Client para que finalice el flujo de autenticación
      llamando al método `Send` del singleton `ActivityMediator` y pasando la URL que
      se recibió.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Ahora que has configurado tu aplicación de Auth0 y el SDK de Auth0, debes configurar el inicio de sesión para tu proyecto.
      Para ello, usarás el método `LoginAsync()` del SDK para crear un botón de inicio de sesión que redirija a los usuarios
      a la página de Auth0 Universal Login.

      ```csharp
      var loginResult = await client.LoginAsync();
      ```

      Si no se produce ningún error, puedes acceder a `User`, `IdentityToken`,
      `AccessToken` y `RefreshToken` en el `LoginResult` que devuelve
      `LoginAsync()`.

      <Note>
        ##### Checkpoint

        Ahora deberías poder iniciar sesión o registrarte con un nombre de usuario y contraseña.

        Haz clic en el botón de inicio de sesión y verifica que:

        * Tu aplicación de Android o iOS te redirija a la página de Auth0 Universal Login.
        * Puedas iniciar sesión o registrarte.
        * Auth0 te redirija a tu aplicación.
      </Note>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Los usuarios que inicien sesión en tu proyecto también necesitarán una forma de cerrar sesión. Crea un botón de cierre de sesión con el método `LogoutAsync()` del SDK.
      Cuando los usuarios cierren sesión, se les redirigirá al endpoint de cierre de sesión de Auth0,
      que luego los redirigirá de inmediato a la URL de cierre de sesión que configuraste antes en este Quickstart.

      ```csharp
      await client.LogoutAsync();
      ```

      <Note>
        ##### Checkpoint

        Ejecuta tu aplicación y haz clic en el botón de cierre de sesión. Comprueba que:

        * Tu aplicación de Android o iOS te redirija a la dirección que especificaste como una de las
          URL de cierre de sesión permitidas en la Configuración de la aplicación.
        * Ya no tengas la sesión iniciada en tu aplicación.
      </Note>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Ahora que tus usuarios pueden iniciar sesión y cerrar sesión, probablemente querrás poder obtener la información de perfil asociada a los usuarios autenticados.
      Por ejemplo, puede que quieras mostrar en tu proyecto el nombre o la foto de perfil de un usuario que haya iniciado sesión.

      El SDK de Auth0 para Android e iOS proporciona información del usuario a través de la propiedad `LoginResult.User`.

      ```csharp
      if loginResult.IsError == false {
            var user = loginResult.User
            var name = user.FindFirst(c => c.Type == "name")?.Value
            var email = user.FindFirst(c => c.Type == "email")?.Value
            var picture = user.FindFirst(c => c.Type == "picture")?.Value
      }
      ```
    </Section>
