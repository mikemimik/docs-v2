<Accordion title="Usa IA para integrar Auth0" icon="microchip-ai" iconType="solid" defaultOpen>
  Si usas un asistente de programación con IA como Claude Code, Cursor o GitHub Copilot, puedes añadir la autenticación de Auth0 automáticamente en cuestión de minutos con [Agent Skills](https://agentskills.io/home).

  **Instala:**

  ```bash
  npx skills add auth0/agent-skills --skill auth0-quickstart --skill auth0-maui
  ```

  **Después, pídele a tu asistente de IA:**

  ```text
  Add Auth0 authentication to my .NET MAUI app
  ```

  Tu asistente de IA creará automáticamente tu aplicación de Auth0, obtendrá las credenciales, instalará el SDK Auth0 OidcClient MAUI, configurará las URL de devolución de llamada e implementará los flujos de inicio y cierre de sesión. [Documentación completa de Agent Skills →](/docs/es/quickstart/agent-skills)
</Accordion>

Esta guía muestra cómo integrar Auth0 con una aplicación .NET MAUI mediante el [SDK Auth0.OidcClient.MAUI](https://github.com/auth0/auth0-oidc-client-net). Al finalizar, tu aplicación admitirá inicio de sesión, cierre de sesión y la visualización de la información del perfil del usuario en **Android**, **iOS**, **macOS** y **Windows** desde una única base de código.

Esta guía usa la versión **1.x** de `Auth0.OidcClient.MAUI`.

<Note>
  ## Requisitos previos

  * **SDK de .NET 8 o .NET 9** instalado ([descargar](https://dotnet.microsoft.com/download))
  * **Workload de MAUI** instalado
  * Una **cuenta de Auth0** ([regístrate gratis](https://auth0.com/signup))
  * **Visual Studio 2022** (17.8+), **JetBrains Rider** o **VS Code** con la extensión de .NET MAUI
</Note>

Verifica tu entorno:

```shellscript
dotnet --version        # Debe ser 8.x o 9.x
dotnet workload list    # Debe incluir maui
```

Si no tienes instalada la carga de trabajo de MAUI, instálala:

```shellscript
dotnet workload install maui
```

<div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Configura tu aplicación de Auth0" stepNumber={1}>
    Configura tu aplicación de Auth0 para obtener las credenciales que necesita tu aplicación MAUI.

    1. Ve a **[Auth0 Dashboard](https://manage.auth0.com/dashboard/) &gt; Applications &gt; Applications**
    2. Selecciona **Create Application**
    3. Introduce un nombre para tu aplicación (por ejemplo, &quot;My MAUI App&quot;), selecciona **Native** como tipo de aplicación y luego selecciona **Create**
    4. Ve a la pestaña **Settings** en la página de detalles de la aplicación
    5. Anota los valores de **Domain** y **Client ID**; los necesitarás más adelante

    En la pestaña **Settings**, desplázate hasta **Application URIs** y configura las siguientes URL. Las aplicaciones .NET MAUI usan un esquema de URI personalizado (por ejemplo, `myapp://callback`) en lugar de una URL HTTP.

    **Allowed Callback URLs:**

    ```text
    myapp://callback
    ```

    **Allowed Logout URLs:**

    ```text
    myapp://callback
    ```

    <Tip>
      Elige un esquema único para tu aplicación. Un nombre de dominio invertido funciona bien; por ejemplo, `com.mycompany.myapp://callback`.
    </Tip>

    Selecciona **Guardar cambios**.

    <Check>
      Tienes una aplicación nativa en Auth0, con tu **Dominio** y **ID de cliente** anotados, y las URL de devolución de llamada y de cierre de sesión configuradas.
    </Check>
  </Step>

  <Step title="Crea tu proyecto MAUI" stepNumber={2}>
    Si ya tienes un proyecto de .NET MAUI, ve al paso 3. Si no, crea uno con la CLI de .NET:

    ```shellscript
    dotnet new maui -n MyMauiApp
    cd MyMauiApp
    ```
  </Step>

  <Step title="Instala el SDK de Auth0 para MAUI" stepNumber={3}>
    Añade el paquete NuGet `Auth0.OidcClient.MAUI` a tu proyecto:

    ```shellscript
    dotnet add package Auth0.OidcClient.MAUI
    ```

    <Check>
      Ejecuta `dotnet restore` para confirmar que el paquete se haya instalado sin errores.
    </Check>
  </Step>

  <Step title="Configurar el manejo específico de la plataforma para la devolución de llamada" stepNumber={4}>
    Las aplicaciones .NET MAUI deben registrar un controlador de devolución de llamada en cada plataforma para que el navegador del sistema pueda redirigir al usuario de vuelta a tu aplicación después de la autenticación. Sigue las instrucciones correspondientes a cada plataforma de destino.

    <Tabs>
      <Tab title="Android">
        Crea un archivo nuevo en `Platforms/Android/WebAuthenticatorActivity.cs`:

        ```csharp Platforms/Android/WebAuthenticatorActivity.cs lines
        using Android.App;
        using Android.Content;
        using Android.Content.PM;

        namespace MyMauiApp.Platforms.Android;

        [Activity(NoHistory = true, LaunchMode = LaunchMode.SingleTop, Exported = true)]
        [IntentFilter(new[] { Intent.ActionView },
                      Categories = new[] { Intent.CategoryDefault, Intent.CategoryBrowsable },
                      DataScheme = CALLBACK_SCHEME)]
        public class WebAuthenticatorActivity : Microsoft.Maui.Authentication.WebAuthenticatorCallbackActivity
        {
            const string CALLBACK_SCHEME = "myapp";
        }
        ```

        Reemplaza `myapp` por el esquema de URI que configuraste en el paso 1.

        <Note>
          El valor de `CALLBACK_SCHEME` debe coincidir exactamente con el esquema de tu `RedirectUri` y con las **Allowed Callback URLs** en Auth0.
        </Note>
      </Tab>

      <Tab title="Windows">
        El flujo de devolución de llamada de Auth0 en Windows depende de la activación del protocolo URI, lo que requiere que tu aplicación esté **empaquetada** (MSIX). La plantilla predeterminada de .NET MAUI crea una aplicación empaquetada, así que no necesitas hacer cambios si usaste `dotnet new maui` o la plantilla MAUI de Visual Studio.

        <Info>
          Si tu `.csproj` contiene `<WindowsPackageType>None</WindowsPackageType>`, tu aplicación está **sin empaquetar** y la activación del protocolo no funcionará. Elimina esa línea o cámbiala a `<WindowsPackageType>MSIX</WindowsPackageType>` para usar una aplicación empaquetada. Para obtener más información, consulta la [documentación de empaquetado de Windows de .NET MAUI](https://learn.microsoft.com/en-us/dotnet/maui/windows/deployment-overview).
        </Info>

        Se requieren dos cambios: registrar el protocolo URI y manejar la activación del protocolo.

        **1. Registra el protocolo** en `Platforms/Windows/Package.appxmanifest`. Agrega el bloque `<Extensions>` dentro del elemento `<Application>` existente:

        ```xml Platforms/Windows/Package.appxmanifest lines
        <Applications>
          <Application Id="App" Executable="$targetnametoken$.exe" EntryPoint="$targetentrypoint$">
            <Extensions>
              <uap:Extension Category="windows.protocol">
                <uap:Protocol Name="myapp"/>
              </uap:Extension>
            </Extensions>
          </Application>
        </Applications>
        ```

        **2. Maneja la activación** en `Platforms/Windows/App.xaml.cs`. Agrega la llamada a `CheckRedirectionActivation` como **primera línea** del constructor:

        ```csharp Platforms/Windows/App.xaml.cs lines
        public App()
        {
            if (Auth0.OidcClient.Platforms.Windows.Activator.Default.CheckRedirectionActivation())
                return;

            this.InitializeComponent();
        }
        ```
      </Tab>

      <Tab title="iOS / macOS">
        No se requiere ninguna configuración específica de la plataforma. El SDK usa automáticamente `ASWebAuthenticationSession` a través del `WebAuthenticator` de MAUI.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Añadir inicio de sesión y cierre de sesión" stepNumber={5}>
    Necesitas crear o modificar tres archivos: un ViewModel con la lógica de inicio y cierre de sesión, una página XAML para la UI y un archivo de code-behind para conectarlos.

    <Tabs>
      <Tab title="MainPageViewModel.cs">
        Crea el ViewModel en `ViewModels/MainPageViewModel.cs`:

        ```csharp ViewModels/MainPageViewModel.cs lines
        using System.ComponentModel;
        using System.Runtime.CompilerServices;
        using System.Windows.Input;
        using Auth0.OidcClient;

        namespace MyMauiApp.ViewModels;

        public class MainPageViewModel : INotifyPropertyChanged
        {
            private readonly Auth0Client _client;
            private string _name;
            private string _email;
            private bool _isAuthenticated;

            public event PropertyChangedEventHandler PropertyChanged;

            public string Name
            {
                get => _name;
                set { _name = value; OnPropertyChanged(); }
            }

            public string Email
            {
                get => _email;
                set { _email = value; OnPropertyChanged(); }
            }

            public bool IsAuthenticated
            {
                get => _isAuthenticated;
                set
                {
                    _isAuthenticated = value;
                    OnPropertyChanged();
                    OnPropertyChanged(nameof(IsNotAuthenticated));
                }
            }

            public bool IsNotAuthenticated => !IsAuthenticated;

            public ICommand LoginCommand { get; }
            public ICommand LogoutCommand { get; }

            public MainPageViewModel(Auth0Client client)
            {
                _client = client;
                LoginCommand = new Command(async () => await LoginAsync());
                LogoutCommand = new Command(async () => await LogoutAsync());
            }

            private async Task LoginAsync()
            {
                var loginResult = await _client.LoginAsync();

                if (loginResult.IsError)
                {
                    if (loginResult.Error == "UserCancel")
                        return; // El usuario cerró el navegador; no es un error

                    await Shell.Current.DisplayAlert("Login failed", loginResult.Error, "OK");
                    return;
                }

                // Lee las claims del perfil de usuario desde el token de ID
                Name = loginResult.User.FindFirst(c => c.Type == "name")?.Value;
                Email = loginResult.User.FindFirst(c => c.Type == "email")?.Value;
                IsAuthenticated = true;
            }

            private async Task LogoutAsync()
            {
                await _client.LogoutAsync();

                Name = null;
                Email = null;
                IsAuthenticated = false;
            }

            private void OnPropertyChanged([CallerMemberName] string propertyName = null)
            {
                PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
            }
        }
        ```
      </Tab>

      <Tab title="MainPage.xaml">
        Configura la UI en `MainPage.xaml`:

        ```xml MainPage.xaml lines
        <?xml version="1.0" encoding="utf-8" ?>
        <ContentPage xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
                     xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
                     x:Class="MyMauiApp.MainPage">

            <VerticalStackLayout Spacing="20" Padding="30" VerticalOptions="Center">

                <!-- Se muestra cuando la sesión está cerrada -->
                <Button Text="Log In"
                        Command="{Binding LoginCommand}"
                        IsVisible="{Binding IsNotAuthenticated}" />

                <!-- Se muestra cuando la sesión está iniciada -->
                <Label Text="{Binding Name, StringFormat='Welcome, {0}!'}"
                       IsVisible="{Binding IsAuthenticated}"
                       FontSize="24"
                       HorizontalOptions="Center" />

                <Label Text="{Binding Email}"
                       IsVisible="{Binding IsAuthenticated}"
                       HorizontalOptions="Center" />

                <Button Text="Log Out"
                        Command="{Binding LogoutCommand}"
                        IsVisible="{Binding IsAuthenticated}" />

            </VerticalStackLayout>
        </ContentPage>
        ```
      </Tab>

      <Tab title="MainPage.xaml.cs">
        Establece el BindingContext en `MainPage.xaml.cs`:

        ```csharp MainPage.xaml.cs lines
        namespace MyMauiApp;

        public partial class MainPage : ContentPage
        {
            public MainPage(MainPageViewModel viewModel)
            {
                InitializeComponent();
                BindingContext = viewModel;
            }
        }
        ```
      </Tab>
    </Tabs>

    <Check>
      Tu proyecto ahora tiene un ViewModel con comandos de inicio y cierre de sesión, una página XAML enlazada a datos y el code-behind conectado.
    </Check>
  </Step>

  <Step title="Registrar servicios e inicializar el cliente de Auth0" stepNumber={6}>
    Ahora registra `Auth0Client`, el ViewModel y la página mediante inyección de dependencias en `MauiProgram.cs`. De este modo, todo queda conectado para que el cliente de Auth0 se inyecte en el ViewModel y el ViewModel se inyecte en la página:

    ```csharp MauiProgram.cs lines
    using Auth0.OidcClient;
    using MyMauiApp.ViewModels;

    public static class MauiProgram
    {
        public static MauiApp CreateMauiApp()
        {
            var builder = MauiApp.CreateBuilder();
            builder
                .UseMauiApp<App>()
                .ConfigureFonts(fonts =>
                {
                    fonts.AddFont("OpenSans-Regular.ttf", "OpenSansRegular");
                });

            // Configura el cliente Auth0 con tus credenciales del Paso 1
            builder.Services.AddSingleton(new Auth0Client(new Auth0ClientOptions
            {
                Domain = "{yourDomain}",
                ClientId = "{yourClientId}",
                RedirectUri = "myapp://callback",
                PostLogoutRedirectUri = "myapp://callback"
            }));

            // Registra la página y el ViewModel creados en el Paso 5
            builder.Services.AddTransient<MainPage>();
            builder.Services.AddTransient<MainPageViewModel>();

            return builder.Build();
        }
    }
    ```

    Reemplaza `{yourDomain}` y `{yourClientId}` por los valores de la configuración de tu aplicación de Auth0 (Paso 1).
    `RedirectUri` y `PostLogoutRedirectUri` son **obligatorios** para las aplicaciones MAUI. Usa la misma URL de devolución de llamada que ingresaste en el Auth0 Dashboard.
  </Step>

  <Step title="Ejecuta tu aplicación" stepNumber={7}>
    Compila y ejecuta tu aplicación .NET MAUI

    **Flujo esperado:**

    1. La aplicación se inicia y muestra el botón **Iniciar sesión**
    2. Pulsa **Iniciar sesión** → se abre el navegador del sistema con Auth0 Universal Login
    3. Completa la autenticación (regístrate o inicia sesión)
    4. El navegador te redirige de nuevo a tu aplicación
    5. La aplicación muestra tu nombre y tu correo electrónico, junto con un botón **Cerrar sesión**
  </Step>
</Steps>

<Check>
  Ahora ya tienes una experiencia de inicio de sesión con Auth0 totalmente funcional en tu aplicación .NET MAUI.
</Check>

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="La URL de devolución de llamada no coincide">
    **Síntoma**: El navegador muestra &quot;Callback URL mismatch. The provided redirect&#95;uri is not in the list of allowed callback URLs.&quot;

    **Solución:**

    1. Confirma que el **ID de cliente** en tu código coincida con la aplicación que configuraste en el Auth0 Dashboard
    2. Borra el campo **Allowed Callback URLs** y vuelve a escribir `myapp://callback` manualmente — al copiar y pegar suelen introducirse espacios finales invisibles o saltos de línea
    3. Asegúrate de que la coincidencia sea exacta: sin barra final, solo en minúsculas y sin espacios en blanco
    4. Selecciona **Save Changes** en el Dashboard y verifica que el valor se haya guardado
    5. Revisa la barra de direcciones del navegador para ver el parámetro de consulta `redirect_uri` y comprobar qué está enviando realmente tu aplicación
  </Accordion>

  <Accordion title="Android: la aplicación no vuelve desde el navegador">
    **Síntoma**: El navegador se abre para iniciar sesión, pero nunca redirige de vuelta a la aplicación.

    **Solución:**

    1. Verifica que `DataScheme` en `WebAuthenticatorActivity.cs` coincida con el esquema de tu `RedirectUri`
    2. Asegúrate de que la Activity tenga `Exported = true`
    3. Confirma que **Allowed Callback URLs** en Auth0 Dashboard coincidan exactamente
  </Accordion>

  <Accordion title="Windows: el inicio de sesión parece quedarse bloqueado">
    **Síntoma**: El navegador se abre, pero en lugar de reanudarse, se abre una segunda instancia de la aplicación.

    **Solución:** Asegúrate de que se llame a `Auth0.OidcClient.Platforms.Windows.Activator.Default.CheckRedirectionActivation()` como la **primera línea** del constructor de `App` en `Platforms/Windows/App.xaml.cs`, y de que el nombre del protocolo en `Package.appxmanifest` coincida con el esquema de tu URI de devolución de llamada.
  </Accordion>

  <Accordion title="Windows: la activación del protocolo no funciona">
    **Síntoma**: Después de iniciar sesión, el navegador muestra un error o no ocurre nada — la aplicación nunca recibe la devolución de llamada.

    **Solución:** Tu aplicación debe ser **empaquetada** (MSIX). Revisa tu archivo `.csproj` para comprobar si tiene un elemento `<WindowsPackageType>`:

    * Si está configurado como `None`, la activación del protocolo no está disponible. Elimina la línea o cámbiala a `<WindowsPackageType>MSIX</WindowsPackageType>`.
    * Si el elemento no está presente, tu aplicación ya está empaquetada de forma predeterminada — verifica que `Package.appxmanifest` contenga la extensión `<uap:Protocol>` del Paso 4.
  </Accordion>

  <Accordion title="Faltan claims en el perfil de usuario">
    **Síntoma**: Los claims `name`, `email` o `picture` no están presentes en `loginResult.User`.

    **Solución:** Verifica que `openid profile email` estén incluidos en `Auth0ClientOptions.Scope`. Si has personalizado el scope, asegúrate de que `openid` esté siempre presente.
  </Accordion>
</AccordionGroup>

<div id="next-steps">
  ## Próximos pasos
</div>

Ahora ya tienes una integración de Auth0 funcionando en tu aplicación .NET MAUI. Explora estos temas para ampliar tu implementación:

<Accordion title="Tokens de actualización">
  El SDK de Auth0 para MAUI admite tokens de actualización para renovar sesiones de forma silenciosa sin volver a pedirle nada al usuario.

  ### Habilitar tokens de actualización

  Agrega `offline_access` a la propiedad `Scope`:

  ```csharp expandable lines
  builder.Services.AddSingleton(new Auth0Client(new Auth0ClientOptions
  {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}",
      RedirectUri = "myapp://callback",
      PostLogoutRedirectUri = "myapp://callback",
      Scope = "openid profile email offline_access"
  }));
  ```

  ### Usar tokens de actualización

  Después del inicio de sesión, almacena el token de actualización y úsalo para renovar la sesión de forma silenciosa:

  ```csharp expandable lines
  // Después del inicio de sesión
  var refreshToken = loginResult.RefreshToken;

  // Más adelante, renueva la sesión
  var refreshResult = await _client.RefreshTokenAsync(refreshToken);

  if (!refreshResult.IsError)
  {
      var newAccessToken = refreshResult.AccessToken;
      var newIdToken = refreshResult.IdentityToken;
  }
  ```

  <Info>
    Si `RefreshToken` es `null` después del inicio de sesión, asegúrate de que **Allow Offline Access** esté habilitado en la configuración de tu API en el Auth0 Dashboard (cuando uses un parámetro `audience`).
  </Info>
</Accordion>

<Accordion title="Llamar a una API protegida">
  Para obtener un token de acceso limitado a tu API, configura `Scope` y pasa un parámetro `audience` a `LoginAsync()`:

  ```csharp expandable lines
  var loginResult = await _client.LoginAsync(new
  {
      audience = "https://myapi.example.com"
  });

  // El token de acceso ahora está limitado a tu API
  var accessToken = loginResult.AccessToken;

  var httpClient = new HttpClient();
  httpClient.DefaultRequestHeaders.Authorization =
      new System.Net.Http.Headers.AuthenticationHeaderValue("Bearer", accessToken);

  var response = await httpClient.GetAsync("https://myapi.example.com/posts");
  ```
</Accordion>

<Accordion title="Organizations (B2B/empresarial)">
  Autentica a los usuarios dentro de una Auth0 Organization específica:

  ```csharp expandable lines
  var loginResult = await _client.LoginAsync(new
  {
      organization = "org_abc123"
  });
  ```

  Para obtener más información, consulta [Organizations](/docs/es/manage-users/organizations).
</Accordion>

<Accordion title="Forzar la reautenticación">
  Usa `MaxAge` para forzar la reautenticación después del tiempo especificado:

  ```csharp expandable lines
  new Auth0ClientOptions
  {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}",
      RedirectUri = "myapp://callback",
      PostLogoutRedirectUri = "myapp://callback",
      MaxAge = TimeSpan.FromMinutes(30)
  }
  ```
</Accordion>

<Accordion title="Personalizar Universal Login">
  Personaliza la página de inicio de sesión de Auth0 para que refleje tu marca, incluidos los colores, los logotipos y el texto.

  Para obtener más información, consulta [Personalizar Universal Login](/docs/es/customize/universal-login-pages).
</Accordion>
