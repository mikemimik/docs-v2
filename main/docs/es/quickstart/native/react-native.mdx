<div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Crear un nuevo proyecto de React Native" stepNumber={1}>
    Crea un nuevo proyecto de React Native para este Quickstart.

    **En tu terminal:**

    ```bash
    npx @react-native-community/cli init Auth0ReactNativeSample
    cd Auth0ReactNativeSample
    ```

    Configura tu proyecto con:

    * **Nombre**: `Auth0ReactNativeSample`
    * **Nombre del paquete**: `com.auth0.samples.reactnative`

    <Tip>
      Esto crea una aplicación de React Native con la versión estable más reciente. El SDK de Auth0 requiere React Native 0.78.0+ y React 19.0.0+.
    </Tip>
  </Step>

  <Step title="Instalar el SDK de Auth0" stepNumber={2}>
    Añade el SDK de React Native de Auth0 a tu proyecto.

    ```bash
    npm install react-native-auth0
    ```

    Para iOS, instala las dependencias nativas:

    ```bash
    cd ios && pod install && cd ..
    ```

    <Tip>
      El SDK se enlaza automáticamente en ambas plataformas. El paso `pod install` instala los módulos nativos de iOS necesarios (Auth0.swift, JWTDecode, SimpleKeychain).
    </Tip>
  </Step>

  <Step title="Configura tu aplicación de Auth0" stepNumber={3}>
    Crea y configura una aplicación de Auth0 para que funcione con tu app de React Native.

    1. Ve al [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. Haz clic en **Applications** &gt; **Applications** &gt; **Create Application**
    3. En la ventana emergente, introduce un nombre para tu app (p. ej., `Auth0 React Native Sample`), selecciona `Native` como tipo de app y haz clic en **Create**
    4. Ve a la pestaña **Settings** en la página de detalles de la aplicación
    5. Anota los valores de **Domain** y **Client ID**

    **Allowed Callback URLs:**

    ```
    org.reactjs.native.example.auth0reactnativesample.auth0://{yourDomain}/ios/org.reactjs.native.example.auth0reactnativesample/callback,
    com.auth0reactnativesample.auth0://{yourDomain}/android/com.auth0reactnativesample/callback
    ```

    **Allowed Logout URLs:**

    ```
    org.reactjs.native.example.auth0reactnativesample.auth0://{yourDomain}/ios/org.reactjs.native.example.auth0reactnativesample/callback,
    com.auth0reactnativesample.auth0://{yourDomain}/android/com.auth0reactnativesample/callback
    ```

    Reemplaza `{yourDomain}` por tu dominio de Auth0 real (por ejemplo, `dev-abc123.us.auth0.com`).

    <Info>
      **Allowed Callback URLs** son una medida de seguridad fundamental para garantizar que los usuarios regresen de forma segura a tu aplicación después de la autenticación. Sin una URL que coincida, el proceso de inicio de sesión fallará y los usuarios verán una página de error de Auth0 en lugar de acceder a tu aplicación.

      **Allowed Logout URLs** son esenciales para ofrecer una experiencia de usuario fluida al cerrar sesión. Sin una URL que coincida, los usuarios no serán redirigidos de vuelta a tu aplicación después del cierre de sesión y, en su lugar, se quedarán en una página genérica de Auth0.

      El esquema de URL incluye `.auth0` después de tu identificador del paquete para garantizar que la devolución de llamada se dirija a tu aplicación específica. Para este Quickstart, el identificador del paquete es `org.reactjs.native.example.auth0reactnativesample`.
    </Info>

    <Warning>
      **Importante**: El esquema de la URL de devolución de llamada debe incluir `.auth0` después de tu identificador del paquete (por ejemplo, `org.reactjs.native.example.auth0reactnativesample.auth0://`). Esto es necesario para que el SDK gestione correctamente la devolución de llamada de autenticación.
    </Warning>
  </Step>

  <Step title="Configurar plataformas nativas" stepNumber={4}>
    Configura iOS y Android para que gestionen la devolución de llamada de autenticación.

    **Configuración de Android:**

    Abre `android/app/build.gradle` y añade los marcadores de posición del manifiesto dentro de `defaultConfig`:

    ```groovy android/app/build.gradle lines
    android {
        defaultConfig {
            applicationId "com.auth0reactnativesample"
            // ... otra configuración
            
            // Agregar los marcadores de posición del manifiesto de Auth0
            manifestPlaceholders = [
                auth0Domain: "{yourDomain}",
                auth0Scheme: "${applicationId}.auth0"
            ]
        }
    }
    ```

    Reemplaza `{yourDomain}` por tu dominio de Auth0 (por ejemplo, `dev-abc123.us.auth0.com`).

    **Configuración de iOS:**

    <Tabs>
      <Tab title="AppDelegate.mm (Objective-C)">
        Abre `ios/Auth0ReactNativeSample/AppDelegate.mm` y agrega el método para gestionar la URL:

        ```objc ios/Auth0ReactNativeSample/AppDelegate.mm lines
        #import <React/RCTLinkingManager.h>

        // Agrega este método dentro del bloque @implementation
        - (BOOL)application:(UIApplication *)app
                    openURL:(NSURL *)url
                    options:(NSDictionary<UIApplicationOpenURLOptionsKey, id> *)options
        {
          return [RCTLinkingManager application:app openURL:url options:options];
        }
        ```
      </Tab>

      <Tab title="AppDelegate.swift (Swift)">
        Abre `ios/Auth0ReactNativeSample/AppDelegate.swift` y agrega el método para gestionar la URL:

        ```swift ios/Auth0ReactNativeSample/AppDelegate.swift lines
        import React

        // Agrega este método dentro de la clase AppDelegate
        func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey : Any] = [:]) -> Bool {
          return RCTLinkingManager.application(app, open: url, options: options)
        }
        ```
      </Tab>
    </Tabs>

    Abre `ios/Auth0ReactNativeSample/Info.plist` y agrega el esquema de URL. Hazlo antes de la etiqueta de cierre `</dict>`:

    ```xml ios/Auth0ReactNativeSample/Info.plist lines
    <key>CFBundleURLTypes</key>
    <array>
      <dict>
        <key>CFBundleTypeRole</key>
        <string>None</string>
        <key>CFBundleURLName</key>
        <string>auth0</string>
        <key>CFBundleURLSchemes</key>
        <array>
          <string>$(PRODUCT_BUNDLE_IDENTIFIER).auth0</string>
        </array>
      </dict>
    </array>
    ```

    <Info>
      El esquema de URL usa tu identificador del paquete con `.auth0` al final. Esto garantiza que la devolución de llamada se dirija a tu aplicación una vez que la autenticación haya finalizado en el navegador.
    </Info>
  </Step>

  <Step title="Componente de la aplicación de Setup" stepNumber={5}>
    Configura el componente principal de tu aplicación según el enfoque de implementación que hayas elegido.

    <Tabs>
      <Tab title="Basado en Hooks (con Provider)">
        Reemplaza el contenido de `App.tsx` y envuelve la aplicación con el componente `Auth0Provider`:

        ```tsx App.tsx lines
        import React from 'react';
        import {Auth0Provider} from 'react-native-auth0';
        import {SafeAreaView, StyleSheet} from 'react-native';
        import MainScreen from './src/MainScreen';

        const App = () => {
          return (
            <Auth0Provider
              domain="{yourDomain}"
              clientId="{yourClientId}">
              <SafeAreaView style={styles.container}>
                <MainScreen />
              </SafeAreaView>
            </Auth0Provider>
          );
        };

        const styles = StyleSheet.create({
          container: {
            flex: 1,
            backgroundColor: '#fff',
          },
        });

        export default App;
        ```

        Reemplaza `{yourDomain}` por tu dominio de Auth0 y `{yourClientId}` por tu ID de cliente del Auth0 Dashboard.

        <Tip>
          `Auth0Provider` inicializa el SDK y proporciona el contexto de autenticación a todos los componentes secundarios mediante el hook `useAuth0`.
        </Tip>
      </Tab>

      <Tab title="Basado en clases (sin Provider)">
        Reemplaza el contenido de `App.tsx` por un contenedor sencillo para tu componente basado en clases:

        ```tsx App.tsx lines
        import React from 'react';
        import {SafeAreaView, StyleSheet} from 'react-native';
        import MainScreen from './src/MainScreen';

        const App = () => {
          return (
            <SafeAreaView style={styles.container}>
              <MainScreen />
            </SafeAreaView>
          );
        };

        const styles = StyleSheet.create({
          container: {
            flex: 1,
            backgroundColor: '#fff',
          },
        });

        export default App;
        ```

        <Tip>
          El enfoque basado en clases no requiere `Auth0Provider`, ya que usa directamente la instancia de la clase `Auth0`. La configuración de Auth0 estará en el componente MainScreen.
        </Tip>
      </Tab>
    </Tabs>
  </Step>

        import {
          View,
          Text,
          Button,
          StyleSheet,
          ActivityIndicator,
          Image,
        } from 'react-native';
        import Auth0, {Credentials} from 'react-native-auth0';

        const auth0 = new Auth0({
          domain: '{yourDomain}',
          clientId: '{yourClientId}',
        });

        interface User {
          name?: string;
          email?: string;
          picture?: string;

          user: User | null;
          isLoading: boolean;

              user: null,
              isLoading: true,
            };

            await this.checkAuthStatus();

            try {
              const hasValidCredentials = await auth0.credentialsManager.hasValidCredentials();
              if (hasValidCredentials) {
                const credentials = await auth0.credentialsManager.getCredentials();

  ## Solución de problemas y opciones avanzadas

<Accordion title="Problemas comunes y soluciones">
  ### Error de &quot;URL de devolución de llamada no coincide&quot;

  **Solución:**

  1. Verifica que la URL exacta (incluido el sufijo `.auth0`) esté en **Allowed Callback URLs** en tu Auth0 Dashboard
  2. Asegúrate de haber agregado tanto la URL de iOS como la de Android
  3. Comprueba que `{yourDomain}` se haya reemplazado por tu dominio real de Auth0

  ### La app no vuelve después del inicio de sesión (iOS)

  **Solución:**

  1. Comprueba que `Info.plist` tenga la entrada `CFBundleURLSchemes` con `$(PRODUCT_BUNDLE_IDENTIFIER).auth0`
  2. Verifica que `AppDelegate.mm` incluya el método para manejar URL
  3. Asegúrate de que el esquema de URL coincida con tu identificador del paquete

  ### La compilación de Android falla

  **Solución:**

  1. Agrega los manifest placeholders `auth0Domain` y `auth0Scheme` a `build.gradle`
  2. Sincroniza el proyecto con los archivos de Gradle
  3. Limpia la compilación: `./gradlew clean`

  ### Error de &quot;PKCE no permitido&quot;

  **Solución:**

  1. Ve a Auth0 Dashboard → Applications → Tu aplicación
  2. Cambia el tipo de aplicación a **Native**
  3. Guarda los cambios y vuelve a intentarlo

  ### La instalación de Pod falla (iOS)

  **Solución:**

  1. Actualiza CocoaPods: `sudo gem install cocoapods`
  2. Actualiza el repositorio de pods: `pod install --repo-update`
  3. Si el problema persiste, elimina `Podfile.lock` y la carpeta `ios/Pods`, y luego ejecuta `pod install` de nuevo

  ### Error de cancelación por parte del usuario

  Manéjalo correctamente en tu función de login:

  ```tsx expandable
  const login = async () => {
    try {
      await authorize({scope: 'openid profile email'});
    } catch (e) {
      if (e.message === 'a0.session.user_cancelled') {
        // El usuario cerró la pantalla de login: manejar correctamente
        console.log('Login cancelled by user');
      } else {
        console.error('Login failed:', e);
      }
    }
  };
  ```

  ### Cuadro de alerta de iOS

  En iOS, los usuarios ven un cuadro de diálogo de permisos: *&quot;App Name&quot; Wants to Use &quot;auth0.com&quot; to Sign In*. Este es el comportamiento esperado de `ASWebAuthenticationSession`. Los usuarios deben tocar **Continuar** para seguir.

  Para personalizar este comportamiento, puedes usar sesiones efímeras (desactiva SSO):

  ```tsx
  await authorize({scope: 'openid profile email'}, {ephemeralSession: true});
  ```
</Accordion>

<Accordion title="Obtener tokens de acceso">
  Usa el método `getCredentials()` para obtener tokens para llamadas a la API:

  ```tsx expandable lines
  import {useAuth0} from 'react-native-auth0';

  const MyComponent = () => {
    const {getCredentials} = useAuth0();

    const callApi = async () => {
      try {
        const credentials = await getCredentials();
        const response = await fetch('https://your-api.com/endpoint', {
          headers: {
            Authorization: `Bearer ${credentials.accessToken}`,
          },
        });
        // Manejar la respuesta
      } catch (e) {
        console.error('Failed to get credentials', e);
      }
    };
  };
  ```

  <Tip>
    Incluye el scope `offline_access` durante el login para recibir un token de actualización: `authorize({scope: 'openid profile email offline_access'})`. Esto habilita la renovación automática del token.
  </Tip>
</Accordion>

<Accordion title="Comprobar el estado de autenticación al abrir la app">
  Usa `hasValidCredentials()` para comprobar si el usuario ya inició sesión:

  ```tsx expandable lines
  import {useAuth0} from 'react-native-auth0';
  import {useEffect} from 'react';

  const App = () => {
    const {hasValidCredentials, getCredentials} = useAuth0();

    useEffect(() => {
      const checkAuth = async () => {
        const isLoggedIn = await hasValidCredentials();
        if (isLoggedIn) {
          const credentials = await getCredentials();
          // El usuario está autenticado, cargar sus datos
        }
      };
      checkAuth();
    }, []);
  };
  ```
</Accordion>

<Accordion title="Despliegue en producción">
  ### Antes de desplegar en producción

  **Use URL de devolución de llamada HTTPS** para mayor seguridad:

  ```text
  https://{yourDomain}/ios/{bundleId}/callback
  https://{yourDomain}/android/{packageName}/callback
  ```

  **Configure Android App Links** en Auth0 Dashboard:

  * Settings → Advanced Settings → Device Settings
  * Agregue la huella digital SHA-256 de su aplicación

  **Configure iOS Universal Links:**

  * Agregue la capacidad Associated Domains en Xcode
  * Agregue `webcredentials:{yourDomain}` a Associated Domains

  **Habilite la autenticación biométrica** para aplicaciones sensibles con `localAuthenticationOptions` en `Auth0Provider`

  **Revise la configuración de seguridad** en Auth0 Dashboard:

  * Habilite **OIDC Conformant** en Advanced Settings
  * Configure adecuadamente la **expiración del token**
  * Configure **Brute Force Protection**
  * Haga pruebas en varios dispositivos y versiones del sistema operativo
  * Implemente un manejo adecuado de errores para fallas de red
