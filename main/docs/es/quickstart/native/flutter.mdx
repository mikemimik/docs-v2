Esta guía muestra cómo integrar Auth0 con una aplicación Flutter mediante el [Auth0 Flutter SDK](https://github.com/auth0/auth0-flutter). Esta guía cubre la configuración para las plataformas **Android**, **iOS** y **macOS**.

<Info>
  El Auth0 Flutter SDK también es compatible con **Web** y **Windows** (Beta). Estas plataformas tienen Quickstarts específicos:

  * [Flutter Web Quickstart](/docs/es/quickstart/spa/flutter) — para aplicaciones Flutter orientadas al navegador
  * [Flutter Windows Quickstart](/docs/es/quickstart/native/flutter-windows) — para aplicaciones de escritorio Flutter en Windows (Beta)
</Info>

<div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Crea un nuevo proyecto de Flutter" stepNumber={1}>
    Crea un nuevo proyecto de Flutter para este Quickstart.

    **En la terminal:**

    1. Ve al directorio de trabajo
    2. Ejecuta: `flutter create auth0_flutter_sample`
    3. Entra en el proyecto: `cd auth0_flutter_sample`
    4. Ábrelo en tu IDE:
       * **VS Code**: `code .`
       * **Android Studio**: `open -a "Android Studio" .`

    ```shellscript
    # Crear nuevo proyecto Flutter
    flutter create auth0_flutter_sample

    # Navegar al directorio del proyecto
    cd auth0_flutter_sample

    # Abrir en VS Code
    code .
    ```

    <Tip>
      Esto crea una aplicación moderna de Flutter con la estructura de proyecto más actual. Ejecuta `flutter doctor` para comprobar que tu entorno esté configurado correctamente.
    </Tip>
  </Step>

  <Step title="Instala el Auth0 Flutter SDK" stepNumber={2}>
    Añade el Auth0 Flutter SDK a tu proyecto usando la CLI de Flutter.

    ```shellscript
    flutter pub add auth0_flutter
    ```

    Esto añadirá `auth0_flutter` a las dependencias de tu `pubspec.yaml`:

    ```yaml pubspec.yaml lines
    dependencies:
      auth0_flutter: ^2.0.0-beta.1
    ```

    <Tip>
      El Auth0 Flutter SDK requiere **Flutter 3.24.0+** y **Dart 3.5.0+**. Ejecuta `flutter doctor` para comprobar que tu entorno cumple estos requisitos.
    </Tip>
  </Step>

  <Step title="Configura tu Auth0 App" stepNumber={3}>
    A continuación, debes crear una nueva aplicación en tu tenant de Auth0 y configurar las URL de devolución de llamada.

    1. Ve al [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. Haz clic en **Applications** &gt; **Applications** &gt; **Create Application**
    3. En la ventana emergente, introduce un nombre para tu aplicación, selecciona `Native` como tipo de aplicación y haz clic en **Create**
    4. Ve a la pestaña **Settings** en la página de detalles de la aplicación
    5. Anota los valores de **Domain** y **ID de cliente**; los necesitarás más adelante

    En la pestaña **Settings**, configura las siguientes URL según la(s) plataforma(s) de destino:

    **Allowed Callback URLs:**

    <Tabs>
      <Tab title="Android">
        ```text
        https://{yourDomain}/android/{yourPackageName}/callback
        ```
      </Tab>

      <Tab title="iOS">
        ```text
        https://{yourDomain}/ios/{yourBundleIdentifier}/callback,
        {yourBundleIdentifier}://{yourDomain}/ios/{yourBundleIdentifier}/callback
        ```
      </Tab>

      <Tab title="macOS">
        ```text
        https://{yourDomain}/macos/{yourBundleIdentifier}/callback,
        {yourBundleIdentifier}://{yourDomain}/macos/{yourBundleIdentifier}/callback
        ```
      </Tab>
    </Tabs>

    **Allowed Logout URLs:**

    Agrega las mismas URL de la configuración de devolución de llamada anterior al campo **Allowed Logout URLs**.

    <Info>
      **Allowed Callback URLs** son una medida de seguridad fundamental para garantizar que los usuarios vuelvan de forma segura a tu aplicación después de la autenticación. Sin una URL coincidente, el proceso de inicio de sesión fallará.

      **Allowed Logout URLs** son esenciales para ofrecer una experiencia de usuario fluida al cerrar sesión. Sin una URL coincidente, los usuarios no serán redirigidos de vuelta a tu aplicación después del cierre de sesión.

      Por ejemplo, si tu dominio de Auth0 es `example.us.auth0.com` y el nombre del paquete de Android es `com.example.myapp`, tu URL de devolución de llamada de Android sería: `https://example.us.auth0.com/android/com.example.myapp/callback`
    </Info>

    <Warning>
      **Importante**: Asegúrate de que el nombre del paquete (Android) o el identificador del paquete (iOS/macOS) en tus URL de devolución de llamada coincida con el identificador real de tu aplicación. Si la autenticación falla, verifica que estos valores sean idénticos.
    </Warning>
  </Step>

  <Step title="Configure su aplicación" stepNumber={4}>
    Se requiere una configuración específica de cada plataforma para habilitar el flujo de autenticación. Sigue las instrucciones correspondientes a cada plataforma que quieras usar.

    <Tabs>
      <Tab title="Android">
        Abre el archivo `android/app/build.gradle` y agrega los siguientes manifest placeholders dentro de `android > defaultConfig`:

        ```groovy android/app/build.gradle lines
        android {
            // ...
            defaultConfig {
                // Agrega la siguiente línea
                manifestPlaceholders += [auth0Domain: "{yourDomain}", auth0Scheme: "https"]
            }
            // ...
        }
        ```

        Reemplaza `{yourDomain}` por tu dominio de Auth0 (por ejemplo, `example.us.auth0.com`).

        **esquema https**

        Para usar el esquema `https` en tu URL de devolución de llamada, configura [Android App Links](https://auth0.com/docs/get-started/applications/enable-android-app-links-support) para tu aplicación.

        **Para la autenticación biométrica (opcional):**

        Si planeas usar autenticación biométrica, actualiza tu `MainActivity.kt` para que extienda `FlutterFragmentActivity`:

        ```kotlin android/app/src/main/kotlin/.../MainActivity.kt lines
        import io.flutter.embedding.android.FlutterFragmentActivity

        class MainActivity: FlutterFragmentActivity() {
        }
        ```
      </Tab>

      <Tab title="iOS">
        Para usar Universal Links (URL de devolución de llamada HTTPS) en iOS 17.4 o versiones posteriores, configura un dominio asociado:

        1. Abre tu aplicación en Xcode: `open ios/Runner.xcworkspace`
        2. Selecciona el target **Runner** y ve a **Signing &amp; Capabilities**
        3. Haz clic en **+ Capability** y agrega **Associated Domains**
        4. Agrega la entrada: `webcredentials:{yourDomain}`
        5. En el [Auth0 Dashboard](https://manage.auth0.com/#/applications), ve a **Settings &gt; Advanced Settings &gt; Device Settings**
        6. En la sección **iOS**, establece tu **Team ID** y **App ID** (identificador del paquete)

        <Info>
          La configuración de Universal Links es opcional. Omite este paso si no necesitas compatibilidad con Universal Links; el SDK usará automáticamente esquemas de URL personalizados como alternativa.
        </Info>
      </Tab>

      <Tab title="macOS">
        Sigue los mismos pasos que en iOS, pero abre `macos/Runner.xcworkspace`.
      </Tab>
    </Tabs>

    <Warning>
      **Android**: Asegúrate de que el valor de `auth0Domain` coincida exactamente con tu dominio de Auth0. Si la autenticación falla, verifica que este valor sea idéntico al dominio que aparece en tu Auth0 Dashboard.

      **iOS/macOS**: Universal Links requiere una cuenta de pago de Apple Developer y iOS 17.4+/macOS 14.4+. En versiones anteriores, el SDK usará automáticamente esquemas de URL personalizados.
    </Warning>
  </Step>

  <Step title="Implement inicio de sesión y cierre de sesión" stepNumber={5}>
    [Universal Login](https://auth0.com/docs/authenticate/login/auth0-universal-login) es la forma más sencilla de configurar la autenticación en tu aplicación. Te recomendamos usarlo para obtener la mejor experiencia, la mayor seguridad y la gama más completa de funcionalidades.

    **Implement Login:**

    Importa el Auth0 Flutter SDK y crea una instancia de `Auth0`:

    ```dart lib/auth_service.dart lines
    import 'package:auth0_flutter/auth0_flutter.dart';

    class AuthService {
      final auth0 = Auth0('{yourDomain}', '{yourClientId}');

      Future<Credentials> login() async {
        final credentials = await auth0.webAuthentication().login(useHTTPS: true);

        // Token de acceso -> credentials.accessToken
        // Token de identidad -> credentials.idToken
        // Perfil de usuario -> credentials.user

        return credentials;
      }
    }
    ```

    **Cierre de sesión de Implement:**

    ```dart lib/auth_service.dart lines
    Future<void> logout() async {
      await auth0.webAuthentication().logout(useHTTPS: true);

      // El usuario ha cerrado sesión
      // Las credenciales han sido eliminadas del almacenamiento seguro
    }
    ```

    <Info>
      **iOS/macOS**: El parámetro `useHTTPS: true` habilita Universal Links en iOS 17.4+ y macOS 14.4+ para reforzar la seguridad.

      **Android**: Si usas un esquema personalizado, pásalo al método `login` para que el SDK pueda redirigir correctamente a la página de inicio de sesión y de regreso:

      ```dart
      await auth0.webAuthentication(scheme: 'YOUR CUSTOM SCHEME').login();
      ```
    </Info>
  </Step>

  <Step title="Mostrar información del perfil de usuario" stepNumber={6}>
    El perfil de usuario se obtiene automáticamente cuando el usuario inicia sesión. El objeto `Credentials` contiene una propiedad `user` con toda la información del perfil de usuario, que se rellena al decodificar el token de identidad.

    ```dart lib/profile_screen.dart lines
    void displayUserProfile(Credentials credentials) {
      final user = credentials.user;
      
      print('User ID: ${user.sub}');
      print('Email: ${user.email}');
      print('Name: ${user.name}');
      print('Picture: ${user.pictureUrl}');
      print('Nickname: ${user.nickname}');
    }
    ```

    <Tip>
      Solicita los scopes adecuados al iniciar sesión para acceder a campos específicos del perfil de usuario. Los scopes predeterminados son `openid`, `profile`, `email` y `offline_access`.
    </Tip>
  </Step>

  <Step title="Ejecuta tu aplicación" stepNumber={7}>
    Compila y ejecuta tu aplicación de Flutter.

    **En tu terminal:**

    ```shellscript
    # Listar dispositivos disponibles
    flutter devices

    # Ejecutar en Android
    flutter run -d android

    # Ejecutar en el simulador de iOS
    flutter run -d ios

    # Ejecutar en macOS
    flutter run -d macos
    ```

    **Flujo esperado:**

    1. La aplicación se inicia con tu interfaz de inicio de sesión
    2. El usuario toca **Iniciar sesión** → Se abre el navegador o una pestaña Custom con Auth0 Universal Login
    3. El usuario completa la autenticación
    4. El navegador redirige de vuelta a tu aplicación
    5. El usuario ya está autenticado y las credenciales quedan almacenadas
  </Step>
</Steps>

<Check>
  **Punto de control**

  Ahora deberías tener una experiencia de inicio de sesión de Auth0 completamente funcional en tu aplicación de Flutter. La app usa autenticación segura a través del navegador y almacena automáticamente las credenciales para mantener la sesión.
</Check>

  ## Solución de problemas y uso avanzado
</div>

<Accordion title="Problemas comunes y soluciones">
  ### La URL de devolución de llamada no coincide

  **Síntoma**: Error &quot;redirect&#95;uri&#95;mismatch&quot; o la autenticación falla sin mostrar ningún error.

  **Soluciones:**

  1. Comprueba que **Allowed Callback URLs** en Auth0 Dashboard coincidan exactamente con la configuración de tu aplicación
  2. Verifica el esquema (`https://` vs `http://`)
  3. Asegúrate de que el nombre del paquete (Android) o el identificador del paquete (iOS/macOS) sea correcto
  4. Comprueba si hay barras finales (`/`)

  ### Android: Chrome Custom Tab no se abre

  **Síntoma**: No ocurre nada al llamar a `login()`.

  **Solución:**

  1. Verifica que `manifestPlaceholders` esté configurado correctamente en `build.gradle`
  2. Asegúrate de que el permiso de internet esté en `AndroidManifest.xml`:
     ```xml
     <uses-permission android:name="android.permission.INTERNET" />
     ```
  3. Comprueba que Chrome u otro navegador esté instalado en el dispositivo

  ### iOS: alerta &quot;Open in App&quot;

  **Síntoma**: Aparece una alerta preguntando si quieres abrirlo en tu aplicación.

  **Solución:** Este es el comportamiento esperado con `ASWebAuthenticationSession`. Para eliminarla:

  * Usa Universal Links (requiere iOS 17.4+ y una cuenta de pago de Apple Developer)
  * O establece `useEphemeralSession: true` (desactiva SSO):

  ```dart expandable
  await auth0.webAuthentication().login(
    useHTTPS: true,
    useEphemeralSession: true,
  );
  ```

  ### Autenticación cancelada por el usuario

  Contrólalo correctamente en tu gestión de errores:

  ```dart expandable
  try {
    final credentials = await auth0.webAuthentication().login(useHTTPS: true);
    // Gestionar el inicio de sesión correcto
  } on WebAuthenticationException catch (e) {
    if (e.code == 'USER_CANCELLED') {
      showMessage('Se canceló el inicio de sesión');
    } else {
      showMessage('Error de inicio de sesión: ${e.message}');
    }
  }
  ```
</Accordion>

<Accordion title="Gestión de credenciales">
  Auth0 Flutter SDK incluye un Credentials Manager integrado que almacena de forma segura las credenciales del usuario. En plataformas móviles, las credenciales se cifran y se almacenan en el almacenamiento seguro de la plataforma (Keychain en iOS/macOS y SharedPreferences cifrado en Android).

  ### Comprobar si hay credenciales almacenadas

  Antes de pedirle al usuario que inicie sesión, comprueba si ya existen credenciales válidas:

  ```dart lib/auth_service.dart expandable lines
  Future<bool> checkAuthentication() async {
    return await auth0.credentialsManager.hasValidCredentials();
  }
  ```

  ### Recuperar credenciales almacenadas

  Recupera las credenciales para obtener tokens de acceso o información del usuario. Credentials Manager renueva automáticamente los tokens vencidos cuando es posible:

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> getCredentials() async {
    return await auth0.credentialsManager.credentials();
  }
  ```

  <Tip>
    No necesitas almacenar manualmente las credenciales después del inicio de sesión: el SDK se encarga de ello automáticamente. Tampoco necesitas renovar manualmente los tokens; Credentials Manager los renueva cuando es necesario.
  </Tip>
</Accordion>

<Accordion title="Gestión de errores">
  Gestiona correctamente los errores de autenticación para ofrecer una buena experiencia de usuario.

  ```dart lib/auth_service.dart expandable lines
  import 'package:auth0_flutter/auth0_flutter.dart';

  Future<void> login() async {
    try {
      final credentials = await auth0.webAuthentication().login(useHTTPS: true);
      // Gestionar el inicio de sesión correcto
    } on WebAuthenticationException catch (e) {
      if (e.code == 'USER_CANCELLED') {
        // El usuario canceló el inicio de sesión
        print('El usuario canceló el inicio de sesión');
      } else {
        // Gestionar otros errores
        print('Error de inicio de sesión: ${e.message}');
      }
    }
  }

  Future<Credentials> getCredentials() async {
    try {
      return await auth0.credentialsManager.credentials();
    } on CredentialsManagerException catch (e) {
      if (e.isNoCredentialsFound) {
        // No hay credenciales almacenadas; el usuario debe iniciar sesión
        throw Exception('Inicia sesión primero');
      } else if (e.isTokenRenewFailed) {
        // El token de actualización venció; se requiere reautenticación
        return await auth0.webAuthentication().login(useHTTPS: true);
      }
      rethrow;
    }
  }
  ```
</Accordion>

<Accordion title="Integración avanzada de Flutter">
  ### Seguridad mejorada de credenciales con biometría

  Implementa autenticación biométrica para acceder a las credenciales en dispositivos móviles:

  ```dart lib/secure_auth_service.dart expandable lines
  class SecureAuthService {
    final auth0 = Auth0('{yourDomain}', '{yourClientId}');
    
    Future<void> enableBiometrics() async {
      // Habilita la autenticación local (Face ID, Touch ID, huella digital)
      await auth0.credentialsManager.enableLocalAuthentication(
        title: 'Autentícate para acceder a tu cuenta',
        cancelTitle: 'Cancelar',
        fallbackTitle: 'Usar código de acceso',
      );
    }
    
    Future<Credentials> getCredentialsWithBiometrics() async {
      // Ahora se requerirá autenticación biométrica
      return await auth0.credentialsManager.credentials();
    }
  }
  ```

  <Info>
    **Android**: Requiere que `MainActivity` extienda `FlutterFragmentActivity`, como se configuró en el paso 4.

    **iOS/macOS**: Requiere agregar `NSFaceIDUsageDescription` a tu `Info.plist`.
  </Info>

  ### Scopes y audience personalizados

  Solicita scopes y audience específicos para tu API:

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> loginWithCustomScopes() async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      scopes: {'openid', 'profile', 'email', 'offline_access', 'read:posts'},
      audience: 'https://myapi.example.com',
      parameters: {'prompt': 'login'},
    );
  }
  ```

  ### Organizations (B2B/empresarial)

  Autentica a los usuarios dentro de una organización específica:

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> loginWithOrganization(String organizationId) async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      organizationId: organizationId,
    );
  }

  // O bien, pide al usuario que seleccione una organización
  Future<Credentials> loginWithOrganizationName(String organizationName) async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      organizationName: organizationName,
    );
  }
  ```
</Accordion>

<Accordion title="Despliegue en producción">
  ### Preparación para la App Store

  * Configura Universal Links (iOS) y App Links (Android) para una autenticación fluida
  * Haz pruebas en varios tamaños de dispositivo y versiones del sistema operativo
  * Implementa un manejo adecuado de errores para fallos de red
  * Agrega reglas de ProGuard para Android si usas ofuscación de código
  * Sigue las políticas específicas de App Store/Play Store de cada plataforma

  ### Consideraciones de seguridad

  * Usa el Credentials Manager integrado para almacenar credenciales en producción
  * Habilita la autenticación biométrica para operaciones sensibles
  * Considera el certificate pinning para reforzar la seguridad de la API
  * Implementa un manejo adecuado de la renovación de tokens
  * Usa `useHTTPS: true` para Universal Links en plataformas compatibles
</Accordion>
