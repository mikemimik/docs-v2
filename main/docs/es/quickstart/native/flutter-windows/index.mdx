<Note>
  **Requisitos previos:**

  * Flutter SDK 3.24.0+ y Dart 3.5.0+.
  * Windows 10 o posterior.
  * Visual Studio 2022 con la carga de trabajo **Desktop development with C++**.
  * Una cuenta de Auth0: [regístrate gratis](https://auth0.com/signup).

  Esta funcionalidad está en **beta** (`auth0_flutter` 2.1.0-beta.1). La API puede cambiar antes de su disponibilidad general.
</Note>

Esta guía te muestra cómo añadir inicio de sesión, cierre de sesión y la visualización del perfil de usuario a una aplicación de escritorio de Flutter para Windows con el SDK `auth0_flutter` y OAuth 2.0 [flujo de código de autorización + PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

<div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Crear un proyecto de Flutter para Windows" stepNumber={1}>
    Crea un nuevo proyecto de Flutter con compatibilidad con Windows.

    ```shellscript
    flutter create --platforms=windows my_app
    cd my_app
    ```

    Asegúrate de que Windows esté disponible:

    ```shellscript
    flutter devices
    ```

    Deberías ver un dispositivo de escritorio con Windows en la lista.

    <Tip>
      Ejecuta `flutter doctor` para comprobar que tu entorno esté configurado correctamente y que se detecte Visual Studio 2022 con las herramientas de desarrollo de escritorio para C++.
    </Tip>
  </Step>

  <Step title="Instalar el SDK de Auth0 para Flutter" stepNumber={2}>
    Añade la versión beta del SDK que incluye compatibilidad con Windows:

    ```shellscript
    flutter pub add auth0_flutter:2.1.0-beta.1
    flutter pub add flutter_dotenv
    ```

    Tu `pubspec.yaml` debe incluir:

    <Snippet file="quickstart/native/flutter-windows/pubspec.yaml.mdx" />

    <Tip>
      El SDK de Auth0 para Flutter requiere **Flutter 3.24.0+** y **Dart 3.5.0+**. Además, la plataforma Windows requiere **Visual Studio 2022** con la carga de trabajo de desarrollo de escritorio en C++.
    </Tip>
  </Step>

  <Step title="Configurar Auth0" stepNumber={3}>
    Crea o configura una aplicación de Auth0 con las URL de devolución de llamada necesarias para la autenticación en Windows de escritorio.

    <Tabs>
      <Tab title="Configuración rápida">
        Crea una aplicación **Native** en tu Auth0 Dashboard con la siguiente configuración:

        | Campo                 | Valor                     |
        | --------------------- | ------------------------- |
        | Allowed Callback URLs | `auth0flutter://callback` |
        | Allowed Logout URLs   | `auth0flutter://callback` |

        Tus credenciales:

        * **Domain:** `{yourDomain}`
        * **ID de cliente:** `{yourClientId}`
      </Tab>

      <Tab title="CLI">
        ```shellscript
        auth0 apps create \
          --name "My Flutter Windows App" \
          --type native \
          --callbacks "auth0flutter://callback" \
          --logout-urls "auth0flutter://callback"
        ```
      </Tab>

      <Tab title="Dashboard">
        1. Ve a [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; Applications**.
        2. Selecciona **Create Application**.
        3. Asígnale el nombre &quot;My Flutter Windows App&quot; y elige **Native** como tipo de aplicación.
        4. Ve a la pestaña **Settings** de tu nueva aplicación.
        5. Establece **Allowed Callback URLs** en `auth0flutter://callback`.
        6. Establece **Allowed Logout URLs** en `auth0flutter://callback`.
        7. Desplázate hacia abajo y haz clic en **Save Changes**.
        8. Copia los valores de **Domain** e **ID de cliente** de la parte superior de Application Settings.
      </Tab>
    </Tabs>

    <Info>
      La URL `auth0flutter://callback` es un esquema personalizado que redirige la devolución de llamada del navegador a tu aplicación de escritorio una vez completada la autenticación.
    </Info>
  </Step>

  <Step title="Configurar variables de entorno" stepNumber={4}>
    Crea un archivo `.env` en la raíz de tu proyecto:

    <AuthCodeBlock children={envSnippet} language="shellscript" filename=".env" />

    Añade el archivo `.env` a tus assets de Flutter en `pubspec.yaml`:

    <Warning>
      Nunca incluyas el archivo `.env` en el control de versiones. Añádelo a `.gitignore`.
    </Warning>
  </Step>

  <Step title="Configurar el runner de Windows" stepNumber={5}>
    El flujo de autenticación de Windows requiere la integración de devolución de llamada en el `runner` de tu aplicación. El plugin de Flutter no recibe automáticamente del sistema operativo las activaciones del esquema de protocolo, por lo que debes forzar una sola instancia y agregar el reenvío de URI.

    Reemplaza el contenido de `windows/runner/main.cpp`:

    <Snippet file="quickstart/native/flutter-windows/main.cpp.mdx" />

    Este código:

    * Fuerza una sola instancia mediante un mutex de Windows.
    * Captura las URI `auth0flutter://callback` que se pasan como argumentos de línea de comandos.
    * Reenvía las URI de inicios secundarios a la instancia en ejecución mediante una canalización con nombre.
    * Establece la variable de entorno `PLUGIN_STARTUP_URL` para que el plugin de Auth0 la lea.
  </Step>

  <Step title="Registrar el esquema de URL personalizado" stepNumber={6}>
    Registra `auth0flutter` como un esquema de URL personalizado para que Windows redirija las URI de devolución de llamada a tu aplicación.

    Crea un archivo `windows/url_scheme.reg`:

    <Snippet file="quickstart/native/flutter-windows/url_scheme.reg.mdx" />

    Sustituye `C:\path\to\your\app.exe` por la ruta real de tu ejecutable compilado. Durante el desarrollo, normalmente es:

    ```text
    build\windows\x64\runner\Debug\my_app.exe
    ```

    Haz doble clic en el archivo `.reg` para importarlo en el Registro de Windows.

    <Info>
      Para una distribución en producción, registra el esquema de URL personalizado mediante programación en el instalador de tu aplicación (MSIX, Inno Setup, etc.) en lugar de depender de un archivo `.reg`.
    </Info>

    **Comprueba que el esquema funcione:**

    Abre el Símbolo del sistema y ejecuta:

    ```shellscript
    start auth0flutter://test
    ```

    Tu aplicación debería iniciarse (o pasar a primer plano si ya está ejecutándose).
  </Step>

  <Step title="Implementar el inicio y cierre de sesión" stepNumber={7}>
    Crea `lib/auth_service.dart` para gestionar la autenticación de Windows:

    <Snippet file="quickstart/native/flutter-windows/auth_service.dart.mdx" />

    <Info>
      El SDK de Auth0 Flutter para Windows no admite actualmente la gestión de credenciales. Debes almacenar manualmente las credenciales si necesitas que las sesiones persistan al reiniciar la aplicación.
    </Info>
  </Step>

  <Step title="Mostrar información del perfil de usuario" stepNumber={8}>
    Crea la UI principal de la aplicación en `lib/main.dart`:

    <Snippet file="quickstart/native/flutter-windows/main.dart.mdx" />
  </Step>

  <Step title="Ejecuta tu aplicación" stepNumber={9}>
    Compila y ejecuta la aplicación:

    ```shellscript
    flutter run -d windows
    ```

    **Flujo esperado:**

    1. La app se abre con un botón de **Iniciar sesión**.
    2. Selecciona **Iniciar sesión**. Se abre en el navegador del sistema la página de Auth0 Universal Login.
    3. Completa la autenticación en el navegador.
    4. El navegador redirige a `auth0flutter://callback` → tu aplicación vuelve al primer plano.
    5. Se muestran el nombre, el correo electrónico y la foto de perfil del usuario.

    <Warning>
      Asegúrate de que el esquema de URL personalizado esté registrado (Paso 6) antes de hacer pruebas. Sin él, la devolución de llamada del navegador no puede llegar a tu aplicación.
    </Warning>
  </Step>
</Steps>

<Check>
  **Verificación**

  Ahora deberías tener una experiencia de inicio de sesión de Auth0 completamente funcional en tu aplicación Flutter para Windows. La aplicación abre el navegador del sistema para Auth0 Universal Login, recibe la devolución de llamada mediante un esquema de URL personalizado y muestra el perfil del usuario autenticado.
</Check>

<div id="troubleshoot-advanced-use">
  ## Solución de problemas y uso avanzado
</div>

<Accordion title="Problemas comunes y soluciones">
  ### El navegador se abre, pero la app no recibe la devolución de llamada

  **Síntoma**: El inicio de sesión con Auth0 se completa en el navegador, pero la app nunca recibe las credenciales.

  **Solución:**

  1. Abre el Editor del Registro → `HKEY_CURRENT_USER\Software\Classes\auth0flutter\shell\open\command` y confirma la ruta de tu `.exe`.
  2. Prueba con `start auth0flutter://test` en el Símbolo del sistema; tu app debería iniciarse.
  3. Asegúrate de que `windows/runner/main.cpp` incluya el servidor de canalización con nombre y el código del mutex.
  4. Comprueba que no haya instancias antiguas ejecutándose en el Administrador de tareas.
  5. Vuelve a compilar por completo: `flutter clean && flutter run -d windows`.

  ### La authentication agota el tiempo de espera después de 5 minutos

  **Síntoma**: Parece que el login se queda bloqueado y al final falla.

  **Solución:** La app nunca recibió la URI de devolución de llamada. Verifica lo siguiente:

  1. La entrada del registro apunta a la ruta correcta del ejecutable.
  2. El nombre del mutex `auth0flutter_single_instance_mutex` sea consistente en `main.cpp`.
  3. Ningún firewall ni antivirus esté bloqueando la canalización con nombre.
  4. Cierra todas las instancias antiguas y vuelve a compilar.

  ### Se abre una segunda instancia de la app en lugar de reenviar la URI

  **Síntoma**: Se abre una ventana nueva en vez de que la app existente reciba la devolución de llamada.

  **Solución:**

  1. Cierra todas las instancias en ejecución desde el Administrador de tareas.
  2. Asegúrate de que el nombre del mutex sea consistente en `main.cpp`.
  3. Vuelve a compilar con el siguiente comando: `flutter clean && flutter run -d windows`.

  ### No se encuentra WindowsWebAuthentication

  **Síntoma**: Error de compilación relacionado con `windowsWebAuthentication`.

  **Solución:** Asegúrate de que tu `pubspec.yaml` especifique la versión beta:

  ```yaml
  dependencies:
    auth0_flutter: 2.1.0-beta.1
  ```

  Ejecuta `flutter pub get` para actualizar.

  ### Error de discrepancia en la URL de devolución de llamada

  **Síntoma**: Error &quot;redirect&#95;uri&#95;mismatch&quot; de Auth0.

  **Solución:**

  1. Verifica que **Allowed Callback URLs** en [Auth0 Dashboard](https://manage.auth0.com/) → Application Settings sea exactamente `auth0flutter://callback`.
  2. Asegúrate de que el parámetro `appCustomURL` en tu código coincida con `'auth0flutter://callback'`.
  3. Comprueba que no haya barras diagonales finales ni espacios en blanco.
</Accordion>

<Accordion title="Uso de un servidor intermediario">
  Cuando Auth0 redirige directamente a un esquema personalizado, el navegador puede mostrar un prompt o dejar una pestaña en blanco. Para una experiencia más fluida, usa un servidor HTTPS intermediario:

  1. Configura un endpoint en el servidor (por ejemplo, `https://your-app.example.com/callback`) que redirija a `auth0flutter://callback?code=...&state=...`.
  2. En [Auth0 Dashboard](https://manage.auth0.com/) → Application Settings, establece **Allowed Callback URLs** como `https://your-app.example.com/callback`.
  3. Pasa ambas URL al método de login:

  ```dart
  final result = await auth0.windowsWebAuthentication().login(
    appCustomURL: 'auth0flutter://callback',
    redirectUrl: 'https://your-app.example.com/callback',
  );
  ```

  La página del servidor puede mostrar &quot;Redirigiendo...&quot; y cerrarse sola para ofrecer una experiencia más limpia.
</Accordion>

<Accordion title="Scopes personalizados y audience">
  Solicita scopes adicionales o un audience de API:

  ```dart
  final result = await auth0.windowsWebAuthentication().login(
    appCustomURL: 'auth0flutter://callback',
    scopes: {'openid', 'profile', 'email', 'read:messages'},
    audience: 'https://your-api.example.com',
  );
  ```

  Configura la API en [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; APIs** antes de usar el parámetro `audience`.
</Accordion>
