<Note>
  **Requisitos previos:**

  * Flutter SDK 3.24.0+ y Dart 3.5.0+.
  * Windows 10 o posterior.
  * Visual Studio 2022 con la carga de trabajo **Desarrollo de escritorio con C++**.
  * Cuenta de Auth0: [regístrate gratis](https://auth0.com/signup).

  Esta funcionalidad está en **beta** (`auth0_flutter` 2.1.0-beta.1). La API puede cambiar antes de la disponibilidad general.
</Note>

Esta guía te muestra cómo agregar inicio de sesión, cierre de sesión y la visualización del perfil de usuario a una aplicación de escritorio de Flutter para Windows con el SDK `auth0_flutter` y OAuth 2.0 [flujo de código de autorización + PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

<div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Crear un proyecto de Flutter para Windows" stepNumber={1}>
    Crea un nuevo proyecto de Flutter con compatibilidad para Windows.

    ```shellscript
    flutter create --platforms=windows my_app
    cd my_app
    ```

    Comprueba que Windows esté disponible:

    ```shellscript
    flutter devices
    ```

    Deberías ver en la lista un dispositivo de escritorio con Windows.

    <Tip>
      Ejecuta `flutter doctor` para comprobar que tu entorno esté configurado correctamente y que se detecte Visual Studio 2022 con el desarrollo de escritorio de C++.
    </Tip>
  </Step>

  <Step title="Instala el SDK de Auth0 para Flutter" stepNumber={2}>
    Agrega la versión beta del SDK que incluye compatibilidad con Windows:

    ```shellscript
    flutter pub add auth0_flutter:2.1.0-beta.1
    flutter pub add flutter_dotenv
    ```

    Tu archivo `pubspec.yaml` debe incluir:

    <Snippet file="quickstart/native/flutter-windows/pubspec.yaml.mdx" />

    <Tip>
      El Auth0 Flutter SDK requiere **Flutter 3.24.0+** y **Dart 3.5.0+**. Además, la plataforma Windows requiere **Visual Studio 2022** con la carga de trabajo de desarrollo de escritorio en C++.
    </Tip>
  </Step>

  <Step title="Configura Auth0" stepNumber={3}>
    Crea o configura una aplicación de Auth0 con las URL de devolución de llamada necesarias para la autenticación en aplicaciones de escritorio de Windows.

    <Tabs>
      <Tab title="Configuración rápida">
        Crea una aplicación **Native** en tu Auth0 Dashboard con la siguiente configuración:

        | Campo                 | Valor                     |
        | --------------------- | ------------------------- |
        | Allowed Callback URLs | `auth0flutter://callback` |
        | Allowed Logout URLs   | `auth0flutter://callback` |

        Tus credenciales:

        * **Domain:** `{yourDomain}`
        * **Client ID:** `{yourClientId}`
      </Tab>

      <Tab title="CLI">
        ```shellscript
        auth0 apps create \
          --name "My Flutter Windows App" \
          --type native \
          --callbacks "auth0flutter://callback" \
          --logout-urls "auth0flutter://callback"
        ```
      </Tab>

      <Tab title="Dashboard">
        1. Ve a [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; Applications**.
        2. Selecciona **Create Application**.
        3. Asígnale el nombre &quot;My Flutter Windows App&quot; y elige **Native** como tipo de aplicación.
        4. Ve a la pestaña **Settings** de tu nueva aplicación.
        5. Configura **Allowed Callback URLs** como `auth0flutter://callback`.
        6. Configura **Allowed Logout URLs** como `auth0flutter://callback`.
        7. Desplázate hacia abajo y haz clic en **Save Changes**.
        8. Copia los valores de **Domain** y **Client ID** de la parte superior de **Application Settings**.
      </Tab>
    </Tabs>

    <Info>
      La URL `auth0flutter://callback` es un esquema personalizado que redirige la devolución de llamada del navegador de vuelta a tu aplicación de escritorio una vez completada la autenticación.
    </Info>
  </Step>

  <Step title="Configurar variables de entorno" stepNumber={4}>
    Crea un archivo `.env` en la raíz de tu proyecto:

    <AuthCodeBlock children={envSnippet} language="shellscript" filename=".env" />

    Añade el archivo `.env` a los assets de Flutter en `pubspec.yaml`:

    <Snippet file="quickstart/native/flutter-windows/pubspec-assets.yaml.mdx" />

    <Warning>
      Nunca subas tu archivo `.env` al control de versiones. Añádelo a `.gitignore`.
    </Warning>
  </Step>

  <Step title="Configurar el runner de Windows" stepNumber={5}>
    El flujo de autenticación de Windows requiere configurar la devolución de llamada en el runner de tu aplicación. El plugin de Flutter no recibe automáticamente las activaciones del esquema de protocolo del sistema operativo, por lo que debes agregar la aplicación de instancia única y el reenvío de URI.

    Reemplaza el contenido de `windows/runner/main.cpp`:

    <Snippet file="quickstart/native/flutter-windows/main.cpp.mdx" />

    Este código:

    * Fuerza una única instancia mediante un mutex de Windows.
    * Captura los URI `auth0flutter://callback` que se pasan como argumentos de la línea de comandos.
    * Reenvía los URI de los inicios secundarios a la instancia en ejecución mediante una canalización con nombre.
    * Establece la variable de entorno `PLUGIN_STARTUP_URL` para que la lea el plugin de Auth0.
  </Step>

  <Step title="Registra el esquema de URL personalizado" stepNumber={6}>
    Registra `auth0flutter` como un esquema de URL personalizado para que Windows dirija las URI de devolución de llamada a tu aplicación.

    Crea un archivo `windows/url_scheme.reg`:

    <Snippet file="quickstart/native/flutter-windows/url_scheme.reg.mdx" />

    Reemplaza `C:\path\to\your\app.exe` con la ruta real de tu ejecutable compilado. Durante el desarrollo, normalmente es:

    ```text
    build\windows\x64\runner\Debug\my_app.exe
    ```

    Haz doble clic en el archivo `.reg` para importarlo en el Registro de Windows.

    <Info>
      Para usarlo en producción, registra el esquema de URL personalizado de forma programática en el instalador de tu aplicación (MSIX, Inno Setup, etc.), en lugar de depender de un archivo `.reg`.
    </Info>

    **Verifica que el esquema funcione:**

    Abre el Símbolo del sistema y ejecuta:

    ```shellscript
    start auth0flutter://test
    ```

    Tu aplicación debería abrirse (o pasar al primer plano si ya se está ejecutando).
  </Step>

  <Step title="Implement inicio de sesión y cierre de sesión" stepNumber={7}>
    Crea `lib/auth_service.dart` para gestionar la Autenticación de Windows:

    <Snippet file="quickstart/native/flutter-windows/auth_service.dart.mdx" />

    <Info>
      Actualmente, el SDK de Auth0 Flutter para Windows no admite la gestión de credenciales. Debes almacenar manualmente las credenciales si necesitas que las sesiones se mantengan entre reinicios de la aplicación.
    </Info>
  </Step>

  <Step title="Mostrar información del perfil del usuario" stepNumber={8}>
    Crea la interfaz principal de la aplicación en `lib/main.dart`:

    <Snippet file="quickstart/native/flutter-windows/main.dart.mdx" />
  </Step>

  <Step title="Ejecuta tu aplicación" stepNumber={9}>
    Compila y ejecuta la aplicación:

    ```shellscript
    flutter run -d windows
    ```

    **Flujo esperado:**

    1. La app se inicia con un botón de **Iniciar sesión**.
    2. Selecciona **Iniciar sesión**. El navegador del sistema abre la página de Universal Login de Auth0.
    3. Completa la autenticación en el navegador.
    4. El navegador redirige a `auth0flutter://callback` → tu app vuelve al primer plano.
    5. Se muestran el nombre, el correo electrónico y la foto de perfil del usuario.

    <Warning>
      Asegúrate de que el esquema de URL personalizado esté registrado (Paso 6) antes de hacer la prueba. Sin él, la devolución de llamada del navegador no podrá llegar a tu aplicación.
    </Warning>
  </Step>
</Steps>

<Check>
  **Verificación**

  Ahora deberías tener una experiencia de inicio de sesión de Auth0 totalmente funcional en tu aplicación Flutter para Windows. La aplicación abre el navegador del sistema con Auth0 Universal Login, recibe la devolución de llamada mediante el esquema de URL personalizado y muestra el perfil del usuario autenticado.
</Check>

<div id="troubleshoot-advanced-use">
  ## Solución de problemas y uso avanzado
</div>

<Accordion title="Problemas comunes y soluciones">
  ### El navegador se abre, pero la app no recibe la devolución de llamada

  **Síntoma**: El login de Auth0 se completa correctamente en el navegador, pero la app nunca recibe las credenciales.

  **Solución:**

  1. Abre el Editor del Registro → `HKEY_CURRENT_USER\Software\Classes\auth0flutter\shell\open\command` y confirma la ruta de tu `.exe`.
  2. Prueba con `start auth0flutter://test` en el Símbolo del sistema; tu app debería iniciarse.
  3. Asegúrate de que `windows/runner/main.cpp` incluya el servidor de canalización con nombre y el código del mutex.
  4. Comprueba que no haya instancias antiguas ejecutándose en el Administrador de tareas.
  5. Vuelve a compilar por completo: `flutter clean && flutter run -d windows`.

  ### La autenticación agota el tiempo de espera tras 5 minutos

  **Síntoma**: El login parece quedarse colgado y, al final, falla.

  **Solución:** La app nunca recibió la URI de devolución de llamada. Verifica lo siguiente:

  1. La entrada del registro apunta a la ruta correcta del ejecutable.
  2. Que el nombre del mutex `auth0flutter_single_instance_mutex` sea consistente en `main.cpp`.
  3. Que ningún firewall ni antivirus esté bloqueando la canalización con nombre.
  4. Cierra todas las instancias antiguas y vuelve a compilar.

  ### Se inicia una segunda instancia de la app en lugar de reenviar la URI

  **Síntoma**: Se abre una ventana nueva en lugar de que la app existente reciba la devolución de llamada.

  **Solución:**

  1. Cierra todas las instancias en ejecución en el Administrador de tareas.
  2. Asegúrate de que el nombre del mutex sea consistente en `main.cpp`.
  3. Vuelve a compilar con el siguiente comando: `flutter clean && flutter run -d windows`.

  ### No se encuentra WindowsWebAuthentication

  **Síntoma**: Error de compilación que hace referencia a `windowsWebAuthentication`.

  **Solución:** Asegúrate de que tu `pubspec.yaml` especifique la versión beta:

  ```yaml
  dependencies:
    auth0_flutter: 2.1.0-beta.1
  ```

  Ejecuta `flutter pub get` para actualizar.

  ### Error de discrepancia en la URL de devolución de llamada

  **Síntoma**: Error &quot;redirect&#95;uri&#95;mismatch&quot; de Auth0.

  **Solución:**

  1. Verifica que **Allowed Callback URLs** en [Auth0 Dashboard](https://manage.auth0.com/) → Application Settings sea exactamente `auth0flutter://callback`.
  2. Asegúrate de que el parámetro `appCustomURL` en tu código coincida: `'auth0flutter://callback'`.
  3. Comprueba que no haya barras diagonales finales ni espacios en blanco.
</Accordion>

<Accordion title="Uso de un servidor intermediario">
  Cuando Auth0 redirige directamente a un esquema personalizado, el navegador puede mostrar un aviso o dejar una pestaña en blanco. Para una experiencia más fluida, usa un servidor HTTPS intermediario:

  1. Configura un endpoint del servidor (por ejemplo, `https://your-app.example.com/callback`) que redirija a `auth0flutter://callback?code=...&state=...`.
  2. En [Auth0 Dashboard](https://manage.auth0.com/) → Application Settings, establece **Allowed Callback URLs** como `https://your-app.example.com/callback`.
  3. Pasa ambas URL al método de login:

  ```dart
  final result = await auth0.windowsWebAuthentication().login(
    appCustomURL: 'auth0flutter://callback',
    redirectUrl: 'https://your-app.example.com/callback',
  );
  ```

  La página del servidor puede mostrar &quot;Redirigiendo...&quot; y cerrarse sola para ofrecer una experiencia más limpia.
</Accordion>

<Accordion title="Scopes personalizados y audience">
  Solicita scopes adicionales o un audience de API:

  ```dart
  final result = await auth0.windowsWebAuthentication().login(
    appCustomURL: 'auth0flutter://callback',
    scopes: {'openid', 'profile', 'email', 'read:messages'},
    audience: 'https://your-api.example.com',
  );
  ```

  Configura la API en [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; APIs** antes de usar el parámetro `audience`.
</Accordion>
