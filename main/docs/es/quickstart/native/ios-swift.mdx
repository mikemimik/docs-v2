  ```bash
  npx skills add auth0/agent-skills --skill auth0-quickstart --skill auth0-swift
  ```

  **Luego, pídele a tu asistente de IA:**

  ```text
  Agrega la autenticación de Auth0 a mi app de iOS
  ```

  Tu asistente de IA creará automáticamente tu aplicación de Auth0, obtendrá las credenciales, añadirá la dependencia del SDK de Auth0.swift, configurará Auth0.plist, establecerá las URL de devolución de llamada e implementará flujos de inicio y cierre de sesión. <div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Crea un nuevo proyecto" stepNumber={1}>
    Crea un nuevo proyecto de iOS o macOS para esta guía de inicio rápido.

    **En Xcode:**

    1. **File** → **New** → **Project** (o **⌘+Shift+N**)
    2. Selecciona una de estas opciones:
       * Pestaña **iOS** → plantilla **App**
       * Pestaña **macOS** → plantilla **App**
    3. Configura tu proyecto:
       * **Product Name**: `Auth0-Sample`
       * **Interface**: SwiftUI
       * **Language**: Swift
       * **Use Core Data**: Desmarcado
       * **Include Tests**: Marcado (recomendado)
    4. Elige una ubicación y haz clic en **Create**

    <Tip>
      Esto crea una app estándar con compatibilidad con SwiftUI y Swift Package Manager, perfecta para integrar Auth0.
    </Tip>
  </Step>

  <Step title="Añade el SDK de Auth0" stepNumber={2}>
    Añade el SDK de Auth0 a tu proyecto con el administrador de paquetes que prefieras.

    <Tabs>
      <Tab title="Swift Package Manager">
        **En Xcode:**

        1. **File** → **Add Package Dependencies...** (o **⌘+Shift+K**)
        2. Introduce la URL del SDK de Auth0:
           ```
           https://github.com/auth0/Auth0.swift
           ```
        3. **Add Package** → Selecciona el target de tu app → **Add Package**
      </Tab>

      <Tab title="CocoaPods">
        1. Crea un `Podfile` en el directorio de tu proyecto:
           ```ruby Podfile
           platform :ios, '14.0' # O platform :osx, '11.0' para macOS
           use_frameworks!

           target 'YourApp' do
             pod 'Auth0', '~> 2.0'
           end
           ```
        2. Instala las dependencias:
           ```bash
           pod install
           ```
        3. Abre el archivo `.xcworkspace` generado (no `.xcodeproj`)
      </Tab>

      <Tab title="Carthage">
        1. Crea un `Cartfile` en el directorio de tu proyecto:
           ```text Cartfile
           github "auth0/Auth0.swift" ~> 2.0
           ```
        2. Ejecuta Carthage:
           ```bash
           carthage update --platform iOS --use-xcframeworks
           ```
           Para macOS, usa `--platform macOS`
        3. Arrastra el `Auth0.xcframework` generado desde `Carthage/Build` a tu proyecto de Xcode
        4. En la configuración **General** de tu target, añade `Auth0.xcframework` a **Frameworks, Libraries, and Embedded Content**
      </Tab>
    </Tabs>
  </Step>

  <Step title="Configurar Auth0" stepNumber={3}>
    Crea una nueva aplicación de Auth0 y configura las URL de devolución de llamada.

    1. Ve a [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. **Applications** &gt; **Create Application** &gt; Ponle un nombre, selecciona **Native** &gt; **Create**
    3. En la pestaña **Settings**, toma nota de tu **ID de cliente** y **Domain**
    4. Agrega las siguientes URL a **Allowed Callback URLs**:

    <Tabs>
      <Tab title="iOS">
        ```
        https://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>

      <Tab title="macOS">
        ```
        https://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>
    </Tabs>

    5. Agrega las siguientes URL a **Allowed Logout URLs**:

    <Tabs>
      <Tab title="iOS">
        ```
        https://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>

      <Tab title="macOS">
        ```
        https://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>
    </Tabs>

    6. Haz clic en **Save Changes**
  </Step>

  <Step title="Configurar las credenciales de la aplicación" stepNumber={4}>
    Crea `Auth0.plist` en el directorio de tu proyecto:

    ```xml Auth0.plist
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
    <plist version="1.0">
    <dict>
        <key>ClientId</key>
        <string>YOUR_AUTH0_CLIENT_ID</string>
        <key>Domain</key>
        <string>{yourDomain}</string>
    </dict>
    </plist>
    ```

    Arrastra `Auth0.plist` a Xcode y asegúrate de que la opción &quot;Add to target&quot; esté marcada.
  </Step>

  <Step title="Crear el servicio de autenticación" stepNumber={5}>
    Crea `AuthenticationService.swift` para gestionar el inicio de sesión, el cierre de sesión y el almacenamiento de tokens.

    <Info>
      **Usa `CredentialsManager` para el almacenamiento de tokens.** La clase `CredentialsManager` almacena las credenciales de forma segura en el Keychain y renueva automáticamente los tokens de acceso vencidos. Úsala siempre; no almacenes tokens en memoria, `UserDefaults` ni `localStorage`.
    </Info>

    1. Haz clic derecho en tu proyecto → **New File...** → **Swift File**
    2. Asígnale el nombre `AuthenticationService`
    3. Reemplaza el contenido por:

    ```swift AuthenticationService.swift expandable lines
    import Foundation
    import Auth0
    import Combine

    @MainActor
    class AuthenticationService: ObservableObject {
        @Published var isAuthenticated = false
        @Published var user: User?
        @Published var isLoading = false
        @Published var errorMessage: String?
        
        private let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
        
        init() {
            Task {
                await checkAuthenticationStatus()
            }
        }
        
        private func checkAuthenticationStatus() async {
            isLoading = true
            defer { isLoading = false }
            
            guard let credentials = try? await credentialsManager.credentials() else {
                isAuthenticated = false
                return
            }
            
            isAuthenticated = true
            // Obtener información del usuario desde el token de ID
            user = credentials.user
        }
        
        func login() async {
            isLoading = true
            errorMessage = nil
            defer { isLoading = false }
            
            do {
                let credentials = try await Auth0
                    .webAuth()
                    .scope("openid profile email offline_access")
                    .start()
                
                _ = credentialsManager.store(credentials: credentials)
                isAuthenticated = true
                // Obtener información del usuario desde el token de ID
                user = credentials.user
            } catch {
                errorMessage = "Login failed: \(error.localizedDescription)"
            }
        }
        
        func logout() async {
            isLoading = true
            defer { isLoading = false }
            
            do {
                try await Auth0
                  .webAuth()
                  .clearSession()
                _ = credentialsManager.clear()
                isAuthenticated = false
                user = nil
            } catch {
                errorMessage = "Logout failed: \(error.localizedDescription)"
            }
        }
    }
    ```
  </Step>

  <Step title="Configurar el flujo de autenticación (opcional)" stepNumber={6}>
    Para mejorar la experiencia del usuario, puede minimizar las alertas del sistema de las siguientes maneras:

    1. Use Universal Links: esto elimina el prompt &#39;Open in &quot;AppName&quot;?&#39; que aparece durante la redirección. Nota: la alerta de permiso de ASWebAuthenticationSession seguirá apareciendo.
    2. Use sesiones efímeras: esto elimina todas las alertas de permiso. Nota: esto desactiva el inicio de sesión único (SSO) y las cookies compartidas.

    <Tip>
      **Omita este paso** para usar el comportamiento predeterminado con una alerta de permiso. Puede configurarlo más adelante.
    </Tip>

    <Tabs>
      <Tab title="Universal Links">
        1. Auth0 Dashboard → **Applications** → tu aplicación → **Settings** → **Advanced Settings** → **Device Settings**
        2. Agregue **ID del equipo de Apple** e **identificador del paquete** → **Save**
        3. Xcode: Target → **Signing &amp; Capabilities** → **+ Capability** → **Associated Domains**
        4. Agregue: `webcredentials:{yourDomain}`

        <Warning>Requiere: una cuenta de pago de Apple Developer, iOS 17.4+/macOS 14.4+</Warning>

        <Tip>
          Ideal para aplicaciones de Production.
        </Tip>
      </Tab>

      <Tab title="Sesión efímera">
        Agregue `.useEphemeralSession()` a la llamada de inicio de sesión en `AuthenticationService.swift`:

        ```swift
        // En la función login()
        let credentials = try await Auth0
            .webAuth()
            .scope("openid profile email offline_access")
            .useEphemeralSession()
            .start()
        ```

        <Info>
          Al usar sesiones efímeras, no necesita llamar a `clearSession()` al hacer cierre de sesión. Solo elimine las credenciales de su aplicación; no hay ninguna cookie compartida que quitar.
        </Info>

        <Tip>
          Configuración rápida, sin alertas, pero los usuarios deben iniciar sesión cada vez (sin SSO).
        </Tip>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Ejecuta tu app" stepNumber={8}>
    Presiona **⌘+R** en Xcode.

    1. Toca &quot;Iniciar sesión&quot; → Alerta de permisos (si usas la opción predeterminada) → Toca &quot;Continuar&quot;
    2. Completa el inicio de sesión en el navegador
    3. ¡Verás tu perfil!
  </Step>
</Steps>

<Check>
  **Comprobación**

  ¡Ahora ya tienes un inicio de sesión con Auth0 totalmente funcional en tu aplicación de iOS o macOS!
</Check>

<div id="troubleshooting-advanced">
  ## Resolución de problemas y opciones avanzadas
</div>

<Accordion title="Problemas comunes y soluciones">
  ### Errores de compilación: no se encuentra el módulo &#39;Auth0&#39;

  **Soluciones**:

  1. **Swift Package Manager**: Revisa **Package Dependencies** → verifica que `Auth0.swift` aparezca en la lista
  2. **CocoaPods**: Asegúrate de abrir el archivo `.xcworkspace`, no `.xcodeproj`
  3. **Carthage**: Verifica que `Auth0.xcframework` esté agregado a **Frameworks, Libraries, and Embedded Content**
  4. Limpia y vuelve a compilar: **⌘+Shift+K** y luego **⌘+R**
  5. Reinicia Xcode si hace falta

  ### La app se cierra: &#39;Auth0.plist not found&#39;

  **Solución**:

  1. Verifica que `Auth0.plist` esté en el navegador de proyectos de Xcode
  2. Selecciona el archivo → Inspector → asegúrate de que el target de tu app esté marcado
  3. Confirma que tenga las claves `ClientId` y `Domain` con tus valores

  ### El navegador se abre, pero nunca vuelve a la app

  **Solución**:

  1. Verifica que las URL de devolución de llamada en Auth0 Dashboard coincidan exactamente con tu identificador del paquete y la plataforma
  2. Para iOS: las URL deben contener `/ios/`; para macOS: `/macos/`
  3. Verifica que el identificador del paquete en Xcode coincida con la configuración de Auth0
  4. Asegúrate de que no haya errores tipográficos en las URL (lo más común: faltan dos puntos o el formato del dominio es incorrecto)
  5. **Usuarios de dominios personalizados**: verifica que estés usando tu dominio personalizado, no el dominio de Auth0

  ### La alerta de permisos aparece siempre

  Este es el comportamiento de seguridad estándar de iOS/macOS cuando se usan esquemas de URL personalizados. Consulta el **Paso 6** para eliminar esta alerta con Universal Links o sesiones efímeras.
</Accordion>

<Accordion title="Configuración del dominio personalizado">
  Si estás usando un [dominio personalizado](/docs/es/customize/custom-domains), usa ese valor en lugar de tu dominio de Auth0 en todas partes.

  **Ejemplo:** Usa `login.example.com` en lugar de `tenant.auth0.com`

  Esto es **necesario** para que ciertas funcionalidades funcionen correctamente:

  * Actualiza `Auth0.plist` con tu dominio personalizado
  * Usa el dominio personalizado en las URL de devolución de llamada y cierre de sesión
  * Para Universal Links, usa: `webcredentials:login.example.com`
</Accordion>

<Accordion title="Despliegue en producción">
  ### Preparación para la App Store

  * Configura Universal Links para eliminar la alerta de permisos
  * Haz pruebas en varias versiones de la plataforma y tamaños de dispositivo
  * Implementa un manejo adecuado de errores para fallos de red
  * Agrega descripciones de uso de privacidad si usas Keychain con biometría
  * Sigue las pautas de revisión de la App Store para los flujos de autenticación

  ### Prácticas recomendadas de seguridad

  * Nunca registres datos sensibles de autenticación en producción
  * Implementa el cumplimiento de App Transport Security (ATS)
  * Usa HTTPS para todas las solicitudes de red
  * **NO** ancles certificados de Auth0 API - [Auth0 no recomienda esta práctica](/docs/es/troubleshoot/product-lifecycle/past-migrations#avoid-pinning-or-fingerprinting-tls-certificates-for-auth0-endpoints)

  ### Optimización del rendimiento

  * Todas las operaciones asíncronas usan correctamente `@MainActor` para las actualizaciones de la UI
  * Las propiedades `@Published` usan un manejo adecuado de memoria
  * Las credenciales se almacenan en caché de forma segura en Keychain para permitir el acceso sin conexión
  * El perfil del usuario se obtiene del token de identidad (sin ninguna solicitud de red adicional)
</Accordion>

<Accordion title="Integración avanzada">
  ### Seguridad reforzada de Keychain con biometría

  Exige Face ID o Touch ID para acceder a las credenciales almacenadas:

  ```swift
  private let credentialsManager: CredentialsManager = {
      let manager = CredentialsManager(authentication: Auth0.authentication())
      manager.enableBiometrics(
          withTitle: "Unlock with Face ID", 
          cancelTitle: "Cancel", 
          fallbackTitle: "Use Passcode"
      )
      return manager
  }()
  ```

  Cuando esta opción está habilitada, los usuarios deben autenticarse con biometría antes de que el SDK pueda recuperar las credenciales almacenadas.

  ### Renovación automática de tokens

  `CredentialsManager` renueva automáticamente los tokens de acceso vencidos:

  ```swift
  // Obtener credenciales: se renuevan automáticamente si vencieron
  func getAccessToken() async throws -> String {
      let credentials = try await credentialsManager.credentials()
      return credentials.accessToken
  }
  ```

  Usa este patrón al hacer llamadas a la API que requieran un token de acceso.

  ### Credenciales compartidas entre extensiones de la app

  Para widgets, extensiones de la app o tareas en segundo plano que necesiten tokens de acceso:

  ```swift
  // Crear un administrador de credenciales compartidas con un grupo de apps
  let credentialsManager = CredentialsManager(
      authentication: Auth0.authentication(),
      storeKey: "credentials",
      storage: .shared(withIdentifier: "group.com.example.myapp")
  )
  ```

  **Requisitos:**

  1. Activa la capacidad **App Groups** en Xcode para todos los targets
  2. Usa el mismo identificador de grupo de apps en todos los targets
  3. Configura el `CredentialsManager` compartido en cada target

  ### Comparación de opciones del flujo de autenticación

  | Funcionalidad                         | Universal Links                      | Sesión efímera | Predeterminado (alerta)   |
  | ------------------------------------- | ------------------------------------ | -------------- | ------------------------- |
  | Alerta de permisos                    | Reducida (sin prompt de redirección) | Ninguna        | Muestra todas las alertas |
  | Compatibilidad con SSO                | Sí                                   | No             | Sí                        |
  | Cuenta de desarrollador de Apple      | Obligatoria                          | No obligatoria | No obligatoria            |
  | Experiencia de usuario                | La mejor                             | Buena          | Aceptable                 |
  | Complejidad de configuración          | Media                                | Fácil          | Fácil                     |
  | Compatibilidad con navegación privada | Sí                                   | Sí             | No                        |

  **Recomendaciones:**

  * **Aplicaciones en producción con SSO**: Universal Links (mejor UX, compatibilidad con SSO, requiere una cuenta de desarrollador de Apple)
  * **Aplicaciones en producción sin SSO**: Sesiones efímeras (sin alertas, configuración más sencilla)
  * **Pruebas/Desarrollo**: Sesiones efímeras (configuración rápida, UX más limpia)
  * **Inicio rápido/Prototipado**: Predeterminado con alertas (sin configuración, puedes migrar más adelante)
</Accordion>
