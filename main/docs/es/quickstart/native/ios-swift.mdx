<Accordion title="Usa IA para integrar Auth0" icon="microchip-ai" iconType="solid" defaultOpen>
  Si usas un asistente de programación con IA como Claude Code, Cursor o GitHub Copilot, puedes añadir la autenticación de Auth0 automáticamente en cuestión de minutos con [agent skills](https://agentskills.io/home).

  **Instala:**

  ```bash
  npx skills add auth0/agent-skills --skill auth0-quickstart --skill auth0-swift
  ```

  **Luego, pídele a tu asistente de IA:**

  ```text
  Add Auth0 authentication to my iOS app
  ```

  Tu asistente de IA creará automáticamente tu aplicación de Auth0, obtendrá las credenciales, añadirá la dependencia del SDK Auth0.swift, configurará Auth0.plist, establecerá las URL de devolución de llamada e implementará los flujos de inicio y cierre de sesión. [Documentación completa de agent skills →](/docs/es/quickstart/agent-skills)
</Accordion>

<div id="get-started">
  ## Primeros pasos
</div>

<Steps>
  <Step title="Crea un nuevo proyecto" stepNumber={1}>
    Crea un nuevo proyecto de iOS o macOS para este Quickstart.

    **En Xcode:**

    1. **File** → **New** → **Project** (o **⌘+Shift+N**)
    2. Selecciona una de estas opciones:
       * pestaña **iOS** → plantilla **App**
       * pestaña **macOS** → plantilla **App**
    3. Configura tu proyecto:
       * **Product Name**: `Auth0-Sample`
       * **Interface**: SwiftUI
       * **Language**: Swift
       * **Use Core Data**: desmarcado
       * **Include Tests**: marcado (recomendado)
    4. Elige una ubicación y haz clic en **Create**

    <Tip>
      Esto crea una app estándar con soporte para SwiftUI y Swift Package Manager, perfecta para la integración con Auth0.
    </Tip>
  </Step>

  <Step title="Añade el SDK de Auth0" stepNumber={2}>
    Agrega el SDK de Auth0 a tu proyecto con el gestor de paquetes que prefieras.

    <Tabs>
      <Tab title="Swift Package Manager">
        **En Xcode:**

        1. **File** → **Add Package Dependencies...** (o **⌘+Shift+K**)
        2. Ingresa la URL del SDK de Auth0:
           ```
           https://github.com/auth0/Auth0.swift
           ```
        3. **Add Package** → Selecciona el target de tu app → **Add Package**
      </Tab>

      <Tab title="CocoaPods">
        1. Crea un `Podfile` en el directorio de tu proyecto:
           ```ruby Podfile
           platform :ios, '14.0' # O platform :osx, '11.0' para macOS
           use_frameworks!

           target 'YourApp' do
             pod 'Auth0', '~> 2.0'
           end
           ```
        2. Instala las dependencias:
           ```bash
           pod install
           ```
        3. Abre el archivo `.xcworkspace` generado (no el `.xcodeproj`)
      </Tab>

      <Tab title="Carthage">
        1. Crea un `Cartfile` en el directorio de tu proyecto:
           ```text Cartfile
           github "auth0/Auth0.swift" ~> 2.0
           ```
        2. Ejecuta Carthage:
           ```bash
           carthage update --platform iOS --use-xcframeworks
           ```
           Para macOS, usa `--platform macOS`
        3. Arrastra el `Auth0.xcframework` generado desde `Carthage/Build` a tu proyecto de Xcode
        4. En la configuración **General** de tu target, agrega `Auth0.xcframework` a **Frameworks, Libraries, and Embedded Content**
      </Tab>
    </Tabs>
  </Step>

  <Step title="Configurar Auth0" stepNumber={3}>
    Crea una nueva aplicación de Auth0 y configura las URL de devolución de llamada.

    1. Ve a [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. **Applications** &gt; **Create Application** &gt; Asígnale un nombre, selecciona **Native** &gt; **Create**
    3. En la pestaña **Settings**, anota tu **ID de cliente** y **Domain**
    4. Agrega las siguientes URL a **Allowed Callback URLs**:

    <Tabs>
      <Tab title="iOS">
        ```
        https://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>

      <Tab title="macOS">
        ```
        https://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>
    </Tabs>

    5. Agrega las siguientes URL a **Allowed Logout URLs**:

    <Tabs>
      <Tab title="iOS">
        ```
        https://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>

      <Tab title="macOS">
        ```
        https://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback,
        YOUR_BUNDLE_IDENTIFIER://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback
        ```
      </Tab>
    </Tabs>

    6. Haz clic en **Save Changes**
  </Step>

  <Step title="Configurar las credenciales de la aplicación" stepNumber={4}>
    Crea `Auth0.plist` en el directorio de tu proyecto:

    ```xml Auth0.plist
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
    <plist version="1.0">
    <dict>
        <key>ClientId</key>
        <string>YOUR_AUTH0_CLIENT_ID</string>
        <key>Domain</key>
        <string>{yourDomain}</string>
    </dict>
    </plist>
    ```

    Arrastra `Auth0.plist` a Xcode y asegúrate de que la opción &quot;Add to target&quot; esté marcada.
  </Step>

  <Step title="Crear el servicio de autenticación" stepNumber={5}>
    Crea `AuthenticationService.swift` para gestionar el inicio y cierre de sesión, y el almacenamiento de tokens.

    <Info>
      **Usa `CredentialsManager` para almacenar los tokens.** La clase `CredentialsManager` guarda las credenciales de forma segura en el Keychain y renueva automáticamente los tokens de acceso vencidos. Úsala siempre; no almacenes tokens en memoria, `UserDefaults` ni `localStorage`.
    </Info>

    1. Haz clic derecho en tu proyecto → **New File...** → **Swift File**
    2. Asígnale el nombre `AuthenticationService`
    3. Sustituye el contenido por:

    ```swift AuthenticationService.swift expandable lines
    import Foundation
    import Auth0
    import Combine

    @MainActor
    class AuthenticationService: ObservableObject {
        @Published var isAuthenticated = false
        @Published var user: User?
        @Published var isLoading = false
        @Published var errorMessage: String?
        
        private let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
        
        init() {
            Task {
                await checkAuthenticationStatus()
            }
        }
        
        private func checkAuthenticationStatus() async {
            isLoading = true
            defer { isLoading = false }
            
            guard let credentials = try? await credentialsManager.credentials() else {
                isAuthenticated = false
                return
            }
            
            isAuthenticated = true
            // Obtener información del usuario desde el token de ID
            user = credentials.user
        }
        
        func login() async {
            isLoading = true
            errorMessage = nil
            defer { isLoading = false }
            
            do {
                let credentials = try await Auth0
                    .webAuth()
                    .scope("openid profile email offline_access")
                    .start()
                
                _ = credentialsManager.store(credentials: credentials)
                isAuthenticated = true
                // Get user info from the ID token
                user = credentials.user
            } catch {
                errorMessage = "Login failed: \(error.localizedDescription)"
            }
        }
        
        func logout() async {
            isLoading = true
            defer { isLoading = false }
            
            do {
                try await Auth0
                  .webAuth()
                  .clearSession()
                _ = credentialsManager.clear()
                isAuthenticated = false
                user = nil
            } catch {
                errorMessage = "Logout failed: \(error.localizedDescription)"
            }
        }
    }
    ```
  </Step>

  <Step title="Configurar el flujo de autenticación (opcional)" stepNumber={6}>
    Para mejorar la experiencia del usuario, puedes minimizar las alertas del sistema de las siguientes maneras:

    1. Usa enlaces universales: así se elimina el aviso &#39;¿Abrir en &quot;AppName&quot;?&#39; que aparece durante la redirección. Nota: la alerta de permiso de ASWebAuthenticationSession seguirá apareciendo.
    2. Usa sesiones efímeras: así se eliminan todas las alertas de permiso. Nota: esto desactiva el inicio de sesión único (SSO) y las cookies compartidas.

    <Tip>
      **Omite este paso** para usar el comportamiento predeterminado con una alerta de permiso. Puedes configurarlo más adelante.
    </Tip>

    <Tabs>
      <Tab title="Enlaces universales">
        1. Auth0 Dashboard → **Applications** → Tu aplicación → **Settings** → **Advanced Settings** → **Device Settings**
        2. Agrega **Apple Team ID** e **identificador del paquete** → **Save**
        3. Xcode: Target → **Signing &amp; Capabilities** → **+ Capability** → **Associated Domains**
        4. Agrega: `webcredentials:{yourDomain}`

        <Warning>Requiere: una cuenta de pago de Apple Developer, iOS 17.4+/macOS 14.4+</Warning>

        <Tip>
          Ideal para aplicaciones en producción.
        </Tip>
      </Tab>

      <Tab title="Sesión efímera">
        Agrega `.useEphemeralSession()` a la llamada de login en `AuthenticationService.swift`:

        ```swift
        // En la función login()
        let credentials = try await Auth0
            .webAuth()
            .scope("openid profile email offline_access")
            .useEphemeralSession()
            .start()
        ```

        <Info>
          Al usar sesiones efímeras, no necesitas llamar a `clearSession()` al cerrar sesión. Solo borra las credenciales de tu aplicación; no hay ninguna cookie compartida que eliminar.
        </Info>

        <Tip>
          Configuración rápida, sin alertas, pero los usuarios tendrán que iniciar sesión cada vez (sin SSO).
        </Tip>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Ejecuta tu aplicación" stepNumber={8}>
    Pulsa **⌘+R** en Xcode.

    1. Toca &quot;Iniciar sesión&quot; → Alerta de permiso (si usas la opción predeterminada) → Toca &quot;Continuar&quot;
    2. Completa el inicio de sesión en el navegador
    3. ¡Verás tu perfil!
  </Step>
</Steps>

<Check>
  **Verificación**

  ¡Ahora ya tienes un inicio de sesión con Auth0 totalmente funcional en tu aplicación para iOS o macOS!
</Check>

<div id="troubleshooting-advanced">
  ## Solución de problemas y Advanced
</div>

<Accordion title="Problemas comunes y soluciones">
  ### Errores de compilación: no se encuentra el módulo &#39;Auth0&#39;

  **Soluciones**:

  1. **Swift Package Manager**: Comprueba **Package Dependencies** → verifica que `Auth0.swift` aparezca en la lista
  2. **CocoaPods**: Asegúrate de abrir el archivo `.xcworkspace`, no `.xcodeproj`
  3. **Carthage**: Verifica que `Auth0.xcframework` esté agregado a **Frameworks, Libraries, and Embedded Content**
  4. Limpia y vuelve a compilar: **⌘+Shift+K** y luego **⌘+R**
  5. Reinicia Xcode si hace falta

  ### La app se cierra: no se encuentra &#39;Auth0.plist&#39;

  **Solución**:

  1. Verifica que `Auth0.plist` esté en el navegador del proyecto de Xcode
  2. Selecciona el archivo → Inspector → asegúrate de que el target de tu app esté marcado
  3. Confirma que tenga las claves `ClientId` y `Domain` con tus valores

  ### El navegador se abre, pero nunca vuelve a la app

  **Solución**:

  1. Comprueba que las URL de devolución de llamada en Auth0 Dashboard coincidan exactamente con tu identificador de paquete y tu plataforma
  2. Para iOS: las URL deben contener `/ios/`; para macOS: `/macos/`
  3. Verifica que el identificador de paquete en Xcode coincida con la configuración de Auth0
  4. Asegúrate de que no haya errores tipográficos en las URL (los más comunes: faltan dos puntos o el formato del dominio es incorrecto)
  5. **Si usas un dominio personalizado**: verifica que estés usando tu dominio personalizado, no el dominio de Auth0

  ### La alerta de permiso aparece cada vez

  Este es el comportamiento de seguridad estándar de iOS/macOS al usar esquemas de URL personalizados. Consulta el **Paso 6** para eliminar esta alerta usando Universal Links o Ephemeral Sessions.
</Accordion>

<Accordion title="Configuración del dominio personalizado">
  Si usas un [dominio personalizado](/docs/es/customize/custom-domains), utiliza ese valor en lugar de tu dominio de Auth0 en todas partes.

  **Ejemplo:** Usa `login.example.com` en lugar de `tenant.auth0.com`

  Esto es **necesario** para que ciertas funciones funcionen correctamente:

  * Actualiza `Auth0.plist` con tu dominio personalizado
  * Usa el dominio personalizado en las URL de devolución de llamada y cierre de sesión
  * Para Universal Links, usa: `webcredentials:login.example.com`
</Accordion>

<Accordion title="Despliegue en producción">
  ### Preparación para App Store

  * Configura Universal Links para eliminar la alerta de permiso
  * Haz pruebas en varias versiones de la plataforma y en distintos tamaños de dispositivo
  * Implementa un manejo de errores adecuado para fallos de red
  * Agrega descripciones de uso de privacidad si usas Keychain con biometría
  * Sigue las App Store Review Guidelines para los flujos de autenticación

  ### Prácticas recomendadas de seguridad

  * Nunca registres datos sensibles de autenticación en producción
  * Implementa el cumplimiento de App Transport Security (ATS)
  * Usa HTTPS para todas las solicitudes de red
  * **NO** fijes certificados de la API de Auth0: [Auth0 no recomienda esta práctica](/docs/es/troubleshoot/product-lifecycle/past-migrations#avoid-pinning-or-fingerprinting-tls-certificates-for-auth0-endpoints)

  ### Optimización del rendimiento

  * Todas las operaciones asíncronas usan correctamente `@MainActor` para actualizar la UI
  * Las propiedades `@Published` usan un manejo de memoria adecuado
  * Las credenciales se almacenan en caché de forma segura en el Keychain para acceso sin conexión
  * El perfil de usuario se recupera del token de ID (sin necesidad de una solicitud de red adicional)
</Accordion>

<Accordion title="Integración avanzada">
  ### Seguridad mejorada del llavero con biometría

  Exige Face ID o Touch ID para acceder a las credenciales almacenadas:

  ```swift
  private let credentialsManager: CredentialsManager = {
      let manager = CredentialsManager(authentication: Auth0.authentication())
      manager.enableBiometrics(
          withTitle: "Unlock with Face ID", 
          cancelTitle: "Cancel", 
          fallbackTitle: "Use Passcode"
      )
      return manager
  }()
  ```

  Cuando esta opción está activada, los usuarios deben autenticarse con biometría antes de que el SDK pueda recuperar las credenciales almacenadas.

  ### Renovación automática de tokens

  El `CredentialsManager` renueva automáticamente los tokens de acceso vencidos:

  ```swift
  // Obtener credenciales: se renuevan automáticamente si vencieron
  func getAccessToken() async throws -> String {
      let credentials = try await credentialsManager.credentials()
      return credentials.accessToken
  }
  ```

  Usa este patrón al hacer llamadas a la API que requieran un token de acceso.

  ### Credenciales compartidas entre extensiones de la app

  Para widgets, extensiones de la app o tareas en segundo plano que necesiten tokens de acceso:

  ```swift
  // Crear un administrador de credenciales compartidas con un grupo de apps
  let credentialsManager = CredentialsManager(
      authentication: Auth0.authentication(),
      storeKey: "credentials",
      storage: .shared(withIdentifier: "group.com.example.myapp")
  )
  ```

  **Requisitos:**

  1. Habilita la capacidad **App Groups** en Xcode para todos los targets
  2. Usa el mismo identificador de grupo de apps en todos los targets
  3. Configura el `CredentialsManager` compartido en cada target

  ### Comparación de opciones del flujo de autenticación

  | Característica                        | Universal Links                      | Sesión efímera | Predeterminado (alerta)   |
  | ------------------------------------- | ------------------------------------ | -------------- | ------------------------- |
  | Alerta de permisos                    | Reducida (sin prompt de redirección) | Ninguna        | Muestra todas las alertas |
  | Compatibilidad con SSO                | Sí                                   | No             | Sí                        |
  | Cuenta de Apple Developer             | Obligatoria                          | No obligatoria | No obligatoria            |
  | Experiencia de usuario                | Óptima                               | Buena          | Aceptable                 |
  | Complejidad de configuración          | Media                                | Fácil          | Fácil                     |
  | Compatibilidad con navegación privada | Sí                                   | Sí             | No                        |

  **Recomendaciones:**

  * **Apps de producción con SSO**: Universal Links (mejor experiencia de usuario, compatibilidad con SSO, requiere una cuenta de Apple Developer)
  * **Apps de producción sin SSO**: Sesiones efímeras (sin alertas, configuración más sencilla)
  * **Pruebas/Desarrollo**: Sesiones efímeras (configuración rápida y experiencia de usuario más limpia)
  * **Inicio rápido/Prototipado**: Predeterminado con alertas (sin configuración; puedes migrar más adelante)
</Accordion>
