<Note>
  **Requisitos previos:** Antes de empezar, asegúrate de tener lo siguiente:

  * **[SDK de .NET 8.0](https://dotnet.microsoft.com/download/dotnet/8.0)** o una versión posterior (o .NET Framework 4.6.2 para proyectos heredados)
  * **[Visual Studio 2022](https://visualstudio.microsoft.com/)** o VS Code con C# Dev Kit

  **Compatibilidad de versiones de .NET:** Este Quickstart funciona con **.NET 8.0**, **.NET 9.0** y **.NET Framework 4.6.2**.
</Note>

<div id="get-started">
  ## Primeros pasos
</div>

Esta guía de inicio rápido muestra cómo añadir la autenticación con Auth0 a una aplicación de escritorio WPF o WinForms. Configurarás Auth0, instalarás el SDK y añadirás el inicio de sesión, el cierre de sesión y la visualización del perfil del usuario mediante Auth0 OIDC Client para [WPF](https://www.nuget.org/packages/Auth0.OidcClient.WPF/) y [WinForms](https://www.nuget.org/packages/Auth0.OidcClient.WinForms/).

<Steps>
  <Step title="Crea tu aplicación" stepNumber={1}>
    Si ya tiene un proyecto de WPF o WinForms, vaya al siguiente paso.

    <Tabs>
      <Tab title=".NET CLI">
        Cree un proyecto nuevo y abra su directorio:

        <Tabs>
          <Tab title="WPF">
            ```bash
            dotnet new wpf -n MyApp
            cd MyApp
            ```
          </Tab>

          <Tab title="WinForms">
            ```bash
            dotnet new winforms -n MyApp
            cd MyApp
            ```
          </Tab>
        </Tabs>
      </Tab>

      <Tab title="Visual Studio">
        1. Abra Visual Studio 2022 y haga clic en **Crear un proyecto**
        2. Busque **Aplicación WPF** o **Aplicación de Windows Forms**
        3. Seleccione la plantilla de C# y haga clic en **Siguiente**
        4. Escriba un nombre para el proyecto y elija una ubicación; luego, haga clic en **Siguiente**
        5. Seleccione **.NET 8.0** (o una versión posterior) como framework de destino y haga clic en **Crear**
      </Tab>
    </Tabs>
  </Step>

  <Step title="Configurar Auth0" stepNumber={2}>
    Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es donde configuras cómo funciona la autenticación en tu proyecto.

    ### Configurar una aplicación

    Ve a [Auth0 Dashboard](https://manage.auth0.com) → **Applications** → **Applications** y crea una aplicación nueva:

    1. Haz clic en **Create Application**
    2. Introduce un nombre para tu aplicación
    3. Selecciona **Native** como tipo de aplicación
    4. Haz clic en **Create**

    En la pestaña **Settings**, anota tu **Domain** y tu **ID de cliente**; los necesitarás para inicializar el SDK.

    ### Configurar URL de devolución de llamada

    Una URL de devolución de llamada es una URL de tu aplicación a la que Auth0 redirige a los usuarios después de autenticarse. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

    En **Application Settings**, agrega lo siguiente a **Allowed Callback URLs**:

    ```
    https://{yourDomain}/mobile
    ```

    ### Configurar las URL de cierre de sesión

    Una URL de cierre de sesión es una URL de tu aplicación a la que Auth0 redirige a los usuarios después de cerrar sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

    En **Application Settings**, agrega lo siguiente a **Allowed Logout URLs**:

    ```
    https://{yourDomain}/mobile
    ```
  </Step>

  <Step title="Instalar el SDK de Auth0" stepNumber={3}>
    Auth0 proporciona paquetes NuGet independientes para WPF y WinForms. Instala el que corresponda al tipo de tu proyecto.

    <Tabs>
      <Tab title="Administrador de paquetes NuGet">
        Abre la **Consola del Administrador de paquetes** (Herramientas → Administrador de paquetes NuGet → Consola del Administrador de paquetes) y ejecuta:

        ```powershell
        # WPF
        Install-Package Auth0.OidcClient.WPF

        # WinForms
        Install-Package Auth0.OidcClient.WinForms
        ```
      </Tab>

      <Tab title=".NET CLI">
        ```bash
        # WPF
        dotnet add package Auth0.OidcClient.WPF

        # WinForms
        dotnet add package Auth0.OidcClient.WinForms
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Crear una instancia de Auth0Client" stepNumber={4}>
    Para integrar Auth0 en tu aplicación, crea una instancia de `Auth0Client` con el **Domain** de Auth0 y el **ID de cliente**. Agrega un campo privado e inicialízalo dentro del constructor existente de la ventana principal o del formulario.

    <Tabs>
      <Tab title="WPF">
        Abre `MainWindow.xaml.cs` y actualízalo de la siguiente manera:

        ```csharp MainWindow.xaml.cs lines
        using Auth0.OidcClient;

        // En caso de que hayas elegido un nombre diferente para tu aplicación,
        // asegúrate de actualizar el espacio de nombres en consecuencia.
        namespace MyApp; 

        public partial class MainWindow : Window
        {
            private Auth0Client _client;

            public MainWindow()
            {
                InitializeComponent();

                _client = new Auth0Client(new Auth0ClientOptions
                {
                    Domain = "{yourDomain}",
                    ClientId = "{yourClientId}"
                });
            }
        }
        ```
      </Tab>

      <Tab title="WinForms">
        Abre `Form1.cs` (o el archivo del formulario principal) y actualízalo de la siguiente manera:

        ```csharp Form1.cs lines
        using Auth0.OidcClient;

        // En caso de que hayas elegido un nombre diferente para tu aplicación,
        // asegúrate de actualizar el espacio de nombres en consecuencia.
        namespace MyApp;

        public partial class Form1 : Form
        {
            private Auth0Client _client;

            public Form1()
            {
                InitializeComponent();

                _client = new Auth0Client(new Auth0ClientOptions
                {
                    Domain = "{yourDomain}",
                    ClientId = "{yourClientId}"
                });
            }
        }
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Añade el inicio de sesión a tu aplicación" stepNumber={5}>
    Usa el método `LoginAsync()` del SDK para iniciar sesión. Al llamarlo, se abre una ventana emergente con la página de Auth0 Universal Login. Después de autenticarse correctamente, Auth0 redirige a la URL de devolución de llamada y el SDK devuelve un `LoginResult`.

    Primero, agrega un botón de inicio de sesión a tu interfaz de usuario:

    <Tabs>
      <Tab title="WPF">
        Abre `MainWindow.xaml` y agrega un `Button` dentro del `<Grid>`:

        ```xml MainWindow.xaml lines
        <Grid>
            <Button x:Name="LoginButton"
                    Content="Log In"
                    Width="120" Height="40"
                    HorizontalAlignment="Center" VerticalAlignment="Center"
                    Click="LoginButton_Click" />
        </Grid>
        ```

        Luego agrega el controlador del clic en `MainWindow.xaml.cs`:

        ```csharp MainWindow.xaml.cs lines
        private async void LoginButton_Click(object sender, RoutedEventArgs e)
        {
            var loginResult = await _client.LoginAsync();

            if (loginResult.IsError == false)
            {
                var user = loginResult.User;
                var name = user.FindFirst(c => c.Type == "name")?.Value;
                var email = user.FindFirst(c => c.Type == "email")?.Value;
                var picture = user.FindFirst(c => c.Type == "picture")?.Value;
            }
        }
        ```
      </Tab>

      <Tab title="WinForms">
        Abre `Form1.cs` y agrega el botón de inicio de sesión mediante programación en el constructor; luego agrega el controlador del clic:

        ```csharp Form1.cs lines
        using Auth0.OidcClient;

        namespace MyApp;

        public partial class Form1 : Form
        {
            private Auth0Client _client;
            private Button loginButton;

            public Form1()
            {
                InitializeComponent();

                _client = new Auth0Client(new Auth0ClientOptions
                {
                    Domain = "{yourDomain}",
                    ClientId = "{yourClientId}"
                });

                loginButton = new Button
                {
                    Text = "Log In",
                    Width = 120,
                    Height = 40,
                    Left = (ClientSize.Width - 120) / 2,
                    Top = (ClientSize.Height - 40) / 2
                };
                loginButton.Click += loginButton_Click;
                Controls.Add(loginButton);
            }

            private async void loginButton_Click(object sender, EventArgs e)
            {
                var loginResult = await _client.LoginAsync();

                if (loginResult.IsError == false)
                {
                    var user = loginResult.User;
                    var name = user.FindFirst(c => c.Type == "name")?.Value;
                    var email = user.FindFirst(c => c.Type == "email")?.Value;
                    var picture = user.FindFirst(c => c.Type == "picture")?.Value;
                }
            }
        }
        ```
      </Tab>
    </Tabs>

    Si no hay errores, puedes acceder a `LoginResult.User`, `LoginResult.IdentityToken`, `LoginResult.AccessToken` y `LoginResult.RefreshToken` en el resultado.
  </Step>

  <Step title="Añade el cierre de sesión a tu aplicación" stepNumber={6}>
    Usa el método `LogoutAsync()` del SDK para cerrar la sesión de los usuarios. Esto abre una ventana emergente, redirige al endpoint de cierre de sesión de Auth0 para eliminar la sesión y luego te redirige a la URL de cierre de sesión que configuraste.

    Primero, agrega un botón de cierre de sesión a tu interfaz de usuario:

    <Tabs>
      <Tab title="WPF">
        Abre `MainWindow.xaml` y agrega un `Button` de cierre de sesión junto al botón de inicio de sesión:

        ```xml MainWindow.xaml lines
            <Grid>

                <StackPanel HorizontalAlignment="Center" VerticalAlignment="Center">
                    <Button x:Name="LoginButton" Content="Login" Width="200" Height="40" 
                            Margin="10" Click="LoginButton_Click" FontSize="16"/>
                    <Button x:Name="LogoutButton" Content="Logout" Width="200" Height="40" 
                            Margin="10" Click="LogoutButton_Click" FontSize="16"/>
                </StackPanel>
            </Grid>
        ```

        Luego, agrega el controlador del clic en `MainWindow.xaml.cs`:

        ```csharp MainWindow.xaml.cs lines
        private async void LogoutButton_Click(object sender, RoutedEventArgs e)
        {
            await _client.LogoutAsync();
        }
        ```
      </Tab>

      <Tab title="WinForms">
        Abre `Form1.cs` y agrega el botón de cierre de sesión en el constructor; luego, agrega su controlador del clic:

        ```csharp Form1.cs lines
        private Button logoutButton;

        public Form1()
        {
            // ... código existente del constructor ...

            logoutButton = new Button
            {
                Text = "Log Out",
                Width = 120,
                Height = 40,
                Left = (ClientSize.Width - 120) / 2,
                Top = (ClientSize.Height - 40) / 2 + 50
            };
            logoutButton.Click += logoutButton_Click;
            Controls.Add(logoutButton);
        }

        private async void logoutButton_Click(object sender, EventArgs e)
        {
            await _client.LogoutAsync();
        }
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Mostrar información del perfil del usuario" stepNumber={7}>
    La propiedad `LoginResult.User` es un [ClaimsPrincipal](https://learn.microsoft.com/en-us/dotnet/api/system.security.claims.claimsprincipal) que contiene el perfil del usuario autenticado. Consulta los claims para mostrar la información del usuario en tu aplicación.

    ```csharp lines
    if (loginResult.IsError == false)
    {
        Debug.WriteLine($"name: {loginResult.User.FindFirst(c => c.Type == "name")?.Value}");
        Debug.WriteLine($"email: {loginResult.User.FindFirst(c => c.Type == "email")?.Value}");
    }
    ```

    Para ver todos los claims que devuelve el token de identidad:

    ```csharp lines
    if (loginResult.IsError == false)
    {
        foreach (var claim in loginResult.User.Claims)
        {
            Debug.WriteLine($"{claim.Type} = {claim.Value}");
        }
    }
    ```
  </Step>
</Steps>

<Check>
  **Comprobación**

  Ahora deberías tener una aplicación WPF o WinForms integrada con Auth0 y funcionando. Ejecuta la aplicación y verifica lo siguiente:

  * Al hacer clic en el botón de inicio de sesión, se abre la página de Auth0 Universal Login en una ventana emergente.
  * Puedes iniciar sesión o registrarte.
  * Después de la autenticación, puedes acceder a la información del usuario desde `LoginResult.User`.
  * Al hacer clic en el botón de cierre de sesión, se cierra la sesión y se te redirige a tu URL de cierre de sesión.
</Check>

<div id="advanced-usage">
  ## Uso avanzado
</div>

<Accordion title="Gestionar errores de inicio de sesión">
  Comprueba `LoginResult.IsError` antes de acceder a los tokens o a las propiedades del usuario. Las propiedades `Error` y `ErrorDescription` contienen detalles cuando falla la authentication.

  ```csharp lines
  var loginResult = await _client.LoginAsync();

  if (loginResult.IsError)
  {
      Debug.WriteLine($"An error occurred during login: {loginResult.Error}");
      // loginResult.ErrorDescription contiene el mensaje de error completo
      return;
  }

  // Aquí puedes acceder con seguridad a los tokens y al usuario
  Debug.WriteLine($"id_token: {loginResult.IdentityToken}");
  Debug.WriteLine($"access_token: {loginResult.AccessToken}");
  ```

  Si el usuario cierra la ventana emergente de inicio de sesión sin autenticarse, `LoginAsync()` devuelve un resultado con `BrowserResultType.UserCancel`. Esto es normal; no lo trates como un error.
</Accordion>

<Accordion title="Tokens de actualización">
  Para obtener un nuevo token de acceso sin que el usuario tenga que volver a iniciar sesión, usa `RefreshTokenAsync()` con el token de actualización del `LoginResult` inicial.

  ```csharp lines
  // Solicita el scope offline_access para recibir un token de actualización
  _client = new Auth0Client(new Auth0ClientOptions
  {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}",
      Scope = "openid profile email offline_access"
  });

  // Almacena el token de actualización del inicio de sesión inicial
  var refreshToken = loginResult.RefreshToken;

  // Intercámbialo más adelante por nuevos tokens
  var refreshResult = await _client.RefreshTokenAsync(refreshToken);

  if (refreshResult.IsError == false)
  {
      var newAccessToken = refreshResult.AccessToken;
  }
  ```

  <Info>
    Los tokens de actualización requieren el scope `offline_access` y deben estar habilitados en Application Settings de tu aplicación de Auth0, en **rotación del Refresh Token**.
  </Info>
</Accordion>

<div id="common-issues">
  ## Problemas comunes
</div>

<AccordionGroup>
  <Accordion title="Se abre la ventana emergente de inicio de sesión, pero la autenticación nunca se completa">
    **Problema:** Se abre la ventana emergente de WebView2 y muestra la página de inicio de sesión, pero después de introducir las credenciales no ocurre nada.

    **Solución:** Microsoft Edge WebView2 Runtime no está instalado en el equipo. Instálalo desde la [página de descarga de Microsoft WebView2](https://developer.microsoft.com/en-us/microsoft-edge/webview2/). WebView2 viene incluido con Windows 11 y con versiones recientes de Windows 10, pero debe instalarse por separado en sistemas más antiguos.
  </Accordion>

  <Accordion title="Error de discrepancia en la URL de devolución de llamada">
    **Problema:** Después de iniciar sesión, Auth0 devuelve un error `callback URL mismatch`.

    **Solución:** El Redirect URI que usa el SDK no coincide con ninguno de los valores de **Allowed Callback URLs** en el Auth0 Dashboard. Agrega `https://{yourDomain}/mobile` a Allowed Callback URLs en Application Settings. El SDK usa esta URL de forma predeterminada.
  </Accordion>

  <Accordion title="Error de discrepancia en la URL de cierre de sesión">
    **Problema:** Después de cerrar sesión, Auth0 devuelve un error sobre una URL de cierre de sesión no reconocida.

    **Solución:** Agrega `https://{yourDomain}/mobile` a **Allowed Logout URLs** en Application Settings.
  </Accordion>

  <Accordion title="LoginResult.IsError es true después de iniciar sesión">
    **Problema:** `LoginResult.IsError` es `true`, pero no hay una indicación clara de la causa.

    **Solución:** Revisa `LoginResult.Error` y `LoginResult.ErrorDescription` para ver los detalles:

    ```csharp lines
    if (loginResult.IsError)
    {
        Debug.WriteLine($"Error: {loginResult.Error}");
        Debug.WriteLine($"Description: {loginResult.ErrorDescription}");
    }
    ```

    Causas comunes:

    * El Application type en el Auth0 Dashboard no está configurado como **Native**
    * **OIDC Conformant** no está habilitado en Advanced Settings → OAuth
    * **JSON Web Token Signature Algorithm** no está configurado como **RS256**
  </Accordion>

  <Accordion title="RefreshToken es null después de iniciar sesión">
    **Problema:** `LoginResult.RefreshToken` es `null`.

    **Solución:** El scope `offline_access` es necesario para recibir un token de actualización. Agrégalo a la opción `Scope`:

    ```csharp lines
    _client = new Auth0Client(new Auth0ClientOptions
    {
        Domain = "{yourDomain}",
        ClientId = "{yourClientId}",
        Scope = "openid profile email offline_access"
    });
    ```

    Asegúrate también de que **rotación del Refresh Token** esté habilitada en Application Settings del Auth0 Dashboard.
  </Accordion>
</AccordionGroup>
