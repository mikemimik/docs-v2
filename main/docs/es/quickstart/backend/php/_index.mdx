    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 proporciona un [SDK de Auth0 para PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y autorización de Auth0 en aplicaciones PHP.

      El SDK de Auth0 para PHP requiere bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) para gestionar solicitudes de red. Si no
      dispone de estas bibliotecas, puede instalar opciones confiables ejecutando los siguientes comandos en su
      terminal:

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora, instala el SDK de Auth0 para PHP ejecutando el siguiente comando en tu terminal:

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configura el SDK de Auth0

      Para que el SDK funcione correctamente, debes establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `strategy`: esta estrategia ayuda a definir el comportamiento del SDK según el caso de uso de tu aplicación. En este
        caso, debes establecerla en la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain: El dominio de tu tenant de Auth0. Por lo general, lo encontrarás en el Auth0 Dashboard, en la
        sección Settings de la aplicación, en el campo Domain. Si usas un dominio personalizado, establece aquí
        el valor de tu dominio personalizado.
      * clientId: El ID de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes encontrarlo en el
        Auth0 Dashboard, en la sección Settings de tu aplicación, en el campo Client ID.
      * clientSecret: El secreto de la aplicación de Auth0 que creaste antes en este Quickstart. El secreto de cliente está en
        el Auth0 Dashboard, en la sección Settings de tu aplicación, en el campo Client Secret.
      * audience: El identificador de la Auth0 API que registraste anteriormente. Debe proporcionarse como un arreglo.

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      A continuación, amplía tu aplicación para recuperar y procesar bearer tokens. Los bearer tokens son tokens de acceso que se envían a
      tu API en solicitudes de clientes en nombre de los usuarios. Los tokens de acceso autorizan o deniegan el acceso a rutas de tu
      aplicación. Esto se conoce como autorización de endpoints.

      La forma más sencilla de recuperar tokens de acceso de una solicitud es usar el método `getBearerToken()`
      del SDK de PHP. Este método obtiene tokens de parámetros GET, cuerpos POST, encabezados de solicitud y otras fuentes. En este
      caso, el SDK de PHP procesa los tokens enviados en solicitudes GET en el parámetro `token` o en el
      encabezado HTTP `Authorization`.

      Ahora que has configurado tu aplicación de Auth0, el SDK de Auth0 para PHP, y tu aplicación extrae bearer
      tokens de las solicitudes, el siguiente paso es configurar la autorización de endpoints para tu proyecto. El método
      `getBearerToken()` que implementaste arriba devuelve una clase `Token` que incluye
      detalles sobre el acceso de la solicitud.

      Dado que el método `getBearerToken()` valida y verifica automáticamente la solicitud entrante, tu
      aplicación determina los detalles del token de acceso evaluando la respuesta del método. Cuando la respuesta es
      null, no se ha proporcionado ningún token válido. De lo contrario, inspecciona el contenido de la respuesta para obtener más información sobre la
      solicitud.

      En el panel interactivo de la derecha, puedes ver una comprobación para determinar si la respuesta es null o no y así filtrar el acceso a
      tu ruta `/api/private`.
