  <Content>
    Auth0 te permite añadir autorización de endpoints basada en tokens a casi cualquier tipo de aplicación de forma rápida. Esta guía
    muestra cómo integrar Auth0, añadir autorización basada en tokens y proteger las rutas de la aplicación con el Auth0
    PHP SDK.

    Para usar esta guía de inicio rápido, necesitarás:

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitas tener una aplicación registrada en el Auth0 Dashboard. La aplicación de Auth0
      es donde configuras cómo quieres que funcione la autenticación en tu proyecto.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente único y alfanumérico que
      el código de tu aplicación usa para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con este Quickstart se actualiza automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones más adelante.

      Si prefieres explorar una configuración completa, también puedes ver una aplicación de ejemplo.

      ### Configurar una API

      Del mismo modo, necesitas crear una nueva API de Auth0 o usar una API existente que represente el proyecto que estás
      integrando desde el [Dashboard](https://manage.auth0.com/#/). Elige un identificador único para la API y toma nota de él. Necesitas
      ese identificador para configurar tu aplicación más abajo.
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 proporciona un [SDK de
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y la autorización con Auth0 en aplicaciones PHP.

      El SDK de Auth0 para PHP requiere tener instaladas bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) para gestionar solicitudes de red. Si no
      tienes bibliotecas disponibles, puedes instalar opciones fiables ejecutando los siguientes comandos en tu
      terminal:

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora, instala el SDK de Auth0 para PHP ejecutando el siguiente comando en la terminal:

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configura el SDK de Auth0

      Para que el SDK funcione correctamente, debes establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `strategy`: La `strategy` define el comportamiento del SDK según el caso de uso de tu aplicación. En este
        caso, debes establecerla en la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain: El dominio de tu tenant de Auth0. Por lo general, lo encontrarás en el Auth0 Dashboard, en la
        sección Settings de la aplicación, en el campo Domain. Si usas un dominio personalizado, establece este valor con el de tu dominio personalizado
        en su lugar.
      * clientId: El ID de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes encontrarlo en el
        Auth0 Dashboard, en la sección Settings de tu aplicación, en el campo Client ID.
      * clientSecret: El secreto de la aplicación de Auth0 que creaste antes en este Quickstart. El secreto de cliente está en
        el Auth0 Dashboard, en la sección Settings de tu aplicación, en el campo Client Secret.
      * audience: El identificador de la Auth0 API que registraste arriba. Debe proporcionarse como un arreglo.

      <Note>
        ##### Checkpoint

        Tu SDK de Auth0 ya está configurado correctamente. Ejecuta tu aplicación para verificar que:

        * El SDK se inicializa correctamente.
        * Tu aplicación no produce ningún error relacionado con Auth0.
      </Note>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      A continuación, amplía tu aplicación para recuperar y procesar bearer tokens. Los bearer tokens son tokens de acceso que se envían a
      tu API con solicitudes de clientes en nombre de los usuarios. Los tokens de acceso autorizan o deniegan el acceso a las rutas de tu
      aplicación. Esto se conoce como autorización de endpoints.

      La forma más sencilla de recuperar tokens de acceso de una solicitud es usar el método `getBearerToken()`
      del SDK de PHP. Este método obtiene tokens de los parámetros GET, los cuerpos POST, los encabezados de la solicitud y otras fuentes. En este
      caso, el SDK de PHP procesa los tokens que se pasan en solicitudes GET mediante el parámetro `token` o mediante el encabezado HTTP
      `Authorization`.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Ahora, instala una biblioteca de enrutamiento para dirigir las solicitudes entrantes a tu aplicación. No es un paso obligatorio,
      pero simplifica la estructura de la aplicación para este Quickstart.

      ```bash lines
      composer require steampixel/simple-php-router
      ```

      Crea un nuevo archivo en tu aplicación llamado `router.php` para definir las rutas. Copia el código del
      panel interactivo de la derecha, en la pestaña **router.php**.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Ahora que has configurado tu aplicación de Auth0, el SDK de Auth0 para PHP, y tu aplicación recupera tokens bearer
      de las solicitudes, el siguiente paso es configurar la autorización del endpoint para tu proyecto. El
      método `getBearerToken()` que implementaste antes devuelve una clase `Token` que incluye
      detalles sobre el acceso de la solicitud.

      Dado que el método `getBearerToken()` valida y verifica automáticamente la solicitud entrante, tu
      aplicación determina los detalles del token de acceso evaluando la respuesta del método. Cuando la respuesta es
      null, no se ha proporcionado ningún token válido. De lo contrario, inspecciona el contenido de la respuesta para obtener más información sobre la
      solicitud.

      En el panel interactivo de la derecha, puedes ver una comprobación de si la respuesta es null o no para filtrar el acceso a
      tu ruta `/api/private`.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      En algunos casos, puede que quieras filtrar el acceso a una ruta específica en función de los scopes solicitados en un token de acceso. Como se muestra en el panel interactivo de la derecha, evalúa el contenido de la propiedad &#39;scope&#39; de la
      respuesta del método `getBearerToken()` para comprobar los scopes concedidos por el token de acceso.
    </Section>
