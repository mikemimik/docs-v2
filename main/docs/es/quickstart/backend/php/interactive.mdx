  <Content>
    Auth0 te permite añadir autorización de endpoints basada en tokens a casi cualquier tipo de aplicación de forma rápida. Esta guía
    muestra cómo integrar Auth0, añadir autorización basada en tokens y proteger las rutas de la aplicación con el Auth0
    PHP SDK.

    Para usar esta guía de inicio rápido, necesitarás:

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitas tener una aplicación registrada en el Auth0 Dashboard. En la aplicación de Auth0
      es donde configuras cómo quieres que funcione la autenticación en tu proyecto.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación en Auth0 se le asigna un ID de cliente único y alfanumérico que
      el código de tu aplicación usa para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con este Quickstart se actualiza automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones más adelante.

      Si prefieres explorar una configuración completa, puedes ver una aplicación de ejemplo.

      ### Configurar una API

      Del mismo modo, necesitas crear una nueva API de Auth0 o usar una API existente que represente el proyecto que estás
      integrando desde el [Dashboard](https://manage.auth0.com/#/). Elige un identificador único para la API y toma nota de él. Necesitas
      ese identificador para configurar tu aplicación a continuación.
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 proporciona un [SDK para
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para facilitar la implementación de la autenticación y autorización de Auth0 en aplicaciones PHP.

      El SDK de Auth0 para PHP requiere bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) instaladas para gestionar las solicitudes de red. Si no
      dispone de bibliotecas, puede instalar opciones fiables ejecutando los siguientes comandos en la
      terminal:

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora, instala el SDK de Auth0 para PHP ejecutando el siguiente comando en la terminal:

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configura el SDK de Auth0

      Para que el SDK funcione correctamente, debes establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `strategy`: La estrategia ayuda a orientar el comportamiento del SDK según el caso de uso de tu aplicación. En este
        caso, debes establecerla en la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain: El dominio de tu tenant de Auth0. Por lo general, lo encontrarás en el Auth0 Dashboard, en los
        Settings de la aplicación, en el campo Domain. Si usas un dominio personalizado, establece este valor en el de tu dominio personalizado
        en su lugar.
      * clientId: El ID de la aplicación de Auth0 que configuraste anteriormente en este Quickstart. Puedes encontrarlo en el
        Auth0 Dashboard, en los Settings de tu aplicación, en el campo ID de cliente.
      * clientSecret: El secreto de la aplicación de Auth0 que creaste anteriormente en este Quickstart. El secreto de cliente está en
        el Auth0 Dashboard, en los Settings de tu aplicación, en el campo secreto de cliente.
      * audience: El identificador de la Auth0 API que registraste anteriormente. Debe proporcionarse como un arreglo.

      <Note>
        ##### Checkpoint

        Tu SDK de Auth0 ya está configurado correctamente. Ejecuta tu aplicación para verificar que:

        * El SDK se inicializa correctamente.
        * Tu aplicación no genera errores relacionados con Auth0.
      </Note>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      A continuación, amplíe su aplicación para recuperar y procesar tokens de portador. Los tokens de portador son tokens de acceso que se envían a
      su API en solicitudes de clientes en nombre de los usuarios. Los tokens de acceso permiten o deniegan el acceso a routes en su
      aplicación. Esto se conoce como autorización de endpoint.

      La forma más sencilla de recuperar tokens de acceso de una solicitud es usar el método `getBearerToken()`
      del SDK de PHP. Este método obtiene tokens de parámetros GET, cuerpos POST, encabezados de solicitud y otras fuentes. En este
      caso, el SDK de PHP procesa los tokens enviados en solicitudes GET en el parámetro `token` o en el
      encabezado HTTP `Authorization`.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Ahora, instala una biblioteca de enrutamiento para dirigir las solicitudes entrantes a tu aplicación. Este no es un paso obligatorio,
      pero simplifica la estructura de la aplicación a efectos de este Quickstart.

      ```bash lines
      composer require steampixel/simple-php-router
      ```

      Crea un archivo nuevo en tu aplicación llamado `router.php` para definir las rutas. Copia el código del
      panel interactivo de la derecha, en la pestaña **router.php**.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Ahora que ha configurado su aplicación de Auth0, el SDK de Auth0 para PHP, y su aplicación recupera bearer
      tokens de las requests, el siguiente paso es configurar la autorización del endpoint para su proyecto. El método
      `getBearerToken()` que implementó arriba devuelve una clase `Token` que incluye
      detalles sobre el acceso de la request.

      Dado que el método `getBearerToken()` valida y verifica automáticamente la request entrante, su
      aplicación determina los detalles del token de acceso evaluando la respuesta del método. Cuando la respuesta es
      null, no se ha proporcionado ningún token válido. De lo contrario, inspeccione el contenido de la respuesta para saber más sobre la
      request.

      En el panel interactivo de la derecha, puede ver una comprobación de si la respuesta es null o no para filtrar el acceso a
      su ruta `/api/private`.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      En algunos casos, puede que quieras restringir el acceso a una ruta específica en función de los scopes solicitados en un
      token de acceso. Como se muestra en el panel interactivo de la derecha, evalúa el contenido de la propiedad &#39;scope&#39; de la
      respuesta del método `getBearerToken()` para verificar los scopes concedidos por el token de acceso.
    </Section>
