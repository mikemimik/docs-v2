Este documento te ayudará a solucionar problemas con tu configuración si obtienes respuestas inesperadas de tu API. Te recomendamos iniciar sesión para seguir este Quickstart con ejemplos configurados para tu cuenta.

Si configuraste correctamente la validación de JWT, podrás obtener respuestas correctas de tu API cuando hagas solicitudes. Sin embargo, si recibes una respuesta 401 (No autorizado) de tu API, se debe a que la configuración de tu middleware de JWT no coincide con el JWT que se envió.

Este documento te ayudará a solucionar problemas con la configuración de tu middleware de JWT.

<div id="how-does-a-token-get-validated">
  ## ¿Cómo se valida un token?
</div>

A la hora de validar un JWT, hay varios aspectos que se deben tener en cuenta:

1. **¿El token tiene un formato válido?** En otras palabras, ¿este token cumple con la estructura de un JSON Web Token (JWT)? Para obtener más información sobre la estructura de un JWT, consulta [esta sección sobre la estructura de un JWT](/docs/es/secure/tokens/json-web-tokens#what-is-the-json-web-token-structure-)
2. **¿Se ha manipulado el token?** La última parte de un JWT es la firma. La firma se utiliza para verificar que el token fue firmado efectivamente por el remitente y que no se ha modificado de ninguna manera.
3. **¿Se ha recibido el token dentro de su período de validez?** Los JWT solo son válidos durante un período de tiempo determinado (como se indica en el claim `exp`).
4. **¿Proviene el token de la autoridad esperada?** Esto consta de 2 partes

   * **Verificación de la firma**: ¿Podemos confirmar que el JWT está correctamente firmado con la key emitida por la autoridad emisora?
   * **Valor del issuer**: El issuer se define en el claim `iss`. Una vez más, ¿este claim coincide con lo que espera tu aplicación?
5. **¿Está destinado el token a la aplicación actual?** Es decir, ¿el claim `aud` del JWT coincide con lo que espera tu aplicación?

<div id="inspecting-a-token">
  ## Inspección de un token
</div>

Una forma rápida de inspeccionar un JWT es usar el sitio web [JWT.io](https://jwt.io/). Tiene un práctico depurador que te permite comprobar rápidamente que un JWT tiene el formato correcto, así como inspeccionar los valores de los distintos claims.

<Frame>![Depuración de un JWT en JWT.io](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-rs256.png)</Frame>

En la captura de pantalla anterior puedes ver que el token se firmó con el algoritmo **RS256**. El **Issuer** del token es **https://jerrie.auth0.com/** y el **Audience** es **https://rs256.test.api**.

En otras palabras, estos valores de la configuración de validación de tu JWT deben coincidir **exactamente**, incluida la barra diagonal final del Issuer, por ejemplo

```
audience = "https://rs256.test.api",
domain = "https://jerrie.auth0.com/"
};
```

Para un token firmado con HS256, la vista del depurador se verá un poco diferente:

<Frame>![Depuración de un JWT en JWT.io](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-hs256.png)</Frame>

En la captura de pantalla anterior, puedes ver que el token se firmó con el algoritmo **HS256**. El **Issuer** del token es **https://jerrie.auth0.com/** y el **Audience** es **https://hs256.test.api**.
