<div id="get-started">
  ## Primeros pasos
</div>

En este Quickstart se muestra cómo proteger endpoints de API de Fastify con tokens de acceso JWT. Crearás una API segura que valida tokens de acceso de Auth0 y otorga acceso a recursos protegidos.

<Steps>
  <Step title="Crear un nuevo proyecto" stepNumber={1}>
    Crea un directorio nuevo para tu API de Fastify e inicializa un proyecto de Node.js.

    ```shellscript
    mkdir auth0-fastify-api && cd auth0-fastify-api
    ```

    Inicializa el proyecto

    ```shellscript
    npm init -y
    ```

    Crear la estructura del proyecto

    ```shellscript
    touch server.js .env
    ```
  </Step>

  <Step title="Instala el SDK de la API de Auth0 para Fastify" stepNumber={2}>
    Instala las dependencias necesarias

    ```shellscript
    npm install @auth0/auth0-fastify-api fastify dotenv
    ```

    Añade scripts de inicio a tu `package.json`:

    ```json package.json
    {
      "name": "auth0-fastify-api",
      "version": "1.0.0",
      "type": "module",
      "main": "server.js",
      "scripts": {
        "start": "node server.js",
        "dev": "node --watch server.js"
      },
      "dependencies": {
        "@auth0/auth0-fastify-api": "^1.2.0",
        "dotenv": "^16.3.1",
        "fastify": "^5.0.0"
      }
    }
    ```
  </Step>

  <Step title="Configura tu API de Auth0" stepNumber={3}>
    A continuación, debes crear una nueva API en tu tenant de Auth0 y agregar las variables de entorno a tu proyecto.

    Tienes dos opciones para configurar tu API de Auth0: usar un comando de CLI o configurarla manualmente desde el Dashboard:

    <Tabs>
      <Tab title="CLI">
        Ejecuta el siguiente comando en el directorio raíz de tu proyecto para crear una API de Auth0:

        <CodeGroup>
          ```shellscript Mac
          # Instalar Auth0 CLI (si aún no está instalado)
          brew tap auth0/auth0-cli && brew install auth0

          # Crear Auth0 API
          auth0 apis create \
            --name "My Fastify API" \
            --identifier https://my-fastify-api.example.com
          ```

          ```powershell Windows
          # Instalar Auth0 CLI (si aún no está instalado)
          scoop bucket add auth0 https://github.com/auth0/scoop-auth0-cli.git
          scoop install auth0

          # Crear Auth0 API
          auth0 apis create `
            --name "My Fastify API" `
            --identifier https://my-fastify-api.example.com
          ```
        </CodeGroup>

        Después de crearla, copia los valores de **Identifier** y **Domain** y luego crea tu archivo `.env`:

        ```bash .env
        AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
        AUTH0_AUDIENCE=YOUR_API_IDENTIFIER
        ```

        <Note>
          Este comando hará lo siguiente:

          1. Comprobar si ya estás autenticado (y te pedirá iniciar sesión si hace falta)
          2. Crear una API de Auth0 con el identificador especificado
          3. Mostrar los detalles de la API, incluidos el dominio y el identificador
        </Note>
      </Tab>

      <Tab title="Dashboard">
        1. Ve al [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
        2. Ve a **Applications** → **APIs** → **Create API**
        3. Introduce un nombre para tu API (por ejemplo, &quot;My Fastify API&quot;)
        4. Define el **Identifier** (por ejemplo, `https://my-fastify-api.example.com`)
           * Este es el audience de tu API y debe tener un formato de URL válido
           * No tiene que ser una URL real; es solo un identificador
        5. Deja **Signing Algorithm** como **RS256**
        6. Haz clic en **Create**
        7. Copia el valor de **Identifier** de la pestaña **Settings**

        Crea tu archivo `.env` con los siguientes valores:

        ```bash .env
        AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
        AUTH0_AUDIENCE=YOUR_API_IDENTIFIER
        ```

        <Warning>
          Reemplaza `YOUR_AUTH0_DOMAIN` por el dominio de tu tenant de Auth0 (por ejemplo, `dev-abc123.us.auth0.com`) y `YOUR_API_IDENTIFIER` por el identificador de tu API del Dashboard (por ejemplo, `https://my-fastify-api.example.com`).
        </Warning>
      </Tab>
    </Tabs>

    <Tip>
      Verifica que tu archivo `.env` exista: `cat .env` (Mac/Linux) o `type .env` (Windows)
    </Tip>
  </Step>

  <Step title="Configurar el plugin de Auth0 API" stepNumber={4}>
    Crea el servidor Fastify y registra el plugin de la API de Auth0:

    ```javascript server.js {1-3,6-7,10-13,16-17} lines
    import 'dotenv/config';
    import Fastify from 'fastify';
    import fastifyAuth0Api from '@auth0/auth0-fastify-api';

    const fastify = Fastify({ logger: true });
    const port = process.env.PORT || 3001;

    // Registrar el plugin de la API de Auth0
    await fastify.register(fastifyAuth0Api, {
      domain: process.env.AUTH0_DOMAIN,
      audience: process.env.AUTH0_AUDIENCE,
    });

    // Iniciar el servidor
    fastify.listen({ port }, (err) => {
      if (err) {
        fastify.log.error(err);
        process.exit(1);
      }
      fastify.log.info(`API server running at http://localhost:${port}`);
    });
    ```

    **Qué hace esto:**

    * Registra el plugin de la API de Auth0 con tu dominio de Auth0 y la audiencia de la API
    * Configura la validación de JWT para las solicitudes entrantes
    * Pone a disposición el preHandler `requireAuth()` para proteger rutas
  </Step>

  <Step title="Crear rutas de API" stepNumber={5}>
    Añade rutas públicas y protegidas a tu `server.js`:

    ```javascript server.js expandable lines
    import 'dotenv/config';
    import Fastify from 'fastify';
    import fastifyAuth0Api from '@auth0/auth0-fastify-api';

    const fastify = Fastify({ logger: true });
    const port = process.env.PORT || 3001;

    // Registrar el plugin de la API de Auth0
    await fastify.register(fastifyAuth0Api, {
      domain: process.env.AUTH0_DOMAIN,
      audience: process.env.AUTH0_AUDIENCE,
    });

    // Ruta pública - no requiere autenticación
    fastify.get('/api/public', async (request, reply) => {
      return {
        message: 'Hello from a public endpoint! You don\'t need to be authenticated to see this.',
        timestamp: new Date().toISOString(),
      };
    });

    // Ruta protegida - requiere un token de acceso válido
    fastify.get('/api/private', {
      preHandler: fastify.requireAuth()
    }, async (request, reply) => {
      return {
        message: 'Hello from a protected endpoint! You successfully authenticated.',
        user: request.user.sub,
        timestamp: new Date().toISOString(),
      };
    });

    // Ruta protegida - devuelve información del usuario desde el token
    fastify.get('/api/profile', {
      preHandler: fastify.requireAuth()
    }, async (request, reply) => {
      return {
        message: 'Your user profile from the access token',
        profile: request.user,
      };
    });

    // Iniciar el servidor
    fastify.listen({ port }, (err) => {
      if (err) {
        fastify.log.error(err);
        process.exit(1);
      }
      fastify.log.info(`API server running at http://localhost:${port}`);
    });
    ```

    **Puntos clave:**

    * Las rutas públicas no requieren autenticación
    * Las rutas protegidas usan `preHandler: fastify.requireAuth()` para exigir un JWT válido
    * `request.user` contiene los claims del JWT decodificados para las solicitudes autenticadas
    * El claim `sub` contiene el identificador único del usuario
  </Step>

  <Step title="Ejecuta tu API" stepNumber={6}>
    Inicia el servidor de desarrollo:

    ```shellscript
    npm run dev
    ```

    Tu API ya está en ejecución en [http://localhost:3001](http://localhost:3001).

    <Info>
      El indicador `--watch` de Node.js 20+ reinicia automáticamente el servidor cuando cambian los archivos.
    </Info>
  </Step>

  <Step title="Prueba tu API" stepNumber={7}>
    Prueba el endpoint público (no requiere autenticación):

    ```bash
    curl http://localhost:3001/api/public
    ```

    Verás:

    ```json
    {
      "message": "Hello from a public endpoint! You don't need to be authenticated to see this.",
      "timestamp": "2024-01-15T10:30:00.000Z"
    }
    ```

    Prueba el endpoint protegido sin token (debería fallar):

    ```bash
    curl http://localhost:3001/api/private
    ```

    Deberías ver un error 401 Unauthorized:

    ```json
    {
      "error": "Unauthorized",
      "message": "No authorization token was found"
    }
    ```

    Para hacer una prueba con un token válido, necesitas:

    1. Crear una aplicación cliente (web o móvil) que autentique a los usuarios
    2. Configurar el cliente para solicitar un token de acceso para tu API (usando el parámetro audience)
    3. Usar ese token de acceso en el encabezado Authorization

    Ejemplo con un token:

    ```bash
    curl http://localhost:3001/api/private \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
    ```
  </Step>
</Steps>

<Check>
  **Checkpoint**

  Ahora deberías tener una API protegida. Tu API:

  1. Acepta solicitudes a endpoints públicos sin autenticación
  2. Rechaza solicitudes a endpoints protegidos sin un token válido
  3. Valida tokens JWT con tu dominio de Auth0 y tu audiencia
  4. Proporciona información del usuario a partir de los claims del token a través de `request.user`
</Check>

<Accordion title="Claims personalizados en tokens con TypeScript">
  Amplía la interfaz Token para añadir seguridad de tipos a los claims personalizados de tus tokens de acceso:

  ```typescript server.ts
  import '@auth0/auth0-fastify-api';

  // Amplía la interfaz Token con tus claims personalizados
  declare module '@auth0/auth0-fastify-api' {
    interface Token {
      sub: string;
      permissions?: string[];
      'https://myapp.com/roles'?: string[];
      email?: string;
      email_verified?: boolean;
    }
  }
  ```

  Ahora TypeScript reconocerá tus claims personalizados:

  ```typescript server.ts
  fastify.get('/api/profile', {
    preHandler: fastify.requireAuth()
  }, async (request, reply) => {
    // TypeScript reconoce estas propiedades
    const userRoles = request.user['https://myapp.com/roles']; // string[] | undefined
    const permissions = request.user.permissions; // string[] | undefined
    const email = request.user.email; // string | undefined

    return {
      userId: request.user.sub,
      roles: userRoles || [],
      permissions: permissions || [],
      email: email,
    };
  });
  ```

  <Note>
    Los claims personalizados deben usar URL con espacio de nombres (por ejemplo, `https://myapp.com/roles`), a menos que sean claims estándar de OIDC. [Más información sobre los claims personalizados](https://auth0.com/docs/secure/tokens/json-web-tokens/create-custom-claims).
  </Note>
</Accordion>

<Accordion title="Autorización basada en permisos">
  Comprueba permisos específicos en el token de acceso:

  ```javascript server.js
  // Middleware para comprobar un permiso específico
  function requirePermission(permission) {
    return async (request, reply) => {
      const permissions = request.user.permissions || [];

      if (!permissions.includes(permission)) {
        return reply.status(403).send({
          error: 'Forbidden',
          message: `Missing required permission: ${permission}`
        });
      }
    };
  }

  // Ruta que requiere el permiso 'read:messages'
  fastify.get('/api/messages', {
    preHandler: [
      fastify.requireAuth(),
      requirePermission('read:messages')
    ]
  }, async (request, reply) => {
    return {
      messages: ['Message 1', 'Message 2', 'Message 3']
    };
  });

  // Ruta que requiere el permiso 'write:messages'
  fastify.post('/api/messages', {
    preHandler: [
      fastify.requireAuth(),
      requirePermission('write:messages')
    ]
  }, async (request, reply) => {
    return {
      message: 'Message created successfully',
      id: 'msg_123'
    };
  });
  ```

  <Note>
    Los permisos deben configurarse en la configuración de tu Auth0 API y concederse a los clientes. [Más información sobre los permisos de API](https://auth0.com/docs/manage-users/access-control/configure-core-rbac/rbac-users/assign-permissions-users).
  </Note>
</Accordion>

<Accordion title="Autorización basada en roles">
  Implementa el control de acceso basado en roles mediante claims personalizados:

  ```javascript server.js
  // Middleware para comprobar un rol específico
  function requireRole(role) {
    return async (request, reply) => {
      const roles = request.user['https://myapp.com/roles'] || [];

      if (!roles.includes(role)) {
        return reply.status(403).send({
          error: 'Forbidden',
          message: `Missing required role: ${role}`
        });
      }
    };
  }

  // Ruta solo para administradores
  fastify.get('/api/admin/users', {
    preHandler: [
      fastify.requireAuth(),
      requireRole('admin')
    ]
  }, async (request, reply) => {
    return {
      users: [
        { id: 1, name: 'User 1' },
        { id: 2, name: 'User 2' }
      ]
    };
  });

  // Ruta para gerente o administrador
  function requireAnyRole(...roles) {
    return async (request, reply) => {
      const userRoles = request.user['https://myapp.com/roles'] || [];
      const hasRole = roles.some(role => userRoles.includes(role));

      if (!hasRole) {
        return reply.status(403).send({
          error: 'Forbidden',
          message: `Missing required role. Need one of: ${roles.join(', ')}`
        });
      }
    };
  }

  fastify.get('/api/reports', {
    preHandler: [
      fastify.requireAuth(),
      requireAnyRole('admin', 'manager')
    ]
  }, async (request, reply) => {
    return { reports: [] };
  });
  ```

  <Note>
    Los roles deben añadirse a los tokens mediante Auth0 Actions. [Aprende a añadir roles a los tokens](https://auth0.com/docs/customize/actions/flows-and-triggers/login-flow/add-user-roles-to-id-and-access-tokens).
  </Note>
</Accordion>

<Accordion title="Configuración de CORS">

  ```javascript server.js
  await fastify.register(cors, {
    origin: ['http://localhost:3000', 'http://localhost:5173'], // URL de tu aplicación web
    credentials: true,
  });
  ```

  En producción, especifica los orígenes exactos:

  ```javascript server.js
  await fastify.register(cors, {
    origin: [
      'https://myapp.com',
      'https://www.myapp.com'
    ],
    credentials: true,
    methods: ['GET', 'POST', 'PUT', 'DELETE'],
  });
  ```
</Accordion>

<Accordion title="Manejo de errores">
  Añade un manejo integral de errores de autenticación:

  ```javascript server.js
  // Manejador de errores personalizado
  fastify.setErrorHandler((error, request, reply) => {
    fastify.log.error(error);

    // Manejar errores de validación de JWT
    if (error.statusCode === 401) {
      return reply.status(401).send({
        error: 'Unauthorized',
        message: error.message || 'Invalid or missing access token',
        code: 'UNAUTHORIZED'
      });
    }

    // Manejar errores de permiso/rol
    if (error.statusCode === 403) {
      return reply.status(403).send({
        error: 'Forbidden',
        message: error.message || 'Insufficient permissions',
        code: 'FORBIDDEN'
      });
    }

    // Manejar otros errores
    return reply.status(error.statusCode || 500).send({
      error: 'Internal Server Error',
      message: 'An unexpected error occurred',
      code: 'INTERNAL_ERROR'
    });
  });

  // Manejador para rutas no encontradas
  fastify.setNotFoundHandler((request, reply) => {
    return reply.status(404).send({
      error: 'Not Found',
      message: `Route ${request.method} ${request.url} not found`,
      code: 'NOT_FOUND'
    });
  });
  ```
</Accordion>

<Accordion title="Limitación de frecuencia">
  Protege tu API del uso abusivo con la limitación de frecuencia:

  ```bash
  npm install @fastify/rate-limit
  ```

  ```javascript server.js
  import rateLimit from '@fastify/rate-limit';

  await fastify.register(rateLimit, {
    max: 100, // Número máximo de solicitudes
    timeWindow: '1 minute', // Ventana de tiempo
    errorResponseBuilder: (request, context) => {
      return {
        error: 'Too Many Requests',
        message: `Rate limit exceeded. Try again in ${context.after}`,
        retryAfter: context.after
      };
    }
  });

  // Aplica límites más estrictos a rutas específicas
  fastify.get('/api/expensive-operation', {
    preHandler: fastify.requireAuth(),
    config: {
      rateLimit: {
        max: 10,
        timeWindow: '1 minute'
      }
    }
  }, async (request, reply) => {
    return { result: 'expensive operation result' };
  });
  ```
</Accordion>

<div id="troubleshooting">
  ## Resolución de problemas
</div>

<AccordionGroup>
  <Accordion title="Problemas comunes y soluciones">
    ### &quot;No se encontró ningún token de autorización&quot;

    **Problema:** La API no puede encontrar el token de acceso en la solicitud.

    **Soluciones:**

    1. Asegúrate de que el encabezado `Authorization` esté presente: `Authorization: Bearer YOUR_TOKEN`
    2. Comprueba que &quot;Bearer&quot; esté incluido antes del token
    3. Verifica que el token no haya expirado

    ### &quot;Token no válido&quot; o &quot;jwt malformado&quot;

    **Problema:** El formato del token no es válido.

    **Soluciones:**

    1. Asegúrate de que estás usando un **token de acceso**, no un token de identidad
    2. El token debe obtenerse con el parámetro `audience` de tu API
    3. Comprueba que el token sea un JWT válido (debe tener tres partes separadas por puntos)

    ### &quot;Firma no válida&quot;

    **Problema:** La firma del token no coincide.

    **Soluciones:**

    1. Verifica que `AUTH0_DOMAIN` coincida con el dominio que emitió el token
    2. Asegúrate de que estás usando el algoritmo de firma RS256 (predeterminado)
    3. Comprueba que el token no haya sido modificado

    ### &quot;Audiencia no válida&quot;

    **Problema:** La audiencia del token no coincide con tu API.

    **Solución:** La aplicación cliente debe solicitar un token con la audiencia correcta:

    ```javascript
    // En tu aplicación cliente
    const token = await getAccessTokenSilently({
      authorizationParams: {
        audience: 'https://my-fastify-api.example.com' // Debe coincidir con el identificador de tu API
      }
    });
    ```

    ### Errores de CORS en el navegador

    **Problema:** El navegador bloquea las solicitudes a la API debido a la política de CORS.

    **Solución:** Instala y configura `@fastify/cors`:

    ```bash
    npm install @fastify/cors
    ```

    ```javascript
    import cors from '@fastify/cors';

    await fastify.register(cors, {
      origin: 'http://localhost:3000', // La URL de tu frontend
      credentials: true
    });
    ```
  </Accordion>
</AccordionGroup>
