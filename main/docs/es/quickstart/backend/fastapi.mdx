    from fastapi_plugin.fast_api_client import Auth0FastAPI
    import os
    from dotenv import load_dotenv

    # Cargar variables de entorno
    load_dotenv()

    app = FastAPI()

    # Inicializar Auth0
    auth0 = Auth0FastAPI(
        domain=os.environ.get("AUTH0_DOMAIN"),
        audience=os.environ.get("AUTH0_AUDIENCE")
    )

    # Ruta pública - no requiere autenticación
    @app.get("/api/public")
    async def public():
        return {
            "message": "Hello from a public endpoint! You don't need to be authenticated to see this."
        }

<Steps>
  <Step title="Crear un nuevo proyecto de FastAPI" stepNumber={1}>
    Crea un nuevo directorio para tu proyecto de FastAPI y configura un entorno virtual.

    ```bash
    mkdir my-fastapi-api
    cd my-fastapi-api
    python3 -m venv venv
    source venv/bin/activate  # Windows: venv\Scripts\activate
    ```
  </Step>

  <Step title="Instala las dependencias" stepNumber={2}>
    Crea un archivo `requirements.txt` con las siguientes dependencias:

    ```txt requirements.txt
    fastapi>=0.115.0
    uvicorn[standard]>=0.34.0
    auth0-fastapi-api>=1.0.0b5
    python-dotenv>=1.0.0
    ```

    Instala las dependencias:

    ```bash
    pip install -r requirements.txt
    ```
  </Step>

  <Step title="Configura tu API de Auth0" stepNumber={3}>
    Tendrás que crear una API en Auth0 para representar tu aplicación FastAPI.

    <Tabs>
      <Tab title="Dashboard">
        1. Ve a [Applications &gt; APIs](https://manage.auth0.com/#/apis) en el Auth0 Dashboard
        2. Haz clic en **Create API**
        3. Introduce un **Name** para tu API (p. ej., &quot;My FastAPI API&quot;)
        4. Establece el **Identifier** con el identificador de tu API (p. ej., `https://my-fastapi-api`)
        5. Deja el **Signing Algorithm** como RS256
        6. Haz clic en **Create**
      </Tab>

      <Tab title="CLI">
        Si tienes instalado [Auth0 CLI](https://github.com/auth0/auth0-cli), puedes crear una API y configurar tu entorno desde la terminal:

        <CodeGroup>
          ```shellscript Mac
          AUTH0_API_NAME="My FastAPI API" && AUTH0_API_IDENTIFIER="https://my-fastapi-api" && brew tap auth0/auth0-cli && brew install auth0 && auth0 login --no-input && auth0 apis create --name "${AUTH0_API_NAME}" --identifier "${AUTH0_API_IDENTIFIER}" --signing-alg RS256 --no-input && echo "AUTH0_DOMAIN=$(auth0 tenants list --json | jq -r '.[] | select(.active == true) | .name')\nAUTH0_AUDIENCE=${AUTH0_API_IDENTIFIER}" > .env
          ```

          ```powershell Windows
          $ApiName = "My FastAPI API"; $ApiIdentifier = "https://my-fastapi-api"; auth0 login --no-input; auth0 apis create -n $ApiName -i $ApiIdentifier --signing-alg RS256 --no-input; $ActiveTenant = (auth0 tenants list --json | ConvertFrom-Json | Where-Object { $_.active -eq $true }).name; "AUTH0_DOMAIN=$ActiveTenant`nAUTH0_AUDIENCE=$ApiIdentifier" | Out-File -FilePath .env -Encoding utf8
          ```
        </CodeGroup>

        Esto creará tu API y generará automáticamente un archivo `.env` con tu configuración de Auth0.
      </Tab>
    </Tabs>

    <Info>
      El **Identifier** es un identificador único para tu API. Se recomienda usar una URL, pero no tiene por qué ser una URL de acceso público: Auth0 no hará llamadas a ella. Este valor no se puede modificar después.
    </Info>

    Toma nota de los valores de **Domain** e **Identifier** (Audience). Los necesitarás en el siguiente paso.
  </Step>

  <Step title="Definir permisos de la API" stepNumber={4}>
    Los permisos (también conocidos como scopes) te permiten definir cómo se puede acceder a tu API. Puedes crear permisos para tu API en el Auth0 Dashboard.

    1. En el Auth0 Dashboard, ve a la pestaña **Permissions** de tu API
    2. Agrega los siguientes permisos:
       * `read:messages` con la descripción &quot;Leer mensajes&quot;
       * `write:messages` con la descripción &quot;Escribir mensajes&quot;

    Estos permisos se usarán para controlar el acceso a endpoints específicos de tu API.
  </Step>

  <Step title="Configurar el cliente de Auth0" stepNumber={5}>
    Crea un archivo `.env` en la raíz de tu proyecto para almacenar la configuración de Auth0:

    ```txt .env
    AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
    AUTH0_AUDIENCE=YOUR_API_IDENTIFIER
    ```

    Sustituye `YOUR_AUTH0_DOMAIN` por tu dominio de Auth0 (p. ej., `dev-abc123.us.auth0.com`) y `YOUR_API_IDENTIFIER` por el identificador que definiste al crear tu API.

    <Warning>
      Nunca subas tu archivo `.env` al control de versiones. Añádelo a tu archivo `.gitignore` para mantener seguras tus credenciales.
    </Warning>

    Ahora crea un archivo `app.py` e inicializa tu aplicación FastAPI con Auth0:

    ```python app.py
    from fastapi import FastAPI, Depends
    from fastapi_plugin.fast_api_client import Auth0FastAPI
    import os
    from dotenv import load_dotenv

    # Cargar variables de entorno
    load_dotenv()

    app = FastAPI()

    # Inicializar Auth0
    auth0 = Auth0FastAPI(
        domain=os.environ.get("AUTH0_DOMAIN"),
        audience=os.environ.get("AUTH0_AUDIENCE")
    )
    ```
  </Step>

  <Step title="Crear rutas protegidas" stepNumber={6}>
    Añade las siguientes rutas a tu archivo `app.py`. Estas rutas muestran distintos niveles de control de acceso:

    ```python app.py
    from fastapi import FastAPI, Depends
    from fastapi_plugin.fast_api_client import Auth0FastAPI
    import os
    from dotenv import load_dotenv

    load_dotenv()

    app = FastAPI()

    auth0 = Auth0FastAPI(
        domain=os.environ.get("AUTH0_DOMAIN"),
        audience=os.environ.get("AUTH0_AUDIENCE")
    )

    # Ruta pública - no requiere autenticación
    @app.get("/api/public")
    async def public():
        return {
            "message": "Hello from a public endpoint! You don't need to be authenticated to see this."
        }

    # Ruta protegida - requiere autenticación
    @app.get("/api/private")
    async def private(claims: dict = Depends(auth0.require_auth())):
        return {
            "message": "Hello from a private endpoint! You need to be authenticated to see this.",
            "user_id": claims.get("sub")
        }

    # Ruta con scope - requiere un permiso específico
    @app.get("/api/private-scoped")
    async def private_scoped(claims: dict = Depends(auth0.require_auth(scopes="read:messages"))):
        return {
            "message": "Hello from a private endpoint! You need to be authenticated and have a scope of read:messages to see this.",
            "user_id": claims.get("sub")
        }
    ```

    El método `require_auth()` valida el token de acceso enviado en el encabezado Authorization. Cuando se llama con el parámetro `scopes`, también verifica que el token contenga el permiso especificado.
  </Step>

  <Step title="Ejecuta tu API" stepNumber={7}>
    Inicia tu aplicación FastAPI:

    ```bash
    uvicorn app:app --reload
    ```

    Tu API ya está en ejecución en `http://localhost:8000`.

    <Check>
      Visita `http://localhost:8000/api/public` en tu navegador. Deberías ver el mensaje público sin necesidad de autenticación.
    </Check>
  </Step>
</Steps>

<div id="call-your-api">
  ### Llama a tu API
</div>

Usa el token de acceso para llamar a tu endpoint protegido:

```bash
curl -X GET http://localhost:8000/api/private -H 'authorization: Bearer YOUR_ACCESS_TOKEN'
```

Deberías recibir una respuesta con el mensaje privado y tu ID de usuario.

Para probar el endpoint con scope limitado, asegúrate de que tu token incluya el scope `read:messages`:

```bash
curl -X GET http://localhost:8000/api/private-scoped -H 'authorization: Bearer YOUR_ACCESS_TOKEN'
```

Si tu token no tiene el scope requerido, recibirás una respuesta 403 Forbidden.

<Accordion title="Validar claims personalizadas">
  Puedes acceder a las claims personalizadas que se hayan agregado al token de acceso mediante [Auth0 Actions](https://auth0.com/docs/customize/actions).

  Accede a las claims personalizadas en el controlador de la ruta:

  ```python
  @app.get("/api/profile")
  async def profile(claims: dict = Depends(auth0.require_auth())):
      return {
          "user_id": claims.get("sub"),
          "email": claims.get("email"),
          "permissions": claims.get("permissions", []),
          "custom_claim": claims.get("https://myapp.example.com/custom_claim")
      }
  ```

  Para agregar claims personalizadas a tus tokens de acceso, crea una Auth0 Action:

  1. Ve a **Actions &gt; Library** en el Auth0 Dashboard
  2. Haz clic en **Create Action**
  3. Selecciona **Build from scratch**
  4. Asigna un nombre a tu Action y selecciona el trigger **Login / Post Login**
  5. Agrega tus claims personalizadas:

  ```javascript
  exports.onExecutePostLogin = async (event, api) => {
    const namespace = 'https://myapp.example.com';

    if (event.authorization) {
      // Agregar claim personalizada al token de acceso
      api.accessToken.setCustomClaim(`${namespace}/roles`, event.user.app_metadata.roles || []);
    }
  };
  ```

  6. Haz clic en **Deploy** y agrega la Action a tu Login flow

  <Info>
    Las claims personalizadas deben usar un formato con espacio de nombres (por ejemplo, `https://myapp.example.com/claim_name`) para evitar conflictos con las claims estándar.
  </Info>
</Accordion>

<Accordion title="Proteger endpoints sin usar claims">
  Si necesitas proteger un endpoint pero no necesitas acceder a las claims, puedes usar el parámetro `dependencies`:

  ```python
  @app.get("/api/protected", dependencies=[Depends(auth0.require_auth())])
  async def protected():
      return {"message": "This endpoint is protected"}
  ```

  Esto valida el token de acceso, pero no inyecta las claims en tu función.
</Accordion>

<Accordion title="Soporte para DPoP">
  <Info>
    DPoP (Demonstrating Proof-of-Possession) se encuentra actualmente en [acceso anticipado](https://auth0.com/docs/troubleshoot/product-lifecycle/product-release-stages). Ponte en contacto con el [equipo de soporte de Auth0](https://support.auth0.com/) para habilitarlo en tu tenant.
  </Info>

  DPoP proporciona mayor seguridad al vincular criptográficamente los tokens de acceso al cliente que los solicitó. Esto evita el robo de tokens y los ataques de repetición.

  El SDK habilita la compatibilidad con DPoP de forma predeterminada. Puedes configurar el comportamiento de DPoP:

  ```python
  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE"),
      dpop_enabled=True,      # Habilitar DPoP (predeterminado: True)
      dpop_required=False     # Requerir DPoP (predeterminado: False)
  )
  ```

  **Modo mixto** (predeterminado) acepta tanto tokens Bearer como DPoP:

  ```python
  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE"),
      dpop_enabled=True,
      dpop_required=False
  )
  ```

  **Modo solo DPoP** rechaza los tokens Bearer:

  ```python
  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE"),
      dpop_required=True
  )
  ```

  Al usar DPoP, los clientes deben incluir tanto el encabezado `Authorization: DPoP <token>` como el encabezado `DPoP: <proof>`. El SDK valida automáticamente la prueba DPoP y la vincula al token de acceso.
</Accordion>

<Accordion title="Configurar para proxy inverso">
  <Warning>
    Solo habilita `trust_proxy` cuando tu aplicación esté detrás de un proxy inverso de confianza. Nunca lo habilites para aplicaciones expuestas directamente a internet.
  </Warning>

  Si tu aplicación se ejecuta detrás de un proxy inverso (nginx, AWS ALB, etc.), debes habilitar la confianza en el proxy para que la validación de DPoP funcione correctamente:

  ```python
  from fastapi import FastAPI

  app = FastAPI()

  # Habilitar confianza en el proxy
  app.state.trust_proxy = True

  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE")
  )
  ```

  Configura tu proxy inverso para reenviar los encabezados necesarios:

  ```nginx
  location /api {
      proxy_pass http://localhost:8000;
      proxy_set_header X-Forwarded-Proto $scheme;
      proxy_set_header X-Forwarded-Host $host;
      proxy_set_header X-Forwarded-Prefix /api;
  }
  ```

  Esto es esencial para la validación de DPoP porque el SDK necesita hacer coincidir la URL exacta que usó el cliente. Sin confianza en el proxy, tu aplicación ve URLs internas mientras que las pruebas DPoP hacen referencia a URLs externas, lo que provoca errores de validación.
</Accordion>

<Accordion title="Manejo de errores">
  El SDK lanza `HTTPException` para los errores de autenticación. FastAPI los gestiona automáticamente y devuelve al cliente las respuestas HTTP correspondientes.

  Puedes implementar un manejo de errores personalizado si es necesario:

  ```python
  from fastapi import Request
  from fastapi.responses import JSONResponse
  from fastapi.exceptions import HTTPException

  @app.exception_handler(HTTPException)
  async def custom_http_exception_handler(request: Request, exc: HTTPException):
      if exc.status_code in [401, 403]:
          return JSONResponse(
              status_code=exc.status_code,
              content={
                  "error": "authentication_failed",
                  "message": "You must be authenticated to access this resource",
                  "details": exc.detail
              }
          )
      return JSONResponse(
          status_code=exc.status_code,
          content={"detail": exc.detail}
      )
  ```

  Los errores de autenticación incluyen:

  * **401 Unauthorized**: Falta el token de acceso, no es válido o ha caducado
  * **403 Forbidden**: El token es válido, pero los permisos (`scopes`) son insuficientes
</Accordion>

<div id="common-issues">
  ## Problemas comunes
</div>

<AccordionGroup>
  <Accordion title="401 Unauthorized - audiencia no válida">
    **Problema**: La validación del token falla con el error &quot;audiencia no válida&quot;.

    **Solución**: Verifica que `AUTH0_AUDIENCE` en tu archivo `.env` coincida exactamente con el **Identifier** que configuraste para tu API en el Auth0 Dashboard.

    1. Abre el Auth0 Dashboard y ve a **Applications &gt; APIs**
    2. Selecciona tu API
    3. Revisa el valor de **Identifier** en la pestaña Settings
    4. Actualiza tu archivo `.env`:
       ```txt
       AUTH0_AUDIENCE=https://your-exact-api-identifier
       ```
    5. Reinicia tu aplicación
  </Accordion>

  <Accordion title="401 Unauthorized - emisor no válido">
    **Problema**: La validación del token falla con el error &quot;emisor no válido&quot;.

    **Solución**: Verifica que `AUTH0_DOMAIN` sea correcto y no incluya el protocolo `https://`.

    Tu dominio debería verse así: `dev-abc123.us.auth0.com`, no `https://dev-abc123.us.auth0.com`.

    Actualiza tu archivo `.env`:

    ```txt
    AUTH0_DOMAIN=dev-abc123.us.auth0.com
    ```
  </Accordion>

  <Accordion title="403 Forbidden - scope insuficiente">
    **Problema**: El endpoint protegido devuelve un error 403 incluso con un token de acceso válido.

    **Solución**: El token de acceso no incluye el scope requerido.

    1. Revisa qué scopes requiere tu endpoint
    2. Al solicitar un token, asegúrate de incluir los scopes requeridos
    3. Verifica que el scope exista en la pestaña Permissions de tu API en el Auth0 Dashboard
    4. Decodifica tu token en [jwt.io](https://jwt.io) para verificar que contiene el claim `scope` con los valores requeridos
  </Accordion>

  <Accordion title="ModuleNotFoundError: No module named 'fastapi_plugin'">
    **Problema**: Python no puede encontrar el Auth0 FastAPI SDK.

    **Solución**: Asegúrate de que el SDK esté instalado en tu entorno virtual activo.

    ```bash
    # Activa tu entorno virtual
    source venv/bin/activate  # Windows: venv\Scripts\activate

    # Instala el SDK
    pip install auth0-fastapi-api

    # Verifica la instalación
    pip show auth0-fastapi-api
    ```
  </Accordion>

  <Accordion title="No se puede conectar a Auth0 (error al obtener JWKS)">
    **Problema**: La aplicación no puede obtener las claves de firma desde Auth0.

    **Solución**: Revisa la conectividad de red y la configuración del dominio.

    1. Verifica que tu dominio sea accesible:
       ```bash
       curl https://YOUR_DOMAIN/.well-known/openid-configuration
       ```

    2. Comprueba que tu firewall permita conexiones HTTPS salientes (puerto 443) a `*.auth0.com`

    3. Si estás detrás de un proxy corporativo, configura las variables de entorno `HTTP_PROXY` y `HTTPS_PROXY`
  </Accordion>

  <Accordion title="Falla la validación de DPoP">
    **Problema**: La autenticación DPoP devuelve errores relacionados con la URL o con la validación de la prueba DPoP.

    **Solución**:

    1. Si estás detrás de un proxy inverso, habilita la confianza en el proxy:
       ```python
       app.state.trust_proxy = True
       ```

    2. Verifica que tu proxy reenvíe estos headers:
       * `X-Forwarded-Proto`
       * `X-Forwarded-Host`
       * `X-Forwarded-Prefix`

    3. Asegúrate de que DPoP esté habilitado para tu tenant (contacta al equipo de soporte de Auth0)

    4. Verifica que el claim `htu` de la prueba DPoP coincida exactamente con la URL de tu solicitud
  </Accordion>
</AccordionGroup>
