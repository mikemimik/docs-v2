<Recipe>
  <Content>
    [Auth0&#39;s Laravel SDK](https://github.com/auth0/laravel-auth0)
    te permite agregar rápidamente autorización basada en tokens y control de acceso a rutas en tu aplicación de Laravel. Esta guía
    muestra cómo integrar Auth0 con una aplicación de [Laravel 9
    o 10](https://github.com/auth0/laravel-auth0#support-policy) nueva (o existente).

    **Las aplicaciones de backend se diferencian de las aplicaciones web tradicionales en que no gestionan la autenticación de usuario ni cuentan con una interfaz de usuario. Proporcionan una API con la que otras aplicaciones pueden interactuar. Aceptan **[tokens de acceso](https://auth0.com/docs/secure/tokens/access-tokens)** desde los encabezados **`Authorization`** de las solicitudes para controlar el acceso a las rutas.**

    Por lo general, se crean aplicaciones de front-end independientes para interactuar con este tipo de backends. Estas pueden ser de cualquier tipo,
    desde [aplicaciones de página única](https://auth0.com/docs/quickstart/spa) hasta [aplicaciones nativas o móviles](https://auth0.com/docs/quickstart/native) (¡para todas ellas Auth0 también
    ofrece SDKs!)

    Cuando los usuarios necesitan interactuar con tu aplicación de backend, primero se autentican con Auth0 a través de la aplicación de frontend. Luego, la aplicación de frontend obtiene un token de acceso de Auth0, que puede usar para realizar solicitudes a tu aplicación de backend en nombre del usuario.

    Como su nombre indica, los [tokens de
    acceso](https://auth0.com/docs/secure/tokens/access-tokens) están diseñados para gestionar cuestiones de control de acceso (autorización) y no contienen información sobre
    el usuario. **Las aplicaciones de backend trabajan exclusivamente con tokens de acceso.** Puedes obtener información sobre el
    usuario que creó el token mediante la [API de Management
    ](https://auth0.com/docs/api/management/v2), como veremos más adelante.

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      **Si aún no tienes configurada una aplicación de Laravel**, abre un shell en un directorio adecuado para un nuevo
      proyecto y ejecuta el siguiente comando:

      ```bash lines wrap
      composer create-project --prefer-dist laravel/laravel auth0-laravel-api ^9.0
      ```

      Todos los comandos de esta guía suponen que los estás ejecutando desde la raíz del directorio de tu proyecto de Laravel, por lo que
      debes ir con `cd` al directorio del nuevo proyecto:

      ```bash lines
      cd auth0-laravel-api
      ```
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Ejecuta el siguiente comando en el directorio de tu proyecto para instalar el [SDK de Auth0 para Laravel](https://github.com/auth0/laravel-auth0):

      ```bash lines
      composer require auth0/login:^7.8 --update-with-all-dependencies
      ```

      A continuación, genera un archivo de configuración del SDK para tu aplicación:

      ```bash lines
      php artisan vendor:publish --tag auth0
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Ejecuta el siguiente comando en el directorio de tu proyecto para descargar la [Auth0 CLI](https://github.com/auth0/auth0-cli):

      ```bash lines wrap
      curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
      ```

      Luego, autentica el CLI con tu cuenta de Auth0 y elige &quot;como usuario&quot; cuando se te pida:

      ```bash lines
      ./auth0 login
      ```

      A continuación, crea una aplicación nueva con Auth0:

      ```bash lines
      ./auth0 apps create \
        --name "My Laravel Backend" \
        --type "regular" \
        --auth-method "post" \
        --callbacks "http://localhost:8000/callback" \
        --logout-urls "http://localhost:8000" \
        --reveal-secrets \
        --no-input \
        --json > .auth0.app.json
      ```

      También deberías crear una API nueva:

      ```bash lines
      ./auth0 apis create \
        --name "My Laravel Backend API" \
        --identifier "https://github.com/auth0/laravel-auth0" \
        --offline-access \
        --no-input \
        --json > .auth0.api.json
      ```

      Esto genera dos archivos en el directorio de tu proyecto que configuran el SDK.

      Como estos archivos contienen credenciales, es importante tratarlos como información confidencial. Debes asegurarte de no
      incluirlos en el control de versiones. Si usas Git, debes agregarlos a tu archivo `.gitignore`:

      ```bash lines
      echo ".auth0.*.json" >> .gitignore
      ```
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      El SDK registra automáticamente su `guard` de autorización en tu aplicación Laravel para usarlo con el
      middleware `api`, que Laravel aplica de forma predeterminada a todas las rutas del archivo
      `routes/api.php` de tu aplicación.

      <Warning>
        Para que el SDK funcione como se espera sin configuración adicional, **debes definir tus rutas en
        el archivo **`routes/api.php`**.**
      </Warning>

      Puedes usar el `guard` de autorización del SDK de Auth0 para restringir el acceso a las rutas de tu aplicación.

      Para rechazar solicitudes que no contengan un token de acceso válido en el encabezado `Authorization`, puedes usar
      el middleware `auth` de Laravel:

      ```php lines
      Route::get('/private', function () {
      return response()->json([
      'message' => 'Your token is valid; you are authorized.',
      ]);
      })->middleware('auth');
      ```

      También puedes exigir que el token proporcionado tenga [permisos](https://auth0.com/docs/manage-users/access-control/rbac) específicos combinando esto
      con el middleware `can` de Laravel:

      ```php lines
      Route::get('/scope', function () {
      return response()->json([
      'message' => 'Your token is valid and has the read:messages permission; you are authorized.',
      ]);
      })->middleware('auth')->can('read:messages');
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      La información sobre el token de acceso proporcionado está disponible mediante la fachada `Auth` de Laravel o la
      función auxiliar `auth()`.

      Por ejemplo, para obtener el identificador y la dirección de correo electrónico del usuario:

      ```php lines
      Route::get('/', function () {
      if (! auth()->check()) {
      return response()->json([
      'message' => 'You did not provide a valid token.',
      ]);
      }
      return response()->json([
      'message' => 'Your token is valid; you are authorized.',
      'id' => auth()->id(),
      'token' => auth()?->user()?->getAttributes(),
      ]);
      });
      ```
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Puedes obtener información sobre el usuario que creó el token de acceso de Auth0 mediante la [Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). El SDK proporciona una práctica capa de abstracción para esta API,
      accesible a través del método `management()` del SDK.

      **Antes de realizar llamadas a la Management API, debes habilitar tu aplicación para que se comunique con la Management
      API.** Esto puede hacerse desde la [página de API del Auth0 Dashboard](https://manage.auth0.com/#/apis/), eligiendo `Auth0 Management API` y
      seleccionando la pestaña &#39;Machine to Machine Applications&#39;. Autoriza tu aplicación Laravel y, a continuación, haz clic en la flecha
      desplegable para elegir los scopes que quieras conceder.

      Para el siguiente ejemplo, debes conceder el scope `read:users`. Puedes encontrar una lista de endpoints de la API y los
      scopes requeridos en [la documentación de la Management
      API](https://auth0.com/docs/api/management/v2).

      ```php lines
      use Auth0\Laravel\Facade\Auth0;
      Route::get('/me', function () {
      $user = auth()->id();
      $profile = cache()->get($user);
      if (null === $profile) {
      $endpoint = Auth0::management()->users();
      $profile = $endpoint->get($user);
      $profile = Auth0::json($profile);
      cache()->put($user, $profile, 120);
      }
      $name = $profile['name'] ?? 'Unknown';
      $email = $profile['email'] ?? 'Unknown';
      return response()->json([
      'name' => $name,
      'email' => $email,
      ]);
      })->middleware('auth');
      ```

      <Info>
        **Debes almacenar en caché la información del usuario en tu aplicación durante periodos breves.** Esto reduce la cantidad
        de solicitudes que tu aplicación envía a Auth0 y mejora el rendimiento. Debes evitar almacenar la información del usuario
        en tu aplicación durante periodos prolongados, ya que esto puede dar lugar a datos desactualizados. Tampoco debes evitar
        almacenar información del usuario más allá de su identificador en bases de datos persistentes.
      </Info>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Ya está listo para iniciar su aplicación Laravel, de modo que pueda aceptar peticiones:

      ```bash lines
      php artisan serve
      ```
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Puedes obtener más información sobre [cómo obtener tokens de acceso aquí](https://auth0.com/docs/secure/tokens/access-tokens/get-access-tokens). Sin embargo, para este Quickstart, puedes usar simplemente un token de acceso
      de [la vista &quot;test&quot; de la configuración de tu API&#39;
      ](https://manage.auth0.com/#/apis).

      <Info>
        La route `/me` que creamos arriba no funcionará con un token de prueba, ya que no hay ningún usuario real
        asociado a este.
      </Info>

      <Note>
        ##### Checkpoint

        Abre una shell e intenta enviar solicitudes a tu aplicación.

        Empieza solicitando la route pública:

        `curl --request GET \ --url http://localhost:8000/api \ --header 'Accept: application/json'`

        A continuación, usa tu token de acceso en el encabezado `Authorization` para solicitar una route protegida:

        `curl --request GET \ --url http://localhost:8000/api/private \ --header 'Accept: application/json' \ --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'`

        Por último, intenta solicitar la route protegida por scope, que solo funcionará si al token de acceso se le ha
        concedido el scope `read:messages`:

        `curl --request GET \ --url http://localhost:8000/api/scope \ --header 'Accept: application/json' \ --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'`
      </Note>
