<div id="by-evan-sims">
  ##### Por Evan Sims
</div>

El SDK de Auth0 para Laravel le permite agregar rápidamente autorización basada en tokens y control de acceso a rutas en su aplicación Laravel. Esta guía muestra cómo integrar Auth0 con una aplicación Laravel 9 o 10, ya sea nueva o existente. Le recomendamos que inicie sesión para seguir este Quickstart con ejemplos configurados para su cuenta.

{/* <Card title="Ver en Github" href="https://github.com/auth0-samples/laravel/tree/7.x/sample" icon="github">
  Requisitos del sistema: Laravel 9 / 10 | PHP 8.0+ | Composer
  </Card> */}

**Las aplicaciones de backend se diferencian de las aplicaciones web tradicionales en que no se encargan de la autenticación de usuario ni tienen una interfaz de usuario. Proporcionan una API con la que otras aplicaciones pueden interactuar. Aceptan [tokens de acceso](https://auth0.com/docs/secure/tokens/access-tokens) desde el encabezado `Authorization` de las solicitudes para controlar el acceso a las rutas.**

Normalmente, se crean aplicaciones de frontend independientes para interactuar con este tipo de backends. Pueden ser desde [aplicaciones de página única](https://auth0.com/docs/quickstart/spa) hasta [aplicaciones nativas o móviles](https://auth0.com/docs/quickstart/native) (¡y Auth0 también proporciona SDKs para todas ellas!)

Cuando los usuarios necesitan interactuar con tu aplicación de backend, primero se autentican con Auth0 mediante la aplicación de frontend. Después, la aplicación de frontend obtiene un token de acceso de Auth0, que puede usar para hacer solicitudes a tu aplicación de backend en nombre del usuario.

Como su nombre indica, los [tokens de acceso](https://auth0.com/docs/secure/tokens/access-tokens) están diseñados para gestionar cuestiones de control de acceso (autorización) y no contienen información sobre el usuario. **Las aplicaciones de backend funcionan exclusivamente con tokens de acceso.** Puedes recuperar información sobre el usuario que creó el token mediante la [Management API](https://auth0.com/docs/api/management/v2), como demostraremos más adelante.

<div id="laravel-installation">
  ## Instalación de Laravel
</div>

**Si aún no has configurado una aplicación de Laravel**, abre un shell en un directorio adecuado para un proyecto nuevo y ejecuta el siguiente comando:

```bash lines
composer create-project --prefer-dist laravel/laravel auth0-laravel-api ^9.0
```

Todos los comandos de esta guía asumen que los estás ejecutando desde el directorio raíz de tu proyecto de Laravel, por lo que debes usar `cd` para ir al directorio del nuevo proyecto:

```bash lines
cd auth0-laravel-api
```

<div id="sdk-installation">
  ## Instalación del SDK
</div>

Ejecuta el siguiente comando en el directorio de tu proyecto para instalar el [SDK de Auth0 para Laravel](https://github.com/auth0/laravel-auth0):

```bash lines
composer require auth0/login:^7.8 --update-with-all-dependencies
```

Luego, genera un archivo de configuración del SDK para tu aplicación:

```bash lines
php artisan vendor:publish --tag auth0
```

<div id="sdk-configuration">
  ## Configuración del SDK
</div>

Ejecuta el siguiente comando desde el directorio de tu proyecto para descargar [Auth0 CLI](https://github.com/auth0/auth0-cli):

```bash lines
curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
```

Luego, autentica la CLI con tu cuenta de Auth0 y elige &quot;como usuario&quot; cuando se te indique:

```bash lines
./auth0 login
```

A continuación, crea una nueva aplicación con Auth0:

```bash lines
./auth0 apps create \
  --name "My Laravel Backend" \
  --type "regular" \
  --auth-method "post" \
  --callbacks "http://localhost:8000/callback" \
  --logout-urls "http://localhost:8000" \
  --reveal-secrets \
  --no-input \
  --json > .auth0.app.json
```

También deberías crear una nueva API:

```bash lines
./auth0 apis create \
  --name "My Laravel Backend API" \
  --identifier "https://github.com/auth0/laravel-auth0" \
  --offline-access \
  --no-input \
  --json > .auth0.api.json
```

Esto genera dos archivos en el directorio de tu proyecto que configuran el SDK.

Como estos archivos contienen credenciales, es importante tratarlos como información confidencial. Debes asegurarte de no subirlos al control de versiones. Si usas Git, debes añadirlos a tu archivo `.gitignore`:

```bash lines
echo ".auth0.*.json" >> .gitignore
```

<div id="access-control">
  ## Control de acceso
</div>

Puedes usar el guard de autorización del SDK de Auth0 para restringir el acceso a las rutas de tu aplicación.

Para rechazar las solicitudes que no incluyan un token de acceso válido en el encabezado `Authorization`, puedes usar el `middleware` `auth` de Laravel:

```php lines
Route::get('/private', function () {
  return response()->json([
    'message' => 'Your token is valid; you are authorized.',
  ]);
})->middleware('auth');
```

También puedes exigir que el token proporcionado tenga [permisos](https://auth0.com/docs/manage-users/access-control/rbac) específicos combinando esto con el middleware `can` de Laravel:

```php lines
Route::get('/scope', function () {
    return response()->json([
      'message' => 'Your token is valid and has the `read:messages` permission; you are authorized.',
    ]);
})->middleware('auth')->can('read:messages');
```

<div id="token-information">
  ## Información del token
</div>

La información sobre el token de acceso proporcionado está disponible mediante la fachada `Auth` de Laravel o la función auxiliar `auth()`.

Por ejemplo, para recuperar el identificador y la dirección de correo electrónico del usuario:

```php lines
Route::get('/', function () {
  if (! auth()->check()) {
    return response()->json([
      'message' => 'You did not provide a valid token.',
    ]);
  }

  return response()->json([
    'message' => 'Your token is valid; you are authorized.',
    'id' => auth()->id(),
    'token' => auth()?->user()?->getAttributes(),
  ]);
});
```

<div id="retrieve-user-information">
  ## Recuperar información del usuario
</div>

Puede obtener información sobre el usuario que creó el token de acceso desde Auth0 mediante la [Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). El SDK proporciona un wrapper práctico para esta API, accesible a través del método `management()` del SDK.

**Antes de realizar llamadas a la Management API, debe habilitar su aplicación para que se comunique con la Management API.** Puede hacerlo desde la [página de API de Auth0 Dashboard](https://manage.auth0.com/#/apis/), eligiendo `Auth0 Management API` y seleccionando la pestaña &#39;Machine to Machine Applications&#39;. Autorice su aplicación de Laravel y, a continuación, haga clic en la flecha desplegable para elegir los scopes que desea conceder.

Para el siguiente ejemplo, debe conceder el scope `read:users`. Puede encontrar una lista de API endpoints y los scopes requeridos en [la documentación de la Management API](https://auth0.com/docs/api/management/v2).

```php lines
use Auth0\Laravel\Facade\Auth0;

Route::get('/me', function () {
  $user = auth()->id();
  $profile = cache()->get($user);

  if (null === $profile) {
    $endpoint = Auth0::management()->users();
    $profile = $endpoint->get($user);
    $profile = Auth0::json($profile);

    cache()->put($user, $profile, 120);
  }

  $name = $profile['name'] ?? 'Unknown';
  $email = $profile['email'] ?? 'Unknown';

  return response()->json([
    'name' => $name,
    'email' => $email,
  ]);
})->middleware('auth');
```

<Info>
  **Deberías almacenar en caché la información del usuario en tu aplicación solo durante períodos breves.** Esto reduce la cantidad de solicitudes que tu aplicación hace a Auth0 y mejora el rendimiento. Debes evitar almacenar la información del usuario en tu aplicación durante períodos prolongados, ya que esto puede hacer que los datos queden desactualizados. También debes evitar almacenar en bases de datos persistentes más información del usuario que su identificador.
</Info>

<div id="run-the-application">
  ## Ejecuta la aplicación
</div>

Ya estás listo para iniciar tu aplicación de Laravel para que pueda aceptar peticiones:

```bash lines
php artisan serve
```

<div id="retrieve-a-test-token">
  ## Obtén un token de prueba
</div>

Puedes obtener más información sobre [cómo obtener tokens de acceso aquí](https://auth0.com/docs/secure/tokens/access-tokens/get-access-tokens). Sin embargo, para este Quickstart, puedes usar simplemente un token de acceso desde [la vista &quot;test&quot; de la configuración de tu API](https://manage.auth0.com/#/apis).

<Info>
  La ruta `/me` que creamos antes no funcionará con un token de prueba, ya que no hay ningún usuario real asociado.
</Info>

<div id="checkpoint">
  ## Checkpoint
</div>

Abre una shell e intenta enviar solicitudes a tu aplicación.

Empieza solicitando la ruta pública:

```bash lines
curl --request GET \
  --url http://localhost:8000/api \
  --header 'Accept: application/json'
```

A continuación, usa tu token de acceso en el encabezado `Authorization` para hacer una solicitud a una ruta protegida:

```bash lines
curl --request GET \
  --url http://localhost:8000/api/private \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

Por último, intenta hacer una solicitud a la ruta protegida por el scope, que solo funcionará si el token de acceso tiene concedido el scope `read:messages`:

```bash lines
curl --request GET \
  --url http://localhost:8000/api/scope \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

Si tienes algún problema, aquí tienes un par de cosas que puedes probar:

* Prueba a ejecutar `php artisan optimize:clear` para borrar la caché de Laravel.
* Asegúrate de que tus archivos `.auth0.app.json` y `.auth0.api.json` estén en la raíz de tu proyecto.
* Asegúrate de haber habilitado tu aplicación Laravel como aplicación de máquina a máquina y de haberle concedido todos los scopes necesarios para la `Auth0 Management API` desde el [Auth0 Dashboard](https://manage.auth0.com/#/apis/).

¿Sigues teniendo problemas? Consulta la [documentación](https://github.com/auth0/laravel-auth0) del SDK o nuestro [centro de documentación](https://auth0.com/docs). También puedes visitar [la comunidad](https://community.auth0.com), donde nuestro equipo y otros miembros de la comunidad pueden ayudarte a resolver tus dudas.
