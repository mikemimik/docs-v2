<div id="by-evan-sims">
  ##### Por Evan Sims
</div>

El Laravel SDK de Auth0 te permite añadir rápidamente autorización basada en tokens y control de acceso a las rutas de tu aplicación Laravel. Esta guía muestra cómo integrar Auth0 en una aplicación Laravel 9 o 10, ya sea nueva o existente. Te recomendamos iniciar sesión para seguir este Quickstart con ejemplos configurados para tu cuenta.

{/* <Card title="Ver en Github" href="https://github.com/auth0-samples/laravel/tree/7.x/sample" icon="github">
  Requisitos del sistema: Laravel 9 / 10 | PHP 8.0+ | Composer
  </Card> */}

**Las aplicaciones de backend se diferencian de las aplicaciones web tradicionales en que no gestionan la autenticación de usuario ni tienen una interfaz de usuario. Proporcionan una API con la que otras aplicaciones pueden interactuar. Aceptan [tokens de acceso](https://auth0.com/docs/secure/tokens/access-tokens) en los encabezados `Authorization` de las solicitudes para controlar el acceso a las rutas.**

Normalmente, se crean aplicaciones de frontend independientes para interactuar con este tipo de backends. Pueden ser desde [aplicaciones de página única](https://auth0.com/docs/quickstart/spa) hasta [aplicaciones nativas o móviles](https://auth0.com/docs/quickstart/native) (¡y Auth0 también ofrece SDKs para todas ellas!)

Cuando los usuarios necesitan interactuar con tu aplicación de backend, primero se autentican con Auth0 mediante la aplicación de frontend. Después, la aplicación de frontend obtiene un token de acceso de Auth0, que puede usar para hacer solicitudes a tu aplicación de backend en nombre del usuario.

Como su nombre indica, los [tokens de acceso](https://auth0.com/docs/secure/tokens/access-tokens) están diseñados para gestionar el control de acceso (autorización) y no contienen información sobre el usuario. **Las aplicaciones de backend funcionan exclusivamente con tokens de acceso.** Puedes obtener información sobre el usuario que creó el token mediante la [Management API](https://auth0.com/docs/api/management/v2), como mostraremos más adelante.

<div id="laravel-installation">
  ## Instalación de Laravel
</div>

**Si todavía no tiene configurada una aplicación de Laravel**, abra una shell en un directorio adecuado para un proyecto nuevo y ejecute el siguiente comando:

```bash lines
composer create-project --prefer-dist laravel/laravel auth0-laravel-api ^9.0
```

Todos los comandos de esta guía asumen que los estás ejecutando desde la raíz de tu proyecto de Laravel, así que debes hacer `cd` al directorio del nuevo proyecto:

```bash lines
cd auth0-laravel-api
```

<div id="sdk-installation">
  ## Instalación del SDK
</div>

Ejecuta el siguiente comando en el directorio de tu proyecto para instalar el [SDK de Auth0 para Laravel](https://github.com/auth0/laravel-auth0):

```bash lines
composer require auth0/login:^7.8 --update-with-all-dependencies
```

Luego, genera un archivo de configuración de SDK para tu aplicación:

```bash lines
php artisan vendor:publish --tag auth0
```

<div id="sdk-configuration">
  ## Configuración del SDK
</div>

Ejecuta el siguiente comando desde el directorio de tu proyecto para descargar la [Auth0 CLI](https://github.com/auth0/auth0-cli):

```bash lines
curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
```

Luego, autentica la CLI con tu cuenta de Auth0 y elige &quot;como usuario&quot; cuando se te pida:

```bash lines
./auth0 login
```

A continuación, crea una nueva aplicación en Auth0:

```bash lines
./auth0 apps create \
  --name "My Laravel Backend" \
  --type "regular" \
  --auth-method "post" \
  --callbacks "http://localhost:8000/callback" \
  --logout-urls "http://localhost:8000" \
  --reveal-secrets \
  --no-input \
  --json > .auth0.app.json
```

También debes crear una nueva API:

```bash lines
./auth0 apis create \
  --name "My Laravel Backend API" \
  --identifier "https://github.com/auth0/laravel-auth0" \
  --offline-access \
  --no-input \
  --json > .auth0.api.json
```

Esto genera dos archivos en el directorio de tu proyecto que configuran el SDK.

Como estos archivos contienen credenciales, es importante tratarlos como información confidencial. Debes asegurarte de no incluirlos en el control de versiones. Si usas Git, debes agregarlos a tu archivo `.gitignore`:

```bash lines
echo ".auth0.*.json" >> .gitignore
```

<div id="access-control">
  ## Control de acceso
</div>

Puedes usar el guard de autorización del SDK de Auth0 para restringir el acceso a las rutas de tu aplicación.

Para rechazar las peticiones que no contengan un token de acceso válido en el encabezado `Authorization`, puedes usar el middleware `auth` de Laravel:

```php lines
Route::get('/private', function () {
  return response()->json([
    'message' => 'Your token is valid; you are authorized.',
  ]);
})->middleware('auth');
```

También puedes requerir que el token proporcionado tenga [permisos](https://auth0.com/docs/manage-users/access-control/rbac) específicos combinando esto con el middleware `can` de Laravel:

```php lines
Route::get('/scope', function () {
    return response()->json([
      'message' => 'Your token is valid and has the `read:messages` permission; you are authorized.',
    ]);
})->middleware('auth')->can('read:messages');
```

<div id="token-information">
  ## Información del token
</div>

Puedes acceder a la información sobre el token de acceso proporcionado a través de la fachada `Auth` de Laravel o de la función auxiliar `auth()`.

Por ejemplo, para obtener el identificador y la dirección de correo electrónico del usuario:

```php lines
Route::get('/', function () {
  if (! auth()->check()) {
    return response()->json([
      'message' => 'You did not provide a valid token.',
    ]);
  }

  return response()->json([
    'message' => 'Your token is valid; you are authorized.',
    'id' => auth()->id(),
    'token' => auth()?->user()?->getAttributes(),
  ]);
});
```

<div id="retrieve-user-information">
  ## Recuperar información del usuario
</div>

Puedes recuperar información sobre el usuario que creó el token de acceso desde Auth0 mediante la [Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). El SDK proporciona un contenedor práctico para esta API, al que se puede acceder a través del método `management()` del SDK.

**Antes de realizar llamadas a la Management API, debes habilitar tu aplicación para que se comunique con la Management API.** Esto puede hacerse desde la [página de API del Auth0 Dashboard](https://manage.auth0.com/#/apis/), eligiendo `Auth0 Management API` y seleccionando la pestaña &#39;Machine to Machine Applications&#39;. Autoriza tu aplicación de Laravel y, a continuación, haz clic en la flecha desplegable para elegir los scopes que quieras conceder.

Para el siguiente ejemplo, debes conceder el scope `read:users`. Puedes encontrar una lista de endpoints de API y los scopes requeridos en [la documentación de la Management API](https://auth0.com/docs/api/management/v2).

```php lines
use Auth0\Laravel\Facade\Auth0;

Route::get('/me', function () {
  $user = auth()->id();
  $profile = cache()->get($user);

  if (null === $profile) {
    $endpoint = Auth0::management()->users();
    $profile = $endpoint->get($user);
    $profile = Auth0::json($profile);

    cache()->put($user, $profile, 120);
  }

  $name = $profile['name'] ?? 'Unknown';
  $email = $profile['email'] ?? 'Unknown';

  return response()->json([
    'name' => $name,
    'email' => $email,
  ]);
})->middleware('auth');
```

<Info>
  **Debes almacenar en caché la información del usuario en tu aplicación solo durante periodos breves.** Esto reduce la cantidad de solicitudes que tu aplicación hace a Auth0 y mejora el rendimiento. Debes evitar almacenar la información del usuario en tu aplicación durante periodos prolongados, ya que esto puede provocar que los datos queden desactualizados. También debes evitar almacenar en bases de datos persistentes información del usuario más allá de su identificador.
</Info>

<div id="run-the-application">
  ## Ejecuta la aplicación
</div>

Ya estás listo para iniciar tu aplicación Laravel y que pueda aceptar peticiones:

```bash lines
php artisan serve
```

<div id="retrieve-a-test-token">
  ## Obtener un token de prueba
</div>

Puedes obtener más información sobre [cómo obtener tokens de acceso aquí](https://auth0.com/docs/secure/tokens/access-tokens/get-access-tokens). Sin embargo, para este Quickstart, puedes usar simplemente un token de acceso de [la vista &quot;test&quot; de la configuración de tu API](https://manage.auth0.com/#/apis).

<Info>
  La ruta `/me` que creamos antes no funcionará con un token de prueba, ya que no hay ningún usuario real asociado.
</Info>

<div id="checkpoint">
  ## Punto de control
</div>

Abra una shell y pruebe a hacer solicitudes a su aplicación.

Comience solicitando la ruta pública:

```bash lines
curl --request GET \
  --url http://localhost:8000/api \
  --header 'Accept: application/json'
```

A continuación, usa tu token de acceso en el encabezado `Authorization` para solicitar una ruta protegida:

```bash lines
curl --request GET \
  --url http://localhost:8000/api/private \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

Por último, intenta hacer una solicitud a la ruta protegida por el scope, que solo funcionará si al token de acceso se le concedió el scope `read:messages`:

```bash lines
curl --request GET \
  --url http://localhost:8000/api/scope \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

Si tienes algún problema, aquí tienes un par de cosas que puedes probar:

* Prueba a ejecutar `php artisan optimize:clear` para borrar la caché de Laravel.
* Asegúrate de que tus archivos `.auth0.app.json` y `.auth0.api.json` estén en la raíz de tu proyecto.
* Asegúrate de haber habilitado tu aplicación Laravel como una aplicación de máquina a máquina y de haberle concedido todos los scopes necesarios para la `Auth0 Management API` desde el [Auth0 Dashboard](https://manage.auth0.com/#/apis/).

¿Sigues teniendo problemas? Consulta la [documentación](https://github.com/auth0/laravel-auth0) del SDK o nuestro [centro de documentación](https://auth0.com/docs). También te recomendamos visitar [la comunidad](https://community.auth0.com), donde nuestro equipo y otros miembros de la comunidad pueden ayudarte a resolver tus dudas.
