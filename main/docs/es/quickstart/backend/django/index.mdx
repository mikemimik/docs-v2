<div id="configure-auth0-apis">
  ## Configurar las APIs de Auth0
</div>

<div id="create-an-api">
  ### Crear una API
</div>

En la sección [APIs](https://manage.auth0.com/#/apis) del Auth0 Dashboard, haz clic en **Create API**. Indica un nombre y un identificador para tu API, por ejemplo, `https://quickstarts/api`. Más adelante usarás el identificador como `audience`, al configurar la verificación del token de acceso. Deja **Signing Algorithm** en **RS256**.

<Frame>![Crear API](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/create-api.png)</Frame>

De forma predeterminada, tu API usa RS256 como algoritmo para firmar tokens. Como RS256 usa un par de claves privada/pública, los tokens se verifican con la clave pública de tu cuenta de Auth0. La clave pública está en formato [JSON Web Key Set (JWKS)](/docs/es/secure/tokens/json-web-tokens/json-web-key-sets) y puedes acceder a ella [aquí](https://\{yourDomain}/.well-known/jwks.json).

<div id="define-permissions">
  ### Definir permisos
</div>

Los permisos te permiten definir cómo se puede acceder a los recursos en nombre del usuario con un token de acceso determinado. Por ejemplo, podrías conceder acceso de lectura al recurso `messages` si los usuarios tienen el nivel de acceso de gerente, y acceso de escritura a ese recurso si tienen el nivel de acceso de administrador.

Puedes definir los permisos permitidos en la vista **Permissions** de la sección [APIs](https://manage.auth0.com/#/apis) del Auth0 Dashboard.

<Frame>![Configurar Permissions](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/configure-permissions.png)</Frame>

<Info>
  Este ejemplo usa el scope `read:messages`.
</Info>

Este ejemplo muestra:

* Cómo comprobar si hay un JSON Web Token (JWT) en el encabezado `Authorization` de una solicitud HTTP entrante.
* Cómo comprobar si el token es válido mediante el [JSON Web Key Set (JWKS)](/docs/es/secure/tokens/json-web-tokens/json-web-key-sets) de tu cuenta de Auth0. Para obtener más información sobre cómo validar Access Tokens, consulta [Validate Access Tokens](/docs/es/secure/tokens/access-tokens/validate-access-tokens).

<div id="add-a-django-remote-user">
  ### Agrega un usuario remoto de Django
</div>

Debes definir una forma de mapear el nombre de usuario del contenido del token de acceso al usuario del sistema de autenticación de Django.

Agrega el componente de middleware [`RemoteUserMiddleware`](https://docs.djangoproject.com/en/2.2/ref/middleware/#django.contrib.auth.middleware.RemoteUserMiddleware) a la lista de middleware, después de `AuthenticationMiddleware`.

```py lines
# apiexample/settings.py

MIDDLEWARE = [
    # ...
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.auth.middleware.RemoteUserMiddleware',
]
```

Agrega `ModelBackend` y `RemoteUserBackend` a los backends de autenticación.

```py lines
# apiexample/settings.py

AUTHENTICATION_BACKENDS = [
    'django.contrib.auth.backends.ModelBackend',
    'django.contrib.auth.backends.RemoteUserBackend',
]
```

Crea el archivo `utils.py` en la carpeta de tu aplicación y define una función que mapee el campo `sub` de `access_token` al nombre de usuario. Luego, el método [authenticate](https://docs.djangoproject.com/en/2.2/ref/contrib/auth/#django.contrib.auth.backends.RemoteUserBackend.authenticate) de [RemoteUserBackend](https://docs.djangoproject.com/en/2.2/ref/contrib/auth/#django.contrib.auth.backends.RemoteUserBackend) creará un usuario remoto en el sistema de autenticación de Django y devolverá un objeto User para ese nombre de usuario.

```py lines
# auth0authorization/utils.py

from django.contrib.auth import authenticate

def jwt_get_username_from_payload_handler(payload):
    username = payload.get('sub').replace('|', '.')
    authenticate(remote_user=username)
    return username
```

<div id="validate-access-tokens">
  ## Validar tokens de acceso
</div>

El archivo `settings.py` contiene la configuración del proyecto de Django.

Añade la aplicación `rest_framework` a la entrada `INSTALLED_APPS`.

```py lines
# apiexample/settings.py

INSTALLED_APPS = [
    # ...
    'rest_framework'
]
```

Añade `JSONWebTokenAuthentication` a `DEFAULT_AUTHENTICATION_CLASSES` de Django REST Framework.

```py lines
# apiexample/settings.py

REST_FRAMEWORK = {
    'DEFAULT_PERMISSION_CLASSES': (
        'rest_framework.permissions.IsAuthenticated',
    ),
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework_jwt.authentication.JSONWebTokenAuthentication',
        'rest_framework.authentication.SessionAuthentication',
        'rest_framework.authentication.BasicAuthentication',
    ),
}
```

Configura [Django REST Framework JWK](https://github.com/Styria-Digital/django-rest-framework-jwt/) definiendo la variable `JWT_AUTH`.

Configura `JWT_AUDIENCE` con el identificador de tu API y `JWT_ISSUER` con tu dominio de Auth0. De forma predeterminada, esos valores se obtendrán del archivo `.env`.

export const codeExample1 = `# apiexample/settings.py

JWT_AUTH = {
    'JWT_PAYLOAD_GET_USERNAME_HANDLER':
        'auth0authorization.utils.jwt_get_username_from_payload_handler',
    'JWT_DECODE_HANDLER':
        'auth0authorization.utils.jwt_decode_token',
    'JWT_ALGORITHM': 'RS256',
    'JWT_AUDIENCE': '{yourApiIdentifier}',
    'JWT_ISSUER': 'https://{yourDomain}/',
    'JWT_AUTH_HEADER_PREFIX': 'Bearer',
}`;

<AuthCodeBlock children={codeExample1} language="py" />

Cree la función para obtener el JWKS de su cuenta de Auth0 y verificar y decodificar el token de acceso recibido.

export const codeExample2 = `# auth0authorization/utils.py

import json

import jwt
import requests

def jwt_decode_token(token):
    header = jwt.get_unverified_header(token)
    jwks = requests.get('https://{}/.well-known/jwks.json'.format('{yourDomain}')).json()
    public_key = None
    for jwk in jwks['keys']:
        if jwk['kid'] == header['kid']:
            public_key = jwt.algorithms.RSAAlgorithm.from_jwk(json.dumps(jwk))

    if public_key is None:
        raise Exception('No se encontró la clave pública.')

    issuer = 'https://{}/'.format('{yourDomain}')
    return jwt.decode(token, public_key, audience='{yourApiIdentifier}', issuer=issuer, algorithms=['RS256'])`;

<div id="validate-scopes">
  ### Validar scopes
</div>

Agrega los siguientes métodos al archivo `views.py` para crear un decorador que comprobará los scopes otorgados en el `access_token`.

```py lines
# auth0authorization/views.py

from functools import wraps
import jwt

from django.http import JsonResponse

def get_token_auth_header(request):
    """Obtiene el Access Token del encabezado Authorization
    """
    auth = request.META.get("HTTP_AUTHORIZATION", None)
    parts = auth.split()
    token = parts[1]

    return token

def requires_scope(required_scope):
    """Determina si el scope requerido está presente en el Access Token
    Args:
        required_scope (str): El scope requerido para acceder al recurso
    """
    def require_scope(f):
        @wraps(f)
        def decorated(*args, **kwargs):
            token = get_token_auth_header(args[0])
            decoded = jwt.decode(token, verify=False)
            if decoded.get("scope"):
                token_scopes = decoded["scope"].split()
                for token_scope in token_scopes:
                    if token_scope == required_scope:
                        return f(*args, **kwargs)
            response = JsonResponse({'message': 'You don\'t have access to this resource'})
            response.status_code = 403
            return response
        return decorated
    return require_scope
```

<div id="protect-api-endpoints">
  ## Proteger los endpoints de la API
</div>

Las rutas que se muestran a continuación están disponibles para las siguientes peticiones:

* `GET /api/public`: disponible para peticiones no autenticadas
* `GET /api/private`: disponible para peticiones autenticadas que contienen un token de acceso sin scopes adicionales
* `GET /api/private-scoped`: disponible para peticiones autenticadas que contienen un token de acceso con el scope `read:messages` concedido

En el archivo `views.py`, agregue los endpoints `public` y `private`. Agregue el decorador `@api_view` a todos los endpoints para indicar que el método requiere autenticación. Por último, agregue el decorador `@permission_classes([AllowAny])` al endpoint `public` para aceptar peticiones no autenticadas.

```py lines
# auth0authorization/views.py

from django.http import JsonResponse
from rest_framework.decorators import api_view, permission_classes
from rest_framework.permissions import AllowAny

@api_view(['GET'])
@permission_classes([AllowAny])
def public(request):
    return JsonResponse({'message': 'Hello from a public endpoint! You don\'t need to be authenticated to see this.'})


@api_view(['GET'])
def private(request):
    return JsonResponse({'message': 'Hello from a private endpoint! You need to be authenticated to see this.'})
```

Usa el decorador `requires_scope` en los métodos que requieran los scopes específicos otorgados. El siguiente método requiere el scope `read:messages` otorgado.

```py lines
# auth0authorization/views.py

@api_view(['GET'])
@requires_scope('read:messages')
def private_scoped(request):
    return JsonResponse({'message': 'Hello from a private endpoint! You need to be authenticated and have a scope of read:messages to see this.'})
```

<div id="add-url-mappings">
  ### Agregar mapeos de URL
</div>

En los pasos anteriores, agregamos métodos al archivo `views.py`. Ahora necesitamos asociar esos métodos con las URL.

Django tiene un [despachador de URL](https://docs.djangoproject.com/en/2.2/topics/http/urls/) que te permite asociar patrones de URL con vistas.

Crea el archivo `urls.py` en la carpeta de tu aplicación. Agrega los patrones de URL.

```py lines
# auth0authorization/urls.py

from django.urls import path

from . import views

urlpatterns = [
    path('api/public', views.public),
    path('api/private', views.private),
    path('api/private-scoped', views.private_scoped),
]
```

El proyecto de Django también tiene un archivo `urls.py`. Añade una referencia al archivo `urls.py` de tu aplicación.

```py lines
# apiexample/urls.py

from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('auth0authorization.urls'))
]
```
