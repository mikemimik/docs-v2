    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Usarás una biblioteca llamada [Authlib](https://github.com/lepture/authlib) para crear un [ResourceProtector](https://docs.authlib.org/en/latest/flask/1/resource-server.html), que es un tipo de [decorador de vista de Django](https://docs.djangoproject.com/en/4.0/topics/http/decorators/) que protege tus recursos (vistas de API) con un
      validador específico.

      El validador verificará el token de acceso que pases al recurso, comprobando que tenga una
      firma válida y claims válidos.

      Puedes usar el validador `JWTBearerTokenValidator` de AuthLib con algunos ajustes para asegurarte de que cumpla
      con nuestros requisitos para [validar tokens de acceso](https://auth0.com/docs/secure/tokens/access-tokens/validate-access-tokens).

      Para crear tu `Auth0JWTBearerTokenValidator`, debes proporcionarle tu `domain` y
      `audience` (identificador de API). Luego obtendrá la clave pública necesaria para verificar la firma del token
      y se la pasará a la clase `JWTBearerTokenValidator`.

      Luego sobrescribirás `claims_options` de la clase para asegurarte de que los claims `expiry`,
      `audience` e `issue` del token se validen según nuestros requisitos.

      Crea el archivo `apiexample/validator.py` con el código del panel interactivo.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      A continuación, crearás tres vistas de API en `apiexample/views.py`:

      * `/api/public`: Un endpoint público que no requiere autenticación.
      * `/api/private`: Un endpoint privado que requiere un JWT de token de acceso válido.
      * `/api/private-scoped`: Un endpoint privado que requiere un JWT de token de acceso válido que contenga el
        `scope` indicado.

      Las routes protegidas tendrán un decorador `require_auth`, que es un `ResourceProtector`
      que usa el `Auth0JWTBearerTokenValidator` que creaste anteriormente.

      Para crear el `Auth0JWTBearerTokenValidator`, le pasarás el dominio de tu tenant y el identificador
      de la API que creaste anteriormente.

      El decorador `require_auth` en la route `private_scoped` acepta un argumento adicional,
      `"read:messages"`, que comprueba si el token de acceso incluye el permiso (`scope`) que creaste anteriormente.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      En los pasos anteriores, agregaste métodos al archivo `views.py`. A continuación, asigna esos métodos a URLs mediante el [URL dispatcher](https://docs.djangoproject.com/en/4.0/topics/http/urls/) de Django, que te permite asociar patrones de URL a vistas.

      Agrega los patrones de URL a tu archivo `apiexample/urls.py`.

      ### Realiza una llamada a tu API

      Para realizar llamadas a tu API, necesitarás un token de acceso. Puedes obtener un token de acceso para fines de prueba
      desde la vista **Test** en tu [configuración de API](https://manage.auth0.com/#/apis).
