<div id="get-started">
  ## Primeros pasos
</div>

Crearás una API de Go con tres endpoints que muestran distintos niveles de protección: acceso público, autenticación con JWT y permisos específicos. La implementación completa usa [go-jwt-middleware v3](https://github.com/auth0/go-jwt-middleware) con la biblioteca estándar `net/http` de Go.

<Steps>
  <Step title="Crear un nuevo proyecto" stepNumber={1}>
    Crea un directorio nuevo para tu API de Go e inicializa un módulo.

    ```shellscript
    mkdir myapi && cd myapi
    go mod init github.com/yourorg/myapi
    ```

    Instala las dependencias necesarias:

    ```shellscript
    go get github.com/auth0/go-jwt-middleware/v3
    go get github.com/joho/godotenv
    go mod download
    ```

    Crea la estructura del proyecto:

    ```shellscript
    mkdir -p cmd/server internal/auth internal/config internal/handlers
    touch .env cmd/server/main.go internal/config/auth.go internal/auth/claims.go internal/auth/validator.go internal/auth/middleware.go internal/handlers/api.go
    ```

    <Accordion title="Ver el go.mod esperado">
      ```go go.mod
      module github.com/yourorg/myapi

      go 1.24

      require (
          github.com/auth0/go-jwt-middleware/v3 v3.1.0
          github.com/joho/godotenv v1.5.1
      )
      ```
    </Accordion>
  </Step>

  <Step title="Configura tu API de Auth0" stepNumber={2}>
    A continuación, debes crear una nueva API en tu tenant de Auth0 y agregar las variables de entorno a tu proyecto.

    Tienes dos opciones para configurar tu API de Auth0: usar un comando de CLI o hacerlo manualmente desde el Dashboard:

    <Tabs>
      <Tab title="CLI">
        Ejecuta el siguiente comando en el directorio raíz de tu proyecto para crear una API de Auth0:

        <CodeGroup>

          ```shellscript Mac
            --name "My Go API" \
          ```

          ```powershell Windows
          scoop bucket add auth0 https://github.com/auth0/scoop-auth0-cli.git
          scoop install auth0

          # Crea una API de Auth0
          auth0 apis create `
            --name "My Go API" `
            --identifier https://my-go-api.example.com
          ```
        </CodeGroup>

        Después de crearla, copia los valores de **Identifier** y **Domain**, y luego crea tu archivo `.env`:

        <Note>
          Este comando hará lo siguiente:

          1. Verificará si ya estás autenticado (y te pedirá iniciar sesión si hace falta)
          2. Creará una API de Auth0 con el identificador especificado
          3. Mostrará los detalles de la API, incluidos el dominio y el identificador

      <Tab title="Dashboard">
        1. Ve al [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
        2. Ve a **Applications** → **APIs** → **Create API**
        3. Introduce un nombre para tu API (por ejemplo, &quot;My Go API&quot;)
        4. Define el **Identifier** (por ejemplo, `https://my-go-api.example.com`)

           * No tiene que ser una URL real; es solo un identificador
        5. Deja **Signing Algorithm** como **RS256**
        6. Haz clic en **Create**
        7. Copia el valor de **Identifier** desde la pestaña **Settings**

        ```bash .env
        AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
        AUTH0_AUDIENCE=YOUR_API_IDENTIFIER
        ```

        <Warning>
          Reemplaza `YOUR_AUTH0_DOMAIN` por el dominio de tu tenant de Auth0 (por ejemplo, `dev-abc123.us.auth0.com`) y `YOUR_API_IDENTIFIER` por el identificador de tu API del Dashboard (por ejemplo, `https://my-go-api.example.com`).
        </Warning>
      </Tab>

      **Security**: Nunca subas archivos `.env` al control de versiones. Agrega `.env` a tu archivo `.gitignore`.

  <Step title="Definir permisos de la API" stepNumber={3}>
    Los permisos (scopes) te permiten definir cómo se puede acceder a los recursos. Por ejemplo, otorga acceso `read` a los gerentes y acceso `write` a los administradores.

    1. En la configuración de tu API, haz clic en la pestaña **Permissions**
    2. Crea el siguiente permiso:

    | Permiso         | Descripción             |
    | --------------- | ----------------------- |
    | `read:messages` | Leer mensajes de la API |

    <Info>
      Este tutorial usa el scope `read:messages` para proteger el endpoint restringido por scope. Puedes definir permisos adicionales según las necesidades de tu aplicación.
    </Info>
  </Step>

  <Step title="Crear un cargador de configuración" stepNumber={4}>
    Crea un package de configuración para cargar y validar las variables de entorno.

    ```go internal/config/auth.go lines
    package config

    import (
        "fmt"
        "os"
    )

    type AuthConfig struct {
        Domain   string
        Audience string
    }

    func LoadAuthConfig() (*AuthConfig, error) {
        domain := os.Getenv("AUTH0_DOMAIN")
        if domain == "" {
            return nil, fmt.Errorf("AUTH0_DOMAIN environment variable required")
        }

        audience := os.Getenv("AUTH0_AUDIENCE")
        if audience == "" {
            return nil, fmt.Errorf("AUTH0_AUDIENCE environment variable required")
        }

        return &AuthConfig{
            Domain:   domain,
            Audience: audience,
        }, nil
    }
    ```

    **Qué hace esto:**

    * Carga el dominio de Auth0 y el audience desde las variables de entorno
    * Valida que la configuración necesaria esté presente al iniciarse
    * Devuelve una estructura `config` con tipos seguros para usarla en toda la aplicación
  </Step>

  <Step title="Crear claims personalizadas y un validador de JWT" stepNumber={5}>
    Los claims personalizados te permiten extraer y validar datos específicos de la aplicación a partir de JWT. El validador es el componente principal que verifica tokens con Auth0.

    <Tabs>
      <Tab title="claims.go">
        ```go internal/auth/claims.go lines
        package auth

        import (
            "context"
            "fmt"
            "strings"
        )

        // CustomClaims contiene datos personalizados que queremos analizar del JWT.
        type CustomClaims struct {
            Scope string `json:"scope"`
        }

        // Validate garantiza que los claims personalizados tengan el formato correcto.
        func (c *CustomClaims) Validate(ctx context.Context) error {
            if c.Scope == "" {
                return nil
            }

            if strings.TrimSpace(c.Scope) != c.Scope {
                return fmt.Errorf("scope claim has invalid whitespace")
            }

            if strings.Contains(c.Scope, "  ") {
                return fmt.Errorf("scope claim contains double spaces")
            }

            return nil
        }

        // HasScope comprueba si nuestros claims incluyen un scope específico.
        func (c *CustomClaims) HasScope(expectedScope string) bool {
            if c.Scope == "" {
                return false
            }

            scopes := strings.Split(c.Scope, " ")
            for _, scope := range scopes {
                if scope == expectedScope {
                    return true
                }
            }
            return false
        }
        ```
      </Tab>

      <Tab title="validator.go">
        ```go internal/auth/validator.go lines
        package auth

        import (
            "fmt"
            "net/url"
            "time"

            "github.com/auth0/go-jwt-middleware/v3/jwks"
            "github.com/auth0/go-jwt-middleware/v3/validator"
        )

        func NewValidator(domain, audience string) (*validator.Validator, error) {
            // Construye la issuer URL (debe incluir la barra final)
            issuerURL, err := url.Parse("https://" + domain + "/")
            if err != nil {
                return nil, fmt.Errorf("failed to parse issuer URL: %w", err)
            }

            // Inicializa el proveedor JWKS con el patrón de opciones de v3
            provider, err := jwks.NewCachingProvider(
                jwks.WithIssuerURL(issuerURL),
                jwks.WithCacheTTL(5*time.Minute),
            )
            if err != nil {
                return nil, fmt.Errorf("failed to create JWKS provider: %w", err)
            }

            // Crea el validador con el patrón de opciones de v3
            jwtValidator, err := validator.New(
                validator.WithKeyFunc(provider.KeyFunc),
                validator.WithAlgorithm(validator.RS256),
                validator.WithIssuer(issuerURL.String()),
                validator.WithAudience(audience),
                validator.WithCustomClaims(func() validator.CustomClaims {
                    return &CustomClaims{}
                }),
                validator.WithAllowedClockSkew(30*time.Second),
            )
            if err != nil {
                return nil, fmt.Errorf("failed to create validator: %w", err)
            }

            return jwtValidator, nil
        }
        ```
      </Tab>
    </Tabs>

    **Puntos clave:**

    * El método `Validate` lo llama automáticamente el middleware después de analizar el JWT
    * `HasScope` analiza scopes separados por espacios para el control de acceso basado en permisos
    * El validador usa caché de JWKS (TTL de 5 min) y permite un desfase de reloj de 30 s
    * El algoritmo RS256 se configura explícitamente para evitar [ataques de confusión de algoritmo](https://auth0.com/blog/critical-vulnerabilities-in-json-web-token-libraries/)
  </Step>

  <Step title="Crear middleware y controladores HTTP" stepNumber={6}>
    El middleware envuelve el validador para las solicitudes HTTP. Los controladores muestran tres niveles de protección: público, privado y restringido por permiso.

    <Tabs>
      <Tab title="middleware.go">
        ```go internal/auth/middleware.go lines
        package auth

        import (
            "log/slog"
            "net/http"

            jwtmiddleware "github.com/auth0/go-jwt-middleware/v3"
            "github.com/auth0/go-jwt-middleware/v3/validator"
        )

        func NewMiddleware(jwtValidator *validator.Validator) (*jwtmiddleware.JWTMiddleware, error) {
            return jwtmiddleware.New(
                jwtmiddleware.WithValidator(jwtValidator),
                jwtmiddleware.WithValidateOnOptions(false),
                jwtmiddleware.WithErrorHandler(func(w http.ResponseWriter, r *http.Request, err error) {
                    slog.Error("JWT validation failed", "error", err, "path", r.URL.Path)
                    w.Header().Set("Content-Type", "application/json")
                    w.WriteHeader(http.StatusUnauthorized)
                    w.Write([]byte(`{"message":"Failed to validate JWT."}`))
                }),
            )
        }
        ```
      </Tab>

      <Tab title="api.go">
        ```go internal/handlers/api.go expandable lines
        package handlers

        import (
            "encoding/json"
            "net/http"

            "github.com/yourorg/myapi/internal/auth"
            jwtmiddleware "github.com/auth0/go-jwt-middleware/v3"
            "github.com/auth0/go-jwt-middleware/v3/validator"
        )

        // PublicHandler - no requiere autenticación
        func PublicHandler(w http.ResponseWriter, r *http.Request) {
            response := map[string]string{
                "message": "¡Hola desde un endpoint público! No necesitas autenticarte para ver esto.",
            }
            w.Header().Set("Content-Type", "application/json")
            json.NewEncoder(w).Encode(response)
        }

        // PrivateHandler - requiere un JWT válido
        func PrivateHandler(w http.ResponseWriter, r *http.Request) {
            response := map[string]string{
                "message": "¡Hola desde un endpoint privado! Necesitas autenticarte para ver esto.",
            }
            w.Header().Set("Content-Type", "application/json")
            json.NewEncoder(w).Encode(response)
        }

        // ScopedHandler - requiere el permiso 'read:messages'
        func ScopedHandler(w http.ResponseWriter, r *http.Request) {
            claims, err := jwtmiddleware.GetClaims[*validator.ValidatedClaims](r.Context())
            if err != nil {
                w.Header().Set("Content-Type", "application/json")
                w.WriteHeader(http.StatusUnauthorized)
                w.Write([]byte(`{"message":"Unauthorized."}`))
                return
            }

            customClaims, ok := claims.CustomClaims.(*auth.CustomClaims)
            if !ok || !customClaims.HasScope("read:messages") {
                w.Header().Set("Content-Type", "application/json")
                w.WriteHeader(http.StatusForbidden)
                w.Write([]byte(`{"message":"Insufficient scope."}`))
                return
            }

            response := map[string]string{
                "message": "¡Hola desde un endpoint privado! Necesitas autenticarte y tener el scope read:messages para ver esto.",
            }
            w.Header().Set("Content-Type", "application/json")
            json.NewEncoder(w).Encode(response)
        }
        ```
      </Tab>
    </Tabs>

    **Niveles de protección:**

    * **Público** (`/api/public`) — No requiere autenticación
    * **Privado** (`/api/private`) — Requiere un JWT válido
    * **Restringido por permiso** (`/api/private-scoped`) — Requiere un JWT válido + el permiso `read:messages`
  </Step>

  <Step title="Crear el servidor principal" stepNumber={7}>
    Integra todo en el punto de entrada principal, con tiempos de espera aptos para producción y un apagado controlado:

    ```go cmd/server/main.go expandable lines
    package main

    import (
        "context"
        "log"
        "net/http"
        "os"
        "os/signal"
        "time"

        "github.com/yourorg/myapi/internal/auth"
        "github.com/yourorg/myapi/internal/config"
        "github.com/yourorg/myapi/internal/handlers"
        "github.com/joho/godotenv"
    )

    func main() {
        // Cargar variables de entorno desde el archivo .env
        if err := godotenv.Load(); err != nil {
            log.Println("No .env file found, using environment variables")
        }

        // Cargar la configuración de Auth0
        cfg, err := config.LoadAuthConfig()
        if err != nil {
            log.Fatalf("Failed to load config: %v", err)
        }

        // Crear el validador de JWT
        jwtValidator, err := auth.NewValidator(cfg.Domain, cfg.Audience)
        if err != nil {
            log.Fatalf("Failed to create validator: %v", err)
        }

        // Crear el middleware HTTP
        middleware, err := auth.NewMiddleware(jwtValidator)
        if err != nil {
            log.Fatalf("Failed to create middleware: %v", err)
        }

        // Configurar las rutas
        mux := http.NewServeMux()
        mux.HandleFunc("/api/public", handlers.PublicHandler)
        mux.Handle("/api/private", middleware.CheckJWT(http.HandlerFunc(handlers.PrivateHandler)))
        mux.Handle("/api/private-scoped", middleware.CheckJWT(http.HandlerFunc(handlers.ScopedHandler)))

        // Configurar el servidor con timeouts para producción
        srv := &http.Server{
            Addr:         ":8080",
            Handler:      mux,
            ReadTimeout:  15 * time.Second,
            WriteTimeout: 15 * time.Second,
            IdleTimeout:  60 * time.Second,
        }

        // Iniciar el servidor en una goroutine
        go func() {
            log.Println("Server starting on :8080")
            if err := srv.ListenAndServe(); err != nil && err != http.ErrServerClosed {
                log.Fatalf("Server failed: %v", err)
            }
        }()

        // Apagado controlado
        quit := make(chan os.Signal, 1)
        signal.Notify(quit, os.Interrupt)
        <-quit

        log.Println("Shutting down server...")
        ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
        defer cancel()

        if err := srv.Shutdown(ctx); err != nil {
            log.Fatalf("Server forced to shutdown: %v", err)
        }

        log.Println("Server exited")
    }
    ```

    <Accordion title="Estructura del proyecto">
      ```
      myapi/
      ├── cmd/
      │   └── server/
      │       └── main.go              # Punto de entrada de la aplicación
      ├── internal/
      │   ├── auth/
      │   │   ├── claims.go            # Claims de JWT personalizadas
      │   │   ├── middleware.go         # Middleware de JWT
      │   │   └── validator.go         # Validador de JWT
      │   ├── config/
      │   │   └── auth.go              # Cargador de configuración
      │   └── handlers/
      │       └── api.go               # Controladores HTTP (públicos, privados, con alcance limitado)
      ├── .env                         # Variables de entorno (no versionadas)
      ├── .gitignore
      ├── go.mod
      └── go.sum
      ```
    </Accordion>
  </Step>

<div id="calling-your-api">
  ## Llamar a tu API
</div>

Puedes llamar a tu API protegida desde cualquier aplicación pasando un token de acceso en el encabezado `Authorization` como token Bearer.

<Accordion title="Ejemplos de código del cliente" defaultOpen>
  <CodeGroup>
    ```bash cURL
    curl --request GET \
      --url http://localhost:8080/api/private \
      --header 'authorization: Bearer YOUR_ACCESS_TOKEN'
    ```

    ```go Go
    package main

    import (
        "fmt"
        "net/http"
        "io"
    )

    func main() {
        url := "http://localhost:8080/api/private"

        req, _ := http.NewRequest("GET", url, nil)
        req.Header.Add("authorization", "Bearer YOUR_ACCESS_TOKEN")

        res, _ := http.DefaultClient.Do(req)
        defer res.Body.Close()

        body, _ := io.ReadAll(res.Body)

        fmt.Println(res)
        fmt.Println(string(body))
    }
    ```

    ```javascript Node.js
    const axios = require("axios").default;

    const options = {
      method: 'GET',
      url: 'http://localhost:8080/api/private',
      headers: {authorization: 'Bearer YOUR_ACCESS_TOKEN'}
    };

    axios.request(options).then(function (response) {
      console.log(response.data);
    }).catch(function (error) {
      console.error(error);
    });
    ```

    ```python Python
    import http.client

    conn = http.client.HTTPConnection("localhost", 8080)
    headers = { 'authorization': "Bearer YOUR_ACCESS_TOKEN" }

    conn.request("GET", "/api/private", headers=headers)

    res = conn.getresponse()
    data = res.read()

    print(data.decode("utf-8"))
    ```
  </CodeGroup>
</Accordion>

      <Steps>

        <Step title="Solicitar un token de acceso">
          Usa el flujo de credenciales de cliente para obtener un token de acceso:

          ```bash
          curl --request POST \
            --url 'https://YOUR_AUTH0_DOMAIN/oauth/token' \
            --header 'content-type: application/x-www-form-urlencoded' \
            --data grant_type=client_credentials \
            --data 'client_id=YOUR_CLIENT_ID' \
            --data client_secret=YOUR_CLIENT_SECRET \
            --data audience=YOUR_API_IDENTIFIER
          ```
        </Step>
      </Steps>

      <Warning>
        **Reutilización de tokens**: A los clientes de Auth0 se les factura en función de la cantidad de tokens de acceso de máquina a máquina emitidos. Una vez que tu aplicación obtenga un token de acceso, debería seguir usándolo hasta que expire para minimizar la cantidad de tokens solicitados.
      </Warning>

  ## Uso avanzado
</div>

<Accordion title="Seguridad DPoP (Proof-of-Possession)">
  **DPoP (Demonstrating Proof-of-Possession)**, según la RFC 9449, proporciona mayor seguridad al evitar el robo de tokens mediante la vinculación criptográfica de claves.

  ```go internal/auth/middleware.go
  func NewMiddleware(jwtValidator *validator.Validator) *jwtmiddleware.JWTMiddleware {
      return jwtmiddleware.New(
          jwtmiddleware.WithValidator(jwtValidator),
          jwtmiddleware.WithDPoPMode(jwtmiddleware.DPoPRequired),
          jwtmiddleware.WithLogger(slog.Default()),
      )
  }
  ```

  **Modos de DPoP:**

  * `DPoPAllowed` (predeterminado) — Acepta tokens Bearer y DPoP
  * `DPoPRequired` — Solo acepta tokens DPoP; rechaza Bearer
  * `DPoPDisabled` — Solo acepta tokens Bearer; rechaza DPoP

  <Note>
    Se recomienda DPoP para las API financieras, las API de atención médica y las aplicaciones empresariales de alta seguridad. Obtén más información en la [documentación de DPoP](https://github.com/auth0/go-jwt-middleware#dpop-support).
  </Note>
</Accordion>

<Accordion title="Configuración de CORS">
  Habilita CORS para permitir solicitudes desde aplicaciones web. Puedes usar un middleware sencillo o una biblioteca como [rs/cors](https://github.com/rs/cors):

  ```bash
  go get github.com/rs/cors
  ```

  ```go cmd/server/main.go
  import "github.com/rs/cors"

  // Wrap the mux with CORS middleware
  handler := cors.New(cors.Options{
      AllowedOrigins:   []string{"http://localhost:3000"},
      AllowedMethods:   []string{"GET", "POST", "PUT", "DELETE"},
      AllowedHeaders:   []string{"Authorization", "Content-Type"},
      AllowCredentials: true,
  }).Handler(mux)

  srv := &http.Server{
      Addr:    ":8080",
      Handler: handler,
  }
  ```

  En producción, especifica orígenes exactos en lugar de comodines.
</Accordion>

<Accordion title="Registro estructurado con slog">
  Habilita el registro detallado para depurar la validación de tokens:

  ```go internal/auth/middleware.go
  middleware := jwtmiddleware.New(
      jwtmiddleware.WithValidator(jwtValidator),
      jwtmiddleware.WithLogger(slog.Default()),
  )
  ```

  Agrega una verificación al iniciar:

  ```go cmd/server/main.go
  log.Printf("Validator configured:")
  log.Printf("  Issuer: https://%s/", cfg.Domain)
  log.Printf("  Audience: %s", cfg.Audience)
  log.Printf("  Algorithm: RS256")
  ```
</Accordion>

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="Problemas comunes y soluciones">
    ### &quot;Failed to validate JWT&quot; o 401 Unauthorized

    **Problema:** La API no puede encontrar ni validar el token de acceso.

    **Soluciones:**

    1. Asegúrate de que el encabezado `Authorization` esté presente: `Authorization: Bearer YOUR_TOKEN`
    2. Comprueba que &quot;Bearer&quot; esté incluido antes del token
    3. Verifica que el token no haya expirado
    4. Asegúrate de que estás usando un **token de acceso**, no un token de identidad

    ### &quot;aud claim mismatch&quot;

    **Problema:** La audiencia del token no coincide con la de tu API.

    **Solución:** Verifica que `AUTH0_AUDIENCE` coincida exactamente con el identificador de tu API en el Auth0 Dashboard. La audiencia NO debe tener una barra diagonal final:

    ```bash
    # Correcto
    AUTH0_AUDIENCE=https://my-go-api.example.com

    # Incorrecto (con barra diagonal final)
    AUTH0_AUDIENCE=https://my-go-api.example.com/
    ```

    La aplicación cliente también debe solicitar un token con el parámetro audience correcto.

    ### &quot;unexpected signing method&quot;

    **Problema:** El algoritmo del token no coincide con la configuración del validador.

    **Soluciones:**

    1. Auth0 usa RS256 de forma predeterminada (asimétrico)
    2. Asegúrate de que tu validador especifique `validator.RS256`
    3. Nunca uses `validator.HS256` para tokens de Auth0, a menos que esté configurado específicamente

    ### endpoint JWKS inaccesible

    **Problema:** El proveedor de almacenamiento en caché de JWKS no puede alcanzar el endpoint de clave pública de Auth0.

    **Soluciones:**

    1. Comprueba la conectividad de red con Auth0 (configuración de firewall/proxy)
    2. Prueba el endpoint JWKS manualmente: `curl https://YOUR_AUTH0_DOMAIN/.well-known/jwks.json`
    3. Verifica que la región de Auth0 sea la correcta (us/eu/au)

    ### Ruta de importación incorrecta

    **Problema:** `cannot find package "github.com/auth0/go-jwt-middleware/v3/..."`

    **Solución:** Asegúrate de que todas las importaciones usen el sufijo `/v3`:

    ```go
    // Correcto
    import "github.com/auth0/go-jwt-middleware/v3/validator"

    // Incorrecto
    import "github.com/auth0/go-jwt-middleware/validator"
    ```

    ### Desfase del reloj / errores de expiración del token

    **Problema:** El reloj del servidor no está sincronizado, lo que hace que tokens válidos parezcan expirados.

    **Solución:** El validador ya incluye una tolerancia de 30 s para el desfase del reloj. Si necesitas más, ajústala:

    ```go
    validator.WithAllowedClockSkew(60*time.Second)
    ```

    ### Error al extraer claims

    **Problema:** `Failed to retrieve claims` al usar genéricos.

    **Solución:** Asegúrate de que estás usando el parámetro de tipo correcto:

    ```go
    claims, err := jwtmiddleware.GetClaims[*validator.ValidatedClaims](r.Context())
    ```
  </Accordion>
</AccordionGroup>
