En este tutorial se muestra cómo usar el módulo `nginx-openid-connect` para añadir autenticación y autorización a su servidor NGINX. Le recomendamos que inicie sesión para seguir este Quickstart con ejemplos configurados para su cuenta.

<Note>
  ### Requisitos del sistema

  Este tutorial y el proyecto semilla se han probado con lo siguiente:

  * NGINX Plus R24
</Note>

**Siga los pasos que se indican a continuación para configurar su aplicación con [**NGINX Plus**](https://www.nginx.com/products/nginx/) para que funcione con Auth0 y Open ID Connect.**

<div id="install-and-enable-nginx-plus-module-njs-module">
  ## Instala y habilita el módulo nginx-plus-module-njs
</div>

Primero, debes instalar el módulo nginx-plus-module-njs para NGINX Plus. Sigue la [guía de instalación de módulos dinámicos](https://docs.nginx.com/nginx/admin-guide/dynamic-modules/nginscript) para instalar paquetes en el sistema operativo anfitrión.
Para las distribuciones de Linux que usan el gestor de paquetes `yum`, instala de la siguiente manera:

```bash lines
sudo yum install nginx-plus-module-njs jq
```

Una vez instalado, debes habilitarlo en NGINX añadiendo la siguiente línea al principio del archivo `/etc/nginx/nginx.conf`:

```bash lines
load_module modules/ngx_http_js_module.so;
```

<div id="checkout-nginx-openid-connect-template-repository">
  ## Descarga el repositorio de plantilla nginx-openid-connect
</div>

Clona el [repositorio de GitHub `nginx-openid-connect`](https://github.com/nginxinc/nginx-openid-connect). Este repositorio incluye una configuración de plantilla.

```bash lines
git clone https://github.com/nginxinc/nginx-openid-connect
```

<div id="configure-with-your-auth0-application-information">
  ## Configura con la información de tu aplicación de Auth0
</div>

Ejecuta el script `configure.sh` dentro de la carpeta nginx-openid-connect para completar la configuración de la plantilla de tu aplicación de Auth0:

export const codeExample1 = `./configure.sh --auth_jwt_key request \
  --client_id {yourClientId} \
  --pkce_enable \
  https://{yourDomain}/.well-known/openid-configuration`;

<div id="set-accept-encoding-type-for-token-and-jwks-endpoints">
  ## Configurar el tipo de Accept-Encoding para los endpoints de token y JWKS
</div>

Agrega el encabezado `Accept-Encoding` en `openid_connect.server_conf`

```conf lines
# openid_connect.server_conf
location = /_jwks_uri {
    internal;
    ...
    proxy_set_header    Content-Length "";           
    proxy_set_header    Accept-Encoding "gzip";          # esto es obligatorio
    ...
}

location = /_token {
    internal;
    ...
    proxy_set_header    Content-Type "application/x-www-form-urlencoded";
    proxy_set_header    Accept-Encoding "gzip";          # esto es obligatorio
    ...
}
```

<div id="copy-openid-connect-config-files-to-nginx-server">
  ## Copie los archivos de configuración de OpenID Connect al servidor NGINX
</div>

Debe copiar cuatro archivos a la carpeta de configuración del servidor NGINX

```sh lines
sudo cp openid_connect.js \ 
   frontend.conf \
   openid_connect_configuration.conf \
   openid_connect.server_conf /etc/nginx/conf.d
```

<div id="configuring-auth0-settings">
  ## Configuración de la aplicación en Auth0
</div>

En la configuración de tu aplicación, agrega un nuevo valor en &quot;Allowed Callback URLs&quot; que sea `https://server-fqdn/_codexch`.

Luego, en Auth0, cambia &quot;Token Endpoint Authentication Method&quot; a &quot;None&quot; para tu aplicación. Esto es necesario para el flujo de código de autorización con PKCE.

<div id="passing-headers-to-upstream-application">
  ## Pasar encabezados a la aplicación upstream
</div>

Edita `/etc/nginx/conf.d/frontend.conf` y agrega encabezados adicionales de `id_token` al destino upstream:

```conf lines
# frontend.conf
# auth_jwt_claim_set $claim_name https://namespace/key;

server {
    include conf.d/openid_connect.server_conf; # Flujo de código de autorización y procesamiento de Relying Party
    error_log /var/log/nginx/error.log debug;  # Reducir el nivel de severidad según sea necesario

    listen 8010; # Usar SSL/TLS en producción
    
    location / {
        # Este sitio está protegido con OpenID Connect
        auth_jwt "" token=$session_jwt;
        error_page 401 = @do_oidc_flow;

        #auth_jwt_key_file $oidc_jwt_keyfile; # Habilitar cuando se use nombre de archivo
        auth_jwt_key_request /_jwks_uri; # Habilitar cuando se use URL

        # Los usuarios autenticados correctamente se envían mediante proxy al backend,
        # con el claim 'sub' enviado como encabezado HTTP
        proxy_set_header username $jwt_claim_sub;
        proxy_set_header x-email $jwt_claim_email;
        #proxy_set_header x-custom $claim_name;             # claim con espacio de nombres

        proxy_pass http://my_backend; # El sitio/aplicación del backend

        access_log /var/log/nginx/access.log main_jwt;
    }
}
```
