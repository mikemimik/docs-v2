<div id="get-started">
  ## Primeros pasos
</div>

Este tutorial muestra cómo configurar Apache para añadir autenticación y autorización a su aplicación web. Le recomendamos iniciar sesión para seguir esta guía de inicio rápido con ejemplos configurados para su cuenta.

<Steps>
  <Step title="Instale y habilite el módulo mod_auth_openidc" stepNumber={1}>
    Primero, instale el módulo `mod_auth_openidc` para Apache.

    Puede obtener los binarios desde [GitHub](https://github.com/OpenIDC/mod_auth_openidc/releases) e instalarlos en su sistema operativo. Si su sistema operativo no es compatible con ninguno de los binarios, también puede [compilarlo desde el código fuente](https://github.com/OpenIDC/mod_auth_openidc/blob/master/INSTALL).

    Una vez que haya instalado el módulo, habilítelo en Apache con el comando `a2enmod`. Para obtener más información, consulte la [página del manual de a2enmod en Ubuntu](https://manpages.ubuntu.com/manpages/focal/man8/a2enmod.8.html):

    ```shellscript
    a2enmod auth_openidc
    ```

    <Info>
      En Windows, puede usar [este script de Powershell](https://github.com/enderandpeter/win-a2enmod#installation) para que `a2enmod` funcione en su sistema.
    </Info>
  </Step>

  <Step title="Configure el módulo con la información de su cuenta de Auth0" stepNumber={2}>
    Actualice su nuevo archivo de configuración (`auth_openidc.conf`), ubicado en la carpeta `/etc/apache2/mods-available`.

    <Info>
      En Windows, debe usar el archivo `/apache/conf/httpd.conf`.
    </Info>

    <AuthCodeBlock children={configSnippet} language="apache" filename="auth_openidc.conf" />
  </Step>

  <Step title="Configurar Auth0" stepNumber={3}>
    En el [Auth0 Dashboard](https://manage.auth0.com/):

    1. Vaya a **Applications** &gt; **Applications** y seleccione su aplicación de la lista.
    2. Cambie a la vista **Settings** y busque la sección **Application URIs**.
    3. Agregue el valor de `OIDCRedirectURI` a **Allowed Callback URLs**.
    4. Busque **Advanced Settings** al final de la página.
    5. Cambie a la vista **OAuth**.
    6. Establezca **JSON Web Token (JWT) Signature Algorithm** en `RS256`.
  </Step>

  <Step title="Autorización" stepNumber={4}>
    Puede configurar Apache para proteger una ubicación específica según el valor de un claim en el token de ID del usuario; para ello, agregue un bloque `Location` a su archivo `auth_openidc.conf`.

    Por ejemplo, podría crear una Action que lea los roles del usuario y luego agregue un claim que conceda acceso a una ubicación protegida:

    ```js lines
    exports.onExecutePostLogin = async (event, api) => {
      const roles = event.authorization.roles; // ['user', 'admin']
      if (roles.includes('admin')) {
        api.idToken.setCustomClaim('folder', 'admin');
      }
    };
    ```
  </Step>
</Steps>
