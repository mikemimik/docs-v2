<div id="get-started">
  ## Primeros pasos
</div>

Esta guía de inicio rápido muestra cómo agregar autenticación de Auth0 a una aplicación de Laravel. Configurarás un inicio de sesión seguro, cierre de sesión, rutas protegidas y acceso al perfil de usuario mediante el [SDK de Auth0 para Laravel](https://github.com/auth0/laravel-auth0).

<Steps>
  <Step title="Crear un nuevo proyecto de Laravel" stepNumber={1}>
    Si ya tienes una aplicación Laravel, ve al paso 2.

    Crea un nuevo proyecto de Laravel:

    ```bash
    composer create-project laravel/laravel auth0-laravel-app
    ```

    Abre el directorio del proyecto:

    ```bash
    cd auth0-laravel-app
    ```
  </Step>

  <Step title="Instala el SDK de Auth0 para Laravel" stepNumber={2}>
    Ejecuta el siguiente comando en el directorio del proyecto para instalar el [SDK de Auth0 para Laravel](https://github.com/auth0/laravel-auth0):

    ```bash
    composer require auth0/login:^7 --update-with-all-dependencies
    ```

    A continuación, publica el archivo de configuración del SDK:

    ```bash
    php artisan vendor:publish --tag auth0
    ```
  </Step>

  <Step title="Configurar credenciales de Auth0" stepNumber={3}>
    Necesitas crear una aplicación de Auth0 y añadir tus credenciales al proyecto. Elige uno de los siguientes métodos:

    <Tabs>
      <Tab title="Configuración rápida (recomendada)">
        Crea una Auth0 App y copia las credenciales precompletadas de `.env` con los valores de configuración correctos.

        <CreateInteractiveApp placeholderText="Laravel" appType="regular_web" allowedCallbackUrls={["http://localhost:8000/callback"]} allowedLogoutUrls={["http://localhost:8000"]} />

        Añade estos valores al archivo `.env` de tu proyecto:

        <AuthCodeBlock children={envSnippet} language="shellscript" filename=".env" />
      </Tab>

      <Tab title="CLI">
        Instala la [Auth0 CLI](https://github.com/auth0/auth0-cli#installation) e inicia sesión en tu cuenta:

        <CodeGroup>
          ```bash Mac
          brew tap auth0/auth0-cli && brew install auth0
          auth0 login
          ```

          ```powershell Windows
          scoop bucket add auth0 https://github.com/auth0/scoop-auth0-cli.git
          scoop install auth0
          auth0 login
          ```
        </CodeGroup>

        Crea una aplicación de Auth0 y una API, y guarda las credenciales en archivos JSON que el SDK lee automáticamente:

        ```bash
        auth0 apps create \
          --name "My Laravel Application" \
          --type "regular" \
          --auth-method "post" \
          --callbacks "http://localhost:8000/callback" \
          --logout-urls "http://localhost:8000" \
          --reveal-secrets \
          --no-input \
          --json > .auth0.app.json

        auth0 apis create \
          --name "My Laravel Application's API" \
          --identifier "https://github.com/auth0/laravel-auth0" \
          --offline-access \
          --no-input \
          --json > .auth0.api.json
        ```

        <Note>
          Estos archivos contienen tus credenciales. Agrégalos a `.gitignore` para mantenerlos fuera del control de versiones:

          ```bash
          echo ".auth0.*.json" >> .gitignore
          ```
        </Note>
      </Tab>

      <Tab title="Dashboard">
        1. Ve al [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; Applications**
        2. Haz clic en **Create Application**, introduce un nombre, selecciona **Regular Web Application** y haz clic en **Create**
        3. Ve a la pestaña **Settings** y copia **Domain**, **Client ID** y **Client Secret**

        Configura las siguientes URL en la pestaña **Settings** de tu aplicación:

        | Campo                 | Valor                            |
        | --------------------- | -------------------------------- |
        | Allowed Callback URLs | `http://localhost:8000/callback` |
        | Allowed Logout URLs   | `http://localhost:8000`          |

        Haz clic en **Save Changes**.

        Después, añade tus credenciales al archivo `.env` de tu proyecto:

        ```bash .env
        AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
        AUTH0_CLIENT_ID=YOUR_AUTH0_CLIENT_ID
        AUTH0_CLIENT_SECRET=YOUR_AUTH0_CLIENT_SECRET
        ```

        <Warning>
          Nunca subas tu `AUTH0_CLIENT_SECRET` al control de versiones. Guárdalo en `.env` y asegúrate de que `.env` esté en tu `.gitignore`.
        </Warning>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Añadir rutas de autenticación" stepNumber={4}>
    El SDK de Auth0 registra automáticamente las siguientes rutas para tu aplicación; no se necesita ninguna configuración adicional de rutas:

    | Route       | Purpose                                                     |
    | ----------- | ----------------------------------------------------------- |
    | `/login`    | Inicia el flujo de inicio de sesión de Auth0                |
    | `/logout`   | Cierra la sesión del usuario y redirige a Auth0             |
    | `/callback` | Gestiona la devolución de llamada de autenticación de Auth0 |

    <Note>
      Si tu aplicación usa Laravel Breeze, Fortify o Jetstream, las rutas `/login`, `/logout` y `/callback` del SDK pueden entrar en conflicto con las rutas registradas por esos paquete. Consulta el [SDK README](https://github.com/auth0/laravel-auth0) para obtener instrucciones sobre el registro manual de rutas.
    </Note>

    Actualiza `routes/web.php` para añadir tu ruta de inicio:

    ```php routes/web.php lines
    <?php

    use Illuminate\Support\Facades\Route;

    Route::get('/', function () {
        if (! auth()->check()) {
            return response('You are not logged in. <a href="/login">Log in</a>');
        }

        $user = auth()->user();
        $name = $user->name ?? 'User';
        $email = $user->email ?? '';

        return response("Hello {$name}! Your email address is {$email}.");
    });
    ```
  </Step>

  <Step title="Protege las rutas con middleware" stepNumber={5}>
    Usa el middleware `auth` de Laravel para exigir autenticación en cualquier ruta. También puedes exigir [permisos](https://auth0.com/docs/manage-users/access-control/rbac) específicos mediante el middleware `can`:

    ```php routes/web.php lines
    <?php

    use Illuminate\Support\Facades\Route;

    // Requiere cualquier usuario autenticado
    Route::get('/private', function () {
        return response('Welcome! You are logged in.');
    })->middleware('auth');

    // Requiere autenticación Y el permiso 'read:messages'
    Route::get('/scope', function () {
        return response('You have the read:messages permission.');
    })->middleware('auth')->can('read:messages');
    ```

    <Info>
      Los permisos se definen en la configuración de tu Auth0 API y se asignan a los usuarios mediante roles. Consulta [Role-Based Access Control](https://auth0.com/docs/manage-users/access-control/rbac) para más detalles.
    </Info>
  </Step>

  <Step title="Ejecuta tu aplicación" stepNumber={6}>
    ```bash
    php artisan serve
    ```

    <Info>
      Tu aplicación se ejecuta en [http://localhost:8000](http://localhost:8000). Si el puerto 8000 ya está en uso, ejecuta `php artisan serve --port=8001` y actualiza las **Allowed Callback URLs** y **Allowed Logout URLs** de tu aplicación en Auth0 para usar el nuevo puerto.
    </Info>
  </Step>
</Steps>

<Check>
  **Verificación**

  Abre [http://localhost:8000](http://localhost:8000) en tu navegador. Prueba estas rutas para verificar tu integración:

  * [http://localhost:8000/login](http://localhost:8000/login) — inicia el flujo de inicio de sesión de Auth0
  * [http://localhost:8000/private](http://localhost:8000/private) — redirige al inicio de sesión si no has iniciado sesión; muestra un mensaje de bienvenida cuando la has iniciado
  * [http://localhost:8000/logout](http://localhost:8000/logout) — cierra la sesión y vuelve a la página de inicio
</Check>

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="Error por discrepancia en la URL de devolución de llamada">
    **Causa:** La URL de devolución de llamada enviada a Auth0 no coincide con ninguna URL de la lista **Allowed Callback URLs** de tu aplicación.

    **Solución:**

    1. Ve al [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; Applications** → tu aplicación → **Settings**
    2. Agrega `http://localhost:8000/callback` a **Allowed Callback URLs**
    3. Haz clic en **Save Changes**

    Asegúrate de que no haya barras diagonales al final y de que el puerto coincida con el de tu servidor en ejecución.
  </Accordion>

  <Accordion title="Error de invalid state / CSRF después del inicio de sesión">
    **Causa:** Hay un problema de configuración de la sesión o de la cookie: la sesión de Laravel no está conservando el valor de state entre la redirección de inicio de sesión y la devolución de llamada.

    **Solución:**

    1. Asegúrate de que `SESSION_DRIVER` en `.env` esté configurado como `file`, `database` o `redis` (no `array`)
    2. Borra la configuración y la caché:

    ```bash
    php artisan config:clear && php artisan cache:clear
    ```

    3. Reinicia tu servidor de desarrollo
  </Accordion>

  <Accordion title="AUTH0_DOMAIN no está configurado o no se cargan las credenciales">
    **Causa:** El SDK no puede encontrar tus credenciales de Auth0.

    **Solución:** Asegúrate de que exista uno de los siguientes archivos en la raíz de tu proyecto:

    * Un archivo `.env` con `AUTH0_DOMAIN`, `AUTH0_CLIENT_ID` y `AUTH0_CLIENT_SECRET`
    * Un archivo `.auth0.app.json` generado por Auth0 CLI

    Después de actualizar `.env`, borra la caché de configuración:

    ```bash
    php artisan config:clear
    ```
  </Accordion>

  <Accordion title="404 en /login, /logout o /callback">
    **Causa:** Las rutas del SDK no están registradas, normalmente porque no se cargó el proveedor de servicios.

    **Solución:**

    1. Confirma que ejecutaste `php artisan vendor:publish --tag auth0`
    2. Verifica que la detección automática del paquete esté habilitada (revisa `composer.json` para `"dont-discover": []`)
    3. Ejecuta `php artisan route:list | grep auth0` para confirmar que las rutas estén registradas
    4. Si faltan rutas, registra manualmente el proveedor de servicios. En Laravel 11+, agrégalo a `bootstrap/providers.php`:

    ```php bootstrap/providers.php
    return [
        App\Providers\AppServiceProvider::class,
        Auth0\Laravel\Auth0ServiceProvider::class,
    ];
    ```
  </Accordion>
</AccordionGroup>

<div id="advanced-usage">
  ## Uso avanzado
</div>

<AccordionGroup>
  <Accordion title="Management API — actualizar metadatos del usuario">
    Puedes actualizar la información del usuario con la [Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). Se puede acceder a todos los endpoints de la Management API mediante el método `Auth0::management()` del SDK.

    **Antes de hacer llamadas a la Management API**, autoriza tu aplicación para acceder a la Management API:

    1. Ve al [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; APIs** → **Auth0 Management API**
    2. Selecciona la pestaña **Machine to Machine Applications**
    3. Autoriza tu aplicación de Laravel y concede los scopes `read:users` y `update:users`

    ```php routes/web.php lines
    <?php

    use Auth0\Laravel\Facade\Auth0;
    use Illuminate\Support\Facades\Route;

    Route::get('/colors', function () {
        $colors = ['red', 'blue', 'green', 'black', 'white', 'yellow', 'purple', 'orange'];

        $users = Auth0::management()->users();

        // Actualiza los metadatos del usuario autenticado con un color favorito aleatorio
        $users->update(
            id: auth()->id(),
            body: [
                'user_metadata' => [
                    'color' => $colors[random_int(0, count($colors) - 1)],
                ],
            ]
        );

        // Recupera y muestra los metadatos actualizados
        $metadata = Auth0::json($users->get(auth()->id()));
        $color = $metadata['user_metadata']['color'] ?? 'desconocido';
        $name = auth()->user()->name;

        return response("¡Hola {$name}! Tu color favorito es {$color}.");
    })->middleware('auth');
    ```

    Puedes consultar una referencia completa de todos los métodos de la Management API en la [documentación del SDK](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md).
  </Accordion>

  <Accordion title="Modelos y repositorios de usuario personalizados">
    El SDK admite modelos y repositorios de usuario personalizados, lo que te permite almacenar y recuperar usuarios desde tu propia base de datos mientras mantienes Auth0 como proveedor de identidad.

    Consulta [User Repositories and Models](https://github.com/auth0/laravel-auth0/blob/main/docs/Users.md) para ver la guía completa de implementación.
  </Accordion>

  <Accordion title="Escuchar eventos del SDK">
    El SDK genera eventos en momentos clave del ciclo de vida de la autenticación —durante el login, el cierre de sesión, la renovación del token y más—, lo que te permite personalizar por completo el comportamiento sin modificar el código principal del SDK.

    Consulta [Hooking Events](https://github.com/auth0/laravel-auth0/blob/main/docs/Events.md) para ver una lista completa de eventos y ejemplos de implementación.
  </Accordion>

  <Accordion title="Control de acceso basado en roles (RBAC)">
    Usa Auth0 RBAC para asignar permisos a los usuarios mediante roles y luego aplicarlos en Laravel con el middleware `can`:

    ```php routes/web.php
    <?php

    use Illuminate\Support\Facades\Route;

    Route::get('/admin', function () {
        return response('Admin area.');
    })->middleware('auth')->can('admin:dashboard');
    ```

    Para configurar RBAC:

    1. Ve al [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; APIs** → tu API → **Permissions**
    2. Agrega los permisos que necesita tu aplicación (por ejemplo, `admin:dashboard`, `read:messages`)
    3. Ve a **User Management &gt; Roles**, crea un rol y asígnale permisos
    4. Asigna el rol a los usuarios desde su página de perfil

    Consulta [Role-Based Access Control](https://auth0.com/docs/manage-users/access-control/rbac) para ver la guía completa.
  </Accordion>
</AccordionGroup>
