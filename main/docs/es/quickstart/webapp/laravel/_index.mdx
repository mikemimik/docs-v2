  <Content>
    [Auth0&#39;s Laravel SDK](https://github.com/auth0/laravel-auth0)
    te permite agregar rápidamente autenticación, gestión de perfil de usuario y control de acceso por rutas a tu
    aplicación Laravel. Esta guía muestra cómo integrar Auth0 con una aplicación [Laravel 9
    o 10](https://github.com/auth0/laravel-auth0#support-policy) nueva (o existente).

    Probar cambio

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      **Si aún no tienes configurada una aplicación de Laravel**, abre un shell en un directorio adecuado para el nuevo
      proyecto y ejecuta el siguiente comando:

      ```sh wrap lines
      composer create-project --prefer-dist laravel/laravel auth0-laravel-app ^9.0
      ```

      Todos los comandos de esta guía asumen que los estás ejecutando desde la raíz del directorio de tu proyecto de Laravel, así que
      debes hacer `cd` al directorio del nuevo proyecto:

      ```sh lines
      cd auth0-laravel-app
      ```

      <LoggedInForm />
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Ejecuta el siguiente comando en el directorio de tu proyecto para instalar el [SDK de Auth0 para Laravel](https://github.com/auth0/laravel-auth0):

      ```sh wrap lines
      composer require auth0/login:^7.8 --update-with-all-dependencies
      ```

      A continuación, genera un archivo de configuración del SDK para tu aplicación:

      ```sh lines
      php artisan vendor:publish --tag auth0
      ```

      <LoggedInForm />
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Ejecute el siguiente comando en el directorio de su proyecto para descargar Auth0 CLI:

      ```sh wrap lines
      curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
      ```

      Luego, autentica la CLI con tu cuenta de Auth0 y, cuando se te solicite, elige &quot;como usuario&quot;:

      ```sh lines
      ./auth0 login
      ```

      A continuación, crea una nueva aplicación en Auth0:

      ```sh lines
      ./auth0 apps create \
        --name "My Laravel Application" \
        --type "regular" \
        --auth-method "post" \
        --callbacks "http://localhost:8000/callback" \
        --logout-urls "http://localhost:8000" \
        --reveal-secrets \
        --no-input \
        --json > .auth0.app.json
      ```

      También deberías crear una nueva API:

      ```sh lines
      ./auth0 apis create \
        --name "My Laravel Application's API" \
        --identifier "https://github.com/auth0/laravel-auth0" \
        --offline-access \
        --no-input \
        --json > .auth0.api.json
      ```

      Esto genera dos archivos en el directorio de tu proyecto que configuran el SDK.

      Dado que estos archivos contienen credenciales, es importante tratarlos como información confidencial. Debes asegurarte de no
      incluirlos en el control de versiones. Si usas Git, debes agregarlos a tu archivo `.gitignore`:

      ```sh lines
      echo ".auth0.*.json" >> .gitignore
      ```

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4" isSingleColumn>
      El SDK registra automáticamente todas las rutas necesarias para que los usuarios de su aplicación se autentiquen.

      Si necesita un mayor control sobre ellas o si entran en conflicto con las rutas existentes de su aplicación, puede
      registrar manualmente los controladores del SDK. Consulte [el README del SDK](https://github.com/auth0/laravel-auth0) para obtener información sobre integraciones avanzadas.

      <LoggedInForm />
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5" isSingleColumn>
      Las funcionalidades de autenticación de Laravel usan &quot;guards&quot; para definir cómo se autentica a los usuarios en cada solicitud. Puedes
      usar el guard de autenticación del SDK de Auth0 para restringir el acceso a las rutas de tu aplicación.

      Para exigir que los usuarios se autentiquen antes de acceder a una ruta, puedes usar el middleware `auth` de Laravel.

      ```text lines
      Route::get('/private', function () {
        return response('Welcome! You are logged in.');
      })->middleware('auth');
      ```

      También puede exigir que los usuarios autenticados tengan [permisos](https://auth0.com/docs/manage-users/access-control/rbac) específicos al combinar esto
      con el middleware `can` de Laravel.

      ```text lines
      Route::get('/scope', function () {
        return response('You have `read:messages` permission, and can therefore access this resource.');

      })->middleware('auth')->can('read:messages');
      ```

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6" isSingleColumn>
      Se puede acceder a la información sobre el usuario autenticado mediante la fachada `Auth` de Laravel o la
      función auxiliar `auth()`.

      Por ejemplo, para obtener el identificador y la dirección de correo electrónico del usuario:

      ```text lines
      Route::get('/', function () {
        if (! auth()->check()) {
        return response('You are not logged in.');

        }
        $user = auth()->user();
        $name = $user->name ?? 'User';
        $email = $user->email ?? '';
        return response("Hello {$name}! Your email address is {$email}.");
      });;
      ```

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      Puede actualizar la información del usuario mediante la [Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). Se puede acceder a todos los endpoints de Management a través del método
      `management()` del SDK.

      **Antes de realizar llamadas a la Management API, debe habilitar su aplicación para que se comunique con la Management
      API.** Esto puede hacerse desde la [página de API del Auth0 Dashboard](https://manage.auth0.com/dashboard/us/dev-gja8kxz4ndtex3rq/apis/), eligiendo
      `Auth0 Management API` y seleccionando la pestaña &#39;Aplicaciones de máquina a máquina&#39;. Autorice su aplicación
      Laravel y, a continuación, haga clic en la flecha desplegable para elegir los scopes que desea conceder.

      Para el siguiente ejemplo, en el que actualizaremos los metadatos de un usuario y le asignaremos un color favorito aleatorio,
      debería conceder los scopes `read:users` y `update:users`. Puede encontrar una lista de los endpoints de la API y los
      scopes requeridos en [la documentación de la Management
      API](https://auth0.com/docs/api/management/v2).

      ```text lines
      use Auth0\Laravel\Facade\Auth0;
      Route::get('/colors', function () {
      $endpoint = Auth0::management()->users();
      $colors = ['red', 'blue', 'green', 'black', 'white', 'yellow', 'purple', 'orange', 'pink', 'brown'];
      $endpoint->update(
      id: auth()-&gt;id(),

      body: [

          'user_metadata' =&gt; [

              'color' =&gt; $colors[random_int(0, count($colors) - 1)]

          ]

      ]

      );
      $metadata = $endpoint->get(auth()->id());
      $metadata = Auth0::json($metadata);
      $color = $metadata['user_metadata']['color'] ?? 'unknown';
      $name = auth()->user()->name;
      return response("Hello {$name}! Your favorite color is {$color}.");
      })->middleware('auth');
      ```

      Una guía de consulta rápida con todos los métodos de la Management API del SDK está [disponible aquí](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md).
