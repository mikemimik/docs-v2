  <Content>
    Auth0 te permite agregar autenticación a casi cualquier tipo de aplicación de forma rápida. Esta guía muestra cómo
    integrar Auth0, añadir autenticación y mostrar información del perfil de usuario en cualquier aplicación PHP con el Auth0 PHP
    SDK.

    Para usar esta guía de inicio rápido, necesitarás:

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitarás tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente único y alfanumérico
      que el código de tu aplicación usará para llamar a las API de Auth0 mediante el SDK.

      Cualquier configuración que realices con este Quickstart se actualizará automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones en el futuro.

      Si prefieres explorar una configuración completa, puedes ver una aplicación de ejemplo.

      ### Configurar URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como
        `http://localhost:3000/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como `http://localhost:3000`.
      </Info>

      ### Configurar orígenes web permitidos

      Un origen web permitido es una URL a la que quieres permitir el acceso a tu flujo de autenticación. Debe
      contener la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar en silencio
      los tokens de autenticación, por lo que tus usuarios cerrarán sesión la próxima vez que visiten tu aplicación o actualicen la
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como `http://localhost:3000`.
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 proporciona un [SDK de
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y la autorización de Auth0 en aplicaciones PHP.

      El SDK de PHP de Auth0 requiere tener instaladas bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) para gestionar las peticiones de red. Si
      no tienes bibliotecas disponibles, puedes instalar opciones fiables ejecutando los siguientes comandos en tu
      terminal:

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora instala el SDK de PHP de Auth0 ejecutando el siguiente comando en tu terminal:

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Configura el SDK de Auth0

      Crea un archivo nuevo en tu aplicación llamado `index.php` y copia en él el código del panel
      interactivo de la derecha, en la pestaña **index.php**.

      Para que el SDK funcione correctamente, debes establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `domain`: El dominio de tu tenant de Auth0. Por lo general, puedes encontrarlo en el Auth0 Dashboard,
        en Settings de tu aplicación, en el campo Domain. Si usas un [dominio personalizado](https://auth0.com/docs/custom-domains), debes establecer aquí el
        valor de ese dominio personalizado.
      * `clientId`: El ID de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes encontrarlo
        en el Auth0 Dashboard, en Settings de tu aplicación, en el campo ID de cliente.
      * `clientSecret`: El secreto de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes
        encontrarlo en el Auth0 Dashboard, en Settings de tu aplicación, en el campo secreto de cliente.
      * `redirectUri`: La URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que
        se hayan autenticado. Esto corresponde a la URL de devolución de llamada que configuraste antes en este Quickstart. También puedes
        encontrar este valor en el Auth0 Dashboard, en Settings de tu aplicación, en el campo Callback URLs. Asegúrate de que
        lo que introduzcas en tu código coincida con lo que configuraste antes; de lo contrario, tus usuarios verán un error.
      * `cookieSecret`: Un valor secreto largo que se usa para cifrar la cookie de sesión. Puedes generar una
        cadena adecuada ejecutando `openssl rand -hex 32` en tu terminal.

      <Info>
        ##### Checkpoint

        Tu SDK de Auth0 ahora debería estar configurado correctamente. Ejecuta tu aplicación para comprobar que:

        * El SDK se inicializa correctamente.
        * Tu aplicación no muestra ningún error relacionado con Auth0.
      </Info>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Ahora instala una biblioteca de enrutamiento para dirigir las solicitudes entrantes a tu aplicación. No es un paso obligatorio,
      pero simplifica la estructura de la aplicación para este Quickstart.

      ```text lines
      composer require steampixel/simple-php-router
      ```

      Crea un archivo nuevo en tu aplicación llamado `router.php` para definir las rutas y copia el código
      del panel interactivo de la derecha.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Ahora que ha configurado su aplicación de Auth0 y el SDK de Auth0 para PHP, necesita configurar el inicio de sesión para su
      proyecto. Para ello, usará el método `login()` del SDK para crear un botón de inicio de sesión que redirija
      a los usuarios a la página de Auth0 Universal Login. Después de que un usuario se autentique correctamente, se le redirigirá a la
      URL de devolución de llamada que configuró antes en este Quickstart.

      Cree un archivo nuevo en su aplicación llamado `login.php` para gestionar el
      proceso de inicio de sesión y copie en él el código del panel interactivo de la derecha, que contiene la lógica necesaria para iniciar sesión.

      <Info>
        ##### Checkpoint

        Ahora debería poder iniciar sesión o registrarse con un nombre de usuario y una contraseña.

        Haga clic en el enlace de inicio de sesión y verifique que:

        * Su aplicación le redirige a la página de Auth0 Universal Login.
        * Puede iniciar sesión o registrarse.
        * Auth0 le redirige de nuevo a su aplicación usando el valor de `redirectUri` que usó
          para configurar el SDK.
      </Info>
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Los usuarios que inician sesión en tu proyecto también necesitarán una forma de cerrar sesión. Gestionaremos un botón de cierre de sesión usando el método
      `logout()` del SDK. Cuando los usuarios cierren sesión, se les redirigirá al endpoint de [logout de Auth0](https://auth0.com/docs/api/authentication?http#logout), que
      luego los redirigirá inmediatamente a la logout URL que configuraste antes en este Quickstart.

      Crea un archivo nuevo en tu aplicación llamado `logout.php` para gestionar el proceso y copia en él el
      código del panel interactivo, que contiene la lógica necesaria para cerrar sesión. Después, actualiza tu archivo
      `index.php` para incluir el botón de cierre de sesión.

      <Info>
        ##### Checkpoint

        Ejecuta tu aplicación y haz clic en el botón de cierre de sesión; verifica que:

        * Tu aplicación te redirija a la dirección que especificaste como una de las Allowed Logout URLs en
          Application Settings.
        * Ya no hayas iniciado sesión en tu aplicación.
      </Info>
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Ahora que sus usuarios pueden iniciar sesión y cerrar sesión, probablemente quiera poder recuperar la [información del perfil](https://auth0.com/docs/users/concepts/overview-user-profile)
      asociada a los usuarios autenticados. Por ejemplo, puede que quiera mostrar el nombre de un usuario que ha iniciado sesión o su
      foto de perfil en su proyecto.

      El SDK de Auth0 para PHP proporciona información del usuario mediante el método `getCredentials()`. Revise el
      código de `profile.php` en el panel interactivo para ver un ejemplo de cómo usarlo.

      Como el método contiene información confidencial relacionada con la identidad del usuario, su disponibilidad depende del
      estado de autenticación del usuario. Para evitar errores al renderizar, siempre debe comprobar si el método
      `getCredentials()` devuelve un `object` o `null` para determinar si
      Auth0 ha autenticado al usuario antes de que su aplicación use los resultados.

      <Info>
        ##### Punto de control

        Verifique que:

        * Puede mostrar `nickname` o cualquier otra propiedad del usuario correctamente después de haber iniciado
          sesión.
      </Info>
    </Section>
