<div id="configure-auth0">
  ## Configurar Auth0
</div>

<div id="get-your-application-keys">
  ### Obtén las claves de tu aplicación
</div>

Cuando te registraste en Auth0, se creó una nueva aplicación para ti, o bien pudiste haber creado una nueva. Necesitarás algunos datos de esa aplicación para comunicarte con Auth0. Puedes obtenerlos en la sección [Configuración de la aplicación](https://manage.auth0.com/#/applications) del Auth0 Dashboard.

<Frame>![Dashboard de la aplicación](https://cdn2.auth0.com/docs/1.14550.0/media/articles/dashboard/client_settings.png)</Frame>

Necesitas la siguiente información:

* **Domain**
* **ID de cliente**
* **secreto de cliente**

<Info>
  Si descargas el ejemplo de la parte superior de esta página, estos datos ya estarán completos.
</Info>

<div id="configure-callback-urls">
  ### Configurar las URL de devolución de llamada
</div>

Una URL de devolución de llamada es una URL de tu aplicación a la que Auth0 redirige al usuario después de autenticarse. La URL de devolución de llamada de tu aplicación debe añadirse al campo **Allowed Callback URLs** en [Configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán iniciar sesión en la aplicación y verán un error.

<Info>
  Si estás siguiendo el proyecto de ejemplo que descargaste desde la parte superior de esta página, la URL de devolución de llamada que debes agregar al campo **Allowed Callback URLs** es `http://127.0.0.1:3000/`.
</Info>

<div id="configure-logout-urls">
  ### Configurar Logout URLs
</div>

Una URL de cierre de sesión es una URL de tu aplicación a la que Auth0 puede redirigir después de que el usuario haya cerrado sesión en el servidor de autorización. Esto se especifica en el parámetro de consulta `returnTo`. La URL de cierre de sesión de tu aplicación debe agregarse al campo **Allowed Logout URLs** en [Configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán cerrar sesión en la aplicación y recibirán un error.

<Info>
  Si estás usando el proyecto de ejemplo que descargaste desde la parte superior de esta página, la URL de cierre de sesión que debes agregar al campo **Allowed Logout URLs** es `http://127.0.0.1:3000`.
</Info>

<div id="integrate-your-php-application">
  ## Integra tu aplicación PHP
</div>

Vamos a crear una aplicación de ejemplo que autentique a un usuario con una aplicación PHP. Aquí seguiremos un enfoque sencillo, adecuado para este formato escrito. Aun así, te recomendamos consultar la [aplicación Quickstart complementaria en GitHub](https://github.com/auth0-samples/auth0-php-web-app/) para ver un ejemplo más completo.

<div id="installing-http-client-and-messaging-factories">
  ### Instalación del cliente HTTP y las fábricas de mensajes
</div>

El SDK de Auth0 para PHP es compatible con muchos estándares de [PHP-FIG](https://www.php-fig.org) que ofrecen opciones de interoperabilidad con tu arquitectura. Dos de especial importancia son [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/). Estos estándares te permiten integrar los componentes de red que prefieras para gestionar mensajes y solicitudes. Deberás instalar bibliotecas compatibles en tu proyecto para que el SDK pueda utilizarlas.

La biblioteca de red más utilizada para PHP es [Guzzle](https://docs.guzzlephp.org/), aunque hay muchas opciones disponibles en la comunidad de PHP. Usemos Guzzle para esta aplicación de ejemplo. Una vez más, desde el directorio de tu proyecto, ejecuta el siguiente comando de shell:

```bash lines
composer require guzzlehttp/guzzle guzzlehttp/psr7 http-interop/http-factory-guzzle
```

<div id="installing-the-php-sdk">
  ### Instalación del SDK para PHP
</div>

El SDK de Auth0 para PHP requiere [Composer](https://getcomposer.org/doc/00-intro.md#installation-linux-unix-macos), una herramienta de gestión de dependencias en PHP. Composer te permite declarar las bibliotecas de las que depende tu proyecto y las instala por ti. Asegúrate de que Composer esté instalado y accesible desde tu shell antes de continuar.

Ejecuta el siguiente comando en la shell dentro del directorio de tu proyecto para instalar el SDK de Auth0 para PHP:

```bash lines
composer require auth0/auth0-php
```

Esto creará una carpeta `vendor` dentro de tu proyecto y descargará todas las dependencias necesarias para usar el SDK de Auth0 para PHP. También creará un archivo `vendor/autoload.php`, que se usa en el ejemplo para cargar todas las clases necesarias para que tu aplicación funcione. Es importante que incluyas este archivo de carga automática en tu proyecto para que el SDK funcione.

<div id="configuring-the-sdk">
  ### Configuración del SDK
</div>

Para comenzar, vamos a crear un archivo `.env` en la raíz del directorio de tu proyecto para almacenar la configuración de nuestra aplicación de ejemplo y completar las variables de entorno:

export const codeExample = `# El ID de cliente de tu aplicación de Auth0
AUTH0_CLIENT_ID={yourClientId}

# La URL del dominio de tu tenant de Auth0
AUTH0_DOMAIN={yourDomain}

# El secreto de cliente de tu aplicación de Auth0
AUTH0_CLIENT_SECRET={yourClientSecret}

# Un valor largo y secreto que se utiliza para cifrar la cookie de sesión.
# Puedes generarlo con \`openssl rand -hex 32\` desde tu shell.
AUTH0_COOKIE_SECRET=

# La URL desde la que se puede acceder a tu aplicación. Actualízala según corresponda.
AUTH0_BASE_URL=http://127.0.0.1:3000`;

<AuthCodeBlock children={codeExample} language="env" />

Como PHP no puede leer por sí solo nuestro archivo `.env`, necesitaremos instalar una biblioteca que nos ayude. Aunque para nuestra aplicación de ejemplo usaremos una biblioteca concreta, en una aplicación real servirá cualquier cargador de &#39;dotenv&#39; que prefieras. Desde el directorio de nuestro proyecto, ejecutemos el siguiente comando en la shell para instalar la biblioteca:

```bash lines
composer require vlucas/phpdotenv
```

A continuación, vamos a crear el archivo fuente de PHP que usaremos en estos ejemplos de código, `index.php`, y a configurar una instancia del SDK de Auth0 para PHP para nuestra aplicación de ejemplo:

```php lines
<?php

// Importar el Autoloader de Composer para hacer accesibles las clases del SDK:
require 'vendor/autoload.php';

// Cargar las variables de entorno desde el archivo .env:
(Dotenv\Dotenv::createImmutable(__DIR__))->load();

// Ahora instanciar la clase Auth0 con nuestra configuración:
$auth0 = new \Auth0\SDK\Auth0([
    'domain' => $_ENV['AUTH0_DOMAIN'],
    'clientId' => $_ENV['AUTH0_CLIENT_ID'],
    'clientSecret' => $_ENV['AUTH0_CLIENT_SECRET'],
    'cookieSecret' => $_ENV['AUTH0_COOKIE_SECRET']
]);
```

<div id="setting-up-your-application-routes">
  ### Configuración de las rutas de tu aplicación
</div>

Las aplicaciones modernas de PHP usan un sistema de enrutamiento para dirigir las solicitudes HTTP entrantes al código que las procesa, lo que determina qué debe ocurrir cuando los usuarios visitan una &quot;página&quot; de nuestra aplicación. No existe una única forma correcta de implementar el enrutamiento en tu aplicación, y hay muchas bibliotecas entre las que puedes elegir para hacerlo. Para nuestra aplicación de ejemplo usaremos una biblioteca en particular, pero en una aplicación real puedes elegir la que prefieras.

Desde el directorio de nuestro proyecto, ejecutemos el siguiente comando de shell para instalar la biblioteca de enrutamiento:

```bash lines
composer require steampixel/simple-php-router
```

A continuación, volvamos a abrir nuestro `index.php` y demos vida a nuestra aplicación. Empieza importando la biblioteca de enrutamiento y definiendo las URL completas de las rutas de nuestra aplicación de ejemplo como constantes con nombre, para mayor comodidad, ya que necesitaremos hacer referencia a ellas en varios lugares de la aplicación:

```php lines
// 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo index.php.

// Importa nuestra biblioteca de enrutamiento:
use Steampixel\Route;

// Define las constantes de ruta:
define('ROUTE_URL_INDEX', rtrim($_ENV['AUTH0_BASE_URL'], '/'));
define('ROUTE_URL_LOGIN', ROUTE_URL_INDEX . '/login');
define('ROUTE_URL_CALLBACK', ROUTE_URL_INDEX . '/callback');
define('ROUTE_URL_LOGOUT', ROUTE_URL_INDEX . '/logout');
```

Ahora podemos pasar a añadir la lógica de enrutamiento de nuestra aplicación y las integraciones del SDK:

<div id="checking-for-a-session">
  ## Comprobar si hay una sesión
</div>

El SDK de Auth0 para PHP tiene un método práctico para comprobar si nuestro usuario se ha autenticado y se ha recuperado su perfil, `getCredentials()`. Añadamos esto a nuestra ruta principal para mostrar el perfil del usuario si ha iniciado sesión o indicar que necesita iniciar sesión.

```php lines
// 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo index.php.

Route::add('/', function() use ($auth0) {
  $session = $auth0->getCredentials();

  if ($session === null) {
    // El usuario no ha iniciado sesión.
    echo '<p>Please <a href="/docs/login">log in</a>.</p>';
    return;
  }

  // El usuario ha iniciado sesión.
  echo '<pre>';
  print_r($session->user);
  echo '</pre>';

  echo '<p>You can now <a href="/docs/logout">log out</a>.</p>';
});
```

Podemos acceder a todas las propiedades del perfil de nuestro usuario desde la propiedad `user` de la respuesta, que es un arreglo. Así, por ejemplo, podemos obtener el nickname del usuario usando `$session->user<a href="https://auth0.com/docs/users/user-profile-structure" target="_blank" rel="noreferrer">nickname`, o su dirección de correo electrónico desde `$session->useremail`. La estructura es un perfil de usuario normalizado, sobre el que puedes obtener más información [aquí](https://auth0.com/docs/users/normalized-user-profiles).

Es importante tener en cuenta que el contenido del perfil de usuario variará en función del proveedor social o los proveedores sociales que uses, por lo que nunca debes dar por hecho que un valor concreto siempre estará ahí dentro de la lógica de tu aplicación. Usa construcciones del lenguaje PHP como `isset` o la coalescencia nula para manejar correctamente la presencia o ausencia de un valor, por ejemplo:

```php lines
// ✋ No es necesario incluir esto en nuestra aplicación de ejemplo, es solo un ejemplo.
$name = $session->user['name'] ?? $session->user['nickname'] ?? $session->user['email'] ?? 'Unknown';
```

<div id="logging-in">
  ## Iniciar sesión
</div>

Ahora vamos a crear nuestra ruta /login, que usará el método `login()` del SDK de Auth0 para PHP para configurar la sesión del usuario y devolver una URL personalizada de la Universal Login Page de Auth0 para que el usuario inicie sesión.

```php lines
// 👆 Continuamos desde los pasos anteriores. Añade esto a tu archivo index.php.

Route::add('/login', function() use ($auth0) {
    // Es buena práctica restablecer las sesiones de usuario cada vez que inicien sesión para evitar errores de "invalid state", en caso de que surjan problemas de red u otros inconvenientes que interrumpan un proceso de inicio de sesión anterior:
    $auth0->clear();

    // Por último, configura la sesión local de la aplicación y redirige al usuario a la Universal Login Page de Auth0 para que se autentique.
    header("Location: " . $auth0->login(ROUTE_URL_CALLBACK));
    exit;
});
```

<div id="handling-authentication-callback">
  ## Gestión de la devolución de llamada de autenticación
</div>

Después de que nuestros usuarios regresen tras autenticarse en la Universal Login Page de Auth0, volverán a nuestra aplicación de ejemplo a través de nuestra ruta de devolución de llamada, `/callback`, que gestionaremos en este paso.

Cuando Auth0 nos devuelve a los usuarios, incluye algunos parámetros esenciales en la consulta de la solicitud HTTP. El método `exchange()` del SDK de Auth0 para PHP se encarga de procesarlos, por lo que completar nuestro flujo de autenticación es sencillo:

```php lines
// 👆 Continuamos desde los pasos anteriores. Añade esto a tu archivo index.php.

Route::add('/callback', function() use ($auth0) {
    // Permite que el SDK complete el flujo de autenticación:
    $auth0->exchange(ROUTE_URL_CALLBACK);

    // Por último, redirige al usuario final de vuelta a la ruta / index para mostrar su perfil:
    header("Location: " . ROUTE_URL_INDEX);
    exit;
});
```

<div id="logging-out">
  ## Cierre de sesión
</div>

Por último, pero no por ello menos importante, vamos a gestionar correctamente el cierre de sesión de nuestros usuarios. El método `logout()` del SDK de Auth0 para PHP se encarga de eliminar las cookies de sesión de nuestra aplicación de ejemplo, redirigir al usuario al [endpoint /logout](https://auth0.com/docs/logout) de Auth0 (que cierra la sesión de la capa de sesión de Auth0 y de cualquier capa de sesión del proveedor de identidad) y luego devolver al usuario a nuestra ruta principal /.

```php lines
// 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo index.php.

Route::add('/logout', function() use ($auth0) {
    // Borra la sesión local del usuario en nuestra app y redirigelo al endpoint de cierre de sesión de Auth0 para borrar su sesión de Auth0.
    header("Location: " . $auth0->logout(ROUTE_URL_INDEX));
    exit;
});
```

<div id="run-your-app">
  ## ¡Ejecuta tu aplicación!
</div>

Por último, necesitamos indicarle a nuestro middleware de enrutamiento que efectivamente enrute las solicitudes:

```php lines
// 👆 Continuamos desde los pasos anteriores. Añade esto a tu archivo index.php.

// Esto le indica a nuestro router que hemos terminado de configurar nuestras rutas y que estamos listos para comenzar a enrutar las solicitudes HTTP entrantes:
Route::run('/');
```

¡Eso es todo! Ya puedes ejecutar tu nueva aplicación; una vez más, desde el directorio de tu proyecto, ejecuta el siguiente comando en la shell:

```bash lines
php -S 127.0.0.1:3000 index.php
```

Abre [http://127.0.0.1:3000](https://127.0.0.1:3000) en tu navegador y pruébala.
