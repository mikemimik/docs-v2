      Para usar los servicios de Auth0, necesitarás tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente único y alfanumérico
      que el código de tu aplicación usará para llamar a las API de Auth0 mediante el SDK.

      Cualquier configuración que hagas con este Quickstart se actualizará automáticamente en tu aplicación del [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones más adelante.

      Si prefieres ver una configuración completa, puedes consultar una aplicación de ejemplo.

      ### Configurar URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como
        `http://localhost:3000/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión desde tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como `http://localhost:3000`.
      </Info>

      ### Configurar orígenes web permitidos

      Un origen web permitido es una URL a la que quieres permitir acceso dentro de tu flujo de autenticación. Debe
      contener la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar silenciosamente
      los tokens de autenticación, por lo que tus usuarios cerrarán sesión la próxima vez que visiten tu aplicación o actualicen la
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 proporciona un [SDK de
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y autorización de Auth0 en aplicaciones PHP.

      El SDK de Auth0 para PHP requiere tener instaladas bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) para gestionar las solicitudes de red. Si
      no dispones de estas bibliotecas, puedes instalar opciones fiables ejecutando los siguientes comandos en tu
      terminal:

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora, instala el SDK de Auth0 para PHP ejecutando el siguiente comando en la terminal:

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Configurar el SDK de Auth0

      Cree un archivo nuevo en su aplicación llamado `index.php` y copie en él el código del panel interactivo
      de la derecha, en la pestaña **index.php**.

      Para que el SDK funcione correctamente, debe establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `domain`: El dominio de su tenant de Auth0. Por lo general, puede encontrarlo en el Auth0 Dashboard,
        en Settings de su aplicación, en el campo Domain. Si está usando un [dominio personalizado](https://auth0.com/docs/custom-domains), debe establecer esta propiedad con el
        valor de su dominio personalizado.
      * `clientId`: El ID de la aplicación de Auth0 que configuró antes en esta guía de inicio rápido. Puede encontrarlo
        en el Auth0 Dashboard, en Settings de su aplicación, en el campo ID de cliente.
      * `clientSecret`: El secreto de la aplicación de Auth0 que configuró antes en esta guía de inicio rápido. Puede
        encontrarlo en el Auth0 Dashboard, en Settings de su aplicación, en el campo Client Secret.
      * `redirectUri`: La URL de su aplicación a la que quiere que Auth0 redirija a los usuarios después de
        autenticarse. Esto corresponde a la URL de devolución de llamada que configuró antes en esta guía de inicio rápido. También puede
        encontrar este valor en el Auth0 Dashboard, en Settings de su aplicación, en el campo Callback URLs. Asegúrese de
        que lo que introduzca en el código coincida con lo que configuró antes; de lo contrario, sus usuarios verán un error.
      * `cookieSecret`: Un valor secreto largo que se utiliza para cifrar la cookie de sesión. Puede generar una
        cadena adecuada ejecutando `openssl rand -hex 32` en su terminal.

      Ahora que tus usuarios pueden iniciar y cerrar sesión, probablemente querrás poder recuperar la [información del perfil](https://auth0.com/docs/users/concepts/overview-user-profile)
      asociada a los usuarios autenticados. Por ejemplo, puede que quieras mostrar el nombre de un usuario que ha iniciado sesión o su
      foto de perfil en tu proyecto.

      El SDK de Auth0 para PHP proporciona información del usuario mediante el método `getCredentials()`. Revisa el código de
      `profile.php` en el panel interactivo para ver un ejemplo de cómo usarlo.

      Como el método contiene información confidencial relacionada con la identidad del usuario, su disponibilidad depende del
      estado de autenticación del usuario. Para evitar errores de renderización, siempre debes comprobar si el método
      `getCredentials()` devuelve un `object` o `null` para determinar si
      Auth0 ha autenticado al usuario antes de que tu aplicación use los resultados.
