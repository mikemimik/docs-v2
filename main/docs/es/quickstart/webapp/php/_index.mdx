      ### Configura una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente

      ### Configura las URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como
        `http://localhost:3000/callback`.
      </Info>

      ### Configura las URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

      ### Configura los orígenes web permitidos

      Un origen web permitido es una URL a la que quieres permitir el acceso dentro de tu flujo de autenticación. Debe
      incluir la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar de forma silenciosa
      los tokens de autenticación, por lo que tus usuarios cerrarán sesión la próxima vez que visiten tu aplicación o actualicen una
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 ofrece un [SDK de
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y la autorización de Auth0 en aplicaciones PHP.

      El SDK de Auth0 para PHP requiere que se instalen bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) para gestionar las solicitudes de red. Si
      no dispones de estas bibliotecas, puedes instalar opciones confiables ejecutando los siguientes comandos en tu
      terminal:

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora instala el SDK de PHP para Auth0 ejecutando el siguiente comando en tu terminal:

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Configura el SDK de Auth0

      Crea un archivo nuevo en tu aplicación llamado `index.php` y copia en él el código del panel
      interactivo de la derecha, en la pestaña **index.php**.

      Para que el SDK funcione correctamente, debes establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `domain`: El dominio de tu tenant de Auth0. Por lo general, puedes encontrarlo en el Auth0 Dashboard
        en Settings de tu aplicación, en el campo Domain. Si utilizas un [dominio personalizado](https://auth0.com/docs/custom-domains), debes establecer aquí el
        valor de ese dominio personalizado.
      * `clientId`: El ID de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes encontrarlo
        en el Auth0 Dashboard en Settings de tu aplicación, en el campo Client ID.
      * `clientSecret`: El secreto de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes
        encontrarlo en el Auth0 Dashboard en Settings de tu aplicación, en el campo Client Secret.
      * `redirectUri`: La URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que
        se autentiquen. Esto corresponde a la URL de devolución de llamada que configuraste antes en este Quickstart. También puedes
        encontrar este valor en el Auth0 Dashboard en Settings de tu aplicación, en el campo Callback URLs. Asegúrate
        de que lo que introduzcas en tu código coincida con lo que configuraste antes; de lo contrario, tus usuarios verán un error.
      * `cookieSecret`: Un valor secreto largo que se utiliza para cifrar la cookie de sesión. Puedes generar una
        cadena adecuada ejecutando `openssl rand -hex 32` en tu terminal.

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6" isSingleColumn>
      Ahora que tus usuarios pueden iniciar y cerrar sesión, seguramente querrás poder recuperar la [información del perfil](https://auth0.com/docs/users/concepts/overview-user-profile)
      asociada a los usuarios autenticados. Por ejemplo, puede que quieras mostrar el nombre o la
      foto de perfil de un usuario que haya iniciado sesión en tu proyecto.

      El SDK de PHP de Auth0 proporciona información del usuario mediante el método `getCredentials()`. Revisa el código de
      `profile.php` en el panel interactivo para ver un ejemplo de cómo usarlo.

      Dado que el método contiene información confidencial relacionada con la identidad del usuario, su disponibilidad depende del
      estado de autenticación del usuario. Para evitar errores de renderizado, siempre debes comprobar si el
      método `getCredentials()` devuelve un `object` o `null` para determinar si
      Auth0 ha autenticado al usuario antes de que tu aplicación use los resultados.

        ##### Punto de control
