      Para usar los servicios de Auth0, necesitarás tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente único y alfanumérico

      ### Configurar URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como
        `http://localhost:3000/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

      ### Configurar Allowed Web Origins

      Un Allowed Web Origin es una URL a la que quieres permitir el acceso dentro de tu flujo de autenticación. Debe
      contener la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar silenciosamente
      los tokens de autenticación, por lo que la sesión de tus usuarios se cerrará la próxima vez que visiten tu aplicación o recarguen una
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 proporciona un [SDK de Auth0 para
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y autorización de Auth0 en aplicaciones PHP.

      El SDK de Auth0 para PHP requiere tener instaladas bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) para gestionar solicitudes de red. Si
      no tienes bibliotecas disponibles, puedes instalar opciones fiables ejecutando los siguientes comandos en tu
      terminal:

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora, instala el SDK de Auth0 para PHP ejecutando el siguiente comando en tu terminal:

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Configura el SDK de Auth0

      Crea un nuevo archivo en tu aplicación llamado `index.php` y copia en él el código del panel interactivo
      de la derecha, en la pestaña **index.php**.

      Para que el SDK funcione correctamente, debes establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `domain`: El dominio de tu tenant de Auth0. Por lo general, puedes encontrarlo en el Auth0 Dashboard,
        en Settings de tu aplicación, en el campo Domain. Si estás usando un [dominio personalizado](https://auth0.com/docs/custom-domains), debes establecer este valor con el
        de tu dominio personalizado.
      * `clientId`: El ID de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes encontrarlo
        en el Auth0 Dashboard, en Settings de tu aplicación, en el campo Client ID.
      * `clientSecret`: El secreto de la aplicación de Auth0 que configuraste antes en este Quickstart. Puedes
        encontrarlo en el Auth0 Dashboard, en Settings de tu aplicación, en el campo Client Secret.
      * `redirectUri`: La URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de
        autenticarse. Esto corresponde a la URL de devolución de llamada que configuraste antes en este Quickstart. También puedes
        encontrar este valor en el Auth0 Dashboard, en Settings de tu aplicación, en el campo Callback URLs. Asegúrate de que
        lo que introduzcas en tu código coincida con lo que configuraste antes; de lo contrario, tus usuarios verán un error.
      * `cookieSecret`: Un valor secreto largo que se utiliza para cifrar la cookie de sesión. Puedes generar una
        cadena adecuada ejecutando `openssl rand -hex 32` en tu terminal.

      Ahora que sus usuarios pueden iniciar sesión y cerrar sesión, probablemente querrá poder recuperar la [información del perfil](https://auth0.com/docs/users/concepts/overview-user-profile)
      asociada a los usuarios autenticados. Por ejemplo, puede que quiera mostrar el nombre de un usuario que haya iniciado sesión o su
      foto de perfil en su proyecto.

      El SDK de Auth0 para PHP proporciona información del usuario mediante el método `getCredentials()`. Revise el archivo
      `profile.php` en el panel interactivo para ver un ejemplo de cómo usarlo.

      Como el método contiene información confidencial relacionada con la identidad del usuario, su disponibilidad depende del
      estado de autenticación del usuario. Para evitar errores al renderizar, siempre debe comprobar si el
      método `getCredentials()` devuelve un `object` o `null` para determinar si
      Auth0 ha autenticado al usuario antes de que su aplicación use los resultados.
