    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitarás tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación en Auth0 se le asigna un ID de cliente alfanumérico y único
      que el código de tu aplicación usará para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con este Quickstart se actualizará automáticamente para tu aplicación en el [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones en el futuro.

      Si prefieres explorar una configuración completa, puedes ver una aplicación de ejemplo.

      ### Configurar URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como
        `http://localhost:3000/auth/auth0/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

      ### Configurar orígenes web permitidos

      Un origen web permitido es una URL a la que quieres permitir el acceso a tu flujo de autenticación. Debe
      contener la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar silenciosamente los
      tokens de autenticación, por lo que se cerrará la sesión de tus usuarios la próxima vez que visiten tu aplicación o actualicen una
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Usa `omniauth-auth0`, una [estrategia personalizada de OmniAuth](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication), para gestionar el flujo de autenticación.

      Agrega las siguientes dependencias a tu `Gemfile`:

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # previene solicitudes de autenticación falsificadas
      ```

      Una vez añadidas tus gemas, instálalas con `bundle install`.
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Cree un archivo de configuración `./config/auth0.yml` para especificar los valores de su dominio de Auth0, el ID de cliente y el secreto del cliente, que encontrará en el Auth0 Dashboard, en **Settings** de la aplicación.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Cree el siguiente archivo de inicialización `./config/initializers/auth0.rb` y [configure](https://github.com/auth0/omniauth-auth0/blob/master/EXAMPLES.md#send-additional-authentication-parameters) el middleware **OmniAuth** con el archivo de configuración
      que creó en el paso anterior.

      Asegúrese de que `callback_path` coincida con el valor indicado en la configuración &quot;Allowed Callback URLs&quot; de su
      aplicación de Auth0.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Cree un controlador de Auth0 para gestionar la devolución de llamada de autenticación, la acción `logout` y los métodos para
      construir la URL de cierre de sesión.

      Ejecute el comando:
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

      Dentro del método callback, asigne a la sesión activa el hash con la información del usuario, que se devuelve como
      `request.env['omniauth.auth']`.

      Para configurar el cierre de sesión, elimine todos los objetos almacenados en la sesión llamando al método `reset_session`
      dentro de la acción `logout`. Después, redirija al endpoint de cierre de sesión de Auth0. Para obtener más información sobre
      `reset_session`, consulte la [documentación de ActionController de Ruby on Rails](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668).
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Agrega estas rutas a tu archivo `./config/routes.rb`.

      Las rutas deben estar configuradas para que Rails sepa cómo dirigir las distintas URL de devolución de llamada de Auth0 al controlador de Auth0 que
      creaste en el paso anterior.

      <Info>
        ##### Verificación

        Ejecuta tu aplicación para comprobar que sigue funcionando como se espera y que no recibes ningún error
        relacionado con Auth0.
      </Info>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Ahora, un usuario puede iniciar sesión en tu aplicación accediendo al endpoint `/auth/auth0`.

      <Warning>
        Para [evitar solicitudes de autenticación fraudulentas](https://github.com/cookpad/omniauth-rails_csrf_protection), usa los métodos auxiliares `link_to` o
        `button_to` con el método `:post`.
      </Warning>

      ```ruby lines
      <!-- Coloca un botón de inicio de sesión en cualquier lugar de tu aplicación -->
      <%= button_to 'Login', '/auth/auth0', method: :post %>
      ```

      <Info>
        ##### Punto de control

        Agrega un botón a tu aplicación que redirija al usuario al endpoint `/auth/auth0` cuando
        lo seleccione. Ten en cuenta que lo rediriges a Auth0 para iniciar sesión y luego de regreso a tu aplicación después de una
        autenticación correcta.
      </Info>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Ahora que ya puedes iniciar sesión en tu aplicación de Rails, necesitas una
      forma de cerrar sesión. Para cerrar la sesión de un usuario, redirígelo a la
      acción `auth/logout`, que lo redirigirá al endpoint de cierre de sesión de Auth0.

      <Info>
        Para probar esto después del paso anterior, puede que tengas que borrar tu sesión y luego redirigir al usuario
        al endpoint de cierre de sesión de Auth0.
      </Info>

      ```ruby lines
      <!-- Coloca un botón de cierre de sesión en cualquier lugar de tu aplicación -->
      <%= button_to 'Logout', 'auth/logout', method: :get %>
      ```

      <Info>
        ##### Checkpoint

        Añade un botón a tu aplicación que redirija al usuario al endpoint `/auth/logout` cuando lo
        seleccione. Comprueba que te redirige a Auth0 y, enseguida, de vuelta a tu aplicación, y que ya no
        tienes la sesión iniciada.
      </Info>
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      Para mostrar el perfil del usuario, tu aplicación debe proporcionar una ruta protegida. Puedes usar un [Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) para controlar el acceso a rutas que se comparten entre varios
      controladores. El concern debe redirigir automáticamente a Auth0 cuando el usuario no esté autenticado. De lo contrario, el
      concern debe devolver el perfil del usuario actual.

      Una vez que tengas un Concern, inclúyelo en cualquier controlador que requiera que el usuario haya iniciado sesión. Después, podrás acceder al
      usuario desde la sesión `session[:userinfo]`, como en el siguiente ejemplo:

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      Una vez cargado el usuario desde la sesión, úsalo para mostrar información en tu frontend:

      ```html lines
      <div>
      <p>Normalized User Profile:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>Full User Profile:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Info>
        ##### Punto de control

        Añade el concern `Secured` a tu aplicación y luego inclúyelo en el controlador al que solo puedan acceder usuarios
        autenticados. Comprueba que los usuarios autenticados tengan acceso a las acciones de ese
        controlador y que los usuarios no autenticados sean redirigidos a Auth0 para autenticarse.
      </Info>
    </Section>
