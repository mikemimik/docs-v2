      Para usar los servicios de Auth0, necesitarás tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación en el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una ya existente que represente
      el proyecto que quieres integrar. A cada aplicación en Auth0 se le asigna un ID de cliente alfanumérico y único
      que el código de tu aplicación usará para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con esta guía de inicio rápido se actualizará automáticamente en tu aplicación dentro del [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones en el futuro.

      Si prefieres explorar una configuración completa, también puedes consultar una aplicación de ejemplo.

      ### Configurar URLs de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como
        `http://localhost:3000/auth/auth0/callback`.
      </Info>

      ### Configurar URLs de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

      ### Configurar orígenes web permitidos

      Un origen web permitido es una URL a la que quieres permitir el acceso dentro de tu flujo de autenticación. Debe
      incluir la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar silenciosamente
      los tokens de autenticación, por lo que tus usuarios cerrarán sesión la próxima vez que visiten tu aplicación o actualicen la
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúralo como `http://localhost:3000`.
      </Info>

      Usa `omniauth-auth0`, una [estrategia personalizada de OmniAuth](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication), para gestionar el flujo de autenticación.

      Agrega las siguientes dependencias a tu `Gemfile`:

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # evita solicitudes de autenticación falsificadas
      ```

      Una vez añadidas las gems, instálalas con `bundle install`.

      Cree un controlador de Auth0 para gestionar la devolución de llamada de autenticación, la acción `logout` y los métodos para
      construir la URL de cierre de sesión.

      Ejecute el comando:
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

      Dentro del método callback, asigne el hash de la información del usuario, devuelta como
      `request.env['omniauth.auth']`, a la sesión activa.

      Para configurar el cierre de sesión, borre todos los objetos almacenados en la sesión llamando al método `reset_session`
      dentro de la acción `logout`. Luego, redirija al endpoint de cierre de sesión de Auth0. Para obtener más información sobre
      `reset_session`, consulte la [documentación de Ruby on Rails ActionController](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668).

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9" isSingleColumn>
      Para mostrar el perfil del usuario, tu aplicación debe proporcionar una ruta protegida. Puedes usar un [Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) para controlar el acceso a rutas que pueden compartirse entre varios
      controladores. El concern debe redirigir automáticamente a Auth0 cuando el usuario no esté autenticado. En caso contrario, el
      concern debe devolver el perfil del usuario actual.

      Una vez que tengas un Concern, inclúyelo en cualquier controlador que requiera que el usuario haya iniciado sesión. Luego, puedes acceder al
      usuario desde la sesión `session[:userinfo]`, como en el siguiente ejemplo:

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      Una vez que cargues al usuario desde la sesión, úsalo para mostrar información en tu frontend:

      ```html lines
      <div>
      <p>Perfil de usuario normalizado:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>Perfil de usuario completo:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Note>
        ##### Punto de control

        Agrega el concern `Secured` a tu aplicación y luego inclúyelo en el controlador que requiera un
        usuario autenticado para poder acceder. Verifica que un usuario autenticado tenga acceso a las acciones de ese
        controlador y que los usuarios no autenticados sean redirigidos a Auth0 para autenticarse.
      </Note>
