      Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente único y alfanumérico
      que el código de tu aplicación usará para llamar a las API de Auth0 a través del SDK.

      Cualquier ajuste que configures con esta guía de inicio rápido se actualizará automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones más adelante.

      Si prefieres ver una configuración completa, puedes consultar una aplicación de ejemplo.

      ### Configurar URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en
        `http://localhost:3000/auth/auth0/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en `http://localhost:3000`.
      </Info>

      ### Configurar orígenes web permitidos

      Un origen web permitido es una URL a la que quieres permitir acceso en tu flujo de autenticación. Debe
      incluir la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar en segundo plano
      los tokens de autenticación, por lo que tus usuarios cerrarán sesión la próxima vez que visiten tu aplicación o actualicen una
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en `http://localhost:3000`.
      </Info>

      Usa `omniauth-auth0`, una [estrategia de OmniAuth](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication) personalizada, para gestionar el flujo de autenticación.

      Agrega las siguientes dependencias a tu `Gemfile`:

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # evita solicitudes de autenticación falsificadas
      ```

      Una vez que hayas añadido tus gems, instálalas con `bundle install`.

      Crea un controlador de Auth0 para manejar la devolución de llamada de autenticación, la acción `logout` y los métodos para
      construir la URL de cierre de sesión.

      Ejecuta el comando:
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

      Dentro del método callback, asigna el hash con la información del usuario, que se devuelve como
      `request.env['omniauth.auth']`, a la sesión activa.

      Para configurar el cierre de sesión, borra todos los objetos almacenados en la sesión llamando al método `reset_session`
      dentro de la acción `logout`. Luego, redirige al endpoint de cierre de sesión de Auth0. Para obtener más información sobre
      `reset_session`, consulta la [documentación de Ruby on Rails ActionController](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668).

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9" isSingleColumn>
      Para mostrar el perfil del usuario, tu aplicación debe proporcionar una ruta protegida. Puedes usar un [Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) para controlar el acceso a rutas que pueden compartirse entre varios
      controladores. El concern debe redirigir automáticamente a Auth0 cuando el usuario no esté autenticado. En caso contrario, el
      concern debe devolver el perfil del usuario actual.

      Una vez que tengas un Concern, inclúyelo en cualquier controlador que requiera un usuario autenticado. Luego podrás acceder al
      usuario desde la sesión `session[:userinfo]`, como en el siguiente ejemplo:

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      Una vez cargado el usuario desde la sesión, úsalo para mostrar información en tu frontend:

      ```html lines
      <div>
      <p>Normalized User Profile:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>Full User Profile:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Note>
        ##### Punto de control

        Agrega el concern `Secured` a tu aplicación y luego inclúyelo en el controlador que requiere que el usuario esté
        autenticado para acceder. Verifica que un usuario autenticado tenga acceso a las acciones de ese
        controlador y que los usuarios no autenticados sean redirigidos a Auth0 para la autenticación.
      </Note>
