    Auth0 te permite agregar autenticación y acceder a la información del perfil de usuario en tu aplicación. Esta guía
    muestra cómo integrar Auth0 con una aplicación Python [Django](https://www.djangoproject.com/) mediante el SDK [Authlib](https://authlib.org/).

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Para usar los servicios de Auth0, necesitarás tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente alfanumérico y único
      que el código de tu aplicación usará para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con este Quickstart se actualizará automáticamente para tu aplicación en el [Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), donde podrás administrar tus aplicaciones más adelante.

      Si prefieres explorar una configuración completa, también puedes ver una aplicación de ejemplo.

      ### Configurar las URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como
        `http://localhost:3000/callback`.
      </Info>

      ### Configurar las URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como `http://localhost:3000`.
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Para esta integración, añadirás varias dependencias, como la biblioteca Authlib. Crea un
      archivo `requirements.txt` en el directorio de tu proyecto e incluye lo siguiente:

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      Ejecute el siguiente comando desde su shell para que estas dependencias queden disponibles:

      ```text lines
      pip install -r requirements.txt
      ```

      A continuación, crea un archivo `.env` en el directorio de tu proyecto. Este archivo guardará las claves de tu cliente y otros
      detalles de configuración.

      ```text lines
      AUTH0_CLIENT_ID={yourClientId}
      AUTH0_CLIENT_SECRET={yourClientSecret}
      AUTH0_DOMAIN={yourDomain}
      ```

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      En este ejemplo, agregarás cuatro rutas para tu aplicación: las rutas de login, devolución de llamada, logout e index.

      * `login` - Cuando los visitantes de tu aplicación visiten la ruta `/login`, llegarán a Auth0 para
        comenzar el flujo de autenticación.
      * `callback` - Después de que tus usuarios terminen de iniciar sesión con Auth0, volverán a tu aplicación
        en la ruta `/callback`. Esta ruta guarda la sesión del usuario y evita que tenga que
        iniciar sesión de nuevo cuando regrese.
      * `logout` - La ruta `/logout` cierra la sesión de los usuarios en tu aplicación. Esta ruta borra
        la sesión del usuario en tu aplicación y redirige al endpoint de logout de Auth0 para garantizar que la sesión ya no
        se conserve. Luego, la aplicación redirige al usuario a la ruta de inicio.
      * `index` - La ruta de inicio renderizará los detalles de un usuario autenticado o permitirá a los visitantes iniciar sesión.
