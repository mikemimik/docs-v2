  <Content>
    Auth0 te permite agregar autenticación y acceder a la información del perfil de usuario en tu aplicación. En esta guía
    se muestra cómo integrar Auth0 con una aplicación Python [Django](https://www.djangoproject.com/) mediante el SDK [Authlib](https://authlib.org/).

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación en el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente único y alfanumérico
      que el código de tu aplicación usará para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con este Quickstart se actualizará automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), donde podrás administrar tus Applications en el futuro.

      Si prefieres revisar una configuración completa, puedes ver una aplicación de ejemplo.

      ### Configurar URL de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como
        `http://localhost:3000/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como `http://localhost:3000`.
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Para esta integración, agregarás varias dependencias de bibliotecas, como Authlib. Crea un
      archivo `requirements.txt` en el directorio de tu proyecto e incluye lo siguiente:

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      Ejecuta el siguiente comando en tu shell para que estas dependencias estén disponibles:

      ```text lines
      pip install -r requirements.txt
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      A continuación, crea un archivo `.env` en el directorio de tu proyecto. Este archivo contendrá las claves de tu cliente y otros
      datos de configuración.

      ```text lines
      AUTH0_CLIENT_ID={yourClientId}
      AUTH0_CLIENT_SECRET={yourClientSecret}
      AUTH0_DOMAIN={yourDomain}
      ```
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Si ya tienes una aplicación de Django configurada, pasa al siguiente paso. Para un nuevo proyecto de aplicación, ejecuta el
      siguiente comando:

      ```text lines
      django-admin startproject webappexample
      ```

      Cambia a la nueva carpeta del proyecto:

      ```text lines
      cd webappexample
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Abre el archivo `webappexample/settings.py` para revisar los valores de `.env`.

      En la parte superior del archivo, agrega las importaciones de `os` y `dotenv`.

      A continuación, debajo de la definición de `BASE_DIR`, agrega la variable `TEMPLATE_DIR`.

      Después, busca la variable `TEMPLATES` y actualiza el valor de `DIRS` para añadir la
      cadena `TEMPLATE_DIR`. Esto determina la ruta de los archivos de plantilla, que crearás en un
      paso posterior. Mantén sin cambios cualquier otro contenido de este arreglo.

      Al final de este archivo, agrega el código para cargar la configuración de Auth0.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Para empezar a crear tu aplicación, abre el archivo `webappexample/views.py` en tu IDE.

      Importa todas las bibliotecas que necesita tu aplicación.

      Ahora puedes configurar Authlib para gestionar la autenticación de tu aplicación con Auth0.

      Obtén más información sobre las opciones de configuración disponibles para el método OAuth `register()` de Authlib en [su documentación.](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      En este ejemplo, agregarás cuatro rutas para tu aplicación: las rutas de login, callback, logout e index.

      * `login` - Cuando los visitantes de tu app visiten la ruta `/login`, llegarán a Auth0 para
        comenzar el flujo de autenticación.
      * `callback` - Después de que tus usuarios terminen de iniciar sesión con Auth0, volverán a tu aplicación
        en la ruta `/callback`. Esta ruta guarda la sesión del usuario y evita que tenga que
        iniciar sesión de nuevo cuando regrese.
      * `logout` - La ruta `/logout` cierra la sesión de los usuarios en tu aplicación. Esta ruta borra
        la sesión del usuario en tu app y redirige al endpoint de logout de Auth0 para garantizar que la sesión ya no
        quede guardada. Luego, la aplicación redirige al usuario a la ruta de inicio.
      * `index` - La ruta de inicio renderizará los detalles de un usuario autenticado o permitirá que los visitantes inicien sesión.
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Reemplaza el contenido de tu archivo `webappexample/urls.py` con el código de la derecha para vincularlo a
      estas nuevas rutas.

      Esto hará que las rutas `/login`, `/callback`, `/logout` y `/`
      apunten a los controladores correctos.
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      A continuación, crearás un archivo de plantilla que se utilizará en la ruta de la página de inicio.

      Crea un nuevo subdirectorio dentro de la carpeta `webappexample` llamado `templates` y crea
      un archivo `index.html`.

      El archivo `index.html` contendrá código de plantilla para mostrar la información del usuario si ha iniciado sesión o
      mostrarle un botón de inicio de sesión si no la ha iniciado.
    </Section>

    <Section id={sections[9].id} title={sections[9].title} stepNumber="10">
      ¡Ya estás listo para ejecutar tu aplicación! Desde el directorio de tu proyecto, abre una terminal y usa:

      ```text lines
      python3 manage.py migrate
      python3 manage.py runserver 3000
      ```

      Su aplicación ya debería estar lista para abrirse en su navegador en [http://localhost:3000](http://localhost:3000).

      <Info>
        ##### Comprobación

        Visite [http://localhost:3000](http://localhost:3000/) para comprobarlo. Debería encontrar un botón de inicio de sesión que lo redirija a
        Auth0 para iniciar sesión y luego de vuelta a su aplicación, donde podrá ver la información de su perfil.
      </Info>
    </Section>
