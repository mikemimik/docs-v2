<div id="configure-auth0">
  ## Configurar Auth0
</div>

<div id="get-your-application-keys">
  ### Obtén las claves de tu aplicación
</div>

Cuando te registraste en Auth0, se creó una aplicación nueva para ti, o es posible que hayas creado una nueva. Necesitarás algunos datos de esa aplicación para comunicarte con Auth0. Puedes obtenerlos en la sección [Configuración de la aplicación](https://manage.auth0.com/#/applications) del Dashboard de Auth0.

<Frame>![Dashboard de la aplicación](https://cdn2.auth0.com/docs/1.14550.0/media/articles/dashboard/client_settings.png)</Frame>

Necesitas la siguiente información:

* **Domain**
* **ID de cliente**
* **Secreto de cliente**

<Info>
  Si descargas el ejemplo que aparece en la parte superior de esta página, estos datos ya estarán completados.
</Info>

<div id="configure-callback-urls">
  ### Configurar URLs de devolución de llamada
</div>

Una URL de devolución de llamada es una URL de tu aplicación a la que Auth0 redirige al usuario después de autenticarse. La URL de devolución de llamada de tu aplicación debe añadirse al campo **Allowed Callback URLs** de [Configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán iniciar sesión en la aplicación y recibirán un error.

<Info>
  Si estás siguiendo el proyecto de ejemplo que descargaste desde la parte superior de esta página, la URL de devolución de llamada que debes agregar al campo **Allowed Callback URLs** es `http://localhost:3000/callback`.
</Info>

<div id="configure-logout-urls">
  ### Configurar Logout URLs
</div>

Una logout URL es una URL de tu aplicación a la que Auth0 puede redirigir después de que el usuario haya cerrado sesión en el servidor de autorización. Esto se especifica en el query parameter `returnTo`. La logout URL de tu aplicación debe añadirse al campo **Allowed Logout URLs** de [Configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán cerrar sesión en la aplicación y recibirán un error.

<Info>
  Si estás siguiendo el proyecto de ejemplo que descargaste de la parte superior de esta página, la logout URL que debes agregar al campo **Allowed Logout URLs** es `http://localhost:3000`.
</Info>

<div id="create-application">
  ## Crear aplicación
</div>

En esta guía te mostramos cómo integrar Auth0 con una aplicación de Python usando el [framework Django](https://www.djangoproject.com/) y [Authlib](https://authlib.org/). Comencemos por asegurarnos de que Django esté instalado en tu sistema. En tu shell, ejecuta el siguiente comando:

```bash lines
pip install django~=4.0
```

Si ya tienes configurada una aplicación de Python, puedes pasar al siguiente paso. De lo contrario, vamos a crear nuestro nuevo proyecto de aplicación. Desde tu shell, ejecuta el siguiente comando y cambia a la carpeta del nuevo proyecto:

```bash lines
django-admin startproject webappexample
cd webappexample
```

<div id="install-dependencies">
  ## Instala las dependencias
</div>

Para esta integración, necesitarás algunas dependencias, como la biblioteca Authlib. Crea un archivo `requirements.txt` en el directorio de tu proyecto e incluye lo siguiente:

```txt lines
# 📁 requirements.txt -----

authlib ~= 1.0
django ~= 4.0
python-dotenv ~= 0.19
requests ~= 2.27
```

Ahora debes ejecutar `pip install -r requirements.txt` desde tu shell para que estas dependencias estén disponibles en tu proyecto.

<div id="configure-your-env-file">
  ## Configura tu archivo `.env`
</div>

A continuación, crea un archivo `.env` en el directorio de tu proyecto. Este archivo almacenará las claves de tu cliente y otros datos de configuración.

export const codeExample = `# 📁 .env -----

<div id="update-settingspy">
  ## Actualiza settings.py
</div>

Tendrás que hacer algunos cambios menores en tu archivo `webappexample/settings.py` para leer esos valores de `.env`. Al principio del archivo, agrega estas importaciones:

```py lines
# 📁 webappexample/settings.py -----

import os
from dotenv import load_dotenv, find_dotenv
```

A continuación, debajo de la definición de &#39;BASE&#95;DIR&#39;, agrega lo siguiente:

```py lines
# 📁 webappexample/settings.py -----

# Agrega la siguiente línea después de 'BASE_DIR = ...'
TEMPLATE_DIR = os.path.join(BASE_DIR, "webappexample", "templates")
```

A continuación, busca la variable `TEMPLATES` y actualiza el valor de `DIRS` para agregar nuestra cadena `TEMPLATE_DIR`. Esto le indica a Django dónde buscar nuestros archivos de plantilla cuando los creemos. Mantén igual cualquier otro contenido de este arreglo.

```py lines
# 📁 webappexample/settings.py -----

TEMPLATES = [
    {
        # Deja las demás líneas como están; solo actualizamos `DIRS`.
        "DIRS": [TEMPLATE_DIR],
    },
]
```

Por último, al final de este archivo, agrega lo siguiente:

```py lines
# 📁 webappexample/settings.py -----

# Cargar el archivo de definición de entorno
ENV_FILE = find_dotenv()
if ENV_FILE:
    load_dotenv(ENV_FILE)


# Cargar la configuración de la aplicación Auth0 en memoria
AUTH0_DOMAIN = os.environ.get("AUTH0_DOMAIN")
AUTH0_CLIENT_ID = os.environ.get("AUTH0_CLIENT_ID")
AUTH0_CLIENT_SECRET = os.environ.get("AUTH0_CLIENT_SECRET")
```

<div id="setup-your-application">
  ## Configura tu aplicación
</div>

Ahora ya estás listo para empezar a escribir tu aplicación. Abre el archivo `webappexample/views.py` en el directorio de tu proyecto.

Empieza importando todas las bibliotecas que va a utilizar tu aplicación:

```py lines
# 📁 webappexample/views.py -----

import json
from authlib.integrations.django_client import OAuth
from django.conf import settings
from django.shortcuts import redirect, render
from django.urls import reverse
from urllib.parse import quote_plus, urlencode
```

Ahora puedes configurar Authlib para encargarse de la autenticación de tu aplicación con Auth0:

```py lines
# 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo webappexample/views.py.

oauth = OAuth()

oauth.register(
    "auth0",
    client_id=settings.AUTH0_CLIENT_ID,
    client_secret=settings.AUTH0_CLIENT_SECRET,
    client_kwargs={
        "scope": "openid profile email",
    },
    server_metadata_url=f"https://{settings.AUTH0_DOMAIN}/.well-known/openid-configuration",
)
```

Puedes consultar más información sobre las opciones de configuración disponibles para el método OAuth `register()` de Authlib en [su documentación.](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)

<div id="setup-your-route-handlers">
  ## Configura los controladores de ruta
</div>

Para esta demostración, agregaremos 4 rutas para tu aplicación: las rutas de login, devolución de llamada, cierre de sesión e índice.

<div id="triggering-authentication-with-login">
  ### Iniciar la autenticación con /login
</div>

Cuando los visitantes de tu aplicación accedan a la ruta `/login`, se les redirigirá a Auth0 para iniciar el flujo de autenticación.

```py lines
# 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo webappexample/views.py.

def login(request):
    return oauth.auth0.authorize_redirect(
        request, request.build_absolute_uri(reverse("callback"))
    )
```

<div id="finalizing-authentication-with-callback">
  ### Finalizar la autenticación con /callback
</div>

Después de que los usuarios terminen de iniciar sesión con Auth0, volverán a tu aplicación en la ruta `/callback`. Esta ruta se encarga de guardar la sesión del usuario para que, cuando vuelva a visitar la aplicación más adelante, no tenga que iniciar sesión de nuevo.

```py lines
# 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo webappexample/views.py.

def callback(request):
    token = oauth.auth0.authorize_access_token(request)
    request.session["user"] = token
    return redirect(request.build_absolute_uri(reverse("index")))
```

<div id="clearing-a-session-with-logout">
  ### Borrar una sesión con /logout
</div>

Como es de esperar, esta ruta se encarga de cerrar la sesión de un usuario en tu aplicación. Borrará la sesión del usuario en tu aplicación y lo redirigirá brevemente al endpoint de cierre de sesión de Auth0 para asegurarse de que su sesión se haya borrado por completo, antes de devolverlo a tu ruta principal (que se explica a continuación).

```py lines
# 👆 Continuamos desde los pasos anteriores. Añade esto a tu archivo webappexample/views.py.

def logout(request):
    request.session.clear()

    return redirect(
        f"https://{settings.AUTH0_DOMAIN}/v2/logout?"
        + urlencode(
            {
                "returnTo": request.build_absolute_uri(reverse("index")),
                "client_id": settings.AUTH0_CLIENT_ID,
            },
            quote_via=quote_plus,
        ),
    )
```

<div id="theres-no-place-like-home">
  ### No hay lugar como /home
</div>

Por último, tu ruta principal servirá para renderizar los detalles de un usuario autenticado o para ofrecer a los visitantes la opción de iniciar sesión.

```py lines
# 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo webappexample/views.py.

def index(request):
    return render(
        request,
        "index.html",
        context={
            "session": request.session.get("user"),
            "pretty": json.dumps(request.session.get("user"), indent=4),
        },
    )
```

<div id="register-your-routes">
  ### Registra tus rutas
</div>

Por último, tendrás que indicarle a Django cómo conectar estas rutas nuevas. Reemplaza el contenido de tu archivo `webappexample/urls.py` por lo siguiente:

```py lines
# 📁 webappexample/urls.py -----

from django.urls import path

from . import views

urlpatterns = [
    path("", views.index, name="index"),
    path("login", views.login, name="login"),
    path("logout", views.logout, name="logout"),
    path("callback", views.callback, name="callback"),
]
```

<div id="add-templates">
  ## Agregar plantillas
</div>

Ahora solo necesitamos crear los archivos de plantilla sencillos que se usan en las rutas mencionadas antes (durante las llamadas a `render()`).

Crea un nuevo subdirectorio dentro de la carpeta `webappexample` llamado `templates` y crea un archivo `index.html`:

```html lines
# 📁 webappexample/templates/index.html -----

<html>
  <head>
    <meta charset="utf-8" />
    <title>Auth0 Example</title>
  </head>
  <body>
    {% if session %}
    <h1>Welcome {{session.userinfo.name}}!</h1>
    <p><a href="{% url 'logout' %}">Logout</a></p>
    <div><pre>{{pretty}}</pre></div>
    {% else %}
    <h1>Welcome Guest</h1>
    <p><a href="{% url 'login' %}">Login</a></p>
    {% endif %}
  </body>
</html>
```

<div id="run-your-application">
  ## Ejecuta tu aplicación
</div>

Ya puedes ejecutar tu aplicación. Desde el directorio de tu proyecto, abre un shell y usa:

```bash lines
python3 manage.py migrate
python3 manage.py runserver 3000
```

Tu aplicación ya debería estar lista para abrirse en tu navegador en `http://localhost:3000`.
