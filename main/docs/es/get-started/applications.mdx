El término **aplicación** o **app** en Auth0 no implica ninguna característica de implementación en particular. Por ejemplo, puede ser una app nativa que se ejecuta en un dispositivo móvil, una aplicación de página única que se ejecuta en un navegador o una aplicación web tradicional que se ejecuta en un servidor.

Auth0 clasifica las aplicaciones según estas características:

* **Tipo de aplicación**: Para agregar autenticación a tu aplicación, debes registrarla en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y seleccionar uno de los siguientes tipos de aplicación:

  * **Aplicación web tradicional**: Aplicaciones web tradicionales que realizan la mayor parte de su lógica en el servidor (como Express.js o ASP.NET). Para aprender a configurar una aplicación web tradicional, consulta [Registrar aplicaciones web clásicas](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps).
  * **Aplicación web de página única (SPA)**: Aplicaciones de JavaScript que realizan la mayor parte de la lógica de su interfaz de usuario en un navegador web y se comunican con un servidor web principalmente mediante API (como AngularJS + Node.js o React). Para aprender a configurar una aplicación web de página única, consulta [Registrar aplicaciones web de página única](/docs/es/get-started/auth0-overview/create-applications/single-page-web-apps).
  * **Aplicación nativa**: Aplicaciones móviles o de escritorio que se ejecutan de forma nativa en un dispositivo (como iOS o Android). Para aprender a configurar una aplicación nativa, consulta [Registrar aplicaciones nativas](/docs/es/get-started/auth0-overview/create-applications/native-apps).
  * **Aplicación de máquina a máquina (M2M)**: Aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en tu backend. Normalmente, usarás esta opción si tienes un servicio que requiere acceso a una API. Para aprender a configurar una aplicación de máquina a máquina, consulta [Registrar aplicaciones de máquina a máquina](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps).
* **Seguridad de las credenciales**: Según la [especificación OAuth 2.0](https://tools.ietf.org/html/rfc6749#section-2.1), las aplicaciones pueden clasificarse como públicas o confidenciales; las aplicaciones confidenciales pueden almacenar credenciales de forma segura, mientras que las aplicaciones públicas no. Para obtener más información, consulta [Aplicaciones confidenciales y públicas](/docs/es/get-started/applications/confidential-and-public-applications).
* **Propiedad**: Que una app se clasifique como propia o de terceros depende de quién la posee y la controla. Las aplicaciones propias están controladas por la misma organización o persona propietaria del dominio de Auth0. Las aplicaciones de terceros permiten que terceros o partners accedan de forma segura a los recursos protegidos detrás de tu API. Para obtener más información, consulta [Aplicaciones propias y de terceros](/docs/es/get-started/applications/first-party-and-third-party-applications).

<div id="manage-applications-settings">
  ## Gestionar la configuración de las aplicaciones
</div>

Registra las aplicaciones en [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications/\{yourClientId}/settings). Además de configurar aplicaciones en el Dashboard, también puedes configurarlas de forma programática, como se describe en la especificación [OpenID Connect (OIDC) registro dinámico de clientes 1.0](https://openid.net/specs/openid-connect-registration-1_0.html).

Puedes crear una configuración más compleja que permita a los usuarios iniciar sesión de manera diferente en distintas aplicaciones. Para obtener más información, consulta [Multi-Tenant Application Best Practices](/docs/es/get-started/auth0-overview/create-tenants/multi-tenant-apps-best-practices) y [Create Multiple Tenants](/docs/es/get-started/auth0-overview/create-tenants/create-multiple-tenants).

De manera predeterminada, Auth0 habilita todas las conexiones asociadas a tu tenant cuando creas una nueva aplicación. Para cambiarlo, [actualiza las conexiones de la aplicación](/docs/es/get-started/applications/update-application-connections) en **Configuración de la aplicación** del Dashboard.

<div id="monitor-applications">
  ## Supervisar aplicaciones
</div>

Puede [supervisar aplicaciones](/docs/es/deploy-monitor/monitor/monitor-applications) y realizar pruebas de extremo a extremo mediante sus propias pruebas. Auth0 almacena [datos de registro](/docs/es/deploy-monitor/logs), incluidas las acciones de los administradores del Dashboard, las autenticaciones de usuario exitosas y fallidas, y las solicitudes de cambio de contraseña. Puede usar la transmisión de logs en [Auth0 Marketplace](https://marketplace.auth0.com/features/log-streaming) para exportar sus datos de registro y usar herramientas como Sumo Logic, Splunk o Mixpanel para analizarlos y almacenarlos.

<div id="remove-applications">
  ## Eliminar aplicaciones
</div>

Puedes [eliminar una aplicación](/docs/es/get-started/applications/remove-applications) desde el Dashboard o con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="manage-client-secrets">
  ## Gestionar secretos de cliente
</div>

Un <Tooltip tip="Servidor de autorización: servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip> es un secreto que solo conocen tu aplicación y el <Tooltip tip="Secreto de cliente: secreto que usa un cliente (aplicación) para autenticarse ante el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>. Protege tus recursos al conceder [tokens](/docs/es/secure/tokens) solo a quienes están autorizados a solicitarlos.

Protege tus secretos de cliente y **nunca** los incluyas en aplicaciones móviles o basadas en navegador. Si tu secreto de cliente alguna vez se ve comprometido, debes [sustituirlo por uno nuevo](/docs/es/get-started/applications/rotate-client-secret) y actualizar todas las aplicaciones autorizadas con el nuevo secreto de cliente.

<div id="grant-types">
  ## Tipos de concesión
</div>

Auth0 ofrece muchos tipos de concesión o flujos de autenticación y autorización, y te permite indicar qué tipos de concesión son adecuados según la propiedad `grant_types` de tu aplicación registrada en Auth0. Para obtener más información, lee [Tipos de concesión de aplicaciones](/docs/es/get-started/applications/application-grant-types).
