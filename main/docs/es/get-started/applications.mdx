El término **aplicación** o **app** en Auth0 no implica ninguna característica de implementación en particular. Por ejemplo, podría ser una app nativa que se ejecuta en un dispositivo móvil, una aplicación de página única que se ejecuta en un navegador o una aplicación web tradicional que se ejecuta en un servidor.

Auth0 categoriza las aplicaciones según estas características:

* **Tipo de aplicación**: Para agregar autenticación a tu aplicación, debes registrarla en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y seleccionar uno de los siguientes tipos de aplicación:

  * **Aplicación web tradicional**: Aplicaciones web tradicionales que ejecutan la mayor parte de su lógica en el servidor (como Express.js o ASP.NET). Para saber cómo configurar una aplicación web tradicional, consulta [Registrar aplicaciones web clásicas](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps).
  * **Aplicación web de página única (SPA)**: Aplicaciones de JavaScript que ejecutan la mayor parte de la lógica de su interfaz de usuario en un navegador web y se comunican con un servidor web principalmente mediante API (como AngularJS + Node.js o React). Para saber cómo configurar una aplicación web de página única, consulta [Registrar aplicaciones web de página única](/docs/es/get-started/auth0-overview/create-applications/single-page-web-apps).
  * **Aplicación nativa**: Aplicaciones móviles o de escritorio que se ejecutan de forma nativa en un dispositivo (como iOS o Android). Para saber cómo configurar una aplicación nativa, consulta [Registrar aplicaciones nativas](/docs/es/get-started/auth0-overview/create-applications/native-apps).
  * **Aplicación máquina a máquina (M2M)**: Aplicaciones no interactivas, como herramientas de línea de comandos, daemons, dispositivos IoT o servicios que se ejecutan en tu backend. Normalmente, usarás esta opción si tienes un servicio que requiere acceso a una API. Para saber cómo configurar una aplicación máquina a máquina, consulta [Registrar aplicaciones máquina a máquina](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps).
* **Seguridad de las credenciales**: Según la [especificación OAuth 2.0](https://tools.ietf.org/html/rfc6749#section-2.1), las aplicaciones pueden clasificarse como públicas o confidenciales; las aplicaciones confidenciales pueden almacenar credenciales de forma segura, mientras que las aplicaciones públicas no. Para obtener más información, consulta [Aplicaciones confidenciales y públicas](/docs/es/get-started/applications/confidential-and-public-applications).
* **Propiedad**: Que una app se clasifique como propia o de terceros depende de quién la posea y la controle. Las aplicaciones propias están controladas por la misma organización o persona que posee el dominio de Auth0. Las aplicaciones de terceros permiten que partes externas o partners accedan de forma segura a recursos protegidos detrás de tu API. Para obtener más información, consulta [Aplicaciones propias y de terceros](/docs/es/get-started/applications/first-party-and-third-party-applications).

<div id="manage-applications-settings">
  ## Administrar la configuración de las aplicaciones
</div>

Registra las aplicaciones en [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications/\{yourClientId}/settings). Además de configurarlas en el Dashboard, también puedes configurar las aplicaciones de forma programática, como se describe en la especificación [registro dinámico de clientes de OpenID Connect (OIDC) 1.0](https://openid.net/specs/openid-connect-registration-1_0.html).

Puedes establecer una configuración más compleja para que los usuarios inicien sesión de forma diferente según la aplicación. Para obtener más información, lee [Prácticas recomendadas para aplicaciones multi-tenant](/docs/es/get-started/auth0-overview/create-tenants/multi-tenant-apps-best-practices) y [Crear varios tenants](/docs/es/get-started/auth0-overview/create-tenants/create-multiple-tenants).

De forma predeterminada, Auth0 habilita todas las conexiones asociadas a tu tenant cuando creas una nueva aplicación. Para cambiarlo, [actualiza las conexiones de la aplicación](/docs/es/get-started/applications/update-application-connections) en **Configuración de la aplicación** del Dashboard.

<div id="monitor-applications">
  ## Supervisar aplicaciones
</div>

Puede [supervisar aplicaciones](/docs/es/deploy-monitor/monitor/monitor-applications) y realizar pruebas de extremo a extremo con sus propias pruebas. Auth0 almacena [datos de registro](/docs/es/deploy-monitor/logs), incluidas las acciones de administradores del Dashboard, las autenticaciones de usuario correctas y fallidas, y las solicitudes de cambio de contraseña. Puede usar la transmisión de logs en [Auth0 Marketplace](https://marketplace.auth0.com/features/log-streaming) para exportar sus datos de registro y usar herramientas como Sumo Logic, Splunk o Mixpanel para analizarlos y almacenarlos.

<div id="remove-applications">
  ## Eliminar aplicaciones
</div>

Puedes [eliminar una aplicación](/docs/es/get-started/applications/remove-applications) desde el Dashboard o con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="manage-client-secrets">
  ## Administrar secretos de cliente
</div>

Un <Tooltip tip="Servidor de autorización: servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, su servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip> es un secreto que solo conocen su aplicación y el <Tooltip tip="Secreto de cliente: secreto que utiliza un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>. Protege sus recursos al conceder [tokens](/docs/es/secure/tokens) solo a entidades autorizadas.

Proteja sus secretos de cliente y **nunca** los incluya en aplicaciones móviles o de navegador. Si su secreto de cliente se ve comprometido en algún momento, debe [rotarlo por uno nuevo](/docs/es/get-started/applications/rotate-client-secret) y actualizar todas las aplicaciones autorizadas con el nuevo secreto de cliente.

<div id="grant-types">
  ## Tipos de concesión
</div>

Auth0 ofrece diversos tipos de concesión o flujos de autenticación y autorización, y te permite indicar qué tipos de concesión son adecuados en función de la propiedad `grant_types` de tu aplicación registrada en Auth0. Para obtener más información, consulta [Tipos de concesión de aplicaciones](/docs/es/get-started/applications/application-grant-types).
