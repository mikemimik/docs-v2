Usa interacciones en lenguaje natural con el [servidor MCP de Auth0](/docs/es/get-started/auth0-mcp-server) para configurar y gestionar rápidamente un sistema completo de autorización de API. Aprovechando la capacidad de [Claude](https://claude.ai/) para interactuar con Auth0 mediante el Model Context Protocol, puedes implementar patrones de autorización complejos en minutos en lugar de horas.

<div id="step-1-explore-current-api-configuration">
  ### Paso 1: Explora la configuración actual de la API
</div>

Antes de crear nuevas API, es importante entender qué ya está configurado en tu tenant. Con el servidor MCP de Auth0, puedes obtener esta información rápidamente mediante lenguaje natural.

Pregúntale a Claude:

```bash wrap lines
Muéstrame todos los resource servers (APIs) configurados actualmente en mi tenant de Auth0.
```

Claude usará la herramienta `auth0_list_resource_servers` para obtener y mostrar tus API existentes. Esto te ofrece una visión general rápida de tus API sin tener que navegar por el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>.

<div id="step-2-create-new-api-definitions">
  ### Paso 2: Crear nuevas definiciones de API
</div>

Ahora que comprendes tu configuración actual, puedes crear nuevas API para tu arquitectura de microservicios. En lugar de configurar cada API por separado desde el Dashboard, puedes definirlas todas de una sola vez.

Pregúntale a Claude:

```bash lines
I need to create three new APIs for our microservices:
1. An inventory API with read and write scopes
2. An orders API with read and write scopes
3. A customers API with read, write, and admin scopes

Each API should enforce scope validation.
```

Claude usará la herramienta `auth0_create_resource_server` varias veces para configurar estas API.

<div id="step-3-implement-authorization-logic">
  ### Paso 3: Implementar la lógica de autorización
</div>

Después de definir tus API, tendrás que implementar una lógica de autorización personalizada. Con el servidor MCP de Auth0, puedes describir tus reglas de negocio en lenguaje natural y Claude generará el código adecuado para las [Actions](/docs/es/customize/actions) de Auth0.

Pregúntale a Claude:

```bash lines
Crea un Action que aplique las siguientes reglas de autorización:
1. Verifica que la aplicación solicitante tenga los scopes adecuados para la API a la que se accede.
2. Para la API de clientes, las solicitudes que usen el scope de administrador solo deben provenir de nuestros IP ranges internos (10.0.0.0/8 y 172.16.0.0/12).
3. Registra todas las decisiones de autorización para fines de auditoría.
```

Claude usará la herramienta `auth0_create_action` para generar y crear una Action con el código correspondiente.

<div id="step-4-deploy-the-authorization-action">
  ### Paso 4: Despliega la Action de autorización
</div>

Cuando estés conforme con el código de la Action, puedes desplegarla en tu entorno de producción con una sola instrucción.

Pídele a Claude:

```bash lines
Deploy this action to run during the token issuance process.
```

Claude usará la herramienta `auth0_deploy_action` para implementar la Action.

<div id="step-5-configure-client-applications">
  ### Paso 5: Configurar las aplicaciones cliente
</div>

A continuación, deberás configurar tus aplicaciones cliente para acceder a las API correspondientes con los scopes correctos.

Pregúntale a Claude:

```bash wrap lines
Create an 'inventory-manager' application on my Auth0 tenant and update the app to access the inventory and orders APIs with read and write permissions, but not the customers API.
```

Claude primero usará `auth0_get_application` para obtener la configuración actual y luego usará `auth0_update_application` para actualizarla.

<div id="step-6-verify-configuration">
  ### Paso 6: Comprueba la configuración
</div>

Para comprobar que tu sistema de autorización funciona correctamente, puedes revisar los registros en busca de cualquier problema de autorización.

Pregúntale a Claude:

```bash wrap lines
Revisa los registros en busca de intentos fallidos de acceso a la API en la última hora. Enfócate en los fallos de autorización.
```

Claude usará `auth0_list_logs` para encontrar las entradas de registro pertinentes.

<div id="step-7-refine-the-authorization-rules">
  ### Paso 7: Ajusta las reglas de autorización
</div>

Según el análisis de los registros, puede que necesites ajustar tus reglas de autorización. Por ejemplo, quizá debas permitir el acceso desde rangos de IP adicionales.

Pregúntale a Claude:

```bash wrap lines
Update the authorization action to also allow requests from our development VPN range 192.168.100.0/24.
```

Claude usará `auth0_get_action` para obtener el código actual de la Action y luego usará `auth0_update_action` para modificarlo.

Por último, Claude usará `auth0_deploy_action` para implementar la Action actualizada.

<div id="conclusion">
  ### Conclusión
</div>

Al usar el servidor MCP de Auth0 con Claude, has implementado un sistema completo de autorización de API mediante:

* La creación de múltiples definiciones de API con los scopes adecuados
* La implementación de lógica de autorización personalizada con restricciones basadas en IP
* La configuración de aplicaciones cliente para acceder a API específicas
* El monitoreo y ajuste de tus reglas de autorización

Este enfoque en lenguaje natural no solo ahorra tiempo, sino que también te permite implementar prácticas recomendadas de seguridad sin necesidad de tener un conocimiento profundo del sistema Actions de Auth0 ni de la sintaxis de consulta de registros.
