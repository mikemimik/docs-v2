Usa interacciones en lenguaje natural con [servidor MCP de Auth0](/docs/es/get-started/auth0-mcp-server) para configurar y gestionar rápidamente un sistema completo de autorización para APIs. Al aprovechar la capacidad de [Claude](https://claude.ai/) para interactuar con Auth0 a través del Model Context Protocol, puedes implementar patrones de autorización complejos en minutos en lugar de horas.

<div id="step-1-explore-current-api-configuration">
  ### Paso 1: Explora la configuración actual de la API
</div>

Antes de crear nuevas API, es importante entender qué está configurado actualmente en tu tenant. Con el servidor MCP de Auth0, puedes obtener esta información rápidamente usando lenguaje natural.

Pregúntale a Claude:

```bash wrap lines
Show me all the resource servers (APIs) currently configured in my Auth0 tenant.
```

Claude usará la herramienta `auth0_list_resource_servers` para obtener y mostrar tus API existentes. Esto te ofrece una visión general rápida de tus API sin tener que navegar por el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>.

<div id="step-2-create-new-api-definitions">
  ### Paso 2: Crear nuevas definiciones de API
</div>

Ahora que entiendes tu configuración actual, puedes crear nuevas API para tu arquitectura de microservicios. En lugar de configurar cada API por separado desde el Dashboard, puedes definirlas todas de una sola vez.

Pregúntale a Claude:

```bash lines
Necesito crear tres nuevas APIs para nuestros microservicios:
1. Una API de inventario con scopes de lectura y escritura
2. Una API de pedidos con scopes de lectura y escritura
3. Una API de clientes con scopes de lectura, escritura y administración

Cada API debe aplicar validación de scopes.
```

Claude utilizará la herramienta `auth0_create_resource_server` varias veces para configurar estas API.

<div id="step-3-implement-authorization-logic">
  ### Paso 3: Implementa la lógica de autorización
</div>

Después de definir tus APIs, tendrás que implementar una lógica de autorización personalizada. Con el servidor MCP de Auth0, puedes describir tus reglas de negocio en lenguaje natural y Claude generará el código adecuado para [Actions](/docs/es/customize/actions) de Auth0.

Pregúntale a Claude:

```bash lines
Create an Action that enforces the following authorization rules:
1. Verify the requesting application has the appropriate scopes for the API being accessed.
2. For the customers API, requests using the admin scope should only come from our internal IP ranges (10.0.0.0/8 and 172.16.0.0/12).
3. Log all authorization decisions for audit purposes.
```

Claude usará la herramienta `auth0_create_action` para generar y crear una Action con el código adecuado.

<div id="step-4-deploy-the-authorization-action">
  ### Paso 4: Despliega la Action de autorización
</div>

Una vez que estés conforme con el código de la Action, puedes desplegarla en tu entorno de producción con una simple instrucción.

Pregúntale a Claude:

```bash lines
Deploy this action to run during the token issuance process.
```

Claude usará la herramienta `auth0_deploy_action` para implementar la Action.

<div id="step-5-configure-client-applications">
  ### Paso 5: Configurar aplicaciones cliente
</div>

A continuación, tendrás que configurar tus aplicaciones cliente para acceder a las APIs adecuadas con los scopes correctos.

Pregunta a Claude:

```bash wrap lines
Create an 'inventory-manager' application on my Auth0 tenant and update the app to access the inventory and orders APIs with read and write permissions, but not the customers API.
```

Claude primero usará `auth0_get_application` para obtener la configuración actual y luego `auth0_update_application` para actualizarla.

<div id="step-6-verify-configuration">
  ### Paso 6: Verificar la configuración
</div>

Para asegurarte de que tu sistema de autorización funciona correctamente, puedes revisar los logs para detectar cualquier problema de autorización.

Pídele a Claude:

```bash wrap lines
Check the logs for any failed API access attempts in the last hour. Focus on authorization failures.
```

Claude usará `auth0_list_logs` para encontrar registros relevantes.

<div id="step-7-refine-the-authorization-rules">
  ### Paso 7: Ajusta las reglas de autorización
</div>

Según el análisis de los registros, puede que tengas que ajustar tus reglas de autorización. Por ejemplo, puede que necesites permitir el acceso desde rangos de IP adicionales.

Pregúntale a Claude:

```bash wrap lines
Actualiza la Action de autorización para que también permita solicitudes desde nuestro rango de VPN de desarrollo 192.168.100.0/24.
```

Claude usará `auth0_get_action` para obtener el código actual de la Action y luego `auth0_update_action` para modificarlo.

Por último, Claude usará `auth0_deploy_action` para implementar la Action actualizada.

<div id="conclusion">
  ### Conclusión
</div>

Al usar el servidor MCP de Auth0 con Claude, has implementado un sistema completo de autorización de API mediante lo siguiente:

* Crear múltiples definiciones de API con los `scopes` adecuados
* Implementar lógica de autorización personalizada con restricciones basadas en IP
* Configurar aplicaciones cliente para acceder a API específicas
* Supervisar y ajustar tus reglas de autorización

Este enfoque en lenguaje natural no solo ahorra tiempo, sino que también te permite aplicar las mejores prácticas de seguridad sin necesidad de tener un conocimiento profundo del sistema Actions de Auth0 ni de la sintaxis de consulta de registros.
