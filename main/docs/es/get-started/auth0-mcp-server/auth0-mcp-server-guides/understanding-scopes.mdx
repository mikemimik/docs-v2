---
description: Comprenda cómo usar scopes en el protocolo de contexto del modelo
title: Comprender los scopes
---

Por motivos de seguridad, el servidor Auth0 Model Context Protocol no concede ningún scope de forma predeterminada, y los scopes obligatorios deben solicitarse durante la inicialización. Una vez que el servidor esté inicializado, puede configurar manualmente los scopes mediante un patrón glob, como se muestra en el ejemplo a continuación:

```bash lines
# Seleccionar todos los scopes de lectura
npx @auth0/auth0-mcp-server init --scopes 'read:*'

# Seleccionar múltiples patrones de scope (separados por comas)
npx @auth0/auth0-mcp-server init --scopes 'read:*,create:clients,update:actions'
```

Según el tipo de operación de administración o la herramienta que quiera usar, es posible que deba solicitar scopes adicionales para ejecutar correctamente la operación de MCP en su cliente. A continuación se muestran los scopes disponibles.

  | Operación de administración                                           | Herramienta MCP                | Scope requerido           | Posibles implicaciones                                                                                                         |
  | --------------------------------------------------------------------- | ------------------------------ | ------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
  | Obtener detalles de una aplicación específica de Auth0                | `auth0_get_application`        | `read:clients`            | Permite ver detalles de configuración de la aplicación que podrían ser sensibles                                               |
  | Obtener detalles de una aplicación específica de Auth0                | `auth0_get_application`        | `read:clients`            | Permite ver detalles de configuración de la aplicación que podrían ser sensibles                                               |
  | Crear una nueva aplicación de Auth0                                   | `auth0_create_application`     | `create:clients`          | Permite crear nuevas aplicaciones que podrían usarse en flujos de autenticación                                                |
  | Actualizar una aplicación existente de Auth0                          | `auth0_update_application`     | `update:clients`          | Puede modificar configuraciones críticas de la aplicación, lo que podría interrumpir la autenticación de los usuarios          |
  | Enumerar todos los servidores de recursos (API) en el tenant de Auth0 | `auth0_list_resource_servers`  | `read:resource_servers`   | Expone todas las configuraciones de API y sus scopes                                                                           |
  | Obtener detalles de un servidor de recursos específico de Auth0       | `auth0_create_resource_server` | `create:resource_servers` | Revela detalles de configuración de la API, incluidos los algoritmos de firma y la configuración de tokens                     |
  | Crear un nuevo servidor de recursos (API) de Auth0                    | `auth0_get_resource_server`    | `read:resource_servers`   | Permite definir nuevas API y scopes que podrían usarse para el control de acceso                                               |
  | Actualizar un servidor de recursos (API) existente de Auth0           | `auth0_update_resource_server` | `update:resource_servers` | Puede modificar configuraciones de API, lo que podría afectar a las aplicaciones que dependen de esas API                      |
  | Enumerar todas las acciones en el tenant de Auth0                     | `auth0_list_actions`           | `read:actions`            | Revela todo el código personalizado que se ejecuta durante los flujos de autenticación                                         |
  | Obtener detalles de una acción específica de Auth0                    | `auth0_get_action`             | `read:actions`            | Expone código y lógica personalizados de los flujos de autenticación                                                           |
  | Crear una nueva acción de Auth0                                       | `auth0_create_action`          | `create:actions`          | Permite crear código personalizado que puede modificar los flujos de autenticación y los datos de los usuarios                 |
  | Actualizar una acción existente de Auth0                              | `auth0_update_action`          | `update:actions`          | Puede modificar código personalizado que afecta a la seguridad y al comportamiento de la autenticación                         |
  | Desplegar una acción de Auth0                                         | `auth0_deploy_action`          | `update:actions`          | Lleva código personalizado a producción, lo que podría afectar la experiencia de autenticación de todos los usuarios           |
  | Enumerar logs del tenant de Auth0                                     | `auth0_list_logs`              | `read:logs`               | Expone información detallada sobre la actividad de los usuarios, los inicios de sesión fallidos y los eventos de seguridad     |
  | Obtener una entrada de log específica por ID                          | `auth0_get_log`                | `read:logs`               | Proporciona datos detallados sobre eventos de autenticación específicos, incluidos potencialmente datos personales             |
  | Enumerar todos los formularios en el tenant de Auth0                  | `auth0_list_forms`             | `read:forms`              | Revela configuraciones de formularios personalizados para inicio de sesión, registro y restablecimiento de contraseña          |
  | Obtener detalles de un formulario específico de Auth0                 | `auth0_get_form`               | `read:forms`              | Expone la configuración detallada del formulario, incluida la validación de campos y el diseño                                 |
  | Crear un nuevo formulario de Auth0                                    | `auth0_create_form`            | `create:forms`            | Permite crear formularios de autenticación personalizados que recopilan datos de usuario                                       |
  | Actualizar un formulario existente de Auth0                           | `auth0_update_form`            | `update:forms`            | Puede modificar formularios de cara al usuario, lo que podría afectar la experiencia de autenticación                          |
  | Publicar un formulario de Auth0                                       | `auth0_publish_form`           | `update:forms`            | Hace que los cambios del formulario se publiquen para todos los usuarios, lo que podría afectar la autenticación en producción |
