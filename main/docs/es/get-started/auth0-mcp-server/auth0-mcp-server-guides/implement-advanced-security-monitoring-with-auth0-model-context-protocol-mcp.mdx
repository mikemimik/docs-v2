Aprovecha [servidor MCP de Auth0](/docs/es/get-started/auth0-mcp-server) y [Claude](https://claude.ai/) para implementar un sofisticado sistema de monitoreo de seguridad y respuesta automatizada para tu tenant de Auth0. Mediante interacciones en lenguaje natural, puedes configurar rápidamente medidas de seguridad que normalmente requerirían scripts complejos y navegar por el panel.

<div id="step-1-set-up-advanced-log-monitoring">
  ### Paso 1: Configura el monitoreo avanzado de logs
</div>

El primer paso para crear un sistema de monitoreo de seguridad es comprender cuáles son los patrones habituales de autenticación en tu entorno. Con el servidor MCP de Auth0, puedes analizar rápidamente tus logs para establecer esta línea base.

Pregúntale a Claude:

```bash wrap lines
Analiza mis logs de Auth0 de la semana pasada para identificar patrones de autenticación normales. Busca rangos de IP comunes, dispositivos, navegadores y patrones por hora del día.
```

Claude usará la herramienta `auth0_search_logs` para recuperar y analizar tus logs. Este análisis te proporciona una base para comprender los patrones normales de autenticación, lo cual es esencial para identificar anomalías.

<div id="step-2-create-security-detection-actions">
  ### Paso 2: Crear Actions para detectar amenazas de seguridad
</div>

Ahora que entiendes los patrones normales, puedes crear Actions que detecten actividades sospechosas. En lugar de escribir manualmente código complejo, puedes describir las reglas de seguridad en lenguaje natural.

Pídele a Claude:

```bash lines
Create an action that detects the following suspicious activities:
1. Multiple failed login attempts within a short time period
2. Logins from unusual geographic locations for a user
3. Logins at unusual times based on the user's history
4. Rapid account switching from the same IP address
```

Claude usará la herramienta `auth0_create_action` para generar y crear una Action con lógica de detección avanzada.

<div id="step-3-create-automated-response-actions">
  ### Paso 3: Crear Actions de respuesta automatizada
</div>

Una vez que tengas la detección configurada, el siguiente paso será crear Actions que respondan a las amenazas detectadas. Estas Actions implementarán tus políticas de seguridad en función del nivel de riesgo.

Pregunta a Claude:

```bash lines
Crea una Action que responda a amenazas de seguridad según la puntuación de riesgo:
1. Para riesgo bajo (puntuación < 30), permitir el inicio de sesión pero registrar el evento
2. Para riesgo medio (puntuación 30-60), solicitar verificación adicional como MFA
3. Para riesgo alto (puntuación > 60), bloquear el inicio de sesión y notificar al equipo de seguridad
```

Claude volverá a usar la herramienta `auth0_create_action` para crear una Action de tipo respuesta.

<div id="step-4-deploy-the-security-actions">
  ### Paso 4: Implementa las Actions de seguridad
</div>

Una vez creadas tus Actions de seguridad, debes implementarlas en el orden correcto para asegurarte de que funcionen bien juntas.

Pídele a Claude:

```bash wrap lines
Despliega primero la Action de detección de seguridad, seguida de la Action de respuesta. Deben ejecutarse en secuencia durante el flujo de inicio de sesión.
```

Claude usará la herramienta `auth0_deploy_action` dos veces para desplegar ambas Actions en el orden correcto.

<div id="step-5-create-custom-security-challenge-forms">
  ### Paso 5: Crear formularios personalizados para desafíos de seguridad
</div>

Para los inicios de sesión de riesgo medio que requieren una verificación adicional, te convendrá crear un [formulario](/docs/es/customize/forms) personalizado que explique la situación al usuario.

Pregúntale a Claude:

```bash wrap lines
Create a custom form for additional security verification that explains to the user why additional verification is needed and provides clear instructions.
```

Claude usará la herramienta `auth0_create_form` para crear un formulario personalizado. Después de crear el formulario, publícalo con la herramienta `auth0_publish_form`.

Pídele a Claude:

```bash lines
Publish the form to my tenant
```

<div id="step-6-monitor-security-events">
  ## Paso 6: Supervisa los eventos de seguridad
</div>

Para asegurarte de que tu sistema de seguridad funcione correctamente, debes supervisar los eventos de seguridad y analizar patrones.

Pregúntale a Claude:

```bash wrap lines
Show me all high-risk security events from the past 24 hours, and identify any patterns or trends, using score-based categorization.
```

Claude usará la herramienta `auth0_list_logs` con criterios específicos para buscar eventos de seguridad.

<div id="step-7-fine-tune-the-security-system">
  ## Paso 7: Ajusta el sistema de seguridad
</div>

Según lo que muestre tu monitoreo, puede que necesites ajustar el sistema de seguridad para reducir los falsos positivos o abordar amenazas específicas.

Pregúntale a Claude:

```bash wrap lines
Update the security detection action to be more sensitive to logins from Russia and China, and less sensitive to unusual login times for users in the IT department.
```

Claude usará `auth0_get_action` para obtener el código actual de la Action y luego usará `auth0_update_action` para modificarlo. Después de actualizar la Action, impleméntala con `auth0_deploy_action`.

Pregúntale a Claude:

```bash lines
Despliega la Action en mi tenant.
```

* Crear mecanismos de respuesta inteligentes que se adaptan a distintos niveles de riesgo
