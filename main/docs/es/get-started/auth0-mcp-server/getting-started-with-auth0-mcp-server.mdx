<Card title="Antes de empezar">
  * Instala [Node.js v18 o superior](https://nodejs.org/en/download) en tu sistema.
  * Crea una [cuenta de Auth0](https://auth0.com/) activa con permisos de administrador.
  * Instala al menos un cliente MCP compatible ([Claude Desktop](https://claude.ai/download), [Cursor](https://www.cursor.com/downloads) o [Windsurf](https://www.windsurf.com/editor), por ejemplo).
</Card>

Potencia a tu equipo y a tus usuarios instalando [servidor MCP de Auth0](/docs/es/get-started/auth0-mcp-server) e integrándolo con varios [clientes MCP](https://modelcontextprotocol.io/clients) de uso habitual.

<div id="installation-and-configuration">
  ## Instalación y configuración
</div>

Instala el servidor MCP de Auth0 ejecutando uno de estos comandos en tu terminal:

<div id="for-claude-desktop-default">
  ### Para Claude Desktop (predeterminado)
</div>

```bash lines
npx @auth0/auth0-mcp-server init
```

<div id="for-cursor">
  ### Para Cursor
</div>

```bash lines
npx @auth0/auth0-mcp-server init --client cursor
```

<div id="for-windsurf">
  ### Para Windsurf
</div>

```bash lines
npx @auth0/auth0-mcp-server init --client windsurf
```

<div id="other-mcp-clients">
  ### Otros clientes MCP
</div>

Para usar el servidor MCP de Auth0 con cualquier otro cliente MCP, puedes añadir manualmente esta configuración al cliente y reiniciarlo para que los cambios surtan efecto:

```json lines
{
  "mcpServers": {
    "auth0": {
      "command": "npx",
      "args": ["-y", "@auth0/auth0-mcp-server", "run"],
      "capabilities": ["tools"],
      "env": {
        "DEBUG": "auth0-mcp"
      }
    }
  }
}
```

<div id="authenticate-with-auth0">
  ## Autentícate con Auth0
</div>

Tu navegador se abrirá automáticamente para iniciar el <Tooltip tip="OAuth 2.0: Marco de autorización que define los protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">flujo de autorización de dispositivo</Tooltip> de <Tooltip tip="OAuth 2.0: Marco de autorización que define los protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">OAuth 2.0</Tooltip>. Inicia sesión en tu cuenta de Auth0 y concede los permisos solicitados.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las credenciales se almacenan de forma segura en el llavero de tu sistema. Si lo deseas, puedes verificar el almacenamiento con tu herramienta de administración del llavero.
</Callout>

<div id="verify-your-integration">
  ## Comprueba tu integración
</div>

<div id="claude-desktop">
  ### Claude Desktop
</div>

1. Reinicia Claude Desktop para asegurarte de que reconozca el servidor MCP de Auth0 que acabas de instalar.
2. En Claude Desktop, haz clic en el icono de **Tools** y verifica que las herramientas de Auth0 (por ejemplo, `auth0_create_action` o `auth0_delete_application`) aparezcan en la lista de **Available MCP Tools**.
3. Inicia una conversación con Claude Desktop y pídele ayuda para administrar tu tenant de Auth0.

<div id="cursor">
  ### Cursor
</div>

1. Reinicia Cursor para asegurarte de que reconozca el servidor MCP de Auth0 que acabas de instalar.
2. En Cursor, abre **Settings &gt; MCP** y verifica que **Auth0** aparezca en la lista de **MCP Servers**.
3. Inicia una conversación con Cursor y pídele ayuda para administrar tu tenant de Auth0.

<div id="windsurf">
  ### Windsurf
</div>

1. Reinicia Windsurf para asegurarte de que reconozca el servidor MCP de Auth0 que acabas de instalar.
2. En Windsurf, abre **Configuración de Windsurf &gt; Cascade** y verifica que **Auth0** aparezca en la lista de **servidores MCP (Model Context Protocol)**.
3. Inicia una conversación con Windsurf y pídele ayuda para administrar tu tenant de Auth0.

<div id="common-troubleshooting-steps">
  ## Pasos habituales para solucionar problemas
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La mayoría de los problemas de conexión se pueden resolver configurando el servidor MCP de Auth0 y reiniciando el cliente MCP.
</Callout>

<div id="get-command-line-help">
  #### Obtén ayuda de la línea de comandos:
</div>

Consulta una lista de comandos compatibles y ejemplos de uso con:

```bash lines
npx @auth0/auth0-mcp-server help
```

<div id="authentication-issues">
  #### Problemas de autenticación
</div>

Asegúrate de que tienes los permisos correctos en tu tenant de Auth0 y prueba a reinicializar con:

```bash lines
npx @auth0/auth0-mcp-server init
```

<div id="connection-failures">
  #### Fallos de conexión:
</div>

1. Confirma que el servidor MCP esté en ejecución con:

   ```ps lines
   ps aux | grep auth0-mcp
   ```

2. Reinicia tu cliente de IA después de comprobar que el servidor está en ejecución.

<div id="claude-cant-connect-to-the-server-error">
  #### Error «Claude Can&#39;t Connect to the Server»:
</div>

1. Reinicia Claude Desktop después de instalarlo.
2. Comprueba que el servidor esté en ejecución con:

   ```ps lines
   ps aux | grep auth0-mcp
   ```

<div id="api-errors">
  #### Errores de la API:
</div>

Verifica los permisos y la expiración de tu token de Auth0. Activa el modo de depuración (consulta más abajo) para ver logs detallados.

<div id="debug-configuration-issues">
  ## Solucionar problemas de configuración
</div>

<div id="enable-debug-mode">
  #### Habilitar el modo de depuración
</div>

Activa el modo de depuración para ver registros detallados:

```bash lines
export DEBUG=auth0-mcp
```

<div id="debug-client-logs">
  #### Logs de depuración del cliente
</div>

Obtén logs detallados del cliente MCP desde Claude Desktop:

```bash lines
# Seguir los registros en tiempo real
tail -n 20 -F ~/Library/Logs/Claude/mcp*.log
```

Obtén registros detallados del servidor MCP al ejecutar el servidor MCP de Auth0 en modo de depuración:

```bash lines
DEBUG=auth0-mcp npx @auth0/auth0-mcp-server run
```

<div id="advanced-debugging-using-mcp-inspector">
  #### Depuración avanzada con MCP Inspector
</div>

Para la resolución avanzada de problemas, usa el MCP Inspector:

```bash wrap lines
npx @modelcontextprotocol/inspector -e DEBUG='auth0-mcp' @auth0/auth0-mcp-server run__text in bold__
```
