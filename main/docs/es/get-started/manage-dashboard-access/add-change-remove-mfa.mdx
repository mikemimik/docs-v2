<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+authentication">
  Autenticación multifactor
</Tooltip>

(MFA) añade un nivel adicional de seguridad a una cuenta de Auth0. Cuando los usuarios con MFA habilitado inician sesión en el

<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">
  Auth0 Dashboard
</Tooltip>

, Auth0 solicita sus credenciales, además de un dato adicional de identificación. Esto garantiza que solo los usuarios válidos puedan acceder a sus cuentas, incluso si un

<Tooltip tip="Actores maliciosos: Entidad (una persona o grupo) que representa una amenaza para el negocio o el entorno con la intención de causar daño." cta="Ver glosario" href="/docs/es/glossary?term=bad+actor">
  actor malicioso
</Tooltip>

ha comprometido un nombre de usuario y una contraseña.

Cualquier usuario del Dashboard puede inscribirse por su cuenta en MFA en [su perfil](https://manage.auth0.com/#/profile). El indicador de MFA en la lista [Auth0 Dashboard &gt; Settings &gt; Tenant Members](https://manage.auth0.com/#/tenant/admins) muestra si un usuario ha habilitado MFA para su cuenta.

Auth0 admite estos factores de autenticación para los usuarios del Dashboard:

* **WebAuthn con claves de seguridad FIDO**: Los autenticadores WebAuthn roaming son extraíbles y multiplataforma, como una YubiKey, y pueden usarse en varios dispositivos. Para autenticarse con un autenticador roaming, los usuarios deben conectar el autenticador a su dispositivo (mediante USB, NFC o Bluetooth) y demostrar presencia (por ejemplo, tocándolo).
* **WebAuthn con biometría del dispositivo**: Los autenticadores de plataforma WebAuthn están vinculados a un dispositivo y solo funcionan en ese dispositivo. Algunos ejemplos son la Touch Bar de MacBook, Windows Hello, Touch ID o Face ID en iOS, y el reconocimiento facial o de huella en Android. Como solo funcionan en el dispositivo al que están vinculados, un usuario debe tener al menos otro factor inscrito en su perfil antes de registrar la biometría del dispositivo.
* **Notificación push mediante Guardian**: Envía notificaciones push al dispositivo registrado de un usuario, normalmente un teléfono móvil o una tableta. El usuario puede permitir o denegar de inmediato el acceso a la cuenta con solo pulsar un botón. El factor push está disponible con la aplicación móvil Guardian para iOS y Android.
* **Contraseñas de un solo uso (OTP)**: Permite a un usuario usar una aplicación autenticadora (como Google Authenticator) en su dispositivo personal. La aplicación genera un OTP que cambia con el tiempo y puede introducirse como segundo factor para verificar la cuenta.
* **Notificación por SMS**: Envía un código de un solo uso por SMS. Luego, Auth0 solicita al usuario que introduzca este código antes de poder completar la autenticación. El SMS como factor de MFA solo está disponible en tenants asociados a un plan de suscripción.

Para saber cómo inscribirse en MFA del Dashboard, lea [Agregar autenticación multifactor para el acceso a Auth0 Dashboard](/docs/es/get-started/manage-dashboard-access/add-change-remove-mfa/add-mfa).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recomienda los factores WebAuthn como los métodos de autenticación más seguros y fáciles de usar. Para obtener más información, lea [Autenticación FIDO con
  WebAuthn](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn).
</Callout>

<Warning>
  Los administradores deben habilitar al menos un factor para usar MFA. Auth0 recomienda encarecidamente configurar varios factores para poder seguir accediendo a su cuenta si pierde su dispositivo principal.

  Una configuración ideal es usar tres factores:

  1. WebAuthn, Guardian u OTP como método principal.
  2. Uno o más números de SMS como respaldo (disponible solo en tenants asociados a un plan de suscripción).
  3. Un código de recuperación.

  Si no puede proporcionar su token de MFA y no tiene métodos de respaldo adecuados, es posible que su cuenta no pueda recuperarse.
</Warning>
