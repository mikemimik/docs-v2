title: Autenticación multifactor para usuarios del Dashboard
---

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+authentication">
  Autenticación multifactor
</Tooltip>

(MFA) agrega un nivel adicional de seguridad a una cuenta de Auth0. Cuando los usuarios con MFA habilitado inician sesión en el

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">
  Auth0 Dashboard
</Tooltip>

, Auth0 solicita sus credenciales y un dato adicional para verificar su identidad. Esto garantiza que solo los usuarios legítimos puedan acceder a sus cuentas, incluso si un

<Tooltip tip="Actor malintencionado: entidad (una persona o grupo) que representa una amenaza para la empresa o el entorno con la intención de causar daño." cta="Ver glosario" href="/docs/es/glossary?term=bad+actor">
  actor malintencionado
</Tooltip>

ha comprometido un nombre de usuario y una contraseña.

Cualquier usuario del Dashboard puede inscribirse por su cuenta en MFA en [su perfil](https://manage.auth0.com/#/profile). El indicador de MFA en la lista [Auth0 Dashboard &gt; Settings &gt; Tenant Members](https://manage.auth0.com/#/tenant/admins) indica si un usuario ha habilitado MFA para su cuenta.

Auth0 admite estos factores de autenticación para los usuarios del Dashboard:

* **WebAuthn con claves de seguridad FIDO**: los autenticadores itinerantes de WebAuthn son extraíbles y multiplataforma, como una YubiKey, y pueden usarse en varios dispositivos. Para autenticarse con un autenticador itinerante, los usuarios deben conectar el autenticador a su dispositivo (mediante USB, NFC o Bluetooth) y confirmar su presencia (por ejemplo, tocándolo).
* **WebAuthn con biometría del dispositivo**: los autenticadores de plataforma de WebAuthn están asociados a un dispositivo y funcionan solo en ese dispositivo. Algunos ejemplos son la Touch Bar de MacBook, Windows Hello, Touch ID o Face ID de iOS, y el reconocimiento facial o de huella digital de Android. Como solo funcionan en el dispositivo asociado, el usuario debe tener inscrito al menos otro factor en su perfil antes de registrar la biometría del dispositivo.
* **Notificación push a través de Guardian**: envía notificaciones push al dispositivo prerregistrado de un usuario, normalmente un teléfono móvil o una tableta. El usuario puede permitir o denegar de inmediato el acceso a la cuenta con solo pulsar un botón. El factor push está disponible con la aplicación móvil Guardian para iOS y Android.
* **Contraseñas de un solo uso (OTP)**: permite a un usuario utilizar una aplicación de autenticación (como Google Authenticator) en su dispositivo personal. La aplicación genera una OTP que cambia con el tiempo y puede introducirse como segundo factor para verificar la cuenta.
* **Notificación por SMS**: envía un código de un solo uso por SMS. Luego, Auth0 solicita al usuario que introduzca este código antes de completar la autenticación. El SMS como método de MFA solo está disponible en tenants asociados a un plan de suscripción.

Para obtener información sobre cómo inscribirse en el MFA del Dashboard, lea [Agregar autenticación multifactor para el acceso a Auth0 Dashboard](/docs/es/get-started/manage-dashboard-access/add-change-remove-mfa/add-mfa).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recomienda los factores de WebAuthn como los métodos de autenticación más seguros y fáciles de usar. Para obtener más información, lea [FIDO Authentication with
  WebAuthn](/docs/es/secure/multi-factor-authentication/fido-authentication-with-webauthn).
</Callout>

<Warning>
  Los administradores deben habilitar al menos un factor para usar MFA. Auth0 recomienda encarecidamente configurar varios factores para que pueda seguir accediendo a su cuenta si pierde su dispositivo principal.

  Una configuración ideal es usar tres factores:

  1. WebAuthn, Guardian u OTP como método principal.
  2. Uno o más números de teléfono para SMS como respaldo (disponible solo en tenants asociados a un plan de suscripción).
  3. Un código de recuperación.

  Si no puede proporcionar su token de MFA y no dispone de métodos de respaldo adecuados, es posible que su cuenta sea irrecuperable.
</Warning>
