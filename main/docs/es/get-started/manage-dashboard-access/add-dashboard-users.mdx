description: Aprende cómo agregar miembros del tenant para que accedan al Auth0 Dashboard.
title: Agregar miembros del tenant
---

Si eres administrador del tenant, puedes agregar colegas al tenant y asignarles un rol específico para darles acceso a las funcionalidades de <Tooltip tip="Auth0 Dashboard: El principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> que necesiten.

<Warning>
  Si tu cuenta de administrador del tenant se autentica con nombre de usuario y contraseña, debes verificar tu dirección de correo electrónico antes de poder invitar a otros miembros.
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si Teams está aprovisionado para tu cuenta empresarial, administra los miembros de tu tenant mediante Auth0 Teams. Para obtener más información, lee [Gestión de miembros del equipo](/docs/es/get-started/auth0-teams/team-member-management).
</Callout>

Cuando agregas nuevos miembros del tenant, puedes asignarles roles que les den acceso sin poner en riesgo las aplicaciones de producción ni la información confidencial. Puedes asignar más de un rol a un usuario y los permisos se combinarán. Para mayor protección, los usuarios pueden activar la [autenticación multifactor](/docs/es/get-started/manage-dashboard-access/add-change-remove-mfa) (<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>). Para obtener más información sobre qué funcionalidades del Dashboard están disponibles con cada rol, lee [Acceso al Dashboard por rol](/docs/es/get-started/manage-dashboard-access/feature-access-by-role).

| Rol                             | Permisos                                                                                                                                                                                               |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Admin**                       | Acceso de lectura y escritura a todos los recursos del Auth0 Dashboard, incluidas las extensiones.                                                                                                     |
| **Editor - Connections**        | Acceso de lectura, escritura y creación a todo tipo de conexiones.                                                                                                                                     |
| **Editor - Key Management**     | Crear y administrar claves criptográficas.                                                                                                                                                             |
| **Editor - Organizations**      | Acceso de lectura, escritura y creación a la configuración de Organizations. Acceso de solo lectura a Users, Roles y Connections.                                                                      |
| **Editor - Specific Apps**      | Acceso de lectura y escritura solo a aplicaciones específicas.                                                                                                                                         |
| **Editor - Users**              | Operaciones de gestión de usuarios (crear, eliminar, bloquear, desbloquear, restablecer MFA, restablecer contraseña, actualizar metadatos, asignar roles, etc.) y acceso a logs.                       |
| **Viewer - Users**              | Acceso de solo lectura a los usuarios y a los logs relacionados con ellos.                                                                                                                             |
| **Viewer - Config Settings**    | Acceso de solo lectura a todos los ajustes de configuración (aplicaciones, APIs, reglas, ajustes de seguridad, etc.), excepto a información confidencial como secretos, facturación, usuarios y logs.  |
| **Support Access - Deprecated** | Acceso a tickets (enviar, ver y actualizar) y métricas agregadas.                                                                                                                                      |
| **Elevated Support Access**     | Acceso de lectura, escritura y creación a **todos** los tickets de soporte creados por **todos los usuarios** de un tenant (Subscripton Tickets en Auth0 Support Center). Acceso a métricas agregadas. |

<Card title="La disponibilidad varía según el plan de Auth0">
  Tanto tu implementación específica de inicio de sesión como tu plan de Auth0 o acuerdo personalizado afectan la disponibilidad de esta funcionalidad. Para obtener más información, lee [Precios](https://auth0.com/pricing).
</Card>

1. Ve a [Dashboard &gt; Settings &gt; Tenant Members](https://manage.auth0.com/#/tenant/admins).
2. Haz clic en **Add Member**.
3. En la pantalla **Add New Tenant Member**, ingresa la dirección de correo electrónico del usuario y selecciona los roles que quieras asignarle.
4. Haz clic en **Invite**.
   Auth0 enviará un correo electrónico al usuario con un enlace en el que deberá hacer clic para aceptar la invitación. Antes de que el usuario acepte la invitación, aparecerá como pendiente en la lista de Invitations. Si haces clic en `...`, verás **Remove invitation** o **Copy invitation link**.

<Card title="Tenants faltantes">
  Los usuarios pueden crear cuentas de Auth0 de distintas maneras. Por ejemplo, pueden registrarse con un proveedor social (p. ej., Google, GitHub) y luego volver a registrarse con su dirección de correo electrónico. Si los miembros de tu tenant no pueden ver todos sus tenants después de iniciar sesión, verifica si tienen varias cuentas de Auth0. Puedes ver el método de registro que usó el miembro del tenant del Dashboard yendo a [Dashboard &gt; Tenant Settings &gt; Tenant Members](https://manage.auth0.com/#/tenant/admins).
</Card>

Cuando el usuario invitado haga clic en el enlace para aceptar la invitación, se le pedirá que inicie sesión con la dirección de correo electrónico a la que se envió la invitación. Luego podrá elegir si iniciar sesión con un proveedor social o con su dirección de correo electrónico y contraseña. Si elige iniciar sesión con su dirección de correo electrónico y contraseña, ya debe haber creado una cuenta con Auth0; de lo contrario, primero tendrá que registrarse y luego volver a aceptar la invitación.

<Callout icon="clock" color="#F59E0B" iconType="regular">
  **Sincronización de Auth0 Support Center**

  Al usar [Auth0 Support Center](https://support.auth0.com) para crear un caso de soporte, el campo CC está limitado a miembros del tenant existentes. Después de que un miembro del tenant acepte una invitación, la sincronización con Auth0 Support Center puede tardar hasta 12 horas y, hasta entonces, no estará disponible para agregarlo al campo CC.
</Callout>
