title: Acceso al Dashboard por rol
---

Como administrador del tenant, puede asignar roles a sus colegas para otorgarles acceso limitado al <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Esto permite que los miembros del tenant completen sus tareas sin poner en riesgo las aplicaciones de producción, al tiempo que se cumple el principio de privilegio mínimo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cada rol recibe acceso a [Support Center](https://support.auth0.com/).
</Callout>

<div id="dashboard-roles">
  ## Roles del Dashboard
</div>

Puedes asignar los siguientes roles para acceder al Auth0 Dashboard:

| Rol                           | Permisos                                                                                                                                                                                               |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Admin**                     | Acceso de lectura y escritura a todos los recursos del Auth0 Dashboard, incluidas las extensiones.                                                                                                     |
| **Editor - Connections**      | Acceso de lectura, escritura y creación a todo tipo de conexiones.                                                                                                                                     |
| **Editor - Key Management**   | Crear y administrar claves criptográficas.                                                                                                                                                             |
| **Editor - Organizations**    | Acceso de lectura, escritura y creación a la configuración de la organización. Acceso de solo lectura a Users, Roles y Connections.                                                                    |
| **Editor - Specific Apps**    | Acceso de lectura y escritura solo a aplicaciones específicas.                                                                                                                                         |
| **Editor - Users**            | Operaciones de gestión de usuarios (crear, eliminar, bloquear, desbloquear, restablecer MFA, restablecer la contraseña, actualizar metadata, asignar roles, etc.) y acceso a los registros.            |
| **Viewer - Users**            | Acceso de solo lectura a los usuarios y a los registros relacionados con ellos.                                                                                                                        |
| **Viewer - Config Settings**  | Acceso de solo lectura a todos los ajustes de configuración (aplicaciones, APIs, reglas, ajustes de seguridad, etc.), excepto a información sensible como secretos, facturación, usuarios y registros. |
| **Support Access - Obsoleto** | Acceso a tickets (enviar, ver y actualizar) y métricas agregadas.                                                                                                                                      |
| **Elevated Support Access**   | Acceso de lectura, escritura y creación a **todos** los tickets de soporte creados por **todos los usuarios** de un tenant (Subscripton Tickets en Auth0 Support Center). Acceso a métricas agregadas. |

<Card title="La disponibilidad varía según el plan de Auth0">
  Tu plan de Auth0 o tu acuerdo personalizado afectan la disponibilidad de esta funcionalidad. Para obtener más información, consulta la [página de precios de Auth0](https://auth0.com/pricing).
</Card>

Los roles de acceso limitado restringen la experiencia de los miembros de un tenant en el Dashboard únicamente a las secciones y acciones necesarias para realizar su trabajo.

A continuación se detallan los permisos de funcionalidad asociados a cada rol. Si un permiso específico no figura en un rol, los miembros del tenant a quienes se les asigne ese rol no tendrán acceso a esa área del Auth0 Dashboard.

<div id="log-events-available-to-user-roles">
  ## Eventos de log disponibles para roles de usuario
</div>

Los registros pueden contener datos confidenciales, como secretos, PII, etc. Es importante no divulgar datos confidenciales a usuarios cuyo rol no requiera esa información. Sin embargo, los roles **Editor - Users** y **Viewer - Users** necesitan tener cierto acceso a los registros para identificar problemas de los usuarios. Por ejemplo, para averiguar si un usuario se registró correctamente, si fue bloqueado, etc.

Permitimos que **Editor - Users** y **Viewer - Users** accedan a un conjunto limitado de tipos de registro relacionados con eventos de usuario. Los eventos de log de la lista proporcionan la información necesaria sobre las acciones de los usuarios, pero no divulgan información confidencial sobre otras partes de la configuración del tenant. Para obtener más información, consulta [Log Event Type Codes](/docs/es/deploy-monitor/logs/log-event-type-codes).

<div id="limitations">
  ## Limitaciones
</div>

* Los usuarios con el rol **Admin** pueden invitar a usuarios **Editor - Specific Apps** a una sola aplicación a la vez. Como solución alternativa, el usuario **Admin** puede [editar su rol](/docs/es/get-started/manage-dashboard-access/edit-dashboard-users) para asignar varias aplicaciones después de que el usuario acepte la invitación.
* Los roles **Viewer - Users** y **Editor - Users** no tienen acceso a las secciones **Dispositivos** y **Aplicaciones autorizadas** de **Users**.
* La [Nueva página de actividad](/docs/es/get-started/auth0-overview/dashboard/activity) solo está visible para los usuarios **Admins** y **Elevated Support Access**. **Editor - Users** y **Viewer - Users** pueden acceder a la actividad diaria (como inicios de sesión o creación de cuentas) a través de la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Stats/get_daily).

<div id="private-cloud-requirements">
  ## Requisitos de Private Cloud
</div>

Los roles **Editor - Users** y **Viewer - Users** requieren que User Search v3 y Logs Search v3 estén habilitados en su entorno. Si sus entornos no son compatibles con estas versiones, estos dos roles no estarán disponibles.
