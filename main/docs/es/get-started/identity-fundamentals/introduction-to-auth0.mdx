title: Introducción a Auth0
---

Auth0 es un proveedor de gestión de identidades y accesos (IAM). Pero ¿qué significa esto? Si has leído [Introducción a la gestión de identidades y accesos (IAM)](/docs/es/get-started/identity-fundamentals/identity-and-access-management), sabes que una solución de IAM actúa como control de acceso a los recursos que ofreces a los clientes, como aplicaciones web, API, etc. Ese control de acceso inicia la autorización, tal como se describe en <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>. La incorporación de la capa de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect añade autenticación para proteger las <Tooltip tip="Digital Identity: Conjunto de atributos que definen a un usuario en particular en el contexto de una función ofrecida por una aplicación concreta." cta="Ver glosario" href="/docs/es/glossary?term=digital+identities">identidades digitales</Tooltip> de tus usuarios y tu producto.

La plataforma de identidad de Auth0 admite distintos tipos de aplicaciones y frameworks. Tanto si tu aplicación es una aplicación web tradicional, una aplicación móvil o una aplicación de máquina a máquina, Auth0 proporciona la configuración de concesión de autorización más segura, o flujo de trabajo, para cada caso. Puedes leer más sobre las concesiones de autorización y elegir la más adecuada para tu aplicación en nuestro artículo [¿Qué flujo de OAuth 2.0 debo usar?](/docs/es/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use)

Además de admitir protocolos seguros, la plataforma de identidad de Auth0 te permite personalizar los servicios de login para adaptarlos a tu negocio, tu tecnología y tu base de clientes. Con <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, puedes crear tu propia instancia de Auth0 para autenticar y autorizar a tus clientes. Puedes configurar los comportamientos de login, conectar tu almacén de datos de usuario, gestionar esos usuarios, elegir una concesión de autorización y establecer factores de autenticación para ofrecer un producto fluido y escalable con una experiencia de usuario impactante.

<div id="get-started">
  ## Primeros pasos
</div>

<div id="identity-fundamentals">
  ### Fundamentos de identidad
</div>

No tienes que ser un experto en IAM para integrar Auth0 en tu aplicación o API, pero sí conviene conocer algunos conceptos clave para elegir la configuración adecuada para tu caso de uso. Para obtener más información, lee nuestro artículo [Introducción a la gestión de identidades y accesos (IAM)](/docs/es/get-started/identity-fundamentals/identity-and-access-management). Si todavía tienes dudas sobre cómo planificar tu implementación, consulta nuestra sección [Escenarios de arquitectura](/docs/es/get-started/architecture-scenarios), donde encontrarás explicaciones paso a paso de escenarios reales.

<div id="integrate-with-auth0">
  ### Integra con Auth0
</div>

Para empezar a integrar tu aplicación con Auth0, puedes comenzar con nuestras [guías interactivas de Quickstart](/docs/es/quickstarts) para la configuración inicial y ajustes rápidos, o registrar tu aplicación manualmente en el Auth0 Dashboard. En el Dashboard, puedes crear un tenant, o tu instancia de Auth0, desde cero. Si prefieres usar SDKs, Auth0 ofrece varias opciones para cada tipo de aplicación. Para ver todas las opciones disponibles, consulta [Auth0 Libraries](/docs/es/libraries).

Puedes iniciar la configuración con detalles generales en la Configuración del tenant del Dashboard, como el nombre que se muestra a tus usuarios, el logotipo de tu empresa, tus URL de devolución de llamada o adónde Auth0 redirige a tus usuarios después de la autenticación. Puedes consultar nuestras recomendaciones en [Configuración del tenant](/docs/es/get-started/tenant-settings).

Una vez que hayas configurado el tenant, puedes crear y configurar tu aplicación o API. Puedes usar las instrucciones de nuestros artículos [Crear aplicaciones](/docs/es/get-started/auth0-overview/create-applications) o [Registrar APIs](/docs/es/get-started/auth0-overview/set-up-apis) como punto de partida.

<div id="authenticate">
  ### Autenticar
</div>

El mecanismo de la autenticación es el formulario de inicio de sesión, que actúa como intermediario para permitir que tus usuarios accedan a tu aplicación. Los usuarios proporcionan credenciales predeterminadas, como nombre de usuario o contraseña, en el formulario de inicio de sesión para verificar sus identidades digitales.

El <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> de Auth0 es un formulario de inicio de sesión que puedes personalizar para adaptarlo a tu marca y configurar para ofrecer acceso seguro. Algunos beneficios de usar Universal Login son:

* Inicio de sesión <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">passwordless</Tooltip> con biometría
* Elección de métodos de <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>, como correo electrónico, voz o Duo
* Capacidades de <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SSO)
* Soporte de localización

Para obtener más información, lee [Universal Login](/docs/es/authenticate/login/auth0-universal-login). Para conocer más sobre las funcionalidades disponibles, lee [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

Una vez que tengas un formulario de inicio de sesión, puedes conectar tu repositorio de usuarios a Auth0. Puedes conectar una base de datos existente o usar un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> social, legal o empresarial, como X o Azure Active Directory. Los usuarios nuevos pueden registrarse con la conexión que hayas configurado.

Una vez que tengas un formulario de inicio de sesión y una conexión a un repositorio de usuarios, puedes establecer protocolos que funcionen en segundo plano cuando los usuarios inicien sesión en tu aplicación. Los protocolos más comunes están asociados con las especificaciones OAuth 2.0 y OpenID Connect (OIDC) que quizá hayas revisado en nuestro artículo sobre [Fundamentos de identidad](/docs/es/get-started/identity-fundamentals).

Otro mecanismo para transmitir información de forma segura durante el inicio de sesión se presenta en forma de tokens. Los tokens del <Tooltip tip="Authorization Server: Servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funcionalidades disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Authorization+Server">servidor de autorización</Tooltip>, la Authentication API de Auth0, transmiten información entre entidades.  Cuando un usuario inicia sesión y se aprueba el acceso, la Authentication API envía un <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>, un <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip>, o ambos, según la concesión de autenticación que estés usando para crear una sesión. Los tokens de acceso contienen información sobre qué scopes, o permisos, tiene el solicitante en tu aplicación, mientras que los tokens de identidad contienen información del solicitante, como metadata del usuario, para mejorar la experiencia del usuario.

Los tokens de la Authentication API son <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y a menudo formato de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Tokens">JSON Web Tokens</Tooltip> (JWT) estructurados con:

* un encabezado que incluye la firma
* el payload que contiene declaraciones y atributos sobre el solicitante
* la firma que verifica que el token sea válido

Para obtener más información sobre los tokens, lee [Access Tokens](/docs/es/secure/tokens/access-tokens), [ID Tokens](/docs/es/secure/tokens/id-tokens) o [JSON Web Tokens](/docs/es/secure/tokens/json-web-tokens).

Otros protocolos, como <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> (Security Assertion Markup Language) y <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/docs/es/glossary?term=WS-Fed">WS-Fed</Tooltip> (Web Service Federation) se usan con sistemas más específicos. SAML funciona con algunos proveedores de identidad, mientras que WS-Fed se usa con productos de Microsoft. Puedes obtener más información en la sección [Protocolos](/docs/es/authenticate/protocols) de nuestra documentación.

<div id="manage-users">
  ### Administrar Users
</div>

Administrar los perfiles de usuario y el acceso puede llevar mucho tiempo. Si eliges administrar usuarios con tu instancia de Auth0, puedes reducir parte de esa complejidad.

Puedes automatizar fácilmente las operaciones CRUD y consultar perfiles de usuario mediante Auth0 Dashboard o la Management API. También puedes agrupar a tus usuarios con Auth0 Organizations para organizar tu base de clientes según tu estilo de administración. Para obtener más información, ve a la sección [Administrar Users](/docs/es/manage-users) de nuestra documentación.

Tu modelo de negocio puede incluir distintos niveles de acceso para tus usuarios. Puede que quieras que un subconjunto de usuarios tenga permisos de solo lectura y otro pueda editar. Authorization Core de Auth0 te permite implementar control de acceso basado en roles. Puedes crear roles, asignarlos a usuarios y definir permisos.

Si quieres administrar el acceso en función del comportamiento del navegador, puedes limitar la duración de una sesión. Una sesión, o la interacción entre la entidad solicitante y tu aplicación o recurso, tiene un límite de duración. Una sesión puede finalizar cuando el usuario cierra el navegador o abandona tu página web. Puedes ampliar las sesiones con <Tooltip tip="Token de actualización: token utilizado para obtener un nuevo token de acceso sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> que renuevan los tokens de acceso. Configura los tokens de actualización en el Dashboard.  Para obtener más información, consulta [Límites de duración de la sesión](/docs/es/manage-users/sessions/session-lifetime-limits) y [Obtener tokens de actualización](/docs/es/secure/tokens/refresh-tokens/get-refresh-tokens).

Las cookies, o cadenas de datos, están vinculadas a la sesión y representan a un usuario autenticado. Permiten que los usuarios autenticados mantengan una sesión y naveguen entre páginas web sin tener que volver a autenticarse.  Cuando se cierra el navegador, este elimina la cookie.

<div id="customize">
  ### Personalizar
</div>

Tu marca es importante, y Auth0 ofrece opciones de personalización para que la experiencia de inicio de sesión se adapte más a tu negocio. Puedes añadir tu logotipo y combinación de colores al formulario de inicio de sesión, así como usar un <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip> para tener el control de la URL de inicio de sesión. Para obtener más información sobre la configuración, consulta [Dominios personalizados](/docs/es/customize/custom-domains).

Universal Login ofrece numerosas funcionalidades para configurar la autenticación según tus necesidades, como autenticación multifactor, autenticación passwordless con biometría del dispositivo y localización. En un nivel más específico, puedes ajustar el texto de los prompt que recibe el usuario cuando necesita completar una acción. Puedes configurar prompt para que tus usuarios se registren, registren un dispositivo para la autenticación o reciban un código por correo electrónico/SMS que luego deban introducir para la verificación. También puedes personalizar las comunicaciones por correo electrónico para dar la bienvenida a nuevos usuarios, verificar la inscripción o restablecer contraseñas con plantillas de correo electrónico. Para obtener más información, consulta [Personalizar los elementos de texto de Universal Login](/docs/es/customize/login-pages/universal-login/customize-text-elements) y [Personalizar plantillas de correo electrónico](/docs/es/customize/email/email-templates).

También puedes configurar ciertos eventos con Auth0 Actions. Las Actions son funciones seguras que se ejecutan durante el runtime. Las Actions se activan en distintos puntos del pipeline y tienen una amplia variedad de usos. Por ejemplo, podrías añadir metadata antes de que el usuario se registre o redirigir usuarios a un sitio externo. Para obtener más información sobre lo que pueden hacer las Auth0 Actions por ti, consulta [Comprender cómo funcionan las Auth0 Actions](/docs/es/customize/actions/actions-overview).

<div id="secure">
  ### Seguridad
</div>

Los ataques maliciosos pueden producirse en cualquier momento. Auth0 ofrece varias opciones de <Tooltip tip="Protección contra ataques: funcionalidades que Auth0 ofrece para detectar y mitigar ataques, incluida la protección contra ataques de fuerza bruta, la limitación de IP sospechosas, la detección de contraseñas vulneradas, la detección de bots y la autenticación multifactor adaptativa." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip>, como <Tooltip tip="Bot Detection: forma de protección contra ataques en la que Auth0 bloquea el tráfico sospechoso de bots al habilitar un CAPTCHA durante el proceso de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> en combinación con Google reCAPTCHA Enterprise para prevenir ciberataques. Para obtener más información sobre la configuración de Bot Detection, consulta [Bot Detection](/docs/es/secure/attack-protection/bot-detection).

Incluso si usas tu propia página de inicio de sesión, Auth0 ofrece otras opciones de seguridad que puedes habilitar en el Auth0 Dashboard:

* <Tooltip tip="Detección de contraseñas vulneradas: forma de protección contra ataques en la que Auth0 notifica a tus usuarios si usan una combinación de nombre de usuario y contraseña que se ha visto comprometida en una filtración de datos en un sitio web o una aplicación de terceros." cta="Ver glosario" href="/docs/es/glossary?term=Breached+Password+Detection">Detección de contraseñas vulneradas</Tooltip>
* <Tooltip tip="Brute-force Protection: forma de protección contra ataques que protege contra ataques de fuerza bruta que se originan desde una sola dirección IP y tienen como objetivo una sola cuenta de usuario." cta="Ver glosario" href="/docs/es/glossary?term=Brute-Force+Protection">Brute Force Protection</Tooltip>
* <Tooltip tip="Suspicious IP Throttling: forma de protección contra ataques que protege tu tenant contra inicios de sesión sospechosos dirigidos a demasiadas cuentas desde una sola dirección IP." cta="Ver glosario" href="/docs/es/glossary?term=Suspicious+IP+Throttling">Suspicious IP Throttling</Tooltip>

La detección de contraseñas vulneradas es una medida de seguridad contra agentes maliciosos con credenciales robadas. La protección contra ataques de fuerza bruta protege una cuenta de usuario específica al limitar la cantidad de intentos de inicio de sesión, lo que bloquea automáticamente la IP maliciosa y envía una notificación a la cuenta marcada. La limitación de IP sospechosas actúa donde termina la protección contra ataques de fuerza bruta, para bloquear el tráfico de cualquier dirección IP que intente realizar registros o inicios de sesión rápidamente.

Otras medidas de seguridad dependen de cómo quieras que tus usuarios se autentiquen. Habilitar la autenticación multifactor (MFA) en Universal Login requiere que los usuarios proporcionen dos o más factores de autenticación. Con Auth0, puedes personalizar MFA para que se active en determinadas circunstancias, como cuando un usuario inicia sesión desde un dispositivo desconocido o desde una dirección IP sospechosa. Para obtener más información sobre cómo configurar MFA, consulta [Adaptive MFA](/docs/es/secure/multi-factor-authentication/adaptive-mfa).

<div id="deploy-and-monitor">
  ### Implementar y supervisar
</div>

Cuando hayas terminado de probar tu instancia de Auth0 y estés listo para implementar, puedes usar nuestras opciones de nube pública o Private Cloud. Para obtener más información sobre las opciones disponibles, lee [Opciones de implementación](/docs/es/deploy-monitor/deployment-options). Si necesitas un entorno multi-tenant, puedes obtener más información sobre [Private Cloud en AWS](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-aws).

Para que tu implementación siga bien encaminada, ofrecemos orientación en forma de recomendaciones previas a la implementación, una lista de verificación de implementación, prácticas recomendadas, soluciones comunes y otros consejos para que la implementación sea lo más fluida posible.

Una vez que tu entorno de producción esté listo para los usuarios, puedes supervisarlo con seguimiento de errores y alertas. System Center Operations Manager te permite monitorizarlo, mientras que los registros de eventos pueden exportarse a una herramienta de análisis para darte visibilidad sobre tendencias, comportamiento de los usuarios o problemas.
