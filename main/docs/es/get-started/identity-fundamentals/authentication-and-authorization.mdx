Aunque a menudo se usan indistintamente, la [autenticación](/docs/es/authenticate) y la autorización representan funciones fundamentalmente diferentes. En este artículo, las comparamos para mostrar cómo protegen las aplicaciones de forma complementaria.

<div id="what-are-authentication-and-authorization">
  ## ¿Qué son la autenticación y la autorización?
</div>

En términos sencillos, la autenticación es el proceso de verificar la identidad de un usuario, mientras que la autorización es el proceso de verificar a qué puede acceder.

Si comparamos estos procesos con un ejemplo del mundo real, cuando pasas por el control de seguridad de un aeropuerto, muestras tu documento de identidad para confirmar tu identidad. Luego, cuando llegas a la puerta de embarque, presentas tu tarjeta de embarque al asistente de vuelo para que pueda autorizarte a subir a tu vuelo y permitirte acceder al avión.

<div id="authentication-vs-authorization">
  ## Autenticación vs. autorización
</div>

Aquí tienes un resumen rápido de las diferencias entre la autenticación y la autorización:

| Autenticación                                                                                                                                    | Autorización                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------ |
| Determina si los usuarios son quienes dicen ser                                                                                                  | Determina a qué pueden y no pueden acceder los usuarios                                                                              |
| Solicita al usuario que valide sus credenciales (por ejemplo, mediante contraseñas, respuestas a preguntas de seguridad o reconocimiento facial) | Verifica si el acceso está permitido mediante políticas y reglas                                                                     |
| Suele realizarse antes de la autorización                                                                                                        | Suele realizarse después de una autenticación satisfactoria                                                                          |
| En general, transmite información a través de un token de ID                                                                                     | En general, transmite información a través de un [Access Token](/docs/es/glossary?term=Access+Token)                                    |
| Por lo general, se rige por el [protocolo OpenID Connect (OIDC)](/docs/es/glossary?term=OpenID+Connect+%28OIDC%29+protocol)                         | Por lo general, se rige por el framework OAuth 2.0                                                                                   |
| Ejemplo: los empleados de una empresa deben autenticarse en la red antes de acceder al correo electrónico corporativo                            | Ejemplo: después de que un empleado se autentique correctamente, el sistema determina a qué información pueden acceder los empleados |

En resumen, el acceso a un recurso está protegido tanto por la autenticación como por la autorización. Si no puedes demostrar tu identidad, no se te permitirá acceder a un recurso. E incluso si puedes demostrar tu identidad, si no estás autorizado para ese recurso, se te seguirá denegando el acceso.

Auth0 tiene productos y servicios de autenticación, como [passwordless](/docs/es/authenticate/passwordless/passwordless-with-universal-login), [autenticación multifactor](/docs/es/secure/multi-factor-authentication) (<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>) e [inicio de sesión único (SSO)](/docs/es/authenticate/single-sign-on), que puedes configurar mediante <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o <Tooltip tip="Management API: API principal de Auth0 para configurar programáticamente tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Para la autorización, Auth0 ofrece [control de acceso basado en roles](/docs/es/manage-users/access-control/rbac) (RBAC) o [autorización detallada](https://docs.fga.dev/fga) <Tooltip tip="Autorización detallada (FGA): producto de Auth0 que permite a usuarios individuales acceder a objetos o recursos específicos." cta="Ver glosario" href="/docs/es/glossary?term=FGA">FGA</Tooltip>).
