Acelera tu flujo de trabajo de desarrollo conectando agentes de IA y LLM directamente a la base de conocimiento de Auth0. Obtén respuestas contextualizadas, ejemplos de código y referencias de API con enlaces a la documentación oficial.

Para ayudar a tus herramientas de IA a indexar y buscar la información que necesitas, Auth0 ofrece:

* Archivos de contexto para LLM (`llms.txt`): archivos Markdown estructurados para ingesta masiva.
* Auth0 Docs MCP Server: MCP Server alojado para que los agentes de IA consulten dinámicamente la base de conocimiento.

<div id="llm-context-files">
  ## Archivos de contexto para LLM
</div>

Usa archivos de contexto para LLM para ayudar a los LLM a comprender y procesar de forma eficiente todo el directorio del sitio de Auth0.

| Nombre          | URL                                    | Descripción                                                                         |
| --------------- | -------------------------------------- | ----------------------------------------------------------------------------------- |
| `llms.txt`      | `https://auth0.com/llms.txt`           | Resúmenes concisos de todos los artículos; ideal para obtener contexto rápidamente. |
| `llms-full.txt` | `https://auth0.com/docs/llms-full.txt` | Contenido completo de todos los artículos; ideal para una indexación exhaustiva.    |

<div id="auth0-docs-mcp-server">
  ## Auth0 Docs MCP Server
</div>

Conecta tus herramientas de IA al Auth0 Docs Model Context Protocol (MCP) Server para acceder en tiempo real a la documentación más actualizada de Auth0. El Auth0 Docs MCP Server expone la herramienta `SearchAuth0Docs`, que permite a los agentes de IA buscar en la base de conocimiento de Auth0 y devolver contenido contextual con enlaces relevantes a la documentación del producto.

El Auth0 Docs MCP Server tiene los siguientes metadatos del endpoint:

```
Server Name: Auth0 Docs
Server URL: https://auth0.com/docs/mcp
Transport Type: HTTP
```

Aprende a conectar tus herramientas de IA con el Auth0 Docs MCP Server siguiendo nuestras guías de integración.

<Tabs>
  <Tab title="ChatGPT">
    Para conectar ChatGPT con el Auth0 Docs MCP Server:

    1. Ve a [Settings &gt; Connectors](https://chatgpt.com/#settings/Connectors).
    2. Selecciona **Create** y proporciona los metadatos de tu servidor MCP:
       * Nombre del Conector: Auth0 Docs MCP Server
       * Descripción: Indexa y busca en la base de conocimiento de Auth0
       * URL del Conector: `https://auth0.com/docs/mcp`
    3. Selecciona **Create**.

    Si todo funciona correctamente, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de OpenAI sobre [Connect from ChatGPT](https://developers.openai.com/apps-sdk/deploy/connect-chatgpt/).
  </Tab>

  <Tab title="Claude">
    Para conectar Claude con Auth0 Docs MCP Server:

    1. Ve a [Settings &gt; Connectors](https://claude.ai/settings/connectors).
    2. Selecciona **Add custom connector**.
    3. Introduce lo siguiente para añadir Auth0 Docs MCP Server:
       * Nombre: Auth0 Docs MCP Server
       * URL: `https://auth0.com/docs/mcp`
    4. Selecciona **Add**.

    Si todo funciona correctamente, Auth0 Docs MCP Server debería aparecer como un conector custom. Para obtener más información, consulta la documentación de Model Context Protocol sobre [Connect to remote MCP Servers](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    Para conectar Claude Code al Auth0 Docs MCP Server:

    En tu terminal, ejecuta:

    ```bash lines
    claude mcp add --transport http auth0-docs-mcp-server https://auth0.com/docs/mcp
    ```

    Para comprobarlo, ejecuta:

    ```bash lines
    claude mcp list
    ```

    Si la instalación se realiza correctamente, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Claude sobre [la instalación de servidores MCP](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Codex">
    Para conectar Codex al Auth0 Docs MCP Server:

    En tu terminal, ejecuta:

    ```bash lines
    codex mcp add auth0-docs-mcp-server --url https://auth0.com/docs/mcp
    ```

    Para comprobarlo, ejecuta:

    ```bash lines
    codex mcp list
    ```

    Si la instalación se realiza correctamente, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Codex sobre [Model Context Protocol](https://developers.openai.com/codex/mcp).
  </Tab>

  <Tab title="Cursor">
    Para conectar Cursor al Auth0 Docs MCP Server:

    1. Pulsa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la paleta de comandos y buscar **Open MCP settings**.
    2. Selecciona **New MCP Server** para editar tu `mcp.json`.
    3. Añade el Auth0 Docs MCP Server:

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "url": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    Si la instalación se completa correctamente, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Cursor sobre [cómo instalar servidores MCP](https://cursor.com/docs/context/mcp#installing-mcp-servers)).
  </Tab>

  <Tab title="VS Code">
    Para conectar VS Code al Auth0 Docs MCP Server:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la paleta de comandos y buscar **MCP: Add Server**.
    2. Selecciona **HTTP** como tipo de MCP Server.
    3. Introduce la URL del MCP Server (`https://auth0.com/docs/mcp`) y establece el ID del servidor como `auth0-docs-mcp-server`.
    4. Selecciona **Global** o **Workspace** para que el MCP Server esté disponible en todos tus espacios de trabajo o solo en el espacio de trabajo actual.
    5. Pulsa **Enter** para completar la configuración. Verifica que Auth0 Docs MCP Server aparezca en **Installed MCP Servers** dentro de **Extensions**.

    Para obtener más información, consulta [Add and manage MCP servers in VS Code](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>

  <Tab title="Windsurf">
    Para conectar Windsurf con el Auth0 Docs MCP Server:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la Configuración de Windsurf y buscar **MCP**.
    2. Selecciona **View Raw Config** para editar tu `mcp_config.json`.
    3. Agrega el Auth0 Docs MCP Server:

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "serverUrl": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    4. Guarda y selecciona **Refresh** para cargar la nueva configuración de MCP.

    Para comprobarlo, escribe `What MCP servers do you have installed?` en el chat de Cascade. Cascade debería mostrar Auth0 Docs MCP Server como un servidor MCP instalado. Para obtener más información, consulta la documentación de Windsurf sobre [Model Context Protocol](https://docs.windsurf.com/plugins/cascade/mcp#model-context-protocol-mcp).
  </Tab>
</Tabs>

<div id="ai-agents-and-mcp-servers">
  ### Agentes de IA y servidores MCP
</div>

* Añade una llamada a herramienta a mi agente de IA para que pueda acceder al Google Calendar del usuario y responder preguntas sobre su agenda y sus reuniones. Usa el Auth0 Docs MCP Server para obtener instrucciones sobre cómo usar Auth0 para proteger el inicio de sesión e implementar la llamada a herramienta al Google Calendar del usuario.
* Actúa como un ingeniero de seguridad. Usa el Auth0 Docs MCP Server para encontrar la guía sobre la autorización para tu MCP Server. Actualiza la configuración actual de mi servidor para exigir OAuth 2.1 en todas las conexiones entrantes y verifica que use el Perfil de compatibilidad del parámetro de recurso para los tokens de acceso.

Si encuentras alguna discrepancia por la que pueda filtrarse el contexto del usuario, sugiere una solución y proporciona el código actualizado.

<div id="troubleshooting">
  ### Resolución de problemas
</div>

* Ayúdame a solucionar un error de `refresh token doesn’t exist` al usar el intercambio de token de actualización. Usa como contexto las instrucciones de Setup de Token Vault.
* El botón de cierre de sesión no está cerrando la sesión en Auth0. Busca en la documentación la estructura correcta de la URL de cierre de sesión federado y actualiza mi controlador de cierre de sesión para usarla.
* Mi flujo de inicio de sesión está fallando con un error `401 Unauthorized`. Usa Auth0 Docs MCP Server para buscar causas comunes de este error con el SDK `auth0-node` y revisa mi archivo `.env` y la inicialización del cliente para detectar discrepancias.
