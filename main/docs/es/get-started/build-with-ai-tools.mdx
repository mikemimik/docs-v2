Acelera tu workflow de desarrollo conectando AI agents y LLM directamente a la base de conocimiento de Auth0. Obtén respuestas contextualizadas, ejemplos de código y referencias de API con enlaces a la documentación oficial.

Para ayudar a tus herramientas de IA a indexar y buscar la información que necesitas, Auth0 proporciona:

* LLM context files (`llms.txt`): Archivos Markdown estructurados para la ingesta masiva.
* Auth0 Docs MCP Server: MCP Server alojado para que los agentes de IA consulten dinámicamente la base de conocimiento.

<div id="llm-context-files">
  ## Archivos de contexto para LLM
</div>

Usa archivos de contexto para LLM para ayudar a los LLM a comprender y procesar de forma eficiente todo el directorio del sitio de Auth0.

| Nombre          | URL                                    | Descripción                                                                         |
| --------------- | -------------------------------------- | ----------------------------------------------------------------------------------- |
| `llms.txt`      | `https://auth0.com/llms.txt`           | Resúmenes concisos de todos los artículos; ideal para obtener contexto rápidamente. |
| `llms-full.txt` | `https://auth0.com/docs/llms-full.txt` | Contenido completo de todos los artículos; ideal para una indexación exhaustiva.    |

<div id="auth0-docs-mcp-server">
  ## Auth0 Docs MCP Server
</div>

Conecta tus herramientas de IA al servidor Model Context Protocol (MCP) de Auth0 Docs para acceder en tiempo real a la documentación más actualizada de Auth0. Auth0 Docs MCP Server expone la herramienta `SearchAuth0Docs`, que permite a los agentes de IA buscar en la base de conocimiento de Auth0 y devolver contenido contextual con enlaces relevantes a la documentación del producto.

Auth0 Docs MCP Server tiene los siguientes metadatos del endpoint:

```
Server Name: Auth0 Docs
Server URL: https://auth0.com/docs/mcp
Transport Type: HTTP
```

Aprende a conectar tus herramientas de IA al Auth0 Docs MCP Server con nuestras guías de integración.

<Tabs>
  <Tab title="ChatGPT">
    Para conectar ChatGPT con el Auth0 Docs MCP Server:

    1. Ve a [Settings &gt; Connectors](https://chatgpt.com/#settings/Connectors).
    2. Selecciona **Crear** e introduce los metadatos de tu servidor MCP:
       * Nombre del conector: Auth0 Docs MCP Server
       * Descripción: Indexa y busca en la base de conocimiento de Auth0
       * URL del conector: `https://auth0.com/docs/mcp`
    3. Selecciona **Crear**.

    Si todo va bien, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de OpenAI sobre [Connect from ChatGPT](https://developers.openai.com/apps-sdk/deploy/connect-chatgpt/).
  </Tab>

  <Tab title="Claude">
    Para conectar Claude a Auth0 Docs MCP Server:

    1. Ve a [Settings &gt; Connectors](https://claude.ai/settings/connectors).
    2. Selecciona **Add custom connector**.
    3. Introduce lo siguiente para añadir Auth0 Docs MCP Server:
       * Nombre: Auth0 Docs MCP Server
       * URL: `https://auth0.com/docs/mcp`
    4. Selecciona **Add**.

    Si todo salió bien, Auth0 Docs MCP Server debería aparecer como un conector personalizado. Para obtener más información, consulta la documentación de Model Context Protocol: [Connect to remote MCP Servers](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    Para conectar Claude Code al Auth0 Docs MCP Server:

    En la terminal, ejecuta:

    ```bash lines
    claude mcp add --transport http auth0-docs-mcp-server https://auth0.com/docs/mcp
    ```

    Para comprobarlo, ejecuta:

    ```bash lines
    claude mcp list
    ```

    Si todo sale bien, Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Claude sobre [cómo instalar servidores MCP](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Codex">
    Para conectar Codex al Auth0 Docs MCP Server:

    En tu terminal, ejecuta:

    ```bash lines
    codex mcp add auth0-docs-mcp-server --url https://auth0.com/docs/mcp
    ```

    Para comprobarlo, ejecuta:

    ```bash lines
    codex mcp list
    ```

    Si se realiza correctamente, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Codex sobre [Model Context Protocol](https://developers.openai.com/codex/mcp).
  </Tab>

  <Tab title="Cursor">
    Para conectar Cursor al Auth0 Docs MCP Server:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la paleta de comandos y buscar **Abrir la configuración de MCP**.
    2. Selecciona **Nuevo MCP Server** para editar tu `mcp.json`.
    3. Agrega el Auth0 Docs MCP Server:

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "url": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    Si la instalación se realiza correctamente, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Cursor sobre [Installing MCP servers](https://cursor.com/docs/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    Para conectar VS Code con el Auth0 Docs MCP Server:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la paleta de comandos y buscar **MCP: Add Server**.
    2. Selecciona **HTTP** como tipo de servidor MCP.
    3. Introduce la URL del MCP Server (`https://auth0.com/docs/mcp`) y establece el ID del servidor como `auth0-docs-mcp-server`.
    4. Selecciona **Global** o **Workspace** para que el servidor MCP esté disponible en todos tus espacios de trabajo o solo en el espacio de trabajo actual.
    5. Pulsa **Enter** para completar la configuración. Comprueba que el Auth0 Docs MCP Server aparezca en **Installed MCP Servers** dentro de **Extensions**.

    Para obtener más información, consulta [Add and manage MCP servers in VS Code](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>

  <Tab title="Windsurf">
    Para conectar Windsurf al Auth0 Docs MCP Server:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la configuración de Windsurf y buscar **MCP**.
    2. Selecciona **Ver configuración sin formato** para editar tu `mcp_config.json`.
    3. Agrega el Auth0 Docs MCP Server:

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "serverUrl": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    4. Guarda y selecciona **Actualizar** para cargar la nueva configuración de MCP.

    Para comprobarlo, pregunta `What MCP servers do you have installed?` en el chat de Cascade. Cascade debería mostrar Auth0 Docs MCP Server como un MCP Server instalado. Para obtener más información, consulta la documentación de Windsurf sobre [Model Context Protocol](https://docs.windsurf.com/plugins/cascade/mcp#model-context-protocol-mcp).
  </Tab>
</Tabs>

Usa Auth0 Docs MCP Server para averiguar cómo configurar la conexión empresarial de GitHub y escribir una Auth0 Action que compruebe si el usuario pertenece a la organización de GitHub durante el inicio de sesión y le deniegue el acceso si no es miembro.

<div id="ai-agents-and-mcp-servers">
  ### Agentes de IA y servidores MCP
</div>

* Añade una llamada a herramienta a mi agente de IA para que pueda acceder a Google Calendar del usuario y responder preguntas sobre su agenda y reuniones. Usa el Auth0 Docs MCP Server para obtener instrucciones sobre cómo usar Auth0 para proteger el inicio de sesión e implementar la llamada a herramienta al Google Calendar del usuario.
* Actúa como ingeniero de seguridad. Usa el Auth0 Docs MCP Server para encontrar la guía sobre la autorización para tu servidor MCP. Actualiza la configuración actual de mi servidor para exigir OAuth 2.1 en todas las conexiones entrantes y verifica que utilice el Resource Parameter Compatibility Profile para los tokens de acceso.
* Examina mis definiciones actuales de herramientas en `server.py`. Usa el Auth0 Docs MCP Server para verificar si mi implementación de Custom Token Exchange cumple con la RFC 8693. Si encuentras discrepancias por las que el contexto del usuario pueda filtrarse, sugiere una solución y proporciona el código actualizado.

<div id="troubleshooting">
  ### Solución de problemas
</div>

* Ayúdame a resolver un error de `refresh token doesn’t exist` al usar el intercambio de token de actualización. Usa las instrucciones de configuración de Token Vault como contexto.
* El botón de cierre de sesión no está cerrando la sesión en Auth0. Busca en la documentación la estructura correcta de la URL de cierre de sesión federado y actualiza mi controlador de cierre de sesión para usarla.
* Mi flujo de inicio de sesión está fallando con un error `401 Unauthorized`. Usa Auth0 Docs MCP Server para consultar las causas más comunes de este error con el SDK `auth0-node` y revisa mi `.env` y la inicialización del cliente para detectar discrepancias.
