  Para usar las funciones de Highly Regulated Identity, debes tener un plan Enterprise con el complemento Highly Regulated Identity. Consulta [Auth0 Pricing](https://auth0.com/pricing/) para obtener más detalles.
</Callout>

Para ayudar a los clientes a configurar su tenant de Auth0 de modo que cumpla con uno de los perfiles de Financial-grade API (FAPI), el modelo de aplicación incluye una propiedad `compliance_level` que se puede establecer en uno de tres valores:

* `null` o undefined: No se requiere ningún nivel de cumplimiento. Este es el valor predeterminado.
* `fapi1_adv_mtls_par`: El cliente quiere que este cliente se comporte de acuerdo con el perfil FAPI1 Advanced mediante [mTLS](/docs/es/get-started/applications/configure-mtls) y [PAR](/docs/es/get-started/applications/configure-par).
* `fapi1_adv_pkj_par`: El cliente quiere que este cliente se comporte de acuerdo con el perfil FAPI1 Advanced mediante [clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [PAR](/docs/es/get-started/applications/configure-par).
* `fapi2_sp_pkj_mtls`: El cliente quiere que este cliente se comporte de acuerdo con el perfil de seguridad de FAPI 2.0 mediante [clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).
* `fapi2_sp_mtls_mtls`: El cliente quiere que este cliente se comporte de acuerdo con el perfil de seguridad de FAPI 2.0 mediante [autenticación de cliente con mTLS](/docs/es/get-started/applications/configure-mtls) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).

Cumplir con un perfil FAPI requiere varios cambios de configuración. Establecer `compliance_level` garantiza que ninguna solicitud de autorización pueda completarse correctamente a menos que tanto la solicitud como la configuración cumplan con el estándar seleccionado.

Por ejemplo, tanto los niveles de cumplimiento `fapi1_adv_pkj_par` como `fapi1_adv_mtls_par` requieren PAR. Si se selecciona cualquiera de estos niveles de cumplimiento, PAR es obligatorio independientemente del valor de la configuración `require_pushed_authorization_requests`. Intentar una autorización sin usar PAR da como resultado la siguiente respuesta de error:

```json lines
{
  “error”: “invalid_request”,
  "error_description": "Las solicitudes de autorización enviadas son requeridas por el nivel de cumplimiento configurado" 
}
```

En algunos casos, definir un nivel de cumplimiento también cambia el comportamiento de Auth0. Por ejemplo, los niveles de cumplimiento `fapi1_adv_pkj_par` y `fapi1_adv_mtls_par` hacen que Auth0 incluya un claim `s_hash` en el <Tooltip tip="token de ID: Credencial destinada al propio cliente y no al acceso a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> devuelto, que contiene un hash SHA256 del valor de state. Esto permite que los tokens de ID actúen como una firma separada.

| Requiere PAR.                                                                                                                                                     | Y                   | Y                    | Y                   | Y                    |
| Requiere PKCE con el método de desafío S256.                                                                                                                      | Y                   | Y                    | Y                   | Y                    |
| Impide el uso de comodines en las devoluciones de llamada permitidas de un cliente.                                                                               | Y                   | Y                    | N                   | N                    |
| Exige el uso de JAR.                                                                                                                                              | Y                   | Y                    | N                   | N                    |
| Garantiza que el payload de JAR esté firmado con el algoritmo PS256.                                                                                              | Y                   | Y                    | N                   | N                    |
| Garantiza que el payload de JAR contenga la claim `nbf` y que no sea anterior en más de 60 minutos.                                                               | Y                   | Y                    | N                   | N                    |
| Garantiza que el payload de JAR contenga la claim `exp` y que no sea posterior en más de 60 minutos a la claim `nbf`.                                             | Y                   | Y                    | N                   | N                    |
| Garantiza que el cliente haya establecido la propiedad `oidc_conformant` en true.                                                                                 | Y                   | Y                    | Y                   | Y                    |
| Requiere el uso de headers `x-fapi-*`                                                                                                                             | Y                   | Y                    | N                   | N                    |

<div id="configure-fapi-compliance-for-a-client">
  ## Configurar el cumplimiento de FAPI para un cliente
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para hacerlo desde Auth0 Dashboard:

    1. Vaya a **Auth0 Dashboard &gt; Applications**.
    2. Seleccione la aplicación.
    3. Seleccione la pestaña **Application Settings**.
    4. Abra la sección **Advanced Settings**.
    5. En la pestaña **OAuth**, seleccione **FAPI Compliance Enforcement Level**.

    Las opciones para configurar el cumplimiento de FAPI son:

    * **None**: No se requiere ningún nivel de cumplimiento. Este es el valor predeterminado.
    * **FAPI 1 Advanced profile using Private Key JWT and PAR**: El cliente desea que esta aplicación se comporte de acuerdo con el perfil avanzado de FAPI 1 mediante [clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [PAR](/docs/es/get-started/applications/configure-par).
    * **FAPI 1 Advanced profile using mTLS and PAR**: El cliente desea que esta aplicación se comporte de acuerdo con el perfil avanzado de FAPI 1 mediante [mTLS](/docs/es/get-started/applications/configure-mtls) y [PAR](/docs/es/get-started/applications/configure-par).
    * **FAPI 2.0 Security Profile with Private Key JWT and certificate binding**: El cliente desea que esta aplicación se comporte de acuerdo con el perfil de seguridad de FAPI 2.0 mediante [autenticación de cliente con clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).
    * **FAPI 2.0 Security Profile with mTLS and certificate binding**: El cliente desea que esta aplicación se comporte de acuerdo con el perfil de seguridad de FAPI 2.0 mediante [autenticación de cliente con mTLS](/docs/es/get-started/applications/configure-mtls) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).
  </Tab>

  <Tab title="Management API">
    Use la [Management API](https://auth0.com/docs/api/management/v2) para establecer la propiedad `compliance_level` con una solicitud `POST` o `PATCH`:

    ```bash lines
    curl --location --request PATCH 'https://{YOUR_DOMAIN}/api/v2/clients/$client_id' \
      --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}' \
      --header 'Content-Type: application/json' \
      --data '{
        "compliance_level": "fapi1_adv_mtls_par"
    }'
    ```

    Para obtener la propiedad `compliance_level`, use una solicitud `GET`:

    ```bash lines
    curl --location 'https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID} \
      --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}'
    ```

    Para el cumplimiento de FAPI 2.0, puede configurar el tiempo de expiración de las solicitudes de autorización enviadas con un valor inferior a 600 segundos; el valor predeterminado es 30 segundos. Puede establecer este valor de expiración mediante la Management API.
  </Tab>
</Tabs>
