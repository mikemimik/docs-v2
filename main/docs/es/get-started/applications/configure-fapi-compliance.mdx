<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar las funcionalidades de Identidad altamente regulada, debes tener un Enterprise Plan con el complemento Identidad altamente regulada. Consulta [Auth0 Pricing](https://auth0.com/pricing/) para obtener más información.
</Callout>

Para ayudar a los clientes a configurar su tenant de Auth0 para que se ajuste a uno de los perfiles de Financial-grade API (FAPI), el modelo de aplicación incluye una propiedad `compliance_level` que puede establecerse en uno de estos tres valores:

* `null` o undefined: No se requiere ningún nivel de cumplimiento. Este es el valor predeterminado.
* `fapi1_adv_mtls_par`: El cliente desea que este cliente se comporte de acuerdo con el perfil FAPI1 Advanced mediante [mTLS](/docs/es/get-started/applications/configure-mtls) y [PAR](/docs/es/get-started/applications/configure-par).
* `fapi1_adv_pkj_par`: El cliente desea que este cliente se comporte de acuerdo con el perfil FAPI1 Advanced mediante [clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [PAR](/docs/es/get-started/applications/configure-par).
* `fapi2_sp_pkj_mtls`: El cliente desea que este cliente se comporte de acuerdo con el perfil de seguridad de FAPI 2.0 mediante [clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).
* `fapi2_sp_mtls_mtls`: El cliente desea que este cliente se comporte de acuerdo con el perfil de seguridad de FAPI 2.0 mediante [autenticación de cliente con mTLS](/docs/es/get-started/applications/configure-mtls) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).

Cumplir con un perfil FAPI requiere varios cambios de configuración. Establecer `compliance_level` garantiza que ninguna solicitud de autorización pueda completarse correctamente a menos que tanto la solicitud como la configuración cumplan con el estándar seleccionado.

Por ejemplo, tanto el nivel de cumplimiento `fapi1_adv_pkj_par` como `fapi1_adv_mtls_par` requieren PAR. Si se selecciona cualquiera de estos niveles de cumplimiento, PAR es obligatorio independientemente del valor de la configuración `require_pushed_authorization_requests`. Intentar una autorización sin usar PAR da como resultado la siguiente respuesta de error:

```json lines
{
  “error”: “invalid_request”,
  "error_description": "La solicitud de autorización push es requerida por el nivel de cumplimiento configurado" 
}
```

En algunos casos, establecer un nivel de cumplimiento también cambia el comportamiento de Auth0. Por ejemplo, los niveles de nivel de cumplimiento `fapi1_adv_pkj_par` y `fapi1_adv_mtls_par` hacen que Auth0 incluya un claim `s_hash` en el <Tooltip tip="token de ID: Credencial destinada al propio cliente y no a acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> devuelto, que contiene un hash SHA256 del valor de state. Esto permite que los tokens de ID actúen como una firma desacoplada.

<div id="configure-fapi-compliance-for-a-client">
  ## Configurar el cumplimiento de FAPI para un cliente
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para hacerlo desde el Auth0 Dashboard:

    1. Ve a **Auth0 Dashboard &gt; Applications**.
    2. Selecciona la aplicación.
    3. Selecciona la pestaña **Application Settings**.
    4. Abre la sección **Advanced Settings**.
    5. En la pestaña **OAuth**, selecciona **FAPI Compliance Enforcement Level**.

    Las opciones para configurar el cumplimiento de FAPI son:

    * **None**: No se requiere ningún nivel de cumplimiento. Este es el valor predeterminado.
    * **FAPI 1 Advanced profile using Private Key JWT and PAR**: Se desea que este cliente se comporte de conformidad con el perfil FAPI1 Advanced usando [clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [PAR](/docs/es/get-started/applications/configure-par).
    * **FAPI 1 Advanced profile using mTLS and PAR**: Se desea que este cliente se comporte de conformidad con el perfil FAPI1 Advanced usando [mTLS](/docs/es/get-started/applications/configure-mtls) y [PAR](/docs/es/get-started/applications/configure-par).
    * **FAPI 2.0 Security Profile with Private Key JWT and certificate binding**: Se desea que este cliente se comporte de conformidad con el perfil de seguridad de FAPI 2.0 usando [autenticación de cliente con clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).
    * **FAPI 2.0 Security Profile with mTLS and certificate binding**: Se desea que este cliente se comporte de conformidad con el perfil de seguridad de FAPI 2.0 usando [autenticación de cliente con mTLS](/docs/es/get-started/applications/configure-mtls) y [mTLS Token Sender-Constraining](/docs/es/secure/sender-constraining/mtls-sender-constraining).
  </Tab>

  <Tab title="Management API">
    Usa la [Management API](https://auth0.com/docs/api/management/v2) para establecer la propiedad `compliance_level` con una solicitud `POST` o `PATCH`:

    ```bash lines
    curl --location --request PATCH 'https://{YOUR_DOMAIN}/api/v2/clients/$client_id' \
      --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}' \
      --header 'Content-Type: application/json' \
      --data '{
        "compliance_level": "fapi1_adv_mtls_par"
    }'
    ```

    Para obtener la propiedad `compliance_level`, usa una solicitud `GET`:

    ```bash lines
    curl --location 'https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID} \
      --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}'
    ```

    Para el cumplimiento de FAPI 2.0, puedes configurar la caducidad de las solicitudes de autorización push con un valor inferior a 600 segundos; el valor predeterminado es 30 segundos. Puedes establecer el valor de caducidad con la Management API.
  </Tab>
</Tabs>
