Cuando registras una aplicación en Auth0, decides si es propia o de terceros según quién la posee y la opera.

* Aplicaciones propias: Son propiedad de tu organización y están operadas por ella. Tú controlas su despliegue, sus credenciales y su comportamiento.
* Aplicaciones de terceros: Son propiedad de una organización externa y están operadas por ella, como un partner, un desarrollador independiente o un agente de IA. Les concedes acceso a tus recursos, pero no puedes controlar directamente qué hacen con ese acceso.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  &quot;De terceros&quot; se refiere al control operativo, no a la autoría. Muchas organizaciones externalizan el desarrollo de sus propias aplicaciones. Por ejemplo, que un contractor desarrolle tu aplicación no la convierte en una aplicación de terceros. La distinción clave es: quién la despliega, quién tiene las credenciales y quién puede detenerla.
</Callout>

Confidencial/pública y propia/de terceros son clasificaciones independientes. Confidencial o pública describe la capacidad de autenticación de la aplicación (si puede almacenar un Secret). Propia o de terceros describe la relación de confianza (quién posee y opera la aplicación).

Tanto las aplicaciones propias como las de terceros pueden ser confidenciales (aplicación web tradicional) o públicas (SPA, Native). Una aplicación web tradicional de terceros es a la vez confidencial y de terceros.

<div id="first-party-applications">
  ## Aplicaciones propias
</div>

Las aplicaciones propias están controladas por la misma organización o persona que posee el dominio de Auth0. Por ejemplo, supongamos que creaste una API de Contoso y una aplicación que inicia sesión en `contoso.com` y consume la API de Contoso. Registrarías tanto la API como la aplicación en el mismo dominio de Auth0, y la aplicación sería una aplicación propia. De forma predeterminada, todas las aplicaciones creadas a través del [Auth0 Dashboard](https://manage.auth0.com/#/applications) son aplicaciones propias.

<div id="third-party-applications">
  ## Aplicaciones de terceros
</div>

Las aplicaciones de terceros están controladas por alguien que, con toda probabilidad, no debería tener acceso administrativo a tu dominio de Auth0. Las aplicaciones de terceros permiten que partes externas o partners accedan de forma segura a los recursos protegidos que están detrás de tu API.

Por ejemplo, si una empresa partner crea un dashboard de análisis de datos para visualizar información de tu servicio, primero debe registrar su aplicación en tu tenant de Auth0 para obtener un <Tooltip tip="ID de cliente: Valor de identificación que Auth0 asigna a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip> y un secreto. Aunque esta aplicación esté registrada en tu entorno, se considera de terceros porque el código y las credenciales pertenecen al partner y son gestionados por este, no por tu organización.

Todas las aplicaciones creadas mediante [registro dinámico de clientes](/docs/es/get-started/applications/dynamic-client-registration) son aplicaciones de terceros. Para obtener más información sobre las aplicaciones de terceros en Auth0, consulta [Third-Party Applications](/docs/es/get-started/applications/third-party-applications).

<div id="first-party-vs-third-party-in-auth0">
  ## Aplicaciones propias vs. de terceros en Auth0
</div>

La siguiente tabla resume las diferencias entre las aplicaciones propias y de terceros en Auth0:

|                                       | **Propias**                                         | **De terceros**                                                                                                                                                                                                                                            |
| ------------------------------------- | --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Acceso a la API**                   | Sigue la política de acceso configurada para la API | Siempre requiere una [autorización de cliente](/docs/es/get-started/applications/application-access-to-apis-client-grants) explícita                                                                                                                          |
| **API del sistema de Auth0**          | Accesibles en los flujos de usuario                 | No accesibles en los flujos de usuario                                                                                                                                                                                                                     |
| **Consentimiento del usuario**        | Puede omitirse (si está habilitado en la API)       | Siempre es obligatorio                                                                                                                                                                                                                                     |
| **Tipos de concesión**                | Todos los tipos de concesión compatibles            | `authorization_code`, `refresh_token` y `client_credentials`                                                                                                                                                                                               |
| **OIDC**                              | Compatible                                          | No compatible. Está previsto para una futura versión.                                                                                                                                                                                                      |
| **Rules**                             | Se ejecutan                                         | No compatible. Produce un error.                                                                                                                                                                                                                           |
| **Protocolos no OAuth** (SAML, WsFed) | Compatible                                          | No compatible                                                                                                                                                                                                                                              |
| **Organizations**                     | Compatible                                          | Acceso de máquina a máquina compatible mediante [autorizaciones de cliente de organización](/docs/es/manage-users/organizations/configure-organizations/configure-organization-client-grants). Los flujos de usuario están previstos para una futura versión. |
| **Formato del ID de cliente**         | Formato estándar                                    | Prefijo `tpc_`                                                                                                                                                                                                                                             |
| **Conexiones**                        | Todas las conexiones habilitadas                    | Conexiones a nivel de dominio                                                                                                                                                                                                                              |

Para obtener más información sobre las aplicaciones de terceros en Auth0, consulta [Third-Party Applications](/docs/es/get-started/applications/third-party-applications).

<div id="application-ownership">
  ## Propiedad de la aplicación
</div>

La propiedad de la aplicación se define en el momento de su creación y no puede cambiarse después. De forma predeterminada, las aplicaciones se crean como propias, lo que aplica configuraciones de seguridad menos restrictivas. Para garantizar que se apliquen los [controles de seguridad](/docs/es/get-started/applications/third-party-applications/security-controls) adecuados, debes identificar correctamente como de terceros las aplicaciones que pertenezcan a partes externas al crearlas desde el Auth0 Dashboard o la Management API. Para saber cómo hacerlo, consulta [Configurar aplicaciones de terceros](/docs/es/get-started/applications/third-party-applications/configure-third-party-applications).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La propiedad de la aplicación es inmutable. No puedes convertir una aplicación de terceros en una aplicación propia ni viceversa.
</Callout>

<div id="check-application-ownership">
  ## Verifica la propiedad de la aplicación
</div>

Para verificar si una aplicación es propia o de terceros:

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Ve a **Aplicaciones &gt; Aplicaciones**.
    2. Selecciona la aplicación. Las aplicaciones de terceros muestran una insignia que indica que pertenecen a terceros.

    <Frame>![Configuración de la aplicación en el Dashboard que muestra la insignia de terceros](/docs/images/third-party-applications/third-party-badge.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Realiza una llamada `GET` al [endpoint Get a Client](https://auth0.com/docs/api/management/v2#!/Clients/get_clients_by_id). Asegúrate de reemplazar los valores de marcador de posición `{YOUR_CLIENT_ID}`
    y `{YOUR_MANAGEMENT_API_ACCESS_TOKEN}` con tu <Tooltip tip="Client ID: valor de identificación asignado a tu recurso registrado por Auth0." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip> y el <Tooltip tip="Access Token: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de la Management API, respectivamente.

    <AuthCodeGroup>
      ```bash cURL
      curl --request GET \
        --url 'https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}?fields=is_first_party&include_fields=true' \
        --header 'authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}'
      ```

      ```csharp C#
      var client = new RestClient("https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}?fields=is_first_party&include_fields=true");
      var request = new RestRequest(Method.GET);
      request.AddHeader("authorization", "Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}");
      IRestResponse response = client.Execute(request);
      ```

      ```go Go
      package main

      import (
      	"fmt"
      	"net/http"
      	"io/ioutil"
      )

      func main() {

      	url := "https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}?fields=is_first_party&include_fields=true"

      	req, _ := http.NewRequest("GET", url, nil)

      	req.Header.Add("authorization", "Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}")

      	res, _ := http.DefaultClient.Do(req)

      	defer res.Body.Close()
      	body, _ := ioutil.ReadAll(res.Body)

      	fmt.Println(res)
      	fmt.Println(string(body))

      }
      ```

      ```java Java
      HttpResponse response = Unirest.get("https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}?fields=is_first_party&include_fields=true")
        .header("authorization", "Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}")
        .asString();
      ```

      ```javascript Node.JS
      var axios = require("axios").default;

      var options = {
        method: 'GET',
        url: 'https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}',
        params: {fields: 'is_first_party', include_fields: 'true'},
        headers: {authorization: 'Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}'}
      };

      axios.request(options).then(function (response) {
        console.log(response.data);
      }).catch(function (error) {
        console.error(error);
      });
      ```

      ```php PHP
      $curl = curl_init();

      curl_setopt_array($curl, [
        CURLOPT_URL => "https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}?fields=is_first_party&include_fields=true",
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_ENCODING => "",
        CURLOPT_MAXREDIRS => 10,
        CURLOPT_TIMEOUT => 30,
        CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
        CURLOPT_CUSTOMREQUEST => "GET",
        CURLOPT_HTTPHEADER => [
          "authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}"

      $response = curl_exec($curl);
      $err = curl_error($curl);

      curl_close($curl);

      if ($err) {
        echo "cURL Error #:" . $err;

        echo $response;
      }
      ```

      ```python Python
      import http.client

      conn = http.client.HTTPSConnection("")

      headers = { 'authorization': "Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}" }

      conn.request("GET", "/{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}?fields=is_first_party&include_fields=true", headers=headers)

      res = conn.getresponse()
      data = res.read()

      print(data.decode("utf-8"))
      ```

      ```ruby Ruby
      require 'uri'
      require 'net/http'
      require 'openssl'

      http.use_ssl = true
      http.verify_mode = OpenSSL::SSL::VERIFY_NONE

      request = Net::HTTP::Get.new(url)
      request["authorization"] = 'Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}'

      response = http.request(request)
      puts response.read_body
      ```

    | Valor                              | Descripción                                                                                                                                        |
    | ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `YOUR_CLIENT_ID`                   | El ID de la aplicación que se va a actualizar.                                                                                                     |
    | `YOUR_MANAGEMENT_API_ACCESS_TOKEN` | [Tokens de acceso de la Management API](https://auth0.com/docs/api/management/v2/tokens) con el [scope](/docs/es/glossary?term=scope) `read:clients`. |

    Si la aplicación es de primera parte, el campo `is_first_party` tendrá el valor `true`. Si la aplicación es de terceros, el campo `is_first_party` tendrá el valor `false`.
