---

En la página [Applications](http://manage.auth0.com/#/applications) del Dashboard, busca tu aplicación en la lista y haz clic en su nombre para ver la configuración disponible. Luego, ve a la pestaña **Credentials**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Es posible que la pestaña Credentials no aparezca en tu tenant si tienes una aplicación pública. Para obtener más información sobre las aplicaciones confidenciales y las aplicaciones públicas, consulta [Aplicaciones confidenciales y públicas](/docs/es/get-started/applications/confidential-and-public-applications).
</Callout>

<div id="authentication-methods">
  ## Métodos de autenticación
</div>

Auth0 ofrece las siguientes formas para que su aplicación se autentique con el <Tooltip tip="Servidor de autorización: Servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, su servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>:

* **Clave privada <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>**: Un método de autenticación asimétrico para aplicaciones confidenciales. Con clave privada JWT, usted genera un par de claves, una pública y otra privada, para usarlas como credenciales. Usted proporciona la clave pública y almacena de forma segura la clave privada en su propio sistema sin compartirla con Auth0. Usa la clave privada para firmar la solicitud enviada al servidor de autorización. Puede administrar las claves públicas a continuación, en la sección **Credenciales disponibles**.
* **<Tooltip tip="Secreto de cliente: Secret utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">Secreto de cliente</Tooltip>:** Un método de autenticación simétrico. En la autenticación con secreto de cliente, usted proporciona el secreto de cliente que Auth0 le asignó al crear la aplicación. Puede ver y copiar el `client_secret` directamente desde esta sección.

  * **Secreto de cliente (Basic)**: Use Basic para una aplicación confidencial que utiliza el esquema de autenticación `HTTP BASIC` para enviar un secreto de cliente.
  * **Secreto de cliente (Post)**: Use Post para una aplicación confidencial que utiliza parámetros del cuerpo de la solicitud para enviar un secreto de cliente.

Para obtener más información sobre los métodos de autenticación y las credenciales asociadas, lea [Credenciales de la aplicación](/docs/es/secure/application-credentials).

<div id="available-credentials">
  ## Credenciales disponibles
</div>

<Frame>![](/docs/images/cdy7uua7fh8z/7vzjXXZ0A9pDj5qsXmOTwh/b4b299c6316216376f7e2858007df0b1/Private_Key_JWT_-_English.png)</Frame>

**Agregar nueva credencial**: Agrega una nueva entrada a la lista si aún no alcanzaste el número máximo de credenciales (2) para la aplicación.

* **Nombre**: Nombre de la credencial. Por ejemplo: 4096.
* **ID de la clave**: Identificador único de la credencial generado por Auth0. La misma credencial no se puede cargar más de una vez.
* **Algoritmo**: Algoritmo que seleccionas para cada credencial. Admitimos RS256, PS256 y RS384.
* **Vence el**: `datetime` en el que la credencial deja de ser válida para el uso declarado. La credencial no se eliminará, pero quedará inoperativa. Esto se puede configurar en la interfaz de usuario habilitando la casilla **Establecer una fecha de vencimiento explícita para esta credencial**.

Usa el menú lateral para habilitar, deshabilitar o eliminar credenciales.
