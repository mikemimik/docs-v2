title: Configuración de credenciales
---

En la página [Applications](http://manage.auth0.com/#/applications) del Dashboard, busca tu aplicación en la lista y haz clic en su nombre para ver la configuración disponible. Luego, cambia a la pestaña **Credentials**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Es posible que la pestaña Credentials no esté visible en tu tenant si tienes una aplicación pública. Para obtener más información sobre las aplicaciones confidenciales y las aplicaciones públicas, consulta [Confidential and Public Applications](/docs/es/get-started/applications/confidential-and-public-applications).
</Callout>

<div id="authentication-methods">
  ## Métodos de autenticación
</div>

Auth0 ofrece las siguientes formas en que tu aplicación puede autenticarse con el <Tooltip tip="Servidor de autorización: Servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>:

* **Clave privada <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, con frecuencia, de token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>**: Un método de autenticación asimétrico para aplicaciones confidenciales. En clave privada JWT, generas un par de claves, una pública y otra privada, para usarlas como credenciales. Proporcionas la clave pública y almacenas de forma segura la clave privada en tu propio sistema sin compartirla con Auth0. Usas la clave privada para firmar la solicitud enviada al servidor de autorización. Puedes administrar las claves públicas a continuación, en la sección **Credenciales disponibles**.
* **<Tooltip tip="Secreto de cliente: Secret usado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">Secreto de cliente</Tooltip>:** Un método de autenticación simétrico. En la autenticación con secreto de cliente, proporcionas el secreto de cliente que Auth0 asignó cuando creaste la aplicación. Puedes ver y copiar el `client_secret` directamente desde esta sección.

  * **Secreto de cliente (Basic)**: Usa Basic para una aplicación confidencial que utiliza el esquema de autenticación `HTTP BASIC` para enviar un secreto de cliente.
  * **Secreto de cliente (Post)**: Usa Post para una aplicación confidencial que utiliza parámetros del cuerpo de la solicitud para enviar un secreto de cliente.

Para obtener más información sobre los métodos de autenticación y las credenciales asociadas, consulta [Credenciales de la aplicación](/docs/es/secure/application-credentials).

<div id="available-credentials">
  ## Credenciales disponibles
</div>

<Frame>![](/docs/images/cdy7uua7fh8z/7vzjXXZ0A9pDj5qsXmOTwh/b4b299c6316216376f7e2858007df0b1/Private_Key_JWT_-_English.png)</Frame>

**Agregar nueva credencial**: Agrega una nueva entrada a la lista si aún no has alcanzado el número máximo de credenciales (2) para la aplicación.

* **Nombre**: Nombre de la credencial. Por ejemplo: 4096.
* **ID de la clave**: Identificador único de la credencial generado por Auth0. La misma credencial no se puede cargar más de una vez.
* **Algoritmo**: Algoritmo que seleccionas para cada credencial. Admitimos RS256, PS256 y RS384.
* **Expira el**: `datetime` en el que la credencial deja de ser válida para el uso indicado. La credencial no se eliminará, pero quedará inoperativa. Esto se puede configurar en la UI activando la casilla **Establecer una fecha de vencimiento explícita para esta credencial**.

Selecciona el menú lateral para habilitar, deshabilitar o eliminar credenciales.
