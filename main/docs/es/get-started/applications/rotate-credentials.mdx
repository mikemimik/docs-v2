---
description: Describe cómo rotar las credenciales existentes en el Auth0 Dashboard.
title: Rotar credenciales
---

Auth0 recomienda rotar el material de claves con regularidad para cumplir sus requisitos de cumplimiento y garantizar que la seguridad no se vea comprometida por claves privadas filtradas. Puede usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para poner en uso nuevas claves. Debe crear una nueva credencial, asociarla con el método de autenticación `private_key_jwt` y eliminar las credenciales antiguas o que ya no use.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El límite actual de almacenamiento de la aplicación es de dos credenciales al mismo tiempo. Para rotar nuevas credenciales de forma periódica, debe eliminar las credenciales sin usar.
</Callout>

<Tabs>
  <Tab title="Usar Auth0 Dashboard">
    Para rotar las credenciales de tu aplicación con Auth0 Dashboard:

    1. Ve a [**Auth0 Dashboard &gt; Applications &gt; Application**](https://manage.auth0.com/#/applications) y selecciona la aplicación que quieres actualizar.
    2. Ve a la pestaña **Credentials**.
    3. En la sección **Available Credentials**, selecciona **Add New Key**.
    4. Define un nombre para la nueva credencial, la clave pública en formato PEM y el algoritmo de la nueva credencial.
    5. Selecciona **Add Credential**.
    6. Para activar la nueva credencial, ve al menú de la credencial y elige **Enable for Private Key JWT use**.
    7. Una vez que hayas actualizado tus aplicaciones para usar la nueva credencial, desactiva la credencial original:

       1. Selecciona **Disable for Private Key JWT Use**.
       2. Una vez desactivada, vuelve al menú de la credencial y selecciona **Delete Credential**.
  </Tab>

  <Tab title="Usar Management API">
    En los siguientes ejemplos de rotación, `credential1` es una credencial existente en uso y `credential2` es una credencial nueva que reemplazará a la existente.

    1. Genera un nuevo par de claves.

    2. Crea el recurso de credencial con una solicitud `POST` a la Management API.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         El límite actual de almacenamiento de la aplicación es de dos credenciales al mismo tiempo. Para rotar credenciales nuevas de forma repetida, debes eliminar las credenciales que no uses.
       </Callout>

    3. Haz una solicitud PATCH al endpoint [actualizar un cliente](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id) de la Management API para asociar la credencial con el método de autenticación `private_key_jwt`:

       ```bash lines
       curl --location --request PATCH 'https://{domain}/api/v2/clients/{clientId} \
         --header 'Authorization: Bearer {managementApiAccessToken} \
         --header 'Content-Type: application/json' \
         --data-raw '{
                 "client_authentication_methods": {
                    "private_key_jwt": {
                       "credentials": [{ "id": {credentialId1} }, { "id": {credentialId2} }]
                    }
                 }
          }'
       ```

       | Parámetro                  | Descripción                                                                                                                                                      |
       | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
       | `clientId`                 | Aplicación que quieres actualizar.                                                                                                                               |
       | `credentialId1`            | ID de la credencial existente en uso.                                                                                                                            |
       | `credentialId2`            | ID de la nueva credencial.                                                                                                                                       |
       | `managementApiAccessToken` | [token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con los scopes `update:clients` y `update:credentials`. |

    4. Actualiza tu aplicación para usar la nueva clave privada para firmar aserciones para la Auth0 Authentication API.

    5. Elimina de tu aplicación la clave que ya no uses.

       ```bash lines
       curl --location --request PATCH 'https://{domain}/api/v2/clients/{clientId} \
         --header 'Authorization: Bearer $management_access_token' \
         --header 'Content-Type: application/json' \
         --data-raw '{
                 "client_authentication_methods": {
                    "private_key_jwt": {
                       "credentials": [{ "id": {credentialId2} }]
                    }
                 }
          }'
       ```

    6. Elimina de tu aplicación la clave que ya no uses. Esto eliminará permanentemente la credencial del almacenamiento. Debes desasociar la credencial de tu aplicación; de lo contrario, no podrás eliminarla.

       ```bash lines
       curl --location --request DELETE 'https://{domain}/api/v2/clients/{clientId}/credentials/{credentialId}' \
         --header 'Authorization: Bearer {managementApiAccessToken}
       ```

       | Parámetro                  | Descripción                                                                                                                                  |
       | -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
       | `clientId`                 | Aplicación que quieres actualizar.                                                                                                           |
       | `credentialId`             | ID de la credencial anterior que quieres eliminar.                                                                                           |
       | `managementApiAccessToken` | [token de acceso para la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope ` delete:credentials`. |
  </Tab>
</Tabs>

<div id="active-credentials">
  ### Credenciales activas
</div>

Para garantizar que no haya tiempo de inactividad, puedes dejar varias credenciales activas durante la rotación. Las aplicaciones pueden seguir funcionando con normalidad con claves anteriores hasta que se actualicen las claves. Las aplicaciones pueden enviar aserciones firmadas con cualquiera de los conjuntos de credenciales activas.

Auth0 recomienda minimizar el tiempo durante el cual se usan varias credenciales. En el siguiente ejemplo se usan varias credenciales asociadas:

```bash lines
curl --location --request PATCH 'https://$tenant/api/v2/clients/$client_id' \
  --header 'Authorization: Bearer $management_access_token' \
  --header 'Content-Type: application/json' \
  --data-raw '{
          "client_authentication_methods": {
             "private_key_jwt": {
                "credentials": [{ "id": $credential1.id }, { "id": $credential2.id }]
             }
          }
 }'
```
