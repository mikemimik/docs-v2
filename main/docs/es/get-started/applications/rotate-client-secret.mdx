Puedes cambiar el <Tooltip tip="Secreto de cliente: secreto que utiliza un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip> de una aplicación desde el <Tooltip tip="Secreto de cliente: secreto que utiliza un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0. Cuando rotas un secreto de cliente, debes actualizar todas las aplicaciones autorizadas con el nuevo valor.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los secretos de cliente no deben almacenarse en aplicaciones cliente públicas. Para obtener más información, lee [Confidential and Public Applications.](/docs/es/get-started/applications/confidential-and-public-applications)
</Callout>

<Warning>
  Los secretos nuevos pueden tardar hasta treinta segundos en estar disponibles mientras se rotan. Para minimizar el tiempo de inactividad, te sugerimos guardar el nuevo secreto de cliente en el código o en la configuración del sistema de tu aplicación como respaldo del secreto anterior. De esta forma, si la solicitud de la aplicación cliente no funciona con el secreto anterior, tu aplicación usará el nuevo secreto.

  Los secretos pueden almacenarse en una lista (o en una estructura similar) hasta que ya no sean necesarios. Una vez que tengas la certeza de que un secreto antiguo ha quedado obsoleto, puedes eliminar su valor del código de tu aplicación.
</Warning>

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. En el Auth0 Dashboard, ve a [Applications &gt; Applications](https://manage.auth0.com/#/applications) y selecciona el nombre de la aplicación que quieres ver.

   <Frame>![Lista de aplicaciones del Dashboard](/docs/images/cdy7uua7fh8z/1ecNwGgFQZxdP57p0tp3jT/cd608fcfae22e195b604e2707e5a848d/App_List_-_EN.png)</Frame>
2. Desplázate hasta la parte inferior de la página **Settings**, busca la **Danger Zone**, selecciona **Rotate** y confirma.
3. Desplázate hasta la parte superior de la página y cambia a la pestaña **Credentials**.
4. Consulta tu nuevo secreto buscando **Client Secret** y seleccionando el icono del ojo.

   <Frame>![Pestaña de información básica de Settings de la aplicación en Dashboard Applications](/docs/images/cdy7uua7fh8z/2GPUw7BODYuYYH3658Upz3/92a49ec57e6b4d07be96093989baac03/2023-04-11_15-34-58.png)</Frame>
5. Actualiza las aplicaciones autorizadas con el nuevo valor.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

1. Llama al endpoint de la Management API [Rotar el secreto de un cliente](https://auth0.com/docs/api/management/v2#!/Clients/post_rotate_secret). Reemplaza los valores de marcador de posición `{yourClientId}` y `MGMT_API_ACCESS_TOKEN` por tu ID de cliente y tu token de acceso de la Management API, respectivamente.

<div id="set-a-custom-client-secret">
  ### Establecer un secreto de cliente personalizado
</div>

Puedes usar el endpoint [actualizar un cliente](https://auth0.com/docs/api/management/v2/#!/Clients/patch_clients_by_id) de la Management API para establecer manualmente un secreto de cliente, en lugar de solicitar la rotación a un secreto generado automáticamente. Tu aplicación se configura con el secreto futuro como respaldo antes de que se realice la rotación.

```bash lines
   curl --request PATCH \
   --url https://{TenantDomain}/api/v2/clients/{ClientID} \
   --header 'Authorization: Bearer {AccessToken}' \
   --header 'Content-Type: application/json' \
   --data '{
      "client_secret": "{CustomClientSecret}"
      }'
```
