---
title: "Revocar el acceso a las API mediante concesiones de aplicación"
---

Configurar tu concesión de aplicación para que los tokens que emites duren solo un breve período significa que, cuando llegue el momento de revocar el acceso a un recurso protegido, puedes simplemente eliminar la concesión. En ese momento, la parte que tiene el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> solo dispone de un tiempo limitado entre el momento en que eliminas la concesión y la expiración del token para realizar solicitudes adicionales a la API. Como esta es una opción fácil (y segura) de implementar, te recomendamos denegar el acceso a tus API y otros recursos protegidos revocando las concesiones de aplicación.

Si, por ejemplo, estás usando una aplicación de máquina a máquina [aplicación](/docs/es/get-started/applications) para acceder a tu [API](/docs/es/get-started/apis) y tienes un partner que hace llamadas a tu API, y al final de tu contrato actual, tú y tu partner deciden no renovar la colaboración. Ahora quieres quitarle a tu partner el acceso a tu API. Sin embargo, el problema es que le has dado a tu partner un token de acceso que dura un mes.

* ¿Qué puedes hacer en esta situación?
* ¿Cómo podrías configurar tu entorno de Auth0 para que este tipo de situaciones sean más fáciles de gestionar en el futuro?

<div id="application-grants">
  ## Concesiones de aplicación
</div>

El principal problema en este escenario es el tiempo durante el que el token de acceso a la API sigue siendo válido: un mes.

De forma predeterminada, Auth0 emite tokens de acceso con una duración de 24 horas. Establecer la duración del token en 24 horas significa que tu partner debe repetir el intercambio de credenciales de cliente (o cualquier otra concesión que hayas implementado) para obtener un nuevo token de acceso cada 24 horas. Para denegar el acceso a tu partner cuando venza tu contrato, basta con eliminar la concesión de aplicación para que, cuando caduque el token que ya tiene, no pueda solicitar uno nuevo.

Puedes cambiar la duración de un token configurando la opción `token_lifetime`. La duración concreta adecuada para tu caso de uso variará, pero te recomendamos establecer este valor lo más corto posible. Un buen punto de partida para determinarlo sería el margen de tiempo que consideres aceptable entre la eliminación de la concesión y el último uso de la API.

<div id="delete-an-application-grant">
  ### Eliminar una concesión de aplicación
</div>

Para eliminar una concesión de aplicación, realiza la llamada `DELETE` correspondiente al <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> [endpoint para eliminar una concesión de aplicación](https://auth0.com/docs/api/management/v2#!/Client_Grants/delete_client_grants_by_id). Como parte de la llamada, tendrás que especificar el ID de la concesión de aplicación que quieres eliminar, que puedes obtener a través del [endpoint para obtener todas las concesiones de aplicación](https://auth0.com/docs/api/management/v2#!/Client_Grants/get_client_grants) de la Management API.

También puedes [actualizar los tipos de concesión de una aplicación](/docs/es/get-started/applications/update-grant-types) desde el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.
