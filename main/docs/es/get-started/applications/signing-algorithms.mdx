title: Algoritmos de firma
---

<Tooltip tip="Algoritmo de firma: Algoritmo utilizado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=Signing+algorithms">Los algoritmos de firma</Tooltip> se usan para firmar los tokens emitidos para tu aplicación o API. Una firma forma parte de un [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens) y se utiliza para verificar que el remitente del token es quien dice ser y para garantizar que el mensaje no se haya modificado durante la transmisión.

Puedes seleccionar entre los siguientes algoritmos de firma:

* **RS256 (RSA Signature with SHA-256)**: Un algoritmo asimétrico, lo que significa que hay dos claves: una clave pública y una clave privada que debe mantenerse en secreto. Auth0 tiene la clave privada que se usa para generar la firma, y el consumidor del <Tooltip tip="JSON Web Token (JWT): Formato estándar del token de ID (y, con frecuencia, del token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> obtiene una clave pública de los endpoints de metadatos proporcionados por Auth0 y la usa para validar la firma del JWT.
* **HS256 (HMAC with SHA-256)**: Un algoritmo simétrico, lo que significa que solo hay una clave privada que debe mantenerse en secreto y que se comparte entre las dos partes. Como la misma clave se usa tanto para generar la firma como para validarla, debes asegurarte de que la clave no se vea comprometida. Esta clave privada (o secreto) se crea cuando registras tu aplicación (<Tooltip tip="Secreto de cliente: Secreto que utiliza un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>) o API (secreto de firma) y eliges el algoritmo de firma HS256.
* **PS256 (RSA Signature with SHA-256)**: Un algoritmo asimétrico, lo que significa que hay dos claves: una clave pública y una clave privada que debe mantenerse en secreto. Auth0 tiene la clave privada que se usa para generar la firma, y el consumidor del JWT obtiene una clave pública de los endpoints de metadatos proporcionados por Auth0 y la usa para validar la firma del JWT. A diferencia de RS256, el mismo header y payload del JWT generarán una firma distinta cada vez.

La práctica más segura, y nuestra recomendación, es usar RS256 porque:

* Con RS256, tienes la certeza de que solo quien posee la clave privada (Auth0) puede firmar tokens, mientras que cualquiera puede comprobar si el token es válido usando la clave pública.
* Con RS256, si la clave privada se ve comprometida, puedes implementar la rotación de claves sin tener que volver a desplegar tu aplicación o API con el nuevo secreto (algo que sí tendrías que hacer si usaras HS256).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si estás configurando la autenticación con clave privada JWT con una conexión empresarial de Okta u OpenID Connect (OIDC), es posible que tengas que elegir [algoritmos de firma adicionales](/docs/es/authenticate/enterprise-connections/private-key-jwt-client-auth#configure-private-key-jwt-client-authentication).
</Callout>
