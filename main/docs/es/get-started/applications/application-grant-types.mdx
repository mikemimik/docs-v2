---
description: Describe los tipos de concesión y cómo se relacionan con las aplicaciones.
title: Tipos de concesión de aplicaciones
---

Los tipos de concesión de aplicaciones (o flujos) son métodos mediante los cuales las aplicaciones pueden obtener [Tokens de acceso](/docs/es/secure/tokens/access-tokens) y con los que usted concede acceso limitado a sus recursos a otra entidad sin exponer credenciales. El [protocolo OAuth 2.0](/docs/es/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use) admite varios tipos de concesión, que permiten distintos tipos de acceso.

Según las necesidades de su aplicación, algunos tipos de concesión son más adecuados que otros. Auth0 ofrece muchos flujos diferentes de autenticación y <Tooltip tip="Flujo de autorización: concesión de autorización (o flujo de trabajo) especificada en el marco de OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flows">flujos de autorización</Tooltip> y le permite indicar qué tipos de concesión son adecuados según la propiedad `grant_types` de su aplicación.

Por ejemplo, si desea proteger una aplicación móvil, el [flujo de código de autorización con Proof Key for Code Exchange (PKCE)](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) es el más adecuado.

Como alternativa, si desea proteger una aplicación del lado del cliente, como una aplicación de página única (SPA), y no está transfiriendo tokens entre servidores, el [flujo implícito con Form Post](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post) es el más adecuado.

Varios tipos de concesión son válidos al registrar aplicaciones. Estos pueden dividirse en las siguientes categorías:

* **Concesiones conformes con las especificaciones**: concesiones definidas por especificaciones externas y conformes con ellas, como <Tooltip tip="OpenID: Estándar abierto de autenticación que permite que las aplicaciones verifiquen las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC).
* **Concesiones de extensión de Auth0**: concesiones específicas de Auth0 que se ajustan al [mecanismo de extensión de OAuth](https://tools.ietf.org/html/rfc6749#section-4.5) para admitir clientes adicionales o proporcionar un puente entre <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip> y otros marcos de confianza.
* **Concesiones heredadas de Auth0**: tipos de concesión tradicionales compatibles solo con clientes heredados. Si usted es un cliente heredado, le recomendamos encarecidamente migrar a una alternativa más segura.

<div id="specification-conforming-grants">
  ### Concesiones conformes con las especificaciones
</div>

| Tipo de concesión                              | Descripción                                                                                                                                 |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `implicit`                                     | [Concesión implícita](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post)                                 |
| `authorization_code`                           | [Concesión Authorization Code](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow)                             |
| `client_credentials`                           | [Concesión de credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow)                     |
| `password`                                     | [Concesión de contraseña del propietario del recurso](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow) |
| `refresh_token`                                | [Usar tokens de actualización](/docs/es/secure/tokens/refresh-tokens/use-refresh-tokens)                                                       |
| `urn:ietf:params:oauth:grant-type:device_code` | [Concesión de autorización de dispositivo](/docs/es/get-started/authentication-and-authorization-flow/device-authorization-flow)               |

<div id="auth0-extension-grants">
  ### Concesiones de extensión de Auth0
</div>

| **Tipo de concesión**                                     | **Descripción**                                                                                                                                      |
| --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`http://auth0.com/oauth/grant-type/password-realm`**    | Usa una concesión de extensión similar a la concesión de contraseña del propietario del recurso, con la posibilidad de indicar un `realm` específico |
| **`http://auth0.com/oauth/grant-type/mfa-oob`**           | Solicitud de concesión OOB para autenticación multifactor                                                                                            |
| **`http://auth0.com/oauth/grant-type/mfa-otp`**           | Solicitud de concesión OTP para autenticación multifactor                                                                                            |
| **`http://auth0.com/oauth/grant-type/mfa-recovery-code`** | Solicitud de concesión de recuperación para autenticación multifactor                                                                                |
| **`http://auth0.com/oauth/grant-type/passwordless/otp`**  | Solicitud de concesión para inicio de sesión passwordless integrado                                                                                  |

<div id="auth0-legacy-grants">
  ### Concesiones heredadas de Auth0
</div>

Las concesiones heredadas incluyen:

* `http://auth0.com/oauth/legacy/grant-type/ro`
* `http://auth0.com/oauth/legacy/grant-type/ro/jwt-bearer`
* `http://auth0.com/oauth/legacy/grant-type/delegation/refresh_token`
* `http://auth0.com/oauth/legacy/grant-type/delegation/id_token`
* `http://auth0.com/oauth/legacy/grant-type/access_token`

Los tipos de concesión heredados son tipos de concesión tradicionales compatibles únicamente para clientes heredados. Si eres un cliente heredado, te recomendamos encarecidamente que migres a una alternativa más segura.

A partir del 8 de junio de 2017, a todas las aplicaciones se les asignó una propiedad `grant_types` que debe completarse. Para evitar cambios en la funcionalidad para los clientes de Auth0 en ese momento, completamos la propiedad `grant_types` de todas las aplicaciones existentes con todos los tipos de concesión heredados de Auth0, de extensiones de Auth0 y conformes con la especificación.

En ese momento, los nuevos clientes de Auth0 ya no podían añadir tipos de concesión heredados a sus aplicaciones. Los tipos de concesión heredados solo están disponibles para clientes anteriores mientras migran a nuevos flujos, a fin de evitar cambios incompatibles. Si eras cliente antes del 8 de junio de 2017, puedes [habilitar un tipo de concesión heredado](/docs/es/get-started/applications/update-grant-types) usando el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0.

Si actualmente estás usando un tipo de concesión heredado, consulta la tabla siguiente para ver cuál de las alternativas seguras deberías usar en su lugar. Por ejemplo, si estás implementando la <Tooltip tip="Passwordless: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">autenticación Passwordless</Tooltip>,

usa [Universal Login](/docs/es/authenticate/login/auth0-universal-login) en lugar del endpoint `oauth/ro`.

<div id="grant-type-mapping">
  ## Mapeo de tipos de concesión
</div>

Una vez registradas, las aplicaciones tienen acceso a distintos tipos de concesión según su tipo de aplicación, en concreto, si la aplicación es confidencial o pública. Además, las aplicaciones propias de confianza tienen acceso a tipos de concesión adicionales.

<div id="public-applications">
  ### Aplicaciones públicas
</div>

Cuando se registra una Aplicación nativa o una Aplicación de página única (SPA) en el Dashboard, se marca automáticamente como aplicación pública, lo que se refleja en que el indicador `token_endpoint_auth_method` se establece en `none`. De forma predeterminada, Auth0 crea aplicaciones públicas con los siguientes `grant_types` habilitados:

* `implicit`
* `authorization_code`
* `refresh_token`

Las aplicaciones nativas también pueden usar el tipo de concesión `device_code`.

Las aplicaciones públicas no pueden usar el tipo de concesión `client_credentials`. Para usar este tipo de concesión, debes configurar la aplicación como confidencial en lugar de pública. Usa el endpoint de Auth0 Management API [actualizar un cliente](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id) para establecer `token_endpoint_auth_method` en `client_secret_post,` `client_secret_basic` o `private_key_jwt`. Para obtener más información, consulta [Credenciales de la aplicación](/docs/es/secure/application-credentials).

<div id="confidential-applications">
  ### Aplicaciones confidenciales
</div>

Cuando se registra una Regular Web Application o una Aplicación de máquina a máquina (M2M) en el Auth0 Dashboard, queda marcada automáticamente como una aplicación confidencial, lo que se refleja en que el indicador `token_endpoint_auth_method` está configurado con cualquier valor distinto de `none`. De forma predeterminada, Auth0 crea aplicaciones confidenciales con los siguientes `grant_types` habilitados:

* `implicit`
* `authorization_code`
* `refresh_token`
* `client_credentials`

<div id="trusted-first-party-applications">
  ### Aplicaciones propias de confianza
</div>

Las aplicaciones propias de confianza tienen habilitados los mismos `grant_types` que las aplicaciones confidenciales, además de los siguientes:

* `password`
* `http://auth0.com/oauth/grant-type/password-realm`
* `http://auth0.com/oauth/grant-type/mfa-oob`
* `http://auth0.com/oauth/grant-type/mfa-otp`
* `http://auth0.com/oauth/grant-type/mfa-recovery-code`

Si usas el Dashboard para habilitar o deshabilitar estos tipos de concesión, ten en cuenta que todos los tipos de concesión de Password y <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip> se habilitan al agregar el tipo de concesión `Password` o `MFA` a tu aplicación. No puedes seleccionarlos por separado.

<div id="third-party-application-restrictions">
  ## Restricciones de las aplicaciones de terceros
</div>

Las [aplicaciones de terceros](/docs/es/get-started/applications/third-party-applications) admiten un conjunto limitado de tipos de concesión, en línea con las [prácticas recomendadas de OAuth 2.1](https://datatracker.ietf.org/doc/html/draft-ietf-oauth-v2-1):

| **Tipo de concesión**                          | **Disponible para aplicaciones de terceros** |
| ---------------------------------------------- | -------------------------------------------- |
| `authorization_code`                           | Sí (PKCE obligatorio)                        |
| `refresh_token`                                | Sí                                           |
| `client_credentials`                           | Sí (solo para clientes confidenciales)       |
| `implicit`                                     | No                                           |
| `password`                                     | No                                           |
| `urn:ietf:params:oauth:grant-type:device_code` | No                                           |
| Concesiones de MFA                             | No                                           |
| Concesiones heredadas                          | No                                           |

Para obtener más información, consulta [Controles de seguridad para aplicaciones de terceros](/docs/es/get-started/applications/third-party-applications/security-controls).
