  Para usar las funcionalidades de Client-Initiated Backchannel Authentication (CIBA), debes tener un Enterprise Plan o un add-on adecuado. Consulta [Auth0 Pricing](https://auth0.com/pricing/) para más detalles.
</Callout>

El [flujo de Client-Initiated Backchannel Authentication (CIBA)](/docs/es/get-started/authentication-and-authorization-flow/client-initiated-backchannel-authentication-flow) es un flujo de autenticación y autorización desacoplado definido por la OpenID Foundation. Puedes usar el flujo CIBA en flujos de trabajo asíncronos en los que el dispositivo que inicia la solicitud de CIBA (el dispositivo de consumo) es distinto del dispositivo que usa el usuario para autenticarse (el dispositivo de autenticación).

Para configurar CIBA en Auth0, debes:

* [Configurar el tipo de concesión de CIBA para tu aplicación](#configure-ciba-grant-type-for-your-application)
* [Habilitar uno o más canales de notificación para tu aplicación](#enable-notification-channels-for-your-application)
* [Configurar el canal de notificación para CIBA](#configure-notification-channel)

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de configurar CIBA para tu aplicación, asegúrate de establecer un [método de autenticación](/docs/es/secure/application-credentials#application-authentication-methods) para tu aplicación. Puedes usar cualquier método de autenticación con el flujo CIBA, incluida la autenticación mTLS, clave privada <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, también del token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> y la autenticación con <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, también del token de acceso) que se usa para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">Client Secret</Tooltip>.

Para establecer el método de autenticación para tu aplicación, consulta [Credential Settings](/docs/es/get-started/applications/credentials).

<div id="configure-ciba-grant-type-for-your-application">
  ## Configura el tipo de concesión CIBA para tu aplicación
</div>

Puedes configurar el tipo de concesión CIBA para tu aplicación con el [Auth0 Dashboard](https://manage.auth0.com/) o la [Management API](https://auth0.com/docs/api/management/v2).

Hay algunas restricciones sobre los tipos de clientes que pueden usar el tipo de concesión CIBA. Solo puedes usar el tipo de concesión CIBA si:

* El cliente es un cliente de primera parte; es decir, la propiedad `is_first_party` es `true`.
* El cliente es confidencial y cuenta con un mecanismo de autenticación; es decir, la propiedad `token_endpoint_auth_method` no debe establecerse en `none`.
* El cliente debe ser compatible con OIDC; es decir, `oidc_conformant` debe ser `true`. Este es el valor predeterminado para todos los clientes nuevos.

Una vez que habilites el tipo de concesión CIBA, se mostrarán los ajustes de configuración de los [canales de notificación](#enable-notification-channels-for-your-application) disponibles para tu aplicación.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para configurar CIBA para tu aplicación con el Auth0 Dashboard:

    1. Ve a **Applications &gt; Applications** en el Auth0 Dashboard.
    2. Crea una aplicación y luego habilita **Client Initiated Backchannel Authentication (CIBA)** en la pestaña **Grant Types**:

    <Frame>![](/docs/images/cdy7uua7fh8z/54TD5VRQVAOWxBdnFEpvTT/223866f22f6c3ddfa87e48221d69ba58/CIBA_grant_-_English.png)</Frame>

    3. Haz clic en **Save Changes**.
  </Tab>

  <Tab title="Management API">
    Para configurar CIBA para tu aplicación con la Management API, usa el endpoint [actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para agregar el tipo de concesión `urn:openid:params:grant-type:ciba` a la lista de tipos de concesión del objeto cliente:

    ```bash lines
    curl --location --request PATCH 'https://{YOUR_DOMAIN}.auth0.com/api/v2/clients/{YOUR_CLIENT_ID}' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}' \
    --data '{
        "grant_types": [
            "authorization_code",
            "refresh_token",
            "urn:openid:params:grant-type:ciba"
        ]
    }'
    ```

    También puedes usar nuestra biblioteca SDK de la Management API para Go. Para obtener más información, consulta [SDKs](/docs/es/libraries):

    ```go lines
    myClient := &Client{
        Name:        auth0.Stringf("CIBA-enabled-client"),
        Description: auth0.String("This is a CIBA enabled client."),
        GrantTypes:  &[]string{"urn:openid:params:grant-type:ciba"},
      }

      err := api.Client.Create(context.Background(), myClient)
    ```
  </Tab>
</Tabs>

<div id="enable-notification-channels-for-your-application">
  ## Habilita los canales de notificación para tu aplicación
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar las funcionalidades de Client-Initiated Backchannel Authentication (CIBA), debes tener un Enterprise Plan o un add-on adecuado. Consulta [Auth0 Pricing](https://auth0.com/pricing/) para obtener más información.
</Callout>

Una vez que hayas habilitado el tipo de concesión de CIBA para tu aplicación, puedes configurar qué canales de notificación estarán habilitados para el flujo de CIBA. Si has habilitado varios canales de notificación, CIBA usa el valor del parámetro `requested_expiry` para determinar qué canal de notificación usar. `requested_expiry` determina la duración máxima, en segundos, durante la cual la sesión de CIBA debe seguir siendo válida:

* [Notificaciones push en dispositivo móvil](#configure-mobile-push-notifications): Si estableces `requested_expiry` en un valor de 300 segundos o menos, CIBA usa el canal de notificación push en dispositivo móvil si está habilitado.
* [Notificaciones por correo electrónico](#configure-email-notifications): Si estableces `requested_expiry` en un valor entre 301 y 259200 segundos (72 horas), CIBA usa el canal de notificación por correo electrónico si está habilitado.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para configurar el canal de notificación de tu aplicación en el Auth0 Dashboard:

    * Ve a **Applications &gt; Applications**.
    * En la configuración de tu aplicación, ve a la sección **Client Initiated Backchannel Authentication (CIBA)** y selecciona los canales de notificación que quieres habilitar.

    <Frame>![](/docs/images/ciba/enable_notification_channel_for_app.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Realiza una solicitud `PATCH` al endpoint `/api/v2/clients` de tu aplicación cliente y configura los canales de notificación para el flujo de CIBA.

    El siguiente ejemplo de código habilita el canal de notificación `guardian-push` para el flujo de CIBA:

    ```bash lines
    curl -L --request PATCH 'https://{YOUR_DOMAIN}/api/v2/clients/{YOUR_CLIENT_ID}' \
    -H 'Content-Type: application/json' \
    -H 'Accept: application/json' \
    -H "Authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}" \
    -d '{
        "async_approval_notification_channels":["guardian-push"]
    }'
    ```
  </Tab>
</Tabs>

<div id="configure-notification-channel">
  ## Configurar el canal de notificación
</div>

Una vez que hayas habilitado los canales de notificación para tu aplicación, configura el canal de notificación para el flujo de CIBA:

* [Configurar las notificaciones push en dispositivos móviles](#configure-mobile-push-notifications)
* [Configurar las notificaciones por correo electrónico](#configure-email-notifications)

<div id="configure-mobile-push-notifications">
  ### Configurar notificaciones push en dispositivos móviles
</div>

Para enviar notificaciones push en dispositivos móviles con CIBA, puedes usar:

* La aplicación Auth0 Guardian
* Una aplicación personalizada integrada con el Auth0 Guardian SDK

Puedes seleccionar qué aplicación usar al [habilitar las notificaciones push de Auth0 Guardian](/docs/es/get-started/applications/configure-client-initiated-backchannel-authentication#enable-auth0-guardian-push-notifications). Si quieres usar una aplicación personalizada, debes integrarla con el [Auth0 Guardian SDK](/docs/es/secure/multi-factor-authentication/auth0-guardian#guardian-sdks). Esto permite que el usuario autorizador apruebe los desafíos de notificación push iniciados por el flujo CIBA desde tu aplicación personalizada.

Para configurar las notificaciones push en dispositivos móviles, asegúrate de:

* [Habilitar las notificaciones push de Auth0 Guardian](#enable-auth0-guardian-push-notifications)
* [Inscribir al usuario autorizador en MFA mediante notificaciones push](#enroll-the-authorizing-user-in-mfa-using-push-notifications)

<div id="enable-auth0-guardian-push-notifications">
  #### Habilita las notificaciones push de Auth0 Guardian
</div>

Usa el Auth0 Dashboard para habilitar el factor [Auth0 Guardian Push Notification](/docs/es/secure/multi-factor-authentication/auth0-guardian#enroll-in-push-notifications) en tu tenant.

En el Auth0 Dashboard:

* Selecciona **Security &gt; Multi-factor Auth.**

* Habilita **Push Notification using Auth0 Guardian**. Esto puede requerir algunos ajustes de configuración de <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor además de nombre de usuario y contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=MFA">MFA</Tooltip>. Para obtener más información, consulta [Configure Push Notifications for MFA](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa).

<Frame>![](/docs/images/cdy7uua7fh8z/7JKyznvsjTupJpTU7NwdSA/54b04e5b734226e1ee73c4165048b4f6/image2.png)</Frame>

* En **Push Notification App**, selecciona la aplicación que prefieras.

<Frame>![](/docs/images/ciba/push_notification_app.png)</Frame>

* Haz clic en **Save**.

<div id="enroll-the-authorizing-user-in-mfa-using-push-notifications">
  #### Inscriba al usuario autorizador en MFA mediante notificaciones push
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si el usuario no está registrado para usar notificaciones push de MFA, Auth0 recurre a las notificaciones por correo electrónico, si están configuradas, en lugar de rechazar la solicitud CIBA.
</Callout>

Tanto para la aplicación Auth0 Guardian como para una aplicación personalizada, debe [inscribir al usuario autorizador en MFA mediante notificaciones push](/docs/es/secure/multi-factor-authentication/auth0-guardian#enroll-in-push-notifications).

Para verificar si el usuario está registrado en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vaya a **User Management &gt; Users** y haga clic en el usuario:

<Frame>![](/docs/images/cdy7uua7fh8z/21qtk4F1cOQHMpbXxXYu75/51ba9c5fd1457ffee3f73893acf3c97f/Screenshot_2025-01-13_at_4.13.44_PM.png)</Frame>

Si ha configurado <Tooltip tip="Multi-factor authentication (MFA): proceso de autenticación de usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-factor+Authentication">autenticación multifactor</Tooltip> como obligatoria en todo momento para su tenant, se pedirá a los usuarios que se registren en MFA en su próximo inicio de sesión. También puede usar [Actions](https://auth0.com/blog/using-actions-to-customize-your-mfa-factors/) para solicitar el registro en MFA.

<div id="configure-email-notifications">
  ### Configura las notificaciones por correo electrónico
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar notificaciones por correo electrónico con CIBA, necesitas el add-on Auth0 for AI Agents con un plan de pago. Para obtener más información, consulta [Auth0 Pricing](https://auth0.com/pricing).
</Callout>

Puedes enviar notificaciones por correo electrónico en el flujo de CIBA. Para configurar las notificaciones por correo electrónico con CIBA, asegúrate de lo siguiente:

* [Configura tu proveedor de correo electrónico](#configure-your-email-provider)
* [Configura tu plantilla de correo electrónico](#configure-your-email-template) para usar la plantilla **Asynchronous Approval**
* [Asegúrate de que el usuario autorizador tenga una dirección de correo electrónico verificada](#ensure-the-authorizing-user-has-a-verified-email-address)

<div id="configure-your-email-provider">
  #### Configura tu proveedor de correo electrónico
</div>

Aunque puedes usar el proveedor de correo electrónico integrado de Auth0 para probar la entrega de correos electrónicos en tus entornos de desarrollo y prueba, debes configurar tu propio proveedor de correo electrónico si quieres usar notificaciones por correo electrónico con CIBA en un entorno de producción.

Para saber cómo configurar tu propio proveedor de correo electrónico, lee [Customize emails](/docs/es/customize/email).

<div id="configure-your-email-template">
  #### Configura tu plantilla de correo electrónico
</div>

Para configurar la plantilla de correo electrónico **Asynchronous Approval**:

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Ve a **Branding &gt; plantilla de correo electrónico**.
    2. En **Plantilla**, selecciona **Asynchronous Approval** en el menú desplegable.
    3. Completa el resto de la configuración de la plantilla siguiendo las instrucciones de [Configurar los campos de la plantilla](/docs/es/customize/email/email-templates#configure-template-fields).
  </Tab>

  <Tab title="Management API">
    Realiza una solicitud `PATCH` al endpoint `/email-templates/async_approval` y sigue las instrucciones de [Configurar los campos de la plantilla](/docs/es/customize/email/email-templates#configure-template-fields) para completar el resto de la configuración de la plantilla. Para obtener más información, consulta la [documentación de la API para actualizar una plantilla de correo electrónico](https://auth0.com/docs/api/management/v2/email-templates/patch-email-templates-by-template-name).

    ```bash lines
    curl -L "https://{YOUR_DOMAIN}.auth0.com/api/v2/email-templates" \
    -H 'Content-Type: application/json' \
    -H 'Accept: application/json' \
    -H "Authorization: Bearer {YOUR_MANAGEMENT_API_ACCESS_TOKEN}" \
    -d '{"template":"async_approval","body":"{{application.name}} says click on {{url}}, binding message: {{binding_message}}","subject":"Action Required","syntax":"liquid","enabled":true,"from":""}'
    ```
  </Tab>
</Tabs>

<div id="ensure-the-authorizing-user-has-a-verified-email-address">
  #### Asegúrese de que el usuario autorizador tenga una dirección de correo electrónico verificada
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si el usuario no tiene una dirección de correo electrónico verificada, Auth0 rechaza la solicitud de CIBA por correo electrónico.
</Callout>

Para enviar una notificación por correo electrónico con CIBA, el usuario autorizador debe tener asociada a su cuenta de usuario una dirección de correo electrónico verificada.

Para verificar en el Auth0 Dashboard que el usuario tiene una dirección de correo electrónico verificada:

1. Vaya a **User Management &gt; Users** y haga clic en el usuario.
2. En **Email**, debe aparecer una dirección de correo electrónico verificada.
