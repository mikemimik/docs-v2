title: Comprobar si la aplicación es confidencial o pública
---

Puede comprobar si una aplicación está registrada en Auth0 como una aplicación confidencial o pública. Para obtener más información, consulte [Aplicaciones confidenciales y públicas](/docs/es/get-started/applications/confidential-and-public-applications).

Auth0 determina si una aplicación es confidencial o pública en función de la configuración **Authentication Method**, que define cómo una aplicación se autentica en el endpoint [Get Token](https://auth0.com/docs/api/authentication/authorization-code-flow/get-token) de Auth0 Authentication API.

1. En el Auth0 Dashboard, vaya a [Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione el nombre de la aplicación que quiera ver.
2. Si la vista **Credentials** no está disponible, la aplicación es pública.
3. Si la vista **Credentials** está disponible, selecciónela y localice el campo **Authentication Method**.

   <Frame>![Configurar la autenticación con clave privada JWT: instrucciones del Auth0 Dashboard](/docs/images/cdy7uua7fh8z/33kfi48tkbMIOQJ8PBxj76/5b79ffda11ad58b27128068057e6e05c/Default_App_-_Creds_-_English.png)</Frame>

Use el método correspondiente:

* **None**: Aplicación pública sin <Tooltip tip="secreto de cliente: Secret utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>.
* **Client Secret Post**: Aplicación que usa parámetros en el cuerpo de la solicitud POST para enviar un secreto de cliente.
* **Client Secret Basic**: Aplicación que usa el esquema de autenticación HTTP BASIC para enviar un secreto de cliente.
* **Private Key <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y con frecuencia formato de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>**: Aplicación que usa autenticación asimétrica.

Estos valores se corresponden con aplicaciones confidenciales y públicas de la siguiente manera:

| Tipo de aplicación | Ejemplo                             | Método de autenticación del endpoint de token             |
| ------------------ | ----------------------------------- | --------------------------------------------------------- |
| **Pública**        | De página única o nativa            | **None**                                                  |
| **Confidencial**   | Web tradicional o máquina a máquina | **Basic**, **Post**, **Private Key JWT**, **Unspecified** |

Las aplicaciones públicas no pueden mantener la confidencialidad de las credenciales necesarias para métodos de autenticación del <Tooltip tip="Endpoint de token: endpoint en el servidor de autorización que se utiliza para solicitar tokens de forma programática." cta="Ver glosario" href="/docs/es/glossary?term=Token+endpoint">endpoint de token</Tooltip> como **Post** y **Basic**.
