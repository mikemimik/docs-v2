description: Describe cómo crear una integración de inicio de sesión único (SSO) para una aplicación. Solo para tenants heredados.
title: Habilitar integraciones de inicio de sesión único para aplicaciones
---

De forma predeterminada, el <Tooltip tip="Inicio de sesión único (SSO): Servicio que, una vez que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=single+sign-on">inicio de sesión único</Tooltip> (SSO) sin fricciones está habilitado para todos los nuevos tenants de Auth0; sin embargo, los **tenants heredados** pueden [decidir si habilitan esta funcionalidad a nivel del tenant](/docs/es/get-started/tenant-settings/enable-sso-for-legacy-tenants). Si no has habilitado el SSO a nivel del tenant, puedes habilitarlo por aplicación.

Puedes habilitar el SSO para tu aplicación mediante el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

<div id="prerequisite">
  ## Requisito previo
</div>

Antes de habilitar SSO para una aplicación, primero debes crear y configurar una conexión para cada <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> que quieras usar. Para los proveedores de identidad social, asegúrate de que la conexión no use claves de desarrollador si utilizas la [experiencia de Universal Login](/docs/es/authenticate/login/auth0-universal-login).

<div id="enable-sso">
  ## Habilitar SSO
</div>

1. Ve a [Auth0 Dashboard &gt; Applications &gt; SSO Integrations](https://manage.auth0.com/#/externalapps) y selecciona el nombre de la integración que quieres ver.
2. Desplázate hasta la parte inferior de la página **Settings**, busca **Use Auth0 instead of the IdP to do Single Sign-on**, activa el toggle y selecciona **Save Changes**.
