Las aplicaciones de terceros son aplicaciones propiedad de una organización externa —un partner, un desarrollador independiente o un agente de IA— y operadas por ella, que autentican a tus usuarios y acceden a tus APIs.

A diferencia de las [aplicaciones propias](/docs/es/get-started/applications/first-party-and-third-party-applications#first-party-applications), que controlas directamente, las aplicaciones de terceros operan de forma independiente: les concedes acceso a tus recursos, pero no puedes controlar lo que hacen con ese acceso. Para obtener más información sobre las diferencias entre aplicaciones propias y de terceros, consulta [Aplicaciones propias y de terceros](/docs/es/get-started/applications/first-party-and-third-party-applications).

Las aplicaciones de terceros tienen las siguientes características únicas:

* **Controles de seguridad reforzados**: Auth0 aplica [controles de seguridad reforzados](/docs/es/get-started/applications/third-party-applications/security-controls) a las aplicaciones de terceros, lo que garantiza que las aplicaciones externas solo puedan acceder a los recursos que autorices explícitamente. Las funcionalidades destinadas a casos de uso de aplicaciones propias no están disponibles.
* **Consentimiento del usuario**: Auth0 siempre requiere el consentimiento del usuario cuando una aplicación de terceros solicita acceso a las APIs. El consentimiento no se puede omitir.
* **Conexiones**: Las aplicaciones de terceros solo pueden autenticar a los usuarios mediante conexiones a nivel de dominio. Para obtener más información, consulta [Promover conexiones al nivel de dominio](/docs/es/authenticate/identity-providers/promote-connections-to-domain-level).

<div id="use-cases">
  ## Casos de uso
</div>

Entre los casos de uso más comunes de las aplicaciones de terceros se incluyen:

* **Integraciones con partners**: Los partners externos crean aplicaciones que consumen tus API en nombre del usuario. Por ejemplo, un proveedor de CRM se integra con tu plataforma para que los clientes en común puedan sincronizar datos entre ambos productos.
* **Agentes de IA y clientes MCP**: Herramientas impulsadas por IA, como Claude Code, VS Code con Copilot o servidores MCP personalizados, se conectan a tus API para realizar acciones en nombre del usuario. Para obtener más información, consulta [Auth for MCP](https://auth0.com/ai/docs/mcp/intro/overview).
* **Ecosistemas de desarrolladores**: Expones API para que desarrolladores externos creen aplicaciones en tu plataforma, ya sea a través de un portal para desarrolladores, un marketplace o [registro dinámico de clientes](/docs/es/get-started/applications/dynamic-client-registration).
* **Cross App Access (XAA)**: Una aplicación para la fuerza laboral en el tenant de otra organización accede a tus API mediante una relación de confianza, en la que la aplicación solicitante se modela como una aplicación de terceros en tu tenant. Para obtener más información, consulta [Cross App Access](/docs/es/secure/call-apis-on-users-behalf/xaa).

<div id="supported-client-types">
  ## Tipos de cliente compatibles
</div>

Las aplicaciones de terceros admiten clientes tanto confidenciales como públicos:

| **Tipo de cliente** | **Tipo de aplicación**         | **Caso de uso**                                |
| ------------------- | ------------------------------ | ---------------------------------------------- |
| Confidencial        | Aplicación web tradicional     | Integraciones de partner del lado del servidor |
| Público             | Aplicación web de página única | Widgets de partner en el navegador             |
| Público             | Nativa                         | Aplicaciones móviles de partner                |

<div id="supported-grant-types">
  ## Tipos de concesión compatibles
</div>

Las aplicaciones de terceros admiten:

* `authorization_code` con PKCE obligatorio
* `refresh_token`
* `client_credentials` (solo para clientes confidenciales)

<div id="get-started">
  ## Primeros pasos
</div>

Para configurar una aplicación de terceros en Auth0, sigue estos pasos:

<div id="step-1-create-the-application">
  ### Paso 1: Crear la aplicación
</div>

[Crea una aplicación de terceros](/docs/es/get-started/applications/third-party-applications/configure-third-party-applications#create-a-third-party-application) con Auth0 Dashboard o la Management API.

<div id="step-2-configure-api-access">
  ### Paso 2: Configurar el acceso a la API
</div>

Las aplicaciones de terceros siempre requieren una autorización explícita para acceder a tus API, incluso cuando la política de acceso de la API está configurada como **Allow All.** Las políticas de acceso a la API se configuran mediante [autorizaciones de cliente](/docs/es/get-started/applications/application-access-to-apis-client-grants).

Puedes configurar permisos predeterminados que se apliquen automáticamente a todas las aplicaciones de terceros. Esto resulta útil si administras muchas aplicaciones de terceros o usas el [registro dinámico de clientes](/docs/es/get-started/applications/dynamic-client-registration). Para obtener más información, consulta [Permisos predeterminados para aplicaciones de terceros](/docs/es/get-started/applications/application-access-to-apis-client-grants#default-permissions-for-third-party-applications).

También puedes definir permisos específicos para cada aplicación mediante [autorizaciones de cliente](/docs/es/get-started/applications/application-access-to-apis-client-grants). Los permisos por aplicación tienen prioridad sobre los predeterminados. Para obtener más información, consulta [Crear una autorización de cliente](/docs/es/get-started/applications/application-access-to-apis-client-grants#create-client-grant).

<div id="step-3-configure-connections">
  ### Paso 3: Configurar conexiones
</div>

Las aplicaciones de terceros solo pueden autenticar a los usuarios mediante conexiones promovidas al nivel de dominio. Las conexiones a nivel de dominio están disponibles para todas las aplicaciones de terceros del tenant.

Para promover una conexión al nivel de dominio, lee [Promover conexiones al nivel de dominio](/docs/es/authenticate/identity-providers/promote-connections-to-domain-level).

<div id="step-4-users-provide-consent">
  ### Paso 4: Los usuarios dan su consentimiento
</div>

Cuando un usuario se autentica a través de una aplicación de terceros, Auth0 muestra un cuadro de diálogo de consentimiento para pedirle que apruebe los permisos solicitados. El consentimiento siempre es obligatorio para las aplicaciones de terceros y no se puede omitir.

Para obtener más información, consulta [consentimiento del usuario y aplicaciones de terceros](/docs/es/get-started/applications/third-party-applications/user-consent-and-third-party-applications).

<div id="dynamic-client-registration">
  ## Registro dinámico de clientes
</div>

El [registro dinámico de clientes](/docs/es/get-started/applications/dynamic-client-registration) crea aplicaciones de terceros con controles de seguridad reforzados de forma predeterminada. Antes de habilitar DCR para aplicaciones de terceros, debes [configurar los permisos predeterminados de la API](/docs/es/get-started/applications/dynamic-client-registration#configure-api-access-for-dcr-clients) para que los clientes registrados dinámicamente puedan acceder a tus APIs.
