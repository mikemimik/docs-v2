<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para usar las funcionalidades de Identidad altamente regulada, debe tener un Enterprise Plan con el add-on Highly Regulated Identity. Consulte [Auth0 Pricing](https://auth0.com/pricing/) para más detalles.
</Callout>

La implementación de Auth0 Push Authorization Request (PAR) se basa en la especificación [OAuth RFC9126: Push Authorization Request](https://www.rfc-editor.org/rfc/rfc9126.html). Para obtener más información, consulte [flujo de código de autorización con solicitudes de autorización push](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/authorization-code-flow-with-par).

De forma predeterminada, PAR no está habilitado en su tenant. Puede habilitarlo en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en la Configuración del tenant. Para obtener más información, lea [Habilitar PAR para un tenant](#enable-par-for-a-tenant).

Después de habilitar PAR para su tenant, puede enviar solicitudes de autorización a los endpoints `/oauth/par` y `/authorize`. Sin embargo, para proteger por completo su <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) especificada en el marco de trabajo OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip>, establezca PAR como obligatorio para una aplicación a través de la [Management API](https://auth0.com/docs/api/management/v2) o en **Application Settings** del Auth0 Dashboard.

<div id="enable-par-for-a-tenant">
  ## Habilitar PAR para un tenant
</div>

Para habilitar PAR para un tenant, utiliza el [Auth0 Dashboard](https://manage.auth0.com/).

1. Ve a Auth0 Dashboard &gt; Settings &gt; Advanced.

2. Desplázate hasta **Settings** y activa **Allow solicitudes de autorización push (PAR)**.

<div id="require-par-for-an-application">
  ## Requerir PAR para una aplicación
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      El tenant debe tener habilitada la opción **Allow solicitudes de autorización push (PAR)** a nivel del tenant antes de habilitar PAR a nivel de la aplicación.
    </Callout>

    1. Vaya a Auth0 Dashboard &gt; Applications.
    2. Seleccione la aplicación.
    3. Seleccione la pestaña **Application Settings**.
    4. En la sección **Authorization Requests**, active el interruptor **Require solicitudes de autorización push (PAR)**.
  </Tab>

  <Tab title="Management API">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      El tenant debe tener habilitada la opción **Allow solicitudes de autorización push (PAR)** a nivel del tenant antes de habilitar PAR a nivel de la aplicación.
    </Callout>

    Use el siguiente ejemplo de código para configurar PAR para su aplicación mediante la Management API:

    ```bash lines
    curl -X PATCH --location 'https://TENANT.auth0.com/api/v2/clients/CLIENT_ID' \
      --header 'Authorization: Bearer MANAGEMENT_ACCESS_TOKEN' \
      --header 'Content-Type: application/json' \
      --data-raw '{
       "require_pushed_authorization_requests": true
    }'
    ```
  </Tab>
</Tabs>
