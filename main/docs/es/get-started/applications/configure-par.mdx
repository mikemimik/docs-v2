  Para usar las funciones de Highly Regulated Identity, debes tener un plan Enterprise con el complemento Highly Regulated Identity. Consulta [Auth0 Pricing](https://auth0.com/pricing/) para obtener más información.
</Callout>

La implementación de Push Authorization Request (PAR) de Auth0 se basa en la especificación [OAuth RFC9126: Push Authorization Request](https://www.rfc-editor.org/rfc/rfc9126.html). Para obtener más información, consulta [Flujo de código de autorización con solicitudes de autorización push](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/authorization-code-flow-with-par).

De forma predeterminada, PAR no está habilitado para tu tenant. Puedes habilitarlo en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en la configuración de tu tenant. Para obtener más información, consulta [Habilitar PAR para un tenant](#enable-par-for-a-tenant).

Después de habilitar PAR para tu tenant, puedes enviar solicitudes de autorización a los endpoints `/oauth/par` y `/authorize`. Sin embargo, para proteger por completo tu <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) especificada en el marco de OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip>, configura PAR como obligatorio para una aplicación mediante la [Management API](https://auth0.com/docs/api/management/v2) o en **Configuración de la aplicación** del Auth0 Dashboard.

<div id="enable-par-for-a-tenant">
  ## Habilitar PAR para un tenant
</div>

Para habilitar PAR para un tenant, usa el [Auth0 Dashboard](https://manage.auth0.com/).

1. Ve a Auth0 Dashboard &gt; Settings &gt; Advanced.

2. Desplázate hasta **Settings** y activa **Allow Pushed Authorization Requests (PAR)**.

<div id="require-par-for-an-application">
  ## Requerir PAR para una aplicación
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Tu tenant debe tener **Allow Pushed Authorization Requests (PAR)** habilitado a nivel de tenant antes de habilitar PAR a nivel de aplicación.
    </Callout>

    1. Ve a Auth0 Dashboard &gt; Applications.
    2. Selecciona la aplicación.
    3. Selecciona la pestaña **Application Settings**.
    4. En la sección **Authorization Requests**, activa el interruptor **Require Pushed Authorization Requests (PAR)**.
  </Tab>

  <Tab title="Management API">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Tu tenant debe tener **Allow Pushed Authorization Requests (PAR)** habilitado a nivel de tenant antes de habilitar PAR a nivel de aplicación.
    </Callout>

    Usa el siguiente ejemplo de código para configurar PAR en tu aplicación mediante la Management API:

    ```bash lines
    curl -X PATCH --location 'https://TENANT.auth0.com/api/v2/clients/CLIENT_ID' \
      --header 'Authorization: Bearer MANAGEMENT_ACCESS_TOKEN' \
      --header 'Content-Type: application/json' \
      --data-raw '{
       "require_pushed_authorization_requests": true
    }'
    ```
  </Tab>
</Tabs>
