  Para usar las funcionalidades de Identidad altamente regulada, debe tener un Enterprise Plan con el add-on Highly Regulated Identity. Consulte [Auth0 Pricing](https://auth0.com/pricing/) para obtener más información.
</Callout>

La implementación de Auth0 Push Authorization Request (PAR) se basa en la especificación [OAuth RFC9126: Push Authorization Request](https://www.rfc-editor.org/rfc/rfc9126.html). Para obtener más información, consulte [Authorization Code Flow with Pushed Authorization Requests](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/authorization-code-flow-with-par).

De forma predeterminada, PAR no está habilitado en su tenant. Puede habilitarlo en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en la configuración de su tenant. Para obtener más información, lea [Habilitar PAR para un tenant](#enable-par-for-a-tenant).

Después de habilitar PAR para su tenant, puede enviar solicitudes de autorización a los endpoints `/oauth/par` y `/authorize`. Sin embargo, para proteger completamente su <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) especificada en el framework OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip>, configure PAR como obligatorio para una aplicación mediante la [Management API](https://auth0.com/docs/api/management/v2) o **configuración de la aplicación** en el Auth0 Dashboard.

<div id="enable-par-for-a-tenant">
  ## Habilitar PAR para un tenant
</div>

Para habilitar PAR para un tenant, usa el [Auth0 Dashboard](https://manage.auth0.com/).

1. Ve a Auth0 Dashboard &gt; Settings &gt; Advanced.

2. Desplázate hacia abajo hasta **Settings** y activa **Allow Pushed Authorization Requests (PAR)**.

<div id="require-par-for-an-application">
  ## Requerir PAR para una aplicación
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Tu tenant debe tener **Permitir solicitudes de autorización push (PAR)** activada a nivel del tenant antes de habilitar PAR a nivel de la aplicación.
    </Callout>

    1. Ve a Auth0 Dashboard &gt; Aplicaciones.
    2. Selecciona la aplicación.
    3. Selecciona la pestaña **Configuración de la aplicación**.
    4. En la sección **Solicitudes de autorización**, habilita el interruptor **Requerir solicitudes de autorización push (PAR)**.
  </Tab>

  <Tab title="Management API">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Tu tenant debe tener **Permitir solicitudes de autorización push (PAR)** activada a nivel del tenant antes de habilitar PAR a nivel de la aplicación.
    </Callout>

    Usa el siguiente ejemplo de código para configurar PAR para tu aplicación con la Management API:

    ```bash lines
    curl -X PATCH --location 'https://TENANT.auth0.com/api/v2/clients/CLIENT_ID' \
      --header 'Authorization: Bearer MANAGEMENT_ACCESS_TOKEN' \
      --header 'Content-Type: application/json' \
      --data-raw '{
       "require_pushed_authorization_requests": true
    }'
    ```
  </Tab>
</Tabs>
