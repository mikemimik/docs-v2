<Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>-Las solicitudes de autorización protegidas con JWT (JAR) permiten empaquetar los parámetros de la solicitud de autorización de OAuth2 en un único parámetro de solicitud JWT, que luego se firma para proteger su integridad.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de configurar tu aplicación para que use JAR, debes [generar un par de claves RSA](/docs/es/secure/application-credentials/generate-rsa-key-pair).

<Warning>
  Debes generar un par de claves distinto para cada uso de credenciales. Por ejemplo, no reutilices el mismo par de claves para JAR y para la autenticación con clave privada JWT.
</Warning>

<div id="configure-jar-for-an-application">
  ## Configurar JAR para una aplicación
</div>

Puedes configurar JAR para una aplicación desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Usa el Auth0 Dashboard para configurar tu aplicación y que use JAR con claves RSA generadas previamente.

    1. Ve a [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/#/applications).
    2. Selecciona la aplicación que quieres usar con JAR.
    3. Selecciona la pestaña **Application Settings**.
    4. En la sección **Authorization Requests**, habilita **Require JWT-Secured Authorization Requests**.
    5. Si no hay ninguna credencial asignada y hay credenciales disponibles, se te pedirá que asignes una credencial existente.

       <Frame>![Dashboard > Aplicación > Configuración > Asignar credenciales existentes](/docs/images/cdy7uua7fh8z/HQHhFWTtdfNa5TnZ1dwx6/e47068cc9e85c538f80476162f4314a3/Existing_Creds_-_English.png)</Frame>
    6. También tendrás la opción de asignar una credencial nueva.

       <Frame>![Auth0 Dashboard > Applications > Configuración > Asignar credenciales nuevas](/docs/images/cdy7uua7fh8z/7JfsCBwytWO6Q7hUvdtSwJ/b85fd39fea7330a31496f51347767ae7/New_Creds_-_EN.png)</Frame>
    7. Agrega y asigna una credencial nueva cargando un par de claves RSA generado previamente. Cuando se te solicite, introduce lo siguiente:

       * **Name**: un nombre para identificar la credencial
       * **Public Key**: la clave pública del certificado X.509 en formato PEM
       * **Algorithm**: selecciona el algoritmo de firma de JAR
       * **Expiration Date**: establece la fecha de vencimiento de la credencial
  </Tab>

  <Tab title="Management API">
    Usa la [Management API](https://auth0.com/docs/api/management/v2) para configurar JAR en tu aplicación mediante la propiedad de configuración del cliente `signed_request_object`. Esta propiedad de objeto contiene los siguientes campos:

    * `required`: obliga a que todas las solicitudes de autorización a `/authorize` y `/oauth/par` usen JAR. Para obtener más información, consulta [flujo de código de autorización con solicitud de autorización protegida con JWT](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/authorization-code-flow-with-jar).
    * `credentials`: un arreglo de ID de credenciales que se usa para verificar firmas.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      El parámetro `credentials` se comporta de forma similar al parámetro de clave privada JWT `client_authentication_methods.private_key_jwt.credentials`, que admite la creación de credenciales cuando creas una aplicación nueva. Para obtener más información, consulta [Configure Private Key JWT](/docs/es/get-started/applications/configure-private-key-jwt).
    </Callout>

    Puedes configurar JAR para una aplicación nueva o para una aplicación existente mediante la Management API.

    #### Configurar JAR para una aplicación nueva

    Cuando crees una aplicación nueva, configura JAR enviando una solicitud POST con `signed_request_object`. En esa solicitud POST, también puedes registrar la credencial de cliente correspondiente (es decir, la clave PEM):

    ```json lines
    POST https://{yourTenant}.auth0.com/api/v2/clients
    Authorization: Bearer <YOUR_ACCESS_TOKEN>
    Content-Type: application/json
    {
      "name": "My App using JAR",
      "signed_request_object": {
          "required": true,
    "credentials": [{
            "name": "My credential for JAR",
            "credential_type": "public_key",
            "pem": "[YOUR PEM FILE CONTENT]",
            "alg": "RS256"
    }]
      },
      "jwt_configuration": {
        "alg": "RS256"
      }
    }
    ```
