title: Configuración de APIs
---

Usa la pestaña **configuración** en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis), para configurar las APIs registradas que pueden consumir tus aplicaciones autorizadas. Para configurar los ajustes de una API, haz clic en **...** junto a una API de la lista y selecciona **configuración**, o haz clic en el nombre de la API. Para aprender a crear y registrar una API, lee [Registrar APIs](/docs/es/get-started/auth0-overview/set-up-apis).

<Frame>![Lista de APIs en Dashboard > Applications > APIs](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>

<div id="settings">
  ## Configuración
</div>

Usa la configuración de esta pestaña para definir la expiración del token, el control de acceso basado en roles (RBAC) y otras opciones de acceso. Haz clic en **Guardar** al final de la pestaña para guardar los cambios.

<div id="general-settings">
  ### Configuración general
</div>

Estos campos se definieron cuando registraste inicialmente la API, excepto en el caso de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0. Solo puedes modificar el **Nombre**.

<Frame>![Dashboard Applications APIs Settings Tab Configuración general](/docs/images/cdy7uua7fh8z/QCaXd3FSeP6SYqLfEJWBr/fbb1bb5b2e12fb099073c33fb39cf44a/dashboard-applications-apis-settings-general-settings.png)</Frame>

* **Id**: Una cadena alfanumérica única generada por Auth0. Esta información es de solo lectura y solo la necesitarás si vas a trabajar directamente con los [endpoints de API Resource Servers de la Management API de Auth0](https://auth0.com/docs/api/management/v2#!/Resource_Servers/get_resource_servers_by_id).
* **Nombre**: Un nombre descriptivo para la API. No afecta a ninguna funcionalidad. No se permiten los siguientes caracteres: `< >`.
* **Identificador**: Un identificador único para tu API. Este valor se establece al crear la API y no puede modificarse después. Recomendamos usar una URL, pero no tiene que ser una URL disponible públicamente; Auth0 no hará ninguna llamada a tu API.

<div id="token-settings">
  ### Configuración del token
</div>

<Frame>![Dashboard - API - Configuración del token - Expiración](/docs/images/cdy7uua7fh8z/2FFfqqmntXL9CnGrSxea9M/810e5542553168ffcd6081a10bc2db2f/image2.png)</Frame>

* **Duración máxima del <Tooltip tip="Access Token: Credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> (segundos)**: El tiempo (en segundos) que transcurre antes de que expire un token de acceso. El valor predeterminado es 86400 segundos (24 horas). El valor máximo que puedes establecer es 2592000 segundos (30 días).
* **Duración del token de acceso para flujo implícito / híbrido (segundos)**: El tiempo (en segundos) que transcurre antes de que expire un token de acceso emitido mediante un flujo implícito o híbrido. El valor predeterminado es 86400 segundos (24 horas). El valor no puede superar la duración máxima del token de acceso.
* **Perfil de <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y a menudo formato de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT)**: El perfil determina el formato de los tokens de acceso emitidos para la API. Los valores disponibles son `Auth0` y `RFC 9068`. Para obtener más información, consulta [Access Token Profiles](/docs/es/secure/tokens/access-tokens/access-token-profiles).
* **JSON Web Token (JWT) <Tooltip tip="Signing Algorithm: Algoritmo utilizado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=Signing+Algorithm">Algoritmo de firma</Tooltip>**: El algoritmo con el que se firman los tokens. La firma se utiliza para verificar que el remitente del JWT es quien dice ser y para garantizar que el mensaje no se haya modificado durante el proceso. Los valores disponibles son [`HS256`](https://en.wikipedia.org/wiki/Symmetric-key_algorithm) y [`RS256`](https://en.wikipedia.org/wiki/Public-key_cryptography). Si seleccionas `RS256` (recomendado), el token se firmará con la clave privada de tu tenant. Este valor se establece cuando se crea tu API y no puede modificarse después. Para obtener más información sobre los algoritmos de firma y cómo funcionan en Auth0, consulta [Signing Algorithms](/docs/es/get-started/applications/signing-algorithms).

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    La firma forma parte de un JWT. Si no estás familiarizado con la estructura de un JWT, consulta [JSON Web Token Structure](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure).
  </Callout>
* **JSON Web Encryption (JWE):** Cuando esta opción está habilitada, los tokens de acceso emitidos se cifran mediante JSON Web Encryption (JWE). Para obtener más información, consulta [JSON Web Encryption](/docs/es/secure/tokens/access-tokens/json-web-encryption).

<div id="rbac-settings">
  ### Configuración de RBAC
</div>

<Frame>![interruptor de RBAC en la configuración de API de Auth0 Dashboard](/docs/images/cdy7uua7fh8z/65tKb6aj0ktc2qXLUVlV3e/641adef615d6af9e5a3b588ff397af87/dashboard-apis-edit_view-settings_rbac-settings.png)</Frame>

* **Habilitar RBAC**: Habilita esta opción para que las políticas de RBAC se apliquen a la API. Para obtener más información, consulta [Control de acceso basado en roles](/docs/es/manage-users/access-control/rbac) y [Habilitar el control de acceso basado en roles para las APIs](/docs/es/get-started/apis/enable-role-based-access-control-for-apis).
* **Agregar permisos en el token de acceso**: Habilita esta opción para agregar el claim de permisos al token de acceso. Esta opción solo está disponible si habilitas RBAC. Puedes configurar los permisos en la pestaña **permisos**.

<div id="access-settings">
  ### Configuración de acceso
</div>

<Frame>![Dashboard Aplicaciones APIs pestaña Configuración Configuración de acceso](/docs/images/cdy7uua7fh8z/JNlWACqoyfmpE89JwWLKy/5a2b595f939167b0e6e1f72b485b08e7/2025-02-25_14-53-21.png)</Frame>

* **Permitir omitir el consentimiento del usuario**: Activa esta opción para que la API omita el consentimiento del usuario en las aplicaciones marcadas como &quot;de origen propio&quot;.
* **Permitir acceso sin conexión**: Activa esta opción para que las aplicaciones puedan solicitar <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> para la API.

<div id="default-policy-setting">
  #### Configuración de la política predeterminada
</div>

<Warning>
  El aseguramiento de la autenticación para la **My Account API** se encuentra actualmente en acceso anticipado con una política de una sola opción. Al usar esta funcionalidad, aceptas los términos aplicables de la prueba gratuita del [Master Subscription Agreement](https://www.okta.com/legal) de Okta. Para obtener más información sobre el ciclo de lanzamiento de productos de Auth0, consulta [Etapas de lanzamiento del producto](/docs/es/troubleshoot/product-lifecycle/product-release-stages). Para participar en el programa, ponte en contacto con el [equipo de soporte de Auth0](https://support.auth0.com).
</Warning>

<Frame>![Configuración de la API de la política predeterminada](/docs/images/api-settings-access-default-policy.png)</Frame>

* **Requerir 2FA**: Si se habilita, Auth0 aplica la política predeterminada a la My Account API para activar la [autenticación escalonada](/docs/es/secure/multi-factor-authentication/step-up-authentication). Los usuarios deben autenticarse con un segundo factor en un plazo de 15 minutos. Para conocer el comportamiento completo y cómo habilitarlo, consulta [My Account API: política predeterminada](/docs/es/manage-users/my-account-api#default-policy).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La política predeterminada no está disponible con Classic Login.
</Callout>

  ## Permisos

  ## Aplicaciones de máquina a máquina
