Puedes agregar permisos a una API con el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Warning>
  De forma predeterminada, cualquier usuario de cualquier aplicación puede solicitar cualquiera de los permisos definidos aquí. Puedes implementar políticas de acceso para limitar este comportamiento mediante [Rules](/docs/es/customize/rules).
</Warning>

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) y haz clic en el nombre de la API que quieres ver.

   <Frame>![Lista de APIs en Applications del Dashboard](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>
2. Ve a la pestaña **Permissions** e ingresa un nombre y una descripción para el permiso que quieres agregar. Asegúrate de no usar ningún nombre de permiso reservado (consulta la sección Nombres reservados).

   <Frame>![Dashboard Agregar permisos de API Pantalla para definir permisos](/docs/images/cdy7uua7fh8z/32Pb185OFs2mC6z2fmunEw/2708c52c2869d016066cc456dd00b6a2/dashboard-applications-apis-permissions.png)</Frame>
3. Haz clic en **Add**. Recuerda que es posible que las aplicaciones individuales necesiten actualizar sus permisos y/o scopes para interactuar correctamente con la API.

<div id="use-the-management-api">
  ## Usa la Management API
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Actualizar los permisos con un objeto vacío los elimina por completo.
</Callout>

Haz una llamada `PATCH` al [endpoint Update Resource Server](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id). Asegúrate de reemplazar los `API_ID`, `MGMT_API_ACCESS_TOKEN`, `PERMISSION_NAME` y `PERMISSION_DESC` placeholder values por tu ID de API, el <Tooltip tip="Access Token: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> de la Management API, los nombres de los permisos y las descripciones de los permisos, respectivamente. Asegúrate de no usar ningún nombre de permiso reservado (consulta la sección de nombres reservados).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Al agregar o actualizar scopes, la Management API requiere que envíes todos los scopes que quieras incluir. Si no envías alguno de los scopes existentes, se eliminará.
</Callout>

| Valor                   | Descripción                                                                                                                       |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `API_ID`                | ID de la API de la que quieres eliminar permisos.                                                                                 |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para la Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:resource_servers`. |
| `PERMISSION_NAME`       | Nombre o nombres de los permisos que quieres conservar para la API especificada.                                                  |
| `PERMISSION_DESC`       | Descripción o descripciones de los permisos que quieres conservar para la API especificada.                                       |

<div id="reserved-names">
  ## Nombres reservados
</div>

Los siguientes nombres de permisos están reservados y no pueden establecerse como permisos de custom API:

* address
* created&#95;at
* email
* email&#95;verified
* family&#95;name
* given&#95;name
* identities
* name
* nickname
* offline&#95;access
* <Tooltip tip="OpenID: Estándar abierto de autenticación que permite que las aplicaciones verifiquen la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=openid">openid</Tooltip>
* phone
* picture
* profile
