Este documento analiza los scopes incluidos en el protocolo de authentication OpenID Connect (OIDC) <Tooltip tip="OpenID: Estándar abierto de authentication que permite a las applications verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip>. Para obtener más información sobre OIDC, consulta [OpenID Connect Protocol](/docs/es/authenticate/protocols/openid-connect-protocol).

Una aplicación usa los scopes de OpenID Connect (OIDC) durante la authentication para autorizar el access a los datos de un usuario, como el nombre y la foto. Cada scope devuelve un conjunto de atributos del usuario, que se denominan claims. Los scopes que una aplicación debe solicitar dependen de los atributos de usuario que necesite. Una vez que el usuario autoriza los scopes solicitados, los claims se devuelven en un <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> y también están disponibles a través del [endpoint `/userinfo`](/docs/es/api/authentication/user-profile/get-user-info).

Por ejemplo, supongamos que has creado una aplicación web tradicional, la has registrado en Auth0 y la has configurado para permitir que un usuario inicie sesión con un username y Password. Una vez que un usuario inicia sesión en tu aplicación, quieres generar y enviar automáticamente un correo electrónico de bienvenida personalizado que incluya el nombre del usuario.

1. Un usuario hace clic en **Login** en tu aplicación.
2. Tu aplicación redirige al usuario al Auth0 Authorization Server (endpoint `/authorize`), incluidos los siguientes scopes:

   * `openid` (obligatorio; para indicar que la aplicación tiene la intención de usar OIDC para verificar la identidad del usuario)
   * `profile` (para que puedas personalizar el correo electrónico con el nombre del usuario)
   * `email` (para que sepas a dónde enviar el correo electrónico de bienvenida)
3. Tu Auth0 Authorization Server redirige al usuario al prompt de inicio de sesión.
4. El usuario se autentica y ve una página de consentimiento en la que se enumeran los scopes que Auth0 proporcionará a tu aplicación, incluido el access a la información de su perfil y a su dirección de correo electrónico.
5. El usuario acepta y autoriza a tu aplicación a tener este nivel de access a su información almacenada por Auth0.
6. Tu aplicación ahora tiene access a la información del perfil del usuario y a su dirección de correo electrónico.

<div id="standard-claims">
  ## Claims estándar
</div>

Las claims estándar están pensadas para proporcionar a una aplicación información del usuario, como el nombre, el correo electrónico y la foto, y están predefinidas para el protocolo OIDC. Estas claims se devuelven en un token de ID y también están disponibles a través del [`/userinfo` endpoint](/docs/es/api/authentication/user-profile/get-user-info).

También puedes crear claims personalizadas, es decir, claims que defines, controlas y agregas a un token mediante [Auth0 Actions](/docs/es/customize/actions/actions-overview). Para obtener más información, consulta [JSON Web Token Claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims).

El scope básico (y obligatorio) para OIDC es `openid`, que indica que una aplicación pretende usar el protocolo OIDC para verificar la identidad de un usuario. Además, una aplicación puede solicitar scopes adicionales indicando los nombres de los scopes solicitados en el parámetro `scope`, separados por espacios.

A continuación se enumeran las claims estándar incluidas en los scopes más utilizados, pero para obtener una lista completa de las claims estándar disponibles, consulta [OIDC specification: Standard Claims on openid.net](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Para obtener una lista completa de scopes, consulta [OIDC specification: Requesting Claims Using Scope Values on openid.net](https://openid.net/specs/openid-connect-core-1_0.html#ScopeClaims).

| Scope     | Claims                                                                                                                                                                                                                                                                                                                           |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `openid`  | (obligatorio) Devuelve la claim `sub`, que identifica de forma única al usuario. En un token de ID, también estarán presentes las claims `iss`, `aud`, `exp`, `iat` y `at_hash`. Para obtener más información sobre las claims del token de ID, consulta [ID Token Structure](/docs/es/secure/tokens/id-tokens/id-token-structure). |
| `profile` | Devuelve claims que representan información básica del perfil, incluidas `name`, `family_name`, `given_name`, `middle_name`, `nickname`, `picture` y `updated_at`.                                                                                                                                                               |
| `email`   | Devuelve la claim `email`, que contiene la dirección de correo electrónico del usuario, y `email_verified`, que es un valor booleano que indica si la dirección de correo electrónico fue verificada por el usuario.                                                                                                             |

Para ver un ejemplo de cómo solicitar claims estándar para tu aplicación, consulta [Casos de uso de ejemplo: Scopes y Claims](/docs/es/get-started/apis/scopes/sample-use-cases-scopes-and-claims).
