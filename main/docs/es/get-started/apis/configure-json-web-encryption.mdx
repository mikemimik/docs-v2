De forma predeterminada, Auth0 emite <Tooltip tip="Access Token: Credencial de autorización, en forma de cadena opaque o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> con formato de [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens) firmado, lo que significa que, aunque su integridad está protegida, los clientes y otros intermediarios aún pueden inspeccionarlos. Esto puede provocar una pérdida de privacidad de la información en datos destinados únicamente a exponerse a un <Tooltip tip="Resource Server: Servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/docs/es/glossary?term=resource+server">servidor de recursos</Tooltip>.

Para evitar la inspección no autorizada de los tokens de acceso, Auth0 admite el uso de tokens de acceso <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> anidados, donde la información de acceso se firma en un JWT y luego se cifra y se representa mediante JSON Web Encryption (JWE). Se espera que los servidores de recursos descifren estos tokens de acceso y verifiquen la firma del payload del JWT, mientras que la información permanece opaca para cualquier otra parte.

<div id="generate-rsa-key-pair">
  ## Generar un par de claves RSA
</div>

Antes de configurar una API para usar JWE, debes [generar un par de claves RSA](/docs/es/secure/application-credentials/generate-rsa-key-pair).

La clave privada debe mantenerse en secreto. Sube la clave pública a Auth0 en formato PEM, como se describe en [Generar un par de claves RSA](/docs/es/secure/application-credentials/generate-rsa-key-pair). Solo el servidor de recursos o el servidor de la API pueden acceder de forma segura a la clave privada para descifrar el token de acceso.

<div id="configure-jwe-for-an-api">
  ## Configurar JWE para una API
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    Usa el [Auth0 Dashboard](https://manage.auth0.com/) para configurar JWE en tu API. Para empezar, activa el toggle **JSON Web Encryption (JWE)** en **Token Settings** de tu API.

    <Frame>![](/docs/images/cdy7uua7fh8z/5xHKQ3rKN7DZogzD5qFDLp/8747f7f3d36c578e0f4622929cefb8e9/Access_Tokens_Settings_-_English.png)</Frame>

    Cuando se te solicite, agrega una clave de JSON Web Encryption (JWE):

    * Ingresa un nombre descriptivo para identificarla fácilmente.
    * Carga un certificado con la clave pública codificada en formato PEM.
    * Selecciona el algoritmo de cifrado.
    * (Opcional) Ingresa un identificador de clave.

    Haz clic en **Add** para guardar la clave JWE, lo que generará una huella digital del certificado.
  </Tab>

  <Tab title="Management API">
    Usa la [Auth0 Management API](https://auth0.com/docs/api/management/v2) para configurar JWE en un servidor de recursos o servidor de la API. Debes configurar JWE para cada API, por lo que cada una tendrá su propia clave pública de cifrado.

    La siguiente solicitud POST configura una nueva API para recibir tokens de acceso cifrados:

    ```bash lines
    curl -X POST --location "https://{domain}/api/v2/resource-servers" \
        -H "Authorization: Bearer {managementAccessToken}" \
        -H "Content-Type: application/json" \
        --data-raw '{
      "name": "{apiName}",
      "identifier": "{apiIdentifier}",
      "token_encryption": {
        "format": "compact-nested-jwe",
        "encryption_key":
        {
          "name": "{credentialName}",
          "pem": "{pem}",
          "alg": "{alg}",
          "kid": "{kid}"
        }
      }
    }'
    ```

    La siguiente tabla explica el significado de los distintos parámetros:

    | Parámetro        | ¿Obligatorio? | Descripción                                                                                                                                                                   |
    | ---------------- | ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `apiName`        | Obligatorio   | El nombre de tu nueva API.                                                                                                                                                    |
    | `apiIdentifier`  | Obligatorio   | El identificador único de tu API. Se usará como audience de tu token.                                                                                                         |
    | `credentialName` | Opcional      | El nombre de tu clave pública.                                                                                                                                                |
    | `pem`            | Obligatorio   | Clave pública codificada en formato PEM.                                                                                                                                      |
    | `alg`            | Obligatorio   | El algoritmo de cifrado debe ser `RSA-OAEP-256`, `RSA-OAEP-384` o `RSA-OAEP-512`.                                                                                             |
    | `kid`            | Opcional      | El identificador que se usa para escribir en el header `kid` de tu JWE token. Puede usarse para identificar la clave utilizada para el cifrado durante la rotación de claves. |

    La respuesta contiene la propiedad `id`, que identifica de forma única el servidor de recursos. También contiene un campo `thumbprint_sha256` generado que puede usarse para identificar la credencial. Auth0 no devolverá material de clave después de la creación inicial (en este caso, tu PEM).

    Hay muchas formas de generar `thumbprint_sha256`. Para más información, consulta la [RFC 8705 OAuth 2.0 Mutual-TLS Client Authentication and Certificate-Bound Access Tokens](https://datatracker.ietf.org/doc/html/rfc8705#name-jwt-certificate-thumbprint-).

    Para asegurarte de haber generado el `thumbprint_sha256` correcto, puedes usar el siguiente ejemplo de código en Node.js para extraer la huella digital:

    ```javascript lines
    const fs = require('fs');
    const crypto = require('crypto');

    const { calculateJwkThumbprint, exportJWK } = require('jose');

    const publicKeyObject = crypto.createPublicKey(fs.readFileSync('./my_cert.pem'));
    exportJWK(publicKeyObject).then((jwk) => {
      calculateJwkThumbprint(jwk, 'sha256').then((thumbprint) => {
        console.log(thumbprint);
      });
    });
    ```
