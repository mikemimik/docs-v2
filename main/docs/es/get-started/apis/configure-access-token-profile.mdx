export const codeExample = `curl --request PATCH \\
  --url 'https://{yourDomain}/api/v2/resource-servers/API_ID' \\
  --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \\
  --header 'content-type: application/json' \\
  --data '{"token_dialect": "TOKEN_DIALECT" }'

Puedes elegir qué perfil de <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> usar para tus API: el perfil de token de acceso de Auth0 o el perfil de token de acceso RFC 9068. El perfil de token de acceso que configures determina el formato de los tokens de acceso emitidos para la API.

De forma predeterminada, Auth0 emite tokens de acceso con el perfil de token de acceso de Auth0. El perfil de token de acceso de Auth0 emite tokens de acceso en formato [JSON Web Tokens (JWTs)](/docs/es/secure/tokens/json-web-tokens), que contienen información sobre una entidad en forma de claims.

Auth0 también admite el perfil de token de acceso RFC 9068. El perfil de token de acceso RFC 9068 emite tokens de acceso en formato <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de token de acceso) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWTs">JWTs</Tooltip> conforme al [IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068). Para obtener más información sobre las diferencias entre estos perfiles de token, consulta [Access Token Profiles](/docs/es/secure/tokens/access-tokens/access-token-profiles).

Cuando [registres una API](/docs/es/get-started/auth0-overview/set-up-apis), puedes seleccionar el perfil de token de acceso en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Después de registrar la API, puedes configurar el perfil de token de acceso en cualquier momento mediante la [Management API](https://auth0.com/docs/api/management/v2) y el [Auth0 Dashboard](https://manage.auth0.com/).

<div id="configure-access-token-profile-for-an-api">
  ## Configurar el perfil de token de acceso para una API
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Ve a [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) y haz clic en el nombre de la API para verla.

    2. Desplázate hasta **Access Token Settings** y selecciona el perfil de token de acceso en **JSON Web Token (JWT) Profile**. El perfil seleccionado determina el formato y los claims de los tokens de acceso emitidos para la API. Los valores admitidos son `Auth0` y `RFC 9068`. Para obtener más información sobre la diferencia entre ambos perfiles, consulta [Access Token Profiles](/docs/es/secure/tokens/access-tokens/access-token-profiles).

    3. Haz clic en **Save**.
  </Tab>

  <Tab title="Management API">

    Reemplaza `API_ID`, `MGMT_API_ACCESS_TOKEN` y `TOKEN_DIALECT` por sus respectivos valores, como se describe en la siguiente tabla:

    | Parámetro               | Descripción                                                                                                                                    |
    | ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
    | `API_ID`                | ID de la API cuyo dialecto de token quieres actualizar.                                                                                        |
    | `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:resource_servers`.                 |
    | `TOKEN_DIALECT`         | Dialecto del token de acceso para la API especificada. Para obtener más información, consulta [Token dialect options](#token-dialect-options). |

    #### Opciones de dialecto de token

    Auth0 admite los siguientes dialectos de token:

    | Valor                   | Descripción                                                                                                                                                                                                                                                                                                                 |
    | ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `access_token`          | El perfil de token de acceso de Auth0 predeterminado genera un token de acceso con formato JSON Web Token (JWT). Para obtener más información, consulta [Access Token Profiles](/docs/es/secure/tokens/access-tokens/access-token-profiles).                                                                                       |
    | `access_token_authz`    | El perfil de token de acceso de Auth0 predeterminado (`access_token`) con el claim `permissions`. Para obtener más información sobre los permisos de RBAC, consulta [Enable Role-Based Access Control for APIs](/docs/es/get-started/apis/enable-role-based-access-control-for-apis).                                              |
    | `rfc9068_profile`       | El perfil de token de acceso RFC 9068 genera un token de acceso en formato JWT conforme a [IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068). Para obtener más información, consulta [Access Token Profiles](/docs/es/secure/tokens/access-tokens/access-token-profiles). |
    | `rfc9068_profile_authz` | El perfil RFC 9068 (`rfc9068_profile`) con el claim `permissions`. Para obtener más información sobre los permisos de RBAC, consulta [Enable Role-Based Access Control for APIs](/docs/es/get-started/apis/enable-role-based-access-control-for-apis).                                                                |
  </Tab>
