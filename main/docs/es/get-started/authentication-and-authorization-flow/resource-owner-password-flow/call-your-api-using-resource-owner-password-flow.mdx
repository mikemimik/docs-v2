<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este tutorial te ayudará a llamar a tu propia API mediante el flujo de contraseña del propietario del recurso. Si quieres saber cómo funciona este flujo y por qué deberías usarlo, consulta [Flujo de contraseña del propietario del recurso](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow).
</Callout>

<Warning>
  Dado que el flujo de contraseña del propietario del recurso (ROP) implica que la aplicación gestione la contraseña del usuario, no debe utilizarse con clientes de terceros.
</Warning>

Auth0 facilita que tu aplicación implemente el flujo de contraseña de <Tooltip tip="Resource Owner: Entidad (como un usuario o una aplicación) capaz de otorgar acceso a un recurso protegido." cta="Ver glosario" href="/docs/es/glossary?term=Resource+Owner">Resource Owner</Tooltip> (a veces llamado concesión de contraseña del propietario del recurso o ROPG) mediante la [Authentication API](https://auth0.com/docs/api/authentication). Sigue leyendo para aprender a llamar directamente a nuestra API.

<div id="prerequisites">
  ## Requisitos previos
</div>

**Antes de comenzar este tutorial:**

* [Registra tu aplicación con Auth0](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps).

  * Selecciona un **Tipo de aplicación** de **Regular Web Apps**.
  * Agrega una **Allowed Callback URL** de `{https://yourApp/callback}`. Este campo no puede quedar sin definir; de lo contrario, se devolverá un mensaje de error.
  * Asegúrate de que los **Tipos de concesión** de tu aplicación incluyan **Password**. Para saber cómo hacerlo, consulta [Update Grant Types](/docs/es/get-started/applications/update-grant-types).
  * Si quieres que tu aplicación pueda usar tokens de actualización, asegúrate de que los **Tipos de concesión** de la aplicación incluyan **token de actualización**. Para saber cómo hacerlo, consulta [Update Grant Types](/docs/es/get-started/applications/update-grant-types). Para obtener más información sobre los tokens de actualización, consulta [Refresh Tokens](/docs/es/secure/tokens/refresh-tokens).
* [Registra tu API con Auth0](/docs/es/get-started/auth0-overview/set-up-apis)

  * Si quieres que tu API reciba tokens de actualización para poder obtener tokens nuevos cuando caduquen los anteriores, habilita **Allow Offline Access**.
* Configura una conexión

  * Asegúrate de que tu conexión pueda autenticar usuarios con nombre de usuario y contraseña (por ejemplo, [conexiones de base de datos](/docs/es/get-started/applications/set-up-database-connections) o conexiones AD/LDAP, ADFS o Azure Active Directory [empresariales](/docs/es/authenticate/identity-providers/enterprise-identity-providers)).
* Actualiza o deshabilita cualquier [rules](/docs/es/customize/rules) para que solo afecten a conexiones específicas. Si recibes un error `access_denied` al probar la concesión Password Owner Resource Grant, podría deberse a una regla de control de acceso.

<div id="configure-tenant">
  ### Configura el tenant
</div>

El flujo de contraseña del propietario del recurso depende de una conexión que pueda autenticar a los usuarios con nombre de usuario y contraseña, por lo que debes establecer la conexión predeterminada del tenant.

1. Ve a [Auth0 Dashboard &gt; Configuración del tenant](https://manage.auth0.com/#/tenant) y desplázate hacia abajo hasta encontrar la opción **Directorio predeterminado**.
2. Introduce el nombre de la conexión que quieras usar. Asegúrate de que pueda autenticar a los usuarios con nombre de usuario y contraseña.

<div id="request-tokens">
  ### Obtener tokens
</div>

Para llamar a tu API, primero debes obtener las credenciales del usuario, normalmente mediante un formulario interactivo. Una vez que tu aplicación tenga las credenciales, debes intercambiarlas por tokens. Para ello, debes enviar una solicitud `POST` a la [URL del token](https://auth0.com/docs/api/authentication#resource-owner-password).

<div id="example-post-to-token-url">
  #### Ejemplo de POST a la URL del token
</div>

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/oauth/token' \
    --header 'content-type: application/x-www-form-urlencoded' \
    --data grant_type=password \
    --data 'username={username}' \
    --data 'password={password}' \
    --data 'audience={yourApiIdentifier}' \
    --data scope=read:sample \
    --data 'client_id={yourClientId}' \
    --data 'client_secret={yourClientSecret}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/oauth/token");
  var request = new RestRequest(Method.POST);
  request.AddHeader("content-type", "application/x-www-form-urlencoded");
  request.AddParameter("application/x-www-form-urlencoded", "grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/oauth/token"

  	payload := strings.NewReader("grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("content-type", "application/x-www-form-urlencoded")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse response = Unirest.post("https://{yourDomain}/oauth/token")
    .header("content-type", "application/x-www-form-urlencoded")
    .body("grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D")
    .asString();
  ```

  ```javascript Node.JS

  var options = {
    method: 'POST',

  }).catch(function (error) {
    console.error(error);
  });
  ```


  ```python Python

  payload = "grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D"

  ```

<div id="parameters">
  ##### Parámetros
</div>

| Nombre del parámetro    | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `grant_type`            | Establézcalo en `password`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `username`              | El nombre de usuario introducido por el usuario.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `password`              | La contraseña introducida por el usuario.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `client_id`             | El ID de cliente de su aplicación. Puede encontrar este valor en [Application Settings](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `client_assertion`      | Un JWT que contiene una aserción firmada con las credenciales de su aplicación. Obligatorio cuando la clave privada JWT es su método de autenticación de la aplicación.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `client_assertion_type` | El valor es `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Obligatorio cuando la clave privada JWT es el método de autenticación de la aplicación.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `client_secret`         | El secreto de cliente de su aplicación. Obligatorio cuando el secreto de cliente es el método de autenticación de la aplicación. [Application Settings](https://manage.auth0.com/#/applications/\{yourClientId}/settings) es `Post` o `Basic`. Si su aplicación no es de alta confianza (por ejemplo, una SPA), no establezca este parámetro.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `audience`              | La audiencia del token, que es su API. Puede encontrarla en el campo **Identifier** de la [pestaña de configuración de su API](https://manage.auth0.com/#/apis).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `scope`                 | Especifica los [scopes](/docs/es/get-started/apis/scopes) para los que desea solicitar autorización, que determinan qué claims (o atributos del usuario) quiere que se devuelvan. Deben separarse con un espacio. Puede solicitar cualquiera de los [scopes estándar de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) sobre usuarios, como `profile` o `email`, [custom claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims#custom-claims) que sigan un [formato con espacio de nombres](/docs/es/secure/tokens/json-web-tokens/create-custom-claims), o cualquier scope compatible con la API de destino (por ejemplo, `read:contacts`). Incluya `offline_access` para obtener un [token de actualización](/docs/es/glossary?term=Refresh+Token) (asegúrese de que el campo **Allow Offline Access** esté habilitado en [Application Settings](https://manage.auth0.com/#/applications)). |

<div id="response">
  #### Respuesta
</div>

Si todo va bien, recibirás una respuesta `HTTP 200` con un payload que incluye los valores `access_token`, `refresh_token`, `id_token`, `token_type` y `expires_in`:

```json lines
{
  "access_token": "eyJz93a...k4laUWw",
  "refresh_token": "GEbRxBN...edjnXbL",
  "id_token": "eyJ0XAi...4faeEoQ",
  "token_type": "Bearer",
  "expires_in": 36000
}
```

<Warning>
  Valida tus tokens antes de guardarlos. Para saber cómo hacerlo, consulta [Validar tokens de ID](/docs/es/secure/tokens/id-tokens/validate-id-tokens) y [Validar tokens de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).
</Warning>

<Warning>
  Los tokens de actualización deben almacenarse de forma segura, ya que permiten que un usuario permanezca autenticado prácticamente para siempre.
</Warning>

<Card title="Flujo de contraseña del propietario del recurso y scopes estándar">
  Como proporcionar una contraseña otorga acceso completo, cualquier intercambio basado en contraseña da acceso a todos los scopes. Por ejemplo, si no incluyes [Scopes de API](/docs/es/get-started/apis/scopes/api-scopes) en la solicitud, todos los scopes de API se incluirán en el token de acceso. Del mismo modo, si incluyes solo el scope `openid` en la solicitud, se devolverán todos los [Scopes de OpenID Connect](/docs/es/get-started/apis/scopes/openid-connect-scopes) estándar de `openid`. En estos casos, el parámetro `scope` se incluirá en la respuesta e indicará los scopes emitidos.
</Card>

<Card title="Obtener información del usuario sin un token de ID">
  Si necesitas la información del usuario, incluye el scope `openid` en tu solicitud. Si la API usa `RS256` como [algoritmo de firma](/docs/es/get-started/applications/signing-algorithms), el token de acceso incluirá `/userinfo` como una audience válida, lo que significa que puedes usarlo para invocar el [endpoint /userinfo](https://auth0.com/docs/api/authentication#get-user-info) y recuperar los claims del usuario.
</Card>

<div id="refresh-tokens">
  ### Tokens de actualización
</div>

Ya has recibido un [token de actualización](/docs/es/secure/tokens/refresh-tokens) si has seguido este tutorial y completaste lo siguiente:

* configuraste tu API para permitir el acceso sin conexión
* incluiste el scope `offline_access` cuando iniciaste la solicitud de autenticación a través del [endpoint de autorización](https://auth0.com/docs/api/authentication/reference#authorize-application).

Puedes usar el <Tooltip tip="Token de actualización: token que se usa para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token">token de actualización</Tooltip> para obtener un nuevo token de acceso. Por lo general, un usuario solo necesitará un nuevo token de acceso cuando el anterior caduque o cuando acceda a un recurso nuevo por primera vez. No es una buena práctica llamar al endpoint para obtener un nuevo token de acceso cada vez que llamas a una API, y Auth0 aplica límites de frecuencia que restringen la cantidad de solicitudes al endpoint que se pueden ejecutar con el mismo token desde la misma IP.

Para actualizar tu token, haz una solicitud `POST` al endpoint `/oauth/token` en la Authentication API con `grant_type=refresh_token`.

  ```ruby Ruby
  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Post.new(url)
  request["content-type"] = 'application/x-www-form-urlencoded'
  request.body = "grant_type=refresh_token&client_id={yourClientId}&refresh_token=%7ByourRefreshToken%7D"

  response = http.request(request)
  puts response.read_body
  ```
</AuthCodeGroup>

<div id="parameters">
  ##### Parámetros
</div>

| Nombre del parámetro | Descripción                                                                                                                                                                                                                                          |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `grant_type`         | Configure este valor como `refresh_token`.                                                                                                                                                                                                           |
| `client_id`          | El ID de cliente de su aplicación. Puede encontrar este valor en [Configuración de la aplicación](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).                                                                                 |
| `refresh_token`      | El token de actualización que se usará.                                                                                                                                                                                                              |
| `scope`              | (opcional) Una lista de permisos de scope solicitados, separados por espacios. Si no se envía, se usarán los scopes originales; de lo contrario, puede solicitar un conjunto reducido de scopes. Tenga en cuenta que debe estar codificado como URL. |

<div id="response">
  #### Respuesta
</div>

Si todo sale bien, recibirás una respuesta `HTTP 200` con un payload que incluye un nuevo `access_token`, su tiempo de validez en segundos (`expires_in`), los valores de `scope` concedidos y `token_type`.

```json lines
{
  "access_token": "eyJ...MoQ",
  "expires_in": 86400,
  "scope": "openid offline_access",
  "token_type": "Bearer"
}
```

<Warning>
  Valida tus tokens antes de guardarlos. Para saber cómo hacerlo, lee [Validar token de ID](/docs/es/secure/tokens/id-tokens/validate-id-tokens) y [Validar tokens de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).
</Warning>

<div id="customize-tokens">
  #### Personaliza los tokens
</div>

Puedes usar Actions para cambiar los scopes que se devuelven en los tokens de acceso y/o añadir claims a los tokens de acceso y <Tooltip tip="token de ID: credencial destinada al propio cliente, no a acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Tokens">tokens de ID</Tooltip>. (Para obtener más información sobre Actions, consulta [Auth0 Actions](/docs/es/customize/actions).) Para ello, añade la siguiente Action, que se ejecutará después de que el usuario se autentique:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  // Agregar claims personalizados al Access Token y al token de ID
  api.accessToken.setCustomClaim('https://foo/bar', 'value');
  api.idToken.setCustomClaim('https://fiz/baz', 'some other value');

  // Modificar el scope del Access Token
  api.accessToken.addScope('foo');
  api.accessToken.addScope('bar');
};
```

Los scopes estarán disponibles en el token una vez que se haya ejecutado la Action.

<Warning>
  Auth0 devuelve la información del perfil en un formato estructurado de claims, tal como lo define la [especificación de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Esto significa que los claims personalizados que se agreguen a los tokens de ID o a los tokens de acceso deben [ajustarse a las directrices y restricciones](/docs/es/secure/tokens/json-web-tokens/create-custom-claims) para evitar posibles conflictos.
</Warning>

<div id="configure-realm-support">
  ### Configurar el uso de realm
</div>

Auth0 proporciona una concesión de extensión que ofrece una funcionalidad similar a la concesión de contraseña del propietario del recurso, pero le permite mantener directorios de usuarios independientes (que corresponden a conexiones independientes) y especificar cuál usar durante el flujo.

Para usar esta variante, debe:

* Establecer el parámetro de solicitud `grant_type` en `http://auth0.com/oauth/grant-type/password-realm`.
* Enviar un parámetro de solicitud adicional llamado `realm` y establecerlo con el nombre del realm al que pertenece el usuario. Por ejemplo, si ha configurado una conexión de base de datos para empleados internos llamada `employees` y el usuario pertenece a ella, establezca `realm` en `employees`.

<Card title="Conexiones como realms">
  Cualquier conexión que admita autenticación activa puede configurarse como un realm, incluidas las [conexiones de base de datos](/docs/es/get-started/applications/set-up-database-connections), las [conexiones passwordless](/docs/es/authenticate/passwordless) y las conexiones empresariales [AD/LDAP](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap), [ADFS](/docs/es/authenticate/identity-providers/enterprise-identity-providers/adfs) y [Azure Active Directory](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).
</Card>

<div id="configure-mfa">
  ### Configurar MFA
</div>

Si necesitas usar el flujo de contraseña del propietario del recurso, pero requieres una autenticación más robusta, puedes agregar <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA). Para saber cómo hacerlo, consulta [Autenticarse con el flujo de contraseña del propietario del recurso y MFA](/docs/es/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa).

<div id="configure-attack-protection">
  ### Configurar la protección contra ataques
</div>

Al usar el flujo de contraseña del propietario del recurso con <Tooltip tip="Protección contra ataques de fuerza bruta: tipo de protección contra ataques que protege frente a ataques de fuerza bruta que se originan en una sola dirección IP y se dirigen a una única cuenta de usuario." cta="Ver glosario" href="/docs/es/glossary?term=brute-force+protection">protección contra ataques de fuerza bruta</Tooltip>, es posible que algunas funcionalidades de <Tooltip tip="Protección contra ataques de fuerza bruta: tipo de protección contra ataques que protege frente a ataques de fuerza bruta que se originan en una sola dirección IP y se dirigen a una única cuenta de usuario." cta="Ver glosario" href="/docs/es/glossary?term=attack+protection">protección contra ataques</Tooltip> no funcionen correctamente. Sin embargo, es posible evitar algunos problemas habituales. Para obtener más información, lee [Cómo evitar problemas habituales con el flujo de contraseña del propietario del recurso y la protección contra ataques](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
