<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este tutorial te ayudará a invocar tu propia API mediante el flujo de contraseña del propietario del recurso. Si quieres saber cómo funciona este flujo y por qué deberías usarlo, consulta [flujo de contraseña del propietario del recurso](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow).
</Callout>

<Warning>
  Dado que el flujo de contraseña del propietario del recurso (ROP) implica que la aplicación gestione la contraseña del usuario, no debe ser utilizado por clientes de terceros.
</Warning>

Auth0 facilita la implementación en tu aplicación del flujo de contraseña de <Tooltip tip="Resource Owner: Entidad (como un usuario o una aplicación) capaz de conceder acceso a un recurso protegido." cta="Ver glosario" href="/docs/es/glossary?term=Resource+Owner">Resource Owner</Tooltip> (a veces llamado Resource Owner Password Grant o ROPG) mediante la [Authentication API](https://auth0.com/docs/api/authentication). Sigue leyendo para aprender a invocar nuestra API directamente.

<div id="prerequisites">
  ## Requisitos previos
</div>

**Antes de comenzar este tutorial:**

* [Registra tu aplicación con Auth0](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps).

  * Selecciona **Regular Web Apps** en **Tipo de aplicación**.
  * Agrega una **URL de devolución de llamada permitida** de `{https://yourApp/callback}`. Este campo no puede ser `undefined`; de lo contrario, se devolverá un mensaje de error.
  * Asegúrate de que los **tipos de concesión** de tu aplicación incluyan **Password**. Para saber cómo, lee [Update Grant Types](/docs/es/get-started/applications/update-grant-types).
  * Si quieres que tu aplicación pueda usar **tokens de actualización**, asegúrate de que los **tipos de concesión** de la aplicación incluyan **token de actualización**. Para saber cómo, lee [Update Grant Types](/docs/es/get-started/applications/update-grant-types). Para obtener más información sobre los **tokens de actualización**, consulta [Refresh Tokens](/docs/es/secure/tokens/refresh-tokens).
* [Registra tu API con Auth0](/docs/es/get-started/auth0-overview/set-up-apis)

  * Si quieres que tu API reciba **tokens de actualización** para poder obtener nuevos tokens cuando caduquen los anteriores, habilita **Allow Offline Access**.
* Configura una conexión

  * Asegúrate de que tu conexión pueda autenticar a los usuarios con nombre de usuario y contraseña (por ejemplo, [conexiones de base de datos](/docs/es/get-started/applications/set-up-database-connections) o conexiones empresariales de AD/LDAP, ADFS o Azure Active Directory [conexiones empresariales](/docs/es/authenticate/identity-providers/enterprise-identity-providers)).
* Actualiza o deshabilita cualquier [rules](/docs/es/customize/rules) para que solo afecte a conexiones específicas. Si obtienes un error `access_denied` mientras pruebas Password Owner Resource Grant, esto podría deberse a una regla de control de acceso.

<div id="configure-tenant">
  ### Configurar el tenant
</div>

El flujo de contraseña del propietario del recurso depende de una conexión que pueda autenticar a los usuarios con nombre de usuario y contraseña, por lo que debes establecer la conexión predeterminada del tenant.

1. Ve a [Auth0 Dashboard &gt; Configuración del tenant](https://manage.auth0.com/#/tenant) y desplázate hacia abajo hasta encontrar la opción **Directorio predeterminado**.
2. Introduce el nombre de la conexión que quieras usar. Asegúrate de que pueda autenticar a los usuarios con nombre de usuario y contraseña.

<div id="request-tokens">
  ### Solicitar tokens
</div>

Para llamar a tu API, primero debes obtener las credenciales del usuario, normalmente mediante un formulario interactivo. Una vez que tu aplicación tenga las credenciales, debes intercambiarlas por tokens. Para hacerlo, debes hacer un `POST` a la [URL del token](https://auth0.com/docs/api/authentication#resource-owner-password).

<div id="example-post-to-token-url">
  #### Ejemplo de POST a la URL del token
</div>

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/oauth/token' \
    --header 'content-type: application/x-www-form-urlencoded' \
    --data grant_type=password \
    --data 'username={username}' \
    --data 'password={password}' \
    --data 'audience={yourApiIdentifier}' \
    --data scope=read:sample \
    --data 'client_id={yourClientId}' \
    --data 'client_secret={yourClientSecret}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/oauth/token");
  var request = new RestRequest(Method.POST);
  request.AddHeader("content-type", "application/x-www-form-urlencoded");
  request.AddParameter("application/x-www-form-urlencoded", "grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"strings"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/oauth/token"

  	payload := strings.NewReader("grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D")

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("content-type", "application/x-www-form-urlencoded")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse response = Unirest.post("https://{yourDomain}/oauth/token")
    .header("content-type", "application/x-www-form-urlencoded")
    .body("grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D")
    .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'POST',
    url: 'https://{yourDomain}/oauth/token',
    headers: {'content-type': 'application/x-www-form-urlencoded'},
    data: new URLSearchParams({
      grant_type: 'password',
      username: '{username}',
      password: '{password}',
      audience: '{yourApiIdentifier}',
      scope: 'read:sample',

      client_secret: '{yourClientSecret}'
    })
  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/oauth/token",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_ENCODING => "",

    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
    CURLOPT_CUSTOMREQUEST => "POST",
    CURLOPT_POSTFIELDS => "grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D",

      "content-type: application/x-www-form-urlencoded"

  if ($err) {
    echo "cURL Error #:" . $err;
  } else {
    echo $response;
  }
  ```

  ```python Python
  import http.client

  conn = http.client.HTTPSConnection("")

  payload = "grant_type=password&username=%7Busername%7D&password=%7Bpassword%7D&audience=%7ByourApiIdentifier%7D&scope=read%3Asample&client_id={yourClientId}&client_secret=%7ByourClientSecret%7D"

  headers = { 'content-type': "application/x-www-form-urlencoded" }

  data = res.read()

  print(data.decode("utf-8"))
  ```

<div id="parameters">
  ##### Parámetros
</div>

| Nombre del parámetro    | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `grant_type`            | Establécelo en `password`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `username`              | El nombre de usuario introducido por el usuario.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `password`              | La contraseña introducida por el usuario.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| `client_id`             | El ID de cliente de tu aplicación. Puedes encontrar este valor en [Configuración de la aplicación](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `client_assertion`      | Un JWT que contiene una aserción firmada con las credenciales de tu aplicación. Obligatorio cuando la clave privada JWT es tu método de autenticación de la aplicación.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| `client_assertion_type` | El valor es `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Obligatorio cuando la clave privada JWT es el método de autenticación de la aplicación.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `client_secret`         | El secreto de cliente de tu aplicación. Obligatorio cuando el secreto de cliente es el método de autenticación de la aplicación y en [Configuración de la aplicación](https://manage.auth0.com/#/applications/\{yourClientId}/settings) es `Post` o `Basic`. Si tu aplicación no es de alta confianza (por ejemplo, una SPA), no configures este parámetro.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `audience`              | La audiencia del token, que es tu API. Puedes encontrarla en el campo **Identifier** de la [pestaña de configuración de tu API](https://manage.auth0.com/#/apis).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `scope`                 | Especifica los [scopes](/docs/es/get-started/apis/scopes) para los que quieres solicitar autorización, que determinan qué claims (o atributos de usuario) quieres que se devuelvan. Deben estar separados por espacios. Puedes solicitar cualquiera de los [scopes estándar de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) sobre los usuarios, como `profile` o `email`, [custom claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims#custom-claims) que sigan un [formato con espacio de nombres](/docs/es/secure/tokens/json-web-tokens/create-custom-claims), o cualquier scope compatible con la API de destino (por ejemplo, `read:contacts`). Incluye `offline_access` para obtener un [token de actualización](/docs/es/glossary?term=Refresh+Token) (asegúrate de que el campo **Allow Offline Access** esté habilitado en [Configuración de la aplicación](https://manage.auth0.com/#/applications)). |

<div id="response">
  #### Respuesta
</div>

Si todo sale bien, recibirás una respuesta `HTTP 200` con un payload que contiene los valores `access_token`, `refresh_token`, `id_token`, `token_type` y `expires_in`:

```json lines
{
  "access_token": "eyJz93a...k4laUWw",
  "refresh_token": "GEbRxBN...edjnXbL",
  "id_token": "eyJ0XAi...4faeEoQ",
  "token_type": "Bearer",
  "expires_in": 36000
}
```

<Warning>
  Valida tus tokens antes de guardarlos. Para saber cómo hacerlo, consulta [Validar tokens de ID](/docs/es/secure/tokens/id-tokens/validate-id-tokens) y [Validar tokens de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).
</Warning>

<Warning>
  Los tokens de actualización deben almacenarse de forma segura, ya que permiten que un usuario permanezca autenticado prácticamente para siempre.
</Warning>

<Card title="flujo de contraseña del propietario del recurso y scopes estándar">
  Como proporcionar una contraseña da acceso completo, cualquier intercambio basado en contraseña da acceso a todos los scopes. Por ejemplo, si no incluyes [Scopes de API](/docs/es/get-started/apis/scopes/api-scopes) en la solicitud, todos los scopes de API se incluirán en el token de acceso. Del mismo modo, si incluyes solo el scope `openid` en la solicitud, se devolverán todos los [Scopes de OpenID Connect](/docs/es/get-started/apis/scopes/openid-connect-scopes) estándar de `openid`. En estos casos, el parámetro `scope` se incluirá en la respuesta e indicará los scopes emitidos.
</Card>

<Card title="Obtener información del usuario sin un token de ID">
  Si necesitas la información del usuario, incluye el scope `openid` en tu solicitud. Si la API usa `RS256` como [algoritmo de firma](/docs/es/get-started/applications/signing-algorithms), el token de acceso incluirá `/userinfo` como audiencia válida, lo que significa que puedes usarlo para invocar el [endpoint /userinfo](https://auth0.com/docs/api/authentication#get-user-info) y recuperar los claims del usuario.
</Card>

<div id="refresh-tokens">
  ### Tokens de actualización
</div>

Ya has recibido un [token de actualización](/docs/es/secure/tokens/refresh-tokens) si has seguido este tutorial y has completado lo siguiente:

* configuraste tu API para permitir el acceso sin conexión
* incluiste el `scope` `offline_access` cuando iniciaste la solicitud de autenticación mediante el [endpoint de autorización](https://auth0.com/docs/api/authentication/reference#authorize-application).

Puedes usar el <Tooltip tip="Token de actualización: token que se usa para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Token">token de actualización</Tooltip> para obtener un nuevo token de acceso. Por lo general, un usuario solo necesitará un nuevo token de acceso después de que caduque el anterior o cuando obtenga acceso a un recurso nuevo por primera vez. No es una buena práctica llamar al endpoint para obtener un nuevo token de acceso cada vez que llamas a una API, y Auth0 aplica límites de frecuencia que restringen la cantidad de solicitudes al endpoint que pueden ejecutarse con el mismo token desde la misma IP.

Para actualizar tu token, haz una solicitud `POST` al endpoint `/oauth/token` en la Authentication API, usando `grant_type=refresh_token`.

  ```ruby Ruby
  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Post.new(url)
  request["content-type"] = 'application/x-www-form-urlencoded'
  request.body = "grant_type=refresh_token&client_id={yourClientId}&refresh_token=%7ByourRefreshToken%7D"

  response = http.request(request)
  puts response.read_body
  ```
</AuthCodeGroup>

<div id="parameters">
  ##### Parámetros
</div>

| Nombre del parámetro | Descripción                                                                                                                                                                                                                              |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `grant_type`         | Establezca este valor en `refresh_token`.                                                                                                                                                                                                |
| `client_id`          | El ID de cliente de su aplicación. Puede encontrar este valor en la [Configuración de la aplicación](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).                                                                  |
| `refresh_token`      | El token de actualización que se utilizará.                                                                                                                                                                                              |
| `scope`              | (opcional) Una lista de scopes solicitados, delimitada por espacios. Si no se envía, se usarán los scopes originales; de lo contrario, puede solicitar un conjunto reducido de scopes. Tenga en cuenta que debe estar codificado en URL. |

<div id="response">
  #### Respuesta
</div>

Si todo sale bien, recibirás una respuesta `HTTP 200` con un payload que incluye un nuevo `access_token`, su tiempo de validez en segundos (`expires_in`), los valores de `scope` otorgados y `token_type`.

```json lines
{
  "access_token": "eyJ...MoQ",
  "expires_in": 86400,
  "scope": "openid offline_access",
  "token_type": "Bearer"
}
```

<Warning>
  Valida tus tokens antes de guardarlos. Para saber cómo hacerlo, consulta [Validar tokens de ID](/docs/es/secure/tokens/id-tokens/validate-id-tokens) y [Validar tokens de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).
</Warning>

<div id="customize-tokens">
  #### Personaliza los tokens
</div>

Puedes usar Actions para cambiar los scopes que se devuelven en los tokens de acceso y/o añadir claims a los tokens de acceso y a los <Tooltip tip="token de ID: credencial destinada al propio cliente, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Tokens">tokens de ID</Tooltip>. (Para obtener más información sobre Actions, consulta [Auth0 Actions](/docs/es/customize/actions).) Para ello, agrega la siguiente Action, que se ejecutará después de que el usuario se autentique:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  // Agregar claims personalizados al Access Token y al token de ID
  api.accessToken.setCustomClaim('https://foo/bar', 'value');
  api.idToken.setCustomClaim('https://fiz/baz', 'some other value');

  // Modificar el scope del Access Token
  api.accessToken.addScope('foo');
  api.accessToken.addScope('bar');
};
```

Los scopes estarán disponibles en el token después de que se ejecute la Action.

<Warning>
  Auth0 devuelve información del perfil en un formato estructurado de claims, tal como se define en la [especificación de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Esto significa que las claims personalizadas que se añadan a los tokens de ID o a los tokens de acceso deben [ajustarse a las directrices y restricciones](/docs/es/secure/tokens/json-web-tokens/create-custom-claims) para evitar posibles colisiones.
</Warning>

<div id="configure-realm-support">
  ### Configurar el soporte para realm
</div>

Auth0 proporciona una concesión de extensión que ofrece una funcionalidad similar a la concesión de contraseña del propietario del recurso, pero te permite mantener directorios de usuarios separados (que se asignan a conexiones independientes) y especificar cuál usar durante el flujo.

Para usar esta variación, debes:

* Establecer el parámetro de solicitud `grant_type` en `http://auth0.com/oauth/grant-type/password-realm`.
* Enviar un parámetro de solicitud adicional llamado `realm` y asignarle el nombre del realm al que pertenece el usuario. Por ejemplo, si has configurado una conexión de base de datos para empleados internos llamada `employees` y tu usuario pertenece a ella, establece `realm` en `employees`.

<Card title="Conexiones como realms">
  Cualquier conexión que admita autenticación activa puede configurarse como un realm, incluidas las [conexiones de base de datos](/docs/es/get-started/applications/set-up-database-connections), las [conexiones passwordless](/docs/es/authenticate/passwordless) y las conexiones empresariales de [AD/LDAP](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap), [ADFS](/docs/es/authenticate/identity-providers/enterprise-identity-providers/adfs) y [Azure Active Directory](/docs/es/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).
</Card>
