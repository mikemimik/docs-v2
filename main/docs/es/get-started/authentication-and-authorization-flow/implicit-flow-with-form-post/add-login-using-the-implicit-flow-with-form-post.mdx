Puedes añadir inicio de sesión a tu aplicación de página única (SPA) mediante el flujo implícito con Form Post. Para saber cómo funciona este flujo y por qué deberías usarlo, lee [flujo implícito con Form Post](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post).

Usa el flujo implícito con Form Post para casos de uso de solo inicio de sesión; si necesitas solicitar <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> al iniciar sesión al usuario para poder llamar a tu API, usa el flujo de código de autorización con PKCE. Para obtener más información, lee [flujo de código de autorización con Proof Key for Code Exchange (PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce)).

Para implementar el flujo implícito con Form Post, puedes usar los siguientes recursos:

* [Express OpenID Connect SDK](https://www.npmjs.com/package/express-openid-connect): La forma más sencilla de implementar el flujo, ya que se encargará de la mayor parte del trabajo. Si usas nuestro SDK de Javascript, asegúrate de implementar las mitigaciones adecuadas para tu arquitectura. Para obtener más información, lee [Referencia de Auth0.js](/docs/es/libraries/auth0js).
* [Authentication API](https://auth0.com/docs/api/authentication): Si prefieres crear tu propia solución, sigue leyendo para aprender a llamar directamente a nuestra API.

Tras iniciar sesión correctamente, tu aplicación tendrá acceso al [token de identidad](/docs/es/secure/tokens/id-tokens) del usuario. El <Tooltip tip="token de ID: credencial destinada al propio cliente, en lugar de al acceso a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> contendrá información básica del perfil del usuario.

<div id="prerequisites">
  ## Requisitos previos
</div>

Registra tu aplicación en Auth0. Para obtener más información, consulta [Registrar aplicaciones de página única](/docs/es/get-started/auth0-overview/create-applications/single-page-web-apps).

* Selecciona **Aplicación web de página única** como **Tipo de aplicación**.
* Agrega `{https://yourApp/callback}` como **URL de devolución de llamada permitida**.
* Asegúrate de que los **Tipos de concesión** de tu aplicación incluyan **Implicit**. Para obtener más información, consulta [Actualizar tipos de concesión](/docs/es/get-started/applications/update-grant-types).

<div id="authorize-user">
  ## Autorizar al usuario
</div>

Solicita la autorización del usuario y redirígelo de nuevo a tu aplicación. Para comenzar el flujo, tendrás que obtener la autorización del usuario. Este paso puede incluir uno o más de los siguientes procesos:

* Autenticar al usuario;
* Redirigir al usuario a un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> para encargarse de la autenticación;
* Comprobar si hay sesiones activas de <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SSO);
* Obtener el consentimiento del usuario para el nivel de permiso solicitado, a menos que ya lo haya otorgado anteriormente.

Para autorizar al usuario, tu aplicación debe redirigirlo a la URL de autorización.

<div id="authorization-url-example">
  ### Ejemplo de URL de autorización
</div>

export const codeExample1 = `https://{yourDomain}/authorize?
    response_type=YOUR_RESPONSE_TYPE&
    response_mode=form_post&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    scope=SCOPE&
    state=STATE&
    nonce=NONCE`;

<div id="parameters">
  ### Parámetros
</div>

| Parameter Name  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `response_type` | Indica el tipo de credencial que Auth0 devolverá (`code` o `token`). Para el flujo implícito, el valor puede ser `id_token`, `token` o `id_token token`. En concreto, `id_token` devuelve un token de ID y `token` devuelve un token de acceso.                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `response_mode` | Especifica el método con el que deben devolverse los parámetros de respuesta. Por motivos de seguridad, el valor debe ser `form_post`. En este modo, los parámetros de respuesta se codifican como valores de formulario HTML que se transmiten mediante el método HTTP POST y en el cuerpo con el formato `application/x-www-form-urlencoded`.                                                                                                                                                                                                                                                                                                                                                      |
| `client_id`     | El ID de cliente de su aplicación. Puede encontrar este valor en la [configuración de su aplicación](https://manage.auth0.com/#/applications/\{yourClientId}/settings).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| `redirect_uri`  | La URL a la que Auth0 redirigirá el navegador una vez que el usuario haya concedido la autorización. Debe especificar esta URL como una URL de devolución de llamada válida en la [Configuración de la aplicación](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).<br /><br />**Advertencia:** Según la [especificación OAuth 2.0](https://tools.ietf.org/html/rfc6749#section-3.1.2), Auth0 elimina todo lo que aparece después del hash y *no* admite ningún fragmento.                                                                                                                                                                                                         |
| `scope`         | Especifica los [scopes](/docs/es/get-started/apis/scopes) para los que desea solicitar autorización, que determinan qué claims (o atributos del usuario) quiere que se devuelvan. Deben separarse con un espacio. Puede solicitar cualquiera de los [scopes estándar de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) sobre los usuarios, como `profile` y `email`, [custom claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims#custom-claims) que sigan un [formato con espacio de nombres](/docs/es/secure/tokens/json-web-tokens/create-custom-claims), o cualquier scope compatible con la API de destino (por ejemplo, `read:contacts`). |
| `state`         | (recomendado) Una cadena alfanumérica arbitraria y opaca que su aplicación agrega a la solicitud inicial y que Auth0 incluye al redirigir de vuelta a su aplicación. Para ver cómo usar este valor para prevenir ataques de falsificación de solicitud entre sitio (CSRF), consulte [Mitigate CSRF Attacks With State Parameters](/docs/es/secure/attack-protection/state-parameters).                                                                                                                                                                                                                                                                                                                  |
| `nonce`         | (obligatorio para `response_type` que contiene `id_token token`; en caso contrario, recomendado) Una cadena aleatoria criptográficamente segura que su aplicación agrega a la solicitud inicial y que Auth0 incluye dentro del token de ID, [utilizada para prevenir ataques de repetición de tokens](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow).                                                                                                                                                                                                                                                    |
| `connection`    | (opcional) Obliga al usuario a iniciar sesión con una conexión específica. Por ejemplo, puede pasar un valor de `github` para enviar al usuario directamente a GitHub para iniciar sesión con su cuenta de GitHub. Si no se especifica, el usuario verá la pantalla de Auth0 Lock con todas las conexiones configuradas. Puede ver una lista de sus conexiones configuradas en la pestaña **Connections** de su aplicación.                                                                                                                                                                                                                                                                          |
| `organization`  | (opcional) ID de la organización que se usará al autenticar a un usuario. Si no se proporciona y su aplicación está configurada para **Display Organization Prompt**, el usuario podrá introducir el nombre de la organización al autenticarse.                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `invitation`    | (opcional) ID del ticket de invitación de la organización. Al [invitar a un miembro a una Organization](/docs/es/manage-users/organizations/configure-organizations/invite-members), su aplicación debe gestionar la aceptación de la invitación reenviando los pares clave-valor `invitation` y `organization` cuando el usuario acepte la invitación.                                                                                                                                                                                                                                                                                                                                                 |

Como ejemplo, el fragmento HTML de su URL de autorización al agregar login a su aplicación podría verse así:

export const codeExample2 = `<a href="https://{yourDomain}/authorize?
  response_type=id_token token&
  response_mode=form_post&
  client_id={yourClientId}&
  redirect_uri={https://yourApp/callback}&
  scope=read:tests&
  state=xyzABC123&
  nonce=eq...hPmz">
  Iniciar sesión
</a>`;

<div id="response">
  ### Respuesta
</div>

Si todo sale bien, recibirás una respuesta `HTTP 302`. Las credenciales solicitadas van codificadas en el cuerpo:

```json lines
HTTP/1.1 302 Found
Content-Type: application/x-www-form-urlencoded
id_token=eyJ...acA&
state=xyzABC123
```

Ten en cuenta que los valores devueltos dependen de lo que hayas solicitado como `response_type`.

| Tipo de respuesta  | Componentes                                                                |
| ------------------ | -------------------------------------------------------------------------- |
| id&#95;token       | token de identidad                                                         |
| token              | token de acceso (más los valores `expires_in` y `token_type`)              |
| id&#95;token token | token de identidad, token de acceso (más los valores `expires_in` y `token_type`) |

Auth0 también devolverá cualquier valor de state que hayas incluido en tu llamada a la URL de autorización.

<Warning>
  Valida tus tokens antes de guardarlos. Para saber cómo hacerlo, lee [Validar tokens de ID](/docs/es/secure/tokens/id-tokens/validate-id-tokens) y [Validar tokens de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).
</Warning>

Los tokens de identidad contienen información del usuario que debe decodificarse y extraerse.

<div id="basic-authentication-request">
  ### Solicitud básica de autenticación
</div>

Este ejemplo muestra la solicitud más básica que puedes realizar al autorizar al usuario en el paso 1. Muestra la pantalla de inicio de sesión de Auth0 y permite al usuario iniciar sesión con cualquiera de tus conexiones configuradas:

export const codeExample3 = `https://{yourDomain}/authorize?
    response_type=id_token&
    response_mode=form_post&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    nonce=NONCE`;

<AuthCodeBlock children={codeExample3} language="http" />

Esto devolverá un token de identidad, que puedes extraer de tu URL de redirección.

<div id="request-users-name-and-profile-picture">
  ### Solicitar el nombre y la foto de perfil del usuario
</div>

Además de la autenticación de usuario habitual, este ejemplo muestra cómo solicitar datos adicionales del usuario, como su nombre y su foto.

Para solicitar el nombre y la foto del usuario, debes añadir los `scopes` adecuados al autorizar al usuario:

export const codeExample4 = `https://{yourDomain}/authorize?
    response_type=id_token token&
    response_mode=form_post&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    scope=openid%20name%20picture&
    state=STATE&
    nonce=NONCE`;

<AuthCodeBlock children={codeExample4} language="http" />

Ahora, tu token de identidad contendrá los claims solicitados de nombre y foto. Cuando decodifiques el token de identidad, será similar a:

```json lines
{
  "name": "jerrie@...",
  "picture": "https://s.gravatar.com/avatar/6222081fd7dcea7dfb193788d138c457?s=480&r=pg&d=https%3A%2F%2Fcdn.auth0.com%2Favatars%2Fje.png",
  "iss": "https://auth0pnp.auth0.com/",
  "sub": "auth0|581...",
  "aud": "xvt...",
  "exp": 1478113129,
  "iat": 1478077129
}
```

<div id="request-user-log-in-with-github">
  ### Solicitar que el usuario inicie sesión con GitHub
</div>

Además de la autenticación de usuario habitual, este ejemplo muestra cómo dirigir a los usuarios directamente a un proveedor de identidad social, como GitHub. Para que este ejemplo funcione, debes ir a [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social) y configurar la conexión correspondiente. Obtén el nombre de la conexión en la pestaña **Settings**.

Para dirigir a los usuarios directamente a la pantalla de inicio de sesión de GitHub, debes pasar el parámetro `connection` y establecer su valor con el nombre de la conexión (en este caso, `github`) al autorizar al usuario:

export const codeExample5 = `https://{yourDomain}/authorize?
    response_type=id_token token&
    response_mode=form_post&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    scope=openid%20name%20picture&
    state=STATE&
    nonce=NONCE&
    connection=github`;

<AuthCodeBlock children={codeExample5} language="http" />

Ahora, tu token de ID contendrá un claim `sub` con el ID único del usuario devuelto por GitHub. Cuando decodifiques el token de ID, tendrá un aspecto similar a:

```json lines
{
  "name": "Jerrie Pelser",
  "nickname": "jerriep",
  "picture": "https://avatars.githubusercontent.com/u/1006420?v=3",
  "iss": "https://auth0pnp.auth0.com/",
  "sub": "github|100...",
  "aud": "xvt...",
  "exp": 1478114742,
  "iat": 1478078742
}
```
