description: Aprende a generar y validar de forma segura un nonce criptográfico para usarlo con el flujo implícito con Form Post.
title: Mitigar ataques de repetición al usar el flujo implícito
---

Para mitigar los ataques de repetición al usar el [flujo implícito con Form Post](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post), se debe enviar un <Tooltip tip="Nonce: Número arbitrario emitido una sola vez en un protocolo de autenticación para detectar y prevenir ataques de repetición." cta="Ver glosario" href="/docs/es/glossary?term=nonce">nonce</Tooltip> en las solicitudes de autenticación, [tal como exige la especificación de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#ImplicitAuthRequest).

La aplicación genera el nonce, lo envía como parámetro `nonce` en la cadena de consulta de la solicitud de autenticación y este se incluye en la respuesta del <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> de Auth0. Esto permite a las aplicaciones correlacionar la respuesta del token de ID de Auth0 con la solicitud de autenticación inicial.

Para obtener más información sobre dónde incluir el nonce, consulta [agregar el inicio de sesión mediante el flujo implícito con Form Post](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/add-login-using-the-implicit-flow-with-form-post).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [Auth0.js](/docs/es/libraries/auth0js) administra por ti los parámetros `state` y `nonce` cuando usas la [autenticación de origen cruzado](/docs/es/authenticate/login/cross-origin-authentication).
</Callout>

<div id="generate-a-cryptographically-random-nonce">
  ## Genera un nonce aleatorio criptográficamente seguro
</div>

Una forma de generar un nonce aleatorio criptográficamente seguro es usar una herramienta como [Nano ID](https://github.com/ai/nanoid) o similar. Sin embargo, esto requiere incluir la herramienta junto con tu código de JavaScript. Si eso no es posible, puedes aprovechar que los [navegadores modernos](http://caniuse.com/#feat=cryptography) pueden usar la [Web Crypto API](https://www.w3.org/TR/WebCryptoAPI/) para generar cadenas aleatorias criptográficamente seguras que sirvan como nonces.

```javascript lines
function randomString(length) {
    var charset = '0123456789ABCDEFGHIJKLMNOPQRSTUVXYZabcdefghijklmnopqrstuvwxyz+/'
    result = ''

    while (length > 0) {
        var bytes = new Uint8Array(16);
        var random = window.crypto.getRandomValues(bytes);

        random.forEach(function(c) {
            if (length == 0) {
                return;
            }
            if (c < charset.length) {
                result += charset[c];
                length--;
            }
        });
    }
    return result;
}
```

<div id="persist-nonces-across-requests">
  ## Persistir nonces entre solicitudes
</div>

El nonce generado debe almacenarse en tu aplicación web mediante cualquiera de los siguientes métodos:

* `HttpOnly` <Tooltip tip="Cookie de sesión: elemento que, cuando está presente, permite considerar autenticado al usuario." cta="Ver glosario" href="/docs/es/glossary?term=session+cookie">cookie de sesión</Tooltip>
* valor de almacenamiento local de HTML5

Por ejemplo:

```js lines
window.localStorage.setItem('nonce', randomString(16));
```

<div id="validate-id-token">
  ## Validar token de identidad
</div>

Una vez que Auth0 devuelva un [token de ID](/docs/es/secure/tokens/id-tokens), este token debe [validarse y decodificarse](/docs/es/secure/tokens/id-tokens/validate-id-tokens) como es habitual.
Su claim `nonce` debe contener exactamente el mismo valor que se envió en la solicitud.
De lo contrario, la aplicación debe rechazar la autenticación.

```javascript lines
var jwt = '...'; // cuerpo del token de ID validado y decodificado
if (jwt.nonce === window.localStorage.getItem('nonce')) {
    // El nonce es correcto
} else {
    // ¡El nonce no es correcto! Es posible que haya un ataque de repetición de token en curso
}
```
