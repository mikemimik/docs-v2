Puedes agregar el inicio de sesión a tu aplicación web tradicional mediante el flujo de código de autorización. Para saber cómo funciona el flujo y por qué deberías usarlo, lee [Authorization Code Flow](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow). Para llamar a tu API desde una aplicación web tradicional, lee [Call Your API Using the Authorization Code Flow](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/call-your-api-using-the-authorization-code-flow).

Para implementar el flujo de código de autorización, Auth0 proporciona los siguientes recursos:

* [Regular Web App Quickstarts](/docs/es/quickstart/webapp): La forma más sencilla de implementar el flujo.
* [Authentication API](https://auth0.com/docs/api/authentication): Si prefieres crear tu propia solución, sigue leyendo para aprender a llamar directamente a nuestra API.

Después de un inicio de sesión exitoso, tu aplicación tendrá acceso al <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip> y al <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> del usuario. El token de identidad contiene información básica del perfil del usuario, y el token de acceso puede usarse para llamar al endpoint `/userinfo` de Auth0 o a tus propias API protegidas. Para obtener más información sobre los tokens de identidad, lee [ID Tokens](/docs/es/secure/tokens/id-tokens). Para obtener más información sobre los tokens de acceso, lee [Access Tokens](/docs/es/secure/tokens/access-tokens).

Solicitarás la autorización del usuario y te redirigirán de regreso a tu aplicación con un `authorization_code`. Luego intercambiarás el código por tokens.

<div id="prerequisites">
  ## Requisitos previos
</div>

Registra tu aplicación en Auth0. Para obtener más información, consulta [Registrar aplicaciones web clásicas](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps).

* Selecciona **Regular Web Application** como **Tipo de aplicación**.
* Agrega una **URL de devolución de llamada permitida**: `{https://yourApp/callback}`.
* Asegúrate de que los **Tipos de concesión** de tu aplicación incluyan **Código de autorización**. Para obtener más información, consulta [Actualizar tipos de concesión](/docs/es/get-started/applications/update-grant-types).

<div id="authorize-user">
  ## Autorizar al usuario
</div>

Para iniciar el flujo, tendrás que obtener la autorización del usuario. Este paso puede incluir uno o más de los siguientes procesos:

* Autenticar al usuario;
* Redirigir al usuario a un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> para encargarse de la autenticación;
* Obtener el consentimiento del usuario para el nivel de permiso solicitado, a menos que ya lo haya otorgado previamente.

Para autorizar al usuario, tu aplicación debe dirigirlo a la [URL de autorización](https://auth0.com/docs/api/authentication#authorization-code-flow).

<div id="authorization-url-example">
  ### Ejemplo de URL de autorización
</div>

export const codeExample1 = `https://{yourDomain}/authorize?
    response_type=code&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    scope={scope}&
    state={state}`;

<div id="parameters">
  ### Parámetros
</div>

| Parameter Name  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `response_type` | Indica el tipo de credencial que devolverá Auth0 (`code` o `token`). Para este flujo, el valor debe ser `code`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `client_id`     | El ID de cliente de tu aplicación. Puedes encontrar este valor en [Application Settings](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `redirect_uri`  | La URL a la que Auth0 redirigirá el navegador una vez que el usuario haya concedido la autorización. El código de autorización estará disponible en el parámetro de URL `code`. Debes especificar esta URL como una URL de devolución de llamada válida en [Application Settings](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).<br /><br />**Advertencia:** Según la [especificación OAuth 2.0](https://tools.ietf.org/html/rfc6749#section-3.1.2), Auth0 elimina todo lo que aparece después del hash y *no* tiene en cuenta ningún fragmento.                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `scope`         | Especifica los [scopes](/docs/es/get-started/apis/scopes) para los que quieres solicitar autorización, que determinan qué claims (o atributos de usuario) quieres que se devuelvan. Deben separarse con un espacio. Para obtener un token de ID en la respuesta, debes especificar al menos el scope `openid`. Si quieres que se devuelva el perfil completo del usuario, puedes solicitar `openid profile`. Puedes solicitar cualquiera de los [scopes estándar de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) sobre usuarios, como `email`, o [custom claims](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims#custom-claims) que sigan un [formato con espacio de nombres](/docs/es/secure/tokens/json-web-tokens/create-custom-claims). Incluye `offline_access` para obtener un [token de actualización](/docs/es/glossary?term=Refresh+Token) (asegúrate de que el campo **Allow Offline Access** esté habilitado en [Application Settings](https://manage.auth0.com/#/applications)). |
| `state`         | (recomendado) Una cadena alfanumérica arbitraria y opaca que tu aplicación añade a la solicitud inicial y que Auth0 incluye al redirigir de vuelta a tu aplicación. Para ver cómo usar este valor para prevenir ataques CSRF, consulta [Mitigate CSRF Attacks With State Parameters](/docs/es/secure/attack-protection/state-parameters).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `connection`    | (opcional) Obliga al usuario a iniciar sesión con una conexión específica. Por ejemplo, puedes pasar un valor de `github` para enviar al usuario directamente a GitHub e iniciar sesión con su cuenta de GitHub. Cuando no se especifica, el usuario ve la pantalla de Auth0 Lock con todas las conexiones configuradas. Puedes ver una lista de tus conexiones configuradas en la pestaña **Connections** de tu aplicación.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `organization`  | (opcional) ID de la organización que se usará al autenticar a un usuario. Si no se proporciona y tu aplicación está configurada para **Display Organization Prompt**, el usuario podrá introducir el nombre de la organización al autenticarse.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `invitation`    | (opcional) ID del ticket de invitación a la organización. Al [invitar a un miembro a una organización](/docs/es/manage-users/organizations/configure-organizations/invite-members), tu aplicación debe gestionar la aceptación de la invitación reenviando los pares clave-valor `invitation` y `organization` cuando el usuario la acepte.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `login_hint`    | (opcional) Rellena el campo de nombre de usuario/correo electrónico de la página de inicio de sesión o registro al redirigir a Auth0. Compatible con la experiencia de Universal Login.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |

Como ejemplo, tu fragmento HTML para la URL de autorización al añadir el inicio de sesión a tu aplicación podría verse así:

export const codeExample2 = `<a href="https://{yourDomain}/authorize?
  response_type=code&
  client_id={yourClientId}&
  redirect_uri={https://yourApp/callback}&
  scope=openid%20profile&
  state=xyzABC123">
  Iniciar sesión
</a>`;

<div id="response">
  ### Respuesta
</div>

Si todo va bien, recibirás una respuesta `HTTP 302`. El código de autorización se incluye al final de la URL:

```http lines
HTTP/1.1 302 Found
Location: {https://yourApp/callback}?code={authorizationCode}&state=xyzABC123
```

<div id="request-tokens">
  ## Solicitar tokens
</div>

Ahora que ya tienes un código de autorización, debes canjearlo por tokens. Para ello, usa el código de autorización (`code`) extraído en el paso anterior y haz una solicitud `POST` a la [URL del token](https://auth0.com/docs/api/authentication#authorization-code).

<div id="parameters">
  ### Parámetros
</div>

| Nombre del parámetro | Descripción                                                                                                                                                                                                                                                                                                                              |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `grant_type`         | Establece este valor en `authorization_code`.                                                                                                                                                                                                                                                                                            |
| `code`               | El `authorization_code` obtenido en el paso anterior de este tutorial.                                                                                                                                                                                                                                                                   |
| `client_id`          | El ID de cliente de tu aplicación. Puedes encontrar este valor en [Application Settings](https://manage.auth0.com/#/Applications/\{yourClientId}/settings).                                                                                                                                                                              |
| `client_secret`      | El secreto de cliente de tu aplicación. Puedes encontrar este valor en [Application Settings](https://manage.auth0.com/#/Applications/\{yourClientId}/settings). Para obtener más información sobre los métodos de autenticación de la aplicación disponibles, consulta [Application Credentials](/docs/es/secure/application-credentials). |
| `redirect_uri`       | La URL de devolución de llamada válida configurada en Application Settings de tu aplicación. Debe coincidir exactamente con el valor de `redirect_uri` enviado a la URL de autorización en el paso anterior de este tutorial. Ten en cuenta que debe estar codificada como URL.                                                          |

<div id="response">
  ### Respuesta
</div>

Si todo sale bien, recibirás una respuesta HTTP 200 con un payload que incluye los valores `access_token`, `refresh_token`, `id_token` y `token_type`:

```json lines
{
  "access_token": "eyJz93a...k4laUWw",
  "refresh_token": "GEbRxBN...edjnXbL",
  "id_token": "eyJ0XAi...4faeEoQ",
  "token_type": "Bearer"
}
```

<Warning>
  Valida tus tokens antes de guardarlos. Para saber cómo hacerlo, consulta [Validar tokens de identidad](/docs/es/secure/tokens/id-tokens/validate-id-tokens) y [Validar tokens de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).
</Warning>

[Los tokens de identidad](/docs/es/secure/tokens/id-tokens) contienen información del usuario que debe decodificarse y extraerse.

[Los tokens de acceso](/docs/es/secure/tokens/access-tokens) se usan para llamar al [endpoint /userinfo de Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-user-info) o a otra API. Si llamas a tu propia API, lo primero que deberá hacer es [validar el token de acceso](/docs/es/secure/tokens/access-tokens/validate-access-tokens).

[Los tokens de actualización](/docs/es/secure/tokens/refresh-tokens) se usan para obtener un nuevo token de acceso o token de identidad después de que el anterior haya caducado. El `refresh_token` solo estará presente en la respuesta si incluiste el scope `offline_access` y habilitaste **Allow Offline Access** para tu API en el Dashboard.

<Warning>
  Los tokens de actualización deben almacenarse de forma segura, ya que permiten que un usuario siga autenticado prácticamente de forma indefinida.
</Warning>

<div id="basic-authentication-request">
  ### Solicitud básica de autenticación
</div>

Este ejemplo muestra la solicitud más sencilla que puedes hacer para autorizar al usuario en el paso 1. Muestra la pantalla de inicio de sesión de Auth0 y permite al usuario iniciar sesión con cualquiera de tus conexiones configuradas:

export const codeExample13 = `https://{yourDomain}/authorize?
    response_type=code&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    scope=openid`;

<AuthCodeBlock children={codeExample13} language="http" />

Ahora, cuando solicites tokens, tu token de identidad contendrá los claims más básicos. Cuando decodifiques el token de identidad, tendrá un aspecto similar a:

```json lines
{
  "iss": "https://auth0pnp.auth0.com/",
  "sub": "auth0|581...",
  "aud": "xvt9...",
  "exp": 1478112929,
  "iat": 1478076929
}
```

<div id="request-users-name-and-profile-picture">
  ### Solicitar el nombre y la foto de perfil del usuario
</div>

Además de la autenticación de usuario habitual, este ejemplo muestra cómo solicitar datos adicionales del usuario, como el nombre y la foto.

Para solicitar el nombre y la foto del usuario, debes agregar los scopes adecuados al autorizar al usuario:

export const codeExample14 = `https://{yourDomain}/authorize?
    response_type=code&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    scope=openid%20name%20picture&
    state={state}`;

<AuthCodeBlock children={codeExample14} language="http" />

Ahora, al solicitar tokens, tu token de identidad contendrá los claims solicitados de nombre y foto. Cuando decodifiques el token de identidad, tendrá un aspecto similar a este:

```json lines
{
  "name": "jerrie@...",
  "picture": "https://s.gravatar.com/avatar/6222081fd7dcea7dfb193788d138c457?s=480&r=pg&d=https%3A%2F%2Fcdn.auth0.com%2Favatars%2Fje.png",
  "iss": "https://auth0pnp.auth0.com/",
  "sub": "auth0|581...",
  "aud": "xvt...",
  "exp": 1478113129,
  "iat": 1478077129
}
```

<div id="request-user-log-in-with-github">
  ### Solicitar el inicio de sesión del usuario con GitHub
</div>

Además de la autenticación de usuario habitual, este ejemplo muestra cómo redirigir a los usuarios directamente a un proveedor de identidad social, como GitHub. Primero deberás configurar la conexión adecuada en [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social) y obtener el nombre de la conexión en la pestaña **Settings**.

Para enviar a los usuarios directamente a la pantalla de inicio de sesión de GitHub, debes pasar el parámetro `connection` y establecer su valor como el nombre de la conexión (en este caso, `github`) al autorizar al usuario en el paso 1:

export const codeExample15 = `https://{yourDomain}/authorize?
    response_type=code&
    client_id={yourClientId}&
    redirect_uri={https://yourApp/callback}&
    scope=openid%20name%20picture&
    state={state}&
    connection=github`;

<AuthCodeBlock children={codeExample15} language="http" />

Ahora, cuando solicites tokens, tu token de identidad incluirá un claim `sub` con el ID único del usuario que devuelve GitHub. Al decodificar el token de identidad, se verá así:

```json lines
{
  "name": "Jerrie Pelser",
  "nickname": "jerriep",
  "picture": "https://avatars.githubusercontent.com/u/1006420?v=3",
  "iss": "https://auth0pnp.auth0.com/",
  "sub": "github|100...",
  "aud": "xvt...",
  "exp": 1478114742,
  "iat": 1478078742
}
```
