[Solicitud de autorización protegida con JWT (JAR)](https://datatracker.ietf.org/doc/html/rfc9101) es una extensión del protocolo OAuth 2.0 que protege la integridad y autenticidad de los parámetros de la solicitud de autorización. Al incluir los parámetros en un [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens) firmado, evitas que intermediarios manipulen o vean datos confidenciales de la solicitud.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de usar solicitud de autorización protegida con JWT, debes:

1. [Generar un par de claves RSA](/docs/es/secure/application-credentials/generate-rsa-key-pair)
2. Registrar la clave pública cargándola en el Auth0 Dashboard, como se describe en [Configurar solicitudes de autorización protegidas con JWT](/docs/es/get-started/applications/configure-jar)

<div id="how-it-works">
  ## Cómo funciona
</div>

En lugar de pasar parámetros como `scope` o `redirect_uri` como texto sin formato en una URL, la aplicación cliente los incluye en un [JSON Web Token (JWT)](/docs/es/secure/tokens/json-web-tokens) firmado como objeto de solicitud:

* Firma: La aplicación cliente firma el JWT con su clave privada.
* Verificación: El servidor de autorización de Auth0 recibe el JWT y verifica la firma con la clave pública que registraste.
* Procesamiento: Si es válido, el servidor de autorización de Auth0 extrae los parámetros. Si un parámetro aparece tanto en la solicitud de autorización protegida con JWT como en la cadena de consulta, prevalece el valor incluido en la solicitud de autorización protegida con JWT.

<div id="generate-the-jar-request">
  ## Genera la solicitud de autorización protegida con JWT
</div>

Usa la [biblioteca JWT de Auth0](https://jwt.io/libraries) para generar un <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> en el lenguaje que prefieras.

<div id="header">
  ### Encabezado
</div>

El encabezado del JWT le indica a Auth0 qué clave y qué algoritmo debe usar para verificarlo. Debe tener los siguientes parámetros:

* `alg`: El algoritmo usado para firmar el JWT. Debe ser RS256, RS384 o PS256.
* `typ`: El tipo de JWT. Debe ser `jwt` o `oauth-authz-req+jwt`.

El encabezado también puede incluir un campo `kid` que identifica la clave usada para firmar el JWT. Si hay un `kid`, Auth0 buscará una clave pública registrada durante la [configuración de la solicitud de autorización protegida con JWT](/docs/es/get-started/applications/configure-jar) cuyo ID de la clave coincida, y usará esa clave para verificar la firma del JWT.

<div id="payload">
  ### Payload
</div>

El payload contiene los parámetros de autorización. Debe contener los siguientes claims:

* `iss`: Debe contener el `client_id` de tu aplicación
* `aud`: Debe ser el dominio de tu tenant, con el protocolo y una barra diagonal final. Por ejemplo, `https://{YOUR_DOMAIN}.auth0.com/`

El JWT también debe contener cualquier parámetro obligatorio para la llamada a `/authorize`. Por ejemplo:

* `client_id`: También debe contener el `client_id` de tu aplicación
* `response_type`: Indica a Auth0 qué flujo de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip> quieres ejecutar. Usa `code` para el flujo de concesión Authorization Code.

El JWT puede contener cualquiera de los parámetros opcionales del <Tooltip tip="Flujo de autorización: Concesión de autorización (o flujo de trabajo) especificada en el marco de OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip> solicitado, como `audience`, `scope`, `state`, `redirect_uri`, entre otros.

Además, el JWT puede contener los siguientes claims opcionales:

* `iat`: Debe ser una fecha numérica.
* `nbf`: Debe ser una fecha numérica que represente un momento del pasado.
* `exp`: Debe ser una fecha numérica que represente un momento en el futuro.
* `jti`: Debe ser una cadena de no más de 64 bytes.

<div id="code-sample-generate-and-sign-jar">
  ## Ejemplo de código: Generar y firmar una solicitud de autorización protegida con JWT
</div>

El siguiente ejemplo de Node.js utiliza la [biblioteca jsonwebtoken](https://www.npmjs.com/package/jsonwebtoken?activeTab=readme) para generar y firmar una solicitud de autorización protegida con JWT:

```javascript lines expandable
const jwt = require('jsonwebtoken');
const crypto = require("crypto");
const fs = require('fs');

const privateKey = fs.readFileSync('{PATH_TO_YOUR_PEM_FILE}');
const client_id = '{YOUR_CLIENT_ID}';
const nonce = crypto.randomBytes(16).toString('hex');

const requestObject = jwt.sign(
{
  iss: client_id,    
  aud: 'https://your_tenant.auth0.com/', // your tenant's domain
  client_id,
  response_type: "code",
  scope: "openid profile",
  redirect_uri : "https://myapp.com/callback", // URL de devolución de llamada de tu aplicación
  nonce
},
privateKey,
{
  keyid: '{YOUR_KID}', // optional key id (kid) value from your public key
  algorithm: 'RS256',
  header: {
    typ: 'oauth-authz-req+jwt',
  },
});

console.log(requestObject);
```

<div id="call-the-authorization-endpoint">
  ## Llamar al endpoint de autorización
</div>

Puedes enviar la solicitud de autorización protegida con JWT al servidor de autorización de Auth0 de las siguientes maneras:

1. [Solicitud de autorización protegida con JWT estándar](#standard-jar-request): Incluye el JWT firmado como una cadena codificada para URL en el parámetro de solicitud.
2. [Pushed Authorization Request](#pushed-authorization-request): Para mayor seguridad y para evitar las limitaciones de longitud de la URL, usa PAR.

<div id="standard-jar-request">
  ### Solicitud estándar de autorización protegida con JWT
</div>

Para llamar al endpoint `/authorize` mediante una solicitud estándar de autorización protegida con JWT:

1. Abra una nueva ventana del navegador.
2. Pase su <Tooltip tip="ID de cliente: valor de identificación asignado a su recurso registrado en Auth0." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip> como parámetro `client_id` y el JWT firmado y codificado en URL como parámetro `request`.

```bash lines
# MacOS
open "https://{YOUR_DOMAIN}.auth0.com/authorize?client_id={YOUR_CLIENT_ID}&request={URL_ENCODED_JWT}"

# Windows
explorer "https://{YOUR_DOMAIN}.auth0.com/authorize?client_id={YOUR_CLIENT_ID}&request={URL_ENCODED_JWT}"
```

<div id="pushed-authorization-request">
  ### Pushed Authorization Request
</div>

Para llamar al endpoint `/authorize` con una solicitud de autorización enviada con PAR:

1. Envía la solicitud de autorización protegida con JWT al endpoint `/oauth/par` mediante una solicitud `POST` por back-channel.
2. Auth0 devolverá un `request_uri`, que luego puedes usar para llamar al endpoint `/authorize`, como en un [flujo PAR normal](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/authorization-code-flow-with-par).

La siguiente solicitud de cURL usa PAR y JAR en conjunto:

```bash lines
curl --location 'https://{YOUR_DOMAIN}.auth0.com/oauth/par' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'client_id={YOUR_CLIENT_ID}' \
--data-urlencode 'client_secret={YOUR_CLIENT_SECRET}' \
--data-urlencode 'request={JWT}'
```
