title: Flujo híbrido
---

Las aplicaciones que pueden almacenar de forma segura [credenciales de cliente](/docs/es/secure/application-credentials) pueden beneficiarse del uso del flujo híbrido (definido en la [especificación de OpenID Connect, sección 3.3](https://openid.net/specs/openid-connect-core-1_0.html#HybridFlowAuth)). El flujo híbrido permite que tu aplicación tenga acceso inmediato a un <Tooltip tip="Token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+token">token de identidad</Tooltip>, al tiempo que garantiza la obtención segura y confiable de <Tooltip tip="Token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> y <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip>. Esto puede ser útil en situaciones en las que tu aplicación necesita acceder de inmediato a información sobre el usuario, pero debe realizar cierto procesamiento antes de poder acceder a recursos protegidos durante un período prolongado.

<div id="how-it-works">
  ## Cómo funciona
</div>

El Flujo híbrido combina pasos del [Flujo implícito con Form Post](/docs/es/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post) y del [Flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow):

1. El usuario selecciona **Login** en la aplicación.
2. La aplicación redirige al usuario al servidor de autorización de Auth0 ([endpoint `/authorize`](https://auth0.com/docs/api/authentication#authorization-code-flow)), y le pasa el parámetro `response_type`, que indica el tipo de credencial solicitada (token de identidad y código de autorización), así como el parámetro `response_mode` con el valor `form_post` para garantizar la seguridad.
3. El servidor de autorización de Auth0 redirige al usuario al prompt de inicio de sesión y autorización.
4. El usuario se autentica con una de las opciones de login configuradas y puede ver una solicitud de consentimiento con la lista de permisos que Auth0 otorgará a la aplicación.
5. El servidor de autorización de Auth0 redirige al usuario de vuelta a la aplicación con un código de autorización de un solo uso y un token de identidad, un token de acceso, o ambos, según el valor de `response_type`.
6. La aplicación envía el código de autorización, el ID de cliente de la aplicación y sus credenciales, como el secreto de cliente o clave privada JWT, al servidor de autorización de Auth0 ([endpoint `/oauth/token`](https://auth0.com/docs/api/authentication?http#authorization-code-flow43)).
7. El servidor de autorización de Auth0 verifica el código de autorización, el ID de cliente de la aplicación y sus credenciales.
8. El servidor de autorización de Auth0 responde con un segundo token de identidad y un token de acceso (y, opcionalmente, un token de actualización).
9. La aplicación puede usar el segundo token de acceso para llamar a una API y acceder a información sobre el usuario.
10. La API responde con los datos solicitados.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si tu aplicación solo necesita usar el Flujo híbrido para el inicio de sesión, no necesitarás un token de actualización ni un token de acceso, solo un token de ID con claims.
</Callout>
