<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Recomendamos encarecidamente usar Actions para ampliar Auth0. Con Actions, tiene acceso a información de tipos más completa, documentación integrada y paquetes `npm` públicos, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lea [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarle con la migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para acompañarle en el proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debe crear nuevas Rules o Hooks solo en su entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar sus Rules a Actions, lea [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar sus Hooks a Actions, lea [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).

Puede cambiar scopes y agregar claims personalizados a los tokens emitidos mediante el [flujo de credenciales de cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow) agregando [Hooks](/docs/es/customize/hooks).

Hooks le permite personalizar el comportamiento de Auth0 mediante código Node.js. Son funciones seguras y autocontenidas asociadas a puntos de extensibilidad específicos de la plataforma de Auth0 (como el flujo de credenciales de cliente). Auth0 invoca los Hooks en runtime para ejecutar su lógica personalizada.

Puede gestionar Hooks mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<Warning>
  Aunque puede crear varios hooks para un mismo punto de extensibilidad, cada punto de extensibilidad solo puede tener un hook habilitado a la vez. Cualquier hook adicional que cree para ese punto de extensibilidad se deshabilitará automáticamente, por lo que deberá habilitarlo de forma explícita. El hook habilitado se ejecutará para todas las applications y APIs.
</Warning>

<div id="create-a-hook">
  ### Crear un Hook
</div>

En este ejemplo, harás lo siguiente:

* agregar una claim arbitraria (`https://foo.com/claim`) al [token de acceso](/docs/es/secure/tokens/access-tokens)
* agregar un permiso adicional a la API que configuraste

Crea un Hook para personalizar el token. Cuando se te pida que elijas un punto de extensibilidad, selecciona `Intercambio de credenciales de cliente` y agrega el siguiente código en el editor:

```js lines
module.exports = function(client, scope, audience, context, cb) {
  var access_token = {};
  access_token['https://foo.com/claim'] = 'bar';
  access_token.scope = scope;
  access_token.scope.push('extra');
  cb(null, access_token);
};
```

<Warning>
  Auth0 devuelve información del perfil en un formato estructurado de claims, tal como se define en la [especificación de OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Esto significa que los custom claims añadidos a los tokens de ID o a los tokens de acceso deben [cumplir las directrices y restricciones](/docs/es/secure/tokens/json-web-tokens/create-custom-claims) para evitar posibles colisiones.
</Warning>

<div id="test-your-hook">
  ### Prueba tu Hook
</div>

Para probar el hook que acabas de crear, debes ejecutar un Intercambio de credenciales de cliente, obtener el <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip>, decodificarlo y revisar su contenido.

<div id="get-token">
  #### Obtener un token
</div>

Para obtener un token, haz una llamada `POST` al [endpoint del flujo de credenciales de cliente](https://auth0.com/docs/api/authentication#client-credentials-flow). Asegúrate de reemplazar los valores de marcador de posición CLIENT&#95;ID, CLIENT&#95;SECRET y API&#95;IDENTIFIER por el <Tooltip tip="ID de cliente: valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip> de tu aplicación, el <Tooltip tip="secreto de cliente: Secret utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo el cliente y el servidor de autorización deben conocerlo, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip> de tu aplicación y el identificador de tu API, respectivamente. Puedes encontrar el ID de cliente y el secreto de cliente en la configuración de tu [aplicación](https://manage.auth0.com/#/applications) y el identificador de la API en la configuración de tu [API](https://manage.auth0.com/#/apis).

<div id="decode-token">
  #### Decodificar el token
</div>

La forma más sencilla de decodificar el token de acceso y consultar su contenido es usar el [depurador de JWT.io](https://jwt.io/#debugger-io).

Copia tu token de acceso y pégalo en el editor. El <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> se decodifica automáticamente y se muestra su contenido.

Ten en cuenta que los dos últimos elementos del **Payload** los ha establecido tu hook:

* `"scope": "extra"`
* `"https://foo.com/claim": "bar"`
