title: Flujo de Custom Token Exchange
---

Custom Token Exchange permite a las aplicaciones intercambiar un token de seguridad existente, denominado `subject_token`, por tokens de acceso, ID y de actualización de Auth0 llamando al endpoint `/oauth/token`, de acuerdo con la [RFC 8693](https://datatracker.ietf.org/doc/html/rfc8693).

<div id="how-it-works">
  ## Cómo funciona
</div>

El flujo de Custom Token Exchange se rige por una única [Action de Custom Token Exchange](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger), que se selecciona específicamente en función del parámetro `subject_token_type` recibido, tal como se configura en un [perfil de Custom Token Exchange](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange).

<Frame>![](/docs/images/custom-token-exchange/cte_how_it_works_diagram.png)</Frame>

Como requisito previo del flujo, debes definir un [perfil de Custom Token Exchange](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-custom-token-exchange-profile) que asigne un `subject_token_type` específico a la Action que gestionará las solicitudes.

Cuando se activa el flujo de Custom Token Exchange:

1. La aplicación envía una solicitud `POST` al endpoint `/oauth/token` con el token de seguridad que se va a intercambiar (el `subject_token`) y su correspondiente `subject_token_type`. La solicitud también puede incluir opcionalmente un `actor_token` y un `actor_token_type` para identificar a la entidad que actúa en nombre del usuario.
2. Auth0 valida la solicitud del cliente y sus credenciales.
3. Se ejecuta la Action de Custom Token Exchange asociada. Tu código personalizado debe decodificar y validar el `subject_token`, aplicar las políticas de autorización necesarias y establecer el usuario de la transacción para aprobar el intercambio. La Action también puede establecer un actor para la transacción mediante `api.authentication.setActor()`, lo que da como resultado un claim `act` en los tokens emitidos que identifica quién está actuando en nombre del usuario.
4. El resto del pipeline estándar de Auth0 se ejecuta para generar un token de acceso de Auth0 y, opcionalmente, un token de identidad y un token de actualización para el usuario establecido.
5. La aplicación solicitante puede usar estos tokens recién emitidos para llamar de forma segura a las API en nombre del usuario establecido en la Action.

<div id="get-started">
  ## Primeros pasos
</div>

Para usar Custom Token Exchange, debe [crear un perfil de Custom Token Exchange](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-custom-token-exchange-profile). El flujo de trabajo general consta de tres pasos principales de configuración, seguidos de la llamada a su API:

1. [Cree e implemente una Action](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-action-for-the-profile) que contendrá la lógica para controlar el intercambio de tokens. En esta Action, escriba la lógica de código personalizada para:
   * Decodificar y validar de forma segura el `subject_token` entrante.
   * Realizar los pasos de autorización necesarios para la transacción.
   * Establecer el usuario correspondiente para la transacción mediante los métodos disponibles en el [objeto de API de Actions de Custom Token Exchange](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger/custom-token-exchange-api-object).
2. [Cree el perfil de Custom Token Exchange](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-custom-token-exchange-profile) en su tenant. El perfil establece un mapeo uno a uno entre un `subject_token_type` específico que usará en sus solicitudes y la Action que acaba de crear.
3. [Habilite Custom Token Exchange para su aplicación](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#enable-custom-token-exchange-for-your-application).

Una vez que haya creado el perfil de Custom Token Exchange, estará listo para:

1. Llamar a su API: ahora puede realizar una solicitud `POST` al endpoint `/oauth/token` con los parámetros necesarios, incluidos `subject_token` y el `subject_token_type` correspondiente al perfil configurado. Para obtener más información, consulte [Llame a su API mediante Custom Token Exchange](/docs/es/get-started/authentication-and-authorization-flow/token-exchange-flow/call-your-api-using-the-custom-token-exchange-flow).
