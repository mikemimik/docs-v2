title: Flujo de Custom Token Exchange
---

Custom Token Exchange permite a las aplicaciones intercambiar un token de seguridad existente —denominado `subject_token`— por tokens de acceso, ID y de actualización de Auth0 llamando al endpoint `/oauth/token`, de acuerdo con la [RFC 8693](https://datatracker.ietf.org/doc/html/rfc8693).

<div id="how-it-works">
  ## Cómo funciona
</div>

El flujo de Custom Token Exchange se rige por una única [Action de Custom Token Exchange](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger), que se selecciona específicamente en función del parámetro `subject_token_type` entrante, tal como se configura en un [Custom Token Exchange Profile](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange).

<Frame>![](/docs/images/custom-token-exchange/cte_how_it_works_diagram.png)</Frame>

Como requisito previo del flujo, debe definir un [Custom Token Exchange Profile](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-custom-token-exchange-profile) que asigne un `subject_token_type` específico a la Action que gestionará las solicitudes.

Cuando se activa el flujo de Custom Token Exchange:

1. La aplicación envía una solicitud `POST` al endpoint `/oauth/token` con el token de seguridad que se va a intercambiar (el `subject_token`) y el `subject_token_type` correspondiente. La solicitud también puede incluir, de forma opcional, un `actor_token` y un `actor_token_type` para identificar a una entidad que actúa en nombre del usuario.
2. Auth0 valida la solicitud del cliente y las credenciales.
3. Se ejecuta la Action de Custom Token Exchange asociada. Su código personalizado debe decodificar y validar el `subject_token`, aplicar las política de autorización necesarias y establecer el usuario de la transacción para aprobar el intercambio. La Action también puede establecer un actor para la transacción mediante `api.authentication.setActor()`, lo que da como resultado un `claim` `act` en los tokens emitidos que identifica quién está actuando en nombre del usuario.
4. A continuación, se ejecuta el resto del pipeline estándar de Auth0 para generar un Auth0 access token y, opcionalmente, un token de identidad y un token de actualización para el usuario establecido.
5. Después, la aplicación solicitante puede usar estos tokens recién emitidos para llamar de forma segura a las API en nombre del usuario establecido en la Action.

<div id="get-started">
  ## Primeros pasos
</div>

Para usar Custom Token Exchange, necesitas [crear un Custom Token Exchange Profile](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-custom-token-exchange-profile). El workflow general incluye tres pasos principales de configuración, seguidos de tu llamada a la API:

1. [Crea e implementa una Action](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-action-for-the-profile) que contendrá la lógica para controlar el intercambio de tokens. En esta Action, escribe la lógica de código personalizado para:
   * Decodificar y validar de forma segura el `subject_token` entrante.
   * Realizar cualquier paso de autorización necesario para la transacción.
   * Establecer el usuario correspondiente para la transacción mediante los métodos disponibles en el [objeto de API de Custom Token Exchange Actions API](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger/custom-token-exchange-api-object).
2. [Crea el Custom Token Exchange Profile](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#create-custom-token-exchange-profile) en tu tenant. El perfil establece un mapeo uno a uno entre un `subject_token_type` específico que usarás en tus solicitudes y la Action que acabas de crear.
3. [Habilita Custom Token Exchange para tu aplicación](/docs/es/authenticate/custom-token-exchange/configure-custom-token-exchange#enable-custom-token-exchange-for-your-application).

Una vez que hayas creado el Custom Token Exchange Profile, estarás listo para hacer lo siguiente:

1. Llamar a tu API: ahora puedes hacer una solicitud `POST` al endpoint `/oauth/token` con los parámetros necesarios, incluidos el `subject_token` y el `subject_token_type` correspondientes a tu perfil configurado. Para obtener más información, consulta [Llama a tu API usando Custom Token Exchange](/docs/es/get-started/authentication-and-authorization-flow/token-exchange-flow/call-your-api-using-the-custom-token-exchange-flow).
