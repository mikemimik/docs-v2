Para usar Custom Token Exchange, realiza una solicitud `POST` al endpoint `/oauth/token` con los siguientes parámetros:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Recuerda que el token de sujeto y el token de actor usados con Custom Token Exchange pueden tener cualquier formato o tipo, siempre que el código de tu Action pueda interpretarlos. Debes implementar una validación robusta de los tokens que recibes y aceptas. Si no lo haces, te expones a distintos vectores de ataque, como la suplantación o los ataques de repetición, lo que puede permitir que actores maliciosos se autentiquen con el ID de usuario de otra persona o actúen en su nombre sin autorización.

  Cada `subject_token_type` se asigna a un Custom Token Exchange Profile y está asociado a una Action que se ejecutará para controlar esa transacción.
</Callout>

| Parámetro            | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `grant_type`         | Para Custom Token Exchange, usa `urn:ietf:params:oauth:grant-type:token-exchange`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| `subject_token_type` | El tipo del token de sujeto. Para Custom Token Exchange, puede ser cualquier URI dentro de un espacio de nombres de tu propiedad, como `http://acme.com/legacy-token` o `urn:acme:legacy-token`.<br /><br />Los siguientes espacios de nombres están reservados y no se pueden usar:<ul><li>`http://auth0.com`</li><li>`https://auth0.com`</li><li>`http://okta.com`</li><li>`https://okta.com`</li><li>`urn:ietf`</li><li>`urn:auth0`</li><li>`urn:okta`</li></ul>                                                                                                                                                                                           |
| `subject_token`      | El token de sujeto, que tu Action debe validar y usar para identificar al usuario.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| `client_id`          | El ID de cliente de la aplicación que estás usando para el intercambio de tokens. Al igual que con otros tipos de concesión, también puedes pasar el ID de cliente en el encabezado Authorization mediante HTTP Basic Auth.                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `client_secret`      | El secreto de cliente de la aplicación que estás usando para el intercambio de tokens. Al igual que con otros tipos de concesión, también puedes pasar el secreto de cliente en el encabezado Authorization mediante HTTP Basic Auth.<br /><br />También hay otras alternativas disponibles, como se explica en la [documentación de referencia de Auth0 Authentication API](https://auth0.com/docs/api/authentication#authentication-methods).<br /><br />Ten en cuenta que Custom Token Exchange puede ser usado por aplicaciones públicas. En ese caso, asegúrate de leer [Attack Protection](#attack-protection).                                         |
| `audience`           | El identificador de API definido en Auth0. Si no está presente, se usará la audience predeterminada del tenant, según se configura en [Configuración del tenant](/docs/es/get-started/tenant-settings).                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `scope`              | (Opcional) El parámetro scope de OAuth2.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `organization`       | (Opcional) El identificador de la organización con el que quieres que se asocie la solicitud. Como alternativa, puedes especificar un nombre de organización si [Use Organization Names in Authentication API](/docs/es/manage-users/organizations/configure-organizations/use-org-name-authentication-api) está permitido para tu tenant. Para obtener más información sobre cómo se procesa la solicitud, consulta el [método de API api.authentication.setOrganization()](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger/custom-token-exchange-api-object#api-authentication-setorganization-organization). |
| `actor_token`        | (Opcional) Un token que identifica al actor o al principal que realiza la delegación en nombre del usuario sujeto. Debe proporcionarse junto con `actor_token_type`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `actor_token_type`   | (Opcional) El tipo del token de actor. Debe proporcionarse junto con `actor_token`. Para los tokens de ID de Auth0, usa `urn:ietf:params:oauth:token-type:id_token` para la validación automática del lado del servidor. Para otros valores, sigue las mismas restricciones de espacio de nombres que `subject_token_type`.                                                                                                                                                                                                                                                                                                                                   |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cuando se establece un actor para la transacción, no se emiten tokens de actualización, independientemente de los scopes solicitados. El scope `offline_access` se excluye de la respuesta. Esto es intencional: los tokens delegados no deben durar más que el intercambio inmediato, lo que garantiza que cada solicitud de acceso posterior pase por la lógica de autorización de la Action de CTE para volver a validar la delegación.

  Tu Action de CTE es responsable de validar de forma segura el `actor_token`, aplicando el mismo rigor (por ejemplo, la verificación de firma criptográfica) que aplicarías al `subject_token`. Cuando `actor_token_type` es `urn:ietf:params:oauth:token-type:id_token`, Auth0 realiza esta validación automáticamente.
</Callout>

Los demás parámetros de la extensión que no se enumeran arriba se incluyen en `event.request.body` de la Action correspondiente.

<div id="sample-request">
  ## Solicitud de ejemplo
</div>

```bash lines
curl --location 'https://{yourDomain}/oauth/token' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'grant_type=urn:ietf:params:oauth:grant-type:token-exchange' \
--data-urlencode 'audience=https://api.acme.com' \
--data-urlencode 'scopes=openid offline_access acme-scope1 acme-scope2' \
--data-urlencode 'subject_token_type=urn:acme:external-idp-migration' \
--data-urlencode 'subject_token=t8e7S2D9trQm73e .... iqBR3GjxDtbDVjpfQU' \
--data-urlencode 'client_id=<YOUR_CLIENT_ID>' \
--data-urlencode 'client_secret=<YOUR_CLIENT_SECRET>'
--data-urlencode 'organization=periscope-acme'
```

<div id="sample-request-with-actor-token">
  ## Ejemplo de solicitud con token de actor
</div>

Al realizar un intercambio de autorización delegada, incluye `actor_token` y `actor_token_type`:

```bash lines
curl --location 'https://{yourDomain}/oauth/token' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'grant_type=urn:ietf:params:oauth:grant-type:token-exchange' \
--data-urlencode 'audience=https://api.acme.com' \
--data-urlencode 'scopes=openid acme-scope1 acme-scope2' \
--data-urlencode 'subject_token_type=urn:acme:legacy-token' \
--data-urlencode 'subject_token=t8e7S2D9trQm73e .... iqBR3GjxDtbDVjpfQU' \
--data-urlencode 'actor_token=eyJhbGciOiJSUzI1NiI .... kXdF9tZ3c' \
--data-urlencode 'actor_token_type=urn:ietf:params:oauth:token-type:id_token' \
--data-urlencode 'client_id=<YOUR_CLIENT_ID>' \
--data-urlencode 'client_secret=<YOUR_CLIENT_SECRET>'
```
