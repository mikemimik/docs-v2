<Card title="Antes de empezar">
  Debes crear una nueva aplicación en Auth0 Dashboard o convertir una aplicación existente antes de continuar. Para obtener más información, lee [Configurar la autenticación con clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt).
</Card>

Debes completar dos pasos al autenticarte con `private_key_jwt:`

1. Crea la aserción de cliente. Esta aserción es un JWT firmado con la clave privada del par de claves que generaste. Para saber cómo generar un par de claves, lee [Configurar la autenticación con clave privada JWT](/docs/es/get-started/applications/configure-private-key-jwt).
2. Usa la aserción para autenticarte ante Auth0.

<div id="build-the-assertion">
  ## Crea la aserción
</div>

Puedes usar uno de los SDK de Auth0 para crear una aserción automáticamente. Si no usas nuestros SDK, tendrás que crearla tú mismo.

La aserción es un <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, con frecuencia, también de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT) que debe contener las siguientes propiedades y claims:

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Todos los claims son obligatorios, salvo que se indique lo contrario. Para obtener más información sobre los claims de JWT, consulta [Claims de JSON Web Token](/docs/es/secure/tokens/json-web-tokens/json-web-token-claims).
</Callout>

* Encabezado

  * `alg`: El algoritmo usado para firmar la aserción. Debe coincidir con el algoritmo especificado cuando creaste la credencial de tu aplicación.
  * `kid`**:** (opcional) El `kid` de la credencial generado por Auth0. El `kid` se crea cuando se crea la credencial.
* Payload

  * `iss`**:** El ID de cliente de tu aplicación. Puedes encontrar este valor en la configuración de tu aplicación en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/dashboard/#/applications/) y seleccionando la pestaña **Settings**.
  * `sub`**:** El ID de cliente de tu aplicación. También puedes encontrar este valor en la configuración de tu aplicación, en [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/dashboard/#/applications/), seleccionando la pestaña **Settings**.
  * `aud`**:** La URL del tenant de Auth0 o del dominio personalizado que recibe la aserción. Por ejemplo: `https://{yourTenant}.auth0.com/`**.** Incluye la barra diagonal final.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si has configurado un dominio personalizado para tu tenant de Auth0, puedes usarlo como claim `aud`. En ese caso, te recomendamos usar el dominio personalizado.
    </Callout>
  * `iat` (opcional), `nbf` (opcional) y `exp`: claims Issued At, Not Before y Expiration configurados con las marcas de tiempo correctas. Se permite una desviación del reloj de hasta 10 segundos para `iat` y `nbf` (si están presentes) para facilitar la interoperabilidad. La aserción del cliente es un token de un solo uso, por lo que recomendamos el menor tiempo de expiración posible. Auth0 admite un máximo de 5 minutos de duración para un token.
  * `jti`: Un ID de claim único creado por el cliente. Recomendamos usar el formato Universally Unique Identifier (UUID).

    <Warning>
      Este JWT es un token de un solo uso y debe tratarse como tal, con una expiración breve. Recomendamos establecer un máximo de 1 minuto.
    </Warning>

A continuación, debes firmar el token con la clave privada que generaste cuando creaste o configuraste tu aplicación para la autenticación con clave privada JWT. Para saber cómo hacerlo, consulta la [especificación de JSON Web Token](https://www.rfc-editor.org/rfc/rfc7519#section-7.1).

Te recomendamos crear el token con herramientas estándar o bibliotecas de terceros que ya admitan esta funcionalidad, en lugar de implementarla tú mismo desde cero. Para obtener más información sobre las bibliotecas compatibles, consulta la lista en [JWT.io](https://jwt.io/libraries).

<div id="example">
  ### Ejemplo
</div>

En el siguiente ejemplo, el script de Node.js usa el [paquete jose](https://github.com/panva/jose) para generar la aserción:

```javascript lines expandable
const { SignJWT } = require('jose')
const crypto = require("crypto");
const uuid = require("uuid");

async function main() {
 const privateKeyPEM = crypto.createPrivateKey(/**
   Lee el contenido de tu clave privada aquí. Recomendamos almacenar tu clave privada
   en una infraestructura segura. 
 */);

 const jwt = await new SignJWT({})
   .setProtectedHeader({ 
      alg: 'RS256', // o RS384 o PS256
      kid: '(OPTIONAL) KID_GENERATED_BY_AUTH0' 
   })
   .setIssuedAt()
   .setIssuer('CLIENT_ID')
   .setSubject('CLIENT_ID')
   .setAudience('https://YOUR_TENANT.auth0.com/') // o tu CUSTOM_DOMAIN
   .setExpirationTime('1m')
   .setJti(uuid.v4())
   .sign(privateKeyPEM);
  console.log(jwt)
}

main();
```

Después de generar el JWT con la información requerida, ya puedes autenticar tu aplicación ante Auth0.

<div id="exchange-assertion-for-access-tokens">
  ## Intercambiar una aserción por tokens de acceso
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El siguiente ejemplo usa el [flujo de credenciales del cliente](/docs/es/get-started/authentication-and-authorization-flow/client-credentials-flow). La autenticación con clave privada JWT puede usarse con otros tipos de concesión que también permiten reemplazar `client_secret` por `client_assertion`.
</Callout>

Para intercambiar la aserción JWT por un <Tooltip tip="Token de acceso: credencial de autorización en forma de cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip>, llama al [endpoint de token](https://auth0.com/docs/api/authentication#authenticate-user) de la Authentication API con los siguientes parámetros:

* `$client_assertion`: aserción JWT
* `$resource_server_identifier`: identificador del <Tooltip tip="Servidor de recursos: servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/docs/es/glossary?term=resource+server">servidor de recursos</Tooltip>. Para obtener más información, consulta [Registrar APIs](/docs/es/get-started/auth0-overview/set-up-apis).

```bash lines
curl --location --request POST 'https://$tenant/oauth/token' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data-urlencode 'grant_type=client_credentials' \
  --data-urlencode 'client_assertion_type=urn:ietf:params:oauth:client-assertion-type:jwt-bearer' \
  --data-urlencode 'client_assertion=$client_assertion' \
  --data-urlencode 'audience=$resource_server_idenifier'
```

<div id="supported-endpoints">
  ## Endpoints compatibles
</div>

Además del endpoint [https://$tenant/oauth/token](https://auth0.com/docs/api/authentication#get-token), los siguientes endpoints de Auth0 Authentication API admiten la autenticación `private_key_jwt` para las aplicaciones configuradas:

* [POST /oauth/revoke](https://auth0.com/docs/api/authentication#revoke-refresh-token)
* [POST /mfa/challenge](https://auth0.com/docs/api/authentication#challenge-request)
* [POST /passwordless/start](https://auth0.com/docs/api/authentication#get-code-or-link)

<div id="assertion-limits">
  ## Límites de la aserción
</div>

La longitud máxima de la aserción JWT es de 2048 bytes.

Los claims de la aserción tienen los siguientes límites:

* `iss`: 64 caracteres
* `sub`: 64 caracteres
* `jti`: 64 caracteres
* `alg`: 16 caracteres
