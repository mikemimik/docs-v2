Las aplicaciones de IA generativa (GenAI) impulsadas por grandes modelos de lenguaje (LLM) y agentes de IA están cambiando la forma en que los usuarios interactúan con el software. A medida que los desarrolladores integran en sus aplicaciones capacidades de IA que gestionan el inicio de sesión, el acceso a datos y la ejecución de herramientas, proteger estas interacciones se vuelve fundamental.

Auth0 for AI Agents aprovecha la sólida plataforma de gestión de identidades y accesos (IAM) de Auth0 para proteger cada capa de tu stack de GenAI. Proporciona seguridad integral, de nivel empresarial, para:

* **Usuarios humanos:** Garantiza que solo las personas autorizadas puedan interactuar con los modelos y las aplicaciones de IA.
* **Agentes de IA:** Controla el acceso autónomo de los agentes de IA a APIs externas mediante scopes de <Tooltip tip="OAuth 2.0: framework de autorización que define protocolos y flujos de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>.
* **APIs y herramientas externas:** Conecta de forma segura los agentes de IA con Google, Slack, GitHub y mucho más. Auth0 renueva e intercambia tokens automáticamente sin intervención del usuario.
* **Datos sensibles:** Aplica permisos granulares de mínimo privilegio para proteger los datos sensibles del acceso no autorizado por parte de agentes de IA.

Para empezar a desarrollar con Auth0 for AI Agents, consulta la [documentación de Auth0 for AI Agents](https://auth0.com/ai/docs).

<div id="use-cases">
  ## Casos de uso
</div>

Auth0 for AI Agents permite a los desarrolladores crear e implementar aplicaciones seguras impulsadas por IA con:

* **Integraciones fluidas:** Se integra de forma nativa con frameworks de IA populares como LangChain, LlamaIndex y el Vercel AI SDK.
* **Controles de seguridad integrales:** Protege los flujos de autenticación con OAuth 2.0 y <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect, y gestiona el acceso tanto de usuarios humanos como de agentes de IA.
* **Autorización asíncrona:** Permite controles con intervención humana para acciones críticas.
* **<Tooltip tip="Autorización detallada (FGA): producto de Auth0 que permite a usuarios individuales acceder a objetos o recursos específicos." cta="Ver glosario" href="/docs/es/glossary?term=Fine-grained+authorization">Autorización detallada</Tooltip> para RAG:** Aplica un control de acceso granular dentro del flujo de Retrieval Augmented Generation (RAG).

<div id="user-authentication">
  ### Autenticación de usuario
</div>

Usa OAuth 2.0 y OpenID Connect para autenticar de forma segura a los usuarios con [Universal Login](/docs/es/authenticate/login/auth0-universal-login). Con [Client-Initiated Account Linking](https://auth0.com/ai/docs/client-initiated-account-linking), los agentes de IA pueden solicitar acceso y conectarse a múltiples <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> en nombre del usuario mediante Auth0.

<div id="call-apis-on-users-behalf">
  ### Llamar a APIs en nombre de los usuarios
</div>

Con [Token Vault](/docs/es/secure/tokens/token-vault), los agentes de IA pueden obtener <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> de proveedores de identidad externos para llamar a sus APIs, todo a través de una sola integración de Auth0. Por ejemplo, puedes usar Token Vault para:

1. Autenticar a un usuario con un proveedor de identidad externo como Google.
2. Usar scopes de OAuth para autorizar el acceso a Google Calendar API.
3. Obtener un token de acceso para llamar a Google Calendar API y programar una reunión.

Auth0 for AI Agents admite los siguientes proveedores de identidad sociales, empresariales y Custom:

* Google
* Microsoft
* Box
* Slack
* GitHub
* OpenID Connect
* Conexión personalizada

<div id="asynchronous-authorization">
  ### Autorización asíncrona
</div>

Añada controles con intervención humana para acciones críticas o diferidas mediante protocolos seguros, como:

* [Client-Initiated Backchannel Authentication (CIBA)](/docs/es/get-started/authentication-and-authorization-flow/client-initiated-backchannel-authentication-flow): Permite que los agentes de IA lleven a cabo de forma asíncrona una autenticación segura por canal secundario y <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) definida en el framework de OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flows">flujos de autorización</Tooltip> cuando se requiere la aprobación del usuario antes de realizar una acción.
* [Rich Authorization Requests (RAR)](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow/authorization-code-flow-with-rar): Refuerza la seguridad al solicitar permisos de autorización específicos para que los usuarios los aprueben mediante notificaciones push.

<div id="authorization-for-rag">
  ### Autorización para RAG
</div>

Usa [Auth0 FGA](https://docs.fga.dev/) para aplicar control de acceso a nivel de documento y de relaciones en tus flujos de Retrieval Augmented Generation (RAG):

* Control de acceso a nivel de documento: realiza comprobaciones de permisos granulares en las fuentes de datos de RAG.
* Control de acceso basado en relaciones (ReBAC): sustituye los roles generales por decisiones precisas y granulares de control de acceso basadas en relaciones.
* Compatibilidad con el cumplimiento normativo: garantiza que solo se recuperen datos autorizados para la privacidad de los datos y el cumplimiento normativo.
