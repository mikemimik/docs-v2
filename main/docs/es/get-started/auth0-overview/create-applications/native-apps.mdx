Para integrar Auth0 con una aplicación nativa, primero debes registrar tu aplicación en Auth0 desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Las aplicaciones nativas deben usar el flujo de código de autorización con PKCE para una autenticación segura. Para obtener más información, consulta [Authorization Code Flow with PKCE](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce). Estas aplicaciones pueden incluir aplicaciones móviles, de escritorio o híbridas que se ejecutan de forma nativa en un dispositivo (por ejemplo, iOS o Android).

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haz clic en **Create Application**.
2. Introduce un nombre descriptivo para tu aplicación, selecciona **Native** y haz clic en **Create**.

   Una vez creada la aplicación, Auth0 te llevará a **Application Settings**, que incluye las siguientes pestañas:

   | Pestaña de configuración | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
   | ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Quick Start**          | Muestra toda la documentación disponible para tu tipo de aplicación.                                                                                                                                                                                                                                                                                                                                                                                               |
   | **Settings**             | Muestra toda la configuración disponible para tu aplicación. De forma predeterminada, la mayoría de las opciones se crean automáticamente. Para obtener más información, consulta [Application Settings](/docs/es/get-started/applications/application-settings).                                                                                                                                                                                                     |
   | **Credentials**          | Muestra el método de autenticación de la aplicación y las credenciales configuradas. Para obtener más información, consulta [Application Credentials](/docs/es/secure/application-credentials)                                                                                                                                                                                                                                                                        |
   | **Add-ons**              | Te permite habilitar plugins asociados a una aplicación. Los add-ons se usan principalmente para integraciones con SAML o WS-Fed y, por lo general, no suelen ser aplicables a las aplicaciones nativas. Para obtener más información, consulta [Enable SAML2 Web App Addon](/docs/es/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) y [Configure WS-Fed Applications](/docs/es/get-started/applications/update-application-connections). |
   | **Connections**          | Te permite habilitar conexiones para tu aplicación. Las conexiones son fuentes de usuarios; pueden habilitarse y compartirse entre varias aplicaciones. Para obtener más información, consulta [Connections](/docs/es/authenticate/identity-providers).                                                                                                                                                                                                               |
   | **Organizations**        | Te permite autenticar usuarios en las organizaciones a las que pertenecen. Para obtener más información, consulta [Organizations](/docs/es/manage-users/organizations).                                                                                                                                                                                                                                                                                               |
   | **Login Experience**     | Te permite configurar la experiencia de inicio de sesión para los usuarios de tu organización. Para obtener más información, consulta [Understand how Auth0 Organizations Work](/docs/es/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                                                                         |

   Para obtener más información, consulta [Application Settings](/docs/es/get-started/applications/application-settings).
3. Si estás desarrollando una aplicación móvil, proporciona los parámetros necesarios de iOS/Android en la sección **Advanced Settings** y haz clic en **Save Changes**.

   * Para aplicaciones iOS, [proporciona tu **Team ID** y tu **App Bundle Identifier**](/docs/es/get-started/applications/enable-universal-links-support-in-apple-xcode).
   * Para aplicaciones Android, [proporciona tu **App Package Name** y tus **Key Hashes**](/docs/es/get-started/applications/enable-android-app-links-support).

<div id="next-steps">
  ## Siguientes pasos
</div>

Una vez que haya registrado y configurado su aplicación, algunos de los siguientes pasos más comunes son:

* Configure una conexión y habilítela para su aplicación.
* Modifique el código de su aplicación para usar la aplicación registrada en Auth0. Consulte nuestros [Auth0 Quickstarts](/docs/es/quickstarts), donde encontrará instrucciones detalladas y ejemplos para diversas tecnologías. También aprenderá a implementar el inicio de sesión y el cierre de sesión, gestionar las sesiones de sus usuarios, recuperar y mostrar información del perfil de usuario, y mucho más.
* Use las [APIs de Auth0](/docs/es/api).

  * La [Authentication API](https://auth0.com/docs/api/authentication) gestiona todas las funciones principales relacionadas con la identidad (por ejemplo, inicio de sesión, cierre de sesión y obtención del perfil de usuario). La mayoría de los usuarios acceden a esta API a través de nuestros Quickstarts, la [biblioteca auth0.js](/docs/es/libraries/auth0js) o el [widget de Lock](/docs/es/libraries/lock). Sin embargo, si está creando manualmente toda su interfaz de autenticación, tendrá que interactuar directamente con esta API.
  * La [Management API](https://auth0.com/docs/api/management/v2) le permite automatizar diversas tareas que también pueden realizarse desde el Dashboard de Auth0 (por ejemplo: crear usuarios, configurar los tipos de concesión de la aplicación).
