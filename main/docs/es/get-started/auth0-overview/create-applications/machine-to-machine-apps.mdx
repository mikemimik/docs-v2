Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debes registrar tu aplicación en Auth0 usando el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Estas aplicaciones pueden incluir aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en tu back-end.

Las aplicaciones M2M están vinculadas a una API y a sus permisos o [scopes](/docs/es/get-started/apis/scopes/api-scopes), que se seleccionan de valores predefinidos. Asegúrate de haber [registrado la API asociada](/docs/es/get-started/auth0-overview/set-up-apis) en Auth0 y definido scopes para la API antes de comenzar este registro.

Si quieres autorizar tu aplicación para acceder solo a la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0, no necesitas registrar una API aparte; Management API ya viene preconfigurada para ti.

Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debes registrar tu aplicación en Auth0 usando el Auth0 Dashboard.

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haz clic en **Create Application**.
2. Ingresa un nombre descriptivo para tu aplicación, selecciona **Machine to Machine Applications** y haz clic en **Create**.
3. Selecciona la API que quieres poder llamar desde tu aplicación.
4. A cada aplicación M2M que accede a una API se le debe conceder un conjunto de permisos (o scopes) que debe otorgar el usuario autenticado. Para obtener más información sobre cómo agregar permisos, consulta [Agregar permisos de API](/docs/es/get-started/apis/add-api-permissions). Selecciona los **Permissions** que quieres que se incluyan en el token de acceso de tu aplicación y haz clic en **Authorize**.

   <Warning>
     Usa los scopes `update:client_grants` y `create:client_grants` solo con aplicaciones con privilegios elevados, ya que permiten que el cliente se otorgue más permisos a sí mismo.
   </Warning>

   Una vez creada la nueva aplicación, puedes configurar **Application Settings**, que incluye las siguientes pestañas:

   | Pestaña de configuración | Descripción                                                                                                                                                                                                                                                                                                                      |
   | ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**          | Muestra todos los ejemplos disponibles para aplicaciones ${application_type}. También te muestra cómo llamar a tu API con varias tecnologías. Para obtener más información sobre cómo aceptar y validar tokens de acceso en tu API, consulta nuestros [Quickstarts de Backend/API](/docs/es/quickstart/backend/nodejs/interactive). |
   | **Settings**             | Muestra toda la configuración disponible para tu aplicación. De forma predeterminada, Auth0 crea la mayor parte de esta configuración por ti.                                                                                                                                                                                    |
   | **Credentials**          | Muestra el método de autenticación de la aplicación y las credenciales configuradas. Para obtener más información, consulta [Application Credentials](/docs/es/secure/application-credentials)                                                                                                                                      |
   | **APIs**                 | Te permite autorizar API adicionales para usarlas con tu aplicación.                                                                                                                                                                                                                                                             |

   Para obtener más información, consulta [Application Settings](/docs/es/get-started/applications/application-settings).

<div id="next-steps">
  ## Próximos pasos
</div>

Una vez que hayas registrado y configurado tu aplicación, algunos pasos habituales a continuación son:

* Configura una conexión de proveedor de identidad y habilítala para tu aplicación (si es necesario para tu caso de uso).
* Modifica el código de tu aplicación para usar la aplicación que registraste en Auth0. Consulta nuestros [Auth0 Quickstarts](/docs/es/quickstarts), donde encontrarás instrucciones detalladas y ejemplos para diversas tecnologías. También aprenderás a implementar el inicio de sesión y el cierre de sesión, gestionar las sesiones de tus usuarios, recuperar y mostrar la información del perfil del usuario, y mucho más.
* Usa las [API de Auth0](/docs/es/api).

  * La [Authentication API](https://auth0.com/docs/api/authentication) gestiona todas las funciones principales relacionadas con la identidad (por ejemplo, inicio de sesión, cierre de sesión y obtención del perfil del usuario). La mayoría de los usuarios utilizan esta API a través de nuestros Quickstarts, la [biblioteca auth0.js](/docs/es/libraries/auth0js) o el [widget de Lock](/docs/es/libraries/lock). Sin embargo, si estás creando manualmente toda la interfaz de autenticación, tendrás que interactuar directamente con esta API.
  * La [Management API](https://auth0.com/docs/api/management/v2) te permite automatizar varias tareas a las que también puedes acceder desde el Dashboard de Auth0 (por ejemplo, crear usuarios y configurar los tipos de concesión de la aplicación).
