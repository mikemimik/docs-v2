Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debe registrar su aplicación en Auth0 con el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Estas aplicaciones pueden incluir aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en su backend.

Las aplicaciones M2M están vinculadas a una API y a sus permisos o [scopes](/docs/es/get-started/apis/scopes/api-scopes), que se seleccionan de valores predefinidos. Asegúrese de haber [registrado la API asociada](/docs/es/get-started/auth0-overview/set-up-apis) en Auth0 y definido scopes para la API antes de comenzar este registro.

Si desea autorizar su aplicación para acceder solo a la Auth0 <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, no necesita registrar una API aparte; la Management API ya está preconfigurada para usted.

Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debe registrar su aplicación en Auth0 usando el Auth0 Dashboard.

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haga clic en **Create Application**.
2. Ingrese un nombre descriptivo para su aplicación, seleccione **Machine to Machine Applications** y haga clic en **Create**.
3. Seleccione la API que quiere poder llamar desde su aplicación.
4. A cada aplicación M2M que accede a una API se le debe conceder un conjunto de permisos (o scopes) que debe otorgar el usuario autenticado. Para obtener información sobre cómo agregar permisos, consulte [Agregar permisos de API](/docs/es/get-started/apis/add-api-permissions). Seleccione los **Permissions** que desea que se emitan como parte del token de acceso de su aplicación y haga clic en **Authorize**.

   <Warning>
     Use los scopes `update:client_grants` y `create:client_grants` solo con aplicaciones con altos privilegios, ya que permiten que el cliente se otorgue permisos adicionales a sí mismo.
   </Warning>

   Una vez creada la nueva aplicación, puede configurar **configuración de la aplicación**, que incluye las siguientes pestañas:

   | Pestaña de configuración | Descripción                                                                                                                                                                                                                                                                                                                     |
   | ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**          | Muestra todos los ejemplos disponibles para aplicaciones ${application_type}. También le muestra cómo puede llamar a su API con varias tecnologías. Para obtener información sobre cómo aceptar y validar tokens de acceso en su API, consulte nuestros [Backend/API Quickstarts](/docs/es/quickstart/backend/nodejs/interactive). |
   | **Settings**             | Muestra toda la configuración disponible para su aplicación. De forma predeterminada, Auth0 crea la mayoría de estas opciones por usted.                                                                                                                                                                                        |
   | **Credentials**          | Muestra el método de autenticación y las credenciales configuradas de la aplicación. Para obtener más información, lea [Application Credentials](/docs/es/secure/application-credentials)                                                                                                                                          |
   | **APIs**                 | Le permite autorizar APIs adicionales para usarlas con su aplicación.                                                                                                                                                                                                                                                           |

   Para obtener más información, lea [configuración de la aplicación](/docs/es/get-started/applications/application-settings).

<div id="next-steps">
  ## Próximos pasos
</div>

Una vez que hayas registrado y configurado tu aplicación, estos son algunos pasos habituales:

* Configura una conexión con un proveedor de identidad y habilítala para tu aplicación (si es necesario para tu caso de uso).
* Modifica el código de tu aplicación para usar la aplicación registrada en Auth0. Consulta nuestros [Auth0 Quickstarts](/docs/es/quickstarts), donde encontrarás instrucciones detalladas y ejemplos para diversas tecnologías. También aprenderás a implementar el inicio de sesión y el cierre de sesión, gestionar las sesiones de usuario, obtener y mostrar la información del perfil de usuario, entre otras cosas.
* Usa las [API de Auth0](/docs/es/api).

  * La [Authentication API](https://auth0.com/docs/api/authentication) gestiona todas las funciones principales relacionadas con la identidad (por ejemplo, inicio de sesión, cierre de sesión y obtención del perfil de usuario). La mayoría de los usuarios utilizan esta API a través de nuestros Quickstarts, la [biblioteca auth0.js](/docs/es/libraries/auth0js) o el [widget de Lock](/docs/es/libraries/lock). Sin embargo, si estás creando manualmente toda la interfaz de usuario de autenticación, tendrás que interactuar directamente con esta API.
  * La [Management API](https://auth0.com/docs/api/management/v2) te permite automatizar varias tareas a las que también puedes acceder desde el Dashboard de Auth0 (por ejemplo, crear usuarios o configurar los tipos de concesión de la aplicación).
