Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debes registrar tu aplicación en Auth0 mediante el <Tooltip tip="Auth0 Dashboard: Producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haz clic en **Create Application**.

2. Introduce un nombre descriptivo para tu aplicación, selecciona **Machine to Machine Applications** y haz clic en **Create**.

3. Selecciona la Management API para autorizar la comunicación M2M, ya que es así como OAG interactúa con Auth0.

4. Haz clic en **Authorize**.

5. A cada aplicación M2M que accede a una API se le debe conceder un conjunto de permisos (o scopes) que debe otorgar el usuario autenticado. Selecciona los **Permissions** que quieres que se emitan como parte del <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de tu aplicación. OAG requiere los siguientes permisos para el **cliente** y los **usuarios:**

* `read:client, write:client, update:client, delete:client`
* `read:users`

6. Haz clic en **Authorize**.

7. Selecciona la pestaña **Credentials** de tu aplicación.

8. Haz clic en **<Tooltip tip="ID de cliente: Valor de identificación asignado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** y cópialo en un editor de texto.

9. Haz clic en **<Tooltip tip="Secreto de cliente: Secreto utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo bastante aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>** y cópialo en un editor de texto.

10. Continúa con el procedimiento [Configurar un proveedor de identidad en Access Gateway](/docs/Configure an Identity Provider in Access Gateway).
