Para integrar Auth0 con una aplicación web clásica, primero debes registrar tu aplicación en Auth0 usando el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Las aplicaciones web clásicas son aplicaciones web tradicionales que ejecutan la mayor parte de la lógica de la aplicación en el servidor (por ejemplo, Express.js, ASP.NET). Para obtener más información sobre los tipos de aplicaciones, consulta [Tipos de aplicación](/docs/es/get-started/applications).

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haz clic en **Create Application**.
2. Introduce un nombre descriptivo para tu aplicación, selecciona **aplicación web clásica** y haz clic en **Create**.

   Una vez creada la nueva aplicación, puedes ver la página de **configuración de la aplicación**, que incluye las siguientes pestañas:

   | Pestaña de configuración | Descripción                                                                                                                                                                                                                                                                                                                                                                                                    |
   | ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**          | Muestra toda la documentación disponible para tu tipo de aplicación.                                                                                                                                                                                                                                                                                                                                           |
   | **Settings**             | Muestra toda la configuración disponible para tu aplicación. De forma predeterminada, la mayoría de los ajustes se crean automáticamente. Para obtener más información, consulta [configuración de la aplicación](/docs/es/get-started/applications/application-settings).                                                                                                                                                  |
   | **Credentials**          | Muestra el método de autenticación de la aplicación y las credenciales configuradas. Para obtener más información, consulta [Application Credentials](/docs/es/secure/application-credentials)                                                                                                                                                                                                                    |
   | **Add-ons**              | Te permite habilitar complementos asociados a una aplicación. Se trata de aplicaciones web SAML o WS-Fed para las que Auth0 genera tokens de acceso. Para obtener más información, consulta [Enable SAML2 Web App Addon](/docs/es/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) y [Configure WS-Fed Applications](/docs/es/get-started/applications/update-application-connections). |
   | **Connections**          | Te permite habilitar conexiones para tu aplicación. Las conexiones son fuentes de usuarios; pueden habilitarse y compartirse entre múltiples aplicaciones. Para obtener más información, consulta [Connections](/docs/es/authenticate/identity-providers).                                                                                                                                                        |
   | **Organizations**        | Te permite autenticar usuarios en las organizaciones a las que pertenecen. Para obtener más información, consulta [Organizations](/docs/es/manage-users/organizations).                                                                                                                                                                                                                                           |
   | **Login Experience**     | Te permite configurar la experiencia de inicio de sesión para los usuarios de tu organización. Para obtener más información, consulta [Understand how Auth0 Organizations Work](/docs/es/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                     |

   Para obtener más información, consulta [configuración de la aplicación](/docs/es/get-started/applications/application-settings).
3. Desplázate hacia abajo y localiza la configuración **Trust Token Endpoint IP Header**. Si la arquitectura de tu aplicación lo permite, habilita esta configuración y haz clic en **Save Changes**. Esta configuración ayuda a proteger contra ataques de fuerza bruta al permitir que Auth0 confíe en los encabezados IP de tu proxy.

<div id="next-steps">
  ## Próximos pasos
</div>

Una vez que hayas registrado y configurado tu aplicación, sigue estos pasos:

1. Configura una conexión y habilítala para tu aplicación.
2. Implementa la autenticación en el código de tu aplicación con la aplicación que registraste en Auth0. Consulta nuestros [Auth0 Quickstarts](/docs/es/quickstarts), donde encontrarás instrucciones detalladas y ejemplos para distintas tecnologías. También aprenderás a implementar el inicio y cierre de sesión, gestionar las sesiones de tus usuarios, recuperar y mostrar la información del perfil de usuario, y mucho más.

* Usa las [Auth0 APIs](/docs/es/api).

  * La [Authentication API](https://auth0.com/docs/api/authentication) gestiona todas las funciones principales relacionadas con la identidad (por ejemplo, iniciar sesión, cerrar sesión y obtener el perfil del usuario). La mayoría de los usuarios utiliza esta API a través de nuestros Quickstarts, la [biblioteca Auth0.js](/docs/es/libraries/auth0js) o el [widget Lock](/docs/es/libraries/lock). Sin embargo, si estás creando manualmente toda la interfaz de usuario de autenticación, puedes interactuar directamente con esta API.
  * La [Management API](https://auth0.com/docs/api/management/v2) te permite automatizar varias tareas a las que también puedes acceder desde el Auth0 Dashboard (por ejemplo, crear usuarios y configurar los tipos de concesión de aplicación).
