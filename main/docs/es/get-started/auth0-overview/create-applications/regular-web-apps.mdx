Para integrar Auth0 con una aplicación web clásica, primero debes registrar tu aplicación en Auth0 mediante el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Las aplicaciones web tradicionales son aplicaciones web convencionales que ejecutan la mayor parte de su lógica en el servidor (por ejemplo, Express.js, ASP.NET). Para obtener más información sobre los tipos de aplicación, consulta [Tipos de aplicación](/docs/es/get-started/applications).

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haz clic en **Create Application**.
2. Introduce un nombre descriptivo para tu aplicación, selecciona **Aplicaciones web clásicas** y haz clic en **Create**.

   Una vez creada la nueva aplicación, podrás ver la página **configuración de la aplicación**, que incluye las siguientes pestañas:

   | Pestaña de configuración | Descripción                                                                                                                                                                                                                                                                                                                                                                                               |
   | ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**          | Muestra toda la documentación disponible para tu tipo de aplicación.                                                                                                                                                                                                                                                                                                                                      |
   | **Settings**             | Muestra toda la configuración disponible para tu aplicación. De forma predeterminada, la mayoría de los ajustes se crearán automáticamente. Para obtener más información, consulta [configuración de la aplicación](/docs/es/get-started/applications/application-settings).                                                                                                                                           |
   | **Credentials**          | Muestra el método de autenticación de la aplicación y las credenciales configuradas. Para obtener más información, consulta [Application Credentials](/docs/es/secure/application-credentials)                                                                                                                                                                                                               |
   | **Add-ons**              | Te permite habilitar plugins asociados a una aplicación. Se trata de aplicaciones web SAML o WS-Fed para las que Auth0 genera tokens de acceso. Para obtener más información, consulta [Enable SAML2 Web App Addon](/docs/es/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) y [Configure WS-Fed Applications](/docs/es/get-started/applications/update-application-connections). |
   | **Connections**          | Te permite habilitar conexiones para tu aplicación. Las conexiones son fuentes de usuarios; pueden habilitarse y compartirse entre varias aplicaciones. Para obtener más información, consulta [Connections](/docs/es/authenticate/identity-providers).                                                                                                                                                      |
   | **Organizations**        | Te permite autenticar usuarios en las organizaciones a las que pertenecen. Para obtener más información, consulta [Organizations](/docs/es/manage-users/organizations).                                                                                                                                                                                                                                      |
   | **Login Experience**     | Te permite configurar la experiencia de inicio de sesión de los usuarios de tu organización. Para obtener más información, consulta [Understand how Auth0 Organizations Work](/docs/es/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                  |

   Para obtener más información, consulta [configuración de la aplicación](/docs/es/get-started/applications/application-settings).
3. Desplázate hacia abajo hasta encontrar la configuración **Trust Token Endpoint IP Header**. Si la arquitectura de tu aplicación lo admite, habilita esta configuración y haz clic en **Save Changes**. Esta configuración ayuda a proteger contra ataques de fuerza bruta al permitir que Auth0 confíe en los encabezados IP de tu proxy.

<div id="next-steps">
  ## Próximos pasos
</div>

Una vez que haya registrado y configurado su aplicación, siga estos pasos:

1. Configure una conexión y habilítela para su aplicación.
2. Implemente la autenticación en el código de su aplicación usando la aplicación que registró en Auth0. Consulte nuestros [Auth0 Quickstarts](/docs/es/quickstarts), donde encontrará instrucciones detalladas y ejemplos para diversas tecnologías. También aprenderá a implementar el inicio de sesión y el cierre de sesión, gestionar las sesiones de los usuarios, recuperar y mostrar información del perfil de usuario, y mucho más.

* Use las [Auth0 APIs](/docs/es/api).

  * La [Authentication API](https://auth0.com/docs/api/authentication) gestiona todas las funciones principales relacionadas con la identidad (por ejemplo, inicio de sesión, cierre de sesión y obtención del perfil de usuario). La mayoría de los usuarios utilizan esta API a través de nuestros Quickstarts, la [biblioteca auth0.js](/docs/es/libraries/auth0js) o el [widget de Lock](/docs/es/libraries/lock). Sin embargo, si está creando manualmente toda la UI de autenticación, puede interactuar directamente con esta API.
  * La [Management API](https://auth0.com/docs/api/management/v2) le permite automatizar varias tareas a las que también puede acceder a través del Auth0 Dashboard (por ejemplo, crear usuarios y configurar los tipos de concesión de la aplicación).
