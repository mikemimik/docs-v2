Auth0 ofrece a los usuarios del tenant una forma más intuitiva de buscar aplicaciones y API registradas, con la posibilidad de crear expresiones de filtro para los campos en los que se puede buscar en Auth0 Dashboard. En lugar de desplazarse por listas paginadas con cientos de recursos, los usuarios del tenant pueden hacer consultas sobre campos como:

* El nombre de tu aplicación (Travel0-native)
* Los metadatos de la aplicación o del usuario (permisos)
* El tipo de aplicación (SPA)
* La API por nombre
* La API por ID

Auth0 Dashboard devuelve resultados coincidentes en tiempo real de todos tus recursos registrados.

<div id="how-it-works">
  ## Cómo funciona
</div>

La barra de búsqueda de las secciones [Applications](https://manage.auth0.com/#/applications) y [APIs](https://manage.auth0.com/#/apis) de Auth0 Dashboard usa una interfaz de filtros guiados y búsqueda booleana. Funciona como un menú desplegable que muestra los campos por los que se puede filtrar. Cuando seleccionas un campo e introduces un valor, el filtro lo aplica como una etiqueta que se puede quitar sobre los resultados. Puedes combinar hasta cinco filtros a la vez.

Cada filtro que añades acota aún más los resultados. Para definir los resultados, aplica lógica `AND/OR` entre distintos campos. Los filtros se reflejan en la URL como parámetros de consulta. Puedes copiar la URL en cualquier momento para compartirla con un compañero de equipo o guardarla en tus marcadores para volver más tarde a la misma vista filtrada.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Ejemplo booleano**

  Si filtras por **Application type** `Native` OR `SPA`, y **cliente de primera parte** `true`, los resultados muestran solo aplicaciones propias que son Native o SPA. Se excluyen las Applications que son Regular Web Apps o de terceros.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportamiento de la búsqueda por tipo de campo
</div>

La búsqueda en Auth0 Dashboard encuentra coincidencias en los resultados según el tipo de campo.

* **Campos String** (name, identifier): Se admite la coincidencia parcial. Al escribir `payments`, se encuentran &quot;Payments-application&quot;, &quot;Corp Payments Service&quot; y otros similares; el orden de las palabras no importa. Para excluir resultados que coincidan con un término, antepón `NOT`, `-` o `No`; por ejemplo, si escribes `NOT test` en el campo name, se devuelven todos los recursos cuyo nombre no contiene &quot;test&quot;. Las búsquedas con comodines requieren al menos tres caracteres; si introduces menos, se muestra el mensaje &quot;Minimum 3 characters&quot; y no se envía la consulta.

* **Campos de coincidencia exacta**

Debes introducir el valor completo del ID. Los IDs parciales no devuelven resultados. Usa estos campos cuando tengas un ID de cliente concreto de un registro o de un ticket de soporte.

* **Campos de metadata**

Puedes buscar metadata por clave y valor, o solo por clave:

* **Key and value**: Introduce una clave y un valor para encontrar recursos donde esa clave exacta coincida con ese valor. Por ejemplo, la clave `environment` y el valor `production` devuelven solo los recursos donde `app_metadata.environment` es igual a `production`. Se admite la coincidencia por prefijo para los valores; al introducir `prod`, coincide con `production`, `prod-eu` y `prod-us`.
  * **Solo key**: Introduce una clave y pulsa **Enter** sin indicar un valor. Esto devuelve todos los recursos que tienen esa clave de metadata configurada, independientemente de su valor. Es útil para encontrar recursos que usen una clave específica.

También puedes buscar claves de metadata anidadas hasta dos niveles de profundidad usando notación de puntos. Por ejemplo, la clave `tier.bronze` y el valor `true` coinciden con recursos donde `app_metadata.tier.bronze` es igual a `true`.

Para excluir resultados que coincidan con un par clave-valor específico, antepón `NOT` al valor. Por ejemplo, la clave `color` y el valor `NOT orange` devuelven todos los recursos donde `color` está configurado con cualquier valor distinto de `orange`. Las claves deben coincidir de forma exacta. No se admiten coincidencias por contenido ni por terminación para metadata. Los valores se indexan hasta 64 caracteres; los valores más largos no se pueden buscar.

* **Campos enum**

Selecciona uno o varios tipos de aplicación de una lista predefinida. Varias selecciones dentro del mismo filtro usan lógica OR. Por ejemplo, seleccionar `Native` y `SPA` devuelve todas las applications Native y SPA.

* **Campos Boolean**

Busca clientes propios seleccionando `true` o `false`. Usa esto para distinguir las aplicaciones propias (las autorizadas para omitir la pantalla de consentimiento) de las de terceros.

<div id="searchable-fields">
  ### Campos que se pueden buscar
</div>

<div id="applications">
  #### Aplicaciones
</div>

| **Campo**                                        | **Descripción**                                                      | **Tipo de coincidencia**          |
| ------------------------------------------------ | -------------------------------------------------------------------- | --------------------------------- |
| **Nombre**                                       | El nombre de la aplicación                                           | Coincidencia parcial              |
| **ID de cliente** (`client_id`)                  | El ID de cliente único                                               | Coincidencia exacta               |
| **ID de cliente externo** (`external_client_id`) | Un identificador externo de la aplicación                            | Coincidencia exacta               |
| **Metadatos**                                    | Metadatos personalizados de clave-valor almacenados en la aplicación | Coincidencia exacta o por prefijo |
| **Tipo de aplicación**                           | Native, SPA, aplicación web tradicional o Machine to Machine         | Selección de enum                 |
| **Cliente de primera parte**                     | Si la aplicación es de primera parte o de tercera parte              | Booleano                          |

<div id="apis">
  #### APIs
</div>

| **Campo**         | **Descripción**                                                             | **Tipo de coincidencia** |
| ----------------- | --------------------------------------------------------------------------- | ------------------------ |
| **Nombre**        | El nombre de la API                                                         | Coincidencia parcial     |
| **Identificador** | El identificador único de audience (por ejemplo, `https://api.example.com`) | Coincidencia parcial     |
| **Id**            | El ID interno del sistema                                                   | Coincidencia exacta      |

<div id="limitations">
  ### Limitaciones
</div>

* Puedes aplicar un máximo de cinco filtros por búsqueda. La barra de búsqueda impide añadir un sexto filtro.
* Es posible que los resultados de búsqueda no reflejen los cambios realizados en los últimos segundos, ya que el índice tiene consistencia eventual.
* En condiciones normales, los resultados se devuelven en un plazo de dos segundos. Si una consulta supera el tiempo de espera, reduce la cantidad de filtros o usa un término de búsqueda más específico.

<div id="example-queries">
  ## Consultas de ejemplo
</div>

<div id="find-an-application-by-client-id">
  ### Buscar una aplicación por ID de cliente
</div>

Usa esta opción cuando tengas un `client_id` de un registro de errores o de un ticket de soporte y necesites localizar la aplicación rápidamente.

1. Ve a **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Selecciona la barra de búsqueda y elige **ID de cliente**.
3. Pega el valor completo de `client_id`. La aplicación correspondiente aparece de inmediato.

<div id="find-applications-using-metadata">
  ### Buscar aplicaciones mediante metadatos
</div>

Si tus aplicaciones almacenan metadatos personalizados, como etiquetas de entorno, niveles del plan o IDs externos, puedes buscar por esos valores.

1. Ve a **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Selecciona la barra de búsqueda y elige **Metadatos**.
3. Introduce la clave de los metadatos, por ejemplo `environment`.
4. Introduce el valor, por ejemplo `production`. Las aplicaciones en las que `metadata.environment` es igual a `production` aparecen en los resultados. Para buscar por prefijo, introduce un valor parcial, como `prod`.

<div id="troubleshooting">
  ## Solución de problemas
</div>

**No se encontraron resultados**

Comprueba que los valores del filtro estén escritos correctamente. En los campos de tipo `String`, prueba con un término más corto o más general. En los campos de metadatos, confirma el nombre exacto de la clave: las claves distinguen entre mayúsculas y minúsculas y deben coincidir exactamente. Los metadatos no admiten coincidencias de tipo &quot;contiene&quot;; usa un prefijo en su lugar. En los campos de coincidencia exacta, como el ID de la aplicación, asegúrate de pegar el valor completo del ID.

**Los resultados parecen desactualizados**

Los resultados de búsqueda son eventualmente consistentes y puede que no reflejen los cambios realizados en los últimos segundos. Si has creado o actualizado un recurso hace poco, espera unos segundos y vuelve a intentar la búsqueda.

**Se agotó el tiempo de espera de la búsqueda**

Reduce el número de filtros activos o usa un término de búsqueda más específico. Si el problema persiste, ponte en contacto con el [equipo de soporte de Auth0](https://support.auth0.com).
