Auth0 ofrece a los usuarios del tenant una experiencia de búsqueda más intuitiva para las Applications y las API registradas, con la posibilidad de crear expresiones de filtro para campos consultables en Auth0 Dashboard. En lugar de desplazarse por listas paginadas con cientos de recursos, el usuario del tenant puede realizar consultas sobre campos como:

* El nombre de tu aplicación (Travel0-native)
* Metadatos de la aplicación o del usuario (permissions)
* Tipo de aplicación (SPA)
* API por nombre
* API por ID

Auth0 Dashboard muestra resultados coincidentes en tiempo real de todos tus recursos registrados.

<div id="how-it-works">
  ## Cómo funciona
</div>

La barra de búsqueda de las secciones [Applications](https://manage.auth0.com/#/applications) y [APIs](https://manage.auth0.com/#/apis) de Auth0 Dashboard usa una interfaz de filtros guiados y búsqueda booleana. Funciona como un menú desplegable que muestra los campos por los que se puede filtrar. Cuando seleccionas un campo e introduces un valor, este se aplica como una etiqueta que se puede quitar sobre los resultados. Puedes combinar hasta cinco filtros a la vez.

Cada filtro que añades reduce aún más los resultados. Para acotarlos, aplica lógica `AND/OR` entre distintos campos. Los filtros se reflejan en la URL como parámetros de consulta. Copia la URL en cualquier momento para compartirla con un compañero de equipo o guardarla en tus marcadores y volver más tarde a la misma vista filtrada.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Ejemplo booleano**

  Si filtras por **tipo de aplicación** `Native` OR `SPA`, y **cliente propio** `true`, los resultados muestran solo aplicaciones propias que son Native o SPA. Se excluyen las aplicaciones que son Regular Web Apps o aplicaciones de terceros.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportamiento de la búsqueda por tipo de campo
</div>

La búsqueda en el Auth0 Dashboard devuelve resultados según el tipo de campo.

* **Campos String** (name, identifier): Se admite la coincidencia parcial. Al introducir `payments`, se encuentran &quot;Payments-application&quot;, &quot;Corp Payments Service&quot; y resultados similares; el orden de las palabras no importa. Para excluir resultados que coincidan con un término, antepón `NOT`, `-` o `No`; por ejemplo, al introducir `NOT test` en el campo de nombre, se devuelven todos los recursos cuyo nombre no contiene &quot;test&quot;. Las búsquedas con comodines requieren al menos tres caracteres; si introduces menos, se muestra un mensaje de &quot;Minimum 3 characters&quot; y no se envía la consulta.

* **Campos de coincidencia exacta**

Debes introducir el valor completo del ID. Los IDs parciales no devuelven resultados. Usa estos campos cuando tengas un ID de cliente específico en un registro o un ticket de soporte.

* **Campos de metadato**

Puedes buscar metadato por clave y valor, o solo por clave:

* **Clave y valor**: Introduce una clave y un valor para encontrar recursos donde esa clave exacta coincida con ese valor. Por ejemplo, la clave `environment` y el valor `production` devuelven solo los recursos donde `app_metadata.environment` es igual a `production`. Se admite la coincidencia por prefijo para los valores; al introducir `prod`, coincide con `production`, `prod-eu` y `prod-us`.
  * **Solo clave**: Introduce una clave y pulsa **Enter** sin indicar un valor. Esto devuelve todos los recursos que tienen esa clave de metadato configurada, independientemente de su valor. Es útil para encontrar recursos que usen una clave específica.

También puedes buscar claves de metadato anidadas hasta dos niveles de profundidad usando notación con puntos. Por ejemplo, la clave `tier.bronze` y el valor `true` coinciden con recursos donde `app_metadata.tier.bronze` es igual a `true`.

Para excluir resultados que coincidan con un par clave-valor específico, antepón `NOT` al valor. Por ejemplo, la clave `color` y el valor `NOT orange` devuelven todos los recursos donde `color` está configurado con cualquier valor distinto de `orange`. Las claves deben coincidir exactamente. No se admiten coincidencias por contenido ni por terminación para metadato. Los valores se indexan hasta 64 caracteres; los valores más largos no se pueden buscar.

* **Campos de enumeración**

Selecciona uno o varios tipos de aplicación de una lista predefinida. Varias selecciones dentro del mismo filtro usan lógica OR. Por ejemplo, seleccionar `Native` y `SPA` devuelve todas las applications Native y SPA.

* **Campos booleanos**

Busca clientes propios seleccionando `true` o `false`. Usa esto para distinguir las aplicaciones propias (las autorizadas a omitir la pantalla de consentimiento) de las de terceros.

<div id="searchable-fields">
  ### Campos consultables
</div>

<div id="applications">
  #### Aplicaciones
</div>

| **Campo**                                        | **Descripción**                                                            | **Tipo de coincidencia**          |
| ------------------------------------------------ | -------------------------------------------------------------------------- | --------------------------------- |
| **Nombre**                                       | El nombre de la aplicación                                                 | Coincidencia parcial              |
| **ID de cliente** (`client_id`)                  | El ID de cliente único                                                     | Coincidencia exacta               |
| **ID de cliente externo** (`external_client_id`) | Un identificador externo de la aplicación                                  | Coincidencia exacta               |
| **Metadatos**                                    | Metadatos personalizados de pares clave-valor almacenados en la aplicación | Coincidencia exacta o por prefijo |
| **Tipo de aplicación**                           | Nativa, SPA, aplicación web tradicional o máquina a máquina                | Selección de enumeración          |
| **Cliente propio**                               | Si la aplicación es de primera parte o de terceros                         | Booleano                          |

<div id="apis">
  #### APIs
</div>

| **Campo**         | **Descripción**                                                                 | **Tipo de coincidencia** |
| ----------------- | ------------------------------------------------------------------------------- | ------------------------ |
| **Nombre**        | El nombre de la API                                                             | Coincidencia parcial     |
| **Identificador** | El identificador único de la audiencia (por ejemplo, `https://api.example.com`) | Coincidencia parcial     |
| **Id**            | El ID interno del sistema                                                       | Coincidencia exacta      |

<div id="limitations">
  ### Limitaciones
</div>

* Puedes aplicar un máximo de cinco filtros por búsqueda. La barra de búsqueda impide añadir un sexto filtro.
* Es posible que los resultados de búsqueda no reflejen los cambios realizados en los últimos segundos, ya que el índice tiene consistencia eventual.
* Los resultados se muestran en un plazo de dos segundos en condiciones normales. Si una consulta supera el tiempo de espera, reduce el número de filtros o usa un término de búsqueda más específico.

<div id="example-queries">
  ## Consultas de ejemplo
</div>

<div id="find-an-application-by-client-id">
  ### Buscar una aplicación por ID de cliente
</div>

Usa esto cuando tengas un `client_id` de un registro de errores o de un ticket de soporte y necesites encontrar la aplicación rápidamente.

1. Ve a **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Selecciona la barra de búsqueda y elige **ID de cliente**.
3. Pega el valor completo de `client_id`. La aplicación correspondiente aparecerá de inmediato.

<div id="find-applications-using-metadata">
  ### Buscar aplicaciones por metadato
</div>

Si sus aplicaciones almacenan metadatos personalizados, como etiquetas de entorno, niveles de plan o IDs externos, puede buscar por esos valores.

1. Vaya a **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Seleccione la barra de búsqueda y elija **Metadato**.
3. Introduzca la clave del metadato, por ejemplo, `environment`.
4. Introduzca el valor, por ejemplo, `production`. Las aplicaciones en las que `metadata.environment` es igual a `production` aparecen en los resultados. Para buscar por prefijo, introduzca un valor parcial, como `prod`.

<div id="troubleshooting">
  ## Solución de problemas
</div>

**No se encontraron resultados**

Comprueba que los valores de los filtros estén escritos correctamente. En los campos de texto, prueba con un término más corto o más general. En los campos de metadatos, confirma el nombre exacto de la clave: las claves distinguen entre mayúsculas y minúsculas y deben coincidir exactamente. No se admite la coincidencia por contenido en los metadatos; usa un prefijo en su lugar. En los campos de coincidencia exacta, como ID de la aplicación, asegúrate de pegar el valor completo del ID.

**Los resultados parecen desactualizados**

Los resultados de búsqueda tienen consistencia eventual y pueden no reflejar los cambios realizados en los últimos segundos. Si hace poco creaste o actualizaste un recurso, espera unos segundos y vuelve a intentar la búsqueda.

**La búsqueda excedió el tiempo de espera**

Reduce el número de filtros activos o usa un término de búsqueda más específico. Si el problema persiste, ponte en contacto con el [equipo de soporte de Auth0](https://support.auth0.com).
