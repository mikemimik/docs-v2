Puedes crear una aplicación manualmente o importarla desde una URL de [Documento de metadatos del ID de cliente (CIMD)](/docs/es/get-started/auth0-overview/create-applications/register-applications-with-cimd) en el Auth0 Dashboard.

<div id="create-manually">
  ## Crear manualmente
</div>

Cuando creas manualmente una aplicación, Auth0 le asigna un <Tooltip tip="ID de cliente: Cadena alfanumérica que sirve como identificador único de una aplicación." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>, una cadena alfanumérica que sirve como identificador único de la aplicación. Usarás este ID en el código de tu aplicación cuando llames a las API de Auth0. Este proceso de registro de la aplicación se denomina registro manual de clientes.

Otro dato importante es el <Tooltip tip="Secreto de cliente: secreto utilizado por un cliente (aplicación) para autenticarse ante el servidor de autorización; solo el cliente y el servidor de autorización deben conocerlo, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">Secreto de cliente</Tooltip>. Debe mantenerse confidencial en todo momento. Si alguien obtiene acceso al secreto de cliente de tu aplicación, puede hacerse pasar por tu aplicación y acceder a recursos protegidos. Es importante que selecciones el tipo de aplicación correcto para ayudar a Auth0 a detectar ciertos riesgos de seguridad.

<div id="import-from-url">
  ## Importar desde una URL
</div>

Puedes registrar aplicaciones importando desde una URL un Documento de metadatos del ID de cliente (CIMD) alojado externamente. CIMD es un archivo JSON que contiene metadatos del cliente y está alojado en un dominio HTTPS seguro controlado por la aplicación. Este proceso de registro de aplicaciones se denomina registro manual con CIMD. Para obtener más información, consulta [Registrar aplicaciones con CIMD](/docs/es/get-started/auth0-overview/create-applications/register-applications-with-cimd).

<div id="manual-client-registration-vs-manual-cimd-registration">
  ## Registro manual de clientes frente a registro manual de CIMD
</div>

La siguiente tabla compara el registro manual de clientes con el registro manual de CIMD:

| Característica                   | Registro manual de clientes              | Registro manual de CIMD                                  |
| -------------------------------- | ---------------------------------------- | -------------------------------------------------------- |
| **ID de cliente**                | Generado por Auth0                       | URL HTTPS (p. ej., `https://yourdomain.com/client.json`) |
| **Autenticación**                | secreto de cliente o certificado         | Solo claves asimétricas (sin secretos)                   |
| **Verificación de la identidad** | Ninguna                                  | Titularidad del dominio mediante alojamiento HTTPS       |
| **Gestión de metadatos**         | Se actualizan mediante Dashboard/API     | CIMD es la fuente de referencia                          |
| **Rotación de claves**           | Manual, una sola clave                   | Se admiten varias claves en `jwks_uri`                   |
| **Tipo de aplicación**           | Todos los tipos                          | Solo aplicaciones de terceros                            |
| **Ideal para**                   | Aplicaciones propias, servicios internos | Clientes MCP, integraciones de terceros                  |

<div id="create-application-in-auth0">
  ## Crear una aplicación en Auth0
</div>

Para crear una aplicación en Auth0:

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Selecciona **Create Application**.
3. Selecciona **Create it manually** o **Import from URL**.
   * **Import from URL**: Sigue las instrucciones de [Registrar aplicaciones con CIMD](/docs/es/get-started/auth0-overview/create-applications/register-applications-with-cimd).
   * **Create it manually**: Asigna a tu nueva aplicación un nombre descriptivo. Luego, selecciona un tipo de aplicación:
     * **Native Applications**: Estas aplicaciones incluyen apps móviles, de escritorio o híbridas que se ejecutan de forma nativa en un dispositivo (por ejemplo, iOS o Android).
     * **Single-Page Web Applications**: Estas aplicaciones incluyen aplicaciones web de página única de JavaScript que ejecutan la mayor parte de la lógica de la interfaz de usuario en un navegador web y se comunican con un servidor web principalmente mediante API (por ejemplo, AngularJS + Node.js o React).
     * **Regular Web Applications**: Estas aplicaciones son aplicaciones web clásicas que ejecutan la mayor parte de la lógica de la aplicación en el servidor (por ejemplo, Express.js o ASP.NET).
     * **Machine-to-Machine Applications**: Estas aplicaciones incluyen aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en tu back-end.
4. Selecciona **Create**.
