Puedes crear una aplicación manualmente o importarla en el Auth0 Dashboard desde la URL de un [Documento de metadatos del ID de cliente (CIMD)](/docs/es/get-started/auth0-overview/create-applications/register-applications-with-cimd).

<div id="create-manually">
  ## Crear manualmente
</div>

Cuando creas manualmente una aplicación, Auth0 le asigna un <Tooltip tip="ID de cliente: Cadena alfanumérica que sirve como identificador único de una aplicación." cta="Ver glosario" href="/docs/es/glossary?term=client+ID">ID de cliente</Tooltip>, una cadena alfanumérica que sirve como identificador único de la aplicación. Usarás este ID en el código de tu aplicación al llamar a las API de Auth0. Este proceso de registro de la aplicación se denomina registro manual de clientes.

Otro dato importante es el <Tooltip tip="Secreto de cliente: Secret usado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip>. Debe mantenerse confidencial en todo momento. Si alguien obtiene acceso al secreto de cliente de tu aplicación, puede hacerse pasar por tu aplicación y acceder a recursos protegidos. Es importante que selecciones el tipo de aplicación correcto para ayudar a Auth0 a detectar ciertos riesgos de seguridad.

<div id="import-from-url">
  ## Importar desde URL
</div>

Puedes registrar aplicaciones importando desde una URL un Documento de metadatos del ID de cliente (CIMD) alojado externamente. CIMD es un archivo JSON que contiene metadatos del cliente y está alojado en un dominio HTTPS seguro controlado por la aplicación. Este proceso de registro de aplicaciones se denomina registro manual de CIMD. Para obtener más información, consulta [Registrar aplicaciones con CIMD](/docs/es/get-started/auth0-overview/create-applications/register-applications-with-cimd).

<div id="manual-client-registration-vs-manual-cimd-registration">
  ## Registro manual de cliente vs. registro manual de CIMD
</div>

La siguiente tabla compara el registro manual de cliente con el registro manual de CIMD:

| Funcionalidad                  | Registro manual de cliente               | Registro manual de CIMD                                  |
| ------------------------------ | ---------------------------------------- | -------------------------------------------------------- |
| **ID de cliente**              | Generado por Auth0                       | URL HTTPS (p. ej., `https://yourdomain.com/client.json`) |
| **Autenticación**              | secreto de cliente o certificado         | Solo claves asimétricas (sin secretos)                   |
| **Verificación de identidad**  | None                                     | titularidad del dominio mediante alojamiento en HTTPS    |
| **Gestión de metadatos**       | Se actualizan mediante Dashboard/API     | La fuente de referencia es el CIMD                       |
| **Tipo de rotación de claves** | Manual, una sola clave                   | Compatible con varias claves en `jwks_uri`               |
| **Tipo de aplicación**         | Todos los tipos                          | Solo aplicaciones de terceros                            |
| **Ideal para**                 | Aplicaciones propias, servicios internos | clientes MCP, integraciones de terceros                  |

<div id="create-application-in-auth0">
  ## Crear una aplicación en Auth0
</div>

Para crear una aplicación en Auth0:

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Seleccione **Create Application**.
3. Seleccione **Create it manually** o **importar desde URL**.
   * **importar desde URL**: Siga las instrucciones de [registrar aplicaciones con CIMD](/docs/es/get-started/auth0-overview/create-applications/register-applications-with-cimd).
   * **Create it manually**: Asigne a su nueva aplicación un nombre descriptivo. Luego, seleccione un tipo de aplicación:
     * **Aplicaciones nativas**: Estas aplicaciones incluyen apps móviles, de escritorio o híbridas que se ejecutan de forma nativa en un dispositivo (por ejemplo, iOS o Android).
     * **Aplicaciones web de página única**: Estas aplicaciones incluyen aplicaciones de Javascript que ejecutan la mayor parte de la lógica de su interfaz de usuario en un navegador web y se comunican con un servidor web principalmente a través de APIs (por ejemplo, AngularJS + Node.js o React).
     * **Aplicaciones web clásicas**: Estas aplicaciones son aplicaciones web tradicionales que ejecutan la mayor parte de la lógica de la aplicación en el servidor (por ejemplo, Express.js o ASP.NET).
     * **Aplicaciones de máquina a máquina**: Estas aplicaciones incluyen aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en su back-end.
4. Seleccione **Create**.
