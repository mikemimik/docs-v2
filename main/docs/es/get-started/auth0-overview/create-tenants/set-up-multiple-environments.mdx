---
description: Describe cómo usar varios tenants de Auth0 para gestionar diversos entornos.
title: Configurar varios entornos
---

Los entornos de desarrollo, staging y producción son fáciles de configurar en Auth0. Simplemente crea un tenant nuevo para cada entorno para garantizar el aislamiento entre ellos. Puedes cambiar fácilmente de un tenant a otro con el selector de tenants del menú superior izquierdo del Dashboard. También puedes configurar distintos administradores para cada uno.

Los [límites de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy) de producción solo se aplican a los tenants etiquetados como `Production`. Asegúrate de que la etiqueta de entorno de tu tenant esté configurada como `Production` antes de entrar en producción, y ten en cuenta las diferencias entre los límites de frecuencia de los distintos tenants, entornos y estructuras de precios.

Puedes poner el nombre que prefieras a tus distintos entornos. Para los entornos de producción, recomendamos encarecidamente usar [dominios personalizados](/docs/es/customize/custom-domains).

Si tienes un plan de suscripción Enterprise, puedes crear tenants secundarios [vinculando varios tenants a una sola suscripción](/docs/es/get-started/auth0-overview/create-tenants/child-tenants); así, cada tenant secundario obtiene acceso a las funcionalidades de pago o mejoradas asociadas a la suscripción. Puedes asegurarte de que tus entornos de desarrollo, staging o pruebas tengan acceso a las mismas funcionalidades que tu entorno de producción vinculándolos todos a la misma suscripción Enterprise.

<div id="tag-the-environment">
  ## Asigna una etiqueta al entorno
</div>

Para cada tenant nuevo que se cree, debes especificar su entorno. Puedes asignar etiquetas de entorno a tus tenants para distinguir entre los entornos de desarrollo, staging y producción.

Si tu tenant es de uso mixto, elige el entorno de nivel más alto. Por ejemplo, un tenant que se usa tanto para desarrollo como para producción debe configurarse como **Production**.

1. Para asignar una etiqueta de entorno a un tenant, ve a [Dashboard &gt; Settings &gt; General](https://manage.auth0.com/#/tenant/general).

   <Frame>![Pestaña General de Settings del tenant en el Dashboard](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. En **Assign Environment Tag**, identifica el entorno de tu tenant como **Development**, **Staging** o **Production**.

   <Frame>![undefined](/docs/images/cdy7uua7fh8z/27OH1jFXce97CSjk7TPeD7/deafc344f675fb11ff5d3d589ffadee9/2025-02-26_14-27-31.png)</Frame>
3. Después de seleccionar el entorno, haz clic en **Save**.

<div id="migration">
  ## Migración
</div>

A través de la [Management API v2](https://auth0.com/docs/api/management/v2), puedes automatizar la migración de assets ([reglas](/docs/es/customize/rules), [conexiones](/docs/es/authenticate/identity-providers) de base de datos, etc.) entre tenants.

Para facilitar la gestión de la configuración, guarda los valores de configuración en el [Dashboard](https://manage.auth0.com/#/rules), en lugar de hardcodearlos en los scripts de tus **reglas** o **conexiones de base de datos**.

Por ejemplo, supongamos que quieres definir una URL para los logs. Una forma de hacerlo es hardcodearla en la rule:

```javascript lines
function(user, context, callback){
      var log_url = 'https://someurl/log';
      ...
    }
```

Sin embargo, este código no es portable, ya que es probable que esta URL cambie del entorno de desarrollo al de producción.

La forma recomendada de trabajar con código que necesitas usar o mover de desarrollo a producción es mediante la sección [Rules](https://manage.auth0.com/#/rules). Si todavía no has creado una rule, tendrás que hacerlo. (De lo contrario, ve al paso 4).

1. Haz clic en **Create Your First Rule**.
2. Elige la plantilla **empty rule**.
3. Introduce un nombre para tu nueva rule y haz clic en **Save**.
4. Ve a [Dashboard &gt; Rules](https://manage.auth0.com/#/rules) y desplázate hasta la parte inferior de la página para establecer tus valores de configuración (usaremos `log_url` como nombre de la clave y `https://someurl/log` como valor); luego, haz clic en **Create**.
5. Ahora ya puedes escribir tu rule. Edita la rule que creaste, introduce el siguiente código en el área de código y haz clic en **Save**.

   ```javascript lines
   function(user, context, callback){
         var log_url = configuration.log_url;
         ...
       }
   ```

Este código es portable y, cuando migres a producción, solo tendrás que cambiar esta configuración en lugar de buscar en tus scripts.

<div id="adldap-connectors">
  ## Conectores AD/LDAP
</div>

Si utiliza varios tenants de Auth0 con AD/LDAP, deberá crear una conexión AD/LDAP y configurar un Conector AD/LDAP para cada tenant. Esto se debe a que cada Conector AD/LDAP está vinculado a una conexión específica dentro de un tenant de Auth0.

Varios Conectores AD/LDAP pueden apuntar al mismo directorio AD o LDAP, pero cada Conector AD/LDAP solo puede ser utilizado por una conexión dentro de un tenant de Auth0.

Si tiene varios directorios AD/LDAP con los que los usuarios se autenticarán (por ejemplo, para dar soporte a distintos departamentos o clientes, cada uno con su propio directorio), puede configurar varios Conectores AD/LDAP dentro de cada tenant de Auth0.
