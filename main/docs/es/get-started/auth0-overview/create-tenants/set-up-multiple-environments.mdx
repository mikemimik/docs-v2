---
description: Describe cómo usar múltiples tenants de Auth0 para administrar varios entornos.
title: Configurar múltiples entornos
---

Los entornos de desarrollo, staging y producción son fáciles de configurar en Auth0. Solo tiene que crear un tenant nuevo para cada entorno a fin de garantizar el aislamiento entre ellos. Puede cambiar fácilmente de un tenant a otro con el selector de tenants del menú superior izquierdo del Dashboard. También puede configurar distintos administradores para cada uno.

Los [límites de frecuencia](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy) de Production solo se aplican a los tenants etiquetados como `Production`. Asegúrese de que la etiqueta de entorno de su tenant esté configurada como `Production` antes de entrar en producción y tenga en cuenta las diferencias entre los límites de frecuencia de los distintos tenants, entornos y estructuras de tarifas.

Puede nombrar sus distintos entornos como prefiera. Para los entornos de producción, le recomendamos encarecidamente usar [dominios personalizados](/docs/es/customize/custom-domains).

Si tiene una suscripción Enterprise, puede crear tenants secundarios [vinculando varios tenants a una sola suscripción](/docs/es/get-started/auth0-overview/create-tenants/child-tenants); cada tenant secundario obtiene acceso a las funcionalidades de pago o mejoradas asociadas con la suscripción. Puede asegurarse de que sus entornos de desarrollo, staging o pruebas tengan acceso a las mismas funcionalidades que su entorno de producción vinculándolos todos a la misma suscripción Enterprise.

<div id="tag-the-environment">
  ## Etiquetar el entorno
</div>

Para cada tenant nuevo que se cree, debes especificar su entorno. Puedes asignar etiquetas de entorno a tus tenants para diferenciar entre los entornos de desarrollo, staging y producción.

Si tu tenant tiene un uso mixto, elige el entorno de nivel superior. Por ejemplo, un tenant que se usa tanto para desarrollo como para producción debe configurarse como **Production**.

1. Para asignar una etiqueta de entorno a un tenant, ve a [Dashboard &gt; Settings &gt; General](https://manage.auth0.com/#/tenant/general).

   <Frame>![Pestaña General de Configuración del tenant en el Dashboard](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. En **Assign Environment Tag**, identifica el entorno de tu tenant como **Development**, **Staging** o **Production**.

   <Frame>![undefined](/docs/images/cdy7uua7fh8z/27OH1jFXce97CSjk7TPeD7/deafc344f675fb11ff5d3d589ffadee9/2025-02-26_14-27-31.png)</Frame>
3. Después de seleccionar el entorno, haz clic en **Save**.

<div id="migration">
  ## Migración
</div>

A través de la [Management API v2](https://auth0.com/docs/api/management/v2), puedes automatizar la migración de recursos ([rules](/docs/es/customize/rules), [conexiones](/docs/es/authenticate/identity-providers) de base de datos, etc.) entre tenants.

Para gestionar la configuración más fácilmente, guarda los valores de configuración en el [Dashboard](https://manage.auth0.com/#/rules) en lugar de codificarlos de forma fija en los scripts de tus **rules** o **conexiones de base de datos**.

Por ejemplo, supongamos que quieres definir una URL para los logs. Una forma de hacerlo es codificarla de forma fija en la rule:

```javascript lines
function(user, context, callback){
      var log_url = 'https://someurl/log';
      ...
    }
```

Sin embargo, este código no es portable, ya que es probable que esta URL cambie de desarrollo a producción.

La forma recomendada de trabajar con código que necesita usar o mover de desarrollo a producción es mediante la sección [Rules](https://manage.auth0.com/#/rules). Si todavía no ha creado una rule, tendrá que hacerlo. (De lo contrario, vaya al paso 4).

1. Haga clic en **Create Your First Rule**.
2. Elija la plantilla **empty rule**.
3. Introduzca un nombre para su nueva rule y haga clic en **Save**.
4. Vaya a [Dashboard &gt; Rules](https://manage.auth0.com/#/rules) y desplácese hasta la parte inferior de la página para establecer los valores de configuración (usaremos `log_url` como nombre de la clave y `https://someurl/log` como valor); luego haga clic en **Create**.
5. Ahora puede escribir su rule. Edite la rule que creó, introduzca el siguiente código en el área de código y haga clic en **Save**.

   ```javascript lines
   function(user, context, callback){
         var log_url = configuration.log_url;
         ...
       }
   ```

Este código es portable y, cuando migre a producción, solo tendrá que cambiar esta configuración en lugar de buscar en sus scripts.

<div id="adldap-connectors">
  ## Conectores AD/LDAP
</div>

Si usa varios tenants de Auth0 con AD/LDAP, deberá crear una conexión AD/LDAP y configurar un Conector AD/LDAP para cada tenant. Esto se debe a que cada Conector AD/LDAP está vinculado a una conexión específica dentro de un tenant de Auth0.

Varios Conectores AD/LDAP pueden apuntar al mismo directorio AD o LDAP, pero cada Conector AD/LDAP solo puede usarse con una conexión dentro de un tenant de Auth0.

Si tiene varios directorios AD/LDAP contra los que los usuarios se autenticarán (por ejemplo, para admitir distintos departamentos o clientes, cada uno con su propio directorio), puede configurar varios Conectores AD/LDAP dentro de cada tenant de Auth0.
