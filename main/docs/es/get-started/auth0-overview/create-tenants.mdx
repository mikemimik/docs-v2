Te guiaremos por los primeros pasos con Auth0 para que te familiarices con los conceptos clave del servicio. Usaremos la empresa **Example-Co** para ilustrar algunos de los pasos necesarios.

<div id="set-up-an-auth0-account">
  ## Configura una cuenta de Auth0
</div>

Si aún no te has [registrado](https://auth0.com/signup) para crear una **cuenta** de Auth0, hazlo (es gratis). Puedes usar nombre de usuario/correo electrónico/teléfono y contraseña, o iniciar sesión con un proveedor social (como LinkedIn, Microsoft, GitHub o Google).

<div id="create-a-tenant-and-domain">
  ## Crear un tenant y un dominio
</div>

Una vez que crees tu cuenta, se te pedirá que crees un **tenant**. Todo comienza con un tenant de Auth0. Aquí es donde configuras tu uso de Auth0 y donde se definen, administran y almacenan los recursos de Auth0, como las [aplicaciones](/docs/es/get-started/applications), las [conexiones](/docs/es/authenticate/identity-providers) y los [perfiles de usuario](/docs/es/get-started/architecture-scenarios/business-to-business/profile-management). Puedes acceder a un tenant de Auth0 a través del [Dashboard](/docs/es/get-started/auth0-overview/dashboard) de Auth0, donde también puedes crear tenants adicionales asociados. Puedes crear más de un tenant de Auth0 para organizar tus tenants de forma que aíslen distintos dominios de usuarios y, al mismo tiempo, respalden tu ciclo de vida de desarrollo de software (SDLC).

**Los nombres de los tenants no se pueden cambiar ni reutilizar una vez eliminados.** Así que asegúrate de estar conforme con el nombre o los nombres antes de crear tus tenants de Auth0.

Determinar el nivel de aislamiento que necesitas para tus dominios de usuario es un paso importante y, junto con tus requisitos de marca, te ayuda a definir cuántos tenants de Auth0 necesitas en tu entorno. La cantidad de tenants de Auth0 que necesitas administrar puede crecer rápidamente, así que piénsalo bien antes de crear varios tenants de Auth0 para producción.

Características del tenant:

* El nombre del tenant debe ser único. Se usará para crear tu dominio propio.
* El nombre del tenant solo puede contener caracteres alfanuméricos en minúscula y guiones (&quot;-&quot;). No puede comenzar ni terminar con un guion.
* El nombre del tenant debe tener un mínimo de 3 caracteres y un máximo de 63 caracteres.
* El nombre del tenant no se puede cambiar después de su creación.
* Puedes crear más de un tenant; de hecho, se recomienda hacerlo para cada entorno que puedas tener, como desarrollo, staging o producción. Para obtener más información, lee [Set Up Multiple Environments](/docs/es/get-started/auth0-overview/create-tenants/set-up-multiple-environments).

El nombre de tu tenant forma parte de tu dominio de Auth0. Puedes personalizarlo usando un <Tooltip tip="Custom Domain: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=custom+domain">dominio personalizado</Tooltip>. Este dominio es la URL base que se usa para acceder a la Auth0 API y la URL en la que se autentican tus usuarios.

<div id="region-locality-and-sub-locality">
  ### Región, localidad y sublocalidad
</div>

El nombre de dominio también se compone del valor de localidad de una **región**. Admitimos los siguientes valores de localidad para la [opción de despliegue de Public Cloud](/docs/es/deploy-monitor/deployment-options):

| Región                    | Localidad | Sublocalidades       |
| ------------------------- | --------- | -------------------- |
| Australia                 | AU        | AU                   |
| Canadá                    | CA        | CA                   |
| Europa                    | EU        | EU, EU-2             |
| Japón                     | JP        | JP                   |
| Reino Unido               | UK        | UK                   |
| Estados Unidos de América | US        | US, US-3, US-4, US-5 |

Cada una de estas localidades se divide en una sublocalidad (o **entorno del tenant**) con un dígito después de la localidad; por ejemplo, `EU-2`. Los entornos del tenant no pueden elegirse manualmente, pero sí pueden especificarse localidades según la región seleccionada, lo que determina el dominio del tenant asignado y la región en la que se alojarán tus datos.

En nuestro ejemplo, **Example-Co** eligió el nombre `example-co` y AU como su región. Por lo tanto, su dominio es `example-co.au.auth0.com`.

<div id="custom-domains">
  ## Dominios personalizados
</div>

Recomendamos usar dominios personalizados, como `example-co.com`, en entornos de producción para ofrecer a sus usuarios una experiencia más segura y fluida.

Los certificados de los dominios personalizados pueden ser administrados por Auth0 o gestionados por usted mismo.

Para obtener más información, consulte [Custom Domains](/docs/es/customize/custom-domains).

<div id="whats-next">
  ## Qué sigue
</div>

* **Crear y registrar aplicaciones**: Ahora que tienes una cuenta y un dominio, debes registrar cada aplicación que usará nuestros servicios en el [Auth0 Dashboard](https://manage.auth0.com/#/applications). Para obtener más información, consulta [Applications in Auth0](/docs/es/get-started/applications) y [Create Applications](/docs/es/get-started/auth0-overview/create-applications).
* **Configurar conexiones**: A continuación, debes definir cómo se autenticarán tus usuarios al iniciar sesión. Auth0 actúa como intermediario entre tu aplicación y el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> que autentica a tus usuarios (como Google o Facebook). La relación entre Auth0 y el proveedor de identidad se conoce como **conexión**. Al usar esta capa de conexión, Auth0 mantiene tu aplicación aislada de cualquier cambio que se produzca en la implementación del proveedor de identidad. Para obtener más información, consulta [Authentication and Authorization](/docs/es/get-started/identity-fundamentals/authentication-and-authorization) y [Connections](/docs/es/authenticate/identity-providers).

<div id="extend-auth0s-functionality">
  ## Amplía la funcionalidad de Auth0
</div>

Auth0 ofrece varias formas de ampliar la funcionalidad de la plataforma:

* [Actions](/docs/es/customize/actions): Actions son funciones seguras, específicas del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma de Auth0. Usa Actions para personalizar y ampliar las capacidades de Auth0 con un inicio de sesión personalizado.
* [Rules](/docs/es/customize/rules): Rules son funciones escritas en JavaScript o C# que se ejecutan en Auth0 justo después de una autenticación correcta y antes de que el control vuelva a tu aplicación. Rules se pueden encadenar para lograr un código modular y se pueden activar o desactivar de forma individual. Puedes usar Rules para:

  * Control de acceso
  * Webhooks
  * Enriquecimiento del perfil
  * Autenticación multifactor (MFA)
* [Hooks](/docs/es/customize/hooks): Hooks te permiten personalizar el comportamiento de Auth0 mediante código en Node.js que se ejecuta en puntos de extensibilidad (comparables a webhooks integrados en un servidor). Son funciones seguras y autocontenidas asociadas a puntos de extensibilidad específicos de la plataforma de Auth0. Auth0 invoca los Hooks en runtime para ejecutar tu lógica personalizada.
* [Extensions](/docs/es/customize/extensions): Las Extensions de Auth0 te permiten instalar aplicaciones o ejecutar comandos/scripts que amplían la funcionalidad básica de Auth0. Puedes usar una de las extensiones predefinidas proporcionadas por Auth0 o crear la tuya propia. Algunas de las acciones que puedes realizar con las extensiones incluyen:

  * Gestionar las autorizaciones de los usuarios (mediante grupos, roles y permisos)
  * Importar/exportar usuarios
  * Exportar registros a otros servicios
  * Desplegar scripts desde repositorios externos
