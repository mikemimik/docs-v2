En algún momento, es posible que necesite cambiar la información almacenada en el perfil de un usuario. El perfil de un usuario (también conocido como la cuenta del usuario) se almacena en Auth0, y puede ser necesario modificar la información que contiene por distintos motivos:

* Actualizaciones de información autogestionadas
* Actualizaciones obligatorias relacionadas con los términos y condiciones de sus organizaciones
* Cambios debidos al cumplimiento normativo

<Warning>
  No puede acceder directamente a un perfil de usuario entre varios tenants de Auth0. Si está implementando varios tenants de Auth0 en producción, es importante que lo tenga en cuenta.
</Warning>

Un [proveedor de identidad](/docs/es/authenticate/identity-providers) completa el perfil de un usuario con los datos proporcionados durante el proceso de inicio de sesión, y a esto se le denomina [perfil de usuario normalizado](/docs/es/manage-users/user-accounts/user-profiles/normalized-user-profiles).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El perfil de usuario normalizado se actualiza a partir del proveedor de identidad durante el inicio de sesión, y puede modificar el conjunto limitado de información que contiene a través de la Auth0 Management API. También puede usar la extensibilidad de Auth0, como [Actions](/docs/es/customize/actions), como alternativa para reemplazar información en el perfil de usuario normalizado. Consulte [Modificación de los datos del perfil de usuario](https://auth0.com/docs/manage-users/user-accounts/user-profiles#change-user-profile-data) para obtener más información.
</Callout>

De forma predeterminada, se crea un perfil de usuario para cada identidad de usuario, y hay varios aspectos que debe considerar:

* ¿Qué debe hacer si necesita almacenar información para ayudar a personalizar la experiencia de un usuario?
* ¿Qué ocurre si necesita almacenar información del usuario que no proviene de un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>?
* ¿Por qué necesitaría almacenar información relacionada con el usuario que este no puede modificar?
* ¿Qué debe hacer si necesita almacenar información relacionada con el usuario que este no puede modificar?
* ¿Qué sucede si un usuario olvida su contraseña?
* ¿Qué debe hacer un usuario si quiere cambiar su contraseña?

Auth0 permite almacenar metadatos en el perfil de un usuario, lo que permite recopilar información adicional, como preferencias de idioma o accesibilidad, con el fin de mejorar la experiencia del usuario. Los metadatos pueden utilizarse para almacenar tanto información que un usuario puede cambiar como información que no puede cambiar; esto último le da la posibilidad de asociar, por ejemplo, un perfil de usuario con registros de sus sistemas existentes sin modificar la implementación existente.

Para los usuarios que olvidan sus contraseñas o a quienes se les permite cambiarla mediante algún mecanismo de autoservicio existente (o uno que tenga previsto implementar), puede aprovechar la funcionalidad de [Restablecimiento de contraseña](#password-reset) proporcionada por Auth0. Esta puede integrarse con su implementación existente y ya viene incorporada en cualquiera de los [widgets de interfaz de usuario](#) de Auth0 listos para usar, incluido [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

También querrá asegurarse de trabajar en todo momento con una [cuenta de usuario verificada](#account-verification). Auth0 también proporciona mecanismos listos para usar para ello. También debe considerar el [cumplimiento normativo](/docs/es/secure/data-privacy-and-compliance), como el [RGPD](https://eugdpr.org/), que establece requisitos muy específicos en lo que respecta a la protección de la privacidad de los ciudadanos de la UE y a las violaciones de la seguridad de los datos.

Aunque Auth0 actualmente no proporciona de forma predeterminada un portal centralizado de gestión de perfiles, para la autogestión de perfiles puede usar la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 para crear el suyo propio o utilizar una interfaz ya creada. Consulte nuestra [guía de la comunidad](https://community.auth0.com/t/how-to-allow-the-end-user-to-update-their-own-profile-information/6228) de Auth0, que describe el endpoint de la Management API. Todas las llamadas a la Management API requerirán el uso de un [token de acceso](/docs/es/secure/tokens/access-tokens).

<Warning>
  La autogestión de perfiles puede plantear problemas tanto de seguridad como de privacidad de los datos. Por ejemplo, puede que quiera permitir que un usuario cambie su dirección de correo electrónico; sin embargo, hacerlo sin seguir una guía de seguridad de práctica recomendada podría provocar que un usuario se quede sin acceso a su cuenta, exponga información de identificación personal (PII) o, peor aún, abra una posible brecha de seguridad.
</Warning>

Como alternativa, puede usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para [gestionar aspectos del perfil de un usuario](/docs/es/manage-users/user-accounts/manage-users-using-the-dashboard). Gestionar el perfil de un usuario a través del Auth0 Dashboard es más bien una tarea administrativa y **no debe** usarse para la gestión de perfiles en autoservicio en un entorno de producción. Sin embargo, la interfaz proporcionada por el Dashboard puede ser extremadamente útil durante el desarrollo, ya que ofrece una forma rápida y sencilla de modificar la información del perfil de un usuario.

<div id="metadata">
  ## Metadatos
</div>

Además de la información del Perfil de Usuario Normalizado, se pueden almacenar metadatos en un perfil de usuario de Auth0. Los metadatos permiten almacenar información que no proviene de un proveedor de identidad, o información que reemplaza la que proporciona un proveedor de identidad.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El uso de metadatos debe seguir las [prácticas recomendadas de almacenamiento de datos de usuario](/docs/es/secure/security-guidance/data-security/user-data-storage#metadata) de Auth0. El almacenamiento de metadatos no está diseñado para ser un almacén de datos de uso general, por lo que, siempre que sea posible, debes usar tu propio sistema de almacenamiento externo. El tamaño y la complejidad de los metadatos también deben mantenerse al mínimo, y la Auth0 Management API tiene un conjunto estricto de directrices en lo que respecta a la actualización y/o eliminación de metadatos asociados a un usuario.
</Callout>

Puedes manipular metadatos mediante la Auth0 Management API y la Auth0 Authentication API. Al igual que ocurre al administrar el Perfil de Usuario Normalizado, las solicitudes a la Management API para manipular metadatos requieren el uso de un [token de acceso](https://auth0.com/docs/api/management/v2/tokens).

<Warning>
  Las solicitudes a la Management API están sujetas a la [política de límite de frecuencia de Auth0](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy). Debes tener esto en cuenta y, para facilitarlo, Auth0 generalmente recomienda usar el [SDK de Auth0](/docs/es/libraries) adecuado para tu entorno de desarrollo en lugar de llamar directamente a nuestras API.
</Warning>

<div id="user-metadata">
  ### Metadatos de usuario
</div>

Los metadatos de usuario (también llamados `user_metadata`) son información que puede almacenarse en un perfil de usuario y que el usuario puede leer y actualizar como parte de cualquier proceso de gestión del perfil de autoservicio. Este tipo de metadatos puede incluir, por ejemplo, el tratamiento del usuario o su idioma preferido, que podría usarse para [personalizar los correos electrónicos](/docs/es/customize/email/email-templates) enviados por Auth0.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Guarda en los metadatos toda la información que quieras usar para personalizar los correos electrónicos de Auth0, y de preferencia en `user_metadata` si el usuario puede modificarla, como la información utilizada para determinar el idioma de un correo electrónico.
</Callout>

<div id="app-metadata">
  ### Metadatos de la aplicación
</div>

Los metadatos de la aplicación (también llamados `app_metadata`) son, por otro lado, información que puede almacenarse en un perfil de usuario, pero a la que **solo se puede acceder o actualizar con la autorización adecuada**; `app_metadata` no es directamente accesible para el usuario. Este tipo de metadatos podría incluir, por ejemplo, un indicador de que el usuario aceptó la última versión vigente de los términos y condiciones, así como la fecha en que los aceptó.

<div id="password-reset">
  ## Restablecimiento de contraseña
</div>

Para los usuarios que olvidan su contraseña o que pueden cambiarla mediante algún mecanismo de autoservicio existente, Auth0 proporciona la funcionalidad de [Restablecimiento de contraseña](/docs/es/authenticate/database-connections/password-change). Puedes integrarla con tu implementación actual y ya está incorporada en los widgets de interfaz de usuario de Auth0 listos para usar, incluidos como parte de [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

<Warning>
  El cambio y el restablecimiento de contraseña solo son compatibles con los tipos de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) de Auth0.
</Warning>

Auth0 <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> ofrece soporte de experiencia de usuario integrado para el restablecimiento de contraseña mediante la funcionalidad de Auth0 Authentication API. Como alternativa, puedes usar la [Auth0 Authentication API](/docs/es/authenticate/database-connections/password-change#authentication-api) a través de uno de los SDKs de Auth0 adecuados para tu entorno de desarrollo. Las plantillas de correo electrónico que se usan durante el flujo de trabajo de restablecimiento de contraseña también se pueden personalizar por completo, tanto si usas los widgets de interfaz de usuario de Auth0 listos para usar como si usas Universal Login personalizado.

Por otro lado, puedes usar Auth0 Management API para [cambiar directamente la contraseña](/docs/es/authenticate/database-connections/password-change#directly-set-the-new-password) de una identidad de usuario definida mediante un tipo de conexión de base de datos. También puedes usar Auth0 Management API como parte de cualquier implementación de gestión de perfiles de autoservicio y de cualquier [personalización de la página Change Password](/docs/es/get-started/architecture-scenarios/business-to-consumer/branding).

<div id="account-verification">
  ## Verificación de la cuenta
</div>

También tendrás que trabajar en todo momento con una cuenta de usuario verificada y utilizar los mecanismos que proporciona Auth0. Asimismo, debes tener en cuenta el cumplimiento normativo, como el [RGPD](https://eugdpr.org/), que establece requisitos muy específicos para proteger la privacidad de los ciudadanos de la UE y prevenir violaciones de la seguridad de los datos.

Auth0 ofrece funcionalidad lista para usar para enviar un [correo electrónico de verificación](/docs/es/customize/email/manage-email-flow) a la dirección de correo electrónico del usuario y verificar su cuenta. De forma predeterminada, Auth0 envía automáticamente correos electrónicos de verificación para cualquier identidad de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) creada como parte del [autoregistro](/docs/es/get-started/architecture-scenarios/business-to-consumer/provisioning#self-sign-up). Sin embargo, Auth0 también proporciona un [endpoint de la Management API](https://auth0.com/docs/api/v2#!/Tickets/post_email_verification) que puedes usar para enviar correos electrónicos de verificación en los casos en que un proveedor social no valida la dirección de correo electrónico durante el registro del usuario.

<div id="blocking-users">
  ## Bloqueo de usuarios
</div>

[El bloqueo del acceso de los usuarios](/docs/es/manage-users/user-accounts/block-and-unblock-users) en Auth0 ofrece una forma de impedir que los usuarios inicien sesión en aplicaciones bajo determinadas condiciones. De forma predeterminada, el Auth0 Dashboard ofrece un mecanismo listo para usar que permite a los administradores bloquear y desbloquear el acceso de los usuarios a todas las aplicaciones, y puedes implementar esta funcionalidad mediante el uso de la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id). También puedes usar la extensibilidad de Auth0 para [deshabilitar el acceso de los usuarios a determinadas aplicaciones](/docs/es/manage-users/user-accounts/manage-user-access-to-applications), así como proporcionar un [control de acceso](/docs/es/get-started/architecture-scenarios/business-to-consumer/authorization) más detallado.

Además, la Auth0 Management API te permite [desbloquear](https://auth0.com/docs/api/management/v2#!/User_Blocks/delete_user_blocks_by_id) a los usuarios deshabilitados debido al uso excesivo de credenciales incorrectas.

<div id="linking-user-accounts">
  ## Vinculación de cuentas de usuario
</div>

De forma predeterminada, hay un perfil de usuario (cuenta de usuario) para cada identidad de usuario. Si habilita el inicio de sesión desde varios proveedores de identidad —mediante Facebook o Google [autenticación social](/docs/es/get-started/architecture-scenarios/business-to-consumer/authentication#social-authentication) y también mediante Auth0 [autenticación con nombre de usuario y contraseña](/docs/es/get-started/architecture-scenarios/business-to-consumer/authentication#username-and-password-authentication)—, cada uno tendrá un perfil de usuario independiente. Puede usar la funcionalidad de Auth0 para [vincular cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking) a fin de crear un único perfil para un usuario que agrupe todas sus identidades asociadas.

El proceso de vinculación de cuentas fusiona perfiles de usuario de dos en dos: en el proceso de vinculación se deben especificar una cuenta principal y una cuenta secundaria. Sin embargo, la cantidad de cuentas que se pueden vincular va más allá de un solo par. Por ejemplo, puede usar como principal una cuenta que ya tenga varias cuentas vinculadas y añadirle una cuenta secundaria adicional. Esto significa que una cuenta de usuario puede tener múltiples identidades asociadas, lo que ofrece varias ventajas:

* Los usuarios pueden iniciar sesión con múltiples identidades sin crear un perfil independiente para cada una.
* Los usuarios registrados pueden usar nuevas identidades de inicio de sesión y seguir utilizando su perfil existente.
* Los usuarios pueden conservar su perfil independientemente de la identidad que usen para iniciar sesión.
* Los usuarios pueden vincularse a una cuenta con más información de identidad para obtener un perfil más completo.
* Sus aplicaciones pueden recuperar datos del perfil de usuario específicos de la conexión.

<div id="de-provisioning">
  ## Desaprovisionamiento
</div>

Es posible que tu aplicación deba permitir que un usuario solicite la eliminación de su cuenta (por ejemplo, para cumplir los requisitos del [RGPD](https://eugdpr.org/)). Puedes implementar esta funcionalidad, junto con otras funciones relacionadas con el perfil, mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Users). La Management API te permite recuperar la información almacenada sobre un usuario y actualizarla según sea necesario.

Auth0 puede satisfacer diversos requisitos relacionados con la privacidad, incluida la visualización de enlaces a avisos de consentimiento durante el registro y la protección de datos para respaldar el derecho de los usuarios a ver y corregir los datos que has recopilado sobre ellos.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El [RGPD](https://eugdpr.org/) y otras directivas de privacidad exigen que los usuarios tengan derecho a ver y corregir los datos que se conservan sobre ellos. También tienen derecho al “olvido”. Puedes usar la Management API para abordar estos requisitos y cumplir tus obligaciones legales.
</Callout>
