En algún momento, puede que necesite cambiar la información almacenada en el perfil de un usuario. El perfil de un usuario (también conocido como la cuenta del usuario) se almacena en Auth0, y puede que sea necesario modificar la información que contiene por distintos motivos:

* Actualizaciones de información en autoservicio
* Actualizaciones obligatorias relacionadas con los términos y condiciones de sus organizaciones
* Cambios debidos al cumplimiento normativo

<Warning>
  No puede acceder directamente al perfil de un usuario en varios tenants de Auth0. Si está implementando varios tenants de Auth0 en producción, esto es algo que debe tener en cuenta.
</Warning>

Un [proveedor de identidad](/docs/es/authenticate/identity-providers) completa el perfil de un usuario con los datos proporcionados durante el proceso de inicio de sesión, y a esto se le denomina [Perfil de usuario normalizado](/docs/es/manage-users/user-accounts/user-profiles/normalized-user-profiles).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El Perfil de usuario normalizado se actualiza a partir del proveedor de identidad durante el inicio de sesión, y puede modificar el conjunto limitado de información que contiene mediante la Auth0 Management API. También puede usar la extensibilidad de Auth0, como [Actions](/docs/es/customize/actions), como alternativa para sobrescribir información del Perfil de usuario normalizado. Consulte [Modificación de datos del perfil de usuario](https://auth0.com/docs/manage-users/user-accounts/user-profiles#change-user-profile-data) para obtener más información.
</Callout>

De forma predeterminada, se crea un perfil de usuario para cada identidad de usuario, y hay varios aspectos que debe tener en cuenta:

* ¿Qué debería hacer si necesita almacenar información para personalizar la experiencia de un usuario?
* ¿Qué ocurre si necesita almacenar información de usuario que no proviene de un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>?
* ¿Por qué necesitaría almacenar información relacionada con el usuario que este no puede modificar?
* ¿Qué debe hacer si necesita almacenar información relacionada con el usuario que este no puede modificar?
* ¿Qué ocurre si un usuario olvida su contraseña?
* ¿Qué debería hacer un usuario si quiere cambiar su contraseña?

Auth0 permite almacenar metadatos en el perfil de un usuario, lo que permite recopilar información adicional, como la preferencia de idioma y/o accesibilidad, para mejorar la experiencia del usuario. Los metadatos pueden usarse para almacenar tanto información que un usuario puede cambiar como información que no puede cambiar; esto último le permite, por ejemplo, asociar un perfil de usuario con registros de sus sistemas existentes sin modificar la implementación actual.

Para los usuarios que olvidan sus contraseñas o a quienes se les permite cambiarla mediante algún mecanismo de autoservicio existente (o uno de autoservicio que tenga previsto), puede aprovechar la funcionalidad de [Restablecimiento de contraseña](#password-reset) que proporciona Auth0. Esta puede integrarse con su implementación existente y ya viene incorporada en cualquier widget de interfaz de usuario de Auth0 listo para usar, incluido [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

También querrá asegurarse de trabajar en todo momento con una [cuenta de usuario verificada](#account-verification). Auth0 también proporciona mecanismos listos para usar para ello. También debería tener en cuenta el [cumplimiento normativo](/docs/es/secure/data-privacy-and-compliance), como el [RGPD](https://eugdpr.org/), que establece requisitos muy específicos cuando se trata de proteger la privacidad de los ciudadanos de la UE y evitar violaciones de la seguridad de los datos.

Aunque Auth0 actualmente no ofrece un portal centralizado de gestión de perfiles listo para usar, para la gestión de perfiles en autoservicio puede usar la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 para crear el suyo propio o utilizar una interfaz ya creada. Consulte nuestra [guía de la comunidad](https://community.auth0.com/t/how-to-allow-the-end-user-to-update-their-own-profile-information/6228) de Auth0, que describe el endpoint de la Management API. Todas las llamadas a la Management API requerirán el uso de un [token de acceso](/docs/es/secure/tokens/access-tokens).

<Warning>
  La gestión de perfiles en autoservicio puede plantear problemas tanto de seguridad como de privacidad de los datos. Por ejemplo, puede que quiera permitir que un usuario cambie su dirección de correo electrónico; sin embargo, hacerlo sin seguir las directrices de seguridad de práctica recomendada podría provocar que un usuario se quede sin acceso a su cuenta, filtre información de identificación personal (PII) o, peor aún, abra una posible brecha de seguridad.
</Warning>

Como alternativa, puedes usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para [administrar aspectos del perfil de un usuario](/docs/es/manage-users/user-accounts/manage-users-using-the-dashboard). Administrar el perfil de un usuario mediante el Auth0 Dashboard responde más a una función administrativa y **no debe** usarse para la gestión de perfiles por autoservicio en un entorno de producción. Sin embargo, la interfaz que proporciona el Dashboard puede ser extremadamente útil durante el desarrollo, ya que ofrece una forma rápida y sencilla de modificar la información del perfil de un usuario.

<div id="metadata">
  ## Metadatos
</div>

Además de la información del Perfil de Usuario Normalizado, se pueden almacenar metadatos en un perfil de usuario de Auth0. Los metadatos permiten almacenar información que no proviene de un proveedor de identidad, o información que reemplaza la que proporciona un proveedor de identidad.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El uso de metadatos debe seguir las [prácticas recomendadas para el almacenamiento de datos de usuario](/docs/es/secure/security-guidance/data-security/user-data-storage#metadata) de Auth0. El almacenamiento de metadatos no está pensado como un almacén de datos de uso general, y, siempre que sea posible, debes seguir usando tu propio sistema de almacenamiento externo. El tamaño y la complejidad de los metadatos también deben mantenerse al mínimo, y la Auth0 Management API establece directrices estrictas para actualizar y/o eliminar metadatos asociados a un usuario.
</Callout>

Puedes manipular metadatos tanto mediante la Auth0 Management API como mediante la Auth0 Authentication API. Al igual que ocurre al administrar el Perfil de Usuario Normalizado, las llamadas a la Management API para manipular metadatos requieren el uso de un [token de acceso](https://auth0.com/docs/api/management/v2/tokens).

<Warning>
  Las llamadas a la Management API están sujetas a la [política de límite de frecuencia de Auth0](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy). Debes tenerlo en cuenta y, para ayudarte, Auth0 generalmente recomienda usar el [SDK de Auth0](/docs/es/libraries) adecuado para tu entorno de desarrollo en lugar de llamar directamente a nuestras API.
</Warning>

<div id="user-metadata">
  ### Metadatos del usuario
</div>

Los metadatos del usuario (también llamados `user_metadata`) son información que puede almacenarse en el perfil de un usuario y que este puede leer y actualizar como parte de cualquier proceso de gestión de perfiles de autoservicio. Estos metadatos pueden incluir, por ejemplo, el tratamiento de un usuario o su idioma preferido, que podría utilizarse para [personalizar los correos electrónicos](/docs/es/customize/email/email-templates) enviados por Auth0.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Guarda en los metadatos toda la información que quieras usar para personalizar los correos electrónicos de Auth0 y, preferiblemente, en `user_metadata` si el usuario puede modificarla, como la información utilizada para determinar el idioma de un correo electrónico.
</Callout>

<div id="app-metadata">
  ### Metadatos de la aplicación
</div>

Los metadatos de la aplicación (también denominados `app_metadata`) son, por otro lado, información que puede almacenarse junto con un perfil de usuario, pero que **solo puede leerse o actualizarse con la autorización adecuada**; `app_metadata` no es directamente accesible para el usuario. Este tipo de metadatos podría incluir, por ejemplo, un indicador de que el usuario aceptó la última versión vigente de los términos y condiciones, así como la fecha en que los aceptó.

<div id="password-reset">
  ## Restablecimiento de contraseña
</div>

Para los usuarios que olvidan su contraseña o que pueden cambiarla mediante algún mecanismo de autoservicio existente, Auth0 ofrece la funcionalidad de [Restablecimiento de contraseña](/docs/es/authenticate/database-connections/password-change). Puede integrarla con su implementación actual, y ya está incorporada en los widgets de interfaz de usuario de Auth0 listos para usar incluidos como parte de [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

<Warning>
  El cambio y el restablecimiento de contraseña solo son compatibles con los tipos de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) de Auth0.
</Warning>

Auth0 <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> ofrece soporte de experiencia de usuario integrado para el restablecimiento de contraseña mediante la funcionalidad de Auth0 Authentication API. Como alternativa, puede usar la [Auth0 Authentication API](/docs/es/authenticate/database-connections/password-change#authentication-api) a través de uno de los Auth0 SDKs adecuados para su entorno de desarrollo. Las plantillas de correo electrónico que se usan durante el flujo de trabajo de restablecimiento de contraseña también se pueden personalizar por completo, tanto si usa los widgets de interfaz de usuario listos para usar de Auth0 como si usa un Universal Login personalizado.

Por otro lado, puede usar Auth0 Management API para [cambiar directamente la contraseña](/docs/es/authenticate/database-connections/password-change#directly-set-the-new-password) de una identidad de usuario definida mediante un tipo de conexión de base de datos. Puede usar Auth0 Management API como parte de cualquier implementación de gestión de perfiles de autoservicio y también como parte de cualquier [personalización de la página Change Password](/docs/es/get-started/architecture-scenarios/business-to-consumer/branding).

<div id="account-verification">
  ## Verificación de la cuenta
</div>

También necesitarás trabajar en todo momento con una cuenta de usuario verificada y utilizar los mecanismos que proporciona Auth0. También debes tener en cuenta el cumplimiento normativo, como el [RGPD](https://eugdpr.org/), que establece requisitos muy específicos para proteger la privacidad de los ciudadanos de la UE y evitar violaciones de la seguridad de los datos.

Auth0 proporciona funcionalidad lista para usar para enviar un [correo electrónico de verificación](/docs/es/customize/email/manage-email-flow) a la dirección de correo electrónico de un usuario y así verificar su cuenta. De forma predeterminada, Auth0 envía automáticamente correos electrónicos de verificación para cualquier identidad de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) creada como parte del [autoregistro](/docs/es/get-started/architecture-scenarios/business-to-consumer/provisioning#self-sign-up). Sin embargo, Auth0 también proporciona un [endpoint de la Management API](https://auth0.com/docs/api/v2#!/Tickets/post_email_verification) que puedes usar para enviar correos electrónicos de verificación en los casos en que un proveedor social no valide la dirección de correo electrónico durante el registro del usuario.

<div id="blocking-users">
  ## Bloqueo de usuarios
</div>

[Bloquear el acceso de usuarios](/docs/es/manage-users/user-accounts/block-and-unblock-users) en Auth0 ofrece una forma de impedir que los usuarios inicien sesión en las aplicaciones en determinadas circunstancias. De forma predeterminada, el Auth0 Dashboard proporciona un mecanismo listo para usar que permite a los administradores bloquear y desbloquear el acceso de los usuarios a todas las aplicaciones, y puedes implementar esta funcionalidad mediante la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id). También puedes usar la extensibilidad de Auth0 para [deshabilitar el acceso de usuarios a determinadas aplicaciones](/docs/es/manage-users/user-accounts/manage-user-access-to-applications), así como para proporcionar un [control de acceso](/docs/es/get-started/architecture-scenarios/business-to-consumer/authorization) más detallado.

Además, la Auth0 Management API te permite [desbloquear](https://auth0.com/docs/api/management/v2#!/User_Blocks/delete_user_blocks_by_id) a usuarios deshabilitados debido al uso excesivo de credenciales incorrectas.

<div id="linking-user-accounts">
  ## Vinculación de cuentas de usuario
</div>

De forma predeterminada, hay un perfil de usuario (cuenta de usuario) para cada identidad de usuario. Si habilita el inicio de sesión con varios proveedores de identidad, a través de Facebook o Google [autenticación social](/docs/es/get-started/architecture-scenarios/business-to-consumer/authentication#social-authentication) y también a través de Auth0 [autenticación con nombre de usuario y contraseña](/docs/es/get-started/architecture-scenarios/business-to-consumer/authentication#username-and-password-authentication), cada una tendrá un perfil de usuario independiente. Puede usar la funcionalidad de Auth0 para la [vinculación de cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking) y crear un único perfil para un usuario que reúna todas sus identidades asociadas.

El proceso de vinculación de cuentas fusiona perfiles de usuario de dos en dos: en el proceso de vinculación se debe especificar una cuenta principal y una cuenta secundaria. Sin embargo, la cantidad de cuentas que se pueden vincular va más allá de un solo par. Por ejemplo, puede usar como principal una cuenta que ya tenga varias cuentas fusionadas con ella y vincularle una cuenta secundaria adicional. Esto significa que una cuenta de usuario puede tener varias identidades asociadas, lo que ofrece varias ventajas:

* Los usuarios pueden iniciar sesión con varias identidades sin crear un perfil independiente para cada una.
* Los usuarios registrados pueden usar nuevas identidades de inicio de sesión y seguir usando su perfil existente.
* Los usuarios pueden conservar su perfil independientemente de la identidad que usen para iniciar sesión.
* Los usuarios pueden vincularse a una cuenta con más información de identidad para tener un perfil más completo.
* Sus aplicaciones pueden recuperar datos del perfil de usuario específicos de la conexión.

<div id="de-provisioning">
  ## Desaprovisionamiento
</div>

Es posible que tu aplicación necesite admitir la solicitud de un usuario para eliminar su cuenta (por ejemplo, puede que debas cumplir los requisitos del [RGPD](https://eugdpr.org/)). Puedes implementar esta funcionalidad, junto con otras funciones relacionadas con el perfil, mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Users). La Management API te permite recuperar la información almacenada sobre un usuario y actualizarla según sea necesario.

Auth0 puede dar soporte a diversos requisitos relacionados con la privacidad, incluida la visualización de enlaces a avisos de consentimiento durante el registro y la protección de datos para respaldar el derecho de los usuarios a ver y corregir los datos que has recopilado sobre ellos.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El [RGPD](https://eugdpr.org/) y otras directivas de privacidad exigen que los usuarios tengan derecho a ver y corregir los datos conservados sobre ellos. También tienen derecho a ser “olvidados”. Puedes usar la Management API para abordar estos requisitos y cumplir tus obligaciones legales.
</Callout>
