En algún momento, es posible que necesite cambiar la información almacenada en el perfil de un usuario. El perfil de un usuario (también conocido como la cuenta del usuario) se almacena en Auth0, y puede que sea necesario modificar la información que contiene por distintos motivos:

* Actualizaciones de información de autoservicio
* Actualizaciones obligatorias relacionadas con los Términos y Condiciones de sus organizaciones
* Cambios debidos al cumplimiento normativo

<Warning>
  No puede acceder directamente al perfil de un usuario en varios tenants de Auth0. Si va a implementar varios tenants de Auth0 en producción, esto es algo que debe tener en cuenta.
</Warning>

Un [proveedor de identidad](/docs/es/authenticate/identity-providers) completa el perfil de un usuario con los datos proporcionados durante el proceso de inicio de sesión, y a esto se le denomina [Perfil de usuario normalizado](/docs/es/manage-users/user-accounts/user-profiles/normalized-user-profiles).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El Perfil de usuario normalizado se actualiza desde el proveedor de identidad durante el inicio de sesión, y puede modificar el conjunto limitado de información que contiene a través de la Auth0 Management API. También puede usar la extensibilidad de Auth0, como [Actions](/docs/es/customize/actions), como alternativa para reemplazar información del Perfil de usuario normalizado. Consulte [Modificación de datos del perfil de usuario](https://auth0.com/docs/manage-users/user-accounts/user-profiles#change-user-profile-data) para obtener más información.
</Callout>

De forma predeterminada, se crea un perfil de usuario para cada identidad de usuario, y hay varias cosas que debe considerar:

* ¿Qué debería hacer si necesita almacenar información para ayudar a personalizar la experiencia de un usuario?
* ¿Qué ocurre si necesita almacenar información del usuario que no procede de un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>?
* ¿Por qué necesitaría almacenar información relacionada con un usuario que este no puede modificar?
* ¿Qué debe hacer si necesita almacenar información relacionada con un usuario que este no puede modificar?
* ¿Qué sucede si un usuario olvida su contraseña?
* ¿Qué debería hacer un usuario si quiere cambiar su contraseña?

Auth0 permite almacenar metadatos en el perfil de un usuario, lo que permite recopilar información adicional, como preferencias de idioma y/o accesibilidad, para mejorar la experiencia del usuario. Los metadatos pueden usarse para almacenar tanto información que un usuario puede cambiar como información que no puede cambiar; esto último le permite asociar, por ejemplo, un perfil de usuario con registros de sus sistemas existentes sin modificar la implementación actual.

Para los usuarios que olvidan sus contraseñas o a quienes se les permite cambiarla mediante algún mecanismo de autoservicio existente (o uno que tenga previsto), puede aprovechar la funcionalidad de [Restablecimiento de contraseña](#password-reset) proporcionada por Auth0. Esto puede integrarse con su implementación existente y ya viene incorporado en cualquier widget de interfaz de usuario de Auth0 listo para usar, incluido [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

También querrá asegurarse de estar trabajando en todo momento con una [cuenta de usuario verificada](#account-verification). Auth0 también proporciona mecanismos listos para usar para hacerlo. También debe considerar el [cumplimiento normativo](/docs/es/secure/data-privacy-and-compliance), como el ([RGPD](https://eugdpr.org/), que tiene requisitos muy específicos cuando se trata de proteger a los ciudadanos de la UE en materia de privacidad y de violación de la seguridad de los datos.

Aunque actualmente Auth0 no proporciona de forma predeterminada un portal centralizado de gestión de perfiles, para fines de gestión de perfiles de autoservicio puede usar la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 para crear el suyo o utilizar una interfaz ya creada. Consulte nuestra [guía de la comunidad](https://community.auth0.com/t/how-to-allow-the-end-user-to-update-their-own-profile-information/6228), que describe el endpoint de la Management API. Todas las llamadas a la Management API requerirán el uso de un [token de acceso](/docs/es/secure/tokens/access-tokens).

<Warning>
  La gestión de perfiles de autoservicio puede plantear problemas tanto de seguridad como de privacidad de los datos. Por ejemplo, puede que quiera permitir que un usuario cambie su dirección de correo electrónico; sin embargo, hacerlo sin seguir las prácticas recomendadas de seguridad podría provocar que un usuario pierda el acceso a su cuenta, filtre información de identificación personal (PII) o, peor aún, abra una posible brecha de seguridad.
</Warning>

Como alternativa, puedes usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para [gestionar aspectos del perfil de un usuario](/docs/es/manage-users/user-accounts/manage-users-using-the-dashboard). Gestionar el perfil de un usuario desde el Auth0 Dashboard es más bien una tarea administrativa y **no debería** usarse para la gestión de perfiles en autoservicio en un entorno de producción. Sin embargo, la interfaz que ofrece el Dashboard puede ser extremadamente útil durante el desarrollo, ya que proporciona una forma rápida y sencilla de modificar la información del perfil de un usuario.

<div id="metadata">
  ## Metadatos
</div>

Además de la información del perfil de usuario normalizado, en un perfil de usuario de Auth0 también se pueden almacenar metadatos. Los metadatos permiten almacenar información que no proviene de un proveedor de identidad, o guardar información que sobrescribe la que proporciona un proveedor de identidad.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El uso de metadato debe seguir las [prácticas recomendadas para el almacenamiento de datos de usuario](/docs/es/secure/security-guidance/data-security/user-data-storage#metadata) de Auth0. El almacenamiento de metadatos no está pensado como un almacén de datos de uso general, por lo que, siempre que sea posible, debe seguir utilizando su propio sistema de almacenamiento externo. Además, el tamaño y la complejidad de los metadatos deben mantenerse al mínimo, y la Auth0 Management API tiene directrices estrictas en lo que respecta a la actualización y/o eliminación de los metadatos asociados a un usuario.
</Callout>

Puede manipular metadatos tanto a través de la Auth0 Management API como de la Auth0 Authentication API. Al igual que ocurre al gestionar el perfil de usuario normalizado, las llamadas a la Management API para manipular metadatos requieren el uso de un [token de acceso](https://auth0.com/docs/api/management/v2/tokens).

<Warning>
  Las llamadas a la Management API están sujetas a la [política de límite de frecuencia de Auth0](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy). Debe tenerlo en cuenta y, para facilitarlo, Auth0 generalmente recomienda usar el [SDK de Auth0](/docs/es/libraries) adecuado para su entorno de desarrollo en lugar de llamar directamente a nuestras API.
</Warning>

<div id="user-metadata">
  ### Metadatos de usuario
</div>

Los metadatos de usuario (también llamados `user_metadata`) son información que puede almacenarse en el perfil de un usuario y que este puede leer y actualizar como parte de cualquier proceso de gestión de perfiles de autoservicio. Este tipo de metadatos puede incluir, por ejemplo, el tratamiento del usuario o su idioma preferido, que podría usarse para [personalizar los correos electrónicos](/docs/es/customize/email/email-templates) enviados por Auth0.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Almacena en los metadatos cualquier información que quieras usar para personalizar los correos electrónicos de Auth0, y preferiblemente en `user_metadata` si el usuario puede cambiarla, como la información que se usa para determinar el idioma de un correo electrónico.
</Callout>

<div id="app-metadata">
  ### Metadatos de la aplicación
</div>

Los metadatos de la aplicación (también denominados `app_metadata`) son, por otro lado, información que puede almacenarse en un perfil de usuario, pero que **solo puede leerse o actualizarse con la autorización adecuada**; `app_metadata` no es directamente accesible para el usuario. Este tipo de metadatos podría ser, por ejemplo, un indicador de que el usuario aceptó la última versión válida de los términos y condiciones, y una fecha que indique cuándo los aceptó.

<div id="password-reset">
  ## Restablecimiento de contraseña
</div>

Para los usuarios que olvidan sus contraseñas o que pueden cambiar su contraseña mediante algún mecanismo de autoservicio existente, Auth0 proporciona la funcionalidad de [Restablecimiento de contraseña](/docs/es/authenticate/database-connections/password-change). Puede integrarla con su implementación actual y ya viene incorporada en los widgets de interfaz de usuario de Auth0 listos para usar incluidos como parte de [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

<Warning>
  El cambio de contraseña y el restablecimiento de contraseña solo son compatibles con los tipos de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) de Auth0.
</Warning>

Auth0 <Tooltip tip="Universal Login: su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> proporciona soporte integrado de experiencia de usuario para el restablecimiento de contraseña mediante la funcionalidad de Auth0 Authentication API. Como alternativa, puede usar la [Auth0 Authentication API](/docs/es/authenticate/database-connections/password-change#authentication-api) a través de uno de los Auth0 SDKs adecuados para su entorno de desarrollo. Las plantillas de correo electrónico utilizadas durante el flujo de restablecimiento de contraseña también se pueden personalizar por completo, tanto si usa los widgets de interfaz de usuario de Auth0 listos para usar como si usa un Universal Login personalizado.

Por otro lado, puede usar Auth0 Management API para [cambiar directamente la contraseña](/docs/es/authenticate/database-connections/password-change#directly-set-the-new-password) de una identidad de usuario definida mediante un tipo de conexión de base de datos. Puede usar Auth0 Management API como parte de cualquier implementación de gestión de perfiles de autoservicio y también como parte de cualquier [personalización de la página Change Password](/docs/es/get-started/architecture-scenarios/business-to-consumer/branding).

<div id="account-verification">
  ## Verificación de la cuenta
</div>

También tendrás que trabajar en todo momento con una cuenta de usuario verificada y aprovechar los mecanismos que ofrece Auth0. También debes tener en cuenta el cumplimiento normativo, como el [RGPD](https://eugdpr.org/), que establece requisitos muy específicos para proteger la privacidad de los ciudadanos de la UE y prevenir violaciones de la seguridad de los datos.

Auth0 ofrece funcionalidades listas para usar para enviar un [correo de verificación](/docs/es/customize/email/manage-email-flow) a la dirección de correo electrónico del usuario y verificar su cuenta. De forma predeterminada, Auth0 envía automáticamente correos de verificación para cualquier identidad de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) creada como parte del [autorregistro](/docs/es/get-started/architecture-scenarios/business-to-consumer/provisioning#self-sign-up). Sin embargo, Auth0 también ofrece un [endpoint de la Management API](https://auth0.com/docs/api/v2#!/Tickets/post_email_verification) que puedes usar para enviar correos de verificación en los casos en que un proveedor social no valide la dirección de correo electrónico durante el registro del usuario.

<div id="blocking-users">
  ## Bloqueo de usuarios
</div>

[El bloqueo del acceso de los usuarios](/docs/es/manage-users/user-accounts/block-and-unblock-users) en Auth0 ofrece una forma de impedir que los usuarios inicien sesión en las aplicaciones en determinadas circunstancias. De forma predeterminada, el Auth0 Dashboard proporciona un mecanismo listo para usar que permite a los administradores bloquear y desbloquear el acceso de los usuarios a todas las aplicaciones, y también puedes implementar esta funcionalidad mediante la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id). También puedes usar la extensibilidad de Auth0 para [deshabilitar el acceso de los usuarios a determinadas aplicaciones](/docs/es/manage-users/user-accounts/manage-user-access-to-applications), así como ofrecer un [control de acceso](/docs/es/get-started/architecture-scenarios/business-to-consumer/authorization) más preciso.

Además, la Auth0 Management API te permite [desbloquear](https://auth0.com/docs/api/management/v2#!/User_Blocks/delete_user_blocks_by_id) a usuarios deshabilitados debido al uso excesivo de credenciales incorrectas.

<div id="linking-user-accounts">
  ## Vinculación de cuentas de usuario
</div>

De forma predeterminada, hay un perfil de usuario (cuenta de usuario) para cada identidad de usuario. Si habilita el inicio de sesión con varios proveedores de identidad, ya sea mediante Facebook o Google [autenticación social](/docs/es/get-started/architecture-scenarios/business-to-consumer/authentication#social-authentication) o mediante la [autenticación con nombre de usuario y contraseña de Auth0](/docs/es/get-started/architecture-scenarios/business-to-consumer/authentication#username-and-password-authentication), cada una tendrá un perfil de usuario independiente. Puede usar la funcionalidad de Auth0 para la [vinculación de cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking) y crear un único perfil para un usuario que agrupe todas sus identidades asociadas.

El proceso de vinculación de cuentas fusiona perfiles de usuario de dos en dos: en el proceso de vinculación se deben especificar una cuenta principal y una cuenta secundaria. No obstante, la cantidad de cuentas que se pueden vincular va más allá de un solo par. Por ejemplo, puede usar como principal una cuenta que ya tenga varias cuentas vinculadas y vincularle una cuenta secundaria adicional. Esto significa que una cuenta de usuario puede tener múltiples identidades asociadas, lo que ofrece varias ventajas:

* Los usuarios pueden iniciar sesión con múltiples identidades sin crear un perfil independiente para cada una.
* Los usuarios registrados pueden usar nuevas identidades de inicio de sesión, pero seguir usando su perfil existente.
* Los usuarios pueden conservar su perfil independientemente de la identidad que usen para iniciar sesión.
* Los usuarios pueden vincular una cuenta con otra que tenga más información de identidad para crear un perfil más completo.
* Sus aplicaciones pueden recuperar datos del perfil de usuario específicos de cada conexión.

<div id="de-provisioning">
  ## Desaprovisionamiento
</div>

Es posible que tu aplicación necesite admitir la solicitud de un usuario para eliminar su cuenta (por ejemplo, puede que debas cumplir los requisitos del [RGPD](https://eugdpr.org/)). Puedes implementar una funcionalidad de este tipo, junto con otras funciones relacionadas con el perfil, mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Users). La Management API te permite recuperar la información almacenada sobre un usuario y actualizarla según sea necesario.

Auth0 puede ayudarte a cumplir diversos requisitos relacionados con la privacidad, incluida la visualización de enlaces a avisos de consentimiento durante el registro y la protección de datos para respaldar el derecho de los usuarios a consultar y corregir los datos que has recopilado sobre ellos.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El [RGPD](https://eugdpr.org/) y otras directivas sobre privacidad exigen que los usuarios tengan derecho a consultar y corregir los datos que se conservan sobre ellos. También tienen derecho al “olvido”. Puedes usar la Management API para dar respuesta a estos requisitos y cumplir tus obligaciones legales.
</Callout>
