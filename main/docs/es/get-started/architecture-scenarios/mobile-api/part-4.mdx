En este documento cubrimos un escenario sencillo: una API utilizada por una aplicación móvil para que los empleados registren sus horas trabajadas.

Aprendimos sobre la concesión Authorization Code con PKCE, qué es un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip>, cómo configurar una API en Auth0, cómo configurar una aplicación móvil para comunicarse de forma segura con esta API, cómo definir y proteger nuestros endpoints de API, cómo usar las bibliotecas proporcionadas para validar el token de acceso y cómo obtener uno nuevo de Auth0.

Comenzamos describiendo el caso de negocio y los requisitos, y luego explicamos cómo puede cumplirse cada requisito y el razonamiento detrás de cada decisión tomada.

Usamos Node.js para la implementación de la API y Android para la aplicación móvil. Aun así, esperamos que, después de leer este documento, puedas crear esta solución con las tecnologías que prefieras.

¡No olvides volver para ver nuevos casos de negocio y escenarios de arquitectura más complejos!
