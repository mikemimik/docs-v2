En este documento abordamos un escenario sencillo: una API utilizada por una aplicación móvil para permitir que los empleados registren sus hojas de hora.

Aprendimos sobre la concesión Authorization Code con PKCE, qué es un <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip>, cómo configurar una API en Auth0, cómo configurar una aplicación móvil para comunicarse de forma segura con esta API, cómo definir y proteger nuestros endpoints de API, cómo usar las bibliotecas proporcionadas para validar el Access Token y cómo obtener uno nuevo de Auth0.

Comenzamos describiendo el caso de negocio y los requisitos, y luego explicamos cómo puede cumplirse cada uno y el razonamiento detrás de las decisiones que se tomaron.

Usamos Node.js para la implementación de la API y Android para la aplicación móvil. Aun así, esperamos que, después de leer este documento, puedas crear esto con las tecnologías que prefieras.

¡No olvides volver para conocer nuevos casos de negocio y escenarios de arquitectura más complejos!
