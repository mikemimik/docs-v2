validatedOn: 2026-02-03

En algún momento, puede que necesite cambiar la información almacenada en el perfil de un usuario. El perfil de un usuario (también conocido como la cuenta del usuario) se almacena en Auth0, y puede que sea necesario modificar la información que contiene por distintos motivos:

* Actualizaciones de información de autoservicio
* Actualizaciones obligatorias relacionadas con los Términos y Condiciones de sus organizaciones
* Cambios debidos al cumplimiento normativo

<Warning>
  No puede acceder directamente al perfil de un usuario a través de varios tenants de Auth0. Si está implementando varios tenants de Auth0 en producción, esto es algo que debe tener en cuenta.
</Warning>

Un [proveedor de identidad](/docs/es/authenticate/identity-providers) completa el perfil de un usuario con los datos proporcionados durante el proceso de inicio de sesión, y a esto se le denomina [Normalized User Profile](/docs/es/manage-users/user-accounts/user-profiles/normalized-user-profiles).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El Normalized User Profile se actualiza a partir del proveedor de identidad durante el inicio de sesión, y puede modificar el conjunto limitado de información que contiene mediante la Auth0 Management API. También puede usar la extensibilidad de Auth0, como [Actions](/docs/es/customize/actions), como alternativa para sobrescribir información del Normalized User Profile. Consulte [Modificación de los datos del perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles#user-profile-data-modification) para obtener más información.
</Callout>

De forma predeterminada, se crea un perfil de usuario por cada identidad de usuario, y hay varios aspectos que debe considerar:

* ¿Qué debería hacer si necesita almacenar información para ayudar a personalizar la experiencia de un usuario?
* ¿Qué ocurre si necesita almacenar información del usuario que no procede de un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>?
* ¿Por qué necesitaría almacenar información relacionada con el usuario que este no puede modificar?
* ¿Qué debe hacer si necesita almacenar información relacionada con el usuario que este no puede modificar?
* ¿Qué sucede si un usuario olvida su contraseña?
* ¿Qué debería hacer un usuario si quiere cambiar su contraseña?
* ¿Cómo proporciona a un administrador de una organización de terceros la capacidad de gestionar a sus usuarios?

Auth0 permite almacenar metadatos en el perfil de un usuario, lo que facilita la recopilación de información adicional, como la preferencia de idioma y/o accesibilidad, con el fin de mejorar la experiencia del usuario. Los metadatos pueden usarse para almacenar tanto información que un usuario puede cambiar como información que no puede cambiar; esto último le brinda la posibilidad de asociar, por ejemplo, un perfil de usuario con registros de sus sistemas existentes sin modificar la implementación actual.

Para los usuarios que olvidan sus contraseñas o a quienes se les permite cambiarla mediante algún mecanismo de autoservicio existente (o uno que tenga previsto implementar), puede aprovechar la funcionalidad de [Restablecimiento de contraseña](#password-reset) proporcionada por Auth0. Esta se puede integrar con su implementación existente y ya viene incorporada en cualquier widget de interfaz de usuario de Auth0 listo para usar, incluido [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

También querrá asegurarse de trabajar en todo momento con una [cuenta de usuario verificada](#account-verification). Auth0 también proporciona mecanismos listos para usar para ello. También debería considerar el [cumplimiento normativo](/docs/es/secure/data-privacy-and-compliance) como el [RGPD](https://gdpr-info.eu/), que tiene requisitos muy específicos cuando se trata de proteger a los ciudadanos de la UE en materia de privacidad y de violaciones de la seguridad de los datos.

Aunque Auth0 actualmente no proporciona un portal centralizado de gestión de perfiles listo para usar, para fines de gestión de perfiles de autoservicio, puede usar la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 para crear el suyo propio o utilizar una interfaz ya desarrollada. Consulte nuestra [guía de la comunidad](https://community.auth0.com/t/how-to-allow-the-end-user-to-update-their-own-profile-information/6228) de Auth0, que describe el endpoint de la Management API. Todas las llamadas a la Management API requerirán el uso de un [token de acceso](/docs/es/secure/tokens/access-tokens).

<Warning>
  La gestión de perfiles de autoservicio puede plantear problemas tanto de seguridad como de privacidad de los datos. Por ejemplo, puede que quiera permitir que un usuario cambie su dirección de correo electrónico; sin embargo, hacerlo sin seguir las prácticas recomendadas de seguridad podría hacer que un usuario pierda el acceso a su cuenta, exponga información de identificación personal (PII) o, peor aún, abra una posible brecha de seguridad.
</Warning>

Como alternativa, puedes usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para [gestionar aspectos del perfil de un usuario](/docs/es/manage-users/user-accounts/manage-users-using-the-dashboard). Gestionar el perfil de un usuario a través del Auth0 Dashboard es más bien una tarea administrativa y **no debe** usarse para la gestión de perfiles en autoservicio en un entorno de producción. Sin embargo, la interfaz que ofrece el Dashboard puede ser extremadamente útil durante el desarrollo, ya que proporciona una forma rápida y sencilla de modificar la información del perfil de un usuario.

Si necesitas ofrecer a tus clientes una forma de permitir que un administrador gestione sus propios usuarios cuando esas credenciales se almacenan en tu sistema, puedes crear algo tú mismo o usar una extensión de Auth0. Consulta [Admin Portal](#admin-portal) para obtener más información.

<div id="metadata">
  ## Metadata
</div>

Además de la información del Normalized User Profile, [Metadata](/docs/es/manage-users/user-accounts/metadata) también puede almacenarse en un perfil de usuario de Auth0. metadata ofrece una forma de almacenar información que no proviene de un proveedor de identidad, o de almacenar información que sobrescribe la que proporciona un proveedor de identidad.

<Info>
  ### Práctica recomendada

  El uso de metadata debe seguir las [prácticas recomendadas para el almacenamiento de datos de usuario](/docs/es/secure/security-guidance/data-security/user-data-storage#metadata) de Auth0. El almacenamiento de metadata no está pensado para usarse como un almacén de datos de propósito general, por lo que, siempre que sea posible, debe seguir usando su propio sistema de almacenamiento externo. El tamaño y la complejidad de metadata también deben mantenerse al mínimo, y la Auth0 Management API establece directrices estrictas para actualizar y/o eliminar metadatos asociada a un usuario.
</Info>

Puede manipular metadatos tanto mediante la Auth0 Management API como mediante la Auth0 Authentication API. Al igual que ocurre al administrar el Normalized User Profile, las llamadas a la Management API para manipular metadatos requieren el uso de un [token de acceso](https://auth0.com/docs/api/management/v2/tokens).

<Warning>
  Las llamadas a la Management API están sujetas a la [política de límite de frecuencia de Auth0](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy). Debe tenerlo en cuenta y, como ayuda, Auth0 generalmente recomienda usar el [SDK de Auth0](/docs/es/libraries) adecuado para su entorno de desarrollo en lugar de llamar directamente a nuestras API.
</Warning>

<div id="user-metadata">
  ### Metadatos de usuario
</div>

Los metadatos de usuario (también llamados `user_metadata`) son información que puede almacenarse en un perfil de usuario y que este puede leer y actualizar como parte de cualquier proceso de gestión autoservicio del perfil. Este tipo de metadatos puede incluir, por ejemplo, la forma de tratamiento de un usuario o su idioma preferido, que podría utilizarse para [personalizar los correos electrónicos](/docs/es/customize/email/email-templates) enviados por Auth0.

<Info>
  ### Práctica recomendada

  Guarda en los metadatos toda la información que quieras usar para personalizar los correos electrónicos de Auth0, y preferiblemente en `user_metadata` si el usuario puede modificarla, como la información que se utiliza para determinar el idioma de un correo electrónico.
</Info>

<div id="app-metadata">
  ### Metadatos de la aplicación
</div>

Los metadatos de la aplicación (también denominados `app_metadata`) son, por otro lado, información que puede almacenarse en un perfil de usuario, pero que **solo puede leerse o actualizarse con la autorización adecuada**; `app_metadata` no es directamente accesible para el usuario. Este tipo de metadatos podría ser, por ejemplo, un indicador de que el usuario aceptó la última versión vigente de los términos y condiciones, así como una fecha que indique cuándo los aceptó.

<div id="password-reset">
  ## Restablecimiento de contraseña
</div>

Para los usuarios que olvidan su contraseña o que pueden cambiarla mediante algún mecanismo de autoservicio existente, Auth0 ofrece la funcionalidad de [Restablecimiento de contraseña](/docs/es/authenticate/database-connections/password-change). Puede integrarla con su implementación actual, y además ya viene incorporada en los widget de interfaz de usuario de Auth0 listos para usar que se incluyen como parte de [Universal Login](/docs/es/authenticate/login/auth0-universal-login).

<Warning>
  El cambio y el restablecimiento de contraseña solo son compatibles con los tipos de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) de Auth0.
</Warning>

Auth0 <Tooltip tip="Universal Login: su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> ofrece compatibilidad nativa de experiencia de usuario para el restablecimiento de contraseña mediante la funcionalidad de Auth0 Authentication API. Como alternativa, puede usar la [Auth0 Authentication API](/docs/es/authenticate/database-connections/password-change#use-the-authentication-api) a través de uno de los Auth0 SDKs adecuados para su entorno de desarrollo. Las plantillas de correo electrónico que se usan durante el flujo de trabajo de restablecimiento de contraseña también se pueden personalizar por completo, tanto si usa los widget de interfaz de usuario de Auth0 listos para usar como una versión personalizada de Universal Login.

Por otro lado, puede usar Auth0 Management API para [cambiar directamente la contraseña](/docs/es/authenticate/database-connections/password-change#directly-set-the-new-password) de una identidad de usuario definida mediante un tipo de conexión de base de datos. Puede usar Auth0 Management API como parte de cualquier implementación de gestión de perfiles de autoservicio, así como de cualquier [personalización de la página Change Password](/docs/es/get-started/architecture-scenarios/business-to-business/branding).

<div id="account-verification">
  ## Verificación de la cuenta
</div>

También tendrás que trabajar siempre con una cuenta de usuario verificada y utilizar los mecanismos que proporciona Auth0. También debes tener en cuenta normativas de cumplimiento como el [RGPD](https://gdpr-info.eu/), que establece requisitos muy específicos para proteger a los ciudadanos de la UE en materia de privacidad y ante violaciones de la seguridad de los datos.

Auth0 ofrece funcionalidad lista para usar para enviar un [correo electrónico de verificación](/docs/es/customize/email/manage-email-flow) a la dirección de correo electrónico de un usuario y verificar su cuenta. De forma predeterminada, Auth0 envía automáticamente correos electrónicos de verificación para cualquier identidad de [conexión de base de datos](/docs/es/authenticate/database-connections/password-change) creada como parte del [autorregistro](/docs/es/get-started/architecture-scenarios/business-to-business/provisioning#self-sign-up). Sin embargo, Auth0 también proporciona un [endpoint de Management API](https://auth0.com/docs/api/v2#!/Tickets/post_email_verification) que puedes usar para enviar correos electrónicos de verificación en los casos en los que un proveedor social no valida la dirección de correo electrónico durante el registro del usuario.

<div id="blocking-users">
  ## Bloqueo de usuarios
</div>

[El bloqueo del acceso de los usuarios](/docs/es/manage-users/user-accounts/block-and-unblock-users) en Auth0 permite impedir que los usuarios inicien sesión en las aplicaciones en determinadas circunstancias. De forma predeterminada, el Auth0 Dashboard ofrece un mecanismo listo para usar que permite a los administradores bloquear y desbloquear el acceso de los usuarios a todas las aplicaciones, y también puedes implementar esta funcionalidad mediante la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id). También puedes usar la extensibilidad de Auth0 para [deshabilitar el acceso de los usuarios a determinadas aplicaciones](/docs/es/manage-users/user-accounts/manage-user-access-to-applications), así como proporcionar un [control de acceso](/docs/es/get-started/architecture-scenarios/business-to-business/authorization) más detallado.

Además, la Auth0 Management API te permite [desbloquear](https://auth0.com/docs/api/management/v2#!/User_Blocks/delete_user_blocks_by_id) a usuarios deshabilitados debido al uso excesivo de credenciales incorrectas.

<div id="admin-portal">
  ## Portal de administración
</div>

Un portal de administración es una aplicación en la que puedes crear usuarios nuevos, editar el perfil de un usuario, ver la actividad de un usuario, etc. Esta aplicación solo debe ser accesible para administradores. Aunque Auth0 proporciona su Dashboard de administración, no se recomienda dar acceso al Dashboard de administración a muchas personas, ya que hay muchas formas en que alguien podría afectar sin querer tu tenant de Auth0. En su lugar, Auth0 ofrece otras dos opciones:

* [**Auth0 Management API**](https://auth0.com/docs/api/management/v2): Con la Auth0 Management API puedes crear fácilmente una aplicación que permita a los administradores gestionar usuarios. Puedes incorporarla a una aplicación existente que ya utilicen tus administradores o crear una nueva con una UI acorde con tus aplicaciones actuales.
* [**Extensión de Administración delegada de Auth0**](/docs/es/customize/extensions/delegated-administration-extension): Esta potente y flexible extensión te permite personalizar la experiencia de administración de usuarios. Puedes adaptarla para que los administradores de tus clientes inicien sesión y solo puedan ver y gestionar usuarios dentro de su organización.

<Info>
  ### Práctica recomendada

  Si proporcionas tu propia forma de administrar usuarios, solo deberías permitir que los administradores envíen a los usuarios un enlace por correo electrónico para cambiar la contraseña, en lugar de permitirles establecer contraseñas directamente. Si debes ir en contra de esta recomendación y permitir que tus administradores establezcan la contraseña de otra persona, deberías obligar al usuario a cambiar su contraseña en el siguiente inicio de sesión para que solo él la conozca (y no también un administrador).
</Info>

<div id="multiple-organization-architecture-multitenancy">
  ## Arquitectura de múltiples organizaciones (multitenencia)
</div>

Muchas plataformas B2B implementan algún tipo de aislamiento y/o branding para la organización de sus clientes, y esto puede añadir complejidad a cualquier sistema de gestión de identidades y accesos (IAM). Si este es su caso, le recomendamos dedicar algo de tiempo a leer nuestra guía y las prácticas recomendadas para este tipo de entorno.
