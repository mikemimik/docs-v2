En este escenario construiremos una API de hojas de hora para una empresa ficticia llamada ExampleCo. La API permitirá agregar registros adicionales de hojas de hora para un empleado o un contratista.

También construiremos una tarea cron que procesará registros de hojas de hora desde un sistema externo hacia la base de datos centralizada de hojas de hora mediante la API.

<Info>
  ### En resumen

  * Auth0 proporciona autenticación y autorización para API como medio para proteger el acceso a los endpoints de API (lee [API Authentication and Authorization](/docs/es/get-started/architecture-scenarios/server-application-api/part-1#api-authentication-and-authorization))
  * Para autorizar una aplicación de máquina a máquina (una CLI, servicio o daemon en el que no interviene la interacción del usuario), Auth0 admite la Concesión de credenciales de cliente (lee [Client Credentials Grant](/docs/es/get-started/architecture-scenarios/server-application-api/part-1#client-credentials-grant))
  * Tanto la aplicación de máquina a máquina como la API deben configurarse en el Auth0 Dashboard (lee [Auth0 Configuration](/docs/es/get-started/architecture-scenarios/server-application-api/part-2))
  * La API estará protegida al garantizar que se envíe un token de acceso válido (implementado como un JSON Web Token) en el encabezado HTTP Authorization cuando se hagan llamadas a la API (lee [Implement the API](/docs/es/get-started/architecture-scenarios/server-application-api/part-3))
  * Tras una autorización correcta, se emite un token de acceso a la aplicación de máquina a máquina (lee [Get an access token](/docs/es/get-started/architecture-scenarios/server-application-api/part-3#get-an-access-token))
  * La aplicación de máquina a máquina puede, a su vez, usar este token de acceso y enviarlo en el encabezado HTTP Authorization para autenticar llamadas a endpoints de API (lee [Invoke the API](/docs/es/get-started/architecture-scenarios/server-application-api/part-3#invoke-the-api))
</Info>

<div id="the-premise">
  ## La premisa
</div>

ExampleCo es una startup de consultoría. Actualmente tiene aproximadamente 100 empleados y también trabaja con contratistas externos. Todos los empleados y los contratistas externos deben completar sus hojas de hora cada semana. Para ello, crearon una aplicación de hojas de hora, un escenario que cubrimos en [inicio de sesión único para aplicaciones web tradicionales](/docs/es/get-started/architecture-scenarios/sso-for-regular-web-apps). Los empleados internos usan esta aplicación web para completar sus hojas de hora, pero algunos de los contratistas externos ya usan otra herramienta para registrar sus hojas de hora. ExampleCo necesita una solución para evitar duplicar el trabajo y decidió crear una tarea cron para leer las entradas de la hoja de hora de este sistema externo y cargarlas automáticamente en el backend de ExampleCo mediante una API.

<div id="goals-requirements">
  ### Objetivos y requisitos
</div>

ExampleCo quiere crear una solución flexible que:

* Permita procesos automatizados que se ejecuten en el backend
* Permita incorporar en la arquitectura futuros lanzamientos de aplicaciones, como una aplicación móvil
* Permita que la API de hoja de hora sea segura y que solo puedan acceder a ella usuarios y aplicaciones autorizados
* Permita compartir gran parte del código y la lógica de negocio de la aplicación con otras aplicaciones
