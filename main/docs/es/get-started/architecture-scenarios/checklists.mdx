Haga clic en los enlaces de abajo para descargar una lista de verificación correspondiente a una fase del SDLC (ciclo de vida del desarrollo de software). Puede abrir la lista de verificación en cualquier aplicación de hoja de cálculo y personalizarla según sus necesidades.

<div id="analyze-checklist">
  ## Lista de verificación para el análisis
</div>

[Descargar la lista de verificación para el análisis](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Analyze-Checklist.xlsx)

En la fase de análisis, analice los requisitos de negocio del usuario final y determine los objetivos del proyecto como parte del plan general del proyecto. Convierta los requisitos y los objetivos en funcionalidades del sistema que la organización pretende desarrollar. Las actividades incluyen:

* Recopilación de requisitos de negocio
* Creación de diagramas de procesos
* Realización de análisis detallados
* Alineación con el plan del proyecto

<div id="design-checklist">
  ## Lista de verificación de Design
</div>

[Descargar la lista de verificación de Design](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Design-Checklist.xlsx)

En la fase de Design, describa las características y operaciones previstas del sistema, incluidas las reglas de negocio, el pseudocódigo, los diseños de pantalla y cualquier otra documentación necesaria. Las actividades incluyen:

* Diseño de infraestructura
* Diseño del modelo del sistema

<div id="build-checklist">
  ## Lista de verificación de desarrollo
</div>

[Descarga de la lista de verificación de desarrollo](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Build-Checklist.xlsx)

En la fase de desarrollo, se desarrolla el sistema mediante la implementación de la infraestructura y el código. Las actividades incluyen:

* Implementación de la infraestructura
* Implementación del código

<div id="test-checklist">
  ## Lista de verificación de pruebas
</div>

[Descargar la lista de verificación de pruebas](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Test-Checklist.xlsx)

En la fase de pruebas, integre y despliegue todo el código implementado en la infraestructura del entorno de pruebas. Luego, las pruebas siguen las actividades del ciclo de vida de las pruebas de software para comprobar si el sistema presenta errores, fallos y defectos, y verificar que las funcionalidades del sistema funcionen como se espera (o no). Las actividades incluyen:

* Escribir casos de prueba
* Ejecutar casos de prueba

<div id="deploy-checklist">
  ## Lista de verificación de despliegue
</div>

[Descargar la lista de verificación de despliegue](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Deploy-Checklist.xlsx)

En la fase de despliegue, implemente el sistema en un entorno de staging o de producción, donde los usuarios reales comienzan a utilizarlo e interactuar con él.

Finalmente, implementará todos los componentes del sistema en el entorno de producción cuando realice una puesta en producción.

<div id="monitor-checklist">
  ## Lista de verificación de monitoreo
</div>

[Descargar la lista de verificación de monitoreo](https://assets.ctfassets.net/cdy7uua7fh8z/4FYZQXQHknEDkLflShHXBz/d8757810ace0fc9dbabad9b1715d0dbf/Monitor-Checklist.xlsx)

En la fase de monitoreo, se realizan mejoras, correcciones y cambios para garantizar que el sistema siga funcionando y se mantenga actualizado, a fin de cumplir los objetivos del negocio y responder a las necesidades de los usuarios. Las actividades incluyen:

* Monitoreo
* Mantenimiento
* Cambios y ajustes
* Actualización y adaptación a las necesidades futuras
