Haz clic en los enlaces de abajo para descargar una lista de verificación correspondiente a una fase del SDLC (ciclo de vida del desarrollo de software). Puedes abrirla en cualquier aplicación de hojas de cálculo y personalizarla según tus necesidades.

<div id="analyze-checklist">
  ## Lista de verificación de análisis
</div>

[Descargar la lista de verificación de análisis](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Analyze-Checklist.xlsx)

En la fase de análisis, se analizan los requisitos empresariales del usuario final y se determinan los objetivos del proyecto como parte del plan general del proyecto. Los requisitos y objetivos se convierten en funciones del sistema que la organización pretende desarrollar. Las actividades incluyen:

* Recopilar requisitos empresariales
* Crear diagramas de procesos
* Realizar un análisis detallado
* Alinear con el plan del proyecto

<div id="design-checklist">
  ## Lista de verificación de Design
</div>

[Descargar la lista de verificación de Design](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Design-Checklist.xlsx)

En la fase de Design, describa las funcionalidades y operaciones previstas del sistema, incluidas las reglas de negocio, el pseudocódigo, los diseños de pantalla y cualquier otra documentación necesaria. Las actividades incluyen:

* Diseño de infraestructura
* Diseño del modelo del sistema

<div id="build-checklist">
  ## Lista de verificación de la fase de desarrollo
</div>

[Descargar la lista de verificación de la fase de desarrollo](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Build-Checklist.xlsx)

Durante la fase de desarrollo, se desarrolla el sistema propiamente dicho mediante la implementación de la infraestructura y el código. Las actividades incluyen:

* Implementación de la infraestructura
* Implementación del código

<div id="test-checklist">
  ## Lista de verificación de pruebas
</div>

[Descargar la lista de verificación de pruebas](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Test-Checklist.xlsx)

En la fase de pruebas, integre y despliegue todo el código implementado en la infraestructura del entorno de pruebas. A continuación, lleve a cabo las actividades del ciclo de vida de las pruebas de software para comprobar si el sistema presenta errores, fallos y defectos, y verificar si las funcionalidades del sistema funcionan como se espera (o no). Las actividades incluyen:

* Escribir casos de prueba
* Ejecutar casos de prueba

<div id="deploy-checklist">
  ## lista de verificación de despliegue
</div>

[Descargar lista de verificación de despliegue](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Deploy-Checklist.xlsx)

En la fase de despliegue, despliega el sistema en un entorno de staging o de producción, donde los usuarios reales empiezan a utilizarlo e interactuar con él.

Por último, despliegas todos los componentes del sistema en el entorno de producción al hacer una puesta en producción.

<div id="monitor-checklist">
  ## Lista de verificación de Monitor
</div>

[Descargar la lista de verificación de Monitor](https://assets.ctfassets.net/cdy7uua7fh8z/4FYZQXQHknEDkLflShHXBz/d8757810ace0fc9dbabad9b1715d0dbf/Monitor-Checklist.xlsx)

En la fase de Monitor, realiza mejoras, correcciones y cambios para garantizar que el sistema siga funcionando y se mantenga actualizado a fin de cumplir los objetivos de negocio y responder a las necesidades de los usuarios. Las actividades incluyen:

* monitoreo
* Mantenimiento
* Cambios y ajustes
* Actualización y adaptación a las necesidades futuras
