Haga clic en los enlaces a continuación para descargar una lista de verificación correspondiente a una fase del SDLC (ciclo de vida del desarrollo de software). Puede abrirla en cualquier aplicación de hojas de cálculo y personalizarla según sus necesidades.

<div id="analyze-checklist">
  ## Lista de verificación de análisis
</div>

[Descargar la lista de verificación de análisis](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Analyze-Checklist.xlsx)

En la fase de análisis, analice los requisitos empresariales de los usuarios finales y defina los objetivos del proyecto como parte del plan general del proyecto. Convierta estos requisitos y objetivos en funciones del sistema que la organización prevé desarrollar. Las actividades incluyen:

* Recopilación de requisitos empresariales
* Creación de diagramas de procesos
* Realización de análisis detallados
* Alineación con el plan del proyecto

<div id="design-checklist">
  ## Lista de verificación de Design
</div>

[Descargar lista de verificación de Design](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Design-Checklist.xlsx)

En la fase de Design, describa las funcionalidades y operaciones deseadas del sistema, incluidas las reglas de negocio, el pseudocódigo, los diseños de pantalla y cualquier otra documentación necesaria. Las actividades incluyen:

* Diseño de la infraestructura
* Diseño del modelo del sistema

<div id="build-checklist">
  ## Lista de verificación de desarrollo
</div>

[Descargar la lista de verificación de desarrollo](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Build-Checklist.xlsx)

En la fase de desarrollo, desarrolle el sistema mediante la implementación de la infraestructura y el código. Las actividades incluyen:

* Implementación de la infraestructura
* Implementación del código

<div id="test-checklist">
  ## Lista de verificación de pruebas
</div>

[Descargar la lista de verificación de pruebas](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Test-Checklist.xlsx)

En la fase de pruebas, integre y despliegue todo el código implementado en la infraestructura del entorno de pruebas. A continuación, las pruebas siguen las actividades del ciclo de vida de las pruebas de software para revisar el sistema en busca de errores, fallos y defectos, y comprobar si las funcionalidades del sistema funcionan como se espera (o no). Las actividades incluyen:

* Redactar casos de prueba
* Ejecutar casos de prueba

<div id="deploy-checklist">
  ## Lista de verificación de despliegue
</div>

[Descargar la lista de verificación de despliegue](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Deploy-Checklist.xlsx)

En la fase de despliegue, el sistema se despliega en un entorno de staging o de Production, donde los usuarios reales comienzan a usarlo e interactuar con él.

Con el tiempo, todos los componentes del sistema se despliegan en el entorno de Production cuando se realiza una publicación en vivo.

<div id="monitor-checklist">
  ## Lista de verificación de monitoreo
</div>

[Descargar la lista de verificación de monitoreo](https://assets.ctfassets.net/cdy7uua7fh8z/4FYZQXQHknEDkLflShHXBz/d8757810ace0fc9dbabad9b1715d0dbf/Monitor-Checklist.xlsx)

En la fase de monitoreo, realice mejoras, correcciones y cambios para garantizar que el sistema siga funcionando y se mantenga actualizado, a fin de cumplir los objetivos del negocio y responder a las necesidades de los usuarios. Las actividades incluyen:

* Monitoreo
* Mantenimiento
* Cambios y ajustes
* Actualización y adaptación a necesidades futuras
