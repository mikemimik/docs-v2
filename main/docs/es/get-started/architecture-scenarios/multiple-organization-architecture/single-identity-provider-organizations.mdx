title: Organizaciones con proveedor de identidad único
---

En el escenario de Organizaciones con <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> (IdP) único, cada organización que usa la funcionalidad [Auth0 Organizations](/docs/es/manage-users/organizations/organizations-overview) utiliza exactamente una conexión de Auth0. Esta funcionalidad permite que cada aplicación seleccione la organización adecuada definida en el tenant de Auth0, que autenticará directamente las credenciales del usuario o reenviará la solicitud de autenticación al IdP correspondiente para que la gestione.

En este escenario, seguiremos usando el ejemplo de [Travel0 Corporate Booking](/docs/es/get-started/architecture-scenarios/multiple-organization-architecture). En este ejemplo, la aplicación de reservas corporativas equivale a tu aplicación y Travel0 corresponde a tu empresa.

<Frame>![Escenarios de arquitectura - Multitenencia - Diagrama - Usuarios aislados, Aplicaciones compartidas, Organizaciones A y B](/docs/images/cdy7uua7fh8z/1YBaMaMozamiTv8hXmNB2D/21e9270a59021a12209bdcb4345147cb/isolated-users_shared-apps_org-ab.png)</Frame>

<Warning>
  La funcionalidad Auth0 Organizations actualmente admite aplicaciones que usan los protocolos [SAML](/docs/es/authenticate/protocols/saml/saml-configuration), [OpenID Connect](/docs/es/authenticate/protocols/openid-connect-protocol) y/o el framework [OAuth 2](/docs/es/authenticate/protocols/oauth). Las aplicaciones que requieren [WS-Fed](/docs/es/authenticate/protocols/ws-fed-protocol) no son compatibles.
</Warning>

Independientemente de cómo se procesen las credenciales, ya sea que las credenciales del usuario se almacenen en una conexión de base de datos (como ocurre con Hoekstra &amp; Associates), se validen mediante una conexión empresarial (como con MetaHexa Bank) o mediante una conexión social, tu aplicación se comporta de la misma manera. Usar la funcionalidad [Organizations](/docs/es/manage-users/organizations/organizations-overview) te permite configurar y controlar la experiencia.
