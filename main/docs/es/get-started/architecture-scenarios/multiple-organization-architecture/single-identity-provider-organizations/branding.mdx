Los recursos de marca asociados a una organización son sumamente valiosos, ya que ofrecen a los usuarios un entorno que reconocen y en el que confían. El uso de elementos de marca reconocibles también aumenta la confianza de los usuarios en que la información que proporcionan (por ejemplo, las credenciales) se gestionará de forma segura. En nuestros escenarios de arquitectura, ofrecemos orientación general sobre varios temas de [B2B Branding](/docs/es/get-started/architecture-scenarios/business-to-business/branding), cuya revisión recomendamos junto con la orientación que se proporciona aquí.

<div id="universal-login">
  ## Universal Login
</div>

La personalización de marca de un entorno con múltiples organizaciones puede ser más compleja que en un entorno estándar, donde solo tienes que gestionar un único conjunto de activos de marca. Sin embargo, [Auth0 Organizations](/docs/es/manage-users/organizations/organizations-overview) aprovechan la [experiencia de Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), que ofrece varias [plantillas de página](/docs/es/customize/login-pages/universal-login/customize-templates) para ayudar a simplificar la personalización de marca. La configuración de la organización en Auth0 también te permite [seleccionar una personalización básica del esquema de colores](/docs/es/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Página de inicio de sesión
</div>

Al usar una [conexión de base de datos o una conexión de base de datos personalizada](/docs/es/authenticate/database-connections), la página de inicio de sesión se utiliza para obtener las credenciales del primer factor de un usuario. Puede configurar la página de inicio de sesión mediante un conjunto de [variables de plantilla](/docs/es/manage-users/organizations/create-first-organization) disponibles para facilitar la personalización por organización.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Puede configurar la página de inicio de sesión para mostrar conexiones empresariales y sociales para la autenticación del primer factor, de modo que la imagen de marca de la página de inicio de sesión resulte pertinente en estos casos.
</Callout>

<div id="password-reset-page">
  ### Página de restablecimiento de contraseña
</div>

Al igual que la [página de inicio de sesión](#login-page), la página de Restablecimiento de contraseña puede configurarse en escenarios que usan una conexión de base de datos o una conexión de base de datos personalizada. En los escenarios que implican una conexión empresarial o una conexión social, los cambios de contraseña los gestiona fuera de Auth0 el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> (IdP) externo, por lo que nunca se muestra una página de Restablecimiento de contraseña de Auth0.

<div id="multi-factor-authentication-pages">
  ### Páginas de autenticación multifactor
</div>

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> (MFA) en Auth0 es compatible con todos los métodos que se usan para obtener las credenciales del primer factor de un usuario (es decir, conexión de base de datos o conexión de base de datos personalizada, conexión empresarial o conexión social). Si usas MFA, probablemente querrás personalizar con tu marca las páginas de la experiencia de <Tooltip tip="Universal Login: tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> asociadas con MFA, lo cual puede hacerse de forma similar a la personalización de marca de la [página de inicio de sesión](#login-page).

<div id="email-templates">
  ## Plantillas de correo electrónico
</div>

Al igual que con las plantillas de página de Universal Login, hay disponible un conjunto de [variables de las plantillas de correo electrónico](/docs/es/customize/email/email-templates/supported-liquid-syntax#general-variables) para facilitar la personalización de los correos electrónicos por organización. Las plantillas de correo electrónico que deban personalizarse variarán en función de cómo se autentiquen los usuarios (es decir, mediante una conexión de base de datos o una conexión de base de datos personalizada, una conexión empresarial o una conexión social). Para obtener más información sobre la personalización de plantillas de correo electrónico al usar la funcionalidad Auth0 Organizations, consulta [Create Your First Organization: Email Templates](/docs/es/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Dominios personalizados
</div>

La funcionalidad Auth0 Organizations solo admite un [dominio personalizado](/docs/es/customize/custom-domains), que se define a nivel del tenant de Auth0. Por lo tanto, le recomendamos que use un nombre de <Tooltip tip="Dominio personalizado: Dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=Custom+Domain">dominio personalizado</Tooltip> que no dependa de la organización. Si realmente necesita implementar nombres de dominio por organización, tendrá que desplegar varios tenants de Auth0 en producción. Como usar varios tenants tiene implicaciones que afectan al <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO), a la gestión del perfil de usuario, etc., debe evaluar cuidadosamente esta opción antes de seguir este enfoque.
