Los elementos de marca asociados con una organización son muy valiosos porque brindan a los usuarios un entorno que conocen y en el que confían. Utilizar elementos de marca reconocibles también aumenta la confianza de los usuarios en que la información que proporcionan (por ejemplo, las credenciales) se gestionará de forma segura. En nuestros escenarios de arquitectura, ofrecemos orientación general sobre diversos temas de [Branding B2B](/docs/es/get-started/architecture-scenarios/business-to-business/branding), que recomendamos revisar junto con la orientación que se proporciona aquí.

<div id="universal-login">
  ## Universal Login
</div>

La personalización de marca de un entorno con múltiples organizaciones puede ser más compleja que en un entorno estándar, donde solo tienes que gestionar un único conjunto de elementos de marca. Sin embargo, [Auth0 Organizations](/docs/es/manage-users/organizations/organizations-overview) aprovecha la [experiencia de Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), que proporciona una serie de [plantillas de página](/docs/es/customize/login-pages/universal-login/customize-templates) para simplificar la personalización de marca. La propia configuración de Auth0 Organization también te permite [seleccionar una combinación básica de colores para la marca](/docs/es/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Página de inicio de sesión
</div>

Al usar una [conexión de base de datos o una conexión de base de datos personalizada](/docs/es/authenticate/database-connections), la página de inicio de sesión se utiliza para obtener las credenciales del primer factor de un usuario. Puedes configurar la página de inicio de sesión mediante un conjunto de [variables de plantilla](/docs/es/manage-users/organizations/create-first-organization) que facilitan la personalización por organización.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Puedes configurar la Página de inicio de sesión para que muestre conexiones empresariales y sociales para la autenticación del primer factor, por lo que la identidad de marca de la Página de inicio de sesión puede ser relevante en estos casos.
</Callout>

<div id="password-reset-page">
  ### Página de Restablecimiento de contraseña
</div>

Al igual que la [página de inicio de sesión](#login-page), la página de Restablecimiento de contraseña se puede configurar en escenarios que usan una conexión de base de datos o una conexión de base de datos personalizada. En los escenarios que involucran una conexión empresarial o una conexión social, los cambios de contraseña se gestionan fuera de Auth0 por el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> (IdP), por lo que nunca se muestra una página de Restablecimiento de contraseña de Auth0.

<div id="multi-factor-authentication-pages">
  ### Páginas de autenticación multifactor
</div>

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación del usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> (MFA) en Auth0 es compatible con todos los métodos que se usan para obtener las credenciales del primer factor de un usuario (es decir, conexión de base de datos o conexión de base de datos personalizada, conexión empresarial o conexión social). Si usas MFA, probablemente querrás personalizar las páginas de la experiencia de <Tooltip tip="Universal Login: tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> asociadas con MFA, lo que puede hacerse de forma similar a la personalización de la [página de inicio de sesión](#login-page).

<div id="email-templates">
  ## Plantillas de correo electrónico
</div>

Al igual que con las plantillas de página de Universal Login, hay disponibles un conjunto de [variables de plantilla de correo electrónico](/docs/es/customize/email/email-templates/supported-liquid-syntax#general-variables) para ayudar a personalizar los correos electrónicos de cada organización. Las plantillas de correo electrónico que deban personalizarse variarán según cómo se autentiquen los usuarios (es decir, mediante una conexión de base de datos o una conexión de base de datos personalizada, una conexión empresarial o una conexión social). Para obtener más información sobre la personalización de plantillas de correo electrónico al usar Auth0 Organizations, consulta [Crea tu primera organización: Plantillas de correo electrónico](/docs/es/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Dominios personalizados
</div>

La función Auth0 Organizations admite solo un [dominio personalizado](/docs/es/customize/custom-domains), que se define en el nivel del tenant de Auth0. Por lo tanto, te recomendamos que uses un nombre de <Tooltip tip="Dominio personalizado: Dominio de terceros con un nombre especializado o distintivo." cta="Ver glosario" href="/docs/es/glossary?term=Custom+Domain">dominio personalizado</Tooltip> que no dependa de la organización. Si realmente necesitas implementar nombres de dominio por organización, será necesario poner en producción varios tenants de Auth0. Dado que usar varios tenants tiene implicaciones que afectan al <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO), a la gestión del perfil de usuario, etc., debes pensarlo bien antes de optar por esta vía.
