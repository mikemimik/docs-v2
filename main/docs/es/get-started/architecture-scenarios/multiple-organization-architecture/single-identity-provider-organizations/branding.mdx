Los recursos de marca asociados a una organización son extremadamente valiosos, ya que proporcionan a los usuarios un entorno que conocen y en el que confían. El uso de recursos de marca reconocibles también refuerza la confianza de los usuarios en que la información que proporcionan (por ejemplo, sus credenciales) se gestionará de forma segura. En nuestros escenarios de arquitectura, ofrecemos orientación general sobre diversos temas de [B2B Branding](/docs/es/get-started/architecture-scenarios/business-to-business/branding), cuya revisión recomendamos junto con la orientación que se proporciona aquí.

<div id="universal-login">
  ## Universal Login
</div>

La personalización de marca en un entorno con múltiples organizaciones puede ser más compleja que en un entorno estándar, donde solo hay que gestionar un único conjunto de recursos de marca. Sin embargo, [Auth0 Organizations](/docs/es/manage-users/organizations/organizations-overview) aprovecha la [experiencia de Universal Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), que ofrece varias [plantillas de página](/docs/es/customize/login-pages/universal-login/customize-templates) para simplificar la personalización de la marca. Además, la propia configuración de Auth0 Organization también le permite [seleccionar una personalización básica de la marca mediante una combinación de colores](/docs/es/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Página de inicio de sesión
</div>

Al usar una [conexión de base de datos o conexión de base de datos personalizada](/docs/es/authenticate/database-connections), la página de inicio de sesión se utiliza para obtener las credenciales del primer factor de un usuario. Puedes configurar la página de inicio de sesión mediante un conjunto de [variables de plantilla](/docs/es/manage-users/organizations/create-first-organization) disponibles para facilitar la personalización para cada organización.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Puedes configurar la página de inicio de sesión para mostrar conexiones empresariales y sociales para la autenticación del primer factor, de modo que la imagen de marca de la página de inicio de sesión sea pertinente también en estos casos.
</Callout>

<div id="password-reset-page">
  ### Página de restablecimiento de contraseña
</div>

Al igual que la [página de inicio de sesión](#login-page), la página de restablecimiento de contraseña se puede configurar en escenarios que usan una conexión de base de datos o una conexión de base de datos personalizada. En los escenarios que implican una conexión empresarial o una conexión social, los cambios de contraseña los gestiona fuera de Auth0 el upstream <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> (IdP), por lo que la página de restablecimiento de contraseña de Auth0 nunca se muestra.

<div id="multi-factor-authentication-pages">
  ### Páginas de autenticación multifactor
</div>

<Tooltip tip="Multi-factor authentication (MFA): Proceso de autenticación de usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> (MFA) en Auth0 es compatible con todos los métodos utilizados para obtener las credenciales del primer factor de un usuario (es decir, conexión de base de datos o conexión de base de datos personalizada, conexión empresarial o conexión social). Si utilizas MFA, probablemente querrás personalizar la identidad visual de las páginas de la experiencia de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> asociadas a MFA, lo que puede hacerse de forma similar a la personalización de la [página de inicio de sesión](#login-page).

<div id="email-templates">
  ## Plantillas de correo electrónico
</div>

Al igual que con las plantillas de página de Universal Login, hay un conjunto de [variables de plantillas de correo electrónico](/docs/es/customize/email/email-templates/supported-liquid-syntax#general-variables) disponibles para ayudar a personalizar los correos electrónicos por organización. Las plantillas de correo electrónico que deban personalizarse variarán en función de cómo se autentiquen los usuarios (es decir, mediante una conexión de base de datos o una conexión de base de datos personalizada, una conexión empresarial o una conexión social). Para obtener más información sobre la personalización de plantillas de correo electrónico al usar la funcionalidad de Auth0 Organizations, consulte [Cree su primera organización: Plantillas de correo electrónico](/docs/es/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Custom Domains
</div>

La funcionalidad Auth0 Organizations admite solo un [dominio personalizado](/docs/es/customize/custom-domains), que se define a nivel del tenant de Auth0. Por lo tanto, te recomendamos usar un nombre de <Tooltip tip="Dominio personalizado: Dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=Custom+Domain">dominio personalizado</Tooltip> que no dependa de la organización. Si realmente necesitas implementar una nomenclatura de dominios por organización, tendrás que desplegar varios tenants de Auth0 en producción. Como el uso de varios tenants tiene implicaciones que afectan al <Tooltip tip="Inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO), la gestión del perfil del usuario, etc., deberías pensarlo detenidamente antes de optar por esta vía.
