<div id="configure-the-cookie-and-oidc-middleware">
  ## Configure la cookie y el middleware OIDC
</div>

A efectos de esta guía, usaremos un hosted login sencillo. Puede usar la cookie estándar y el middleware OIDC disponible en ASP.NET Core, así que asegúrese de instalar los paquetes NuGet.

```bash lines
Install-Package Microsoft.AspNetCore.Authentication.Cookies
Install-Package Microsoft.AspNetCore.Authentication.OpenIdConnect
```

A continuación, configura la cookie y el middleware OIDC en el pipeline de middleware de tu aplicación.

export const codeExample = `public class Startup
{
    public void ConfigureServices(IServiceCollection services)
    {
        // Agrega servicios de autenticación
        services.AddAuthentication(
            options => options.SignInScheme = CookieAuthenticationDefaults.AuthenticationScheme);

        // Código omitido por brevedad...
    }

    public void Configure(IApplicationBuilder app, IHostingEnvironment env, ILoggerFactory loggerFactory, IOptions<Auth0Settings> auth0Settings)
    {
        // Código omitido por brevedad...

        // Agrega el middleware de cookies
        app.UseCookieAuthentication(new CookieAuthenticationOptions
        {
            AutomaticAuthenticate = true,
            AutomaticChallenge = true
        });

        // Agrega el middleware OIDC
        app.UseOpenIdConnectAuthentication(new OpenIdConnectOptions("Auth0")
        {
            // Establece la autoridad en tu dominio de Auth0
            Authority = "https://{yourDomain}/",

            // Configura el ID de cliente de Auth0 y el secreto de cliente
            ClientId = {yourClientId},
            ClientSecret = {yourClientSecret},

            // No autenticar ni solicitar autenticación automáticamente
            AutomaticAuthenticate = false,
            AutomaticChallenge = false,

            // Establece el tipo de respuesta como code
            ResponseType = "code",

            CallbackPath = new PathString("/signin-auth0"),

            // Configura el emisor de claims para que sea Auth0
            ClaimsIssuer = "Auth0"
        });

        // Código omitido por brevedad...
    }
}`;

Como puedes ver en el código anterior, hemos configurado dos tipos diferentes de middleware de autenticación.

El primero es el middleware de cookies, que se registró con la llamada a `UseCookieAuthentication`. El segundo es el middleware OIDC, que se registró con la llamada a `UseOpenIdConnectAuthentication`.

Una vez que el usuario haya iniciado sesión en Auth0 usando el middleware OIDC, su información se almacenará automáticamente en una <Tooltip tip="Cookie de sesión: Entidad que, cuando está presente, permite considerar al usuario como autenticado." cta="Ver glosario" href="/docs/es/glossary?term=session+cookie">cookie de sesión</Tooltip>. Todo lo que tienes que hacer es configurar el middleware como se muestra arriba, y este se encargará de gestionar la sesión del usuario.

El middleware de <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) también extraerá todos los claims del <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> que Auth0 envía una vez que el usuario se ha autenticado, y los añadirá como claims a la `ClaimsIdentity`.

<div id="implement-the-logout">
  ## Implementar el cierre de sesión
</div>

Puede controlar tanto la sesión de la aplicación como la sesión de Auth0 mediante el método `SignOutAsync` de la clase `AuthenticationManager`, pasando el esquema de autenticación del que desea cerrar sesión.

Por ejemplo, para cerrar sesión en el middleware de cookies y, de ese modo, borrar la cookie de autenticación de su aplicación, puede realizar la siguiente llamada:

```csharp wrap lines
await HttpContext.Authentication.SignOutAsync(CookieAuthenticationDefaults.AuthenticationScheme);
```

De forma similar, puedes cerrar la sesión del usuario en Auth0 llamando al método `SignOutAsync` y pasando `Auth0` como el esquema de autenticación para cerrar la sesión.

```csharp lines
await HttpContext.Authentication.SignOutAsync("Auth0");
```

Sin embargo, para que lo anterior funcione, también tendrás que agregar configuración adicional al registrar el middleware OIDC, gestionando el evento `OnRedirectToIdentityProviderForSignOut`. En ese evento, tendrás que redirigir al [endpoint de cierre de sesión de Auth0](https://auth0.com/docs/api/authentication/reference#logout) para eliminar la cookie de Auth0.

```csharp lines
app.UseOpenIdConnectAuthentication(new OpenIdConnectOptions("Auth0")
{
    // Se omite parte del código por brevedad
    Events = new OpenIdConnectEvents
    {
        OnRedirectToIdentityProviderForSignOut = context =>
        {
            context.Response.Redirect($"https://{auth0Settings.Value.Domain}/v2/logout?client_id={auth0Settings.Value.ClientId}&returnTo={context.Request.Scheme}://{context.Request.Host}/");
            context.HandleResponse();

            return Task.FromResult(0);
        }
    }
});
```

También deberá asegurarse de agregar la URL de su aplicación a las **Allowed Logout URLs** de su aplicación en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>. Para obtener más información, consulte [Cierre de sesión](/docs/es/authenticate/login/logout).

<div id="implement-admin-permissions">
  ## Permisos de Admin de Implement
</div>

La forma más sencilla de integrar los grupos en una aplicación ASP.NET Core es usar la [autorización basada en roles](https://docs.asp.net/en/latest/security/authorization/roles.html) integrada de ASP.NET Core. Para lograrlo, tendremos que agregar un Claim de tipo

```http lines
http://schemas.microsoft.com/ws/2008/06/identity/claims/role
```

para cada uno de los grupos a los que está asignado un usuario.

Una vez agregados los claims, podemos asegurarnos fácilmente de que una acción específica esté disponible solo para usuarios `Admin` decorando el claim con el atributo `[Authorize(Roles = "Admin")]`. También puedes comprobar desde el código si un usuario pertenece a un role específico haciendo una llamada a `User.IsInRole("Admin")` desde el controlador.

El middleware OIDC de ASP.NET agregará automáticamente todos los claims devueltos en el <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, formato de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip> como claims a la `ClaimsIdentity`. Por lo tanto, tendríamos que extraer la información del claim `authorization`, deserializar el cuerpo JSON del claim y, para cada uno de los grupos, agregar un claim `http://schemas.microsoft.com/ws/2008/06/identity/claims/role` a la `ClaimsIdentity`.

```csharp lines expandable
app.UseOpenIdConnectAuthentication(new OpenIdConnectOptions("Auth0")
{
    // Se omite parte de la configuración por brevedad

    Events = new OpenIdConnectEvents
    {
        OnTicketReceived = context =>
        {
            var options = context.Options as OpenIdConnectOptions;

            // Obtener el ClaimsIdentity
            var identity = context.Principal.Identity as ClaimsIdentity;
            if (identity != null)
            {
                // Agregar los grupos como roles
                var authzClaim = context.Principal.FindFirst(c => c.Type == "authorization");
                if (authzClaim != null)
                {
                    var authorization = JsonConvert.DeserializeObject<Auth0Authorization>(authzClaim.Value);
                    if (authorization != null)
                    {
                        foreach (var group in authorization.Groups)
                        {
                            identity.AddClaim(new Claim(ClaimTypes.Role, group, ClaimValueTypes.String, options.Authority));
                        }
                    }
                }
            }

            return Task.FromResult(0);
        }
    }
});
```

```csharp lines
[Authorize(Roles = "Admin")]

{          
    return View();
}
```
