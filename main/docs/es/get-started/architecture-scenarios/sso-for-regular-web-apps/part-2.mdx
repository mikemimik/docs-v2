En esta sección, revisaremos toda la configuración que debemos aplicar desde el [Auth0 Dashboard](https://manage.auth0.com/#).

<div id="application">
  ## Aplicación
</div>

La parte de configuración de Auth0 comienza registrando la aplicación de parte de horas en el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip> como una **aplicación**. Una aplicación realiza solicitudes a recursos protegidos en nombre del <Tooltip tip="Resource Owner: Entidad (como un usuario o una aplicación) capaz de otorgar acceso a un recurso protegido." cta="Ver glosario" href="/docs/es/glossary?term=resource+owner">propietario del recurso</Tooltip> (usuario final).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El término &quot;aplicación&quot; no implica ninguna característica de implementación en particular. Una aplicación puede ser una web app, una aplicación móvil o una SPA. En el caso de ExampleCo, es una web app de ASP.NET Core.
</Callout>

Las principales características de una aplicación en Auth0 son:

* **Nombre**: El nombre canónico de la aplicación. Se usa para identificar la aplicación en el portal, correos electrónicos, logs y más.
* **<Tooltip tip="Client ID: Valor de identificación dado por Auth0 a su recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>** (solo lectura): El identificador único de la aplicación. Este es el ID que se usa en la aplicación al configurar la autenticación con Auth0. Es una cadena alfanumérica generada automáticamente.
* **<Tooltip tip="Client Secret: Secret utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Client+secret">secreto de cliente</Tooltip>** (solo lectura): Una cadena que se utiliza para firmar y validar tokens que se usarán en los distintos flujos de autenticación. Se genera automáticamente y debe mantenerse confidencial.
* **Dominio**: El nombre de dominio asignado a la cuenta de Auth0. El formato del dominio es `{account-name}.auth0.com` o `{account-name}.{location}.auth0.com`, por ejemplo `ExampleCo.auth0.com`.
* **URL de devolución de llamada**: La URL a la que se redirige al usuario después de autenticarse.

<div id="create-an-application">
  ### Crear una aplicación
</div>

El escenario de ExampleCo incluye solo una aplicación: la aplicación web de partes de horas. Por lo tanto, debemos configurar una aplicación en Auth0.

Para registrar una conexión de base de datos, ve al [Dashboard](https://manage.auth0.com/#) y, en el menú lateral, selecciona [Applications](https://manage.auth0.com/#/applications).

Haz clic en el botón **+ Create Application**. Se te pedirá el nombre y el tipo de la aplicación. Llamaremos a nuestra aplicación `Timesheet-App` y seleccionaremos `Regular Web Applications` como tipo de aplicación.

Cuando hagas clic en **Create** irás a la [vista Quickstart](https://manage.auth0.com/#/applications/\{yourClientId}/quickstart). Aquí puedes elegir la tecnología que piensas usar para crear tu aplicación y se mostrará la guía de inicio rápido correspondiente.

Las otras vistas disponibles son:

* [Settings](https://manage.auth0.com/#/applications/\{yourClientId}/settings)
* [Addons](https://manage.auth0.com/#/applications/\{yourClientId}/addons)
* [Connections](https://manage.auth0.com/#/applications/\{yourClientId}/connections)

<div id="configure-callback-urls">
  ### Configurar URLs de devolución de llamada
</div>

El campo **Allowed Callback URLs** contiene las URL a las que Auth0 redirigirá después de que el usuario se haya autenticado para que <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) complete el proceso de autenticación. Puedes especificar varias URL válidas separándolas por comas. Puedes usar el asterisco como comodín para subdominios, por ejemplo `*.google.com`. Asegúrate de especificar el protocolo, `http://` o `https://`; de lo contrario, la devolución de llamada puede fallar en algunos casos.

La URL de devolución de llamada de nuestro proyecto de ejemplo es `http://localhost:5000/signin-auth0`. Establece este valor en el campo **Allowed Callback URLs** si piensas usar nuestro ejemplo; de lo contrario, agrega la URL en la que decidiste desplegar tu aplicación.

<div id="connections">
  ## Conexiones
</div>

El siguiente paso es configurar los <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> que se usarán para la autenticación en la aplicación web. Cada proveedor de identidad se corresponde con una **conexión** en Auth0. Cada aplicación necesita al menos una conexión, y cada conexión puede usarse en más de una aplicación.

ExampleCo necesita configurar dos conexiones: una conexión de Active Directory para los empleados internos y una conexión de base de datos para los usuarios externos.

<Info>
  ### Proveedores de identidad compatibles

  Auth0 admite una amplia variedad de protocolos y proveedores de identidad:

  * Social: Permita que sus usuarios inicien sesión con Google, Facebook, LinkedIn, Github y muchos más.
  * Empresarial: Permita que sus usuarios inicien sesión con Active Directory, ADFS, LDAP, <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=SAML-P">SAML-P</Tooltip> y muchos más.
  * Conexiones de base de datos: Cree su propio repositorio de usuarios configurando una nueva conexión de base de datos y autentique a sus usuarios con correo electrónico/nombre de usuario y contraseña. Las credenciales pueden almacenarse de forma segura en el repositorio de usuarios de Auth0 o en su propia base de datos.
  * Autenticación <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip>: Permita que sus usuarios inicien sesión sin necesidad de recordar una contraseña y utilicen un canal de autenticación como SMS o correo electrónico.
</Info>

<div id="create-a-database-connection">
  ### Crear una conexión de base de datos
</div>

Para registrar una conexión de base de datos, sigue estos pasos:

1. En el [Dashboard](https://manage.auth0.com/#), ve a **Authentication &gt; Database**.
2. Selecciona **+ Create DB Connection**. En el prompt, introduce un nombre para la conexión y configura los requisitos adicionales según sea necesario.
3. Revisa el prompt para asegurarte de que sea correcto y, a continuación, selecciona **Create**.

Después de crear la conexión de base de datos, debes habilitar tus aplicaciones para la conexión. Una aplicación debe estar habilitada para la conexión para que los usuarios puedan acceder a ella usando esa conexión. Para habilitar tus aplicaciones, ve a la pestaña Applications de la nueva conexión y activa el interruptor de cada aplicación que necesite usar la conexión.

Para obtener más información sobre las conexiones de base de datos, consulta [Database Identity Providers](/docs/es/authenticate/database-connections).

<div id="create-an-active-directory-ldap-connection">
  ### Crear una conexión de Active Directory / LDAP
</div>

A continuación, configura tu conexión de Active Directory / LDAP. En el [Auth0 Dashboard](https://manage.auth0.com/#), ve a **Authentication &gt; empresarial**.

Allí debes crear la conexión AD / LDAP e instalar el Conector AD. Puedes encontrar más detalles en estos documentos:

* [Cómo conectar tu Active Directory con Auth0](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap)
* [Cómo instalar el Conector de Active Directory/LDAP](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El Conector AD/LDAP es un puente entre tu Active Directory y el servicio Auth0. Este puente es necesario porque AD normalmente está restringido a tu red interna, y Auth0 es un servicio en la nube que se ejecuta en un contexto completamente distinto. [Más información](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector)
</Callout>

Una vez que hayas configurado la conexión y el conector, asegúrate de habilitar tu aplicación para que use esta conexión AD / LDAP:

<Info>
  ### Compatibilidad con Kerberos

  El Conector AD/LDAP es compatible con Kerberos para facilitar la autenticación de tus usuarios cuando están en una máquina unida a un dominio dentro de la red corporativa. Para activar Kerberos en Active Directory, solo tienes que habilitar la opción en el Dashboard. Después de habilitar Kerberos, también podrás configurar los **rangos de IP**. Cuando los usuarios provengan de estos rangos de direcciones IP, esta información se expondrá en el endpoint de SSO, lo que significa que los SDKs del lado del cliente, como auth0.js y Lock, podrán detectar la compatibilidad con Kerberos y permitir la autenticación integrada de Windows.
  [Más información](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-with-kerberos)

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    Si habilitas Kerberos, tendrás que hacer algunos cambios en el archivo de configuración de AD/LDAP. Para más detalles, consulta: [Modificar la configuración del Conector AD/LDAP](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-connector-to-auth0).
  </Callout>
</Info>

Ahora que hemos diseñado nuestra solución y analizado las configuraciones necesarias del lado de Auth0, podemos continuar con la integración de Auth0 con nuestra aplicación web de partes de horas.
