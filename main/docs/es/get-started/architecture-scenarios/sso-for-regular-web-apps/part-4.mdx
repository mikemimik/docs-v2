En este tutorial cubrimos un escenario sencillo: una aplicación web tradicional, alojada en la nube, que usa Auth0 para la autenticación y aprovecha el repositorio de usuarios existente de Active Directory. Aprendimos qué ofrece <Tooltip tip="OpenID: Estándar abierto para autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) y por qué era la mejor opción para este caso de uso, cómo funciona el flujo de autenticación, qué es un <Tooltip tip="OpenID: Estándar abierto para autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> y cómo validarlo y manipularlo, cómo configurar aplicaciones y conexiones en el <Tooltip tip="Auth0 Dashboard: El principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>, cómo implementar el inicio de sesión y el cierre de sesión de usuarios con Lock, y cómo funcionan la gestión de sesiones y el control de acceso.

Empezamos describiendo el caso de uso y los requisitos, y luego explicamos cómo puede cumplirse cada uno y el razonamiento detrás de cada decisión tomada.

Usamos ASP .NET Core para la implementación de ejemplo de la aplicación web, aunque esperamos que, después de leer este documento, puedas crear una aplicación web de este tipo con el framework que prefieras.
