En este tutorial abordamos un escenario sencillo: una aplicación web tradicional, alojada en la nube, que usa Auth0 para la autenticación y aprovecha el repositorio de usuarios existente de Active Directory. Aprendimos qué ofrece <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) y por qué era la mejor opción para este caso de negocio, cómo funciona el flujo de autenticación, qué es un <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip> y cómo validarlo y manipularlo, cómo configurar aplicaciones y conexiones en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>, cómo implementar el inicio de sesión y el cierre de sesión de los usuarios con Lock, y cómo funcionan la gestión de sesiones y el control de acceso.

Empezamos describiendo el caso de negocio y los requisitos, y después explicamos cómo puede cumplirse cada uno y qué razonamiento hubo detrás de cada decisión tomada.

Usamos ASP .NET Core para la implementación de ejemplo de la aplicación web; aun así, esperamos que, después de leer este documento, puedas crear una aplicación web de este tipo con el framework que prefieras.
