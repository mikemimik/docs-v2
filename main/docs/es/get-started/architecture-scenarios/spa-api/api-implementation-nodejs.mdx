<div id="nodejs-api-implementation-spas-api">
  # Implementación de API en Node.js (SPAs + API)
</div>

Este documento forma parte del escenario de Arquitectura de SPAs + API y explica cómo implementar la API en Node.js. Consulta el escenario para obtener más información sobre la solución implementada.

El código fuente completo de la implementación de API en Node.js se encuentra en [este repositorio de GitHub](https://github.com/auth0-samples/auth0-pnp-exampleco-timesheets/tree/master/timesheets-api/node).

<div id="step-1-define-the-api-endpoints">
  ## Paso 1. Defina los endpoints de la API
</div>

Usaremos el [framework de aplicaciones web Express](http://expressjs.com/) para crear nuestra API de Node.js.

<div id="create-a-packagejson-file">
  ### Crea un archivo package.json
</div>

Crea una carpeta para tu API, entra en ella y ejecuta `npm init`. Esto creará tu archivo `package.json`.

Puedes dejar la configuración predeterminada o modificarla como prefieras.

El archivo `package.json` de nuestro ejemplo tiene el siguiente aspecto:

```json lines
{
  "name": "timesheets-api",
  "version": "1.0.0",
  "description": "API used to add timesheet entries for employees and contractors",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/auth0-samples/auth0-pnp-timesheets.git"
  },
  "author": "Auth0",
  "license": "MIT",
  "bugs": {
    "url": "https://github.com/auth0-samples/auth0-pnp-timesheets/issues"
  },
  "homepage": "https://github.com/auth0-samples/auth0-pnp-timesheets#readme"
}
```

<div id="install-the-dependencies">
  ### Instala las dependencias
</div>

A continuación, debemos configurar nuestras dependencias. Usaremos los siguientes módulos:

* **express**: Este módulo añade el [framework web de aplicaciones Express](https://expressjs.com/).
* **cors**: Este módulo añade compatibilidad para habilitar [CORS](https://en.wikipedia.org/wiki/Cross-origin_resource_sharing), lo cual es necesario porque se llamará a la API desde una aplicación de página única que se ejecuta en un dominio distinto dentro de un navegador web.
* **jwks-rsa**: Esta biblioteca recupera claves de firma RSA desde un endpoint JWKS (JSON Web Key Set). Con `expressJwtSecret`, podemos generar un proveedor de secretos que proporcione la clave de firma correcta a `express-jwt` según el `kid` del encabezado del <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, formato de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWT">JWT</Tooltip>. Para obtener más información, consulta el [repositorio de GitHub de node-jwks-rsa](https://github.com/auth0/node-jwks-rsa).
* **express-jwt**: Este módulo te permite autenticar solicitudes HTTP mediante tokens JWT en tus aplicaciones de Node.js. Proporciona varias funciones que facilitan el trabajo con JWT. Para obtener más información, consulta el [repositorio de GitHub de express-jwt](https://github.com/auth0/express-jwt).
* **body-parser**: Este es un middleware de Node.js para analizar el cuerpo de las solicitudes. Extrae todo el contenido del cuerpo de un flujo de solicitud entrante y lo expone en `req.body` de una forma más fácil de usar. Para obtener más información y conocer varias alternativas, consulta el repositorio de GitHub de body-parser.

Para instalar estas dependencias, ejecuta lo siguiente:

```bash wrap lines
npm install express cors express-jwt jwks-rsa body-parser express-jwt-authz --save
```

<div id="implement-the-endpoints">
  ### Implementa los endpoints
</div>

Ve al directorio de tu API y crea un archivo `server.js`. Tu código debe:

* Cargar las dependencias.
* Implementar los endpoints.
* Iniciar el servidor de la API.

Esta es nuestra implementación de ejemplo:

```javascript lines
const express = require('express');
const app = express();
const { expressjwt: jwt } = require('express-jwt');
const jwksRsa = require('jwks-rsa');
const cors = require('cors');
const bodyParser = require('body-parser');

// Habilitar CORS
app.use(cors());

// Habilitar el uso de middleware de análisis del cuerpo de la solicitud
app.use(bodyParser.json());
app.use(bodyParser.urlencoded({
  extended: true
}));

// Crear el endpoint de la API de parte de horas
app.post('/timesheets', function(req, res){
  res.status(201).send({message: "This is the POST /timesheets endpoint"});
})

// Iniciar el servidor de la API en localhost:8080
app.listen(8080);
```

Inicia el servidor de la API con `node server` y realiza una solicitud HTTP POST a `localhost:8080/timesheets`. Deberías ver una respuesta JSON con el mensaje `This is the POST /timesheets endpoint`.

Ahora ya tenemos nuestro endpoint, pero cualquiera puede llamarlo. Ve al siguiente párrafo para ver cómo podemos solucionar esto.

<div id="step-2-secure-the-api-endpoints">
  ## Paso 2. Protege los endpoints de la API
</div>

Para validar nuestro token usaremos la función `jwt`, proporcionada por el [middleware express-jwt](https://github.com/auth0/express-jwt#usage), y `jwks-rsa` para recuperar nuestro secreto. Las bibliotecas hacen lo siguiente:

1. `express-jwt` decodificará el token y pasará la solicitud, el header y el payload a `jwksRsa.expressJwtSecret`.
2. Luego, `jwks-rsa` descargará todas las claves de firma desde el endpoint JWKS y comprobará si alguna coincide con el `kid` del header del JWT. Si ninguna de las claves de firma coincide con el `kid` entrante, se generará un error. Si hay una coincidencia, pasaremos la clave de firma correcta a `express-jwt`.
3. Después, `express-jwt` continuará con su propia lógica para validar la firma del token, la expiración, `audience` y `issuer`.

Los pasos que seguiremos en nuestro código son:

* Crear la función middleware para validar el <Tooltip tip="token de acceso: Credencial de autorización, en forma de una cadena opaque o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip>.
* Habilitar el uso del middleware en nuestras rutas.

También puedes escribir algo de código para guardar realmente la hoja de hora en una base de datos. Esta es nuestra implementación de ejemplo (se omite parte del código por brevedad):

export const codeExample = `// configurar las dependencias - código omitido

// Habilitar CORS - código omitido

// Crear middleware para validar el JWT
const checkJwt = jwt({
  // Proporcionar dinámicamente una clave de firma en función del kid del encabezado y de las Signing Keys proporcionadas por el endpoint JWKS
  secret: jwksRsa.expressJwtSecret({
    cache: true,
    rateLimit: true,
    jwksRequestsPerMinute: 5,
    jwksUri: \`https://{yourDomain}/.well-known/jwks.json\`
  }),

  // Validar la audience y el issuer
  audience: '{YOUR_API_IDENTIFIER}', //reemplazar con la audience de tu API, disponible en Dashboard > APIs
  issuer: 'https://{yourDomain}/',
  algorithms: [ 'RS256' ]
});

// Habilitar el uso del middleware de análisis del cuerpo de la solicitud - código omitido

// crear el endpoint de la API de parte de horas - código omitido
app.post('/timesheets', checkJwt, function(req, res){
  var timesheet = req.body;

  // Guardar la hoja de hora en la base de datos...

  //enviar la respuesta

// iniciar el servidor de la API en localhost:8080 - código omitido`;

Si iniciamos nuestro servidor ahora y hacemos un HTTP POST a `localhost:8080/timesheets`, deberíamos obtener el error message `Missing or invalid token` (lo cual es perfectamente normal, ya que no enviamos un token de acceso en nuestra solicitud).

Para probar también el caso en el que sí funciona, necesitamos:

* Obtener un token de acceso. Para más detalles sobre cómo hacerlo, consulta: [Get an Access Token](/docs/es/get-started/architecture-scenarios/server-application-api#get-an-access-token).
* Invocar la API agregando un encabezado `Authorization` a nuestra solicitud con el valor `Bearer ACCESS_TOKEN` (donde `ACCESS_TOKEN` es el valor del token que obtuvimos en el primer paso).

<div id="step-3-check-the-application-permissions">
  ## Paso 3. Verifica los permisos de la aplicación
</div>

En este paso añadiremos a nuestra implementación la capacidad de comprobar si la aplicación tiene permisos (o `scope`) para usar nuestro endpoint y crear una hoja de hora. En particular, queremos asegurarnos de que el token tenga el `scope` correcto, que es `batch:upload`.

Para ello, usaremos el paquete de Node.js `express-jwt-authz`, así que añádelo a tu proyecto:

```bash lines
npm install express-jwt-authz --save
```

Ahora es tan sencillo como agregar una llamada a `jwtAuthz(...)` a tu middleware para garantizar que el JWT contenga un scope específico y así poder ejecutar un endpoint determinado.

Vamos a añadir una dependency adicional. La biblioteca **express-jwt-authz**, que se usa junto con express-jwt, valida el [JWT](/docs/es/secure/tokens/json-web-tokens) y garantiza que tenga los permisos correctos para llamar al endpoint deseado. Para obtener más información, consulta el [GitHub repository de express-jwt-authz](https://github.com/auth0/express-jwt-authz).

Esta es nuestra implementation de ejemplo (se omite parte del código por brevedad):

```javascript lines
// establecer dependencias - parte del código omitida
const jwtAuthz = require('express-jwt-authz');

// Habilitar CORS - código omitido

// Crear middleware para verificar el JWT - código omitido

// Habilitar el uso del middleware de análisis del cuerpo de la solicitud - código omitido

// crear el endpoint de la API de parte de horas
app.post('/timesheets', checkJwt, jwtAuthz(['create:timesheets'], { customUserKey: 'auth' }), function(req, res){
  var timesheet = req.body;

  // Guardar la hoja de hora en la base de datos...

  //enviar la respuesta
  res.status(201).send(timesheet);
})

// iniciar el servidor de la API en localhost:8080 - código omitido
```

Si invocamos nuestra API con un token que no incluya este scope, deberíamos obtener el mensaje de error Forbidden con el código de estado HTTP `403`. Puedes comprobarlo eliminando este scope de tu API.

<div id="step-4-determine-the-user-identity">
  ## Paso 4. Determinar la identidad del usuario
</div>

El `middleware` `express-jwt`, que se usa para validar el JWT, también establece `req.auth` con la información contenida en el JWT. Si quieres usar el `claim` `sub`` para identificar de forma única al usuario, puedes usar simplemente `req.auth.sub&#96;.

Sin embargo, en el caso de la aplicación de hojas de horas, queremos usar la dirección de correo electrónico del usuario como identificador único.

Lo primero que debemos hacer es escribir una rule que agregue la dirección de correo electrónico del usuario al token de acceso. Ve a la sección [Rules](https://manage.auth0.com/#/rules%7D) del Dashboard y haz clic en el botón **Create Rule**.

Puedes darle a la rule un nombre descriptivo, por ejemplo `Add email to Access Token`, y luego usar el siguiente código para la rule:

```javascript lines
function (user, context, callback) {
  const namespace = 'https://api.exampleco.com/';
  context.accessToken[namespace + 'email'] = user.email;
  callback(null, user, context);
}
```

El `namespace` se usa para garantizar que el claim tenga un nombre único y no entre en conflicto con los nombres de ninguno de los claims estándar de OIDC. Sin embargo, Auth0 admite custom claims con espacio de nombres y sin espacio de nombres. Para obtener más información sobre los custom claims, consulta [Create Custom Claims](/docs/es/secure/tokens/json-web-tokens/create-custom-claims).

A continuación, dentro de tu API, puedes recuperar el valor del claim desde `req.auth` y usarlo como la identidad única del usuario, que puedes asociar con los registros de la hoja de hora.

```js lines
app.get('/timesheets', checkJwt, jwtAuthz(['read:timesheets'], { customUserKey: 'auth' }), function(req, res) {
  var timesheet = req.body;

  // Asociar la hoja de horas con el usuario actual
  var userId = req.auth['https://api.exampleco.com/email'];
  timesheet.user_id = userId;

  // Guardar la hoja de horas en la base de datos...

  //enviar la respuesta
  res.status(201).send(timesheet);
});
```
