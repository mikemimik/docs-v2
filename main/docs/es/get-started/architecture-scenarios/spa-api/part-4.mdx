title: Conclusión (SPAs + API)
---

En este documento cubrimos un escenario sencillo en el que una API es utilizada por una aplicación de página única (SPA) para permitir que los empleados registren sus horas de trabajo.

Aprendimos sobre la concesión implícita, qué es un <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip>, cómo configurar una API en Auth0, cómo configurar una aplicación SPA para comunicarse de forma segura con esta API, cómo definir y proteger nuestros endpoints de API, cómo usar las bibliotecas proporcionadas para validar el Access Token y cómo obtener uno nuevo de Auth0.

Comenzamos describiendo el caso de uso y los requisitos, y luego explicamos cómo puede cumplirse cada uno y el razonamiento detrás de cada decisión que se tomó.

Usamos Node.js para la implementación de la API y Angular para la SPA. Aun así, esperamos que, después de revisar este documento, puedas desarrollar esto con las tecnologías que prefieras.
