title: Conclusión (SPAs + API)
---

En este documento analizamos un escenario sencillo en el que una Single-Page Application (SPA) usa una API para permitir que los empleados registren sus horas de trabajo.

Aprendimos sobre el flujo implícito, qué es un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip>, cómo configurar una API en Auth0, cómo configurar una aplicación SPA para comunicarse de forma segura con esta API, cómo definir y proteger nuestros endpoints de API, cómo usar las bibliotecas proporcionadas para validar el token de acceso y cómo obtener uno nuevo de Auth0.

Comenzamos describiendo el caso de uso y los requisitos, y luego explicamos cómo puede cumplirse cada uno y el razonamiento detrás de cada decisión que se tomó.

Usamos Node.js para implementar la API y Angular para la SPA. Aun así, esperamos que, después de leer este documento, puedas crear esto con las tecnologías que prefieras.
