En esta sección revisaremos toda la configuración que debemos aplicar en el [Auth0 Dashboard](https://manage.auth0.com/#).

<div id="create-the-api">
  ## Crear la API
</div>

Vaya a la [sección APIs](https://manage.auth0.com/#/apis) del Dashboard y haga clic en el botón **Crear API**.

Se le pedirá que proporcione los siguientes datos para su API:

* **Nombre**: un nombre descriptivo para la API. No afecta a ninguna funcionalidad.
* **Identificador**: un identificador único para la API. Recomendamos usar una URL, pero tenga en cuenta que no tiene que ser una URL disponible públicamente; Auth0 no hará ninguna llamada a su API. Este valor no se puede modificar después.
* **<Tooltip tip="Signing Algorithm: algoritmo usado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=Signing+Algorithm">Signing Algorithm</Tooltip>**: el algoritmo con el que se firmarán los tokens. Los valores disponibles son `HS256` y `RS256`. Al seleccionar `RS256`, el token se firmará con la clave privada del tenant. Para obtener más información sobre los algoritmos de firma, consulte [Signing Algorithms](/docs/es/get-started/applications/signing-algorithms).

Complete la información requerida y haga clic en el botón **Crear**.

En la configuración de tu API, ve a la pestaña Permissions. En esta sección puedes agregar los scopes para este caso de uso: `read:timesheets`, `create:timesheets`, `delete:timesheets` y `approve:timesheets`.

<div id="create-the-application">
  ## Crear la aplicación
</div>

Hay cuatro tipos de aplicación en Auth0:

* **Aplicación nativa** (utilizada por aplicaciones móviles o de escritorio),
* **Aplicación web de página única**,
* **Aplicación web tradicional** y
* **Aplicación de máquina a máquina** (utilizada por CLI, demonios o servicios que se ejecutan en tu backend).

Para este escenario, queremos crear una nueva aplicación para nuestra SPA, así que usaremos aplicación web de página única como tipo de aplicación.

Para crear una nueva aplicación, ve al [Dashboard](https://manage.auth0.com/#) y haz clic en la opción de menú [Applications](https://manage.auth0.com/#/applications) de la izquierda. Haz clic en el botón **+ Create Application**.

Asigna un nombre a tu aplicación (usaremos `Timesheets SPA`) y selecciona `Single-Page Web App` como tipo.

Haz clic en **Create**.

Eso es todo por ahora. Cuando hayamos terminado con la implementación de la SPA, volveremos al Dashboard y a la sección Settings de esta aplicación para hacer algunos cambios en su configuración.

<div id="define-roles">
  ### Definir roles
</div>

A continuación, vamos a configurar los dos roles: employee y gerente.

Ve a la pestaña **roles**, haz clic en el botón **Create Role** y selecciona la aplicación **Timesheets SPA**.

A continuación, sigue el mismo proceso para crear un rol `gerente` y asegúrate de haber seleccionado todos los permisos.

<div id="assign-users-to-roles">
  ### Asigna usuarios a roles
</div>

Debes asignar a todos los usuarios al rol `Gerente` o al rol `Employee`.

Puedes hacerlo yendo a la pestaña **Users** de Authorization Extension y seleccionando un usuario.

En la pantalla de información del usuario, ve a la pestaña **roles**. Haz clic en **Add Role to User** y selecciona el rol adecuado.

<div id="configuring-the-authorization-extension">
  ### Configuración de la Authorization Extension
</div>

También tendrás que asegurarte de que la Rule de la Authorization Extension esté publicada.

Para ello, haz clic en tu avatar en la esquina superior derecha de la Authorization Extension y selecciona **Configuración**.

Asegúrate de que **Permissions** estén habilitados y, a continuación, haz clic en **Publicar Rule**.

<div id="create-a-rule-to-validate-token-scopes">
  ### Cree una Rule para validar los scopes del token
</div>

El paso final de este proceso es crear una Rule para comprobar si los scopes contenidos en un <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> son válidos en función de los permisos asignados al usuario. Cualquier scope que no sea válido para un usuario debe eliminarse del Access Token.

En su <Tooltip tip="Auth0 Dashboard: El principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vaya a la pestaña **Rules**. Debería ver la Rule creada por la Authorization Extension. Haga clic en el botón **Create Rule** y seleccione la plantilla **Empty Rule**. Puede asignarle un nombre a la Rule, como **Access Token Scopes**, y luego especificar el siguiente código para la Rule:

```javascript lines
function (user, context, callback) {
  var permissions = user.permissions || [];
  var requestedScopes = context.request.body.scope || context.request.query.scope;
  var filteredScopes = requestedScopes.split(' ').filter( function(x) {
    return x.indexOf(':') < 0;
  });

  var allScopes = filteredScopes.concat(permissions);
  context.accessToken.scope = allScopes.join(' ');

  callback(null, user, context);
}
```

El código anterior garantizará que todos los tokens de acceso solo contengan los scopes con el formato correcto (p. ej., `action:area` o `delete:timesheets`) que sean válidos según los permisos del usuario. Cuando hayas terminado, puedes hacer clic en el botón **Save**.

Las Rules se ejecutan en el orden en que aparecen en la página Rules, así que asegúrate de que la nueva rule que creaste esté ubicada debajo de la rule de Authorization Extension, para que se ejecute después de la Authorization Extension rule.
