En esta sección revisaremos toda la configuración que debemos aplicar en el [Auth0 Dashboard](https://manage.auth0.com/#).

<div id="create-the-api">
  ## Crear la API
</div>

Ve a la [sección APIs](https://manage.auth0.com/#/apis) del Dashboard y haz clic en el botón **Create API**.

Se te pedirá que proporciones los siguientes datos para tu API:

* **Nombre**: un nombre descriptivo para la API. No afecta al funcionamiento.
* **Identificador**: un identificador único para la API. Te recomendamos usar una URL, pero ten en cuenta que no tiene que ser una URL pública; Auth0 no llamará a tu API en ningún caso. Este valor no se puede modificar después.
* **<Tooltip tip="Signing Algorithm: algoritmo usado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/docs/es/glossary?term=Signing+Algorithm">Signing Algorithm</Tooltip>**: el algoritmo con el que se firmarán los tokens. Los valores disponibles son `HS256` y `RS256`. Al seleccionar RS256, el token se firmará con la clave privada del tenant. Para obtener más información sobre los algoritmos de firma, consulta [Signing Algorithms](/docs/es/get-started/applications/signing-algorithms).

<Frame>![Dashboard - Applications - APIs - Create API - Popup](/docs/images/cdy7uua7fh8z/4kIH4BQc4vDel2zhJ6lwCE/094cff6e8e1788c39796eb28ab1925ca/2024-06-27_14-50-24.png)</Frame>

Completa la información requerida y haz clic en el botón **Create**.

<div id="configure-the-scopes">
  ## Configura los scopes
</div>

Una vez creada la aplicación, tendrás que configurar los scopes que las aplicaciones pueden solicitar durante la autorización.

En la configuración de tu API, ve a la pestaña Permissions. En esta sección puedes agregar los scopes para nuestro caso de uso: `read:timesheets`, `create:timesheets`, `delete:timesheets` y `approve:timesheets`.

<Frame>![Dashboard - Applications - APIs - Permissions](/docs/images/cdy7uua7fh8z/xxMOKezob6gNHJ0CcKhcg/ccbb9bbab29576a96ed8425329ecffb4/dashboard-apis-edit_view-permissions.png)</Frame>

<div id="create-the-application">
  ## Crear la aplicación
</div>

Hay cuatro tipos de aplicaciones en Auth0:

* **Aplicación nativa** (utilizada por aplicaciones móviles o de escritorio),
* **Aplicación web de página única**,
* **Aplicación web tradicional** y
* **Aplicación Machine to Machine** (utilizada por CLI, daemons o servicios que se ejecutan en tu backend).

Para este escenario, queremos crear una nueva aplicación para nuestra SPA, por lo que usaremos Aplicación de página única como tipo de aplicación.

Para crear una nueva aplicación, ve al [Dashboard](https://manage.auth0.com/#) y haz clic en la opción de menú [Applications](https://manage.auth0.com/#/applications) de la izquierda. Haz clic en el botón **+ Create Application**.

Asigna un nombre a tu aplicación (usaremos `Timesheets SPA`) y selecciona `Single-Page Web App` como el tipo.

Haz clic en **Create**.

Eso es todo por ahora. Cuando terminemos con la implementación de la SPA, volveremos al Dashboard y a la configuración de esta aplicación para hacer algunos cambios.

<div id="configure-the-authorization-extension">
  ## Configurar la Authorization Extension
</div>

Deberá asegurarse de que la Authorization Extension esté instalada en su tenant. Puede consultar la [documentación de la Authorization Extension](/docs/es/customize/extensions/authorization-extension) para obtener más información sobre cómo hacerlo.

<div id="define-permissions">
  ### Definir permisos
</div>

Ahora definirá los permisos necesarios, de acuerdo con los scopes que ya definió: `read:timesheets`, `create:timesheets`, `delete:timesheets` y `approve:timesheets`.

En la Authorization Extension, haga clic en la pestaña **Permissions** y luego en el botón **Create Permission**.

En el cuadro de diálogo, complete los detalles de cada permiso.

Asegúrese de que el nombre del permiso sea exactamente el mismo que el scope correspondiente:

<Frame>![Dashboard - Extensions - Authorization Extension - Create Permission](/docs/images/cdy7uua7fh8z/6G4X14X3A73IDRwOrkmttq/e0c20c44a32e0b39a6d8fb72731ffbaf/create-permission.png)</Frame>

Continúe creando los permisos para todos los scopes restantes:

<div id="define-roles">
  ### Definir roles
</div>

A continuación, configuremos los dos roles: Employee y gerente.

Ve a la pestaña **Roles**, haz clic en el botón **Create Role** y selecciona la aplicación **Timesheets SPA**.

Establece **Name** y **Description** como `Employee`, y selecciona los permisos `delete:timesheets`, `create:timesheets` y `read:timesheets`. Haz clic en **Save**.

A continuación, sigue el mismo proceso para crear un rol de `Manager` y asegúrate de haber seleccionado todos los permisos.

<div id="assign-users-to-roles">
  ### Asigna Users a roles
</div>

Debes asignar todos los usuarios al rol `gerente` o al rol `Employee`.

Puedes hacerlo yendo a la pestaña **Users** en Authorization Extension y seleccionando un usuario.

En la pantalla de información del usuario, ve a la pestaña **Roles**. Haz clic en **Add Role to User** y selecciona el rol adecuado.

<div id="configuring-the-authorization-extension">
  ### Configuración de Authorization Extension
</div>

También deberás asegurarte de que la Rule de Authorization Extension esté publicada.

Para ello, haz clic en el avatar de tu usuario en la esquina superior derecha de Authorization Extension y selecciona **Configuración**.

Asegúrate de que **Permissions** estén habilitados y, luego, haz clic en **Publicar Rule**.

<div id="create-a-rule-to-validate-token-scopes">
  ### Crear una Rule para validar los scopes del token
</div>

El último paso de este proceso es crear una Rule para comprobar si los scopes incluidos en un <Tooltip tip="Access Token: Credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Access Token</Tooltip> son válidos según los permisos asignados al usuario. Cualquier scope que no sea válido para un usuario debe eliminarse del Access Token.

En tu <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, ve a la pestaña **Rules**. Deberías ver la Rule creada por la Authorization Extension. Haz clic en el botón **Create Rule** y selecciona la plantilla **Empty Rule**. Puedes darle un nombre a la Rule, como **Access Token Scopes**, y luego especificar el siguiente código para la Rule:

```javascript lines
function (user, context, callback) {
  var permissions = user.permissions || [];
  var requestedScopes = context.request.body.scope || context.request.query.scope;
  var filteredScopes = requestedScopes.split(' ').filter( function(x) {
    return x.indexOf(':') < 0;
  });

  var allScopes = filteredScopes.concat(permissions);
  context.accessToken.scope = allScopes.join(' ');

  callback(null, user, context);
}
```

El código anterior garantizará que todos los Access Tokens solo contengan los scopes con el formato correcto (p. ej., `action:area` o `delete:timesheets`) que sean válidos según los permisos del usuario. Cuando termines, puedes hacer clic en el botón **Save**.

Las Rules se ejecutan en el orden en que aparecen en la página Rules, así que asegúrate de que la nueva Rule que creaste esté ubicada debajo de la Rule de Authorization Extension, para que se ejecute después de la Authorization Extension rule.
