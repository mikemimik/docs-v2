Esta guía es relevante para **todas** las partes interesadas del proyecto. Recomendamos leerla completa al menos una vez, incluso si ya has comenzado tu recorrido con Auth0. Ofrecemos una guía de planificación del proyecto en formato PDF, información sobre cómo empezar cada fase del proceso de implementación y listas de verificación para ayudarte a gestionar las tareas de cada fase.

Hay muchas formas distintas de integrar Auth0 en la arquitectura de un proyecto B2B IAM. La flexibilidad de Auth0 permite cubrir una amplia variedad de casos de uso; sin embargo, es posible que tu proyecto no necesite todas las capacidades que ofrece Auth0. Saber qué implementar, cuándo hacerlo y cuál es la mejor manera de abordarlo te ayudará a centrarte en completar las tareas necesarias en el momento oportuno.

<div id="multiple-organization-architecture-multitenancy">
  ## Arquitectura de múltiples organizaciones (Multitenancy)
</div>

Muchas plataformas B2B implementan algún tipo de aislamiento y/o branding para las organizaciones de sus clientes, y esto puede añadir complejidad a cualquier sistema de gestión de identidades y accesos (IAM). Si este es su caso, le recomendamos que se tome un tiempo para consultar nuestra guía y las prácticas recomendadas para este tipo de entorno.

[Arquitectura de múltiples organizaciones](/docs/es/get-started/architecture-scenarios/multiple-organization-architecture)
