`Auth0ComponentProvider` es la capa de orquestación de [Auth0 Universal Components](/docs/es/get-started/universal-components/universal-components-overview). Los Auth0 SDKs administran sesiones y tokens, y `Auth0ComponentProvider` traduce ese estado de identidad en un contexto funcional de interfaz de usuario alineado con la marca. `Auth0ComponentProvider` se asegura de que los componentes, como la inscripción en MFA, tengan los permisos, los datos en caché y los estilos necesarios.

<div id="benefits">
  ## Beneficios
</div>

`Auth0ComponentProvider` es el contenedor raíz obligatorio para todos los Auth0 Universal Components. Los beneficios de usar este contenedor son:

* **Alineación de identidades**: Cierra la brecha entre los [Auth0 SDKs](/docs/es/libraries) y la [My Account API](/docs/es/manage-users/my-account-api) para garantizar que las solicitudes se firmen con tokens con alcance de usuario.
* **Optimización del rendimiento**: Implementa una caché compartida de **TanStack Query**, ajustada específicamente para flujos de identidad, a fin de evitar llamadas a la API redundantes y cambios en el diseño.
* **Consistencia del sistema de diseño**: Propaga variables de **Tailwind CSS** y temas compatibles con **Shadcn** en todo el árbol de componentes.
* **Comentarios globales**: Administra un sistema unificado de notificaciones toast para alertas de seguridad y el estado del flujo de trabajo.

<div id="integration-architecture">
  ## Arquitectura de integración
</div>

Anida `Auth0ComponentProvider` en tu proveedor de autenticación (`Auth0Provider`). Esta configuración permite acceder al estado de autenticación y a los métodos de obtención de token necesarios para llamar a las API de Auth0.

<div id="configure-auth0componentprovider">
  ## Configurar `Auth0ComponentProvider`
</div>

<Tabs>
  <Tab title="React">
    ```tsx App.tsx
    import { Auth0Provider } from "@auth0/auth0-react";
    import { Auth0ComponentProvider } from "@auth0/universal-components-react/spa";

    function App() {
      return (
        <Auth0Provider
          domain="your-tenant.auth0.com"
          clientId="YOUR_CLIENT_ID"
          authorizationParams={{ redirect_uri: window.location.origin }}
          interactiveErrorHandler='popup' // Necesario para gestionar los desafíos de autenticación step-up mediante el popup de Universal Login
        >
          <Auth0ComponentProvider domain="your-tenant.auth0.com">
            {/* Tu aplicación con Auth0 Universal Components */}
          </Auth0ComponentProvider>
        </Auth0Provider>
      );
    }
    ```

    **toastSettings**

    La configuración de Toast admite dos tipos de proveedor: **Sonner** (predeterminado) o **custom**. Cada proveedor tiene su propia estructura de configuración para mayor seguridad de tipos.

    <Tabs>
      <Tab title="Proveedor Sonner (predeterminado)">
        | Propiedad              | Tipo            | Predeterminado | Descripción                                                                                                                                                                               |
        | ---------------------- | --------------- | -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
        | `provider`             | `"sonner"`      | `"sonner"`     | Usa la biblioteca integrada de toasts Sonner                                                                                                                                              |
        | `settings.position`    | `ToastPosition` | `"top-right"`  | Posición en la que aparecen los toasts: &quot;top-left&quot;, &quot;top-right&quot;, &quot;bottom-left&quot;, &quot;bottom-right&quot;, &quot;top-center&quot;, &quot;bottom-center&quot; |
        | `settings.duration`    | `number`        | `4000`         | Duración en milisegundos antes de que el toast se cierre automáticamente (valor predeterminado de Sonner)                                                                                 |
        | `settings.maxToasts`   | `number`        | -              | Número máximo de toasts visibles al mismo tiempo                                                                                                                                          |
        | `settings.dismissible` | `boolean`       | `true`         | Si los toasts se pueden cerrar manualmente mediante la interacción del usuario (valor predeterminado de Sonner)                                                                           |
        | `settings.closeButton` | `boolean`       | `true`         | Si se muestra un botón de cierre explícito en los toasts                                                                                                                                  |

        ```tsx Sonner Provider Example
        const toastSettings = {
          provider: 'sonner', // Opcional, este es el valor predeterminado
          settings: {
            position: 'top-center',
            duration: 6000,
            maxToasts: 5,
            dismissible: true,
            closeButton: true
          }
        };
        ```
      </Tab>

      <Tab title="Custom Provider">
        | Propiedad         | Tipo                         | Obligatorio | Descripción                                  |
        | ----------------- | ---------------------------- | ----------- | -------------------------------------------- |
        | `provider`        | `"custom"`                   | Sí          | Usa tu implementación personalizada de toast |
        | `methods.success` | `(message: string) => void`  | No          | Controlador Custom para toast de éxito       |
        | `methods.error`   | `(message: string) => void`  | No          | Controlador Custom para toast de error       |
        | `methods.warning` | `(message: string) => void`  | No          | Controlador Custom para toast de advertencia |
        | `methods.info`    | `(message: string) => void`  | No          | Controlador Custom para toast informativo    |
        | `methods.dismiss` | `(toastId?: string) => void` | No          | Controlador Custom para cerrar               |

        ```tsx Custom Provider Example
        const toastSettings = {
          provider: 'custom',
          methods: {
            success: (message: string) => {
              // Tu implementación personalizada para toast de éxito
              myToastLibrary.success(message);
            },
            error: (message: string) => {
              // Tu implementación personalizada para toast de error
              myToastLibrary.error(message);
            },
            warning: (message: string) => {
              // Tu implementación personalizada para toast de advertencia
              myToastLibrary.warning(message);
            },
            info: (message: string) => {
              // Tu implementación personalizada para toast informativo
              myToastLibrary.info(message);
            },
            dismiss: (toastId?: string) => {
              // Tu implementación personalizada para cerrar
              myToastLibrary.dismiss(toastId);
            }
          }
        };
        ```

    ## Estado y rendimiento

    Ajusta con precisión el almacenamiento en caché de TanStack Query para cada componente de Auth0. Los valores predeterminados mantienen los datos actualizados durante dos minutos, los eliminan después de cinco minutos y omiten las recargas al recuperar el foco de la ventana.

    <table class="table">
      <thead>
        <tr>
          <th>Propiedad</th>
          <th>Tipo</th>
          <th>Predeterminado</th>
          <th>Descripción</th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            <code>enabled</code>
          </td>

          <td>
            <code>boolean</code>
          </td>

          <td>
            <code>true</code>
          </td>

          <td>
            Activa o desactiva la caché por completo. Cuando se establece en <code>false</code>, se deshabilitan los datos obsoletos y las entradas en caché se eliminan rápidamente.
          </td>
        </tr>

        <tr>
          <td>
            <code>staleTime</code>
          </td>

          <td>
            <code>number</code>
          </td>

          <td>
            <code>120000</code>
          </td>

          <td>
            Milisegundos antes de que los datos queden obsoletos. El valor predeterminado es de dos minutos. Auméntelo para dashboards y redúzcalo para flujos de trabajo críticos.
          </td>
        </tr>

        <tr>
          <td>
            <code>gcTime</code>
          </td>

          <td>
            <code>number</code>
          </td>

          <td>
            <code>300000</code>
          </td>

          <td>
            Milisegundos antes de que se eliminen las consultas inactivas. El valor predeterminado es de cinco minutos.
          </td>
        </tr>

        <tr>
          <td>
            <code>refetchOnWindowFocus</code>
          </td>

          <td>
            <code>boolean | &quot;always&quot;</code>
          </td>

          <td>
            <code>false</code>
          </td>

          <td>
            Controla si las consultas se vuelven a obtener cuando el navegador recupera el foco. Use{" "}
            <code>&quot;always&quot;</code> para garantizar una actualización estricta.
          </td>
        </tr>
      </tbody>
    </table>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Desactivar el almacenamiento en caché:** Pasa `{ enabled: false }`. Esto establece automáticamente `staleTime` en 0 y reduce la ventana de recolección de basura a cinco segundos, para que en cada renderizado se obtengan datos nuevos.

      **Consejo:** Mantén habilitado el almacenamiento en caché, pero reduce `staleTime` al integrar con paneles de administración que requieren actualizaciones casi en tiempo real.
    </Callout>

    <Tabs>
      <Tab title="Ajustar el TTL">
        ```tsx
        <Auth0ComponentProvider
          domain="your-tenant.auth0.com"
          cacheConfig={{
            staleTime: 10 * 60 * 1000,
            gcTime: 15 * 60 * 1000,
            refetchOnWindowFocus: true,
          }}
        >
          <App />
        </Auth0ComponentProvider>
        ```
      </Tab>

      <Tab title="Desactivar caché">
        ```tsx
        <Auth0ComponentProvider
          domain="your-tenant.auth0.com"
          cacheConfig={{ enabled: false }}
        >
          <App />
        </Auth0ComponentProvider>
        ```

    ## Configurar el modo proxy

    Para aplicaciones de Next.js, usa el modo proxy con autenticación del lado del servidor:

    ```tsx layout.tsx
    import { Auth0ComponentProvider } from "@auth0/universal-components-react/rwa";

    export default function RootLayout({ children }) {
      return (
        <html lang="en">
          <body>
            <Auth0ComponentProvider
              mode="proxy"
              domain="your-tenant.auth0.com"
              proxyConfig={{ baseUrl: "/api/auth" }}
            >
              {children}
            </Auth0ComponentProvider>
          </body>
        </html>
      );
    }
    ```
