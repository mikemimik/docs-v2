Auth0 Universal Components se basa en un sistema de diseño flexible que utiliza [Tailwind CSS](https://tailwindcss.com/docs/installation/using-vite) y [primitivas de Radix UI](https://www.radix-ui.com/primitives/docs/overview/introduction). El estilo se gestiona mediante [`Auth0ComponentProvider`](/docs/es/get-started/universal-components/auth0-component-provider), lo que te permite elegir entre preajustes de alto nivel o sobrescrituras detalladas de variables CSS.

Esta arquitectura garantiza que tus flujos de identidad tengan el aspecto y el comportamiento de partes nativas de tu aplicación.

<Tabs>
  <Tab title="React">
    ## Primeros pasos

    1. Importa la hoja de estilos.

    ```tsx
    import "@auth0/universal-components-react/styles";
    ```

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Usuarios de Tailwind v4:** Añadan `@import "@auth0/universal-components-react/tailwind"` a su archivo CSS.
    </Callout>

    2. Añade el color de tu marca.

    ```css
    :root {
      --primary: #4f46e5; /* el color de tu marca */
      --primary-foreground: #ffffff; /* texto en los botones */
    }
    ```

    Listo. Todos los botones, enlaces y estados activos ahora usan tu marca.

    ## Temas predefinidos

    Pasa `themeSettings` a `Auth0ComponentProvider` para cambiar entre temas integrados.

    ```tsx App.tsx
    <Auth0ComponentProvider
      themeSettings={{
        mode: "light", // 'light' | 'dark'
        theme: "default", // 'default' | 'minimal' | 'rounded'
      }}
    >
      <App />
    </Auth0ComponentProvider>
    ```

    ## Componentes de estilo

    Cada componente acepta una prop `styling` para sobrescrituras puntuales sin afectar los estilos globales.

    ```tsx
    <SsoProviderTable
      styling={{
        variables: {
          common: {
            "--primary": "#059669", // override solo para este componente
          },
          light: { "--card": "#f8fafc" },
          dark: { "--card": "#1e293b" },
        },
        classes: {
          "SsoProviderTable-header": "shadow-lg",
          "SsoProviderTable-table": "rounded-xl",
        },
      }}
    />
    ```

    **Cuándo usar:**

    * `variables` - sobrescrituras de propiedades personalizadas de CSS restringidas al componente
    * `classes` - clases de Tailwind o personalizadas aplicadas a partes específicas del componente

    ## Personalizaciones comunes

    ### Colores de marca (Hex)

    Convierte el color hexadecimal de tu marca al formato que usa CSS:

    ```css
    :root {
      /* Usando hex directamente */
      --primary: #4f46e5;
      --primary-foreground: #ffffff;

      /* O oklch para una mejor manipulación del color */
      --primary: oklch(50% 0.2 265); /* morado */
    }
    ```

    ### Esquinas redondeadas

    Para una estética más redondeada en todos los componentes:

    ```css
    :root {
      --radius-lg: 16px; /* botones, entradas */
      --radius-xl: 20px; /* tarjetas */
      --radius-2xl: 24px; /* modales */
    }
    ```

    ### Tipografía compacta

    Para interfaces de usuario más densas:

    ```css
    :root {
      --font-size-page-header: 1.75rem;
      --font-size-heading: 1.25rem;
      --font-size-title: 1rem;
      --font-size-body: 0.875rem;
    }
    ```

    ## Modo oscuro

    Los componentes responden automáticamente a la configuración `mode` en `themeSettings`. Para sincronizar con el modo oscuro de tu aplicación:

    ```tsx
    function App() {
      const [isDark, setIsDark] = useState(false);

      return (
        <Auth0ComponentProvider
          themeSettings={{
            mode: isDark ? "dark" : "light",
          }}
        >
          <YourApp />
        </Auth0ComponentProvider>
      );
    }
    ```

    O usar la preferencia del sistema:

    ```tsx
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;

    <Auth0ComponentProvider
      themeSettings={{ mode: prefersDark ? 'dark' : 'light' }}
    >
    ```

    ## Componentes de estilo

    Cada componente acepta una prop `styling` para sobrescrituras puntuales sin afectar los estilos globales.

    ```tsx
    <SsoProviderTable
      styling={{
        variables: {
          common: {
            "--primary": "#059669", // override solo para este componente
          },
          light: { "--card": "#f8fafc" },
          dark: { "--card": "#1e293b" },
        },
        classes: {
          "SsoProviderTable-header": "shadow-lg",
          "SsoProviderTable-table": "rounded-xl",
        },
      }}
    />
    ```

    **Cuándo usar:**

    * `variables` - sobrescrituras de propiedades personalizadas de CSS restringidas al componente
    * `classes` - clases de Tailwind o personalizadas aplicadas a partes específicas del componente

    ## Personalizaciones comunes

    ### Colores de marca (Hex)

    Convierte el color hexadecimal de tu marca al formato que se usa en CSS:

    ```css
    :root {
      /* Usando hex directamente */
      --primary: #4f46e5;
      --primary-foreground: #ffffff;

      /* O oklch para una mejor manipulación del color */
      --primary: oklch(50% 0.2 265); /* morado */
    }
    ```

    ### Esquinas redondeadas

    Para una estética más redondeada en todos los componentes:

    ```css
    :root {
      --radius-lg: 16px; /* botones, entradas */
      --radius-xl: 20px; /* tarjetas */
      --radius-2xl: 24px; /* modales */
    }
    ```

    ### Tipografía compacta

    Para interfaces de usuario más densas:

    ```css
    :root {
      --font-size-page-header: 1.75rem;
      --font-size-heading: 1.25rem;
      --font-size-title: 1rem;
      --font-size-body: 0.875rem;
    }
    ```

    ## Modo oscuro

    Los componentes responden automáticamente a la configuración `mode` en `themeSettings`. Para sincronizar con el modo oscuro de tu aplicación:

    ```tsx
    function App() {
      const [isDark, setIsDark] = useState(false);

      return (
        <Auth0ComponentProvider
          themeSettings={{
            mode: isDark ? "dark" : "light",
          }}
        >
          <YourApp />
        </Auth0ComponentProvider>
      );
    }
    ```

    O usar la preferencia del sistema:

    ```tsx
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;

    <Auth0ComponentProvider
      themeSettings={{ mode: prefersDark ? 'dark' : 'light' }}
    >
    ```
