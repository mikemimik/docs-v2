---
title: Crear una interfaz de Delegated Admin con My Organization API
description: Describe cómo usar Universal Components para crear interfaces de Administración delegada con My Organization API de Auth0.
sidebarTitle: Crear una interfaz de Delegated Admin
---

Universal Components le permite crear un Dashboard de administración de autoservicio dentro de su propia aplicación. Esto permite que sus clientes B2B, delegados como administradores de su [Auth0 Organization](/docs/es/manage-users/organizations), gestionen su propia imagen de marca, proveedores de identidad y dominios verificados sin necesidad de acceder al Auth0 Dashboard ni de recurrir a un equipo de soporte para realizar acciones administrativas básicas.

<div id="how-it-works">
  ## Cómo funciona
</div>

Universal Components aprovecha la [My Organization API](/docs/es/api/myorganization), que opera con tokens con alcance de usuario, para crear un [Dashboard administrativo de SSO de autoservicio](/docs/es/authenticate/enterprise-connections/self-service-enterprise-configuration). Cuando un administrador inicia sesión en el panel integrado en tu aplicación, el SDK de Auth0 obtiene un token de acceso que contiene scopes específicos de la organización para privilegios de administrador. Los componentes usan este token para llamar a la API en nombre del usuario que inició sesión, lo que garantiza que los administradores delegados solo puedan modificar la Auth0 Organization a la que pertenecen.

<div id="available-components">
  ### Componentes disponibles
</div>

Cada componente se asigna a endpoints específicos dentro de [My Organization API](/docs/es/api/myorganization) para realizar tareas administrativas.

| **Componente**                                                                                                                                              | **API endpoint**                      |
| :---------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------ |
| **OrganizationDetailsEdit** — Edita el nombre de la organización, el nombre para mostrar, los colores de branding y el logotipo.                            | `/my-org/details`                     |
| **SsoProviderTable** — Enumera, habilita, deshabilita, elimina y desvincula proveedores de identidad de SSO.                                                | `/my-org/identity-providers`          |
| **SsoProviderCreate** — Asistente de varios pasos para crear proveedores de SSO (Okta, ADFS, SAML, OIDC, Google Workspace, Azure AD, Ping Federate).        | `/my-org/identity-providers`          |
| **SsoProviderEdit** — Configura los ajustes de autenticación, habilita el aprovisionamiento y administra los tokens de SCIM y las asociaciones de dominios. | `/my-org/identity-providers/{idp_id}` |
| **DomainTable** — Crea, verifica, elimina y asocia dominios con proveedores de identidad.                                                                   | `/my-org/domains`                     |

<div id="configure-auth0-dashboard">
  ## Configurar Auth0 Dashboard
</div>

Antes de usar cualquier componente de My Organization, debes configurar tu tenant de Auth0 con las [API](/docs/es/api), las aplicaciones y los [scopes](/docs/es/api/myorganization) adecuados.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">

    ## **Crear aplicación**

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Este ejemplo usa una aplicación web de página única (SPA). Para los frameworks de React con renderizado del lado del servidor, consulta la pestaña de Next.js.
    </Callout>

    1. Ve a [**Auth0 Dashboard &gt; Applications &gt; Applications**](https://manage.auth0.com/#/applications). Selecciona **Create Application**.

    2. Elige **Single Page Web Applications**.

    3. En la pestaña **Settings**, añade `http://localhost:5173` a los siguientes campos:
       <br />A. Allowed Callback URLs
       <br />B. Allowed Logout URLs

    4. En **Experiencia de inicio de sesión**, selecciona:
       <br />A. Business Users.
       <br />B. (Opcional) Prompt for Organization.

    ## **Habilitar la My Organization API**

    1. Vaya a [**Dashboard &gt; Applications &gt; APIs**](https://manage.auth0.com/#/applications).
    2. Seleccione **[My Organization API](/docs/es/api/myorganization)** y asegúrese de que esté habilitada para su tenant.

    ## **Configurar el acceso a la aplicación**

    1. Vaya a la pestaña **Acceso a la aplicación**.
    2. Seleccione **Edit** en la aplicación que desea usar.
    3. Configure los siguientes ajustes:
       <br />A. [Perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile#connection-profile): Seleccione o cree un perfil con mapeo de atributos de conexión.
       <br />B. [Perfil de atributos de usuario](/docs/es/authenticate/enterprise-connections/user-attribute-profile#user-attribute-profile): Seleccione o cree un perfil con mapeo de atributos de usuario.
       <br />C. [Proveedores de identidad](/docs/es/authenticate/identity-providers) compatibles: Habilite los proveedores que sus clientes puedan usar.
       <br />D. Comportamiento de eliminación de conexiones: Elija **Permitir** o **Permitir si está vacía**.
       <br />    • **Permitir**: Los usuarios pueden eliminar conexiones, lo que elimina a todos los usuarios de esa conexión.
       <br />    • **Permitir si está vacía**: Los usuarios solo pueden eliminar conexiones sin usuarios.
       <br />E. Autorización de acceso de usuarios: Elija **No autorizado**, **Autorizado** o **Todos**.
       <br />    • **No autorizado**: No se concede ningún permiso.
       <br />    • **Autorizado**: Seleccione permisos específicos.
       <br />    • **Todos**: Incluye todos los permisos actuales y futuros.
       <br />F. Autorización de acceso de credenciales del cliente: Elija **No autorizado**, **Autorizado** o **Todos**.
    4. Selecciona **Guardar**.

    ## **Setup Database y usuario**

    1. Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) para crear una conexión de base de datos.
    2. Seleccione la pestaña **Applications** de la conexión y habilite su nueva aplicación SPA.
    3. Cree un usuario de prueba en esta base de datos para realizar las pruebas iniciales.

    ## **Setup Role**

    1. Crea un rol (p. ej., &quot;Organization Admin&quot;).
    2. Agrega al rol los scopes requeridos de My Organization API.

    <Accordion title="Scopes requeridos">
      ```
          read:my_org:details
          update:my_org:details
          create:my_org:identity_providers
          read:my_org:identity_providers
          update:my_org:identity_providers
          delete:my_org:identity_providers
          update:my_org:identity_providers_detach
          create:my_org:identity_providers_domains
          delete:my_org:identity_providers_domains
          read:my_org:domains
          delete:my_org:domains
          create:my_org:domains
          update:my_org:domains
          read:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_scim_tokens
          delete:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_provisioning
          read:my_org:identity_providers_provisioning
          delete:my_org:identity_providers_provisioning
          read:my_org:configuration
      ```
    </Accordion>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      El token del usuario solo incluirá los permisos que existan tanto en su rol asignado como en la configuración de User Access Authorization establecida en el paso anterior.
    </Callout>

    ## **Crear organización**

    1. Vaya a [**Organizations &gt; Crear organización**](https://manage.auth0.com/#/organizations).
    2. Configura la organización de Auth0:
       <br />A. En **Members**: añade tu usuario de prueba y asígnale el rol &quot;Organization Admin&quot;.
       <br />B. En **Connections**: habilita tu conexión de base de datos.

    ## **Configurar Variables de Entorno**

    1. Crea un archivo `.env` en la raíz de tu proyecto React:

    ```bash
    VITE_AUTH0_DOMAIN=your-domain.auth0.com
      VITE_AUTH0_CLIENT_ID=your-spa-client-id
    ```
