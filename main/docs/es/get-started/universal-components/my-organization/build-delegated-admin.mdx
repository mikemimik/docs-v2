---
title: Crear una interfaz de Delegated Admin con My Organization API
description: Describe cómo usar Universal Components para crear interfaces de administración delegada con My Organization API de Auth0.
sidebarTitle: Crear una interfaz de Delegated Admin
---

Universal Components te permite crear un panel de administración de autoservicio dentro de tu propia aplicación. Esto permite que tus clientes B2B, designados como administradores de tu [organización de Auth0](/docs/es/manage-users/organizations), gestionen su propia imagen de marca, proveedores de identidad y dominios verificados sin necesidad de acceder a Auth0 Dashboard ni de recurrir a un equipo de soporte para realizar acciones administrativas básicas.

<div id="how-it-works">
  ## Cómo funciona
</div>

Universal Components aprovecha la [My Organization API](/docs/es/api/myorganization), que funciona con tokens con alcance de usuario, para crear un [panel administrativo de SSO de autoservicio](/docs/es/authenticate/enterprise-connections/self-service-enterprise-configuration). Cuando un administrador inicia sesión en el panel integrado en tu aplicación, el SDK de Auth0 obtiene un token de acceso que contiene scopes específicos de la organización para privilegios de administrador. Los componentes usan este token para llamar a la API en nombre del usuario que inició sesión, lo que garantiza que los administradores delegados solo puedan modificar la organización de Auth0 específica a la que pertenecen.

<div id="available-components">
  ### Componentes disponibles
</div>

Cada componente está asociado a endpoints específicos dentro de la [My Organization API](/docs/es/api/myorganization) para realizar tareas administrativas.

| **Componente**                                                                                                                                       | **endpoint de API**                   |
| :--------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------ |
| **OrganizationDetailsEdit** — Edita el nombre de la organización, el nombre para mostrar, los colores de la marca y el logotipo.                     | `/my-org/details`                     |
| **SsoProviderTable** — Muestra, habilita, deshabilita, elimina y desvincula proveedores de identidad para SSO.                                       | `/my-org/identity-providers`          |
| **SsoProviderCreate** — Asistente de varios pasos para crear proveedores de SSO (Okta, ADFS, SAML, OIDC, Google Workspace, Azure AD, Ping Federate). | `/my-org/identity-providers`          |
| **SsoProviderEdit** — Configura la autenticación, habilita el aprovisionamiento y gestiona tokens SCIM y asociaciones de dominios.                   | `/my-org/identity-providers/{idp_id}` |
| **DomainTable** — Crea, verifica, elimina y asocia dominios con proveedores de identidad.                                                            | `/my-org/domains`                     |

<div id="configure-auth0-dashboard">
  ## Configura Auth0 Dashboard
</div>

Antes de usar cualquier componente de My Organization, debes configurar tu tenant de Auth0 con las [APIs](/docs/es/api), aplicaciones y [scopes](/docs/es/api/myorganization) adecuadas.

  <Tab title="React">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">

    ## **Crear aplicación**

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Este ejemplo usa una aplicación de una sola página (SPA). Para frameworks de React con renderizado del lado del servidor, consulta la pestaña de Next.js.
    </Callout>

    1. Ve a [**Auth0 Dashboard &gt; Applications &gt; Applications**](https://manage.auth0.com/#/applications). Selecciona **Create Application**.

    2. Elige **Single Page Web Applications**.

    3. En la pestaña **Settings**, añade `http://localhost:5173` a los siguientes campos:
       <br />A. Allowed Callback URLs
       <br />B. Allowed Logout URLs

    4. En **Login Experience**, selecciona:
       <br />A. Business Users.
       <br />B. (Opcional) Prompt for Organization.

    ## **Habilitar la My Organization API**

    1. Ve a [**Dashboard &gt; Applications &gt; APIs**](https://manage.auth0.com/#/applications).
    2. Selecciona **[My Organization API](/docs/es/api/myorganization)** y asegúrate de que esté habilitada en tu tenant.

    ## **Configurar el acceso a la aplicación**

    1. Ve a la pestaña **Acceso a la aplicación**.
    2. Selecciona **Editar** en la aplicación que quieras usar.
    3. Configura las siguientes opciones:
       <br />A. [Perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile#connection-profile): Selecciona o crea un perfil con mapeo de atributos de conexión.
       <br />B. [Perfil de atributos de usuario](/docs/es/authenticate/enterprise-connections/user-attribute-profile#user-attribute-profile): Selecciona o crea un perfil con mapeo de atributos de usuario.
       <br />C. [Proveedores de identidad compatibles](/docs/es/authenticate/identity-providers): Habilita los proveedores que tus clientes puedan usar.
       <br />D. Comportamiento de eliminación de la conexión: Elige **Permitir** o **Permitir si está vacía**.
       <br />    • **Permitir**: Los usuarios pueden eliminar conexiones, lo que elimina a todos los usuarios de esa conexión.
       <br />    • **Permitir si está vacía**: Los usuarios solo pueden eliminar conexiones que no tengan usuarios.
       <br />E. Autorización de acceso de usuarios: Elige **No autorizado**, **Autorizado** o **Todos**.
       <br />    • **No autorizado**: No se concede ningún permiso.
       <br />    • **Autorizado**: Selecciona permisos específicos.
       <br />    • **Todos**: Incluye todos los permisos actuales y futuros.
       <br />F. Autorización de acceso para credenciales de cliente: Elige **No autorizado**, **Autorizado** o **Todos**.
    4. Selecciona **Guardar**.

    ## **Setup Database y usuario**

    1. Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) para crear una conexión de base de datos.
    2. Seleccione la pestaña **Applications** de la conexión y habilite su nueva aplicación SPA.
    3. Cree un usuario de prueba en esta conexión de base de datos para las pruebas iniciales.

    ## **Setup Role**

    1. Crea un rol (p. ej., &quot;Administrador de la organización&quot;).
    2. Agrega al rol los scopes necesarios de My Organization API.

    <Accordion title="Scopes requeridos">
      ```
          read:my_org:details
          update:my_org:details
          create:my_org:identity_providers
          read:my_org:identity_providers
          update:my_org:identity_providers
          delete:my_org:identity_providers
          update:my_org:identity_providers_detach
          create:my_org:identity_providers_domains
          delete:my_org:identity_providers_domains
          read:my_org:domains
          delete:my_org:domains
          create:my_org:domains
          update:my_org:domains
          read:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_scim_tokens
          delete:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_provisioning
          read:my_org:identity_providers_provisioning
          delete:my_org:identity_providers_provisioning
          read:my_org:configuration
      ```
    </Accordion>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      El token del usuario solo incluirá los permisos que existan tanto en el rol que tenga asignado como en la configuración de autorización de acceso del usuario establecida en el paso anterior.
    </Callout>

    ## **Crear organización**

    1. Ve a [**Organizations &gt; Crear organización**](https://manage.auth0.com/#/organizations).
    2. Configure Auth0 Organization:
       <br />A. En **Members**: añada su usuario de prueba y asígnele el rol de &quot;Administrador de la organización&quot;.
       <br />B. En **Connections**: habilite su conexión de base de datos.

    ## **Configurar Variables de Entorno**

    1. Crea un archivo `.env` en la raíz de tu proyecto de React:

    ```bash
    VITE_AUTH0_DOMAIN=your-domain.auth0.com
      VITE_AUTH0_CLIENT_ID=your-spa-client-id
    ```
