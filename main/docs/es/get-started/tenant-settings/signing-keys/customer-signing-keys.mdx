<Warning>
  La clave de firma proporcionada por el cliente está actualmente en acceso anticipado. Para obtener más información, consulta las etapas de lanzamiento del producto. Si quieres participar en Claves de firma proporcionadas por el cliente, debes tener un plan Enterprise. Ponte en contacto con tu gerente técnico de cuentas para obtener más información.
</Warning>

Auth0 usa claves criptográficas para firmar <Tooltip tip="Token de acceso: Credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> para garantizar que los tokens sean emitidos por Auth0. Auth0 firma los tokens con una clave de firma privada y publica la clave pública en el endpoint [JWKS](/docs/es/secure/tokens/json-web-tokens/json-web-key-sets)) `well-known` del tenant para <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, formato de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip>: `https://{yourDomain}/.well-known/``jwks.json`.

Si generas tus propias claves de firma, puedes importar la clave pública al endpoint JWKS `/.well-known` de tu tenant. Auth0 puede ser el único punto de distribución para las claves de firma de Auth0 y tus claves de firma personalizadas. Si tu caso de uso requiere que tu aplicación esté integrada en dispositivos remotos, puedes migrar tus claves de firma existentes para que Auth0 publique la clave pública en tu endpoint `/.well-known`.

Cuando se emiten tokens de acceso con tus claves de firma personalizadas como resultado de una autenticación de usuario y un <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) especificado en el framework OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip>:

* El sistema de gestión de claves de Auth0 es responsable de proteger las claves de firma que generas y proporcionas.
* Auth0 emite tokens para [autenticación y autorización](/docs/es/get-started/identity-fundamentals/authentication-and-authorization) como <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>.
* Auth0 publica una lista de claves públicas válidas en una URL de distribución específica del tenant.
* Una solución proxy recibe los tokens emitidos por Auth0 y los pone a disposición de aplicaciones y API heredadas.

  <Warning>
    Los tokens firmados con claves del cliente no pueden ser consumidos por las [API de Auth0](/docs/es/api).
  </Warning>
* Las aplicaciones del cliente usan los tokens de acceso emitidos para consumir API y recursos.
* Los servidores de API y recursos validan la firma de los tokens de acceso que se les presentan recuperando la clave pública de la distribución JWKS de Auth0 para verificar la firma del token.

<div id="how-it-works">
  ## Cómo funciona
</div>

<Frame>![](/docs/images/cdy7uua7fh8z/4nhWTT6iqS5CBWwXNuE5Ay/4542db5febc85dc5938958fa4b205a7e/2025-03-26_12-09-22.png)</Frame>

1. La parte pública de la clave de firma se importa a Auth0 y queda pendiente de publicarse en las claves de Auth0 como un [JWKS](/docs/es/secure/tokens/json-web-tokens/json-web-key-sets) agregado.
2. La clave pública de firma se publica en el endpoint well-known del [dominio personalizado](/docs/es/customize/custom-domains) de tu tenant: `{yourCustomDomain}/.well-known/jwks.json`.
3. Auth0 emite tokens de acceso. Esos tokens se personalizan y se vuelven a firmar con tu clave de firma.
4. Tu aplicación usa los tokens personalizados para acceder a tus recursos, como las API.
5. Auth0 distribuye tu clave publicándola en el endpoint `/.wellknown`.

<div id="prerequisites">
  ## Requisitos previos
</div>

Para importar claves de firma personalizadas en Auth0, debes:

* Configurar un <Tooltip tip="Custom Domain: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/docs/es/glossary?term=Custom+Domain">Custom Domain</Tooltip>. El JWKS agregado se publica desde un dominio personalizado y no está disponible en el dominio canónico. Para obtener más información, consulta [Custom Domains.](/docs/es/customize/custom-domains)
* Usar Private Cloud de Auth0. Solo puedes importar claves de firma en entornos de Private Cloud. Para obtener más información, consulta nuestro Private Cloud en [AWS](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-aws) y [Azure](/docs/es/deploy-monitor/deploy-private-cloud/private-cloud-on-azure).

<div id="configure-your-custom-keys-with-management-api">
  ## Configure sus claves personalizadas con Management API
</div>

Use la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para cargar sus claves personalizadas en formato JWKS.

Para crear nuevas claves personalizadas (y reemplazar todas las existentes), haga una llamada `PUT` al endpoint [Create or replace custom signing keys](https://auth0.com/docs/api/management/v2/keys/put-custom-signing-keys) para importar un conjunto de claves públicas personalizadas en Auth0. Las claves personalizadas existentes se reemplazarán por el nuevo conjunto. Puede importar hasta diez claves personalizadas en formato JWKS.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Una vez importadas, las claves públicas personalizadas se agregan al inicio de la lista de claves generadas por Auth0 y se publicarán en la URL del endpoint well-known de su tenant para distribuir JWKS.
</Callout>

Para obtener una lista de las claves personalizadas existentes, haga una llamada `GET` al endpoint [Get custom signing keys](https://auth0.com/docs/api/management/v2/keys/get-custom-signing-keys) para recuperar el conjunto de claves personalizadas como un arreglo de objetos JWK.

Para eliminar las claves personalizadas existentes, haga una llamada `DELETE` al endpoint [Delete custom keys](https://auth0.com/docs/api/management/v2/keys/delete-custom-signing-keys) para eliminar las claves personalizadas.
