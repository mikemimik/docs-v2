Puede rotar manualmente una clave de firma de forma periódica para cambiar la clave JSON Web Key (JWK) que utilizan las aplicaciones y las API para validar tokens. Si su aplicación o API **no** admite este cambio de clave e intenta usar una clave de firma vencida para verificar un token, la solicitud de autenticación fallará.

<Warning>
  Auth0 le recomienda realizar primero la rotación de claves de firma en un tenant de desarrollo y, después, verificar que sus aplicaciones y API sigan funcionando como se espera. Una vez que confirme que todo funciona correctamente, realice la misma rotación de claves de firma en su tenant de producción.
</Warning>

Aunque Auth0 firma con una sola clave de firma a la vez, el documento de descubrimiento de <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) de su tenant siempre contiene varias claves. El documento de descubrimiento de OIDC siempre incluirá tanto la clave actual como la siguiente, y también puede incluir la clave anterior si aún no se ha revocado. Para ofrecer una experiencia fluida en caso de emergencia, su aplicación debe poder usar cualquiera de las claves especificadas en el documento. Para obtener más información sobre los documentos de descubrimiento de OpenID Connect, lea [Localizar conjuntos de claves web JSON](/docs/es/secure/tokens/json-web-tokens/locate-json-web-key-sets).

<Warning>
  Para darle tiempo a actualizar su aplicación con la nueva clave de firma, todos los tokens firmados con la clave anterior seguirán siendo válidos hasta que revoque la clave anterior. Para obtener más información, lea [Revocar claves de firma](/docs/es/get-started/tenant-settings/signing-keys/revoke-signing-keys).
</Warning>

Puede rotar la clave de firma de la aplicación de su tenant mediante el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0.

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Dashboard &gt; Settings &gt; Signing Keys](https://manage.auth0.com/#/tenant/signing_keys).

   <Frame>![Pestaña Signing Keys de Configuración del tenant en el Dashboard](/docs/images/cdy7uua7fh8z/7r8t3EGctFmvkCgPrU0i2R/f79ac74dac5cded37b39bd8a8b80951d/dashboard-tenant-settings-signing-keys.png)</Frame>
2. En **Rotation Settings**, ubica **Rotate Signing Key** y selecciona **Rotate Key**.
3. Haz clic en **Rotate** para confirmar.

   <Frame>![Confirmación de rotación en la pestaña Signing Keys de Settings del Dashboard](/docs/images/cdy7uua7fh8z/6Ofp24mgt8ZrvaxF2wLA5N/5cca1a3267ffdc07f61d7990b4e85989/Rotating_Key_Warning_-_English.png)</Frame>

<div id="use-the-management-api">
  ## Usa la Management API
</div>

1. Para obtener una lista de las claves de firma, realiza una llamada `GET` al endpoint [Get all Application Signing Keys](https://auth0.com/docs/api/management/v2#!/Keys/get_signing_keys).
2. Para rotar la clave de firma, realiza una llamada `POST` al endpoint [Rotate the Application Signing Key](https://auth0.com/docs/api/management/v2#!/Keys/post_signing_keys). Asegúrate de sustituir el valor de marcador de posición `MGMT_API_ACCESS_TOKEN` por tu token de acceso de la Management API.

<div id="key-rotation-impact">
  ## Impacto de la rotación de claves
</div>

<div id="apis-and-api-gateways-accepting-access-tokens">
  ### APIs y gateways de API que aceptan tokens de acceso
</div>

La mayoría del middleware y de los gateways de API utilizan el endpoint de JSON Web Key Set (JWKS) para recuperar las claves de firma actuales y futuras a intervalos determinados. Si su middleware o sus gateways de API **no** admiten este endpoint y requieren que configure manualmente un archivo `*.cer`, deberá coordinar la rotación de la clave de firma en Auth0 con la reconfiguración de su middleware y sus gateways.

<div id="regular-web-applications">
  ### Aplicaciones web tradicionales
</div>

Al rotar la clave de firma en Auth0, tendrás que coordinar la reconfiguración de las aplicaciones que utilizan <Tooltip tip="Web Service Federation (WS-Fed): Protocolo para gestionar identidades de usuario entre dominios." cta="Ver glosario" href="/docs/es/glossary?term=WS-Fed">WS-Fed</Tooltip> o <Tooltip tip="Web Service Federation (WS-Fed): Protocolo para gestionar identidades de usuario entre dominios." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>. Esto suele suceder cuando cargas el nuevo certificado público o reconfiguras la aplicación al introducir la URL de metadatos de WS-Fed/SAML. Esto cambiará la clave JWKS, que las aplicaciones usan para validar tokens. Asegúrate de que tu implementación no dé por hecho que las claves JWKS no cambian.
