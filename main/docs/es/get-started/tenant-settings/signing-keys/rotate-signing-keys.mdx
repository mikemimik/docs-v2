Puedes rotar manualmente una clave de firma de forma periódica para cambiar la clave web JSON (JWK) que usan las aplicaciones y las API para validar tokens. Si tu aplicación o API **no** admite este cambio de clave y trata de usar una clave de firma vencida para verificar un token, la solicitud de autenticación fallará.

<Warning>
  Auth0 recomienda que primero ejecutes la rotación de la clave de firma en un tenant de desarrollo y luego verifiques que tus aplicaciones y API sigan funcionando como se espera. Después de comprobar que todo funciona correctamente, realiza la misma rotación de la clave de firma en tu tenant de producción.
</Warning>

Aunque Auth0 firma con una sola clave de firma a la vez, tu tenant&#39;s <Tooltip tip="OpenID: Open standard for authentication that allows applications to verify users' identities without collecting and storing login information." cta="View Glossary" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) el documento de descubrimiento siempre contiene varias claves. El documento de descubrimiento de OIDC siempre incluirá tanto la clave actual como la siguiente, y también puede incluir la clave anterior si aún no ha sido revocada. Para ofrecer una experiencia fluida en caso de emergencia, tu aplicación debe poder usar cualquiera de las claves especificadas en el documento. Para obtener más información sobre los documentos de descubrimiento de OpenID Connect, lee [Locate JSON Web Key Sets](/docs/es/secure/tokens/json-web-tokens/locate-json-web-key-sets).

<Warning>
  Para darte tiempo de actualizar tu aplicación con la nueva clave de firma, todos los tokens firmados con la clave anterior seguirán siendo válidos hasta que revoques la clave anterior. Para obtener más información, lee [Revocar claves de firma](/docs/es/get-started/tenant-settings/signing-keys/revoke-signing-keys).
</Warning>

Puedes rotar la clave de firma de la aplicación de tu tenant con el <Tooltip tip="Auth0 Dashboard: Auth0's main product to configure your services." cta="View Glossary" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: A product to allow customers to perform administrative tasks." cta="View Glossary" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0.

<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

1. Ve a [Dashboard &gt; Settings &gt; Signing Keys](https://manage.auth0.com/#/tenant/signing_keys).

   <Frame>![Pestaña Signing Keys de la Configuración del tenant en el Dashboard](/docs/images/cdy7uua7fh8z/7r8t3EGctFmvkCgPrU0i2R/f79ac74dac5cded37b39bd8a8b80951d/dashboard-tenant-settings-signing-keys.png)</Frame>
2. En **Rotation Settings**, busca **Rotate Signing Key** y selecciona **Rotate Key**.
3. Haz clic en **Rotate** para confirmar.

   <Frame>![Confirmación de rotación en la pestaña Signing Keys de Settings del Dashboard](/docs/images/cdy7uua7fh8z/6Ofp24mgt8ZrvaxF2wLA5N/5cca1a3267ffdc07f61d7990b4e85989/Rotating_Key_Warning_-_English.png)</Frame>

<div id="use-the-management-api">
  ## Usa la Management API
</div>

1. Para obtener una lista de las claves de firma, haz una llamada `GET` al endpoint [Get all Application Signing Keys](https://auth0.com/docs/api/management/v2#!/Keys/get_signing_keys).
2. Para rotar la clave de firma, haz una llamada `POST` al endpoint [Rotate the Application Signing Key](https://auth0.com/docs/api/management/v2#!/Keys/post_signing_keys). Asegúrate de reemplazar el valor de marcador de posición `MGMT_API_ACCESS_TOKEN` por tu token de acceso de la Management API.

<div id="key-rotation-impact">
  ## Impacto de la rotación de claves
</div>

<div id="apis-and-api-gateways-accepting-access-tokens">
  ### APIs y gateways de API que aceptan tokens de acceso
</div>

La mayoría del middleware y los gateways de API utilizan el endpoint de JSON Web Key Set (JWKS) para recuperar las claves de firma actuales y futuras a intervalos determinados. Si su middleware o sus gateways de API **no** admiten este endpoint y requieren que configure manualmente un archivo `*.cer`, deberá coordinar la rotación de las claves de firma en Auth0 con la reconfiguración de su middleware y sus gateways.

<div id="regular-web-applications">
  ### Aplicaciones web tradicionales
</div>

Al rotar la clave de firma en Auth0, deberá coordinar la reconfiguración de las aplicaciones que utilizan <Tooltip tip="Web Service Federation (WS-Fed): Protocolo para administrar identidades de usuario entre dominios." cta="Ver glosario" href="/docs/es/glossary?term=WS-Fed">WS-Fed</Tooltip> o <Tooltip tip="Web Service Federation (WS-Fed): Protocolo para administrar identidades de usuario entre dominios." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip>. Esto suele suceder cuando carga el nuevo certificado público o vuelve a configurar la aplicación introduciendo la URL de metadatos de WS-Fed/SAML. Esto cambiará la clave de JWKS que las aplicaciones usan para validar tokens; asegúrese de que su implementación no dé por hecho que las claves de JWKS no cambian.
