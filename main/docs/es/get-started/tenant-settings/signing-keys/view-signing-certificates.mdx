Puede ver los <Tooltip tip="Secreto de cliente: Secret usado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secrets">secretos de cliente</Tooltip> y las claves de firma de las aplicaciones de su tenant mediante el <Tooltip tip="Secreto de cliente: Secret usado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. La clave de firma de la aplicación se utiliza para firmar <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de ID</Tooltip>, <Tooltip tip="Token de ID: Credencial destinada al propio cliente, en lugar de al acceso a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip>, aserciones <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=SAML">SAML</Tooltip> y aserciones <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/docs/es/glossary?term=WS-Fed">WS-Fed</Tooltip> que se envían a su aplicación. Estas claves son distintas de las que se usan para firmar interacciones con las conexiones, incluida la firma de solicitudes SAML a <Tooltip tip="Web Service Federation (WS-Fed): Protocolo para gestionar identidades de usuarios entre dominios." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> (IdPs) y el cifrado de respuestas de los IdPs. De forma predeterminada, las aserciones SAML para las conexiones de IdP se firman, y lo recomendamos. Para obtener más información, lea [SAML Identity Provider Configuration Settings](/docs/es/authenticate/protocols/saml/saml-identity-provider-configuration-settings).

<div id="use-the-dashboard">
  ## Usar el Dashboard
</div>

<div id="tenant-settings">
  ### Configuración del tenant
</div>

1. Ve a [Dashboard &gt; Configuración &gt; Claves de firma](https://manage.auth0.com/#/tenant/signing_keys).

   <Frame>![Pestaña Claves de firma de Configuración del tenant en el Dashboard](/docs/images/cdy7uua7fh8z/7r8t3EGctFmvkCgPrU0i2R/f79ac74dac5cded37b39bd8a8b80951d/dashboard-tenant-settings-signing-keys.png)</Frame>
2. En la sección **Configuración de rotación**, ubica **Lista de claves válidas** y **Lista de claves revocadas**.

   1. La sección **Lista de claves válidas** muestra la clave de firma actual que usa tu tenant, además de la siguiente clave de firma que se asignará si decides rotar tus claves de firma. Si ya has rotado claves de firma anteriormente, esta sección también muestra las claves usadas previamente.
   2. La sección **Lista de claves revocadas** muestra las últimas tres claves revocadas de tu tenant.

<div id="application-settings">
  ### Configuración de la aplicación
</div>

También puedes ver la clave de firma o el secreto de cliente de una aplicación, según el tipo de <Tooltip tip="Algoritmo de firma: algoritmo utilizado para firmar digitalmente tokens y garantizar que el token no haya sido alterado." cta="Ver glosario" href="/docs/es/glossary?term=signing+algorithm">algoritmo de firma</Tooltip> que estés usando.

<div id="if-using-the-rs256-signing-algorithm">
  #### Si utilizas el algoritmo de firma RS256
</div>

1. Ve a [Dashboard &gt; Applications](https://manage.auth0.com/#/applications) y selecciona el nombre de la aplicación para verla.
2. Desplázate hasta la parte inferior de la pestaña **Settings** y selecciona **Advanced Settings**.
3. Ve a la pestaña **Certificados** y localiza el campo **Certificado de firma**.

   <Frame>![Pestaña Certificados de Advanced Settings en Dashboard Applications](/docs/images/cdy7uua7fh8z/5eB167clrfOiBEtyrMOtkn/fba0b689fda32cb9fd16eba1c0725c65/dashboard-applications-applications-settings-advanced-certificates.png)</Frame>

<div id="if-using-the-hs256-signing-algorithm">
  #### Si usas el algoritmo de firma HS256
</div>

1. Ve a [Dashboard &gt; Applications](https://manage.auth0.com/#/applications) y selecciona el nombre de la aplicación que quieres ver.
2. En **Información básica**, localiza el campo **secreto de cliente**.

   <Frame>![Dashboard Applications pestaña Configuración de la aplicación Información básica](/docs/images/cdy7uua7fh8z/2GPUw7BODYuYYH3658Upz3/92a49ec57e6b4d07be96093989baac03/2023-04-11_15-34-58.png)</Frame>

<div id="use-the-management-api">
  ## Usa la Management API
</div>

<div id="get-all-signing-keys">
  ### Obtener todas las claves de firma
</div>

Realiza una solicitud `GET` al endpoint [`/signing_keys/get_signing_keys`](https://auth0.com/docs/api/management/v2#!/signing_keys/get_signing_keys). Asegúrate de reemplazar el valor de marcador de posición `{yourMgmtApiAccessToken}` por tu token de acceso de la Management API.

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
     --url 'https://{yourDomain}/api/v2/keys/signing' \
     --header 'authorization: Bearer {yourMgmtApiAccessToken}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/keys/signing");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/keys/signing"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse response = Unirest.get("https://{yourDomain}/api/v2/keys/signing")
     .header("authorization", "Bearer {yourMgmtApiAccessToken}")
     .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'GET',
    url: 'https://{yourDomain}/api/v2/keys/signing',
    headers: {authorization: 'Bearer {yourMgmtApiAccessToken}'}
  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP
  $curl = curl_init();

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/api/v2/keys/signing",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_ENCODING => "",
    CURLOPT_MAXREDIRS => 10,
    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
    CURLOPT_CUSTOMREQUEST => "GET",
    CURLOPT_HTTPHEADER => [
      "authorization: Bearer {yourMgmtApiAccessToken}"
    ],
  ]);

  $response = curl_exec($curl);
  $err = curl_error($curl);

  curl_close($curl);

  if ($err) {
    echo "cURL Error #:" . $err;
  } else {
    echo $response;
  }
  ```

  ```python Python
  import http.client

  conn = http.client.HTTPSConnection("")

  headers = { 'authorization': "Bearer {yourMgmtApiAccessToken}" }

  conn.request("GET", "/{yourDomain}/api/v2/keys/signing", headers=headers)

  res = conn.getresponse()
  data = res.read()

  print(data.decode("utf-8"))
  ```

  ```ruby Ruby
  require 'uri'
  require 'net/http'
  require 'openssl'

  url = URI("https://{yourDomain}/api/v2/keys/signing")

  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Get.new(url)
  request["authorization"] = 'Bearer {yourMgmtApiAccessToken}'

  response = http.request(request)
  puts response.read_body
  ```
</AuthCodeGroup>

| Valor                   | Descripción                                                                                                                 |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN` | [token de acceso para la Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `read:signing_keys`. |

<div id="get-a-single-signing-key">
  ### Obtén una clave de firma específica
</div>

Haz una solicitud `GET` al endpoint [`/signing_keys/get_signing_key`](https://auth0.com/docs/api/management/v2#!/signing_keys/get_signing_key). Asegúrate de reemplazar los valores de marcador de posición `{yourKeyId}` y `{yourMgmtApiAccessToken}` por el ID de tu clave de firma y el token de acceso de la Management API, respectivamente.

<AuthCodeGroup>
  ```bash cURL

     --header 'authorization: Bearer {yourMgmtApiAccessToken}'
  ```
