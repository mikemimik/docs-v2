---
title: "Agent Skills pour Auth0"
description: "Utilisez les Agent Skills d’Auth0 pour intégrer l’authentification à vos applications avec des assistants de programmation IA comme Claude Code, Cursor et GitHub Copilot"
---

Les Agent Skills sont des instructions structurées qui aident les assistants de programmation IA à intégrer correctement l’authentification Auth0. Installez-les une seule fois, puis demandez à votre assistant d’ajouter la connexion, de protéger les routes, de sécuriser les API, de configurer MFA, de migrer depuis un autre fournisseur ou de personnaliser votre page de connexion — il s’occupe du reste.

Fonctionne avec Claude Code, Cursor, GitHub Copilot et [40+ autres agents](https://agentskills.io/clients) compatibles avec le format [Agent Skills](https://agentskills.io).

<div id="prerequisites">
  ## Prérequis
</div>

* Un [compte Auth0](https://auth0.com/signup) (forfait gratuit offert)
* Votre **domaine** Auth0 et votre **ID client** dans l’[Auth0 Dashboard](https://manage.auth0.com)
* Un assistant IA de codage (Claude Code, Cursor ou GitHub Copilot)

<div id="install">
  ## Installer
</div>

<Tabs>
  <Tab title="Claude Code">
    Auth0 est offert dans le marketplace officiel des plugins Claude Code (`claude-plugins-official`). Assurez-vous d’abord que le marketplace est à jour, puis installez-le :

    ```bash
    /plugin marketplace update claude-plugins-official
    /plugin install auth0@claude-plugins-official
    ```

    Vous pouvez aussi taper `/plugin` dans une session, aller à l’onglet **Discover**, puis rechercher « Auth0 ».

    Vous pouvez également l’installer à partir de votre terminal sans démarrer de session :

    ```bash
    claude plugin marketplace update claude-plugins-official
    claude plugin install auth0@claude-plugins-official
    ```

    <Note>
      Si vous obtenez une erreur « not found », ajoutez d’abord le marketplace, puis réessayez l’installation :

      ```bash
      /plugin marketplace add anthropics/claude-plugins-official
      /plugin install auth0@claude-plugins-official
      ```
    </Note>
  </Tab>

  <Tab title="Cursor">
    Auth0 est offert dans le [marketplace Cursor](https://cursor.com/marketplace/auth0). Ouvrez la fiche et cliquez sur **Add** pour l’installer.

    Vous pouvez aussi l’installer via **Cursor Settings &gt; Rules &gt; Add Rule &gt; Remote Rule (GitHub)**, puis entrer l’[URL du repository Auth0 Agent Skills](https://github.com/auth0/agent-skills).
  </Tab>

  <Tab title="GitHub Copilot">
    Installez-le à l’aide du [Skills CLI](https://github.com/vercel-labs/skills) pour GitHub Copilot :

    ```bash
    npx skills add auth0/agent-skills --agent github-copilot
    ```
  </Tab>

  <Tab title="Autres agents">
    Le [Skills CLI](https://github.com/vercel-labs/skills) fonctionne avec Claude Code, Cursor, Copilot, Codex et [plus de 40 autres agents](https://agentskills.io/clients) :

    ```bash
    npx skills add auth0/agent-skills --all
    ```

    Ciblez des agents précis avec `--agent` :

    ```bash
    npx skills add auth0/agent-skills --agent claude-code cursor
    ```
  </Tab>
</Tabs>

<div id="try-it-now">
  ## Essayez-le maintenant
</div>

Après l’installation, collez l’un des éléments suivants dans votre assistant de codage IA :

```
Add Auth0 login to my app
```

```
Secure my Express API with Auth0 JWT validation
```

```
Migrate from Firebase Auth to Auth0
```

<Tip>
  Donnez du contexte dans votre requête — mentionnez votre framework, le type de router, et si vous avez besoin de la connexion, de la protection de l’API ou des deux. Plus la requête est précise, meilleur sera le résultat.
</Tip>

<div id="how-it-works">
  ## Comment ça fonctionne
</div>

<Steps>
  <Step title="Décrivez ce dont vous avez besoin">
    Dites à votre assistant ce que vous voulez faire — ajouter la connexion à votre application React, sécuriser une API Spring Boot, migrer depuis Cognito, ajouter la MFA ou personnaliser votre page de connexion. Vous n’avez pas besoin de savoir quel skill ou SDK utiliser.
  </Step>

  <Step title="Le bon skill s’active">
    L’assistant lit les fichiers de votre projet (`package.json`, `requirements.txt`, `build.gradle`, etc.), identifie votre framework et active le skill correspondant parmi les 25 disponibles.
  </Step>

  <Step title="Vous obtenez du code fonctionnel">
    Le skill guide votre assistant à travers toute l’implémentation — installation du bon SDK, création des routes d’authentification, configuration des variables d’environnement, mise en place de la protection des routes et application des pratiques exemplaires d’Auth0.
  </Step>
</Steps>

<Info>
  Vous n’avez pas besoin de sélectionner les skills manuellement. La détection du framework gère automatiquement les skills liés aux SDK, et vous pouvez invoquer directement des skills de fonctionnalité comme MFA ou Migration en décrivant ce dont vous avez besoin.
</Info>

<div id="available-skills">

<div id="get-started">
  ### Pour commencer
</div>

| Skill                 | Ce qu’il fait                                                                                                                                                                                  | Essayez-le                                           |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| **Quickstart Router** | Détecte votre framework et configure Auth0 avec le bon SDK                                                                                                                                     | `Ajouter l’authentification Auth0 à mon application` |
| **Migration**         | Migre les utilisateurs et le code depuis Firebase, Cognito, Supabase, Clerk ou une solution d’authentification personnalisée                                                                   | `Aidez-moi à migrer de Firebase Auth vers Auth0`     |
| **CLI**               | Référence des commandes de l’Auth0 CLI — gérez les applications, les API, les utilisateurs, les rôles, les organisations, les Actions, les logs, les domaines personnalisés et Universal Login | `Créer une application Auth0 à l’aide de la CLI`     |
