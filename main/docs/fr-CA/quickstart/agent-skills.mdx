---
title: "Agent Skills pour Auth0"
description: "Utilisez les agent skills d’Auth0 pour intégrer l’authentification à vos applications avec des assistants IA de programmation comme Claude Code, Cursor et GitHub Copilot"
---

Les Agent Skills sont des instructions structurées qui aident les assistants IA de programmation à intégrer correctement l’authentification Auth0. Installez-les une seule fois, puis demandez à votre assistant d’ajouter la connexion, de protéger les routes, de sécuriser les API, de configurer la MFA, de migrer depuis un autre fournisseur ou de personnaliser votre page de connexion — il s’occupe du reste.

Fonctionne avec Claude Code, Cursor, GitHub Copilot et [40+ autres agents](https://agentskills.io/clients) qui prennent en charge le format [Agent Skills](https://agentskills.io).

<div id="prerequisites">
  ## Prérequis
</div>

* Un [compte Auth0](https://auth0.com/signup) (forfait gratuit offert)
* Votre **domaine** Auth0 et votre **client ID** dans le [Auth0 Dashboard](https://manage.auth0.com)
* Un assistant de codage IA (Claude Code, Cursor ou GitHub Copilot)

<div id="install">
  ## Installation
</div>

<Tabs>
  <Tab title="Claude Code">
    Auth0 est offert dans la Marketplace officielle des plugins Claude Code (`claude-plugins-official`). Assurez-vous d’abord que la Marketplace est à jour, puis installez-le :

    ```bash
    /plugin marketplace update claude-plugins-official
    /plugin install auth0@claude-plugins-official
    ```

    Vous pouvez aussi taper `/plugin` dans une session, aller à l’onglet **Discover**, puis rechercher « Auth0 ».

    Vous pouvez également l’installer depuis votre terminal sans démarrer de session :

    ```bash
    claude plugin marketplace update claude-plugins-official
    claude plugin install auth0@claude-plugins-official
    ```

    <Note>
      Si vous obtenez une erreur « not found », ajoutez d’abord la Marketplace, puis réessayez l’installation :

      ```bash
      /plugin marketplace add anthropics/claude-plugins-official
      /plugin install auth0@claude-plugins-official
      ```
    </Note>
  </Tab>

  <Tab title="Cursor">
    Auth0 se trouve dans la [Marketplace Cursor](https://cursor.com/marketplace/auth0). Ouvrez la fiche, puis cliquez sur **Add** pour l’installer.

    Vous pouvez aussi l’installer via **Cursor Settings &gt; Rules &gt; Add Rule &gt; Remote Rule (GitHub)** et entrer l’[URL du repository Auth0 Agent Skills](https://github.com/auth0/agent-skills).
  </Tab>

  <Tab title="GitHub Copilot">
    Installez-le à l’aide de la [CLI Skills](https://github.com/vercel-labs/skills) pour GitHub Copilot :

    ```bash
    npx skills add auth0/agent-skills --agent github-copilot
    ```
  </Tab>

  <Tab title="Autres agents">
    La [CLI Skills](https://github.com/vercel-labs/skills) fonctionne avec Claude Code, Cursor, Copilot, Codex et [plus de 40 autres agents](https://agentskills.io/clients) :

    ```bash
    npx skills add auth0/agent-skills --all
    ```

    Ciblez des agents précis avec `--agent` :

    ```bash
    npx skills add auth0/agent-skills --agent claude-code cursor
    ```
  </Tab>
</Tabs>

<div id="try-it-now">
  ## Essayez-le dès maintenant
</div>

Après l’installation, collez l’un des éléments suivants dans votre assistant IA de codage :

```
Add Auth0 login to my app
```

```
Secure my Express API with Auth0 JWT validation
```

```
Migrate from Firebase Auth to Auth0
```

<Tip>
  Fournissez du contexte dans votre requête — mentionnez votre framework, le type de router et si vous avez besoin du login, de la protection de l’API ou des deux. Plus votre requête est précise, meilleur sera le résultat.
</Tip>

<div id="how-it-works">
  ## Fonctionnement
</div>

<Steps>
  <Step title="Décrivez ce dont vous avez besoin">
    Dites à votre assistant ce que vous voulez faire — ajouter la connexion à votre application React, sécuriser une API Spring Boot, migrer depuis Cognito, ajouter la MFA ou personnaliser votre page de connexion. Vous n’avez pas besoin de savoir quel skill ou SDK utiliser.
  </Step>

  <Step title="Le skill approprié s’active">
    L’assistant lit les fichiers de votre projet (`package.json`, `requirements.txt`, `build.gradle`, etc.), identifie votre framework et active le skill correspondant parmi les 25 offerts.
  </Step>

  <Step title="Vous obtenez un code fonctionnel">
    Le skill guide votre assistant dans l’ensemble de la mise en œuvre — installation du bon SDK, création des routes d’authentification, configuration des variables d’environnement, mise en place de la protection des routes et application des bonnes pratiques d’Auth0.
  </Step>
</Steps>

<Info>
  Vous n’avez pas besoin de choisir les skills manuellement. La détection du framework gère automatiquement les skills liés au SDK, et vous pouvez lancer directement des skills axés sur des fonctionnalités comme la MFA ou la migration simplement en décrivant ce dont vous avez besoin.
</Info>
