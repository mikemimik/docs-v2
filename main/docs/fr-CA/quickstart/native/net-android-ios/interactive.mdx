  <Content>
    Auth0 vous permet d&#39;ajouter l&#39;authentification à presque n&#39;importe quel type d&#39;application rapidement. Ce guide explique comment
    intégrer Auth0, ajouter l&#39;authentification et afficher les renseignements du profil utilisateur dans n&#39;importe quelle application .NET Android et iOS
    à l&#39;aide des SDK Auth0 pour [Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/) et [iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS).

    <Info>
      Ce guide de démarrage rapide porte sur .NET Android et iOS, puisqu’ils représentent la nouvelle génération de
      `Xamarin.Android` et `Xamarin.iOS`. Si vous utilisez encore
      `Xamarin.Android` et `Xamarin.iOS`, vous pouvez suivre ce guide, car l’intégration est
      identique et les SDKs sont compatibles.
    </Info>

    Pour utiliser ce guide de démarrage rapide, vous aurez besoin de :

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Pour utiliser les services Auth0, vous devez avoir une application configurée dans l’Auth0 Dashboard. L’application Auth0 est
      l’endroit où vous configurerez le fonctionnement de l’authentification pour votre projet.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle &quot;Application native&quot;, ou sélectionnez une application existante qui
      représente le projet que vous souhaitez intégrer. Chaque application dans Auth0 se voit attribuer un client ID alphanumérique unique
      que le code de votre application utilisera pour effectuer des requêtes aux API Auth0 par l’entremise du SDK.

      Tous les paramètres que vous configurez à l’aide de ce Quickstart seront automatiquement mis à jour pour votre application dans le [Dashboard](https://manage.auth0.com/#/), où vous
      pourrez gérer vos applications plus tard.

      Si vous préférez consulter une configuration complète, vous pouvez plutôt voir un exemple d’application.

      ### Configurer les URL de rappel

      Une URL de rappel est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      authentification. Si elle n’est pas définie, les utilisateurs ne seront pas redirigés vers votre application après s’être connectés.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la sur l’une des URL suivantes, selon
        votre plateforme :

        * **Android** : `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS** : `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android** : `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS** : `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      ### Configurer les URL de déconnexion

      Une URL de déconnexion est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      déconnexion. Si elle n’est pas définie, les utilisateurs ne pourront pas se déconnecter de votre application et recevront une erreur.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la sur l’une des URL suivantes, selon
        votre plateforme :

        * **Android** : `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS** : `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android** : `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS** : `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      Enfin, assurez-vous que le **Type d’application** de votre application est défini sur **Native** dans les [Settings de l’application](https://manage.auth0.com/#/applications/).
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 offre des SDK [Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/) et [iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS) pour simplifier l’implémentation de l’authentification Auth0
      dans les applications Android et iOS en .NET.

      Utilisez le Gestionnaire de package NuGet (Outils -&gt; Gestionnaire de package de bibliothèque -&gt; Console du Gestionnaire de package) pour installer le
      package `Auth0.OidcClient.AndroidX` ou `Auth0.OidcClient.iOS`, selon que vous
      développez une application Android ou iOS.

      Vous pouvez aussi utiliser la console du Gestionnaire de package NuGet (`Install-Package`) ou la
      CLI `dotnet` (`dotnet add`).

      ```
      Install-Package Auth0.OidcClient.AndroidX
      Install-Package Auth0.OidcClient.iOS
      ```

      ```
      dotnet add Auth0.OidcClient.AndroidX
      dotnet add Auth0.OidcClient.iOS
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Pour intégrer Auth0 à votre application, créez une instance de la classe `Auth0Client`, en lui passant
      une instance de `Auth0ClientOptions` qui contient votre domaine Auth0 et votre Client ID.

      ```csharp
      using Auth0.OidcClient;
      var client = new Auth0Client(new Auth0ClientOptions {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}"
      }, this);
      ```

      Par défaut, le SDK utilisera Chrome Custom Tabs sur Android et ASWebAuthenticationSession sur iOS.

      <Note>
        ##### Point de contrôle

        Votre `Auth0Client` devrait maintenant être correctement instancié. Exécutez votre application pour vérifier que :

        * L’`Auth0Client` est correctement instancié dans l’`Activity` (Android) ou le
          `UIViewController` (iOS).
        * Votre application ne génère aucune erreur liée à Auth0.
      </Note>
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Une fois l’utilisateur authentifié avec succès, il sera redirigé vers l’URL de rappel que vous avez configurée plus tôt dans ce
      Quickstart.

      Pour gérer le rappel sur les appareils Android, vous devez enregistrer un intent qui prend en charge cette URL de rappel. Une façon simple
      de le faire consiste à enregistrer l’intent sur la même activité à partir de laquelle vous avez appelé la méthode LoginAsync pour
      instancier le flux d’authentification.

      Assurez-vous de remplacer `YOUR_ANDROID_PACKAGE_NAME` dans l’exemple de code par le véritable Package Name de votre
      application, par exemple `com.mycompany.myapplication`, et assurez-vous que tout le texte de
      `DataScheme`, `DataHost` et `DataPathPrefix` est en minuscules. Définissez également
      `LaunchMode = LaunchMode.SingleTask` pour l’Activity ; sinon, le système créera une nouvelle instance
      de l’activité chaque fois que l’URL de rappel sera appelée.

      De plus, vous devez gérer l’intent dans l’événement `OnNewIntent` de votre classe `Activity`.
      Vous devez indiquer au Auth0 OIDC Client de terminer le flux d’authentification en appelant la méthode `Send`
      du singleton `ActivityMediator`, en lui transmettant l’URL reçue.

      ```charp
      protected override void OnNewIntent(Intent intent)
      {

            base.OnNewIntent(intent);

            ActivityMediator.Instance.Send(intent.DataString);

      }
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Après s’être authentifié avec succès, l’utilisateur sera redirigé vers l’URL de rappel que vous avez configurée plus tôt dans ce
      Quickstart.

      Pour traiter l’URL de rappel sur les appareils iOS :

      1. Ouvrez le fichier `Info.plist` de votre application dans Visual Studio, puis accédez à l’onglet &#42;&#42;Advanced &#42;&#42;.
      2. Sous **Types d’URL**, cliquez sur le bouton &#42;&#42;Ajouter un type d’URL &#42;&#42;.
      3. Définissez l’&#42;&#42;identifiant &#42;&#42; sur Auth0, les &#42;&#42;schémas d’URL &#42;&#42; comme le `Bundle Identifier` de votre application,
         et le &#42;&#42;rôle &#42;&#42; sur `None`.

      Voici un exemple de la représentation XML de votre fichier `info.plist` après avoir ajouté le
      type d’URL :

      ```xml
      <key>CFBundleURLTypes</key>
      <array>
      <dict>
      <key>CFBundleTypeRole</key>
      <string>None</string>
      <key>CFBundleURLName</key>
      <string>Auth0</string>
      <key>CFBundleURLSchemes</key>
      <array>
      <string>YOUR_BUNDLE_IDENTIFIER</string>
      </array>
      </dict>
      </array>
      ```

      De plus, vous devez gérer l’URL de rappel dans l’événement `OpenUrl` de votre
      classe `AppDelegate`. Vous devez indiquer à Auth0 OIDC Client de terminer le flux d’authentification en
      appelant la méthode `Send` du singleton `ActivityMediator`, en lui transmettant l’URL
      reçue.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Maintenant que vous avez configuré votre application Auth0 et le SDK Auth0, vous devez mettre en place la connexion pour votre projet.
      Pour ce faire, vous utiliserez la méthode `LoginAsync()` du SDK afin de créer un bouton de connexion qui redirige les utilisateurs
      vers la page Universal Login d’Auth0.

      ```csharp
      var loginResult = await client.LoginAsync();
      ```

      S’il n’y a pas d’erreur, vous pouvez accéder à `User`, `IdentityToken`,
      `AccessToken` et `RefreshToken` dans le `LoginResult` renvoyé par
      `LoginAsync()`.

      <Note>
        ##### Point de contrôle

        Vous devriez maintenant être en mesure de vous connecter ou de vous inscrire à l’aide d’un nom d’utilisateur et d’un mot de passe.

        Cliquez sur le bouton de connexion et vérifiez que :

        * Votre application Android ou iOS vous redirige vers la page Universal Login d’Auth0.
        * Vous pouvez vous connecter ou vous inscrire.
        * Auth0 vous redirige vers votre application.
      </Note>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Les utilisateurs qui se connectent à votre projet auront aussi besoin d’un moyen de se déconnecter. Créez un bouton de déconnexion à l’aide de la méthode `LogoutAsync()` du SDK.
      Lorsque les utilisateurs se déconnectent, ils sont redirigés vers votre endpoint de logout Auth0,
      qui les redirige ensuite immédiatement vers l’URL de logout que vous avez configurée plus tôt dans ce Quickstart.

      ```csharp
      await client.LogoutAsync();
      ```

      <Note>
        ##### Point de contrôle

        Lancez votre application et cliquez sur le bouton de déconnexion. Vérifiez que :

        * Votre application Android ou iOS vous redirige vers l’adresse que vous avez indiquée parmi les URL de déconnexion autorisées dans les Paramètres de l’application.
        * Vous n’êtes plus connecté à l’application.
      </Note>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Maintenant que vos utilisateurs peuvent se connecter et se déconnecter, vous voudrez probablement pouvoir récupérer les informations de profil associées aux utilisateurs authentifiés.
      Par exemple, vous pourriez vouloir afficher le nom ou la photo de profil d’un utilisateur connecté dans votre projet.

      Le SDK Auth0 pour Android et iOS fournit des informations sur l’utilisateur par l’intermédiaire de la propriété `LoginResult.User`.

      ```csharp
      if loginResult.IsError == false {
            var user = loginResult.User
            var name = user.FindFirst(c => c.Type == "name")?.Value
            var email = user.FindFirst(c => c.Type == "email")?.Value
            var picture = user.FindFirst(c => c.Type == "picture")?.Value
      }
      ```
    </Section>
