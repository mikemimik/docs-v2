<Note>
  **Prérequis :**

  * Flutter SDK 3.24.0+ et Dart 3.5.0+.
  * Windows 10 ou version ultérieure.
  * Visual Studio 2022 avec la charge de travail **Desktop development with C++**.
  * Compte Auth0 — [inscrivez-vous gratuitement](https://auth0.com/signup).

  Cette fonctionnalité est en **bêta** (`auth0_flutter` 2.1.0-beta.1). L’API peut changer avant la disponibilité générale.
</Note>

Ce guide vous montre comment ajouter la connexion, la déconnexion et l’affichage du profil utilisateur à une application de bureau Flutter pour Windows à l’aide du SDK `auth0_flutter` avec OAuth 2.0 [Flux du code d’autorisation + PKCE](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

<div id="get-started">
  ## Premiers pas
</div>

<Steps>
  <Step title="Créer un projet Flutter pour Windows" stepNumber={1}>
    Créez un nouveau projet Flutter avec la prise en charge de Windows.

    ```shellscript
    flutter create --platforms=windows my_app
    cd my_app
    ```

    Vérifiez que Windows est disponible :

    ```shellscript
    flutter devices
    ```

    Vous devriez voir un appareil Windows pour ordinateur de bureau dans la liste.

    <Tip>
      Exécutez `flutter doctor` pour vous assurer que votre environnement est correctement configuré et que Visual Studio 2022 avec les outils de développement d’applications de bureau en C++ est détecté.
    </Tip>
  </Step>

  <Step title="Installer le SDK Auth0 Flutter" stepNumber={2}>
    Ajoutez la version bêta du SDK qui prend en charge Windows :

    ```shellscript
    flutter pub add auth0_flutter:2.1.0-beta.1
    flutter pub add flutter_dotenv
    ```

    Votre `pubspec.yaml` doit inclure :

    <Snippet file="quickstart/native/flutter-windows/pubspec.yaml.mdx" />

    <Tip>
      L’Auth0 Flutter SDK nécessite **Flutter 3.24.0+** et **Dart 3.5.0+**. La plateforme Windows nécessite également **Visual Studio 2022** avec la charge de travail de développement d’applications de bureau en C++.
    </Tip>
  </Step>

  <Step title="Configurer Auth0" stepNumber={3}>
    Créez ou configurez une application Auth0 avec les URL de rappel requises pour l’authentification sur Windows Desktop.

    <Tabs>
      <Tab title="Configuration rapide">
        Créez une application **Native** dans votre Auth0 Dashboard avec les paramètres suivants :

        | Champ                 | Valeur                    |
        | --------------------- | ------------------------- |
        | Allowed Callback URLs | `auth0flutter://callback` |
        | Allowed Logout URLs   | `auth0flutter://callback` |

        Vos informations d’identification :

        * **Domain:** `{yourDomain}`
        * **Client ID:** `{yourClientId}`
      </Tab>

      <Tab title="CLI">
        ```shellscript
        auth0 apps create \
          --name "My Flutter Windows App" \
          --type native \
          --callbacks "auth0flutter://callback" \
          --logout-urls "auth0flutter://callback"
        ```
      </Tab>

      <Tab title="Dashboard">
        1. Accédez à [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; Applications**.
        2. Sélectionnez **Create Application**.
        3. Donnez-lui le nom &quot;My Flutter Windows App&quot; et choisissez **Native** comme type d’application.
        4. Ouvrez l’onglet **Settings** de votre nouvelle application.
        5. Définissez **Allowed Callback URLs** sur `auth0flutter://callback`.
        6. Définissez **Allowed Logout URLs** sur `auth0flutter://callback`.
        7. Faites défiler la page vers le bas et cliquez sur **Save Changes**.
        8. Copiez les valeurs **Domain** et **Client ID** en haut de la page Application Settings.
      </Tab>
    </Tabs>

    <Info>
      L’URL `auth0flutter://callback` est un schéma personnalisé qui redirige l’URL de rappel du navigateur vers votre application de bureau une fois l’authentification terminée.
    </Info>
  </Step>

    Le flux d’authentification Windows exige la configuration du callback dans le runner de votre application. Le plugin Flutter ne reçoit pas automatiquement du système d’exploitation les activations du schéma de protocole — vous devez donc ajouter la gestion d’instance unique et le transfert d’URI.

    Remplacez le contenu de `windows/runner/main.cpp` :

    <Snippet file="quickstart/native/flutter-windows/main.cpp.mdx" />

    Ce code :

    * Assure qu’une seule instance s’exécute à l’aide d’un mutex Windows.
    * Capture les URI `auth0flutter://callback` passés en argument sur la ligne de commande.
    * Transfère les URI des lancements secondaires à l’instance déjà en cours d’exécution au moyen d’un tube nommé.
    * Définit la variable d’environnement `PLUGIN_STARTUP_URL` pour que le plugin Auth0 puisse la lire.
  </Step>

  <Step title="Enregistrez le schéma d’URL personnalisé" stepNumber={6}>
    Enregistrez `auth0flutter` comme schéma d’URL personnalisé pour que Windows redirige les URI de rappel vers votre application.

    Créez un fichier `windows/url_scheme.reg` :

    <Snippet file="quickstart/native/flutter-windows/url_scheme.reg.mdx" />

    Remplacez `C:\path\to\your\app.exe` par le chemin réel vers votre fichier exécutable. En cours de développement, il s’agit généralement de :

    ```text
    build\windows\x64\runner\Debug\my_app.exe
    ```

    Double-cliquez sur le fichier `.reg` pour l’importer dans le registre de Windows.

    <Info>
      En production, enregistrez le schéma d’URL personnalisé par programmation dans le programme d’installation de votre application (MSIX, Inno Setup, etc.) au lieu de vous fier à un fichier `.reg`.
    </Info>

    **Vérifiez que le schéma fonctionne :**

    Ouvrez l’invite de commandes et exécutez :

    ```shellscript
    start auth0flutter://test
    ```

    Votre application devrait s’ouvrir (ou passer au premier plan si elle est déjà en cours d’exécution).
  </Step>

  <Step title="Mettre en place la connexion et la déconnexion" stepNumber={7}>
    Créez `lib/auth_service.dart` pour gérer l’authentification Windows :

    <Snippet file="quickstart/native/flutter-windows/auth_service.dart.mdx" />

    <Info>
      Le SDK Flutter Windows d’Auth0 ne prend actuellement pas en charge la gestion des identifiants. Vous devez enregistrer manuellement les identifiants si vous voulez conserver les sessions après le redémarrage de l’application.
    </Info>
  </Step>

  <Step title="Afficher les renseignements du profil utilisateur" stepNumber={8}>
    Créez l’interface principale de l’application dans `lib/main.dart` :

    <Snippet file="quickstart/native/flutter-windows/main.dart.mdx" />
  </Step>

  <Step title="Lancez votre application" stepNumber={9}>
    Compilez et exécutez l’application :

    ```shellscript
    flutter run -d windows
    ```

    **Flux attendu :**

    1. L’application s’ouvre avec un bouton **Se connecter**.
    2. Sélectionnez **Se connecter**. Le navigateur du système ouvre la page Auth0 Universal Login.
    3. Terminez l’authentification dans le navigateur.
    4. Le navigateur redirige vers `auth0flutter://callback` → votre application revient au premier plan.
    5. Le nom de l’utilisateur, son courriel et sa photo de profil s’affichent.

    <Warning>
      Assurez-vous que le schéma d’URL personnalisé est enregistré (étape 6) avant de tester. Sans cela, le callback du navigateur ne peut pas atteindre votre application.
    </Warning>
  </Step>
</Steps>

  **Vérification**

  Vous devriez maintenant disposer d’une expérience de connexion Auth0 entièrement fonctionnelle dans votre application Flutter pour Windows. L’application ouvre le navigateur du système pour Auth0 Universal Login, reçoit le rappel au moyen du schéma d’URL personnalisé et affiche le profil de l’utilisateur authentifié.
</Check>

<div id="troubleshoot-advanced-use">
  ## Dépannage et utilisation avancée
</div>

<Accordion title="Problèmes courants et solutions">
  ### Le navigateur s’ouvre, mais l’application ne reçoit pas le rappel

  **Symptôme** : la connexion Auth0 réussit dans le navigateur, mais l’application ne reçoit jamais les identifiants.

  **Correctif :**

  1. Ouvrez l’Éditeur du Registre → `HKEY_CURRENT_USER\Software\Classes\auth0flutter\shell\open\command` et confirmez le chemin vers votre `.exe`.
  2. Testez avec `start auth0flutter://test` dans l’Invite de commandes — votre application devrait se lancer.
  3. Assurez-vous que `windows/runner/main.cpp` inclut le code du serveur de pipe et du mutex.
  4. Vérifiez qu’aucune ancienne instance n’est en cours d’exécution dans le Gestionnaire des tâches.
  5. Recompilez complètement : `flutter clean && flutter run -d windows`.

  ### L’authentification expire après 5 minutes

  **Symptôme** : La connexion semble bloquée et finit par échouer.

  **Correctif :** L’application n’a jamais reçu l’URI de rappel. Vérifiez :

  1. Que l’entrée du registre pointe vers le bon chemin de l’exécutable.
  2. Que le nom du mutex `auth0flutter_single_instance_mutex` est cohérent dans `main.cpp`.
  3. Qu’aucun pare-feu ou antivirus ne bloque le named pipe.
  4. Fermez toutes les anciennes instances et recompilez.

  ### Une deuxième instance de l’application se lance au lieu de transférer l’URI

  **Symptôme** : Une nouvelle fenêtre s’ouvre au lieu que l’application existante reçoive le rappel.

  **Correctif :**

  1. Fermez toutes les instances en cours d’exécution dans le Gestionnaire des tâches.
  2. Assurez-vous que le nom du mutex est cohérent dans `main.cpp`.
  3. Recompilez avec la commande suivante : `flutter clean && flutter run -d windows`.

  ### WindowsWebAuthentication introuvable

  **Symptôme** : Erreur de compilation faisant référence à `windowsWebAuthentication`.

  **Correctif :** Assurez-vous que votre `pubspec.yaml` précise la version bêta :

  ```yaml
  dependencies:
    auth0_flutter: 2.1.0-beta.1
  ```

  Exécutez `flutter pub get` pour mettre à jour.

  ### Erreur de non-correspondance de l’URL de rappel

  **Symptôme** : Erreur &quot;redirect&#95;uri&#95;mismatch&quot; provenant d’Auth0.

  **Correctif :**

  1. Vérifiez que **Allowed Callback URLs** dans [Auth0 Dashboard](https://manage.auth0.com/) → Application Settings est exactement `auth0flutter://callback`.
  2. Assurez-vous que le paramètre `appCustomURL` dans votre code correspond à : `'auth0flutter://callback'`.
  3. Vérifiez qu’il n’y a ni barre oblique finale ni espace.
</Accordion>

<Accordion title="Utiliser un serveur intermédiaire">
  Quand Auth0 redirige directement vers un schéma personnalisé, le navigateur peut afficher une invite ou laisser un onglet vide. Pour une expérience plus fluide, utilisez un serveur HTTPS intermédiaire :

  1. Configurez un endpoint serveur (par ex., `https://your-app.example.com/callback`) qui redirige vers `auth0flutter://callback?code=...&state=...`.
  2. Dans [Auth0 Dashboard](https://manage.auth0.com/) → Application Settings, définissez **Allowed Callback URLs** à `https://your-app.example.com/callback`.
  3. Passez les deux URL à la méthode de connexion :

  ```dart
  final result = await auth0.windowsWebAuthentication().login(
    appCustomURL: 'auth0flutter://callback',
    redirectUrl: 'https://your-app.example.com/callback',
  );
  ```

  La page du serveur peut afficher &quot;Redirecting...&quot; et se fermer d’elle-même pour une expérience plus fluide.
</Accordion>

<Accordion title="Scopes personnalisés et audience">
  Demandez des scopes supplémentaires ou une audience d’API :

  ```dart
  final result = await auth0.windowsWebAuthentication().login(
    appCustomURL: 'auth0flutter://callback',
    scopes: {'openid', 'profile', 'email', 'read:messages'},
    audience: 'https://your-api.example.com',
  );
  ```

  Configurez l’API dans [Auth0 Dashboard](https://manage.auth0.com/) → **Applications &gt; APIs** avant d’utiliser le paramètre `audience`.
</Accordion>
