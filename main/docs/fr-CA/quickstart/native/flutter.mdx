
  ```text

  ```

  Votre assistant IA créera automatiquement votre application Auth0, récupérera les identifiants, ajoutera la dépendance du SDK auth0&#95;flutter, configurera les URL de rappel Android et iOS, et implémentera la connexion/déconnexion avec Web Auth ainsi que le stockage sécurisé des identifiants. [Documentation complète d’Agent Skills →](/quickstart/agent-skills)

Ce guide montre comment intégrer Auth0 à une application Flutter à l’aide du [Auth0 Flutter SDK](https://github.com/auth0/auth0-flutter). Il couvre la configuration des plateformes **Android**, **iOS** et **macOS**.

<Info>
  Le Auth0 Flutter SDK prend aussi en charge **Web** et **Windows** (bêta). Ces plateformes ont des Quickstarts dédiés :

  * [Flutter Web Quickstart](/docs/fr-CA/quickstart/spa/flutter) — pour les applications Flutter destinées au navigateur
  * [Flutter Windows Quickstart](/docs/fr-CA/quickstart/native/flutter-windows) — pour les applications de bureau Flutter sur Windows (bêta)
</Info>

<div id="get-started">
  ## Premiers pas
</div>

<Steps>
  <Step title="Créer un nouveau projet Flutter" stepNumber={1}>
    Créez un nouveau projet Flutter pour ce Quickstart.

    **Dans votre terminal :**

    1. Accédez à votre répertoire de travail
    2. Exécutez : `flutter create auth0_flutter_sample`
    3. Placez-vous dans le répertoire du projet : `cd auth0_flutter_sample`
    4. Ouvrez-le dans votre IDE :
       * **VS Code** : `code .`
       * **Android Studio** : `open -a "Android Studio" .`

    ```shellscript
    # Créer un nouveau projet Flutter
    flutter create auth0_flutter_sample

    # Accéder au répertoire du projet
    cd auth0_flutter_sample

    # Ouvrir dans VS Code
    code .
    ```

    <Tip>
      Cette commande crée une application Flutter moderne avec la structure de projet la plus récente. Exécutez `flutter doctor` pour vérifier que votre environnement est configuré correctement.
    </Tip>
  </Step>

  <Step title="Installez le SDK Auth0 Flutter" stepNumber={2}>
    Ajoutez le SDK Auth0 Flutter à votre projet à l’aide de la CLI Flutter.

    ```shellscript
    flutter pub add auth0_flutter
    ```

    Cela ajoutera `auth0_flutter` aux dépendances de votre `pubspec.yaml` :

    ```yaml pubspec.yaml lines
    dependencies:
      auth0_flutter: ^2.0.0-beta.1
    ```

    <Tip>
      Le SDK Auth0 Flutter nécessite **Flutter 3.24.0+** et **Dart 3.5.0+**. Exécutez `flutter doctor` pour vérifier que votre environnement répond à ces exigences.
    </Tip>
  </Step>

  <Step title="Configurez votre Auth0 App" stepNumber={3}>
    La prochaine étape consiste à créer une nouvelle application dans votre tenant Auth0 et à configurer les URL de rappel.

    1. Accédez au [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. Cliquez sur **Applications** &gt; **Applications** &gt; **Create Application**
    3. Dans la fenêtre contextuelle, entrez un nom pour votre application, sélectionnez `Native` comme type d’application, puis cliquez sur **Create**
    4. Passez à l’onglet **Settings** de la page des détails de l’application
    5. Prenez en note les valeurs **Domain** et **Client ID** - vous en aurez besoin plus tard

    Dans l’onglet **Settings**, configurez les URL suivantes selon la ou les plateformes cibles :

    **Allowed Callback URLs :**

    <Tabs>
      <Tab title="Android">
        ```text
        https://{yourDomain}/android/{yourPackageName}/callback
        ```
      </Tab>

      <Tab title="iOS">
        ```text
        https://{yourDomain}/ios/{yourBundleIdentifier}/callback,
        {yourBundleIdentifier}://{yourDomain}/ios/{yourBundleIdentifier}/callback
        ```
      </Tab>

      <Tab title="macOS">
        ```text
        https://{yourDomain}/macos/{yourBundleIdentifier}/callback,
        {yourBundleIdentifier}://{yourDomain}/macos/{yourBundleIdentifier}/callback
        ```
      </Tab>
    </Tabs>

    **Allowed Logout URLs :**

    Ajoutez dans le champ **Allowed Logout URLs** les mêmes URL que celles indiquées dans la configuration des URL de rappel ci-dessus.

    <Info>
      Les **Allowed Callback URLs** constituent une mesure de sécurité essentielle pour garantir que les utilisateurs sont redirigés de façon sécuritaire vers votre application après l’authentification. Sans URL correspondante, le processus de connexion échouera.

      Les **Allowed Logout URLs** sont essentielles pour offrir une expérience utilisateur fluide lors de la déconnexion. Sans URL correspondante, les utilisateurs ne seront pas redirigés vers votre application après la déconnexion.

      Par exemple, si votre domaine Auth0 est `example.us.auth0.com` et que le nom du package Android est `com.example.myapp`, votre URL de rappel Android serait : `https://example.us.auth0.com/android/com.example.myapp/callback`
    </Info>

    <Warning>
      **Important** : Assurez-vous que le nom du package (Android) ou l’identifiant du bundle (iOS/macOS) dans vos URL de rappel correspond bien à l’identifiant réel de votre application. Si l’authentification échoue, vérifiez que ces valeurs sont identiques.
    </Warning>
  </Step>

  <Step title="Configurer votre application" stepNumber={4}>
    Une configuration propre à la plateforme est requise pour activer le flux d’authentification. Suivez les instructions pour chaque plateforme ciblée.

    <Tabs>
      <Tab title="Android">
        Ouvrez le fichier `android/app/build.gradle` et ajoutez les espaces réservés du manifeste suivants dans `android > defaultConfig` :

        ```groovy android/app/build.gradle lines
        android {
            // ...
            defaultConfig {
                // Ajoutez la ligne suivante
                manifestPlaceholders += [auth0Domain: "{yourDomain}", auth0Scheme: "https"]
            }
            // ...
        }
        ```

        Remplacez `{yourDomain}` par votre domaine Auth0 (p. ex., `example.us.auth0.com`).

        **schéma https**

        Pour utiliser le schéma `https` pour votre URL de rappel, configurez les [Android App Links](https://auth0.com/docs/get-started/applications/enable-android-app-links-support) pour votre application.

        **Pour l’authentification biométrique (facultatif) :**

        Si vous prévoyez utiliser l’authentification biométrique, mettez à jour votre `MainActivity.kt` afin qu’elle étende `FlutterFragmentActivity` :

        ```kotlin android/app/src/main/kotlin/.../MainActivity.kt lines
        import io.flutter.embedding.android.FlutterFragmentActivity

        class MainActivity: FlutterFragmentActivity() {
        }
        ```
      </Tab>

      <Tab title="iOS">
        Pour utiliser Universal Links (URL de rappel HTTPS) sur iOS 17.4+, configurez un domaine associé :

        1. Ouvrez votre application dans Xcode : `open ios/Runner.xcworkspace`
        2. Sélectionnez la cible **Runner** et accédez à **Signing &amp; Capabilities**
        3. Cliquez sur **+ Capability** et ajoutez **Associated Domains**
        4. Ajoutez l’entrée : `webcredentials:{yourDomain}`
        5. Dans le [Auth0 Dashboard](https://manage.auth0.com/#/applications), accédez à **Settings &gt; Advanced Settings &gt; Device Settings**
        6. Dans la section **iOS**, définissez votre **Team ID** et votre **App ID** (identifiant du bundle)

        <Info>
          La configuration de Universal Links est facultative. Ignorez cette étape si vous n’avez pas besoin de la prise en charge de Universal Links : le SDK utilisera automatiquement les schémas d’URL personnalisés.
        </Info>
      </Tab>

      <Tab title="macOS">
        Suivez les mêmes étapes que pour iOS, mais ouvrez `macos/Runner.xcworkspace` à la place.
      </Tab>
    </Tabs>

    <Warning>
      **Android** : Assurez-vous que la valeur `auth0Domain` correspond exactement à votre domaine Auth0. Si l’authentification échoue, vérifiez que cette valeur est identique au domaine affiché dans votre Auth0 Dashboard.

      **iOS/macOS** : Universal Links nécessitent un compte Apple Developer payant et iOS 17.4+/macOS 14.4+. Sur les versions antérieures, le SDK utilisera automatiquement les schémas d’URL personnalisés.
    </Warning>
  </Step>

  <Step title="Implement Connexion et Déconnexion" stepNumber={5}>
    [Universal Login](https://auth0.com/docs/authenticate/login/auth0-universal-login) est la façon la plus simple de configurer l’authentification dans votre application. Nous vous recommandons de l’utiliser pour profiter de la meilleure expérience, du meilleur niveau de sécurité et de l’éventail de fonctionnalités le plus complet.

    **Implement Login :**

    Importez l’Auth0 Flutter SDK et créez une instance `Auth0` :

    ```dart lib/auth_service.dart lines
    import 'package:auth0_flutter/auth0_flutter.dart';

    class AuthService {
      final auth0 = Auth0('{yourDomain}', '{yourClientId}');

      Future<Credentials> login() async {
        final credentials = await auth0.webAuthentication().login(useHTTPS: true);

        // Jeton d'accès -> credentials.accessToken
        // Jeton d'ID -> credentials.idToken
        // Profil utilisateur -> credentials.user

        return credentials;
      }
    }
    ```

    **Implement Déconnexion :**

    ```dart lib/auth_service.dart lines
    Future<void> logout() async {
      await auth0.webAuthentication().logout(useHTTPS: true);

      // L'utilisateur est maintenant déconnecté
      // Les informations d'identification ont été supprimées du stockage sécurisé
    }
    ```

    <Info>
      **iOS/macOS** : Le paramètre `useHTTPS: true` active les liens universels sur iOS 17.4+ et macOS 14.4+ pour une sécurité accrue.

      **Android** : Si vous utilisez un schéma personnalisé, transmettez-le à la méthode `login` pour que le SDK puisse acheminer correctement l’utilisateur vers la page de connexion, puis l’en ramener :

      ```dart
      await auth0.webAuthentication(scheme: 'YOUR CUSTOM SCHEME').login();
      ```
    </Info>
  </Step>

  <Step title="Afficher les renseignements du profil utilisateur" stepNumber={6}>
    Le profil utilisateur est récupéré automatiquement lorsque l’utilisateur se connecte. L’objet `Credentials` contient une propriété `user` qui comprend tous les renseignements du profil utilisateur, obtenus par décodage du `ID token`.

    ```dart lib/profile_screen.dart lines
    void displayUserProfile(Credentials credentials) {
      final user = credentials.user;
      
      print('User ID: ${user.sub}');
      print('Email: ${user.email}');
      print('Name: ${user.name}');
      print('Picture: ${user.pictureUrl}');
      print('Nickname: ${user.nickname}');
    }
    ```

    <Tip>
      Demandez les scopes appropriés au moment de la connexion pour accéder à certains champs du profil utilisateur. Les scopes par défaut sont `openid`, `profile`, `email` et `offline_access`.
    </Tip>
  </Step>

  <Step title="Exécutez votre application" stepNumber={7}>
    Compilez et exécutez votre application Flutter.

    **Dans votre terminal :**

    ```shellscript
    # Lister les appareils disponibles
    flutter devices

    # Exécuter sur Android
    flutter run -d android

    # Exécuter sur le simulateur iOS
    flutter run -d ios

    # Exécuter sur macOS
    flutter run -d macos
    ```

    **Déroulement attendu :**

    1. L’application se lance avec votre interface de connexion
    2. L’utilisateur appuie sur **Se connecter** → le navigateur ou l’onglet personnalisé s’ouvre avec Auth0 Universal Login
    3. L’utilisateur termine l’authentification
    4. Le navigateur redirige ensuite l’utilisateur vers votre application
    5. L’utilisateur est maintenant authentifié et ses identifiants sont enregistrés
  </Step>
</Steps>

<Check>
  **Checkpoint**

  Vous devriez maintenant avoir une expérience de connexion Auth0 entièrement fonctionnelle dans votre application Flutter. L’application utilise une authentification sécurisée par navigateur et stocke automatiquement les identifiants pour maintenir la session.
</Check>

<div id="troubleshooting-advanced-usage">
  ## Résolution des problèmes et utilisation avancée
</div>

<Accordion title="Problèmes courants et solutions">
  ### Incompatibilité de l’URL de rappel

  **Symptôme** : Erreur &quot;redirect&#95;uri&#95;mismatch&quot; ou l’authentification échoue sans message.

  **Solutions :**

  1. Vérifiez que les **Allowed Callback URLs** dans le Auth0 Dashboard correspondent exactement à la configuration de votre application
  2. Vérifiez le schéma (`https://` vs `http://`)
  3. Assurez-vous que le nom du package (Android) ou l’identifiant du bundle (iOS/macOS) est correct
  4. Vérifiez s’il y a une barre oblique finale

  ### Android : l’onglet Chrome personnalisé ne s’ouvre pas

  **Symptôme** : Rien ne se produit lors de l’appel à `login()`.

  **Correctif :**

  1. Vérifiez que `manifestPlaceholders` est correctement défini dans `build.gradle`
  2. Assurez-vous que la permission Internet figure dans `AndroidManifest.xml` :
     ```xml
     <uses-permission android:name="android.permission.INTERNET" />
     ```
  3. Vérifiez que Chrome ou un autre navigateur est installé sur l’appareil

  ### iOS : alerte &quot;Open in App&quot;

  **Symptôme** : Une boîte d’alerte s’affiche pour vous demander d’ouvrir l’application.

  **Correctif :** Il s’agit du comportement attendu avec `ASWebAuthenticationSession`. Pour la supprimer :

  * Utilisez les Universal Links (nécessite iOS 17.4+ et un compte Apple Developer payant)
  * Ou définissez `useEphemeralSession: true` (désactive le SSO) :

  ```dart expandable
  await auth0.webAuthentication().login(
    useHTTPS: true,
    useEphemeralSession: true,
  );
  ```

  ### Authentification annulée par l’utilisateur

  Gérez ce cas correctement dans votre traitement des erreurs :

  ```dart expandable
  try {
    final credentials = await auth0.webAuthentication().login(useHTTPS: true);
    // Gérer une ouverture de session réussie
  } on WebAuthenticationException catch (e) {
    if (e.code == 'USER_CANCELLED') {
      showMessage('La connexion a été annulée');
    } else {
      showMessage('Échec de la connexion : ${e.message}');
    }
  }
  ```
</Accordion>

<Accordion title="Gestion des identifiants">
  Le Auth0 Flutter SDK comprend un Credentials Manager intégré qui stocke de façon sécuritaire les identifiants de l’utilisateur. Sur les plateformes mobiles, les identifiants sont chiffrés et stockés dans le stockage sécurisé de la plateforme (Keychain sur iOS/macOS, SharedPreferences chiffré sur Android).

  ### Vérifier les identifiants stockés

  Avant d’inviter l’utilisateur à ouvrir une session, vérifiez si des identifiants valides existent déjà :

  ```dart lib/auth_service.dart expandable lines
  Future<bool> checkAuthentication() async {
    return await auth0.credentialsManager.hasValidCredentials();
  }
  ```

  ### Récupérer les identifiants stockés

  Récupérez les identifiants pour accéder aux jetons d’accès ou aux renseignements de l’utilisateur. Le Credentials Manager actualise automatiquement les jetons expirés lorsque c’est possible :

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> getCredentials() async {
    return await auth0.credentialsManager.credentials();
  }
  ```

  <Tip>
    Vous n’avez pas besoin de stocker manuellement les identifiants après l’ouverture de session : le SDK s’en charge automatiquement. Vous n’avez pas non plus besoin d’actualiser manuellement les jetons; le Credentials Manager le fait au besoin.
  </Tip>
</Accordion>

<Accordion title="Gestion des erreurs">
  Gérez les erreurs d’authentification correctement afin d’offrir une bonne expérience utilisateur.

  ```dart lib/auth_service.dart expandable lines
  import 'package:auth0_flutter/auth0_flutter.dart';

  Future<void> login() async {
    try {
      final credentials = await auth0.webAuthentication().login(useHTTPS: true);
      // Gérer une ouverture de session réussie
    } on WebAuthenticationException catch (e) {
      if (e.code == 'USER_CANCELLED') {
        // L’utilisateur a annulé l’ouverture de session
        print('Connexion annulée par l’utilisateur');
      } else {
        // Gérer les autres erreurs
        print('Erreur de connexion : ${e.message}');
      }
    }
  }

  Future<Credentials> getCredentials() async {
    try {
      return await auth0.credentialsManager.credentials();
    } on CredentialsManagerException catch (e) {
      if (e.isNoCredentialsFound) {
        // Aucun identifiant stocké, l’utilisateur doit ouvrir une session
        throw Exception('Veuillez d’abord ouvrir une session');
      } else if (e.isTokenRenewFailed) {
        // Le jeton d’actualisation a expiré, une nouvelle authentification est requise
        return await auth0.webAuthentication().login(useHTTPS: true);
      }
      rethrow;
    }
  }
  ```
</Accordion>

<Accordion title="Intégration Flutter avancée">
  ### Sécurité renforcée des identifiants avec la biométrie

  Implémentez l’authentification biométrique pour accéder aux identifiants sur mobile :

  ```dart lib/secure_auth_service.dart expandable lines
  class SecureAuthService {
    final auth0 = Auth0('{yourDomain}', '{yourClientId}');
    
    Future<void> enableBiometrics() async {
      // Activer l’authentification locale (Face ID, Touch ID, empreinte digitale)
      await auth0.credentialsManager.enableLocalAuthentication(
        title: 'Authenticate to access your account',
        cancelTitle: 'Cancel',
        fallbackTitle: 'Use passcode',
      );
    }
    
    Future<Credentials> getCredentialsWithBiometrics() async {
      // Cela exigera désormais une authentification biométrique
      return await auth0.credentialsManager.credentials();
    }
  }
  ```

  <Info>
    **Android**: Nécessite que `MainActivity` étende `FlutterFragmentActivity`, comme configuré à l’étape 4.

    **iOS/macOS**: Nécessite d’ajouter `NSFaceIDUsageDescription` à votre `Info.plist`.
  </Info>

  ### Scopes personnalisés et audience

  Demandez des scopes précis et une audience pour votre API :

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> loginWithCustomScopes() async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      scopes: {'openid', 'profile', 'email', 'offline_access', 'read:posts'},
      audience: 'https://myapi.example.com',
      parameters: {'prompt': 'login'},
    );
  }
  ```

  ### Organizations (B2B/Enterprise)

  Authentifiez les utilisateurs au sein d’une organisation précise :

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> loginWithOrganization(String organizationId) async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      organizationId: organizationId,
    );
  }

  // Ou inviter l’utilisateur à sélectionner une organisation
  Future<Credentials> loginWithOrganizationName(String organizationName) async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      organizationName: organizationName,
    );
  }
  ```
</Accordion>

<Accordion title="Déploiement en production">
  ### Préparation pour l’App Store

  * Configurez les Universal Links (iOS) et les App Links (Android) pour une authentification fluide
  * Effectuez des tests sur plusieurs tailles d’écran et versions du système d’exploitation
  * Mettez en place une gestion adéquate des erreurs en cas d’échec réseau
  * Ajoutez des règles ProGuard pour Android si vous utilisez l’obscurcissement du code
  * Respectez les politiques propres aux plateformes App Store/Play Store

  ### Considérations de sécurité

  * Utilisez le Credentials Manager intégré pour stocker les identifiants en production
  * Activez l’authentification biométrique pour les opérations sensibles
  * Envisagez l’épinglage de certificats pour renforcer la sécurité de l’API
  * Mettez en place une gestion adéquate de l’actualisation des tokens
  * Utilisez `useHTTPS: true` pour les Universal Links sur les plateformes prises en charge
</Accordion>
