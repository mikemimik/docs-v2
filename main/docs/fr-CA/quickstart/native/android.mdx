<Accordion title="Utilisez l’IA pour intégrer Auth0" icon="microchip-ai" iconType="solid" defaultOpen>
  Si vous utilisez un AI coding assistant comme Claude Code, Cursor ou GitHub Copilot, vous pouvez ajouter l’authentification Auth0 automatiquement en quelques minutes à l’aide d’[Agent Skills](https://agentskills.io/home).

  **Installation :**

  ```bash
  npx skills add auth0/agent-skills --skill auth0-quickstart --skill auth0-android
  ```

  **Ensuite, demandez à votre assistant IA :**

  ```text
  Add Auth0 authentication to my Android app
  ```

  Votre assistant IA créera automatiquement votre application Auth0, récupérera les identifiants, ajoutera la dépendance du SDK Android Auth0, configurera les espaces réservés du manifeste et implémentera les flux de connexion et de déconnexion. [Documentation complète sur Agent Skills →](/docs/fr-CA/quickstart/agent-skills)
</Accordion>

<div id="get-started">
  ## Pour commencer
</div>

<Steps>
  <Step title="Créer un nouveau projet Android" stepNumber={1}>
    Créez un nouveau projet Android pour ce Quickstart.

    **Dans Android Studio :**

    1. **File** → **New** → **New Project**
    2. Sélectionnez **Phone and Tablet** → le modèle **Empty Activity**
    3. Configurez votre projet :
       * **Name** : `Auth0-Android-Sample`
       * **Package name** : `com.auth0.samples.android`
       * **Language** : **Kotlin**
       * **Minimum SDK** : **API 24 (Android 7.0)**
       * **Build configuration language** : **Kotlin DSL**
    4. Cliquez sur **Finish**

    <Tip>
      Cela crée une application Android moderne avec Kotlin et Gradle Kotlin DSL, selon les meilleures pratiques actuelles du développement Android.
    </Tip>
  </Step>

  <Step title="Ajouter le SDK Auth0 avec Gradle" stepNumber={2}>
    Ajoutez le SDK Android Auth0 à votre projet à l’aide de Gradle.

    **Mettez à jour le fichier `build.gradle.kts` de votre application :**

    ```kotlin app/build.gradle.kts lines
    dependencies {   
        // SDK Auth0
        implementation("com.auth0.android:auth0:3.14.0")
    }
    ```

    **Ajoutez des espaces réservés du manifeste dans le fichier `build.gradle.kts` de votre application :**

    ```kotlin app/build.gradle.kts lines
    android {
        defaultConfig {
            // Ajoutez ces espaces réservés du manifeste
            manifestPlaceholders += mapOf(
                "auth0Domain" to "@string/com_auth0_domain", // sera défini à l'étape suivante
                "auth0Scheme" to "https"
            )
        }
    }

    ```

    **Ajoutez l’autorisation Internet à `AndroidManifest.xml` :**

    ```xml app/src/main/AndroidManifest.xml lines
    <?xml version="1.0" encoding="utf-8"?>
    <manifest xmlns:android="http://schemas.android.com/apk/res/android"
        xmlns:tools="http://schemas.android.com/tools">
        <uses-permission android:name="android.permission.INTERNET" />
    </manifest>
    ```

    <Tip>
      Le Auth0 SDK gère automatiquement la résolution des dépendances et intègre un stockage sécurisé des jetons.
    </Tip>
  </Step>

  <Step title="Configurez votre Auth0 App" stepNumber={3}>
    Ensuite, vous devez créer une nouvelle application dans votre tenant Auth0 et ajouter la configuration à votre projet Android.

    D’abord, préparez votre fichier `app/src/main/res/values/strings.xml` avec des valeurs fictives :

    ```xml app/src/main/res/values/strings.xml
    <?xml version="1.0" encoding="utf-8"?>
    <resources>
        <string name="com_auth0_domain">{yourDomain}</string>
        <string name="com_auth0_client_id">YOUR_AUTH0_CLIENT_ID</string>
        <string name="com_auth0_scheme">https</string>
    </resources>
    ```

    1. Accédez au [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. Cliquez sur **Applications** &gt; **Applications** &gt; **Create Application**
    3. Dans la fenêtre contextuelle, saisissez un nom pour votre application, sélectionnez `Native` comme type d’application, puis cliquez sur **Create**
    4. Ouvrez l’onglet **Settings** dans la page Application Details
    5. Remplacez `{yourDomain}` et `YOUR_AUTH0_CLIENT_ID` dans le fichier `strings.xml` par les valeurs **Domain** et **Client ID** du Dashboard

    Enfin, dans l’onglet **Settings** de votre page Application Details, configurez les URL suivantes :

    **Allowed Callback URLs :**

    ```
    https://{yourDomain}/android/PACKAGE_NAME/callback
    ```

    **URLs de déconnexion autorisées :**

    ```
    https://{yourDomain}/android/PACKAGE_NAME/callback
    ```

    Remplacez `{yourDomain}` par votre véritable domaine Auth0 (p. ex., `dev-abc123.us.auth0.com`).

    <Info>
      **Allowed Callback URLs** constituent une mesure de sécurité essentielle pour garantir que les utilisateurs sont redirigés de façon sécuritaire vers votre application après l’authentification. Sans URL correspondante, le processus de connexion échouera et les utilisateurs verront une page d’erreur Auth0 au lieu d’accéder à votre application.

      **Allowed Logout URLs** sont essentiels pour offrir une expérience utilisateur fluide après la déconnexion. Sans URL correspondante, les utilisateurs ne seront pas redirigés vers votre application après la déconnexion et resteront plutôt sur une page Auth0 générique.

      Le schéma d’URL inclut le nom de votre package (`com.auth0.samples.android`) afin de garantir que le callback est acheminé vers votre application.
    </Info>

    <Warning>
      **Important** : Assurez-vous que le nom du package dans vos URL de callback correspond à votre `applicationId` dans `build.gradle.kts`. Si l’authentification échoue, vérifiez que ces valeurs sont identiques.
    </Warning>

    <Tip>
      Lorsque vous utilisez le schéma `https` (tel qu’il est configuré ci-dessus), vous devez configurer Android App Links pour qu’Android achemine l’URL de callback directement vers votre application au lieu de l’ouvrir dans un navigateur. Consultez la section **Configurer Android App Links** sous [Dépannage et Advanced](#troubleshooting--advanced) ci-dessous.
    </Tip>
  </Step>

  <Step title="Initialiser le SDK Auth0" stepNumber={4}>
    Créez une instance d’Auth0 dans votre Activity afin de communiquer avec Auth0.

    **Dans votre `MainActivity.kt` :**

    ```kotlin MainActivity.kt lines
    import com.auth0.android.Auth0
    import com.auth0.android.authentication.AuthenticationException
    import com.auth0.android.callback.Callback
    import com.auth0.android.provider.WebAuthProvider
    import com.auth0.android.result.Credentials

    class MainActivity : ComponentActivity() {
        private lateinit var auth0: Auth0

        override fun onCreate(savedInstanceState: Bundle?) {
            super.onCreate(savedInstanceState)
            
            // Initialiser Auth0
            auth0 = Auth0.getInstance(
                getString(R.string.com_auth0_client_id),
                getString(R.string.com_auth0_domain)
            )
        }
    }
    ```

    <Tip>
      L’instance Auth0 est initialisée avec votre client ID et votre domaine, tels qu’ils sont définis dans le fichier `strings.xml` que vous avez configuré plus tôt. Cette instance sera utilisée pour toutes les opérations d’authentification.
    </Tip>
  </Step>

  <Step title="Implement Login et Logout" stepNumber={5}>
    **Implémentez la connexion :** Utilisez WebAuthProvider pour lancer la page de connexion Universal Login.

    Ajoutez ces méthodes à votre `MainActivity` :

    <Tabs>
      <Tab title="Callback Kotlin">
        ```kotlin MainActivity.kt lines
        private fun login() {
            WebAuthProvider.login(auth0)
                .withScheme("https")
                .withScope("openid profile email offline_access")
                .start(this, object : Callback<Credentials, AuthenticationException> {
                    override fun onSuccess(credentials: Credentials) {
                        // Enregistrer les informations d’authentification
                        // L’utilisateur est authentifié
                    }

                    override fun onFailure(exception: AuthenticationException) {
                        // Gérer les cas d’erreur
                    }
                })
        }
        ```
      </Tab>

      <Tab title="Coroutine">
        ```kotlin MainActivity.kt lines
        private fun login() {
            lifecycleScope.launch {
                try {
                    val credentials = WebAuthProvider.login(auth0)
                        .withScheme("https")
                        .withScope("openid profile email offline_access")
                        .await(this@MainActivity)
                    // Enregistrer les informations d’authentification
                    // L’utilisateur est authentifié
                } catch (exception: AuthenticationException) {
                    // Gérer l’erreur
                }
            }
        }
        ```
      </Tab>
    </Tabs>

    **Implémentez la déconnexion :** Utilisez WebAuthProvider pour effacer la session de l’utilisateur.

    <Tabs>
      <Tab title="Callback Kotlin">
        ```kotlin MainActivity.kt lines
        private fun logout() {
            WebAuthProvider.logout(auth0)
                .withScheme("https")
                .start(this, object : Callback<Void?, AuthenticationException> {
                    override fun onSuccess(result: Void?) {
                        // Effacer les informations d’authentification enregistrées
                        // L’utilisateur est déconnecté
                    }

                    override fun onFailure(exception: AuthenticationException) {
                        // Gérer l’erreur
                    }
                })
        }
        ```
      </Tab>

      <Tab title="Coroutine">
        ```kotlin MainActivity.kt lines
        private fun logout() {
            lifecycleScope.launch {
                try {
                    WebAuthProvider.logout(auth0)
                        .withScheme("https")
                        .await(this@MainActivity)
                    // Effacer les informations d’authentification enregistrées
                    // L’utilisateur est déconnecté
                } catch (exception: AuthenticationException) {
                    // Gérer l’erreur
                }
            }
        }
        ```
      </Tab>
    </Tabs>

    <Info>
      Les méthodes `login()` et `logout()` doivent être appelées lorsque l’utilisateur appuie sur les boutons correspondants dans votre UI. Le code utilise `this` (qui fait référence à l’Activity) comme paramètre de contexte, ce qui est nécessaire pour que WebAuthProvider lance Chrome Custom Tabs et gère le flux d’authentification.
    </Info>
  </Step>

  <Step title="Lancez votre application" stepNumber={6}>
    Compilez et exécutez votre application Android.

    **Dans Android Studio :**

    ```shellscript
    # Synchroniser le projet avec les fichiers Gradle (ou utiliser « Sync Now » dans Android Studio)
    ./gradlew clean build

    # Compiler et installer sur l'appareil connecté ou l'émulateur
    ./gradlew installDebug

    # Ou exécuter directement depuis Android Studio
    # Cliquer sur le bouton « Run » ou appuyer sur Shift+F10
    ```

    **Déroulement attendu :**

    1. L’application s’ouvre avec un bouton &quot;Se connecter&quot; et une icône de bouclier
    2. Touchez &quot;Se connecter&quot; → un Chrome Custom Tab s’ouvre → Terminez la connexion
    3. Retour automatique à l’application
    4. Succès !!

    <Warning>
      Android affichera une boîte de dialogue de sélection du navigateur si plusieurs navigateurs sont installés. Les Chrome Custom Tabs offrent la meilleure expérience utilisateur pour l’authentification Auth0.
    </Warning>
  </Step>
</Steps>

  **Vérification**

<div id="troubleshooting-advanced">
  ## Dépannage et options avancées
</div>

<Accordion title="Problèmes courants et solutions">
  ### Chrome Custom Tab ne redirige pas vers l’application

  **Solutions :**

  1. Vérifiez que les **Allowed Callback URLs** dans Auth0 Dashboard correspondent exactement à votre `applicationId`
  2. Vérifiez que les espaces réservés du manifeste dans `build.gradle.kts` sont corrects
  3. Assurez-vous que les URL HTTPS et les URL avec schéma personnalisé sont toutes les deux configurées
  4. Nettoyez puis régénérez le projet : **Build** → **Clean Project** → **Rebuild Project**

  ### L’application plante : &#39;Auth0 domain not found&#39;

  **Correctif :**

  1. Vérifiez que les valeurs `com_auth0_domain` et `com_auth0_client_id` sont correctes
  2. Assurez-vous qu’il n’y a pas de faute de frappe dans le format du domaine (il ne doit pas inclure `https://`)

  ### Erreurs de build avec les dépendances

  **Correctif :**

  1. Mettez à jour Android Gradle Plugin vers la version la plus récente dans `build.gradle` (au niveau du projet)
  2. Synchronisez le projet : **File** → **Sync Project with Gradle Files**
  3. Nettoyez la build : `./gradlew clean build`

  ### Authentification annulée par l’utilisateur

  Gérez cela correctement dans votre fonction de rappel d’erreur :

  ```kotlin {3,4,5,6} expandable
  override fun onFailure(exception: AuthenticationException) {
      when {
          exception.isAuthenticationCanceled -> 
              showMessage("Login was cancelled")
          exception.isBrowserAppNotAvailable -> 
              showMessage("No browser available")
          else -> 
              showMessage("Login failed: ${exception.getDescription()}")
      }
  }
  ```

  ### Erreur : aucun navigateur compatible

  * Installez Chrome ou un autre navigateur moderne sur votre appareil/émulateur
  * Activez Chrome Custom Tabs pour une meilleure expérience utilisateur
  * Testez sur un appareil réel avec Chrome installé
</Accordion>

<Accordion title="Configurer Android App Links">
  [Android App Links](https://developer.android.com/training/app-links) permettent à votre application de se définir comme le gestionnaire par défaut des callback URLs d’Auth0, pour une expérience d’authentification plus sécurisée et fluide. Sans App Links, Android peut afficher une boîte de dialogue de désambiguïsation demandant à l’utilisateur de choisir entre votre application et un navigateur.

  <Note>
    Les App Links utilisent des callbacks avec schéma `https` vérifié. C’est plus sécuritaire que les schémas d’URL personnalisés, qui peuvent être vulnérables à des [attaques d’usurpation de client](https://datatracker.ietf.org/doc/html/rfc8252#section-8.6).
  </Note>

  ### Obtenir l’empreinte de votre certificat de signature

  Vous avez besoin de l’empreinte SHA256 du certificat de signature de votre application. Exécutez la commande suivante dans votre terminal :

  ```shellscript
  # Pour les builds de débogage
  keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android

  # Pour les builds de production
  keytool -list -v -keystore my-release-key.keystore
  ```

  Copiez la valeur de l’empreinte **SHA256** dans la sortie.

  ### Configurer dans Auth0 Dashboard

  1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis sélectionnez votre application
  2. Faites défiler jusqu’au bas de la page **Settings** et sélectionnez **Show Advanced Settings**
  3. Sélectionnez l’onglet **Device Settings**
  4. Sous **Android**, entrez :
     * **App Package Name**: Votre `applicationId` (p. ex., `com.auth0.samples.android`)
     * **SHA256 Cert Fingerprints**: L’empreinte que vous avez copiée ci-dessus
  5. Cliquez sur **Save Changes**

  ### Vérifier la configuration

  Auth0 génère automatiquement le fichier `assetlinks.json` utilisé par Android pour vérifier votre application. Testez-le en accédant à :

  ```
  https://{yourDomain}/.well-known/assetlinks.json
  ```

  Vous devriez voir une réponse JSON contenant le nom de votre package et l’empreinte du certificat :

  ```json
  [{
    "relation": ["delegate_permission/common.handle_all_urls"],
    "target": {
      "namespace": "android_app",
      "package_name": "[YOUR_PACKAGE_NAME]",
      "sha256_cert_fingerprints": ["YOUR_SHA256_FINGERPRINT"]
    }
  }]
  ```

  <Info>
    Le Quickstart utilise déjà `https` comme schéma dans les espaces réservés du manifeste et les appels `WebAuthProvider`, ce qui est requis pour App Links. Aucun changement de code n’est nécessaire si vous avez suivi les étapes ci-dessus.
  </Info>

  Pour en savoir plus, consultez la documentation [Enable Android App Links Support](/docs/fr-CA/get-started/applications/enable-android-app-links-support) et le guide [Verify App Links](https://developer.android.com/training/app-links/verify-site-associations) d’Android.
</Accordion>

<Accordion title="Utiliser un schéma d’URL personnalisé">
  Si vous ne pouvez pas utiliser Android App Links (par exemple, si vous ciblez des versions de l’API Android antérieures à 23), vous pouvez configurer un schéma d’URL personnalisé à la place.

  <Warning>
    Les schémas d’URL personnalisés sont moins sécurisés que les App Links, car ils peuvent être vulnérables aux [attaques d’usurpation de client](https://datatracker.ietf.org/doc/html/rfc8252#section-8.6). Utilisez les App Links chaque fois que possible.
  </Warning>

  1. Mettez à jour l’espace réservé `auth0Scheme` dans le manifeste de votre `app/build.gradle.kts` :

  ```kotlin app/build.gradle.kts lines
  android {
      defaultConfig {
          manifestPlaceholders += mapOf(
              "auth0Domain" to "@string/com_auth0_domain",
              "auth0Scheme" to "myapp" // Utilisez un schéma personnalisé unique
          )
      }
  }
  ```

  2. Mettez à jour les **Allowed Callback URLs** et **Allowed Logout URLs** dans les paramètres de votre application du [Auth0 Dashboard](https://manage.auth0.com/#/applications) pour utiliser le schéma personnalisé :

  ```
  myapp://{yourDomain}/android/PACKAGE_NAME/callback
  ```

  3. Indiquez le schéma personnalisé lors de l’appel à `WebAuthProvider` :

  ```kotlin MainActivity.kt lines
  WebAuthProvider.login(auth0)
      .withScheme("myapp")
      .withScope("openid profile email offline_access")
      .start(this, callback)
  ```

  <Note>
    Les schémas personnalisés [ne peuvent contenir que des lettres minuscules](https://developer.android.com/guide/topics/manifest/data-element).
  </Note>
</Accordion>

<Accordion title="Déploiement en production">
  ### Préparation pour l’App Store

  * Configurez Android App Links pour une authentification sans friction
  * Testez sur plusieurs versions d’Android et tailles d’écran
  * Mettez en place une gestion adéquate des erreurs en cas de panne réseau
  * Ajoutez des règles ProGuard pour le SDK Auth0 si vous utilisez l’obfuscation du code
  * Respectez les politiques du Google Play Store pour les flux d’authentification

  ### Considérations de sécurité

  * Utilisez `SecureCredentialsManager` pour stocker les identifiants en production
  * Mettez en place le certificate pinning pour renforcer la sécurité de l’API
  * Envisagez Android Keystore pour mieux protéger les identifiants
  * Activez l’authentification biométrique pour les opérations sensibles
</Accordion>

<Accordion title="Intégration Android avancée">
  ### Sécurité renforcée des identifiants

  Mettez en place l’authentification biométrique pour accéder aux identifiants :

  ```kotlin AuthenticationManager.kt expandable lines
  class AuthenticationManager(private val context: Context) {
      
      private val credentialsManager: SecureCredentialsManager
      
      init {
          val authentication = AuthenticationAPIClient(auth0)
          val storage = SharedPreferencesStorage(context)
          credentialsManager = SecureCredentialsManager(context, authentication, storage)
          
          // Activer l’authentification biométrique
          credentialsManager.requireAuthentication(
              context as FragmentActivity,
              REQUEST_CODE_BIOMETRIC,
              "Biometric Authentication",
              "Please authenticate to access your account"
          )
      }
      
      companion object {
          private const val REQUEST_CODE_BIOMETRIC = 321
      }
  }
  ```

  ### Scopes personnalisés et audience

  Demandez des scopes précis et l’audience de votre API :

  ```kotlin AuthenticationManager.kt expandable lines
  fun login() {
      WebAuthProvider.login(auth0)
          .withScheme("https")
          .withScope("openid profile email offline_access read:posts")
          .withAudience("https://myapi.example.com")
          .withParameter("prompt", "login")
          .start(context as MainActivity, loginCallback)
  }
  ```

  ### Configuration réseau

  Gérez la sécurité réseau et le certificate pinning :

  ```xml app/src/main/res/xml/network_security_config.xml expandable lines
  <?xml version="1.0" encoding="utf-8"?>
  <network-security-config>
      <domain-config>
          <domain includeSubdomains="true">your-auth0-domain.auth0.com</domain>
          <pin-set>
              <pin digest="SHA-256">AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA=</pin>
          </pin-set>
      </domain-config>
  </network-security-config>
  ```

  Ajoutez ceci à `AndroidManifest.xml` :

  ```xml
  <application
      android:networkSecurityConfig="@xml/network_security_config"
      ... />
  ```
</Accordion>
