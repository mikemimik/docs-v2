    Auth0 vous permet d&#39;ajouter l&#39;authentification et d&#39;accéder aux informations du profil utilisateur dans votre application. Ce guide

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Pour utiliser les services Auth0, vous devez avoir une application configurée dans l’Auth0 Dashboard. L’application Auth0 est
      l’endroit où vous configurerez le fonctionnement de l’authentication pour le projet que vous développez.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle application Auth0 ou sélectionner une application existante qui représente
      le projet que vous souhaitez intégrer. Chaque application dans Auth0 se voit attribuer un client ID
      alphanumérique unique que le code de votre application utilisera pour appeler les API Auth0 au moyen du SDK.

      Tous les paramètres que vous configurez à l’aide de ce Quickstart seront automatiquement mis à jour pour votre Application dans le [Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), où vous pourrez gérer vos Applications par la suite.

      Si vous préférez consulter une configuration complète, vous pouvez plutôt afficher un exemple d’application.

      ### Configurer les callback URLs

      Une callback URL est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      authentication. Si elle n’est pas définie, les utilisateurs ne seront pas redirigés vers votre application après s’être connectés.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la sur
        `http://localhost:3000/callback`.
      </Info>

      ### Configurer les Logout URLs

      Une logout URL est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      logout. Si elle n’est pas définie, les utilisateurs ne pourront pas se déconnecter de votre application et recevront une erreur.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la sur `http://localhost:3000`.
      </Info>

      Pour cette intégration, vous ajouterez plusieurs dépendances, comme la bibliothèque Authlib. Créez un
      fichier `requirements.txt` dans le répertoire de votre projet et incluez ce qui suit :

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      Exécutez la commande suivante dans votre shell afin que ces dépendances soient disponibles :

      ```text lines
      pip install -r requirements.txt
      ```

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      Dans cet exemple, vous ajouterez quatre routes à votre application : les routes de connexion, de rappel, de déconnexion et d’index.

      * `login` - Lorsque les visiteurs de votre application accèdent à la route `/login`, ils sont redirigés vers Auth0 pour
        amorcer le flux d’authentification.
      * `callback` - Une fois que vos utilisateurs ont terminé de se connecter avec Auth0, ils reviennent à votre application
        par la route `/callback`. Cette route enregistre la session de l’utilisateur et lui évite
        d’avoir à se reconnecter lors de sa prochaine visite.
      * `logout` - La route `/logout` déconnecte les utilisateurs de votre application. Cette route efface
        la session utilisateur dans votre application et redirige vers le point de terminaison de déconnexion d’Auth0 afin de s’assurer que la session n’est plus
        conservée. Ensuite, l’application redirige l’utilisateur vers votre route d’accueil.
      * `index` - La route d’accueil affichera les détails d’un utilisateur authentifié ou permettra aux visiteurs de se connecter.
