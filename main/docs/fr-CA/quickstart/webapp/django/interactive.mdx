  <Content>
    Auth0 vous permet d&#39;ajouter l&#39;authentification et d&#39;accéder aux renseignements du profil utilisateur dans votre application. Ce guide
    explique comment intégrer Auth0 à une application Python [Django](https://www.djangoproject.com/) à l&#39;aide du SDK [Authlib](https://authlib.org/).

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Pour utiliser les services Auth0, vous devez avoir une application configurée dans l’Auth0 Dashboard. L’application Auth0 est
      l’endroit où vous configurerez le fonctionnement de l’authentification pour le projet que vous développez.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle application Auth0 ou sélectionner une application existante qui représente
      le projet que vous souhaitez intégrer. Chaque application dans Auth0 se voit attribuer un client ID unique alphanumérique
      que le code de votre application utilisera pour appeler les API Auth0 au moyen du SDK.

      Tous les paramètres que vous configurez à l’aide de ce Quickstart seront automatiquement mis à jour pour votre Application dans le [Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), où vous pourrez gérer vos Applications par la suite.

      Si vous préférez explorer une configuration complète, vous pouvez plutôt consulter un exemple d’application.

      ### Configurer les URL de rappel

      Une URL de rappel est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      authentification. Si elle n’est pas définie, les utilisateurs ne seront pas redirigés vers votre application après s’être connectés.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la à
        `http://localhost:3000/callback`.
      </Info>

      ### Configurer les URL de logout

      Une URL de logout est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      déconnexion. Si elle n’est pas définie, les utilisateurs ne pourront pas se déconnecter de votre application et recevront une erreur.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la à `http://localhost:3000`.
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Pour cette intégration, vous ajouterez plusieurs dépendances, comme la bibliothèque Authlib. Créez un
      fichier `requirements.txt` dans le répertoire de votre projet et incluez ce qui suit :

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      Exécutez la commande suivante dans votre shell pour que ces dépendances soient disponibles :

      ```text lines
      pip install -r requirements.txt
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Ensuite, créez un fichier `.env` dans le répertoire de votre projet. Ce fichier contiendra les clés du client et d’autres
      détails de configuration.

      ```text lines
      AUTH0_CLIENT_ID={yourClientId}
      AUTH0_CLIENT_SECRET={yourClientSecret}
      AUTH0_DOMAIN={yourDomain}
      ```
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Si vous avez déjà mis en place une application Django, passez à l’étape suivante. Pour un nouveau projet d’application, exécutez la
      commande suivante :

      ```text lines
      django-admin startproject webappexample
      ```

      Placez-vous dans le nouveau dossier du projet :

      ```text lines
      cd webappexample
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Ouvrez le fichier `webappexample/settings.py` pour passer en revue les valeurs du fichier `.env`.

      En haut du fichier, ajoutez les importations `os` et `dotenv`.

      Ensuite, sous la définition de `BASE_DIR`, ajoutez la variable `TEMPLATE_DIR`.

      Ensuite, repérez la variable `TEMPLATES` et mettez à jour la valeur `DIRS` pour y ajouter notre
      chaîne `TEMPLATE_DIR`. Cela détermine le chemin des fichiers de gabarit, que vous créerez dans une
      étape suivante. Conservez tel quel tout autre contenu de ce tableau.

      À la fin de ce fichier, ajoutez le code permettant de charger la configuration Auth0.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Pour commencer à créer votre application, ouvrez le fichier `webappexample/views.py` dans votre IDE.

      Importez toutes les bibliothèques dont votre application a besoin.

      Vous pouvez maintenant configurer Authlib pour qu’il gère l’authentification de votre application avec Auth0.

      Pour en savoir plus sur les options de configuration offertes par la méthode OAuth `register()` d’Authlib, consultez [sa documentation.](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Dans cet exemple, vous ajouterez quatre routes pour votre application : les routes login, callback, logout et d’index.

      * `login` - Lorsque des visiteurs de votre application accèdent à la route `/login`, ils sont redirigés vers Auth0 pour
        amorcer le flux d’authentification.
      * `callback` - Une fois que vos utilisateurs se sont connectés avec Auth0, ils sont redirigés vers votre application
        à la route `/callback`. Cette route enregistre la session de l’utilisateur et lui évite
        d’avoir à se reconnecter lors de sa prochaine visite.
      * `logout` - La route `/logout` déconnecte les utilisateurs de votre application. Cette route efface
        la session utilisateur dans votre application et redirige vers le endpoint de logout d’Auth0 afin de s’assurer que la session n’est plus
        conservée. Ensuite, l’application redirige l’utilisateur vers votre route d’accueil.
      * `index` - La route d’accueil affichera les détails d’un utilisateur authentifié ou permettra aux visiteurs de se connecter.
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Remplacez le contenu de votre fichier `webappexample/urls.py` par le code à droite afin de configurer
      ces nouvelles routes.

      Les routes `/login`, `/callback`, `/logout` et `/` pointeront ainsi
      vers les bons gestionnaires.
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      Ensuite, vous créerez un fichier de gabarit utilisé dans la route de la page d’accueil.

      Créez un nouveau sous-répertoire nommé `templates` dans le dossier `webappexample`, puis créez
      un fichier `index.html`.

      Le fichier `index.html` contiendra le code du gabarit pour afficher les renseignements de l’utilisateur s’il est connecté, ou lui présenter
      un bouton de connexion s’il est déconnecté.
    </Section>

    <Section id={sections[9].id} title={sections[9].title} stepNumber="10">
      Vous êtes prêt à exécuter votre application ! Dans le répertoire de votre projet, ouvrez un shell et utilisez :

      ```text lines
      python3 manage.py migrate
      python3 manage.py runserver 3000
      ```

      Vous devriez maintenant pouvoir ouvrir votre application dans votre navigateur à l’adresse [http://localhost:3000](http://localhost:3000).

      <Info>
        ##### Point de contrôle

        Allez à l’adresse [http://localhost:3000](http://localhost:3000/) pour le vérifier. Vous devriez y voir un bouton de connexion qui vous redirige vers
        Auth0 pour vous connecter, puis de retour vers votre application pour afficher les informations de votre profil.
      </Info>
    </Section>
