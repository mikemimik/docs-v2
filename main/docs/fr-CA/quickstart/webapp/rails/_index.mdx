    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Pour utiliser les services Auth0, vous devez configurer une application dans l’Auth0 Dashboard. L’application Auth0 est
      l’endroit où vous configurerez le fonctionnement de l’authentification pour le projet que vous développez.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle application Auth0 ou sélectionner une application existante qui correspond
      au projet que vous souhaitez intégrer. Chaque application dans Auth0 reçoit un ID client alphanumérique unique
      que le code de votre application utilisera pour effectuer des requêtes vers les API Auth0 au moyen du SDK.

      Tous les paramètres que vous configurez à l’aide de ce Quickstart seront automatiquement mis à jour pour votre application dans le [Dashboard](https://manage.auth0.com/#/), où vous
      pourrez gérer vos Applications par la suite.

      Si vous préférez consulter une configuration complète, vous pouvez plutôt afficher un exemple d’application.

      ### Configurer les URL de rappel

      Une URL de rappel est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      authentification. Si elle n’est pas définie, les utilisateurs ne seront pas redirigés vers votre application après s’être connectés.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la à
        `http://localhost:3000/auth/auth0/callback`.
      </Info>

      ### Configurer les URL de déconnexion

      Une URL de déconnexion est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      déconnexion. Si elle n’est pas définie, les utilisateurs ne pourront pas se déconnecter de votre application et recevront une erreur.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la à `http://localhost:3000`.
      </Info>

      ### Configurer les origines Web autorisées

      Une origine Web autorisée est une URL que vous souhaitez autoriser à accéder à votre flux d’authentification. Elle doit
      contenir l’URL de votre projet. Si elle n’est pas correctement définie, votre projet ne pourra pas actualiser silencieusement
      les jetons d’authentification; vos utilisateurs seront donc déconnectés lors de leur prochaine visite dans votre application ou lorsqu’ils actualiseront une
      page.

      <Info>
        Si vous suivez notre projet d’exemple, définissez-la à `http://localhost:3000`.
      </Info>

      Utilisez `omniauth-auth0`, une [stratégie OmniAuth](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication) personnalisée, pour prendre en charge le processus d’authentification.

      Ajoutez les dépendances suivantes à votre `Gemfile` :

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # empêche les requêtes d'authentification falsifiées
      ```

      Une fois les gems ajoutés, installez-les avec `bundle install`.

      Créez un contrôleur Auth0 pour gérer le callback d’authentification, l’action `logout` et les méthodes permettant de
      construire l’URL de déconnexion.

      Exécutez la commande :
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

      Dans la méthode callback, assignez à la session active le hash des renseignements sur l’utilisateur, renvoyé dans
      `request.env['omniauth.auth']`.

      Pour configurer la déconnexion, effacez tous les objets stockés dans la session en appelant la méthode `reset_session`
      dans l’action `logout`. Ensuite, redirigez vers l’endpoint de déconnexion d’Auth0. Pour en savoir plus sur
      `reset_session`, consultez la [documentation Ruby on Rails ActionController](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668).

      Un utilisateur peut maintenant se connecter à votre application en accédant au point de terminaison `/auth/auth0`.

      <Warning>
        Pour [éviter les requêtes d’authentification falsifiées](https://github.com/cookpad/omniauth-rails_csrf_protection), utilisez les méthodes utilitaires `link_to` ou
        `button_to` avec la méthode `:post`.
      </Warning>

      ```ruby lines
      <!-- Placez un bouton de connexion n'importe où dans votre application -->
      <%= button_to 'Login', '/auth/auth0', method: :post %>
      ```

      <Note>
        ##### Point de contrôle

        Ajoutez à votre application un bouton qui redirige l’utilisateur vers le point de terminaison `/auth/auth0` lorsqu’il
        clique dessus. Notez que l’utilisateur est redirigé vers Auth0 pour se connecter, puis renvoyé vers votre application après une
        authentification réussie.

      Maintenant que vous pouvez vous connecter à votre application Rails, vous avez besoin d’un
      moyen de vous déconnecter. Pour déconnecter un utilisateur, redirigez-le vers l’action `auth/logout`, qui le redirige

      </Info>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9" isSingleColumn>
      Pour afficher le profil de l&#39;utilisateur, votre application doit comporter une route protégée. Vous pouvez utiliser un [Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) pour contrôler l&#39;accès à des routes pouvant être partagées entre plusieurs
      contrôleurs. Le Concern doit rediriger automatiquement vers Auth0 lorsque l&#39;utilisateur n&#39;est pas authentifié. Sinon, le
      Concern doit renvoyer le profil de l&#39;utilisateur actuel.

      Une fois que vous avez un Concern, incluez-le dans tout contrôleur qui nécessite un utilisateur connecté. Vous pouvez ensuite accéder au
      profil de l&#39;utilisateur dans la session `session[:userinfo]`, comme dans l&#39;exemple suivant :

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      Une fois l’utilisateur récupéré à partir de la session, utilisez ses informations dans votre interface :

      ```html lines
      <div>
      <p>Normalized User Profile:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>Full User Profile:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Note>
        ##### Vérification

        Ajoutez le concern `Secured` à votre application, puis incluez-le dans le contrôleur qui nécessite qu’un
        utilisateur soit authentifié pour y accéder. Vérifiez qu’un utilisateur authentifié peut accéder aux actions de ce
        contrôleur et que les utilisateurs non authentifiés sont redirigés vers Auth0 pour s’authentifier.
      </Note>
