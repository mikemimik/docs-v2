<div id="troubleshooting">
  # Dépannage
</div>

Voici quelques rubriques de dépannage si vous rencontrez des problèmes lors de l’utilisation du Quickstart Rails.

<div id="using-a-reverse-proxy">
  ## Utilisation d’un proxy inverse
</div>

Le paramètre `redirect_uri` qu’OmniAuth génère lors de la redirection vers la connexion est basé sur l’en-tête `Host` transmis à Rails. Cela peut entraîner l’envoi d’URL de rappel incorrectes lors de l’utilisation de cette stratégie (et d’OmniAuth en général) avec un proxy inverse. Vous pouvez modifier l’hôte utilisé par OmniAuth à l’aide de l’extrait suivant :

```
OmniAuth.config.full_host = lambda do |env|
    scheme         = env['rack.url_scheme']
    local_host     = env['HTTP_HOST']
    forwarded_host = env['HTTP_X_FORWARDED_HOST']
    forwarded_host.blank? ? "#{scheme}://#{local_host}" : "#{scheme}://#{forwarded_host}"
end
```

[Consultez ce fil StackOverflow pour en savoir plus](https://stackoverflow.com/a/7135029/728480).

<div id="actioncontrollerinvalidauthenticitytoken">
  ## ActionController::InvalidAuthenticityToken
</div>

Cela est probablement dû à l’absence d’un jeton CSRF nécessaire pour envoyer la requête de connexion au moyen de POST. Si vous inspectez le bouton de connexion dans votre navigateur, vous devriez voir quelque chose comme ceci :

```erb lines
<a data-method="post" href="auth/auth0">Login</a>
```

... et dans l’élément `<head>` de la page, vous devriez inclure des balises méta CSRF comme celles-ci :

```erb lines
<meta name="csrf-param" content="authenticity_token">
<meta name="csrf-token" content="UY2XpKwxzwBWalxFVJ8yKsao/33it7If09BnZewpHifVPSpFJd2LrA7xgQn6VQrhZNGjgZoLI3kV+bkQHtr+Rw==">
```

Une fois ces éléments en place, Rails convertira le lien de connexion afin qu’il transmette le jeton CSRF au backend par POST pour le vérifier avant de rediriger vers la connexion.

<div id="actiondispatchcookiescookieoverflow">
  ## ActionDispatch::Cookies::CookieOverflow
</div>

Cette erreur signifie qu’une session par cookie est utilisée et que, puisque l’ensemble du profil est stocké, elle dépasse la taille maximale de 4 ko. Si vous n’arrivez pas à accéder au profil utilisateur, ou si vous obtenez une erreur semblable à `NoMethodError`, `undefined method '[]' for nil:NilClass`, essayez d’utiliser le stockage en mémoire pendant le développement.

Accédez à `/config/initializers/session_store.rb` (ou créez-le s’il n’existe pas) et ajoutez ce qui suit :

```
Rails.application.config.session_store :cache_store
```

Rendez-vous dans `/config/environments/development.rb` et ajoutez ce qui suit :

```
config.cache_store = :memory_store
```

Redémarrez votre serveur Rails pour que ces changements soient pris en compte.

Il est recommandé d&#39;utiliser un stockage en mémoire, comme MemCached, pour les applications de production.

<div id="ssl-issues">
  ## Problèmes SSL
</div>

Dans certaines configurations, Ruby peut ne pas réussir à trouver les certificats d’autorité de certification (certificats CA).

Téléchargez l’ensemble des certificats CA dans le répertoire du projet :

```bash lines
curl -L -o lib/ca-bundle.crt http://curl.haxx.se/ca/ca-bundle.crt
```

Ajoutez cet initialiseur dans `config/initializers/fix_ssl.rb` :

```rb lines
# config/initializers/fix_ssl.rb

require 'open-uri'
require 'net/https'

module Net
  class HTTP
    alias_method :original_use_ssl=, :use_ssl=

    def use_ssl=(flag)
      path = ( Rails.env == "development") ? "lib/ca-bundle.crt" : "/usr/lib/ssl/certs/ca-certificates.crt"
      self.ca_file = Rails.root.join(path).to_s
      self.verify_mode = OpenSSL::SSL::VERIFY_PEER
      self.original_use_ssl = flag
    end
  end
end
```

<div id="failure-messageinvalid_credentials">
  ## &quot;failure message=invalid_credentials&quot;
</div>

Ce problème ne se produit pas en local, mais peut survenir dans un environnement de préproduction ou de production. Le message d’erreur peut s’afficher comme suit :

```
omniauth: (auth0) Authentication failure! invalid_credentials: OAuth2::Error, server_error: The redirect URI is wrong. You send [wrong url], and we expected [callback url set in your app settings]
```

Pour résoudre ce problème, ajoutez ce qui suit dans `config/environments/staging.rb` ou `production.rb` :

```
OmniAuth.config.full_host = "http://www.example.com"
```

Remplacez `http://www.example.com` par l’URL réelle que vous utiliserez dans votre application.
