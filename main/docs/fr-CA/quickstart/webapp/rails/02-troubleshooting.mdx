<div id="troubleshooting">
  # Dépannage
</div>

Voici quelques problèmes de dépannage que vous pourriez rencontrer lors de l’utilisation du Quickstart Rails.

<div id="using-a-reverse-proxy">
  ## Utilisation d’un proxy inverse
</div>

Le paramètre `redirect_uri` qu’OmniAuth génère lors de la redirection vers la page de connexion est basé sur l’en-tête `Host` transmis à Rails. Cela peut entraîner la transmission d’URL de rappel incorrectes lors de l’utilisation de cette stratégie (et d’OmniAuth en général) avec un proxy inverse. Vous pouvez ajuster l’hôte utilisé par OmniAuth à l’aide de l’extrait suivant :

```
OmniAuth.config.full_host = lambda do |env|
    scheme         = env['rack.url_scheme']
    local_host     = env['HTTP_HOST']
    forwarded_host = env['HTTP_X_FORWARDED_HOST']
    forwarded_host.blank? ? "#{scheme}://#{local_host}" : "#{scheme}://#{forwarded_host}"
end
```

[Consultez cette discussion sur Stack Overflow pour en savoir plus](https://stackoverflow.com/a/7135029/728480).

<div id="actioncontrollerinvalidauthenticitytoken">
  ## ActionController::InvalidAuthenticityToken
</div>

Cela est probablement dû à l’absence du jeton CSRF requis pour envoyer la requête de connexion par POST. Si vous examinez le bouton de connexion dans votre navigateur, vous devriez voir quelque chose comme ceci :

```erb lines
<a data-method="post" href="auth/auth0">Login</a>
```

... et dans l’élément `<head>` de la page, vous devriez trouver des balises méta CSRF comme celles-ci :

```erb lines
<meta name="csrf-param" content="authenticity_token">
<meta name="csrf-token" content="UY2XpKwxzwBWalxFVJ8yKsao/33it7If09BnZewpHifVPSpFJd2LrA7xgQn6VQrhZNGjgZoLI3kV+bkQHtr+Rw==">
```

Avec ces éléments en place, Rails convertira le lien de connexion pour envoyer le jeton CSRF au backend par POST afin de le vérifier avant de rediriger vers la page de connexion.

<div id="actiondispatchcookiescookieoverflow">
  ## ActionDispatch::Cookies::CookieOverflow
</div>

Cette erreur signifie qu’une session par cookie est utilisée et que, comme l’ensemble du profil est stocké, elle dépasse la taille maximale de 4 ko. Si vous n’arrivez pas à accéder au profil, ou si vous obtenez une erreur semblable à `NoMethodError`, `undefined method '[]' for nil:NilClass`, essayez d’utiliser le stockage en mémoire pendant le développement.

Allez à `/config/initializers/session_store.rb` (ou créez-le s’il n’existe pas) et ajoutez ce qui suit :

```
Rails.application.config.session_store :cache_store
```

Accédez à `/config/environments/development.rb` et ajoutez ce qui suit :

```
config.cache_store = :memory_store
```

Redémarrez votre serveur Rails pour que ces modifications soient prises en compte.

Il est recommandé d’utiliser un stockage en mémoire, comme MemCached, pour les applications en production.

<div id="ssl-issues">
  ## Problèmes SSL
</div>

Dans certaines configurations, Ruby pourrait ne pas être en mesure de trouver les certificats d’autorité de certification (certificats CA).

Téléchargez le paquet de certificats CA dans le répertoire du projet :

```bash lines
curl -L -o lib/ca-bundle.crt http://curl.haxx.se/ca/ca-bundle.crt
```

Ajoutez ce code d’initialisation à `config/initializers/fix_ssl.rb` :

```rb lines
# config/initializers/fix_ssl.rb

require 'open-uri'
require 'net/https'

module Net
  class HTTP
    alias_method :original_use_ssl=, :use_ssl=

    def use_ssl=(flag)
      path = ( Rails.env == "development") ? "lib/ca-bundle.crt" : "/usr/lib/ssl/certs/ca-certificates.crt"
      self.ca_file = Rails.root.join(path).to_s
      self.verify_mode = OpenSSL::SSL::VERIFY_PEER
      self.original_use_ssl = flag
    end
  end
end
```

<div id="failure-messageinvalid_credentials">
  ## &quot;message d’erreur=invalid_credentials&quot;
</div>

Ce problème ne survient pas lorsque vous travaillez en local, mais il peut se produire dans un environnement de préproduction ou de production. Le message d’erreur peut s’afficher comme suit :

```
omniauth: (auth0) Authentication failure! invalid_credentials: OAuth2::Error, server_error: The redirect URI is wrong. You send [wrong url], and we expected [callback url set in your app settings]
```

Pour régler ce problème, ajoutez ce qui suit à `config/environments/staging.rb` ou `production.rb` :

```
OmniAuth.config.full_host = "http://www.example.com"
```

Remplacez `http://www.example.com` par l’URL réelle que vous utiliserez dans votre application.
