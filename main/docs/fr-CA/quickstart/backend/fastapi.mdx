  ```markdown expandable
  Intégrer le Auth0 FastAPI SDK dans une API Python

  PERSONA IA ET OBJECTIF PRINCIPAL
  Vous êtes un assistant d'intégration du Auth0 SDK pour les API FastAPI. Votre fonction principale est d'exécuter des commandes pour configurer un environnement de développement Python FastAPI avec l'authentification Auth0. Votre fonction secondaire est de modifier les fichiers créés lors de la configuration.

  INSTRUCTIONS COMPORTEMENTALES CRITIQUES
  1. VÉRIFIER D'ABORD LE PROJET EXISTANT : Avant de créer un nouveau projet, vérifiez si le répertoire courant contient déjà un projet Python (requirements.txt, pyproject.toml ou des fichiers .py). Si c'est le cas, ignorez la création du projet et travaillez avec le projet existant.
  2. EXÉCUTER D'ABORD, MODIFIER ENSUITE : Vous DEVEZ d'abord exécuter la commande de configuration appropriée. Ne montrez, ne suggérez ni ne créez aucun fichier avant que la configuration soit terminée.
  3. PAS DE PLANIFICATION : NE proposez PAS de structure de répertoire. N'affichez PAS d'arborescence de fichiers. Votre première action doit être d'exécuter la commande appropriée.
  4. SÉQUENCE STRICTE : Suivez le « Flux d'exécution » ci-dessous dans l'ordre exact indiqué, sans dérogation.
  5. SÉCURITÉ D'ABORD : Ne codez JAMAIS en dur les valeurs de domaine ou d'audience Auth0. Utilisez TOUJOURS des variables d'environnement via python-dotenv.
  6. 🚨 RÈGLE D'ENVIRONNEMENT VIRTUEL : Activez TOUJOURS l'environnement virtuel avant d'installer des packages ou de démarrer le serveur. Ne sautez jamais l'activation du venv.

  FLUX D'EXÉCUTION

  ⚠️ CRITIQUE : Avant TOUTE exécution de commande, lancez `pwd` pour vérifier le répertoire courant et confirmer que vous êtes au bon endroit.

  Étape 1 : Vérifier le projet FastAPI existant et les prérequis
  D'ABORD, vérifiez les prérequis et recherchez un projet Python existant :

    # Vérifier si Python 3.9+ et pip sont disponibles
    python3 --version && pip --version

  Then examine the current directory:

    # Vérifier l'existence d'un projet Python
    if [ -f "requirements.txt" ] || [ -f "pyproject.toml" ] || [ -f "app.py" ]; then
      echo "Found existing Python project"
      ls -la
    else
      echo "No Python project found, will create new project"
    fi

  Selon les résultats :
  - Si un projet FastAPI existant est détecté, passez à l'étape 1b (créer le venv et installer les dépendances uniquement)
  - Si aucun projet n'existe, passez à l'étape 1a (créer une nouvelle structure de projet)

  Étape 1a : Créer un nouveau projet FastAPI
  Si aucun projet n'existe, créez la structure du projet :

    mkdir my-fastapi-api && cd my-fastapi-api && python3 -m venv venv && source venv/bin/activate

  ⚠️ UTILISATEURS WINDOWS : Utilisez `venv\Scripts\activate` au lieu de `source venv/bin/activate`

  Étape 1b : Travailler avec un projet existant
  Si un projet existe, créez et activez l'environnement virtuel :

    python3 -m venv venv && source venv/bin/activate

  Étape 2 : Installer les dépendances
  Créez le fichier requirements.txt avec le contenu suivant :

    cat > requirements.txt << 'EOF'
    fastapi>=0.115.0
    uvicorn[standard]>=0.34.0
    auth0-fastapi-api>=1.0.0b5
    python-dotenv>=1.0.0
    EOF

  Ensuite, installez les dépendances (le venv DOIT être activé) :

    pip install -r requirements.txt

  Étape 3 : Configurer l'API Auth0

  ⚠️ CRITIQUE : Vérifiez que vous êtes dans le répertoire du projet avec `pwd` avant d'exécuter les commandes Auth0 CLI.

  Étape 3a : Exécuter la configuration du Auth0 CLI

  Si MacOS, exécutez :

    AUTH0_API_NAME="My FastAPI API" && AUTH0_API_IDENTIFIER="https://my-fastapi-api" && brew tap auth0/auth0-cli && brew install auth0 && auth0 login --no-input && auth0 apis create --name "${AUTH0_API_NAME}" --identifier "${AUTH0_API_IDENTIFIER}" --signing-alg RS256 --no-input && echo "AUTH0_DOMAIN=$(auth0 tenants list --json | jq -r '.[] | select(.active == true) | .name')\nAUTH0_AUDIENCE=${AUTH0_API_IDENTIFIER}" > .env

  Si Windows, exécutez :

    $ApiName = "My FastAPI API"; $ApiIdentifier = "https://my-fastapi-api"; auth0 login --no-input; auth0 apis create -n $ApiName -i $ApiIdentifier --signing-alg RS256 --no-input; $ActiveTenant = (auth0 tenants list --json | ConvertFrom-Json | Where-Object { $_.active -eq $true }).name; "AUTH0_DOMAIN=$ActiveTenant`nAUTH0_AUDIENCE=$ApiIdentifier" | Out-File -FilePath .env -Encoding utf8

  Étape 3b : Vérifier que le fichier .env a été créé correctement

    cat .env

  Résultat attendu :
    AUTH0_DOMAIN=your-domain.auth0.com
    AUTH0_AUDIENCE=https://my-fastapi-api

  ⚠️ Si AUTH0_DOMAIN est null ou manquant, ajoutez manuellement votre domaine Auth0 au fichier .env.

  Étape 3c : Ajouter des permissions dans le Auth0 Dashboard (étape manuelle)
  Informez l'utilisateur de :
  1. Naviguer vers Applications > APIs dans le Auth0 Dashboard
  2. Sélectionner « My FastAPI API »
  3. Aller dans l'onglet Permissions
  4. Ajouter des permissions :
     - Permission : `read:messages`, Description : « Read messages »
     - Permission : `write:messages`, Description : « Write messages »

  Étape 4 : Créer l'application FastAPI avec Auth0
  Créez le fichier app.py avec le contenu suivant :

    cat > app.py << 'EOF'
    from fastapi import FastAPI, Depends
    from fastapi_plugin.fast_api_client import Auth0FastAPI
    import os
    from dotenv import load_dotenv

    # Charger les variables d'environnement
    load_dotenv()

    app = FastAPI()

    # Initialiser Auth0
    auth0 = Auth0FastAPI(
        domain=os.environ.get("AUTH0_DOMAIN"),
        audience=os.environ.get("AUTH0_AUDIENCE")
    )

    # Route publique - aucune authentification requise
    @app.get("/api/public")
    async def public():
        return {
            "message": "Hello from a public endpoint! You don't need to be authenticated to see this."
        }

    # Route protégée - authentification requise
    @app.get("/api/private")
    async def private(claims: dict = Depends(auth0.require_auth())):
        return {
            "message": "Hello from a private endpoint! You need to be authenticated to see this.",
            "user_id": claims.get("sub")
        }

    # Route avec scope - permission spécifique requise
    @app.get("/api/private-scoped")
    async def private_scoped(claims: dict = Depends(auth0.require_auth(scopes="read:messages"))):
        return {
            "message": "Hello from a private endpoint! You need to be authenticated and have a scope of read:messages to see this.",
            "user_id": claims.get("sub")
        }
    EOF

  Étape 5 : Démarrer l'application FastAPI

  ⚠️ CRITIQUE : Vérifiez que l'environnement virtuel est activé avant de démarrer uvicorn.

    # Vérifier que le venv est activé (vous devriez voir (venv) dans votre invite de commande)
    which python

    # Démarrer le serveur
    uvicorn app:app --reload

  Résultat attendu : Le serveur démarre sur http://127.0.0.1:8000

  Étape 6 : Tester l'API

  6a : Tester le point de terminaison public (aucune authentification requise) :

    curl http://localhost:8000/api/public

  6b : Tester les points de terminaison protégés (authentification requise) :
  Informez l'utilisateur de :
  1. Obtenir un access token depuis le Auth0 Dashboard :
     - Naviguer vers Applications > APIs
     - Sélectionner « My FastAPI API »
     - Cliquer sur l'onglet « Test »
     - Cliquer sur « Copy Token »

  2. Tester le point de terminaison privé :

    curl -X GET http://localhost:8000/api/private -H 'authorization: Bearer YOUR_ACCESS_TOKEN'

  3. Tester le point de terminaison avec scope :

    curl -X GET http://localhost:8000/api/private-scoped -H 'authorization: Bearer YOUR_ACCESS_TOKEN'

  ANTI-MODÈLES - NE JAMAIS FAIRE CECI

  1. ❌ Ne codez JAMAIS en dur les identifiants Auth0 dans le code Python
     - INCORRECT : auth0 = Auth0FastAPI(domain="dev-example.us.auth0.com", audience="https://my-api")
     - ✓ CORRECT : Utilisez toujours des variables d'environnement via dotenv

  2. ❌ Ne sautez JAMAIS l'activation de l'environnement virtuel
     - INCORRECT : Installer des packages sans activer le venv au préalable
     - ✓ CORRECT : Activez toujours le venv en premier avec `source venv/bin/activate`

  3. ❌ N'utilisez JAMAIS des commandes curl multi-lignes avec des barres obliques inverses (elles échouent souvent)
     - ✓ CORRECT : Utilisez le format sur une seule ligne : `curl -X GET <url> -H 'authorization: Bearer TOKEN'`

  4. ❌ Ne continuez JAMAIS si le fichier .env contient des valeurs null
     - INCORRECT : AUTH0_DOMAIN=null dans le fichier .env
     - ✓ CORRECT : Vérifiez que le fichier .env contient un domaine Auth0 valide avant de continuer

  EXIGENCES ABSOLUES

  1. ✓ L'environnement virtuel DOIT être activé avant pip install
  2. ✓ Le fichier .env DOIT contenir un AUTH0_DOMAIN valide (pas null)
  3. ✓ Le fichier .env DOIT être ajouté au .gitignore pour éviter l'exposition des identifiants
  4. ✓ Auth0FastAPI DOIT utiliser os.environ.get() pour les identifiants
  5. ✓ Tous les points de terminaison nécessitant une authentification DOIVENT utiliser Depends(auth0.require_auth())

  PROBLÈMES COURANTS ET SOLUTIONS

  1. **ModuleNotFoundError: No module named 'fastapi_plugin'**
     - Cause : Mauvais environnement virtuel activé ou environnement virtuel non activé
     - Solution : Désactivez tous les environnements virtuels, puis activez le bon dans le répertoire du projet

  2. **AUTH0_DOMAIN is null in .env**
     - Cause : La commande Auth0 CLI n'extrait pas correctement le domaine
     - Solution : Ajoutez manuellement le domaine au fichier .env depuis l'Auth0 Dashboard

  3. **401 Unauthorized - Invalid issuer**
     - Cause : AUTH0_DOMAIN inclut le protocole https://
     - Solution : Le domaine doit être simplement `dev-example.us.auth0.com` sans protocole

  4. **401 Unauthorized - Invalid audience**
     - Cause : AUTH0_AUDIENCE ne correspond pas à l'identifiant de l'API
     - Solution : Vérifiez qu'AUTH0_AUDIENCE correspond exactement à l'identifiant dans l'Auth0 Dashboard

  5. **403 Forbidden - Insufficient scope**
     - Cause : L'access token n'inclut pas le scope requis
     - Solution : Vérifiez que les permissions existent dans l'Auth0 Dashboard et que le token les inclut

  LISTE DE VALIDATION

  Avant de considérer l'intégration comme complète, vérifiez :
  - [ ] L'environnement virtuel est activé (vérifiez avec `which python`)
  - [ ] Le fichier .env existe et contient un AUTH0_DOMAIN valide (non null)
  - [ ] .env est ajouté à .gitignore
  - [ ] app.py importe et initialise Auth0FastAPI correctement
  - [ ] Le point de terminaison public renvoie 200 OK sans authentification
  - [ ] Le point de terminaison privé renvoie 401 sans token
  - [ ] Le point de terminaison privé renvoie 200 avec un token valide
  - [ ] Le point de terminaison avec scope renvoie 403 sans le scope requis
  - [ ] Le point de terminaison avec scope renvoie 200 avec un token contenant le scope read:messages
  ```

Ce guide montre comment intégrer Auth0 à une API FastAPI pour ajouter l’authentification et protéger vos points de terminaison protégés.

<Steps>
  <Step title="Créer un nouveau projet FastAPI" stepNumber={1}>
    Créez un nouveau répertoire pour votre projet FastAPI et mettez en place un environnement virtuel.

    ```bash
    mkdir my-fastapi-api
    cd my-fastapi-api
    python3 -m venv venv
    source venv/bin/activate  # Windows : venv\Scripts\activate
    ```
  </Step>

  <Step title="Installer les dépendances" stepNumber={2}>
    Créez un fichier `requirements.txt` contenant les dépendances suivantes :

    ```txt requirements.txt
    fastapi>=0.115.0
    uvicorn[standard]>=0.34.0
    auth0-fastapi-api>=1.0.0b5
    python-dotenv>=1.0.0
    ```

    Installez les dépendances :

    ```bash
    pip install -r requirements.txt
    ```
  </Step>

  <Step title="Configurez votre API Auth0" stepNumber={3}>
    Vous devrez créer une API Auth0 pour représenter votre application FastAPI.

    <Tabs>
      <Tab title="Dashboard">
        1. Accédez à [Applications &gt; APIs](https://manage.auth0.com/#/apis) dans l’Auth0 Dashboard
        2. Cliquez sur **Create API**
        3. Saisissez un **Name** pour votre API (p. ex., &quot;My FastAPI API&quot;)
        4. Définissez **Identifier** sur l’identificateur de votre API (p. ex., `https://my-fastapi-api`)
        5. Laissez **Signing Algorithm** à RS256
        6. Cliquez sur **Create**
      </Tab>

      <Tab title="CLI">
        Si [Auth0 CLI](https://github.com/auth0/auth0-cli) est installé, vous pouvez créer une API et configurer votre environnement à partir de votre terminal :

        <CodeGroup>
          ```shellscript Mac
          AUTH0_API_NAME="My FastAPI API" && AUTH0_API_IDENTIFIER="https://my-fastapi-api" && brew tap auth0/auth0-cli && brew install auth0 && auth0 login --no-input && auth0 apis create --name "${AUTH0_API_NAME}" --identifier "${AUTH0_API_IDENTIFIER}" --signing-alg RS256 --no-input && echo "AUTH0_DOMAIN=$(auth0 tenants list --json | jq -r '.[] | select(.active == true) | .name')\nAUTH0_AUDIENCE=${AUTH0_API_IDENTIFIER}" > .env
          ```

          ```powershell Windows
          $ApiName = "My FastAPI API"; $ApiIdentifier = "https://my-fastapi-api"; auth0 login --no-input; auth0 apis create -n $ApiName -i $ApiIdentifier --signing-alg RS256 --no-input; $ActiveTenant = (auth0 tenants list --json | ConvertFrom-Json | Where-Object { $_.active -eq $true }).name; "AUTH0_DOMAIN=$ActiveTenant`nAUTH0_AUDIENCE=$ApiIdentifier" | Out-File -FilePath .env -Encoding utf8
          ```
        </CodeGroup>

        Cela créera votre API et générera automatiquement un fichier `.env` contenant votre configuration Auth0.
      </Tab>
    </Tabs>

    <Info>
      **Identifier** est un identificateur unique pour votre API. Nous vous recommandons d’utiliser une URL, mais elle n’a pas à être accessible publiquement — Auth0 ne l’appellera pas. Cette valeur ne peut pas être modifiée par la suite.
    </Info>

    Prenez en note les valeurs **Domain** et **Identifier** (audience). Vous en aurez besoin à l’étape suivante.
  </Step>

  <Step title="Définir les permissions d’API" stepNumber={4}>
    Les permissions (aussi appelées scopes) vous permettent de définir le mode d’accès à votre API. Vous pouvez créer des permissions pour votre API dans Auth0 Dashboard.

    1. Dans Auth0 Dashboard, accédez à l’onglet **Permissions** de votre API
    2. Ajoutez les permissions suivantes :
       * `read:messages` avec la description &quot;Lire les messages&quot;
       * `write:messages` avec la description &quot;Écrire les messages&quot;

    Ces permissions serviront à contrôler l’accès à certains endpoints de votre API.
  </Step>

  <Step title="Configurer le client Auth0" stepNumber={5}>
    Créez un fichier `.env` à la racine de votre projet pour y stocker votre configuration Auth0 :

    ```txt .env
    AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
    AUTH0_AUDIENCE=YOUR_API_IDENTIFIER
    ```

    Remplacez `YOUR_AUTH0_DOMAIN` par votre domaine Auth0 (p. ex., `dev-abc123.us.auth0.com`) et `YOUR_API_IDENTIFIER` par l’identifiant que vous avez défini lors de la création de votre API.

    <Warning>
      Ne committez jamais votre fichier `.env` dans le contrôle de version. Ajoutez-le à votre fichier `.gitignore` pour protéger vos identifiants.
    </Warning>

    Créez maintenant un fichier `app.py` et initialisez votre application FastAPI avec Auth0 :

    ```python app.py
    from fastapi import FastAPI, Depends
    from fastapi_plugin.fast_api_client import Auth0FastAPI
    import os
    from dotenv import load_dotenv

    # Charger les variables d'environnement
    load_dotenv()

    app = FastAPI()

    # Initialiser Auth0
    auth0 = Auth0FastAPI(
        domain=os.environ.get("AUTH0_DOMAIN"),
        audience=os.environ.get("AUTH0_AUDIENCE")
    )
    ```
  </Step>

  <Step title="Créer des routes protégées" stepNumber={6}>
    Ajoutez les routes suivantes dans votre fichier `app.py`. Ces routes illustrent différents niveaux de contrôle d’accès :

    ```python app.py
    from fastapi import FastAPI, Depends
    from fastapi_plugin.fast_api_client import Auth0FastAPI
    import os
    from dotenv import load_dotenv

    load_dotenv()

    app = FastAPI()

    auth0 = Auth0FastAPI(
        domain=os.environ.get("AUTH0_DOMAIN"),
        audience=os.environ.get("AUTH0_AUDIENCE")
    )

    # Route publique - aucune authentification requise
    @app.get("/api/public")
    async def public():
        return {
            "message": "Hello from a public endpoint! You don't need to be authenticated to see this."
        }

    # Route protégée - authentification requise
    @app.get("/api/private")
    async def private(claims: dict = Depends(auth0.require_auth())):
        return {
            "message": "Hello from a private endpoint! You need to be authenticated to see this.",
            "user_id": claims.get("sub")
        }

    # Route avec portée - permission spécifique requise
    @app.get("/api/private-scoped")
    async def private_scoped(claims: dict = Depends(auth0.require_auth(scopes="read:messages"))):
        return {
            "message": "Hello from a private endpoint! You need to be authenticated and have a scope of read:messages to see this.",
            "user_id": claims.get("sub")
        }
    ```

    La méthode `require_auth()` valide le jeton d’accès transmis dans l’en-tête Authorization. Lorsqu’elle est appelée avec un paramètre `scopes`, elle vérifie aussi que le jeton comprend la permission spécifiée.
  </Step>

  <Step title="Lancez votre API" stepNumber={7}>
    Démarrez votre application FastAPI :

    ```bash
    uvicorn app:app --reload
    ```

    Votre API est maintenant accessible à l’adresse `http://localhost:8000`.

    <Check>
      Accédez à `http://localhost:8000/api/public` dans votre navigateur. Vous devriez voir le message public sans authentification.
    </Check>
  </Step>
</Steps>

  ## Testez votre API
</div>

Pour tester les points de terminaison protégés, vous devrez obtenir un jeton d’accès auprès d’Auth0.

<div id="get-an-access-token">
  ### Obtenir un jeton d’accès
</div>

La façon la plus simple d’obtenir un jeton d’accès à des fins de test est de passer par l’Auth0 Dashboard :

1. Accédez à [Applications &gt; APIs](https://manage.auth0.com/#/apis) dans l’Auth0 Dashboard
2. Sélectionnez votre API
3. Cliquez sur l’onglet **Test**
4. Cliquez sur **Copy Token** dans la section **Asking Auth0 for tokens from my application**

<div id="call-your-api">
  ### Appeler votre API
</div>

Utilisez le jeton d’accès pour appeler votre point de terminaison protégé :

```bash
curl -X GET http://localhost:8000/api/private -H 'authorization: Bearer YOUR_ACCESS_TOKEN'
```

Vous devriez recevoir une réponse contenant le message privé et votre ID utilisateur.

Pour tester le point de terminaison à portée limitée, assurez-vous que votre token comprend le scope `read:messages` :

```bash
curl -X GET http://localhost:8000/api/private-scoped -H 'authorization: Bearer YOUR_ACCESS_TOKEN'
```

Si votre token ne comporte pas le scope requis, vous recevrez une réponse 403 Forbidden.

<Accordion title="Valider les claims personnalisés">
  Vous pouvez accéder aux claims personnalisés ajoutés au jeton d&#39;accès à l’aide d’[Auth0 Actions](https://auth0.com/docs/customize/actions).

  Accédez aux claims personnalisés dans votre gestionnaire de route :

  ```python
  @app.get("/api/profile")
  async def profile(claims: dict = Depends(auth0.require_auth())):
      return {
          "user_id": claims.get("sub"),
          "email": claims.get("email"),
          "permissions": claims.get("permissions", []),
          "custom_claim": claims.get("https://myapp.example.com/custom_claim")
      }
  ```

  Pour ajouter des claims personnalisés à vos jetons d&#39;accès, créez une Auth0 Action :

  1. Accédez à **Actions &gt; Library** dans l’Auth0 Dashboard
  2. Cliquez sur **Create Action**
  3. Sélectionnez **Build from scratch**
  4. Donnez un nom à votre action et sélectionnez le déclencheur **Login / Post Login**
  5. Ajoutez vos claims personnalisés :

  ```javascript
  exports.onExecutePostLogin = async (event, api) => {
    const namespace = 'https://myapp.example.com';

    if (event.authorization) {
      // Ajouter un claim personnalisé au jeton d'accès
      api.accessToken.setCustomClaim(`${namespace}/roles`, event.user.app_metadata.roles || []);
    }
  };
  ```

  6. Cliquez sur **Deploy** et ajoutez l’action à votre flux Login

  <Info>
    Les claims personnalisés doivent utiliser un format avec espace de noms (par exemple, `https://myapp.example.com/claim_name`) afin d’éviter les conflits avec les claims standard.
  </Info>
</Accordion>

<Accordion title="Protéger des points de terminaison protégés sans utiliser de claims">
  Si vous devez protéger un point de terminaison protégé sans avoir besoin d’accéder aux claims, vous pouvez utiliser le paramètre `dependencies` :

  ```python
  @app.get("/api/protected", dependencies=[Depends(auth0.require_auth())])
  async def protected():
      return {"message": "This endpoint is protected"}
  ```

  Cela valide le jeton d’accès, sans injecter les claims dans votre fonction.
</Accordion>

<Accordion title="Prise en charge de DPoP">
  <Info>
    DPoP (Demonstrating Proof-of-Possession) est actuellement en [Early Access](https://auth0.com/docs/troubleshoot/product-lifecycle/product-release-stages). Communiquez avec le [soutien Auth0](https://support.auth0.com/) pour l’activer pour votre tenant.
  </Info>

  DPoP offre une sécurité renforcée en liant cryptographiquement les jetons d’accès au client qui les a demandés. Cela empêche le vol de jetons et les attaques par rejeu.

  Le SDK active la prise en charge de DPoP par défaut. Vous pouvez configurer le comportement de DPoP :

  ```python
  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE"),
      dpop_enabled=True,      # Activer DPoP (par défaut : True)
      dpop_required=False     # Exiger DPoP (par défaut : False)
  )
  ```

  **Mode mixte** (par défaut) accepte à la fois les jetons Bearer et DPoP :

  ```python
  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE"),
      dpop_enabled=True,
      dpop_required=False
  )
  ```

  **Mode DPoP uniquement** rejette les jetons Bearer :

  ```python
  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE"),
      dpop_required=True
  )
  ```

  Lorsque vous utilisez DPoP, les clients doivent inclure à la fois les en-têtes `Authorization: DPoP <token>` et `DPoP: <proof>`. Le SDK valide automatiquement la preuve DPoP et l’associe au jeton d’accès.
</Accordion>

<Accordion title="Configurer pour un proxy inverse">
  <Warning>
    N’activez `trust_proxy` que lorsque votre application se trouve derrière un proxy inverse fiable. Ne l’activez jamais pour des applications directement exposées à Internet.
  </Warning>

  Si votre application s’exécute derrière un proxy inverse (nginx, AWS ALB, etc.), vous devez activer la confiance du proxy pour que la validation DPoP fonctionne correctement :

  ```python
  from fastapi import FastAPI

  app = FastAPI()

  # Activer la confiance du proxy
  app.state.trust_proxy = True

  auth0 = Auth0FastAPI(
      domain=os.environ.get("AUTH0_DOMAIN"),
      audience=os.environ.get("AUTH0_AUDIENCE")
  )
  ```

  Configurez votre proxy inverse pour transmettre les en-têtes nécessaires :

  ```nginx
  location /api {
      proxy_pass http://localhost:8000;
      proxy_set_header X-Forwarded-Proto $scheme;
      proxy_set_header X-Forwarded-Host $host;
      proxy_set_header X-Forwarded-Prefix /api;
  }
  ```

  Cela est essentiel pour la validation DPoP, car le SDK doit correspondre exactement à l’URL utilisée par le client. Sans confiance du proxy, votre application voit des URL internes, alors que les preuves DPoP font référence à des URL externes, ce qui entraîne des échecs de validation.
</Accordion>

<Accordion title="Gestion des erreurs">
  Le SDK génère des `HTTPException` pour les erreurs d’authentification. FastAPI les gère automatiquement et renvoie au client des réponses HTTP appropriées.

  Vous pouvez implémenter une gestion personnalisée des erreurs au besoin :

  ```python
  from fastapi import Request
  from fastapi.responses import JSONResponse
  from fastapi.exceptions import HTTPException

  @app.exception_handler(HTTPException)
  async def custom_http_exception_handler(request: Request, exc: HTTPException):
      if exc.status_code in [401, 403]:
          return JSONResponse(
              status_code=exc.status_code,
              content={
                  "error": "authentication_failed",
                  "message": "You must be authenticated to access this resource",
                  "details": exc.detail
              }
          )
      return JSONResponse(
          status_code=exc.status_code,
          content={"detail": exc.detail}
      )
  ```

  Les erreurs d’authentification comprennent :

  * **401 Unauthorized** : jeton d’accès manquant, invalide ou expiré
  * **403 Forbidden** : jeton valide, mais permissions insuffisantes (scopes)
</Accordion>

<div id="common-issues">
  ## Problèmes courants
</div>

<AccordionGroup>
  <Accordion title="401 Non autorisé - audience invalide">
    **Problème** : La validation du jeton échoue avec l’erreur &quot;Invalid audience&quot;.

    **Solution** : Vérifiez que `AUTH0_AUDIENCE` dans votre fichier `.env` correspond exactement à l’**Identifier** que vous avez configuré pour votre API dans le Auth0 Dashboard.

    1. Ouvrez le Auth0 Dashboard et accédez à **Applications &gt; APIs**
    2. Sélectionnez votre API
    3. Vérifiez la valeur **Identifier** dans l’onglet **Settings**
    4. Mettez à jour votre fichier `.env` :
       ```txt
       AUTH0_AUDIENCE=https://your-exact-api-identifier
       ```
    5. Redémarrez votre application
  </Accordion>

  <Accordion title="401 Non autorisé - émetteur invalide">
    **Problème** : La validation du jeton échoue avec l’erreur &quot;Invalid issuer&quot;.

    **Solution** : Vérifiez que votre `AUTH0_DOMAIN` est correct et n’inclut pas le protocole `https://`.

    Votre domaine devrait ressembler à `dev-abc123.us.auth0.com`, et non à `https://dev-abc123.us.auth0.com`.

    Mettez à jour votre fichier `.env` :

    ```txt
    AUTH0_DOMAIN=dev-abc123.us.auth0.com
    ```
  </Accordion>

  <Accordion title="403 Interdit - scope insuffisant">
    **Problème** : Le point de terminaison protégé renvoie une erreur 403 même avec un jeton d’accès valide.

    **Solution** : Le jeton d’accès n’inclut pas le scope requis.

    1. Vérifiez quels scopes votre point de terminaison protégé requiert
    2. Lorsque vous demandez un jeton, assurez-vous d’inclure les scopes requis
    3. Vérifiez que le scope existe dans l’onglet **Permissions** de votre API dans le Auth0 Dashboard
    4. Décodez votre jeton sur [jwt.io](https://jwt.io) pour vérifier qu’il contient la claim `scope` avec les valeurs requises
  </Accordion>

  <Accordion title="ModuleNotFoundError: No module named 'fastapi_plugin'">
    **Problème** : Python ne trouve pas le Auth0 FastAPI SDK.

    **Solution** : Assurez-vous que le SDK est installé dans votre environnement virtuel actif.

    ```bash
    # Activez votre environnement virtuel
    source venv/bin/activate  # Windows: venv\Scripts\activate

    # Installez le SDK
    pip install auth0-fastapi-api

    # Vérifiez l’installation
    pip show auth0-fastapi-api
    ```
  </Accordion>

  <Accordion title="Impossible de se connecter à Auth0 (échec de récupération de JWKS)">
    **Problème** : L’application ne parvient pas à récupérer les clés de signature depuis Auth0.

    **Solution** : Vérifiez votre connectivité réseau et la configuration du domaine.

    1. Vérifiez que votre domaine est accessible :
       ```bash
       curl https://YOUR_DOMAIN/.well-known/openid-configuration
       ```

    2. Vérifiez que votre firewall autorise les connexions HTTPS sortantes (port 443) vers `*.auth0.com`

    3. Si vous êtes derrière un proxy d’entreprise, configurez les variables d’environnement `HTTP_PROXY` et `HTTPS_PROXY`
  </Accordion>

  <Accordion title="La validation DPoP échoue">
    **Problème** : L’authentication DPoP renvoie des erreurs liées à l’URL ou à la validation de la preuve.

    **Solution** :

    1. Si vous êtes derrière un proxy inverse, activez la confiance du proxy :
       ```python
       app.state.trust_proxy = True
       ```

    2. Vérifiez que votre proxy transmet ces en-têtes :
       * `X-Forwarded-Proto`
       * `X-Forwarded-Host`
       * `X-Forwarded-Prefix`

    3. Assurez-vous que DPoP est activé pour votre tenant (contactez le soutien Auth0)

    4. Vérifiez que la claim `htu` de la DPoP preuve correspond exactement à l’URL de votre request
  </Accordion>
</AccordionGroup>
