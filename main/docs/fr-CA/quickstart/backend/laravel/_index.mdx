    [Auth0&#39;s Laravel SDK](https://github.com/auth0/laravel-auth0)
    vous permet d&#39;ajouter rapidement une autorisation basée sur les jetons et un contrôle d&#39;accès aux routes de votre application Laravel. Ce guide
    montre comment intégrer Auth0 à une application [Laravel 9
    ou 10](https://github.com/auth0/laravel-auth0#support-policy) nouvelle ou existante.

    **Les applications backend se distinguent des applications web traditionnelles en ce qu&#39;elles ne gèrent pas l&#39;authentification des utilisateurs et ne disposent pas d&#39;interface utilisateur. Elles fournissent une API avec laquelle d&#39;autres applications peuvent interagir. Elles acceptent des **[jetons d&#39;accès](https://auth0.com/docs/secure/tokens/access-tokens)** provenant des en-têtes **`Authorization`** dans les requêtes afin de contrôler l&#39;accès aux routes.**

    Des applications frontales distinctes sont généralement conçues pour interagir avec ces types de backends. Il peut s&#39;agir d&#39;[applications monopage](https://auth0.com/docs/quickstart/spa) ou d&#39;[applications natives ou mobiles](https://auth0.com/docs/quickstart/native) (pour lesquelles Auth0 fournit également des SDK !)

    Lorsque les utilisateurs doivent interagir avec votre application back-end, ils s&#39;authentifient d&#39;abord auprès d&#39;Auth0 à l&#39;aide de l&#39;application front-end. L&#39;application front-end obtient ensuite un jeton d&#39;accès auprès d&#39;Auth0, qu&#39;elle peut utiliser pour envoyer des requêtes à votre application back-end au nom de l&#39;utilisateur.

    Comme leur nom l&#39;indique, les [jetons
    d&#39;accès](https://auth0.com/docs/secure/tokens/access-tokens) sont conçus pour gérer les questions de contrôle d&#39;accès (autorisation) et ne contiennent aucune information sur
    l&#39;utilisateur. **Les applications backend fonctionnent exclusivement avec des jetons d&#39;accès.** Vous pouvez récupérer des informations sur
    l&#39;utilisateur qui a créé le jeton à l&#39;aide de l&#39;[API Management
    ](https://auth0.com/docs/api/management/v2), ce que nous verrons plus loin.

      **Si vous n’avez pas encore configuré d’application Laravel**, ouvrez un shell dans le répertoire où vous souhaitez créer le nouveau
      projet et exécutez la commande suivante :

      ```bash lines wrap
      composer create-project --prefer-dist laravel/laravel auth0-laravel-api ^9.0
      ```

      Toutes les commandes de ce guide supposent que vous les exécutez depuis le répertoire racine de votre projet Laravel, donc
      vous devez faire `cd` dans le répertoire du nouveau projet :

      ```bash lines
      cd auth0-laravel-api
      ```

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Exécutez la commande suivante dans le répertoire de votre projet pour installer le [Auth0 Laravel SDK](https://github.com/auth0/laravel-auth0) :

      ```bash lines
      composer require auth0/login:^7.8 --update-with-all-dependencies
      ```

      Ensuite, générez un fichier de configuration du SDK pour votre application :

      ```bash lines
      php artisan vendor:publish --tag auth0
      ```

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Exécutez la commande suivante dans le répertoire de votre projet pour télécharger l’[Auth0 CLI](https://github.com/auth0/auth0-cli) :

      ```bash lines wrap
      curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
      ```

      Ensuite, authentifiez le CLI avec votre compte Auth0 en choisissant &quot;en tant qu’utilisateur&quot; lorsqu’on vous le demande :

      ```bash lines
      ./auth0 login
      ```

      Ensuite, créez une nouvelle application avec Auth0 :

      ```bash lines
      ./auth0 apps create \
        --name "My Laravel Backend" \
        --type "regular" \
        --auth-method "post" \
        --callbacks "http://localhost:8000/callback" \
        --logout-urls "http://localhost:8000" \
        --reveal-secrets \
        --no-input \
        --json > .auth0.app.json
      ```

      Vous devriez également créer une nouvelle API :

      ```bash lines
      ./auth0 apis create \
        --name "My Laravel Backend API" \
        --identifier "https://github.com/auth0/laravel-auth0" \
        --offline-access \
        --no-input \
        --json > .auth0.api.json
      ```

      Cela crée deux fichiers dans le répertoire de votre projet qui configurent le SDK.

      Comme ces fichiers contiennent des identifiants, il est important de les traiter comme des données sensibles. Vous devez vous assurer de ne pas
      les enregistrer dans le contrôle de version. Si vous utilisez Git, vous devriez les ajouter à votre fichier `.gitignore` :

      ```bash lines
      echo ".auth0.*.json" >> .gitignore
      ```

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4" isSingleColumn>
      Le SDK enregistre automatiquement son mécanisme d’autorisation auprès de votre application Laravel pour l’utiliser avec le
      middleware `api`, que Laravel applique par défaut à toutes les routes du fichier
      `routes/api.php` de votre application.

      <Warning>
        Pour que le SDK fonctionne comme prévu sans configuration supplémentaire, **vous devriez définir vos routes dans
        le **fichier `routes/api.php`**.**
      </Warning>

      Vous pouvez utiliser le mécanisme d’autorisation du SDK Auth0 pour restreindre l’accès aux routes de votre application.

      Pour rejeter les requêtes qui ne contiennent pas de jeton d’accès valide dans l’en-tête `Authorization`, vous pouvez utiliser
      le middleware `auth` de Laravel :

      ```php lines
      Route::get('/private', function () {
      return response()->json([
      'message' => 'Your token is valid; you are authorized.',
      ]);
      })->middleware('auth');
      ```

      Vous pouvez aussi exiger que le token fourni ait des [permissions](https://auth0.com/docs/manage-users/access-control/rbac) précises en combinant le tout
      avec le middleware `can` de Laravel :

      ```php lines
      Route::get('/scope', function () {
      return response()->json([
      'message' => 'Your token is valid and has the read:messages permission; you are authorized.',
      ]);
      })->middleware('auth')->can('read:messages');
      ```

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5" isSingleColumn>
      Des renseignements sur le jeton d’accès fourni sont accessibles au moyen de la façade `Auth` de Laravel ou de la
      fonction utilitaire `auth()`.

      Par exemple, pour récupérer l’identifiant de l’utilisateur et son adresse courriel :

      ```php lines
      Route::get('/', function () {
      if (! auth()->check()) {
      return response()->json([
      'message' => 'You did not provide a valid token.',
      ]);
      }
      return response()->json([
      'message' => 'Your token is valid; you are authorized.',
      'id' => auth()->id(),
      'token' => auth()?->user()?->getAttributes(),
      ]);
      });
      ```

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6" isSingleColumn>
      Vous pouvez récupérer des renseignements sur l’utilisateur qui a créé le jeton d’accès depuis Auth0 à l’aide de l’[Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). Le SDK fournit un wrapper pratique pour cette API,
      accessible au moyen de la méthode `management()` du SDK.

      **Avant d’effectuer des appels à la Management API, vous devez autoriser votre application à communiquer avec la Management
      API.** Pour ce faire, accédez à la [page API de l’Auth0 Dashboard](https://manage.auth0.com/#/apis/), choisissez `Auth0 Management API`, puis
      sélectionnez l’onglet « Machine to Machine Applications ». Autorisez votre application Laravel, puis cliquez sur la
      flèche vers le bas pour choisir les scopes que vous souhaitez accorder.

      Pour l’exemple suivant, vous devez accorder le scope `read:users`. La liste des endpoints d’API et des
      scopes requis se trouve dans [la documentation de la Management
      API](https://auth0.com/docs/api/management/v2).

      ```php lines
      use Auth0\Laravel\Facade\Auth0;
      Route::get('/me', function () {
      $user = auth()->id();
      $profile = cache()->get($user);
      if (null === $profile) {
      $endpoint = Auth0::management()->users();
      $profile = $endpoint->get($user);
      $profile = Auth0::json($profile);
      cache()->put($user, $profile, 120);
      }
      $name = $profile['name'] ?? 'Unknown';
      $email = $profile['email'] ?? 'Unknown';
      return response()->json([
      'name' => $name,
      'email' => $email,
      ]);
      })->middleware('auth');
      ```

      <Info>
        **Vous devriez mettre en cache les renseignements de l’utilisateur dans votre application pendant de courtes périodes.** Cela réduit le nombre
        de requêtes envoyées par votre application à Auth0 et améliore les performances. Évitez de conserver les renseignements de l’utilisateur
        dans votre application pendant de longues périodes, car cela peut entraîner l’utilisation de données périmées. Évitez aussi de
        conserver dans des bases de données persistantes des renseignements sur l’utilisateur autres que son identifiant.
      </Info>

        ##### Checkpoint

        Ouvrez un shell et essayez d&#39;envoyer des requêtes à votre application.

        Commencez par appeler la route publique :

        `curl --request GET \ --url http://localhost:8000/api \ --header 'Accept: application/json'`

        Ensuite, utilisez votre jeton d&#39;accès dans un en-tête `Authorization` pour appeler une route protégée :

        `curl --request GET \ --url http://localhost:8000/api/private \ --header 'Accept: application/json' \ --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'`

        `read:messages` a été accordé au jeton d&#39;accès :
