    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 fournit un [SDK
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) pour simplifier la mise en œuvre de l’authentification et de l’autorisation Auth0 dans les applications PHP.

      Le SDK PHP Auth0 exige l’installation de [PSR-17](https://www.php-fig.org/psr/psr-17/) et de [PSR-18](https://www.php-fig.org/psr/psr-18/), des bibliothèques HTTP compatibles pour gérer les requêtes réseau. Si vous
      n’avez pas de bibliothèques à votre disposition, vous pouvez installer des options fiables en exécutant les commandes suivantes dans votre
      terminal :

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Maintenant, installez l’Auth0 PHP SDK en exécutant la commande suivante dans votre terminal :

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configurer le SDK Auth0

      Pour que le SDK fonctionne correctement, vous devez définir les propriétés suivantes dans le SDK Auth0 pendant l’initialisation :

      * `strategy` : La stratégie aide à orienter le comportement du SDK selon le cas d’utilisation de votre application. Dans ce
        cas, vous devez la définir à la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain : Le domaine de votre tenant Auth0. En général, vous le trouverez dans l’Auth0 Dashboard, sous les
        Settings de l’Application, dans le champ Domain. Si vous utilisez un domaine personnalisé, définissez plutôt cette valeur sur celle de votre domaine personnalisé.
      * clientId : L’ID de l’application Auth0 que vous avez configurée plus tôt dans ce Quickstart. Vous pouvez le trouver dans l’Auth0 Dashboard,
        sous les Settings de votre Application, dans le champ Client ID.
      * clientSecret : Le secret de l’application Auth0 que vous avez créée plus tôt dans ce Quickstart. Le secret client se trouve dans l’Auth0 Dashboard,
        sous les Settings de votre Application, dans le champ Client Secret.
      * audience : L’identifiant de l’API Auth0 que vous avez enregistrée ci-dessus. Cette valeur doit être fournie sous forme de tableau.

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Ensuite, développez votre application pour récupérer et traiter les bearer tokens. Les bearer tokens sont des jetons d’accès transmis à
      votre API dans des requêtes envoyées par des clients au nom d’utilisateurs. Les jetons d’accès accordent ou refusent l’accès aux routes de votre
      application. C&#39;est ce qu&#39;on appelle l&#39;autorisation des points de terminaison.

      La façon la plus simple de récupérer des jetons d’accès dans une requête consiste à utiliser la méthode `getBearerToken()`
      du SDK PHP. Cette méthode extrait les jetons à partir des paramètres GET, des corps POST, des en-têtes de requête et d&#39;autres sources. Dans ce
      cas, le SDK PHP traite les jetons transmis dans des requêtes GET au moyen du paramètre `token` ou de l&#39;en-tête HTTP
      `Authorization`.

      Maintenant que vous avez configuré votre application Auth0, le Auth0 PHP SDK et que votre application récupère des jetons Bearer
      à partir des requêtes, la prochaine étape consiste à configurer l’autorisation des points de terminaison de votre projet. La méthode
      `getBearerToken()` que vous avez implémentée ci-dessus retourne une classe `Token` qui comprend
      des détails sur l’accès associé à la requête.

      Comme la méthode `getBearerToken()` valide et vérifie automatiquement la requête entrante, votre
      application détermine les détails du jeton d’accès en évaluant la réponse de la méthode. Lorsque la réponse est
      nulle, aucun jeton valide n’a été fourni. Sinon, examinez le contenu de la réponse pour en savoir plus sur la
      requête.

      Dans le panneau interactif à droite, vous pouvez voir une vérification qui détermine si la réponse est nulle ou non afin de filtrer l’accès à
      votre route `/api/private`.
