      Auth0 offre un [SDK
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) pour simplifier l’intégration de l’authentification et de l’autorisation Auth0 dans les applications PHP.

      Le SDK PHP d’Auth0 nécessite l’installation de bibliothèques HTTP compatibles [PSR-17](https://www.php-fig.org/psr/psr-17/) et [PSR-18](https://www.php-fig.org/psr/psr-18/) pour gérer les requêtes réseau. Si vous
      n’avez pas de bibliothèques à votre disposition, vous pouvez installer des options fiables en exécutant les commandes suivantes dans votre
      terminal :

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ensuite, installez l’Auth0 PHP SDK en exécutant la commande suivante dans votre terminal :

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configurer le SDK Auth0

      Pour que le SDK fonctionne correctement, vous devez définir les propriétés suivantes dans le SDK Auth0 lors de l’initialisation :

      * `strategy` : la stratégie oriente le comportement du SDK selon le cas d’utilisation de votre application. Dans ce
        cas, vous devez lui attribuer la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain : le domaine de votre tenant Auth0. En général, vous le trouverez dans l’Auth0 Dashboard, sous les
        Settings de votre application, dans le champ Domain. Si vous utilisez un custom domain, définissez plutôt cette valeur sur celle de votre custom domain.
      * clientId : l’ID de l’Application Auth0 que vous avez configurée plus tôt dans ce Quickstart. Vous pouvez le trouver dans l’Auth0 Dashboard,
        sous les Settings de votre application, dans le champ Client ID.
      * clientSecret : le secret de l’application Auth0 que vous avez créée plus tôt dans ce Quickstart. Le Client Secret se trouve dans l’Auth0 Dashboard,
        sous les Settings de votre application, dans le champ Client Secret.
      * audience : l’identifiant de l’Auth0 API que vous avez enregistrée ci-dessus. Cette valeur doit être fournie sous forme de tableau.

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Ensuite, ajoutez à votre application la capacité de récupérer et de traiter les bearer tokens. Les bearer tokens sont des jetons d’accès envoyés à
      votre API dans des requêtes de clients au nom d’utilisateurs. Les jetons d’accès accordent ou rejettent l’accès aux routes de votre
      application. C’est ce qu’on appelle l’autorisation des points de terminaison.

      La façon la plus simple de récupérer des jetons d’accès à partir d’une requête est d’utiliser la méthode `getBearerToken()`
      du SDK PHP. Cette méthode récupère les jetons à partir des paramètres GET, des corps de requête POST, des en-têtes de requête et d’autres sources. Dans ce
      cas, le SDK PHP traite les jetons transmis dans le paramètre `token` des requêtes GET ou dans l’en-tête HTTP
      `Authorization`.

      Maintenant que vous avez configuré votre application Auth0, le SDK Auth0 PHP et que votre application récupère des
      jetons Bearer à partir des requêtes, l’étape suivante consiste à configurer l’autorisation de l’endpoint pour votre projet. La méthode
      `getBearerToken()` que vous avez implémentée ci-dessus renvoie une classe `Token` qui contient des
      détails sur l’accès de la requête.

      Comme la méthode `getBearerToken()` valide et vérifie automatiquement la requête entrante, votre
      application détermine les détails du jeton d’accès en évaluant la réponse de la méthode. Lorsque la réponse est
      null, aucun jeton valide n’a été fourni. Sinon, examinez le contenu de la réponse pour en savoir plus sur la
      requête.

      Dans le panneau interactif à droite, vous pouvez voir une vérification indiquant si la réponse est null ou non afin de filtrer l’accès à
      votre route `/api/private`.
