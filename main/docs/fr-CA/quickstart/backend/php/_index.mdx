      Auth0 offre un [SDK
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) pour simplifier l’implémentation de l’authentification et de l’autorisation Auth0 dans les applications PHP.

      Le SDK PHP d’Auth0 nécessite l’installation de bibliothèques HTTP compatibles avec [PSR-17](https://www.php-fig.org/psr/psr-17/) et [PSR-18](https://www.php-fig.org/psr/psr-18/) pour gérer les requêtes réseau. Si vous
      n’avez pas ces bibliothèques, vous pouvez installer des options fiables en exécutant les commandes suivantes dans votre
      terminal :

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Maintenant, installez le SDK Auth0 PHP en exécutant la commande suivante dans votre terminal :

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configurer l’Auth0 SDK

      Pour que l’Auth0 SDK fonctionne correctement, vous devez définir les propriétés suivantes lors de l’initialisation :

      * `strategy` : La stratégie permet d’orienter le comportement du SDK selon le cas d’utilisation de votre application. Dans ce
        cas, vous devez définir cette valeur sur la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain : Le domaine de votre tenant Auth0. En général, vous le trouverez dans l’Auth0 Dashboard, sous les
        Settings de l’Application, dans le champ Domain. Si vous utilisez un domaine personnalisé, définissez plutôt cette valeur sur celle de votre domaine personnalisé.
      * clientId : L’ID de l’Application Auth0 que vous avez configurée plus tôt dans ce Quickstart. Vous le trouverez dans l’Auth0 Dashboard, sous les
        Settings de votre Application, dans le champ Client ID.
      * clientSecret : Le secret de l’application Auth0 que vous avez créée plus tôt dans ce Quickstart. Le Client Secret se trouve dans l’Auth0 Dashboard,
        sous les Settings de votre Application, dans le champ Client Secret.
      * audience : L’identifiant de l’Auth0 API que vous avez enregistrée ci-dessus. Cette valeur doit être fournie sous forme de tableau.

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Ensuite, développez votre application pour récupérer et traiter des jetons Bearer. Les jetons Bearer sont des jetons d’accès fournis à
      votre API dans des requêtes provenant de clients au nom des utilisateurs. Les jetons d’accès autorisent ou rejettent l’accès aux routes de votre
      application. C’est ce qu’on appelle l’autorisation des points de terminaison.

      La façon la plus simple de récupérer des jetons d’accès à partir d’une requête consiste à utiliser la méthode `getBearerToken()`
      du SDK PHP. Cette méthode récupère les jetons à partir des paramètres GET, des corps de requête POST, des en-têtes de requête et d’autres sources. Dans ce
      cas, le SDK PHP traite les jetons transmis dans des requêtes GET au moyen du paramètre `token` ou de l’en-tête HTTP
      `Authorization`.

      Maintenant que vous avez configuré votre application Auth0, le SDK Auth0 PHP et que votre application récupère des jetons Bearer
      à partir des requêtes, la prochaine étape consiste à configurer l’autorisation des points de terminaison pour votre projet. La méthode
      `getBearerToken()` que vous avez implémentée ci-dessus renvoie une classe `Token` qui comprend
      des détails sur l’accès associé à la requête.

      Comme la méthode `getBearerToken()` valide et vérifie automatiquement la requête entrante, votre
      application détermine les détails du jeton d’accès en évaluant la réponse de la méthode. Lorsque la réponse est
      null, aucun jeton valide n’a été fourni. Sinon, examinez le contenu de la réponse pour en savoir plus sur la
      requête.

      Dans le panneau interactif à droite, vous pouvez voir une vérification pour déterminer si la réponse est nulle ou non afin de contrôler l’accès à
      votre route `/api/private`.
