  <Content>
    Auth0 vous permet d&#39;ajouter rapidement une autorisation de point de terminaison basée sur des jetons à presque n&#39;importe quel type d&#39;application. Ce guide
    explique comment intégrer Auth0, ajouter une autorisation basée sur des jetons et protéger les routes de l&#39;application à l&#39;aide du SDK
    PHP Auth0.

    Pour utiliser ce guide de démarrage rapide, vous aurez besoin de :

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Pour utiliser les services Auth0, vous devez avoir une application enregistrée dans l’Auth0 Dashboard. L’application Auth0
      est l’endroit où vous configurez le fonctionnement de l’authentification pour votre projet.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle application Auth0 ou sélectionner une application existante qui représente
      le projet que vous souhaitez intégrer. Chaque application Auth0 se voit attribuer un ID client alphanumérique unique que
      le code de votre application utilise pour envoyer des requêtes aux API Auth0 au moyen du SDK.

      Tous les paramètres que vous configurez à l’aide de ce Quickstart sont automatiquement appliqués à votre application dans le [Dashboard](https://manage.auth0.com/#/), où vous
      pourrez gérer vos applications par la suite.

      Si vous préférez explorer une configuration complète, vous pouvez plutôt consulter un exemple d’application.

      ### Configurer une API

      De la même façon, vous devez créer une nouvelle API Auth0 ou utiliser une API existante qui représente le projet que vous
      souhaitez intégrer à partir du [Dashboard](https://manage.auth0.com/#/). Choisissez un identifiant unique pour l’API et notez-le. Vous aurez besoin
      de cet identifiant pour configurer votre application ci-dessous.
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 fournit un [SDK
      PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) pour simplifier l’implémentation de l’authentification et de l’autorisation Auth0 dans les applications PHP.

      Le SDK PHP Auth0 nécessite l’installation de [PSR-17](https://www.php-fig.org/psr/psr-17/) et de [PSR-18](https://www.php-fig.org/psr/psr-18/), ainsi que de bibliothèques HTTP compatibles pour gérer les requêtes réseau. Si vous
      n’avez pas ces bibliothèques à votre disposition, vous pouvez installer des options fiables en exécutant les commandes suivantes dans votre
      terminal :

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Maintenant, installez l’Auth0 PHP SDK en exécutant la commande suivante dans votre terminal :

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configurer l’Auth0 SDK

      Pour que le SDK fonctionne correctement, vous devez définir les propriétés suivantes dans l’Auth0 SDK lors de l’initialisation :

      * `strategy` : La stratégie permet d’orienter le comportement du SDK selon le cas d’utilisation de votre application. Dans ce
        cas, vous devez la définir à la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain : Le domaine de votre tenant Auth0. En général, vous le trouverez dans l’Auth0 Dashboard, sous les
        Settings de l’Application, dans le champ Domain. Si vous utilisez un domaine personnalisé, définissez plutôt cette valeur sur celle de votre domaine personnalisé.
      * clientId : L’ID de l’Application Auth0 que vous avez configurée plus tôt dans ce Quickstart. Vous pouvez le trouver dans l’Auth0 Dashboard, sous les
        Settings de votre Application, dans le champ Client ID.
      * clientSecret : Le secret de l’Application Auth0 que vous avez créée plus tôt dans ce Quickstart. Le Client Secret se trouve dans
        l’Auth0 Dashboard, sous les Settings de votre Application, dans le champ Client Secret.
      * audience : L’identifiant de l’Auth0 API que vous avez enregistrée ci-dessus. Cette valeur doit être fournie sous forme de tableau.

      <Note>
        ##### Vérification

        Votre Auth0 SDK est maintenant correctement configuré. Exécutez votre application pour vérifier que :

        * Le SDK s’initialise correctement.
        * Votre application ne génère aucune erreur liée à Auth0.
      </Note>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Ensuite, développez votre application pour récupérer et traiter les jetons porteurs. Les jetons porteurs sont des jetons d’accès fournis à
      votre API dans des requêtes provenant de clients au nom des utilisateurs. Les jetons d’accès accordent ou rejettent l’accès aux routes de votre
      application. C’est ce qu’on appelle l’autorisation des points de terminaison.

      La façon la plus simple de récupérer des jetons d’accès à partir d’une requête consiste à utiliser la méthode `getBearerToken()`
      du SDK PHP. Cette méthode récupère les jetons à partir des paramètres GET, des corps de requête POST, des en-têtes de requête et d’autres sources. Dans ce
      cas, le SDK PHP traite les jetons transmis dans les requêtes GET par le paramètre `token` ou par l’en-tête HTTP
      `Authorization`.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Maintenant, installez une bibliothèque de routage pour acheminer les requêtes entrantes vers votre application. Cette étape n&#39;est pas obligatoire,
      mais elle simplifie la structure de l&#39;application dans le cadre de ce Quickstart.

      ```bash lines
      composer require steampixel/simple-php-router
      ```

      Créez un nouveau fichier dans votre application appelé `router.php` pour définir les routes. Copiez-y le code du
      panneau interactif à droite, sous l’onglet **router.php**.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Maintenant que vous avez configuré votre application Auth0, l’Auth0 PHP SDK, et que votre application récupère des jetons Bearer
      à partir des requêtes, l’étape suivante consiste à configurer l’autorisation des points de terminaison pour votre projet. La méthode
      `getBearerToken()` que vous avez implémentée ci-dessus renvoie une classe `Token` qui comprend
      des renseignements sur l’accès associé à la requête.

      Comme la méthode `getBearerToken()` valide et vérifie automatiquement la requête entrante, votre
      application détermine les détails du jeton d’accès en évaluant la réponse de la méthode. Lorsque la réponse est
      null, aucun jeton valide n’a été fourni. Sinon, examinez le contenu de la réponse pour en savoir plus sur la
      requête.

      Dans le panneau interactif à droite, vous pouvez voir une vérification qui détermine si la réponse est null ou non afin de contrôler l’accès à
      votre route `/api/private`.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Dans certains cas, vous pourriez vouloir restreindre l’accès à une route précise en fonction des portées demandées dans un jeton
      d’accès. Comme le montre le panneau interactif à droite, évaluez le contenu de la propriété &#39;scope&#39; de la
      réponse de la méthode `getBearerToken()` pour vérifier les portées accordées par le jeton d’accès.
    </Section>
