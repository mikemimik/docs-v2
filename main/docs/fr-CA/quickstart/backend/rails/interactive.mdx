    Ce tutoriel effectue la validation du jeton d&#39;accès à l&#39;aide du gem [jwt](https://github.com/jwt/ruby-jwt) dans une classe `Auth0Client` personnalisée. Un Concern appelé
    `Secured` est utilisé pour autoriser les endpoints qui nécessitent une authentification via un jeton d&#39;accès entrant.

    Si vous n&#39;avez pas encore créé d&#39;API dans votre tableau de bord Auth0, utilisez le sélecteur interactif pour créer une nouvelle API Auth0
    ou sélectionnez une API existante pour votre projet.

    Pour configurer votre première API via le tableau de bord Auth0, consultez [notre guide de démarrage](https://auth0.com/docs/get-started/auth0-overview/set-up-apis).

    Chaque API Auth0 utilise l&#39;identifiant d&#39;API, dont votre application a besoin pour valider le jeton d&#39;accès.

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Les permissions vous permettent de définir de quelle façon il est possible d’accéder aux ressources pour le compte de l’utilisateur au moyen d’un jeton d’accès donné. Par
      exemple, vous pourriez choisir d’accorder un accès en lecture à la ressource `messages` si les utilisateurs ont un niveau
      d’accès de gestionnaire, et un accès en écriture à cette ressource s’ils ont un niveau d’accès d’administrateur.

      Vous pouvez définir les permissions autorisées dans la vue **Permissions** de la section [APIs](https://manage.auth0.com/#/apis) de l’Auth0 Dashboard.

      </Frame>

      <Info>
        Cet exemple utilise le scope `read:messages`.

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Installez la gem &#42;&#42;jwt &#42;&#42;.

      gem 'jwt'

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Créez une classe appelée `Auth0Client`. Cette classe décode et vérifie le jeton d’accès entrant provenant
      de l’en-tête `Authorization` de la requête.

      La classe `Auth0Client` récupère la clé publique de votre tenant Auth0, puis l’utilise pour vérifier la
      signature du jeton d’accès. La structure `Token` définit une méthode `validate_permissions` pour
      rechercher un `scope` particulier dans un jeton d’accès en fournissant un tableau des scopes requis et vérifier s’ils
      sont présents dans la charge utile du jeton.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Créez un Concern appelé `Secured` qui recherche le jeton d’accès dans l’en-tête `Authorization`
      d’une requête entrante.

      Si le jeton est présent, `Auth0Client.validate_token` utilisera la gem `jwt` pour vérifier
      la signature du jeton et valider ses claims.

      En plus de vérifier que le jeton d’accès est valide, le Concern comprend aussi un mécanisme pour confirmer que le
      jeton possède le **scope** nécessaire pour accéder aux ressources demandées. Dans cet exemple, nous définissons une
      méthode `validate_permissions` qui reçoit un bloc et vérifie les permissions en appelant la
      méthode `Token.validate_permissions` de la classe `Auth0Client`.

      Pour la route `/private-scoped`, les scopes définis seront croisés avec les scopes provenant du
      payload afin de déterminer s’il contient un ou plusieurs éléments de l’autre tableau.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      En ajoutant le Concern `Secure` à votre contrôleur d’application, vous n’aurez qu’à utiliser un
      filtre `before_action` dans le contrôleur qui requiert une autorisation.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Créez un contrôleur pour gérer le point de terminaison public `/api/public`.

      Le point de terminaison `/public` ne nécessite aucune autorisation; aucun `before_action` n’est donc requis.
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Créez un contrôleur pour gérer les endpoints privés : `/api/private` et
      `/api/private-scoped`.

      `/api/private` est disponible pour les requêtes authentifiées contenant un access token sans
      scopes supplémentaires.

      `/api/private-scoped` est disponible pour les requêtes authentifiées contenant un jeton d&#39;accès avec la
      portée `read:messages` accordée

      Les endpoints protégés doivent appeler la méthode `authorize` du concern `Secured`. Pour ce faire, utilisez `before_action :authorize`, ce qui garantit que la méthode `Secured.authorize` est appelée avant chaque action du `PrivateController`.

      ### Effectuer un appel vers votre API

      Pour effectuer des appels à votre API, vous avez besoin d&#39;un jeton d&#39;accès. Vous pouvez obtenir un jeton d&#39;accès à des fins de test depuis la vue **Test** dans vos [paramètres
      d&#39;API](https://manage.auth0.com/#/apis).
