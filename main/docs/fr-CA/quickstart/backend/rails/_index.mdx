    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Créez une classe appelée `Auth0Client`. Cette classe décode et vérifie le jeton d’accès entrant extrait
      de l’en-tête `Authorization` de la requête.

      La classe `Auth0Client` récupère la clé publique de votre tenant Auth0, puis l’utilise pour vérifier la
      signature du jeton d’accès. La structure `Token` définit une méthode `validate_permissions` pour
      rechercher un `scope` précis dans un jeton d’accès en fournissant un tableau de scopes requis et vérifier s’ils
      sont présents dans la charge utile du jeton.

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4" isSingleColumn>
      Créez un Concern appelé `Secured` qui recherche le jeton d’accès dans l’en-tête `Authorization`
      d’une requête entrante.

      Si le token est présent, `Auth0Client.validate_token` utilisera la gem `jwt` pour vérifier
      la signature du token et valider ses claims.

      En plus de vérifier que le jeton d’accès est valide, le Concern inclut aussi un mécanisme pour confirmer que le
      token a le **scope** suffisant pour accéder aux ressources demandées. Dans cet exemple, nous définissons une
      méthode `validate_permissions` qui reçoit un bloc et vérifie les permissions en appelant la
      méthode `Token.validate_permissions` de la classe `Auth0Client`.

      Pour la route `/private-scoped`, les scopes définis seront croisés avec les scopes provenant du
      payload afin de déterminer s’il contient un ou plusieurs éléments de l’autre array.

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      Créez un contrôleur pour gérer les endpoints privés : `/api/private` et
      `/api/private-scoped`.

      `/api/private` est disponible pour les requêtes authentifiées contenant un access token sans
      portées supplémentaires.

      `/api/private-scoped` est disponible pour les requêtes authentifiées contenant un jeton d&#39;accès avec la portée
      `read:messages` accordée

      Les endpoints protégés doivent appeler la méthode `authorize` du concern `Secured`. Pour ce faire, utilisez `before_action :authorize`, ce qui garantit que la méthode `Secured.authorize` est appelée avant chaque action du `PrivateController`.
