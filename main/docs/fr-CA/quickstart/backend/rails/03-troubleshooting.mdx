Ce document vous aidera à dépanner votre configuration si votre API renvoie des réponses inattendues. Nous vous recommandons d’ouvrir une session pour suivre ce Quickstart avec des exemples configurés pour votre compte.

Si vous avez correctement configuré la validation du JWT, votre API devrait renvoyer des réponses appropriées lorsque vous effectuez des requêtes. Cependant, si votre API renvoie une réponse 401 (Non autorisé), c’est que la configuration de votre middleware JWT ne correspond pas au JWT transmis.

Ce document vous aidera à dépanner la configuration de votre middleware JWT.

<div id="how-does-a-token-get-validated">
  ## Comment un jeton est-il validé ?
</div>

Lors de la validation d’un JWT, plusieurs éléments doivent être pris en compte :

1. **Le jeton est-il bien formé ?** Autrement dit, ce jeton respecte-t-il la structure d’un JSON Web Token (JWT) ? Pour en savoir plus sur la structure d’un JWT, veuillez consulter [cette section sur la structure d’un JWT](/docs/fr-CA/secure/tokens/json-web-tokens#what-is-the-json-web-token-structure-)
2. **Le jeton a-t-il été altéré ?** La dernière partie d’un JWT est la signature. La signature sert à vérifier que le jeton a bien été signé par l’expéditeur et qu’il n’a été modifié d’aucune façon.
3. **Le jeton a-t-il été reçu pendant sa période de validité ?** Les JWT ne sont valides que pendant une période déterminée (comme indiqué dans la claim `exp`).
4. **Le jeton provient-il de l’autorité attendue ?** Cela comporte 2 parties

   * **Vérification de la signature** : Pouvons-nous confirmer que le JWT est correctement signé à l’aide de la clé émise par l’autorité émettrice ?
   * **Valeur de l’émetteur** : L’émetteur est défini dans la claim `iss`. Encore une fois, cette claim correspond-elle à ce que votre application attend ?
5. **Le jeton est-il destiné à l’application actuelle ?** Autrement dit, la claim `aud` du JWT correspond-elle à ce que votre application attend ?

<div id="inspecting-a-token">
  ## Inspection d’un jeton
</div>

Un moyen rapide d’inspecter un JWT consiste à utiliser le site Web [JWT.io](https://jwt.io/). Il comprend un débogueur pratique qui vous permet de vérifier rapidement qu’un JWT est bien formé et d’inspecter les valeurs des différents claims.

<Frame>![Débogage d’un JWT sur JWT.io](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-rs256.png)</Frame>

Dans la capture d’écran ci-dessus, vous pouvez voir que le jeton a été signé à l’aide de l’algorithme **RS256**. L’**Issuer** du jeton est **https://jerrie.auth0.com/**, et l’**Audience** est **https://rs256.test.api**.

Autrement dit, ces valeurs dans l’enregistrement de votre configuration de validation du JWT doivent correspondre **exactement** — y compris la barre oblique finale de l’Issuer, comme

```
audience = "https://rs256.test.api",
domain = "https://jerrie.auth0.com/"
};
```
