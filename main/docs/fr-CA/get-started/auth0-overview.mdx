title: Aperçu d’Auth0
---

Auth0 est une solution flexible, prête à l’emploi, qui permet d’ajouter des services d’authentification et d’autorisation à vos applications. Votre équipe et votre organisation peuvent ainsi éviter les coûts, les délais et les risques associés à la création de leur propre solution pour authentifier les utilisateurs et gérer leurs autorisations.

Voici quelques exemples de cas d’utilisation d’Auth0 :

* Vous avez créé une application formidable et vous souhaitez y ajouter l’authentification et l’autorisation des utilisateurs. Vos utilisateurs devraient pouvoir se connecter soit avec un identifiant (`username`, courriel ou numéro de téléphone) et un mot de passe, soit avec leurs comptes sociaux (comme Facebook ou X). Vous souhaitez récupérer le profil de l’utilisateur après la connexion afin de personnaliser l’UI et d’appliquer vos politiques d’autorisation.
* Vous avez créé une API et vous souhaitez la sécuriser avec [OAuth 2.0](/docs/fr-CA/authenticate/protocols/oauth).
* Vous avez plus d’une application et vous souhaitez mettre en place l’[authentification unique (SSO)](/docs/fr-CA/authenticate/single-sign-on).
* Vous avez créé une application frontale en JavaScript et une application mobile, et vous voulez qu’elles puissent toutes deux accéder à votre API de façon sécurisée.
* Vous avez une application web qui doit authentifier les utilisateurs au moyen de [Security Assertion Markup Language (SAML)](/docs/fr-CA/authenticate/protocols/saml/saml-configuration).
* Vous estimez que les mots de passe ne suffisent plus et vous voulez que vos utilisateurs se connectent à l’aide de codes à usage unique envoyés par courriel ou par SMS.
* Si l’une des adresses courriel de vos utilisateurs est compromise dans une fuite de données publique sur un site, vous voulez en être avisé, puis avertir les utilisateurs et/ou les empêcher de se connecter à votre application jusqu’à ce qu’ils réinitialisent leur mot de passe.
* Vous souhaitez agir de manière proactive pour bloquer les adresses IP suspectes si elles effectuent plusieurs tentatives de connexion échouées consécutives, afin d’éviter les attaques DDoS.
* Vous faites partie d’une grande organisation qui souhaite fédérer son service d’annuaire d’entreprise existant pour permettre aux employés de se connecter aux diverses applications internes et applications tierces à l’aide de leurs identifiants d’entreprise existants.
* Vous ne voulez pas (ou vous ne savez pas comment) mettre en œuvre votre propre solution de gestion des utilisateurs. Réinitialisation des mots de passe, création, provisionnement, blocage et suppression des utilisateurs, ainsi que l’UI pour gérer tout cela. Vous voulez simplement vous concentrer sur votre application.
* Vous souhaitez imposer l’[authentification multifacteur (MFA)](/docs/fr-CA/secure/multi-factor-authentication) lorsque vos utilisateurs veulent accéder à des données sensibles.
* Vous cherchez une solution de gestion des identités qui vous aidera à rester à jour face aux exigences de conformité en constante augmentation de SOC2, du RGPD, de PCI DSS, de HIPAA et d’autres encore.
* Vous voulez surveiller les utilisateurs sur votre site ou dans votre application. Vous prévoyez utiliser ces données pour créer des entonnoirs, mesurer la fidélisation des utilisateurs et améliorer votre flux d’inscription.
* Vous voulez une politique d’autorisation robuste pour permettre à vos utilisateurs d’accéder aux ressources selon [leur relation](https://docs.fga.dev/authorization-concepts) avec la ressource ou [leur rôle](/docs/fr-CA/manage-users/access-control/rbac) dans votre organisation.
