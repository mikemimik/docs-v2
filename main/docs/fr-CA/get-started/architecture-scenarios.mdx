<div id="application-architecture-scenarios">
  ## Scénarios d’architecture d’application
</div>

Ces scénarios décrivent les différents types d’architectures technologiques que votre application peut adopter et comment Auth0 peut vous aider. Leur objectif est de vous guider tout au long du processus de mise en œuvre, du début à la fin.

| Lire...                                                                                                                | Pour apprendre...                                                                                                                                                                             |
| ---------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Organisations multiples](/docs/fr-CA/get-started/architecture-scenarios/multiple-organization-architecture)                 | à configurer une architecture d’application qui utilise soit des organisations à fournisseur d’identité unique, soit des organisations à plusieurs fournisseurs d’identité.                   |
| [Authentification unique pour les Regular Web Apps](/docs/fr-CA/get-started/architecture-scenarios/sso-for-regular-web-apps) | à configurer et à mettre en œuvre une application web qui utilise le SSO pour les employés utilisant Active Directory et les sous-traitants utilisant un référentiel d’utilisateurs distinct. |
| [Application serveur + API](/docs/fr-CA/get-started/architecture-scenarios/server-application-api)                           | à configurer une API de feuille de temps à l’aide d’une tâche cron pour consigner des entrées de feuille de temps.                                                                            |
| [Application monopage + API](/docs/fr-CA/get-started/architecture-scenarios/spa-api)                                         | à configurer une API de feuille de temps à l’aide d’une application monopage pour consigner des entrées de feuille de temps.                                                                  |
| [Application mobile + API](/docs/fr-CA/get-started/architecture-scenarios/mobile-api)                                        | à configurer une API de feuille de temps à l’aide d’une application mobile pour consigner des entrées de feuille de temps dans une base de données centralisée.                               |
