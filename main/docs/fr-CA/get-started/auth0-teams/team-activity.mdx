description: Team Activity permet aux Team Owners de consulter et d’auditer les journaux d’événements générés par un membre de l’équipe.
title: Team Activity
---

Team Activity permet aux Team Owners de consulter les journaux d’audit liés aux activités effectuées par les membres dans leur compte Auth0.

Le rapport Team Activity se trouve dans la section **Reports** du Teams Dashboard.

Team Activity offre une vue sommaire du membre qui a généré l’événement, du type d’événement et de la date à laquelle l’événement a été généré.

Vous pouvez sélectionner **View Details** pour obtenir plus d’informations, comme la description de l’événement, le statut de l’événement, l’adresse IP de l’utilisateur qui a déclenché l’événement, les détails de l’agent utilisateur, et plus encore.

<div id="event-types">
  ## Types d’événements
</div>

Les types d’événements de Team Activity sont décrits ci-dessous.

| Type d’événement       | Description                                                                            |
| ---------------------- | -------------------------------------------------------------------------------------- |
| Invitation d’équipe    | Invitation de membre de l’équipe créée<br />Invitation de membre de l’équipe supprimée |
| Membre de l’équipe     | Membre de l’équipe supprimé<br />Rôle du membre de l’équipe mis à jour                 |
| Politique de sécurité  | Politique de sécurité mise à jour<br />Connexions sociales mises à jour                |
| Paramètres de l’équipe | Paramètres de l’équipe mis à jour                                                      |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le soutien de l’authentification des utilisateurs du Dashboard dans Auth0 et le Teams Dashboard, ainsi que d’autres événements, sera offert ultérieurement.
</Callout>

<div id="event-schema">
  ## Schéma des événements
</div>

Le schéma des événements de Team Activity est décrit ci-dessous.

```json lines
{  
    "id": "string", 
    "team_id": "string",
    "user_id": "string",
    "type": "string",
    "description": "string",
    "status": "string",
    "details": "object",
    "ip": "string",
    "hostname": "string",
    "user_agent": "string",
    "version": "string",
    "created_at": "date",
    "team_slug": "string",
    "user": "object"
}
```

| Nom           | Type     | Description                                                                              |
| ------------- | -------- | ---------------------------------------------------------------------------------------- |
| `id`          | string   | Identifiant unique du journal d’événement.                                               |
| `team_id`     | string   | Identifiant unique de l’équipe.                                                          |
| `user_id`     | string   | L’identifiant unique de l’utilisateur qui a généré l’événement.                          |
| `type`        | string   | Type d’événement.                                                                        |
| `description` | string   | Résumé du type d’événement.                                                              |
| `status`      | string   | État de l’événement. Les valeurs possibles sont `Success` ou `Failure`.                  |
| `details`     | object   | Détails de l’événement enregistré.                                                       |
| `ip`          | string   | Adresse IP de l’utilisateur qui a généré le journal d’événement.                         |
| `hostname`    | string   | FQDN de l’Auth0 App sur laquelle l’événement s’est produit.                              |
| `user_agent`  | string   | Navigateur Web, ainsi que le type et la version du système d’exploitation de l’appareil. |
| `version`     | string   | Version du schéma selon lequel le journal d’activité a été généré.                       |
| `created_at`  | datetime | Date et heure auxquelles le journal d’événement a été généré.                            |
| `team_slug`   | string   | Identifiant textuel unique de l’équipe.                                                  |
| `user`        | object   | Contient le nom, l’URL de l’avatar et le statut d’appartenance au Team (booléen).        |
