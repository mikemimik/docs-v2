title: Team Activity
---

Team Activity permet aux Team Owner de consulter les journaux d’audit associés aux activités des membres sur leur compte Auth0.

Le rapport Team Activity se trouve dans la section **Reports** du Teams Dashboard.

Team Activity fournit une vue d’ensemble du membre qui a généré l’événement, du type d’événement et de la date à laquelle l’événement a été généré.

Vous pouvez sélectionner **View Details** pour obtenir plus d’informations, comme la description de l’événement, le statut de l’événement, l’adresse IP de l’utilisateur qui a déclenché l’événement, les détails de l’agent utilisateur, et plus encore.

<div id="event-types">
  ## Types d’événements
</div>

Les types d’événements de Team Activity sont décrits ci-dessous.

| Type d’événement       | Description                                                                                |
| ---------------------- | ------------------------------------------------------------------------------------------ |
| Invitation d’équipe    | Invitation d’un membre de l’équipe créée<br />Invitation d’un membre de l’équipe supprimée |
| Membre de l’équipe     | Membre de l’équipe supprimé<br />Rôle du membre de l’équipe mis à jour                     |
| Politique de sécurité  | Politique de sécurité mise à jour<br />Connexions sociales mises à jour                    |
| Paramètres de l’équipe | Paramètres de l’équipe mis à jour                                                          |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le soutien pour l’authentification des utilisateurs du Dashboard dans Auth0 et Teams Dashboard, ainsi que pour d’autres événements, sera offert ultérieurement.
</Callout>

<div id="event-schema">
  ## Schéma d’événement
</div>

Le schéma d’événement de Team Activity est décrit ci-dessous.

```json lines
{  
    "id": "string", 
    "team_id": "string",
    "user_id": "string",
    "type": "string",
    "description": "string",
    "status": "string",
    "details": "object",
    "ip": "string",
    "hostname": "string",
    "user_agent": "string",
    "version": "string",
    "created_at": "date",
    "team_slug": "string",
    "user": "object"
}
```

| Nom           | Type     | Description                                                                            |
| ------------- | -------- | -------------------------------------------------------------------------------------- |
| `id`          | string   | Identifiant unique de l’entrée du journal d’événements.                                |
| `team_id`     | string   | Identifiant unique de l’équipe.                                                        |
| `user_id`     | string   | Identifiant unique de l’utilisateur qui a généré l’événement.                          |
| `type`        | string   | Type d’événement.                                                                      |
| `description` | string   | Résumé du type d’événement.                                                            |
| `status`      | string   | Statut de l’événement. Les valeurs possibles sont `Success` ou `Failure`.              |
| `details`     | object   | Détails de l’événement consigné.                                                       |
| `ip`          | string   | Adresse IP de l’utilisateur qui a généré l’entrée du journal d’événements.             |
| `hostname`    | string   | Nom de domaine pleinement qualifié (FQDN) de l’Auth0 App où l’événement s’est produit. |
| `user_agent`  | string   | Navigateur Web ainsi que type et version du système d’exploitation de l’appareil.      |
| `version`     | string   | Version du schéma selon laquelle le journal d’activité a été généré.                   |
| `created_at`  | datetime | Date et heure auxquelles l’entrée du journal d’événements a été générée.               |
| `team_slug`   | string   | Identifiant textuel unique de l’équipe.                                                |
| `user`        | object   | Contient le nom, l’URL de l’avatar et le statut d’appartenance à l’équipe (booléen).   |
