title: Team Activity
---

Team Activity permet aux Team Owner de consulter les journaux d’audit associés aux activités des membres sur leur compte Auth0.

Le rapport Team Activity se trouve dans la section **Reports** du Teams Dashboard.

Team Activity fournit une vue d’ensemble du membre qui a généré l’événement, du type d’événement et de la date à laquelle l’événement a été généré.

Vous pouvez sélectionner **View Details** pour obtenir plus d’informations, comme la description de l’événement, le statut de l’événement, l’adresse IP de l’utilisateur qui a déclenché l’événement, les détails de l’agent utilisateur, et plus encore.

<div id="event-types">
  ## Types d’événements
</div>

Les types d’événements de Team Activity sont décrits ci-dessous.

| Type d’événement      | Description                                                                      |
| --------------------- | -------------------------------------------------------------------------------- |
| Invitation d’équipe   | Invitation de membre d’équipe créée<br />Invitation de membre d’équipe supprimée |
| Membre d’équipe       | Membre d’équipe supprimé<br />Rôle du membre d’équipe mis à jour                 |
| Politique de sécurité | Politique de sécurité mise à jour<br />Connexions sociales mises à jour          |
| Paramètres d’équipe   | Paramètres d’équipe mis à jour                                                   |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La prise en charge de l’authentification des utilisateurs du Dashboard dans Auth0 et le Teams Dashboard, ainsi que d’autres événements, sera offerte ultérieurement.
</Callout>

<div id="event-schema">
  ## Schéma d’événement
</div>

Le schéma d’événement de Team Activity est décrit ci-dessous.

```json lines
{  
    "id": "string", 
    "team_id": "string",
    "user_id": "string",
    "type": "string",
    "description": "string",
    "status": "string",
    "details": "object",
    "ip": "string",
    "hostname": "string",
    "user_agent": "string",
    "version": "string",
    "created_at": "date",
    "team_slug": "string",
    "user": "object"
}
```

| Nom           | Type     | Description                                                                             |
| ------------- | -------- | --------------------------------------------------------------------------------------- |
| `id`          | chaîne   | Identifiant unique du journal d’événements.                                             |
| `team_id`     | chaîne   | Identifiant unique de l’équipe.                                                         |
| `user_id`     | chaîne   | Identifiant unique de l’utilisateur qui a généré l’événement.                           |
| `type`        | chaîne   | Type d’événement.                                                                       |
| `description` | chaîne   | Résumé du type d’événement.                                                             |
| `status`      | chaîne   | Statut de l’événement. Les valeurs possibles sont `Success` ou `Failure`.               |
| `details`     | objet    | Détails de l’événement consigné.                                                        |
| `ip`          | chaîne   | Adresse IP de l’utilisateur qui a généré l’entrée du journal d’événements.              |
| `hostname`    | chaîne   | FQDN de l’Auth0 App dans laquelle l’événement s’est produit.                            |
| `user_agent`  | chaîne   | Navigateur Web ainsi que le type et la version du système d’exploitation de l’appareil. |
| `version`     | chaîne   | Version du schéma selon laquelle le journal d’activité a été généré.                    |
| `created_at`  | datetime | Date et heure de génération du journal d’événements.                                    |
| `team_slug`   | chaîne   | Identifiant textuel unique de l’équipe.                                                 |
| `user`        | objet    | Contient le nom, l’URL de l’avatar et le statut d’appartenance à l’équipe (booléen).    |
