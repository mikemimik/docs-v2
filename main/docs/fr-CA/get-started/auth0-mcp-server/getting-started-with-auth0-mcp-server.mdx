<Card title="Avant de commencer">
  * Installez [Node.js v18 ou une version ultérieure](https://nodejs.org/en/download) sur votre système.
  * Créez un [compte Auth0](https://auth0.com/) actif avec des permissions d’administrateur.
  * Installez au moins un client MCP pris en charge ([Claude Desktop](https://claude.ai/download), [Cursor](https://www.cursor.com/downloads) ou [Windsurf](https://www.windsurf.com/editor), par exemple).
</Card>

Donnez à votre équipe et à vos utilisateurs les outils nécessaires en installant [Auth0 Model Context Protocol (MCP) Server](/docs/fr-CA/get-started/auth0-mcp-server) et en l’intégrant à plusieurs [clients MCP](https://modelcontextprotocol.io/clients) couramment utilisés.

<div id="installation-and-configuration">
  ## Installation et configuration
</div>

Installez Auth0 MCP Server en exécutant l’une des commandes suivantes dans votre terminal :

<div id="for-claude-desktop-default">
  ### Pour Claude Desktop (par défaut)
</div>

```bash lines
npx @auth0/auth0-mcp-server init
```

<div id="for-cursor">
  ### Pour Cursor
</div>

```bash lines
npx @auth0/auth0-mcp-server init --client cursor
```

<div id="for-windsurf">
  ### Pour Windsurf
</div>

```bash lines
npx @auth0/auth0-mcp-server init --client windsurf
```

<div id="other-mcp-clients">
  ### Autres clients MCP
</div>

Pour utiliser Auth0 MCP Server avec n’importe quel autre client MCP, vous pouvez ajouter manuellement cette configuration au client, puis le redémarrer pour que les modifications prennent effet :

```json lines
{
  "mcpServers": {
    "auth0": {
      "command": "npx",
      "args": ["-y", "@auth0/auth0-mcp-server", "run"],
      "capabilities": ["tools"],
      "env": {
        "DEBUG": "auth0-mcp"
      }
    }
  }
}
```

<div id="authenticate-with-auth0">
  ## Authentifiez-vous avec Auth0
</div>

Votre navigateur s’ouvrira automatiquement pour lancer le <Tooltip tip="OAuth 2.0 : Cadre d’autorisation qui définit les protocoles et les processus d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip> <Tooltip tip="OAuth 2.0 : Cadre d’autorisation qui définit les protocoles et les processus d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=authorization+flow">flux d’autorisation sur appareil</Tooltip>. Connectez-vous à votre compte Auth0 et accordez les autorisations demandées.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les informations d’identification sont stockées de façon sécurisée dans le trousseau de votre système. Au besoin, vous pouvez le vérifier à l’aide de votre outil de gestion du trousseau.
</Callout>

<div id="verify-your-integration">
  ## Vérifiez votre intégration
</div>

<div id="claude-desktop">
  ### Claude Desktop
</div>

1. Redémarrez Claude Desktop pour vous assurer qu’il reconnaît l’Auth0 MCP Server nouvellement installé.
2. Dans Claude Desktop, cliquez sur l’icône **Tools** et vérifiez que les outils Auth0 (par exemple, `auth0_create_action` ou `auth0_delete_application`) s’affichent dans la liste **Available MCP Tools**.
3. Lancez une conversation avec Claude Desktop et demandez-lui de vous aider à gérer votre tenant Auth0.

<div id="cursor">
  ### Cursor
</div>

1. Redémarrez Cursor pour vous assurer qu’il reconnaît l’Auth0 MCP Server nouvellement installé.
2. Dans Cursor, ouvrez **Settings &gt; MCP** et vérifiez que **Auth0** apparaît dans la liste **MCP Servers**.
3. Lancez une conversation avec Cursor et demandez-lui de vous aider à gérer votre tenant Auth0.

<div id="windsurf">
  ### Windsurf
</div>

1. Redémarrez Windsurf pour vous assurer qu’il reconnaît l’**Auth0 MCP Server** nouvellement installé.
2. Dans Windsurf, ouvrez **Paramètres de Windsurf &gt; Cascade** et vérifiez que **Auth0** figure dans la liste des **serveurs MCP (Model Context Protocol)**.
3. Lancez une conversation avec Windsurf et demandez de l’aide pour gérer votre tenant Auth0.

<div id="common-troubleshooting-steps">
  ## Étapes de dépannage courantes
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La plupart des problèmes de connexion peuvent être résolus en configurant Auth0 MCP Server et en redémarrant le client MCP.
</Callout>

<div id="get-command-line-help">
  #### Obtenir de l’aide sur la ligne de commande :
</div>

Consultez la liste des commandes prises en charge et des exemples d’utilisation avec :

```bash lines
npx @auth0/auth0-mcp-server help
```

<div id="authentication-issues">
  #### Problèmes d’authentification
</div>

Assurez-vous que les permissions dans votre tenant Auth0 sont correctes, puis essayez de réinitialiser avec :

```bash lines
npx @auth0/auth0-mcp-server init
```

<div id="connection-failures">
  #### Échecs de connexion :
</div>

1. Confirmez que le serveur MCP est bien en cours d’exécution avec :

   ```ps lines
   ps aux | grep auth0-mcp
   ```

2. Redémarrez votre client d’IA après avoir vérifié que le serveur est en cours d’exécution.

<div id="claude-cant-connect-to-the-server-error">
  #### Erreur « Claude Can&#39;t Connect to the Server » :
</div>

1. Redémarrez Claude Desktop après l’installation.
2. Vérifiez que le serveur est en cours d’exécution avec :

   ```ps lines
   ps aux | grep auth0-mcp
   ```

<div id="api-errors">
  #### Erreurs d’API :
</div>

Vérifiez les permissions de votre jeton Auth0 et sa date d’expiration. Activez le mode de débogage (voir ci-dessous) pour consulter des journaux détaillés.

<div id="debug-configuration-issues">
  ## Problèmes de configuration du débogage
</div>

<div id="enable-debug-mode">
  #### Activer le mode de débogage
</div>

Activez le mode de débogage pour consulter des journaux détaillés :

```bash lines
export DEBUG=auth0-mcp
```

<div id="debug-client-logs">
  #### Journaux de débogage du client
</div>

Obtenez les journaux détaillés du client MCP dans Claude Desktop :

```bash lines
# Suivre les journaux en temps réel
tail -n 20 -F ~/Library/Logs/Claude/mcp*.log
```

Obtenez des logs détaillés du serveur MCP en exécutant Auth0 MCP Server en mode de débogage :

```bash lines
DEBUG=auth0-mcp npx @auth0/auth0-mcp-server run
```

<div id="advanced-debugging-using-mcp-inspector">
  #### Débogage avancé avec MCP Inspector
</div>

Pour effectuer un dépannage avancé, utilisez MCP Inspector :

```bash wrap lines
npx @modelcontextprotocol/inspector -e DEBUG='auth0-mcp' @auth0/auth0-mcp-server run__text in bold__
```
