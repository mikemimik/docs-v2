Utilisez des interactions en langage naturel avec [Auth0 MCP Server](/docs/fr-CA/get-started/auth0-mcp-server) pour configurer et gérer rapidement un système complet d’autorisation d’API. En tirant parti de la capacité de [Claude](https://claude.ai/) à interagir avec Auth0 au moyen du Model Context Protocol, vous pouvez mettre en œuvre des schémas d’autorisation complexes en quelques minutes plutôt qu’en quelques heures.

<div id="step-1-explore-current-api-configuration">
  ### Étape 1 : Explorer la configuration actuelle des API
</div>

Avant de créer de nouvelles API, il est important de comprendre ce qui est déjà configuré dans votre tenant. Avec Auth0 MCP Server, vous pouvez obtenir rapidement ces renseignements en langage naturel.

Demandez à Claude :

```bash wrap lines
Show me all the resource servers (APIs) currently configured in my Auth0 tenant.
```

Claude utilisera l’outil `auth0_list_resource_servers` pour récupérer et afficher vos API existantes. Cela vous donne une vue d’ensemble rapide de vos API sans avoir à passer par le <Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>.

<div id="step-2-create-new-api-definitions">
  ### Étape 2 : Créer de nouvelles définitions d’API
</div>

Maintenant que vous comprenez votre configuration actuelle, vous pouvez créer de nouvelles API pour votre architecture de microservices. Au lieu de configurer chaque API séparément dans le tableau de bord, vous pouvez toutes les définir en une seule fois.

Demandez à Claude :

```bash lines
Je dois créer trois nouvelles API pour nos microservices :
1. Une API d'inventaire avec des scopes de lecture et d'écriture
2. Une API de commandes avec des scopes de lecture et d'écriture
3. Une API de clients avec des scopes de lecture, d'écriture et admin

Chaque API doit appliquer la validation des scopes.
```

Claude utilisera l’outil `auth0_create_resource_server` à plusieurs reprises pour configurer ces API.

<div id="step-3-implement-authorization-logic">
  ### Étape 3 : Implémenter la logique d’autorisation
</div>

Une fois vos API définies, vous devrez implémenter une logique d’autorisation personnalisée. Avec Auth0 MCP Server, vous pouvez décrire vos règles métier en langage naturel, et Claude générera le code Auth0 [Actions](/docs/fr-CA/customize/actions) approprié.

Demandez à Claude :

```bash lines
Crée une Action qui applique les règles d'autorisation suivantes :
1. Vérifie que l'application requérante dispose des scopes appropriés pour l'API accédée.
2. Pour l'API customers, les requêtes utilisant le scope admin doivent provenir uniquement de nos plages d'adresses IP internes (10.0.0.0/8 et 172.16.0.0/12).
3. Consigne toutes les décisions d'autorisation à des fins d'audit.
```

Claude utilisera l’outil `auth0_create_action` pour générer et créer une Action contenant le code approprié.

<div id="step-4-deploy-the-authorization-action">
  ### Étape 4 : Déployer l’Action d’autorisation
</div>

Une fois satisfait du code de l’Action, vous pouvez la déployer dans votre environnement Production à l’aide d’une simple instruction.

Demandez à Claude :

```bash lines
Déployez cette action pour qu'elle s'exécute pendant le processus d'émission de jetons.
```

Claude utilisera l’outil `auth0_deploy_action` pour déployer l’Action.

<div id="step-5-configure-client-applications">
  ### Étape 5 : Configurer les applications clientes
</div>

Ensuite, vous devrez configurer vos applications clientes pour accéder aux API voulues avec les portées appropriées.

Demandez à Claude :

```bash wrap lines
Create an 'inventory-manager' application on my Auth0 tenant and update the app to access the inventory and orders APIs with read and write permissions, but not the customers API.
```

Claude utilisera d’abord `auth0_get_application` pour récupérer la configuration actuelle, puis `auth0_update_application` pour la mettre à jour.

<div id="step-6-verify-configuration">
  ### Étape 6 : Vérifier la configuration
</div>

Pour vous assurer que votre système d’autorisation fonctionne correctement, vous pouvez consulter les journaux pour repérer tout problème d’autorisation.

Demandez à Claude :

```bash wrap lines
Check the logs for any failed API access attempts in the last hour. Focus on authorization failures.
```

Claude utilisera `auth0_list_logs` pour trouver les entrées de journal pertinentes.

<div id="step-7-refine-the-authorization-rules">
  ### Étape 7 : Affiner les règles d’autorisation
</div>

D’après l’analyse des journaux, vous devrez peut-être affiner vos règles d’autorisation. Par exemple, vous devrez peut-être autoriser l’accès à partir de plages d’adresses IP supplémentaires.

Demandez à Claude :

```bash wrap lines
Mettez à jour l'action d'autorisation pour autoriser également les requêtes provenant de notre plage VPN de développement 192.168.100.0/24.
```

Claude utilisera `auth0_get_action` pour récupérer le code actuel de l’Action, puis `auth0_update_action` pour le modifier.

Enfin, Claude utilisera `auth0_deploy_action` pour déployer l’Action mise à jour.

<div id="conclusion">
  ### Conclusion
</div>

En utilisant Auth0 MCP Server avec Claude, vous avez mis en place un système complet d’autorisation d’API en :

* créant plusieurs définitions d’API avec les scopes appropriés
* implémentant une logique d’autorisation personnalisée avec des restrictions basées sur l’adresse IP
* configurant les applications clientes pour accéder à des API précises
* surveillant et peaufinant vos règles d’autorisation

Cette approche en langage naturel vous fait non seulement gagner du temps, mais vous permet aussi de mettre en œuvre les pratiques exemplaires en matière de sécurité sans devoir posséder une expertise approfondie des Actions d’Auth0 ni de la syntaxe des requêtes de journal.

  ## Prochaines étapes
</div>

* Mettre en place des règles d’autorisation supplémentaires selon les attributs des utilisateurs

* Créer des tableaux de bord personnalisés pour visualiser les tendances d’utilisation de l’API
