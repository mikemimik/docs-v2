Consultez ce guide de référence pour découvrir des exemples en langage naturel qui vous permettront d’interagir avec votre tenant Auth0 à l’aide d’Auth0 MCP Server et de tout [client MCP](https://modelcontextprotocol.io/clients).

<div id="application-management">
  ## Gestion des applications
</div>

| Outil                       | Description                                                               | Exemples en langage naturel                                                                                                                                                                                                                                                                                                          |
| --------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `auth0_list_applications`   | Lister toutes les applications du tenant Auth0 ou les rechercher par nom. | &quot;Montre-moi toutes mes applications Auth0.&quot;<br /><br />&quot;Trouve les applications qui ont &#39;api&#39; dans leur nom.&quot;<br /><br />&quot;Quelles applications ai-je dans mon tenant Auth0?&quot;                                                                                                                   |
| `auth0_get_application `    | Obtenir les détails d’une application Auth0 précise.                      | &quot;Montre-moi les détails de l&#39;application appelée &#39;Customer Portal&#39;.&quot;<br /><br />&quot;Obtiens des renseignements sur mon application avec le Client ID {clientID}.&quot;<br /><br />&quot;Quelles sont les URL de rappel de mon &#39;Mobile App&#39;?&quot;                                                    |
| `auth0_create_application ` | Créer une nouvelle application Auth0.                                     | &quot;Crée une nouvelle application monopage appelée &#39;Analytics Dashboard&#39;.&quot;<br /><br />&quot;Configure une nouvelle application mobile native appelée &#39;iOS Client&#39;.&quot;<br /><br />&quot;Crée une application machine-to-machine pour notre service d’arrière-plan.&quot;                                    |
| `auth0_update_application`  | Mettre à jour une application Auth0 existante.                            | &quot;Mets à jour les URL de rappel de mon &#39;Web App&#39; pour inclure `https://staging.example.com/callback`.&quot;<br /><br />&quot;Change l’URL de déconnexion de &#39;Customer Portal&#39;.&quot;<br /><br />&quot;Ajoute des métadonnées d’environnement de développement à mon application &#39;Admin Dashboard&#39;.&quot; |

<div id="combining-operations">
  ## Combiner plusieurs opérations
</div>

Vous pouvez aussi demander à [Claude](https://claude.ai/) d’effectuer plusieurs opérations connexes à la suite :

<div id="set-up-a-new-api-with-client-access">
  #### Configurer une nouvelle API avec Client Access
</div>

```bash wrap lines
Create a new API called "Inventory API" with read and write scopes, then update my "Admin Dashboard" application to have access to it.
```

<div id="create-and-deploy-a-security-action">
  #### Créer et déployer une Action de sécurité
</div>

```bash wrap lines
Créez une Action qui exige le MFA pour les utilisateurs qui se connectent depuis l'extérieur de notre réseau d'entreprise, puis déployez-la dans le flux de connexion.
```

<div id="analyze-security-events">
  #### Analyser les événements de sécurité
</div>

```bash wrap lines
Show me all failed login attempts from the past week, and identify any patterns or suspicious activity.
```

<div id="updating-branding-across-forms">
  #### Mettre à jour l’image de marque dans les formulaires
</div>

```bash wrap lines
Update all our Auth0 Form to use our new logo at <https://example.com/new-logo.png> and change the primary color to #0066CC.
```

<div id="best-practices">
  ## Meilleures pratiques
</div>

1. Soyez précis dans vos requêtes : incluez des noms, des IDs ou d&#39;autres renseignements permettant d&#39;identifier les ressources visées.
2. Utilisez un langage naturel : pas besoin de mémoriser les noms d&#39;outils ni les paramètres; décrivez simplement ce que vous souhaitez accomplir.
3. Vérifiez avant de confirmer : lorsque Claude propose des modifications, prenez le temps de les examiner attentivement avant de confirmer.
4. Commencez simplement : faites d&#39;abord des requêtes simples pour vous familiariser avec la façon dont Claude interprète vos requêtes.

<div id="troubleshooting">
  ## Dépannage
</div>

Si Claude ne parvient pas à effectuer une opération, vérifiez les points suivants :

1. Authentification : assurez-vous d’être connecté au moyen d’Auth0 CLI (`auth0 login`).
2. Permissions : vérifiez que votre compte dispose des permissions nécessaires dans Auth0.
3. Précision : essayez d’être plus précis dans votre demande.
