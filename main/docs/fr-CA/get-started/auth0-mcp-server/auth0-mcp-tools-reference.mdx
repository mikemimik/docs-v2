Consultez ce guide de référence pour voir des exemples en langage naturel vous permettant d’interagir avec votre tenant Auth0 à l’aide d’Auth0 MCP Server et de n’importe quel [client MCP](https://modelcontextprotocol.io/clients).

<div id="application-management">
  ## Gestion des applications
</div>

| Outil                       | Description                                                               | Exemples en langage naturel                                                                                                                                                                                                                                                                                                          |
| --------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `auth0_list_applications`   | Lister toutes les applications du tenant Auth0 ou les rechercher par nom. | &quot;Montre-moi toutes mes applications Auth0.&quot;<br /><br />&quot;Trouve les applications dont le nom contient &#39;api&#39;.&quot;<br /><br />&quot;Quelles applications ai-je dans mon tenant Auth0?&quot;                                                                                                                    |
| `auth0_get_application `    | Obtenir les détails d’une application Auth0 en particulier.               | &quot;Montre-moi les détails de l’application appelée &#39;Customer Portal&#39;.&quot;<br /><br />&quot;Obtiens des renseignements sur mon application avec l’ID client {clientID}.&quot;<br /><br />&quot;Quelles sont les URL de rappel de mon &#39;Mobile App&#39;?&quot;                                                         |
| `auth0_create_application ` | Créer une nouvelle application Auth0.                                     | &quot;Crée une nouvelle application monopage appelée &#39;Analytics Dashboard&#39;.&quot;<br /><br />&quot;Configure une nouvelle application native appelée &#39;iOS Client&#39;.&quot;<br /><br />&quot;Crée une application machine à machine pour notre service d’arrière-plan.&quot;                                            |
| `auth0_update_application`  | Mettre à jour une application Auth0 existante.                            | &quot;Mets à jour les URL de rappel de mon &#39;Web App&#39; pour inclure `https://staging.example.com/callback`.&quot;<br /><br />&quot;Change l’URL de déconnexion de &#39;Customer Portal&#39;.&quot;<br /><br />&quot;Ajoute des métadonnées d’environnement de développement à mon application &#39;Admin Dashboard&#39;.&quot; |

<div id="combining-operations">
  ## Combiner plusieurs opérations
</div>

Vous pouvez aussi demander à [Claude](https://claude.ai/) d’effectuer plusieurs opérations connexes à la suite :

<div id="set-up-a-new-api-with-client-access">
  #### Configurer une nouvelle API avec accès client
</div>

```bash wrap lines
Create a new API called "Inventory API" with read and write scopes, then update my "Admin Dashboard" application to have access to it.
```

<div id="create-and-deploy-a-security-action">
  #### Créer et déployer une Action de sécurité
</div>

```bash wrap lines
Créez une Action qui exige le MFA pour les utilisateurs qui se connectent depuis l'extérieur de notre réseau d'entreprise, puis déployez-la dans le flux de connexion.
```

<div id="analyze-security-events">
  #### Analyser les événements de sécurité
</div>

```bash wrap lines
Show me all failed login attempts from the past week, and identify any patterns or suspicious activity.
```

<div id="updating-branding-across-forms">
  #### Mise à jour de l’image de marque dans tous les Forms
</div>

```bash wrap lines
Mettez à jour tous nos Forms Auth0 pour utiliser notre nouveau logo à l'adresse <https://example.com/new-logo.png> et changez la couleur principale en #0066CC.
```

<div id="best-practices">
  ## Bonnes pratiques
</div>

1. Soyez précis dans vos demandes : indiquez les noms, les ID ou tout autre renseignement permettant d’identifier les ressources précises auxquelles vous faites référence.
2. Utilisez un langage naturel : nul besoin de mémoriser les noms des outils ou les paramètres; décrivez simplement ce que vous voulez accomplir.
3. Vérifiez avant de confirmer : lorsque Claude propose des modifications, examinez-les attentivement avant de confirmer.
4. Commencez simplement : commencez par des requêtes simples pour vous familiariser avec la manière dont Claude interprète vos demandes.

<div id="troubleshooting">
  ## Dépannage
</div>

Si Claude ne parvient pas à effectuer une opération, tenez compte des points suivants :

1. Authentification : Assurez-vous d’être connecté au moyen de l’Auth0 CLI (`auth0 login`).
2. Permissions : Vérifiez que votre compte dispose des permissions nécessaires dans Auth0.
3. Précision : Essayez d’être plus précis dans votre demande.
