title: Client Credentials Flow
---

Le Client Credentials Flow (défini dans [OAuth 2.0 RFC 6749, section 4.4](https://tools.ietf.org/html/rfc6749#section-4.4)) consiste pour une application à échanger ses identifiants d’application, comme le <Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">client ID</Tooltip> et le <Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">client secret</Tooltip>, contre un <Tooltip tip="Client Secret : secret utilisé par un client (application) pour s’authentifier auprès du Authorization Server; il ne doit être connu que du client et du Authorization Server et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip>.

Ce flow est particulièrement adapté aux applications Machine-to-Machine (M2M), comme les CLI, les démons ou les services backend, puisque le système doit authentifier et autoriser l’application plutôt qu’un utilisateur.

<div id="how-it-works">
  ## Fonctionnement
</div>

<Frame>![Flows - Client Credentials - Diagramme de séquence d’autorisation(w/Border)](/docs/images/cdy7uua7fh8z/4Ph562CAccmCUkKNXuEIlQ/47581872e40e87b2cca95aecf7f42c5b/diagram.png)</Frame>

1. L’application envoie ses identifiants à Auth0 Authorization Server. Pour en savoir plus sur les méthodes d’authentification du client, consultez [Application Credentials](/docs/fr-CA/secure/application-credentials).
2. Auth0 Authorization Server valide les identifiants de l’application.
3. Auth0 Authorization Server renvoie un jeton d’accès.
4. L’application peut utiliser le jeton d’accès pour appeler une API en son propre nom. Pour en savoir plus sur ce processus, consultez [Valider les JSON Web Tokens](/docs/fr-CA/secure/tokens/json-web-tokens/validate-json-web-tokens).
5. L’API renvoie les données demandées.

<div id="how-to-implement-it">
  ## Comment le mettre en œuvre
</div>

La façon la plus simple de mettre en œuvre le Client Credentials Flow est de suivre nos [Quickstarts backend](/docs/fr-CA/quickstart/backend).

Sinon, vous pouvez utiliser l’Auth0 Authentication API pour mettre en œuvre le Client Credentials Flow. Pour en savoir plus, consultez [Call Your API Using the Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).
