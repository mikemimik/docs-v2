title: Flux Client Credentials
---

Le flux Client Credentials (défini dans [OAuth 2.0 RFC 6749, section 4.4](https://tools.ietf.org/html/rfc6749#section-4.4)) permet à une application d’échanger ses identifiants d’application, comme le <Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">client ID</Tooltip> et le <Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">client secret</Tooltip>, contre un <Tooltip tip="Client Secret : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne devrait être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip>.

Ce flux convient particulièrement aux applications machine à machine (M2M), comme les CLI, les démons ou les services d’arrière-plan, parce que le système doit authentifier et autoriser l’application plutôt qu’un utilisateur.

<div id="how-it-works">
  ## Comment ça fonctionne
</div>

<Frame>![Flows - Identifiants du client - Diagramme de séquence d’autorisation (avec bordure)](/docs/images/cdy7uua7fh8z/4Ph562CAccmCUkKNXuEIlQ/47581872e40e87b2cca95aecf7f42c5b/diagram.png)</Frame>

1. L’application envoie ses identifiants au serveur d’autorisation Auth0. Pour en savoir plus sur les méthodes d’authentification du client, consultez [Identifiants de l’application](/docs/fr-CA/secure/application-credentials).
2. Le serveur d’autorisation Auth0 valide les identifiants de l’application.
3. Le serveur d’autorisation Auth0 renvoie un jeton d’accès.
4. L’application peut utiliser le jeton d’accès pour appeler une API en son propre nom. Pour en savoir plus sur ce processus, consultez [Valider les JSON Web Tokens](/docs/fr-CA/secure/tokens/json-web-tokens/validate-json-web-tokens).
5. L’API renvoie les données demandées.

<div id="how-to-implement-it">
  ## Comment l’implémenter
</div>

Le moyen le plus simple d’implémenter le flux Client Credentials consiste à suivre nos [Quickstarts backend](/docs/fr-CA/quickstart/backend).

Vous pouvez aussi utiliser l’Auth0 Authentication API pour implémenter le flux Client Credentials. Pour en savoir plus, consultez [Appeler votre API à l’aide du flux Client Credentials](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).
