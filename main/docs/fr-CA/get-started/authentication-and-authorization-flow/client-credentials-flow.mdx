title: Client Credentials Flow
---

Le Client Credentials Flow (défini dans [OAuth 2.0 RFC 6749, section 4.4](https://tools.ietf.org/html/rfc6749#section-4.4)) permet à une application d’échanger ses informations d’identification, comme le <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">client ID</Tooltip> et le <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">secret client</Tooltip>, contre un <Tooltip tip="Secret client : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip>.

Ce flux convient particulièrement aux applications machine à machine (M2M), comme les CLI, les démons ou les services backend, car le système doit authentifier et autoriser l’application plutôt qu’un utilisateur.

<div id="how-it-works">
  ## Fonctionnement
</div>

<Frame>![Flows - Client Credentials - diagramme de séquence d’autorisation (avec bordure)](/docs/images/cdy7uua7fh8z/4Ph562CAccmCUkKNXuEIlQ/47581872e40e87b2cca95aecf7f42c5b/diagram.png)</Frame>

1. L’Application envoie ses identifiants à l’Auth0 Authorization Server. Pour en savoir plus sur les méthodes d’authentification du client, consultez [Application Credentials](/docs/fr-CA/secure/application-credentials).
2. L’Auth0 Authorization Server valide les identifiants de l’Application.
3. L’Auth0 Authorization Server renvoie un jeton d’accès.
4. L’Application peut utiliser le jeton d’accès pour appeler une API en son propre nom. Pour en savoir plus sur ce processus, consultez [Validate JSON Web Tokens](/docs/fr-CA/secure/tokens/json-web-tokens/validate-json-web-tokens).
5. L’API renvoie les données demandées.

<div id="how-to-implement-it">
  ## Comment l’implémenter
</div>

La façon la plus simple d’implémenter le Client Credentials Flow consiste à suivre nos [Quickstarts backend](/docs/fr-CA/quickstart/backend).

Vous pouvez également utiliser l’Auth0 Authentication API pour implémenter le Client Credentials Flow. Pour en savoir plus, consultez [Appelez votre API à l’aide du Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).
