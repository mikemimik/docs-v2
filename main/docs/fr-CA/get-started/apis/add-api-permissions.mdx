Vous pouvez ajouter des permissions à une API à l’aide du <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<Warning>
  Par défaut, tout utilisateur de toute application peut demander n’importe quelle permission définie ici. Vous pouvez mettre en place des politiques d’accès pour limiter ce comportement au moyen de [rules](/docs/fr-CA/customize/rules).
</Warning>

<div id="use-the-dashboard">
  ## Utiliser le Dashboard
</div>

1. Accédez à [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis), puis cliquez sur le nom de l’API à consulter.

   <Frame>![Liste des API dans Dashboard Applications](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>
2. Accédez à l’onglet **Permissions**, puis entrez un nom et une description pour la permission que vous souhaitez ajouter. Assurez-vous de ne pas utiliser de noms de permission réservés (voir la section Noms réservés).

   <Frame>![Écran Dashboard Add API Permissions API Define Permissions](/docs/images/cdy7uua7fh8z/32Pb185OFs2mC6z2fmunEw/2708c52c2869d016066cc456dd00b6a2/dashboard-applications-apis-permissions.png)</Frame>
3. Cliquez sur **Add**. N’oubliez pas que certaines Applications peuvent nécessiter une mise à jour des permissions et/ou des scopes pour interagir correctement avec l’API.

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Mettre à jour les permissions avec un objet vide supprime complètement les permissions.
</Callout>

Effectuez une requête `PATCH` vers l’[endpoint Update Resource Server](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id). Assurez-vous de remplacer les valeurs de l’espace réservé `API_ID`, `MGMT_API_ACCESS_TOKEN`, `PERMISSION_NAME` et `PERMISSION_DESC` par votre ID d’API, votre <Tooltip tip="Jeton d’accès : information d’authentification, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, le ou les noms de permission et la ou les descriptions de permission, respectivement. Assurez-vous de ne pas utiliser de noms de permission réservés (consultez la section Noms réservés).

    --header 'content-type: application/json' \
    --data '{ "scopes": [ { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" }, { "value": "PERMISSION_NAME", "description": "PERMISSION_DESC" } ] }'

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Lorsque vous ajoutez ou mettez à jour des scopes, le Management API exige que vous transmettiez tous les scopes à inclure. Si certains scopes existants ne sont pas transmis, ils seront supprimés.
</Callout>

| Valeur                  | Description                                                                                                                     |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `API_ID`                | ID de l’API pour laquelle vous voulez supprimer des permissions.                                                                |
| `MGMT_API_ACCESS_TOKEN` | [jeton d’accès for the Management API](https://auth0.com/docs/api/management/v2/tokens) avec le scope `update:resource_servers`. |
| `PERMISSION_NAME`       | Nom(s) de la ou des permissions que vous voulez conserver pour l’API spécifiée.                                                 |
| `PERMISSION_DESC`       | Description(s) conviviale(s) de la ou des permissions que vous voulez conserver pour l’API spécifiée.                           |

<div id="reserved-names">
  ## Noms réservés
</div>

Les noms de permission suivants sont réservés et ne peuvent pas être définis comme des permissions d’API personnalisées :

* address
* created&#95;at
* email
* email&#95;verified
* family&#95;name
* given&#95;name
* identities
* name
* nickname
* offline&#95;access
* <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker leurs informations de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=openid">openid</Tooltip>
* phone
* picture
* profile
