---

Utilisez l’onglet **Settings** dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, à [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis), pour configurer les API enregistrées auxquelles vos applications autorisées peuvent accéder. Pour configurer les paramètres d’une API, cliquez sur **...** à côté de l’API dans la liste et sélectionnez **Settings**, ou cliquez sur le nom de l’API. Pour savoir comment créer et enregistrer une API, consultez [Enregistrer des API](/docs/fr-CA/get-started/auth0-overview/set-up-apis).

<Frame>![Liste des API dans Dashboard > Applications > APIs](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>

<div id="settings">
  ## Settings
</div>

Utilisez les paramètres de cet onglet pour configurer l’expiration du jeton, le contrôle d’accès en fonction des rôles (RBAC) et d’autres paramètres d’accès. Cliquez sur **Enregistrer** au bas de l’onglet pour enregistrer les modifications.

<div id="general-settings">
  ### General Settings
</div>

Ces champs ont été définis au moment de l’enregistrement initial de l’API, sauf dans le cas de la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> d’Auth0. Vous pouvez uniquement modifier le **Name**.

<Frame>![Onglet Settings généraux des Settings des API des applications du Dashboard](/docs/images/cdy7uua7fh8z/QCaXd3FSeP6SYqLfEJWBr/fbb1bb5b2e12fb099073c33fb39cf44a/dashboard-applications-apis-settings-general-settings.png)</Frame>

* **Id** : Une chaîne alphanumérique unique générée par Auth0. Ce renseignement est en lecture seule et vous n’en aurez besoin que si vous travaillez directement avec les [points de terminaison Resource Servers de la Management API d’Auth0](https://auth0.com/docs/api/management/v2#!/Resource_Servers/get_resource_servers_by_id).
* **Name** : Un nom convivial pour l’API. N’affecte aucune fonctionnalité. Les caractères suivants ne sont pas autorisés : `< >`.
* **Identifier** : Un identifiant unique pour votre API. Cette valeur est définie lors de la création de l’API et ne peut pas être modifiée par la suite. Nous recommandons d’utiliser une URL, mais elle n’a pas besoin d’être accessible publiquement; Auth0 n’appellera jamais votre API.

<div id="token-settings">
  ### Paramètres des jetons
</div>

<Frame>![Dashboard - API - Paramètres des jetons - Expiration](/docs/images/cdy7uua7fh8z/2FFfqqmntXL9CnGrSxea9M/810e5542553168ffcd6081a10bc2db2f/image2.png)</Frame>

* **Durée de vie maximale du <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> (secondes)** : La durée (en secondes) pendant laquelle un jeton d’accès reste valide avant d’expirer. La valeur par défaut est de 86400 secondes (24 heures). La valeur maximale que vous pouvez définir est de 2592000 secondes (30 jours).
* **Durée de vie du jeton d’accès pour le flux implicite / hybride (secondes)** : La durée (en secondes) pendant laquelle un jeton d’accès émis au moyen d’un flux implicite ou hybride reste valide avant d’expirer. La valeur par défaut est de 86400 secondes (24 heures). Cette valeur ne peut pas dépasser la durée de vie maximale du jeton d’accès.
* **Profil <Tooltip tip="JSON Web Token (JWT) : format standard de ID Token (et souvent de jeton d’accès) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT)** : Le profil détermine le format des jetons d’accès émis pour l’API. Les valeurs offertes sont `Auth0` et `RFC 9068`. Pour en savoir plus, consultez [Profils de jetons d’accès](/docs/fr-CA/secure/tokens/access-tokens/access-token-profiles).
* **JSON Web Token (JWT) <Tooltip tip="Algorithme de signature : algorithme utilisé pour signer numériquement des jetons afin de garantir que le jeton n’a pas été altéré." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Signing+Algorithm">Algorithme de signature</Tooltip>** : L’algorithme utilisé pour signer les jetons. La signature sert à vérifier que l’expéditeur du JWT est bien celui qu’il prétend être et à garantir que le message n’a pas été modifié en cours de route. Les valeurs offertes sont [`HS256`](https://en.wikipedia.org/wiki/Symmetric-key_algorithm) et [`RS256`](https://en.wikipedia.org/wiki/Public-key_cryptography). Si vous sélectionnez `RS256` (recommandé), le jeton sera signé avec la clé privée de votre tenant. Cette valeur est définie lors de la création de votre API et ne peut pas être modifiée par la suite. Pour en savoir plus sur les algorithmes de signature et leur fonctionnement dans Auth0, consultez [Algorithmes de signature](/docs/fr-CA/get-started/applications/signing-algorithms).

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    La signature fait partie d’un JWT. Si la structure d’un JWT ne vous est pas familière, veuillez consulter [Structure d’un JSON Web Token](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-structure).
  </Callout>
* **JSON Web Encryption (JWE) :** Lorsqu’elle est activée, les jetons d’accès émis sont chiffrés au moyen du format JSON Web Encryption (JWE). Pour en savoir plus, consultez [JSON Web Encryption](/docs/fr-CA/secure/tokens/access-tokens/json-web-encryption).

<div id="rbac-settings">
  ### RBAC Settings
</div>

<Frame>![bascule RBAC des paramètres d’API dans Auth0 Dashboard](/docs/images/cdy7uua7fh8z/65tKb6aj0ktc2qXLUVlV3e/641adef615d6af9e5a3b588ff397af87/dashboard-apis-edit_view-settings_rbac-settings.png)</Frame>

* **Activer RBAC** : Activez ce paramètre pour appliquer les politiques RBAC à l’API. Pour en savoir plus, consultez [Role-Based Access Control](/docs/fr-CA/manage-users/access-control/rbac) et [Enable Role-Based Access Control for APIs](/docs/fr-CA/get-started/apis/enable-role-based-access-control-for-apis).
* **Ajouter les permissions au jeton d’accès** : Activez ce paramètre pour ajouter la revendication Permissions au jeton d’accès. Ce paramètre est offert uniquement si vous activez RBAC. Vous pouvez configurer les permissions dans l’onglet **Permissions**.

<div id="access-settings">
  ### Access Settings
</div>

<Frame>![Dashboard Applications API onglet Settings Paramètres d’accès](/docs/images/cdy7uua7fh8z/JNlWACqoyfmpE89JwWLKy/5a2b595f939167b0e6e1f72b485b08e7/2025-02-25_14-53-21.png)</Frame>

* **Autoriser l’omission du consentement de l’utilisateur** : Activez ce paramètre pour permettre à l’API de ne pas demander le consentement de l’utilisateur pour les applications marquées comme « first party ».
* **Autoriser l’accès hors ligne** : Activez ce paramètre pour permettre aux applications de demander des <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se connecter de nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+tokens">jetons de rafraîchissement</Tooltip> pour l’API.

<div id="default-policy-setting">
  #### Paramètre Default Policy
</div>

<Warning>
  Le niveau d’assurance de l’authentification pour la **My Account API** est actuellement en Early Access avec une politique à option unique. En utilisant cette fonctionnalité, vous acceptez les modalités applicables de l’essai gratuit dans le [Master Subscription Agreement](https://www.okta.com/legal) d’Okta. Pour en savoir plus sur le cycle de publication des produits d’Auth0, consultez [Product Release Stages](/docs/fr-CA/troubleshoot/product-lifecycle/product-release-stages). Pour participer au programme, communiquez avec [Auth0 Support](https://support.auth0.com).
</Warning>

<Frame>![Paramètre API Default Policy](/docs/images/api-settings-access-default-policy.png)</Frame>

* **Exiger la 2FA** : Lorsqu’elle est activée, Auth0 applique la Default Policy à la My Account API pour déclencher la [Step-Up Authentication](/docs/fr-CA/secure/multi-factor-authentication/step-up-authentication). Les utilisateurs doivent s’authentifier avec un second facteur dans les 15 minutes. Pour connaître le comportement complet et savoir comment l’activer, consultez [My Account API: Default Policy](/docs/fr-CA/manage-users/my-account-api#default-policy).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La Default Policy n’est pas offerte avec Classic Login.
</Callout>

<div id="permissions">
  ## Permissions
</div>

Utilisez les paramètres de l’onglet **Permissions** pour définir les permissions (scopes) utilisées par l’API. Pour en savoir plus, consultez [Ajouter des permissions d’API](/docs/fr-CA/get-started/apis/add-api-permissions) et [Supprimer des permissions d’API](/docs/fr-CA/get-started/apis/delete-api-permissions).

<div id="machine-to-machine-applications">
  ## Applications machine à machine
</div>

Si vous avez des applications machine à machine, elles figureront dans une liste sous cet onglet. Utilisez les bascules pour autoriser les applications de la liste. Pour en savoir plus, consultez [Enregistrer des applications machine à machine](/docs/fr-CA/get-started/auth0-overview/create-applications/machine-to-machine-apps).
