Les politiques d&#39;accès aux API pour les applications vous permettent de contrôler la façon dont les applications accèdent à vos API enregistrées dans Auth0. Ces politiques définissent comment les applications interagissent avec une API, notamment si elles peuvent obtenir un jeton d&#39;accès pour accéder aux ressources de l&#39;API.

Vous pouvez configurer la politique d&#39;accès à l&#39;API pour chaque application de chaque API enregistrée dans le Dashboard Auth0. Pour en savoir plus, consultez [Configurer la politique d&#39;accès de l&#39;application à l&#39;API](#configure-api-application-access-policy).

<div id="user-delegated-access-vs-client-access">
  ## Accès délégué à l’utilisateur vs. accès client
</div>

Vous pouvez configurer des politiques d’accès à l’API de l’application distinctes pour l’accès délégué à l’utilisateur et l’accès client (machine-to-machine) :

* **Accès client** : utilisé pour l’accès machine-to-machine, qui correspond au [flux d’identifiants du client](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow).
* **Accès délégué à l’utilisateur** : utilisé pour tous les flux d’accès qui génèrent un jeton d’accès associé à un utilisateur final, ce qui permet à l’application d’accéder à une API au nom de l’utilisateur. Les flux d’accès délégué à l’utilisateur n’incluent pas le flux d’identifiants du client. Pour en savoir plus sur les flux d’accès délégué à l’utilisateur, consultez [Flux d’authentification et d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow).

<div id="application-api-access-policies">
  ## Politiques d&#39;accès des applications aux API
</div>

Les politiques d&#39;accès des applications aux API sont les suivantes :

| Politique               | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           | Flux d&#39;accès                                                                                                                                                                                               |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `All apps allowed`      | Lorsqu&#39;elle est configurée pour une API, toute application de votre tenant peut obtenir un jeton d&#39;accès pour l&#39;API. Aucun octroi au client précis n&#39;est requis.                                                                                                                                                                                                                                                                                                                      | Il s&#39;agit de la valeur par défaut pour l&#39;accès délégué à l&#39;utilisateur lorsque vous créez une API. Vous pouvez uniquement configurer `allow_all` pour l&#39;accès délégué à l&#39;utilisateur. |
| `Per-app authorization` | Lorsqu&#39;elle est configurée pour une API, seules les applications pour lesquelles un octroi au client est défini peuvent obtenir un jeton d&#39;accès pour l&#39;API. L&#39;octroi au client établit les permissions maximales qu&#39;une application peut demander à l&#39;API. Pour en savoir plus sur la création et la gestion des octrois au client, consultez [Accès des applications aux API : octrois au client](/docs/fr-CA/get-started/applications/application-access-to-apis-client-grants). | Il s&#39;agit de la valeur par défaut pour le flux des identifiants du client lorsque vous créez une API.                                                                                                      |
| `No apps allowed`       | Lorsqu&#39;elle est configurée pour une API, aucune application ne peut obtenir de jeton d&#39;accès pour l&#39;API, quels que soient les autres paramètres ou octrois. L&#39;accès est entièrement restreint.                                                                                                                                                                                                                                                                                        | Vous pouvez configurer `deny_all` à la fois pour l&#39;accès utilisateur et l&#39;accès client.                                                                                                                |

Lors de la configuration de la politique d&#39;accès des applications d&#39;une API, Auth0 recommande d&#39;utiliser `Per-app authorization`, car cette option applique le principe du moindre privilège. Pour en savoir plus, consultez [Accès des applications aux API : octrois au client](/docs/fr-CA/get-started/applications/application-access-to-apis-client-grants).

Lorsque vous définissez la politique d&#39;accès des applications d&#39;une API sur `Per-app authorization`, vous devez fournir explicitement les portées requises dans la requête de jeton. Cela ne s&#39;applique pas aux requêtes de jeton d&#39;actualisation : si vous omettez les portées, le serveur d&#39;autorisation suppose que l&#39;application demande toutes les portées qui lui ont été accordées dans le jeton d&#39;accès d&#39;origine. Par conséquent, le serveur d&#39;autorisation renvoie un jeton d&#39;accès avec les mêmes portées que celles accordées initialement par le propriétaire de la ressource.

<div id="configure-api-application-access-policy">
  ## Configurer la politique d&#39;accès des applications à l&#39;API
</div>

Vous pouvez configurer la politique d&#39;accès des applications d&#39;une API à l&#39;aide de l&#39;Auth0 Dashboard ou de la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour configurer la politique d&#39;accès des applications d&#39;une API :

    1. Accédez à [Dashboard &gt;  Applications &gt; APIs](https://manage.auth0.com/#/apis) et sélectionnez votre API.
    2. Sélectionnez l&#39;onglet **Settings** et faites défiler jusqu&#39;à **Application Access Policy** pour configurer les politiques **User-Delegated Access** et **Client Access**.
       * Réglez la politique **User-Delegated Access** sur **All apps allowed**, **Per-app authorization** ou **No apps allowed**.
         * **All apps allowed** : Les applications sont autorisées à accéder à l&#39;API au nom de l&#39;utilisateur.
         * **Per-app authorization** : Les applications doivent disposer d&#39;un octroi au client pour accéder à l&#39;API au nom de l&#39;utilisateur.
         * **No apps allowed** : Les applications n&#39;ont pas accès à l&#39;API au nom de l&#39;utilisateur.
       * Réglez la politique **Client Access** sur **Per-app authorization** ou **No apps allowed**.
         * **Per-app authorization** : Les Machine-to-Machine Applications sont autorisées à accéder à cette API tant qu&#39;elles disposent de l&#39;octroi au client correspondant.
         * **No apps allowed** : Restreint l&#39;accès machine-to-machine à cette API.
    3. Sélectionnez **Save** pour enregistrer l&#39;**Application Access Policy**.

    Lorsque **Per-app authorization** est la politique configurée pour l&#39;API, sélectionnez l&#39;onglet **Application Access**, puis **Edit** pour autoriser **User-Delegated Access**, **Client Access** ou les deux pour chaque application.

    * Pour **User-Delegated Access**, sélectionnez **Grant Access**, puis les permissions souhaitées. Vous pouvez aussi sélectionner **Always grant all permissions**.
    * Pour **Client Access**, sélectionnez **Grant Access**, puis les permissions souhaitées. Vous pouvez aussi sélectionner **Always grant all permissions**. Pour **Organization Support**, sélectionnez :
      * **None** : L&#39;accès machine-to-machine ne peut pas être limité à une organization.
      * **Optional** : L&#39;accès machine-to-machine peut être limité à une organization.
      * **Required** : L&#39;accès machine-to-machine doit être limité à une organization.
  </Tab>

  <Tab title="Management API">
    Vous pouvez configurer la politique d&#39;accès des applications d&#39;une API en mettant à jour sa propriété `subject_type_authorization` dans la collection `resource-servers` de la [Management API](/docs/fr-CA/api/management/v2/resource-servers/patch-resource-servers-by-id).

    L&#39;objet `subject_type_authorization` contient deux objets imbriqués, `user` et `client`, chacun avec un attribut de politique que vous pouvez définir sur l&#39;une des trois politiques d&#39;accès décrites dans [Application API access policies](#application-api-access-policies).

    **API existante**

    Pour configurer la politique d&#39;accès des applications d&#39;une API existante, effectuez une request `PATCH` vers le endpoint `/resource-servers/{id}` :

    ```bash lines
    curl --location --request PATCH 'https://{yourDomain}/api/v2/resource-servers/{RESOURCE_SERVER_ID}' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_TOKEN}' \
    --data '{
        "subject_type_authorization": {
            "user": {
                "policy": "require_client_grant"
            },
            "client": {
                "policy": "deny_all"
            }
        }
    }'
    ```

    **Nouvelle API**

    Pour configurer la politique d&#39;accès des applications lors de la création d&#39;une nouvelle API, effectuez une request `POST` vers le endpoint `/resource-servers` :

    ```bash lines
    curl --location 'https://{yourDomain}/api/v2/resource-servers' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_TOKEN}' \
    --data '{
      "name": "{YOUR_NEW_API_NAME}",
      "identifier": "{YOUR_NEW_API_IDENTIFIER}",
      "scopes": [
        {
          "value": "{SCOPE_VALUE}",
          "description": "{SCOPE_DESCRIPTION}"
        }
      ],
      "subject_type_authorization": {
        "user": {
          "policy": "require_client_grant"
        },
        "client": {
          "policy": "deny_all"
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="api-access-policies-and-third-party-applications">
  ## Politiques d&#39;accès à l&#39;API et applications tierces
</div>

Les [applications tierces](/docs/fr-CA/get-started/applications/third-party-applications) exigent toujours un octroi au client explicite pour accéder à une API, quelle que soit la politique d&#39;accès configurée pour l&#39;API.

| **Politique d&#39;accès à l&#39;API** | **Applications de première partie** | **Applications tierces**       |
| ------------------------------------- | ----------------------------------- | ------------------------------ |
| **Tout autoriser**                    | Accès accordé                       | Nécessite un octroi au client |
| **Exiger un octroi au client**        | Nécessite un octroi au client       | Nécessite un octroi au client |
| **Refuser**                           | Accès refusé                        | Accès refusé                   |

Vous pouvez aussi configurer les [permissions par défaut pour toutes les applications tierces](/docs/fr-CA/get-started/applications/third-party-applications/configure-third-party-applications#default-permissions-for-all-third-party-applications) :

1. Accédez à **Applications &gt; APIs** et sélectionnez l&#39;API.
2. Sous l&#39;onglet **Settings**, accédez à **Default Permissions for Third Party Applications**.
3. Sélectionnez **Authorized** pour **User-delegated Access** ou **Client Access**, puis sélectionnez les scopes à accorder.
4. Sélectionnez **Save**.

Pour en savoir plus, consultez [Configurer des politiques d&#39;accès à l&#39;API pour les applications tierces](/docs/fr-CA/get-started/applications/third-party-applications/configure-third-party-applications#configure-api-access-policies).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Certains tenants existants peuvent avoir des applications tierces dont le comportement en matière de politique d&#39;accès diffère. Pour en savoir plus, consultez [Permissive Mode for Third-Party Applications](/docs/fr-CA/get-started/applications/third-party-applications/permissive-mode).
</Callout>
