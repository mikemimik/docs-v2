API Access Policies for Applications vous permet de contrôler la manière dont les applications accèdent à vos API enregistrées dans Auth0. Ces politiques définissent la façon dont les applications interagissent avec une API, notamment si elles peuvent obtenir avec succès un jeton d’accès pour accéder aux ressources de l’API.

Vous pouvez configurer la politique d’accès à l’API de l’application pour chaque API enregistrée dans Auth0 Dashboard. Pour en savoir plus, consultez [Configurer la politique d’accès à l’API de l’application](#configure-api-application-access-policy).

<div id="user-delegated-access-vs-client-access">
  ## Accès délégué par l’utilisateur vs. accès client
</div>

Vous pouvez configurer des politiques d’accès à l’API de l’application distinctes pour l’accès délégué par l’utilisateur et l’accès client (machine-to-machine) :

* **Accès client** : utilisé pour l’accès machine-to-machine, qui correspond au [Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow).
* **Accès délégué par l’utilisateur** : utilisé pour tous les flux d’accès qui génèrent un jeton d’accès associé à un utilisateur final et permettent à l’application d’accéder à une API au nom de l’utilisateur. Les flux d’accès délégué par l’utilisateur n’incluent pas le Client Credentials Flow. Pour en savoir plus sur les flux d’accès délégué par l’utilisateur, consultez les [flux d’authentification et d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow).

<div id="application-api-access-policies">
  ## Politiques d&#39;accès des applications aux API
</div>

Les politiques d&#39;accès des applications aux API sont les suivantes :

| Politique               | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | Flux d&#39;accès                                                                                                                                                                                               |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `All apps allowed`      | Lorsqu&#39;elle est configurée pour une API, toute application de votre tenant peut obtenir un jeton d&#39;accès pour l&#39;API. Aucune autorisation particulière n&#39;est requise.                                                                                                                                                                                                                                                                                            | Il s&#39;agit de la valeur par défaut pour l&#39;accès délégué par l&#39;utilisateur lorsque vous créez une API. Vous pouvez uniquement configurer `allow_all` pour l&#39;accès délégué par l&#39;utilisateur. |
| `Per-app authorization` | Lorsqu&#39;elle est configurée pour une API, seules les applications pour lesquelles un client grant est défini peuvent obtenir un jeton d&#39;accès pour l&#39;API. Le client grant définit les autorisations maximales qu&#39;une application peut demander à l&#39;API. Pour en savoir plus sur la création et la gestion des client grants, consultez [Application Access to APIs: Client Grants](/docs/fr-CA/get-started/applications/application-access-to-apis-client-grants). | Il s&#39;agit de la valeur par défaut pour le Client Credentials Flow lorsque vous créez une API.                                                                                                              |
| `No apps allowed`       | Lorsqu&#39;elle est configurée pour une API, aucune application ne peut obtenir de jeton d&#39;accès pour l&#39;API, peu importe les autres paramètres ou grants. L&#39;accès est entièrement restreint.                                                                                                                                                                                                                                                                        | Vous pouvez configurer `deny_all` pour l&#39;accès utilisateur et l&#39;accès client.                                                                                                                          |

Lors de la configuration de la politique d&#39;accès des applications d&#39;une API, Auth0 recommande d&#39;utiliser `Per-app authorization`, qui repose sur le principe du moindre privilège. Pour en savoir plus, consultez [Application Access to APIs: Client Grants](/docs/fr-CA/get-started/applications/application-access-to-apis-client-grants).

Lorsque vous définissez la politique d&#39;accès des applications d&#39;une API sur `Per-app authorization`, vous devez explicitement fournir les scopes requis dans la requête de jeton. Cela ne s&#39;applique pas aux requêtes de jeton de refresh token : si vous omettez les scopes, le serveur d&#39;autorisation suppose que l&#39;application souhaite tous les scopes qui lui ont été accordés dans le jeton d&#39;accès initial. Par conséquent, le serveur d&#39;autorisation renvoie un jeton d&#39;accès avec les mêmes scopes que ceux initialement accordés par le propriétaire de la ressource.

<div id="configure-api-application-access-policy">
  ## Configurer la politique d’accès des applications à l’API
</div>

Vous pouvez configurer la politique d’accès des applications d’une API à l’aide de l’Auth0 Dashboard ou de la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour configurer la politique d’accès des applications d’une API :

    1. Accédez à [Dashboard &gt;  Applications &gt; APIs](https://manage.auth0.com/#/apis) et sélectionnez votre API.
    2. Sélectionnez l’onglet **Settings** et faites défiler la page jusqu’à **Application Access Policy** pour configurer les politiques **User-Delegated Access** et **Client Access**.
       * Définissez la politique **User-Delegated Access** sur **Toutes les applications autorisées**, **Autorisation par application** ou **Aucune application autorisée**.
         * **Toutes les applications autorisées** : les applications sont autorisées à accéder à l’API au nom de l’utilisateur.
         * **Autorisation par application** : les applications doivent disposer d’un client grant pour accéder à l’API au nom de l’utilisateur.
         * **Aucune application autorisée** : les applications se voient refuser l’accès à l’API au nom de l’utilisateur.
       * Définissez la politique **Client Access** sur **Autorisation par application** ou **Aucune application autorisée**.
         * **Autorisation par application** : les applications machine-to-machine sont autorisées à accéder à cette API tant qu’elles disposent du client grant correspondant.
         * **Aucune application autorisée** : restreint l’accès machine-to-machine à cette API.
    3. Sélectionnez **Save** pour enregistrer la **Application Access Policy**.

    Lorsque **Autorisation par application** est la politique configurée pour l’API, sélectionnez l’onglet **Application Access**, puis **Edit** pour autoriser **User-Delegated Access**, **Client Access** ou les deux pour chaque application.

    * Pour **User-Delegated Access**, sélectionnez **Grant Access**, puis les permissions voulues. Vous pouvez aussi sélectionner **Always grant all permissions**.
    * Pour **Client Access**, sélectionnez **Grant Access**, puis les permissions voulues. Vous pouvez aussi sélectionner **Always grant all permissions**. Pour **Organization Support**, sélectionnez :
      * **None** : l’accès machine-to-machine ne peut pas être limité à une organization.
      * **Optional** : l’accès machine-to-machine peut être limité à une organization.
      * **Required** : l’accès machine-to-machine doit être limité à une organization.
  </Tab>

  <Tab title="Management API">
    Vous pouvez configurer la politique d’accès des applications d’une API en mettant à jour sa propriété `subject_type_authorization` dans la collection `resource-servers` de la [Management API](/docs/fr-CA/api/management/v2/resource-servers/patch-resource-servers-by-id).

    L’objet `subject_type_authorization` contient deux objets imbriqués, `user` et `client`, chacun avec un attribut `policy` que vous pouvez définir sur l’une des trois politiques d’accès décrites dans [Application API access policies](#application-api-access-policies).

    **API existante**

    Pour configurer la politique d’accès des applications pour une API existante, envoyez une requête `PATCH` au point de terminaison `/resource-servers/{id}` :

    ```bash lines
    curl --location --request PATCH 'https://{yourDomain}/api/v2/resource-servers/{RESOURCE_SERVER_ID}' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_TOKEN}' \
    --data '{
        "subject_type_authorization": {
            "user": {
                "policy": "require_client_grant"
            },
            "client": {
                "policy": "deny_all"
            }
        }
    }'
    ```

    **Nouvelle API**

    Pour configurer la politique d’accès des applications lors de la création d’une nouvelle API, envoyez une requête `POST` au point de terminaison `/resource-servers` :

    ```bash lines
    curl --location 'https://{yourDomain}/api/v2/resource-servers' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_TOKEN}' \
    --data '{
      "name": "{YOUR_NEW_API_NAME}",
      "identifier": "{YOUR_NEW_API_IDENTIFIER}",
      "scopes": [
        {
          "value": "{SCOPE_VALUE}",
          "description": "{SCOPE_DESCRIPTION}"
        }
      ],
      "subject_type_authorization": {
        "user": {
          "policy": "require_client_grant"
        },
        "client": {
          "policy": "deny_all"
        }
      }
    }'
    ```
  </Tab>
</Tabs>

<div id="api-access-policies-and-third-party-applications">
  ## Politiques d&#39;accès à l&#39;API et applications tierces
</div>

Les [applications tierces](/docs/fr-CA/get-started/applications/third-party-applications) exigent toujours un client grant explicite pour accéder à une API, quelle que soit la politique d&#39;accès configurée pour celle-ci.

| **Politique d&#39;accès à l&#39;API** | **Applications de première partie** | **Applications tierces** |
| ------------------------------------- | ----------------------------------- | ------------------------ |
| **Autoriser tout**                    | Accès accordé                       | Client grant requis      |
| **Exiger un client grant**            | Client grant requis                 | Client grant requis      |
| **Refuser**                           | Accès refusé                        | Accès refusé             |

Vous pouvez aussi configurer les [permission par défaut pour toutes les applications tierces](/docs/fr-CA/get-started/applications/third-party-applications/configure-third-party-applications#default-permissions-for-all-third-party-applications) :

1. Accédez à **Applications &gt; APIs** et sélectionnez l&#39;API.
2. Dans l&#39;onglet **Settings**, accédez à **Default Permissions for Third Party Applications**.
3. Sélectionnez **Authorized** pour **User-delegated Access** ou **Client Access**, puis sélectionnez les scopes à accorder.
4. Sélectionnez **Save**.

Pour en savoir plus, consultez [Configurer les politiques d&#39;accès à l&#39;API pour les applications tierces](/docs/fr-CA/get-started/applications/third-party-applications/configure-third-party-applications#configure-api-access-policies).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Certains tenants existants peuvent avoir des applications tierces dont le comportement en matière de politique d&#39;accès diffère. Pour en savoir plus, consultez [Mode permissif pour les applications tierces](/docs/fr-CA/get-started/applications/third-party-applications/permissive-mode).
</Callout>
