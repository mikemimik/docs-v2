title: Créer des applications machine à machine pour les tests
---

Lorsque vous créez un compte, une instance par défaut de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> est créée dans la section API de l’<Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Un exemple d’application machine à machine de test est automatiquement créé. Les instructions suivantes vous permettent de créer une autre application de test à utiliser avec cette instance de la Management API et d’utiliser le jeton de test généré avant de mettre en place votre propre configuration de Production.

Notez que le jeton de test situé sous API Explorer sert uniquement à tester l’accès à la Management API. Pour en savoir plus sur les jetons de la Management API, consultez [Obtenir des jetons d’accès à la Management API pour la production](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production). Si vous souhaitez utiliser le jeton de test, consultez [Obtenir des jetons d’accès à la Management API pour les tests](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-testing)

1. Accédez à [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) et sélectionnez **Auth0 Management API**.
2. Sélectionnez l’onglet **API Explorer**.
3. Cliquez sur **Create &amp; Authorize a Test Application**. Une nouvelle application est créée et autorisée à accéder à la Management API.

<Frame>![Dashboard Applications APIs Auth0 Management API Explorer Tab Authorize and Test](/docs/images/cdy7uua7fh8z/6G5M1JY9iYnkYXfDHwTvR/d09119af4f7d6d45c9b39001fd189d2f/dashboard-applications-apis-mgmt-api-authorize-and-test.png)</Frame>

L’application créée dans les étapes ci-dessus a obtenu toutes les portées de la Management API. Cela signifie qu’elle peut accéder à tous les endpoints à des fins de test. Cependant, les applications n’obtiennent généralement pas l’accès à toutes les portées, mais seulement à celles qui sont requises.

<Card title="Comment savoir quelles portées/autorisations sont requises pour la Management API ?">
  Chaque application machine à machine qui accède à une API doit se voir accorder un ensemble de [portées](/docs/fr-CA/get-started/apis/scopes/api-scopes). Les portées sont des autorisations qui doivent être accordées par le propriétaire. Chaque endpoint de [Auth0 Management API v2](https://auth0.com/docs/api/management/v2) exige des portées précises. Pour voir les portées/autorisations requises pour chaque endpoint, accédez au [Management API Explorer](https://auth0.com/docs/api/management/v2#!) et trouvez l’endpoint que vous souhaitez appeler. Chaque endpoint comporte une section appelée **Scopes** qui répertorie toutes les portées acceptées par cet endpoint.
</Card>

<div id="example-get-all-connections-endpoint">
  ## Exemple : point de terminaison Get All Connections
</div>

Le point de terminaison [Get All Connections](https://auth0.com/docs/api/management/v2#!/Clients/get_connections) accepte le scope `read:connections`, tandis que le point de terminaison [Create a Connection](https://auth0.com/docs/api/management/v2#!/Clients/post_connections) accepte le scope `write:connections`. Notre jeton machine à machine ne devrait avoir besoin que du scope `read:connections` pour accéder aux données de ce point de terminaison.

Si vous avez plusieurs applications qui doivent accéder à la Management API, vous devriez créer des applications machine à machine distinctes pour chaque application dans Auth0, plutôt qu’une seule application machine à machine.
