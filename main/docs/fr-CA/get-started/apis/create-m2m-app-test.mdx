title: Créer des applications machine-to-machine pour les tests
---

Lorsque vous créez un compte, une instance par défaut de la <Tooltip tip="Management API: Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> est créée dans la section API de l’<Tooltip tip="Management API: Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Une application machine-to-machine de test est également créée automatiquement. Les instructions suivantes vous permettent de créer une autre application de test à utiliser avec cette instance de la Management API et d’utiliser le jeton de test généré avant de mettre en place votre propre configuration de Production.

Veuillez noter que le jeton de test situé sous API Explorer sert uniquement à tester l’accès à la Management API. Pour en savoir plus sur les jetons de la Management API, consultez [Obtenir des jetons d’accès à la Management API pour la production](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production). Si vous souhaitez utiliser le jeton de test, consultez [Obtenir des jetons d’accès à la Management API pour les tests](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-testing)

1. Accédez à [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) et sélectionnez **Auth0 Management API**.
2. Sélectionnez l’onglet **API Explorer**.
3. Cliquez sur **Create &amp; Authorize a Test Application**. Une nouvelle application est créée et autorisée à accéder à la Management API.

<Frame>![Onglet API Explorer de Dashboard Applications APIs Auth0 Management API Autoriser et tester](/docs/images/cdy7uua7fh8z/6G5M1JY9iYnkYXfDHwTvR/d09119af4f7d6d45c9b39001fd189d2f/dashboard-applications-apis-mgmt-api-authorize-and-test.png)</Frame>

L’application créée dans les étapes ci-dessus s’est vu accorder tous les scopes de la Management API. Cela signifie qu’elle peut accéder à tous les points de terminaison à des fins de test. Toutefois, en règle générale, les applications n’ont pas accès à tous les scopes; elles ne sont autorisées qu’aux scopes requis.

<Card title="Comment puis-je savoir quels scopes/autorisations sont requis pour la Management API ?">
  Chaque application machine-to-machine qui accède à une API doit se voir accorder un ensemble de [Scopes](/docs/fr-CA/get-started/apis/scopes/api-scopes). Les scopes sont des autorisations qui doivent être accordées par le propriétaire. Chaque point de terminaison d’[Auth0 Management API v2](https://auth0.com/docs/api/management/v2) exige des scopes précis. Pour voir les scopes/autorisations requis pour chaque point de terminaison, accédez au [Management API Explorer](https://auth0.com/docs/api/management/v2#!) et repérez le point de terminaison que vous souhaitez appeler. Chaque point de terminaison comporte une section appelée **Scopes** qui répertorie tous les scopes acceptés par ce point de terminaison.
</Card>

<div id="example-get-all-connections-endpoint">
  ## Exemple : point de terminaison Get All Connections
</div>

Le point de terminaison [Get All Connections](https://auth0.com/docs/api/management/v2#!/Clients/get_connections) accepte le scope `read:connections`, tandis que le point de terminaison [Create a Connection](https://auth0.com/docs/api/management/v2#!/Clients/post_connections) accepte le scope `write:connections`. Notre jeton machine à machine ne devrait avoir besoin que du scope `read:connections` pour accéder aux données de ce point de terminaison.

Si vous avez plusieurs applications qui doivent accéder à la Management API, vous devriez créer des applications machine-to-machine distinctes pour chacune de vos applications dans Auth0, plutôt qu’une seule application machine-to-machine.
