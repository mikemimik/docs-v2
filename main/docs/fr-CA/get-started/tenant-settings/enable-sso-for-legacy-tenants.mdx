title: Activer l’authentification unique pour les tenants
---

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Uniquement pour les tenants Legacy. Par défaut, Seamless SSO est activé pour tous les nouveaux tenants Auth0; toutefois, les tenants Legacy peuvent choisir d’activer ou non cette fonctionnalité.
</Callout>

Vous pouvez activer le <Tooltip tip="Authentification unique (SSO) : service qui, après la connexion d’un utilisateur à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip> pour votre tenant dans l’<Tooltip tip="Authentification unique (SSO) : service qui, après la connexion d’un utilisateur à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Si vous choisissez de ne pas activer le SSO au niveau du tenant, vous pouvez [l’activer pour chaque application](/docs/fr-CA/get-started/applications/enable-sso-for-applications). Si ce paramètre n’apparaît pas dans le Dashboard, cela signifie que Seamless SSO est déjà activé; cette bascule est réservée aux tenants Legacy.

1. Accédez à [Dashboard &gt; Tenant Settings](https://manage.auth0.com/#/tenant).

   <Frame>![Dashboard Tenant Settings, onglet General Settings](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. Accédez à l’onglet **Advanced**.
3. Faites défiler jusqu’à la section **Log In Session Management**. Repérez **Enable Seamless SSO**.
4. Renseignez les champs **Inactivity timeout** (valeur par défaut : 4320 minutes) et **Require log in after** (valeur par défaut : 10080 minutes).

Une fois terminé, vous devriez aussi [configurer les paramètres de durée de vie de votre session](/docs/fr-CA/manage-users/sessions/configure-session-lifetime-settings).
