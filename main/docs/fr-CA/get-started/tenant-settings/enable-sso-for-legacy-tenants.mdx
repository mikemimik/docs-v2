title: Activer le Single Sign-On pour les tenants
---

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Uniquement pour les tenants Legacy. Par défaut, le Seamless Single Sign-On (SSO) est activé pour tous les nouveaux tenants Auth0; toutefois, les tenants Legacy peuvent choisir d’activer ou non cette fonctionnalité.
</Callout>

Vous pouvez activer le <Tooltip tip="Single Sign-On (SSO) : service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement aux autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip> pour votre tenant à l’aide de l’<Tooltip tip="Single Sign-On (SSO) : service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement aux autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Si vous choisissez de ne pas activer le SSO au niveau du tenant, vous pouvez [l’activer pour chaque application](/docs/fr-CA/get-started/applications/enable-sso-for-applications). Si ce paramètre n’est pas affiché dans le Dashboard, c’est que Seamless SSO est déjà activé; cette bascule est strictement réservée aux tenants Legacy.

1. Accédez à [Dashboard &gt; Tenant Settings](https://manage.auth0.com/#/tenant).

   <Frame>![Onglet Paramètres généraux des paramètres du tenant dans le Dashboard](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. Accédez à l’onglet **Advanced**.
3. Faites défiler jusqu’à la section **Log In Session Management**. Repérez **Enable Seamless SSO**.
4. Saisissez les valeurs des champs **Inactivity timeout** (4320 minutes par défaut) et **Require log in after** (10080 minutes par défaut).

Une fois terminé, vous devriez aussi [configurer vos paramètres de durée de vie de session](/docs/fr-CA/manage-users/sessions/configure-session-lifetime-settings).
