Dans le [scénario d’architecture Serveur + API](/docs/fr-CA/get-started/architecture-scenarios/server-application-api), nous avons présenté un scénario machine-to-machine simple : une API utilisée pour importer des entrées de feuille de temps dans les systèmes d’ExampleCo, ainsi qu’une tâche cron utilisée par des sous-traitants externes pour envoyer leurs feuilles de temps au moyen de cette API.

Nous avons vu le Client Credentials Grant et les <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+tokens">jetons d’accès</Tooltip>, comment configurer une API dans Auth0, comment configurer une Machine-to-Machine Application pour communiquer de façon sécurisée avec cette API, comment définir et sécuriser nos points de terminaison d’API, comment utiliser les bibliothèques fournies pour valider le jeton d’accès et comment en obtenir un nouveau auprès d’Auth0.

Nous avons utilisé Node.js pour la mise en œuvre de l’API et Python pour le processus serveur non interactif. Nous espérons que cet exemple vous donnera les moyens de créer votre propre mise en œuvre avec les technologies de votre choix.
