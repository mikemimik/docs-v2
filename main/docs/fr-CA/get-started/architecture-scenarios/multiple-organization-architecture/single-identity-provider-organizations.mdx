---

Dans le scénario des Organizations à <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d&#39;identité</Tooltip> (IdP) unique, chaque organisation qui utilise la fonctionnalité [Auth0 Organizations](/docs/fr-CA/manage-users/organizations/organizations-overview) utilise exactement une connexion Auth0. Cette fonctionnalité permet à chaque application de sélectionner l&#39;organisation appropriée définie dans le tenant Auth0, qui authentifiera directement les identifiants de l&#39;utilisateur ou transmettra la demande d&#39;authentification à l&#39;IdP approprié pour qu&#39;il s&#39;en charge.

Dans ce scénario, nous continuerons d&#39;utiliser l&#39;exemple [Travel0 Corporate Booking](/docs/fr-CA/get-started/architecture-scenarios/multiple-organization-architecture). Dans cet exemple, l&#39;application de réservation d&#39;entreprise correspond à votre application, et Travel0 à votre entreprise.

<Frame>![Scénarios d'architecture - Multitenance - Diagramme - Utilisateurs isolés, applications partagées, organisations A et B](/docs/images/cdy7uua7fh8z/1YBaMaMozamiTv8hXmNB2D/21e9270a59021a12209bdcb4345147cb/isolated-users_shared-apps_org-ab.png)</Frame>

<Warning>
  La fonctionnalité Auth0 Organizations prend actuellement en charge les applications qui utilisent les protocoles [SAML](/docs/fr-CA/authenticate/protocols/saml/saml-configuration), [Open ID Connect](/docs/fr-CA/authenticate/protocols/openid-connect-protocol) et/ou le framework [OAuth 2](/docs/fr-CA/authenticate/protocols/oauth). Les applications qui nécessitent [WS-Fed](/docs/fr-CA/authenticate/protocols/ws-fed-protocol) ne sont pas prises en charge.
</Warning>

Peu importe comment les identifiants sont traités — qu&#39;ils soient stockés dans une connexion de base de données (comme pour Hoekstra &amp; Associates), validés au moyen d&#39;une connexion d&#39;entreprise (comme pour MetaHexa Bank) ou d&#39;une connexion sociale — votre application se comporte de la même façon. En tirant parti de la fonctionnalité [Organizations](/docs/fr-CA/manage-users/organizations/organizations-overview), vous pouvez configurer et contrôler l&#39;expérience.
