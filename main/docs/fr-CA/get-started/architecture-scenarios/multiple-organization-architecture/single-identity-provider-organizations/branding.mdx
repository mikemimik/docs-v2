title: 'Fournisseur d’identité unique : Image de marque'
---

Les éléments d’image de marque associés à une organisation sont extrêmement précieux, car leur utilisation offre aux utilisateurs un environnement qu’ils connaissent et en lequel ils ont confiance. L’utilisation d’éléments d’image de marque reconnus renforce également la confiance des utilisateurs quant au traitement sûr et sécurisé des renseignements qu’ils fournissent (par exemple, leurs informations d’authentification). Dans nos scénarios d’architecture, nous proposons des conseils généraux sur plusieurs sujets liés à [l’image de marque B2B](/docs/fr-CA/get-started/architecture-scenarios/business-to-business/branding), que nous vous recommandons de consulter en complément des conseils fournis ici.

<div id="universal-login">
  ## Universal Login
</div>

L’image de marque d’un environnement à organisations multiples peut être plus complexe que celle d’un environnement standard, où il n’y a qu’un seul ensemble d’éléments d’image de marque à gérer. [Auth0 Organizations](/docs/fr-CA/manage-users/organizations/organizations-overview), toutefois, s’appuie sur l’[expérience Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), qui offre plusieurs [gabarits de page](/docs/fr-CA/customize/login-pages/universal-login/customize-templates) pour simplifier la personnalisation de l’image de marque. La configuration d’une Auth0 Organization permet aussi de [sélectionner une image de marque fondée sur un jeu de couleurs de base](/docs/fr-CA/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Page de connexion
</div>

Lorsque vous utilisez une [connexion de base de données ou de base de données personnalisée](/docs/fr-CA/authenticate/database-connections), la page de connexion sert à obtenir les identifiants de premier facteur d’un utilisateur. Vous pouvez configurer la page de connexion à l’aide d’un ensemble de [variables de modèle](/docs/fr-CA/manage-users/organizations/create-first-organization) qui facilitent la personnalisation par organisation.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pouvez configurer la page de connexion pour afficher les connexions Enterprise et Social pour l’authentification de premier facteur, de sorte que l’image de marque de la page de connexion reste pertinente dans ces cas.
</Callout>

<div id="password-reset-page">
  ### Page de réinitialisation du mot de passe
</div>

Comme la [page de connexion](#login-page), la page de réinitialisation du mot de passe peut être configurée dans les scénarios qui utilisent une connexion de base de données ou une connexion de base de données personnalisée. Dans les scénarios qui font intervenir une connexion d’entreprise ou une connexion sociale, les changements de mot de passe sont gérés à l’extérieur d’Auth0 par le <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) en amont; une page de réinitialisation du mot de passe Auth0 n’est donc jamais affichée.

<div id="multi-factor-authentication-pages">
  ### Pages de l’authentification multifacteur
</div>

<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip> (MFA) dans Auth0 est prise en charge pour toutes les méthodes servant à obtenir les identifiants de premier facteur d’un utilisateur (c.-à-d. une connexion de base de données ou une connexion de base de données personnalisée, une connexion d’entreprise ou une connexion sociale). Si vous utilisez le MFA, vous voudrez probablement personnaliser l’image de marque des pages de l’expérience <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> associées au MFA, ce qui peut se faire de façon semblable à l’image de marque de la [page de connexion](#login-page).

<div id="email-templates">
  ## Modèles de courriel
</div>

Comme pour les gabarits de page Universal Login, un ensemble de [variables de modèles de courriel](/docs/fr-CA/customize/email/email-templates/supported-liquid-syntax#general-variables) est disponible pour faciliter la personnalisation des courriels propres à chaque organisation. Les modèles de courriel à personnaliser varient selon la façon dont les utilisateurs sont authentifiés (autrement dit, au moyen d’une connexion de base de données ou de base de données personnalisée, d’une connexion d’entreprise ou d’une connexion sociale). Pour en savoir plus sur la personnalisation des modèles de courriel lors de l’utilisation de la fonctionnalité Auth0 Organizations, consultez [Créer votre première organisation : Modèles de courriel](/docs/fr-CA/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Domaines personnalisés
</div>

La fonctionnalité Auth0 Organizations ne prend en charge qu’un seul [domaine personnalisé](/docs/fr-CA/customize/custom-domains), défini au niveau du locataire Auth0. Par conséquent, nous vous recommandons d’utiliser un nom de <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Custom+Domain">domaine personnalisé</Tooltip> indépendant de toute organisation. Si vous devez absolument adopter une convention de nommage de domaine propre à chaque organisation, plusieurs locataires Auth0 devront alors être déployés en production. Comme l’utilisation de plusieurs locataires a des répercussions sur l’<Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO), la gestion des profils d’utilisateur, et ainsi de suite, vous devriez réfléchir attentivement avant d’emprunter cette voie.
