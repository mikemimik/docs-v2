title: 'Single Identity Provider : Image de marque'
---

Les éléments d’image de marque associés à une organisation sont extrêmement précieux, car ils offrent aux utilisateurs un environnement qu’ils connaissent et en lequel ils ont confiance. L’utilisation d’éléments d’image de marque reconnus renforce également la confiance des utilisateurs quant au fait que les renseignements qu’ils fournissent (par exemple, leurs identifiants) seront traités en toute sécurité. Dans nos scénarios d’architecture, nous fournissons des conseils généraux sur plusieurs sujets liés à [l’image de marque B2B](/docs/fr-CA/get-started/architecture-scenarios/business-to-business/branding), que nous vous recommandons de consulter en complément des conseils fournis ici.

<div id="universal-login">
  ## Universal Login
</div>

L’image de marque d’un environnement multi-organisation peut être plus complexe que celle d’un environnement standard, où vous n’avez qu’un seul ensemble d’éléments de marque à gérer. [Auth0 Organizations](/docs/fr-CA/manage-users/organizations/organizations-overview), toutefois, s’appuie sur l’[expérience Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), qui offre un certain nombre de [modèles de page](/docs/fr-CA/customize/login-pages/universal-login/customize-templates) pour simplifier la personnalisation de l’image de marque. La configuration d’une Auth0 Organization vous permet également de [sélectionner une image de marque de base fondée sur un jeu de couleurs](/docs/fr-CA/manage-users/organizations/configure-organizations/create-organizations/create-organizations).

<div id="login-page">
  ### Page de connexion
</div>

Lorsque vous utilisez une [connexion de base de données ou une connexion de base de données personnalisée](/docs/fr-CA/authenticate/database-connections), la page de connexion sert à recueillir les identifiants du premier facteur d’un utilisateur. Vous pouvez configurer la page de connexion à l’aide d’un ensemble de [variables de modèle](/docs/fr-CA/manage-users/organizations/create-first-organization) offertes pour faciliter la personnalisation pour chaque organisation.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pouvez configurer la page de connexion pour afficher des connexions Enterprise et sociales pour l’authentification du premier facteur, afin que l’image de marque de la page de connexion soit adaptée à ces cas.
</Callout>

<div id="password-reset-page">
  ### Page de réinitialisation du mot de passe
</div>

Comme la [page de connexion](#login-page), la page de réinitialisation du mot de passe peut être configurée pour les scénarios qui utilisent une connexion de base de données ou une connexion de base de données personnalisée. Pour les scénarios faisant appel à une connexion d’entreprise ou à une connexion sociale, les changements de mot de passe sont gérés à l’extérieur d’Auth0 par le <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) sous-jacent; une page de réinitialisation du mot de passe Auth0 n’est donc jamais affichée.

<div id="multi-factor-authentication-pages">
  ### Pages d’authentification multifacteur
</div>

<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui ajoute un facteur au nom d’utilisateur et au mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip> (MFA) dans Auth0 est prise en charge pour toutes les méthodes servant à obtenir les informations d’identification du premier facteur d’un utilisateur (c.-à-d. une connexion de base de données ou une connexion de base de données personnalisée, une connexion d’entreprise ou une connexion sociale). Si vous utilisez le MFA, vous voudrez probablement appliquer votre image de marque aux pages de l’expérience <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> associées au MFA, de façon semblable à l’image de marque de la [page de connexion](#login-page).

<div id="email-templates">
  ## Modèles de courriel
</div>

Comme pour les modèles de page Universal Login, un ensemble de [variables des modèles de courriel](/docs/fr-CA/customize/email/email-templates/supported-liquid-syntax#general-variables) est disponible pour faciliter la personnalisation des courriels propres à chaque organisation. Les modèles de courriel à personnaliser varient selon la façon dont les utilisateurs sont authentifiés (autrement dit, au moyen d’une connexion de base de données ou connexion de base de données personnalisée, d’une connexion d’entreprise ou d’une connexion sociale). Pour en savoir plus sur la personnalisation des modèles de courriel avec la fonctionnalité Auth0 Organizations, consultez [Créer votre première organisation : Modèles de courriel](/docs/fr-CA/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Domaines personnalisés
</div>

La fonctionnalité Auth0 Organizations ne prend en charge qu’un seul [domaine personnalisé](/docs/fr-CA/customize/custom-domains), défini au niveau du tenant Auth0. Par conséquent, nous vous recommandons d’utiliser un nom de <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou distinctif." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Custom+Domain">domaine personnalisé</Tooltip> qui soit indépendant de l’organisation. Si vous devez absolument adopter une convention de nommage de domaines propre à chaque organisation, plusieurs tenants Auth0 devront alors être déployés en production. Comme l’utilisation de plusieurs tenants a des répercussions sur l’<Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur se connecte à une application, connecte automatiquement cet utilisateur aux autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO), la gestion du profil utilisateur, et plus encore, vous devriez bien y réfléchir avant de vous engager dans cette voie.
