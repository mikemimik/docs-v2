title: 'Fournisseur d’identité unique : Image de marque'
---

Les éléments d’image de marque associés à une organisation sont extrêmement précieux, car leur utilisation offre aux utilisateurs un environnement qu’ils connaissent et auquel ils font confiance. L’utilisation d’éléments de marque reconnaissables renforce aussi la confiance des utilisateurs quant au fait que les renseignements qu’ils fournissent (par exemple, leurs identifiants) seront traités de façon sûre et sécuritaire. Dans nos scénarios d’Architecture, nous présentons des recommandations générales sur plusieurs sujets liés à l’[image de marque B2B](/docs/fr-CA/get-started/architecture-scenarios/business-to-business/branding), que nous vous recommandons de consulter en complément des conseils fournis ici.

<div id="universal-login">
  ## Universal Login
</div>

L’image de marque d’un environnement multi-organisation peut être plus complexe que celle d’un environnement standard, où il n’y a qu’un seul ensemble d’éléments de marque à gérer. [Auth0 Organizations](/docs/fr-CA/manage-users/organizations/organizations-overview), toutefois, s’appuie sur l’[expérience Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), qui propose plusieurs [gabarits de page](/docs/fr-CA/customize/login-pages/universal-login/customize-templates) pour simplifier la personnalisation de l’image de marque. La configuration d’une Auth0 organisation permet également de [sélectionner une image de marque reposant sur un jeu de couleurs de base](/docs/fr-CA/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Page de connexion
</div>

Lorsqu’une [connexion de base de données ou de base de données personnalisée](/docs/fr-CA/authenticate/database-connections) est utilisée, la page de connexion sert à recueillir les informations d’identification du premier facteur d’un utilisateur. Vous pouvez configurer la page de connexion à l’aide d’un ensemble de [variables de modèle](/docs/fr-CA/manage-users/organizations/create-first-organization) qui permettent une personnalisation par organisation.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pouvez configurer la page de connexion pour afficher les connexions Enterprise et Social pour l’authentification du premier facteur, afin que l’image de marque de la page de connexion soit adaptée à ces cas.
</Callout>

<div id="password-reset-page">
  ### Page de réinitialisation du mot de passe
</div>

Comme la [page de connexion](#login-page), la page de réinitialisation du mot de passe peut être configurée dans les scénarios qui utilisent une connexion de base de données ou une connexion à une base de données personnalisée. Dans les scénarios impliquant une connexion d’entreprise ou une connexion sociale, les changements de mot de passe sont gérés à l’extérieur d’Auth0 par le <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) en amont; par conséquent, une page de réinitialisation du mot de passe Auth0 n’est jamais affichée.

<div id="multi-factor-authentication-pages">
  ### Pages d’authentification multifacteur
</div>

<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip> (MFA) dans Auth0 est prise en charge pour toutes les méthodes utilisées pour obtenir les identifiants du premier facteur d’un utilisateur (c.-à-d. une connexion de base de données, une connexion de base de données personnalisée, une connexion d’entreprise ou une connexion sociale). Si vous utilisez la MFA, vous voudrez probablement personnaliser les pages de l’expérience <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> associées à la MFA, ce qui peut se faire de façon semblable à la personnalisation de la [page de connexion](#login-page).

<div id="email-templates">
  ## Modèles de courriel
</div>

Comme pour les gabarits de page Universal Login, un ensemble de [variables de modèle de courriel](/docs/fr-CA/customize/email/email-templates/supported-liquid-syntax#general-variables) est offert pour faciliter la personnalisation des courriels pour chaque organisation. Les modèles de courriel à personnaliser varient selon la façon dont les utilisateurs sont authentifiés (autrement dit, au moyen d’une connexion à une base de données ou d’une connexion à une base de données personnalisée, d’une connexion d’entreprise ou d’une connexion sociale). Pour en savoir plus sur la personnalisation des modèles de courriel lors de l’utilisation de la fonctionnalité Auth0 Organizations, consultez [Créer votre première organisation : Modèles de courriel](/docs/fr-CA/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Domaines personnalisés
</div>

La fonctionnalité Auth0 Organizations ne prend en charge qu’un seul [Custom Domain](/docs/fr-CA/customize/custom-domains), défini au niveau du tenant Auth0. Par conséquent, nous vous recommandons d’utiliser un nom de <Tooltip tip="Custom Domain : Domaine tiers portant un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Custom+Domain">Custom Domain</Tooltip> qui soit indépendant de l’organisation. Si vous devez absolument adopter une convention de nommage de domaine propre à chaque organisation, plusieurs tenants Auth0 devront être déployés en production. Comme l’utilisation de plusieurs tenants a des répercussions sur l’<Tooltip tip="Single Sign-On (SSO) : Service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO), la gestion du profil utilisateur, etc., vous devriez y réfléchir attentivement avant de vous engager dans cette voie.
