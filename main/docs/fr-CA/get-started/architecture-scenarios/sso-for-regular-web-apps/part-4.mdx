Dans ce tutoriel, nous avons présenté un scénario simple : une application web traditionnelle, hébergée dans le cloud, qui utilise Auth0 pour l’authentification, tout en s’appuyant sur le référentiel d’utilisateurs Active Directory existant. Nous avons vu ce qu’offre <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les informations de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) et pourquoi il convenait mieux à ce cas d’usage, comment fonctionne le flux d’authentification, ce qu’est un <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les informations de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+Token">ID Token</Tooltip> et comment le valider et le manipuler, comment configurer les applications et les connexions dans l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>, comment implémenter le login et le logout des utilisateurs à l’aide de Lock, ainsi que le fonctionnement de la gestion des sessions et du contrôle d’accès.

Nous avons commencé par décrire le cas d’usage et les exigences, puis nous avons expliqué comment chacune de ces exigences pouvait être satisfaite et le raisonnement qui sous-tend chacun des choix effectués.

Nous avons utilisé ASP .NET Core pour la mise en œuvre de l’exemple d’application web, mais nous espérons qu’après avoir parcouru ce document, vous serez en mesure de créer une telle application web à l’aide du framework de votre choix.
