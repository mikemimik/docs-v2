Accélérez votre workflow de développement en connectant directement des agents d’IA et des LLM à la base de connaissances d’Auth0. Obtenez des réponses adaptées au contexte, des exemples de code et des références d’API, avec des liens vers la documentation officielle.

Pour aider vos outils d’IA à indexer l’information dont vous avez besoin et à la rechercher, Auth0 fournit :

* fichiers de contexte pour les LLM (`llms.txt`) : des fichiers Markdown structurés pour l’ingestion en bloc.
* Auth0 Docs MCP Server : un MCP Server hébergé permettant aux agents d’IA d’interroger dynamiquement la base de connaissances.

<div id="llm-context-files">
  ## Fichiers de contexte pour les LLM
</div>

Utilisez les fichiers de contexte pour les LLM afin d’aider ces modèles à comprendre et à assimiler efficacement l’ensemble du répertoire du site Auth0.

| Nom             | URL                                    | Description                                                                      |
| --------------- | -------------------------------------- | -------------------------------------------------------------------------------- |
| `llms.txt`      | `https://auth0.com/llms.txt`           | Résumés concis de tous les articles ; idéal pour obtenir rapidement du contexte. |
| `llms-full.txt` | `https://auth0.com/docs/llms-full.txt` | Contenu complet de tous les articles ; idéal pour une indexation approfondie.    |

<div id="auth0-docs-mcp-server">
  ## Auth0 Docs MCP Server
</div>

Connectez vos outils d’IA au serveur Model Context Protocol (MCP) d’Auth0 Docs pour accéder en temps réel à la documentation Auth0 la plus récente. Le serveur Auth0 Docs MCP Server expose l’outil `SearchAuth0Docs`, qui permet aux agents IA d’effectuer des recherches dans la base de connaissances d’Auth0 et de renvoyer du contenu contextuel avec des liens pertinents vers la documentation du produit.

Le serveur Auth0 Docs MCP Server comporte les métadonnées d’endpoint suivantes :

```
Server Name: Auth0 Docs
Server URL: https://auth0.com/docs/mcp
Transport Type: HTTP
```

Découvrez comment connecter vos outils d’IA à l’Auth0 Docs MCP Server en suivant nos guides d’intégration.

<Tabs>
  <Tab title="ChatGPT">
    Pour connecter ChatGPT au Auth0 Docs MCP Server :

    1. Accédez à [Settings &gt; Connectors](https://chatgpt.com/#settings/Connectors).
    2. Sélectionnez **Create** et fournissez les métadonnées de votre serveur MCP :
       * Nom du connecteur : Auth0 Docs MCP Server
       * Description : Indexe et effectue des recherches dans la base de connaissances d’Auth0
       * URL du connecteur : `https://auth0.com/docs/mcp`
    3. Sélectionnez **Create**.

    Si tout fonctionne, le Auth0 Docs MCP Server devrait s’afficher comme serveur MCP installé. Pour en savoir plus, consultez la documentation d’OpenAI intitulée [Connect from ChatGPT](https://developers.openai.com/apps-sdk/deploy/connect-chatgpt/).
  </Tab>

  <Tab title="Claude">
    Pour connecter Claude au serveur MCP Auth0 Docs :

    1. Accédez à [Settings &gt; Connectors](https://claude.ai/settings/connectors).
    2. Sélectionnez **Add custom connector**.
    3. Saisissez les renseignements suivants pour ajouter le serveur MCP Auth0 Docs :
       * Nom : Auth0 Docs MCP Server
       * URL : `https://auth0.com/docs/mcp`
    4. Sélectionnez **Add**.

    Si tout fonctionne, le serveur MCP Auth0 Docs devrait s’afficher comme connecteur personnalisé. Pour en savoir plus, consultez la documentation de Model Context Protocol : [Connect to remote MCP Servers](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    Pour connecter Claude Code à Auth0 Docs MCP Server :

    Dans le terminal, exécutez :

    ```bash lines
    claude mcp add --transport http auth0-docs-mcp-server https://auth0.com/docs/mcp
    ```

    Pour vérifier, exécutez :

    ```bash lines
    claude mcp list
    ```

    Si l’installation réussit, le Auth0 Docs MCP Server devrait figurer parmi les serveurs MCP installés. Pour en savoir plus, consultez la documentation de Claude sur [l’installation des serveurs MCP](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Codex">
    Pour connecter Codex au serveur MCP Auth0 Docs :

    Dans votre terminal, exécutez :

    ```bash lines
    codex mcp add auth0-docs-mcp-server --url https://auth0.com/docs/mcp
    ```

    Pour vérifier, exécutez :

    ```bash lines
    codex mcp list
    ```

    Si tout se passe bien, Auth0 Docs MCP Server devrait figurer parmi les serveurs MCP installés. Pour en savoir plus, consultez la documentation de Codex sur le [Model Context Protocol](https://developers.openai.com/codex/mcp).
  </Tab>

  <Tab title="Cursor">
    Pour connecter Cursor au serveur MCP Auth0 Docs :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir la palette de commandes et rechercher **Open MCP settings**.
    2. Sélectionnez **New MCP Server** pour modifier votre `mcp.json`.
    3. Ajoutez le serveur MCP Auth0 Docs :

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "url": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    Si l’installation réussit, l’Auth0 Docs MCP Server devrait apparaître comme serveur MCP installé. Pour en savoir plus, consultez la documentation de Cursor sur [Installing MCP servers](https://cursor.com/docs/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    Pour connecter VS Code au Auth0 Docs MCP Server :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir la palette de commandes et rechercher **MCP: Add Server**.
    2. Sélectionnez **HTTP** comme type de serveur MCP.
    3. Entrez l’URL du serveur MCP (`https://auth0.com/docs/mcp`) et définissez le Server ID sur `auth0-docs-mcp-server`.
    4. Sélectionnez **Global** ou **Workspace** pour rendre le serveur MCP accessible dans tous vos espaces de travail ou dans votre espace de travail actuel.
    5. Appuyez sur **Enter** pour terminer la configuration. Assurez-vous que le Auth0 Docs MCP Server apparaît sous **Installed MCP Servers** dans vos **Extensions**.

    Pour en savoir plus, consultez [Add and manage MCP servers in VS Code](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>

  <Tab title="Windsurf">
    Pour connecter Windsurf au Auth0 Docs MCP Server :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir les paramètres de Windsurf et rechercher **MCP**.
    2. Sélectionnez **View Raw Config** pour modifier votre `mcp_config.json`.
    3. Ajoutez le Auth0 Docs MCP Server :

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "serverUrl": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    4. Enregistrez, puis sélectionnez **Actualiser** pour charger la nouvelle configuration MCP.

    Pour vérifier, posez la question `What MCP servers do you have installed?` dans le chat de Cascade. Cascade devrait afficher Auth0 Docs MCP Server parmi les serveurs MCP installés. Pour en savoir plus, consultez la documentation [Model Context Protocol](https://docs.windsurf.com/plugins/cascade/mcp#model-context-protocol-mcp) de Windsurf.
  </Tab>
</Tabs>

  ### Agents d’IA et serveurs MCP
</div>

* Ajoutez à mon agent d’IA un appel d’outil qui peut accéder au Google Calendar de l’utilisateur et répondre à des questions sur son horaire et ses réunions. Utilisez l’Auth0 Docs MCP Server pour obtenir des instructions sur l’utilisation d’Auth0 afin de sécuriser la connexion et d’implémenter l’appel d’outil au Google Calendar de l’utilisateur.
* Agissez comme un ingénieur en sécurité. Utilisez l’Auth0 Docs MCP Server pour trouver le guide sur l’autorisation de votre serveur MCP. Mettez à jour la configuration actuelle de mon serveur pour exiger OAuth 2.1 pour toutes les connexions entrantes et vérifiez qu’elle utilise le profil de compatibilité des paramètres de ressource pour les jetons d’accès.
* Examinez mes définitions d’outils actuelles dans `server.py`. Utilisez l’Auth0 Docs MCP Server pour vérifier si ma mise en œuvre de l’Échange de jetons personnalisé est conforme à la RFC 8693. Si vous trouvez des écarts susceptibles d’exposer le contexte utilisateur, suggérez un correctif et fournissez le code mis à jour.

<div id="troubleshooting">
  ### Dépannage
</div>

* Aidez-moi à résoudre une erreur `refresh token doesn’t exist` lors de l’utilisation du refresh token exchange. Utilisez les instructions de configuration de Token Vault comme contexte.
* Le bouton de logout n’efface pas la session côté Auth0. Recherchez dans la documentation la structure correcte de l’URL de Federated Logout et mettez à jour mon handler de logout pour l’utiliser.
* Mon login flow échoue avec une `401 Unauthorized error`. Utilisez Auth0 Docs MCP Server pour rechercher les causes courantes de cette erreur avec le SDK `auth0-node` et vérifiez s’il y a des incohérences dans mon `.env` et l’initialisation du client.
