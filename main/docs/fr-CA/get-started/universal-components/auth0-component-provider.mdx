`Auth0ComponentProvider` est la couche d’orchestration de [Auth0 Universal Components](/docs/fr-CA/get-started/universal-components/universal-components-overview). Les Auth0 SDKs gèrent les sessions et les jetons, et `Auth0ComponentProvider` convertit cet état d’identité en un contexte d’interface utilisateur fonctionnel et conforme à votre image de marque. `Auth0ComponentProvider` veille à ce que les composants, comme l’inscription à l’authentification multifacteur (MFA), disposent des autorisations nécessaires, des données mises en cache et du style visuel requis.

<div id="benefits">
  ## Avantages
</div>

`Auth0ComponentProvider` est l’enveloppe racine requise pour tous les Auth0 Universal Components. Voici les avantages de son utilisation :

* **Alignement des identités** : Comble l’écart entre les [Auth0 SDKs](/docs/fr-CA/libraries) et la [My Account API](/docs/fr-CA/manage-users/my-account-api) afin de garantir que les requêtes sont signées avec des jetons à portée utilisateur.
* **Optimisation des performances** : Implémente un cache **TanStack Query** partagé, spécialement optimisé pour les workflows d’identité, afin d’éviter les appels d’API redondants et les décalages de mise en page.
* **Cohérence du système de design** : Propage les variables **Tailwind CSS** et les thèmes compatibles avec **Shadcn** dans toute l’arborescence des composants.
* **Rétroaction centralisée** : Gère un système unifié de notifications toast pour les alertes de sécurité et l’état du workflow.

<div id="integration-architecture">
  ## Architecture d’intégration
</div>

Placez `Auth0ComponentProvider` dans votre fournisseur d’authentification (`Auth0Provider`). Cette configuration donne accès à l’état d’authentification et aux méthodes d’obtention de token nécessaires pour appeler les API Auth0.

<div id="configure-auth0componentprovider">
  ## Configurer `Auth0ComponentProvider`
</div>

<Tabs>
  <Tab title="React">
    ```tsx App.tsx
    import { Auth0Provider } from "@auth0/auth0-react";
    import { Auth0ComponentProvider } from "@auth0/universal-components-react/spa";

    function App() {
      return (
        <Auth0Provider
          domain="your-tenant.auth0.com"
          clientId="YOUR_CLIENT_ID"
          authorizationParams={{ redirect_uri: window.location.origin }}
          interactiveErrorHandler='popup' // Requis pour gérer les défis d'authentification par élévation de privilèges via le popup Universal Login
        >
          <Auth0ComponentProvider domain="your-tenant.auth0.com">
            {/* Votre application avec Auth0 Universal Components */}
          </Auth0ComponentProvider>
        </Auth0Provider>
      );
    }
    ```

    **toastSettings**

    Les paramètres Toast prennent en charge deux types de fournisseurs : **Sonner** (par défaut) ou **custom**. Chaque fournisseur possède sa propre structure de configuration pour une meilleure sécurité des types.

    <Tabs>
      <Tab title="Fournisseur Sonner (par défaut)">
        | Property               | Type            | Default       | Description                                                                                                                                                                               |
        | ---------------------- | --------------- | ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
        | `provider`             | `"sonner"`      | `"sonner"`    | Utilise la bibliothèque intégrée de toasts Sonner                                                                                                                                         |
        | `settings.position`    | `ToastPosition` | `"top-right"` | Emplacement où les toasts s’affichent : &quot;top-left&quot;, &quot;top-right&quot;, &quot;bottom-left&quot;, &quot;bottom-right&quot;, &quot;top-center&quot;, &quot;bottom-center&quot; |
        | `settings.duration`    | `number`        | `4000`        | Durée en millisecondes avant la fermeture automatique du toast (valeur par défaut de Sonner)                                                                                              |
        | `settings.maxToasts`   | `number`        | -             | Nombre maximal de toasts visibles à la fois                                                                                                                                               |
        | `settings.dismissible` | `boolean`       | `true`        | Indique si l’utilisateur peut fermer manuellement les toasts (valeur par défaut de Sonner)                                                                                                |
        | `settings.closeButton` | `boolean`       | `true`        | Indique s’il faut afficher un bouton de fermeture sur les toasts                                                                                                                          |

        ```tsx Sonner Provider Example
        const toastSettings = {
          provider: 'sonner', // Facultatif, c’est la valeur par défaut
          settings: {
            position: 'top-center',
            duration: 6000,
            maxToasts: 5,
            dismissible: true,
            closeButton: true
          }
        };
        ```

    ## État et performance

    Affinez la mise en cache de TanStack Query pour chaque composant Auth0. Par défaut, les données restent fraîches pendant deux minutes, sont supprimées après cinq minutes, et les récupérations au focus de la fenêtre sont ignorées.

    <table class="table">
      <thead>
        <tr>
          <th>Propriété</th>
          <th>Type</th>
          <th>Par défaut</th>
          <th>Description</th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            <code>enabled</code>
          </td>

          <td>
            <code>boolean</code>
          </td>

          <td>
            <code>true</code>
          </td>

          <td>
            Active ou désactive complètement la mise en cache. Lorsque cette option est définie à <code>false</code>, les données périmées ne sont plus utilisées
            et les entrées mises en cache sont rapidement supprimées.
          </td>
        </tr>

        <tr>
          <td>
            <code>staleTime</code>
          </td>

          <td>
            <code>number</code>
          </td>

          <td>
            <code>120000</code>
          </td>

          <td>
            Nombre de millisecondes avant que les données deviennent périmées. La valeur par défaut est de deux minutes. Augmentez-la
            pour les tableaux de bord, réduisez-la pour les flux de travail critiques.
          </td>
        </tr>

        <tr>
          <td>
            <code>gcTime</code>
          </td>

          <td>
            <code>number</code>
          </td>

          <td>
            <code>300000</code>
          </td>

          <td>
            Nombre de millisecondes avant que les requêtes inactives soient supprimées par le ramasse-miettes. La valeur par défaut est de cinq minutes.
          </td>
        </tr>

        <tr>
          <td>
            <code>refetchOnWindowFocus</code>
          </td>

          <td>
            <code>boolean | &quot;always&quot;</code>
          </td>

          <td>
            <code>false</code>
          </td>

          <td>
            Détermine si les requêtes sont relancées lorsque le navigateur reprend le focus. Utilisez{" "}
            <code>&quot;always&quot;</code> pour garantir une fraîcheur maximale.
          </td>
        </tr>
      </tbody>
    </table>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Désactiver la mise en cache :** passez `{ enabled: false }`. Cela règle automatiquement `staleTime` à 0 et réduit à cinq secondes la fenêtre de ramasse-miettes, de sorte que chaque rendu récupère des données fraîches.

      **Conseil de pro :** laissez la mise en cache activée, mais réduisez `staleTime` lors de l’intégration à des interfaces d’administration qui exigent des mises à jour quasi en temps réel.
    </Callout>

    <Tabs>
      <Tab title="Régler le TTL">
        ```tsx
        <Auth0ComponentProvider
          domain="your-tenant.auth0.com"
          cacheConfig={{
            staleTime: 10 * 60 * 1000,
            gcTime: 15 * 60 * 1000,
            refetchOnWindowFocus: true,
          }}
        >
          <App />
        </Auth0ComponentProvider>
        ```
      </Tab>

      <Tab title="Désactiver le cache">
        ```tsx
        <Auth0ComponentProvider
          domain="your-tenant.auth0.com"
          cacheConfig={{ enabled: false }}
        >
          <App />
        </Auth0ComponentProvider>
        ```

    Pour les applications Next.js, utilisez le mode proxy avec l&#39;authentification côté serveur :

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Désactiver la mise en cache :** passez `{ enabled: false }`. Cela définit automatiquement `staleTime` à 0 et réduit la fenêtre de garbage collection à 5 secondes, afin que chaque rendu récupère des données à jour.

      **Conseil de pro :** gardez la mise en cache activée, mais réduisez `staleTime` lors de l’intégration à des panneaux d’administration qui nécessitent des mises à jour quasi en temps réel.
    </Callout>
