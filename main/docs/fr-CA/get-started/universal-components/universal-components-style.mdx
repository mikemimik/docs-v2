Auth0 Universal Components repose sur un système de conception flexible qui utilise [Tailwind CSS](https://tailwindcss.com/docs/installation/using-vite) et les [primitives d’interface de Radix UI](https://www.radix-ui.com/primitives/docs/overview/introduction). La mise en forme est gérée par le [`Auth0ComponentProvider`](/docs/fr-CA/get-started/universal-components/auth0-component-provider), ce qui vous permet de choisir entre des préréglages de haut niveau ou des remplacements précis de variables CSS.

Cette architecture garantit que vos parcours d’identité s’intègrent naturellement à votre application.

<Tabs>
  <Tab title="React">
    ## Premiers pas

    1. Importez la feuille de style.

    ```tsx
    import "@auth0/universal-components-react/styles";
    ```

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Utilisateurs de Tailwind v4 :** ajoutez `@import "@auth0/universal-components-react/tailwind"` à votre fichier CSS.
    </Callout>

    2. Ajoutez la couleur de votre marque.

    ```css
    :root {
      --primary: #4f46e5; /* couleur de votre marque */
      --primary-foreground: #ffffff; /* texte sur les boutons */
    }
    ```

    Terminé. Tous les boutons, liens et états actifs utilisent désormais votre marque.

    ***

    ## Préréglages de thème

    Passez `themeSettings` à `Auth0ComponentProvider` pour alterner entre les thèmes intégrés.

    ```tsx App.tsx
    <Auth0ComponentProvider
      themeSettings={{
        mode: "light", // 'light' | 'dark'
        theme: "default", // 'default' | 'minimal' | 'rounded'
      }}
    >
      <App />
    </Auth0ComponentProvider>
    ```

    ## Composants de style

    Chaque composant accepte une prop `styling` pour des overrides ciblés sans affecter les styles globaux.

    ```tsx
    <SsoProviderTable
      styling={{
        variables: {
          common: {
            "--primary": "#059669", // remplacement uniquement pour ce composant
          },
          light: { "--card": "#f8fafc" },
          dark: { "--card": "#1e293b" },
        },
        classes: {
          "SsoProviderTable-header": "shadow-lg",
          "SsoProviderTable-table": "rounded-xl",
        },
      }}
    />
    ```

    **Quand utiliser :**

    * `variables` - remplacements de propriétés CSS personnalisées propres au composant
    * `classes` - classes Tailwind ou personnalisées appliquées à des parties précises du composant

    ## Personnalisations courantes

    ### Couleurs de marque (Hex)

    Convertissez la couleur hexadécimale de votre marque au format utilisé en CSS :

    ```css
    :root {
      /* Utilisation directe de l'hexadécimal */
      --primary: #4f46e5;
      --primary-foreground: #ffffff;

      /* Ou oklch pour une meilleure manipulation des couleurs */
      --primary: oklch(50% 0.2 265); /* violet */
    }
    ```

    ### Coins arrondis

    Pour une esthétique plus arrondie dans tous les composants :

    ```css
    :root {
      --radius-lg: 16px; /* boutons, champs de saisie */
      --radius-xl: 20px; /* cartes */
      --radius-2xl: 24px; /* fenêtres modales */
    }
    ```

    ### Typographie compacte

    Pour les interfaces plus denses :

    ```css
    :root {
      --font-size-page-header: 1.75rem;
      --font-size-heading: 1.25rem;
      --font-size-title: 1rem;
      --font-size-body: 0.875rem;
    }
    ```

    ***

    ## Mode sombre

    Les composants répondent automatiquement au paramètre `mode` dans `themeSettings`. Pour synchroniser avec le mode sombre de votre application :

    ```tsx
    function App() {
      const [isDark, setIsDark] = useState(false);

      return (
        <Auth0ComponentProvider
          themeSettings={{
            mode: isDark ? "dark" : "light",
          }}
        >
          <YourApp />
        </Auth0ComponentProvider>
      );
    }
    ```

    Ou utiliser les préférences du système :

    ```tsx
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;

    <Auth0ComponentProvider
      themeSettings={{ mode: prefersDark ? 'dark' : 'light' }}
    >
    ```

    ## Composants de style

    Chaque composant accepte une prop `styling` pour des surcharges ciblées sans affecter les styles globaux.

    ```tsx
    <SsoProviderTable
      styling={{
        variables: {
          common: {
            "--primary": "#059669", // remplacement uniquement pour ce composant
          },
          light: { "--card": "#f8fafc" },
          dark: { "--card": "#1e293b" },
        },
        classes: {
          "SsoProviderTable-header": "shadow-lg",
          "SsoProviderTable-table": "rounded-xl",
        },
      }}
    />
    ```

    **Quand utiliser :**

    * `variables` - redéfinitions de propriétés CSS personnalisées propres au composant
    * `classes` - classes Tailwind ou personnalisées appliquées à des parties précises du composant

    ## Personnalisations courantes

    ### Couleurs de marque (Hex)

    Convertissez la couleur hexadécimale de votre marque au format utilisé en CSS :

    ```css
    :root {
      /* Utilisation directe de l'hexadécimal */
      --primary: #4f46e5;
      --primary-foreground: #ffffff;

      /* Ou oklch pour une meilleure manipulation des couleurs */
      --primary: oklch(50% 0.2 265); /* violet */
    }
    ```

    ### Coins arrondis

    Pour une esthétique plus arrondie dans tous les composants :

    ```css
    :root {
      --radius-lg: 16px; /* boutons, champs de saisie */
      --radius-xl: 20px; /* cartes */
      --radius-2xl: 24px; /* fenêtres modales */
    }
    ```

    ### Typographie compacte

    Pour les interfaces plus denses :

    ```css
    :root {
      --font-size-page-header: 1.75rem;
      --font-size-heading: 1.25rem;
      --font-size-title: 1rem;
      --font-size-body: 0.875rem;
    }
    ```

    ## Mode sombre

    Les composants répondent automatiquement au paramètre `mode` dans `themeSettings`. Pour vous synchroniser avec le mode sombre de votre application :

    ```tsx
    function App() {
      const [isDark, setIsDark] = useState(false);

      return (
        <Auth0ComponentProvider
          themeSettings={{
            mode: isDark ? "dark" : "light",
          }}
        >
          <YourApp />
        </Auth0ComponentProvider>
      );
    }
    ```

    Ou utiliser les préférences du système :

    ```tsx
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;

    <Auth0ComponentProvider
      themeSettings={{ mode: prefersDark ? 'dark' : 'light' }}
    >
    ```
