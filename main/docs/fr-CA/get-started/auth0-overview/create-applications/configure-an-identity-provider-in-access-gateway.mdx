title: Configurer un fournisseur d’identité dans Access Gateway
---

Pour intégrer Auth0 à une application machine-to-machine (M2M), vous devez d’abord enregistrer votre application dans Auth0 à partir de l’<Tooltip tip="Auth0 Dashboard : interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.

2. Saisissez un nom descriptif pour votre application, sélectionnez **Machine to Machine Applications**, puis cliquez sur **Create**.

3. Sélectionnez la Management API pour autoriser la communication M2M, car c’est ainsi qu’OAG interagit avec Auth0.

4. Cliquez sur **Authorize**.

5. Chaque application M2M qui accède à une API doit recevoir un ensemble de permissions (ou de scopes), qui doivent être accordées par l’utilisateur authentifié. Sélectionnez les **Permissions** que vous souhaitez inclure dans le <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> de votre application. OAG exige les permissions suivantes pour le **client** et les **utilisateurs :**

* `read:client, write:client, update:client, delete:client`
* `read:users`

6. Cliquez sur **Authorize**.

7. Sélectionnez l’onglet **Credentials** de votre application.

8. Cliquez sur le **<Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+ID">Client ID</Tooltip>** et copiez-le dans un éditeur de texte.

9. Cliquez sur le **<Tooltip tip="Secret du client : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Secret">Client Secret</Tooltip>** et copiez-le dans un éditeur de texte.

10. Poursuivez avec la procédure [Configurer un fournisseur d’identité dans Access Gateway](/docs/Configure an Identity Provider in Access Gateway).
