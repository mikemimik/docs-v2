Pour intégrer Auth0 à une application Web monopage, vous devez d’abord enregistrer votre application dans Auth0 à l’aide du Dashboard.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Saisissez un nom descriptif pour votre application, sélectionnez **Single-Page Web Applications**, puis cliquez sur **Create**.

   Une fois la nouvelle application créée, vous pouvez consulter la page **Application Settings**, qui comprend les onglets suivants :

   | Onglet de paramètres | Description                                                                                                                                                                                                                                                                                                                                                                                                                                |
   | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Quick Start**      | Affiche toute la documentation disponible pour votre type d’application.                                                                                                                                                                                                                                                                                                                                                                   |
   | **Settings**         | Affiche tous les paramètres disponibles pour votre application. Par défaut, la plupart des paramètres sont créés pour vous. Pour en savoir plus, consultez [Application Settings](/docs/fr-CA/get-started/applications/application-settings).                                                                                                                                                                                                    |
   | **Credentials**      | Affiche la méthode d’authentification de l’application et les informations d’identification configurées. Pour en savoir plus, consultez [Application Credentials](/docs/fr-CA/secure/application-credentials)                                                                                                                                                                                                                                    |
   | **Add-ons**          | Vous permet d’activer les extensions associées à une application. Elles sont généralement utilisées pour les intégrations SAML ou WS-Fed, dont les SPA n’ont habituellement pas besoin. Pour en savoir plus, consultez [Enable SAML2 Web App Addon](/docs/fr-CA/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) et [Configure WS-Fed Applications](/docs/fr-CA/get-started/applications/update-application-connections). |
   | **Connections**      | Vous permet d’activer des connexions pour votre application. Les connexions sont des sources d’utilisateurs ; elles peuvent être activées et partagées entre plusieurs applications. Pour en savoir plus, consultez [Connections](/docs/fr-CA/authenticate/identity-providers).                                                                                                                                                                  |
   | **Organizations**    | Vous permet d’authentifier les utilisateurs dans les organisations auxquelles ils appartiennent. Pour en savoir plus, consultez [Organizations](/docs/fr-CA/manage-users/organizations).                                                                                                                                                                                                                                                         |
   | **Login Experience** | Vous permet de configurer l’expérience de connexion des utilisateurs dans votre organisation. Pour en savoir plus, consultez [Understand how Auth0 Organizations Work](/docs/fr-CA/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                                                          |

   Pour en savoir plus, consultez [Application Settings](/docs/fr-CA/get-started/applications/application-settings).

<div id="next-steps">
  ## Prochaines étapes
</div>

Une fois votre application enregistrée, passez en revue et complétez les étapes de configuration suivantes :

* Configurez au moins une connexion (comme une connexion de base de données ou une connexion sociale) et activez-la pour votre application. Sans connexion activée, les utilisateurs ne peuvent pas s’authentifier.
* Modifiez le code de votre application pour utiliser l’application que vous avez enregistrée dans Auth0. Consultez nos [Auth0 Quickstarts](/docs/fr-CA/quickstarts), où vous trouverez des instructions détaillées pour implémenter le login et le logout, gérer vos sessions utilisateur, récupérer et afficher les renseignements du profil utilisateur, et plus encore.
* Utilisez les [API Auth0](/docs/fr-CA/api).

  * L’[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les principales fonctions liées à l’identité (par exemple, le login, le logout et l’obtention du profil utilisateur). La plupart des utilisateurs utilisent cette API à l’aide de nos Quickstarts, du [Auth0 SPA SDK](/docs/fr-CA/libraries/auth0-spa-js) ou du [Lock widget](/docs/fr-CA/libraries/lock). Pour les applications Legacy, consultez la [auth0.js library](/docs/fr-CA/libraries/auth0js). Cependant, si vous créez manuellement toute votre interface utilisateur d’authentification, vous devrez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d’automatiser diverses tâches également accessibles à partir du Dashboard d’Auth0 (par exemple : créer des utilisateurs, configurer les types d’autorisation de l’application).
