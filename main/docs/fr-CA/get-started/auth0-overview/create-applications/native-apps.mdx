Pour intégrer Auth0 à une application native, vous devez d’abord enregistrer votre application dans Auth0 à l’aide du <Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Les applications natives devraient utiliser le flux de code d’autorisation avec PKCE pour une authentification sécurisée. Pour en savoir plus, consultez [Flux de code d’autorisation avec PKCE](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce). Il peut s’agir d’applications mobiles, de bureau ou hybrides qui s’exécutent nativement sur un appareil (par exemple, iOS, Android).

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Entrez un nom descriptif pour votre application, sélectionnez **Native**, puis cliquez sur **Create**.

   Une fois l’application créée, Auth0 vous redirige vers **Application Settings**, qui comprend les onglets suivants :

   | Onglet des paramètres | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
   | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Quick Start**       | Affiche toute la documentation offerte pour votre type d’application.                                                                                                                                                                                                                                                                                                                                                                                                    |
   | **Settings**          | Affiche tous les paramètres offerts pour votre application. Par défaut, la plupart des paramètres sont créés pour vous. Pour en savoir plus, consultez [Application Settings](/docs/fr-CA/get-started/applications/application-settings).                                                                                                                                                                                                                                      |
   | **Credentials**       | Affiche la méthode d’authentification de l’application et les informations d’identification configurées. Pour en savoir plus, consultez [Application Credentials](/docs/fr-CA/secure/application-credentials)                                                                                                                                                                                                                                                                  |
   | **Add-ons**           | Vous permet d’activer des modules d’extension associés à une application. Les add-ons sont principalement utilisés pour les intégrations SAML ou WS-Fed et ne s’appliquent généralement pas aux applications natives. Pour en savoir plus, consultez [Enable SAML2 Web App Addon](/docs/fr-CA/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) et [Configure WS-Fed Applications](/docs/fr-CA/get-started/applications/update-application-connections). |
   | **Connections**       | Vous permet d’activer des connexions pour votre application. Les connexions sont des sources d’utilisateurs; elles peuvent être activées et partagées entre plusieurs applications. Pour en savoir plus, consultez [Connections](/docs/fr-CA/authenticate/identity-providers).                                                                                                                                                                                                 |
   | **Organizations**     | Vous permet d’authentifier des utilisateurs dans les organisations auxquelles ils appartiennent. Pour en savoir plus, consultez [Organizations](/docs/fr-CA/manage-users/organizations).                                                                                                                                                                                                                                                                                       |
   | **Login Experience**  | Vous permet de configurer l’expérience de connexion pour les utilisateurs de votre Organization. Pour en savoir plus, consultez [Understand how Auth0 Organizations Work](/docs/fr-CA/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                                                                                     |

   Pour en savoir plus, consultez [Application Settings](/docs/fr-CA/get-started/applications/application-settings).
3. Si vous développez une application mobile, fournissez les paramètres iOS/Android nécessaires dans la section **Advanced Settings**, puis cliquez sur **Save Changes**.

   * Pour les applications iOS, [indiquez votre **Team ID** et votre **App Bundle Identifier**](/docs/fr-CA/get-started/applications/enable-universal-links-support-in-apple-xcode).
   * Pour les applications Android, [indiquez votre **App Package Name** et vos **Key Hashes**](/docs/fr-CA/get-started/applications/enable-android-app-links-support).

<div id="next-steps">
  ## Prochaines étapes
</div>

Une fois votre application enregistrée et configurée, voici quelques étapes courantes à suivre :

* Configurer une connexion et l’activer pour votre application.
* Modifier le code de votre application pour utiliser l’application enregistrée dans Auth0. Consultez nos [Auth0 Quickstarts](/docs/fr-CA/quickstarts), où vous trouverez des instructions détaillées et des exemples pour diverses technologies. Vous y apprendrez aussi à implémenter le login et le logout, à gérer les sessions utilisateur, à récupérer et à afficher les renseignement du profil utilisateur, et bien plus encore.
* Utiliser les [API Auth0](/docs/fr-CA/api).

  * L’[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les principales fonctions liées à l’identité (par exemple, le login, le logout et l’obtention du profil utilisateur). La plupart des utilisateurs utilisent cette API au moyen de nos Quickstarts, de la [bibliothèque Auth0.js](/docs/fr-CA/libraries/auth0js) ou du [widget Lock](/docs/fr-CA/libraries/lock). Toutefois, si vous concevez toute votre UI d’authentication manuellement, vous devrez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d’automatiser diverses tâches également accessibles dans le Dashboard d’Auth0 (par exemple : créer des utilisateurs, définir les types d’autorisation de l’application).
