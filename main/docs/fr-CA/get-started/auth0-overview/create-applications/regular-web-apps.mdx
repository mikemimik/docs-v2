Pour intégrer Auth0 à une application Web classique, vous devez d’abord enregistrer votre application dans Auth0 à l’aide du <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Les applications Web classiques sont des applications Web traditionnelles qui exécutent la majeure partie de leur logique sur le serveur (par exemple, Express.js, ASP.NET). Pour en savoir plus sur les types d’application, consultez [Application Types](/docs/fr-CA/get-started/applications).

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Entrez un nom descriptif pour votre application, sélectionnez **Regular Web Applications**, puis cliquez sur **Create**.

   Une fois la nouvelle application créée, vous pouvez afficher la page **Application Settings**, qui comprend les onglets suivants :

   | Onglet des paramètres | Description                                                                                                                                                                                                                                                                                                                                                                                                                   |
   | --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**       | Affiche toute la documentation offerte pour votre type d’application.                                                                                                                                                                                                                                                                                                                                                         |
   | **Settings**          | Affiche tous les paramètres offerts pour votre application. Par défaut, la plupart des paramètres sont créés pour vous. Pour en savoir plus, consultez [Application Settings](/docs/fr-CA/get-started/applications/application-settings).                                                                                                                                                                                           |
   | **Credentials**       | Affiche la méthode d’authentification de l’application et les informations d’identification configurées. Pour en savoir plus, consultez [Application Credentials](/docs/fr-CA/secure/application-credentials)                                                                                                                                                                                                                       |
   | **Add-ons**           | Vous permet d’activer les modules complémentaires associés à une application. Il s’agit d’applications Web SAML ou WS-Fed pour lesquelles Auth0 génère des jetons d’accès. Pour en savoir plus, consultez [Enable SAML2 Web App Addon](/docs/fr-CA/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) et [Configure WS-Fed Applications](/docs/fr-CA/get-started/applications/update-application-connections). |
   | **Connections**       | Vous permet d’activer des connexions pour votre application. Les connexions sont des sources d’utilisateurs; elles peuvent être activées et partagées entre plusieurs applications. Pour en savoir plus, consultez [Connections](/docs/fr-CA/authenticate/identity-providers).                                                                                                                                                      |
   | **Organizations**     | Vous permet d’authentifier les utilisateurs dans les organisations auxquelles ils appartiennent. Pour en savoir plus, consultez [Organizations](/docs/fr-CA/manage-users/organizations).                                                                                                                                                                                                                                            |
   | **Login Experience**  | Vous permet de configurer l’expérience de connexion des utilisateurs de votre Organization. Pour en savoir plus, consultez [Understand how Auth0 Organizations Work](/docs/fr-CA/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                                               |

   Pour en savoir plus, consultez [Application Settings](/docs/fr-CA/get-started/applications/application-settings).
3. Faites défiler la page vers le bas et repérez le paramètre **Trust Token Endpoint IP Header**. Si l’architecture de votre application le prend en charge, activez ce paramètre et cliquez sur **Save Changes**. Ce paramètre aide à vous protéger contre les attaques par force brute en permettant à Auth0 de se fier aux en-têtes IP de votre proxy.

<div id="next-steps">
  ## Prochaines étapes
</div>

Une fois votre application enregistrée et configurée, suivez les étapes suivantes :

1. Configurez une connexion et activez-la pour votre application.
2. Implémentez l’authentification dans le code de votre application à l’aide de l’application que vous avez enregistrée dans Auth0. Consultez nos [Auth0 Quickstarts](/docs/fr-CA/quickstarts), où vous trouverez des instructions détaillées et des exemples pour diverses technologies. Vous apprendrez aussi à implémenter le login et le logout, à gérer vos sessions utilisateur, à récupérer et à afficher les renseignements du profil utilisateur, et bien plus encore.

* Utilisez les [API Auth0](/docs/fr-CA/api).

  * L’[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les principales fonctions liées à l’identité (par exemple, le login, le logout et l’obtention du profil utilisateur). La plupart des utilisateurs utilisent cette API au moyen de nos Quickstarts, de la [bibliothèque auth0.js](/docs/fr-CA/libraries/auth0js) ou du [widget Lock](/docs/fr-CA/libraries/lock). Toutefois, si vous créez toute votre UI d’authentification manuellement, vous pouvez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d’automatiser diverses tâches auxquelles vous pouvez aussi accéder dans l’Auth0 Dashboard (par exemple, créer des utilisateurs et définir les types d’octroi de l’application).
