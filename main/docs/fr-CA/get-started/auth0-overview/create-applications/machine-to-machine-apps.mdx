Pour intégrer Auth0 à une application machine à machine (M2M), vous devez d’abord enregistrer votre application dans Auth0 à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Ces applications peuvent inclure des applications non interactives, comme des outils en ligne de commande, des démons, des appareils IoT ou des services exécutés sur votre backend.

Les applications M2M sont associées à une API et à ses permissions ou [scopes](/docs/fr-CA/get-started/apis/scopes/api-scopes), qui sont sélectionnés à partir de valeurs prédéfinies. Assurez-vous d’avoir déjà [enregistré l’API associée](/docs/fr-CA/get-started/auth0-overview/set-up-apis) dans Auth0 et défini des scopes pour l’API avant de commencer cet enregistrement.

Si vous souhaitez autoriser votre application à accéder uniquement à l’Auth0 <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, vous n’avez pas besoin d’enregistrer une API distincte; la Management API est déjà préremplie pour vous.

Pour intégrer Auth0 à une application machine à machine (M2M), vous devez d’abord enregistrer votre application dans Auth0 à l’aide de l’Auth0 Dashboard.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Saisissez un nom descriptif pour votre application, sélectionnez **Machine to Machine Applications**, puis cliquez sur **Create**.
3. Sélectionnez l’API que vous souhaitez appeler à partir de votre application.
4. Chaque application M2M qui accède à une API doit se voir accorder un ensemble de permissions (ou de scopes) qui devraient être accordées par l’utilisateur authentifié. Pour savoir comment ajouter des permissions, consultez [Add API Permissions](/docs/fr-CA/get-started/apis/add-api-permissions). Sélectionnez les **Permissions** que vous souhaitez inclure dans le access token de votre application, puis cliquez sur **Authorize**.

   <Warning>
     Utilisez les scopes `update:client_grants` et `create:client_grants` uniquement avec des applications à privilèges élevés, car ils permettent au client de s’accorder d’autres permissions.
   </Warning>

   Une fois la nouvelle application créée, vous pouvez configurer les **paramètres de l’application**, qui comprennent les onglets suivants :

   | Onglet Settings | Description                                                                                                                                                                                                                                                                                                                       |
   | --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start** | Affiche tous les exemples disponibles pour les applications ${application_type}. Il montre également comment appeler votre API à l’aide de diverses technologies. Pour savoir comment accepter et valider des Access Tokens dans votre API, consultez nos [Backend/API Quickstarts](/docs/fr-CA/quickstart/backend/nodejs/interactive). |
   | **Settings**    | Affiche tous les paramètres disponibles pour votre application. Par défaut, Auth0 crée la plupart de ces paramètres pour vous.                                                                                                                                                                                                    |
   | **Credentials** | Affiche la méthode d’authentication de l’application et les credentials configurés. Pour en savoir plus, consultez [Application Credentials](/docs/fr-CA/secure/application-credentials)                                                                                                                                                |
   | **APIs**        | Permet d’autoriser des API supplémentaires à utiliser avec votre Application.                                                                                                                                                                                                                                                     |

   Pour en savoir plus, consultez [paramètres de l’application](/docs/fr-CA/get-started/applications/application-settings).

<div id="next-steps">
  ## Prochaines étapes
</div>

Une fois votre application enregistrée et configurée, voici quelques étapes courantes à suivre :

* Configurez une connexion à un fournisseur d’identité et activez-la pour votre application (au besoin selon votre cas d’utilisation).
* Modifiez le code de votre application pour utiliser votre application enregistrée dans Auth0. Consultez nos [Quickstarts Auth0](/docs/fr-CA/quickstarts), où vous trouverez des instructions détaillées et des exemples pour diverses technologies. Vous y apprendrez aussi à implémenter le login et le logout, à gérer vos sessions utilisateur, à récupérer et à afficher les renseignements du profil utilisateur, et plus encore.
* Utilisez les [API Auth0](/docs/fr-CA/api).

  * L’[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les principales fonctions liées à l’identité (par exemple, le login, le logout et l’obtention du profil utilisateur). La plupart des utilisateurs utilisent cette API par l’entremise de nos Quickstarts, de la [bibliothèque auth0.js](/docs/fr-CA/libraries/auth0js) ou du [widget Lock](/docs/fr-CA/libraries/lock). Toutefois, si vous créez toute votre UI d’authentification manuellement, vous devrez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d’automatiser diverses tâches auxquelles vous pouvez aussi accéder dans le Dashboard d’Auth0 (par exemple : créer des utilisateurs, configurer les types d’octroi de l’application).
