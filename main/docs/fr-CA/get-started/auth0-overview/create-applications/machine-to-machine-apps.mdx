Pour intégrer Auth0 à une application machine-to-machine (M2M), vous devez d’abord enregistrer votre application dans Auth0 à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Ces applications peuvent inclure des applications non interactives, comme des outils en ligne de commande, des démons, des appareils IoT ou des services exécutés sur votre backend.

Les applications M2M sont liées à une API et à ses permissions ou [scopes](/docs/fr-CA/get-started/apis/scopes/api-scopes), qui sont sélectionnés à partir de valeurs prédéfinies. Assurez-vous d’avoir déjà [enregistré l’API associée](/docs/fr-CA/get-started/auth0-overview/set-up-apis) dans Auth0 et défini des scopes pour l’API avant de commencer cet enregistrement.

Si vous souhaitez autoriser votre application à accéder uniquement à la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> d’Auth0, vous n’avez pas besoin d’enregistrer une API distincte; la Management API est déjà préremplie pour vous.

Pour intégrer Auth0 à une application machine-to-machine (M2M), vous devez d’abord enregistrer votre application dans Auth0 à l’aide de l’Auth0 Dashboard.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Entrez un nom descriptif pour votre application, sélectionnez **Machine to Machine Applications**, puis cliquez sur **Create**.
3. Sélectionnez l’API que vous voulez appeler depuis votre application.
4. Chaque application M2M qui accède à une API doit recevoir un ensemble de permissions (ou de scopes) qui seront accordés à l’utilisateur authentifié. Pour savoir comment ajouter des permissions, consultez [permission d’API](/docs/fr-CA/get-started/apis/add-api-permissions). Sélectionnez les **Permissions** que vous souhaitez inclure dans le jeton d’accès de votre application, puis cliquez sur **Authorize**.

   <Warning>
     Utilisez les scopes `update:client_grants` et `create:client_grants` uniquement avec des applications hautement privilégiées, car ils permettent au client de s’accorder des permissions supplémentaires.
   </Warning>

   Une fois la nouvelle application créée, vous pouvez configurer les **paramètres de l’application**, qui comprennent les onglets suivants :

   | Onglet de paramètres | Description                                                                                                                                                                                                                                                                                                                |
   | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**      | Affiche tous les exemples offerts pour les applications ${application_type}. Il montre aussi comment appeler votre API à l’aide de diverses technologies. Pour savoir comment accepter et valider des jetons d’accès dans votre API, consultez nos [Quickstarts Backend/API](/docs/fr-CA/quickstart/backend/nodejs/interactive). |
   | **Settings**         | Affiche tous les paramètres disponibles pour votre application. Par défaut, Auth0 crée la plupart de ces paramètres pour vous.                                                                                                                                                                                             |
   | **Credentials**      | Affiche la méthode d’authentification de l’application et les identifiants configurés. Pour en savoir plus, consultez [Application Credentials](/docs/fr-CA/secure/application-credentials)                                                                                                                                      |
   | **APIs**             | Permet d’autoriser des API supplémentaires à utiliser avec votre application.                                                                                                                                                                                                                                              |

   Pour en savoir plus, consultez [paramètres de l’application](/docs/fr-CA/get-started/applications/application-settings).

<div id="next-steps">
  ## Prochaines étapes
</div>

Une fois que vous avez enregistré et configuré votre application, voici quelques prochaines étapes courantes :

* Configurez une connection de fournisseur d&#39;identité et activez-la pour votre application (au besoin selon votre cas d&#39;utilisation).
* Modifiez le code de votre application pour utiliser l&#39;application que vous avez enregistrée dans Auth0. Consultez nos [Auth0 Quickstarts](/docs/fr-CA/quickstarts), où vous trouverez des instructions détaillées et des exemples pour une variété de technologies. Vous y apprendrez aussi comment implémenter le login et le logout, gérer vos sessions utilisateur, récupérer et afficher les renseignements du profil utilisateur, et plus encore.
* Utilisez les [API Auth0](/docs/fr-CA/api).

  * L&#39;[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les principales fonctions liées à l&#39;identité (par exemple, le login, le logout et l&#39;obtention du profil utilisateur). La plupart des utilisateurs utilisent cette API au moyen de nos Quickstarts, de la [bibliothèque auth0.js](/docs/fr-CA/libraries/auth0js) ou du [widget Lock](/docs/fr-CA/libraries/lock). Cependant, si vous créez manuellement toute votre UI d&#39;authentication, vous devrez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d&#39;automatiser diverses tâches qui sont aussi accessibles à partir du Dashboard dans Auth0 (par exemple : créer des utilisateurs, définir les types d’autorisation de l’application).
