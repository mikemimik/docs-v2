Auth0 offre aux utilisateurs du tenant une expérience de recherche plus intuitive pour les Applications et les API enregistrées, avec la possibilité de créer des expressions de filtre pour les champs interrogeables dans Auth0 Dashboard. Au lieu de faire défiler des listes paginées comptant des centaines de ressources, l’utilisateur de votre tenant peut effectuer des requêtes sur des champs comme :

* Le nom de votre application (Travel0-native)
* Les métadonnées de l’application ou de l’utilisateur (permissions)
* Le type d’application (SPA)
* Une API par nom
* Une API par ID

Auth0 Dashboard retourne en temps réel les résultats correspondants parmi l’ensemble de vos ressources enregistrées.

<div id="how-it-works">
  ## Comment ça fonctionne
</div>

La barre de recherche des sections [Applications](https://manage.auth0.com/#/applications) et [APIs](https://manage.auth0.com/#/apis) de l’Auth0 Dashboard utilise une interface de filtres guidés et une recherche booléenne. Elle fonctionne comme un menu déroulant qui répertorie les champs pouvant être filtrés. Lorsque vous sélectionnez un champ et saisissez une valeur, le filtre applique cette valeur comme étiquette amovible au-dessus des résultats. Vous pouvez combiner jusqu’à cinq filtres à la fois.

Chaque filtre ajouté affine davantage les résultats. Pour préciser les résultats, appliquez la logique `AND/OR` entre différents champs. Les filtres se reflètent dans l’URL sous forme de paramètres de requête. Vous pouvez copier l’URL à tout moment pour la partager avec un collègue ou l’ajouter à vos favoris pour revenir plus tard à la même vue filtrée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Exemple booléen**

  Si vous filtrez par **Type d’application** `Native` OR `SPA`, et par **Client propriétaire** `true`, les résultats n’affichent que les applications propriétaires de type Native ou SPA. Les applications de type Regular Web Apps ou les applications tierces sont exclues.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportement de la recherche selon le type de champ
</div>

La recherche dans Auth0 Dashboard renvoie des résultats selon le type de champ.

* **Champs String** (nom, identificateur) : la correspondance partielle est prise en charge. Si vous entrez `payments`, vous trouverez « Payments-application », « Corp Payments Service » et des résultats semblables — l’ordre des mots n’a pas d’importance. Pour exclure les résultats qui correspondent à un terme, ajoutez le préfixe `NOT`, `-` ou `No` — par exemple, si vous entrez `NOT test` dans le champ du nom, toutes les ressources dont le nom ne contient pas « test » sont renvoyées. Les recherches avec caractères génériques nécessitent au moins trois caractères; si vous en entrez moins, un message « Minimum 3 characters » s’affiche et la requête n’est pas soumise.

* **Champs à correspondance exacte**

Vous devez entrer la valeur complète de l’ID. Les ID partiels ne renvoient aucun résultat. Utilisez ces champs lorsque vous avez un Client ID précis provenant d’un log ou d’un ticket de soutien.

* **Champs metadata**

Vous pouvez rechercher dans les metadata par clé et valeur, ou par clé seulement :

* **Clé et valeur** : entrez une clé et une valeur pour trouver les ressources où cette clé exacte correspond à cette valeur. Par exemple, la clé `environment` et la valeur `production` renvoient uniquement les ressources où `app_metadata.environment` est égal à `production`. La correspondance par préfixe est prise en charge pour les valeurs — entrer `prod` correspond à `production`, `prod-eu` et `prod-us`.
  * **Clé seulement** : entrez une clé et appuyez sur **Entrée** sans valeur. Cela renvoie toutes les ressources pour lesquelles cette clé de metadata est définie, peu importe sa valeur. C’est utile pour trouver les ressources qui utilisent une clé particulière.

Vous pouvez aussi rechercher des clés de metadata imbriquées jusqu’à deux niveaux de profondeur à l’aide de la notation par points. Par exemple, la clé `tier.bronze` et la valeur `true` correspondent aux ressources où `app_metadata.tier.bronze` est égal à `true`.

Pour exclure les résultats qui correspondent à une paire clé-valeur précise, ajoutez le préfixe `NOT` à la valeur. Par exemple, la clé `color` et la valeur `NOT orange` renvoient toutes les ressources où `color` est défini à autre chose que `orange`. Les clés doivent correspondre exactement. Les correspondances de type contient et se termine par ne sont pas prises en charge pour les metadata. Les valeurs sont indexées jusqu’à 64 caractères; les valeurs plus longues ne sont pas interrogeables.

* **Champs d’énumération**

Sélectionnez un ou plusieurs types d’application dans une liste prédéfinie. Les sélections multiples dans le même filtre utilisent une logique OR. Par exemple, si vous sélectionnez `Native` et `SPA`, toutes les applications Native et SPA sont renvoyées.

* **Champs Boolean**

Recherchez des clients propriétaires en sélectionnant `true` ou `false`. Utilisez ceci pour distinguer les applications propriétaires (celles autorisées à ignorer l’écran de consentement) des applications tierces.

<div id="searchable-fields">
  ### Champs interrogeables
</div>

<div id="applications">
  #### Applications
</div>

| **Champ**                                    | **Description**                                                                        | **Type de correspondance**           |
| -------------------------------------------- | -------------------------------------------------------------------------------------- | ------------------------------------ |
| **Nom**                                      | Le nom de l’application                                                                | Correspondance partielle             |
| **Client ID** (`client_id`)                  | Le client ID unique                                                                    | Correspondance exacte                |
| **ID client externe** (`external_client_id`) | Un identifiant externe de l’application                                                | Correspondance exacte                |
| **Metadata**                                 | Métadonnées personnalisées sous forme de paires clé-valeur stockées dans l’application | Correspondance exacte ou par préfixe |
| **Type d’application**                       | Native, SPA, application Web traditionnelle ou Machine to Machine                      | Sélection dans une énumération       |
| **Client propriétaire**                      | Indique si l’application est propriétaire ou tierce                                    | Booléen                              |

<div id="apis">
  #### API
</div>

| **Champ**       | **Description**                                                          | **Type de correspondance** |
| --------------- | ------------------------------------------------------------------------ | -------------------------- |
| **Nom**         | Le nom de l’API                                                          | Correspondance partielle   |
| **Identifiant** | L’identifiant d’audience unique (par exemple, `https://api.example.com`) | Correspondance partielle   |
| **ID**          | L’ID interne du système                                                  | Correspondance exacte      |

<div id="limitations">
  ### Limites
</div>

* Vous pouvez appliquer un maximum de cinq filtres par recherche. La barre de recherche empêche d’ajouter un sixième filtre.
* Les résultats de recherche peuvent ne pas tenir compte des modifications apportées au cours des dernières secondes, car l’index ne devient cohérent qu’après un court délai.
* Les résultats s’affichent en moins de deux secondes dans des conditions normales. Si une requête expire, réduisez le nombre de filtres ou utilisez un terme de recherche plus précis.

<div id="example-queries">
  ## Exemples de requêtes
</div>

<div id="find-an-application-by-client-id">
  ### Trouver une application par ID client
</div>

Utilisez cette procédure lorsque vous avez un `client_id` provenant d’un journal d’erreurs ou d’un ticket de soutien et que vous devez trouver rapidement l’application.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **Client ID**.
3. Collez la valeur complète de `client_id`. L’application correspondante s’affiche immédiatement.

<div id="find-applications-using-metadata">
  ### Trouver des applications à l’aide des métadonnées
</div>

Si vos applications stockent des métadonnées personnalisées — comme des étiquettes d’environnement, des niveaux d’abonnement ou des ID externes — vous pouvez faire une recherche à partir de ces valeurs.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **Metadata**.
3. Entrez la clé de métadonnées, par exemple `environment`.
4. Entrez la valeur, par exemple `production`. Les applications dont la valeur `metadata.environment` est égale à `production` s’affichent dans les résultats. Pour faire une recherche par préfixe, entrez une valeur partielle comme `prod`.

<div id="troubleshooting">
  ## Dépannage
</div>

**Aucun résultat trouvé**

Vérifiez que les valeurs du filtre sont correctement orthographiées. Pour les champs de type String, essayez un terme plus court ou plus général. Pour les champs de métadonnées, confirmez le nom exact de la clé — les clés respectent la casse et doivent correspondre exactement. La correspondance de type « contient » n’est pas prise en charge pour les métadonnées; utilisez plutôt un préfixe. Pour les champs à correspondance exacte, comme l’ID d’application, assurez-vous de coller la valeur complète de l’ID.

**Les résultats semblent ne pas être à jour**

Les résultats de recherche reposent sur une cohérence éventuelle et peuvent ne pas refléter les changements apportés dans les dernières secondes. Si vous avez récemment créé ou mis à jour une ressource, attendez quelques secondes, puis relancez la recherche.

**Délai de recherche dépassé**

Réduisez le nombre de filtres actifs ou utilisez un terme de recherche plus précis. Si le problème persiste, communiquez avec [Auth0 Support](https://support.auth0.com).
