Auth0 offre aux utilisateurs du tenant une fonction de recherche plus intuitive pour les Applications et les API enregistrées, avec la possibilité de créer des expressions de filtre pour les champs interrogeables dans Auth0 Dashboard. Au lieu de faire défiler des listes paginées contenant des centaines de ressources, les utilisateurs du tenant peuvent effectuer des requêtes sur des champs comme :

* Le nom de votre application (Travel0-native)
* Les métadonnées de l’application ou de l’utilisateur (permissions)
* Le type d’application (SPA)
* L’API par nom
* L’API par ID

Auth0 Dashboard renvoie en temps réel les résultats correspondants parmi toutes vos ressources enregistrées."

<div id="how-it-works">
  ## Fonctionnement
</div>

La barre de recherche des sections [Applications](https://manage.auth0.com/#/applications) et [APIs](https://manage.auth0.com/#/apis) de l’Auth0 Dashboard utilise une interface de filtres guidés et la recherche booléenne. Elle fonctionne comme un menu déroulant qui affiche les champs pouvant être filtrés. Lorsque vous sélectionnez un champ et saisissez une valeur, le filtre l’applique sous forme de balise amovible au-dessus des résultats. Vous pouvez combiner jusqu’à cinq filtres à la fois.

Chaque filtre ajouté affine davantage les résultats. Pour cibler les résultats, appliquez une logique `AND/OR` entre différents champs. Les filtres se répercutent dans l’URL sous forme de paramètres de requête. Vous pouvez copier l’URL en tout temps pour la partager avec un collègue ou l’ajouter à vos signets afin de revenir plus tard à la même vue filtrée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Exemple booléen**

  Si vous filtrez par **Type d’application** `Native` OR `SPA`, et **Client de première partie** `true`, les résultats n’affichent que les applications de première partie de type Native ou SPA. Les applications de type Regular Web Apps et les applications tierces sont exclues.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportement de la recherche par type de champ
</div>

La recherche dans Auth0 Dashboard renvoie des correspondances selon le type de champ.

* **Champs de type chaîne** (name, identifier) : La correspondance partielle est prise en charge. La saisie de `payments` trouve « Payments-application », « Corp Payments Service » et des résultats semblables — l’ordre des mots n’a pas d’importance. Pour exclure les résultats qui correspondent à un terme, faites-le précéder de `NOT`, `-` ou `No` — par exemple, la saisie de `NOT test` dans le champ name renvoie toutes les ressources dont le nom ne contient pas « test ». Les recherches avec caractères génériques exigent au moins trois caractères; en saisir moins affiche le message « Minimum 3 characters » et n’exécute pas la requête.

* **Champs à correspondance exacte**

Vous devez saisir la valeur complète de l’ID. Les IDs partiels ne renvoient aucun résultat. Utilisez ces champs lorsque vous avez un Client ID précis provenant d’un journal ou d’un ticket de soutien.

* **Champs de métadonnées**

Vous pouvez rechercher dans les métadonnées par clé et valeur, ou par clé seulement :

* **Key and value** : Saisissez une clé et une valeur pour trouver les ressources où cette clé correspond exactement à cette valeur. Par exemple, la clé `environment` et la valeur `production` renvoient uniquement les ressources où `app_metadata.environment` est égal à `production`. La correspondance par préfixe est prise en charge pour les valeurs — la saisie de `prod` correspond à `production`, `prod-eu` et `prod-us`.
  * **Key only** : Saisissez une clé et appuyez sur **Enter** sans valeur. Cela renvoie toutes les ressources pour lesquelles cette clé de métadonnées est définie, quelle que soit sa valeur. Pratique pour trouver les ressources qui utilisent une clé précise, peu importe sa valeur.

Vous pouvez aussi rechercher des clés de métadonnées imbriquées jusqu’à deux niveaux de profondeur à l’aide de la notation par points. Par exemple, la clé `tier.bronze` et la valeur `true` correspondent aux ressources où `app_metadata.tier.bronze` est égal à `true`.

Pour exclure les résultats qui correspondent à une paire clé-valeur précise, faites précéder la valeur de `NOT`. Par exemple, la clé `color` et la valeur `NOT orange` renvoient toutes les ressources où `color` est défini à autre chose que `orange`. Les clés doivent correspondre exactement. Les correspondances « contains » et « ends-with » ne sont pas prises en charge pour les métadonnées. Les valeurs sont indexées jusqu’à 64 caractères; les valeurs plus longues ne sont pas interrogeables.

* **Champs d’énumération**

Sélectionnez un ou plusieurs types d’application dans une liste prédéfinie. Plusieurs sélections dans le même filtre utilisent une logique OU. Par exemple, sélectionner `Native` et `SPA` renvoie toutes les applications Native et SPA.

* **Champs booléens**

Recherchez les clients first-party en sélectionnant `true` ou `false`. Utilisez cette option pour distinguer les applications first-party (celles autorisées à ignorer l’écran de consentement) des applications third-party.

<div id="searchable-fields">
  ### Champs interrogeables
</div>

<div id="applications">
  #### Applications
</div>

| **Champ**                                    | **Description**                                                                        | **Type de correspondance**           |
| -------------------------------------------- | -------------------------------------------------------------------------------------- | ------------------------------------ |
| **Nom**                                      | Le nom de l’application                                                                | Correspondance partielle             |
| **Client ID** (`client_id`)                  | Le Client ID unique                                                                    | Correspondance exacte                |
| **Client ID externe** (`external_client_id`) | Un identifiant externe de l’application                                                | Correspondance exacte                |
| **Métadonnées**                              | Métadonnées personnalisées sous forme de paires clé-valeur stockées dans l’application | Correspondance exacte ou par préfixe |
| **Type d’application**                       | Native, SPA, application Web traditionnelle ou Machine to Machine                      | Sélection dans une énumération       |
| **Client de première partie**                | Indique si l’application est de première partie ou tierce                              | Booléen                              |

<div id="apis">
  #### APIs
</div>

| **Champ**       | **Description**                                                          | **Type de correspondance** |
| --------------- | ------------------------------------------------------------------------ | -------------------------- |
| **Nom**         | Le nom de l’API                                                          | Correspondance partielle   |
| **Identifiant** | L’identifiant d’audience unique (par exemple, `https://api.example.com`) | Correspondance partielle   |
| **ID**          | L’ID interne du système                                                  | Correspondance exacte      |

<div id="limitations">
  ### Limites
</div>

* Vous pouvez appliquer un maximum de cinq filtres par recherche. La barre de recherche empêche l’ajout d’un sixième filtre.
* Il se peut que les résultats de recherche ne reflètent pas les modifications apportées au cours des dernières secondes, car l’index se met à jour avec un léger délai.
* Les résultats sont renvoyés en moins de deux secondes dans des conditions normales. Si une requête dépasse le délai d’attente, réduisez le nombre de filtres ou utilisez un terme de recherche plus précis.

<div id="example-queries">
  ## Exemples de requêtes
</div>

<div id="find-an-application-by-client-id">
  ### Trouver une application à partir de son ID client
</div>

Utilisez cette procédure lorsque vous avez un `client_id` provenant d’un journal d’erreurs ou d’un ticket de soutien et que vous devez localiser rapidement l’application.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **Client ID**.
3. Collez la valeur complète de `client_id`. L’application correspondante s’affiche immédiatement.

<div id="find-applications-using-metadata">
  ### Trouver des applications à l’aide des métadonnées
</div>

Si vos applications stockent des métadonnées personnalisées — comme des étiquettes d’environnement, des niveaux de forfait ou des ID externes — vous pouvez rechercher ces valeurs.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **Metadata**.
3. Entrez la clé de métadonnées, par exemple `environment`.
4. Entrez la valeur, par exemple `production`. Les applications dont `metadata.environment` est égal à `production` s’affichent dans les résultats. Pour faire une recherche par préfixe, entrez une valeur partielle comme `prod`.

<div id="troubleshooting">
  ## Dépannage
</div>

**Aucun résultat trouvé**

Vérifiez que les valeurs des filtres sont bien orthographiées. Pour les champs de type chaîne, essayez un terme plus court ou plus général. Pour les champs de métadonnées, confirmez le nom exact de la clé — les clés respectent la casse et doivent correspondre exactement. La correspondance de type « contient » n’est pas prise en charge pour les métadonnées; utilisez plutôt un préfixe. Pour les champs à correspondance exacte, comme l’ID de l’application, assurez-vous de coller la valeur complète de l’ID.

**Les résultats semblent obsolètes**

Les résultats de recherche sont soumis à une cohérence éventuelle et peuvent ne pas refléter les modifications apportées au cours des dernières secondes. Si vous avez récemment créé ou mis à jour une ressource, attendez quelques secondes, puis relancez la recherche.

**Délai d’attente de la recherche dépassé**

Réduisez le nombre de filtres actifs ou utilisez un terme de recherche plus précis. Si le problème persiste, communiquez avec [Auth0 Support](https://support.auth0.com).
