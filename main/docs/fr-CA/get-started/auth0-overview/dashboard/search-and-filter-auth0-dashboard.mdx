Auth0 offre aux utilisateurs du tenant une fonction de recherche plus intuitive pour les applications et les API enregistrées, avec la possibilité de créer des expressions de filtre pour les champs interrogeables dans Auth0 Dashboard. Plutôt que de parcourir des listes paginées comptant des centaines de ressources, les utilisateurs du tenant peuvent lancer des requêtes sur des champs comme :

* Le nom de votre application (Travel0-native)
* Les métadonnées de l’application ou de l’utilisateur (permissions)
* Le type d’application (SPA)
* Une API par nom
* Une API par ID

Auth0 Dashboard renvoie en temps réel les résultats correspondants parmi toutes vos ressources enregistrées.

<div id="how-it-works">
  ## Fonctionnement
</div>

La barre de recherche des sections [Applications](https://manage.auth0.com/#/applications) et [APIs](https://manage.auth0.com/#/apis) de l’Auth0 Dashboard utilise une interface de filtres guidés et une recherche booléenne. Elle fonctionne comme un menu déroulant qui affiche les champs pouvant être filtrés. Lorsque vous sélectionnez un champ et saisissez une valeur, celle-ci est appliquée sous forme d’étiquette amovible au-dessus des résultats. Vous pouvez combiner jusqu’à cinq filtres à la fois.

Chaque filtre ajouté affine davantage les résultats. Pour cibler les résultats, appliquez une logique `AND/OR` entre différents champs. Les filtres se reflètent dans l’URL sous forme de paramètres de requête. Vous pouvez copier l’URL à tout moment pour la partager avec un collègue ou l’ajouter à vos signets afin de revenir plus tard à la même vue filtrée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Exemple booléen**

  Si vous filtrez par **Type d’application** `Native` OR `SPA`, et **Client de première partie** `true`, les résultats n’affichent que les applications de première partie de type Native ou SPA. Les Applications de type Regular Web Apps ou les applications tierces sont exclues.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportement de la recherche par type de champ
</div>

La recherche dans Auth0 Dashboard trouve des résultats en fonction du type de champ.

* **Champs de type chaîne** (nom, identifiant) : La correspondance partielle est prise en charge. Si vous saisissez `payments`, vous trouverez « Payments-application », « Corp Payments Service » et des résultats semblables — l’ordre des mots n’a pas d’importance. Pour exclure les résultats correspondant à un terme, ajoutez le préfixe `NOT`, `-` ou `No` — par exemple, si vous saisissez `NOT test` dans le champ du nom, toutes les ressources dont le nom ne contient pas « test » sont renvoyées. Les recherches avec caractère générique exigent au moins trois caractères; si vous en saisissez moins, le message « Minimum 3 characters » s’affiche et la requête n’est pas envoyée.

* **Champs à correspondance exacte**

Vous devez saisir la valeur complète de l’ID. Les IDs partiels ne renvoient aucun résultat. Utilisez ces champs lorsque vous avez un Client ID précis provenant d’un journal ou d’un ticket de soutien.

* **Champs de métadonnées**

Vous pouvez rechercher dans les métadonnées par clé et valeur, ou par clé seulement :

* **Clé et valeur** : Saisissez une clé et une valeur pour trouver les ressources où cette clé exacte correspond à cette valeur. Par exemple, la clé `environment` et la valeur `production` renvoient uniquement les ressources où `app_metadata.environment` est égal à `production`. La correspondance par préfixe est prise en charge pour les valeurs — saisir `prod` correspond à `production`, `prod-eu` et `prod-us`.
  * **Clé seulement** : Saisissez une clé et appuyez sur **Entrée** sans valeur. Cela renvoie toutes les ressources pour lesquelles cette clé de métadonnées est définie, quelle que soit sa valeur. C’est utile pour trouver les ressources qui utilisent cette clé, peu importe la valeur.

Vous pouvez aussi rechercher des clés de métadonnées imbriquées jusqu’à deux niveaux de profondeur en utilisant la notation par points. Par exemple, la clé `tier.bronze` et la valeur `true` correspondent aux ressources où `app_metadata.tier.bronze` est égal à `true`.

Pour exclure les résultats qui correspondent à une paire clé-valeur précise, ajoutez le préfixe `NOT` à la valeur. Par exemple, la clé `color` et la valeur `NOT orange` renvoient toutes les ressources où `color` est défini à autre chose que `orange`. Les clés doivent correspondre exactement. Les correspondances de type contient et se termine par ne sont pas prises en charge pour les métadonnées. Les valeurs sont indexées jusqu’à 64 caractères; les valeurs plus longues ne sont pas interrogeables.

* **Champs d’énumération**

Sélectionnez un ou plusieurs types d’Application dans une liste prédéfinie. Les sélections multiples dans le même filtre utilisent une logique OU. Par exemple, si vous sélectionnez `Native` et `SPA`, toutes les applications Native et SPA sont renvoyées.

* **Champs booléens**

Recherchez des clients first-party en sélectionnant `true` ou `false`. Utilisez cette option pour distinguer les applications first-party (celles autorisées à ignorer l’écran de consentement) des applications third-party.

<div id="searchable-fields">
  ### Champs interrogeables
</div>

<div id="applications">
  #### Applications
</div>

| **Champ**                                    | **Description**                                                                        | **Type de correspondance**           |
| -------------------------------------------- | -------------------------------------------------------------------------------------- | ------------------------------------ |
| **Nom**                                      | Le nom de l’application                                                                | Correspondance partielle             |
| **Client ID** (`client_id`)                  | L’ID client unique                                                                     | Correspondance exacte                |
| **ID client externe** (`external_client_id`) | Un identifiant externe pour l’application                                              | Correspondance exacte                |
| **Métadonnées**                              | Métadonnées personnalisées sous forme de paires clé-valeur stockées dans l’application | Correspondance exacte ou par préfixe |
| **Type d’application**                       | Native, SPA, application Web traditionnelle ou Machine to Machine                      | Sélection d’une énumération          |
| **client de première partie**                      | Indique si l’application est propriétaire ou tierce partie                             | Booléen                              |

<div id="apis">
  #### API
</div>

| **Champ**       | **Description**                                                             | **Type de correspondance** |
| --------------- | --------------------------------------------------------------------------- | -------------------------- |
| **Nom**         | Le nom de l’API                                                             | Correspondance partielle   |
| **Identifiant** | L’identifiant unique de l’audience (par exemple, `https://api.example.com`) | Correspondance partielle   |
| **Id**          | L’ID interne du système                                                     | Correspondance exacte      |

<div id="limitations">
  ### Limites
</div>

* Vous pouvez appliquer un maximum de cinq filtres par recherche. La barre de recherche empêche l’ajout d’un sixième filtre.
* Les résultats de recherche peuvent ne pas refléter les modifications apportées au cours des dernières secondes, car l’index n’est mis à jour partout qu’après un court délai.
* Les résultats s’affichent en moins de deux secondes dans des conditions normales. Si une requête expire, réduisez le nombre de filtres ou utilisez un terme de recherche plus précis.

<div id="example-queries">
  ## Exemples de requêtes
</div>

<div id="find-an-application-by-client-id">
  ### Trouver une application à l’aide du Client ID
</div>

Utilisez cette procédure lorsque vous avez un `client_id` provenant d’un journal d’erreurs ou d’un ticket de soutien et que vous devez retrouver rapidement l’application.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche, puis choisissez **Client ID**.
3. Collez la valeur complète de `client_id`. L’application correspondante s’affiche immédiatement.

<div id="find-applications-using-metadata">
  ### Trouver des applications à l’aide des métadonnées
</div>

Si vos applications stockent des métadonnées personnalisées — comme des libellés d’environnement, des paliers de forfait ou des ID externes — vous pouvez effectuer une recherche à partir de ces valeurs.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **métadonnées**.
3. Saisissez la clé de métadonnées, par exemple `environment`.
4. Saisissez la valeur, par exemple `production`. Les applications pour lesquelles `metadata.environment` est égal à `production` s’affichent dans les résultats. Pour une correspondance par préfixe, saisissez une valeur partielle comme `prod`.

<div id="troubleshooting">
  ## Dépannage
</div>

**Aucun résultat trouvé**

Vérifiez que les valeurs de filtre sont orthographiées correctement. Pour les champs de type chaîne, essayez un terme plus court ou plus général. Pour les champs de métadonnées, confirmez le nom exact de la clé — les clés sont sensibles à la casse et doivent correspondre exactement. La correspondance de type « contient » n’est pas prise en charge pour les métadonnées; utilisez plutôt un préfixe. Pour les champs à correspondance exacte comme l’ID d’application, assurez-vous de coller la valeur complète de l’ID.

**Les résultats semblent obsolètes**

Les résultats de recherche ne sont pas mis à jour instantanément et peuvent ne pas refléter les modifications apportées au cours des dernières secondes. Si vous avez récemment créé ou mis à jour une ressource, attendez quelques secondes, puis relancez la recherche.

**La recherche a expiré**

Réduisez le nombre de filtres actifs ou utilisez un terme de recherche plus précis. Si le problème persiste, contactez [Auth0 Support](https://support.auth0.com).
