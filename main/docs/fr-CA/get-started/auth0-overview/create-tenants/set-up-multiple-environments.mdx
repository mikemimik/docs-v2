---
description: Décrit comment utiliser plusieurs tenants Auth0 pour gérer divers environnements.
title: Configurer plusieurs environnements
---

Les environnements de développement, de staging et de production sont faciles à configurer dans Auth0. Créez simplement un nouveau tenant pour chaque environnement afin de garantir leur isolement. Vous pouvez facilement passer d’un tenant à l’autre à l’aide du sélecteur de tenants dans le menu en haut à gauche du Dashboard. Vous pouvez également configurer des administrateurs différents pour chacun.

Les [limites de débit](/docs/fr-CA/troubleshoot/customer-support/operational-policies/rate-limit-policy) de production s’appliquent uniquement aux tenants marqués comme `Production`. Assurez-vous que la balise d’environnement de votre tenant est définie sur `Production` avant de passer en production, et prenez note des différences entre les limites de débit selon les tenants, les environnements et les structures tarifaires.

Vous pouvez nommer vos différents environnements comme vous le souhaitez. Pour les environnements de production, nous recommandons fortement d’utiliser des [domaines personnalisés](/docs/fr-CA/customize/custom-domains).

Si vous avez un abonnement Enterprise, vous pouvez créer des tenants enfants en [liant plusieurs tenants à un seul abonnement](/docs/fr-CA/get-started/auth0-overview/create-tenants/child-tenants) ; chaque tenant enfant a alors accès aux fonctionnalités payantes ou améliorées associées à l’abonnement. Vous pouvez faire en sorte que vos environnements de développement, de staging ou de test aient accès aux mêmes fonctionnalités que votre environnement de production en les liant tous au même abonnement Enterprise.

<div id="tag-the-environment">
  ## Attribuez une balise d’environnement
</div>

Pour chaque nouveau tenant créé, vous devez préciser son environnement. Vous pouvez attribuer des balises d’environnement à vos tenants pour distinguer les environnements de développement, de staging et de production.

Si votre tenant est à usage mixte, choisissez l’environnement le plus élevé. Par exemple, un tenant utilisé à la fois pour le développement et la production doit être défini sur **Production**.

1. Pour attribuer une balise d’environnement à un tenant, accédez à [Dashboard &gt; Settings &gt; General](https://manage.auth0.com/#/tenant/general).

   <Frame>![Onglet General dans Tenant Settings du Dashboard](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. Sous **Assign Environment Tag**, indiquez si l’environnement de votre tenant est **Development**, **Staging** ou **Production**.

   <Frame>![undefined](/docs/images/cdy7uua7fh8z/27OH1jFXce97CSjk7TPeD7/deafc344f675fb11ff5d3d589ffadee9/2025-02-26_14-27-31.png)</Frame>
3. Après avoir sélectionné l’environnement, cliquez sur **Save**.

<div id="migration">
  ## Migration
</div>

Grâce à la [Management API v2](https://auth0.com/docs/api/management/v2), vous pouvez automatiser la migration des ressources ([rules](/docs/fr-CA/customize/rules), [connections](/docs/fr-CA/authenticate/identity-providers) de base de données, etc.) entre les tenants.

Pour simplifier la gestion de la configuration, enregistrez vos valeurs de configuration dans le [Dashboard](https://manage.auth0.com/#/rules) plutôt que de les coder en dur dans vos scripts de **rules** ou de **connections** de base de données.

Par exemple, supposons que vous vouliez définir une URL pour les logs. Une façon de faire consiste à la coder en dur dans la rule :

```javascript lines
function(user, context, callback){
      var log_url = 'https://someurl/log';
      ...
    }
```

Ce code, toutefois, n’est pas portable, puisque cette URL changera probablement entre le développement et la production.

La façon recommandée de travailler avec du code que vous devez utiliser ou déplacer du développement vers la production consiste à passer par la section [Rules](https://manage.auth0.com/#/rules). Si vous n’avez pas encore créé de Rule, vous devrez le faire. (Sinon, passez directement à l’étape 4.)

1. Cliquez sur **Create Your First Rule**.
2. Choisissez le modèle **empty rule**.
3. Entrez un nom pour votre nouvelle Rule, puis cliquez sur **Save**.
4. Accédez à [Dashboard &gt; Rules](https://manage.auth0.com/#/rules), puis faites défiler jusqu’au bas de la page pour définir vos valeurs de configuration (nous utiliserons `log_url` comme nom de clé, et `https://someurl/log` comme valeur), puis cliquez sur **Create**.
5. Vous pouvez maintenant écrire votre Rule. Modifiez la Rule que vous avez créée, entrez le code suivant dans la zone de code, puis cliquez sur **Save**.

   ```javascript lines
   function(user, context, callback){
         var log_url = configuration.log_url;
         ...
       }
   ```

Ce code est portable et, lorsque vous migrerez vers la production, vous n’aurez qu’à modifier ce paramètre plutôt que de chercher dans vos scripts.

<div id="adldap-connectors">
  ## Connecteurs AD/LDAP
</div>

Si vous utilisez plusieurs tenants Auth0 avec AD/LDAP, vous devrez créer une connexion AD/LDAP et configurer un connecteur AD/LDAP pour chaque tenant. En effet, chaque connecteur AD/LDAP est lié à une connexion précise dans un tenant Auth0.

Plusieurs connecteurs AD/LDAP peuvent pointer vers le même répertoire AD ou LDAP, mais chaque connecteur AD/LDAP ne peut être utilisé que par une seule connexion au sein d’un seul tenant Auth0.

Si vous avez plusieurs répertoires AD/LDAP auprès desquels les utilisateurs s’authentifieront (par exemple, pour prendre en charge différents départements ou clients, chacun ayant son propre répertoire), vous pouvez configurer plusieurs connecteurs AD/LDAP dans chaque tenant Auth0.
