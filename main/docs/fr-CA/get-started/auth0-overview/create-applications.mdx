Vous pouvez créer une application manuellement ou l’importer à partir de l’URL d’un [document de métadonnées de l’ID client (CIMD)](/docs/fr-CA/get-started/auth0-overview/create-applications/register-applications-with-cimd) dans Auth0 Dashboard.

<div id="create-manually">
  ## Créer manuellement
</div>

Lorsque vous créez manuellement une application, Auth0 lui attribue un <Tooltip tip="ID client : chaîne alphanumérique qui sert d’identifiant unique à une application." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">ID client</Tooltip>, une chaîne alphanumérique qui sert d’identifiant unique à l’application. Vous utiliserez cet ID dans le code de votre application lorsque vous appellerez les API d’Auth0. Ce processus d’enregistrement de l’application s’appelle l’enregistrement manuel du client.

Un autre élément important est le <Tooltip tip="Secret client : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">Secret client</Tooltip>. Il doit demeurer confidentiel en tout temps. Si quelqu’un a accès au secret client de votre application, cette personne peut se faire passer pour votre application et accéder à des ressources protégées. Il est important de sélectionner le bon type d’application afin d’aider Auth0 à détecter certains risques de sécurité.

<div id="import-from-url">
  ## Importer à partir d’une URL
</div>

Vous pouvez enregistrer des applications en important depuis une URL un document de métadonnées de l’ID client (CIMD) hébergé à l’externe. Le CIMD est un fichier JSON contenant les métadonnées du client, hébergé sur un domaine HTTPS sécurisé contrôlé par l’application. Ce processus d’enregistrement d’application s’appelle l’enregistrement manuel avec CIMD. Pour en savoir plus, consultez [Enregistrer des applications avec CIMD](/docs/fr-CA/get-started/auth0-overview/create-applications/register-applications-with-cimd).

<div id="manual-client-registration-vs-manual-cimd-registration">
  ## Enregistrement manuel du client vs. enregistrement manuel CIMD
</div>

Le tableau suivant compare l’enregistrement manuel du client et l’enregistrement manuel CIMD :

| Fonctionnalité                 | Enregistrement manuel du client                    | Enregistrement manuel CIMD                                  |
| ------------------------------ | -------------------------------------------------- | ----------------------------------------------------------- |
| **ID client**                  | Généré par Auth0                                   | URL HTTPS (p. ex., `https://yourdomain.com/client.json`)    |
| **Authentification**           | Secret client ou certificat                        | Clés asymétriques uniquement (aucun secret)                 |
| **Vérification de l’identité** | Aucune                                             | Validation de la propriété du domaine par hébergement HTTPS |
| **Gestion des métadonnées**    | Mise à jour via Dashboard/API                      | Le CIMD constitue la source de vérité                       |
| **Rotation des clés**          | Manuelle, une seule clé                            | Plusieurs clés prises en charge dans `jwks_uri`             |
| **Type d’application**         | Tous les types                                     | Applications tierces uniquement                             |
| **Idéal pour**                 | Applications de première partie, services internes | Clients MCP, intégrations tierces                           |

<div id="create-application-in-auth0">
  ## Créer une application dans Auth0
</div>

Pour créer une application dans Auth0 :

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Sélectionnez **Create Application**.
3. Sélectionnez **Create it manually** ou **Import from URL**.
   * **Import from URL** : Suivez les instructions dans [Enregistrer des applications avec CIMD](/docs/fr-CA/get-started/auth0-overview/create-applications/register-applications-with-cimd).
   * **Create it manually** : Donnez un nom explicite à votre nouvelle application. Ensuite, sélectionnez un type d’application :
     * **Native Applications** : Ces applications comprennent des applis mobiles, de bureau ou hybrides qui s’exécutent nativement sur un appareil (p. ex., iOS, Android).
     * **Single-Page Web Applications** : Ces applications comprennent des applications Javascript qui exécutent la majeure partie de la logique de leur interface utilisateur dans un navigateur Web et communiquent principalement avec un serveur Web au moyen d’API (p. ex., AngularJS + Node.js ou React).
     * **Regular Web Applications** : Ces applications sont des applications Web traditionnelles qui exécutent la majeure partie de leur logique sur le serveur (p. ex., Express.js, ASP.NET).
     * **Machine-to-Machine Applications** : Ces applications comprennent des applications non interactives, comme des outils en ligne de commande, des démons, des appareils IoT ou des services exécutés sur votre serveur principal.
4. Sélectionnez **Create**.
