Sur la page [Applications](http://manage.auth0.com/#/applications) du Dashboard, repérez votre application dans la liste, puis cliquez sur son nom pour afficher les paramètres disponibles. Passez ensuite à l’onglet **information d’authentification**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’onglet information d’authentification peut ne pas être visible dans votre tenant si vous avez une application publique. Pour en savoir plus sur les applications confidentielles et les applications publiques, consultez [Confidential and Public Applications](/docs/fr-CA/get-started/applications/confidential-and-public-applications).
</Callout>

<div id="authentication-methods">
  ## Méthodes d’authentification
</div>

Auth0 offre les moyens suivants pour permettre à votre application de s’authentifier auprès du <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=authorization+server">serveur d’autorisation</Tooltip> :

* **Clé privée <Tooltip tip="JSON Web Token (JWT) : format standard d’ID token (et souvent de jeton d’accès) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JWT">JWT</Tooltip>** : Une méthode d’authentification asymétrique pour les applications confidentielles. Avec Private Key JWT, vous générez une paire de clés, publique et privée, à utiliser comme informations d’authentification. Vous fournissez la clé publique et conservez la clé privée de façon sécurisée dans votre propre système, sans la partager avec Auth0. Vous utilisez la clé privée pour signer la requête envoyée au serveur d’autorisation. Vous pouvez gérer les clés publiques ci-dessous dans la section **Informations d’authentification disponibles**.
* **<Tooltip tip="Client secret : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Secret">Client secret</Tooltip> :** Une méthode d’authentification symétrique. Dans l’authentification par client secret, vous fournissez le client secret attribué par Auth0 lorsque vous avez créé l’application. Vous pouvez afficher et copier le `client_secret` directement dans cette section.

  * **Client secret (Basic)** : Utilisez Basic pour une application confidentielle qui utilise le schéma d’authentification `HTTP BASIC` afin d’envoyer un client secret.
  * **Client secret (Post)** : Utilisez Post pour une application confidentielle qui utilise des paramètres du corps de la requête afin d’envoyer un client secret.

Pour en savoir plus sur les méthodes d’authentification et les informations d’authentification associées, consultez [Informations d’authentification de l’application](/docs/fr-CA/secure/application-credentials).

<div id="available-credentials">
  ## Informations d’authentification disponibles
</div>

<Frame>![](/docs/images/cdy7uua7fh8z/7vzjXXZ0A9pDj5qsXmOTwh/b4b299c6316216376f7e2858007df0b1/Private_Key_JWT_-_English.png)</Frame>

**Ajouter une nouvelle information d’authentification** : ajoute une nouvelle entrée à la liste si vous n’avez pas atteint le nombre maximal d’informations d’authentification (2) pour l’application.

* **Nom** : nom de l’information d’authentification. Par exemple : 4096.
* **Key ID** : identifiant unique de l’information d’authentification généré par Auth0. La même information d’authentification ne peut pas être téléversée plus d’une fois.
* **Algorithme** : algorithme que vous sélectionnez pour chaque information d’authentification. Nous prenons en charge RS256, PS256 et RS384.
* **Expire le** : `datetime` à partir duquel l’information d’authentification n’est plus valide pour l’utilisation déclarée. L’information d’authentification ne sera pas supprimée, mais elle sera inutilisable. Cette option peut être configurée dans l’interface utilisateur en activant la case à cocher **Définir une date d’expiration explicite pour cette information d’authentification**.

Utilisez le menu latéral pour activer, désactiver ou supprimer des informations d’authentification.
