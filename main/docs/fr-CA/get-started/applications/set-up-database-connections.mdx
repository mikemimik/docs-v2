title: Configurer des connexions de base de données
---

Vous pouvez configurer des connexions de base de données pour les applications à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Les connexions de base de données configurées peuvent être utilisées pour se connecter à votre application. Pour configurer une connexion, suivez les étapes ci-dessous.

<AccordionGroup>
  <Accordion icon="circle-1" title="1 Ouvrez l’Auth0 Dashboard">
    Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](http://manage.auth0.com/#/connections/database), puis sélectionnez **Create DB Connection**.

    <Frame>![Dashboard - Authentication - Database - Liste des connexions de base de données](/docs/images/cdy7uua7fh8z/1Hf6ctA9xhsLOQO9IyG0VU/9061b8436368e003b165efb890d86473/DB_Connections_-_EN.png)</Frame>
  </Accordion>

  <Accordion icon="circle-2" title="Nommez la connexion">
    Entrez un nom pour votre connexion, puis sélectionnez **Create**.

    <Frame>![Dashboard - Authentication - Database - Nouvelle connexion DB](/docs/images/cdy7uua7fh8z/224NpYTYh56bICfblpAJ6O/711d7fff6565d5dff7bde62ac5bf485a/Create_Database_Connections_-_English.png)</Frame>
  </Accordion>

  <Accordion icon="circle-3" title="Activez la connexion">
    Sélectionnez la vue **Applications**, puis activez le commutateur pour chaque application Auth0 qui doit pouvoir utiliser cette connexion.

    <Frame>![Dashboard - Connections - Database - Modifier - Vue Applications](/docs/images/cdy7uua7fh8z/Kjd9GRzWgHJis31lQVJlW/a9fcea0cc19b6602a94971b85b155bb0/2025-02-26_14-44-12.png)</Frame>
  </Accordion>
</AccordionGroup>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour permettre aux [applications tierces](/docs/fr-CA/get-started/applications/third-party-applications) d’authentifier des utilisateurs au moyen d’une connexion de base de données, la connexion doit être promue à l’échelle du domaine. Pour en savoir plus, consultez [Promouvoir les connexions au niveau du domaine](/docs/fr-CA/authenticate/identity-providers/promote-connections-to-domain-level).
</Callout>
