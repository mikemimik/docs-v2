---
title: "Révoquer l’accès aux API à l’aide des autorisations d’application"
---

En configurant votre autorisation d’application de façon à ce que les jetons que vous émettez n’aient qu’une courte durée de vie, vous pourrez, au moment de révoquer l’accès à une ressource protégée, simplement supprimer l’autorisation. À ce moment-là, la partie qui détient le <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> ne dispose que d’un délai limité entre la suppression de l’autorisation et l’expiration du jeton pour effectuer d’autres requêtes API. Comme il s’agit d’une option simple (et sûre) à mettre en œuvre, nous vous recommandons de révoquer l’accès à vos API et à vos autres ressources protégées en révoquant les autorisations d’application.

Si, par exemple, vous utilisez une [application](/docs/fr-CA/get-started/applications) machine à machine pour accéder à votre [API](/docs/fr-CA/get-started/apis) et qu’un partenaire appelle votre API, puis qu’à la fin de votre contrat actuel, vous décidez tous deux de ne pas renouveler votre partenariat. Vous souhaitez maintenant supprimer l’accès de votre partenaire à votre API. Le problème, toutefois, c’est que vous lui avez donné un jeton d’accès valide pendant un mois.

* Comment pourriez-vous configurer votre environnement Auth0 pour faciliter la gestion de telles situations à l’avenir ?

<div id="application-grants">
  ## Autorisations d’application
</div>

Le principal enjeu dans ce scénario est la durée de validité du jeton d’accès à l’API : un mois.

Par défaut, Auth0 émet des jetons d’accès valides pendant 24 heures. Définir la durée de vie du jeton à 24 heures signifie que votre partenaire devra répéter l’échange client credentials (ou tout autre type d’autorisation que vous avez mis en place) pour obtenir un nouveau jeton d’accès toutes les 24 heures. Pour refuser l’accès à votre partenaire en raison de l’expiration de votre contrat, vous pouvez simplement supprimer l’autorisation d’application afin que, lorsque son jeton actuel expirera, il ne puisse pas en demander un nouveau.

Vous pouvez modifier la durée de vie d’un jeton en définissant l’option `token_lifetime`. La durée de vie appropriée dépendra de votre cas d’utilisation, mais nous vous recommandons de régler cette valeur au minimum possible. Un bon point de départ pour la déterminer est le délai que vous jugez acceptable entre la suppression de l’autorisation et la dernière utilisation de l’API.

<div id="delete-an-application-grant">
  ### Supprimer une autorisation d’application
</div>

Pour supprimer une autorisation d’application, effectuez l’appel `DELETE` approprié à l’<Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> [point de terminaison Delete an Application Grant](https://auth0.com/docs/api/management/v2#!/Client_Grants/delete_client_grants_by_id). Dans le cadre de cet appel, vous devrez préciser l’ID de l’autorisation d’application que vous voulez supprimer. Vous pouvez obtenir cet ID au moyen du [point de terminaison Get all Application Grants](https://auth0.com/docs/api/management/v2#!/Client_Grants/get_client_grants) de la Management API.

Vous pouvez aussi [mettre à jour les types d’octroi d’une application](/docs/fr-CA/get-started/applications/update-grant-types) dans l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.&quot;
