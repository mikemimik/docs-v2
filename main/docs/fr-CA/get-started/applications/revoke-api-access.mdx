---
title: "Révoquer l’accès aux API à l’aide d’autorisations d’application"
---

En configurant votre autorisation d’application pour que les jetons que vous émettez aient une courte durée de vie, lorsque vient le temps de révoquer l’accès à une ressource protégée, vous pouvez simplement supprimer l’autorisation. À ce moment-là, la partie qui détient le <Tooltip tip="Jeton d’accès : identifiant d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> ne dispose que d’un délai limité entre la suppression de l’autorisation et l’expiration du jeton pour effectuer d’autres requêtes à l’API. Comme il s’agit d’une option simple à mettre en œuvre (et sécuritaire), nous vous recommandons de rejeter l’accès à vos API et à vos autres ressources protégées en révoquant les autorisations d’application.

Si, par exemple, vous utilisez une [application](/docs/fr-CA/get-started/applications) Machine-to-Machine pour accéder à votre [API](/docs/fr-CA/get-started/apis), et qu’un partenaire appelle votre API, puis qu’à la fin de votre contrat actuel, vous décidez ensemble de ne pas renouveler votre partenariat, vous voudrez alors retirer à ce partenaire l’accès à votre API. Le problème, toutefois, c’est que vous lui avez donné un jeton d’accès valide pendant un mois.

* Que pouvez-vous faire dans cette situation ?
* Comment pourriez-vous configurer votre environnement Auth0 pour que ce genre de situation soit plus facile à gérer à l’avenir ?

<div id="application-grants">
  ## Autorisations d’application
</div>

Le principal enjeu dans ce scénario est la durée de validité du jeton d’accès à l’API : un mois.

Par défaut, Auth0 émet des jetons d’accès valides pendant 24 heures. Régler la durée de vie du jeton à 24 heures signifie que votre partenaire doit répéter l’échange client credentials (ou tout autre type d’autorisation que vous avez mis en place) pour obtenir un nouveau jeton d’accès toutes les 24 heures. Pour rejeter l’accès accordé à votre partenaire en raison de l’expiration de votre contrat, vous pouvez simplement supprimer l’autorisation d’application afin que, lorsque son jeton actuel expirera, il ne puisse pas en demander un nouveau.

Vous pouvez modifier la durée de vie d’un jeton en définissant l’option `token_lifetime`. La durée précise qui convient à votre cas d’utilisation variera, mais nous vous recommandons de régler cette valeur à la durée la plus courte possible. Un bon point de départ pour déterminer cette valeur serait le délai que vous jugez acceptable entre la suppression de l’autorisation et la dernière utilisation de l’API.

<div id="delete-an-application-grant">
  ### Supprimer une autorisation d’application
</div>

Pour supprimer une autorisation d’application, effectuez l’appel `DELETE` approprié à l’<Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> [endpoint Delete an Application Grant](https://auth0.com/docs/api/management/v2#!/Client_Grants/delete_client_grants_by_id). Dans cet appel, vous devrez indiquer l’ID de l’autorisation d’application que vous voulez supprimer; vous pouvez l’obtenir au moyen de l’[endpoint Get all Application Grants](https://auth0.com/docs/api/management/v2#!/Client_Grants/get_client_grants) de la Management API.

Vous pouvez aussi [mettre à jour les types d’octroi d’une Application](/docs/fr-CA/get-started/applications/update-grant-types) dans l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.
