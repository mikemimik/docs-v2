title: "Delegation"
description: "Point de terminaison obsolète qui génère un jeton de délégation pour transférer l’identité d’un utilisateur d’une application vers un module complémentaire d’API."
---

<div id="endpoint">
  ## Point de terminaison
</div>

`POST /delegation`

<Note>
  Par défaut, cette fonctionnalité est désactivée pour les tenants qui n’utilisaient aucun module complémentaire au 8 juin 2017. Les tenants Legacy qui utilisent actuellement un module complémentaire nécessitant la délégation peuvent continuer à utiliser cette fonctionnalité. Si la fonctionnalité de délégation est modifiée ou retirée à un moment donné, les clients qui l’utilisent actuellement en seront avisés à l’avance et disposeront de suffisamment de temps pour migrer.
</Note>

Un jeton de délégation peut être obtenu et utilisé lorsqu’une application doit appeler l’API d’un module complémentaire d’application, comme Firebase ou SAP, enregistré et configuré dans Auth0, dans le même tenant que le programme appelant.

À partir d’un token existant, ce point de terminaison génère un nouveau token signé avec le secret de l’application `target`. Cela sert à transmettre l’identité de l’utilisateur de l’application vers une API.

<div id="remarks">
  ### Remarques
</div>

* La valeur de `scope` `profile` demande l&#39;accès aux claims de profil par défaut de l&#39;utilisateur final, soit : `name`, `family_name`, `given_name`, `middle_name`, `nickname`, `preferred_username`, `profile`, `picture`, `website`, `gender`, `birthdate`, `zoneinfo`, `locale` et `updated_at`.

* La valeur de `scope` `email` demande l&#39;accès aux claims `email` et `email_verified`.

* Delegation n&#39;est **pas pris en charge** dans la version 8 de [auth0.js](https://auth0.com/docs/libraries/auth0js). Pour voir un exemple avec la version 7 de la bibliothèque, consultez le [guide de référence d&#39;auth0.js v7](https://auth0.com/docs/libraries/auth0js/v7).

* Ce point de terminaison est limité à 10 requêtes par minute provenant de la même adresse IP et associées au même `user_id`.

* Ce point de terminaison retourne trois headers de réponse HTTP qui fournissent des données pertinentes sur ses limites de débit :
  * `X-RateLimit-Limit` : Nombre de requêtes autorisées par minute.
  * `X-RateLimit-Remaining` : Nombre de requêtes encore disponibles. Chaque nouvelle requête réduit ce nombre de 1. À chaque minute écoulée, une requête est rajoutée; ce nombre augmente donc de 1.
  * `X-RateLimit-Reset` : Temps restant avant la réinitialisation de la limite de débit (`X-RateLimit-Limit`). La valeur est exprimée en [secondes d&#39;époque UTC](https://en.wikipedia.org/wiki/Unix_time).
