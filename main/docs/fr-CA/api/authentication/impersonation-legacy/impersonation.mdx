<div id="endpoint">
  ## Endpoint
</div>

`POST /users/{user_id}/impersonate`

Utilisez cet endpoint pour obtenir une URL d’usurpation d’identité afin de vous connecter en tant qu’un autre utilisateur. Utile pour le dépannage.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison ne peut être utilisé qu’avec les informations d’authentification **Global Client**.
* Pour distinguer les ouvertures de session réelles des ouvertures de session par usurpation d’identité, le profil de l’utilisateur impersoné contiendra des propriétés supplémentaires `impersonated` et `impersonator`. Par exemple : `"impersonated": true, "impersonator": {"user_id": "auth0|...", "email": "admin@example.com"}`.
* Pour une application Web traditionnelle, vous devez définir les `additionalParameters` : attribuez à `response_type` la valeur `code`, à `callback_url` l’URL de rappel vers laquelle Auth0 redirigera avec le code d’autorisation, et à `scope` les claims JWT que vous voulez inclure dans le JWT.
