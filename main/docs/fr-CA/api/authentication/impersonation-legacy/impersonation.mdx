<div id="endpoint">
  ## Point de terminaison
</div>

`POST /users/{user_id}/impersonate`

Utilisez ce point de terminaison pour obtenir une URL d’usurpation d’identité et vous connecter comme un autre utilisateur. Utile pour le dépannage.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison ne peut être utilisé qu’avec les identifiants du **Global Client**.
* Pour distinguer les connexions réelles des connexions par usurpation d’identité, le profil de l’utilisateur usurpé contiendra des propriétés supplémentaires `impersonated` et `impersonator`. Par exemple : `"impersonated": true, "impersonator": {"user_id": "auth0|...", "email": "admin@example.com"}`.
* Pour une application Web traditionnelle, vous devez définir `additionalParameters` : définissez `response_type` sur `code`, `callback_url` sur l’URL de rappel vers laquelle Auth0 redirigera avec le code d’autorisation, et `scope` sur les claims JWT que vous souhaitez inclure dans le JWT.
