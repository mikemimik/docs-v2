<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/ro`

<Note>
  Ce point de terminaison fait partie du pipeline d’authentification hérité et a été remplacé par l’[octroi par mot de passe](#resource-owner-password). Pour plus d’information sur le pipeline d’authentification le plus récent, consultez [Introducing OIDC Conformant Authentication](/api-auth/intro).
</Note>

Utilisez ce point de terminaison pour l’authentification active par API. À partir des identifiants de l’utilisateur et de la `connection` indiquée, il effectuera l’authentification auprès du fournisseur et renverra un objet JSON contenant le jeton d’accès et le jeton d’ID.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison fonctionne uniquement avec les connexions de base de données, les connexions sans mot de passe, Active Directory/LDAP, Windows Azure AD et ADFS.

* La principale différence entre l’authentification passive et l’authentification active est que la première a lieu dans le navigateur, par l’intermédiaire de la [Auth0 Login Page](http://manage.auth0.com/login), tandis que la seconde peut être invoquée de n’importe où (par un script, de serveur à serveur, etc.).

* L’exemple de script auth0.js utilise la version 8 de la bibliothèque. Si vous utilisez la version 7 d’auth0.js, veuillez consulter ce [guide de référence](https://auth0.com/docs/libraries/auth0js/v7).

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre application.
</ParamField>

<ParamField body="username" type="string" required>
  Nom d’utilisateur ou adresse courriel de l’utilisateur pour se connecter.
</ParamField>

<ParamField body="password" type="string" required>
  Mot de passe de l’utilisateur pour se connecter.
</ParamField>

<ParamField body="connection" type="string" required>
  Le nom de la connexion à utiliser pour se connecter.
</ParamField>

<ParamField body="scope" type="string">
  Définissez cette valeur sur `openid` pour récupérer également un jeton d’ID.
</ParamField>

<ParamField body="grant_type" type="string" required>
  Définissez cette valeur sur `password` ou `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="device" type="string">
  Obligatoire lorsque `grant_type` est `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="id_token" type="string">
  Obligatoire lorsque `grant_type` est `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema statusCode="200">
  <ResponseField name="id_token" type="string">
    Le jeton ID.
  </ResponseField>

  <ResponseField name="access_token" type="string">
    Le jeton d’accès.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    Le type de jeton. Généralement `Bearer`.
  </ResponseField>
</ResponseSchema>

<ResponseSchema statusCode="400">
  <ResponseField name="error" type="string">
    Code d’erreur.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Description de l’erreur.
  </ResponseField>
</ResponseSchema>
