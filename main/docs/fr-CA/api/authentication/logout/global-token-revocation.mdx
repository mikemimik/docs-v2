---
title: "Révocation globale des jetons"
description: "Révoquez les cookies de session et les jetons d’actualisation d’un utilisateur dans l’ensemble des applications grâce à Universal Logout d’Okta."
---

<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/global-token-revocation/connection/{connection_name}`

Utilisez ce point de terminaison avec [Okta Workforce Identity Cloud Universal Logout](https://developer.okta.com/docs/guides/oin-universal-logout-overview/) pour déconnecter les utilisateurs de vos applications. Une requête à ce point de terminaison révoque les cookies de session et les jetons d’actualisation, mais pas les jetons d’accès. Pour en savoir plus, consultez [Universal Logout](https://auth0.com/docs/authenticate/login/logout/universal-logout). La requête doit être authentifiée avant la révocation des sessions utilisateur. Consultez [Authentification du point de terminaison](https://developer.okta.com/docs/guides/oin-universal-logout-overview/#endpoint-authentication).

<div id="path-parameters">
  ## Paramètres de chemin
</div>

<ParamField path="connection_name" type="string" required>
  Le nom de la connexion pour laquelle vous voulez révoquer la session de l’utilisateur.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="subject" type="object">
  <Expandable title="propriétés">
    <ParamField body="format" type="string" required />

    <ParamField body="iss" type="string" required>
      L’URL de l’émetteur (généralement l’URL de votre serveur d’autorisation).
    </ParamField>

    <ParamField body="sub" type="string" required>
      L’identifiant du sujet (généralement l’identifiant utilisateur).
    </ParamField>
  </Expandable>
</ParamField>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                                                                                 |
| ------ | ----------------------------------------------------------------------------------------------------------- |
| 200    | Cookies de session et jetons d’actualisation révoqués avec succès.                                          |
| 400    | Requête invalide, possiblement en raison d’un format `subject` invalide ou de l’absence d’authentification. |
| 401    | Non autorisé, l’authentification a échoué.                                                                  |
| 500    | Erreur interne du serveur.                                                                                  |
