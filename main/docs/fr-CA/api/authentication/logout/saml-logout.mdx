---
title: "Déconnexion SAML"
description: "Déconnecter un utilisateur d’un tenant Auth0 configuré en tant que fournisseur d’identité SAML."
---

<div id="endpoint">
  ## Point de terminaison
</div>

`POST /samlp/{client_id}/logout`

Utilisez ce point de terminaison pour déconnecter un utilisateur d’un tenant Auth0 configuré comme fournisseur d’identité SAML (IdP). Le comportement de déconnexion est déterminé par la configuration du module complémentaire SAML2 Web App pour l’application du tenant Auth0 agissant comme IdP SAML. Pour en savoir plus, consultez [Déconnecter les utilisateurs des fournisseurs d’identité SAML](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-saml-idps#configure-slo-when-auth0-is-the-saml-idp).

<div id="remarks">
  ### Remarques
</div>

* Le corps de la requête POST doit contenir un message SAML `<LogoutRequest>` valide. Pour en savoir davantage, consultez [Assertions and Protocols for the OASIS Security Assertion Markup Language (SAML) V2.0 on Oasis](https://docs.oasis-open.org/security/saml/v2.0/saml-core-2.0-os.pdf).

<div id="path-parameters">
  ## Paramètres du chemin
</div>

<ParamField path="client_id" type="string" required>
  ID client de votre application configurée avec le [module complémentaire SAML2 Web App](https://auth0.com/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon).
</ParamField>

  ## Messages de réponse
</div>

| Statut | Description                                                           |
| ------ | --------------------------------------------------------------------- |
| 200    | Déconnexion réussie                                                   |
| 400    | Requête incorrecte, probablement en raison d’un message SAML invalide |
