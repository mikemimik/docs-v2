<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/token`

Utilisez ce point de terminaison pour actualiser un jeton d’accès à l’aide du jeton d’actualisation obtenu lors de l’autorisation.

<div id="learn-more">
  ## En savoir plus
</div>

* [Jetons d’actualisation](https://auth0.com/docs/secure/tokens/refresh-tokens)

<div id="headers">
  ## En-têtes
</div>

<ParamField header="DPoP" type="string">
  Une preuve DPoP pour la requête. Ce paramètre est facultatif et n’est requis que si votre application utilise le mécanisme Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="grant_type" type="string" required>
  Indique le flux utilisé. Pour actualiser un jeton d’accès, cette valeur doit être définie sur `refresh_token`.

  Valeurs autorisées : `refresh_token`
</ParamField>

<ParamField body="client_id" type="string" required>
  Le Client ID de votre application, qui identifie l’application qui effectue la requête.
</ParamField>

<ParamField body="client_secret" type="string">
  Le Client Secret de votre application. Obligatoire lorsque la méthode d’authentification du point de terminaison des jetons est définie sur `Post` ou `Basic`.
</ParamField>

<ParamField body="refresh_token" type="string" required>
  Le jeton d’actualisation qui a déjà été émis au client. Ce token est utilisé pour obtenir un nouveau jeton d’accès.
</ParamField>

<ParamField body="scope" type="string">
  Une liste de scopes demandés, séparés par des espaces. S’il est omis, les scopes d’origine seront utilisés; sinon, vous pouvez demander un ensemble réduit de scopes. Notez que cette valeur doit être encodée dans l’URL.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    Le nouveau jeton d’accès émis.
  </ResponseField>

  <ResponseField name="refresh_token" type="string">
    Le jeton d’actualisation.
  </ResponseField>

  <ResponseField name="id_token" type="string">
    Le jeton d’identité, le cas échéant.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    Le type de jeton. Généralement `Bearer`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La durée de validité du jeton d’accès, en secondes.
  </ResponseField>

  <ResponseField name="scope" type="string">
    Les scopes du jeton d’accès.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                                                                                |
| ------ | ---------------------------------------------------------------------------------------------------------- |
| 200    | Le jeton d’accès a été actualisé avec succès.                                                              |
| 400    | Requête incorrecte. Il manque des paramètres requis à la requête ou celle-ci est mal formée.               |
| 401    | Non autorisé. Les identifiants du client ne sont pas valides ou le jeton d’actualisation n’est pas valide. |
| 403    | Interdit. Le jeton d’actualisation n’est pas valide ou a expiré.                                           |
