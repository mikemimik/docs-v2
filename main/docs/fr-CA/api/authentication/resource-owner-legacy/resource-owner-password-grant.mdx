<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/ro`

À partir des informations d’identification de l’utilisateur, ce point de terminaison authentifie l’utilisateur auprès du fournisseur et renvoie un objet JSON contenant le jeton d’accès et un ID Token.

<Note>
  Ce point de terminaison fait partie du pipeline d’authentification hérité et a été remplacé par le [Password Grant](#resource-owner-password). Pour en savoir plus sur le pipeline d’authentification le plus récent, consultez [Présentation de l’authentification conforme à OIDC](/api-auth/intro).
</Note>

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison fonctionne uniquement avec les connexions de base de données, les connexions Passwordless, Active Directory/LDAP, Windows Azure AD et ADFS.
* Le scope `profile` demande l’accès aux claims par défaut du profil de l’utilisateur final, notamment `name`, `family_name`, `given_name` et d’autres.
* Le scope `email` demande l’accès aux claims `email` et `email_verified`.
