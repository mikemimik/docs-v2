<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/device/code`

Ce flux est conçu pour permettre aux appareils aux capacités de saisie limitées d’accéder à une API. Utilisez ce point de terminaison pour obtenir un code d’appareil qui permettra à l’utilisateur d’autoriser l’appareil.

<div id="request-example">
  ## Exemple de requête
</div>

```http
POST https://{yourDomain}/oauth/device/code
Content-Type: application/x-www-form-urlencoded

client_id=${account.clientId}&scope=SCOPE&audience=API_IDENTIFIER
```

<div id="response-values">
  ### Valeurs de réponse
</div>

| Valeur                      | Description                                                                    |
| :-------------------------- | :----------------------------------------------------------------------------- |
| `device_code`               | Le code unique de l’appareil.                                                  |
| `user_code`                 | Le code que l’utilisateur doit saisir pour autoriser l’appareil.               |
| `verification_uri`          | L’URL que l’utilisateur doit consulter pour autoriser l’appareil.              |
| `verification_uri_complete` | L’URL complète, y compris le code utilisateur, pour y accéder facilement.      |
| `expires_in`                | La durée de validité des codes de l’appareil et de l’utilisateur, en secondes. |
| `interval`                  | L’intervalle d’interrogation, en secondes, pour demander un jeton.             |

<div id="remarks">
  ### Remarques
</div>

* Incluez `offline_access` dans le `scope` pour obtenir un Refresh Token.
* Utilisez le code d’appareil renvoyé pour demander un jeton d’accès au token endpoint.

<div id="token-request-example">
  ### Exemple de requête de jeton
</div>

```http
POST https://{yourDomain}/oauth/token
Content-Type: application/x-www-form-urlencoded

client_id=${account.clientId}&device_code=YOUR_DEVICE_CODE&grant_type=urn:ietf:params:oauth:grant-type:device_code
```

<div id="responses">
  ### Réponses
</div>

<div id="200">
  #### 200
</div>

Une requête réussie renvoie un jeton d’accès.

```json
HTTP/1.1 200 OK
Content-Type: application/json
{
   "access_token": "eyJz93a...k4laUWw",
   "id_token": "eyJ...0NE",
   "refresh_token": "eyJ...MoQ",
   "expires_in": 86400,
   "token_type": "Bearer"
}
```

<div id="error-responses">
  ### Réponses d’erreur
</div>

```json
HTTP/1.1 403 Forbidden
Content-Type: application/json
{ "error": "authorization_pending", "error_description": "User has yet to authorize device code." }
```

```json
HTTP/1.1 429 Too Many Requests
Content-Type: application/json
{ "error": "slow_down", "error_description": "You are polling faster than the specified interval of 5 seconds." }
```

```json
HTTP/1.1 403 Forbidden
Content-Type: application/json
{ "error": "access_denied", "error_description": "User cancelled the confirmation prompt." }
```

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  L’ID de votre application.
</ParamField>

<ParamField body="scope" type="string">
  Les portées d’autorisation que vous souhaitez demander.
</ParamField>

<ParamField body="audience" type="string">
  L’identifiant unique de l’API cible à laquelle vous souhaitez accéder.
</ParamField>

<ParamField body="resource" type="string">
  L’identifiant de l’API cible (serveur de ressources) à laquelle vous souhaitez accéder. Il doit correspondre à un identifiant d’API enregistré dans votre tenant Auth0. Il est utilisé comme alternative à `audience` lorsque le [profil de compatibilité du paramètre de ressource](https://auth0.com/docs/get-started/tenant-settings#settings-advanced) du tenant est défini sur `compatibility`.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="device_code" type="string">
    Le code unique de l’appareil. Lorsque l’utilisateur accède au `verification_uri` dans le navigateur de son appareil, ce code est associé à sa session.
  </ResponseField>

  <ResponseField name="user_code" type="string">
    Le code que l’utilisateur doit saisir à l’adresse `verification_uri`.
  </ResponseField>

  <ResponseField name="verification_uri" type="string">
    L’URL que l’utilisateur doit consulter pour autoriser l’appareil.
  </ResponseField>

  <ResponseField name="verification_uri_complete" type="string">
    L’URL complète que l’utilisateur doit consulter, y compris le `user_code`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La durée de validité, en secondes, du `device_code` et du `user_code`.
  </ResponseField>

  <ResponseField name="interval" type="integer">
    L’intervalle, en secondes, auquel l’application doit interroger pour obtenir un jeton.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                    |
| ------ | ---------------------------------------------- |
| 200    | Renvoie les codes d’appareil et d’utilisateur. |
