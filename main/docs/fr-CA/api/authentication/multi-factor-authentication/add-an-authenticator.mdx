<div id="endpoint">
  ## Point de terminaison
</div>

`POST /mfa/associate`

Associe ou ajoute un nouvel authentificateur pour l’authentification multifacteur (MFA).

Si l’utilisateur a des authentificateurs actifs, un Access Token avec le scope `enroll` et l’`audience` définie à `https://{yourDomain}/mfa/` est requis pour utiliser ce point de terminaison.

Si l’utilisateur n’a aucun authentificateur actif, vous pouvez utiliser le `mfa_token` de l’erreur `mfa_required` au lieu d’un Access Token pour cette requête.

Après l’ajout d’un authentificateur, celui-ci doit être vérifié. Pour le vérifier, utilisez les valeurs de réponse de la requête `/mfa/associate` au lieu de celles renvoyées par le point de terminaison `/mfa/challenge`, puis poursuivez le flux de vérification.

Un champ `recovery_codes` est inclus dans la réponse la première fois qu’un authentificateur est ajouté. Vous pouvez utiliser `recovery_codes` pour effectuer l’authentification multifacteur, comme indiqué ci-dessus dans [Verify with recovery code](/multi-factor-authentication/verify-with-recovery-code).

Pour accéder à ce point de terminaison, vous devez définir un Access Token dans l’en-tête Authorization, avec les claims suivantes :

* `scope`: `enroll`
* `audience`: `https://{yourDomain}/mfa/`

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  Le Client ID de votre application.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT contenant une assertion signée avec les informations d’identification de votre application. Obligatoire lorsque Private Key JWT est votre méthode d’authentification de l’application.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  La valeur est `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Obligatoire lorsque Private Key JWT est la méthode d’authentification de l’application.
</ParamField>

<ParamField body="client_secret" type="string" required>
  Le Client Secret de votre application. Obligatoire lorsque le champ Token Endpoint Authentication Method dans vos Paramètres de l’application est défini sur `Post` ou `Basic`.
</ParamField>

<ParamField body="authenticator_types" type="array" required>
  La valeur est un tableau contenant les valeurs &#39;otp&#39; ou &#39;oob&#39;.
</ParamField>

<ParamField body="oob_channels" type="array">
  Obligatoire si `authenticator_types` comprend `oob`.
</ParamField>

<ParamField body="phone_number" type="string">
  Le numéro de téléphone à utiliser pour SMS ou Voice. Obligatoire si `oob_channels` comprend `sms` ou `voice`.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="oob_code" type="string">
    Le code pour l’authentification hors bande.
  </ResponseField>

  <ResponseField name="binding_method" type="string">
    Indique la méthode de liaison.
  </ResponseField>

  <ResponseField name="authenticator_type" type="string">
    Le type d’authentificateur.
  </ResponseField>

  <ResponseField name="oob_channels" type="string">
    Les canaux OOB utilisés par l’authentificateur.
  </ResponseField>

  <ResponseField name="recovery_codes" type="array">
    Un tableau de codes de récupération générés par Auth0 pour l’utilisateur. Lorsqu’un utilisateur ajoute un authentificateur, les codes de récupération ne s’affichent qu’une seule fois.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                                          |
| ------ | -------------------------------------------------------------------- |
| 200    | Réponse réussie pour l’ajout d’un authentificateur.                  |
| 400    | Requête incorrecte en raison de paramètres manquants ou non valides. |
| 401    | Non autorisé, jeton d’accès ou mfa&#95;token invalide.               |
