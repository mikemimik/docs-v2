<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/token`

Vérifie l’authentification multifacteur (MFA) à l’aide d’un mot de passe à usage unique (OTP). Pour vérifier la MFA avec un OTP, invitez l’utilisateur à récupérer le code OTP, puis envoyez une requête à l’endpoint /oauth/token. La requête doit contenir le code OTP, le mfa&#95;token reçu (dans l’erreur mfa&#95;required) et le grant&#95;type défini sur http://auth0.com/oauth/grant-type/mfa-otp. La réponse est identique à celle des grant types password ou http://auth0.com/oauth/grant-type/password-realm. Pour en savoir plus, consultez [Associer des authentificateurs OTP](/mfa/guides/mfa-api/otp).

<div id="headers">
  ## En-têtes
</div>

<ParamField header="DPoP" type="string">
  Une preuve DPoP pour la requête. Ce champ est facultatif et n’est requis que si votre application utilise Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="grant_type" type="string" required>
  Indique le flux utilisé. Pour l’OTP MFA, utilisez `http://auth0.com/oauth/grant-type/mfa-otp`.
</ParamField>

<ParamField body="client_id" type="string">
  Le Client ID de votre application.
</ParamField>

<ParamField body="client_secret" type="string">
  Le Client Secret de votre application.
</ParamField>

<ParamField body="mfa_token" type="string" required>
  Le `mfa_token` que vous avez reçu dans l’erreur `mfa_required`.
</ParamField>

<ParamField body="otp" type="string" required>
  Le code OTP fourni par l’utilisateur.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT contenant une assertion signée avec les identifiants de votre application.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  La valeur est `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    Le jeton d’accès de la session authentifiée.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    Le type de jeton délivré.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La durée de validité du jeton d’accès, en secondes.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                                                           |
| ------ | ------------------------------------------------------------------------------------- |
| 200    | Vérification de l’OTP réussie.                                                        |
| 400    | Requête invalide, par exemple en raison de paramètres manquants ou d’un OTP invalide. |
