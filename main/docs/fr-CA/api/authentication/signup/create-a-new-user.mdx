<div id="endpoint">
  ## Point de terminaison
</div>

`POST /dbconnections/signup`

À partir des identifiants d’un utilisateur et d’une `connection`, ce point de terminaison crée un nouvel utilisateur.

Ce point de terminaison fonctionne uniquement avec les connexions de base de données.

<div id="remarks">
  ### Remarques
</div>

* Le script d’exemple auth0.js utilise la version 8 de la bibliothèque. Si vous utilisez la version 7 d’auth0.js, veuillez consulter ce [guide de référence](https://auth0.com/docs/libraries/auth0js/v7).

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre client.
</ParamField>

<ParamField body="email" type="string">
  L’adresse courriel de l’utilisateur.
</ParamField>

<ParamField body="password" type="string" required>
  Le mot de passe souhaité par l’utilisateur.
</ParamField>

<ParamField body="phone_number" type="string">
  Le numéro de téléphone de l’utilisateur.
</ParamField>

<ParamField body="connection" type="string" required>
  Le nom de la base de données configurée pour votre client.
</ParamField>

<ParamField body="username" type="string">
  Le nom d’utilisateur de l’utilisateur. Valide uniquement si la connexion exige un nom d’utilisateur.
</ParamField>

<ParamField body="given_name" type="string">
  Le ou les prénoms de l’utilisateur.
</ParamField>

<ParamField body="family_name" type="string">
  Le ou les noms de famille de l’utilisateur.
</ParamField>

<ParamField body="name" type="string">
  Le nom complet de l’utilisateur.
</ParamField>

<ParamField body="nickname" type="string">
  Le surnom de l’utilisateur.
</ParamField>

<ParamField body="picture" type="string">
  Un URI pointant vers la photo de l’utilisateur.
</ParamField>

<ParamField body="user_metadata" type="object">
  Les métadonnées utilisateur à associer à l’utilisateur. S’il est défini, le champ doit être un objet ne contenant pas plus de dix propriétés. Les noms de propriétés peuvent comporter au maximum 100 caractères, et les valeurs de propriétés doivent être des chaînes de caractères d’au plus 500 caractères.
</ParamField>
