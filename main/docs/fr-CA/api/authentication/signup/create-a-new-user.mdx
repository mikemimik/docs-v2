<div id="endpoint">
  ## Endpoint
</div>

`POST /dbconnections/signup`

À partir de l’identifiant d’un utilisateur et d’une `connection`, cet endpoint crée un nouvel utilisateur.

Cet endpoint fonctionne uniquement avec les connexions de base de données.

<div id="remarks">
  ### Remarques
</div>

* L’exemple de script auth0.js utilise la version 8 de la bibliothèque. Si vous utilisez la version 7 d’auth0.js, veuillez consulter ce [guide de référence](https://auth0.com/docs/libraries/auth0js/v7).

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre client.
</ParamField>

<ParamField body="email" type="string">
  L’adresse courriel de l’utilisateur.
</ParamField>

<ParamField body="password" type="string" required>
  Le mot de passe souhaité par l’utilisateur.
</ParamField>

<ParamField body="phone_number" type="string">
  Le numéro de téléphone de l’utilisateur.
</ParamField>

<ParamField body="connection" type="string" required>
  Le nom de la connexion de base de données configurée pour votre client.
</ParamField>

<ParamField body="username" type="string">
  Le nom d’utilisateur. Valide uniquement si la connexion exige un nom d’utilisateur.
</ParamField>

<ParamField body="given_name" type="string">
  Le ou les prénoms de l’utilisateur.
</ParamField>

<ParamField body="family_name" type="string">
  Le ou les noms de famille de l’utilisateur.
</ParamField>

<ParamField body="name" type="string">
  Le nom complet de l’utilisateur.
</ParamField>

<ParamField body="nickname" type="string">
  Le surnom de l’utilisateur.
</ParamField>

<ParamField body="picture" type="string">
  Un URI pointant vers la photo de profil de l’utilisateur.
</ParamField>

<ParamField body="user_metadata" type="object">
  Les métadonnées utilisateur à associer à l’utilisateur. Si ce champ est défini, il doit s’agir d’un objet ne contenant pas plus de dix propriétés. Les noms de propriété peuvent comporter un maximum de 100 caractères, et les valeurs de propriété doivent être des chaînes d’au plus 500 caractères.
</ParamField>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                           |
| ------ | ----------------------------------------------------- |
| 200    | Utilisateur créé avec succès                          |
| 400    | Requête invalide : paramètres manquants ou invalides. |
| 409    | Conflit : l’utilisateur existe déjà.                  |
