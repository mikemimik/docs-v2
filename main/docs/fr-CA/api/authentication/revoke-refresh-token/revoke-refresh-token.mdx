<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/revoke`

Utilisez ce point de terminaison pour invalider un jeton d’actualisation s’il a été compromis.

Le comportement de ce point de terminaison dépend de l’état de la bascule [Refresh Token Revocation Deletes Grant](https://auth0.com/docs/secure/tokens/refresh-tokens/revoke-refresh-tokens#choose-whether-token-revocation-deletes-grants).
Si cette bascule est activée, chaque requête de révocation invalide non seulement le jeton visé, mais aussi tous les autres jetons associés au même grant d’autorisation. Cela signifie que **tous les jetons d’actualisation émis pour le même utilisateur, la même application et la même audience seront révoqués**.
Si cette bascule est désactivée, seul le jeton d’actualisation est révoqué, tandis que le grant demeure intact.

<div id="remarks">
  ## Remarques
</div>

* Pour les applications non confidentielles qui ne peuvent pas protéger le Client Secret (par exemple, les applications natives), l’endpoint permet de ne pas transmettre de Client Secret, mais l’application elle-même doit avoir la propriété `tokenEndpointAuthMethod` définie à `none`. Vous pouvez le faire soit dans l’UI ([Dashboard &gt; Applications &gt; Application Settings](https://manage.auth0.com/dashboard)), soit à l’aide de la [Management API](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id).

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre application.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT contenant une assertion signée avec les identifiants de votre application. Obligatoire lorsque Private Key JWT est la méthode d’authentification de l’application.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  La valeur est `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Obligatoire lorsque Private Key JWT est la méthode d’authentification de l’application.
</ParamField>

<ParamField body="client_secret" type="string">
  Le `client_secret` de votre application. Obligatoire lorsque Client Secret Basic ou Client Secret Post est la méthode d’authentification de l’application. Requis spécifiquement pour les applications Web traditionnelles **uniquement**.
</ParamField>

<ParamField body="token" type="string" required>
  Le jeton d’actualisation que vous souhaitez révoquer.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  Une réponse 200 en cas de succès ne contient aucun corps. Les réponses d’erreur utilisent le schéma suivant :

  <ResponseField name="error" type="string">
    Code d’erreur.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Description de l’erreur.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut  | Description                |
| ------- | -------------------------- |
| 200     | (corps de la réponse vide) |
| 400     | Requête incorrecte         |
| 401     | Non autorisé               |
| default | Erreur inattendue          |
