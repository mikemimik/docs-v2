<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/token`

Lorsque les utilisateurs sont redirigés vers votre URL de rappel, vous devez effectuer une requête `POST` au point de terminaison `oauth/token` pour échanger un code d’autorisation contre un jeton d’accès ou un jeton d’identité, ou les deux.

<div id="remarks">
  ### Remarques
</div>

Pour envoyer une requête au point de terminaison `/oauth/token`, vous devez :

* Définir le type de contenu de la requête sur `application/x-www-form-urlencoded`
* Utiliser des chaînes de caractères pour tous les paramètres transmis
* Inclure un paramètre supplémentaire d’authentification de l’application dans la requête (p. ex. `client_secret`, ou `client_assertion` et `client_assertion_type` pour l’authentification du client par JSON Web Token, ou transmettre les en-têtes `client-certificate` et `client-certificate-ca-verified` lorsque vous utilisez Mutual TLS).

<div id="headers">
  ## En-têtes
</div>

<ParamField header="DPoP" type="string">
  Une preuve DPoP pour la requête. Ce paramètre est facultatif et n’est requis que si votre application utilise Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="grant_type" type="string" required>
  Indique le flux; utilisez `authorization_code` avec un code d’autorisation valide.
</ParamField>

<ParamField body="code" type="string">
  Le code d’autorisation de la requête initiale à `/authorize`.
</ParamField>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre application. Obligatoire.
</ParamField>

<ParamField body="request_uri" type="string">
  Ce paramètre est requis uniquement s’il est défini sur le point de terminaison `GET` `/oauth/par`.
</ParamField>

<ParamField body="code_verifier" type="string">
  Clé aléatoire sécurisée sur le plan cryptographique, utilisée pour générer le `code_challenge`. Recommandée si `code_challenge` a été fourni.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    Le jeton d’accès permettant d’accéder aux ressources protégées.
  </ResponseField>

  <ResponseField name="refresh_token" type="string">
    Jeton permettant d’obtenir de nouveaux jetons d’accès.
  </ResponseField>

  <ResponseField name="id_token" type="string">
    Le jeton d’identité.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    Le type de jeton. Généralement, `Bearer`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La durée de validité du jeton d’accès, en secondes.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                           |
| ------ | ------------------------------------- |
| 200    | Réponse réussie contenant les jetons. |
