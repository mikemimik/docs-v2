---

<div id="endpoint">
  ## Endpoint
</div>

`GET /samlp/{clientId}`

Utilisez cet endpoint pour accepter une requête SAML afin d’amorcer une connexion.

Vous pouvez aussi inclure un paramètre `connection` pour vous connecter avec un fournisseur précis. Si aucune connexion n’est précisée, la [Auth0 Login Page](https://auth0.com/docs/authenticate/login/auth0-universal-login) s’affichera.

Au besoin, les requêtes de connexion initiées par le SP peuvent inclure un paramètre `organization` pour authentifier les utilisateurs dans le contexte d’une organisation. Pour en savoir plus, consultez [Organizations](https://auth0.com/docs/manage-users/organizations).

<div id="path-parameters">
  ## Paramètres du chemin
</div>

<ParamField path="clientId" type="string" required>
  Client ID de votre application.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="connection" type="string">
  Connexion à utiliser pour la connexion.
</ParamField>

<ParamField body="organization" type="string">
  Identifiant de l’organisation, si l’authentification s’effectue dans le contexte d’une organisation.
</ParamField>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                        |
| ------ | ---------------------------------- |
| 200    | Requête SAML acceptée avec succès. |
