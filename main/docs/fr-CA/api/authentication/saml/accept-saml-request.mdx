title: "Accepter une requête"
description: "Accepter une requête d’authentification SAML pour lancer un processus de connexion."
---

<div id="endpoint">
  ## Point de terminaison
</div>

`GET /samlp/{clientId}`

Utilisez ce point de terminaison pour accepter une requête SAML afin d’amorcer une connexion.

Vous pouvez aussi inclure un paramètre `connection` pour vous connecter au moyen d’un fournisseur précis. Si aucune connexion n’est précisée, la [Auth0 Login Page](https://auth0.com/docs/authenticate/login/auth0-universal-login) s’affichera.

Les requêtes de connexion initiées par le SP peuvent aussi inclure un paramètre `organization` pour authentifier les utilisateurs dans le contexte d’une organisation. Pour en savoir plus, consultez [Organizations](https://auth0.com/docs/manage-users/organizations).

<div id="path-parameters">
  ## Paramètres de chemin
</div>

<ParamField path="clientId" type="string" required>
  Le Client ID de votre application.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="connection" type="string">
  Connexion à utiliser pour la connexion.
</ParamField>

<ParamField body="organization" type="string">
  ID de l’organisation, si l’authentification se fait dans le contexte d’une organisation.
</ParamField>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                        |
| ------ | ---------------------------------- |
| 200    | Requête SAML acceptée avec succès. |
