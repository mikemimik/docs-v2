---

<div id="endpoint">
  ## Point de terminaison
</div>

`GET /samlp/{clientId}`

Utilisez ce point de terminaison pour accepter une requête SAML afin d’amorcer une connexion.

Vous pouvez également inclure un paramètre `connection` pour vous connecter au moyen d’un fournisseur précis. Si aucune connexion n’est précisée, la [Auth0 Login Page](https://auth0.com/docs/authenticate/login/auth0-universal-login) s’affichera.

Les requêtes de connexion initiées par le SP peuvent également inclure un paramètre `organization` pour authentifier les utilisateurs dans le contexte d’une organisation. Pour en savoir plus, consultez [Organizations](https://auth0.com/docs/manage-users/organizations).

<div id="path-parameters">
  ## Paramètres du chemin
</div>

<ParamField path="clientId" type="string" required>
  Le Client ID de votre application.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="connection" type="string">
  Connexion à utiliser lors de la connexion.
</ParamField>

<ParamField body="organization" type="string">
  ID de l’organisation, si l’authentification se fait dans le contexte d’une organisation.
</ParamField>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                             |
| ------ | --------------------------------------- |
| 200    | Acceptation réussie de la requête SAML. |
