<div id="endpoint">
  ## Point de terminaison
</div>

`POST /tokeninfo`

<Note>
  Ce point de terminaison fait partie du pipeline d’authentification héritée et sera désactivé pour les personnes qui utilisent notre plus récent pipeline conforme à OIDC. Nous vous encourageons plutôt à utiliser le [point de terminaison /userinfo](#get-user-info). Pour en savoir plus sur le plus récent pipeline d’authentification, consultez [Introducing OIDC Conformant Authentication](/api-auth/intro).
</Note>

Ce point de terminaison valide un JSON Web Token (JWT) (signature et expiration) et renvoie les renseignements sur l’utilisateur associés à la propriété `sub` (ID utilisateur) du token.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison renverra trois en-têtes de réponse HTTP qui fournissent des données pertinentes sur ses limites de débit :
  * `X-RateLimit-Limit` : Nombre de requêtes autorisées par minute.
  * `X-RateLimit-Remaining` : Nombre de requêtes disponibles. Chaque nouvelle requête réduit ce nombre de 1. À chaque minute qui passe, des requêtes sont ajoutées de nouveau; ce nombre augmente donc de 1 chaque fois.
  * `X-RateLimit-Reset` : Temps restant avant la réinitialisation de la limite de débit (`X-RateLimit-Limit`). La valeur est en [secondes depuis l’époque Unix (UTC)](https://en.wikipedia.org/wiki/Unix_time).

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="email_verified" type="boolean">
    Indique si l’adresse courriel de l’utilisateur a été vérifiée.
  </ResponseField>

  <ResponseField name="email" type="string">
    L’adresse courriel de l’utilisateur.
  </ResponseField>

  <ResponseField name="clientID" type="string">
    L’ID client associé au jeton.
  </ResponseField>

  <ResponseField name="updated_at" type="string">
    Date et heure de la dernière mise à jour des renseignements de l’utilisateur.
  </ResponseField>

  <ResponseField name="name" type="string">
    Le nom complet de l’utilisateur.
  </ResponseField>

  <ResponseField name="picture" type="string">
    L’URL de la photo de profil de l’utilisateur.
  </ResponseField>

  <ResponseField name="user_id" type="string">
    L’identifiant unique de l’utilisateur.
  </ResponseField>

  <ResponseField name="nickname" type="string">
    Le pseudonyme de l’utilisateur.
  </ResponseField>

  <ResponseField name="provider" type="string">
    Le fournisseur d’identité de l’utilisateur.
  </ResponseField>

  <ResponseField name="connection" type="string">
    La connexion utilisée pour authentifier l’utilisateur.
  </ResponseField>

  <ResponseField name="isSocial" type="boolean">
    Indique si l’utilisateur s’est authentifié au moyen d’un fournisseur social.
  </ResponseField>

  <ResponseField name="created_at" type="string">
    Date et heure de création de l’utilisateur.
  </ResponseField>

  <ResponseField name="global_client_id" type="string">
    L’ID client global.
  </ResponseField>
</ResponseSchema>
