<div id="endpoint">
  ## Point de terminaison
</div>

`POST /tokeninfo`

<Note>
  Ce point de terminaison fait partie du pipeline d’authentification héritée et sera désactivé pour les utilisateurs de notre plus récent pipeline conforme à OIDC. Nous vous recommandons d’utiliser plutôt le [point de terminaison /userinfo](#get-user-info). Pour en savoir plus sur le plus récent pipeline d’authentification, consultez [Introducing OIDC Conformant Authentication](/api-auth/intro).
</Note>

Ce point de terminaison valide un JSON Web Token (JWT) (signature et expiration) et renvoie les renseignements sur l’utilisateur associés à la propriété `sub` (ID utilisateur) du jeton.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison renvoie trois en-têtes de réponse HTTP qui fournissent des données pertinentes sur ses limites de débit :
  * `X-RateLimit-Limit` : Nombre de requêtes autorisées par minute.
  * `X-RateLimit-Remaining` : Nombre de requêtes disponibles. Chaque nouvelle requête réduit ce nombre de 1. Chaque minute écoulée rétablit une requête, donc ce nombre augmente de 1 à chaque fois.
  * `X-RateLimit-Reset` : Temps restant avant la réinitialisation de la limite de débit (`X-RateLimit-Limit`). La valeur est exprimée en [secondes d’époque UTC](https://en.wikipedia.org/wiki/Unix_time).

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="email_verified" type="boolean">
    Indique si l’adresse courriel de l’utilisateur a été vérifiée.
  </ResponseField>

  <ResponseField name="email" type="string">
    L’adresse courriel de l’utilisateur.
  </ResponseField>

  <ResponseField name="clientID" type="string">
    Le client ID associé au jeton.
  </ResponseField>

  <ResponseField name="updated_at" type="string">
    Date et heure de la dernière mise à jour des renseignements de l’utilisateur.
  </ResponseField>

  <ResponseField name="name" type="string">
    Le nom complet de l’utilisateur.
  </ResponseField>

  <ResponseField name="picture" type="string">
    URL de la picture de profil de l’utilisateur.
  </ResponseField>

  <ResponseField name="user_id" type="string">
    L’identifiant unique de l’utilisateur.
  </ResponseField>

  <ResponseField name="nickname" type="string">
    Le nickname de l’utilisateur.
  </ResponseField>

  <ResponseField name="provider" type="string">
    L’identity provider de l’utilisateur.
  </ResponseField>

  <ResponseField name="connection" type="string">
    La connection utilisée pour authentifier l’utilisateur.
  </ResponseField>

  <ResponseField name="isSocial" type="boolean">
    Indique si l’utilisateur s’est authentifié au moyen d’un social provider.
  </ResponseField>

  <ResponseField name="created_at" type="string">
    Date et heure de création de l’utilisateur.
  </ResponseField>

  <ResponseField name="global_client_id" type="string">
    Le client ID global.
  </ResponseField>
</ResponseSchema>
