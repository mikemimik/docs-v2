<div id="endpoint">
  ## Point de terminaison
</div>

`GET /wsfed/{clientId}`

Ce point de terminaison accepte une requête WS-Federation pour amorcer une connexion.

<div id="remarks">
  ### Remarques
</div>

* Le paramètre `wtrealm` doit être dans l’un de ces formats :
  * `urn:clientID` (par exemple, `urn:${account.clientId}`)
  * Si ce paramètre ne commence pas par `urn`, le tableau `client.clientAliases` est utilisé pour la recherche. Ce paramètre ne peut être défini qu’avec la Management API [/api/v2/clients](https://auth0.com/docs/api/management/v2/clients/get-clients).
* Le paramètre `whr` est mappé à la connexion comme suit : `urn:CONNECTION_NAME`. Par exemple, `urn:google-oauth2` indique une connexion avec Google. Si aucun paramètre `whr` n’est inclus, l’utilisateur sera redirigé vers la [Auth0 Login Page](https://auth0.com/docs/login_page).

<div id="path-parameters">
  ## Paramètres de chemin
</div>

<ParamField path="client-id" type="string" required>
  Le `client-id` de votre application.
</ParamField>

<div id="query-parameters">
  ## Paramètres de requête
</div>

<ParamField query="wtrealm" type="string">
  Peut être utilisé à la place de `client-id`.
</ParamField>

<ParamField query="whr" type="string">
  Le nom de la connexion (utilisé pour contourner la page de connexion).
</ParamField>

<ParamField query="wctx" type="string">
  L’état de votre application.
</ParamField>

<ParamField query="wreply" type="string">
  L’URL de rappel.
</ParamField>

  ## Messages de réponse
