<div id="endpoint">
  ## Point de terminaison
</div>

`GET /authorize`

<div id="social">
  ## Social
</div>

Vous pouvez connecter votre service Auth0 à un fournisseur d’identité sociale et permettre à vos utilisateurs de se connecter à votre application avec Facebook, Google, Apple ou d’autres fournisseurs pris en charge. Pour en savoir plus sur les fournisseurs pris en charge, consultez [Marketplace](https://marketplace.auth0.com/features/social-connections).

Pour authentifier les utilisateurs avec un fournisseur d’identité sociale, effectuez une requête `GET` vers le point de terminaison `/authorize`. Cette requête renverra une redirection `302` vers le fournisseur d’identité sociale indiqué dans le paramètre `connection`.

<Note>
  Les connexions sociales prennent uniquement en charge l’authentification par navigateur (passive), car la plupart des fournisseurs d’identité sociale n’autorisent pas la saisie d’un nom d’utilisateur et mot de passe dans des applications qui ne leur appartiennent pas. L’utilisateur sera donc redirigé vers la page de connexion du fournisseur.
</Note>

<div id="remarks">
  ### Remarques
</div>

* Si `response_type=token`, une fois l’utilisateur authentifié auprès du fournisseur, il sera redirigé vers l’`URL de rappel` de votre application, avec le jeton d’accès et le jeton d’identité transmis dans le `location.hash` de l’adresse. Cela est utilisé pour les applications monopages ainsi que pour les SDK mobiles natifs.

* L’exemple de script auth0.js utilise la version 8 de la bibliothèque. Si vous utilisez la version 7 d’auth0.js, veuillez consulter ce [guide de référence](https://auth0.com/docs/libraries/auth0js).

<div id="query-parameters">
  ## Paramètres de requête
</div>

<ParamField query="response_type" type="string" required>
  Indique le type de token (`code` ou `token`)

  Valeurs autorisées : `code`, `token`
</ParamField>

<ParamField query="client_id" type="string" required>
  Le client&#95;id de votre application
</ParamField>

<ParamField query="connection" type="string">
  Le nom d’un fournisseur d’identité sociale configuré pour votre application
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  L’URL vers laquelle Auth0 redirigera après l’autorisation
</ParamField>

<ParamField query="state" type="string">
  Valeur opaque pour prévenir les attaques CSRF
</ParamField>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                     |
| ------ | ----------------------------------------------- |
| 302    | Redirige vers le fournisseur d’identité sociale |
