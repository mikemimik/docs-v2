Si vous utilisez la structure de base de données hébergée par Auth0 pour stocker vos utilisateurs, vous pouvez tester la connexion en authentifiant les utilisateurs dans le même tenant ou dans un tenant distinct.

<div id="test-with-import-users-to-auth0-enabled">
  ## Test avec Import Users to Auth0 activé
</div>

Lorsque l’option Import Users to Auth0 est activée, les utilisateurs sont progressivement migrés de votre référentiel d’utilisateurs externe vers le répertoire d’utilisateurs Auth0. Pendant ce processus, les utilisateurs sont d’abord authentifiés auprès du répertoire externe la première fois qu’ils se connectent avec succès. Toutes les tentatives de connexion suivantes sont ensuite effectuées auprès du répertoire d’utilisateurs Auth0. Pour en savoir plus, consultez la section sur les migrations automatiques dans [Import and Export Users](/docs/fr-CA/manage-users/user-migration).

<div id="create-a-test-tenant-and-application">
  ### Créez un tenant et une application de test
</div>

1. Utilisez l’Auth0 Dashboard pour [créer un nouveau tenant](/docs/fr-CA/get-started/auth0-overview/create-tenants).
2. Accédez à [Applications &gt; Applications](https://manage.auth0.com/#/applications/applications) et créez une application [Machine-to-Machine](https://manage.auth0.com/#/applications/applications/create).

   <Warning>
     Vous devez noter le **Client ID** et le **Client Secret** lorsque vous créez les scripts de base de données personnalisés. Vous devez noter le **Client ID** pour activer des scopes dans Management API.
   </Warning>
3. Activez les grants **Password** et **Client Credential** pour cette application (les deux grants doivent être activés pour ce test).

   <Frame>![Auth0 Dashboard > Applications > Advanced Settings](/docs/images/cdy7uua7fh8z/4caAveuhYurCDb5F8mdImm/38496efe6f670d828542b02a56e5ca64/Grant_Types_-_English.png)</Frame>
4. Pour autoriser votre application, accédez à [Applications &gt; APIs](https://manage.auth0.com/#/apis).
5. Sélectionnez **Management API**.
6. Sous l’onglet **Machine-to-Machine Applications**, utilisez la bascule pour autoriser votre application de test.
7. Sélectionnez le menu déroulant pour activer les scopes suivants de l’Auth0 Management API :

   * `read:users`
   * `update:users`
   * `delete:users`
   * `create:users`
   * `read:users_app_metadata`
   * `update:users_app_metadata`
   * `create:users_app_metadata`

     <Frame>![Auth0 Dashboard > APIs > Management API >  M2M Tab > App Permissions](/docs/images/cdy7uua7fh8z/2vW8zD1iIObuqb0i59yw77/c43bafa05b271a4343af51cb5c8ce1ce/2024-03-14_10-28-19.png)</Frame>

<div id="create-test-database-connections">
  ### Créer des connexions de base de données de test
</div>

Après avoir créé un tenant et une application dans Dashboard, créez une connexion de base de données source et une connexion de base de données cible.

1. Accédez à [Authentication &gt; Database](https://manage.auth0.com/#/connections/database) pour créer une nouvelle connexion de base de données qui servira de source.
2. Donnez un nom à votre connexion source de test, activez **Requires Username**, puis sélectionnez **Create**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Vous pouvez créer des utilisateurs pour vos tests en accédant à [User Management &gt; Users](https://manage.auth0.com/#/users). Après avoir sélectionné **Create** **Users** et rempli les champs requis, choisissez la base de données source dans le champ **Connection**.
   </Callout>
3. Créez une deuxième base de données qui servira de cible avec la même configuration qu’à l’étape 2.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Vous pouvez définir la **Password Policy** sur **Non-empty password required** dans les bases de données source et cible afin d’utiliser des mots de passe simples dans vos tests.
   </Callout>
4. Dans votre base de données cible, passez à la vue **Custom Database** et activez **Use my own database**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Si vous voulez faire des tests avec Organizations et vos bases de données personnalisées, activez **Context object in database scripts**.
   </Callout>
5. Passez à la vue **Settings** et activez **Import Users to Auth0**.
6. Passez à la vue **Custom Database**, puis repérez la section **Database settings**. Ajoutez les renseignements suivants à partir de votre base de données source créée à l’étape 1 :

| Clé               | Valeur                                                                 |
| ----------------- | ---------------------------------------------------------------------- |
| `client_id`       | ID client de l’application que vous avez créée.                        |
| `client_secret`   | Secret client de l’application que vous avez créée.                    |
| `auth0_domain`    | Nom de votre tenant dans le domaine Auth0 : `yourTenant.us.auth0.com`. |
| `source_database` | Nom de la connexion source.                                            |

7. Mettez à jour les scripts d’action de base de données [Login](/docs/fr-CA/authenticate/database-connections/custom-db/test-custom-database-connections#login) et [Get User](/docs/fr-CA/authenticate/database-connections/custom-db/test-custom-database-connections#get-user) dans votre base de données cible. Pour en savoir plus sur les pratiques exemplaires relatives à l’utilisation des scripts d’action de base de données, consultez [Pratiques exemplaires pour les connexions de base de données personnalisées et les scripts d’action](/docs/fr-CA/authenticate/database-connections/custom-db/custom-database-connections-scripts).
8. Sélectionnez **Enregistrer et essayer** pour chaque script. Vous devriez surveiller la sortie `console.log` dans Actions Real-time Logs. Pour en savoir plus, consultez [Actions Real-time Logs](/docs/fr-CA/customize/actions/actions-real-time-logs).
9. Sélectionnez **Try Connection** pour tester la connexion en direct.

<div id="test-without-import-users-enabled">
  ## Test sans activer Import Users to Auth0
</div>

1. Répétez les étapes de [Créer un tenant et une application de test](/docs/fr-CA/authenticate/database-connections/custom-db/test-custom-database-connections#create-a-test-tenant-and-application).
2. Créez une connexion de base de données de test.
3. Assurez-vous que **Import Users to Auth0** est désactivé dans les paramètres de votre base de données source. Les utilisateurs s’authentifieront auprès du référentiel externe d’utilisateurs à chaque tentative de connexion.
4. Mettez à jour tous les scripts des actions de base de données à l’aide des exemples ci-dessous.

<div id="get-user-script">
  ### Script Get User
</div>

Le script [Get User](/docs/fr-CA/authenticate/database-connections/custom-db/templates/get-user) implémente une fonction exécutable qui détermine l’état actuel d’un utilisateur.

Lorsque **Import Users to Auth0** est activé, le script Get User s’exécute lorsqu’un utilisateur tente de s’inscrire afin de vérifier si l’utilisateur existe déjà dans le référentiel d’utilisateurs externe.

Le script Get User s’exécute également lorsqu’un utilisateur tente de :

* modifier l’adresse courriel d’un utilisateur (script [Change Email](/docs/fr-CA/authenticate/database-connections/custom-db/templates/change-email))
* se connecter (script [Login](/docs/fr-CA/authenticate/database-connections/custom-db/templates/login))
* modifier le mot de passe d’un utilisateur (script [Change Password](/docs/fr-CA/authenticate/database-connections/custom-db/templates/change-password))

Lorsque **Import Users to Auth0** est désactivé, le script Get User s’exécute lorsqu’un utilisateur tente de s’inscrire afin de vérifier si l’utilisateur existe déjà dans le référentiel d’utilisateurs externe. Si un utilisateur existe déjà dans le référentiel d’utilisateurs externe, le script Create n’est pas exécuté.

Le script Get User s’exécute également lorsqu’un utilisateur tente de :

* créer un utilisateur (script [Create](/docs/fr-CA/authenticate/database-connections/custom-db/templates/create))
* modifier l’adresse courriel d’un utilisateur (script [Change Email](/docs/fr-CA/authenticate/database-connections/custom-db/templates/change-email))
* modifier le mot de passe d’un utilisateur (script [Change Password](/docs/fr-CA/authenticate/database-connections/custom-db/templates/change-password))

<div id="example">
  #### Exemple
</div>

```js lines expandable
async function getUser(user, context, callback) {
    log(`Script started.`);
    log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
    let accessToken = await getAccessToken();
    accessToken = accessToken.access_token;
    if (!accessToken) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);
    log(`The Access Token is available. Searching for user "${user}" in "${configuration.source_database}"`);
    user = user.toLowerCase();
    const searchQuery = encodeURI(`identities.connection:"${configuration.source_database}"+AND+(email:${user} OR username:${user})`);
    var options = {
        method: `GET`,
        url: `https://${configuration.auth0_domain}/api/v2/users?q=${searchQuery}`,
        headers: {
            Authorization: `Bearer ${accessToken}`,
        }
    };
    request(options, function (error, response) {
        if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
        let search_results = JSON.parse(response.body);
        let profile = null;
        if (search_results.length > 0) {
            log(`A user "${user}" is FOUND in "${configuration.source_database}" database.`);
            profile = {
                user_id: search_results[0].user_id.toString(),
                nickname: search_results[0].nickname,
                username: search_results[0].username,
                email: search_results[0].email
            };
        } else {
            log(`A user "${user}" is NOT FOUND in "${configuration.source_database}" database.`);
        }
        log(`Script completed!`);
        return callback(null, profile);
    });
    /* -- OBTENIR UN JETON D'ACCÈS VIA LES IDENTIFIANTS CLIENT -- */
    async function getAccessToken() {
        var options = {
            method: `POST`,
            url: `https://${configuration.auth0_domain}/oauth/token`,
            headers: {
                "Content-Type": `application/x-www-form-urlencoded`,
            },
            form: {
                grant_type: `client_credentials`,
                client_id: configuration.client_id,
                client_secret: configuration.client_secret,
                audience: `https://${configuration.auth0_domain}/api/v2/`
            },
            json: true
        };
        return new Promise(function (resolve) {
            request(options, function (error, response) {
                resolve(error || response.body);
            });
        });
    }
    /* -- JOURNALISATION -- */
    function log(message, error = false) {
        const script_name = `GET USER`;
        const error_label = error ? `(ERROR)` : ``;
        const return_message = `${script_name}: ${error_label} ${message}`;
        console.log(return_message);
        if (error) return callback(new Error(return_message));
    }
}
```

<div id="login-script">
  ### Script Login
</div>

Le script [Login](/docs/fr-CA/authenticate/database-connections/custom-db/templates/login) implémente une fonction exécutable qui authentifie un utilisateur lors de sa connexion. Si l’utilisateur existe dans la base de données cible (Auth0), il l’authentifie à l’aide de cet enregistrement. Sinon, il authentifie l’utilisateur à l’aide de son enregistrement dans la base de données source (externe).

<div id="example">
  #### Exemple
</div>

```javascript lines expandable
function login(usernameOrEmail, password, context, callback) {
  log(`Script started.`);
  const jwt = require('jsonwebtoken');
  const options = {
    method: `POST`,
    url: `https://${configuration.auth0_domain}/oauth/token`,
    headers: { "Content-Type": `application/x-www-form-urlencoded` },
    json: true,
    form: {
      grant_type: `http://auth0.com/oauth/grant-type/password-realm`,
      client_id: configuration.client_id,
      client_secret: configuration.client_secret,
      username: usernameOrEmail,
      password: password,
      realm: `${configuration.source_database}`
    }
  };
  request(options, function (error, response, body) {
    log(`Attempting to authenticate a user "${usernameOrEmail}" against "${configuration.source_database}" database in "${configuration.auth0_domain}" tenant.`);
    if (error) return log(`Cannot connect to "${configuration.auth0_domain}" database.`, true);
    if (response.statusCode !== 200) {
      console.log(`LOGIN: (ERROR) ${response.body.error_description}`);
      return callback(new WrongUsernameOrPasswordError(usernameOrEmail, `LOGIN: (ERROR) ${response.body.error_description}`));
    }
    log(`Successfuly authenticated user "${usernameOrEmail}" against "${configuration.source_database}" database in "${configuration.auth0_domain}" tenant.`);
    const decoded_id_token = jwt.decode(body.id_token);
    const profile = {
      user_id: decoded_id_token.sub,
      nickname: decoded_id_token.nickname,
      username: decoded_id_token.username,
      email: decoded_id_token.email
    };
    log(`Script completed.`);
    return callback(null, profile);
  });
  /* -- JOURNALISATION -- */
  function log(message, error = false) {
    const script_name = `LOGIN`;
    const error_label = error ? `(ERROR)` : ``;
    const return_message = `${script_name}: ${error_label} ${message}`;
    console.log(return_message);
    if (error) return callback(new Error(return_message));
  }
}
```

<div id="create-script">
  ### Script Create
</div>

Le [script Create](/docs/fr-CA/authenticate/database-connections/custom-db/templates/create) implémente une fonction exécutable qui crée l’enregistrement utilisateur correspondant dans la base de données externe lorsqu’un utilisateur s’inscrit par l’intermédiaire de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>, ou qu’il est créé dans l’<Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou au moyen de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<div id="example">
  #### Exemple
</div>

```js lines expandable
async function create(user, context, callback) {
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
  let accessToken = await getAccessToken();
  if (!accessToken.access_token) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);
  accessToken = accessToken.access_token;
  log(`The Access Token is available. Attempting to create a user "${user.email}" in "${configuration.source_database}"`);
  const options = {
    method: `POST`,
    url: `https://${configuration.auth0_domain}/api/v2/users`,
    headers: {
      Authorization: `Bearer ${accessToken}`,
      "Content-Type": `application/x-www-form-urlencoded`
    },
    form: {
      connection: configuration.source_database,
      email: user.email,
      password: user.password,
      username: user.username
    },
    json: true
  };
  request(options, function (error, response) {
    if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
    switch (response.statusCode) {
      case 201:
        log(`The user "${user.email}" is successfuly created in "${configuration.source_database}" database.`);
        return callback(null);
      case 409:
        return callback(new ValidationError(`user_exists`, `The user already exists in "${configuration.source_database}" database.`));
      default:
        return log(`Failed to create a user "${user.email}" in "${configuration.source_database}" database. Error: "${response.statusCode}, ${response.body.message}"`, true);
    }
  });

    var options = {
      method: `POST`,
      url: `https://${configuration.auth0_domain}/oauth/token`,
      headers: {
        "Content-Type": `application/x-www-form-urlencoded`,
      },
      form: {
        grant_type: `client_credentials`,
        client_id: configuration.client_id,
        client_secret: configuration.client_secret,
        audience: `https://${configuration.auth0_domain}/api/v2/`
      },
      json: true
    };
    return new Promise(function (resolve) {
      request(options, function (error, response) {
        resolve(error || response.body);
      });
    });

  function log(message, error = false) {
```

<div id="verify-script">

</div>

Le [script Verify](/docs/fr-CA/authenticate/database-connections/custom-db/templates/verify) implémente une fonction exécutable qui met à jour le statut de vérification de l’adresse courriel d’un utilisateur dans la base de données externe lorsqu’il clique sur le lien contenu dans le courriel de vérification envoyé par Auth0.

  #### Exemple
</div>

```js lines expandable
async function verify(user, context, callback) {
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);

  };
  request(options, function (error, response) {
    if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
```

<div id="change-password-script">
  ### script Change Password
</div>

Le [script Change Password](/docs/fr-CA/authenticate/database-connections/custom-db/templates/change-password) implémente une fonction exécutable qui met à jour le mot de passe d’un utilisateur dans la base de données externe lorsqu’un flux de changement de mot de passe est lancé depuis Auth0 Dashboard ou l’Auth0 Management API.

<div id="example">
  #### Exemple
</div>

```js lines expandable
async function changePassword(user, newPassword, context, callback) {
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
  let accessToken = await getAccessToken();

  log(`The Access Token is available. Searching for user "${user}" in "${configuration.source_database}" database.`);
  user = user.toLowerCase();
  const searchQuery = encodeURI(`identities.connection:"${configuration.source_database}"+AND+(email:${user} OR username:${user})`);
  var options = {
    method: `GET`,
    url: `https://${configuration.auth0_domain}/api/v2/users?q=${searchQuery}`,
    headers: {
      Authorization: `Bearer ${accessToken}`,
    }
  };
  request(options, function (error, response) {
    if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
    let search_results = JSON.parse(response.body);
    if (search_results.length > 0) {
      log(`A user "${user}" is found in "${configuration.source_database}" database.`);
      const user_id = search_results[0].user_id.toString();
      log(`Attempting to change password for user "${user_id}" in "${configuration.source_database}" database.`);
      const options = {
        method: `PATCH`,
        url: `https://${configuration.auth0_domain}/api/v2/users/${user_id}`,
        headers: {
          Authorization: `Bearer ${accessToken}`,
          "Content-Type": `application/x-www-form-urlencoded`
        },
        form: {
          password: newPassword,
        },
        json: true
      };
      request(options, function (error, response) {
        if (error) return log(`Cannot connect to "${configuration.source_database}" database.`, true);
        switch (response.statusCode) {
          case 200:
            log(`The user "${user}" password successfully changed in "${configuration.source_database}" database.`);
            return callback(null, true);
          default:
            return log(`Failed to change password for "${user}"  in "${configuration.source_database}" database. Error: "${response.statusCode}, ${response.body.message}"`, true);
        }
      });
    } else {
      log(`A user "${user}" is not found in "${configuration.source_database}" database. Unable to change password.`, true);
    }
  });
  /* -- OBTENIR UN JETON D'ACCÈS VIA LES IDENTIFIANTS CLIENT -- */
  async function getAccessToken() {
    var options = {
      method: `POST`,
      url: `https://${configuration.auth0_domain}/oauth/token`,
      headers: {
        "Content-Type": `application/x-www-form-urlencoded`,
      },
      form: {
        grant_type: `client_credentials`,
        client_id: configuration.client_id,
        client_secret: configuration.client_secret,
        audience: `https://${configuration.auth0_domain}/api/v2/`
      },
      json: true
    };
    return new Promise(function (resolve) {
      request(options, function (error, response) {
        resolve(error || response.body);
      });
    });
  }
  /* -- JOURNALISATION -- */
  function log(message, error = false) {
    const script_name = `CHANGE PASSWORD`;
    const error_label = error ? `(ERROR)` : ``;
    const return_message = `${script_name}: ${error_label} ${message}`;
    console.log(return_message);
    if (error) return callback(new Error(return_message));
  }
}
```

<div id="change-email-script">
  ### Script Change Email
</div>

Le script [Change Email](/docs/fr-CA/authenticate/database-connections/custom-db/templates/change-email) implémente une fonction exécutable qui met à jour l’adresse courriel d’un utilisateur lorsque celui-ci modifie son adresse courriel ou le statut de vérification de celle-ci.

Ce script n’est pas offert dans l’Auth0 Dashboard. Vous devez appeler le point de terminaison [Update a connection](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) de la Management API et fournir la propriété `options.customScripts.change_email`.

</div>

```js lines expandable
  log(`Script started.`);
  log(`Requesting an Access Token from "${configuration.auth0_domain}".`);
  let accessToken = await getAccessToken();
  if (!accessToken.access_token) return log(`Failed to get an Access Token from "${configuration.auth0_domain}".`, true);
  accessToken = accessToken.access_token;
  log(`The Access Token is available. Searching for user "${user}" in "${configuration.source_database}" database.`);
  user = user.toLowerCase();
  const searchQuery = encodeURI(`identities.connection:"${configuration.source_database}"+AND+(email:${user} OR username:${user})`);
  var options = {
    method: `GET`,
    url: `https://${configuration.auth0_domain}/api/v2/users?q=${searchQuery}`,
    headers: {
      Authorization: `Bearer ${accessToken}`,
    }
```
