Si vous avez votre propre base de données d’utilisateurs, vous pouvez l’utiliser comme <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> dans Auth0 pour authentifier les utilisateurs. Suivez le processus pour créer la connexion de base de données personnalisée, créer les scripts d’action de base de données et ajouter des paramètres de configuration.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Assurez-vous que votre base de données comprend les champs appropriés pour stocker les [attributs du profil utilisateur](/docs/fr-CA/manage-users/user-accounts/user-profiles/user-profile-structure#user-profile-attributes), comme `id`, `nickname`, `email` et `password`. Pour en savoir plus sur le schéma du profil utilisateur d’Auth0, consultez [Profils utilisateur normalisés](/docs/fr-CA/manage-users/user-accounts/user-profiles/normalized-user-profiles). Pour en savoir plus sur la mise à jour des profils utilisateur, consultez [Mettre à jour les profils utilisateur à l’aide de votre base de données](/docs/fr-CA/manage-users/user-accounts/user-profiles/update-user-profiles-using-your-database).
</Callout>

Auth0 vous permet de créer des connexions et des scripts pour la plupart des bases de données couramment utilisées, notamment :

* ASP.NET Identity
* MongoDB
* MySQL
* PostgreSQL
* SQLServer
* Windows Azure SQL Database
* Services Web accessibles au moyen de l’authentification Basic

Vous pouvez vous connecter à tout type de base de données ou de service Web au moyen d’un script personnalisé correctement configuré. Notez que le nettoyage des entrées doit être effectué côté client; Auth0 n’assainit ni ne valide aucune combinaison de nom d’utilisateur et de mot de passe transmise par une base de données personnalisée. Les scripts de base de données personnalisée sont assujettis à la [politique de limitation de débit](/docs/fr-CA/troubleshoot/customer-support/operational-policies/rate-limit-policy) d’Auth0.

<Card title="Pare-feu réseau">
  Si vous êtes derrière un pare-feu, cette fonctionnalité peut exiger que vous ajoutiez les adresses IP Auth0 appropriées à la liste d’autorisation pour fonctionner correctement.
</Card>

<div id="create-the-connection-in-the-auth0-dashboard">
  ## Créer la connexion dans l’Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database), puis sélectionnez **Create DB Connection**.
2. Configurez les paramètres de la connexion, puis sélectionnez **Create** :

| Paramètre                                             | Définition                                                                                                                                                                                         |
| ----------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Name**                                              | Nom de la connexion. Il doit commencer et se terminer par un caractère alphanumérique, contenir uniquement des caractères alphanumériques et des traits d’union, et ne pas dépasser 35 caractères. |
| **Choose one or more attributes as user identifiers** | Choisissez le ou les [attributs d’identifiant](/docs/fr-CA/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers) pour la connexion.                              |
| **Choose one or more authentication methods**         | Choisissez une méthode d’authentification à utiliser pour soumettre les utilisateurs à une vérification.                                                                                           |
| **Use my own database**                               | Activez cette bascule si vous avez une base de données à connecter à Auth0.                                                                                                                        |
| **Disable Sign Ups**                                  | Empêche l’inscription de nouveaux utilisateurs à partir d’endpoints publics.                                                                                                                       |
| **Promote Connection to Domain Level**                | Fait passer cette connexion au niveau du domaine afin qu’elle puisse être utilisée avec des applications tierces.                                                                                  |

Une fois la connexion créée par Auth0, vous aurez accès aux vues suivantes (en plus de la vue **Settings**) :

* Attributes
  * Authentication Methods
  * Custom Database
  * Applications

3. Sélectionnez la vue **Custom Database**, puis activez le commutateur **Use my own database**.

   <Frame>![Activer l’option Use my own database dans Custom Database](/docs/images/cdy7uua7fh8z/3kgHDpBFdVWNq9XOfhsTXI/2efa43e44793b8dd69b9f9e8b54bf752/2025-02-25_10-20-20.png)</Frame>

<div id="create-database-action-scripts">
  ## Créer des scripts d&#39;action de base de données
</div>

L&#39;activation du commutateur **Use my own database** active la section **Database Action Scripts**, dans laquelle vous créerez des scripts pour configurer le fonctionnement de l&#39;authentification lorsque vous utilisez votre base de données. Vous pouvez écrire vos scripts d&#39;action de base de données, ou sélectionner un modèle dans la liste déroulante **Templates** et le modifier au besoin.

Vous **devez** configurer un script `login`; les scripts supplémentaires liés aux fonctionnalités utilisateur sont facultatifs.

Les scripts d&#39;action de base de données disponibles sont :

| Nom                        | Description                                                                                                                       | Paramètres             |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------- | ---------------------- |
| **Login**<br />Obligatoire | S&#39;exécute chaque fois qu&#39;un utilisateur tente d&#39;ouvrir une session.                                                   | `email`, `password`    |
| **Create**                 | S&#39;exécute lorsqu&#39;un utilisateur s&#39;inscrit.                                                                            | `user`                 |
| **Verify**                 | S&#39;exécute après qu&#39;un utilisateur a suivi le lien de vérification.                                                        | `email`                |
| **Change Password**        | S&#39;exécute lorsqu&#39;un utilisateur clique sur le lien de confirmation après une demande de réinitialisation du mot de passe. | `email`, `newPassword` |
| **Get User**               | Récupère un profil utilisateur à partir de votre base de données sans authentifier l&#39;utilisateur.                             | `email`                |
| **Delete**                 | S&#39;exécute lorsqu&#39;un utilisateur est supprimé à l&#39;aide de l&#39;API ou de l&#39;Auth0 Dashboard.                       | `id`                   |

Pour en savoir plus sur les scripts, consultez [Custom Database Action Script Templates](/docs/fr-CA/authenticate/database-connections/custom-db/templates) et [Custom Database Action Script Execution Best Practices](/docs/fr-CA/authenticate/database-connections/custom-db/custom-database-connections-scripts/execution).

<div id="create-a-login-script">
  ### Créer un script Login
</div>

<Card title="Éviter les collisions d’ID utilisateur avec plusieurs bases de données">
  La propriété `id` (ou, sinon, `user_id`) du profil utilisateur renvoyé sera utilisée par Auth0 pour identifier l’utilisateur.

  Si vous utilisez plusieurs connexions de base de données personnalisées, la valeur de `id` doit être unique dans l’ensemble de ces connexions afin d’éviter les collisions d’ID utilisateur. Nous vous recommandons de faire précéder la valeur de `id` par le nom de la connexion (sans espaces). Pour en savoir plus sur les ID utilisateur, consultez [Identifier les utilisateurs](/docs/fr-CA/manage-users/user-accounts/identify-users).
</Card>

Les étapes suivantes utilisent un exemple de script Login pour une base de données MySQL

1. Après avoir activé le commutateur **Use my own database**, la section **Database Action Scripts** est activée. Assurez-vous d’être dans l’onglet **Login**.
2. Utilisez la liste déroulante **Templates** pour sélectionner le modèle de script de base de données MySQL.

   ```javascript lines expandable
   function login(email, password, callback) {
     var bcrypt = require('bcrypt');
     var mysql = require('mysql');
     var connection = mysql.createConnection({
       host: 'localhost',
       user: 'me',
       password: 'secret',
       database: 'mydb'
     });
     connection.connect();
     var query = "SELECT id, nickname, email, password " +
       "FROM users WHERE email = ?";
     connection.query(query, [email], function (err, results) {
       if (err) return callback(err);
       if (results.length === 0) return callback(new WrongUsernameOrPasswordError(email));
       var user = results[0];
       bcrypt.compare(password, user.password, function (err, isValid) {
         if (err) {
           callback(err);
         } else if (!isValid) {
           callback(new WrongUsernameOrPasswordError(email));
         } else {
           callback(null, {
             // Ce préfixe (remplacez-le par le nom de votre propre BD personnalisée)
             // assure l’unicité entre différentes BD personnalisées s’il y a un
             // risque de collision (p. ex. si l’ID utilisateur est une adresse courriel ou un entier)
             id: 'MyConnection1|' + user.id.toString(),
             nickname: user.nickname,
             email: user.email
           });
         }
       });
     });
   }
   ```

   Le script ci-dessus se connecte à une base de données MySQL et exécute une requête pour récupérer le premier utilisateur pour lequel `email == user.email`.
   À l’aide de la méthode `bcrypt.compareSync`, il valide ensuite que les mots de passe correspondent et, en cas de succès, renvoie un objet contenant les renseignement du profil utilisateur, y compris `id`, `nickname` et `email`.
   Ce script suppose que vous avez une table `users` contenant ces colonnes. La valeur `id` renvoyée par le script Login sert à construire l’attribut d’ID utilisateur du profil utilisateur.
3. Sélectionnez **Enregistrer et essayer**.
4. Dans la fenêtre modale, entrez votre courriel et votre mot de passe, puis cliquez sur **Try**. Une liste déroulante vous permet de choisir parmi les runtimes disponibles, le runtime actuel étant affiché en premier.

   <Frame>![Fenêtre modale pour essayer le script Login.](/docs/images/cdy7uua7fh8z/8uBGZhHYZwOcO19ljJNNp/ccd69d84fdbdff90dc779a6b1ea8a6fd/db-connection-try-login-script.png)</Frame>

Les modèles de script ne sont pas utilisés par Auth0 tant que vous n’avez pas sélectionné **Save** ou **Try**. Cela reste vrai même si vous ne modifiez qu’un seul script et n’apportez aucun changement aux autres. Vous devez choisir **Save** au moins une fois pour que tous les scripts soient pris en compte.

<div id="custom-database-support-for-organizations">
  ## Prise en charge des connexions de base de données personnalisées pour les organisations
</div>

Si vous utilisez des connexions de base de données personnalisées avec les organisations, vous pouvez utiliser le paramètre `context` dans la signature du script pour rendre accessibles à vos scripts d’action de base de données personnalisée des détails sur l’organisation, comme `id`, `name` et `metadata`. Les données de l’organisation sont transmises dans l’objet de contexte lorsque l’action associée se produit dans le contexte d’une organisation; par exemple, lorsqu’un utilisateur s’authentifie à l’invite de connexion d’une organisation, le script d’action de connexion est transmis. Pour en savoir plus, consultez [Database Connections](/docs/fr-CA/authenticate/database-connections).

<div id="enable-context-object">
  ### Activer l&#39;objet de contexte
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/dashboard/us/#/connections/database/).
2. Sélectionnez votre connexion de base de données.
3. Sous l&#39;onglet **Custom Database**, repérez **Context objects in database scripts** et sélectionnez **Enable**.

   <Warning>
     Cette fonctionnalité ne peut plus être désactivée une fois activée.
   </Warning>
4. À l&#39;invite de confirmation, sélectionnez **Confirm**.

<div id="use-organization-context">
  ### Utiliser le contexte d’organisation
</div>

Une fois l’objet de contexte `context` activé dans Dashboard, vous pouvez ajouter un objet de contexte `context` à vos scripts d’action de base de données personnalisée, en l’insérant directement avant le paramètre callback. Lorsque des événements sont déclenchés avec le contexte d’organisation, les données correspondantes sont mises à la disposition de vos scripts d’action de base de données personnalisée dans le format ci-dessous :

<div id="add-configuration-parameters">
  ## Ajouter des paramètres de configuration
</div>

Vous pouvez stocker des paramètres, comme les identifiants nécessaires pour vous connecter à votre base de données, dans la section **Paramètres** sous l’éditeur de script. Ils seront accessibles dans tous vos scripts, et vous pourrez accéder à leurs valeurs à l’aide de l’objet `configuration` dans vos scripts d’action de base de données (c.-à-d. `configuration.MYSQL_PASSWORD`).

Utilisez les paramètres ajoutés dans vos scripts pour configurer la connexion. Par exemple, vous pourriez ajouter ce qui suit au script Login MySQL :

```javascript lines
function login (username, password, callback) {
  var mysql = require('mysql');
  var connection = mysql.createConnection({
    host     : configuration.MYSQL_HOST,
    user     : 'me',
    password : configuration.MYSQL_PASSWORD,
    database : 'mydb'
  });
}
```
