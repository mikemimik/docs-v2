Si vous avez votre propre base de données d’utilisateurs, vous pouvez l’utiliser comme <Tooltip tip="Identity Provider (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> dans Auth0 pour authentifier les utilisateurs. Suivez le processus pour créer la connexion de base de données personnalisée, créer des scripts d’action de base de données et ajouter des paramètres de configuration.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Assurez-vous que votre base de données comporte les champs appropriés pour stocker les [attributs de profil utilisateur](/docs/fr-CA/manage-users/user-accounts/user-profiles/user-profile-structure#user-profile-attributes), comme `id`, `nickname`, `email` et `password`. Pour en savoir plus sur le schéma de profil utilisateur d’Auth0, consultez [Profils utilisateur normalisés](/docs/fr-CA/manage-users/user-accounts/user-profiles/normalized-user-profiles). Pour en savoir plus sur la mise à jour des profils utilisateur, consultez [Mettre à jour les profils utilisateur à l’aide de votre base de données](/docs/fr-CA/manage-users/user-accounts/user-profiles/update-user-profiles-using-your-database).
</Callout>

Auth0 vous permet de créer des connexions et des scripts pour la plupart des bases de données couramment utilisées, notamment :

* ASP.NET Identity
* MongoDB
* MySQL
* PostgreSQL
* SQLServer
* Windows Azure SQL Database
* Services Web accessibles par authentification Basic

Vous pouvez vous connecter à tout type de base de données ou de service Web à l’aide d’un script personnalisé correctement configuré. Notez que l’assainissement des entrées doit être effectué du côté du client; Auth0 n’assainit ni ne valide aucune combinaison de nom d’utilisateur et de mot de passe transmise par une base de données personnalisée. Les scripts de base de données personnalisée sont assujettis à la [politique de limitation du débit](/docs/fr-CA/troubleshoot/customer-support/operational-policies/rate-limit-policy) d’Auth0.

  Si vous êtes derrière un pare-feu, cette fonctionnalité peut vous obliger à ajouter les adresses IP Auth0 appropriées à la liste d’autorisation pour fonctionner correctement.
</Card>

<div id="create-the-connection-in-the-auth0-dashboard">
  ## Créer la connexion dans l’Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database), puis sélectionnez **Create DB Connection**.
2. Configurez les paramètres de la connexion, puis sélectionnez **Create** :

| Paramètre                                             | Définition                                                                                                                                                                                    |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Name**                                              | Le nom de la connexion. Il doit commencer et se terminer par un caractère alphanumérique, contenir uniquement des caractères alphanumériques et des tirets, et ne pas dépasser 35 caractères. |
| **Choose one or more attributes as user identifiers** | Choisissez un ou plusieurs [attributs d’identification](/docs/fr-CA/authenticate/database-connections/activate-and-configure-attributes-for-flexible-identifiers) pour la connexion.                |
| **Choose one or more authentication methods**         | Choisissez une méthode d’authentification pour soumettre les utilisateurs à une demande de vérification.                                                                                      |
| **Use my own database**                               | Activez cette bascule si vous avez une base de données que vous souhaitez connecter à Auth0.                                                                                                  |
| **Disable Sign Ups**                                  | Empêche les nouveaux utilisateurs de s’inscrire à partir de endpoints publics.                                                                                                                |
| **Promote Connection to Domain Level**                | Fait passer cette connexion au niveau du domaine afin qu’elle puisse être utilisée avec des applications tierces.                                                                             |

Une fois la connexion créée par Auth0, vous disposerez des vues suivantes (en plus de la vue **Settings**) :

* Attributs
  * Méthodes d’authentification
  * Base de données personnalisée
  * Applications

3. Sélectionnez la vue **Custom Database**, puis activez le commutateur **Use my own database**.

   <Frame>![Activer l’option Use My Own Database de la base de données personnalisée](/docs/images/cdy7uua7fh8z/3kgHDpBFdVWNq9XOfhsTXI/2efa43e44793b8dd69b9f9e8b54bf752/2025-02-25_10-20-20.png)</Frame>

<div id="create-database-action-scripts">
  ## Créer des scripts d’action de base de données
</div>

L’activation du commutateur **Use my own database** active la section **Database Action Scripts**, où vous créerez des scripts pour configurer le fonctionnement de l’authentification lorsque vous utilisez votre base de données. Vous pouvez rédiger vos scripts d’action de base de données ou sélectionner un modèle dans la liste déroulante **Templates** et le modifier au besoin.

Vous **devez** configurer un script `login` ; les scripts supplémentaires liés aux fonctionnalités utilisateur sont facultatifs.

Les scripts d’action de base de données disponibles sont :

| Nom                        | Description                                                                                                               | Paramètres             |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------- | ---------------------- |
| **Login**<br />Obligatoire | S’exécute chaque fois qu’un utilisateur tente de se connecter.                                                            | `email`, `password`    |
| **Create**                 | S’exécute lorsqu’un utilisateur s’inscrit.                                                                                | `user`                 |
| **Verify**                 | S’exécute après qu’un utilisateur a suivi le lien de vérification.                                                        | `email`                |
| **Change Password**        | S’exécute lorsqu’un utilisateur clique sur le lien de confirmation après une demande de réinitialisation du mot de passe. | `email`, `newPassword` |
| **Get User**               | Récupère le profil d’un utilisateur à partir de votre base de données sans authentifier l’utilisateur.                    | `email`                |
| **Delete**                 | S’exécute lorsqu’un utilisateur est supprimé à l’aide de l’API ou de l’Auth0 Dashboard.                                   | `id`                   |

Pour en savoir plus sur les scripts, consultez [Custom Database Action Script Templates](/docs/fr-CA/authenticate/database-connections/custom-db/templates) et [Pratiques exemplaires pour l’exécution des scripts d’action de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db/custom-database-connections-scripts/execution).

<div id="create-a-login-script">
  ### Créer un script Login
</div>

<Card title="Évitez les collisions d’ID utilisateur avec plusieurs bases de données">
  La propriété `id` (ou `user_id`, à défaut) du profil utilisateur renvoyé sera utilisée par Auth0 pour identifier l’utilisateur.

  Si vous utilisez plusieurs connexions de base de données personnalisées, la valeur de `id` doit être unique dans l’ensemble de ces connexions afin d’éviter les collisions d’ID utilisateur. Nous vous recommandons de faire précéder la valeur de `id` du nom de la connexion (en supprimant les espaces). Pour en savoir plus sur les ID utilisateur, consultez [Identifier les utilisateurs](/docs/fr-CA/manage-users/user-accounts/identify-users).
</Card>

Les étapes suivantes utilisent un exemple de script Login pour une base de données MySQL.

1. Après avoir activé le commutateur **Use my own database**, la zone **Database Action Scripts** est activée. Assurez-vous d’être dans l’onglet **Login**.
2. Utilisez la liste déroulante **Templates** pour sélectionner le modèle de script de base de données MySQL.

   ```javascript lines expandable
   function login(email, password, callback) {
     var bcrypt = require('bcrypt');
     var mysql = require('mysql');
     var connection = mysql.createConnection({
       host: 'localhost',
       user: 'me',
       password: 'secret',
       database: 'mydb'
     });
     connection.connect();
     var query = "SELECT id, nickname, email, password " +
       "FROM users WHERE email = ?";
     connection.query(query, [email], function (err, results) {
       if (err) return callback(err);
       if (results.length === 0) return callback(new WrongUsernameOrPasswordError(email));
       var user = results[0];
       bcrypt.compare(password, user.password, function (err, isValid) {
         if (err) {
           callback(err);
         } else if (!isValid) {
           callback(new WrongUsernameOrPasswordError(email));
         } else {
           callback(null, {
             // Ce préfixe (remplacez-le par le nom de votre propre BD personnalisée)
             // garantit l’unicité entre différentes BD personnalisées s’il y a une
             // possibilité de collisions (par ex., si l’ID utilisateur est une adresse courriel ou un entier)
             id: 'MyConnection1|' + user.id.toString(),
             nickname: user.nickname,
             email: user.email
           });
         }
       });
     });
   }
   ```

   Le script ci-dessus se connecte à une base de données MySQL et exécute une requête pour récupérer le premier utilisateur dont `email == user.email`.
   À l’aide de la méthode `bcrypt.compare`, il valide ensuite que les mots de passe correspondent et, en cas de réussite, retourne un objet contenant les renseignements du profil utilisateur, y compris `id`, `nickname` et `email`.
   Ce script suppose que vous avez une table `users` contenant ces colonnes. Le `id` renvoyé par le script Login est utilisé pour construire l’attribut d’ID utilisateur du profil utilisateur.
3. Sélectionnez **Enregistrer et essayer**.
4. Dans la fenêtre modale, entrez votre courriel et votre mot de passe, puis cliquez sur **Try**. Une liste déroulante s’affiche alors, vous permettant de sélectionner un runtime parmi ceux qui sont disponibles, le runtime actuel apparaissant en haut.

   <Frame>![Fenêtre modale pour essayer le script Login.](/docs/images/cdy7uua7fh8z/8uBGZhHYZwOcO19ljJNNp/ccd69d84fdbdff90dc779a6b1ea8a6fd/db-connection-try-login-script.png)</Frame>

Les modèles de script ne sont pas utilisés par Auth0 tant que vous n’avez pas sélectionné **Save** ou **Try**. Cela demeure vrai même si vous ne modifiez qu’un seul script et n’apportez aucun changement aux autres. Vous devez choisir **Save** au moins une fois pour que tous les scripts soient en place.

<div id="custom-database-support-for-organizations">
  ## Prise en charge des connexions de base de données personnalisées pour Organizations
</div>

Si vous utilisez des connexions de base de données personnalisées avec Organizations, vous pouvez utiliser le paramètre `context` dans la signature du script pour mettre à la disposition de vos scripts d’action de base de données personnalisée les détails de l’organisation, comme `id`, `name` et `metadata`. Les données de l’organisation sont transmises dans l’objet context lorsque l’action associée se produit dans le contexte d’une organisation; par exemple, lorsqu’un utilisateur s’authentifie sur l’écran de connexion d’une organisation, le script d’action Login reçoit ces données. Pour en savoir plus, consultez [Database Connections](/docs/fr-CA/authenticate/database-connections).

<div id="enable-context-object">
  ### Activer l’objet de contexte
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/dashboard/us/#/connections/database/).
2. Sélectionnez votre connexion de base de données.
3. Sous l’onglet **Custom Database**, repérez **Context objects in database scripts** et sélectionnez **Enable**.

   <Warning>
     Cette fonctionnalité ne peut pas être désactivée une fois activée.
   </Warning>
4. Dans le prompt de confirmation, sélectionnez **Confirm**.

<div id="use-organization-context">
  ### Utiliser le contexte d’organisation
</div>

Une fois l’objet de contexte activé dans le Dashboard, vous pouvez ajouter un objet `context` à vos scripts d’action de base de données personnalisée, en l’insérant directement avant le paramètre `callback`. Lorsque des événements sont déclenchés avec le contexte d’organisation, les données correspondantes sont mises à la disposition de vos scripts d’action de base de données personnalisée dans le format ci-dessous :

<div id="add-configuration-parameters">
  ## Ajouter des paramètres de configuration
</div>

Vous pouvez stocker des paramètres, comme les identifiants requis pour vous connecter à votre base de données, dans la section **Settings** sous l’éditeur de scripts. Ils seront accessibles dans tous vos scripts, et vous pourrez accéder aux valeurs des paramètres à l’aide de l’objet `configuration` dans vos scripts d’action de base de données (c.-à-d. `configuration.MYSQL_PASSWORD`).

Utilisez les paramètres ajoutés dans vos scripts pour configurer la connexion. Par exemple, vous pourriez ajouter ce qui suit au script MySQL Login :

```javascript lines
function login (username, password, callback) {
  var mysql = require('mysql');
  var connection = mysql.createConnection({
    host     : configuration.MYSQL_HOST,
    user     : 'me',
    password : configuration.MYSQL_PASSWORD,
    database : 'mydb'
  });
}
```
