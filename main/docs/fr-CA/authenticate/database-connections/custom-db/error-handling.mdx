---
title: Dépannage des bases de données personnalisées
description: Comment gérer les erreurs et faire du dépannage lorsque vous utilisez votre propre base de données comme fournisseur d’identité.
---

Vous pourriez recevoir des erreurs de vos bases de données personnalisées. Vous pouvez aussi renvoyer des erreurs à partir des scripts d’action de votre connexion de base de données personnalisée à des fins de dépannage.

<div id="custom-database-error-types">
  ## Types d’erreurs des bases de données personnalisées
</div>

Les connexions de base de données peuvent renvoyer les erreurs suivantes :

| Erreur                                                                              | Script Login      | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ----------------------------------------------------------------------------------- | ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `new WrongUsernameOrPasswordError(`<br />`  <email or user_id>, <message>`<br />`)` | Login             | Se produit lorsque les identifiants de l’utilisateur sont invalides. Si vous ne transmettez aucun argument, ou si le premier argument est évalué comme faux, l’erreur sera consignée comme un événement `fu` (nom d’utilisateur/courriel invalide) avec une chaîne vide pour `user_id`. Si vous transmettez un premier argument évalué comme vrai, l’erreur sera consignée comme un événement `fp` (l’utilisateur existe, mais le mot de passe est invalide) avec une valeur `user_id` de `auth0\|<first_argument>`. Pour en savoir plus, consultez [Log Event Type Codes](/docs/fr-CA/deploy-monitor/logs/log-event-type-codes). |
| `new ValidationError(`<br />`  <user_exists>, <message>`<br />`)`                   | Create            | Se produit lorsqu’un utilisateur existe déjà dans votre base de données.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `callback(<null>,<false>)`                                                          | Change Password   | Se produit lorsque le mot de passe de l’utilisateur n’a pas été mis à jour.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `callback(<null>)`                                                                  | Get User          | Se produit lorsque l’utilisateur est introuvable.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| `new UnauthorizedError(<message>)`                                                  | All Login Scripts | Se produit lorsqu’un problème survient lors de la tentative d’accès à la base de données. Il s’agit de la méthode privilégiée pour transmettre les erreurs des bases de données personnalisées à l’interface utilisateur.                                                                                                                                                                                                                                                                                                                                                                                                   |
| `new Error(<message>)`                                                              | All Login Scripts | Se produit lorsqu’un problème survient lors de la tentative d’accès à la base de données.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `error code: auth0_idp_error;` `user already exists`                                | Create            | Se produit dans un répertoire d’identités existant lorsque vous tentez de créer un utilisateur qui existe déjà dans votre répertoire d’identités existant ou dans Auth0.                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `error code: auth0_idp_error;` `user already exists`                                | Delete            | Se produit dans un répertoire d’identités existant lorsque vous tentez de supprimer un utilisateur à l’aide de scripts de base de données personnalisée. L’erreur peut aussi se présenter sous la forme « Error! There was a problem deleting `user@example.com` » ou « Sandbox Error: Please implement the Delete script for this database connection at `https://manage.auth0.com/#/connections/database` ».                                                                                                                                                                                                              |

<div id="return-errors">
  ## Renvoyer des erreurs
</div>

Pour renvoyer une erreur, appelez la fonction `callback()` et passez `error` comme premier paramètre :

```js lines
callback(error);
```

Par exemple :

```js wrap lines
callback(new ValidationError('email-too-long', 'Email is too long.'));
```

<div id="return-errors-when-using-lock">
  ### Renvoyer les erreurs lors de l’utilisation de Lock
</div>

Si vous utilisez Lock, vous pouvez [personnaliser les messages d’erreur](/docs/fr-CA/customize/login-pages/classic-login/customize-lock-error-messages) en les ajoutant au dictionnaire.

<div id="troubleshoot-errors">
  ## Résoudre les erreurs
</div>

Testez le script à l’aide du bouton **Essayer**.

Si vous n’obtenez pas le résultat attendu ou si une erreur s’affiche, consultez les [journaux en temps réel d’Actions](/docs/fr-CA/customize/actions/actions-real-time-logs), ajoutez des instructions `console.log()` à votre script, puis réessayez la connexion. La sortie de `console.log()` s’affiche dans la fenêtre des journaux en temps réel d’Actions.

Vous pourriez rencontrer des erreurs lorsque vous utilisez des scripts d’action pour importer, créer ou supprimer des utilisateurs. Utilisez `console.logs` pour vérifier que l’erreur est bien liée à l’utilisation de scripts d’action. Pour en savoir plus, consultez [Modèles de scripts d’action de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db/templates).
