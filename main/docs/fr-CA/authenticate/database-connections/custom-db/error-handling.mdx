---
title: Dépannage des bases de données personnalisées
description: Comment gérer les erreurs et faire du dépannage lorsque vous utilisez votre propre base de données comme fournisseur d’identité.
---

Vous pourriez recevoir des erreurs provenant de bases de données personnalisées. Vous pouvez également renvoyer des erreurs à partir des scripts d’action de votre connexion de base de données personnalisée à des fins de dépannage.

<div id="custom-database-error-types">
  ## Types d’erreurs de base de données personnalisée
</div>

Les connexions de base de données peuvent renvoyer les erreurs suivantes :

| Erreur                                                                              | Login Script           | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------------------------------------------------------------------------------- | ---------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `new WrongUsernameOrPasswordError(`<br />`  <email or user_id>, <message>`<br />`)` | Login                  | Se produit lorsque les identifiants de l’utilisateur sont invalides. Si aucun argument n’est transmis, ou si le premier argument est une valeur falsy, l’erreur sera consignée comme un événement `fu` (username/email invalide) avec une chaîne vide comme `user_id`. Si le premier argument est une valeur truthy, l’erreur sera consignée comme un événement `fp` (l’utilisateur existe, mais le Password est invalide) avec une valeur `user_id` de `auth0\|<first_argument>`. Pour en savoir plus, consultez [Codes de type d’événement de journal](/docs/fr-CA/deploy-monitor/logs/log-event-type-codes). |
| `new ValidationError(`<br />`  <user_exists>, <message>`<br />`)`                   | Create                 | Se produit lorsqu’un utilisateur existe déjà dans votre base de données.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `callback(<null>,<false>)`                                                          | Change Password        | Se produit lorsque le Password de l’utilisateur n’a pas été mis à jour.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `callback(<null>)`                                                                  | Get User               | Se produit lorsque l’utilisateur est introuvable.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `new UnauthorizedError(<message>)`                                                  | Tous les scripts Login | Se produit lorsqu’un problème survient pendant une tentative d’accès à la base de données. C’est la façon privilégiée de transmettre les erreurs des bases de données personnalisées au front-end.                                                                                                                                                                                                                                                                                                                                                                                                        |
| `new Error(<message>)`                                                              | Tous les scripts Login | Se produit lorsqu’un problème survient pendant une tentative d’accès à la base de données.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| `error code: auth0_idp_error;` `user already exists`                                | Create                 | Se produit dans un magasin d’identités hérité lorsque vous tentez de créer un utilisateur qui existe déjà dans votre magasin d’identités hérité ou dans Auth0.                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| `error code: auth0_idp_error;` `user already exists`                                | Delete                 | Se produit dans un magasin d’identités hérité lorsque vous tentez de supprimer un utilisateur à l’aide de scripts de base de données personnalisée. L’erreur peut aussi s’afficher ainsi : &quot;Error! There was a problem deleting `user@example.com`&quot; ou &quot;Sandbox Error: Please implement the Delete script for this database connection at `https://manage.auth0.com/#/connections/database`&quot;.                                                                                                                                                                                         |

<div id="return-errors">
  ## Retourner des erreurs
</div>

Pour retourner une erreur, appelez la fonction `callback()` et passez `error` en premier paramètre :

```js lines
callback(error);
```

Par exemple :

```js wrap lines
callback(new ValidationError('email-too-long', 'Email is too long.'));
```

<div id="return-errors-when-using-lock">
  ### Retourner des erreurs lors de l’utilisation de Lock
</div>

Si vous utilisez Lock, vous pouvez [personnaliser les messages d’erreur](/docs/fr-CA/customize/login-pages/classic-login/customize-lock-error-messages) en les ajoutant au dictionnaire.

<div id="troubleshoot-errors">
  ## Résoudre les erreurs
</div>

Testez le script à l’aide du bouton **Try**.

Si vous n’obtenez pas le résultat attendu ou si vous recevez une erreur, utilisez [Actions Real-time Logs](/docs/fr-CA/customize/actions/actions-real-time-logs), ajoutez des instructions `console.log()` à votre script, puis réessayez la connexion. La sortie de `console.log()` s’affiche dans la fenêtre Actions Real-time Logs.

Vous pourriez rencontrer des erreurs lorsque vous utilisez des scripts d&#39;action pour importer, créer ou supprimer des utilisateurs. Utilisez `console.logs` pour vérifier que l’erreur découle de l’utilisation de scripts d&#39;action. Pour en savoir plus, consultez [Custom Database Action Script Templates](/docs/fr-CA/authenticate/database-connections/custom-db/templates).
