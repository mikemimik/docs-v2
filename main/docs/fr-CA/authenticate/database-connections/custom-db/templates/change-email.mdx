---
description: Décrit le script d’action de base de données personnalisée permettant de modifier l’adresse courriel d’un utilisateur.
title: Modèle de script Change Email
---

Le script Change Email implémente la fonction définie lorsque l’adresse courriel d’un utilisateur ou le statut de vérification de son adresse courriel change. Nous recommandons de nommer cette fonction `changeEmail`.

Ce script n’est utilisé que dans un [scénario d’authentification héritée](/docs/fr-CA/authenticate/database-connections/custom-db/overview-custom-db-connections), et il est requis si vous voulez mettre à jour l’adresse courriel d’un utilisateur (et/ou le statut de vérification de son adresse courriel) dans Auth0 et dans votre base de données externe au cours de la même opération.

Le script Change Email ne peut pas être configuré dans le <Tooltip tip="Auth0 Dashboard : produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Pour gérer ce script, vous devez utiliser le endpoint Auth0 <Tooltip tip="Management API : produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> [Create a connection](https://auth0.com/docs/api/management/v2#!/Connections/post_connections) ou [Update a connection](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id), ou encore l’[Auth0 Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool).

<Warning>
  Si vous gérez les Database Action Scripts d’une connexion à base de données personnalisée dans l’Auth0 Dashboard, le script Change Email sera automatiquement supprimé lorsque vous enregistrerez.

  Pour éviter que le script Change Email soit supprimé par accident, utilisez l’Auth0 Management API ou l’Auth0 Deploy CLI pour gérer la connexion.

  Si vous modifiez manuellement l’adresse courriel d’un utilisateur dans votre base de données personnalisée, vous devez mettre à jour l’utilisateur séparément; Auth0 ne détectera pas automatiquement le changement.
</Warning>

<div id="changeemail-function">
  ## Fonction changeEmail
</div>

La fonction changeEmail doit :

* Mettre à jour l’adresse courriel de l’utilisateur dans la base de données externe.
* Renvoyer une erreur si l’opération échoue ou si une erreur survient.

<div id="definition">
  ### Définition
</div>

La fonction `changeEmail` accepte quatre paramètres et renvoie une fonction `callback` :

```js lines
changeEmail(email, newEmail, verified, callback): function
```

| Paramètre  | Type     | Description                                                                                        |
| ---------- | -------- | -------------------------------------------------------------------------------------------------- |
| `email`    | String   | Adresse courriel actuelle de l’utilisateur.                                                        |
| `newEmail` | String   | Valeur à définir comme nouvelle adresse courriel de l’utilisateur dans la base de données externe. |
| `verified` | Boolean  | État de vérification de la nouvelle adresse courriel.                                              |
| `callback` | Function | Sert à transmettre les données d’erreur dans le pipeline.                                          |

<div id="example">
  ### Exemple
</div>

Voici un exemple en pseudo-Javascript montrant comment vous pourriez implémenter la fonction `changeEmail` :

```javascript lines expandable
function (email, newEmail, verified, callback) {
  // Préparer l'appel d'API
  let options = {
    url: "https://example.com/api/users",
    action: "PATCH",
    body: {
      email: email,
      new_email: newEmail,
      email_verified: verified
    }
  };

  // Appeler l'API
  send(options, err => {
    // Retourner la valeur `false` dans le callback si l'opération a échoué
    if (err && err.id == "FAIL_CHANGE_EMAIL") {
      return callback(null, false);
    } else if (err) {
      // Retourner une erreur dans le callback si une erreur non spécifiée s'est produite
      return callback(new Error("My custom error message."));
    }

    // Retourner la valeur `true` dans le callback si l'opération a réussi
    return callback(null, true);
  });
}
```

<div id="callback-function">
  ## Fonction callback
</div>

La fonction `callback` accepte deux paramètres et renvoie une fonction.

<div id="definition">
  ### Définition
</div>

```js lines
callback(error, operationResult): function
```

| Paramètre         | Type    | Obligatoire | Description                                                       |
| ----------------- | ------- | ----------- | ----------------------------------------------------------------- |
| `error`           | object  | Obligatoire | Contient les données d’erreur.                                    |
| `operationResult` | Boolean | Facultatif  | Indique le résultat de l’opération de changement d’adresse courriel. |

<div id="return-a-success">
  ### Retourner un succès
</div>

Si l’opération de changement d’adresse courriel a réussi, appelez la fonction `callback` et transmettez une valeur `null` comme paramètre `error` ainsi qu’une valeur `true` comme paramètre `operationResult` :

```js lines
return callback(null, true);
```

<div id="return-a-failure">
  ### Signaler un échec
</div>

Si l’opération de changement d’adresse courriel a échoué, appelez la fonction `callback` en transmettant la valeur `null` pour le paramètre `error` et la valeur `false` pour le paramètre `operationResult` :

```js lines
return callback(null, false);
```

<div id="return-an-error">
  ### Retourner une erreur
</div>

Si une erreur s’est produite, appelez la fonction `callback` et transmettez les renseignements pertinents sur l’erreur dans le paramètre `error` :

```js lines
return callback(new Error("My custom error message."));
```
