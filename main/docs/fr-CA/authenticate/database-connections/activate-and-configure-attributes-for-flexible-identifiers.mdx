<Warning>
  L’activation des identifiants flexibles dans votre tenant peut entraîner des changements incompatibles dans votre environnement de production. Pour en savoir plus sur les problèmes potentiels et les limites, consultez [Identifiants flexibles et attributs](/docs/fr-CA/authenticate/database-connections/flexible-identifiers-and-attributes).
</Warning>

Un attribut de connexion de base de données, comme le courriel, le numéro de téléphone ou le nom d’utilisateur, fait partie des identifiants flexibles. Les utilisateurs finaux s’inscrivent, s’authentifient et réinitialisent leur mot de passe à l’aide d’un attribut. Chaque attribut a une configuration distincte.

<div id="activate-attributes">
  ## Activer les attributs
</div>

<Tabs>
  <Tab title="Nouvelle connexion" icon="database">
    1. Accédez à [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Sélectionnez **+Créer une connexion DB**.
    3. Entrez un nom pour la connexion et choisissez les attributs.
    4. Choisissez la méthode d’authentification pour la connexion.
  </Tab>

  <Tab title="Connexion existante" icon="database">
    1. Accédez à [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Dans la liste, sélectionnez la connexion de base de données que vous souhaitez mettre à jour.
    3. Sélectionnez l’onglet **Attributs**, puis choisissez **Activer**.
    4. Lorsqu’on vous demande de confirmer que vous avez testé votre configuration dans un environnement de développement, sélectionnez **Continuer**.
    5. Dans l’écran **Attributs**, sélectionnez **Ajouter un attribut** ou **Configurer** un attribut existant.
  </Tab>
</Tabs>

Des valeurs par défaut sont attribuées à une nouvelle connexion et peuvent être modifiées une fois la connexion créée.

| Paramètre                                                      | Valeur           |
| -------------------------------------------------------------- | ---------------- |
| Utiliser `\{attribute}` comme identifiant                      | `True`           |
| Autoriser l’inscription avec `\{attribute}`                    | Obligatoire      |
| Exiger `\{attribute}` dans le profil utilisateur               | `True`           |
| Longueur du nom d’utilisateur                                  | Min : 1 Max : 15 |
| Autoriser les noms d’utilisateur au format courriel            | `False`          |
| Autoriser les noms d’utilisateur au format numéro de téléphone | `False`          |

<div id="configure-attributes">
  ## Configurer les attributs
</div>

Sélectionnez l’onglet **Attribut** dans la connexion. Choisissez l’un des Attributs disponibles, puis sélectionnez **Configurer**. Une fois la configuration des attributs mise à jour, sélectionnez **Enregistrer**.

<div id="configure-or-remove-existing-attributes-on-a-connection">
  ### Configurer ou supprimer des attributs existants sur une connexion
</div>

Tous les attributs peuvent être modifiés ou supprimés dans votre <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Les utilisateurs ne peuvent pas se connecter, s’inscrire ni réinitialiser leur mot de passe à l’aide de l’attribut supprimé, et au moins un attribut doit demeurer actif sur chaque connexion.

Pour modifier ou supprimer un attribut, accédez à l’onglet **Attribut**, puis choisissez **Supprimer**.

<div id="revert-a-connection">
  ### Rétablir une connexion
</div>

Pour rétablir une connexion dans son état d’origine, utilisez le point de terminaison <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> [Mettre à jour une connexion](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) avec un objet `options` qui n’inclut pas le paramètre `attributes`, comme indiqué ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Si vous utilisez le paramètre `options`, l’objet `options` au complet est remplacé. Pour éviter les données partielles ou d’autres problèmes, assurez-vous que tous les paramètres actuels sont présents en copiant le contenu de l’objet `options` de la connexion et en l’incluant dans l’appel PATCH. Le seul paramètre à supprimer est `attributes`.
  * Si `username` est activé comme attribut dans votre tenant, l’exécution de l’action suivante vous obligera à modifier vos paramètres `username`. Vous devrez reconfigurer vos paramètres de nom d’utilisateur si vous choisissez cette option.
</Callout>

{
"options": {
       }
}
