<Warning>
  L’activation des identifiants flexibles dans votre tenant peut entraîner des changements incompatibles dans votre environnement de production. Pour en savoir plus sur les problèmes potentiels et les limitations, consultez [Identifiants flexibles et attributs](/docs/fr-CA/authenticate/database-connections/flexible-identifiers-and-attributes).
</Warning>

Un attribut d’une connexion de base de données, comme l’e-mail, le numéro de téléphone ou le nom d’utilisateur, fait partie des identifiants flexibles. Les utilisateurs finaux s’inscrivent, s’authentifient et réinitialisent leur mot de passe à l’aide d’un attribut. Chaque attribut a sa propre configuration.

<div id="activate-attributes">
  ## Activer les attributs
</div>

<Tabs>
  <Tab title="Nouvelle connexion" icon="database">
    1. Accédez à [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Sélectionnez **+Create DB Connection**.
    3. Entrez un nom pour la connexion et choisissez les attributs.
    4. Choisissez la méthode d’authentification de la connexion.
  </Tab>

  <Tab title="Connexion existante" icon="database">
    1. Accédez à [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database).
    2. Sélectionnez dans la liste la connexion de base de données que vous souhaitez mettre à jour.
    3. Sélectionnez l’onglet **Attributs** et choisissez **Activer**.
    4. Lorsque vous êtes invité à confirmer que vous avez testé votre configuration dans un environnement de développement, sélectionnez **Poursuivre**.
    5. À l’écran Attributs, sélectionnez **Ajouter un attribut** ou **Configurer** un attribut existant.
  </Tab>
</Tabs>

Des valeurs par défaut sont attribuées à une nouvelle connexion et peuvent être modifiées après sa création.

| Paramètre                                                      | Valeur           |
| -------------------------------------------------------------- | ---------------- |
| Utiliser `\{attribute}` comme identifiant                      | `True`           |
| Autoriser l’inscription avec `\{attribute}`                    | Obligatoire      |
| Exiger `\{attribute}` dans le profil utilisateur               | `True`           |
| Longueur du nom d’utilisateur                                  | Min : 1 Max : 15 |
| Autoriser les noms d’utilisateur au format courriel            | `False`          |
| Autoriser les noms d’utilisateur au format numéro de téléphone | `False`          |

<div id="configure-attributes">
  ## Configurer les attributs
</div>

Sélectionnez l’onglet **Attribut** de la connexion. Choisissez l’un des attributs disponibles, puis sélectionnez **Configurer**. Une fois la configuration des attributs mise à jour, sélectionnez **Enregistrer**.

<div id="configure-or-remove-existing-attributes-on-a-connection">
  ### Configurer ou supprimer les attributs existants d’une connexion
</div>

Tous les attributs peuvent être modifiés ou supprimés dans votre <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Les utilisateurs ne peuvent pas se connecter, s’inscrire ni réinitialiser leur mot de passe à l’aide de l’attribut supprimé, et au moins un attribut doit rester actif pour chaque connexion.

Pour modifier ou supprimer un attribut, accédez à l’onglet **Attributs**, puis choisissez **Supprimer**.

<div id="revert-a-connection">
  ### Rétablir une connexion
</div>

Pour rétablir une connexion dans son état d’origine, utilisez le endpoint [Update a connection](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) du <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> avec un objet `options` qui n’inclut pas le paramètre `attributes`, comme indiqué ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Si vous utilisez le paramètre `options`, l’objet `options` est entièrement remplacé. Pour éviter les données partielles ou d’autres problèmes, assurez-vous que tous les paramètres actuels sont inclus en recopiant le contenu de l’objet `options` de la connexion et en l’ajoutant à l’appel PATCH. Le seul paramètre à retirer est `attributes`.
  * Si l’attribut `username` est activé dans votre tenant, l’exécution de l’action suivante vous oblige à modifier vos paramètres `username`. Vous devrez reconfigurer vos paramètres de nom d’utilisateur si vous choisissez cette approche.
</Callout>

{
"options": {
       }
}
