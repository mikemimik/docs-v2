title: Surveiller les événements liés aux clés d’accès dans les journaux du locataire
---

Vous pouvez surveiller les flux de clé d’accès associés à vos connexions de base de données dans les [journaux du locataire](/docs/fr-CA/deploy-monitor/logs).

Vous pouvez consulter les événements de journal dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ([Monitoring &gt; Logs](https://manage.auth0.com/#/logs)) ou récupérer les journaux au moyen de la [Management API](https://auth0.com/docs/api/management/v2/logs/get-logs).

| Code d’événement                | Scénario                                                                                                        | Notes                                                                                                                                                                                                                                                                                                             |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ss`                            | Lors de l’inscription, un utilisateur a enregistré une clé d’accès comme méthode d’authentification principale. | Cet événement de journal contient le champ de type array `details.authentication_methods` avec la valeur `passkey`.<br /><br />Si un utilisateur s’est inscrit avec un mot de passe, ce champ de type array contient la valeur `pwd`.                                                                             |
| `s`                             | Lors de la connexion, un utilisateur s’est authentifié avec une clé d’accès.                                    | Cet événement de journal contient l’array `details.prompts`, qui répertorie l’ensemble des prompts qu’un utilisateur a parcourus pendant la connexion.<br /><br />Si un utilisateur s’est authentifié à l’aide d’une clé d’accès, l’array contient un élément avec le champ suivant : `"performed_amr": ["phr"]`. |
| `gd_enrollment_complete`        | Un flux d’inscription progressive ou locale a été mené à bien.                                                  | Cet événement de journal contient le champ `details.authenticator.type` avec la valeur `passkey` si un utilisateur a inscrit une clé d’accès.                                                                                                                                                                     |
| `gd_webauthn_enrollment_failed` | Une erreur s’est produite durant un flux d’inscription progressive ou locale.                                   | Cet événement de journal contient le champ array `details.authentication_methods` avec la valeur `passkey` si un utilisateur a tenté d’inscrire une clé d’accès.                                                                                                                                                  |
