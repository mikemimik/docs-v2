---
title: Connection Profile
description: Utilisez un Connection Profile pour préciser comment les paramètres privés d’une connexion Auth0 doivent être configurés lorsqu’elle est créée par des tiers.
---

Auth0 propose des connexions Enterprise pour authentifier les utilisateurs auprès d’un fournisseur d’identité fédéré externe (IdP), comme Okta, Microsoft Entra ID, Google Workspace, entre autres. Une connexion configurée comprend des paramètres propres au protocole pour intégrer l’authentification unique (SSO), le provisionnement des utilisateurs et les intégrations de logout avec l’IdP externe, ainsi que des paramètres privés qui régissent la manière dont la connexion interagit avec les fonctionnalités Universal Login et Organizations d’Auth0.

Le Connection Profile (CP) permet aux développeurs Auth0 de préciser comment les paramètres privés d’une connexion Auth0 doivent être configurés lorsqu’elle est créée par des tiers

<div id="how-it-works">
  ## Fonctionnement
</div>

* **Définition du profil** : Un administrateur crée un Connection Profile qui définit les valeurs des propriétés à inscrire dans la connexion chaque fois qu’elle est créée à l’aide d’une des fonctionnalités de délégation d’administration d’Auth0.

* **Portée flexible** : Les profils sont aujourd’hui liés aux flux Self-Service Enterprise Configuration et Okta Express Configuration, mais ils sont conçus pour être réutilisés plus largement, notamment pour le provisionnement, l’onboarding, la gestion des droits et les futures fonctionnalités d’Auth0.

<div id="connection-profile-properties">
  ## Propriétés de Connection Profile
</div>

Un Connection Profile est un objet JSON qui prend en charge les propriétés configurables suivantes, appliquées à toutes les connexions nouvellement créées.

| **Propriété**                             | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `connection_name_prefix_template`         | Représente le préfixe à utiliser pour nommer les connexions. Correspond au paramètre `name` d’une connexion. Cette valeur prend en charge la substitution de variables pour l’Organization ID et le nom de l’organisation. Le workflow OIN doit remplacer ces valeurs pour générer le préfixe final. Les variables sont placées entre accolades ({}). <p /> Variables prises en charge : <p /> <ul><li>`org_id` : l’Organization ID</li><li>`org_name` : le [nom de l’organisation](/docs/fr-CA/manage-users/organizations/configure-organizations/create-organizations#auth0-dashboard)</li></ul> <p /> Exemple : `con-{org_id}-` |
| `enabled_features`                        | Cette liste indique les fonctionnalités prises en charge par la connexion configurée. Les fonctionnalités qui ne figurent pas dans la liste ne sont pas autorisées.<p /> Valeurs prises en charge : <ul><li>`scim` : lorsqu’elle est présente, SCIM peut être configuré sur la connexion.</li><li>`universal_logout` : lorsqu’elle est présente, la fonctionnalité Universal Logout peut être utilisée avec cette connexion.</li></ul>                                                                                                                                                                                      |
| `organization.assign_membership_on_login` | Indique si les utilisateurs doivent automatiquement se voir attribuer une Membership dans l’organisation au moment du login. Correspond à la propriété `assign_membership_on_login` de la sous-ressource `enabled_connections` de l’organisation.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Pour les intégrations Express Configuration avec Okta, `optional` est traité comme `required`</Callout> <p />                                                                                                                                                                                                |

| Valeur de Connection Profile  | Valeur de `assign_membership_on_login` dans Enable Connections                                                                                                                                      |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `none`                        | `false`                                                                                                                                                                                             |
| `optional`                    | `false`                                                                                                                                                                                             |
| `required`                    | `true`                                                                                                                                                                                              |
| `organization`                | Un objet contenant les paramètres appliqués à l’organisation sous laquelle la connexion est créée.                                                                                                 |
| `organization.show_as_button` | Indique si la connexion doit être affichée sous forme de bouton sur l’écran de login une fois qu’une organisation a été choisie. Correspond à la valeur `show_as_button` de la connexion activée. |

| Valeur de Connection Profile | Valeur `show_as_button` de la connexion |
| ---------------------------- | ---------------------------------------- |
| `none`                       | `false`                                  |
| `optional`                   | `true`                                   |
| `required`                   | `true`                                   |

<div id="example-connection-profile">
  ## Exemple de Connection Profile
</div>

```json
{
  "organization": {
    "show_as_button": "none",
    "assign_membership_on_login": "none"
  },
  "connection_name_prefix_template": "ec-{org_id}-",
  "enabled_features": [
    "scim",
    "universal_logout"
  ]
}
```

<div id="create-and-manage-connection-profiles">
  ## Créer et gérer les Connection Profiles
</div>

Un Connection Profile est généré automatiquement lorsque vous utilisez la fonctionnalité [Express Configuration](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/okta/express-configuration) avec Okta. Vous pouvez également créer et gérer des Connection Profiles à l’aide de l’Auth0 Management API.

<div id="configure-with-management-api">
  ### Configurer avec la Management API
</div>

Pour utiliser la Management API, vous devez obtenir un [jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens). Pour gérer les Connection Profiles, les points de terminaison suivants de la Management API sont disponibles :

* `POST` `/api/v2/connection-profiles`
* `GET` `/api/v2/connection-profiles`
* `PATCH` `/api/v2/connection-profiles/{id}`
* `GET` `/api/v2/connection-profiles/{id}`
* `GET` `/api/v2/connection-profiles/templates`
