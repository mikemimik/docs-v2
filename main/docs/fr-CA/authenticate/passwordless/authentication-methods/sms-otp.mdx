Vous pouvez configurer l’authentification <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> pour envoyer un mot de passe à usage unique (OTP) à un utilisateur par SMS. Pour en savoir plus, consultez [Configurer l’email ou le SMS pour l’authentification Passwordless](/docs/fr-CA/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

Auth0 prend en charge [Twilio](https://www.twilio.com) ainsi qu’un fournisseur téléphonique personnalisé pour envoyer des OTP. Si vous souhaitez utiliser un fournisseur téléphonique personnalisé avec Auth0 Actions, consultez [Configurer un fournisseur téléphonique personnalisé](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) ou [Configurer un fournisseur téléphonique personnalisé avec Terraform](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/configure-a-custom-phone-provider-with-terraform).

<div id="how-it-works">
  ## Comment ça fonctionne
</div>

Lorsqu’un nouvel utilisateur reçoit un code et le saisit pour la première fois dans votre application, son profil utilisateur est créé dans la connection `sms` avant qu’Auth0 l’authentifie.

Si le numéro de téléphone auquel l’OTP a été envoyé correspond à celui d’un utilisateur existant, Auth0 authentifie cet utilisateur :

<div id="configure-the-phone-provider">
  ## Configurer le fournisseur téléphonique
</div>

1. Dans l’Auth0 Dashboard, accédez à [Branding &gt; Phone Provider](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Activez la bascule Phone Message Provider.

   <Frame>![Auth0 Dashboard > Branding > Phone Provider > Activer Phone Message Provider](/docs/images/cdy7uua7fh8z/1k93jvdxDq2FZLiRdvr5aR/d917431cd0265943c5118174a8673118/2025-02-24_16-43-14.png)</Frame>
3. Sélectionnez votre fournisseur de messages téléphoniques.

   1. Pour utiliser Twilio, suivez [Configurer Twilio comme fournisseur de messagerie téléphonique](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-twilio-as-a-phone-messaging-provider).
   2. Pour utiliser un fournisseur téléphonique personnalisé, suivez [Configurer un fournisseur téléphonique personnalisé](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) afin d’utiliser Auth0 Actions.
4. Sélectionnez votre mode d’envoi.
5. **Enregistrez** vos modifications.

<div id="legacy-sms-gateway">
  ## Ancienne passerelle SMS
</div>

Si le fournisseur téléphonique personnalisé de l’Auth0 Action ne répond pas à votre cas d’usage, vous pouvez gérer la passerelle SMS personnalisée dans les [connexions SMS Passwordless](/docs/fr-CA/authenticate/passwordless) avec <Tooltip tip="Management API : Un produit qui permet aux clients d'effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Pour en savoir plus, consultez [Configurer une passerelle SMS personnalisée pour les connexions Passwordless](/docs/fr-CA/authenticate/passwordless/authentication-methods/use-sms-gateway-passwordless).

<div id="configure-passwordless-sms-settings">
  ### Configurer les paramètres de Passwordless SMS
</div>

1. Dans **Message**, saisissez le texte du corps du SMS.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     L’espace réservé `@@password@@` sera automatiquement remplacé par le mot de passe à usage unique envoyé à l’utilisateur.
   </Callout>
2. Ajustez les paramètres d’**expiration de l’OTP** et de **longueur de l’OTP**.

   * Seul le dernier mot de passe à usage unique (ou lien) émis sera accepté. Dès qu’un nouveau est émis, tous les autres sont invalidés. Une fois utilisé, le plus récent est lui aussi invalidé.
   * Seulement trois tentatives infructueuses de saisie du mot de passe à usage unique sont permises. Après cela, il faudra demander un nouveau code.
   * Le mot de passe à usage unique émis sera valide pendant trois minutes (par défaut) avant d’expirer.
   * Si vous choisissez de prolonger la durée de validité de votre mot de passe à usage unique, vous devriez aussi augmenter la longueur du code. Sinon, un attaquant dispose de plus de temps pour tenter de deviner un code court.
3. Décidez si vous voulez **Disable Sign Ups**. Si vous activez ce paramètre, vous pouvez autoriser l’accès passwordless uniquement aux utilisateurs existants, mais cela peut exposer votre application à des attaques par énumération d’utilisateurs. Pour en savoir plus, consultez [Bonnes pratiques relatives aux connexions Passwordless](/docs/fr-CA/authenticate/passwordless/best-practices).
4. Sélectionnez **Enregistrer**.

<div id="multi-language-support">
  #### Prise en charge multilingue
</div>

Le champ **Message** prend en charge plusieurs langues.

Pour indiquer une langue, appelez le [point de terminaison Get Code or Link de l’Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) et définissez la valeur de l’en-tête `x-request-language`. Si cet en-tête n’est pas défini, la langue est extraite de l’en-tête `accept-language`, que le navigateur définit automatiquement.

<div id="message-syntax">
  #### Syntaxe du champ Message
</div>

La zone **Message** accepte la syntaxe Liquid. Vous pouvez utiliser cette syntaxe, combinée aux valeurs des paramètres, pour générer par programmation des éléments du message.

Par exemple, vous pouvez utiliser le paramètre `request_language` pour changer la langue du message :

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ password }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ password }}
{% endif %}
```

Les paramètres suivants sont disponibles lors de la définition du modèle de message :

| Paramètre            | Description                                                    |
| -------------------- | -------------------------------------------------------------- |
| `password` or `code` | Le mot de passe à utiliser.                                    |
| `phone_number`       | Le numéro de téléphone de l’utilisateur.                       |
| `application.name`   | Le nom de l’application avec laquelle l’utilisateur s’inscrit. |
| `request_language`   | La langue demandée pour le contenu du message.                 |

Pour en savoir plus sur l’utilisation de Liquid, consultez [Liquid for Designers sur Github](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers).

  ### Activer les applications
</div>
