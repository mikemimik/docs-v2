---
title: Authentification Passwordless avec Embedded Login
description: Décrit comment implémenter l’authentification Passwordless avec Universal Login.
---

Pour intégrer l’interface utilisateur de connexion dans votre application, vous pouvez utiliser notre [API Passwordless intégrée](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/relevant-api-endpoints) ou nos SDKs.

Selon le type d’application que vous voulez créer, vous devrez l’implémenter différemment :

* Pour les Single Page Applications (SPAs) (par exemple, Angular ou React), consultez [Authentification Passwordless intégrée pour les Single Page Applications (SPA)](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/spa).
* Pour les Native applications (par exemple, iOS, Android ou les applications de bureau), consultez [Connexion Passwordless intégrée dans les Native Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/native).
* Pour les Regular Web Applications (par exemple, NodeJS, Java, Rails ou .NET), consultez [Connexion Passwordless intégrée dans les Regular Web Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/webapps).

L’Embedded Login dans les applications Web régulières et les SPA nécessite une configuration de l’authentification cross-origin. Pour en savoir plus, consultez [Authentification cross-origin](/docs/fr-CA/authenticate/login/cross-origin-authentication).
