---
title: "Authentification Passwordless avec Embedded Login"
description: "Décrit comment mettre en œuvre l’authentification Passwordless avec Universal Login."
---

Pour intégrer l’interface utilisateur de login à votre application, vous pouvez utiliser notre [API Passwordless intégrée](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/relevant-api-endpoints) ou nos SDKs.

Selon le type d’application que vous souhaitez créer, l’implémentation diffère :

* Pour les Single Page Applications (SPAs) (par exemple, Angular ou React), consultez [Authentification Passwordless intégrée pour les Single Page Applications (SPA)](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/spa).
* Pour les Native Applications (par exemple, iOS, Android ou les applications de bureau), consultez [Login Passwordless intégré dans les Native Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/native).
* Pour les Regular Web Applications (par exemple, NodeJS, Java, Rails ou .NET), consultez [Login Passwordless intégré dans les Regular Web Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/webapps).

Le login intégré dans les applications Web régulières et les SPAs nécessite la configuration de l’authentification cross-origin. Pour en savoir plus, consultez [Cross-Origin Authentication](/docs/fr-CA/authenticate/login/cross-origin-authentication).
