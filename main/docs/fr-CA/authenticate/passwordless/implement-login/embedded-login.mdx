---
title: Authentification Passwordless avec connexion intégrée
description: Décrit comment implémenter l’authentification Passwordless avec Universal Login.
---

Pour intégrer l’interface utilisateur de connexion à votre application, vous pouvez utiliser notre [API Passwordless intégrée](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/relevant-api-endpoints) ou nos SDKs.

Selon le type d’application que vous souhaitez créer, vous devez l’implémenter différemment :

* Pour les applications monopages (SPA) (par exemple, Angular ou React), consultez [Authentification Passwordless intégrée pour les applications monopages (SPA)](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/spa).
* Pour les applications natives (par exemple, iOS, Android ou les applications de bureau), consultez [Connexion Passwordless intégrée dans les applications natives](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/native).
* Pour les applications Web traditionnelles (par exemple, NodeJS, Java, Rails ou .NET), consultez [Connexion Passwordless intégrée dans les applications Web traditionnelles](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/webapps).

La connexion intégrée dans les applications Web traditionnelles et les SPA nécessite la configuration de l’authentification inter-origine. Pour en savoir plus, consultez [Authentification inter-origine](/docs/fr-CA/authenticate/login/cross-origin-authentication).
