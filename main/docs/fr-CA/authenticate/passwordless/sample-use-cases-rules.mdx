Avec les [rules](/docs/fr-CA/customize/rules), vous pouvez gérer des cas plus complexes que ce qui est possible avec les seules [connexions sans mot de passe](/docs/fr-CA/authenticate/passwordless). Par exemple, vous pouvez ajouter des mesures supplémentaires pour mieux confirmer la possession d’une adresse courriel ou d’un appareil.

<div id="require-multi-factor-authentication-for-users-who-are-outside-the-corporate-network">
  ## Exiger l’authentification multifacteur pour les utilisateurs à l’extérieur du réseau d’entreprise
</div>

Supposons que vous vouliez exiger l’[authentification multifacteur (MFA)](/docs/fr-CA/secure/multi-factor-authentication) pour tous les utilisateurs qui accèdent à l’application au moyen d’une connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip> depuis l’extérieur de votre réseau d’entreprise.

À l’aide d’une règle, vous pouvez vérifier si un utilisateur s’authentifie au moyen d’une méthode sans mot de passe (`sms`, `email`) et, si l’adresse IP de sa session se trouve à l’extérieur du réseau d’entreprise défini, lui demander un deuxième facteur d’authentification.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pouvez aussi déclencher cette règle selon d’autres critères, par exemple si l’IP actuelle correspond à l’IP AllowList de l’utilisateur ou si la géolocalisation indique qu’il se trouve dans un pays différent de celui indiqué dans son profil utilisateur.
</Callout>

Pour ce faire, vous devez [créer la règle suivante](/docs/fr-CA/customize/rules/create-rules):

```javascript lines
function(user, context, callback) {
  const ipaddr = require('ipaddr.js');
  const corp_network = "192.168.1.134/26";
  const current_ip = ipaddr.parse(context.request.ip);
  // la méthode d'authentification est-elle sans mot de passe et l'IP est-elle hors du réseau d'entreprise ?
  const passwordlessOutside = context.authentication.methods.find(
    (method) => (
      ((method.name === 'sms') || (method.name === 'email')) && 
      (!current_ip.match(ipaddr.parseCIDR(corp_network)))
    )
  );

  // si oui, exiger l'AMF
  if (passwordlessOutside) {
    context.multifactor = {
      provider: 'any',
      allowRememberBrowser: false
    };
  }
  callback(null, user, context);
}
```
