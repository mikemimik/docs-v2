---
description: Décrit certaines limites de l’utilisation des connexions sans mot de passe.
title: Limites des connexions sans mot de passe
---

Le type d’authentification que vous souhaitez utiliser dépend de la version de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> que vous avez configurée pour votre application. Si vous utilisez l’expérience Universal Login, vous pouvez configurer [l’authentification sans mot de passe avec WebAuthN et Device Biometrics](/docs/fr-CA/authenticate/login/auth0-universal-login/passwordless-login/webauthn-device-biometrics) ou utiliser des connexions <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip>, comme décrit ci-dessous.

Vous pouvez configurer l’authentification sans mot de passe par courriel ou par SMS à l’aide des méthodes suivantes :

1. Universal Login + paramètres de connexion sans mot de passe
2. Classic Login + Lock
3. Classic Login + interface utilisateur personnalisée + Auth0.js

Pour en savoir plus sur ces implémentations, consultez [Configurer le courriel ou les SMS pour l’authentification sans mot de passe](/docs/fr-CA/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="errors">
  ### Erreurs
</div>

Si vous utilisez une connexion intégrée, nous vous recommandons d’utiliser les <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou de fantaisie." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=custom+domains">domaines personnalisés</Tooltip>. Pour en savoir plus, consultez [Custom Domains](/docs/fr-CA/customize/custom-domains).

<div id="considerations">
  ## Considérations
</div>

* Vous ne pouvez pas créer d’utilisateurs sans mot de passe dans le <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Créez-les directement avec la [Management API](https://auth0.com/docs/api/management/v2#!/Users/post_users).
* Les <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Refresh+tokens">jetons d’actualisation</Tooltip> ne peuvent pas être récupérés lorsque vous utilisez un lien magique sans mot de passe par courriel; seul un OTP peut l’être.
