Auth0 offre des connexions d’entreprise pour authentifier les utilisateurs au moyen d’un <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> (IdP) externe fédéré, comme Azure AD, Google Workspace, PingFederate, et bien d’autres.

<Card title="La disponibilité varie selon le forfait Auth0">
  La disponibilité de cette fonctionnalité dépend de votre forfait Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [la page des tarifs d’Auth0](https://auth0.com/pricing).
</Card>

<div id="create-an-enterprise-connection">
  ## Créer une connexion d’entreprise
</div>

Auth0 prend en charge nativement de nombreux fournisseurs d’identité. Pour en savoir plus, consultez [les fournisseurs d’identité d’entreprise](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers).

<div id="openid-connect-oidc-protocol">
  ### Protocole OpenID Connect (OIDC)
</div>

Les connexions d’entreprise qui utilisent <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les renseignements de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OpenID">OpenID</Tooltip> Connect ou Okta Workforce comme fournisseur d’identité peuvent prendre en charge ce qui suit :

* [Proof Key for Code Exchange (PKCE)](https://www.oauth.com/oauth2-servers/pkce/)

* Claims d’attribut et mappage du <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip>

* Intégration UserInfo

Vous pouvez mettre en œuvre ces fonctionnalités pour les connexions [OpenID Connect](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/oidc) ou [Okta Workforce](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/okta). Pour en savoir plus, consultez [Configure PKCE and Claim Mapping for OIDC Connections](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/configure-pkce-claim-mapping-for-oidc).

<div id="view-enterprise-connections">
  ## Voir les connexions d’entreprise
</div>

<Tabs>
  <Tab title="Dashboard">
    Accédez à [**Auth0 Dashboard &gt; Authentication &gt; Enterprise**](https://manage.auth0.com/#/connections/enterprise) pour voir tous les types de connexions d’entreprise offerts. Sélectionnez un type de connexion (par exemple, SAML) pour vérifier s’il existe des connexions configurées de ce type.

    Vous pouvez aussi sélectionner une connexion configurée et consulter l’onglet **Applications** pour voir si elle est activée pour certaines applications.
  </Tab>

  <Tab title="Management API">
    Appelez le point de terminaison [Get all Connections de l’Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Connections/get_connections) pour obtenir des renseignements sur vos connexions. Incluez le paramètre `strategy` pour filtrer par type de connexion.

    Les valeurs `strategy` du type de connexion d’entreprise sont les suivantes :

    * `ad` (Active Directory/LDAP)
    * `adfs` (ADFS)
    * `auth0-adldap`
    * `google-apps` (Google Workspace)
    * `ip`
    * `office365`
    * `oidc` (OpenID Connect)
    * `pingfederate` (PingFederate)
    * `samlp` (SAML)
    * `sharepoint`
    * `waad` (Microsoft Azure AD)
  </Tab>
</Tabs>

<div id="what-is-an-active-enterprise-connection">
  ## Qu’est-ce qu’une connexion d’entreprise active?
</div>

Une connexion d’entreprise est considérée comme active si, au cours du mois en cours, elle remplit les deux conditions suivantes :

* Elle a été activée pour une application.
* Elle a enregistré une activité utilisateur (par exemple, une connexion, une inscription ou un échange de jetons).

Si une connexion d’entreprise n’a jamais été activée pour une application, ou si elle a été activée sans enregistrer d’activité utilisateur au cours du mois en cours, elle n’est pas considérée comme active.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La connexion Okta Workforce Enterprise n’est pas prise en compte dans votre limite de connexions d’entreprise actives.
</Callout>
