---

Auth0 prend en charge la connexion d’entreprise pour les applications Web et les applications natives. La connexion d’entreprise est une méthode d’authentification qui permet aux utilisateurs de se connecter à une application à l’aide des identifiants existants d’un fournisseur d’identité d’entreprise, comme Google Workspace ou Microsoft Azure Active Directory (Entra ID). Cela se distingue de la connexion d’applications à un fournisseur externe et de leur autorisation afin qu’elles puissent accéder à des API externes au nom de l’utilisateur. Pour en savoir plus, consultez [Authentification de l’utilisateur vs comptes connectés](/docs/fr-CA/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).

Auth0 prend en charge les fournisseurs d’identité d’entreprise suivants, prêts à l’emploi :

* [Active Directory/LDAP](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap)
* [ADFS](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/adfs)
* [Azure Active Directory Native](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory-native)
* [Google Workspace](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/google-apps)
* [OpenID Connect](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/oidc)
* [Okta](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/okta)
* [PingFederate](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/ping-federate)
* [SAML](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/saml)
* [Azure Active Directory](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les forfaits B2B et Enterprise d’Auth0 incluent un nombre illimité de [Okta Enterprise Connections](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/okta). Pour en savoir plus, consultez [la page des tarifs d’Auth0](https://auth0.com/pricing).
</Callout>

Vous pouvez aussi découvrir des connexions d’entreprise prises en charge par des partenaires dans l’[Auth0 Marketplace](https://marketplace.auth0.com/features/enterprise-connections).

<div id="openid-connect-oidc-protocol">
  ## Protocole OpenID Connect (OIDC)
</div>

Les connexions d’entreprise qui utilisent <Tooltip tip="OpenID: Norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker leurs renseignements de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OpenID">OpenID</Tooltip> Connect ou Okta Workforce comme <Tooltip tip="OpenID: Norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker leurs renseignements de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> peuvent prendre en charge les éléments suivants :

* Clé de preuve pour l’échange de code (PKCE)

  * Pour en savoir plus sur PKCE, consultez [Protéger les applications avec PKCE](https://www.oauth.com/oauth2-servers/pkce/).
* Mappage des claims d’attribut et du <Tooltip tip="Access Token: Informations d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisées pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip>
* Intégration UserInfo

Vous pouvez actuellement mettre en œuvre ces fonctionnalités pour les connexions [OpenID Connect](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/oidc) ou [Okta Workforce](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/okta). Pour en savoir plus, consultez [Configurer PKCE et le mappage des claims pour les connexions OIDC](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/configure-pkce-claim-mapping-for-oidc).
