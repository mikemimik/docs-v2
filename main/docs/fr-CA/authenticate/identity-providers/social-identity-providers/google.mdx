Auth0 prend en charge Sign in with Google, ce qui permet à vos utilisateurs de s’authentifier avec leur compte Google via <Tooltip tip="OAuth 2.0 : Cadre d’autorisation qui définit les protocoles et workflows d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>. Ajouter Google comme option de connexion sociale offre une expérience de connexion rapide et familière, que la plupart des utilisateurs connaissent déjà et à laquelle ils font confiance.

Pour configurer une connexion sociale Google, vous devez :

1. Créer des identifiants OAuth Google dans Google Auth Platform.
2. Configurer et tester une connexion sociale Google dans l’Auth0 Dashboard.

<Card title="Avant de commencer">
  Avant de configurer Google comme connexion sociale, vous devez disposer des éléments suivants :

  1. Un [compte Auth0](https://auth0.com/signup). Si vous n’en avez pas, vous pouvez vous inscrire gratuitement.
  2. Un [compte Google Developer](https://console.cloud.google.com/).
  3. Un [projet Google](https://developers.google.com/workspace/guides/create-project).
  4. Une application enregistrée dans l’[Auth0 Dashboard](https://manage.auth0.com/#/applications).
</Card>

<div id="google-auth-platform">
  ## Google Auth Platform
</div>

La Google Auth Platform est une section de la [Google Cloud Console](https://console.cloud.google.com/) qui vous permet de gérer vos applications et vos identifiants OAuth pour vous connecter et appeler les API Google. Pour en savoir plus, consultez [Get started with the Google Auth Platform](https://developers.google.com/identity/protocols/oauth2) dans la documentation de Google.

Utilisez la Google Auth Platform pour :

1. Configurer l’écran de consentement Google.
2. Créer un client Google OAuth 2.0.

<div id="configure-the-google-consent-screen">
  ### Configurer l’écran de consentement Google
</div>

Avant de créer un ID client OAuth, vous devez configurer l’écran de consentement OAuth avec des renseignements sur votre application. Lorsque vous utilisez OAuth 2.0 à des fins d’autorisation, votre application demande un ou plusieurs scopes d’accès à un compte Google. Google affiche alors un écran de consentement à l’utilisateur, avec un résumé de votre projet, de ses politiques et des scopes d’accès demandés.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si votre application demande des scopes OAuth sensibles ou utilise une image personnalisée, [Google la limite à 100 connexions jusqu’à ce que l’écran de consentement OAuth soit vérifié](https://support.google.com/cloud/answer/10311615). La vérification de l’écran de consentement peut prendre plusieurs jours.
</Callout>

Dans Google Cloud Console, [configurez votre écran de consentement Google OAuth](https://support.google.com/cloud/answer/10311615) :

<Steps>
  <Step title="Configurer l’image de marque">
    Accédez à **Google Auth Platform &gt; Branding**. Pour **Authorized domains**, saisissez `auth0.com`. Si vous utilisez un domaine personnalisé, saisissez plutôt votre domaine personnalisé.
  </Step>

  <Step title="Configurer l’audience">
    Accédez à **Google Auth Platform &gt; Audience**. Pour **User type**, sélectionnez **Make External**. Dans **Test Users**, vous pouvez ajouter les adresses courriel que vous souhaitez utiliser pour les tests.
  </Step>

  <Step title="Configurer l’accès aux données">
    Accédez à **Google Auth Platform &gt; Data Access** pour ajouter ou supprimer des scopes. Pour en savoir plus, consultez [OAuth 2.0 Scopes for Google APIs](https://developers.google.com/identity/protocols/oauth2/scopes) dans la documentation de Google.
  </Step>

  <Step title="Enregistrer vos modifications">
    Suivez le reste des instructions pour terminer la configuration de votre écran de consentement Google OAuth. Sélectionnez **Save Changes**.
  </Step>
</Steps>

<div id="create-a-google-oauth-20-client">
  ### Créer un client Google OAuth 2.0
</div>

Pour créer un client Google OAuth 2.0, vous avez besoin de votre domaine Auth0, que vous trouverez dans l’Auth0 Dashboard.

Accédez à [Auth0 Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/custom_domains) pour trouver votre domaine :

* Si vous n’avez pas configuré de domaine personnalisé, votre nom de domaine Auth0 est `YOUR_TENANT_NAME.YOUR_REGIONAL_SUBDOMAIN.auth0.com`. Votre URI de redirection est `https://YOUR_TENANT_NAME.YOUR_REGIONAL_SUBDOMAIN.auth0.com/login/callback`.
* Si vous avez configuré un domaine personnalisé, utilisez ce domaine à la place. Votre URI de redirection est `https://YOUR_CUSTOM_DOMAIN/login/callback`.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous avez créé votre tenant américain avant juin 2020, votre nom de domaine Auth0 n’inclut pas le sous-domaine régional : `YOUR_TENANT_NAME.auth0.com`.
</Callout>

Dans la Google Cloud Console, créez un nouveau client OAuth 2.0 :

<Steps>
  <Step title="Créer un nouveau client">
    Accédez à **Google Auth Platform &gt; Clients**. Sélectionnez **New Client**.
  </Step>

  <Step title="Sélectionner le type d’application">
    Pour **Application type**, sélectionnez **Web application**.
  </Step>

  <Step title="Saisir les détails du client">
    Saisissez les renseignements suivants pour votre client OAuth 2.0 :

    | **Champ**                         | **Valeur**                           |
    | --------------------------------- | ------------------------------------ |
    | **Name**                          | Le nom de votre client OAuth 2.0     |
    | **Authorized JavaScript origins** | `https://YOUR_DOMAIN`                |
    | **Authorized redirect URIs**      | `https://YOUR_DOMAIN/login/callback` |

    Remplacez `YOUR_DOMAIN` par le domaine Auth0 trouvé à l’étape précédente.
  </Step>

  <Step title="Créer le client">
    Sélectionnez **Create**. Copiez le **Client ID** et le **Client Secret** affichés par Google.
  </Step>
</Steps>

<div id="configure-auth0">
  ## Configurer Auth0
</div>

Utilisez l’Auth0 Dashboard pour créer et configurer une connexion sociale Google avec les identifiants OAuth que vous avez créés.

<Steps>
  <Step title="Créer une connexion sociale Google">
    Accédez à [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/social). Sélectionnez **Créer une connexion**, puis **Google/Gmail**.
  </Step>

  <Step title="Entrer un nom et vos identifiants Google">
    Dans **Général**, entrez un **Nom** pour la connexion ainsi que les identifiants du client Google OAuth 2.0 que vous avez créé :

    * **Client ID** : L’identifiant unique de votre application.
    * **Client Secret** : Le secret utilisé par Auth0 pour s’authentifier auprès de Google au nom de votre application.
  </Step>

  <Step title="Sélectionner les permissions et enregistrer">
    Sélectionnez les permissions que vous souhaitez activer pour la connexion. La plupart des applications ont besoin de **Basic Profile** pour l’adresse courriel de l’utilisateur et l’indicateur d’adresse courriel vérifiée, ainsi que de **Extended Profile** pour le nom, la photo et d’autres détails du profil.

    Sélectionnez **Save Changes**.
  </Step>
</Steps>

<div id="test-the-connection">
  ## Tester la connexion
</div>

Une fois votre connexion sociale Google créée, vérifiez qu’elle fonctionne avant de la déployer en production.

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/social).
2. Sélectionnez la connexion Google dans la liste.
3. Sélectionnez **Tester la connexion**.
4. Connectez-vous avec votre compte Google lorsque vous y êtes invité.
5. Confirmez qu’Auth0 renvoie les données du profil de l’utilisateur, y compris son nom et son adresse courriel.
