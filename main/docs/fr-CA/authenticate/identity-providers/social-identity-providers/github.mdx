Auth0 prend en charge la connexion avec GitHub, ce qui permet à vos utilisateurs de s’authentifier avec leur compte GitHub au moyen de <Tooltip tip="OAuth 2.0 : cadre d’autorisation qui définit les protocoles et les flux d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>. Ajouter GitHub comme option de connexion sociale offre une expérience de connexion rapide et familière aux développeurs et aux utilisateurs techniques.

Pour configurer cette connexion, vous devez enregistrer une application OAuth dans GitHub, puis configurer la connexion dans l’Auth0 Dashboard à l’aide des identifiants fournis par GitHub.

<Card title="Avant de commencer">
  Avant de configurer GitHub comme connexion sociale, vous devez avoir :

  1. Un [compte Auth0](https://auth0.com/signup). Si vous n’en avez pas, vous pouvez vous inscrire gratuitement.
  2. Un [compte GitHub](https://github.com/signup).
  3. Une application enregistrée dans l’[Auth0 Dashboard](https://manage.auth0.com/#/applications).
</Card>

<div id="configure-github">
  ## Configurer GitHub
</div>

Créez une application OAuth dans GitHub afin de générer les identifiants dont Auth0 a besoin pour établir la connexion.

<Steps>
  <Step title="Ouvrir les paramètres de développeur GitHub">
    Connectez-vous à votre compte GitHub. Sélectionnez votre photo de profil dans le coin supérieur droit, puis sélectionnez **Settings**. Dans la barre latérale gauche, sélectionnez **Developer settings**, puis **OAuth Apps**.
  </Step>

  <Step title="Enregistrer une nouvelle application OAuth">
    Sélectionnez **New OAuth App**. Si vous n’avez encore créé aucune application, sélectionnez **Register a new application**.

    Remplissez les champs suivants :

    | **Champ**                      | **Valeur**                                                                                             |
    | ------------------------------ | ------------------------------------------------------------------------------------------------------ |
    | **Application name**           | Un nom que vos utilisateurs reconnaîtront et auquel ils feront confiance, par exemple `My App (Auth0)` |
    | **Homepage URL**               | L’URL complète de la page d’accueil de votre application, par exemple `https://myapp.example.com`      |
    | **Application description**    | (Facultatif) Une description affichée à tous les utilisateurs de votre application                     |
    | **Authorization callback URL** | `https://YOUR_AUTH0_DOMAIN/login/callback`                                                             |

    Remplacez `YOUR_AUTH0_DOMAIN` par le domaine de votre tenant Auth0. Vous trouverez cette valeur dans [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), sous l’onglet **Settings**.

    Vous pouvez laisser **Enable Device Flow** décoché, sauf si votre application exige une autorisation par appareil.

    Sélectionnez **Register application**.
  </Step>

  <Step title="Copier le Client ID et le Client Secret">
    Après l’enregistrement, GitHub affiche la page des détails de l’application. Copiez la valeur du **Client ID**.

    Sélectionnez **Generate a new client secret** pour créer un secret. Copiez immédiatement la valeur générée.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      GitHub n’affiche le Client Secret qu’une seule fois. Conservez-le en lieu sûr avant de quitter la page. Si vous le perdez, vous devrez en générer un nouveau.
    </Callout>
  </Step>
</Steps>

<div id="configure-auth0">
  ## Configurer Auth0
</div>

Ajoutez la connexion GitHub à votre tenant Auth0 et saisissez les identifiants de la section précédente.

<Steps>
  <Step title="Créer une connexion GitHub">
    Accédez à [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/social) et sélectionnez **Create Connection**. Sélectionnez **GitHub** dans la liste des fournisseurs.
  </Step>

  <Step title="Sélectionner l’objectif de la connexion">
    Sous **Purpose**, précisez comment cette connexion sera utilisée : pour la connexion, les comptes liés ou les deux.
  </Step>

  <Step title="Saisir vos identifiants GitHub">
    Dans **General**, saisissez le **Client ID** et le **Client Secret** que vous avez copiés depuis GitHub.
  </Step>

  <Step title="Sélectionner les permissions">
    Sous **Permissions**, sélectionnez les permissions requises par votre application. La plupart des applications ont besoin de **Email address** pour l’adresse courriel de l’utilisateur et de **Read user** pour les renseignements du profil.
  </Step>

  <Step title="Enregistrer et activer la connexion">
    Sélectionnez **Create** pour enregistrer la connexion.

    Sélectionnez l’onglet **Applications** et activez la connexion pour les applications qui doivent offrir la connexion avec GitHub.
  </Step>
</Steps>

<div id="test-the-connection">
  ## Tester la connexion
</div>

Vérifiez que la connexion GitHub fonctionne avant de la déployer en production.

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/social).
2. Sélectionnez la connexion GitHub dans la liste.
3. Sélectionnez **Try Connection**.
4. Authentifiez-vous avec votre compte GitHub lorsque vous y êtes invité.
5. Confirmez qu’Auth0 renvoie les données de profil utilisateur, y compris le nom d’utilisateur GitHub et l’adresse courriel de l’utilisateur.
