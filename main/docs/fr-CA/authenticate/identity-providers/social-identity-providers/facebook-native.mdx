Vous pouvez ajouter des fonctionnalités à votre application native afin de permettre à vos utilisateurs de s’authentifier avec Facebook de façon native, directement dans l’application. Cela ne nécessite aucune redirection par navigateur Web et permet aux applications mobiles de se conformer à la [Facebook Developer Policy](https://developers.facebook.com/policy/), qui exige que les applications mobiles utilisent le SDK Facebook pour [Android](https://developers.facebook.com/docs/android) ou [iOS](https://developers.facebook.com/docs/ios) pour authentifier les utilisateurs.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Lors de l’intégration aux Facebook SDKs, vos applications partageront des données avec Facebook. Assurez-vous de bien comprendre quelles données sont partagées et d’en faire état correctement dans la politique de confidentialité de votre application. Auth0 n’a aucun contrôle sur les données qui seront partagées avec Facebook par l’intermédiaire du SDK.

  Consultez la page [Facebook GDPR](https://www.facebook.com/business/m/one-sheeters/gdpr-developer-faqs) pour obtenir plus d’informations sur les données recueillies par le SDK Facebook et Facebook Login.
</Callout>

<div id="how-it-works">
  ## Fonctionnement
</div>

Le flux de connexion Facebook Login natif fonctionne comme suit :

* **Étape 1** : L’application authentifie un utilisateur à l’aide du SDK Facebook et obtient un <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip>.
* **Étape 2** : L’application utilise ce jeton d’accès pour demander un [Facebook Session Info Access Token](https://developers.facebook.com/docs/facebook-login/access-tokens/session-info-access-token) spécial.
* **Étape 3** : Utilisez le SDK Facebook pour récupérer le profil de l’utilisateur.
* **Étape 4** : L’application peut ensuite utiliser le jeton Facebook Session Info pour s’authentifier auprès d’Auth0.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de configurer Facebook Login natif pour votre application native via Auth0, vous devez :

1. [Configurer Facebook comme connexion Auth0](https://marketplace.auth0.com/integrations/facebook-social-connection)
2. [Utiliser le SDK Facebook approprié dans votre application](https://developers.facebook.com/docs/apis-and-sdks/)
3. Accéder à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et créer une application dans Auth0 (si ce n’est pas déjà fait).
4. Au bas de la page Settings, sélectionnez **Show Advanced Settings**, puis la vue **Device Settings**. Sous **Native Social Login**, activez la bascule **Enable Sign In with Facebook**.

<div id="implementation">
  ## Mise en œuvre
</div>

Du point de vue de votre application, le processus d’authentification d’un profil d’utilisateur au moyen de Facebook Login natif comporte quatre étapes :

<div id="step-1">
  ### Étape 1
</div>

L’application authentifie un utilisateur au moyen du Facebook SDK. Elle obtiendra un jeton d’accès auprès de Facebook.

<div id="step-2">
  ## Étape 2
</div>

L’application utilise le jeton d’accès pour demander un [Facebook Session Info Access Token](https://developers.facebook.com/docs/facebook-login/access-tokens/session-info-access-token).

Cette requête prendra une forme semblable à la suivante :

```json wrap lines
GET https://graph.facebook.com/v5.0/oauth/access_token?grant_type=fb_attenuate_token&client_id=457704041391802&fb_exchange_token=<facebook_access_token>
```

et la réponse :

```json lines
{
    "access_token": "XAAGgR4b...1lHWNCpqrAhcpoAZDZD",
    "token_type": "bearer",
    "expires_in": 5183924
}
```

<div id="step-3">
  ### Étape 3
</div>

L’application doit récupérer le profil utilisateur depuis Facebook à l’aide du Facebook SDK, ce qui donnera lieu à une requête semblable à la suivante :

```json wrap lines
GET https://graph.facebook.com/v5.0/<facebook user id>?access_token=<facebook access token>&fields=email,name
```

<div id="step-4">
  ### Étape 4
</div>

L’application peut ensuite utiliser le jeton d’accès des renseignements de session et le profil utilisateur Facebook pour s’authentifier auprès d’Auth0 en appelant le point de terminaison `/oauth/token` d’Auth0 au moyen du flux Token Exchange avec le type de jeton `facebook-session-access-token`. Si tout se passe bien, Auth0 renverra une réponse d’échange normale, avec en plus le profil utilisateur. Le profil utilisateur doit être un objet JSON encodé sous forme de chaîne de caractères.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample = `POST https://{yourDomain}/oauth/token

grant_type: 'urn:ietf:params:oauth:grant-type:token-exchange'
subject_token_type: 'http://auth0.com/oauth/token-type/facebook-info-session-access-token'
audience: 'your-api'
scope: 'read:appointments openid profile email email_verified'
subject_token: 'XAAGgR4b...1lHWNCpqrUHZAEtUuZAhcpoAZDZD'
client_id: '{yourClientId}'
user_profile: '{"email":"john@example.com", "name":"John Doe"}'`;

<AuthCodeBlock children={codeExample} language="json" />

et la réponse d’Auth0 :

```json lines
{
    "access_token": "eyJ0eXA..yXQaPLVXg",
    "id_token": "eyJ0.tFE5HPipdOsA",
    "scope": "openid profile email read:appointments",
    "expires_in": 86400,
    "token_type": "Bearer"
}
```

<div id="user-profile-and-email-validation">
  ## Profil d’utilisateur et validation du courriel
</div>

Dans l’exemple précédent, vous deviez récupérer le profil d’utilisateur à partir de Facebook et l’inclure dans l’appel à `/oauth/token`. En effet, le jeton d’accès de session Facebook ne peut pas servir à récupérer directement le profil, et le jeton d’accès Facebook ne peut pas être envoyé directement au serveur en raison des [AppStore Review Guidelines d’Apple](https://developer.apple.com/app-store/review/guidelines). Il doit donc être récupéré dans le client et envoyé à Auth0 de cette manière.

Comme Auth0 ne peut pas garantir que le profil d’utilisateur est le même que celui renvoyé par Facebook, il attribuera la valeur `false` au champ `email_verified`.

<div id="logout">
  ## Déconnexion
</div>

Comme l’implémentation de connexion native n’a pas recours aux flux standards basés sur le navigateur, les propriétaires d’application doivent aussi veiller à effectuer la déconnexion correctement. Lorsqu’une application doit effectuer une déconnexion, elle doit aussi [révoquer le Refresh Token d’Auth0](https://auth0.com/docs/api/authentication#revoke-refresh-token).
