---
description: Découvrez comment redemander à l’utilisateur d’accorder des permissions à votre application.
title: Gérer les permissions d’autorisation refusées
---

Lorsque vos utilisateurs autorisent votre application, certains fournisseurs (comme Facebook) leur permettent de sélectionner les attributs qu’ils souhaitent partager.

Par défaut, cette sélection n’est effectuée que lorsque l’utilisateur autorise l’application pour la première fois. Si votre utilisateur choisit de ne pas autoriser certains attributs (comme son courriel) requis par votre application, il ne pourra pas accéder à votre application.

Dans de tels cas, l’utilisateur devra être invité de nouveau à accorder la permission pour les attributs requis afin de se connecter.

<div id="re-prompt-for-permissions">
  ## Demander à nouveau les permissions
</div>

En définissant le paramètre **prompt=consent** lorsque vous appelez le point de terminaison [/authorize](https://auth0.com/docs/api/authentication/reference#social) de l’[API d’autorisation](https://auth0.com/docs/api/authentication), l’utilisateur sera de nouveau invité à accorder les permissions à votre application.

Ce paramètre peut aussi être défini à l’aide de Lock comme [paramètre d’authentification](/docs/fr-CA/libraries/lock/lock-authentication-parameters), avec **prompt: &#39;consent&#39;**.

Vous pouvez aussi le définir avec [Auth0.js](https://github.com/auth0/auth0.js) en utilisant **prompt: &#39;consent&#39;**.
