Une fois l&#39;utilisateur connecté, vous pouvez récupérer son profil utilisateur, puis l&#39;`accessToken` associé, pour appeler les API du <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP), comme décrit dans [Appeler une API d’un fournisseur d’identité](/docs/fr-CA/authenticate/identity-providers/calling-an-external-idp-api).

Toutefois, si vous recevez `Access Denied` lorsque vous appelez l’API de l’IdP, c’est probablement parce que vous n’avez pas demandé les permissions appropriées pour l’utilisateur au moment de la connexion. Vous pouvez demander les permissions appropriées de l’une des deux façons suivantes.

<div id="change-identity-provider-settings">
  ## Modifier les paramètres du fournisseur d’identité
</div>

Pour configurer les scopes/autorisations requis de l’utilisateur, accédez à [Auth0 Dashboard &gt; Authentification &gt; Social](https://manage.auth0.com/#/connections/social), puis sélectionnez un IdP. Vous pouvez sélectionner les autorisations requises affichées sur l’écran de configuration.

Par exemple, si vous cliquez sur la connexion **Google / Gmail**, vous pouvez configurer des autorisations propres à Google :

<Frame>![Autorisations pour Google](/docs/images/cdy7uua7fh8z/61ACa6hnMtO5aUjus0fCb7/31411373a18463f272107e1124445c60/dashboard-connections-social-create_google.png)</Frame>

<div id="pass-scopes-to-authorize-endpoint">
  ## Transmettre des scopes au point de terminaison Authorize
</div>

Vous pouvez aussi transmettre les scopes ou autorisations que vous souhaitez demander sous forme de liste séparée par des virgules dans le paramètre `connection_scope` lors de l’appel au [point de terminaison Authorize](https://auth0.com/docs/api/authentication#login). Par exemple, si vous voulez demander à Google les scopes `https://www.googleapis.com/auth/contacts.readonly` et `https://www.googleapis.com/auth/analytics`, vous pouvez les transmettre avec le paramètre `connection` pour vous assurer que l’utilisateur ouvre une session avec son compte Google :

export const codeExample = `https://{yourDomain}/authorize
  ?response_type=id_token
  &client_id={yourClientId}
  &redirect_uri={https://yourApp/callback}
  &scope=openid%20profile
  &connection=google-oauth2
  &connection_scope=https%3A%2F%2Fwww.googleapis.com%2Fauth%2Fanalytics%2Chttps%3A%2F%2Fwww.googleapis.com%2Fauth%2Fcontacts.readonly
  &nonce=abc`;

</Callout>
