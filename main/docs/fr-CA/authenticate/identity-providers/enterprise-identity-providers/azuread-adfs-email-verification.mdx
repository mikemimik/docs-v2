---
description: Décrit comment contrôler la façon dont le champ `email_verified` est défini pour Azure AD et ADFS.
title: Vérification du courriel pour Azure AD et ADFS
---

Le profil utilisateur Auth0 comprend un champ `email_verified`, qui peut être défini de différentes façons selon le type de connexion. Pour les connexions de base de données, les utilisateurs doivent passer par un processus de validation du courriel pour que leur adresse courriel soit vérifiée. Pour les connexions fédérées, les <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d&#39;identité</Tooltip> peuvent renvoyer le champ `email_verified` selon leurs propres critères.

Azure AD et ADFS ne peuvent pas garantir que les courriels qu&#39;ils renvoient ont été vérifiés :

* Dans ADFS, l&#39;administrateur ADFS peut configurer n&#39;importe quelle adresse courriel.
* Dans Azure AD, selon la façon dont le tenant Azure AD est configuré, les adresses courriel renvoyées par Azure AD peuvent correspondre ou non à des boîtes aux lettres Office. Auth0 ne peut pas le savoir.

Cependant, si vous savez comment Azure AD ou ADFS est configuré et géré, vous pouvez choisir de considérer les courriels de ces comptes comme vérifiés.

Pour répondre à ces deux besoins, les connexions Azure AD et ADFS ont une propriété **Vérification du courriel** avec deux valeurs :

* Toujours définir `email_verified` à `true`
* Toujours définir `email_verified` à `false`

La connexion Azure AD a aussi une propriété **Utiliser un endpoint commun**. Lorsqu&#39;elle est activée, l&#39;utilisateur peut s&#39;authentifier avec n&#39;importe quel tenant Azure AD. Comme il n&#39;est pas possible de présumer que n&#39;importe quel tenant Azure AD renverra des courriels vérifiés, la propriété **Vérification du courriel** devra être définie à **Toujours définir** `email_verified` **à** `false`.

Lorsque la propriété est définie à **Toujours définir** `email_verified` **à** `false`, les utilisateurs auront `email_verified` défini à `false` à leur prochaine connexion, à moins que la [synchronisation des attributs du profil utilisateur à chaque connexion](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) soit désactivée.

<div id="azure-adadfs-email-verification-migration-setting">
  ## Paramètre de migration vérification du courriel pour les connexions Azure AD/ADFS
</div>

Dans les versions précédentes, Auth0 définissait toujours le champ `email_verified` sur `true` dans les connexions Azure AD et ADFS. Si vous utilisiez des connexions Azure AD et ADFS auparavant, vous disposerez d’un paramètre de tenant qui remplacera le paramètre de connexion pour **vérification du courriel** et conservera le comportement précédent.

Vous trouverez le nouveau paramètre de tenant dans [Auth0 Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced). Repérez la section **Migrations**, puis le paramètre **Default to** `Email Verification` **setting for Azure AD/ADFS connections**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Ce paramètre est disponible uniquement si :

  * le tenant est antérieur à la date d’introduction de la migration
  * le tenant a une ou plusieurs connexions ADFS ou Azure AD actives
</Callout>

<Frame>![Dashboard - Paramètres avancés du tenant - Migrations](/docs/images/cdy7uua7fh8z/4K1rQIv6p96KPCYOrhjXz4/be0f7b104d743fd0d3d42d18df97e144/dashboard-tenant-edit_view-advanced_migrations_azure-adfs.png)</Frame>

Lorsque ce paramètre est désactivé, `email_verified` sera toujours défini sur `true` pour les connexions Azure AD/ADFS. Lorsqu’il est activé, le paramètre `Email Verification` au niveau de la connexion sera utilisé.

<div id="email-verification-flow-for-azure-adadfs-connections">
  ## Flux de vérification du courriel pour les connexions Azure AD/ADFS
</div>

Si votre application exige que les courriels des utilisateurs d’une connexion Azure AD/ADFS soient toujours vérifiés, vous pouvez activer l’option **Enable email verification flow during login for Azure AD and ADFS connections** dans la section **Advanced Settings** du tenant.

Après s’être authentifié pour la première fois avec un courriel non vérifié, l’utilisateur devra vérifier son courriel en saisissant un code à usage unique envoyé à son compte de courriel :

<Frame>![Auth0 - Invite de vérification du courriel - Code à usage unique](/docs/images/cdy7uua7fh8z/i92GJgntVuU5QK37adugH/b767d4db92de5bafd23113042eca2600/Verify_Email_-_English.png)</Frame>

Si l’utilisateur termine cette étape, le champ `email_verified` sera défini à `true`, et il ne sera plus invité à effectuer la vérification du courriel, à moins qu’Azure AD ou ADFS renvoie une adresse courriel différente pour cet utilisateur.

Ce nouvel écran s’affiche dans l’expérience <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>, même si vous utilisez Classic Login. Pour savoir comment le personnaliser, consultez [Personnaliser les pages Universal Login](/docs/fr-CA/customize/login-pages/universal-login/customize-templates).

Pour savoir comment personnaliser le courriel envoyé aux utilisateurs, consultez [Vérifier les courriels avec Auth0](/docs/fr-CA/manage-users/user-accounts/verify-emails).

<Warning>
  Lorsque Azure AD ne renvoie pas de claim `email`, Auth0 mappe le [UserPrincipalName Azure](https://docs.microsoft.com/en-us/azure/active-directory/hybrid/plan-connect-userprincipalname) à l’adresse courriel. Rien ne garantit que la valeur `UserPrincipalName` corresponde à une boîte aux lettres; Auth0 **n’affichera donc pas** l’invite de vérification du courriel, et le champ `email_verified` de l’utilisateur sera défini à `false`.
</Warning>
