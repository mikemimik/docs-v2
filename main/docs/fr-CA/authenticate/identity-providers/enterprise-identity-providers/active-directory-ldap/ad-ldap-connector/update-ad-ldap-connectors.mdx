---
description: Mettre à niveau un AD/LDAP Connector Auth0 installé vers la plus récente version sous Windows ou Linux, tout en conservant la configuration existante et le lien avec le tenant.
title: Mettre à jour les AD/LDAP Connectors
---

S’il existe plusieurs instances de l’AD/LDAP Connector dans votre déploiement, Auth0 recommande d’effectuer les étapes ci-dessous pour chaque instance, une à la fois, afin qu’une seule instance soit indisponible à un moment donné.

<div id="admin-console-windows">
  ## Admin Console (Windows)
</div>

1. Ouvrez l’Admin Console en accédant à `http://localhost:8357/`. Si l’Admin Console a un accès sortant à Internet, elle vérifiera si une nouvelle version est disponible et l’indiquera en haut de la page :

   <Frame>![Écran de mise à jour de l’Admin Console des AD/LDAP Connectors](/docs/images/cdy7uua7fh8z/6Yp2uRnEU3RMeCqgHa6Ztl/b9c04ab83b9fd725910291b01ac9d05d/connector-update-available.png)</Frame>
2. Accédez à l’onglet **Update** pour mettre à jour l’installation sur la machine actuelle vers la version la plus récente. La mise à jour prendra environ 2 minutes et les journaux de mise à jour s’afficheront dans l’Admin Console.

<div id="updater-script-windows">
  ## Script de mise à jour (Windows)
</div>

Le script de mise à jour permet de mettre à jour l’AD/LDAP Connector en ligne de commande en exécutant les étapes suivantes :

1. Vérifier si une mise à jour est disponible.
2. Sauvegarder la configuration existante, les certificats et profileMapper.js.
3. Désinstaller l’AD/LDAP Connector.
4. Télécharger la mise à jour.
5. Installer l’AD/LDAP Connector.
6. Restaurer la configuration existante, les certificats et profileMapper.js.
7. Démarrer le service Windows.
8. Pour exécuter le script de mise à jour, exécutez la commande suivante en ligne de commande :
   `@powershell -NoProfile -ExecutionPolicy Bypass -Command "iex ((new-object net.webclient).DownloadString('https://cdn.auth0.com/connector/windows/update-connector.ps1'))"`

Le script de mise à jour utilise des commandes PowerShell précises qui ne sont disponibles que dans Windows PowerShell 3.0 et les versions ultérieures. Si vous utilisez Windows 2008 ou Windows 2008 R2, vous devrez peut-être d’abord mettre à jour votre version de PowerShell.

Pour en savoir plus, consultez [Télécharger Windows Management Framework](https://www.microsoft.com/en-us/download/details.aspx?id=34595).

<div id="manual-update-windowslinux">
  ## Mise à jour manuelle (Windows/Linux)
</div>

1. Vérifiez la version installée. Survolez l’indicateur de status du Connector dans la console. L’infobulle indiquera le status actuel et la version installée.
2. Téléchargez la dernière version du [programme d’installation](https://cdn.auth0.com/connector/windows/adldap-5.0.10.msi?_ga=2.58973869.1675886805.1589821069-1123834949.1586212750).
   Pour les autres plateformes, utilisez la [version](https://github.com/auth0/ad-ldap-connector/releases/tag/v5.0.10) du GitHub repository.
3. Sauvegardez votre configuration actuelle à partir de l’Admin Console ou manuellement.

   1. Dans l’Admin Console, cliquez sur le bouton **Download** pour générer un fichier .zip contenant le fichier `config.json`, le dossier `certs` et le fichier `lib\\profileMapper.js`.
   2. Pour le faire manuellement, sauvegardez les mêmes fichiers que ceux indiqués ci-dessus à partir du dossier `%Program Files(x86)%\Auth0\AD LDAP Connector\`. Ce chemin s’applique à Windows. Sur les autres plateformes, les fichiers peuvent se trouver ailleurs, mais il s’agira des mêmes ressources.
4. Exécutez le programme d’installation et suivez les instructions.
5. Fermez la boîte de dialogue Configuration sans rien modifier.
6. Restaurez les fichiers à partir de l’Admin Console ou manuellement.

   1. Dans l’Admin Console, téléversez le fichier .zip dans l’onglet **Import / Export**.
   2. Pour le faire manuellement, copiez les fichiers que vous avez sauvegardés précédemment.
7. Redémarrez le service **Auth0 AD LDAP** à partir de la console des services.
