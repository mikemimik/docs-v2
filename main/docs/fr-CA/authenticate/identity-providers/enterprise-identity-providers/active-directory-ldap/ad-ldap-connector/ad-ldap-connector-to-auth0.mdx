---
description: Décrit comment faire pointer des instances d’AD/LDAP Connector vers de nouvelles connexions dans Auth0.
title: Faire pointer AD/LDAP Connector vers des connexions Auth0
---

Vous devrez peut-être faire pointer votre instance d’AD/LDAP Connector vers une nouvelle connexion dans Auth0, par exemple si vous avez migré vers un nouveau tenant Auth0 ou si vous avez changé le nom de la connexion Auth0. Comme vous ne pouvez pas renommer les connexions dans Auth0, vous devez créer une nouvelle connexion AD/LDAP et faire pointer vos instances Connector existantes vers celle-ci.

1. Créez la nouvelle connexion AD/LDAP dans le Dashboard Auth0 et copiez l’**URL DE TICKET** obtenue. Si vous utilisez des domaines personnalisés, vous devrez remplacer la partie `{yourDomain}` de l’**URL DE TICKET** par votre domaine personnalisé, par exemple `identity.fabrikam.com`.
2. Sur l’hôte AD/LDAP Connector, dans l’application Connector Admin, exportez les paramètres existants à partir de l’onglet **Import / Export**. Il s’agit simplement d’une précaution au cas où quelque chose, dans les étapes suivantes, écraserait accidentellement vos paramètres personnalisés. Si le Connector s’exécute sur un hôte qui ne dispose pas d’un navigateur Web permettant d’accéder au site Web de Connector Admin, faites simplement une copie de votre fichier `config.json`.
3. Sur l’hôte AD/LDAP Connector, modifiez le fichier `config.json` et remplacez la valeur de la propriété `PROVISIONING_TICKET` par l’**URL DE TICKET** que vous avez copiée à l’étape 1.
4. Si vous êtes passé d’un tenant Auth0 à un autre, supprimez du fichier `config.json` la propriété nommée `urn:auth0:{oldAuth0TenantName}`. Si elle n’est pas supprimée, le Connector continuera de fonctionner, mais ces anciennes données de configuration ne sont pas nécessaires.
5. Redémarrez le service AD/LDAP Connector (le service **Auth0 ADLDAP** dans Windows).
6. Consultez les logs du Connector (onglet **Troubleshooting** dans l’outil Connector Admin ou surveillez le fichier `logs.log`) et assurez-vous qu’une entrée récente ressemble à ceci : `2016-03-10T22:47:32.970Z - debug: [2016-03-10 22:47:32] Loading settings from ticket: {yourTicketUrl}/info`
7. Assurez-vous que la nouvelle connexion AD/LDAP dans le Dashboard Auth0 s’affiche maintenant comme connectée (le point à gauche de la nouvelle connexion est vert et non rouge).
8. Effectuez un test d’authentification à l’aide de votre nouvelle connexion et assurez-vous de voir également de l’activité dans les logs de votre Connector.
