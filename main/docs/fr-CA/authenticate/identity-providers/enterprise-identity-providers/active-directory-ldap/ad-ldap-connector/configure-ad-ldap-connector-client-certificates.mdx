Le Connector AD/LDAP permet aussi aux utilisateurs de s’authentifier au moyen d’un certificat installé sur leur ordinateur ou leur appareil.

<div id="enable-client-certificates">
  ## Activer les certificats clients
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Enterprise &gt; Active Directory/LDAP](https://manage.auth0.com/#/connections/enterprise/ad), puis sélectionnez la connexion que vous souhaitez configurer.
2. Activez l’option **Utiliser l’authentification par certificat SSL client** dans les paramètres.
3. Indiquez des plages d’adresses IP dans le champ **Plages d’adresses IP**. Seuls les utilisateurs provenant des plages d’adresses IP indiquées seront invités à s’authentifier à l’aide de certificats clients. Les utilisateurs provenant d’autres plages d’adresses IP seront invités à ouvrir une session à l’aide du formulaire de connexion par nom d’utilisateur et mot de passe.

<div id="configure-certificates">
  ## Configurer les certificats
</div>

Une fois la connexion AD/LDAP configurée dans Auth0, vous devrez configurer les certificats dans l’Connector AD/LDAP. Pour prendre en charge les certificats clients, les éléments suivants sont requis :

* Un certificat SSL pour l’URL publique, car l’interaction entre l’utilisateur final et le Connector devra se faire en HTTPS.
* Un ou plusieurs certificats d’AC.
* Un certificat client signé par l’AC pour chaque utilisateur devant s’authentifier à l’aide de certificats clients.

1. Avant de téléverser des certificats dans l’Connector AD/LDAP, convertissez les certificats X.509 au format Base64. Utilisez Base64 ou Certutil sur Windows Server. Pour en savoir plus, consultez [Base64 Decode](https://www.base64decode.org/) sur Base64decode ou [Certutil.exe](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/certutil#BKMK_encode) dans la documentation Microsoft.
2. Téléversez les certificats SSL et d’AC dans l’Connector AD/LDAP :

   <Frame>![Écran de configuration de l’authentification de l’Connector AD/LDAP avec des certificats clients](/docs/images/cdy7uua7fh8z/2zRux90ZHATIBFWyHlodue/c426e5a475ff528a467db6409980de12/connector-client-cert-config.png)</Frame>
3. Pour tester, générez une AC auto-signée et des certificats clients à l’aide de **makecert.exe** sous Windows, qui fait partie du SDK Windows :

   ```bash lines
   SET ClientCertificateName=jon
       SET RootCertificateName=FabrikamRootCA
       "C:\Program Files (x86)\Microsoft SDKs\Windows\v7.1A\Bin\makecert.exe" -sky exchange -r -n "CN=%RootCertificateName%" -pe -a sha1 -len 2048 -ss My "%RootCertificateName%.cer"
       "C:\Program Files (x86)\Microsoft SDKs\Windows\v7.1A\Bin\makecert.exe" -n "CN=%ClientCertificateName%" -pe -sky exchange -m 96 -ss My -in "%RootCertificateName%" -is my -a sha1
   ```

   Il est important que le sujet du certificat client soit au format `CN=AD_USERNAME`, par exemple `CN=jon`.

Dans une application, lorsqu’un utilisateur lance le flux de connexion à l’aide d’une connexion AD/LDAP :

```js lines
auth.signin({
       popup: true,
       connection: 'FabrikamAD',
       scope: 'openid name email'
     }, onLoginSuccess, onLoginFailed);
```

Si l&#39;adresse IP de l&#39;utilisateur se trouve dans la plage d’adresses IP configurée, on lui demandera de s&#39;authentifier à l’aide d’un certificat client :
