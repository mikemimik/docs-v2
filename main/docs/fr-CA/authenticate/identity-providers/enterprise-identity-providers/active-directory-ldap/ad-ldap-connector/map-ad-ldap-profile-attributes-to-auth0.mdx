---
description: Décrit comment mapper les attributs de profil AD/LDAP aux attributs du profil utilisateur Auth0 à l’aide du Profile Mapper dans la Connector Admin Console.
title: Mapper les attributs de profil AD/LDAP au profil utilisateur Auth0
---

Vous pouvez modifier le mappage des attributs de profil AD/LDAP vers les attributs du profil utilisateur Auth0.

1. Dans un navigateur, accédez à `http://localhost:8357` pour lancer la **Connector Admin Console** sur le serveur AD/LDAP Connector.
2. Cliquez sur **Profile Mapper** pour ouvrir un écran d’éditeur de code qui mappe les attributs d’un service d’annuaire source (représenté par la variable `raw_data`) vers une variable renvoyée pour remplir le profil utilisateur Auth0.
   La première partie de la fonction instancie une variable appelée `profile` et comprend un mappage pour la partie principale du profil utilisateur Auth0. Vous pouvez définir des attributs supplémentaires en dessous à l’aide d’une syntaxe de la forme suivante :
   `profile['department'] = raw_data['companydept'];`
   Dans cet exemple, `department` est le nom de l’attribut dans le profil utilisateur Auth0 et `companydept` est le nom de l’attribut dans le service d’annuaire source (comme AD).
3. Cliquez sur **Save** lorsque vous avez terminé de modifier la configuration LDAP.
4. Une fois que le Connector a exécuté les tests, assurez-vous qu’ils sont tous réussis.

| Test   | Description                                                                                                                       | Dépannage                                                                                                          |
| ------ | --------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Test 1 | Tente d’établir une connexion TCP au serveur LDAP et au port spécifié.                                                            | Vérifiez la connectivité réseau de base et les paramètres du pare-feu qui pourraient empêcher une telle connexion. |
| Test 2 | Tente d’effectuer une liaison LDAP sur le serveur LDAP et le port spécifié, avec le nom d’utilisateur et le mot de passe fournis. | Vérifiez la chaîne de connexion LDAP, le chemin de recherche, le nom d’utilisateur et le mot de passe.             |
| Test 3 | Tente d’effectuer une recherche LDAP dans l’annuaire pour vérifier les privilèges du nom d’utilisateur spécifié.                  | Vérifiez les privilèges du nom d’utilisateur dans l’annuaire cible.                                                |
| Test 4 | Tente d’établir une connexion au serveur Auth0.                                                                                   | Vérifiez la connectivité réseau et les paramètres du pare-feu qui pourraient empêcher une telle connexion.         |
