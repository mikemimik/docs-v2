---
description: Décrit comment mapper les attributs de profil AD/LDAP aux attributs du profil utilisateur Auth0 à l’aide du Profile Mapper dans la Connector Admin Console.
title: Mapper les attributs de profil AD/LDAP au profil utilisateur Auth0
---

Vous pouvez modifier le mapping des attributs de profil AD/LDAP vers les attributs du profil utilisateur Auth0.

1. Dans un navigateur, accédez à `http://localhost:8357` pour ouvrir la **Connector Admin Console** sur le serveur AD/LDAP Connector.
2. Cliquez sur **Profile Mapper** pour ouvrir un écran d’éditeur de code qui mappe les attributs d’un service d’annuaire source (représenté par la variable `raw_data`) vers une variable renvoyée pour remplir le profil utilisateur Auth0.
   La première partie de la fonction instancie une variable appelée `profile` et comprend un mapping pour la partie principale du profil utilisateur Auth0. Vous pouvez définir des attributs supplémentaires plus bas à l’aide d’une syntaxe de la forme suivante :
   `profile['department'] = raw_data['companydept'];`
   Dans cet exemple, `department` est le nom de l’attribut dans le profil utilisateur Auth0 et `companydept` est le nom de l’attribut dans le service d’annuaire source (comme AD).
3. Cliquez sur **Save** lorsque vous avez terminé de modifier la configuration LDAP.
4. Une fois que le Connector a exécuté les tests, assurez-vous qu’ils sont tous réussis.

| Test   | Description                                                                                                          | Dépannage                                                                                                          |
| ------ | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Test 1 | Tente d’établir une connexion TCP au serveur LDAP et au port indiqués.                                               | Vérifiez la connectivité réseau de base et les paramètres du firewall qui pourraient empêcher une telle connexion. |
| Test 2 | Tente d’effectuer une liaison LDAP sur le serveur LDAP et le port indiqués, avec le username et le Password fournis. | Vérifiez la Connection string LDAP, le chemin de recherche, le username et le Password.                            |
| Test 3 | Tente d’effectuer une recherche LDAP dans l’annuaire afin de vérifier les privilèges du username indiqué.            | Vérifiez les privilèges du username dans l’annuaire cible.                                                         |
| Test 4 | Tente d’établir une connexion au serveur Auth0.                                                                      | Vérifiez la connectivité réseau et les paramètres du firewall qui pourraient empêcher une telle connexion.         |
