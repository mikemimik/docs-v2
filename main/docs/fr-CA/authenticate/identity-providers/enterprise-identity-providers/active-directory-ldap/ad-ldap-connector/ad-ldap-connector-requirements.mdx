---
description: Énumère tous les prérequis pour installer et configurer le Connector.
title: Configuration système requise pour AD/LDAP Connector
---

En règle générale, AD/LDAP Connector doit être installé par un administrateur système ou un ingénieur d’exploitation plutôt que par un développeur, puisqu’il nécessite souvent un accès aux ressources de production. Voici une liste de contrôle des éléments à prendre en compte avant l’installation proprement dite :

<div id="host-servers">
  ## Serveurs hôtes
</div>

Le Connector peut être installé sur un serveur existant, même sur un contrôleur de domaine. Cependant, il est le plus souvent installé sur des machines virtuelles mises en place spécialement pour le Connector. Dans tous les cas, le serveur hôte doit répondre aux spécifications et configurations matérielles et logicielles suivantes :

<div id="hardware-requirements">
  ### Exigences matérielles
</div>

* **Architecture** : x86 ou x86-64
* **Cœurs du processeur** : min. 1, 2 recommandés
* **Stockage** : 500 Mo d’espace disque libre
* **Système d’exploitation** : Le Connector peut fonctionner sous Windows ou Linux. Windows est requis si l’authentification Kerberos est utilisée.
* **Mémoire vive** : min. 2 Go

<div id="windows-version">
  ### Version de Windows
</div>

Nous recommandons d’utiliser une version de Windows Server prise en charge, par exemple Windows Server 2016 ou Windows Server 2019. Le Connector peut également s’exécuter sous Windows Server 2012 R2.

<div id="time-synchronization">
  ### Synchronisation de l’heure
</div>

Il est très important que l’horloge du serveur hôte du Connector soit synchronisée automatiquement avec un serveur NTP. Sinon, le Connector ne pourra pas démarrer et signalera une erreur de désynchronisation de l’horloge.

<div id="outbound-connectivity">
  ## Connectivité réseau sortante
</div>

Le serveur hôte a besoin d’une connectivité réseau sortante vers les services suivants :

<div id="auth0">
  ### Auth0
</div>

Le Connector doit être installé sur un serveur disposant d’une connectivité sortante vers le service Auth0 à l’adresse suivante : `https://{yourDomain}` sur le port 443.

Le Connector peut être installé et configuré derrière un serveur proxy, mais nous ne le recommandons pas. Activez un proxy à l’aide de la variable d’environnement ou de configuration `HTTP_PROXY`.

<div id="ldap">
  ### LDAP
</div>

Le Connector doit être installé sur un serveur qui a accès au serveur LDAP sur le port 389 pour ldap ou 636 pour ldaps. Avant d’installer le Connector, vous devez connaître la chaîne de connexion LDAP ainsi que le DN de base requis pour vous connecter à votre annuaire LDAP.

<div id="inbound-connectivity">
  ## Connectivité entrante
</div>

Vous n’avez pas besoin d’activer la connectivité entrante vers le Connector, sauf si l’authentification Kerberos ou l’authentification par certificat est activée. Dans ces cas, les serveurs sur lesquels le Connector est installé doivent être accessibles depuis le navigateur de vos utilisateurs par le port 443. Si plus d’une instance du Connector est installée, vous devriez utiliser un répartiteur de charge pour acheminer le trafic vers l’un ou l’autre des connecteurs.

Pour en savoir plus, consultez [Configurer l’authentification du AD/LDAP Connector avec Kerberos](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-with-kerberos) ou [Configurer l’authentification du AD/LDAP Connector avec des certificats client](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-client-certificates).

<div id="service-account">
  ## Compte de service
</div>

Le Connector sera exécuté à l’aide d’un compte de service, qui doit correspondre à un utilisateur du domaine ayant au minimum un accès en lecture à l’annuaire. Vous aurez besoin du nom d’utilisateur et du mot de passe de ce compte lors de l’installation.

<div id="one-connector-per-auth0-tenant">
  ## Un Connector par tenant Auth0
</div>

Si vous créez plusieurs tenants Auth0, par exemple pour isoler les environnements de développement et de production, vous devrez configurer une connexion AD/LDAP dans l’<Tooltip tip="Auth0 Dashboard : l’interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ainsi qu’un AD/LDAP Connector pour chaque tenant Auth0 qui a besoin de ce type d’authentification. Un Connector est associé à une connexion précise dans un tenant Auth0.

Il est possible d’avoir plusieurs Connectors dans un même tenant Auth0 si vous avez plusieurs annuaires AD/LDAP auprès desquels les utilisateurs s’authentifieront. Par exemple, pour prendre en charge différents départements ou clients, chacun ayant son propre annuaire. De plus, plusieurs Connectors peuvent pointer vers le même annuaire AD ou LDAP, mais un Connector ne peut être utilisé que par une seule Auth0 Connection dans un seul tenant Auth0.

<div id="high-availability">
  ## Haute disponibilité
</div>

Le Connector peut être installé sur plusieurs serveurs hôtes afin d’assurer la haute disponibilité et la redondance (la plupart des organisations en déploient deux) si l’un d’eux devient indisponible. Chaque serveur doit répondre aux mêmes exigences indiquées ci-dessus. Aucun répartiteur de charge n’est nécessaire, puisque cette fonction est assurée par le serveur Auth0 lui-même, sauf si vous activez Kerberos ou l’authentification par certificat client.

Pour en savoir plus, consultez [Déployer des connecteurs AD/LDAP pour les environnements de haute disponibilité.](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-high-availability)
