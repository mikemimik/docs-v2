---
description: Répertorie tous les prérequis pour l’installation et la configuration du connecteur.
title: Configuration système requise pour AD/LDAP Connector
---

En général, l’installation d’AD/LDAP Connector doit être effectuée par un administrateur système ou un ingénieur d’exploitation plutôt que par un développeur, puisqu’elle exige souvent l’accès à des ressources de production. Voici une liste de vérification des éléments à considérer avant l’installation proprement dite :

<div id="host-servers">
  ## Serveurs hôtes
</div>

Le Connector peut être installé sur un serveur existant, même sur un contrôleur de domaine. Cependant, on l’installe le plus souvent sur des machines virtuelles mises en place uniquement pour le Connector. Dans tous les cas, le serveur hôte doit respecter les spécifications et configurations matérielles et logicielles suivantes :

<div id="hardware-requirements">
  ### Configuration matérielle requise
</div>

* **Architecture**: x86 ou x86-64
* **Cœurs de processeur**: min. 1, 2 recommandés
* **Espace disque**: 500 MB d’espace libre sur le disque
* **Système d’exploitation**: Le Connector peut s’exécuter sous Windows ou Linux. Windows est nécessaire si l’authentification Kerberos est utilisée.
* **RAM**: min. 2 Go

<div id="windows-version">
  ### Version de Windows
</div>

Nous recommandons d’utiliser une version prise en charge de Windows Server, comme Windows Server 2016 ou Windows Server 2019. Le Connector peut aussi fonctionner sous Windows Server 2012 R2.

<div id="time-synchronization">
  ### Synchronisation de l’heure
</div>

Il est très important que l’horloge du serveur hôte du Connector soit synchronisée automatiquement avec un serveur NTP. Sinon, le Connector ne démarrera pas et signalera une erreur de désynchronisation de l’horloge.

<div id="outbound-connectivity">
  ## Connectivité sortante
</div>

Le serveur hôte doit pouvoir se connecter aux services suivants :

<div id="auth0">
  ### Auth0
</div>

Le Connector doit être installé sur un serveur ayant une connectivité sortante vers le service Auth0 à l’adresse suivante : `https://{yourDomain}` sur le port 443.

Le Connector peut être installé et configuré derrière un serveur proxy, mais nous ne le recommandons pas. Activez un proxy à l’aide de la variable d’environnement ou de configuration `HTTP_PROXY`.

<div id="ldap">
  ### LDAP
</div>

Le Connector doit être installé sur un serveur ayant accès au serveur LDAP sur le port 389 pour ldap ou 636 pour ldaps. Avant d’installer le Connector, vous devez connaître la chaîne de connexion LDAP et le DN de base nécessaires pour vous connecter à votre annuaire LDAP.

<div id="inbound-connectivity">
  ## Connectivité entrante
</div>

Vous n’avez pas besoin d’activer la connectivité entrante vers le Connector, sauf si Kerberos ou l’authentification par certificat sont activés. Dans ces cas, le ou les serveurs sur lesquels le Connector est installé doivent être accessibles depuis les navigateurs de vos utilisateurs sur le port 443. Si plus d’une instance du Connector est installée, vous devriez utiliser un répartiteur de charge pour acheminer le trafic vers l’un ou l’autre des connecteurs.

Pour en savoir plus, consultez [Configurer l’authentification d’AD/LDAP Connector avec Kerberos](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-with-kerberos) ou [Configurer l’authentification d’AD/LDAP Connector avec des certificats clients](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-client-certificates).

<div id="service-account">
  ## Compte de service
</div>

Le Connector s’exécute à l’aide d’un compte de service, qui doit être un utilisateur du domaine et disposer, au minimum, d’un accès en lecture à l’annuaire. Vous aurez besoin du nom d’utilisateur et du mot de passe de ce compte au moment de l’installation.

<div id="one-connector-per-auth0-tenant">
  ## Un Connector par tenant Auth0
</div>

Si vous configurez plusieurs tenants Auth0, par exemple pour isoler les environnements de développement et de production, vous devrez mettre en place une connexion AD/LDAP dans le <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, ainsi qu’un AD/LDAP Connector pour chaque tenant Auth0 qui a besoin de cette forme d’authentification. Un Connector est associé à une connexion précise dans un tenant Auth0.

Il est possible d’avoir plusieurs Connectors dans un même tenant Auth0 si vous disposez de plusieurs annuaires AD/LDAP auprès desquels les utilisateurs s’authentifieront. Par exemple, pour prendre en charge différents départements ou clients ayant chacun leur propre annuaire. De plus, plusieurs Connectors peuvent pointer vers le même annuaire AD ou LDAP, mais un Connector ne peut être utilisé que par une seule connexion Auth0 au sein d’un même tenant Auth0.

<div id="high-availability">
  ## Haute disponibilité
</div>

Le Connector peut être installé sur plusieurs serveurs hôtes afin d’assurer la haute disponibilité et la redondance (la plupart des organisations en déploient deux), si jamais l’un des serveurs devient indisponible. Chaque serveur doit respecter les mêmes exigences que celles indiquées ci-dessus. Aucun répartiteur de charge n’est nécessaire, puisque cette fonction est assurée par le serveur Auth0 lui-même, à moins d’activer l’authentification Kerberos ou l’authentification par certificat client.

Pour en savoir plus, consultez [Déployer des Connectors AD/LDAP pour les environnements à haute disponibilité.](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-high-availability)
