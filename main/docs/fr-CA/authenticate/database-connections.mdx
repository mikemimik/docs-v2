Auth0 fournit des connexions de base de données pour authentifier les utilisateurs à l’aide d’un identifiant (courriel, nom d’utilisateur ou numéro de téléphone) et d’un mot de passe ou de [clés d’accès](/docs/fr-CA/authenticate/database-connections/passkeys/configure-passkey-policy). Ces informations d’authentification sont stockées de manière sécurisée dans le répertoire d’utilisateurs d’Auth0 ou dans votre propre base de données.

Vous pouvez créer une nouvelle connexion de base de données et gérer les connexions existantes dans [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database).

<div id="database-options">
  ## Options de la connexion Database
</div>

<div id="use-auth0-user-store">
  ### Utiliser le répertoire d’utilisateurs d’Auth0
</div>

Par défaut, Auth0 fournit l’infrastructure de base de données nécessaire pour stocker vos utilisateurs. Ce scénario offre les meilleures performances pour le processus d’authentification, puisque toutes les données sont stockées dans Auth0.

La base de données hébergée par Auth0 est hautement sécurisée. Les mots de passe ne sont jamais stockés ni consignés en texte clair; ils sont hachés avec **bcrypt**. Il est également possible d’appliquer différents niveaux d’exigences de sécurité pour les mots de passe. Pour en savoir plus, consultez [Force du mot de passe dans les connexions de base de données Auth0](/docs/fr-CA/authenticate/database-connections/password-strength).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour les connexions de base de données, Auth0 limite le nombre de tentatives de connexion répétées par utilisateur et par adresse IP. Pour en savoir plus, consultez [Limites de connexion à la base de données](/docs/fr-CA/troubleshoot/customer-support/operational-policies/rate-limit-policy#database-login-limits).
</Callout>

Pour utiliser l’infrastructure de base de données d’Auth0 comme répertoire d’utilisateurs, plusieurs options s’offrent à vous pour migrer vos utilisateurs vers Auth0. La fonctionnalité de [migration automatique](/docs/fr-CA/manage-users/user-migration/configure-automatic-migration-from-your-database) ajoute vos utilisateurs à la base de données d’Auth0 un par un, à mesure qu’ils se connectent, et vous évite de demander à tous vos utilisateurs de réinitialiser leur mot de passe en même temps. Vous pouvez aussi utiliser la Management API pour créer une tâche afin [d’importer des utilisateurs](/docs/fr-CA/manage-users/user-migration/bulk-user-imports). Pour en savoir plus, consultez [Scénarios de migration des utilisateurs](/docs/fr-CA/manage-users/user-migration/user-migration-scenarios) pour voir des exemples de migration.

<div id="use-your-own-database">
  ### Utilisez votre propre base de données
</div>

Si vous disposez déjà d’un répertoire d’utilisateurs ou si vous souhaitez stocker les informations d’authentification des utilisateurs sur votre propre serveur, Auth0 vous permet de vous connecter à une [base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db) ou à un référentiel, et de l’utiliser comme fournisseur d’identité.

<Frame>![](/docs/images/cdy7uua7fh8z/3SNKwx0KzkUwbodqUQbtvW/df720f8871b56dd7ec399a24aa621c36/Authentication>Database>use-my-own-database.png)</Frame>

Dans ce scénario, vous fournissez le script Login qui sert à authentifier l’utilisateur et qui s’exécute chaque fois qu’un utilisateur tente de se connecter. Au besoin, vous pouvez [créer des scripts](/docs/fr-CA/authenticate/database-connections/custom-db/templates) pour l’inscription, la vérification du courriel, la réinitialisation du mot de passe et la suppression d’utilisateurs.

Les scripts sont du code Node.js. Auth0 fournit des [modèles](/docs/fr-CA/authenticate/database-connections/custom-db/templates) pour les bases de données les plus courantes, comme ASP.NET Membership Provider, MongoDB, MySQL, PostgreSQL, SQL Server et Windows Azure SQL Database, ainsi que pour un service Web accessible par authentification Basic. En pratique, vous pouvez vous connecter à presque n’importe quel type de base de données ou de service Web au moyen d’un script personnalisé.
