<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La MFA n’est pas compatible avec les transactions d’échange de jetons personnalisé lorsqu’un acteur est défini au moyen de `api.authentication.setActor()`. Si la MFA est requise (par une politique ou une Action post-connexion) et que l’Action d’échange de jetons personnalisé définit un acteur, la transaction échoue avec une erreur `400` : `MFA is not supported using actor_token with the requested token exchange profile.`
</Callout>

Pour ajouter une protection contre le vol de jetons ou d’autres risques de sécurité, vous pouvez ajouter l’authentification multifacteur à une requête d’échange de jetons personnalisé à l’aide de l’une des méthodes suivantes :

* [Activer et configurer votre politique MFA ](/docs/fr-CA/secure/multi-factor-authentication/enable-mfa) pour votre tenant
* Utiliser `api.multifactor.enable()` dans une Action post-connexion

Lorsque vous ajoutez la MFA, le serveur d’autorisation Auth0 rejette l’appel initial d’échange de jetons personnalisé vers le point de terminaison `/token` avec une erreur `mfa_required`. Vous pouvez utiliser cette erreur avec l’[API MFA Auth0](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api) pour lancer une demande de vérification d’un facteur et le valider afin d’obtenir les jetons Auth0 d’accès, d’ID et d’actualisation demandés.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’échange de jetons personnalisé ne prend pas en charge `api.authentication.challengeWith()` ni `api.authentication.enrollWith()`. Si vous utilisez ces méthodes avec votre Action post-connexion, la transaction échouera avec une erreur non récupérable. Assurez-vous de ne pas utiliser ces méthodes lorsque `event.transaction.protocol==oauth2-token-exchange`, selon la valeur de `subject_token_type`.
</Callout>

Lorsqu’une transaction d’échange de jetons personnalisé est associée à une organisation, `api.multifactor.enable()` et les politiques MFA ne sont pas non plus prises en charge.

Pour en savoir plus sur l’utilisation de l’autorisation MFA, consultez [S’authentifier à l’aide du flux Resource Owner Password avec MFA](/docs/fr-CA/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa), puisque l’échange de jetons personnalisé suit le même modèle. Vous pouvez aussi consulter un exemple détaillé dans [Cas d’utilisation : effectuer une MFA pendant un échange de jetons personnalisé](/docs/fr-CA/authenticate/custom-token-exchange/cte-example-use-cases#use-case%3A-perform-mfa-during-custom-token-exchange).
