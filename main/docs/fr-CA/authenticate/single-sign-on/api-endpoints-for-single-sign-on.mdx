title: Points de terminaison d’API pour l’authentification unique
---

Lorsque vous mettez en œuvre l’<Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, ouvre automatiquement sa session dans les autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-on">authentification unique</Tooltip> (SS0) et en configurez les paramètres, les points de terminaison d’API suivants vous seront utiles.

<div id="authentication-api-endpoints">
  ## Points de terminaison de l’Authentication API
</div>

<div id="post-logincallback">
  ### POST /login/callback
</div>

Pour les flux SSO lancés par un <Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP), le point de terminaison [POST /login/callback](https://auth0.com/docs/api/authentication#idp-initiated-single-sign-on-sso-flow) peut accepter une requête d’authentification unique <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des informations d’authentification sans mot de passe." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SAML">SAML</Tooltip> provenant du fournisseur d’identité.

<div id="management-api">
  ## Management API
</div>

<Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> propose plusieurs points de terminaison pour vous aider à gérer le SSO.

Pour appeler l’un des points de terminaison de la Management API, vous devrez [obtenir un jeton d’accès](https://auth0.com/docs/api/management/v2/tokens).

<div id="clients">
  ### Clients
</div>

Toutes vos applications clientes/intégrations SSO comprennent des renseignements pertinents pour votre mise en œuvre du SSO. Vous pouvez récupérer ou modifier ces renseignements en appelant l’un des points de terminaison Clients.

Voici les paramètres pertinents pour chaque client :

* `sso` : indicateur applicable uniquement aux clients créés par l’entremise d’intégrations SSO. Si `true`, Auth0 gère le SSO; si `false`, l’IdP gère le SSO.
* `sso_disabled` : indicateur servant à activer ou désactiver le SSO. Si `true`, le SSO est désactivé; si `false`, le SSO est activé. Cette option peut être définie uniquement au moyen de la Management API.
* `app_type` : type d’application. Si le client a été créé à l’aide de l’une des intégrations SSO intégrées d’Auth0, le nom de l’intégration SSO sera indiqué (par exemple, `box` ou `concur` au lieu de `native` ou `spa`)

<div id="get-all-clients">
  #### Obtenir tous les clients
</div>

Le point de terminaison [GET /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/get_clients) permet d’obtenir des renseignements sur les applications clientes que vous avez configurées pour votre tenant.

<div id="create-a-client">
  #### Créer un client
</div>

Le point de terminaison [POST /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/post_clients) permet de créer une nouvelle application cliente.

<div id="get-a-client">
  #### Récupérer un client
</div>

Le point de terminaison [GET /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/get_clients_by_id) permet d’obtenir des renseignements sur un client précis que vous avez configuré dans votre tenant.

<div id="update-a-client">
  #### Mettre à jour un client
</div>

Le point de terminaison [PATCH /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id) permet de mettre à jour un client donné, y compris ses paramètres liés au SSO.

<div id="tenants">
  ### Tenants
</div>

Auth0 vous permet de contrôler les paramètres suivants au niveau du tenant, qui peuvent avoir une incidence sur votre mise en œuvre du SSO :

* `session_lifetime` : durée pendant laquelle la session Auth0 de l’utilisateur reste valide.
* `idle_session_lifetime` : durée pouvant s’écouler avant que l’utilisateur doive se reconnecter en raison de son inactivité.

<div id="get-tenant-settings">
  ### Obtenir les paramètres du tenant
</div>

Le point de terminaison [GET /api/v2/tenants/settings](https://auth0.com/docs/api/management/v2#!/Tenants/get_settings) permet de récupérer les paramètres de votre tenant.

<div id="update-tenant-settings">
  ### Mettre à jour les paramètres du tenant
</div>

Le point de terminaison [POST /api/v2/connections](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings) vous permet de mettre à jour les paramètres de votre tenant.
