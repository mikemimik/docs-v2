---

Lors de la mise en œuvre de l’<Tooltip tip="Authentification unique (SSO) : service qui, une fois qu’un utilisateur ouvre une session dans une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-on">authentification unique</Tooltip> (SS0) et de la configuration de ses paramètres, les points de terminaison d’API suivants vous seront utiles.

<div id="authentication-api-endpoints">
  ## Points de terminaison de l’Authentication API
</div>

<div id="post-logincallback">
  ### POST /login/callback
</div>

Pour les flux SSO lancés par un <Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">Identity Provider</Tooltip> (IdP), le point de terminaison [POST /login/callback](https://auth0.com/docs/api/authentication#idp-initiated-single-sign-on-sso-flow) peut accepter une requête de connexion <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SAML">SAML</Tooltip> provenant du fournisseur d’identité.

<div id="management-api">
  ## Management API
</div>

La <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> comporte plusieurs points de terminaison conçus pour vous aider à gérer le SSO.

Pour appeler l’un des points de terminaison de la Management API, vous devrez [obtenir un jeton d’accès](https://auth0.com/docs/api/management/v2/tokens).

<div id="clients">
  ### Clients
</div>

Toutes vos applications clientes/intégrations SSO comprennent des renseignements pertinents pour votre implémentation SSO. Vous pouvez récupérer ou modifier ces renseignements en appelant l’un des points de terminaison Clients.

Les paramètres pertinents pour chaque client comprennent :

* `sso` : indicateur applicable uniquement aux clients créés au moyen d’intégrations SSO. Si `true`, Auth0 gère le SSO; si `false`, c’est l’IdP qui gère le SSO.
* `sso_disabled` : indicateur servant à activer ou désactiver le SSO. Si `true`, le SSO est désactivé; si `false`, le SSO est activé. Cette option ne peut être définie qu’au moyen de la Management API.
* `app_type` : type d’application. Si le client a été créé à l’aide de l’une des intégrations SSO intégrées d’Auth0, le nom de l’intégration SSO serait indiqué (par ex., `box` ou `concur` au lieu de `native` ou `spa`)

<div id="get-all-clients">
  #### Obtenir tous les clients
</div>

Le point de terminaison [GET /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/get_clients) permet d&#39;obtenir des renseignements sur les applications clientes que vous avez configurées dans votre tenant.

<div id="create-a-client">
  #### Créer un client
</div>

Le point de terminaison [POST /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/post_clients) permet de créer une nouvelle application cliente.

<div id="get-a-client">
  #### Obtenir un client
</div>

Le point de terminaison [GET /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/get_clients_by_id) permet d’obtenir des renseignements sur un client précis que vous avez configuré pour votre tenant.

<div id="update-a-client">
  #### Mettre à jour un client
</div>

Le point de terminaison [PATCH /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id) permet de mettre à jour un client donné, y compris ses paramètres de SSO.

<div id="tenants">
  ### Tenants
</div>

Auth0 vous permet de contrôler les paramètres suivants au niveau du tenant, qui peuvent avoir une incidence sur votre implémentation de l’SSO :

* `session_lifetime` : durée pendant laquelle la session Auth0 de l’utilisateur reste valide.
* `idle_session_lifetime` : durée pouvant s’écouler avant que l’utilisateur doive se reconnecter en raison de son inactivité.

<div id="get-tenant-settings">
  ### Obtenir les paramètres du tenant
</div>

Le point de terminaison [GET /api/v2/tenants/settings](https://auth0.com/docs/api/management/v2#!/Tenants/get_settings) permet d’obtenir les paramètres de votre tenant.

<div id="update-tenant-settings">
  ### Mettre à jour les paramètres du tenant
</div>

Le point de terminaison [POST /api/v2/connections](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings) vous permet de mettre à jour les paramètres de votre tenant.
