Utilisez la configuration <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé qui permet à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=SAML">SAML</Tooltip> suivante pour Oracle Eloqua Marketing Cloud.

<div id="configure-saml-sso-in-auth0">
  ## Configurer SAML SSO dans Auth0
</div>

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis créez une nouvelle application ou cliquez sur le nom de l’application que vous souhaitez mettre à jour.
2. Accédez à l’onglet **Addons** et activez la bascule **SAML2 Web App**.
3. Dans l’onglet **Settings**, définissez **Application Callback URL** sur : `https://login.eloqua.com/auth/saml2/acs`.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. Collez le code suivant dans la zone de texte **Settings**, puis cliquez sur **Debug**.

   ```json lines
   {
       "audience": "<Service Provider Entity Url from within Eloqua>",
        "recipient": "https://login.eloqua.com/auth/saml2/acs",
          "mappings": {
       "user_id": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier",
       "email": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
       "name": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name",
          },
        "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
        "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
       ],
        "destination": "https://login.eloqua.com/auth/saml2/acs"
         }
   ```

   Notez que `audience` est défini sur `http://foo`. Cette valeur sera modifiée une fois la configuration du fournisseur d’identité terminée dans Eloqua.
5. Faites défiler jusqu’au bas de l’onglet et cliquez sur **Enable**.
6. Dans l’onglet **Usage**, repérez **Identity Provider Metadata** et cliquez sur **Download** pour télécharger le fichier de métadonnées. Vous en aurez besoin lorsque vous configurerez Auth0 comme fournisseur d’identité dans Eloqua.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Usage Tab](/docs/images/cdy7uua7fh8z/3T90BDpyTXFUWDp1JkncBU/86bdba6bed8130b34b42b25c9bf2d232/dashboard-applications-applications-addons-saml2-web-app-usage.png)</Frame>

<div id="configure-saml-sso-in-oracle-eloqua">
  ## Configurer SAML SSO dans Oracle Eloqua
</div>

1. Ouvrez une session en tant qu’**Admin** dans Eloqua, puis cliquez sur **Settings &gt; Users &gt; Single Sign-On &gt; Identity Provider Settings**.
2. Cliquez sur **Upload Identity Provider from Metadata**. Téléversez les **Identity Provider Metadata** que vous avez téléchargées depuis l’onglet **Usage** dans l’Auth0 Dashboard.
3. Parcourez vos fichiers pour trouver le fichier de métadonnées téléchargé.
4. Modifiez le fournisseur d’identité nouvellement ajouté et suivez les étapes ci-dessous :

   1. Définissez le **User Identity Mapping** sur `Assertion contains the Email Address from the User object`, puis cliquez sur **Save**.
   2. Dans les **Identity Provider Details**, copiez l’**URL d’entité du fournisseur de services**.
   3. Dans les paramètres IdP d’ELOQUA, s’il s’agit de l’IdP par défaut, marquez cet IdP comme valeur par défaut.

<div id="update-audience-restriction-in-auth0">
  ## Mettre à jour la restriction d’audience dans Auth0
</div>

Utilisez la **URL d’entité du fournisseur de services** copiée dans les paramètres du <Tooltip tip="Identity Provider (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> dans Eloqua pour définir la restriction `audience` dans le <Tooltip tip="Auth0 Dashboard : Principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur le nom de l’application que vous voulez mettre à jour.
2. Accédez à l’onglet **Addons** et activez la bascule **SAML2 Web App**.
3. Dans l’onglet **Settings**, remplacez la valeur de `audience` par celle que vous avez copiée depuis Eloqua.

<div id="test-configuration">
  ## Configuration de test
</div>

**La connexion à Eloqua avec Auth0** devrait maintenant être activée. Vous pouvez vous connecter à Eloqua au moyen d’une connexion initiée par l’IdP ou par le SP.

* Pour la connexion initiée par le SP, accédez à &lt;https://login.eloqua.com/auth/saml2&gt;, entrez le nom de votre entreprise et lancez le processus de connexion SAML avec Auth0.
* Pour la connexion initiée par l’IdP, utilisez l’**URL de connexion du fournisseur d’identité** définie ici : **Dashboard &gt; Applications &gt; Applications** dans l’onglet **Addons**, **SAML 2 Web App**, onglet **Usage**.
