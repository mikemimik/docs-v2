description: Découvrez comment configurer Auth0 en tant que fournisseur d'identité à l'aide du module complémentaire SAML2 Web App pour Workday.
title: Configurer Workday en tant que fournisseur de services SAML
---

Utilisez la configuration <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des informations d’authentification sans mot de passe." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SAML">SAML</Tooltip> suivante pour Workday.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis créez une application ou cliquez sur le nom d’une application à mettre à jour.
2. Accédez à l’onglet **Addons** et activez la bascule **SAML2 Web App**.
3. Dans l’onglet **Settings**, définissez l’**Application Callback URL** comme suit : `https://impl.workday.com/<tenant>/login-saml.htmld`.

   <Frame>![Dashboard Applications Applications onglet Addons SAML2 Web App onglet Settings](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. Collez le code suivant dans la zone de texte **Settings**, puis cliquez sur **Debug**.

   ```json lines
   {
         "audience": "http://www.workday.com",
         "recipient": "https://www.myworkday.com/<tenant>/login-saml.htmld",
         "mappings": {
         },
         "createUpnClaim":       false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities":        false,
         "signResponse":         true,
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:X509SubjectName",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
         ],
         "authnContextClassRef": "urn:oasis:names:tc:SAML:2.0:ac:classes:X509",
       }
   ```

   Modifiez le sous-domaine `impl` selon le centre de données Workday que vous utilisez.
5. Faites défiler jusqu’au bas de la page et cliquez sur **Enable**.
6. Dans l’onglet **Usage**, repérez **Identity Provider Metadata**, puis cliquez sur **Download** pour télécharger le fichier de métadonnées. Vous en aurez besoin lorsque vous configurerez Auth0 en tant que fournisseur d&#39;identité.
