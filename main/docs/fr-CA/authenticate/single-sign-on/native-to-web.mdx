La fonctionnalité Native to Web <Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO) d’Auth0 offre une expérience utilisateur fluide en assurant le passage des utilisateurs authentifiés de votre [application native](/docs/fr-CA/authenticate/login/native-login) vers votre application web.

Si votre application web s’appuie sur des WebViews intégrées ou des navigateurs externes pour offrir des fonctionnalités étendues ou avancées, vos utilisateurs peuvent passer d’un environnement natif à un environnement web dans le cadre de la même session authentifiée.

En liant les jetons de transfert de session à l’appareil concerné au moyen de l’adresse IP ou de l’ASN, la continuité et la sécurité de la session sont préservées, ce qui garantit que le contexte d’authentification reste sécurisé tout au long de la transition.

<div id="how-it-works">
  ## Fonctionnement
</div>

<Frame>![Flux de travail de Native to Web SSO](/docs/images/cdy7uua7fh8z/4gFE2RP5ZCEcukiNF3Gpae/f957034f63c36d5a11f6a64d988c00e9/sequence-diagram.png)</Frame>

1. Un utilisateur se connecte à votre application native.
2. Auth0 authentifie l’utilisateur et renvoie un jeton d’accès, un jeton d’actualisation et un jeton d’identification.
3. Votre application native appelle le point de terminaison [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow-with-pkce/get-token-pkce) d’Auth0 pour échanger un jeton d’actualisation contre un jeton de transfert de session. Le jeton de transfert de session est associé à une adresse IP ou à un ASN précis.
4. Auth0 renvoie le jeton de transfert de session pour l’authentification auprès d’une application web.
5. Auth0 autorise le jeton de transfert de session au moyen d’un cookie ou comme paramètre d’URL, puis renvoie un [code d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow).
6. L’application web échange le code d’autorisation contre des jetons d’accès ou d’actualisation au point de terminaison [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow/get-token).
7. L’application web initialise une session pour l’utilisateur.

Découvrez comment [Configurer et implémenter Native to Web SSO](/docs/fr-CA/authenticate/single-sign-on/native-to-web/configure-implement-native-to-web)

<div id="limitations">
  ## Limites
</div>

* Une fois que Native to Web SSO est activé dans une application, le paramètre `session_transfer_token` fonctionne uniquement avec Native to Web SSO.
* <Tooltip tip="Refresh Token : Jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se connecter à nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Refresh+Tokens">Les jetons d’actualisation</Tooltip> issus d’une transaction antérieure de jeton de transfert de session ne génèrent pas de nouveaux jetons de transfert de session.
