description: Découvrez comment le pipeline conforme à OIDC influence votre utilisation de l’authentification unique (SSO).
title: Authentification unique avec OIDC
---

Dans le contexte du pipeline d’authentification conforme à OIDC, l’authentification unique (<Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip>) doit se faire au niveau du <Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=authorization+server">serveur d’autorisation</Tooltip> (c.-à-d. Auth0) plutôt qu’au niveau de l’application, ce qui signifie que vous devez utiliser <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> et rediriger les utilisateurs vers la page de connexion. Pour en savoir plus, consultez [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) et [Single Sign-On](/docs/fr-CA/authenticate/single-sign-on).

De façon générale, lors de l’utilisation de l’authentification unique :

1. Si l’utilisateur n’est pas connecté localement, vous devez le rediriger vers votre page de connexion Auth0 (`/authorize`) pour l’authentifier au moyen d’un flux basé sur une redirection, comme le [flux de code d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow) ou l’[Implicit Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post), selon le type d’application.
2. Si l’utilisateur est déjà connecté par SSO, Auth0 l’authentifiera immédiatement sans qu’il ait à saisir de nouveau ses identifiants.

Pour déterminer si les utilisateurs sont connectés par SSO, utilisez l’authentification silencieuse, qui réauthentifie l’utilisateur s’il est déjà connecté ou renvoie une erreur s’il doit s’authentifier. Dans le pipeline d’authentification existant, cela pouvait se faire à l’aide du point de terminaison `/ssodata`, qui est désuet dans le pipeline conforme à OIDC. Pour en savoir plus sur la façon de configurer l’authentification silencieuse, consultez [Configure Silent Authentication](/docs/fr-CA/authenticate/login/configure-silent-authentication).
