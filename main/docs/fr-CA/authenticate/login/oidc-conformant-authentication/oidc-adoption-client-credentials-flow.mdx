---
description: Découvrez comment le pipeline conforme à OIDC a une incidence sur le Client Credentials Flow.
title: Client Credentials Flow avec OIDC
---

Le pipeline conforme à OIDC permet d’utiliser le [Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow), qui permet aux applications de s’authentifier en leur propre nom (plutôt qu’au nom d’un utilisateur) afin d’obtenir par programmation, et en toute sécurité, un accès à une API.

Ce mécanisme n’existe pas dans le pipeline Legacy; on utilise plutôt le [flux avec mot de passe du propriétaire de la ressource](/docs/fr-CA/get-started/authentication-and-authorization-flow/resource-owner-password-flow) pour le simuler en créant un **utilisateur de service**. Nous déconseillons fortement cette approche et recommandons plutôt d’utiliser le Client Credentials Flow, qui permet de définir des permissions fines pour chaque application d’API.

Pour savoir comment exécuter un Client Credentials Flow, consultez [Call API Using the Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).
