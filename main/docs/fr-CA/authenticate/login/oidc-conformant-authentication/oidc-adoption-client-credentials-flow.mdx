---
description: Découvrez comment le pipeline conforme à OIDC affecte le flux Client Credentials.
title: Flux Client Credentials avec OIDC
---

Le pipeline conforme à OIDC permet d’utiliser le [flux Client Credentials](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow), qui permet aux applications de s’authentifier en leur propre nom (plutôt qu’au nom d’un utilisateur) afin d’obtenir, de façon programmatique et sécurisée, l’accès à une API.

Cet échange n’existe pas dans le pipeline hérité; on utilise plutôt le [flux de mot de passe du propriétaire de la ressource](/docs/fr-CA/get-started/authentication-and-authorization-flow/resource-owner-password-flow) pour le simuler en créant un **utilisateur de service**. Nous déconseillons fortement cette approche au profit du flux Client Credentials, qui permet de définir des permissions granulaires pour chaque application.

Pour savoir comment exécuter un flux Client Credentials, consultez [Call API Using the Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).
