---
description: Découvrez comment le pipeline conforme à OIDC influe sur votre utilisation de la délégation.
title: Délégation avec OIDC
---

<Warning>
  Par défaut, la délégation est désactivée pour les tenants qui n’utilisaient aucun module complémentaire en date du 8 juin 2017. Les tenants existants qui utilisent actuellement un module complémentaire nécessitant la délégation peuvent continuer à utiliser cette fonctionnalité. Si la fonctionnalité de délégation devait être modifiée ou retirée du service, les clients qui l’utilisent actuellement en seraient avisés à l’avance et disposeraient d’un délai suffisant pour migrer.
</Warning>

Traditionnellement, la délégation sert à :

* Échanger un <Tooltip tip="ID Token : Information d’identification destinée au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+token">ID token</Tooltip> émis pour une application contre un nouveau jeton émis pour une autre application.
* Obtenir un nouvel ID token à l’aide d’un [jeton d’actualisation](/docs/fr-CA/secure/tokens/refresh-tokens).
* Échanger un ID token contre un jeton d’API tiers (p. ex., Firebase, AWS).

Comme le pipeline conforme à OIDC exige que les ID tokens ne soient plus utilisés pour sécuriser les API et que les <Tooltip tip="Jeton d’actualisation : Jeton utilisé pour obtenir un nouvel Access Token sans obliger les utilisateurs à se connecter de nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> soient utilisés uniquement au point de terminaison [`/oauth/token`](https://auth0.com/docs/api/authentication#refresh-token), le point de terminaison [`/delegation`](https://auth0.com/docs/api/authentication#delegation) est déprécié.

Les applications conformes à OIDC ne peuvent être ni la source ni la cible des requêtes de délégation.

<div id="third-party-apis">
  ## API tierces
</div>

Comme il n’existe aucun mécanisme conforme à OIDC pour obtenir des tokens d’API tierces (p. ex. Firebase, AWS), délégation peut toujours être utilisée pour obtenir des tokens d’API tierces.
