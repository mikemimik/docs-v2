title: Expérience Classic Login
---

Classic Login est une expérience de connexion hébergée par Auth0 qui repose sur JavaScript pour la personnalisation. Implémenter Classic Login est moins complexe que d’intégrer directement le processus d’authentification à votre application, et cela peut aider à éviter les risques liés à l’authentification cross-origin.

Classic Login s’appuie sur les bibliothèques JavaScript d’Auth0 ([Lock.js](/docs/fr-CA/libraries/lock), [auth0.js](/docs/fr-CA/libraries/auth0js), <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> Widget, et Password Reset). Lorsque vous personnalisez les pages Classic Login dans l’<Tooltip tip="Auth0 Dashboard : interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, les modèles par défaut utilisent les mêmes bibliothèques JavaScript afin d’assurer une transition plus naturelle entre l’interface utilisateur par défaut et une interface personnalisée.

Après avoir choisi un modèle par défaut pour vos pages Classic Login, vous pouvez le modifier selon vos besoins. Vous pouvez aussi personnaliser différents éléments du comportement et de l’apparence du widget Lock. Les modèles Auth0.js offrent une souplesse supplémentaire, car ils vous permettent de créer une interface utilisateur personnalisée et de l’adapter au style de votre application.

<Warning>
  À l’heure actuelle, les efforts de développement d’Auth0 sont principalement axés sur Universal Login, et Classic Login ne reçoit plus de mises à jour. À moins que votre cas d’utilisation particulier n’exige l’expérience Classic Login, il est recommandé d’implémenter Universal Login.

  Pour comparer les fonctionnalités des expériences de connexion, consultez [Universal Login vs. Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login)
</Warning>
