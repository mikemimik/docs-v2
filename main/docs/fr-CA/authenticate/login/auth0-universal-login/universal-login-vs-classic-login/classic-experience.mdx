title: Expérience Classic Login
---

Classic Login est une expérience de connexion hébergée par Auth0 qui s’appuie sur JavaScript pour la personnalisation. Mettre en œuvre Classic Login est moins complexe que d’intégrer directement le processus d’authentification à votre application, et cela peut aider à prévenir les risques liés à l’authentification cross-origin.

Classic Login repose sur les bibliothèques JavaScript d’Auth0 ([Lock.js](/docs/fr-CA/libraries/lock), [auth0.js](/docs/fr-CA/libraries/auth0js), le widget <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur supplémentaire en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> et Password Reset). Lorsque vous personnalisez les pages Classic Login dans le <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, les modèles par défaut utilisent les mêmes bibliothèques JavaScript afin d’assurer une transition plus naturelle entre l’interface utilisateur par défaut et une interface personnalisée.

Après avoir choisi un modèle par défaut pour vos pages Classic Login, vous pouvez le modifier selon vos besoins. Vous pouvez aussi personnaliser divers éléments de comportement et d’apparence du widget Lock. Les modèles Auth0.js offrent une plus grande souplesse, puisque vous pouvez créer une interface utilisateur personnalisée et l’adapter au style de votre application.

<Warning>
  Pour le moment, les efforts de développement actifs d’Auth0 sont axés sur Universal Login, et Classic Login n’est plus mis à jour. À moins que votre cas d’utilisation exige précisément l’expérience Classic, il est recommandé de mettre en œuvre Universal Login.

  Pour comparer les fonctionnalités des différentes expériences de connexion, consultez [Universal Login vs. Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login)
</Warning>
