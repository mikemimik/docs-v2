title: Page d’erreur Auth0 par défaut
---

Si votre URL de rappel est valide, lorsqu’une erreur d’autorisation se produit, le <Tooltip tip="serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=Authorization+Server">serveur d’autorisation</Tooltip> renvoie à votre URL de rappel les paramètres `error` et `state` appropriés. Si votre URL de rappel n’est pas valide, votre application affichera la page d’erreur générique Auth0 par défaut. Vous pouvez également personnaliser les pages d’erreur. Pour en savoir plus, consultez [Custom Error Pages](/docs/fr-CA/customize/login-pages/custom-error-pages).

Pour trouver le nom par défaut de la page d’erreur générique, consultez [Use Version Control to Manage Your Universal Login Pages](/docs/fr-CA/customize/login-pages/classic-login/version-control).

<div id="display">
  ## Affichage
</div>

La page d’erreur générique affiche les renseignements suivants si vous avez configuré les champs correspondants dans vos [Paramètres du locataire](https://manage.auth0.com/#/tenant/).

| Champ               | Description                                                 |
| ------------------- | ----------------------------------------------------------- |
| Nom affiché         | Nom de votre entreprise                                     |
| URL du logo         | URL du logo de votre entreprise                             |
| Courriel de soutien | Adresse courriel de l’équipe de soutien de votre entreprise |
| URL de soutien      | URL de la page de soutien de votre entreprise               |

<div id="parameters">
  ## Paramètres
</div>

Le serveur d’autorisation renvoie des paramètres ajoutés à l’URL sous forme de chaîne de requête.

| Paramètre           | Description                                                                          |
| ------------------- | ------------------------------------------------------------------------------------ |
| `client_id`         | Identifiant de l’application Auth0.                                                  |
| `connection`        | Connexion utilisée au moment de l’erreur.                                            |
| `lang`              | Langue définie au moment de l’erreur.                                                |
| `error`             | Code de l’erreur.                                                                    |
| `error_description` | Description de l’erreur.                                                             |
| `tracking`          | Identifiant utilisé par Auth0 pour retrouver les erreurs dans les journaux internes. |

Les paramètres affichés varient selon le type d’erreur et sont propres à la requête. Par exemple, si la requête ne contenait pas de `client_id` et qu’une erreur s’est produite, aucun <Tooltip tip="Client ID : valeur d’identification attribuée par Auth0 à votre ressource enregistrée." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">ID client</Tooltip> ne sera renvoyé par la page d’erreur.
