title: Configurer le courriel ou les SMS pour l’authentification Passwordless
---

Auth0 vous permet de configurer <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> pour que les utilisateurs puissent s’authentifier au moyen d’un Magic Link ou d’un mot de passe à usage unique (OTP) envoyé par courriel, ou d’un OTP envoyé par SMS.

Pour activer l’authentification <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> par SMS ou par courriel, vous devez définir une [connexion Passwordless](/docs/fr-CA/authenticate/passwordless) pour votre application.

Une fois la connexion créée, vous devez configurer Universal Login pour l’utiliser. Vous avez trois options :

<div id="universal-login">
  ## Universal Login
</div>

Cette méthode utilise l’expérience Universal Login, qui prend en charge de façon native les connexions Passwordless. Pour en savoir plus, consultez [Passwordless with Universal Login](/docs/fr-CA/authenticate/passwordless/passwordless-with-universal-login).

<div id="universal-login-lock-passwordless">
  ## Universal Login + Lock (passwordless)
</div>

Cette méthode utilise une page de connexion personnalisée avec le modèle **Lock (passwordless)** et permet d’authentifier l’utilisateur avec le [Lock for Web SDK](/docs/fr-CA/libraries/lock) :

1. Accédez à [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings) et cliquez sur l’onglet **Login**.
2. Activez la bascule **Custom Login Page**, puis sélectionnez le modèle **Lock (passwordless)**. Le modèle HTML sera mis à jour avec du code qui utilise le widget Lock et les options de personnalisation Passwordless.
3. Personnalisez le modèle, puis cliquez sur **Save Changes**.

Vous pouvez utiliser du HTML et du CSS pour personnaliser le formulaire de connexion et prévisualiser les modifications dans le Dashboard. Pour en savoir plus sur la personnalisation du modèle **Lock (passwordless)**, consultez la section Passwordless dans [Lock for Web SDK](/docs/fr-CA/libraries/lock).

<div id="universal-login-custom-ui-auth0js">
  ## Universal Login + UI personnalisée + Auth0.js
</div>

Cette méthode utilise une page de connexion personnalisée avec le modèle **Custom Login Form** et authentifie les utilisateurs à l’aide du [SDK Auth0.js](/docs/fr-CA/libraries/auth0js). Pour en savoir plus, consultez [Authentification Passwordless avec Universal Login](/docs/fr-CA/authenticate/passwordless/implement-login/universal-login).
