---
title: Universal Login vs. Classic Login
description: Compare les fonctionnalités offertes par les expériences Universal Login et Classic Login
---

[Auth0 Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) fournit une fonctionnalité essentielle d’un <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités offertes à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=authorization+server">serveur d’autorisation</Tooltip> : le login flow. Lorsqu’un utilisateur doit prouver son identité pour avoir accès à votre application, vous pouvez le rediriger vers <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités offertes à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> et laisser Auth0 gérer le processus d’authentification.

Universal Login est la principale solution de connexion hébergée d’Auth0. [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) offre des outils de personnalisation conviviaux et une expérience plus simple et plus rapide pour les utilisateurs finaux. Auth0 prend aussi en charge [Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience), une expérience de connexion hébergée qui utilise des contrôles JavaScript pour la personnalisation.

<Warning>
  À l’heure actuelle, les efforts de développement d’Auth0 sont axés sur Universal Login, et Classic Login ne reçoit plus de mises à jour. À moins que votre cas d’usage n’exige l’expérience Classic, il est recommandé d’adopter Universal Login.
</Warning>
