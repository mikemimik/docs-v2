title: Embedded Login
---

Embedded Login permet à vos utilisateurs de se connecter directement à votre application et de transmettre leurs identifiants au serveur Auth0 pour l’authentification. Nous ne recommandons pas l’utilisation d’Embedded Login. Pour en savoir plus, consultez [Universal Login centralisé ou Embedded Login](/docs/fr-CA/authenticate/login/universal-vs-embedded-login).

Si vous décidez d’utiliser Embedded Login, vous devez [configurer votre application pour Cross-Origin Resource Sharing](/docs/fr-CA/get-started/applications/set-up-cors) et il est recommandé de [configurer un domaine personnalisé](/docs/fr-CA/customize/custom-domains). Vous pouvez ensuite intégrer le [SDK Lock](/docs/fr-CA/libraries/lock) ou le [SDK Auth0.js](/docs/fr-CA/libraries/auth0js) à votre application, ou appeler directement l’[Auth0 Authentication API](https://auth0.com/docs/api/authentication).

Auth0 prend en charge Embedded Login avec des connexions <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> pour plusieurs types d’application :

* [Connexion Passwordless intégrée dans les Native Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/native)
* [Connexion Passwordless intégrée dans les Regular Web Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Authentification Passwordless intégrée pour les single-page apps](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/spa)
