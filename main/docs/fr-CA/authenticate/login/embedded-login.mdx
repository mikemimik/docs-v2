title: Embedded Login
---

Embedded Login permet à vos utilisateurs de se connecter directement à votre application et de transmettre leurs identifiants au serveur Auth0 pour l’authentification. Nous ne recommandons pas d’utiliser Embedded Login. Pour en savoir plus, consultez [Centralized Universal Login vs. Embedded Login](/docs/fr-CA/authenticate/login/universal-vs-embedded-login).

Si vous décidez d’utiliser Embedded Login, vous devez [configurer votre application pour le Cross-Origin Resource Sharing](/docs/fr-CA/get-started/applications/set-up-cors) et devriez [configurer un domaine personnalisé](/docs/fr-CA/customize/custom-domains). Vous pouvez ensuite implémenter le [SDK Lock](/docs/fr-CA/libraries/lock) ou le [SDK Auth0.js](/docs/fr-CA/libraries/auth0js) dans votre application, ou appeler directement l’[Auth0 Authentication API](https://auth0.com/docs/api/authentication).

Auth0 prend en charge Embedded Login avec des connexions <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> pour plusieurs types d’applications :

* [Embedded Passwordless Login dans les Native Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/native)
* [Embedded Passwordless Login dans les Regular Web Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Authentification Passwordless intégrée pour les single-page apps](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/spa)
