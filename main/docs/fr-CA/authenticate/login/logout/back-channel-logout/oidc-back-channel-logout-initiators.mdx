Les OIDC Back-Channel Logout Initiators vous permettent de déconnecter à distance les utilisateurs de leurs applications en fonction des événements de fin de session. Les OIDC Back-Channel Logout Initiators fonctionnent d’un protocole à l’autre — par exemple, une requête de logout initiée par le <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> (IdP-initiated) <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SAML">SAML</Tooltip> — et ne sont pas touchés par les restrictions sur les témoins tiers.

Cette fonctionnalité est une extension de la spécification OIDC back-channel standard. Vous pouvez la configurer pour déclencher une requête de déconnexion OIDC Back-Channel pour des événements de fin de session précis, comme un changement de mot de passe ou l’expiration d’une session, ou pour tous les événements de fin de session.

Les administrateurs peuvent activer cette fonctionnalité pour des applications précises à l’aide de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> d’Auth0.

<div id="how-oidc-back-channel-logout-initiators-work">
  ## Fonctionnement des OIDC Back-Channel Logout Initiators
</div>

Les initiateurs associent une réponse OIDC Back-Channel Logout à un événement de fin de session. Ils captent l’événement et l’utilisent pour déclencher un jeton de déconnexion OIDC pour toutes les applications associées à la session en question.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les initiateurs ne contrôlent pas la gestion des sessions dans votre tenant, ni les événements de fin de session.
</Callout>

<div id="configure-oidc-back-channel-logout-initiators">
  ## Configurer les OIDC Back-Channel Logout Initiators
</div>

Vous pouvez configurer les OIDC Back-Channel Logout Initiators à l’aide de l’Auth0 Management API.

<div id="management-api">
  ### Management API
</div>

Vous pouvez configurer les OIDC Back-Channel Logout Initiators d’une application à l’aide de la Management API en utilisant le endpoint [Update a Client](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id).

1. Obtenez un [jeton d’accès à la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec le scope `update:clients`.
2. Appelez le endpoint [Update a Client](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) avec les données de configuration appropriées dans le payload. Par exemple, pour déconnecter une application après un événement de changement de mot de passe, fournissez ce qui suit :

   {/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample1 = `   PATCH /api/v2/clients/{yourClientId}
{
  ...
  "oidc_logout": {
    "backchannel_logout_urls": ["https://example.com/cb"]
    "backchannel_logout_initiators": {
      "mode":"custom",
      "selected_initiators": ["rp-logout", "idp-logout", "password-changed"]
    }
  }
  ...
}
`;

<div id="properties">
  #### Propriétés
</div>

L’objet `backchannel_logout_initiators` comprend les propriétés suivantes :

| Propriété             | Type   | Obligatoire ?                      | Description                                            | Valeurs prises en charge                                                                                                           |
| --------------------- | ------ | ---------------------------------- | ------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| `mode`                | string | Obligatoire                        | Méthode de configuration pour activer les initiateurs. | `custom`, `all`                                                                                                                    |
| `selected_initiators` | array  | Obligatoire si `mode` est `custom` | Liste des initiateurs à activer.                       | `rp-logout`, `idp-logout`, `password-changed`, `session-expired`, `session-revoked`, `account-deleted`, `email-identifier-changed` |

<div id="mode-property">
  ##### propriété `mode`
</div>

La propriété `mode` détermine la méthode de configuration utilisée pour activer les initiateurs.

Par défaut, elle est définie sur `custom`, ce qui vous permet de préciser quels initiateurs vous souhaitez activer. Si vous voulez que votre application se déconnecte chaque fois que la session de l’IdP prend fin, définissez-la sur `all`.

La propriété `mode` prend en charge les valeurs suivantes :

| Valeur   | Description                                                                       |
| -------- | --------------------------------------------------------------------------------- |
| `custom` | Active uniquement les initiateurs figurant dans le tableau `selected_initiators`. |
| `all`    | Active automatiquement tous les initiateurs actuels et futurs.                    |

<div id="selected_initiators-property">
  ##### Propriété `selected_initiators`
</div>

La propriété `selected_initiators` contient la liste des initiateurs à activer pour l’application spécifiée.

La propriété `selected_initiators` accepte les valeurs suivantes :

| Valeur                     | Description                                                         |
| -------------------------- | ------------------------------------------------------------------- |
| `rp-logout`                | La requête est initiée par une partie de confiance (RP).            |
| `idp-logout`               | La requête est initiée par un fournisseur d’identité externe (IdP). |
| `password-changed`         | La requête est initiée par un changement de mot de passe.           |
| `session-expired`          | La requête est initiée par l’expiration de la session.              |
| `session-revoked`          | La requête est initiée par la suppression de la session.            |
| `account-deleted`          | La requête est initiée par la suppression d’un compte.              |
| `email-identifier-changed` | La requête est initiée par un changement d’identifiant de courriel. |

<div id="selected-initiators-only">
  #### Initiateurs sélectionnés uniquement
</div>

Seuls les initiateurs requis (`rp-logout` et `idp-logout`) seront activés par défaut. Tout initiateur supplémentaire, y compris ceux qui seront ajoutés à l’avenir, doit d’abord être sélectionné avant de pouvoir déclencher une déconnexion de votre application.

Sélectionnez cette option si vous voulez que votre application ne se déconnecte qu’en réponse aux initiateurs que vous avez sélectionnés.

<div id="all-supported-initiators">
  #### Tous les initiateurs pris en charge
</div>

Vous serez automatiquement abonné à tous les initiateurs pris en charge, y compris à ceux qui seront ajoutés ultérieurement.

Sélectionnez cette option si vous voulez que votre application déconnecte l’utilisateur chaque fois que la session de l’IdP prend fin.
