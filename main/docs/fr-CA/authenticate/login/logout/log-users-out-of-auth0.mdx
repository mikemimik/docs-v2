Auth0 implémente le [RP-Initiated Logout 1.0](https://openid.net/specs/openid-connect-rpinitiated-1_0.html) d’<Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker de renseignements de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OpenID">OpenID</Tooltip> Connect pour la déconnexion de l’utilisateur final. Cette norme fait partie de la collection des [spécifications finales](https://openid.net/developers/specs/) d’OpenID Connect.

<div id="how-it-works">
  ## Fonctionnement
</div>

RP-Initiated Logout est un scénario dans lequel une <Tooltip tip="Relying Party : entité (comme un service ou une application) qui dépend d’un fournisseur d’identité tiers pour authentifier un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=relying+party">partie de confiance</Tooltip> (l’utilisateur) demande au fournisseur OpenID (Auth0) de le déconnecter.

1. L’utilisateur initie une demande de déconnexion dans votre application.
2. Votre application dirige l’utilisateur vers le point de terminaison [OIDC Logout](https://auth0.com/docs/api/authentication#oidc-logout) de l’Auth0 Authentication API.
3. Auth0 redirige l’utilisateur vers la destination appropriée en fonction des [paramètres du point de terminaison OIDC Logout](#oidc-logout-endpoint-parameters) fournis.

<div id="configure-rp-initiated-logout">
  ## Configurer RP-Initiated Logout
</div>

Pour configurer RP-Initiated Logout, vous devez vous assurer que votre application peut repérer le paramètre `end_session_endpoint` dans le [document de métadonnées de découverte](https://openid.net/specs/openid-connect-discovery-1_0.html#ProviderConfig) de votre tenant Auth0 et qu’elle appelle le point de terminaison OIDC Logout avec les paramètres nécessaires.

<div id="enable-endpoint-discovery">
  ### Activer la découverte du point de terminaison
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour les tenants Auth0 créés à compter du 14 novembre 2023, **RP-Initiated Logout End Session Endpoint Discovery** est activée par défaut.
</Callout>

Vous pouvez activer **RP-Initiated Logout End Session Endpoint Discovery** dans le <Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou à l’aide de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> Auth0.

<Tabs>
  <Tab title="Dashboard">
    Pour activer **RP-Initiated Logout End Session Endpoint Discovery** dans le Dashboard :

    1. Accédez à [Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced).
    2. Repérez la section **Login and Logout**.
    3. Activez la bascule **RP-Initiated Logout End Session Endpoint Discovery**.
  </Tab>

  <Tab title="Management API">
    Pour activer **RP-Initiated Logout End Session Endpoint Discovery** avec la Management API :

    1. [Obtenez un jeton d’accès à la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) qui inclut le scope `update:tenant_settings`.
    2. Appelez le point de terminaison [Update tenant settings](https://auth0.com/docs/api/management/v2/tenants/patch-settings) de la Management API avec la charge utile suivante :

       ```json JSON lines
       {
         "oidc_logout": {
           "rp_logout_end_session_endpoint_discovery": true
         }
       }
       ```
  </Tab>
</Tabs>

<div id="call-the-oidc-logout-endpoint">
  ### Appelez le point de terminaison OIDC Logout
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La plupart des [bibliothèques SDK](/docs/fr-CA/libraries) d’Auth0 sont conformes à OIDC et prennent en charge RP-Initiated Logout nativement.
</Callout>

Lorsque vous appelez le point de terminaison OIDC Logout, Auth0 recommande de fournir le paramètre `id_token_hint`.

Si votre application ne peut pas stocker les <Tooltip tip="ID Token : jeton destiné au client lui-même plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+tokens">jetons ID</Tooltip> de façon sécuritaire, vous pouvez fournir les paramètres `logout_hint` et `client_id` à la place.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pouvez appeler le point de terminaison OIDC Logout avec des informations redondantes.

  Par exemple, vous pouvez envoyer les paramètres `id_token_hint` et `logout_hint`, ou les paramètres `id_token_hint` et `client_id`.

  Dans tous les cas, Auth0 vérifie la cohérence des données utilisateur et de session, et renvoie une erreur en cas d’incohérence.
</Callout>

<div id="oidc-logout-endpoint-parameters">
  #### Paramètres du point de terminaison OIDC Logout
</div>

Le point de terminaison [OIDC Logout](https://auth0.com/docs/api/authentication#oidc-logout) de l’Authentication API prend en charge les paramètres suivants :

| Paramètre                  | Obligatoire ? | Description                                                                                                                                                                                     |
| -------------------------- | ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id_token_hint`            | Recommandé    | jeton d’ID précédemment émis pour l’utilisateur. Il indique quel utilisateur déconnecter.                                                                                                         |
| `logout_hint`              | Facultatif    | Valeur de l’ID de session (`sid`) qui indique quel utilisateur déconnecter.                                                                                                                     |
| `post_logout_redirect_uri` | Facultatif    | Valeur de l’URL de redirection qui indique vers où rediriger l’utilisateur après la déconnexion.                                                                                                |
| `client_id`                | Facultatif    | ID client de votre application.                                                                                                                                                                 |
| `federated`                | Facultatif    | Indique à Auth0 de déconnecter l’utilisateur de son fournisseur d’identité.                                                                                                                     |
| `state`                    | Facultatif    | Valeur opaque que l’application ajoute à la request de déconnexion initiale et qu’Auth0 inclut lors de la redirection vers le `post_logout_redirect_uri`.                                       |
| `ui_locales`               | Facultatif    | Liste de locales séparées par des espaces, utilisée pour restreindre la liste de langues de la request. La première locale de la liste doit correspondre à la locale activée dans votre tenant. |

<div id="id_token_hint-parameter">
  #### paramètre `id_token_hint`
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recommande d’utiliser le paramètre `id_token_hint` lorsque vous appelez le point de terminaison OIDC Logout.
</Callout>

La valeur du paramètre `id_token_hint` doit être le jeton d’ID qu’Auth0 a émis pour l’utilisateur après son authentification.

Le jeton d’ID contient, aux fins de vérification, la claim enregistrée issuer (`iss`), l’<Tooltip tip="Audience : identificateur unique de l’audience d’un jeton émis. Appelée aud dans un jeton, sa valeur contient l’ID d’une application (Client ID) pour un ID Token ou d’une API (API identifier) pour un jeton d’accès." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip> (`aud`) et l’ID de session Auth0 (`sid`). Pour en savoir plus sur les claims du jeton d’ID, consultez [Structure du jeton d’ID](/docs/fr-CA/secure/tokens/id-tokens/id-token-structure).

<div id="examples">
  ##### Exemples
</div>

<AuthCodeGroup>
  ```http HTTP
  https://{yourDomain}/oidc/logout?id_token_hint={yourIdToken}&post_logout_redirect_uri={yourCallbackUrl}
  ```

  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/oidc/logout' \
    --header 'content-type: application/x-www-form-urlencoded' \
    --data 'id_token_hint={yourIdToken}' \
    --data 'post_logout_redirect_uri={yourCallbackUrl}'
  ```
</AuthCodeGroup>

<div id="logout_hint-parameter">
  #### paramètre logout_hint
</div>

La valeur du paramètre `logout_hint` doit correspondre à l’ID de session (`sid`) de la session Auth0 actuelle de l’utilisateur.

L’ID de session (`sid`) est fourni sous forme de claim enregistré dans le jeton d’ID qu’Auth0 a émis à l’utilisateur après qu’il se soit authentifié.

<Warning>
  Vous devez utiliser l’ID de session (`sid`) associé au jeton d’ID émis par Auth0 au moment où la session actuelle a débuté. Auth0 ignore les requêtes dont les valeurs sont aléatoires ou ne correspondent pas aux données de la session en cours.
</Warning>

<div id="allowed-logout-urls-guidelines">
  ##### Consignes relatives aux Allowed Logout URLs
</div>

Lorsque vous mettez à jour **Allowed Logout URLs**, suivez les consignes ci-dessous pour éviter les erreurs de validation :

* Séparez plusieurs URL par une virgule (`,`).
* Incluez le protocole de l’URL (par exemple, `https://`).

Vous pouvez utiliser un astérisque (`*`) comme caractère générique pour les sous-domaines (par exemple, `https://*.example.com`), mais nous vous recommandons de ne pas utiliser de caractères génériques dans les environnements de production. Pour en savoir plus, consultez [Espaces réservés d’URL pour les sous-domaines](/docs/fr-CA/get-started/applications/wildcards-for-subdomains#wildcard-url-placeholders).

<div id="add-query-string-parameters-to-post_logout_redirect_uri">
  ##### Ajouter des paramètres de requête à post_logout_redirect_uri
</div>

Le point de terminaison OIDC Logout analyse les paramètres de la chaîne de requête dans l’URL fournie au paramètre `post_logout_redirect_uri`.

Vous devez inclure l’**URL exacte avec les paramètres de la chaîne de requête** dans vos **Allowed Logout URLs**, sinon la requête de logout sera refusée. L’URL doit correspondre exactement, y compris tous les noms et toutes les valeurs des paramètres de requête.

Par exemple, si vous transmettez `https://example.com/logout?myParam=1234` au paramètre `post_logout_redirect_uri` (encodé sous la forme `https%3A%2F%2Fexample.com%2Flogout%3FmyParam%3D1234`), vous devez inclure l’URL complète `https://example.com/logout?myParam=1234` dans vos **Allowed Logout URLs**.

<Warning>
  Les valeurs dynamiques des paramètres de requête ne sont pas prises en charge. Chaque combinaison unique de noms et de valeurs de paramètres de requête doit être enregistrée comme une entrée distincte dans vos **Allowed Logout URLs**.
</Warning>

<div id="ui_locales-parameter">
  #### paramètre ui_locales
</div>

La valeur du paramètre `ui_locales` doit être une liste de [locales prises en charge](/docs/fr-CA/customize/internationalization-and-localization/universal-login-internationalization#new-universal-login-experience-localization), séparée par des espaces.

La première valeur de la liste doit correspondre au [réglage Langue par défaut de votre tenant](/docs/fr-CA/get-started/tenant-settings#languages).

<div id="federated-parameter">
  #### paramètre `federated`
</div>

Le paramètre `federated` ne nécessite aucune valeur.

Si vous incluez le paramètre `federated` lorsque vous appelez le point de terminaison OIDC Logout, Auth0 tente de [déconnecter l’utilisateur de son fournisseur d’identité](/docs/fr-CA/authenticate/login/logout/log-users-out-of-idps).

<div id="logout-consent-prompt">
  ## Invite de consentement à la déconnexion
</div>

La norme OIDC prévoit que le processus de déconnexion doit être interrompu pour demander le consentement de l’utilisateur si le fournisseur OpenID ne peut pas vérifier que la requête a bien été effectuée par l’utilisateur.

Auth0 applique ce comportement en affichant une invite de consentement à la déconnexion s’il détecte l’une des conditions suivantes :

* Ni le paramètre `id_token_hint` ni le paramètre `logout_hint` n’est fourni.
* Le claim `sid` du jeton d’ID ne correspond pas à la session du navigateur associée à la requête.
* La valeur du paramètre `logout_hint` ne correspond pas aux données de la session en cours.

<Frame>![](/docs/images/cdy7uua7fh8z/5Sycg1aMJ1CLZwJj19Omv4/655fa33b439217d0410fba1335c63a63/Logout_-_English.png)</Frame>

Si l’utilisateur confirme la requête de déconnexion, Auth0 poursuit le processus de déconnexion.

<div id="disable-the-logout-consent-prompt">
  ### Désactiver l’invite de consentement à la déconnexion
</div>

Vous pouvez désactiver l’invite de consentement à la déconnexion. Si vous le faites, Auth0 ne tentera pas de détecter les comportements anormaux et acceptera automatiquement les requêtes de déconnexion.

Pour désactiver l’invite de consentement à la déconnexion dans le Dashboard :

1. Accédez à [Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced).
2. Désactivez la bascule **Show RP-Initiated Logout End-User Confirmation**.
