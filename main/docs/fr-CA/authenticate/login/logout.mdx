<div id="logout-session-layers">
  ## Couches de session de Logout
</div>

Lorsque vous mettez en œuvre la fonctionnalité de Logout, il y a généralement trois couches de session à prendre en compte :

1. **Application Session Layer** : La première couche est la session dans votre application. Même si votre application utilise Auth0 pour authentifier les utilisateurs, vous devez quand même garder une trace du fait que l’utilisateur s’est connecté à votre application. Dans une application Web traditionnelle, cela se fait en stockant l’information dans un cookie. [Déconnectez les utilisateurs de vos applications](/docs/fr-CA/authenticate/login/logout/log-users-out-of-applications) en effaçant leurs sessions. Vous devez gérer la session de l’application dans votre application.

2. **Auth0 Session Layer** : Auth0 maintient également une session pour l’utilisateur et stocke ses renseignements dans un cookie. La prochaine fois qu’un utilisateur sera redirigé vers l’écran Auth0 Lock, ses renseignements seront conservés. [Déconnectez les utilisateurs d’Auth0](/docs/fr-CA/authenticate/login/logout/log-users-out-of-auth0) en effaçant le cookie de Single Sign-On (SSO).

3. **Identity Provider Session Layer** : La dernière couche de session est celle de l’Identity Provider (par exemple, Facebook ou Google). Lorsque des utilisateurs tentent de se connecter avec l’un de ces fournisseurs alors qu’ils sont déjà connectés auprès de ce fournisseur, ils n’auront pas à se reconnecter. Il se peut qu’on demande aux utilisateurs d’autoriser le partage de leurs renseignements avec Auth0 et, par conséquent, avec votre application. Il n’est pas nécessaire de déconnecter les utilisateurs de cette couche de session, mais vous pouvez forcer le Logout. (Pour en savoir plus, consultez [Log Users Out of Identity Providers](/docs/fr-CA/authenticate/login/logout/log-users-out-of-idps) et [Log Users Out of SAML Identity Providers](/docs/fr-CA/authenticate/login/logout/log-users-out-of-saml-idps).)

<div id="redirect-users-after-logout">
  ## Rediriger les utilisateurs après le Logout
</div>

Après leur Logout, vous pouvez [rediriger les utilisateurs](/docs/fr-CA/authenticate/login/logout/redirect-users-after-logout) vers une URL précise. Vous devez enregistrer l’URL de redirection dans les paramètres de votre tenant ou de votre application. Après le Logout, Auth0 redirige uniquement vers les URL figurant dans la liste d’autorisation. Si vous avez besoin de redirections différentes pour chaque application, vous pouvez ajouter les URL à la liste d’autorisation dans les paramètres de votre application.
