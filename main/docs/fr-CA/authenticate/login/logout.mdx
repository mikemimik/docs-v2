<div id="logout-session-layers">
  ## Couches de session de déconnexion
</div>

Lorsque vous mettez en œuvre la fonctionnalité de déconnexion, vous devez généralement tenir compte de trois couches de session :

1. **Couche de session de l&#39;application** : La première couche est la session dans votre application. Même si votre application utilise Auth0 pour authentifier les utilisateurs, vous devez quand même garder une trace du fait que l&#39;utilisateur s&#39;est connecté à votre application. Dans une application Web traditionnelle, vous le faites en stockant des informations dans un cookie. [Déconnectez les utilisateurs de vos applications](/docs/fr-CA/authenticate/login/logout/log-users-out-of-applications) en effaçant leurs sessions. Vous devez gérer la session de l&#39;application dans votre application.

2. **Couche de session Auth0** : Auth0 maintient également une session pour l&#39;utilisateur et stocke ses renseignements dans un cookie. La prochaine fois qu&#39;un utilisateur sera redirigé vers l&#39;écran Auth0 Lock, ses renseignements seront conservés. [Déconnectez les utilisateurs d&#39;Auth0](/docs/fr-CA/authenticate/login/logout/log-users-out-of-auth0) en effaçant le cookie d&#39;authentification unique (SSO).

3. **Couche de session du fournisseur d&#39;identité** : La dernière couche de session est celle du fournisseur d&#39;identité (par exemple, Facebook ou Google). Lorsque les utilisateurs tentent de se connecter avec l&#39;un de ces fournisseurs alors qu&#39;ils y sont déjà connectés, ils n&#39;auront pas à se reconnecter. On peut toutefois leur demander d&#39;autoriser le partage de leurs renseignements avec Auth0 et, par conséquent, avec votre application. Il n&#39;est pas nécessaire de déconnecter les utilisateurs de cette couche de session, mais vous pouvez forcer la déconnexion. (Pour en savoir plus, consultez [Déconnecter les utilisateurs des fournisseurs d&#39;identité](/docs/fr-CA/authenticate/login/logout/log-users-out-of-idps) et [Déconnecter les utilisateurs des fournisseurs d&#39;identité SAML](/docs/fr-CA/authenticate/login/logout/log-users-out-of-saml-idps).)

<div id="redirect-users-after-logout">
  ## Rediriger les utilisateurs après la déconnexion
</div>

Une fois les utilisateurs déconnectés, vous pouvez [rediriger les utilisateurs](/docs/fr-CA/authenticate/login/logout/redirect-users-after-logout) vers une URL précise. Vous devez enregistrer l’URL de redirection dans les paramètres de votre tenant ou de votre application. Auth0 redirige uniquement vers les URL de la liste d’autorisation après la déconnexion. Si vous avez besoin de redirections différentes pour chaque application, vous pouvez ajouter les URL à votre liste d’autorisation dans les paramètres de votre application.
