Le protocole <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) est un standard ouvert basé sur XML pour l’authentification et l’autorisation entre deux entités, sans mot de passe :

* Le **fournisseur de services** (SP) accepte de faire confiance au <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> pour authentifier les utilisateurs.
* Le **fournisseur d’identité** (IdP) authentifie les utilisateurs et fournit aux fournisseurs de services une assertion d’authentification indiquant qu’un utilisateur a bien été authentifié.

Auth0 prend en charge le protocole SAML et peut agir comme IdP, comme SP, ou les deux, notamment pour :

* les applications Web SAML2
* les intégrations SAML <Tooltip tip="Single Sign-On (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip>
* le SSO initié par l’IdP
* Microsoft Active Directory Federation Services (ADFS)
* la signature et le chiffrement des requêtes SAML

<div id="supported-saml-bindings-and-options">
  ## Bindings et options SAML pris en charge
</div>

Auth0 prend en charge les bindings SAML suivants :

* Redirection HTTP
* HTTP POST

Auth0 prend en charge les options SAML suivantes :

* Profil SSO de navigateur Web
* Profil Single Logout
* Profil de gestion du Name Identifier
* Profil de mappage du Name Identifier
