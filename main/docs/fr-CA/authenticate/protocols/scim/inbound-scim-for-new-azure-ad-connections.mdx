<Card title="Avant de commencer">
  Assurez-vous que votre tenant Auth0 est connecté à Azure AD au moyen du [type de connexion Microsoft Azure AD](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

  Cette intégration nécessite l’enregistrement de deux applications dans Azure AD : l’intégration OpenID Connect et l’intégration SCIM. Pour simplifier ce processus de configuration pour vos clients, envisagez de [publier votre application dans la galerie d’applications Azure Active Directory](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/v2-howto-app-gallery-listing#submit-your-application).
</Card>

Auth0 peut être intégré à Microsoft Azure Active Directory (désormais appelé Microsoft Entra ID) au moyen du [type de connexion Microsoft Azure AD](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2), qui utilise le protocole <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les renseignements de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) pour l’authentification des utilisateurs. Ces instructions portent sur les nouvelles connexions Azure AD; pour les anciennes connexions utilisant des identifiants utilisateur pairwise (`sub)`, consultez [Anciennes connexions](/docs/fr-CA/authenticate/protocols/scim/inbound-scim-for-older-azure-ad-connections).

<div id="configure-scim-settings-in-auth0">
  ## Configurer les paramètres SCIM dans Auth0
</div>

1. Ouvrez l’[Auth0 Dashboard](https://manage.auth0.com/#/), puis accédez à **Authentication &gt; Enterprise &gt; Microsoft Azure AD &gt; [your-connection] &gt; Settings.**
2. Assurez-vous que **User ID Attribute Type** est défini sur **User Object Identifier (oid)** et que la bascule **Use Common Endpoint** est **Disabled**.
3. Sélectionnez l’onglet **provisionnement** et désactivez **Sync user profile attributes at each login**, sauf si vous souhaitez [synchroniser des attributs supplémentaires à la connexion](/docs/fr-CA/authenticate/protocols/scim/configure-inbound-scim).
4. Dans la même section, activez **Sync user profiles using SCIM**.
5. Sous l’onglet **Mapping**, assurez-vous que le paramètre **SCIM attribute containing the User ID** est défini sur **externalId**.
6. Vérifiez les **Additional Mappings** pour vous assurer que les attributs SCIM étendus sont bien mappés aux attributs Auth0 de votre choix. Consultez [attribute mapping](/docs/fr-CA/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) pour plus de détails.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Récupérer l’URL du point de terminaison SCIM et le jeton
</div>

Cette section utilise le <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, mais ces étapes peuvent aussi être effectuées avec la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Consultez la section [Deployment Guidelines](/docs/fr-CA/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines) pour connaître les pratiques exemplaires.

1. Dans le [Auth0 Dashboard](https://manage.auth0.com/#/), accédez à l’onglet SCIM **Setup**, puis copiez la **URL du point de terminaison SCIM** et collez-la dans un endroit sûr.
2. Générez un jeton SCIM en cliquant sur **Generate New Token**, puis définissez une date d’expiration pour le jeton, si vous le souhaitez.
3. Sélectionnez les scopes que vous souhaitez autoriser. Les scopes par défaut requis par Azure AD sont `get:users`, `post:users`, `patch:users` et `delete:users`. Vous pouvez également sélectionner `get:groups`, `post:groups`, `patch:groups` et `delete:groups` afin de provisionner des groupes.

<div id="configure-scim-in-azure-ad-for-oidc-apps">
  ## Configurer SCIM dans Azure AD pour les applications OIDC
</div>

1. Confirmez qu’une application OpenID Connect a déjà été enregistrée pour gérer l’authentification des utilisateurs dans la section **Microsoft Entra ID &gt; Inscriptions d’applications** du [portail Azure](https://portal.azure.com).
2. Confirmez que l’option **Affectation requise** de votre application OpenID Connect est définie sur **Oui** dans la section **Microsoft Entra ID &gt; Applications d’entreprise &gt; [your-oidc-app] &gt; Gérer &gt; Propriétés**, et que des utilisateurs lui sont affectés dans l’onglet **Utilisateurs et groupes**.
3. Ensuite, enregistrez une nouvelle application **Hors galerie** dans le [portail Azure](https://portal.azure.com) en accédant à **Microsoft Entra ID &gt; Applications d’entreprise &gt; Nouvelle application &gt; Créer votre propre application,** en entrant un nom d’application, puis en sélectionnant **Créer**.
4. Accédez à l’onglet **Utilisateurs et groupes** et affectez-y les mêmes utilisateurs et groupes Azure AD que ceux affectés à l’application OpenID Connect enregistrée.
5. Sélectionnez l’onglet **Provisionnement**, sélectionnez **Commencer**, puis choisissez **Automatique** comme **Mode de provisionnement.**
6. Sélectionnez **Informations d’identification de l’administrateur**, puis entrez la valeur **URL du point de terminaison SCIM** que vous avez enregistrée plus tôt comme **URL du locataire**. À la fin de l’URL, ajoutez le paramètre de requête `?aadOptscim062020` pour appliquer le correctif aux [problèmes connus d’Azure AD décrits ici.](https://learn.microsoft.com/en-us/azure/active-directory/app-provisioning/application-provisioning-config-problem-scim-compatibility)
7. Collez la valeur du jeton dans le champ **Jeton secret** et sélectionnez **Enregistrer.**
8. Accédez à **Mappages**.
9. Pour provisionner des groupes au besoin, confirmez que **Provisionner les groupes Entra ID** est défini sur **Activé**. Sinon, définissez cette option sur **Désactivé**.
10. Sélectionnez **Provisionner les utilisateurs Microsoft Entra ID,** puis accédez à **Mappages d’attributs** et modifiez les attributs de la ligne contenant `externalId` et `mailNickname`.
11. Dans l’écran **Modifier l’attribut**, remplacez **Attribut source** par **objectId,** puis choisissez **OK**.
12. Revenez à **Mappages d’attributs** et sélectionnez la ligne contenant `emails[type eq "work"].value and mail`.
13. Dans l’écran **Modifier l’attribut**, remplacez **Faire correspondre les objets à l’aide de cet attribut** par **Oui**, puis définissez **Priorité de correspondance** sur **2** et choisissez **OK.** L’écran de mappage d’attributs ressemble à ceci pendant que vous continuez à utiliser la section **Mappages d’attributs** pour configurer des attributs SCIM supplémentaires :

<Frame>![Mappage d’attributs SAML Azure](/docs/images/cdy7uua7fh8z/6JnGm4IeBglKiHNoVPX5vs/ddc17baeca48f02a6309b57ab87f845e/attributemap2.jpg)</Frame>

**Enregistrez** les mappages d’attributs, puis sélectionnez **X** dans le coin supérieur droit pour revenir à l’écran **Provisionnement**.

<div id="testing">
  ## Test
</div>

1. Dans l’écran **Vue d’ensemble** de l’application d’entreprise, accédez à **Gérer &gt; provisionnement &gt; provisionnement à la demande** pour tester la connexion SCIM.
2. Accédez à **Sélectionner un utilisateur ou un groupe** et entrez le nom d’un utilisateur ou d’un groupe que vous avez attribué à l’application, puis sélectionnez l’élément et choisissez **Approvisionner**. Cela crée l’utilisateur ou le groupe dans le tenant Auth0.
3. Réglez **État du provisionnement** sur **Activé** afin d’[approvisionner tous les utilisateurs et groupes attribués](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status).
