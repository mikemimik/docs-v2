Auth0 prend en charge des profils d’algorithmes prédéfinis, qui offrent un moyen simple de définir une liste d’algorithmes cryptographiques sécurisés sans devoir préciser chaque algorithme individuellement pour votre cas d’utilisation. Ces profils correspondent à un ensemble prédéfini d’algorithmes qui sert de référence pour l’usage prévu.

<div id="naming-conventions">
  ## Conventions de nommage
</div>

Les profils sont nommés d’après l’année de leur création, avec un numéro incrémentiel pour distinguer les versions ultérieures d’une même année. Dans la mesure du possible, il convient d’utiliser la version la plus récente du profil afin de bénéficier de la configuration la plus sécuritaire.

<div id="supported-profiles">
  ## Profils pris en charge
</div>

Auth0 en tant que fournisseur de services SAML (SP)

| **Profil** | **Algorithmes pris en charge**                                                                                                                                         |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `v2026-1`  | <ul> <li>http://www.w3.org/2009/xmlenc11#aes128-gcm</li> <li>http://www.w3.org/2009/xmlenc11#aes192-gcm</li> <li>http://www.w3.org/2009/xmlenc11#aes256-gcm</li> </ul> |

Pour en savoir plus sur la configuration du profil d’authentication d’une connection, consultez [Signer et chiffrer les requêtes SAML](/docs/fr-CA/authenticate/protocols/saml/saml-sso-integrations/sign-and-encrypt-saml-requests).
