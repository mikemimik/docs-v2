Auth0 prend en charge des profils d’algorithmes prédéfinis pour vous permettre de définir facilement une liste d’algorithmes cryptographiques sécurisés, sans avoir à spécifier chaque algorithme individuellement pour votre cas d’utilisation. Ces profils représentent un ensemble prédéfini d’algorithmes qui sert de référence pour le modèle d’utilisation visé.

<div id="naming-conventions">
  ## Conventions de dénomination
</div>

Les profils sont nommés selon leur année de création, avec un numéro séquentiel pour distinguer les versions publiées la même année. Dans la mesure du possible, il convient d’utiliser la version la plus récente du profil afin de bénéficier de la configuration la plus sécuritaire.

<div id="supported-profiles">
  ## Profils pris en charge
</div>

Auth0 comme fournisseur de services SAML (SP)

| **Profil** | **Algorithmes pris en charge**                                                                                                                                         |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `v2026-1`  | <ul> <li>http://www.w3.org/2009/xmlenc11#aes128-gcm</li> <li>http://www.w3.org/2009/xmlenc11#aes192-gcm</li> <li>http://www.w3.org/2009/xmlenc11#aes256-gcm</li> </ul> |

Pour en savoir plus sur la façon de configurer le profil d’authentification d’une connection, consultez [Signer et chiffrer les requêtes SAML](/docs/fr-CA/authenticate/protocols/saml/saml-sso-integrations/sign-and-encrypt-saml-requests).
