Auth0 prend en charge des profils d’algorithmes prédéfinis pour offrir un moyen simple de définir une liste d’algorithmes cryptographiques sécurisés, sans avoir à préciser chaque algorithme individuellement selon votre cas d’utilisation. Ces profils correspondent à un ensemble prédéfini d’algorithmes qui sert de point de référence pour l’usage prévu.

<div id="naming-conventions">
  ## Conventions de dénomination
</div>

Les profils sont nommés selon leur année de création, avec un numéro séquentiel pour indiquer les versions publiées la même année. Dans la mesure du possible, utilisez la version la plus récente du profil pour bénéficier de la configuration la plus sécuritaire.

<div id="supported-profiles">
  ## Profils pris en charge
</div>

Auth0 comme fournisseur de services SAML (SP)

| **Profil** | **Algorithmes pris en charge**                                                                                                                                         |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `v2026-1`  | <ul> <li>http://www.w3.org/2009/xmlenc11#aes128-gcm</li> <li>http://www.w3.org/2009/xmlenc11#aes192-gcm</li> <li>http://www.w3.org/2009/xmlenc11#aes256-gcm</li> </ul> |

Pour en savoir plus sur la configuration du profil d’authentification d’une connexion, consultez [Signer et chiffrer les requêtes SAML](/docs/fr-CA/authenticate/protocols/saml/saml-sso-integrations/sign-and-encrypt-saml-requests).
