---
description: Décrit comment utiliser des certificats ou des clés dans des Rules ou des requêtes à la Management API lorsqu’une représentation du fichier sous forme de chaîne est requise.
title: Utiliser des certificats et des clés sous forme de chaînes
---

Vous pouvez utiliser la commande suivante dans un shell UNIX pour concaténer un certificat au format PEM avec `\n` (saut de ligne d’échappement) :

```bash wrap lines
awk 'NF {sub(/\r/, ""); printf "%s\\n",$0;}' certificate.pem | pbcopy
```

Cela copie la valeur dans votre presse-papiers pour que vous puissiez la coller au besoin.

<div id="more-information">
  ### Plus d’informations
</div>

Lorsque vous travaillez avec des certificats ou des clés dans des rules ou des requêtes v2 de <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, vous aurez probablement besoin d’une représentation sous forme de chaîne du fichier.

Si vous ouvrez un fichier de certificat (`cer`, `pem`) dans un éditeur de texte, vous verrez quelque chose comme ceci :

```text lines
-----BEGIN CERTIFICATE-----
MIICzDCCAbQCCQDH8GvxPIeH+DANBgkqhkiG9w0BAQsFADAoMQswCQYDVQQGEwJh
cjEZMBcGA1UEAwwQaHR0cHM6Ly9uaWNvLmNvbTAeFw0xOTA0MDgxODA3NTVaFw0y
//
// autres lignes d'informations encodées en base64
//
nSWyabd+LiBGtLTMB+ZLbOIi3EioWPGw/nHOI8jzPrqhiCLuZCSQmiqrLQYNsc1W
-----END CERTIFICATE-----
```

Les lignes entre l’en-tête `-----BEGIN CERTIFICATE-----` et le pied de page `-----END CERTIFICATE-----` contiennent des données binaires encodées en base64. Les clés publiques et les clés privées (fichiers .key) se ressemblent, mais avec un en-tête et un pied de page différents.

Pour obtenir la représentation sous forme de chaîne d’un fichier de certificat ou de clé, vous devrez tout concaténer sur une seule ligne, en remplaçant les sauts de ligne réels du fichier par une séquence `\n` (saut de ligne d’échappement). Ainsi, à partir de l’exemple ci-dessus, vous obtiendriez quelque chose comme ceci :

```text lines
"-----BEGIN CERTIFICATE-----
\nMIICzDCCAbQCCQDH8GvxPIeH+DANBgkqhkiG9w0BAQsFADAoMQswCQYDVQQGEwJh\n
[..all the other lines..]
-----END CERTIFICATE-----\n"
```
