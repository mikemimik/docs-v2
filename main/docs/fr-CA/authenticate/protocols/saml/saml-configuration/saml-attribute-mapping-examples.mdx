description: Fournit des exemples de la façon de mapper les attributs SAML lorsque Auth0 est le fournisseur d'identité.
title: Mapper les attributs SAML avec Auth0 comme IdP/module complémentaire SAML
---

Lorsque Auth0 est le <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=IdP">IdP</Tooltip>, vous pouvez mapper les attributs utilisateur au moyen du module complémentaire SAML2 d&#39;Auth0. Des erreurs peuvent survenir si les attributs sont mal configurés. Par exemple, un utilisateur saisit correctement son nom d&#39;utilisateur et son mot de passe, mais ne parvient pas à se connecter à l&#39;application, même si les logs dans l&#39;<Tooltip tip="Auth0 Dashboard : principal produit d'Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> indiquent des événements de connexion réussis. Il est aussi possible que des renseignements sur l&#39;utilisateur, comme son nom ou son adresse courriel, manquent dans votre application.

<div id="use-cases">
  ## Cas d’utilisation
</div>

Le profil d’utilisateur ci-dessous constitue un exemple pour les scénarios suivants.

```javascript lines
//EXEMPLE de profil d'utilisateur IdP
{
   "created_at": "2021-06-21T13:26:08.579Z",
   "email": "testuser@example.com",
...
   "fav_genre": "fiction",
   "user_metadata": {
       "fav_streaming_service": "hulu"
   }
...
}
```

<div id="no-mappings-object">
  #### Aucun objet mappings
</div>

Lorsqu’on utilise le module complémentaire SAML2, un objet mappings vide est généré par défaut.

Dans cet exemple, `fav_genre` et `user_metadata.fav_streaming_service` ne sont pas définis, mais ils peuvent être personnalisés et associés à la <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SAML">réponse SAML</Tooltip> remplie par Auth0.

Dans l’exemple ci-dessous, `"fav_genre": "fiction"` est associé à l’attribut  [http://schemas.auth0.com/fav&#95;genre](http://schemas.auth0.com/fav_food) dans la réponse SAML avec la valeur `fiction`, tandis que `"user_metadata": {"fav_streaming_service": "hulu"}` n’apparaît pas du tout dans la réponse SAML.

Réponse SAML résultante envoyée par l’IdP :

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/fav_genre" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
...
       </saml:AttributeStatement>
   </saml:Assertion>
</samlp:Response>
```

<div id="standard-mappings-example">
  #### Exemple de mappages standard
</div>

Dans l’exemple précédent, le fait de ne pas personnaliser l’objet mappings a produit un attribut [http://schemas.auth0.com/fav&#95;genre](http://schemas.auth0.com/fav_food) dans la réponse SAML avec la valeur `"fiction"`.

Ensuite, mappez les attributs dans l’objet Mappings des paramètres du module complémentaire SAML2 pour en tenir compte.

Une fois cela fait, notez que la valeur `"fiction"` est la même dans la réponse SAML, mais que le nom de l’attribut dans la réponse SAML a changé, passant de la valeur par défaut [http://schemas.auth0.com/fav&#95;fiction](http://schemas.auth0.com/fav_food) à [http://schemas.auth0.com/books](http://schemas.auth0.com/dinner).

Configuration de l’objet mappings du module complémentaire SAML2 :

```json lines
"mappings": {
   "fav_genre": "http://schemas.auth0.com/books"
 }
```

Ce mappage produit la réponse suivante :

```xml lines
<samlp:Response xmlns:samlp="urn:oasis:names:tc:SAML:2.0:protocol" ID="_e30cb5f29249a82846eb" InResponseTo="_e33996d83f953ce46225185b3a1c0ad8" Version="2.0" IssueInstant="2021-11-03T21:34:42.493Z" Destination="https://example-dev-tenant.us.auth0.com/login/callback">
...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/books" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
               <saml:AttributeValue xsi:type="xs:string">
                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
...
       </saml:AttributeStatement>
   </saml:Assertion>
</samlp:Response>
```

<div id="mapping-the-same-value-to-multiple-attributes">
  #### Associer la même valeur à plusieurs attributs
</div>

Dans certains cas, il peut être nécessaire d&#39;associer la réponse SAML à plusieurs attributs ayant la même valeur.

Dans ce cas, il est possible d&#39;associer la même valeur du profil utilisateur à plusieurs attributs dans la réponse SAML.

Comment configurer l&#39;objet mappings du module complémentaire SAML2 :

```json lines
"mappings": {
   "fav_genre": [
     "http://schemas.auth0.com/movies",
     "http://schemas.auth0.com/books",
     "http://schemas.auth0.com/television"
   ]
 }
```

Ce mappage produit la réponse suivante :

...
       <saml:AttributeStatement xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
           <saml:Attribute Name="http://schemas.auth0.com/movies" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">

                   fiction
               </saml:AttributeValue>
           </saml:Attribute>
           <saml:Attribute Name="http://schemas.auth0.com/books" NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri">
