---
title: Ouvrir et gérer des tickets de soutien
description: Décrit comment ouvrir et gérer des tickets dans Support Center.
---

Utilisez [Auth0 Support Center](https://support.auth0.com) pour créer des tickets si vous avez des questions ou des problèmes. Tous les rôles de membre du tenant peuvent accéder à Support Center.

Si vous êtes déjà client de Private Cloud, vous devez créer un tenant Auth0 dans le Public Cloud, aussi appelé le « Support Tenant », pour vous connecter à Support Center. Ce tenant peut aussi être utilisé à des fins de développement et de test, sans frais supplémentaires. Veuillez communiquer avec votre Technical Account Manager ou votre Sales Executive pour associer votre tenant dans le Public Cloud à votre abonnement Private Cloud existant.

<div id="open-a-new-ticket">
  ## Ouvrir un nouveau ticket
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Seuls les clients payants et en période d’essai peuvent ouvrir des tickets de soutien. Pour tous les autres clients, sélectionnez **Demander à notre Community** pour utiliser les ressources de la [Community](https://community.auth0.com/).
</Callout>

1. Accédez au [Support Center](https://support.auth0.com) et sélectionnez **Ouvrir un ticket**.
2. Sélectionnez le tenant pour lequel vous avez besoin d’aide dans le menu déroulant **Tenant concerné**.
3. Sous **Avec quoi pouvons-nous vous aider ?**, sélectionnez le type de problème qui correspond le mieux à votre situation.
4. Si on vous le demande, choisissez le [niveau de gravité](#severity-levels) approprié (non disponible pour les clients ayant un tenant d’essai).
5. Fournissez un résumé clair de la demande ou du problème dans le champ **Résumé de la demande**.
6. Fournissez autant de détails que possible sur le problème ou la demande dans la zone **Description**. Lorsque vous écrivez dans cette zone, vous pouvez mettre votre texte en forme avec [Markdown](https://guides.github.com/features/mastering-markdown) et prévisualiser ce que vous avez écrit en cliquant sur **Aperçu**. Lorsque vous rédigez votre description, essayez d’inclure les renseignements suivants, s’ils s’appliquent :

   1. Qu’essayiez-vous de faire ?
   2. Quel résultat attendiez-vous ?
   3. À quel moment les choses ont-elles mal tourné ?
   4. Y avait-il des messages d’erreur ou des captures d’écran montrant le problème ? Si oui, veuillez les inclure.
   5. Cela a-t-il déjà fonctionné auparavant, ou essayez-vous une nouvelle configuration ?
   6. Le problème se produit-il pour tous les utilisateurs ou seulement pour quelques-uns ?
   7. Quelles étapes de dépannage avez-vous essayées ?
7. Lorsque vous avez rempli tous les champs, sélectionnez **SOUMETTRE**.

<div id="severity-levels">
  ### Niveaux de gravité
</div>

| Niveau         | Type                                                                         | Description                                                                                                                                                                                                                                                              |
| -------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **1 (Urgent)** | Incident critique                                                            | Défaut entraînant une panne complète ou partielle du système, ou une situation qui rend la plateforme Auth0 inutilisable ou indisponible en production pour tous vos utilisateurs.                                                                                       |
| **2 (Élevé)**  | Répercussions importantes sur les activités                                  | Défaut entraînant une situation où des fonctionnalités essentielles sont touchées ou une dégradation importante des performances est observée ; le problème persiste et touche de nombreux utilisateurs et/ou des fonctionnalités essentielles.                          |
| **3 (Normal)** | Problèmes mineurs de fonctionnalité ou de fonctionnement / Question générale | Le défaut fait en sorte qu’un composant de la plateforme Auth0 ne fonctionne pas comme prévu ou comme indiqué dans la documentation, ou il s’agit d’une demande de renseignements de vos représentants concernant des questions ou problèmes techniques d’ordre général. |
| **4 (Low)**    | Problème mineur / Demande d’amélioration                                     | Demande de renseignements sur les capacités, la navigation, l’installation ou la configuration de la plateforme Auth0 ; demande d’amélioration.                                                                                                                          |

<div id="update-an-existing-ticket">
  ## Mettre à jour un ticket existant
</div>

1. Accédez à la page [Support Center](https://support.auth0.com) et sélectionnez **Home**.
2. Sélectionnez le ticket que vous souhaitez mettre à jour en cliquant sur son **Résumé de la demande**.
3. Saisissez tout renseignement supplémentaire dans la zone de texte, puis sélectionnez **REPLY**.

Si vous êtes le demandeur du ticket et que celui-ci est attribué à un agent, sans être résolu ni fermé, vous pouvez changer le statut de votre ticket à **Solved** en cochant la case **Submit as solved** à côté du bouton **REPLY**.

<div id="reopening-a-closed-ticket">
  ## Réouverture d’un ticket fermé
</div>

Si votre ticket a été fermé, mais que vous souhaitez poursuivre vos échanges avec Auth0 au sujet du problème, créez un nouveau ticket dans le Support Center. Assurez-vous d’inclure le numéro de référence du ou des tickets d’origine.

Lorsqu’un dossier est fermé, Auth0 supprime définitivement toutes les pièces jointes pour des raisons de sécurité.

<div id="manage-subscription-tickets">
  ## Gérer les tickets d’abonnement
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour gérer les tickets d’abonnement, le rôle [Elevated Support Access](/docs/fr-CA/get-started/manage-dashboard-access/feature-access-by-role) doit vous être attribué.

  Si vous êtes un administrateur de tenant, vous pouvez vous attribuer le rôle Elevated Support Access pour voir tous les tickets ouverts par tous les utilisateurs dans vos tenants.
</Callout>

Pour gérer les tickets d’abonnement :

1. Accédez à [Auth0 Support Center](https://support.auth0.com)
2. Sélectionnez **Tickets** dans le menu de navigation.
3. Passez à la vue **Subscription Tickets**.
