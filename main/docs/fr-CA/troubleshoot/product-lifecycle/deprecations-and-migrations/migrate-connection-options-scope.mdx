title: Migrer vers les scopes d’options de connexion de la Management API
---

​Auth0 introduit deux nouveaux scopes de la Management API pour les options de connexion : [`read:connections_options`](/docs/fr-CA/get-started/apis/scopes) et `update:connections_options`.

Nous déprécions également la possibilité d’accéder au champ `options` ou de le mettre à jour sans les scopes appropriés, ce qui entraîne les changements incompatibles suivants :

* Les réponses des points de terminaison [Get All Connections](/docs/fr-CA/api/management/v2/connections/get-connections) et [Get Connection by Id](/docs/fr-CA/api/management/v2/connections/get-connections-by-id) n’incluent pas le champ `options`, sauf si l’appelant possède le scope `read:connections_options`.
* Les requêtes au point de terminaison [Update a Connection](/docs/fr-CA/api/management/v2/connections/patch-connections-by-id) échouent si le corps de la requête contient le champ `options` et que l’appelant ne possède pas le scope `update:connections_options`. Les réponses réussies n’incluent pas le champ `options`, sauf si l’appelant possède le scope `read:connections_options`.

Pour continuer à utiliser le champ `options`, vous devez mettre à jour vos autorisations d’application afin d’inclure les nouveaux scopes. Suivez les étapes ci-dessous pour déterminer si votre tenant et vos intégrations sont touchés, migrer vers les scopes propres aux options, puis vous soustraire au comportement déprécié.

<div id="identify-affected-tenants">
  ## Identifier les tenants concernés
</div>

Utilisez Auth0 Dashboard pour vérifier si un tenant peut accéder aux options de connexion sans les nouveaux scopes et s’il doit être migré.

1. Accédez à [Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Faites défiler la page jusqu’à la section **Migrations**.
3. Repérez la bascule **Allow Connections Management** without Options Scopes :
   A. Si la bascule est **activée**, votre tenant peut toujours accéder aux options de connexion sans les nouveaux scopes, et vous devez effectuer la migration avant la date limite.
   B. Si la bascule est absente ou **désactivée**, aucune autre action n’est requise.

<div id="identify-affected-integrations">
  ## Identifier les intégrations concernées
</div>

Consultez les journaux `depnote` du tenant pour déterminer quelles applications et intégrations utilisent les points de terminaison concernés. L’exemple de requête suivant montre comment rechercher les journaux d’avis de dépréciation du tenant propres à cette dépréciation :

```
type:depnote AND description:Allow\ Connections\ Management\ without\ Options\ Scopes*
```

Pour en savoir plus, consultez [Syntaxe des requêtes de recherche dans les journaux](docs/deploy-monitor/logs/log-search-query-syntax).

Une fois que vous avez les résultats de la requête, examinez le champ `client_id` pour identifier chaque application susceptible d’être touchée. Compte tenu de la nature des changements, la présence d’un avis de dépréciation pour une application cliente donnée ne signifie pas nécessairement que l’application est touchée.

​Si le champ `client_id` est vide, cela signifie que la requête vers le point de terminaison touché provient d’une extension. Plusieurs extensions Auth0 peuvent générer dans les journaux du tenant des avis de dépréciation qui sont de faux positifs. Une fois les étapes propres aux extensions indiquées dans la section Mettre à jour les extensions terminées, vous pouvez ignorer les avis de dépréciation associés à un identifiant client vide.

<div id="update-application-grants">
  ## Mettre à jour les autorisations d&#39;application
</div>

Pour les applications qui récupèrent ou mettent à jour le champ `options` au moyen des points de terminaison concernés, mettez à jour leurs autorisations d’accès à la Management API afin d’inclure les scopes nécessaires.

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/dashboard/#/apis).
2. Sélectionnez **Auth0 Management API**, puis l’onglet **Machine to Machine Applications**.
3. À côté de l’application qui doit être mise à jour, sélectionnez l’icône de flèche vers le bas pour afficher la section.
4. Dans la section Permissions, recherchez `“connections_options”`. Activez l’un ou les deux nouveaux scopes. Ensuite, choisissez **Update** pour enregistrer vos modifications.
   <Frame>![Mettre à jour les permissions de la Management API](/docs/images/cdy7uua7fh8z/update-connections-permissions.png)</Frame>

<div id="update-extensions">
  ## Mettre à jour les extensions
</div>

Mettez à jour l’extension [Delegated Administration Dashboard](/docs/fr-CA/customize/extensions/delegated-administration-extension/install-delegated-admin-extension) vers sa version la plus récente. Cette mise à jour est nécessaire, car l’extension doit avoir accès au champ des options de connexion pour fonctionner correctement. Si vous utilisez cette extension, assurez-vous de la mettre à jour pour qu’elle continue de fonctionner correctement.

<div id="opt-out-finalize-migration">
  ## Désactivation (Finaliser la migration)
</div>

Une fois que vos intégrations ne dépendent plus du comportement obsolète, nous vous recommandons de configurer votre tenant pour refuser explicitement la gestion des connexions sans scopes Options. Vous pouvez ainsi choisir exactement quand votre tenant adoptera le nouveau comportement, ce qui vous donne davantage de contrôle sur votre processus de migration.

1. Accédez à [Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Dans la section Migrations, désactivez la bascule **Allow Connections Management without Options Scopes**.

Si vous rencontrez des problèmes, vous pouvez réactiver cette même bascule pour rétablir temporairement le comportement précédent pendant que vous corrigez le problème.
