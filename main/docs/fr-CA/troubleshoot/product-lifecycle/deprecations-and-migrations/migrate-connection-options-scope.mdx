title: Migrer vers les scopes des options de connexion de la Management API
---

​Auth0 introduit deux nouveaux [scopes](/docs/fr-CA/get-started/apis/scopes) de la Management API pour les options de connexion : `read:connections_options` et `update:connections_options`.

Nous retirons également progressivement la possibilité d’accéder au champ `options` ou de le mettre à jour sans les scopes appropriés, ce qui entraîne les changements non rétrocompatibles suivants :

* Les réponses des points de terminaison [Obtenir toutes les connexions](/docs/fr-CA/api/management/v2/connections/get-connections) et [Obtenir une connexion par ID](/docs/fr-CA/api/management/v2/connections/get-connections-by-id) n’incluent pas le champ `options`, à moins que l’appelant ait le scope `read:connections_options`.
* Les requêtes au point de terminaison [Mettre à jour une connexion](/docs/fr-CA/api/management/v2/connections/patch-connections-by-id) échouent si le corps de la requête contient le champ `options` et que l’appelant n’a pas le scope `update:connections_options`. Les réponses réussies n’incluent pas le champ `options`, à moins que l’appelant ait le scope `read:connections_options`.

Pour continuer à utiliser le champ `options`, vous devez mettre à jour vos autorisations d’application pour y inclure les nouveaux scopes. Suivez les étapes ci-dessous pour déterminer si votre tenant et vos intégrations sont touchés, migrer vers les scopes propres aux options et, enfin, vous soustraire au comportement obsolète.

<div id="identify-affected-tenants">
  ## Identifier les tenants touchés
</div>

Utilisez Auth0 Dashboard pour vérifier si un tenant est autorisé à accéder aux options de connection sans les nouveaux scopes et doit être migré.

1. Accédez à [Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Faites défiler la page jusqu’à la section **Migrations**.
3. Repérez la bascule **Allow Connections Management** without Options Scopes :
   A. Si la bascule est **activée**, votre tenant peut encore accéder aux options de connection sans les nouveaux scopes, et vous devez effectuer la migration avant la date limite.
   B. Si la bascule est absente ou **désactivée**, aucune autre action n’est requise.

<div id="identify-affected-integrations">
  ## Identifier les intégrations touchées
</div>

Examinez les journaux de tenant `depnote` pour déterminer quelles applications et intégrations utilisent les points de terminaison touchés. L’exemple de requête suivant montre comment interroger les journaux de tenant d’avis de dépréciation liés à cette dépréciation :

```
type:depnote AND description:Allow\ Connections\ Management\ without\ Options\ Scopes*
```

Pour en savoir plus, consultez [Log Search Query Syntax](docs/deploy-monitor/logs/log-search-query-syntax).

Une fois que vous avez les résultats de la requête, examinez le champ `client_id` pour identifier chaque application susceptible d’être touchée. En raison de la nature des changements, la présence d’un avis de dépréciation pour une application cliente donnée ne signifie pas nécessairement que l’application est touchée.

​Si le champ `client_id` est vide, cela signifie que la requête vers le endpoint touché provenait d’une extension. Plusieurs extensions Auth0 peuvent générer des tenant logs d’avis de dépréciation qui sont des faux positifs. Une fois que vous avez terminé les étapes propres aux extensions indiquées dans la section Update extensions, vous pouvez ignorer les avis de dépréciation dont l’identifiant client est vide.

<div id="update-application-grants">
  ## Mettre à jour les autorisations d’application
</div>

Pour les applications qui récupèrent ou mettent à jour le champ `options` à l’aide des points de terminaison touchés, mettez à jour leurs autorisations pour la Management API afin d’inclure les scopes nécessaires.

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/dashboard/#/apis).
2. Sélectionnez **Auth0 Management API**, puis l’onglet **Machine to Machine Applications**.
3. À côté de l’application à mettre à jour, sélectionnez l’icône de flèche vers le bas pour développer la section.
4. Dans la section Permissions, recherchez `“connections_options”`. Activez l’un ou les deux nouveaux scopes. Ensuite, choisissez **Update** pour enregistrer vos modifications.
   <Frame>![Mettre à jour les permissions de la Management API](/docs/images/cdy7uua7fh8z/update-connections-permissions.png)</Frame>

<div id="update-extensions">
  ## Mettre à jour les extensions
</div>

Mettez à jour l’extension [Delegated Administration Dashboard](/docs/fr-CA/customize/extensions/delegated-administration-extension/install-delegated-admin-extension) vers sa version la plus récente. Cette mise à jour est nécessaire, car l’extension doit avoir accès au champ option de connexion pour fonctionner correctement. Si vous utilisez cette extension, assurez-vous de la mettre à jour pour en préserver le bon fonctionnement.

<div id="opt-out-finalize-migration">
  ## Désactivation (finaliser la migration)
</div>

Une fois que vos intégrations ne dépendent plus du comportement déprécié, nous vous recommandons de configurer votre tenant pour désactiver explicitement la gestion des connexions sans scopes Options. Cela vous permet de choisir exactement quand votre tenant adopte le nouveau comportement, et vous donne ainsi plus de contrôle sur votre processus de migration.

1. Accédez à [Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Dans la section Migrations, désactivez l’option **Allow Connections Management without Options Scopes**.

Si vous rencontrez des problèmes, vous pouvez réactiver cette même bascule pour rétablir temporairement le comportement précédent pendant que vous corrigez le problème.
