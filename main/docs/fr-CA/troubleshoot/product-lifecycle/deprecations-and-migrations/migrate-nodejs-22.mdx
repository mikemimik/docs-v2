---
description: Décrit les éléments d’Auth0 Actions touchés par Node.js 18
title: Migrer de Node 18 à Node 22
---

À compter du 15 janvier 2025, l’environnement d’exécution Node 22 pour l’extensibilité est offert en disponibilité générale (GA) dans l’ensemble de notre offre d’extensibilité. Cela comprend Actions, Rules, Hooks, les scripts de base de données et les connexions sociales personnalisées. Nous vous encourageons fortement à passer à Node 22 dès que possible afin de respecter les meilleures pratiques en matière de sécurité du code.

<Warning>
  Le code de niveau module de vos Actions est maintenant exécuté une fois par instance de votre Action et peut être réutilisé, au besoin, dans plusieurs exécutions.
</Warning>

<div id="general-considerations">
  ### Considérations générales
</div>

* Pour connaître les modules offerts dans le nouvel environnement d’exécution, vous pouvez consulter [Can I require: Auth0 Extensibility](https://auth0-extensions.github.io/canirequire/).
* La directive `use npm` n’est plus prise en charge dans Node 22.
* L’accès en écriture au système de fichiers a été désactivé.

<div id="marketplace-integrations">
  ### Intégrations du Marketplace
</div>

<div id="social-connections-integrations">
  #### Intégrations des connexions sociales
</div>

Utilisez la [Management API](https://auth0.com/docs/api/management/v2) pour obtenir la liste complète des connexions sociales touchées par un changement de version de l’environnement d’exécution Node. Plus précisément, toutes les connexions sociales potentiellement touchées, qu’elles aient été créées explicitement comme connexion sociale personnalisée ou ajoutées au départ par l’intermédiaire du Marketplace, ont l’attribut `strategy` dont la valeur est `oauth1` ou `oauth2`.

Vous pouvez ensuite parcourir par pages toutes les connexions sociales personnalisées existantes dans un tenant donné à l’aide du endpoint [GET all connections](https://auth0.com/docs/api/management/v2#!/Connections/get_connections). Par exemple, les paramètres de requête suivants renvoient les noms et les identificateurs d’un maximum de 100 connexions sociales personnalisées :

```http wrap lines
/api/v2/connections?strategy=oauth1&strategy=oauth2&include_totals=true&fields=name&per_page=100
```

<Note>
  Le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ne permet pas de mettre à jour les scripts des connexions sociales personnalisées ajoutées via le Marketplace. Si une modification du script est nécessaire pour assurer la compatibilité avec Node 22, vous devez utiliser la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.
</Note>

<div id="migration-tasks">
  ## Tâches de migration
</div>

<div id="create-new-custom-actions">
  ### Créer de nouvelles Actions personnalisées
</div>

Pour créer une nouvelle Action personnalisée avec Node 22 à partir d’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Actions &gt; Library](https://manage.auth0.com/#/dashboard/actions/library).
2. Sélectionnez **Create Action &gt; Build from scratch**.
3. Dans le champ **Runtime***, sélectionnez **Node 22 (Recommended)**.
4. Rédigez vos Actions personnalisées en Node 22, testez-les et déployez-les quand elles sont prêtes.

<div id="upgrade-existing-custom-actions">
  ### Mettre à niveau les Actions personnalisées existantes
</div>

Vous pouvez mettre à niveau individuellement les Actions personnalisées existantes qui utilisent Node 12, 16 ou 18 vers Node 22 et [revenir à la version précédente](/docs/fr-CA/customize/actions/manage-versions#revert-actions) à l’aide de l’ancien environnement d’exécution. Pour mettre les Actions à niveau vers Node 22, créez et déployez une nouvelle version de la mise en œuvre existante avec les modifications requises, puis configurez-la pour utiliser Node 22 comme environnement d’exécution.

<div id="choose-node-22-for-other-extensibility-products">
  ### Choisir Node 22 pour les autres offres d’extensibilité
</div>

L’environnement d’exécution utilisé pour les autres offres d’extensibilité (hors Actions) est défini globalement dans les paramètres avancés du tenant. La modification de ce paramètre a une incidence simultanée sur les fonctionnalités suivantes :

* Rules
* Hooks
* scripts de base de données personnalisés
* scripts de connexion sociale personnalisée

Pour modifier le paramètre d’environnement d’exécution de l’extensibilité du tenant dans le Auth0 Dashboard :

1. Accédez à **Dashboard &gt; Settings &gt; Advanced**.
2. Faites défiler la page jusqu’à la section **Extensibility**.
3. Pour **Runtime**, sélectionnez **Node 22**.
4. Pour **Legacy Extensibility**, sélectionnez la version de **Runtime**, s’il y a lieu.

Comme il s’agit d’un paramètre global qui a une incidence sur plusieurs fonctionnalités d’extensibilité à la fois, nous vous recommandons d’effectuer d’abord cette étape dans votre tenant de développement, de terminer les tests de toutes les fonctionnalités d’extensibilité applicables, puis de passer à votre tenant de production seulement lorsque vous ne constatez aucun problème en développement.

Plus précisément, pour les scripts de base de données personnalisés, vous pouvez [suivre les étapes expliquées sur cette page](/docs/fr-CA/authenticate/database-connections/custom-db/custom-database-connections-scripts/environment) afin de vérifier individuellement un script avec une version précise de l’environnement d’exécution avant de modifier la version globale de l’environnement d’exécution.
