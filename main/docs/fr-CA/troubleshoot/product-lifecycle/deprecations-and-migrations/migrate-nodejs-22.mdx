---
description: Décrit les éléments d’Auth0 Actions concernés par Node.js 18
title: Migrer de Node 18 à Node 22
---

Depuis le 15 janvier 2025, le runtime d’extensibilité Node 22 est offert de façon générale (GA) dans l’ensemble de notre offre d’extensibilité. Cela comprend Actions, Rules, Hooks, les scripts de base de données et les connexions sociales personnalisées. Nous vous recommandons fortement de passer à Node 22 dès que possible afin de respecter les meilleures pratiques en matière de sécurité du code.

<Warning>
  Le code au niveau du module dans vos Actions n’est désormais exécuté qu’une seule fois par instance de votre Action et peut être réutilisé au besoin pour de nombreuses exécutions.
</Warning>

<div id="general-considerations">
  ### Considérations générales
</div>

* Pour connaître les modules offerts dans le nouveau runtime, vous pouvez consulter [Can I require: Auth0 Extensibility](https://auth0-extensions.github.io/canirequire/).
* La directive `use npm` n’est plus offerte dans Node 22.
* L’accès en écriture au système de fichiers a été désactivé.

<div id="marketplace-integrations">
  ### Intégrations du Marketplace
</div>

<div id="social-connections-integrations">
  #### Intégrations des connexions sociales
</div>

Utilisez la [Management API](https://auth0.com/docs/api/management/v2) pour obtenir la liste complète des connexions sociales touchées par un changement de version du runtime Node. Plus précisément, toutes les connexions sociales potentiellement touchées, qu’elles aient été créées explicitement comme connexion sociale personnalisée ou ajoutées initialement par l’entremise du Marketplace, ont l’attribut `strategy` dont la valeur est `oauth1` ou `oauth2`.

Vous pouvez ensuite parcourir, par pagination, toutes les connexions sociales personnalisées existantes dans un tenant donné à l’aide du endpoint [GET all connections](https://auth0.com/docs/api/management/v2#!/Connections/get_connections). Par exemple, les options de requête suivantes renvoient les noms et les identifiants d’un maximum de 100 connexions sociales personnalisées :

```http wrap lines
/api/v2/connections?strategy=oauth1&strategy=oauth2&include_totals=true&fields=name&per_page=100
```

<Note>
  Le <Tooltip tip="Auth0 Dashboard : le principal produit d'Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ne permet pas de mettre à jour les scripts des connexions sociales personnalisées ajoutées à partir du Marketplace. Si une modification du script est nécessaire pour assurer la compatibilité avec Node 22, vous devez utiliser la <Tooltip tip="Management API : un produit qui permet aux clients d'effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.
</Note>

<div id="migration-tasks">
  ## Tâches de migration
</div>

<div id="create-new-custom-actions">
  ### Créer de nouvelles Actions personnalisées
</div>

Pour créer une nouvelle Action personnalisée avec Node 22 dans l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Actions &gt; Library](https://manage.auth0.com/#/dashboard/actions/library).
2. Sélectionnez **Create Action &gt; Build from scratch**.
3. Dans le champ **Runtime***, sélectionnez **Node 22 (Recommended)**.
4. Rédigez vos Actions personnalisées en Node 22, testez-les, puis déployez-les lorsqu’elles sont prêtes.

<div id="upgrade-existing-custom-actions">
  ### Mettre à niveau les Actions personnalisées existantes
</div>

Vous pouvez mettre à niveau individuellement les Actions personnalisées existantes basées sur Node 12, 16 ou 18 vers Node 22 et [revenir à la version précédente](/docs/fr-CA/customize/actions/manage-versions#revert-actions) en utilisant l’ancien runtime. Mettez les Actions à niveau vers Node 22 en créant et en déployant une nouvelle version de l’implémentation existante, avec les modifications requises, puis en la configurant pour utiliser Node 22 comme runtime.

<div id="choose-node-22-for-other-extensibility-products">
  ### Choisir Node 22 pour les autres produits d’extensibilité
</div>

Le runtime utilisé pour les autres offres d’extensibilité (hors Actions) est défini globalement dans les paramètres avancés du tenant. La modification de ce paramètre a une incidence simultanée sur les fonctionnalités suivantes :

* Rules
* Hooks
* scripts de base de données personnalisés
* scripts de connexion sociale personnalisée

Pour modifier le paramètre de runtime d’extensibilité du tenant dans l’Auth0 Dashboard :

1. Accédez à **Dashboard &gt; Paramètres &gt; Avancé**.
2. Faites défiler jusqu’à la section **Extensibility**.
3. Pour **Runtime**, sélectionnez **Node 22**.
4. Pour **Legacy Extensibility**, sélectionnez la version de **Runtime**, le cas échéant.

Comme il s’agit d’un paramètre global qui a une incidence sur plusieurs fonctionnalités d’extensibilité à la fois, nous vous recommandons d’effectuer d’abord cette étape dans votre tenant de développement, de terminer les tests de toutes les fonctionnalités d’extensibilité applicables, puis de passer à votre tenant de production seulement lorsque vous ne constatez aucun problème en développement.

Plus précisément, pour les scripts Custom DB, vous pouvez [suivre les étapes expliquées sur cette page](/docs/fr-CA/authenticate/database-connections/custom-db/custom-database-connections-scripts/environment) afin de vérifier individuellement un script avec une version précise du runtime avant de modifier la version globale du runtime.
