description: Découvrez comment mettre à jour la configuration de votre connexion Auth0 pour remplacer l’utilisation du champ `enabled_clients` par des points de terminaison de connexion dédiés.
title: Migrer la gestion des clients activés vers des points de terminaison de connexion dédiés
validatedOn: 2026-04-20
---

Auth0 retire progressivement le champ `enabled_clients` des points de terminaison de la Management API [Get All Connections](/docs/fr-CA/api/management/v2/connections/get-connections), [Get Connections](/docs/fr-CA/api/management/v2/connections/get-connections-by-id) et [Update a Connection](/docs/fr-CA/api/management/v2/connections/patch-connections-by-id). Les clients Auth0 peuvent récupérer les clients activés pour une connexion à l’aide du point de terminaison [Get enabled clients for a connection](https://auth0.com/docs/api/management/v2/connections/get-connection-clients). Pour mettre à jour les clients déjà configurés pour une connexion, utilisez le point de terminaison [Update enabled clients for a connection](https://auth0.com/docs/api/management/v2/connections/patch-clients).

Pour vous préparer à ce changement et vous assurer que vos intégrations continuent de fonctionner sans problème, consultez les étapes ci-dessous pour vérifier si vous êtes concernés et migrer vers les nouveaux points de terminaison.

Pour en savoir plus sur les échéanciers de migration, consultez l’article sur les [Dépréciations et migrations](/docs/fr-CA/troubleshoot/product-lifecycle/deprecations-and-migrations).

<div id="verify-affected-tenants">
  ## Vérifier les tenants touchés
</div>

<Warning>
  Les administrateurs de tenant Private Cloud peuvent suivre ces instructions si leur [déploiement Private Cloud](/docs/fr-CA/deploy-monitor/deploy-private-cloud) est à la version `v202608` ou ultérieure. Pour en savoir plus sur les versions des déploiements Private Cloud, consultez [Comment vérifier le numéro de version et la date de déploiement de l’Auth0 Private Cloud](https://support.auth0.com/center/s/article/How-to-check-the-Auth0-private-cloud-s-release-number-and-deployment-date).
</Warning>

Vous pouvez utiliser l’Auth0 Dashboard pour vérifier si un tenant a accès à la fonctionnalité obsolète et doit être migré.

1. Accédez à [Auth0 Dashboard &gt; Paramètres du tenant &gt; Avancé](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Sous **Migrations**, vérifiez la **bascule Legacy Management of Connection&#39;s Enabled Clients**.
   * La bascule est **activée** : votre tenant autorise l’accès au champ obsolète lors de la récupération et de la mise à jour des connexions, et vous devez effectuer la migration avant la date limite.
   * La bascule est **désactivée** : aucune autre mesure n’est requise.

<div id="identify-affected-integrations">
  ## Identifier les intégrations concernées
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Les tenants Public Cloud ont commencé à recevoir ces avis de dépréciation dans la première moitié de février 2026.
  * Les tenants Private Cloud ont commencé à recevoir ces avis de dépréciation lorsque leurs déploiements Private Cloud ont atteint la version `v202608`.
</Callout>

Pour déterminer quelles applications et intégrations utilisent les points de terminaison obsolètes, consultez les [journaux](/docs/fr-CA/deploy-monitor/logs#logs) du tenant pour y repérer les avis de dépréciation.

Utilisez la requête suivante pour rechercher les avis propres à cette dépréciation :

```bash lines
type:depnote AND description:Legacy\ Management\ of\ Connection's\ Enabled\ Clients*
```

Examinez le résultat de la requête pour le champ `client_id` afin d’identifier les applications potentiellement touchées.

Certains services de la plateforme Auth0 et certaines extensions peuvent générer des avis de dépréciation dans les journaux du tenant :

* Si le champ `client_id` est vide, l’avis provient d’une extension ou de l’Auth0 Dashboard. Après avoir vérifié l’utilisation des extensions ci-dessous, vous pouvez ignorer les avis sans ID client.
* Si la valeur ne correspond à aucune des applications clientes du tenant, l’avis provient d’un service de la plateforme et peut être ignoré.

De plus, la présence d’un avis de dépréciation pour une application cliente donnée n’indique pas nécessairement que l’application est touchée. Par exemple, si l’une de vos applications récupère, au moyen de `GET`, des informations sur une connexion, y compris le champ `enabled_clients` obsolète, mais ne traite pas le champ renvoyé dans la réponse, la dépréciation prévue de ce champ n’a aucune incidence.

<div id="review-extensions-usage">
  ## Vérifier l’utilisation des extensions
</div>

Les extensions suivantes peuvent déclencher des avis de dépréciation dans les journaux du tenant :

* [User Import/Export](/docs/fr-CA/customize/extensions/user-import-export-extension#user-import-export-extension) : Cette extension ne dépend pas de la fonctionnalité obsolète. Ces avis sont des faux positifs qui se déclenchent lorsque l’extension récupère des renseignements sur la connexion.
* [Custom Social Connections](https://github.com/auth0/custom-social-connections) : Cette extension dépend de la fonctionnalité obsolète et pose des problèmes.
* [Delegated Administration Dashboard](/docs/fr-CA/customize/extensions/delegated-administration-extension/install-delegated-admin-extension) : Les anciennes versions de cette extension peuvent dépendre de la fonctionnalité obsolète et poser des problèmes. Mettez-la à jour vers la version 4.8 ou ultérieure à partir de l’Auth0 Dashboard.

Les deux premières extensions sont remplacées par des fonctionnalités de l’Auth0 Dashboard qui répondent à leurs cas d’utilisation et ne sont plus mises à jour.

* Utilisez la fonctionnalité [Bulk User Import / Export](/docs/fr-CA/manage-users/user-migration/bulk-user-import-export) du Dashboard pour remplacer l’extension User Import/Export.
* Utilisez la fonctionnalité [Social Identity Providers](/docs/fr-CA/authenticate/identity-providers/social-identity-providers#auth0-dashboard) du Dashboard pour remplacer l’extension Custom Social Connections.

<div id="update-your-integrations">
  ## Mettez à jour vos intégrations
</div>

Pour les applications et les intégrations qui utilisent les points de terminaison de connexion concernés pour récupérer (`GET`) ou mettre à jour (`PATCH`) le champ `enabled_clients`, vous devez les modifier afin d’utiliser les points de terminaison dédiés à la gestion des clients :

* **Pour lire :** [Obtenir les clients activés pour une connexion](https://auth0.com/docs/api/management/v2/connections/get-connection-clients).
* **Pour écrire :** [Mettre à jour les clients activés pour une connexion](https://auth0.com/docs/api/management/v2/connections/patch-clients).

<div id="update-sdks-and-tools">
  ## Mettre à jour les SDK et les outils
</div>

Les versions suivantes des SDK et des outils prennent en charge les nouveaux points de terminaison :

* [node-auth0](https://github.com/auth0/node-auth0): `4.25.0+`
* [go-auth0](https://github.com/auth0/go-auth0/): `1.21.0+`
* [auth0-java](https://github.com/auth0/auth0-java): `2.21.0+`
* [auth0-python](https://github.com/auth0/auth0-python): `5.1.0+`
* [auth0.net](https://github.com/auth0/auth0.net): `7.38.0+`
* [Auth0 Deploy CLI](https://auth0.com/docs/deploy-monitor/deploy-cli-tool): `8.27.0+`
* [Auth0 Terraform Provider](https://auth0.com/docs/deploy-monitor/auth0-terraform-provider): `1.29.0+`

Auth0 Deploy CLI et Auth0 Terraform Provider n’exigent aucune modification à la façon dont les connexions et les clients correspondants sont représentés dans leur modèle de données. Vous devez utiliser la version la plus récente offerte pour garantir que l’outil utilise les points de terminaison dédiés.

Les SDK Ruby et PHP de la Management API sont en attente de mise à jour.

<div id="opt-out-finalize-migration">
  ## Désactivation (finaliser la migration)
</div>

Une fois que vous vous êtes assuré que les intégrations ne dépendent plus du comportement obsolète, vous devriez configurer le tenant pour le désactiver explicitement.

* **Pourquoi désactiver tôt ?** Cela vous permet de contrôler le moment exact où un tenant donné passe au nouveau comportement. En cas de situation imprévue, vous pouvez rapidement rétablir la bascule pour restaurer le comportement d’origine.
* **Comment désactiver :** Accédez à **[Auth0 Dashboard &gt; Paramètres du tenant &gt; Avancé](https://manage.auth0.com/dashboard/#/tenant/advanced).**  Sous Migrations, désactivez l’option **Legacy Management of Connection&#39;s Enabled Clients**.
