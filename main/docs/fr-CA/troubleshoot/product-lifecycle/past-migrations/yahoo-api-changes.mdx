---
description: Décrit les changements apportés à l’API Yahoo qui touchent la structure du profil utilisateur.
title: Modifications de l’API Yahoo
---

Yahoo a modifié l’API que les applications doivent utiliser pour récupérer le profil utilisateur : au lieu de la [Social Directory API](https://developer.yahoo.com/oauth/social-directory-eol/), elles doivent maintenant utiliser l’endpoint Yahoo `/userinfo`. Ce changement signifie que la structure du profil utilisateur des utilisateurs Yahoo dans Auth0 changera.

Auparavant, Auth0 chargeait toutes les données de profil renvoyées par Yahoo et ajoutait ces champs supplémentaires, mappés à partir du profil Yahoo.

Yahoo a cessé de renvoyer `url`, `profileUrl`, `isConnected` et plusieurs autres champs répertoriés dans la [documentation de Yahoo](https://developer.yahoo.com/oauth/social-directory-eol/) (voir « List Of Attributes Deprecated in Social Directory Profile Api »). Ces autres champs ne feront donc plus partie du profil.

L’endpoint Yahoo `/userinfo` renvoie différents champs selon les permissions d’API que vous configurez dans la définition de la [Yahoo Application](https://developer.yahoo.com/apps/). Yahoo vous permet d’accorder l’une des quatre permissions dans la section **Profile (Social Directory)** des permissions :

* Lecture publique de base
* Lecture publique étendue
* Lecture et écriture publiques
* Lecture et écriture publiques et privées

<div id="actions">
  ## Actions
</div>

Lorsque vous configurez la connexion Yahoo dans votre <Tooltip tip="Auth0 Dashboard : interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, sélectionnez l’attribut qui correspond aux permissions que vous avez accordées dans votre configuration Yahoo. Si vous choisissez un attribut qui ne correspond pas à ce que vous avez indiqué dans Yahoo, la transaction de connexion échouera.

Si vous ne sélectionnez aucune permission dans les paramètres de la connexion Auth0, Auth0 demandera par défaut le scope `openid`, qui renvoie les champs de profil correspondant à la permission d’API que vous avez spécifiée dans la Yahoo Application. Par exemple, si votre application Yahoo est configurée avec la permission **Read Public Extended**, Yahoo renvoie les champs suivants :

* `sub, name`
* `given_name`
* `family_name`
* `locale`
* `email`
* `email_verified`
* `birthdate`
* `profile_images`
* `picture`
* `preferred_username`
* `phone_number`
* `nickname`

Si vous utilisez la connexion Yahoo pour authentifier des utilisateurs et obtenir leurs informations de base, votre application continuera de fonctionner sans changement.

Si votre application accède à des champs du profil utilisateur qui ne sont plus disponibles, vous devrez alors activer le bon **Connection Attribute** dans l’Auth0 Dashboard et ajuster le code de votre application pour utiliser les bons noms de champs.
