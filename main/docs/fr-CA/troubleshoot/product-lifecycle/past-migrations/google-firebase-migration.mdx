title: Migrer de Google Cloud Messaging vers Firebase Cloud Messaging
---

Depuis le 11 avril 2019, [Google a déclaré Google Cloud Messaging (GCM) obsolète](https://firebase.googleblog.com/2018/04/time-to-upgrade-from-gcm-to-fcm.html) et l’a remplacé par Firebase Cloud Messaging (FCM).

<div id="features-affected">
  ## Fonctionnalités concernées
</div>

Les SDK Guardian d’Auth0 pour iOS et Android vous aident à créer des applications mobiles personnalisées avec les fonctionnalités Guardian, afin d’offrir un accès sécurisé à l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> (MFA) au moyen de notifications push.

La bibliothèque Guardian Android SDK d’Auth0 a été conçue pour envoyer des notifications push au moyen de GCM. Notez que les applications existantes devraient [continuer de fonctionner telles quelles](https://aws.amazon.com/blogs/messaging-and-targeting/the-end-of-google-cloud-messaging-and-what-it-means-for-your-apps/).

<div id="actions">
  ## Actions
</div>

Consultez [la documentation de Google](https://developers.google.com/cloud-messaging/android/android-migrate-fcm) pour savoir comment migrer de GCM vers FCM.

La principale différence entre l’envoi de notifications à GCM et à FCM réside dans le payload reçu dans la notification. S’il était possible pour les clients existants qui utilisaient l’Android SDK d’adapter le payload reçu avant d’appeler la méthode du SDK, nous avons mis à niveau la bibliothèque pour qu’elle accepte le nouveau payload, ce qui facilite l’adoption de FCM. Plus de détails [ici](https://github.com/auth0/Guardian.Android/pull/84).

La version 0.4.0 du Guardian Android SDK est déjà offerte dans Maven Central et inclut cette modification. L’application d’exemple a aussi été mise à niveau et peut être testée en fournissant le fichier `google-services.json` et un `guardian-url`.
