---
description: Décrit comment utiliser les paramètres de pagination pour les requêtes adressées à certains points de terminaison de Management API v2.
title: Migrer vers les requêtes paginées des points de terminaison de Management API v2
---

Après le **26 janvier 2021** (ou la version 2202 de Private Cloud), les requêtes envoyées aux points de terminaison v2 de <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> renverront un maximum de 50 éléments pour les tenants dans le Public Cloud. Pour récupérer plus d’éléments, vous devez inclure les paramètres `page` et `per_page`. À partir du **21 juillet 2020** (ou de la version 2108 de Private Cloud), Auth0 affichera les journaux du tenant ainsi qu’une bascule de migration pour vous aider à vous préparer à ce changement.

Les tenants touchés sont ceux qui répondent aux critères suivants :

* Tenants Public Cloud créés **avant le 21 juillet 2020** ou déploiements Private Cloud en version 2108
* Effectuent activement des appels aux points de terminaison touchés sans transmettre le paramètre `per_page` pour des requêtes pouvant renvoyer plus d’un résultat.

Les tenants suivants adopteront par défaut le nouveau comportement de pagination et continueront de fonctionner comme aujourd’hui après le 26 janvier 2021 :

* Tenants Public Cloud créés **le 21 juillet 2020 ou après** ou déploiements Private Cloud en version 2108 ou ultérieure
* N’utilisent aucun des points de terminaison touchés
* Utilisent les points de terminaison touchés et transmettent le paramètre `per_page`, ou effectuent des requêtes qui renverront toujours un seul résultat.

<div id="endpoints-affected">
  ## Points de terminaison affectés
</div>

Les appels aux points de terminaison suivants de la Management API v2 sont affectés :

* [`GET /api/v2/clients`](https://auth0.com/docs/api/management/v2#!/Clients/get_clients)
* [`GET /api/v2/client_grants`](https://auth0.com/docs/api/management/v2#!/Clients/client_grants)
* [`GET /api/v2/grants`](https://auth0.com/docs/api/management/v2#!/Clients/grants)
* [`GET /api/v2/connections`](https://auth0.com/docs/api/management/v2#!/Clients/connections)
* [`GET /api/v2/device-credentials`](https://auth0.com/docs/api/management/v2#!/Clients/device_credentials)
* [`GET /api/v2/resource-servers`](https://auth0.com/docs/api/management/v2#!/Clients/resource_servers)
* [`GET /api/v2/rules`](https://auth0.com/docs/api/management/v2#!/Clients/rules)

Des avis de dépréciation seront consignés dans le journal du tenant pour toutes les requêtes sans options de pagination qui retournent actuellement plus d’un élément, à raison d’une fois par heure, pour chaque combinaison différente de client et de point de terminaison.

<div id="actions">
  ## Actions
</div>

1. Remplacez toutes les appels aux points de terminaison concernés en fournissant les paramètres `page` et `per_page`.

| Paramètre  | Type    | Description                                                                                      |
| ---------- | ------- | ------------------------------------------------------------------------------------------------ |
| `page`     | Integer | Index de la page de résultats à retourner. La première page est 0.                               |
| `per_page` | Integer | Nombre de résultats par page. La pagination est désactivée si ce paramètre n&#39;est pas envoyé. |

2. Assurez-vous que vous ne voyez plus d&#39;avis de dépréciation dans les journaux de votre tenant. Vérifiez si une requête a retourné plus de 50 éléments. Consultez le champ `details.size_exceeded` et vérifiez s&#39;il est défini à `true`.

   1. Utilisez la requête de journal suivante pour retourner toutes les appels sans options de pagination qui renvoient plus d&#39;un résultat :
      `type:depnote AND description:*Unpaginated*`
   2. Utilisez la requête de journal suivante pour retourner toutes les appels sans options de pagination qui renvoient plus de 50 résultats :
      `type:depnote AND description:*Unpaginated* AND details.size_exceeded:true`

   Pour identifier l&#39;application qui effectue la requête, les journaux incluront le `client_id` utilisé pour l&#39;effectuer. Vous pouvez aussi trouver le point de terminaison utilisé dans le champ `details.path` des journaux.
3. Désactivez les requêtes non paginées de la Management API pour votre tenant. Accédez à [Dashboard &gt; Tenant Settings &gt; Advanced &gt; Migration](https://manage.auth0.com/#/tenant/advanced). Cela simulera le comportement attendu une fois la fenêtre de migration terminée, de sorte que les appels aux points de terminaison concernés retournent jusqu&#39;à 50 résultats. Vous pourrez réactiver les requêtes non paginées à tout moment avant cette date.

<div id="update-extensions">
  ### Mettre à jour les extensions
</div>

Vous devrez peut-être mettre à jour les versions antérieures de [Auth0 Extensions](/docs/fr-CA/customize/extensions) et les extensions personnalisées vers les versions les plus récentes afin de vous assurer qu’elles n’effectuent que des requêtes paginées.

1. Vérifiez le journal du tenant pour repérer les avis de dépréciation associés à des clients dont l’ID correspond à l’URL d’une extension. Cela signifie que vous devrez mettre cette extension à jour.
2. Accédez à [Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions), sélectionnez **Installed Extensions**, puis cliquez sur le lien **Update** de l’extension, s’il est présent.

<div id="update-deploy-cli-tool">
  ### Mettre à jour l’outil Deploy CLI
</div>

Si vous utilisez l’outil `auth0-deploy-cli`, vous devez le mettre à jour vers la version **5.3.0 ou une version ultérieure**. Pour en savoir plus, consultez [l’outil Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool).
