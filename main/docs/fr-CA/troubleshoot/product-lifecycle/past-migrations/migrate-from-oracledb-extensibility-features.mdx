title: Migrer à partir des fonctionnalités d’extensibilité oracledb
---

À compter du 21 juin 2023, Auth0 ne prendra plus en charge la connexion aux bases de données Oracle à partir de Node.js dans les fonctionnalités d’extensibilité. Auth0 prenait auparavant en charge la connexion aux bases de données Oracle au moyen du module Node.js `oracledb`.

<div id="affected-workflows">
  ## Flux de travail touchés
</div>

Passez en revue vos fonctionnalités d’extensibilité pour repérer le code qui contient `require(oracledb)` ou  `require('oracledb@4.2.0')`.

Passez en revue les flux ou fonctionnalités ci-dessous qui permettent d’exécuter du code d’extensibilité afin d’y repérer des références à [oracledb](https://www.npmjs.com/package/oracledb) :

* [Hooks](/docs/fr-CA/customize/hooks)
* [Rules](/docs/fr-CA/customize/rules)
* [Custom Database Scripts](/docs/fr-CA/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Custom Social Connections](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Il n’existe aucun remplacement officiel prévu pour cette fonctionnalité. Auth0 recommande de transférer la logique `oracledb` vers un service distinct dans votre propre infrastructure. Après avoir créé ce service dans votre infrastructure, vous pouvez utiliser une fonction d’Extensibilité pour envoyer une requête HTTP qui déclenche la logique de la base de données Oracle.

Pour en savoir plus sur l’implémentation du module `oracledb` dans votre propre infrastructure, consultez le [guide Quickstart d’OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Par exemple, dans un script Login de Custom Database qui utilise `oracledb`, supprimez la fonction qui appelle `oracledb` et utilisez plutôt un client HTTP générique pour récupérer la réponse.

Pour plus d’informations sur la création d’Actions ou la migration vers Actions, [consultez la documentation Auth0](/docs/fr-CA/customize/actions).
