title: Migrer depuis les fonctionnalités d’extensibilité `oracledb`
---

À compter du 21 juin 2023, Auth0 ne prendra plus en charge les connexions aux bases de données Oracle depuis Node.js dans les fonctionnalités d’extensibilité. Auparavant, Auth0 prenait en charge les connexions aux bases de données Oracle au moyen du module Node.js `oracledb`.

<div id="affected-workflows">
  ## Flux de travail concernés
</div>

Vérifiez vos fonctionnalités d’extensibilité pour repérer tout code contenant `require(oracledb)` ou  `require('oracledb@4.2.0')`.

Examinez les flux ou fonctionnalités ci-dessous qui permettent d’utiliser du code d’extensibilité afin d’y repérer des références à [oracledb](https://www.npmjs.com/package/oracledb) :

* [Hooks](/docs/fr-CA/customize/hooks)
* [Rules](/docs/fr-CA/customize/rules)
* [Scripts de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Il n’existe aucun remplacement officiel prévu pour cette fonctionnalité. Auth0 recommande plutôt de déplacer la logique `oracledb` vers un service distinct dans votre propre infrastructure. Une fois ce service créé dans votre infrastructure, vous pouvez utiliser une fonction d’Extensibilité pour envoyer une requête HTTP qui déclenchera la logique Oracle Database.

Pour en savoir plus sur l’implémentation du module `oracledb` dans votre propre infrastructure, consultez le [guide Quickstart OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Par exemple, dans un script Login d’une base de données personnalisée qui utilise `oracledb`, supprimez la fonction qui appelle `oracledb` et remplacez-la par un client HTTP générique pour récupérer la réponse.

Pour en savoir plus sur la création d’Actions ou sur la migration vers Actions, [consultez la documentation Auth0](/docs/fr-CA/customize/actions).
