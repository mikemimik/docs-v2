title: Migrer à partir des fonctionnalités d’extensibilité oracledb
---

À compter du 21 juin 2023, Auth0 n’offrira plus de soutien pour les connexions aux bases de données Oracle à partir de Node.js dans les fonctionnalités d’extensibilité. Auth0 offrait auparavant la prise en charge des connexions aux bases de données Oracle au moyen du module Node.js `oracledb`.

<div id="affected-workflows">
  ## Flux de travail touchés
</div>

Passez en revue vos fonctionnalités d’extensibilité pour repérer le code qui contient `require(oracledb)` ou `require('oracledb@4.2.0')`.

Passez en revue les flux ou fonctionnalités suivants qui permettent d’exécuter du code d’extensibilité afin de repérer des références à [oracledb](https://www.npmjs.com/package/oracledb) :

* [Hooks](/docs/fr-CA/customize/hooks)
* [Rules](/docs/fr-CA/customize/rules)
* [Scripts de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Aucun remplacement officiel n’est prévu pour cette fonctionnalité. Auth0 recommande de remplacer la logique `oracledb` par un service distinct au sein de votre propre infrastructure. Une fois ce service créé dans votre infrastructure, vous pouvez utiliser une fonction d’Extensibilité pour envoyer une requête HTTP qui déclenchera la logique Oracle Database.

Pour en savoir plus sur la façon d’implémenter le module `oracledb` dans votre propre infrastructure, consultez le [guide Quickstart OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Par exemple, dans un script Login de base de données personnalisée qui utilise `oracledb`, supprimez la fonction qui appelle `oracledb` et remplacez-la par un client HTTP générique pour récupérer la réponse.

Pour en savoir plus sur la création d’Actions ou la migration vers Actions, [consultez la documentation Auth0](/docs/fr-CA/customize/actions).
