title: Migrer depuis les fonctionnalités d’extensibilité oracledb
---

Depuis le 21 juin 2023, Auth0 ne prend plus en charge la connexion à des bases de données Oracle depuis Node.js dans les fonctionnalités d’extensibilité. Auth0 prenait auparavant en charge la connexion à des bases de données Oracle au moyen du module Node.js `oracledb`.

<div id="affected-workflows">
  ## Flux de travail touchés
</div>

Passez en revue vos fonctionnalités d’extensibilité pour repérer le code qui contient `require(oracledb)` ou `require('oracledb@4.2.0')`.

Passez en revue les flux ou fonctionnalités suivants qui permettent d’ajouter du code d’extensibilité afin d’y repérer des références à [oracledb](https://www.npmjs.com/package/oracledb) :

* [Hooks](/docs/fr-CA/customize/hooks)
* [Rules](/docs/fr-CA/customize/rules)
* [Scripts de base de données personnalisés](/docs/fr-CA/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Il n’existe aucun remplacement officiel prévu pour cette fonctionnalité. Auth0 recommande de déplacer la logique `oracledb` vers un service distinct au sein de votre propre infrastructure. Une fois ce service créé dans votre infrastructure, vous pouvez utiliser une fonction d’Extensibilité pour effectuer une requête HTTP qui déclenchera la logique Oracle Database.

Pour en savoir plus sur la façon d’implémenter le module `oracledb` dans votre propre infrastructure, consultez le [guide Quickstart d’OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Par exemple, dans un script Login d’une base de données personnalisée utilisant `oracledb`, remplacez la fonction qui appelle `oracledb` par un client HTTP générique pour récupérer la réponse.

Pour en savoir plus sur la création d’Actions ou la migration vers Actions, [consultez la documentation Auth0](/docs/fr-CA/customize/actions).
