title: Migrer depuis les extensions de journaux
---

Les extensions de journaux Auth0 ci-dessous sont maintenant obsolètes et seront prises en charge jusqu’aux dates de fin de vie (EOL) indiquées plus bas. Il n’est plus possible de créer de nouvelles extensions à partir de cette liste d’extensions de journaux obsolètes. Vous pouvez obtenir une fonctionnalité équivalente à l’aide de flux d’événements de journalisation ou d’intégrations sur l’[Auth0 Marketplace](https://marketplace.auth0.com/)

Les éléments suivants atteindront leur fin de vie dans le Private Cloud le **6 janvier 2023** :

* Auth0 Authentication API Webhooks
* Auth0 <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> Webhooks
* Logs to Cloudwatch
* Logs to Logentries
* Logs to Loggly
* Logs to Logstash
* Logs to Papertrail
* Logs to Splunk
* Logs to Sumo Logic

Les éléments suivants atteindront leur fin de vie dans le Public Cloud le **2 mai 2023** :

* Auth0 Authentication API Webhooks
* Auth0 Management API Webhooks
* Logs to Cloudwatch
* Logs to Logentries
* Logs to Loggly
* Logs to Logstash
* Logs to Papertrail
* Logs to Splunk
* Logs to Sumo Logic
* Logs to Segment
* Logs to Mixpanel
* Logs to AppInsights
* Logs to Azure Blob Storage

À compter des dates de fin de vie indiquées, Auth0 n’offrira plus de soutien pour les extensions de journaux installées figurant dans la liste ci-dessus. Sur cette page, vous trouverez des instructions pour migrer depuis des extensions de journaux précises.

<div id="auth0-authentication-api-webhooks">
  ## Webhooks de l’Auth0 Authentication API
</div>

Assurez-vous que votre destination peut traiter un tableau d’objets de journal.

1. [Connectez-vous à votre Auth0 Dashboard](https://manage.auth0.com/).
2. Accédez à **Extensions** et sélectionnez votre **webhook Auth0 Authentication API** dans l’onglet **Installed Extensions**.
3. Copiez le **Webhook URL** et l’**Authorization Header** (si vous en avez un).
4. Ensuite, accédez à **Monitoring &gt; Streams &gt; New Event Stream**.
5. Sélectionnez [**Custom Webhook**](/docs/fr-CA/customize/log-streams/custom-log-streams), entrez un nom, puis cliquez sur **Create**.
6. Configurez votre webhook.

   * Dans le champ **Payload URL**, entrez l’URL du webhook que vous avez copiée à partir de l’extension.
   * Si vous en avez un, définissez **Authorization Token** sur l’**Authorization Header** que vous avez copié à partir de l’extension.
   * Réglez **Content Format** sur JSON Array.
7. Cliquez sur **Save**.
8. Retournez à **Extensions &gt; Installed Extensions** et désactivez votre extension **Auth0 Authentication API webhooks**.

<div id="auth0-management-api-webhooks">
  ## Webhooks de l’Auth0 Management API
</div>

Assurez-vous que votre destination peut traiter un tableau d’objets de journal.

1. [Connectez-vous à votre Auth0 Dashboard](https://manage.auth0.com/).
2. Allez à **Extensions** et sélectionnez votre **webhook Auth0 Management API** dans l’onglet **Installed Extensions**.
3. Copiez le **Webhook URL** et le **Authorization Header** (si vous en avez un).
4. Allez à **Monitoring &gt; Streams &gt; New Event Stream**.
5. Sélectionnez [**Custom Webhook**](/docs/fr-CA/customize/log-streams/custom-log-streams), entrez un nom, puis cliquez sur **Create**.
6. Configurez votre webhook.

   * Dans le champ **Payload URL**, entrez l’URL du webhook que vous avez copiée depuis l’extension.
   * Si vous en avez un, définissez le **Authorization Token** à la valeur du **Authorization Header** que vous avez copié depuis l’extension.
   * Réglez **Content Format** sur JSON Array.
7. Cliquez sur **Save**.
8. Retournez à **Extensions &gt; Installed Extensions** et désactivez votre extension **Auth0 Management API webhooks**.

<div id="cloudwatch">
  ## CloudWatch
</div>

Configurez l’[intégration Amazon EventBridge](https://marketplace.auth0.com/integrations/amazon-log-streaming) offerte dans l’Auth0 Marketplace. Une fois les événements disponibles sur votre bus d’événements, [créez des règles](https://docs.aws.amazon.com/eventbridge/latest/userguide/create-event-bus.html) pour acheminer ces événements vers AWS CloudWatch. Assurez-vous de désactiver votre extension Auth0 Logs to CloudWatch :

1. [Connectez-vous à votre Auth0 Dashboard](https://manage.auth0.com/).
2. Accédez à **Extensions &gt; Installed Extensions**.
3. Désactivez l’extension **Auth0 Logs to CloudWatch**.

<div id="loggly">
  ## Loggly
</div>

1. [Connectez-vous à votre compte Loggly](https://www.loggly.com/login/).
2. Ouvrez le menu **Logs** et sélectionnez **Source Setup**.
3. Accédez à l’onglet **Customer Tokens**.
4. Copiez votre jeton.
5. [Connectez-vous à votre Auth0 Dashboard](https://manage.auth0.com/).
6. Accédez à **Monitoring &gt; Streams &gt; New Event Stream**.
7. Sélectionnez [**Custom Webhook**](/docs/fr-CA/customize/log-streams/custom-log-streams), saisissez un nom, puis cliquez sur **Create**.
8. Configurez votre webhook.

   * Pour **Payload URL**, remplacez `LOGGLY_TOKEN` par le jeton copié depuis votre compte Loggly : `https://logs-01.loggly.com/bulk/{LOGGLY_TOKEN}/tag/auth0-{TENANT_NAME}/`.
   * Laissez **Authorization Token** vide.
   * Réglez **Content Type** sur `application/json`.
   * Réglez **Content Format** sur JSON Lines.
9. Cliquez sur **Save**.
10. Accédez à **Extensions &gt; Installed Extensions** et désactivez votre extension **Auth0 Logs to Loggly**.

<div id="logstash">
  ## Logstash
</div>

1. [Connectez-vous à votre Auth0 Dashboard](https://manage.auth0.com/).
2. Accédez à **Monitoring &gt; Streams &gt; New Event Stream**.
3. Sélectionnez [**Custom Webhook**](/docs/fr-CA/customize/log-streams/custom-log-streams), entrez un nom, puis cliquez sur **Create**.
4. Configurez votre webhook.

   * Dans le champ **Payload URL**, entrez l’URL de votre serveur Logstash.
   * Définissez le **Authorization Token** sur `Basic <CREDENTIALS>`, en remplaçant `CREDENTIALS` par l’encodage Base64 de votre nom d’utilisateur et de votre mot de passe Logstash, séparés par un deux-points (`<USER:PASSWORD>`). Vous trouverez vos identifiants HTTP Logstash dans votre fichier de configuration Logstash.
5. Cliquez sur **Save**.
6. Accédez à **Extensions &gt; Installed Extensions** et désactivez votre extension **Auth0 Logs to Logstash**.

<div id="papertrail">
  ## Papertrail
</div>

1. Connectez-vous à Papertrail.
2. Accédez à [Log Destinations](https://papertrailapp.com/account/destinations), puis cliquez sur [Create Log Destination](https://papertrailapp.com/destinations/new).
3. Saisissez une description et configurez les paramètres de la destination.
4. Dans le volet **Accept Connections via ...**, sélectionnez **Token**.
5. Cliquez sur **Create**.
6. [Connectez-vous à votre Auth0 Dashboard](https://manage.auth0.com/).
7. Accédez à **Monitoring &gt; Streams &gt; New Event Stream**.
8. Sélectionnez [**Custom Webhook**](/docs/fr-CA/customize/log-streams/custom-log-streams), entrez un nom, puis cliquez sur **Create**.
9. Configurez votre webhook.

   * Définissez **Payload URL** sur `https://logs.collector.solarwinds.com/v1/logs`.
   * Définissez **Authorization Token** sur `Basic <CREDENTIALS>`, en remplaçant `CREDENTIALS` par l’encodage Base64 du token Papertrail obtenu dans les paramètres de destination de Papertrail.
   * Définissez **Content Format** sur JSON Lines.
10. Cliquez sur **Save**.
11. Accédez à **Extensions &gt; Installed Extensions** et désactivez votre extension **Auth0 Logs to Papertrail**.

<div id="segment">
  ## Segment
</div>

1. Connectez-vous à votre Auth0 Dashboard
2. Dans **Extensions &gt; Installed Extensions**, repérez votre extension Segment.
3. Prenez en note votre **Write Key**, car vous en aurez besoin pour votre nouveau log flux vers Segment.
4. Désactivez votre extension Segment.
5. Prenez en note l’horodatage UTC du dernier log reçu dans Segment. Cet horodatage vous permettra de créer votre nouveau flux tout en évitant les logs en double dans Segment.
6. Accédez à **Monitoring &gt; Log Streams**, puis créez un nouveau log flux Segment. Entrez votre **Write Key** dans le champ approprié ainsi que l’horodatage noté dans la section **Start From**.
7. Créez le flux

<div id="mixpanel">
  ## Mixpanel
</div>

1. Connectez-vous à votre compte Mixpanel, puis ouvrez **Project Settings**. Prenez en note votre **Project ID** et la **Data Residency** (la région).
2. Mixpanel ne recommande plus d’utiliser le jeton de votre projet pour l’importation d’événements et recommande plutôt d’utiliser le mécanisme de compte de service. Dans la page **Project Settings**, sélectionnez **Service Accounts**, puis créez un nouveau compte de service avec des permissions d’administrateur ou de propriétaire. Prenez en note les identifiants.
3. Connectez-vous à votre Auth0 Dashboard.
4. Dans **Monitoring &gt; Log Streams**, créez un nouveau flux Mixpanel. Entrez l’ID du projet, la région et les identifiants du compte de service, puis créez le flux.
5. N’oubliez pas de désactiver votre ancienne extension. Allez dans **Extensions &gt; Installed Extensions** et désactivez l’**extension Auth0 Logs to Mixpanel**.

<div id="appinsights">
  ## AppInsights
</div>

1. Configurez un [Azure EventGrid Log flux](https://marketplace.auth0.com/integrations/azure-log-streaming) ou utilisez votre flux EventGrid existant.
2. Suivez les [instructions publiées par Azure](https://urldefense.com/v3/__https://learn.microsoft.com/en-us/azure/event-grid/auth0-log-stream-app-insights__;!!PwKahg!9T9C3qhBRgQQHChQkQqnmHDDnX99yWJ4W8F4rLVs6qrwwbnFCY3JcCfF486d1DLqRFEghVnJZ9gJnronDv0$) pour définir une destination pour les événements de journalisation.
3. Les journaux peuvent être transformés différemment par la diffusion des journaux. Consultez le [code de Log Extenstion](https://github.com/auth0-extensions/auth0-logs-to-provider/blob/master/server/lib/senders/appinsights.js), maintenant obsolète, pour en savoir plus.

Transformations notables apportées aux journaux :

* Le client appInsight est remplacé :

  * Définit une heure personnalisée correspondant à l’heure du journal
  * Définit le système d’exploitation et sa version pour qu’ils correspondent aux informations du journal
  * Définit la balise du type d’appareil pour indiquer si le journal provient d’une source mobile
  * Définit la balise d’adresse IP pour qu’elle corresponde aux informations du journal
  * Définit les balises user id, account id, user agent et auth user id pour qu’elles correspondent aux journaux
* L’enregistrement est nettoyé afin de supprimer les propriétés suivantes si elles sont vides :

  * ip
  * user&#95;id
  * user&#95;name
  * connection
  * client&#95;name
  * description
* La propriété isMobile de l’enregistrement est modifiée pour contenir une chaîne « yes » ou « no »
* La propriété details de l’enregistrement est modifiée pour être une chaîne
* La propriété details de l’enregistrement est tronquée à 8185 caractères

<div id="azure-blob-storage">
  ## Azure Blob Storage
</div>

1. Configurez un [flux de journaux Azure EventGrid](https://marketplace.auth0.com/integrations/azure-log-streaming) ou utilisez votre flux EventGrid existant.
2. Suivez les [instructions publiées par Azure](https://urldefense.com/v3/__https://learn.microsoft.com/en-us/azure/event-grid/auth0-log-stream-blob-storage__;!!PwKahg!9T9C3qhBRgQQHChQkQqnmHDDnX99yWJ4W8F4rLVs6qrwwbnFCY3JcCfF486d1DLqRFEghVnJZ9gJ5k9lKMM$) pour définir une destination pour les événement de journalisation.
3. Les journaux peuvent être transformés différemment par la diffusion des journaux. Veuillez consulter le [code de Log Extenstion](https://github.com/auth0-extensions/auth0-logs-to-provider/blob/master/server/lib/senders/blobstorage.js) obsolète pour plus de détails

Parmi les transformations notables apportées aux journaux, on compte :

* Les champs `type_code` et `type` de l’enregistrement sont ajustés. `Type` correspond à la description en clair de `type_code`
* Les champs `os`, `os_version`, `device` et `device_version` de l’enregistrement sont ajustés si l’enregistrement contient des informations `user_agent`
