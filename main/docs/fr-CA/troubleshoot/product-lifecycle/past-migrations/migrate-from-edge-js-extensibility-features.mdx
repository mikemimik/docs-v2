title: Migrer à partir des fonctionnalités d’extensibilité edge.js
---

À compter du 21 juin 2023, Auth0 ne prendra plus en charge les fonctionnalités d’extensibilité .NET, C# et Node.js.

<div id="affected-workflows">
  ## Workflows concernés
</div>

Passez en revue vos fonctionnalités d’extensibilité pour repérer le code contenant `require(edge)`.

Passez en revue les flows ou fonctionnalités suivants qui permettent d’ajouter du code d’extensibilité afin d’y repérer des références à `edge.js` :

* [Hooks](/docs/fr-CA/customize/hooks)
* [Rules](/docs/fr-CA/customize/rules)
* [Custom Database Scripts](/docs/fr-CA/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Custom Social Connections](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Aucun remplacement officiel n’est prévu pour cette fonctionnalité.

Une option de migration consiste à remplacer la logique `edge` par un service distinct dans votre propre infrastructure. Une fois ce service créé dans votre infrastructure, vous pouvez utiliser une fonction d’Extensibilité pour envoyer une requête HTTP qui déclenchera l’opération requise.

Par exemple, dans un script Login de Custom Database qui utilise `edge`, supprimez la fonction qui appelle `edge` et remplacez-la par un client HTTP générique pour récupérer la réponse.

Une autre option de migration consiste à réécrire la fonctionnalité d’Extensibilité en JavaScript plutôt qu’en .NET/C#.

Pour en savoir plus sur la création d’Actions ou la migration vers Actions, [consultez la documentation Auth0](/docs/fr-CA/customize/actions).
