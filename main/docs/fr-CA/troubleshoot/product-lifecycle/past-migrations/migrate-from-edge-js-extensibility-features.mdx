title: Migrer depuis les fonctionnalités d’extensibilité edge.js
---

À compter du 21 juin 2023, Auth0 n’offrira plus de soutien pour les fonctionnalités d’extensibilité Node.js en .NET et en C#.

<div id="affected-workflows">
  ## Flux de travail concernés
</div>

Passez en revue vos fonctionnalités d’extensibilité afin de repérer le code qui contient `require(edge)`.

Passez en revue les flows ou fonctionnalités ci-dessous qui permettent d’exécuter du code d’extensibilité afin de repérer les références à `edge.js` :

* [Hooks](/docs/fr-CA/customize/hooks)
* [Rules](/docs/fr-CA/customize/rules)
* [Scripts de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Aucun remplacement officiel n’est prévu pour cette fonctionnalité.

Une option de migration consiste à remplacer la logique `edge` par un service distinct dans votre propre infrastructure. Une fois ce service créé dans votre infrastructure, vous pouvez utiliser une fonction d’extensibilité pour envoyer une requête HTTP qui déclenchera l’opération requise.

Par exemple, dans un script Login de base de données personnalisée utilisant `edge`, supprimez la fonction qui appelle `edge` et utilisez plutôt un client HTTP générique pour récupérer la réponse.

Une autre option de migration consiste à réécrire la fonctionnalité d’extensibilité en JavaScript plutôt qu’en .NET/C#.

Pour en savoir plus sur la création d’Actions ou la migration vers Actions, [consultez la documentation Auth0](/docs/fr-CA/customize/actions).
