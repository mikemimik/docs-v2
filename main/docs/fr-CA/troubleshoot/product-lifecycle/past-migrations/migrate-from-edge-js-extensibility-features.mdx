title: Migrer depuis les fonctionnalités d’extensibilité edge.js
---

À compter du 21 juin 2023, Auth0 n’assurera plus le soutien des fonctionnalités d’extensibilité Node.js pour .NET et C#.

<div id="affected-workflows">
  ## Workflows touchés
</div>

Vérifiez vos fonctionnalités d’extensibilité afin de repérer le code qui contient `require(edge)`.

Vérifiez les flux ou fonctionnalités suivants qui permettent d’utiliser du code d’extensibilité pour repérer les références à `edge.js` :

* [Hooks](/docs/fr-CA/customize/hooks)
* [Rules](/docs/fr-CA/customize/rules)
* [Custom Database Scripts](/docs/fr-CA/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Custom Social Connections](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Aucun remplacement officiel n’est prévu pour cette fonctionnalité.

Une option de migration consiste à remplacer la logique `edge` par un service distinct dans votre propre infrastructure. Une fois ce service créé dans votre infrastructure, vous pouvez utiliser une fonction d’Extensibilité pour effectuer une requête HTTP et déclencher l’opération requise.

Par exemple, dans un script Login de Base de données personnalisée qui utilise `edge`, supprimez la fonction qui appelle `edge` et utilisez plutôt un client HTTP générique pour récupérer la réponse.

Une autre option de migration consiste à réécrire la fonctionnalité d’Extensibilité en JavaScript plutôt qu’en .NET/C#.

Pour en savoir plus sur la création d’Actions ou la migration vers Actions, [consultez la documentation Auth0](/docs/fr-CA/customize/actions).
