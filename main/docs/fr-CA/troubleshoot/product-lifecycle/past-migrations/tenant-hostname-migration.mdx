description: Décrit comment vérifier les journaux du tenant pour repérer les avis de dépréciation afin de migrer le domaine de votre application
title: Migrer la validation du nom d’hôte du tenant
---

À compter du 9 juin 2022 dans Public Cloud et du 9 septembre 2022 dans Private Cloud, Auth0 ajoute une couche de validation à l’Authentication API. Si Auth0 a détecté que les appels de vos applications à l’Authentication API pourraient être touchés par ce changement, nous avons ajouté des avis de dépréciation dans les journaux du tenant ainsi qu’un indicateur de migration pour vous aider à vous y préparer.

<div id="affected-endpoints">
  ### Points de terminaison concernés
</div>

À compter du 9 juin 2022 dans Public Cloud et du 9 septembre 2022 dans Private Cloud, tous les appels aux points de terminaison de l’Authentication API ci-dessous qui ne sont pas correctement validés seront rejetés. Auth0 recommande de prendre les mesures nécessaires pour migrer l’URL de domaine de votre application ou l’identifiant d’appel d’API dans le même tenant bien avant cette date.

Les points de terminaison concernés sont :

* `/oauth/token`
* `/co/authenticate`
* `/userinfo`
* `/login`
* `/oauth/revoke`
* `/mfa/challenge`
* `/p/<connection-type>/<ticket>` (point de terminaison de provisionnement de la connexion d’entreprise)

<div id="review-tenant-logs">
  ## Examiner les journaux du tenant
</div>

D’abord, vérifiez les journaux de votre tenant pour repérer les avis de dépréciation et confirmer si vous devez migrer votre application.

1. Accédez à [Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs).

   1. Recherchez dans les journaux `type:depnote AND description:ignore*request*host*header*` afin de trouver l’avis de dépréciation indiquant quelles applications sont touchées et doivent être migrées. L’entrée de journal comprend l’ID de toute application touchée ainsi que le message suivant :
      `Ignore request Host header: This feature is being deprecated. Please see https://auth0.com/docs/product-lifecycle/deprecations-and-migrations``/tenant-hostname-migration.`
   2. Repérez la section **Details &gt; Raw** du journal. Vous pourrez y identifier le `client_id` de l’application à mettre à jour, ou le `connection_id` dans le cas d’un point de terminaison de provisionnement.

      <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
        Vérifiez aussi le champ `tenant_from_host`. Ce champ contient le tenant Auth0 identifié à partir du domaine reçu dans vos appels d’API.
      </Callout>
2. Modifiez toutes les applications concernées.

   1. S’il existe une discordance entre le tenant et le tenant de domaine, vous devez modifier les identifiants envoyés ou l’URL de domaine, ainsi que tout autre paramètre de requête mal configuré.
   2. Le tenant de domaine doit correspondre au tenant associé au `client_id` ou au `connection_id`.

Une fois la migration de tous les tenants concernés terminée, les journaux du tenant n’afficheront plus d’avis de dépréciation associés à cette migration.

<div id="verify-migration">
  ## Vérifier la migration
</div>

Une fois que vous avez migré vos applications et configuré les noms d’hôte non validés, vérifiez vos changements en désactivant le comportement obsolète au moment de votre choix, avant le 9 juin 2022 ou le 9 septembre 2022.

1. Accédez à [Dashboard &gt; Tenant Settings &gt; Advanced &gt; Migrations](https://manage.auth0.com/#/tenant/advanced).
2. Désactivez le bouton bascule **Ignore request Host header** toggle**.** La désactivation de cette bascule force la validation pour votre tenant et complète la migration.

Si la validation du nom d’hôte ne fonctionne pas comme prévu après la désactivation de cette bascule, vous recevrez une erreur 4xx indiquant que le tenant du domaine et le tenant associé au `client_id` ou au `connection_id` ne correspondent pas.

Une fois que toutes les migrations des applications ont été effectuées avec succès et confirmées dans les environnements de production, vous pouvez alors désactiver définitivement le commutateur afin de vous assurer que les fonctionnalités obsolètes ne peuvent plus être utilisées. Après le 9 juin 2022 dans le Public Cloud et le 9 septembre 2022 dans le Private Cloud, Auth0 appliquera la validation du nom d’hôte et le commutateur associé sera supprimé des paramètres de votre tenant.
