title: Changements apportés à l’API Graph de Facebook
---

Depuis le 1er août 2018, Facebook a modifié les autorisations et les champs de l’API Graph de Facebook qui peuvent être demandés. Auth0 a mis à jour les connexions Facebook pour tenir compte de ces changements et a modifié l’interface de la connexion pour la rendre plus claire. Consultez le [journal des modifications de Facebook Login : changements récents apportés à Facebook Login](https://developers.facebook.com/docs/facebook-login/changelog#2018-07-02) pour tous les détails et les dates clés.

<div id="features-affected">
  ## Fonctionnalités touchées
</div>

Il se peut que la mise à jour n’exige aucune modification de votre code ou de votre configuration, mais votre application pourrait recevoir des données de profil supplémentaires si les autorisations existantes le permettent. Gardez à l’esprit ce qui suit :

* Si votre connexion Facebook est configurée pour demander l’une des autorisations supprimées, votre <Tooltip tip="Jeton d’accès : identifiant d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> ne les inclura pas dans le scope.
* Si votre application Facebook est marquée comme &quot;development&quot;, il se peut que vous voyiez encore temporairement une erreur lorsque vous essayez la connexion.
* Si vous ajoutez de nouvelles autorisations à la connexion, les utilisateurs finaux devront consentir la prochaine fois qu’ils ouvriront une session. Consultez la documentation de Facebook pour savoir comment gérer les actions pour les utilisateurs qui n’ont pas une autorisation donnée.

<div id="facebook-login-permissions">
  ### Autorisations de connexion Facebook
</div>

Les [autorisations de Facebook Login](https://developers.facebook.com/docs/facebook-login/permissions) sont demandées par votre application lorsqu’un utilisateur se connecte avec Facebook. Si l’utilisateur se connecte pour la première fois ou si les autorisations ont changé, une fenêtre de consentement s’affiche dans Facebook pour présenter les nouvelles autorisations demandées. Une fois ces autorisations accordées, votre application peut alors agir au nom de cet utilisateur au moyen d’un jeton d’accès Facebook.

L’interface de connexion Facebook a été mise à jour pour afficher à la fois le nom courant et le nom machine de toutes les autorisations affichées. Il est ainsi plus facile de trouver les autorisations dont vous avez besoin et de les associer au code que vous exécutez à l’aide de ces noms d’autorisation.

<div id="permissions-added">
  #### Autorisations ajoutées
</div>

Les autorisations suivantes ont été ajoutées à l’interface de connexion Facebook :

* business&#95;management
* groups&#95;access&#95;member&#95;info
* leads&#95;retrieval
* pages&#95;manage&#95;instant&#95;articles
* publish&#95;to&#95;groups
* publish&#95;to&#95;groups
* user&#95;age&#95;range
* user&#95;gender
* user&#95;link

<div id="permissions-removed">
  #### Autorisations retirées
</div>

Les autorisations suivantes ont été retirées de l’interface de connexion Facebook :

* read&#95;custom&#95;friendlists
* rsvp&#95;event
* user&#95;about&#95;me
* user&#95;actions-books
* user&#95;actions-fitness
* user&#95;actions-music
* user&#95;actions-news
* user&#95;actions-video
* user&#95;education&#95;history
* user&#95;games&#95;activity
* user&#95;relationship&#95;details
* user&#95;relationships
* user&#95;religion&#95;politics
* user&#95;website
* user&#95;work&#95;history

<div id="permissions-moved-to-deprecated">
  #### Autorisations déplacées vers la section Deprecated
</div>

Les autorisations suivantes ont été déplacées vers la section **Deprecated** et ne doivent pas être utilisées avec la plus récente version de la Graph API :

* publish&#95;actions
* user&#95;managed&#95;groups

<div id="facebook-graph-api-fields">
  ### Champs du Facebook Graph API
</div>

Le [Facebook Graph API](https://developers.facebook.com/docs/graph-api/reference/v3.2/user) est utilisé après la connexion d’un utilisateur pour récupérer les données de profil de l’utilisateur Auth0. Les autorisations d’accès aux données utilisateur demandées déterminent quelles informations sont récupérées à partir du Graph API. Les champs renvoyés dépendent des autorisations demandées et de la présence de ces champs dans le profil Facebook de l’utilisateur.

Cette modification a mis à niveau le Graph API de la version 2.8 à la version 3.2 et demandera les champs de données utilisateur suivants à la connexion :

* address (ajouté)
* age&#95;range
* birthday
* context
* cover
* currency (ajouté)
* devices
* email
* favorite&#95;athletes
* favorite&#95;teams
* first&#95;name
* gender
* hometown
* id
* inspirational&#95;people
* install&#95;type (ajouté)
* installed
* is&#95;verified
* languages
* last&#95;name
* link
* locale
* location
* meeting&#95;for (ajouté)
* middle&#95;name
* name
* name&#95;format
* picture
* public&#95;key (ajouté)
* quotes
* security&#95;settings (ajouté)
* short&#95;name (ajouté)
* significant&#95;other
* sports (ajouté)
* third&#95;party&#95;id
* timezone
* updated&#95;time
* verified
* video&#95;upload&#95;limits (ajouté)
* viewer&#95;can&#95;send&#95;gift (ajouté)
