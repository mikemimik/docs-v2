---
description: Décrit les changements liés à la suppression par Facebook de l’accès au champ de contexte social dans le profil.
title: Dépréciation du champ Social Context de Facebook
---

À compter du **30 juillet 2020**, les connexions Facebook qui demandent le champ `context` échoueront; Auth0 cessera donc de le demander pour toutes les connexions.

Le 30 avril, [Facebook a déclaré obsolète](https://developers.facebook.com/docs/graph-api/changelog/4-30-2019-endpoint-deprecations) l’utilisation du champ **Social Context** pour les nouvelles applications. Auth0 a continué de demander ce champ par défaut pour les connexions Facebook créées avant le 30 avril 2019. Pour vous assurer que ce champ n’est pas demandé avant le 30 juillet, décochez le champ **Social context** dans la section Données utilisateur de la connexion :

<Frame>![Dépréciation du champ Social Context de Facebook](/docs/images/cdy7uua7fh8z/4oi3foRuwYYOveDlq2vJM/a57cd954ceeaf6d7722796be7c743b76/facebook-context.png)</Frame>

Une fois le champ **Social context** décoché, les données du profil n’incluront plus le champ `context`. Ce champ contient les éléments suivants :

```text lines
"context": {
      "mutual_likes": {"data": [],"summary": {"total_count": 0}},
      "id": "dXNlcl9...UZD"
    }
```

<div id="actions">
  ## Actions
</div>

Si vous n’utilisez pas le champ `context` du profil Facebook renvoyé par Auth0 dans votre application, celle-ci continuera de fonctionner sans aucun changement. Sinon, vous devrez adapter le code de votre application pour qu’il n’en dépende pas.

Pour vous assurer que votre application ne sera pas touchée le 30 juillet, nous vous recommandons de décocher le champ **Social context** dans les propriétés de la connexion Facebook.
