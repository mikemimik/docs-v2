description: Décrit les changements découlant de la dépréciation de l’API d’authentification d’Instagram.
title: Dépréciation de la connexion Instagram
---

Depuis le 31 mars 2020, Facebook a désactivé les anciennes API d’Instagram au profit d’un nouvel ensemble d’API :

* [Instagram Graph API](https://developers.facebook.com/docs/instagram-api), conçue pour les comptes professionnels Instagram, et non pour l’authentification des utilisateurs finaux.
* [Instagram Basic Display API](https://developers.facebook.com/docs/instagram-basic-display-api), une API OAuth2 qui vous permet d’accorder à une application tierce l’accès aux données de base de votre compte Instagram.

Facebook exige que les applications intègrent Facebook Login pour l’authentification. Facebook n’approuvera pas les applications qui utilisent l’Instagram Basic Display API pour l’authentification.

<div id="actions">
  ## Actions
</div>

Pour permettre à vos utilisateurs existants de continuer à accéder à votre application, vous devrez demander aux utilisateurs qui s’authentifient avec Instagram de le faire autrement, puis utiliser Account Linking pour lier la nouvelle identité à l’ancienne.

Voici un exemple de flux :

* L’utilisateur s’authentifie avec Instagram.
* L’application informe l’utilisateur qu’il ne pourra plus s’authentifier avec Instagram et qu’il devra désormais le faire autrement.
* L’application présente à l’utilisateur les options d’authentification qui s’offrent à lui, par exemple :

  * Facebook
  * Nom d’utilisateur et mot de passe
* Une fois que l’utilisateur s’est authentifié d’une autre façon, vous liez les comptes au moyen d’Account Linking.

Bien que nous puissions remplacer notre mise en œuvre actuelle et utiliser le [flux OAuth Instagram Basic Display](https://developers.facebook.com/docs/instagram-basic-display-api/guides/getting-access-tokens-and-permissions), cela ne serait pas accepté selon les règles de Facebook. Vous devrez créer une application Instagram sur Facebook et, dans cette application, un avis indique :

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Notez que Basic Display n’est pas un outil d’authentification. Les données renvoyées par l’API ne peuvent pas être utilisées pour authentifier les utilisateurs de votre application ni pour les connecter à votre application. Si votre application utilise des données d’API pour authentifier des utilisateurs, elle sera rejetée pendant l’examen de l’application. Si vous avez besoin d’une solution d’authentification, utilisez plutôt Facebook Login.
</Callout>

Cela signifie que même si Auth0 prenait en charge ce flux, votre application Instagram ne serait pas approuvée par Facebook.

Si vous devez accéder aux données Instagram, vous devrez authentifier votre utilisateur autrement (par exemple, au moyen de Facebook Login ou d’un nom d’utilisateur et mot de passe), et implémenter le flux Instagram <Tooltip tip="OAuth 2.0 : framework d’autorisation qui définit les protocoles et les workflows d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> dans votre application.
