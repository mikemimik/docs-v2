---
description: Décrit les changements découlant de la dépréciation de l’Authentication API d’Instagram.
title: Dépréciation de la connexion Instagram
---

Depuis le 31 mars 2020, Facebook a désactivé les anciennes API d’Instagram au profit d’un nouvel ensemble d’API :

* [Instagram Graph API](https://developers.facebook.com/docs/instagram-api), conçue pour les comptes professionnels Instagram, et non pour l’authentification des utilisateurs finaux.
* [Instagram Basic Display API](https://developers.facebook.com/docs/instagram-basic-display-api), une API OAuth2, qui vous permet d’accorder à une application tierce l’accès aux données de base de votre compte Instagram.

Facebook exige que les applications intègrent Facebook Login pour l’authentification. Facebook n’approuvera pas les applications qui utilisent l’Instagram Basic Display API pour l’authentification.

<div id="actions">
  ## Actions
</div>

Pour permettre à vos utilisateurs actuels de continuer à accéder à votre application, vous devrez demander aux utilisateurs qui s&#39;authentifient avec Instagram de le faire autrement, puis utiliser la Liaison de comptes pour lier la nouvelle identité à l&#39;ancienne.

Voici un exemple de flux :

* L&#39;utilisateur s&#39;authentifie avec Instagram.
* L&#39;application informe l&#39;utilisateur qu&#39;il ne pourra plus s&#39;authentifier avec Instagram et qu&#39;il devra utiliser une autre méthode.
* L&#39;application présente les options d&#39;authentification offertes à l&#39;utilisateur, par exemple :

  * Facebook
  * Nom d&#39;utilisateur et mot de passe
* Une fois que l&#39;utilisateur s&#39;est authentifié d&#39;une autre façon, vous liez les comptes à l&#39;aide de la Liaison de comptes.

Même si nous pouvions remplacer notre mise en œuvre actuelle et utiliser le [flux OAuth Instagram Basic Display](https://developers.facebook.com/docs/instagram-basic-display-api/guides/getting-access-tokens-and-permissions), cela ne serait pas accepté par les politiques de Facebook. Vous devriez créer une application Instagram dans Facebook et, dans cette application, un avis indique :

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Veuillez noter que Basic Display n&#39;est pas un outil d&#39;authentification. Les données renvoyées par l&#39;API ne peuvent pas être utilisées pour authentifier les utilisateurs de votre application ni pour leur permettre d&#39;ouvrir une session dans votre application. Si votre application utilise des données d&#39;API pour authentifier des utilisateurs, elle sera rejetée pendant l&#39;App Review. Si vous avez besoin d&#39;une solution d&#39;authentification, utilisez plutôt Facebook Login.
</Callout>

Cela signifie que même si Auth0 implémentait ce flux, votre application Instagram ne serait pas approuvée par Facebook.

Si vous devez accéder aux données Instagram, vous devrez authentifier votre utilisateur d&#39;une autre façon (par exemple, en utilisant Facebook Login ou un nom d&#39;utilisateur et un mot de passe), et implémenter le flux <Tooltip tip="OAuth 2.0 : framework d'autorisation qui définit les protocoles et les workflows d'autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> d&#39;Instagram dans votre application.
