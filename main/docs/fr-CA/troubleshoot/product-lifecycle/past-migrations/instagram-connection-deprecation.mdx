---
description: Décrit les changements découlant de la dépréciation de l’Authentication API d’Instagram.
title: Dépréciation de la connexion Instagram
---

À compter du 31 mars 2020, Facebook a désactivé les API Legacy d’Instagram au profit d’un nouvel ensemble d’API :

* [Instagram Graph API](https://developers.facebook.com/docs/instagram-api), conçue pour les comptes Instagram professionnels, et non pour l’authentification des utilisateurs finaux.
* [Instagram Basic Display API](https://developers.facebook.com/docs/instagram-basic-display-api), une API OAuth2 qui vous permet d’accorder à une application tierce l’accès aux données de base de votre compte Instagram.

Facebook exige que les applications intègrent Facebook Login pour l’authentification. Facebook n’approuvera pas les applications qui utilisent l’Instagram Basic Display API pour l’authentification.

<div id="actions">
  ## Actions
</div>

Pour permettre à vos utilisateurs actuels de continuer à accéder à votre application, vous devrez demander à ceux qui s’authentifient avec Instagram de le faire autrement, puis utiliser Account Linking pour lier la nouvelle identité à l’ancienne.

Voici un exemple de flux :

* L’utilisateur s’authentifie avec Instagram.
* L’application informe l’utilisateur qu’il ne pourra plus s’authentifier avec Instagram et qu’il devra désormais utiliser une autre méthode.
* L’application présente les options d’authentification offertes à l’utilisateur, par exemple :

  * Facebook
  * Nom d’utilisateur et mot de passe
* Une fois l’utilisateur authentifié autrement, vous liez les comptes au moyen d’Account Linking.

Bien que nous puissions remplacer notre mise en œuvre actuelle et utiliser le [flux OAuth Instagram Basic Display](https://developers.facebook.com/docs/instagram-basic-display-api/guides/getting-access-tokens-and-permissions), cela ne serait pas accepté par les politiques de Facebook. Vous devrez créer une application Instagram dans Facebook et, dans cette application, un avis indique :

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Notez que Basic Display n’est pas un outil d’authentification. Les données renvoyées par l’API ne peuvent pas être utilisées pour authentifier les utilisateurs de votre application ni pour les connecter à votre application. Si votre application utilise des données d’API pour authentifier des utilisateurs, elle sera refusée lors de l’examen de l’application. Si vous avez besoin d’une solution d’authentification, utilisez plutôt Facebook Login.
</Callout>

Cela signifie que, même si Auth0 implémentait ce flux, votre application Instagram ne serait pas approuvée par Facebook.

Si vous avez besoin d’accéder aux données d’Instagram, vous devrez authentifier votre utilisateur autrement (par exemple, à l’aide de Facebook Login ou d’un nom d’utilisateur et mot de passe), puis implémenter le flux <Tooltip tip="OAuth 2.0: framework d’autorisation qui définit les protocoles et les workflows d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> d’Instagram dans votre application.
