<div class="api_card_wrapper">
  <Card title="Authentication API" href="/docs/fr-CA/api/authentication" arrow />
</div>

<Columns cols={2}>
  <Column>
    L’Authentication API expose les fonctionnalités d’identité d’Auth0 et des protocoles d’identité pris en charge (y compris OpenID Connect, OAuth et SAML).

    En règle générale, vous devriez utiliser cette API au moyen de l’un des Auth0 SDKs, comme [Auth0.js](/docs/fr-CA/libraries/auth0js), ou d’une bibliothèque comme [Lock](/docs/fr-CA/libraries/lock). Toutefois, si vous créez manuellement votre UI d’authentification, vous devrez appeler l’Authentication API directement.
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Obtenir des [jetons](/docs/fr-CA/secure/tokens) lors de l’authentification
    * Demander le profil d’un utilisateur à l’aide d’un [jeton d’accès](/docs/fr-CA/secure/tokens/access-tokens)
    * Échanger des [jetons d’actualisation](/docs/fr-CA/secure/tokens/refresh-tokens) contre de nouveaux jetons d’accès
    * Demander une vérification pour l’[authentification multifacteur (MFA)](/docs/fr-CA/secure/multi-factor-authentication)
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="Management API" href="/docs/fr-CA/api/management/v2" arrow />
</div>

<Columns cols={2}>
  <Column>
    La Management API vous permet de gérer votre compte Auth0 par programmation afin d’automatiser la configuration de votre environnement. La plupart des tâches que vous pouvez effectuer dans le Dashboard de gestion Auth0 peuvent aussi être effectuées par programmation à l’aide de cette API.
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Enregistrer vos applications et vos API dans Auth0
    * Configurer des [connexions](/docs/fr-CA/connections) permettant à vos utilisateurs de s’authentifier
    * [Gérer les utilisateurs](/docs/fr-CA/manage-users)
    * [Lier des comptes d’utilisateur](/docs/fr-CA/manage-users/user-accounts/user-account-linking/link-user-accounts)
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="My Account API" href="/docs/fr-CA/api/myaccount" arrow />
</div>

<Columns cols={2}>
  <Column>
    La My Account API d’Auth0 fournit un ensemble dédié de points de terminaison permettant aux utilisateurs de gérer les renseignements de leur propre compte. Vous pouvez utiliser ces API pour créer des expériences en libre-service dans vos applications ou ajouter progressivement des renseignements à un compte d’utilisateur.
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Permettre à l’utilisateur final de gérer ses propres facteurs
    * Mettre à jour les détails d’une méthode d’authentification
    * Connaître les méthodes d’authentification disponibles
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="My Organization API" href="/docs/fr-CA/api/myorganization" arrow />
</div>

<Columns cols={2}>
  <Column>
    La My Organization API d’Auth0 fournit une interface sécurisée, limitée à l’Organization, qui permet à vos clients d’affaires de gérer leurs propres Organizations dans votre Auth0 tenant. Cette API sert de fondation technique à l’administration déléguée intégrée et aux intégrations axées sur les API. Pour en savoir plus, consultez [My Organzation API and Embeddable UI Components](/docs/fr-CA/manage-users/my-organization-api).
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Détails de l’Auth0 Organization (nom, image de marque, nom d’affichage)
    * Configuration, propriété et relations propres à l’Organization
    * Identity Providers (IdPs) et provisionnement SCIM
    * Configuration des domaines et du Home Realm Discovery (HRD)
  </Column>
</Columns>

<div id="pagination">
  ## Pagination
</div>

Pour les tenants du Public Cloud, les points de terminaison de la Management API renvoient un maximum de 50 résultats par réponse. S’il y a plus de résultats que ce qu’une réponse peut contenir, incluez les paramètres `page` et `per_page` dans votre requête pour en récupérer davantage.

Consultez [Migrer vers les requêtes paginées du point de terminaison de la Management API v2](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-to-paginated-queries) pour savoir comment mettre à jour vos applications.
