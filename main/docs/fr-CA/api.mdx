<div class="api_card_wrapper">
  <Card title="Authentication API" href="/docs/fr-CA/api/authentication" arrow />
</div>

<Columns cols={2}>
  <Column>
    L’Authentication API expose des fonctionnalités d’identité pour Auth0 et les protocoles d’identité pris en charge (y compris OpenID Connect, OAuth et SAML).

    En général, vous devriez utiliser cette API par l’intermédiaire de l’un des Auth0 SDKs, comme [Auth0.js](/docs/fr-CA/libraries/auth0js), ou d’une bibliothèque comme [Lock](/docs/fr-CA/libraries/lock). Toutefois, si vous créez manuellement votre interface utilisateur d’authentication, vous devrez appeler l’Authentication API directement.
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Obtenir des [tokens](/docs/fr-CA/secure/tokens) pendant l’authentication
    * Demander le profil d’un utilisateur à l’aide d’un [Access Token](/docs/fr-CA/secure/tokens/access-tokens)
    * Échanger des [Refresh Tokens](/docs/fr-CA/secure/tokens/refresh-tokens) contre de nouveaux Access Tokens
    * Demander un challenge pour la [multi-factor authentication (MFA)](/docs/fr-CA/secure/multi-factor-authentication)
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="Management API" href="/docs/fr-CA/api/management/v2" arrow />
</div>

<Columns cols={2}>
  <Column>
    La Management API vous permet de gérer votre compte Auth0 par programmation, afin d’automatiser la configuration de votre environnement. La plupart des tâches que vous pouvez effectuer dans le Dashboard de gestion d’Auth0 peuvent aussi être réalisées par programmation à l’aide de cette API.
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Enregistrer vos applications et vos API dans Auth0
    * Configurer des [connections](/docs/fr-CA/connections) avec lesquelles vos utilisateurs peuvent s’authentifier
    * [Gérer les utilisateurs](/docs/fr-CA/manage-users)
    * [Lier des comptes d’utilisateur](/docs/fr-CA/manage-users/user-accounts/user-account-linking/link-user-accounts)
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="My Account API" href="/docs/fr-CA/api/myaccount" arrow />
</div>

<Columns cols={2}>
  <Column>
    L’Auth0 My Account API fournit un ensemble dédié d’endpoints permettant aux utilisateurs de gérer les renseignements de leur propre compte. Vous pouvez utiliser ces API pour créer des expériences en libre-service dans vos applications ou ajouter progressivement des renseignements à un compte utilisateur.
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Permettre à l’end-user de gérer ses propres factors
    * Mettre à jour les détails d’une authentication method
    * Savoir quelles authentication methods sont disponibles
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="My Organization API" href="/docs/fr-CA/api/myorganization" arrow />
</div>

<Columns cols={2}>
  <Column>
    L’Auth0 My Organization API fournit une interface sécurisée à portée Organization qui permet à vos clients d’affaires de gérer leurs propres Organizations dans votre Auth0 tenant. Cette API sert de fondement technique à l’administration déléguée intégrée et aux intégrations API-first. Pour en savoir plus, consultez [My Organzation API and Embeddable UI Components](/docs/fr-CA/manage-users/my-organization-api).
  </Column>

  <Column>
    Voici quelques exemples de tâches :

    * Détails de l’Auth0 Organization (nom, branding, nom d’affichage)
    * Configuration, propriété et relations propres à l’Organization
    * Identity Providers (IdPs) et SCIM provisioning
    * Configuration des domains et de Home Realm Discovery (HRD)
  </Column>
</Columns>

<div id="pagination">
  ## Pagination
</div>

Pour les tenants Public Cloud, les endpoints de la Management API renvoient un maximum de 50 résultats par réponse. Lorsqu&#39;il y a plus de résultats qu&#39;une réponse peut en contenir, incluez les paramètres `page` et `per_page` dans votre requête pour récupérer plus de résultats.

Consultez [Migrate to Management API v2 Endpoint Paginated Queries](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-to-paginated-queries) pour savoir comment mettre à jour vos applications.
