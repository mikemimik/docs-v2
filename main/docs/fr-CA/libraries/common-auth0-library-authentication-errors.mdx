title: Erreurs d’authentification courantes des bibliothèques Auth0
---

Les actions de vos utilisateurs ou les données qu’ils saisissent lors de l’inscription ou de la connexion peuvent entraîner des erreurs. Voici une liste des erreurs les plus courantes que vous pourriez rencontrer si vous utilisez l’une des bibliothèques Auth0 pour l’authentification.

<div id="sign-up">
  ## S’inscrire
</div>

En cas d’échec de l’inscription, les erreurs les plus courantes sont les suivantes :

| **Erreur**                                      | **Description**                                                                                               |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **invalid&#95;password**                        | Si le mot de passe utilisé n’est pas conforme à la politique de mot de passe de la connexion                  |
| **invalid&#95;signup**                          | L’utilisateur que vous tentez d’inscrire n’est pas valide                                                     |
| **password&#95;dictionary&#95;error**           | Le mot de passe choisi est trop courant                                                                       |
| **password&#95;no&#95;user&#95;info&#95;error** | Le mot de passe choisi est fondé sur des renseignements de l’utilisateur                                      |
| **password&#95;strength&#95;error**             | Le [mot de passe est trop faible](/docs/fr-CA/authenticate/database-connections/password-strength)                  |
| **unauthorized**                                | Si vous ne pouvez pas vous inscrire à cette application. Cela peut être dû à la violation d’une règle précise |
| **user&#95;exists**                             | L’utilisateur que vous tentez d’inscrire est déjà inscrit                                                     |
| **username&#95;exists**                         | Le nom d’utilisateur avec lequel vous tentez de vous inscrire est déjà utilisé                                |

<div id="log-in">
  ## Se connecter
</div>

En cas d’échec de connexion, les erreurs les plus courantes sont les suivantes :

| **Error**                             | **Description**                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **access&#95;denied**                 | Lors de l’utilisation d’une authentification Web, le serveur de ressources refuse l’accès conformément aux spécifications OAuth2                                                                                                                                                                                                                                                                          |
| **invalid&#95;user&#95;password**     | Le nom d’utilisateur et/ou le mot de passe utilisés pour l’authentification ne sont pas valides                                                                                                                                                                                                                                                                                                           |
| **mfa&#95;invalid&#95;code**          | Le code d’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication+%28MFA%29">authentification multifacteur (MFA)</Tooltip> fourni par l’utilisateur est invalide ou expiré |
| **mfa&#95;registration&#95;required** | L’administrateur a exigé l’[authentification multifacteur](/docs/fr-CA/secure/multi-factor-authentication), mais l’utilisateur ne l’a pas encore configurée                                                                                                                                                                                                                                                     |
| **mfa&#95;required**                  | L’utilisateur doit fournir le code d’[authentification multifacteur](/docs/fr-CA/secure/multi-factor-authentication) pour s’authentifier                                                                                                                                                                                                                                                                        |
| **password&#95;leaked**               | Si le mot de passe a été compromis et qu’il faut en utiliser un autre                                                                                                                                                                                                                                                                                                                                     |
| **PasswordHistoryError**              | Le mot de passe fourni pour l’inscription ou la mise à jour a déjà été utilisé (signalé lorsque la fonctionnalité d’[historique des mots de passe](/docs/fr-CA/authenticate/database-connections/password-options#password-history) est activée)                                                                                                                                                                |
| **PasswordStrengthError**             | Le mot de passe fourni ne respecte pas les [exigences de robustesse](/docs/fr-CA/authenticate/database-connections/password-strength) de la connexion                                                                                                                                                                                                                                                           |
| **too&#95;many&#95;attempts**         | Le compte est bloqué en raison d’un trop grand nombre de tentatives de connexion                                                                                                                                                                                                                                                                                                                          |
| **unauthorized**                      | L’utilisateur avec lequel vous tentez de vous connecter est bloqué                                                                                                                                                                                                                                                                                                                                        |
