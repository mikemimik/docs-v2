Vous pouvez personnaliser l’apparence de votre widget Lock de différentes façons. La meilleure et la plus sûre consiste à utiliser les options Javascript fournies.

<div id="javascript-options">
  ## Options de JavaScript
</div>

Vous pouvez configurer diverses personnalisations de votre Lock au moyen du paramètre `options` lorsque vous instanciez votre Lock. Certaines vous permettent de personnaliser votre interface utilisateur. Les options de personnalisation de l’interface utilisateur sont encore en cours d’élaboration - nous prévoyons en ajouter davantage au fil du temps.

D’abord, vous définirez l’objet `options`, qui contient les options que vous souhaitez personnaliser. Ensuite, vous devrez inclure cet objet `options` comme troisième paramètre lorsque vous instanciez Lock; vous trouverez plus de détails à ce sujet ci-dessous.

<div id="theming-options">
  ### Options de thème
</div>

Quelques options de thème sont actuellement offertes, dans l’espace de noms de la propriété `theme`.

#### logo {String}

<Frame>![Personnalisation de l’interface utilisateur de Lock - Logo](/docs/images/cdy7uua7fh8z/FdOAKKy71rTbFjIlZEL8D/2eef96008095d85388daaa461707901d/lock-theme-logo.png)</Frame>

La valeur de `logo` est l’URL d’une image qui sera placée dans l’en-tête de Lock; par défaut, le logo d’Auth0 est utilisé. La résolution minimale recommandée est de 200 pixels (largeur) sur 200 pixels (hauteur).

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png'
  }
};
```

#### primaryColor {String}

<Frame>![Personnalisation de l’interface utilisateur de Lock - couleur principale](/docs/images/cdy7uua7fh8z/1qQjeTmXJyIziv400VPaOn/48dd3cb8a72a6e88b2634b4b8a8bed63/lock-theme-primarycolor.png)</Frame>

La propriété `primaryColor` définit la couleur principale de Lock; toutes les couleurs utilisées dans le widget sont calculées à partir de celle-ci. Cette option est utile lorsque vous fournissez un `logo` personnalisé, afin de vous assurer que toutes les couleurs s’agencent bien avec la palette de couleurs du `logo`. La valeur par défaut est `#ea5323`.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png',
    primaryColor: '#31324F'
  }
};
```

<div id="authbuttons-object">
  #### authButtons `{Object}`
</div>

Permet de personnaliser les boutons de Lock. Chaque connexion personnalisée dont vous souhaitez personnaliser le bouton doit être indiquée par son nom, chacune avec son propre jeu de paramètres. Les paramètres personnalisables sont indiqués ci-dessous :

* **displayName** &#123;String&#125; : Le nom à afficher à la place du nom de la connexion dans le libellé du bouton, par exemple `LOGIN WITH MYCONNECTION` pour la connexion).
* **primaryColor** &#123;String&#125; : La couleur d’arrière-plan du bouton. La valeur par défaut est `#eb5424`.
* **foregroundColor** &#123;String&#125; : La couleur du texte du bouton. La valeur par défaut est `#FFFFFF`.
* **icon** &#123;String&#125; : L’URL de l’icône de cette connexion. Par exemple : `http://site.com/logo.png`.

```js lines
var options = {
  theme: {
    authButtons: {
      "testConnection": {
        displayName: "Test Conn",
        primaryColor: "#b7b7b7",
        foregroundColor: "#000000",
        icon: "http://example.com/icon.png"
      },
      "testConnection2": {
        primaryColor: "#000000",
        foregroundColor: "#ffffff",
      }
    }
  }
};
```

<div id="customizing-text">
  ### Personnalisation du texte
</div>

L’option `languageDictionary` permet de personnaliser tous les textes affichés dans Lock. Par défaut, sa valeur est {}. Voir ci-dessous un exemple.

```js lines
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Log me in"
  },
};
```

<Frame>![Personnalisation du texte dans l’interface utilisateur de Lock à l’aide de l’option languageDictionary](/docs/images/cdy7uua7fh8z/iNvBLxxmea2tiWBhzwKX5/db0d7f86d0f7f95f70e5edcd34ab736f/lock-languagedictionary.png)</Frame>

Pour obtenir la liste complète des éléments personnalisables à l’aide de `languageDictionary`, consultez la [spécification anglaise du Language Dictionary](https://github.com/auth0/lock/blob/master/src/i18n/en.js) dans le dépôt.

<div id="instantiating-lock">
  ### Instancier Lock
</div>

Enfin, vous devrez instancier votre Lock à l’aide de l’objet `options` que vous avez défini avec vos options personnalisées.

`var lock = new Auth0Lock('{yourClientId}', '{yourDomain}', options);`

<div id="overriding-css">
  ## Surcharge du CSS
</div>

Il n&#39;est pas recommandé de personnaliser votre Lock en surchargeant son CSS. Le problème, c&#39;est qu&#39;avec les nouvelles versions de Lock, certains styles peuvent changer, ce qui peut entraîner des problèmes imprévus si vous surchargez le CSS. De plus, il est possible de ne pas remarquer l&#39;utilisation de styles à d&#39;autres endroits et, même si le changement peut sembler correct dans une vue, il pourrait ne pas l&#39;être dans une autre.

Si vous comptez quand même surcharger le CSS pour personnaliser davantage votre Lock, nous vous recommandons d&#39;utiliser une version corrective précise de Lock plutôt qu&#39;une version majeure ou mineure, afin de limiter les résultats inattendus qui peuvent survenir lorsque vous modifiez les styles, puis qu&#39;une autre version corrective est déployée et risque d&#39;entraîner un comportement inattendu dans votre interface utilisateur en raison de ces changements. Vous pouvez le faire en veillant à préciser cette version corrective (`x.y.z`) lorsque vous incluez Lock, ou en la téléchargeant.

De plus, nous vous recommandons bien sûr de tester vos modifications CSS de façon exhaustive, afin de vous assurer que l&#39;expérience correspond bien à celle que vous souhaitez offrir à vos clients.
