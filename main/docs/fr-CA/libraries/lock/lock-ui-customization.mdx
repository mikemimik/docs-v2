Vous pouvez personnaliser l’apparence de votre widget Lock de plusieurs façons. La meilleure et la plus sûre consiste à utiliser les options Javascript fournies.

<div id="javascript-options">
  ## Options JavaScript
</div>

Vous pouvez configurer diverses options de personnalisation de votre Lock au moyen du paramètre `options` lorsque vous instanciez votre Lock. Certaines vous permettent de personnaliser votre interface utilisateur. Les options de personnalisation de l’interface utilisateur sont encore en cours d’élaboration - nous prévoyons en ajouter d’autres au fil du temps.

D’abord, vous définirez l’objet `options`, qui contiendra les options que vous souhaitez personnaliser. Ensuite, vous devrez inclure cet objet d’options comme troisième paramètre lorsque vous instanciez Lock; vous trouverez plus de détails à ce sujet ci-dessous.

<div id="theming-options">
  ### Options de thème
</div>

Il existe actuellement quelques options de thème, regroupées sous la propriété `theme`.

#### logo {String}

<Frame>![Personnalisation de l’interface utilisateur de Lock - Logo](/docs/images/cdy7uua7fh8z/FdOAKKy71rTbFjIlZEL8D/2eef96008095d85388daaa461707901d/lock-theme-logo.png)</Frame>

La valeur de `logo` est l’URL d’une image qui sera placée dans l’en-tête de Lock et, par défaut, le logo d’Auth0 sera utilisé. La résolution minimale recommandée est de 200 pixels (largeur) sur 200 pixels (hauteur).

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png'
  }
};
```

#### primaryColor {String}

<Frame>![Personnalisation de l’interface Lock - couleur principale](/docs/images/cdy7uua7fh8z/1qQjeTmXJyIziv400VPaOn/48dd3cb8a72a6e88b2634b4b8a8bed63/lock-theme-primarycolor.png)</Frame>

La propriété `primaryColor` définit la couleur principale du Lock ; toutes les couleurs utilisées dans le widget sont calculées à partir de celle-ci. Cette option est utile lorsque vous fournissez un `logo` personnalisé, afin de vous assurer que toutes les couleurs s’harmonisent bien avec la palette de couleurs du `logo`. La valeur par défaut est `#ea5323`.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png',
    primaryColor: '#31324F'
  }
};
```

<div id="authbuttons-object">
  #### authButtons `{Object}`
</div>

Permet de personnaliser les boutons dans Lock. Chaque connexion personnalisée dont vous voulez personnaliser le bouton doit être indiquée par son nom, avec son propre ensemble de paramètres. Les paramètres personnalisables sont indiqués ci-dessous :

* **displayName** &#123;String&#125;: Le nom à afficher à la place du nom de la connexion lors de la création du titre du bouton, par exemple `LOGIN WITH MYCONNECTION` pour la connexion).
* **primaryColor** &#123;String&#125;: La couleur d’arrière-plan du bouton. La valeur par défaut est `#eb5424`.
* **foregroundColor** &#123;String&#125;: La couleur du texte du bouton. La valeur par défaut est `#FFFFFF`.
* **icon** &#123;String&#125;: L’URL de l’icône de cette connexion. Par exemple : `http://site.com/logo.png`.

```js lines
var options = {
  theme: {
    authButtons: {
      "testConnection": {
        displayName: "Test Conn",
        primaryColor: "#b7b7b7",
        foregroundColor: "#000000",
        icon: "http://example.com/icon.png"
      },
      "testConnection2": {
        primaryColor: "#000000",
        foregroundColor: "#ffffff",
      }
    }
  }
};
```

<div id="customizing-text">
  ### Personnalisation du texte
</div>

L’option `languageDictionary` permet de personnaliser tous les textes affichés dans Lock. La valeur par défaut est {}. Voir l’exemple ci-dessous.

```js lines
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Log me in"
  },
};
```

<Frame>![Personnalisation du texte dans la Lock UI à l’aide de l’option languageDictionary](/docs/images/cdy7uua7fh8z/iNvBLxxmea2tiWBhzwKX5/db0d7f86d0f7f95f70e5edcd34ab736f/lock-languagedictionary.png)</Frame>

Pour obtenir la liste complète des éléments personnalisables à l’aide de `languageDictionary`, consultez l’[English Language Dictionary Specification](https://github.com/auth0/lock/blob/master/src/i18n/en.js) dans le repository.

<div id="instantiating-lock">
  ### Instanciation de Lock
</div>

Enfin, vous pouvez instancier votre Lock à l’aide de l’objet `options` que vous avez défini et qui contient vos options personnalisées.

`var lock = new Auth0Lock('{yourClientId}', '{yourDomain}', options);`

<div id="overriding-css">
  ## Surcharger le CSS
</div>

Nous ne recommandons pas de personnaliser votre Lock en surchargeant son CSS. Le problème, c’est qu’avec les nouvelles versions de Lock, certains styles peuvent changer, ce qui peut entraîner des problèmes imprévus si vous surchargez le CSS. De plus, il est possible de passer à côté de l’utilisation de styles ailleurs et, même si la modification peut sembler correcte dans une vue, elle pourrait ne pas l’être dans une autre.

Si vous comptez tout de même surcharger le CSS pour pousser davantage le style de votre Lock, nous vous recommandons d’utiliser une version de correctif précise de Lock plutôt qu’une version majeure ou mineure, afin de limiter les résultats inattendus qui pourraient survenir si vous modifiez les styles, puis qu’un autre correctif est déployé et entraîne un comportement imprévu dans votre interface utilisateur en raison de ces changements. Pour ce faire, assurez-vous d’indiquer cette version de correctif (`x.y.z`) lorsque vous incluez Lock, ou lorsque vous le téléchargez.

De plus, nous vous recommandons bien sûr de tester vos modifications au CSS de façon exhaustive, afin de vous assurer que l’expérience correspond bien à celle que vous souhaitez offrir à vos clients.
