Lock peut être configuré au moyen du paramètre `options` passé au constructeur. Ces options peuvent modifier le comportement du widget Lock, la façon dont il gère les connexions, les champs d’inscription supplémentaires requis pour votre projet, les paramètres de langue et les chaînes de texte, les couleurs et les images du widget, et bien plus encore. Consultez l’index ci-dessous si vous savez ce que vous cherchez, ou parcourez les options pour obtenir plus de détails.

<div id="ui-options">
  ## Options de l’interface utilisateur
</div>

### allowAutocomplete {Boolean}

Détermine si les champs de courriel ou de nom d’utilisateur autorisent l’autocomplétion (`<input autocomplete />`). La valeur par défaut est `false`.

`allowAutocomplete: true`

### allowPasswordAutocomplete {Boolean}

Détermine si le champ de mot de passe autorise ou non le remplissage automatique (`<input autocomplete />`). La valeur par défaut est `false`.

Réglez `allowPasswordAutocomplete` sur `true` pour assurer la prise en charge des gestionnaires de mots de passe et éviter d’autres comportements indésirables.

`allowPasswordAutocomplete: true`

### allowShowPassword {Boolean}

Cette option détermine si une case à cocher doit être ajoutée à l’UI afin de permettre à l’utilisateur d’afficher son mot de passe pendant la saisie lorsqu’elle est sélectionnée. Par défaut, cette option est définie à `false`.

`allowShowPassword: true`

Lock avec `allowShowPassword` défini à `true` et la bascule activée pour afficher le mot de passe :

<Frame>![option de Lock : Autoriser l’affichage du mot de passe](/docs/images/cdy7uua7fh8z/6G8LJaNjaMl5zUrwGi5UJQ/44dc951b70077b0ec6bc04d2f0413b12/lock-allowshowpassword.png)</Frame>

### allowedConnections {Array}

Tableau des connexions à utiliser pour les actions `signin|signup|reset`. Par défaut, toutes les connexions activées sont utilisées.

`allowedConnections: ['Username-Password-Authentication']`

`allowedConnections: ['twitter', 'facebook', 'linkedin']`

`allowedConnections: ['qraftlabs.com']`

Exemples de `allowedConnections` :

<Frame>![option de Lock : Connexions autorisées - Base de données](/docs/images/cdy7uua7fh8z/1mllPDLFCCETwQ9MeGEf3W/763728971302656618ef3bd131ab83b1/lock-allowedconnections-database.png)</Frame><Frame>![option de Lock : Connexions autorisées - Social](/docs/images/cdy7uua7fh8z/7uYN4ebrLLM8bjx1fee8j0/1e716cca3e1bae990063e9226dfe9ac2/lock-allowedconnections-social.png)</Frame>

### autoclose {Boolean}

Détermine si Lock se fermera automatiquement après une connexion réussie. La valeur par défaut est false. Si Lock n’est pas `closable`, il ne se fermera pas, même si cette option est définie à true.

`autoclose: true`

### autofocus {Boolean}

Si cette valeur est true, le focus est placé sur le premier champ du widget. La valeur par défaut est `false` lors du rendu sur un appareil mobile, ou si une option `container` est fournie; elle est `true` dans tous les autres cas.

`autofocus: false`

### avatar {Objet}

Par défaut, Gravatar est utilisé pour récupérer l’avatar et le nom d’affichage de l’utilisateur, mais vous pouvez les obtenir ailleurs grâce à l’option `avatar`.

```js lines
var options = {
  avatar: {
    url: function(email, cb) {
      // Obtenir l'URL de l'avatar pour l'adresse courriel saisie par l'utilisateur, Lock
      // préchargera l'image avant de l'afficher.
      // Notez qu'en cas d'erreur, vous appelez cb avec l'erreur dans
      // le premier argument au lieu de `null`.
      var url = obtainAvatarUrl(email);
      cb(null, url);
    },
    displayName: function(email, cb) {
      // Obtenir le nom d'affichage pour l'adresse courriel saisie par l'utilisateur.
      // Notez qu'en cas d'erreur, vous appelez cb avec l'erreur dans
      // le premier argument au lieu de `null`.
      var displayName = obtainDisplayName(email);
      cb(null, displayName);
    }
  }
};
```

Ou, si vous ne voulez afficher aucun avatar du tout, passez simplement `null`.

`avatar: null`

Comportement par défaut avec Gravatar :

<Frame>![Option de Lock : Avatar](/docs/images/cdy7uua7fh8z/69Nz0QoQifPWpfBZwX15J0/204aad93cde406f0ed3771ce38766f56/lock-avatar.png)</Frame>

### closable {Boolean}

Détermine si Lock peut être fermé. Lorsqu’une option `container` est fournie, sa valeur est toujours `false` ; sinon, la valeur par défaut est `true`.

`closable: false`

<Frame>![Option de Lock : Closable](/docs/images/cdy7uua7fh8z/2qwXtPQev1nlpyOQauXKBO/f967e1157972ecbf5b030b3be3600f2e/lock-closable.png)</Frame>

### container {String}

L’`ID` de l’élément HTML où le widget sera affiché.

Le widget s’affiche ainsi directement dans votre `div`, plutôt que dans une fenêtre modale contextuelle.

```html lines
<div id="hiw-login-container"></div>

<script>
  var options = {
    container: 'hiw-login-container'
  };

  // initialiser
  var lock = new Auth0Lock('xxxxxx', '<account>.auth0.com', options);

  // afficher
  lock.show();
</script>
```

<Frame>![Lock Option : conteneur](/docs/images/cdy7uua7fh8z/5L4S47Yfset2DR9pjcqVrX/1c4d1928cf75acf40129ffd01394bf33/lock-container.png)</Frame>

### message flash {Object}

Affiche un message flash `error` ou `success` lorsque Lock s’affiche. Cet objet comporte les propriétés suivantes :

* type &#123;String&#125; : Le type de message; les types pris en charge sont `error`, `info` et `success`
* text &#123;String&#125; : Le texte à afficher.

```js lines
var options = {
  flashMessage: {
    type: 'success',
    text: 'Welcome!'
  }
};
```

### language {String}

Indique la langue du widget. La valeur par défaut est « en ». Consultez le [répertoire d’internationalisation](https://github.com/auth0/lock/blob/master/src/i18n/) pour voir la liste à jour des langues offertes.

`language: 'es'`

<Frame>![Option de Lock : Langue](/docs/images/cdy7uua7fh8z/3u3lytCUBDbcKmXFV34OUv/5cf5d7e1af9593a3b094f32815a4da11/lock-language.png)</Frame>

### languageDictionary {Objet}

Permet de personnaliser tous les textes affichés dans Lock. La valeur par défaut est {"{}"}. Consultez la [spécification du Language Dictionary en anglais](https://github.com/auth0/lock/blob/master/src/i18n/en.js) pour obtenir la liste complète des valeurs `languageDictionary` pouvant être modifiées à l’aide de cet objet.

```js lines
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Log me in"
  },
};
```

<Frame>![Option de Lock : Language Dictionary](/docs/images/cdy7uua7fh8z/5203ofwtagtIKGT4HnIPmk/8ad9cdd151a2cdb18b8a42ee47dbf679/lock-languagedictionary.png)</Frame>

De plus, consultez la page [Personnaliser les messages d’erreur](/docs/fr-CA/customize/login-pages/classic-login/customize-lock-error-messages) ou la page [Internationalisation](/docs/fr-CA/customize/internationalization-and-localization/lock-internationalization) pour en savoir plus sur l’utilisation de l’option `languageDictionary`.

### popupOptions {Objet}

Permet de personnaliser l’emplacement de la fenêtre contextuelle à l’écran. Toutes les options de positionnement et de taille acceptées par window.open sont prises en charge. La valeur par défaut est {"{}"}.

Options des paramètres de `window.open`. Cela s’applique uniquement si `redirect` est défini à `false`.

```js lines
var options = {
  auth: {
      redirect: false
  },
  popupOptions: { width: 300, height: 400, left: 200, top: 300 }
};
```

### rememberLastLogin {Boolean}

Détermine s’il faut afficher ou non un écran qui vous permet d’ouvrir rapidement une session avec le compte utilisé la dernière fois.
Demande les données de <Tooltip tip="Single Sign-On (SSO) : service qui, après qu’un utilisateur ouvre une session dans une application, lui ouvre automatiquement une session dans d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-on">Single Sign-on</Tooltip> (SSO) et active le message **La dernière fois, vous avez ouvert une session avec**. La valeur par défaut est `true`. Cette information provient de la session Auth0 de l’utilisateur; cette fonctionnalité reste donc disponible tant que sa session Auth0 est active (et cette durée est configurable).

`rememberLastLogin: false`

Les nouveaux tenants [ont automatiquement Seamless SSO activé](/docs/fr-CA/get-started/tenant-settings/enable-sso-for-legacy-tenants). Lorsque cette option est activée, l’option `rememberLastLogin` n’est plus pertinente, car s’il existe déjà une session, la page <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> ne s’affichera pas du tout. L’utilisation de Seamless SSO est fortement recommandée, car elle offre une expérience d’authentification fluide : les utilisateurs ouvrent une session une seule fois et n’auront pas à saisir de nouveau leurs informations d’identification lorsqu’ils passent d’applications que vous avez créées à des applications tierces. Si l’utilisateur n’est pas connecté, il sera redirigé vers l’écran de connexion, comme prévu. De plus, le message **La dernière fois, vous avez ouvert une session avec** ne sera pas disponible dans les circonstances suivantes :

* Vous avez utilisé Lock dans une [page de connexion hébergée](/docs/fr-CA/authenticate/login/auth0-universal-login) avec une session établie au moyen de l’[authentification sans mot de passe](/docs/fr-CA/libraries/lock/lock-configuration#passwordlessmethod-string-).
* Vous avez utilisé Lock dans un [scénario de connexion intégrée](/docs/fr-CA/authenticate/login/universal-vs-embedded-login) avec `responseType: code` (ce qui indique le [Authorization Code Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow), utilisé pour les Regular Web Apps).

### scrollGlobalMessagesIntoView {Boolean}

Détermine si `globalMessage` doit être défilé jusqu’à la zone d’affichage de l’utilisateur. La valeur par défaut est `true`.

`scrollGlobalMessagesIntoView: false`

var options = {

    }

### authButtons {Objet}

Permet de personnaliser les boutons de Lock avec des connexions OAuth2 personnalisées. Chaque connexion personnalisée dont vous voulez personnaliser le bouton doit être indiquée par son nom, avec son propre ensemble de paramètres. Les paramètres personnalisables sont listés ci-dessous :

* **displayName** &#123;Chaîne&#125;: Le nom à afficher à la place du nom de la connexion lors de la création du libellé du bouton, par exemple `LOGIN WITH MYCONNECTION` pour le login).
* **primaryColor** &#123;Chaîne&#125;: La couleur d’arrière-plan du bouton. La valeur par défaut est `#eb5424`.
* **foregroundColor** &#123;Chaîne&#125;: La couleur du texte du bouton. La valeur par défaut est `#FFFFFF`.
* **icon** &#123;Chaîne&#125;: L’URL de l’icône de cette connexion. Par exemple : `http://site.com/logo.png`.

```js lines
var options = {
  theme: {
    authButtons: {
      "testConnection": {
        displayName: "Test Conn",
        primaryColor: "#b7b7b7",
        foregroundColor: "#000000",
        icon: "http://example.com/icon.png"
      },
      "testConnection2": {
        primaryColor: "#000000",
        foregroundColor: "#ffffff",
      }
    }
  }
};
```

### labeledSubmitButton {Boolean}

Cette option indique si le bouton de soumission doit être accompagné d’un libellé, et sa valeur par défaut est `true`. Lorsqu’elle est définie à `false`, une icône s’affiche à la place.

```js lines
var options = {
  theme: {
    labeledSubmitButton: false
  }
};
```

<Frame>![option de Lock : bouton de soumission avec libellé](/docs/images/cdy7uua7fh8z/5PNCvsbYa98pKZwWHVPS9S/39cb5ac63818ec5558c451b1e37f735f/lock-theme-labeledsubmitbutton.png)</Frame>

Si la propriété label est définie sur true, ce qui est la valeur par défaut, le texte du libellé peut être personnalisé au moyen de l’option [languageDictionary](#languagedictionary-object-).

### logo {String}

La valeur de `logo` est l’URL d’une image qui sera affichée dans l’en-tête de Lock; par défaut, le logo d’Auth0 est utilisé. Une hauteur maximale de `58px` est recommandée pour une meilleure expérience utilisateur.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png'
  }
};
```

<Frame>![Personnalisation de l’interface de Lock - logo](/docs/images/cdy7uua7fh8z/FdOAKKy71rTbFjIlZEL8D/2eef96008095d85388daaa461707901d/lock-theme-logo.png)</Frame>

### primaryColor {String}

La propriété `primaryColor` définit la couleur principale de Lock ; toutes les couleurs utilisées dans le widget seront calculées à partir de celle-ci. Cette option est utile si vous fournissez un `logo` personnalisé, afin de vous assurer que toutes les couleurs s’agencent bien avec la palette du `logo`. La valeur par défaut est `#ea5323`.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png',
    primaryColor: '#31324F'
  }
};
```

<div id="authentication-options">
  ## Options d’authentification
</div>

Les options d’authentification sont regroupées dans la propriété `auth` de l’objet `options`.

Le scope par défaut utilisé par Lock est `openid profile email`.

```js lines
var options = {
  auth: {
   params: {
    param1: "value1",
    scope: "openid profile email"
   },
   autoParseHash: true,
   redirect: true,
   redirectUrl: "some url",
   responseMode: "form_post",
   responseType: "token",
   sso: true,
   connectionScopes: {
    connectionName: [ 'scope1', 'scope2' ]
   }
  }
};
```

### audience {String}

L’option `audience` indique l’API qui utilisera le <Tooltip tip="jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> reçu après l’authentification.

  auth: {
    audience: 'https://{yourDomain}/userinfo',

}`;

### autoParseHash {Boolean}

Lorsque `autoParseHash` est défini à `true`, Lock analysera la chaîne `window.location.hash` au moment de son initialisation. S&#39;il est défini à `false`, vous devrez reprendre manuellement l&#39;authentification à l&#39;aide de la méthode [resumeAuth](/docs/fr-CA/libraries/lock/lock-api-reference).

```js lines
var options = {
  auth: {
    autoParseHash: false
  }
};
```

### connectionScopes {Object}

Cette option vous permet de définir les scopes à envoyer à la connexion oauth2/social lors de l’authentification.

```js lines
var options = {
  auth: {
    connectionScopes: {
      'facebook': ['scope1', 'scope2']
    }
  }
};
```

Vous pouvez obtenir auprès du fournisseur concerné la liste des scopes propres à vos connexions sociales. Par exemple, la documentation de [Facebook for Developers](https://developers.facebook.com/docs/facebook-login/permissions/) contient une liste des permissions distinctes pouvant être demandées pour votre connexion.

### params {Object}

Vous pouvez envoyer des paramètres au moment de démarrer une connexion en les ajoutant à l’objet options. L’exemple ci-dessous ajoute un paramètre `state` dont la valeur est `foo`, ainsi qu’un paramètre `scope` (qui comprend la portée, puis les attributs demandés).

```js lines
var options = {
  auth: {
    params: {
      state: 'foo',
      scope: 'openid email user_metadata app_metadata picture'
    }
  }
};
```

Pour en savoir plus sur les paramètres pris en charge, consultez [Paramètres d’authentification de Lock](/docs/fr-CA/libraries/lock/lock-authentication-parameters).

### redirect {Boolean}

La valeur par défaut est `true`. Lorsqu’elle est définie à `true`, le mode de redirection est utilisé. Si elle est définie à `false`, le [mode popup](/docs/fr-CA/libraries/lock/lock-authentication-modes) est utilisé.

```js lines
var options = {
  auth: {
    redirect: false
  }
};
```

### redirectUrl {String}

L’URL vers laquelle Auth0 redirigera l’utilisateur après l’authentification. Par défaut, la valeur est la chaîne vide &quot;&quot; (aucune URL de redirection).

```js lines
var options = {
  auth: {
    redirectUrl: 'http://testurl.com'
  }
};
```

Lorsque `redirectUrl` est fourni (c’est-à-dire défini à une valeur non vide), l’option `responseType` prendra par défaut la valeur `code` si elle n’est pas définie manuellement.

### responseMode {String}

Doit être défini à `"form_post"` si vous voulez que le code ou le token soit transmis par une requête HTTP POST à `redirectUrl`, au lieu d’être inclus dans les paramètres de requête ou le fragment de l’URL.

Sinon, cette option doit être omise, et elle l’est par défaut.

```js lines
var options = {
  auth: {
    responseMode: 'form_post'
  }
};
```

### responseType {String}

La valeur de `responseType` doit être définie à &quot;token&quot; pour les applications à page unique, et à &quot;code&quot; dans les autres cas. La valeur par défaut est &quot;code&quot; lorsque redirectUrl est fourni, et &quot;token&quot; sinon.

```js lines
var options = {
  auth: {
    responseType: 'token'
  }
};
```

Lorsque `responseType` est défini sur `code`, Lock n’affichera jamais le message **La dernière fois que vous avez ouvert une session avec**, et demandera toujours à l’utilisateur de fournir ses informations d’authentification.

## sso {Booléen}

La valeur par défaut est true. Lorsqu’elle est définie à true, elle active le [Single Sign-On](/docs/fr-CA/authenticate/single-sign-on) dans Lock.

```js lines
var options = {
  auth: {
    sso: false
  }
};
```

  ## Options Database
</div>

### additionalSignUpFields {Array}

Il est possible d’ajouter des champs de saisie supplémentaires à l’écran d’inscription à l’aide de l’option `additionalSignUpFields`. Chaque option ajoutée de cette façon sera ensuite ajoutée au `user_metadata` de l’utilisateur. Pour en savoir plus, consultez [Comprendre le fonctionnement des métadonnées dans les profils d’utilisateur](/docs/fr-CA/manage-users/user-accounts/metadata). Chaque entrée doit avoir un `name` et un `placeholder`, et vous pouvez aussi fournir une URL pour `icon`. De plus, vous pouvez définir la valeur initiale avec l’option `prefill`, qui peut être une chaîne contenant la valeur ou une fonction qui la récupère. Les autres options dépendent du type de champ, défini au moyen de l’option `type`, dont la valeur par défaut est &quot;text&quot;.

<Card title="Option additionalSignUpFields destinée à être utilisée uniquement avec l’inscription base de données">
  Les `additionalSignUpFields` sont destinés à être utilisés uniquement avec les inscriptions base de données. Si vous proposez aussi des inscriptions sociales, vous pouvez demander les renseignements supplémentaires après l’inscription des utilisateurs (consultez cette [page sur l’inscription personnalisée](/docs/fr-CA/libraries/custom-signup) pour en savoir plus). Vous pouvez utiliser la clé i18n `databaseAlternativeSignupInstructions` pour afficher ces instructions.
</Card>

Les nouveaux champs s’affichent sous les champs de saisie habituels de l’inscription, dans l’ordre où ils sont fournis.

<div id="text-fields">
  #### Champs de texte
</div>

Les champs de texte constituent le type par défaut des champs d’inscription supplémentaires. Notez qu’une fonction `validator` peut également être fournie.

```js lines
var options = {
  additionalSignUpFields: [{
    name: "address",
    placeholder: "enter your address",
    // Les propriétés suivantes sont facultatives
    icon: "https://example.com/assests/address_icon.png",
    prefill: "street 123",
    validator: function(address) {
      return {
         valid: address.length >= 10,
         hint: "Must have 10 or more chars" // facultatif
      };
    }
  },
  {
    name: "full_name",
    placeholder: "Enter your full name"
  }]
}
```

Si vous ne précisez pas de `validator`, le champ de texte sera **obligatoire**. Si vous voulez que le champ de texte soit facultatif, utilisez un validateur qui renvoie toujours `true`, comme ceci :

```js lines
var options = {
  additionalSignUpFields: [{
    name: "favorite_color",
    placeholder: "Enter your favorite color (optional)",
    validator: function() { 
      return true;
    }
  }]
}
```

Si vous voulez enregistrer la valeur de l’attribut à la racine de votre profil, utilisez `storage: 'root'`. Seul un sous-ensemble de valeurs peut être enregistré de cette manière. La liste des attributs qui peuvent être ajoutés à la racine de votre profil se trouve [ici](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id). Par défaut, chaque champ supplémentaire d’inscription est stocké dans l’objet user&#95;metadata.

```js lines
var options = {
  additionalSignUpFields: [{
    name: "name",
    storage: "root"
  }]
};
```

<Frame>![Option de Lock : champs d’inscription supplémentaires](/docs/images/cdy7uua7fh8z/2KPDGZb3caq57caJ90w53k/e092bdb4a4d93589aa006f796ff20fca/lock-additionalsignupfields.png)</Frame>

<div id="select-field">
  #### Champ de sélection
</div>

Le champ d’inscription `type: "select"` vous permet d’utiliser des éléments de sélection afin que l’utilisateur puisse choisir une valeur.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "select",
    name: "location",
    placeholder: "choose your location",
    options: [
      {value: "us", label: "United States"},
      {value: "fr", label: "France"},
      {value: "ar", label: "Argentina"}
    ],
    // Les propriétés suivantes sont facultatives
    icon: "https://example.com/assests/location_icon.png",
    prefill: "us"
  }]
}
```

Les éléments du tableau `options` pour les champs `select` doivent respecter le format suivant :
`{label: “non empty string”, value: “non empty string”}`, et au moins une option doit être définie.

Les valeurs `options` et `prefill` peuvent être fournies à l’aide d’une fonction :

```js lines
var options = {
  additionalSignUpFields: [{
    type: "select",
    name: "location",
    placeholder: "choose your location",
    options: function(cb) {
      // obtenir les options, en cas d'erreur, appelez cb avec l'erreur dans le
      // premier arg au lieu de null
      cb(null, options);
    },
    icon: "https://example.com/assests/location_icon.png",
    prefill: function(cb) {
      // obtenir le préremplissage, en cas d'erreur, appelez cb avec l'erreur dans le
      // premier arg au lieu de null
      cb(null, prefill);
    }
  }]
}
```

<div id="checkbox-field">
  #### Champ de type case à cocher
</div>

Le troisième type de champ d’inscription personnalisé est `type: "checkbox"`. La valeur `prefill` permet de définir l’état par défaut de la case à cocher (`true` ou `false`) et elle est obligatoire.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "checkbox",
    name: "newsletter",
    prefill: "true",
    placeholder: "I hereby agree that I want to receive marketing emails from your company"
  }]
}
```

<div id="hidden-field">
  #### Champ masqué
</div>

Le champ Signup `type: "hidden"` permet d’utiliser un champ masqué avec une valeur fixe.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "hidden",
    name: "signup_code",
    value: "abc123"
  }]
}
```

Dans certains cas d’usage, il peut être possible d’utiliser les données `additionalSignUpFields` dans les modèles de courriel, par exemple une option de préférence linguistique, dont la valeur pourrait ensuite servir à définir la langue des communications par courriel générées à partir de modèles.

### allowLogin {Boolean}

Lorsque cette option est réglée sur `false`, le widget n&#39;affiche pas l&#39;écran de connexion. C&#39;est utile si vous voulez utiliser le widget uniquement pour les inscriptions (les onglets de connexion et d&#39;inscription dans l&#39;écran d&#39;inscription seront masqués) ou pour réinitialiser des mots de passe (le bouton de retour dans l&#39;écran de mot de passe oublié sera masqué). Dans ce cas, vous devrez peut-être aussi définir les options `initialScreen`, `allowForgotPassword` et `allowSignUp`. Par défaut, sa valeur est `true`.

`allowLogin: false`

<Frame>![Option Lock : Autoriser la connexion](/docs/images/cdy7uua7fh8z/6gI4aSLHXLzhcHiR9aWMET/d05c2a08a3895278b01150ecf894cb5f/lock-allowlogin.png)</Frame>

### allowForgotPassword {Boolean}

Lorsque `allowForgotPassword` est défini à false, le lien « Vous avez oublié votre mot de passe? » est masqué dans l’écran de connexion, ce qui rend l’écran mot de passe oublié inaccessible. Par défaut, sa valeur est true. Si vous utilisez une connexion de base de données avec une base de données personnalisée qui ne comporte pas de script Change Password, l’écran mot de passe oublié ne sera pas disponible.

`allowForgotPassword: false`

<Frame>![Option Lock : Autoriser mot de passe oublié](/docs/images/cdy7uua7fh8z/1L6wJS3UnWK1VGKYMFhAB9/2ddcf2d462255e78a8b78958c2503809/lock-allowforgotpassword.png)</Frame>

### allowSignUp {Boolean}

Lorsqu’elle est définie sur `false`, cette option masque les onglets de connexion et d’inscription sur l’écran de connexion, ce qui rend l’écran d’inscription inaccessible. La valeur par défaut est `true`. Gardez à l’esprit que si la connexion à la base de données a les inscriptions désactivées, ou si vous utilisez une base de données personnalisée qui n’a pas de script Create, l’écran d’inscription ne sera pas disponible.

Gardez également à l’esprit que cette option contrôle **uniquement** l’apparence côté client et n’empêche pas complètement les visiteurs anonymes déterminés de créer un nouveau compte. Si vous souhaitez empêcher complètement les nouveaux utilisateurs de s’inscrire, vous devez utiliser l’option **Disable Sign Ups** dans le Dashboard, dans les paramètres de la connexion.

`allowSignUp: false`

<Frame>![Option Lock : Allow Signup](/docs/images/cdy7uua7fh8z/2BNlYe99EFlRmIlhoLZiEB/f280d865cb16fa5fecda952271cb18bf/lock-allowsignup.png)</Frame>

### defaultDatabaseConnection {String}

Spécifie la connexion de base de données à utiliser lorsqu’il y en a plus d’une.

`defaultDatabaseConnection: 'test-database'`

### initialScreen {String}

Le nom de l’écran qui s’affiche à l’ouverture du widget. Les valeurs acceptées sont `login`, `signUp` et `forgotPassword`. Si cette option n’est pas précisée, le widget affichera par défaut le premier écran disponible dans cette liste.

`initialScreen: 'forgotPassword'`

### loginAfterSignUp {Boolean}

Détermine si l’utilisateur sera automatiquement connecté après une inscription réussie. La valeur par défaut est `true`.

`loginAfterSignUp: false`

### forgotPasswordLink {String}

Définissez l’URL d’une page permettant à l’utilisateur de réinitialiser son mot de passe. Lorsqu’elle est définie avec une chaîne non vide, l’utilisateur est redirigé vers l’URL fournie lorsqu’il clique sur le lien « Vous ne vous souvenez pas de votre mot de passe ? » dans l’écran de connexion.

`forgotPasswordLink: 'https://yoursite.com/reset-password'`

### showTerms {Boolean}

Lorsque cette option est définie sur `true`, la chaîne `languageDictionary.signUpTerms` s’affiche. La valeur par défaut est `true`.

`showTerms: false`

### mustAcceptTerms {Booléen}

Lorsque cette option est définie à `true`, une case à cocher s’affiche à côté des conditions générales d’utilisation et doit être cochée avant l’inscription. Les conditions générales d’utilisation peuvent être définies au moyen de l’option `languageDictionary`. Cette option ne s’applique qu’aux utilisateurs qui s’inscrivent au moyen de connexions de base de données. La valeur par défaut est `false`.

`mustAcceptTerms: true`

### prefill {Objet}

Permet de définir la valeur initiale des champs de courriel et/ou de username. Si ce paramètre est omis, aucune valeur initiale n’est fournie.

```js lines
var options = {
  prefill: {
    email: "someone@auth0.com",
    username: "someone",
    phoneNumber: "+1234567890"
  }
};
```

### signUpLink {String}

Définissez l’URL à utiliser lorsqu’on clique sur le bouton d’inscription. Lorsqu’elle est définie sur une chaîne non vide, cette option force `allowSignUp` à `true`.

`signUpLink: 'https://yoursite.com/signup'`

### usernameStyle {String}

Détermine ce qui sera utilisé pour identifier l’utilisateur dans une connexion de base de données dont l’indicateur `requires_username` est activé (si ce n’est pas le cas, l’option `usernameStyle` sera ignorée). Les valeurs possibles sont `"username"` et `"email"`. Par défaut, `username` et `email` sont tous deux autorisés; définir cette option limitera les tentatives de connexion à l’un ou à l’autre.

`usernameStyle: 'username'`

### defaultEnterpriseConnection {String}

Indique la connexion d’entreprise à utiliser pour la connexion avec un `username` et un `Password` lorsqu’il y a plus d’une connexion disponible ou lorsqu’une connexion de base de données est configurée. Si un `defaultDatabaseConnection` est fourni, la connexion de base de données sera utilisée et cette option sera ignorée.

`defaultEnterpriseConnection: 'test-database'`

<div id="passwordless-options">
  ## Options d’authentification sans mot de passe
</div>

### passwordlessMethod {String}

Lorsque vous utilisez `Auth0LockPasswordless` avec une connexion par courriel, vous pouvez utiliser cette option pour choisir si vous envoyez un [code](/docs/fr-CA/authenticate/passwordless) ou un [Magic Link](/docs/fr-CA/authenticate/passwordless) pour authentifier l’utilisateur. Les valeurs possibles pour les connexions par courriel sont `code` et `link`. La valeur par défaut est `code`. Les connexions SMS <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip> utiliseront toujours `code`.

`passwordlessMethod: code`

<div id="other-options">
  ## Autres options
</div>

### configurationBaseUrl {String}

Remplace l’URL de base des paramètres de l’application. Par défaut, cette option utilise le domaine fourni. Cette option n’est requise que si, dans votre cas d’usage, votre application ne doit pas utiliser le comportement par défaut.

`configurationBaseUrl: "https://www.example.com"`

### languageBaseUrl {String}

Remplace l’URL source des traductions de langue fournies par Auth0. Par défaut, cette option utilise l’URL du CDN d’Auth0 `https://cdn.auth0.com`, puisque c’est là que sont stockées toutes les traductions fournies. En indiquant une autre valeur, vous pouvez utiliser une autre source pour les traductions de langue, au besoin.

`languageBaseUrl: "https://www.example.com"`

### hashCleanup {booléen}

Lorsque l’option `hashCleanup` est activée, elle supprime le hachage de l’URL de rappel après l’authentification de l’utilisateur. Sa valeur par défaut est `true`.

`hashCleanup: false`

### connectionResolver {Fonction}

Lorsqu’elle est utilisée, cette fonction fournit un point d’extensibilité qui permet de choisir la connexion à utiliser en fonction du nom d’utilisateur.

Prend `username`, `context` et `callback` comme paramètres. La fonction `callback` attend un objet comme celui-ci : `{type: 'database', name: 'connection name'}`. **Cela fonctionne uniquement avec les connexions de base de données.** Gardez à l’esprit que ce résolveur s’exécute lors de l’événement `onSubmit` du formulaire; il doit donc rester simple et rapide.

Il s’agit d’une fonctionnalité bêta. Si vous trouvez un bogue, veuillez ouvrir un ticket GitHub [issue](https://github.com/auth0/lock/issues/new).

### leeway {Integer}

L’option `leeway` peut être définie comme un entier — une valeur en secondes — afin de tenir compte d’un décalage entre les horloges lors de l’expiration des <Tooltip tip="ID Token : information d’authentification destinée au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+Token">ID Token</Tooltip>. En général, cette valeur ne dépasse pas une ou deux minutes.

`leeway: 30`
