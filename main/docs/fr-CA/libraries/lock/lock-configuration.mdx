Lock peut être configuré à l’aide du paramètre `options` passé au constructeur. Ces options peuvent modifier le comportement du widget Lock, la façon dont il gère les connexions, les champs d’inscription supplémentaires requis pour votre projet, la langue et les libellés, les couleurs et les images du widget, et bien plus encore. Consultez l’index ci-dessous si vous savez ce que vous cherchez, ou parcourez les options pour obtenir plus de détails.

<div id="ui-options">
  ## Options de l’UI
</div>

### allowAutocomplete {Boolean}

Indique si les champs de courriel ou de nom d’utilisateur prennent en charge le remplissage automatique (`<input autocomplete />`). La valeur par défaut est `false`.

`allowAutocomplete: true`

### allowPasswordAutocomplete {Boolean}

Détermine si le champ de mot de passe permet ou non l’autocomplétion (`<input autocomplete />`). La valeur par défaut est `false`.

Réglez `allowPasswordAutocomplete` à `true` pour prendre en charge les gestionnaires de mots de passe et éviter d’autres comportements indésirables.

`allowPasswordAutocomplete: true`

### allowShowPassword {Boolean}

Cette option détermine s’il faut ajouter ou non une case à cocher dans l’UI qui, une fois sélectionnée, permet à l’utilisateur d’afficher son mot de passe pendant qu’il le saisit. Par défaut, l’option est réglée à `false`.

`allowShowPassword: true`

Lock avec `allowShowPassword` réglé à `true` et la bascule activée pour afficher le mot de passe :

<Frame>![Option Lock : autoriser l’affichage du mot de passe](/docs/images/cdy7uua7fh8z/6G8LJaNjaMl5zUrwGi5UJQ/44dc951b70077b0ec6bc04d2f0413b12/lock-allowshowpassword.png)</Frame>

### allowedConnections {Tableau}

Tableau des connexions utilisées pour les actions `signin|signup|reset`. Par défaut, toutes les connexions activées sont utilisées.

`allowedConnections: ['Username-Password-Authentication']`

`allowedConnections: ['twitter', 'facebook', 'linkedin']`

`allowedConnections: ['qraftlabs.com']`

Exemples de `allowedConnections` :

<Frame>![Option Lock : Connexions autorisées - Database](/docs/images/cdy7uua7fh8z/1mllPDLFCCETwQ9MeGEf3W/763728971302656618ef3bd131ab83b1/lock-allowedconnections-database.png)</Frame><Frame>![Option Lock : Connexions autorisées - Social](/docs/images/cdy7uua7fh8z/7uYN4ebrLLM8bjx1fee8j0/1e716cca3e1bae990063e9226dfe9ac2/lock-allowedconnections-social.png)</Frame>

### autoclose {Boolean}

Détermine si Lock se fermera automatiquement après une connexion réussie. La valeur par défaut est false. Si Lock n’est pas `closable`, il ne se fermera pas, même si cette option est définie à true.

`autoclose: true`

### autofocus {Boolean}

Si cette option est activée, le focus est placé sur le premier champ du widget. Par défaut, la valeur est `false` sur un appareil mobile ou si une option `container` est fournie; dans tous les autres cas, la valeur par défaut est `true`.

`autofocus: false`

### avatar {Object}

Par défaut, Gravatar sert à récupérer l’avatar et le nom d’affichage de l’utilisateur, mais vous pouvez aussi les obtenir ailleurs à l’aide de l’option `avatar`.

```js lines
var options = {
  avatar: {
    url: function(email, cb) {
      // Récupérer l'URL de l'avatar pour le courriel saisi par l'utilisateur, Lock
      // préchargera l'image avant de l'afficher.
      // Notez qu'en cas d'erreur, vous appelez cb avec l'erreur dans
      // le premier argument au lieu de `null`.
      var url = obtainAvatarUrl(email);
      cb(null, url);
    },
    displayName: function(email, cb) {
      // Récupérer le nom d'affichage pour le courriel saisi par l'utilisateur.
      // Notez qu'en cas d'erreur, vous appelez cb avec l'erreur dans
      // le premier argument au lieu de `null`.
      var displayName = obtainDisplayName(email);
      cb(null, displayName);
    }
  }
};
```

Ou, si vous ne voulez afficher aucun avatar, passez simplement `null`.

`avatar: null`

Comportement par défaut avec Gravatar :

<Frame>![Option de Lock : Avatar](/docs/images/cdy7uua7fh8z/69Nz0QoQifPWpfBZwX15J0/204aad93cde406f0ed3771ce38766f56/lock-avatar.png)</Frame>

### closable {Boolean}

Indique si Lock peut être fermé. Si l’option `container` est fournie, cette valeur est toujours `false`; sinon, la valeur par défaut est `true`.

`closable: false`

<Frame>![Option de Lock : Fermable](/docs/images/cdy7uua7fh8z/2qwXtPQev1nlpyOQauXKBO/f967e1157972ecbf5b030b3be3600f2e/lock-closable.png)</Frame>

### container {String}

L’`id` de l’élément HTML dans lequel le widget s’affichera.

Le widget s’affichera directement dans votre `div` plutôt que dans une fenêtre modale contextuelle.

```html lines
<div id="hiw-login-container"></div>

<script>
  var options = {
    container: 'hiw-login-container'
  };

  // initialiser
  var lock = new Auth0Lock('xxxxxx', '<account>.auth0.com', options);

  // afficher
  lock.show();
</script>
```

<Frame>![Option Lock : Conteneur](/docs/images/cdy7uua7fh8z/5L4S47Yfset2DR9pjcqVrX/1c4d1928cf75acf40129ffd01394bf33/lock-container.png)</Frame>

### flashMessage {Objet}

Affiche un message flash `error` ou `success` lorsque Lock s’affiche. Cet objet comporte les propriétés suivantes :

* type &#123;String&#125;: Le type de message; les types pris en charge sont `error`, `info` et `success`
* text &#123;String&#125;: Le texte à afficher.

```js lines
var options = {
  flashMessage: {
    type: 'success',
    text: 'Welcome!'
  }
};
```

### language {String}

Indique la langue du widget. La valeur par défaut est « en ». Consultez le [répertoire d’internationalisation](https://github.com/auth0/lock/blob/master/src/i18n/) pour voir la liste à jour des langues offertes.

`language: 'es'`

<Frame>![Lock Option : Langue](/docs/images/cdy7uua7fh8z/3u3lytCUBDbcKmXFV34OUv/5cf5d7e1af9593a3b094f32815a4da11/lock-language.png)</Frame>

### languageDictionary {Object}

Permet de personnaliser tous les textes affichés dans Lock. La valeur par défaut est {"{}"}. Consultez la [spécification du Language Dictionary en anglais](https://github.com/auth0/lock/blob/master/src/i18n/en.js) pour obtenir la liste complète des valeurs de `languageDictionary` qui peuvent être modifiées à l’aide de cet objet.

```js lines
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Log me in"
  },
};
```

<Frame>![Option de Lock : dictionnaire de langue](/docs/images/cdy7uua7fh8z/5203ofwtagtIKGT4HnIPmk/8ad9cdd151a2cdb18b8a42ee47dbf679/lock-languagedictionary.png)</Frame>

Consultez également la page [Personnalisation des messages d’erreur](/docs/fr-CA/customize/login-pages/classic-login/customize-lock-error-messages) ou la page [Internationalisation](/docs/fr-CA/customize/internationalization-and-localization/lock-internationalization) pour en savoir plus sur l’utilisation de l’option `languageDictionary`.

### popupOptions {object}

Permet de personnaliser l’emplacement de la fenêtre contextuelle à l’écran. Toutes les options de position et de taille acceptées par window.open sont prises en charge. La valeur par défaut est {"{}"}.

Options des fonctionnalités de `window.open`. Cela s’applique uniquement si `redirect` est défini à `false`.

```js lines
var options = {
  auth: {
      redirect: false
  },
  popupOptions: { width: 300, height: 400, left: 200, top: 300 }
};
```

### rememberLastLogin {Boolean}

Détermine s’il faut ou non afficher un écran qui vous permet d’ouvrir rapidement une session avec le compte utilisé la dernière fois.
Demande des données d’<Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-on">Authentification unique</Tooltip> (SSO) et active un message **La dernière fois, vous avez ouvert une session avec**. La valeur par défaut est `true`. Ces informations proviennent de la session Auth0 de l’utilisateur; cette fonctionnalité reste donc disponible tant que dure sa session Auth0 (qui est configurable).

`rememberLastLogin: false`

Les nouveaux tenants [ont automatiquement Seamless SSO activé](/docs/fr-CA/get-started/tenant-settings/enable-sso-for-legacy-tenants). Lorsque cette option est activée, l’option `rememberLastLogin` n’est plus pertinente, car s’il existe déjà une session, la page <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> ne s’affichera pas du tout. L’utilisation de Seamless SSO est fortement recommandée, car elle offre une expérience d’authentification fluide : les utilisateurs ouvrent une session une seule fois et n’auront pas à saisir de nouveau leurs identifiants lorsqu’ils naviguent entre les applications que vous avez créées ou des applications tierces. Si l’utilisateur n’est pas connecté, il sera redirigé vers l’écran de connexion, comme prévu. De plus, le message **La dernière fois, vous avez ouvert une session avec** ne sera pas disponible dans les circonstances suivantes :

* Vous avez utilisé Lock dans une [page de connexion hébergée](/docs/fr-CA/authenticate/login/auth0-universal-login) avec la session établie à l’aide de l’[authentification sans mot de passe](/docs/fr-CA/libraries/lock/lock-configuration#passwordlessmethod-string-).
* Vous avez utilisé Lock dans un [scénario de connexion intégrée](/docs/fr-CA/authenticate/login/universal-vs-embedded-login) où `responseType: code` (indiquant le [flux de code d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow), qui est utilisé pour les Regular Web Apps).

### scrollGlobalMessagesIntoView {Booléen}

Détermine s’il faut faire défiler la page pour afficher un `globalMessage` dans la fenêtre de l’utilisateur. La valeur par défaut est `true`.

`scrollGlobalMessagesIntoView: false`

### authButtons {Objet}

Permet de personnaliser les boutons dans Lock pour les connexions OAuth2 personnalisées. Chaque connexion personnalisée dont vous souhaitez personnaliser le bouton doit être indiquée par son nom, avec son propre ensemble de paramètres. Les paramètres personnalisables sont indiqués ci-dessous :

* **displayName** &#123;Chaîne&#125;: Le nom à afficher à la place du nom de la connexion lors de la création du libellé du bouton, par exemple `LOGIN WITH MYCONNECTION` pour la connexion).
* **primaryColor** &#123;Chaîne&#125;: La couleur d’arrière-plan du bouton. La valeur par défaut est `#eb5424`.
* **foregroundColor** &#123;Chaîne&#125;: La couleur du texte du bouton. La valeur par défaut est `#FFFFFF`.
* **icon** &#123;Chaîne&#125;: L’URL de l’icône de cette connexion. Par exemple : `http://site.com/logo.png`.

```js lines
var options = {
  theme: {
    authButtons: {
      "testConnection": {
        displayName: "Test Conn",
        primaryColor: "#b7b7b7",
        foregroundColor: "#000000",
        icon: "http://example.com/icon.png"
      },
      "testConnection2": {
        primaryColor: "#000000",
        foregroundColor: "#ffffff",
      }
    }
  }
};
```

### labeledSubmitButton {Boolean}

Cette option indique si le bouton d’envoi doit afficher un libellé, et sa valeur par défaut est `true`. Lorsqu’elle est définie à `false`, une icône s’affiche à la place.

```js lines
var options = {
  theme: {
    labeledSubmitButton: false
  }
};
```

<Frame>![Option Lock : bouton d’envoi avec libellé](/docs/images/cdy7uua7fh8z/5PNCvsbYa98pKZwWHVPS9S/39cb5ac63818ec5558c451b1e37f735f/lock-theme-labeledsubmitbutton.png)</Frame>

Si `label` est défini sur true, ce qui est la valeur par défaut, le texte du libellé peut être personnalisé à l’aide de l’option [languageDictionary](#languagedictionary-object-).

### logo {String}

La valeur de `logo` est l’URL d’une image qui sera placée dans l’en-tête de Lock, et le logo d’Auth0 est utilisé par défaut. La hauteur maximale recommandée est de `58px` pour une meilleure expérience utilisateur.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png'
  }
};
```

<Frame>![Personnalisation de l’interface de Lock - Logo](/docs/images/cdy7uua7fh8z/FdOAKKy71rTbFjIlZEL8D/2eef96008095d85388daaa461707901d/lock-theme-logo.png)</Frame>

### primaryColor {String}

La propriété `primaryColor` définit la couleur principale de Lock ; toutes les couleurs utilisées dans le widget seront calculées à partir de celle-ci. Cette option est utile lorsque vous fournissez un `logo` personnalisé, afin de veiller à ce que toutes les couleurs s’harmonisent bien avec la palette du `logo`. La valeur par défaut est `#ea5323`.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png',
    primaryColor: '#31324F'
  }
};
```

<div id="authentication-options">
  ## Options d’authentification
</div>

Les options d’authentification sont regroupées dans la propriété `auth` de l’objet `options`.

Le scope par défaut utilisé par Lock est `openid profile email`.

```js lines
var options = {
  auth: {
   params: {
    param1: "value1",
    scope: "openid profile email"
   },
   autoParseHash: true,
   redirect: true,
   redirectUrl: "some url",
   responseMode: "form_post",
   responseType: "token",
   sso: true,
   connectionScopes: {
    connectionName: [ 'scope1', 'scope2' ]
   }
  }
};
```

### audience {String}

L’option `audience` indique l’API qui consommera le <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> reçu après l’authentification.

export const codeExample = `var options = {
  auth: {
    audience: 'https://{yourDomain}/userinfo',
  }
}`;

### autoParseHash {Boolean}

Lorsque `autoParseHash` est défini sur `true`, Lock analysera la chaîne `window.location.hash` au moment de son instanciation. S’il est défini sur `false`, vous devrez reprendre manuellement l’authentification à l’aide de la méthode [resumeAuth](/docs/fr-CA/libraries/lock/lock-api-reference).

```js lines
var options = {
  auth: {
    autoParseHash: false
  }
};
```

### connectionScopes {Object}

Cette option vous permet de définir les scopes à envoyer à la connexion sociale OAuth2 pour l’authentification.

```js lines
var options = {
  auth: {
    connectionScopes: {
      'facebook': ['scope1', 'scope2']
    }
  }
};
```

Vous pouvez obtenir auprès du fournisseur concerné une liste des scopes propres à vos connexions sociales. Par exemple, la documentation de référence [Facebook for Developers](https://developers.facebook.com/docs/facebook-login/permissions/) contient une liste d’autorisations distinctes pouvant être demandées pour votre connexion.

### params {Object}

Vous pouvez envoyer des paramètres au moment de lancer une connexion en les ajoutant à l’objet options. L’exemple ci-dessous ajoute un paramètre `state` ayant pour valeur `foo`, ainsi qu’un paramètre `scope` (qui comprend le scope, puis les attributs demandés).

```js lines
var options = {
  auth: {
    params: {
      state: 'foo',
      scope: 'openid email user_metadata app_metadata picture'
    }
  }
};
```

Pour en savoir plus sur les paramètres pris en charge, consultez [Paramètres d’authentification de Lock](/docs/fr-CA/libraries/lock/lock-authentication-parameters).

### redirect {Boolean}

La valeur par défaut est `true`. Lorsqu’elle est définie à `true`, le mode de redirection est utilisé. Si elle est définie à `false`, le [mode popup](/docs/fr-CA/libraries/lock/lock-authentication-modes) est utilisé.

```js lines
var options = {
  auth: {
    redirect: false
  }
};
```

### redirectUrl {String}

L’URL vers laquelle Auth0 redirigera l’utilisateur après l’authentification. Par défaut, la valeur est la chaîne vide &quot;&quot; (aucune URL de redirection).

```js lines
var options = {
  auth: {
    redirectUrl: 'http://testurl.com'
  }
};
```

Lorsque `redirectUrl` est fourni (c’est-à-dire qu’il n’est pas vide), l’option `responseType` prendra par défaut la valeur `code` si elle n’est pas définie manuellement.

### responseMode {String}

Doit être défini sur `"form_post"` si vous voulez que le code ou le token soit transmis dans une requête HTTP POST au `redirectUrl`, plutôt que d’être inclus dans les paramètres de requête ou le fragment de l’URL.

Sinon, cette option doit être omise; par défaut, elle l’est.

```js lines
var options = {
  auth: {
    responseMode: 'form_post'
  }
};
```

### responseType {String}

La valeur de `responseType` doit être définie sur &quot;token&quot; pour les applications à page unique, et sur &quot;code&quot; dans les autres cas. Par défaut, la valeur est &quot;code&quot; lorsque redirectUrl est fourni, et &quot;token&quot; sinon.

```js lines
var options = {
  auth: {
    responseType: 'token'
  }
};
```

Lorsque `responseType` est défini sur `code`, Lock n’affichera jamais le message **Votre dernière connexion avec** et invitera toujours l’utilisateur à saisir ses identifiants.

## sso {Boolean}

La valeur par défaut est true. Lorsqu’elle est définie sur true, l’[authentification unique](/docs/fr-CA/authenticate/single-sign-on) est activée dans Lock.

```js lines
var options = {
  auth: {
    sso: false
  }
};
```

  ## Options de base de données
</div>

### additionalSignUpFields {Array}

Des champs de saisie supplémentaires peuvent être ajoutés à l’écran d’inscription à l’aide de l’option `additionalSignUpFields`. Chaque option ajoutée de cette façon sera ensuite ajoutée au `user_metadata` de l’utilisateur. Pour en savoir plus, consultez [Comprendre le fonctionnement des métadonnées dans les profils utilisateur](/docs/fr-CA/manage-users/user-accounts/metadata). Chaque champ doit avoir un `name` et un `placeholder`, et vous pouvez aussi fournir une URL pour `icon`. De plus, une valeur initiale peut être fournie avec l’option `prefill`, qui peut être une chaîne contenant la valeur ou une fonction qui l’obtient. Les autres options dépendent du type de champ, qui est défini à l’aide de l’option `type` et dont la valeur par défaut est `"text"`.

<Card title="Option additionalSignUpFields destinée uniquement aux inscriptions base de données">
  Les `additionalSignUpFields` sont destinés à être utilisés uniquement avec les inscriptions base de données. Si vous avez aussi des inscriptions sociales, vous pouvez demander les renseignements supplémentaires après l’inscription des utilisateurs (consultez cette [page sur l’inscription personnalisée](/docs/fr-CA/libraries/custom-signup) pour plus de détails). Vous pouvez utiliser la clé i18n `databaseAlternativeSignupInstructions` pour afficher ces instructions.
</Card>

Les nouveaux champs sont affichés sous les champs de saisie d’inscription habituels, dans l’ordre où ils sont fournis.

<div id="text-fields">
  #### Champs de texte
</div>

Les champs de texte sont le type de champ d’inscription supplémentaire par défaut. Notez qu’il est aussi possible de fournir une fonction `validator`.

```js lines
var options = {
  additionalSignUpFields: [{
    name: "address",
    placeholder: "enter your address",
    // Les propriétés suivantes sont facultatives
    icon: "https://example.com/assests/address_icon.png",
    prefill: "street 123",
    validator: function(address) {
      return {
         valid: address.length >= 10,
         hint: "Must have 10 or more chars" // facultatif
      };
    }
  },
  {
    name: "full_name",
    placeholder: "Enter your full name"
  }]
}
```

Si vous ne spécifiez pas de `validator`, le champ de texte sera **obligatoire**. Si vous voulez que le champ de texte soit facultatif, utilisez un validateur qui retourne toujours `true`, comme ceci :

```js lines
var options = {
  additionalSignUpFields: [{
    name: "favorite_color",
    placeholder: "Enter your favorite color (optional)",
    validator: function() { 
      return true;
    }
  }]
}
```

Si vous souhaitez enregistrer la valeur de l’attribut à la racine de votre profil, utilisez `storage: 'root'`. Seul un sous-ensemble de valeurs peut être enregistré de cette façon. La liste des attributs qui peuvent être ajoutés à la racine de votre profil se trouve [ici](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id). Par défaut, chaque champ d’inscription supplémentaire est enregistré dans l’objet user&#95;metadata.

```js lines
var options = {
  additionalSignUpFields: [{
    name: "name",
    storage: "root"
  }]
};
```

<div id="select-field">
  #### Champ de sélection
</div>

Le champ d’inscription `type: "select"` vous permet d’utiliser des éléments de sélection afin que l’utilisateur puisse choisir une valeur.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "select",
    name: "location",
    placeholder: "choose your location",
    options: [
      {value: "us", label: "United States"},
      {value: "fr", label: "France"},
      {value: "ar", label: "Argentina"}
    ],
    // Les propriétés suivantes sont facultatives
    icon: "https://example.com/assests/location_icon.png",
    prefill: "us"
  }]
}
```

Les éléments du tableau `options` des champs `select` doivent respecter le format suivant :
`{label: “non empty string”, value: “non empty string”}`, et au moins une option doit être définie.

Les valeurs `options` et `prefill` peuvent être fournies à l’aide d’une fonction :

```js lines
var options = {
  additionalSignUpFields: [{
    type: "select",
    name: "location",
    placeholder: "choose your location",
    options: function(cb) {
      // obtenir les options, en cas d'erreur, appelez cb avec l'erreur dans le
      // premier arg au lieu de null
      cb(null, options);
    },
    icon: "https://example.com/assests/location_icon.png",
    prefill: function(cb) {
      // obtenir le préremplissage, en cas d'erreur, appelez cb avec l'erreur dans le
      // premier arg au lieu de null
      cb(null, prefill);
    }
  }]
}
```

<div id="checkbox-field">
  #### Champ de case à cocher
</div>

Le troisième type de champ d’inscription personnalisé est `type: "checkbox"`. La valeur `prefill` permet de définir l’état par défaut de la case à cocher (`true` ou `false`) et elle est obligatoire.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "checkbox",
    name: "newsletter",
    prefill: "true",
    placeholder: "I hereby agree that I want to receive marketing emails from your company"
  }]
}
```

<div id="hidden-field">
  #### Champ masqué
</div>

Le champ d’inscription `type: "hidden"` vous permet d’utiliser un champ masqué à valeur fixe.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "hidden",
    name: "signup_code",
    value: "abc123"
  }]
}
```

Certains cas d’utilisation peuvent utiliser les données de `additionalSignUpFields` dans des modèles de courriel, par exemple une option de préférence linguistique dont la valeur pourrait ensuite servir à définir la langue des communications par courriel générées à partir de modèles.

### allowLogin {Boolean}

Lorsque cette option est définie sur `false`, le widget n’affiche pas l’écran de connexion. C’est utile si vous souhaitez utiliser le widget uniquement pour les inscriptions (les onglets de connexion et d’inscription sur l’écran d’inscription seront masqués) ou pour réinitialiser des mots de passe (le bouton de retour sur l’écran de mot de passe oublié sera masqué). Dans ce cas, vous devrez peut-être aussi préciser les options `initialScreen`, `allowForgotPassword` et `allowSignUp`. Par défaut, cette option est définie sur `true`.

`allowLogin: false`

<Frame>![Option de Lock : Autoriser la connexion](/docs/images/cdy7uua7fh8z/6gI4aSLHXLzhcHiR9aWMET/d05c2a08a3895278b01150ecf894cb5f/lock-allowlogin.png)</Frame>

### allowForgotPassword {Boolean}

Lorsqu’elle est définie sur false, `allowForgotPassword` masque le lien « Vous ne vous souvenez plus de votre mot de passe? » dans l’écran de connexion, ce qui rend l’écran Mot de passe oublié inaccessible. La valeur par défaut est true. Si vous utilisez une connexion de base de données avec une base de données personnalisée qui n’a pas de script Change Password, l’écran Mot de passe oublié ne sera pas disponible.

`allowForgotPassword: false`

<Frame>![Option Lock : Autoriser le mot de passe oublié](/docs/images/cdy7uua7fh8z/1L6wJS3UnWK1VGKYMFhAB9/2ddcf2d462255e78a8b78958c2503809/lock-allowforgotpassword.png)</Frame>

### allowSignUp {Boolean}

Lorsqu’elle est définie à `false`, cette option masque les onglets de connexion et d’inscription dans l’écran de connexion, ce qui rend l’écran d’inscription inaccessible. Sa valeur par défaut est `true`. Gardez à l’esprit que si la connexion de base de données a les inscriptions désactivées ou si vous utilisez une base de données personnalisée qui n’a pas de script **Create**, l’écran d’inscription ne sera pas disponible.

Gardez également à l’esprit que cette option contrôle **uniquement** l’apparence côté client et n’empêche pas complètement les nouvelles inscriptions de la part de visiteurs anonymes déterminés. Si vous voulez empêcher complètement de nouveaux utilisateurs de s’inscrire, vous devez utiliser l’option **Disable Sign Ups** dans le Dashboard, dans les paramètres de la connexion.

`allowSignUp: false`

<Frame>![Option Lock : Autoriser l’inscription](/docs/images/cdy7uua7fh8z/2BNlYe99EFlRmIlhoLZiEB/f280d865cb16fa5fecda952271cb18bf/lock-allowsignup.png)</Frame>

### defaultDatabaseConnection {String}

Indique la connexion de base de données à utiliser lorsqu’il y en a plusieurs.

`defaultDatabaseConnection: 'test-database'`

### initialScreen {String}

Le nom de l’écran qui s’affiche à l’ouverture du widget. Les valeurs valides sont `login`, `signUp` et `forgotPassword`. Si cette option n’est pas précisée, le widget affichera par défaut le premier écran disponible dans cette liste.

`initialScreen: 'forgotPassword'`

### loginAfterSignUp {Boolean}

Indique si l’utilisateur sera automatiquement connecté après une inscription réussie. La valeur par défaut est `true`.

`loginAfterSignUp: false`

### forgotPasswordLink {String}

Définissez l’URL d’une page permettant à l’utilisateur de réinitialiser son mot de passe. Lorsqu’une valeur non vide est définie, l’utilisateur est redirigé vers l’URL fournie en cliquant sur le lien « Vous ne vous souvenez pas de votre mot de passe? » dans l’écran de connexion.

`forgotPasswordLink: 'https://yoursite.com/reset-password'`

### showTerms {Boolean}

Lorsqu’il est défini à `true`, la chaîne `languageDictionary.signUpTerms` s’affiche. La valeur par défaut est `true`.

`showTerms: false`

### mustAcceptTerms {Boolean}

Lorsque cette option est définie sur `true`, une case à cocher s’affiche à côté des conditions générales et doit être cochée avant de s’inscrire. Les conditions générales peuvent être définies au moyen de l’option `languageDictionary`. Cette option ne s’applique qu’aux utilisateurs qui s’inscrivent au moyen de connexions de base de données. La valeur par défaut est `false`.

`mustAcceptTerms: true`

### prefill {Objet}

Permet de définir la valeur initiale des champs `email` et/ou `username`. Si ce paramètre est omis, aucune valeur initiale n’est fournie.

```js lines
var options = {
  prefill: {
    email: "someone@auth0.com",
    username: "someone",
    phoneNumber: "+1234567890"
  }
};
```

### signUpLink {String}

Définissez l’URL à ouvrir lorsque vous cliquez sur le bouton d’inscription. Lorsqu’elle contient une chaîne non vide, cette option force `allowSignUp` à `true`.

`signUpLink: 'https://yoursite.com/signup'`

### usernameStyle {String}

Détermine quel identifiant sera utilisé pour un utilisateur dans une connexion de base de données dont l’indicateur `requires_username` est activé (si ce n’est pas le cas, l’option `usernameStyle` sera ignorée). Les valeurs possibles sont `"username"` et `"email"`. Par défaut, `username` et `email` sont tous deux autorisés; définir cette option limitera les connexions à l’un ou à l’autre.

`usernameStyle: 'username'`

### defaultEnterpriseConnection {String}

Spécifie la connexion d’entreprise qui permet de se connecter à l’aide d’un username et d’un mot de passe, et qui sera utilisée lorsqu’il y en a plus d’une disponible ou qu’il existe une connexion de base de données. Si un `defaultDatabaseConnection` est fourni, la connexion de base de données sera utilisée et cette option sera ignorée.

`defaultEnterpriseConnection: 'test-database'`

### defaultADUsernameFromEmailPrefix {Boolean}

Détermine le nom d’utilisateur AD de remplacement à partir du préfixe de l’adresse courriel. La valeur par défaut est `true`.

`defaultADUsernameFromEmailPrefix: false`

<div id="passwordless-options">
  ## Options Passwordless
</div>

### passwordlessMethod {String}

Lorsque vous utilisez `Auth0LockPasswordless` avec une connexion par courriel, vous pouvez utiliser cette option pour choisir entre l’envoi d’un [code](/docs/fr-CA/authenticate/passwordless) ou d’un [lien magique](/docs/fr-CA/authenticate/passwordless) pour authentifier l’utilisateur. Les valeurs possibles pour les connexions par courriel sont `code` et `link`. La valeur par défaut est `code`. Les connexions <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip> par SMS utilisent toujours `code`.

`passwordlessMethod: code`

<div id="other-options">
  ## Autres options
</div>

### configurationBaseUrl {String}

Remplace l’URL de base des paramètres de l’application. Par défaut, cette option utilise le domaine fourni. Cette option n’est nécessaire que si, dans votre cas d’utilisation, votre application ne doit pas utiliser le comportement par défaut.

`configurationBaseUrl: "https://www.example.com"`

### languageBaseUrl {String}

Remplace l’URL source des traductions fournies par Auth0. Par défaut, cette option utilise l’URL du CDN d’Auth0 `https://cdn.auth0.com`, puisque c’est à cet endroit que toutes les traductions fournies sont stockées. En indiquant une autre valeur, vous pouvez utiliser une autre source pour les traductions si nécessaire.

`languageBaseUrl: "https://www.example.com"`

### hashCleanup {Boolean}

Lorsque l’option `hashCleanup` est activée, elle supprime la partie hachée de l’URL de rappel après l’authentification de l’utilisateur. Par défaut, sa valeur est `true`.

`hashCleanup: false`

### connectionResolver {Function}

Lorsqu’elle est utilisée, cette fonction fournit un point d’extensibilité qui permet de choisir la connexion à utiliser en fonction des renseignements sur le nom d’utilisateur.

Prend `username`, `context` et `callback` comme paramètres. Le `callback` attend un objet comme celui-ci : `{type: 'database', name: 'connection name'}`. **Cela fonctionne uniquement avec les connexions de base de données.** Gardez à l’esprit que ce résolveur s’exécute lors de l’événement `onSubmit` du formulaire, donc il doit rester simple et rapide.

Il s’agit d’une fonctionnalité bêta. Si vous trouvez un bogue, veuillez ouvrir un [issue](https://github.com/auth0/lock/issues/new) sur GitHub.

### leeway {Integer}

L’option `leeway` peut être définie comme un nombre entier — une valeur en secondes — pour tenir compte du décalage entre les horloges lors de l’expiration des <Tooltip tip="ID Token : justificatif destiné au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+Token">ID Token</Tooltip>. En général, cette valeur ne dépasse pas une ou deux minutes.

`leeway: 30`
