Lock offre des options qui permettent d’activer ou de désactiver facilement la journalisation, ainsi que d’ajuster d’autres paramètres connexes.

<div id="loglevel">
  ## logLevel
</div>

Par défaut, la valeur est `.off` et les niveaux de journalisation Syslog sont pris en charge.

```swift lines
.withOptions {
  $0.logLevel = .all
}
```

<div id="loghttprequest">
  ## logHttpRequest
</div>

Cette option vous permet de choisir si vous voulez ou non enregistrer les requêtes à l’API Auth0.swift. Par défaut, cette valeur est `false`.

```swift lines
.withOptions {
  $0.logHttpRequest = true
}
```

<div id="loggeroutput">
  ## loggerOutput
</div>

Vous pouvez préciser un gestionnaire de sortie du logger; par défaut, l’instruction `print` est utilisée.

```swift lines
.withOptions {
  $0.loggerOutput = CleanroomLockLogger()
}
```

Dans le code ci-dessus, `loggerOutput` a été configuré pour utiliser [CleanroomLogger](https://github.com/emaloney/CleanroomLogger). On peut généralement y parvenir en implémentant le protocole `loggerOutput`. Vous pouvez bien sûr utiliser votre bibliothèque de logger préférée. Voici un exemple montrant comment gérer la sortie du logger avec CleanroomLogger.

```swift lines
class CleanroomLockLogger: LoggerOutput {
  func message(_ message: String, level: LoggerLevel, filename: String, line: Int) {
    let channel: LogChannel?
    switch level {
    case .debug:
        channel = Log.debug
    case .error:
        channel = Log.error
    case .info:
        channel = Log.info
    case .verbose:
        channel = Log.verbose
    case .warn:
        channel = Log.warning
    default:
        channel = nil
    }
    channel?.message(message, filePath: filename, fileLine: line)
  }
}
```
