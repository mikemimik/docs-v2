Lock <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> prend en charge l’authentification sans mot de passe au moyen de connexions par courriel et par SMS. Pour utiliser l’authentification Passwordless, vous devez avoir la version 2.14.0 ou ultérieure de Lock.Swift.

Pour afficher Lock, ajoutez l’extrait de code suivant à votre `UIViewController`.

```swift lines
Lock
    .passwordless()
    .withOptions {
         $0.oidcConformant = true
    }
    // withConnections, withOptions, withStyle, etc.
    .onAuth { credentials in
      // Enregistrer l'objet Credentials
    }
    .present(from: self)
```

Passwordless ne peut être utilisé qu’avec une seule connexion et privilégiera les connexions par courriel plutôt que par SMS.

<div id="passwordless-method">
  ### Méthode Passwordless
</div>

Lorsque vous utilisez Lock Passwordless, la valeur par défaut de `passwordlessMethod` est `.code`, ce qui envoie à l’utilisateur un code à usage unique pour se connecter. Si vous souhaitez utiliser les [liens universels](/docs/fr-CA/get-started/applications/enable-universal-links-support-in-apple-xcode), vous pouvez ajouter ce qui suit :

```swift lines
.withOptions {
    $0.passwordlessMethod = .magicLink
}
```

<div id="activity-callback">
  ### Callback de l’Activity
</div>

Si vous utilisez Lock Passwordless et que vous avez spécifié l’option `.magicLink` pour envoyer à l’utilisateur un lien universel, vous devrez ajouter ce qui suit à votre `AppDelegate.swift` :

```swift lines
func application(_ application: UIApplication, continue userActivity: NSUserActivity, restorationHandler: @escaping ([Any]?) -> Void) -> Bool {
    return Lock.continueAuth(using: userActivity)
}
```
