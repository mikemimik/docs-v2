Lock <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> prend en charge l’authentification sans mot de passe au moyen de connexions par courriel et par SMS. Pour utiliser l’authentification Passwordless, vous devez utiliser Lock.Swift version 2.14.0 ou ultérieure.

Pour afficher Lock, ajoutez l’extrait suivant à votre `UIViewController`.

```swift lines
Lock
    .passwordless()
    .withOptions {
         $0.oidcConformant = true
    }
    // withConnections, withOptions, withStyle, et ainsi de suite.
    .onAuth { credentials in
      // Enregistrer l'objet Credentials
    }
    .present(from: self)
```

Passwordless ne peut être utilisé qu’avec une seule connexion et donnera priorité aux connexions Email plutôt qu’aux connexions SMS.

<div id="passwordless-method">
  ### Méthode Passwordless
</div>

Lorsque vous utilisez Lock Passwordless, `passwordlessMethod` est réglé par défaut sur `.code`, ce qui envoie à l’utilisateur un code à usage unique pour se connecter. Si vous souhaitez utiliser les [liens universels](/docs/fr-CA/get-started/applications/enable-universal-links-support-in-apple-xcode), vous pouvez ajouter ce qui suit :

```swift lines
.withOptions {
    $0.passwordlessMethod = .magicLink
}
```

<div id="activity-callback">
  ### Callback de l’Activity
</div>

Si vous utilisez Lock Passwordless et que vous avez spécifié l’option `.magicLink` pour envoyer un lien universel à l’utilisateur, vous devrez ajouter ce qui suit à votre `AppDelegate.swift` :

```swift lines
func application(_ application: UIApplication, continue userActivity: NSUserActivity, restorationHandler: @escaping ([Any]?) -> Void) -> Bool {
    return Lock.continueAuth(using: userActivity)
}
```
