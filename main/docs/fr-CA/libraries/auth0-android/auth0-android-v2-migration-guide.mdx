La version 2 du Auth0 Android SDK comprend de nombreuses améliorations pour intégrer Auth0 à votre application Android. **Cette mise à niveau comporte des changements non rétrocompatibles.** Veuillez consulter ce guide pour comprendre les modifications requises lors de la migration vers la v2.

<div id="requirements-changes">
  ## Changements aux exigences
</div>

La v2 nécessite Android API version 21 ou une version ultérieure, ainsi que Java 8+.

Voici ce qu’il faut dans `build.gradle` pour cibler le bytecode Java 8 pour Android et les plugiciels Kotlin, respectivement :

```kotlin lines
android {
  compileOptions {
      sourceCompatibility JavaVersion.VERSION_1_8
      targetCompatibility JavaVersion.VERSION_1_8
  }

  kotlinOptions {
      jvmTarget = '1.8'
  }
}
```

<div id="open-id-connect-only">
  ## OpenID Connect uniquement
</div>

v2 ne prend en charge que les applications conformes à OIDC. Lorsque vous utilisez ce SDK pour authentifier des utilisateurs, seuls le pipeline d’authentification Auth0 actuel et les endpoints actuels sont utilisés. Nous avons supprimé la méthode `setOIDCConformant(boolean value)` de la classe `Auth0`.

Pour en savoir plus, consultez le [protocole OpenID Connect](/docs/fr-CA/authenticate/protocols/openid-connect-protocol).

<div id="authorization-code-with-pkce">
  ## Code d’autorisation avec PKCE
</div>

v2 ne prend en charge que le flux [Code d’autorisation avec clé de preuve pour l’échange de code (PKCE)](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce). Nous avons supprimé la méthode `withResponseType(@ResponseType int type)` de `WebAuthProvider.Builder`.

<div id="removal-of-webview-support">
  ## Fin de la prise en charge de WebView
</div>

Nous avons supprimé la possibilité, désormais obsolète, de s’authentifier à l’aide d’un composant `WebView`. À la place, des applications de navigateur externes seront toujours utilisées afin d’accroître la sécurité et la confiance des utilisateurs.

<div id="removal-of-legacy-authentication-api-support">
  ## Suppression de la prise en charge des API d’authentification hérité
</div>

Nous avons supprimé les méthodes et les classes servant à appeler les [API d’authentification](https://auth0.com/docs/api/authentication) classées comme hérité. Consultez la liste complète dans la section **Classes et interfaces supprimées** ci-dessous.

<div id="request-interfaces-changes">
  ## Modifications des interfaces de requête
</div>

Le package `com.auth0.android.request` définit les interfaces de haut niveau servant à créer et à exécuter des requêtes HTTP. Historiquement, un problème fréquent était l’impossibilité d’ajouter des en-têtes personnalisés et des paramètres de requête à tous les types de requêtes. Nous avons refactorisé les interfaces de requête pour permettre de personnaliser n’importe quelle requête avec des en-têtes supplémentaires et des paramètres de requête. Notez que ces paramètres doivent maintenant être de type `String`.

L’interface de haut niveau `Request` définit maintenant les méthodes suivantes :

* `public fun addParameters(parameters: Map<String, String>): Request<T, U>`
* `public fun addParameter(name: String, value: String): Request<T, U>`
* `public fun addHeader(name: String, value: String): Request<T, U>`

Nous avons supprimé les interfaces suivantes :

* `ParameterizableRequest`
* `AuthRequest`

`AuthenticationAPIClient` comporte donc aussi de nombreux changements aux types de retour des méthodes. En voici quelques exemples :

* Toutes les méthodes qui retournaient `ParameterizableRequest`, `TokenRequest` ou `DatabaseConnectionRequest` retournent maintenant un `Request`

* Toute méthode qui retournait `AuthRequest` retourne maintenant `AuthenticationRequest`

Si vous utilisez directement le type de retour de l’une de ces méthodes, vous devez modifier votre code pour tenir compte du type indiqué ci-dessus. Si vous enchaînez un appel à `start` ou `execute` pour effectuer la requête, aucun changement n’est nécessaire.

L’interface `AuthenticationRequest` ne comprend plus de méthode `setAccessToken("{ACCESS-TOKEN}")`. Cette méthode servait à définir le token pour la requête envoyée au point de terminaison hérité de l’Authentication API [/oauth/access&#95;token](https://auth0.com/docs/api/authentication#social-with-provider-s-access-token), désactivé depuis juin 2017. Si vous devez envoyer un paramètre portant ce nom, veuillez utiliser `addParameter("access_token", "{ACCESS-TOKEN}")`.

Toutes les classes qui implémentaient `ParameterizableRequest` ou `AuthRequest` ont été mises à jour pour tenir compte de ces changements.

Consultez la liste complète dans la section **Classes et interfaces supprimées** ci-dessous.

<div id="networking-client-customization">
  ## Personnalisation du client réseau
</div>

La v2 améliore la façon dont vous pouvez personnaliser la manière dont cette bibliothèque effectue des requêtes HTTP. Une nouvelle interface, `NetworkingClient`, définit le contrat d’exécution des requêtes HTTP. Vous pouvez configurer la classe `Auth0` avec une instance de `NetworkingClient`. Cela vous permet de fournir votre propre implémentation afin d’avoir un contrôle total sur le client réseau. De plus, le client réseau par défaut fourni par cette bibliothèque offre plusieurs points de personnalisation en exemple pour faciliter la configuration des personnalisations courantes.

<div id="timeout-configuration">
  ### Configuration du délai d’attente
</div>

export const codeExample1 = `// Avant
val account = Auth0("{yourClientId}", "{yourDomain}")

account.connectTimeoutInSeconds = 30
account.readTimeoutInSeconds = 30

val authAPI = AuthenticationAPIClient(account)

// Après
val netClient = DefaultClient(
    connectTimeout = 30,
    readTimeout = 30
)

val account = Auth0("{yourClientId}", "{yourDomain}")
account.networkingClient = netClient`;

<div id="detailed-change-listing">
  ## Liste détaillée des changements
</div>

<div id="files-that-changed-their-package">
  ### Fichiers dont le package a changé
</div>

* Nous avons déplacé la classe `com.auth0.android.authentication.request.ProfileRequest` vers `com.auth0.android.request.ProfileRequest`.
* Nous avons déplacé la classe `com.auth0.android.authentication.request.SignUpRequest` vers `com.auth0.android.request.SignUpRequest`.

<div id="internal-package">
  ### Package interne
</div>

La version précédente de cette bibliothèque rendait accessibles quelques classes du package `com.auth0.android.request.internal`. Celles-ci n&#39;ont jamais été destinées à faire partie de l&#39;API publique de la bibliothèque; la documentation Java en déconseillait l&#39;utilisation directe. Comme la base de code a migré vers Kotlin, nous tirons maintenant parti du modificateur `internal` et l&#39;utilisons au besoin. Ces classes peuvent sembler être des classes `public` dans les projets Java, mais ne les utilisez pas. Elles ne font toujours pas partie de l&#39;API publique de cette bibliothèque.

Nous ne fournirons pas de soutien pour ces classes. Nous les modifierons au besoin, sans préavis.

<div id="classes-and-interfaces-removed">
  ### Classes et interfaces supprimées
</div>

* La classe `com.auth0.android.util.Base64` a été supprimée. Utilisez `android.util.Base64` à la place.
* L’interface `com.auth0.android.request.ParameterizableRequest` a été supprimée. La possibilité d’ajouter des en-têtes de requête et des paramètres a été déplacée vers l’interface `com.auth0.android.request.Request`.
* L’interface `com.auth0.android.request.AuthRequest` a été supprimée. Utilisez plutôt l’interface `com.auth0.android.request.AuthenticationRequest`.
* La classe `com.auth0.android.provider.WebAuthActivity` a été supprimée. Des applications de navigateur externes seront désormais toujours utilisées pour l’authentification.
* La classe `com.auth0.android.result.Delegation` a été supprimée. Elle servait de résultat pour la requête vers le point de terminaison hérité de l’Authentication API, désactivé depuis juin 2017.
* La classe `com.auth0.android.authentication.request.DelegationRequest` a été supprimée. Elle servait à représenter la requête vers le point de terminaison hérité de l’Authentication API, désactivé depuis juin 2017.
* La classe `com.auth0.android.util.Telemetry` a été renommée `com.auth0.android.util.Auth0UserAgent`.
* La classe `com.auth0.android.request.AuthorizableRequest` a été supprimée. Vous pouvez obtenir le même résultat avec `addHeader("Authorization", "Bearer {TOKEN_VALUE}")`.
* La classe `com.auth0.android.authentication.request.TokenRequest` a été supprimée. La possibilité de définir un code verifier ainsi que des en-têtes de requête et des paramètres a été déplacée vers l’interface `com.auth0.android.request.Request`.
* La classe `com.auth0.android.authentication.request.DatabaseConnectionRequest` a été supprimée. La possibilité de définir des en-têtes de requête et des paramètres a été déplacée vers l’interface `com.auth0.android.request.Request`.
* La classe `com.auth0.android.provider.VoidCallback` a été supprimée. La possibilité d’utiliser un callback sans argument a été remplacée par `Callback<Void, AuthenticationException>`.
* L’interface `com.auth0.android.request.ErrorBuilder` a été supprimée.
* La classe `com.auth0.android.RequestBodyBuildException` a été supprimée.
* La classe `com.auth0.android.util.CheckHelper` a été supprimée.
* L’interface `com.auth0.android.util.JsonRequired` a été supprimée.
* La classe `com.auth0.android.util.JsonRequiredTypeAdapterFactory` a été supprimée.

<div id="constants-removed">
  ### Constantes supprimées
</div>

* `ParameterBuilder.GRANT_TYPE_JWT` a été supprimée.
* `ParameterBuilder.ID_TOKEN_KEY` a été supprimée.
* `ParameterBuilder.DEVICE_KEY` a été supprimée.
* `ParameterBuilder.ACCESS_TOKEN_KEY` a été supprimée.
* `ResponseType.CODE`, `ResponseType.ID_TOKEN` et `ResponseType.ACCESS_TOKEN` ont été supprimées.

<div id="classes-and-interfaces-changed">
  ### Classes et interfaces modifiées
</div>

* `SignupRequest` implémente maintenant `AuthenticationRequest` au lieu de `AuthRequest`, qui a été supprimé
* `AuthorizableRequest` étend maintenant `Request` au lieu de `ParameterizableRequest`, qui a été supprimé
* `BaseCallback` est obsolète; utilisez plutôt `Callback`.

<div id="constructors-changed">
  ### Constructeurs modifiés
</div>

* Vous ne pouvez plus créer `AuthenticationAPIClient` à partir d’un `Context`. Utilisez plutôt `AuthenticationAPIClient(auth0: Auth0)`. Vous pouvez créer une instance de `Auth0` à l’aide d’un `Context`.
* Vous ne pouvez plus créer `UsersAPIClient` à partir d’un `Context`. Utilisez plutôt `UsersAPIClient(auth0: Auth0, token: String)`. Vous pouvez créer une instance de `Auth0` à l’aide d’un `Context`.
* `SignupRequest` exige maintenant que le deuxième paramètre soit un `AuthenticationRequest`.
* `ProfileRequest` exige maintenant un `AuthenticationRequest` et un `Request<UserProfile, AuthenticationException>`.
* Vous ne pouvez plus créer `Credentials` avec une valeur « expires in ». Indiquez plutôt la date d’expiration ou « expires at ». En règle générale, vous n’aurez pas à créer `Credentials` vous-même.

<div id="methods-removed-or-changed">
  ### Méthodes supprimées ou modifiées
</div>

<div id="auth0-methods-removed">
  #### Méthodes Auth0 supprimées
</div>

* `setOIDCConformant(boolean enabled)` et `isOIDCConformant()` ont été supprimées. Le SDK prend désormais uniquement en charge les applications conformes à OIDC.
* `doNotSendTelemetry()` a été supprimée. Il n’existe pas de méthode de remplacement.
* `setWriteTimeoutInSeconds(seconds)` et `getWriteTimeoutInSeconds(seconds)` ont été supprimées. Il n’existe pas de méthode de remplacement.
* `setReadTimeoutInSeconds(seconds)` et `getReadTimeoutInSeconds(seconds)` ont été supprimées. Vous pouvez personnaliser directement le délai d’expiration de lecture dans la classe `DefaultClient`. Utilisez `DefaultClient(readTimeout = 123)`.
* `setConnectTimeoutInSeconds(seconds)` et `getConnectTimeoutInSeconds(seconds)` ont été supprimées. Vous pouvez personnaliser directement le délai d’expiration de connexion dans la classe `DefaultClient`. Utilisez `DefaultClient(connectTimeout = 123)`.
* `setLoggingEnabled(boolean enabled)` et `isLoggingEnabled()` ont été supprimées. Vous pouvez activer directement la journalisation du trafic réseau dans la classe `DefaultClient`. Utilisez `DefaultClient(enableLogging = true)`. Utilisez-la uniquement à des fins de débogage et ne l’activez jamais dans les environnements de production.
* `setTLS12Enforced()` et `isTLS12Enforced()` ont été supprimées. Le SDK prend désormais en charge TLS moderne par défaut.

<div id="authenticationapiclient-methods-removed-or-changed">
  #### Méthodes `AuthenticationAPIClient` supprimées ou modifiées
</div>

* `setUserAgent(String)` a été supprimée. Vous pouvez définir les en-têtes à envoyer directement pour chaque instance de `Request` à l’aide de la méthode `addHeader("{NAME}", "{VALUE}")`, ou de façon globale pour l’instance du client réseau au moyen du paramètre de constructeur `DefaultClient(defaultHeaders = mapOf())`.

Les méthodes et classes propres à l’appel des [Authentication APIs](https://auth0.com/docs/api/authentication) classées comme hérité ont été supprimées dans la v2. Nous avons supprimé les méthodes suivantes :

* `delegation()`, `delegationWithIdToken("{ID-TOKEN}")`, `delegationWithRefreshToken("{REFRESH-TOKEN}")` et `delegationWithIdToken("{ID-TOKEN}", "{API-TYPE}}")`. La prise en charge des points de terminaison de la hérité Authentication API a été supprimée.
* `loginWithOAuthAccessToken("{TOKEN}", {CONNECTION})`. Pour certains fournisseurs d’identité sociale, vous pouvez utiliser `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")` à la place.
* `tokenInfo("{ID-TOKEN}")`. Utilisez plutôt `userInfo("{ACCESS-TOKEN}")`.

Les méthodes qui retournaient un `ParameterizableRequest` retournent maintenant un `Request` :

* `userInfo("{ACCESS-TOKEN}")`
* `revokeToken("{REFRESH-TOKEN}")`
* `renewAuth("{REFRESH-TOKEN}")`
* `passwordlessWithEmail("{EMAIL}", PasswordlessType, "{CONNECTION}")`
* `passwordlessWithSMS("{PHONE-NUNBER}", PasswordlessType, "{CONNECTION}")`
* `fetchJsonWebKeys()`

Les méthodes qui retournaient un `AuthRequest` retournent maintenant un `AuthenticationRequest` :

* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}", "{REALM-OR-CONNECTION}")`
* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}")`
* `loginWithOTP("{MFA-TOKEN}", "{OTP}")`
* `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")`
* `loginWithPhoneNumber("{PHONE-NUMBER}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`
* `loginWithEmail("{EMAIL}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`

Les méthodes qui retournaient un `DatabaseConnectionRequest` retournent maintenant un `Request` :

* `createUser("{EMAIL}", "{PASSWORD}", "{USERNAME}", "{CONNECTION}")`
* `resetPassword("{EMAIL}","{CONNECTION}")`

Les méthodes qui retournaient un `TokenRequest` retournent maintenant un `Request` :

* `token("{AUTHORIZATION-CODE}", "{CODE-VERIFIER}", "{REDIRECT-URI}")`

<div id="authenticationrequest-methods-removed">
  #### Méthodes d’`AuthenticationRequest` supprimées
</div>

* `addAuthenticationParameters(parameters)` a été supprimée. Utilisez plutôt `addParameters(parameters)`.
* `setDevice("{DEVICE}").` Utilisez plutôt `addParameter("device", "{VALUE}")`.

<div id="webauthproviderbuilder-methods-removed">
  #### Méthodes supprimées de WebAuthProvider.Builder
</div>

* `useCodeGrant(boolean useCodeGrant)`. Il n’y a pas de remplacement; seul le flux Code + PKCE est pris en charge dans la v2.
* `useBrowser(boolean useBrowser)`. Il n’y a pas de remplacement; cette bibliothèque ne prend plus en charge l’authentification WebView.
* `useFullscreen(boolean useFullscreen)`. Il n’y a pas de remplacement; cette bibliothèque ne prend plus en charge l’authentification WebView.
* `withResponseType(@ResponseType int type)`. Il n’y a pas de remplacement; seul le flux Code + PKCE est pris en charge dans la v2.
* `start(activity: Activity, callback: AuthCallback, requestCode: Int)` a été supprimée. Utilisez plutôt `start(activity: Activity, callback: Callback<Credentials, AuthenticationException>)`.

<div id="webauthproviderlogoutbuilder-methods-removed">
  #### Méthodes de WebAuthProvider.LogoutBuilder supprimées
</div>

* `start(context: Context, callback: VoidCallback)`. Utilisez plutôt `start(context: Context, callback: Callback<Void, AuthenticationException>)`.

<div id="webauthprovider-methods-removed">
  #### Méthodes `WebAuthProvider` retirées
</div>

* `init(account: Auth0)` a été retirée. Utilisez plutôt `login(account: Auth0)`.
* `init(context: Context)` a été retirée. Utilisez plutôt `login(account: Auth0)`.
* `resume(requestCode: Int, resultCode: Int, intent: Intent)` a été retirée. Utilisez plutôt `resume(intent: Intent)`.
* `init(account: Auth0)` a été retirée. Utilisez plutôt `login(account: Auth0)`.

<div id="parameterbuilder-methods-removed">
  #### Méthodes supprimées de ParameterBuilder
</div>

* `setDevice("{DEVICE}")` a été supprimée. Utilisez `set("device", "{VALUE}")` à la place.

<div id="usersapiclient-methods-changed">
  #### Méthodes de UsersAPIClient modifiées
</div>

* `setUserAgent(String)` a été supprimée. Vous pouvez définir les en-têtes à envoyer directement pour chaque instance de `Request` à l’aide de la méthode `addHeader("{NAME}", "{VALUE}")`, ou globalement pour l’instance du client réseau au moyen du paramètre de constructeur `DefaultClient(defaultHeaders = mapOf())`.

Les méthodes qui renvoyaient un `ParameterizableRequest` renvoient maintenant un `Request` :

* `link("{PRIMARY-USER-ID}", "{SECONDARY-TOKEN}")`
* `unlink("{PRIMARY-USER-ID}", "{SECONDARY-USER-ID}", "{SECONDARY-PROVIDER}")`
* `updateMetadata("{USER-ID}", userMetadata)`
* `getProfile("{USER-ID}")`

<div id="requestfactory-methods-removed-or-changed">
  #### Méthodes de `RequestFactory` supprimées ou modifiées
</div>

La classe `RequestFactory` contient des méthodes qui facilitent l’envoi de requêtes HTTP, y compris la sérialisation et la désérialisation du corps de la requête JSON et de la réponse. Comme elle fait partie du package `com.auth0.android.request.internal`, elle n’est pas destinée à un usage public, mais puisqu’il s’agit d’un type public, un résumé des changements est présenté ci-dessous.

* Toutes les méthodes de requête ont été refactorisées pour être découplées d’une bibliothèque de requêtes HTTP précise (par exemple, `OkHttp`)
* Toutes les méthodes de requête retournent maintenant un `Request`.
* Toutes les méthodes de requête sont maintenant en minuscules (par exemple, `POST()` -&gt; `post()`).
* La méthode `authenticationPOST` a été supprimée sans remplacement, puisque toutes les instances de `Request` peuvent être paramétrées avec les en-têtes nécessaires.

<div id="profilerequest-methods-changed">
  #### Méthodes de `ProfileRequest` modifiées
</div>

* La méthode `addParameters` exige maintenant que la valeur soit une Map de String à String, au lieu de String à Object (`addParameters(mapOf("key" to "val"))`)

<div id="signuprequest-methods-changed">
  #### Méthodes de SignUpRequest modifiées
</div>

Les méthodes qui définissent des paramètres exigent maintenant que la valeur soit une Map de String à String, plutôt qu’un String à Object :

* `addAuthenticationParameters(mapOf("key" to "val"))`
* `addSignupParameters(mapOf("key" to "val"))`
* `addParameters(mapOf("key" to "val"))`

De plus, `setDevice("device")` a été supprimée. Utilisez plutôt `addParameter("device", "{VALUE}")`.
