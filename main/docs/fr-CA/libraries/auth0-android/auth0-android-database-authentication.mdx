<Warning>
  L’authentification par nom d’utilisateur/adresse courriel et mot de passe à partir d’applications natives est désactivée par défaut pour les nouveaux tenants depuis le 8 juin 2017. Il est recommandé aux utilisateurs d’utiliser Universal Login et d’effectuer plutôt une Web Authentication. Si vous souhaitez tout de même continuer, vous devrez d’abord activer le Password Grant Type dans votre Dashboard. Consultez [Application Grant Types](/docs/fr-CA/get-started/applications/application-grant-types) pour en savoir plus.
</Warning>

<div id="log-in-with-a-database-connection">
  ## Se connecter avec une connexion de base de données
</div>

Pour vous connecter à l’aide d’une connexion de base de données, appelez `login` en fournissant l’**adresse courriel** de l’utilisateur, son **mot de passe** et la **connexion** à utiliser pour l’authentification. La réponse est un objet `Credentials`.

```kotlin lines
authentication
    .login("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(payload: Credentials) {
            // Connecté!
        }

        override fun onFailure(error: AuthenticationException) {
            // Erreur!
        }
    })
```

Le scope par défaut est `openid profile email`.

<div id="sign-up-with-a-database-connection">
  ## S’inscrire avec une connexion de base de données
</div>

Pour vous inscrire avec une connexion de base de données, appelez la méthode `signUp` en lui transmettant l’adresse courriel, le mot de passe et le nom de la connexion.

```kotlin lines
authentication
    .signUp("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(result: Credentials) {
            // Inscription et connexion réussies!
        }

        override fun onFailure(error: AuthenticationException) {
            // Erreur!
        }
    });
```
