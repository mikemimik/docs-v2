Auth0.Android peut être configuré au moyen de plusieurs options, présentées ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les options ci-dessous ont changé depuis la version V1 antérieure. Veuillez consulter le guide de migration pour en savoir plus.
</Callout>

<div id="withconnection">
  ## withConnection
</div>

L’option `withConnection` vous permet de préciser la connexion avec laquelle vous souhaitez vous authentifier.

```kotlin lines
WebAuthProvider.login(account)
                .withConnection("twitter")
                .start(this, callback)
```

<div id="withscope">
  ## withScope
</div>

L’utilisation de scope vous permet de renvoyer des revendications précises pour certains champs de votre requête. L’ajout de paramètres à `withScope` vous permet d’ajouter davantage de scope. Consultez [Portées](/docs/fr-CA/get-started/apis/scopes) pour plus de détails.

```kotlin lines
WebAuthProvider.login(account)
                .withScope("openid profile email")
                .start(this, callback)
```

Le scope par défaut est `openid profile email`.

<div id="withconnectionscope">
  ## withConnectionScope
</div>

Il peut arriver que vous deviez vous authentifier avec des scopes de connexion particuliers, ou des permissions, auprès du fournisseur d’authentification en question. Consultez [Ajout de scopes pour un IdP externe](/docs/fr-CA/authenticate/identity-providers/adding-scopes-for-an-external-idp). Toutefois, si vous avez besoin d’un accès précis dans une situation particulière de votre application, vous pouvez transmettre des paramètres à `withConnectionScope`. Vous trouverez la liste complète des paramètres disponibles dans les paramètres de cette connexion dans le [Dashboard](https://manage.auth0.com/#), ou dans la documentation du fournisseur d’authentification. Le scope demandé ici s’ajoute à ceux précisés dans les paramètres de la connexion dans le Dashboard.

```kotlin lines
WebAuthProvider.login(account)
                .withConnectionScope("email", "profile", "calendar:read")
                .start(this, callback)
```

<div id="withparameters">
  ## withParameters
</div>

Pour envoyer des paramètres supplémentaires lors de l’authentification, utilisez `withParameters.`

```kotlin lines
val parameters = mapOf("param1" to "value1")

WebAuthProvider.login(account)
                .withParameters(parameters)
                .start(this, callback)
```

<div id="withheaders">
  ## withHeaders
</div>

Pour envoyer des en-têtes personnalisés à l’endpoint d’autorisation, utilisez `withHeaders.`

```kotlin lines
val headers = mapOf("header1" to "value1")

WebAuthProvider.login(account)
                .withHeaders(headers)
                .start(this, callback)
```

<div id="withscheme">
  ## withScheme
</div>

Si vous n’utilisez pas les &quot;App Links&quot; d’Android ou si vous souhaitez utiliser un schéma différent pour le Redirect URI, utilisez `withScheme`. Mettez à jour l’espace réservé `auth0Scheme` du Manifest dans le fichier `app/build.gradle`, ainsi que l’AllowList **Allowed Callback URLs** dans le [Dashboard](https://manage.auth0.com/#), sous les Settings de l’Application, afin qu’ils correspondent au schéma choisi.

```kotlin lines
WebAuthProvider.login(account)
                .withScheme("myapp")
                .start(this, callback)
```

<Warning>
  Le schéma doit être en minuscules et ne doit pas contenir de traits de soulignement.
</Warning>

<div id="withaudience">
  ## withAudience
</div>

Pour fournir une <Tooltip tip="Audience : identifiant unique du destinataire visé par un token émis. Appelée aud dans un token, sa valeur contient l’ID soit d’une application (Client ID) pour un ID Token, soit d’une API (API Identifier) pour un Access Token." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip>, utilisez `withAudience`.

```kotlin lines
WebAuthProvider.login(account)
                .withAudience("https://YOUR_DOMAIN/userinfo")
                .start(this, callback)
```
