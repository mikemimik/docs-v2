Auth0.Android peut être configuré au moyen de diverses options, présentées ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les options ci-dessous ont été modifiées par rapport à la version V1 précédente. Veuillez consulter le guide de migration pour plus de détails.
</Callout>

<div id="withconnection">
  ## withConnection
</div>

L’option `withConnection` vous permet d’indiquer la connexion avec laquelle vous souhaitez vous authentifier.

```kotlin lines
WebAuthProvider.login(account)
                .withConnection("twitter")
                .start(this, callback)
```

<div id="withscope">
  ## withScope
</div>

L’utilisation de scopes vous permet de renvoyer des claims précis pour des champs précis de votre requête. L’ajout de paramètres à `withScope` vous permet d’ajouter d’autres scopes. Consultez [Scopes](/docs/fr-CA/get-started/apis/scopes) pour plus de détails.

```kotlin lines
WebAuthProvider.login(account)
                .withScope("openid profile email")
                .start(this, callback)
```

Le scope par défaut est `openid profile email`.

<div id="withconnectionscope">
  ## withConnectionScope
</div>

Il peut arriver que vous deviez vous authentifier avec des scopes de connexion particuliers, ou des permissions, auprès du fournisseur d’authentification concerné. Consultez [Adding Scopes for an External IDP](/docs/fr-CA/authenticate/identity-providers/adding-scopes-for-an-external-idp). Cependant, si vous avez besoin d’un accès particulier dans un cas précis de votre application, vous pouvez passer des paramètres à `withConnectionScope`. Vous trouverez la liste complète des paramètres disponibles dans les paramètres de cette connexion, dans le [Dashboard](https://manage.auth0.com/#), ou dans la documentation du fournisseur d’authentification. Le scope demandé ici s’ajoute à ceux définis dans les paramètres de la connexion dans le Dashboard.

```kotlin lines
WebAuthProvider.login(account)
                .withConnectionScope("email", "profile", "calendar:read")
                .start(this, callback)
```

<div id="withparameters">
  ## withParameters
</div>

Pour envoyer des paramètres supplémentaires lors de l’authentication, utilisez `withParameters.`

```kotlin lines
val parameters = mapOf("param1" to "value1")

WebAuthProvider.login(account)
                .withParameters(parameters)
                .start(this, callback)
```

<div id="withheaders">
  ## withHeaders
</div>

Pour envoyer des en-têtes personnalisés au point de terminaison d’autorisation, utilisez `withHeaders.`

```kotlin lines
val headers = mapOf("header1" to "value1")

WebAuthProvider.login(account)
                .withHeaders(headers)
                .start(this, callback)
```

<div id="withscheme">
  ## withScheme
</div>

Si vous n’utilisez pas les « App Links » d’Android ou si vous voulez utiliser un schéma différent pour l’URI de redirection, utilisez `withScheme`. Mettez à jour l’espace réservé `auth0Scheme` du Manifest dans le fichier `app/build.gradle`, ainsi que l’AllowList **Allowed Callback URLs** dans les Settings de l’Application du [Dashboard](https://manage.auth0.com/#), afin qu’ils correspondent au schéma choisi.

```kotlin lines
WebAuthProvider.login(account)
                .withScheme("myapp")
                .start(this, callback)
```

<Warning>
  Le schéma doit être en minuscules et ne pas contenir de traits de soulignement.
</Warning>

<div id="withaudience">
  ## withAudience
</div>

Pour fournir une <Tooltip tip="Audience : identifiant unique de l’audience d’un token émis. Appelée aud dans un token, sa valeur contient l’ID d’une application (Client ID) pour un ID Token, ou d’une API (API Identifier) pour un access token." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip>, utilisez `withAudience`.

```kotlin lines
WebAuthProvider.login(account)
                .withAudience("https://YOUR_DOMAIN/userinfo")
                .start(this, callback)
```
