<div id="add-login-to-your-android-application">
  ## Ajouter la connexion à votre application Android
</div>

Vous pouvez permettre à l’utilisateur de se connecter à l’aide de la méthode `WebAuthProvider.login` :

```kotlin lines
WebAuthProvider.login(account)
  .withScheme(getString(R.string.com_auth0_scheme))
  .withScope("openid profile email")
  .start(this, object : Callback<Credentials, AuthenticationException> {
    override fun onFailure(exception: AuthenticationException) {
       // Authentification échouée
     }

      override fun onSuccess(credentials: Credentials) {
         // Authentification réussie
       }
  })
```

Le résultat de l’authentification sera transmis à la fonction de rappel `onSuccess`.

Consultez la [configuration d’Auth0.Android](/docs/fr-CA/libraries/auth0-android/auth0-android-configuration) pour voir les autres options offertes par la classe `WebAuthProvider`.

<div id="add-logout-to-your-android-application">
  ## Ajoutez la déconnexion à votre application Android
</div>

Pour déconnecter l’utilisateur, appelez la méthode `WebAuthProvider.logout`. Le résultat de la déconnexion sera transmis dans la fonction de rappel `onSuccess`.

Cette méthode supprime le cookie que le navigateur a défini au moment de l’authentification, ce qui oblige les utilisateurs à saisir de nouveau leurs identifiants la prochaine fois qu’ils tenteront de s’authentifier.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si la déconnexion est annulée, vous pourriez vouloir ramener l’utilisateur à l’endroit où il se trouvait avant de tenter de se déconnecter.
</Callout>

```kotlin lines
WebAuthProvider.logout(account)
  .withScheme("demo")
  .start(this, object: Callback<Void?, AuthenticationException> {
    override fun onSuccess(payload: Void?) {
      // L'utilisateur a été déconnecté!
    }

    override fun onFailure(error: AuthenticationException) {
      // Une erreur s'est produite!
    }
  })
```

<div id="show-the-users-profile">
  ## Afficher le profil de l&#39;utilisateur
</div>

Utilisez la classe `AuthenticationAPIClient` pour récupérer le profil de l&#39;utilisateur à partir d&#39;Auth0. Cela nécessite :

* Le <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> reçu lors de la phase de connexion
* Le scope `profile` à inclure lors de l&#39;appel à `WebAuthProvider.login`
* Le scope `email` (si vous souhaitez récupérer l&#39;adresse courriel de l&#39;utilisateur)

Cet exemple présente une fonction qui récupère le profil de l&#39;utilisateur et l&#39;affiche à l&#39;écran :

```kotlin lines
var client = AuthenticationAPIClient(account)

// Utilisez le jeton d'accès reçu pour appeler `userInfo` et obtenir le profil depuis Auth0.
client.userInfo(accessToken)
  .start(object : Callback<UserProfile, AuthenticationException> {
      override fun onFailure(exception: AuthenticationException) {
          // Une erreur s'est produite !
      }

      override fun onSuccess(profile: UserProfile) {
        // Nous avons le profil de l'utilisateur !
        val email = profile.email
        val name = profile.name
      }
})
```
