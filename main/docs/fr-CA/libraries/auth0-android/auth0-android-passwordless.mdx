<Warning>
  L’authentification Passwordless à partir des applications natives est désactivée par défaut pour les nouveaux tenants depuis le 8 juin 2017. Nous vous recommandons plutôt d’utiliser Universal Login et de passer par Web Authentication. Si vous souhaitez tout de même continuer, vous devrez d’abord activer le Grant Type MFA dans votre dashboard. Consultez [Application Grant Types](/docs/fr-CA/get-started/applications/application-grant-types) pour en savoir plus.
</Warning>

<Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> peut se faire par courriel ou par SMS, soit en envoyant à l’utilisateur un code, soit en envoyant un lien contenant un code. Toutes les méthodes d’authentification Passwordless nécessitent deux étapes : demander le code, puis le saisir pour vérification.

<div id="set-up-the-android-sdk">
  ## Configurer le SDK Android
</div>

D’abord, [configurez le SDK Android](/docs/fr-CA/libraries/auth0-android) afin de pouvoir utiliser les méthodes Passwordless ci-dessous.

<div id="configure-auth0-and-the-android-sdk">
  ## Configurer Auth0 et le SDK Android
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’utilisation de l’API Passwordless nécessite une configuration avec la version 1.20 ou ultérieure du SDK Android d’Auth0.
</Callout>

Pour utiliser l’API Passwordless à partir d’un client natif, activez le type d’autorisation Passwordless OTP pour votre application dans **Dashboard &gt; Applications &gt; (YOUR APPLICATION) &gt; Settings &gt; Advanced Settings &gt; Grant Types**.

<div id="request-the-code">
  ## Demander le code
</div>

Cet exemple demande le code en appelant `passwordlessWithEmail` avec l’adresse courriel de l’utilisateur, `PasswordlessType.CODE` et le nom de la connexion en paramètres. En cas de réussite, vous pourriez informer l’utilisateur que son code lui a été envoyé et, au besoin, le rediriger vers l’endroit où il devra le saisir.

```kotlin lines
authentication
    .passwordlessWithEmail("username@domain.com", PasswordlessType.CODE, "my-passwordless-connection")
    .start(object: Callback<Void?, AuthenticationException>() {
        override fun onSuccess(result: Void?) {
            // Code envoyé !
        }

        override fun onFailure(error: AuthenticationException) {
            // Erreur !
        }
    })
```

Vous pouvez utiliser la méthode `passwordlessWithSms` pour envoyer le code par SMS.

<div id="input-the-code">
  ## Saisissez le code
</div>

Une fois que l’utilisateur a reçu un code, il peut le saisir. Appelez la méthode `loginWithEmail` et transmettez l’adresse courriel de l’utilisateur, le code qu’il a reçu et le nom de la connexion concernée. Si l’opération réussit, vous recevez un objet `Credentials` en réponse.

```kotlin lines
authentication
    .loginWithEmail("username@domain.com", "123456", "my-passwordless-connection")
    .start(object: Callback<Credentials, AuthenticationException>() {
        override fun onSuccess(result: Credentials) {
            // Connecté !
        }

        override fun onFailure(error: AuthenticationException) {
            // Erreur !
        }
    })
```

Vous pouvez utiliser la méthode `loginWithSms` pour envoyer le code reçu par SMS et authentifier l’utilisateur.

Le scope utilisé par défaut est `openid profile email`.
