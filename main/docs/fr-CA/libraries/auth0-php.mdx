Le SDK Auth0-PHP s’intègre à vos applications PHP et offre des fonctionnalités simples de connexion et d’inscription. Il prend en charge les <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> sociaux comme Facebook, Google ou X, ainsi que les fournisseurs d’entreprise comme Active Directory. Le SDK fournit des méthodes pratiques pour accéder aux points de terminaison d’Authentication et de Management d’Auth0.

Le dépôt Auth0-PHP est open source et [hébergé sur GitHub](https://github.com/auth0/auth0-PHP). Nous apprécions toutes les contributions, y compris les signalements de bogues, les propositions d’amélioration et les demandes de tirage.

<div id="requirements">
  ## Prérequis
</div>

* PHP 7.4+ (8.0+ recommandé)
* [Composer](https://getcomposer.org/doc/00-intro.md)

<div id="installation">
  ## Installation
</div>

L’installation de l’Auth0 PHP SDK nécessite [Composer](https://getcomposer.org/doc/00-intro.md#installation-linux-unix-macos), l’outil standard de gestion des dépendances pour PHP. Composer vous permet de déclarer les bibliothèques dont votre projet a besoin et les installe pour vous. Assurez-vous que Composer est installé et accessible depuis votre shell avant de continuer.

Ensuite, exécutez la commande shell suivante dans le répertoire de votre projet pour installer le SDK :

```php lines
composer require auth0/auth0-php
```

Cela créera un sous-dossier `vendor` dans votre projet et téléchargera toutes les dépendances nécessaires pour utiliser le SDK PHP. Cela créera également un fichier `vendor/autoload.php`, nécessaire au fonctionnement du SDK avec votre application, que nous importerons plus tard.

<div id="getting-started">
  ## Pour commencer
</div>

Pour utiliser les API Authentication et Management d’Auth0, vous aurez besoin d’un compte Auth0 gratuit et d’une Application :

1. Rendez-vous sur [auth0.com/signup](https://auth0.com/signup) et créez votre compte.
2. Une fois dans le Dashboard, accédez à **Applications**, puis à **Create Application**.
3. Donnez un nom à votre Application, sélectionnez **Regular Web Application**, puis **Create**
4. Cliquez sur l’onglet **Settings** pour obtenir les identifiants requis ci-dessous. Pour en savoir plus, consultez [Application Settings](/docs/fr-CA/get-started/applications/application-settings).

<div id="configure-the-sdk">
  ## Configurer le SDK
</div>

Il est recommandé d’utiliser des [variables d’environnement](https://secure.php.net/manual/en/reserved.variables.environment.php) pour stocker et charger les identifiants Auth0 sensibles. Vous éviterez ainsi d’avoir à les coder en dur dans votre application. Créons un fichier `.env` à la racine du répertoire de notre projet pour y stocker les identifiants de l’application :

La façon la plus simple d’utiliser des variables d’environnement dans votre projet est d’employer une bibliothèque comme [PHP Dotenv](https://github.com/josegonzalez/php-dotenv) avec un fichier `.env` local. Créez un fichier `.env` (assurez-vous qu’il n’est pas accessible publiquement et qu’il est exclu du contrôle de version), puis ajoutez les valeurs suivantes :

export const codeExample = `# L'URL du domaine du tenant Auth0.
# Si nous utilisons un domaine personnalisé, assurez-vous de définir cette valeur à la place.
AUTH0_DOMAIN='https://{yourDomain}'

# L'ID client de notre application Auth0.
AUTH0_CLIENT_ID='{yourClientId}'

# Le secret client de notre application Auth0.
AUTH0_CLIENT_SECRET='{yourClientSecret}'

# Une longue valeur secrète que nous utiliserons pour chiffrer les cookies de session. Vous pouvez la générer à l'aide de \`openssl rand -hex 32\` dans notre shell.
AUTH0_COOKIE_SECRET='VOIR LE COMMENTAIRE CI-DESSUS'

# L'URL de base de notre application.
AUTH0_BASE_URL='http://127.0.0.1:3000'`;

<AuthCodeBlock children={codeExample} language="php" />

Vous ne devriez jamais enregistrer ce fichier dans un système de contrôle de version ni le partager de façon non sécuritaire. Son contenu doit être manipulé avec soin et traité comme un mot de passe.
Comme PHP ne peut pas lire nativement notre fichier `.env`, vous devrez installer une bibliothèque PHP pour le faire. Pour les besoins de cette documentation, nous utiliserons `vlucas/phpdotenv`, mais n’importe quelle bibliothèque « dotenv » de votre choix fera l’affaire. Depuis le répertoire de votre projet, exécutez la commande shell suivante pour installer la bibliothèque :

```php lines
composer require vlucas/phpdotenv
```

<div id="initialize-the-sdk">
  ## Initialiser le SDK
</div>

Nous sommes prêts à configurer et à initialiser une instance du SDK dans notre nouvelle application PHP. Commençons par créer le fichier source PHP que nous utiliserons pour cette démonstration, `index.php`, puis utilisons l’extrait suivant pour nous lancer :

```php lines
<?php

// Importer le chargeur automatique de Composer pour rendre les classes du SDK accessibles :
require 'vendor/autoload.php';

// Charger nos variables d'environnement depuis le fichier .env :
(Dotenv\Dotenv::createImmutable(__DIR__))->load();

// Instancier maintenant la classe Auth0 avec notre configuration :
$auth0 = new \Auth0\SDK\Auth0([
    'domain' => $_ENV['AUTH0_DOMAIN'],
    'clientId' => $_ENV['AUTH0_CLIENT_ID'],
    'clientSecret' => $_ENV['AUTH0_CLIENT_SECRET'],
    'cookieSecret' => $_ENV['AUTH0_COOKIE_SECRET']
]);
```

Félicitations ! Votre application est maintenant configurée et prête à être utilisée avec Auth0.
