<div id="where-can-i-see-errors-related-to-my-screen-and-how-to-handle-them">
  ## Où puis-je voir les erreurs liées à l’écran et comment les gérer
</div>

Le SDK offre plusieurs façons de détecter et de consulter les erreurs pendant le processus :

1. **Méthode `getErrors` pour les erreurs spécifiques à l’écran**
   La fonction getter de niveau supérieur `getErrors` permet de récupérer des erreurs d’authentification détaillées lorsque la validation côté serveur échoue.
   ```javascript
      import Login from "@auth0/auth0-acul-js/login";

      const loginIdManager = new Login();

      const errors = loginIdManager.getErrors();
      if (errors) {
      console.error("Login failed:", errors);
   }
   ```
2. **Erreurs spécifiques à l’écran**
   Chaque écran comprend une propriété `transaction.errors` qui renvoie une liste d’erreurs propres à cet écran. Ces erreurs sont générées après la validation finale de la requête, ce qui vous aide à repérer les problèmes à corriger.

En utilisant ces méthodes, vous pouvez facilement repérer et corriger les erreurs tout au long du flux du SDK.

<div id="can-i-use-this-sdk-from-a-cdn">
  ## Puis-je utiliser ce SDK à partir d’un CDN ?
</div>

À l’heure actuelle, ce SDK n’est disponible à l’installation que par le registre npm et ne peut pas être utilisé directement à partir d’un CDN.

Pour installer le SDK, exécutez la commande suivante :

```sh
npm install @auth0/auth0-acul-js
```

Une fois installé, vous pouvez l’importer et l’utiliser dans vos projets JavaScript, TypeScript ou ReactJS.

<div id="can-i-build-a-reactjs-application-with-this-sdk">
  ## Puis-je créer une application ReactJS avec ce SDK?
</div>

Oui, ce SDK s’intègre facilement à une application ReactJS.

Bien que le SDK soit écrit en JavaScript pur, il est entièrement compatible avec tous les frameworks basés sur JavaScript, notamment React, Angular, Vue et d’autres. Vous pouvez l’utiliser sans problème dans vos composants ou hooks React.
