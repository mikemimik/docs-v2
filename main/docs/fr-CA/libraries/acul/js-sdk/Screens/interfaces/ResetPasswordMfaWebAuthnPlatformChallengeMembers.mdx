<ParamField body="continueWithPasskey" type="Promise<void>">
  Lance le défi de l’authentificateur de plateforme WebAuthn en appelant `navigator.credentials.get()`
  à l’aide des options de défi fournies dans `screen.publicKey`.
  En cas de réussite, soumet l’assertion d’authentification obtenue à Auth0 avec `action: "default"`.

  Une Promise qui se résout lorsque la tentative de vérification est soumise.
  Une opération réussie entraîne généralement une redirection.

  #### Exceptions

  Lève une erreur si `screen.publicKey` est absent,
  si `navigator.credentials.get()` échoue (p. ex., annulation par l’utilisateur `NotAllowedError`),
  ou si la soumission du formulaire à Auth0 échoue. Si `navigator.credentials.get()` échoue
  avec une `DOMException`, il est recommandé d’intercepter cette erreur et d’appeler `reportBrowserError`.

  ```typescript Example
  // En supposant que 'sdk' est une instance de ResetPasswordMfaWebAuthnPlatformChallenge
  try {
    await sdk.continueWithPasskey({
      rememberDevice: true // si l’utilisateur a coché la case et que sdk.screen.showRememberDevice est true
    });
    // En cas de réussite, Auth0 gère la redirection.
  } catch (error) {
    console.error("Platform authenticator verification failed:", error);
    // S’il s’agit d’une erreur de l’API WebAuthn (DOMException), la signaler
    if (error instanceof DOMException && error.name && error.message) {
      await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
    }
    // Vérifiez sdk.transaction.errors pour les messages de validation côté serveur si la page se recharge.
  }
  ```

<ParamField body="reportBrowserError" type="Promise<void>">
  Signale une erreur côté navigateur survenue pendant l’appel d’API `navigator.credentials.get()`.
  Cela sert à informer Auth0 de problèmes comme l’annulation par l’utilisateur, le délai d’expiration ou un matériel non pris en charge.
  Soumet les détails de l’erreur avec `action: "showError::{errorDetailsJsonString}"`.

  Une Promise qui se résout lorsque le signalement de l’erreur est soumis.

  #### Exceptions

  Lève une erreur si la soumission du formulaire échoue (p. ex., erreur réseau).

  ```typescript Example
  // En supposant que 'sdk' est une instance de ResetPasswordMfaWebAuthnPlatformChallenge
  // et que 'webAuthnError' est une DOMException provenant d’un appel échoué à navigator.credentials.get().
  try {
    await sdk.reportBrowserError({
      error: { name: webAuthnError.name, message: webAuthnError.message }
    });
    // Auth0 peut réafficher la page ou rediriger selon l’erreur.
  } catch (submitError) {
    console.error("Failed to report WebAuthn browser error:", submitError);
  }
  ```

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Permet à l’utilisateur de ne pas utiliser le défi de plateforme WebAuthn et de sélectionner une autre méthode d’AMF.
  Cette action soumet `action: "pick-authenticator"` à Auth0, ce qui devrait rediriger
  l’utilisateur vers un écran de sélection du facteur d’AMF pour le processus de réinitialisation du mot de passe.

  Une Promise qui est résolue lorsque l’action « pick-authenticator » est soumise.

  #### Exceptions

  Si la soumission du formulaire échoue.

  ```typescript Example
  // En supposant que 'sdk' est une instance de ResetPasswordMfaWebAuthnPlatformChallenge
  try {
    await sdk.tryAnotherMethod();
    // En cas de succès, Auth0 gère la redirection vers la sélection d’une méthode d’AMF.
  } catch (error) {
    console.error("Failed to switch MFA method:", error);
  }
  ```
