   * // Après l'attente, vérifiez toujours sdk.transaction.errors pour les problèmes de validation côté serveur.

<ParamField body="continueWithNewNickname" type="Promise<void>">
  Soumet le nouveau surnom de la clé de sécurité WebAuthn.
  Cette action correspond au moment où l’utilisateur saisit un nouveau surnom et clique sur le bouton « Enregistrer » ou « Continuer ».
  Le SDK enverra ce nouveau surnom par requête POST au point de terminaison Auth0 `/u/mfa-webauthn-change-key-nickname`.

  Si le nouveau surnom est valide et que la mise à jour réussit, Auth0 redirigera généralement l’utilisateur
  vers l’écran suivant approprié (par ex., vers la gestion des facteurs AMF ou un écran de confirmation de réussite).
  Si le surnom n’est pas valide (par ex., vide, trop long ou trop court), Auth0 réaffichera l’écran
  &#39;mfa-webauthn-change-key-nickname&#39;, et le tableau `transaction.errors` dans le contexte du SDK
  sera mis à jour avec les détails de l’échec de validation.

  Une Promise qui se résout lorsque la soumission du formulaire est lancée.
  Elle ne renvoie pas directement de données lorsqu’elle se résout, puisqu’une redirection ou
  le réaffichage de la page constitue généralement le résultat.

  #### Lève

  Lève une erreur si `payload.nickname` n’est pas fourni ou n’est pas une chaîne de caractères,
  ou si le `FormHandler` rencontre un problème irrécupérable pendant la soumission (par ex., une erreur réseau).
  Les erreurs de validation provenant d’Auth0 (comme un surnom invalide) ne sont pas levées comme erreurs JavaScript,
  mais se reflètent dans `this.transaction.errors` après l’opération.

  ```typescript Example
  // En supposant que 'sdk' est une instance de MfaWebAuthnChangeKeyNickname
  const newNickname = "My Favorite YubiKey"; // Valeur saisie par l’utilisateur
  try {
    await sdk.continueWithNewNickname({ nickname: newNickname });
    // En cas de réussite, la page est redirigée.
  } catch (error) {
    // Ce catch sert à intercepter les erreurs inattendues pendant l’appel du SDK lui-même.
    console.error("Failed to submit the new nickname:", error);
  }
  // Après le await, vérifiez toujours sdk.transaction.errors pour les problèmes de validation côté serveur.
  if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
    sdk.transaction.errors.forEach(err => {
      if (err.field === 'nickname') {
        // Affiche err.message lié au champ de saisie du surnom.
        // p. ex., "Name is too short", "Name is required"
      }
    });
  }
  ```
