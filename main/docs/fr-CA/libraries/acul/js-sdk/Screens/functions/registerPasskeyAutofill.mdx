title: "registerPasskeyAutofill"
---

> **registerPasskeyAutofill**(`inputId`?: `string`): `Promise`&lt;`void`&gt;

Initialise une requête passive d’information d’identification WebAuthn à l’aide de `navigator.credentials.get()` avec `mediation: "conditional"`. Lorsqu’elle est prise en charge, cette fonctionnalité permet au navigateur d’afficher les clés d’accès enregistrées directement dans la liste déroulante de saisie semi-automatique du champ de nom d’utilisateur. La sélection d’une clé d’accès complète l’authentification sans nécessiter d’interaction supplémentaire de l’utilisateur.

Appelez cette méthode une seule fois lorsque l’écran de connexion est initialisé (par exemple, au chargement de la page). Si la médiation conditionnelle n’est pas prise en charge par le navigateur, la méthode ne fait rien, en toute sécurité.

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="inputId?" type="string">
  L’ID de l’élément `<input>` du nom d’utilisateur (sans `#`). Exemple : `"username"`. Lorsqu’il est fourni, le SDK valide l’existence de l’élément, confirme qu’il s’agit d’un `<input>` et remplace son attribut `autocomplete` par `"username webauthn"`.

  S’il est omis, vous devez configurer manuellement l’élément `<input>` :

  ```html
  <input id="username" autocomplete="username webauthn" />
  ```
</ParamField>

<div id="returns">
  ## Retourne
</div>

`Promise`&lt;`void`&gt;

Se résout une fois l’inscription à l’UI conditionnelle terminée.

<div id="example">
  ## Exemple
</div>

```typescript
import LoginId from '@auth0/auth0-acul-js/login-id';

async function initializeLogin() {
  const loginId = new LoginId();
  // Assurez-vous que l'élément HTML input associé existe :
  // <input id="username" autocomplete="username webauthn" />
  await loginId.registerPasskeyAutofill('username');
}

initializeLogin().catch(console.error);
```

<div id="gotchas">
  ## Points à noter
</div>

* L’attribut `autocomplete` **doit contenir exactement** `"username webauthn"`. Si vous incluez des jetons sans rapport, comme `"email"` ou `"text"`, les navigateurs n’afficheront pas le menu déroulant des clés d’accès.
* L’écrasement de l’attribut est intentionnel et nécessaire pour garantir un comportement uniforme d’un navigateur à l’autre. Ne vous fiez pas à la fusion ou à l’ajout de valeurs `autocomplete` existantes.
* Si la médiation conditionnelle n’est pas prise en charge par le navigateur, le SDK n’effectuera simplement aucune opération.

<div id="remarks">
  ## Remarques
</div>

Cette méthode s’appuie sur l’utilitaire interne `registerPasskeyAutofill()` et renvoie un `AbortController` en arrière-plan pour gérer la durée de la requête. Elle ne doit être appelée qu’une seule fois durant le cycle de vie de la page.
