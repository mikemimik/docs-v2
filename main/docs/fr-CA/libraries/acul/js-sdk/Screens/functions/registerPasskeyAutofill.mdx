---
title: "registerPasskeyAutofill"
---

> **registerPasskeyAutofill**(`inputId`?: `string`): `Promise`&lt;`void`&gt;

Initialise une requête passive d’information d’identification WebAuthn à l’aide de `navigator.credentials.get()` avec `mediation: "conditional"`. Lorsqu’elle est prise en charge, elle permet au navigateur d’afficher les clés d’accès enregistrées directement dans le menu déroulant de saisie semi-automatique du champ de nom d’utilisateur. La sélection d’une clé d’accès complète l’authentification sans nécessiter d’interaction supplémentaire de l’utilisateur.

Appelez cette méthode une seule fois lorsque l’écran de connexion est initialisé (par exemple, au chargement de la page). Si Conditional Mediation n’est pas pris en charge par le navigateur, la méthode est simplement sans effet.

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="inputId?" type="string">
  L’ID de l’élément `<input>` pour le nom d’utilisateur (sans `#`). Exemple : `"username"`. Lorsqu’il est fourni, le SDK vérifie que l’élément existe, confirme qu’il s’agit d’un `<input>` et remplace son attribut `autocomplete` par `"username webauthn"`.

  S’il est omis, vous devez configurer manuellement l’élément `<input>` :

  ```html
  <input id="username" autocomplete="username webauthn" />
  ```
</ParamField>

<div id="returns">
  ## Renvoie
</div>

`Promise`&lt;`void`&gt;

Se résout lorsque l’inscription à l’UI conditionnelle est terminée.

<div id="example">
  ## Exemple
</div>

```typescript
import LoginId from '@auth0/auth0-acul-js/login-id';

async function initializeLogin() {
  const loginId = new LoginId();
  // Assurez-vous que l'entrée HTML associée existe :
  // <input id="username" autocomplete="username webauthn" />
  await loginId.registerPasskeyAutofill('username');
}

initializeLogin().catch(console.error);
```

<div id="gotchas">
  ## Points à noter
</div>

* L’attribut `autocomplete` **doit contenir exactement** `"username webauthn"`. L’ajout de valeurs non pertinentes comme `"email"` ou `"text"` empêchera les navigateurs d’afficher la liste déroulante des clés d’accès.
* Le remplacement de l’attribut est intentionnel et nécessaire pour garantir un comportement uniforme d’un navigateur à l’autre. Ne comptez pas sur la fusion ou l’ajout aux valeurs `autocomplete` existantes.
* Si le navigateur ne prend pas en charge Conditional Mediation, le SDK n’effectuera simplement aucune action, en toute sécurité.

<div id="remarks">
  ## Remarques
</div>

Cette méthode s’appuie sur l’utilitaire interne `registerPasskeyAutofill()` et renvoie un `AbortController` en arrière-plan pour gérer la durée de vie de la requête. Elle ne devrait être appelée qu’une seule fois par cycle de vie de la page.
