> **changeLanguage**(`options`: [`LanguageChangeOptions`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions)): `Promise`&lt;`void`&gt;

Soumet la nouvelle préférence de langue au serveur avec l’action `change-language`, puis réaffiche l’interface dans la langue sélectionnée.

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="options" type="LanguageChangeOptions" required>
  La configuration du changement de langue. Consultez [`LanguageChangeOptions`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).

  <Expandable title="options">
    <ParamField body="language" type="string" required>
      Un code de locale pour la langue cible (p. ex. `'en'`, `'fr'`, `'es'`). Il doit correspondre à l’une des locales activées dans votre Auth0 tenant.
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Portée d’enregistrement de la préférence de langue. Actuellement, seule la valeur `"session"` est prise en charge.
    </ParamField>
  </Expandable>
</ParamField>

<div id="returns">
  ## Renvoie
</div>

`Promise`&lt;`void`&gt;

Se résout une fois que la préférence de langue a été soumise et que l’écran s’affiche de nouveau dans la langue sélectionnée.
