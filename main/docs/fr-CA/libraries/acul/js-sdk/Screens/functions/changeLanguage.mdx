> **changeLanguage**(`options`: [`LanguageChangeOptions`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions)): `Promise`&lt;`void`&gt;

Soumet au serveur la nouvelle préférence de paramètre régional avec l’action `change-language` et réaffiche l’interface dans la langue sélectionnée.

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="options" type="LanguageChangeOptions" required>
  La configuration du changement de langue. Consultez [`LanguageChangeOptions`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).

  <Expandable title="options">
    <ParamField body="language" type="string" required>
      Un code de paramètre régional pour la langue cible (par ex. `'en'`, `'fr'`, `'es'`). Il doit correspondre à l’un des paramètres régionaux activés dans votre tenant Auth0.
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Portée de l’enregistrement de la préférence de langue. À l’heure actuelle, seule la valeur `"session"` est prise en charge.
    </ParamField>
  </Expandable>
</ParamField>

<div id="returns">
  ## Renvoie
</div>

`Promise`&lt;`void`&gt;

Se résout lorsque la préférence de paramètre régional a été transmise et que l’écran se réaffiche dans la langue sélectionnée.
