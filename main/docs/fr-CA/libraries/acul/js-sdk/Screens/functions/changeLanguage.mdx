> **changeLanguage**(`options`: [`LanguageChangeOptions`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions)): `Promise`&lt;`void`&gt;

Soumet la nouvelle préférence de langue au serveur au moyen de l’action `change-language` et réaffiche l’interface dans la langue sélectionnée.

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="options" type="LanguageChangeOptions" required>
  La configuration du changement de langue. Consultez [`LanguageChangeOptions`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).

  <Expandable title="options">
    <ParamField body="language" type="string" required>
      Un code de paramètre régional pour la langue cible (p. ex. `'en'`, `'fr'`, `'es'`). Il doit correspondre à l’un des paramètres régionaux activés dans votre tenant Auth0.
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Portée d’enregistrement de la préférence de langue. À l’heure actuelle, seul `"session"` est pris en charge.
    </ParamField>
  </Expandable>
</ParamField>

<div id="returns">
  ## Renvoie
</div>

`Promise`&lt;`void`&gt;

Se résout une fois la préférence de langue soumise et l’écran réaffiché dans la langue sélectionnée.
