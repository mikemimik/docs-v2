La classe LoginPasswordlessEmailCode implémente la fonctionnalité de l’écran `login-passwordless-email-code`. Cet écran recueille l’adresse courriel de l’utilisateur et le code.

<div id="constructors">
  ## Constructeurs
</div>

Créez une instance du gestionnaire d’écran LoginPasswordlessEmailCode :

```typescript Example
import LoginPasswordlessEmailCode from '@auth0/auth0-acul-js/login-passwordless-email-code';
const loginPasswordlessEmailCodeManager = new LoginPasswordlessEmailCode();
loginPasswordlessEmailCodeManager.submitCode({
  code: '123456',
});
```

<ParamField body="resendManager" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/ResendControl">ResendControl</a></span>}>
  Cette méthode offre une fonctionnalité de renvoi avec gestion du délai d’expiration. Elle retourne un objet [`ResendControl`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/ResendControl) doté d’une méthode `startResend()` qui déclenche le renvoi et impose une période d’attente avant que l’utilisateur puisse renvoyer de nouveau.

  ```typescript Example
  import LoginPasswordlessEmailCode from '@auth0/auth0-acul-js/login-passwordless-email-code';

  const loginPasswordlessEmailCodeManager = new LoginPasswordlessEmailCode();
  const { startResend } = loginPasswordlessEmailCodeManager.resendManager({
    timeoutSeconds: 15,
    onStatusChange: (remainingSeconds, isDisabled) => {
      console.log(`Resend available in ${remainingSeconds}s, disabled: ${isDisabled}`);
    },
    onTimeout: () => {
      console.log('Resend is now available');
    }
  });

  // Appeler startResend lorsque l’utilisateur clique sur le bouton de renvoi
  startResend();
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [StartResendOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/StartResendOptions).
    </ParamField>

    <ParamField body="timeoutSeconds?" type="number">
      La durée de la période d’attente, en secondes, avant que l’utilisateur puisse renvoyer le code à nouveau.
    </ParamField>

    <ParamField body="onStatusChange?" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/OnStatusChangeCallback">OnStatusChangeCallback</a></span>}>
      Fonction de rappel invoquée chaque fois que le statut du renvoi change; elle retourne le nombre de secondes restantes et indique si la fonctionnalité de renvoi est actuellement désactivée.
    </ParamField>

    <ParamField body="onTimeout?" type="() => void">
      Fonction de rappel invoquée lorsque la période d’attente expire et que le renvoi redevient possible.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="switchConnection" type="Promise<void>">
  Cette méthode permet de basculer entre une connexion de base de données (mot de passe) et une connexion sans mot de passe (courriel/SMS OTP).

  ```typescript Example
  import LoginPasswordlessEmailCode from '@auth0/auth0-acul-js/login-passwordless-email-code';
  const loginPasswordlessEmailCodeManager = new LoginPasswordlessEmailCode();
  loginPasswordlessEmailCodeManager.switchConnection({
    connection: 'email',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SwitchConnectionOptionsEmailCode](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/SwitchConnectionOptionsEmailCode).
    </ParamField>

    <ParamField body="connection" type="string" required>
      Le nom de la connexion vers laquelle basculer. Utilisez `'email'` ou `'sms'` pour une connexion sans mot de passe, ou un nom de connexion DB pour l’authentification par mot de passe.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22;">
      Champs personnalisés supplémentaires précédés de `ulp-` (par exemple, `'ulp-custom-field'`).
