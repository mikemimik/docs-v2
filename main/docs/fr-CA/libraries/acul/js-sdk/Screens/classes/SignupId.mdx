---
title: "SignupId"
description: "Décrit toutes les propriétés et méthodes disponibles pour personnaliser l’écran `signup-id` dans Universal Login."
---

La classe SignupId implémente les fonctionnalités de l’écran `signup-id`. Cet écran recueille l’identifiant de l’utilisateur.

<Frame>
  <img style={{maxHeight:"400px"}} src="/docs/images/cdy7uua7fh8z/5lzGYj0k4VcxvPCBYH37SO/ebd291cc0dbd673a70f93530f724bf65/Signup_ID.png" alt="Identifiant d’inscription" />
</Frame>

<div id="constructors">
  ## Constructeurs
</div>

Créez une instance du gestionnaire d’écran SignupId :

```typescript Example
import SignupId from '@auth0/auth0-acul-js/signup-id';
const signupIdManager = new SignupId();
signupIdManager.signup({
  email: 'test@example.com',
});
```

<div id="methods">
  ## Méthodes
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Cette méthode modifie la langue d’affichage de la page Universal Login.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      Le code de langue souhaité (par exemple, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Lorsqu’elle est définie à `'session'`, la langue sélectionnée est conservée pendant toute la session.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Champs personnalisés supplémentaires préfixés par `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedSignup" type="Promise<void>">
  Cette méthode gère l’inscription au moyen de différentes identités sociales. Par exemple : Google, Facebook, etc.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.federatedSignup({
    connection: 'google-oauth2',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [FederatedSignupOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      Le nom de la connexion sociale à utiliser.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Cette méthode récupère, à partir du contexte, le tableau des erreurs de transaction, ou un tableau vide s’il n’y en a pas. Elle renvoie un tableau d’objets d’erreur provenant du contexte de transaction.
</ParamField>

<ParamField body="getSignupIdentifiers" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Identifier">Identifier</a>[] | null</span>}>
  Cette méthode renvoie la liste des identifiants activés pour le formulaire `signup-id`, en indiquant pour chacun s’il est obligatoire ou facultatif selon la configuration de la transaction. Un tableau d’objets d’identifiant (par exemple : email, phone, username).

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  const identifiers = signupIdManager.getSignupIdentifiers();
  // [{ type: 'email', required: true }, { type: 'username', required: false }]
  ```
</ParamField>

<ParamField body="pickCountryCode" type="Promise<void>">
  Cette méthode redirige l’utilisateur vers la liste de sélection de l’indicatif de pays, où il peut mettre à jour l’indicatif de pays de son numéro de téléphone.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.pickCountryCode();
  ```
</ParamField>

<ParamField body="signup" type="Promise<void>">
  Cette méthode gère la configuration associée à `signup-id`. Elle permet à de nouveaux utilisateurs de s’inscrire au moyen de différents identifiants.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.signup({
    email: 'test@example.com',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SignupOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/SignupOptions).
    </ParamField>

    <ParamField body="email?" type="string">
      L’adresse courriel.
    </ParamField>

    <ParamField body="username?" type="string">
      L’identifiant de l’utilisateur.
    </ParamField>

    <ParamField body="phone?" type="string">
      Le numéro de téléphone de l’utilisateur.
    </ParamField>

    <ParamField body="captcha?" type="string">
      Le code captcha ou la réponse du fournisseur CAPTCHA. Cette propriété est requise si votre tenant Auth0 a [Bot Detection](/docs/fr-CA/secure/attack-protection/bot-detection) activé.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Champs personnalisés supplémentaires préfixés par `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validateUsername" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  Cette méthode valide un nom d’utilisateur donné en fonction de la politique de nom d’utilisateur actuelle définie dans le contexte de la transaction. Elle retourne un objet indiquant si le nom d’utilisateur est valide et pour quelle raison.

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  const result = signupIdManager.validateUsername('myusername');
  // result => { valid: true, errors: [] }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="username" type="string" required>
      La chaîne de caractères du nom d’utilisateur à valider.
    </ParamField>
  </Expandable>
</ParamField>
