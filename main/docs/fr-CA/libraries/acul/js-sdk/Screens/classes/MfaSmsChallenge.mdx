La classe MfaSmsChallenge implémente les fonctionnalités de l’écran `mfa-sms-challenge`. Cet écran s’affiche lorsque l’utilisateur doit saisir un code SMS pour vérifier son identité.

<div id="constructors">
  ## Constructeurs
</div>

Créez une instance du gestionnaire de l’écran AMF SMS Challenge :

```typescript Example
import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

const mfaSmsChallenge = new MfaSmsChallenge();
await mfaSmsChallenge.continueMfaSmsChallenge({
  code: '123456',
  rememberDevice: true,
});
```

<div id="methods">
  ## Méthodes
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Cette méthode modifie la langue d’affichage de la page Universal Login.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  mfaSmsChallenge.changeLanguage({
    language: 'fr',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      Le code de locale de la langue souhaitée (par exemple, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Lorsqu’elle est définie sur `'session'`, la langue sélectionnée est enregistrée pour toute la durée de la session.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="continueMfaSmsChallenge" type="Promise<void>">
  Cette méthode soumet le AMF SMS Challenge avec le code fourni.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.continueMfaSmsChallenge({
    code: '123456',
    rememberDevice: true,
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaSmsChallengeOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaSmsChallengeOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      Le code SMS saisi par l’utilisateur.
    </ParamField>

    <ParamField body="rememberDevice?" type="boolean">
      Lorsqu’il est défini sur `true`, cet appareil est enregistré comme fiable afin que l’utilisateur n’ait pas à effectuer l’AMF lors des connexions ultérieures à partir du même appareil.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getACall" type="Promise<void>">
  Cette méthode fait passer la méthode de vérification à un appel vocal.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.getACall();
  ```
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Cette méthode récupère le tableau des erreurs de transaction à partir du contexte, ou un tableau vide s’il n’y en a aucune.
</ParamField>

<ParamField body="pickSms" type="Promise<void>">
  Cette méthode mène à l’écran où l’utilisateur peut choisir un autre numéro SMS.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.pickSms();
  ```
</ParamField>

<ParamField body="resendCode" type="Promise<void>">
  Cette méthode renvoie le code de vérification SMS à l’utilisateur.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.resendCode();
  ```
</ParamField>

<ParamField body="resendManager" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/ResendControl">ResendControl</a></span>}>
  Cette méthode offre une fonctionnalité de renvoi avec un délai d’expiration configurable et une gestion du statut.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  const { startResend } = mfaSmsChallenge.resendManager({
    timeoutSeconds: 15,
    onStatusChange: (remainingSeconds, isDisabled) => {
      console.log(`Resend available in ${remainingSeconds}s, disabled: ${isDisabled}`);
    },
    onTimeout: () => {
      console.log('Resend is now available');
    },
  });

  startResend();
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [StartResendOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/StartResendOptions).
    </ParamField>

    <ParamField body="onStatusChange?" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/OnStatusChangeCallback">OnStatusChangeCallback</a></span>}>
      Fonction de rappel appelée chaque fois que le statut du renvoi change; elle reçoit le nombre de secondes restantes et indique si le renvoi est actuellement désactivé.
    </ParamField>

    <ParamField body="onTimeout?" type="() => void">
      Fonction de rappel appelée à l’expiration du délai de renvoi, lorsque l’option de renvoi redevient disponible.
    </ParamField>

    <ParamField body="timeoutSeconds?" type="number">
      Le nombre de secondes avant que l’option de renvoi devienne disponible.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Cette méthode permet à l’utilisateur d’essayer une autre méthode d’AMF.

  ```typescript Example
  import MfaSmsChallenge from '@auth0/auth0-acul-js/mfa-sms-challenge';

  const mfaSmsChallenge = new MfaSmsChallenge();
  await mfaSmsChallenge.tryAnotherMethod();
  ```
</ParamField>
