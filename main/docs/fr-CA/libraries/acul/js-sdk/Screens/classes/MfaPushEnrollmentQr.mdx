const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();
await mfaPushEnrollmentQr.pickAuthenticator();

<ParamField body="pollingManager" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode lance et gère le polling pour une inscription à l’AMF par notification push.

  Elle crée une session de polling qui vérifie de façon répétée le point de terminaison d’inscription à l’AMF par notification push à l’intervalle spécifié jusqu’à ce que l’inscription soit approuvée ou qu’une erreur survienne. Lorsque la condition d’approbation est remplie, la fonction de rappel [onCompleted](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) fournie est appelée et le polling s’arrête automatiquement.

  Utilisez le [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) retourné pour démarrer, arrêter ou vérifier l’état du processus de polling en tout temps.

  Retourne une instance de [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) qui expose :

  * `startPolling()` — Démarre ou reprend le polling.
  * `stopPolling()` — Annule le polling immédiatement.
  * `isRunning()` — Indique si le polling est actuellement actif.

  ```typescript Example
  import MfaPushEnrollmentQr from '@auth0/auth0-acul-js/mfa-push-enrollment-qr';

  const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();

  // Démarre le polling toutes les 5 secondes jusqu’à ce que l’inscription push soit approuvée
  const control = mfaPushEnrollmentQr.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushEnrollmentQr.continue(),
    onError: (error) => console.error('Erreur de polling :', error),
  });

  // Plus tard, annule le polling au besoin
  control.stopPolling();
  ```

  **Remarques**

  * Les réponses HTTP 429 (limite de débit) sont gérées en interne : le polling attend automatiquement que la limite de débit soit réinitialisée avant de réessayer.
  * Vous pouvez appeler `startPolling()` plusieurs fois sans risque; cela n’a aucun effet si le polling est déjà en cours.

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en millisecondes, entre des requêtes de polling consécutives. S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Fonction de rappel facultative exécutée une fois que l’inscription à l’AMF par notification push est approuvée avec succès et que le polling est terminé. Elle est appelée exactement une fois, après quoi le polling s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Fonction de rappel facultative appelée si une erreur survient pendant le polling. Elle reçoit un objet `ULError` contenant `status` (code d’état HTTP) et `responseText` (corps brut de la réponse).
    </ParamField>
  </Expandable>
</ParamField>
