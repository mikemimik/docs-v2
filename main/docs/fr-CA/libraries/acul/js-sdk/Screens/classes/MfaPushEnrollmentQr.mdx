<div id="constructors">
  ## Constructeurs
</div>

Créez une instance du gestionnaire de l’écran de code QR d’inscription à MFA Push :

```typescript Example
import MfaPushEnrollmentQr from '@auth0/auth0-acul-js/mfa-push-enrollment-qr';

const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();
await mfaPushEnrollmentQr.pickAuthenticator();
```

<ParamField body="pollingManager" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode lance et gère l’interrogation pour une inscription à l’AMF push.

  Elle crée une session d’interrogation qui vérifie de façon répétée le point de terminaison de l’inscription à l’AMF push à l’intervalle indiqué, jusqu’à ce que l’inscription soit approuvée ou qu’une erreur survienne. Lorsque la condition d’approbation est remplie, le callback [onCompleted](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) fourni est appelé et l’interrogation s’arrête automatiquement.

  Utilisez le [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) retourné pour démarrer, arrêter ou vérifier l’état du processus d’interrogation à tout moment.

  Retourne une instance [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) qui expose :

  * `startPolling()` — Démarre ou reprend l’interrogation.
  * `stopPolling()` — Annule immédiatement l’interrogation.
  * `isRunning()` — Indique si l’interrogation est actuellement active.

  ```typescript Example
  import MfaPushEnrollmentQr from '@auth0/auth0-acul-js/mfa-push-enrollment-qr';

  const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();

  // Démarrer l’interrogation toutes les 5 secondes jusqu’à ce que l’inscription push soit approuvée
  const control = mfaPushEnrollmentQr.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushEnrollmentQr.continue(),
    onError: (error) => console.error('Polling error:', error),
  });

  // Plus tard, annuler l’interrogation au besoin
  control.stopPolling();
  ```

  **Remarques**

  * Les réponses HTTP 429 (limite de débit) sont gérées en interne : l’interrogation attend automatiquement que la limite de débit soit réinitialisée avant de réessayer.
  * Vous pouvez appeler `startPolling()` plusieurs fois en toute sécurité ; cela n’a aucun effet si l’interrogation est déjà en cours.

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en millisecondes, entre des requêtes d’interrogation consécutives. S’il est omis, l’intervalle interne par défaut du SDK est utilisé (habituellement 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Callback facultatif exécuté une fois que l’inscription à l’AMF push a été approuvée avec succès et que l’interrogation est terminée. Il est appelé exactement une fois, après quoi l’interrogation s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Callback facultatif invoqué si une erreur survient pendant l’interrogation. Reçoit un objet `ULError` contenant `status` (code d’état HTTP) et `responseText` (corps de la réponse brut).
    </ParamField>
  </Expandable>
</ParamField>
