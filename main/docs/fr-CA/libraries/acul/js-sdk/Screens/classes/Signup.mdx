---
title: "Signup"
description: "Décrit toutes les propriétés et méthodes disponibles pour personnaliser l’écran `signup` d’Universal Login."
---

La classe Signup implémente la fonctionnalité de l’écran `signup`. Cet écran permet de saisir l’identifiant et le mot de passe de l’utilisateur. Selon votre tenant, cet identifiant peut être une adresse courriel, un numéro de téléphone ou un nom d’utilisateur.

<div id="constructors">
  ## Constructeurs
</div>

Créez une instance du gestionnaire de l’écran d’inscription :

```typescript Example
import Signup from '@auth0/auth0-acul-js/signup';
const signupManager = new Signup();
signupManager.signup({
  email: 'test@example.com',
  password: 'P@$wOrd123!',
});
```

<div id="methods">
  ## Méthodes
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Cette méthode change la langue d’affichage de la page Universal Login.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      Le code de locale de la langue souhaitée (par exemple, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Lorsqu’elle est définie à `'session'`, la langue sélectionnée est conservée pendant toute la durée de la session.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Des champs personnalisés supplémentaires préfixés par `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedSignup" type="Promise<void>">
  Cette méthode gère la soumission du formulaire d’inscription sociale. Elle est semblable à la méthode [signup](#signup), mais elle est utilisée pour les inscriptions sociales.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.federatedSignup({
    connection: 'google-oauth2',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [FederatedSignupPayloadOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupPayloadOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      Le nom de connexion sociale à utiliser.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Cette méthode récupère, à partir du contexte, le tableau des erreurs de transaction, ou un tableau vide s’il n’y en a aucune. Un tableau d’objets d’erreur provenant du contexte de transaction.
</ParamField>

<ParamField body="getSignupIdentifiers" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Identifier">Identifier</a>[] | null</span>}>
  Cette méthode renvoie la liste des identifiants activés pour le formulaire d’inscription, en indiquant pour chacun s’il est obligatoire ou facultatif selon la configuration de la transaction. Un tableau d’objets d’identifiant (par exemple, courriel, téléphone, nom d’utilisateur).

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const identifiers = signupManager.getSignupIdentifiers();
  // [{ type: 'email', required: true }, { type: 'username', required: false }]
  ```
</ParamField>

<ParamField body="pickCountryCode" type="Promise<void>">
  Cette méthode redirige l’utilisateur vers la liste de sélection de l’indicatif de pays, où il peut mettre à jour le préfixe du numéro de téléphone.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.pickCountryCode();
  ```
</ParamField>

<ParamField body="signup" type="Promise<void>">
  Cette méthode gère la soumission du formulaire d’inscription.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  signupManager.signup({
    email: 'test@example.com',
    password: 'P@$wOrd123!',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SignupPayloadOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/SignupPayloadOptions).
    </ParamField>

    <ParamField body="email?" type="string">
      L’adresse courriel.
    </ParamField>

    <ParamField body="username?" type="string">
      L’identifiant de l’utilisateur.
    </ParamField>

    <ParamField body="phoneNumber?" type="string">
      Le numéro de téléphone de l’utilisateur.
    </ParamField>

    <ParamField body="password?" type="string">
      Le mot de passe de l’utilisateur.
    </ParamField>

    <ParamField body="captcha?" type="string">
      Le code captcha ou la réponse du fournisseur CAPTCHA. Cette propriété est requise si [Bot Detection](/docs/fr-CA/secure/attack-protection/bot-detection) est activé dans votre tenant Auth0.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Des champs personnalisés supplémentaires préfixés par `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validatePassword" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/PasswordValidationResult">PasswordValidationResult</a></span>}>
  Cette méthode renvoie un objet de type [PasswordValidationResult](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/PasswordValidationResult) indiquant si le mot de passe est valide et, le cas échéant, pourquoi.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const result = signupManager.validatePassword('P@$wOrd123!');
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="password" type="string" required>
      Le mot de passe de l’utilisateur.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validateUsername" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  Cette méthode valide un nom d’utilisateur donné en fonction de la politique de nom d’utilisateur actuelle définie dans le contexte de la transaction. Elle renvoie un objet indiquant si le nom d’utilisateur est valide et, le cas échéant, pourquoi.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';
  const signupManager = new Signup();
  const result = signupManager.validateUsername('myusername');
  // result => { valid: true, errors: [] }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="username" type="string" required>
      La chaîne de nom d’utilisateur à valider.
    </ParamField>
  </Expandable>
</ParamField>
