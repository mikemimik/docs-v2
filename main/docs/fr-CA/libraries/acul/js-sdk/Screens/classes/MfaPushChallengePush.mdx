La classe MfaPushChallengePush implémente les fonctionnalités de l’écran `mfa-push-challenge-push`. Cet écran s’affiche lorsqu’un utilisateur doit confirmer une notification push pendant l’authentification AMF.

  ## Constructeurs

```typescript Example
import MfaPushChallengePush from '@auth0/auth0-acul-js/mfa-push-challenge-push';

const mfaPushChallengePush = new MfaPushChallengePush();
await mfaPushChallengePush.continue();
```

  ```typescript Example
  const mfaPushChallengePush = new MfaPushChallengePush();
  mfaPushChallengePush.changeLanguage({
    language: 'fr',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).

    <ParamField body="language" type="string" required>

<ParamField body="pollingManager" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode démarre et gère le polling pour un défi push AMF.

  Elle crée une session de polling qui vérifie de façon répétée l’endpoint du défi push AMF à l’intervalle indiqué, jusqu’à ce que le défi soit approuvé ou qu’une erreur survienne. Lorsque la condition d’approbation est remplie, la fonction de rappel [onCompleted](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) fournie est invoquée et le polling s’arrête automatiquement.

  Utilisez le [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) retourné pour démarrer, arrêter ou vérifier le statut du processus de polling à tout moment.

  Retourne une instance [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) qui expose :

  * `startPolling()` — Démarre ou reprend le polling.
  * `stopPolling()` — Annule le polling immédiatement.
  * `isRunning()` — Indique si le polling est actuellement actif.

  ```typescript Example
  import MfaPushChallengePush from '@auth0/auth0-acul-js/mfa-push-challenge-push';

  const mfaPushChallengePush = new MfaPushChallengePush();

  // Démarre le polling toutes les 5 secondes jusqu’à ce que le défi push soit approuvé
  const control = mfaPushChallengePush.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushChallengePush.continue(),
    onError: (error) => console.error('Erreur de polling :', error),
  });

  // Plus tard, annulez le polling au besoin
  control.stopPolling();
  ```

  **Remarques**

  * Les réponses HTTP 429 (rate limit) sont gérées en interne : le polling attend automatiquement que la limite soit réinitialisée avant de réessayer.
  * Vous pouvez appeler `startPolling()` plusieurs fois sans risque; cela n’a aucun effet s’il est déjà en cours d’exécution.

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en millisecondes, entre des requêtes de polling consécutives. S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Fonction de rappel facultative exécutée une fois que le défi push AMF a été approuvé et que le polling est terminé. Elle est appelée exactement une fois, après quoi le polling s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Fonction de rappel facultative invoquée si une erreur se produit pendant le polling. Reçoit un objet `ULError` contenant `status` (code d’état HTTP) et `responseText` (corps de la réponse brut).
    </ParamField>
  </Expandable>
</ParamField>
