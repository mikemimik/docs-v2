La classe MfaPushChallengePush implémente les fonctionnalités de l’écran `mfa-push-challenge-push`. Cet écran s’affiche lorsqu’un utilisateur doit confirmer une notification push dans le cadre de l’AMF.

<ParamField body="pollingManager" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode lance et gère le polling d’un défi push d’AMF.

  Elle crée une session de polling qui vérifie de façon répétée l’endpoint du défi push d’AMF selon l’intervalle indiqué, jusqu’à ce que le challenge soit approuvé ou qu’une erreur survienne. Lorsque la condition d’approbation est remplie, la fonction de rappel [onCompleted](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) fournie est appelée et le polling s’arrête automatiquement.

  Utilisez l’instance [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) retournée pour démarrer, arrêter ou vérifier l’état du processus de polling en tout temps.

  Retourne une instance [MfaPushPollingControl](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) qui expose :

  * `startPolling()` — Démarre ou reprend le polling.
  * `stopPolling()` — Arrête le polling immédiatement.
  * `isRunning()` — Indique si le polling est actuellement actif.

  ```typescript Example
  import MfaPushChallengePush from '@auth0/auth0-acul-js/mfa-push-challenge-push';

  const mfaPushChallengePush = new MfaPushChallengePush();

  // Démarrer le polling toutes les 5 secondes jusqu’à ce que le défi push d’AMF soit approuvé
  const control = mfaPushChallengePush.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushChallengePush.continue(),
    onError: (error) => console.error('Polling error:', error),
  });

  // Plus tard, arrêter le polling au besoin
  control.stopPolling();
  ```

  **Remarques**

  * Les réponses HTTP 429 (rate limit) sont gérées à l’interne : le polling attend automatiquement que la limite soit réinitialisée avant de réessayer.
  * Vous pouvez appeler `startPolling()` plusieurs fois en toute sécurité ; cela n’a aucun effet si le processus est déjà en cours.

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en millisecondes, entre des requêtes de polling consécutives. S’il est omis, l’intervalle interne par défaut du SDK est utilisé (habituellement 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Fonction de rappel facultative exécutée une fois que le défi push d’AMF a été approuvé avec succès et que le polling est terminé. Elle est appelée exactement une fois, après quoi le polling s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Fonction de rappel facultative invoquée si une erreur survient pendant le polling. Elle reçoit un objet `ULError` contenant `status` (code d’état HTTP) et `responseText` (corps de la réponse brut).
    </ParamField>
  </Expandable>
</ParamField>
