title: "SignupPassword"
description: "Décrit toutes les propriétés et méthodes disponibles pour personnaliser l’écran `signup-password` de l’Universal Login."
---

La classe SignupPassword implémente les fonctionnalités de l’écran `signup-password`. Cet écran recueille le mot de passe de l’utilisateur.

  <img style={{maxHeight:"400px"}} src="/docs/images/cdy7uua7fh8z/5h85r3sVhe8mVw7rDkDwTD/02c3521d9d6c4d490c1a64ae37ecca72/Signup_Password_with_Flexible_IDs.png" alt="Mot de passe d’inscription avec ID flexibles" />
</Frame>

<div id="constructors">
  ## Constructeurs
</div>

Créez une instance du gestionnaire de l’écran SignupPassword :

```typescript Example
import SignupPassword from '@auth0/auth0-acul-js/signup-password';
const signupPasswordManager = new SignupPassword();
signupPasswordManager.signup({
  email: 'test@example.com',
  password: 'P@$wOrd123!',
});
```

<div id="methods">
  ## Méthodes
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Cette méthode modifie la langue d’affichage de la page Universal Login.

  ```typescript Example
  import SignupPassword from '@auth0/auth0-acul-js/signup-password';
  const signupPasswordManager = new SignupPassword();
  signupPasswordManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      Le code de langue de la langue souhaitée (par exemple, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Lorsqu’elle est définie sur `'session'`, la langue sélectionnée est conservée pendant toute la durée de la session.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Champs personnalisés supplémentaires préfixés par `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedSignup" type="Promise<void>">
  Cette méthode permet l’inscription au moyen de différentes identités sociales. Par exemple : Google, Facebook, etc.

  ```typescript Example
  import SignupPassword from '@auth0/auth0-acul-js/signup-password';
  const signupPasswordManager = new SignupPassword();
  signupPasswordManager.federatedSignup({
    connection: 'google-oauth2',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [FederatedSignupPasswordPayloadOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupPasswordPayloadOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      Le nom de la connexion sociale à utiliser.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Cette méthode récupère, à partir du contexte, le tableau des erreurs de la transaction, ou un tableau vide s’il n’y en a aucune.
</ParamField>

<ParamField body="signup" type="Promise<void>">
  Cette méthode gère la soumission du formulaire d’inscription avec mot de passe.

  ```typescript Example
  import SignupPassword from '@auth0/auth0-acul-js/signup-password';
  const signupPasswordManager = new SignupPassword();
  signupPasswordManager.signup({
    email: 'test@example.com',
    password: 'P@$wOrd123!',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SignupPasswordOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/SignupPasswordOptions).
    </ParamField>

    <ParamField body="email?" type="string">
      L’adresse courriel de l’utilisateur.
    </ParamField>

    <ParamField body="username?" type="string">
      L’identifiant de l’utilisateur.
    </ParamField>

    <ParamField body="phoneNumber?" type="string">
      Le numéro de téléphone de l’utilisateur.
    </ParamField>

    <ParamField body="password" type="string" required>
      Le mot de passe de l’utilisateur.
    </ParamField>

    <ParamField body="captcha?" type="string">
      Le code captcha ou la réponse du fournisseur CAPTCHA. Cette propriété est requise si la fonctionnalité [Bot Detection](/docs/fr-CA/secure/attack-protection/bot-detection) est activée dans votre tenant Auth0.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Champs personnalisés supplémentaires préfixés par `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="switchConnection" type="Promise<void>">
  Cette méthode bascule la connexion d’authentification pendant le flux signup-password.

  ```typescript Example
  import SignupPassword from '@auth0/auth0-acul-js/signup-password';
  const signupPasswordManager = new SignupPassword();
  signupPasswordManager.switchConnection({
    connection: 'Username-Password-Authentication',
  });
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SwitchConnectionOptionsSignupPassword](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/SwitchConnectionOptionsSignupPassword).
    </ParamField>

    <ParamField body="connection" type="string" required>
      L’identifiant de la connexion vers laquelle basculer pendant le flux d’authentification signup-password.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22;">
      Champs personnalisés supplémentaires préfixés par `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validatePassword" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/PasswordValidationResult">PasswordValidationResult</a></span>}>
  Cette méthode valide une chaîne de mot de passe en fonction de la politique de mots de passe de la transaction en cours. Elle renvoie un objet indiquant si le mot de passe est valide et, le cas échéant, pourquoi.

  ```typescript Example
  import SignupPassword from '@auth0/auth0-acul-js/signup-password';
  const signupPasswordManager = new SignupPassword();
  const result = signupPasswordManager.validatePassword('MyP@ssw0rd!');
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="password" type="string" required>
      La chaîne de mot de passe à valider.
    </ParamField>
  </Expandable>
</ParamField>
