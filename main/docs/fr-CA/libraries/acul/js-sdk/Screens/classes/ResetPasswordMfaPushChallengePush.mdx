La classe ResetPasswordMfaPushChallengePush implémente la fonctionnalité de l’écran `reset-password-mfa-push-challenge-push`. Cet écran s’affiche lorsqu’une notification push a été envoyée à l’appareil de l’utilisateur à l’étape d’AMF du flux de réinitialisation du mot de passe.

<ParamField body="pollingManager" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode lance et gère la vérification périodique du défi push d’AMF. Elle vérifie à intervalles réguliers si la notification push a été approuvée et appelle le callback `onCompleted` lorsque l’approbation est détectée. Les réponses HTTP 429 de limite de débit sont gérées en interne — la vérification périodique attend que la limite de débit soit réinitialisée avant de réessayer.

  Le [`MfaPushPollingControl`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) renvoyé expose `startPolling()`, `stopPolling()` et `isRunning()` pour gérer le cycle de vie de la vérification périodique.

  ```typescript Example
  import ResetPasswordMfaPushChallengePush from '@auth0/auth0-acul-js/reset-password-mfa-push-challenge-push';
  const resetPasswordMfaPushChallengePushManager = new ResetPasswordMfaPushChallengePush();
  const control = resetPasswordMfaPushChallengePushManager.pollingManager({
    intervalMs: 5000,
    onCompleted: () => resetPasswordMfaPushChallengePushManager.continue(),
    onError: (error) => console.error('Polling error:', error),
  });

  control.startPolling();

  // Annuler la vérification périodique au besoin
  control.stopPolling();
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      L’intervalle, en millisecondes, entre les tentatives de vérification.
    </ParamField>

    <ParamField body="onCompleted?" type="() => void">
      Callback appelé lorsque le défi push est approuvé et que la vérification périodique s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError?" type="(error: Error) => void">
      Callback appelé si une erreur survient pendant la vérification périodique.
    </ParamField>
  </Expandable>
</ParamField>
