<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer un processus d’interrogation d’une notification push AMF.

  Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
  à laquelle l’endpoint du challenge est interrogé et la manière dont la réussite ou l’échec sont gérés.

  ```ts Example
  export type MfaPollingOptions = {
    /**
     * Intervalle facultatif, en **millisecondes**, entre deux requêtes d’interrogation consécutives.
     *
     * - S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5000 ms).
     * - Choisissez une valeur qui offre un bon équilibre entre la réactivité et les limites de débit.
     *
     * @defaultValue Dépend de l’implémentation (généralement 5000 ms)
     */
    intervalMs?: number;

    /**
     * Fonction de rappel facultative exécutée une fois que le MFA push challenge est
     * approuvé avec succès et que l’interrogation est terminée.
     *
     * - Appelée exactement **une** fois, après quoi l’interrogation s’arrête automatiquement.
     * - Utile pour faire progresser le flux de connexion (par ex., appeler `continue()`).
     */
    onCompleted?: () => void;

    /**
     * Fonction de rappel facultative invoquée si une erreur survient pendant l’interrogation.
     *
     * Reçoit un objet {@link ULError} contenant :
     * - `status` — le code d’état HTTP de la requête ayant échoué
     * - `responseText` — le corps brut de la réponse, s’il est disponible
     *
     * - Appelée une fois par événement d’erreur; l’interrogation peut être relancée selon le type d’erreur.
     * - À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
     */
    onError?: (error: ULError) => void;
  };
  ```

  <Expandable title="Propriétés">
    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en **millisecondes**, entre deux requêtes d’interrogation consécutives.

      * S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5000 ms).
      * Choisissez une valeur qui offre un bon équilibre entre la réactivité et les limites de débit.

      #### Valeur par défaut

      ```ts
      Dépend de l’implémentation (généralement 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Fonction de rappel facultative exécutée une fois que le MFA push challenge est
      approuvé avec succès et que l’interrogation est terminée.

      * Appelée exactement **une** fois, après quoi l’interrogation s’arrête automatiquement.
      * Utile pour faire progresser le flux de connexion (par ex., appeler `continue()`).
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Fonction de rappel facultative invoquée si une erreur survient pendant l’interrogation.

      Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error) contenant :

      * `status` — le code d’état HTTP de la requête ayant échoué

      * `responseText` — le corps brut de la réponse, s’il est disponible

      * Appelée une fois par événement d’erreur; l’interrogation peut être relancée selon le type d’erreur.

      * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
