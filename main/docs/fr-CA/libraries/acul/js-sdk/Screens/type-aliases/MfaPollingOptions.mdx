<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer un processus de scrutation par notification push d’AMF.

  Spécifiez ces options au démarrage d’une session de scrutation pour contrôler la fréquence
  à laquelle le point de terminaison du challenge est interrogé et la façon dont le succès ou l’échec sont gérés.

  ```ts Example
  export type MfaPollingOptions = {
    /**
     * Intervalle facultatif, en **millisecondes**, entre deux requêtes
     * de scrutation consécutives.
     *
     * - S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5000 ms).
     * - Choisissez une valeur qui équilibre la réactivité et les contraintes de limite de débit.
     *
     * @defaultValue Spécifique à l’implémentation (généralement 5000 ms)
     */
    intervalMs?: number;

    /**
     * Fonction de rappel facultative exécutée une fois que le challenge push d’AMF
     * a été approuvé avec succès et que la scrutation est terminée.
     *
     * - Appelée exactement **une** fois, après quoi la scrutation s’arrête automatiquement.
     * - Utile pour faire avancer le flux de connexion (par ex., en appelant `continue()`).
     */
    onCompleted?: () => void;

    /**
     * Fonction de rappel facultative invoquée si une erreur survient pendant la scrutation.
     *
     * Reçoit un objet {@link ULError} contenant :
     * - `status` — le code d’état HTTP de la requête ayant échoué
     * - `responseText` — le corps de la réponse brut, s’il est disponible
     *
     * - Appelée une fois par événement d’erreur; la scrutation peut réessayer selon le type d’erreur.
     * - À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
     */
    onError?: (error: ULError) => void;
  };
  ```

  <Expandable title="Propriétés">

      Intervalle facultatif, en **millisecondes**, entre deux requêtes de scrutation consécutives.

      * Choisissez une valeur qui équilibre la réactivité et les contraintes de limite de débit.

      #### Valeur par défaut

      ```ts
      Spécifique à l’implémentation (généralement 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">

    <ParamField body="onError()?" type="(error) => void">
      Fonction de rappel facultative invoquée si une erreur survient pendant la scrutation.

      Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error) contenant :

      * `status` — le code d’état HTTP de la requête ayant échoué

      * `responseText` — le corps de la réponse brut, s’il est disponible

      * Appelée une fois par événement d’erreur; la scrutation peut réessayer selon le type d’erreur.

      * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
