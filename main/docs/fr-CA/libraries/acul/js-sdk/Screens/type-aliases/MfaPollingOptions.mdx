<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer un processus de polling des notifications push MFA.

  Fournissez ces options au démarrage d’une session de polling pour contrôler la fréquence
  à laquelle l’endpoint du défi est interrogé et la manière dont le succès ou l’échec est géré.

  ```ts Example
  export type MfaPollingOptions = {
    /**
     * Intervalle facultatif, en **millisecondes**, entre des requêtes de polling consécutives.
     *
     * - S’il est omis, l’intervalle par défaut interne du SDK est utilisé (généralement 5000 ms).
     * - Choisissez une valeur qui offre un bon équilibre entre réactivité et limites de débit.
     *
     * @defaultValue Dépend de l’implémentation (généralement 5000 ms)
     */
    intervalMs?: number;

    /**
     * Callback facultatif exécuté une fois que le défi MFA push est
     * approuvé avec succès et que le polling est terminé.
     *
     * - Appelé exactement **une** fois, après quoi le polling s’arrête automatiquement.
     * - Utile pour faire progresser le flux de connexion (p. ex., appeler `continue()`).
     */
    onCompleted?: () => void;

    /**
     * Callback facultatif invoqué si une erreur survient pendant le polling.
     *
     * Reçoit un objet {@link ULError} contenant :
     * - `status` — le code d’état HTTP de la requête ayant échoué
     * - `responseText` — le corps brut de la réponse, s’il est disponible
     *
     * - Appelé une fois par événement d’erreur; le polling peut réessayer selon le type d’erreur.
     * - À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
     */
    onError?: (error: ULError) => void;
  };
  ```

  <Expandable title="Propriétés">
    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en **millisecondes**, entre des requêtes de polling consécutives.

      * S’il est omis, l’intervalle par défaut interne du SDK est utilisé (généralement 5000 ms).
      * Choisissez une valeur qui offre un bon équilibre entre réactivité et limites de débit.

      #### Valeur par défaut

      ```ts
      Dépend de l’implémentation (généralement 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Callback facultatif exécuté une fois que le défi MFA push est approuvé avec succès
      et que le polling est terminé.

      * Appelé exactement **une** fois, après quoi le polling s’arrête automatiquement.
      * Utile pour faire progresser le flux de connexion (p. ex., appeler `continue()`).
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Callback facultatif invoqué si une erreur survient pendant le polling.

      Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error) contenant :

      * `status` — le code d’état HTTP de la requête ayant échoué

      * `responseText` — le corps brut de la réponse, s’il est disponible

      * Appelé une fois par événement d’erreur; le polling peut réessayer selon le type d’erreur.

      * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
