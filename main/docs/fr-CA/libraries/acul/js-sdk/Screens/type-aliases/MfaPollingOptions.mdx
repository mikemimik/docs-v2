<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer le processus d’interrogation d’une notification push d’AMF.

  Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
  à laquelle l’endpoint du défi est interrogé et la manière dont les réussites ou les échecs sont gérés.

  ```ts Example
  export type MfaPollingOptions = {
    /**
     * Intervalle facultatif, en **millisecondes**, entre des requêtes d’interrogation consécutives.
     *
     * - S’il est omis, l’intervalle `default` interne du SDK est utilisé (généralement 5000 ms).
     * - Choisissez une valeur qui offre un bon équilibre entre réactivité et limites de débit.
     *
     * @defaultValue Spécifique à la mise en œuvre (couramment 5000 ms)
     */
    intervalMs?: number;

    /**
     * Fonction de rappel facultative exécutée une fois que le défi push d’AMF est
     * approuvé avec succès et que l’interrogation se termine.
     *
     * - Appelée exactement **une** fois, après quoi l’interrogation s’arrête automatiquement.
     * - Utile pour faire avancer le flux de connexion (par ex., en appelant `continue()`).
     */
    onCompleted?: () => void;

    /**
     * Fonction de rappel facultative invoquée si une erreur survient pendant l’interrogation.
     *
     * Reçoit un objet {@link ULError} contenant :
     * - `status` — le code d’état HTTP de la requête échouée
     * - `responseText` — le corps de la réponse brut, s’il est disponible
     *
     * - Appelée une fois par événement d’erreur; l’interrogation peut être retentée selon le type d’erreur.
     * - À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
     */
    onError?: (error: ULError) => void;
  };
  ```

  <Expandable title="Propriétés">
    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en **millisecondes**, entre des requêtes d’interrogation consécutives.

      * S’il est omis, l’intervalle `default` interne du SDK est utilisé (généralement 5000 ms).
      * Choisissez une valeur qui offre un bon équilibre entre réactivité et limites de débit.

      #### Valeur par défaut

      ```ts
      Spécifique à la mise en œuvre (couramment 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Fonction de rappel facultative exécutée une fois que le défi push d’AMF est approuvé avec succès
      et que l’interrogation se termine.

      * Appelée exactement **une** fois, après quoi l’interrogation s’arrête automatiquement.
      * Utile pour faire avancer le flux de connexion (par ex., en appelant `continue()`).
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Fonction de rappel facultative invoquée si une erreur survient pendant l’interrogation.

      Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error) contenant :

      * `status` — le code d’état HTTP de la requête échouée

      * `responseText` — le corps de la réponse brut, s’il est disponible

      * Appelée une fois par événement d’erreur; l’interrogation peut être retentée selon le type d’erreur.

      * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
