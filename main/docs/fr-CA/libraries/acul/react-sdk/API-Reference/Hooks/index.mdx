<div id="methods">
  ## Méthodes
</div>

* [useAuth0Themes](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes) : Hook React pour obtenir les options du thème actuel avec une configuration aplatie à partir du contexte de l’image de marque.
* [useChangeLanguage](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage) : Hook React qui retourne une fonction pour changer la langue d’affichage sur l’écran ACUL actuel.
* [useCurrentScreen](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen) : Hook React qui récupère le contexte et l’état de l’écran actuel.
* [useErrors](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useErrors) : Hook React pour lire et gérer les erreurs dans ACUL (Advanced Customization of Universal Login), y compris toutes les erreurs de validation et les erreurs côté serveur. Il regroupe les erreurs en trois types : - `server` — erreurs renvoyées par Auth0 ou votre propre backend. - `client` — erreurs provenant de la validation côté client (p. ex., saisie de formulaire invalide). - `developer` — erreurs causées par une intégration incorrecte ou une mauvaise utilisation du SDK.
* [useLoginIdentifiers](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useLoginIdentifiers) : Retourne une liste des types d’identifiant actifs (comme `'email'`, `'phone'` ou `'username'`) actuellement utilisés dans le flux d’authentification ou le processus de connexion.
* [useMfaPolling](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useMfaPolling) : Hook React pour gérer le polling de MFA push (p. ex., en attendant l’approbation d’une notification push) sur un écran Auth0 Advanced Customization of Universal Login (ACUL).
* [useResend](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useResend) : Ce hook React gère les actions de « renvoi » (p. ex., renvoyer un code de vérification) sur les écrans ACUL.
* [useSignupIdentifiers](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useSignupIdentifiers) : Retourne une liste des identifiants activés (email, phone ou username), chacun avec son statut `required`, selon les identifiants d’inscription de l’écran actuel.
* [usePasswordValidation](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/usePasswordValidation) : Ce hook React valide un mot de passe selon la politique de mot de passe Auth0 actuelle et retourne un résultat structuré indiquant si le mot de passe respecte chaque règle.
* [useUsernameValidation](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useUsernameValidation) : Hook React pour valider un username selon la politique de username Auth0 actuelle.
