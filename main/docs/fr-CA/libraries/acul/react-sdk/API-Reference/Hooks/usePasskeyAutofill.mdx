<ParamField body="usePasskeyAutofill()" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsePasskeyAutofillResult">UsePasskeyAutofillResult</a></span>}>
  Hook React qui active la fonctionnalité d’interface conditionnelle du navigateur (remplissage automatique par passkey) pour les
  champs d’identifiant de connexion sur les écrans de la personnalisation avancée de Universal Login (ACUL) d’Auth0.

  Ce hook initialise l’API Conditional Mediation du navigateur, qui affiche les
  passkeys d’appareil enregistrées dans la liste déroulante d’autocomplétion du champ de nom d’utilisateur sans nécessiter
  d’interaction supplémentaire de la part de l’utilisateur.

  * Le hook s’enregistre **une seule fois par cycle de vie de la page** — l’appeler plusieurs fois ne pose aucun problème.
  * **Fonctionne en mode dégradé** sur les navigateurs non pris en charge sans bloquer l’interaction de l’utilisateur.
  * L’association du `inputRef` retourné à un élément `input` garantit que le bon
    attribut `autocomplete="username webauthn"` est appliqué automatiquement.
  * Si votre `input` déclare déjà `autocomplete="username webauthn"` dans le balisage,
    **l’association de la ref est facultative** — le hook s’enregistre quand même correctement.

  ### Fonctionnalités clés

  * **Conditional Mediation automatique** — initialise le remplissage automatique par passkey sans
    aucun appel manuel à l’API du navigateur.
  * **Conception avec ref facultative** — fonctionne avec ou sans association de `inputRef` à l’élément `input`.
  * **Solution de repli sans friction** — les environnements non pris en charge sont gérés silencieusement,
    ce qui préserve le flux de connexion normal.

  ## Paramètres

  Aucun.

  ## Retourne

  [`UsePasskeyAutofillResult`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsePasskeyAutofillResult)

  Objet contenant :

  * `inputRef` *(facultatif)* — une ref React à associer à l’élément `input` du nom d’utilisateur. Garantit
    que le bon attribut `autocomplete` est défini lorsqu’il est associé.

  ## Écrans pris en charge

  * `login-id`

  ```tsx Example
  import { usePasskeyAutofill } from '@auth0/auth0-acul-react/login-id';

  export function LoginIdentifierInput() {
    const { inputRef } = usePasskeyAutofill();

    return (
      <input
        ref={inputRef}
        type="text"
        name="username"
        placeholder="Email or username"
      />
    );
  }
  ```

  ## Remarques

  * Si `inputRef` n’est pas associé à un `input`, assurez-vous que l’élément `input` déclare explicitement
    `autocomplete="username webauthn"` afin que le navigateur puisse l’associer au remplissage automatique par passkey.
  * Le hook n’a aucun effet dans les environnements où l’API Conditional Mediation n’est pas disponible
    (p. ex. les anciens navigateurs ou les contextes non HTTPS) — aucune erreur n’est générée.
  * Appelez `usePasskeyAutofill` une fois au niveau supérieur de votre composant; ne l’appelez pas
    de façon conditionnelle ni à l’intérieur de gestionnaires d’événements.
