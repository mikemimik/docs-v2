<ParamField body="useUsernameValidation" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  Hook React permettant de valider un nom d&#39;utilisateur en fonction de la politique de nom d&#39;utilisateur Auth0 actuellement en vigueur.

  Ce hook vérifie le nom d&#39;utilisateur fourni par rapport à toutes les règles de validation configurées
  et renvoie un résultat structuré indiquant s&#39;il est valide.
  Au besoin, il peut aussi envoyer les erreurs de validation au gestionnaire global d&#39;erreurs pour que les
  composants d&#39;UI qui surveillent le champ `username` puissent automatiquement les afficher ou y réagir.

  ### Fonctionnalités clés

  * **Validation adaptée à la politique du tenant** — vérifie le nom d&#39;utilisateur en fonction des règles de la politique de nom d&#39;utilisateur Auth0 configurée pour le tenant.
  * **Intégration au gestionnaire d&#39;erreurs** — transmet au besoin automatiquement les échecs de validation aux composants d&#39;erreur du formulaire.
  * **Recalcul optimisé** — effectue un nouveau calcul uniquement lorsque `username` ou `options.includeInErrors` changent.

  ## Paramètres

  <ParamField body="username" type="string">
    La chaîne de caractères du nom d&#39;utilisateur à valider.
  </ParamField>

  <ParamField body="options" type="{ includeInErrors?: boolean }">
    Configuration facultative du hook.

    <Expandable title="propriétés">
      <ParamField body="includeInErrors" type="boolean">
        Lorsque `true`, les erreurs de validation sont stockées dans le gestionnaire global d&#39;erreurs sous le champ `username`. La valeur par défaut est `false`.
      </ParamField>
    </Expandable>
  </ParamField>

  ## Valeur de retour

  [`UsernameValidationResult`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult)

  Un objet [UsernameValidationResult](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult) contenant :

  * `isValid` — `true` si le nom d&#39;utilisateur respecte toutes les règles configurées.
  * `errors` — un tableau d&#39;erreurs de validation par règle contenant `code`, `message` et `isValid`.

  ## Écrans pris en charge

  * `signup`
  * `signup-id`

  ```tsx Example
  import { useUsernameValidation } from "@auth0/auth0-acul-react/signup";

  export function UsernameField() {
    const { isValid, errors } = useUsernameValidation(username, { includeInErrors: true });

    return (
      <div>
        <input
          value={username}
          onChange={e => setUsername(e.target.value)}
          aria-invalid={!isValid}
        />

        {!isValid && (
          <ul>
            {errors.map(err => (
              <li key={err.code}>{err.message}</li>
            ))}
          </ul>
        )}
      </div>
    );
  }
  ```

  ## Remarques

  * Lorsque `includeInErrors` est activé, le hook met automatiquement à jour les erreurs dans le magasin d&#39;erreurs, qui peut être utilisé par le hook `useErrors`.
  * Le hook effectue un nouveau calcul uniquement lorsque `username` ou `options.includeInErrors` changent.
  * Appelez `useUsernameValidation` au niveau supérieur de votre composant; ne l&#39;appelez pas de façon conditionnelle ni dans des gestionnaires d&#39;événements.
  * Le chemin d&#39;importation doit correspondre à l&#39;écran — utilisez `signup` ou `signup-id` selon le cas.
</ParamField>
