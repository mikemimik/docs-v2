```tsx
useUsernameValidation(
  username: string,
  options?: { includeInErrors?: boolean },
): UsernameValidationResult
```

<ParamField body="useUsernameValidation" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  Hook React permettant de valider un `username` selon la politique de nom d’utilisateur Auth0 actuelle.

  Ce hook vérifie le `username` fourni par rapport à toutes les règles de validation configurées
  et renvoie un résultat structuré indiquant s’il est valide.
  Au besoin, il peut aussi envoyer les erreurs de validation au gestionnaire d’erreurs global afin que les
  composants d’UI qui surveillent le champ `username` puissent automatiquement les afficher ou y réagir.

  ### Fonctionnalités clés

  * **Validation selon la politique du tenant** — vérifie le `username` selon les règles de la politique de nom d’utilisateur Auth0 configurées pour le tenant.
  * **Intégration au gestionnaire d’erreurs** — transmet facultativement les échecs de validation aux composants d’erreur du formulaire.
  * **Recalcul optimisé** — recalcule uniquement lorsque `username` ou `options.includeInErrors` changent.

  ## Paramètres

  <ParamField body="username" type="string">
    La chaîne `username` à valider.
  </ParamField>

  <ParamField body="options" type="{ includeInErrors?: boolean }">
    Configuration facultative du hook.

    <Expandable title="propriétés">
      <ParamField body="includeInErrors" type="boolean">
        Lorsque `true`, les erreurs de validation sont stockées dans le gestionnaire d’erreurs global sous le champ `username`. La valeur par défaut est `false`.
      </ParamField>
    </Expandable>
  </ParamField>

  ## Valeur renvoyée

  [`UsernameValidationResult`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult)

  Un objet [UsernameValidationResult](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UsernameValidationResult) contenant :

  * `isValid` — `true` si le `username` respecte toutes les règles configurées.
  * `errors` — un tableau d’erreurs de validation par règle avec `code`, `message` et `isValid`.

  ## Écrans pris en charge

  * `signup`
  * `signup-id`

  ```tsx Exemple
  import { useUsernameValidation } from "@auth0/auth0-acul-react/signup";

  export function UsernameField() {
    const { isValid, errors } = useUsernameValidation(username, { includeInErrors: true });

    return (
      <div>
        <input
          value={username}
          onChange={e => setUsername(e.target.value)}
          aria-invalid={!isValid}
        />

        {!isValid && (
          <ul>
            {errors.map(err => (
              <li key={err.code}>{err.message}</li>
            ))}
          </ul>
        )}
      </div>
    );
  }
  ```

  ## Remarques

  * Lorsque `includeInErrors` est activé, le hook met automatiquement à jour les erreurs dans le stockage d’erreurs, qui peut ensuite être utilisé par le hook `useErrors`.
  * Le hook ne recalcule que lorsque `username` ou `options.includeInErrors` changent.
  * Appelez `useUsernameValidation` au niveau supérieur de votre composant; ne l’appelez pas de façon conditionnelle ni dans des gestionnaires d’événements.
  * Le chemin d’importation doit correspondre à l’écran — utilisez `signup` ou `signup-id` selon le cas.
</ParamField>
