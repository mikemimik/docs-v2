<ParamField body="useMfaPolling(options?: MfaPollingOptions)" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPollingResult">MfaPollingResult</a></span>}>
  Hook React permettant de gérer le polling d’un MFA push (par ex., en attendant l’approbation d’une push notification)
  sur un écran Auth0 Advanced Customization of Universal Login (ACUL).

  Ce hook configure et contrôle une boucle de polling de longue durée qui vérifie de façon répétée
  l’endpoint du MFA push challenge jusqu’à ce que l’une des situations suivantes se produise :

  * Le challenge est **approuvé ou refusé** par l’utilisateur, ce qui déclenche `options.onCompleted`.
  * Une **erreur** survient (erreur réseau, réponse autre que 200/429), ce qui déclenche `options.onError`.
  * Le **composant est démonté** ou `stopPolling()` est appelé, ce qui annule le polling.

  ### Principales fonctionnalités

  * `isRunning` est **réactif** — il se met automatiquement à jour si la boucle de polling
    s’arrête d’elle-même ou est annulée.
  * Utilise une **instance de polling unique et stable** (`useRef`) pour éviter
    les requêtes réseau en double et les redémarrages involontaires lors des nouveaux rendus de React.
  * **Nettoyage automatique** au démontage : aucun minuteur orphelin ni aucune requête XHR en fuite.

  ## Paramètres

  <ParamField body="options" type={<span><a href='/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/type-aliases/MfaPollingOptions'>MfaPollingOptions</a></span>}>
    Spécifie l’intervalle de polling, le callback de réussite (`onCompleted`) et le gestionnaire d’erreur facultatif (`onError`).
  </ParamField>

  ## Renvoie

  [`MfaPollingResult`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPollingResult) contenant :

  * `isRunning` — `true` tant que le polling est actif.
  * `startPolling()` — démarre ou reprend le polling.
  * `stopPolling()` — arrête immédiatement le polling.

  ## Écrans pris en charge

  * `mfa-push-challenge-push`
  * `reset-password-mfa-push-challenge-push`
  * `mfa-push-enrollment-qr`

  ```tsx Example
  import { useMfaPolling } from '@auth0/auth0-acul-react/mfa-push-challenge-push';

  export function MfaPushStatus() {
    const { isRunning, startPolling, stopPolling } = useMfaPolling({
      intervalMs: 5000,
      onCompleted: () => console.log('Push approved!/denied'),
      onError: (error) => console.error('Polling error:', error)
    });

    return (
      <div>
        <button onClick={startPolling} disabled={isRunning}>
          {isRunning ? 'Waiting for approval…' : 'Start MFA Polling'}
        </button>
        {isRunning && <button onClick={stopPolling}>Cancel</button>}
      </div>
    );
  }
  ```

  ## Remarques

  * Le callback `onError` reçoit un objet [ULError](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error)
    avec `status` et `responseText`, qui décrivent la réponse du serveur.
  * Les réponses internes de limitation du débit (`429`) sont gérées automatiquement :
    le polling attend la fenêtre de réinitialisation avant de réessayer.
  * Appeler `startPolling()` à répétition pendant l’exécution est sûr et idempotent.
</ParamField>
