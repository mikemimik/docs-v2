MembresDuConsentementPersonnalisé

````ts Example
export interface CustomizedConsentMembers extends BaseMembers {
  /**
   * Fournit l'accès aux propriétés et données propres à l'écran Customized Consent,
   * y compris la liste des `scopes` et des `authorizationDetails` demandés.
   * @type {ScreenMembersOnCustomizedConsent}
   */
  screen: ScreenMembersOnCustomizedConsent;

  /**
   * Soumet la décision de l'utilisateur d'accepter (accorder) les permissions demandées et les détails d'autorisation.
   * Cette action envoie une requête au point de terminaison `/u/customized-consent` avec `action: "accept"`.
   * L'état de la transaction est automatiquement inclus.
   *
   * @param {CustomOptions} [payload] - Facultatif. Un objet pour toute paire clé-valeur personnalisée
   *                                    à envoyer avec la requête. Ces paramètres seront
   *                                    inclus dans les données de formulaire soumises au serveur.
   * @returns {Promise<void>} Une promesse qui se résout lorsque l'action d'acceptation est soumise avec succès.
   *                          En cas de succès, Auth0 redirige généralement l'utilisateur vers l'application
   *                          ou vers l'étape suivante du flux d'authentification.
   * @throws {Error} Génère une erreur si la soumission du formulaire échoue (p. ex., problème réseau, état invalide).
   *                 Les erreurs côté serveur (comme « invalid_request ») seront reflétées dans `this.transaction.errors`
   *                 après l'opération, plutôt que d'être levées comme des erreurs JavaScript.
   *
   * @example
   * ```typescript
   * // En supposant que 'customizedConsentManager' est une instance de la classe SDK CustomizedConsent
   * try {
   *   await customizedConsentManager.accept();
   *   // En cas de succès, la page sera généralement redirigée.
   * } catch (error) {
   *   // Gérer les erreurs inattendues lors de la soumission elle-même.
   *   console.error("Failed to submit consent acceptance:", error);
   * }
   * // Après l'attente, vérifier customizedConsentManager.transaction.errors pour les problèmes de validation côté serveur.
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * Soumet la décision de l'utilisateur de rejeter les permissions demandées et les détails d'autorisation.
   * Cette action envoie une requête au point de terminaison `/u/customized-consent` avec `action: "deny"`.
   * L'état de la transaction est automatiquement inclus.
   *
   * @param {CustomOptions} [payload] - Facultatif. Un objet pour toute paire clé-valeur personnalisée
   *                                    à envoyer avec la requête. Ces paramètres seront
   *                                    inclus dans les données de formulaire soumises au serveur.
   * @returns {Promise<void>} Une promesse qui se résout lorsque l'action de rejet est soumise avec succès.
   *                          En cas de succès, Auth0 redirige généralement l'utilisateur, en affichant
   *                          potentiellement un message d'accès refusé ou en retournant une erreur à l'application.
   * @throws {Error} Génère une erreur si la soumission du formulaire échoue (p. ex., problème réseau, état invalide).
   *                 Les erreurs côté serveur sont reflétées dans `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // En supposant que 'customizedConsentManager' est une instance de la classe SDK CustomizedConsent
   * try {
   *   await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // Exemple d'option personnalisée
   *   // En cas de succès, la page sera généralement redirigée.
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // Après l'attente, vérifier customizedConsentManager.transaction.errors pour les problèmes de validation côté serveur.
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}
````

<ParamField body="accept" type="Promise<void>">
  Soumet la décision de l’utilisateur d’accepter (d’accorder) les permissions demandées et les détails d’autorisation.
  Cette action envoie une requête au point de terminaison `/u/customized-consent` avec `action: "accept"`.
  L’état de la transaction est inclus automatiquement.

  Une promesse qui se résout lorsque l’action d’acceptation est soumise avec succès.
  En cas de succès, Auth0 redirige généralement l’utilisateur vers l’application
  ou vers l’étape suivante du flux d’authentification.

  #### Lève

  Lève une erreur si la soumission du formulaire échoue (p. ex., problème réseau, état non valide).
  Les erreurs côté serveur (comme &quot;invalid&#95;request&quot;) sont reflétées dans `this.transaction.errors`
  après l’opération, plutôt que d’être levées comme erreurs JavaScript.

  ```typescript Example
  // En supposant que 'customizedConsentManager' est une instance de la classe SDK CustomizedConsent
  try {
    await customizedConsentManager.accept();
    // En cas de succès, l’utilisateur sera généralement redirigé.
  } catch (error) {
    // Gérer les erreurs inattendues pendant la soumission elle-même.
    console.error("Failed to submit consent acceptance:", error);
  }
  // Après le await, vérifiez customizedConsentManager.transaction.errors pour les problèmes de validation côté serveur.
  ```
