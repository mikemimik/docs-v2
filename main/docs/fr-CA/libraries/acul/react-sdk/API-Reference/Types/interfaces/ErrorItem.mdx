---
title: "ErrorItem"
---

Représente une entrée d’erreur unique dans le stockage des erreurs d’ACUL. Est renvoyé par [`useErrors`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useErrors) au sein de l’objet [`UseErrorsResult`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult). Étend l’interface `Error` de base avec un `id`, un `type` et un `label` facultatif.

```typescript
interface ErrorItem {
  id: string;
  code: string;
  message: string;
  type?: ErrorType;
  field?: string;
  label?: string;
  rules?: PasswordComplexityRule[];
}
```

<div id="properties">
  ## Propriétés
</div>

<ParamField body="id" type="string" required>
  Un identifiant unique pour cette instance d’erreur. Sert à cibler une erreur précise avec `dismiss(id)`.
</ParamField>

<ParamField body="code" type="string" required>
  Un code d’erreur lisible par machine qui permet d’identifier le type d’erreur. Exemple : `"invalid_password"`, `"username_exists"`.
</ParamField>

<ParamField body="message" type="string" required>
  Une description de l’erreur compréhensible par un humain, pouvant être affichée dans l’UI.
</ParamField>

<ParamField body="type" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/type-aliases/ErrorType">ErrorType</a>}>
  Classe l’erreur comme `"auth0"`, `"validation"` ou `"configuration"`. Utilisé pour le filtrage avec `errors.byType()`.
</ParamField>

<ParamField body="field" type="string">
  Le champ de formulaire associé à l’erreur, le cas échéant. Exemple : `"username"`, `"email"`. Utilisé pour le filtrage avec `errors.byField()`.
</ParamField>

<ParamField body="label" type="string">
  Un libellé d’affichage facultatif pour l’erreur, utilisé lorsqu’un nom de champ plus convivial est nécessaire en plus du message d’erreur.
</ParamField>

<ParamField body="rules" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PasswordComplexityRule">PasswordComplexityRule</a>[]</span>}>
  Règles de complexité du mot de passe associées à cette erreur, lorsque l’erreur est liée à la validation du mot de passe.
</ParamField>
