---
title: "ErrorItem"
---

Représente une entrée d’erreur du magasin d’erreurs d’ACUL. Renvoyé par [`useErrors`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useErrors) comme partie de l’objet [`UseErrorsResult`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult). Étend l’interface `Error` de base avec un `id`, un `type` et un `label` facultatif.

```typescript
interface ErrorItem {
  id: string;
  code: string;
  message: string;
  type?: ErrorType;
  field?: string;
  label?: string;
  rules?: PasswordComplexityRule[];
}
```

<div id="properties">
  ## Propriétés
</div>

<ParamField body="id" type="string" required>
  Un identifiant unique pour cette occurrence de l’erreur. Sert à cibler une erreur précise avec `dismiss(id)`.
</ParamField>

<ParamField body="code" type="string" required>
  Un code d’erreur lisible par machine qui identifie le type d’erreur. Exemple : `"invalid_password"`, `"username_exists"`.
</ParamField>

<ParamField body="message" type="string" required>
  Une description de l’erreur lisible par un humain, pouvant être affichée dans l’interface utilisateur.
</ParamField>

<ParamField body="type" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/type-aliases/ErrorType">ErrorType</a>}>
  Classe l’erreur comme `"auth0"`, `"validation"` ou `"configuration"`. Sert au filtrage avec `errors.byType()`.
</ParamField>

<ParamField body="field" type="string">
  Le champ de formulaire associé à l’erreur, s’il y a lieu. Exemple : `"username"`, `"email"`. Sert au filtrage avec `errors.byField()`.
</ParamField>

<ParamField body="label" type="string">
  Un label d’affichage facultatif pour l’erreur, utilisé lorsqu’un nom de champ plus convivial est nécessaire à côté du message d’erreur.
</ParamField>

<ParamField body="rules" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PasswordComplexityRule">PasswordComplexityRule</a>[]</span>}>
  Règles de complexité du mot de passe associées à cette erreur, lorsque l’erreur concerne la validation du mot de passe.
</ParamField>
