    <ParamField body="options" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/type-aliases/MfaPollingOptions">MfaPollingOptions</a></span>}>
      <ParamField body="MfaPollingOptions" type="object">
        Options de configuration pour gérer un processus d’interrogation pour une notification d’AMF par push.

        Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
        à laquelle le endpoint du challenge est interrogé et la façon dont le succès
        ou l’échec est géré.

        <Expandable title="propriétés">
          <ParamField body="intervalMs?" type="number">
            Intervalle facultatif, en **millisecondes**, entre des requêtes d’interrogation consécutives.

            * S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5 000 ms).
            * Choisissez une valeur qui établit un bon équilibre entre la réactivité et les limites de débit.
          </ParamField>

          <ParamField body="onCompleted()?" type="void">
            Callback facultatif exécuté une fois que le challenge d’AMF par push a été
            approuvé avec succès et que l’interrogation est terminée.

            * Appelé exactement **une** fois, après quoi l’interrogation s’arrête automatiquement.
            * Utile pour faire progresser le flux de connexion (par ex., en appelant `continue()`).
          </ParamField>

          <ParamField body="onError()?" type="void">
            Callback facultatif invoqué si une erreur se produit pendant l’interrogation.

            Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) contenant :

            * `status` — le code d’état HTTP de la requête ayant échoué

            * `responseText` — le corps de la réponse brut, s’il est disponible

            * Appelé une fois par événement d’erreur; l’interrogation peut réessayer selon le type d’erreur.

            * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
