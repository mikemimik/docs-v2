MfaWebAuthnPlatformChallengeMembers

````ts Example
export interface MfaWebAuthnPlatformChallengeMembers extends BaseMembers {
  /**
   * Propriétés et données propres à l'écran, incluant les options de défi WebAuthn et la préférence de mémorisation de l'appareil.
   * @type {ScreenMembersOnMfaWebAuthnPlatformChallenge}
   */
  screen: ScreenMembersOnMfaWebAuthnPlatformChallenge;

  /**
   * Lance le défi de plateforme WebAuthn de l'authentificateur.
   * Cette méthode appelle en interne `navigator.credentials.get()` en utilisant les options de défi
   * fournies dans `screen.publicKey`.
   * En cas de réussite, elle soumet les informations d'identification résultantes à Auth0 avec `action: "default"`.
   *
   * Si `navigator.credentials.get()` échoue (p. ex., annulation par l'utilisateur, délai d'attente ou autres erreurs de l'API WebAuthn),
   * il est recommandé d'appeler `reportBrowserError` avec les détails de l'erreur.
   *
   * @param {VerifyPlatformAuthenticatorOptions} [options] - Paramètres facultatifs pour le processus de vérification,
   * tels que `rememberDevice` (si `screen.showRememberDevice` est vrai) et d'autres options personnalisées.
   * @returns {Promise<void>} Une promesse qui se résout lorsque la tentative de vérification est soumise.
   *                          Une opération réussie entraîne généralement une redirection.
   * @throws {Error} Génère une erreur si `screen.publicKey` est manquant,
   *                 si `navigator.credentials.get()` échoue avec une erreur inattendue non gérée par `getPasskeyCredentials`,
   *                 ou si la soumission du formulaire à Auth0 échoue.
   *
   * @example
   * ```typescript
   * // Supposons que 'sdk' est une instance de MfaWebAuthnPlatformChallenge
   * try {
   *   const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
   *   await sdk.verify({ rememberDevice: remember });
   *   // En cas de réussite, Auth0 gère la redirection.
   * } catch (error) {
   *   console.error("Platform authenticator verification failed:", error);
   *   // S'il s'agit d'une erreur de l'API WebAuthn, la signaler
   *   if (error.name && error.message) { // Vérification de base pour une erreur de type DOMException
   *     await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
   *   }
   *   // Vérifier sdk.transaction.errors pour les messages de validation côté serveur si la page se recharge.
   * }
   * ```
   */
  verify(options?: VerifyPlatformAuthenticatorOptions): Promise<void>;

  /**
   * Signale une erreur spécifique de l'API WebAuthn (provenant de `navigator.credentials.get()`) à Auth0.
   * Cette méthode doit être utilisée lorsque `verify()` (ou un appel manuel à `navigator.credentials.get()`) échoue en raison
   * d'un problème WebAuthn côté navigateur (p. ex., annulation par l'utilisateur `NotAllowedError`, délai d'attente).
   * Elle soumet les détails de l'erreur avec `action: "showError::{errorDetailsJsonString}"` et une `response` vide.
   *
   * @param {ReportBrowserErrorOptions} options - Contient l'objet `error` (avec `name` et `message`
   * provenant de la DOMException de l'API WebAuthn) et toute autre option personnalisée.
   * @returns {Promise<void>} Une promesse qui se résout lorsque le rapport d'erreur est soumis.
   * @throws {Error} Génère une erreur si la soumission du formulaire échoue (p. ex., erreur réseau, état invalide).
   *
   * @example
   * ```typescript
   * // Supposons que 'sdk' est une instance de MfaWebAuthnPlatformChallenge
   * // Et que webAuthnDomException est un objet d'erreur provenant d'un appel échoué à navigator.credentials.get().
   * try {
   *   await sdk.reportBrowserError({
   *     error: { name: webAuthnDomException.name, message: webAuthnDomException.message }
   *   });
   *   // Auth0 traitera cette erreur et pourra réafficher la page ou effectuer une redirection.
   * } catch (submitError) {
   *   console.error("Failed to report WebAuthn browser error:", submitError);
   * }
   * ```
   */
  reportBrowserError(options: ReportBrowserErrorOptions): Promise<void>;

  /**
   * Permet à l'utilisateur de se désinscrire du défi de plateforme WebAuthn et de sélectionner une autre méthode AMF.
   * Cette action soumet `action: "pick-authenticator"` à Auth0, ce qui devrait diriger
   * l'utilisateur vers un écran de sélection de facteur AMF.
   *
   * @param {TryAnotherMethodOptions} [options] - Paramètres personnalisés facultatifs à envoyer avec la requête.
   * @returns {Promise<void>} Une promesse qui se résout lorsque l'action 'pick-authenticator' est soumise.
   * @throws {Error} Génère une erreur si la soumission du formulaire échoue.
   *
   * @example
   * ```typescript
   * // Supposons que 'sdk' est une instance de MfaWebAuthnPlatformChallenge
   * try {
   *   await sdk.tryAnotherMethod();
   *   // En cas de réussite, Auth0 gère la redirection vers la sélection AMF.
   * } catch (error) {
   *   console.error("Failed to switch MFA method:", error);
   * }
   * ```
   */
  tryAnotherMethod(options?: TryAnotherMethodOptions): Promise<void>;
}

<ParamField body="reportBrowserError" type="Promise<void>">
  Signale à Auth0 une erreur précise de la WebAuthn API (provenant de `navigator.credentials.get()`).
  Cette méthode doit être utilisée lorsque `verify()` (ou un appel manuel à `navigator.credentials.get()`) échoue en raison
  d’un problème WebAuthn du côté du navigateur (p. ex., annulation par l’utilisateur `NotAllowedError`, délai d’expiration).
  Elle soumet les détails de l’erreur avec `action: "showError::{errorDetailsJsonString}"` et une `response` vide.

  Une promesse qui se résout lorsque le signalement d’erreur est soumis.

  #### Throws

  Lance une erreur si la soumission du formulaire échoue (p. ex., erreur réseau, `state` invalide).

  ```typescript Example
  // En supposant que 'sdk' est une instance de MfaWebAuthnPlatformChallenge
  // Et que webAuthnDomException est un objet d’erreur provenant d’un appel échoué à navigator.credentials.get().
  try {
    await sdk.reportBrowserError({
      error: { name: webAuthnDomException.name, message: webAuthnDomException.message }
    });
    // Auth0 traitera cette erreur et pourra réafficher la page ou rediriger l’utilisateur.
  } catch (submitError) {
    console.error("Failed to report WebAuthn browser error:", submitError);
  }
  ```

<ParamField body="verify" type="Promise<void>">
  Lance le défi de l’authenticator de plateforme WebAuthn.
  Cette méthode appelle en interne `navigator.credentials.get()` à l’aide des options de défi
  fournies dans `screen.publicKey`.
  Si elle réussit, elle soumet les informations d’identification obtenues à Auth0 avec `action: "default"`.

  Si `navigator.credentials.get()` échoue (p. ex. en cas d’annulation par l’utilisateur, de timeout ou d’autres erreurs de l’API WebAuthn),
  il est recommandé d’appeler `reportBrowserError` avec les détails de l’erreur.

  Une promesse qui est résolue lorsque la tentative de vérification est soumise.
  Une opération réussie entraîne généralement une redirection.

  #### Throws

  Lève une erreur si `screen.publicKey` est absent,
  si `navigator.credentials.get()` échoue avec une erreur inattendue non gérée par `getPasskeyCredentials`,
  ou si la soumission du formulaire à Auth0 échoue.

  ```typescript Example
  // En supposant que 'sdk' est une instance de MfaWebAuthnPlatformChallenge
  try {
    const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
    await sdk.verify({ rememberDevice: remember });
    // En cas de réussite, Auth0 gère la redirection.
  } catch (error) {
    console.error("Platform authenticator verification failed:", error);
    // S'il s'agit d'une erreur de l'API WebAuthn, signalez-la
    if (error.name && error.message) { // Vérification de base pour une erreur de type DOMException
      await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
    }
    // Vérifiez sdk.transaction.errors pour les messages de validation côté serveur si la page se recharge.
  }
  ```
