---
title: "ErrorType"
---

Type d’union de chaînes qui catégorise les erreurs gérées par [`useErrors`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useErrors). Utilisé pour filtrer les erreurs avec `errors.byType()`, il est présent dans chaque [`ErrorItem`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ErrorItem) sous la forme de la propriété `kind`.

```typescript
type ErrorType = "auth0" | "validation" | "configuration";
```

<div id="values">
  ## Valeurs
</div>

<ParamField body="&#x22;auth0&#x22;" type="ErrorType">
  Erreurs renvoyées par Auth0 ou par votre propre backend pendant l’authentification ou l’autorisation.
</ParamField>

<ParamField body="&#x22;validation&#x22;" type="ErrorType">
  Erreurs provenant de la validation du formulaire côté client, par exemple lorsqu’une donnée saisie est invalide avant qu’une requête soit envoyée.
</ParamField>

<ParamField body="&#x22;configuration&#x22;" type="ErrorType">
  Erreurs causées par une intégration incorrecte du SDK ou une mauvaise configuration, par exemple lorsqu’un hook est appelé en dehors du context requis. Elles ne s’affichent que lorsque `includeDevErrors: true` est défini dans [`UseErrorOptions`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorOptions).
</ParamField>
