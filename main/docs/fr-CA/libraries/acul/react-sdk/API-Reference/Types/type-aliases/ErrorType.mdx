---
title: "ErrorType"
---

Un type d’union de chaînes de caractères qui classe les erreurs gérées par [`useErrors`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useErrors). Il sert à filtrer les erreurs avec `errors.byType()` et figure dans chaque [`ErrorItem`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ErrorItem) comme propriété `kind`.

```typescript
type ErrorType = "auth0" | "validation" | "configuration";
```

<div id="values">
  ## Valeurs
</div>

<ParamField body="&#x22;auth0&#x22;" type="ErrorType">
  Erreurs renvoyées par Auth0 ou par votre propre backend pendant l’authentification ou l’autorisation.
</ParamField>

<ParamField body="&#x22;validation&#x22;" type="ErrorType">
  Erreurs de validation du formulaire côté client, par exemple lorsque des données invalides sont saisies avant l’envoi d’une requête.
</ParamField>

<ParamField body="&#x22;configuration&#x22;" type="ErrorType">
  Erreurs causées par une mauvaise intégration du SDK ou une configuration incorrecte, par exemple lorsqu’un hook est appelé hors du contexte requis. Elles s’affichent uniquement lorsque `includeDevErrors: true` est défini dans [`UseErrorOptions`](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorOptions).
</ParamField>
