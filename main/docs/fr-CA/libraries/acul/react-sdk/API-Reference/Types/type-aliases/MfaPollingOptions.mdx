title: "Alias de type : MfaPollingOptions"
---

```ts Example
type MfaPollingOptions = {
  intervalMs?: number;
  onCompleted?: () => void;
  onError?: (error: Error) => void;
}
```

<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer un processus d’interrogation périodique des notifications poussées d’AMF.

  Fournissez ces options au démarrage d’une session d’interrogation afin de contrôler la fréquence
  à laquelle le point de terminaison du défi est interrogé et la manière dont le succès ou l’échec est géré.

  <Expandable title="propriétés">
    <ParamField body="intervalMs" type="number">
      Intervalle facultatif, en **millisecondes**, entre deux requêtes d’interrogation consécutives.

      * S’il n’est pas précisé, l’intervalle par défaut interne du SDK est utilisé (habituellement 5000 ms).
      * Choisissez une valeur qui offre un bon équilibre entre la réactivité et les limites de débit.

      ```ts Default Value
      Propre à la mise en œuvre (généralement 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted" type="() => void">
      Fonction de rappel facultative exécutée une fois que le défi d’AMF par notification poussée a été
      approuvé avec succès et que l’interrogation est terminée.

      * Appelée exactement **une seule fois**, après quoi l’interrogation s’arrête automatiquement.
      * Utile pour faire avancer le flux de connexion (p. ex., appeler `continue()`).
    </ParamField>

    <ParamField body="onError" type="(error: Error) => void">
      Fonction de rappel facultative invoquée si une erreur survient pendant l’interrogation.

      Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) contenant :

      * `status` — le code d’état HTTP de la requête échouée

      * `responseText` — le corps de la réponse brut, s’il est disponible

      * Appelée une fois par événement d’erreur; l’interrogation peut être reprise selon le type d’erreur.

      * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.

      **Propriétés**

      <ParamField body="error" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error">Error</a>}>
        Un objet d’erreur contenant le code d’état HTTP et le corps de la réponse brut de la requête d’interrogation échouée.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>
