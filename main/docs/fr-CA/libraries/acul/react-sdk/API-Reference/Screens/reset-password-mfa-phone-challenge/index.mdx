L’écran `reset-password-mfa-phone-challenge` s’affiche durant le flux de réinitialisation du mot de passe lorsqu’une authentification multifacteur par téléphone est requise. Il invite l’utilisateur à choisir un mode d’envoi (SMS ou appel vocal) pour recevoir un code de vérification et poursuivre le processus de réinitialisation.

<div id="import">
  ## Importation
</div>

Chaque écran possède son propre ensemble de hooks et de méthodes. Le SDK prend en charge **l’importation partielle** et **l’importation racine** pour chaque écran.

* L’importation partielle vous permet d’inclure uniquement le code nécessaire à votre cas d’utilisation.
* L’importation racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez qu’une compilation unifiée prenne en charge tous les écrans possibles.

```jsx Import Example
// importation racine
import { useResetPasswordMfaPhoneChallenge } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useResetPasswordMfaPhoneChallenge,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  continueMethod,
  tryAnotherMethod,
} from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';

function ResetPasswordMfaPhoneChallengeScreen() {
  const { continueMethod } = useResetPasswordMfaPhoneChallenge();
  return (
    <button onClick={() => continueMethod({ type: 'sms' })}>
      Send Code via SMS
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 dans l’écran `reset-password-mfa-phone-challenge`. Importez-les depuis `@auth0/auth0-acul-react/reset-password-mfa-phone-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les configurations de l’image de marque, comme le logo, les couleurs et les paramètres du thème affichés dans l’écran `reset-password-mfa-phone-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit des configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `reset-password-mfa-phone-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organization de l’utilisateur si la réinitialisation du mot de passe est propre à une Organization. Il renvoie `null` lorsqu’aucun contexte d’Organization n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No Organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```

<div id="methods">
  ## Méthodes
</div>

<ParamField body="continueMethod" type="Promise<void>">
  Cette méthode soumet le mode d’envoi choisi par l’utilisateur pour envoyer le code de vérification et poursuivre l’étape d’AMF du flux de réinitialisation du mot de passe.

  ```jsx Example
  import { useResetPasswordMfaPhoneChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';

  function SendCodeForm() {
    const { continueMethod } = useResetPasswordMfaPhoneChallenge();
    return (
      <button onClick={() => continueMethod({ type: 'sms' })}>
        Send Code via SMS
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [ResetPasswordMfaPhoneChallengeContinueOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaPhoneChallengeContinueOptions).
    </ParamField>

    <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">
      Le mode d’envoi sélectionné par l’utilisateur pour recevoir le code de vérification.

      * `sms` : Envoie le code par message texte.
      * `voice` : Envoie le code par appel vocal.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Cette méthode redirige l’utilisateur pour qu’il sélectionne une autre méthode d’AMF.

  ```jsx Example
  import { useResetPasswordMfaPhoneChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useResetPasswordMfaPhoneChallenge();
    return (
      <button onClick={() => tryAnotherMethod({ type: 'sms' })}>
        Try Another Method
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions).
    </ParamField>

    <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">
      Le mode d’envoi qui était affiché ou sélectionné initialement sur l’écran actuel.
      Ce renseignement est requis par le point de terminaison d’API pour traiter correctement l’action &#39;pick-authenticator&#39;.

      * `sms` : Indique que l’écran actuel était celui du défi SMS.
      * `voice` : Indique que l’écran actuel était celui du défi vocal.
    </ParamField>
  </Expandable>
</ParamField>
