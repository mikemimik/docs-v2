L’écran `mfa-sms-challenge` invite l’utilisateur à saisir un code à usage unique envoyé à son téléphone par SMS pour compléter la vérification AMF.

<div id="import">
  ## Importation
</div>

Chaque écran a son propre ensemble de hooks et de méthodes. Le SDK prend en charge l’**importation partielle** et l’**importation à la racine** pour chaque écran.

* L’**importation partielle** vous permet d’inclure uniquement le code dont vous avez besoin pour votre cas d’utilisation.
* L’**importation à la racine** vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous voulez une version unifiée capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useMfaSmsChallenge } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useMfaSmsChallenge,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  useResend,
  // Méthodes
  continueMfaSmsChallenge,
  getACall,
  pickSms,
  resendCode,
  tryAnotherMethod,
} from "@auth0/auth0-acul-react/mfa-sms-challenge";

function MfaSmsChallengeScreen() {
  const { continueMfaSmsChallenge } = useMfaSmsChallenge();
  return (
    <button onClick={() => continueMfaSmsChallenge({ code: '123456' })}>
      Verify Code
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 sur l’écran `mfa-sms-challenge`. Importez-les à partir de `@auth0/auth0-acul-react/mfa-sms-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les configurations d’image de marque, comme le logo, les couleurs et les paramètres de thème affichés sur l’écran `mfa-sms-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit les configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `mfa-sms-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organization de l’utilisateur si le flux d’AMF est limité à une Organization. Renvoie `null` lorsqu’aucun contexte d’Organization n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnMfaSmsChallenge">ScreenMembersOnMfaSmsChallenge</a></span>}>
  Ce hook contient des détails propres à l’écran `mfa-sms-challenge`, y compris sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `mfa-sms-challenge`, comme l’état actuel du flux d’AMF.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembersOnMfaSmsChallenge">UntrustedDataMembersOnMfaSmsChallenge</a></span>}>
  Ce hook gère les données non fiables transmises à l’écran, comme les valeurs préremplies provenant des paramètres d’URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Ce hook fournit des détails sur l’utilisateur actif, notamment `username`, `email` et les méthodes d’authentification disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/mfa-sms-challenge';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useMfaSmsChallenge" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaSmsChallengeMembers">MfaSmsChallengeMembers</a>}>
  Ce hook renvoie toutes les méthodes et tout le contexte disponibles dans l’écran `mfa-sms-challenge`.
</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="continueMfaSmsChallenge" type="Promise<void>">
  Cette méthode envoie le code à usage unique par SMS pour terminer la vérification AMF.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function VerifyCodeForm() {
    const { continueMfaSmsChallenge } = useMfaSmsChallenge();
    return (
      <button onClick={() => continueMfaSmsChallenge({ code: '123456' })}>
        Vérifier le code
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaSmsChallengeOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaSmsChallengeOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      Le code à usage unique envoyé par SMS au téléphone de l&#39;utilisateur.
    </ParamField>

    <ParamField body="rememberDevice" type="boolean">
      Indique s&#39;il faut mémoriser l&#39;appareil et ne pas demander l&#39;AMF lors des prochaines connexions.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getACall" type="Promise<void>">
  Cette méthode fait passer la vérification AMF à un appel vocal plutôt qu&#39;à un SMS.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function GetACallButton() {
    const { getACall } = useMfaSmsChallenge();
    return (
      <button onClick={() => getACall()}>
        Recevoir un appel à la place
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="pickSms" type="Promise<void>">
  Cette méthode redirige vers l&#39;écran de sélection du numéro de téléphone pour le SMS.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function PickSmsButton() {
    const { pickSms } = useMfaSmsChallenge();
    return (
      <button onClick={() => pickSms()}>
        Utiliser un autre numéro de téléphone
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="resendCode" type="Promise<void>">
  Cette méthode renvoie le code à usage unique par SMS au téléphone de l&#39;utilisateur.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function ResendCodeButton() {
    const { resendCode } = useMfaSmsChallenge();
    return (
      <button onClick={() => resendCode()}>
        Renvoyer le code
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Cette méthode redirige vers l&#39;écran de sélection de la méthode AMF afin que l&#39;utilisateur puisse choisir un autre facteur d&#39;authentification.

  ```jsx Example
  import { useMfaSmsChallenge } from '@auth0/auth0-acul-react/mfa-sms-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useMfaSmsChallenge();
    return (
      <button onClick={() => tryAnotherMethod()}>
        Essayer une autre méthode
      </button>
    );
  }
  ```
</ParamField>
