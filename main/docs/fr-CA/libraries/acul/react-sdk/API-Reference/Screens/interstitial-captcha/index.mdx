---

L’écran `interstitial-captcha` s’affiche lorsqu’un défi CAPTCHA est requis dans le flux d’authentification. Il invite l’utilisateur à remplir le CAPTCHA avant de poursuivre.

<div id="import">
  ## Importation
</div>

Chaque écran a son propre ensemble de hooks et de méthodes. Le SDK prend en charge l’**importation partielle** et l’**importation racine** pour chaque écran.

* L’importation partielle vous permet d’inclure uniquement le code nécessaire à votre cas d’utilisation.
* L’importation racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez qu’une version unifiée prenne en charge tous les écrans possibles.

```jsx Import Example
// importation racine
import { useInterstitialCaptcha } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useInterstitialCaptcha,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  submitCaptcha,
} from '@auth0/auth0-acul-react/interstitial-captcha';

function InterstitialCaptchaScreen() {
  const { submitCaptcha } = useInterstitialCaptcha();
  return (
    <button onClick={() => submitCaptcha({ captcha: 'token' })}>Submit</button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte d’Auth0 sur l’écran `interstitial-captcha`. Importez-les depuis `@auth0/auth0-acul-react/interstitial-captcha`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les configurations de l’image de marque, comme le logo, les couleurs et les paramètres de thème affichés sur l’écran `interstitial-captcha`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/interstitial-captcha';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit les configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `interstitial-captcha`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/interstitial-captcha';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organization de l’utilisateur si le défi CAPTCHA est limité à l’Organization. Il retourne `null` lorsqu’aucun contexte d’Organization n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/interstitial-captcha';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>Aucun contexte d’Organization</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/interstitial-captcha';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembers">ScreenMembers</a></span>}>
  Ce hook contient des détails propres à l’écran `interstitial-captcha`, y compris sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/interstitial-captcha';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/interstitial-captcha';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `interstitial-captcha`, comme les connexions actives et l’état actuel du flux.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/interstitial-captcha';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Ce hook gère les données non fiables transmises à l’écran, comme les paramètres préremplis provenant de la chaîne de requête de l’URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/interstitial-captcha';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Ce hook fournit des renseignements sur l’utilisateur actif, notamment `username`, `email` et les méthodes d’authentification disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/interstitial-captcha';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useInterstitialCaptcha" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/InterstitialCaptchaMembers">InterstitialCaptchaMembers</a>}>
  Ce hook renvoie toutes les méthodes et tout le contexte disponibles à l’écran `interstitial-captcha`.
</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="submitCaptcha" type="Promise<void>">
  Cette méthode envoie le jeton CAPTCHA afin de terminer la vérification et de poursuivre le flux d’authentification.

  ```jsx Example
  import { useInterstitialCaptcha } from '@auth0/auth0-acul-react/interstitial-captcha';

  function SubmitButton() {
    const { submitCaptcha } = useInterstitialCaptcha();
    return (
      <button onClick={() => submitCaptcha({ captcha: 'token' })}>
        Continue
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SubmitCaptchaOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/SubmitCaptchaOptions).
    </ParamField>

    <ParamField body="captcha" type="string" required>
      Le jeton de réponse CAPTCHA.
    </ParamField>
  </Expandable>
</ParamField>

<div id="commonutility-hooks">
  ## Hooks communs/utilitaires
</div>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Ce hook renvoie une fonction permettant de changer la langue d’affichage sur l’écran ACUL actuel.
</ParamField>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Ce hook récupère le contexte et l’état de l’écran actuel.
</ParamField>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Ce hook lit et gère les erreurs du serveur, du client et du développeur à l’écran.
</ParamField>
