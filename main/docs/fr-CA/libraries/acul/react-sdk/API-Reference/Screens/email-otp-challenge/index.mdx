L’écran `email-otp-challenge` s’affiche lorsqu’un utilisateur doit vérifier son identité en saisissant un code à usage unique envoyé à son adresse courriel. Il permet à l’utilisateur d’entrer le code ou de demander qu’il soit renvoyé.

<div id="import">
  ## Importation
</div>

Chaque écran possède son propre ensemble de hooks et de méthodes. Le SDK prend en charge **partial import** et **root import** pour chaque écran.

* L’utilisation de **partial import** vous permet d’inclure uniquement le code nécessaire à votre cas d’utilisation.
* L’utilisation de **root import** vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez une version unifiée capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useEmailOTPChallenge } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useEmailOTPChallenge,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  useResend,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  submitCode,
  resendCode,
} from '@auth0/auth0-acul-react/email-otp-challenge';

function EmailOtpChallengeScreen() {
  const { submitCode } = useEmailOTPChallenge();
  return (
    <button onClick={() => submitCode({ code: '123456' })}>Submit</button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks propres à l’écran qui offrent un accès en lecture seule aux données de contexte d’Auth0 sur l’écran `email-otp-challenge`. Importez-les depuis `@auth0/auth0-acul-react/email-otp-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les paramètres de branding, comme le logo, les couleurs et les paramètres du thème affichés sur l’écran `email-otp-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/email-otp-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit des données liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `email-otp-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/email-otp-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’organisation de l’utilisateur si le défi OTP est limité à l’organisation. Renvoie `null` lorsqu’aucun contexte d’organisation n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/email-otp-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>Aucun contexte d’organisation</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt en cours dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/email-otp-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnEmailOTPChallenge">ScreenMembersOnEmailOTPChallenge</a></span>}>
  Ce hook contient des détails propres à l’écran `email-otp-challenge`, notamment sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/email-otp-challenge';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/email-otp-challenge';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `email-otp-challenge`, comme les connections actives et l’état actuel du flux.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/email-otp-challenge';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Ce hook gère les données non fiables transmises à l’écran, comme les paramètres préremplis provenant des chaînes de requête d’URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/email-otp-challenge';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Ce hook fournit des renseignements sur l’utilisateur actif, y compris le `username`, l’`email` et les méthodes d’authentification disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/email-otp-challenge';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useEmailOTPChallenge" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/EmailOTPChallengeMembers">EmailOTPChallengeMembers</a>}>
  Ce hook renvoie toutes les méthodes et tout le contexte disponibles sur l’écran `email-otp-challenge`.
</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="resendCode" type="Promise<void>">
  Cette méthode renvoie le code à usage unique à l’adresse courriel de l’utilisateur.

  ```jsx Example
  import { useEmailOTPChallenge } from '@auth0/auth0-acul-react/email-otp-challenge';

  function ResendButton() {
    const { resendCode } = useEmailOTPChallenge();
    return (
      <button onClick={() => resendCode()}>
        Resend Code
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="submitCode" type="Promise<void>">
  Cette méthode soumet le code à usage unique saisi par l’utilisateur.

  ```jsx Example
  import { useEmailOTPChallenge } from '@auth0/auth0-acul-react/email-otp-challenge';

  function SubmitButton() {
    const { submitCode } = useEmailOTPChallenge();
    return (
      <button onClick={() => submitCode({ code: '123456' })}>
        Continue
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [OtpCodeOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OtpCodeOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      Le code OTP saisi par l’utilisateur.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">

</ParamField>
