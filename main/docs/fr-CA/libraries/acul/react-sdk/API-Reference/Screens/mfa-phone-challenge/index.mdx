L’écran `mfa-phone-challenge` invite l’utilisateur à vérifier son identité en saisissant un code envoyé à son téléphone par SMS ou par appel vocal.

<div id="import">
  ## Importation
</div>

Chaque écran a son propre ensemble de hooks et de méthodes. Le SDK prend en charge l’**importation partielle** et l’**importation racine** pour chaque écran.

* L’importation partielle vous permet d’inclure uniquement le code nécessaire à votre cas d’usage.
* L’importation racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez une build unifiée pour prendre en charge tous les écrans possibles.

```jsx Import Example
// importation racine
import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useMfaPhoneChallenge,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  continueMethod,
  pickPhone,
  tryAnotherMethod,
} from "@auth0/auth0-acul-react/mfa-phone-challenge";

function MfaPhoneChallengeScreen() {
  const { continueMethod } = useMfaPhoneChallenge();
  return (
    <button onClick={() => continueMethod({ type: 'sms' })}>
      Send Code via SMS
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Des hooks limités à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 sur l’écran `mfa-phone-challenge`. Importez-les depuis `@auth0/auth0-acul-react/mfa-phone-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit la configuration d’image de marque, comme le logo, les couleurs et les paramètres du thème affichés sur l’écran `mfa-phone-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit la configuration liée au client, comme `id`, `name` et `logoUrl`, pour l’écran `mfa-phone-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’organisation de l’utilisateur si le flux AMF est associé à une organisation. Il renvoie `null` lorsqu’aucun contexte d’organisation n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnMfaPhoneChallenge">ScreenMembersOnMfaPhoneChallenge</a></span>}>
  Ce hook contient des détails propres à l’écran `mfa-phone-challenge`, notamment sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function TenantInfo() {

  }
  ```

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `mfa-phone-challenge`, comme l’état actuel du flux AMF.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  ```

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>


<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>


</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="continueMethod" type="Promise<void>">
  Cette méthode lance le défi par téléphone en envoyant un code de vérification au téléphone de l’utilisateur selon le mode d’envoi spécifié.

  ```jsx Example
  import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react/mfa-phone-challenge';

  function PhoneChallengeForm() {
    const { continueMethod } = useMfaPhoneChallenge();
    return (
      <button onClick={() => continueMethod({ type: 'sms' })}>
        Send Code via SMS
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPhoneChallengeContinueOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPhoneChallengeContinueOptions).
    </ParamField>

    <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;" required>
      Le type de défi à effectuer.

      * `sms` : Envoyer le code par message texte.
      * `voice` : Envoyer le code par appel vocal.

      #### Par défaut

      ```ts
      'sms'
      ```
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="pickPhone" type="Promise<void>">
  Cette méthode ouvre l’écran de sélection du numéro de téléphone afin que l’utilisateur puisse choisir le numéro à utiliser pour le défi.

  ```jsx Example
  import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react/mfa-phone-challenge';

  function PickPhoneButton() {
    const { pickPhone } = useMfaPhoneChallenge();
    return (
      <button onClick={() => pickPhone()}>
        Choose Phone Number
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Cette méthode ouvre l’écran de sélection de la méthode d’AMF afin que l’utilisateur puisse choisir un autre facteur d’authentification.

  ```jsx Example
  import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react/mfa-phone-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useMfaPhoneChallenge();
    return (
      <button onClick={() => tryAnotherMethod()}>
        Try Another Method
      </button>
    );
  }
  ```
</ParamField>
