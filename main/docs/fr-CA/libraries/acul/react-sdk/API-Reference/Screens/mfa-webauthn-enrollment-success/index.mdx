L’écran `mfa-webauthn-enrollment-success` confirme que l’utilisateur a bien enregistré une clé de sécurité WebAuthn et lui permet de poursuivre le flux d’authentification.

<div id="import">
  ## Importation
</div>

Chaque écran a son propre ensemble de hooks et de méthodes. Le SDK prend en charge l’**importation partielle** et l’**importation depuis la racine** pour chaque écran.

* L’importation partielle vous permet d’inclure uniquement le code nécessaire à votre cas d’utilisation.
* L’importation depuis la racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous voulez qu’une compilation unifiée prenne en charge tous les écrans possibles.

```jsx Import Example
// importation racine
import { useMfaWebAuthnEnrollmentSuccess } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useMfaWebAuthnEnrollmentSuccess,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  continueMethod,
} from "@auth0/auth0-acul-react/mfa-webauthn-enrollment-success";

function MfaWebAuthnEnrollmentSuccessScreen() {
  const { continueMethod } = useMfaWebAuthnEnrollmentSuccess();
  return (
    <button onClick={() => continueMethod()}>
      Continue
    </button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Des hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 dans l’écran `mfa-webauthn-enrollment-success`. Importez-les depuis `@auth0/auth0-acul-react/mfa-webauthn-enrollment-success`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit des configurations d’image de marque, comme le logo, les couleurs et les paramètres du thème affichés sur l’écran `mfa-webauthn-enrollment-success`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit des configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `mfa-webauthn-enrollment-success`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des informations sur l’Organization de l’utilisateur si le flux MFA est limité à une Organization. Il renvoie `null` lorsqu’aucun contexte d’Organization n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>Aucun contexte d’Organization</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnMfaWebAuthnEnrollmentSuccess">ScreenMembersOnMfaWebAuthnEnrollmentSuccess</a></span>}>
  Ce hook contient des détails propres à l’écran `mfa-webauthn-enrollment-success`, y compris sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `mfa-webauthn-enrollment-success`, comme l’état actuel du flux MFA.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Ce hook gère les données non fiables transmises à l’écran, comme les valeurs préremplies à partir des paramètres d’URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Ce hook fournit des renseignements sur l’utilisateur actif, y compris `username`, `email` et les méthodes d’authentification disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useMfaWebAuthnEnrollmentSuccess" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaWebAuthnEnrollmentSuccessMembers">MfaWebAuthnEnrollmentSuccessMembers</a>}>
  Ce hook renvoie toutes les méthodes et le contexte disponibles dans l’écran `mfa-webauthn-enrollment-success`.
</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="continueMethod" type="Promise<void>">
  Cette méthode fait progresser le flux une fois que l’utilisateur a inscrit avec succès une clé de sécurité WebAuthn.

  ```jsx Example
  import { useMfaWebAuthnEnrollmentSuccess } from '@auth0/auth0-acul-react/mfa-webauthn-enrollment-success';

  function ContinueButton() {
    const { continueMethod } = useMfaWebAuthnEnrollmentSuccess();
    return (
      <button onClick={() => continueMethod()}>
        Continuer
      </button>
    );
  }
  ```
</ParamField>

<div id="commonutility-hooks">
  ## Hooks communs/utilitaires
</div>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">
  Ce hook récupère, à partir du contexte de l’image de marque, les options du thème actuel sous forme de configuration aplatie.
</ParamField>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Ce hook renvoie une fonction qui permet de changer la langue d’affichage sur l’écran ACUL actuel.
</ParamField>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Ce hook récupère le contexte et l’état de l’écran actuel.
</ParamField>

<ParamField body={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Ce hook permet de lire et de gérer les erreurs du serveur, du client et du développeur à l’écran.
</ParamField>
