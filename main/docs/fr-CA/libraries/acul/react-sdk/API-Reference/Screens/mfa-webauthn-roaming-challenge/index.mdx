L’écran `mfa-webauthn-roaming-challenge` demande à l’utilisateur d’effectuer un défi d’AMF à l’aide d’une clé de sécurité itinérante, comme une YubiKey ou un autre authentificateur matériel FIDO2.

<div id="import">
  ## Importation
</div>

Chaque écran possède son propre ensemble de hooks et de méthodes. Le SDK prend en charge l’**importation partielle** et l’**importation racine** pour chaque écran.

* L’importation partielle vous permet d’inclure uniquement le code nécessaire à votre cas d’utilisation.
* L’importation racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez une build unifiée capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useMfaWebAuthnRoamingChallenge,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  reportWebAuthnError,
  tryAnotherMethod,
  verify,
} from "@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge";

function MfaWebAuthnRoamingChallengeScreen() {
  const { verify } = useMfaWebAuthnRoamingChallenge();
  return (
    <button onClick={() => verify({ rememberDevice: true })}>
      Verify with Security Key
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Des hooks associés à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 sur l’écran `mfa-webauthn-roaming-challenge`. Importez-les depuis `@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les configurations d’image de marque, comme le logo, les couleurs et les paramètres de thème affichés sur l’écran `mfa-webauthn-roaming-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit des configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `mfa-webauthn-roaming-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organization de l’utilisateur si le flux AMF est limité à cette Organization. Renvoie `null` lorsqu’aucun contexte d’Organization n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnMfaWebAuthnRoamingChallenge">ScreenMembersOnMfaWebAuthnRoamingChallenge</a></span>}>
  Ce hook contient des détails propres à l’écran `mfa-webauthn-roaming-challenge`, y compris sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `mfa-webauthn-roaming-challenge`, comme l’état actuel du flux AMF.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>


<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>


<ParamField body="useMfaWebAuthnRoamingChallenge" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaWebAuthnRoamingChallengeMembers">MfaWebAuthnRoamingChallengeMembers</a>}>

</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="reportWebAuthnError" type="Promise<void>">
  Cette méthode signale une erreur WebAuthn survenue pendant le défi par clé de sécurité itinérante.

  ```jsx Example
  import { useMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';

  function HandleWebAuthnError({ error }) {
    const { reportWebAuthnError } = useMfaWebAuthnRoamingChallenge();
    return (
      <button onClick={() => reportWebAuthnError({ error })}>
        Report Error
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaWebAuthnRoamingChallengeReportErrorOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaWebAuthnRoamingChallengeReportErrorOptions).
    </ParamField>

    <ParamField body="error" type={<span><a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/WebAuthnErrorDetails">WebAuthnErrorDetails</a></span>} required>
      L&#39;objet d&#39;erreur de la WebAuthn API (`navigator.credentials.get()`) à signaler.

      <ParamField body="name" type="string" required>
        Le nom de l&#39;erreur (p. ex., `'NotAllowedError'`).
      </ParamField>

      <ParamField body="message" type="string" required>
        Le message d&#39;erreur décrivant le problème.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Cette méthode redirige vers l&#39;écran de sélection de la méthode d&#39;AMF afin que l&#39;utilisateur puisse choisir un autre facteur d&#39;authentification.

  ```jsx Example
  import { useMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useMfaWebAuthnRoamingChallenge();
    return (
      <button onClick={() => tryAnotherMethod()}>
        Try Another Method
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="verify" type="Promise<void>">
  Cette méthode lance la vérification de la clé de sécurité itinérante pour terminer le défi d&#39;AMF.

  ```jsx Example
  import { useMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';

  function VerifyButton() {
    const { verify } = useMfaWebAuthnRoamingChallenge();
    return (
      <button onClick={() => verify({ rememberDevice: true })}>
        Verify with Security Key
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [VerifySecurityKeyOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/VerifySecurityKeyOptions).
    </ParamField>

    <ParamField body="rememberDevice" type="boolean">
      Facultatif. Si la valeur est true, tente de mémoriser le navigateur pour les prochains défis d&#39;AMF.
      Correspond au champ de formulaire `rememberBrowser`.
    </ParamField>
  </Expandable>
</ParamField>
