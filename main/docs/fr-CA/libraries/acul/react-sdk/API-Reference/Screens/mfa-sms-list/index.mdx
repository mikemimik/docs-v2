L’écran `mfa-sms-list` affiche une liste des numéros de téléphone déjà inscrits et permet à l’utilisateur d’en sélectionner un pour recevoir un code de vérification par SMS.

<div id="import">
  ## Importation
</div>

Chaque écran possède son propre ensemble de hooks et de méthodes. Le SDK prend en charge **partial import** et **root import** pour chaque écran.

* L’utilisation de **partial import** vous permet d’inclure uniquement le code nécessaire à votre cas d’utilisation précis.
* L’utilisation de **root import** vous permet de charger tous les écrans à partir d’un seul package, ce qui est utile si vous souhaitez une compilation unifiée capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useMfaSmsList } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useMfaSmsList,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  backAction,
  selectPhoneNumber,
} from "@auth0/auth0-acul-react/mfa-sms-list";

function MfaSmsListScreen() {
  const { selectPhoneNumber } = useMfaSmsList();
  return (
    <button onClick={() => selectPhoneNumber({ index: 0 })}>
      Use This Phone Number
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Des hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte d’Auth0 sur l’écran `mfa-sms-list`. Importez-les depuis `@auth0/auth0-acul-react/mfa-sms-list`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit des configurations d’image de marque, comme le logo, les couleurs et les paramètres de thème affichés sur l’écran `mfa-sms-list`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-sms-list';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit des configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `mfa-sms-list`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-sms-list';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’organisation de l’utilisateur si le flux d’AMF est limité à l’organisation. Il renvoie `null` lorsqu’aucun contexte d’organisation n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-sms-list';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-sms-list';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembers">ScreenMembers</a></span>}>
  Ce hook contient des détails propres à l’écran `mfa-sms-list`, y compris sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-sms-list';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-sms-list';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `mfa-sms-list`, comme l’état actuel du flux d’AMF.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-sms-list';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Ce hook gère les données non fiables transmises à l’écran, comme les valeurs préremplies provenant des paramètres d’URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/mfa-sms-list';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Ce hook fournit des renseignements sur l’utilisateur actif, notamment `username`, `email` et les méthodes d’authentification disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/mfa-sms-list';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useMfaSmsList" type={<a href="/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaSmsListMembers">MfaSmsListMembers</a>}>
  Ce hook renvoie toutes les méthodes et tout le contexte disponibles sur l’écran `mfa-sms-list`.
</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="backAction" type="Promise<void>">
  Cette méthode permet de revenir à l’écran précédent dans le flux d’AMF.

  ```jsx Example
  import { useMfaSmsList } from '@auth0/auth0-acul-react/mfa-sms-list';

  function BackButton() {
    const { backAction } = useMfaSmsList();
    return (
      <button onClick={() => backAction()}>
        Go Back
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="selectPhoneNumber" type="Promise<void>">
  Cette méthode sélectionne un numéro de téléphone dans la liste des numéros déjà inscrits pour l’utiliser pour la vérification par SMS.

  ```jsx Example
  import { useMfaSmsList } from '@auth0/auth0-acul-react/mfa-sms-list';

  function PhoneNumberSelector() {
    const { selectPhoneNumber } = useMfaSmsList();
    return (
      <button onClick={() => selectPhoneNumber({ index: 0 })}>
        Use This Phone Number
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaSmsListOptions](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaSmsListOptions).
    </ParamField>

    <ParamField body="index" type="number" required>
      L’index du numéro de téléphone à sélectionner.
    </ParamField>
  </Expandable>
</ParamField>
