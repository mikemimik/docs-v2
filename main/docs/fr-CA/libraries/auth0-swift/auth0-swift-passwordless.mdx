L’authentification <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> permet aux utilisateurs de se connecter uniquement avec une adresse courriel ou un numéro de téléphone, ce qui réduit les obstacles liés à la nécessité de mémoriser un mot de passe. L’authentification passwordless peut se faire par courriel ou par SMS, soit en envoyant un code à l’utilisateur, soit en lui envoyant un lien contenant un code.

Pour utiliser l’authentification Passwordless, vous devez utiliser la version `1.20.0` d’Auth0.Swift ou une version ultérieure.

<div id="how-passwordless-works">
  ## Fonctionnement de Passwordless
</div>

Passwordless nécessite deux étapes :

1. Demander le code
2. Saisir le code

Avec les liens, le principe est le même, mais le fonctionnement diffère légèrement, puisque l’utilisateur n’a pas à saisir lui-même un code. Le code est inclus dans l’URL.

<div id="step-1-request-the-code">
  ### Étape 1 : Demander le code
</div>

Dans cet exemple, la demande de code se fait en appelant `startPasswordless` avec le courriel de l’utilisateur et le type de connexion. Le paramètre `type` prendra par défaut la valeur `Code`. En cas de succès, vous afficherez probablement un message à l’utilisateur pour l’informer que son code lui a été envoyé, puis vous le redirigerez peut-être vers une vue où il pourra saisir ce code.

```swift lines
Auth0
   .authentication()
   .startPasswordless(email: "support@auth0.com")
   .start { result in
       switch result {
       case .success:
           print("Sent OTP to support@auth0.com!")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="step-2-input-the-code">
  ### Étape 2 : Saisir le code
</div>

Une fois que l’utilisateur a reçu un code, il peut le saisir. Appelez la méthode `login` et transmettez l’adresse courriel de l’utilisateur, le code qu’il a reçu et le nom de la connexion concernée. Si l’opération réussit, vous recevrez un objet Credentials dans la réponse.

```swift lines
Auth0
   .authentication()
   .login(
       email: "support@auth0.com", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

Si vous utilisiez SMS, l’appel ressemblerait plutôt à cet exemple :

```swift lines
Auth0
   .authentication()
   .login(
       phoneNumber: "+4591131761367", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="passwordless-parameters">
  ## Paramètres Passwordless
</div>

L’authentification Passwordless peut être lancée avec différents paramètres.

Par exemple :

`.startPasswordless(email: String, type: String, connection: String)`

ou

`.startPasswordless(phoneNumber: String, type: String, connection: String)`

| **Paramètre** | **Obligatoire** | **Description**                                                                                                                                                             |
| ------------- | --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `email`       | obligatoire     | (String) `email` ou `phoneNumber` est requis (pas les deux), selon celui qui sera utilisé.                                                                                  |
| `phoneNumber` | obligatoire     | (String) `email` ou `phoneNumber` est requis (pas les deux), selon celui qui sera utilisé.                                                                                  |
| `type`        | facultatif      | (String) Le type de transaction Passwordless à utiliser, soit `.Code`, soit `.iOSLink`. La valeur par défaut est `.Code`.                                                   |
| `connection`  | facultatif      | (String) Le nom de la connexion à utiliser pour l’authentification Passwordless. La valeur par défaut est `sms` pour la version SMS ou `email` pour la version par courriel |
