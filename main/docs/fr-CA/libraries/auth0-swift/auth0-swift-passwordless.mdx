L’authentification <Tooltip tip="Sans mot de passe : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">sans mot de passe</Tooltip> permet aux utilisateurs de se connecter à l’aide d’une simple adresse courriel ou d’un numéro de téléphone, ce qui réduit les obstacles liés à la nécessité de se souvenir d’un mot de passe. L’authentification sans mot de passe peut se faire par courriel ou par SMS, soit en envoyant à l’utilisateur un code, soit en lui envoyant un lien qui contient un code.

Pour utiliser l’authentification sans mot de passe, il vous faut Auth0.Swift version `1.20.0` ou ultérieure.

<div id="how-passwordless-works">
  ## Comment fonctionne sans mot de passe
</div>

sans mot de passe nécessite deux étapes :

1. Demander le code
2. Saisir le code

Avec les liens, le processus est le même, mais se déroule un peu différemment, puisque l’utilisateur n’a pas à saisir lui-même un code. Le code est inclus dans l’URL.

<div id="step-1-request-the-code">
  ### Étape 1 : Demander le code
</div>

Dans cet exemple, la demande de code se fait en appelant `startPasswordless` avec l’adresse courriel de l’utilisateur et le type de connexion. Le paramètre `type` prendra par défaut la valeur `Code`. En cas de réussite, vous afficherez probablement un message à l’utilisateur pour l’informer que son code a été envoyé, puis vous le redirigerez peut-être vers une vue où il pourra le saisir.

```swift lines
Auth0
   .authentication()
   .startPasswordless(email: "support@auth0.com")
   .start { result in
       switch result {
       case .success:
           print("Sent OTP to support@auth0.com!")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="step-2-input-the-code">
  ### Étape 2 : Entrez le code
</div>

Une fois que l’utilisateur a reçu un code, il peut le saisir. Appelez la méthode `login` en transmettant l’adresse courriel de l’utilisateur, le code qu’il a reçu et le nom de la connexion concernée. Si l’opération réussit, la réponse contiendra un objet Credentials.

```swift lines
Auth0
   .authentication()
   .login(
       email: "support@auth0.com", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

Si vous avez utilisé le SMS, l’appel ressemblerait plutôt à l’exemple suivant :

```swift lines
Auth0
   .authentication()
   .login(
       phoneNumber: "+4591131761367", 
       code: "123456", 
       audience: "https://myapi.com/api",
       scope: "openid email")
   .start { result in
       switch result {
       case .success(let credentials):
           print("Access Token: \(credentials.accessToken)")
       case .failure(let error):
           print(error)
       }
   }
```

<div id="passwordless-parameters">
  ## Paramètres de sans mot de passe
</div>

L&#39;authentification sans mot de passe peut être lancée avec différents paramètres.

Par exemple :

`.startPasswordless(email: String, type: String, connection: String)`

ou

`.startPasswordless(phoneNumber: String, type: String, connection: String)`

| **Paramètre** | **Obligatoire** | **Description**                                                                                                                                                              |
| ------------- | --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `email`       | obligatoire     | (String) `email` ou `phoneNumber` est requis (pas les deux), selon celui qui sera utilisé.                                                                                   |
| `phoneNumber` | obligatoire     | (String) `email` ou `phoneNumber` est requis (pas les deux), selon celui qui sera utilisé.                                                                                   |
| `type`        | facultatif      | (String) Le type de transaction sans mot de passe à utiliser : `.Code` ou `.iOSLink`. La valeur par défaut est `.Code`.                                                           |
| `connection`  | facultatif      | (String) Le nom de la connexion à utiliser pour l&#39;authentification sans mot de passe. La valeur par défaut est `sms` pour la surcharge SMS ou `email` pour la surcharge email |
