Voici le scénario : après l’authentification de l’utilisateur, vous souhaitez stocker ses identifiants et les utiliser tant qu’ils sont valides. Une fois qu’ils expirent, vous voudrez les renouveler à l’aide du `refreshToken` afin d’éviter de réafficher la page de connexion. Au lieu de le faire automatiquement, vous demandez à l’utilisateur de valider son identité à l’aide de son empreinte digitale ou de son visage.

Vous utiliserez l’utilitaire [Credentials Manager](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) dans [Auth0.swift](https://github.com/auth0/Auth0.swift/) pour simplifier la gestion des identifiants de l’utilisateur et effectuer une authentification biométrique.

<div id="getting-started">
  ## Pour commencer
</div>

Commencez par importer le module `Auth0` :

`import Auth0`

<div id="credentials-manager">
  ### Credentials Manager
</div>

Avant de récupérer les identifiants, vous pouvez aussi activer l’authentification biométrique (Face ID ou Touch ID) prise en charge par votre appareil iOS.

Commencez par configurer le Credentials Manager. Activez ensuite la biométrie. Vous pouvez aussi fournir un titre à afficher dans la fenêtre d’invite.

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Nous vous recommandons fortement d’ajouter le paramètre [NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) au fichier `Info.plist` de votre projet afin d’indiquer la raison de l’utilisation de Face ID. Dans certains cas, si vous ne fournissez pas de chaîne de description et que l’utilisateur tente de s’authentifier avec Face ID, la tentative peut échouer.

```xml lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

<div id="login">
  ### Login
</div>

Affichez la page <Tooltip tip="Universal Login : votre application redirige l’utilisateur vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier son identité." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> et, une fois l’authentification réussie, transmettez les identifiants au Credentials Manager.

export const codeExample = `Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // Traiter l’erreur
        case .success(let credentials):
            // Stocker les informations d’authentification en toute sécurité avec le Credentials Manager
            credentialsManager.store(credentials: credentials)
        }
}`;

<div id="renew-user-credentials">
  ### Renouveler les identifiants de l’utilisateur
</div>

Si vous devez renouveler les identifiants de l’utilisateur, vous pouvez appeler la méthode `credentials` de Credentials Manager.

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // Gérer l'erreur
        // Revenir à l'écran de connexion
    }
    // Continuer à rediriger l'utilisateur car l'authentification a réussi
}
```

Il n’est pas nécessaire d’enregistrer manuellement les nouveaux identifiants, car le Credentials Manager s’en charge lors du renouvellement.
