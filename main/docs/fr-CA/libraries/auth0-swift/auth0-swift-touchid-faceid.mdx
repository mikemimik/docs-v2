Voici le scénario : après l’authentification de l’utilisateur, vous souhaitez stocker ses informations d’authentification et les utiliser tant qu’elles sont valides. Lorsqu’elles expirent, vous voudrez les actualiser à l’aide du `refreshToken` afin d’éviter d’afficher de nouveau la page de connexion. Au lieu de le faire automatiquement, vous exigez que l’utilisateur valide son identité à l’aide de son empreinte digitale ou de la reconnaissance faciale.

Vous utiliserez l’utilitaire [Credentials Manager](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) dans [Auth0.swift](https://github.com/auth0/Auth0.swift/) pour simplifier la gestion des informations d’authentification de l’utilisateur et effectuer une authentification biométrique.

<div id="getting-started">
  ## Premiers pas
</div>

Commencez par importer le module `Auth0` :

`import Auth0`

<div id="credentials-manager">
  ### Credentials Manager
</div>

Avant de récupérer les informations d’authentification, vous pouvez aussi utiliser l’authentification biométrique (Face ID ou Touch ID) prise en charge par votre appareil iOS.

Commencez par configurer le Credentials Manager. Activez ensuite la biométrie. Vous pouvez aussi fournir un titre à afficher dans l’invite.

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Nous vous recommandons fortement d’ajouter le paramètre [NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) au fichier `Info.plist` de votre projet afin d’indiquer la raison de l’utilisation de Face ID. Dans certains cas, si vous ne fournissez pas de chaîne de description et que l’utilisateur tente une authentification avec Face ID, sa tentative peut échouer.

```xml lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

<div id="login">
  ### Connexion
</div>

Affichez la page <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> puis, une fois l’authentification réussie, transmettez les informations d’authentification au Credentials Manager.

export const codeExample = `Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // Gérer l’erreur
        case .success(let credentials):
            // Stocker les informations d’authentification de manière sécurisée avec le Credentials Manager
            credentialsManager.store(credentials: credentials)
        }
}`;

<div id="renew-user-credentials">
  ### Renouveler les informations d’authentification de l’utilisateur
</div>

Si vous devez renouveler les informations d’authentification de l’utilisateur, vous pouvez appeler la méthode `credentials` du Credentials Manager.

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // Gérer l'erreur
        // Solution de secours vers l'écran de connexion
    }
    // Continuer à rediriger l'utilisateur car l'authentification a réussi
}
```

Il n’est pas nécessaire d’enregistrer manuellement les nouvelles informations d’authentification, car le Credentials Manager s’en charge lors du renouvellement.
