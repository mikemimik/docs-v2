Voici le scénario : après l’authentification de l’utilisateur, vous voulez stocker ses identifiants et les utiliser tant qu’ils sont valides. Lorsqu’ils expirent, vous voudrez les renouveler à l’aide du `refreshToken` afin d’éviter d’afficher de nouveau la page de connexion. Au lieu de le faire automatiquement, vous demandez à l’utilisateur de se valider avec son empreinte digitale ou son visage.

Vous utiliserez l’utilitaire [Credentials Manager](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) dans [Auth0.swift](https://github.com/auth0/Auth0.swift/) pour simplifier la gestion des identifiants de l’utilisateur et effectuer une authentification biométrique.

<div id="getting-started">
  ## Pour commencer
</div>

D’abord, importez le module `Auth0` :

`import Auth0`

<div id="credentials-manager">
  ### Credentials Manager
</div>

Avant de récupérer les identifiants, vous pouvez aussi utiliser l’authentification biométrique (Face ID ou Touch ID) prise en charge par votre appareil iOS.

Commencez par configurer le Credentials Manager, puis activez la biométrie. Vous pouvez aussi fournir un titre à afficher dans l’invite.

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Nous vous recommandons fortement d’ajouter le paramètre [NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) à votre fichier `Info.plist` afin d’afficher la raison de l’utilisation de Face ID. Dans certains cas, si vous ne fournissez pas de chaîne de description et que l’utilisateur tente une authentification par Face ID, sa tentative pourrait échouer.

```xml lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

<div id="login">
  ### Login
</div>

Affichez la page <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>, puis, une fois l’authentification réussie, transmettez les identifiants au Credentials Manager.

export const codeExample = `Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // Gérer l’erreur
        case .success(let credentials):
            // Stocker les identifiants de façon sécuritaire à l’aide du Credentials Manager
            credentialsManager.store(credentials: credentials)
        }
}`;

<div id="renew-user-credentials">
  ### Renouveler les identifiants de l’utilisateur
</div>

Lorsque vous devez renouveler les identifiants de l’utilisateur, vous pouvez appeler la méthode `credentials` du Credentials Manager.

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // Gérer l'erreur
        // Solution de secours vers l'écran de connexion
    }
    // Continuer à rediriger l'utilisateur car l'authentification a réussi
}
```

Il n’est pas nécessaire d’enregistrer manuellement les nouveaux identifiants, car le Credentials Manager s’en charge au moment du renouvellement.
