Dans cet article, vous verrez comment migrer votre application monopage (SPA) de [auth0.js](/docs/fr-CA/libraries/auth0js) vers [auth0-spa-js](/docs/fr-CA/libraries/auth0-single-page-app-sdk). Vous trouverez ci-dessous des scénarios utilisant auth0.js ainsi que le code équivalent avec auth0-spa-js.

<div id="functionality-that-cannot-be-migrated">
  ## Fonctionnalités qui ne peuvent pas être migrées
</div>

Il n’est pas possible de migrer directement toutes les fonctionnalités d’auth0.js vers auth0-spa-js. Parmi les scénarios qui ne peuvent pas être migrés directement, on compte :

* la [connexion intégrée avec nom d’utilisateur et mot de passe](https://auth0.github.io/auth0.js/WebAuth.html#login), ainsi que la [connexion intégrée sans mot de passe](https://auth0.github.io/auth0.js/WebAuth.html#passwordlessLogin)
* l’[inscription](https://auth0.github.io/auth0.js/WebAuth.html#signup) d’un utilisateur
* [obtenir un profil utilisateur depuis le point de terminaison /userinfo](https://auth0.github.io/auth0.js/Authentication.html#userInfo)
* [demander l’envoi d’un courriel pour modifier le mot de passe de l’utilisateur](https://auth0.github.io/auth0.js/WebAuth.html#changePassword)
* [lier des utilisateurs avec la Management API](https://auth0.github.io/auth0.js/Management.html#linkUser)
* [obtenir un utilisateur avec la Management API](https://auth0.github.io/auth0.js/Management.html#getUser)
* [mettre à jour les attributs utilisateur avec la Management API](https://auth0.github.io/auth0.js/Management.html#patchUserAttributes)
* [mettre à jour les métadonnées utilisateur avec la Management API](https://auth0.github.io/auth0.js/Management.html#patchUserMetadata)
* Certaines options configurables dans Auth0.js n’ont pas non plus d’équivalent dans auth0-spa-js. `responseType` en est un exemple. Il y a une raison pour laquelle il n’existe pas de correspondance directe 1:1 pour chaque option. Dans ce cas-ci, `responseType` n’est pas nécessaire, puisque le SDK est conçu uniquement pour les SPA et qu’il n’est donc pas nécessaire de modifier le type de réponse.

<div id="authentication-parameters-are-not-modified-anymore">
  ## Les paramètres d’authentification ne sont plus modifiés
</div>

Auth0.js convertit en interne les paramètres personnalisés que vous définissez de `camelCase` en `snake_case`. Par exemple, si vous définissez `deviceType: 'offline'`, auth0.js envoie en réalité `device_type: 'offline'` au serveur.

Lorsque vous utilisez auth0-spa-js, les paramètres personnalisés **ne sont pas convertis** de `camelCase` en `snake_case`. Il transmet les paramètres exactement tels que vous les envoyez au <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=authorization+server">serveur d’autorisation</Tooltip>.

<div id="get-the-user-information">
  ## Obtenir les renseignements de l’utilisateur
</div>

<div id="auth0js">
  ### auth0.js
</div>

La fonction `userInfo()` envoie une requête au point de terminaison [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) et renvoie le profil utilisateur.

* [userInfo()](https://auth0.github.io/auth0.js/Authentication.html#userInfo)

```jsx lines
window.addEventListener('load', () => {
  auth0.client.userInfo(accessToken, function(err, user) {
    console.log(user)
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Contrairement à auth0.js, l’Auth0 SPA SDK n’effectue pas de requête vers le point de terminaison `/userinfo` pour obtenir le profil utilisateur. Au lieu de cela, `Auth0Client.getUser()` renvoie les renseignements sur l’utilisateur tirés du `id_token` décodé.

* [Auth0Client.getUser()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getUser)

```jsx lines
window.addEventListener('load', async () => {
  const user = await auth0.getUser();
  console.log(user);
});
```

<div id="refresh-tokens">
  ## Jetons d’actualisation
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({}, function(err, authResult) {
    // Jetons d'authentification ou erreur
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Le SDK Auth0 SPA s’occupe d’actualiser vos <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jetons d’accès</Tooltip>. Chaque fois que vous appelez `getTokenSilently`, vous obtenez soit un jeton d’accès valide, soit une erreur si aucune session n’est active dans Auth0.

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently();
});
```

<div id="get-a-token-for-a-different-audience-or-with-more-scopes">
  ## Obtenir un jeton pour une autre audience ou avec plus de scopes
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  }, function(err, authResult) {
    // Jetons d'authentification ou erreur
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

L’Auth0 SPA SDK gère pour vous l’actualisation du jeton d’accès. Chaque fois que vous appelez `getTokenSilently`, vous obtiendrez soit un jeton d’accès valide, soit une erreur s’il n’y a pas de session chez Auth0.

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)
* [GetTokenSilentlyOptions](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```

Utilisez [getTokenWithPopup](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenWithPopup) pour ouvrir une fenêtre contextuelle et permettre à l’utilisateur de donner son consentement à la nouvelle API :

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenWithPopup({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```
