Voici une liste de problèmes que vous pourriez rencontrer en utilisant la bibliothèque PHP Auth0, ainsi que des pistes pour les résoudre.

**J&#39;obtiens une exception « Invalid State » quand j&#39;essaie de me connecter.**

La [validation du state](/docs/fr-CA/secure/attack-protection/state-parameters) a été ajoutée dans la version 5.1.0 pour améliorer la sécurité. Par défaut, elle utilise le stockage de session et s&#39;effectue automatiquement si vous utilisez une combinaison de `Auth0::login()` et de toute méthode qui appelle `Auth0::exchange()` dans votre callback.

Si vos utilisateurs rencontrent cette erreur :

* Assurez-vous que votre application n&#39;appelle pas accidentellement `Auth0::login()` plus d&#39;une fois, ce qui pourrait invalider le state stocké sur l&#39;appareil de l&#39;utilisateur final.
* Vérifiez que l&#39;utilisateur final utilise un navigateur moderne sur son appareil et qu&#39;il ne bloque pas les cookies.

**J&#39;obtiens** **`curl error 60: SSL certificate problem: self-signed certificate in certificate chain`** **sous Windows**

Il s&#39;agit d&#39;un problème courant avec les versions récentes de PHP sous **Windows OS** (il est lié à une incompatibilité entre Windows et la base de données des autorités de certification d&#39;OpenSSL).

1. Téléchargez cette base de données CA `https://curl.haxx.se/ca/cacert.pem` vers `c:/cacert.pem`.
2. Modifiez votre php.ini et ajoutez `openssl.cafile=c:/cacert.pem`. (Il doit pointer vers le fichier que vous avez téléchargé.)

**Mon hébergeur ne permet pas d&#39;utiliser Composer**

Le SDK PHP nécessite Composer pour gérer les dépendances (bibliothèques PHP externes). Si Composer n&#39;est pas autorisé à être installé globalement sur votre hébergement, vous pouvez quand même l&#39;installer localement pour l&#39;exécuter à partir de votre compte shell utilisateur. Vous trouverez les instructions sur le site Web de Composer : &lt;https://getcomposer.org/doc/00-intro.md#locally&gt;
