Voici une liste de problèmes que vous pourriez rencontrer lors de l’utilisation de la bibliothèque PHP d’Auth0, ainsi que des pistes pour les résoudre.

**J’obtiens une exception « Invalid State » lorsque j’essaie de me connecter.**

La [validation de l’état](/docs/fr-CA/secure/attack-protection/state-parameters) a été ajoutée dans la version 5.1.0 pour renforcer la sécurité. Par défaut, elle utilise le stockage en session et s’exécute automatiquement si vous utilisez une combinaison de `Auth0::login()` et de toute méthode qui appelle `Auth0::exchange()` dans votre callback.

Si vos utilisateurs rencontrent cette erreur:- Assurez-vous que votre application n’appelle pas accidentellement `Auth0::login()` plus d’une fois, ce qui pourrait invalider l’état stocké sur l’appareil de l’utilisateur final.- Assurez-vous que l’utilisateur final utilise un navigateur moderne sur son appareil et qu’il ne bloque pas les cookies.

**J’obtiens** **`curl error 60: SSL certificate problem: self-signed certificate in certificate chain`** **sous Windows**

Il s’agit d’un problème courant avec les versions les plus récentes de PHP sous **Windows OS** (il est lié à une incompatibilité entre Windows et la base de données des AC d’OpenSSL).

1. Téléchargez cette base de données d’AC `https://curl.haxx.se/ca/cacert.pem` vers `c:/cacert.pem`.
2. Modifiez votre php.ini et ajoutez `openssl.cafile=c:/cacert.pem`. (Il doit pointer vers le fichier que vous avez téléchargé.)

**Mon hébergeur ne permet pas d’utiliser Composer**

Le SDK PHP nécessite Composer pour gérer les dépendances (bibliothèques PHP externes). Si Composer n’est pas autorisé à être installé globalement sur votre hébergeur, vous pouvez quand même l’installer localement pour l’exécuter à partir de votre compte shell utilisateur. Vous trouverez les instructions à ce sujet sur le site Web de Composer : &lt;https://getcomposer.org/doc/00-intro.md#locally&gt;
