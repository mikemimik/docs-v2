Le SDK Auth0 PHP fournit une classe `Auth0\SDK\Token` utilisée pour traiter des <Tooltip tip="JSON Web Token (JWT) : format standard de ID Token (et souvent d’Access Token) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Tokens">JSON Web Tokens</Tooltip> (JWT). Elle vous permet de décoder, de valider et de vérifier des jetons à utiliser dans votre application. Vous trouverez plus d’information sur les JWT et sur la façon de les créer et de les décoder sur [jwt.io](https://jwt.io/).

La classe peut traiter les jetons HS256 et RS256. Les deux types exigent que l’algorithme et les audiences valides soient configurés dans le SDK avant le traitement. Les jetons HS256 exigent que le <Tooltip tip="Client Secret : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne devrait être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">secret client</Tooltip> soit configuré. Les jetons RS256 exigent un émetteur autorisé, qui sert à récupérer un fichier JWKs pendant le processus de décodage. ([Plus d’information sur les algorithmes de signature ici](https://auth0.com/blog/navigating-rs256-and-jwks/).)

<div id="prerequisites">
  ## Prérequis
</div>

La documentation ci-dessous suppose que vous avez suivi les étapes du [guide de prise en main PHP](/docs/fr-CA/libraries/auth0-php) et que vous poursuivez à partir du code qui y est fourni.

<div id="example-usage">
  ## Exemple d’utilisation
</div>

Voici un exemple de petit processeur de JSON Web Token basé sur des URL, fondé sur la classe `Token` du SDK.

```php lines expandable
<?php

// Importez le chargeur automatique Composer pour rendre les classes du SDK accessibles :
require 'vendor/autoload.php';

// Chargez nos variables d'environnement depuis le fichier .env :
(Dotenv\Dotenv::createImmutable(__DIR__))->load();

$token = filter_var($_GET['token'] ?? null, FILTER_UNSAFE_RAW, FILTER_NULL_ON_FAILURE);
$algorithm = filter_var($_GET['algorithm'] ?? 'HS256', FILTER_UNSAFE_RAW, FILTER_NULL_ON_FAILURE);

if ($token === null) {
    die('No `token` request parameter.');
}

if (! in_array($algorithm, ['HS256', 'RS256'])) {
    die('Invalid `algorithm` supplied.');
}

// Le SDK Auth0 inclut un utilitaire pratique de traitement de token que nous allons utiliser ici :
$token = new \Auth0\SDK\Token([
    'domain' => $env['AUTH0_DOMAIN'],
    'clientId' => $env['AUTH0_CLIENT_ID'],
    'clientSecret' => $env['AUTH0_CLIENT_SECRET'],
    'tokenAlgorithm' => $algorithm
], $token, \Auth0\SDK\Token::TYPE_ID_TOKEN);

// Vérifiez le token : (Cela lèvera une \Auth0\SDK\Exception\InvalidTokenException si la vérification échoue.)
$token->verify();

// Validez les claims du token : (Cela lèvera une \Auth0\SDK\Exception\InvalidTokenException si la validation échoue.)
$token->validate();

echo '<pre>';
print_r($token->toArray(), true);
echo '</pre>';
```

Les méthodes `verify()` et `validate()` offrent plusieurs arguments d’option permettant de personnaliser leur comportement, notamment pour valider les claims de <Tooltip tip="Nonce : nombre arbitraire émis une seule fois dans un protocole d’authentification afin de détecter et de prévenir les attaques par rejeu." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=nonce">nonce</Tooltip>, limiter le délai maximal écoulé depuis le `auth_time` d’un jeton, définir la tolérance d’horloge `leeway` pour les vérifications de temps, et plus encore. Ces méthodes sont entièrement commentées afin que vous puissiez passer en revue ces options dans le code source ou dans l’IDE de votre choix.
