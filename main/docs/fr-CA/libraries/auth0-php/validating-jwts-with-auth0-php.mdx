Le SDK Auth0 PHP fournit une classe `Auth0\SDK\Token` servant à traiter les <Tooltip tip="JSON Web Token (JWT) : format standard d’ID Token (et souvent d’access token) utilisé pour représenter des claims de manière sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Tokens">JSON Web Tokens</Tooltip> (JWT). Elle vous permet de décoder, de valider et de vérifier les jetons afin que votre application puisse les utiliser. Vous trouverez plus d’informations sur les JWT et sur la façon de les créer et de les décoder sur [jwt.io](https://jwt.io/).

La classe prend en charge les jetons HS256 et RS256. Les deux types exigent que l’algorithme et des audiences valides soient configurés dans le SDK avant le traitement. Les jetons HS256 exigent que le <Tooltip tip="Client Secret : secret utilisé par un client (application) pour s’authentifier auprès de l’Authorization Server; il ne doit être connu que du client et de l’Authorization Server et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">client secret</Tooltip> soit configuré. Les jetons RS256 exigent un émetteur autorisé, qui est utilisé pour récupérer un fichier JWKS pendant le processus de décodage. ([En savoir plus sur les algorithmes de signature ici](https://auth0.com/blog/navigating-rs256-and-jwks/).)

<div id="prerequisites">
  ## Prérequis
</div>

La documentation ci-dessous suppose que vous avez suivi les étapes du [guide de démarrage avec PHP](/docs/fr-CA/libraries/auth0-php) et que vous poursuivez à partir du code qui y est fourni.

<div id="example-usage">
  ## Exemple d’utilisation
</div>

Voici un exemple simple de processeur de JSON Web Jeton reposant sur une URL et fondé sur la classe `Token` du SDK.

```php lines expandable
<?php

// Importation du chargeur automatique Composer pour rendre les classes du SDK accessibles :
require 'vendor/autoload.php';

// Chargement des variables d'environnement depuis le fichier .env :
(Dotenv\Dotenv::createImmutable(__DIR__))->load();

$token = filter_var($_GET['token'] ?? null, FILTER_UNSAFE_RAW, FILTER_NULL_ON_FAILURE);
$algorithm = filter_var($_GET['algorithm'] ?? 'HS256', FILTER_UNSAFE_RAW, FILTER_NULL_ON_FAILURE);

if ($token === null) {
    die('No `token` request parameter.');
}

if (! in_array($algorithm, ['HS256', 'RS256'])) {
    die('Invalid `algorithm` supplied.');
}

// Le SDK Auth0 inclut un utilitaire pratique de traitement de token que nous utiliserons ici :
$token = new \Auth0\SDK\Token([
    'domain' => $env['AUTH0_DOMAIN'],
    'clientId' => $env['AUTH0_CLIENT_ID'],
    'clientSecret' => $env['AUTH0_CLIENT_SECRET'],
    'tokenAlgorithm' => $algorithm
], $token, \Auth0\SDK\Token::TYPE_ID_TOKEN);

// Vérification du token : (Cela lèvera une \Auth0\SDK\Exception\InvalidTokenException si la vérification échoue.)
$token->verify();

// Validation des claims du token : (Cela lèvera une \Auth0\SDK\Exception\InvalidTokenException si la validation échoue.)
$token->validate();

echo '<pre>';
print_r($token->toArray(), true);
echo '</pre>';
```

`verify()` et `validate()` offrent tous deux un certain nombre d’arguments d’options qui permettent de personnaliser leur comportement, notamment la validation des claims de <Tooltip tip="Nonce : Nombre arbitraire émis une seule fois dans un protocole d’authentification pour détecter et prévenir les attaques par rejeu." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=nonce">nonce</Tooltip>, la restriction du délai maximal depuis le `auth_time` d’un jeton, la tolérance d’horloge `leeway` pour les vérifications temporelles, et plus encore. Ces méthodes sont entièrement commentées, ce qui vous permet de consulter ces options dans le code source ou dans l’IDE de votre choix.
