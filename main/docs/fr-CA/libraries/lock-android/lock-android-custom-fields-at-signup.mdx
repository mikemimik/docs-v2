---
description: Ajout de champs supplémentaires à l’inscription avec Lock.Android
title: 'Lock.Android: champs personnalisés à l’inscription'
---

**Lock.Android** vous permet d’ajouter des champs supplémentaires que l’utilisateur doit remplir avant de créer un nouveau compte. Ces champs s’affichent sur un deuxième écran, après que l’utilisateur a rempli les champs de base (courriel, nom d’utilisateur, mot de passe).

<div id="create-the-custom-fields">
  ## Créer les champs personnalisés
</div>

Créez un nouvel objet `CustomField` en lui passant ces 4 paramètres obligatoires.

1. Icon : le `int` qui pointe vers la ressource que vous voulez utiliser comme icône (gardez-la petite).
2. Type : le `FieldType` à utiliser pour ce champ. Le type définit la disposition du clavier et, parfois, la validation des entrées.
3. Key : le `String` qui identifie cette valeur dans le JSON de résultat. Il ne doit pas être répété ! Si des clés de champ sont répétées, le deuxième champ sera retiré de la liste.
4. Hint : le `@StringRes` du texte à afficher comme indication dans le champ.

De plus, vous pouvez préciser où le champ sera stocké dans le profil de l&#39;utilisateur. Consultez la section [Storage](#storage) ci-dessous pour en savoir plus.

Vous pouvez aussi créer des champs qui ne seront pas affichés à l&#39;utilisateur, mais qui peuvent servir à inclure des metadata supplémentaires de façon dynamique après l&#39;inscription.

```kotlin lines
// Champ standard
val nameField = CustomField(R.drawable.ic_name, FieldType.TYPE_NAME, "first_name", R.string.hint_first_name, CustomField.Storage.PROFILE_ROOT)

// Champ masqué
val androidField = HiddenField("android_version", Build.VERSION.SDK_INT.toString(), Storage.USER_METADATA)

val customFields = listOf(nameField, androidField)
```

Répétez les étapes ci-dessus autant de fois que nécessaire pour créer le nombre de champs souhaité.

<div id="use-the-custom-fields">
  ## Utiliser les champs personnalisés
</div>

Passez la liste des champs personnalisés à l’instance Lock lors de sa création, à l’aide de la méthode `withSignUpFields()`.

```kotlin lines
val lock = Lock.newBuilder(auth0, callback)
              .withSignUpFields(customFields)
              //...
              .build(this)
```

Si vous avez activé l’inscription dans le Dashboard de l’application, les utilisateurs seront invités à remplir les champs restants après avoir rempli les champs de base (courriel/username, mot de passe) et cliqué sur **Submit**. L’utilisateur doit remplir tous les champs personnalisés avant de pouvoir terminer l’inscription.

Au moment de l’inscription, les champs supplémentaires sont rattachés à l’attribut `user_metadata` ou définis directement à la racine du profil utilisateur, selon l’option Storage choisie. Vous pouvez accéder à ces propriétés en interrogeant le profil utilisateur à tout moment, même à partir du Dashboard, dans la section Users.

<div id="field-types">
  ## Types de champs
</div>

Chaque champ personnalisé ne peut être associé qu’à un seul `FieldType`.

* TYPE&#95;NAME
* TYPE&#95;NUMBER
* TYPE&#95;PHONE&#95;NUMBER
* TYPE&#95;EMAIL

<div id="storage">
  ## Storage
</div>

Chaque champ personnalisé ne peut être associé qu’à un seul `Storage`. Vous pouvez choisir de l’enregistrer au root level, dans un attribut root du profil, ou dans l’attribut `user_metadata`. Pour préciser l’emplacement de stockage, utilisez le constructeur à cinq paramètres et transmettez le paramètre `Storage` de votre choix. Par défaut, les champs sont enregistrés dans l’attribut `user_metadata`.

Choix possibles :

* PROFILE&#95;ROOT
* USER&#95;METADATA (par défaut)

Pour que les champs soient enregistrés au root level du profil de l’utilisateur, leurs clés doivent correspondre à celles indiquées dans la [documentation de l’endpoint](https://auth0.com/docs/api/authentication#signup).
