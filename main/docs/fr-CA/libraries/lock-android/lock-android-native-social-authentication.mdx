<div id="native-provider-google">
  ## Provider natif - Google
</div>

Vous pouvez utiliser Google AuthProvider pour vous connecter avec ou sans **Lock**. Assurez-vous de suivre les instructions dans la section [Setup](#setup).

[Lock-Google.Android](https://github.com/okta-graveyard/Lock-Google.Android)
nécessite l’API Android 15 ou une version ultérieure, ainsi que Google Play Services 10.+

<div id="latest-version-of-lock-google">
  ### Dernière version de Lock-Google
</div>

Lock-Google est offert sur [Maven Central](https://central.sonatype.com/). Pour l’installer, ajoutez simplement la ligne suivante à votre fichier `build.gradle` :

`compile 'com.auth0.android:lock-google:1.+'`

Vous pouvez consulter la version la plus récente dans le [Readme](https://github.com/okta-graveyard/Lock-Google.Android#install) du dépôt ou dans [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock-google).

<div id="lock-google-setup">
  ### Configuration de Lock-Google
</div>

<div id="google-cloud-console">
  #### Google Cloud Console
</div>

1. Accédez à la [Google Cloud Console](https://console.cloud.google.com) et créez un nouveau projet.
2. Remplissez l’[écran de consentement OAuth](https://console.cloud.google.com/apis/credentials/consent) en fournissant au minimum une adresse courriel valide et un nom.
3. Dans le panneau de navigation à gauche, cliquez sur [Credentials](https://console.cloud.google.com/apis/credentials).
4. Créez un nouvel identifiant en cliquant sur le bouton [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) et en choisissant **OAuth client ID**. Ensuite, choisissez **Web Application** et donnez-lui un nom comme &quot;Auth0 Server Google-OAuth&quot;. Remplissez les **Authorized redirect URIs** en y saisissant votre URL de rappel, qui devrait ressembler à `https://{yourDomain}/login/callback`. Assurez-vous d’appuyer sur ENTRÉE avant de quitter le champ, puis cliquez sur le bouton Create. Prenez en note les valeurs `CLIENT ID` et `CLIENT SECRET`, car nous les utiliserons plus tard.
5. Créez un nouvel identifiant en cliquant sur le bouton [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) et en choisissant **OAuth client ID**. Ensuite, choisissez **Android** et donnez-lui un nom comme &quot;Auth0 Android Google-OAuth&quot;. Obtenez le **SHA-1** du certificat que vous utilisez pour signer votre application et saisissez-le dans le premier champ. Si vous avez besoin d’aide pour obtenir le SHA-1, consultez [cette section](#certificate-fingerprints). Enfin, remplissez le dernier champ avec le **Package Name** de votre application Android, puis cliquez sur le bouton Create. Prenez en note la valeur `CLIENT ID`, car nous l’utiliserons plus tard.

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Accédez à l’Auth0 Dashboard et cliquez sur [Social Connections](https://manage.auth0.com/#/connections/social).
2. Cliquez sur **Google** et une boîte de dialogue s’ouvrira.
3. Remplissez le champ &quot;Client ID&quot; avec la valeur `CLIENT ID` obtenue à l’étape 4 de la section **Google Cloud Console** ci-dessus.
4. Remplissez le champ &quot;Client Secret&quot; avec la valeur `CLIENT SECRET` obtenue à l’étape 4 de la section **Google Cloud Console** ci-dessus.
5. Remplissez le champ &quot;Allowed Mobile Client IDs&quot; avec le `CLIENT ID` obtenu à l’étape 5 de la section **Google Cloud Console** ci-dessus.
6. Cliquez sur le bouton Save.
7. Accédez à l’Auth0 Dashboard et cliquez sur [Applications](https://manage.auth0.com/#/applications). Si vous n’en avez pas encore créé, faites-le d’abord, puis accédez à la page de configuration de votre application.
8. Au bas de la page, cliquez sur le lien &quot;Show Advanced Settings&quot;, puis accédez à l’onglet &quot;Device Settings&quot;.
9. Dans la section Android, remplissez le champ **Package Name** avec le nom de package de votre application. Enfin, remplissez le champ **Key Hashes** avec le SHA-256 du certificat que vous utilisez pour signer votre application. Si vous avez besoin d’aide pour obtenir le SHA-256, consultez [cette](#certificate-fingerprints) section. Cliquez sur le bouton &quot;Save Changes&quot;.

<div id="android-application">
  #### Application Android
</div>

1. Dans votre application Android, créez une nouvelle ressource de type String dans le fichier `res/strings.xml`. Nommez-la `google_server_client_id` et donnez-lui comme valeur le `CLIENT_ID` obtenu à l’étape 5 de la section **Setup** de **Google Cloud Console** ci-dessus.
2. Ajoutez les Metadata de version de Google Play Services au fichier `AndroidManifest.xml`, à l’intérieur de la balise Application.

   ```xml lines
   <meta-data
       android:name="com.google.android.gms.version"
       android:value="@integer/google_play_services_version" />
   ```
3. Ajoutez la permission Android Internet à votre fichier `AndroidManifest.xml`.

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. Lors de la création d’une nouvelle instance de `GoogleAuthProvider`, transmettez la valeur `google_server_client_id` obtenue précédemment comme premier paramètre :

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private GoogleAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new GoogleAuthProvider(getString(R.string.google_server_client_id), client);
     }

     // ...
   }
   ```

Si vous avez besoin d’aide supplémentaire pour configurer le SDK Google, veuillez consulter le [Sign-In for Android Guide](https://developers.google.com/identity/android-credential-manager).

<div id="usage-without-lock">
  ### Utilisation sans Lock
</div>

Si vous comptez utiliser ce fournisseur sans **Lock**, assurez-vous de redéfinir la méthode `onActivityResult()` et de rediriger l’appel vers l’instance du fournisseur. Enfin, appelez start pour lancer le processus d’authentification.

```kotlin lines
// Définissez vos propres codes de requête
private static final int RC_PERMISSIONS = 101;
private static final int RC_AUTHENTICATION = 102;

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    if (provider.authorize(requestCode, resultCode, data)) {
        return;
    }
    super.onActivityResult(requestCode, resultCode, data);
}

private void beginAuthentication(){
  provider.start(this, callback, RC_PERMISSIONS, RC_AUTHENTICATION);
}
```

Et voilà, vous êtes prêt à lancer l’application et à vous connecter avec le fournisseur natif Google !!

### Autres options

<div id="using-a-custom-connection-name">
  #### Utiliser un nom de connexion personnalisé
</div>

Pour utiliser un nom personnalisé de connexion sociale pour vous authentifier auprès d’Auth0, créez l’instance GoogleAuthProvider à l’aide du deuxième constructeur :

```kotlin wrap lines
GoogleAuthProvider provider = new GoogleAuthProvider("my-connection", "google-server-client-id", client);
```

<div id="send-additional-authentication-parameters">
  #### Envoyer des paramètres d’authentification supplémentaires
</div>

Pour envoyer des paramètres supplémentaires dans l’appel d’authentification, utilisez `setParameters`.

```kotlin lines
Map<String, Object> parameters = new HashMap<>();
//Ajouter des entrées
provider.setParameters(parameters);
```

<div id="requesting-a-custom-google-scope">
  #### Demander une portée Google personnalisée
</div>

Par défaut, la portée `Scopes.PLUS_LOGIN` est demandée. Vous pouvez personnaliser les portées en appelant `setScopes` avec la liste des portées. Chaque API Google (Auth, Drive, Plus..) définit sa propre liste de portées Google.

```kotlin wrap lines
provider.setScopes(Arrays.asList(new Scope(Scopes.PLUS_ME), new Scope(Scopes.PLUS_LOGIN)));
```

<div id="requesting-custom-android-runtime-permissions">
  #### Demander des autorisations Android personnalisées à l’exécution
</div>

Ce Provider n’exige aucune autorisation spéciale dans le Manifest Android pour authentifier l’utilisateur. Toutefois, si votre cas d’utilisation l’exige, vous pouvez laisser l’AuthProvider s’en charger. Utilisez la méthode `setRequiredPermissions`.

```kotlin wrap lines
provider.setRequiredPermissions(new String[]{"android.permission.GET_ACCOUNTS"});
```

Si vous n’utilisez pas Lock, vous devrez gérer vous-même le résultat de la demande de permissions. Pour ce faire, assurez-vous que votre activité implémente l’interface `ActivityCompat.OnRequestPermissionsResultCallback`. Lorsque la méthode `onRequestPermissionsResult` est appelée, transmettez le résultat au provider en appelant `provider.onRequestPermissionsResult`.

<div id="log-out-clear-account">
  #### Déconnecter / effacer le compte.
</div>

Pour déconnecter l’utilisateur afin qu’on lui demande de nouveau ses identifiants la prochaine fois, appelez `clearSession`. Après cela, l’état du fournisseur sera invalide et vous devrez appeler `start` de nouveau avant d’essayer d’`authorize` un résultat. L’appel à `stop` a le même effet.

`provider.clearSession();`

<div id="remember-the-last-login">
  #### Se souvenir de la dernière connexion
</div>

Par défaut, ce fournisseur mémorise le dernier compte utilisé pour se connecter. Si vous voulez modifier ce comportement, utilisez la méthode suivante.

`provider.rememberLastLogin(false);`

<div id="certificate-fingerprints">
  #### Empreintes de certificat
</div>

Lorsque vous créez un nouvel identifiant <Tooltip tip="OAuth 2.0 : cadre d’autorisation qui définit les protocoles et les processus d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> dans la Google Cloud Console, vous devez fournir la valeur SHA-1 du certificat que vous utilisez pour signer votre application. Lorsque vous terminez la Configuration de votre application dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vous devez aussi fournir la valeur SHA-256. Voici un exemple de commande à exécuter dans le terminal pour obtenir cette valeur, ainsi qu’un exemple de résultat.

Commande :

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

Exemple de sortie :

`no71633JAC3qgzQYCbskprUr55k=`

Si vous avez besoin d’aide, vous pouvez consulter ce [Keystores Guide](/docs/fr-CA/libraries/auth0-android/android-development-keystores-hashes) pour obtenir ces valeurs.

<div id="lock-facebook-setup">
  ### Configuration de Lock-Facebook
</div>

<div id="facebook-developers-console">
  #### Facebook Developers Console
</div>

1. Accédez à la [Facebook Developers Console](https://developers.facebook.com/) et créez une nouvelle App : choisissez &quot;Android&quot; et donnez-lui un nom valide. Cliquez sur &quot;Create new Facebook App ID&quot;.
2. Ajoutez le **Package Name** de votre application ainsi que le nom de la **classe Activity** dans laquelle vous utilisez le fournisseur, puis cliquez sur le bouton Next.
3. Ajoutez les **Key Hashes** SHA-1 encodés en Base64 des certificats que vous utilisez pour signer votre application, puis cliquez sur le bouton Next. Si vous avez besoin d&#39;aide pour obtenir le SHA-1, consultez [cette section](#certificate-fingerprints).
4. Enfin, faites défiler la page jusqu&#39;en haut et cliquez sur le bouton Skip Quickstart pour accéder à la page de votre application Facebook.
5. En haut de la page, vous trouverez les valeurs `APP ID` et `APP SECRET`. Enregistrez-les, car vous en aurez besoin plus tard.
6. À gauche, vous trouverez le volet de navigation. Cliquez sur Settings, puis sur Basic. Activez le commutateur **Single Sign-On** et cliquez sur le bouton Save.
7. Cliquez sur Settings, puis sur Advanced. Activez le commutateur **Native or desktop app?**.

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Accédez à l’Auth0 Dashboard et cliquez sur [Social Connections](https://manage.auth0.com/#/connections/social).
2. Cliquez sur **Facebook**; une boîte de dialogue s’ouvrira.
3. Remplissez le champ &quot;App ID&quot; avec la valeur `APP ID` obtenue à l’étape 5 de la section **Facebook Developers Console** ci-dessus.
4. Remplissez le champ &quot;App Secret&quot; avec la valeur `APP SECRET` obtenue à l’étape 5 de la section **Facebook Developers Console** ci-dessus.
5. Cliquez sur le bouton Save.
6. Accédez à l’Auth0 Dashboard et cliquez sur [Applications](https://manage.auth0.com/#/applications). Si vous n’en avez pas encore créé une, faites-le d’abord, puis ouvrez la page de configuration de votre application.
7. Au bas de la page, cliquez sur le lien &quot;Show Advanced Settings&quot;, puis accédez à l’onglet &quot;Device Settings&quot;.
8. Dans la section Android, remplissez le champ **Package Name** avec le nom du package de votre application. Enfin, remplissez le champ **Key Hashes** avec la valeur SHA-256 du certificat que vous utilisez pour signer votre application. Si vous avez besoin d’aide pour obtenir cette valeur SHA-256, consultez [cette](#certificate-fingerprints) section. Cliquez sur le bouton &quot;Save Changes&quot;.

<div id="android-application">
  #### Application Android
</div>

1. Dans votre application Android, créez une nouvelle ressource String dans le fichier `res/strings.xml`. Nommez-la `facebook_app_id` et attribuez-lui comme valeur l’`App ID` obtenu à l’étape 5 de la section **Facebook Developers Console** ci-dessus.
2. Ajoutez `FacebookActivity` et les métadonnées `facebook_app_id` au fichier `AndroidManifest.xml`, à l’intérieur de la balise Application.

   ```xml lines
   <activity
       android:name="com.facebook.FacebookActivity"
       android:configChanges="keyboard|keyboardHidden|screenLayout|screenSize|orientation"
       android:label="@string/app_name"/>
   <meta-data
       android:name="com.facebook.sdk.ApplicationId"
       android:value="@string/facebook_app_id" />
   ```
3. Ajoutez la permission Internet d’Android à votre fichier `AndroidManifest.xml`.

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. Créez une nouvelle instance de `FacebookAuthProvider`.

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private FacebookAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new FacebookAuthProvider(client);
     }

     // ...
   }
   ```

Si vous avez besoin d’aide supplémentaire pour configurer le Facebook SDK, veuillez consulter le [Guide de démarrage](https://developers.facebook.com/docs/android/getting-started).
