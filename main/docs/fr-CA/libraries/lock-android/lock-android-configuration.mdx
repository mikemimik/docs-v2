description: Modifier l’apparence et le comportement de Lock.Android
title: "Lock.Android : Configuration"
---

Vous pouvez utiliser les options ci-dessous pour configurer les fonctions et l’apparence de Lock pour Android.

Les options de configuration sont ajoutées au builder Lock selon le format suivant :

```kotlin lines
val lock = Lock.newBuilder(account, callback)
    // Options de configuration
    .closable(true)
    .allowSignUp(false)
    .setPrivacyURL('http://example.com/privacy')
    .setTermsURL('http://example.com/terms')
    // Fin des options de configuration
    .build(this)
```

<div id="general-options">
  ## Options générales
</div>

* **closable(boolean)** : Indique si `LockActivity` peut être fermé. Par défaut, il ne peut pas l’être.
* **allowedConnections(List)** : Filtre les connexions autorisées à partir de la liste configurée dans le Dashboard. Par défaut, si cette valeur est vide, toutes les connexions définies dans le Dashboard seront disponibles.
* **hideMainScreenTitle(boolean)** : Si cette option est définie sur `true`, l’en-tête de l’écran principal n’affichera pas le titre.

<div id="authentication-options">
  ## Options d’authentification
</div>

* **withAuthenticationParameters(Map&lt;String, Object&gt;)**: Définit des paramètres d’authentification supplémentaires à envoyer lors de chaque requête de connexion ou d’inscription. Le `scope` par défaut utilisé dans les requêtes d’authentification est `openid`. Si vous souhaitez en spécifier un autre, utilisez `withAuthenticationParameters` et ajoutez une valeur différente pour la clé `scope`.
* **withScope(String)**: Modifie le `scope` demandé lors d’une requête d’authentification.
* **withAudience(String)**: Modifie l’<Tooltip tip="Audience : identificateur unique de l’audience d’un token émis. Appelée aud dans un token, sa valeur contient l’ID d’une application (Client ID) pour un ID Token ou d’une API (API Identifier) pour un Access Token." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip> demandée lors d’une requête d’authentification.

<div id="database-options">
  ## Options de base de données
</div>

* **withUsernameStyle(int)**: Définit s’il faut demander uniquement le courriel, uniquement le nom d’utilisateur ou les deux. Les valeurs acceptées sont `USERNAME` et `EMAIL`. Par défaut, cette option respecte la configuration du Dashboard pour le paramètre `requires_username`.
* **loginAfterSignUp(boolean)**: Indique si, après un événement `SignUp`, l’utilisateur doit être connecté automatiquement. La valeur par défaut est `true`.
* **initialScreen(int)**: Permet de personnaliser le formulaire qui s’affichera en premier au lancement de Lock. Les valeurs acceptées sont `LOG_IN`, `SIGN_UP` et `FORGOT_PASSWORD`. Par défaut, `LOG_IN` est l’écran initial.
* **allowSignUp(boolean)**: Affiche le formulaire d’inscription si une connexion de base de données est configurée et qu’elle est autorisée dans le Dashboard. La valeur par défaut est `true`.
* **allowLogIn(boolean)**: Affiche le formulaire de connexion si une connexion de base de données est configurée. La valeur par défaut est `true`.
* **allowForgotPassword(boolean)**: Affiche le formulaire mot de passe oublié si une connexion de base de données est configurée et qu’elle est autorisée dans le Dashboard. La valeur par défaut est `true`.
* **allowShowPassword(boolean)**: Affiche un bouton pour basculer la visibilité de la saisie dans un champ Password. La valeur par défaut est `true`.
* **setDefaultDatabaseConnection(String)**: Définit la connexion de base de données par défaut. C’est utile si votre application a plusieurs connexions de base de données configurées.
* **enableEnterpriseWebAuthenticationFor(List)**: Filtre les connexions d’entreprise qui lanceront un flux d’authentification Web dans le navigateur lorsqu’on clique dessus. Seules les stratégies &#39;ad&#39;, &#39;adfs&#39; ou &#39;waad&#39; sont autorisées.
* **withSignUpFields(List)**: Affiche un deuxième écran avec des champs supplémentaires que l’utilisateur doit remplir après avoir saisi son nom d’utilisateur ou son courriel et son mot de passe dans l’écran d’inscription. Les valeurs soumises de cette façon peuvent être stockées dans le profil utilisateur à l’aide d’un attribut racine ou de l’attribut `user_metadata`. Pour plus d’info, consultez [Lock Android: Custom Fields at Signup](/docs/fr-CA/libraries/lock-android/lock-android-custom-fields-at-signup).
* **setPrivacyURL(String)**: Permet de personnaliser l’URL de la politique de confidentialité. La valeur par défaut est `https://auth0.com/privacy`.
* **setTermsURL(String)**: Permet de personnaliser l’URL des Conditions d’utilisation. La valeur par défaut est `https://auth0.com/terms`.
* **setSupportURL(String)**: Permet de définir une URL de soutien qui sera affichée si une erreur non récupérable survient dans Lock.
* **setMustAcceptTerms(boolean)**: Force l’utilisateur à accepter les Conditions d’utilisation et la politique avant de s’inscrire. La valeur par défaut est `false`.
* **useLabeledSubmitButton(boolean)**: Si cette option est définie à `true`, elle affichera un libellé du mode actuel (s’inscrire / se connecter) dans le bouton de soumission au lieu d’une icône. La valeur par défaut est `true`. Si l’option `hideMainScreenTitle` est définie à true, ce paramètre est ignoré et un libellé sera utilisé quand même.

<div id="oauth-options">
  ## Options OAuth
</div>

* **withAuthStyle(String, int)**: Personnalise l’apparence d’une connexion donnée (nom) à l’aide d’un style précis. Consultez [ce document sur les connexions OAuth personnalisées](/docs/fr-CA/libraries/lock-android/lock-android-custom-theming) pour plus d’informations.
* **withConnectionScope(String, String...)**: Permet de préciser des scopes supplémentaires pour un nom de connexion donné, qui seront demandés avec ceux définis dans les paramètres de la connexion dans le [Auth0 Dashboard](https://manage.auth0.com/#). Les scopes ne sont validés d’aucune manière et doivent être reconnus par le fournisseur d’authentification indiqué. Pour en obtenir la liste, consultez le [Auth0 Dashboard](https://manage.auth0.com/#) dans les paramètres de la connexion en question.
* **withScheme(String)**: Permet de modifier le schéma du `redirect_uri` envoyé lors de la requête authorize. Par défaut, le schéma est `https`. Si ce paramètre est modifié, l’espace réservé du manifeste pour `auth0Scheme` ainsi que les « Allowed Callbacks URLs » dans le tableau de bord de l’application Auth0 doivent aussi être mis à jour.

<div id="passwordless-options">
  ## Options Passwordless
</div>

* **useCode()**: Envoyer un code plutôt qu’un lien par courriel/SMS pour l’authentification <Tooltip tip="Passwordless : Forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip>.
* **useLink()**: Envoyer un lien plutôt qu’un code par courriel/SMS pour l’authentification Passwordless.
* **rememberLastLogin(boolean)**: Indique si l’adresse courriel ou le numéro de téléphone utilisé lors de la dernière authentification réussie sera enregistré afin de connecter automatiquement l’utilisateur la prochaine fois qu’une authentification Passwordless sera demandée.
