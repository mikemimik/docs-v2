Lorsque l’authentification est effectuée avec le scope `offline_access`, un <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se connecter de nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Refresh+Token">jeton d’actualisation</Tooltip> est renvoyé avec les informations d’identification. Cette valeur peut être utilisée pour demander un nouveau <Tooltip tip="Jeton d’accès : jeton utilisé pour accéder à une ressource protégée pour une durée limitée." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> et éviter de redemander ses informations d’identification à l’utilisateur.

Les jetons doivent être stockés dans un espace de stockage sécurisé après une authentification réussie. Gardez à l’esprit que les jetons d’actualisation **n’expirent jamais**. Pour demander un nouveau jeton, utilisez le `AuthenticationAPIClient` de A`uth0.Android`.

<div id="using-refresh-token">
  ## Utilisation du jeton d’actualisation
</div>

```kotlin lines
val refreshToken: String = // Récupérer le jeton d’actualisation depuis un stockage sécurisé
val account = Auth0(this)

val client = AuthenticationAPIClient(account)
client.renewAuth(refreshToken)
  .start(object: Callback<Credentials, AuthenticationException> {
  override fun onFailure(exception: AuthenticationException) {
       // Erreur
   }

   override fun onSuccess(credentials: Credentials) {
       // Utiliser les informations d’identification
   }
})
```
