Lorsque l’authentification est effectuée avec la portée `offline_access`, un <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un jeton d’accès renouvelé sans obliger les utilisateurs à se connecter de nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Refresh+Token">jeton d’actualisation</Tooltip> est renvoyé avec les identifiants. Cette valeur peut être utilisée pour demander un nouveau <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un jeton d’accès renouvelé sans obliger les utilisateurs à se connecter de nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> et éviter de redemander ses identifiants à l’utilisateur.

Les jetons doivent être stockés dans un espace de stockage sécurisé après une authentification réussie. Gardez à l’esprit que les jetons d’actualisation **n’expirent jamais**. Pour demander un nouveau jeton, utilisez `AuthenticationAPIClient` de A`uth0.Android`.

<div id="using-refresh-token">
  ## Utilisation du jeton d’actualisation
</div>

```kotlin lines
val refreshToken: String = // Récupérer le Refresh Token depuis un stockage sécurisé
val account = Auth0(this)

val client = AuthenticationAPIClient(account)
client.renewAuth(refreshToken)
  .start(object: Callback<Credentials, AuthenticationException> {
  override fun onFailure(exception: AuthenticationException) {
       // Error
   }

   override fun onSuccess(credentials: Credentials) {
       // Utiliser les credentials
   }
})
```
