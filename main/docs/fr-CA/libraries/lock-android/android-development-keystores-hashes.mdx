Lorsque vous créez un nouvel identifiant <Tooltip tip="OAuth 2.0 : cadre d’autorisation qui définit les protocoles et les workflows d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> pour de nombreuses connections, vous devez fournir le SHA-1 du certificat utilisé pour signer votre application. Lorsque vous terminez la configuration de votre application dans le [Auth0 Dashboard](https://manage.auth0.com/#), vous devez aussi fournir la valeur SHA-256.

Repérez le certificat utilisé pour signer votre application. Si vous n’en avez pas, vous pouvez le générer. Pour les applications de production, c’est ce que vous devriez faire.

Pendant le développement, vous pouvez signer votre application avec le certificat `android.keystore` par défaut, généré automatiquement lors de l’installation du SDK. Dans cet exemple, nous allons utiliser ce keystore par défaut. Pour générer les hachages de clés à l’aide de ce keystore, suivez les exemples ci-dessous.

<div id="generating-your-key-hashes">
  ## Génération de vos hachages de clé
</div>

**Sous Windows :**

`keytool -list -v -keystore "%USERPROFILE%\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android`

**Sous Linux / macOS :**

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

<div id="using-your-key-hashes">
  ## Utiliser vos hachages de clé
</div>

Une fois vos hachages de clé générés, copiez la valeur SHA256 obtenue, puis allez dans les paramètres de votre application dans le [Auth0 Dashboard](https://manage.auth0.com/#/applications). Cliquez sur « Show Advanced Settings », puis, dans l’onglet « Device Settings », sous « Android », indiquez le nom du package de votre application dans le champ « App Package Name » et la valeur SHA256 copiée dans le champ « Key Hashes ». N’oubliez pas d’enregistrer les modifications.

Si vous n’ajoutez pas l’URL de rappel aux paramètres de l’application ni les hachages de clé aux paramètres de l’appareil de l’application, le serveur Auth0 ne renverra pas le résultat de l’appel à votre application.
