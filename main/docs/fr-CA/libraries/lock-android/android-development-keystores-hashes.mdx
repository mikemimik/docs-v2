Lors de la création d’un nouvel identifiant <Tooltip tip="OAuth 2.0 : Cadre d’autorisation qui définit les protocoles et les flux d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> pour de nombreuses connexions, vous devrez fournir le SHA-1 du certificat utilisé pour signer votre application. Lorsque vous terminez la configuration de votre application dans le [Auth0 Dashboard](https://manage.auth0.com/#), vous devrez aussi fournir la valeur SHA-256.

Repérez le certificat utilisé pour signer votre application. Si vous n’en avez pas, vous pouvez en générer un. Pour les applications de production, il est recommandé de le faire.

Pendant le développement, vous pouvez signer votre application avec le certificat `android.keystore` par défaut, généré automatiquement lors de l’installation du SDK. Dans cet exemple, nous allons utiliser ce keystore par défaut. Pour générer les hachages de clé à l’aide de ce keystore, suivez les exemples ci-dessous.

<div id="generating-your-key-hashes">
  ## Génération de vos hachages de clé
</div>

**Sous Windows :**

`keytool -list -v -keystore "%USERPROFILE%\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android`

**Sous Linux / macOS :**

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

<div id="using-your-key-hashes">
  ## Utilisation de vos hachages de clés
</div>

Une fois vos hachages de clés générés, copiez la valeur SHA256 obtenue, puis accédez aux paramètres de votre application dans le [Auth0 Dashboard](https://manage.auth0.com/#/applications). Cliquez sur &quot;Show Advanced Settings&quot; et, dans l’onglet &quot;Device Settings&quot;, sous &quot;Android&quot;, saisissez le nom du package de votre application dans le champ &quot;App Package Name&quot;, puis la valeur SHA256 que vous avez copiée dans le champ &quot;Key Hashes&quot;. N’oubliez pas d’enregistrer les modifications.

Si vous n’ajoutez ni l’URL de callback aux paramètres de l’application ni les hachages de clé aux paramètres de l’appareil de l’application, le serveur Auth0 ne renverra pas le résultat de la requête à votre application.
