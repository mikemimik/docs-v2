Lorsque vous créez un nouvel identifiant <Tooltip tip="OAuth 2.0 : Cadre d’autorisation qui définit les protocoles et les processus d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> pour plusieurs connexions, vous devez fournir l’empreinte SHA-1 du certificat utilisé pour signer votre application. Lorsque vous terminez la configuration de votre application dans le [Auth0 Dashboard](https://manage.auth0.com/#), vous devez aussi fournir la valeur SHA-256.

Repérez le certificat que vous utilisez pour signer votre application. Si vous n’en avez pas, vous pouvez en générer un. Pour les applications de production, c’est ce que vous devriez faire.

Pendant le développement, vous pouvez signer votre application avec le certificat `android.keystore` par défaut, généré automatiquement lors de l’installation du SDK. Dans cet exemple, nous allons utiliser ce keystore par défaut. Pour générer les hachages de clé à l’aide de ce keystore, suivez les exemples ci-dessous.

<div id="generating-your-key-hashes">
  ## Générer vos hachages de clé
</div>

**Sous Windows :**

`keytool -list -v -keystore "%USERPROFILE%\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android`

**Sous Linux / macOS :**

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

<div id="using-your-key-hashes">
  ## Utilisation de vos hachages de clé
</div>

Une fois vos hachages de clé générés, copiez la valeur SHA256 obtenue et accédez aux paramètres de votre application dans le [Auth0 Dashboard](https://manage.auth0.com/#/applications). Cliquez sur &quot;Show Advanced Settings&quot;, puis, dans l’onglet &quot;Device Settings&quot;, sous &quot;Android&quot;, saisissez le nom de package de votre application dans le champ &quot;App Package Name&quot;, et la valeur SHA256 copiée dans le champ &quot;Key Hashes&quot;. N’oubliez pas d’enregistrer les modifications.

Si vous n’ajoutez pas l’URL de rappel aux paramètres de l’application ni les hachages de clé aux paramètres de l’appareil de l’application, le serveur Auth0 ne renverra pas le résultat à votre application.
