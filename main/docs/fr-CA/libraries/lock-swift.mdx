Ce guide de référence vous montrera comment implémenter l’interface utilisateur de Lock et vous fournira tous les détails nécessaires pour configurer et personnaliser Lock afin de l’utiliser comme UI pour vos besoins d’authentification. Toutefois, si vous souhaitez en faire plus avec Auth0 et Swift, par exemple enregistrer, appeler et actualiser des <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Tokens">jetons d’accès</Tooltip>, obtenir des renseignements sur le profil de l’utilisateur, et plus encore, consultez le [SDK Auth0.swift](/docs/fr-CA/libraries/auth0-swift). Vous pouvez aussi consulter le [Quickstart Swift](/docs/fr-CA/quickstart/native/ios-swift) pour suivre des exemples complets et voir les options, tant pour utiliser Lock comme interface que pour utiliser une interface personnalisée.

Consultez le [dépôt Lock.swift](https://github.com/auth0/Lock.swift) sur GitHub.

<div id="cocoapods">
  ### Cocoapods
</div>

Si vous utilisez [Cocoapods](https://cocoapods.org), ajoutez cette ligne à votre `Podfile` :

`pod 'Lock', '~> 2.0'`

Exécutez ensuite `pod install`.
Pour en savoir plus sur Cocoapods, consultez [la documentation officielle](https://guides.cocoapods.org/using/getting-started.html).

<div id="carthage">
  ### Carthage
</div>

Si vous utilisez [Carthage](https://github.com/Carthage/Carthage), ajoutez la ligne suivante à votre `Cartfile` :

`github "auth0/Lock.swift" ~> 2.0`
Ensuite, exécutez `carthage bootstrap`.
Pour en savoir plus sur l’utilisation de Carthage, consultez [la documentation officielle](https://github.com/Carthage/Carthage#if-youre-building-for-ios-tvos-or-watchos).

Si vous utilisez Swift Package Manager, ouvrez l’élément de menu suivant dans Xcode :

**File &gt; Swift Packages &gt; Add Package Dependency...**

`https://github.com/auth0/Lock.swift.git`

<div id="integrate-with-your-application">
  ### Intégrer votre application
</div>

Lock doit être avisé lorsque l’application reçoit une demande d’ouverture d’une URL. Vous pouvez le faire dans le fichier `AppDelegate`.

```swift lines
func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey : Any] = [:]) -> Bool {
  return Lock.resumeAuth(url, options: options)
}
```

<div id="import-lock">
  ### Importer Lock
</div>

Importez **Lock** là où vous en aurez besoin

`import lock`

<div id="auth0-credentials">
  ### Identifiants Auth0
</div>

Pour utiliser Lock, vous devez fournir votre <Tooltip tip="Client ID : valeur d’identification attribuée par Auth0 à votre ressource enregistrée." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Id">Client Id</Tooltip> et votre domaine Auth0, que vous trouverez dans votre [Auth0 Dashboard](https://manage.auth0.com/#), dans les paramètres de votre application.

Dans le bundle de votre application, vous pouvez ajouter un fichier `plist` nommé `Auth0.plist` qui contiendra vos identifiants au format suivant.

<div id="implementation-of-lock-classic">
  ## Implémentation de Lock Classic
</div>

Lock Classic gère l’authentification avec les connexions Database, Social et Enterprise.

<div id="oidc-conformant-mode">
  ### Mode conforme à OIDC
</div>

Il est fortement recommandé d’utiliser ce SDK en mode conforme à OIDC. Lorsque ce mode est activé, il oblige le SDK à utiliser le pipeline d’authentification actuel d’Auth0 et l’empêche d’accéder aux points de terminaison Legacy. Par défaut, cette valeur est `false`

```swift lines
.withOptions {
    $0.oidcConformant = true
}
```

Pour afficher Lock, ajoutez l’extrait de code suivant dans votre UIViewController.

```swift lines
Lock
    .classic()
    // withConnections, withOptions, withStyle, et ainsi de suite
    .withOptions {
      $0.oidcConformant = true
      $0.scope = "openid profile"
    }
    .onAuth { credentials in
      // Sauvegardons la valeur de credentials.accessToken
    }
    .present(from: self)
```

<div id="use-auth0swift-library-to-access-user-profile">
  ## Utiliser la bibliothèque Auth0.Swift pour accéder au profil utilisateur
</div>

Pour accéder aux renseignements du profil utilisateur, vous devrez utiliser la bibliothèque `Auth0.Swift` :

```swift lines
Auth0
   .authentication()
   .userInfo(withAccessToken: accessToken)
   .start { result in
       switch result {
       case .success(let profile):
           print("User Profile: \(profile)")
       case .failure(let error):
           print("Failed with \(error)")
       }
   }
```

Consultez la [documentation de la bibliothèque Auth0.Swift](/docs/fr-CA/libraries/auth0-swift) pour en savoir plus sur les façons de l’utiliser.

<div id="specify-connections">
  ## Spécifier les connexions
</div>

Lock chargera automatiquement les connexions configurées pour votre application. Si vous souhaitez remplacer le comportement par défaut, vous pouvez indiquer manuellement quelles connexions afficher aux utilisateurs comme options d’authentification. Pour ce faire, appelez la méthode et fournissez une closure qui permet de préciser la ou les connexions.

Ajout d’une connexion de base de données :

```swift lines
.withConnections {
    connections.database(name: "Username-Password-Authentication", requiresUsername: true)
}
```

Ajouter plusieurs connexions sociales :

```swift lines
.withConnections {
    connections.social(name: "facebook", style: .Facebook)
    connections.social(name: "google-oauth2", style: .Google)
}
```

<div id="styling-and-customization">
  ## Style et personnalisation
</div>

Lock offre de nombreuses options de style pour vous aider à appliquer l’identité visuelle de votre marque à Lock à l’aide de `withStyle`. Par exemple, vous pouvez modifier la couleur principale et le texte de l’en-tête de votre widget Lock :

<div id="customize-your-title-logo-and-primary-color">
  ### Personnalisez votre titre, votre logo et votre couleur principale
</div>

```swift lines
.withStyle {
  $0.title = "Company LLC"
  $0.logo = LazyImage(named: "company_logo")
  $0.primaryColor = UIColor(red: 0.6784, green: 0.5412, blue: 0.7333, alpha: 1.0)
}
```

Vous trouverez l’ensemble des options de style permettant de modifier l’apparence de Lock dans votre application dans le [guide de personnalisation](/docs/fr-CA/libraries/lock-swift/lock-swift-customization).

<div id="configuration-options">
  ## Options de configuration
</div>

Lock offre de nombreuses options de configuration. Voici un exemple de configuration de Lock qui permet de le fermer, de le limiter aux seuls noms d’utilisateur (et non aux adresses courriel) et d’afficher uniquement les écrans Login et de réinitialisation du mot de passe.

```swift lines
Lock
  .classic()
  .withOptions {
    $0.closable = true
    $0.usernameStyle = [.Username]
    $0.allow = [.Login, .ResetPassword]
  }
```

Vous trouverez dans le [guide de configuration](/docs/fr-CA/libraries/lock-swift/lock-swift-configuration-options) toutes les options de configuration du comportement qui vous permettent de modifier le fonctionnement de Lock dans votre application.

<div id="password-manager-support">
  ## Prise en charge des gestionnaires de mots de passe
</div>

Par défaut, la prise en charge de [1Password](https://1password.com/) est activée pour les connexions de base de données. Toutefois, pour que l’option s’affiche sur les écrans de connexion et d’inscription, l’utilisateur doit avoir installé l’application 1Password. Vous pouvez désactiver la prise en charge de 1Password à l’aide de la propriété enabled de passwordManager.

```swift lines
.withOptions {
    $0.passwordManager.enabled = false
}
```

Par défaut, `appIdentifier` correspondra à l’identifiant du bundle de l’application et `displayName` au nom d’affichage de l’application. Vous pouvez les personnaliser comme suit :

```swift lines
.withOptions {
    $0.passwordManager.appIdentifier = "www.myapp.com"
    $0.passwordManager.displayName = "My App"
}
```

Vous devrez ajouter ce qui suit au fichier `info.plist` de votre application :

```xml lines
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>org-appextension-feature-password-management</string>
</array>
```
