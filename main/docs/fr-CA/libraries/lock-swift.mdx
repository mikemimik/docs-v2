Ce guide de référence vous montrera comment implémenter l’interface utilisateur de Lock et vous donnera les détails nécessaires pour configurer et personnaliser Lock afin de l’utiliser comme UI pour vos besoins en authentication. Toutefois, si vous souhaitez en apprendre davantage sur ce que vous pouvez faire avec Auth0 et Swift, par exemple comment enregistrer, utiliser et actualiser des <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Tokens">jetons d’accès</Tooltip>, obtenir des renseignements sur le profil utilisateur, et plus encore, consultez le [Auth0.swift SDK](/docs/fr-CA/libraries/auth0-swift). Vous pouvez aussi jeter un coup d’œil au [Quickstart Swift](/docs/fr-CA/quickstart/native/ios-swift) pour parcourir des exemples complets et voir les options, tant pour utiliser Lock comme interface que pour utiliser une interface personnalisée.

Consultez le [dépôt Lock.swift](https://github.com/auth0/Lock.swift) sur GitHub.

<div id="cocoapods">
  ### Cocoapods
</div>

Si vous utilisez [Cocoapods](https://cocoapods.org), ajoutez cette ligne à votre `Podfile` :

`pod 'Lock', '~> 2.0'`

Exécutez ensuite `pod install`.
Pour en savoir plus sur Cocoapods, consultez [la documentation officielle](https://guides.cocoapods.org/using/getting-started.html).

<div id="carthage">
  ### Carthage
</div>

Si vous utilisez [Carthage](https://github.com/Carthage/Carthage), ajoutez la ligne suivante à votre `Cartfile` :

`github "auth0/Lock.swift" ~> 2.0`
Exécutez ensuite `carthage bootstrap`.
Pour en savoir davantage sur l’utilisation de Carthage, consultez [la documentation officielle](https://github.com/Carthage/Carthage#if-youre-building-for-ios-tvos-or-watchos).

<div id="integrate-with-your-application">
  ### Intégrer votre application
</div>

Lock doit être informé lorsque l’application reçoit une demande d’ouverture d’URL. Vous pouvez le faire dans le fichier `AppDelegate`.

```swift lines
func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey : Any] = [:]) -> Bool {
  return Lock.resumeAuth(url, options: options)
}
```

<div id="import-lock">
  ### Importer Lock
</div>

Importez **Lock** là où vous en aurez besoin

`import lock`

<div id="implementation-of-lock-classic">
  ## Mise en œuvre de Lock Classic
</div>

Lock Classic prend en charge l’authentification au moyen des connexions Database, Social et Enterprise.

<div id="oidc-conformant-mode">
  ### Mode conforme à OIDC
</div>

Il est fortement recommandé d’utiliser ce SDK en mode conforme à OIDC. Lorsque ce mode est activé, il oblige le SDK à utiliser le pipeline d’authentification actuel d’Auth0 et l’empêche d’accéder aux points de terminaison hérités. Par défaut, cette valeur est `false`

```swift lines
.withOptions {
    $0.oidcConformant = true
}
```

Pour afficher Lock, ajoutez l’extrait de code suivant à votre UIViewController.

```swift lines
Lock
    .classic()
    // withConnections, withOptions, withStyle, et ainsi de suite
    .withOptions {
      $0.oidcConformant = true
      $0.scope = "openid profile"
    }
    .onAuth { credentials in
      // Sauvegardons la valeur de credentials.accessToken
    }
    .present(from: self)
```

<div id="use-auth0swift-library-to-access-user-profile">
  ## Utiliser la bibliothèque Auth0.Swift pour accéder au profil utilisateur
</div>

Pour accéder aux renseignements du profil utilisateur, vous devrez utiliser la bibliothèque `Auth0.Swift` :

```swift lines
Auth0
   .authentication()
   .userInfo(withAccessToken: accessToken)
   .start { result in
       switch result {
       case .success(let profile):
           print("User Profile: \(profile)")
       case .failure(let error):
           print("Failed with \(error)")
       }
   }
```

Consultez la [documentation de la bibliothèque Auth0.Swift](/docs/fr-CA/libraries/auth0-swift) pour en savoir plus sur son utilisation.

<div id="specify-connections">
  ## Spécifier les connexions
</div>

Lock chargera automatiquement les connexions configurées pour votre application. Si vous souhaitez déroger au comportement par défaut, vous pouvez indiquer manuellement quelles connexions il doit présenter aux utilisateurs comme options d’authentification. Pour ce faire, appelez la méthode et fournissez une closure permettant de préciser la ou les connexions.

Ajouter une connexion de base de données :

```swift lines
.withConnections {
    connections.database(name: "Username-Password-Authentication", requiresUsername: true)
}
```

Ajout de plusieurs connexions sociales :

```swift lines
.withConnections {
    connections.social(name: "facebook", style: .Facebook)
    connections.social(name: "google-oauth2", style: .Google)
}
```

<div id="styling-and-customization">
  ## Style et personnalisation
</div>

Lock offre de nombreuses options de style pour vous aider à appliquer votre propre image de marque à Lock grâce à `withStyle`. Par exemple, en modifiant la couleur primaire et le texte de l’en-tête de votre widget Lock :

<div id="customize-your-title-logo-and-primary-color">
  ### Personnalisez le titre, le logo et la couleur principale
</div>

```swift lines
.withStyle {
  $0.title = "Company LLC"
  $0.logo = LazyImage(named: "company_logo")
  $0.primaryColor = UIColor(red: 0.6784, green: 0.5412, blue: 0.7333, alpha: 1.0)
}
```

Vous pouvez consulter toutes les options de style permettant de modifier l’apparence de Lock pour votre application dans le [guide de personnalisation](/docs/fr-CA/libraries/lock-swift/lock-swift-customization).

<div id="configuration-options">
  ## Options de configuration
</div>

Il existe de nombreuses options pour configurer le comportement de Lock. Vous trouverez ci-dessous un exemple de configuration de Lock permettant de le fermer, de le limiter aux seuls noms d’utilisateur (et non aux adresses courriel) et d’afficher uniquement les écrans Login et Réinitialiser le mot de passe.

```swift lines
Lock
  .classic()
  .withOptions {
    $0.closable = true
    $0.usernameStyle = [.Username]
    $0.allow = [.Login, .ResetPassword]
  }
```

Vous pouvez consulter l’ensemble des options de configuration du comportement qui permettent de modifier le fonctionnement de Lock pour votre application dans le [Guide de configuration](/docs/fr-CA/libraries/lock-swift/lock-swift-configuration-options).

<div id="password-manager-support">
  ## Compatibilité avec les gestionnaires de mots de passe
</div>

Par défaut, la compatibilité avec [1Password](https://1password.com/) est activée pour les connexions de base de données. Le soutien de 1Password exige tout de même que l’utilisateur ait installé l’application 1Password pour que l’option soit visible dans les écrans de connexion et d’inscription. Vous pouvez désactiver le soutien de 1Password à l’aide de la propriété enabled de passwordManager.

```swift lines
.withOptions {
    $0.passwordManager.enabled = false
}
```

Par défaut, `appIdentifier` correspondra à l’identifiant du bundle de l’application et `displayName`, au nom d’affichage de l’application. Vous pouvez les personnaliser comme suit :

```swift lines
.withOptions {
    $0.passwordManager.appIdentifier = "www.myapp.com"
    $0.passwordManager.displayName = "My App"
}
```

Vous devrez ajouter les éléments suivants au fichier `info.plist` de votre application :

```xml lines
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>org-appextension-feature-password-management</string>
</array>
```
