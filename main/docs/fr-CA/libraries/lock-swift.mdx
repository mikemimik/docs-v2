Ce guide de référence vous montrera comment mettre en œuvre l’interface utilisateur de Lock et vous donnera des détails sur la façon de configurer et de personnaliser Lock afin de l’utiliser comme interface utilisateur pour vos besoins d’authentification. Cependant, si vous souhaitez en apprendre davantage sur ce que vous pouvez faire avec Auth0 et Swift, notamment comment enregistrer, utiliser dans des requêtes et actualiser des <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Tokens">jetons d’accès</Tooltip>, obtenir des renseignements sur le profil de l’utilisateur, et plus encore, consultez le [Auth0.swift SDK](/docs/fr-CA/libraries/auth0-swift). Ou jetez un coup d’œil au [Quickstart Swift](/docs/fr-CA/quickstart/native/ios-swift) pour suivre des exemples complets et voir les options, autant pour utiliser Lock comme interface que pour utiliser une interface personnalisée.

Consultez le [dépôt Lock.swift](https://github.com/auth0/Lock.swift) sur GitHub.

<div id="cocoapods">
  ### Cocoapods
</div>

Si vous utilisez [Cocoapods](https://cocoapods.org), ajoutez cette ligne à votre `Podfile` :

`pod 'Lock', '~> 2.0'`

Ensuite, exécutez `pod install`.
Pour en savoir plus sur Cocoapods, consultez [la documentation officielle](https://guides.cocoapods.org/using/getting-started.html).

<div id="carthage">
  ### Carthage
</div>

Si vous utilisez [Carthage](https://github.com/Carthage/Carthage), ajoutez la ligne suivante à votre `Cartfile` :

`github "auth0/Lock.swift" ~> 2.0`
Exécutez ensuite `carthage bootstrap`.
Pour plus d’information sur l’utilisation de Carthage, consultez [la documentation officielle](https://github.com/Carthage/Carthage#if-youre-building-for-ios-tvos-or-watchos).

Si vous utilisez Swift Package Manager, ouvrez l’élément de menu suivant dans Xcode :

`https://github.com/auth0/Lock.swift.git`

Ensuite, cliquez sur **Next** et suivez les étapes restantes.

<div id="integrate-with-your-application">
  ### Intégrer votre application
</div>

Lock doit être informé lorsqu’on demande à l’application d’ouvrir une URL. Vous pouvez le faire dans le fichier `AppDelegate`.

```swift lines
func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey : Any] = [:]) -> Bool {
  return Lock.resumeAuth(url, options: options)
}
```

<div id="import-lock">
  ### Importation de Lock
</div>

Importez **Lock** partout où vous en avez besoin

`import lock`

<div id="auth0-credentials">
  ### Identifiants Auth0
</div>

Pour utiliser Lock, vous devez fournir votre Auth0 <Tooltip tip="Client ID : valeur d’identification fournie par Auth0 à votre ressource enregistrée." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Id">Client ID</Tooltip> et votre Domaine, que vous trouverez dans l’[Auth0 Dashboard](https://manage.auth0.com/#), dans les paramètres de votre application.

Dans le bundle de votre application, vous pouvez ajouter un fichier `plist` nommé `Auth0.plist` contenant vos identifiants, selon le format suivant.

export const codeExample = `<?xml version="1.0" encoding="UTF-8"?>

<dict>
  <key>ClientId</key>
  <string>{yourClientId}</string>
  <key>Domain</key>
  <string>{yourDomain}</string>

</plist>`;

<div id="implementation-of-lock-classic">
  ## Mise en œuvre de Lock Classic
</div>

Lock Classic gère l’authentification au moyen de connexions Database, Social et Enterprise.

<div id="oidc-conformant-mode">
  ### Mode conforme à OIDC
</div>

Il est fortement recommandé d’utiliser ce SDK en mode conforme à OIDC. Lorsque ce mode est activé, il oblige le SDK à utiliser le pipeline d’authentification actuel d’Auth0 et l’empêche d’accéder à des points de terminaison hérités. Par défaut, cette valeur est `false`

```swift lines
.withOptions {
    $0.oidcConformant = true
}
```

Pour afficher Lock, ajoutez l’extrait de code suivant dans votre UIViewController.

```swift lines
Lock
    .classic()
    // withConnections, withOptions, withStyle, et ainsi de suite
    .withOptions {
      $0.oidcConformant = true
      $0.scope = "openid profile"
    }
    .onAuth { credentials in
      // Sauvegardons la valeur de credentials.accessToken
    }
    .present(from: self)
```

<div id="use-auth0swift-library-to-access-user-profile">
  ## Utiliser la bibliothèque Auth0.Swift pour accéder au profil utilisateur
</div>

Pour accéder aux renseignements du profil utilisateur, vous devrez utiliser la bibliothèque `Auth0.Swift` :

```swift lines
Auth0
   .authentication()
   .userInfo(withAccessToken: accessToken)
   .start { result in
       switch result {
       case .success(let profile):
           print("User Profile: \(profile)")
       case .failure(let error):
           print("Failed with \(error)")
       }
   }
```

Consultez la [documentation de la bibliothèque Auth0.Swift](/docs/fr-CA/libraries/auth0-swift) pour en savoir plus sur son utilisation.

<div id="specify-connections">
  ## Spécifier les connexions
</div>

Lock chargera automatiquement les connexions configurées pour votre application. Si vous souhaitez modifier le comportement par défaut, vous pouvez préciser manuellement quelles connexions il doit présenter aux utilisateurs comme options d’authentification. Pour ce faire, appelez la méthode et fournissez une fonction de fermeture permettant de préciser la ou les connexions.

Ajouter une connexion de base de données :

```swift lines
.withConnections {
    connections.database(name: "Username-Password-Authentication", requiresUsername: true)
}
```

Ajout de plusieurs connexions sociales :

```swift lines
.withConnections {
    connections.social(name: "facebook", style: .Facebook)
    connections.social(name: "google-oauth2", style: .Google)
}
```

<div id="configuration-options">
  ## Options de configuration
</div>

Il existe de nombreuses options pour configurer le comportement de Lock. Voici un exemple de Lock configuré de façon à pouvoir être fermé, à être limité aux seuls noms d’utilisateur (et non aux adresses courriel), et à n’afficher que les écrans Connexion et Réinitialiser le mot de passe.

```swift lines
Lock
  .classic()
  .withOptions {
    $0.closable = true
    $0.usernameStyle = [.Username]
    $0.allow = [.Login, .ResetPassword]
  }
```

Vous pouvez consulter l’ensemble des options de configuration du comportement qui permettent de modifier le fonctionnement de Lock pour votre application dans le [guide de configuration](/docs/fr-CA/libraries/lock-swift/lock-swift-configuration-options).

<div id="password-manager-support">
  ## Prise en charge du gestionnaire de mots de passe
</div>

Par défaut, la prise en charge de [1Password](https://1password.com/) est activée pour les connexions de base de données. Pour que l’option s’affiche dans les écrans de connexion et d’inscription, l’utilisateur doit tout de même avoir installé l’application 1Password. Vous pouvez désactiver la prise en charge de 1Password à l’aide de la propriété enabled de passwordManager.

```swift lines
.withOptions {
    $0.passwordManager.enabled = false
}
```

Par défaut, `appIdentifier` correspondra au Bundle Identifier de l’application et `displayName` à son nom d’affichage. Vous pouvez les personnaliser comme suit :

```swift lines
.withOptions {
    $0.passwordManager.appIdentifier = "www.myapp.com"
    $0.passwordManager.displayName = "My App"
}
```

Vous devrez ajouter les éléments suivants au fichier `info.plist` de votre application :

```xml lines
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>org-appextension-feature-password-management</string>
</array>
```
