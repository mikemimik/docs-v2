Ce guide de référence vous montrera comment implémenter l’interface utilisateur de Lock et vous donnera des détails sur la façon de configurer et de personnaliser Lock afin de l’utiliser comme interface utilisateur pour répondre à vos besoins en matière d’authentification. Toutefois, si vous souhaitez en apprendre davantage sur ce que vous pouvez faire avec Auth0 et Swift, notamment comment enregistrer, utiliser et actualiser des <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Tokens">jetons d’accès</Tooltip>, obtenir des renseignements sur le profil utilisateur et plus encore, consultez le [SDK Auth0.swift](/docs/fr-CA/libraries/auth0-swift). Vous pouvez aussi jeter un coup d’œil au [Quickstart Swift](/docs/fr-CA/quickstart/native/ios-swift) pour parcourir des exemples complets et voir les options, autant pour utiliser Lock comme interface que pour utiliser une interface personnalisée.

Consultez le [dépôt Lock.swift](https://github.com/auth0/Lock.swift) sur GitHub.

  ## Installation

<div id="cocoapods">
  ### Cocoapods
</div>

Si vous utilisez [Cocoapods](https://cocoapods.org), ajoutez cette ligne à votre `Podfile` :

`pod 'Lock', '~> 2.0'`

Exécutez ensuite `pod install`.
Pour en savoir plus sur Cocoapods, consultez [sa documentation officielle](https://guides.cocoapods.org/using/getting-started.html).

<div id="carthage">
  ### Carthage
</div>

Si vous utilisez [Carthage](https://github.com/Carthage/Carthage), ajoutez la ligne suivante à votre `Cartfile` :

`github "auth0/Lock.swift" ~> 2.0`
Exécutez ensuite `carthage bootstrap`.
Pour en savoir plus sur l’utilisation de Carthage, consultez [la documentation officielle](https://github.com/Carthage/Carthage#if-youre-building-for-ios-tvos-or-watchos).

<div id="spm">
  ### SPM
</div>

Si vous utilisez Swift Package Manager, ouvrez l’élément de menu suivant dans Xcode :

**File &gt; Swift Packages &gt; Add Package Dependency...**

Dans la fenêtre **Choose Package Repository**, ajoutez cette URL :

`https://github.com/auth0/Lock.swift.git`

Cliquez ensuite sur **Next** et suivez les étapes restantes.

<div id="integrate-with-your-application">
  ### Intégrez votre application
</div>

Lock doit être avisé lorsque l’application reçoit une demande d’ouverture d’URL. Vous pouvez le faire dans le fichier `AppDelegate`.

```swift lines
func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey : Any] = [:]) -> Bool {
  return Lock.resumeAuth(url, options: options)
}
```

<div id="import-lock">
  ### Importer Lock
</div>

Importez **Lock** là où vous en aurez besoin

`import lock`

<div id="auth0-credentials">
  ### Identifiants Auth0
</div>

Pour utiliser Lock, vous devez fournir votre Auth0 <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Id">ID client</Tooltip> et votre domaine, que vous trouverez dans l’[Auth0 Dashboard](https://manage.auth0.com/#), sous les paramètres de votre application.

Dans le bundle de votre application, vous pouvez ajouter un fichier `plist` nommé `Auth0.plist` qui inclura vos identifiants au format suivant.

<div id="implementation-of-lock-classic">
  ## Mise en œuvre de Lock Classic
</div>

Lock Classic gère l’authentification avec les connexions Database, Social et Enterprise.

<div id="oidc-conformant-mode">
  ### Mode conforme à OIDC
</div>

Il est fortement recommandé d’utiliser ce SDK en mode conforme à OIDC. Lorsque ce mode est activé, il force le SDK à utiliser le pipeline d’authentification actuel d’Auth0 et l’empêche d’accéder aux points de terminaison hérités. Par défaut, cette valeur est `false`

```swift lines
.withOptions {
    $0.oidcConformant = true
}
```

Pour afficher Lock, ajoutez le code suivant dans votre UIViewController.

```swift lines
Lock
    .classic()
    // withConnections, withOptions, withStyle, et ainsi de suite
    .withOptions {
      $0.oidcConformant = true
      $0.scope = "openid profile"
    }
    .onAuth { credentials in
      // Sauvegardons la valeur de credentials.accessToken
    }
    .present(from: self)
```

<div id="use-auth0swift-library-to-access-user-profile">
  ## Utiliser la bibliothèque Auth0.Swift pour accéder au profil utilisateur
</div>

Pour accéder aux renseignements du profil utilisateur, vous devrez utiliser la bibliothèque `Auth0.Swift` :

```swift lines
Auth0
   .authentication()
   .userInfo(withAccessToken: accessToken)
   .start { result in
       switch result {
       case .success(let profile):
           print("User Profile: \(profile)")
       case .failure(let error):
           print("Failed with \(error)")
       }
   }
```

Consultez la [documentation de la bibliothèque Auth0.Swift](/docs/fr-CA/libraries/auth0-swift) pour en savoir plus sur son utilisation.

<div id="specify-connections">
  ## Spécifier les connexions
</div>

Lock chargera automatiquement les connexions configurées pour votre application. Si vous souhaitez remplacer le comportement par défaut, vous pouvez préciser manuellement les connexions qu’il doit afficher aux utilisateurs comme options d’authentification. Pour ce faire, appelez la méthode et fournissez une closure permettant de préciser la ou les connexions.

Ajout d’une connexion de base de données :

```swift lines
.withConnections {
    connections.database(name: "Username-Password-Authentication", requiresUsername: true)
}
```

Ajout de plusieurs connexions sociales :

```swift lines
.withConnections {
    connections.social(name: "facebook", style: .Facebook)
    connections.social(name: "google-oauth2", style: .Google)
}
```

  ## Style et personnalisation
</div>

Lock offre de nombreuses options de style pour vous aider à adapter Lock à l’identité de votre marque à l’aide de `withStyle`. Par exemple, en modifiant la couleur principale et le texte d’en-tête de votre widget Lock :

<div id="configuration-options">
  ## Options de configuration
</div>

Il existe de nombreuses options pour configurer le comportement de Lock. Voici ci-dessous un exemple de Lock configuré pour pouvoir être fermé, être limité aux seuls noms d’utilisateur (et non aux adresses courriel) et n’afficher que les écrans Login et Reset Password.

```swift lines
Lock
  .classic()
  .withOptions {
    $0.closable = true
    $0.usernameStyle = [.Username]
    $0.allow = [.Login, .ResetPassword]
  }
```

Vous pouvez consulter l’ensemble des options de configuration du comportement permettant de modifier le fonctionnement de Lock dans votre application dans le [Guide de configuration](/docs/fr-CA/libraries/lock-swift/lock-swift-configuration-options).

<div id="password-manager-support">
  ## Prise en charge des gestionnaires de mots de passe
</div>

Par défaut, la prise en charge de [1Password](https://1password.com/) est activée pour les connexions de base de données. Pour que l’option s’affiche dans les écrans de connexion et d’inscription, l’utilisateur doit néanmoins avoir installé l’application 1Password. Vous pouvez désactiver la prise en charge de 1Password à l’aide de la propriété enabled de passwordManager.

```swift lines
.withOptions {
    $0.passwordManager.enabled = false
}
```

Par défaut, `appIdentifier` correspondra au Bundle Identifier de l’application et `displayName`, à son nom d’affichage. Vous pouvez les personnaliser comme suit :

```swift lines
.withOptions {
    $0.passwordManager.appIdentifier = "www.myapp.com"
    $0.passwordManager.displayName = "My App"
}
```

Vous devrez ajouter ce qui suit au fichier `info.plist` de votre application :

```xml lines
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>org-appextension-feature-password-management</string>
</array>
```
