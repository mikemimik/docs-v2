Auth0.Android est une bibliothèque côté client que vous pouvez utiliser dans votre application Android pour authentifier les utilisateurs et accéder aux [API Auth0](/docs/fr-CA/api).

Consultez le [dépôt Auth0.Android](https://github.com/auth0/Auth0.Android) sur GitHub.

<div id="requirements">
  ## Exigences
</div>

Android API version 21 ou une version ultérieure est requise.

<div id="installation">
  ## Installation
</div>

Ajoutez le SDK à votre projet. La bibliothèque enverra des requêtes aux API Authentication et Management d’Auth0.

<div id="add-auth0-to-gradle">
  ### Ajouter Auth0 à Gradle
</div>

Dans la section des dépendances du fichier `build.gradle` de votre application, ajoutez ce qui suit :

```kotlin lines
dependencies {
  // Ajouter le SDK Android Auth0
  implementation 'com.auth0.android:auth0:2.+'
}
```

Si Android Studio signale un problème avec le signe `+`, ou si vous préférez utiliser une version précise, consultez la plus récente dans [Maven](https://central.sonatype.com/artifact/com.auth0.android/auth0).

<Card title="Synchroniser le projet avec Gradle">
  N’oubliez pas de synchroniser à l’aide de l’invite d’Android Studio ou d’exécuter `./gradlew clean build` à partir de la ligne de commande.
</Card>

Dans la section `android`, définissez Java 8 comme bytecode cible pour les plugins Android et Kotlin, respectivement.

```kotlin lines
android {
  compileOptions {
    sourceCompatibility JavaVersion.VERSION_1_8
    targetCompatibility JavaVersion.VERSION_1_8
  }
  kotlinOptions {
    jvmTarget = '1.8'
  }
}
```

<div id="permissions">
  ### Autorisations
</div>

Ouvrez le fichier `AndroidManifest.xml` de votre application et ajoutez l’autorisation suivante.

```text lines
<uses-permission android:name="android.permission.INTERNET" />
```

<div id="configure-for-universal-login">
  ## Configurer pour Universal Login
</div>

D’abord, accédez à [Dashboard &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur le nom de l’application que vous souhaitez modifier. Dans **Allowed Callback URLs**, assurez-vous que votre URL respecte ce format :
`https://YOUR_DOMAIN/android/{YOUR_APP_PACKAGE_NAME}/callback`

Ensuite, remplacez `{YOUR_APP_PACKAGE_NAME}` par le véritable nom de package de votre application. Vous le trouverez dans votre fichier `app/build.gradle`, comme valeur de `applicationId`.

Puis, dans votre fichier `app/build.gradle`, ajoutez les [espaces réservés](https://developer.android.com/build/manage-manifests) pour les propriétés Auth0 Domain et Auth0 Scheme, que la bibliothèque utilisera pour enregistrer un intent-filter qui intercepte l’URI de rappel.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous ne prévoyez pas utiliser la fonctionnalité Web Authentication et ne déclarez pas les espaces réservés, vous pouvez redéclarer manuellement l’Activity dans le fichier Manifest de votre application Android afin de la retirer du fichier Manifest fusionné. Ainsi, Gradle ne signalera pas l’absence des espaces réservés.

  Veuillez [consulter la FAQ](https://github.com/auth0/auth0.android/blob/main/FAQ.md) pour en savoir plus.
</Callout>

```kotlin lines
apply plugin: 'com.android.application'

android {
    compileSdkVersion 30
    defaultConfig {
        applicationId "com.auth0.samples"
        minSdkVersion 21
        targetSdkVersion 30
        //...

        //---> Ajouter la ligne suivante
        manifestPlaceholders = [auth0Domain: "@string/com_auth0_domain", auth0Scheme: "https"]
        //<---
    }
    //...
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Il est recommandé de définir des ressources réutilisables comme `@string/com_auth0_domain` plutôt que de les coder en dur.
</Callout>

<div id="initialize-auth0">
  ## Initialiser Auth0
</div>

Créez un nouvel objet `Auth0` à l’aide de votre <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">ID client</Tooltip> Auth0 et de votre valeur de domaine. Cet objet sera ensuite utilisé lors des interactions avec les points de terminaison d’Auth0.

```kotlin lines
val auth0 = Auth0("{yourClientId}", "YOUR_DOMAIN")
```

Vous pouvez aussi initialiser cet objet à l’aide d’un Context Android, comme le montre [cet exemple d’utilisation](https://github.com/auth0/Auth0.Android/tree/master#usage).

<div id="reset-user-password">
  ### Réinitialiser le mot de passe de l’utilisateur
</div>

Pour lancer la réinitialisation du mot de passe d’un utilisateur, appelez `resetPassword` en passant en paramètres l’adresse courriel de l’utilisateur et le nom de la connexion de base de données.

Les requêtes de réinitialisation du mot de passe échoueront en cas d’erreurs réseau, mais n’échoueront pas si l’adresse courriel indiquée n’existe pas dans la base de données (pour des raisons de sécurité).
