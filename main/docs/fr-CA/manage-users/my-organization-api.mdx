L’API My Organization d’Auth0 fournit une interface sécurisée, circonscrite à l’Organization, qui permet à vos clients commerciaux de gérer leurs propres organisations dans votre tenant Auth0. Cette API sert de fondement technique à l’administration déléguée intégrée et aux intégrations axées sur les API.

<div id="core-capabilities">
  ## Capacités principales
</div>

À l’heure actuelle, l’API prend en charge la gestion des éléments suivants :

* les détails de [Auth0 Organization](/docs/fr-CA/manage-users/organizations) (nom, image de marque, nom d’affichage)
* la configuration, la propriété et les relations de l’Organization
* les [fournisseurs d’identité (IdP)](/docs/fr-CA/authenticate/identity-providers) et le provisionnement [SCIM](/docs/fr-CA/authenticate/protocols/scim)
* la configuration des domaines et de Home Realm Discovery (HRD)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’API My Organization vous permet d’exercer un contrôle technique approfondi sur votre intégration. Pour accélérer le déploiement, nous vous recommandons fortement de commencer par les composants d’interface utilisateur intégrables, les SDK et les applications d’exemple. Les composants d’interface utilisateur intégrables et les applications d’exemple réduisent considérablement le temps et les efforts nécessaires pour offrir une expérience en libre-service à vos clients et à vos utilisateurs finaux.
</Callout>

<div id="set-up-my-organization-api">
  ## Configurer My Organization API
</div>

<div id="activate-the-my-organization-api-in-auth0-dashboard">
  ### Activer My Organization API dans Auth0 Dashboard
</div>

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis)**.
2. Repérez la bannière My Organization API.
3. Sélectionnez **Activer**.
   <Frame>![Auth0 Dashboard>Authentification>APIs](/docs/images/cdy7uua7fh8z/my-org-api-activate.png)</Frame>
4. L’API apparaît dans votre liste Applications &gt; API sous le nom My Organization API.

Une fois My Organization API activée :

* Auth0 désactive l’API par défaut pour toutes les applications clientes.
* Vous devez accorder l’accès aux applications et aux rôles à l’aide de [octroi client](/docs/fr-CA/get-started/applications/application-access-to-apis-client-grants) ou de politiques [RBAC](/docs/fr-CA/manage-users/access-control/configure-core-rbac/enable-role-based-access-control-for-apis).
* Vos clients d’affaires peuvent récupérer les détails d’une Organization ou configurer des IdP au nom de leurs propres Organizations.

<div id="default-settings">
  #### Paramètres par défaut
</div>

**Domaine Auth0 ou domaine personnalisé**

My Organization API prend en charge l’utilisation de votre domaine Auth0 canonique ou de votre domaine personnalisé, mais vous devez utiliser le même tout au long du processus, ce qui comprend les éléments suivants :

* Demander un jeton d’accès
* Définir la valeur d’audience ou `aud`
* Appeler le point de terminaison de My Organization API

Pour en savoir plus sur l’utilisation des domaines personnalisés dans Auth0, consultez [Custom Domains](/docs/fr-CA/customize/custom-domains).

**Politiques d’accès**

Par défaut, My Organization API est activée avec les politiques d’accès des applications à l’API suivantes :

* `require_client_grant` pour les flux utilisateur
* `deny-all` pour les flux machine-à-machine

Pour qu’une application puisse accéder à My Organization API au nom de l’utilisateur, créez un octroi client pour cette application afin de définir les portées maximales qu’elle peut demander. Vous pouvez aussi permettre à toute application de votre tenant de demander n’importe quelle portée en remplaçant les flux d’accès utilisateur par `allow_all`.

<Warning>
  Auth0 ne recommande pas d’utiliser `allow_all` pour les flux d’accès utilisateur, parce que My Organization API expose des renseignements et des opérations sensibles. Vous devriez appliquer le principe du moindre privilège afin de vous assurer que les applications obtiennent uniquement l’accès dont elles ont réellement besoin, ce qui réduit au minimum les risques potentiels pour la sécurité.
</Warning>

Les autorisations finales accordées à l’application seront déterminées par l’intersection des portées autorisées par la politique d’accès de l’application à l’API, des permissions de contrôle d’accès basé sur les rôles (RBAC) attribuées à l’utilisateur final et de tout consentement donné par l’utilisateur (le cas échéant).

Pour en savoir plus sur la gestion des politiques d’accès des applications à l’API et des octrois client qui y sont associés, consultez [Application Access to APIs: Client Grants](/docs/fr-CA/get-started/applications/application-access-to-apis-client-grants).

**Durée de vie des jetons**

My Organization API émet des jetons d’accès dont la durée de vie est fixe, soit 600 secondes (10 minutes). Cette courte durée est une mesure de sécurité délibérée conçue pour protéger votre tenant et ses ressources.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  My Organization API demeurera toujours à activation explicite pour des raisons de sécurité. La désactivation de l’API supprime l’accès pour toutes les applications connectées jusqu’à sa réactivation.
</Callout>

### Configurer les attributs de l’application cliente

[Créez une application](/docs/fr-CA/get-started/auth0-overview/create-applications) dans Auth0 pour l’utiliser avec l’API My Organization. Une fois créée, accédez à **[Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis)** et autorisez l’API My Organization, y compris les scopes que vous voulez accorder à l’application.

Votre application doit fournir l’objet `my_organization_configuration`, sinon l’API My Organization renverra une erreur et rejettera la requête. Vous pouvez utiliser les propriétés suivantes avec l’objet `my_organization_configuration` :

| **Propriété**                  | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `connection_profile_id`        | **ID du Connection Profile.** ID du [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile) utilisé avec l’application lors de l’utilisation de l’API My Organization. S’il n’est pas fourni, les fonctionnalités de l’API My Organization qui exigent la présence d’un Connection Profile ne fonctionneront pas. Cet ID doit faire référence à un Connection Profile valide dans le même tenant.                                                                                                                                                                                                                                                                                                                                              |
| `user_attribute_profile_id`    | **ID du User Attribute Profile**. ID du [User Attribute Profile](/docs/fr-CA/authenticate/enterprise-connections/user-attribute-profile) utilisé avec l’application lors de l’utilisation de l’API My Organization. S’il n’est pas fourni, les fonctionnalités de l’API My Organization qui exigent la présence d’un User Attribute Profile ne fonctionneront pas. Cet ID doit faire référence à un User Attribute Profile valide dans le même tenant.                                                                                                                                                                                                                                                                                                                          |
| `allowed_strategies`           | **Tableau de chaînes.** Chaque chaîne est unique et fait référence à une stratégie prise en charge. Les stratégies prises en charge — les valeurs de l’énumération — sont les suivantes : `pingfederate`, `ad`, `adfs`, `waad`, `google-apps`, `okta`, `oidc` et `samlp`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `connection_deletion_behavior` | **Énumération (allow, allow&#95;if&#95;empty).** Décrit le comportement de l’API My Organization lorsqu’un utilisateur final tente de supprimer une connexion au moyen de l’API My Organization à partir de cette application. Les valeurs et la description de l’énumération sont les suivantes : <p /> 1. `allow` : si l’utilisateur possède le scope approprié, il peut supprimer la connexion, ce qui entraîne la suppression de tous les utilisateurs issus de cette connexion. <p /> 2.`allow_if_empty` : si l’utilisateur possède le scope approprié, il peut supprimer la connexion uniquement s’il n’y a aucun utilisateur dans la connexion. Si des utilisateurs sont présents, l’API My Organization renverra une erreur et ne procédera pas à la suppression. |

<div id="generate-an-access-token">
  ## Générer un jeton d’accès
</div>

Le My Organization API ne peut être appelé qu’avec un jeton d’accès associé à un utilisateur, obtenu au moyen de l’un des [flux OAuth 2.0 pris en charge](https://auth0.com/docs/get-started/authentication-and-authorization-flow).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous prévoyez autoriser le My Organization API à effectuer des opérations sensibles, nous vous recommandons fortement d’utiliser l’[authentification renforcée](/docs/fr-CA/secure/multi-factor-authentication/step-up-authentication) afin d’appliquer des politiques de sécurité supplémentaires au moyen de l’authentification multifacteur (MFA).
</Callout>

<div id="example-with-authorization-code-flow">
  ### Exemple avec le flux de code d’autorisation
</div>

Utilisez le [flux de code d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow) pour les applications Web confidentielles avec un <Tooltip tip="Secret client : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne devrait être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Secret">Secret client</Tooltip>.

```bash
curl --request POST \
--url 'https://YOUR_DOMAIN/oauth/token' \
--header 'content-type: application/x-www-form-urlencoded' \
--data 'grant_type=authorization_code' \
--data 'client_id=YOUR_CLIENT_ID' \
--data 'client_secret=YOUR_CLIENT_SECRET' \
--data 'code=AUTH_CODE' \
--data 'redirect_uri=https://yourapp/callback' \
--data 'audience=https://YOUR_DOMAIN/my-org/'
```

**Exemple de réponse**

```json
{
  "access_token": "eyJz93a...k4laUWw",
  "token_type": "Bearer",
  "expires_in": 600,
  "scope": "read:my_org:details update:my_org:identity_providers"
}
```

<div id="example-with-authorization-code-flow-with-pkce">
  ### Exemple avec le flux de code d’autorisation avec PKCE
</div>

Utilisez le [flux de code d’autorisation avec clé de preuve pour l’échange de code (PKCE)](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) pour les applications publiques sans secret client, les applications monopages, les applications mobiles ou natives et les outils CLI.

```bash
curl --request POST \
--url 'https://YOUR_DOMAIN/oauth/token' \
--header 'content-type: application/x-www-form-urlencoded' \
--data 'grant_type=authorization_code' \
--data 'client_id=YOUR_CLIENT_ID' \
--data 'code=AUTH_CODE' \
--data 'code_verifier=CODE_VERIFIER' \
--data 'redirect_uri=https://yourapp/callback' \
--data 'audience=https://YOUR_DOMAIN/my-org/'
```

<div id="audience">
  ### Audience
</div>

L’audience et l’URL de base de My Organization API sont `https://{yourDomain}/my-org/`. Les jetons doivent inclure l’audience `https://YOUR_DOMAIN/my-org/`. Les jetons d’autres API (comme `/me` ou `/api/v2/`) ne fonctionneront pas.

<div id="user-profiles">

L’API My Organization utilise les [Connection Profiles](/docs/fr-CA/authenticate/enterprise-connections/connection-profile) et les [User Attribute Profiles](/docs/fr-CA/authenticate/enterprise-connections/user-attribute-profile) pour définir la structure, les restrictions et les règles des configurations créées par des clients tiers.

</div>

Le Connection Profile permet aux développeurs d’Auth0 de préciser comment les paramètres privés d’une Auth0 connexion doivent être configurés lorsqu’elle est créée par des tiers. Pour en savoir plus sur le fonctionnement du Connection Profile, ses mappages d’attributs et ses remplacements, consulter des exemples et apprendre à en configurer un, lisez [Connection Profiles](/docs/fr-CA/authenticate/enterprise-connections/connection-profile).

</div>

Le User Attribute Profile (UAP) offre une manière cohérente de définir, de gérer et de faire le mappage des attributs utilisateur dans des protocoles comme SCIM, SAML et OIDC. Pour en savoir plus sur le fonctionnement de l’UAP, ses mappages d’attributs et ses overrides, voir des exemples et apprendre à en configurer un, consultez [User Attribute Profiles](/docs/fr-CA/authenticate/enterprise-connections/user-attribute-profile).

<div id="per-organization-rate-limits">
  ### Limites de débit par Organization
</div>

En plus des limites de débit du niveau de service, la My Organization API applique également des limites de débit par Organization. Ces limites sont conçues pour assurer une répartition équitable des ressources et éviter qu’une seule Organization n’ait une incidence sur les performances globales de votre tenant. En imposant ces limites, nous atténuons l’effet de « voisin bruyant », afin qu’une hausse soudaine d’activité provenant d’une Organization ne monopolise pas les ressources partagées et n’ait pas d’incidence sur une autre au sein du même environnement. Chaque Organization se voit attribuer un nombre précis de requêtes par seconde (RPS), pour les opérations de lecture comme d’écriture.

| **Tier**                | **Lecture par Organization (RPS)** | **Écriture par Organization (RPS)** |
| ----------------------- | ---------------------------------- | ----------------------------------- |
| **Free**                | 4                                  | 2                                   |
| **Public Self-Service** | 4                                  | 2                                   |
| **Public Enterprise**   | 8                                  | 4                                   |
| **Private Basic**       | 8                                  | 4                                   |
| **Private Performance** | 16                                 | 8                                   |

<div id="cross-origin-requests">
  ## Requêtes cross-origin
</div>

Si vous comptez appeler directement la My Organization API à partir d’une application browser-based (comme une Single Page Application) exécutée sur un domaine différent de celui de votre tenant Auth0, vous serez confronté aux politiques de sécurité des navigateurs appelées [Cross-Origin Resource Sharing (CORS)](/docs/fr-CA/get-started/applications/set-up-cors). Par défaut, les navigateurs bloquent ces requêtes cross-origin.

Pour permettre à votre application d’envoyer des requêtes à l’API avec succès, vous devez ajouter le domaine de votre application (son « origine ») à la configuration de votre client :

1. Accédez à [**Auth0 Dashboard &gt; Applications**](https://manage.auth0.com/#/applications). Sélectionnez l’application à afficher.
2. Sous Cross-Origin Authentication, activez la bascule Allow Cross-Origin Authentication.
3. Repérez Allowed Origins (CORS), puis saisissez l’URL d’origine de votre application.
4. Sélectionnez **Save**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous n’avez pas besoin d’utiliser CORS pour votre application, assurez-vous que la bascule Allow Cross-Origin Authentication est désactivée. En ajoutant l’URL de votre application à cette liste, vous indiquez à Auth0 de faire confiance aux requêtes provenant de cette origine, ce qui permet à votre application client-side d’accéder à l’API.
</Callout>

<div id="log-events">

</div>

Pour faciliter l’audit et le suivi granulaires, l’API My Organization génère un ensemble précis d’événements de journalisation qui lui sont propres. Ces codes d’événement vous permettent de suivre les activités liées à toutes les ressources gérées par l’API, plus précisément : la configuration, les détails de l’Organization, les IdPs et les domaines. Si vous souhaitez en savoir plus sur les schémas des événements de journalisation, vous pouvez consulter notre [référentiel GitHub](https://github.com/auth0/auth0-log-schemas).

| **Code d’événement**                        | **Événement**                                                   | **Description de l’événement**                                                                     |
| ------------------------------------------- | --------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `my_organization_api_config_failed`         | Échec de la configuration de l’API My Organization              | Échec d’un appel API vers la ressource de configuration du service API My Organization             |
| `my_organization_api_org_details_succeeded` | Réussite des détails de l’Organization de l’API My Organization | Appel API réussi vers la ressource de détails de l’Organization du service API My Organization     |
| `my_organization_api_org_details_failed`    | Échec des détails de l’Organization de l’API My Organization    | Échec d’un appel API vers la ressource de détails de l’Organization du service API My Organization |
| `my_organization_api_idp_succeeded`         | Réussite du fournisseur d’identité de l’API My Organization     | Appel API réussi vers la ressource de fournisseur d’identité du service API My Organization        |
| `my_organization_api_idp_failed`            | Échec du fournisseur d’identité de l’API My Organization        | Échec d’un appel API vers la ressource de fournisseur d’identité du service API My Organization    |
| `my_organization_api_domain_succeeded`      | Réussite du domaine de l’API My Organization                    | Appel API réussi vers la ressource de domaine du service API My Organization                       |
| `my_organization_api_domain_failed`         | Échec du domaine de l’API My Organization                       | Échec d’un appel API vers la ressource de domaine du service API My Organization                   |

<div id="organization-connection-ownership">
  ## Propriété des connections d’Organization
</div>

L’API introduit un modèle de propriété pour distinguer les connections gérées par l’Admin du tenant et celles autogérées par l’Organization. Ce modèle est contrôlé par la propriété `organization_access_level`.

**Propriété clé** : `organization_access_level`

| **Valeur d’énumération** | **Description**                                                                                                                                                                                                                                                                                                   |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `none`                   | Association attribuée par l’administrateur du tenant. Cette connection ne peut pas être consultée ni modifiée au moyen de la My Organization API.                                                                                                                                                                 |
| `readonly`               | Association attribuée par l’administrateur du tenant. Elle peut être consultée, mais ne peut pas être modifiée au moyen de la My Organization API.                                                                                                                                                                |
| `limited`                | Association attribuée par l’administrateur du tenant. Des renseignements et des modifications limités sont autorisés au moyen de la My Organization API, plus précisément pour les attributs `show_as_button` et `is_enabled`.                                                                                    |
| `full`                   | Association attribuée par l’administrateur du tenant. Les utilisateurs de la My Organization API pourront modifier les attributs de connexion propres à l’Organization sans restrictions supplémentaires, plus précisément les attributs `show_as_button`, `is_enabled`, `options`, `display_name` et `domains`. |

**Endpoints de la Management API pour les Connections** :

* `GET` [`/api/v2/organizations/{id}/connections`](https://auth0.com/docs/api/management/v2/organizations/get-organization-connection)
* `POST` [`/api/v2/organizations/{id}/connections`](https://auth0.com/docs/api/management/v2/organizations/post-organization-connection)
* `PATCH` [`/api/v2/organizations/{id}/connections/{id}`](https://auth0.com/docs/api/management/v2/organizations/patch-organization-connection)
* `DELETE` [`/api/v2/organizations/{id}/connections/{id}`](https://auth0.com/docs/api/management/v2/organizations/delete-organization-connection)

Lorsque vous appelez les endpoints `/connections`, utilisez les mêmes scopes que pour les endpoints `/enabled_connections` :

* `create:organization_connections`
* `read:organization_connections`
* `delete:organization_connections`
* `update:organization_connections`

Consultez les attributs supplémentaires du schéma :

| **Propriété**                  | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `is_enabled`                   | **Boolean**. Active ou désactive une connection pour une Organization. En désactivant la connection, il sera possible de conserver les valeurs de l’objet connection enregistrées tout en gardant la connection désactivée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `organization_access_level`    | **Énumération (null, none, readonly, limited, full)**. Détermine le type d’accès qu’un utilisateur aura lorsqu’il utilise la My Organization API : 1. `null` 2. `none` - Association attribuée par l’administrateur du tenant. Cette connection ne peut pas être consultée ni modifiée au moyen de la My Organization API. 3. `readonly` - Association attribuée par l’administrateur du tenant. Cette connection peut être consultée, mais ne peut pas être modifiée au moyen de la My Organization API. 4. `limited` - Association attribuée par l’administrateur du tenant. Des renseignements et des modifications limités sont autorisés au moyen de la My Organization API. 5. `full` - Association attribuée par l’administrateur du tenant. Les utilisateurs de la My Organization API pourront modifier la connexion sans restrictions supplémentaires lorsqu’ils utilisent la My Organization API. Remarque : indépendamment de `organization_access_level`, les modifications effectuées au moyen de la My Organization API restent toujours assujetties aux restrictions appliquées par le Connection Profile. |
| `organization_connection_name` | **String**. Stocke le nom fourni lors de la création de la connexion. Ce champ est calculé par la My Organization API en évaluant le `connection_name_prefix_template` du Connection Profile. Ce champ n’est visible et modifiable qu’au moyen de la Management API.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |

Remarques :

* Ces endpoints acceptent un query parameter facultatif de `is_enabled=true/false` et, s’il est présent, n’affichent que les connections ayant la valeur `is_enabled` spécifiée.
* `organization_access_level` ne peut être modifié qu’au moyen de la Management API.
* Si l’attribut `name` n’est pas défini, il doit être renseigné au moyen de la Management API avant de changer `organization_access_level` de `none` à toute autre valeur.
