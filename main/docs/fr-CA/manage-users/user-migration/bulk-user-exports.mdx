Vous pouvez utiliser le point de terminaison `POST /api/v2/jobs/users-exports` pour créer une tâche qui exporte tous les utilisateurs associés à une [connexion](/docs/fr-CA/authenticate/identity-providers), ou tous les utilisateurs du tenant.

Pour obtenir la liste des champs du profil utilisateur pouvant être exportés, consultez [User Profile Structure](/docs/fr-CA/manage-users/user-accounts/user-profiles/user-profile-structure).

Lorsque vous créez votre tâche, vous devrez fournir :

* l’[ID de la connexion](/docs/fr-CA/authenticate/identity-providers/locate-the-connection-id) à partir de laquelle vous souhaitez exporter des utilisateurs (facultatif)
* le format du fichier d’exportation (CSV ou compatible avec JSON)
* le nombre maximal d’enregistrements d’utilisateurs à exporter (facultatif; si omis, tous les enregistrements seront exportés)
* les champs liés à l’utilisateur (comme l’ID utilisateur ou le nom) que vous souhaitez inclure dans l’exportation

Vous aurez aussi besoin d’un [jeton d’accès à la Management API](https://auth0.com/docs/api/management/v2/tokens).

<div id="create-a-request-body">
  ## Créer un corps de requête
</div>

Au besoin, trouvez le `connection_id` et le nom de domaine de votre tenant Auth0 dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Créez un nouveau fichier texte avec le corps de la requête ci-dessous :

```json lines
{
   "connection_id":"connection_id",
   "format":"csv",
   "limit":20,
   "fields":[
      {
         "name":"user_id"
      },
      {
         "name":"email"
      },
      {
         "name":"user_metadata.country"
      }
   ]
}
```

Mettez à jour le `connection_id` avec l’ID de votre connexion de base de données, ou retirez-le pour exporter tous les utilisateurs du tenant.

<div id="include-user-metadata-in-exported-csv">
  ## Inclure les métadonnées utilisateur dans le CSV exporté
</div>

Si vous exportez des données utilisateur au format CSV et souhaitez inclure des métadonnées, indiquez chaque champ de métadonnées à exporter. Vous pouvez exporter jusqu’à 30 champs.

<Warning>
  Vous ne pouvez pas exporter l’intégralité de `app_metadata` ou de `user_metadata` en CSV. Vous devez préciser explicitement les champs des objets de métadonnées.

  Pour exporter `app_metadata` ou `user_metadata` comme objets uniques, utilisez le format compatible avec JSON et incluez le champ voulu dans le paramètre `fields` du corps de la requête. Par exemple :

  `{"name":"app_metadata"}`

  Comme vous ne pouvez exporter que 30 champs au maximum, il est recommandé d’utiliser le format JSON si les données utilisateur comportent de nombreux champs.
</Warning>

Par exemple, pour des métadonnées structurées comme ceci :

```json lines
{
  "consent": {
      "given": true,
      "date": "01/23/2019",
      "text_details": "{yourURL}"
  }
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Dans les fichiers CSV d’exportation des utilisateurs, nous ajoutons des caractères d’échappement aux types de données String conformément aux normes de l’OWASP visant à atténuer les risques d’injection CSV :

  * Les guillemets doubles sont précédés d’un autre guillemet double.
  * Chaque chaîne est précédée d’une apostrophe.
  * Chaque chaîne est encadrée de guillemets doubles.

  Cela ne s’applique pas aux dates générées par Auth0 au format ISO 8601.
</Callout>

<div id="json-compatible-format">
  ### Format compatible avec JSON
</div>

Si vous exportez les données dans un format compatible avec JSON, vous n&#39;avez qu&#39;à fournir la propriété root ; vous n&#39;avez pas besoin de nommer chaque propriété interne individuellement, puisqu&#39;elles seront incluses automatiquement.

Les fichiers d&#39;exportation d&#39;Auth0 utilisent le format [NDJSON](https://github.com/ndjson/ndjson-spec) en raison de leur grande taille, tandis que la fonctionnalité d&#39;importation attend un fichier JSON.

Avant de pouvoir importer des utilisateurs à l&#39;aide d&#39;une exportation générée par Auth0, vous devrez convertir le fichier du format NDJSON au format JSON à l&#39;aide de la bibliothèque de votre choix (comme [jq](https://stedolan.github.io/jq/)).

Dans ce cas, pour le même exemple que celui utilisé précédemment, la requête ressemblerait à ceci :

<div id="find-export-data">
  ## Trouver les données exportées
</div>

Vous pouvez accéder à vos fichiers d’exportation à l’aide de l’URL fournie comme valeur du paramètre **location**. Le nom de votre tenant correspond aussi au nom de votre fichier. Par exemple, si le nom de votre tenant est `auth0docs`, votre fichier sera `auth0docs.csv` ou `auth0docs.json`. Lorsque vous accédez à l’URL, le téléchargement du fichier démarre automatiquement.

Le lien de téléchargement est valide pendant 60 secondes. Une fois ce délai expiré, vous avez 24 heures pour utiliser de nouveau l’URL avant l’expiration de la tâche.

<div id="job-cleanup">
  ## Nettoyage des tâches
</div>

Toutes les données liées à vos tâches sont automatiquement supprimées après 24 heures et ne sont plus accessibles par la suite. Par conséquent, **nous vous recommandons fortement de stocker les résultats des tâches au moyen du mécanisme de stockage de votre choix**.

<div id="filter-example">
  ## Exemple de filtre
</div>

Vous pourriez vouloir filtrer votre fichier `.csv` exporté pour n’en conserver qu’un sous-ensemble de données, par exemple la date de la dernière connexion des utilisateurs à votre application.

1. Faites un appel `POST` au point de terminaison [créer une tâche d’exportation des utilisateurs](https://auth0.com/docs/api/management/v2/jobs/post-users-exports) de la Management API pour exporter les utilisateurs vers un fichier `.csv` :

   1. Créez une nouvelle tâche :

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/users-exports' \
      -H 'Content-Type: application/json' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN' \
      -d '{"format":"csv"}'
      ```

   2. Vérifiez le statut :

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/JOB_ID' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN'
      ```

2. Accédez au fichier dans vos téléchargements sur votre ordinateur local.

3. Utilisez un outil en ligne de commande pour ajouter un filtre. Dans cet exemple, nous utilisons [Miller](https://miller.readthedocs.io/en/latest/installing-miller/) :

   ```bash wrap lines
   mlr --csv filter '$last_login >= "2024-01-01T00:00:00Z"' file.csv > filtered.csv
   ```
