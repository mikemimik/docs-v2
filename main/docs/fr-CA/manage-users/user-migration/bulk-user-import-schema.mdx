Le fichier d’utilisateurs doit contenir un tableau des données utilisateur au format JSON.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La taille maximale du fichier pour une importation en bloc est de 500 KB. Vous devrez lancer plusieurs importations si vos données dépassent cette limite.
</Callout>

</div>

| ---------------------- | ------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------- |

Les données stockées dans `app_metadata` ne peuvent pas être modifiées par les utilisateurs. | `email_verified`       | booléen | Indique si l’utilisateur a vérifié son adresse courriel. La valeur par défaut est `false` si `email` est mis à jour par upsert, mais pas `email_verified`.                                                                                                                                                                                                                                                                                                                                                                                                                                    | Oui                                            |

<div id="app-metadata">
  ## Métadonnées d’application
</div>

L’objet `user.app_metadata` ne doit **contenir aucune** de ces propriétés :

* `__tenant`
* `_id`
* `blocked`
* `clientID`
* `created_at`
* `email_verified`
* `email`
* `globalClientID`
* `global_client_id`
* `identities`
* `lastIP`
* `lastLogin`
* `loginsCount`
* `metadata`
* `multifactor_last_modified`
* `multifactor`
* `updated_at`
* `user_id`

<div id="custom-password-hash">
  ## Hachage de mot de passe personnalisé
</div>

L’objet `user.custom_password_hash` peut être utilisé à la place de la propriété `user.password_hash` lorsque le hachage du mot de passe de l’utilisateur a été créé à l’aide d’un autre algorithme. Notez que ce champ et `password_hash` s’excluent mutuellement.

L’objet `user.custom_password_hash` comporte les propriétés suivantes :

| Propriété           | Type   | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `algorithm`         | chaîne | L’algorithme utilisé pour hacher le mot de passe. Doit être l’un des suivants : <ul><li>`argon2`</li><li>`bcrypt`</li><li>`hmac`</li><li>`ldap`</li><li>`md4`</li><li>`md5`</li><li>`sha1`</li><li>`sha256`</li><li>`sha512`</li><li>`pbkdf2`</li><li>`scrypt`</li></ul>                                                                                                                                                                                                                           |
| `hash`              | objet  |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `hash.value`        | chaîne | Le hachage du mot de passe.                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `hash.encoding`     | chaîne | L’encodage du hachage fourni. Doit être l’un des suivants : <ul><li>`base64`</li><li>`hex`</li><li>`utf8`</li></ul>Les variantes hexadécimales en majuscules et en minuscules sont prises en charge, de même que le base64 encodé dans l’URL.                                                                                                                                                                                                                                                      |
| `hash.digest`       | chaîne | L’algorithme utilisé pour générer le hachage HMAC. Doit être l’un des suivants : <ul><li>`md4`</li><li>`md5`</li><li>`ripemd160`</li><li>`sha1`</li><li>`sha224`</li><li>`sha256`</li><li>`sha384`</li><li>`sha512`</li><li>`whirlpool`</li></ul>                                                                                                                                                                                                                                                  |
| `hash.key`          | objet  | La clé utilisée pour générer le hachage HMAC.                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `hash.key.value`    | chaîne | La valeur de la clé.                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `hash.key.encoding` | chaîne | L’encodage de la clé. Doit être l’un des suivants : <ul><li>`base64`</li><li>`hex`</li><li>`utf8`</li></ul>Par défaut, `hash.key.encoding` est `utf8`.                                                                                                                                                                                                                                                                                                                                             |
| `salt`              | objet  |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `salt.value`        | chaîne | La valeur du salt utilisée pour générer le hachage.                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| `salt.encoding`     | chaîne | L’encodage du salt fourni. Doit être l’un des suivants : <ul><li>`base64`</li><li>`hex`</li><li>`utf8`</li></ul> Les variantes hexadécimales en majuscules et en minuscules sont prises en charge, de même que le base64 encodé dans l’URL. Par défaut, `salt.encoding` est `utf8`.                                                                                                                                                                                                                |
| `salt.position`     | chaîne | La position du salt au moment du calcul du hachage. Par défaut, `salt.position` est `prefix`.                                                                                                                                                                                                                                                                                                                                                                                                      |
| `password.encoding` | chaîne | L’encodage du mot de passe utilisé pour générer le hachage. Doit être l’un des suivants : <ul><li>`ascii`</li><li>`utf8`</li><li>`utf16le`</li><li>`ucs2`</li><li>`latin1`</li><li>`binary`</li></ul> À la connexion, le mot de passe fourni par l’utilisateur sera transcodé depuis `password.encoding` avant d’être vérifié par rapport au hachage fourni. Par exemple, si votre hachage a été généré à partir d’une chaîne encodée en `ucs2`, vous devrez alors définir : ` "encoding": "ucs2"` |
| `keylen`            | entier | Longueur de clé souhaitée, en octets, pour le hachage scrypt. Doit être un entier supérieur à zéro.<br />Ce paramètre est requis lorsque `algorithm` est défini sur `scrypt`.                                                                                                                                                                                                                                                                                                                      |
| `cost`              | entier | Paramètre de coût processeur/mémoire utilisé pour le hachage scrypt. Doit être une puissance de deux supérieure à un. Par défaut, `cost` est 16384.<br />Ce paramètre est utilisé uniquement lorsque `algorithm` est défini sur `scrypt`.                                                                                                                                                                                                                                                          |
| `blockSize`         | entier | Paramètre de taille de bloc utilisé pour le hachage scrypt. Doit être un entier positif. Par défaut, `blockSize` est 8.<br />Ce paramètre est utilisé uniquement lorsque `algorithm` est défini sur `scrypt`.                                                                                                                                                                                                                                                                                      |
| `parallelization`   | entier | Paramètre de parallélisation utilisé pour le hachage scrypt. Doit être un entier positif. Par défaut, `parallelization` est 1.<br />Ce paramètre est utilisé uniquement lorsque `algorithm` est défini sur `scrypt`.                                                                                                                                                                                                                                                                               |

<div id="update-custom-password-hash">
  ### Mettre à jour le hachage de mot de passe personnalisé
</div>

Pendant le processus d’importation en bloc, vous pouvez mettre à jour le `custom_password_hash` si l’utilisateur ne s’est pas connecté à l’aide du `custom_password_hash` importé initialement. Par exemple, vous pouvez envoyer le JSON ci-dessous deux fois au point de terminaison[`/api/v2/jobs/users-imports`](https://auth0.com/docs/api/management/v2?_ga=2.169020323.346193905.1601302257-746691936.1587131255#!/Jobs/post_users_imports), avec des valeurs différentes pour `custom_password_hash`. Lors du deuxième envoi, définissez l’indicateur `upsert` à `true`.

```json lines
[
    {
    	"user_id": "2000",
        "email": "examplecouser20@gmail.com",
        "given_name": "ExampleCo User",
        "name" : "ExampleCoUser20",
        "custom_password_hash": {
            "algorithm": "bcrypt",
            "hash": {
                "value": "$2a$10$aHF7mbpWT6tZ7PJVtwtjNelaKbszikcYBCB2jibvbFcGFmOsu/s4K"
            }
        }
    }
]
```

Vous pouvez utiliser le [Bcrypt Password Generator](https://www.browserling.com/tools/bcrypt) sur browserling.com pour générer des hachages de mot de passe bcrypt.

<div id="supported-hash-algorithms">
  ### Algorithmes de hachage pris en charge
</div>

Auth0 prend actuellement en charge l’importation de mots de passe utilisateur hachés avec :

* [Argon2](https://github.com/p-h-c/phc-winner-argon2)
* [bcrypt](https://auth0.com/blog/hashing-in-action-understanding-bcrypt/)
* [LDAP](https://tools.ietf.org/html/rfc2307#section-5.3) (`RFC-2307 "userPassword"`)
* [HMAC](https://tools.ietf.org/html/rfc2104)
* [MD4](https://tools.ietf.org/html/rfc1320)
* [MD5](https://tools.ietf.org/html/rfc1321)
* [SHA1](https://tools.ietf.org/html/rfc3174)
* [SHA256 et SHA512](https://tools.ietf.org/html/rfc4634)
* [PBKDF2](https://tools.ietf.org/html/rfc2898#section-5.2)
* [scrypt](https://datatracker.ietf.org/doc/rfc7914/)

Veuillez consulter les sections suivantes lorsque vous fournissez un `custom_password_hash`.

<div id="argon2">
  #### Argon2
</div>

Lorsque `algorithm` est défini sur `argon2` :

* `hash.encoding` doit être `utf8`.
* `hash.salt` n&#39;est pas autorisé.
* `hash.value` doit être au format de chaîne PHC, comme indiqué dans [P-H-C / phc-string-format](https://github.com/P-H-C/phc-string-format/blob/master/phc-sf-spec.md) sur GitHub. Il doit également être conforme aux exigences précisées dans [Auth0 / magic](https://github.com/auth0/magic#magicpasswordhash--magicverifypassword) sur GitHub.
* `hash.value` doit inclure le salt encodé en base64 (comme indiqué dans la documentation `PHC`).

<div id="bcrypt">
  #### bcrypt
</div>

Lorsque `algorithm` est défini sur `bcrypt` :

* `hash.encoding` doit être `utf8`.
* `hash.salt` peut être utilisé avec l’encodage et la position du salt.
* `hash.value` doit inclure l’un de ces préfixes :

  * `$2a$`
  * `$2b$`
  * `$2y$`

  Les autres préfixes, comme `$2$`, `$sha1$` et `$2x$`, ne sont pas pris en charge pour le moment.

Par exemple, cette valeur a été générée à partir de la chaîne `hello` à l’aide d’un paramètre de coût de 10 :

`$2b$10$nFguVi9LsCAcvTZFKQlRKeLVydo8ETv483lkNsSFI/Wl1Rz1Ypo1K`

L’algorithme `bcrypt` traite au maximum 72 octets d’entrée lors du calcul des hachages de mot de passe ou de l’exécution de comparaisons, et la longueur de `salt.value` compte dans cette limite de 72 octets. Toute entrée qui dépasse la limite de 72 octets est tronquée; par exemple, si le salt consomme 10 octets, la longueur maximale du mot de passe pour le hachage ou la comparaison est de 62 octets.

Les mots de passe qui dépassent cette limite réduite sont tronqués, ce qui peut affaiblir la robustesse du mot de passe ou entraîner des collisions de hachage. Validez toujours la longueur des mots de passe avant le hachage.

<div id="hmac">
  #### HMAC
</div>

Lorsque `algorithm` est défini sur `hmac` :

* `hash.encoding` doit être soit `hex`, soit `base64`.
* `hash.digest` est requis et doit correspondre à l’un des éléments suivants :

  * `md4`
  * `md5`
  * `ripemd160`
  * `sha1`
  * `sha224`
  * `sha256`
  * `sha384`
  * `sha512`
  * `whirlpool`
* `hash.key.value` est requis.
* `hash.key.encoding` doit être soit `base64`, soit `hex`, soit `utf8`.

<div id="ldap">
  #### LDAP
</div>

Lorsque `algorithm` est défini sur `ldap` :

* `hash.encoding` doit être `utf8`.
* `salt` n’est pas autorisé.
* `hash.value` doit respecter le format décrit dans la [section 5.3 de la RFC-2307](https://tools.ietf.org/html/rfc2307#section-5.3) sur l’IETF Datatracker.
* Le schéma doit être l’un des suivants : `md5|smd5|sha*|ssha*`. Consultez [cette page](https://www.openldap.org/faq/data/cache/347.html) pour en savoir plus.
* Notez que le schéma [crypt](https://www.openldap.org/faq/data/cache/344.html) n’est **pas pris en charge** en raison d’un comportement dépendant du système ou de la mise en œuvre. Pour en savoir plus, consultez [Open LDAP Admin Guide - 14.4.2. CRYPT password storage scheme](https://www.openldap.org/doc/admin24/guide.html#CRYPT%20password%20storage%20scheme).

<div id="md-or-sha">
  #### MD ou SHA
</div>

Lorsque `algorithm` est défini à `md4`, `md5`, `sha1`, `sha256` ou `sha512` :

* `hash.encoding` doit être `hex` ou `base64`.

<div id="pbkdf2">
  #### PBKDF2
</div>

Lorsque `algorithm` est défini sur `pbkdf2` :

* `hash.encoding` doit être `utf8`.
* `hash.salt` n’est pas autorisé.
* `hash.value` doit être au format PHC string format, comme précisé dans [P-H-C / phc-string-format](https://github.com/P-H-C/phc-string-format/blob/master/phc-sf-spec.md) sur GitHub.
* `hash.value` doit inclure le salt encodé en B64 (en base64, sans les caractères de remplissage `=`, comme précisé dans la documentation `PHC`).
* `hash.value` devrait inclure les paramètres `i` (itérations) et `l` (keylen). Si ces paramètres sont omis, les valeurs par défaut seront `i=100000` et `l=64`.
* L’`id` doit être au format `pbkdf2-<digest>` (`pbkdf2-sha512`, `pbkdf2-md5`, etc). Les algorithmes de condensé pris en charge sont :

  * `RSA-MD4`
  * `RSA-MD5`
  * `RSA-MDC2`
  * `RSA-RIPEMD160`
  * `RSA-SHA1`
  * `RSA-SHA1-2`
  * `RSA-SHA224`
  * `RSA-SHA256`
  * `RSA-SHA384`
  * `RSA-SHA512`
  * `md4`
  * `md4WithRSAEncryption`
  * `md5`
  * `md5WithRSAEncryption`
  * `mdc2`
  * `mdc2WithRSA`
  * `ripemd`
  * `ripemd160`
  * `ripemd160WithRSA`
  * `rmd160`
  * `sha1`
  * `sha1WithRSAEncryption`
  * `sha224`
  * `sha224WithRSAEncryption`
  * `sha256`
  * `sha256WithRSAEncryption`
  * `sha384`
  * `sha384WithRSAEncryption`
  * `sha512`
  * `sha512WithRSAEncryption`
  * `ssl3-md5`
  * `ssl3-sha1`
  * `whirlpool`

<div id="scrypt">
  #### scrypt
</div>

Lorsque `algorithm` est défini sur `scrypt` :

* `hash.encoding` doit être soit `hex`, soit `base64`.
* Le paramètre `keylen` est requis.
* Le paramètre `cost` peut être spécifié; sinon, sa valeur par défaut sera 16384.
* Le paramètre `blockSize` peut être spécifié; sinon, sa valeur par défaut sera 8.
* Le paramètre `parallelization` peut être spécifié; sinon, sa valeur par défaut sera 1.

<div id="mfa-factors">
  ### Facteurs MFA
</div>

Le tableau `user.mfa_factors` contient les inscriptions <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> de l’utilisateur. Pour en savoir plus, consultez [Authentification multifacteur dans Auth0](/docs/fr-CA/secure/multi-factor-authentication). L’importation des inscriptions évite aux utilisateurs d’avoir à se réinscrire au MFA après l’importation. Les types d’inscription pris en charge sont :

| Propriété     | Type   | Description                                                                                                                                                                                                                                        |
| ------------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `email`       | objet |                                                                                                                                                                                                                                                    |
| `email.value` | chaîne | L’adresse courriel utilisée pour le MFA.                                                                                                                                                                                                           |
| `phone`       | objet |                                                                                                                                                                                                                                                    |
| `phone.value` | chaîne | Le numéro de téléphone utilisé pour le MFA par SMS. Il doit inclure un indicatif de pays et commencer par `+`, par exemple : `"+12125550001"`                                                                                                      |
| `totp`        | objet |                                                                                                                                                                                                                                                    |
| `totp.secret` | chaîne | Le secret OTP pour l’authentification MFA avec des applications d’authentification (Google Authenticator, Microsoft Authenticator, Authy, 1Password, LastPass). Il doit être encodé en Base32 sans remplissage, par exemple : `"JBTWY3DPEHPK3PNP"` |
