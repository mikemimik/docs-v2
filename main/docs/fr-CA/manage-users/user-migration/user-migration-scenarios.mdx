Voici quelques exemples de scénarios pour migrer des utilisateurs de Gigya, d’Okta et de Stormpath vers Auth0. Chacun de ces scénarios part du principe que vous avez des comptes sur ces plateformes.

<div id="prerequisites">
  ## Prérequis
</div>

Configurez la [connexion de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db).

1. Créez une nouvelle connexion de base de données dans [Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database). Assurez-vous de sélectionner la vue **Custom Database** et d’activer le commutateur **Use my own database**.
2. Connectez la base de données à l’application : dans les paramètres de votre connexion de base de données, sélectionnez la vue **Applications**. Repérez la section **Applications Using This Connection** et activez la connexion de base de données pour chaque application.

<div id="scenario-1-migrate-users-from-gigya-to-auth0">
  ## Scénario 1 : Migrer des utilisateurs de Gigya vers Auth0
</div>

1. Exportez les utilisateurs de Gigya :
   Utilisez [IdentitySync de Gigya](https://help.sap.com/docs/SAP_CUSTOMER_DATA_CLOUD/8b8d6fffe113457094a17701f63e3d6a/2b717202e42a4165aa7f75f7eecab64f.html) pour transformer et exporter les données utilisateur afin qu’elles correspondent à un schéma cible. Pour en savoir plus sur ce processus, consultez [Gigya IdentitySync: Using IdentitySync](https://help.sap.com/docs/SAP_CUSTOMER_DATA_CLOUD/8b8d6fffe113457094a17701f63e3d6a/2b717202e42a4165aa7f75f7eecab64f.html?locale=en-US).<br /><br />

   Suivez les instructions pour convertir les données utilisateur de votre base de données Gigya selon le bon [schéma](/docs/fr-CA/manage-users/user-migration/bulk-user-import-database-schema-and-examples), puis exportez les données converties au format JSON.
2. Importez vos utilisateurs de Gigya dans Auth0 :
   Vous pouvez importer des utilisateurs au moyen de la [User Import/Export Extension](/docs/fr-CA/manage-users/user-migration/user-import-export-extension) ou de la [Management API](https://auth0.com/docs/api/management/v2).

   * **User Import/Export Extension** : Accédez à [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions), sélectionnez l’extension **User Import / Export** et installez-la. Une fois l’extension installée, vous pouvez cliquer dessus pour ouvrir l’interface d’importation et d’exportation.<br /><br />

     Faites glisser le fichier JSON exporté de vos utilisateurs Gigya dans la zone de téléversement prévue à cet effet, puis sélectionnez la base de données que vous avez créée plus tôt. Sélectionnez **Commencer l’importation des utilisateurs** pour démarrer l’importation. Pour en savoir plus, consultez [User Import/Export Extension](/docs/fr-CA/manage-users/user-migration/user-import-export-extension).
   * **Management API** : [Créez un job](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports) pour importer vos utilisateurs dans Auth0. Pour obtenir des instructions détaillées, consultez [Bulk User Imports](/docs/fr-CA/manage-users/user-migration/bulk-user-imports).

<div id="scenario-2-migrate-users-from-okta-to-auth0">
  ## Scénario 2 : Migrer les utilisateurs d’Okta vers Auth0
</div>

1. Activez l’importation des utilisateurs : allez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) et sélectionnez votre connexion de base de données. Dans **Settings**, activez l’option **Import Users to Auth0**.
2. Créez le [script Login](/docs/fr-CA/authenticate/database-connections/custom-db/templates/login) : le script **Login** s’exécute lorsqu’un utilisateur tente de se connecter, mais que son compte est introuvable dans la base de données Auth0. Vous devez créer un script qui appellera l’endpoint Okta [Primary Authentication](https://developer.okta.com/docs/api/resources/authn.html#primary-authentication), en transmettant l’adresse courriel et le mot de passe dans les paramètres `username` et `password`.
   Après une authentication réussie, Okta renverra un [objet de transaction d’authentication](https://developer.okta.com/docs/api/resources/authn.html#authentication-transaction-model), qui contient le [profil de l’utilisateur](https://developer.okta.com/docs/api/resources/authn.html#user-profile-object) dans les [ressources intégrées](https://developer.okta.com/docs/api/resources/authn.html#embedded-resources). Vous pourrez ensuite extraire les renseignements de l’utilisateur et les transmettre à Auth0 dans la fonction de rappel.<br /><br />

   Dans la vue **Base de données personnalisée** de votre connexion à la base de données, repérez **Scripts d’action de base de données** et sélectionnez **Connexion**.

   ```javascript lines expandable
   function login (email, password, callback) {
     // Remplacez {yourOktaDomain} par votre propre domaine Okta
     var url = 'https:/{yourOktaDomain}/api/v1/authn';

     // Effectuez la requête POST pour authentifier un utilisateur
     request({
       url: url,
       method: 'POST',
       headers: {
         'Content-Type': 'application/json',
         'Accept': 'application/json'
       },
       body: {
         username: email,
         password: password,
         options: {
           multiOptionalFactorEnroll: false,
           warnBeforePasswordExpired: false
         }
       },
       json: true
     }, function (error, response, body) {
       // Vérifiez que la réponse est réussie
       if (response.statusCode !== 200) return callback();

       // Récupérez l'utilisateur depuis le corps de la réponse
       var user = body._embedded.user;

       // Définissez les données à stocker dans Auth0 et migrez l'utilisateur
       return callback(null, {
           user_id : user.id,
           username: user.profile.login,
           email: user.profile.login,
           // Nous définissons email_verified à true pour les utilisateurs, car nous supposons que s'ils étaient
           // des utilisateurs valides dans Okta, ils ont déjà vérifié leur adresse courriel
           // Si ce champ n'est pas défini, l'utilisateur recevra un courriel lui demandant de vérifier
           // son compte
           email_verified: true,
           name: user.profile.firstName + ' ' + user.profile.lastName
         });
     });
   }
   ```

   Cliquez sur le bouton **Try** au-dessus du script pour vérifier que le script fonctionne.
3. Créez le [script Get User](/docs/fr-CA/authenticate/database-connections/custom-db/templates/get-user) : le script **Get User** s’exécute lorsque l’utilisateur tente de réinitialiser son mot de passe, mais que son compte est introuvable dans la base de données Auth0. Vous devrez créer un script qui appelle l’endpoint Okta [Get User with Login](https://developer.okta.com/docs/api/resources/users.html#get-user-with-login), en transmettant l’adresse courriel de l’utilisateur comme paramètre `login`.
   Vous devrez aussi [créer un token d’API](https://developer.okta.com/docs/api/getting_started/getting_a_token.html), qui doit être transmis à cet endpoint dans l’en-tête `Authorization`.
   Si la requête réussit, Okta renverra un [objet `user`](https://developer.okta.com/docs/api/resources/users.html#user-model) contenant les renseignements de l’utilisateur. Encore une fois, vous pouvez extraire les renseignements de l’utilisateur et les transmettre à Auth0 dans la fonction de rappel.<br /><br />

   Dans la vue **Base de données personnalisée** de votre connexion à la base de données, repérez **Scripts d’action de la base de données** et sélectionnez **Récupérer l’utilisateur**.

   ```javascript lines expandable
   function getByEmail(email, callback) {
     // Remplacez {yourOktaDomain} par votre propre domaine Okta
     var url = 'https://{yourOktaDomain}/api/v1/users/' + encodeURIComponent(email);

     // Effectuez une requête GET pour trouver un utilisateur par courriel
     // Remplacez {yourOktaApiToken} par un jeton API Okta 
     // (voir https://developer.okta.com/docs/api/getting_started/getting_a_token.html) 
     request({
       url: url,
       method: 'GET',
       headers: {
         'Content-Type': 'application/json',
         'Accept': 'application/json',
         'Authorization': 'SSWS ' + '{yourOktaApiToken}'
       },
       json: true
     }, function (error, response, body) {
       // Vérifiez que la réponse est bien réussie
       if (response.statusCode !== 200) return callback();

       // Définissez les données à stocker dans Auth0 et migrez l'utilisateur
       return callback(null, {
         user_id: body.id,
         username: body.profile.login,
         email: body.profile.login,
         email_verified: true,
         name: body.profile.firstName + ' ' + body.profile.lastName
       });
     });
   }
   ```

   Cliquez sur le bouton **Try** au-dessus du script pour tester si le script fonctionne.
4. Testez la connexion de base de données personnalisée : cliquez sur **Try connection**. Le [widget Auth0 Lock](/docs/fr-CA/libraries/lock) s’affichera. Entrez l’adresse courriel et le mot de passe de l’utilisateur Okta, puis cliquez sur **Log In.** Vous devriez voir une page Web indiquant que la connexion fonctionne, accompagnée de renseignements sur l’utilisateur.
5. Pour voir les utilisateurs nouvellement importés, accédez à [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).

<div id="scenario-3-migrate-users-from-stormpath-to-auth0">
  ## Scénario 3 : Migrer les utilisateurs de Stormpath vers Auth0
</div>

1. Activez l’importation des utilisateurs : Accédez à [Auth0 Dashboard &gt; Authentification &gt; Database](https://manage.auth0.com/#/connections/database) et sélectionnez votre connexion de base de données. Dans **Settings**, activez l’option **Import Users to Auth0**.
2. Activez les applications pour votre connexion : Sélectionnez la vue **Applications**. Activez les applications qui utiliseront la connexion de base de données.
3. Créez le [script Login](/docs/fr-CA/authenticate/database-connections/custom-db/templates/login) : le script **Login** s’exécute lorsqu’un utilisateur tente de se connecter, mais que son compte est introuvable dans la base de données Auth0. Vous devrez créer un script qui appelle l’API de Stormpath pour authentifier l’utilisateur en transmettant ses identifiants comme paramètres `username` et `password`.<br /><br />

   Après une authentification réussie, la réponse de Stormpath inclura l’URL du compte d’utilisateur. Envoyez une deuxième requête à l’URL du compte pour récupérer les informations de l’utilisateur. Vous pourrez ensuite extraire les informations de l’utilisateur et les transmettre à Auth0 dans la fonction de rappel.

   ```javascript lines expandable
   function login(username, password, callback) {
     // Remplacez l'attribut {yourStormpathClientId} par votre ID Stormpath
     var url = 'https://api.stormpath.com/v1/applications/{yourStormpathClientId}/loginAttempts';

     // Stormpath exige que les informations d'identification de l'utilisateur soient transmises sous forme de message encodé en base64
     var message = username + ':' + password;
     var pass = new Buffer(message).toString('base64');

     // Ici, nous effectuons la requête POST pour authentifier un utilisateur
     request({
       url: url,
       method: 'POST',
       auth: {
         // Votre ID client API
         user: '{yourStormpathClientId}',
         // Votre secret client API
         password: '{yourStormpathClientSecret}'
       },
       headers: {
         'Content-Type': 'application/json'
       },
       json: {
         type: 'basic',
         // Transmission des informations d'identification encodées en base64
         value: pass
       }
     }, function (error, response, body) {
       // Si la réponse est réussie, on continue
       if (response.statusCode !== 200) return callback();
       // Une réponse réussie retourne une URL pour récupérer les informations de l'utilisateur
       var accountUrl = body.account.href;

       // On effectue une deuxième requête pour obtenir les informations de l'utilisateur. Cette fois, ce sera une requête GET
       request({
         url: accountUrl,
         method: 'GET',
         auth: {
           // Votre ID client API
           user: '{yourStormpathClientId}',
           // VOTRE secret client API
           password: '{yourStormpathClientSecret}'
         }
       }, function (errorUserInfo, responseUserInfo, bodyUserInfo) {
         // Si on obtient une réponse réussie, on la traite
         if (responseUserInfo.statusCode !== 200) return callback();

         var parsedBody = JSON.parse(bodyUserInfo);
         // Pour obtenir l'identifiant de l'utilisateur, on retire la partie de l'API Stormpath
         var id = parsedBody.href.replace('https://api.stormpath.com/v1/accounts/', '');

         // Enfin, on définit les données à stocker dans Auth0 et on migre l'utilisateur
         return callback(null, {
           user_id : id,
           username: parsedBody.username,
           email: parsedBody.email,
           // On définit email_verified à true pour les utilisateurs, en supposant que s'ils étaient
           // des utilisateurs valides dans Stormpath, ils ont déjà vérifié leur courriel
           // Si ce champ n'est pas défini, l'utilisateur recevra un courriel lui demandant de vérifier
           // son compte
           email_verified: true,
           // Ajoutez tout champ supplémentaire que vous souhaitez transférer depuis Stormpath
         });
       });
     });
   }
   ```

   Cliquez sur le bouton **Try** au-dessus du script pour le tester et vérifier s’il fonctionne.
4. Créez le [script Get User](/docs/fr-CA/authenticate/database-connections/custom-db/templates/get-user) : le script **Get User** est exécuté lorsque l’utilisateur tente de réinitialiser son mot de passe, mais que son compte est introuvable dans la base de données Auth0. Vous devrez créer un script qui appellera l’endpoint `/accounts` de l’API Stormpath, en transmettant l’adresse courriel de l’utilisateur comme paramètre `email`.
   En cas de réussite, Stormpath renverra les renseignements de l’utilisateur. Vous pourrez extraire ces renseignements et les transmettre à Auth0 dans la fonction de rappel.

   ```javascript lines expandable
   function getByEmail(email, callback) {
     // Remplacez l'attribut {yourStormpathClientId} par votre ID Stormpath
     var url = 'https://api.stormpath.com/v1/applications/{yourStormpathClientId}/accounts';

     request({
       url: url,
       method: 'GET',
       auth: {
         // Votre ID client API
         user: '{yourStormpathClientId}',
         // VOTRE Secret client API
         password: '{yourStormpathClientSecret}'
       },
       qs: { q: email }
     }, function (error, response, body) {
       if (response.statusCode !== 200) return callback();

       var parsedBody = JSON.parse(body);
       var user = parsedBody.items[0];

       if (!user) return callback();

       var id = user.href.replace('https://api.stormpath.com/v1/accounts/', '');

       return callback(null, {
         user_id: id,
         username: user.username,
         email: user.email,
         email_verified: true,
         // Ajoutez tout champ supplémentaire que vous souhaitez conserver depuis Stormpath
       });
     });
   }
   ```

   Cliquez sur le bouton **Try** au-dessus du script pour tester le script et vérifier s’il fonctionne.
5. Testez la connexion de base de données personnalisée : cliquez sur **Try connection**. Le [widget Auth0 Lock](/docs/fr-CA/libraries/lock) s’affichera. Entrez l’adresse courriel et le mot de passe de l’utilisateur Stormpath, puis cliquez sur **Log In.** Vous devriez voir une page Web indiquant que la connexion fonctionne, avec des renseignements sur l’utilisateur.
6. Pour voir les utilisateurs nouvellement importés, accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
