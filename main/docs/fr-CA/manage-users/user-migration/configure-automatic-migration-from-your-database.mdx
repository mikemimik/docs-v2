title: Configurer la migration automatique à partir de votre base de données
---

Configurez votre connexion Database pour migrer automatiquement vos utilisateurs d’un répertoire d’utilisateurs externe vers Auth0 à l’aide de [scripts d’action de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db/templates).

<Warning>
  Si vous essayez d’utiliser plus d’une méthode de migration (par exemple, la migration automatique, puis l’importation en bloc d’utilisateurs), vous pourriez obtenir une erreur `DUPLICATED_USER`. Cette erreur indique que l’utilisateur existe dans le répertoire d’utilisateurs interne d’Auth0, mais pas dans votre tenant. Pour corriger cette erreur, supprimez l’utilisateur à l’aide de l’endpoint [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de l’Auth0 Management API, puis réessayez l’importation.
</Warning>

<div id="configure-database-connection-as-custom">
  ## Configurer une connexion Database en mode personnalisé
</div>

Créez une connexion Database, puis définissez-la en mode personnalisé.

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) et sélectionnez la base de données à consulter.
2. Sélectionnez la vue **Custom Database**, puis activez la bascule **Use my own database**.

   <Frame>![Auth0 Dashboard Authentication Database Connection Custom Database Settings Use Own Database Enabled](/docs/images/cdy7uua7fh8z/11HPAdVwJMmnWbzMVjHCJ8/f75ff47f613a41778f266643f29ab2b1/2025-02-27_17-25-36.png)</Frame>
3. Sélectionnez la vue **Settings**, activez la bascule **Import Users to Auth0**, puis sélectionnez **Save**.

<div id="configure-database-action-scripts">
  ## Configurer Database Action Scripts
</div>

Sélectionnez la vue **Custom Database**, puis repérez **Database Action Scripts**.

<Frame>![Dashboard Authentication Database Connection Custom Database tab Database Action Scripts](/docs/images/cdy7uua7fh8z/4LBIvvjaABo51It4eVCmX3/4ef14765c79cc5b7ecfa3f3155caa9eb/Screen_Shot_2021-05-18_at_8.54.55_PM.png)</Frame>

<div id="login">
  ### Login
</div>

Le script [Login](/docs/fr-CA/authenticate/database-connections/custom-db/templates/login) s’exécute chaque fois qu’un utilisateur qu’Auth0 ne trouve pas tente de se connecter. Il vérifie que l’utilisateur existe dans le répertoire d’utilisateurs externe sans lui redemander son mot de passe.

<div id="get-user">
  ### Get User
</div>

Le script [Get User](/docs/fr-CA/authenticate/database-connections/custom-db/templates/get-user) s’exécute dans l’un ou l’autre des scénarios suivants :

* Un utilisateur lance une tentative d’inscription.
* Un utilisateur lance un processus de changement de mot de passe.
* Un utilisateur saisit son identifiant de connexion lorsqu’il utilise Identifier First + Biometrics.
* Le point de terminaison [Update a User&#39;s Email or Username](https://auth0.com/docs/api/v2#!/Users/patch_users_by_id) ou [Create User](https://auth0.com/docs/api/management/v2#!/Users/post_users) de la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> est appelé.

Si un utilisateur qui n’a pas encore été migré confirme un changement de mot de passe et se connecte avec succès, son profil d’utilisateur sera créé dans Auth0 avec le nouveau mot de passe. Ce profil d’utilisateur contient toutes les données renvoyées par le script Get User. Toutes les connexions ultérieures de cet utilisateur se feront directement dans Auth0.

<div id="verify-user-migration-is-complete">
  ## Vérifiez que la migration des utilisateurs est terminée
</div>

Vérifiez que tous les utilisateurs ont été migrés vers le répertoire d’utilisateurs d’Auth0 à l’aide du <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou à l’aide de l’Auth0 Management API.

1. Accédez à [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users), puis passez en revue la liste des utilisateurs.
2. Utilisez le endpoint [List or Search Users](https://auth0.com/docs/api/v2#!/Users/get_users) de la Management API.

<div id="disconnect-external-user-store">
  ## Déconnecter le référentiel d’utilisateurs externe
</div>

Reconfigurez les Database Action Scripts en fonctions vides. Cela empêchera Auth0 d’interroger votre référentiel d’utilisateurs externe pour authentifier les utilisateurs.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Assurez-vous de laisser l’option **Import Users to Auth0** activée. Si vous désactivez cette option, Auth0 utilisera uniquement les scripts pour authentifier les utilisateurs et effectuer d’autres actions liées aux utilisateurs, au lieu d’utiliser les utilisateurs importés localement.
</Callout>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database), puis sélectionnez votre connexion Database.

2. Passez à la vue **Custom Database**, puis repérez **Database Action Scripts.**

3. Mettez à jour le script **Login**.

   ```javascript lines
   function login (email, password, callback) {
     return callback(null, null);
   }
   ```

4. Mettez à jour le script **Get User**.

   ```javascript lines
   function getByEmail (email, callback) {
     return callback(null, null);
   }
   ```

<div id="troubleshoot-user-migration-issues">
  ## Résoudre les problèmes liés à la migration des utilisateurs
</div>

Si vous rencontrez des problèmes avec la migration automatique, consultez d’abord les [bonnes pratiques relatives aux connexions de base de données personnalisées et aux scripts d’action](/docs/fr-CA/authenticate/database-connections/custom-db/custom-database-connections-scripts).

<div id="user-already-exists">
  ### L’utilisateur existe déjà
</div>

Voici les scénarios les plus courants dans lesquels vous pourriez rencontrer ce problème :

* Vous supprimez un utilisateur de votre connexion Database, puis vous essayez de recréer cet utilisateur.
* Vous tentez de créer un nouvel utilisateur dans votre connexion Database alors que l’utilisateur existe déjà dans votre répertoire d’utilisateurs externe.
* Le processus de migration de l’utilisateur a été interrompu.

Pendant le processus de migration de l’utilisateur, Auth0 crée d’abord un profil utilisateur partiel dans un répertoire d’utilisateurs interne, puis crée un profil utilisateur complet dans votre connexion Database. Si un problème survient et empêche la création de ce profil utilisateur complet, vous pourriez obtenir l’erreur `The user already exists`.

Vous pouvez généralement résoudre ce problème en supprimant l’utilisateur de la connexion Database, en supprimant l’utilisateur du répertoire d’utilisateurs interne, puis en relançant le processus de migration.

1. Vérifiez les instructions `console.log()` dans les [Actions Real-time Logs](/docs/fr-CA/customize/actions/actions-real-time-logs).
2. Supprimez l’utilisateur à l’aide du point de terminaison [Delete a User](https://auth0.com/docs/api/management/v2#!/Users/delete_users_by_id) de la Management API.
3. Supprimez l’utilisateur à l’aide du point de terminaison [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de la Management API.
4. Demandez à l’utilisateur de se connecter ou de changer son mot de passe pour relancer la migration.

<div id="missing-metadata">
  ### Métadonnées manquantes
</div>

Si un utilisateur est interrompu pendant le flux de connexion ou de changement de mot de passe qui lance le processus de migration, Auth0 pourrait ne pas être en mesure de transférer ses métadonnées (`user_metadata` ou `app_metadata`) avec ses autres données de profil.

Vous pouvez atténuer ce scénario en créant une [Action](/docs/fr-CA/customize/actions) qui vérifie si le profil de l’utilisateur ne contient pas de métadonnées, les récupère depuis le magasin d’utilisateurs externe, puis les stocke dans Auth0.
