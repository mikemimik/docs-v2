description: Découvrez les meilleures pratiques pour la recherche d’utilisateurs dans Auth0
title: Meilleures pratiques pour la recherche d’utilisateurs
---

Voici quelques bonnes pratiques pour la recherche d’utilisateurs :

* Vous avez besoin d’un jeton pour envoyer des requêtes à la <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Pour en savoir plus, consultez [Jetons d’accès de la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens).
* Pour effectuer des requêtes de recherche d’utilisateurs, la portée `read:users` est requise.
* Tirez parti de la recherche non déterministe : si votre application n’a pas besoin d’un ordre stable pour les résultats de recherche et que vos requêtes sont assez courtes pour ne pas dépendre de la pagination, définissez `primary_order=false` afin d’améliorer les performances des requêtes.
* Pour obtenir les résultats de recherche les plus récents, utilisez un endpoint à **cohérence immédiate** pendant les processus d’authentification, comme [Get Users by ID](/docs/fr-CA/manage-users/user-search/retrieve-users-with-get-users-by-id-endpoint) et [Get Users by Email](/docs/fr-CA/manage-users/user-search/retrieve-users-with-get-users-by-email-endpoint). Les recherches effectuées au moyen de ces endpoints reflètent les résultats de toutes les opérations d’écriture réussies, y compris celles survenues peu avant votre requête.
* Utilisez un schéma bien connu pour les métadonnées :

  * Utilisez des types de données cohérents pour les propriétés.
  * Évitez les noms de propriété dynamiques.
  * Évitez les schémas trop volumineux et les structures trop profondes.
  * Évitez de stocker des données dont vous n’avez pas besoin à des fins d’authentification et d’autorisation.
* Les requêtes de recherche expirent (code d’état HTTP 503) si elles ne se terminent pas en deux secondes ou moins. Les requêtes qui prennent plus de temps indiquent soit qu’elles sont coûteuses, soit qu’elles contiennent une erreur qui les empêche de se terminer rapidement.
* Les requêtes de recherche portant sur des attributs définis par l’utilisateur dans `app_metadata` et `user_metadata` peuvent expirer. Nous vous conseillons de ne pas utiliser [List or search users](https://auth0.com/docs/api/management/v2/users/get-users) pour des opérations critiques.
* N’utilisez pas de critère de recherche qui retourne un grand volume de données (plus de 1000 résultats).
* N’utilisez pas de requêtes d’existence (par exemple, « donnez-moi tous les utilisateurs qui ont une propriété, peu importe sa valeur »).
* N’interrogez pas les API de recherche en boucle.
* N’effectuez pas de requêtes de recherche d’utilisateurs dans les points d’extension du flux de connexion, comme les Rules ou les Actions `post-login`.
* N’utilisez pas de champs de métadonnées volumineux (essayez de limiter les champs de métadonnées à 2 KB ou moins).
* L’utilisation de caractères génériques dans les recherches peut nuire aux performances. Dans certains cas, les recherches avec caractères génériques sur de grands ensembles de données peuvent entraîner des erreurs d’expiration. Nous recommandons également d’éviter les caractères génériques au début du terme de recherche; leur utilisation en suffixe offre de meilleures performances.
* Échappez le caractère d’espacement pour améliorer les performances (p. ex., `q=name:John Doe` devrait s’écrire `q=name:John\ Doe`).
