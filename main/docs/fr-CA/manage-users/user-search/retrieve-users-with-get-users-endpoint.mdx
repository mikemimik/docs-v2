L’[`point de terminaison GET /api/v2/users`](https://auth0.com/docs/api/management/v2#!/Users/get_users) vous permet de récupérer une liste d’utilisateurs. Avec ce point de terminaison, vous pouvez :

* Effectuer une recherche selon divers critères
* Sélectionner les champs à renvoyer
* Trier les résultats renvoyés

Ce point de terminaison est **à cohérence éventuelle**; par conséquent, nous vous recommandons de l’utiliser pour des processus administratifs, comme la modification du nom d’affichage d’un utilisateur existant.

<div id="request-example">
  ## Exemple de requête
</div>

Pour rechercher des utilisateurs, effectuez une requête `GET` vers le [point de terminaison `/api/v2/users`](https://auth0.com/docs/api/management/v2#!/Users/get_users). La requête doit inclure un [jeton d’accès à la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens). Passez votre requête de recherche dans le paramètre `q` et définissez le paramètre `search_engine` sur `v3`.

Par exemple, pour rechercher un utilisateur dont l’adresse courriel est exactement `jane@exampleco.com`, utilisez `q=email:"jane@exampleco.com"` :

  curl --request GET \
    --url 'https://{yourDomain}/api/v2/users?q=email%3A%22jane%40exampleco.com%22&search_engine=v3' \
    --header 'authorization: Bearer {yourMgmtApiAccessToken}'
  ```

  IRestResponse response = client.Execute(request);
  ```

<div id="query-examples">
  ## Exemples de requêtes
</div>

Voici quelques exemples de requêtes que vous pouvez effectuer avec la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

| Cas d’utilisation                                                                                                                                         | Requête                                                                    |
| --------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Rechercher tous les utilisateurs dont le nom contient &quot;john&quot;                                                                                    | `name:\*john\*`                                                            |
| Rechercher tous les utilisateurs dont le nom est exactement &quot;jane&quot;                                                                              | `name:"jane"`                                                              |
| Rechercher tous les noms d’utilisateur qui commencent par &quot;john&quot;                                                                                | `name:john*`                                                               |
| Rechercher les noms d’utilisateur qui commencent par &quot;jane&quot; et se terminent par &quot;smith&quot;                                               | `name:jane*smith`                                                          |
| Rechercher tous les utilisateurs dont l’adresse courriel est exactement &quot;john@exampleco.com&quot;                                                    | `email:"john@exampleco.com"`                                               |
| Rechercher tous les utilisateurs dont l’adresse courriel est exactement &quot;john@exampleco.com&quot; ou &quot;jane@exampleco.com&quot; à l’aide de `OR` | `email:("john@exampleco.com" OR "jane@exampleco.com")`                     |
| Rechercher les utilisateurs sans adresse courriel vérifiée                                                                                                | `email_verified:false OR NOT \_exists_\:email_verified`                    |
| Rechercher les utilisateurs ayant le champ `user_metadata` nommé `full_name` avec la valeur &quot;John Smith&quot;                                        | `user_metadata.full_name:"John Smith"`                                     |
| Rechercher les utilisateurs d’une connexion spécifique                                                                                                    | `identities.connection:"google-oauth2"`                                    |
| Rechercher tous les utilisateurs qui ne se sont jamais connectés                                                                                          | `(NOT \_exists_\:logins_count OR logins_count:0)`                          |
| Rechercher tous les utilisateurs qui se sont connectés avant 2018                                                                                         | `last_login:[* TO 2017-12-31]`                                             |
| Rechercher tous les utilisateurs dont la dernière connexion remonte à décembre 2017                                                                       | `last_login:[2017-11 TO 2017-12]`, `last_login:[2017-12-01 TO 2017-12-31]` |
| Rechercher tous les utilisateurs ayant un nombre de connexions &gt;= 100 et &lt;= 200                                                                     | `logins_count:[100 TO 200]`                                                |
| Rechercher tous les utilisateurs ayant un nombre de connexions &gt;= 100                                                                                  | `logins_count:[100 TO *]`                                                  |
| Rechercher tous les utilisateurs ayant un nombre de connexions &gt; 100 et &lt; 200                                                                       | `logins_count:\{100 TO 200}`                                               |
| Rechercher tous les utilisateurs dont le domaine de l’adresse courriel est &quot;exampleco.com&quot;                                                      | `email.domain:"exampleco.com"`                                             |

<div id="limitations">
  ## Limites
</div>

* Le point de terminaison renvoie un maximum de 50 utilisateurs, même si davantage d’utilisateurs correspondent à votre requête.
* Si vous devez renvoyer plus de 50 utilisateurs, utilisez le paramètre `page` pour afficher plus de pages de résultats. Chaque page contient 50 utilisateurs. Par exemple, vous pouvez préciser `&page=2` pour afficher les résultats 51 à 100, préciser `&page=3` pour afficher les résultats 101 à 150, et ainsi de suite. Cependant, ce point de terminaison ne renvoie jamais plus de 1 000 utilisateurs au total avec les mêmes critères de recherche, même avec la pagination.
* Une limite de 1 Mo par utilisateur s’applique aux données utilisateur pouvant être indexées, interrogées et renvoyées par le [point de terminaison de recherche d’utilisateurs](https://auth0.com/docs/api/management/v2/users/get-users). Pour en savoir plus sur l’incidence de cette limite sur les métadonnées personnalisées de plus de 1 Mo, consultez [noms de champs et types de données des métadonnées](/docs/fr-CA/manage-users/user-accounts/metadata/metadata-fields-data#size-limits-and-storage). Le [point de terminaison Get User](https://auth0.com/docs/api/management/v2/users/get-users-by-id) doit être utilisé pour récupérer tous les attributs utilisateur des profils utilisateur surdimensionnés.
* Par défaut, le point de terminaison `GET /api/v2/users` renvoie les résultats dans un ordre déterministe afin qu’une même requête produise chaque fois des résultats logiquement ordonnés de la même façon. Ce comportement est contrôlé par le paramètre de requête `primary_order` :

  * `primary_order=true` (par défaut) : les résultats de requêtes identiques sont ordonnés de façon cohérente.
  * `primary_order=false` : les résultats sont renvoyés dans un ordre non déterministe, ce qui peut améliorer les performances pour les requêtes complexes.
* Si votre application n’exige pas un ordre cohérent des résultats de recherche et que vos requêtes sont assez courtes pour ne pas dépendre de la pagination, définissez `primary_order=false` pour améliorer les performances des requêtes.
* Si vous avez besoin d’une exportation complète de tous vos utilisateurs, utilisez le [job d’exportation](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_exports) ou l’extension [User Import / Export](/docs/fr-CA/customize/extensions/user-import-export-extension).
* Si vous obtenez l’erreur `414 Request-URI Too Large`, cela signifie que votre chaîne de requête dépasse la longueur prise en charge. Dans ce cas, affinez votre recherche.

Nous ne recommandons **pas** d’utiliser ce point de terminaison pour :

* Les opérations qui exigent une cohérence immédiate. Utilisez plutôt le [point de terminaison Get Users by Email](/docs/fr-CA/manage-users/user-search/retrieve-users-with-get-users-by-email-endpoint) ou le [point de terminaison Get Users by ID](/docs/fr-CA/manage-users/user-search/retrieve-users-with-get-users-by-id-endpoint).
* Les exportations d’utilisateurs. Utilisez plutôt le [point de terminaison User Export](/docs/fr-CA/manage-users/user-migration/bulk-user-exports).
* Les opérations qui exigent la recherche d’utilisateurs dans le cadre de processus d’authentification. Utilisez plutôt le point de terminaison Get Users by Email ou le point de terminaison Get Users by ID.
* La recherche d’utilisateurs pour la [liaison de comptes](/docs/fr-CA/manage-users/user-accounts/user-account-linking) par courriel. Utilisez plutôt le point de terminaison Get Users by Email.
