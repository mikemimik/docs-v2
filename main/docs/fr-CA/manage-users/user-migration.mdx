<Warning>
  L’**extension User Import/Export d’utilisateurs** sera abandonnée en septembre 2025. Vous pouvez accéder aux mêmes fonctionnalités désormais offertes [directement dans le tableau de bord de gestion Auth0](/docs/fr-CA/manage-users/user-migration/bulk-user-import-export).
</Warning>

Auth0 prend en charge l’importation d’utilisateurs à partir d’applications externes à l’aide de connexions de base de données personnalisées, de l’Auth0 <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> ou de l’extension User Import/Export d’utilisateurs.

<div id="automatic-migrations">
  ## Migrations automatiques
</div>

Auth0 prend en charge la migration automatique des utilisateurs d’une [connexion de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db) vers Auth0. Cette pratique est parfois appelée **migration progressive** ou **migration paresseuse**. En activant cette fonctionnalité, vos utilisateurs :

* Sont migrés vers Auth0 la première fois qu’ils se connectent après que vous avez configuré l’intégration.
* N’ont pas à réinitialiser leur mot de passe en raison de la migration.

<Card title="La disponibilité varie selon le forfait Auth0">
  La disponibilité de cette fonctionnalité dépend à la fois de votre mise en œuvre de la connexion et de votre forfait Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Card>

Lorsqu’un utilisateur s’authentifie au moyen d’une connexion de base de données personnalisée configurée pour l’importation vers Auth0, le processus suivant a lieu :

* Auth0 authentifie les utilisateurs migrés à l’aide de la base de données Auth0.
* Si l’utilisateur n’a pas encore été migré, Auth0 exécute votre script Login personnalisé et, une fois l’authentification réussie, ajoute l’utilisateur à la base de données Auth0.
* Lors des connexions suivantes, les informations d’identification de l’utilisateur sont récupérées dans Auth0, **PAS** dans votre base de données personnalisée.
* Les nouveaux utilisateurs sont automatiquement ajoutés à la base de données Auth0.

<div id="bulk-user-imports-with-the-management-api">
  ## Importations en bloc d’utilisateurs avec la Management API
</div>

Si vous avez déjà une base de données d’utilisateurs, vous pouvez utiliser le point de terminaison de la Management API [`/post_users_imports`](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports) pour alimenter une connexion de base de données avec ces données.

<div id="migrate-users-with-the-user-importexport-feature">
  ## Migrer des utilisateurs avec la fonctionnalité User Import/Export
</div>

La fonctionnalité User Import/Export vous permet de :

* Importer en bloc les utilisateurs existants de votre base de données dans Auth0.
* Rechercher et exporter une partie (ou la totalité) de vos utilisateurs de base de données dans Auth0.

Pour utiliser l’importation en bloc, vous devez avoir le rôle Editor - Users dans le Dashboard ou le rôle Admin. Pour utiliser l’exportation en bloc, vous devez avoir le rôle Viewer - Users, Editor - Users ou Admin dans le Dashboard.

Vous pouvez importer et exporter des données utilisateur à l’aide de la [fonctionnalité User Import/Export](/docs/fr-CA/manage-users/user-migration/bulk-user-import-export) offerte dans le Dashboard.
