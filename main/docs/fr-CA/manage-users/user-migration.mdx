Auth0 prend en charge l’importation d’utilisateurs depuis des applications externes au moyen de connexions de base de données personnalisées, de l’Auth0 <Tooltip tip="Management API : produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> ou de l’extension User Import/Export.

<div id="automatic-migrations">
  ## Migrations automatiques
</div>

Auth0 prend en charge la migration automatique des utilisateurs d’une [connexion de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db) vers Auth0. Cette pratique est parfois appelée **migration progressive** ou **migration paresseuse**. En activant cette fonctionnalité, vos utilisateurs :

* Sont migrés vers Auth0 la première fois qu’ils se connectent après la mise en place de l’intégration.
* N’ont pas à réinitialiser leur mot de passe à la suite de la migration.

<Card title="La disponibilité varie selon le forfait Auth0">
  Votre mise en œuvre de connexion, ainsi que votre forfait Auth0 ou votre entente personnalisée, déterminent si cette fonctionnalité est offerte. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Card>

Lorsqu’un utilisateur s’authentifie au moyen d’une connexion de base de données personnalisée marquée pour importation vers Auth0, le processus suivant a lieu :

* Auth0 authentifie les utilisateurs migrés à l’aide de la base de données Auth0.
* Si l’utilisateur n’a pas encore été migré, Auth0 exécute votre script Login personnalisé et, une fois la connexion réussie, ajoute l’utilisateur à la base de données Auth0.
* Lors des connexions suivantes, les identifiants de l’utilisateur sont récupérés dans Auth0, **PAS** dans votre base de données personnalisée.
* Les nouveaux utilisateurs sont automatiquement ajoutés à la base de données Auth0.

<div id="bulk-user-imports-with-the-management-api">
  ## Importations en bloc d’utilisateurs avec la Management API
</div>

Si vous avez déjà une base de données d’utilisateurs, vous pouvez utiliser notre [`/post_users_imports`](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports) point de terminaison de la Management API pour importer ces données dans une connexion de base de données.

<div id="migrate-users-with-the-user-importexport-feature">
  ## Migrer des utilisateurs à l’aide de la fonctionnalité User Import/Export
</div>

La fonctionnalité User Import/Export vous permet de :

* Importer en bloc vos utilisateurs de base de données existants dans Auth0.
* Rechercher et exporter une partie (ou la totalité) de vos utilisateurs de base de données Auth0.

Pour utiliser l’importation en bloc, vous devez avoir le rôle Dashboard Editor - Users ou Admin. Pour utiliser l’exportation en bloc, vous devez avoir le rôle Dashboard Viewer - Users, Editor - Users ou Admin.

Vous pouvez importer et exporter des données d’utilisateur à l’aide de la [fonctionnalité User Import/Export](/docs/fr-CA/manage-users/user-migration/bulk-user-import-export) disponible dans le Dashboard.
