Auth0 prend en charge l’importation d’utilisateurs à partir d’applications externes au moyen de connexions de base de données personnalisées, de l’Auth0 <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> ou de l’extension d’importation/exportation d’utilisateurs.

<div id="automatic-migrations">
  ## Migrations automatiques
</div>

Auth0 prend en charge la migration automatique des utilisateurs d’une [connexion de base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db) vers Auth0. Cette pratique est parfois appelée **migration progressive** ou **migration paresseuse**. En activant cette fonctionnalité, vos utilisateurs :

* sont migrés vers Auth0 la première fois qu’ils se connectent après la configuration de l’intégration.
* n’ont pas à réinitialiser leur mot de passe à cause de la migration.

<Card title="La disponibilité varie selon le forfait Auth0">
  La disponibilité de cette fonctionnalité dépend à la fois de votre mise en œuvre de connexion et de votre forfait Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Tarification](https://auth0.com/pricing).
</Card>

Lorsqu’un utilisateur s’authentifie au moyen d’une connexion de base de données personnalisée marquée pour l’importation vers Auth0, le processus suivant a lieu :

* Auth0 authentifie les utilisateurs migrés auprès de la base de données Auth0.
* Si l’utilisateur n’a pas encore été migré, Auth0 exécute votre script Login personnalisé et, si la connexion réussit, ajoute l’utilisateur à la base de données Auth0.
* Lors des connexions suivantes, les informations d’authentification de l’utilisateur sont récupérées depuis Auth0, **PAS** depuis votre base de données personnalisée.
* Les nouveaux utilisateurs sont automatiquement ajoutés à la base de données Auth0.

<div id="bulk-user-imports-with-the-management-api">
  ## Importations en bloc d’utilisateurs avec la Management API
</div>

Si vous avez déjà une base de données d’utilisateurs, vous pouvez utiliser notre endpoint de la Management API [`/post_users_imports`](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports) pour alimenter une connexion de base de données avec ces renseignements.

<div id="migrate-users-with-the-user-importexport-feature">
  ## Migrer des utilisateurs avec la fonctionnalité d’importation/exportation d’utilisateurs
</div>

La fonctionnalité d’importation/exportation d’utilisateurs vous permet :

* D’importer en bloc vos utilisateurs existants de votre base de données dans Auth0.
* De rechercher et d’exporter certains ou la totalité de vos utilisateurs de base de données Auth0.

Pour utiliser l’importation en bloc, vous devez avoir le rôle Dashboard Editor - Users ou Admin. Pour utiliser l’exportation en bloc, vous devez avoir le rôle Dashboard Viewer - Users, Editor - Users ou Admin.

Vous pouvez importer et exporter les données utilisateur à l’aide de la [fonctionnalité d’importation/exportation d’utilisateurs](/docs/fr-CA/manage-users/user-migration/bulk-user-import-export) accessible dans le Dashboard.
