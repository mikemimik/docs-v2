title: Gérer les utilisateurs à l’aide de la Management API
---

En plus d’utiliser le Dashboard, vous pouvez récupérer, créer, mettre à jour ou supprimer des utilisateurs à l’aide de la [Management API](https://auth0.com/docs/api/management/v2#!/Users/get_users). Si vous souhaitez appeler directement la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, vous devrez d’abord générer le <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> approprié. Pour en savoir plus, consultez [Access Tokens for the Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens). Vous pouvez aussi utiliser un SDK pour mettre en œuvre la fonctionnalité dont vous avez besoin afin d’appeler la Management API depuis votre application. Pour obtenir la liste des SDKs offerts, consultez [la section sur les SDKs de notre matrice de soutien](/docs/fr-CA/troubleshoot/customer-support/product-support-matrix).

Pour savoir comment gérer l’accès à <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> pour les membres de votre équipe, consultez [Manage Dashboard Access](/docs/fr-CA/get-started/manage-dashboard-access).

<div id="permissions">
  ## Permissions
</div>

Utilisez les permissions suivantes pour obtenir un jeton d’accès pour les applications de machine à machine. Pour en savoir plus, consultez [Enregistrer des applications de machine à machine.](/docs/fr-CA/get-started/auth0-overview/create-applications/machine-to-machine-apps)

| Permissions                 | Descriptions                                                                                                                                                                                                                                             |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `read:users`                | Les applications peuvent rechercher et récupérer tous les enregistrements d’utilisateurs stockés dans un tenant Auth0. Cela comprend la lecture par les points de terminaison `/api/v2/users` , `/api/v2/users-by-email` et `/api/v2/jobs/users-export`. |
| `create:users`              | Les applications peuvent créer des utilisateurs dans n’importe quelle connexion de base de données ou Passwordless d’un tenant Auth0. Cela comprend la création par les points de terminaison `/api/v2/users` et `/api/v2/jobs/users-import`.            |
| `update:users`              | Les applications peuvent mettre à jour tout utilisateur stocké dans un tenant Auth0. Cela comprend la mise à jour par le point de terminaison `/api/v2/users`.                                                                                           |
| `update:users_app_metadata` | Les applications peuvent mettre à jour uniquement l’attribut `user.app_metadata` pour tout utilisateur stocké dans un tenant Auth0. Cela comprend la mise à jour par le point de terminaison `/api/v2/users endpoint`.                                   |
| `delete:users`              | Les applications peuvent supprimer tout utilisateur stocké dans un tenant Auth0. Cela comprend la suppression par le point de terminaison `/api/v2/users`.                                                                                               |

<div id="limitations">
  ## Limites
</div>

Par défaut, les attributs du profil utilisateur fournis par des <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> autres qu’Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, puisqu’ils sont mis à jour par le fournisseur d’identité chaque fois que l’utilisateur se connecte. Pour en savoir plus sur les attributs provenant des fournisseurs d’identité, consultez [Configurer la connexion du fournisseur d’identité pour les mises à jour du profil utilisateur](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Pour pouvoir modifier les attributs racine `name`, `nickname`, `given_name`, `family_name` ou `picture` du profil utilisateur normalisé, vous devez [configurer la synchronisation de votre connexion avec Auth0](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) afin que les attributs utilisateur soient mis à jour par le fournisseur d’identité uniquement lors de la création du profil utilisateur. Ces attributs racine pourront ensuite être modifiés individuellement ou au moyen d’importations en bloc

Vous pouvez définir des mots de passe à l’aide des points de terminaison `create` ou `update`, mais, pour des raisons de sécurité, les mots de passe ne peuvent pas être affichés avec les commandes `get` ou `list user`. Le panneau de droite de l’API Explorer fournit des indications sur les attributs du profil utilisateur qui peuvent être consultés ou modifiés pour un appel donné.

<div id="endpoints">
  ## Points de terminaison
</div>

* Utilisez le point de terminaison [`/users`](https://auth0.com/docs/api/management/v2/users/get-users) pour [obtenir des renseignements sur tous les utilisateurs](https://auth0.com/docs/api/management/v2/users/get-users). Vous pouvez aussi inclure des critères de recherche pour trouver des utilisateurs précis.
* Utilisez [`/user_id`](https://auth0.com/docs/api/management/v2/users/get-users-by-id) pour [obtenir des renseignements sur un utilisateur](https://auth0.com/docs/api/management/v2/users/get-users-by-id) à partir du `user_id`. Le `user_id` est un identifiant interne composé d’un nom de connexion et d’un identifiant unique de l’utilisateur. Le `user_id` est différent du <Tooltip tip="Jeton ID : information d’identification destinée au client lui-même plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+token">jeton ID</Tooltip>.
* Le point de terminaison [`/userinfo`](https://auth0.com/docs/api/authentication/reference#get-user-info) prend en entrée le [jeton d’accès Auth0 et renvoie les renseignements du profil utilisateur](https://auth0.com/docs/api/authentication/reference#get-user-info). Ce point de terminaison inclura les résultats de toutes les rules qui pourraient avoir modifié le profil utilisateur pendant la transaction d’authentication, mais le profil utilisateur obtenu ne sera filtré par aucune [portée](/docs/fr-CA/get-started/apis/scopes/api-scopes).
* Le point de terminaison [`/tokeninfo`](https://auth0.com/docs/api/authentication/reference#get-token-info) prend en entrée le [jeton ID Auth0 et renvoie les renseignements du profil utilisateur](https://auth0.com/docs/api/authentication/reference#get-token-info). Ce point de terminaison renverra un résultat qui n’inclut pas les résultats des rules qui modifient le profil utilisateur.
