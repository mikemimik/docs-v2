title: Gérer l'accès des utilisateurs aux applications
---

Tous les utilisateurs associés à un même tenant Auth0 sont partagés entre les applications du tenant (et ont donc accès à ces applications).

* Pour séparer les utilisateurs et restreindre leur accès, nous vous recommandons de [créer un tenant supplémentaire](/docs/fr-CA/get-started/auth0-overview/create-tenants/create-multiple-tenants) avec les applications auxquelles vous voulez donner accès à certains groupes d&#39;utilisateurs.

* Pour restreindre l&#39;accès des utilisateurs à certaines applications, vous pouvez utiliser des [rules](/docs/fr-CA/customize/rules).

  * Dans une rule, les variables `context.clientName` et `context.clientID` sont disponibles pour vérifier quelle application l&#39;utilisateur utilise pour se connecter.
  * Consultez [cette rule comme exemple](https://github.com/auth0/rules/blob/aeaf93bc058408e260192d0941a688963449d6be/src/rules/simple-user-whitelist-for-app.js).

* Pour empêcher des utilisateurs d&#39;accéder à certaines applications, configurez une nouvelle connexion et accordez l&#39;accès uniquement à une application précise.

  * Allez dans [Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social), puis cliquez sur une connexion.
  * Dans l&#39;onglet **Applications**, activez ou désactivez les applications.

* Pour désactiver l&#39;accès des utilisateurs à vos applications, vous pouvez [bloquer et débloquer des utilisateurs](/docs/fr-CA/manage-users/user-accounts/block-and-unblock-users) dans le Dashboard.
