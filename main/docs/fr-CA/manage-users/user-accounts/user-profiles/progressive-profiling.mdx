description: Comprenez comment le profilage progressif permet de recueillir des renseignements sur les utilisateurs au fil du temps, à mesure qu'ils interagissent avec votre site Web ou votre application.
title: Comprendre le fonctionnement du profilage progressif
---

Le profilage progressif consiste à recueillir graduellement des renseignements supplémentaires sur vos utilisateurs à mesure qu&#39;ils interagissent avec votre site Web ou votre application. Vous pouvez recueillir des renseignements détaillés sur un utilisateur uniquement lorsque cela est pertinent dans le cadre de ses activités.

Par exemple, vous pourriez recueillir le nom, l&#39;adresse courriel et le mot de passe de l&#39;utilisateur lors de l&#39;inscription, puis lui demander le nom de son entreprise et son titre de poste la prochaine fois qu&#39;il se connecte. Vous pouvez aussi utiliser [Auth0 Forms](/docs/fr-CA/customize/forms/configure-progressive-profile-form) pour concevoir un flux qui recueille des renseignements sur l&#39;utilisateur.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Lorsque vous demandez des renseignements supplémentaires aux utilisateurs, évitez de leur demander des renseignements que vous avez peut-être déjà. Par exemple, si un utilisateur s&#39;inscrit au moyen d&#39;une connexion sociale qui utilise le protocole OpenID Connect (comme Google ou Facebook), le fournisseur d&#39;identité (IdP) peut retourner plus de renseignements sur l&#39;utilisateur que vous ne le pensez. Pour en savoir plus, consultez [OpenID Connect Scopes](/docs/fr-CA/get-started/apis/scopes/openid-connect-scopes#standard-claims).
</Callout>

La mise en œuvre du profilage progressif améliore l&#39;expérience de vos utilisateurs en :

* raccourcissant les formulaires d&#39;inscription.
* évitant les questions répétitives.
* recueillant des renseignements plus pertinents.
* augmentant les taux de conversion.

<div id="how-it-works">
  ## Fonctionnement
</div>

Chaque fois qu’un utilisateur s’authentifie auprès d’Auth0, Auth0 met à jour son profil utilisateur. Les données utilisées pour mettre à jour son profil peuvent provenir de différentes sources :

* Attributs fournis par le <Tooltip tip="Fournisseur d’identité (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=IdP">IdP</Tooltip>.
* Attributs créés dynamiquement avec [Auth0 Actions](/docs/fr-CA/customize/actions).
* Attributs renvoyés par des requêtes à des API comme [FullContact](https://www.fullcontact.com/) et [Clearbit](https://clearbit.com/).
* Attributs gérés par l’application.

<div id="use-action-triggers">
  ## Utiliser les déclencheurs d’action
</div>

Avec le déclencheur d’action `post-login`, vous pouvez modifier les champs `user_metadata` et `app_metadata` du profil d’un utilisateur pendant le flux de connexion. C’est utile pour des tâches comme stocker des données propres à l’application sur le profil de l’utilisateur, enregistrer si certaines opérations ont été effectuées pour un utilisateur, ou mettre en cache les résultats d’opérations coûteuses sur le profil de l’utilisateur afin de pouvoir les réutiliser lors de connexions futures. Pour en savoir plus, consultez [Gérer les métadonnées utilisateur à l’aide du déclencheur d’action post-login](/docs/fr-CA/manage-users/user-accounts/metadata/manage-user-metadata).

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

Vous pouvez utiliser le <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> endpoint [Update a User](https://auth0.com/docs/api/management/v2/#!/Users/patch_users_by_id) d’Auth0 pour mettre à jour les données de profil d’un utilisateur authentifié, y compris les champs `app_metadata` et `user_metadata`. Pour en savoir plus, consultez [gérer les métadonnées à l’aide de la Management API](/docs/fr-CA/manage-users/user-accounts/metadata/manage-metadata-api).

<div id="use-the-lock-library">
  ## Utiliser la bibliothèque Lock
</div>

Vous pouvez utiliser l’option `additionalSignUpFields` pour ajouter des champs personnalisés aux formulaires d’inscription des utilisateurs. Lorsqu’un utilisateur saisit une valeur dans un champ personnalisé, Auth0 l’enregistre dans le champ `user_metadata` de son profil. Pour en savoir plus, consultez [Options de configuration de Lock](/docs/fr-CA/libraries/lock/lock-configuration#additionalsignupfields-array-).
