Vous pouvez mettre à jour les préférences d’une connexion pour un <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> en amont afin de contrôler à quel moment les mises à jour des attributs racine du profil utilisateur sont autorisées dans <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou au moyen de la <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

Par défaut, les attributs du profil utilisateur fournis par des fournisseurs d’identité autres qu’Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, car ils sont mis à jour par le fournisseur d’identité chaque fois que l’utilisateur se connecte.

Pour pouvoir modifier les attributs racine name, nickname, given&#95;name, family&#95;name ou picture du profil utilisateur normalisé, vous devez configurer la synchronisation de votre connexion avec Auth0 pour que les attributs utilisateur soient mis à jour depuis le fournisseur d’identité uniquement lors de la création du profil utilisateur.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Synchroniser les utilisateurs à la connexion** ne s’applique pas aux connexions Passwordless et ne peut pas être activé dans l’interface utilisateur du tenant ni au moyen de la Management API.
</Callout>

Vous pouvez modifier les [attributs racine individuellement](/docs/fr-CA/manage-users/user-accounts/user-profiles/root-attributes/update-root-attributes-for-users) ou au moyen d’une [importation en bloc](/docs/fr-CA/manage-users/user-accounts/user-profiles/root-attributes/set-root-attributes-during-user-import) à l’aide de la Management API.

<div id="use-the-dashboard">
  ## Utiliser le Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentification](https://manage.auth0.com/#/connections), puis sélectionnez le type de connexion : Database, Social, Enterprise ou Passwordless.
2. Sélectionnez le nom d’une connexion pour en afficher les paramètres.
3. Repérez la section **Advanced**, placez le commutateur **Synchroniser les attributs du profil utilisateur à chaque connexion** sur le réglage souhaité, puis sélectionnez **Save**.

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

Avant de terminer cette étape, vous devriez d’abord [récupérer les valeurs existantes de l’objet `options` de la connexion](/docs/fr-CA/authenticate/identity-providers/retrieve-connection-options) pour éviter d’écraser les valeurs actuelles. Sinon, les paramètres absents de l’objet d’origine seront perdus après la mise à jour.

Effectuez un appel `PATCH` à l’[endpoint Update a Connection](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id). Assurez-vous d’inclure les valeurs d’origine des options dans l’appel pour éviter d’écraser les valeurs actuelles. Assurez-vous également de remplacer les valeurs de l’espace réservé `CONNECTION_ID`, `MGMT_API_ACCESS_TOKEN` et `ATTRIBUTE_UPDATE_VALUE` par l’ID de votre connexion, le <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> de la Management API et la valeur de mise à jour de l’attribut, respectivement.

| Valeur                   | Description                                                                                                                                                                                                                                       |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CONNECTION_ID`          | ID de la connexion pour laquelle vous souhaitez autoriser les mises à jour des attributs racine.                                                                                                                                                  |
| `MGMT_API_ACCESS_TOKEN`  | [Jeton d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) avec la portée `update:connections`.                                                                                                                      |
| `ATTRIBUTE_UPDATE_VALUE` | Indique à quel moment vous souhaitez autoriser les mises à jour des attributs racine du profil utilisateur. Les valeurs valides sont `on_first_login` et `on_each_login`. La valeur par défaut pour les nouvelles connexions est `on_each_login`. |
