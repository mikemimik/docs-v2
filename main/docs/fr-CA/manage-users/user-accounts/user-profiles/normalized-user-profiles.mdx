---
description: Comprendre comment Auth0 normalise les propriétés utilisateur courantes dans le profil utilisateur.
title: Profils utilisateur normalisés
---

Le profil utilisateur normalisé est une méthode propre à Auth0 pour normaliser et stocker les renseignements associés à l’utilisateur. Comme chaque <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d&#39;identité</Tooltip> (IdP) fournit un ensemble différent de renseignements sur un utilisateur, Auth0 normalise les propriétés de profil courantes en une représentation de l’utilisateur indépendante du protocole lorsqu’il stocke les claims liés à l’utilisateur. Par exemple, `family_name` dans le profil utilisateur normalisé contient des renseignements qu’un IdP peut renvoyer sous la forme de `surname` ou `last_name`.

Ainsi, les claims Auth0 inclus dans le profil utilisateur normalisé diffèrent de l’ensemble standard de claims qui peuvent être renvoyées dans les <Tooltip tip="ID Token : justificatif destiné au client lui-même, plutôt qu'à l'accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+Tokens">ID Tokens</Tooltip> depuis le [endpoint `oauth/token` de l&#39;Authentication API](https://auth0.com/docs/api/authentication#get-token) ou dans la réponse de l’endpoint [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (qui suivent tous deux les exigences détaillées dans la spécification OIDC).

Pour en savoir plus sur les attributs inclus dans le profil utilisateur normalisé et comprendre comment les utiliser, consultez [Structure du profil utilisateur](/docs/fr-CA/manage-users/user-accounts/user-profiles/user-profile-structure).
