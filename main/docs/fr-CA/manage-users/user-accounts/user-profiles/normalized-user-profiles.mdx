description: Découvrez comment Auth0 normalise les propriétés utilisateur courantes dans le profil utilisateur.
title: Profils utilisateur normalisés
---

Le profil utilisateur normalisé est la façon dont Auth0 standardise et stocke les renseignements liés à l’utilisateur. Comme chaque <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) fournit un ensemble différent de renseignements sur un utilisateur, Auth0 normalise les propriétés de profil courantes en une représentation de l’utilisateur indépendante du protocole au moment de stocker les claims liées à l’utilisateur. Par exemple, `family_name` dans le profil utilisateur normalisé contient des renseignements qui peuvent être renvoyés par un IdP sous la forme de `surname` ou `last_name`.

Ainsi, les claims Auth0 incluses dans le profil utilisateur normalisé diffèrent de l’ensemble standard de claims qui peuvent être renvoyées dans les <Tooltip tip="Jeton d’identification : information d’identification destinée au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+Tokens">jetons d’identification</Tooltip> depuis le point de terminaison [`oauth/token` de l’Authentication API](https://auth0.com/docs/api/authentication#get-token) ou dans la réponse du point de terminaison [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (qui respectent tous deux les exigences détaillées dans la spécification OIDC).

Pour en savoir plus sur les attributs inclus dans le profil utilisateur normalisé et sur la façon de les utiliser, consultez [Structure du profil utilisateur](/docs/fr-CA/manage-users/user-accounts/user-profiles/user-profile-structure).
