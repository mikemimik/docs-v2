---

Les attributs qu&#39;Auth0 associe à un schéma commun sont listés ci-dessous.

Champs toujours générés :

* `name` : le nom complet de l&#39;utilisateur.
* `nickname` : par défaut, la partie locale de l&#39;adresse courriel de l&#39;utilisateur.
* `picture` : l&#39;URL de la [photo de l&#39;utilisateur](/docs/fr-CA/manage-users/user-accounts/change-user-picture). Si elle n&#39;est pas disponible, Auth0 utilise l&#39;image Gravatar associée à l&#39;adresse courriel de l&#39;utilisateur.
* `user_id` : l&#39;identifiant unique de l&#39;utilisateur. Il est unique par connexion, mais identique pour toutes les applications qui s&#39;authentifient au moyen de cette connexion.

Par défaut, les attributs `name`, `nickname` et `picture` d&#39;un utilisateur fournis par des <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d&#39;identité</Tooltip> autres qu&#39;Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, puisqu&#39;ils sont mis à jour par le fournisseur d&#39;identité chaque fois que l&#39;utilisateur se connecte. Si vous souhaitez pouvoir modifier ces attributs, vous devez [configurer la synchronisation de votre connexion avec Auth0](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) afin que les attributs utilisateur ne soient mis à jour à partir du fournisseur d&#39;identité qu&#39;au moment de la création du profil utilisateur. Les attributs racine pourront ensuite être [modifiés individuellement](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) ou [par importation en bloc](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) à l&#39;aide de la <Tooltip tip="Management API : un produit qui permet aux clients d'effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

Champs générés lorsque les détails sont disponibles :

* `email` : l&#39;adresse courriel de l&#39;utilisateur.
* `email_verified` : une valeur booléenne indiquant si l&#39;adresse courriel de l&#39;utilisateur a été vérifiée.
* `given_name` : le prénom de l&#39;utilisateur.
* `family_name` : le nom de famille de l&#39;utilisateur.

Lorsque vous créez un utilisateur à l&#39;aide du [point de terminaison Create a User de la Management API](https://auth0.com/docs/api/management/v2#!/Users/post_users), vous pouvez soumettre `given_name` et `family_name`. Par défaut, les attributs `given_name` et `family_name` d&#39;un utilisateur fournis par des fournisseurs d&#39;identité autres qu&#39;Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, puisqu&#39;ils sont mis à jour par le fournisseur d&#39;identité chaque fois que l&#39;utilisateur se connecte. Si vous souhaitez pouvoir modifier ces attributs, vous devez [configurer la synchronisation de votre connexion avec Auth0](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) afin que les attributs utilisateur ne soient mis à jour à partir du fournisseur d&#39;identité qu&#39;au moment de la création du profil utilisateur. Les attributs racine pourront ensuite être [modifiés individuellement](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) ou [par importation en bloc](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) à l&#39;aide de la Management API

Si vous écrivez un script de connexion pour une [base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db) , il vous incombe de retourner les informations du profil utilisateur. Une propriété `user_id` unique et immuable est obligatoire pour identifier correctement l&#39;utilisateur (voir [Identifier les utilisateurs de façon unique](/docs/fr-CA/manage-users/user-accounts/identify-users)).

<div id="additional-attributes">
  ## Attributs supplémentaires
</div>

Le profil utilisateur comprend un tableau d’identités. Dans le cas le plus courant (connexion avec un seul fournisseur), le tableau ne contient qu’un seul élément. Si plusieurs comptes sont liés à l’utilisateur, le tableau comportera un élément pour chaque compte associé. Consultez [Liaison de comptes utilisateur](/docs/fr-CA/manage-users/user-accounts/user-account-linking) pour en savoir plus.

Le tableau `identities` contient les attributs suivants :

* `connection` : le nom de la connexion.
* `isSocial` : indique si le fournisseur est un fournisseur social.
* `provider` : le fournisseur de la connexion.
* `user_id` : l’identificateur unique de l’utilisateur pour cette connexion.

Auth0 transmet à votre application toutes les autres propriétés fournies par le fournisseur d’identité, même si elles ne sont pas mappées aux attributs standard indiqués ci-dessus.
