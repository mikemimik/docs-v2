title: Schéma du profil utilisateur normalisé
---

Les attributs qu’Auth0 associe à un schéma commun sont énumérés ci-dessous.

Champs toujours générés :

* `name`: le nom complet de l’utilisateur.
* `nickname`: par défaut, la partie locale de l’adresse courriel de l’utilisateur.
* `picture`: l’URL de la [photo de l’utilisateur](/docs/fr-CA/manage-users/user-accounts/change-user-picture). Si elle n’est pas disponible, Auth0 utilise l’image Gravatar associée à l’adresse courriel de l’utilisateur.
* `user_id`: l’identifiant unique de l’utilisateur. Il est unique par connexion, mais le même pour toutes les applications qui s’authentifient au moyen de cette connexion.

Par défaut, les attributs `name`, `nickname` et `picture` d’un utilisateur fournis par des <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> autres qu’Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, puisqu’ils sont mis à jour par le fournisseur d’identité chaque fois qu’un utilisateur se connecte. Si vous souhaitez pouvoir modifier ces attributs, vous devez [configurer la synchronisation de votre connexion avec Auth0](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) afin que les attributs de l’utilisateur soient mis à jour à partir du fournisseur d’identité uniquement lors de la création du profil utilisateur. Les attributs racine pourront ensuite être [modifiés individuellement](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) ou [par importation en bloc](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) à l’aide de la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

Champs générés lorsque l’information est disponible :

* `email`: l’adresse courriel de l’utilisateur.
* `email_verified`: une valeur booléenne indiquant si l’adresse courriel de l’utilisateur a été vérifiée.
* `given_name`: le prénom de l’utilisateur.
* `family_name`: le nom de famille de l’utilisateur.

Lors de la création d’un utilisateur avec le [point de terminaison « Create a user » de la Management API](https://auth0.com/docs/api/management/v2#!/Users/post_users), vous pouvez soumettre `given_name` et `family_name`. Par défaut, les attributs `given_name` et `family_name` d’un utilisateur fournis par des fournisseurs d’identité autres qu’Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, puisqu’ils sont mis à jour par le fournisseur d’identité chaque fois qu’un utilisateur se connecte. Si vous souhaitez pouvoir modifier ces attributs, vous devez [configurer la synchronisation de votre connexion avec Auth0](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) afin que les attributs de l’utilisateur soient mis à jour à partir du fournisseur d’identité uniquement lors de la création du profil utilisateur. Les attributs racine pourront ensuite être [modifiés individuellement](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) ou [par importation en bloc](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) à l’aide de la Management API

Si vous rédigez un script de connexion pour une [base de données personnalisée](/docs/fr-CA/authenticate/database-connections/custom-db), il vous incombe de renvoyer les renseignements du profil utilisateur. Une propriété `user_id` unique et immuable est obligatoire pour identifier correctement l’utilisateur (consultez [Identifier les utilisateurs de manière unique](/docs/fr-CA/manage-users/user-accounts/identify-users)).

<div id="additional-attributes">
  ## Attributs supplémentaires
</div>

Le profil utilisateur comprend un tableau d’identités. Dans le cas le plus courant (lorsqu’un utilisateur se connecte avec un seul fournisseur), le tableau ne contient qu’un seul élément. Si l’utilisateur a plusieurs comptes liés, le tableau comporte un élément pour chaque compte associé. Consultez [Liaison de comptes d’utilisateurs](/docs/fr-CA/manage-users/user-accounts/user-account-linking) pour en savoir plus.

Le tableau `identities` contient les attributs suivants :

* `connection` : le nom de la connexion.
* `isSocial` : indique si le fournisseur est un fournisseur social.
* `provider` : le fournisseur de la connexion.
* `user_id` : l’identifiant unique de l’utilisateur pour cette connexion.

Auth0 transmet à votre application toutes les autres propriétés fournies par le fournisseur d’identité, même si elles ne sont pas mappées aux attributs standard indiqués ci-dessus.
