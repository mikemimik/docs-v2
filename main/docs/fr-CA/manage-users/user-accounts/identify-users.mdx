---
description: Découvrez comment identifier vos utilisateurs de façon unique.
title: Identifier les utilisateurs
---

Il existe deux options recommandées pour identifier vos utilisateurs de façon unique :

1. Au moyen de la propriété `user_id`. Son unicité est garantie (dans un tenant) pour chaque utilisateur (par exemple, `{identity provider id}|{unique id in the provider}` ou `facebook|1234567890`). Un utilisateur peut avoir la même propriété `user_id` dans plusieurs tenants Auth0, mais cette cohérence n&#39;est pas garantie.
2. Au moyen d&#39;une clé naturelle, comme la propriété `email`. Dans ce cas, il est recommandé d&#39;activer la vérification de l&#39;adresse courriel et d&#39;utiliser cette option uniquement avec les fournisseurs qui exigent que les utilisateurs vérifient leur adresse courriel.

Pour les connexions de base de données [standard](/docs/fr-CA/authenticate/database-connections/auth0-user-store) et [personnalisées](/docs/fr-CA/authenticate/database-connections/custom-db), évitez de créer des `user_id` en double dans un tenant pour différentes connexions de base de données. Si vous prévoyez des collisions possibles entre des ID provenant de différentes connexions, vous devriez utiliser un préfixe qui identifie la connexion.

L&#39;exemple de code suivant montre une connexion de base de données personnalisée identifiée par le préfixe `MyConnection1|` :

```javascript lines
function login (email, password, callback) {
  var user = getUserFromDB(email);
  var profile = {
    user_id: 'MyConnection1|' + user.id,
    email: user.email,
    [...]
  };
  callback(null, profile);
}
```
