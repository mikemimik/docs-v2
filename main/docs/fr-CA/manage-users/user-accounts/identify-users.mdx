---
description: Découvrez comment identifier vos utilisateurs de manière unique.
title: Identifier les utilisateurs
---

Il existe deux options recommandées pour identifier vos utilisateurs de manière unique :

1. Au moyen de la propriété `user_id`. Son unicité est garantie (dans un tenant) pour chaque utilisateur (par exemple, `{identity provider id}|{unique id in the provider}` ou `facebook|1234567890`). Un utilisateur peut avoir la même propriété `user_id` dans plusieurs tenants Auth0, mais cette cohérence n’est pas garantie.
2. Au moyen d’une clé naturelle, comme la propriété `email`. Dans ce cas, il est recommandé d’activer la vérification de l’adresse courriel et d’utiliser cette option uniquement avec des fournisseurs d’identité qui exigent que les utilisateurs vérifient leur adresse courriel.

Pour les connexions de base de données [standard](/docs/fr-CA/authenticate/database-connections/auth0-user-store) et [personnalisées](/docs/fr-CA/authenticate/database-connections/custom-db), évitez de créer des `user_id` en double dans un tenant pour différentes connexions de base de données. Si vous prévoyez de possibles collisions entre des IDs provenant de différentes connexions, vous devriez utiliser un préfixe qui identifie la connexion.

L’exemple de code suivant montre une connexion de base de données personnalisée identifiée par le préfixe `MyConnection1|` :

```javascript lines
function login (email, password, callback) {
  var user = getUserFromDB(email);
  var profile = {
    user_id: 'MyConnection1|' + user.id,
    email: user.email,
    [...]
  };
  callback(null, profile);
}
```
