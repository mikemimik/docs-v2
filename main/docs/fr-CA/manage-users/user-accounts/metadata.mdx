Auth0 offre un système complet pour stocker des métadonnées dans le profil utilisateur Auth0. Vous pouvez utiliser les métadonnées pour effectuer les opérations suivantes :

* Stocker des données propres à l’application dans le profil utilisateur.
* Indiquer si des opérations précises ont eu lieu ou non pour un utilisateur.
* Mettre en cache les résultats d’opérations coûteuses dans le profil utilisateur afin de pouvoir les réutiliser lors de connexions ultérieures.
* Stocker des informations qui ne proviennent pas d’un <Tooltip tip="Fournisseur d’identité (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> ou qui remplacent celles qu’un fournisseur d’identité fournit.

Les métadonnées peuvent être modifiées dans le cadre du flux de connexion d’un utilisateur.

Vous pouvez configurer la synchronisation de la connexion afin que les attributs racine de l’utilisateur soient mis à jour par le fournisseur d’identité uniquement lors de la création du profil utilisateur. Vous pouvez ensuite modifier les attributs racine individuellement ou par importation en bloc. Pour en savoir plus, consultez [Configurer la connexion du fournisseur d’identité pour les mises à jour du profil utilisateur](/docs/fr-CA/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

<div id="metadata-types">
  ## Types de métadonnées
</div>

Auth0 utilise trois types de métadonnées pour stocker des renseignements précis.

| Type de métadonnées                | Nom du champ                                                                                                                         | Description                                                                                                                                                                                                                                                                                                                                 |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Informations utilisateur**       | `user_metadata`                                                                                                                      | Stocke des attributs utilisateur, comme des préférences, qui n’ont pas d’incidence sur le fonctionnement de base d’un utilisateur. Ces données **peuvent** être modifiées par les utilisateurs connectés si vous créez un formulaire à l’aide de la Management API et **ne doivent pas** être utilisées comme stockage sécurisé de données. |
| **Informations d’accès**           | `app_metadata`                                                                                                                       | Stocke des renseignements comme les permissions, le forfait Auth0 et les ID externes qui peuvent avoir une incidence sur l’accès de l’utilisateur aux fonctionnalités. Ces données **ne peuvent pas** être modifiées par les utilisateurs et des restrictions s’appliquent à ce qui peut être stocké dans ce champ.                         |
| **Informations sur l’application** | `client_metadata` dans l’objet `Client`, `context.clientMetadata` dans Rules et `event.client.metadata` dans les Actions post-login. | Stocke des renseignements sur une application (ou un *client* selon la terminologie OIDC OAuth2). Par exemple, l’URL de la page d’accueil de l’application (toute valeur qu’Auth0 ne définit pas dans les paramètres de l’application).                                                                                                     |

<div id="manage-metadata">
  ## Gérer les métadonnées
</div>

Vous pouvez créer et mettre à jour des métadonnées à l’aide de Rules, de l’Authentication API, de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> et de la bibliothèque Lock.

Pour les scripts de base de données personnalisés et le script `fetchUserProfile` des connexions sociales personnalisées, vous pouvez créer et mettre à jour des métadonnées à l’aide de l’objet `metadata`.

<div id="use-actions">
  ### Utiliser les Actions
</div>

Les [Actions](/docs/fr-CA/customize/actions) sont des fonctions sécurisées, propres au tenant et versionnées, écrites en Node.js, qui s’exécutent à certains moments dans la plateforme Auth0. Les Actions servent à personnaliser et à étendre les capacités d’Auth0 à l’aide d’une logique personnalisée. Elles peuvent aussi servir à [enrichir le profil utilisateur](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

Par exemple, vous pouvez créer une Action `post-login` qui utilise des claims personnalisées pour copier les propriétés `user_metadata` dans les <Tooltip tip="ID Token : jeton destiné au client lui-même plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+tokens">jetons ID</Tooltip>. Vous pouvez ensuite récupérer le `user_metadata` des utilisateurs à partir du endpoint [Get User Info](https://auth0.com/docs/api/authentication#get-user-info) de l’Authentication API.

Pour en savoir plus, consultez [Gérer les métadonnées utilisateur à l’aide du déclencheur d’action post-login](/docs/fr-CA/manage-users/user-accounts/metadata/manage-user-metadata).

<div id="use-the-management-api">
  ### Utiliser la Management API
</div>

Un utilisateur peut demander un <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> avec les scopes appropriés et utiliser les [endpoints de la Management API](https://auth0.com/docs/api/management/v2) suivants pour consulter, créer ou mettre à jour `user_metadata`, ou supprimer une configuration <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip>.

| Tâche         | Endpoint                                                                                                                              | Scope                          |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ |
| Consulter     | [`GET /api/v2/users/{id}`](https://auth0.com/docs/api/management/v2#!/Users/get_users_by_id)                                          | `read:current_user`            |
| Créer         | [`PATCH /api/v2/users/{id}`](https://auth0.com/docs/api/management/v2#!/Users/post_users)                                             | `create:current_user_metadata` |
| Mettre à jour | [`PATCH /api/v2/users/{id}`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id)                                      | `update:current_user_metadata` |
| Supprimer     | [`DELETE /api/v2/users/{id}/multifactor/{provider}`](https://auth0.com/docs/api/management/v2#!/Users/delete_multifactor_by_provider) | `update:users`                 |

Un jeton d’accès est nécessaire pour appeler l’Auth0 Management API. Pour en savoir plus, consultez [Jetons d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) et [Obtenir des jetons d’accès à la Management API pour les SPA](https://auth0.com/docs/api/management/v2/get-access-tokens-for-spas).

<div id="use-the-dashboard">
  ### Utiliser le Dashboard
</div>

Utilisez le Auth0 Dashboard pour configurer les métadonnées d’application, qui contiennent des paires clé-valeur. Pour en savoir plus, consultez [Configurer les métadonnées d’application](/docs/fr-CA/get-started/applications/configure-application-metadata).

<div id="use-the-lock-library">
  ### Utilisez la bibliothèque Lock
</div>

Utilisez la bibliothèque [Lock](/docs/fr-CA/libraries/lock) pour définir, ajouter, lire et mettre à jour `user_metadata.` Les propriétés de `user_metadata` se lisent de la même manière que n’importe quelle autre propriété du profil utilisateur. Par exemple, l’extrait de code suivant récupère la valeur associée à `user_metadata.hobby` et l’attribue à un élément de la page :

```javascript lines
// Utiliser l'accessToken obtenu lors de l'authentification pour appeler getUserInfo
lock.getUserInfo(accessToken, function(error, profile) {
  if (!error) {
    document.getElementById('hobby').textContent = profile.user_metadata.hobby;
  }
});
```

Vous pouvez utiliser `additionalSignUpFields` pour ajouter des champs personnalisés dans les formulaires d’inscription des utilisateurs. Lorsqu’un utilisateur saisit des données dans un champ personnalisé, Auth0 stocke les valeurs entrées dans le `user_metadata` de cet utilisateur. Pour en savoir plus sur l’ajout de `user_metadata` lors de l’inscription, consultez [Additional Signup Fields](/docs/fr-CA/libraries/lock/lock-configuration#additionalsignupfields).

<div id="custom-database-connections-and-metadata">
  ## Connexions de base de données personnalisées et métadonnées
</div>

Si vous avez une [connexion de base de données personnalisée](/docs/fr-CA/authenticate/database-connections#using-your-own-user-store), vous pouvez utiliser le point de terminaison [`/dbconnections/signup`](https://auth0.com/docs/api/authentication?shell#signup) de l&#39;Authentication API pour définir le `user_metadata` d&#39;un utilisateur. Pour en savoir plus sur l&#39;utilisation des métadonnées dans le cadre d&#39;un processus d&#39;inscription personnalisé, consultez [Inscription personnalisée](/docs/fr-CA/libraries/custom-signup).

Lorsque vous définissez le champ `user_metadata` à l&#39;aide du point de terminaison `/dbconnections/signup` de l&#39;Authentication API, vous pouvez définir au maximum 10 champs de type chaîne et 500 caractères.

<div id="custom-emails-and-metadata">
  ## Courriels personnalisés et métadonnées
</div>

Utilisez les métadonnées pour stocker l’information que vous souhaitez utiliser pour personnaliser les courriels Auth0. Par exemple, utilisez `user_metadata.lang` si vous voulez que l’utilisateur puisse modifier la valeur de ce champ, puis servez-vous de cette information pour personnaliser la langue du courriel. Pour en savoir plus, consultez [Personnaliser les modèles de courriel](/docs/fr-CA/customize/email/email-templates).
