---
description: Comment afficher les utilisateurs et les détails de leur profil.
title: Afficher les détails de l’utilisateur
---

La page [Utilisateurs](https://manage.auth0.com/#/users) répertorie les utilisateurs associés à vos applications. Pour ouvrir un utilisateur en particulier, cliquez sur la photo de profil ou sur le nom de l’utilisateur dans la colonne « Name ». La page Détails de l’utilisateur s’ouvre et affiche les informations sur cet utilisateur.

La page Détails de l’utilisateur comprend des liens vers cinq onglets :

1. Détails
2. Appareils
3. Historique
4. Emplacements
5. JSON brut

<div id="user-details-details-tab">
  ## Détails de l’utilisateur : onglet Détails
</div>

L’onglet Détails contient trois sections qui offrent un aperçu général des renseignements figurant dans le profil de l’utilisateur :

* **Identité de l’utilisateur** fournit, en un coup d’œil, des détails sur l’utilisateur, notamment l’identifiant utilisé, les Connections associées et les droits d’accès.
* **Métadonnées** affiche les renseignements `app_metadata` et `user_metadata`. Vous pouvez modifier ces valeurs.
* **<Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">Fournisseur d’identité</Tooltip> Attributs** affiche les renseignements récupérés auprès du fournisseur d’authentification. Notez que les attributs du fournisseur d’identité sont en lecture seule.
* **Clés d’accès** affiche les [clés d’accès](/docs/fr-CA/authenticate/database-connections/passkeys) créées par l’utilisateur.
* **<Tooltip tip="Multi-factor authentication (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip>** affiche les facteurs [MFA](/docs/fr-CA/secure/multi-factor-authentication) que l’utilisateur a configurés.
  Si vous activez [Multiple Custom Domains](/docs/fr-CA/customize/custom-domains/multiple-custom-domains), vous disposez de l’option **Domain** pour sélectionner le domaine de notification de votre tenant.

<div id="user-details-devices-tab">
  ## Détails de l’utilisateur : onglet Appareils
</div>

L’onglet Appareils répertorie les appareils à partir desquels l’utilisateur a demandé une authentification. Le fait de demander une autorisation sur un appareil lie cet appareil au compte de l’utilisateur.

Les informations de connexion de l’utilisateur sont associées au <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se connecter de nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Refresh+Token">jeton d’actualisation</Tooltip> attribué à cet appareil. Pour révoquer le jeton d’actualisation, cliquez sur **Dissocier** à côté de l’appareil.

<div id="user-details-history-tab">
  ## Détails de l’utilisateur : onglet Historique
</div>

L’onglet Historique affiche un historique de l’activité du compte de l’utilisateur pour les 2 derniers jours.

Les journaux comprennent des renseignements sur :

* Les événements qui se sont produits
* Le moment où les événements se sont produits
* Les applications associées aux événements
* Le fournisseur d’identité utilisé pour l’authentification
* Les adresses IP d’origine des événements
* Le lieu d’origine des événements

<div id="user-details-locations-tab">
  ## Détails de l’utilisateur : onglet Emplacements
</div>

L’onglet Emplacements affiche une carte avec des repères indiquant l’endroit ou les endroits où l’utilisateur se trouvait lorsqu’il s’est connecté aux applications.

<div id="user-details-raw-json-tab">
  ## Détails de l’utilisateur : onglet JSON brut
</div>

L’onglet JSON brut affiche, au format JSON, toutes les informations contenues dans le profil de l’utilisateur afin que vous puissiez consulter rapidement l’ensemble des renseignements disponibles à son sujet.
