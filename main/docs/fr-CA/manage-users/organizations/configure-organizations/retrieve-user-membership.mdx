Lorsque vous travaillez avec les [organisations](/docs/fr-CA/manage-users/organizations/organizations-overview) par programmation, vous devrez peut-être récupérer la liste des organisations dont un utilisateur est membre.

<div id="management-api">
  ## Management API
</div>

<Warning>
  Jusqu’à 1000 membres d’organisation peuvent être affichés dans l’Auth0 Dashboard ou au moyen de la Management API, bien qu’il puisse en exister davantage.
</Warning>

Effectuez un appel `GET` au point de terminaison `Get User Organizations`. Assurez-vous de remplacer les valeurs de l’espace réservé `USER_ID` et `MGMT_API_ACCESS_TOKEN` par le user ID et votre <Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Jeton d’accès : Informations d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisées pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip>, respectivement.

  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/users/USER_ID/organizations' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users/USER_ID/organizations");
  ```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Si votre domaine Auth0 correspond au nom de votre tenant, il s’agit de votre sous-domaine régional (à moins que votre tenant soit dans la région des États-Unis et ait été créé avant juin 2020), suivi de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et a été créé avant juin 2020, alors votre domaine serait `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir de votre nom de domaine personnalisé.
</Callout>

| Valeur | Description |
| `USER_ID` | ID de l’utilisateur dont vous voulez récupérer l’appartenance à l’organisation. |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `read:organizations`. |

<div id="response-status-codes">
  ### Codes d’état de la réponse
</div>

Voici les codes d’état de réponse possibles :

| Code d’état | Code d’erreur          | Message                                                                                                | Cause                                                                                                     |
| ----------- | ---------------------- | ------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| `200`       |                        | Organizations récupérées avec succès.                                                                  |                                                                                                           |
| `400`       | `invalid_uri`          | `invalid_request_uri`                                                                                  | Le chemin n’est pas valide.                                                                               |
| `400`       | `invalid_query_string` | Chaîne de requête non valide. Le message varie selon la cause.                                         | La chaîne de requête n’est pas valide.                                                                    |
| `401`       |                        | Jeton non valide.                                                                                      |                                                                                                           |
| `401`       |                        | Signature non valide reçue lors de la validation du JSON Web Token.                                    |                                                                                                           |
| `403`       | `insufficient_scope`   | Scope insuffisant ; l’un des éléments suivants était attendu : `read:organizations`.                   | Tentative de lire ou de modifier un field qui n’est pas autorisé avec les scopes du bearer token fournis. |
| `404`       |                        | Utilisateur introuvable                                                                                |                                                                                                           |
| `429`       |                        | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                           |
