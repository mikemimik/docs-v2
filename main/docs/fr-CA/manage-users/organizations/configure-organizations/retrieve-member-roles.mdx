Lorsque vous travaillez avec les [organisations](/docs/fr-CA/manage-users/organizations/organizations-overview) de façon programmatique, vous pourriez avoir besoin de récupérer la liste des rôles attribués à un membre d&#39;une organisation.

Vous pouvez consulter les rôles associés à un membre d&#39;une organisation au moyen de l&#39;<Tooltip tip="Auth0 Dashboard : le principal produit d'Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Management API : un produit qui permet aux clients d'effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; organisations](https://manage.auth0.com/#/organizations), puis sélectionnez l’organisation dont vous voulez consulter l’appartenance.
2. Sélectionnez la vue **Members**, puis l’utilisateur dont vous voulez consulter les rôles.

<div id="management-api">
  ## Management API
</div>

Effectuez un appel `GET` au point de terminaison `Get Organization Member Roles`. Assurez-vous de remplacer les valeurs de l’espace réservé `ORG_ID`, `USER_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre organisation, l’ID utilisateur du membre et le <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, respectivement.

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Si votre domaine Auth0 est composé du nom de votre tenant, de votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), puis de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir de votre nom de domaine personnalisé.
</Callout>

| Valeur                  | Description                                                                                                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ORG_ID`                | ID de l’organisation pour laquelle vous voulez récupérer les rôles d’un membre.                                                                        |
| `USER_ID`               | ID utilisateur du membre dont vous voulez récupérer les rôles.                                                                                         |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `read:organization_member_roles`. |

<div id="response-status-codes">
  #### Codes d’état de la réponse
</div>

Les codes d’état de la réponse possibles sont les suivants :

| Code d’état | Code d’erreur          | Message                                                                                                 | Cause                                                                                                      |
| ----------- | ---------------------- | ------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| `200`       |                        | Rôles récupérés avec succès.                                                                            |                                                                                                            |
| `400`       | `invalid_query_string` | Chaîne de requête invalide. Le message varie selon la cause.                                            | La chaîne de requête n’est pas valide.                                                                     |
| `401`       |                        | Jeton invalide.                                                                                         |                                                                                                            |
| `401`       |                        | Signature invalide reçue lors de la validation du JSON Web Token.                                       |                                                                                                            |
| `401`       |                        | Le client n’est pas global.                                                                             |                                                                                                            |
| `403`       | `insufficient_scope`   | Portée insuffisante; au moins une des portées suivantes est requise : `read:organization_member_roles`. | Tentative de lecture ou d’écriture d’un champ non autorisée avec les portées fournies par le bearer token. |
| `429`       |                        | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset.  |                                                                                                            |
