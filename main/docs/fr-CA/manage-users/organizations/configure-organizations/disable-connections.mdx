<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Pour désactiver une connexion dans le Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) et sélectionnez l’organisation dont vous souhaitez configurer les connexions.
2. Désactivez le champ **Enabled** pour la connexion que vous souhaitez désactiver.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La désactivation d’une connexion ne supprime pas son association avec l’organisation. En désactivant une connexion, vous pouvez conserver les valeurs de `show_as_button`, `assign_membership_on_login`, etc., tout en laissant la connexion désactivée.
</Callout>

Pour supprimer une connexion d’une organisation dans le Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) et sélectionnez l’organisation pour laquelle vous souhaitez configurer des connexions.
2. Sélectionnez la vue **Connections**, repérez la connexion que vous souhaitez supprimer, puis ouvrez son menu **More Options** (**...**).
3. Sélectionnez **Remove Connection**, puis confirmez.

<div id="management-api">
  ## Management API
</div>

Pour désactiver une connexion dans le Auth0 Dashboard :
Effectuez un appel `PATCH` au point de terminaison `Update Organization Connections`. Assurez-vous de remplacer les valeurs de remplacement `ORG_ID`, `MGMT_API_ACCESS_TOKEN` et `CONNECTION_ID` par votre ID d’organisation, votre Management API <Tooltip tip="Jeton d’accès : information d’identification Authorization, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">Jeton d’accès</Tooltip> et votre ID de connexion, respectivement.

Assurez-vous de remplacer les valeurs de remplacement `ORG_ID`, `MGMT_API_ACCESS_TOKEN` et `CONNECTION_ID` par votre ID d’organisation, votre <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API et votre ID de connexion, respectivement.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Si votre domaine Auth0 correspond au nom de votre tenant, il sera formé de votre sous-domaine régional (à moins que votre tenant se trouve dans la région des États-Unis et ait été créé avant juin 2020), suivi de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir du nom de votre domaine personnalisé.
</Callout>

| Valeur                  | Description                                                                                                                                             |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de l’Organisation pour laquelle vous souhaitez désactiver une connexion.                                                                             |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour l’API de gestion](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `delete:organization_connections`. |
| `CONNECTION_ID`         | ID de la connexion que vous souhaitez désactiver pour l’Organisation.                                                                                   |

<div id="response-status-codes">
  ### Codes d&#39;état de la réponse
</div>

Les codes d&#39;état de la réponse possibles sont les suivants :

| Code d&#39;état | Code d&#39;erreur    | Message                                                                                                | Cause                                                                                              |
| --------------- | -------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| `204`           |                      | Connexion supprimée de l’Organisation avec succès.                                                     |                                                                                                    |
| `401`           |                      | Token invalide.                                                                                        |                                                                                                    |
| `401`           |                      | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                                    |
| `401`           |                      | Le client n’est pas global.                                                                            |                                                                                                    |
| `403`           | `insufficient_scope` | Scope insuffisant ; une des valeurs suivantes était attendue : `delete:organization_connections`.      | Tentative de lecture ou d’écriture d’un champ non autorisé avec les scopes du bearer token fourni. |
| `429`           |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                    |
