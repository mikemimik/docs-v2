Lorsque vous travaillez avec des [organisations](/docs/fr-CA/manage-users/organizations/organizations-overview) de façon programmatique, il se peut que vous deviez récupérer la liste des membres d’une organisation.

Vous pouvez consulter les membres associés à une organisation à l’aide de l’<Tooltip tip="Auth0 Dashboard : l’interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : l’interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’organisation dont vous voulez consulter les membres.
2. Sélectionnez la vue **Members**.

<div id="management-api">
  ## Management API
</div>

<Warning>
  Jusqu’à 1000 membres d’une organisation peuvent être affichés dans l’Auth0 Dashboard ou au moyen de la Management API, bien qu’il puisse en exister davantage.
</Warning>

Effectuez une requête `GET` vers le endpoint `Get Organization Members`. Assurez-vous de remplacer les valeurs de remplacement `ORG_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre organisation et le <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, respectivement.

  ```bash cURL
  curl --request GET \

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Si votre domaine Auth0 est formé du nom de votre tenant, de votre sous-domaine régional (à moins que votre tenant soit situé dans la région des États-Unis et ait été créé avant juin 2020), puis de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant était situé aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir du nom de votre domaine personnalisé.
</Callout>

| Valeur                  | Description                                                                                                                                       |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de l’organisation dont vous voulez récupérer les membres.                                                                                      |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `read:organization_members`. |

<div id="response-status-codes">
  ### Codes d’état de réponse
</div>

Les codes d’état de réponse possibles sont les suivants :

| Code d’état | Code d’erreur        | Message                                                                                                | Cause                                                                                                 |
| ----------- | -------------------- | ------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------- |
| `200`       |                      | Membres récupérés avec succès.                                                                         |                                                                                                       |
| `400`       | `invalid_paging`     | La page demandée dépasse le maximum autorisé de 1000 enregistrements.                                  | L’API est limitée à retourner un maximum de 1000 enregistrements.                                     |
| `400`       | `invalid_body`       | Corps de la requête invalide. Le message varie selon la cause.                                         | La charge utile de la requête n’est pas valide.                                                       |
| `401`       |                      | Token invalide.                                                                                        |                                                                                                       |
| `401`       |                      | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                                       |
| `401`       |                      | Le client n’est pas global.                                                                            |                                                                                                       |
| `403`       | `insufficient_scope` | Scope insuffisant ; l’un des éléments suivants est attendu : `read:organization_members`.              | Tentative de lire ou d’écrire un champ qui n’est pas autorisé avec les scopes du bearer token fourni. |
| `429`       |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                       |
