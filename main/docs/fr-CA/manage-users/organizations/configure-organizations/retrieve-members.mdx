Lorsque vous travaillez avec [organizations](/docs/fr-CA/manage-users/organizations/organizations-overview) par programmation, vous devrez peut-être récupérer la liste des membres d’une organisation.

Vous pouvez consulter les membres associés à une organisation au moyen de l’<Tooltip tip="Auth0 Dashboard : le principal produit d'Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de l’<Tooltip tip="Auth0 Dashboard : le principal produit d'Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’organisation dont vous souhaitez consulter les membres.
2. Sélectionnez la vue **Members**.

<div id="management-api">
  ## Management API
</div>

<Warning>
  Jusqu’à 1000 membres de l’organisation peuvent être affichés dans l’Auth0 Dashboard ou au moyen de la Management API, mais il peut en exister davantage.
</Warning>

Envoyez une requête `GET` au point de terminaison `Get Organization Members`. Assurez-vous de remplacer les valeurs de remplacement `ORG_ID` et `MGMT_API_ACCESS_TOKEN` par votre identifiant d’organisation et votre <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, respectivement.

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Votre domaine Auth0 est constitué du nom de votre tenant, de votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), suivi de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir du nom de votre domaine personnalisé.
</Callout>

| Valeur                  | Description                                                                                                                                        |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | Identifiant de l’organisation dont vous voulez récupérer les membres.                                                                              |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `read:organization_members`. |

<div id="response-status-codes">
  ### Codes d’état de la réponse
</div>

Les codes d’état de la réponse possibles sont les suivants :

| Code d’état | Code d’erreur        | Message                                                                                                | Cause                                                                                        |
| ----------- | -------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------- |
| `200`       |                      | Membres récupérés avec succès.                                                                         |                                                                                              |
| `400`       | `invalid_paging`     | La page demandée dépasse le maximum autorisé de 1000 enregistrements.                                  | L’API est limitée à un maximum de 1000 enregistrements renvoyés.                             |
| `400`       | `invalid_body`       | Corps de la requête invalide. Le message varie selon la cause.                                         | Le corps de la requête n’est pas valide.                                                     |
| `401`       |                      | Jeton invalide.                                                                                        |                                                                                              |
| `401`       |                      | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                              |
| `401`       |                      | Le client n’est pas un client global.                                                                  |                                                                                              |
| `403`       | `insufficient_scope` | Portée insuffisante ; l’une des portées suivantes est requise : `read:organization_members`.           | Tentative de lire ou d’écrire un champ non autorisé avec les portées du jeton Bearer fourni. |
| `429`       |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                              |
