Vous pouvez activer des connexions précises pour chaque [organisation](/docs/fr-CA/manage-users/organizations/organizations-overview) afin d’offrir aux utilisateurs différentes options de connexion. Après avoir activé une connexion, elle est ajoutée à l’écran de connexion de l’organisation, et les utilisateurs peuvent s’authentifier au moyen de cette connexion pour accéder à vos applications.

Pour activer une connexion pour une organisation, la connexion **doit** déjà exister dans votre tenant. Les connexions prises en charge comprennent les [connexions de base de données](/docs/fr-CA/authenticate/database-connections), les [connexions sociales](/docs/fr-CA/authenticate/identity-providers/social-identity-providers) et les [connexions d’entreprise](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers).

<div id="organization-properties">
  ## Propriétés de l’organisation
</div>

Lorsque vous utilisez les organisations, certaines connexions comportent des propriétés supplémentaires que vous pouvez configurer :

| **Propriété**                                        | **Types de connexion**                   | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ---------------------------------------------------- | ---------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Nom de la connexion de l’organisation                | Tous les types de connexion              | Ce nom est fourni par l’administrateur de l’organisation au moyen de la [My Organization API](/docs/fr-CA/manage-users/my-organization-api). S’il n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux administrateurs de l’organisation lorsqu’ils consultent cette connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Niveau d’accès de l’administrateur de l’organisation | Tous les types de connexion              | Cette propriété détermine le niveau d’accès dont disposent les administrateurs de l’organisation pour cette connexion. Les valeurs possibles sont : <ul><li>**Aucun** : géré par l’administrateur du tenant, non visible et non modifiable au moyen de la My Organization API.</li> <li>**Lecture seule** : géré par l’administrateur du tenant, visible au moyen de la My Organization API.</li> <li>**Limité** : les administrateurs de l’organisation peuvent modifier certains paramètres non sensibles au moyen de la My Organization API.</li> <li>**Complet** : les administrateurs de l’organisation peuvent modifier tous les paramètres autorisés pour cette connexion, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile).</li></ul>                                                                                         |
| Adhésion lors de l’authentification                  | Tous les types de connexion              | Lorsqu’elle est activée, cette propriété attribue automatiquement une adhésion à l’organisation aux utilisateurs finaux la première fois qu’ils s’authentifient avec la connexion. L’adhésion lors de l’authentification est utile dans les cas où l’on peut supposer que tous les utilisateurs en mesure de s’authentifier avec une connexion donnée sont membres d’une organisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| Inscription à l’organisation                         | Connexions de base de données uniquement | Cette propriété détermine si les utilisateurs finaux peuvent accéder à un lien d’inscription sur l’écran de connexion leur permettant d’obtenir une adhésion à une organisation. Pour utiliser l’inscription à l’organisation, vous devez aussi activer l’adhésion lors de l’authentification. L’inscription à l’organisation est utile dans les cas où les utilisateurs ont besoin d’un accès en libre-service pour rejoindre des organisations, par exemple : <ul><li>Lorsque les organisations servent à modéliser des populations d’utilisateurs à adhésion ouverte, comme des espaces de travail.</li> <li>Dans des cas d’utilisation B2B2C où les utilisateurs peuvent créer librement des comptes.</li></ul>                                                                                                                                                                                                   |
| Afficher la connexion sous forme de bouton           | Connexions d’entreprise uniquement       | Facultatif. Cette propriété détermine si une connexion donnée s’affiche comme option sur l’écran de connexion de l’organisation. Les utilisateurs finaux peuvent aussi s’authentifier en utilisant le profil d’authentification [Identifier First with Home Realm Discovery](/docs/fr-CA/authenticate/login/auth0-universal-login/identifier-first) combiné au flux de connexion d’organisation [Prompt for Credentials](/docs/fr-CA/manage-users/organizations/login-flows-for-organizations#configure-the-login-flow-for-your-application). Cela fonctionne seulement si vous ne fournissez **pas** de `org_id` avec la requête d’autorisation.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Si cette option est désactivée, les utilisateurs finaux peuvent quand même s’authentifier au moyen de la connexion en envoyant directement le paramètre de connexion dans la requête d’autorisation. </Callout> |

<div id="configure-organization-connections">
  ## Configurer les connexions d’organisation
</div>

Vous pouvez configurer les connexions pour les organisations à l’aide du <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    #### Auth0 Dashboard

    Pour activer une connexion dans Auth0 Dashboard :

    1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’Organization que vous voulez configurer.
    2. Sélectionnez l’onglet **Connections**, puis **Add Connections**.
    3. Choisissez la connexion que vous voulez activer, puis sélectionnez **Add Connection**.
    4. Dans la section Organization Connection, configurez les deux propriétés requises pour utiliser une connexion via la My Organization API :

    * Repérez **Organization Connection Name**. S’il n’est pas renseigné, indiquez le nom que vous souhaitez voir affiché aux administrateurs de l’Organization lorsqu’ils consultent cette connexion.
    * Repérez **Organization Admin Access Level** et, s’il n’est pas renseigné, choisissez le niveau d’accès que vous souhaitez accorder aux administrateurs de l’Organization pour cette connexion via la My Organization API.

    5. Dans la section Authentication, repérez **Membership On Authentication** et choisissez d’activer ou de désactiver l’ajout automatique des membres. Lorsqu’elle est activée, cette option ajoute automatiquement comme membres de l’Organization tous les utilisateurs qui se connectent avec cette connexion.
    6. **Pour les connexions Database seulement** : Dans la section Organization Signup, choisissez d’activer ou de désactiver les inscriptions en libre-service. Lorsqu’elle est activée, cette option permet aux utilisateurs d’accéder à un lien d’inscription dans l’écran de connexion afin de créer leur compte et d’obtenir automatiquement l’adhésion à l’Organization.

       * Pour activer cette propriété, vous devez d’abord activer **Membership on Authentication**.
       * **Remarque** : Organization Signup a priorité sur la configuration d’inscription de la connexion Database. Lorsque Organization Signup est activé, les utilisateurs peuvent quand même s’inscrire à une Organization ou accepter une invitation à une Organization, même si les inscriptions sont désactivées au niveau de la connexion Database.
    7. **Pour les connexions Enterprise seulement** : Dans la section Connection button, activez au besoin la propriété **Display connection as a button** pour afficher la connexion comme option dans l’écran de connexion de l’Organization.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         Si toutes les connexions activées dans l’Organization sont des connexions Enterprise et qu’elles sont toutes masquées, Auth0 renvoie l’erreur `Message: no connections enabled for the organization are visible` lorsque les utilisateurs accèdent à l’application.
       </Callout>
    8. Sélectionnez **Save**.
  </Tab>

  <Tab title="Management API">
    #### Management API

    Pour activer une connexion via la Management API :

    Effectuez un appel `POST` vers le point de terminaison `Create Organization Connections`. Assurez-vous de remplacer les valeurs fictives suivantes par les informations appropriées :

    * Remplacez `{orgId}` par votre ID d’organisation.
    * Remplacez `{mgmtApiAccessToken}` par votre jeton d’accès à l’API de gestion.
    * Remplacez `{connectionId}` par un ID de connexion précis.
    * Remplacez `{assignMembershipOption}` par `true` ou `false` selon votre sélection Membership on Authentication.
    * Remplacez `{isEnabled}` par `true` ou `false` selon que vous souhaitez activer la connexion ou non.
    * **Pour les connexions de base de données seulement** : remplacez `{isSignupEnabled}` par `true` ou `false` selon votre sélection d’inscription.
    * **Pour les connexions d’entreprise seulement** : remplacez `{showAsButtonOption}` par `true` ou `false` selon votre sélection Connection Button.
    * **Pour l’utilisation de l’API My Organization** : remplacez `{organizationAccessLevel}` et `{organizationConnectionName}` par les valeurs appropriées.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Trouvez votre domaine Auth0**

      Votre domaine Auth0 correspond au nom de votre tenant, à votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), suivi de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

      Si vous utilisez des [domaines personnalisés](/docs/fr-CA/customize/custom-domains), il doit s’agir de votre nom de domaine personnalisé.
    </Callout>

    | `{orgId}`                      | ID de l’Organization pour laquelle vous souhaitez activer une connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
    | `{mgmtApiAccessToken}`         | [Jeton d’accès pour l’API de gestion](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) doté de la portée `create:organization_connections`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
    | `{connectionId}`               | ID de la connexion que vous souhaitez activer pour l’Organization indiquée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
    | `{assignMembershipOption}`     | Indique si vous voulez que les utilisateurs qui ouvrent une session avec cette connexion obtiennent automatiquement leur appartenance à l’Organisation. Lorsque la valeur est définie à `true`, les utilisateurs obtiennent automatiquement leur appartenance. Lorsque la valeur est définie à `false`, ils ne l’obtiennent pas automatiquement.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
    | `{isEnabled}`                  | Permet d’activer ou de désactiver une connexion pour une Organisation. En désactivant une connexion, il est possible de conserver les valeurs de `show_as_button`, `assign_membership_on_login`, etc., tout en laissant la connexion désactivée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
    | `{isSignupEnabled}`            | Détermine si les utilisateurs peuvent accéder à un lien d’inscription en libre-service sur l’écran de connexion. Lorsque cette option est définie sur `true`, le lien d’inscription s’affiche sur l’écran. Lorsqu’elle est définie sur `false`, le lien reste masqué. **Remarque** : pour activer cette option, vous devez aussi définir `{assignMembershipOption}` sur `true`.                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
    | `{showAsButtonOption}`         | Indique si vous souhaitez afficher une connexion d’entreprise précise comme option dans l’invite de connexion de l’Organization. Lorsqu’elle est définie sur `true`, la connexion s’affiche sous forme de bouton dans l’invite. Lorsqu’elle est définie sur `false`, la connexion est masquée dans l’invite.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
    | `{organizationAccessLevel}`    | Détermine le niveau d’accès qu’un Admin d’organisation aura lors de l’utilisation de l’API My Organization. Lorsqu’elle est définie sur `none`, la connexion sera gérée par le Tenant Admin et ne sera ni visible ni modifiable via l’API My Organization. Lorsqu’elle est définie sur `read-only`, la connexion sera gérée par le Tenant Admin et sera visible via l’API My Organization, mais non modifiable. Lorsqu’elle est définie sur `limited`, les Admins d’organisation peuvent modifier des paramètres sélectionnés non sensibles via l’API My Organization. Lorsqu’elle est définie sur `full`, les Admins d’organisation peuvent modifier tous les paramètres autorisés pour cette connexion, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile). |
    | `{organizationConnectionName}` | Ce nom est fourni par l’Admin de l’Organization au moyen de la My Organization API. S’il n’est pas renseigné, indiquez le nom que vous souhaitez que les Admins de l’Organization voient lorsqu’ils consultent cette connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

    ##### Codes d&#39;état de la réponse

    Les codes d&#39;état de réponse possibles sont les suivants :

    | **Code d’état** | **Code d’erreur**    | **Message**                                                                                            | **Cause**                                                                                            |
    | --------------- | -------------------- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
    | `201`           |                      | Connexion ajoutée à une organisation avec succès.                                                      |                                                                                                      |
    | `401`           |                      | Jeton non valide.                                                                                      |                                                                                                      |
    | `401`           |                      | Signature non valide reçue lors de la validation du JSON Web Token.                                    |                                                                                                      |
    | `401`           |                      | Le client n’est pas global.                                                                            |                                                                                                      |
    | `403`           | `insufficient_scope` | Scope insuffisant ; une des valeurs suivantes est attendue : `create:organizations_connections`.       | Tentative de lecture ou d’écriture d’un champ non autorisée avec les scopes du jeton porteur fourni. |
    | `429`           |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                      |
