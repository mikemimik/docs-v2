En utilisant cette fonctionnalité, vous acceptez les conditions applicables de l’essai gratuit énoncées dans le [Contrat-cadre d’abonnement d’Okta](https://www.okta.com/legal/). Vous pouvez activer des connexions précises pour chaque [organisation](/docs/fr-CA/manage-users/organizations/organizations-overview) afin d’offrir aux utilisateurs différentes options de connexion. Après avoir activé une connexion, elle est ajoutée au prompt de connexion de l’organisation, et les utilisateurs peuvent s’authentifier au moyen de cette connexion pour accéder à vos applications.

Pour activer une connexion pour une organisation, la connexion **doit** déjà exister dans votre tenant. Les connexions prises en charge comprennent les [connexions de base de données](/docs/fr-CA/authenticate/database-connections), les [connexions sociales](/docs/fr-CA/authenticate/identity-providers/social-identity-providers) et les [connexions d’entreprise](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers).

<div id="organization-properties">
  ## Propriétés de l’organisation
</div>

Lorsque vous utilisez des organisations, certaines connexions comportent des propriétés supplémentaires que vous pouvez configurer :

| **Propriété**                                        | **Types de connexion**                  | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ---------------------------------------------------- | --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Nom de la connexion de l’organisation                | Tous les types de connexion             | Ce nom est fourni par l’administrateur de l’organisation au moyen de la [My Organization API](/docs/fr-CA/manage-users/my-organization-api). S’il n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux administrateurs de l’organisation lorsqu’ils consultent cette connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Niveau d’accès de l’administrateur de l’organisation | Tous les types de connexion             | Cette propriété détermine le niveau d’accès dont disposent les administrateurs de l’organisation pour cette connexion. Les valeurs disponibles sont : <ul><li>**None** : géré par l’administrateur du tenant, non visible et non modifiable au moyen de la My Organization API.</li> <li>**Read-only** : géré par l’administrateur du tenant, visible au moyen de la My Organization API.</li> <li>**Limited** : les administrateurs de l’organisation peuvent modifier certains paramètres non sensibles au moyen de la My Organization API.</li> <li>**Full** : les administrateurs de l’organisation peuvent modifier tous les paramètres autorisés pour cette connexion, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile).</li></ul>                                                                                                                       |
| Membership On Authentication                         | Tous les types de connexion             | Lorsqu’elle est activée, cette propriété attribue automatiquement une appartenance à une organisation aux utilisateurs finaux la première fois qu’ils s’authentifient avec la connexion. Membership On Authentication est utile dans les scénarios où l’on peut supposer que tous les utilisateurs en mesure de s’authentifier avec une connexion donnée sont membres d’une organisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Organization Signup                                  | Connexions de base de données seulement | Cette propriété détermine si les utilisateurs finaux peuvent accéder à un lien d’inscription dans le prompt de connexion, leur permettant d’obtenir une appartenance à une organisation. Pour utiliser Organization Signup, vous devez aussi activer Membership On Authentication. Organization Signup est utile dans les scénarios où les utilisateurs ont besoin d’un accès en libre-service pour se joindre à des organisations, par exemple : <ul><li>Lorsque les organisations servent à modéliser des populations d’utilisateurs à adhésion ouverte, comme des espaces de travail.</li> <li>Dans les cas d’utilisation Business-to-Consumer (B2B2C) où les utilisateurs peuvent librement créer un compte.</li></ul>                                                                                                                                                                                                                      |
| Afficher la connexion sous forme de bouton           | Connexions d’entreprise seulement       | Facultatif. Cette propriété détermine si une connexion donnée s’affiche ou non comme option dans le prompt de connexion de l’organisation. Les utilisateurs finaux peuvent aussi s’authentifier à l’aide du profil d’authentification [Identifier First with Home Realm Discovery](/docs/fr-CA/authenticate/login/auth0-universal-login/identifier-first) en combinaison avec le flux de connexion de l’organisation [Prompt for Credentials](/docs/fr-CA/manage-users/organizations/login-flows-for-organizations#configure-the-login-flow-for-your-application). Cela fonctionne uniquement si vous **ne** fournissez **pas** de `org_id` avec la demande d’autorisation.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Si cette option est désactivée, les utilisateurs finaux peuvent quand même s’authentifier au moyen de la connexion en envoyant directement le paramètre de connexion dans la demande d’autorisation. </Callout> |

<div id="configure-organization-connections">
  ## Configurer les connexions d’organisation
</div>

Vous pouvez configurer les connexions pour les organisations à partir de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    #### Auth0 Dashboard

    Pour activer une connexion dans Auth0 Dashboard :

    1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’Organization que vous souhaitez configurer.
    2. Sélectionnez l’onglet **Connections**, puis **Add Connections**.
    3. Choisissez la connexion à activer, puis sélectionnez **Add Connection**.
    4. Dans la section Organization Connection, configurez les deux propriétés requises pour utiliser une connexion au moyen de My Organization API :

    * Repérez **Organization Connection Name**. S’il n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux administrateurs de l’Organization lorsqu’ils consultent cette connexion.
    * Repérez **Organization Admin Access Level** et, s’il n’est pas renseigné, choisissez le niveau d’accès que vous souhaitez accorder aux administrateurs de l’Organization pour cette connexion au moyen de My Organization API.

    5. Dans la section Authentication, repérez **Membership On Authentication** et choisissez d’activer ou de désactiver l’ajout automatique des membres. Lorsqu’elle est activée, cette option ajoute automatiquement à l’Organization tous les utilisateurs qui ouvrent une session avec cette connexion.
    6. **Pour les Database Connections uniquement** : dans la section Organization Signup, choisissez d’activer ou de désactiver les inscriptions en libre-service. Lorsqu’elle est activée, les utilisateurs peuvent accéder à un lien d’inscription dans l’écran de connexion pour créer leur compte et obtenir automatiquement l’adhésion à l’Organization.

       * Pour activer cette propriété, vous devez d’abord activer **Membership on Authentication**.
       * **Remarque** : Organization Signup a priorité sur la configuration d’inscription de la Database Connection. Lorsque Organization Signup est activé, les utilisateurs peuvent quand même s’inscrire à une Organization ou accepter une invitation à une Organization, même si les inscriptions sont désactivées au niveau de la Database Connection.
    7. **Pour les Enterprise connections uniquement** : dans la section Connection button, activez au besoin la propriété **Display connection as a button** pour afficher la connexion comme option dans l’écran de connexion de l’Organization.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         Si toutes les connexions activées dans l’Organization sont des connexions d’entreprise et qu’elles sont toutes masquées, Auth0 renvoie l’erreur `Message: no connections enabled for the organization are visible` lorsque les utilisateurs accèdent à l’application.
       </Callout>
    8. Sélectionnez **Save**.
  </Tab>

  <Tab title="Management API">
    #### Management API

    Pour activer une connexion via la Management API :

    Effectuez un appel `POST` vers le point de terminaison `Create Organization Connections`. Assurez-vous de remplacer les valeurs de l&#39;espace réservé suivantes par les informations appropriées :

    * Remplacez `{orgId}` par votre ID d’organisation.
    * Remplacez `{mgmtApiAccessToken}` par votre jeton d’accès à la Management API.
    * Remplacez `{connectionId}` par l’ID d’une connexion précise.
    * Remplacez `{assignMembershipOption}` par `true` ou `false` selon le paramètre Adhésion lors de l’authentification que vous avez sélectionné.
    * Remplacez `{isEnabled}` par `true` ou `false` selon que vous voulez activer la connexion ou non.
    * **Pour les connexions de base de données seulement** : remplacez `{isSignupEnabled}` par `true` ou `false` selon l’option d’inscription que vous avez sélectionnée.
    * **Pour les connexions d’entreprise seulement** : remplacez `{showAsButtonOption}` par `true` ou `false` selon l’option de bouton de connexion que vous avez sélectionnée.
    * **Pour utiliser My Organization API** : remplacez `{organizationAccessLevel}` et `{organizationConnectionName}` par les valeurs appropriées.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Trouvez votre domaine Auth0**

      Votre domaine Auth0 est formé du nom de votre tenant, de votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), puis de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

      Si vous utilisez des [domaines personnalisés](/docs/fr-CA/customize/custom-domains), il devrait s’agir de votre nom de domaine personnalisé.
    </Callout>

    | `{mgmtApiAccessToken}`         | [Jeton d’accès pour l’API de gestion](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `create:organization_connections`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |

    | `{assignMembershipOption}`     | Indique si vous souhaitez que les utilisateurs qui se connectent avec cette connexion obtiennent automatiquement une adhésion à l’Organization. Lorsqu’elle est définie sur `true`, les utilisateurs obtiennent automatiquement une adhésion. Lorsqu’elle est définie sur `false`, ils n’en obtiennent pas automatiquement.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
    | `{isEnabled}`                  | Active ou désactive une connexion pour une Organisation. La désactivation d’une connexion permet de conserver les valeurs de `show_as_button`, `assign_membership_on_login`, etc., tout en laissant la connexion désactivée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
    | `{isSignupEnabled}`            | Détermine si les utilisateurs peuvent accéder à un lien d’inscription libre-service sur le prompt de connexion. Lorsqu’elle est définie à `true`, le lien d’inscription s’affiche sur le prompt. Lorsqu’elle est définie à `false`, le lien reste masqué. **Remarque** : pour activer cette option, vous devez aussi définir `{assignMembershipOption}` à `true`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
    | `{showAsButtonOption}`         | Indique si vous voulez qu’une connexion Enterprise précise s’affiche comme option sur l’invite de connexion de l’Organization. Lorsqu’elle est définie sur `true`, la connexion s’affiche sous forme de bouton sur l’invite. Lorsqu’elle est définie sur `false`, la connexion est masquée sur l’invite.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
    | `{organizationAccessLevel}`    | Détermine le type d’accès dont disposera un Admin d’Organization lorsqu’il utilise la My Organization API. Lorsque la valeur est définie sur `none`, la connection sera gérée par le Tenant Admin et ne sera ni visible ni modifiable via la My Organization API. Lorsque la valeur est définie sur `read-only`, la connection sera gérée par le Tenant Admin et sera visible via la My Organization API, mais non modifiable. Lorsque la valeur est définie sur `limited`, les Admins d’Organization peuvent modifier certains Settings sélectionnés non sensibles via la My Organization API. Lorsque la valeur est définie sur `full`, les Admins d’Organization peuvent modifier tous les Settings autorisés pour cette connection, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile). |
    | `{organizationConnectionName}` | Ce nom est fourni par l’Admin de l’Organization via la My Organization API. S’il n’est pas renseigné, indiquez le nom que vous souhaitez que les Admins de l’Organization voient lorsqu’ils consultent cette connection.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

    ##### Codes d&#39;état de la réponse

    Les codes d&#39;état de réponse possibles sont les suivants :

    | **Code de statut** | **Code d’erreur**    | **Message**                                                                                            | **Cause**                                                                                          |
    | ------------------ | -------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
    | `201`              |                      | Connexion ajoutée avec succès à une organisation.                                                      |                                                                                                    |
    | `401`              |                      | Jeton invalide.                                                                                        |                                                                                                    |
    | `401`              |                      | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                                    |
    | `401`              |                      | Le client n’est pas global.                                                                            |                                                                                                    |
    | `403`              | `insufficient_scope` | Scope insuffisant ; une des valeurs suivantes est attendue : `create:organizations_connections`.       | Tentative de lecture ou d’écriture d’un field non autorisé avec les scopes du bearer token fourni. |
    | `429`              |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                    |
