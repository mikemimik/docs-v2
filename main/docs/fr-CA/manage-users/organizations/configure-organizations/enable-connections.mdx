Vous pouvez activer des connexions précises pour chaque [organisation](/docs/fr-CA/manage-users/organizations/organizations-overview) afin d’offrir aux utilisateurs différentes options de connexion. Après avoir activé une connexion, elle est ajoutée à l’écran de connexion de l’organisation, et les utilisateurs peuvent s’authentifier au moyen de cette connexion pour accéder à vos applications.

Pour activer une connexion pour une organisation, la connexion **doit** déjà exister dans votre tenant. Les connexions prises en charge comprennent les [connexions de base de données](/docs/fr-CA/authenticate/database-connections), les [connexions sociales](/docs/fr-CA/authenticate/identity-providers/social-identity-providers) et les [connexions d’entreprise](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers).

<div id="organization-properties">
  ## Propriétés des organisations
</div>

Lorsque vous utilisez des organisations, certaines connexions ont des propriétés supplémentaires que vous pouvez configurer :

| **Propriété**                                        | **Types de connexion**                   | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ---------------------------------------------------- | ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Nom de la connexion de l’organisation                | Tous les types de connexion              | Ce nom est fourni par l’administrateur de l’organisation à l’aide de l’[API My Organization](/docs/fr-CA/manage-users/my-organization-api). S’il n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux administrateurs d’organisation lorsqu’ils consultent cette connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Niveau d’accès de l’administrateur de l’organisation | Tous les types de connexion              | Cette propriété détermine le niveau d’accès dont disposent les administrateurs d’organisation pour cette connexion. Les valeurs possibles sont : <ul><li>**Aucun** : géré par l’administrateur de tenant, non visible et non modifiable à l’aide de l’API My Organization.</li> <li>**Lecture seule** : géré par l’administrateur de tenant, visible à l’aide de l’API My Organization.</li> <li>**Limité** : les administrateurs d’organisation peuvent modifier certains paramètres non sensibles à l’aide de l’API My Organization.</li> <li>**Complet** : les administrateurs d’organisation peuvent modifier tous les paramètres autorisés pour cette connexion, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile).</li></ul>                                                                                                       |
| Membership On Authentication                         | Tous les types de connexion              | Lorsqu’elle est activée, cette propriété attribue automatiquement une appartenance à l’organisation aux utilisateurs finaux la première fois qu’ils s’authentifient avec la connexion. Membership On Authentication est utile lorsqu’on peut présumer que tous les utilisateurs capables de s’authentifier avec une connexion donnée sont membres d’une organisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Organization Signup                                  | Connexions de base de données uniquement | Cette propriété détermine si les utilisateurs finaux peuvent accéder à un lien d’inscription dans l’écran de connexion leur permettant d’obtenir une appartenance à une organisation. Pour utiliser Organization Signup, vous devez aussi activer Membership On Authentication. Organization Signup est utile lorsque les utilisateurs ont besoin d’un accès libre-service pour se joindre à des organisations, par exemple : <ul><li>lorsque les organisations servent à modéliser des groupes d’utilisateurs à adhésion ouverte, comme des workspaces;</li> <li>dans des cas d’utilisation Business-to-Business-to-Consumer (B2B2C) où les utilisateurs peuvent librement créer un compte.</li></ul>                                                                                                                                                                                                                  |
| Afficher la connexion sous forme de bouton           | Connexions d’entreprise uniquement       | Facultatif. Cette propriété détermine si une connexion donnée s’affiche comme option dans l’écran de connexion de l’organisation. Les utilisateurs finaux peuvent aussi s’authentifier à l’aide du profil d’authentification [Identifier First with Home Realm Discovery](/docs/fr-CA/authenticate/login/auth0-universal-login/identifier-first), combiné au flux de connexion de l’organisation [Prompt for Credentials](/docs/fr-CA/manage-users/organizations/login-flows-for-organizations#configure-the-login-flow-for-your-application). Cela fonctionne uniquement si vous ne fournissez **pas** de `org_id` avec la requête d’autorisation.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Si cette option est désactivée, les utilisateurs finaux peuvent quand même s’authentifier au moyen de la connexion en envoyant directement le paramètre de connexion dans la requête d’autorisation. </Callout> |

<div id="configure-organization-connections">
  ## Configurer les connexions d’organisation
</div>

Vous pouvez configurer les connexions pour les organisations à l’aide de l’<Tooltip tip="Auth0 Dashboard : l’interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    #### Auth0 Dashboard

    Pour activer une connexion dans Auth0 Dashboard :

    1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’organization que vous souhaitez configurer.
    2. Sélectionnez l’onglet **Connections**, puis **Add Connections**.
    3. Choisissez la connexion que vous souhaitez activer, puis sélectionnez **Add Connection**.
    4. Dans la section Organization Connection, configurez les deux propriétés requises pour utiliser une connexion au moyen de la My Organization API :

    * Repérez **Organization Connection Name**. Si ce champ n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux administrateurs de l’organization lorsqu’ils consultent cette connexion.
    * Repérez **Organization Admin Access Level** et, si ce champ n’est pas renseigné, choisissez le niveau d’accès que vous souhaitez accorder aux administrateurs de l’organization pour cette connexion au moyen de la My Organization API.

    5. Dans la section Authentication, repérez **Membership On Authentication** et choisissez d’activer ou de désactiver l’ajout automatique des membres. Lorsqu’elle est activée, cette option ajoute automatiquement comme membres de l’organization tous les utilisateurs qui se connectent au moyen de cette connexion.
    6. **Pour les Database connections seulement** : Dans la section Organization Signup, choisissez d’activer ou de désactiver les inscriptions en libre-service. Lorsqu’elle est activée, les utilisateurs peuvent accéder à un lien d’inscription dans le login prompt pour créer leur compte et devenir automatiquement membres de l’organization.

       * Pour activer cette propriété, vous devez d’abord activer **Membership on Authentication**.
       * **Remarque** : Organization Signup a priorité sur la configuration d’inscription de la Database connection. Lorsque Organization Signup est activée, les utilisateurs peuvent quand même s’inscrire à une organization ou accepter une invitation à une organization, même si les inscriptions sont désactivées au niveau de la Database connection.
    7. **Pour les Enterprise connections seulement** : Dans la section Connection button, activez au besoin la propriété **Display connection as a button** pour afficher la connexion comme option dans le login prompt de l’organization.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         Si toutes les connexions activées dans l’organization sont des enterprise connections et qu’elles sont toutes masquées, Auth0 renvoie une erreur indiquant `Message: no connections enabled for the organization are visible` lorsque les utilisateurs accèdent à l’application.
       </Callout>
    8. Sélectionnez **Save**.
  </Tab>

  <Tab title="Management API">
    #### Management API

    Pour activer une connexion via la Management API :

    Effectuez une requête `POST` vers le point de terminaison `Create Organization Connections`. Assurez-vous de mettre à jour les valeurs d&#39;espace réservé suivantes avec les informations appropriées :

    * Remplacez `{orgId}` par votre ID d’organisation.
    * Remplacez `{mgmtApiAccessToken}` par votre jeton d’accès à l’API Management.
    * Remplacez `{connectionId}` par l’ID d’une connexion spécifique.
    * Remplacez `{assignMembershipOption}` par `true` ou `false` selon votre sélection pour l’adhésion lors de l’authentification.
    * Remplacez `{isEnabled}` par `true` ou `false` selon que vous voulez activer la connexion ou non.
    * **Pour les connexions de base de données seulement** : remplacez `{isSignupEnabled}` par `true` ou `false` selon votre sélection pour l’inscription.
    * **Pour les connexions d’entreprise seulement** : remplacez `{showAsButtonOption}` par `true` ou `false` selon l’option choisie pour le bouton de connexion.
    * **Pour utiliser l’API My Organization** : remplacez `{organizationAccessLevel}` et `{organizationConnectionName}` par les valeurs appropriées.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Trouvez votre domaine Auth0**

      Votre domaine Auth0 est formé du nom de votre tenant, de votre sous-domaine régional (à moins que votre tenant soit dans la région des États-Unis et ait été créé avant juin 2020), puis de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

      Si vous utilisez des [domaines personnalisés](/docs/fr-CA/customize/custom-domains), il doit s’agir de votre nom de domaine personnalisé.
    </Callout>

    | `{orgId}`                      | ID de l’Organisation pour laquelle vous souhaitez activer une connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
    | `{mgmtApiAccessToken}`         | [Jeton d’accès pour l’API de gestion](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) doté de la portée `create:organization_connections`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
    | `{connectionId}`               | ID de la connexion que vous souhaitez activer pour l’Organisation spécifiée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
    | `{assignMembershipOption}`     | Indique si vous souhaitez que les utilisateurs qui se connectent avec cette connexion deviennent automatiquement membres de l’Organisation. Lorsque la valeur est définie à `true`, les utilisateurs deviendront automatiquement membres. Lorsque la valeur est définie à `false`, ils ne deviendront pas automatiquement membres.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
    | `{isEnabled}`                  | Active ou désactive une connexion pour une organisation. En désactivant une connexion, vous pouvez conserver les valeurs de `show_as_button`, `assign_membership_on_login`, etc., tout en laissant la connexion désactivée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
    | `{isSignupEnabled}`            | Détermine si les utilisateurs peuvent accéder à un lien d’inscription libre-service dans le prompt de connexion. Lorsque cette option est définie sur `true`, le lien d’inscription s’affiche dans le prompt. Lorsqu’elle est définie sur `false`, le lien demeure masqué. **Remarque** : pour activer cette option, vous devez aussi définir `{assignMembershipOption}` sur `true`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
    | `{showAsButtonOption}`         | Indique si vous souhaitez qu’une connexion Enterprise particulière s’affiche en tant qu’option sur l’écran de connexion de l’Organization. Lorsque cette valeur est définie sur `true`, la connexion s’affiche sous forme de bouton sur l’écran. Lorsque cette valeur est définie sur `false`, la connexion est masquée sur l’écran.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
    | `{organizationAccessLevel}`    | Détermine le type d’accès qu’un Organization Admin aura lors de l’utilisation de la My Organization API. Lorsque la valeur est `none`, la connexion sera gérée par l’administrateur de tenant et ne sera ni visible ni modifiable via la My Organization API. Lorsque la valeur est `read-only`, la connexion sera gérée par l’administrateur de tenant et sera visible via la My Organization API, mais ne pourra pas être modifiée. Lorsque la valeur est `limited`, les Organization Admins peuvent modifier certains paramètres sélectionnés et non sensibles via la My Organization API. Lorsque la valeur est `full`, les Organization Admins peuvent modifier tous les paramètres autorisés pour cette connexion, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile). |
    | `{organizationConnectionName}` | Ce nom est fourni par l’Admin de l’Organisation au moyen de la My Organization API. S’il n’est pas renseigné, indiquez le nom que vous souhaitez que les Admins de l’Organisation voient lorsqu’ils consultent cette connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

    ##### Codes d&#39;état de la réponse

    Les codes d&#39;état de réponse possibles sont les suivants :

    | **Code d’état** | **Code d’erreur**    | **Message**                                                                                            | **Cause**                                                                                          |
    | --------------- | -------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
    | `201`           |                      | Connexion ajoutée avec succès à une organisation.                                                      |                                                                                                    |
    | `401`           |                      | Jeton invalide.                                                                                        |                                                                                                    |
    | `401`           |                      | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                                    |
    | `401`           |                      | Le client n’est pas global.                                                                            |                                                                                                    |
    | `403`           | `insufficient_scope` | Portée insuffisante; l’une des portées suivantes est requise : `create:organizations_connections`.     | Tentative de lecture ou d’écriture d’un champ non autorisé avec les scopes du bearer token fourni. |
    | `429`           |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                    |
