<Card title="La disponibilité varie selon votre plan Auth0">
  La disponibilité de cette fonctionnalité dépend à la fois de votre mise en œuvre de connexion et de votre plan Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Card>

Si vous souhaitez attribuer un membre à une [organisation](/docs/fr-CA/manage-users/organizations/organizations-overview), mais que l’utilisateur n’existe pas encore dans votre banque de données (ou qu’il existe déjà dans votre banque de données, sans être encore membre de votre organisation), vous pouvez l’inviter à rejoindre l’organisation. Dans ce cas, l’utilisateur recevra un courriel contenant un lien lui permettant de créer un compte ou d’ouvrir une session et de rejoindre l’organisation, au besoin avec des rôles prédéfinis, puis d’être redirigé vers votre application afin de pouvoir commencer à l’utiliser immédiatement.

<Warning>
  Les rôles de membre d’organisation sont distincts des rôles attribués aux utilisateurs au moyen du [Role-Based Access Control](/docs/fr-CA/manage-users/access-control/rbac) d’Auth0. Lorsqu’un rôle prédéfini est appliqué à un utilisateur qui accepte une invitation à une organisation, le rôle attribué s’applique uniquement lorsque l’utilisateur ouvre une session au moyen de cette organisation. Vous pouvez [voir les rôles des membres d’organisation](/docs/fr-CA/manage-users/organizations/configure-organizations/retrieve-member-roles) dans le Dashboard.
</Warning>

Pour configurer le flux d’invitation des membres :

1. [Intégrer le flux d’invitation des membres à votre application](#integrate-the-membership-invitation-flow-into-your-application)
2. [Personnaliser le flux d’invitation utilisateur](#customize-the-user-invitation-flow)
3. [Envoyer des invitations aux membres](#send-membership-invitations)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Limites**

  * Auth0 peut envoyer des invitations utilisateur uniquement par courriel. Si vous préférez envoyer l’invitation autrement, vous pouvez choisir de générer uniquement une URL d’invitation.
  * Les utilisateurs invités doivent ouvrir une session ou créer un compte avec l’adresse courriel à laquelle l’invitation a été envoyée.
</Callout>

<div id="integrate-the-membership-invitation-flow-into-your-application">
  ## Intégrez le flux d’invitation des membres à votre application
</div>

Pour que le mécanisme d’invitation des utilisateurs fonctionne, vous devez configurer une route dans votre application dans le cadre du flux d’acceptation de l’invitation et [configurer une route de connexion par défaut associée](/docs/fr-CA/authenticate/login/auth0-universal-login/configure-default-login-routes) pour votre tenant ou votre application. Un lien vers l’URI configurée sera inclus dans le courriel d’invitation envoyé aux utilisateurs.

<div id="configure-the-uri">
  ### Configurer l’URI
</div>

Si l’URI doit être commune à toutes vos applications, vous pouvez définir le **Tenant Login URL** dans [Auth0 Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced). Sinon, vous pouvez définir l’URI pour chaque application à l’aide de **URI de connexion de l’application** dans les paramètres de votre application, auxquels vous pouvez accéder depuis [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).

<div id="specify-route-behavior">
  ### Spécifier le comportement de la route
</div>

La route de votre application doit accepter les paramètres `invitation` et `organization` dans la chaîne de requête, puis appeler le endpoint [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de l’API d’authentification Auth0 avec ces deux paramètres.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour prendre en charge les scénarios multilocataires où le nom de l’organisation est utilisé comme sous-domaine ou variable de chemin dans votre application (par exemple, `acme.myapp.com`), le paramètre `organization_name` est également inclus dans le lien d’invitation envoyé aux utilisateurs. Il n’est pas nécessaire d’envoyer ce paramètre au endpoint [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de l’API d’authentification Auth0.
</Callout>

Par exemple, si vous avez une application compatible avec les organisations avec un **URI de connexion de l’application** défini sur `https://myapp.com/login`, le lien dans l’invitation par courriel envoyée à l’utilisateur final serait : `https://myapp.com/login?invitation={invitation_ticket_id}&organization={organization_id}&organization_name={organization_name}`.

<div id="customize-the-user-invitation-flow">
  ## Personnaliser le flux d’invitation des utilisateurs
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour personnaliser le flux d’invitation des utilisateurs, vous devez utiliser des domaines personnalisés. Les forfaits Free comprennent un domaine personnalisé gratuit; des domaines personnalisés supplémentaires sont offerts avec tout forfait payant.
</Callout>

Pour personnaliser l’écran d’invitation de l’utilisateur et le courriel qu’il reçoit, modifiez le prompt de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> ainsi que les modèles de courriel. Pour en savoir plus, consultez ces pages :

* [Modèles de page Universal Login](/docs/fr-CA/customize/login-pages/universal-login/customize-templates)
* [Internationalisation d’Universal Login](/docs/fr-CA/customize/internationalization-and-localization/universal-login-internationalization)
* [Personnaliser les modèles de courriel](/docs/fr-CA/customize/email/email-templates)

<div id="localization">
  ### Localisation
</div>

Comme la langue de l’utilisateur invité n’est pas connue d’avance, vous pouvez envoyer un paramètre facultatif `user_metadata` à l’endpoint d’invitation à l’aide de la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Lors de la localisation du flux d’invitation des utilisateurs, vous pouvez accéder à ce paramètre au moyen du modèle de courriel **Invitation utilisateur** et [utiliser la même logique que pour les autres modèles de courriel multilingues](/docs/fr-CA/customize/email/email-templates).

Pour que l’utilisateur invité voie l’invitation dans la bonne langue, votre application devra envoyer le paramètre `ui_locales`, ou le navigateur de l’utilisateur invité devra envoyer l’en-tête `Accept-Language` approprié. Pour en savoir plus, consultez [Personnaliser les modèles de courriel : Modèles de courriel multilingues](/docs/fr-CA/customize/email/email-templates).

<div id="send-membership-invitations">
  ## Envoyer des invitations aux membres
</div>

Vous pouvez [envoyer des invitations aux membres d’une organisation](/docs/fr-CA/manage-users/organizations/configure-organizations/send-membership-invitations) aux utilisateurs au moyen de l’<Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la Management API. Si vous préférez envoyer l’invitation à l’aide de votre propre service de courriel, vous pouvez choisir de générer uniquement une URL d’invitation. Sinon, l’utilisateur recevra un courriel contenant le lien, ce qui lui permettra de se connecter à l’organisation à l’aide d’une connexion configurée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Lorsqu’un utilisateur accepte une invitation, son adresse courriel sera marquée comme vérifiée.

  Si l’utilisateur accepte une invitation au moyen d’une connexion fédérée, l’identity provider doit renvoyer l’adresse courriel correspondant à celle à laquelle l’invitation a été envoyée. L’email verification ne sera pas remplacée par la valeur renvoyée par l’identity provider lors des connexions ultérieures.

  Si l’utilisateur n’accepte pas l’invitation avant son expiration, l’invitation sera marquée comme expirée et finira par être supprimée de l’[Auth0 Dashboard](/docs/fr-CA/get-started/auth0-overview/dashboard).
</Callout>
