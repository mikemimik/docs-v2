---

<Card title="La disponibilité varie selon le plan Auth0">
  Votre mise en œuvre de connexion ainsi que votre plan Auth0 ou votre entente personnalisée déterminent la disponibilité de cette fonctionnalité. Pour en savoir plus, consultez [Tarification](https://auth0.com/pricing).
</Card>

Si vous souhaitez attribuer un membre à une [organisation](/docs/fr-CA/manage-users/organizations/organizations-overview), mais que l’utilisateur n’existe pas encore dans votre banque de données (ou qu’il existe déjà dans votre banque de données, sans être encore membre de votre organisation), vous pouvez l’inviter à rejoindre l’organisation. Dans ce cas, l’utilisateur recevra un courriel contenant un lien lui permettant de créer un compte ou de se connecter, puis de rejoindre l’organisation, éventuellement avec des rôles prédéfinis, avant d’être redirigé vers votre application pour pouvoir commencer à l’utiliser immédiatement.

<Warning>
  Les rôles des membres d’une organisation sont distincts des rôles attribués aux utilisateurs au moyen du [Role-Based Access Control](/docs/fr-CA/manage-users/access-control/rbac) d’Auth0. Lorsqu’un rôle prédéfini est appliqué à un utilisateur qui accepte une invitation à une organisation, ce rôle s’applique uniquement lorsque l’utilisateur se connecte par l’entremise de cette organisation. Vous pouvez [voir les rôles des membres d’organisation](/docs/fr-CA/manage-users/organizations/configure-organizations/retrieve-member-roles) dans le Dashboard.
</Warning>

Pour configurer le flux d’invitation à l’adhésion :

1. [Intégrer le flux d’invitation à l’adhésion à votre application](#integrate-the-membership-invitation-flow-into-your-application)
2. [Personnaliser le flux d’invitation d’utilisateur](#customize-the-user-invitation-flow)
3. [Envoyer des invitations d’adhésion](#send-membership-invitations)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Limites**

  * Auth0 ne peut envoyer des invitations d’utilisateur que par courriel. Si vous préférez envoyer l’invitation autrement, vous pouvez choisir de générer uniquement une URL d’invitation.
  * Les utilisateurs invités doivent se connecter ou créer un compte avec l’adresse courriel à laquelle l’invitation a été envoyée.
</Callout>

<div id="integrate-the-membership-invitation-flow-into-your-application">
  ## Intégrez le flux d’invitation à l’adhésion à votre application
</div>

Pour que les invitations d’utilisateurs fonctionnent, vous devez configurer une route dans votre application dans le cadre du processus d’acceptation de l’invitation et [configurer une route de connexion par défaut associée](/docs/fr-CA/authenticate/login/auth0-universal-login/configure-default-login-routes) pour votre tenant ou votre application. Un lien vers l’URI configuré sera inclus dans le courriel d’invitation envoyé aux utilisateurs.

<div id="configure-the-uri">
  ### Configurer l’URI
</div>

Si l’URI doit être la même pour toutes vos applications, vous pouvez définir **Tenant Login URL** dans [Auth0 Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced). Sinon, vous pouvez définir l’URI pour chaque application à l’aide de **URI de connexion de l’application** dans les paramètres de votre application, accessibles depuis [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).

<div id="specify-route-behavior">
  ### Définir le comportement de la route
</div>

La route de votre application doit accepter les paramètres `invitation` et `organization` dans la chaîne de requête, puis appeler le point de terminaison [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de l’Auth0 Authentication API avec ces deux paramètres.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour prendre en charge les scénarios multitenant où le nom de l’organisation est utilisé comme sous-domaine ou variable de chemin dans votre application (par exemple, `acme.myapp.com`), le paramètre `organization_name` est aussi inclus dans le lien d’invitation envoyé aux utilisateurs. Il n’est pas nécessaire d’envoyer ce paramètre au point de terminaison [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de l’Auth0 Authentication API.
</Callout>

Par exemple, si vous avez une application avec prise en charge des organisations et qu’une **URI de connexion de l’application** est définie à `https://myapp.com/login`, le lien contenu dans l’invitation par courriel envoyée à l’utilisateur final serait : `https://myapp.com/login?invitation={invitation_ticket_id}&organization={organization_id}&organization_name={organization_name}`.

<div id="customize-the-user-invitation-flow">
  ## Personnaliser le processus d’invitation de l’utilisateur
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour personnaliser le processus d’invitation de l’utilisateur, vous devez utiliser des domaines personnalisés. Le plan Free comprend un domaine personnalisé gratuit; des domaines personnalisés supplémentaires sont offerts avec n’importe quel forfait payant.
</Callout>

Pour personnaliser le prompt d’invitation de l’utilisateur et le courriel qu’il reçoit, modifiez le prompt de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> ainsi que les modèles de courriel. Pour en savoir plus, consultez les pages suivantes :

* [Universal Login Page Templates](/docs/fr-CA/customize/login-pages/universal-login/customize-templates)
* [Universal Login Internationalization](/docs/fr-CA/customize/internationalization-and-localization/universal-login-internationalization)
* [Personnaliser les modèles de courriel](/docs/fr-CA/customize/email/email-templates)

<div id="localization">
  ### Localisation
</div>

Comme la langue de l’utilisateur invité n’est pas connue d’avance, vous pouvez envoyer un paramètre facultatif `user_metadata` au point de terminaison d’invitation à l’aide de l’<Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Pour localiser le processus d’invitation de l’utilisateur, vous pouvez accéder à ce paramètre dans le modèle de courriel **User Invitation** et [utiliser la même logique que pour les autres modèles de courriel multilingues](/docs/fr-CA/customize/email/email-templates).

Pour que l’utilisateur invité voie l’invitation dans la langue appropriée, votre application devra envoyer le paramètre `ui_locales`, ou le navigateur de l’utilisateur invité devra envoyer l’en-tête `Accept-Language` approprié. Pour en savoir plus, consultez [Personnaliser les modèles de courriel : modèles de courriel multilingues](/docs/fr-CA/customize/email/email-templates).

<div id="send-membership-invitations">
  ## Envoyer des invitations d’adhésion
</div>

Vous pouvez [envoyer des invitations d’adhésion à l’organisation](/docs/fr-CA/manage-users/organizations/configure-organizations/send-membership-invitations) aux utilisateurs au moyen de l’<Tooltip tip="Auth0 Dashboard: le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la Management API. Si vous préférez envoyer l’invitation à l’aide de votre propre service de courriel, vous pouvez choisir de générer seulement une URL d’invitation. Sinon, l’utilisateur recevra un courriel contenant le lien, ce qui lui permettra de se connecter à l’organisation au moyen d’une connexion configurée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Lorsqu’un utilisateur accepte une invitation, son adresse courriel est marquée comme vérifiée.

  Si l’utilisateur accepte une invitation au moyen d’une connexion fédérée, le fournisseur d’identité doit alors renvoyer l’adresse courriel correspondant à celle à laquelle l’invitation a été envoyée. La vérification de l’adresse courriel ne sera pas remplacée par la valeur renvoyée par le fournisseur d’identité lors des connexions suivantes.

  Si l’utilisateur n’accepte pas une invitation avant son expiration, l’invitation sera marquée comme expirée et finira par être supprimée de l’[Auth0 Dashboard](/docs/fr-CA/get-started/auth0-overview/dashboard).
</Callout>
