Vous pouvez retirer des membres des [organisations](/docs/fr-CA/manage-users/organizations/organizations-overview) à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Retirer un utilisateur d’une organisation supprime uniquement l’adhésion de cet utilisateur à l’organisation en question; cela ne supprime pas l’utilisateur de la connexion ni du tenant Auth0. Le processus de suppression prend environ 5 à 10 secondes.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Pour supprimer un membre dans l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’organisation pour laquelle vous souhaitez configurer l’adhésion.
2. Sélectionnez la vue **Members**, repérez le membre que vous souhaitez supprimer, puis ouvrez son menu **More Options** (**...**).
3. Sélectionnez **Remove member**, puis confirmez.

<div id="management-api">
  ## Management API
</div>

Pour supprimer des membres à l’aide de la Management API :
Effectuez un appel `DELETE` au point de terminaison `Delete Organization Members`. Assurez-vous de remplacer les valeurs de l’espace réservé `ORG_ID`, `MGMT_API_ACCESS_TOKEN` et `USER_ID` par votre ID d’organisation, votre jeton d’accès de la Management API <Tooltip tip="Jeton d’accès : information d’authentification, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> et votre ID utilisateur, respectivement. Vous pouvez supprimer jusqu’à 10 membres par requête.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Votre domaine Auth0 correspond au nom de votre tenant, suivi de votre sous-domaine régional (à moins que votre tenant se trouve dans la région des États-Unis et ait été créé avant juin 2020), puis de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir de votre nom de domaine personnalisé.
</Callout>

| Valeur                  | Description                                                                                                                                         |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de l’organisation à partir de laquelle vous voulez retirer des membres.                                                                          |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `delete:organization_members`. |
| `USER_ID`               | ID du ou des utilisateurs que vous voulez retirer de l’organisation spécifiée.                                                                      |

<div id="response-status-codes">
  ### Codes d’état de la réponse
</div>

Les codes d’état de réponse possibles sont les suivants :

| Code d’état | Code d’erreur          | Message                                                                                                | Cause                                                                                       |
| ----------- | ---------------------- | ------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------- |
| `204`       |                        | Utilisateurs supprimés de l’organisation avec succès.                                                  |                                                                                             |
| `400`       | `invalid_body`         | Corps de la requête invalide. Le message varie selon la cause.                                         | La charge utile de la requête n’est pas valide.                                             |
| `400`       | `invalid_query_string` | Chaîne de requête invalide. Le message varie selon la cause.                                           | La chaîne de requête n’est pas valide.                                                      |
| `401`       |                        | Jeton invalide.                                                                                        |                                                                                             |
| `401`       |                        | Signature invalide reçue pour la validation du JSON Web Token.                                         |                                                                                             |
| `401`       |                        | Le client n’est pas global.                                                                            |                                                                                             |
| `403`       | `insufficient_scope`   | Portée insuffisante ; l’une des portées suivantes est requise : `delete:organization_members`.         | Tentative de lire ou d’écrire un champ non autorisé par les portées du bearer token fourni. |
| `429`       |                        | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                             |
