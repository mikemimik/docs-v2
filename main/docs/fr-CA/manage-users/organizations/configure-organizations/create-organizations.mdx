Vous pouvez créer des [organisations](/docs/fr-CA/manage-users/organizations/organizations-overview) en utilisant soit le <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, soit la <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Pour créer une organisation dans l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations).
2. Sélectionnez **Créer une organisation**.
3. Entrez les renseignements de base de votre organisation, puis sélectionnez **Ajouter une organisation** :

   | **Champ**           | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                 |
   | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Nom**             | Nom de l’organisation que vous souhaitez créer. C’est le nom qu’un utilisateur final saisirait dans l’invite pre-login pour indiquer l’organisation par laquelle il souhaite se connecter. Il s’agit d’un identifiant logique unique. Il peut contenir des lettres minuscules, des chiffres, des traits de soulignement (`_`) et des tirets (`-`). Il peut commencer par un chiffre. Il doit compter entre 1 et 50 caractères. |
   | **Nom d’affichage** | Nom convivial à afficher.                                                                                                                                                                                                                                                                                                                                                                                                       |
4. Repérez la section **Branding** et personnalisez votre organisation, puis sélectionnez **Enregistrer les modifications** :

   | **Champ**                             | **Description**                                                                                           |
   | ------------------------------------- | --------------------------------------------------------------------------------------------------------- |
   | **Logo de l’organisation**            | Logo à afficher. La résolution minimale recommandée est de 200 pixels (largeur) sur 200 pixels (hauteur). |
   | **Couleur principale**                | Couleur des éléments principaux.                                                                          |
   | **Couleur d’arrière-plan de la page** | Couleur de l’arrière-plan.                                                                                |
5. Repérez la section **Metadata** et ajoutez à l’organisation toutes les paires clé-valeur de métadonnées nécessaires, puis sélectionnez **Ajouter**.

<div id="management-api">
  ## Management API
</div>

Pour créer une organisation au moyen de la Management API :

Effectuez un appel `POST` au point de terminaison `Create Organizations`. Assurez-vous de remplacer les valeurs de remplacement par les valeurs appropriées de votre tenant. Pour en savoir plus, consultez le tableau des paramètres ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Votre domaine Auth0 est composé du nom de votre tenant, de votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), puis de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir de votre nom de domaine personnalisé.
</Callout>

| **Valeur**                 | **Description**                                                                                                                                                                                                                                                                                                                                                                                                 |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN`    | [Jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec le scope `create:organizations`.                                                                                                                                                                                                                                                                    |
| `ORG_NAME`                 | Nom de l’organisation que vous souhaitez créer. Il s’agit du nom qu’un utilisateur final saisirait dans l’invite pre-login pour indiquer l’organisation par laquelle il veut se connecter. Identifiant logique unique. Peut contenir des lettres minuscules, des chiffres, des traits de soulignement (`_`) et des traits d’union (`-`). Peut commencer par un chiffre. Doit contenir entre 1 et 50 caractères. |
| `ORG_DISPLAY_NAME`         | Facultatif. Nom convivial de l’organisation pouvant être affiché dans le flux de connexion et les modèles de courriel.                                                                                                                                                                                                                                                                                          |
| `ORG_LOGO`                 | Facultatif. URL du logo de l’organisation.                                                                                                                                                                                                                                                                                                                                                                      |
| `ORG_PRIMARY_COLOR`        | Facultatif. Code de couleur HEX des éléments principaux.                                                                                                                                                                                                                                                                                                                                                        |
| `ORG_BACKGROUND_COLOR`     | Facultatif. Code de couleur HEX de l’arrière-plan.                                                                                                                                                                                                                                                                                                                                                              |
| `KEY`/`VALUE`              | Facultatif. Paires clé-valeur de type String représentant les métadonnées de l’organisation. Maximum de 255 caractères chacune. Maximum de 25 paires de métadonnées.                                                                                                                                                                                                                                            |
| `CONNECTION_ID`            | Facultatif. ID de la connexion que vous souhaitez activer pour l’organisation indiquée. Les connexions activées s’affichent dans l’invite de connexion de l’organisation, afin que les utilisateurs puissent accéder à votre ou vos applications par leur intermédiaire.                                                                                                                                        |
| `ASSIGN_MEMBERSHIP_OPTION` | Facultatif. Indique si vous voulez que les utilisateurs qui se connectent avec la connexion activée obtiennent automatiquement un statut de membre dans l’organisation indiquée. Lorsque la valeur est `true`, les utilisateurs obtiennent automatiquement ce statut. Lorsque la valeur est `false`, ils ne l’obtiennent pas automatiquement.                                                                   |

| **Code d’état** | **Code d’erreur**       | **Message**                                                                                            | **Cause**                                                                                       |
| --------------- | ----------------------- | ------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------- |
| `201`           |                         | Organisation créée avec succès.                                                                        |                                                                                                 |
| `400`           | `invalid_body`          | Corps de la requête invalide. Le message varie selon la cause.                                         | La charge utile de la requête n’est pas valide.                                                 |
| `400`           | `invalid_query_string`  | Chaîne de requête invalide. Le message varie selon la cause.                                           | La chaîne de requête n’est pas valide.                                                          |
| `401`           |                         | Jeton invalide.                                                                                        |                                                                                                 |
| `401`           |                         | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                                 |
| `401`           |                         | Le client n’est pas global.                                                                            |                                                                                                 |
| `403`           | `insufficient_scope`    | Portée insuffisante ; l’une des valeurs suivantes est requise : `create:organizations`.                | Tentative de lire ou de modifier un champ non autorisé avec les portées du bearer token fourni. |
| `409`           | `organization_conflict` | Une organisation portant le même nom existe déjà.                                                      | Une organisation portant le même nom existe déjà.                                               |
