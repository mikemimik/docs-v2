Lorsque vous travaillez avec [Auth0 Organizations](/docs/fr-CA/manage-users/organizations/organizations-overview) par programmation, vous pourriez avoir besoin de récupérer une liste de connexions pour une organisation.

Vous pouvez consulter les connexions associées à une organisation au moyen de <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’organisation.
2. Sélectionnez la vue **Connections**.

<div id="management-api">
  ## Management API
</div>

Effectuez un appel `GET` au point de terminaison `Get Organization Connections`. Assurez-vous de remplacer les valeurs d’exemple `ORG_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre organisation et le <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, respectivement.

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/organizations/ORG_ID/connections' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/organizations/ORG_ID/connections");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  	"net/http"
  	"io/ioutil"
  ```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Votre domaine Auth0 correspond au nom de votre tenant, à votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), puis à `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des [domaines personnalisés](/docs/fr-CA/customize/custom-domains), il doit s’agir de votre nom de domaine personnalisé.
</Callout>

| Valeur                  | Description                                                                                                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ORG_ID`                | ID de l’Organisation dont vous voulez récupérer les connexions.                                                                                        |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) ayant la portée `read:organization_connections`. |

<div id="response-status-codes">
  ### Codes d’état de réponse
</div>

Les codes d’état de réponse possibles sont les suivants :

| Code d’état | Code d’erreur        | Message                                                                                                | Cause                                                                                              |
| ----------- | -------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| `200`       |                      | Connexions récupérées avec succès.                                                                     |                                                                                                    |
| `400`       | `invalid_body`       | Corps de la requête non valide. Le message varie selon la cause.                                       | La charge utile de la requête n’est pas valide.                                                    |
| `401`       |                      | Jeton invalide.                                                                                        |                                                                                                    |
| `401`       |                      | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                                    |
| `401`       |                      | Le client n’est pas global.                                                                            |                                                                                                    |
| `403`       | `insufficient_scope` | Portée insuffisante ; une des portées suivantes est requise : `read:organization_connections`.         | Tentative de lire ou de modifier un champ non autorisée avec les portées fournies du jeton Bearer. |
| `429`       |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                    |
