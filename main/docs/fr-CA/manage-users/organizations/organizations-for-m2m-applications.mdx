Le [Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow) est utilisé par une application pour obtenir des <Tooltip tip="Jeton d’accès : identifiant d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+tokens">jetons d’accès</Tooltip> pour elle-même plutôt que pour un utilisateur. Cela s’avère utile dans des cas d’utilisation machine à machine, comme les robots, les CLI, les processus backend ou les processus démon, ou toute application qui consomme des API en son propre nom. Même si les utilisateurs finaux ne sont pas impliqués, le Client Credentials Flow devrait tout de même être limité à chaque organisation afin de s’assurer que seules les applications autorisées peuvent accéder aux données de chaque organisation.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour limiter chaque requête à une organisation donnée, incluez le claim `org_id` correspondant dans les jetons machine à machine. Les serveurs API doivent tenir compte de ce claim et l’appliquer lorsqu’ils autorisent l’accès aux ressources et aux données d’API. Pour en savoir plus, consultez [Utiliser les jetons et les Organizations](/docs/fr-CA/manage-users/organizations/using-tokens).
</Callout>

L’accès machine à machine pour les Organizations vous permet de définir les organisations auxquelles une application donnée peut accéder pour chaque API au moyen du Client Credentials Flow. Pour en savoir plus, consultez ce qui suit :

<div id="open-up-apis-to-third-party-applications">
  ## Ouvrir les API aux applications tierces
</div>

À l’aide d’Auth0 Organizations et de la <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, Travel0 offre un portail libre-service où les clients peuvent créer et gérer leur propre organisation. Travel0 veut permettre facilement à ses clients de créer des bots pour aider les utilisateurs finaux à trouver et à acheter des aventures. C’est pourquoi, dans ce portail, Travel0 permet aux clients d’enregistrer leurs applications (p. ex. les bots) afin d’utiliser la Travel0 API en leur propre nom au moyen d’un accès machine-to-machine. Dans ce cas d’usage, l’accès entre organisations doit être rigoureusement contrôlé afin que les applications appartenant à une organisation ne puissent accéder qu’aux données de cette organisation par l’intermédiaire de la Travel0 API.

**accès machine à machine pour les organisations vous permet de configurer l’accès Client Credentials pour chaque API en l’associant à une organisation précise**. Dans le diagramme suivant, les applications de notre organisation d’exemple, org&#95;X, ne peuvent accéder à la Travel0 API que dans le contexte de org&#95;X. Vous pouvez aussi configurer l’accès d’une même application à plusieurs organisations lorsque des agrégateurs sont utilisés. Avec accès machine à machine pour les organisations, vous contrôlez, API par API, quelles applications peuvent accéder à une organisation donnée au moyen d’un accès machine-to-machine.

<div id="isolation-of-organizations-for-internal-applications">
  ## Isolement des organisations pour les applications internes
</div>

Travel0 a aussi certains processus internes et outils CLI qui doivent accéder à l’API Travel0 au moyen du Client Credentials Flow. Pour appliquer une stratégie unifiée de contrôle d’accès à l’API, Travel0 veut que les requêtes provenant de ses propres applications soient limitées à une organisation précise afin de garantir que seules les données appropriées sont accessibles dans chaque cas.

<Frame>![](/docs/images/cdy7uua7fh8z/5bMEPlmTLGgBn8fq5Fp59S/fe874efcfba549114ad062bd7e3d874d/image7.png)</Frame>

Machine-to-Machine Access to Organizations vous permet de configurer une application afin qu’elle puisse accéder à n’importe quelle organisation pour chaque API de votre tenant au moyen du Client Credentials Flow.
