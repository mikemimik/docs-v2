<Card title="Avant de commencer">
  Assurez-vous d’avoir :

  * Un tenant Auth0 avec un [Custom Domain](/docs/fr-CA/customize/custom-domains) configuré pour votre application

  * [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_actions.html) installé et authentifié.

  * Une [Action post-login](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)

  * Une session de tenant Auth0 utilisant la politique de sessions [non persistantes](/docs/fr-CA/manage-users/sessions/session-lifecycle)

  * (Facultatif) Configurer les valeurs du délai d’inactivité et du délai d’expiration absolu de la session
</Card>

L’expérience **Keep Me Signed In** permet aux utilisateurs de rester connectés sur un appareil de confiance.

<Frame>![Image de connexion](/docs/images/cdy7uua7fh8z/sessions-use-case/keep-me-signed-in-login.png)</Frame>

* Si la case à cocher **Keep Me Signed In** est sélectionnée, une session persistante est établie et reste active même après le redémarrage du navigateur.

* Si la case à cocher **Keep Me Signed In** n’est pas sélectionnée, une session non persistante est établie et expire à la fermeture du navigateur.

En utilisant des [sessions non persistantes (éphémères)](/docs/fr-CA/manage-users/sessions/session-lifecycle) avec [Actions](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger), le serveur gère cette logique de façon sécurisée sans exposer les jetons d’actualisation à l’interface client.

En utilisant des prompts personnalisés, Auth0 CLI et des sessions éphémères via Actions, vous pouvez mettre en place un flux « Keep Me Signed In » sécurisé et moderne, sans jetons d’actualisation, sans contournements de session côté client et avec un contrôle complet du backend.

Cette solution est idéale pour les clients qui privilégient :

* Une gestion des sessions de navigateur fondée sur le modèle Zero Trust

* Une persistance minimale des sessions

* Des dérogations pour les appareils de confiance

<div id="1-configure-a-universal-login-page-and-add-a-custom-prompt">
  ## 1. Configurer une page Universal Login et ajouter un prompt personnalisé
</div>

Pour recueillir la préférence de l’utilisateur pour **Keep Me Signed In**, vous devez personnaliser [Universal Login](/docs/fr-CA/customize/login-pages/universal-login/customize-signup-and-login-prompts) en configurant un gabarit de page et en ajoutant un prompt personnalisé à l’aide d’Auth0 CLI.

<div id="optional-enable-the-universal-login-page-template">
  ### Facultatif : activer le gabarit de page Universal Login
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cette étape n’est nécessaire que si aucun gabarit de page Universal Login n’est configuré pour votre tenant.
</Callout>

Utilisez la commande suivante pour créer ou mettre à jour le gabarit de page :

```bash lines
auth0 universal-login templates update
```

L’éditeur de personnalisation ouvre le gabarit de la page de connexion. Vous pouvez utiliser n’importe quelle structure HTML qui convient à votre image de marque.

Voici un exemple minimal de modèle avec un arrière-plan et un widget dans un conteneur :

```html lines
<!DOCTYPE html>
<html lang="{{locale}}">
  <head>
    {%- auth0:head -%}
    <style>
      body {
        background-image: url("https://images.unsplash.com/photo-1643916861364-02e63ce3e52f");
        background-size: cover;
        background-position: center;
      }
      .prompt-wrapper {
        display: flex;
        align-items: center;
        justify-content: center;
        height: 100%;
        background-color: rgba(0, 0, 0, 0.3);
      }
    </style>
    <title>{{ prompt.screen.texts.pageTitle }}</title>
  </head>
  <body class="_widget-auto-layout">
    {% if prompt.name == "login" or prompt.name == "signup" %}
      <div class="prompt-wrapper">
        {%- auth0:widget -%}
      </div>
    {% else %}
      {%- auth0:widget -%}
    {% endif %}
  </body>
</html>
```

Enregistrez et fermez l’éditeur, puis Auth0 CLI vous demandera de confirmer :

```bash lines
Voulez-vous enregistrer le modèle ? Oui

▸ Le modèle pour la nouvelle expérience Universal Login a été mis à jour avec succès!
```

<div id="2-add-a-keep-me-signed-in-checkbox-using-auth0-cli">
  ## 2. Ajouter une case à cocher Keep Me Signed In avec Auth0 CLI
</div>

Pour ajouter une case à cocher à l’écran Universal Login, suivez ces étapes :

<div id="launch-the-customization-editor">
  ### Ouvrez l’éditeur de personnalisation
</div>

1. Ouvrez l’éditeur de personnalisation du prompt login-id ou du prompt de connexion, selon la configuration de votre tenant :

```bash lines
auth0 universal-login customize -p login-id
```

2. Sélectionnez **standard** comme mode de rendu

```bash lines
Rendering Mode: standard
```

L’éditeur de personnalisation s’ouvre à une adresse locale :

```bash lines
http://localhost:52649
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous ne savez pas si votre tenant utilise `login` ou `login-id`, consultez la [personnalisation des prompts](/docs/fr-CA/customize/login-pages/universal-login/customize-signup-and-login-prompts) d’Auth0.
</Callout>

<div id="configure-the-checkbox">
  ### Configurer la case à cocher
</div>

Ouvrez l’option **Partials** dans l’éditeur de personnalisation :

1. Réglez **Prompt** à : `login-id` (ou `login`)

2. Réglez **Screen** à : `login-id` (ou `login`)

3. Réglez **Partial** à : `form-content-end`

<Frame>![Auth0 CLI partials](/docs/images/cdy7uua7fh8z/sessions-use-case/partials.png)</Frame>

Ajoutez l’extrait HTML suivant :

```html lines
<div class="ulp-field">
  <input type="checkbox" name="ulp-remember-me" id="remember-me">
  <label for="remember-me">Stay signed in on this device?</label>
</div>
```

Cela affiche la case à cocher sous le formulaire de connexion.

<div id="deploy-your-changes">
  ### Déployer vos modifications
</div>

Sélectionnez **Déployer les modifications** en haut à droite pour enregistrer et appliquer la personnalisation.

  ## Checkpoint

Connectez-vous à l’aide de Universal Login.

<div id="3-create-a-post-login-action-using-auth0-cli">
  ## 3. Créer une Post-Login Action à l’aide d’Auth0 CLI
</div>

Créez un nouveau fichier nommé `set-session-persistence.js` pour l’utiliser avec Auth0 CLI et ajoutez le code suivant :

```javascript lines
exports.onExecutePostLogin = async (event, api) => {

  // Appliquer le changement uniquement lorsque le paramètre est disponible
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode('persistent');
  }
};
```

Cette Action fait en sorte que le cookie de session reste enregistré après le redémarrage du navigateur lorsque l’utilisateur coche la case. Pour prolonger également la durée de vie de la session sur les appareils de confiance, consultez [Définir la durée de la session pour les appareils de confiance](#set-session-duration-for-trusted-devices) ci-dessous.

Utilisez Auth0 CLI pour créer ou mettre à jour votre Action post-login à l’aide du fichier `set-session-persistence.js` :

```bash lines
auth0 actions create --name kmsi --trigger post-login --code "$(cat set-session-persistence.js)"
```

Auth0 CLI renvoie l’ID de l’Action :

```bash lines
ID             f54c4aae-431f-4290-a7e0-cfc6a3f28386
NAME           kmsi
TYPE           post-login
STATUS         pending
DEPLOYED       ✗
LAST DEPLOYED
LAST UPDATED   0 seconds ago
CREATED        0 seconds ago

                   if (event.request.body['ulp-remember-me']) {
                     api.session.setCookieMode("persistent");
                   } else {
                     api.session.setCookieMode("non-persistent");
                   }
                 };
```

Déployez votre Action à l’aide de son `id` :

```bash lines
auth0 actions deploy <id>
```

Accédez à [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post-Login](https://manage.auth0.com/#/actions/triggers/post-login) pour ajouter la nouvelle action au flux en cours, puis sélectionnez **Apply**.

<div id="set-session-duration-for-trusted-devices">
  ### Définir la durée de la session pour les appareils de confiance
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [`setCookieMode()`](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object#api-session-setcookiemode-mode) contrôle uniquement la façon dont le cookie de session est transmis au navigateur. Un témoin persistant reste en place même après le redémarrage du navigateur, tandis qu’un témoin non persistant (de session) est supprimé à la fermeture du navigateur. Cela ne modifie pas le moment d’expiration de la session côté serveur. Si vous voulez que les appareils de confiance aient une durée de vie de session plus longue, vous devez définir explicitement l’expiration à l’aide de `api.session.setExpiresAt()` ou de `api.session.setIdleExpiresAt()`.
</Callout>

Pour accorder aux appareils de confiance une durée de vie de session plus longue, utilisez `api.session.setExpiresAt()` ou `api.session.setIdleExpiresAt()` avec `setCookieMode()` :

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode("persistent");

    const now = Date.now();
    api.session.setExpiresAt(now + 7 * 24 * 60 * 60 * 1000); // 7 jours
  } else {
    api.session.setCookieMode("nonpersistent");
  }
};
```

<div id="4-test-implementation">
  ## 4. Tester la mise en œuvre
</div>

Pour vérifier le comportement de la session, vous pouvez :

Connectez-vous à l’aide d’Universal Login, puis cochez la case **Keep Me Signed** In.

* Dans les journaux du tenant, recherchez (types d’événements `s` ou `ssa`)

  * `cookie: { mode: "non-persistent" }`

  * `cookie: { mode: "persistent" }`

* Recherchez `event.session.cookie.mode` dans vos Actions
