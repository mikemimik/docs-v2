<Card title="Avant de commencer">
  Assurez-vous d’avoir :

  * Un [domaine personnalisé](/docs/fr-CA/customize/custom-domains) configuré pour votre application dans votre tenant Auth0

  * [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_actions.html) installé et authentifié.

  * Une [Action post-login](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)

  * Une session de tenant Auth0 utilisant la politique de sessions [non persistantes](/docs/fr-CA/manage-users/sessions/session-lifecycle)

  * (Facultatif) Configurer les valeurs du délai d’inactivité de la session et du délai d’expiration absolu
</Card>

L’expérience **Keep Me Signed In** permet aux utilisateurs de rester connectés sur un appareil de confiance.

<Frame>![Image de connexion](/docs/images/cdy7uua7fh8z/sessions-use-case/keep-me-signed-in-login.png)</Frame>

* Si la case à cocher **Keep Me Signed In** est sélectionnée, une session persistante est créée et reste active même après le redémarrage du navigateur.

* Si la case à cocher **Keep Me Signed In** n’est pas sélectionnée, une session non persistante est créée et expire à la fermeture du navigateur.

En utilisant des [sessions non persistantes (éphémères)](/docs/fr-CA/manage-users/sessions/session-lifecycle) avec [Actions](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger), le serveur gère cette logique de manière sécuritaire, sans exposer de jetons d’actualisation à l’interface utilisateur.

À l’aide de prompts personnalisés, d’Auth0 CLI et de sessions éphémères via Actions, vous pouvez mettre en place un flux « Keep Me Signed In » moderne et sécurisé, sans jetons d’actualisation, sans contournements de session côté client et avec un contrôle complet du backend.

Cette solution est idéale pour les clients qui privilégient :

* Une gestion des sessions de navigateur fondée sur le principe de confiance zéro

* Une persistance minimale des sessions

* Des dérogations pour les appareils de confiance

<div id="1-configure-a-universal-login-page-and-add-a-custom-prompt">
  ## 1. Configurer une page Universal Login et ajouter un prompt personnalisé
</div>

Pour recueillir la préférence **Keep Me Signed In** de l’utilisateur, vous devez personnaliser [Universal Login](/docs/fr-CA/customize/login-pages/universal-login/customize-signup-and-login-prompts) en configurant un gabarit de page et en ajoutant un prompt personnalisé au moyen d’Auth0 CLI.

<div id="optional-enable-the-universal-login-page-template">
  ### Facultatif : Activer le gabarit de page Universal Login
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cette étape est requise uniquement si aucun gabarit Universal Login n’est configuré pour votre tenant.
</Callout>

Utilisez la commande suivante pour créer ou mettre à jour le gabarit de page :

```bash lines
auth0 universal-login templates update
```

L’éditeur de personnalisation ouvre le gabarit de la page de connexion. Vous pouvez utiliser n’importe quelle structure HTML qui cadre avec votre image de marque.

Voici un exemple minimal de gabarit avec un arrière-plan et un widget dans un conteneur :

```html lines
<!DOCTYPE html>
<html lang="{{locale}}">
  <head>
    {%- auth0:head -%}
    <style>
      body {
        background-image: url("https://images.unsplash.com/photo-1643916861364-02e63ce3e52f");
        background-size: cover;
        background-position: center;
      }
      .prompt-wrapper {
        display: flex;
        align-items: center;
        justify-content: center;
        height: 100%;
        background-color: rgba(0, 0, 0, 0.3);
      }
    </style>
    <title>{{ prompt.screen.texts.pageTitle }}</title>
  </head>
  <body class="_widget-auto-layout">
    {% if prompt.name == "login" or prompt.name == "signup" %}
      <div class="prompt-wrapper">
        {%- auth0:widget -%}
      </div>
    {% else %}
      {%- auth0:widget -%}
    {% endif %}
  </body>
</html>
```

Enregistrez et fermez l’éditeur; Auth0 CLI vous demande de confirmer :

```bash lines
Voulez-vous enregistrer le modèle ? Oui

▸ Le modèle pour la nouvelle expérience Universal Login a été mis à jour avec succès!
```

<div id="2-add-a-keep-me-signed-in-checkbox-using-auth0-cli">
  ## 2. Ajouter une case à cocher Keep Me Signed In à l’aide d’Auth0 CLI
</div>

Pour ajouter une case à cocher sur l’écran Universal Login, suivez les étapes suivantes :

<div id="launch-the-customization-editor">
  ### Ouvrez l’éditeur de personnalisation
</div>

1. Ouvrez l’éditeur de personnalisation pour le login-id ou le prompt de connexion, selon la configuration de votre tenant :

```bash lines
auth0 universal-login customize -p login-id
```

2. Sélectionnez **standard** comme mode de rendu

```bash lines
Rendering Mode: standard
```

L’éditeur de personnalisation se lance à une adresse locale :

```bash lines
http://localhost:52649
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous ne savez pas si votre tenant utilise `login` ou `login-id`, consultez la [personnalisation des prompts](/docs/fr-CA/customize/login-pages/universal-login/customize-signup-and-login-prompts) d’Auth0.
</Callout>

<div id="configure-the-checkbox">
  ### Configurer la case à cocher
</div>

Ouvrez l’option **Partials** dans l’éditeur de personnalisation :

1. Définissez **Prompt** sur : `login-id` (ou `login`)

2. Définissez **Screen** sur : `login-id` (ou `login`)

3. Définissez **Partial** sur : `form-content-end`

<Frame>![Partials d’Auth0 CLI](/docs/images/cdy7uua7fh8z/sessions-use-case/partials.png)</Frame>

Ajoutez l’extrait HTML suivant :

```html lines
<div class="ulp-field">
  <input type="checkbox" name="ulp-remember-me" id="remember-me">
  <label for="remember-me">Stay signed in on this device?</label>
</div>
```

Cela affiche la case à cocher sous le formulaire de connexion.

<div id="deploy-your-changes">
  ### Déployez vos modifications
</div>

Sélectionnez **Déployer les modifications** dans l’angle supérieur droit pour enregistrer et appliquer la personnalisation.

<div id="3-create-a-post-login-action-using-auth0-cli">
  ## 3. Créez une Post-Login Action avec Auth0 CLI
</div>

Créez un nouveau fichier nommé `set-session-persistence.js` pour l’utiliser avec Auth0 CLI, puis ajoutez le code suivant :

```javascript lines
exports.onExecutePostLogin = async (event, api) => {

  // Appliquer le changement uniquement lorsque le paramètre est disponible
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode('persistent');
  }
};
```

Cette Action permet de conserver le cookie de session après le redémarrage du navigateur lorsque l’utilisateur coche la case. Pour prolonger aussi la durée de la session sur les appareils de confiance, consultez [Définir la durée de la session pour les appareils de confiance](#set-session-duration-for-trusted-devices) ci-dessous.

Utilisez Auth0 CLI pour créer ou mettre à jour votre Action post-login à l’aide du fichier `set-session-persistence.js` :

```bash lines
auth0 actions create --name kmsi --trigger post-login --code "$(cat set-session-persistence.js)"
```

Auth0 CLI renvoie l’ID de l’Action :

```bash lines
ID             f54c4aae-431f-4290-a7e0-cfc6a3f28386
NAME           kmsi
TYPE           post-login
STATUS         pending
DEPLOYED       ✗
LAST DEPLOYED
LAST UPDATED   il y a 0 seconde
CREATED        il y a 0 seconde
CODE           exports.onExecutePostLogin = async (event, api) => {
                   if (event.request.body['ulp-remember-me']) {
                     api.session.setCookieMode("persistent");
                   } else {
                     api.session.setCookieMode("non-persistent");
                   }
                 };
```

Déployez votre Action à l’aide de son `id` :

```bash lines
auth0 actions deploy <id>
```

Accédez à [Auth0 Dashboard &gt; Actions &gt; Triggers &gt; Post-Login](https://manage.auth0.com/#/actions/triggers/post-login) pour ajouter la nouvelle action au flow actuel, puis cliquez sur **Apply**.

<div id="set-session-duration-for-trusted-devices">
  ### Définir la durée de la session pour les appareils de confiance
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [`setCookieMode()`](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object#api-session-setcookiemode-mode) contrôle uniquement la façon dont le cookie de session est transmis au navigateur. Un cookie persistant survit au redémarrage du navigateur, tandis qu’un cookie non persistant (de session) est effacé lorsque le navigateur se ferme. Cela ne modifie pas la date d’expiration de la session côté serveur. Si vous voulez que les appareils de confiance aient une durée de vie de session plus longue, vous devez définir explicitement l’expiration à l’aide de `api.session.setExpiresAt()` ou de `api.session.setIdleExpiresAt()`.
</Callout>

Pour donner aux appareils de confiance une durée de vie de session plus longue, utilisez `api.session.setExpiresAt()` ou `api.session.setIdleExpiresAt()` avec `setCookieMode()` :

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.request.body['ulp-remember-me']) {
    api.session.setCookieMode("persistent");

    const now = Date.now();
    api.session.setExpiresAt(now + 7 * 24 * 60 * 60 * 1000); // 7 jours
  } else {
    api.session.setCookieMode("nonpersistent");
  }
};
```

<div id="4-test-implementation">
  ## 4. Tester la mise en œuvre
</div>

Pour vérifier le comportement de la session, vous pouvez :

Vous connecter avec Universal Login, puis cocher la case **Keep Me Signed** In.

* Dans les journaux du tenant, recherchez (types d’événement `s` ou `ssa`)

  * `cookie: { mode: "non-persistent" }`

  * `cookie: { mode: "persistent" }`

* Recherchez `event.session.cookie.mode` dans vos Actions
