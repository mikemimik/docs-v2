---

Auth0 utilise des [sessions](/docs/fr-CA/manage-users/sessions) pour maintenir l’état d’authentification d’un utilisateur d’une requête à l’autre.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les points de terminaison de gestion des sessions d’Auth0 Management API sont offerts aux clients ayant un plan Enterprise. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Callout>

<div id="management-api-endpoints">
  ## Points de terminaison de la Management API
</div>

La [Management API](https://auth0.com/docs/api/management/v2) fournit un accès hors bande aux mécanismes internes des sessions utilisateur dans la [couche de session Auth0](/docs/fr-CA/manage-users/sessions/session-layers), ainsi que des méthodes de suppression pour en forcer la fin.

<div id="session-resource">
  ### Ressource de session
</div>

Vous pouvez consulter ou supprimer une session précise à l’aide des points de terminaison suivants :

| Nom                                  | URL                                   | Portée(s) requise(s) |
| ------------------------------------ | ------------------------------------- | -------------------- |
| Consulter une session précise par ID | `GET /api/v2/sessions/{sessionId}`    | `read:sessions`      |
| Supprimer une session précise par ID | `DELETE /api/v2/sessions/{sessionId}` | `delete:sessions`    |

<div id="user-resource">
  ### Ressource utilisateur
</div>

Vous pouvez afficher la liste de toutes les sessions d’un utilisateur donné ou les supprimer à l’aide des points de terminaison suivants :

| Nom                                                | URL                                      | Portée(s) requise(s) |
| -------------------------------------------------- | ---------------------------------------- | -------------------- |
| Afficher les détails des sessions d’un utilisateur | `GET /api/v2/users/{userId}/sessions`    | `read:sessions`      |
| Supprimer toutes les sessions de l’utilisateur     | `DELETE /api/v2/users/{userId}/sessions` | `delete:sessions`    |

<div id="session-properties">
  ## Propriétés de la session
</div>

Les points de terminaison de session renvoient des renseignements pertinents sur la session et son historique.

| Champ                               | Description                                                                                                                                                                                                                                                                |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID de session                       | L’ID de session est un identifiant persistant de la session dans le tenant Auth0.<br /><br />Notez que l’ID de session correspond à la claim `sid` déjà présente dans les ID Tokens et les Logout Tokens, et peut servir à établir des correspondances entre ces éléments. |
| Informations temporelles            | Informations sur la création de la session, l’heure d’authentification et l’expiration.                                                                                                                                                                                    |
| Informations sur l’appareil         | La propriété de l’appareil retrace les détails liés à l’agent utilisateur (par exemple, le navigateur) utilisé lors des interactions avec cette session Auth0.                                                                                                             |
| Informations sur l’authentification | Contient des renseignements sommaires sur les méthodes utilisées pour s’authentifier dans cette session.                                                                                                                                                                   |

Pour obtenir des renseignements détaillés sur ces champs, consultez la [documentation de Management API](https://auth0.com/docs/api/management/v2).

<div id="oidc-back-channel-logout-initiators">
  ## OIDC Back-Channel Logout Initiators
</div>

Les événements de suppression de session sont associés à [OIDC Back-Channel Logout](/docs/fr-CA/authenticate/login/logout/back-channel-logout) via l’initiateur `session-deleted`. Pour en savoir plus, consultez [OIDC Back-Channel Logout Initiators](/docs/fr-CA/authenticate/login/logout/back-channel-logout/oidc-back-channel-logout-initiators).

<div id="sessions-and-refresh-tokens">
  ## Sessions et jetons d’actualisation
</div>

Les sessions et les <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un jeton d’accès renouvelé sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> fonctionnent de concert pour rendre l’authentification des utilisateurs plus fluide tout en optimisant la sécurité. Pour en savoir plus, consultez [Pratiques exemplaires de gestion des sessions d’application sur le blogue Auth0](https://auth0.com/blog/application-session-management-best-practices).

Les jetons d’actualisation peuvent demeurer actifs après l’expiration ou la suppression d’une session, ou après la déconnexion de l’utilisateur. Vous pouvez utiliser la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> pour [gérer les jetons d’actualisation](/docs/fr-CA/secure/tokens/manage-refresh-tokens-with-auth0-management-api) de façon indépendante ou conjointement avec les sessions.

<div id="limitations">
  ## Limites
</div>

Les opérations de session Delete s’exécutent de manière asynchrone et finissent par devenir cohérentes.
