title: Gérer les sessions utilisateur avec l’Auth0 Management API
---

Auth0 utilise des [sessions](/docs/fr-CA/manage-users/sessions) pour maintenir l’état d’authentication d’un utilisateur d’une requête à l’autre.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les points de terminaison de gestion des sessions de l’Auth0 Management API sont offerts aux clients ayant un forfait Enterprise. Pour en savoir plus, consultez la page [Tarification](https://auth0.com/pricing).
</Callout>

<div id="management-api-endpoints">
  ## Points de terminaison de la Management API
</div>

La [Management API](https://auth0.com/docs/api/management/v2) fournit un accès out-of-band au fonctionnement interne des sessions utilisateur dans la [Auth0 Session Layer](/docs/fr-CA/manage-users/sessions/session-layers), ainsi que des méthodes de suppression permettant d’en forcer la terminaison.

<div id="session-resource">
  ### Ressource de session
</div>

Vous pouvez consulter ou supprimer une session spécifique à l’aide des points de terminaison suivants :

| Nom                                     | URL                                   | scope(s) requis   |
| --------------------------------------- | ------------------------------------- | ----------------- |
| Inspecter une session spécifique par ID | `GET /api/v2/sessions/{sessionId}`    | `read:sessions`   |
| Supprimer une session spécifique par ID | `DELETE /api/v2/sessions/{sessionId}` | `delete:sessions` |

<div id="user-resource">
  ### Ressource utilisateur
</div>

Vous pouvez afficher ou supprimer toutes les sessions d’un utilisateur donné à l’aide des points de terminaison suivants :

| Nom                                                | URL                                      | scope requis      |
| -------------------------------------------------- | ---------------------------------------- | ----------------- |
| Afficher les détails des sessions d’un utilisateur | `GET /api/v2/users/{userId}/sessions`    | `read:sessions`   |
| Supprimer toutes les sessions de l’utilisateur     | `DELETE /api/v2/users/{userId}/sessions` | `delete:sessions` |

<div id="session-properties">
  ## Propriétés de la session
</div>

Les points de terminaison de session renvoient des renseignements pertinents sur la session et son historique.

| Champ                                 | Description                                                                                                                                                                                                                                                    |
| ------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID de session                         | L’ID de session est un identifiant persistant de la session dans le tenant Auth0.<br /><br />Notez que l’ID de session correspond au claim `sid` déjà présent dans les ID Tokens et les Logout Tokens, et qu’il peut servir à faire le lien entre ces entités. |
| Dates et heures pertinentes           | Renseignements sur la création de la session, l’heure d’authentification et l’expiration.                                                                                                                                                                      |
| Renseignements sur l’appareil         | La propriété device indique des détails liés à l’agent utilisateur (par exemple, le navigateur) utilisé lors des interactions avec cette session Auth0.                                                                                                        |
| Renseignements sur l’authentification | Contient des renseignements sommaires sur les méthodes utilisées pour authentifier l’utilisateur dans cette session.                                                                                                                                           |

Pour obtenir des renseignements détaillés sur ces champs, consultez la [documentation de la Management API](https://auth0.com/docs/api/management/v2).

<div id="oidc-back-channel-logout-initiators">
  ## Initiateurs de la déconnexion OIDC par canal arrière
</div>

Les événements de suppression de session sont associés à la [déconnexion OIDC par canal arrière](/docs/fr-CA/authenticate/login/logout/back-channel-logout) par l’initiateur `session-deleted`. Pour en savoir plus, consultez [Initiateurs de la déconnexion OIDC par canal arrière](/docs/fr-CA/authenticate/login/logout/back-channel-logout/oidc-back-channel-logout-initiators).

<div id="sessions-and-refresh-tokens">
  ## Sessions et jetons d’actualisation
</div>

Les sessions et les <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se connecter de nouveau." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> fonctionnent de concert pour réduire les frictions liées à l’authentification des utilisateurs tout en optimisant la sécurité. Pour en savoir plus, consultez [les pratiques exemplaires de gestion des sessions d’application sur le blogue Auth0](https://auth0.com/blog/application-session-management-best-practices).

Les jetons d’actualisation peuvent demeurer actifs après l’expiration ou la suppression d’une session, ou après la déconnexion de l’utilisateur. Vous pouvez utiliser la <Tooltip tip="Management API : produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> pour [gérer les jetons d’actualisation](/docs/fr-CA/secure/tokens/manage-refresh-tokens-with-auth0-management-api) de façon indépendante ou de concert avec les sessions.

<div id="limitations">
  ## Limites
</div>

Les opérations de suppression de session s’exécutent de façon asynchrone et deviennent cohérentes après un court délai.
