---
description: Décrit les sessions non persistantes dans Auth0.
title: Sessions non persistantes
---

Avec les sessions persistantes, les utilisateurs peuvent fermer leur navigateur et conserver leur session lorsqu’ils le rouvrent sur le même appareil. Pour améliorer l’expérience utilisateur et offrir un accès plus sécurisé, les sessions non persistantes invalident un <Tooltip tip="Cookie de session : entité qui, lorsqu’elle est présente, permet de considérer l’utilisateur comme authentifié." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=session+cookie">cookie de session</Tooltip> lorsque le navigateur est fermé.

<Warning>
  La biométrie comme type d’authentification de premier facteur ne fonctionne pas avec les sessions non persistantes.
</Warning>

<div id="browser-limitations">
  ## Limites du navigateur
</div>

Dans certains cas, les sessions non persistantes **ne peuvent pas** être imposées par les paramètres du tenant. Par exemple :

* L’utilisateur a activé le paramètre de restauration de session dans son navigateur; la restauration de la session rétablit aussi le cookie de session.
* L’utilisateur ferme un onglet, mais pas la fenêtre du navigateur; le cookie de session n’est pas supprimé tant que la session ne prend pas fin en raison de l’expiration pour inactivité ou de l’expiration absolue.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Dans ces cas, les sessions non persistantes n’offrent **aucune** sécurité supplémentaire à la session.
</Callout>

<div id="sessions">
  ## Sessions
</div>

Les sessions se composent d’un cookie dans le navigateur et d’un enregistrement de session sur le serveur. Dans les sessions non persistantes, les cookies ne sont pas enregistrés, et un délai d’expiration du tenant est défini afin que les utilisateurs n’aient pas à se déconnecter manuellement d’un appareil.

La durée de vie de la session détermine pendant combien de temps le système conserve la session de connexion et se configure par tenant. L’expiration absolue d’une session est définie au moment de sa création. Vous pouvez activer les paramètres non persistants pour les sessions utilisateur, ou permettre à la session de se poursuivre même si le navigateur est fermé. Vous pouvez ajuster les paramètres d’expiration absolue et de non-persistance dans le Dashboard ou la <Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Pour en savoir plus, consultez [Configurer les paramètres de durée de vie des sessions](/docs/fr-CA/manage-users/sessions/configure-session-lifetime-settings).

<div id="cookies">
  ## Cookies
</div>

La plateforme Auth0 utilise trois types de cookies : de session, <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip> et d’appareil. Tous deviennent non persistants une fois votre tenant Auth0 configuré. Le délai d’expiration du tenant entraîne l’expiration de la session.

Lorsque les sessions non persistantes sont activées, les cookies suivants sont émis en tant que cookies de session lors des interactions avec le <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=authorization+server">serveur d’autorisation</Tooltip> :

* `auth0` / `auth0_compat`
* `auth0-mf` / `auth0-mf_compat`
* `did` / `did_compat`

Pour en savoir plus sur les cookies, consultez [Cookies de l’Authentication API](/docs/fr-CA/manage-users/cookies/authentication-api-cookies).

<div id="tenant-configuration">
  ## Configuration du tenant
</div>

Vous pouvez configurer votre tenant pour gérer les sessions de l’une des deux façons suivantes :

* **Persistent** : les cookies de session sont conservés lorsque le navigateur est fermé.
* **Non-Persistent** : les cookies de session ne sont pas conservés. Le navigateur conserve les cookies de session jusqu’à la fin de la session de navigation. Pour en savoir plus, consultez [HTTP State Management Mechanism on IETF Datatracker](https://datatracker.ietf.org/doc/rfc2109/).

Pour configurer ces paramètres dans le Dashboard :

1. Accédez à [Dashboard &gt; Tenant Settings](https://manage.auth0.com/#/tenant) et sélectionnez la vue **Advanced**.
2. Faites défiler jusqu’à la section **Session Expiration**, repérez **Idle Session Lifetime** et **Maximum Session Lifetime**, entrez les paramètres souhaités, puis sélectionnez **Save.**

   | Paramètres               | Description                                                                                                                                                                                                                                                                                        |
   | ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | Idle Session Lifetime    | Durée (en minutes) après laquelle la session d’un utilisateur expire s’il n’a pas interagi avec le serveur d’autorisation. Les limites du système s’appliqueront si elle dépasse 4 320 minutes (3 jours) pour Essential ou Professional, ou 144 000 minutes (100 jours) pour les forfaits Enterprise. |
   | Maximum Session Lifetime | Durée (en minutes) après laquelle un utilisateur devra se reconnecter, peu importe son activité. Les limites du système s’appliqueront si elle dépasse 43 200 minutes (30 jours) pour Developer ou Developer Pro, ou 525 600 minutes (365 jours) pour les forfaits Enterprise.                     |

<div id="management-api">
  ## Management API
</div>

Vous pouvez utiliser Management API pour activer les sessions non persistantes, ainsi que pour définir les valeurs de durée de vie des sessions, de durée d’inactivité des sessions et des propriétés des cookies de session. Pour en savoir plus, consultez le [endpoint Tenant Update de Management API](https://auth0.com/docs/api/management/v2/tenants/patch-settings).
