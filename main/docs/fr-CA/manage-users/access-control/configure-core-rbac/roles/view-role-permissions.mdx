Vous pouvez consulter les [permissions](/docs/fr-CA/manage-users/access-control/rbac) ajoutées à un rôle dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou à l’aide de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Les permissions et les rôles attribués sont utilisés avec l’ensemble de fonctionnalités Authorization Core de l’API.

<div id="prerequisites">
  ## Prérequis
</div>

Pour que le contrôle d’accès basé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API au moyen du Dashboard ou de la Management API. La fonctionnalité Authorization Core se distingue de l’Authorization Extension. Pour comparer les deux, consultez [Authorization Core vs. Authorization Extension](/docs/fr-CA/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Dashboard
</div>

1. Allez à [Dashboard &gt; User Management &gt; Roles](https://manage.auth0.com/#/roles) et cliquez sur le nom du rôle que vous voulez consulter.
2. Cliquez sur la vue **Permissions**.

Les renseignements suivants s’affichent pour chaque permission :

| Colonne     | Description                                                           |
| ----------- | --------------------------------------------------------------------- |
| Nom         | Nom de la permission tiré de la définition de la permission.          |
| Description | Description de la permission tirée de la définition de la permission. |
| API         | Nom de l’API à laquelle la permission est associée.                   |

<div id="management-api">
  ## Management API
</div>

Effectuez une requête `GET` vers le [point de terminaison Get Role Permissions](https://auth0.com/docs/api/management/v2#!/Roles/get_role_permission). Assurez-vous de remplacer les valeurs de l’espace réservé `ROLE_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre rôle et votre <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, servant à accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> de la Management API, respectivement.

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/roles/ROLE_ID/permissions' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/roles/ROLE_ID/permissions");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/roles/ROLE_ID/permissions"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

</AuthCodeGroup>

| Valeur                  | Description                                                                                                          |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------- |
| `ROLE_ID`               | L’ID du rôle dont vous voulez obtenir les permissions.                                                               |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) avec la portée `read:roles`. |
