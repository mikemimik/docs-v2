title: Voir les utilisateurs attribués à des rôles
---

Vous pouvez afficher les utilisateurs attribués à un [rôle](/docs/fr-CA/manage-users/access-control/rbac) à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Les rôles sont utilisés avec l’ensemble de fonctionnalités Authorization Core pour les API.

<div id="prerequisites">
  ## Prérequis
</div>

Pour que le contrôle d’accès basé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API au moyen du Dashboard ou de la Management API. La fonctionnalalité Authorization Core est différente de l’Authorization Extension. Pour comparer les deux, consultez [Authorization Core vs. Authorization Extension](/docs/fr-CA/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Dashboard
</div>

1. Allez à [Dashboard &gt; User Management &gt; Roles](https://manage.auth0.com/#/users), puis cliquez sur le nom du rôle à consulter.
2. Cliquez sur l’onglet **Users**.

Les renseignements suivants s’affichent pour chaque utilisateur :

| Colonne          | Description                                                        |
| ---------------- | ------------------------------------------------------------------ |
| Picture          | L’image de l’utilisateur tirée du profil d’utilisateur.            |
| Name             | Le nom de l’utilisateur tiré du profil d’utilisateur.              |
| Adresse courriel | L’adresse courriel de l’utilisateur tirée du profil d’utilisateur. |

<div id="management-api">
  ## Management API
</div>

Pour récupérer une liste d’utilisateurs attribués à un rôle, consultez [Obtenir les utilisateurs attribués à un rôle](https://auth0.com/docs/api/management/v2#!/Roles/get_role_user).
