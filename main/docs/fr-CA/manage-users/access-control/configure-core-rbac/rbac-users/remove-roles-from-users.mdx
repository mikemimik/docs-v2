Vous pouvez supprimer les [rôles](/docs/fr-CA/manage-users/access-control/rbac) attribués à un utilisateur à l’aide du <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Les rôles attribués sont utilisés avec l’ensemble de fonctionnalités Authorization Core pour les API.

<div id="prerequisites">
  ## Prérequis
</div>

Pour que le contrôle d’accès basé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API à l’aide du Dashboard ou de la Management API. La fonctionnalité Authorization Core est différente de l’Authorization Extension. Pour les comparer, consultez [Authorization Core vs. Authorization Extension](/docs/fr-CA/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Dashboard
</div>

1. Allez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users), puis cliquez sur le nom de l’utilisateur à consulter.
2. Cliquez sur l’onglet **Roles**, puis sur l’icône de corbeille à côté du rôle que vous souhaitez supprimer.

<div id="management-api">
  ## Management API
</div>

Effectuez un appel `DELETE` au [point de terminaison Delete User Roles](https://auth0.com/docs/api/management/v2#!/Users/delete_user_roles). Assurez-vous de remplacer les valeurs d’exemple `USER_ID`, `MGMT_API_ACCESS_TOKEN` et `ROLE_ID` par votre ID d’utilisateur, votre jeton d’accès de la Management API <Tooltip tip="Jeton d’accès : information d’authentification envoyée dans l’en-tête Authorization, sous la forme d’une chaîne opaque ou d’un JWT, servant à accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> et vos ID de rôle, respectivement.

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \

    --header 'content-type: application/json' \

| Valeur                  | Description                                                                                                           |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | ID de l’utilisateur à mettre à jour.                                                                                  |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) avec le scope `update:users`. |
| `ROLE_ID`               | ID du ou des rôles que vous souhaitez retirer à l’utilisateur spécifié.                                               |
