Vous pouvez supprimer les permissions directement attribuées à un utilisateur à l’aide du <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Les permissions attribuées sont utilisées avec l’ensemble de fonctionnalités Authorization Core pour les API. Pour en savoir plus, consultez [Role-Based Access Control](/docs/fr-CA/manage-users/access-control/rbac).

<div id="prerequisites">
  ## Prérequis
</div>

Pour que le contrôle d’accès fondé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API au moyen du Dashboard ou de la Management API. La fonctionnalité Authorization Core est différente de l’Authorization Extension. Pour en savoir plus, consultez [Authorization Core vs. Authorization Extension](/docs/fr-CA/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="use-the-dashboard">
  ## Utiliser le Dashboard
</div>

1. Accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users), puis cliquez sur le nom de l’utilisateur.
2. Cliquez sur la vue **Permissions**, puis sur l’icône de corbeille à côté de la permission que vous souhaitez supprimer, puis confirmez.

   <Frame>![Auth0 Dashboard : détails de l’utilisateur et liste des permissions attribuées à cet utilisateur](/docs/images/cdy7uua7fh8z/4vShfxoUcSpCDnM5FnAbsR/de85942623430e3cedc097addbaa2f9d/User_Details_-_Permissions_-_English.png)</Frame>

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

Faites une requête `DELETE` vers le [endpoint Delete User Permissions](https://auth0.com/docs/api/management/v2#!/Users/delete_permissions). Assurez-vous de remplacer les valeurs de l’espace réservé `USER_ID`, `MGMT_API_ACCESS_TOKEN`, `API_ID` et `PERMISSION_NAME` par votre ID utilisateur, le <Tooltip tip="Jeton d’accès : information d’authentification, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> de la Management API, les ID d’API et les noms de permission, respectivement.

</AuthCodeGroup>

| Valeur                  | Description                                                                                                            |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | L’ID de l’utilisateur à mettre à jour.                                                                                 |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) avec la portée `update:users`. |
| `API_ID`                | ID de la ou des API associées à la ou aux permissions que vous souhaitez supprimer pour l’utilisateur indiqué.         |
| `PERMISSION_NAME`       | Nom de la ou des permissions que vous souhaitez supprimer pour l’utilisateur indiqué.                                  |
