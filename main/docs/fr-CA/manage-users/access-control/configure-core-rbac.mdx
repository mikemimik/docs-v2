description: Découvrez comment configurer les fonctionnalités d’Authorization Core d’Auth0 pour le contrôle d’accès basé sur les rôles (RBAC) de vos API.
title: Configurer les fonctionnalités d’Authorization Core pour le contrôle d’accès basé sur les rôles
---

Les fonctionnalités de base d’Authorization d’Auth0 permettent de mettre en œuvre un contrôle d’accès basé sur les rôles (RBAC) pour vos API.

Pour que le RBAC fonctionne correctement, vous devez l’activer pour votre API à l’aide du Dashboard ou de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Pour en savoir plus, consultez [Activer le contrôle d’accès basé sur les rôles pour les API](/docs/fr-CA/get-started/apis/enable-role-based-access-control-for-apis).

La fonctionnalité Authorization Core est différente de l’Authorization Extension. Pour comparer ces deux produits, consultez [Authorization Core vs. Authorization Extension](/docs/fr-CA/manage-users/access-control/authorization-core-vs-authorization-extension).

Pour tirer le meilleur parti de cette fonctionnalité de base, vous devriez :

1. [Enregistrer une API avec Auth0](/docs/fr-CA/get-started/auth0-overview/set-up-apis)
2. [Définir des permissions pour l’API](/docs/fr-CA/get-started/apis/add-api-permissions)
3. [Créer des rôles](/docs/fr-CA/manage-users/access-control/configure-core-rbac/roles/create-roles)
4. [Attribuer des rôles aux utilisateurs](/docs/fr-CA/manage-users/access-control/configure-core-rbac/rbac-users/assign-roles-to-users)
5. [Attribuer des permissions aux utilisateurs](/docs/fr-CA/manage-users/access-control/configure-core-rbac/rbac-users/assign-permissions-to-users), au besoin.
