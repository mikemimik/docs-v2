Voici les trois méthodes prises en charge pour authentifier l’Auth0 Deploy CLI auprès de votre tenant.

<div id="client-credentials">
  ## Identifiants du client
</div>

L’[authentification avec les identifiants du client](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow) exige que vous fournissiez directement à l’application une paire composée d’un <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">ID client</Tooltip> et d’un <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">secret client</Tooltip>. Cette option est simple et permet de configurer l’outil le plus rapidement possible.

Pour configurer l’authentification avec les identifiants du client, définissez les champs suivants dans votre [fichier de configuration](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) :

| Champ                 | Valeur                                    |
| --------------------- | ----------------------------------------- |
| `AUTH0_CLIENT_ID`     | ID client de votre application Auth0.     |
| `AUTH0_CLIENT_SECRET` | secret client de votre application Auth0. |

<div id="private-key-jwt">
  ## Private key JWT
</div>

L’authentification [Private key JWT](/docs/fr-CA/get-started/applications/configure-private-key-jwt) exige que vous configuriez une clé privée pour l’application et une clé publique pour le tenant distant. Cette option est idéale si vous ne souhaitez pas stocker d’identifiants dans Auth0.

Pour configurer l’authentification par clé privée <Tooltip tip="JSON Web Token (JWT) : format standard de ID Token (et souvent de access token) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JWT">JWT</Tooltip>, définissez les champs suivants dans votre [fichier de configuration](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) :

| Champ                            | Valeur                                                              |
| -------------------------------- | ------------------------------------------------------------------- |
| `AUTH0_CLIENT_SIGNING_KEY_PATH`  | Chemin relatif vers le certificat de la clé de signature JWT.       |
| `AUTH0_CLIENT_SIGNING_ALGORITHM` | Facultatif. Algorithme de signature JWT utilisé pour le certificat. |

<div id="access-token">
  ## Jeton d’accès
</div>

L’[authentification par jeton d’accès](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) exige que vous fournissiez directement à l’application un <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> d’Auth0. Cette option confère plus de responsabilités aux développeurs, mais peut permettre des flux de travail souples et précis au besoin.

Pour configurer l’authentification par jeton d’accès, transmettez un jeton d’accès Auth0 pour la Management API au moyen de la variable d’environnement `AUTH0_ACCESS_TOKEN` [variable d’environnement](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).
