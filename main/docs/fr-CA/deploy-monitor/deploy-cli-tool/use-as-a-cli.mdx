title: Utiliser comme CLI

Le Deploy CLI peut être utilisé comme utilitaire de ligne de commande autonome. C&#39;est une façon simple de gérer la configuration de votre tenant Auth0 dans des flux de travail CI/CD.

Pour obtenir la liste complète des paramètres de configuration, veuillez consulter [Configure the Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export-command">
  ## commande export
</div>

Récupération des configurations du tenant Auth0 sur la machine locale.

<div id="output_folder-o">
  ### `--output_folder`, `-o`
</div>

Chemin. Indique le répertoire cible dans lequel les fichiers de configuration seront enregistrés.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Chemin d’accès. Indique le fichier de configuration défini par l’utilisateur (`config.json`). Pour en savoir plus, consultez [Configurer le Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="format-f">
  ### `--format`, `-f`
</div>

Options : `yaml` ou `directory`. Détermine le format des fichiers de configuration des ressources exportés. Pour en savoir plus, consultez [Formats de configuration des ressources disponibles](/docs/fr-CA/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="export_ids-e">
  ### `--export_ids`, `-e`
</div>

Booléen. Lorsqu’elle est activée, cette option exporte les champs d’identifiant de chaque ressource. Par défaut : `false`.

<div id="env">
  ### `--env`
</div>

Booléen. Indique si l’outil doit prendre en compte les variables d’environnement ou non. Par défaut : `true`.

<div id="debug">
  ### `--debug`
</div>

Booléen. Active une journalisation des erreurs plus détaillée; utile pour le dépannage. Par défaut : `false`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Une URL pour acheminer les requêtes via un proxy. À définir uniquement si vous êtes derrière un proxy.

<div id="examples">
  ### Exemples
</div>

<div id="fetching-auth0-tenant-configuration-in-the-yaml-format">
  #### Récupération de la configuration du tenant Auth0 au format YAML
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local`

<div id="fetching-auth0-tenant-configuration-in-directory-json-format">
  #### Récupération de la configuration du tenant Auth0 au format directory (JSON)
</div>

`a0deploy export -c=config.json --format=directory --output_folder=local`

<div id="fetching-auth0-tenant-configurations-with-ids-of-all-assets">
  #### Récupération de la configuration du tenant Auth0 avec les ID de toutes les ressources
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local --export_ids=true`

<div id="import-command">
  ## commande import
</div>

Application des configurations de la machine locale au tenant Auth0.

<div id="input_file-i">
  ### `--input_file`, `-i`
</div>

Chemin. Indique l’emplacement des fichiers de configuration des ressources. Pour les formats YAML, ce paramètre pointe vers le fichier `tenant.yaml`; pour les formats de directory, il pointe vers le directory de configuration des ressources.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Chemin d’accès. Indique le fichier de configuration défini par l’utilisateur (`config.json`). Pour en savoir plus, consultez [Configurer le Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="env">
  ### `--env`
</div>

Booléen. Indique si l’outil doit prendre en compte les variables d’environnement ou non. Par défaut : `true`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Une URL pour acheminer les requêtes via un proxy. N’utilisez cette option que si vous êtes derrière un proxy.

<div id="debug">
  ### `--debug`
</div>

Booléen. Active une journalisation des erreurs plus détaillée; utile pour le dépannage. Par défaut : `false`.

<div id="examples">
  ### Exemples
</div>

<div id="deploying-configuration-for-yaml-formats">
  #### Déployer la configuration au format YAML
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml`

<div id="deploying-configuration-for-directory-format">
  #### Déploiement de la configuration au format directory
</div>

`a0deploy import -c=config.json --input_file=local`

<div id="deploying-configuration-with-environment-variables-ignored">
  #### Déploiement de la configuration sans tenir compte des variables d’environnement
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml --env=false`
