En général, les fichiers de configuration des ressources du Deploy CLI correspondent étroitement aux schémas de payload de l’[Auth0 Management API](https://auth0.com/docs/api/management/v2), mais il y a quelques nuances importantes à garder à l’esprit.

<div id="client-grants">
  ## Autorisations de client
</div>

L’autorisation de client associée à Deploy CLI n’est délibérément ni exportée ni configurable directement. Cela vise à éviter des changements incompatibles; autrement, l’outil pourrait révoquer l’accès ou même planter en pleine importation. Dans un contexte multilocataire et multi-environnement, on s’attend à ce que les nouveaux tenants aient déjà un client désigné pour Deploy CLI, comme indiqué dans [Pour commencer](/docs/fr-CA/deploy-monitor/deploy-cli-tool).

<div id="prompts">
  ## Prompts
</div>

Les prompts de texte personnalisé multilingue suivent une hiérarchie précise. Sous la propriété de ressource `prompts` au niveau racine se trouve une propriété propriétaire, `customText`, qui sert à regrouper les traductions de texte personnalisé avec d’autres paramètres des prompts. Sous `customText` se trouve le code de langue à deux caractères. Vient ensuite le prompt ID, suivi du screen ID, puis du text ID.

<div id="hierarchy">
  ### Hiérarchie
</div>

```yaml lines
prompts:
  customText:
    <LANGUAGE>: # code de langue à deux caractères
      <PROMPT_ID>: # ID du prompt
        <SCREEN_ID>: # ID de l'écran du prompt
          <TEXT_ID>: 'Some text'
```

<div id="example">
  ### Exemple
</div>

```yaml lines
prompts:
  identifier_first: true
  universal_login_experience: classic
  customText:
    en:
      login:
        login:
          description: Description de connexion en anglais
          buttonText: Texte du bouton
      mfa:
        mfa-detect-browser-capabilities:
          pickAuthenticatorText: 'Essayer une autre méthode'
          reloadButtonText: 'Recharger'
          noJSErrorTitle: 'JavaScript requis'
        mfa-login-options:
          pageTitle: 'Se connecter à ${clientName}'
          authenticatorNamesSMS: 'SMS'
```

<div id="databases">
  ## Bases de données
</div>

Lors de la gestion des connexions de base de données, les valeurs de `options.customScripts` renvoient à des fichiers Javascript précis, en fonction du chemin du dossier de sortie. Par ailleurs, le payload correspond de près à celui de l’[Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Connections/post_connections).

<div id="yaml-example">
  ### Exemple en YAML
</div>

Arborescence des dossiers en mode YAML :

```text lines
./databases/
    /Username-Password-Authentication
        /change_password.js   
        /create.js   
        /delete.js   
        /get_user.js   
        /login.js   
        /verify.js   
./tenant.yaml
```

 Contenu de `tenant.yaml` :

```yaml lines
databases:
  - name: Username-Password-Authentication
    # ...
    options:
      # ...
      customScripts:
        change_password: ./databases/Username-Password-Authentication/change_password.js
        create: ./databases/Username-Password-Authentication/create.js
        delete: ./databases/Username-Password-Authentication/delete.js
        get_user: ./databases/Username-Password-Authentication/get_user.js
        login: ./databases/Username-Password-Authentication/login.js
        verify: ./databases/Username-Password-Authentication/verify.js
```

### Exemple de répertoire

Structure des dossiers en mode répertoire :

```text lines
./database-connections/
    ./Username-Password-Authentication/
        ./change_password.js
        ./create.js
        ./database.json
        ./delete.js
        ./get_user.js
        ./login.js
        ./verify.js
```

Contenu de `database.json` :

```json lines
{
  "options": {
    "customScripts": {
      "change_password": "./change_password.js",
      "create": "./create.js",
      "delete": "./delete.js",
      "get_user": "./get_user.js",
      "login": "./login.js",
      "verify": "./verify.js"
    }
  }
}
```

<div id="universal-login">
  ## Universal Login
</div>

<div id="pages">
  ### Pages
</div>

Lorsque vous remplacez le <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l'Authorization Server d'Auth0, pour vérifier l'identité d'un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> par du HTML personnalisé, les contenus d’erreur, de connexion, d’<Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l'Authorization Server d'Auth0, pour vérifier l'identité d'un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifactorielle</Tooltip> et de réinitialisation du mot de passe se trouvent sur des pages HTML distinctes.

#### Exemple YAML

Arborescence des dossiers en mode YAML :

```text lines
./pages/
    /error_page.html
    /guardian_multifactor.html
    /login.html
    /password_reset.html
./tenant.yaml
```

Contenu de `tenant.yaml` :

```yaml lines
pages:
  - name: error_page
    html: ./pages/error_page.html
    show_log_link: false
    url: https://mycompany.org/error
  - name: guardian_multifactor
    enabled: true
    html: ./pages/guardian_multifactor.html
  - name: login
    enabled: false
    html: ./pages/login.html
  - name: password_reset
    enabled: true
    html: ./pages/password_reset.html
```

<div id="directory-example">
  #### Exemple de répertoire
</div>

Arborescence des dossiers en mode répertoire :

```text lines
./pages/
    ./error_page.html
    ./error_page.json
    ./guardian_multifactor.html
    ./guardian_multifactor.json
    ./login.html
    ./login.json
    ./password_reset.html
    ./password_reset.json
```

Contenu de `login.json` :

```json lines
{
  "name": "login",
  "enabled": false,
  "html": "./login.html"
}
```

Contenu de `error_page.json` :

```json lines
{
  "html": "./error_page.html",
  "show_log_link": false,
  "url": "https://mycompany.org/error",
  "name": "error_page"
}
```

Contenu de `guardian_multifactor.json` :

```json lines
{
  "enabled": true,
  "html": "./guardian_multifactor.html",
  "name": "guardian_multifactor"
}
```

Contenu de `password_reset.json` :

```json lines
{
  "enabled": true,
  "html": "./password_reset.html",
  "name": "password_reset"
}
```
