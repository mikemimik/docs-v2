description: Découvrez les formats de configuration disponibles lorsque vous utilisez Auth0 Deploy CLI.
title: Formats de configuration des ressources disponibles
---

L’état des ressources Auth0 peut être défini dans deux formats de fichier de configuration : YAML et répertoire (JSON). Lorsque vous utilisez la commande `export` d’Auth0 Deploy CLI, vous serez invité à choisir l’un ou l’autre.

<div id="yaml">
  ## YAML
</div>

Le format YAML se présente principalement sous la forme d’un fichier plat tenant.yaml, accompagné de fichiers de code supplémentaires pour des ressources comme les Actions et les modèles de courriel. Le fichier unique facilite le suivi des modifications au fil du temps dans le contrôle de version. De plus, il élimine une part de l’ambiguïté liée aux noms de répertoires et de fichiers, qui n’est pas forcément évidente au premier abord.

<div id="directory-json">
  ## Répertoire (JSON)
</div>

Le format en répertoire répartit les types de ressources dans des répertoires distincts, chaque ressource étant stockée dans un fichier JSON dédié. Ce format facilite la séparation conceptuelle entre chaque type de ressource, ainsi qu’entre les ressources individuelles elles-mêmes. De plus, le Auth0 Deploy CLI reprend de très près les structures de données définies dans l’[Auth0 Management API](https://auth0.com/docs/api/management/v2), donc il peut être utile de consulter les exemples JSON dans la documentation lorsque vous utilisez ce format.

<div id="how-to-choose">
  ## Comment choisir
</div>

Le choix du format à utiliser devrait d’abord relever de votre préférence. Les deux formats sont des solutions viables qui permettent d’accomplir la même tâche, mais chacun présente des forces et des faiblesses légèrement différentes, décrites ci-dessus. Assurez-vous d’évaluer chacun en fonction de votre contexte. Surtout, ce choix n’est pas définitif, et vous pouvez passer de l’un à l’autre au moyen de la commande `import`.
