description: Découvrez les formats de configuration disponibles lorsque vous utilisez le Deploy CLI d’Auth0.
title: Formats de configuration des ressources disponibles
---

L’état des ressources Auth0 est présenté dans deux formats de fichiers de configuration : YAML et répertoire (JSON). Lorsque vous utilisez la commande `export` du Deploy CLI, vous serez invité à choisir l’un ou l’autre.

<div id="yaml">
  ## YAML
</div>

Le format YAML prend principalement la forme d’un fichier plat tenant.yaml, accompagné de fichiers de code supplémentaires pour des ressources comme les Actions et les modèles d’e-mails. Le fait d’avoir un seul fichier facilite le suivi des modifications au fil du temps dans le contrôle de version. De plus, cela élimine une part de l’ambiguïté entourant les noms de répertoires et de fichiers, qui n’est pas toujours évidente au premier abord.

<div id="directory-json">
  ## Répertoire (JSON)
</div>

Le format de répertoire répartit les types de ressources dans des répertoires distincts, chaque ressource étant stockée dans un fichier JSON dédié. Ce format facilite la séparation conceptuelle entre chaque type de ressource, ainsi qu’entre les ressources individuelles elles-mêmes. De plus, le Deploy CLI reflète de près les structures de données définies dans l’[Auth0 Management API](https://auth0.com/docs/api/management/v2); il peut donc être utile de consulter les exemples JSON de la documentation lorsque vous utilisez ce format.

<div id="how-to-choose">
  ## Comment choisir
</div>

Le choix du format à utiliser devrait avant tout dépendre de vos préférences. Les deux formats sont des solutions viables qui permettent d’accomplir la même tâche, mais ils présentent des forces et des faiblesses légèrement différentes, décrites ci-dessus. Assurez-vous d’évaluer chacun en fonction de votre contexte. Il est important de noter que ce choix n’est pas définitif et qu’il est possible de passer de l’un à l’autre à l’aide de la commande `import`.
