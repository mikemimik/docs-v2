description: Découvrez les formats de configuration disponibles lors de l’utilisation d’Auth0 Deploy CLI.
title: Formats de configuration des ressources disponibles
---

La configuration des ressources Auth0 peut être exprimée dans deux formats de fichier différents : YAML et répertoire (JSON). Lorsque vous utilisez la commande `export` de Deploy CLI, vous devrez choisir l’un ou l’autre.

<div id="yaml">
  ## YAML
</div>

Le format YAML se présente principalement sous la forme d’un fichier unique tenant.yaml, accompagné de fichiers de code supplémentaires pour des ressources comme les actions et les modèles de courriel. Ce fichier unique simplifie le suivi des changements au fil du temps dans le contrôle de version. De plus, il élimine une part de l’ambiguïté liée aux noms de répertoires et de fichiers, qui peut ne pas être évidente au premier abord.

<div id="directory-json">
  ## Répertoire (JSON)
</div>

Le format de répertoire répartit les types de ressources dans des répertoires distincts, chaque ressource étant enregistrée dans un fichier JSON dédié. Ce format permet de distinguer plus facilement chaque type de ressource, ainsi que les ressources individuelles elles-mêmes. De plus, le Deploy CLI reprend de près les structures de données définies dans l’[Auth0 Management API](https://auth0.com/docs/api/management/v2), donc il peut être utile de consulter les exemples JSON dans la documentation lorsque vous utilisez ce format.

<div id="how-to-choose">
  ## Comment choisir
</div>

Le choix du format à utiliser devrait avant tout reposer sur vos préférences. Les deux formats sont des solutions viables qui permettent d’accomplir la même tâche, mais ils présentent des forces et des faiblesses légèrement différentes, décrites ci-dessus. Assurez-vous d’évaluer chacun en fonction de votre propre contexte. Surtout, ce choix n’est pas définitif, et vous pouvez passer de l’un à l’autre à l’aide de la commande `import`.
