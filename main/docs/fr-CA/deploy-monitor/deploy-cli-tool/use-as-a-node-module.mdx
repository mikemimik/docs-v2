---

Le Deploy CLI peut être utilisé non seulement comme CLI autonome, mais aussi comme module Node.js. Vous pouvez ainsi gérer les ressources Auth0 dans des scripts Node.js plus souples.

Pour obtenir une liste complète des options de configuration, veuillez consulter [Configurer le Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="dump-function">
  ## fonction dump
</div>

Récupère les configurations du tenant Auth0 sur la machine locale.

### Paramètres

<div id="format">
  #### format
</div>

Options : `yaml` ou `directory`. Détermine le format des fichiers de configuration des ressources exportés. Pour en savoir plus, consultez [Formats de configuration des ressources disponibles](/docs/fr-CA/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="output_folder">
  #### output_folder
</div>

Chemin. Spécifie le répertoire cible dans lequel les fichiers de configuration seront écrits.

<div id="config">
  #### config
</div>

Objet. Configure le comportement de l’utilitaire. Pour en savoir plus, consultez [Configurer le Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="config_file">
  #### config_file
</div>

Chemin. Spécifie le fichier de configuration personnalisé (`config.json`). Pour en savoir plus, consultez [Configurer le Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export_ids">
  #### export_ids
</div>

Booléen. Si cette option est activée, les champs d’identifiant de chaque ressource seront exportés. Par défaut : `false`.

<div id="env">
  #### env
</div>

Booléen. Indique si l’outil doit charger les variables d’environnement. Par défaut : `false`.

<div id="proxy_url">
  #### proxy_url
</div>

Une URL pour faire passer les requêtes par un serveur proxy. À définir uniquement si vous êtes derrière un proxy.

<div id="example">
  ### Exemple
</div>

```jsx lines
import { dump } from 'auth0-deploy-cli';

dump({
  output_folder: './local',
  format: 'yaml',
  config: {
    AUTH0_DOMAIN: '<YOUR_AUTH0_TENANT_DOMAIN>',
    AUTH0_CLIENT_ID: '<YOUR_AUTH0_CLIENT_ID>',
    AUTH0_CLIENT_SECRET: '<YOUR_AUTH0_CLIENT_SECRET>',
  },
})
  .then(() => {
    console.log('Exportation de la configuration Auth0 réussie');
  })
  .catch((err) => {
    console.log('Erreur lors de l\'exportation de la configuration Auth0 :', err);
  });
```

<div id="deploy-function">
  ## fonction de déploiement
</div>

Applique les configurations de la machine locale au tenant Auth0.

<div id="argument-parameters">
  ### Paramètres des arguments
</div>

<div id="input_file">
  #### input_file
</div>

Chemin. Indique l’emplacement des fichiers de configuration des ressources. Pour les formats YAML, il pointera vers le fichier `tenant.yaml`. Pour les formats de répertoire, il pointera vers le répertoire de configuration des ressources.

<div id="config">
  #### config
</div>

Objet. Définit le comportement de l’utilitaire. Pour en savoir plus, consultez [Configurer le Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="config_file">
  #### config_file
</div>

Chemin. Indique le fichier de configuration personnalisé (`config.json`). Pour en savoir plus, consultez [Configurer le Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export_ids">
  #### export_ids
</div>

Booléen. Si cette option est activée, les champs d’identifiant de chaque ressource seront exportés. Par défaut : `false`.

<div id="env">
  #### env
</div>

Booléen. Indique si l’outil doit charger les variables d’environnement. Par défaut : `false`.

<div id="proxy_url">
  #### proxy_url
</div>

Une URL servant à acheminer les requêtes par un serveur proxy. Définissez-la uniquement si vous passez par un proxy.

<div id="example">
  ### Exemple
</div>

```jsx lines
import { deploy } from 'auth0-deploy-cli';

deploy({
  input_file: './local/tenant.yaml',
  config: {
    AUTH0_DOMAIN: '<YOUR_AUTH0_TENANT_DOMAIN>',
    AUTH0_CLIENT_ID: '<YOUR_AUTH0_CLIENT_ID>',
    AUTH0_CLIENT_SECRET: '<YOUR_AUTH0_CLIENT_SECRET>',
  },
})
  .then(() => {
    console.log('Configuration Auth0 appliquée au locataire avec succès');
  })
  .catch((err) => {
    console.log('Erreur lors de l\'application de la configuration au tenant Auth0 :', err);
  });
```
