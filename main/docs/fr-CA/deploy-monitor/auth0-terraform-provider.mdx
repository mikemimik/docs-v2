description: Découvrez comment utiliser Auth0 Terraform Provider pour représenter les configurations de votre tenant sous forme de code.
title: Auth0 Terraform Provider
---

Le Deploy CLI n’est pas le seul outil offert pour gérer la configuration de votre tenant Auth0; il existe aussi un [Terraform Provider officiellement pris en charge](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) est un outil tiers qui permet de représenter la configuration de vos ressources infonuagiques sous forme de code. Il repose sur un framework d’extensions bien établi qui prend en charge un large éventail de fournisseurs infonuagiques, y compris Auth0.

Auth0 Terraform Provider est aussi offert pour [OpenTofu](https://opentofu.org/), l’outil open source compatible avec Terraform. Vous trouverez le provider Auth0 dans le [registre OpenTofu](https://search.opentofu.org/provider/auth0/auth0/latest).

Le Deploy CLI et le Terraform Provider sont tous deux là pour vous aider à gérer les configurations de votre tenant Auth0, mais chacun a ses avantages et ses inconvénients.

Vous pourriez envisager d’utiliser Auth0 Terraform Provider si :

* Vos processus de développement utilisent déjà Terraform ou OpenTofu
* Vos besoins de gestion du tenant sont très précis ou ne concernent que quelques ressources en particulier

Vous pourriez préférer ne pas utiliser Auth0 Terraform Provider si :

* Votre processus de développement n’utilise pas Terraform ou OpenTofu, ce qui demande une configuration initiale supplémentaire
* Vos processus de développement visent surtout à gérer vos tenants en bloc
* Votre tenant comporte déjà beaucoup de ressources, ce qui peut demander des efforts importants pour les « importer »
