title: Auth0 Terraform Provider
---

Le Deploy CLI n’est pas le seul outil offert pour gérer la configuration de votre tenant Auth0 : il existe aussi un [Terraform Provider officiellement pris en charge](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) est un outil tiers qui permet de représenter la configuration de vos ressources infonuagiques sous forme de code. Il s’appuie sur un framework de plugiciels bien établi qui prend en charge un large éventail de fournisseurs infonuagiques, dont Auth0.

L’Auth0 Terraform Provider est aussi offert pour [OpenTofu](https://opentofu.org/), l’outil libre compatible avec Terraform. Vous trouverez le fournisseur Auth0 dans le [registre OpenTofu](https://search.opentofu.org/provider/auth0/auth0/latest).

Le Deploy CLI et le Terraform Provider servent tous deux à vous aider à gérer les configurations de votre tenant Auth0, mais chacun a ses propres avantages et inconvénients.

Vous pourriez envisager l’Auth0 Terraform Provider si :

* Vos workflows de développement utilisent déjà Terraform ou OpenTofu
* Vos besoins en gestion de tenant sont granulaires ou ne concernent que quelques ressources précises

Vous pourriez ne pas envisager l’Auth0 Terraform Provider si :

* Votre workflow de développement n’utilise ni Terraform ni OpenTofu, ce qui demande une configuration initiale supplémentaire
* Vos workflows de développement visent surtout à gérer vos tenants en bloc
* Votre tenant comporte déjà beaucoup de ressources, ce qui peut demander des efforts importants pour les « importer »
