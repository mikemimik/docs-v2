title: Surveiller Auth0 à l’aide de System Center Operations Manager
---

Vous pouvez surveiller Auth0 en tant qu’application Web standard à l’aide de System Center Operations Manager (SCOM) ou de tout outil prenant en charge les transactions synthétiques. Nous vous recommandons d’ajouter une transaction de connexion synthétique qui inclut les extensions dont dépendent vos applications (comme les rules qui exécutent du code personnalisé pour l’intégration à d’autres services de votre entreprise).

<div id="set-up-scom">
  ## Configurer SCOM
</div>

1. Ajoutez une nouvelle instance SCOM à l’aide de l’**Assistant d’ajout de Monitoring** :

| Champ                                                | Description                                          |
| ---------------------------------------------------- | ---------------------------------------------------- |
| Nom                                                  | Nom descriptif de cette instance SCOM.               |
| Description                                          | Description de ce que surveille cette instance SCOM. |
| Sélectionner le pack d’administration de destination | Pack d’administration par défaut                     |

Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

2. Cliquez sur **Ajouter** pour saisir les URL que SCOM doit surveiller. Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

3. Cliquez sur **Ajouter** pour configurer un emplacement à partir duquel effectuer la surveillance.

4. Dans la boîte de dialogue, recherchez **Emplacement interne - Agent**. Sélectionnez l’adresse appropriée, puis cliquez sur **Ajouter**. Cliquez ensuite sur **OK** pour terminer la sélection de l’emplacement. Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

5. Définissez la fréquence à laquelle SCOM collecte les données de chaque point de terminaison :

| Données                                           | Fréquence               |
| ------------------------------------------------- | ----------------------- |
| Fréquence de Test                                 | 60 secondes             |
| Intervalle de collecte des données de performance | 60 secondes             |
| Délai d’expiration du Test                        | 30 secondes             |
| Code d’état HTTP                                  | Supérieur ou égal à 400 |

6. Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

<div id="run-scom-tests">
  ## Exécuter les tests SCOM
</div>

1. Cliquez sur **Lancer le test** pour tester chaque point de terminaison et vous assurer que les paramètres de connexion fournis sont corrects.
2. Une fois la configuration de votre instance SCOM terminée, vous pouvez consulter l’activité dans l’onglet **Monitoring**.

<div id="review-test-results">
  ## Consulter les résultats du test
</div>

Cliquez sur **Statut de l’application Web** pour afficher les renseignements recueillis par SCOM.
