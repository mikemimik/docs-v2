title: Monitor Auth0 Using System Center Operations Manager
---

Vous pouvez surveiller Auth0 comme une application Web standard à l’aide de System Center Operations Manager (SCOM) ou de tout outil prenant en charge les transactions synthétiques. Nous recommandons d’ajouter une transaction de connexion synthétique qui inclut les extensions dont dépendent vos applications (comme les Rules qui exécutent du code personnalisé pour l’intégration aux autres services de votre entreprise).

<div id="set-up-scom">
  ## Configurer SCOM
</div>

1. Ajoutez une nouvelle instance SCOM à l’aide de l’**Add Monitoring Wizard** :

| Champ                              | Description                                          |
| ---------------------------------- | ---------------------------------------------------- |
| Name                               | Nom descriptif de l’instance SCOM.                   |
| Description                        | Description de ce que surveille cette instance SCOM. |
| Select destination management pack | Pack Management par défaut                           |

Une fois terminé, cliquez sur **Next** pour continuer.

2. Cliquez sur **Add** pour entrer les URL que vous voulez que SCOM surveille. Une fois terminé, cliquez sur **Next** pour continuer.

3. Cliquez sur **Add** pour configurer l’emplacement à partir duquel vous voulez effectuer la surveillance.

4. Dans la fenêtre contextuelle, recherchez **Internal location - Agent**. Sélectionnez l’adresse appropriée, puis cliquez sur **Add**. Cliquez ensuite sur **Ok** pour terminer la sélection de l’emplacement. Une fois terminé, cliquez sur **Next** pour continuer.

5. Définissez la fréquence à laquelle SCOM recueille des données à partir de chaque point de terminaison :

| Données                                           | Fréquence               |
| ------------------------------------------------- | ----------------------- |
| Fréquence des tests                               | 60 secondes             |
| Intervalle de collecte des données de performance | 60 secondes             |
| Délai d’expiration du test                        | 30 secondes             |
| Code d’état HTTP                                  | Supérieur ou égal à 400 |

6. Une fois terminé, cliquez sur **Next** pour continuer.

<div id="run-scom-tests">
  ## Exécuter les tests SCOM
</div>

1. Cliquez sur **Run Test** pour tester chaque point de terminaison et vous assurer que les paramètres de connexion fournis sont corrects.
2. Une fois la configuration de votre instance SCOM terminée, vous pouvez consulter l’activité dans l’onglet **Monitoring**.

<div id="review-test-results">
  ## Consulter les résultats du test
</div>

Cliquez sur **État de l’application Web** pour afficher les informations recueillies par SCOM.
