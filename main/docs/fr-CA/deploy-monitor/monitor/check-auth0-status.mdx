Accédez à la page [Auth0 Public Cloud Status](https://status.auth0.com) pour vérifier la disponibilité du service de la version infonuagique publique d’Auth0.

Sélectionnez une **Région** dans le menu déroulant pour voir l’état des services de base (comme l’authentification utilisateur et l’authentification machine à machine) et des services de soutien (comme <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip>, <Tooltip tip="Management API : produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, les journaux du tenant et le CDN).

Pour en savoir plus sur les régions offertes, consultez la table [Region, locality, and sub-locality](/docs/fr-CA/get-started/auth0-overview/create-tenants#region-locality-and-sub-locality) dans [Créer des tenants](/docs/fr-CA/get-started/auth0-overview/create-tenants).

<div id="subscribe-to-public-cloud-status-updates">
  ## S’abonner aux mises à jour sur l’état de Public Cloud
</div>

Sélectionnez une **région**, puis **Subscribe** pour recevoir des mises à jour. Vous pouvez choisir l’une des deux options suivantes pour consulter l’état :

1. Suivez [@auth0status](https://x.com/auth0status) sur X pour obtenir les plus récentes mises à jour sur l’état du service.
2. Abonnez-vous au flux Atom ou RSS d’Auth0 pour recevoir les mises à jour d’état qui touchent votre tenant. Remplacez `{YOUR_TENANT_DOMAIN}` par le **Domain** de votre tenant.

Pour trouver le **Domain** de votre tenant, accédez à [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/dashboard/#/applications), sélectionnez une application, puis accédez à **Settings**. Le **Domain** de votre tenant figure sous **Basic Information**.

<div id="check-auth0-status-for-private-cloud">
  ## Vérifier l’état d’Auth0 pour Private Cloud
</div>

Accédez à [Auth0 Public Cloud Status](https://status.auth0.com), puis sélectionnez **Private Cloud Login**.

Une fois authentifié, vous pouvez consulter l’état de tout environnement hors développement (par exemple, `travel0-prod-1`) s’il est disponible sur la plateforme convergée d’Auth0.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour accéder à l’état de votre environnement Private Cloud, vous devez être membre du tenant de soutien Public Cloud associé. Pour en savoir plus, consultez [Open and Manage Support Tickets](/docs/fr-CA/troubleshoot/customer-support/open-and-manage-support-tickets) et [Manage Dashboard Access](/docs/fr-CA/get-started/manage-dashboard-access).
</Callout>

Pour revenir à la page Auth0 Public Cloud Status, sélectionnez **Auth0 Public Cloud Status** dans la navigation en haut à droite.

<div id="subscribe-to-private-cloud-status-updates">
  ## S’abonner aux mises à jour de l’état de Private Cloud
</div>

Sélectionnez un environnement, puis sélectionnez **Subscribe**.

Abonnez-vous au flux Atom ou RSS d’Auth0 en copiant l’URL générée dans le champ « Subscribe » afin de recevoir les mises à jour de l’état qui touchent votre environnement.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’URL générée du flux ATOM / RSS est unique à chaque environnement Private Cloud et contient des jetons qui peuvent faire l’objet d’une rotation en [communiquant avec le soutien](https://support.auth0.com/).
</Callout>
