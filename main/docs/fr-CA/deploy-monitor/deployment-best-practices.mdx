<div id="use-automated-deployment-for-actions">
  ## Utiliser le déploiement automatisé pour les Actions
</div>

Développer une Action dans l’éditeur du <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> est un excellent moyen de l’implémenter et de la tester pendant la phase de développement. Cependant, lorsqu’il faut la déployer dans des environnements de test automatisés ou de production, un mécanisme plus automatisé s’impose : le copier-coller de code entre des tenants Auth0 n’est pas une méthode satisfaisante.

<div id="set-up-version-control-for-actions">
  ### Configurer le contrôle de version pour Actions
</div>

Les modifications apportées à une Action dans l’Auth0 Dashboard sont enregistrées à l’état de brouillon jusqu’à leur déploiement. Après le déploiement, une version est créée, mais vous devez associer l’Action à un trigger pour qu’elle s’active. Nous recommandons d’utiliser le contrôle de version pour assurer la gestion des changements et la révision du code. [Actions prend nativement en charge le versionnage](docs/customize/actions/manage-versions) dans le Dashboard.

Auth0 offre des méthodes de déploiement automatisé pour les Actions entre des environnements de tenant Auth0. Les intégrations Auth0 suivantes vous permettent de mettre à jour les Actions à partir du système de contrôle de version, manuellement ou automatiquement (par exemple, lorsqu’un changement est détecté dans le système de contrôle de version) : [GitHub Actions](https://marketplace.auth0.com/integrations/github-actions), [GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline), [Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline) et [Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline).

<div id="use-deploy-cli-tool-for-action-import-and-export">
  ### Utiliser l’outil Deploy CLI pour l’importation et l’exportation d’Actions
</div>

L’outil Auth0 Deploy CLI automatise le déploiement des Actions entre les tenants Auth0. Il fonctionne avec des fichiers YAML locaux ou une structure de répertoires, conjointement avec l’Auth0 <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, et permet d’exporter des Actions depuis un tenant Auth0, puis de les importer dans un tenant Auth0. De plus, l’outil offre un contrôle programmatique sur l’ordre et la configuration d’environnement d’une Action dans le cadre de l’automatisation du déploiement. Pour en savoir plus, consultez [Deploy CLI Tool](/docs/fr-CA/deploy-monitor/deploy-cli-tool).

Comme pratique exemplaire, utilisez Auth0 Deploy CLI dans presque tous les cas de déploiement vers des environnements de test ou de production. Bien que les intégrations Marketplace offrent une détection automatisée des changements, Deploy CLI permet de contrôler précisément ce qui est déployé, quand et comment.

<div id="set-up-test-tenants">
  ## Configurer des tenants de test
</div>

Nous vous recommandons de configurer un tenant de test distinct dans Auth0 pour tester en toute sécurité les Actions avant de les déployer en production. Pour en savoir plus, consultez [Configurer plusieurs environnements](/docs/fr-CA/get-started/auth0-overview/create-tenants/set-up-multiple-environments).
