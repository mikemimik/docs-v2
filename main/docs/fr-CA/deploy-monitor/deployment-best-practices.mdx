<div id="use-automated-deployment-for-actions">
  ## Utiliser le déploiement automatisé pour Actions
</div>

Coder une Action dans l’éditeur du <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> est une excellente façon de développer et de tester pendant la phase de développement. Cependant, au moment de déployer dans des environnements de test automatisé ou de production, il faut un mécanisme plus automatisé; copier-coller du code d’un tenant Auth0 à l’autre n’est pas une méthode satisfaisante.

<div id="set-up-version-control-for-actions">
  ### Configurer le contrôle de version pour Actions
</div>

Les modifications apportées à une Action dans l’Auth0 Dashboard sont enregistrées à l’état de brouillon jusqu’à leur déploiement. Après le déploiement, une version est créée, mais vous devez associer l’Action à un déclencheur pour l’activer. Nous recommandons d’utiliser un contrôle de version pour assurer la gestion des changements et la révision du code. [Actions prend en charge la gestion des versions](docs/customize/actions/manage-versions) nativement dans le Dashboard.

Auth0 offre des méthodes de déploiement automatisé pour Actions entre des environnements de tenant Auth0. Les intégrations Auth0 suivantes vous permettent de mettre à jour des Actions à partir du système de contrôle de version, autant manuellement qu’automatiquement (par exemple, lorsqu’un changement dans le système de contrôle de version est détecté) : [GitHub Actions](https://marketplace.auth0.com/integrations/github-actions), [GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline), [Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline) et [Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline).

<div id="use-deploy-cli-tool-for-action-import-and-export">
  ### Utiliser l’outil Deploy CLI pour importer et exporter des Actions
</div>

L’outil Auth0 Deploy CLI automatise le déploiement des Actions entre des tenants Auth0. Il fonctionne avec des fichiers YAML locaux ou des fichiers basés sur des répertoires, en combinaison avec l’Auth0 <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, et permet d’exporter des Actions à partir d’un tenant Auth0, puis de les importer dans un tenant Auth0. De plus, l’outil offre un contrôle programmatique sur l’ordre et la configuration d’environnement d’une Action dans le cadre de l’automatisation du déploiement. Pour en savoir plus, consultez [Deploy CLI Tool](/docs/fr-CA/deploy-monitor/deploy-cli-tool).

Bonne pratique : utilisez Auth0 Deploy CLI dans presque tous les cas de déploiement vers des environnements de test ou de production. Bien que les intégrations Marketplace offrent une détection automatisée des changements, Deploy CLI permet de contrôler précisément ce qui est déployé, à quel moment et de quelle façon.

<div id="set-up-test-tenants">
  ## Configurer des tenants de test
</div>

Nous vous recommandons de configurer un tenant de test distinct dans Auth0 afin de tester les Actions en toute sécurité avant de les déployer en Production. Pour en savoir plus, consultez [Configurer plusieurs environnements](/docs/fr-CA/get-started/auth0-overview/create-tenants/set-up-multiple-environments).
