---
description: Décrit les attributs et les valeurs provenant des prompts utilisateur dans les journaux du tenant Auth0.
title: Détails des prompts dans les journaux du tenant
---

Les journaux du tenant Auth0 fournissent les détails des transactions liées aux actions d’authentification des utilisateurs. Les journaux du tenant consignent des renseignements sur les processus de connexion, de login et de vérification à des fins d’examen. Vous pouvez voir comment l’utilisateur s’est authentifié, quelles informations d’identification il a fournies, la durée de la transaction et l’état de la connection.

Vous trouverez ci-dessous les valeurs par défaut consignées dans les journaux du tenant :

| Attribut      | Description                                                                |
| ------------- | -------------------------------------------------------------------------- |
| `completedAt` | Heure de fin de l’étape, y compris le temps d’interaction de l’utilisateur |
| `elapsedTime` | Différence entre `completedAt` et `initiatedAt`                            |
| `flow`        | Varie selon l’expérience de login (c.-à-d. Universal ou Classic)           |
| `initiatedAt` | Heure de début de l’étape                                                  |
| `name`        | Nom de l’étape                                                             |

<div id="login">
  ## login
</div>

L’étape de login fournit les valeurs suivantes :

| Attribut    | Description                                                                    |
| ----------- | ------------------------------------------------------------------------------ |
| `flow`      | Peut prendre deux valeurs, selon qu’il s’agit de Universal ou de Classic Login |
| `user_id`   | ID Auth0 de l’utilisateur authentifié à cette étape                            |
| `user_name` | Nom de l’utilisateur authentifié à cette étape                                 |

Champs supplémentaires contenant les renseignements suivants :

| Attribut        | Description                                                         |
| --------------- | ------------------------------------------------------------------- |
| `connection`    | Connection utilisée pour authentifier l’utilisateur                 |
| `connection_id` | ID de la connection utilisée pour authentifier l’utilisateur        |
| `strategy`      | Stratégie de la connection utilisée pour authentifier l’utilisateur |
| `identity`      | Identité authentifiée de l’utilisateur                              |

Valeurs des événements supplémentaires dans le flux de login :

| Nom                            | Expérience Universal Login | Description                                                                                     |
| ------------------------------ | -------------------------- | ----------------------------------------------------------------------------------------------- |
| `federated-authenticate`       | Universal                  | Connection fédérée utilisée pour l’authentification, sociale, juridique ou Enterprise           |
| `lock-password-authenticate`   | Classic                    | Utilisateur authentifié avec un nom d’utilisateur et un mot de passe                            |
| `oauth1-authenticate`          | Classic                    | Utilisateur authentifié au moyen d’une fédération oauth1                                        |
| `oauth2-authenticate`          | Classic                    | Utilisateur authentifié au moyen d’une fédération oauth2                                        |
| `oidc-authenticate`            | Classic                    | Utilisateur authentifié au moyen d’une fédération oidc                                          |
| `prompt-authenticate`          | Universal                  | Utilisateur authentifié avec un nom d’utilisateur et un mot de passe                            |
| `prompt-authenticate-password` | Universal                  | Utilisateur authentifié avec un nom d’utilisateur et un mot de passe dans le flow `idfirst`     |
| `prompt-signup`                | Universal                  | Utilisateur inscrit à l’aide d’un nom d’utilisateur et d’un mot de passe                        |
| `prompt-signup-password`       | Universal                  | Utilisateur inscrit à l’aide d’un nom d’utilisateur et d’un mot de passe dans le flow `idfirst` |
| `saml-authenticate`            | Classic                    | Utilisateur authentifié au moyen d’une fédération SAML                                          |
| `wsfed-authenticate`           | Classic                    | Utilisateur authentifié au moyen d’une fédération WS-FED                                        |

<div id="consent">
  ## consentement
</div>

L’étape de consentement fournit les valeurs suivantes :

| Attribut    | Description                                                                                               |
| ----------- | --------------------------------------------------------------------------------------------------------- |
| `grantInfo` | Le `{"{myApiIdentifier}"}` ou l’identifiant de l’API pour laquelle l’utilisateur a accordé une permission |
| `audience`  | Audience du token                                                                                         |
| `id`        | Grant de l’utilisateur enregistré dans la base de données                                                 |
| `scope`     | Scope approuvé par l’utilisateur                                                                          |

<div id="login-email-verification">
  ## login-email-verification
</div>

Le prompt de vérification du courriel fournit les valeurs suivantes :

| Attribut                    | Description                                                     |
| --------------------------- | --------------------------------------------------------------- |
| ` login-email-verification` | Est inclus si le prompt de vérification du courriel a été affiché |

<div id="redirect">
  ## redirection
</div>

L’étape de redirection fournit les valeurs suivantes :

| Attribut | Description                                    |
| -------- | ---------------------------------------------- |
| `URL`    | URL vers laquelle l’utilisateur a été redirigé |

<div id="mfa">
  ## mfa
</div>

<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification d’un utilisateur qui utilise un facteur supplémentaire au nom d’utilisateur et au mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-factor+authentication">Authentification multifacteur</Tooltip> fournit les valeurs suivantes :

| Attribut        | Description                                                                                                                      |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `flow`          | Peut prendre deux valeurs selon qu’il est utilisé avec l’expérience Classic (`mfa`) ou l’expérience Universal (`universal-mfa`). |
| `performed_acr` | Valeur `acr` qui sera incluse dans le `id_token`.                                                                                |
| `performed_amr` | Valeur `amr` qui sera incluse dans le `id_token`.                                                                                |
| `provider`      | Provider utilisé pour effectuer l’authentification multifacteur.                                                                 |
