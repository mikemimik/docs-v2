title: Renseignements personnels identifiables dans les journaux Auth0
---

Bon nombre des journaux du tenant Auth0 peuvent contenir des renseignements personnels identifiables (PII). Les événements d’authentification génèrent des entrées de journal qui peuvent inclure des PII d’utilisateurs. Et si vous utilisez l’Auth0 <Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> ou le Dashboard pour ajouter des utilisateurs, Auth0 consigne les détails du compte d’utilisateur.

Ces champs dans les entrées de journal peuvent contenir des PII :

* Nom
* Numéro de téléphone (utilisé pour l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip>)
* Adresse courriel
* Adresse IP (qui peut révéler l’emplacement)
* Tout renseignement utilisateur personnalisé que vous définissez

Auth0 ne consigne jamais de <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+tokens">jetons d’accès</Tooltip> provenant d’Auth0 ou de tout <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip>. Lorsqu’un échange de code d’autorisation a lieu, les journaux n’affichent qu’un code partiel (par exemple : `code: 31XXXXX`).
