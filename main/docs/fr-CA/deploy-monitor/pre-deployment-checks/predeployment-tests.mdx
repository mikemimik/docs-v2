title: Exécuter des tests avant le déploiement
---

Vous devriez exécuter des tests unitaires et d’intégration avant de déployer Auth0 dans une application ou un service en production. Effectuer des tests sur les API d’Auth0 peut faire en sorte que votre compte soit [soumis à une limite de débit](/docs/fr-CA/troubleshoot/customer-support/operational-policies/rate-limit-policy); nous vous recommandons donc de créer des API d’Auth0 simulées pendant les tests. Selon votre environnement de développement, vos outils de test peuvent aussi offrir des fonctionnalités de simulation d’API. Il existe également de nombreux outils de simulation d’API, comme [MockServer](https://www.mock-server.com/) ou [JSON Server](https://github.com/typicode/json-server), que vous pouvez utiliser pour créer rapidement de fausses API à des fins de test.

Si vous avez un abonnement Enterprise, vous pouvez demander des [tests de charge](/docs/fr-CA/troubleshoot/customer-support/operational-policies/load-testing-policy) sur Auth0.

<div id="performance-tests">
  ## Tests de performance
</div>

Si vous rencontrez des problèmes pendant les tests de performance, utilisez les méthodes suivantes pour les résoudre.

<div id="auth0-dashboard">
  ### Auth0 Dashboard
</div>

La section [Monitoring &gt; Logs](https://manage.auth0.com/#/logs) du Dashboard contient des données sur :

* les actions effectuées dans le Dashboard par les administrateurs
* les authentifications effectuées par vos utilisateurs

Il existe aussi des [extensions](/docs/fr-CA/customize/extensions) que vous pouvez utiliser à des fins de journalisation, notamment pour exporter des logs vers des outils tiers et recueillir de l’information sur l’utilisation de code personnalisé dans votre compte.

<div id="third-party-test-tools">
  ### Outils de test tiers
</div>

Il existe plusieurs outils de test tiers que vous pouvez utiliser pour effectuer des tests de performance sur des API RESTful. Voici quelques options à envisager (notez qu’Auth0 n’appuie aucun produit ni outil en particulier) :

* [Apache JMeter](https://jmeter.apache.org/)
* [Artillery](https://www.artillery.io/)
* [Micro Focus LoadRunner](https://www.radview.com/)
* [Loader](https://loader.io/)
* [RadView Webload](https://www.radview.com/)
* [SmartBear LoadUI](https://smartbear.com/)
* [Vegeta](https://github.com/tsenart/vegeta)
* [Wrk](https://github.com/wg/wrk)

Ces outils génèrent des journaux d’activité qui vous aident à repérer tout élément préoccupant. Si vous avez besoin d’aide pour interpréter votre journal ou cerner le problème potentiel, veuillez contacter le soutien.

<div id="http-archive-files">
  ### Fichiers d’archive HTTP
</div>

Si vous constatez un problème reproductible, nous vous recommandons de [créer un fichier d’archive HTTP (HAR)](https://support.auth0.com/center/s/article/generate-and-analyze-har-files-to-troubleshoot-auth0-issues) et de l’envoyer à notre équipe de soutien pour obtenir du soutien supplémentaire.
