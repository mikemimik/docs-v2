title: Exécuter des tests avant le déploiement
---

Vous devriez exécuter des tests unitaires et d’intégration avant de déployer Auth0 dans une application ou un service en production. Effectuer des tests sur les API Auth0 peut faire en sorte que votre compte soit [limité en nombre de requêtes](/docs/fr-CA/troubleshoot/customer-support/operational-policies/rate-limit-policy); nous vous recommandons donc de créer des API Auth0 simulées pendant les tests. Selon votre environnement de développement, vos outils de test peuvent aussi offrir des fonctionnalités de simulation d’API. Il existe également de nombreux outils de simulation d’API, comme [MockServer](https://www.mock-server.com/) ou [JSON Server](https://github.com/typicode/json-server), que vous pouvez utiliser pour créer rapidement de fausses API à des fins de test.

Si vous avez un abonnement Enterprise, vous pouvez demander des [tests de charge](/docs/fr-CA/troubleshoot/customer-support/operational-policies/load-testing-policy) sur Auth0.

<div id="performance-tests">
  ## Tests de performance
</div>

Si vous rencontrez des problèmes lors des tests de performance, utilisez les méthodes suivantes pour effectuer le dépannage.

<div id="auth0-dashboard">
  ### Auth0 Dashboard
</div>

La section [Monitoring &gt; Logs](https://manage.auth0.com/#/logs) du Dashboard contient des données sur :

* les actions effectuées dans le Dashboard par les administrateurs
* les authentifications de vos utilisateurs

Il existe aussi des [extensions](/docs/fr-CA/customize/extensions) que vous pouvez utiliser à des fins de journalisation, notamment pour exporter des journaux vers des outils tiers et recueillir des renseignements sur l’utilisation de code personnalisé dans votre compte.

<div id="third-party-test-tools">
  ### Outils de test tiers
</div>

Il existe un certain nombre d’outils de test tiers que vous pouvez utiliser pour effectuer des tests de performance sur des API RESTful. Voici quelques options à considérer (notez qu’Auth0 n’approuve aucun produit ou outil en particulier) :

* [Apache JMeter](https://jmeter.apache.org/)
* [Artillery](https://www.artillery.io/)
* [Micro Focus LoadRunner](https://www.radview.com/)
* [Loader](https://loader.io/)
* [RadView Webload](https://www.radview.com/)
* [SmartBear LoadUI](https://smartbear.com/)
* [Vegeta](https://github.com/tsenart/vegeta)
* [Wrk](https://github.com/wg/wrk)

Ces outils fournissent des journaux d’activité qui vous aident à repérer tout ce qui peut poser problème. Si vous avez besoin d’aide pour interpréter vos journaux ou cerner le problème potentiel, veuillez contacter le soutien.

<div id="http-archive-files">
  ### Fichiers d’archive HTTP
</div>

Si vous constatez un problème que vous pouvez reproduire, nous vous recommandons de [créer un fichier d’archive HTTP (HAR)](https://support.auth0.com/center/s/article/generate-and-analyze-har-files-to-troubleshoot-auth0-issues) et de l’envoyer à notre équipe de soutien pour obtenir une aide supplémentaire.
