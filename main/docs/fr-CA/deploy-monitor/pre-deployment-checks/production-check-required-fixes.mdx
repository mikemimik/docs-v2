---
description: Correctifs à apporter à votre application Auth0 avant le déploiement en production
title: 'Vérifications de préparation à la production : correctifs critiques'
---

Les vérifications suivantes permettent de déterminer si vous avez satisfait à toutes les exigences pour réussir un déploiement en Production.

Pour en savoir plus sur les résultats de vos vérifications, consultez [Exécuter les vérifications de production](/docs/fr-CA/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Vérification                                                                                                                                                                                   | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Toutes les Actions utilisent une [version recommandée de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)                                          | Configurez toutes les [Actions](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) pour qu’elles utilisent une version recommandée de NodeJS afin de profiter des correctifs de sécurité et des nouvelles fonctionnalités.                                                                                                                                                                                                                                                                              |
| [Autoriser les ID Tokens pour l’authentification à l’API Management v2](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) est désactivé          | Les capacités permettant d’utiliser des jetons d’identité pour autoriser certains points de terminaison Users et Device Credentials de la Management API sont en cours de dépréciation. Après avoir migré vers les jetons d’accès, assurez-vous que la bascule [Allow ID Tokens for Management API v2 Authentication](https://manage.auth0.com/#/tenant/advanced) est désactivée. Si ce paramètre n’apparaît pas, c’est que votre tenant a été créé après la dépréciation de cette fonctionnalité; il est donc déjà désactivé par défaut. |
| allowLegacyRoGrantTypes                                                                                                                                                                        | L’ancien [endpoint oauth/ro](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) dans les paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications pour utiliser des endpoints pris en charge, assurez-vous de désactiver cet endpoint.                                                                                                                                                                                                                     |
| allowOtherLegacyGrantTypes                                                                                                                                                                     | Le [point de terminaison hérité oauth/access&#95;token](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) dans les paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                           |
| allowLegacyTokenInfoEndpoint                                                                                                                                                                   | Le [point de terminaison tokeninfo](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) hérité dans les paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications afin qu&#39;elles utilisent des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                 |
| allowLegacyDelegationGrantTypes                                                                                                                                                                | Le [point de terminaison de délégation](/docs/fr-CA/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation) dans les paramètres de migration du tenant est en cours d’abandon. Après avoir mis à jour vos applications pour qu’elles utilisent des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                                |
| [Allowed Callback URLs](/docs/fr-CA/authenticate/login/redirect-users-after-login) ne doit pas inclure localhost                                                                                     | Vérifie que les [URL de rappel autorisées de l’application](https://manage.auth0.com/#/applications) ne pointent pas vers `localhost`, `127.0.0.1`, etc.                                                                                                                                                                                                                                                                                                                                                                                  |
| [Origines autorisées (CORS)](/docs/fr-CA/authenticate/login/cross-origin-authentication) ne devrait pas être localhost                                                                               | Valide que la [Location URL](https://manage.auth0.com/#/applications) de la page ne pointe pas vers `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| [Origines Web autorisées](/docs/fr-CA/get-started/applications/application-settings) ne doivent pas inclure `localhost`                                                                              | Vérifie que les [URL des origines Web autorisées](https://manage.auth0.com/#/applications) ne pointent pas vers `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                              |
| [Fournisseur de courriel](/docs/fr-CA/customize/email/smtp-email-providers) doit être configuré                                                                                                      | Vérifie que le [fournisseur de courriel personnalisé](https://manage.auth0.com/#/emails/provider) est configuré.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Les Hooks sont dépréciés et doivent être [migrés vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions)                                                                  | Les Hooks sont dépréciés. Nous vous recommandons fortement de passer à [Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions) et d’entamer le processus de migration. Les Hooks existants resteront actifs jusqu’au 18 nov. 2024.                                                                                                                                                                                                                                                                                       |
| [Multi-Factor Auth](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) devrait être configuré (prérequis : MFA est configuré) | Si vous utilisez MFA avec le facteur [Message téléphonique](https://manage.auth0.com/#/mfa), configurez-le avec Twilio ou un fournisseur d’envoi de messages personnalisé afin d’éviter les limites de débit du fournisseur d’envoi de messages par défaut.                                                                                                                                                                                                                                                                               |
| L’[ancien profil utilisateur](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) doit être désactivé                                           | Les flux d’authentification hérités qui permettent aux ID Tokens et à l’endpoint `/userinfo` d’inclure le profil utilisateur complet sont dépréciés. Une fois la migration vers les nouvelles API conformes à OIDC terminée, assurez-vous que la bascule [Legacy User Profile](https://manage.auth0.com/#/account/advanced) est désactivée.                                                                                                                                                                                               |
| Les Rules sont en cours de retrait et doivent être [migrées vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions)                                                       | Les Rules sont en voie de dépréciation. Nous vous recommandons fortement de commencer à utiliser [Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions) et d’amorcer le processus de migration. Les Rules actuellement en place resteront actives jusqu’au 18 nov. 2024.                                                                                                                                                                                                                                                |
| [Connexions sociales](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/devkeys) ne doivent pas utiliser les clés de développeur Auth0                                           | Vérifie que les [Social Connections](https://manage.auth0.com/#/connections/social) n’utilisent pas les clés de développeur Auth0 par défaut.                                                                                                                                                                                                                                                                                                                                                                                             |
| Support Email doit être configuré                                                                                                                                                              | S’assure que le [Support Email](https://manage.auth0.com/#/account) est configuré dans « Tenant Settings ».                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Support URL est configurée                                                                                                                                                                     | S’assure que le [Support URL](https://manage.auth0.com/#/account) est configuré dans Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| L’étiquette d’environnement du tenant doit être configurée                                                                                                                                     | S’assure que la [balise d’environnement du tenant](https://support.auth0.com/tenants/public) est correctement définie sur Production, Staging ou Development. Toute modification de cette balise a une incidence sur la limite de requêtes de votre tenant.                                                                                                                                                                                                                                                                               |
| Le tenant est configuré pour utiliser une [version NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22) recommandée par défaut                          | Mettez à jour votre tenant pour utiliser une [version recommandée de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) afin de profiter des correctifs de sécurité et des nouvelles fonctionnalités.                                                                                                                                                                                                                                                                                           |
| URI de connexion du tenant                                                                                                                                                                     | Veillez à ce que [Tenant Login URI](https://manage.auth0.com/#/tenant/advanced) soit configuré dans Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Utiliser le domaine personnalisé dans les modèles de courriel à l’image de votre marque                                                                                                        | Assurez-vous que vos modèles de courriel utilisent un [domaine personnalisé](/docs/fr-CA/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails). Nous recommandons vivement de personnaliser tous les courriels destinés aux utilisateurs afin d’utiliser votre domaine personnalisé et l’image de marque de votre entreprise.                                                                                                                                                                         |
