---
description: Correctifs que vous devez apporter à votre Auth0 Application avant le déploiement en production
title: 'Vérifications de préparation à la production : correctifs critiques'
---

Les vérifications suivantes permettent de déterminer si vous avez satisfait à toutes les exigences nécessaires à un déploiement réussi en production.

Pour en savoir plus sur les résultats de vos vérifications, consultez [Exécuter les vérifications de production](/docs/fr-CA/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Vérification                                                                                                                                                                                                 | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Toutes les Actions s’exécutent avec une [version recommandée de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)                                                 | Configurez toutes les [Actions](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) pour utiliser une version recommandée de NodeJS afin de bénéficier des correctifs de sécurité et des nouvelles fonctionnalités.                                                                                                                                                                                                                                                                         |
| [Autoriser les ID Tokens pour l’authentification à la Management API v2](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) est désactivée                      | La possibilité d’utiliser des ID Tokens pour autoriser certains endpoints Users et Device Credentials de la Management API est en cours de dépréciation. Une fois la migration vers les Access Tokens terminée, assurez-vous que la bascule [Allow ID Tokens for Management API v2 Authentication](https://manage.auth0.com/#/tenant/advanced) est désactivée. Si vous ne voyez pas ce paramètre, c’est que votre tenant a été créé après la dépréciation de cette fonctionnalité; elle est donc déjà désactivée par défaut. |
| allowLegacyRoGrantTypes                                                                                                                                                                                      | Le [point de terminaison `oauth/ro` hérité](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) dans les paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                  |
| allowOtherLegacyGrantTypes                                                                                                                                                                                   | Le [point de terminaison oauth/access&#95;token ancien](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) dans les paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications pour utiliser les points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                              |
| allowLegacyTokenInfoEndpoint                                                                                                                                                                                 | Le [point de terminaison tokeninfo](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) hérité dans les paramètres de migration du tenant est en voie de dépréciation. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                   |
| allowLegacyDelegationGrantTypes                                                                                                                                                                              | Le [point de terminaison de délégation](/docs/fr-CA/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation) hérité dans les paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                |
| [Allowed Callback URLs](/docs/fr-CA/authenticate/login/redirect-users-after-login) ne doivent pas inclure Localhost                                                                                                | Vérifie que les [URL de rappel autorisées de l’application](https://manage.auth0.com/#/applications) ne pointent pas vers `localhost`, `127.0.0.1`, etc.                                                                                                                                                                                                                                                                                                                                                                     |
| [Origines autorisées (CORS)](/docs/fr-CA/authenticate/login/cross-origin-authentication) ne doivent pas être Localhost                                                                                             | Valide que l’[URL de la page](https://manage.auth0.com/#/applications) ne pointe pas vers `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                       |
| [Origines Web autorisées](/docs/fr-CA/get-started/applications/application-settings) ne doivent pas inclure `localhost`                                                                                            | Vérifie que les [URL des origines Web autorisées](https://manage.auth0.com/#/applications) ne pointent pas vers `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                 |
| [Fournisseur de courriel](/docs/fr-CA/customize/email/smtp-email-providers) doit être configuré                                                                                                                    | Vérifie que le [fournisseur de courriel personnalisé](https://manage.auth0.com/#/emails/provider) est configuré.                                                                                                                                                                                                                                                                                                                                                                                                             |
| Les Hooks sont dépréciés et doivent être [migrés vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions)                                                                                | Les Hooks sont dépréciés. Nous vous recommandons fortement de commencer à utiliser [Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions) et d’entamer le processus de migration. Les Hooks existants resteront actifs jusqu’au 18 novembre 2024.                                                                                                                                                                                                                                                          |
| [Authentification multifacteur](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) doit être configurée (Prérequis : la MFA est configurée) | Si vous utilisez MFA avec le [facteur Phone Message](https://manage.auth0.com/#/mfa), configurez-le avec Twilio ou un fournisseur de livraison personnalisé pour éviter les limites de débit du fournisseur de livraison par défaut.                                                                                                                                                                                                                                                                                         |
| [Profil utilisateur hérité](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) doit être désactivé                                                           | Les flux d’authentification hérités qui permettent aux ID Tokens et au point de terminaison `/userinfo` d’inclure le profil utilisateur complet sont en voie de dépréciation. Après avoir terminé la migration vers les nouvelles API conformes à OIDC, assurez-vous que la bascule [Profil utilisateur hérité](https://manage.auth0.com/#/account/advanced) est désactivée.                                                                                                                                                 |
| Les Rules sont dépréciées et doivent être [migrées vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions)                                                                              | Les Rules sont en voie d’abandon. Nous vous recommandons fortement de commencer à utiliser [Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions) et d’amorcer le processus de migration. Les Rules existantes resteront actives jusqu’au 18 nov. 2024.                                                                                                                                                                                                                                                    |
| [Connexions sociales](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/devkeys) ne devraient pas utiliser les clés de développeur Auth0                                                       | Vérifie que les [connexions sociales](https://manage.auth0.com/#/connections/social) n’utilisent pas les clés de développeur Auth0 par défaut.                                                                                                                                                                                                                                                                                                                                                                               |
| L’adresse courriel de soutien doit être configurée                                                                                                                                                           | Vérifie que [Support Email](https://manage.auth0.com/#/account) est configuré dans Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| L’URL de soutien est configurée                                                                                                                                                                              | S’assurer que le [Support URL](https://manage.auth0.com/#/account) est configuré dans Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                                       |
| L’étiquette d’environnement du tenant doit être configurée                                                                                                                                                   | Veille à ce que l’[étiquette d’environnement du tenant](https://support.auth0.com/tenants/public) soit définie correctement sur Production, Staging ou Development. Les modifications apportées à cette étiquette ont une incidence sur la limite de requêtes de votre tenant.                                                                                                                                                                                                                                               |
| Le tenant est configuré pour utiliser la [version de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22) recommandée par défaut                                      | Mettez votre tenant à niveau vers une [version recommandée de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) afin de profiter des correctifs de sécurité et des nouvelles fonctionnalités.                                                                                                                                                                                                                                                                                     |
| URI de connexion du tenant                                                                                                                                                                                   | Vérifie que [l’URI de connexion du tenant](https://manage.auth0.com/#/tenant/advanced) est configurée dans les paramètres du tenant.                                                                                                                                                                                                                                                                                                                                                                                         |
| Utiliser un domaine personnalisé dans les modèles de courriel à l’image de votre marque                                                                                                                      | Assurez-vous que vos modèles de courriel utilisent un [domaine personnalisé](/docs/fr-CA/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails). Nous recommandons fortement de personnaliser tous les courriels destinés aux utilisateurs afin qu’ils utilisent votre domaine personnalisé et l’image de marque de votre entreprise.                                                                                                                                                     |
