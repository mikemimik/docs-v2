---
description: Correctifs que vous devez apporter à votre application Auth0 avant le déploiement en production
title: 'Vérifications de préparation à la production : correctifs critiques'
---

Les vérifications suivantes permettent de déterminer si vous avez satisfait à toutes les exigences nécessaires à un déploiement réussi en Production.

Pour en savoir plus sur les résultats de ces vérifications, consultez [Exécuter les vérifications de production](/docs/fr-CA/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Contrôle                                                                                                                                                                                                     | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Toutes les Actions s’exécutent avec une [version recommandée de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)                                                 | Mettez toutes les [Actions](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) à niveau vers une version recommandée de NodeJS afin de bénéficier des correctifs de sécurité et des nouvelles fonctionnalités.                                                                                                                                                                                                                                                                                                    |
| [Autoriser les ID Tokens pour l’authentification à l’API Management v2](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) est désactivé                        | La possibilité d’utiliser des ID Tokens pour autoriser certains points de terminaison Users et Device Credentials du Management API est en cours de dépréciation. Une fois la migration vers les Access Tokens terminée, assurez-vous que la bascule [Autoriser les ID Tokens pour l’authentification à Management API v2](https://manage.auth0.com/#/tenant/advanced) est désactivée. Si ce paramètre n’apparaît pas, c’est que votre tenant a été créé après la dépréciation de cette fonctionnalité et qu’il est donc déjà désactivé par défaut. |
| allowLegacyRoGrantTypes                                                                                                                                                                                      | L’ancien [point de terminaison oauth/ro](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) dans les paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications pour utiliser les points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                                            |
| allowOtherLegacyGrantTypes                                                                                                                                                                                   | L’ancien [point de terminaison oauth/access&#95;token](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) dans les paramètres de migration du locataire est en voie d’être abandonné. Après avoir mis à jour vos applications pour utiliser les points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                   |
| allowLegacyTokenInfoEndpoint                                                                                                                                                                                 | L’ancien [point de terminaison tokeninfo](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) des paramètres de migration du tenant est en cours de dépréciation. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                                               |
| allowLegacyDelegationGrantTypes                                                                                                                                                                              | Le [point de terminaison de délégation](/docs/fr-CA/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation) hérité dans les paramètres de migration du tenant est en voie d’être retiré. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.                                                                                                                                                                                          |
| [Allowed Callback URLs](/docs/fr-CA/authenticate/login/redirect-users-after-login) ne doivent pas inclure localhost                                                                                                | Vérifie que les [URL de rappel autorisées de l’application](https://manage.auth0.com/#/applications) ne pointent pas vers `localhost`, `127.0.0.1`, etc.                                                                                                                                                                                                                                                                                                                                                                                            |
| [Origines autorisées (CORS)](/docs/fr-CA/authenticate/login/cross-origin-authentication) ne devraient pas être définies sur localhost                                                                              | Vérifie que l’[URL d’emplacement](https://manage.auth0.com/#/applications) de la page ne pointe pas vers `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                               |
| [Origines Web autorisées](/docs/fr-CA/get-started/applications/application-settings) ne doivent pas inclure `localhost`                                                                                            | Vérifie que les [URL d’origine Web autorisées](https://manage.auth0.com/#/applications) ne pointent pas vers `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                           |
| [Fournisseur de courriel](/docs/fr-CA/customize/email/smtp-email-providers) doit être configuré                                                                                                                    | Vérifie que le [fournisseur de courriel personnalisé](https://manage.auth0.com/#/emails/provider) est configuré.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Les Hooks sont dépréciés et doivent être [migrés vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions)                                                                                | Les Hooks sont dépréciés. Nous vous recommandons fortement de passer à [Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions) et d’amorcer le processus de migration. Les Hooks existants resteront actifs jusqu’au 18 novembre 2024.                                                                                                                                                                                                                                                                                             |
| [Authentification multifacteur](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) doit être configurée (Prérequis : la MFA est configurée) | Si vous utilisez la MFA avec le [facteur Phone Message](https://manage.auth0.com/#/mfa), configurez-le avec un fournisseur d’envoi Twilio ou personnalisé pour éviter les limites de débit du fournisseur d’envoi par défaut.                                                                                                                                                                                                                                                                                                                       |
| [Ancien profil utilisateur](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) doit être désactivé                                                           | Les flux d’authentification hérités qui permettent aux ID Tokens et au point de terminaison `/userinfo` d’inclure le profil utilisateur complet sont dépréciés. Une fois la migration vers les nouvelles API conformes à OIDC terminée, assurez-vous que la bascule [Legacy User Profile](https://manage.auth0.com/#/account/advanced) est désactivée.                                                                                                                                                                                              |
| Les Rules sont dépréciées et doivent être [migrées vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions)                                                                              | Les Rules sont dépréciées. Nous vous recommandons fortement de commencer à utiliser [Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions) et d’amorcer le processus de migration. Les Rules existantes resteront actives jusqu’au 18 nov. 2024.                                                                                                                                                                                                                                                                                  |
| [Connexions sociales](/docs/fr-CA/authenticate/identity-providers/social-identity-providers/devkeys) ne doivent pas utiliser les clés de développeur d’Auth0                                                       | Vérifie que les [connexions sociales](https://manage.auth0.com/#/connections/social) n’utilisent pas les clés de développeur Auth0 par défaut.                                                                                                                                                                                                                                                                                                                                                                                                      |
| L’adresse courriel de soutien doit être configurée                                                                                                                                                           | Vérifie que le [Support Email](https://manage.auth0.com/#/account) est configuré dans les Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| L’URL de soutien est configurée                                                                                                                                                                              | Vérifie que le [Support URL](https://manage.auth0.com/#/account) est configuré dans Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| L’étiquette d’environnement du locataire doit être configurée                                                                                                                                                | S’assure que l’[étiquette d’environnement du tenant](https://support.auth0.com/tenants/public) est réglée correctement à Production, Staging ou Development. Toute modification de cette étiquette a une incidence sur la limite de requêtes de votre tenant.                                                                                                                                                                                                                                                                                       |
| Le tenant est configuré pour utiliser la [version de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22) recommandée par défaut                                      | Mettez à jour votre tenant vers une [version recommandée de NodeJS](/docs/fr-CA/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) pour profiter des correctifs de sécurité et des nouvelles fonctionnalités.                                                                                                                                                                                                                                                                                                                 |
| URI de connexion du locataire                                                                                                                                                                                | Veillez à ce que l’[URI de connexion du tenant](https://manage.auth0.com/#/tenant/advanced) soit configuré dans les paramètres du tenant.                                                                                                                                                                                                                                                                                                                                                                                                           |
| Utiliser le domaine personnalisé dans les modèles de courriels aux couleurs de votre marque                                                                                                                  | Assurez-vous que vos modèles de courriel utilisent un [domaine personnalisé](/docs/fr-CA/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails). Nous recommandons fortement de personnaliser tous les courriels destinés aux utilisateurs pour qu’ils utilisent votre domaine personnalisé et reflètent l’image de marque de votre entreprise.                                                                                                                                                                  |
