L’option de déploiement Private Cloud comprend deux fonctionnalités complémentaires : Geo-failover et PCI Compliance. Si vous avez des questions sur ces fonctionnalités, veuillez [communiquer avec Auth0 Sales](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales).

<div id="private-cloud-geo-failover">
  ## Geo-failover de Private Cloud
</div>

Le Geo-failover est une configuration de basculement actif-passif qui permet aux clients de définir une région secondaire de basculement à utiliser en cas de panne majeure dans une région. En règle générale, toute région du même fournisseur infonuagique offrant une latence inférieure à 100 ms peut être choisie comme région de basculement. Dans certains cas, toutefois, les fournisseurs infonuagiques privilégient des régions de basculement précises. L’équipe d’onboarding d’Auth0 travaillera avec vous pour sélectionner les régions les mieux adaptées à votre cas d’utilisation.

De plus, une troisième région « d’arbitrage » est requise pour l’un de nos magasins de données. Cette troisième région ne reçoit jamais de trafic en direct, mais elle est nécessaire pour élire un nouveau leader en cas de basculement.

Si une défaillance régionale est détectée, le Geo-failover est déclenché par Auth0 pour le compte du client. L’ensemble du processus est automatisé une fois déclenché. Lorsqu’il est confirmé que la région ne connaît plus de problèmes, Auth0 rétablira le service dans la région principale.
