L’option de déploiement Private Cloud comprend deux fonctionnalités de module complémentaire : Geo-failover et la conformité PCI. Si vous avez des questions au sujet de ces fonctionnalités, veuillez [contacter Auth0 Sales](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales).

<div id="private-cloud-geo-failover">
  ## Private Cloud Geo-Failover
</div>

Le Geo-failover est une configuration de basculement actif-passif qui permet aux clients de préciser une région de basculement secondaire à utiliser en cas de panne régionale majeure. En général, toute région du même fournisseur infonuagique offrant une latence inférieure à 100 ms peut être choisie comme région de basculement. Dans certains cas, toutefois, les fournisseurs infonuagiques privilégient des régions de basculement précises. L’équipe d’onboarding d’Auth0 travaillera avec vous pour sélectionner les régions les mieux adaptées à votre use case.

De plus, une troisième région de « départage » est requise pour l’un de nos systèmes de stockage de données. Cette troisième région ne reçoit jamais de trafic en direct, mais elle est nécessaire pour élire un nouveau leader en cas de basculement.

Si une défaillance régionale est détectée, Auth0 déclenche le Geo-failover au nom du client. L’ensemble du processus est automatisé une fois déclenché. Lorsqu’une confirmation est reçue indiquant que la région ne connaît plus de problèmes, Auth0 rebascule vers la région principale.
