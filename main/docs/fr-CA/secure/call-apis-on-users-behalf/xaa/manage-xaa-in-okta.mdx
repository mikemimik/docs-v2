Une fois la configuration de votre environnement de test de bout en bout terminée, vous pouvez gérer, dans l’Okta Admin Console, la façon dont les applications XAA autorisées peuvent se connecter entre elles.

<Frame>![](/docs/images/xaa/manage_xaa_in_okta.png)</Frame>

Dans l’Okta Admin Console :

1. Accédez à **Applications &gt; Applications** et sélectionnez votre Resource App (p. ex. Todo0).
2. Sous l’onglet **Manage Connections**, ajoutez :
   * Requesting Apps : les applications qui peuvent se connecter à votre application SaaS
   * Resource Apps : les applications auxquelles votre application SaaS peut se connecter

Pour l’environnement de test de bout en bout, ajoutez Agent0, ou l’application que vous souhaitez utiliser pour les tests, comme Requesting App autorisée.
