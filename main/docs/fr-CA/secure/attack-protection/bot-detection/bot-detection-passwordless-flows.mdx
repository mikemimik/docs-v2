title: Ajouter Bot Detection aux flux sans mot de passe
---

Si vous utilisez des flux de connexion et d’inscription <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip>, vous pouvez activer <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Bot+Detection">Bot Detection</Tooltip> pour afficher une étape CAPTCHA lorsque Auth0 détermine qu’une requête présente un risque élevé.

<Warning>
  Si vous utilisez le [SDK Lock](/docs/fr-CA/libraries/lock) avec l’expérience Classic Login, vous devez effectuer une mise à jour vers la version 11.35.0 ou ultérieure pour utiliser Bot Detection avec les flux sans mot de passe.

  Si vous utilisez le [SDK Auth0.js](/docs/fr-CA/libraries/auth0js), vous devez effectuer une mise à jour vers la version 9.20.0 ou ultérieure pour utiliser Bot Detection avec les flux sans mot de passe.
</Warning>

<div id="configure-bot-detection-for-passwordless-flows">
  ## Configurer Bot Detection pour les flux sans mot de passe
</div>

Vous pouvez utiliser le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> pour configurer le comportement de la fonction CAPTCHA.

1. Dans le [Auth0 Dashboard](https://manage.auth0.com/dashboard), accédez à [Sécurité &gt; Protection contre les attaques &gt; Bot Detection](https://manage.auth0.com/dashboard/#/security/attack-protection/bot-detection).
2. Repérez la section **Réponse**.
3. Pour le paramètre **Exiger un CAPTCHA pour les flux sans mot de passe**, sélectionnez quand exiger un CAPTCHA :

   * **Jamais** : les utilisateurs n’ont pas à remplir de CAPTCHA pour se connecter.
   * **En cas de risque** : les utilisateurs doivent remplir un CAPTCHA si la tentative de connexion présente un risque élevé.
   * **Toujours** : les utilisateurs doivent toujours remplir un CAPTCHA pour se connecter.
