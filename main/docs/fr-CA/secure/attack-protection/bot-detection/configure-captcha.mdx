Auth0 prend en charge plusieurs intégrations avec des fournisseurs de CAPTCHA tiers pour [Bot Detection](/docs/fr-CA/secure/attack-protection/bot-detection).

<div id="google-recaptcha-enterprise-on-google-cloud-platform-gcp">
  ## Google reCAPTCHA Enterprise sur Google Cloud Platform (GCP)
</div>

Vous pouvez utiliser Google reCAPTCHA Enterprise comme service de <Tooltip tip="Bot Detection : forme de protection contre les attaques dans laquelle Auth0 bloque le trafic de robots suspect en activant un CAPTCHA pendant le processus de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Bot+Detection">Bot Detection</Tooltip> pour le trafic qui dépasse 1 million d’évaluations par mois.

Pour savoir comment configurer reCAPTCHA Enterprise sur GCP, consultez [Set up reCAPTCHA Enterprise ... on Google Documentation](https://cloud.google.com/recaptcha-enterprise/docs/authenticate-apis).

<div id="locate-your-gcp-project-id">
  ### Trouvez l’ID de votre projet GCP
</div>

1. Accédez à votre [Google Cloud Dashboard](https://console.cloud.google.com/).
2. Sélectionnez votre projet dans le menu déroulant pour afficher l’**ID du projet** correspondant.
3. Prenez en note l’**ID du projet**, car vous en aurez besoin plus tard.

<div id="enable-recaptcha-enterprise">
  ### Activer reCAPTCHA Enterprise
</div>

1. Dans le [Google Cloud Dashboard](https://console.cloud.google.com/), entrez reCAPTCHA Enterprise API dans la barre de recherche, puis sélectionnez-le dans les résultats de Marketplace.
2. Sélectionnez **Activer** pour activer l’API.

<div id="create-a-site-key">
  ### Créer une clé de site
</div>

1. Dans le [Google Cloud Dashboard](https://console.cloud.google.com/), saisissez reCAPTCHA Enterprise dans la barre de recherche, puis sélectionnez-le dans les résultats du Marketplace.
2. Sélectionnez **Create Key**.
3. Sous **Domain List**, ajoutez le domaine de votre tenant Auth0.
4. Activez le bouton bascule pour **Disable domain verification**.
5. Activez le bouton bascule pour **Use checkbox challenge** afin d’activer la case à cocher de vérification Google « Je ne suis pas un robot ».
6. Sous **Challenge Security**, sélectionnez un niveau de difficulté pour la demande de vérification.
7. Sélectionnez **Create Key**.
8. Prenez en note la **Site Key**, car vous en aurez besoin plus tard.

<div id="create-an-api-key">
  ### Créer une clé API
</div>

1. Dans le [Google Cloud Dashboard](https://console.cloud.google.com/), saisissez Credentials dans la barre de recherche, puis sélectionnez cette option dans les résultats.
2. Sélectionnez **Create Credentials &gt; API Key**. Prenez note de l’**API Key**, car vous en aurez besoin plus tard.
3. Dans la fenêtre contextuelle, sélectionnez **Edit Key**.
4. Sous **Application Restrictions**, sélectionnez **IP addresses (web servers, cron jobs, etc.)**, puis entrez les [adresses IP d’Auth0 pour les listes d’autorisation](/docs/fr-CA/secure/security-guidance/data-security/allowlist).
5. Sous **API Restrictions**, sélectionnez **Restrict Key**, choisissez **reCAPTCHA Enterprise API** dans la liste déroulante, puis cliquez sur OK.
6. Sélectionnez **Save**.

<div id="hcaptcha">
  ## hCaptcha
</div>

Le widget hCaptcha peut protéger vos applications contre les robots, les pourriels et d’autres formes d’abus automatisés.

Pour en savoir plus sur hCaptcha et pour savoir comment obtenir les détails de configuration, consultez [Switch from reCAPTCHA to hCaptcha on hCaptcha Docs](https://docs.hcaptcha.com/switch).

<div id="friendly-captcha">
  ## Friendly Captcha
</div>

Friendly Captcha est un système qui aide à prévenir le pourriel sur votre site Web. Vous pouvez ajouter le widget Friendly Captcha à votre application Web pour lutter contre le pourriel, avec peu d’incidence sur l’expérience utilisateur.

Pour en savoir plus sur Friendly Captcha et sur la façon d’obtenir les renseignements de configuration, consultez [Installation dans la documentation de Friendly Captcha](https://docs.friendlycaptcha.com/#/installation?id=installation).

<div id="arkose">
  ## Arkose
</div>

Arkose Labs offre une solution de détection de bots pour atténuer les attaques menées par des humains et par des bots. Pour en savoir plus sur l’intégration d’Arkose comme fournisseur de CAPTCHA, consultez [Arkose Bot Manager](https://www.arkoselabs.com/arkose-bot-manager/).

<Warning>
  Vous devez mettre à niveau Auth0.js vers la version 9.23 ou ultérieure pour utiliser Arkose comme fournisseur de CAPTCHA.
</Warning>

<div id="configure-callback-action-with-auth0js">
  ### Configurer l’action de rappel avec Auth0.js
</div>

Lorsque vous utilisez Arkose comme fournisseur de CAPTCHA, la méthode `renderCaptcha` du client WebAuth d’Auth0.js vous permet de transmettre une fonction de rappel. Le deuxième paramètre de la fonction de rappel est un objet payload contenant la fonction `triggerCaptcha`. La fonction `triggerCaptcha` peut servir à configurer la façon dont votre application gère et affiche l’invite CAPTCHA.

Pour en savoir plus sur la configuration de la méthode `renderCaptcha`, consultez [renderCaptcha](https://auth0.github.io/auth0.js/WebAuth.html#renderCaptcha) dans la documentation d’Auth0.js.
