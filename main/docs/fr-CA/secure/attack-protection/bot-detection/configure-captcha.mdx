Auth0 prend en charge plusieurs intégrations de fournisseurs CAPTCHA tiers pour [Bot Detection](/docs/fr-CA/secure/attack-protection/bot-detection).

<div id="google-recaptcha-enterprise-on-google-cloud-platform-gcp">
  ## Google reCAPTCHA Enterprise sur Google Cloud Platform (GCP)
</div>

Vous pouvez utiliser Google reCAPTCHA Enterprise comme service de <Tooltip tip="Bot Detection : forme de protection contre les attaques où Auth0 bloque le trafic de robots suspect en activant un CAPTCHA pendant le processus de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Bot+Detection">Bot Detection</Tooltip> lorsque le trafic dépasse 1 million d’évaluations par mois.

Pour savoir comment configurer reCAPTCHA Enterprise sur GCP, consultez [Set up reCAPTCHA Enterprise ... on Google Documentation](https://cloud.google.com/recaptcha-enterprise/docs/authenticate-apis).

<div id="locate-your-gcp-project-id">
  ### Repérez l’ID de votre projet GCP
</div>

1. Accédez à votre [Google Cloud Dashboard](https://console.cloud.google.com/).
2. Sélectionnez votre projet dans le menu déroulant pour afficher l’**ID du projet** correspondant.
3. Prenez en note l’**ID du projet**, car vous en aurez besoin plus tard.

<div id="enable-recaptcha-enterprise">
  ### Activer reCAPTCHA Enterprise
</div>

1. Dans le [Google Cloud Dashboard](https://console.cloud.google.com/), saisissez reCAPTCHA Enterprise API dans la barre de recherche, puis sélectionnez-le parmi les résultats du Marketplace.
2. Sélectionnez **Enable** pour activer l’API.

<div id="create-a-site-key">
  ### Créer une clé de site
</div>

1. Dans le [Google Cloud Dashboard](https://console.cloud.google.com/), saisissez reCAPTCHA Enterprise dans la barre de recherche, puis sélectionnez-le dans les résultats du Marketplace.
2. Sélectionnez **Créer une clé**.
3. Sous **Liste des domaines**, ajoutez le domaine de votre tenant Auth0.
4. Activez la bascule pour **Désactiver la vérification du domaine**.
5. Activez la bascule pour **Utiliser le test par case à cocher** afin d’activer la case à cocher de vérification « Je ne suis pas un robot » de Google.
6. Sous **Sécurité du test**, sélectionnez un niveau de difficulté pour le test.
7. Sélectionnez **Créer une clé**.
8. Prenez en note la **Clé de site**, car vous en aurez besoin plus tard.

<div id="create-an-api-key">
  ### Créer une clé API
</div>

1. Dans le [Google Cloud Dashboard](https://console.cloud.google.com/), entrez Credentials dans la barre de recherche, puis sélectionnez cette option dans les résultats.
2. Sélectionnez **Create Credentials &gt; API Key**. Prenez note de la **clé API**, car vous en aurez besoin plus tard.
3. Dans la fenêtre contextuelle, sélectionnez **Edit Key**.
4. Sous **Application Restrictions**, sélectionnez **IP addresses (web servers, cron jobs, etc.)**, puis saisissez les [adresses IP Auth0 pour les listes d’autorisation](/docs/fr-CA/secure/security-guidance/data-security/allowlist).
5. Sous **API Restrictions**, sélectionnez **Restrict Key**, choisissez **reCAPTCHA Enterprise API** dans la liste déroulante, puis cliquez sur OK.
6. Sélectionnez **Save**.

<div id="hcaptcha">
  ## hCaptcha
</div>

Le widget hCaptcha peut protéger vos applications contre les robots, le pourriel et d’autres formes d’abus automatisés.

Pour en savoir plus sur hCaptcha et pour savoir comment obtenir les détails de configuration, consultez [Switch from reCAPTCHA to hCaptcha on hCaptcha Docs](https://docs.hcaptcha.com/switch).

<div id="friendly-captcha">
  ## Friendly Captcha
</div>

Friendly Captcha est un système qui permet de prévenir le spam sur votre site web. Vous pouvez ajouter le widget Friendly Captcha à votre application web pour lutter contre le spam, avec peu d’impact sur l’expérience utilisateur.

Pour en savoir plus sur Friendly Captcha et sur la façon d’obtenir les détails de configuration, consultez [Installation dans la documentation de Friendly Captcha](https://docs.friendlycaptcha.com/#/installation?id=installation).

<div id="arkose">
  ## Arkose
</div>

Arkose Labs offre une solution de détection des bots pour atténuer les attaques d’origine humaine et celles menées par des bots. Pour en savoir plus sur l’intégration fluide d’Arkose comme fournisseur CAPTCHA, consultez [Arkose Bot Manager](https://www.arkoselabs.com/arkose-bot-manager/).

<Warning>
  Vous devez passer à Auth0.js v9.23+ pour utiliser Arkose comme fournisseur CAPTCHA.
</Warning>

<div id="configure-callback-action-with-auth0js">
  ### Configurer l’action de rappel avec Auth0.js
</div>

Lorsque vous utilisez Arkose comme fournisseur de CAPTCHA, la méthode `renderCaptcha` du client WebAuth d’Auth0.js vous permet de transmettre une fonction de rappel. Le deuxième paramètre de cette fonction de rappel est un objet payload contenant la fonction `triggerCaptcha`. La fonction `triggerCaptcha` peut être utilisée pour configurer la manière dont votre application gère et affiche l’écran CAPTCHA.

Pour en savoir plus sur la configuration de la méthode `renderCaptcha`, consultez [renderCaptcha](https://auth0.github.io/auth0.js/WebAuth.html#renderCaptcha) dans la documentation d’Auth0.js.
