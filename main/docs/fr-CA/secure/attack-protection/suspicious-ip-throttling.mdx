<Tooltip tip="Suspicious IP Throttling : forme de protection contre les attaques qui protège votre tenant contre les connexions suspectes visant un trop grand nombre de comptes à partir d’une seule adresse IP." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Suspicious+IP+Throttling">Suspicious IP Throttling</Tooltip> bloque le trafic provenant de toute adresse IP qui effectue rapidement un trop grand nombre de tentatives de connexion ou d’inscription. Cela aide à protéger vos applications contre les attaques à volume élevé qui ciblent plusieurs comptes.

Suspicious IP Throttling est activé par défaut lorsque vous créez votre tenant Auth0. Lorsqu’Auth0 détecte un grand nombre de tentatives d’inscription ou de tentatives de connexion échouées provenant d’une adresse IP, il répond aux tentatives suivantes avec le [code d’état HTTP 429 Too Many Requests](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/429) jusqu’à ce que cette adresse IP ne soit plus soumise à la limitation.

Pour en savoir plus sur les inscriptions et les tentatives de connexion échouées consignées par votre tenant, consultez [les codes de type de log](/docs/fr-CA/deploy-monitor/logs/log-event-type-codes).

<div id="configure-suspicious-ip-throttling">
  ## Configurer Suspicious IP Throttling
</div>

Vous pouvez configurer Suspicious IP Throttling dans [Auth0 Dashboard](https://manage.auth0.com/#) ou au moyen de [Auth0 Management API](https://auth0.com/docs/api/management/v2/attack-protection/patch-suspicious-ip-throttling).

<Warning>
  Auth0 recommande fortement de ne pas désactiver Suspicious IP Throttling.
</Warning>

1. Accédez à [Dashboard &gt; Sécurité &gt; Protection contre les attaques](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling**.
2. Sélectionnez la bascule en haut de la page pour activer ou désactiver Suspicious IP Throttling.

<div id="allow-trusted-ip-addresses-exceed-throttling-limits">
  ## Autoriser des adresses IP de confiance à dépasser les limites de throttling
</div>

Vous pouvez exempter jusqu’à 100 adresses IP distinctes ou plages CIDR (IPv4 ou IPv6) de **Suspicious IP Throttling** en les ajoutant à l’**IP AllowList**. Auth0 ne bloque pas les administrateurs de tenant et ne leur envoie pas d’alerte lorsque ces adresses IP dépassent les limites de throttling.

1. Accédez à [Dashboard &gt; Sécurité &gt; Protection contre les attaques](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling**.
2. Dans le champ **IP AllowList**, entrez les adresses IP ou plages CIDR que vous souhaitez autoriser à effectuer un nombre illimité de tentatives de connexion et d’inscription. Séparez plusieurs adresses IP ou plages CIDR par des virgules.

<div id="configure-response">
  ## Configurer la réponse
</div>

Par défaut, Auth0 envoie un courriel aux administrateurs de tenant lorsqu’une adresse IP est jugée suspecte.

Vous pouvez configurer la façon dont Auth0 réagit aux tentatives de connexion ou d’inscription à volume élevé.

<Warning>
  Activer les fonctionnalités de protection contre les attaques sans activer de paramètres de réponse met en mode Monitoring, qui consigne uniquement les événements connexes dans votre journal de tenant. Pour en savoir plus, consultez [View Attack Protection Log Events](/docs/fr-CA/secure/attack-protection/view-attack-protection-events).
</Warning>

1. Accédez à [Dashboard &gt; Sécurité &gt; Protection contre les attaques](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling.**
2. Repérez la section **Response**.
3. Dans la section **Block Settings**, activez **Limit high-velocity traffic targeting too many accounts** pour limiter le trafic provenant d’adresses IP qui dépassent le seuil de connexions ou d’inscriptions.
4. Dans la section **Notifications**, activez **Send notification to account administrator** pour envoyer automatiquement un courriel aux administrateurs de tenant lorsqu’une adresse IP dépasse le seuil de connexions ou d’inscriptions.
5. Sélectionnez **Save**.

<div id="configure-throttling-limits-and-rates">
  ## Configurer les limites et les taux de limitation
</div>

Vous pouvez personnaliser la façon dont Auth0 applique une limitation aux adresses IP suspectes. Vous pouvez modifier :

* Le nombre maximal de tentatives de connexion et d&#39;inscription échouées autorisées à partir d&#39;une adresse IP donnée.
* La fréquence à laquelle les adresses IP limitées peuvent effectuer de nouvelles tentatives de connexion et d&#39;inscription.

Pour en savoir plus sur le nombre maximal de tentatives, le taux de limitation et leur fonctionnement, consultez [Comment fonctionne Suspicious IP Throttling](#how-suspicious-ip-throttling-works).

1. Accédez à [Dashboard &gt; Sécurité &gt; Protection contre les attaques](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling.**

   <Frame>![Personnalisation de Suspicious IP Throttling dans le Dashboard Auth0](/docs/images/cdy7uua7fh8z/5yLYmDn0FvDB061WgTH2fk/0f3dce552a41aa5756a2eca788944965/Screen_Shot_2021-09-29_at_10.19.32_AM.png)</Frame>
2. Repérez la section **Detection**.
3. Sélectionnez **Custom** pour **Suspicious IP Thresholds.**
4. Configurez les paramètres de **Login Threshold** :

   * **Maximum Attempts** : Entrez le nombre de tentatives de connexion échouées qu&#39;une même adresse IP peut effectuer en une journée avant qu&#39;Auth0 bloque la tentative suivante.
   * **Taux de limitation** : Entrez la fréquence à laquelle de nouveaux jetons de connexion sont accordés.
5. Configurez les paramètres de **Signup Threshold** :

   * **Maximum Attempts** : Entrez le nombre de tentatives d&#39;inscription qu&#39;une même adresse IP peut effectuer en une minute avant qu&#39;Auth0 bloque la tentative suivante.
   * **Taux de limitation** : Entrez la fréquence à laquelle de nouveaux jetons d&#39;inscription sont accordés.
6. Sélectionnez **Enregistrer**.

<div id="how-suspicious-ip-throttling-works">
  ### Fonctionnement de Suspicious IP Throttling
</div>

Auth0 comptabilise et autorise séparément les tentatives de connexion et d’inscription. Les adresses IP pour lesquelles les tentatives de connexion sont suspendues peuvent quand même tenter de s’inscrire. Les adresses IP pour lesquelles les tentatives d’inscription sont suspendues peuvent quand même tenter de se connecter.

<div id="login-attempts">
  #### Tentatives de connexion
</div>

Auth0 limite toute adresse IP qui effectue trop de tentatives de connexion infructueuses en une journée. Le taux de limitation détermine le nombre de tentatives de connexion qu’Auth0 accorde à une adresse IP, réparties uniformément sur 24 heures. Par exemple, un taux de limitation de 100 signifie qu’Auth0 accorde une nouvelle tentative environ toutes les 15 minutes.

<div id="signup-attempts">
  #### Tentatives d’inscription
</div>

Auth0 limite temporairement toute adresse IP qui effectue un trop grand nombre de tentatives d’inscription en une minute. Par exemple, si une adresse IP effectue 50 tentatives d’inscription, Auth0 bloque les tentatives suivantes. Contrairement aux connexions, les tentatives d’inscription n’ont pas besoin d’être des échecs.

Le taux de limitation des inscriptions détermine le nombre de tentatives d’inscription qu’Auth0 accorde à une adresse IP, réparties uniformément sur 24 heures. Par exemple, un taux de limitation de 72 000 signifie qu’Auth0 accorde une nouvelle tentative environ toutes les secondes.

<div id="malformed-requests-and-schema-validation-errors">
  #### Requêtes malformées et erreurs de validation du schéma
</div>

Les requêtes de connexion et d’inscription qui ne respectent pas la structure ou le format requis ne sont pas considérées comme des tentatives valides et, par conséquent, ne sont pas prises en compte dans le seuil que vous avez configuré.

Pour en savoir plus sur les requêtes de connexion et d’inscription, consultez [API d’authentification Auth0](https://auth0.com/docs/api/authentication).

<div id="special-cases">
  ## Cas particuliers
</div>

Comme la limitation du trafic des IP suspectes dépend de l&#39;adresse IP de l&#39;utilisateur, les cas d&#39;utilisation suivants nécessitent une configuration supplémentaire :

* **Utilisation du <Tooltip tip="Resource Owner : entité (comme un utilisateur ou une application) capable d’accorder l’accès à une ressource protégée." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Resource+Owner">Resource Owner</Tooltip> Password Grant depuis le backend d&#39;une application** : L&#39;utilisation de cet appel ne permet pas d&#39;obtenir l&#39;adresse IP de l&#39;utilisateur; toutefois, pour que la limitation du trafic des IP suspectes fonctionne correctement, vous pouvez configurer votre application pour envoyer l&#39;adresse IP de l&#39;utilisateur dans la requête. Consultez [Éviter les problèmes courants avec le flux Resource Owner Password et la protection contre les attaques : envoyer l&#39;adresse IP de l&#39;utilisateur depuis votre serveur](/docs/fr-CA/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
* **Authentification d&#39;un grand nombre d&#39;utilisateurs à partir de la même adresse IP** : Les utilisateurs qui passent par un proxy sont plus susceptibles d&#39;atteindre les limites définies et de déclencher la limitation. Vous pouvez éviter de déclencher la limitation par erreur en configurant une AllowList pour l&#39;adresse IP et la plage CIDR du proxy. Consultez [Éviter les problèmes courants avec le flux Resource Owner Password et la protection contre les attaques : Configurer votre Application pour faire confiance à l&#39;adresse IP](/docs/fr-CA/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
