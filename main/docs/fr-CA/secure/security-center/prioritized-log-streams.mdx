title: Prioritized Log Streams
---

**Prioritized Log Streams** est une fonctionnalité du Dashboard qui met en évidence les événements de sécurité. En cas d’attaque ou de hausse de l’activité des utilisateurs, Prioritized Log Streams et les événements prédéfinis resteront actifs afin que l’automatisation et les processus de sécurité en aval ne soient pas touchés.

Vous pouvez créer un nouveau flux de journaux prioritaire à l’aide d’un service pris en charge ou configurer un webhook personnalisé pour envoyer les données vers un service que vous utilisez déjà, et tous les journaux peuvent être exportés vers un service d’analyse d’événements. Les Prioritized Log Streams ne comptent pas dans la limite de flux de journaux indiquée dans le plan tarifaire.

<div id="configure-a-prioritized-log-stream">
  #### Configurer un flux de journaux priorisé
</div>

1. Suivez les instructions de la page [Log Streams](/docs/fr-CA/customize/log-streams) pour configurer un nouveau flux de journaux.
2. Pendant la configuration du nouveau flux, accédez à l’onglet **Settings** dans votre Auth0 Dashboard et cochez la case **Prioritized Log** pour activer les événements de sécurité prédéfinis décrits ci-dessous. Sélectionnez **Save**.
3. Accédez à votre page **Stream** et repérez le libellé **Prioritized**.

<div id="prioritized-logs-event-types">
  #### Types d’événements de log prioritaires
</div>

Auth0 classe les codes d’événement suivants parmi les événements liés à la sécurité :

| Code d’événement  | Événement                                          |
| ----------------- | -------------------------------------------------- |
| `limit_mu`        | Adresse IP bloquée                                 |
| `limit_sul`       | Compte bloqué (limitation IP)                      |
| `limit_wc`        | Compte bloqué (Brute Force Protection)             |
| `pwd_leak`        | Mot de passe compromis à la connexion              |
| `ublkdu`          | Blocage de connexion de l’utilisateur levé         |
| `signup_pwd_leak` | Mot de passe compromis à l’inscription             |
| `reset_pwd_leak`  | Mot de passe compromis lors de la réinitialisation |
| `gd_send_sms`     | SMS de MFA envoyé                                  |
