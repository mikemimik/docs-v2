title: Prioritized Log Streams
---

**Prioritized Log Streams** est une fonctionnalité du Dashboard qui met en avant les événements de sécurité. En cas d&#39;attaque ou d&#39;augmentation de l&#39;activité des utilisateurs, Prioritized Log Streams et les événements prédéfinis resteront actifs afin que l&#39;automatisation et les processus de sécurité en aval ne soient pas touchés.

Vous pouvez créer un nouveau Prioritized Log Stream à l&#39;aide d&#39;un service pris en charge ou configurer un Webhook personnalisé pour diffuser en continu vers un service que vous utilisez déjà, et tous les logs peuvent être exportés vers un service d&#39;analyse d&#39;événements. Les Prioritized Log Streams ne sont pas comptabilisés dans la limite de log streams indiquée dans votre forfait.

<div id="configure-a-prioritized-log-stream">
  #### Configurer un Prioritized Log Stream
</div>

1. Suivez les instructions de la [page Log Streams](/docs/fr-CA/customize/log-streams) pour configurer un nouveau log stream.
2. Pendant la configuration du nouveau stream, accédez à l’onglet **Settings** de votre Auth0 Dashboard, puis cochez la case **Prioritized Log** pour activer les événements de sécurité prédéfinis décrits ci-dessous. Cliquez sur **Save**.
3. Accédez à votre page **Stream** et allez au libellé **Prioritized**.

<div id="prioritized-logs-event-types">
  #### Types d’événements de journaux prioritaires
</div>

Auth0 classe les codes d’événement suivants parmi les événements liés à la sécurité :

| Code d’événement  | Événement                                                      |
| ----------------- | -------------------------------------------------------------- |
| `limit_mu`        | Adresse IP bloquée                                             |
| `limit_sul`       | Compte bloqué (limitation des requêtes IP)                     |
| `limit_wc`        | Compte bloqué (protection contre les attaques par force brute) |
| `pwd_leak`        | Mot de passe compromis à la connexion                          |
| `ublkdu`          | Levée du blocage de connexion de l’utilisateur                 |
| `signup_pwd_leak` | Mot de passe compromis à l’inscription                         |
| `reset_pwd_leak`  | Mot de passe compromis lors de la réinitialisation             |
| `gd_send_sms`     | SMS MFA envoyé                                                 |
