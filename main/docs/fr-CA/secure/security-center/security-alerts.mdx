---

**Les alertes de surveillance de sécurité** permettent aux utilisateurs de configurer les seuils des métriques de sécurité dans leur Security Center. Il est possible de définir, pour chaque seuil, des valeurs numériques pour l’alerte, l’avertissement et le retour à la normale, ainsi que de configurer les notifications d’alerte utilisateur pour être avisé lorsqu’une métrique de menace dépasse un seuil défini.

| Option              | Description                                                                                                                                           |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| Alerte              | Valeur numérique obligatoire qui génère un avis d’alerte lorsque la métrique évaluée dépasse la valeur fournie.                                       |
| Avertissement       | Valeur numérique facultative qui génère un avis d’avertissement lorsque la métrique évaluée dépasse la valeur fournie.                                |
| Retour à la normale | Valeur numérique facultative qui génère un avis de retour à la normale lorsque la métrique évaluée revient à une valeur qui ne dépasse plus le seuil. |

<div id="configure-and-update-alert-thresholds">
  #### Configurer et mettre à jour les seuils d’alerte
</div>

Les seuils sont calculés à partir d’une moyenne mobile pondérée pour une métrique donnée et peuvent être personnalisés dans votre <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Chaque seuil défini est visible dans les graphiques des métriques de surveillance des menaces et agrégé sur une base horaire; si aucun seuil de retour à la normale n’est configuré, la récupération se fait par défaut juste en dessous du seuil d’avertissement ou d’alerte défini.

<Warning>
  Les seuils s’appliquent à la métrique d’origine, sans filtres. Lorsque des filtres sont appliqués, les courbes de tendance du seuil d’origine et de la vue filtrée sont visibles.
</Warning>

<Frame>![Une deuxième capture d’écran de notre produit Seuils du Security Center](/docs/images/cdy7uua7fh8z/1Kd9d33WUCdlVa1SwEFkKQ/496722ce86bc00cc7f58d1e04e1fd006/MFA_success_rate_threshold_editor_-_english.png)</Frame>

1. Accédez à [**Sécurité &gt; Security Center &gt; Surveillance des menaces**](https://manage.auth0.com/#/security/center/bot-detection) et choisissez un graphique de métrique.
2. Sélectionnez l’icône **Voir les détails** dans le coin supérieur droit.
3. Accédez au panneau Seuils affiché sous la vue détaillée du graphique et choisissez **Créer.**
4. Donnez un nom au seuil et configurez les paramètres suivants :

   * Quand le seuil doit déclencher un avertissement
   * Quand le seuil doit déclencher une alerte
   * Quand le seuil doit revenir à la normale
5. Si des destinations de notification ont été configurées, les options suivantes sont offertes :

   * Sélectionnez une destination pour recevoir les avis d’alerte, d’avertissement et de retour à la normale pour la métrique,
   * Créez une nouvelle destination en sélectionnant le **+**
   * Désactivez temporairement ou indéfiniment les notifications pour toutes les destinations de seuil dans la liste déroulante **Désactiver les notifications**.
6. Sélectionnez **Enregistrer.**

Les seuils peuvent aussi être mis à jour ou supprimés dans l’écran de vue développée. Les différents seuils d’un même graphique se trouvent derrière le chevron de l’étiquette Threshold, en haut à droite.

<div id="manage-notification-destinations">
  #### Gérer les destinations de notification
</div>

Les destinations de notification sont des endpoints auxquels sont envoyés les avis d’alerte, d’avertissement et de retour à la normale. Chaque tenant est limité à deux endpoints de destination, et il est recommandé d’utiliser un éditeur de webhook tiers pour personnaliser le message de la notification.

1. Accédez à la page Manage Destinations dans votre Dashboard à partir du panneau de configuration Seuil, ou en allant à [**Sécurité &gt; Security Center &gt; Manage Destinations**](https://manage.auth0.com/#/security/center/manage-destinations)
2. Sélectionnez le bouton **New Destination** et fournissez les détails suivants :

   1. Nom
   2. URL de destination
   3. Jeton d’autorisation
3. Choisissez **Save.** Pour supprimer un endpoint de destination, accédez à **More Actions &gt; Delete**.

<div id="notification-payload-data">
  #### Données du payload de notification
</div>

Les champs racine inclus dans le payload de notification sont les suivants :

* `id` : ID de l’alerte du Security Center
* `time` : date et heure UTC du dépassement du seuil
* `source` : URL du dashboard Historique des alertes du Security Center d’Auth0
* `data` : objet de données contenant les renseignements pertinents sur la notification d’alerte

Voici l’objet de données inclus dans le payload de notification :

* `id` : ID de l’alerte du Security Center
* `tenant` : tenant d’où provient la notification du Security Center
* `evaluated_metric` : métrique de menace à laquelle le seuil s’applique
* `state` : état de la notification (ALERT/WARN/RECOVERED)
* `metric_value` : valeur de la métrique évaluée au cours des 60 dernières minutes
* `alert_threshold` : valeur du seuil d’alerte configuré pour la métrique de menace
* `warn_threshold` : valeur du seuil d’avertissement configuré pour la métrique de menace
* `recovery_threshold` : valeur du seuil de retour à la normale configuré pour la métrique de menace
* `triggered_at` : date et heure UTC du dépassement du seuil.

<div id="view-alert-history">
  #### Consulter l’historique des alertes
</div>

Vous pouvez consulter les avis d’alerte, d’avertissement et de retour à la normale dans [**Sécurité &gt; Security Center &gt; Historique des alertes**](https://manage.auth0.com/#/security/center/alert-history). Tous les avis sont aussi envoyés à vos destinations de notification configurées.
