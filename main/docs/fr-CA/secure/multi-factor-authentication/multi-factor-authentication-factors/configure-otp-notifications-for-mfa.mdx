description: Décrit comment configurer les notifications de mots de passe à usage unique temporisés (OTP) pour la MFA.
title: Configurer les notifications OTP pour la MFA
---

Auth0 prend en charge les mots de passe à usage unique (OTP) comme facteurs d’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip>. Pour que les utilisateurs puissent utiliser les OTP, vous devez d’abord [les activer comme facteur MFA](/docs/fr-CA/secure/multi-factor-authentication/enable-mfa) dans votre tenant Auth0.

Une fois activés pour votre application, les utilisateurs peuvent s’enrôler aux OTP lorsqu’ils accèdent à votre application. Pour ce faire, ils doivent installer une application d’authentification sur leur appareil, par exemple :

* Auth0 Guardian ([Google Play](https://play.google.com/store/apps/details?id=com.auth0.guardian) / [App Store](https://itunes.apple.com/us/app/auth0-guardian/id1093447833))
* Authy ([Google Play](https://play.google.com/store/apps/details?id=com.authy.authy) / [App Store](https://itunes.apple.com/us/app/authy/id494168017))
* Google Authenticator ([Google Play](https://play.google.com/store/apps/details?id=com.google.android.apps.authenticator2) / [App Store](https://itunes.apple.com/us/app/google-authenticator/id388497605))
* Microsoft Authenticator ([Google Play](https://play.google.com/store/apps/details?id=com.azure.authenticator) / [App Store](https://itunes.apple.com/us/app/microsoft-authenticator/id983156458))

<div id="user-otp-workflow">
  ## Flux de travail des OTP pour l’utilisateur
</div>

Lorsqu’un utilisateur tente de s’inscrire au moyen d’un <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>, il est invité à scanner un code QR pour s’enrôler aux OTP.

<Frame>![Exemple de configuration d’un code QR OTP d’Auth0 Guardian](/docs/images/cdy7uua7fh8z/2J6vWaonPbCvpuBUu9b8q1/b4ec4e66fc86021e5e31d000acaf18e2/QR_Code_-_EN_-_Small.png)</Frame>

Si l’appareil de l’utilisateur est détecté comme étant un appareil mobile, Universal Login saute l’écran du code QR et affiche plutôt directement un code d’enrôlement dans le prompt.

<Frame>![L’écran temporaire d’enrôlement au mot de passe à usage unique tel qu’affiché sur un appareil mobile. ](/docs/images/cdy7uua7fh8z/5QvQ9FLYnVZXINFAeLkIr9/04df9be004169ac06b8d9fee456dc599/OTP_prompt_-_English.png)</Frame>

Une fois l’utilisateur enrôlé aux OTP, il peut les utiliser pour se connecter à votre application. Pour ce faire, il récupère d’abord un code à usage unique dans son application d’authentification préférée.

<Frame>![Exemple de code OTP Google d’Auth0 Guardian](/docs/images/cdy7uua7fh8z/1vVY2NLQwsOwdHIPU4Qj3B/026255e1549c3ec5bdc790280590d210/google-auth-screenshot.png)</Frame>

Ensuite, il saisit le code à usage unique dans le prompt de connexion.

<Frame>![Exemple de prompt de connexion OTP d’Auth0 Guardian](/docs/images/cdy7uua7fh8z/4dntQCYqgx5QyLZKW7quxR/20cab4bcde8270914d2c2979810cd5f5/OTP_challenge_-_EN_-_Small.png)</Frame>
