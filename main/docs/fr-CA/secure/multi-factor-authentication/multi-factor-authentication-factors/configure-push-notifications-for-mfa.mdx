title: Configurer les notifications push pour la MFA
---

Les SDKs [Guardian](/docs/fr-CA/secure/multi-factor-authentication/auth0-guardian) pour [iOS](/docs/fr-CA/secure/multi-factor-authentication/auth0-guardian/guardian-for-ios-sdk) et [Android](/docs/fr-CA/secure/multi-factor-authentication/auth0-guardian/guardian-for-android-sdk) vous permettent d&#39;utiliser une application conçue sur mesure avec des services de notifications push propres à chaque fournisseur. Vous pouvez configurer les notifications push pour la plateforme AWS Simple Notification Service (SNS) ou utiliser les services Direct to Vendor pour ajouter directement dans Auth0 les identifiants Firebase Cloud Messaging (FCM) et Apple Push Notification (APN).

<div id="aws-sns-configuration">
  ## Configuration d’AWS SNS
</div>

Avant de configurer les notifications push, suivez les prérequis ci-dessous :

1. [Créez une Platform Application SNS](https://console.aws.amazon.com/sns/v3/home?region=us-east-1#/mobile/push-notifications/platform-applications) à l’aide de la console de gestion AWS et notez son ARN.
2. Créez un AWS Access Key ID autorisé à créer des points de terminaison de Platform Application. Guardian crée automatiquement un point de terminaison de Platform Application avec le jeton d’appareil approprié dans le cadre d’une inscription réussie.

<div id="configure-push-notifications-with-aws-sns">
  ### Configurer les notifications push avec AWS SNS
</div>

Pour recevoir des notifications push de Guardian, vous devez d’abord remplacer les paramètres SNS par défaut de Guardian. Vous pourrez ensuite configurer les notifications push pour Android et iOS.

1. Dans l’Auth0 Dashboard, accédez à [Sécurité &gt; Authentification multifacteur](https://manage.auth0.com/#/security/mfa) et sélectionnez **Push via Auth0 Guardian**. Si la méthode est inactive, sélectionnez la bascule en haut à droite pour l’activer.
2. Sélectionnez **Custom** pour afficher des paramètres supplémentaires.
3. Dans la section Service de notifications push, choisissez **Amazon SNS.**

<Frame>![Auth0 Dashboard > Sécurité > Authentification multifacteur > Guardian > Custom > Amazon SNS](/docs/images/cdy7uua7fh8z/6PKSyH99GJ2CalJoG38sfZ/fbe7229dcc6f7d303a582520001b3dc1/Amazon_SNS.png)</Frame>

1. Dans la section Configuration d’Amazon SNS, renseignez les champs suivants :

   | Champ                 | Description                         |
   | --------------------- | ----------------------------------- |
   | AWS Access Key ID     | Votre AWS Access Key ID.            |
   | AWS Secret Access Key | Votre AWS Secret Access Key.        |
   | AWS Region            | La région de votre application AWS. |
2. Sélectionnez **Save** pour enregistrer vos modifications.

<div id="configure-push-notifications-for-android-using-fcm">
  ## Configurer les notifications push pour Android avec FCM
</div>

<Tabs>
  <Tab title="Obtenir les identifiants FCM">
    Pour configurer les notifications push Android dans votre Auth0 Dashboard, commencez par récupérer les identifiants de votre serveur FCM. Pour plus de détails, consultez la [documentation Firebase](https://firebase.google.com/docs/admin/setup#initialize-sdk) de Google.

    1. Accédez à la console Firebase et sélectionnez le projet approprié.
    2. Ouvrez le menu **Settings** et choisissez **Service Accounts**.
    3. Sélectionnez **Generate New Private Key** pour télécharger votre fichier d’identifiants.

    Une fois vos identifiants récupérés, vous pouvez configurer les notifications push dans votre Auth0 Dashboard.
  </Tab>

  <Tab title="Configurer Auth0 Dashboard">
    Après avoir récupéré vos identifiants, vous pouvez configurer les notifications push Android dans votre Auth0 Dashboard.

    1. Accédez à [Sécurité &gt; Authentification multifacteur](https://manage.auth0.com/#/guardian) et sélectionnez **Push Notification using Auth0 Guardian**.
    2. Dans la section **Push Notification App**, sélectionnez **Custom** pour afficher les paramètres supplémentaires.
    3. Dans la section de configuration de l’application Android, configurez les éléments suivants :

       | Paramètre                     | Description                                                                                                                                                                 |
       | ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
       | Activer l’application Android | Permet à Auth0 d’envoyer des notifications push à votre application personnalisée. Si cette option n’est pas activée, les notifications push ne pourront pas être envoyées. |
       | URL du Google Play Store      | Fournissez un lien vers l’application personnalisée que les utilisateurs doivent télécharger pour configurer l’authentification multifacteur.                               |
       | Identifiants du serveur FCM   | Téléversez vos [identifiants du serveur FCM](https://firebase.google.com/docs/admin/setup#initialize-sdk).                                                                  |
    4. Sélectionnez **Enregistrer** pour enregistrer vos modifications.
  </Tab>
</Tabs>

<div id="configure-push-notifications-for-ios-using-apns">
  ## Configurer les notifications push pour iOS à l’aide d’APNs
</div>

<Tabs>
  <Tab title="Obtenir le certificat APN">
    Pour configurer les notifications push iOS dans votre Auth0 Dashboard, commencez par récupérer votre certificat APN.

    1. Générez un [nouveau certificat APN](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns#2947597) à partir de votre compte développeur Apple.

    2. Installez le certificat localement.

    3. Repérez le certificat **Apple Sandbox Push Services: {AppId}** dans Keychain Access.

    4. Exportez le certificat et la clé privée associée dans un fichier P12 sans mot de passe (nom par défaut : `Certificates.p12`).

    5. Pour générer un fichier P12 avec Triple DES plutôt que RC2, exécutez :

       ```bash lines
       openssl pkcs12 -in Certificates.p12 -legacy -nocerts -nodes -out pk.pem -passin pass:"" &&
       openssl pkcs12 -in Certificates.p12 -legacy -nokeys -out cert.crt -passin pass:"" &&
       openssl pkcs12 -export -inkey pk.pem -in cert.crt -descert -out Certificate_3des.p12 -passout pass:"" &&
       rm pk.pem cert.crt
       ```

    6. Supprimez `Certificates.p12`, puis utilisez `Certificate_3des.p12` comme certificat APN dans l’Auth0 Dashboard.

    Après avoir récupéré vos identifiants, vous pouvez configurer les notifications push dans votre Auth0 Dashboard.
  </Tab>

  <Tab title="Configurer l’Auth0 Dashboard">
    Après avoir récupéré vos identifiants, vous pouvez configurer les notifications push iOS à partir de votre Auth0 Dashboard.

    1. Accédez à [Sécurité &gt; Authentification multifacteur](https://manage.auth0.com/#/guardian) et sélectionnez **Push Notification using Auth0 Guardian**.
    2. Dans la section Push Notification App, sélectionnez **Custom** pour afficher des paramètres supplémentaires.
    3. Dans la section iOS App Configuration, configurez les éléments suivants :

       | Champ                              | Description                                                                                                                                                                                 |
       | ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
       | Activer l’application iOS          | Permet à Auth0 d’envoyer des notifications push à votre application personnalisée. Si cette option n’est pas activée, les notifications push ne peuvent pas être envoyées.                  |
       | URL de l’App Store d’Apple         | Fournissez un lien vers l’application personnalisée que les utilisateurs doivent télécharger pour configurer la MFA.                                                                        |
       | ID du bundle APNs                  | Fourni par la console de développement iOS.                                                                                                                                                 |
       | Certificat APNs                    | Téléversez votre [certificat P12](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns). |
       | Environnement de l’application iOS | Définissez-le selon la configuration de votre compte développeur Apple.                                                                                                                     |
    4. Sélectionnez **Save** pour enregistrer vos modifications.
  </Tab>
</Tabs>
