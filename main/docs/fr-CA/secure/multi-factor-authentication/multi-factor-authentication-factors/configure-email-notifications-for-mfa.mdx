title: Configurer les notifications par courriel pour la MFA
---

Utilisez le courriel comme facteur d’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> (MFA) afin d’offrir aux utilisateurs une façon d’effectuer la MFA lorsqu’ils n’ont pas accès à leur facteur principal (p. ex. ils n’ont pas leur appareil pour recevoir un SMS ou une notification push).

<Card title="La disponibilité varie selon l’implémentation de connexion">
  L’implémentation de connexion que vous utilisez détermine si cette fonctionnalité est offerte. Pour en savoir plus, consultez [Universal Login vs. Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).
</Card>

Le courriel n’est pas une véritable MFA, car il ne représente pas un facteur différent du mot de passe. Il ne représente pas **quelque chose que j’ai** ni **quelque chose que je suis**, mais simplement **quelque chose que je connais** (le mot de passe du courriel). Il est aussi moins robuste que d’autres facteurs, puisqu’il n’est sécurisé qu’au même niveau que le courriel lui-même (par exemple, s’il est chiffré de bout en bout).

Les utilisateurs n’ont pas besoin de s’enrôler explicitement pour utiliser la MFA par courriel. Ils peuvent l’utiliser lorsqu’ils ont une adresse courriel vérifiée. C’est le cas lorsqu’ils :

* Terminent le flux de vérification de l’adresse courriel, qui met à jour le champ `email_verified` à l’aide de l’<Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.
* Se connectent avec une connexion qui fournit des adresses courriel vérifiées (comme Google).

Vous ne pouvez activer le courriel comme facteur de MFA que si un autre facteur est déjà activé.

Une fois la MFA par courriel activée, les utilisateurs seront invités à effectuer la MFA avec un autre facteur activé. S’ils sélectionnent **Essayer une autre méthode**, puis choisissent **Courriel**, ils recevront un courriel contenant un code à 6 chiffres qu’ils devront saisir pour terminer le flux d’authentification.

<div id="configure-email-notifications">
  ## Configurer les notifications par courriel
</div>

Vous pouvez inscrire explicitement le courriel comme facteur MFA à l’aide de l’[MFA API](/docs/fr-CA/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa/enroll-and-challenge-email-authenticators). Si les utilisateurs ont une adresse courriel vérifiée et un ou plusieurs courriels inscrits explicitement, ils peuvent choisir l’adresse à utiliser pour effectuer la MFA lorsqu’ils se connectent au moyen de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>.

1. Accédez à [Dashboard &gt; Sécurité &gt; Multi-factor Auth](https://manage.auth0.com/#/multifactor-auth) et activez la bascule **courriel**.
   Vous ne pourrez l’activer que si un autre facteur est déjà activé.
2. Auth0 fournit un fournisseur de courriel de test, mais il ne permet d’envoyer qu’un nombre limité de courriels. Vous devriez donc configurer votre propre fournisseur de courriel. Pour en savoir plus, consultez [configurer les fournisseurs de courriel SMTP externes](/docs/fr-CA/customize/email/smtp-email-providers).
