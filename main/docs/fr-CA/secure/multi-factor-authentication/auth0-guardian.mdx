Auth0 Guardian est une application mobile pour les appareils iOS et Android qui permet aux utilisateurs d’effectuer une [authentification multifacteur (MFA)](/docs/fr-CA/secure/multi-factor-authentication) au moyen de notifications push ou de mots de passe temporaires à usage unique.

Auth0 Guardian peut envoyer des notifications push aux appareils inscrits des utilisateurs (généralement des téléphones mobiles ou des tablettes) ou générer des mots de passe à usage unique directement dans l’application. Les utilisateurs peuvent ensuite répondre rapidement à ces notifications push ou récupérer un mot de passe à usage unique pour terminer leur connexion.

Les utilisateurs peuvent télécharger l’application Auth0 Guardian depuis l’[Apple App Store](https://apps.apple.com/us/app/auth0-guardian/id1093447833) ou le [Google Play Store](https://play.google.com/store/apps/details?id=com.auth0.guardian). Vous pouvez aussi intégrer les fonctionnalités d’Auth0 Guardian à votre propre application personnalisée à l’aide du Guardian SDK.

<div id="push-notifications">
  ## Notifications push
</div>

Pour utiliser les notifications push, les utilisateurs doivent avoir installé sur leur appareil soit l’application Auth0 Guardian, soit une application personnalisée créée avec le Guardian SDK. Lorsqu’un utilisateur tente de s’authentifier, des notifications push sont envoyées à l’application installée. L’utilisateur doit répondre à la notification pour finaliser sa connexion, ce qui prouve à la fois qu’il connaît ses identifiants de connexion et qu’il possède l’appareil configuré pour la <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip>.

Les notifications push d’Auth0 peuvent être mises en place à l’aide d’AWS Simple Notification Service (SNS) ou d’un ou de plusieurs des services directs suivants pour configurer des intégrations propres à chaque fournisseur :

* Firebase Cloud Messaging (FCM)

  * Pour en savoir plus sur la modification du contenu des notifications push, consultez la [documentation de FCM](https://firebase.google.com/docs/cloud-messaging/concept-options).
* Apple Push Notification (APN)

  * Pour en savoir plus sur la modification du contenu des notifications push, consultez la [documentation d’Apple](https://developer.apple.com/documentation/usernotifications/modifying-content-in-newly-delivered-notifications).

<div id="enroll-in-push-notifications">
  ### S’inscrire aux notifications push
</div>

Lorsque vous utilisez Auth0 Guardian pour configurer les notifications push, les utilisateurs sont invités à télécharger l’application mobile la première fois qu’ils s’inscrivent ou se connectent à votre application. Si vous utilisez le Guardian SDK pour mettre en œuvre des notifications push dans une application personnalisée, les utilisateurs ne sont invités à télécharger aucune application pendant l’inscription.

Vous pouvez activer les notifications push à partir du <Tooltip tip="Auth0 Dashboard : produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> sous [Security &gt; Multi-factor Auth &gt; Push Notification using Auth0 Guardian](https://manage.auth0.com/#/security/mfa/guardian).

<Frame>![Auth0 Dashboard > Security > Multi-factor Auth > Push Notification using Auth0 Guardian](/docs/images/cdy7uua7fh8z/fwjSKFNhsvObzdPDiA79N/65850980f88029cb23eef5eaef4f9b20/Auth0_Dashboard_-_Push_Notification_screen.png)</Frame>

La première fois qu’un utilisateur s’inscrit ou se connecte à votre application, [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) affiche un code QR qu’il peut utiliser pour enregistrer l’application Auth0 Guardian ou une application personnalisée utilisant le Guardian SDK comme facteur d’authentification secondaire. L’utilisateur dispose de peu de temps pour balayer ce code avec l’application désignée afin de terminer son inscription.

Une fois l’utilisateur inscrit, il peut utiliser les notifications push comme facteur d’authentification. Chaque fois qu’il tente de se connecter à votre application, il reçoit une notification push sur son appareil par l’entremise de l’application Auth0 Guardian ou d’une application personnalisée utilisant le Guardian SDK. L’utilisateur doit approuver cette demande pour réussir à se connecter à votre application.

Pour savoir comment réinitialiser la MFA pour les utilisateurs qui ont perdu leurs appareils ou leurs codes de récupération, consultez [Reset Multi-Factor Authentication and Recovery Codes](/docs/fr-CA/secure/multi-factor-authentication/reset-user-mfa).

<div id="temporary-one-time-passwords">
  ## Mots de passe à usage unique temporaires
</div>

L’application Auth0 Guardian et le Guardian SDK prennent également en charge l’utilisation de mots de passe à usage unique temporaires (OTP) comme facteurs d’authentification secondaires. L’application et le SDK peuvent tous deux générer des OTP temporaires que les utilisateurs peuvent utiliser pour répondre aux défis MFA de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>.

Par défaut, l’application Auth0 Guardian affiche les applications auxquelles l’utilisateur est inscrit. Après avoir sélectionné une application, l’OTP correspondant s’affiche. Chaque OTP à 6 chiffres reste valide pendant 30 secondes. Les OTP qui approchent de leur expiration s’affichent en rouge. À l’expiration, l’application génère immédiatement un nouvel OTP. Pour éviter les échecs d’authentification, les utilisateurs ne devraient pas utiliser les OTP affichés en rouge

Dans les applications Guardian SDK personnalisées, l’apparence des OTP peut varier.

<div id="use-temporary-one-time-passwords">
  ### Utiliser des mots de passe à usage unique temporaires
</div>

Vous pouvez utiliser les OTP de deux façons principales :

* Comme solution de rechange aux notifications push si l’application Auth0 Guardian ou une autre application personnalisée ne reçoit pas de demande push.
* Comme défi MFA, si vous activez le [facteur One-Time Password](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors) dans votre tenant.

<AccordionGroup>
  <Accordion title="Méthode de rechange : utiliser les OTP comme solution de rechange aux notifications push">
    Si un utilisateur ne reçoit pas de notification push par l’entremise d’Auth0 Guardian ou d’une autre application personnalisée, il peut utiliser un OTP temporaire pour effectuer plutôt son défi MFA.

    **Exemple de parcours utilisateur** :

    1. Un utilisateur tente de se connecter à votre application. Il atteint l’écran MFA de Universal Login, mais ne reçoit pas de notification push.
    2. Dans l’écran MFA, l’utilisateur sélectionne le bouton **Manually Enter Code**. Il doit sélectionner cette option pour s’authentifier à l’aide d’OTP.

       <Frame>![Exemple d’écran MFA affichant le bouton Manually Enter Code](/docs/images/cdy7uua7fh8z/7reEOLI5CWxbpRzgZY7gxg/58da830927a33545d9a576974ec3ba6e/MFA_-_Push_-_English.png)</Frame>
    3. L’utilisateur ouvre Auth0 Guardian ou l’application personnalisée basée sur le Guardian SDK, puis sélectionne l’application à laquelle il tente d’accéder.
    4. L’utilisateur copie l’OTP affiché au bas de la page Account Detail.
    5. L’utilisateur saisit l’OTP dans l’écran MFA pour terminer le défi.
  </Accordion>

  <Accordion title="Défi MFA : utiliser les OTP comme défi MFA">
    Pour permettre aux utilisateurs de s’authentifier avec des OTP, vous devez activer **One-Time Password** comme [facteur MFA](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors) pour votre tenant. Une fois cette option activée, les utilisateurs peuvent répondre aux écrans MFA à l’aide d’OTP lorsqu’ils se connectent au moyen de Universal Login. Pour activer les facteurs MFA, accédez à [Auth0 Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).

    Les utilisateurs peuvent s’inscrire au facteur OTP lors de leur première inscription ou de leur première connexion à votre application. Selon le type d’appareil utilisé, [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) affiche l’un des éléments suivants :

    * **Pour les appareils non mobiles** : Universal Login affiche un code QR que l’utilisateur peut numériser pour poursuivre l’inscription.

    <Frame>![Exemple d’écran d’inscription OTP affichant le lien Trouble Scanning?](/docs/images/cdy7uua7fh8z/3kbyOBiLOpJiY8vI0kUqwg/b10f58a3b3ace74de613e28517726069/OTP_Challenge_3_-_English.png)</Frame>

    * **Pour les appareils mobiles** : Universal Login saute l’étape du code QR et affiche plutôt directement le code d’inscription.

    <Frame>![L’écran d’inscription au mot de passe à usage unique temporaire tel qu’affiché sur un appareil mobile. ](/docs/images/cdy7uua7fh8z/5QvQ9FLYnVZXINFAeLkIr9/04df9be004169ac06b8d9fee456dc599/OTP_prompt_-_English.png)</Frame>

    Dans un cas comme dans l’autre, les utilisateurs peuvent suivre les étapes affichées à l’écran pour enregistrer soit l’application Auth0 Guardian, soit une application personnalisée basée sur le Guardian SDK comme facteur d’authentification secondaire. En règle générale, ce processus consiste à demander à l’utilisateur d’ajouter votre application dans l’application appropriée et d’obtenir l’OTP correspondant pour cette application. L’utilisateur saisit ensuite l’OTP dans l’écran Universal Login pour terminer le processus d’inscription.

    La prochaine fois que l’utilisateur tentera de se connecter à votre application après avoir terminé l’inscription, il pourra s’authentifier en saisissant un OTP provenant de son application Auth0 Guardian ou de son application personnalisée basée sur le Guardian SDK pour répondre au défi.
  </Accordion>
</AccordionGroup>

<div id="user-security-settings">
  ### Paramètres de sécurité de l’utilisateur
</div>

Dans l’application Auth0 Guardian, les utilisateurs peuvent activer un code d’accès et la biométrie comme couches de sécurité supplémentaires sur iOS et Android. Si un utilisateur active une ou plusieurs de ces options, il doit franchir ces étapes de vérification avant de pouvoir répondre aux notifications push ou obtenir des codes à usage unique.

Pour activer ces options sur un appareil iOS ou Android, les utilisateurs peuvent suivre les étapes ci-dessous.

<Tabs>
  <Tab title="iOS">
    Pour activer les paramètres de sécurité de l’application Auth0 Guardian sur iOS :

    1. Dans l’application, sélectionnez l’**icône d’engrenage** pour ouvrir le menu Paramètres.
    2. Sélectionnez **Passcode** pour activer la protection par code d’accès.
    3. Définissez et confirmez un code d’accès à 6 chiffres.

    L’application est maintenant protégée par le code d’accès. L’utilisateur doit saisir ce code d’accès avant de pouvoir répondre aux notifications push ou obtenir des OTP.

    Après avoir activé la protection par code d’accès, l’utilisateur peut configurer la biométrie de l’appareil comme mesure de sécurité de rechange. Pour activer la biométrie de l’appareil :

    1. Dans l’application, sélectionnez l’**icône d’engrenage** pour ouvrir le menu Paramètres.
    2. Activez la biométrie de l’appareil en sélectionnant l’une des options offertes, comme **Face ID** ou **Touch ID**.
  </Tab>

  <Tab title="Android">
    Pour activer les paramètres de sécurité de l’application Auth0 Guardian sur Android :

    1. Dans l’application, sélectionnez le **menu à trois points** pour ouvrir le menu Paramètres.
    2. Activez le commutateur de l’option **Always Ask for Passphrase**.
    3. Saisissez la phrase secrète définie lors de la configuration de l’application Auth0 Guardian.

    L’application est maintenant protégée par la phrase secrète d’Auth0 Guardian. L’utilisateur doit saisir cette phrase secrète avant de pouvoir répondre aux notifications push ou obtenir des OTP.

    Après avoir activé la protection par phrase secrète, l’utilisateur peut configurer la biométrie de l’appareil comme mesure de sécurité de rechange. Pour activer la biométrie de l’appareil :

    1. Dans l’application, sélectionnez le **menu à trois points** pour ouvrir le menu Paramètres.
    2. Activez le commutateur de l’option **Use Biometrics to unlock**.
  </Tab>
</Tabs>

Dans la section Language du menu Settings, les utilisateurs peuvent sélectionner leur langue de préférence. Par défaut, l’application utilise la même langue que le système de l’appareil.

<div id="customization-options-for-protected-applications">
  ### Options de personnalisation des applications protégées
</div>

Pour permettre aux utilisateurs de distinguer plus facilement leurs applications protégées dans l’application Auth0 Guardian, chaque application peut être personnalisée avec un nom distinct, une couleur d’accentuation et une icône.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le nom par défaut attribué à votre application dans l’application Auth0 Guardian est déterminé par votre [nom de tenant](/docs/fr-CA/get-started/tenant-settings/find-your-tenant-name-or-tenant-id) ou par le Friendly Name défini dans vos [paramètres du tenant](/docs/fr-CA/get-started/tenant-settings). De même, le logo par défaut provient de l’URL du logo fournie dans les paramètres du tenant.

  Pour modifier les paramètres du tenant, accédez à [Auth0 Dashboard &gt; Settings &gt; General](https://manage.auth0.com/dashboard/#/tenant/general).
</Callout>

Pour personnaliser une application protégée :

1. Sélectionnez une application dans la liste.

   * **Conseil** : Vous pouvez aussi balayer l’application vers la gauche pour afficher les options Edit et Delete.
2. Dans la page Account Detail, choisissez **Edit**.
3. Mettez à jour le nom, la couleur et l’icône de l’application au besoin.
4. Sélectionnez **Save**.

<div id="guardian-sdks">
  ## Guardian SDK
</div>

Vous pouvez [installer le Guardian SDK](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/install-guardian-sdk) (offert pour [iOS](/docs/fr-CA/secure/multi-factor-authentication/auth0-guardian/guardian-for-ios-sdk) et [Android](/docs/fr-CA/secure/multi-factor-authentication/auth0-guardian/guardian-for-android-sdk)) pour créer votre propre application d’authentification multifacteur avec un contrôle total sur l’image de marque et l’apparence. Avec le Guardian SDK, vous pouvez créer vos propres applications mobiles personnalisées qui fonctionnent comme Guardian ou intégrer certaines fonctionnalités de Guardian, comme la réception de notifications push dans vos applications mobiles existantes. Un cas d’usage typique serait une application bancaire. Vous pouvez utiliser le Guardian SDK dans votre application mobile existante pour recevoir et confirmer des notifications push lorsqu’une personne effectue une transaction au guichet automatique.

<div id="use-actions-to-enable-multi-factor-with-auth0-guardian">
  ## Utilisez Actions pour activer l’authentification multifacteur avec Auth0 Guardian
</div>

Pour activer Auth0 Guardian dans une Action, passez `guardian` comme valeur du paramètre `provider` lorsque vous activez l’authentification multifacteur :

```js lines
exports.onExecutePostLogin = async (event, api) => {
  api.multifactor.enable('guardian', { allowRememberBrowser: false });
};
```

Pour obliger vos utilisateurs à se connecter avec Auth0 Guardian à chaque fois, créez l’Action avec `allowRememberBrowser: false`.

<div id="multi-factor-with-auth0-guardian-and-authorization-extension">
  ## Authentification multifacteur avec Auth0 Guardian et Authorization Extension
</div>

Ce modèle fournit un exemple et un point de départ pour déclencher l’authentification multifacteur avec Auth0 Guardian au moyen de notifications push lorsqu’une condition est satisfaite.

À la première connexion, l’utilisateur peut enregistrer l’appareil. Consultez Activer l’authentification multifacteur pour en savoir plus sur M

```js lines
exports.onExecutePostLogin = async (event, api) => {
const groups = event.user.app_metadata.authorization.groups;
const GROUPS_WITH_MFA = {
// Ajouter ici les groupes qui nécessitent l'AMF
// Exemple
admins: true
};

const needsMFA = !!groups.find(function (group) {
return GROUPS_WITH_MFA[group];
});

if (needsMFA) {
// facultatif, vrai par défaut. Mettre à false pour forcer l'authentification Guardian à chaque connexion.
// Voir https://auth0.com/docs/secure/multi-factor-authentication/customize-mfa#change-frequency-of-mfa-prompts pour plus de détails
api.multifactor.enable('guardian', { allowRememberBrowser: false });
}

};
```
