<div id="use-the-dashboard">
  ## Utiliser le Dashboard
</div>

Activez **WebAuthn with Device Biometrics** en accédant à [Dashboard &gt; Sécurité &gt; Multifactor Auth](https://manage.auth0.com/#/security/mfa). Vous devrez activer un facteur supplémentaire, car il ne peut pas être le seul facteur activé.

<div id="configure-relying-party">
  ## Configurer le Relying Party
</div>

WebAuthn rend l’hameçonnage impossible en liant les informations d’identification à l’origine du navigateur. Les utilisateurs ne peuvent pas non plus utiliser WebAuthn pour un site auprès duquel ils ne se sont pas inscrits.

Le fait de lier les informations d’identification à l’origine signifie que si vous configurez un <Tooltip tip="Custom Domain : domaine tiers portant un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=custom+domain">domaine personnalisé</Tooltip> ou que vous le modifiez, les utilisateurs inscrits avant ce changement ne pourront pas s’authentifier.

WebAuthn définit un [attribut d’ID de Relying Party](https://www.w3.org/TR/webauthn-2/#relying-party-identifier), qui vous permet de préciser le domaine utilisé pour authentifier les utilisateurs. Vous pouvez le définir comme n’importe quel suffixe de domaine enregistrable de l’origine du navigateur. Par exemple, si le domaine personnalisé est `login.example.com`, vous pouvez configurer l’ID du <Tooltip tip="Relying Party : entité (comme un service ou une application) qui dépend d’un fournisseur d’identité tiers pour authentifier un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Relying+Party">Relying Party</Tooltip> à `example.com`. Cela permet aux utilisateurs de s’authentifier sur n’importe quel domaine `example.com` avec leurs informations d’identification WebAuthn.

Auth0 vous permet de préciser l’ID du Relying Party seulement si vous avez configuré un domaine personnalisé. Si le domaine personnalisé change, vous devez mettre à jour l’ID du Relying Party.

<div id="device-recognition">
  ## Reconnaissance de l’appareil
</div>

Lorsque vous activez WebAuthn with Device Biometrics, Auth0 tentera d’enrôler progressivement tous les appareils d’un utilisateur final qui sont compatibles avec WebAuthn. Les navigateurs dans lesquels Javascript est désactivé ou qui ne prennent pas en charge un authentificateur de plateforme WebAuthn n’offriront pas l’option de s’enrôler ou de s’authentifier à l’aide de la biométrie de l’appareil.

Comme il n’existe aucun moyen déterministe de savoir si un appareil donné a été enrôlé ou non sans soumettre l’utilisateur à une vérification WebAuthn, Auth0 se fie à l’agent utilisateur pour décider quoi faire. Le comportement dépend du système d’exploitation.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Dans le scénario le plus courant, où les utilisateurs emploient toujours le même navigateur et utilisent un appareil mobile ainsi qu’un ordinateur portable ou de bureau, les nuances de comportement décrites ci-dessous n’auront pas d’incidence sur l’expérience utilisateur.
</Callout>

<div id="windows-and-ios-145">
  ### Windows et iOS 14.5+
</div>

Sous Windows et iOS 14.5+, l’authentificateur de plateforme WebAuthn est enregistré au **niveau du système d’exploitation.** Les utilisateurs peuvent s’inscrire avec un navigateur, puis se connecter avec n’importe lequel. Si Auth0 détecte qu’un appareil est déjà inscrit, les utilisateurs auront l’option de s’authentifier avec Face ID / Touch ID / Windows Hello. S’ils se sont inscrits avec ce même appareil, ils pourront s’authentifier. Sinon, l’authentification échouera et ils devront utiliser une autre méthode d’authentification.

<div id="macos">
  ### macOS
</div>

Sur Mac, l’authentificateur de plateforme WebAuthn est enregistré au niveau du système du **navigateur**. Les utilisateurs devront s’enrôler avec WebAuthn dans chaque navigateur qu’ils utilisent. Si Auth0 détecte que l’utilisateur s’est enrôlé dans Chrome sur un Mac, il aura l’option de s’authentifier avec Touch ID lorsqu’il se connectera à Chrome sur un Mac. S’il s’est enrôlé sur le même Mac, il pourra s’authentifier. Sinon, la tentative échouera et il devra utiliser une autre méthode d’authentification. S’il essaie de s’enrôler dans Safari sur le même Mac, on lui demandera d’effectuer l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui ajoute un facteur au nom d’utilisateur et mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> avec l’autre méthode d’authentification, puis **il sera invité à s’enrôler** avec Touch ID.

<div id="android">
  ### Android
</div>

Sur Android, **seul Chrome** prend en charge les authentificateurs de plateforme WebAuthn. Si Auth0 détecte que les utilisateurs ont un appareil Android enregistré, l’option de s’authentifier avec la reconnaissance faciale ou d’empreintes d’Android leur sera proposée. S’ils se sont enregistrés avec ce même appareil Android, ils pourront s’authentifier. Sinon, l’authentification échouera et ils devront utiliser une autre méthode d’authentification.

<div id="device-support">
  ## Prise en charge des appareils
</div>

L’utilisateur doit avoir activé une autre méthode MFA avant d’utiliser la biométrie de l’appareil.

Pour utiliser la biométrie de l’appareil, le navigateur doit avoir JavaScript activé et prendre en charge les authentificateurs de plateforme WebAuthn. Si ces conditions ne sont pas remplies, Auth0 n’offrira pas l’option d’inscrire l’appareil ni de s’authentifier avec celui-ci. Auth0 demandera alors à l’utilisateur d’utiliser un autre facteur.

Les versions les plus récentes des navigateurs et des systèmes d’exploitation les plus courants prennent en charge WebAuthn avec Security Keys. Pour en savoir plus, consultez la [section sur la compatibilité des navigateurs sur webauthn.me](https://webauthn.me/browser-support).

<div id="limitations">
  ## Limitations
</div>

* Il n’existe aucun moyen d’enregistrer les données biométriques d’un appareil avec WebAuthn en dehors de l’écran Progressive Enrollment.
* Lors de l’utilisation de l’[API MFA](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api), vous pouvez afficher et supprimer les inscriptions WebAuthn, mais vous ne pouvez pas les enregistrer.
* Les utilisateurs ne peuvent enregistrer qu’un seul appareil par type avec WebAuthn with Device Biometrics (un téléphone, une tablette, un portable/ordinateur de bureau). Si un utilisateur veut enregistrer un autre appareil du même type, le premier appareil doit être désinscrit.
