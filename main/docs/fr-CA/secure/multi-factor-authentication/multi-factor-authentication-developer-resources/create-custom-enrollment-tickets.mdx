Si vous souhaitez personnaliser et gérer les [inscriptions de vos utilisateurs à l’authentification multifacteur (MFA)](/docs/fr-CA/secure/multi-factor-authentication), vous pouvez utiliser les Custom Enrollment Tickets. Les Custom Enrollment Tickets d’Auth0 génèrent un lien à usage unique permettant à vos utilisateurs de s’inscrire à l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification des utilisateurs qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> avec le ou les [facteurs](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors) que vous avez précisés.

<div id="use-cases">
  ### Cas d’utilisation
</div>

Utilisez Custom Enrollment Tickets pour :

* L’intégration de nouveaux utilisateurs
* La configuration des paramètres utilisateur
* Définir les facteurs parmi lesquels vous voulez que les clients choisissent lors de l’inscription
* Permettre aux utilisateurs d’inscrire plus d’un facteur

<div id="configure-custom-enrollment-tickets">
  ### Configurer les Custom Enrollment Tickets
</div>

Effectuez un appel POST au point de terminaison [`/guardian/post_ticket`](https://auth0.com/docs/api/management/v2#!/Guardian/post_ticket) de <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

| Paramètre                    | Obligatoire | Description                                                                                                                                                                                |
| ---------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `allow_multiple_enrollments` | Facultatif  | Permet à un utilisateur déjà inscrit d’ajouter des facteurs supplémentaires, à l’exclusion du courriel.                                                                                    |
| `factor`                     | Facultatif  | Indique le facteur avec lequel l’utilisateur doit s’inscrire, à l’exclusion du courriel. Utilisé avec `allow_multiple_enrollments`, il peut préciser plus d’un facteur pour l’inscription. |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Ces paramètres ne peuvent être utilisés qu’avec [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) et ne peuvent pas être utilisés avec Classic Login ni avec des pages MFA personnalisées.
</Callout>

Management API renvoie un ticket d’inscription contenant un `ticket_id` et un `ticket_url`. Le `ticket_id` est réservé à un usage interne. Faites parvenir le `ticket_url` à l’utilisateur à l’aide du paramètre `send_email` pour lancer le processus d’inscription. Les tickets expirent après 5 jours et sont à usage unique. Un utilisateur qui reçoit un ticket ne pourra s’inscrire qu’une seule fois.

Exemple de réponse :

```json lines
{
  "ticket_id": "gten_8b2f5e90d2c848dc9230d34f",
  "ticket_url": "https://guardianadnrdoifcmtest0.eu.auth0.com/guardian/enroll?ticket=gten_8b2f5e90d2c848dc9230d34f"
}
```

Si vous activez [Multiple Custom Domains](/docs/fr-CA/customize/custom-domains/multiple-custom-domains), vous devez inclure l’en-tête HTTP `auth0-custom-domain`. Pour en savoir plus, consultez [Multiple Custom Domains](/docs/fr-CA/customize/custom-domains/multiple-custom-domains#customize-email-handling-using-the-management-api).

<div id="custom-enrollment-tickets-with-classic-login">
  ### Custom Enrollment Tickets avec Classic Login
</div>

Si vous utilisez l’expérience [Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) et que vous devez personnaliser l’apparence de la page lorsque l’utilisateur accède à `ticket_url`, vous pouvez modifier la page MFA.

1. Accédez à [Dashboard &gt; Branding &gt; Universal Login &gt; options Advanced &gt; Multi-factor Authentication.](https://manage.auth0.com/#/mfa_page)
2. Si **Customize MFA Page** n’est pas sélectionné, activez cette option.
3. Mettez à jour la variable `ticket`.
   Exemple :

```liquid lines
{% if ticket %}
<h4 class="message">Welcome, {{ userData.email }}, enroll your device below</h4>
{% else %}
<h4 class="message">Welcome back, {{ userData.email }}, authenticate below</h4>
{% endif %}
```
