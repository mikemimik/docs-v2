title: Personnaliser l’authentification multifactorielle pour Classic Login
---

Vous pouvez activer l’<Tooltip tip="Authentification multifactorielle (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifactorielle</Tooltip> dans la section [Dashboard &gt; Sécurité &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Vous pouvez personnaliser les pages MFA à l’aide des options de base de l’image de marque dans l’<Tooltip tip="Authentification multifactorielle (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Pour aller plus loin dans la personnalisation, vous pouvez fournir votre propre code HTML pour la page MFA. Lorsque vous utilisez votre propre code HTML, Classic Login utilise le MFA Widget d’Auth0 avec les limitations suivantes :

* Il ne prend pas en charge l’authentification multifactorielle par courriel.
* Si les utilisateurs ont activé plus d’un facteur, ils ne peuvent pas choisir celui à utiliser. Le widget MFA leur demandera de se connecter avec le facteur le plus sûr.
* Il n’utilise pas les fonctionnalités d’internationalisation de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le Authorization Server d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>. Pour en savoir plus, consultez [Universal Login Internationalization](/docs/fr-CA/customize/internationalization-and-localization/universal-login-internationalization).

<div id="customize-html-for-the-mfa-page">
  ## Personnaliser le HTML de la page MFA
</div>

Pour personnaliser la page MFA, accédez à l’onglet [Dashboard &gt; Branding &gt; Universal Login &gt; Multi-factor Authentication](https://manage.auth0.com/#/mfa_page) et activez la bascule **Customize MFA Page**. Vous pouvez ensuite utiliser l’éditeur de texte pour modifier votre HTML, mettre en forme votre page à l’aide de CSS et modifier le Javascript utilisé pour récupérer des variables personnalisées. Une fois vos modifications terminées, assurez-vous de cliquer sur **Save Changes**.

Les pages MFA fonctionnent sans personnalisation, et Auth0 mettra à jour les scripts inclus au besoin. Toutefois, une fois la personnalisation activée, c’est à vous d’effectuer les mises à jour et d’assurer la maintenance du script (y compris la modification des numéros de version, comme celui du MFA Widget), puisque Auth0 ne peut plus le mettre à jour automatiquement.

Pour en savoir plus sur la façon de remplacer le texte dans plusieurs parties du processus MFA pour Classic Login, consultez [MFA Theme Language Dictionary](/docs/fr-CA/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary). Pour en savoir plus sur les options de thème du MFA Widget, consultez [MFA Widget Theme Options](/docs/fr-CA/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options).

<div id="reset-page">
  ## Réinitialiser la page
</div>

Si vous souhaitez rétablir une version antérieure du design, vous avez deux options :

* Revenir au dernier modèle enregistré en cliquant sur **Réinitialiser à la dernière version** ;
* Revenir au modèle par défaut fourni par Auth0 en cliquant sur **Réinitialiser aux valeurs par défaut**.

<div id="render-enrollment-invitations">
  ## Afficher les invitations d&#39;inscription
</div>

La page peut s’afficher dans deux scénarios différents. Si un utilisateur a été dirigé vers cette page expressément pour l’inscription (par exemple, à partir d’un courriel contenant un lien d’inscription), la propriété `ticket` sera disponible. Sinon, la propriété `requestToken` sera disponible.

<div id="html-and-liquid-syntax">
  ## Syntaxe HTML et Liquid
</div>

La page hébergée utilise la syntaxe Liquid pour les modèles. Pour en savoir plus, consultez [Liquid for Designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) sur GitHub. Les paramètres suivants sont offerts pour vous aider à générer votre page :

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

La plupart des paramètres utilisés dans le MFA Widget doivent être transmis à Guardian, comme indiqué dans le modèle par défaut fourni dans la section de personnalisation. Si vous avez besoin d’un niveau de personnalisation plus avancé, vous pouvez utiliser [auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example).

Pour en savoir plus, consultez [Personnaliser les pages utilisateur MFA](/docs/fr-CA/secure/multi-factor-authentication/customize-mfa).
