Vous pouvez activer l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> (MFA) pour votre tenant dans l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

Lorsque vous activez l’authentification multifacteur, sélectionnez les facteurs (comme les notifications push ou les messages téléphoniques) que vous souhaitez rendre accessibles aux utilisateurs, puis définissez une politique pour déterminer quand vous exigez qu’ils s’authentifient.

<div id="select-factors">
  ## Sélectionner des facteurs
</div>

Auth0 prend en charge divers facteurs que vous pouvez activer afin que vos utilisateurs puissent terminer le processus d’authentification.

<div id="independent-factors">
  ### Facteurs indépendants
</div>

Vous devez activer et configurer au moins un facteur indépendant pour exiger l’authentification multifacteur (MFA) pour les utilisateurs de votre tenant. Vous pouvez activer simultanément plusieurs facteurs indépendants, selon vos besoins.

Les facteurs indépendants disponibles sont :

* [WebAuthn with FIDO Security Keys](/docs/fr-CA/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)
* [One-time Password (OTP)](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)
* [notification push using Auth0 Guardian](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa)
* [Phone Message](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)
* [Cisco Duo Security](/docs/fr-CA/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="dependent-factors">
  ### Facteurs dépendants
</div>

Vous pouvez activer un ou plusieurs facteurs dépendants dès qu’au moins un facteur indépendant a déjà été activé et configuré pour les utilisateurs de votre tenant.

Les facteurs dépendants disponibles sont :

* [WebAuthn avec biométrie FIDO](/docs/fr-CA/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)
* [Email](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)
* [Code de récupération](/docs/fr-CA/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="enable-mfa-in-the-auth0-dashboard">
  ## Activer MFA dans le Auth0 Dashboard
</div>

1. Accédez à [Dashboard &gt; Sécurité &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
2. Dans la section **Factors**, activez et configurez les facteurs que vous voulez rendre accessibles aux utilisateurs.
3. Dans la section **Define policies**, sélectionnez une politique pour déterminer à quel moment les utilisateurs seront invités à utiliser MFA.

   * **Jamais** : MFA n’est requise pour aucune connexion.
   * **Utiliser Adaptive MFA** : MFA est requise selon l’évaluation du risque effectuée par Auth0. Pour en savoir plus, consultez [Activer Adaptive MFA](/docs/fr-CA/secure/multi-factor-authentication/adaptive-mfa).
   * **Toujours** : MFA est requise pour toutes les connexions.
4. Si vous choisissez la politique **Jamais** ou **Toujours**, la section **MFA Risk Assessors** s’affiche. Par défaut, le paramètre **Enable Adaptive MFA Risk Assessment** est désactivé. Activez la bascule pour évaluer et consigner le risque de toutes les transactions de connexion dans les logs de votre tenant.
5. Dans la section **Paramètres supplémentaires**, configurez les options suivantes au besoin :

   * **Afficher les options de Multi-factor Authentication :** Activez cette bascule pour permettre aux utilisateurs de sélectionner des facteurs d’authentification lors de l’inscription. Lorsqu’elle est désactivée, Auth0 présente automatiquement aux utilisateurs les facteurs d’authentification les plus sécuritaires au moment de l’inscription.

     <Frame>![Vue de l’utilisateur final des options de Multi-factor Authentication](/docs/images/cdy7uua7fh8z/3Mo3EXUk3ZFK4hZKck0yOO/37268f33f70d1e09032f0db317ac11d3/MFA_Options_-_English.png)</Frame>
   * **Personnaliser les facteurs MFA à l’aide d’Actions** : Activez cette bascule pour personnaliser vos flux MFA à l’aide des [Actions](/docs/fr-CA/customize/actions) `post-login`. Cela vous permet de soumettre les utilisateurs à un facteur précis ou à une séquence de facteurs, ainsi que d’utiliser des informations contextuelles pour créer des expériences plus personnalisées. Pour en savoir plus, consultez [Personnaliser la sélection MFA pour Universal Login](/docs/fr-CA/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
6. Cliquez sur **Enregistrer**.

<div id="customize-mfa-flow">
  ## Personnaliser le flux MFA
</div>

Vous pouvez utiliser Auth0 Actions pour personnaliser le flux MFA des utilisateurs de votre tenant. Par exemple, vous pouvez créer une Action pour exiger la MFA dans certaines circonstances ou pour imposer l’utilisation d’un facteur précis.

Pour en savoir plus, consultez [Customize Adaptive MFA](/docs/fr-CA/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).
