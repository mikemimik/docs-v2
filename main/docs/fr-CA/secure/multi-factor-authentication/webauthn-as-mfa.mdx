title: WebAuthn comme authentification multifacteur
---

Lorsque les utilisateurs s’authentifient avec WebAuthn, ils utilisent quelque chose **qu’ils possèdent** comme facteur d’authentification : une clé de sécurité ou un appareil.

Les clés de sécurité et la biométrie de l’appareil prennent toutes deux en charge la **vérification de l’utilisateur**, qui exige que les utilisateurs fournissent quelque chose **qu’ils connaissent** (un NIP ou un code d’accès) et quelque chose **qu’ils sont** (comme des caractéristiques biométriques).

Lorsqu’on utilise la biométrie de l’appareil, la vérification de l’utilisateur est toujours effectuée. Pour effectuer la vérification avec des clés de sécurité, vous [devez configurer Auth0](/docs/fr-CA/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa) pour exiger un NIP. Les utilisateurs devront alors saisir un NIP, qui est stocké uniquement dans la clé de sécurité, pour terminer l’authentification. Lorsque la vérification de l’utilisateur est effectuée, les utilisateurs peuvent alors se connecter avec WebAuthn comme seule méthode d’authentification pour bénéficier de l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip>.

En utilisant WebAuthn pour l’authentification avec la vérification de l’utilisateur, vous remplacez non seulement le mot de passe par une méthode beaucoup plus simple à utiliser, mais vous éliminez aussi la nécessité d’une étape d’authentification supplémentaire lorsqu’une authentification multifacteur est requise.
