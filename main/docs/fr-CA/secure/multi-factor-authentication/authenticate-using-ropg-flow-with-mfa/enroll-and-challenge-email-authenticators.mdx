Auth0 fournit un parcours intégré d’inscription et d’authentification <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> avec [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login). Toutefois, si vous souhaitez créer votre propre interface utilisateur, vous pouvez utiliser la [MFA API](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api) pour le faire.

Lorsque le courriel est activé comme facteur, tous les utilisateurs ayant des adresses courriel vérifiées pourront s’en servir pour effectuer la MFA.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de pouvoir utiliser les API MFA, vous devez activer le type d’octroi MFA pour votre application. Accédez à [Auth0 Dashboard &gt; Applications &gt; Advanced Settings &gt; Grant Types](https://manage.auth0.com/#/applications) et sélectionnez **MFA**.

* [Configurer Email](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa) en tant que facteur dans le Dashboard ou à l’aide de la [Management API](https://auth0.com/docs/api/management/v2#!/Guardian/put_factors_by_name).

<div id="enroll-with-email">
  ## Inscription avec Email
</div>

Pour permettre aux utilisateurs d’ajouter des adresses courriel, en plus de leur adresse courriel vérifiée associée à leur identité principale, vous devez suivre les étapes suivantes.

<div id="get-mfa-token">
  ### Obtenir un jeton MFA
</div>

Selon le moment où vous lancez l’inscription, vous pouvez obtenir un <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> pour utiliser l’API MFA de différentes façons :

* Si vous effectuez l’inscription pendant l’authentification, consultez [S’authentifier avec le Resource Owner Password Grant et la MFA](/docs/fr-CA/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa).
* Si vous souhaitez permettre à l’utilisateur d’inscrire un facteur à tout moment, consultez [Gérer les inscriptions des facteurs MFA](/docs/fr-CA/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authenticator-factors-mfa-api).

<div id="enroll-authenticator">
  ### Inscrire l’authentificateur
</div>

Envoyez une requête `POST` au endpoint MFA Associate pour inscrire l’authentificateur de l’utilisateur. Le bearer token requis par ce endpoint est le MFA token obtenu à l’étape précédente.

Utilisez les paramètres suivants :

| Paramètre              | Valeur                                                    |
| ---------------------- | --------------------------------------------------------- |
| `authentication_types` | `[oob]`                                                   |
| `oob_channels`         | `[email]`                                                 |
| `email`                | `email@address.com`, l’adresse courriel de l’utilisateur. |

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/mfa/associate' \
    --header 'authorization: Bearer MFA_TOKEN' \
    --header 'content-type: application/json' \
    --data '{ "authenticator_types": ["oob"], "oob_channels": ["email"], "email" : "email@address.com" }'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/mfa/associate");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer MFA_TOKEN");
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{ "authenticator_types": ["oob"], "oob_channels": ["email"], "email" : "email@address.com" }", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  }

Si la requête réussit, vous recevez une réponse comme celle-ci :

```json lines
{
    "authenticator_type": "oob",
    "binding_method": "prompt",
    "oob_code" : "Fe26..nWE",
    "oob_channel": "email",
    "recovery_codes": [ "N3BGPZZWJ85JLCNPZBDW6QXC" ]
  }
```

Si vous obtenez l’erreur `User is already enrolled`, c’est que l’utilisateur a déjà un facteur MFA configuré. Avant d’associer un autre facteur à l’utilisateur, vous devez d’abord lui envoyer un défi à l’aide du facteur existant.

S’il s’agit de la première fois que l’utilisateur associe un authentificateur, vous remarquerez que la réponse comprend `recovery_codes`. Les codes de récupération servent à accéder au compte de l’utilisateur s’il perd l’accès au compte ou à l’appareil utilisé pour son authentification à deuxième facteur. Il s’agit de codes à usage unique, et de nouveaux codes sont générés au besoin.

<div id="confirm-email-enrollment">
  ### Confirmer l’inscription par courriel
</div>

L’utilisateur devrait recevoir un courriel contenant le code à six chiffres qu’il devra saisir dans l’application.

Pour terminer l’inscription, faites une requête `POST` vers le point de terminaison [**`/oath/token`**](https://auth0.com/docs/api/authentication#get-token). Incluez le `oob_code` renvoyé dans la réponse précédente ainsi que le `binding_code` avec la valeur indiquée dans le courriel.

À ce stade, l’authentificateur est entièrement associé et prêt à être utilisé, et vous disposez des jetons d’authentification de l’utilisateur.

Vous pouvez vérifier à tout moment si un authentificateur a été confirmé en appelant le point de terminaison MFA Authenticators. Si l’authentificateur est confirmé, la valeur renvoyée pour `active` est `true`.

Au besoin, vous pouvez personnaliser les courriels que les utilisateurs reçoivent. Consultez [Personnaliser les modèles de courriel](/docs/fr-CA/customize/email/email-templates) pour en savoir plus.

<div id="customize-mfa">
  ## Personnaliser la MFA
</div>

<Warning>
  La MFA personnalisable avec les flux Resource Owner Password Grant, Embedded ou Refresh Token est en Early Access. En utilisant cette fonctionnalité, vous acceptez les conditions applicables de l’essai gratuit dans le [Master Subscription Agreement](https://www.okta.com/legal/) d’Okta. Pour en savoir plus sur les étapes de lancement d’Auth0, consultez [Product Release Stages](/docs/fr-CA/troubleshoot/product-lifecycle/product-release-stages). Pour participer à l’accès anticipé, communiquez avec [Auth0 Support](https://support.auth0.com/).
</Warning>

La MFA personnalisable permet aux utilisateurs d’enregistrer les facteurs pris en charge par votre application de leur choix et de les utiliser pour les défis.

Lors de l’authentification au point de terminaison `oauth/token`, la réponse renvoie l’erreur `mfa_required`, qui comprend le `mfa_token` à utiliser avec la MFA API ainsi que le paramètre `mfa_requirements`, contenant une liste d’authentificateurs :

```json
{
  "error": "mfa_required",
  "error_description": "Multifactor authentication required",
  "mfa_token": "Fe26...Ha",
  "mfa_requirements": {
    "challenge": [
      { "type": "otp" },
      { "type": "push-notification" },
      { "type": "phone" },
      { "type": "recovery-code" }
      { "type": "email"} //fonctionne uniquement avec challenge
    ]
  }
}
```

Utilisez le `mfa_token` pour appeler le point de terminaison [`mfa/authenticator`](/docs/fr-CA/api/authentication/muti-factor-authentication/list-authenticators) afin d’obtenir la liste de tous les facteurs auxquels l’utilisateur est déjà inscrit et de repérer le même type que celui pris en charge par votre application.  Vous devez aussi obtenir l’`authenticator_type` correspondant pour lancer les défis :

```json
[
  {
    "type": "recovery-code",
    "id": "recovery-code|dev_qpOkGUOxBpw6R16t",
    "authenticator_type": "recovery-code",
    "active": true
  },
  {
    "type": "otp",
    "id": "totp|dev_6NWz8awwC8brh2dN",
    "authenticator_type": "otp",
    "active": true
  }
]
```

Forcez l’étape de défi MFA en appelant le endpoint [`request/mfa/challenge`](/docs/fr-CA/api/authentication/muti-factor-authentication/request-mfa-challenge).

Personnalisez davantage votre processus MFA avec Auth0 Actions. Pour en savoir plus, consultez [Actions Triggers : post-challenge - objet API](/docs/fr-CA/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object).
