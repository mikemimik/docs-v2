---
description: Découvrez comment réinitialiser l’authentification multifacteur d’un utilisateur s’il perd son appareil mobile et n’a pas de code de récupération.
title: Réinitialiser l’authentification multifacteur et les codes de récupération d’un utilisateur
---

Si un utilisateur a perdu son appareil mobile, il peut utiliser son code de récupération pour se connecter. S’il n’a pas de code de récupération, il devra demander à l’administrateur de son tenant de réinitialiser son <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> (MFA). Cette action équivaut à retirer ou à supprimer l’inscription de l’utilisateur à la MFA. Les paramètres MFA associés à l’utilisateur seront supprimés, ce qui lui permettra de configurer la MFA comme s’il était un nouvel utilisateur lors de sa prochaine tentative de connexion. Pour réinitialiser la MFA d’un administrateur plutôt que celle d’un utilisateur final, veuillez contacter [Auth0 Support](/docs/fr-CA/troubleshoot/customer-support).

<div id="use-the-dashboard">
  ## Utiliser le Dashboard
</div>

1. Accédez à [Dashboard &gt; Users Management &gt; Users](https://manage.auth0.com/#/users).
2. Cliquez sur l’utilisateur dont vous souhaitez réinitialiser le MFA.
3. Cliquez sur le bouton **Actions** en haut à droite de l’écran.
4. Sélectionnez **Reset Multi-factor** dans le menu déroulant. Les Admins verront aussi un lien **Reset MFA** au bas de l’onglet **Multi-Factor Authentication** de la page **User Details** si l’utilisateur a déjà activé le MFA. Ces deux méthodes fonctionnent de la même manière. Une fenêtre contextuelle s’affichera pour confirmer votre choix.
5. Cliquez sur **Yes, reset it** pour réinitialiser le MFA de l’utilisateur.

La prochaine fois que l’utilisateur se connectera, il devra configurer son MFA comme un nouvel utilisateur.

<div id="use-the-management-api">
  ## Utiliser l’API Management
</div>

En tant qu’administrateur, vous pouvez aussi utiliser la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> pour supprimer l’inscription d’un utilisateur à la MFA à l’aide de `DELETE` `/api/v2/users/{id}/authentication-methods`. Si l’utilisateur a plus d’une inscription à la MFA, vous devrez répéter l’opération pour chaque inscription.

<div id="recovery-codes">
  ## Codes de récupération
</div>

Avec la plupart des facteurs MFA, l’utilisateur final reçoit un code de récupération au moment de l’inscription, qu’il doit noter et conserver en lieu sûr. S’il n’a pas son appareil avec lui ou s’il est temporairement incapable d’utiliser son processus MFA habituel, l’utilisateur peut se connecter en entrant ce code après son nom d’utilisateur et son mot de passe. Si un code de récupération est utilisé, un nouveau code de récupération sera alors fourni.

Si un utilisateur désinstalle Guardian puis le réinstalle plus tard, il pourrait être invité à entrer son code de récupération. Si le code de récupération a été perdu, l’utilisateur peut réinstaller l’application en désactivant la restauration automatique de sa sauvegarde Guardian. Pour ce faire, l’utilisateur devra désinstaller Guardian, désactiver temporairement la restauration automatique des sauvegardes dans les paramètres de son appareil (la marche à suivre varie selon l’appareil), puis réinstaller l’application. Il devra ensuite ajouter son ou ses comptes MFA à l’application comme s’il s’agissait d’une première configuration. Si les sauvegardes automatiques ou la restauration automatique ne sont pas activées sur l’appareil de l’utilisateur, la réinstallation de l’application ne demandera pas de code de récupération et l’utilisateur devra ajouter son ou ses comptes MFA comme lors d’une première configuration.
