---
description: Découvrez comment réinitialiser l’authentification multifacteur (MFA) d’un utilisateur s’il perd son appareil mobile et n’a pas de code de récupération.
title: Réinitialiser l’authentification multifacteur et les codes de récupération d’un utilisateur
---

Si un utilisateur a perdu son appareil mobile, il peut utiliser son code de récupération pour se connecter. S’il n’a pas de code de récupération, il devra demander à son administrateur de tenant de réinitialiser son <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> (MFA). Cette action revient à supprimer l’inscription de l’utilisateur à l’authentification multifacteur. Les paramètres d’authentification multifacteur associés à l’utilisateur seront supprimés, ce qui lui permettra de configurer l’authentification multifacteur comme s’il s’agissait d’un nouvel utilisateur lors de sa prochaine tentative de connexion. Pour réinitialiser l’authentification multifacteur d’un administrateur plutôt que celle d’un utilisateur final, veuillez communiquer avec le [soutien Auth0](/docs/fr-CA/troubleshoot/customer-support).

<div id="use-the-dashboard">
  ## Utiliser le Dashboard
</div>

1. Allez à [Dashboard &gt; Users Management &gt; Users](https://manage.auth0.com/#/users).
2. Cliquez sur l’utilisateur dont vous ուզումज़ réinitialiser l’authentification multifacteur.
3. Cliquez sur le bouton **Actions** en haut à droite de l’écran.
4. Sélectionnez **Reset Multi-factor** dans le menu déroulant. Les Admins verront aussi un lien **Reset MFA** au bas de l’onglet **Multi-Factor Authentication** de la page **User Details** si l’utilisateur a déjà configuré l’authentification multifacteur. Ces deux méthodes fonctionnent de la même manière. Une fenêtre contextuelle s’affichera pour confirmer votre décision.
5. Cliquez sur **Yes, reset it** pour réinitialiser l’authentification multifacteur de l’utilisateur.

La prochaine fois que l’utilisateur se connectera, il devra configurer son authentification multifacteur comme un nouvel utilisateur.

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

En tant qu’administrateur, vous pouvez également utiliser la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> pour supprimer l’inscription à l’authentification multifacteur d’un utilisateur à l’aide de `DELETE` `/api/v2/users/{id}/authentication-methods`. Si l’utilisateur a plus d’une inscription à l’authentification multifacteur, vous devrez répéter l’opération pour chaque inscription.

<div id="recovery-codes">
  ## Codes de récupération
</div>

Avec la plupart des facteurs d’authentification multifacteur, l’utilisateur final reçoit un code de récupération lors de l’inscription, qu’il doit noter et garder secret. S’il n’a pas son appareil avec lui ou s’il est temporairement incapable d’utiliser son processus habituel d’authentification multifacteur pour une autre raison, l’utilisateur peut se connecter en entrant ce code après son nom d’utilisateur et mot de passe. Si un code de récupération est utilisé, un nouveau code de récupération lui sera alors fourni.

Si un utilisateur désinstalle Guardian puis le réinstalle plus tard, il se peut qu’on lui demande d’entrer son code de récupération. Si le code de récupération a été perdu, l’utilisateur peut effectuer une nouvelle installation de l’application en désactivant la restauration automatique de sa sauvegarde Guardian. Pour ce faire, l’utilisateur devra désinstaller Guardian, désactiver temporairement la restauration automatique des sauvegardes dans les paramètres de son appareil (la procédure varie selon l’appareil), puis réinstaller l’application. Il devra ensuite ajouter son ou ses comptes d’authentification multifacteur à l’application comme s’il s’agissait d’une configuration initiale. Si les sauvegardes automatiques ou la restauration automatique ne sont pas activées sur l’appareil de l’utilisateur, la réinstallation de l’application ne demandera pas de code de récupération, et l’utilisateur devra ajouter son ou ses comptes d’authentification multifacteur comme lors d’une première configuration.
