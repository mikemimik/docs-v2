<Card title="Avant de commencer">
  * Abonnez-vous à un plan Enterprise avec le module complémentaire Adaptive MFA. Consultez la [tarification Auth0](https://auth0.com/pricing/) pour plus de détails.
  * Configurez et activez une connexion Database ou une connexion Active Directory.
  * Configurez et activez au moins un facteur MFA.
</Card>

Utilisez <Tooltip tip="Authentification multifacteur adaptative : authentification multifacteur (MFA) qui n’est déclenchée pour les utilisateurs que lorsqu’une tentative de connexion est considérée comme étant à faible niveau de confiance." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Adaptive+MFA">Adaptive MFA</Tooltip> pour déclencher <Tooltip tip="Authentification multifacteur adaptative : authentification multifacteur (MFA) qui n’est déclenchée pour les utilisateurs que lorsqu’une tentative de connexion est considérée comme étant à faible niveau de confiance." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> lorsqu’Auth0 détermine qu’une tentative de connexion présente un risque, et pour consigner les Risk Assessments de toutes les transactions de connexion dans les journaux du tenant.

<div id="enable-adaptive-mfa">
  ## Activer Adaptive MFA
</div>

Vous pouvez activer Adaptive MFA dans le <Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou avec la <Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> d’Auth0.

<Tabs>
  <Tab title="Dashboard">
    1. Accédez à [**Dashboard &gt; Sécurité &gt; Multi-factor Auth**](https://manage.auth0.com/#/security/mfa).

    <Frame>![Auth0 Dashboard Sécurité Multi-factor Auth politique Adaptive MFA](/docs/images/cdy7uua7fh8z/4IlQi0LXOPJdYjOuo09xtE/944ce27c115cb43133aac78d9b6b7886/MFA_Factors_-_English.png)</Frame>

    2. Dans la section **Factors**, activez et configurez au moins un facteur MFA. Pour en savoir plus, consultez [facteur de l’authentification multifacteur](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors).

    3. Dans la section **Define policies**, repérez **Require Multi-factor Auth**, puis sélectionnez **Use Adaptive MFA**. Risk Assessment sera automatiquement activé et consigné dans les journaux de votre tenant.

    4. Dans le champ **Device Trust Duration**, définissez le nombre de jours pendant lesquels un appareil reste approuvé avant que l’utilisateur doive s’authentifier avec l’authentification MFA. La durée par défaut est de 30 jours, mais vous pouvez augmenter ou diminuer le nombre de vérifications demandées à vos utilisateurs.

       <Warning>
         Les clients Auth0 sont responsables de toute diminution du niveau de sécurité résultant du remplacement de la période de mémorisation de l’appareil par une durée supérieure à la configuration standard recommandée par Okta.
       </Warning>

       * Vous pouvez définir la durée de confiance de l’appareil entre 1 et 365 jours.
       * Si vous modifiez cette durée, la nouvelle valeur s’appliquera à l’appareil de vos utilisateurs lors de leur prochaine connexion.

    5. Sélectionnez **Enregistrer**.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si vous utilisez le facteur `email` avec [Identifier First Authentication](/docs/fr-CA/authenticate/login/auth0-universal-login/identifier-first), vous devez mettre à jour les attributs de courriel dans [Dashboard &gt; Database Connections &gt; Authentication Methods](https://manage.auth0.com/#/connections/database). Dans l’onglet Email Configuration, assurez-vous que l’attribut de courriel est actif. Ensuite, définissez **Allow Signup** sur **Required** et activez l’option de courriel **Require** dans le profil utilisateur.

      <Frame>![Auth0 Dashboard > Authentication > Database Connections > Authentication Methods](/docs/images/cdy7uua7fh8z/ql7yYX1GnJaiPQnjQQBTs/7ce5b2a51be21e4b3b6dc65b97b4b3eb/Email_Config_-_English.png)</Frame>
    </Callout>
  </Tab>

  <Tab title="Management API">
    1. Obtenez un [jeton d’accès à la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec la portée `update:mfa_policies`.

    2. Appelez le endpoint [Set the multi-factor authentication policies](https://auth0.com/docs/api/management/v2/guardian/put-policies) de la Management API avec le payload approprié.

    3. Si vous souhaitez modifier la valeur par défaut de **Device Trust Duration** de 30 jours, appelez [Update New Device Accessor](https://auth0.com/docs/api/management/v2/risk-assessments/patch-new-device). Vous devez ajouter les portées suivantes à votre jeton d’accès à la Management API :
       * `read:attack_protection`
       * `update:attack_protection`
       <Warning>
         Les clients Auth0 sont responsables de toute diminution du niveau de sécurité résultant du remplacement de la période de mémorisation de l’appareil par une durée supérieure à la configuration standard recommandée par Okta.
       </Warning>
  </Tab>
</Tabs>

<div id="enable-adaptive-mfa-risk-assessment">
  ## Activer Risk Assessment d’Adaptive MFA
</div>

Si vous n’êtes pas encore prêt à activer Adaptive MFA, mais souhaitez commencer à l’entraîner à analyser les comportements de connexion, vous pouvez activer Adaptive MFA Risk Assessment indépendamment.

1. Accédez à [Dashboard &gt; Sécurité &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
2. Repérez la section **Define policies**.
3. Dans **MFA Risk Assessors**, sélectionnez **Enable Adaptive MFA Risk Assessment**.
4. Sélectionnez **Save**.

<div id="customize-adaptive-mfa">
  ## Personnaliser Adaptive MFA
</div>

Vous pouvez personnaliser le comportement d’Adaptive MFA afin d’offrir la meilleure expérience à vos utilisateurs tout en assurant la sécurité. Pour en savoir plus, consultez [Personnaliser Adaptive MFA](/docs/fr-CA/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les Actions qui déclenchent MFA priment sur le comportement par défaut d’Adaptive MFA.
</Callout>

<div id="limitations">
  ## Limitations
</div>

Les informations d’évaluation dans les journaux du tenant sont uniquement disponibles pour les flux interactifs. Auth0 ne prend pas en charge l’enregistrement des informations d’évaluation pour les flux <Tooltip tip="Resource Owner : entité (comme un utilisateur ou une application) capable d’accorder l’accès à une ressource protégée." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Resource+Owner">Resource Owner</Tooltip> Password Grant (ROPG) lorsque Adaptive MFA n’est pas activé. Pour en savoir plus sur les limitations des flux d’authentification, consultez [Adaptive MFA](/docs/fr-CA/secure/multi-factor-authentication/adaptive-mfa).
