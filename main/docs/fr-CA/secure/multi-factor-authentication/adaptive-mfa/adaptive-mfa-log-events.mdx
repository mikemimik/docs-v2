---

Les journaux de votre tenant contiennent des entrées pour les événements de connexion réussis et Failed, y compris des renseignements liés aux scores de Risk Assessment d’<Tooltip tip="Adaptive Multi-factor Authentication : authentification multifacteur (MFA) qui n’est déclenchée pour les utilisateurs que lorsqu’une tentative de connexion est déterminée comme étant une connexion à faible niveau de confiance." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Adaptive+MFA">Adaptive MFA</Tooltip>. Il s’agit des mêmes entrées que celles qui peuvent être utilisées dans les objets context de règle. Les entrées d’Adaptive <Tooltip tip="Adaptive Multi-factor Authentication : authentification multifacteur (MFA) qui n’est déclenchée pour les utilisateurs que lorsqu’une tentative de connexion est déterminée comme étant une connexion à faible niveau de confiance." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> ont la structure suivante :

<div id="attack-protection-details">
  ## Détails de la protection contre les attaques
</div>

En plus de fournir des renseignements sur l’exécution des règles, Auth0 ajoute à vos journaux du tenant des renseignements sur l’Adaptive MFA <Tooltip tip="Protection contre les attaques : fonctionnalités qu’Auth0 fournit pour détecter et atténuer les attaques, y compris la protection contre les attaques par force brute, le throttling des IP suspectes, la détection des mots de passe compromis, la détection de bots et l’authentification multifacteur adaptative." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=attack+protection">protection contre les attaques</Tooltip>. La même structure de données que celle qui apparaît dans les règles se retrouve dans les journaux du tenant :

```json lines expandable
{
  "date": "2020-06-26T15:12:43.654Z",
  "type": "s",
  "details": {
	"anomalyDetection": {
  	  "confidence": "high",
  	  "assessments": {
    	    "UntrustedIP": {
      	    "confidence": "high",
      	    "reason": "ip not found",
      	    "code": "ip_not_found"
    	    },
    	    "NewDevice": {
      	    "confidence": "medium",
      	    "reason": "match useragent",
      	    "code": "match_useragent"
    	    },
    	    "ImpossibleTravel": {
      	    "confidence": "high",
      	    "reason": "minimal travel",
      	    "code": "minimal_travel_from_last_login"
    	    }
  	  }
	},
  },
  "description": "Successful login"
}
```
