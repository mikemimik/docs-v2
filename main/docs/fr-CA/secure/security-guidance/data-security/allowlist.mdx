description: Répertorie les adresses IP d’Auth0 à utiliser dans des listes d’autorisation pour le trafic entrant derrière un pare-feu.
title: Adresses IP d’Auth0 pour les listes d’autorisation
---

Si vous avez du code personnalisé qui s’exécute dans Auth0 et qui appelle un service à l’intérieur de votre réseau, ou si vous avez configuré dans Auth0 un fournisseur SMTP sur site, vous pourriez devoir configurer votre pare-feu pour autoriser le trafic entrant provenant d’Auth0.

Les fonctionnalités qui peuvent exiger que vous autorisiez le trafic entrant provenant d’Auth0 comprennent :

* [Actions](/docs/fr-CA/customize/actions)
* [Rules](/docs/fr-CA/customize/rules)
* [Hooks](/docs/fr-CA/customize/hooks)
* [Custom Database Action Scripts](/docs/fr-CA/authenticate/database-connections/custom-db/templates)
* [Log Streams](/docs/fr-CA/customize/log-streams)
* [Fournisseurs de courriel SMTP externes](/docs/fr-CA/customize/email/smtp-email-providers)

<div id="outbound-calls">
  ## Appels sortants
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les adresses IP peuvent changer. En cas de changement d’adresse IP, Auth0 envoie des notifications plusieurs mois avant l’entrée en vigueur des changements. Les listes sont à jour au moment de la rédaction.
</Callout>

Lorsque Auth0 effectue des appels sortants, les adresses IP sont statiques. Auth0 convertit les adresses IP internes en l’une des options affichées lorsqu’il communique au moyen de la NAT.

<div id="public-cloud">
  ### Public Cloud
</div>

Pour les tenants Public Cloud, les adresses IP que vous devez autoriser dans votre pare-feu dépendent de la région du tenant.

<div id="machine-readable-list">
  #### Liste lisible par machine
</div>

Vous pouvez accéder à une liste des plages CIDR d’adresses IP du CDN Auth0 en liste lisible par machine : https://cdn.auth0.com/ip-ranges.json.

<div id="schema">
  #### Schéma
</div>

La liste lisible par machine contient les propriétés suivantes :

| **Propriété**     | **Type de données** | **Description**                                                                                                                                                                                                                                                                                                                                                            |
| :---------------- | :------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `last_updated_at` | timestamp (ISO8601) | Date de la plus récente mise à jour de la liste.                                                                                                                                                                                                                                                                                                                           |
| `regions`         | objet               | Collection de [régions de tenants Auth0](/docs/fr-CA/get-started/auth0-overview/create-tenants#region-locality-and-sub-locality). Chaque objet de région de tenant contient la propriété suivante : * `ipv4_cidrs` : Tableau de chaînes. Liste des plages CIDR IPv4 pour la région donnée.                                                                                       |
| `changelog`       | tableau d’objets    | Historique complet des ajouts et suppressions de plages CIDR IP. Chaque objet contient les propriétés suivantes : * `date` : Chaîne. Horodatage au format AAAA-MM-JJ. * `region` : Chaîne. Région Auth0 touchée. * `action` : Chaîne. Action effectuée. Les valeurs possibles sont : `add`, `remove`. * `ipv4_cidrs` : Tableau de chaînes. Liste des plages CIDR touchées. |

<div id="detecting-changes">
  #### Détection des changements
</div>

Vous pouvez détecter par programmation les changements apportés à la liste lisible par machine en vous basant sur la propriété last&#95;updated&#95;at ou sur la réponse d&#39;Auth0 à une requête HTTP conditionnelle. Auth0 met cette liste à jour avant toute incidence fonctionnelle sur les tenants existants. Prévoyez une vérification hebdomadaire automatisée pour surveiller les changements et maintenir des configurations de pare-feu adéquates.

Si vous envoyez une requête HTTP conditionnelle, vous pouvez utiliser l&#39;en-tête If-None-Match ou If-Modified-Since. Si la liste a été modifiée, Auth0 renvoie une réponse HTTP 200 (OK) avec la liste mise à jour dans le corps de la réponse. Sinon, Auth0 renvoie une réponse 304 (Not Modified) sans corps.
Pour en savoir plus sur les requêtes HTTP conditionnelles, consultez [RFC 9110 : HTTP Semantics on RFC Editor](https://www.rfc-editor.org/rfc/rfc9110.html#name-conditional-requests).

<div id="example-of-conditional-http-request-if-none-match-header">
  #### Exemple de requête HTTP conditionnelle (en-tête If-None-Match)
</div>

```curl
# La première requête.
$ curl https://cdn.auth0.com/ip-ranges.json
HTTP/2 200
last-modified: Tue, 22 Apr 2025 07:50:08 GMT
etag: "1652cbe64a22099d62424fbec1022da5"
...

# Le contenu n'a pas changé, la réponse est `HTTP 304`
$ curl https://cdn.auth0.com/ip-ranges.json \
       -H "If-None-Match: 1652cbe64a22099d62424fbec1022da5" 
HTTP/2 304
last-modified: Tue, 22 Apr 2025 07:50:08 GMT
etag: "1652cbe64a22099d62424fbec1022da5"
...

# Le contenu a changé, la réponse est `HTTP 200` avec un nouvel `etag` :
$ curl https://cdn.auth0.com/ip-ranges.json \
       -H "If-None-Match: 1652cbe64a22099d62424fbec1022da5" 
HTTP/2 200
last-modified: Mon, 07 Jul 2025 01:04:18 GMT
etag: "8e58c97d39a67706c644cbe3b5c682d3"
...
```

<div id="private-cloud">
  ### Private Cloud
</div>

Pour les tenants Private Cloud, les adresses IP que vous devez autoriser dans votre pare-feu sont propres à l’environnement du tenant. Auth0 peut recevoir les adresses IP privées de votre tenant si vous activez des fonctionnalités comme Tenant Logs, <Tooltip tip="Suspicious IP Throttling : forme de protection contre les attaques qui protège votre tenant contre les connexions suspectes visant un trop grand nombre de comptes à partir d’une seule adresse IP." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Suspicious+IP+throttling">Suspicious IP throttling</Tooltip>, Custom Databases et Actions qui en dépendent.

Ces adresses IP sont appelées **adresses IP de sortie principales** et figurent dans les données de configuration de l’environnement accessibles dans le [Auth0 Support Center](https://support.auth0.com/tenants/private).

<div id="inbound-calls">
  ## Appels entrants
</div>

Les adresses IP associées aux appels entrants vers Auth0 peuvent varier, puisqu’il n’y a pas d’adresses IP fixes sur les répartiteurs de charge. Dans ce cas, les règles du pare-feu doivent s’appuyer sur le nom du service (par exemple : `<YOUR_TENANT>.<YOUR_REGION>.auth0.com`).

Si votre abonnement Auth0 vous permet de configurer un <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou de fantaisie." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=custom+domain">domaine personnalisé</Tooltip> autogéré, vous pouvez le configurer de façon à lui attribuer une adresse IP statique. Les domaines personnalisés autogérés vous donnent le contrôle du point d’entrée réseau et vous permettent de garantir que l’adresse IP demeure fixe. Pour en savoir plus sur les forfaits, consultez [Tarification Auth0](https://auth0.com/pricing).
