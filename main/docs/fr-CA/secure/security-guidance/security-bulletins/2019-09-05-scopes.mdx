description: Cas où un code de règle personnalisé mal conçu peut créer des vulnérabilités dans le flux d’authentification.
title: Bulletin de sécurité Auth0 sur l’attribution de scopes selon l’adresse courriel
---

<div id="overview">
  ## Vue d’ensemble
</div>

Si vous :

* utilisez des règles pour attribuer des scopes aux utilisateurs en fonction de leurs adresses courriel **et**
* votre application utilise plusieurs connexions

il est possible que vos scopes soient compromis.

<div id="how-this-works">
  ## Fonctionnement
</div>

Auth0 exige que les adresses courriel soient uniques pour chaque connexion. Cependant, il n’y a aucune restriction pour chaque application.

Il est donc possible que l’utilisateur A s’insive à l’application au moyen d’une connexion, puis que l’utilisateur B s’y inscrive avec la même adresse courriel au moyen d’une autre connexion.

Si vos règles attribuent des scopes aux utilisateurs en fonction de leur adresse courriel, le deuxième utilisateur se verra alors attribuer les mêmes scopes que le premier, même s’il s’agit d’une personne différente.

<div id="how-do-i-fix-this">
  ## Comment puis-je corriger cela?
</div>

La solution la plus simple consiste à exiger que les utilisateurs vérifient leur adresse courriel après leur inscription, avant d’être autorisés à se connecter.
