description: Situations où un code de règle personnalisé inadéquat peut créer des vulnérabilités dans le flux d’authentification.
title: Bulletin de sécurité Auth0 sur l’attribution de portées en fonction de l’adresse courriel
---

<div id="overview">
  ## Aperçu
</div>

Si vous :

* utilisez des règles pour attribuer des scopes aux utilisateurs en fonction de leur adresse courriel **et**
* votre application utilise plusieurs connexions

il est possible que vos scopes soient compromis.

<div id="how-this-works">
  ## Comment ça fonctionne
</div>

Auth0 exige que les adresses courriel soient uniques pour chaque connexion. Toutefois, il n’y a aucune restriction par application.

Il est donc possible que l’utilisateur A s’inscrive à l’application au moyen d’une connexion, tandis que l’utilisateur B s’inscrive à la même application avec la même adresse courriel au moyen d’une autre connexion.

Si vos règles attribuent des scopes aux utilisateurs en fonction de leur adresse courriel, le deuxième utilisateur se voit alors attribuer les mêmes scopes que le premier, même s’il s’agit d’une personne différente.

<div id="how-do-i-fix-this">
  ## Comment corriger cela ?
</div>

La mesure d’atténuation la plus simple consiste à obliger les utilisateurs à vérifier leur adresse courriel après leur inscription, avant d’être autorisés à se connecter.
