description: Cas où du code de règle personnalisé inadéquat peut créer des vulnérabilités dans le processus d’authentification.
title: Bulletin de sécurité Auth0 sur l’attribution de portées selon l’adresse courriel
---

<div id="overview">
  ## Aperçu
</div>

Si vous :

* utilisez des Rules pour attribuer des portées aux utilisateurs en fonction de leur adresse courriel **et**
* votre application utilise plusieurs connexions

il est possible que vos portées soient compromises.

<div id="how-this-works">
  ## Fonctionnement
</div>

Auth0 exige que les adresses courriel soient uniques pour chaque connexion. Toutefois, il n’y a aucune restriction par application.

Il est donc possible que l’utilisateur A s’inscrive à l’application au moyen d’une connexion, tandis que l’utilisateur B s’inscrit à la même application avec la même adresse courriel au moyen d’une autre connexion.

Si vos Rules attribuent des portées aux utilisateurs en fonction de leur adresse courriel, le deuxième utilisateur se retrouve alors avec les mêmes portées que le premier, même s’il s’agit d’une personne différente.

<div id="how-do-i-fix-this">
  ## Comment corriger ce problème?
</div>

La façon la plus simple d’atténuer ce risque consiste à exiger que les utilisateurs vérifient leur adresse courriel après leur inscription et avant d’être autorisés à se connecter.
