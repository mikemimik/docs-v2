**Publication** : 21 décembre 2022

**Numéros CVE** : CVE-2022-23539, CVE-2022-23541, CVE-2022-23540

<div id="overview">
  ### Vue d’ensemble
</div>

Auth0 a publié une nouvelle version majeure de la bibliothèque `jsonwebtoken` pour corriger quatre vulnérabilités.

Nous vous recommandons de consulter les avis de sécurité suivants et de passer à cette nouvelle version majeure :

* Un type de clé non restreint pourrait mener à l’utilisation de clés Legacy : [CVE-2022-23539](https://github.com/auth0/node-jsonwebtoken/security/advisories/GHSA-8cf7-32gw-wr33)
* Une mise en œuvre non sécurisée de la fonction de récupération de clé pourrait permettre de forger des jetons Public/Private de RSA vers HMAC : [CVE-2022-23541](https://github.com/auth0/node-jsonwebtoken/security/advisories/GHSA-hjrf-2m68-5959)
* Un algorithme par défaut non sécurisé dans <Tooltip tip="JSON Web Token (JWT) : format standard de l’ID Token (et souvent de l’Access Token) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=jwt">jwt</Tooltip>.verify() pourrait permettre de contourner la validation de la signature : [CVE-2022-23540](https://github.com/auth0/node-jsonwebtoken/security/advisories/GHSA-qwph-4952-7xr6)

<div id="am-i-affected">
  ### Suis-je concerné ?
</div>

Vous pourriez être concerné si vous utilisez `jsonwebtoken` dans une version &lt;= 8.5.1, selon la configuration. Veuillez consulter les avis de sécurité correspondants pour plus de détails.

<div id="how-to-fix-that">
  ### Comment corriger cela ?
</div>

Si vous utilisez `jsonwebtoken`, passez à la version 9.0.0 ou ultérieure. Une configuration supplémentaire pourrait être nécessaire. Veuillez consulter chaque avis de sécurité pour en savoir plus.

<div id="will-this-update-impact-my-users">
  ### Cette mise à jour aura-t-elle des répercussions sur mes utilisateurs?
</div>

Le passage à la version 9.0.0 peut avoir des répercussions sur vos utilisateurs, selon votre configuration et les besoins de votre application. Veuillez consulter chacun des avis de sécurité pour en savoir plus.
