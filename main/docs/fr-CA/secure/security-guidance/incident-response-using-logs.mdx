L’examen des journaux pour évaluer l’impact d’une attaque est une étape essentielle de votre plan d’intervention en cas d’incident. Sur cette page, vous verrez comment consulter les journaux dans l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, ainsi que quelques exemples de requêtes de recherche pour repérer les signes d’une attaque et examiner l’activité du compte.

<div id="check-auth0-logs">
  ## Consultez les journaux d’Auth0
</div>

1. [Connectez-vous à l’Auth0 Dashboard](https://manage.auth0.com/#)
2. La page Logs se trouve sous **Monitoring** dans le menu de gauche.
3. Sur la page Logs, vous verrez une barre de recherche, un filtre et un sélecteur de date.

<Frame>![Dashboard Monitoring Logs](/docs/images/cdy7uua7fh8z/2l9LfKEjrprVmHfWt80VX0/4e48bc6ad3a8b591014e23fdb7353405/2025-01-28_15-07-43.png)</Frame>

Sélectionnez un événement de journal dans la liste pour voir un **Summary** de l’événement ainsi que d’autres **Details**, y compris le **Raw JSON**.

<div id="log-structure">
  ### Structure du journal
</div>

Chaque événement de journal comporte les champs suivants :

| Champ           | Description                                                                               |
| --------------- | ----------------------------------------------------------------------------------------- |
| `date`          | Horodatage indiquant le moment où cet événement s&#39;est produit.                        |
| `log_id`        | L&#39;ID de l&#39;événement de journal                                                    |
| `type`          | Le type d&#39;événement de journal                                                        |
| `description`   | La description de l&#39;événement.                                                        |
| `connection`    | Le nom de la connexion associée à l&#39;événement.                                        |
| `connection_id` | L&#39;ID de la connexion associée à l&#39;événement.                                      |
| `client_id`     | L&#39;ID du client associé à l&#39;événement                                              |
| `client_name`   | Le nom du client associé à l&#39;événement.                                               |
| `ip`            | L&#39;adresse IP d&#39;où provenait la requête à l&#39;origine de l&#39;événement de journal. |
| `user_agent`    | L&#39;agent utilisateur associé à l&#39;événement.                                        |
| `details`       | Un objet contenant des renseignements supplémentaires sur cet événement de journal.       |
| `user_id`       | L&#39;ID de l&#39;utilisateur associé à l&#39;événement.                                  |
| `user_name`     | Le nom d&#39;utilisateur associé à l&#39;événement.                                       |
| `strategy`      | La stratégie de connexion associée à l&#39;événement.                                     |
| `strategy_type` | Le type de stratégie de connexion associé à l&#39;événement.                              |

<div id="indicators-of-an-attack">
  ## Indicateurs d’une attaque
</div>

Il peut être difficile de repérer une attaque rapidement, mais voici quelques éléments à surveiller dans votre journal, ainsi que des exemples de requêtes de recherche :

* Un nombre élevé de connexions échouées avec des noms d’utilisateur invalides, ou de tentatives de connexion visant des utilisateurs inexistants.

  * `type:"fu"`
  * `description:"missing username parameter"`
  * `description:"Wrong email or password"`
* Un grand nombre de comptes atteignant la limite de tentatives de connexion échouées.

  * `type:"limit_wc"`
* Un nombre élevé de tentatives de connexion utilisant un mot de passe divulgué.

  * `type:"pwd_leak"`

Pendant votre enquête, prenez en note les adresses IP, les applications ciblées, ainsi que les connexions ou <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère des identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> utilisés.

<div id="identify-compromised-user-accounts">
  ## Identifier les comptes d’utilisateur compromis
</div>

Pour repérer les comptes d’utilisateur qui pourraient avoir été compromis, vous pouvez rechercher :

* Des événements de connexion réussis provenant d’une adresse IP suspecte :

  * `type:"s" AND ip:"99.xxx.xxx.xxx"`

<div id="check-compromised-user-account-activity">
  ## Vérifier l’activité d’un compte utilisateur compromis
</div>

Après avoir identifié un compte utilisateur compromis, vous voudrez vérifier l’activité du compte :

* Recherchez d’autres événements dans le journal avec le même `user_id` : `user_id:"auth0|ABC123"`
* Vérifiez les champs d’événement de journal `client_name` ou `client_id` pour voir à quelles applications on a accédé. Prenez note du moment où l’accès a eu lieu.
* Vérifiez s’il y a eu un accès administratif ou des modifications à la configuration d’Auth0
* Recherchez des appels récents à l’<Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> : `type:"sapi"`

<div id="delete-or-block-users-from-the-dashboard">
  ## Supprimer ou bloquer des utilisateurs depuis le Dashboard
</div>

1. Accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
2. Recherchez l’utilisateur à supprimer ou à bloquer.
3. Cliquez sur le bouton &quot;**...**&quot; tout à droite de l’utilisateur.
4. Sélectionnez **Block** ou **Delete**, puis confirmez.
