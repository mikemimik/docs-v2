L’examen des journaux pour évaluer l’impact d’une attaque est une étape essentielle de votre plan d’intervention en cas d’incident. Cette page vous explique comment accéder aux journaux dans le <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, et présente quelques exemples de requêtes de recherche dans les journaux pour repérer les signes d’une attaque et examiner l’activité du compte.

<div id="check-auth0-logs">
  ## Vérifier les journaux Auth0
</div>

1. [Connectez-vous à l’Auth0 Dashboard](https://manage.auth0.com/#)
2. La page Journaux se trouve sous **Monitoring** dans le menu de gauche.
3. Sur la page Journaux, vous verrez une barre de recherche, une option de filtre et un sélecteur de date.

<Frame>![Dashboard Monitoring Journaux](/docs/images/cdy7uua7fh8z/2l9LfKEjrprVmHfWt80VX0/4e48bc6ad3a8b591014e23fdb7353405/2025-01-28_15-07-43.png)</Frame>

Sélectionnez un journal dans la liste pour voir un **Summary** de l’événement ainsi que d’autres **Details**, y compris le JSON brut.

<div id="log-structure">
  ### Structure du journal
</div>

Chaque événement de journal comprend les champs suivants :

| Champ           | Description                                                                         |
| --------------- | ----------------------------------------------------------------------------------- |
| `date`          | Horodatage indiquant quand cet événement s’est produit.                             |
| `log_id`        | L’ID de l’événement de journal                                                      |
| `type`          | Le type d’événement de journal                                                      |
| `description`   | La description de l’événement.                                                      |
| `connection`    | Le nom de la connexion associée à l’événement.                                      |
| `connection_id` | L’ID de la connexion associée à l’événement.                                        |
| `client_id`     | L’ID client associé à l’événement                                                   |
| `client_name`   | Le nom du client associé à l’événement.                                             |
| `ip`            | L’adresse IP d’où provenait la requête à l’origine de l’événement de journal.       |
| `user_agent`    | L’agent utilisateur associé à l’événement.                                          |
| `details`       | Un objet contenant des renseignements supplémentaires sur cet événement de journal. |
| `user_id`       | L’ID utilisateur associé à l’événement.                                             |
| `user_name`     | Le nom d’utilisateur associé à l’événement.                                         |
| `strategy`      | La stratégie de connexion associée à l’événement.                                   |
| `strategy_type` | Le type de stratégie de connexion associé à l’événement.                            |

<div id="indicators-of-an-attack">
  ## Indicateurs d’une attaque
</div>

Il peut être difficile de repérer rapidement une attaque, mais voici quelques éléments à surveiller dans vos journaux, ainsi que des exemples de requêtes de recherche :

* Un nombre élevé de connexions échouées avec des noms d’utilisateur invalides ou de tentatives de connexion visant des utilisateurs inexistants.

  * `type:"fu"`
  * `description:"missing username parameter"`
  * `description:"Wrong email or password"`
* Un grand nombre de comptes atteignant la limite de tentatives de connexion échouées.

  * `type:"limit_wc"`
* Un nombre élevé de tentatives de connexion utilisant un mot de passe compromis.

  * `type:"pwd_leak"`

Au cours de votre enquête, prenez note des adresses IP, des applications ciblées et des connexions ou <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> utilisés.

<div id="identify-compromised-user-accounts">
  ## Identifier les comptes d’utilisateur compromis
</div>

Pour repérer les comptes d’utilisateur qui pourraient avoir été compromis, vous pouvez rechercher :

* Des événements de connexion réussie à partir d’une adresse IP suspecte :

  * `type:"s" AND ip:"99.xxx.xxx.xxx"`

<div id="check-compromised-user-account-activity">
  ## Vérifier l’activité d’un compte utilisateur compromis
</div>

Après avoir identifié un compte utilisateur compromis, vous voudrez vérifier l’activité du compte :

* Recherchez d’autres événements du journal ayant le même `user_id` : `user_id:"auth0|ABC123"`
* Vérifiez les champs `client_name` ou `client_id` de l’événement du journal pour voir à quelles applications l’accès a eu lieu. Prenez en note le moment où l’accès s’est produit.
* Vérifiez s’il y a eu un accès administrateur ou des changements à la configuration d’Auth0
* Recherchez les appels récents à <Tooltip tip="Management API : Un produit permettant aux clients d'effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> : `type:"sapi"`

<div id="delete-or-block-users-from-the-dashboard">
  ## Supprimer ou bloquer des utilisateurs à partir du dashboard
</div>

1. Accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
2. Recherchez l’utilisateur à supprimer ou à bloquer.
3. Cliquez sur le bouton &quot;**...**&quot; à l’extrême droite de l’utilisateur.
4. Sélectionnez **Block** ou **Delete**, puis confirmez.
