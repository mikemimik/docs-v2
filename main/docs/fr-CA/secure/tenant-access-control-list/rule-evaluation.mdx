title: Évaluation des règles de la Tenant Access Control List
sidebarTitle: Évaluation des règles
description: Découvrez comment le moteur déterministe de Tenant ACL évalue les correspondances de règles et leur exécution.
---

Tenant ACL utilise un moteur déterministe pour évaluer les requêtes entrantes. Pour déterminer si une action (autoriser, bloquer ou rediriger) doit être exécutée, le moteur suit les étapes d’évaluation suivantes :

1. Ordre d’évaluation (priorité)

2. Correspondance des conditions (signal)

3. Fin de la correspondance avec exception pour le mode de surveillance

<div id="evaluation-order">
  ## Ordre d’évaluation
</div>

Le Tenant ACL évalue les règles par ordre croissant de numéro de priorité. L’évaluation commence par le plus petit numéro, soit la priorité 0, puis se poursuit séquentiellement.

Si deux règles peuvent toutes deux correspondre à une requête, celle dont le numéro de priorité est le plus bas se déclenchera en premier.

<div id="condition-matching">
  ## Correspondance des conditions
</div>

Pour qu’une règle donnée déclenche son action, elle doit respecter la logique de correspondance. Cette logique détermine comment le moteur traite plusieurs valeurs et plusieurs types de signaux :

| Type de logique            | Comportement | Description                                                                                                                                                                                                                                  | Exemple                                                                                |
| -------------------------- | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| Correspondance des signaux | OR           | Pour un signal donné, une requête doit correspondre à au moins un élément de la liste pour satisfaire cette condition du signal.                                                                                                             | Si `ipv4_cidrs` contient plusieurs CIDR IP, au moins un d’entre eux doit correspondre. |
| Signaux multiples          | AND          | Si une règle contient plusieurs types de signaux différents, la requête doit correspondre à tous les types de signaux fournis. Exception : si une règle contient à la fois `ipv4_cidrs` et `ipv6_cidrs`, un seul des deux doit correspondre. | Si une règle contient à la fois `ipv4_cidrs` et `asn`, les deux doivent correspondre.  |

Les types de signaux qui ne sont pas fournis dans une règle sont ignorés et n’ont aucune incidence sur la logique d’évaluation de la règle.

<div id="match-termination">
  ## Fin de la correspondance
</div>

Tenant ACL applique la première correspondance. Une fois que les conditions d&#39;une règle sont entièrement remplies, le comportement dépend de si la règle est en [mode de surveillance](./enable-monitoring-mode) ou non.

| Comportement       | Mode de surveillance désactivé                                                                            | Mode de surveillance activé                                                             |
| ------------------ | --------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **Journalisation** | Un événement de log `acls_summary` est déclenché.                                                         | Un événement de log `acls_summary` est déclenché.                                       |
| **Exécution**      | L&#39;action définie par la règle est exécutée immédiatement.                                             | L&#39;action définie par la règle n&#39;est pas exécutée.                               |
| **Évaluation**     | L&#39;évaluation prend fin. Aucune règle Tenant ACL suivante ou de priorité inférieure n&#39;est évaluée. | L&#39;évaluation se poursuit jusqu&#39;à la règle suivante dans la liste des priorités. |
