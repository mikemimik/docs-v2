<div id="configure-orts">
  ## Configurer les ORT
</div>

Les jetons d’actualisation en ligne (ORT) doivent être activés au niveau de l’[API (Resource Server)](/docs/fr-CA/get-started/apis/api-settings) dans l’Auth0 Dashboard ou à l’aide de la Management API.

<div id="configure-using-the-dashboard">
  ### Configurer à l’aide du Dashboard
</div>

1. Accédez à [Dashboard &gt; Applications &gt; API](https://manage.auth0.com/#/apis).

2. Sélectionnez l’API que vous voulez configurer.

3. Dans l’onglet **Settings**, activez la bascule **Allow Online Access**.

   <Frame>![Dashboard Applications APIs Settings Allow Online Access](/docs/images/refresh-tokens/allow-online-access.png)</Frame>

4. Sélectionnez **Save**.

Pour consulter et configurer les valeurs du délai d’inactivité et de la durée de vie absolue des sessions, consultez les paramètres [Tenant Session Expiration](https://manage.auth0.com/#/tenant/advanced). Pour en savoir plus, consultez [Configurer la durée de vie de la session](/docs/fr-CA/manage-users/sessions/configure-session-lifetime).

<div id="configure-using-the-management-api">
  ### Configurer avec la Management API
</div>

Pour activer les ORT, envoyez une requête `PATCH` au point de terminaison [Mettre à jour le serveur de ressources](/docs/fr-CA/api/management/v2/resource-servers/patch-resource-servers-by-id) :

```bash
curl --request PATCH \
  --url 'https://<YOUR_DOMAIN>/api/v2/resource-servers/<YOUR_RESOURCE_SERVER_ID>' \
  --header 'authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
  --header 'content-type: application/json' \
  --data '{"allow_online_access": true}'
```

Pour vérifier la configuration, faites une requête `GET` au serveur de ressources :

```bash
curl --request GET \
  --url 'https://<YOUR_DOMAIN>/api/v2/resource-servers/<YOUR_RESOURCE_SERVER_ID>' \
  --header 'authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>'
```

La réponse inclut la propriété `allow_online_access` :

```json
{
  "id": "resource-server-id",
  "name": "My API",
  "identifier": "https://my-api.example.com",
  "allow_online_access": true,
  ...
}
```

<div id="get-an-ort">
  ## Obtenir un ORT
</div>

Pour obtenir un ORT, incluez la valeur de [scope](/docs/fr-CA/get-started/apis/scopes) `online_access` dans votre demande d’autorisation en utilisant le [flux du code d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow) ou le [flux du code d’autorisation avec PKCE](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

```bash
https://<YOUR_DOMAIN>/authorize \
    audience=<YOUR_API_AUDIENCE> \
    scope=openid profile online_access \
    response_type=code \
    client_id=<YOUR_CLIENT_ID> \
    redirect_uri=<https://YOUR_APP/callback> \
    state=<OPAQUE_VALUE>
```

Une fois que l’utilisateur s’est authentifié, échangez le code d’autorisation contre des jetons :

```bash
curl --request POST \
  --url 'https://<YOUR_DOMAIN>/oauth/token' \
  --header 'content-type: application/x-www-form-urlencoded' \
  --data grant_type=authorization_code \
  --data 'client_id=<YOUR_CLIENT_ID>' \
  --data 'code=<YOUR_AUTHORIZATION_CODE>' \
  --data 'redirect_uri=<https://YOUR_APP/callback>' \
  --data 'code_verifier=<YOUR_CODE_VERIFIER>'
```

La réponse du token comprend l’ORT :

```json
{
  "access_token": "eyJ...",
  "id_token": "eyJ...",
  "refresh_token": "ORT...",
  "token_type": "Bearer",
  "expires_in": 86400
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les ORT commencent par `ORT` pour les distinguer des jetons d’actualisation. Toutefois, vous devez traiter le jeton comme opaque et ne pas vous fier à sa structure interne.
</Callout>

<div id="use-an-ort">
  ## Utiliser un ORT
</div>

Échangez l’ORT contre un nouveau [jeton d’accès](/docs/fr-CA/secure/tokens/access-tokens) lorsque le jeton d’accès actuel expire ou est sur le point d’expirer :

```bash
curl --request POST \
  --url 'https://<YOUR_DOMAIN>/oauth/token' \
  --header 'content-type: application/x-www-form-urlencoded' \
  --data grant_type=refresh_token \
  --data 'client_id=<YOUR_CLIENT_ID>' \
  --data 'refresh_token=<YOUR_ONLINE_REFRESH_TOKEN>'
```

La réponse contient un nouveau jeton d’accès.
Si votre requête inclut la portée `openid`, la réponse comprend un nouveau [jeton d’identité](/docs/fr-CA/secure/tokens/id-tokens):

```json
{
  "access_token": "eyJ...",
  "id_token": "eyJ...",
  "token_type": "Bearer",
  "expires_in": 86400
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 n’émet pas de nouveau `refresh_token` et, comme les ORT ne changent pas, vous continuez à utiliser le même ORT lors des échanges suivants.
</Callout>

<div id="session-extension-behavior">
  ### Comportement de la prolongation de session
</div>

Chaque échange de jeton réussi :

* **Réinitialise le délai d’inactivité de la session :** Le délai d’inactivité est rétabli à sa durée maximale.
* **Ne prolonge pas la durée de vie absolue :** L’expiration absolue de la session demeure inchangée.
* **Maintient le SSO :** D’autres applications peuvent obtenir des jetons par SSO tant que la session est active.

<div id="revoke-an-ort">
  ## Révoquer un ORT
</div>

Lorsque vous révoquez un ORT, cela met fin à l’ensemble de la session Auth0, et pas seulement au jeton. Cela invalide tous les ORT liés à cette session et met fin au SSO de l’utilisateur.

```bash
curl --request POST \
  --url 'https://<YOUR_DOMAIN>/oauth/revoke' \
  --header 'content-type: application/x-www-form-urlencoded' \
  --data 'client_id=<YOUR_CLIENT_ID>' \
  --data 'token=<YOUR_ONLINE_REFRESH_TOKEN>'
```

<div id="use-orts-with-actions">
  ## Utiliser les ORT avec les Actions
</div>

Vous pouvez utiliser les ORT avec les [Actions](/docs/fr-CA/customize/actions/actions-overview) d’Auth0 et le [déclencheur post-login](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

Avec les Actions, vous pouvez :

* Utiliser l’objet `event.refresh_token` pour déterminer si le jeton est un ORT.
* Accéder à des données propres à la session à l’aide de l’objet `event.session` pour prendre des décisions selon l’état actuel de la session

```javascript
exports.onExecutePostLogin = async (event, api) => {
  // Vérifier si le jeton est un jeton d'actualisation en ligne (ORT)
  
  if (event.refresh_token?.access == 'online') {
    // Le jeton est un ORT, vous pouvez ensuite référencer event.session et api.session
    console.log('Exchanging Online Refresh Token bound to Session ID: ', event.session?.id);
    // Récupérer les métadonnées de session et les ajouter aux jetons
    // En supposant que les métadonnées de session ont été stockées au préalable
    const importantInformation = event.session?.metadata?.importantInformation;
    api.accessToken.setCustomClaim('info', importantInformation);
    api.idToken.setCustomClaim('info', importantInformation);
  }
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 révoque l’intégralité de la session utilisateur, plutôt que seulement le jeton, lorsque la méthode `api.refreshToken.revoke()` est utilisée avec un ORT.
</Callout>
