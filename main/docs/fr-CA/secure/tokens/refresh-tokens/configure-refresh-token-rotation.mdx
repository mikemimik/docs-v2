Configurez la rotation du <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+token">jeton d’actualisation</Tooltip> pour chaque application au moyen du Dashboard ou du [Auth0 SPA SDK](/docs/fr-CA/libraries/auth0-single-page-app-sdk). Lorsque la <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+token+rotation">rotation des jetons d’actualisation</Tooltip> est activée, la transition se fait de façon transparente pour l’utilisateur. L’application utilise le jeton d’actualisation précédent, non expiré et sans rotation, puis l’échange contre un jeton d’actualisation avec rotation.

Les scénarios de migration prennent en charge la révocation automatique des jetons lors du passage d’un jeton d’actualisation sans rotation à un jeton d’actualisation avec rotation, et vice-versa.

* **Lorsque la rotation des jetons d’actualisation est activée** :
  L’échange d’un jeton d’actualisation sans rotation génère un nouveau jeton d’actualisation avec rotation et supprime tous les jetons sans rotation émis pour le même `client_id`, le <Tooltip tip="Serveur de ressources : serveur hébergeant des ressources protégées. Les serveurs de ressources acceptent les requêtes de ressources protégées et y répondent." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=resource+server">serveur de ressources</Tooltip>, l’utilisateur et le tenant.
* **Lorsque la rotation des jetons d’actualisation est désactivée** :
  L’échange d’un jeton d’actualisation avec rotation génère un jeton d’actualisation sans rotation et révoque la famille de jetons d’actualisation avec rotation émise pour le même `client_id`, le serveur de ressources, l’utilisateur et le tenant.

<div id="configure-in-the-dashboard">
  ## Configurer dans le Dashboard
</div>

1. Accédez à [Dashboard &gt; Applications](https://manage.auth0.com/#/applications).
2. Sélectionnez l’application que vous voulez configurer.
3. Accédez à l’onglet **Settings**.
4. Sous **rotation des jetons d’actualisation**, activez **Allow rotation des jetons d’actualisation**.

   <Frame>![Dashboard Applications Applications Settings Tab Refresh Token Rotation](/docs/images/cdy7uua7fh8z/Rk6i8MUVWZG34P7K6aurb/9457a6ea7ca5942522778c7ed7925023/Screenshot_2024-11-05_at_18.14.22.png)</Frame>
5. Entrez la valeur de **Rotation Overlap Period** (en secondes) pour le jeton d’actualisation afin de tenir compte du délai de tolérance entre la requête et la réponse avant de déclencher la détection automatique de réutilisation.
   Cette période de chevauchement aide à éviter les problèmes de simultanéité lorsqu’un jeton d’actualisation avec rotation est échangé plusieurs fois dans un délai donné. Pendant cette fenêtre de tolérance, les fonctionnalités de détection de compromission ne s’appliquent pas et un nouveau jeton d’actualisation avec rotation est émis. Seul le token précédent peut être réutilisé; si l’avant-dernier est échangé, la détection de compromission sera déclenchée.
6. Sélectionnez **Save Changes**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La rotation des jetons d’actualisation est uniquement prise en charge pour les applications conformes à OIDC avec le type d’autorisation Refresh Token activé.
</Callout>

<div id="configure-with-the-auth0-spa-sdk">
  ## Configurer avec le SDK Auth0 SPA
</div>

Vous pouvez utiliser le SDK Auth0 SPA pour activer la rotation des jetons d’actualisation. Vous devez activer l’accès hors ligne et demander le scope `offline_access` dans le SDK client.

1. Installez la dernière version du SDK `auth0-spa-js` :
   `npm install @auth0/auth0-spa-js`

2. Activez cette fonctionnalité dans le SDK en définissant `useRefreshTokens: true` pour commencer à envoyer le scope `offline_access`.

   {/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

   <AuthCodeBlock children={codeExample} language="javascript" />

3. Configurez les paramètres de rotation des jetons d’actualisation. Par exemple :

   ```http lines
   PATCH /api/v2/clients/{client_id}
   {
     "refresh_token": {
       "rotation_type": "rotating",
       "expiration_type": "expiring",
       "token_lifetime": "2592000",
       "leeway": 3
     }
   }
   ```

   | Attribut          | Description                                                                                                                                                                                                                                                                                                                                |
   | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | `rotation_type`   | Chaîne de texte : &quot;rotating&quot; ou &quot;non-rotating&quot;                                                                                                                                                                                                                                                                         |
   | `expiration_type` | Chaîne de texte : &quot;expiring&quot; ou &quot;non-expiring&quot;                                                                                                                                                                                                                                                                         |
   | `token_lifetime`  | La période d’expiration par défaut du jeton d’actualisation, lorsque la **rotation des jetons d’actualisation** est activée, est de 30 jours (2 592 000 secondes). Vous pouvez la configurer jusqu’à 1 an (31 557 600 secondes). **La durée de vie n’est pas prolongée lorsque les jetons sont remplacés.**                                |
   | `leeway`          | Permet d’utiliser le même jeton d’actualisation pendant une certaine période afin de tenir compte d’éventuels problèmes de concurrence réseau qui, autrement, invalideraient le jeton si le client tentait de réessayer avec le même jeton d’actualisation. Par défaut, cette marge de tolérance est désactivée. Configurable en secondes. |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La rotation des jetons d’actualisation est prise en charge uniquement pour les applications conformes à OIDC lorsque le type d’autorisation Refresh Token est activé.
</Callout>

<div id="automatic-reuse-detection">
  ## Détection automatique de la réutilisation
</div>

Si un jeton déjà invalidé est utilisé, tous les jetons d’actualisation émis depuis l’émission de ce jeton seront immédiatement révoqués, de même que l’octroi, et l’utilisateur devra s’authentifier de nouveau.
