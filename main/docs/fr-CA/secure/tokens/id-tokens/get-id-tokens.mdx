description: Décrit comment demander un jeton d’identité contenant des claims sur l'utilisateur lors de l'authentification, en incluant des scopes OIDC.
title: Obtenir des jetons d’identité
---

Pour obtenir un <Tooltip tip="ID Token: Informations d'identification destinées au client lui-même, plutôt qu'à l'accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+token">jeton d’identité</Tooltip>, vous devez le demander lors de l&#39;authentification des utilisateurs. Auth0 permet à votre application d&#39;authentifier facilement les utilisateurs au moyen de :

* [Quickstarts](/docs/fr-CA/quickstarts) : le moyen le plus simple de mettre en place l&#39;authentification. Ils peuvent vous montrer comment utiliser [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login), le widget Lock et les SDK Auth0 propres à chaque langage et framework. Notre [documentation Lock](/docs/fr-CA/libraries/lock) et la [documentation Auth0.js](/docs/fr-CA/libraries/auth0js) expliquent en détail comment récupérer un jeton d’identité après l&#39;authentification.
* [Authentication API](https://auth0.com/docs/api/authentication) : si vous préférez créer votre propre solution, vous pouvez appeler directement notre API. Vous devez d&#39;abord savoir [quel flux utiliser](/docs/fr-CA/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use) avant de suivre le tutoriel correspondant.

<div id="control-id-token-contents">
  ## Contrôler le contenu du jeton d’identité
</div>

Vous contrôlez quelles claims sur l’utilisateur authentifié sont incluses dans le jeton d’identité utilisé par votre application en incluant des [OpenID Connect Scopes](/docs/fr-CA/get-started/apis/scopes/openid-connect-scopes) précis dans le paramètre `scope` lorsque vous demandez des jetons pendant l’authentification des utilisateurs.

Vous pouvez aussi créer des [custom claims](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims), c’est-à-dire des claims que vous définissez, contrôlez et ajoutez à un jeton à l’aide d’une rule. Comme pour les autres <Tooltip tip="JSON Web Token (JWT) : format standard de l’ID token (et souvent de l’Access Token) utilisé pour représenter des claims de façon sécuritaire entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JWTs">JWTs</Tooltip>, vous devriez [valider un jeton d’identité](/docs/fr-CA/secure/tokens/id-tokens/validate-id-tokens) avant de supposer que son contenu est fiable.

<div id="renew-id-tokens">
  ## Renouveler les jetons d’identité
</div>

Par défaut, un jeton d’identité est valide pendant 36 000 secondes (10 heures). En cas de préoccupations liées à la sécurité, vous pouvez [raccourcir la durée de validité du jeton](/docs/fr-CA/secure/tokens/id-tokens/update-id-token-lifetime), mais n’oubliez pas que l’une des fonctions de ce jeton est d’améliorer les performances en mettant en cache les informations de l’utilisateur.

Une fois le jeton d’identité expiré, vous pouvez le renouveler. Pour renouveler le jeton d’identité, vous pouvez soit réauthentifier l’utilisateur avec Auth0, soit utiliser un [jeton d’actualisation](/docs/fr-CA/secure/tokens/refresh-tokens).
