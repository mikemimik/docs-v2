---
description: Décrit comment demander un jeton d’identité lors de l’authentification des utilisateurs, en incluant des claims sur l’utilisateur au moyen des scopes OIDC.
title: Obtenir des jetons ID
---

Pour obtenir un <Tooltip tip="Jeton ID : information d’identification destinée au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+token">jeton d’identité</Tooltip>, vous devez le demander au moment d’authentifier les utilisateurs. Auth0 permet à votre application d’authentifier facilement les utilisateurs à l’aide de :

* [Quickstarts](/docs/fr-CA/quickstarts) : la façon la plus simple de mettre en œuvre l’authentification. Ils peuvent vous montrer comment utiliser [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login), le Lock widget et les SDKs d’Auth0 propres à chaque langage et framework. Notre [documentation Lock](/docs/fr-CA/libraries/lock) et notre [documentation Auth0.js](/docs/fr-CA/libraries/auth0js) donnent des précisions sur la façon de récupérer un jeton d’identité après l’authentification.
* [Authentication API](https://auth0.com/docs/api/authentication) : si vous préférez créer votre propre solution, vous pouvez appeler directement notre API. Vous devez d’abord savoir [quel flow utiliser](/docs/fr-CA/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use) avant de suivre le tutoriel correspondant.

<div id="control-id-token-contents">
  ## Contrôler le contenu de l’jeton d’identité
</div>

Vous contrôlez les claims concernant l’utilisateur authentifié qui sont incluses dans l’jeton d’identité utilisé par votre application en incluant des [OpenID Connect Scopes](/docs/fr-CA/get-started/apis/scopes/openid-connect-scopes) précis dans le paramètre `scope` lorsque vous demandez des jetons pendant l’authentification des utilisateurs.

Vous pouvez aussi créer des [custom claims](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims), c’est-à-dire des claims que vous définissez, contrôlez et ajoutez à un jeton à l’aide d’une rule. Comme pour les autres <Tooltip tip="JSON Web Token (JWT) : format standard d’jeton d’identité (et souvent de Access Token) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JWTs">JWTs</Tooltip>, vous devriez [valider un jeton d’identité](/docs/fr-CA/secure/tokens/id-tokens/validate-id-tokens) avant de supposer que son contenu est digne de confiance.

<div id="renew-id-tokens">
  ## Renouveler les jetons d’identité
</div>

Par défaut, un jeton d’identité est valide pendant 36 000 secondes (10 heures). En cas de préoccupations liées à la sécurité, vous pouvez [réduire la durée de validité du token](/docs/fr-CA/secure/tokens/id-tokens/update-id-token-lifetime), mais n&#39;oubliez pas que l&#39;un des objectifs de ce token est d&#39;améliorer les performances grâce à la mise en cache des renseignements sur l&#39;utilisateur.

Une fois un jeton d’identité expiré, vous pourriez vouloir le renouveler. Pour renouveler le jeton d’identité, vous pouvez soit authentifier de nouveau l&#39;utilisateur avec Auth0, soit utiliser un [refresh token](/docs/fr-CA/secure/tokens/refresh-tokens).
