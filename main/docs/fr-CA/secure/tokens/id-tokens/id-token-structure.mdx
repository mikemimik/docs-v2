title: Structure du jeton d’identité
---

Les <Tooltip tip="Jeton d’identification : information d’identification destinée au client lui-même, plutôt que pour accéder à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+tokens">jetons d’identité</Tooltip> suivent la norme [JSON Web Token (JWT)](/docs/fr-CA/secure/tokens/json-web-tokens), ce qui signifie que leur structure de base correspond à la [structure JWT](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-structure) typique et qu’ils contiennent des [claims JWT](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims) standard sur le jeton lui-même.

Au-delà de ce qui est requis pour <Tooltip tip="Jeton d’identification : information d’identification destinée au client lui-même, plutôt que pour accéder à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JWT">JWT</Tooltip>, les jetons d’identité contiennent aussi des claims sur l’utilisateur authentifié, qui sont prédéfinis par le protocole <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les informations de connexion." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) et sont donc appelés claims OIDC standard. Voici quelques claims OIDC standard :

* `name`
* `nickname`
* `picture`
* `email`
* `email_verified`

Pour obtenir la liste complète des claims OIDC standard, consultez la [spécification OIDC : Claims standard](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Vous pouvez aussi créer des [claims personnalisés](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims), c’est-à-dire des claims que vous définissez, contrôlez et ajoutez à un jeton à l’aide d’Auth0 Actions.

Vous contrôlez les claims OIDC inclus dans le jeton d’identité utilisé par votre application en incluant des [scopes OpenID Connect](/docs/fr-CA/get-started/apis/scopes/openid-connect-scopes) précis dans un paramètre lorsque vous demandez des jetons pendant l’authentification des utilisateurs. Pour savoir comment demander un jeton d’identité, consultez [Obtenir des jetons d’identification](/docs/fr-CA/secure/tokens/id-tokens/get-id-tokens).

<div id="sample-id-token">
  ## Exemple de jeton d’identité
</div>

Cet exemple montre le contenu d’un jeton d’identité. Remarquez que la valeur <Tooltip tip="Audience : Identifiant unique du destinataire prévu d’un jeton émis. Nommée aud dans un jeton, sa valeur contient l’ID d’une application (Client ID) pour un jeton d’ID ou d’une API (identifiant de l’API) pour un jeton d’accès." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip> (située dans la claim `aud`) correspond à l’identifiant de l’application, ce qui signifie que seule cette application en particulier doit consommer le jeton. Pour en savoir plus sur les claims de cet exemple, consultez [JSON Web Token Claims](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims).

```json lines
{
  "iss": "http://my-domain.auth0.com",
  "sub": "auth0|123456",
  "aud": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "name": "Jane Doe",
  "given_name": "Jane",
  "family_name": "Doe",
  "gender": "female",
  "birthdate": "0000-10-31",
  "email": "janedoe@example.com",
  "picture": "http://example.com/janedoe/me.jpg"
}
```
