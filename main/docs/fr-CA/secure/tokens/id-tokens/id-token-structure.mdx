title: Structure du jeton d’identité
---

<Tooltip tip="Jeton d’identité : information d’identification destinée au client lui-même, plutôt qu’à l’accès à une ressource." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=ID+tokens">Les jetons d’identité</Tooltip> suivent la norme [JSON Web Token (JWT)](/docs/fr-CA/secure/tokens/json-web-tokens), ce qui signifie que leur structure de base correspond à la [structure JWT](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-structure) habituelle et qu’ils contiennent des [claims JWT](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims) standard portant sur le jeton lui-même.

En plus de ce qui est requis pour <Tooltip tip="Jeton d’identité : information d’identification destinée au client lui-même, plutôt qu’à l’accès à une ressource." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=JWT">JWT</Tooltip>, les jetons d’identité contiennent aussi des claims concernant l’utilisateur authentifié. Ces claims sont prédéfinis par le protocole <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les renseignements de connexion." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) et sont donc appelés claims OIDC standard. Voici quelques claims OIDC standard :

* `name`
* `nickname`
* `picture`
* `email`
* `email_verified`

Pour obtenir la liste complète des claims OIDC standard, consultez [OIDC specification: Standard Claims](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Vous pouvez aussi créer des [claims personnalisés](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims), c’est-à-dire des claims que vous définissez, contrôlez et ajoutez à un jeton à l’aide d’Auth0 Actions.

Vous contrôlez quels claims OIDC sont inclus dans le jeton d’identité utilisé par votre application en incluant des [OpenID Connect Scopes](/docs/fr-CA/get-started/apis/scopes/openid-connect-scopes) précis dans un paramètre lorsque vous demandez des jetons pendant l’authentification des utilisateurs. Pour savoir comment demander un jeton d’identité, consultez [Get ID Tokens](/docs/fr-CA/secure/tokens/id-tokens/get-id-tokens).

<div id="sample-id-token">
  ## Exemple de jeton d’identité
</div>

Cet exemple montre le contenu d’un jeton d’identité. Remarquez que la valeur <Tooltip tip="Audience : identificateur unique de l’audience d’un jeton émis. Nommée aud dans un jeton, sa valeur contient l’ID d’une application (Client ID) pour un jeton d’identité, ou d’une API (identificateur d’API) pour un jeton d’accès." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip> (située dans le claim `aud`) correspond à l’identificateur de l’application, ce qui signifie que seule cette application précise doit consommer le jeton. Pour en savoir plus sur les claims de cet exemple, consultez [Claims de JSON Web Token](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-claims).

```json lines
{
  "iss": "http://my-domain.auth0.com",
  "sub": "auth0|123456",
  "aud": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "name": "Jane Doe",
  "given_name": "Jane",
  "family_name": "Doe",
  "gender": "female",
  "birthdate": "0000-10-31",
  "email": "janedoe@example.com",
  "picture": "http://example.com/janedoe/me.jpg"
}
```
