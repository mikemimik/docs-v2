title: Jetons d’accès du fournisseur d’identité
---

<Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+providers">Fournisseurs d’identité</Tooltip> émettent des <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+tokens">jetons d’accès</Tooltip> tiers après l’authentification des utilisateurs auprès de ce fournisseur. Vous pouvez utiliser ces jetons d’accès pour appeler l’API du fournisseur tiers qui les a émis. Par exemple, vous pouvez utiliser un jeton d’accès émis après l’authentification d’un utilisateur avec Facebook pour appeler la Facebook Graph API.

Pour les utilisateurs individuels, les jetons du fournisseur d’identité sont disponibles dans le tableau `identities` de l’objet `user`, sous l’élément correspondant à la connection en question. Pour accéder de façon sécurisée aux jetons d’un utilisateur précis, vous avez besoin d’un [jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) qui inclut la portée `read:user_idp_tokens`. Vous pouvez ensuite effectuer un appel HTTP GET vers le [point de terminaison Get a User](https://auth0.com/docs/api/management/v2/#!/Users/get_users_by_id) pour récupérer les jetons.

Pour les administrateurs des Enterprise connections configurées à l’aide d’un flux de consentement <Tooltip tip="OAuth 2.0 : cadre d’autorisation qui définit les protocoles et les flux d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>, les jetons du fournisseur d’identité sont disponibles dans l’objet `connection`. Pour accéder de façon sécurisée aux jetons d’une connection, vous avez besoin d’un [jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) qui inclut la portée `read:connections`. Vous pouvez ensuite effectuer un appel HTTP GET vers le [point de terminaison Get a Connection](https://auth0.com/docs/api/management/v2/connections/get-connections-by-id) pour récupérer les jetons.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le contenu des jetons d’accès tiers varie selon le fournisseur d’identité émetteur. Comme les jetons sont créés et gérés par un tiers (comme Facebook, GitHub, etc.), leur période de validité varie selon l’émetteur. Si vous pensez que ces jetons ont été compromis, vous devez les révoquer ou les réinitialiser auprès du tiers qui les a émis.
</Callout>

<div id="renew-third-party-tokens">
  ## Renouveler les jetons tiers
</div>

Il n’existe pas de méthode normalisée pour renouveler les jetons d’accès du fournisseur d’identité par l’intermédiaire d’Auth0. Le mécanisme de renouvellement de ces jetons d’accès varie selon le fournisseur. Pour certains fournisseurs d’identité, Auth0 peut stocker un [jeton d’actualisation](/docs/fr-CA/secure/tokens/refresh-tokens), que vous pouvez utiliser pour obtenir un nouveau jeton d’accès auprès du fournisseur d’identité. Voici une liste de quelques fournisseurs d’identité :

* BitBucket
* Google OAuth 2.0 (transmettez le paramètre `access_type=offline`, ainsi que le paramètre `connection_scope` avec les scopes requis, lors de l’appel au point de terminaison Auth0 `/authorize`)
* Tout autre fournisseur d’identité OAuth 2.0
* SharePoint
* Azure AD

Vous pouvez récupérer des <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> de la même manière que vous récupérez les jetons d’accès. Consultez les indications ci-dessus pour en savoir plus.

<div id="validate-third-party-tokens">
  ## Valider les jetons tiers
</div>

En règle générale, si vous avez reçu un jeton d’accès d’un fournisseur d’identité, vous n’avez pas besoin de le valider. Vous pouvez le transmettre à l’émetteur, qui s’occupe du reste.
