title: Obtenir des jetons d’accès pour la Management API à des fins de test
---

<Warning>
  Cette méthode pour obtenir des jetons d’accès est **réservée aux tests**. N’obtenez pas manuellement de jetons à longue durée de vie et ne les utilisez pas dans vos applications, car cela annule les avantages de sécurité qu’offrent les jetons.
</Warning>

La première fois que vous obtenez un jeton de test pour la <Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>, c’est lorsque vous terminez la configuration dans le <Tooltip tip="Auth0 Dashboard : Le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Vous n’aurez pas à refaire cette étape, sauf si vous créez un nouveau tenant. Nous vous recommandons de créer un jeton de test uniquement pour autoriser l’accès à la Management API, au lieu de réutiliser un autre jeton que vous pourriez déjà avoir.

<div id="prerequisites">
  ## Prérequis
</div>

* [Créer des applications machine à machine à des fins de test](/docs/fr-CA/get-started/apis/create-m2m-app-test)

<div id="get-access-tokens-manually">
  ## Obtenir des jetons d’accès manuellement
</div>

1. Dans votre Auth0 Dashboard, accédez à **Applications &gt; APIs &gt;**  [Auth0 Management API](https://manage.auth0.com/#/apis/management/explorer). Sélectionnez l’onglet **API Explorer** et repérez un jeton généré automatiquement dans la section **Token**.
2. Sélectionnez l’icône **Copy** à droite du jeton. Vous pouvez maintenant effectuer des appels autorisés à la [Management API](https://auth0.com/docs/api/management/v2) à l’aide de ce jeton.

   <Frame>![APIs - Management API access tokens - Testing - Copy token](/docs/images/cdy7uua7fh8z/3Y4I9YJR77j73FCYDiegnC/44f32f76d27357bc39aad9ee7c4d5561/management-api-access-token.png)</Frame>
3. Au besoin, définissez le délai d’expiration. Par défaut, le jeton a un délai d’expiration de 86400 secondes, soit 24 heures. Après cette période, le jeton expire et ne peut plus être utilisé. Pour modifier le délai d’expiration, mettez à jour **Token Expiration (Seconds)** et cliquez sur **Update &amp; Regenerate Token**.

<Warning>
  Ces jetons **ne peuvent pas être révoqués**; les longs délais d’expiration ne sont donc pas recommandés. Nous vous recommandons plutôt d’utiliser de courts délais d’expiration et d’émettre un nouveau jeton chaque fois que vous en avez besoin. Vous pouvez vérifier l’expiration à l’aide de [jwt.io](https://jwt.io/) pour décoder le jeton.
</Warning>

<div id="use-access-tokens-for-testing">
  ## Utiliser des jetons d’accès pour les tests
</div>

Vous pouvez utiliser le <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> que vous venez de créer à des fins de test pour appeler manuellement un endpoint au moyen du [Management API Explorer](https://auth0.com/docs/api/management/v2).

1. Accédez au [Management API Explorer](https://auth0.com/docs/api/management/v2#!), puis sélectionnez le bouton **Set API Token**.
2. Remplissez le champ **API Token**. Des renseignements supplémentaires s’affichent ensuite dans la fenêtre contextuelle, comme les scopes accordés à l’application.
3. Sélectionnez **Set Token** pour enregistrer votre jeton et fermer la fenêtre contextuelle. Le bouton **Set API Token** affiche maintenant **Authorized**.
4. Pour commencer les tests, accédez à l’endpoint que vous souhaitez appeler et saisissez les paramètres requis ou facultatifs dans les champs à droite. Sélectionnez ensuite **Test Endpoint**.
