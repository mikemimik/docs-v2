---
description: Un JSON Web Key Set est un objet JSON qui représente un ensemble de JSON Web Keys (des objets JSON représentant des clés cryptographiques).
title: JSON Web Key Sets
---

Le JSON Web Key Set (JWKS) est un ensemble de clés qui contient les clés publiques servant à vérifier tout <Tooltip tip="JSON Web Token (JWT) : format standard de ID Token (et souvent de Access Token) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Token+%28JWT%29">JSON Web Token (JWT)</Tooltip> émis par le <Tooltip tip="Authorization Server : serveur centralisé qui aide à définir les limites de l’accès d’un utilisateur. Par exemple, votre Authorization Server peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Authorization+Server">Authorization Server</Tooltip> et signé à l’aide de l’[algorithme de signature](/docs/fr-CA/get-started/applications/signing-algorithms) RS256.

Lors de la création d’applications et d’API dans Auth0, deux algorithmes sont pris en charge pour signer les <Tooltip tip="Authorization Server : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre authorization server peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JWTs">JWTs</Tooltip> : **RS256** et **HS256**. RS256 génère une signature asymétrique, ce qui signifie qu’une clé privée doit être utilisée pour signer le JWT et qu’une autre clé publique doit être utilisée pour en vérifier la signature.

Auth0 utilise la [spécification JSON Web Key (JWK)](https://tools.ietf.org/html/rfc7517) pour représenter les clés cryptographiques servant à signer les jetons RS256. Cette spécification définit deux structures de données de haut niveau : **JSON Web Key (JWK)** et **JSON Web Key Set (JWKS)**. Voici les définitions tirées de la spécification :

| Élément                     | Description                                                                                                                              |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **JSON Web Key (JWK)**      | Un objet JSON qui représente une clé cryptographique. Les membres de l’objet représentent les propriétés de la clé, y compris sa valeur. |
| **JSON Web Key Set (JWKS)** | Un objet JSON qui représente un ensemble de JWK. L’objet JSON DOIT avoir un membre `keys`, qui est un tableau de JWK.                    |

Auth0 expose un endpoint JWKS pour chaque tenant, à l’adresse `https://{yourDomain}/.well-known/jwks.json`. Cet endpoint contient le JWK utilisé pour vérifier tous les JWT émis par Auth0 pour ce tenant.

<Warning>
  À l’heure actuelle, Auth0 ne signe qu’avec un seul JWK à la fois ; toutefois, il est important de partir du principe que cet endpoint pourrait contenir plusieurs JWK. Par exemple, plusieurs clés peuvent se trouver dans le JWKS lors de la [rotation des clés de signature d’application](/docs/fr-CA/get-started/tenant-settings/signing-keys/rotate-signing-keys).
</Warning>
