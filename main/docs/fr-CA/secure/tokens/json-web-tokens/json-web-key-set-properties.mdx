Chaque propriété de la clé est définie par la spécification JWK [RFC 7517 Section 4](https://tools.ietf.org/html/rfc7517#section-4) ou, pour les propriétés propres à un algorithme, par la [RFC 7518](https://tools.ietf.org/html/rfc7518).

| Nom de la propriété | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `alg`               | L’algorithme cryptographique précis utilisé avec la clé. Il s’agit d’un paramètre facultatif. Par défaut, Auth0 inclut dans le jeu de clés Web JSON (JWKS) l’algorithme de signature défini au niveau du tenant, qui est ensuite publié. Pour permettre aux clés d’être utilisées avec plusieurs algorithmes plutôt qu’avec un seul, par exemple RS256, désactivez la **bascule Include Signing Algorithms in jeu de clés Web JSON** dans les paramètres avancés du tenant du Dashboard. Cela supprime le paramètre `alg` et oblige les consommateurs du JWKS à interpréter les algorithmes de signature selon leurs besoins. |
| `kty`               | La famille d’algorithmes cryptographiques utilisée avec la clé.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `use`               | L’utilisation prévue de la clé ; `sig` représente la signature.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `x5c`               | La chaîne de certificats x.509. La première entrée du tableau est le certificat à utiliser pour la vérification du token ; les autres certificats peuvent servir à vérifier ce premier certificat.                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `n`                 | Le module de la [clé publique RSA](https://tools.ietf.org/html/rfc7518#page-30).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `e`                 | L’exposant de la [clé publique RSA](https://tools.ietf.org/html/rfc7518#page-30).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `kid`               | L’identifiant unique de la clé.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `x5t`               | L’empreinte du certificat x.509 (empreinte SHA-1).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 prend uniquement en charge HMAC et RSA ; il ne prend pas actuellement en charge le chiffrement à courbe elliptique (ECDSA).
</Callout>
