<div id="management-api-endpoints">
  ## Points de terminaison pour la Management API
</div>

La <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> d’Auth0 fournit des points de terminaison pour gérer les jetons d’actualisation individuellement ou en tant que collection.

Ces points de terminaison complètent la [révocation alternative des jetons d’actualisation](/docs/fr-CA/secure/tokens/refresh-tokens/revoke-refresh-tokens) et remplacent les points de terminaison de la ressource `device-credentials` en offrant des propriétés étendues et des opérations de révocation en bloc.

<div id="refresh-token-resource">
  ### Ressource de jeton d’actualisation
</div>

Vous pouvez consulter ou révoquer un jeton d’actualisation précis à l’aide des points de terminaison suivants :

| Nom                                                          | URL                                       | Portée(s) requise(s)    |
| ------------------------------------------------------------ | ----------------------------------------- | ----------------------- |
| Consulter les informations d’un jeton d’actualisation par ID | `GET /api/v2/refresh-tokens/{tokenId}`    | `read:refresh_tokens`   |
| Révoquer un jeton d’actualisation par ID                     | `DELETE /api/v2/refresh-tokens/{tokenId}` | `delete:refresh_tokens` |

<div id="user-resource">
  ### Ressource utilisateur
</div>

Vous pouvez lister ou révoquer tous les jetons d’actualisation d’un utilisateur donné à l’aide des points de terminaison suivants :

| Nom                                                       | URL                                            | Portée(s) requise(s)    |
| --------------------------------------------------------- | ---------------------------------------------- | ----------------------- |
| Lister tous les jetons d’actualisation d’un utilisateur   | `GET /api/v2/users/{userId}/refresh-tokens`    | `read:refresh_tokens`   |
| Révoquer tous les jetons d’actualisation d’un utilisateur | `DELETE /api/v2/users/{userId}/refresh-tokens` | `delete:refresh_tokens` |

<div id="refresh-token-properties">
  ## Propriétés du jeton d’actualisation
</div>

Les <Tooltip tip="Point de terminaison de jeton : point de terminaison sur le serveur d’autorisation utilisé pour demander des jetons par programmation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=token+endpoints">points de terminaison de jeton</Tooltip> du jeton d’actualisation renvoient des renseignements pertinents sur le jeton et son historique.

| Champ                                                      | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ---------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID du jeton d’actualisation                                | Un identifiant unique et propre au tenant du jeton d’actualisation dans l’API. Il ne s’agit pas du jeton lui-même, qui est un secret à échanger avec le point de terminaison de jeton de l’Authentication API.<br /><br />Les jetons d’actualisation peuvent rester valides au-delà d’une session. Le jeton d’actualisation conserve ce renseignement, peu importe le cycle de vie de la session.                                                                                                                                                                                                                                                      |
| ID de session                                              | L’ID de session est un identifiant persistant de la session dans le tenant Auth0. Notez que l’ID de session correspond à la claim `sid` déjà présente dans les ID Tokens et les Logout Tokens, et peut être utilisée pour établir des correspondances entre ces entités.<br /><br />Le jeton d’actualisation contient des références à la session qui a servi à créer le jeton. L’ID de session peut être utilisée pour récupérer des renseignements supplémentaires au moyen du point de terminaison de la Management API [inspecter une session précise par ID](/docs/fr-CA/manage-users/sessions/manage-user-sessions-with-auth0-management-api). |
| Dates pertinentes                                          | Renseignements sur la création et l’expiration du jeton d’actualisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Renseignements sur le client et les serveurs de ressources | Renseignements sommaires sur les applications et les API associées à un jeton d’actualisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

<div id="limitations">
  ## Limites
</div>

* Les opérations de révocation de jetons d’actualisation (`DELETE`) s’exécutent de façon asynchrone et finissent par devenir cohérentes.
* Les jetons d’actualisation émis le 21-09-2023 ou après cette date (le 22-02-2024 pour les tenants de la région US-3) contiennent la propriété d’ID de session (`session_id`) avec la valeur appropriée. Les jetons d’actualisation émis avant cette date contiennent cette propriété avec une valeur `null`.
