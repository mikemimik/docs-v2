<div id="management-api-endpoints">
  ## Points de terminaison de la Management API
</div>

L’Auth0 <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> fournit des points de terminaison pour gérer les jetons d’actualisation individuellement ou sous forme de collection.

Ces points de terminaison complètent les [autres méthodes de révocation des jetons d’actualisation](/docs/fr-CA/secure/tokens/refresh-tokens/revoke-refresh-tokens) et remplacent les points de terminaison de la ressource `device-credentials` en y ajoutant des propriétés étendues et des opérations de révocation en bloc.

<div id="refresh-token-resource">
  ### Ressource de jeton d’actualisation
</div>

Vous pouvez consulter ou révoquer un jeton d’actualisation précis à l’aide des points de terminaison suivants :

| Nom                                                            | URL                                       | Portée(s) requise(s)    |
| -------------------------------------------------------------- | ----------------------------------------- | ----------------------- |
| Consulter les renseignements d’un jeton d’actualisation par ID | `GET /api/v2/refresh-tokens/{tokenId}`    | `read:refresh_tokens`   |
| Révoquer un jeton d’actualisation par ID                       | `DELETE /api/v2/refresh-tokens/{tokenId}` | `delete:refresh_tokens` |

<div id="user-resource">
  ### Ressource utilisateur
</div>

Vous pouvez lister ou révoquer tous les jetons d’actualisation d’un utilisateur donné à l’aide des points de terminaison suivants :

| Nom                                                       | URL                                            | Portée(s) requise(s)    |
| --------------------------------------------------------- | ---------------------------------------------- | ----------------------- |
| Lister tous les jetons d’actualisation d’un utilisateur   | `GET /api/v2/users/{userId}/refresh-tokens`    | `read:refresh_tokens`   |
| Révoquer tous les jetons d’actualisation d’un utilisateur | `DELETE /api/v2/users/{userId}/refresh-tokens` | `delete:refresh_tokens` |

<div id="refresh-token-properties">
  ## Propriétés du jeton d’actualisation
</div>

Les <Tooltip tip="Token Endpoint : endpoint sur l'Authorization Server utilisé pour demander des jetons par programmation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=token+endpoints">token endpoints</Tooltip> de refresh renvoient des renseignements pertinents sur le token et son historique.

| Champ                                                      | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID du jeton d’actualisation                                | Identifiant unique et propre au tenant du jeton d’actualisation dans l’API. Il ne s’agit pas du token lui-même, mais d’un secret à échanger avec le token endpoint dans l’Authentication API.<br /><br />Les jetons d’actualisation peuvent survivre à une session. Le jeton d’actualisation conserve cette information, peu importe le cycle de vie de la session.                                                                                                                                                                                                                                               |
| ID de session                                              | L’ID de session est un identifiant persistant de la session dans l’Auth0 tenant. Notez qu’il correspond au claim `sid`, déjà présent dans les ID Tokens et les Logout Tokens, et qu’il peut servir à établir des renvois croisés entre ces entités.<br /><br />Le jeton d’actualisation contient des références à la session utilisée pour créer le token. L’ID de session peut servir à récupérer des renseignements supplémentaires au moyen de l’endpoint du Management API [introspect a specific session by ID](/docs/fr-CA/manage-users/sessions/manage-user-sessions-with-auth0-management-api). |
| Heure pertinente                                           | Renseignements sur la création du jeton d’actualisation et sur son expiration.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Renseignements sur le client et les serveurs de ressources | Renseignements sommaires sur les applications et les API associées à un jeton d’actualisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

<div id="limitations">
  ## Limitations
</div>

* Les opérations de révocation de jeton d’actualisation (`DELETE`) s’exécutent de manière asynchrone et finissent par devenir cohérentes.
* Les jetons d’actualisation émis à compter du 21-09-2023 (22-02-2024 pour les tenants de la région US-3) contiennent la propriété d’ID de session (`session_id`) avec la valeur appropriée. Les jetons d’actualisation émis avant cette date contiennent cette propriété avec une valeur `null`.
