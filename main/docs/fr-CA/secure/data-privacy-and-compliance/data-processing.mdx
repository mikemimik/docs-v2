Ce document présente les données que détient Auth0, ainsi que la façon dont elles sont traitées.

<div id="data-auth0-possesses">
  ## Données détenues par Auth0
</div>

Toutes les données qu’Auth0 possède au sujet d’un utilisateur final se trouvent dans le profil utilisateur Auth0. Les attributs précis contenus dans le profil utilisateur varient selon la mise en œuvre du client et dépendent d’un certain nombre de facteurs, comme le type de connexion, le consentement de l’utilisateur pendant le flux d’authentification et le fait que vous ayez ou non enrichi les profils utilisateur avec des renseignements supplémentaires.

<Warning>
  Les métadonnées Auth0 ne constituent pas un stockage de données sécurisé et ne doivent pas être utilisées pour stocker des renseignements sensibles, comme des secrets à risque élevé et des renseignements personnels identifiables (PII), tels que des numéros d’assurance sociale ou de carte de crédit. Les clients d’Auth0 sont fortement encouragés à évaluer les données stockées dans les métadonnées et à n’y conserver que celles qui sont nécessaires aux fins de la gestion des identités et des accès.
</Warning>

<div id="when-data-is-stored">
  ## Quand les données sont stockées
</div>

Les renseignements du profil utilisateur Auth0 sont stockés dans Auth0 lorsque vous utilisez une connexion de base de données. Si un utilisateur se connecte à l’aide de tout autre type de connexion (y compris des connexions de base de données personnalisées), Auth0 stocke les renseignements fournis par le <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> externe pour de futures requêtes.

<div id="how-data-is-used">
  ## Utilisation des données
</div>

Les données personnelles stockées dans Auth0 sont utilisées uniquement pour fournir ses services, notamment l’authentification des utilisateurs

<div id="how-can-i-delete-my-data">
  ## Comment puis-je supprimer mes données ?
</div>

En tant que responsable du traitement, le client est chargé de gérer et de supprimer les données au moyen des fonctionnalités libre-service, notamment :

* [Supprimer ou réinitialiser des tenants](/docs/fr-CA/troubleshoot/customer-support/manage-subscriptions/delete-or-reset-tenant) - Utilisez cette fonctionnalité pour supprimer un tenant. Lorsque vous supprimez un tenant, le système supprime définitivement toutes les données du client qui y sont associées.
* [Gérer les utilisateurs à l’aide de la Management API](/docs/fr-CA/manage-users/user-accounts/manage-users-using-the-management-api) - Utilisez la Management API pour récupérer, créer, mettre à jour ou supprimer des utilisateurs.
* [Gérer les utilisateurs dans Dashboard](/docs/fr-CA/manage-users/user-accounts/manage-users-using-the-dashboard) - Utilisez le Dashboard pour créer, afficher, modifier ou supprimer manuellement des utilisateurs.

<div id="when-is-customer-data-automatically-deleted">
  ## Quand les données du client sont-elles supprimées automatiquement?
</div>

Auth0 conserve les données du client tant que le client est considéré comme actif.

Auth0 considère un client comme inactif lorsqu’il remplit les deux conditions suivantes :

1. Le client utilise un plan Free (y compris ceux qui ont choisi le niveau Free ou qui y ont été rétrogradés).

2. Tous les tenants associés ne montrent aucune activité pendant une période continue de 150 jours.

Après 150 jours d’inactivité, le système prévoit la suppression définitive des tenants du client inactif et de toutes les données associées.
