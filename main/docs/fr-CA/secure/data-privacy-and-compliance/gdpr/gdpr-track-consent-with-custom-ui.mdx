Dans ce tutoriel, nous verrons comment utiliser auth0.js ou les API Auth0 pour demander des renseignements sur le consentement et enregistrer les données saisies dans les métadonnées de l’utilisateur. Pour en savoir plus, consultez [Comprendre le fonctionnement des métadonnées dans les profils d’utilisateur](/docs/fr-CA/manage-users/user-accounts/metadata).

Si vous souhaitez plutôt faire le suivi du consentement à l’aide de Lock, consultez [GDPR : Suivre le consentement avec Lock](/docs/fr-CA/secure/data-privacy-and-compliance/gdpr/gdpr-track-consent-with-lock).

Il vous incombe en dernier ressort de comprendre le GDPR et de vous y conformer, bien qu’Auth0 vous aide à respecter les exigences du GDPR dans la mesure du possible.

<div id="overview">
  ## Aperçu
</div>

Nous recueillerons les renseignements de consentement dans différents scénarios et les enregistrerons dans les métadonnées de l’utilisateur.

Tous les scénarios enregistreront les propriétés suivantes dans les métadonnées de l’utilisateur :

* `consentGiven` (vrai/faux) indique si l’utilisateur a donné son consentement (vrai) ou non (faux)
* `consentTimestamp` (horodatage Unix) indique à quel moment l’utilisateur a donné son consentement

Par exemple :

```json lines
{
  "consentGiven": "true"
  "consentTimestamp": "1525101183"
}
```

Nous verrons quatre mises en œuvre différentes pour cela :

1. une qui affiche une case à cocher, fonctionne avec les connexions de base de données et utilise la bibliothèque auth0.js pour créer l’utilisateur (utilisée par les applications à page unique). Pour en savoir plus, consultez la [référence Auth0.js](/docs/fr-CA/libraries/auth0js).
2. une qui affiche une case à cocher, fonctionne avec les connexions de base de données et utilise l’Authentication API pour créer l’utilisateur (utilisée par les applications Web traditionnelles)
3. une qui affiche une case à cocher, fonctionne avec les connexions sociales et utilise la Management API pour mettre à jour les renseignements de l’utilisateur (utilisée avec des SPA ou des applications Web traditionnelles)
4. une qui redirige vers une autre page où il est possible de consulter les Conditions générales et/ou la politique de confidentialité, puis de donner son consentement (utilisée avec des SPA ou des applications Web traditionnelles)

<div id="option-1-use-auth0js">
  ## Option 1 : Utiliser auth0.js
</div>

Dans cette section, nous utiliserons une simple application à page unique et personnaliserons le widget de connexion pour ajouter un indicateur permettant aux utilisateurs de fournir des renseignements de consentement. Au lieu de créer une application à partir de zéro, nous allons [utiliser l’exemple JavaScript du Quickstart d’Auth0](/docs/fr-CA/quickstart/spa/vanillajs). Nous utiliserons aussi les pages de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> d’Auth0 pour mettre en œuvre Universal Login, au lieu d’intégrer la connexion à notre application. Pour en savoir plus sur Universal Login, consultez [Auth0 Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login). Pour en savoir plus sur les différences entre Universal Login et la connexion intégrée, consultez [Centralized Universal Login vs. Embedded Login](/docs/fr-CA/authenticate/login/universal-vs-embedded-login).

Cela fonctionne **uniquement** pour les database connections (nous utiliserons l’infrastructure d’Auth0 au lieu de configurer notre propre base de données).

1. Allez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et créez une nouvelle application. Choisissez `Single Web Page Applications` comme type. Allez à **Settings** et définissez **Allowed Callback URLs** sur `http://localhost:3000`.

   Ce champ contient l’ensemble des URL vers lesquelles Auth0 est autorisé à rediriger les utilisateurs après leur authentification. Notre application d’exemple s’exécutera à `http://localhost:3000`, donc nous définissons cette valeur.

2. Copiez les valeurs de **Client Id** et **Domain**. Vous en aurez besoin dans un instant.

3. Allez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) et créez une nouvelle connexion. Cliquez sur **Create DB Connection**, donnez un nom à la nouvelle connexion, puis cliquez sur **Save**. Allez à l’onglet **Applications** de la connexion et assurez-vous que votre application nouvellement créée est activée.

4. [Téléchargez l’exemple JavaScript SPA](/docs/fr-CA/quickstart/spa/vanillajs).

5. [Définissez les valeurs de Client ID et de Domain](https://github.com/auth0-samples/auth0-javascript-samples/tree/master/01-Login#set-the-client-id-and-domain).

6. Allez à [Auth0 Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings). Dans l’onglet **Login**, activez le bouton bascule.

7. Dans la liste déroulante **Default Templates**, assurez-vous que `Custom Login Form` est sélectionné. Le code est déjà prérempli.

8. Définissez la valeur de la variable `databaseConnection` sur le nom de la connexion de base de données utilisée par votre application.

   ```javascript lines
   //code simplifié pour plus de clarté
   	var databaseConnection = 'test-db';
   	//code simplifié pour plus de clarté
   ```

9. Pour ajouter un champ pour la métadonnée `consentGiven`, ajoutez une case à cocher au formulaire. Dans notre exemple, nous configurerons la case à cocher pour qu’elle soit cochée par défaut et désactivée, afin que l’utilisateur ne puisse pas la décocher. Vous pouvez adapter cela selon les besoins de votre entreprise.

   ```javascript lines
   //code simplifié pour plus de clarté
       <div class="form-group">
         <label for="name">Je consens au traitement des données</label>
         <input
           type="checkbox"
           id="userConsent"
           checked disabled>
       </div>
       //code simplifié pour plus de clarté
   ```

10. Modifiez la fonction d’inscription afin de définir les métadonnées. Notez que nous définissons la valeur des métadonnées comme une chaîne de caractères de valeur `true`, et non comme une valeur booléenne, et que nous utilisons `toString` pour convertir le nombre en chaîne de caractères. Cela est dû à une restriction du point de terminaison [**Signup** de l’Authentication API](https://auth0.com/docs/api/authentication#signup), qui n’accepte que des chaînes de caractères comme valeurs.

    ```text lines
    //code simplifié pour plus de clarté
        webAuth.redirect.signupAndLogin({
          connection: databaseConnection,
          email: email,
          password: password,
          user_metadata: { consentGiven: 'true', consentTimestamp: Date.now().toString() }
        }, function(err) {
          if (err) displayError(err);
        });
        //code simplifié pour plus de clarté
    ```

11. Pour voir à quoi ressemblera le widget de connexion, cliquez sur l’onglet **Preview**.

<Frame>![Dashboard Branding Universal Login Classic Login Onglet Formulaire de connexion personnalisé](/docs/images/cdy7uua7fh8z/4m3WA0sKMoR0C1KVnVmZ1G/b311bdbc6c48b4910eac49cc8f1b9ba8/2025-02-26_15-17-18.png)</Frame>

1. Pour tester cette configuration, exécutez l’application et accédez à `http://localhost:3000`. Inscrivez-vous en tant que nouvel utilisateur. Ensuite, allez dans [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) et recherchez ce nouvel utilisateur. Accédez à **User Details** et faites défiler la page jusqu’à la section **Metadata**. Dans la zone de texte **user&#95;metadata**, vous devriez voir la métadonnée `consentGiven` définie sur `true`.

Pour savoir comment faire, poursuivez votre lecture : le paragraphe suivant utilise ce point de terminaison.

<div id="option-3-use-the-api-social">
  ## Option 3 : Utiliser l’API (social)
</div>

Si vous utilisez des connexions sociales, vous ne pouvez pas utiliser l’Authentication API pour créer l’utilisateur dans Auth0, puisque ce point de terminaison fonctionne uniquement avec les connexions de base de données.

Vous devez plutôt permettre à l’utilisateur de s’inscrire avec le fournisseur social (ce qui créera un enregistrement utilisateur dans Auth0), puis utiliser la Management API pour mettre à jour les informations de l’utilisateur.

Avant d’appeler la Management API, vous devez obtenir un jeton valide. Pour en savoir plus, consultez [Get Management API Access Tokens for Production](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).

L’article lié utilise le Client Credentials Flow pour obtenir un jeton, ce que vous ne pouvez pas faire à partir d’une application exécutée dans le navigateur. Vous pouvez plutôt utiliser l’Implicit Flow. Pour en savoir plus sur le Client Credentials Flow, consultez [Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow). Pour en savoir plus sur l’Implicit Flow, consultez [Implicit Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post).

Définissez le paramètre de requête **<Tooltip tip="Audience : Identifiant unique de l’audience d’un jeton émis. Appelé aud dans un jeton, sa valeur contient l’ID soit d’une application (Client ID) pour un ID Token, soit d’une API (API Identifier) pour un Access Token." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip>** à `https://YOUR_DOMAIN/api/v2/` et le paramètre **scope** à la portée `create:current_user_metadata`. Vous pouvez utiliser le <Tooltip tip="Access Token : Information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">Access Token</Tooltip> que vous recevrez dans la réponse pour appeler le point de terminaison [**Update User**](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) de la Management API.

Une fois que vous avez un jeton valide, utilisez l’extrait suivant pour mettre à jour les métadonnées de l’utilisateur.

<div id="option-4-redirect-to-another-page">
  ## Option 4 : Rediriger vers une autre page
</div>

Si vous souhaitez afficher plus d’information à l’utilisateur, vous pouvez, au moment de l’inscription, le rediriger vers une autre page pour lui demander son consentement ainsi que tout renseignement supplémentaire, puis le rediriger de nouveau afin de terminer la transaction d’authentication. Vous pouvez le faire au moyen de redirect rules. Cette même règle peut servir à enregistrer l’information de consentement dans les métadonnées de l’utilisateur afin de pouvoir en faire le suivi et d’éviter de redemander le consentement lors de la prochaine connexion. Pour en savoir plus, consultez [Redirect Users from Within Rules](/docs/fr-CA/customize/rules/redirect-users).

Vous devrez héberger ce formulaire, et son URL devra être accessible publiquement. À une étape ultérieure de ce tutoriel, vous devrez fournir à Auth0 l’URL à laquelle le formulaire est accessible.

1. Ajoutez la règle de redirection. Allez à [Auth0 Dashboard &gt; Auth Pipeline &gt; Rules](https://manage.auth0.com/#/rules), puis cliquez sur **Create Rule**. Sous **Rules Templates**, sélectionnez **empty rule**. Remplacez le nom par défaut de la règle, `empty rule`, par un nom plus explicite (p. ex., `Redirect to consent form`).

2. Ajoutez le code Javascript suivant dans l’éditeur de script, puis cliquez sur **Save** pour enregistrer vos modifications.

   ```js lines
   exports.onExecutePostLogin = async (event, api) => {
       const { consentGiven } = event.user.user_metadata || {};

       // rediriger vers le formulaire de consentement si l’utilisateur n’a pas encore donné son consentement
       if (!consentGiven && api.redirect.canRedirect()) {
         const options = {
           query: {
             auth0_domain: `${event.tenant.id}.auth0.com`,
           },
         };
         api.redirect.sendUserTo(event.secrets.CONSENT_FORM_URL, options);
       }
   };

   // si l’utilisateur clique sur 'I agree' dans le formulaire de consentement, enregistrez-le dans son profil afin qu’on ne lui redemande pas
   exports.onContinuePostLogin = async (event, api) => {
     if (event.request.body.confirm === "yes") {
       api.user.setUserMetadata("consentGiven", true);
       api.user.setUserMetadata("consentTimestamp", Date.now());
       return;
     } else {
       return api.access.deny("User did not consent");
     }
   };
   ```

3. Retournez à [Auth0 Dashboard &gt; Auth0 Pipeline &gt; Rules](https://manage.auth0.com/#/rules), puis faites défiler jusqu’au bas de la page jusqu’à la section **Settings**. Créez une paire clé-valeur comme suit :

   1. **Key** : `CONSENT_FORM_URL`
   2. **Value** : `your-consent-form-url.com`

Assurez-vous de fournir l’URL accessible publiquement où se trouve votre formulaire de consentement.

Lorsque vous configurez la redirection vers votre formulaire de consentement pour l’utiliser dans un environnement Production, assurez-vous de consulter [Trusted Callback URLs](https://github.com/auth0/rules/tree/master/redirect-rules/simple#trusted-callback-urls) et [Data Integrity](https://github.com/auth0/rules/tree/master/redirect-rules/simple#data-integrity) en ce qui concerne la sécurité.

Si vous avez besoin d’un consent prompt spécialisé, par exemple pour un consentement parental, vous devrez créer votre propre formulaire de consentement personnalisé. Gardez à l’esprit que les lois varient d’un pays à l’autre.

La configuration est terminée, passons au test !
