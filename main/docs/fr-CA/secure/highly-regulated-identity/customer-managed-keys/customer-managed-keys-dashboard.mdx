title: Configurer Customer Managed Keys avec le Dashboard
---

Auth0 sécurise les secrets et les données de votre tenant à l’aide d’une Auth0 Environment Root Key, au sommet de la hiérarchie de clés du chiffrement par enveloppe. L’Auth0 Environment Root Key et la Customer Provided Root Key sont stockées dans le module de sécurité matériel (HSM) du fournisseur de services infonuagiques Auth0 correspondant, AWS ou Azure.

<div id="bring-your-own-key">
  ## Bring Your Own Key
</div>

Avec Bring Your Own Key, les utilisateurs qui ont le [Key Management Editor role](/docs/fr-CA/get-started/manage-dashboard-access/add-dashboard-users) peuvent utiliser le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> pour remplacer l’Auth0 Environment Root Key par défaut par leur propre Customer Provided Root Key.

Les clients peuvent téléverser de façon sécurisée leur propre Root Key, qui contient leur propre matériel cryptographique, afin de :

* Répondre à des exigences personnalisées en matière de génération et de provenance des clés pour l’Auth0 Environment Root Key.
* Répondre à des exigences particulières d’installation ou de durée de vie des clés pour l’Auth0 Environment Root Key.

<Warning>
  En important votre propre Customer Provided Root Key avec Bring Your Own Key, vous retirez implicitement à Auth0 la gestion du cycle de vie de la Customer Provided Root Key, sauf en ce qui concerne sa suppression.
</Warning>

Pour commencer, accédez à Dashboard &gt; Settings &gt; Encryptions keys

<Frame>![Dashboard > Settings > Clés de chiffrement](/docs/images/cdy7uua7fh8z/1qmfCSl7cOugrHIAdxSyAt/6c7d7185920e61809d9423c8e3d4c4f2/Encryption_Keys_-_EN.png)</Frame>

Sélectionnez **Téléverser une clé** pour lancer le processus d’importation de votre Customer Provided Root Key. Cela ouvre la boîte de dialogue d’importation :

<Frame>![Dashboard > Settings > Clés de chiffrement > Téléverser](/docs/images/cdy7uua7fh8z/1GJPgT1Be7Wm6G6ldCVW4q/96e5a326aa643f29bb50aea76fce27aa/image__2_.png)</Frame>

Lorsque vous sélectionnez **Téléverser une clé**, puis **Télécharger**, cela lance le processus Bring Your Own Key :

1. Crée une clé d’enveloppement publique et la télécharge sur votre système.
2. Prenez la clé d’enveloppement publique et enveloppez votre propre matériel cryptographique avec celle-ci à l’aide de votre propre système de gestion des clés afin de créer une clé de chiffrement encapsulée (la Customer Provided Root Key).
3. Téléversez votre clé de chiffrement encapsulée et sélectionnez **Save**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Une fois téléversée, la clé de chiffrement encapsulée remplace l’Auth0 Environment Root Key dans le module de sécurité matériel (AWS ou Azure) en tant que Customer Provided Root Key.
</Callout>

<div id="cryptographic-material-requirements">
  ## Exigences relatives au matériel cryptographique
</div>

Utilisez votre système de gestion des clés pour encapsuler votre propre matériel cryptographique avec la clé d’enveloppement publique et créer la clé de chiffrement encapsulée. Utilisez les paramètres suivants pour l’algorithme [CKM&#95;RSA&#95;AES&#95;KEY&#95;WRAP](https://docs.oasis-open.org/pkcs11/pkcs11-curr/v2.40/cos01/pkcs11-curr-v2.40-cos01.html#_Toc408226894), selon votre fournisseur de services infonuagiques Auth0 (AWS ou Azure) :

<div id="auth0-on-aws-cloud">
  ### Auth0 sur le cloud AWS
</div>

* Longueur de la clé d’enveloppement publique : 3072 bits
* Algorithme : CKG&#95;MGF1&#95;SHA256
* Longueur de la clé AES temporaire pour CKM&#95;AES&#95;KEY&#95;WRAP&#95;PAD : 256 bits
* Type de la clé racine fournie par le client : clé symétrique AES de 256 bits

<div id="auth0-on-azure-cloud">
  ### Auth0 dans le cloud Azure
</div>

* Longueur de la clé d’enveloppement publique : 2048 bits
* Algorithme : CKG&#95;MGF1&#95;SHA-1
* Longueur de la clé AES temporaire pour CKM&#95;AES&#95;KEY&#95;WRAP&#95;PAD : 256 bits
* Type de Customer Provided Root Key : clé privée RSA de 2048 bits
* Encodage de la clé privée : PKCS #8 - ASN.1 DER
