Auth0 peut grandement simplifier le processus d’authentification des applications SharePoint. Auth0 négociera un <Tooltip tip="Jeton d’accès : information d’autorisation d’identification, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> que vous pourrez ensuite utiliser pour appeler les API SharePoint.

Vous n’aurez besoin d’aucune bibliothèque particulière. Vous pouvez utiliser n’importe lequel des SDKs pris en charge par Auth0.

<div id="step-1-register-your-application-in-auth0">
  ## Étape 1. Enregistrez votre application dans Auth0
</div>

Il suffit d’enregistrer une nouvelle application dans Auth0 comme vous le feriez normalement : **Applications &gt; NEW**. Choisissez l’un des SDKs offerts pour obtenir des instructions détaillées. Gardez le `client_id` à portée de main, car vous en aurez besoin à l’étape suivante.

<div id="step-2-create-a-package-for-your-application">
  ## Étape 2. Créez un package pour votre application
</div>

Vous devez obtenir un **<Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+ID">Client ID</Tooltip>** et un **<Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Secret">Client Secret</Tooltip>** pour votre application. Il existe plusieurs façons d’enregistrer votre application selon l’utilisation prévue.

[Enregistrer des modules complémentaires SharePoint (Microsoft Docs)](https://docs.microsoft.com/en-us/sharepoint/dev/sp-add-ins/register-sharepoint-add-ins) présente les différentes façons d’enregistrer votre application dans SharePoint. Cette étape du tutoriel utilisera la méthode la plus simple : l’enregistrement automatique dans un tenant précis (le vôtre).

<div id="open-sharepoint-online">
  ### Ouvrir SharePoint Online
</div>

L’URL du tableau de bord est `https://{your Office365 tenant}.sharepoint.com/_layouts/15/appregnew.aspx`

<div id="complete-the-information-in-the-form">
  ### Remplissez les renseignements dans le formulaire :
</div>

Comme Auth0 s’interpose entre votre application et l’infrastructure d’Office 365, vous devez utiliser cette URL pour l’application :

**Domaine de l’application** :

export const codeExample1 = `{yourDomain}`;

<AuthCodeBlock children={codeExample1} language="http" />

**URI de redirection** :

export const codeExample2 = `https://{yourDomain}/login/callback?SP_APP_TOKEN&connection=CONNECTION&client_id={yourClientId}&redirect_uri={https://yourApp/callback}`;

<AuthCodeBlock children={codeExample2} language="http" />

* `connection` est simplement le nom que vous utiliserez dans les connexions d’Auth0 (par exemple, &quot;sharepoint&quot;).
* `client_id` identifie votre application dans Auth0 (créée à l’étape 1).
* `redirect_uri` correspond à l’emplacement dans votre application où vos utilisateurs seront finalement redirigés une fois le processus terminé. Si vous ne le précisez pas, ce sera toujours l’URL de rappel de l’application définie dans Auth0 (cela peut être localhost)

<div id="package-the-app-and-upload-to-sharepoint">
  ### Créez le package de l’application et téléversez-le dans SharePoint :
</div>

Remplissez les champs du manifeste de votre application dans Visual Studio :

<Frame>![SharePoint - Visual Studio - Manifeste de l’application](/docs/images/cdy7uua7fh8z/36dscLHlLHZjJx1rLgrQ7N/8d64ec7a51f4e053627ecb5dfd3c7707/90SEc.png)</Frame>

Notez que le `Query string` sera exactement le même que le `URI de redirection` que vous avez indiqué précédemment. Cliquez ensuite avec le bouton droit sur le projet et sélectionnez `Publish` :

Créez un **Publishing Profile** (vous devrez saisir le même **Client Id** et **Client Secret** obtenus dans le tableau de bord de SharePoint).

Cliquez sur **Package** et téléversez le fichier obtenu dans SharePoint.

<div id="step-3-create-the-connection-in-auth0">
  ## Étape 3. Créer la connexion dans Auth0
</div>

La dernière étape de l’intégration consiste à ajouter une connexion SharePoint dans Auth0.

Vous aurez besoin de :

* `Connection Name`. Il s’agit d’un nom arbitraire. Il doit correspondre à celui que vous avez saisi à l’étape 2.
* `Client Id` &amp; `Client Secret`. Ils doivent eux aussi correspondre à ce que vous avez saisi à l’étape 2.
* `Test SharePoint Site Url`. Il s’agit de l’URL du site SharePoint utilisée pour tester la connexion (par exemple, lorsque vous cliquez sur le bouton &#39;Try&#39; dans le Dashboard). Elle n’est jamais utilisée à l’exécution, car les utilisateurs suivront toujours le lien vers votre site depuis SharePoint.

Les utilisateurs installeront votre application à partir de l’Office Marketplace. Lorsqu’ils cliqueront sur le lien, ils seront redirigés vers Auth0, qui négociera le jeton d’accès pour vous, puis vers votre application. Votre application recevra un `User Profile` qui ressemblera à ceci :

<Frame>![Office Marketplace - Redirect -  User Profile Example](/docs/images/cdy7uua7fh8z/4piqwmx1ONWMd3kjTFMq4I/714545f5cda3ab1d1dd5c620bc96b083/8Xp6x.png)</Frame>

Les propriétés suivantes seront incluses : `cacheKey`, `refresh_token`, `host` et `site`. Elles vous permettront d’effectuer des requêtes vers les API de SharePoint (comme les listes).
