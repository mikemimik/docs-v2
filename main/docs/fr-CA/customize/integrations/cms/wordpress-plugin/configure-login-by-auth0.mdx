title: Configure Login by Auth0
---

Login by Auth0 peut être configuré à l’aide du Setup Wizard dans le plugin, ou manuellement si vous souhaitez mieux contrôler le processus. Pour en savoir plus, consultez [Install Login by Auth0](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/install-login-by-auth0). Les instructions ci-dessous peuvent aussi être utiles si le Setup Wizard ne s’est pas terminé correctement ou pour le dépannage de problèmes de connexion.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous devrez être connecté à votre compte Auth0 avant de commencer les étapes ci-dessous. Si vous n’en avez pas encore, [créez-en un](https://auth0.com/signup).
</Callout>

<div id="auth0-configuration">
  ## Configuration d’Auth0
</div>

Votre tenant Auth0 doit être configuré pour accepter les requêtes de connexion provenant de votre site WordPress et utiliser au moins une connexion comme source d’identités utilisateur, qu’il s’agisse d’une base de données Auth0, d’une connexion sociale ou d’un annuaire d’entreprise. Pour en savoir plus sur les tenants, consultez [Create Tenants](/docs/fr-CA/get-started/auth0-overview/create-tenants). Pour en savoir plus sur les connexions, consultez [Identity Providers](/docs/fr-CA/authenticate/identity-providers).

<div id="application-setup">
  ### Configuration de l’Application
</div>

1. Créez une Application pour votre site WordPress :

* **Si vous faites du troubleshooting avec le Setup Wizard**, accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis cherchez une Application dont le nom ressemble à celui de votre site WordPress. Si vous n’en trouvez pas, cela signifie qu’aucune Application n’a été créée par le Wizard. Redémarrez le Setup Wizard dans WordPress ou suivez l’étape ci-dessous pour créer une Application manuellement.
* **Si vous effectuez la configuration manuellement**, accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis sélectionnez **Create**. Entrez un nom pour l’Application, sélectionnez **Regular Web Applications**, puis sélectionnez **Create**.

1. Sélectionnez la vue **Settings** de l’Application. Vous y verrez votre Domain, votre Client ID et votre Client Secret, qui sont utilisés dans **wp-admin &gt; Auth0 &gt; Settings** pour connecter WordPress à Auth0.
2. **Application Type** doit être défini sur **Regular Web Application** et **Token Endpoint Authentication Method** doit être défini sur **Post**
3. Faites défiler jusqu’à **Allowed Callback URLs** et entrez l’URL de votre site WordPress avec `?auth0=1` ajouté :

<Warning>
  L’URL de rappel indiquée ici **ne doit pas** être mise en cache, sinon vous pourriez voir une erreur « Invalid state » pendant le login. Pour en savoir plus, consultez [Résoudre les erreurs Invalid State du WordPress Plugin](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/troubleshoot-wordpress-plugin-invalid-state-errors).
</Warning>

1. Entrez la **WordPress Address (URL)** de votre site WordPress (là où le site est accessible publiquement) et, si elle est différente, la **Site Address (URL)** (à partir de laquelle wp-admin est servi) dans le champ **Allowed Web Origins**. Ces deux valeurs se trouvent dans l’écran des réglages généraux de votre site WordPress.
2. Entrez l’URL de login de votre site WordPress dans le champ **Allowed Logout URLs**
3. Laissez le champ **Allowed Origins (CORS)** vide (il utilisera les valeurs de **Allowed Callback URLs** ci-dessus)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Assurez-vous d’utiliser le bon protocole pour votre site (http ou https) et prenez comme base l’URL d’accueil indiquée dans **wp-admin &gt; Settings &gt; General &gt; WordPress Address (URL)** pour tous les champs d’URL ci-dessus.
</Callout>

1. Faites défiler vers le bas et sélectionnez **Show Advanced Settings**, puis la vue **OAuth**. Assurez-vous que **JsonWebToken Signature Algorithm** est défini sur RS256. Si ce paramètre doit être modifié plus tard, il faudra aussi le modifier ici ainsi que dans wp-admin (voir Settings &gt; Basic ci-dessous).
2. Activez **OIDC Conformant**.
3. Cliquez sur l’onglet **Grant Types** et sélectionnez au minimum **Authorization Code** et **Client Credentials**.
4. Cliquez sur **Save Changes**.

<div id="authorize-the-application-for-the-management-api">
  ### Autoriser l’Application pour la Management API
</div>

Pour que votre site WordPress puisse effectuer certaines actions au nom de votre tenant Auth0, vous devrez autoriser l’Application créée ci-dessus à accéder à la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Ce n’est pas obligatoire, mais cela permettra de récupérer les données utilisateur complètes à la connexion (y compris `user_metadata` et `app_metadata`), de modifier l’adresse courriel et le mot de passe des utilisateurs, et de renvoyer le courriel de vérification lorsque des adresses courriel vérifiées sont requises.

1. Assurez-vous que votre Application autorise le grant **Client Credentials** (étape 10 dans la section ci-dessus).
2. Accédez à [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis).
3. Sélectionnez **Auth0 Management API**, puis la vue **Machine to Machine Applications**.
4. Repérez l’Application WordPress et sélectionnez **Unauthorized** pour accorder l’accès.
5. Dans le panneau qui s’affiche, sélectionnez uniquement les scopes `read:users` et `update:users`, puis sélectionnez **Update** (vous pouvez faire une recherche à l’aide du champ **Filter scopes**).

<div id="database-connection-setup">
  ### Configuration de la Database connexion
</div>

Les Database connexions permettent la connexion avec nom d’utilisateur et mot de passe pour des dossiers d’utilisateurs stockés dans Auth0. Ce type de connexion n’est pas obligatoire et peut être ignoré si vous utilisez uniquement des connexions <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip>, sociales ou d’entreprise.

1. Si vous avez utilisé l’assistant pendant la configuration, accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) et cherchez une connexion dont le nom ressemble à celui de l’Application configurée ci-dessus. Sinon, vous pouvez créer une nouvelle connexion, utiliser une connexion existante ou utiliser la valeur par défaut **Username-Password-Authentication**. Sélectionnez le nom d’une connexion existante pour afficher ses paramètres, ou sélectionnez **Create DB Connection** et suivez les étapes.
2. Sélectionnez la vue **Applications** et activez l’Application créée ci-dessus.

<div id="wordpress-configuration">
  ## Configuration de WordPress
</div>

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et sélectionnez l’Application créée ci-dessus.
2. Dans un nouvel onglet ou une nouvelle fenêtre, connectez-vous à `wp-admin` pour votre site WordPress, puis accédez à **wp-admin &gt; Auth0 &gt; Settings**.
3. Copiez **Domain**, **Client ID** et **Client Secret** depuis la page de votre Auth0 Application vers les paramètres de WordPress à l’aide des boutons **Copy to Clipboard** à côté de chaque champ.
4. Faites défiler la page vers le bas et sélectionnez **Save Changes**.

<div id="php-constant-setting-storage">
  ## Stockage des paramètres à l’aide de constantes PHP
</div>

Les paramètres du plugin peuvent être enregistrés dans la base de données (par défaut) ou définis au moyen d’une constante PHP portant un nom précis. Cela permet de stocker plus sécuritairement des données sensibles comme le <Tooltip tip="Client Secret : secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+secret">secret client</Tooltip>, le jeton d’API et le jeton de migration (à condition que le fichier dans lequel ils sont définis soit lui aussi stocké de façon sécurisée; pour en savoir plus, consultez [Hardening WordPress on wordpress.org](https://developer.wordpress.org/advanced-administration/security/hardening/)).

La constante **doit** être définie avant le chargement du plugin, sinon elle ne sera pas utilisée. Cela doit se faire dans votre fichier `wp-config.php` ou dans un plugin à usage obligatoire. Pour en savoir plus, consultez [Must Use Plugins on wordpress.org](https://developer.wordpress.org/advanced-administration/plugins/mu-plugins/). Si la constante est définie dans le fichier `functions.php` de votre thème ou dans un plugin chargé après Auth0, la valeur sera ignorée.

Les constantes PHP se définissent comme suit :

Le nom de constante par défaut doit être `AUTH0_ENV_` suivi du nom de l’option à remplacer, en majuscules (le préfixe peut être modifié avec le filtre `auth0_settings_constant_prefix`; pour en savoir plus, consultez [Extend Login by Auth0 WordPress Plugin](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/extend-login-by-auth0)). Toutes les options du plugin peuvent être remplacées, et leurs clés se trouvent dans la méthode `WP_Auth0_Options::defaults()`.

**Remarque :** La valeur `migration_token` est générée par le plugin lorsque la migration des utilisateurs est activée. S’il y a déjà une valeur dans l’interface d’administration, assurez-vous de définir la constante à cette même valeur. Si cette valeur doit changer, elle doit aussi être modifiée dans les scripts personnalisés de la connexion de base de données utilisée dans le <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>.

Le champ de paramètre changera son affichage en fonction de cette nouvelle valeur et indiquera la constante utilisée à titre de référence. Cette valeur sera automatiquement utilisée partout dans le plugin.

**Important :** L’enregistrement de la page des paramètres après la définition d’une valeur de constante validera les valeurs définies par constante (sans toutefois les modifier) et les supprimera du tableau d’options enregistré dans la base de données. Si vous ne faites que tester cette fonctionnalité, n’enregistrez pas les paramètres dans la page d’administration WordPress tant que vous n’êtes pas prêt à supprimer cette valeur.

Tous les sites d’un réseau WordPress multisite utiliseront la même valeur de constante, ce qui en fait une façon simple de configurer un réseau à l’aide d’une seule Application et d’une seule connexion de base de données.

<div id="plugin-settings">
  ## Paramètres du plugin
</div>

<div id="basic">
  ### Basic
</div>

* **Domain :** Le Domain copié à partir des Application Settings de votre Dashboard. Le nom de l’option est `domain`.
* **<Tooltip tip="Custom Domain : domaine tiers avec un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Custom+Domain">Custom Domain</Tooltip> :** Le Custom Domain de votre tenant, s’il est configuré. Pour en savoir plus, consultez [Custom Domains](/docs/fr-CA/customize/custom-domains). Le nom de l’option est `custom_domain`.
* **<Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+ID">Client ID</Tooltip> :** Le Client ID copié à partir des Application Settings de votre Dashboard. Le nom de l’option est `client_id`.
* **Client Secret :** Le Client Secret copié à partir des Application Settings de votre Dashboard. Le nom de l’option est `client_secret`.
* **<Tooltip tip="JSON Web Token (JWT) : format standard de ID Token (et souvent de access token) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JWT">JWT</Tooltip> Signature Algorithm** L’algorithme utilisé pour signer les jetons dans les Application Settings avancés, onglet <Tooltip tip="OAuth 2.0 : framework d’autorisation qui définit les protocoles et les workflows d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> ; la valeur par défaut est RS256. Le nom de l’option est `client_signing_algorithm`.
* **JWKS Cache Time (in minutes) :** Durée de conservation des informations JWKS lors de l’utilisation de l’algorithme de signature JWT RS256. Le nom de l’option est `cache_expiration`.
* **Original Login Form on wp-login.php :** Permet d’autoriser ou de bloquer l’accès à la page de login principale de WordPress. Le nom de l’option est `wordpress_login_enabled`. Le nom de l’option du code de la page de login est `wle_code`.

  * **Never** n’autorisera pas l’affichage du formulaire de login principal de WordPress.
  * **Via a link under the Auth0 form** affichera un lien vers le formulaire de login principal de WordPress directement sous celui intégré d’Auth0 sur `wp-login.php`. Il est aussi possible d’accéder directement à la page de login en ajoutant `?wle` à l’URL de login.
  * **When &quot;wle&quot; query parameter is present** autorisera l’accès direct à la page de login en ajoutant `?wle` à l’URL de login. Cela contournera la redirection vers la Universal Login Page.
  * **When &quot;wle&quot; query parameter contains specific code** autorisera l’accès direct à la page de login en ajoutant `?wle=` suivi d’un code à l’URL de login. Le code est généré automatiquement et s’affichera sous les contrôles de ce paramètre. Cela contournera la redirection vers la Universal Login Page.
* **Allow Signups :** L’inscription des utilisateurs sera offerte seulement si l’option WordPress Anyone can register est activée. Vous trouverez ce paramètre sous **Settings &gt; General &gt; Membership**.

<div id="features">
  ### Fonctionnalités
</div>

* **<Tooltip tip="Universal Login : Votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> Page :** Redirige la page `wp-login.php` vers la page Universal Login pour l’authentification par <Tooltip tip="Single Sign-on (SSO) : Service qui, après qu’un utilisateur se connecte à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-on">Single Sign-on</Tooltip> (SSO) en utilisant toutes les connexions actives pour cette application. Le nom de l’option est `auto_login`.
* **Méthode de connexion automatique :** Une seule connexion active à utiliser pour l’authentification lorsque **Universal Login Page** est activée. Laissez ce champ vide pour afficher toutes les connexions actives sur la page Universal Login. Le nom de l’option est `auto_login_method`.
* **Déconnexion Auth0 :** Activez cette option pour vous déconnecter d’Auth0 lorsque vous fermez votre session WordPress. Le nom de l’option est `singlelogout`.
* **Remplacer les avatars WordPress :** Force WordPress à utiliser les avatars d’Auth0. Le nom de l’option est `override_wp_avatars`.

<div id="embedded">
  ### Intégré
</div>

Les options ci-dessous n’ont aucun effet sur la page Universal Login (pour en savoir plus sur les options de personnalisation, consultez [Auth0 Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login)).

* **Connexion Passwordless :** Activez cette option pour activer la connexion Passwordless sur tous les formulaires de connexion Auth0 intégrés. Les connexions Passwordless sont gérées dans le dashboard Auth0, et au moins une doit être active et activée pour cette application afin que cela fonctionne. Le nom de l’option est `passwordless_enabled`.
* **URL de l’icône :** Définit l’icône affichée au-dessus du formulaire de connexion Auth0 intégré. Le nom de l’option est `icon_url`.
* **Titre du formulaire :** Définit le titre du formulaire de connexion Auth0 intégré. Le nom de l’option est `form_title`.
* **Activer l’intégration Gravatar :** Lorsque l’utilisateur saisit son courriel, son image Gravatar associée s’affiche dans le formulaire de connexion Auth0 intégré. Le nom de l’option est `gravatar`.
* **Style du nom de connexion :** En sélectionnant **Email**, les utilisateurs devront saisir leur adresse courriel pour se connecter. Réglez cette option sur **Username** si vous ne voulez pas obliger le nom d’utilisateur à être une adresse courriel valide. Le nom de l’option est `username_style`. Le nom de l’option est `client_secret_b64_encoded`.
* **Couleur principale :** Pour en savoir plus sur ce paramètre, consultez [Lock Configuration Options](/docs/fr-CA/libraries/lock/lock-configuration). Le nom de l’option est `primary_color`.
* **Paramètres supplémentaires :** Un objet JSON valide qui comprend les options à utiliser pour appeler Lock. Cela remplace toutes les autres options définies ci-dessus. Pour obtenir la liste des options offertes, consultez Lock: options configurables par l’utilisateur (p. ex. : `{"disableResetAction": true }`). Le nom de l’option est `extra_conf`.
* **Utiliser une URL JS Lock personnalisée :** Lorsque cette option est désactivée, WordPress utilisera automatiquement la plus récente version testée de Lock (formulaire de connexion Auth0 intégré). Lorsqu’elle est activée, les administrateurs peuvent fournir une URL personnalisée de Lock. Le nom de l’option est `custom_cdn_url`.
* **URL JS Lock personnalisée :** Une URL valide pointant vers une version de Lock. Ce champ sera automatiquement masqué lorsque **Utiliser une URL JS Lock personnalisée** est désactivée. Le nom de l’option est `cdn_url`.
* **connexions à afficher :** Indiquez ici chacun des <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> avec lesquels vous voulez permettre aux utilisateurs de se connecter. Si ce champ est laissé vide, tous les fournisseurs activés seront autorisés. (Pour en savoir plus, consultez [Lock UI Customization](/docs/fr-CA/libraries/lock/lock-ui-customization).) Le nom de l’option est `lock_connections`.

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    Si vous avez activé la connexion Passwordless, vous devez indiquer ici tous les fournisseurs d’identité sociaux autorisés. Pour en savoir plus, consultez [.social(options, callback) dans notre GitHub repository](https://github.com/auth0/lock-passwordless#socialoptions-callback).
  </Callout>

<div id="advanced">
  ### Avancé
</div>

* **Exiger une adresse courriel vérifiée :** Si cette option est activée, l’utilisateur doit avoir une adresse courriel vérifiée pour se connecter. Cela peut empêcher certaines connexions de fonctionner correctement si elles ne fournissent pas d’adresse courriel ni d’indicateur `email_verified` dans les données du profil utilisateur. Le nom de l’option est `requires_verified_email`.
* **Ignorer des stratégies :** Si Exiger une adresse courriel vérifiée est activé, ce paramètre s’affichera. Ce champ accepte des noms de stratégie pour contourner l’exigence d’adresse courriel vérifiée lors de la connexion et de l’association de comptes. Cela ne doit **être utilisé que** pour les stratégies qui ne fournissent pas d’indicateur `email_verified`.
* **Mémoriser la session utilisateur :** Par défaut, les sessions utilisateur durent deux jours. Activez ce paramètre pour garder les sessions utilisateur actives pendant 14 jours. Le nom de l’option est `remember_users_session`.

  * Ce paramètre doit être inférieur ou égal au paramètre [délai d’inactivité](/docs/fr-CA/manage-users/sessions/session-lifetime-limits)docs/manage-users/sessions/session-lifetime-limits de votre tenant afin de garantir le maintien adéquat de la session de l’utilisateur.
* **URL de redirection de connexion :** Si elle est définie, redirige les utilisateurs vers l’URL indiquée après la connexion. Cela n’affecte pas la connexion au moyen du shortcode `[auth0]`. Le nom de l’option est `default_login_redirection`. Pour modifier la redirection du shortcode, ajoutez un attribut `redirect_to`, comme ceci :

  `[auth0 redirect_to="http://yourdomain.com/redirect-here"]`
* **Forcer le callback HTTPS :** Activez cette option si votre site permet HTTPS, mais ne l’impose pas. Cela forcera les callbacks Auth0 à utiliser HTTPS si l’URL d’accueil de votre site n’est pas définie en HTTPS. Le nom de l’option est `force_https_callback`.
* **Approvisionnement automatique :** Les nouveaux utilisateurs provenant d’Auth0 doivent-ils être enregistrés dans la base de données WordPress si les nouvelles inscriptions ne sont pas autorisées ? Cela créera des utilisateurs WordPress qui n’existent pas encore lorsqu’ils se connectent par Auth0 (par exemple, si un utilisateur est créé dans l’Auth0 Dashboard). Le nom de l’option est `auto_provisioning`.

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    Si les inscriptions sont autorisées dans WordPress, de nouveaux utilisateurs seront créés независимоamment de ce paramètre.
  </Callout>
* **Migration des utilisateurs :** L’activation de cette option exposera les services Web de migration Auth0. Cependant, la connexion devra être configurée manuellement dans l’[Auth0 Dashboard](https://manage.auth0.com/#). Pour en savoir plus sur le processus de migration, consultez [User Migration in Login by Auth0 WordPress Plugin](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0). Le bouton **Generate New Migration Token** permet de remplacer le token enregistré par un nouveau. Assurez-vous d’avoir la page de configuration de votre database connexion ouverte à l’onglet **Custom Database** afin de pouvoir remplacer le token existant par le nouveau dans les deux scripts. Le nom de l’option est `migration_ws`. Le nom de l’option du token de migration est `migration_token`.
* **AllowList d’IP de migration :** Seules les requêtes provenant des IP indiquées seront autorisées à accéder au service Web de migration. Le nom de l’option est `migration_ips_filter`.
* **IP proxy valide :** Indiquez l’adresse IP de votre proxy ou de votre répartiteur de charge pour activer les vérifications d’IP pour les connexions et les services Web de migration. Le nom de l’option est `valid_proxy_ip`.
* **Domaine du serveur Auth0 :** Le domaine Auth0 est utilisé par l’assistant de configuration pour récupérer les renseignements de votre compte. Le nom de l’option est `auth0_server_domain`.
