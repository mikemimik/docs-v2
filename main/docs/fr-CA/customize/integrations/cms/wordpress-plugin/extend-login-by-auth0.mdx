---
description: Découvrez comment étendre le Login by Auth0 WordPress Plugin avec des hooks, des filtres et des fonctions.
title: Étendre Login by Auth0 WordPress Plugin
---

Les extensions WordPress peuvent être étendues pour répondre à vos besoins précis à l’aide d’actions et de filtres permettant d’exécuter du code personnalisé à des moments précis de l’exécution. Ce document présente les hooks existants dans l’extension Login by Auth0. Nous sommes heureux d’examiner et d’approuver de nouveaux filtres et de nouvelles actions qui vous aideront à intégrer encore davantage cette extension. Pour en savoir plus, consultez la section Contributing dans le [`README` du GitHub repository de l’extension](https://github.com/auth0/wordpress?tab=readme-ov-file#contributing).

<div id="wordpress-actions">
  ## Actions de WordPress
</div>

Les actions de WordPress exécutent du code personnalisé à des étapes précises du traitement. Pour en savoir plus sur les actions de WordPress, consultez [WordPress Plugin Handbook: Actions on wordpress.org](https://developer.wordpress.org/plugins/hooks/actions/).

<div id="auth0_before_login">
  ### auth0_before_login
</div>

Cette action s’exécute dans `WP_Auth0_LoginManager` une fois qu’un utilisateur a été authentifié avec succès, mais avant qu’il soit connecté à WordPress. Elle peut être utilisée pour interrompre le processus de connexion, au besoin, à l’aide de `wp_die()` ou en levant une exception.

<div id="auth0_user_login">
  ### auth0_user_login
</div>

Cette action s’exécute dans `WP_Auth0_LoginManager` après l’authentification réussie d’un utilisateur et sa connexion à WordPress. Elle peut servir à définir des valeurs de métadonnées précises, à envoyer des notifications ou à interroger d’autres services.

<div id="wpa0_user_created">
  ### wpa0_user_created
</div>

Cette action s’exécute dans `WP_Auth0_Users` juste après la création réussie d’un utilisateur WordPress. Elle peut servir à modifier les données de l’utilisateur, à ajouter des métadonnées utilisateur supplémentaires ou à déclencher d’autres actions associées aux nouveaux utilisateurs.

<div id="filters">
  ## Filtres
</div>

Dans WordPress, les filtres exécutent aussi du code personnalisé à des moments précis du traitement, mais ils renvoient toujours une valeur modifiée du même type que celle qui leur a été passée. Pour en savoir plus sur les filtres, consultez [WordPress Plugin Handbook: Filters on wordpress.org](https://developer.wordpress.org/plugins/hooks/filters/).

<div id="auth0_create_user_data">
  ### auth0_create_user_data
</div>

Ce filtre peut être utilisé pour modifier le tableau de données utilisateur lors de la création d’un nouvel utilisateur dans WordPress à partir d’un profil utilisateur Auth0 entrant. Il reçoit 2 paramètres :

* `$user_data` correspond aux données utilisateur extraites du profil Auth0.
* `$userinfo` correspond au profil utilisateur Auth0.

Ce filtre doit toujours retourner un tableau de données qui sera transmis à la fonction principale `wp_insert_user()`.

<div id="auth0_get_wp_user">
  ### auth0_get_wp_user
</div>

Ce filtre est appelé une fois que l’extension a trouvé l’utilisateur associé à la connexion (selon le `user_id` Auth0) et permet de remplacer le comportement par défaut par des règles de correspondance personnalisées (par exemple, toujours établir la correspondance à partir de l’adresse courriel).

Si le filtre renvoie null, la recherche se fera par courriel. Pour en savoir plus, consultez [Intégrer à WordPress](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/integrate-with-wordpress).

<div id="auth0_verify_email_page">
  ### auth0_verify_email_page
</div>

Ce filtre est exécuté dans `WP_Auth0_Email_Verification` pour modifier le HTML généré lorsqu’un utilisateur qui se connecte doit vérifier son adresse courriel avant d’avoir accès au site. Notez que ce HTML est transmis à `wp_die()`, où il est modifié avant d’être affiché (consultez la définition de `_default_wp_die_handler()` dans le core pour en savoir plus).

<div id="auth0_get_auto_login_connection">
  ### auth0_get_auto_login_connection
</div>

Ce filtre est utilisé dans `WP_Auth0_LoginManager` pour modifier la connexion utilisée dans le processus de connexion automatique. Le réglage défini dans wp-admin est récupéré, puis transmis à ce filtre.

<div id="wp_auth0_get_option">
  ### wp_auth0_get_option
</div>

Ce filtre est utilisé par les fonctions et méthodes de récupération d’options pour modifier la valeur renvoyée.

<div id="auth0_migration_ws_authenticated">
  ### auth0_migration_ws_authenticated
</div>

Ce filtre est utilisé dans `WP_Auth0_Routes` pour modifier l’objet `WP_User`, qui est encodé en JSON puis renvoyé à Auth0 pendant une migration d’utilisateurs.

<div id="wpa0_should_create_user">
  ### wpa0_should_create_user
</div>

Ce filtre est utilisé dans `WP_Auth0_Users` pour déterminer si un utilisateur doit être créé. La valeur initiale transmise est `TRUE`. Si `FALSE` est renvoyé pour quelque raison que ce soit, l’inscription sera refusée et l’utilisateur qui s’inscrit verra un message d’erreur (`WP_Auth0_UsersRepo::create()`).

<div id="auth0_login_css">
  ### auth0_login_css
</div>

Ce filtre sert à modifier le CSS de la page de connexion, y compris le widget de connexion lui-même. Ce filtre s’exécute avant que le CSS ne soit récupéré depuis la page des paramètres de wp-admin.

<div id="auth0_login_form_tpl">
  ### auth0_login_form_tpl
</div>

Filtre le modèle utilisé pour le formulaire de connexion Auth0. Ce filtre doit renvoyer le chemin d’un fichier contenant du HTML qui remplace le contenu de `wp-content/plugins/auth0/templates/auth0-login-form.php`. Le JS standard d’initialisation de Lock recherche un attribut ID `auth0-login-form` pour instancier le formulaire de connexion; assurez-vous donc qu’il est présent, ou remplacez le fichier `wp-content/plugins/auth0/assets/js/lock-init.js` par le vôtre.

<div id="auth0_settings_fields">
  ### auth0_settings_fields
</div>

Ce filtre sert à modifier un champ de formulaire existant ou à en ajouter un nouveau. Il doit renvoyer un tableau `$options` modifié contenant vos changements ou ajouts. Les nouveaux champs doivent avoir une fonction de rappel associée au champ, comme illustré ci-dessous.

<div id="auth0_auth_scope">
  ### auth0_auth_scope
</div>

Ce filtre permet aux développeurs d’ajouter ou de modifier le scope demandé lors de la connexion. Il peut servir à ajouter des [claims personnalisées](/docs/fr-CA/secure/tokens/json-web-tokens/create-custom-claims) ou à demander un <Tooltip tip="Refresh Token : jeton servant à obtenir un nouvel Access Token sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Refresh+Token">Refresh Token</Tooltip>.

<div id="auth0_nonce_cookie_name">
  ### auth0_nonce_cookie_name
</div>

Utilisez ce filtre pour modifier le nom du cookie utilisé pour la validation du <Tooltip tip="Nonce : nombre arbitraire émis une seule fois dans un protocole d’authentification afin de détecter et de prévenir les attaques par rejeu." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=nonce">nonce</Tooltip>. Consultez le filtre `auth0_state_cookie_name` ci-dessous pour obtenir un exemple.

<div id="auth0_state_cookie_name">
  ### auth0_state_cookie_name
</div>

Utilisez ce filtre pour modifier le nom du cookie utilisé pour la valeur du paramètre state. Vous pouvez y ajouter un préfixe ou un suffixe, ou le remplacer entièrement. Pour en savoir plus sur le paramètre state, consultez [Prévenir les attaques et rediriger les utilisateurs avec les paramètres state d’OAuth 2.0](/docs/fr-CA/secure/attack-protection/state-parameters).
Assurez-vous d’utiliser des caractères valides pour toute modification apportée :

`A <cookie-name> can be any US-ASCII characters except control characters (CTLs), spaces, or tabs. It also must not contain a separator character like the following: ( ) < > @ , ; : \ " / ? = { }.`

Pour en savoir plus sur l’en-tête de réponse HTTP `Set-Cookie`, consultez [Set-Cookie dans MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Set-Cookie).

<div id="auth0_settings_constant_prefix">
  ### auth0_settings_constant_prefix
</div>

Utilisez ce filtre pour modifier le préfixe de la constante utilisée pour surcharger les paramètres de l’extension. Veuillez noter que ce filtre **doit** s’exécuter avant le chargement de l’extension Auth0; il doit donc se trouver dans une extension MU. Pour en savoir plus sur les extensions MU, consultez [Must Use Plugins on wordpress.org](https://developer.wordpress.org/advanced-administration/plugins/mu-plugins/).

<div id="auth0_authorize_url_params">
  ### auth0_authorize_url_params
</div>

Ce filtre permet aux développeurs de modifier, au besoin, les paramètres du endpoint `/authorize`. La fonction doit retourner un tableau associatif de paramètres d’URL. Pour en savoir plus sur l’utilisation de ces paramètres, consultez [Authentication API Explorer : Login](https://auth0.com/docs/api/authentication#login).

<div id="auth0_authorize_url">
  ### auth0_authorize_url
</div>

Ce filtre permet aux développeurs de modifier l’URL `/authorize` complète avant qu’elle soit utilisée. La fonction doit renvoyer une URL valide sous forme de chaîne de caractères. Pour en savoir plus sur l’utilisation de cette URL, consultez [Authentication API Explorer : Login](https://auth0.com/docs/api/authentication#login).

<div id="auth0_die_on_login_output">
  ### auth0_die_on_login_output
</div>

Ce filtre vous permet de modifier ou de remplacer le contenu HTML transmis à `wp_die()` lorsqu’une erreur survient à la connexion. Ce filtre n’a aucun effet sur le contenu de la vérification du courriel (voir auth0&#95;verify&#95;email&#95;page).

<div id="auth0_coo_auth0js_url">
  ### auth0_coo_auth0js_url
</div>

Ce filtre vous permet de remplacer l’URL CDN par défaut d’Auth0.js lors du chargement de la page COO de solution de secours.

<div id="auth0_slo_return_to">
  ### auth0_slo_return_to
</div>

Ce filtre vous permet de remplacer l’URL `returnTo` par défaut lors de votre déconnexion d’Auth0.

<div id="auth0_logout_url">
  ### auth0_logout_url
</div>

Ce filtre vous permet de remplacer l’URL de déconnexion d’Auth0. Pour en savoir plus sur son fonctionnement, consultez [Déconnexion](/docs/fr-CA/authenticate/login/logout).

<div id="auth0_use_management_api_for_userinfo">
  ### auth0_use_management_api_for_userinfo
</div>

Ce filtre détermine s’il faut utiliser les données du profil utilisateur récupérées à partir de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> lorsque vous n’utilisez pas le flux de connexion implicite. Renvoyez la valeur booléenne `true` (par défaut) pour utiliser l’API, ou `false` pour utiliser le <Tooltip tip="ID token : justificatif destiné au client lui-même, plutôt que pour accéder à une ressource." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=ID+token">ID token</Tooltip>.

<div id="auth0_lock_options">
  ### auth0_lock_options
</div>

Ce filtre permet de modifier les options du formulaire de connexion Embedded Lock utilisé dans les shortcodes, les widgets et sur la page wp-login.php lorsque **Features &gt; <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> Page** est désactivée.

<div id="auth0_jwt_leeway">
  ### auth0_jwt_leeway
</div>

Ce filtre permet d’ajuster le délai de tolérance utilisé pour valider les jetons d’identité et doit renvoyer un nombre entier de secondes.

<div id="auth0_jwt_max_age">
  ### auth0_jwt_max_age
</div>

Ce filtre vous permet de régler le paramètre d’URL `max_age` transmis dans l’URL d’autorisation.

<div id="auth0_authorize_state">
  ### auth0_authorize_state
</div>

Ce filtre vous permet de filtrer les données de `state` avant qu’elles soient encodées et utilisées lors de la connexion. Ces données seront vérifiées après une connexion réussie et fournies telles quelles afin que vous puissiez les utiliser.
