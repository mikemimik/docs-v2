<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour installer ou personnaliser des extensions, vous devez utiliser un site WordPress.org auto-hébergé. Un site WordPress.com ne permet pas d’installer des extensions. Pour en savoir plus sur les différences, consultez [WordPress.com and WordPress.org on wordpress.com](https://wordpress.com/support/com-vs-org/).
</Callout>

Ce plugin peut être ajouté à votre site WordPress à l’aide de l’écran **Plugins** dans wp-admin :

1. Connectez-vous à un site WordPress existant en tant qu’administrateur.
2. Accédez à **Plugins &gt; Add New** dans le menu d’administration de gauche.
3. Recherchez « Login by Auth0 »
4. Pour le plugin Login by Auth0, cliquez sur **Install Now**, puis sur **Activate**.

Pour en savoir plus sur ce processus, consultez [WordPress.com and WordPress.org on wordpress.com](https://wordpress.com/support/com-vs-org/).

Dès que le plugin est activé, vous êtes redirigé vers le début du Setup Wizard.

Si vous n’avez pas encore de compte Auth0, [i**nscrivez-vous gratuitement pour en créer un**](https://auth0.com/signup) avant de continuer.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 ne remplacera pas vos formulaires de connexion tant que le **Domain**, le **Client ID** et le **Client Secret** n’auront pas été ajoutés à l’onglet **Basic** de la page des paramètres.
</Callout>

<div id="multisite-setup">
  ## Configuration multisite
</div>

Le plugin Login by Auth0 est compatible avec les réseaux multisites WordPress. Le plugin peut être activé à l’échelle du réseau pour protéger automatiquement les sites du réseau (une fois la configuration terminée) ou activé seulement pour une partie du réseau.

Il existe plusieurs façons de configurer un réseau de sites dans Auth0 :

1. **Tous les sites peuvent partager à la fois une Application et une connexion de base de données**

   1. Effectuez toutes les étapes du Setup Wizard pour le site principal.
   2. Configurez manuellement tous les autres sites à l’aide du **Domain**, du **Client ID** et du **Client Secret** du site principal dans l’onglet Basic de la page des paramètres d’Auth0.
   3. Mettez à jour les **Allowed Callback URLs**, les **Allowed Web Origins** et les **Allowed Logout URLs** de l’Application pour inclure chaque site (des caractères génériques peuvent être utilisés si votre réseau utilise des sous-domaines).
2. **Chaque site peut avoir sa propre Application et partager une connexion de base de données**

   1. Effectuez toutes les étapes du Setup Wizard pour le site principal.
   2. Ensuite, [créez manuellement une Application pour chaque site](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0) et ajoutez chacune d’elles à la connexion de base de données créée précédemment.
   3. Ajoutez les valeurs **Domain**, **Client ID** et **Client Secret** dans l’onglet Basic de la page des paramètres d’Auth0 pour chaque site.
3. **Chaque site peut avoir sa propre Application et sa propre connexion de base de données.** Dans ce cas, effectuez toutes les étapes du Setup Wizard pour chaque site.

Chacune des options ci-dessus comporte des compromis. L’option 1 comporte le moins d’éléments distincts à gérer dans Auth0, mais si votre réseau compte des centaines de sites et que vous n’utilisez pas de sous-domaines, vous pourriez vous heurter à des limites quant au nombre d’URL de rappel. L’option 2 exigera la gestion de nombreuses Applications distinctes, mais vous permettra de configurer différemment l’Application de chaque site.

Comme toujours, si vous avez des questions sur ce processus de configuration, [créez une publication dans notre Community et ajoutez-lui l’étiquette &quot;wordpress&quot;](https://community.auth0.com/tags/wordpress).

<div id="setup-wizard">
  ## Setup Wizard
</div>

Le Setup Wizard tentera de créer tous les composants nécessaires à l’utilisation d’Auth0 sur votre site WordPress. Si vous avez déjà une Application ou une connexion de base de données que vous souhaitez utiliser, veuillez consulter les étapes de configuration manuelle ci-dessous.

<div id="option-1-standard-setup">
  ### Option 1 : Configuration standard
</div>

Cela créera et configurera une application et une connexion de base de données pour ce site.

Tout d’abord, [générez un jeton de Management API](https://auth0.com/docs/api/management/v2/get-access-tokens-for-test#get-access-tokens-manually). Une fois le jeton généré, prenez en note le nom de domaine utilisé dans le champ **Identifier** de l’onglet **Settings**. Par exemple, si votre **Identifier** est `https://tenant-name.auth0.com/api/v2/`, le domaine du tenant est `tenant-name.auth0.com`. Pour en savoir plus sur les domaines de tenant, consultez [Create Tenants](/docs/fr-CA/get-started/auth0-overview/create-tenants).

De retour dans le Setup Wizard de l’administrateur WordPress, cliquez sur **Standard**. Dans la fenêtre modale qui s’affiche, cliquez sur **Start Standard Setup**.

Entrez le domaine du tenant et le jeton d’API ci-dessus. Ce jeton sert uniquement au processus de configuration et ne sera pas enregistré dans la base de données.

Si la première partie de la configuration se termine correctement, vous verrez l’écran « Configure your social connections ». Cliquez sur **Next** pour poursuivre la configuration en migrant votre compte administrateur.

Cette étape associe votre utilisateur WordPress à un utilisateur Auth0 qui vous autorise à vous connecter. Vous pouvez choisir le même mot de passe que pour votre compte administrateur ou un mot de passe différent, mais assurez-vous qu’il respecte les politiques de robustesse des mots de passe de la connexion de base de données utilisée. Pour en savoir plus, consultez [Password Strength in Auth0 Database Connections: Password Policies](/docs/fr-CA/authenticate/database-connections/password-strength#password-policieslicies).

Le Setup Wizard doit être exécuté jusqu’au bout pour que votre site soit correctement configuré. Si le Wizard échoue pour quelque raison que ce soit avant l’écran « setup successful », consultez le journal d’erreurs du plugin à **wp-admin &gt; Auth0 &gt; Error Log** ainsi que les étapes ci-dessous pour déterminer le problème.

Pour recommencer complètement le processus, supprimez toutes les applications ou connexions de base de données qui ont été créées dans le <Tooltip tip="Auth0 Dashboard : principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Dans WordPress, accédez à **Auth0 &gt; Settings &gt; Basic**, supprimez les champs Domain, <Tooltip tip="Client Secret : secret utilisé par un client (application) pour s’authentifier auprès de l’Authorization Server; il ne doit être connu que du client et de l’Authorization Server et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+ID">Client ID</Tooltip> et <Tooltip tip="Client ID : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Client+Secret">Client Secret</Tooltip>, puis cliquez sur **Save**. Cliquez maintenant sur **Setup Wizard** dans le menu d’administration pour recommencer le processus.

Si vous n’arrivez toujours pas à l’installer, [publiez une discussion dans notre Community](https://community.auth0.com/tags/wordpress) avec les messages d’erreur affichés dans l’Error Log, et nous serons heureux de vous aider !

<div id="option-2-user-migration-setup">
  ### Option 2 : Configuration de la migration des utilisateurs
</div>

Cette option créera et configurera une Application et une connexion de base de données, ainsi que la migration des données de votre base de données WordPress. Elle nécessite une connexion entrante depuis les serveurs Auth0 et ne pourra pas être modifiée par la suite sans entraîner une perte de données. Pour en savoir plus, consultez [User Migration in Login by Auth0 WordPress Plugin](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).

::: warning
Si vous avez plus d’une connexion de base de données personnalisée dans Auth0, vous devrez vous assurer que les ID utilisateur comportent un espace de noms afin d’éviter les conflits. Cela se fait automatiquement pour les sites qui installent la version 3.11.0 ou une version ultérieure. Si vos connexions sont ou ont été créées avec une version antérieure, consultez les étapes de dépannage dans [User Migration in Login by Auth0 WordPress Plugin](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).
:::

Une fois le processus de configuration terminé, déconnectez-vous de votre site WordPress, puis essayez de vous reconnecter à l’aide de vos identifiants WordPress existants dans le formulaire de connexion Auth0. Cela devrait créer un utilisateur Auth0 lié à votre compte WordPress.

<div id="option-3-manual-setup">
  ### Option 3 : Configuration manuelle
</div>

Cette option ignore la configuration automatique et vous permet de créer et de configurer votre propre Application et connexion de base de données (voir ci-dessous). Utilisez cette option si vous voulez que ce site utilise une Application existante ou une connexion de base de données.

<div id="option-4-import-setup">
  ### Option 4 : Importer une configuration
</div>

Le site peut aussi être configuré en important les paramètres d’un autre site. C’est utile si vous passez d’un environnement à un autre ou si vous avez déjà un site WordPress semblable configuré.

<div id="setup-complete">
  ## Configuration terminée
</div>

Lorsque l’écran « Done » s’affiche, Auth0 est activé et prêt à accepter les connexions et, s’il y a lieu, les inscriptions.

C’est le bon moment pour vérifier que l’essentiel fonctionne sur votre site avant de modifier les paramètres par défaut :

1. Déconnectez-vous de WordPress et confirmez que le formulaire Auth0 s’affiche maintenant à l’adresse `/wp-login.php`.
2. Connectez-vous avec l’utilisateur Auth0 créé ci-dessus.
3. Déconnectez-vous, puis essayez de créer un compte avec une autre adresse courriel (si l’option « Anyone can register » est activée dans les réglages généraux de WordPress).
4. Essayez de vous connecter à l’aide d’une connexion sociale (si vous les avez activées).

Vous êtes maintenant prêt à [configurer l’application](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0).

<div id="manual-setup">
  ## Configuration manuelle
</div>

Le plugin peut être configuré à l’aide du Setup Wizard intégré (décrit ci-dessous) ou manuellement, en créant une application et en attribuant des connexions. Le processus de configuration entièrement manuel peut être utilisé si l’assistant vous pose problème, si vous avez déjà suivi le processus de configuration, ou si vous voulez partager une connexion de base de données entre plusieurs applications.

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#), puis sélectionnez **Create**.
2. Donnez à votre application un nom descriptif, sélectionnez **Regular Web Applications**, puis sélectionnez **Create**.
3. Configurez l’application pour votre installation WordPress. Pour savoir comment configurer l’application, consultez [Configure Login by Auth0](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0) et suivez toutes les sections jusqu’à « Update Auth0 settings in WordPress ».

Une fois les étapes ci-dessus terminées, votre site sera configuré et prêt à être utilisé.
