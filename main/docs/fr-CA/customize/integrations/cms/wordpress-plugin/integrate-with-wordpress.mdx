---
description: Décrit les scénarios d’intégration d’Auth0 à WordPress
title: Intégrer WordPress
---

Le plugin Login by Auth0 gère automatiquement les flux de connexion et de création de compte en créant des comptes d’utilisateur ou en les associant aux données de profil Auth0 reçues. Le processus de connexion et le processus d’inscription sont semblables, et un compte sera créé ou associé en fonction des données de votre base de données WordPress. Une connexion avec Auth0 crée un compte WordPress, tandis qu’une inscription avec Auth0 associe un compte WordPress existant.

Si vous utilisez le paramètre de migration des utilisateurs dans le plugin, le flux de connexion sera légèrement différent de ce qui est expliqué ci-dessous. Pour en savoir plus, consultez [Migration des utilisateurs dans le plugin WordPress Login by Auth0](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).

Voici les étapes du processus :

1. L’utilisateur accède à la page de connexion du site WordPress. Il peut s’agir de la page de connexion principale à `[SITE URL]/wp-login.php` ou d’une page contenant un widget ou un shortcode.
2. L’utilisateur saisit son nom d’utilisateur et son mot de passe, clique sur une icône de connexion sociale pour utiliser un autre fournisseur d’identité, ou termine le processus Passwordless dans le formulaire de connexion Auth0, Lock.
3. Auth0 tente d’authentifier l’utilisateur au moyen de la méthode sélectionnée.

   1. Si une tentative de connexion ou d’inscription avec un nom d’utilisateur et un mot de passe, ou avec Passwordless, échoue, un message d’erreur s’affiche dans Lock.
   2. En cas de réussite, le processus se poursuit.
4. L’utilisateur est redirigé vers le point de terminaison `/authorize`, avec un ticket de connexion et une valeur `state` générés par le plugin. Une fois cette étape terminée, le compte utilisateur Auth0 a été créé, et le reste du processus se déroule sur le site WordPress.
5. Le processus de connexion lui-même varie selon que vous utilisez le [Flux de code d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow) ou le [Flux implicite](/docs/fr-CA/authenticate/login/oidc-conformant-authentication/oidc-adoption-implicit-flow) :

   1. Pour les connexions avec le flux de code d’autorisation :

      1. L’utilisateur est redirigé vers une URL de rappel, `SITE URL/index.php?auth0=1`, avec un code d’autorisation et la même valeur `state` dans les paramètres d’URL.
      2. La valeur `state` est validée. Si la validation échoue, une erreur &quot;state invalide&quot; s’affiche et le processus de connexion s’arrête. Pour en savoir plus sur la validation de `state`, consultez [Dépannage de Login by Auth0 WordPress Plugin](/docs/fr-CA/customize/integrations/cms/wordpress-plugin/troubleshoot-login-by-auth0).
      3. Le jeton d’identité est validé afin de s’assurer que rien n’a été modifié pendant le transfert. Si le jeton d’identité n’est pas valide, un message d’erreur s’affiche et le processus de connexion s’arrête (consultez la page de dépannage pour en savoir plus sur la validation du jeton d’identité)
      4. Les données du profil utilisateur sont récupérées au moyen de la Management API à l’aide du flux machine-to-machine. Pour en savoir plus, consultez [Flux machine-to-machine](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow).
   2. Pour les connexions avec le flux implicite :

      1. L’utilisateur est redirigé vers une URL de rappel, `SITE URL/wp-login.php?auth0=implicit`, avec un jeton d’identité et la même valeur `state` dans un lien d’ancrage.
      2. Ce lien d’ancrage est analysé en JS, puis renvoyé par POST à une URL de rappel, `SITE URL/index.php?auth0=implicit`, avec ces 2 mêmes valeurs dans les paramètres d’URL.
      3. Le jeton d’identité est validé afin de s’assurer que rien n’a été modifié pendant le transfert. Si le jeton d’identité n’est pas valide, un message d’erreur s’affiche et le processus de connexion s’arrête (consultez la page de dépannage pour en savoir plus sur la validation du jeton d’identité)
      4. Les renseignements contenus dans le jeton d’identité valide sont utilisés comme données du profil utilisateur.
6. Le processus d’authentification Auth0 est terminé et le plugin tente d’associer les données de profil à un utilisateur dans WordPress.
7. Le plugin vérifie si le site exige une adresse courriel (onglet **Advanced** des paramètres du plugin) et si le profil reçu a l’indicateur `email_verified` activé.

   1. Si le site exige une adresse courriel et que l’utilisateur ne fournit pas d’adresse courriel (certains fournisseurs d’identité sociale, comme X, n’incluent pas d’adresse courriel), le processus de connexion s’interrompt et affiche le message d’erreur &quot;Ce compte n’a pas d’adresse courriel associée.&quot;
   2. Si le site exige une adresse courriel et que l’indicateur `email_verified` de l’utilisateur n’est pas défini à `true`, le processus de connexion s’interrompt et affiche le message d’erreur &quot;Ce site exige une adresse courriel vérifiée&quot;, ainsi qu’un lien pour renvoyer le courriel de vérification. Ce message continue de s’afficher jusqu’à ce que l’utilisateur ait vérifié son adresse courriel.
   3. Si le site n’exige pas d’adresse courriel ou que l’indicateur `email_verified` de l’utilisateur est défini à `true`, le processus de connexion se poursuit.
8. Le plugin vérifie s’il existe un utilisateur dans la base de données WordPress dont une valeur `usermeta` correspond à l’ID utilisateur Auth0 reçu (ce qui signifie que l’utilisateur s’est déjà inscrit ou s’est déjà connecté avec Auth0) :

   1. Si un utilisateur correspondant à l’ID utilisateur reçu est trouvé, le processus de connexion se poursuit.
   2. Si aucun utilisateur correspondant à l’ID utilisateur Auth0 reçu n’est trouvé, le plugin recherche une adresse courriel correspondant à l’utilisateur transmis :

      1. Si une correspondance est trouvée, cet utilisateur est sélectionné et le processus de connexion se poursuit.
      2. Si aucune correspondance n’est trouvée, le plugin vérifie si l’inscription est activée sur le site WordPress :

         1. Si l’inscription est désactivée, le processus de connexion s’arrête avec un message d’erreur indiquant &quot;Impossible de créer l’utilisateur. Le processus d’inscription n’est pas disponible&quot;`.`
         2. Si l’inscription est activée, un nouvel utilisateur est créé et le processus de connexion se poursuit.
9. L’utilisateur trouvé ou créé est mis à jour avec les données de profil Auth0 reçues, y compris son ID utilisateur Auth0.
10. L’utilisateur est connecté à son compte WordPress au moyen de `wp_set_auth_cookie`, et l’action `do_login` du noyau est déclenchée.
11. L’utilisateur est redirigé vers une page du site, qui peut être celle définie par défaut dans l’onglet **Advanced** des paramètres de l’extension, l’URL de connexion d’origine si un shortcode ou un widget a été utilisé, ou une autre URL fournie durant le processus de connexion.

L’utilisateur est maintenant connecté à Auth0 et à son compte WordPress, tous deux étant associés à son ID utilisateur Auth0.
