<div id="step-1-set-up-the-amazon-api-gateway">
  ## Étape 1 - Configurer Amazon API Gateway
</div>

Après avoir terminé cette étape, vous aurez :

* Configuré Amazon API Gateway à l’aide de fonctions AWS Lambda pour exécuter la logique de votre service, qui stocke et récupère des animaux de compagnie dans une table Amazon DynamoDB;
* Créé deux méthodes de service REST accessibles sans authentification pour récupérer et mettre à jour une liste d’animaux de compagnie.

Avant de commencer, assurez-vous d’avoir installé [Node.js](https://nodejs.org/).

<div id="step-1-create-the-amazon-dynamodb-table">
  ### Étape 1. Créer la table Amazon DynamoDB
</div>

Dans la [console Amazon DynamoDB](https://console.aws.amazon.com/dynamodb), cliquez sur **Create Table**.

<Frame>![Console DynamoDB - Créer une table](/docs/images/cdy7uua7fh8z/2ZPhi7Ddx4TznSXEEhUSDK/52a06412f7c5e6d0ef0d13c6ffe17611/dynamodb-create-table.png)</Frame>

Configurez les paramètres associés à la table :

* **Nom de la table** : Pets
* **Clé primaire** : username
* **Type de clé primaire** : String
* **Utiliser les paramètres par défaut** : décoché
* **Unités de capacité de lecture** : 3
* **Unités de capacité d’écriture** : 3

<Frame>![Console DynamoDB - Configurer la table nouvellement créée](/docs/images/cdy7uua7fh8z/6MYJV97Xa3DxajG8DYj2dx/e428ab2ce137176d7771c61a9c528f7c/configure-newly-created-table.png)</Frame>

Cliquez sur **Create** pour créer la table avec les paramètres que vous avez fournis.

Pendant la création de la table, notez l’Amazon Resource Name (ARN) dans la section Table details. Vous aurez besoin de l’ARN de la table à l’étape suivante.

<Frame>![Console DynamoDB - Détails de la table - Amazon Resource Name](/docs/images/cdy7uua7fh8z/4ZFemddtUD8tRi0NUvzXJ2/38abd951974cad385ef7417f48adf359/table-arn.png)</Frame>

<div id="step-2-create-the-policy-that-grants-aws-lambda-functions-access-to-the-dynamodb-pets-table">
  ### Étape 2. Créer la stratégie qui accorde aux fonctions AWS Lambda l&#39;accès à la table Pets de DynamoDB
</div>

Accédez à la [console AWS IAM](https://console.aws.amazon.com/iam).

Cliquez sur **Roles** dans le menu de gauche, puis sur le bouton **Create New Role**.

<Frame>![Console IAM - Roles - Create New Roles](/docs/images/cdy7uua7fh8z/4mgCTXLpycHLA8RbYoEFim/5d9eb81be979355aa9cd10d476ead515/roles.png)</Frame>

Nommez le rôle `APIGatewayLambdaExecRole` et cliquez sur **Next Step**.

Sélectionnez le type de rôle. Sous AWS Service Roles, sélectionnez AWS Lambda.

<Frame>![Console IAM - Sélectionner le type de rôle - Service Roles - AWS Lambda](/docs/images/cdy7uua7fh8z/326kKGbahlvIwVFTi61Gfw/026d1fc0c8c6e3a489ec4fb840f551da/select-role-type.png)</Frame>

À l&#39;écran Attach Policy, passez cette étape en cliquant sur **Next Step**. À ce stade, vérifiez les renseignements que vous avez fournis. Si tout semble correct, cliquez sur **Create Role**. Une fois l&#39;opération terminée, vous devriez voir votre rôle dans la liste sur la page d&#39;accueil d&#39;IAM.

Sélectionnez le rôle que vous venez de créer, **APIGatewayLambdaExecRole**. Cliquez sur la flèche vers le bas pour Inline Policies, puis sur le lien **Click Here**.

Sélectionnez Custom Policy, puis cliquez sur **Select**. Nommez la stratégie `LogAndDynamoDBAccess` et ajoutez le code suivant comme document de stratégie (assurez-vous d&#39;abord de mettre à jour l&#39;Amazon Resource Name (ARN) de votre table DynamoDB). Cliquez sur **Apply Policy**.

```json lines
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AccessCloudwatchLogs",
      "Action": ["logs:*"],
      "Effect": "Allow",
      "Resource": "arn:aws:logs:*:*:*"
    },
    {
      "Sid": "PetsDynamoDBReadWrite",
                  "Effect": "Allow",
      "Action": [
                  "dynamodb:DeleteItem",
                  "dynamodb:GetItem",
                  "dynamodb:PutItem",
                  "dynamodb:UpdateItem"
                  ],
      "Resource": ["DYNAMODB_TABLE_ARN_VALUE_FROM_PREVIOUS_STEP"]
    }
   ]
}
```

<div id="step-2-create-the-aws-lambda-functions">
  ### Étape 2. Créez les fonctions AWS Lambda
</div>

Les trois prochaines étapes servent à créer les fonctions AWS Lambda qui permettent de récupérer et de mettre à jour les renseignements sur les animaux de compagnie dans la table DynamoDB.

<div id="create-the-lambda-function-for-getpetinfo">
  #### Créer la fonction AWS Lambda pour GetPetInfo
</div>

Dans la [console AWS Lambda](https://console.aws.amazon.com/lambda), sélectionnez **Create a Lambda Function** (si vous n’avez encore jamais créé de fonction AWS Lambda, cliquez sur **Get Started Now**).

À l’écran Select blueprint, cliquez sur **Blank Function**.

Vous serez ensuite invité à configurer les déclencheurs. Cliquez sur **Next** pour continuer. Vous n’avez rien à faire à cette étape.

Enfin, on vous demandera de configurer la fonction.

Remplissez les champs appropriés avec les renseignements suivants :

* **Nom** : `GetPetInfo`
* **Environnement d’exécution** : Node.js 6.10

Collez le code suivant dans la zone **fonction Lambda code** pour lire les informations sur l’animal à partir de la table DynamoDB.

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();

exports.handler = function(event, context) {
   var cb = function(err, data) {
      if(err) {
         console.log('error on GetPetsInfo: ',err);
         context.done('Unable to retrieve pet information', null);
      } else {
         if(data.Item && data.Item.pets) {
             context.done(null, data.Item.pets);
         } else {
              context.done(null, {});
         }
      }
   };

   dynamo.getItem({TableName:"Pets", Key:{username:"default"}}, cb);
};
```

Pour rôle, sélectionnez **Choose an existing role**. Ensuite, choisissez **APIGatewayLambdaExecRole** comme Existing Role. Laissez tous les autres paramètres à leurs valeurs par défaut.

Cliquez sur **Next** pour vérifier les informations que vous avez fournies. Si tout est correct, cliquez sur **Create function**.

Cliquez sur **Test**, en laissant Input test event à sa valeur par défaut (qui utilise le modèle Hello World). Une fois le test terminé, vous devriez voir une sortie vide (`{}`) dans la section Execution Result. La table est vide.

<div id="create-the-lambda-function-for-updatepetinfo">
  #### Créez la fonction Lambda pour UpdatePetInfo
</div>

Reprenez les instructions utilisées pour créer la fonction `GetPetInfo`, mais utilisez plutôt le code de fonction suivant :

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();
exports.handler = function(event, context) {
    var item = { username:"default",
                 pets: event.pets || {}
            };

    var cb = function(err, data) {
        if(err) {
            console.log(err);
            context.fail('unable to update pets at this time');
        } else {
            console.log(data);
                context.done(null, data);
        }
    };
    dynamo.putItem({TableName:"Pets", Item:item}, cb);
};
```

Mettez la fonction à l’essai en cliquant sur le menu déroulant Actions, puis en choisissant **Configurer l’événement d’exemple**. Saisissez les données d’exemple suivantes, puis cliquez sur **Soumettre** :

```json lines
{
  "pets": [{
    "id": 1,
    "type": "dog",
    "price": 249.99
  }]
}
```

Vous devriez voir un résultat vide (`{}`).

Retournez à votre fonction Lambda `GetPetInfo` et cliquez de nouveau sur **Test**. Vous devriez maintenant voir un seul animal.

<div id="create-the-third-lambda-function">
  #### Créez la troisième fonction Lambda
</div>

Vous allez créer une autre fonction Lambda. Même si cette fonction ne fera rien, elle est requise par la méthode OPTIONS pour CORS, comme il sera expliqué dans une section ultérieure.

En suivant les étapes décrites ci-dessus, créez une fonction Lambda nommée `NoOp`. Le code de la fonction sera le suivant :

```js lines
exports.handler = function(event, context) {
    context.succeed('');
}
```

Au lieu de créer cette troisième fonction Lambda, vous pouvez choisir de [créer une méthode OPTIONS](#method-options) dans API Gateway.

<div id="step-3-create-the-amazon-api-gateway-api">
  ### Étape 3. Créer l’API dans Amazon API Gateway
</div>

Vous allez créer une API avec deux méthodes : l’une effectuera un `GET` pour récupérer les renseignements sur l’animal de compagnie, et l’autre un `POST` pour les transmettre.

<div id="method-get-pet-information">
  #### Méthode : GET Pet Information
</div>

Accédez à la [Amazon API Gateway Console](https://console.aws.amazon.com/apigateway), puis cliquez sur **Create API**. S’il s’agit de la première fois que vous créez une API, vous verrez plutôt un écran vous invitant à cliquer sur **Get Started**.

S’il s’agit de la première fois que vous créez une API, vous serez invité à créer une Example API. Cliquez sur **OK** pour fermer la fenêtre contextuelle, puis choisissez le bouton radio **New API** au lieu de **Example API**.

Nommez l’API `SecurePets`, puis cliquez sur **Create API**.

Accédez à l’onglet Resources de l’API `SecurePets`, puis cliquez sur **Create Resource**.

Nommez la ressource `Pets`, puis cliquez de nouveau sur **Create Resource**.

Dans le volet de gauche, sélectionnez `/pets`, puis cliquez sur le bouton **CreateMethod**.

Dans la liste déroulante, sélectionnez GET, puis cliquez sur le bouton de coche. Entrez les valeurs de configuration suivantes pour la méthode `GET` :

* **Integration type** : Lambda Function ;
* **Lambda Region** : la région où vous vous trouvez ;
* **Lambda Function** : GetPetInfo.

Cliquez sur **Save**, puis sur **OK** lorsque l’invite s’affiche dans la fenêtre contextuelle pour accorder des autorisations à la fonction Lambda.

Dans la fenêtre Method Execution qui s’affiche ensuite, cliquez sur **Test**.

Vous devriez voir l’unique animal renvoyé dans le corps de la réponse.

<div id="method-post-pet-information">
  #### Méthode : POST des renseignements sur l’animal de compagnie
</div>

La création de l’API utilisée pour `POST` des renseignements sur l’animal de compagnie est semblable à celle de l’API utilisée pour `GET` des renseignements sur l’animal de compagnie.

Dans le volet de gauche, sélectionnez `/pets`, puis cliquez sur **CreateMethod**.

Dans la liste déroulante, sélectionnez POST, puis cliquez sur le bouton en forme de coche.

Pour Type d’intégration, sélectionnez Fonction Lambda, choisissez la région où vous vous trouvez, puis sélectionnez UpdatePetInfo comme fonction Lambda.

Cliquez sur **Save**, puis sur **OK** lorsqu’une invite s’affiche dans la fenêtre contextuelle pour accorder les autorisations à la fonction Lambda.

**Test**, puis collez ce qui suit dans le corps de la requête :

```json lines
{"pets": [
    {"id": 1, "type": "dog", "price": 249.99},
    {"id": 2, "type": "cat", "price": 124.99}
  ]
}
```

Vous devriez voir un résultat vide (`{}`).

Retournez à la méthode GET, puis cliquez de nouveau sur **Test** pour constater que le corps de la réponse indique que deux animaux sont répertoriés dans la table :

```json lines
[
  {
    "id": 1,
    "price": 249.99,
    "type": "dog"
  },
  {
    "id": 2,
    "price": 124.99,
    "type": "cat"
  }
]
```

<div id="method-options">
  #### Méthode : OPTIONS
</div>

Au lieu de créer une fonction Lambda qui ne fait rien, vous pouvez créer une méthode `OPTIONS` sur API Gateway.

Dans le volet de gauche, sélectionnez `/pets`, puis cliquez sur **CreateMethod**. Dans la liste déroulante, sélectionnez OPTIONS, puis cliquez sur le bouton de validation. Sélectionnez Mock comme Type d’intégration. Cliquez sur **Save**.

En laissant le corps de la réponse vide, cliquez sur **Test**. Vous devriez recevoir un corps de la réponse indiquant `no data`.

À ce stade, les fonctions AWS Lambda et les méthodes d’Amazon API Gateway sont définies sans sécurité.
