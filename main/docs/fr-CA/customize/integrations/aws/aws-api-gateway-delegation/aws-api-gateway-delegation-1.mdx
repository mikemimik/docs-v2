<Warning>
  Cette fonctionnalité utilise Delegation. Par défaut, Delegation est désactivée pour les tenants qui n’utilisaient pas d’add-on en date du 8 juin 2017. Les tenants existants qui utilisent actuellement un add-on nécessitant Delegation peuvent continuer à utiliser cette fonctionnalité. Si la fonctionnalité Delegation est modifiée ou retirée du service à un moment donné, les clients qui l’utilisent actuellement en seront avisés à l’avance et disposeront de suffisamment de temps pour migrer. De plus, notez que Delegation ne prend pas en charge l’utilisation de domaines personnalisés; toute fonctionnalité qui en dépend pourrait donc ne pas être entièrement fonctionnelle avec un domaine personnalisé.
</Warning>

<div id="step-1-set-up-the-amazon-api-gateway">
  ## Étape 1 - Configurer Amazon API Gateway
</div>

Après avoir terminé cette étape, vous aurez :

* configuré Amazon API Gateway à l’aide de fonctions AWS Lambda pour exécuter la logique de votre service, qui stocke et récupère des animaux de compagnie dans une table Amazon DynamoDB;
* créé deux méthodes de service REST non authentifiées pour récupérer et mettre à jour une liste d’animaux de compagnie.

Avant de commencer, veuillez avoir installé [Node.js](https://nodejs.org/).

<div id="step-1-create-the-amazon-dynamodb-table">
  ### Étape 1. Créer la table Amazon DynamoDB
</div>

Dans la [console Amazon DynamoDB](https://console.aws.amazon.com/dynamodb), cliquez sur **Create Table**.

<Frame>![Console DynamoDB - Create Table](/docs/images/cdy7uua7fh8z/2ZPhi7Ddx4TznSXEEhUSDK/52a06412f7c5e6d0ef0d13c6ffe17611/dynamodb-create-table.png)</Frame>

Configurez les variables associées à la table :

* **Nom de la table** : Pets
* **Clé primaire** : username
* **Type de clé primaire** : String
* **Utiliser les paramètres par défaut** : décoché
* **Unités de capacité de lecture** : 3
* **Unités de capacité d’écriture** : 3

<Frame>![Console DynamoDB - Configurer la table nouvellement créée](/docs/images/cdy7uua7fh8z/6MYJV97Xa3DxajG8DYj2dx/e428ab2ce137176d7771c61a9c528f7c/configure-newly-created-table.png)</Frame>

Cliquez sur **Create** pour créer la table avec les paramètres indiqués.

Pendant la création de la table, notez le nom de ressource Amazon (ARN) dans la section des détails de la table. Vous aurez besoin de l’ARN de la table à l’étape suivante.

<Frame>![Console DynamoDB - Détails de la table - nom de ressource Amazon](/docs/images/cdy7uua7fh8z/4ZFemddtUD8tRi0NUvzXJ2/38abd951974cad385ef7417f48adf359/table-arn.png)</Frame>

<div id="step-2-create-the-policy-that-grants-aws-lambda-functions-access-to-the-dynamodb-pets-table">
  ### Étape 2. Créez la politique qui accorde aux fonctions AWS Lambda l’accès à la table Pets de DynamoDB
</div>

Accédez à la [console AWS IAM](https://console.aws.amazon.com/iam).

Cliquez sur **Roles** dans le menu de gauche, puis sur le bouton **Create New Role**.

<Frame>![Console IAM - Roles - Créer de nouveaux rôles](/docs/images/cdy7uua7fh8z/4mgCTXLpycHLA8RbYoEFim/5d9eb81be979355aa9cd10d476ead515/roles.png)</Frame>

Nommez le rôle `APIGatewayLambdaExecRole`, puis cliquez sur **Next Step**.

Sélectionnez le type de rôle. Sous AWS Service Roles, sélectionnez AWS Lambda.

<Frame>![Console IAM - Sélectionner le type de rôle - Rôles de service - AWS Lambda](/docs/images/cdy7uua7fh8z/326kKGbahlvIwVFTi61Gfw/026d1fc0c8c6e3a489ec4fb840f551da/select-role-type.png)</Frame>

À l’écran Attach Policy, passez cette étape en cliquant sur **Next Step**. À ce stade, vérifiez les renseignements que vous avez fournis. Si tout semble correct, cliquez sur **Create Role**. Une fois l’opération terminée, votre rôle devrait s’afficher sur la page d’accueil d’IAM.

Sélectionnez le rôle que vous venez de créer, **APIGatewayLambdaExecRole**. Cliquez sur la flèche vers le bas de Inline Policies, puis sur le lien **Click Here**.

Sélectionnez Custom Policy, puis cliquez sur **Select**. Nommez la politique `LogAndDynamoDBAccess` et ajoutez le code suivant comme document de politique (veillez d’abord à mettre à jour le nom de ressource Amazon (ARN) de votre table DynamoDB). Cliquez sur **Apply Policy**.

```json lines
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AccessCloudwatchLogs",
      "Action": ["logs:*"],
      "Effect": "Allow",
      "Resource": "arn:aws:logs:*:*:*"
    },
    {
      "Sid": "PetsDynamoDBReadWrite",
                  "Effect": "Allow",
      "Action": [
                  "dynamodb:DeleteItem",
                  "dynamodb:GetItem",
                  "dynamodb:PutItem",
                  "dynamodb:UpdateItem"
                  ],
      "Resource": ["DYNAMODB_TABLE_ARN_VALUE_FROM_PREVIOUS_STEP"]
    }
   ]
}
```

<div id="step-2-create-the-aws-lambda-functions">
  ### Étape 2. Créez les fonctions AWS Lambda
</div>

Les trois prochaines étapes servent à créer les fonctions AWS Lambda pour récupérer et mettre à jour les renseignements sur les animaux de compagnie dans la table DynamoDB.

<div id="create-the-lambda-function-for-getpetinfo">
  #### Créer la fonction Lambda pour GetPetInfo
</div>

Dans la [console AWS Lambda](https://console.aws.amazon.com/lambda), sélectionnez **Create a Lambda Function** (si vous n&#39;avez encore jamais créé de fonction AWS Lambda, cliquez sur **Get Started Now**).

À l&#39;écran Select blueprint, cliquez sur **Blank Function**.

Vous serez ensuite invité à Configurer les déclencheurs. Cliquez sur **Next** pour continuer. Vous n&#39;avez pas à le faire à cette étape.

Enfin, il vous sera demandé de Configurer la fonction.

Remplissez les champs appropriés avec les renseignements suivants :

* **Nom** : `GetPetInfo`
* **Runtime** : Node.js 6.10

Collez le code suivant dans la zone **Lambda function code** pour lire les informations sur l&#39;animal à partir de la table DynamoDB.

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();

exports.handler = function(event, context) {
   var cb = function(err, data) {
      if(err) {
         console.log('error on GetPetsInfo: ',err);
         context.done('Unable to retrieve pet information', null);
      } else {
         if(data.Item && data.Item.pets) {
             context.done(null, data.Item.pets);
         } else {
              context.done(null, {});
         }
      }
   };

   dynamo.getItem({TableName:"Pets", Key:{username:"default"}}, cb);
};
```

Pour Role, sélectionnez **Choisir un rôle existant**. Ensuite, choisissez **APIGatewayLambdaExecRole** comme rôle existant. Laissez tous les autres paramètres à leur valeur par défaut.

Cliquez sur **Suivant** pour passer en revue les renseignements que vous avez fournis. Si tout semble correct, cliquez sur **Créer une fonction**.

Cliquez sur **Test** en laissant l’événement de test d’entrée à sa valeur par défaut (qui utilise le modèle Hello World). Une fois le test terminé, vous devriez voir une sortie vide (`{}`) dans la section Résultat de l’exécution. La table est vide.

<div id="create-the-lambda-function-for-updatepetinfo">
  #### Créez la fonction Lambda pour UpdatePetInfo
</div>

Reprenez les instructions utilisées pour créer la fonction `GetPetInfo`, mais utilisez plutôt le code de fonction suivant :

```javascript lines
var AWS = require('aws-sdk');
var DOC = require('dynamodb-doc');
var dynamo = new DOC.DynamoDB();
exports.handler = function(event, context) {
    var item = { username:"default",
                 pets: event.pets || {}
            };

    var cb = function(err, data) {
        if(err) {
            console.log(err);
            context.fail('unable to update pets at this time');
        } else {
            console.log(data);
                context.done(null, data);
        }
    };
    dynamo.putItem({TableName:"Pets", Item:item}, cb);
};
```

Testez la fonction en cliquant sur le menu déroulant Actions et en choisissant **Configurer un événement d’exemple**. Saisissez les données d’exemple suivantes et cliquez sur **Soumettre** :

```json lines
{
  "pets": [{
    "id": 1,
    "type": "dog",
    "price": 249.99
  }]
}
```

Vous devriez voir un résultat vide (`{}`).

Retournez à votre fonction Lambda `GetPetInfo` et cliquez de nouveau sur **Test**. Vous devriez maintenant voir un seul animal.

<div id="create-the-third-lambda-function">
  #### Créer la troisième fonction Lambda
</div>

Vous allez créer une autre fonction Lambda. Bien que cette fonction ne fasse rien, elle est nécessaire à la méthode OPTIONS pour CORS, comme expliqué plus loin.

En suivant les étapes décrites ci-dessus, créez une fonction Lambda nommée `NoOp`. Le code de la fonction sera le suivant :

```js lines
exports.handler = function(event, context) {
    context.succeed('');
}
```

Au lieu de créer cette troisième fonction Lambda, vous pouvez aussi [créer une méthode OPTIONS](#method-options) dans API Gateway.

<div id="step-3-create-the-amazon-api-gateway-api">
  ### Étape 3. Créer l’API Amazon API Gateway
</div>

Vous allez créer une API avec deux méthodes : l’une utilisera `GET` pour récupérer les renseignements sur l’animal de compagnie, et l’autre utilisera `POST` pour envoyer les renseignements sur l’animal de compagnie.

<div id="method-get-pet-information">
  #### Méthode : GET Pet Information
</div>

Accédez à la [Amazon API Gateway Console](https://console.aws.amazon.com/apigateway), puis cliquez sur **Create API**. Si c&#39;est la première fois que vous créez une API, un écran vous invitant à **Get Started** s&#39;affichera à la place.

Si c&#39;est la première fois que vous créez une API, vous serez invité à créer une API d&#39;exemple. Cliquez sur **OK** pour fermer la fenêtre contextuelle, puis sélectionnez le bouton radio **New API** au lieu du bouton **Example API**.

Nommez l&#39;API `SecurePets`, puis cliquez sur **Create API**.

Accédez à l&#39;onglet Resources de l&#39;API `SecurePets`, puis cliquez sur l&#39;action **Create Resource**.

Nommez la ressource `Pets`, puis cliquez de nouveau sur **Create Resource**.

Dans le volet de gauche, sélectionnez `/pets`, puis cliquez sur le bouton **CreateMethod**.

Dans la liste déroulante, sélectionnez GET, puis cliquez sur le bouton avec la coche. Indiquez les valeurs de configuration suivantes pour la méthode `GET` :

* **Integration type** : Lambda Function;
* **Lambda Region** : la Region où vous vous trouvez;
* **Lambda Function** : GetPetInfo.

Cliquez sur **Save**, puis sur **OK** lorsque la fenêtre contextuelle vous invite à accorder les permissions à la fonction Lambda.

Dans la fenêtre Method Execution qui s&#39;affiche ensuite, cliquez sur **Test**.

Vous devriez voir le seul animal renvoyé dans le corps de la réponse.

<div id="method-post-pet-information">
  #### Méthode : POST des informations sur l’animal de compagnie
</div>

La création de l’API utilisée pour `POST` des informations sur l’animal de compagnie est semblable à celle utilisée pour `GET` des informations sur l’animal de compagnie.

Dans le volet de gauche, sélectionnez `/pets`, puis cliquez sur **CreateMethod**.

Dans la liste déroulante, sélectionnez POST, puis cliquez sur le bouton de coche.

Sélectionnez Lambda Function pour Integration Type, sélectionnez la région où vous vous trouvez, puis sélectionnez UpdatePetInfo comme fonction Lambda.

Cliquez sur **Save**, puis sur **OK** lorsque la fenêtre contextuelle vous invite à accorder les autorisations à la fonction Lambda.

**Test**, puis collez ce qui suit dans le corps de la requête :

```json lines
{"pets": [
    {"id": 1, "type": "dog", "price": 249.99},
    {"id": 2, "type": "cat", "price": 124.99}
  ]
}
```

Vous devriez voir un résultat vide (`{}`).

Retournez à la méthode GET, puis cliquez de nouveau sur **Test** pour voir que le corps de la réponse indique que deux animaux sont répertoriés dans la table :

```json lines
[
  {
    "id": 1,
    "price": 249.99,
    "type": "dog"
  },
  {
    "id": 2,
    "price": 124.99,
    "type": "cat"
  }
]
```

<div id="method-options">
  #### Méthode : OPTIONS
</div>

Au lieu de créer une fonction Lambda qui n’effectue rien, vous pouvez créer une méthode `OPTIONS` dans API Gateway.

Dans le volet de gauche, sélectionnez `/pets`, puis cliquez sur **CreateMethod**. Dans la liste déroulante, sélectionnez OPTIONS, puis cliquez sur le bouton en forme de coche. Sélectionnez Mock comme Integration Type. Cliquez sur **Save**.

En laissant le corps de la réponse vide, cliquez sur **Test**. Vous devriez recevoir un corps de la réponse indiquant `no data`.

À ce stade, les fonctions AWS Lambda et les méthodes d’Amazon API Gateway sont définies sans configuration de sécurité.
