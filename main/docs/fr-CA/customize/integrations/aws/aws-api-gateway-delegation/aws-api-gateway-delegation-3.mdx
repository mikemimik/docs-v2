<Warning>
  Cette fonctionnalité utilise la délégation. Par défaut, la délégation est désactivée pour les tenants qui n’utilisaient pas de module complémentaire au 8 juin 2017. Les tenants Legacy qui utilisent actuellement un module complémentaire nécessitant la délégation peuvent continuer à utiliser cette fonctionnalité. Si la fonctionnalité de délégation est modifiée ou retirée du service à l’avenir, les clients qui l’utilisent actuellement en seront avisés à l’avance et disposeront d’un délai suffisant pour migrer. De plus, notez que la délégation ne prend pas en charge l’utilisation de domaines personnalisés; par conséquent, toute fonctionnalité qui en dépend pourrait ne pas être entièrement fonctionnelle avec un domaine personnalisé.
</Warning>

<div id="step-3-build-the-application">
  ## Étape 3 - Créer l’application
</div>

Dans cette étape, vous créerez une application monopage sans serveur à l’aide du framework AngularJS, que vous hébergerez dans un bucket AWS S3 configuré pour servir de site Web statique.

<div id="step-1-set-up-your-sample-application">
  ### Étape 1. Configurez votre application d’exemple
</div>

Pour démarrer simplement, [téléchargez un projet d’exemple](https://github.com/auth0/auth0-aws/tree/master/examples/api-gateway/client) propre à ce tutoriel. Connectez-vous pour que vos informations d’identification Auth0 soient préconfigurées.

Copiez le contenu de ce projet de départ dans un dossier local nommé `pets`, que vous utiliserez pour le reste de ce tutoriel. Dans ce dossier, mettez à jour `auth0-variables.js` avec le `AUTH0_CLIENT_ID` et le `AUTH0_CLIENT_ID` de votre application Auth0 (ces renseignements sont accessibles dans le [Dashboard de gestion](https://manage.auth0.com/#/applications) de l’application en question).

<div id="aws-s3-bucket">
  #### Bucket AWS S3
</div>

Assurez-vous d’avoir [créé le bucket AWS S3 configuré pour servir de site Web statique](http://docs.aws.amazon.com/gettingstarted/latest/swh/website-hosting-intro.html). Pendant la configuration, copiez le contenu du dossier `pets` vers votre bucket S3 pour alimenter le site Web en contenu.

Si vous utilisez un bucket existant, vous pouvez déplacer les fichiers avec la CLI AWS à l’aide de la commande suivante.

```bash lines
aws s3 cp --recursive --acl "public-read" ./ s3://{yourBucket}/
```

Avant de poursuivre, assurez-vous qu’au moins un utilisateur est associé à votre connexion Username-Password-Authentication (ou à la connexion de base de données associée à l’Application que vous utilisez). Pour profiter pleinement des fonctionnalités de votre application d’exemple et de son intégration avec AWS, vous aurez besoin de cet utilisateur pour tester l’authentification et obtenir l’accès.

Enfin, assurez-vous qu’Auth0 autorise l’authentification à partir de votre site Web en fournissant l’URL dans le champ **Allowed Origins** de la page **Settings** de votre Application. L’URL de votre site Web devrait ressembler à ceci :

`http://your-bucket.s3-website-us-east-1.amazonaws.com`

Si vous ne connaissez pas votre URL, vous pouvez la trouver sous l’onglet **Properties** de votre bucket S3.

Avant d’aller plus loin, testez la connexion à votre application. Ouvrez `http://your-bucket-domain/index.html` dans votre navigateur. Après la connexion, une boîte d’alerte indiquant &quot;getPets not implemented&quot; devrait s’afficher :

Vous devriez aussi voir la page d’affichage des animaux.

<div id="modify-the-role-and-principal-strings">
  #### Modifier les chaînes de caractères role et principal
</div>

Pour modifier les chaînes de caractères `role` et `principal` (qui correspondent aux deux dernières lignes de l’instruction `if` contenue dans la fonction fournie), indiquez les valeurs appropriées dans [Rules](https://manage.auth0.com/#/rules) :

```javascript lines
function (user, context, callback) {
  if (context.clientID === 'CLIENT_ID' &&
      context.protocol === 'delegation') {
    // définir les paramètres AWS
    context.addonConfiguration = context.addonConfiguration || {};
    context.addonConfiguration.aws = context.addonConfiguration.aws || {};
    context.addonConfiguration.aws.principal = 'arn:aws:iam::[omitted]:saml-provider/auth0-provider';
    context.addonConfiguration.aws.role = 'arn:aws:iam::[omitted]:role/auth0-role';
  }

  callback(null, user, context);
}
```

Assurez-vous de mettre à jour les valeurs ARN `role` et `[principal]` avec celles de votre intégration.

Copiez les fichiers mis à jour dans votre bucket S3 pour votre site Web.

Au besoin, vous pouvez définir un point d’arrêt dans le navigateur à `store.set('awstoken', delegation.Credentials);`. Lorsque vous vous déconnectez puis vous reconnectez, inspectez `delegation.Credentials` une fois au point d’arrêt. Vous verrez des valeurs familières comme AccessKeyId et SecretAccessKey :

```js lines
{
    AccessKeyId: "ASIAJB...BNQ",
    SecretAccessKey: "vS+b6...2Noav",
    SessionToken: "AQoDYBqsivOV...DdQW0gsKr8rgU=",
    Expiration: "2015-08-27T14:48:32.000Z"
}
```

Si vous ne voyez pas ces valeurs, assurez-vous d’avoir activé le module complémentaire Amazon Web Services dans l’onglet Addons de votre Application Auth0.

<div id="configure-the-getpets-method">
  #### Configurer la méthode `getPets`
</div>

Ouvrez `home.js` dans le dossier `home`, puis mettez à jour le contenu de `getPets` avec une méthode pour récupérer les données sur les animaux de compagnie (veillez à mettre à jour la région si vous n’exécutez pas l’application dans `us-east-1`) :

```javascript lines
function getPets() {
    // ceci n'est pas authentifié
    var apigClient = apigClientFactory.newClient({
        region: 'us-east-1' // La région où l'API est déployée
    });

    apigClient.petsGet({},{})
      .then(function(response) {
        console.log(response);
        $scope.pets = response.data;
        $scope.$apply();
      }).catch(function (response) {
        alert('pets get failed');
        showError(response);
    });
}
```

Copiez le code mis à jour dans votre S3 bucket. Actualisez la page pour voir deux animaux s’afficher (si vous avez exécuté le test décrit précédemment sur vos API qui ont créé ces animaux).

<div id="update-pets-with-the-aws-api-service">
  ### Mettre à jour les animaux de compagnie avec AWS API Service
</div>

Maintenant que vous avez une application Auth0 fonctionnelle avec API Gateway, vous allez ajouter une méthode pour mettre à jour la table `pets`.

Modifiez la logique de la méthode `putPets` afin de mettre à jour les animaux de compagnie à l’aide de votre fonction d’API. Cette fonction servira autant à ajouter qu’à supprimer des animaux de compagnie.

```javascript lines
function putPets(updatedPets) {
    var body = {pets: updatedPets};

    var apigClient = apigClientFactory.newClient({
        region: 'us-east-1' // définissez ceci selon la région dans laquelle vous travaillez.
    });

    apigClient.petsPost({},body)
      .then(function(response) {
        console.log(response);
       }).catch(function (response) {
        alert('pets update failed');
        showError(response);
      });
}
```

Copiez le code mis à jour dans votre bucket S3. Testez la méthode :

1. Déconnectez-vous, puis reconnectez-vous.
2. Entrez des valeurs pour `Pet Type` et `Pet Price`.
3. Cliquez sur **Save** pour envoyer vos données.

Vous devriez voir un message indiquant : &quot;Nous avons un `<Pet Type>` à vendre pour `<Pet Price>`&quot;, avec un bouton rouge **REMOVE** à sa gauche.

Pour renforcer la sécurité, ajoutez la fonction `getSecureApiClient` au début de la méthode `putPets` :

```javascript lines
function putPets(updatedPets) {
     var apigClient = getSecureApiClient();
}
```

Copiez le code dans votre bucket S3.

La fonction `getSecureApiClient` fournie récupère le jeton AWS dans le local storage, obtenu au moyen de délégation vers l’API, et utilise la clé d’accès, la clé secrète et le token de session :

```javascript lines
function getSecureApiClient() {
    var awstoken = store.get('awstoken');

    return apigClientFactory.newClient({
        accessKey: awstoken.AccessKeyId,
        secretKey: awstoken.SecretAccessKey,
        sessionToken: awstoken.SessionToken,
        region: 'us-east-1' // La région dans laquelle vous travaillez.
    });
  }
```
