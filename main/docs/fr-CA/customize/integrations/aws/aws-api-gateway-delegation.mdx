<Warning>
  Cette fonctionnalité utilise Delegation. Par défaut, Delegation est désactivée pour les tenants qui n’utilisaient pas d’add-on au 8 juin 2017. Les tenants Legacy qui utilisent actuellement un add-on nécessitant Delegation peuvent continuer à utiliser cette fonctionnalité. Si la fonctionnalité Delegation est modifiée ou retirée du service à un moment donné, les clients qui l’utilisent actuellement en seront avisés à l’avance et disposeront de suffisamment de temps pour migrer. De plus, notez que Delegation ne prend pas en charge l’utilisation de domaines personnalisés; toute fonctionnalité qui en dépend pourrait donc ne pas être entièrement fonctionnelle avec un domaine personnalisé.
</Warning>

Avec AWS, vous pouvez créer des API et des applications puissantes, sans serveur et hautement évolutives grâce à AWS Lambda, Amazon API Gateway et à une application JavaScript.

Une application sans serveur exécute du code personnalisé comme service de calcul, sans qu’il soit nécessaire de maintenir un environnement d’exploitation pour héberger votre service. À la place, un service comme [AWS Lambda](https://aws.amazon.com/lambda/) exécute votre code pour vous.

Amazon API Gateway étend les capacités d’AWS Lambda en ajoutant une couche de service devant vos fonctions Lambda afin de renforcer la sécurité, de gérer les transformations des messages d’entrée et de sortie et d’offrir des capacités comme le throttling et l’audit. Une approche sans serveur simplifie vos besoins opérationnels, puisque des enjeux comme la mise à l’échelle et la tolérance aux pannes relèvent désormais du service de calcul qui exécute votre code.

Cependant, vous voulez souvent associer vos API à vos utilisateurs existants, qu’ils proviennent de fournisseurs d’identité sociale comme Twitter et Facebook, ou de votre propre organisation par l’intermédiaire d’Active Directory ou d’une base de données client. Ce tutoriel montre comment autoriser l’accès à vos méthodes Amazon API Gateway pour vos utilisateurs existants à l’aide d’Auth0 Delegation pour AWS et de l’intégration avec AWS Identity and Access Management (IAM).

Ensuite, ce tutoriel vous guide dans la configuration d’Amazon API Gateway à l’aide de fonctions AWS Lambda, la sécurisation de ces fonctions avec des rôles AWS IAM, puis l’utilisation d’Auth0 Delegation pour obtenir un jeton pour le rôle AWS IAM. Il vous montrera ensuite comment attribuer différentes permissions à diverses catégories d’utilisateurs, comme les utilisateurs d’une base de données interne ou les utilisateurs d’un fournisseur d’identité sociale, et comment transmettre l’identité à l’aide d’un <Tooltip tip="JSON Web Token (JWT) : format standard de ID Token (et souvent de jeton d’accès) utilisé pour représenter des claims de façon sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT).
