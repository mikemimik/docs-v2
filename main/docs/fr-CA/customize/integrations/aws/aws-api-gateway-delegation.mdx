  Cette fonctionnalité utilise la délégation. Par défaut, la délégation est désactivée pour les tenants qui n’utilisaient pas d’add-on au 8 juin 2017. Les tenants Legacy qui utilisent actuellement un add-on nécessitant la délégation peuvent continuer à utiliser cette fonctionnalité. Si la fonctionnalité de délégation est modifiée ou retirée du service un jour, les clients qui l’utilisent actuellement en seront avisés à l’avance et disposeront de suffisamment de temps pour migrer. De plus, notez que la délégation ne prend pas en charge l’utilisation de domaines personnalisés; les fonctionnalités qui en dépendent pourraient donc ne pas être entièrement fonctionnelles avec un domaine personnalisé.
</Warning>

Avec AWS, vous pouvez créer des API et des applications puissantes, sans serveur et hautement évolutives grâce à AWS Lambda, Amazon API Gateway et à une application JavaScript.

Une application sans serveur exécute du code personnalisé dans un service de calcul, sans qu’il soit nécessaire de maintenir un environnement d’exploitation pour héberger votre service. À la place, un service comme [AWS Lambda](https://aws.amazon.com/lambda/) exécute votre code pour vous.

Amazon API Gateway étend les capacités d’AWS Lambda en ajoutant une couche de service devant vos fonctions Lambda afin de renforcer la sécurité, de gérer les transformations des messages d’entrée et de sortie, et d’offrir des capacités comme la limitation du débit et l’audit. Une approche sans serveur simplifie vos besoins opérationnels, puisque des aspects comme la mise à l’échelle et la tolérance aux pannes relèvent désormais du service de calcul qui exécute votre code.

Cependant, vous voudrez souvent lier vos API à vos utilisateurs existants, qu’ils proviennent de fournisseurs d’identité sociale comme Twitter et Facebook, ou de votre propre organisation via Active Directory ou une base de données client. Ce didacticiel montre comment autoriser l’accès à vos méthodes Amazon API Gateway pour vos utilisateurs existants en utilisant la délégation Auth0 pour AWS et l’intégration avec AWS Identity and Access Management (IAM).

Ensuite, le didacticiel vous guide dans la configuration d’Amazon API Gateway à l’aide de fonctions AWS Lambda, la sécurisation de ces fonctions avec des rôles AWS IAM, puis l’utilisation de la délégation Auth0 pour obtenir un jeton pour le rôle AWS IAM. Il vous montrera ensuite comment attribuer différentes autorisations à diverses catégories d’utilisateurs, comme les utilisateurs d’une base de données interne ou d’un fournisseur d’identité sociale, et comment transmettre l’identité au moyen d’un <Tooltip tip="JSON Web Token (JWT) : format standard de jeton d’identification (et souvent de jeton d’accès) utilisé pour représenter des claims de manière sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT).
