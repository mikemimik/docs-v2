Il existe de nombreuses façons d’intégrer Auth0 à des produits et services Amazon, selon vos besoins précis.

<div id="product-integrations">
  ## Intégrations de produits
</div>

Vous pouvez en apprendre davantage sur l’utilisation des produits ci-dessous en complément de vos services Auth0 et AWS :

* [CloudFront](/docs/fr-CA/customize/custom-domains/self-managed-certificates/configure-aws-cloudfront-for-use-as-reverse-proxy) : utilisez-le comme proxy inverse avec votre <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=custom+domain">domaine personnalisé</Tooltip>.
* [Simple Email Service (SES)](/docs/fr-CA/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider) : gérez les communications par courriel avec vos utilisateurs.
* [EventBridge](https://marketplace.auth0.com/integrations/amazon-log-streaming) : transmettez les journaux à EventBridge.
* [Cognito](/docs/fr-CA/customize/integrations/aws/amazon-cognito) : utilisez-le comme backend pour votre application.
* AWS S3 et DynamoDB :

  * Auth0 fournit des points de terminaison de passerelle VPC pour accéder à AWS S3 et DynamoDB, ce qui peut être nécessaire pour les [Actions](/docs/fr-CA/customize/actions). Cela nécessite des stratégies IAM basées sur `aws:SourceVpc`, comme décrit dans [Gateway endpoints for Amazon S3](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-s3.html) et [Gateway endpoints for Amazon DynamoDB](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-ddb.html). Veuillez communiquer avec le [soutien Auth0](https://www.support.auth0.com) pour obtenir plus de détails sur les identifiants VPC.

<div id="identity-and-access-management-iam-integrations">
  ## Intégrations de gestion des identités et des accès (IAM)
</div>

Nous avons plusieurs articles pratiques pour configurer AWS afin de le faire fonctionner avec Auth0 :

* [Sécuriser les endpoints d’AWS API Gateway à l’aide d’autoriseurs personnalisés](/docs/fr-CA/customize/integrations/aws/aws-api-gateway-custom-authorizers) : Acceptez les <Tooltip tip="Jeton d'accès : information d'autorisation, sous la forme d'une chaîne opaque ou d'un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+tokens">jetons d&#39;accès</Tooltip> émis par Auth0.
* [Utiliser les balises de session AWS avec les API et les ressources AWS](/docs/fr-CA/customize/integrations/aws/session-tags) : Ajoutez des balises aux ressources et attribuez aux utilisateurs des paires clé/valeur pour le RBAC.
* [Configurer AWS pour l’authentification unique](/docs/fr-CA/customize/integrations/aws/configure-amazon-web-services-for-sso) : Permettez à vos utilisateurs de se connecter à AWS au moyen de n’importe quel <Tooltip tip="Fournisseur d'identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d&#39;identité</Tooltip> pris en charge.
* [Ajouter la connexion Amazon à votre application](/docs/fr-CA/customize/integrations/aws/configure-amazon-web-services-for-sso) : Permettez aux utilisateurs de se connecter à votre application avec leur profil Amazon.
