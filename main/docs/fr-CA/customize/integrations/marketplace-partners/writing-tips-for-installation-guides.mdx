Si vous ne pouvez pas consulter un guide de style complet, consultez ces conseils pour améliorer sensiblement votre contenu.

<div id="use-active-voice">
  ## Utilisez la voix active
</div>

Évitez la voix passive (verbe `être` + participe passé). Pour rendre votre texte plus dynamique, plus clair et (souvent) plus concis, essayez d’indiquer clairement qui agit et d’employer la voix active chaque fois que possible.

| Préféré                                                                                                                                                                                                                                                                                                       | Déconseillé                                                                                                                                                                                                                                                           |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Une fois que vous avez ajouté la configuration requise, Auth0 traitera toutes les connexions de votre tenant au moyen de cette Rule. Avant d’activer l’intégration en Production, assurez-vous d’avoir correctement configuré tous les composants et d’avoir effectué la vérification dans un tenant de test. | Une fois la configuration requise ajoutée, toutes les connexions de votre tenant seront traitées par cette Rule. Assurez-vous que tous les composants ont été correctement configurés et vérifiés dans un tenant de test avant d’activer l’intégration en Production. |

<div id="make-link-titles-scannable">
  ## Rendez les titres de liens faciles à repérer
</div>

Lorsque vous ajoutez des liens vers d’autres ressources, assurez-vous que le lien décrit l’action ou la tâche présentée dans le document vers lequel il mène. Un texte de lien générique nuit au repérage rapide, car il oblige les lecteurs à lire tout le texte autour du lien pour savoir quoi sélectionner.

| À privilégier                                                                                                                                                                                                 | À éviter                                            |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| Vous pouvez utiliser l’Auth0 Dashboard ou la Management API pour [personnaliser le texte de l’expérience Universal Login](/docs/fr-CA/customize/login-pages/universal-login/customize-templates).                   | Cliquez [ici](/docs/fr-CA/get-started).                   |
| Pour savoir comment attribuer des rôles aux membres d’Organizations, consultez [Ajouter des rôles aux membres d’une organization](/docs/fr-CA/manage-users/organizations/configure-organizations/add-member-roles). | Vous pouvez consulter [ce lien](/docs/fr-CA/get-started). |

<div id="write-device-agnostic-directions">
  ## Rédigez des instructions qui ne dépendent pas de l’appareil
</div>

N’oubliez pas que les interfaces utilisateur s’affichent différemment selon l’appareil. N’oubliez pas non plus que certains utilisateurs peuvent consulter votre contenu à l’aide de technologies d’adaptation.

| Préféré                           | Déconseillé                                                    |
| --------------------------------- | -------------------------------------------------------------- |
| Sélectionnez **Create**.          | Cliquez sur **Create**.                                        |
| Passez à la vue **Applications**. | Passez à l’onglet **Applications**.                            |
| Repérez **Client Secret**.        | Faites défiler l’écran vers le haut jusqu’à **Client Secret**. |

<div id="use-verbs-and-nounadjectives-properly">
  ## Utilisez correctement les verbes, les noms et les adjectifs
</div>

Faites attention aux mots qui peuvent servir à la fois de verbes, de noms ou d’adjectifs. Les formes verbales exigent généralement un espace entre les mots, tandis que les noms et les adjectifs peuvent s’écrire en un seul mot.

| À privilégier                 | À éviter                       |
| ----------------------------- | ------------------------------ |
| Log in to your account.       | Login to your account.         |
| Auth0 processes logins.       | Auth0 processes log ins.       |
| Set up your hardware.         | Setup your hardware.           |
| Describe your hardware setup. | Describe your hardware set up. |

<div id="use-sentence-case-for-headings">
  ## Utilisez la casse de phrase dans les titres
</div>

Mettez une majuscule uniquement au premier mot ainsi qu’aux noms propres et aux noms de produits Auth0.

| À privilégier                   | À éviter                        |
| ------------------------------- | ------------------------------- |
| Ajouter une dépendance          | Ajouter une Dépendance          |
| Valider la reprise de connexion | Valider la Reprise de Connexion |
| Créer une Action                | Créer une action                |

<div id="use-simple-tense-for-headings">
  ## Utilisez des temps simples dans les titres
</div>

Évitez les tournures inutilement longues en employant des temps simples dans les titres.

| Préféré                                     | Déconseillé                                         |
| ------------------------------------------- | --------------------------------------------------- |
| Attribuer et modifier des utilisateurs      | Attribution et modification des utilisateurs        |
| Connecter une base de données personnalisée | Comment connecter une base de données personnalisée |

<div id="write-steps-concisely">
  ## Rédigez les étapes avec concision
</div>

Lorsque vous rédigez un texte d’introduction pour des étapes, allez droit au but et soyez concis. N’oubliez pas que les utilisateurs savent déjà quoi faire lorsqu’une liste d’étapes numérotées leur est présentée.

| Préféré                 | À éviter                       |
| ----------------------- | ------------------------------ |
| Pour installer le SDK : | Suivez les étapes ci-dessous : |

<div id="set-user-expectations-in-steps">
  ## Établissez les attentes des utilisateurs dans les étapes
</div>

Indiquez aux utilisateurs le résultat attendu avant de décrire ce qu’ils doivent faire.

| Recommandé                                                                      | Déconseillé                                                                    |
| ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Pour activer cette intégration, sélectionnez **Enregistrer les modifications**. | Sélectionnez **Enregistrer les modifications** pour activer cette intégration. |

<div id="list-actions-within-steps-in-order">
  ## Présentez les actions à l’intérieur des étapes dans l’ordre
</div>

Dans chaque étape, indiquez à l’utilisateur ce qu’il doit faire dans l’ordre.

| À privilégier                                                                    | À éviter                                                                   |
| -------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Repérez la section **Intégrations** et sélectionnez **Ajouter une intégration**. | Sélectionnez **Ajouter une intégration** dans la section **Intégrations**. |

<div id="use-notes-and-warning-appropriately">
  ## Utilisez les notes et les avertissements à bon escient
</div>

Les notes et les avertissements n’ont pas la même fonction. Les notes servent à fournir des renseignements généraux utiles, tandis que les avertissements signalent des éléments dont le non-respect peut entraîner un échec.

Si un renseignement doit être présenté sous forme d’avertissement, vous devriez l’inclure exactement à l’endroit où l’utilisateur devra effectuer l’action connexe. Les utilisateurs ne devraient pas avoir à chercher les avertissements.
