Si vous n’avez pas accès à un guide de style complet, consultez ces conseils pour améliorer considérablement votre contenu.

<div id="use-active-voice">
  ## Utilisez la voix active
</div>

Évitez la voix passive (verbe `être` + participe passé). Pour rendre votre texte plus dynamique, plus clair et (souvent) plus court, essayez d’utiliser des agents et la voix active chaque fois que possible.

| Préférable                                                                                                                                                                                                                                                               | Déconseillé                                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Une fois la configuration requise ajoutée, Auth0 traitera toutes les connexions de votre tenant au moyen de cette Rule. Avant d’activer l’intégration en production, assurez-vous que tous les composants sont correctement configurés et validés sur un tenant de test. | Une fois la configuration requise ajoutée, toutes les connexions de votre tenant seront traitées par cette Rule. Assurez-vous que tous les composants ont été correctement configurés et validés sur un tenant de test avant d’activer l’intégration en production. |

<div id="make-link-titles-scannable">
  ## Rendez les titres de liens faciles à parcourir
</div>

Lorsque vous ajoutez des liens vers d’autres ressources, assurez-vous que le lien décrit l’action ou la tâche dans le document vers lequel il renvoie. Un texte de lien générique nuit à la facilité de repérage, car il oblige les lecteurs à lire tout le texte autour du lien pour savoir quoi sélectionner.

| À privilégier                                                                                                                                                                                                 | À éviter                                         |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| Vous pouvez utiliser l’Auth0 Dashboard ou la Management API pour [personnaliser le texte de l’expérience Universal Login](/docs/fr-CA/customize/login-pages/universal-login/customize-templates).                   | Cliquez [ici](/docs/fr-CA/get-started).                |
| Pour savoir comment attribuer des rôles aux membres d’organizations, consultez [Ajouter des rôles aux membres d’une organization](/docs/fr-CA/manage-users/organizations/configure-organizations/add-member-roles). | Vous pouvez suivre [ce lien](/docs/fr-CA/get-started). |

<div id="write-device-agnostic-directions">
  ## Rédigez des instructions qui ne dépendent pas de l’appareil
</div>

N’oubliez pas que les interfaces utilisateur s’affichent différemment d’un appareil à l’autre. N’oubliez pas non plus que les utilisateurs peuvent utiliser des appareils adaptés pour consulter votre contenu.

| Préférable                        | À éviter                                               |
| --------------------------------- | ------------------------------------------------------ |
| Sélectionnez **Create**.          | Cliquez sur **Create**.                                |
| Passez à la vue **Applications**. | Passez à l’onglet **Applications**.                    |
| Repérez **Client Secret**.        | Faites défiler vers le haut jusqu’à **Client Secret**. |

<div id="use-verbs-and-nounadjectives-properly">
  ## Utilisez correctement les verbes et les noms/adjectifs
</div>

Faites attention aux mots qui peuvent servir à la fois de verbes et de noms ou d’adjectifs. En anglais, les formes verbales prennent habituellement un espace entre les mots, tandis que les noms et les adjectifs peuvent s’écrire en un seul mot.

| Préféré                       | Déconseillé                    |
| ----------------------------- | ------------------------------ |
| Log in to your account.       | Login to your account.         |
| Auth0 processes logins.       | Auth0 processes log ins.       |
| Set up your hardware.         | Setup your hardware.           |
| Describe your hardware setup. | Describe your hardware set up. |

<div id="use-sentence-case-for-headings">
  ## Utilisez la casse de phrase pour les titres
</div>

Mettez une majuscule uniquement au premier mot, ainsi qu’aux noms propres et aux noms de produits Auth0.

| À privilégier                   | À éviter                        |
| ------------------------------- | ------------------------------- |
| Ajouter une dépendance          | Ajouter une Dépendance          |
| Valider la reprise de connexion | Valider la Reprise de connexion |
| Créer une Action                | Créer une action                |

<div id="use-simple-tense-for-headings">
  ## Utilisez le temps simple pour les titres
</div>

Évitez les formulations inutilement longues en employant des temps simples dans les titres.

| Préféré                                     | Déconseillé                                         |
| ------------------------------------------- | --------------------------------------------------- |
| Attribuer et modifier des utilisateurs      | Attribution et modification des utilisateurs        |
| Connecter une base de données personnalisée | Comment connecter une base de données personnalisée |

<div id="write-steps-concisely">
  ## Rédigez les étapes de façon concise
</div>

Lorsque vous rédigez un texte d’introduction pour des étapes, allez droit au but et restez concis. N’oubliez pas que les utilisateurs savent déjà quoi faire lorsqu’une liste d’étapes numérotées leur est présentée.

| Préféré                 | À éviter                       |
| ----------------------- | ------------------------------ |
| Pour installer le SDK : | Suivez les étapes ci-dessous : |

<div id="set-user-expectations-in-steps">
  ## Définissez les attentes des utilisateurs dans les étapes
</div>

Indiquez d’abord aux utilisateurs le résultat attendu, puis décrivez ce qu’ils doivent faire pour l’obtenir.

| Préféré                                                                         | Déconseillé                                                                    |
| ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Pour activer cette intégration, sélectionnez **Enregistrer les modifications**. | Sélectionnez **Enregistrer les modifications** pour activer cette intégration. |

<div id="list-actions-within-steps-in-order">
  ## Énumérez les actions dans les étapes, dans l’ordre
</div>

Dans chaque étape, indiquez à l’utilisateur quoi faire dans l’ordre où il doit le faire.

| Préféré                                                                             | À éviter                                                                   |
| ----------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Repérez la section **Intégrations**, puis sélectionnez **Ajouter une intégration**. | Sélectionnez **Ajouter une intégration** sous la section **Intégrations**. |

<div id="use-notes-and-warning-appropriately">
  ## Utilisez les notes et les avertissements de façon appropriée
</div>

Les notes et les avertissements n’ont pas la même fonction. Les notes devraient présenter des renseignements généraux utiles, tandis que les avertissements contiennent des renseignements dont le non-respect peut entraîner un échec.

Si des renseignements doivent être présentés sous forme d’avertissement, vous devriez placer l’avertissement exactement à l’endroit où l’utilisateur doit effectuer l’action correspondante. Les utilisateurs ne devraient pas avoir à chercher les avertissements.
