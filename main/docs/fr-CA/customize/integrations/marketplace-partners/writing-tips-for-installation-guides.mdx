Si vous ne pouvez pas consulter un guide de style complet, les conseils suivants vous aideront à améliorer considérablement votre contenu.

<div id="use-active-voice">
  ## Utilisez la voix active
</div>

Évitez la voix passive (être + participe passé). Pour rendre votre texte plus dynamique, plus clair et (souvent) plus court, essayez d’employer des sujets concrets et la voix active chaque fois que possible.

| Préféré                                                                                                                                                                                                                                                                      | Déconseillé                                                                                                                                                                                                                                                                   |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Une fois la configuration requise ajoutée, Auth0 traitera toutes les connexions de votre tenant à l’aide de cette Rule. Avant d’activer l’intégration dans Production, assurez-vous d’avoir bien configuré tous les éléments et de les avoir validés dans un tenant de test. | Une fois la configuration requise ajoutée, toutes les connexions de votre tenant seront traitées par cette Rule. Veuillez vous assurer que tous les éléments ont été correctement configurés et validés dans un tenant de test avant d’activer l’intégration dans Production. |

<div id="make-link-titles-scannable">
  ## Rendez les titres de liens faciles à balayer
</div>

Lorsque vous créez des liens vers d’autres ressources, assurez-vous que le lien décrit l’action ou la tâche du document vers lequel il renvoie. Un texte de lien générique nuit au balayage rapide, car il oblige les lecteurs à lire tout le texte autour du lien pour savoir quoi sélectionner.

| Préférable                                                                                                                                                                                                   | Déconseillé                                         |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------- |
| Vous pouvez utiliser l’Auth0 Dashboard ou la Management API pour [personnaliser le texte de l’expérience Universal Login](/docs/fr-CA/customize/login-pages/universal-login/customize-templates).                  | Cliquez [ici](/docs/fr-CA/get-started).                   |
| Pour savoir comment attribuer des rôles aux membres d’Organizations, consultez [Ajouter des rôles aux membres de l’Organization](/docs/fr-CA/manage-users/organizations/configure-organizations/add-member-roles). | Vous pouvez consulter [ce lien](/docs/fr-CA/get-started). |

<div id="write-device-agnostic-directions">
  ## Rédigez des instructions qui ne dépendent pas de l’appareil
</div>

N’oubliez pas que les interfaces utilisateur s’affichent différemment d’un appareil à l’autre. N’oubliez pas non plus que des utilisateurs peuvent consulter votre contenu à l’aide de technologies d’assistance.

| Préféré                           | Déconseillé                                            |
| --------------------------------- | ------------------------------------------------------ |
| Sélectionnez **Create**.          | Cliquez sur **Create**.                                |
| Passez à la vue **Applications**. | Passez à l’onglet **Applications**.                    |
| Repérez **Client Secret**.        | Faites défiler vers le haut jusqu’à **Client Secret**. |

<div id="use-verbs-and-nounadjectives-properly">
  ## Employez correctement les verbes et les noms/adjectifs
</div>

Faites attention aux mots qui peuvent être employés à la fois comme verbes et comme noms ou adjectifs. Les formes verbales exigent généralement un espace entre les mots, tandis que les noms et les adjectifs peuvent être composés.

| À privilégier                 | À éviter                       |
| ----------------------------- | ------------------------------ |
| Log in to your account.       | Login to your account.         |
| Auth0 processes logins.       | Auth0 processes log ins.       |
| Set up your hardware.         | Setup your hardware.           |
| Describe your hardware setup. | Describe your hardware set up. |

<div id="use-sentence-case-for-headings">
  ## Utilisez la casse de phrase dans les titres
</div>

Mettez une majuscule uniquement au premier mot, ainsi qu’aux noms propres et aux noms de produits Auth0.

| À privilégier                   | À éviter                        |
| ------------------------------- | ------------------------------- |
| Ajouter une dépendance          | Ajouter une Dépendance          |
| Valider la reprise de connexion | Valider La Reprise De Connexion |
| Créer une Action                | Créer une action                |

<div id="use-simple-tense-for-headings">
  ## Utilisez un temps simple dans les titres
</div>

Évitez les tournures inutilement longues en employant des temps simples dans les titres.

| À privilégier                               | À éviter                                            |
| ------------------------------------------- | --------------------------------------------------- |
| Assigner et modifier des utilisateurs       | Comment assigner et modifier des utilisateurs       |
| Connecter une base de données personnalisée | Comment connecter une base de données personnalisée |

<div id="write-steps-concisely">
  ## Rédigez des étapes concises
</div>

Lorsque vous rédigez le texte d’introduction d’une série d’étapes, allez droit au but et soyez concis. N’oubliez pas que les utilisateurs savent déjà ce qu’ils ont à faire avec une liste d’étapes numérotées.

| Préféré                 | Déconseillé                    |
| ----------------------- | ------------------------------ |
| Pour installer le SDK : | Suivez les étapes ci-dessous : |

<div id="set-user-expectations-in-steps">
  ## Définissez les attentes des utilisateurs dans les étapes
</div>

Indiquez aux utilisateurs quel sera le résultat de leur action avant de décrire l’action qu’ils doivent effectuer.

| Préférable                                                                      | Déconseillé                                                                    |
| ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Pour activer cette intégration, sélectionnez **Enregistrer les modifications**. | Sélectionnez **Enregistrer les modifications** pour activer cette intégration. |

<div id="list-actions-within-steps-in-order">
  ## Énumérez les actions dans chaque étape, dans l’ordre
</div>

Dans chaque étape, indiquez à l’utilisateur ce qu’il doit faire dans l’ordre.

| Recommandé                                                                          | À éviter                                                                   |
| ----------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Repérez la section **Intégrations**, puis sélectionnez **Ajouter une intégration**. | Sélectionnez **Ajouter une intégration** dans la section **Intégrations**. |

<div id="use-notes-and-warning-appropriately">
  ## Utilisez correctement les notes et les avertissements
</div>

Les notes et les avertissements n’ont pas la même fonction. Les notes devraient fournir des renseignements généraux utiles à connaître, tandis que les avertissements contiennent des renseignements qui peuvent entraîner un échec si on n’en tient pas compte.

Si des renseignements doivent être présentés sous forme d’avertissement, placez l’avertissement exactement là où l’utilisateur doit effectuer l’action correspondante. Les utilisateurs ne devraient pas avoir à chercher les avertissements.
