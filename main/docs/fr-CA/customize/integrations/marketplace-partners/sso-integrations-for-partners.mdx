<Tooltip tip="Single Sign-On (SSO) : service qui, après qu'un utilisateur se connecte à une application, le connecte automatiquement à d'autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip> (authentification unique) utilise une identité Auth0 pour se connecter, au moyen de <Tooltip tip="Single Sign-On (SSO) : service qui, après qu'un utilisateur se connecte à une application, le connecte automatiquement à d'autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SAML">SAML</Tooltip>, à un service externe. Cela permet aux clients ayant des applications internes d&#39;utiliser l&#39;authentification unique pour accéder à des fournisseurs tiers à l&#39;aide de leur identité Auth0.

Les [intégrations SSO offertes dans Marketplace](https://marketplace.auth0.com/features/sso-integrations) définissent à la fois les valeurs de configuration SAML standard requises pour connecter Auth0 à des fournisseurs externes et les champs nécessaires pour des valeurs supplémentaires propres au client. Dans ces intégrations, Auth0 agit comme <Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d&#39;identité</Tooltip> dans un scénario initié par le fournisseur de services (SP). Pour savoir comment configurer Auth0 pour ces intégrations, consultez [Configurer Auth0 comme fournisseur d&#39;identité SAML](/docs/fr-CA/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

La partie fournie par nos partenaires est la configuration SAML par défaut nécessaire au fonctionnement de cette transaction. Elle comprendra des valeurs ou des champs demandés par le client pour certains ou l&#39;ensemble des attributs répertoriés dans [Personnaliser les assertions SAML : attributs d&#39;assertion SAML](/docs/fr-CA/authenticate/protocols/saml/saml-configuration/customize-saml-assertions).

Pour créer et tester une intégration, [créez un tenant](https://auth0.com/signup) dans Auth0 et suivez la documentation ci-dessus pour créer une intégration SSO fonctionnelle avec votre service.

De plus, pour vous aider à créer votre intégration, vous pouvez utiliser notre [intégration SSO](https://github.com/auth0-marketplace/TEMPLATE-sso).

<div id="submit-your-sso-integration">
  ## Soumettez votre intégration SSO
</div>

Si vous êtes prêt à devenir partenaire et à soumettre votre intégration, veuillez créer un compte partenaire dans le [portail des partenaires](https://partners.marketplace.auth0.com/) et faire ce qui suit :

1. Remplissez le profil partenaire, qui comprend une brève description de votre entreprise, du contenu juridique et marketing, le logo de l’entreprise, les conditions d’utilisation, la politique de confidentialité, les coordonnées et le courriel de soutien.
2. Soumettez votre intégration, y compris le code d’intégration, le guide d’installation et des renseignements comme le nom et la description. Vous pouvez également ajouter, au besoin, 3 propositions de valeur à votre intégration.

**Nous aurons besoin de la configuration et de la documentation suivantes :**

* Le JSON utilisé pour configurer l’Application avec le module complémentaire SAML fonctionnel
* Une liste des champs qui doivent être affichés au client qui configure l’intégration (comme **URL de rappel**, **<Tooltip tip="Audience : identifiant unique de l’audience d’un jeton émis. Nommée aud dans un jeton, sa valeur contient l’ID d’une application (Client ID) pour un ID Token ou d’une API (API Identifier) pour un jeton d’accès." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Audience">Audience</Tooltip>**, et **certificat de signature**)
* Un guide d’installation rédigé en Markdown.
