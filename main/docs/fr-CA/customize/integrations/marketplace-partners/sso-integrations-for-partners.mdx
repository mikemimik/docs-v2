<Tooltip tip="Single Sign-On (SSO) : Service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip> (authentification unique) intégration utilise une identité Auth0 pour se connecter, via <Tooltip tip="Single Sign-On (SSO) : Service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SAML">SAML</Tooltip>, à un service externe. Cela permet aux clients qui ont des applications internes d’utiliser le SSO pour accéder à des fournisseurs tiers au moyen de leur identité Auth0.

Les [intégrations SSO offertes dans Marketplace](https://marketplace.auth0.com/features/sso-integrations) définissent à la fois les valeurs de configuration SAML standard nécessaires pour connecter Auth0 à des fournisseurs externes, ainsi que les champs requis pour des valeurs additionnelles propres au client. Dans ces intégrations, Auth0 agit comme <Tooltip tip="Identity Provider (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> dans un scénario initié par le fournisseur de services (SP). Pour savoir comment configurer Auth0 pour ces intégrations, consultez [Configurer Auth0 comme fournisseur d’identité SAML](/docs/fr-CA/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

La partie fournie par nos partenaires est la configuration SAML par défaut nécessaire au bon fonctionnement de cette transaction. Elle comprendra des valeurs ou des champs demandés par le client pour certains ou l’ensemble des attributs énumérés dans [Personnaliser les assertions SAML : attributs d’assertion SAML](/docs/fr-CA/authenticate/protocols/saml/saml-configuration/customize-saml-assertions).

Pour créer et tester une intégration, [créez un tenant](https://auth0.com/signup) dans Auth0 et suivez la documentation ci-dessus pour créer une intégration SSO fonctionnelle avec votre service.

De plus, pour vous aider à créer votre intégration, vous pouvez vous appuyer sur notre [intégration SSO](https://github.com/auth0-marketplace/TEMPLATE-sso).

<div id="submit-your-sso-integration">
  ## Soumettez votre intégration SSO
</div>

Si vous êtes prêt à devenir partenaire et à soumettre votre intégration, veuillez créer un compte partenaire dans le [portail des partenaires](https://partners.marketplace.auth0.com/) et suivre les étapes suivantes :

1. Remplissez le profil partenaire, qui comprend une brève description de votre entreprise, du contenu juridique et marketing, le logo de l’entreprise, les conditions d’utilisation, la politique de confidentialité, les coordonnées et l’adresse courriel du soutien.
2. Soumettez votre intégration, y compris le code d’intégration, le guide d’installation et des renseignements comme le nom et la description. Vous pouvez également ajouter 3 propositions de valeur à votre intégration.

**Nous aurons besoin des éléments de configuration et de la documentation suivants :**

* Le JSON utilisé pour configurer l’Application de l’add-on SAML fonctionnelle
* Une liste des champs qui doivent être affichés au client qui configure l’intégration (comme **URL de rappel**, **<Tooltip tip="Audience : identifiant unique du destinataire visé par un jeton émis. Appelée aud dans un jeton, sa valeur contient l’ID d’une application (Client ID) pour un ID Token ou d’une API (API Identifier) pour un jeton d’accès." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Audience">Audience</Tooltip>**, et **certificat de signature**)
* Un guide d’installation rédigé en Markdown.
