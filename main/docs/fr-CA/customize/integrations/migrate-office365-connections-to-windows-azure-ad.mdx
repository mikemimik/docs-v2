---

Office365 est déprécié. Vous devriez migrer vos connexions Office365 vers des connexions Windows Azure AD.

Depuis le début, nous prenons en charge l’authentification des utilisateurs avec Office365. Office365 a toujours utilisé Windows Azure AD en arrière-plan, mais il n’y avait pas de bonne interface utilisateur pour créer une « application » dans Windows Azure AD. C’est pourquoi vous deviez la créer dans le Seller Dashboard. Dorénavant, Microsoft veut que vous utilisiez Windows Azure AD, et vous pouvez maintenant créer facilement un annuaire associé à votre compte Office365 et à l’application.

<div id="how-to-migrate-to-azure">
  ## Comment migrer vers Azure
</div>

1. Créez un abonnement à Windows Azure AD (gratuit)
2. Créez un annuaire (qui sera associé à votre compte Office365) et une application. Pour en savoir plus, consultez [Connectez votre application à Microsoft Azure Active Directory](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

Si vous utilisiez le `user_id` dans votre application, sachez qu’il passera de `office365|....some-guid....` à `waad|...email....`.
