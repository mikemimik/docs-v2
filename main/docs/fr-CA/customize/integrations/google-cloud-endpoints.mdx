[Google Cloud Endpoints (GCE)](https://cloud.google.com/endpoints/) est un système de gestion d’API qui offre des fonctionnalités pour vous aider à créer, à maintenir et à sécuriser vos API. GCE utilise [OpenAPI](https://www.openapis.org/) pour définir les endpoints de votre API, ses entrées et sorties, ses erreurs et sa description de sécurité.

Pour en savoir plus sur la spécification OpenAPI, consultez le dépôt [OpenAPI Specification](https://github.com/OAI/OpenAPI-Specification) sur GitHub.

Ce tutoriel vous montrera comment sécuriser Google Cloud Endpoints avec Auth0.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, vous aurez besoin d’une API GCE déployée. Si vous n’avez pas encore créé d’API, consultez le [Cloud Endpoints Quickstart](https://cloud.google.com/endpoints/docs/quickstart-endpoints) dans la documentation de Google.

Ce guide vous montrera comment créer une API GCE simple avec un seul endpoint, `/airportName`, qui renvoie le nom d’un aéroport à partir de son [code IATA](https://en.wikipedia.org/wiki/IATA_airport_code) à trois lettres.

<div id="update-the-api-configuration">
  ## Mettre à jour la configuration de l’API
</div>

Ensuite, nous mettrons à jour le fichier de configuration OpenAPI de la API GCE. Dans l’exemple d’API créé pendant le Quickstart, ce fichier est `openapi.yaml`.

<div id="add-security-definitions">
  ### Ajouter des définitions de sécurité
</div>

Ouvrez le fichier de configuration et ajoutez une nouvelle section `securityDefinitions`. Dans cette section, ajoutez une nouvelle définition (`auth0_jwt`) avec les champs suivants :

| Field                | Description                                                                                                                                                                                                |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `authorizationUrl`   | L’URL d’autorisation doit être définie sur `"https://{yourDomain}/authorize"`                                                                                                                              |
| `flow`               | Le flux utilisé par le schéma de sécurité OAuth2. Les valeurs valides sont `"implicit"`, `"password"`, `"application"` ou `"accessCode"`.                                                                  |
| `type`               | Le type du schéma de sécurité. Les valeurs valides sont `"basic"`, `"apiKey"` ou `"oauth2"`                                                                                                                |
| `x-google-issuer`    | L’émetteur d’un justificatif d’authentification doit être défini sur `"https://{yourDomain}/"`                                                                                                             |
| `x-google-jwks_uri`  | L’URI de l’ensemble de clés publiques pour valider la signature du [JSON Web Token (JWT)](/docs/fr-CA/glossary?term=JSON+Web+Token+%28JWT%29). Définissez-la sur `"https://\{yourDomain}/.well-known/jwks.json"` |
| `x-google-audiences` | L’identifiant de l’API; assurez-vous que cette valeur correspond à celle que vous avez définie dans le Auth0 Dashboard pour l’API.                                                                         |

<div id="update-the-endpoint">
  ### Mettre à jour l’endpoint
</div>

Maintenant, mettez à jour l’endpoint en ajoutant un champ `security` avec le `securityDefinition` que nous avons créé à l’étape précédente.

```yaml lines
paths:
  "/airportName":
    get:
      description: "Get the airport name for a given IATA code."
      operationId: "airportName"
      parameters:
        -
          name: iataCode
          in: query
          required: true
          type: string
      responses:
        200:
          description: "Success."
          schema:
            type: string
        400:
          description: "The IATA code is invalid or missing."
      security:
       - auth0_jwt: []
```

Dans l’exemple ci-dessus, le champ `security` indique au proxy GCE que notre chemin d’accès `/airportName` doit être protégé par la définition `auth0-jwt`.

Effectuez une requête `GET` vers votre API avec un en-tête Authorization de `Bearer {ACCESS_TOKEN}` pour y accéder avec autorisation :
