title: Configurer Azure CDN comme proxy inverse
---

<Card title="La disponibilité varie selon le plan Auth0">
  Votre plan Auth0 ou votre entente personnalisée détermine si cette fonctionnalité est offerte. Pour en savoir plus, consultez la page [Tarification](https://auth0.com/pricing).
</Card>

Pour configurer Azure CDN comme proxy inverse, un plan Azure CDN Premium est requis.

1. [Configurez les domaines personnalisés avec des certificats autogérés](/docs/fr-CA/customize/custom-domains/self-managed-certificates) si ce n’est pas déjà fait. Prenez en note les valeurs **Origin Domain Name** et **cname-api-key**, car vous en aurez besoin plus tard.
2. Connectez-vous à [Azure Portal](https://portal.azure.com/).
3. [Créez un profil Azure CDN et un endpoint](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint). Pour les paramètres de l’endpoint CDN, indiquez la valeur **Origin Domain Name** de l’Auth0 Dashboard.
4. [Ajoutez votre domaine personnalisé à votre endpoint Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup).
5. [Configurez HTTPS pour votre domaine personnalisé Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate). Ce processus vous oblige à vérifier que vous êtes propriétaire du domaine. Une fois cette étape terminée, le déploiement du certificat dans tous les points de présence du CDN peut prendre jusqu’à 6 heures.
6. Configurez la communication du domaine personnalisé avec Auth0 à l’aide du moteur de règles Azure CDN. (Pour en savoir plus, consultez [Override HTTP behavior using the Azure CDN from Verizon Premium rules engine](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) dans la documentation de Microsoft.) [Créez une nouvelle règle Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) avec les paramètres suivants :

   | Paramètre        | Valeur                                                                                                 |
   | ---------------- | ------------------------------------------------------------------------------------------------------ |
   | Nom/Description  | Domaine personnalisé Auth0                                                                             |
   | Type de requêtes | Sélectionnez l’option **Edge CName**, puis choisissez votre nom de domaine personnalisé dans la liste. |
7. Ajoutez les **Fonctionnalités** suivantes à votre règle Azure CDN :
   Nous vous recommandons de créer une autre règle Azure CDN afin de rejeter l’utilisation du CNAME **azureedge.net**.

   | Paramètre                               | Valeur                                                                                                                          |
   | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
   | Contourner le cache                     | **Activé**                                                                                                                      |
   | Modifier l’en-tête de requête du client | Sélectionnez **Remplacer**, saisissez **cname-api-key** comme nom, puis entrez la clé API CNAME fournie par Auth0 comme valeur. |
8. Une fois la règle Azure CDN approuvée, le statut passera de Pending XML à Active XML. À ce stade, Azure CDN publiera les règles et les certificats. Quand Azure aura terminé de traiter tous les changements, vous pourrez utiliser votre domaine personnalisé.
