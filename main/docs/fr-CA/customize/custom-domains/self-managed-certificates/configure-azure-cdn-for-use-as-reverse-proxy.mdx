title: Configurer Azure CDN comme proxy inverse
---

<Card title="La disponibilité varie selon le plan Auth0">
  La disponibilité de cette fonctionnalité dépend de votre plan Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Card>

Pour configurer Azure CDN comme proxy inverse, un plan Azure CDN Premium est requis.

1. [Configurez les domaines personnalisés avec des Self-Managed Certificates](/docs/fr-CA/customize/custom-domains/self-managed-certificates) si ce n’est pas déjà fait. Prenez en note les valeurs **Origin Domain Name** et **cname-api-key**, car vous en aurez besoin plus tard.
2. Connectez-vous à [Azure Portal](https://portal.azure.com/).
3. [Créez un nouveau profil Azure CDN et un endpoint](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint). Dans les paramètres de l’endpoint CDN, indiquez le **Origin Domain Name** provenant de l’Auth0 Dashboard.
4. [Ajoutez votre domaine personnalisé à votre endpoint Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup).
5. [Configurez HTTPS pour votre domaine personnalisé Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate). Ce processus exige que vous vérifiiez la propriété du domaine. Une fois cette étape terminée, le déploiement du certificat à tous les emplacements POP du CDN peut prendre jusqu’à 6 heures.
6. Configurez la communication du domaine personnalisé avec Auth0 à l’aide du moteur Rules d’Azure CDN. (Pour en savoir plus, consultez [Override HTTP behavior using the Azure CDN from Verizon Premium rules engine](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) dans la documentation Microsoft.) [Créez une nouvelle Rule Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) avec les paramètres suivants :

   | Paramètre        | Valeur                                                                                                   |
   | ---------------- | -------------------------------------------------------------------------------------------------------- |
   | Nom/Description  | Auth0 Custom Domain                                                                                      |
   | Type de requêtes | Sélectionnez l’option **Edge CName**, puis sélectionnez votre nom de domaine personnalisé dans la liste. |
7. Ajoutez les **Features** suivantes à votre Rule Azure CDN :
   Nous vous recommandons de créer une autre Rule Azure CDN pour rejeter l’utilisation du CNAME **azureedge.net**.

   | Paramètre                               | Valeur                                                                                                                         |
   | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
   | Contourner le cache                     | **Activé**                                                                                                                     |
   | Modifier l’en-tête de requête du client | Sélectionnez **Override**, saisissez **cname-api-key** comme nom, puis entrez la clé API CNAME fournie par Auth0 comme valeur. |
8. Une fois la Rule Azure CDN approuvée, le status passera de Pending XML à Active XML. À partir de ce moment, Azure CDN publiera les règles et les certificats. Quand Azure aura terminé le traitement de tous les changements, vous pourrez utiliser votre domaine personnalisé.
