<Card title="La disponibilité varie selon le forfait Auth0">
  Le mode de connexion que vous avez mis en œuvre, ainsi que votre forfait Auth0 ou votre entente personnalisée, déterminent si cette fonctionnalité est offerte. Pour en savoir plus, consultez [Tarification](https://auth0.com/pricing).
</Card>

Pour configurer un produit Akamai comme proxy inverse, procédez comme suit :

1. Vérifiez que vous êtes propriétaire de votre domaine personnalisé. Pour en savoir plus, consultez [Verify Ownership](/docs/fr-CA/customize/custom-domains/auth0-managed-certificates#verify-ownership).
2. Accédez à [Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) pour obtenir les paramètres de distribution, de l’en-tête d’origine personnalisé, du nom d’hôte d’origine et du comportement de cache par défaut. Utilisez ensuite ces renseignements pour mettre à jour les paramètres de votre domaine personnalisé Akamai afin qu’il pointe vers le domaine du serveur proxy inverse. Pour en savoir plus, consultez [Configurer le proxy inverse](/docs/fr-CA/customize/custom-domains/self-managed-certificates#configure-reverse-proxy).
3. Mettez à jour l’en-tête IP client du domaine personnalisé à l’aide du point de terminaison PATCH  `/api/v2/custom-domains/{id}` de la Management API, comme ci-dessous :

   ```json lines
   { 
   "custom_client_ip_header": "true-client-ip" 
   }
   ```

   Pour en savoir plus, consultez [Update custom domain configuration](https://auth0.com/docs/api/management/v2#!/Custom_Domains/patch_custom_domains_by_id).

Pour savoir comment configurer Akamai comme proxy inverse à l’aide des renseignements ci-dessus, consultez la documentation d’Akamai ou le soutien à la clientèle.
