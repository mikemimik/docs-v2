<Warning>
  Pour personnaliser les modèles de téléphone, vous devez [configurer un fournisseur de téléphonie externe](../configure-phone-messaging-providers).
</Warning>

<div id="customize-phone-templates">
  ## Personnaliser les Phone Templates
</div>

Pour personnaliser un modèle de téléphone :

1. Accédez à [**Dashboard &gt; Branding &gt; Phone Templates**](https://manage.auth0.com/#/phone).

2. Dans la liste déroulante **Template**, sélectionnez le modèle de téléphone que vous voulez mettre à jour.

3. Activez la bascule **Status** pour activer le modèle s’il ne l’est pas déjà.

4. Mettez à jour le numéro de téléphone dans le champ **From** et/ou le **modèle de message texte**.

5. Sélectionnez **Save** pour enregistrer vos modifications, **Try** pour les tester ou **Reset** pour annuler vos modifications.

<div id="from-phone-number">
  ### Numéro de téléphone **From**
</div>

<Warning>
  Si vous utilisez les [Twilio Messaging Services](https://www.twilio.com/docs/messaging/services) comme source pour vos SMS, vous ne pouvez pas définir un numéro de téléphone **From** personnalisé. C’est plutôt Twilio qui choisit un numéro dans le groupe d’expéditeurs de votre Messaging Service, selon votre configuration Twilio.
</Warning>

Le champ **From** définit le numéro de téléphone à partir duquel les messages texte sont envoyés. Le numéro de téléphone **From** par défaut est celui que vous avez défini lorsque vous avez configuré votre fournisseur de téléphonie.

Pour définir un numéro de téléphone **From** personnalisé pour un modèle de téléphone, entrez le numéro de téléphone dans ce champ.

<div id="text-message-template">
  ### Modèle de message texte
</div>

Le champ **Modèle de message texte** définit le contenu du message texte. Chaque modèle comprend un corps de message par défaut que vous pouvez modifier ou supprimer entièrement pour rédiger le vôtre.

Ce champ prend en charge la syntaxe Liquid pour le contenu généré dynamiquement. Pour en savoir plus, consultez [Syntaxe Liquid prise en charge pour les modèles de messages texte](./supported-liquid-syntax).

<div id="test-updated-templates">
  ## Tester les modèles mis à jour
</div>

Pour tester un modèle de téléphone, cliquez sur **Try**. Auth0 envoie le message téléphonique à une application par défaut portant le nom de votre tenant. Pour tester des modèles pour différentes applications, créez un utilisateur fictif pour suivre les flux de messages téléphoniques.

Vous pouvez déclencher manuellement des courriels de vérification pour des applications et des utilisateurs précis à l’aide du point de terminaison [Send an email address verification email](https://auth0.com/docs/api/management/v2#!/Jobs/post_verification_email) de la Management API.
