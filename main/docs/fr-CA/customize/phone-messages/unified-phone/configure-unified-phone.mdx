---

<Card title="Avant de commencer">
  Si vous utilisez [l’authentification Passwordless SMS](/docs/fr-CA/authenticate/passwordless/authentication-methods/use-sms-gateway-passwordless), les [notifications MFA par SMS et appel vocal](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa), ou Phone as ID, vous devez configurer un fournisseur téléphonique avec Unified Phone Experience.

  Si vous avez besoin de fournisseurs différents pour la MFA, Passwordless et/ou Phone as ID, vous devez configurer un [fournisseur téléphonique personnalisé](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) à l’aide d’Auth0 Actions.
</Card>

Unified Phone Experience centralise toutes les configurations de fournisseurs téléphoniques en un seul endroit dans votre tenant. Cette centralisation évite d’avoir à configurer un fournisseur téléphonique plusieurs fois pour différents facteurs d’authentification par téléphone.

Si vous voulez essayer les capacités de messagerie téléphonique, vous devez configurer un fournisseur téléphonique au niveau du tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 n’indique pas à l’utilisateur final si un message téléphonique a bien été envoyé à l’aide du fournisseur au niveau du tenant. Pour résoudre les problèmes, consultez les [journaux du tenant](/docs/fr-CA/deploy-monitor/logs).
</Callout>

<div id="configure-mfa">
  ## Configurer la MFA
</div>

Vous pouvez utiliser Unified Phone Experience avec les méthodes de livraison et les facteurs SMS, Voice et SMS + Voice. Pour en savoir plus sur l’<Tooltip tip="authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip> avec Auth0, consultez [Multi-Factor Authentication](/docs/fr-CA/secure/multi-factor-authentication).

1. Dans l’Auth0 Dashboard, accédez à [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Activez la bascule Phone Message Provider.

   <Frame>![Auth0 Dashboard > Branding > Phone Provider > Activer Phone Message Provider](/docs/images/cdy7uua7fh8z/1k93jvdxDq2FZLiRdvr5aR/d917431cd0265943c5118174a8673118/2025-02-24_16-43-14.png)</Frame>
3. Sélectionnez votre fournisseur de messages téléphoniques.

   1. Pour utiliser Twilio comme phone provider, suivez [Configure Twilio as a Phone Messaging Provider](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-twilio-as-a-phone-messaging-provider).
   2. Pour utiliser un fournisseur personnalisé, suivez [Configure a Custom Phone Provider](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) afin d’utiliser Auth0 Actions.
4. Sélectionnez votre ou vos méthodes de livraison.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Les facteurs MFA souhaités doivent correspondre à un sous-ensemble des méthodes de livraison que vous configurez à l’échelle du tenant.
   </Callout>
5. **Save** vos modifications.
6. Une fois terminé, accédez à [Sécurité &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
7. Sélectionnez **Phone Message**.
8. Choisissez les facteurs d’authentification.
9. **Save** vos modifications.

<div id="configure-passwordless">
  ## Configurer Passwordless
</div>

Vous pouvez utiliser Unified Phone Experience avec une connexion SMS <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip>. Pour en savoir plus sur l’authentification passwordless avec Auth0, consultez [Passwordless Connections](/docs/fr-CA/authenticate/passwordless).

1. Dans le tableau de bord Auth0, accédez à [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Activez la bascule Phone Message Provider.

   <Frame>![Auth0 Dashboard > Branding > Phone Provider > Enable Phone Message Provider](/docs/images/cdy7uua7fh8z/1k93jvdxDq2FZLiRdvr5aR/d917431cd0265943c5118174a8673118/2025-02-24_16-43-14.png)</Frame>
3. Sélectionnez votre fournisseur de messages téléphoniques.

   1. Pour utiliser Twilio comme fournisseur téléphonique, suivez [Configure Twilio as a Phone Messaging Provider](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-twilio-as-a-phone-messaging-provider).
   2. Pour utiliser un fournisseur personnalisé, suivez [Configure a Custom Phone Provider](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) pour utiliser Auth0 Actions.
4. Sélectionnez votre mode de livraison.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Si vous ne sélectionnez pas **Text** comme mode de livraison configuré, votre intégration à Passwordless ne fonctionnera pas.
   </Callout>
5. **Save** vos modifications.
6. Une fois la configuration terminée, accédez à [**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless).
7. **Save** vos modifications.
8. Si vous utilisez Twilio :

   1. Accédez à [**Applications &gt; Applications**](https://manage.auth0.com/#/applications).
   2. Sélectionnez la ou les applications que vous souhaitez utiliser avec votre connexion SMS Twilio pour Passwordless.
   3. Sous **Connections**, activez l’option SMS (Twilio).
   4. Retournez à [**Authentication &gt; Passwordless &gt; SMS**](https://manage.auth0.com/#/connections/passwordless).
   5. Sous l’onglet **Applications**, activez toutes les applications que vous souhaitez utiliser pour Passwordless SMS.
   6. **Save** vos modifications.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si votre cas d’utilisation exige l’expérience Legacy pour [MFA](/docs/fr-CA/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) et [Passwordless](/docs/fr-CA/authenticate/passwordless/authentication-methods/use-sms-gateway-passwordless), communiquez avec [Auth0 Support](/docs/fr-CA/troubleshoot/customer-support).
</Callout>

<div id="test-your-phone-provider">
  ### Testez votre fournisseur téléphonique
</div>

Testez le fournisseur du tenant avant de l’utiliser dans un environnement de production :

1. Accédez à [**Image de marque &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Sélectionnez **Envoyer un message de test** pour tester le fournisseur du tenant avant de l’utiliser dans un environnement de production.

<div id="configure-phone-templates">
  ## Configurer les modèles de messages téléphoniques
</div>

Le fournisseur téléphonique défini au niveau du tenant utilise la [syntaxe Liquid](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) pour injecter des variables de contexte dans le contenu de vos messages téléphoniques. Pour configurer vos modèles de messages téléphoniques, suivez les instructions dans [Personnaliser les modèles de messages téléphoniques](/docs/fr-CA/customize/phone-messages/phone-templates/customize).

<div id="disable-phone-provider">
  ## Désactiver le fournisseur téléphonique
</div>

Si vous configurez l’Unified Phone Experience, puis la désactivez, les intégrations existantes avec votre fournisseur téléphonique pourraient cesser de fonctionner, y compris votre facteur téléphonique MFA.
