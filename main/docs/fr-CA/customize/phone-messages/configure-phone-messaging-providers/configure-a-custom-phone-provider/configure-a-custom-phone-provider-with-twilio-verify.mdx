title: Configurer un fournisseur téléphonique personnalisé avec Twilio Verify
---

Vous pouvez configurer un fournisseur téléphonique personnalisé avec [Twilio Verify](https://www.twilio.com/docs/verify), qui contribue à prévenir la fraude et à protéger les comptes d’utilisateur. Cette opération définit Twilio Verify comme votre fournisseur téléphonique personnalisé.

<div id="prerequisites">
  ## Prérequis
</div>

Vous devez avoir un compte [Twilio](https://twilio.com/) avec une option valide de livraison par SMS et/ou par appel téléphonique.

<div id="set-up-twilio-verify-service">
  ## Configurer le service Twilio Verify
</div>

Vous pouvez créer un service Twilio Verify dans la [console Twilio](https://www.twilio.com/console/verify/services) ou à l’aide de l’[API REST de Verify](https://www.twilio.com/docs/verify/api/service#create-a-verification-service). Vous aurez besoin des éléments suivants pour connecter votre fournisseur téléphonique personnalisé Auth0 à Twilio Verify :

* TWILIO&#95;ACCOUNT&#95;SID
* TWILIO&#95;AUTH&#95;TOKEN
* TWILIO&#95;VERIFY&#95;SID

<div id="connect-custom-phone-provider-with-twilio-verify">
  ## Connecter un fournisseur téléphonique personnalisé à Twilio Verify
</div>

1. Dans le Auth0 Dashboard, accédez à [Branding &gt; Phone Provider](https://manage.auth0.com/dashboard/#/phone/templates/phone/provider). Vous serez redirigé vers la page **Phone Message Provider**.
2. Sélectionnez **Custom** comme **Phone Provider**.
3. Sous **Provider Configuration**, ajoutez l’exemple de code suivant pour permettre à l’API Twilio d’envoyer des messages téléphoniques au numéro de téléphone d’un utilisateur :

```javascript lines expandable
/**
* Handler à exécuter lors de l'envoi d'une notification téléphonique
* @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
* @param {CustomPhoneProviderAPI} api - Méthodes et utilitaires pour modifier le comportement de l'envoi d'une notification téléphonique.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {
 const { TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_VERIFY_SID } = event.secrets;

  const client = require('twilio')(TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN);

  // mapper la valeur voice d'auth0 à call
  const messageType = event.notification.delivery_method === 'voice' ? 'call' : 'sms';

  const { recipient, code } = event.notification;

  // ajouter ceci pour appliquer le correctif à « 1333444999  »
  // « 333 444 5555 »
  const sanitizedNumber = recipient.replace(/\s/g, '').trim();

  await client.verify.v2.services(TWILIO_VERIFY_SID)
    .verifications.create({
      to: sanitizedNumber,
      channel: messageType,
      customCode: code
    })

};
```

3. Cliquez sur l’icône **Key** pour ouvrir le menu **Secrets**. Ajoutez les valeurs suivantes depuis la configuration de votre service Twilio Verify :

* TWILIO&#95;ACCOUNT&#95;SID
* TWILIO&#95;AUTH&#95;TOKEN
* TWILIO&#95;VERIFY&#95;SID

4. Pour charger la bibliothèque d’assistance Twilio, cliquez sur l’icône **Dependency**. Ensuite, cliquez sur le bouton **Add Dependency**. Lorsque la fenêtre contextuelle **Add Dependency** s’affiche, saisissez les éléments suivants :

* **Nom :** Twilio
* **Version :** latest (devrait se remplir automatiquement)

Lorsque vous cliquez sur **Create**, Auth0 recherche la bibliothèque d’assistance Twilio et charge la version la plus récente.

5. Cliquez sur **Save**. Lorsque vous cliquez sur **Save,** l’Action s’enregistre et se déploie automatiquement. Pour tester la configuration du fournisseur téléphonique personnalisé avant de l’utiliser dans un environnement de production, cliquez sur **Send Test Message**.
