title: Configurer un fournisseur téléphonique personnalisé avec Terraform
---

Vous pouvez configurer un [fournisseur téléphonique personnalisé](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) avec le provider Auth0 pour Terraform. Le provider Auth0 pour Terraform sert à interagir avec l’[Auth0 Management API](https://auth0.com/docs/api/management/v2) afin de configurer un tenant Auth0. Pour en savoir plus, consultez la documentation d’[Auth0 Provider](https://registry.terraform.io/providers/auth0/auth0/latest/docs) de Terraform.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Si vous utilisez Terraform pour configurer un fournisseur téléphonique personnalisé, supprimez le fournisseur [de téléphonie](/docs/fr-CA/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) personnalisé existant de votre tenant Auth0.
  * Auth0 impose une limite d’une seule Action pour les Actions associées aux triggers suivants : custom-phone-provider et custom-email-provider.
</Callout>

Les étapes suivantes expliquent comment configurer un fournisseur téléphonique personnalisé à l’aide de Terraform :

1. Créez une [`auth0_action`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/phone_provider) :

```terraform lines
resource "auth0_action" "custom_phone_provider" {
  name    = "Custom Phone Provider"
  runtime = "node22"
  deploy  = true
  code    = <<-EOT
    /**
    * Gestionnaire à exécuter lors de l'envoi d'une notification téléphonique
    * @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
    * @param {CustomPhoneProviderAPI} api - Méthodes et utilitaires pour modifier le comportement de l'envoi d'une notification téléphonique.
    */
    exports.onExecuteCustomPhoneProvider = async (event, api) => {
      // Insérer le code ici
      return;
    };
  EOT
  supported_triggers {
    id      = "custom-phone-provider"
    version = "v1"
  }
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour lier une ressource Terraform `auth0_action` à un `auth0_trigger_action`, définissez `deploy = true` pour `auth0_action`. Pour en savoir plus, consultez la documentation Terraform sur [auth0&#95;action](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/action).
</Callout>

2. Liez le `auth0_action` au `auth0_trigger_action` custom-phone-provider :

```terraform lines
resource "auth0_trigger_action" "custom_phone_provider" {
  trigger = "custom-phone-provider"
  actions {
    id           = auth0_action.custom_phone_provider.id
    display_name = auth0_action.custom_phone_provider.name
  }
  depends_on = [
    auth0_action.custom_phone_provider
  ]
}
```

3. Configurez la ressource `auth0_phone_provider` pour utiliser l’`auth0_action` :

```terraform lines
resource "auth0_phone_provider" "custom_phone_provider" {
  depends_on = [auth0_trigger_actions.custom_phone_provider] # S'assurer que l'action est créée en premier avec `custom-phone-provider` comme supported_triggers
  name       = "custom"                         # Indique une mise en œuvre personnalisée
  disabled   = false                            # Désactiver le fournisseur de téléphone par défaut
  configuration {
    delivery_methods = ["text", "voice"]
  }
  credentials {}
},
```

Pour en savoir plus, consultez la documentation Terraform de [auth0&#95;phone&#95;provider](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/phone_provider).

<div id="troubleshoot">
  ## Dépannage
</div>

**Scénario : le fournisseur téléphonique personnalisé configuré avec Terraform ne prend pas effet, et l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> affiche une Action différente.**

Il se peut que vous ayez plusieurs Actions de fournisseur personnalisé avec `deploy: true`. Auth0 limite à une seule Action les Actions associées aux triggers suivants : `custom-phone-provider` et `custom-email-provider`.

* Si vous êtes dans cette situation, nous vous recommandons de [dresser la liste de vos Actions](https://auth0.com/docs/api/management/v2/actions/get-actions) afin d’identifier et de supprimer tout doublon.
* Si vous vous retrouvez dans cette situation après avoir géré vos fournisseurs personnalisés de plusieurs façons (Terraform, Auth0 Dashboard, <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> et/ou les Auth0 SDKs), il est conseillé de réinitialiser la configuration du fournisseur personnalisé. Nous vous recommandons d’éviter d’utiliser plusieurs méthodes pour gérer votre fournisseur personnalisé.

Pour réinitialiser le fournisseur téléphonique personnalisé, accédez à [Auth0 Dashboard &gt; Branding &gt; Phone Provider](https://manage.auth0.com/#/phone/templates/phone/provider). Ensuite, sélectionnez le bouton **Reset** sous Provider Configuration pour rétablir les paramètres par défaut de votre fournisseur personnalisé.
