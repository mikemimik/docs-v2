<div id="set-language-option">
  ## Définir l’option de langue
</div>

Pour utiliser les exemples suivants, vous devez d’abord inclure Lock dans votre page :

`<script src="https://cdn.auth0.com/js/lock/11.14/lock.min.js"></script>`

Ensuite, définissez votre objet `options` et incluez l’option `language`. L’option `language` doit être une chaîne de caractères correspondant au nom du fichier dans le répertoire `i18n` [de Lock](https://github.com/auth0/lock/tree/master/src/i18n). Puis, instanciez Lock.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample1 = `// Sélectionnez une langue prise en charge
var options = {
  language: 'es'
};

// Initialisation d'Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample1} language="javascript" />

Les données de traduction de Lock proviennent des fichiers de langue, qui contiennent des paires clé-valeur correspondant à différentes traductions. Pour certaines langues, il se peut que certaines valeurs soient manquantes; dans ce cas, l’avertissement suivant s’affichera : `language does not have property <missing prop>`. Nous vous invitons à soumettre une [pull request](https://github.com/auth0/lock/tree/master/src/i18n) pour ajouter ces valeurs manquantes. Vous pouvez aussi définir les valeurs manquantes dans les `options` de Lock (voir ci-dessous).

<div id="replace-dictionary-terms">
  ## Remplacer les termes du dictionnaire
</div>

Vous pouvez aussi personnaliser certaines entrées précises du dictionnaire à l’aide de l’option `languageDictionary`. C’est utile si vous utilisez l’une des langues prises en charge, mais souhaitez modifier le libellé de quelques éléments. Par exemple, vous pourriez reformuler le `title` ou modifier la façon dont d’autres libellés s’affichent à l’utilisateur, tout en laissant intact le reste du texte du widget.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample2 = `// Personnaliser certains attributs de languageDictionary
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Me connecter"
  },
};

// Initialisation de notre Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample2} language="javascript" />

Pour obtenir un exemple des noms de propriétés `languageDictionary` disponibles et de la façon de structurer un fichier `language`, consultez le [fichier de dictionnaire anglais pour Lock](https://github.com/auth0/lock/blob/master/src/i18n/en.js).

L’option `languageBaseUrl`, qui prend une valeur de type chaîne (une URL), remplace l’URL source des traductions fournies par Auth0. Par défaut, elle utilise l’URL du CDN d’Auth0 `https://cdn.auth0.com`, puisque c’est à cet endroit que sont stockées les traductions fournies. En indiquant une autre valeur, vous pouvez utiliser votre propre source pour les traductions, selon les besoins de vos applications. Votre source de traductions doit être un fichier JavaScript.
