[Auth0 Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) offre une fonctionnalité essentielle d’un <Tooltip tip="Authorization Server : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=authorization+server">serveur d’autorisation</Tooltip> : le flux de connexion. Lorsqu’un utilisateur doit prouver son identité pour accéder à votre application, vous pouvez le rediriger vers <Tooltip tip="Authorization Server : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> et laisser Auth0 gérer le processus d’authentification.

Universal Login est la principale solution de connexion hébergée d’Auth0. [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) propose des outils de personnalisation faciles à utiliser et offre une expérience plus simple et plus rapide aux utilisateurs finaux. Auth0 prend aussi en charge [Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience), une expérience de connexion hébergée qui utilise des contrôles JavaScript pour la personnalisation.

Pour comparer les expériences de connexion, consultez [Universal Login vs. Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).
