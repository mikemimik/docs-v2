<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [SendGrid](https://sendgrid.com). Si vous avez un abonnement Microsoft Azure, vous pouvez obtenir un compte gratuit dans le Marketplace Azure.

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes à partir d’[adresses IP Auth0 précises](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

<div id="configure-sendgrid">
  ## Configurer SendGrid
</div>

1. Connectez-vous à votre compte [SendGrid](https://sendgrid.com).
2. Accédez à **Settings &gt; API Keys** dans SendGrid, puis cliquez sur **Create API Key**.
3. Saisissez un nom pour votre clé et sélectionnez **Restricted Access**.
4. Sous **Access Details**, cliquez pour afficher **Mail Send**.
5. Réglez la permission **Mail Send** sur **Full Access**.
6. Cliquez sur **Save** et copiez la valeur de la clé.
7. Accédez à [Dashboard &gt; Branding &gt; fournisseur de courriel](https://manage.auth0.com/#/branding/email_provider) dans Auth0.
8. Activez la bascule **Use my own fournisseur de courriel**.
9. Cliquez sur le logo **SendGrid**.
10. Saisissez une adresse courriel **From**, collez la **API Key** de SendGrid que vous avez copiée, puis cliquez sur **Save**.

Si vous utilisez l’[authentification sans mot de passe par courriel](/docs/fr-CA/authenticate/passwordless#email), la valeur de l’adresse courriel **From** de votre connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip> doit correspondre à la valeur de l’adresse courriel **From** utilisée pour SendGrid.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour voir les codes d’erreur.

Une fois la configuration effectuée, la page [Activité des courriels](https://sendgrid.com/logs/index) de SendGrid affiche des renseignements sur la livraison de tous les courriels envoyés à vos utilisateurs.
