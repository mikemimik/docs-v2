<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Microsoft 365](https://www.microsoft.com/en-us/microsoft-365) avec un abonnement actif.

* Votre abonnement Microsoft 365 doit inclure une licence autorisant l’envoi de courriels par l’entremise d’Exchange Online. Pour utiliser Microsoft Graph API avec Exchange Online, vous devez disposer au minimum d’un abonnement payant au forfait Microsoft Business Basic.

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes provenant de [certaines adresses IP d’Auth0](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

<div id="configure-microsoft-365-exchange-online">
  ## Configurer Microsoft 365 Exchange Online
</div>

1. Ouvrez une session sur votre compte [Microsoft Azure](https://azure.microsoft.com/) avec vos identifiants Microsoft 365.
2. Enregistrez une ressource [Microsoft Entra ID Application](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application) si vous n’en avez pas déjà une.
3. Accédez à votre ressource Active Directory Application. Sélectionnez **Services &gt; App registrations**, puis l’application que vous utiliserez pour envoyer des courriels.
4. Configurez les permissions d’application requises en accédant à **Manage &gt; API permissions**.
5. Ajoutez la [permission](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#api-permissions) `Mail.Send` en sélectionnant **Add a permission &gt; Microsoft Graph &gt; Application permissions**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Avant de configurer **Application permissions**, vous devez obtenir le consentement d’un administrateur.
   </Callout>

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Par défaut, `Mail.Send` accorde la permission d’envoyer des courriels au nom de n’importe quel utilisateur de votre tenant. Pour restreindre l’accès à une boîte aux lettres précise, utilisez [RBAC for Applications in Exchange Online](https://learn.microsoft.com/en-us/exchange/permissions-exo/application-rbac) afin de limiter la permission à un principal de service précis.
   </Callout>
6. Récupérez les configurations de fournisseur requises :

   1. Accédez à **Application Overview** pour récupérer le [Client ID et le Tenant ID](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#application-id-client-id) de votre application.
   2. Accédez à **Manage &gt; Certificates &amp; secrets** et récupérez la **Value** pour créer un [secret](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#certificates--secrets) d’application.
   3. Récupérez le **User Email** de l’organisation qui envoie les courriels au moyen de l’application. Accédez à **Users &gt; Active users**, sélectionnez votre utilisateur autorisé, puis récupérez son **User Email** dans ses propriétés.
7. Accédez à Auth0 [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider).
8. Activez la bascule **Use my own email provider**.
9. Cliquez sur le logo **Microsoft 365**.
10. Saisissez l’adresse courriel **From**. Ensuite, entrez le **Tenant ID**, le **Client ID** et le **Client Secret** de l’étape 6, puis cliquez sur **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez aucun courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour vérifier les codes d’erreur.

Une fois la configuration terminée, la page [Utilisation du courriel](https://admin.microsoft.com/#/reportsUsage/EmailActivity) de Microsoft Exchange affiche des informations sur la livraison de tous les courriels envoyés à vos utilisateurs.
