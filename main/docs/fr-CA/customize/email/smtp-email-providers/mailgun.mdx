<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Mailgun](https://mailgun.com).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes à partir d’[adresses IP Auth0 précises](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

<div id="configure-mailgun">
  ## Configurer Mailgun
</div>

1. Connectez-vous à votre compte [Mailgun](https://mailgun.com).
2. Accédez à **Sending &gt; Domains** dans Mailgun, puis ajoutez votre domaine personnalisé. Mailgun permet d’envoyer des courriels uniquement à partir de domaines vérifiés.
3. Accédez à **Settings &gt; API Keys** dans Mailgun. Votre clé API a été créée lors de l’ouverture de votre compte; copiez-la à partir de **Private API Key**.
4. Accédez à [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider) dans Auth0.
5. Activez la bascule **Use my own email provider**.
6. Cliquez sur le logo **Mailgun**.
7. Indiquez une adresse courriel **From**, saisissez le **Domain** que vous avez ajouté précédemment à Mailgun, saisissez l’**API Key** Mailgun que vous avez copiée précédemment, puis sélectionnez votre **Region**.
8. Cliquez sur **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, vérifiez vos [logs Auth0](https://manage.auth0.com/#/logs) pour repérer les codes d’erreur.

Une fois la configuration effectuée, le [Dashboard Mailgun](https://app.mailgun.com/app/dashboard) affiche des renseignements sur la livraison de tous les courriels envoyés à vos utilisateurs.
