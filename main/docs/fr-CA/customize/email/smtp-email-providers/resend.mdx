<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Resend](https://resend.com).

* Vous devez vérifier au moins un domaine avec Resend avant de pouvoir envoyer des courriels. Pour en savoir plus, consultez la [documentation de Resend sur la vérification de domaine](https://resend.com/docs/dashboard/domains/introduction).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes à partir d’[adresses IP spécifiques d’Auth0](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

<div id="configure-resend">
  ## Configurer Resend
</div>

1. Connectez-vous à votre compte [Resend](https://resend.com).
2. Accédez à **clés d’API** et cliquez sur **Create clé d’API**.
3. Entrez un nom pour votre clé, sélectionnez le niveau d’autorisation et le domaine appropriés, puis cliquez sur **Add**.

<Frame>
  <img src="/docs/images/customize/emails/providers/resend-create-api-key.png" />
</Frame>

4. Copiez la valeur de la clé d’API. Vous ne pourrez plus la voir par la suite.
5. Accédez à Auth0 [Dashboard &gt; Branding &gt; fournisseur de courriel](https://manage.auth0.com/#/branding/email_provider).
6. Activez la bascule **Use my own fournisseur de courriel**.
7. Sélectionnez le logo **Resend**.
8. Entrez une adresse courriel **From**, collez l’**clé d’API** de Resend que vous avez copiée, puis cliquez sur **Save**.

L’adresse courriel **From** doit utiliser un domaine que vous avez vérifié avec Resend. Si vous utilisez l’[authentification sans mot de passe par courriel](/docs/fr-CA/authenticate/passwordless#email), la valeur de l’adresse courriel **From** pour votre connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip> doit correspondre à celle que vous utilisez pour Resend.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez vos [logs Auth0](https://manage.auth0.com/#/logs) pour vérifier les codes d’erreur.

Une fois la configuration effectuée, la page [Emails](https://resend.com/emails) de Resend affiche des données de livraison pour tous les courriels envoyés à vos utilisateurs.
