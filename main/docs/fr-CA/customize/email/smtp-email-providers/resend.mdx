<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Resend](https://resend.com).

* Vous devez vérifier au moins un domaine avec Resend avant de pouvoir envoyer des courriels. Pour en savoir plus, consultez la [documentation de Resend sur la vérification de domaine](https://resend.com/docs/dashboard/domains/introduction).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes en provenance d’[adresses IP Auth0](/docs/fr-CA/secure/security-guidance/data-security/allowlist) précises.

<div id="configure-resend">
  ## Configurer Resend
</div>

1. Connectez-vous à votre compte [Resend](https://resend.com).
2. Accédez à **API Keys** et cliquez sur **Create API Key**.
3. Saisissez un nom pour votre clé, sélectionnez le niveau d’autorisation et le domaine appropriés, puis cliquez sur **Add**.

<Frame>
  <img src="/docs/images/customize/emails/providers/resend-create-api-key.png" />
</Frame>

4. Copiez la valeur de la clé API. Vous ne pourrez plus la voir par la suite.
5. Accédez à Auth0 [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider).
6. Activez la bascule **Use my own email provider**.
7. Sélectionnez le logo **Resend**.
8. Saisissez une adresse courriel **From**, collez la **API Key** Resend que vous avez copiée, puis cliquez sur **Save**.

L’adresse courriel **From** doit utiliser un domaine que vous avez vérifié avec Resend. Si vous utilisez l’[authentification sans mot de passe par courriel](/docs/fr-CA/authenticate/passwordless#email), la valeur de l’adresse courriel **From** pour votre connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip> doit correspondre à celle que vous utilisez pour Resend.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez aucun courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour vérifier les codes d’erreur.

Une fois la configuration effectuée, la page [Courriels](https://resend.com/emails) de Resend affiche des informations sur la livraison de tous les courriels envoyés à vos utilisateurs.
