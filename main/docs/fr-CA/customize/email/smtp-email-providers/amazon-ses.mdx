<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Amazon AWS](http://aws.amazon.com/ses/).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes en provenance d’[adresses IP précises d’Auth0](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

Vous pouvez utiliser des identifiants API ou SMTP. Pour en savoir plus, consultez le guide Amazon [Using Credentials With Amazon SES](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/using-credentials.html).

<div id="configure-amazon-ses-with-api-credentials">
  ## Configurer Amazon SES avec des identifiants API
</div>

1. Connectez-vous à votre compte [Amazon AWS](http://aws.amazon.com/ses/).
2. [Vérifiez votre domaine](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/verify-domains.html).
3. [Vérifiez les adresses courriel](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/verify-email-addresses.html)
4. [Demandez l’accès à l’environnement de production](https://docs.aws.amazon.com/ses/latest/DeveloperGuide/request-production-access.html?icmpid=docs_ses_console).
5. [Obtenez vos clés d’accès AWS](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/get-aws-keys.html). Copiez-les; vous devrez entrer ces valeurs dans Auth0.
6. [Attachez une policy avec les permissions appropriées](http://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_manage.html). Attachez une policy avec les permissions `ses:SendRawEmail` et `ses:SendEmail`, comme dans cet exemple :

   ```json lines
   {
       "Version": "2012-10-17",
       "Statement": [
           {
               "Effect": "Allow",
               "Action": [
                   "ses:SendRawEmail",
                   "ses:SendEmail"
               ],
               "Resource": "*"
           }
       ]
   }
   ```
7. Accédez à Auth0 [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider).
8. Activez la bascule **Use my own email provider**.
9. Cliquez sur le logo **Amazon Web Services**.
10. Indiquez une adresse courriel **From**, saisissez votre **Access Key ID** AWS et votre **Secret Access Key**, puis sélectionnez votre **Region**.
11. Cliquez sur **Save**.

<div id="configure-amazon-ses-with-smtp-credentials">
  ## Configurer Amazon SES avec des identifiants SMTP
</div>

1. Connectez-vous à votre compte [Amazon AWS](https://aws.amazon.com/ses/).
2. [Vérifier votre domaine](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/verify-domains.html).
3. [demander l’accès à l’environnement de production](https://docs.aws.amazon.com/ses/latest/DeveloperGuide/request-production-access.html?icmpid=docs_ses_console).
4. [Obtenez vos identifiants SMTP](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/smtp-credentials.html). Copiez les identifiants de sécurité; vous devrez saisir ces valeurs dans Auth0.
5. Accédez à [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/emails/provider).
6. Activez la bascule **Use my own email provider**.
7. Cliquez sur le logo **SMTP**.
8. Saisissez une adresse courriel **From**, puis entrez les valeurs **Host**, **Port**, **Username** et **Password** de votre serveur SMTP, puis cliquez sur **Save**. Vous pouvez utiliser `email-smtp.us-east-1.amazonaws.com` (en remplaçant `us-east-1` par la région appropriée) pour **Host** et `587` pour **Port**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour voir les codes d’erreur.

Une fois la configuration effectuée, la [console Amazon SES](https://console.aws.amazon.com/ses) affiche des renseignements sur la livraison de tous les courriels envoyés à vos utilisateurs.
