<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Mandrill](https://www.mandrill.com/signup/).

* Vous devez configurer votre fournisseur afin d’autoriser les connexions entrantes en provenance de certaines [adresses IP d’Auth0](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

<div id="configure-mandrill">
  ## Configurer Mandrill
</div>

1. Connectez-vous à [Mandrill](https://www.mandrill.com/signup/).
2. Accédez à [Settings](https://mandrillapp.com/settings) dans Mandrill, puis cliquez sur **Add API key**. Copiez la valeur de cette clé.
3. Accédez à [Dashboard &gt; Branding &gt; fournisseur de courriel](https://manage.auth0.com/#/branding/email_provider) dans Auth0.
4. Activez la bascule **Use my own fournisseur de courriel**.
5. Cliquez sur le logo **Mandrill**.
6. Saisissez une adresse courriel **From** et entrez l’**API Key** Mandrill que vous avez copiée précédemment.
7. Cliquez sur **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez aucun courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour y repérer tout code d’erreur.

Une fois la configuration terminée, la page [Outbound Activity](https://mandrillapp.com/activity) de Mandrill affiche l’état de livraison de tous les courriels envoyés à vos utilisateurs.
