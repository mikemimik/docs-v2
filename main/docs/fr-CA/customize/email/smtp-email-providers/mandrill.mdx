<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Mandrill](https://www.mandrill.com/signup/).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes en provenance d’[adresses IP spécifiques d’Auth0](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

<div id="configure-mandrill">
  ## Configurer Mandrill
</div>

1. Connectez-vous à [Mandrill](https://www.mandrill.com/signup/).
2. Accédez à [Settings](https://mandrillapp.com/settings) dans Mandrill, puis cliquez sur **Add API key**. Copiez la valeur de cette clé.
3. Accédez à [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider) dans Auth0.
4. Activez la bascule **Use my own email provider**.
5. Cliquez sur le logo **Mandrill**.
6. Indiquez une adresse courriel dans le champ **From** et entrez l’**API Key** Mandrill que vous avez copiée précédemment.
7. Cliquez sur **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez aucun courriel après quelques minutes, consultez les [journaux Auth0](https://manage.auth0.com/#/logs) pour vérifier les codes d’erreur.

Une fois la configuration terminée, la page [Outbound Activity](https://mandrillapp.com/activity) de Mandrill affiche des renseignements sur la livraison de tous les courriels envoyés à vos utilisateurs.
