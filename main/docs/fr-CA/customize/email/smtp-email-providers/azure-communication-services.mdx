<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Microsoft Azure](https://azure.microsoft.com/en-us).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes provenant d’[adresses IP Auth0 précises](/docs/fr-CA/secure/security-guidance/data-security/allowlist).

<div id="configure-azure-communication-services">
  ## Configurer Azure Communication Services
</div>

D’abord, créez une ressource dans Microsoft Azure :

1. Ouvrez une session dans votre compte [Microsoft Azure](https://azure.microsoft.com/en-us).
2. Entrez « Communication Services » dans la barre de recherche.
3. Créez une ressource [Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource) si vous n’en avez pas déjà une.
4. Sélectionnez **Create**.
5. Dans les paramètres **Tools**, sélectionnez **Keys**. Récupérez votre [Connection string Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource).
6. Sous **Primary Key**, copiez la valeur **Connection string**.
   Une Connection string valide respecte le format `endpoint=COMMUNICATION_SERVICES_URL;accesskey=ACCESS_KEY`.

Ensuite, configurez votre fournisseur de courriel dans Auth0 :

1. Accédez à [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider).

2. Sélectionnez **Use my own email provider**.

3. Sélectionnez **Azure Communication Services**.

4. Entrez une adresse courriel **From**.

   <Warning>
     Azure Communication Services prend uniquement en charge une adresse courriel brute (`user@example.com`) dans le champ **From**.
   </Warning>

5. Ajoutez la **Connection string** d’Azure Communication Services.

6. Sélectionnez **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez aucun courriel après quelques minutes, consultez vos [logs Auth0](https://manage.auth0.com/#/logs) pour vérifier les codes d’erreur.

Une fois la configuration effectuée, la page [Metrics Monitor](https://learn.microsoft.com/en-us/azure/communication-services/concepts/metrics#chat-api-request-metric-operations) de Communication Services affiche des renseignements sur la livraison de tous les courriels envoyés à vos utilisateurs.
