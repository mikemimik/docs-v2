<div id="configure-an-email-provider-action-using-the-auth0-dashboard">
  ## Configurer une Action de fournisseur de courriel à l’aide d’Auth0 Dashboard
</div>

Pour utiliser Auth0 Dashboard afin de configurer votre tenant avec une Action de fournisseur de courriel personnalisé :

1. Accédez à [Auth0 Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/templates/provider).

2. Activez la bascule **Use my own email provider**.

3. Dans la section **Email Provider**, sélectionnez **Custom Provider**.

4. Dans le champ **From**, saisissez l’adresse courriel par défaut à partir de laquelle les courriels seront envoyés.

5. Dans l’éditeur de code, écrivez le code de votre Action pour envoyer des messages à votre fournisseur de courriel. Consultez la documentation de votre fournisseur pour connaître les détails de son API ou de sa connexion SMTP.

   Dans le menu de gauche de l’éditeur de code, vous pouvez cliquer sur l’icône de clé pour ajouter des secrets (par exemple, pour vous authentifier auprès d’une API), puis sur la boîte pour ajouter des dépendances.

6. Lorsque vous avez terminé d’écrire l’Action, cliquez sur **Save** pour la déployer.

Pour tester votre configuration, cliquez sur **Send Test Email**.

Comme pour les autres Actions, vous pouvez utiliser la [Management API](https://auth0.com/docs/api/management/v2/actions/get-actions) pour gérer l’Action et ses [versions](/docs/fr-CA/customize/actions/manage-versions). Toutes les Actions sont assujetties aux mêmes [limitations](/docs/fr-CA/customize/actions/limitations).

<div id="configure-an-email-provider-action-using-terraform">
  ## Configurer une Action de fournisseur de courriel avec Terraform
</div>

Le [Terraform Auth0 provider](https://registry.terraform.io/providers/auth0/auth0/latest/docs) utilise l’[Auth0 Management API](https://auth0.com/docs/api/management/v2) pour effectuer des opérations sur la plateforme Auth0.

Vous pouvez utiliser le Terraform Auth0 provider pour créer une Action de fournisseur de courriel personnalisé et configurer votre tenant afin de l’utiliser.

<div id="1-unbind-or-delete-conflicting-actions">
  ### 1. Dissocier ou supprimer les Actions en conflit
</div>

Vous ne pouvez lier qu’une seule Action au déclencheur `custom-email-provider`.

Si votre tenant est déjà configuré avec une Action de fournisseur de courriel personnalisé, réinitialisez-la avant de créer une nouvelle Action de fournisseur de courriel avec Terraform :

1. Accédez à [Auth0 Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/templates/provider).
2. Dans la section **Email Provider**, cliquez sur **Custom Provider**.
3. Sous l’éditeur de code des Actions, cliquez sur **Reset**.

Vous pouvez vérifier si d’autres Actions déployées sont liées au déclencheur `custom-email-provider` à l’aide de la Management API :

1. [Listez vos Actions](https://auth0.com/docs/api/management/v2/actions/get-actions) pour repérer les doublons.
2. [Mettez à jour les liaisons de leur déclencheur](https://auth0.com/docs/api/management/v2/actions/patch-bindings) ou [supprimez les Actions](https://auth0.com/docs/api/management/v2/actions/delete-action).

<div id="2-create-a-new-email-provider-action">
  ### 2. Créez une nouvelle Action de fournisseur de courriel
</div>

Utilisez la ressource [Terraform `auth0_action`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/action) pour créer une Action qui prend en charge le déclencheur `custom-email-provider`. Définissez `deploy = true` pour créer immédiatement une nouvelle version de l’Action.

```terraform lines
resource "auth0_action" "custom_email_provider" {
  name    = "Custom Email Provider"
  runtime = "node20"
  deploy  = true
  code    = <<-EOT
    /**
    * Gestionnaire à exécuter lors de l'envoi d'une notification par courriel
    * @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
    * @param {CustomEmailProviderAPI} api - Méthodes et utilitaires pour modifier le comportement d'envoi d'une notification par courriel.
    */
    exports.onExecuteCustomEmailProvider = async (event, api) => {
      // Insérez votre code ici
      return;
    };
  EOT
  supported_triggers {
    id      = "custom-email-provider"
    version = "v1"
  }
}
```

Dans le squelette de la fonction, écrivez votre code Action pour acheminer les messages à votre fournisseur de courriel. Consultez la documentation de votre fournisseur pour obtenir les détails de son API ou de son serveur SMTP.

<div id="3-bind-the-action-to-the-email-provider-trigger">
  ### 3. Associer l’Action au déclencheur du fournisseur de courriel
</div>

Utilisez la [ressource Terraform `auth0_trigger_action`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/trigger_action) pour associer l’Action au déclencheur `custom-email-provider` :

```terraform lines
resource "auth0_trigger_action" "custom_email_provider" {
  trigger = "custom-email-provider"
  actions {
    id           = auth0_action.custom_email_provider.id
    display_name = auth0_action.custom_email_provider.name
  }
   depends_on = [
    auth0_action.custom_email_provider
  ]
 }
```

<div id="4-configure-your-tenants-email-provider-with-the-action">
  ### 4. Configurez l’email provider de votre tenant avec l’Action
</div>

Utilisez la [ressource Terraform `auth0_email_provider`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/email_provider) pour configurer votre tenant afin d’utiliser l’Action de fournisseur de courriel :

```terraform lines
resource "auth0_email_provider" "custom_email_provider" {
  name                 = "custom"
  enabled              = true
  default_from_address = "accounts@example.com"
  credentials {}
  depends_on = [
    auth0_trigger_actions.custom_email_provider
  ]
```

Si vous créez une nouvelle Action de fournisseur de courriel avec Terraform et qu’elle ne s’applique pas, il se peut qu’une Action déployée soit déjà associée au déclencheur `custom-email-provider`. Pour résoudre ce problème, suivez la première étape de cet article pour [dissocier ou supprimer les Actions en conflit](#1-unbind-or-delete-conflicting-actions).

<div id="example-email-provider-action">
  ## Exemple d’Action de fournisseur de courriel
</div>

Voici un exemple d’Action pour le déclencheur `custom-email-provider`.

Dans cet exemple de code, la fonction `onExecuteCustomEmailProvider` reçoit deux arguments de l’[objet d’événement `custom-email-provider`](/docs/fr-CA/customize/email/configure-a-custom-email-provider/action-triggers-custom-email-provider-event-object) : `event`, qui contient des renseignements sur l’utilisateur et le contexte de la notification, et `api`, qui fournit des méthodes utilitaires pour personnaliser le comportement lors de l’envoi des notifications.

```javascript lines expandable
/**
 * Handler à exécuter lors de l'envoi d'une notification par courriel.
 * @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
 * @param {CustomEmailProviderAPI} api - Méthodes et utilitaires pour modifier le comportement lors de l'envoi d'une notification par courriel.
 */
exports.onExecuteCustomEmailProvider = async (event, api) => {
  // Définir le payload du courriel
  const emailPayload = {
    from: {
      name: "Test Sender",
      email: "sender@example.com"
    },
    to: [{ email: event.user.email }],
    subject: event.notification.message_type,
    html: event.notification.html,
    text: event.notification.text,
  };
  try {
    // Effectuer l'appel d'API pour envoyer le courriel
    const response = await fetch('https://api.example.com/send-email', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${event.secrets.api_key}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(emailPayload),
    });
    if (response.ok) {
      console.log('Email sent successfully');
    } else if (response.status >= 500) {
      api.notification.retry(
        `Internal Server Error received from Messaging Proxy. Status code: ${response.status}.`
      );
      return;
    }
  } catch (error) {
    console.error(`Error sending email: ${error.message}`);
    api.notification.drop(`An unexpected error occurred. Error: ${error.message}`);
  }
  return;
};
```

Pour en savoir plus sur les Actions, consultez [Comprendre le fonctionnement des Actions Auth0](/docs/fr-CA/customize/actions/actions-overview).
