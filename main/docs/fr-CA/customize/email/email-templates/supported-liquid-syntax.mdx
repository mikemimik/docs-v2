Pour un aperçu de la syntaxe Liquid, consultez l’[introduction à LiquidJS](https://liquidjs.com/tutorials/intro-to-liquid.html). Vous y trouverez un bref résumé des deux types de balisage de Liquid :

* **Sorties**, qui génèrent du HTML.

  La syntaxe des sorties consiste en une variable ou une valeur, ainsi qu’une liste facultative de filtres, entourées de `{{` et `}}`. Par exemple, `{{ user.name }}`.

* **Balises**, qui contrôlent le rendu du modèle et appliquent une logique.

  La syntaxe des balises consiste en un nom de balise et des arguments facultatifs entourés de `{%` et `%}`. Par exemple, `{% if user_metadata.lang == 'it' %} Ciao! {% endif %}`.

Cette page répertorie les balises, les filtres et les variables pris en charge dans les modèles de courriel.

<div id="tags">
  ## Balises
</div>

Tous les modèles de courriel prennent en charge les balises Liquid suivantes :

| Catégorie        | Balises                                 |
| ---------------- | --------------------------------------- |
| Itération        | `for`, `cycle`, `tablerow`              |
| Contrôle de flux | `if`, `unless`, `elsif`, `else`, `case` |
| Variable         | `assign`, `increment`, `decrement`      |
| Fichier          | `include`, `layout`                     |
| Langage          | `raw`, `comment`                        |

Vous pouvez consulter des descriptions et des exemples de balises Liquid sur la [page des balises de LiquidJS](https://liquidjs.com/tags/overview.html).

<div id="debug-tag">
  ### balise de débogage
</div>

Pour faciliter le débogage et le développement, notre balise personnalisée `{% debug %}` affiche un résumé des variables du modèle mises à la disposition de votre modèle au moment de son rendu.

<div id="filters">
  ## Filtres
</div>

Tous les modèles de courriel prennent en charge les filtres Liquid suivants :

| Catégorie | Filtres                                                                                                                                                                                            |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Math      | `plus`, `minus`, `modulo`, `times`, `floor`, `ceil`, `round`, `divided_by`, `abs`, `at_least`, `at_most`                                                                                           |
| Chaîne    | `append`, `prepend`, `capitalize`, `upcase`, `downcase`, `strip`, `lstrip`, `rstrip`, `strip_newlines`, `split`, `replace`, `replace_first`, `remove`, `remove_first`, `truncate`, `truncatewords` |
| HTML/URI  | `escape`, `escape_once`, `url_encode`, `url_decode`, `strip_html`, `newline_to_br`                                                                                                                 |
| Tableau   | `slice`, `map`, `sort`, `sort_natural`, `first`, `last`, `join`, `reverse`, `size`                                                                                                                 |
| Date      | `date`                                                                                                                                                                                             |
| Divers    | `default`                                                                                                                                                                                          |

Vous trouverez des descriptions et des exemples des filtres Liquid sur la [page des filtres de LiquidJS](https://liquidjs.com/filters/overview.html).

<div id="general-variables">
  ## Variables générales
</div>

Les variables suivantes sont disponibles dans les champs **From Address**, **Subject** et **Message** de chaque modèle (sauf indication contraire).

Pour utiliser des propriétés non fiables dans les modèles de courriel, échappez-les à l’aide du filtre `escape` (par exemple, `{{ user.name | escape }}`).

* L’objet `application`, qui donne accès aux propriétés client standard comme `application.name` et `application.client_metadata`

* `request_language`, pour la sélection de la langue de l’utilisateur

* `connection.name`

<Warning>
  - Dans le modèle de courriel d’inscription à l’AMF, la variable `connection.name` n’est pas disponible.
</Warning>

* Renseignements liés au tenant définis dans [Tenant Settings](https://manage.auth0.com/#/tenant) :
  * `tenant`, qui correspond au nom brut du tenant
  * `friendly_name`
  * `support_email`
  * `support_url`

* Les variables `user` suivantes :
  * `user.email`
  * `user.email_verified`
  * `user.picture` (**non fiable**)
  * `user.nickname` (**non fiable**)
  * `user.given_name` (**non fiable**)
  * `user.family_name` (**non fiable**)
  * `user.name` (**non fiable**)
  * `user.app_metadata`, attributs utilisateur pouvant avoir une incidence sur les fonctionnalités de base d’un utilisateur, par exemple le fonctionnement d’une application ou ce à quoi l’utilisateur peut accéder, y compris son plan de soutien, ses rôles de sécurité et ses groupes de contrôle d’accès
  * `user.user_metadata` (**non fiable** s’il contient des renseignements fournis par l’utilisateur), attributs utilisateur qui n’ont pas d’incidence sur les fonctionnalités de base d’un utilisateur, y compris les préférences de l’utilisateur

<Warning>
  Dans le modèle de courriel Passwordless OTP, seule la propriété `email` de l’objet `user` est disponible.
</Warning>

* Lorsqu’un utilisateur se connecte au moyen d’une [Organization](/docs/fr-CA/manage-users/organizations/organizations-overview), les variables supplémentaires suivantes sont disponibles :
  * `organization.id`
  * `organization.display_name`
  * `organization.name`
  * `organization.metadata`
  * `organization.branding.logo_url`
  * `organization.branding.colors.primary`
  * `organization.branding.colors.page_background`

* Variables associées à plusieurs <Tooltip tip="Domaine personnalisé : domaine tiers ayant un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=custom+domains">domaines personnalisés</Tooltip> :

  * `custom_domain.domain`, le nom de domaine personnalisé utilisé (p. ex., `login.example.com`)
  * `custom_domain.domain_metadata`, les champs de métadonnées du domaine personnalisé (paires clé-valeur) que vous pouvez utiliser pour personnaliser le contenu du courriel selon le domaine

Lorsque vous utilisez [plusieurs domaines personnalisés](/docs/fr-CA/customize/custom-domains/multiple-custom-domains), Auth0 utilise automatiquement le domaine personnalisé approprié pour les liens dans les courriels. Vous pouvez utiliser ces variables pour personnaliser le contenu du courriel en fonction du domaine personnalisé qui a déclenché l’envoi du courriel.

**Exemple** : Personnaliser l’image de marque du courriel en fonction des métadonnées du domaine personnalisé :

```liquid
{% if custom_domain.domain_metadata.brand == "BrandA" %}
  <h1 style="color: #0000FF;">Welcome to Brand A</h1>
{% elsif custom_domain.domain_metadata.brand == "BrandB" %}
  <h1 style="color: #FF0000;">Welcome to Brand B</h1>
{% else %}
  <h1>Welcome</h1>
{% endif %}
```

**Exemple** : Incluez des renseignements de soutien propres au domaine :

```liquid
{% if custom_domain.domain_metadata.support_email %}
  <p>Need help? Contact us at {{ custom_domain.domain_metadata.support_email }}</p>
{% else %}
  <p>Need help? Contact us at {{ support_email }}</p>
{% endif %}
```

Pour utiliser des domaines personnalisés dans les courriels, assurez-vous d’avoir :

1. Configuré et vérifié vos [domaines personnalisés](/docs/fr-CA/customize/custom-domains)
2. Activé **Use Custom Domain in Emails** dans **Dashboard** &gt; **Branding** &gt; **Custom Domains**
3. Défini un [domaine personnalisé par défaut](/docs/fr-CA/customize/custom-domains/multiple-custom-domains/default-domain) si vous utilisez plusieurs domaines personnalisés

<div id="redirect-to-variables">
  ### Variables de Redirect To
</div>

Dans les modèles de courriel comportant un champ d’URL **Redirect To**, seules les trois variables suivantes sont prises en charge :

* `application.name` (ou son synonyme `client.name`)
* `application.clientID`
* `application.callback_domain` (ou son synonyme `client.callback_domain`), qui contient l’origine de la première URL figurant dans la liste **Allowed Callback URL** de l’application. Comme il s’agit d’une origine, elle comprend aussi le protocole (par exemple `https://`), en plus du domaine.

  Le domaine de rappel vous permet de rediriger les utilisateurs vers un chemin de l’application qui a déclenché l’action à l’aide d’une syntaxe comme `{{ application.callback_domain }}/result_page`.

  ### Variables du modèle de courriel d’inscription à l’AMF
</div>

* `link`, le lien d’inscription à l’AMF.

<div id="passwordless-otp-email-template-variables">
  ### Variables du modèle de courriel Passwordless OTP
</div>

* `send`, avec une valeur de `link`, `link_ios`, `link_android` ou `code`, selon le type de courriel sans mot de passe demandé
* `code`, avec le code à usage unique permettant d’accéder à l’application
* `link`, avec le lien sur lequel l’utilisateur peut cliquer pour accéder à l’application (uniquement pour les courriels sans mot de passe de type lien)
* `operation`, avec une valeur de `change_email` pour une opération de changement d’adresse courriel sans mot de passe
