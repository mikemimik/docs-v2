title: Modèles pour les Actions
---

La galerie de modèles Actions propose une variété de modèles de départ pour créer une Action. Ces modèles fournissent des exemples de code couvrant divers déclencheurs et cas d’utilisation, comme l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-factor+Authentication">authentification multifacteur</Tooltip>, le contrôle d’accès et l’enrichissement du profil.

Les modèles fournissent du code de départ que vous pouvez modifier et utiliser comme base dans le cadre d’un déclencheur. Une fois que vous avez sélectionné un modèle pour votre cas d’utilisation précis, vous pouvez ensuite personnaliser le code afin de répondre aux besoins particuliers de votre application.

Pour un guide étape par étape sur l’utilisation des modèles Actions, consultez [Créez votre première Action](/docs/fr-CA/customize/actions/write-your-first-action).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les modèles Actions sont aussi offerts à l’adresse suivante : https://github.com/auth0/opensource-marketplace.
</Callout>

<div id="best-practices">
  ## Meilleures pratiques
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les modèles Actions sont conçus uniquement à des fins d’apprentissage.

  Ces modèles se composent d’extraits de code incomplets et ne doivent en aucun cas être utilisés dans un environnement de production. Il incombe aux développeurs de développer de façon sécuritaire et de compléter le modèle Actions avant de le déployer dans un environnement de production.
</Callout>

* Assurez-vous que les modèles sont adaptés à votre environnement. Comme le code est incomplet, vous devez personnaliser et compléter les exemples avant de les importer dans un environnement de production.
* Ne déployez pas un modèle sans l’avoir d’abord modifié et complété.
* Les modèles Actions contiennent souvent des balises dans les commentaires, qui ne doivent être ni modifiées ni supprimées.
* Utilisez toujours la version la plus récente de tout package `npm`.
* Les modèles Actions ciblent par défaut les versions les plus récentes de Node.js. Après avoir créé un modèle, vous pouvez le modifier pour prendre en charge d’autres versions de Node.js au besoin.

Veuillez consulter l’ensemble des [directives de codage pour Actions](/docs/fr-CA/customize/actions/action-coding-guidelines), car elles s’appliquent également ici.
