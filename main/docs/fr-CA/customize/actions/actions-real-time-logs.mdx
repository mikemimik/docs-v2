---

La fonctionnalité Journaux en temps réel d’Actions affiche en temps réel tous les journaux générés par le code personnalisé de votre compte. Cela comprend toute la sortie de `console.log` ainsi que les exceptions. Ces journaux sont utiles pour déboguer vos Actions, vos scripts de base de données personnalisée et vos connexions sociales personnalisées.

<Warning>
  Lorsque vous utilisez les Journaux en temps réel d’Actions, faites preuve de prudence si vous consignez des renseignements sensibles. Par exemple, les paramètres reçus en entrée par les fonctions pour les Actions, les scripts de base de données personnalisée ou les connexions sociales personnalisées peuvent contenir des renseignements sensibles explicites ou implicites. La consignation d’objets complets peut entraîner la divulgation de renseignements sensibles dans les journaux en temps réel.

  Auth0 déconseille fortement ce type de pratique. **Nous vous recommandons de bien connaître les objets que vous consignez et de vous assurer qu’aucun renseignement sensible n’est consigné, tout en évitant de consigner explicitement quelque type de renseignement sensible que ce soit.**
</Warning>

<div id="how-to-use">
  ## Comment l’utiliser
</div>

Pour consulter les journaux en temps réel de votre code personnalisé :

1. Accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs).
2. Attendez que le flux se connecte. Les tentatives de connexion se font automatiquement.
3. Une fois la connexion établie, les entrées de journal s’affichent dans le panneau principal à mesure qu’elles surviennent.

<div id="features">
  ## Fonctionnalités
</div>

Le panneau Journaux en temps réel d’Actions vous permet de surveiller l’activité du système en temps réel. Il fournit un retour en direct sur les Actions exécutées dans le Dashboard, ce qui facilite l’identification des problèmes, la confirmation du bon déroulement des opérations ou simplement l’observation du comportement du système en temps réel. Consultez ci-dessous la liste détaillée des fonctionnalités actuellement offertes par cette fonctionnalité.

<div id="live-log-streaming">
  ### Diffusion en direct des journaux
</div>

Les entrées de journal sont diffusées en direct à mesure qu’elles sont générées par Actions. Chaque fois qu’une Action est exécutée, l’entrée de journal correspondante s’affiche automatiquement dans le panneau des journaux, sans qu’il soit nécessaire d’actualiser la page.

<div id="connection-status">
  ### État de la connexion
</div>

Le haut du panneau affiche l’état actuel de la connexion entre votre navigateur et le flux de logs. Cela vous permet de vérifier si des logs sont bien en cours de réception. Les états courants comprennent :

| État                         | Description                                 |
| ---------------------------- | ------------------------------------------- |
| Connexion au flux de logs... | Tentative d’établir une connexion.          |
| CONNECTED                    | Connecté avec succès et réception des logs. |
| DISCONNECTED                 | N’est pas connecté au flux pour le moment.  |

<div id="search-functionality">
  ### Fonction de recherche
</div>

Utilisez la barre de recherche en haut du panneau pour filtrer les entrées de log. Vous pouvez faire une recherche par :

* Mots-clés ou expressions
* Codes d’erreur précis
* Messages de log exacts

C’est particulièrement utile lorsque vous parcourez un grand volume de logs pour trouver une Action ou un problème précis.

<div id="options-menu">
  ### Menu **Options** (...)
</div>

Cliquez sur le menu **Options** dans le coin supérieur droit du panneau des logs pour accéder à des commandes supplémentaires.

* **Heure locale :** Bascule l’affichage de l’horodatage des entrées de log entre l’heure par défaut du serveur (souvent UTC) et l’heure locale du navigateur de l’utilisateur.
* **Télécharger :** Permet de télécharger les logs actuellement affichés au format JSON pour une analyse hors ligne, le partage ou l’archivage.
* **Effacer les logs :** Efface toutes les entrées de log actuellement visibles dans la fenêtre d’affichage. Cela ne supprime pas les logs du stockage système, seulement de la vue actuelle.

<div id="scroll-to-bottom">
  ### Faire défiler jusqu’en bas
</div>

Lorsque de nouvelles entrées arrivent et qu’elles ne sont plus visibles à l’écran, un bouton **Faire défiler jusqu’en bas** apparaît dans le panneau de journaux. En cliquant dessus, vous accéderez à l’entrée de journal la plus récente, ce qui vous permet de rester à jour avec les dernières activités.

<div id="debug-an-action">
  ## Déboguer une Action
</div>

L’exemple suivant explique comment créer une Action Hello World générique, l’exécuter et utiliser les Journaux en temps réel d’Actions pour voir les résultats.

1. Suivez les instructions de [Write Your First Action](/docs/fr-CA/customize/actions/write-your-first-action) pour écrire une Post Login Action à l’aide de l’exemple de code ci-dessous :

   ```javascript lines
   /**
   * Gestionnaire qui sera appelé pendant l’exécution d’un flux PostLogin.
   *
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   exports.onExecutePostLogin = async (event, api) => {
     console.log('Hello World!');
   };

   /**
   * Gestionnaire qui sera invoqué lorsque cette action reprend après une redirection externe. Si votre
   * fonction onExecutePostLogin n’effectue pas de redirection, cette fonction peut être ignorée sans risque.
   *
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   // exports.onContinuePostLogin = async (event, api) => {
   // };
   ```

2. Ouvrez un nouvel onglet et accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs)

3. Une fois le stream connecté, revenez à votre éditeur d’Action et lancez un test.

4. Vos résultats devraient s’afficher en temps réel dans les Journaux en temps réel d’Actions.

<div id="supported-statements">
  ### Instructions prises en charge
</div>

Le tableau suivant présente les instructions de console JavaScript prises en charge par la fonctionnalité de journalisation en temps réel :

| Instruction       | Description                                                                                                  | Cas d’utilisation                                                                   |
| ----------------- | ------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------- |
| `console.log()`   | Journalisation générale d’informations.                                                                      | Messages d’information de base, valeurs de variables.                               |
| `console.info()`  | Messages informatifs mettant en évidence la progression ou l’état de l’application.                          | Signalement d’opérations réussies, étapes clés de l’application.                    |
| `console.warn()`  | Indique des problèmes potentiels ou des situations inattendues qui ne constituent pas des erreurs critiques. | Signalement de problèmes non critiques, fonctionnalités obsolètes.                  |
| `console.error()` | Signale les erreurs survenues dans l’application.                                                            | Indication d’échecs, d’exceptions et de problèmes critiques.                        |
| `console.trace()` | Affiche une trace de la pile, montrant la séquence des appels de fonction menant au point actuel.            | Débogage de séquences d’appels complexes, compréhension de l’origine des problèmes. |
