---

La fonctionnalité Journaux en temps réel d’Actions affiche en temps réel tous les journaux du code personnalisé de votre compte. Cela comprend toute la sortie de `console.log` ainsi que les exceptions. Ces journaux sont utiles pour déboguer vos Actions, vos scripts de base de données personnalisée et vos connexions sociales personnalisées.

<Warning>
  Lorsque vous utilisez les Journaux en temps réel d’Actions, faites preuve de prudence si vous consignez des renseignements sensibles. Par exemple, les paramètres reçus en entrée par les fonctions pour les Actions, les scripts de base de données personnalisée ou les connexions sociales personnalisées peuvent contenir des renseignements sensibles explicites ou implicites. La consignation d’objets complets peut entraîner la divulgation de renseignements sensibles dans les journaux en temps réel.

  Auth0 déconseille fortement de telles pratiques. **Nous vous recommandons de bien connaître les objets que vous consignez, de vous assurer qu’aucun renseignement sensible n’est consigné et d’éviter de consigner explicitement tout type de renseignement sensible.**
</Warning>

<div id="how-to-use">
  ## Utilisation
</div>

Pour consulter les logs en temps réel de votre code personnalisé :

1. Accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs).
2. Attendez que le flux soit connecté. Les tentatives de connexion se font automatiquement.
3. Une fois la connexion établie, les entrées de log s’affichent dans le panneau principal dès qu’elles sont générées.

<div id="features">
  ## Fonctionnalités
</div>

Le panneau Journaux en temps réel d’Actions vous permet de surveiller l’activité du système au fur et à mesure qu’elle se produit. Il fournit un retour en direct sur les Actions exécutées à partir du tableau de bord, ce qui facilite l’identification des problèmes, la confirmation du bon déroulement des opérations ou simplement l’observation du comportement du système en temps réel. Vous trouverez ci-dessous une liste détaillée des fonctionnalités actuellement offertes par cette fonctionnalité.

<div id="live-log-streaming">
  ### Diffusion en direct des journaux
</div>

Les entrées de journal sont diffusées en direct au fur et à mesure qu’elles sont générées par Actions. Chaque fois qu’une Action est exécutée, l’entrée de journal correspondante apparaît automatiquement dans le panneau des journaux, sans qu’il soit nécessaire d’actualiser la page.

<div id="connection-status">
  ### État de la connexion
</div>

Le haut du panneau affiche l’état actuel de la connexion entre votre navigateur et le flux de journaux. Cela vous permet de vérifier si des journaux sont bien reçus en temps réel. Les états courants comprennent :

| État                             | Description                                                          |
| -------------------------------- | -------------------------------------------------------------------- |
| Connexion au flux de journaux... | Tentative d’établir une connexion.                                   |
| CONNECTÉ                         | La connexion est établie et les journaux sont en cours de réception. |
| DÉCONNECTÉ                       | Aucune connexion au flux n’est établie pour le moment.               |

<div id="search-functionality">
  ### Fonction de recherche
</div>

Utilisez la barre de recherche en haut du panneau pour filtrer les entrées de journal. Vous pouvez faire une recherche par :

* Mots-clés ou expressions
* Codes d’erreur précis
* Messages de journal exacts

C’est particulièrement utile lorsque vous parcourez de grands volumes de journaux à la recherche d’une Action ou d’un problème précis.

<div id="options-menu">
  ### Menu Options (...)
</div>

Cliquez sur le menu **Options** dans le coin supérieur droit du panneau des journaux pour accéder à des commandes supplémentaires.

* **Heure locale :** Bascule l’affichage de l’horodatage des entrées de journal entre l’heure par défaut du serveur (souvent UTC) et l’heure locale du navigateur de l’utilisateur.
* **Télécharger :** Vous permet de télécharger les journaux actuellement affichés au format JSON pour analyse hors ligne, partage ou archivage.
* **Effacer les journaux :** Efface toutes les entrées de journal actuellement visibles dans la fenêtre d’affichage. Cela ne supprime pas les journaux du stockage du système, seulement de votre vue actuelle.

<div id="scroll-to-bottom">
  ### Défiler vers le bas
</div>

Lorsque de nouvelles entrées arrivent et ne sont plus visibles, un bouton **Défiler vers le bas** apparaît dans le panneau des journaux. En cliquant dessus, vous accéderez à l’entrée de journal la plus récente, ce qui vous permet de toujours voir l’activité la plus récente.

<div id="debug-an-action">
  ## Déboguer une Action
</div>

L’exemple suivant explique comment créer une Action générique Hello World, l’exécuter et utiliser les Journaux en temps réel d’Actions pour voir les résultats.

1. Suivez les instructions de [Write Your First Action](/docs/fr-CA/customize/actions/write-your-first-action) pour rédiger une Action Post Login à l’aide de l’exemple de code ci-dessous :

   ```javascript lines
   /**
   * Handler qui sera appelé pendant l’exécution d’un flux PostLogin.
   *
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   exports.onExecutePostLogin = async (event, api) => {
     console.log('Hello World!');
   };

   /**
   * Handler qui sera invoqué lorsque cette action reprendra après une redirection externe. Si votre
   * fonction onExecutePostLogin n’effectue pas de redirection, vous pouvez ignorer cette fonction en toute sécurité.
   *
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   // exports.onContinuePostLogin = async (event, api) => {
   // };
   ```

2. Ouvrez un nouvel onglet, puis accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs)

3. Une fois le flux connecté, revenez à votre Action editor et exécutez un test.

4. Vos résultats devraient s’afficher en temps réel dans les Journaux en temps réel d’Actions.

<div id="supported-statements">
  ### Instructions prises en charge
</div>

Le tableau suivant présente les instructions de la console Javascript prises en charge par la fonctionnalité de journalisation en temps réel :

| Instruction       | Description                                                                                                  | Cas d&#39;utilisation                                                                  |
| ----------------- | ------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------- |
| `console.log()`   | Journalisation générale d&#39;informations.                                                                  | Messages d&#39;information de base, valeurs de variables.                              |
| `console.info()`  | Messages informatifs mettant en évidence la progression ou l&#39;état de l&#39;application.                  | Signalement d&#39;opérations réussies, étapes clés de l&#39;application.               |
| `console.warn()`  | Indique des problèmes potentiels ou des situations inattendues qui ne constituent pas des erreurs critiques. | Signalement de problèmes non critiques, fonctionnalités obsolètes.                     |
| `console.error()` | Signale des erreurs survenues dans l&#39;application.                                                        | Signalement d&#39;échecs, d&#39;exceptions et de problèmes critiques.                  |
| `console.trace()` | Affiche une trace de pile montrant la séquence des appels de fonction ayant mené au point actuel.            | Débogage de flux d&#39;appels complexes, compréhension de l&#39;origine des problèmes. |
