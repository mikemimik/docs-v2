---

La fonctionnalité Journaux en temps réel d’Actions affiche en temps réel tous les journaux du code personnalisé de votre compte. Cela comprend toute sortie `console.log` ainsi que les exceptions. Ces journaux sont utiles pour déboguer vos Actions, vos scripts de base de données personnalisée et vos connexions sociales personnalisées.

<Warning>
  Lorsque vous utilisez les Journaux en temps réel d’Actions, faites preuve de prudence si vous consignez des renseignements sensibles. Par exemple, les paramètres reçus en entrée par les fonctions pour les Actions, les scripts de base de données personnalisée ou les connexions sociales personnalisées peuvent contenir des renseignements sensibles, explicites ou implicites. La consignation d’objets complets peut entraîner la divulgation de renseignements sensibles dans les journaux en temps réel.

  Auth0 déconseille fortement ce genre de pratiques. **Nous vous recommandons de bien savoir quels objets vous consignez, de vous assurer qu’aucun renseignement sensible n’est consigné et d’éviter de consigner explicitement tout type de renseignement sensible.**
</Warning>

<div id="how-to-use">
  ## Utilisation
</div>

Pour consulter les journaux en temps réel de votre code personnalisé :

1. Accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs).
2. Attendez que le flux se connecte. Les tentatives de connexion se font automatiquement.
3. Une fois la connexion établie, les entrées de journal s’affichent dans le panneau principal au fur et à mesure.

<div id="features">
  ## Fonctionnalités
</div>

Le panneau journaux en temps réel d’Actions vous permet de surveiller l’activité du système au moment où elle se produit. Il fournit une rétroaction en direct sur les Actions exécutées dans le Dashboard, ce qui facilite l’identification des problèmes, la confirmation du bon déroulement des opérations ou simplement l’observation du comportement du système en temps réel. Vous trouverez ci-dessous une liste détaillée des fonctionnalités actuellement offertes par cette fonctionnalité.

<div id="live-log-streaming">
  ### Diffusion en direct des journaux
</div>

Les entrées de journal sont diffusées en direct dès qu’elles sont générées par Actions. Chaque fois qu’une Action s’exécute, l’entrée de journal correspondante s’affiche automatiquement dans le panneau des journaux, sans qu’il soit nécessaire d’actualiser la page.

<div id="connection-status">
  ### État de la connexion
</div>

La partie supérieure du panneau affiche l’état actuel de la connexion entre votre navigateur et le flux de journaux. Cela vous aide à vérifier si des journaux sont bien reçus. Les états courants comprennent :

| État                             | Description                                  |
| -------------------------------- | -------------------------------------------- |
| Connexion au flux de journaux... | Tentative d’établir une connexion.           |
| CONNECTÉ                         | Connexion établie et réception des journaux. |
| DÉCONNECTÉ                       | N’est pas actuellement connecté au flux.     |

<div id="search-functionality">
  ### Fonction de recherche
</div>

Utilisez la barre de recherche en haut du panneau pour filtrer les entrées du journal. Vous pouvez faire une recherche par :

* Mots-clés ou expressions
* Codes d’erreur précis
* Messages de journal exacts

C’est particulièrement utile lorsque vous parcourez de gros volumes de journaux pour trouver une Action ou un problème précis.

<div id="options-menu">
  ### Menu Options (...)
</div>

Cliquez sur le menu **Options** dans le coin supérieur droit du panneau des journaux pour accéder à des commandes supplémentaires.

* **Heure locale :** Permet de basculer l’affichage de l’horodatage des entrées de journal entre l’heure par défaut du serveur (souvent UTC) et l’heure locale du navigateur de l’utilisateur.
* **Télécharger :** Permet de télécharger les journaux actuellement affichés sous forme de fichier JSON pour une analyse hors ligne, le partage ou l’archivage.
* **Effacer les journaux :** Efface toutes les entrées de journal actuellement visibles dans la fenêtre d’affichage. Cela ne supprime pas les journaux du stockage du système, mais seulement de la vue actuelle.

<div id="scroll-to-bottom">
  ### Faire défiler vers le bas
</div>

Lorsque de nouvelles entrées arrivent et qu’elles sont hors du champ de vision, un bouton **Faire défiler vers le bas** apparaît dans le panneau des journaux. En cliquant dessus, vous serez redirigé vers l’entrée de journal la plus récente, ce qui vous permet de rester à jour avec la plus récente activité.

<div id="debug-an-action">
  ## Déboguer une Action
</div>

L’exemple suivant montre comment créer une Action Hello World générique, l’exécuter et utiliser les journaux en temps réel d’Actions pour voir les résultats.

1. Suivez les instructions de [Créez votre première Action](/docs/fr-CA/customize/actions/write-your-first-action) pour créer une Post Login Action à l’aide de l’exemple de code ci-dessous :

   ```javascript lines
   /**
   * Gestionnaire qui sera appelé pendant l’exécution d’un flux PostLogin.
   *
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   exports.onExecutePostLogin = async (event, api) => {
     console.log('Hello World!');
   };

   /**
   * Gestionnaire qui sera invoqué lorsque cette action reprendra après une redirection externe. Si votre
   * fonction onExecutePostLogin n’effectue pas de redirection, cette fonction peut être ignorée sans risque.
   *
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   // exports.onContinuePostLogin = async (event, api) => {
   // };
   ```

2. Ouvrez un nouvel onglet et accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs)

3. Une fois le stream connecté, revenez à votre Action editor et lancez un Test.

4. Les résultats devraient s’afficher en temps réel dans les journaux en temps réel d’Actions.

<div id="supported-statements">
  ### Instructions prises en charge
</div>

Le tableau suivant présente les instructions de console Javascript prises en charge par la fonctionnalité de journalisation en temps réel :

| Instruction       | Description                                                                                                  | Cas d’utilisation                                                              |
| ----------------- | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------ |
| `console.log()`   | Journalisation générale d’informations.                                                                      | Messages d’information de base, valeurs de variables.                          |
| `console.info()`  | Messages d’information mettant en évidence la progression ou l’état de l’application.                        | Signalement des opérations réussies, étapes importantes de l’application.      |
| `console.warn()`  | Indique des problèmes potentiels ou des situations inattendues qui ne constituent pas des erreurs critiques. | Signalement de problèmes non critiques, fonctionnalités obsolètes.             |
| `console.error()` | Signale des erreurs survenues dans l’application.                                                            | Indication d’échecs, d’exceptions et de problèmes critiques.                   |
| `console.trace()` | Affiche une trace de la pile, montrant la séquence des appels de fonction menant au point actuel.            | Débogage de flux d’appels complexes, compréhension de l’origine des problèmes. |
