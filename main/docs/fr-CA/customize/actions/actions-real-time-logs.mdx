title: Journaux en temps réel des Actions
---

La fonctionnalité Journaux en temps réel des Actions affiche tous les journaux du code personnalisé de votre compte en temps réel. Cela comprend toutes les sorties `console.log` et les exceptions. Ces journaux sont utiles pour déboguer vos Actions, vos scripts de base de données personnalisée et vos connexions sociales personnalisées.

<Warning>
  Lorsque vous utilisez les Journaux en temps réel des Actions, faites attention à ne pas consigner de renseignements sensibles. Par exemple, les paramètres reçus en entrée par les Actions, les scripts de base de données personnalisée ou les connexions sociales personnalisées peuvent contenir des renseignements sensibles, explicites ou implicites. La consignation d’objets complets peut entraîner la divulgation de renseignements sensibles dans les journaux en temps réel.

  Auth0 déconseille fortement ce type de pratique. **Nous vous recommandons de porter une attention particulière aux objets que vous consignez et de vous assurer qu’aucun renseignement sensible n’est consigné, tout en évitant de consigner explicitement quelque renseignement sensible que ce soit.**
</Warning>

<div id="how-to-use">
  ## Utilisation
</div>

Pour consulter les journaux en temps réel de votre code personnalisé :

1. Accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs).
2. Attendez que le flux soit connecté. Les tentatives de connexion se font automatiquement.
3. Une fois la connexion établie, les entrées de journal s’affichent dans le panneau principal à mesure qu’elles surviennent.

<div id="features">
  ## Fonctionnalités
</div>

Le panneau Actions Real-time logs vous permet de surveiller l’activité du système en temps réel. Il fournit une rétroaction en direct sur les Actions exécutées à partir du tableau de bord, ce qui facilite l’identification des problèmes, la confirmation du bon déroulement des opérations ou simplement l’observation du comportement du système en temps réel. Vous trouverez ci-dessous une liste détaillée des fonctionnalités actuellement offertes par cette fonctionnalité.

<div id="live-log-streaming">
  ### Diffusion en direct des journaux
</div>

Les entrées de journal sont diffusées en direct à mesure qu’elles sont générées par Actions. Chaque fois qu’une Action est exécutée, l’entrée de journal correspondante s’affiche automatiquement dans le panneau des journaux, sans qu’il soit nécessaire d’actualiser la page.

<div id="connection-status">
  ### État de la connexion
</div>

Le haut du panneau affiche l’état actuel de la connexion entre votre navigateur et le flux de journaux. Cela vous permet de vérifier si des journaux sont bien reçus. Les états courants comprennent :

| État                             | Description                                  |
| -------------------------------- | -------------------------------------------- |
| Connexion au flux de journaux... | Tentative d’établir une connexion.           |
| CONNECTÉ                         | Connexion établie et réception des journaux. |
| DÉCONNECTÉ                       | Aucune connexion au flux pour le moment.     |

<div id="search-functionality">
  ### Fonction de recherche
</div>

Utilisez la barre de recherche en haut du panneau pour filtrer les entrées du journal. Vous pouvez effectuer une recherche par :

* Mots-clés ou expressions
* Codes d’erreur précis
* Messages de journal exacts

C’est particulièrement utile lorsque vous parcourez de gros volumes de journaux pour trouver une Action ou un problème précis.

<div id="options-menu">
  ### Menu Options (...)
</div>

Cliquez sur le menu **Options** dans le coin supérieur droit du panneau des journaux pour accéder à des commandes supplémentaires.

* **Heure locale :** Permet d’alterner l’affichage de l’horodatage des entrées de journal entre l’heure par défaut du serveur (souvent UTC) et l’heure locale du navigateur de l’utilisateur.
* **Télécharger :** Permet de télécharger les journaux actuellement affichés sous forme de fichier JSON pour les analyser hors ligne, les partager ou les archiver.
* **Effacer les journaux :** Efface toutes les entrées de journal actuellement visibles dans la fenêtre d’affichage. Cela ne supprime pas les journaux du stockage système, seulement de votre vue actuelle.

<div id="scroll-to-bottom">
  ### Faire défiler jusqu’en bas
</div>

Lorsque de nouvelles entrées arrivent et ne sont plus visibles, un bouton **Faire défiler jusqu’en bas** apparaît dans le panneau des journaux. En cliquant dessus, vous accéderez à l’entrée de journal la plus récente, afin d’être toujours au fait de l’activité la plus récente.

<div id="debug-an-action">
  ## Déboguer une Action
</div>

L&#39;exemple suivant explique comment créer une Action générique Hello World, l&#39;exécuter et utiliser les Journaux en temps réel des Actions pour voir les résultats.

1. Suivez les instructions de [Créer votre première Action](/docs/fr-CA/customize/actions/write-your-first-action) pour écrire une Post Login Action à l&#39;aide de l&#39;exemple de code ci-dessous :

   ```javascript lines
   /**
   * Handler qui sera appelé pendant l'exécution d'un flow PostLogin.
   *
   * @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement du login.
   */
   exports.onExecutePostLogin = async (event, api) => {
     console.log('Hello World!');
   };

   /**
   * Handler qui sera invoqué lorsque cette action reprend après une redirection externe. Si votre
   * fonction onExecutePostLogin n'effectue pas de redirection, cette fonction peut être ignorée sans problème.
   *
   * @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement du login.
   */
   // exports.onContinuePostLogin = async (event, api) => {
   // };
   ```

2. Ouvrez un nouvel onglet, puis accédez à [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs)

3. Une fois le flux connecté, revenez à votre Action editor et exécutez un test.

4. Les résultats devraient s&#39;afficher en temps réel dans vos Journaux en temps réel des Actions.

<div id="supported-statements">
  ### Instructions prises en charge
</div>

Le tableau suivant présente les instructions de la console JavaScript prises en charge par la fonctionnalité de journalisation en temps réel :

| Instruction       | Description                                                                                                  | Cas d’utilisation                                                                      |
| ----------------- | ------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------- |
| `console.log()`   | Journalisation générale d’informations.                                                                      | Messages d’information de base, valeurs de variables.                                  |
| `console.info()`  | Messages d’information mettant en évidence la progression ou l’état de l’application.                        | Signalement d’opérations réussies, jalons de l’application.                            |
| `console.warn()`  | Indique des problèmes potentiels ou des situations inattendues qui ne constituent pas des erreurs critiques. | Signalement de problèmes non critiques, fonctionnalités obsolètes.                     |
| `console.error()` | Signale les erreurs survenues dans l’application.                                                            | Indication des échecs, des exceptions et des problèmes critiques.                      |
| `console.trace()` | Affiche une trace de la pile, montrant la séquence des appels de fonction menant au point actuel.            | Débogage d’enchaînements d’appels complexes, compréhension de l’origine des problèmes. |
