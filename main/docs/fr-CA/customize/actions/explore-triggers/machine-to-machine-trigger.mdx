Le déclencheur Machine to Machine s’exécute lorsqu’un <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">Jeton d’accès</Tooltip> est émis au moyen du [Client Credentials Flow](/docs/fr-CA/get-started/authentication-and-authorization-flow/client-credentials-flow).

<Frame>![Schéma illustrant le Machine to Machine Flow d’Actions et le moment où les déclencheurs qu’il contient s’exécutent.](/docs/images/cdy7uua7fh8z/1JPl54LFWCUh5StuglZS2o/41f89372526574c3b8cdac4d5ba38072/Machine_to_Machine_Flow.png)</Frame>

Les Actions de ce flux sont bloquantes (synchrones), ce qui signifie qu’elles s’exécutent dans le cadre du processus du déclencheur et empêchent le reste du pipeline Auth0 de s’exécuter tant que l’Action n’est pas terminée.

<div id="m2m-client-credentials">
  ### M2M / Client Credentials
</div>

Le déclencheur `credentials-exchange` est une fonction exécutée avant le renvoi du jeton d’accès.

<div id="common-use-cases">
  ## Cas d’utilisation courants
</div>

<div id="access-control">
  ### Contrôle d’accès
</div>

Une Action credentials-exchange peut servir à rejeter un jeton d’accès selon une logique personnalisée.

```javascript lines
/**
 * @param {Event} event - Détails sur la demande d'octroi des identifiants client.
 * @param {CredentialsExchangeAPI} api - Interface dont les méthodes permettent de modifier le comportement de l'octroi des identifiants client.
 */
exports.onExecuteCredentialsExchange = async (event, api) => {
  if (event.request.geoip.continentCode === "NA") {
    api.access.deny('invalid_request', "Access from North America is not allowed.");
  }
};
```

<div id="add-custom-claims-to-the-access-token">
  ### Ajouter des revendications personnalisées au jeton d’accès
</div>

Une action credentials-exchange peut être utilisée pour ajouter des revendications personnalisées à un jeton d’accès.

```javascript lines
/**
 * @param {Event} event - Détails sur la requête de l’octroi des identifiants client.
 * @param {CredentialsExchangeAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de l’octroi des identifiants client.
 */
exports.onExecuteCredentialsExchange = async (event, api) => {
  api.accessToken.setCustomClaim("https://my-api.exampleco.com/request-ip", event.request.ip);  
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Nous recommandons fortement d’utiliser une revendication personnalisée dotée d’un espace de noms, sous la forme d’un URI. Pour en savoir plus sur les revendications personnalisées avec ou sans espace de noms, consultez [Créer des revendications personnalisées](/docs/fr-CA/secure/tokens/json-web-tokens/create-custom-claims).
</Callout>
