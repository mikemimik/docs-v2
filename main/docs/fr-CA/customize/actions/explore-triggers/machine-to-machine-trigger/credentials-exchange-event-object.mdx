<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="objet">
  Renseignements sur la transaction Credentials Exchange.

  <Expandable title="propriétés de la transaction" defaultOpen>
    <ResponseField name="correlation_id" type="string" post={["optional"]}>
      L’ID de corrélation peut être fourni dans la requête d’authentification initiale lorsque l’application redirige vers Universal Login. Vous pouvez utiliser cette valeur pour corréler les journaux et les requêtes de votre code Action avec le parcours de l’utilisateur.
    </ResponseField>

    <ResponseField name="requested_scopes" type="tableau de chaînes de caractères">
      Les scopes précisés, le cas échéant, lors de la requête du jeton d’accès.
    </ResponseField>

    <ResponseField name="target_scopes" type="tableau de chaînes de caractères" post={["optional"]}>
      &#91;Accès anticipé&#93; L’ensemble de scopes cibles actif pour le jeton d’accès. Il est initialisé à partir des autorisations du client et mis à jour immédiatement par les méthodes de scope cible de api.transaction dans les Actions en cours et suivantes. Une fois toutes les Actions terminées, ces scopes sont recoupés avec les autorisations du client. Les scopes qui ne figurent pas dans les autorisations sont omis sans avertissement du jeton d’accès final.
    </ResponseField>
  </Expandable>
</ResponseField>
