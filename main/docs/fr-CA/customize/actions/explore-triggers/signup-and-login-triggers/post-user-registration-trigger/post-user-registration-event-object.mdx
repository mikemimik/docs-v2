<div id="eventsecurity_context">
  ## `event.security_context`
</div>

<ResponseField name="event.security_context" type="object" post={["optional"]}>
  Un objet contenant des signatures d’empreinte. Il n’est disponible que si le client utilise Cloudflare. L’empreinte JA3/JA4 peut être `null` ou vide dans certains cas. Le cas le plus courant concerne les requêtes HTTP, puisque JA3 et JA4 sont calculés au niveau de TLS. Elle peut aussi être vide parce que le Worker envoie des requêtes à l’intérieur de la même zone ou vers une zone qui n’est pas proxyfiée (ou vers un tiers).

  <Expandable title="propriétés de security_context" defaultOpen>
    <ResponseField name="ja3" type="string" post={["optional"]}>
      Signature d’empreinte JA3. Elle n’est disponible que si le client utilise une connexion TLS.
    </ResponseField>

    <ResponseField name="ja4" type="string" post={["optional"]}>
      Signature d’empreinte JA4. Elle n’est disponible que si le client utilise une connexion TLS.
    </ResponseField>
  </Expandable>
</ResponseField>
