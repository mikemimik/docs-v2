<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="dictionary">
  Détails sur la transaction en cours d&#39;échange de jetons personnalisé.

  <Expandable title="propriétés de la transaction" defaultOpen>
    <ResponseField name="actor_token" type="string" post={["optional"]}>
      Le jeton de l’acteur fourni dans la requête d’échange de jeton.
    </ResponseField>

    <ResponseField name="actor_token_type" type="string" post={["optional"]}>
      Le type du jeton d’acteur fourni dans la requête d’échange de jetons.
    </ResponseField>

    <ResponseField name="actor_token_user" type="dictionary" post={["optional"]}>
      L&#39;utilisateur représenté par le jeton d&#39;acteur. Cette valeur ne sera présente que si le actor&#95;token&#95;type est urn:ietf:params:oauth:token-type:id&#95;token et que le jeton d&#39;acteur fourni dans la requête d&#39;échange de jeton est un ID token valide généré par Auth0.

      <Expandable title="propriétés de actor_token_user" defaultOpen>
        <ResponseField name="app_metadata" type="dictionary">
          Champs personnalisés qui stockent des renseignements sur un utilisateur et qui influencent son accès, comme le forfait de soutien, les rôles de sécurité ou les groupes de contrôle d’accès.
        </ResponseField>

        <ResponseField name="created_at" type="string">
          Horodatage indiquant à quel moment le profil d’utilisateur a été créé pour la première fois.
        </ResponseField>

        <ResponseField name="courriel" type="string" post={["optional"]}>
          (unique) Adresse courriel de l’utilisateur.
        </ResponseField>

        <ResponseField name="email_verified" type="boolean">
          Indique si l’utilisateur a vérifié son adresse courriel.
        </ResponseField>

        <ResponseField name="nom_de_famille" type="string" post={["optional"]}>
          Nom de famille de l’utilisateur.
        </ResponseField>

        <ResponseField name="given_name" type="string" post={["optional"]}>
          Prénom de l’utilisateur.
        </ResponseField>

        <ResponseField name="dernière_réinitialisation_du_mot_de_passe" type="string" post={["optional"]}>
          Horodatage indiquant la dernière réinitialisation ou modification du mot de passe de l’utilisateur. À la création de l’utilisateur, ce champ n’existe pas. Cette propriété est offerte uniquement pour les connexions de base de données.
        </ResponseField>

        <ResponseField name="nom" type="string" post={["optional"]}>
          Nom complet de l’utilisateur.
        </ResponseField>

        <ResponseField name="surnom" type="string" post={["optional"]}>
          Pseudo de l’utilisateur.
        </ResponseField>

        <ResponseField name="phone_number" type="string" post={["optional"]}>
          Numéro de téléphone de l’utilisateur.
        </ResponseField>

        <ResponseField name="phone_verified" type="boolean" post={["optional"]}>
          Indique si l’utilisateur a vérifié son numéro de téléphone.
        </ResponseField>

        <ResponseField name="photo" type="string" post={["optional"]}>
          URL de la [photo de profil de l’utilisateur](https://auth0.com/docs/users/change-user-picture).
        </ResponseField>

        <ResponseField name="updated_at" type="string">
          Horodatage indiquant la date et l’heure de la dernière mise à jour/modification du profil de l’utilisateur.
        </ResponseField>

        <ResponseField name="id_utilisateur" type="string">
          (unique) Identifiant unique de l&#39;utilisateur.
        </ResponseField>

        <ResponseField name="user_metadata" type="dictionary">
          Des champs personnalisés qui stockent des renseignements sur un utilisateur sans incidence sur ce à quoi il a accès, comme son adresse professionnelle, son adresse personnelle ou ses préférences.
        </ResponseField>

        <ResponseField name="nom d’utilisateur" type="string" post={["optional"]}>
          (unique) Nom d’utilisateur.
        </ResponseField>

        <ResponseField name="enrolledFactors" type="array of objects" post={["optional"]}>
          Un tableau des facteurs d’authentification auxquels l’utilisateur est inscrit.

          <Expandable title="propriétés de enrolledFactors">
            <ResponseField name="type" type="string">
              Le type de facteur d’authentification, comme `push-notification`, `phone`, `email`, `otp`, `webauthn-roaming` et `webauthn-platform`.
            </ResponseField>

            <ResponseField name="options" type="dictionary" post={["optional"]}>
              Options supplémentaires décrivant cette instance du facteur inscrit.
            </ResponseField>
          </Expandable>
        </ResponseField>

        <ResponseField name="multifacteur" type="array of strings" post={["optional"]}>
          Liste des fournisseurs d’authentification multifacteur (MFA) auprès desquels l’utilisateur est inscrit. Ce tableau est mis à jour lorsque l’utilisateur s’inscrit à l’authentification multifacteur et lorsqu’un administrateur réinitialise les inscriptions d’un utilisateur à la MFA.
        </ResponseField>

        <ResponseField name="identités" type="array of objects">
          Contient les renseignements récupérés auprès du fournisseur d’identité utilisé initialement par l’utilisateur pour s’authentifier. Les utilisateurs peuvent aussi lier leur profil à plusieurs fournisseurs d’identité; ces identités figureront alors également dans ce tableau. Le contenu de chaque objet de fournisseur d’identité varie selon le fournisseur.

          <Expandable title="propriétés de l’élément identities">
            <ResponseField name="connection" type="string" post={["optional"]}>
              Nom de la connexion Auth0 utilisée pour authentifier l’utilisateur.
            </ResponseField>

            <ResponseField name="isSocial" type="boolean" post={["optional"]}>
              Indique s’il s’agit d’une connexion sociale.
            </ResponseField>

            <ResponseField name="profileData" type="dictionary" post={["optional"]}>
              Renseignements sur l’utilisateur associés à la connexion. Lorsque des profils sont liés, ce champ contient les renseignements sur l’utilisateur associés aux comptes secondaires.
            </ResponseField>

            <ResponseField name="provider" type="string" post={["optional"]}>
              Nom de l’entité qui authentifie l’utilisateur, comme Facebook, Google, SAML ou votre propre fournisseur.
            </ResponseField>

            <ResponseField name="user_id" type="string" post={["optional"]}>
              Identifiant unique de l’utilisateur pour cette connexion/ce fournisseur.
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="requested_scopes" type="array of strings">
      Les scopes demandés (le cas échéant) qui sont fournis dans la demande d’échange de jeton.
    </ResponseField>

    <ResponseField name="requested_token_type" type="string">
      Le type de token qu’Auth0 doit générer. Par exemple : urn:ietf:params:oauth:token-type:access&#95;token.
    </ResponseField>

    <ResponseField name="subject_token" type="string">
      Le jeton de sujet fourni dans la demande d’échange de jetons.
    </ResponseField>

    <ResponseField name="subject_token_type" type="string">
      Le subject&#95;token&#95;type fourni dans la requête d’échange de jeton.
    </ResponseField>
  </Expandable>
</ResponseField>
