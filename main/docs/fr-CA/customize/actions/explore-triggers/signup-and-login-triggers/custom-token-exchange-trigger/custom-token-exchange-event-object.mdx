<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="dictionary">
  Détails sur la transaction d&#39;échange personnalisé de jetons en cours.

  <Expandable title="propriétés de la transaction" defaultOpen>
    <ResponseField name="actor_token" type="string" post={["optional"]}>
      Le jeton de l’acteur fourni dans la requête d’échange de jetons.
    </ResponseField>

    <ResponseField name="actor_token_type" type="string" post={["optional"]}>
      Le type du jeton de l’acteur fourni dans la requête d’échange de jetons.
    </ResponseField>

    <ResponseField name="actor_token_user" type="dictionary" post={["optional"]}>
      L&#39;utilisateur représenté par le jeton d&#39;acteur. Cette valeur ne sera présente que si le actor&#95;token&#95;type est urn:ietf:params:oauth:token-type:id&#95;token et que le jeton d&#39;acteur fourni dans la requête d&#39;échange de jeton est un ID token valide généré par Auth0.

      <Expandable title="actor_token_user propriétés" defaultOpen>
        <ResponseField name="app_metadata" type="dictionary">
          Des champs personnalisés qui stockent des renseignements sur un utilisateur et qui influencent son accès, comme le forfait de soutien, les rôles de sécurité ou les groupes de contrôle d’accès.
        </ResponseField>

        <ResponseField name="created_at" type="string">
          Horodatage indiquant la date de création initiale du profil utilisateur.
        </ResponseField>

        <ResponseField name="courriel" type="string" post={["optional"]}>
          (unique) Adresse courriel de l’utilisateur.
        </ResponseField>

        <ResponseField name="email_verified" type="boolean">
          Indique si l’utilisateur a vérifié son adresse courriel.
        </ResponseField>

        <ResponseField name="nom_de_famille" type="string" post={["optional"]}>
          Nom de famille de l’utilisateur.
        </ResponseField>

        <ResponseField name="given_name" type="string" post={["optional"]}>
          Prénom de l’utilisateur.
        </ResponseField>

        <ResponseField name="last_password_reset" type="string" post={["optional"]}>
          Horodatage indiquant la date et l’heure de la dernière réinitialisation/modification du mot de passe de l’utilisateur. Lors de la création de l’utilisateur, ce champ n’existe pas. Cette propriété n’est disponible que pour les connexions de base de données.
        </ResponseField>

        <ResponseField name="nom" type="string" post={["optional"]}>
          Nom complet de l’utilisateur.
        </ResponseField>

        <ResponseField name="pseudonyme" type="string" post={["optional"]}>
          Le pseudonyme de l’utilisateur.
        </ResponseField>

        <ResponseField name="phone_number" type="string" post={["optional"]}>
          Numéro de téléphone de l’utilisateur.
        </ResponseField>

        <ResponseField name="phone_verified" type="boolean" post={["optional"]}>
          Indique si l’utilisateur a vérifié son numéro de téléphone.
        </ResponseField>

        <ResponseField name="photo" type="string" post={["optional"]}>
          URL de la [photo de profil de l’utilisateur](https://auth0.com/docs/users/change-user-picture).
        </ResponseField>

        <ResponseField name="updated_at" type="string">
          Horodatage indiquant la date et l’heure de la dernière mise à jour/modification du profil de l’utilisateur.
        </ResponseField>

        <ResponseField name="ID_utilisateur" type="string">
          (unique) Identifiant unique de l&#39;utilisateur.
        </ResponseField>

        <ResponseField name="user_metadata" type="dictionary">
          Les champs personnalisés qui servent à stocker des renseignements sur un utilisateur sans avoir d’incidence sur ce à quoi il peut accéder, comme son adresse professionnelle, son adresse domiciliaire ou ses préférences.
        </ResponseField>

        <ResponseField name="nom d’utilisateur" type="string" post={["optional"]}>
          (unique) Nom d’utilisateur.
        </ResponseField>

        <ResponseField name="facteursInscrits" type="array of objects" post={["optional"]}>
          Un tableau de facteurs d’authentification que l’utilisateur a enregistrés.

          <Expandable title="propriétés de enrolledFactors">
            <ResponseField name="type" type="string">
              Le type de facteur d’authentification, par exemple `push-notification`, `phone`, `email`, `otp`, `webauthn-roaming` et `webauthn-platform`.
            </ResponseField>

            <ResponseField name="options" type="dictionary" post={["optional"]}>
              Options supplémentaires décrivant cette instance du facteur enregistré.
            </ResponseField>
          </Expandable>
        </ResponseField>

        <ResponseField name="multifacteur" type="array of strings" post={["optional"]}>
          Liste des fournisseurs d’authentification multifacteur (MFA) auprès desquels l’utilisateur est inscrit. Ce tableau est mis à jour lorsque l’utilisateur s’inscrit à l’authentification multifacteur (MFA) et lorsqu’un administrateur réinitialise les inscriptions MFA d’un utilisateur.
        </ResponseField>

        <ResponseField name="identités" type="array of objects">
          Contient les renseignements récupérés auprès du fournisseur d&#39;identité par lequel l&#39;utilisateur s&#39;authentifie initialement. Les utilisateurs peuvent aussi lier leur profil à plusieurs fournisseurs d&#39;identité; ces identités apparaîtront alors également dans ce tableau. Le contenu de chaque objet de fournisseur d&#39;identité varie selon le fournisseur.

          <Expandable title="propriétés de l'élément identities">
            <ResponseField name="connection" type="string" post={["optional"]}>
              Nom de la connexion Auth0 utilisée pour authentifier l&#39;utilisateur.
            </ResponseField>

            <ResponseField name="isSocial" type="boolean" post={["optional"]}>
              Indique si la connexion est de type social.
            </ResponseField>

            <ResponseField name="profileData" type="dictionary" post={["optional"]}>
              Renseignements sur l&#39;utilisateur associés à la connexion. Lorsque des profils sont liés, il contient les renseignements utilisateur associés aux comptes secondaires.
            </ResponseField>

            <ResponseField name="provider" type="string" post={["optional"]}>
              Nom de l&#39;entité qui authentifie l&#39;utilisateur, comme Facebook, Google, SAML ou votre propre fournisseur.
            </ResponseField>

            <ResponseField name="user_id" type="string" post={["optional"]}>
              Identifiant unique de l&#39;utilisateur pour cette connexion ou ce fournisseur.
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="requested_scopes" type="array of strings">
      Les scopes demandés (le cas échéant) dans la demande d’échange de jeton.
    </ResponseField>

    <ResponseField name="type_de_jeton_demandé" type="string">
      Le type de token généré par Auth0. Par exemple : urn:ietf:params:oauth:token-type:access&#95;token.
    </ResponseField>

    <ResponseField name="subject_token" type="string">
      Le jeton de sujet fourni dans la requête d’échange de jetons.
    </ResponseField>

    <ResponseField name="subject_token_type" type="string">
      Le subject&#95;token&#95;type fourni dans la requête d’échange de jeton.
    </ResponseField>
  </Expandable>
</ResponseField>
