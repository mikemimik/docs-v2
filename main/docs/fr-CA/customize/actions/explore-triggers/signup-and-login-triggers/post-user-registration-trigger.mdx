Le déclencheur Post-user Registration s’exécute après qu’un utilisateur a été ajouté à une connexion Database ou <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip>.

<Frame>![Schéma du flux Actions Post User Registration.](/docs/images/cdy7uua7fh8z/4bqF9YEPYQshnJGCx39pws/079798c354f4a83355a756a6a02650d5/post-user-registration-flow.png)</Frame>

Les Actions de ce flux sont non bloquantes (asynchrones), ce qui signifie que le pipeline Auth0 poursuit son exécution sans attendre que l’Action ait terminé. Par conséquent, le résultat de l’Action n’a pas d’incidence sur la transaction Auth0.

<div id="triggers">
  ## Déclencheurs
</div>

<div id="post-user-registration">
  ### Post User Registration
</div>

Le déclencheur `post-user-registration` s’exécute après la création d’un utilisateur pour une connexion Database ou Passwordless. Ce déclencheur peut servir à aviser un autre système qu’un utilisateur s’est inscrit à votre application. Plusieurs actions peuvent y être associées, et elles s’exécuteront dans l’ordre. Toutefois, elles s’exécuteront de façon asynchrone et ne bloqueront pas le processus d’inscription de l’utilisateur.

  ## Cas d’utilisation courants

<div id="notify-slack-when-a-new-user-registers">
  ### Envoyer une notification à Slack lorsqu’un nouvel utilisateur s’inscrit
</div>

```javascript lines
/**
* Gestionnaire qui sera appelé lors de l'exécution d'un flux PostUserRegistration.
* 
 * @param {Event} event - Détails sur le contexte et l'utilisateur qui s'est inscrit.
 * @param {PostUserRegistrationAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de l'enregistrement d'un nouvel utilisateur.
 */

exports.onExecutePostUserRegistration = async (event, api) => {
  const { IncomingWebhook } = require("@slack/webhook");
  const webhook = new IncomingWebhook(event.secrets.SLACK_WEBHOOK_URL);

  const text = `New User: ${event.user.email}`;
  const channel = '#some_channel';

  webhook.send({ text, channel });
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour que cette Action s’exécute correctement, elle doit contenir un secret nommé `SLACK_WEBHOOK_URL` et avoir une dépendance sur le package `@slack/webhook` de `npm`.
</Callout>

<div id="store-the-auth0-user-id-in-a-remote-system">
  ### Stocker l’ID utilisateur d’Auth0 dans un système distant
</div>

Une Action post-user-registration peut servir à stocker l’ID utilisateur d’Auth0 dans un système distant.

```javascript lines
/**
* Handler qui sera appelé lors de l'exécution d'un flux PostUserRegistration.
* 
* @param {Event} event - Détails sur le contexte et l'utilisateur qui s'est inscrit.
* @param {PostUserRegistrationAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de l'inscription après enregistrement d'un utilisateur.
*/

const axios = require("axios");

exports.onExecutePostUserRegistration = async (event, api) => {
  await axios.post("https://my-api.exampleco.com/users", { params: { email: event.user.email }});
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour utiliser une bibliothèque `npm` comme `axios`, vous devez l’ajouter à l’Action en tant que dépendance. Pour en savoir plus, consultez la section &quot;Ajouter une dépendance&quot; dans [Écrivez votre première Action](/docs/fr-CA/customize/actions/write-your-first-action).
</Callout>

<div id="deny-access-to-specific-ja3ja4-fingerprints">
  ### Rejeter l’accès à certaines empreintes JA3/JA4
</div>

L’objet `event.security_context` contient les valeurs des empreintes JA3/JA4 pour la transaction en cours.

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const clientJa4 = event?.security_context?.ja4;
  console.log('[ACTION]', {clientJa4});
  const badFingerprints = ['t13d1517h2_8daaf6152771_b6f405a00624','t13d1516h2_8daaf6152771_d8a2da3f94cd'];
  if (clientJa4 && badFingerprints.includes(clientJa4)){
    api.access.deny('suspicious_tls_fingerprint', 'Your TLS fingerprint has been flagged as suspicious');
  }
};
```
