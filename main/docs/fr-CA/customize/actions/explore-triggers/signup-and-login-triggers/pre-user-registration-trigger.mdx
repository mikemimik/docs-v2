Le déclencheur Pre-user Registration s’exécute avant qu’un utilisateur soit ajouté à une connexion Database ou à une connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip>.

<Frame>![Diagramme illustrant le flux Pre User Registration d’Actions.](/docs/images/cdy7uua7fh8z/2KtUcZhaBcT12GxjhBJZFG/9633b4454ac1c06c0deed6c97e70fe7d/pre-user-registration-flow.png)</Frame>

Les Actions de ce flux sont bloquantes (synchrones), ce qui signifie qu’elles s’exécutent dans le cadre du processus d’un déclencheur et empêchent le reste du pipeline Auth0 de s’exécuter tant que l’Action n’est pas terminée.

<div id="triggers">
  ## Triggers
</div>

<div id="pre-user-registration">
  ### Pre-user Registration
</div>

Le déclencheur `pre-user-registration` s’exécute lorsqu’un utilisateur tente de s’inscrire au moyen d’une connexion Database ou Passwordless. Ce déclencheur peut servir à ajouter des métadonnées au profil de l’utilisateur avant sa création ou à rejeter une inscription au moyen d’une logique personnalisée.

<Warning>
  À l’heure actuelle, vous ne pouvez pas utiliser les Actions `pre-user-registration` pour ajouter des métadonnées aux utilisateurs Passwordless.
</Warning>

<div id="common-use-cases">
  ## Cas d’utilisation courants
</div>

<div id="deny-registration-by-location">
  ### Rejeter l’inscription en fonction de l’emplacement
</div>

Une Action de préinscription utilisateur peut être utilisée pour empêcher un utilisateur de s’inscrire.

```javascript lines
/**
 * @param {Event} event - Détails sur l'événement d'inscription.
 * @param {PreUserRegistrationAPI} api
 */
exports.onExecutePreUserRegistration = async (event, api) => {
  if (event.request.geoip.continentCode === "NA") {

    // localiser le message d'erreur 
    const LOCALIZED_MESSAGES = {
      en: 'You are not allowed to register.',
      es: 'No tienes permitido registrarte.'
    };

    const userMessage = LOCALIZED_MESSAGES[event.request.language] || LOCALIZED_MESSAGES['en'];
    api.access.deny('no_signups_from_north_america', userMessage);
  }
};
```

<div id="set-metadata-in-the-user-profile">
  ### Définir des métadonnées dans le profil utilisateur
</div>

Une Action de préinscription de l’utilisateur peut servir à ajouter des métadonnées au profil utilisateur avant sa création.

<Warning>
  Vous ne pouvez pas actuellement utiliser les Actions `pre-user-registration` pour ajouter des métadonnées aux utilisateurs passwordless.
</Warning>

```javascript lines
/**
 * @param {Event} event - Details about registration event.
 * @param {PreUserRegistrationAPI} api
 */
exports.onExecutePreUserRegistration = async (event, api) => {
  api.user.setUserMetadata("screen_name", "username");  
};
```

<div id="store-a-user-id-from-another-system-in-the-user-profile">
  ### Stocker l’ID d’un utilisateur d’un autre système dans le profil utilisateur
</div>

Une Action Pre-User Registration peut servir à stocker l’ID d’un utilisateur d’un autre système dans le profil utilisateur.

```javascript lines expandable
const axios = require('axios');

const REQUEST_TIMEOUT = 2000; // Exemple de délai d'attente

/**
* Handler qui sera appelé lors de l'exécution d'un flow PreUserRegistration.
*
* @param {Event} event - Détails sur le contexte et l'utilisateur qui tente de s'inscrire.
* @param {PreUserRegistrationAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de l'inscription.
*/
exports.onExecutePreUserRegistration = async (event, api) => {
  try {
    // Définir un secret USER_SERVICE_URL = 'https://yourservice.com'
    const remoteUser = await axios.get(event.secrets.USER_SERVICE_URL, {
      timeout: REQUEST_TIMEOUT,
      params: {
        email: event.user.email 
      }
    });

    if (remoteUser) {
      api.user.setAppMetadata('my-api-user-id', remoteUser.id); 
    }
  } catch (err) {
    api.validation.error('custom_error', 'Custom Error');
  }
};
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour utiliser une bibliothèque `npm` comme `axios`, vous devez l’ajouter comme dépendance de l’Action. Pour en savoir plus, consultez la section &quot;Ajouter une dépendance&quot; de [Écrire votre première Action](/docs/fr-CA/customize/actions/write-your-first-action).
</Callout>

<div id="deny-access-to-specific-ja3ja4-fingerprints">
  ### Rejeter l’accès pour certaines empreintes JA3/JA4
</div>

L’objet `event.security_context` contient les valeurs d’empreinte JA3/JA4 de la transaction en cours.

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const clientJa4 = event?.security_context?.ja4;
  console.log('[ACTION]', {clientJa4});
  const badFingerprints = ['t13d1517h2_8daaf6152771_b6f405a00624','t13d1516h2_8daaf6152771_d8a2da3f94cd'];
  if (clientJa4 && badFingerprints.includes(clientJa4)){
    api.access.deny('suspicious_tls_fingerprint', 'Your TLS fingerprint has been flagged as suspicious');
  }
};
```
