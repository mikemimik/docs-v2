L’objet API du déclencheur post-change-password d’Actions comprend :

<div id="apicache">
  ## `api.cache`
</div>

Stockez et récupérez des données qui sont conservées d’une exécution à l’autre.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Supprime l’enregistrement correspondant à la valeur mise en cache associée à la clé fournie, s’il existe.

Renvoie un objet `CacheWriteResult` avec `type: "success"` si une valeur a été supprimée du cache. En cas d’échec, l’opération renvoie `type: "error"`. En cas d’erreur, l’objet renvoyé comporte une propriété
`code` indiquant la nature de l’échec.

| Paramètre | Description                                                |
| --------- | ---------------------------------------------------------- |
| `key`     | *String*. La clé de l’enregistrement stocké dans le cache. |

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Récupère un enregistrement décrivant une valeur mise en cache à la
`key` fournie, s’il existe. Si un enregistrement est trouvé, la valeur mise en cache se
trouve dans la propriété `value` de l’objet retourné.

Retourne un enregistrement du cache si un élément est trouvé dans le cache pour la
`key` fournie. Les enregistrements du cache sont des objets avec une
propriété `value` contenant la valeur mise en cache, ainsi qu’une
propriété `expires_at` indiquant le délai d’expiration maximal de
l’enregistrement en millisecondes depuis l’époque Unix.

**Important :**
Ce cache est conçu pour des données éphémères de courte durée. Il se peut que des éléments ne soient pas
disponibles dans des transactions ultérieures, même s’ils sont encore dans leur
durée de validité.

| Paramètre | Description                                                |
| --------- | ---------------------------------------------------------- |
| `key`     | *String*. La clé de l’enregistrement stocké dans le cache. |

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, [options])`
</div>

Stocke ou met à jour une valeur de type chaîne dans le cache à la clé spécifiée.

Les valeurs stockées dans ce cache sont limitées au déclencheur dans lequel elles sont définies.
Elles sont assujetties aux
[limites du cache Actions](/docs/fr-CA/customize/actions/limitations).

Les valeurs stockées de cette façon ont une durée de vie pouvant aller jusqu’aux valeurs
`ttl` ou `expires_at` spécifiées. Si aucune durée de vie n’est
spécifiée, une durée de vie par défaut de 15 minutes sera utilisée. Les durées de vie ne peuvent
pas dépasser la durée maximale indiquée dans
[limites du cache Actions](/docs/fr-CA/customize/actions/limitations).

| Parameter            | Description                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `key`                | *String*. La clé de l’enregistrement stocké dans le cache.                                                                                                                                                                                                                                                                                                                                                                                    |
| `value`              | *String*. La valeur de l’enregistrement à stocker.                                                                                                                                                                                                                                                                                                                                                                                            |
| `options`            | *Optional object*. Options permettant d’ajuster le comportement du cache.                                                                                                                                                                                                                                                                                                                                                                     |
| `options.expires_at` | *Optional number*. L’heure d’expiration absolue en millisecondes depuis l’unix epoch. Bien que les enregistrements mis en cache puissent être évincés plus tôt, ils ne seront jamais conservés au-delà de la valeur `expires_at` fournie.<br />*Remarque :* Cette valeur ne doit pas être fournie si une valeur a également été fournie pour `ttl`. Si les deux options sont fournies, la plus rapprochée des deux expirations sera utilisée. |
| `options.ttl`        | *Optional number*. La durée de vie de cette entrée du cache, en millisecondes. Bien que les valeurs mises en cache puissent être évincées plus tôt, elles ne seront jamais conservées au-delà de la valeur `ttl` fournie.<br />*Remarque :* Cette valeur ne doit pas être fournie si une valeur a également été fournie pour `expires_at`. Si les deux options sont fournies, la plus rapprochée des deux expirations sera utilisée.          |
