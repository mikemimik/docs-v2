L’objet d’API du déclencheur send-phone-message d’Actions comprend :

<div id="apicache">
  ## `api.cache`
</div>

Stockez et récupérez des données qui sont conservées d’une exécution à l’autre.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Supprime un enregistrement décrivant une valeur en cache associée à la clé fournie, s’il existe.

Retourne un objet `CacheWriteResult` avec `type: "success"` si une valeur a été supprimée du cache. Une opération échouée retourne `type: "error"`. En cas d’erreur, l’objet retourné aura une propriété
`code` qui indique la nature de l’échec.

| Paramètre | Description                                                |
| --------- | ---------------------------------------------------------- |
| `key`     | *String*. La clé de l’enregistrement stocké dans le cache. |

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Récupère un enregistrement décrivant une valeur en cache à la
`key` fournie, s’il existe. Si un enregistrement est trouvé, la valeur en cache
se trouve dans la propriété `value` de l’objet retourné.

Retourne un enregistrement du cache si un élément est trouvé dans le cache pour la
`key` fournie. Les enregistrements du cache sont des objets avec une
propriété `value` contenant la valeur en cache, ainsi qu’une
propriété `expires_at` indiquant l’expiration maximale de
l’enregistrement en millisecondes depuis l’époque Unix.

**Important :**
Ce cache est conçu pour des données éphémères de courte durée. Il se peut que des éléments ne soient pas
disponibles dans des transactions ultérieures, même s’ils sont encore dans leur
durée de vie.

| Paramètre | Description                                                |
| --------- | ---------------------------------------------------------- |
| `key`     | *String*. La clé de l’enregistrement stocké dans le cache. |

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, [options])`
</div>

Stocke ou met à jour une valeur de type chaîne dans le cache à la clé spécifiée.

Les valeurs stockées dans ce cache sont limitées au déclencheur dans lequel elles sont définies.
Elles sont soumises aux
[limites du cache Actions](/docs/fr-CA/customize/actions/limitations).

Les valeurs stockées de cette façon auront une durée de vie pouvant aller jusqu’aux valeurs
`ttl` ou `expires_at` spécifiées. Si aucune durée de vie n’est
spécifiée, une durée de vie par défaut de 15 minutes sera utilisée. Les durées de vie ne peuvent
pas dépasser la durée maximale indiquée dans
[limites du cache Actions](/docs/fr-CA/customize/actions/limitations).

| Paramètre            | Description                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `key`                | *String*. La clé de l’enregistrement stocké dans le cache.                                                                                                                                                                                                                                                                                                                                                                                    |
| `value`              | *String*. La valeur de l’enregistrement à stocker.                                                                                                                                                                                                                                                                                                                                                                                            |
| `options`            | *Optional object*. Options permettant d’ajuster le comportement du cache.                                                                                                                                                                                                                                                                                                                                                                     |
| `options.expires_at` | *Optional number*. L’heure d’expiration absolue en millisecondes depuis l’époque Unix. Bien que les enregistrements mis en cache puissent être supprimés plus tôt, ils ne seront jamais conservés au-delà de la valeur `expires_at` fournie.<br />*Remarque :* Cette valeur ne doit pas être fournie si une valeur a aussi été fournie pour `ttl`. Si les deux options sont fournies, l’expiration la plus rapprochée des deux sera utilisée. |
| `options.ttl`        | *Optional number*. La durée de vie de cette entrée de cache en millisecondes. Bien que les valeurs mises en cache puissent être supprimées plus tôt, elles ne seront jamais conservées au-delà de la valeur `ttl` fournie.<br />*Remarque :* Cette valeur ne doit pas être fournie si une valeur a aussi été fournie pour `expires_at`. Si les deux options sont fournies, l’expiration la plus rapprochée des deux sera utilisée.            |
