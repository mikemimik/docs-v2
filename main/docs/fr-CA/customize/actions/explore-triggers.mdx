sidebarTitle: Aperçu
title: Explorer les déclencheurs
---

Les déclencheurs sont des points du processus Auth0 où des Actions peuvent être ajoutées afin d’apporter la personnalisation requise. Plusieurs Actions peuvent être placées dans un déclencheur et s’exécutent en séquence. À l’exception de l’échange de jeton personnalisé, les déclencheurs peuvent être synchrones (bloquants) ou asynchrones (non bloquants). Chaque déclencheur comporte des objets et des API précis. Ils peuvent varier selon votre connexion (par exemple, une connexion <Tooltip tip="Sans mot de passe : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip>).

Pour consulter les objets de référence et les API associés aux Actions d’un déclencheur donné, sélectionnez le déclencheur ci-dessous, en gardant à l’esprit la définition d’Auth0 d’une [connexion sans mot de passe](/docs/fr-CA/authenticate/passwordless).

<div id="signup-and-login">
  #### Inscription et connexion
</div>

Lorsqu’un utilisateur est ajouté à une connexion de base de données ou sans mot de passe (inscription), ou dans le cadre du processus d’authentification d’un utilisateur (connexion).

| Déclencheur              | Description                                                                               |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se déclenche avant la création d’un utilisateur.                                          |
| `post-user-registration` | Se déclenche de façon asynchrone après la création d’un utilisateur.                      |
| `post-login`             | Se déclenche après l’authentification d’un utilisateur, mais avant l’émission d’un jeton. |

<div id="mfa-notifications">
  #### Notifications MFA
</div>

Lors de l’utilisation de SMS comme facteur pour l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">authentification multifacteur</Tooltip> (MFA) ou pour configurer un Custom Provider.

| déclencheur              | Description                                                                                                                           |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se déclenche lorsque vous utilisez un Custom Provider pour envoyer les messages liés à l’inscription et au processus de vérification. |

<div id="password-reset">
  #### Réinitialisation du mot de passe
</div>

Lorsqu’une demande de réinitialisation du mot de passe est soumise.

| Déclencheur                     | Description                                                                               |
| ------------------------------- | ----------------------------------------------------------------------------------------- |
| `post-change-password`          | Se déclenche après qu’un mot de passe a été modifié.                                      |
| `password-reset-post-challenge` | Se déclenche après la première vérification et avant la réinitialisation du mot de passe. |

<div id="machine-to-machine">
  #### Machine to Machine
</div>

Lorsqu’une application, plutôt qu’un utilisateur, est authentifiée et autorisée à utiliser vos API.

| déclencheur                | Description                                        |
| ---------------------- | -------------------------------------------------- |
| `credentials-exchange` | Se déclenche avant qu’un jeton d’accès soit retourné. |

<div id="custom-token-exchange">
  ### Échange de jeton personnalisé
</div>

Lorsqu’une requête d’échange de jeton personnalisé est reçue pour authentifier un utilisateur.

| Déclencheur             | Description                                                                            |
| ----------------------- | -------------------------------------------------------------------------------------- |
| `custom-token-exchange` | Se déclenche dès la première étape de la transaction, avant le déclencheur post-login. |
