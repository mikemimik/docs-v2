title: Explorer les déclencheurs
---

Les déclencheurs sont des points du processus Auth0 où des Actions peuvent être ajoutées pour répondre à des besoins de personnalisation. Plusieurs Actions peuvent être placées dans un déclencheur et s’exécutent en séquence. À l’exception de l’échange de jeton personnalisé, les déclencheurs peuvent être synchrones (bloquants) ou asynchrones (non bloquants). Chaque déclencheur possède des objets et des API qui lui sont propres. Ils peuvent varier selon votre connexion (par exemple, une connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip>).

Pour consulter les objets de référence et les API des Actions associées à un déclencheur précis, sélectionnez le déclencheur ci-dessous et prenez connaissance de la définition d’Auth0 d’une [connexion sans mot de passe](/docs/fr-CA/authenticate/passwordless).

<div id="signup-and-login">
  #### Inscription et connexion
</div>

Lorsqu’un utilisateur est ajouté à une connexion de base de données ou sans mot de passe (inscription), ou pendant la transaction d’authentification d’un utilisateur (connexion).

| Déclencheur              | Description                                                                               |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se déclenche avant la création d’un utilisateur.                                          |
| `post-user-registration` | Se déclenche de façon asynchrone après la création d’un utilisateur.                      |
| `post-login`             | Se déclenche après l’authentification d’un utilisateur, mais avant l’émission d’un jeton. |

<div id="mfa-notifications">
  #### Notifications MFA
</div>

Lorsqu’on utilise le SMS comme facteur pour l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">authentification multifacteur</Tooltip> (MFA) ou pour configurer un fournisseur personnalisé.

| Déclencheur          | Description                                                                                                                                 |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se déclenche lorsqu’un fournisseur personnalisé est utilisé pour envoyer les messages liés à l’inscription et au processus de vérification. |

<div id="password-reset">
  #### Réinitialisation du mot de passe
</div>

Lorsqu’une demande de réinitialisation du mot de passe est lancée.

| Déclencheur                     | Description                                                                                           |
| ------------------------------- | ----------------------------------------------------------------------------------------------------- |
| `post-change-password`          | Se déclenche après qu’un mot de passe a été modifié.                                                  |
| `password-reset-post-challenge` | Se déclenche une fois la première vérification terminée et avant la réinitialisation du mot de passe. |

<div id="machine-to-machine">
  #### Machine to Machine
</div>

Lorsqu’une application, plutôt qu’un utilisateur, est authentifiée et autorisée à utiliser vos API.

| Trigger                | Description                                             |
| ---------------------- | ------------------------------------------------------- |
| `credentials-exchange` | Se déclenche avant qu’un jeton d’accès ne soit renvoyé. |

<div id="custom-token-exchange">
  ### Échange de jeton personnalisé
</div>

Lorsqu’une requête d’échange de jeton personnalisé est reçue pour connecter un utilisateur.

| Déclencheur             | Description                                                                       |
| ----------------------- | --------------------------------------------------------------------------------- |
| `custom-token-exchange` | Se déclenche à la première étape de la transaction, avant le déclencheur post-login. |
