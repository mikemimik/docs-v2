sidebarTitle: Vue d’ensemble
title: Explorer les déclencheurs
---

Les déclencheurs sont des points du processus Auth0 où des Actions peuvent être ajoutées pour répondre à des besoins de personnalisation. Plusieurs Actions peuvent être placées dans un déclencheur et s’exécuter en séquence. À l’exception de l’Échange de jeton personnalisé, les déclencheurs peuvent être synchrones (bloquants) ou asynchrones (non bloquants). Chaque déclencheur dispose de ses propres objets et API. Ils peuvent varier selon votre connexion (par exemple, une connexion <Tooltip tip="Sans mot de passe : forme d'authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=passwordless">sans mot de passe</Tooltip>).

Pour consulter les objets de référence et les API propres aux Actions d’un déclencheur donné, sélectionnez le déclencheur ci-dessous, en gardant à l’esprit la définition qu’Auth0 donne d’une [connexion sans mot de passe](/docs/fr-CA/authenticate/passwordless).

<div id="signup-and-login">
  #### Inscription et connexion
</div>

Lorsqu’un utilisateur est ajouté à une connexion de base de données ou à une connexion sans mot de passe (inscription), ou dans le cadre de la transaction d’authentification d’un utilisateur (connexion).

| Déclencheur              | Description                                                                               |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se déclenche avant la création d’un utilisateur.                                          |
| `post-user-registration` | Se déclenche de façon asynchrone après la création d’un utilisateur.                      |
| `post-login`             | Se déclenche après l’authentification d’un utilisateur, mais avant l’émission d’un jeton. |

<div id="mfa-notifications">
  #### Notifications MFA
</div>

Lors de l’utilisation du SMS comme facteur pour l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Multi-Factor+Authentication">authentification multifacteur</Tooltip> (MFA) ou pour configurer un fournisseur personnalisé.

| Déclencheur          | Description                                                                                                            |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se déclenche lorsqu’un fournisseur personnalisé est utilisé pour envoyer les messages d’enrôlement et de vérification. |

<div id="password-reset">
  #### Réinitialisation du mot de passe
</div>

Lorsqu’une demande de réinitialisation du mot de passe est amorcée.

| Déclencheur                     | Description                                                                                         |
| ------------------------------- | --------------------------------------------------------------------------------------------------- |
| `post-change-password`          | Se déclenche après qu’un mot de passe a été modifié.                                                |
| `password-reset-post-challenge` | Se déclenche une fois la première vérification terminée, avant la réinitialisation du mot de passe. |

<div id="machine-to-machine">
  #### Machine to Machine
</div>

Quand une application, et non un utilisateur, est authentifiée et autorisée à utiliser vos API.

| Déclencheur            | Description                                         |
| ---------------------- | --------------------------------------------------- |
| `credentials-exchange` | Se déclenche avant qu’un jeton d’accès soit renvoyé. |

<div id="custom-token-exchange">
  ### Échange de jeton personnalisé
</div>

Lorsqu’une demande d’échange de jeton personnalisé est reçue pour ouvrir une session utilisateur.

| Déclencheur             | Description                                                                          |
| ----------------------- | ------------------------------------------------------------------------------------ |
| `custom-token-exchange` | Se déclenche à la première étape de la transaction, avant le déclencheur post-login. |
