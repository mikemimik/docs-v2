---
description: Limites propres aux activités de migration lors du passage de Rules et Hooks vers Actions
title: Limites de migration vers Actions
---

Pour tirer parti des fonctionnalités offertes par Auth0 Actions, vous devriez migrer votre code actuel à partir de Rules, de Hooks ou de versions antérieures d’Actions. Toutefois, vous devez connaître les limites actuelles de la migration entre les anciennes fonctionnalités d’extensibilité et Actions.

<div id="general-differences">
  ## Différences générales
</div>

* Actions ne peuvent retourner que des codes d’erreur `access denied` et ne prennent pas en charge le renvoi d’un code d’erreur `unauthorized`.
* Les claims standard d’un `accessToken` ne peuvent pas être supprimés au moyen d’une Action.

<div id="rule-specific-differences">
  ## Différences propres aux Rules
</div>

* L’objet d’événement des Actions Post Login n’offre actuellement aucun équivalent à `context.connectionOptions`, disponible dans les Rules.
* L’objet d’événement des Actions Post Login n’offre actuellement aucun équivalent pour accéder, au moyen de l’objet `user.identities`, à un token provenant d’un IdP en amont pour les utilisateurs fédérés, comme c’est le cas dans les Rules.
* Les claims standard d’un `idToken` ne peuvent pas être supprimées au moyen d’une Action.
* Les Actions prennent uniquement en charge la modification de l’utilisateur principal parmi les comptes liés.
