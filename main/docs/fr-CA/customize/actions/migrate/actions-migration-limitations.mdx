---
description: Limitations propres aux activités de migration lors du passage de Rules et Hooks vers Actions
title: Limites de la migration vers Actions
---

Pour tirer parti des fonctionnalités offertes par Auth0 Actions, vous devriez migrer votre code actuel à partir de Rules, de Hooks ou de versions antérieures d’Actions. Toutefois, vous devez connaître les limites actuelles de la migration entre les anciennes offres d’extensibilité et Actions.

<div id="general-differences">
  ## Différences générales
</div>

* Actions ne peut retourner que des codes d’erreur `access denied` et ne permet pas de retourner un code d’erreur `unauthorized`.
* Les claims standard d’un `accessToken` ne peuvent pas être supprimés au moyen d’une Action.

<div id="rule-specific-differences">
  ## Différences propres aux Rules
</div>

* L’objet Event dans les Actions Post Login ne fournit actuellement pas d’équivalent à `context.connectionOptions`, qui est disponible dans les Rules.
* L’objet Event dans les Actions Post Login ne fournit actuellement pas d’équivalent permettant d’accéder, via l’objet `user.identities`, à un token provenant d’un IdP en amont pour les utilisateurs fédérés, comme c’est possible dans les Rules.
* Les claims standard d’un `idToken` ne peuvent pas être supprimées à l’aide d’une Action.
* Les Actions permettent uniquement de modifier l’utilisateur principal parmi des comptes liés.
