---
description: Limites propres aux activités de migration lors du passage de Rules et Hooks vers Actions
title: Limites de la migration vers Actions
---

Pour tirer parti des fonctionnalités offertes par Auth0 Actions, vous devriez migrer votre code actuel depuis Rules, Hooks ou des versions antérieures d’Actions. Toutefois, vous devez connaître les limites actuelles liées à la migration entre les offres d’extensibilité héritées et Actions.

<div id="general-differences">
  ## Différences générales
</div>

* Les Actions ne peuvent retourner que des codes d’erreur `access denied` et ne prennent pas en charge le renvoi d’un code d’erreur `unauthorized`.
* Les claims standard d’un `accessToken` ne peuvent pas être supprimés à l’aide d’une Action.

<div id="rule-specific-differences">
  ## Différences propres aux Rules
</div>

* L’objet Event dans les Actions Post Login ne fournit actuellement aucune alternative à `context.connectionOptions`, disponible dans les Rules.
* L’objet Event dans les Actions Post Login ne fournit actuellement aucune alternative permettant d’accéder, via l’objet `user.identities`, aux jetons de l’IdP en amont pour les utilisateurs fédérés, comme c’est le cas dans les Rules.
* Les claims standard d’un `idToken` ne peuvent pas être supprimées à l’aide d’une Action.
* Les Actions prennent uniquement en charge la modification de l’utilisateur principal entre des comptes liés.
