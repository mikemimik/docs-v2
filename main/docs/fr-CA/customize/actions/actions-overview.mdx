Les Actions sont des fonctions sécurisées, propres au tenant et [versionnées](/docs/fr-CA/customize/actions/manage-versions), écrites en Node.js, qui s’exécutent à certains points de la plateforme Auth0. Les Actions servent à personnaliser et à étendre les capacités d’Auth0 à l’aide d’une logique personnalisée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 Actions utilise le [modèle de programmation basé sur les promesses](https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Async_JS/Promises) de JavaScript moderne, qui sert de fondement aux fonctions asynchrones dans Actions.
</Callout>

Les processus qui peuvent être étendus de cette façon sont appelés des déclencheurs. Les regroupements de déclencheurs représentent le pipeline logique par lequel l’information circule à une étape précise du parcours Auth0. Plusieurs Actions peuvent être ajoutées à un déclencheur, chaque Action s’exécutant dans l’ordre où elle a été placée. Certains déclencheurs s’exécutent de façon synchrone, ce qui bloque le flux dans lequel ils interviennent, tandis que d’autres s’exécutent de façon asynchrone, comme l’indique le tableau ci-dessous.

Les Actions sont un élément clé de notre offre globale d’extensibilité chez Auth0. Avec Actions, vous pouvez ajouter la logique personnalisée essentielle à vos flux de connexion et d’identité, selon vos besoins. Les Actions vous permettent aussi de connecter des intégrations externes qui enrichissent votre expérience globale d’extensibilité. Par exemple, vous pouvez ajouter une Action à votre déclencheur de connexion pour vérifier des pièces justificatives, comme un permis de conduire ou un passeport, au moyen d’un partenaire du Marketplace spécialisé dans la vérification d’identité.

<div id="what-can-you-do-with-actions">
  ## Que pouvez-vous faire avec Actions?
</div>

Ce qu’une Action peut faire dépend de l’endroit où elle s’exécute dans l’environnement d’exécution d’Auth0.

<AccordionGroup>
  <Accordion title="Inscription et connexion : lorsqu’un utilisateur est ajouté à une connexion de base de données ou Passwordless.">
    | Déclencheur                  | Description                                                                                                         |
    | ------------------------ | ------------------------------------------------------------------------------------------------------------------- |
    | `pre-user-registration`  | Se déclenche avant la création d’un utilisateur. Ne s’exécute pas pour les connexions sociales.                     |
    | `post-user-registration` | Se déclenche de façon asynchrone après la création d’un utilisateur. Ne s’exécute pas pour les connexions sociales. |
    | `post-login`             | Se déclenche après l’authentification d’un utilisateur, mais avant qu’un jeton ne soit émis.                        |
  </Accordion>

  <Accordion title="Notifications MFA : lors de l’utilisation des SMS comme facteur pour l’authentification multifacteur (MFA) ou pour configurer un fournisseur personnalisé.">
    | Déclencheur              | Description                                                                                                                                                                        |
    | -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `send-phone-message` | Se déclenche lorsqu’un fournisseur personnalisé est utilisé pour envoyer les messages d’enrôlement et du processus de vérification. Ne s’exécute pas pour les connexions sociales. |
  </Accordion>

  <Accordion title="Réinitialisation du mot de passe : lorsqu’une demande de réinitialisation du mot de passe est lancée.">
    | Déclencheur                         | Description                                                                                           |
    | ------------------------------- | ----------------------------------------------------------------------------------------------------- |
    | `post-change-password`          | Se déclenche après la modification d’un mot de passe. Ne s’exécute pas pour les connexions sociales.  |
    | `password-reset-post-challenge` | Se déclenche une fois la première vérification terminée et avant la réinitialisation du mot de passe. |
  </Accordion>

  <Accordion title="Machine to Machine : lorsqu’une application, plutôt qu’un utilisateur, est authentifiée et autorisée à utiliser vos API.">
    | Déclencheur                | Description                                      |
    | ---------------------- | ------------------------------------------------ |
    | `credentials-exchange` | Se déclenche avant le renvoi d’un jeton d’accès. |
  </Accordion>
</AccordionGroup>

<div id="key-benefits-of-actions">
  ## Principaux avantages des Actions
</div>

<div id="improved-developer-experience">
  ### Expérience développeur améliorée
</div>

* Lors de la modification d’une Action dans le <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vous aurez accès à des informations de typage détaillées et à de la documentation intégrée sur les possibilités offertes par chaque déclencheur, ce qui permet de découvrir facilement les fonctionnalités prises en charge par chaque déclencheur.
* Une Action peut être modifiée et testée sans affecter la version qui gère actuellement le trafic de production.
* Si un problème est détecté dans une Action, il est possible de revenir à une version précédente.

<div id="reuse-custom-functions">
  ### Réutiliser des fonctions personnalisées
</div>

* Créez, gérez et partagez des fonctions d’une action à l’autre à l’aide des [Actions Modules](/docs/fr-CA/customize/actions/modules/actions-modules-overview).

<div id="access-to-npm-packages">
  ### Accès aux paquets npm
</div>

* Vous pouvez utiliser la quasi-totalité des [paquets `npm` publics](https://www.npmjs.com/) dans Actions.

<div id="observability">
  ### Observabilité
</div>

* Lorsque Auth0 exécute des Actions, il recueille des indicateurs clés à leur sujet et les associe aux [journaux Auth0](/docs/fr-CA/deploy-monitor/logs).
* Pour le débogage, utilisez la fonctionnalité [Journaux en temps réel d’Actions](/docs/fr-CA/customize/actions/actions-real-time-logs).

<div id="multiple-actions-on-every-trigger">
  ### Plusieurs Actions pour chaque déclencheur
</div>

* Chaque déclencheur d’Action prend en charge plusieurs Actions indépendantes.
