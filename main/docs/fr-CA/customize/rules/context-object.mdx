<Warning>
  La date de fin de vie (EOL) de Rules et Hooks est le **18 novembre 2026**, et ils ne sont plus offerts aux nouveaux tenants créés à compter du **16 octobre 2023**. Les tenants existants avec des Hooks actifs conserveront l&#39;accès au produit Hooks jusqu&#39;à leur fin de vie.

  Nous vous recommandons fortement d&#39;utiliser Actions pour étendre Auth0. Avec Actions, vous avez accès à des informations de type enrichies, à de la documentation intégrée et à des packages `npm` publics, et vous pouvez connecter des intégrations externes qui améliorent votre expérience globale d&#39;extensibilité. Pour en savoir plus sur ce qu&#39;offrent Actions, consultez [Comprendre le fonctionnement des Auth0 Actions](/docs/fr-CA/customize/actions/actions-overview).

  Pour vous aider dans votre migration, nous proposons des guides pour [migrer de Rules vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions) et [migrer de Hooks vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions). Nous avons également une page dédiée, [Move to Actions](https://auth0.com/extensibility/movetoactions), qui présente des comparaisons de fonctionnalités, [une démo d&#39;Actions](https://www.youtube.com/watch?v=UesFSY1klrI) et d&#39;autres ressources pour vous accompagner dans votre migration.

  Pour en savoir plus sur la dépréciation de Rules et Hooks, consultez notre billet de blogue : [Préparer la fin de vie de Rules et Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

L&#39;objet `context` stocke des renseignements contextuels sur la transaction d&#39;authentification en cours, comme l&#39;adresse IP de l&#39;utilisateur, l&#39;application ou l&#39;emplacement.

Si vous modifiez le contenu des jetons à l&#39;aide de l&#39;objet context dans une rule, vos modifications seront reflétées dans les jetons une fois l&#39;exécution de toutes les rules terminée. Si votre application exige aussi <Tooltip tip="Authentification multifacteur (MFA) : processus d'authentification de l'utilisateur qui utilise un facteur en plus du nom d'utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multifactor+authentication">l&#39;authentification multifacteur</Tooltip> ou le consentement de l&#39;utilisateur, l&#39;utilisateur devra d&#39;abord intervenir avant que les modifications apportées au jeton soient disponibles.

| `context.request`            | Un objet contenant des informations utiles sur la requête. Il peut aussi avoir la valeur `undefined`. Il comprend les propriétés suivantes :<ul> <li>`userAgent` : le user-agent de l’application qui tente de se connecter.</li> <li>`ip` : l’adresse IP d’origine de l’utilisateur qui tente de se connecter. Nous exposons des adresses IPv6 sur nos points de terminaison publics (par exemple, travel0.us.auth0.com). Si une requête provient d’une machine compatible avec IPv6, cette valeur contiendra une adresse IPv6. Si vous manipulez manuellement des adresses IP, nous vous suggérons d’utiliser la [bibliothèque ipaddr.js@1.9.0](https://www.npmjs.com/package/ipaddr.js/v/1.9.0).</li> <li>`hostname` : le nom d’hôte utilisé pour le flux d’authentification.</li> <li>`query` : un objet contenant les propriétés de la chaîne de requête de la transaction de connexion envoyée par l’application.</li> <li>`body` : le corps de la requête POST dans les transactions de connexion utilisées avec les protocoles `oauth2-resource-owner`, `oauth2-resource-owner-jwt-bearer` ou `wstrust-usernamemixed`.</li> <li>`geoip` : un objet contenant des informations géographiques liées à l’adresse IP. Il comprend les propriétés suivantes :<ul> <li>`country_code` : un code à deux caractères pour le pays associé à l’adresse IP.</li> <li>`country_code3` : un code à trois caractères pour le pays associé à l’adresse IP.</li> <li>`country_name` : le nom du pays associé à l’adresse IP.</li> <li>`city_name` : le nom de la ville ou de la municipalité associé à l’adresse IP.</li> <li>`latitude` : la latitude associée à l’adresse IP.</li> <li>`longitude` : la longitude associée à l’adresse IP.</li> <li>`time_zone` : le fuseau horaire associé à l’adresse IP.</li> <li>`continent_code` : un code à deux caractères pour le continent associé à l’adresse IP.</li> <li>`subdivision_code` : le code ISO 3166-2 de la subdivision/région de premier niveau.</li> <li>`subdivision_name` : le nom anglais de cette subdivision/région.</li></ul></li></ul> |
| `context.primaryUser`        | L’ID utilisateur unique du compte principal de l’utilisateur. Sert à [lier des comptes d’utilisateur](/docs/fr-CA/manage-users/user-accounts/user-account-linking#automatic-account-linking) provenant de divers fournisseurs d’identité.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| `context.authentication`     | Un objet contenant des informations liées à la transaction d’authentification, avec les propriétés suivantes :<br /><br />`methods`: un tableau d’objets contenant les méthodes d’authentification qu’un utilisateur a effectuées pendant sa session. Par exemple, un utilisateur qui a effectué une authentification par mot de passe suivie d’une MFA peut avoir les méthodes suivantes :<br />`json lines [ { "name": "pwd", "timestamp": 1434454643024 }, { "name": "mfa", "timestamp": 1534454643881 } ] `<br />Les objets de méthode contiennent les propriétés suivantes : <ul> <li> `name`: une chaîne représentant le nom de la méthode d’authentification utilisée. Elle peut prendre l’une des valeurs suivantes (d’autres valeurs pourraient être prises en charge à l’avenir) :<br /><ul> <li>`federated`: une connexion sociale ou d’entreprise a été utilisée pour authentifier l’utilisateur</li> <li>`pwd`: une connexion de base de données a été utilisée pour authentifier l’utilisateur</li> <li>`sms`: une connexion SMS [Passwordless](/docs/fr-CA/glossary?term=Passwordless) a été utilisée pour authentifier l’utilisateur</li> <li>`email`: une connexion Passwordless Email a été utilisée pour authentifier l’utilisateur</li> <li>`mfa`: l’utilisateur a effectué une authentification multifacteur</li> </ul> </li> <li>`timestamp`: un entier indiquant l’heure, en secondes, à laquelle la méthode d’authentification a eu lieu, selon l’époque Unix</li> </ul> Vous pouvez consulter un exemple de cas d’utilisation de la propriété `context.authentication.methods` dans la [règle Require MFA once per session](https://github.com/auth0/rules/blob/master/src/rules/require-mfa-once-per-session.js).                                                                                                                                                                                                                                                                                                                                                            |
| `context.authorization`      | Un objet contenant des renseignements liés à la transaction d’autorisation avec les propriétés suivantes :<ul> <li>`roles` : un tableau de chaînes contenant les noms des [rôles](/docs/fr-CA/glossary?term=roles) attribués à un utilisateur. Vous pouvez consulter un exemple de cas d’utilisation qui utilise la propriété `context.authorization.roles` pour ajouter des rôles aux jetons dans [Exemples de cas d’utilisation : Rules avec l’autorisation](/docs/fr-CA/manage-users/access-control/sample-use-cases-rules-with-authorization#add-user-roles-to-tokens).</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `context.organization`       | Objet contenant des renseignements sur l’organisation. Comprend les propriétés suivantes :<ul> <li>`id` : String contenant l’ID de l’organisation avec laquelle l’utilisateur ouvre une session.</li><li>`name` : String contenant le nom de l’organisation (tel que défini dans le Auth0 Dashboard).</li><li>`metadata` : Dictionnaire de paires clé-valeur de type String contenant d’autres propriétés de l’organisation.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
