  Nous vous recommandons fortement d&#39;utiliser Actions pour étendre Auth0. Avec Actions, vous avez accès à des informations de type détaillées, à de la documentation intégrée et à des packages `npm` publics, et vous pouvez connecter des intégrations externes qui améliorent votre expérience globale d&#39;extensibilité. Pour en savoir plus sur ce qu&#39;offrent les Actions, consultez [Comprendre le fonctionnement des Auth0 Actions](/docs/fr-CA/customize/actions/actions-overview).

  Pour vous aider dans votre migration, nous mettons à votre disposition des guides qui vous aideront à [migrer des Rules vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions) et à [migrer des Hooks vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions). Nous avons également une page dédiée [Move to Actions](https://auth0.com/platform/extensibility/movetoactions) qui présente des comparaisons de fonctionnalités, [une démonstration d&#39;Actions](https://www.youtube.com/watch?v=UesFSY1klrI) et d&#39;autres ressources pour vous accompagner tout au long de votre migration.

  Pour en savoir plus sur la dépréciation des Rules et des Hooks, consultez notre billet de blogue : [Préparer la fin de vie des Rules et des Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Comme nous prévoyons retirer les fonctions Rules et Hooks en 2026, vous ne devriez créer de nouveaux Rules ou Hooks que dans votre environnement de développement, et uniquement pour tester la migration vers Actions.

  Pour savoir comment migrer vos Rules vers Actions, consultez [Migrate from Rules to Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions). Pour savoir comment migrer vos Hooks vers Actions, consultez [Migrate from Hooks to Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions).

Vous pouvez créer vos propres règles pour répondre à vos besoins fonctionnels précis. Vous pouvez modifier un modèle de règle existant ou choisir de partir de zéro à l&#39;aide de l&#39;un de nos exemples. Auth0 fournit un certain nombre de règles et de modèles de règle pour vous aider à atteindre vos objectifs. Pour en voir la liste, consultez notre[dépôt GitHub des rules](https://github.com/auth0/rules).

<div id="how-rules-work">
  ## Fonctionnement des Rules
</div>

Les Rules sont des fonctions JavaScript qui s’exécutent lorsqu’un utilisateur s’authentifie auprès de votre application. Elles s’exécutent une fois le processus d’authentification terminé, et vous pouvez les utiliser pour personnaliser et étendre les capacités d’Auth0. Pour des raisons de sécurité, le code de vos Rules s’exécute de manière isolée du code des autres tenants Auth0 dans un sandbox. Les Rules s’exécutent également pendant le flux d’actualisation des jetons. Pour en savoir plus, consultez [Refresh Tokens](/docs/fr-CA/secure/tokens/refresh-tokens).

Dans Auth0, le flux de la transaction d’authentification fonctionne comme suit lorsque vous utilisez des Rules :

<Frame>![Diagramme des Rules dans le flux d’authentification](/docs/images/cdy7uua7fh8z/2gtBtkPChIyguA24x6enx2/ffbb8e21e86920ef9914f6fc126dc1df/flow.png)</Frame>

1. Une application envoie une requête d’authentification à Auth0.
2. Auth0 achemine la requête vers un fournisseur d’identité au moyen d’une connexion configurée.
3. L’utilisateur s’authentifie avec succès.
4. Le jeton d’ID et/ou le jeton d’accès passent par le pipeline des Rules, puis sont envoyés à l’application.

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

Effectuez une requête `POST` au [point de terminaison Create Rule](https://auth0.com/docs/api/management/v2#!/Rules/post_rules). Veillez à remplacer les valeurs de remplacement `MGMT_API_ACCESS_TOKEN`, `RULE_NAME`, `RULE_SCRIPT`, `RULE_ORDER` et `RULE_ENABLED` par votre <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip>, le nom de la règle, le script de la règle, son numéro d’ordre et sa valeur d’activation, respectivement.

| Valeur                    | Description                                                                                                                                                                                                                                                                                   |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN`   | Jeton d’accès pour la Management API avec la portée `create:rules`.                                                                                                                                                                                                                           |
| `RULE_NAME`               | Nom de la règle que vous souhaitez créer. Le nom de la règle ne peut contenir que des caractères alphanumériques, des espaces et des traits d’union; il ne peut pas commencer ou se terminer par des espaces ou des traits d’union.                                                           |
| `RULE_SCRIPT`             | Script qui contient le code de la règle. Il doit correspondre à ce que vous saisiriez si vous créiez une nouvelle règle dans le Dashboard.                                                                                                                                                    |
| `RULE_ORDER` (optional)   | Entier qui représente l’ordre dans lequel la règle doit être exécutée par rapport aux autres règles. Les règles associées à des nombres plus petits sont exécutées avant celles associées à des nombres plus grands. Si aucun numéro d’ordre n’est fourni, la règle sera exécutée en dernier. |
| `RULE_ENABLED` (optional) | Booléen qui indique si la règle est activée (`true`) ou désactivée (`false`).                                                                                                                                                                                                                 |

<Warning>
  Nous exposons des adresses IPv6 dans nos points de terminaison publics (p. ex., `travel0.us.auth0.com`). Si une requête provient d’une machine qui prend en charge IPv6, la propriété `context.request.ip` contiendra une adresse IPv6. Si vous manipulez manuellement des adresses IP, nous vous suggérons d’utiliser la [bibliothèque ipaddr.js@1.9.0](https://www.npmjs.com/package/ipaddr.js/v/1.9.0).
</Warning>

<div id="manage-rate-limits">
  ## Gérer les limites de débit
</div>

Pour les règles qui appellent les API Auth0, vous devriez toujours tenir compte de la limitation du débit en vérifiant l’en-tête `X-RateLimit-Remaining` et en réagissant de façon appropriée lorsque le nombre renvoyé approche de 0. Vous devriez aussi prévoir une logique pour les cas où vous dépassez les limites de débit accordées et recevez le code d’état HTTP 429 (Trop de requêtes) ; dans ce cas, si une nouvelle tentative est nécessaire, il vaut mieux prévoir un délai progressif afin d’éviter une boucle infinie de tentatives. Pour en savoir plus sur les limites de débit, consultez [Rate Limit Policy For Auth0 APIs](/docs/fr-CA/troubleshoot/customer-support/operational-policies/rate-limit-policy).

<div id="available-modules">
  ## Modules disponibles
</div>

Les Rules s’exécutent dans un sandbox JavaScript configuré pour une version précise de Node.js.

Le sandbox prend en charge toutes les versions du langage JavaScript (et la syntaxe associée) pour la version de Node.js configurée, ainsi qu’un grand nombre de modules Node.js. Pour consulter la liste des modules de sandbox pris en charge, consultez [Can I require: Auth0 Extensibility](https://auth0-extensions.github.io/canirequire/).
