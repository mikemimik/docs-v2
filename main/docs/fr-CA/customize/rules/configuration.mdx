L&#39;objet global `configuration` est disponible dans Rules pour stocker les valeurs couramment utilisées, comme des URL. Les renseignements sensibles, comme les identifiants ou les clés API, doivent être stockés dans l&#39;objet `configuration` et non dans votre code Rules.

<div id="configure-values">
  ## Configurer des valeurs
</div>

Vous pouvez définir des valeurs de configuration dans vos [paramètres des Rules](https://manage.auth0.com/#/rules/) du Dashboard.

Pour modifier la valeur d&#39;une clé de configuration, supprimez le paramètre de configuration existant et remplacez-le par la nouvelle valeur. Vous devez avoir créé au moins une Rule pour voir la zone de configuration; sinon, la démo des Rules s&#39;affiche à la place.

<div id="use-the-configuration-object">
  ## Utiliser l’objet de configuration
</div>

Vous pouvez accéder à toutes les valeurs de configuration que vous définissez par clé au moyen de l’objet `configuration` dans votre code Rules.

```javascript lines
var MY_API_KEY = configuration.MY_API_KEY;
```

L’exemple suivant montre une Rule qui envoie un message Slack lorsqu’un nouvel utilisateur s’inscrit. Le [webhook Slack](https://api.slack.com/incoming-webhooks) correspond à une valeur de `configuration` définie à l’aide de la clé `SLACK_HOOK_URL`.

```javascript lines
function (user, context, callback) {
  // court-circuiter si l'utilisateur s'est déjà inscrit ou utilise un Refresh Token
  if (context.stats.loginsCount > 1 || context.protocol === 'oauth2-refresh-token') {
    return callback(null, user, context);
  }

  // obtenez l'URL du hook Slack ici : https://slack.com/services/10525858050
  const SLACK_HOOK = configuration.SLACK_HOOK_URL;

  const slack = require('slack-notify')(SLACK_HOOK);
  const message = 'New User: ' + (user.name || user.email) + ' (' + user.email + ')';
  const channel = '#some_channel';

  slack.success({
    text: message,
    channel: channel
  });

  // ne pas attendre la fin de l'appel d'API Slack, retourner immédiatement (la requête continuera dans le sandbox)`
  callback(null, user, context);
}
```
