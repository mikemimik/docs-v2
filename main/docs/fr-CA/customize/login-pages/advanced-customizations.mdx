**Advanced Customizations for Universal Login (ACUL)** étend les capacités de [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience#universal-login-experience) pour prendre en charge la gestion de plusieurs images de marque, des configurations de sécurité complexes et l’intégration de nombreuses plateformes d’analyse.

<video autoPlay muted loop playsInline className="w-full aspect-video rounded-xl" src="/docs/images/videos/ACUL.mp4" />

Avec ACUL, vous pouvez écrire du code dans le framework de votre choix, intégrer vos plateformes d’analyse et de tests A/B existantes, et définir l’expérience utilisateur. Les [ACUL SDKs](/docs/fr-CA/customize/login-pages/advanced-customizations/reference) font le lien entre votre code personnalisé et Universal Login.

<div id="benefits">
  ## Avantages
</div>

* Étendez les capacités d’Universal Login d’Auth0 pour prendre en charge plusieurs images de marque dans un même tenant.
* Profitez des fonctionnalités d’Auth0, comme [Passkeys](/docs/fr-CA/authenticate/database-connections/passkeys), [Bot Detection](/docs/fr-CA/secure/attack-protection/bot-detection) et [MFA](/docs/fr-CA/secure/multi-factor-authentication), sans avoir à reconstruire votre logique d’authentification.
* Intégrez des outils d’analyse avancés et des tests A/B pour optimiser le parcours utilisateur.
* Utilisez les ACUL SDKs pour développer avec votre propre pile technologique. Vous pouvez :
  * Utiliser toute bibliothèque tierce de CSS, de JS ou de composants
  * Intégrer votre plateforme d’analyse préférée, comme Heap, Mixpanel, etc.
  * Développer avec votre framework frontend préféré, comme Vue ou React

<div id="use-cases">
  ## Cas d’utilisation
</div>

Vous devriez utiliser ACUL avec votre mise en œuvre de Universal Login si :

* Vous devez appliquer des normes précises d’image de marque à vos flux d’authentification.
* Vous devez offrir une expérience multimarque.
* Vous devez effectuer des tests A/B au niveau de l’interface utilisateur ou obtenir des analyses détaillées.
* Vous devez offrir un Universal Login localisé dans des langues qui ne sont pas actuellement prises en charge par Auth0.
* Vous devez ajouter une logique complexe ou personnalisée à vos flux d’authentification.

Pour en savoir plus, consultez [Cas d’utilisation d’ACUL](/docs/fr-CA/customize/login-pages/advanced-customizations/acul-use-cases).

<div id="how-it-works">
  ## Comment ça fonctionne
</div>

<img className="block dark:hidden" src="/docs/images/cdy7uua7fh8z/ACUL/Final-Light-ACUL.png" alt="Comment ça fonctionne ACUL" />

<img className="hidden dark:block" src="/docs/images/cdy7uua7fh8z/ACUL/Final-Dark-ACUL.png" alt="Comment ça fonctionne ACUL" />

La **phase de compilation** commence lorsque vous créez vos écrans d’authentification personnalisés dans votre propre environnement.

Utilisez les [ACUL SDKs](/docs/fr-CA/customize/login-pages/advanced-customizations/reference) pour soumettre votre code personnalisé afin de définir des prompts et des écrans avec votre pile technologique frontend préférée, comme React ou Angular. Lorsque vous poussez des commits, votre pipeline CI/CD exécute le processus de compilation, compile votre application en ressources statiques Javascript et CSS, puis téléverse ces ressources dans votre stockage infonuagique privé, qui vous sert de réseau de diffusion de contenu (CDN).

La **phase d’exécution** commence lorsque votre application redirige les utilisateurs finaux vers votre page de connexion Auth0.

Auth0 reconnaît les écrans d’authentification que vous avez configurés pour utiliser ACUL et fournit une page hôte minimale. Le navigateur de l’utilisateur charge cette page, qui contient des balises HTML pointant vers les URL de vos ressources sur votre CDN. Le navigateur télécharge votre Javascript et votre CSS, vérifie leur intégrité à l’aide des hachages Subresource Integrity (SRI), puis affiche votre interface utilisateur personnalisée. À partir de là, le code de votre application et le SDK ACUL gèrent le reste de l’interaction de l’utilisateur.

<div id="requirements">
  ## Exigences
</div>

Pour commencer à utiliser Advanced Customizations for Universal Login, vous devez avoir :

* Un tenant de développement Auth0 configuré avec [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) et un [domaine personnalisé](/docs/fr-CA/customize/custom-domains).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Nous vous recommandons de créer un tenant pour les tests, le développement et la production, avec les balises d’environnement appropriées.
</Callout>

* Une [application propriétaire](/docs/fr-CA/get-started/auth0-overview/create-applications#create-applications) Auth0
  * Un CDN avec un pipeline CI/CD intégré, comme GitHub, pour héberger des ressources.
