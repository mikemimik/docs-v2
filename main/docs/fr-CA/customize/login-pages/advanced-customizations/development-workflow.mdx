Il existe deux modes principaux pour développer vos écrans ACUL : le développement local et les tests d’intégration sur un tenant réel. 

<div id="local-development">
  ## Développement local
</div>

Personnalisez vos écrans ACUL localement avec la commande `auth0 acul dev` de l’[Auth0 CLI](https://github.com/auth0/auth0-cli/). Faites rapidement des itérations sur vos écrans ACUL, sans délai ni dépendance au backend, grâce à [Universal Login Context Inspector](https://github.com/auth0/ul-context-inspector).

Universal Login Context Inspector fournit des données de contexte simulées pour tester vos écrans d’authentification selon les configurations propres au tenant.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/UniversalLoginContextInspector.png" alt="Universal Login Context Inspector" />
</Frame>

* Sélectionnez **Screen** pour voir les écrans ACUL configurés disponibles avec des données de contexte simulées par défaut.
  * Sélectionnez **Data Source** pour choisir des données de contexte simulées fournies par **Auth0 CDN** ou des données de contexte simulées provenant du répertoire `public/screens/{prompt}/{screen}` de votre projet de **Local Development**.
  * Sélectionnez l’icône **Download JSON** pour télécharger les données de contexte de l’écran sélectionné. 

<div id="live-tenant-integration-testing">
  ## Tests d’intégration avec un tenant réel
</div>

L’intégration avec un tenant réel au moyen de la commande Auth0 CLI `auth0 acul dev --connected` permet d’effectuer des tests de bout en bout de l’interface utilisateur de vos écrans personnalisés et de vérifier l’intégration avec votre tenant Auth0.
Utilisez le mode `--connected` pour les tests d’intégration lorsque l’interface utilisateur est presque terminée, car chaque rechargement redémarre le flux d’authentification.

Lors de l’utilisation de `auth0 acul dev --connected`

* Sélectionnez le répertoire de votre projet ACUL.
  * Sélectionnez `npm run build` pour générer le projet et configurer votre tenant Auth0.
  * Sélectionnez le **port** à utiliser pour servir vos ressources locales, soit `55444` par défaut, puis démarrez le serveur local en arrière-plan.
  * Au besoin, permettez au mode `--connected` de surveiller en continu les modifications apportées au répertoire `/dist/assets` du projet.

Ouvrez un nouveau terminal et exécutez `auth0 test login` pour tester le flux de connexion sur votre tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En mode `--connected`, la commande Auth0 CLI surveille les modifications de fichiers, régénère les ressources et met automatiquement à jour la configuration de votre tenant.
</Callout>

Pour en savoir plus, consultez [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_acul_dev.html) sur GitHub.
