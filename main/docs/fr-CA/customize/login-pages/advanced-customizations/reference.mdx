Auth0 propose différents SDKs à utiliser avec ACUL.

<div id="acul-sdks">
  ## ACUL SDKs
</div>

* [Auth0 ACUL React SDK](/docs/fr-CA/libraries/acul/react-sdk)
  Fournit des hooks React pour simplifier l’intégration aux applications React.

  * [Auth0 ACUL JS SDK](/docs/fr-CA/libraries/acul/js-sdk)
    La bibliothèque JavaScript de base (écrite en TypeScript), qui convient à tout framework frontend ou au JavaScript pur.

<div id="acul-sample-applications">
  ## Applications d’exemple ACUL
</div>

Les [applications d’exemple ACUL](https://github.com/auth0-samples/auth0-acul-samples) comprennent une chaîne de compilation complète, des flux de travail de déploiement et des exemples de mise en œuvre pour les écrans courants de Universal Login. Ces applications d’exemple vous aident à développer vos écrans. Les exemples sont offerts en React à l’aide des SDK React et JS d’ACUL.

Vous pouvez utiliser la commande `auth0 acul init` pour initialiser un projet React de départ avec les SDK React ou JS d’ACUL. Pour en savoir plus, consultez [Auth0 CLI](https://github.com/auth0/auth0-cli) sur GitHub.

<div id="acul-resources">
  ## Ressources ACUL
</div>

Auth0 offre plusieurs ressources pour personnaliser l’expérience Universal Login.

<div id="auth0-management-api">
  ### Auth0 Management API
</div>

Mettez à jour des écrans individuels d’Universal Login ou configurez plusieurs écrans à la fois à l’aide de Management API. Pour utiliser Auth0 Management API, vous devez avoir un [jeton d’accès](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens).

**Configuration d’un écran individuel**

* Le point de terminaison [`GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/get-rendering) vous permet de récupérer la configuration d’un écran précis.

  * Le point de terminaison [`PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-rendering) vous permet de configurer ACUL pour un écran précis.

**Configuration des écrans en bloc**

* Le point de terminaison [`GET /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/get-all-rendering) vous permet de récupérer la configuration de tous les écrans.

  * Le point de terminaison [`PATCH /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-bulk-rendering) vous permet de mettre à jour des écrans en bloc.

<div id="auth0-developer-tools">
  ### Outils de développement Auth0
</div>

Auth0 propose plusieurs outils pour simplifier la gestion du tenant, y compris la configuration ACUL.

* **Auth0 CLI**<p />
  Utilisez les commandes `acul` pour initialiser des projets, développer localement et déployer vos configurations d’écran. Pour en savoir plus, consultez [Auth0 CLI](https://github.com/auth0/auth0-cli) sur GitHub.

  * **Auth0 Deploy CLI**<p />
    Gérez toute la configuration de votre tenant comme du code, y compris les paramètres de rendu ACUL définis dans des fichiers JSON. Pour en savoir plus, consultez [Auth0 Deploy CLI](https://github.com/auth0/auth0-deploy-cli) sur GitHub.

  * **Auth0 Terraform provider**<p />
    Définissez vos configurations d’écran ACUL. Pour en savoir plus, consultez [`promt_screen_renderer resource`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) sur Terraform.
