title: Personnaliser les pages Classic Login
---

<Warning>
  À l’heure actuelle, les efforts de développement d’Auth0 se concentrent sur Universal Login, et Classic Login n’est plus mis à jour. Pour personnaliser vos pages de connexion, Auth0 recommande [Advanced Customizations for Universal Login](/docs/fr-CA/customize/login-pages/advanced-customizations).
</Warning>

[Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) est une expérience de connexion hébergée par Auth0 qui s’appuie sur JavaScript pour personnaliser les pages.

Dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vous pouvez configurer des options de personnalisation de base qui s’appliquent à toutes les pages de connexion. Pour une personnalisation plus poussée, vous pouvez modifier directement le modèle HTML d’une ou de plusieurs pages.

De plus, vous pouvez gérer le contenu de vos pages à l’aide du [système de contrôle de versio](/docs/fr-CA/customize/login-pages/classic-login/version-control) de votre choix.

<div id="basic-customization">
  ## Personnalisation de base
</div>

Vous pouvez configurer les options suivantes dans l’Auth0 Dashboard, sous [Branding &gt; Universal Login &gt; onglet Settings](https://manage.auth0.com/#/login_settings) :

* Logo de l’entreprise (taille recommandée : 150 x 150 pixels)
* Couleur principale
* Couleur d’arrière-plan

Par défaut, ces paramètres s’appliquent à toutes les Classic Login pages, comme l’écran de connexion ou la page de réinitialisation du mot de passe. Si vous modifiez directement le modèle HTML d’une ou de plusieurs pages et que vous changez des attributs liés à ces options, ces attributs remplaceront ces paramètres.

<div id="advanced-customization">
  ## Personnalisation avancée
</div>

Pour personnaliser davantage, vous pouvez modifier directement le modèle HTML des pages suivantes :

* Login (voir ci-dessous)
* [Réinitialisation du mot de passe](/docs/fr-CA/customize/login-pages/classic-login/customize-password-reset-page)
* [Authentification multifacteur](/docs/fr-CA/secure/multi-factor-authentication/customize-mfa/customize-mfa-classic-login)

<div id="responsibility-for-updates">
  ### Responsabilité des mises à jour
</div>

**Lorsque vous activez la personnalisation pour une page précise, vous assumez la responsabilité de sa maintenance**, puisqu’elle ne recevra plus automatiquement les mises à jour d’Auth0. Cette maintenance comprend notamment la mise à jour des numéros de version des Auth0 SDKs ou widgets inclus.

Comme des renseignements sensibles liés à la sécurité transitent souvent par la page de connexion, l’introduction de vulnérabilités de script intersite (XSS) est une source de préoccupation. De plus, si votre organisation utilise la [Home Realm Discovery](/docs/fr-CA/get-started/architecture-scenarios/business-to-business/authentication#home-realm-discovery) conjointement avec Classic Login, une liste des connections disponibles est publiquement visible dans la propriété Auth0.setClient. En conséquence, Auth0 vous encourage à faire preuve de prudence lorsque vous utilisez du JavaScript tiers sur votre page de connexion.

<div id="enable-page-customization">
  ### Activer la personnalisation d’une page
</div>

Pour activer la personnalisation d’une page donnée, suivez les étapes ci-dessous :

1. Dans l’Auth0 Dashboard, accédez à [Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings).
2. Sélectionnez l’un des onglets suivants :

   * **Login**
   * **Password Reset**
   * **Multi-Factor Authentication**
3. Activez la bascule **Customize Page** située en haut de l’onglet.
4. Répétez les étapes 2 et 3 au besoin.

Après avoir activé la personnalisation d’une page, vous pouvez modifier son modèle HTML dans l’éditeur de code situé sous la bascule.

<Warning>
  Si vous activez la personnalisation pour examiner le code de la page, mais décidez de ne pas la personnaliser, assurez-vous de désactiver la bascule **Customize Page** lorsque vous avez terminé afin qu’Auth0 puisse afficher les pages par défaut.
</Warning>

<div id="customize-the-login-page">
  ### Personnaliser la page de connexion
</div>

Pour personnaliser le gabarit de la page de connexion, vous devez d’abord choisir un modèle de base à partir duquel travailler.

1. Dans l’Auth0 Dashboard, accédez à [Branding &gt; Universal Login &gt; onglet Login](https://manage.auth0.com/#/login_page).
2. Activez la bascule **Customize Login Page**.
3. Au-dessus de l’éditeur de code HTML, sélectionnez une option dans le menu **Modèles par défaut**.

Les modèles offerts utilisent les bibliothèques suivantes :

* [Lock](/docs/fr-CA/libraries/lock)
* [Lock (mode Passwordless)](/docs/fr-CA/libraries/lock)
* [Auth0.js](/docs/fr-CA/libraries/auth0js)
* [Authentication API](https://auth0.com/docs/api/authentication)

Vous pouvez utiliser ces bibliothèques dans Classic Login ou les intégrer directement à une application. Pour mieux comprendre comment personnaliser votre expérience de connexion, consultez la documentation de la bibliothèque de votre choix.

<div id="lock-template">
  #### Modèle Lock
</div>

Si vous comptez utiliser la page de connexion par défaut et n’avez besoin que de modifications mineures, vous pouvez utiliser le modèle Lock pour modifier le comportement du widget Lock sur la page de connexion.

Par exemple, vous pouvez configurer le widget Lock pour :

* Diriger par défaut les utilisateurs vers la page d’inscription plutôt que vers la page de connexion
* Personnaliser la page de connexion avec différentes couleurs, différents textes ou différentes langues par défaut
* Présenter aux utilisateurs seulement une ou plusieurs méthodes de connexion précises

Pour en savoir plus sur la personnalisation du widget Lock, consultez [Lock Configuration Options](/docs/fr-CA/libraries/lock/lock-configuration).

<div id="custom-login-form-template">
  #### Modèle Custom Login Form
</div>

Si vous voulez modifier considérablement la page de connexion, vous pouvez utiliser le modèle Custom Login Form comme point de départ. Ce modèle vous montre comment obtenir les valeurs dont vous avez besoin à l’aide du SDK auth0.js. Vous pouvez ensuite modifier le style et la mise en page de la page de connexion comme bon vous semble. Toute personnalisation CSS **doit** être intégrée directement dans le code, car vous ne pouvez pas héberger de fichier CSS distinct dans votre tenant Auth0.
