title: Contrôle de version des pages de Classic Login
---

<Warning>
  À l’heure actuelle, les efforts de développement d’Auth0 sont axés sur Universal Login, et Classic Login ne reçoit plus de mises à jour. Pour personnaliser vos pages de connexion, Auth0 recommande [Advanced Customizations for Universal Login](/docs/fr-CA/customize/login-pages/advanced-customizations).
</Warning>

Vous pouvez utiliser un logiciel de contrôle de version pour gérer le code source de vos pages de Classic Login. Le contrôle de version de votre contenu peut être particulièrement utile si vous utilisez un processus CI/CD pour la configuration de votre tenant et si vous avez personnalisé le HTML d’une ou de plusieurs pages de Classic Login.

Pour mettre en place le contrôle de version, utilisez l’intégration de l’Auth0 Marketplace correspondant à la méthode de votre choix :

* [GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline)
* [GitHub Actions](https://marketplace.auth0.com/integrations/github-actions)
* [Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline)
* [Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline)

Le processus général de déploiement comprend les étapes suivantes :

1. Créez un dossier dans votre dépôt de contrôle de version en lui donnant le nom approprié (`pages`).
2. Créez une page HTML (`login.html`, `password_reset.html`, `guardian_multifactor.html` ou `error_page.html`) dans ce dossier.
3. Créez un fichier JSON portant le même nom que votre page HTML pour chaque page hébergée que vous souhaitez gérer avec le contrôle de version. Pour activer la page, le fichier JSON doit contenir ce qui suit :

`{
"enabled": true
}`

Par exemple :
`your-repo/pages/error_page.html
your-repo/pages/error_page.json`
