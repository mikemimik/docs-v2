Vous pouvez personnaliser les pages <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> en fournissant un gabarit de page créé à l’aide du [langage de gabarits Liquid](https://shopify.github.io/liquid/basics/introduction/). Les gabarits de page vous permettent de définir le contenu affiché autour des prompts Universal Login, comme la boîte de connexion ou une demande de vérification <Tooltip tip="Multi-factor authentication (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=MFA">MFA</Tooltip>. Comme le même gabarit de page est utilisé pour toutes les pages du flux de connexion, cette méthode de personnalisation vous permet de mettre en place facilement une expérience cohérente et fidèle à votre image de marque pour les utilisateurs.

Pour utiliser des gabarits de page personnalisés, vous devez configurer un [domaine personnalisé](/docs/fr-CA/customize/custom-domains) pour votre tenant. De plus, vous pouvez uniquement mettre à jour les gabarits de page Universal Login avec la [Management API](https://auth0.com/docs/api/management/v2).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le terme `prompt` désigne une étape précise du flux de connexion, comme la page d’inscription ou une demande de vérification MFA. Les variables et les exemples de code sur cette page peuvent utiliser le terme `prompt` ou le terme `widget`. Bien que ces termes soient synonymes dans le contexte d’Universal Login, ils ne sont **pas** interchangeables dans votre code.  Pour personnaliser correctement vos pages, vérifiez que vous utilisez le terme approprié pour chaque élément ajouté à votre code.
</Callout>

<div id="page-template-requirements">
  ## Exigences du gabarit de page
</div>

Lorsque vous créez un gabarit de page Universal Login, vous devez inclure les balises suivantes :

| Balise         | Description                                                                                                                                        |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth0:widget` | Contient le HTML du prompt affiché sur chaque page du flux de connexion, comme la page de connexion ou la page de réinitialisation du mot de passe |
| `auth0:head`   | Contient les balises requises pour afficher le prompt                                                                                              |

Pour centrer le prompt sur la page, ajoutez `class="_widget-auto-layout"` à l’élément `<body>`. Vous pouvez omettre cet attribut pour positionner le prompt manuellement au besoin.

<Warning>
  **Limites du gabarit de page :**

  * Les noms de classes CSS changent chaque fois qu’Auth0 génère le projet. Toute règle CSS personnalisée qui cible ces classes cessera de fonctionner à chaque nouvelle génération.
  * La structure HTML des pages Universal Login peut changer. Évitez les personnalisations qui en dépendent afin de prévenir toute interruption.

  Pour en savoir plus, consultez [la personnalisation CSS](#css-customization).
</Warning>

<div id="example-template">
  ##### Exemple de gabarit :
</div>

L’exemple suivant montre le gabarit de page Universal Login le plus simple que vous puissiez créer avec les balises requises :

```liquid lines
<!DOCTYPE html>
{% assign resolved_dir = dir | default: "auto" %}
<html lang="{{locale}}" dir="{{resolved_dir}}">
  <head>
    {%- auth0:head -%}
  </head>
  <body class="_widget-auto-layout">
    {%- auth0:widget -%}
  </body>
</html>
```

<Warning>
  Si vous utilisez Storybook pour afficher votre gabarit, sachez que la balise `<script>` fait échouer le rendu, car Storybook n’arrive pas à interpréter correctement ces balises.  Pour contourner ce problème, utilisez les caractères accent grave (&#96;) et signe plus (+) pour injecter correctement la balise `<script>` dans le code de votre gabarit.

  **Exemple** :

  ``<scr`+`ipt>console.log("test");</scr`+`ipt>``
</Warning>

<div id="page-template-variables">
  ## Variables de gabarit de page
</div>

Les gabarits de page prennent en charge diverses variables de contexte qui influent sur le rendu d’une page.

Par exemple, vous pouvez utiliser ces variables pour :

* Afficher un contenu différent selon l’application associée au flux de connexion. Par exemple, vous pourriez gérer deux marques qui nécessitent des mises en page différentes.
* Afficher un contenu différent selon le prompt concerné. Par exemple, vous pourriez vouloir ajouter de l’information sur ce que votre application offre sur la page de connexion, mais préférer que le flux MFA n’affiche que le prompt de défi MFA.
* Ajouter un pied de page contenant des renseignements de soutien aux utilisateurs, comme des liens vers votre page de soutien ou vos coordonnées.

<div id="custom-query-parameters">
  ### Paramètres de requête personnalisés
</div>

<Warning>
  Auth0 recommande de traiter les paramètres `ext-` comme non fiables, puisque ces valeurs peuvent provenir d’entrées d’URL contrôlées par un attaquant. Vous devriez suivre les étapes recommandées pour atténuer les risques de sécurité :

  * Utilisez des sorties HTML échappées dans les modèles
  * N’interpolez pas les valeurs `ext-` dans des blocs `<script>` ou des attributs de gestionnaires d’événements
  * Validez les schémas JavaScript ou de données avant d’utiliser ces valeurs dans des URL

  Veuillez noter que la liste de caractères autorisés n’élimine pas tous les risques de XSS dans tous les contextes de rendu. Pour en savoir plus sur les directives et les recommandations de sécurité, consultez [Security Guidance](/docs/fr-CA/secure/security-guidance).
</Warning>

Vous pouvez aussi utiliser des paramètres de requête dans le contexte en les transmettant au point de terminaison `/authorize` au moment d’amorcer la demande d’authentication. Ces paramètres de requête personnalisés doivent avoir le préfixe `ext-`.

L’exemple suivant utilise les paramètres de requête `ext-ga` et `ext-test` dans le modèle de page de connexion :

```html lines
<!DOCTYPE html>
<html>
  <head>
    {%- auth0:head -%}
  </head>
  <body>
    {%- auth0:widget -%}
  </body>
  <pre style='background: wheat'>
    <b>Value of the ext-ga parameter:</b>{{ transaction.params.ext-ga }}
    <b>Value of the ext-test parameter:</b>{{ transaction.params.ext-test }}
  </pre>
</html>
```

Les paramètres de requête personnalisés sont soumis aux limites suivantes :

* Chaque nom de paramètre `ext-` doit être unique
* Une requête d’autorisation peut contenir au maximum dix paramètres `ext-`
* Le nom du paramètre `ext-` doit commencer par ext-, ne contenir que [a-zA-z0-9&#95;-] et comporter au maximum 28 caractères, comme dans l’exemple suivant : `/^ext-[\w-]{1,28}$/`
* La valeur du paramètre `ext-` ne doit contenir que [a-zA-Z0-9-.*~@+ /:&#95;] et comporter au maximum 255 caractères, comme dans l’exemple suivant : `/^[-\w.*~@+ /:]{1,255}$/`

<Warning>

  * Les noms de classe CSS changent chaque fois qu’Auth0 génère le projet. Le CSS personnalisé qui cible ces classes cessera de fonctionner à chaque nouvelle génération.
  * La structure HTML des pages Universal Login peut changer. Évitez les personnalisations qui dépendent de la structure HTML afin de prévenir toute interruption.
</Warning>

Voici quelques éléments que vous pouvez personnaliser à l’aide du CSS :

* Vous pouvez redimensionner le prompt de connexion en plaçant les variables ci-dessous dans des balises `<style>` de l’élément `<head>`.

  * Utilisez `--prompt-width` pour ajuster la largeur du conteneur. Sa valeur par défaut est de 400px.
  * Pour [Forms](/docs/fr-CA/customize/forms) : utilisez `--form-max-width` pour définir la largeur maximale du formulaire. Sa valeur par défaut est de 500px.

    * Pour appliquer votre code uniquement à la page Form, incluez `{% if prompt.name == "custom-form" %} dans l’élément <head>`.
* Vous pouvez utiliser une police Google en l’important et en remplaçant la variable CSS `--font-family`.
* Vous pouvez masquer le logo du tenant en ajoutant `class="_hide-prompt-logo"` dans l’élément `<body>`.
* Vous pouvez spécifier un logo personnalisé en ajoutant `class="_use-custom-prompt-logo"` dans l’élément `<body>`. Cela vous permettrait, par exemple, de modifier le logo de la page de connexion selon l’application :

```html lines expandable
<!DOCTYPE html>
<html lang="{{locale}}">

  <head>
    <title>Welcome to {{ application.name }} </title>
    {%- auth0:head -%}
    <style>
      :root {
        --prompt-width: 800px;
      }
      {% if application.name == "Auth0 Community" %}
      #custom-prompt-logo {
      background-image: url('https://cdn.auth0.com/manhattan/versions/1.3312.0/assets/badge.png');
      }
      {% elsif application.name == "Auth0 Dashboard" %}
      #custom-prompt-logo {
      background-image: url('https://cdn.auth0.com/blog/auth0rta/theme/logos/auth0-logo-black.png');
      }
      {% endif %}
  </style>

  </head>

  <body class="_widget-auto-layout _use-custom-prompt-logo">
    {%- auth0:widget -%}
  </body>

</html>
```

La mise en œuvre actuelle ne permet pas de personnaliser davantage le CSS. Si vous regardez le HTML généré, vous verrez du code comme :

```text lines
.c10d15918.c7b3b8672 {
  background: #D00E17;
}
```

<div id="using-the-auth0-cli">
  ## Utiliser l’Auth0 CLI
</div>

Vous pouvez utiliser l’[Auth0 CLI](https://github.com/auth0/auth0-cli) pour mettre à jour facilement les gabarits de page.

Dans l’Auth0 CLI, exécutez :

`auth0 universal-login customize`

L’Auth0 CLI ouvrira deux fenêtres :

* Une fenêtre de navigateur avec un [Storybook](https://storybook.js.org/) qui affiche la page de connexion avec le gabarit de page appliqué :

<Frame>![Storybook des gabarits de page](/docs/images/cdy7uua7fh8z/1Ms4jj0pdMowoSZi54i72S/a2adcffa34a491ed7a587094884ad0f1/storybook.png)</Frame>

* L’éditeur par défaut, avec le code du gabarit de page :

<Frame>![undefined](/docs/images/cdy7uua7fh8z/3fTqpTmRRgVeLu5p8cioWa/e5a5c8a12dc93f5de95f12581ce0e5ff/vs-code.png)</Frame>

Vous pouvez maintenant modifier le code du gabarit de page et prévisualiser les changements dans la fenêtre de votre navigateur.

Lorsque vous fermerez la fenêtre, il vous sera demandé si vous souhaitez enregistrer le modèle. Si vous répondez **Oui**, le modèle sera téléversé dans votre tenant.

<div id="troubleshooting">
  ## Dépannage
</div>

Si le gabarit de page ne s’applique pas, assurez-vous d’accéder à `{customDomain}/authorize`. Si vous accédez à `{yourDomain}/authorize`, Auth0 n’affichera pas le gabarit de page.
