<Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip> se compose d’un ensemble de pages qui facilitent plusieurs actions liées au compte, comme la connexion, l’inscription à des <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=multi-factor+authentication">facteurs d’authentification multifacteur</Tooltip> ou la réinitialisation des mots de passe. Auth0 fournit le texte de ces pages en plusieurs langues.

Dans certains cas, vous souhaiterez peut-être modifier le texte de ces pages afin qu’il corresponde mieux au ton ou aux besoins particuliers de votre application.

Vous pouvez personnaliser le texte affiché sur les pages de [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) dans toutes les langues prises en charge à partir de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Les définitions, valeurs et variables suivantes s’appliquent aux deux méthodes.

<div id="configure-prompts-and-screens">
  ## Configurer les prompts et les écrans
</div>

Vous pouvez personnaliser n’importe lequel des [prompts et écrans](#prompt-values) disponibles dans l’Auth0 Dashboard ou à l’aide de la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Utiliser le Dashboard">
    Vous pouvez utiliser le Dashboard pour personnaliser les textes des prompts de New Universal Login.

    1. Dans le Dashboard, accédez à [**Branding &gt; Universal Login &gt; Advanced Options**](https://manage.auth0.com/#/custom_text), puis sélectionnez **Custom Text**.
    2. Sélectionnez le **Prompt** et le **Screen** que vous souhaitez modifier, puis apportez les modifications voulues. Un aperçu en direct est disponible à droite.
    3. Lorsque vous êtes prêt à afficher vos modifications dans votre production environment, sélectionnez **Save Changes.**
  </Tab>

  <Tab title="Utiliser l’API Management">
    Vous pouvez utiliser la Management API pour personnaliser les invites de texte du New Universal Login.

    1. Obtenez un [jeton d’accès à la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les scopes `read:prompts` et `update:prompts`.
    2. Envoyez une requête au point de terminaison [Obtenir le texte personnalisé d’un prompt](https://auth0.com/docs/api/management/v2#!/Prompts/get_custom_text_by_language) de la Management API pour récupérer les données de configuration existantes pour le prompt et la langue indiqués.
    3. Modifiez les données de configuration comme vous le souhaitez, en vous référant aux [paramètres pris en charge](#supported-parameters) et aux [valeurs de prompt](#prompt-values).
    4. Effectuez une requête au point de terminaison [Définir le texte personnalisé pour un prompt donné](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de la Management API avec les données de configuration mises à jour.

       <Warning>
         Le point de terminaison [Set custom text for a specific prompt](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de la Management API remplace l’ensemble des données de configuration existantes du prompt indiqué par celles fournies dans le paramètre `body`.

         Si vous souhaitez configurer plusieurs valeurs de `screen` pour un prompt donné, vous devez toutes les inclure dans une seule requête à la Management API.
       </Warning>

    #### Exemple

    Vous pouvez définir le composant de texte `description` de l&#39;écran `login` du prompt `login` à `Log in to ACME's website` en effectuant une requête à l&#39;endpoint [Set custom text for a specific prompt](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de la Management API, comme suit :

    <AuthCodeGroup>
      ```bash cURL
      curl --request PUT \
        --url 'https://{yourDomain}/api/v2/prompts/login/custom-text/en' \
        --header 'authorization: Bearer {yourMgmtApiAccessToken}' \
        --header 'content-type: application/json' \
        --data '{ "login": { "description": "Login to ACME'''s Website" } }'
      ```

      ```csharp C#
      var client = new RestClient("https://{yourDomain}/api/v2/prompts/login/custom-text/en");
      var request = new RestRequest(Method.PUT);
      request.AddHeader("content-type", "application/json");
      request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
      request.AddParameter("application/json", "{ "login": { "description": "Login to ACME's Website" } }", ParameterType.RequestBody);
      IRestResponse response = client.Execute(request);
      ```

      ```go Go
      package main

      import (
      	"fmt"
      	"strings"
      	"net/http"
      	"io/ioutil"
      )

      func main() {

      	url := "https://{yourDomain}/api/v2/prompts/login/custom-text/en"

      	payload := strings.NewReader("{ "login": { "description": "Login to ACME's Website" } }")

      	req, _ := http.NewRequest("PUT", url, payload)

      	req.Header.Add("content-type", "application/json")
      	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

      	res, _ := http.DefaultClient.Do(req)

      	defer res.Body.Close()
      	body, _ := ioutil.ReadAll(res.Body)

      	fmt.Println(res)
      	fmt.Println(string(body))

      }
      ```

<div id="supported-parameters">
  ### Paramètres pris en charge
</div>

| Paramètre  | Définition                                                                                                                                                                                                                                  |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `prompt`   | L’étape du flux de connexion. Pour consulter la liste des valeurs disponibles, voir [Valeurs de prompt](#prompt-values).                                                                                                                    |
| `language` | La langue à laquelle la personnalisation s’applique. Pour consulter la liste des langues offertes, voir [Internationalisation de Universal Login](/docs/fr-CA/customize/internationalization-and-localization/universal-login-internationalization). |
| `screen`   | L’écran précis dans un prompt.                                                                                                                                                                                                              |
| `key`      | L’identifiant d’un composant textuel.                                                                                                                                                                                                       |
| `text`     | Le message affiché à l’utilisateur.                                                                                                                                                                                                         |

<div id="prompt-values">

</div>

Le terme `prompt` désigne une étape particulière du flux de connexion. Chaque `prompt` comporte au moins un `screen`, et chaque `screen` comprend un ensemble de composants textuels que vous pouvez personnaliser.

<div id="variables">
  ## Variables
</div>

Certains écrans contiennent des variables dans le texte qui sont remplacées à l’exécution en fonction d’informations contextuelles. Comme des variables différentes sont disponibles pour chaque écran, il n’est pas garanti qu’une variable donnée fonctionne ailleurs que sur son écran d’origine.

| Variable                      | Description                                                                                                      |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| $&#123;clientName&#125;       | Nom de l’application Auth0                                                                                       |
| $&#123;connectionName&#125;   | Nom de la connexion (p. ex. Google)                                                                              |
| $&#123;tenantName&#125;       | Nom du tenant Auth0                                                                                              |
| $&#123;organizationName&#125; | Nom de l’organisation de l’utilisateur si celui-ci s’authentifie dans le contexte d’une Organisation             |
| $&#123;companyName&#125;      | Nom du tenant Auth0, ou nom de l’Organisation si l’utilisateur s’authentifie dans le contexte d’une Organisation |
| $&#123;userName&#125;         | Nom de l’utilisateur connecté                                                                                    |
| $&#123;email&#125;            | Adresse courriel de l’utilisateur connecté                                                                       |
| $&#123;appName&#125;          | Nom de l’application Guardian Push personnalisée                                                                 |
