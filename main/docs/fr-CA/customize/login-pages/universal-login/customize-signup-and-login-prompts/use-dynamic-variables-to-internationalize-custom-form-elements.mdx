title: Utiliser des variables dynamiques pour internationaliser des éléments de formulaire personnalisés
---

Vous pouvez afficher les [personnalisations de l’écran d’inscription](/docs/fr-CA/customize/login-pages/universal-login/customize-signup-and-login-prompts) différemment selon les données contextuelles. Pour ce faire, on utilise la variable `locale` pour afficher conditionnellement des champs de formulaire et définir le comportement de validation.

Le cas d’utilisation suivant utilise la variable `locale` pour afficher un message de Conditions d’utilisation et une case à cocher. Toute variable exposée dans les [Modèles de page](/docs/fr-CA/customize/login-pages/universal-login/customize-templates#available-variables) peut être remplacée.

<div id="prerequisites">
  ### Prérequis
</div>

* Le tenant a un [domaine personnalisé](/docs/fr-CA/customize/custom-domains) vérifié
* Le tenant a un modèle de page défini

<div id="enable-fr-and-es-locales">
  ### Activer les paramètres régionaux fr et es
</div>

Accédez à votre <Tooltip tip="Auth0 Dashboard : l’interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, puis accédez à Settings &gt; General et activez les paramètres régionaux :

<Frame>![](/docs/images/cdy7uua7fh8z/4yCGYTZ3RDyeULWIFcyHSv/01ce1985b1f8655aff0a0a2b4db6bee0/Language_Picker_-_EN.png)</Frame>

<div id="conditionally-render-custom-fields">
  ### Afficher les Custom Fields de façon conditionnelle
</div>

Utilisez <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> pour ajouter un gabarit partiel au conteneur du signup prompt `form-content-end` de <Tooltip tip="Universal Login : Votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Universal+Login">Universal Login</Tooltip>. Si la locale est `es` ou `fr`, vous voudrez peut-être afficher un message sur les Conditions d’utilisation ainsi qu’une case à cocher.

```liquid lines
{% if locale == 'fr' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      J'accepte les
      <a href="https://fr.example.com/tos">termes et conditions</a>
    </label>
  </div>
{% endif %}
{% if locale == 'es' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      Estoy de acuerdo con los
      <a href="https://es.example.com/tos">términos y condiciones</a>
    </label>
  </div>
{% endif %}
```

La requête suivante est envoyée au point de terminaison **Définir le texte personnalisé pour un prompt donné** dans l’[Management API](https://auth0.com/docs/api/management/v2/prompts/put-custom-text-by-language) :

```javascript lines
// PUT prompts/signup/partials

{
  "signup": {
    "form-content-end": "{% if locale..."
  }
}
```

Votre prompt d’inscription affiche maintenant un message sur les conditions d’utilisation et une case à cocher uniquement lorsque la locale est définie sur `fr` ou `es` :

<Frame>![](/docs/images/cdy7uua7fh8z/3zBadIHnkl9bIWP9BH1lLj/b2a6d9932be3bbb150b4dbbf76bc6599/2024-01-31_16-17-04.png)</Frame><Frame>![](/docs/images/cdy7uua7fh8z/3rwVd1h1Av7QzOEEWoCHhF/519a2eb5ac99c3801861c3372b81c0b9/2024-01-31_16-17-16.png)</Frame><Frame>![](/docs/images/cdy7uua7fh8z/2qYzAI6QVBLmqCW2FTAiGj/f19c59d79c67c63782b70995dbd223c3/2024-01-31_16-17-24.png)</Frame>

Pour tester le résultat, accédez au **Dashboard de gestion** et allez à [Branding &gt; Universal Login &gt; Options de personnalisation](https://manage.auth0.com/dashboard/us/dev-6endizjt/universal-login/customizations/colors), puis cliquez avec le bouton droit sur le **bouton Try** et copiez l’adresse du lien. Ajoutez les paramètres de requête suivants à l’URL copiée, puis ouvrez la nouvelle URL : `&screen_hint=signup&ui_locales=fr` (ou `es`).

<div id="add-validation">
  ### Ajouter une validation
</div>

Lorsque la locale est `fr,` vous pouvez vérifier que la case à cocher est cochée avant de continuer. Mettez à jour le gabarit partiel à l’aide du code de validation suivant :

```liquid lines expandable
{% if locale == 'fr' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      J'accepte les
      <a href="https://fr.example.com/tos">termes et conditions</a>
    </label>

    <!-- NOUVEAU -->
    <div
      class="ulp-error-info"
      data-ulp-validation-function="requiredFunction"
      data-ulp-validation-event-listeners="change">
      Vous devez accepter les termes et conditions pour continuer
    </div>
    <!-- FIN NOUVEAU -->
  </div>
{% endif %}
{% if locale == 'es' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      Estoy de acuerdo con los
      <a href="https://es.example.com/tos">términos y condiciones</a>
    </label>
  </div>
{% endif %}

<!-- NOUVEAU -->
<script>
  function requiredFunction(element, formSubmitted) {
    if (! formSubmitted) {
      return true;
    }
    return element.checked;
  }
</script>
<!-- FIN NOUVEAU -->
```

Avec cette validation, seuls les utilisateurs dont la `locale` est `fr` doivent accepter les ToS.

<Frame>![](/docs/images/cdy7uua7fh8z/7jBPuIe62PDNiIPUYzVVix/74de519659de4a8b4329165e799b1815/2024-01-31_16-20-54.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les validations de ce cas d’utilisation se font côté client et visent à simplifier l’expérience utilisateur. Leur exécution n’est pas garantie ; par exemple, l’utilisateur peut avoir désactivé Javascript dans son navigateur. De plus, les validations peuvent être modifiées par des acteurs curieux ou malveillants. Pour assurer l’intégrité de la logique de validation, les validations côté client devraient être jumelées à des validations côté serveur.
</Callout>
