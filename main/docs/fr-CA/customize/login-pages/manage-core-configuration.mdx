La page de Core configuration vous permet de gérer les paramètres suivants pour Universal Login et Classic Login :

* **Moteur de connexion** : Passez de Universal Login à Classic Login, et inversement.
* **Iframe embedding** : Permettez le chargement des flux de connexion dans des iframes pour les environnements de développement.

<div id="settings">
  ## Paramètres
</div>

<div id="universal-or-classic-login">
  ### Universal Login ou Classic Login
</div>

Choisissez l’expérience de connexion utilisée par votre tenant. Modifier ce paramètre affecte toutes les applications qui redirigent vers votre page de connexion Auth0. Pour en savoir plus, consultez [Universal Login vs. Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<Steps>
  <Step title="Accéder à Core configuration">
    Accédez à **Branding &gt; Universal Login &gt; Personnalisations pro-code** et sélectionnez **Core configuration**.
  </Step>

  <Step title="Sélectionner le type de connexion">
    Sous **Settings**, sélectionnez **Universal Login** ou **Classic Login**.
  </Step>
</Steps>

<div id="iframe-embedding">
  ### Iframe embedding
</div>

Activez Universal Login ou Classic Login pour qu’ils se chargent lorsque l’application et le flux de connexion sont intégrés à un iframe, comme sur des plateformes d’UI générative telles que Vercel. Lorsque cette option est activée, Iframe embedding s’applique uniquement dans les environnements de développement.

<Warning>
  Lorsque Iframe embedding est activé, les en-têtes de sécurité qui protègent contre le clickjacking sont désactivés.
</Warning>

<Steps>
  <Step title="Accéder à Core configuration">
    Accédez à **Branding &gt; Universal Login &gt; Personnalisations pro-code** et sélectionnez **Core configuration**.
  </Step>

  <Step title="Activer la prise en charge des iframes">
    Sous **Paramètres d’iframe**, activez **Activer la prise en charge des plateformes d’UI générative**.
  </Step>

  <Step title="Configurer les origines autorisées">
    Sous **URL d’iframe autorisées pour la connexion**, entrez les origines autorisées.

    Pour v0, entrez les origines suivantes :

    * `https://v0.app`
    * `https://*.v0.app`
    * `https://*.vusercontent.net`

    Lorsqu’une application est créée à partir d’une plateforme d’UI générative comme Vercel à l’aide d’une intégration Auth0 disponible, Auth0 préremplit les origines autorisées à partir de la plateforme. Vérifiez toujours les origines préremplies pour vous assurer qu’elles sont complètes. Pour en savoir plus, consultez [Intégrer Vercel](/docs/fr-CA/customize/integrations/integrate-with-vercel).
  </Step>
</Steps>
