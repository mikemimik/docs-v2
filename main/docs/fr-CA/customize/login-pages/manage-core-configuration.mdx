La page Configuration principale vous permet de gérer les paramètres suivants pour Universal Login et Classic Login :

* **Moteur de connexion** : Basculez entre Universal et Classic Login.
* **Intégration dans un iframe** : Autorisez le chargement des flux de connexion dans des iframes pour les environnements de développement.

<div id="settings">
  ## Paramètres
</div>

<div id="universal-or-classic-login">
  ### Universal Login ou Classic Login
</div>

Choisissez l’expérience de connexion utilisée par votre tenant. La modification de ce paramètre a une incidence sur toutes les applications qui redirigent vers votre page de connexion d’Auth0. Pour en savoir plus, consultez [Universal Login vs. Classic Login](/docs/fr-CA/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<Steps>
  <Step title="Accéder à la configuration principale">
    Accédez à **Image de marque &gt; Universal Login &gt; Personnalisations pro-code** et sélectionnez **Configuration principale**.
  </Step>

  <Step title="Sélectionner le type de connexion">
    Sous **Paramètres**, sélectionnez **Universal Login** ou **Classic Login**.
  </Step>
</Steps>

<div id="iframe-embedding">
  ### Intégration dans une iframe
</div>

Activez Universal Login ou Classic Login pour qu’ils se chargent lorsque l’application et le flux de connexion sont intégrés dans une iframe, comme avec des plateformes d’UI générative telles que Vercel. Lorsqu’elle est activée, l’intégration dans une iframe s’applique uniquement aux environnements de développement.

<Warning>
  Lorsque l’intégration dans une iframe est activée, les en-têtes de sécurité qui protègent contre les attaques de clickjacking sont désactivés.
</Warning>

<Steps>
  <Step title="Accéder à configuration principale">
    Accédez à **image de marque &gt; Universal Login &gt; Personnalisations pro-code** et sélectionnez **configuration principale**.
  </Step>

  <Step title="Activer la prise en charge des iframes">
    Sous **Iframe settings**, activez la bascule **Enable support for generative UI platforms**.
  </Step>

  <Step title="Configurer les origines autorisées">
    Sous **Allowed iframe URLs for Login**, saisissez les origines autorisées.

    Pour v0, saisissez les origines suivantes :

    * `https://v0.app`
    * `https://*.v0.app`
    * `https://*.vusercontent.net`

    Lorsqu’une application est créée à partir d’une plateforme d’UI générative comme Vercel à l’aide d’une intégration Auth0 disponible, Auth0 préremplit les origines autorisées à partir de la plateforme. Vérifiez toujours les origines préremplies pour vous assurer qu’elles sont complètes. Pour en savoir plus, consultez [Integrate with Vercel](/docs/fr-CA/customize/integrations/integrate-with-vercel).
  </Step>
</Steps>
