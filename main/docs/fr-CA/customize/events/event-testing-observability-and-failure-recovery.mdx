Les sections suivantes présentent des renseignements sur le test des flux d’événements et la gestion des événements.

<div id="test-the-event-stream">
  ## Tester le flux d’événements
</div>

Après avoir créé et activé un flux d’événements, vous pouvez tester ce nouveau flux. Actuellement, Auth0 ne prend pas en charge l’envoi d’événements vers des flux d’événements désactivés.

Pour tester votre flux d’événements, commencez par créer un utilisateur pour votre application en exécutant la commande suivante avec les identifiants de votre choix :

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    auth0 users create # suivez les invites pour créer un utilisateur de test
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -X POST https://$TENANT/api/v2/users \
      -d '{
            "email": "nick-testing-21@example.com",
            "password": "Password123!",
            "connection": "Username-Password-Authentication"
          }'
    ```
  </Tab>
</Tabs>

Une fois l’utilisateur créé, sa création devrait déclencher l’événement `user.created` dans votre flux d’événements et renvoyer une réponse. Dans cet exemple, des webhooks sont utilisés pour acheminer le flux d’événements.

```json lines expandable
"Webhook received": {
  "id": "evt_6SfwXrtm8nVcjzzNCFFH1L",
  "source": "urn:auth0:",
  "specversion": "1.0",
  "type": "user.created",
  "time": "2025-02-02T17:56:01.573Z",
  "data": {
    "object": {
      "created_at": "2025-02-02T17:56:01.566Z",
      "email": "nick-testing-21@example.com",
      "email_verified": false,
      "identities": [
        {
          "connection": "Username-Password-Authentication",
          "isSocial": false,
          "provider": "auth0",
          "user_id": "679fb1b1e809445f129bc18d"
        }
      ],
      "name": "nick-testing-21@example.com",
      "nickname": "nick-testing-21",
      "picture": "test.png",
      "updated_at": "2025-02-02T17:56:01.566Z",
      "user_id": "auth0|679fb1b1e809445f129bc18d"
    }
  },
  "a0tenant": "acme",
  "a0stream": "est_38ANnv8pQhqsZrX2VFbftv",
  "a0purpose": "test"
}
```

Vous pouvez afficher la liste des échecs de livraison des événements pour un flux d’événements donné en exécutant :

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    auth0 api get event-streams/$EVENT_STREAM_ID/deliveries
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -H "Authorization: Bearer $TOKEN" \
      "https://$TENANT/api/v2/event-streams/${EVENT_STREAM_ID}/deliveries" | jq .
    ```
  </Tab>
</Tabs>

Si le système renvoie un tableau vide pour les échecs de livraison, votre flux d’événements fonctionne correctement.

<div id="observability">
  ## Observabilité
</div>

Pour assurer la fiabilité des flux d’événements, mettez en place un processus de surveillance qui vérifie périodiquement les échecs de livraison et déclenche des alertes au besoin.

Les échecs de livraison peuvent être causés par :

* Flux d’événements désactivés.
* Flux d’événements indisponibles.
* Erreurs de connexion.

<div id="polling-deliveries-endpoint">
  ### Interrogation périodique du point de terminaison Deliveries
</div>

Interrogez le point de terminaison Deliveries toutes les **5 minutes** pour vérifier s’il y a des événements en échec. Utilisez l’appel d’API suivant pour récupérer les livraisons d’événements récentes :

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    auth0 api get event-streams/$EVENT_STREAM_ID/deliveries
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -H "Authorization: Bearer $TOKEN" \
      "https://$TENANT/api/v2/event-streams/${EVENT_STREAM_ID}/deliveries" | jq .
    ```
  </Tab>
</Tabs>

Si des échecs sont détectés :

* Consignez les détails des événements en échec.
* Alertez l’équipe ou le système approprié afin d’examiner et de diagnostiquer le problème.

<div id="recovery">
  ## Rétablissement
</div>

La livraison des événements sera automatiquement relancée. Pour en savoir plus, consultez [Best Practices](/docs/fr-CA/customize/events/event-testing-observability-and-failure-recovery#batch-redelivery-of-failed-events).

En cas d’échec, utilisez la Redelivery API pour relancer la livraison des événements.

<div id="redeliver-a-single-failed-event">
  ### Renvoyer un seul événement en échec
</div>

Pour relancer un événement en échec précis, utilisez :

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash lines
    EVENT_ID=<your event id>
    auth0 api POST event-streams/$EVENT_STREAM_ID/redeliver/$EVENT_ID --data '{}'
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -X POST -H "Authorization: Bearer $TOKEN" \
      "https://<your-domain>/api/v2/event-streams/$EVENT_STREAM_ID/redeliver/$EVENT_ID"
    ```
  </Tab>
</Tabs>

<div id="batch-redelivery-of-failed-events">
  ### Réexpédition par lot des événements en échec
</div>

Pour réessayer tous les événements en échec du flux d’événements :

<Tabs>
  <Tab title="Auth0 CLI">
    ```bash wrap lines
    auth0 api POST event-streams/$EVENT_STREAM_ID/redeliver --data '{}'
    ```
  </Tab>

  <Tab title="cURL">
    ```bash lines
    curl -s -X POST -H "Authorization: Bearer $TOKEN" \
      "https://<your-tenant>.auth0.com/api/v2/event-streams/<event_stream_id>/redeliver"
    ```
  </Tab>
</Tabs>

<div id="auto-disable-mechanism">
  ### Mécanisme de désactivation automatique
</div>

Auth0 désactive automatiquement un flux d’événements en cas d’échecs trop nombreux.

| Condition              | Mesure appliquée                                    |
| ---------------------- | --------------------------------------------------- |
| 500 échecs consécutifs | Le flux d’événements est automatiquement désactivé. |
| 5000 échecs au total   | Le flux d’événements est automatiquement désactivé. |

* Si un flux d’événements est automatiquement désactivé, vous **devez** utiliser la Redelivery API pour traiter les événements en échec.
* Le flux d’événements **ne peut pas** être réactivé manuellement tant que le nombre total d’échecs ne tombe pas sous 5000.
* Une fois le nombre d’échecs réduit, vous pouvez réactiver le flux au moyen de l’API.
