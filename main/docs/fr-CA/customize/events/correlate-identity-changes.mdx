title: Corréler les changements d’identité avec des systèmes externes
---

Avec Event Streams, vous pouvez réagir aux changements d’identité dans Auth0 et les répercuter dans des systèmes externes comme des plateformes CRM, des services de facturation ou des outils de gestion des licences SaaS. Plutôt que d’interroger périodiquement la Management API pour obtenir des mises à jour, votre intégration reçoit les événements en temps réel et les met en correspondance avec les enregistrements de vos systèmes en aval.

<div id="why-correlate-identity-changes">
  ## Pourquoi corréler les changements d’identité
</div>

Corréler les événements d’identité d’Auth0 avec des systèmes externes est utile lorsque vous devez :

* Mettre à jour un dossier dans un CRM lorsque l’adresse courriel ou les données de profil d’un utilisateur changent.
* Aviser un système de facturation ou de gestion des licences lorsqu’un compte est désactivé ou supprimé.
* Déclencher des flux de travail de conformité lorsque des attributs utilisateur changent, par exemple lors de mises à jour du consentement ou du rôle.
* Veiller à ce que les segments d’utilisateurs dans les plateformes de marketing restent alignés sur les données d’identité actuelles.

<div id="how-it-works">
  ## Comment ça fonctionne
</div>

1. Auth0 publie un événement lorsqu’un profil d’utilisateur est créé, mis à jour ou supprimé.
2. Votre Event Stream achemine cet événement vers une destination (webhook, AWS EventBridge ou Auth0 Action).
3. Votre gestionnaire mappe l’utilisateur Auth0 à un enregistrement dans le système externe et applique la mise à jour appropriée.

Les types d’événements suivants sont pertinents pour la corrélation des données :

| Type d’événement | Moment où il se déclenche                            |
| ---------------- | ---------------------------------------------------- |
| `user.created`   | Un nouveau profil d’utilisateur est créé dans Auth0. |
| `user.updated`   | Un profil d’utilisateur existant est modifié.        |
| `user.deleted`   | Un profil d’utilisateur est supprimé d’Auth0.        |

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, assurez-vous d’avoir :

* un tenant Auth0 avec Events activé. Pour en savoir plus sur la disponibilité selon votre plan, consultez [Créer un Event Stream](/docs/fr-CA/customize/events/create-an-event-stream).
* un Event Stream actif abonné aux types d’événements dont vous avez besoin. Pour en savoir plus, consultez [Créer un Event Stream](/docs/fr-CA/customize/events/create-an-event-stream).
* des identifiants d’API pour le système externe que vous comptez mettre à jour (par exemple, une clé d’API de CRM ou un jeton OAuth).

<div id="implement-identity-correlation">
  ## Corrélation des identités avec Implement
</div>

Les sections ci-dessous montrent comment corréler des événements Auth0 avec une plateforme CRM. Les fonctions gestionnaires sont les mêmes, quelle que soit la destination de votre Event Stream. La section [Acheminer les événements par type](#route-events-by-type) montre comment acheminer les événements vers des destinations webhook et Auth0 Action.

<div id="map-auth0-users-to-external-records">
  ### Mapper les utilisateurs Auth0 à des enregistrements externes
</div>

La plupart des systèmes externes identifient les contacts par adresse courriel ou par un ID externe stocké dans `app_metadata` d’Auth0. Définissez une fonction de recherche qui permet de retrouver l’enregistrement externe correspondant à un utilisateur Auth0 donné.

```javascript wrap lines
async function findCrmContact(user) {
    const externalId = user.app_metadata?.crm_contact_id;

    if (externalId) {
        return externalId;
    }

    // Recourir à la recherche par courriel
    const response = await fetch(
        `https://api.example-crm.com/contacts?email=${encodeURIComponent(user.email)}`,
        { headers: { "Authorization": `Bearer ${CRM_API_TOKEN}` } }
    );

    const results = await response.json();
    return results.length > 0 ? results[0].id : null;
}
```

<div id="handle-usercreated">
  ### Gérer user.created
</div>

Lorsqu’un nouvel utilisateur est créé dans Auth0, créez le contact correspondant dans votre système externe.

```javascript wrap lines
async function handleUserCreated(user) {
    const response = await fetch("https://api.example-crm.com/contacts", {
        method: "POST",
        headers: {
            "Authorization": `Bearer ${CRM_API_TOKEN}`,
            "Content-Type": "application/json"
        },
        body: JSON.stringify({
            email: user.email,
            name: user.name,
            phone: user.phone_number,
            externalId: user.user_id
        })
    });

    if (!response.ok) {
        throw new Error(`CRM create failed: ${response.status}`);
    }
}
```

<div id="handle-userupdated">
  ### Traiter user.updated
</div>

Lorsqu’un profil d’utilisateur est modifié, mettez à jour le contact CRM correspondant avec les nouvelles données.

```javascript wrap lines
async function handleUserUpdated(user) {
    const contactId = await findCrmContact(user);

    if (!contactId) {
        console.log(`No CRM contact found for user ${user.user_id}, skipping.`);
        return;
    }

    await fetch(`https://api.example-crm.com/contacts/${contactId}`, {
        method: "PATCH",
        headers: {
            "Authorization": `Bearer ${CRM_API_TOKEN}`,
            "Content-Type": "application/json"
        },
        body: JSON.stringify({
            email: user.email,
            name: user.name,
            phone: user.phone_number
        })
    });
}
```

<div id="handle-userdeleted">
  ### Traiter user.deleted
</div>

Lorsqu’un utilisateur est supprimé d’Auth0, désactivez ou marquez le dossier correspondant dans votre système externe.

```javascript wrap lines
async function handleUserDeleted(user) {
    const contactId = await findCrmContact(user);

    if (!contactId) {
        return;
    }

    await fetch(`https://api.example-crm.com/contacts/${contactId}`, {
        method: "PATCH",
        headers: {
            "Authorization": `Bearer ${CRM_API_TOKEN}`,
            "Content-Type": "application/json"
        },
        body: JSON.stringify({ status: "deactivated" })
    });
}
```

<div id="route-events-by-type">
  ### Acheminer les événements par type
</div>

Utilisez un routeur principal pour acheminer chaque événement vers le bon gestionnaire. Les exemples ci-dessous montrent comment acheminer les événements vers des destinations webhook et Auth0 Action.

<Tabs>
  <Tab title="Webhook">
    ```javascript wrap lines
    app.post("/webhook", async (req, res) => {
        const { type, data } = req.body;
        const user = data.object;

        try {
            switch (type) {
                case "user.created":
                    await handleUserCreated(user);
                    break;
                case "user.updated":
                    await handleUserUpdated(user);
                    break;
                case "user.deleted":
                    await handleUserDeleted(user);
                    break;
                default:
                    console.log(`Unhandled event type: ${type}`);
            }

            res.sendStatus(204);
        } catch (err) {
            console.error("Error processing event:", err);
            res.status(500).json({ error: "Internal server error" });
        }
    });
    ```

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Renvoyez une réponse HTTP `2XX` le plus rapidement possible. Si vos appels d’API externes sont lents, placez l’événement dans une file d’attente interne et traitez-le de façon asynchrone. Pour en savoir plus, consultez [Pratiques exemplaires pour Events](/docs/fr-CA/customize/events/events-best-practices).
    </Callout>
  </Tab>

  <Tab title="Auth0 Action">
    ```javascript wrap lines
    exports.onExecuteEventStream = async (event, api) => {
        const eventType = event.message.type;
        const user = event.message.data.object;

        switch (eventType) {
            case "user.created":
                await handleUserCreated(user);
                break;
            case "user.updated":
                await handleUserUpdated(user);
                break;
            case "user.deleted":
                await handleUserDeleted(user);
                break;
            default:
                console.log(`Unhandled event type: ${eventType}`);
        }
    };
    ```

<div id="handle-edge-cases">
  ## Gérer les cas particuliers
</div>

Lors de la mise en correspondance des données d’identité entre les systèmes, tenez compte des éléments suivants :

* **Enregistrements externes manquants.** Un événement `user.updated` peut être reçu pour un utilisateur qui n’existe pas encore dans votre système externe. Décidez s’il faut créer l’enregistrement ou le consigner dans un journal pour vérification manuelle.
* **Limites de requêtes sur les API externes.** Si Auth0 envoie une rafale d’événements (par exemple, pendant une importation en bloc), votre gestionnaire peut dépasser les limites de requêtes de l’API externe. Utilisez une file d’attente asynchrone avec temporisation progressive pour rester dans les limites.
* **Données partielles.** Les événements `user.updated` n’incluent pas tous les champs du profil. Appliquez uniquement les champs présents dans le payload de l’événement afin d’éviter d’écraser des données par des valeurs vides.

<div id="example-correlate-with-hubspot-crm">
  ## Exemple : Établir la corrélation avec HubSpot CRM
</div>

L’Action Auth0 suivante illustre un gestionnaire de corrélation complet qui crée, met à jour et supprime des contacts dans HubSpot CRM. L’Action utilise l’API Contacts de HubSpot pour rechercher des contacts existants par courriel et effectuer l’opération appropriée.

```javascript wrap lines
exports.onExecuteEventStream = async (event, api) => {
    const eventType = event.message.type;
    const userData = event.message.data.object;
    const HUBSPOT_TOKEN = event.secrets.HUBSPOT_TOKEN;
    const BASE_URL = "https://api.hubapi.com/crm/v3/objects/contacts";

    const headers = {
        "Authorization": `Bearer ${HUBSPOT_TOKEN}`,
        "Content-Type": "application/json"
    };

    switch (eventType) {
        case "user.created": {
            const response = await fetch(BASE_URL, {
                method: "POST",
                headers,
                body: JSON.stringify({
                    properties: {
                        email: userData.email,
                        firstname: userData.given_name,
                        lastname: userData.family_name
                    }
                })
            });

            if (!response.ok) {
                throw new Error(`HubSpot create failed: ${response.status}`);
            }
            break;
        }
        case "user.updated": {
            // Rechercher un contact existant par courriel
            const searchResponse = await fetch(`${BASE_URL}/search`, {
                method: "POST",
                headers,
                body: JSON.stringify({
                    filterGroups: [{
                        filters: [{
                            propertyName: "email",
                            operator: "EQ",
                            value: userData.email
                        }]
                    }]
                })
            });

            const searchResult = await searchResponse.json();

            if (searchResult.total > 0) {
                const contactId = searchResult.results[0].id;
                await fetch(`${BASE_URL}/${contactId}`, {
                    method: "PATCH",
                    headers,
                    body: JSON.stringify({
                        properties: {
                            firstname: userData.given_name,
                            lastname: userData.family_name
                        }
                    })
                });
            }
            break;
        }
        case "user.deleted": {
            const searchResponse = await fetch(`${BASE_URL}/search`, {
                method: "POST",
                headers,
                body: JSON.stringify({
                    filterGroups: [{
                        filters: [{
                            propertyName: "email",
                            operator: "EQ",
                            value: userData.email
                        }]
                    }]
                })
            });

            const searchResult = await searchResponse.json();

            if (searchResult.total > 0) {
                const contactId = searchResult.results[0].id;
                await fetch(`${BASE_URL}/${contactId}`, {
                    method: "DELETE",
                    headers
                });
            }
            break;
        }
        default:
            console.log(`Unhandled event type: ${eventType}`);
    }
};
```

Pour stocker la clé API HubSpot de façon sécuritaire, ajoutez un secret nommé `HUBSPOT_TOKEN` dans l’éditeur d’Actions. Pour en savoir plus, consultez [Secrets des Actions](/docs/fr-CA/customize/actions/write-your-first-action#add-a-secret).

<div id="verify-correlation">
  ## Vérifier la corrélation
</div>

Après avoir déployé votre gestionnaire, créez ou mettez à jour un utilisateur test dans Auth0, puis confirmez ce qui suit :

1. L’enregistrement correspondant dans votre système externe reflète la modification.
2. Supprimez l’utilisateur test dans Auth0. Confirmez que l’enregistrement externe est désactivé ou supprimé.
3. Consultez les journaux de votre gestionnaire pour repérer toute erreur ou tout événement ignoré.

Pour en savoir plus sur la mise à l’essai des Event Streams, consultez [Tests d’événements, observabilité et reprise après incident](/docs/fr-CA/customize/events/event-testing-observability-and-failure-recovery).
