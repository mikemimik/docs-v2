<Warning>
  La date de fin de vie (EOL) de Rules et Hooks est fixée au **18 novembre 2026**, et ils ne sont plus offerts aux nouveaux tenants créés à compter du **16 octobre 2023**. Les tenants existants ayant des Hooks actifs conserveront l’accès au produit Hooks jusqu’à sa fin de vie.

  Nous vous recommandons fortement d’utiliser Actions pour étendre Auth0. Avec Actions, vous avez accès à des informations de type détaillées, à de la documentation intégrée et à des packages `npm` publics, et vous pouvez connecter des intégrations externes qui bonifient votre expérience globale d’extensibilité. Pour en savoir plus sur ce qu’offre Actions, consultez [Comprendre le fonctionnement d’Auth0 Actions](/docs/fr-CA/customize/actions/actions-overview).

  Pour faciliter votre migration, nous proposons des guides qui vous aideront à [migrer de Rules vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions) et à [migrer de Hooks vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions). Nous avons aussi une page dédiée, [Passer à Actions](https://auth0.com/extensibility/movetoactions), qui présente des comparatifs de fonctionnalités, [une démo d’Actions](https://www.youtube.com/watch?v=UesFSY1klrI) et d’autres ressources pour vous accompagner dans votre démarche de migration.

  Pour en savoir plus sur la dépréciation de Rules et Hooks, consultez notre billet de blogue : [Préparer la fin de vie de Rules et Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Comme nous prévoyons retirer les fonctions Rules et Hooks en 2026, vous ne devriez créer de nouvelles Rules ou de nouveaux Hooks que dans votre environnement de développement, et uniquement pour tester la migration vers Actions.

  Pour savoir comment migrer vos Rules vers Actions, consultez [Migrer de Rules vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions). Pour savoir comment migrer vos Hooks vers Actions, consultez [Migrer de Hooks vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Vous pouvez mettre à jour les Hook Secrets ajoutés à un Hook donné à l’aide du Dashboard ou de l’<Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

Les Hook Secrets peuvent aussi être importés et exportés à l’aide de l’outil Deploy Command-Line Interface (CLI). Pour en savoir plus, consultez [l’outil Deploy CLI](/docs/fr-CA/deploy-monitor/deploy-cli-tool).

<div id="update-hook-secrets-using-the-dashboard">
  ## Mettre à jour les Hook Secrets dans le Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/), puis cliquez sur l’icône en forme de crayon à côté du Hook que vous voulez modifier.
2. Dans l’éditeur de Hook, cliquez sur l’icône de clé à molette, puis sur **Secrets**.
3. Cliquez sur l’icône en forme de crayon et de papier à côté de la valeur du Hook Secret que vous voulez modifier.
4. Modifiez le nom et/ou la valeur du Hook Secret sélectionné, puis cliquez sur **Save**.

<div id="update-hook-secrets-using-the-management-api">
  ## Mettre à jour les Hook Secrets à l’aide de la Management API
</div>

Effectuez une requête `PATCH` vers l’[endpoint Update Hook Secrets](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_secrets). Assurez-vous de remplacer les valeurs de l’espace réservé `HOOK_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre hook et votre <Tooltip tip="Access Token : information d’identification d’autorisation, sous forme de chaîne opaque ou de JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">Access Token</Tooltip> de la Management API, respectivement.

| Valeur                  | Description                                                                                                                                                                                                     |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN` | Access Tokens pour la Management API avec le [scope](/docs/fr-CA/glossary?term=scope) `update:hooks`. Pour en savoir plus, consultez [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |
| `HOOK_ID`               | ID du hook pour lequel vous voulez mettre à jour des secrets.                                                                                                                                                   |
| `HOOK_SECRET_KEY`       | Nom du secret que vous voulez mettre à jour pour le hook indiqué. Cet endpoint accepte un objet de paires clé-valeur.                                                                                           |
| `HOOK_SECRET_VALUE`     | Valeur du secret que vous voulez mettre à jour pour le hook indiqué. Cet endpoint accepte un objet de paires clé-valeur.                                                                                        |

<Warning>
  Lors de la récupération des Hook Secrets configurés pour un Hook donné, les valeurs contiendront le texte d’espace réservé suivant : `_VALUE_NOT_SHOWN`. Veillez à ne pas renvoyer ce texte d’espace réservé lors d’une mise à jour, sinon la valeur réelle du secret sera écrasée.
</Warning>
