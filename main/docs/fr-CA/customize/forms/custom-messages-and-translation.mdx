Vous pouvez personnaliser les messages de Form afin d’offrir à vos utilisateurs une expérience Form unique. De plus, vous pouvez traduire les composants du formulaire, les erreurs et d’autres messages personnalisés dans d’autres langues pour offrir une expérience localisée.

<div id="messages">
  ## Messages
</div>

Dans l’éditeur de formulaires, vous pouvez personnaliser les messages d’erreur et créer d’autres messages personnalisés pour vos utilisateurs.

<div id="error-messages">
  ### Messages d’erreur
</div>

Vous pouvez créer des messages d’erreur personnalisés pour remplacer les messages d’erreur par défaut de vos formulaires.

Pour créer des messages d’erreur personnalisés, suivez les étapes ci-dessous :

1. Dans l’Auth0 Dashboard, accédez à **Actions &gt; Forms**.
2. Dans le menu de gauche de l’éditeur de formulaires, sélectionnez **Messages**.
3. Sélectionnez l’onglet **Errors****.**
4. Saisissez vos messages d’erreur personnalisés dans les champs appropriés pour remplacer les messages par défaut.
5. Sélectionnez **Publish** pour enregistrer.

<div id="custom-messages">
  ### Messages personnalisés
</div>

Vous pouvez créer des messages personnalisés à ajouter aux messages par défaut afin d’offrir une expérience Form encore plus personnalisée. Par exemple, vous pouvez ajouter un message personnalisé qui s’affiche aux utilisateurs lorsqu’une action renvoie une erreur dans un flow.

<div id="create-custom-messages">
  #### Créer des messages personnalisés
</div>

Pour créer des messages personnalisés, suivez les étapes ci-dessous :

1. Dans le menu de gauche de l’éditeur de formulaires, sélectionnez **Messages**.
2. Sélectionnez l’onglet **Custom**. Ensuite, sélectionnez **Add New Item**.
3. À gauche, ajoutez une clé unique pour votre message**.** Ensuite**, saisissez** votre message personnalisé dans le champ à droite.
4. Sélectionnez **Publish** pour enregistrer.

<Frame>![Dashboard > Forms > Custom messages and translations > Message personnalisé](/docs/images/cdy7uua7fh8z/1upBOwWqntPPQF8fqGeN6T/a65674d05c823c5fbe703265558693e4/custom-message.png)</Frame>

<div id="add-custom-messages-to-your-forms">
  #### Ajoutez des messages personnalisés à vos formulaires
</div>

Pour faire référence à vos messages personnalisés dans vos formulaires, utilisez la fonction `{{t('custom_unique_key')}}` avec la clé unique du message personnalisé.

<Frame>![Dashboard > Forms > Custom messages and translations > fonction de traduction](/docs/images/cdy7uua7fh8z/45p4V7w5gw7DQZyJ08ECSH/d040720d9aa19f6e97e21defcd428e55/translation-function.png)</Frame>

<div id="translations">
  ## Translations
</div>

Vous pouvez utiliser la fonctionnalité Translations pour offrir une expérience utilisateur internationale dans vos formulaires. Cette fonctionnalité s’appuie sur la langue du navigateur de l’utilisateur pour déterminer la langue appropriée pour vos formulaires.
Par défaut, des traductions de certains messages d’erreur standards sont offertes en espagnol. Toutefois, pour assurer une expérience utilisateur de qualité, vous pouvez définir les traductions de vos composants de formulaire, de vos erreurs et de vos messages personnalisés dans la langue de votre choix.

Pour activer les traductions, suivez les étapes ci-dessous :

1. Dans l’Auth0 Dashboard, accédez à **Actions &gt; Forms**.
2. Sélectionnez **Translations** dans le menu de gauche de l’éditeur de formulaires.
3. Cochez **Enable additional languages**.
4. Sélectionnez les langues souhaitées dans le menu déroulant **Additional languages**.

<div id="translate-messages">
  ### Traduire les messages
</div>

Avec l’éditeur de langue, vous pouvez définir des traductions pour les composants de votre formulaire, les erreurs et d’autres messages personnalisés.

Pour définir vos traductions, suivez les étapes ci-dessous :

1. Sélectionnez la langue souhaitée dans le menu déroulant Select language.
2. Sélectionnez l’un des onglets suivants pour commencer à définir vos traductions :

   1. **Components**
   2. **Errors**
   3. **Custom**
3. Pour chaque champ disponible, ajoutez les traductions nécessaires.
4. Sélectionnez **Publish** pour enregistrer.
