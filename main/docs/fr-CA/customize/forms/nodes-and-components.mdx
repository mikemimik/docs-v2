Forms vous permet d’ajouter des nœuds et des composants, comme des champs, des blocs et des widgets, avec lesquels vos utilisateurs peuvent interagir dans le formulaire. Vous pouvez ensuite réutiliser les données saisies dans ces composants dans vos formulaires et vos flows.

<div id="form-nodes">
  ## Nœuds de formulaire
</div>

Il existe cinq types de nœuds de formulaire : Start, Step, flux, Router et écran de fin. Ces nœuds jouent un rôle bien précis dans la structure du formulaire.

<Frame>![Dashboard >  Forms > Form Editor](/docs/images/cdy7uua7fh8z/5V4GsHxxC2cjxx1aODCxPd/309850c72bbce224e7a8d3f5af34114c/form-editor-preview.png)</Frame>

<div id="step-node">
  ## nœud d’étape
</div>

Les nœuds d’étape représentent visuellement le form. Ils peuvent contenir n’importe quel nombre de composants, comme des champs, des blocs et des widgets, dans lesquels l’utilisateur peut saisir ses données.

<Frame>![Dashboard > Foms > Step node](/docs/images/cdy7uua7fh8z/2tbkcXkVon0BawCvYgnvIz/e2ce566320af18a6b08702f04f763126/form-step-node.png)</Frame>

<div id="field-components">
  ### Composants de champ
</div>

Les champs sont des composants d’interface utilisateur permettant aux utilisateurs de saisir leurs renseignements dans Forms.

<Frame>![Dashboard > Forms > Fields](/docs/images/cdy7uua7fh8z/6fuBdJcbDybzcVaiByUkoQ/2ad29c8dbbcc6ddf850f5efc004a75e0/forms-fields.png)</Frame>

<div id="field-settings">
  #### Paramètres du champ
</div>

Les champs comportent un certain nombre de paramètres par défaut que vous pouvez configurer selon vos besoins.

Les paramètres de champ par défaut sont :

| Paramètres        | Description                                   |
| ----------------- | --------------------------------------------- |
| ID                | L’identifiant unique                          |
| Libellé           | Afficher ou masquer un libellé                |
| Obligatoire       | La valeur de saisie est obligatoire           |
| Texte d’aide      | Afficher un texte d’aide                      |
| Texte indicatif   | Texte visible pour l’utilisateur              |
| Valeur par défaut | Valeur par défaut visible pour l’utilisateur  |
| Temporaire        | Activer ou désactiver le masquage des données |

<div id="field-validation-rules">
  #### Règles de validation des champs
</div>

| Paramètres                    | Description                               |
| ----------------------------- | ----------------------------------------- |
| Longueur min. / Longueur max. | Exige une longueur pour la valeur saisie. |
| Valeur min. / Valeur max.     | Exige une valeur numérique.               |

<div id="available-fields-in-forms">
  #### Champs disponibles dans Forms
</div>

Les champs disponibles, ainsi que leurs paramètres propres et leurs types de données en sortie, sont :

<AccordionGroup>
  <Accordion title="Champ de texte">
    La saisie standard permet à l’utilisateur de saisir n’importe quelle chaîne de caractères.

    ##### Paramètres du champ de texte

    Le paramètre du champ de texte est le suivant :

    | Paramètres | Description                                     |
    | ---------- | ----------------------------------------------- |
    | Multiligne | Active la saisie de texte sur plusieurs lignes. |

    ##### Valeur de sortie du champ de texte

    Le type de données de la valeur de sortie du champ de texte est `string`.

    ```json lines
    {
      "text_field_id": "Auth0"
    }
    ```
  </Accordion>

  <Accordion title="Champ de courriel">
    La saisie standard permet à l’utilisateur d’entrer des adresses courriel sous forme de chaîne de caractères.

    ##### Valeur de sortie du champ Courriel

    Le type de données de la valeur de sortie du champ Courriel est `string`.

    ```json lines
    {
      "email_field_id": "username@domain.com"
    }
    ```
  </Accordion>

  <Accordion title="Champ de téléphone">
    La saisie standard permet à l’utilisateur d’entrer des numéros de téléphone.

    ##### Paramètres du champ de téléphone

    Les paramètres du champ de téléphone sont les suivants :

    | Paramètres                | Description                                                                                                               |
    | ------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
    | Sélecteur de pays         | Activez ou désactivez le sélecteur de pays. Par défaut, il affiche le pays correspondant à l’adresse IP de l’utilisateur. |
    | Texte indicatif du filtre | Texte visible par l’utilisateur dans la fenêtre de recherche de pays.                                                     |

    ##### Valeur de sortie du champ de téléphone

    Le type de données de la valeur de sortie du champ de téléphone est soit une `string`, soit, si le sélecteur de pays est activé, un `object`.

    Valeur de sortie du champ de téléphone lorsque le type de données est une `string` :

    ```json lines
    {
      "phone_field_id": "8005550175"
    }
    ```

    Le champ phone renvoie une valeur de type `object` :

    ```json lines
    {
      "phone_field_id": {
        "national_number": "8005550175",
        "national_format": "(800) 555-0175",
        "international_number": "+18005550175",
        "international_format": "+1 800-555-0175",
        "country_code_iso": "US",
        "country_code_number": "1"
      }
    }
    ```
  </Accordion>

  <Accordion title="Champ de sélection">
    Champ standard qui permet à un utilisateur de sélectionner une ou plusieurs valeurs.

    ##### Paramètres du champ Choice

    Les paramètres du champ Choice sont :

    | Paramètres                       | Description                                                                    |
    | -------------------------------- | ------------------------------------------------------------------------------ |
    | Choix multiple                   | Active plusieurs options de réponse.                                           |
    | Autre option                     | Permet aux utilisateurs d’entrer leur propre option.                           |
    | Avancé &gt; Valeurs internes     | Attribue une valeur interne à chaque option.                                   |
    | Avancé &gt; Modification en bloc | Permet de modifier en bloc les étiquettes et les valeurs internes des options. |

    ##### Valeur de sortie du champ Choice

    Le type de données de la valeur de sortie du champ Choice est soit une `string` ou, si le choix multiple est activé, un `array of strings` .

    La valeur de sortie du champ Choice en tant que `string` :

    ```json lines
    {
      "choice_field_id": "Option A"
    }
    ```

    La valeur du type de données de sortie du champ choice est un  `array of strings`:

    ```json lines
    {
      "choice_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ Cartes">
    Champ standard qui permet à un utilisateur de sélectionner une ou plusieurs valeurs d’image.

    ##### Paramètres du champ Cards

    Les paramètres du champ Cards sont :

    | Paramètres                   | Description                                  |
    | ---------------------------- | -------------------------------------------- |
    | Choix multiples              | Active plusieurs options de sélection.       |
    | Masquer les libellés         | Masque les libellés des cartes.              |
    | Avancé &gt; Valeurs internes | Attribue une valeur interne à chaque option. |

    ##### Valeur de sortie du champ Cards

    Le type de données de la valeur de sortie du champ Cards est soit une `string` ou, si le choix multiple est activé, un `array of strings`.

    La valeur de sortie du champ Cards comme `string` :

    ```json lines
    {
      "card_field_id": "Option A"
    }
    ```

    Le champ cards renvoie une valeur de type `array of strings` :

    ```json lines
    {
      "card_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ URL">
    Une entrée standard permet à l’utilisateur de saisir une URL.

    ##### Valeur de sortie du champ URL

    Le type de données de la valeur de sortie du champ URL est une chaîne de caractères :

    ```json lines
    {
      "url_field_id": "https://auth0.com"
    }
    ```
  </Accordion>

  <Accordion title="Champ de mot de passe">
    Champ de saisie standard permettant à l’utilisateur d’entrer un mot de passe ou une valeur secrète.

    ##### Paramètres du champ de mot de passe

    Les paramètres du champ de mot de passe sont les suivants :

    | Paramètres                          | Description                                                                                                                                                                                                                                                                |
    | ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Exiger un mot de passe complexe     | Les mots de passe doivent contenir une lettre majuscule, une lettre minuscule, un symbole et un chiffre.                                                                                                                                                                   |
    | Appliquer les directives du NIST    | Le mot de passe doit contenir au moins 8 caractères et ne peut pas correspondre à un mot de passe figurant dans des corpus de fuites de données antérieures, à un mot du dictionnaire, à une suite de caractères répétitifs ou séquentiels, ni à des mots liés au contenu. |
    | Ajouter un indicateur de robustesse | Un indicateur graphique avec une échelle rouge, jaune et verte s’affiche sous le champ de mot de passe pour indiquer la robustesse du mot de passe saisi.                                                                                                                  |
    | Hachage                             | Sélectionnez un algorithme pour hacher la valeur saisie directement dans le navigateur.                                                                                                                                                                                    |

    ##### Valeur de sortie du champ de mot de passe

    Le type de données de la valeur de sortie du champ de mot de passe est un `string` masqué :

    ```json lines
    {
      "password_field_id": "███"
    }
    ```
  </Accordion>

  <Accordion title="Champ de paiement">
    Champ de saisie standard qui permet à l’utilisateur d’entrer des renseignements de paiement au moyen d’un fournisseur de paiement. Les paiements sont traités avant le nœud de fin.

    ##### Paramètres du champ de paiement

    Les paramètres du champ de paiement sont :

    | Paramètres          | Description                                                                                                                              |
    | ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
    | Type de paiement    | Sélectionnez un paiement ponctuel (Charge) ou récurrent (Subscription).                                                                  |
    | Montant             | Indiquez le montant d’un paiement ponctuel (Charge).                                                                                     |
    | Devise              | Indiquez la devise d’un paiement ponctuel (Charge).                                                                                      |
    | Action d’abonnement | Sélectionnez l’option permettant de créer un abonnement existant ou de le mettre à jour.                                                 |
    | Action du client    | Sélectionnez l’option permettant de créer un client existant ou de le mettre à jour.                                                     |
    | Options d’affichage | Affichez et modifiez les libellés du numéro de carte, de la date d’expiration et du code de sécurité, ainsi que les sceaux de confiance. |

    ##### Valeur de sortie du champ de paiement

    Le type de données de la valeur de sortie du champ de paiement est soit une chaîne de caractères, soit un objet pour les flux après la soumission.

    Le type de données de la valeur de sortie du champ de paiement au format `string` :

    ```json lines
    {
      "payment_field_id": "pm_1P19e..."
    }
    ```

    Type de données de la valeur de sortie du champ de paiement pour les abonnements sous forme d’`object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "price_ids": [
          "price_1ONHR..."
        ],
        "subscription_id": "sub_1P1A...",
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```

    Le type de données de la valeur de sortie du champ de paiement pour un débit en tant qu’`object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "amount": 100,
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```
  </Accordion>

  <Accordion title="Champ légal">
    Entrée standard permettant d’ajouter un texte juridique et une case à cocher.

    ##### Valeur de sortie du champ juridique

    Le type de données de la valeur de sortie du champ juridique est `boolean`.

    ```json lines
    {
      "legal_field_id": true
    }
    ```
  </Accordion>

  <Accordion title="Champ déroulant">
    Champ standard permettant à un utilisateur de sélectionner une ou plusieurs valeurs dans une liste déroulante.

    ##### Paramètres du champ de liste déroulante

    Les paramètres du champ de liste déroulante sont :

    | Paramètres                         | Description                                             |
    | ---------------------------------- | ------------------------------------------------------- |
    | Sélection multiple                 | Active plusieurs options de choix.                      |
    | Advanced &gt; Valeurs internes     | Attribue une valeur interne à chaque option.            |
    | Advanced &gt; Modification en bloc | Modifie en bloc les étiquettes et les valeurs internes. |

    ##### Valeur de sortie du champ de liste déroulante

    Le type de données de la valeur de sortie du champ de liste déroulante est soit une `string` ou, si le choix multiple est activé, un `array of strings`.

    Valeur de sortie du champ de liste déroulante en tant que `string` :

    ```json lines
    {
      "dropdown_field_id": "Option A"
    }
    ```

    Le champ de liste déroulante renvoie une valeur de type `tableau de chaînes` :

    ```json lines
    {
      "dropdown_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ numérique">
    Champ de saisie standard permettant à l’utilisateur d’entrer des valeurs numériques.

    ##### Valeur de sortie du champ Number

    Le type de donnée de la valeur de sortie du champ Number est `number`.

    ```json lines
    {
      "number_field_id": 123
    }
    ```
  </Accordion>

  <Accordion title="Champ de date et d’heure">
    Champ standard qui permet à l’utilisateur de saisir une date ou une heure.

    ##### Paramètres du champ date/heure

    Les paramètres du champ date/heure comprennent :

    | Clé    | Description                           |
    | ------ | ------------------------------------- |
    | Format | Sélectionnez le format Date ou Heure. |

    ##### Valeur de sortie du champ date/heure

    Pour une date, la valeur du type de données de sortie du champ date/heure est une `string` :

    ```json lines
    {
      "date_time_field_id": "2023-04-11"
    }
    ```

    La valeur du type de données de sortie du champ date/heure pour l’heure est de type `string` :

    ```json lines
    {
      "date_time_field_id": "23:15"
    }
    ```
  </Accordion>

  <Accordion title="Champ booléen">
    Entrée standard permettant à l’utilisateur de saisir des valeurs vraies ou fausses.

    ##### Valeur de sortie du champ booléen

    Le type de données de la valeur de sortie du champ booléen est de type booléen.

    ```json lines
    {
      "boolean_field_id": true
    }
    ```
  </Accordion>

  <Accordion title="Champ personnalisé">
    Champ standard que vous pouvez personnaliser afin de créer un champ sur mesure.

    ##### Paramètres du champ personnalisé

    Les paramètres du champ personnalisé sont :

    | Paramètres  | Description                                                                                                                                         |
    | ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Params      | Ajoutez des paires clé-valeur que vous pourrez référencer dans le code source du champ personnalisé.                                                |
    | Code source | Ajoutez votre code Javascript dans le champ personnalisé.                                                                                           |
    | Schéma JSON | Par défaut, le champ personnalisé accepte n’importe quel format de valeur. Cependant, vous pouvez utiliser un schéma JSON pour valider les valeurs. |

    Pour en savoir plus, consultez [Composants de champ personnalisé pour Forms](/docs/fr-CA/customize/forms/custom-field-components).
  </Accordion>
</AccordionGroup>

<div id="blocks-components">
  ### Composants bloc
</div>

Les blocs sont des composants UI qui ajoutent des fonctionnalités à vos formulaires, mais ne servent pas à recueillir de renseignements sur les utilisateurs.

<Frame>![Dashboard > Forms > Components > Blocks](/docs/images/cdy7uua7fh8z/7IijftJnuwcevL4bdHiaKw/f0376ca75303bb58d0df87e6af75e347/forms-block.png)</Frame>

<div id="available-blocks-in-forms">
  #### Blocs disponibles dans Forms
</div>

Les blocs disponibles sont les suivants :

<AccordionGroup>
  <Accordion title="Bloc de bouton Suivant">
    Un bouton qui permet aux utilisateurs de passer au nœud de formulaire suivant.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      * Avant de continuer, les règles de validation des champs sont appliquées et vérifiées côté client et côté serveur. Si elles échouent, un message d’erreur s’affiche pour l’utilisateur.
      * Si le nœud suivant est un nœud de flux, un indicateur de chargement s’affiche jusqu’à la fin du flux. Si le flux échoue ou renvoie un message d’erreur personnalisé, un message d’erreur s’affiche.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton Précédent">
    Un bouton qui permet aux utilisateurs de revenir au nœud d’étape précédent.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un utilisateur sélectionne le bouton précédent, les règles de validation des champs sont exécutées de nouveau, même si aucune modification n’a été apportée aux données saisies avant de continuer.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton de saut">
    Un bouton qui permet aux utilisateurs d’ignorer l’étape actuelle et d’accéder directement au nœud de destination.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Lorsque l’étape actuelle est ignorée, aucune validation n’est effectuée et aucune information saisie à cette étape n’est recueillie.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton de renvoi">
    Un composant qui permet aux utilisateurs de relancer un flux afin de générer et d’envoyer un nouveau code OTP

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un utilisateur sélectionne le bouton de renvoi, le flux associé génère et envoie un nouveau code OTP.
    </Callout>

    | Paramètres                   | Description                                                                                                                                                                                                                                                                                                                                                   |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Alignement du texte          | Sélectionnez l’alignement du texte : à gauche, au centre ou à droite                                                                                                                                                                                                                                                                                          |
    | Texte                        | Texte par défaut affiché lorsque les utilisateurs sélectionnent le bouton de renvoi                                                                                                                                                                                                                                                                           |
    | Texte du bouton              | Texte du bouton                                                                                                                                                                                                                                                                                                                                               |
    | Texte d’attente              | Texte affiché lorsque les utilisateurs ont sélectionné le bouton de renvoi. Remplace les paramètres `Text` et `Button text`. Utilisez la variable `“{{remaining_seconds}}”` pour afficher dynamiquement le nombre de secondes restantes avant que le bouton d’envoi soit de nouveau activé. Par exemple : « Renvoyer dans `{{remaining_seconds}}` secondes ». |
    | Flux                         | Flux exécuté après que l’utilisateur a sélectionné le bouton de renvoi.                                                                                                                                                                                                                                                                                       |
    | Nombre maximal de tentatives | Nombre maximal de fois où l’utilisateur peut sélectionner le bouton de renvoi                                                                                                                                                                                                                                                                                 |
    | Temps d’attente              | Temps d’attente entre les tentatives, en secondes.                                                                                                                                                                                                                                                                                                            |
  </Accordion>

  <Accordion title="Bloc de texte enrichi">
    Un bloc de texte enrichi pour personnaliser le nœud d’étape avec des informations supplémentaires.
  </Accordion>

  <Accordion title="Bloc séparateur">
    Une ligne pour séparer différentes sections du nœud d’étape. Elle peut contenir un court texte.
  </Accordion>

  <Accordion title="Bloc HTML">
    Un bloc HTML pour créer votre UI personnalisée.
  </Accordion>

  <Accordion title="Bloc d’image">
    Un bloc d’image pour personnaliser l’étape en y ajoutant des images.
  </Accordion>
</AccordionGroup>

<div id="widgets-components">
  ### Composants Widgets
</div>

Les widgets sont des composants prédéfinis dotés d’intégrations tierces qui ajoutent une logique côté client et côté serveur à vos formulaires.

<Frame>![Dashboard > Forms > Components > Widget](/docs/images/cdy7uua7fh8z/3J0IuWLP7utOPKSHc82IE9/a523afe0c240c63f714ef5fb102f8dab/forms-widget.png)</Frame>

<div id="available-widgets-in-forms">
  #### Widgets disponibles dans Forms
</div>

Les widgets disponibles sont :

<AccordionGroup>
  <Accordion title="Widget d’adresse Google">
    Un widget qui permet à l’utilisateur de saisir et de valider son adresse.

    ##### Paramètres du widget d’adresse Google

    Les paramètres du widget d’adresse Google comprennent :

    | Paramètres | Description                                                       |
    | ---------- | ----------------------------------------------------------------- |
    | Clé API    | Nécessite une clé API Google Maps pour authentifier les requêtes. |

    ##### Valeur de sortie du widget d’adresse Google

    Le type de données de la valeur de sortie du widget d’adresse Google est un `object`.

    ```json lines
    {
      "google_address_widget_id": {
        "type": "Feature",
        "geometry": {
          "type": "Point",
          "coordinates": [
            -73.9654415,
            40.8054491
          ]
        },
        "properties": {
          "geocoding": {
            "type": "house",
            "housenumber": "2880",
            "street": "Broadway",
            "city": "New York",
            "county": "New York County",
            "state": "New York",
            "country": "United States",
            "postcode": "10025"
          }
        }
      }
    }
    ```
  </Accordion>

  <Accordion title="Google reCAPTCHA">
    Un widget qui aide à protéger votre site Web contre le spam et les abus en vérifiant que l’utilisateur est un être humain et non un robot.

    Le widget prend en charge :

    * Basé sur un score (v3)
    * Défi (v2)

      * Badge reCAPTCHA invisible

    Différences entre `v2` et `v3` :

    * `v2` exige une interaction de l’utilisateur avec des défis visibles, comme cliquer sur une case à cocher ou résoudre des énigmes.
    * `v3` s’exécute en arrière-plan et fournit un score fondé sur le comportement de l’utilisateur, sans exiger d’interaction de sa part. Lorsque vous utilisez cette version, assurez-vous d’implémenter une logique métier supplémentaire pour traiter le score et déterminer si une autre action est nécessaire.

    ##### Paramètres du widget Google reCAPTCHA

    Les paramètres du widget Google reCAPTCHA comprennent :

    | Paramètres  | Description                                                                                                                                                                                                                                            |
    | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | Clé de site | La clé de site publique utilisée pour initialiser reCAPTCHA sur votre site Web. Vous pouvez la créer dans la [console Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) ou dans votre projet Google Cloud Platform.                     |
    | Clé secrète | La clé secrète utilisée pour communiquer de façon sécurisée avec le service reCAPTCHA côté serveur. Vous pouvez la créer dans la [console Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) ou dans votre projet Google Cloud Platform. |

    ##### Valeur de sortie de Google reCAPTCHA

    Le type de données de la valeur de sortie du widget Google reCAPTCHA est un `object`.

    Exemple de réponse `v2` :

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com"
      }
    }
    ```

    Exemple de réponse `v3` :

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com",
        "score": 0.9
      }
    }
    ```
  </Accordion>

  <Accordion title="Verifiable Credentials">
    Un widget qui permet à l’utilisateur de vérifier son identité à l’aide de justificatifs vérifiables stockés dans son portefeuille numérique.

    ##### Paramètres du widget Verifiable Credentials

    Les paramètres du widget Verifiable Credentials comprennent :

    | Paramètres              | Description                                                                                                                                                                                                                          |
    | ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | URL                     | La valeur d’URL utilisée pour générer le code QR. Cette valeur est renvoyée dans la propriété `engagement` de la requête de vérification.                                                                                            |
    | Texte du lien           | Texte de remplacement affiché pour les utilisateurs qui ne peuvent pas numériser le code QR et préfèrent ouvrir le lien directement sur leur appareil.                                                                               |
    | Taille                  | La taille du code QR.                                                                                                                                                                                                                |
    | Jeton public            | Le jeton d’accès requis pour utiliser le point de terminaison de polling. Veuillez vous assurer que le jeton est généré avec uniquement la portée `read:verification_request` afin d’éviter d’exposer l’accès à d’autres ressources. |
    | ID de vérification      | L’ID de vérification généré lorsque vous lancez la requête de vérification.                                                                                                                                                          |
    | Temps d’attente maximal | Le délai maximal d’attente pour que le processus de vérification se termine. Une fois ce délai atteint, le polling s’arrête et une erreur est générée, quel que soit l’état de la vérification.                                      |

    ##### Valeur de sortie de Verifiable Credentials

    Le type de données de la valeur de sortie du widget Verifiable Credentials est un `object`.

    ```json lines
    {
      "verifiable_credentials_widget_id": {
        "state": "honored",
        "reason": "...", // Le champ reason contient des renseignements supplémentaires (s’ils sont disponibles) concernant l’état de la requête de vérification. 
        "presentation": {} // La présentation contient les claims fournis par le portefeuille en réponse à la définition de présentation.
      }
    }
    ```
  </Accordion>
</AccordionGroup>

<div id="router-node">
  ## Nœud Router
</div>

Les nœuds Router vous permettent d’ajouter des règles pour créer des branchements conditionnels entre les nœuds.

Par défaut, un nœud Router comprend une seule règle de transit nommée **Cas par défaut** qui mène à d’autres nœuds. Vous pouvez ajouter des règles supplémentaires en fonction d’un ensemble de conditions qu’une variable doit remplir pour être ensuite dirigée vers d’autres nœuds. Pour en savoir plus, consultez [Router.](/docs/fr-CA/customize/forms/routers)
