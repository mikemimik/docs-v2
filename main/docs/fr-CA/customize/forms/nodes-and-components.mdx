Forms vous permet d’ajouter des nœuds et des composants, comme des champs, des blocs et des widgets, avec lesquels vos utilisateurs peuvent interagir dans le formulaire. Vous pouvez ensuite référencer, dans vos forms et flows, les données saisies à partir des composants.

<div id="form-nodes">
  ## Nœuds de formulaire
</div>

Il existe cinq types de nœuds de formulaire : Start, Step, Flow, Router et Ending screen. Ces nœuds remplissent une fonction bien précise dans la structure du formulaire.

<Frame>![Dashboard >  Forms > Éditeur de formulaire](/docs/images/cdy7uua7fh8z/5V4GsHxxC2cjxx1aODCxPd/309850c72bbce224e7a8d3f5af34114c/form-editor-preview.png)</Frame>

<div id="step-node">
  ## Step node
</div>

Les step nodes constituent la représentation visuelle du formulaire. Ils peuvent contenir un nombre quelconque de composants, comme des champs, des blocs et des widgets dans lesquels l’utilisateur peut saisir ses données.

<Frame>![Dashboard > Foms > Step node](/docs/images/cdy7uua7fh8z/2tbkcXkVon0BawCvYgnvIz/e2ce566320af18a6b08702f04f763126/form-step-node.png)</Frame>

<div id="field-components">
  ### Composants de champ
</div>

Les champs sont des composants d’interface utilisateur qui permettent aux utilisateurs de saisir leurs renseignements dans Forms.

<Frame>![Dashboard > Forms > Fields](/docs/images/cdy7uua7fh8z/6fuBdJcbDybzcVaiByUkoQ/2ad29c8dbbcc6ddf850f5efc004a75e0/forms-fields.png)</Frame>

<div id="field-settings">
  #### Paramètres des champs
</div>

Les champs comportent plusieurs paramètres par défaut que vous pouvez configurer selon vos besoins.

Les paramètres par défaut d’un champ sont :

| Paramètres            | Description                                   |
| --------------------- | --------------------------------------------- |
| ID                    | L’identifiant unique                          |
| Libellé               | Afficher ou masquer un libellé                |
| Obligatoire           | La valeur saisie est obligatoire              |
| Texte d’aide          | Afficher un texte d’aide                      |
| Texte de remplacement | Texte visible par l’utilisateur               |
| Valeur par défaut     | Valeur par défaut visible par l’utilisateur   |
| Transitoire           | Activer ou désactiver le masquage des données |

<div id="field-validation-rules">
  #### Règles de validation des champs
</div>

| Paramètres                    | Description                                                   |
| ----------------------------- | ------------------------------------------------------------- |
| Longueur min. / Longueur max. | Définit la longueur minimale ou maximale de la valeur saisie. |
| Valeur min. / Valeur max.     | Définit la valeur numérique minimale ou maximale.             |

<div id="available-fields-in-forms">
  #### Champs disponibles dans Forms
</div>

Les champs disponibles, avec leurs paramètres spécifiques et leurs types de données de sortie, sont les suivants :

<AccordionGroup>
  <Accordion title="Champ de texte">
    L’entrée standard permet à l’utilisateur de saisir n’importe quelle chaîne de caractères.

    ##### Paramètres du champ de texte

    Le paramètre du champ de texte est :

    | Paramètres | Description                                     |
    | ---------- | ----------------------------------------------- |
    | Multiligne | Active la saisie de texte sur plusieurs lignes. |

    ##### Valeur de sortie du champ de texte

    Le type de données de la valeur de sortie du champ de texte est `string`.

    ```json lines
    {
      "text_field_id": "Auth0"
    }
    ```
  </Accordion>

  <Accordion title="Champ de courriel">
    L’entrée standard permet à un utilisateur de saisir des adresses courriel sous forme de texte.

    ##### Valeur de sortie du champ Email

    Le type de données de la valeur de sortie du champ Email est `string`.

    ```json lines
    {
      "email_field_id": "username@domain.com"
    }
    ```
  </Accordion>

  <Accordion title="Champ de numéro de téléphone">
    La saisie standard permet à l’utilisateur d’entrer des numéros de téléphone.

    ##### Paramètres du champ phone

    Les paramètres du champ phone sont :

    | Paramètres                | Description                                                                                                           |
    | ------------------------- | --------------------------------------------------------------------------------------------------------------------- |
    | Sélecteur de pays         | Active ou désactive le sélecteur de pays. Par défaut, il affiche l’emplacement de l’utilisateur selon son adresse IP. |
    | Texte indicatif du filtre | Texte visible par l’utilisateur dans la fenêtre de recherche de pays.                                                 |

    ##### Valeur de sortie du champ phone

    Le type de données de la valeur de sortie du champ phone est soit une `string`, soit, si le sélecteur de pays est activé, un `object`.

    La valeur de sortie du champ phone sous forme de `string` :

    ```json lines
    {
      "phone_field_id": "8005550175"
    }
    ```

    Le champ phone renvoie une valeur de type `object` :

    ```json lines
    {
      "phone_field_id": {
        "national_number": "8005550175",
        "national_format": "(800) 555-0175",
        "international_number": "+18005550175",
        "international_format": "+1 800-555-0175",
        "country_code_iso": "US",
        "country_code_number": "1"
      }
    }
    ```
  </Accordion>

  <Accordion title="Champ à choix">
    Entrée standard qui permet à l’utilisateur de sélectionner une ou plusieurs valeurs.

    ##### Paramètres du champ de sélection

    Les paramètres du champ de sélection sont :

    | Settings                           | Description                                                                       |
    | ---------------------------------- | --------------------------------------------------------------------------------- |
    | Choix multiples                    | Active plusieurs options de sélection.                                            |
    | Autre option                       | Permet aux utilisateurs de saisir leur propre option.                             |
    | Advanced &gt; Valeurs internes     | Attribuez une valeur interne à chaque option de sélection.                        |
    | Advanced &gt; Modification en bloc | Modifiez en bloc les étiquettes et les valeurs internes des options de sélection. |

    ##### Valeur de sortie du champ de sélection

    Le type de données de la valeur de sortie du champ de sélection est soit un `string`, soit, si les choix multiples sont activés, un `array of strings` .

    La valeur de sortie du champ de sélection lorsque le type de données est `string` :

    ```json lines
    {
      "choice_field_id": "Option A"
    }
    ```

    Le champ `choice` renvoie une valeur de type `array of strings` :

    ```json lines
    {
      "choice_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ Cartes">
    Champ de saisie standard qui permet à l’utilisateur de sélectionner une ou plusieurs valeurs d’image.

    ##### Paramètres du champ Cards

    Les paramètres du champ Cards sont :

    | Paramètres                   | Description                                   |
    | ---------------------------- | --------------------------------------------- |
    | Choix multiples              | Active plusieurs options de sélection.        |
    | Masquer les libellés         | Masque les libellés des cartes.               |
    | Avancé &gt; Valeurs internes | Attribuer une valeur interne à chaque option. |

    ##### Valeur de sortie du champ Cards

    Le type de données de la valeur de sortie du champ Cards est soit une `string` ou, si le choix multiple est activé, un `array of strings`.

    La valeur de sortie du champ Cards sous forme de `string` :

    ```json lines
    {
      "card_field_id": "Option A"
    }
    ```

    Le champ cards renvoie une valeur de type de données `array of strings` :

    ```json lines
    {
      "card_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ URL">
    La saisie standard permet à l’utilisateur de saisir une valeur URL.

    ##### Valeur de sortie du champ URL

    Le type de données de la valeur de sortie du champ URL est une chaîne de caractères :

    ```json lines
    {
      "url_field_id": "https://auth0.com"
    }
    ```
  </Accordion>

  <Accordion title="Champ de mot de passe">
    Champ de saisie standard permettant à l’utilisateur d’entrer un mot de passe ou une valeur secrète.

    ##### Paramètres du champ Mot de passe

    Les paramètres du champ de mot de passe sont les suivants :

    | Paramètres                          | Description                                                                                                                                                                                                                   |
    | ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Exiger un mot de passe complexe     | Les mots de passe doivent contenir une lettre majuscule, une lettre minuscule, un symbole et un chiffre.                                                                                                                      |
    | Appliquer les directives du NIST    | Le mot de passe doit comporter au moins 8 caractères et ne pas contenir de mots de passe issus de bases de données compromises, de mots du dictionnaire, de caractères répétitifs ou séquentiels, ni de mots liés au contenu. |
    | Ajouter un indicateur de robustesse | Un indicateur graphique avec une échelle rouge, jaune et verte s’affiche sous le champ de mot de passe pour indiquer la robustesse du mot de passe saisi.                                                                     |
    | Hachage                             | Sélectionnez un algorithme pour hacher la valeur saisie directement dans le navigateur.                                                                                                                                       |

    ##### Valeur de sortie du champ Mot de passe

    Le type de données de la valeur de sortie du champ de mot de passe est une `string` masquée :

    ```json lines
    {
      "password_field_id": "███"
    }
    ```
  </Accordion>

  <Accordion title="Champ de paiement">
    Champ standard permettant à un utilisateur de saisir des renseignements de paiement à l’aide d’un fournisseur de services de paiement. Les paiements sont traités avant le nœud Ending.

    ##### Paramètres du champ de paiement

    Les paramètres du champ de paiement sont les suivants :

    | Paramètres          | Description                                                                                                                     |
    | ------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
    | Type de paiement    | Sélectionnez un paiement unique (Charge) ou récurrent (Subscription).                                                           |
    | Montant             | Indiquez le montant d’un paiement unique (Charge).                                                                              |
    | Devise              | Indiquez la devise d’un paiement unique (Charge).                                                                               |
    | Action d’abonnement | Sélectionnez la création ou la mise à jour d’un abonnement existant.                                                            |
    | Action du client    | Sélectionnez la création ou la mise à jour d’un client existant.                                                                |
    | Options d’affichage | Affichez et modifiez les libellés de Numéro de carte, Date d’expiration et Code de sécurité, ainsi que les sceaux de confiance. |

    ##### Valeur de sortie du champ de paiement

    Le type de données de la valeur de sortie du champ de paiement est soit une chaîne de caractères, soit un objet pour les flux après la soumission.

    Le type de données de la valeur de sortie du champ de paiement sous forme de `string` :

    ```json lines
    {
      "payment_field_id": "pm_1P19e..."
    }
    ```

    Le type de données de la valeur de sortie du champ de paiement pour les abonnements est de type `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "price_ids": [
          "price_1ONHR..."
        ],
        "subscription_id": "sub_1P1A...",
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```

    Le type de données de la valeur de sortie du champ de paiement pour charge est de type `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "amount": 100,
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```
  </Accordion>

  <Accordion title="Domaine juridique">
    Champ standard qui ajoute un texte juridique et une case à cocher.

    ##### Valeur de sortie du champ juridique

    Le type de données de la valeur de sortie du champ juridique est `boolean`.

    ```json lines
    {
      "legal_field_id": true
    }
    ```
  </Accordion>

  <Accordion title="Champ à liste déroulante">
    Champ standard qui permet à un utilisateur de sélectionner une ou plusieurs valeurs dans une liste déroulante.

    ##### Paramètres du champ de liste déroulante

    Les paramètres du champ de liste déroulante sont :

    | Paramètres                       | Description                                            |
    | -------------------------------- | ------------------------------------------------------ |
    | Sélecteur multiple               | Active plusieurs options de choix.                     |
    | Avancé &gt; Valeurs internes     | Définir une valeur interne pour chaque option.         |
    | Avancé &gt; Modification en bloc | Modifier en bloc les libellés et les valeurs internes. |

    ##### Valeur de sortie du champ de liste déroulante

    Le type de données de la valeur de sortie du champ de liste déroulante est soit une `string` ou, si le choix multiple est activé, un `array of strings` .

    La valeur de sortie du champ de liste déroulante en tant que `string` :

    ```json lines
    {
      "dropdown_field_id": "Option A"
    }
    ```

    Le champ de liste déroulante renvoie une valeur du type de données `array of strings` :

    ```json lines
    {
      "dropdown_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ numérique">
    Champ de saisie standard qui permet à l’utilisateur d’entrer des valeurs numériques.

    ##### Valeur de sortie du champ numérique

    Le type de données de la valeur de sortie du champ numérique est `number`.

    ```json lines
    {
      "number_field_id": 123
    }
    ```
  </Accordion>

  <Accordion title="Champ de date et d’heure">
    Champ standard permettant à l’utilisateur de saisir soit une date, soit une heure.

    ##### Paramètres du champ Date / Heure

    Les paramètres du champ date/heure comprennent :

    | Key    | Description                           |
    | ------ | ------------------------------------- |
    | Format | Sélectionnez le format Date ou Heure. |

    ##### Valeur de sortie du champ Date / Heure

    Pour une date, la valeur du type de données de sortie du champ date/heure est une `string` :

    ```json lines
    {
      "date_time_field_id": "2023-04-11"
    }
    ```

    La valeur du type de données de sortie du champ date/heure pour l’heure est de type `string` :

    ```json lines
    {
      "date_time_field_id": "23:15"
    }
    ```
  </Accordion>

  <Accordion title="Champ booléen">
    Saisie standard permettant à l’utilisateur d’entrer des valeurs vrai ou faux.

    ##### Valeur de sortie du champ booléen

    Le type de données de la valeur de sortie du champ booléen est booléen.

    ```json lines
    {
      "boolean_field_id": true
    }
    ```
  </Accordion>

  <Accordion title="Champ personnalisé">
    Champ standard que vous pouvez personnaliser pour créer votre propre champ.

    ##### Paramètres du champ personnalisé

    Les paramètres du champ personnalisé sont :

    | Paramètres  | Description                                                                                                                                         |
    | ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Params      | Ajoutez des paires clé-valeur que vous pourrez référencer dans le code source du champ personnalisé.                                                |
    | Code source | Ajoutez votre code Javascript au champ personnalisé.                                                                                                |
    | Schéma JSON | Par défaut, le champ personnalisé accepte n’importe quel format de valeur. Cependant, vous pouvez utiliser un schéma JSON pour valider les valeurs. |

    Pour en savoir plus, consultez [Composants de champs personnalisés pour Forms](/docs/fr-CA/customize/forms/custom-field-components).
  </Accordion>
</AccordionGroup>

<div id="blocks-components">
  ### Composants Blocks
</div>

Les Blocks sont des composants UI qui ajoutent des fonctionnalités à vos formulaires sans recueillir d’informations sur les utilisateurs.

<Frame>![Dashboard > Forms > Components > Blocks](/docs/images/cdy7uua7fh8z/7IijftJnuwcevL4bdHiaKw/f0376ca75303bb58d0df87e6af75e347/forms-block.png)</Frame>

<div id="available-blocks-in-forms">
  #### Blocs disponibles dans Forms
</div>

Les blocs disponibles sont :

<AccordionGroup>
  <Accordion title="Bloc de bouton Suivant">
    Un bouton qui permet aux utilisateurs de passer au nœud de formulaire suivant.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      * Avant de continuer, les règles de validation des champs sont appliquées et validées côté client et côté serveur. Si elles échouent, un message d’erreur s’affiche pour l’utilisateur.
      * Si le nœud suivant est un nœud de flux, un indicateur de chargement s’affiche jusqu’à la fin du flux. Si le flux échoue ou renvoie un message d’erreur personnalisé, un message d’erreur s’affiche.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton Précédent">
    Un bouton qui permet aux utilisateurs de revenir au nœud Step précédent.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un utilisateur sélectionne le bouton précédent, les règles de validation des champs sont exécutées de nouveau, même si aucune donnée saisie n’a été modifiée avant de continuer.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton de saut">
    Un bouton qui permet aux utilisateurs d’ignorer l’étape actuelle et de passer directement au nœud de destination.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Lorsque l’étape actuelle est ignorée, nous ne validons pas et ne recueillons pas non plus les renseignements saisis à cette étape.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton de renvoi">
    Un composant qui permet aux utilisateurs de réexécuter un flux afin de générer et d’envoyer un nouveau code OTP

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un utilisateur sélectionne le bouton de renvoi, le flux associé génère et envoie un nouveau code OTP.
    </Callout>

    | Paramètres                   | Description                                                                                                                                                                                                                                                                                                                                                 |
    | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Alignement du texte          | Sélectionnez l’alignement du texte : à gauche, centré ou à droite                                                                                                                                                                                                                                                                                           |
    | Texte                        | Texte par défaut affiché lorsque les utilisateurs sélectionnent le bouton de renvoi                                                                                                                                                                                                                                                                         |
    | Texte du bouton              | Texte du bouton                                                                                                                                                                                                                                                                                                                                             |
    | Texte d’attente              | Texte affiché lorsque les utilisateurs ont sélectionné le bouton de renvoi. Remplace les paramètres `Text` et `Button text`. Utilisez la variable `“{{remaining_seconds}}”` pour afficher dynamiquement le nombre de secondes restantes avant que le bouton d’envoi soit de nouveau activé. Par exemple : “Renvoyer dans `{{remaining_seconds}}` secondes”. |
    | Flow                         | Flux exécuté après que l’utilisateur a sélectionné le bouton de renvoi.                                                                                                                                                                                                                                                                                     |
    | Nombre maximal de tentatives | Le nombre maximal de fois que l’utilisateur peut sélectionner le bouton de renvoi                                                                                                                                                                                                                                                                           |
    | Temps d’attente              | Le temps d’attente entre les tentatives, en secondes.                                                                                                                                                                                                                                                                                                       |
  </Accordion>

  <Accordion title="Bloc de texte enrichi">
    Un bloc de texte enrichi pour personnaliser le nœud Step avec des renseignements supplémentaires.
  </Accordion>

  <Accordion title="Bloc de séparateur">
    Une ligne pour diviser les différentes sections du nœud Step. Elle peut contenir un court texte.
  </Accordion>

  <Accordion title="Bloc HTML">
    Un bloc HTML pour créer votre interface utilisateur personnalisée.
  </Accordion>

  <Accordion title="Bloc d’image">
    Un bloc d’image pour personnaliser l’étape en y ajoutant des images.
  </Accordion>
</AccordionGroup>

<div id="widgets-components">
  ### Composants widget
</div>

Les widgets sont des composants préconfigurés dotés d’intégrations tierces qui ajoutent de la logique côté client et côté serveur à vos formulaires.

<Frame>![Dashboard > Forms > Components > Widget](/docs/images/cdy7uua7fh8z/3J0IuWLP7utOPKSHc82IE9/a523afe0c240c63f714ef5fb102f8dab/forms-widget.png)</Frame>

<div id="available-widgets-in-forms">
  #### Widgets disponibles dans Forms
</div>

Le widget disponible est :

<AccordionGroup>
  <Accordion title="Widget d’adresse Google">
    Un widget qui permet à l’utilisateur de saisir et de valider son adresse.

    ##### Paramètres du widget d’adresse Google

    Les paramètres du widget d’adresse Google comprennent :

    | Paramètres | Description                                                       |
    | ---------- | ----------------------------------------------------------------- |
    | Clé API    | Nécessite une clé API Google Maps pour authentifier les requêtes. |

    ##### Valeur de sortie du widget d’adresse Google

    Le type de données de la valeur de sortie du widget d’adresse Google est un `object`.

    ```json lines
    {
      "google_address_widget_id": {
        "type": "Feature",
        "geometry": {
          "type": "Point",
          "coordinates": [
            -73.9654415,
            40.8054491
          ]
        },
        "properties": {
          "geocoding": {
            "type": "house",
            "housenumber": "2880",
            "street": "Broadway",
            "city": "New York",
            "county": "New York County",
            "state": "New York",
            "country": "United States",
            "postcode": "10025"
          }
        }
      }
    }
    ```
  </Accordion>

  <Accordion title="Google reCAPTCHA">
    Un widget qui aide à protéger votre site Web contre le pourriel et les abus en vérifiant qu’un utilisateur est un humain et non un robot.

    Le widget prend en charge :

    * Basé sur le score (v3)
    * Challenge (v2)

      * Badge reCAPTCHA invisible

    Différences entre `v2` et `v3` :

    * `v2` exige que l’utilisateur interagisse avec des défis visibles, comme cliquer sur une case à cocher ou résoudre des casse-têtes.
    * `v3` s’exécute en arrière-plan et fournit un score fondé sur le comportement de l’utilisateur, sans exiger d’interaction. Lorsque vous utilisez cette version, assurez-vous d’implémenter une logique métier supplémentaire pour gérer le score et déterminer si une autre action est nécessaire.

    ##### Paramètres du widget Google reCAPTCHA

    Les paramètres du widget Google reCAPTCHA comprennent :

    | Paramètres  | Description                                                                                                                                                                                                                                            |
    | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | Clé de site | La clé de site publique utilisée pour initialiser reCAPTCHA sur votre site Web. Vous pouvez la créer dans la [console Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) ou dans votre projet Google Cloud Platform.                     |
    | Clé secrète | La clé secrète utilisée pour communiquer de façon sécurisée avec le service reCAPTCHA côté serveur. Vous pouvez la créer dans la [console Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) ou dans votre projet Google Cloud Platform. |

    ##### Valeur de sortie de Google reCAPTCHA

    Le type de données de la valeur de sortie du widget Google reCAPTCHA est un `object`.

    Exemple de réponse `v2` :

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com"
      }
    }
    ```

    Exemple de réponse `v3` :

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com",
        "score": 0.9
      }
    }
    ```
  </Accordion>

  <Accordion title="Justificatifs vérifiables">
    Un widget qui permet à l’utilisateur de vérifier son identité à l’aide de justificatifs vérifiables stockés dans son portefeuille numérique.

    ##### Paramètres du widget Justificatifs vérifiables

    Les paramètres du widget Justificatifs vérifiables comprennent :

    | Paramètres              | Description                                                                                                                                                                                                                        |
    | ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | URL                     | La valeur URL utilisée pour générer le code QR. Cette valeur est renvoyée dans la propriété `engagement` de la requête de vérification.                                                                                            |
    | Texte du lien           | Texte de remplacement qui s’affichera pour les utilisateurs qui ne peuvent pas balayer le code QR et qui préfèrent ouvrir le lien directement sur leur appareil.                                                                   |
    | Taille                  | La taille du code QR.                                                                                                                                                                                                              |
    | Jeton public            | Le access token requis pour utiliser le point de terminaison de polling. Veuillez vous assurer que le token est généré uniquement avec le scope `read:verification_request` afin d’éviter d’exposer l’accès à d’autres ressources. |
    | ID de vérification      | L’ID de vérification généré lorsque vous démarrez la requête de vérification.                                                                                                                                                      |
    | Temps d’attente maximal | Le délai maximal d’attente pour que le processus de vérification se termine. Une fois ce délai atteint, le polling s’arrête et une erreur est générée, peu importe l’état de la vérification.                                      |

    ##### Valeur de sortie de Justificatifs vérifiables

    Le type de données de la valeur de sortie du widget Justificatifs vérifiables est un `object`.

    ```json lines
    {
      "verifiable_credentials_widget_id": {
        "state": "honored",
        "reason": "...", // Le champ reason contient des renseignements supplémentaires (s’ils sont disponibles) concernant l’état de la requête de vérification. 
        "presentation": {} // La présentation contient les claims fournis par le portefeuille en réponse à la définition de présentation.
      }
    }
    ```
  </Accordion>

<div id="router-node">
  ## Nœud Router
</div>

Les nœuds Router vous permettent d’ajouter des règles afin de créer des embranchements logiques conditionnels entre les nœuds.

Par défaut, un nœud Router comprend une seule règle de transit nommée **Cas par défaut** qui mène à d’autres nœuds. Vous pouvez ajouter d’autres règles en fonction d’un ensemble de conditions qu’une variable doit remplir pour mener ensuite à d’autres nœuds. Pour en savoir plus, consultez [Router.](/docs/fr-CA/customize/forms/routers)
