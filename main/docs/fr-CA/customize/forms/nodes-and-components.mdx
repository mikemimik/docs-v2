Forms vous permet d’ajouter des nœuds et des composants, comme des champs, des blocs et des widgets, avec lesquels vos utilisateurs peuvent interagir dans le formulaire. Vous pouvez ensuite référencer, dans vos forms et flux, les données saisies à partir des composants.

<div id="form-nodes">
  ## Nœuds de formulaire
</div>

Il existe cinq types de nœuds de formulaire : Start, étape, Flow, Router et écran de fin. Ces nœuds jouent un rôle très précis dans la structure du formulaire.

<Frame>![Dashboard > Forms > Form editor](/docs/images/cdy7uua7fh8z/5V4GsHxxC2cjxx1aODCxPd/309850c72bbce224e7a8d3f5af34114c/form-editor-preview.png)</Frame>

<div id="step-node">
  ## Nœud d’étape
</div>

Les nœuds d’étape représentent visuellement le formulaire. Ils peuvent contenir autant de composants que nécessaire, comme des champs, des blocs et des widgets, dans lesquels l’utilisateur peut saisir ses données.

<Frame>![Dashboard > Foms > Nœud d’étape](/docs/images/cdy7uua7fh8z/2tbkcXkVon0BawCvYgnvIz/e2ce566320af18a6b08702f04f763126/form-step-node.png)</Frame>

<div id="field-components">
  ### Composants de champ
</div>

Les champs sont des composants d’interface utilisateur dans lesquels les utilisateurs saisissent leurs renseignements dans Forms.

<Frame>![Dashboard > Forms > Champs](/docs/images/cdy7uua7fh8z/6fuBdJcbDybzcVaiByUkoQ/2ad29c8dbbcc6ddf850f5efc004a75e0/forms-fields.png)</Frame>

<div id="field-settings">
  #### Paramètres du champ
</div>

Les champs comportent un certain nombre de paramètres par défaut que vous pouvez configurer selon vos besoins.

Les paramètres par défaut du champ sont :

| Paramètres        | Description                                   |
| ----------------- | --------------------------------------------- |
| ID                | ID unique                                     |
| Libellé           | Afficher ou masquer un libellé                |
| Obligatoire       | La saisie est obligatoire                     |
| Texte d’aide      | Afficher un texte d’aide                      |
| Texte indicatif   | Texte visible pour l’utilisateur              |
| Valeur par défaut | Valeur par défaut visible pour l’utilisateur  |
| Temporaire        | Activer ou désactiver le masquage des données |

<div id="field-validation-rules">
  #### Règles de validation des champs
</div>

| Paramètres                    | Description                               |
| ----------------------------- | ----------------------------------------- |
| Longueur min. / Longueur max. | Exige une longueur pour la valeur saisie. |
| Valeur min. / Valeur max.     | Exige une valeur numérique.               |

<div id="available-fields-in-forms">
  #### Champs disponibles dans Forms
</div>

Les champs disponibles, avec leurs paramètres spécifiques et leurs types de données de sortie, sont :

<AccordionGroup>
  <Accordion title="Champ de texte">
    L’entrée standard permet à l’utilisateur de saisir n’importe quelle valeur textuelle.

    ##### Paramètres du champ de texte

    Le paramètre du champ de texte est :

    | Paramètres | Description                                     |
    | ---------- | ----------------------------------------------- |
    | Multiligne | Permet la saisie de texte sur plusieurs lignes. |

    ##### Valeur de sortie du champ de texte

    Le type de données de la valeur de sortie du champ de texte est `string`.

    ```json lines
    {
      "text_field_id": "Auth0"
    }
    ```
  </Accordion>

  <Accordion title="Champ d’adresse courriel">
    La saisie standard permet à l’utilisateur de saisir des adresses courriel sous forme de chaîne de caractères.

    ##### Valeur de sortie du champ courriel

    Le type de données de la valeur de sortie du champ courriel est `string`.

    ```json lines
    {
      "email_field_id": "username@domain.com"
    }
    ```
  </Accordion>

  <Accordion title="Champ Numéro de téléphone">
    La saisie standard permet à l’utilisateur d’entrer des numéros de téléphone.

    ##### Paramètres du champ de téléphone

    Les paramètres du champ de téléphone sont les suivants :

    | Paramètres                | Description                                                                                                   |
    | ------------------------- | ------------------------------------------------------------------------------------------------------------- |
    | Sélecteur de pays         | Active ou désactive le sélecteur de pays. Par défaut, il affiche l’emplacement de l’utilisateur selon son IP. |
    | Texte indicatif du filtre | Texte visible par l’utilisateur dans la fenêtre de recherche de pays.                                         |

    ##### Valeur de sortie du champ de téléphone

    Le type de données de la valeur de sortie du champ de téléphone est soit une `string`, soit, si le sélecteur de pays est activé, un `object`.

    Valeur de sortie du champ de téléphone comme `string` :

    ```json lines
    {
      "phone_field_id": "8005550175"
    }
    ```

    La valeur du type de données de sortie du champ phone est de type `object` :

    ```json lines
    {
      "phone_field_id": {
        "national_number": "8005550175",
        "national_format": "(800) 555-0175",
        "international_number": "+18005550175",
        "international_format": "+1 800-555-0175",
        "country_code_iso": "US",
        "country_code_number": "1"
      }
    }
    ```
  </Accordion>

  <Accordion title="Champ de sélection">
    Champ standard qui permet à l’utilisateur de sélectionner une ou plusieurs valeurs.

    ##### Paramètres du champ de choix

    Les paramètres du champ de choix sont :

    | Paramètres                       | Description                                                                   |
    | -------------------------------- | ----------------------------------------------------------------------------- |
    | Choix multiples                  | Active plusieurs options de réponse.                                          |
    | Autre option                     | Permet aux utilisateurs de saisir leur propre option.                         |
    | Avancé &gt; Valeurs internes     | Attribuez une valeur interne à chaque option de réponse.                      |
    | Avancé &gt; Modification en bloc | Modifiez en bloc les libellés et les valeurs internes des options de réponse. |

    ##### Valeur de sortie du champ de choix

    Le type de données de la valeur de sortie du champ de choix est soit une `string` ou, si le choix multiple est activé, un `array of strings` .

    Le type de données de la valeur de sortie du champ de choix en tant que `string` :

    ```json lines
    {
      "choice_field_id": "Option A"
    }
    ```

    La valeur du type de données de sortie du champ de sélection est un `array of strings` :

    ```json lines
    {
      "choice_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ de cartes">
    Entrée standard qui permet à l’utilisateur de sélectionner des valeurs d’image à choix unique ou à choix multiples.

    ##### Paramètres du champ Cartes

    Les paramètres du champ Cartes sont :

    | Paramètres                   | Description                                   |
    | ---------------------------- | --------------------------------------------- |
    | Choix multiples              | Active plusieurs options de choix.            |
    | Masquer les libellés         | Masque les libellés des cartes.               |
    | Avancé &gt; Valeurs internes | Attribuer une valeur interne à chaque option. |

    ##### Valeur de sortie du champ Cartes

    Le type de données de la valeur de sortie du champ Cartes est soit une `string` ou, si les choix multiples sont activés, un `array of strings`.

    La valeur de sortie du champ Cartes sous forme de `string` :

    ```json lines
    {
      "card_field_id": "Option A"
    }
    ```

    Le type de données de sortie du champ cards est un `array of strings` :

    ```json lines
    {
      "card_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ URL">
    La saisie standard permet à l’utilisateur d’entrer une valeur d’URL.

    ##### Valeur de sortie du champ URL

    Le type de données de la valeur de sortie du champ URL est une chaîne de caractères :

    ```json lines
    {
      "url_field_id": "https://auth0.com"
    }
    ```
  </Accordion>

  <Accordion title="Champ de mot de passe">
    Champ standard qui permet à l’utilisateur de saisir un mot de passe ou une valeur secrète.

    ##### Paramètres du champ de mot de passe

    Les paramètres du champ de mot de passe sont :

    | Paramètres                          | Description                                                                                                                                                                                                                               |
    | ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Exiger un mot de passe complexe     | Les mots de passe doivent contenir une majuscule, une minuscule, un symbole et un chiffre.                                                                                                                                                |
    | Appliquer les directives du NIST    | Le mot de passe ne peut pas comporter moins de 8 caractères ni figurer dans des corpus de fuites de données antérieures, ni contenir des mots du dictionnaire, des caractères répétitifs ou séquentiels, ou des mots propres au contexte. |
    | Ajouter un indicateur de robustesse | Un indicateur graphique avec une échelle rouge, jaune et verte s’affiche sous le champ de mot de passe pour indiquer la robustesse du mot de passe saisi.                                                                                 |
    | Hachage                             | Sélectionnez un algorithme pour hacher la valeur saisie directement dans le navigateur.                                                                                                                                                   |

    ##### Valeur de sortie du champ de mot de passe

    Le type de données de la valeur de sortie du champ de mot de passe est une `string` masquée :

    ```json lines
    {
      "password_field_id": "███"
    }
    ```
  </Accordion>

  <Accordion title="Champ de paiement">
    Champ standard qui permet à un utilisateur de saisir des renseignements de paiement au moyen d’un fournisseur de paiement. Les paiements sont traités avant le nœud de fin.

    ##### Paramètres du champ de paiement

    Les paramètres du champ de paiement sont :

    | Paramètres          | Description                                                                                                                     |
    | ------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
    | Type de paiement    | Sélectionnez un paiement ponctuel (Charge) ou récurrent (Subscription).                                                         |
    | Montant             | Indiquez le montant d’un paiement ponctuel (Charge).                                                                            |
    | Devise              | Indiquez la devise d’un paiement ponctuel (Charge).                                                                             |
    | Action d’abonnement | Sélectionnez la création ou la mise à jour d’un abonnement existant.                                                            |
    | Action du client    | Sélectionnez la création ou la mise à jour d’un client existant.                                                                |
    | Options d’affichage | Affichez et modifiez les libellés de Numéro de carte, Date d’expiration et Code de sécurité, ainsi que les sceaux de confiance. |

    ##### Valeur de sortie du champ de paiement

    Le type de données de la valeur de sortie du champ de paiement est soit une chaîne de caractères, soit un objet pour les flux après soumission.

    Le type de données de la valeur de sortie du champ de paiement sous forme de `string` :

    ```json lines
    {
      "payment_field_id": "pm_1P19e..."
    }
    ```

    Le type de données de la valeur de sortie du champ de paiement pour les abonnements sous forme d’`object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "price_ids": [
          "price_1ONHR..."
        ],
        "subscription_id": "sub_1P1A...",
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```

    Le type de données de la valeur de sortie du champ de paiement pour charge est de type `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "amount": 100,
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```
  </Accordion>

  <Accordion title="Champ légal">
    Champ standard qui ajoute un texte juridique et une case à cocher.

    ##### Valeur de sortie du champ des mentions légales

    Le type de données de la valeur de sortie du champ des mentions légales est un `boolean`.

    ```json lines
    {
      "legal_field_id": true
    }
    ```
  </Accordion>

  <Accordion title="Champ déroulant">
    Champ standard qui permet à un utilisateur de sélectionner une ou plusieurs valeurs dans une liste déroulante.

    ##### Paramètres du champ de liste déroulante

    Les paramètres du champ de liste déroulante sont :

    | Paramètres                       | Description                                           |
    | -------------------------------- | ----------------------------------------------------- |
    | Sélecteur multiple               | Active plusieurs options de choix.                    |
    | Avancé &gt; Valeurs internes     | Attribuez une valeur interne à chaque option.         |
    | Avancé &gt; Modification en bloc | Modifiez en bloc l’étiquette et les valeurs internes. |

    ##### Valeur de sortie du champ de liste déroulante

    Le type de données de la valeur de sortie du champ de liste déroulante est soit une `string`, soit, si le choix multiple est activé, un `array of strings` .

    La valeur de sortie du champ de liste déroulante de type `string` :

    ```json lines
    {
      "dropdown_field_id": "Option A"
    }
    ```

    La valeur de sortie du champ de liste déroulante est de type `array of strings` :

    ```json lines
    {
      "dropdown_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Champ numérique">
    Champ de saisie standard qui permet à l’utilisateur d’entrer des valeurs numériques.

    ##### Valeur de sortie du champ Number

    La valeur de sortie du champ Number est de type `number`.

    ```json lines
    {
      "number_field_id": 123
    }
    ```
  </Accordion>

  <Accordion title="Champ date / heure">
    Entrée standard qui permet à l’utilisateur de saisir une date ou une heure.

    ##### Paramètres du champ Date / Heure

    Les paramètres du champ Date / Heure comprennent :

    | Clé    | Description                           |
    | ------ | ------------------------------------- |
    | Format | Sélectionnez le format Date ou Heure. |

    ##### Valeur de sortie du champ Date / Heure

    Pour une date, la valeur de sortie du type de données du champ Date / Heure est une `string` :

    ```json lines
    {
      "date_time_field_id": "2023-04-11"
    }
    ```

    La valeur du type de données de sortie du champ date/heure pour l’heure est de type `string` :

    ```json lines
    {
      "date_time_field_id": "23:15"
    }
    ```
  </Accordion>

  <Accordion title="Champ booléen">
    Champ de saisie standard permettant à l’utilisateur d’entrer des valeurs vraies ou fausses.

    ##### Valeur de sortie du champ booléen

    Le type de données de la valeur de sortie du champ booléen est un booléen.

    ```json lines
    {
      "boolean_field_id": true
    }
    ```
  </Accordion>

  <Accordion title="Champ personnalisé">
    Champ standard que vous pouvez personnaliser pour créer votre propre champ.

    ##### Paramètres du champ personnalisé

    Les paramètres du champ personnalisé sont les suivants :

    | Paramètres  | Description                                                                                                                                      |
    | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
    | Params      | Ajoutez des paires clé-valeur auxquelles le code source du champ personnalisé peut faire référence.                                              |
    | Code source | Ajoutez votre code Javascript au champ personnalisé.                                                                                             |
    | JSON Schema | Par défaut, le champ personnalisé accepte n’importe quel format de valeur. Toutefois, vous pouvez utiliser JSON Schema pour valider les valeurs. |

    Pour en savoir plus, consultez [Composants de champ personnalisé pour Forms](/docs/fr-CA/customize/forms/custom-field-components).
  </Accordion>
</AccordionGroup>

<div id="blocks-components">
  ### Composants « Blocks »
</div>

Les Blocks sont des composants d’interface utilisateur qui ajoutent des fonctionnalités à vos formulaires sans recueillir d’informations sur l’utilisateur.

<Frame>![Dashboard > Forms > Components > Blocks](/docs/images/cdy7uua7fh8z/7IijftJnuwcevL4bdHiaKw/f0376ca75303bb58d0df87e6af75e347/forms-block.png)</Frame>

<div id="available-blocks-in-forms">
  #### Blocs disponibles dans Forms
</div>

Les blocs disponibles sont :

<AccordionGroup>
  <Accordion title="Bloc de bouton Suivant">
    Un bouton qui permet aux utilisateurs de passer au nœud de formulaire suivant.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      * Avant de continuer, les règles de validation des champs sont appliquées et validées côté client et côté serveur. Si elles échouent, un message d&#39;erreur s&#39;affiche pour l&#39;utilisateur.
      * Si le nœud suivant est un nœud de flux, un indicateur de chargement s&#39;affiche jusqu&#39;à ce que le flux soit terminé. Si le flux échoue ou renvoie un message d&#39;erreur personnalisé, un message d&#39;erreur s&#39;affiche.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton Précédent">
    Un bouton qui permet aux utilisateurs de revenir au step node précédent.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un utilisateur sélectionne le bouton précédent, les règles de validation des champs sont exécutées de nouveau, même si aucune modification n&#39;a été apportée aux données saisies avant de continuer.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton Saut">
    Un bouton qui permet aux utilisateurs d&#39;ignorer l&#39;étape en cours et de passer directement au nœud de destination.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Lorsque l&#39;étape en cours est ignorée, nous ne validons pas et ne recueillons pas non plus les renseignements saisis à cette étape.
    </Callout>
  </Accordion>

  <Accordion title="Bloc de bouton de renvoi">
    Un composant qui permet aux utilisateurs de réexécuter un flux pour générer et envoyer un nouveau code OTP

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un utilisateur sélectionne le bouton de renvoi, le flux associé génère et envoie un nouveau code OTP.
    </Callout>

    | Settings                     | Description                                                                                                                                                                                                                                                                                                                                            |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | Alignement du texte          | Sélectionnez l&#39;alignement du texte : gauche, centré ou droite                                                                                                                                                                                                                                                                                      |
    | Texte                        | Texte par défaut affiché lorsque les utilisateurs sélectionnent le bouton de renvoi                                                                                                                                                                                                                                                                    |
    | Texte du bouton              | Texte du bouton                                                                                                                                                                                                                                                                                                                                        |
    | Texte d&#39;attente          | Texte affiché lorsque les utilisateurs ont sélectionné le bouton de renvoi. Remplace les paramètres `Text` et `Button text`. Utilisez la variable `“{{remaining_seconds}}”` pour afficher dynamiquement le nombre de secondes restantes avant que le bouton d&#39;envoi soit réactivé. Par exemple : “Renvoyer dans `{{remaining_seconds}}` secondes”. |
    | Flow                         | Flux exécuté après que l&#39;utilisateur a sélectionné le bouton de renvoi.                                                                                                                                                                                                                                                                            |
    | Nombre maximal de tentatives | Nombre maximal de tentatives où l&#39;utilisateur peut sélectionner le bouton de renvoi                                                                                                                                                                                                                                                                |
    | Temps d&#39;attente          | Temps d&#39;attente entre les tentatives, en secondes.                                                                                                                                                                                                                                                                                                 |
  </Accordion>

  <Accordion title="Bloc de texte enrichi">
    Un bloc de texte enrichi pour personnaliser le step node avec des renseignements supplémentaires.
  </Accordion>

  <Accordion title="Bloc de séparateur">
    Une ligne pour séparer différentes sections du step node. Elle peut contenir un court texte.
  </Accordion>

  <Accordion title="Bloc HTML">
    Un bloc HTML pour créer votre UI personnalisée.
  </Accordion>

  <Accordion title="Bloc d'image">
    Un bloc d&#39;image pour personnaliser l&#39;étape en y ajoutant des images.
  </Accordion>
</AccordionGroup>

<div id="widgets-components">
  ### Composants de widget
</div>

Les widgets sont des composants prédéfinis, dotés d’intégrations de tierces parties, qui ajoutent une logique côté client et côté serveur à vos formulaires.

<Frame>![Dashboard > Forms > Components > Widget](/docs/images/cdy7uua7fh8z/3J0IuWLP7utOPKSHc82IE9/a523afe0c240c63f714ef5fb102f8dab/forms-widget.png)</Frame>

<div id="available-widgets-in-forms">
  #### Widgets disponibles dans Forms
</div>

Les widgets disponibles sont :

<AccordionGroup>
  <Accordion title="Widget d'adresse Google">
    Un widget qui permet à l&#39;utilisateur de saisir et de valider son adresse.

    ##### Paramètres du widget d&#39;adresse Google

    Les paramètres du widget d&#39;adresse Google comprennent :

    | Paramètres | Description                                                       |
    | ---------- | ----------------------------------------------------------------- |
    | Clé API    | Nécessite une clé API Google Maps pour authentifier les requêtes. |

    ##### Valeur de sortie du widget d&#39;adresse Google

    Le type de données de la valeur de sortie du widget d&#39;adresse Google est un `object`.

    ```json lines
    {
      "google_address_widget_id": {
        "type": "Feature",
        "geometry": {
          "type": "Point",
          "coordinates": [
            -73.9654415,
            40.8054491
          ]
        },
        "properties": {
          "geocoding": {
            "type": "house",
            "housenumber": "2880",
            "street": "Broadway",
            "city": "New York",
            "county": "New York County",
            "state": "New York",
            "country": "United States",
            "postcode": "10025"
          }
        }
      }
    }
    ```
  </Accordion>

  <Accordion title="Google reCAPTCHA">
    Un widget qui aide à protéger votre site Web contre le pourriel et les abus en vérifiant qu&#39;un utilisateur est bien une personne et non un bot.

    Le widget prend en charge :

    * Basé sur le score (v3)
    * Défi (v2)

      * Badge reCAPTCHA invisible

    Différences entre `v2` et `v3` :

    * `v2` nécessite une interaction de l&#39;utilisateur avec des défis visibles, par exemple cliquer sur une case à cocher ou résoudre des casse-têtes.
    * `v3` s&#39;exécute en arrière-plan et attribue un score en fonction du comportement de l&#39;utilisateur, sans nécessiter d&#39;interaction. Lorsque vous utilisez cette version, veillez à mettre en place une logique métier supplémentaire pour traiter le score et déterminer si une autre action est requise.

    ##### Paramètres du widget Google reCAPTCHA

    Les paramètres du widget Google reCAPTCHA comprennent :

    | Paramètres  | Description                                                                                                                                                                                                                                            |
    | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | Clé de site | La clé de site publique utilisée pour initialiser reCAPTCHA sur votre site Web. Vous pouvez la créer dans la [console Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) ou dans votre projet Google Cloud Platform.                     |
    | Clé secrète | La clé secrète utilisée pour communiquer de façon sécurisée avec le service reCAPTCHA côté serveur. Vous pouvez la créer dans la [console Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) ou dans votre projet Google Cloud Platform. |

    ##### Valeur de sortie de Google reCAPTCHA

    Le type de données de la valeur de sortie du widget Google reCAPTCHA est un `object`.

    Exemple de réponse `v2` :

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com"
      }
    }
    ```

    Exemple de réponse `v3` :

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com",
        "score": 0.9
      }
    }
    ```
  </Accordion>

  <Accordion title="Justificatifs vérifiables">
    Un widget qui permet à l&#39;utilisateur de vérifier son identité au moyen de justificatifs vérifiables stockés dans son portefeuille numérique.

    ##### Paramètres du widget Justificatifs vérifiables

    Les paramètres du widget Justificatifs vérifiables comprennent :

    | Paramètres                  | Description                                                                                                                                                                                                                                              |
    | --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | URL                         | La valeur URL utilisée pour générer le QR code. Cette valeur est renvoyée dans la propriété `engagement` de la requête de vérification.                                                                                                                  |
    | Texte du lien               | Texte de remplacement affiché pour les utilisateurs qui ne peuvent pas numériser le QR code et qui préfèrent ouvrir le lien directement sur leur appareil.                                                                                               |
    | Taille                      | La taille du QR code.                                                                                                                                                                                                                                    |
    | Jeton public                | Le jeton d’accès requis pour appeler le point de terminaison d’interrogation. Veuillez vous assurer que le token est généré uniquement avec la portée `read:verification_request` afin d&#39;éviter d&#39;exposer l&#39;accès à d&#39;autres ressources. |
    | ID de vérification          | L&#39;ID de vérification généré lorsque vous lancez la requête de vérification.                                                                                                                                                                          |
    | Temps d&#39;attente maximal | Le délai maximal d&#39;attente pour que le processus de vérification se termine. Une fois ce délai atteint, l&#39;interrogation s&#39;arrête et une erreur est générée, quel que soit le statut de vérification.                                         |

    ##### Valeur de sortie de Justificatifs vérifiables

    Le type de données de la valeur de sortie du widget Justificatifs vérifiables est un `object`.

    ```json lines
    {
      "verifiable_credentials_widget_id": {
        "state": "honored",
        "reason": "...", // Le champ reason contient des renseignements supplémentaires (s'ils sont disponibles) sur l'état de la requête de vérification. 
        "presentation": {} // La présentation contient les claims fournis par le portefeuille en réponse à la définition de présentation.
      }
    }
    ```

<div id="router-node">
  ## Nœud Router
</div>

Les nœuds Router vous permettent d’ajouter des règles pour créer des embranchements conditionnels entre les nœuds.

Par défaut, un nœud Router comporte une seule règle de passage nommée **Default Case** qui le relie à d’autres nœuds. Vous pouvez ajouter des règles supplémentaires selon un ensemble de conditions qu’une variable doit remplir pour ensuite la relier à d’autres nœuds. Pour en savoir plus, consultez [Router.](/docs/fr-CA/customize/forms/routers)
