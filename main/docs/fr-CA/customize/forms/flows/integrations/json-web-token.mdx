Cette liste d’actions <Tooltip tip="JSON Web Token (JWT) : format standard d’ID Token (et souvent d’Access Token) utilisé pour représenter des claims de manière sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> vous permet de générer, de vérifier et de décoder des JWTs dans vos flows.

  ## Configurez votre JWT Vault Connection
</div>

Découvrez comment configurer une Vault Connection pour vos Flow Actions de JSON Web Token à la page [JWT Vault Connection](/docs/fr-CA/customize/forms/vaults/jwt).

<div id="sign-json-web-token">
  ## Signer un JSON Web Token
</div>

Génère un JSON Web Token.

<Frame>![](/docs/images/cdy7uua7fh8z/3XQqUMKrmrc8fUkCsDhI2T/43f7f8d38673d8fa4f9dba7242e044d9/sign-json-web-token.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre          | Description                                                                           |
| ------------------ | ------------------------------------------------------------------------------------- |
| Payload            | Données à encoder. Nous recommandons de les formater conformément aux normes OpenID.  |
| Sujet              | Identifie le sujet du JWT.                                                            |
| Émetteur           | Identifie l’entité qui a émis le JWT.                                                 |
| Audience           | Identifie les destinataires visés par le JWT. Par exemple : admin.your&#95;domain.com |
| Délai d’expiration | Indique le moment à partir duquel le JWT ne doit plus être accepté pour traitement.   |

<div id="output-object">
  ### Objet de sortie
</div>

| Propriété | Type   | Description                              |
| --------- | ------ | ---------------------------------------- |
| `token`   | String | Une chaîne de caractères JSON Web Token. |

<div id="output-object-example">
  ### Exemple d’objet de sortie
</div>

```json lines
{
  "token": "eyJhbGciOiJIUzI1N..."
}
```

<div id="decode-json-web-token">
  ## Décoder un JSON Web Token
</div>

Décode le JSON Web Token fourni.

<Frame>![](/docs/images/cdy7uua7fh8z/7I25WVtppllW6qhC5sALvH/70143bbe0d96920e1d9dfabbb1d6aeff/decode-json-web-token.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre           | Description                                           |
| ------------------- | ----------------------------------------------------- |
| Token (obligatoire) | Chaîne de caractères JSON Web Token qui sera décodée. |

<div id="output-object">
  ### Objet de sortie
</div>

| Propriété | Type  | Description                                   |
| --------- | ----- | --------------------------------------------- |
| `payload` | objet | Le contenu décodé et valide du JSON Web Token |

<div id="output-object-example">
  ### Exemple d’objet de sortie
</div>

```json lines
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "1234567890",
    "name": "John Doe",
    "iat": 1516239022
  },
  "signature": "SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c"
}
```

<div id="verify-json-web-token">
  ## Vérifier le JSON Web Token
</div>

Vérifie les données du JSON Web Token pour déterminer s’il est toujours intact ou s’il a été modifié, afin d’en garantir l’authenticité.

<Frame>![](/docs/images/cdy7uua7fh8z/15MDJcfZqtC46h6mRPOGcz/55e0785db066a54f836d30199fa5f295/verify-json-web-token.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre           | Description                                 |
| ------------------- | ------------------------------------------- |
| Token (obligatoire) | Chaîne du JSON Web Token qui sera vérifiée. |
| émetteur            | émetteur du JWT qui sera vérifié.           |
| Audience            | Audience destinataire du JWT à vérifier.    |

<div id="output-object">
  ### Objet de sortie
</div>

| Propriété | Type    | Description                                                                      |
| --------- | ------- | -------------------------------------------------------------------------------- |
| `valid`   | Booléen | Retourne `true` ou `false` selon que le JWT possède ou non une signature valide. |
| `cause`   | Chaîne  | Si la propriété `valid` est `false`, un message s’affiche.                       |
| `payload` | Objet   | Le contenu décodé et valide du JSON Web Token.                                   |

<div id="output-object-example">
  ### Exemple d’objet de sortie
</div>

```json lines
{
  "valid": true,
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "1234567890",
    "name": "Jane Doe",
    "iat": 1516239022
  },
  "signature": "SflKxwRJSMe..."
}
```

```json lines
{
  "valid": false,
  "cause": "INVALID_SIGNATURE"
}
```
