Cette liste d’actions de logique vous permet d’ajouter une logique conditionnelle au formulaire, de stocker des variables partagées, de mapper des valeurs et d’afficher des messages d’erreur personnalisés.

<div id="ifthen-condition">
  ## Condition if/then
</div>

Ajoute une action conditionnelle avec des branches Vrai/Faux pour personnaliser le comportement du flux en fonction des données d’entrée ou des propriétés.

<Frame>![](/docs/images/cdy7uua7fh8z/4ehTUSu8W1LibUd1uj3x5s/00d76b15c68746ed74e44127804c9557/if-then-condition-action.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre | Description                                                                                                                                                                               |
| --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Condition | Action conditionnelle configurable qui déclenche les branches `True` / `False` selon le résultat. Vous pouvez utiliser les opérateurs `AND` / `OR` pour imbriquer différentes conditions. |

<div id="output-object">
  ### Objet de sortie
</div>

| Propriété | Type    | Description                                                   |
| --------- | ------- | ------------------------------------------------------------- |
| `result`  | Boolean | Retourne `true` ou `false` selon le résultat de la condition. |

<div id="output-object-example">
  ### Exemple d’objet de sortie
</div>

```json lines
{
  "result": true
}
```

<div id="store-shared-variable">
  ## Stocker une variable partagée
</div>

Stocke des variables dans un état privé accessible dans votre formulaire et dans tous ses flux.

Vous pouvez référencer ces variables partagées à l’aide de `{{vars.*}}`**.** Pour en savoir plus, consultez [Variables](/docs/fr-CA/customize/forms/variables).

<Frame>![](/docs/images/cdy7uua7fh8z/4MO9wvChhnCJOg9OAdWuEf/98182a2bd8b6cf4c4baebefb96bce121/stored-shared-variable.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre         | Description                                                                                                                                                                |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Champs de données | Les variables à stocker dans un état privé pour les utiliser dans votre formulaire et tout au long de son flux. Utilisez des paires clé-valeur pour ajouter des variables. |

<div id="show-error-message">
  ## Afficher un message d’erreur
</div>

Affiche un message d’erreur personnalisé à l’utilisateur.

<Warning>
  Cette action de Flow met fin au Flow. Les actions insérées après celle-ci ne seront pas exécutées.
</Warning>

<Frame>![](/docs/images/cdy7uua7fh8z/1Kd4SyaBtAC9S8nfWFeGfh/6e42b16c0ab69dc6d026fb3601133fbe/show-error-message-flow-action.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre        | Description                                     |
| ---------------- | ----------------------------------------------- |
| Message d’erreur | Le message d’erreur à afficher à l’utilisateur. |

<div id="input-value-mapping">
  ## Mappage des valeurs d’entrée
</div>

Associe une variable d’entrée à une sortie selon ses valeurs.

<Frame>![](/docs/images/cdy7uua7fh8z/4VtAr4vCSXtKlSaxtiXQhV/bcb07221f688483aa2010d80394ae630/input-value-mapping.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre                     | Description                                                                                                                       |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| Valeur d’entrée (obligatoire) | La variable d’entrée.                                                                                                             |
| Table de mappage              | Mappe une valeur de variable d’entrée à une valeur de sortie.                                                                     |
| Valeur de sortie par défaut   | Définit une valeur de sortie par défaut à utiliser lorsqu’aucune valeur de sortie n’est indiquée dans la table de mappage.      |

<div id="output-object">
  ### Objet de sortie
</div>

| Propriété | Type   | Description                                      |
| --------- | ------ | ------------------------------------------------ |
| `result`  | String | Retourne la valeur mappée à une valeur d’entrée. |

<div id="output-object-example">
  ### Exemple d’objet de sortie
</div>

```json lines
{
	"result": "Please enter a valid email address."
}
```
