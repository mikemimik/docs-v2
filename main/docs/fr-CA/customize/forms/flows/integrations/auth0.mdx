<div id="custom-api-request">
  ## Requête d’API personnalisée
</div>

Envoyez une requête d’API personnalisée aux endpoints de l’API Auth0.

<Frame>![Dashboard > Personnaliser > Flows > Intégration > Auth0](/docs/images/cdy7uua7fh8z/66uV9iuhwfnXUdE1quLOyx/2ae3ff26335acbcd172cbb40618a5a90/Screenshot_2025-01-24_at_18.50.26.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre        | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Chemin           | Le chemin absolu du point de terminaison de l’API.<br />( Exemple : `/api/v2/organizations` )<br />**Remarque :** Certains endpoints, par exemple : [ Get a user ](https://auth0.com/docs/api/management/v2/users/get-users-by-id), exigent que les paramètres URI soient encodés lorsqu’ils contiennent des caractères non alphanumériques, comme `-`, `.`, `_` ou `~` ([caractères](https://datatracker.ietf.org/doc/html/rfc3986#section-2.3)).<br />Pour ce faire, vous pouvez utiliser la fonction encodeURIComponent :<br />`{{ functions.encodeURIComponent(context.user.user_id) }}` |
| Méthode          | Les méthodes HTTP offertes comprennent : <ul> <li>`GET`</li> <li>`POST`</li> <li>`PUT`</li> <li>`PATCH`</li> <li>`DELETE`</li> </ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Corps            | Le corps de la requête.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| En-têtes         | Les en-têtes de la requête.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Paramètres d’URL | Les paramètres d’URL. Ces paramètres sont automatiquement encodés dans l’URL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |

<div id="output-object">
  ### Objet de sortie
</div>

Consultez l’[Auth0 Management API](https://auth0.com/docs/api/management/v2/introduction) pour connaître les réponses attendues et les codes d’état.

<div id="send-email">
  ## Envoyer un courriel
</div>

Envoyez un courriel personnalisé à l’aide de votre [service de courriel configuré](/docs/fr-CA/customize/email).

<Warning>
  Le [Service de courriel](/docs/fr-CA/customize/email) doit être activé avant d’utiliser cette Action.
</Warning>

<Frame>![](/docs/images/forms/flows/integrations/auth0-notifications-send-email.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre                                                | Description                                                                                                                                                                                                                                                        |
| -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Utiliser une adresse courriel d’expéditeur personnalisée | Par défaut, nous utiliserons l’adresse courriel d’expéditeur que vous avez configurée dans les paramètres personnalisés de votre fournisseur de courriel. Activez cette option si vous souhaitez utiliser une autre adresse courriel ou un autre nom d’expéditeur. |
| Adresse courriel du destinataire                         | L’adresse courriel du destinataire.                                                                                                                                                                                                                                |
| Variables personnalisées                                 | Ajoutez des variables personnalisées à insérer dans votre modèle de courriel. Une fois configurées, vous pouvez y faire référence à l’aide de la syntaxe `{{custom_vars.your_variable_key}}`.                                                                      |
| Objet                                                    | L’objet du courriel. Prend en charge les variables personnalisées et la syntaxe Liquid.                                                                                                                                                                            |
| Message                                                  | Le message du courriel est au format HTML, utilise la syntaxe Liquid et prend en charge les variables personnalisées.                                                                                                                                              |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Objet et Message peuvent utiliser une combinaison de la syntaxe Liquid avec des [variables courantes](https://auth0.com/docs/customize/email/email-templates/supported-liquid-syntax) ou des variables personnalisées à l’aide de la syntaxe `{{custom_vars.your_variable_key}}`.

  Utilisez des variables personnalisées pour transmettre de l’information entre le contexte de Forms / Flows et le contenu d’Objet et de Message.
</Callout>

<div id="output-object">
  ### Objet de sortie
</div>

| Propriété | Type    | Description                                                                                           |
| --------- | ------- | ----------------------------------------------------------------------------------------------------- |
| success   | Boolean | Indique si le courriel a bien été mis en file d’attente pour l’envoi, que la livraison échoue ou non. |

<div id="output-object-example">
  ### Exemple d’objet de sortie
</div>

```json
{
  "success": true
}
```

<div id="send-sms">
  ## Envoyer un SMS
</div>

Envoyez un SMS personnalisé à l’aide de votre [fournisseur de téléphonie configuré](https://auth0.com/docs/customize/phone-messages/configure-phone-messaging-providers).

<Warning>
  Le [fournisseur de téléphonie](https://auth0.com/docs/customize/phone-messages/configure-phone-messaging-providers) doit être activé pour permettre l’envoi de messages texte avant d’utiliser cette Action.
</Warning>

<Frame>![](/docs/images/forms/flows/integrations/auth0-notifications-send-sms.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre                | Description                                                                                                                                                                                                                                       |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| De                       | Par défaut, nous utiliserons le numéro de téléphone de l’émetteur configuré dans vos paramètres personnalisés du fournisseur de téléphone. Saisissez une valeur personnalisée si vous souhaitez utiliser un autre numéro de téléphone d’émetteur. |
| À                        | Le numéro de téléphone du destinataire.                                                                                                                                                                                                           |
| Variables personnalisées | Ajoutez des variables personnalisées à insérer dans votre modèle de message téléphonique. Une fois configurées, vous pouvez les référencer à l’aide de la syntaxe `{{custom_vars.your_variable_key}}`.                                                       |
| Message                  | Le message téléphonique utilise la syntaxe Liquid et prend en charge les variables personnalisées.                                                                                                                                                |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le Message peut combiner la syntaxe Liquid avec des [variables communes](https://auth0.com/docs/customize/email/email-templates/supported-liquid-syntax) ou des variables personnalisées au moyen de la syntaxe `{{custom_vars.your_variable_key}}`.

  Utilisez des variables personnalisées pour transmettre de l’information entre le contexte de Forms / Flows et le contenu du Message.
</Callout>

<div id="output-object">
  ### Objet de sortie
</div>

| Propriété | Type    | Description                                                                                                       |
| --------- | ------- | ----------------------------------------------------------------------------------------------------------------- |
| success   | Boolean | Indique si le message téléphonique a bien été mis en file d’attente pour l’envoi, que la livraison échoue ou non. |

<div id="output-object-example">
  ### Exemple d’objet de sortie
</div>

```json
{
  "success": true
}
```
