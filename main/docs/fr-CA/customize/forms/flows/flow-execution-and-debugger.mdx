---
description: Décrit comment passer en revue les exécutions de flux et déboguer celles de Forms.
title: Exécution et débogage
---

Chaque fois qu’un flux s’exécute, Auth0 enregistre une entrée que vous pouvez utiliser pour en déboguer l’exécution. Cela permet de mieux comprendre la cause d’une erreur d’exécution d’un flux. Par exemple : un flux peut échouer en raison de propriétés manquantes, de fautes de frappe, de clés API incorrectes, etc.

<div id="review-flow-executions">
  ## Examiner les exécutions de flux
</div>

Pour examiner une entrée d’exécution d’un flux, suivez les étapes ci-dessous :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Forms**, puis sélectionnez **Flows**.
2. Sélectionnez un flux dans la liste.
3. Sélectionnez **Executions**.

<Frame>![Dashboard > Actions > Forms > Flows > Execution](/docs/images/cdy7uua7fh8z/448jfjHtgCxfIeDOakuHyd/fba64deda0cf920f399de2190e94bdc9/Flows-execution.png)</Frame>

Chaque entrée d’exécution comporte quatre propriétés :

* **Date d’exécution** : La date d’exécution du flux.
* **ID d’exécution** : L’ID unique de l’exécution du flux.
* **Status** : L’étape actuelle de l’exécution du flux. Il existe quatre statuts d’exécution possibles :

  * **Running** : L’exécution est actuellement en cours.
  * **Failed** : L’exécution a échoué en raison d’une erreur interne, comme l’absence de paramètres requis dans une action de flux.
  * **Completed** : L’exécution s’est terminée sans erreur interne.
* **View Details** : Ouvre la fenêtre de débogage de l’exécution.

<Frame>![Dashboard > Actions >  Forms > Flows > Executions](/docs/images/cdy7uua7fh8z/1VqWEaVZg5lrXatXe9dbFB/dbcf7edca5aabdfc544f04cb18be4149/Flow-execution-details.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Completed et Failed sont des statuts finaux. Lorsqu’une exécution atteint l’un ou l’autre de ces statuts, le flux est entièrement traité.
</Callout>

<div id="debug-flow-executions">
  ## Déboguer les exécutions d’un flux
</div>

Pour faciliter le débogage ou le dépannage d’un flux, vous pouvez consulter les données d’entrée, de sortie et d’erreur du flux en accédant à **View Details** de l’exécution.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/5aeniTfHXplDyZfwCKBcgK/a2098a476edde76da10a527b0a606e95/Flow-error.png)</Frame>

Pour déboguer une erreur qui interrompt l’exécution du flux, suivez ces étapes :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Forms**
2. Repérez le flux lié au step node.

   <Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/4LjULYyo0nhtNowm7J5okz/27ea3f8386bb5015d447c48facf6b90f/flow-error-linked.png)</Frame>
3. Sélectionnez **Edit Flow.**
4. Sélectionnez **Executions**.
5. Sélectionnez **View Details** de l’exécution Failed.

Dans l’exemple ci-dessous, la vue **Input** de `Update user` affiche une valeur nulle pour la propriété `user_id`.

<Frame>![Dashboard > Actions > Forms > Flow > Execution](/docs/images/cdy7uua7fh8z/6QNYKDqG9AIOEzQkP6Fnnr/db885e146179a0490eddd5ad6cb518dc/Flow-execution-failed.png)</Frame>

La vue **Error** indique que la validation du schéma de l’action a échoué parce que le paramètre obligatoire `user_id` est manquant. Le flux renvoie une erreur interne qui interrompt son exécution.

<Frame>![Dashboard > Actions > Forms > Flow > Execution > Error](/docs/images/cdy7uua7fh8z/1kaKihtJtv8VvNAyTkW4cC/6b34bcecc8642d7d97aaf2c8dec51ebc/flow-execution-error-view.png)</Frame>

L’examen de l’action **Update user** révèle une erreur de syntaxe dans le paramètre `User ID` : la syntaxe correcte est `{{context.user.user_id}}`. Une fois la syntaxe corrigée, testez le flux pour confirmer. Pour en savoir plus sur la syntaxe des variables, consultez [Variables](/docs/fr-CA/customize/forms/variables).

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/4YLRVNbzK5LjJEQjFXIJXM/5003833784ce5b0d9f63730ed46ff047/Action-typo-error.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si un champ est configuré comme transitoire ou si sa valeur est masquée à l’aide d’une [helper function](/docs/fr-CA/customize/forms/variables#helper-functions), la valeur n’est pas disponible pour le débogage.

  <Frame>![Dashboard > Actions > Forms > Flows > Execution](/docs/images/cdy7uua7fh8z/4MVtjnVD50cG63ocN5a458/c0337a94d1f4c21f28d06e28ec59121e/masked-information.png)</Frame>
</Callout>
