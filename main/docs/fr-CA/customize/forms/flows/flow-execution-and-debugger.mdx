---
description: Décrit comment consulter les exécutions de Flow et déboguer les exécutions de Forms.
title: Exécution et débogueur
---

Chaque fois qu’un flow s’exécute, Auth0 enregistre une entrée que vous pouvez utiliser pour en déboguer l’exécution. C’est utile pour comprendre pourquoi une erreur d’exécution s’est produite dans un flow. Par exemple : un flow peut échouer en raison de propriétés manquantes, de fautes de frappe, de clés API incorrectes, etc.

<div id="review-flow-executions">
  ## Vérifier les exécutions de Flow
</div>

Pour consulter une exécution de Flow, suivez les étapes ci-dessous :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Forms**, puis sélectionnez **Flows**.
2. Sélectionnez un Flow dans la liste.
3. Sélectionnez **Exécutions**.

<Frame>![Dashboard > Actions > Forms > Flows > Exécution](/docs/images/cdy7uua7fh8z/448jfjHtgCxfIeDOakuHyd/fba64deda0cf920f399de2190e94bdc9/Flows-execution.png)</Frame>

Chaque exécution comporte quatre propriétés :

* **Date d’exécution** : La date d’exécution du Flow.
* **ID d’exécution** : L’ID unique de l’exécution du Flow.
* **Statut** : L’étape actuelle de l’exécution du Flow. Il existe quatre statuts d’exécution possibles :

  * **En cours** : L’exécution est en cours.
  * **Échouée** : L’exécution a échoué en raison d’une erreur interne, par exemple des paramètres requis manquants dans une action de Flow.
  * **Terminée** : L’exécution s’est terminée sans erreur interne.
* **afficher les détails** : Ouvre la fenêtre de débogage de l’exécution.

<Frame>![Dashboard > Actions >  Forms > Flows > Exécutions](/docs/images/cdy7uua7fh8z/1VqWEaVZg5lrXatXe9dbFB/dbcf7edca5aabdfc544f04cb18be4149/Flow-execution-details.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Terminée et Échouée sont des statuts finaux. Lorsqu’une exécution atteint l’un ou l’autre de ces statuts, le Flow est entièrement traité.
</Callout>

<div id="debug-flow-executions">
  ## Déboguer les exécutions de Flow
</div>

Pour faciliter le débogage ou le dépannage d’un Flow, vous pouvez examiner ses données d’entrée, de sortie et d’erreur en accédant à **Afficher les détails** de l’exécution.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/5aeniTfHXplDyZfwCKBcgK/a2098a476edde76da10a527b0a606e95/Flow-error.png)</Frame>

Pour déboguer une erreur qui interrompt l’exécution du Flow, suivez ces étapes :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Forms**
2. Repérez le Flow lié au nœud d’étape.

   <Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/4LjULYyo0nhtNowm7J5okz/27ea3f8386bb5015d447c48facf6b90f/flow-error-linked.png)</Frame>
3. Sélectionnez **Modifier le Flow.**
4. Sélectionnez **Exécutions**.
5. Sélectionnez **Afficher les détails** de l’exécution ayant échoué.

Dans l’exemple ci-dessous, la vue **Entrée** de `Update user` affiche une valeur nulle pour la propriété `user_id`.

<Frame>![Dashboard > Actions > Forms > Flow > Execution](/docs/images/cdy7uua7fh8z/6QNYKDqG9AIOEzQkP6Fnnr/db885e146179a0490eddd5ad6cb518dc/Flow-execution-failed.png)</Frame>

La vue **Erreur** indique que la validation du schéma de l’action a échoué parce que le `user_id` requis est manquant. Le Flow renvoie une erreur interne qui interrompt son exécution.

<Frame>![Dashboard > Actions > Forms > Flow > Execution > Error](/docs/images/cdy7uua7fh8z/1kaKihtJtv8VvNAyTkW4cC/6b34bcecc8642d7d97aaf2c8dec51ebc/flow-execution-error-view.png)</Frame>

L’examen de l’action **Update user** révèle une erreur de syntaxe dans le paramètre `User ID` ; la syntaxe correcte est `{{context.user.user_id}}`. Une fois la syntaxe corrigée, testez le Flow pour confirmer. Pour en savoir plus sur la syntaxe des variables, consultez [Variables](/docs/fr-CA/customize/forms/variables).

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/4YLRVNbzK5LjJEQjFXIJXM/5003833784ce5b0d9f63730ed46ff047/Action-typo-error.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si un champ est configuré comme transitoire ou si sa valeur est masquée à l’aide d’une [fonction utilitaire](/docs/fr-CA/customize/forms/variables#helper-functions), la valeur n’est pas disponible pour le débogage.

  <Frame>![Dashboard > Actions > Forms > Flows > Execution](/docs/images/cdy7uua7fh8z/4MVtjnVD50cG63ocN5a458/c0337a94d1f4c21f28d06e28ec59121e/masked-information.png)</Frame>
</Callout>
