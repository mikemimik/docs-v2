Les variables vous permettent d’accéder aux données contenues dans Forms et Flows afin de créer une logique opérationnelle personnalisée et des automatisations.

Voici quelques points de base à considérer lors de l’utilisation des variables :

* Les variables sont entourées d’accolades `{{ variable }}`.
* Si une variable n’existe pas ou ne peut pas être résolue, cela a le même effet qu’une variable `undefined`.
* Vous pouvez utiliser des opérateurs de coalescence nulle pour gérer les variables `null` ou `undefined`, par exemple : `{{fields.foo ?? fields.bar}}`.

<div id="available-variables">
  ## Variables disponibles
</div>

Vous pouvez utiliser différents types de variables pour référencer et transformer les données que vous recueillez auprès des clients avec Forms et Flows.

| Variable                   | Syntaxe           | Où            | Description                                                                 |
| -------------------------- | ----------------- | ------------- | --------------------------------------------------------------------------- |
| Contexte                   | `{{context.*}}`   | Forms / Flows | Accéder aux données de contexte de la transaction en cours                  |
| Champs du formulaire       | `{{fields.*}}`    | Forms / Flows | Accéder aux données de vos champs de formulaire et de vos champs masqués    |
| Variables partagées        | `{{vars.*}}`      | Forms / Flows | Accéder aux données stockées dans les variables partagées                   |
| Sortie des actions de Flow | `{{actions.*}}`   | Flows         | Accéder aux données de la réponse de sortie des actions de Flow précédentes |
| Fonctions utilitaires      | `{{functions.*}}` | Forms / Flows | Fonctions utilitaires servant à transformer les données                     |

<div id="context-variables">
  ### Variables de contexte
</div>

Forms et les Flows héritent automatiquement des variables de contexte de la transaction en cours. Vous pouvez accéder aux variables de contexte suivantes :

* L’objet `user`, avec accès aux propriétés suivantes :

  * `user.user_id`
  * `user.username`
  * `user.name`
  * `user.given_name`
  * `user.family_name`
  * `user.nickname`
  * `user.email`
  * `user.email_verified`
  * `user.phone_number`
  * `user.phone_verified`
  * `user.picture`
  * `user.user_metadata`
  * `user.app_metadata`
  * `user.created_at`
  * `user.updated_at`
  * `user.last_password_reset`
  * `user.identities`
* L’objet `organization`, avec accès aux propriétés suivantes :

  * `organization.id`
  * `organization.name`
  * `organization.display_name`
  * `organization.metadata`
* L’objet `client`, avec accès aux propriétés suivantes :

  * `client.client_id`
  * `client.name`
* L’objet `tenant`, avec accès à la propriété suivante :

  * `tenant.name`
* L’objet `transaction`, avec accès aux propriétés suivantes :

  * `transaction.state`
  * `transaction.correlation_id`
* L’objet `custom_domain`, avec accès aux propriétés suivantes :

  * `custome_domain.domain`
  * `custome_domain.domain_metadata`

<div id="form-fields-variables">
  ### Variables des champs du formulaire
</div>

Les variables de champ contiennent les données recueillies auprès de l’utilisateur dans les champs de saisie du formulaire, ainsi que les données des champs masqués. Les données de chaque champ varient selon son type. Par exemple : `{{fields.first_name}}` fait référence à la valeur saisie dans un champ ayant l’ID `first_name`.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/4D42A6ww4XvFxweoxh04Rd/016c2ed9d25bc2267ed3ed31bf8b2bf6/form-fields-variable.png)</Frame>

<div id="flow-output-actions-variables">
  ### Variables de sortie des actions de Flow
</div>

Lorsqu’une action de Flow est exécutée, elle génère un objet de sortie auquel il est possible de faire référence dans les actions suivantes. Par exemple : `{{actions.generate_otp.code}}` fait référence au code de sortie de l’action ayant l’ID `generate_otp`.

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/6jYtEdYqSGXm3DDv0mxQ0o/2d48012629745e07137bf6ba42f4d829/flow-output-variable.png)</Frame>

<div id="shared-variables">
  ### Variables partagées
</div>

Vous pouvez stocker des variables à partager avec des flows ultérieurs ou même avec des composants de formulaire dans la même transaction. Les variables partagées sont éphémères et associées à un parcours de formulaire précis. Leurs valeurs ne sont pas accessibles à l’extérieur de ce parcours. Par exemple : `{{vars.external_id}}` fait référence à la valeur d’une variable partagée dont l’ID est `external_id`.

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/4571ExRTHL0KZKJKX9pNwj/abb631b8608a6356b0e5b3784cf8b02f/flow-shared-variable.png)</Frame>

<Warning>
  Variables partagées :

  * sont internes (côté serveur) et ne sont pas exposées au navigateur (côté client).
  * peuvent être résolues et exposées au navigateur si vous faites référence à une variable partagée dans un composant de formulaire.
  * peuvent contenir des renseignements sensibles; faites donc preuve de prudence lorsque vous accédez à une variable partagée afin d’éviter toute exposition inutile ou involontaire.
</Warning>

<div id="helper-functions">
  ### Fonctions utilitaires
</div>

Les fonctions utilitaires vous permettent de transformer des données et d’effectuer des opérations simples. Par exemple : `{{functions.toString(fields.privacy_policies)}}` transforme la valeur du champ ayant l’ID `privacy_policies` à l’aide de la fonction `toString()`.

<div id="variables-best-practices">
  ## Bonnes pratiques pour les variables
</div>

<div id="use-meaningful-ids">
  ### Utilisez des ID explicites
</div>

Lorsque vous ajoutez un champ de Form ou une action de Flow, l’ID est généré automatiquement.

Remplacez l’ID par une valeur plus descriptive afin de pouvoir reconnaître l’information qu’il contient. Par exemple, si vous créez un champ de Form qui stocke le prénom de l’utilisateur, remplacez l’ID par `first_name`.

<div id="be-mindful-when-updating-existing-ids">
  ### Faites preuve de prudence lorsque vous mettez à jour des IDs existants
</div>

Si une variable est déjà utilisée et que vous devez modifier l’ID, repérez tous les endroits où il est référencé et mettez-le à jour en conséquence afin d’éviter des valeurs non définies ou vides.

<div id="check-flow-executions">
  ### Vérifier les exécutions du Flow
</div>

Si vous voyez une valeur de variable inattendue, utilisez la vue [**Exécutions**](/docs/fr-CA/customize/forms/flows/flow-execution-and-debugger) pour résoudre le problème lié à la variable.

<div id="use-the-autocomplete-menu">
  ### Utilisez le menu d’autocomplétion
</div>

Le menu d’autocomplétion vous aide à trouver et à référencer correctement les variables existantes.
