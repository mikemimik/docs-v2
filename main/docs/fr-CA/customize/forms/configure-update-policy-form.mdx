<Card title="Avant de commencer">
  1. Créez une [Machine-to-Machine Application](/docs/fr-CA/get-started/auth0-overview/create-applications/machine-to-machine-apps) avec les scopes suivants activés :

  * `read:users`
  * `update:users`
  * `create:users`
  * `read:users_app_metadata`
  * `update:users_app_metadata`
  * `create:users_app_metadata`

  2. Ajoutez une [connexion Vault](/docs/fr-CA/customize/forms/vault) à l’aide des identifiants de l’application M2M.
</Card>

Forms for Actions vous permet de créer des flux personnalisés de politique de confidentialité et d’enregistrer le consentement dans l’objet `app_metadata`.

<Frame>![Dashboard > Actions > Forms > cas d’utilisation de la politique de confidentialité](/docs/images/cdy7uua7fh8z/6i68dMvHDTijfRmSgN1x8Q/ad900e7d4c0fb34645921435472b8684/privavy-policy-form.png)</Frame>

Les sections ci-dessous expliquent comment créer un formulaire de mise à jour de la politique à l’aide de nœuds et de flux, ainsi que les étapes pour ajouter votre formulaire à une Action [Post Login](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="create-a-form-from-scratch">
  ## Créer un formulaire à partir de zéro
</div>

Suivez ces étapes pour créer un formulaire et recueillir des renseignements :

1. Ouvrez l’éditeur de formulaires en sélectionnant **Auth0 Dashboard &gt; Actions &gt; Forms.**
2. Sélectionnez **Créer un formulaire** &gt; **Commencer à partir de zéro**.

Par défaut, un nouveau formulaire contient un nœud Start, un nœud Step et un nœud Ending screen.

<Frame>![Dashboard > Forms > Cas d’utilisation > Formulaire](/docs/images/cdy7uua7fh8z/2TZ9jSjgEcARFpULI6hz4R/a4319a1bd482f2ee5cc23100fdb2f42c/New-form-Forms-for-action-use-case.png)</Frame>

<div id="configure-the-step-node">
  ### Configurer le nœud Step
</div>

Le nœud Step est l’interface graphique que les utilisateurs voient. Ajoutez des [Champs](/docs/fr-CA/customize/forms/nodes-and-components) au nœud Step pour recueillir la nouvelle valeur d’acceptation de l’accord en suivant ces étapes :

1. Faites glisser un champ **Rich text** du **menu Components** vers le nœud **Step**.

   * **Rich text** : Entrez une description des mises à jour apportées à votre politique de confidentialité.
   * Sélectionnez **Enregistrer**.
2. Faites glisser un champ **Legal** vers le nœud **Step**.

   * **ID** : Entrez `privacy_policies`.
   * **Required** : Cochez la case.
   * **Legal** : Entrez `I agree to the Privacy Policy`.
   * Sélectionnez **Enregistrer**.
3. Sélectionnez **Publier** pour enregistrer.

<Frame>![Dashboard > Actions > Forms > Cas d’utilisation du nœud Step de la politique de confidentialité](/docs/images/cdy7uua7fh8z/59zFFjQjAnjogkc2QGIqqJ/0ad66b1b636544122aa4e65968b76c2c/privacy-policy-step-node.png)</Frame>

<div id="configure-the-flow-node">
  ### Configurer le nœud de flux
</div>

Ajoutez un nœud **Flux** après le nœud **Step** pour mettre à jour `app_metadata` et reprendre le flux d’authentification en suivant ces étapes :

1. Sélectionnez **Flux** au bas de l’éditeur de formulaires.
2. Supprimez le lien existant entre les nœuds **Step** et **Écran de fin**.
3. Sélectionnez le nouveau **Flux** &gt; **Cliquez pour ajouter un flux** &gt; **Créer un nouveau flux.**

   * Saisissez `Update app_metadata` dans le champ **Nom**.
   * Sélectionnez **Créer**.
   * Sélectionnez **Enregistrer**.
4. Reliez le nœud **Flux** aux nœuds **Step** et **Écran de fin**, comme illustré ci-dessous.
5. Sélectionnez **Publier** pour enregistrer.

<Frame>![Dashboard > Actions > Forms > Formulaire](/docs/images/cdy7uua7fh8z/6YIkzm7SrpM9LW76AXuRzh/353c518fe82bbaa440836da1d83596d6/link-policies.png)</Frame>

Ajoutez une action Auth0 de mise à jour de l’utilisateur au flux en suivant ces étapes :

1. Sélectionnez le **Flux** **Update app&#95;metadata** &gt; **Modifier le flux** pour ouvrir l’éditeur de flux dans un nouvel onglet.
2. Sous StartAction, sélectionnez l’icône **+** pour ajouter une action **Mettre à jour l’utilisateur**. Remplissez les champs ci-dessous, puis sélectionnez **Enregistrer** pour continuer.

   * **Connexion** : Dans le menu déroulant, sélectionnez la connexion Vault de votre application M2M.

   * **ID utilisateur** : Saisissez `{{context.user.user_id}}`.

   * **Corps** : Copiez-collez le code suivant pour mettre à jour `app_metadata` avec les propriétés `privacy_policies` et `privacy_policies_timestamp`.

     ```text lines
     {
       "app_metadata": {
         "privacy_policies": true,
         "privacy_policies_timestamp": "{{ functions.toTimestamp() }}"
       }
     }
     ```

   * Sélectionnez **Enregistrer**.
3. Sélectionnez **Publier** pour enregistrer.

<Frame>![Dashboard > Actions > Forms > Code du cas d’utilisation de la politique de confidentialité](/docs/images/cdy7uua7fh8z/CvMNIwWrOUwaFW4X3G28p/1fb70ae57a20749d9ffaaba8dc0b7e26/update-user-privacy.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le flux **Update app&#95;metadata** ajoute la valeur `privacy_policies true` à l’objet `app_metadata` et génère aussi un horodatage à l’aide d’une [fonction utilitaire.](/docs/fr-CA/customize/forms/variables)
</Callout>

<div id="retrieve-the-form-render-code">
  ## Récupérer le code de rendu du formulaire
</div>

Récupérez le code de rendu de votre formulaire pour afficher le formulaire au moyen d’une Action Post Login personnalisée en suivant ces étapes :

* Dans l’éditeur de formulaires, sélectionnez **`<> Render`**.
* Sélectionnez **Copy**.

<Frame>![Dashboard > Actions > Forms > Form > Embed](/docs/images/cdy7uua7fh8z/2EIUzc539sq9lwoTKbjoJv/496a864c0fc44a782c9cadca119dbfb0/embed.png)</Frame>

<div id="create-a-post-login-action">
  ## Créer une Action Post Login
</div>

Créez une Action Post Login pour afficher votre formulaire en suivant ces étapes :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Flows &gt; Login.**
2. Sélectionnez l’icône **+**, puis **Build from scratch** :

   * **Name** : Entrez `Render Update Policy Form`.
   * **Trigger** : Sélectionnez `Login / Post Login`.
   * **Runtime** : Sélectionnez la version recommandée.
3. Sélectionnez **Create**.

Pour configurer l’Action Post Login :

1. Supprimez le code existant dans l’éditeur de code.
2. Collez le code de rendu du formulaire dans l’éditeur de code.
3. Modifiez le code pour définir la logique conditionnelle qui affichera le formulaire.

   ```javascript lines
   /**
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   exports.onExecutePostLogin = async (event, api) => {
     const FORM_ID = 'REPLACE_WITH_YOUR_FORM_ID';

     if (event.user.app_metadata.privacy_policies !== true) {
       api.prompt.render(FORM_ID);
     }
   }

   exports.onContinuePostLogin = async (event, api) => { }
   ```

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     L’exemple de code ci-dessus affichera le formulaire si la propriété `privacy_policies` est absente des app metadata de l’utilisateur.
   </Callout>
4. Sélectionnez **Deploy.**
5. Glissez-déposez l’Action **Render Update Policy Form** dans le flux **Login**.

   <Frame>![Dashboard > Forms > Use Case > Action de connexion pour afficher le formulaire de mise à jour de la politique](/docs/images/cdy7uua7fh8z/5TOwDgevZsBqd7yZynUYWU/41ca909dc3007aed4f40ff794a05efa5/postlogin.png)</Frame>
6. Sélectionnez **Apply.**

<div id="test-implementation">
  ### Tester la mise en œuvre
</div>

Testez la mise en œuvre en suivant ces étapes :

1. Connectez-vous avec un utilisateur existant dont l’attribut app`_metadata` `privacy_policies` ne contient aucune valeur.
2. L’Action Post Login dans le flux Login affichera le formulaire et demandera l’acceptation.
3. Sélectionnez **Auth0 Dashboard &gt; Gestion des utilisateurs &gt; Utilisateurs,** repérez l’utilisateur et vérifiez que l’attribut `app_metadata` `privacy_policies` contient la valeur `true` ainsi qu’un horodatage.
