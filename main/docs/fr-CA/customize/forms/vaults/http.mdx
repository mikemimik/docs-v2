<div id="configure-your-http-vault-connection-options">
  ## Configurez les options de votre connexion Vault HTTP
</div>

Vous pouvez configurer la connexion Vault HTTP avec différentes options d’authentification, puis l’utiliser dans vos HTTP Request Flow Actions.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configurer votre connexion Vault HTTP pour Bearer Token
</div>

Pour configurer une connexion Vault HTTP à l’aide de l’autorisation **Bearer Token**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre    | Description                                                      |
| ------------ | ---------------------------------------------------------------- |
| Bearer Token | Jeton statique utilisé pour l’autorisation d’un service externe. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configurez votre connexion Vault HTTP pour OAuth Client Credentials
</div>

Pour configurer une connexion Vault HTTP à l’aide de l’autorisation **Client Credentials**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre                      | Description                                                                                               |
| ------------------------------ | --------------------------------------------------------------------------------------------------------- |
| Point de terminaison du tenant | Point de terminaison du tenant du serveur d’autorisation qui émet les jetons d’accès.                     |
| Client ID                      | Client ID de votre application M2M autorisée pour l’API personnalisée.                                    |
| Client Secret                  | Client Secret de votre application M2M autorisée pour l’API personnalisée.                                |
| Audience (facultatif)          | Audience de l’API personnalisée. Habituellement utilisée pour les API personnalisées définies dans Auth0. |
| Resource (facultatif)          | Ressource de l’API personnalisée. Paramètre standard dont le sens est semblable à celui d’Audience.       |
| Scope (facultatif)             | Liste de scopes au format chaîne de caractères, séparés par des espaces.                                  |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configurer votre connexion Vault HTTP pour l’autorisation par API Key
</div>

Pour configurer une connexion Vault HTTP avec l’autorisation **API Key**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre | Description                                                                                                          |
| --------- | -------------------------------------------------------------------------------------------------------------------- |
| Nom       | Nom de la clé API à ajouter comme en-tête ou paramètre de requête.                                                   |
| Valeur    | Valeur de la clé API à ajouter comme en-tête ou paramètre de requête.                                                |
| Ajouter à | Par défaut, il s’agit de l’en-tête. Détermine si la clé API doit être ajoutée comme en-tête ou paramètre de requête. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configurer votre connexion Vault HTTP pour l’authentification Basic
</div>

Pour configurer une connexion Vault HTTP à l’aide de l’autorisation **authentification Basic**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre         | Description                                                                                                              |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------ |
| Nom d’utilisateur | Nom d’utilisateur à concaténer et à encoder en base64, puis à ajouter comme paramètre de l’en-tête d’autorisation Basic. |
| Mot de passe      | Mot de passe à concaténer et à encoder en base64, puis à ajouter comme paramètre de l’en-tête d’autorisation Basic.      |

<Warning>
  La configuration connexion Vault HTTP avec authentification Basic est appelée à remplacer l’option actuellement intégrée à l’action HTTP Request de Flows.
</Warning>
