  ## Configurer les options de votre connexion HTTP Vault
</div>

connexion HTTP Vault peut être configurée avec différentes options d’authentification, puis utilisée dans vos HTTP Request Flow Actions.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configurer votre connexion HTTP Vault avec le Bearer Token
</div>

Pour configurer une connexion HTTP Vault avec l’autorisation **Bearer Token**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre    | Description                                                           |
| ------------ | --------------------------------------------------------------------- |
| Bearer Token | Jeton statique utilisé pour l’autorisation auprès du service externe. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configurer votre connexion HTTP Vault pour OAuth Client Credentials
</div>

Pour configurer une connexion HTTP Vault avec l’autorisation **Client Credentials**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre                      | Description                                                                                             |
| ------------------------------ | ------------------------------------------------------------------------------------------------------- |
| Point de terminaison du tenant | Point de terminaison du tenant du serveur d’autorisation qui émet les jetons d’accès.                   |
| Client ID                      | Client ID de votre application M2M autorisée pour l’API personnalisée.                                  |
| Client Secret                  | Client Secret de votre application M2M autorisée pour l’API personnalisée.                              |
| Audience (facultatif)          | Audience de l’API personnalisée. Généralement utilisée pour les API personnalisées définies dans Auth0. |
| Resource (facultatif)          | Ressource de l’API personnalisée. Paramètre standard dont le sens est semblable à celui d’Audience.     |
| Scope (facultatif)             | Liste de portées au format chaîne de caractères, séparées par des espaces.                              |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configurer votre connexion HTTP Vault pour une clé API
</div>

Pour configurer une connexion HTTP Vault à l’aide de l’autorisation par **clé API**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre | Description                                                                                                           |
| --------- | --------------------------------------------------------------------------------------------------------------------- |
| Nom       | Nom de la clé API à ajouter à l’en-tête ou au paramètre de requête.                                                   |
| Valeur    | Valeur de la clé API à ajouter à l’en-tête ou au paramètre de requête.                                                |
| Ajouter à | Par défaut, il s’agit de l’en-tête. Détermine si la clé API doit être ajoutée à l’en-tête ou au paramètre de requête. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configurer votre connexion HTTP Vault pour l’authentification Basic
</div>

Pour configurer une connexion HTTP Vault à l’aide de l’**authentification Basic**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre         | Description                                                                                                       |
| ----------------- | ----------------------------------------------------------------------------------------------------------------- |
| Nom d’utilisateur | Nom d’utilisateur à concaténer et à encoder en base64, puis à ajouter au paramètre d’en-tête Authorization Basic. |
| Mot de passe      | Mot de passe à concaténer et à encoder en base64, puis à ajouter au paramètre d’en-tête Authorization Basic.      |

<Warning>
  La configuration connexion HTTP Vault avec authentification Basic est appelée à remplacer l’option actuellement intégrée à l’Action HTTP Request de Flows.
</Warning>
