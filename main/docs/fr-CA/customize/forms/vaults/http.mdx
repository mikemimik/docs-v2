<div id="configure-your-http-vault-connection-options">
  ## Configurez les options de votre connexion Vault HTTP
</div>

connexion Vault HTTP peut être configurée avec différentes options d’authentification, puis être utilisée dans vos HTTP Request Flow Actions.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configurez votre connexion Vault HTTP pour un jeton porteur
</div>

Pour configurer une connexion Vault HTTP à l’aide de l’autorisation **Bearer Token**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre    | Description                                                             |
| ------------ | ----------------------------------------------------------------------- |
| Bearer Token | Jeton statique utilisé pour l’autorisation auprès d’un service externe. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configurez votre connexion Vault HTTP pour OAuth Client Credentials
</div>

Pour configurer une connexion Vault HTTP avec l’autorisation **Client Credentials**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre                      | Description                                                                                             |
| ------------------------------ | ------------------------------------------------------------------------------------------------------- |
| Point de terminaison du tenant | Point de terminaison du tenant du serveur d’autorisation qui émet les jetons d’accès.                   |
| Client ID                      | Client ID de votre application M2M autorisée à utiliser l’API personnalisée.                            |
| Client Secret                  | Client Secret de votre application M2M autorisée à utiliser l’API personnalisée.                        |
| Audience (facultatif)          | Audience de l’API personnalisée. Généralement utilisée pour les API personnalisées définies dans Auth0. |
| Resource (facultatif)          | Ressource de l’API personnalisée. Paramètre standard dont le sens est semblable à celui d’Audience.     |
| Scope (facultatif)             | Liste de portées au format chaîne de caractères, séparées par des espaces.                              |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configurez votre connexion Vault HTTP pour une clé API
</div>

Pour configurer une connexion Vault HTTP à l’aide de l’autorisation **clé API**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre | Description                                                                                                      |
| --------- | ---------------------------------------------------------------------------------------------------------------- |
| Nom       | Nom de la clé API à ajouter comme en-tête ou paramètre de requête.                                                |
| Valeur    | Valeur de la clé API à ajouter comme en-tête ou paramètre de requête.                                             |
| Ajouter à | Par défaut, c’est En-tête. Détermine si la clé API doit être ajoutée comme en-tête ou comme paramètre de requête. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configurez votre connexion Vault HTTP pour l’authentification Basic
</div>

Pour configurer une connexion Vault HTTP à l’aide de l’**authentification Basic**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre         | Description                                                                                                             |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------- |
| Nom d’utilisateur | Nom d’utilisateur à concaténer et à encoder en base64, puis à ajouter comme paramètre de l’en-tête Authorization Basic. |
| Mot de passe      | Mot de passe à concaténer et à encoder en base64, puis à ajouter comme paramètre de l’en-tête Authorization Basic.      |

<Warning>
  La configuration connexion Vault HTTP avec authentification Basic remplacera éventuellement l’option actuellement intégrée à l’Action HTTP Request de Flows.
</Warning>
