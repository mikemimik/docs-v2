title: Extension User Import / Export
---

<Warning>
  L’**extension User Import/Export** sera **dépréciée** en septembre 2025. Vous pouvez dès maintenant accéder aux mêmes fonctionnalités [directement dans le Dashboard de gestion Auth0](/docs/fr-CA/manage-users/user-migration/bulk-user-import-export).
</Warning>

L’**extension User Import / Export** vous permet de :

* Importer en bloc vos utilisateurs de base de données existants dans Auth0
* Rechercher et exporter certains (ou tous) de vos utilisateurs de base de données Auth0

Pour obtenir la liste des champs de profil utilisateur pouvant être importés et exportés, consultez [User Profile Attributes](/docs/fr-CA/manage-users/user-accounts/user-profiles/user-profile-structure).

Vous devez être Admin du Dashboard pour utiliser cette extension.

<div id="install-the-extension">
  ## Installez l’extension
</div>

Pour installer cette extension, accédez à [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions), puis sélectionnez **User Import / Export**.

L’extension ne nécessite aucune configuration supplémentaire avant son installation; sélectionnez donc **Installer** dans la fenêtre contextuelle d’information pour poursuivre.

<div id="use-the-extension">
  ## Utiliser l’extension
</div>

Après avoir installé votre extension, elle apparaîtra dans la liste de vos extensions installées. Cliquez sur **Import / Export Extension** pour la lancer.

La première fois que vous lancerez l’extension, on vous demandera de l’autoriser à accéder à votre compte Auth0 pour les activités indiquées.

<Frame>![Dashboard Extensions autoriser l’extension Users Import/Export](/docs/images/cdy7uua7fh8z/5CCKmxUv1HN5nDj9jL8qel/d7f02ccf3f4be166b00bbf217e14144f/Screen_Shot_2021-05-18_at_9.11.01_PM.png)</Frame>

Cliquez sur la **coche** pour continuer.

Il y a deux façons d’utiliser cette extension :

* Importer en bloc les utilisateurs de votre base de données existante dans Auth0
* Rechercher et exporter certains (ou tous) de vos utilisateurs de base de données Auth0

Ces deux cas d’utilisation sont expliqués plus en détail ci-dessous.

<div id="import-users">
  ### Importer des utilisateurs
</div>

Par défaut, chaque fois que vous ouvrez l’extension, l’écran **Importation des utilisateurs** s’affiche (si vous êtes sur l’écran d’exportation, vous pouvez revenir à cet écran en cliquant sur **Import** dans la barre de navigation de gauche).

Pour importer vos utilisateurs, glissez-déposez un fichier JSON valide ([schéma et exemples ici](/docs/fr-CA/manage-users/user-migration/bulk-user-import-database-schema-and-examples)) dans la zone portant la mention **Drop your file here, or click to select**. Vous pouvez aussi cliquer sur cette zone pour parcourir vos fichiers et sélectionner le fichier JSON approprié. Le fichier JSON doit contenir la liste des utilisateurs que vous comptez importer.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous utilisez un fichier d’exportation provenant d’un tenant Auth0, vous devez convertir le fichier exporté de `ndjson` en [JSON](/docs/fr-CA/manage-users/user-migration/bulk-user-exports#json-compatible-format). Pour conserver les mêmes ID utilisateur, vous devez supprimer le préfixe `auth0|` de tous les ID utilisateur importés.

  Le processus d’importation ajoute automatiquement le préfixe `auth0|` aux ID utilisateur importés. Si vous ne supprimez pas le préfixe `auth0|` avant l’importation, les ID utilisateur deviendront `auth0|auth0|...`
</Callout>

Sélectionnez la connexion de base de données dans laquelle vos utilisateurs seront importés. Assurez-vous que la connexion choisie a été activée pour au moins une application.

Cliquez sur **Start Importing Users** pour lancer le processus d’importation.

Une fois l’opération terminée, le message **Completed** suivant s’affiche.

<Frame>![Dashboard Extensions Users Import Export Import Complete](/docs/images/cdy7uua7fh8z/7hyE3DWBJuFuAulAAnZlfN/23303644d4c706f4c0937bd363bf6699/import-complete.png)</Frame>

Une fois vos utilisateurs importés, vous pouvez les gérer individuellement dans [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).

<div id="export-users">
  ### Exporter des utilisateurs
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 utilise le format ndjson en raison de la taille importante des fichiers d’exportation. Avant de pouvoir importer des utilisateurs, vous devrez convertir le format **ndjson** en **json** à l’aide de la bibliothèque de votre choix (comme [jq](https://stedolan.github.io/jq/)). Lorsque vous exportez des utilisateurs qui seront importés plus tard, les noms de champs utilisateur doivent conserver leurs valeurs par défaut et ne pas être mappés à un nom de colonne.
</Callout>

Pour exporter vos utilisateurs Auth0 existants associés à des connexions de base de données, sélectionnez **Export**.

<Frame>![Page Export Users de Dashboard Extensions Users Import Export](/docs/images/cdy7uua7fh8z/5wUhbGMjVQKIE9vx2SBIA3/a952aee6cc6027234631334c7174c4a8/export-users.png)</Frame>

Sous **User Fields**, vous pouvez choisir quels attributs utilisateur ou quelles expressions doivent être inclus dans l’exportation. L’attribut utilisateur peut être une valeur statique comme `user.user_metadata.name`, ou une expression Javascript comme `user.user_metadata.name || user.name`. Les expressions seront évaluées au moment de l’exportation. La valeur du **nom de colonne** correspond à la façon dont elle sera représentée dans l’exportation.

Vous pouvez cliquer sur le bouton **Add Default Fields** pour sélectionner automatiquement les champs par défaut et remplir leurs noms de colonne (c’est aussi une bonne façon de visualiser comment les paramètres/expressions s’afficheront).

Vous pouvez supprimer les attributs/expressions superflus en cliquant sur l’icône de **corbeille** correspondante.

Sous **Settings**, vous pouvez :

* Configurer l’affichage de vos utilisateurs exportés en indiquant l’**attribut utilisateur** selon lequel les utilisateurs doivent être triés (ainsi que le choix d’un tri croissant ou décroissant)
* Choisir votre **format d’exportation**; vous pouvez choisir entre des fichiers JSON et CSV

  <Frame>![Format d’exportation des paramètres de Dashboard Extensions Users Import Export](/docs/images/cdy7uua7fh8z/1HVpo1IevSRJbFvwrEKkzf/761fa914f6c33cdee147e07b2c43f7c7/settings.png)</Frame>

Quand vous êtes prêt, cliquez sur **Export X Users** (où `X` est le nombre d’utilisateurs que vous exportez).

Vous pouvez télécharger le fichier contenant vos utilisateurs une fois l’exportation terminée; vous devrez peut-être utiliser un outil d’extraction tiers si vous utilisez Windows 10 ou une version antérieure.

<div id="update-users">
  ## Mettre à jour les utilisateurs
</div>

L’Auth0 <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> permet de mettre à jour en bloc le profil utilisateur dans `user_metadata` et `app_metadata` à l’aide de l’extension User Import/Export.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 ne prend pas actuellement en charge l’importation en bloc pour Passwordless. Utilisez l’[Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) pour mettre à jour les utilisateurs un à un.
</Callout>

<div id="export-users">
  ### Exporter des utilisateurs
</div>

En utilisant le [processus d’exportation ci-dessus](/docs/fr-CA/customize/extensions/user-import-export-extension#export-users), créez un fichier d’exportation complet comprenant tous les attributs de vos utilisateurs.

Choisissez le format JSON pour l’exportation.

Téléchargez le fichier contenant vos utilisateurs une fois l’exportation terminée.

<div id="update-your-configuration">
  ### Mettez à jour votre configuration
</div>

Faites une **copie** du fichier JSON que vous avez téléchargé à l’étape précédente.

<Warning>
  Ce processus peut entraîner une perte de données. Créez une copie de sauvegarde de votre base de données d’utilisateurs en cas de corruption des données.
</Warning>

Ouvrez la copie du fichier JSON dans un éditeur de texte. Votre schéma utilisateur devrait ressembler à celui ci-dessous :

```json lines expandable
[
   {
      "Id":"auth0|626737a9813c000069daf462",
      "Nickname":"testuser1",
      "Name":"testuser1@auth0.com",
      "email":"testuser1@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-04-26T00:07:05.098Z",
      "Updated At":"2022-07-06T14:54:19.214Z"
   },
   {
      "Id":"auth0|62d1a1853bdcf11ff5dca672",
      "Nickname":"testuser2",
      "Name":"testuser2@auth0.com",
      "email":"testuser2@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-07-15T17:19:01.531Z",
      "Updated At":"2022-07-15T17:19:01.532Z"
   },
   {
      "Id":"auth0|62d1a1a524f32a4185c9ce23",
      "Nickname":"testuser3",
      "Name":"testuser3@auth0.com",
      "email":"testuser3@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-07-15T17:19:33.484Z",
      "Updated At":"2022-07-15T17:19:33.484Z"
   }
]
```

Ne modifiez pas l’adresse courriel de l’utilisateur, car c’est la clé utilisée pour repérer les utilisateurs dans la base de données.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’ID doit comporter un `connection|id` unique. Si le même ID d’utilisateur existe dans plusieurs connexions, votre mise à jour échouera.
</Callout>

Pour ajouter un attribut, créez un nouveau champ dans chaque schéma utilisateur et attribuez-lui une valeur. Dans l’exemple ci-dessous, l’attribut &quot;Nom de famille&quot; a la valeur &quot;Smith&quot;, suivie d’une virgule.

Vous pouvez aussi ajouter l’attribut `email_verified` à plusieurs utilisateurs et définir sa valeur à `true`.

```json lines
{
   "Id":"auth0|626737a9813c000069daf462",
   "Nickname":"testuser1",
   "Family Name":"Smith",
   "Name":"testuser1@auth0.com",
   "email":"testuser1@auth0.com",
   "email_verified":true,
   "Connection":"Username-Password-Authentication",
   "Created At":"2022-04-26T00:07:05.098Z",
   "Updated At":"2022-07-06T14:54:19.214Z"
}
```

Enregistrez les modifications dans votre fichier d’utilisateurs.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Ce processus ne permet pas de bloquer plusieurs utilisateurs à la fois. Vous devez bloquer les utilisateurs individuellement dans Auth0 Dashboard ou à l’aide de la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).
</Callout>

<div id="import-updated-user-profile-data">
  ### Importer des données de profil utilisateur mises à jour
</div>

Le endpoint `users-import` vous permet de téléverser votre fichier de profil utilisateur mis à jour et de mettre à jour plusieurs utilisateurs en un seul job. Accédez au [endpoint Create import users job](https://auth0.com/docs/api/management/v2/#!/Jobs/post_users_imports).

Ajoutez votre fichier utilisateur au paramètre `users`, puis ajoutez l’ID de la connexion de base de données que vous souhaitez mettre à jour.

La valeur `upsert` doit être `true` pour mettre à jour les renseignements d’un utilisateur existant.
Pour appliquer vos mises à jour, sélectionnez **Try** ou utilisez Postman pour importer les utilisateurs.

Utilisez le `job_id` pour vérifier le statut au moyen du [endpoint](https://auth0.com/docs/api/management/v2/#!/Jobs/get_jobs_by_id) `jobs`.
