Utilisez l’extension <Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement aux autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-On">Authentification unique</Tooltip> (SSO) Dashboard pour gérer la connexion SSO de vos utilisateurs dans plusieurs applications d’entreprise. Pour en savoir plus, [consultez cette extension sur GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).

<div id="create-auth0-application">
  ## Créer une application Auth0
</div>

Avant d’[ajouter l’extension SSO Dashboard](/docs/fr-CA/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension), vous devez créer une application dans Auth0 :

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis sélectionnez **+Create Application**.
2. Entrez un nom descriptif pour votre application (par exemple, SSO Dashboard), sélectionnez **Single-Page Web Application** comme **Application Type**, puis sélectionnez **Create**.
3. Dans l’onglet **Settings**, définissez les **Allowed Callback URLs** en fonction de la région de votre tenant :

   1. Pour les **Admins** : `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/admins/login`
   2. Pour les **Users** : `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/login`

      | Région | Domaine régional |
      | ------ | ---------------- |
      | US-1   | us.webtask.io    |
      | US-3   | us.webtask.run   |
      | US-4   | us.webtask.run   |
      | US-5   | us.webtask.run   |
      | EU     | eu.webtask.io    |
      | EU-2   | eu.webtask.run   |
      | AU     | au.webtask.io    |
      | JP-1   | jp.webtask.run   |
      | UK-1   | uk.webtask.run   |
      | CA-1   | ca.webtask.run   |
4. Sélectionnez et copiez la valeur du **Client ID**.
5. Allez au bas de la page, puis sélectionnez **Advanced Settings**.
6. Sélectionnez l’onglet **OAuth**, puis collez la valeur du **Client ID** dans le champ **Allowed APPs / APIs**.
7. Définissez **JsonWebToken Signature Algorithm** sur **RS256**.
8. Sélectionnez **Save Changes**.

Par défaut, tous les types de connexion sont activés afin que les utilisateurs puissent se connecter au SSO Dashboard. Si vous souhaitez modifier ce comportement, accédez à l’onglet **Connections** de l’application.

<div id="install-sso-dashboard-extension">
  ## Installer l’extension SSO Dashboard
</div>

Ensuite, vous devrez [installer l’extension SSO Dashboard](/docs/fr-CA/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension) et [ajouter des applications](/docs/fr-CA/customize/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard) au tableau de bord SSO.
