---

Utilisez l’extension <Tooltip tip="Single Sign-On (SSO) : Service qui, après qu’un utilisateur se connecte à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip> Dashboard pour gérer la connexion SSO de vos utilisateurs à plusieurs applications d’entreprise.

<div id="prerequisites">
  ## Prérequis
</div>

* [Créer l’application SSO Dashboard](/docs/fr-CA/customize/extensions/single-sign-on-dashboard-extension/create-sso-dashboard-application).
* [Installer l’extension SSO Dashboard Extension](/docs/fr-CA/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension).

<div id="add-applications">
  ## Ajouter des applications
</div>

1. Accédez à [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions), puis sélectionnez votre nouvelle extension SSO Dashboard.
   Si vous êtes administrateur, vous pouvez aussi vous connecter au SSO Dashboard à l’aide de `https://{yourTenant}.<REGION>8.webtask.io/auth0-sso-dashboard/admins/login`.
2. Dans le coin supérieur droit, sélectionnez **Settings** dans le menu déroulant sous le nom de votre tenant.
3. Cliquez sur **Create App** pour ajouter une nouvelle application.

   Le formulaire **New Application** s’affiche.
4. Remplissez les champs suivants pour la nouvelle application :

   | Champ           | Description                                                                                                                                                                                                                                                                                                                       |
   | --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Type**        | Menu déroulant dans lequel vous sélectionnez SAML, OpenID-Connect ou WS-Federation selon le type d’application.                                                                                                                                                                                                                   |
   | **Application** | Menu déroulant dans lequel vous sélectionnez l’application que vous souhaitez ajouter.                                                                                                                                                                                                                                            |
   | **Name**        | Le nom est rempli automatiquement en fonction de l’application sélectionnée. Vous pouvez le modifier ou conserver le nom par défaut.                                                                                                                                                                                              |
   | **Logo**        | URL du logo que vous souhaitez utiliser comme icône pour l’application.                                                                                                                                                                                                                                                           |
   | **Callback**    | Une des **Allowed Callback URLs** dans vos [Application Settings](https://manage.auth0.com/#/applications) pour l’application.                                                                                                                                                                                                    |
   | **Connection**  | (*Facultatif*) Le type de connexion. Vous pouvez ajouter ou modifier vos types de connexion disponibles dans la [section Connections du Dashboard de gestion Auth0](https://manage.auth0.com/#/connections/database). Si aucune connexion n’est définie et que l’utilisateur n’est pas connecté, il verra l’**Auth0 Login Page**. |
   | **Custom URL**  | (*Facultatif*) Cochez cette case pour utiliser une URL personnalisée au lieu de l’URL Auth0. Si vous cochez la case, un champ s’affiche dans lequel vous pouvez entrer votre URL personnalisée.                                                                                                                                   |
   | **Enabled**     | Cochez cette case pour rendre l’application visible (publiée) pour vos utilisateurs.                                                                                                                                                                                                                                              |
5. Sélectionnez **Create**. Votre nouvelle application apparaîtra alors sur la page **Applications** du tableau de bord SSO, avec les autres applications qui ont été créées.
6. Cliquez sur une application pour tester la connexion.
