---

Dans l’Application exposée par l’extension Delegated Administration, deux vues sont proposées : **Users** et **Logs**. Dans la vue **Users**, vous pouvez afficher et modifier les utilisateurs associés à votre compte Auth0.

Par défaut, tous les utilisateurs sont affichés, mais vous pouvez filtrer la liste en configurant un Filter Hook.

Pour en savoir plus sur le Filter Hook, consultez [Delegation Administration: Filter Hook](/docs/fr-CA/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-filter-hook).

<div id="user-actions-in-delegated-administration-dashboard">
  ## Actions des utilisateurs dans le tableau de bord Delegated Administration
</div>

Pour limiter le nombre d’options offertes à une personne ayant accès au Dashboard fourni par l’extension Delegated Administration, configurez un Access Hook.

Pour en savoir plus sur les Access Hooks, consultez [Delegated Administration : Access Hook](/docs/fr-CA/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-access-hook).

L’option **Modifier le profil** est offerte uniquement si vous avez configuré des champs personnalisés.

Notez la nouvelle option **Réinitialiser le mot de passe** offerte par l’extension. Cette option enverra un courriel à l’utilisateur afin qu’il puisse choisir un nouveau mot de passe. Pour ce faire, sélectionnez un utilisateur, puis sélectionnez **Actions** &gt; **Réinitialiser le mot de passe**.

Un courriel sera alors envoyé à l’utilisateur, contenant un lien pour modifier son mot de passe.

Si votre profil indique que vous avez le rôle `Delegated Admin - Administrator`, la vue **Logs** vous permet de consulter la liste des authentifications effectuées par vos utilisateurs (cette vue est visible uniquement pour les utilisateurs ayant le rôle `Delegated Admin - Administrator`). Le contenu de cette vue correspond à un sous-ensemble des données affichées dans [Auth0 Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs). Le tableau de bord Logs affiche également des données sur les actions administratives effectuées dans le <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

<div id="create-users">
  ## Créer des utilisateurs
</div>

Vous pouvez créer un nouvel utilisateur en cliquant sur le bouton **+ Create User** dans la vue **Users**. Vous devez indiquer une adresse courriel et un mot de passe. Selon votre rôle, il se peut que vous ne puissiez pas définir le **départemen**t auquel le nouvel utilisateur appartient.

Par exemple, les utilisateurs ayant le rôle `Delegated Admin - Administrator` peuvent voir le champ **Department** et en sélectionner n’importe quelle valeur.

<Frame>![Extensions - Administration Dashboard - Create admin](/docs/images/cdy7uua7fh8z/4vrbU6nRuIkF30ZPM4yHnG/73fd2060fe9618c9f465025cb56deb02/create-user-admin.png)</Frame>

À l’inverse, Kelly, qui a le rôle `Delegated Admin - User` et appartient au département des finances, ne peut pas voir le champ **Department**. L’utilisateur qu’elle crée sera automatiquement attribué au département des finances.

<Frame>![Extensions - Delegated Administration Dashboard - Create user](/docs/images/cdy7uua7fh8z/2CSdiPbyiyPsURHH9X5VVZ/8416222627933bd35c044aa15c22dfae/create-user-kelly.png)</Frame>

<Card title="Expiration de session de Delegated Administration">
  Par défaut, le token expire après 10 heures. Toutefois, pour des raisons de sécurité, lorsque vous utilisez Delegated Administration, Auth0 n’enregistre pas de token dans les cookies ni dans `sessionStorage`. Vous devez ouvrir une nouvelle session chaque fois que la page est rechargée.
</Card>
