---

Dans l’Application exposée par l’extension Delegated Administration, deux vues sont offertes : **Users** et **Logs**. Dans la vue **Users**, vous pouvez afficher et modifier les utilisateurs associés à votre compte Auth0.

Par défaut, tous les utilisateurs sont affichés, mais vous pouvez filtrer la liste en configurant un Filter Hook.

Pour en savoir plus sur le Filter Hook, consultez [Delegated Administration : Filter Hook](/docs/fr-CA/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-filter-hook).

<div id="user-actions-in-delegated-administration-dashboard">
  ## Actions utilisateur dans le tableau de bord Delegated Administration
</div>

Pour limiter le nombre d’options offertes à une personne ayant accès au tableau de bord exposé par l’extension Delegated Administration, configurez un Hook d’accès.

Pour en savoir plus sur les Hooks d’accès, consultez [Delegated Administration: Access Hook](/docs/fr-CA/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-access-hook).

L’option **Modifier le profil** est offerte seulement si vous avez configuré des champs personnalisés.

Notez la nouvelle option **Réinitialiser le mot de passe**, accessible par l’intermédiaire de l’extension. Cette option enverra un courriel à l’utilisateur lui permettant de choisir un nouveau mot de passe. Pour ce faire, sélectionnez un utilisateur, puis sélectionnez **Actions** &gt; **Réinitialiser le mot de passe**.

Un courriel sera alors envoyé à l’utilisateur, contenant un lien pour modifier le mot de passe.

Si votre profil indique que vous avez le rôle `Delegated Admin - Administrator`, la vue **Logs** vous permet de voir une liste des authentifications effectuées par vos utilisateurs (cette vue est visible uniquement pour les utilisateurs ayant le rôle `Delegated Admin - Administrator`). Le contenu de cette vue est un sous-ensemble des données affichées dans [Auth0 tableau de bord &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs). Le tableau de bord Logs affiche également des données sur les actions administratives effectuées dans l’<Tooltip tip="Auth0 tableau de bord : produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 tableau de bord</Tooltip>.

<div id="create-users">
  ## Créer des utilisateurs
</div>

Vous pouvez créer un nouvel utilisateur en sélectionnant le bouton **+ Create User** dans la vue **Users**. Vous devez indiquer l’adresse courriel et le mot de passe. Selon votre rôle, il se peut que vous ne puissiez pas définir le **département** auquel le nouvel utilisateur appartient.

Par exemple, les utilisateurs ayant le rôle `Delegated Admin - Administrator` peuvent voir le champ **Département** et en sélectionner n’importe quelle valeur.

<Frame>![Extensions - Tableau de bord d’administration - Créer un administrateur](/docs/images/cdy7uua7fh8z/4vrbU6nRuIkF30ZPM4yHnG/73fd2060fe9618c9f465025cb56deb02/create-user-admin.png)</Frame>

En revanche, Kelly, qui a le rôle `Delegated Admin - User` et appartient au département des finances, ne peut pas voir le champ **Département**. L’utilisateur qu’elle crée sera automatiquement affecté au département des finances.

<Frame>![Extensions - Delegated Administration tableau de bord - Créer un utilisateur](/docs/images/cdy7uua7fh8z/2CSdiPbyiyPsURHH9X5VVZ/8416222627933bd35c044aa15c22dfae/create-user-kelly.png)</Frame>

<Card title="Délai d’expiration de la session Delegated Administration">
  Par défaut, le jeton expire après 10 heures. Toutefois, pour des raisons de sécurité, lorsque vous utilisez Delegated Administration, Auth0 n’enregistre pas de jeton dans des cookie ni dans `sessionStorage`. Vous devez ouvrir une nouvelle session chaque fois que la page est rechargée.
</Card>
