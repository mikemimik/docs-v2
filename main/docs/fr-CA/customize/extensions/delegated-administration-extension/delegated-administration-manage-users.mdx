Dans l’application exposée par l’extension Administration déléguée, deux vues sont disponibles : **Users** et **Logs**. Dans la vue **User**s, vous pouvez consulter les utilisateurs associés à votre compte Auth0 et les modifier.

Par défaut, tous les utilisateurs sont affichés, mais vous pouvez filtrer la liste en configurant un Hook de filtrage.

Pour plus d’informations sur le Hook de filtrage, consultez [Administration déléguée : Hook de filtrage](/docs/fr-CA/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-filter-hook).

<div id="user-actions-in-delegated-administration-dashboard">
  ## Actions utilisateur dans le tableau de bord de Administration déléguée
</div>

Pour limiter le nombre d’options offertes à une personne ayant accès au Dashboard exposé par l’extension Administration déléguée, configurez un Access Hook.

Pour en savoir plus sur les Access Hooks, consultez [Administration déléguée: Access Hook](/docs/fr-CA/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-access-hook).

L’option **Modifier le profil** n’est offerte que si vous avez configuré des champs personnalisés.

Notez la nouvelle option **Réinitialiser le mot de passe** offerte par l’extension. Cette option enverra un courriel à l’utilisateur afin qu’il puisse choisir un nouveau mot de passe. Pour ce faire, sélectionnez un utilisateur, puis sélectionnez **Actions** &gt; **Réinitialiser le mot de passe**.

Cela enverra à l’utilisateur un courriel contenant un lien pour modifier son mot de passe.

Si votre profil indique que vous avez le rôle `Delegated Admin - Administrator`, la vue **Logs** vous permet de voir une liste des authentifications effectuées par vos utilisateurs (cette vue n’est visible que pour les utilisateurs ayant le rôle `Delegated Admin - Administrator`). Le contenu de cette vue correspond à un sous-ensemble des données affichées dans [Auth0 Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs). Le tableau de bord Logs affiche également des données sur les actions administratives effectuées dans l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

<div id="create-users">
  ## Créer des utilisateurs
</div>

Vous pouvez créer un nouvel utilisateur en cliquant sur le bouton **+ Create User** dans la vue **Users**. Vous devez indiquer une adresse courriel et un mot de passe. Selon votre rôle, il se peut que vous ne puissiez pas définir le **départemen**t auquel le nouvel utilisateur appartient.

Par exemple, les utilisateurs ayant le rôle `Delegated Admin - Administrator` peuvent voir le champ **Département** et en sélectionner n’importe quelle valeur.

<Frame>![Extensions - Tableau de bord d’administration - Créer un administrateur](/docs/images/cdy7uua7fh8z/4vrbU6nRuIkF30ZPM4yHnG/73fd2060fe9618c9f465025cb56deb02/create-user-admin.png)</Frame>

À l’inverse, Kelly, qui a le rôle `Delegated Admin - User` et fait partie du département des finances, ne peut pas voir le champ **Département**. L’utilisateur qu’elle crée sera automatiquement affecté au département des finances.

<Frame>![Extensions - Administration déléguée Dashboard - Créer un utilisateur](/docs/images/cdy7uua7fh8z/2CSdiPbyiyPsURHH9X5VVZ/8416222627933bd35c044aa15c22dfae/create-user-kelly.png)</Frame>

<Card title="Délai d’expiration de la session de Administration déléguée">
  Par défaut, la durée d’expiration du token est de 10 heures. Toutefois, pour des raisons de sécurité, lorsque vous utilisez Administration déléguée, Auth0 n’enregistre pas de token dans des cookies ni dans `sessionStorage`. Vous devez ouvrir une nouvelle session chaque fois que la page est rechargée.
</Card>
