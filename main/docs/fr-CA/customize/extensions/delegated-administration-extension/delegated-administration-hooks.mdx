title: "Delegated Administration : Extension Hooks"
---

Si vous êtes un utilisateur à qui le rôle **Delegated Admin - Administrator** a été attribué, vous pouvez gérer les différents Extension Hooks et requêtes qui vous permettent de personnaliser le comportement de l’extension Delegated Administration.

Pour accéder à la section de configuration :

1. Connectez-vous au Delegated Administration Dashboard.
2. Cliquez sur votre nom dans le coin supérieur droit. Un menu déroulant s’affiche; cliquez sur l’option **Configurer**.

La page **Configuration** vers laquelle vous êtes redirigé vous permet de gérer vos Hooks et vos requêtes.

<div id="extension-hooks-context">
  ## Contexte des Extension Hooks
</div>

L’objet de contexte (**ctx**) fournit quelques fonctions utilitaires ainsi que des informations sur la requête en cours. Les méthodes et propriétés suivantes sont disponibles dans chaque Extension Hook :

* Logging
* Caching
* Custom Data
* Payload and Request
* Remote Calls

<div id="logging">
  ### Journalisation
</div>

Pour ajouter un message aux journaux en temps réel d’Actions (que vous pouvez consulter dans les Journaux en temps réel d’Actions), appelez la méthode **log** :

```javascript lines
function(ctx, callback) {
  ctx.log('Logging action:', ctx.payload.action);
  return callback();
}
```

<div id="caching">
  ### Mise en cache
</div>

Pour mettre quelque chose en cache (comme une longue liste de départements), vous pouvez le stocker dans l’objet **global** du contexte. Cet objet restera disponible jusqu’à ce que le conteneur Actions Real-Time Logs soit recyclé.

```javascript lines
function(ctx, callback) {
  ctx.global.departments = [ 'IT', 'HR', 'Finance' ];
  return callback();
}
```

<div id="custom-data">
  ### Données personnalisées
</div>

Vous pouvez stocker des données personnalisées dans l’extension. Ce champ peut contenir jusqu’à 400 ko de données.

```javascript lines
var data = {
departments: [ 'IT', 'HR', 'Finance' ]
};

ctx.write(data)
.then(function() {
    // ...
})
.catch(function(err) {
    // ...
});
```

Pour lire les données :

```js lines
ctx.read()
.then(function(data) {
    // ...
})
.catch(function(err) {
    // ...
});
```

<div id="payload-and-request">
  ### Payload et requête
</div>

Chaque Extension Hook expose le payload ou la requête en cours avec des renseignements précis. La requête contiendra toujours des renseignements sur l’utilisateur connecté au Dashboard Users :

```javascript lines
function(ctx, callback) {
  ctx.log('Current User:', ctx.request.user);
  return callback();
}
```

<div id="remote-calls">
  ### Appels à distance
</div>

Si vous souhaitez appeler un service externe (comme une API) pour valider des données ou charger des affiliations, vous pouvez le faire à l’aide du module `request`.

```javascript lines
function(ctx, callback) {
var request = require('request');
    request('http://api.mycompany.com/departments', function (error, response, body) {
        if (error) {
        return callback(error);
        }

        // ...
    });
}
```

<div id="hook-contract">
  ### Contrat du Hook
</div>

* `ctx` : L’objet de contexte

  * `payload` : L’objet payload

    * `action` : L’action en cours d’exécution (par exemple, `delete:user`)
    * `user` : L’utilisateur sur lequel l’action est exécutée
* `callback(error)` : Le callback auquel vous pouvez transmettre une erreur si l’accès est refusé

<div id="sample-use">
  ### Exemple d’utilisation
</div>

Kelly gère le département des finances et ne devrait pouvoir accéder qu’aux utilisateurs de son département.

```javascript lines
function(ctx, callback) {
  if (ctx.payload.action === 'delete:user') {
    return callback(new Error('You are not allowed to delete users.'));
  }

  // Obtenir le département à partir des métadonnées de l'utilisateur actuel.
  var department = ctx.request.user.app_metadata && ctx.request.user.app_metadata.department;
  if (!department || !department.length) {
    return callback(new Error('The current user is not part of any department.'));
  }

  // Le département TI peut accéder à tous les utilisateurs.
  if (department === 'IT') {
    return callback();
  }

  ctx.log('Verifying access:', ctx.payload.user.app_metadata.department, department);

  if (!ctx.payload.user.app_metadata.department || ctx.payload.user.app_metadata.department !== department) {
    return callback(new Error('You can only access users within your own department.'));
  }

  return callback();
}
```

Si ce hook n’est pas configuré, tous les utilisateurs seront accessibles.

Noms d’action pris en charge :

* `read:user`
* `delete:user`
* `reset:password`
* `change:password`
* `change:username`
* `change:email`
* `read:devices`
* `read:logs`
* `remove:multifactor-provider`
* `block:user`
* `unblock:user`
* `send:verification-email`
