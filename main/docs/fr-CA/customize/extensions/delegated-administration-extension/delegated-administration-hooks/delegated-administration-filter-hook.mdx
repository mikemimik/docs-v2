---
description: Découvrez comment utiliser le Filter Hook avec la Delegated Administration Extension.
title: 'Delegated Administration : Filter Hook'
---

Par défaut, les utilisateurs ayant le rôle **Delegated Admin - User** peuvent voir tous les utilisateurs associés au compte Auth0. Toutefois, vous pouvez utiliser le **Filter Hook** pour filtrer les données qui leur sont affichées.

<div id="hook-contract">
  ## Contrat du Hook
</div>

* **ctx** : objet de contexte.
* **callback(error, query)** : fonction de rappel dans laquelle vous pouvez renvoyer une erreur ou la requête lucene utilisée pour filtrer les utilisateurs. L’extension enverra cette requête au [**point de terminaison Get Users**](https://auth0.com/docs/api/management/v2#!/Users/get_users) de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

Pour en savoir plus sur la requête lucene, consultez [User Search Query Syntax](/docs/fr-CA/manage-users/user-search/user-search-query-syntax).

<div id="sample-use">
  ### Exemple d’utilisation
</div>

Si Kelly gère le service des finances, elle ne devrait voir que les utilisateurs qui font aussi partie du service des finances. Nous filtrerons les utilisateurs en fonction du service de l’utilisateur actuel (qui, dans ce cas-ci, correspond respectivement au service des finances et à Kelly).

```javascript lines
function(ctx, callback) {
  // Obtenir le département à partir des métadonnées de l'utilisateur actuel.
  var department = ctx.request.user.app_metadata && ctx.request.user.app_metadata.department;
  if (!department || !department.length) {
    return callback(new Error('The current user is not part of any department.'));
  }

  // Le département TI peut voir tous les utilisateurs.
  if (department === 'IT') {
    return callback();
  }

  // Retourner la requête Lucene.
  return callback(null, 'app_metadata.department:"' + department + '"');
}
```

<div id="search-engine-override">
  ### Remplacement du moteur de recherche
</div>

Vous pouvez remplacer le moteur de recherche par défaut en indiquant votre choix dans la réponse.

```javascript lines
// Retourner la requête Lucene.
return callback(null, { query: 'app_metadata.department:"' + department + '"', searchEngine: 'v2' });
```

<div id="notes">
  ## Remarques
</div>

N’utilisez pas de guillemets simples, de guillemets doubles ni d’autres caractères spéciaux (comme **+** ou **-**) dans les termes que vous souhaitez utiliser pour le filtrage. Cela peut causer des problèmes avec la requête Lucene.

Si vous utilisez plusieurs opérateurs comme OR, NOT ou AND, mettez ces paramètres de recherche entre parenthèses [`()`] afin de bien délimiter la portée de chaque opérateur.

Si vous ne configurez pas ce Hook, la recherche renvoie **tous les utilisateurs**.
