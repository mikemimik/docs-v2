description: Découvrez comment utiliser le Write Hook avec la Delegated Administration Extension.
title: 'Delegated Administration : Write Hook'
---

Le Write Hook, qui s’exécute chaque fois que vous créez ou mettez à jour un utilisateur, vous permet notamment de :

* modifier le mot de passe de l’utilisateur;
* modifier l’adresse courriel de l’utilisateur;
* mettre à jour le profil de l’utilisateur.

Vous pouvez aussi utiliser le Write Hook pour définir automatiquement des valeurs par défaut pour les utilisateurs nouvellement créés. Par exemple, vous pouvez vouloir attribuer automatiquement à un utilisateur le même groupe, le même département ou le même fournisseur que ceux auxquels vous êtes assigné.

<Warning>
  Auth0 prend uniquement en charge la création d’utilisateurs avec Database Connections.
</Warning>

<div id="hook-contract">
  ## Contrat du Hook
</div>

* **ctx**: Objet de contexte.

  * **request.originalUser**: Valeurs actuelles de l’utilisateur, où le **payload** contient le nouvel ensemble de champs. Disponible uniquement lorsque la méthode est **update**.
  * **payload**: Objet payload

    * **memberships**: Tableau des appartenances sélectionnées dans l’UI lors de la création de l’utilisateur.
    * **email**: Adresse courriel de l’utilisateur.
    * **password**: Mot de passe de l’utilisateur.
    * **connection**: Nom de la connexion à la base de données.
    * **app&#95;metadata**: Données incluses si un champ personnalisé en cours de modification est enregistré dans `app_metadata`.
    * **user&#95;metadata**: Données incluses si un champ personnalisé en cours de modification est enregistré dans `user_metadata`.
  * **userFields**: Tableau des champs utilisateur (s’il est précisé dans la requête de paramètres)
  * **method**: Soit **create**, soit **update**, selon qu’il s’agit d’un appel de création ou de mise à jour.
* **callback(error, user)**: Callback auquel vous pouvez renvoyer une erreur et l’objet utilisateur qui doit être envoyé à la <Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

Pour en savoir plus sur **userFields**, consultez [Delegated Administration: Settings Query Hook](/docs/fr-CA/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-settings-query-hook).

<div id="sample-use">
  ## Exemple d’utilisation
</div>

Kelly gère le département des finances. Lorsqu’elle crée des utilisateurs, ceux-ci doivent être ajoutés comme membres du département des finances.

```javascript lines expandable
function(ctx, callback) {
  var newProfile = {
    email: ctx.payload.email,
    password: ctx.payload.password,
    connection: ctx.payload.connection,
    user_metadata: ctx.payload.user_metadata,
    app_metadata: {
      department: ctx.payload.memberships && ctx.payload.memberships[0],
      ...ctx.payload.app_metadata
    }
  };

  if (!ctx.payload.memberships || ctx.payload.memberships.length === 0) {
    return callback(new Error('The user must be created within a department.'));
  }

  // Obtenir le département à partir des métadonnées de l'utilisateur actuel.
  var currentDepartment = ctx.request.user.app_metadata && ctx.request.user.app_metadata.department;
  if (!currentDepartment || !currentDepartment.length) {
    return callback(new Error('The current user is not part of any department.'));
  }

  // Si vous n'êtes pas dans le département TI, vous ne pouvez créer des utilisateurs que dans votre propre département.
  // Le département TI peut créer des utilisateurs dans tous les départements.
  if (currentDepartment !== 'IT' && ctx.payload.memberships[0] !== currentDepartment) {
    return callback(new Error('You can only create users within your own department.'));
  }

  if (ctx.method === 'update') {
    // En cas de mise à jour, ne définir que les champs à envoyer
    Object.keys(newProfile).forEach(function(key) {
      if (newProfile[key] === ctx.request.originalUser[key]) delete newProfile[key];
    });
  }

  // Voici le payload qui sera envoyé à l'API v2. Vous avez un contrôle total sur la façon dont l'utilisateur est créé dans l'API v2.
  return callback(null, newProfile);
}
```
