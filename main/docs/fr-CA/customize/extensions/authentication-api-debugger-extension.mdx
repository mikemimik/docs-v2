---

L’extension Authentication API Debugger vous permet de tester différents points de terminaison de l’Auth0 Authentication API.

<div id="install-the-extension">
  ## Installer l’extension
</div>

Pour installer cette extension :

1. Accédez à [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions), puis sélectionnez **Auth0 Authentication API Debugger**. La fenêtre **Install Extension** s’ouvre.
2. Sélectionnez **Install**.

<div id="authorize-the-extension">
  ## Autoriser l’extension
</div>

Comme l’extension communiquera avec la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> en votre nom pour récupérer des renseignements sur les Applications que vous avez configurées dans votre <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vous devrez autoriser cet accès.

Pour autoriser l’extension :

1. Si vous suivez ce guide, vous devriez déjà vous trouver dans la vue [Extensions installées](https://manage.auth0.com/#/extensions) de [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions). Sélectionnez **Auth0 Authentication API Debugger** pour lancer l’extension. Une boîte de dialogue de consentement s’affichera pour demander l’accès à votre compte.
2. Donnez votre consentement pour permettre à l’extension d’accéder à votre compte. Une fois le consentement accordé, l’extension se lance.

<div id="configure-the-extension">
  ## Configurer l’extension
</div>

Après avoir installé l’extension, vous devez effectuer la configuration de base. Pour configurer l’extension :

1. Si vous suivez ce guide, vous avez déjà lancé l’extension. Sélectionnez la vue **Configuration**.

   <Frame>![Dashboard - Extensions - Authentication API Debugger - Configuration](/docs/images/cdy7uua7fh8z/5riI24ArqUAKRXhZfNWIQ5/ab7d6825288752132d14e54db1e563d0/configuration.png)</Frame>
2. Définissez les paramètres de configuration suivants, puis sélectionnez **Save** :

* **Domain** : Le domaine associé à votre tenant. Ce champ est en lecture seule et s’affiche à titre informatif.
* **Application** : l’Application pour laquelle vous voulez lancer des flux d’authentification. Vous pouvez gérer vos Applications dans [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
* **Callback URL** : l’URL de rappel de cette extension. Vous devez aussi ajouter cette URL aux **Allowed Callback URLs** dans les paramètres de l’Application sélectionnée.
* **State** : Données d’état facultatives pouvant être envoyées avec le flux d’authentification.
* **Connection** : Le nom de la connexion que vous voulez utiliser pour vous connecter. Vous pouvez utiliser ce paramètre pour contourner l’écran de connexion principal et accéder directement à l’écran de connexion de l’<Tooltip tip="Identity Provider (IdP) : Service qui stocke et gère des identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+Provider">Identity Provider</Tooltip> pertinent.

<div id="execute-the-flows">
  ## Exécuter les flux
</div>

Après avoir configuré l’extension, vous pouvez exécuter n’importe quel flux en sélectionnant la vue **OAuth2/OIDC**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour certains flux, vous devrez peut-être indiquer des paramètres supplémentaires en plus de ceux de la configuration de base. Ces paramètres dépendent du flux exécuté; nous vous suggérons donc de consulter la [documentation de l’Auth0 Authentication API](/docs/fr-CA/api) pour en savoir plus sur les paramètres applicables à chaque flux.
</Callout>

<Frame>![Dashboard - Extensions - Authentication API Debugger - Paramètres OAuth OIDC](/docs/images/cdy7uua7fh8z/72iYPxZU9OiHkKkHtgPTjI/fc166f3ae74e595c8d916276e78edb99/oauth-oidc-settings.png)</Frame>

Une fois un flux exécuté, vous verrez un écran affichant les informations renvoyées par Auth0. Les informations exactes renvoyées dépendront du flux exécuté. Voici un exemple de résultats après l’exécution d’un flux **OAuth2/OIDC Login** normal :

<Frame>![Dashboard - Extensions - Authentication API Debugger - Flux exécuté](/docs/images/cdy7uua7fh8z/13TRS8SQUd4OsfLYRWMoe6/3478778905ad253160022db1f1f6f31a/flow-executed.png)</Frame>

Pour exécuter un autre flux, sélectionnez la vue **Login** pour revenir à l’écran **Configuration**.
