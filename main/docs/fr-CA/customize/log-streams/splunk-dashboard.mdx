Vous pouvez utiliser **Auth0 App for Splunk** pour visualiser les données de votre tenant Auth0 dans un tableau de bord Splunk. Ce tableau de bord vous permet de surveiller l’état du trafic de connexion d’un tenant. L’application vous permet d’utiliser les agrégations recommandées par Auth0 ou de vous en servir comme point de départ pour créer vos propres visualisations personnalisées.

<div id="install-the-auth0-app-for-splunk">
  ## Installer Auth0 App for Splunk
</div>

1. Connectez-vous à [Splunk](https://www.splunk.com/).
2. Accédez à la page de l’application [Auth0 App for Splunk](https://splunkbase.splunk.com/app/5193).
3. Sélectionnez le bouton vert **Install**.
4. Entrez votre nom d’utilisateur et votre mot de passe, acceptez les conditions générales, puis sélectionnez **Login and Install**.
5. Dans le widget **Complete** qui s’affiche, sélectionnez le bouton **Open the App**.
   L’Auth0 tableau de bord s’affiche avec les filtres par défaut appliqués.

<div id="dashboard-filters">
  ## Filtres du tableau de bord
</div>

Les filtres suivants vous permettent d’explorer plus en détail les caractéristiques de votre trafic. Vous pouvez saisir `*` pour effectuer une recherche parmi toutes les valeurs de ce champ.

| Filtre            | Description                                                                                                                                                                                                                            |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Plage de temps    | Un champ temporel Splunk qui vous permet de choisir la période pendant laquelle afficher les événements.                                                                                                                               |
| Index             | L’`index` Splunk dans lequel vous voulez effectuer la recherche. Lors de la création de votre `Http Event Collector Token`, si vous avez précisé un index particulier, vous pouvez sélectionner cette valeur dans la liste déroulante. |
| Source HTTP       | Le nom `source` dans Splunk.                                                                                                                                                                                                           |
| IP                | L’adresse IP dont vous voulez examiner le trafic. Cela correspond à `data.ip` dans l’événement de journal.                                                                                                                             |
| Client            | Le client dont vous voulez examiner le trafic. Cela correspond à `data.client_name` dans l’événement de journal.                                                                                                                       |
| Pays              | Le pays dont vous voulez examiner le trafic. Il s’agit d’un champ obtenu à l’aide de la commande de recherche `iplocation data.ip` dans la requête. Cela ajoute un champ `Country` à chaque événement de journal.                      |
| Nom d’utilisateur | L’adresse courriel dont vous voulez examiner le trafic de connexion. Cela correspond à `data.user_name` dans l’événement de journal.                                                                                                   |

<div id="customize-dashboard">
  ## Personnaliser le tableau de bord
</div>

Vous pouvez personnaliser votre tableau de bord de sécurité Splunk Auth0 pour y ajouter des widgets de données personnalisées.

1. Accédez à votre tableau de bord de sécurité Auth0, puis sélectionnez **modifier** dans le coin supérieur droit.
2. Dans le panneau Modifier le tableau de bord, sélectionnez **ajouter un panneau**, puis choisissez un type de contenu, par exemple un graphique linéaire, un événement ou un graphique en aires.
3. Saisissez la plage de temps, le titre du contenu et la requête de recherche pour la visualisation des données.
4. Sélectionnez **Ajouter au tableau de bord** pour ajouter votre nouveau widget de données.
