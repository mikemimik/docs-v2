<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Mandrill](https://www.mandrill.com/signup/).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes provenant de certaines [adresses IP d’Auth0](/fr-CA/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-mandrill">
  ## Configurer Mandrill
</div>

1. Connectez-vous à [Mandrill](https://www.mandrill.com/signup/).
2. Accédez à la page [Settings](https://mandrillapp.com/settings) de Mandrill, puis cliquez sur **Add API key**. Copiez la valeur de cette clé.
3. Accédez à [Auth0 Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider).
4. Activez la bascule **Use my own email provider**.
5. Cliquez sur le logo **Mandrill**.
6. Entrez une adresse courriel **From** et saisissez la **clé API** Mandrill que vous avez copiée précédemment.
7. Cliquez sur **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer le courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez aucun courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour vérifier les codes d’erreur.

Une fois la configuration effectuée, la page [Activité sortante](https://mandrillapp.com/activity) de Mandrill affiche des renseignements sur la livraison de tous les courriels envoyés à vos utilisateurs.
