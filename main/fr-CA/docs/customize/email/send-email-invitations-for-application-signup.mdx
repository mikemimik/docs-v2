Une façon de gérer les inscriptions à une application Auth0 consiste à utiliser des invitations d’utilisateurs.

Dans un processus typique d’invitation d’utilisateur, un administrateur de l’application crée un nouveau compte utilisateur, puis envoie à l’utilisateur un courriel d’invitation contenant un lien d’activation. Lorsque l’utilisateur suit ce lien, il définit un mot de passe pour le compte, puis peut se connecter.

Un système d’invitation d’utilisateurs peut être utile dans des cas d’utilisation comme la restriction des inscriptions ou la création de comptes en lot.

<div id="implementing-user-invitations-for-application-sign-up-on-auth0">
  ## Mise en œuvre des invitations d’utilisateur pour l’inscription à une application sur Auth0
</div>

Dans Auth0, vous pouvez mettre en œuvre des invitations d’utilisateur à l’aide du processus de changement de mot de passe.

* En adaptant le style de la page de réinitialisation du mot de passe de Universal Login ou de Classic Login pour prendre aussi en charge une expérience d’invitation d’utilisateur, vous pouvez utiliser un lien de changement de mot de passe comme invitation à un nouveau compte utilisateur.

* En personnalisant de la même façon le modèle de courriel de changement de mot de passe, vous pouvez utiliser le processus de changement de mot de passe d’Auth0 pour générer le lien de changement de mot de passe et envoyer le courriel d’invitation à l’aide du [fournisseur SMTP configuré](/fr-CA/docs/customize/email/smtp-email-providers) de votre tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous disposez déjà d’une solution de courriel externe et ne souhaitez pas configurer de fournisseur SMTP dans votre tenant, vous pouvez [utiliser la Management API pour générer le password change ticket](https://auth0.com/docs/api/management/v2/tickets/post-password-change) et envoyer vous-même par courriel à l’utilisateur le lien du ticket, au lieu d’utiliser le modèle de courriel et le workflow d’Auth0.
</Callout>

Cette mise en œuvre repose sur une propriété de métadonnées utilisateur (nommée `user.app_metadata.needsInvitation` dans ces instructions) qui indique que l’utilisateur a une invitation en attente : lorsqu’elle est à `true`, l’utilisateur reçoit l’expérience d’invitation d’utilisateur; sinon, il reçoit l’expérience habituelle de réinitialisation du mot de passe.

<div id="1-customize-the-password-change-email-template">
  ### 1. Personnalisez le modèle de courriel de changement de mot de passe
</div>

D’abord, [personnalisez le modèle de courriel de changement de mot de passe (lien)](/fr-CA/docs/customize/email/email-templates/customize-email-templates) pour prendre en charge à la fois les processus d’invitation d’utilisateur et de changement de mot de passe.

[Utilisez Liquid](/fr-CA/docs/customize/email/email-templates/supported-liquid-syntax) pour vérifier si la propriété de métadonnées utilisateur d’une invitation en attente est définie et envoyer le message approprié. Par exemple :

```liquid lines
{% if user.app_metadata.needsInvitation %}
    // Contenu du courriel d'invitation de l'utilisateur ici
{% else %}
    // Contenu du courriel de changement de mot de passe ici
{% endif %}
```

De plus, ajoutez un paramètre de requête au lien de réinitialisation du mot de passe dans le corps du message afin d’indiquer quelle expérience l’utilisateur doit voir sur la page de réinitialisation du mot de passe. Par exemple, vous pouvez ajouter `type=invite` ou `type=reset`.

<div id="2-style-the-password-reset-page">
  ### 2. Personnaliser la page de réinitialisation du mot de passe
</div>

Selon le paramètre de requête que vous avez défini dans le template, personnalisez la page de réinitialisation du mot de passe pour prendre en charge à la fois les workflows d’invitation d’utilisateur et de changement de mot de passe, selon le paramètre de requête.

Pour obtenir les instructions avec Universal Login, consultez [Personnaliser Universal Login](/fr-CA/docs/customize/login-pages/universal-login). Pour obtenir les instructions avec Classic Login, consultez [Personnaliser la page de réinitialisation du mot de passe](/fr-CA/docs/customize/login-pages/classic-login/customize-password-reset-page).

<div id="3-create-a-post-login-action-to-unset-the-invitation-property">
  ### 3. Créer une Action post-login pour désactiver la propriété d’invitation
</div>

Une fois que l’utilisateur a défini son mot de passe et qu’il se connecte pour la première fois, il n’a plus besoin d’invitation et devrait voir l’expérience de changement de mot de passe par la suite.

Pour ce faire, [créez une Action post-login](/fr-CA/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) qui définit `user.app_metadata.needsInvitation` à `false`:

```javascript
exports.onExecutePostLogin = async (event, api) => {
    if (event.user.app_metadata.needsInvitation) {
        api.user.setAppMetadata("needsInvitation", false);
    }
};
```

<div id="4-send-user-invitations">
  ### 4. Envoyer des invitations à des utilisateurs
</div>

Pour envoyer une invitation à un utilisateur, [créez un nouvel utilisateur de base de données Auth0 à partir du Dashboard](/fr-CA/docs/authenticate/database-connections) ou au moyen du [endpoint Create a User](https://auth0.com/docs/api/management/v2/#!/Users/post_users) dans la Management API, puis :

* Définissez le paramètre `user.email_verified` sur `false`.

* Définissez `user.app_metadata.needsInvitation` sur `true`.

Ensuite, appelez l’[endpoint Change password](https://auth0.com/docs/api/authentication/change-password/change-password) de l’Authentication API pour envoyer le courriel d’invitation à l’utilisateur.

  ## En savoir plus
</div>

Pour choisir où l’utilisateur est redirigé après avoir défini son mot de passe :

* Pour Universal Login, [personnalisez la route de connexion du processus de réinitialisation du mot de passe](/fr-CA/docs/authenticate/login/auth0-universal-login/configure-default-login-routes#complete-password-reset-flow).

* Pour Classic Login, définissez le champ **Redirect To** dans le modèle de courriel.

Pour en savoir plus sur la configuration des courriels, vous pouvez consulter :

* [Personnaliser les modèles de courriel](/fr-CA/docs/customize/email/email-templates)

* [Configurer des fournisseurs de messagerie SMTP externes](/fr-CA/docs/customize/email/smtp-email-providers)
