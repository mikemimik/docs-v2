<Warning>
  La date de fin de vie (EOL) des Rules et des Hooks est fixée au **18 novembre 2026**, et ils ne sont plus offerts aux nouveaux tenants créés à compter du **16 octobre 2023**. Les tenants existants qui ont des Hooks actifs conserveront l’accès au produit Hooks jusqu’à la fin de vie.

  Nous vous recommandons fortement d’utiliser Actions pour étendre Auth0. Avec Actions, vous avez accès à des informations de type enrichies, à de la documentation intégrée et à des packages `npm` publics, et vous pouvez connecter des intégrations externes qui améliorent votre expérience globale d’extensibilité. Pour en savoir plus sur ce qu’offrent les Actions, consultez [Understand How Auth0 Actions Work](/docs/fr-CA/customize/actions/actions-overview).

  Pour vous aider dans votre migration, nous proposons des guides pour [migrer des Rules vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-rules-to-actions) et [migrer des Hooks vers Actions](/docs/fr-CA/customize/actions/migrate/migrate-from-hooks-to-actions). Nous avons aussi une page dédiée, [Move to Actions](https://auth0.com/extensibility/movetoactions), qui présente des comparaisons de fonctionnalités, [une démo d’Actions](https://www.youtube.com/watch?v=UesFSY1klrI) et d’autres ressources pour vous accompagner dans votre migration.

  Pour en savoir plus sur la dépréciation des Rules et des Hooks, consultez notre billet de blogue : [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

L’objet global `configuration` est disponible dans les Rules pour stocker des valeurs couramment utilisées, comme des URL. Les renseignements sensibles, comme des identifiants ou des clés API, devraient être stockés dans l’objet `configuration` et non dans votre code Rules.

<div id="configure-values">
  ## Configurer les valeurs
</div>

Vous pouvez définir des valeurs de configuration dans vos [paramètres de Rules](https://manage.auth0.com/#/rules/) du Dashboard.

Pour modifier la valeur d’une clé de configuration, supprimez le paramètre existant et remplacez-le par la nouvelle valeur. Vous devez avoir créé au moins une Rule pour voir la section de configuration; sinon, la démo de Rules s’affiche à la place.

<div id="use-the-configuration-object">
  ## Utiliser l’objet `configuration`
</div>

Vous pouvez accéder à toutes les valeurs de configuration que vous définissez dans votre code Rules à l’aide de leur clé, au moyen de l’objet `configuration`.

```javascript lines
var MY_API_KEY = configuration.MY_API_KEY;
```

L’exemple suivant montre une Rule qui envoie un message Slack lorsqu’un nouvel utilisateur s’inscrit. Le [Slack Webhook](https://api.slack.com/incoming-webhooks) est une valeur de `configuration` définie avec la clé `SLACK_HOOK_URL`.

```javascript lines
function (user, context, callback) {
  // court-circuiter si l'utilisateur s'est déjà inscrit ou utilise un Refresh Token
  if (context.stats.loginsCount > 1 || context.protocol === 'oauth2-refresh-token') {
    return callback(null, user, context);
  }

  // obtenez l'URL du hook Slack ici : https://slack.com/services/10525858050
  const SLACK_HOOK = configuration.SLACK_HOOK_URL;

  const slack = require('slack-notify')(SLACK_HOOK);
  const message = 'New User: ' + (user.name || user.email) + ' (' + user.email + ')';
  const channel = '#some_channel';

  slack.success({
    text: message,
    channel: channel
  });

  // ne pas attendre la fin de l'appel à l'API Slack, retourner immédiatement (la requête continuera dans le sandbox)`
  callback(null, user, context);
}
```
