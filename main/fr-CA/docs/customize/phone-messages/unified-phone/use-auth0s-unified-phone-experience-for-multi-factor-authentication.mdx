<Card title="Avant de commencer">
  Pour migrer vers Unified Phone Experience, vous devez :

  * Avoir activé et configuré dans votre tenant les notifications SMS, voix ou SMS+voix pour l’authentification multifacteur (MFA), ou
  * Avoir activé et configuré Passwordless SMS dans votre tenant

  Si vous n’avez configuré ni MFA ni Passwordless, consultez [Configurer Unified Phone Experience d’Auth0](/fr-CA/docs/customize/phone-messages/unified-phone/configure-unified-phone) pour partir du début.

  Si vous avez à la fois des intégrations MFA et Passwordless avec des fournisseurs différents, vous devez utiliser un [Custom Phone Provider](/fr-CA/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider).

  Si vous utilisez Phone comme ID, vous utilisez déjà automatiquement Unified Phone Experience d’Auth0 et aucune action supplémentaire n’est nécessaire.
</Card>

Unified Phone Experience centralise toutes les configurations de votre fournisseur de téléphonie à un seul endroit dans votre tenant. Vous n’avez pas besoin de configurer un fournisseur de téléphonie plusieurs fois pour différents facteurs d’authentification par téléphone. Pour en savoir plus sur les facteurs téléphoniques pour <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du username et du Password, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=MFA">MFA</Tooltip>, consultez [Notifications SMS et vocales pour MFA](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa).

Si vous avez déjà configuré un fournisseur de téléphonie pour les [notifications SMS et vocales pour l’authentification multifacteur (MFA)](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa), vous pouvez migrer cette configuration vers Unified Phone Experience et l’appliquer au niveau du tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 n’indique pas à l’utilisateur final si un message téléphonique a bien été envoyé avec le fournisseur au niveau du tenant. Pour résoudre les problèmes, consultez les [logs du tenant](/fr-CA/docs/deploy-monitor/logs).
</Callout>

<div id="update-your-phone-provider">
  ## Mettez à jour votre fournisseur de téléphonie
</div>

Pour utiliser Unified Phone Experience, vous devez :

* Vérifier les paramètres actuels
* Configurer votre fournisseur :

  * Configurer Twilio
  * Configurer un fournisseur de téléphonie personnalisé
* Mettre à jour vos facteurs téléphoniques et vos méthodes de livraison
* Mettre à jour les endpoints de l’<Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>
* Mettre à jour les déclencheurs d’Auth0 Actions

<div id="verify-current-settings">
  ### Vérifier les paramètres actuels
</div>

1. Pour vérifier vos paramètres MFA actuels, accédez à [**Auth0 Dashboard &gt; Security &gt; Multi-factor Auth**](https://manage.auth0.com/#/security/mfa).
2. Sélectionnez **Phone Message**.
3. Les nouveaux tenants Auth0 peuvent utiliser l’Unified Phone Experience, et une bannière vous indique de configurer un fournisseur de téléphonie au niveau du tenant :

   <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/mfa-phone-message-upe.png)</Frame>

* Les tenants existants datant d’avant mars 2025 peuvent configurer les notifications **SMS and Voice** pour la **Multi-Factor Authentication (MFA)** en plus de l’Unified Phone Experience.

  <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/mfa-phone-message-no-upe.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recommande d’utiliser l’Unified Phone Experience. Si vous devez utiliser l’expérience [SMS and Voice notifications for Multi-Factor Authentication (MFA)](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) pour configurer votre fournisseur de téléphonie, et que la bascule **Use  Tenant-Level Messaging Provider** est absente, communiquez avec [Auth0 Support](/fr-CA/docs/troubleshoot/customer-support).
</Callout>

4. Pour vérifier les paramètres de votre tenant, accédez à [**Dashboard &gt; Tenant Settings**](https://manage.auth0.com/#/tenant/advanced).
5. Sélectionnez **Advanced**.
6. Repérez **Unified Phone Experience**.
   <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/tenant-settings-upe.png)</Frame>
7. Activez la bascule au besoin.

<div id="configure-twilio-as-a-tenant-level-provider">
  ### Configurer Twilio comme fournisseur de téléphonie au niveau du tenant
</div>

Si vous utilisez Twilio comme fournisseur de téléphonie pour l’authentification multifacteur, vous devez ajouter le SID Twilio et l’AuthToken à Unified Phone Experience. Vous pouvez copier le SID Twilio à partir de votre configuration existante, mais vous devrez obtenir la valeur de l’AuthToken auprès de Twilio.

1. Accédez à [**Image de marque &gt; Fournisseur de téléphonie**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Sélectionnez et configurez votre fournisseur :

   <Frame>![Branding > Phone Provider > Twilio](/docs/images/cdy7uua7fh8z/6TESTIdmhUL0LEitACadIK/fcdc34e18a6b66b47370f6d754c085b5/2025-02-21_12-49-53.png)</Frame>

   1. Choisissez Twilio comme fournisseur de téléphonie.
   2. Choisissez vos modes de livraison.
   3. Saisissez les paramètres Twilio.

      <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
        Les facteurs MFA souhaités doivent correspondre à un sous-ensemble des modes de livraison que vous configurez au niveau du tenant.
      </Callout>
3. **Enregistrer** vos modifications.
4. Sélectionnez **Envoyer un message de test** pour tester le fournisseur au niveau du tenant avant de l’utiliser dans un environnement de production.
5. Activez un fournisseur au niveau du tenant.

   1. Pour la MFA, accédez à [**Sécurité &gt; Authentification multifacteur &gt; Message téléphonique**.](https://manage.auth0.com/#/security/mfa)
   2. Sélectionnez **Utiliser le fournisseur de messagerie au niveau du tenant**.

      <Warning>
        Si aucune bascule n’existe dans votre tenant, vous utilisez déjà Unified Phone Experience.
      </Warning>
6. **Enregistrer** vos modifications.

<div id="configure-a-custom-provider-with-auth0-actions">
  ### Configurer un fournisseur personnalisé avec Auth0 Actions
</div>

Unified Phone Experience utilise le déclencheur Actions `custom-phone-provider` pour les notifications téléphoniques, plutôt que le déclencheur Actions `send-phone-message` pour les notifications téléphoniques personnalisées.

Vous ne pouvez pas migrer directement votre déclencheur Actions `send-phone-message` vers le déclencheur Actions `custom-phone-provider` pour les raisons suivantes :

* Il n’existe pas de correspondance directe 1:1 entre les types de messages et les types de notifications.
* La migration des secrets et des dépendances associés au déclencheur Actions `send-phone-message` pourrait entraîner des risques de sécurité ou nuire à l’expérience du tenant.
* Les Actions peuvent prendre en charge plusieurs déclencheurs, ce qui peut compliquer la migration. Par exemple, les Actions peuvent prendre en charge à la fois les déclencheurs `send-phone-message` et `post-login`.
* Les déclencheurs Actions `send-phone-message` et `custom-phone-provider` prennent en charge différents types d’événements :

  * `send-phone-message` prend en charge les types de message `SMS and Voice`. Pour en savoir plus, consultez [Action triggers: send-phone-message Event Object](/fr-CA/docs/customize/actions/explore-triggers/mfa-notifications-trigger/send-phone-message-event-object).
  * `custom-phone-provider` prend en charge les types de notification `otp_verify`, `otp_enroll`, `blocked_account`, `change_password` et `password_breach`. Pour en savoir plus, consultez [Actions triggers: custom-phone-provider Event Object](/fr-CA/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/actions-triggers-custom-phone-provider-event-object).
* Vous ne pouvez définir qu’une seule Action par fournisseur de téléphonie personnalisé. Si vous avez déjà défini une Action pour un fournisseur de téléphonie personnalisé, vous **devez** la désactiver avant de l’associer à une nouvelle Action.

Par conséquent, vous devez convertir votre `send-phone-message` existant en nouveau `custom-phone-provider` à l’aide des nouvelles variables de contexte :

Consultez l’exemple du fournisseur de téléphonie personnalisé :

```javascript lines expandable
const { fetch } = require('undici');
/**
* Handler à exécuter lors de l'envoi d'une notification téléphonique
* @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
* @param {CustomPhoneProviderAPI} api - Méthodes et utilitaires pour modifier le comportement de l'envoi d'une notification téléphonique.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {

  if (event.notification.message_type.startsWith('otp')) {
    const body = {
      from: event.notification.from,
      action: event.notification.message_type === 'otp_verify' ? 'second-factor-authenticator' : 'enrollment',
      language: event.notification.locale,
      recipient: event.notification.recipient,
      message_type: event.notification.delivery_method,
      text: event.notification.as_text,
      code: event.notification.code,
    };

    const response = await fetch(event.secrets['MY_ENDPOINT'], {
      method: 'POST',
      headers: {
        authorization: `Bearer ${event.secrets['MY_SECRET']}`,
      },
      body: JSON.stringify(body),
    });

    console.log(response.status); // 200
    console.log(await response.text()); // corps de la réponse de la requête!
  }

  return;
};
```

<div id="configure-mfa-custom-phone-provider">
  #### Configurer le `custom-phone-provider` de MFA
</div>

Pour continuer et configurer votre déclencheur MFA `custom-phone-provider` :

1. Accédez à [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Sélectionnez votre fournisseur :

   1. Choisissez **Custom**.
   2. Sous **Provider Configuration**, ajoutez le code traduit de votre Action `send-phone-message` dans l’éditeur de code intégré.
3. **Save** vos modifications.
4. Sélectionnez **Send Test Message** pour tester le fournisseur au niveau du tenant avant de l’utiliser dans un environnement de **Production**.
5. Revenez à [**Security &gt; Multi-factor Auth &gt; Phone Message**](https://manage.auth0.com/#/security/mfa) pour activer **Use Tenant-level Messaging Provider**.
6. **Save** vos modifications.
