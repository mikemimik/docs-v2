<Warning>
  La date de fin de vie (EOL) de Rules et Hooks est fixée au **18 novembre 2026**, et ils ne sont plus offerts aux nouveaux tenants créés à compter du **16 octobre 2023**. Les tenants existants ayant des Hooks actifs conserveront l’accès au produit Hooks jusqu’à la fin de vie.

  Nous vous recommandons fortement d’utiliser Actions pour étendre Auth0. Avec Actions, vous avez accès à des informations de typage détaillées, à de la documentation intégrée et à des packages `npm` publics, et vous pouvez connecter des intégrations externes qui enrichissent votre expérience globale d’extensibilité. Pour en savoir plus sur ce qu’offrent Actions, consultez [Comprendre le fonctionnement d’Auth0 Actions](/fr-CA/docs/customize/actions/actions-overview).

  Pour vous aider dans votre migration, nous proposons des guides qui vous aideront à [migrer de Rules vers Actions](/fr-CA/docs/customize/actions/migrate/migrate-from-rules-to-actions) et à [migrer de Hooks vers Actions](/fr-CA/docs/customize/actions/migrate/migrate-from-hooks-to-actions). Nous avons aussi une page dédiée, [Passez à Actions](https://auth0.com/extensibility/movetoactions), qui présente des comparaisons de fonctionnalités, [une démo d’Actions](https://www.youtube.com/watch?v=UesFSY1klrI) et d’autres ressources pour vous accompagner dans votre parcours de migration.

  Pour en savoir plus sur la dépréciation de Rules et Hooks, consultez notre billet de blogue : [Préparer la fin de vie de Rules et Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Comme nous prévoyons retirer les fonctionnalités Rules et Hooks en 2026, vous ne devriez créer de nouvelles Rules ou de nouveaux Hooks que dans votre environnement de développement, et uniquement pour tester la migration vers Actions.

  Pour savoir comment migrer vos Rules vers Actions, consultez [Migrer de Rules vers Actions](/fr-CA/docs/customize/actions/migrate/migrate-from-rules-to-actions). Pour savoir comment migrer vos Hooks vers Actions, consultez [Migrer de Hooks vers Actions](/fr-CA/docs/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Vous pouvez mettre à jour les Hooks à l’aide du Dashboard ou de <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>.

Vous pouvez aussi importer et exporter des Hooks à l’aide de l’outil [Deploy Command-Line Interface (CLI)](/fr-CA/docs/deploy-monitor/deploy-cli-tool).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Au besoin, vous pouvez ajouter des secrets (comme des clés Twilio ou des chaînes de connexion à une base de données) aux Hooks.
</Callout>

<div id="rename-hooks-using-the-dashboard">
  ## Renommer des hooks dans le Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/) et cliquez sur l’icône d’engrenage à côté du hook que vous souhaitez renommer.
2. Cliquez sur **Rename**.
3. Entrez un nouveau nom pour le Hook, puis cliquez sur **Rename**.

   <Frame>![Auth0 - Renommer des hooks](/docs/images/cdy7uua7fh8z/2Xp8Z3uYGoGZrF5EGXLPLQ/6c68c77f97133b2066219a68540b1e73/Screen_Shot_2021-05-19_at_12.08.00_PM.png)</Frame>

<div id="update-hook-scripts-using-the-dashboard">
  ## Mettre à jour les scripts de hook à l’aide du Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Auth Pipeline &gt; Hooks](https://manage.auth0.com/#/) et cliquez sur l’icône en forme de crayon à côté du hook que vous souhaitez mettre à jour.

   <Frame>![Sélection d’un Hook dans Auth0 Hooks](/docs/images/cdy7uua7fh8z/2y2R713TkAvQWIMZPMd1zh/bf0a657261f2d090492660ea5962e0a3/dashboard-hooks-list.png)</Frame>
2. Mettez le hook à jour dans le Hook Editor, puis cliquez sur l’icône de disquette pour enregistrer.

   <Frame>![Éditeur de Hook Auth0](/docs/images/cdy7uua7fh8z/26u71QdXxZxg5zIH8SJQAg/e9bbd25b99d9b8fd44c281b6de83a5a3/dashboard-hooks-edit_menu.png)</Frame>

<div id="update-hooks-using-the-management-api">
  ## Mettre à jour les hooks à l’aide de la Management API
</div>

Effectuez un appel `PATCH` au [point de terminaison Update a Hook](https://auth0.com/docs/api/management/v2/#!/Hooks/patch_hooks_by_id). Assurez-vous de remplacer les valeurs d’espace réservé `HOOK_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre Hook et votre <Tooltip tip="Jeton d’accès : information d’authentification transmise dans l’en-tête Authorization, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, respectivement.

</AuthCodeGroup>

| Valeur                  | Description                                                                                                                                                                                                      |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `HOOK_ID`               | ID du Hook à mettre à jour.                                                                                                                                                                                      |
| `MGMT_API_ACCESS_TOKEN` | Jetons d’accès pour la Management API avec la [scope](/fr-CA/docs/glossary?term=scope) `update:hooks`. Pour en savoir plus, consultez [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |
| `HOOK_NAME`             | Nom du Hook à créer.                                                                                                                                                                                             |
| `HOOK_SCRIPT`           | Script contenant le code du Hook. Il doit correspondre à ce que vous saisiriez si vous créiez un nouveau hook dans le Dashboard.                                                                                 |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La propriété `enabled` indique si la règle est activée (`true`) ou désactivée (`false`).
</Callout>

<Card title="Gérer les limites de débit lors d’appels aux API depuis des Hooks">
  Si vous appelez des API Auth0 depuis le script d’un Hook, vous devrez gérer les limites de débit. Pour ce faire, vérifiez l’en-tête `X-RateLimit-Remaining` et réagissez de façon appropriée lorsque la valeur retournée approche de 0.

  Ajoutez également une logique pour gérer les cas où vous dépassez les limites de débit fournies et recevez le code d’état HTTP `429` (`Too Many Requests`). Dans ce cas, si une nouvelle tentative est nécessaire, il est préférable de prévoir un délai progressif afin d’éviter une boucle infinie de nouvelles tentatives.
</Card>
