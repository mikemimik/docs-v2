<Card title="Avant de commencer">
  1. Créez une [application machine à machine](/docs/fr-CA/get-started/auth0-overview/create-applications/machine-to-machine-apps) avec les scopes suivants activés :

  * `read:users`
  * `update:users`
  * `create:users`
  * `read:users_app_metadata`
  * `update:users_app_metadata`
  * `create:users_app_metadata`

  2. Ajoutez une [connexion Vault](/docs/fr-CA/customize/forms/vault) à l’aide des identifiants de l’application M2M.
</Card>

Forms pour Actions vous permet de créer des étapes supplémentaires dans vos flux d’inscription ou de connexion, et de les rendre obligatoires avant d’autoriser les utilisateurs à accéder à votre application.

<Frame>![Dashboard > Actions > Forms > Cas d’utilisation : formulaire d’inscription personnalisé](/docs/images/cdy7uua7fh8z/ZOQNi6SrEC9OREcBbdiJ3/d79f84fb61acd5473bdf932527e26d58/custom-signup-form.png)</Frame>

Les sections ci-dessous expliquent comment créer des étapes d’inscription supplémentaires sous forme de formulaires à l’aide de nœuds et de flux, ainsi que comment ajouter votre formulaire à une Action [Post Login](/docs/fr-CA/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

<div id="create-a-form-from-scratch">
  ## Créer un formulaire à partir de zéro
</div>

Suivez ces étapes pour créer un formulaire afin de recueillir des informations :

1. Ouvrez le Form editor en sélectionnant **Auth0 Dashboard &gt; Actions &gt; Forms.**
2. Sélectionnez **Create Form** &gt; **Start from scratch**.

Par défaut, un nouveau formulaire comprend un nœud Start, un nœud Step et un nœud Ending screen.

<Frame>![Dashboard > Forms > Cas d’utilisation > Formulaire](/docs/images/cdy7uua7fh8z/2TZ9jSjgEcARFpULI6hz4R/a4319a1bd482f2ee5cc23100fdb2f42c/New-form-Forms-for-action-use-case.png)</Frame>

<div id="configure-the-step-node">
  ### Configurer le nœud Step
</div>

Le nœud Step est l’interface graphique visible pour les utilisateurs. Ajoutez des [champs](/docs/fr-CA/customize/forms/nodes-and-components) au nœud Step pour recueillir le nom complet et le titre du poste de l’utilisateur en suivant les étapes ci-dessous :

1. Faites glisser un champ **Texte enrichi** du **menu Components** vers le nœud **Step**.

   * **Texte enrichi** : Entrez un message personnalisé.
   * Sélectionnez **Enregistrer**.
2. Faites glisser un champ **Texte** dans le nœud **Step**.

   * **ID** : Entrez `full_name`.
   * **Libellé** : Entrez `Full name`.
   * **Libellé** : Cochez la case.
   * **Obligatoire** : Cochez la case.
   * Sélectionnez **Enregistrer**.
3. Faites glisser un champ **Texte** dans le nœud **Step**.

   * **ID** : Entrez `job_title`.
   * **Libellé** : Entrez `Job title.`
   * **Libellé** : Cochez la case.
   * **Obligatoire** : Cochez la case.
   * Sélectionnez **Enregistrer**.
4. Sélectionnez **Publier** pour enregistrer.

<Frame>![Dashboard > Actions > Forms > Cas d’utilisation Custom Signup Steps Step node](/docs/images/cdy7uua7fh8z/6Kcn0ai9fsVsqHwZpTGByG/37cbf4ffee63a7010926b15f06dd20ac/custom-signup-step-node.png)</Frame>

<div id="configure-the-flow-node">
  ### Configurer le nœud **Flow**
</div>

Ajoutez un nœud **Flow** après le nœud **Step** pour mettre à jour le `user_metadata` et reprendre le flux d’authentification en suivant les étapes ci-dessous :

1. Sélectionnez **Flow** au bas du Form editor.
2. Supprimez le lien existant entre les nœuds **Step** et **Ending Screen**.
3. Sélectionnez le nouveau **Flow** &gt; **Click to add a flow** &gt; **Create a new flow.**

   * Saisissez `Update user_metadata` dans le champ **Name**.
   * Sélectionnez **Create**.
   * Sélectionnez **Save**.
4. Reliez le nœud **Flow** aux nœuds **Step** et **Ending Screen**, comme illustré ci-dessous.
5. Sélectionnez **Publish** pour enregistrer.

<Frame>![Dashboard > Actions > Forms > Cas d’utilisation Custom Signup Step Update User metadata flow](/docs/images/cdy7uua7fh8z/4nTdYdA2Hu5vK65smkdvJg/db3294ebc25a7c2636be96362a3ea56e/custom-signup-step-flow.png)</Frame>

Ajoutez une action Auth0 **Update user** au flux en suivant les étapes ci-dessous :

1. Sélectionnez **Flow** **Update user&#95;metadata** &gt; **Edit flow** pour ouvrir le Form editor dans un nouvel onglet.
2. Sous le StartAction, sélectionnez l’icône **+** pour ajouter une Action **Update user**. Remplissez les champs ci-dessous, puis sélectionnez **Save** pour continuer.

   * **Connection** : Dans la liste déroulante, sélectionnez la connexion Vault de votre M2M Application.

   * **User ID** : Saisissez `{{context.user.user_id}}`.

   * **Body** : Copiez-collez le code suivant pour mettre à jour le `user_metadata` avec les propriétés `full_name` et `job_title`.

     ```json lines
     {
       "user_metadata": {
         "full_name": "{{fields.full_name}}",
         "job_title": "{{fields.job_title}}"
       }
     }
     ```

   * Sélectionnez **Save**.
3. Sélectionnez **Publish** pour enregistrer.

<div id="retrieve-your-form-render-code">
  ## Récupérez le code de rendu de votre formulaire
</div>

Récupérez le code de rendu de votre formulaire pour afficher le formulaire à l’aide d’une action Post Login personnalisée en suivant ces étapes :

* Dans le Form editor, sélectionnez **`<> Render`**.
* Sélectionnez **Copy**.

<Frame>![Dashboard > Actions > Forms > Form > Embed](/docs/images/cdy7uua7fh8z/2EIUzc539sq9lwoTKbjoJv/496a864c0fc44a782c9cadca119dbfb0/embed.png)</Frame>

<div id="create-a-post-login-action">
  ## Créer une action Post Login
</div>

Créez une action post-login pour afficher votre formulaire en suivant les étapes ci-dessous :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Flows &gt; Login.**
2. Sélectionnez l’icône **+**, puis **Build from scratch** :

   * **Name** : Entrez `Render Additional Signup Form`.
   * **Trigger** : Sélectionnez `Login / Post Login`.
   * **Runtime** : Sélectionnez la version recommandée.
3. Sélectionnez **Create**.

Pour configurer l’action post-login :

1. Supprimez le code existant dans l’éditeur Code.
2. Collez le code de rendu du formulaire dans l’éditeur Code.
3. Modifiez le code pour définir la logique conditionnelle qui affichera le formulaire.

   ```javascript lines
   /**
   * @param {Event} event - Détails sur l’utilisateur et le contexte dans lequel il se connecte.
   * @param {PostLoginAPI} api - Interface dont les méthodes peuvent être utilisées pour modifier le comportement de la connexion.
   */
   exports.onExecutePostLogin = async (event, api) => {
     const FORM_ID = 'REPLACE_WITH_YOUR_FORM_ID';

     if (
       !event.user.user_metadata.full_name &&
       !event.user.user_metadata.job_title
     ) {
       api.prompt.render(FORM_ID);
     }
   }

   exports.onContinuePostLogin = async (event, api) => { }
   ```

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     L’exemple de code ci-dessus affichera le formulaire si les propriétés de métadonnées utilisateur `full_name` et `job_title` sont absentes.
   </Callout>
4. Sélectionnez **Deploy.**
5. Glissez-déposez l’action **Render Additional Signup Form** dans le flow **Login**.

   <Frame>![Dashboard > Forms > Cas d’utilisation Custom Signup Steps Login Flow](/docs/images/cdy7uua7fh8z/1WBOicIOcH98hKj6Lq9HF2/4951e30a0cc08c7c5daa0116eaa17e11/Screenshot_2024-03-01_at_15.48.00.png)</Frame>
6. Sélectionnez **Apply**.

<div id="test-implementation">
  ### Tester la mise en œuvre
</div>

Testez la mise en œuvre en suivant ces étapes :

* Inscrivez un nouvel utilisateur à une application.
* L’Action post-login du flow Login affichera le formulaire et demandera les renseignements.
* Sélectionnez **<Tooltip tip="Auth0 Dashboard: principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> &gt; User Management &gt; Users,** repérez le nouvel utilisateur et vérifiez que ses attributs `full_name` et `job_title` contiennent les renseignements saisis à l’étape d’inscription supplémentaire.
