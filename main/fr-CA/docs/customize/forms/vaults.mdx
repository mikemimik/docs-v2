Le vault vous permet de stocker de façon sécurisée des secrets ou des paramètres communs (par exemple : des clés API, <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+tokens">jetons d’accès</Tooltip>, <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip>) pour authentifier des requêtes auprès de services externes ou d’intégrations en tant que connexions Vault.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Certaines connexions exigent que l’utilisateur fournisse des identifiants (par exemple, avec une application Auth0 machine à machine (M2M), vous générez un jeton d’accès avec une durée d’expiration et un jeton d’actualisation). La connexion stocke les deux jetons et génère un nouveau jeton d’accès avant son expiration.
  * Certains services externes exigent l’autorisation de scopes pour générer des identifiants d’API. Si vous modifiez les scopes, mettez à jour la connexion à ces services pour éviter les problèmes de connexion.
  * Lorsqu’une connexion est ajoutée, elle est prête à être utilisée dans vos flows. Toute mise à jour d’une connexion existante est automatiquement propagée.
</Callout>

<div id="create-and-modify-vault-connections">
  ## Créer et modifier des connexions Vault
</div>

Créez et modifiez des connexions Vault à partir de la page Vault ou de l’éditeur de flux lorsqu’une action nécessite des identifiants.

<div id="create-vault-connections-from-the-vault-page">
  ### Créer des connexions Vault à partir de la page Vault
</div>

Ajoutez une connexion Vault à partir de la page Vault en suivant ces étapes :

1. Ouvrez le Form editor en sélectionnant **Auth0 Dashboard &gt; Actions &gt; Forms.**
2. Dans le coin supérieur gauche, sélectionnez le menu du tenant, puis **Vault** pour ouvrir la page Vault.

   <Frame>![Dashboard > Forms > Vault](/docs/images/cdy7uua7fh8z/1sPNPKO6nuEDCuiYGRWSur/d85dc616170eaf2d7d5f321750863ccf/vault-connection.png)</Frame>
3. Sélectionnez **Add Vault Connection**.
4. Sélectionnez l’action souhaitée, puis **Continue**.
5. Ajoutez un **Connection name** pour la connexion Vault, puis sélectionnez **Continue**.
6. Saisissez les identifiants.
   Si vous n’avez pas les identifiants requis, vous pouvez sélectionner **Create an empty connection** et les ajouter plus tard. Les connexions vides affichent une icône orange indiquant l’absence d’identifiants.
7. Sélectionnez **Add Vault Connection**.

<div id="create-vault-connections-from-the-flow-editor">
  ### Créer des connexions Vault à partir de l’éditeur de flux
</div>

Ajoutez une connexion Vault à partir d’une action de Flow, par exemple Twilio, en suivant ces étapes :

1. Ouvrez l’éditeur de flux en sélectionnant **Auth0 Dashboard &gt; Actions &gt; Forms &gt; Flows**
2. Sélectionnez un Flow et ajoutez une action **Send SMS with Twilio**.
3. Dans le paramètre **Vault connection** de l’action, sélectionnez l’icône **+**.

   <Frame>![Dashboard > Forms > Vault ](/docs/images/cdy7uua7fh8z/1AsaznhRKxm5LZxPlcbfUw/10e5a8af700aab909f5b03fcd266da7e/twilio-vault-connection.png)</Frame>
4. Saisissez un **Connection name** Vault, puis sélectionnez **Continue**.
5. Entrez les identifiants.
   Si vous n’avez pas les identifiants requis, vous pouvez sélectionner **Create an empty connection** et ajouter les identifiants plus tard. Les connexions vides affichent une icône orange pour indiquer qu’il manque des identifiants.

   <Frame>![Dashboard > Forms > Vault > paramètres de connexion Twilio](/docs/images/cdy7uua7fh8z/yobXanutPRCO3FWek8cOk/bc29e39fb8593cff35e2738234bfe6fe/twilio-vault-settings.png)</Frame>
6. Sélectionnez **Add Vault Connection.**
