---
description: Découvrez comment afficher des Forms à l’aide d’Actions.
title: Afficher des Forms à l’aide d’Actions
---

Une fois que vous avez créé une Form, vous pouvez l’afficher avec [Actions](/fr-CA/docs/customize/actions) au moyen de la méthode `api.prompt.render()` :

```javascript lines
// Exemple utilisant le déclencheur post-login

exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id');
}

exports.onContinuePostLogin = async (event, api) => {
  // Ajoutez votre logique après avoir rempli la Form
}
```

Remplacez form&#95;id par l’ID du Form. Vous pouvez repérer l’ID dans l’URL du Form, par exemple : `ap_pUMG...`, ou le sélectionner dans l’onglet **Render** du Form editor.
Dans le [Actions Code editor](/fr-CA/docs/customize/actions/write-your-first-action#create-an-action), vous pouvez définir la logique opérationnelle pour déterminer quand et comment afficher le Form.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {

  // Afficher la Form uniquement si l'utilisateur n'a pas de métadonnées company_name
  if (!event.user.user_metadata.company_name) {
    api.prompt.render(':form_id');
  }
}

exports.onContinuePostLogin = async (event, api) => {
  // Ajoutez votre logique après avoir rempli la Form
}
```

Pour en savoir plus sur ces objets, consultez :

* [Objet Event](/fr-CA/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object) : Découvrez les objets Event ainsi que leurs propriétés.
* [Objet API](/fr-CA/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object) : Découvrez les objets API ainsi que leurs méthodes.

<div id="populate-values-for-existing-fields-and-hidden-fields-client-side">
  ## Renseigner les valeurs des champs existants et des champs masqués (côté client)
</div>

Vous pouvez renseigner les valeurs des champs existants et des champs masqués à l’aide de la propriété `fields` comme deuxième argument de votre méthode `render`.

Dans l’exemple ci-dessous, la valeur `Jane` renseigne le champ `first_name`.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id', {
    fields: {
      first_name: 'Jane',
    }
  });
}

exports.onContinuePostLogin = async (event, api) => {
  // Ajoutez votre logique après avoir rempli la Form
}
```

<Warning>
  N’insérez pas de renseignements sensibles, car les valeurs renseignées sont exposées dans le navigateur (côté client).
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Assurez-vous que le champ `ID` mentionné dans le deuxième argument est accessible côté client dans vos [nœuds et composants](/fr-CA/docs/customize/forms/nodes-and-components); sinon, la valeur ne sera pas renseignée.
</Callout>

<div id="inject-custom-data-with-shared-variables-server-side">
  ## Injecter des données personnalisées avec des variables partagées (côté serveur)
</div>

Vous pouvez injecter des variables côté serveur à l’aide de la propriété `vars` comme deuxième argument de votre méthode `render`. Cela permet d’injecter des renseignements sensibles sans les exposer au côté client.

Dans l’exemple ci-dessous, la valeur `123456789` remplit la variable `external_user_id`.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id', {
    vars: {
      external_user_id: '123456789',
    }
  });
}

exports.onContinuePostLogin = async (event, api) => {
  // Ajoutez votre logique après avoir rempli la Form
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les variables partagées `{{vars.*}}` ne sont soumises à aucune restriction quant au type de données.

  Les variables partagées `{{vars.*}}` sont toujours masquées automatiquement.

  Contrairement au remplissage des variables `{{fields.*}}`, la propriété `{{vars.*}}` n’a pas besoin d’exister dans votre Form avant que vous y injectiez des valeurs.

  Vous pouvez utiliser la propriété `{{vars.*}}` pour faire référence aux variables partagées dans vos Forms et les Flows qui y sont liés. Pour en savoir plus, consultez [Variables](/fr-CA/docs/customize/forms/variables).
</Callout>

<div id="fields-and-shared-variables-data-in-actions">
  ## Données des champs et des variables partagées dans les Actions
</div>

Les données recueillies dans vos Forms pour les champs et les variables partagées sont automatiquement accessibles dans `event.prompt`, dans la fonction de reprise de l’action en cours :

* La propriété `id`, avec l’ID du prompt que vous affichez.
* L’objet `fields`, qui contient toutes les données de vos champs et de vos champs masqués.
* L’objet `vars`, qui contient toutes les données de vos variables partagées.

<Accordion title="Exemple d’objet event.prompt">
  ```json lines
  {
    "id": "ap_fuVuFiiQWN3mTEujWTy966",
    "fields": {
      "first_name": "Jane",
      "company_name": "Okta"
    },
    "vars": {
      "external_crm_uuid": "f8f32e6f-2329-49bd-bf21-fa8b0bea2652",
      "api_hostname": "api.example.com"
    }
  }
  ```
</Accordion>

Dans l’exemple ci-dessous, `api.user.setUserMetadata` renseigne l’attribut `company_name` de `user_metadata` à partir de la propriété `event.prompt.fields.company_name` recueillie dans votre Form.

```javascript lines
exports.onExecutePostLogin = async (event, api) => {
  api.prompt.render(':form_id');
}

exports.onContinuePostLogin = async (event, api) => {
  api.user.setUserMetadata('company_name', event.prompt.fields.company_name);
}
```

<div id="restrictions-and-limitations">
  ## Restrictions et limites
</div>

* Vous ne pouvez pas [rediriger](/fr-CA/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/redirect-with-actions) un utilisateur et afficher une Form dans la même Action. Si vous devez utiliser les deux, envisagez d’utiliser des Actions différentes.
* Vous ne pouvez afficher qu’une seule Form par Action. Si vous devez afficher plus d’une Form, vous devez le faire dans des Actions différentes.
* La taille maximale de la propriété `fields` est de 24 KB.
* La méthode `api.prompt.render()` est offerte dans les déclencheurs suivants :

  * [post-login](/fr-CA/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-api-object)
  * [post-challenge](/fr-CA/docs/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-api-object)
