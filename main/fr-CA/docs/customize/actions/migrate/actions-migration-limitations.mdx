description: Limites propres à la migration lors du passage de Rules & Hooks vers Actions
title: Limites de la migration vers Actions
---

Pour tirer parti des fonctionnalités offertes par Auth0 Actions, vous devriez migrer votre code actuel à partir de Rules, de Hooks ou de versions antérieures d’Actions. Cependant, vous devez connaître les limites actuelles de la migration entre les anciennes fonctionnalités d’extensibilité et Actions.

<div id="general-differences">
  ## Différences générales
</div>

* Les Actions ne peuvent renvoyer que des codes d’erreur `access denied` et ne prennent pas en charge le renvoi d’un code d’erreur `unauthorized`.
* Les claims standards d’un `accessToken` ne peuvent pas être supprimés à l’aide d’une Action.

<div id="rule-specific-differences">
  ## Différences propres aux Rules
</div>

* L’objet event des Actions Post Login ne fournit actuellement pas d’équivalent à `context.connectionOptions`, qui est disponible dans les Rules.
* L’objet event des Actions Post Login ne fournit actuellement pas d’équivalent permettant d’accéder, pour les utilisateurs fédérés, à un token provenant d’un IdP en amont au moyen de l’objet `user.identities`, comme c’est le cas dans les Rules.
* Les claims standard d’un `idToken` ne peuvent pas être supprimés à l’aide d’une Action.
* Les Actions prennent uniquement en charge la modification de l’utilisateur principal parmi des comptes liés.
