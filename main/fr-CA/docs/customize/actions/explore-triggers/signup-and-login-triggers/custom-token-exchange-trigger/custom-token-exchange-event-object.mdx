<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="dictionary">
  Détails sur la transaction d&#39;échange personnalisé de jetons en cours.

  <Expandable title="propriétés de la transaction" defaultOpen>
    <ResponseField name="actor_token" type="string" post={["optional"]}>
      Le jeton de l’acteur fourni dans la requête d’échange de jetons.
    </ResponseField>

    <ResponseField name="actor_token_type" type="string" post={["optional"]}>
      Le type de jeton d’acteur fourni dans la requête d’échange de jetons.
    </ResponseField>

    <ResponseField name="actor_token_user" type="dictionary" post={["optional"]}>
      L&#39;utilisateur représenté par le jeton d&#39;acteur. Cette valeur ne sera présente que si le actor&#95;token&#95;type est urn:ietf:params:oauth:token-type:id&#95;token et que le jeton d&#39;acteur fourni dans la demande d&#39;échange de jeton est un ID token valide généré par Auth0.

      <Expandable title="propriétés de actor_token_user" defaultOpen>
        <ResponseField name="app_metadata" type="dictionary">
          Des champs personnalisés contenant des renseignements sur un utilisateur qui influencent son accès, comme le forfait d’assistance, les rôles de sécurité ou les groupes de contrôle d’accès.
        </ResponseField>

        <ResponseField name="created_at" type="string">
          Horodatage indiquant à quel moment le profil de l’utilisateur a été créé pour la première fois.
        </ResponseField>

        <ResponseField name="courriel" type="string" post={["optional"]}>
          (unique) Adresse courriel de l’utilisateur.
        </ResponseField>

        <ResponseField name="email_verified" type="boolean">
          Indique si l’utilisateur a vérifié son adresse courriel.
        </ResponseField>

        <ResponseField name="nom_de_famille" type="string" post={["optional"]}>
          Nom de famille de l’utilisateur.
        </ResponseField>

        <ResponseField name="prénom" type="string" post={["optional"]}>
          Prénom de l’utilisateur.
        </ResponseField>

        <ResponseField name="dernière_réinitialisation_du_mot_de_passe" type="string" post={["optional"]}>
          Horodatage de la dernière réinitialisation ou modification du mot de passe de l’utilisateur. Lors de la création de l’utilisateur, ce champ n’existe pas. Cette propriété est disponible uniquement pour les connexions de base de données.
        </ResponseField>

        <ResponseField name="nom" type="string" post={["optional"]}>
          Nom complet de l’utilisateur.
        </ResponseField>

        <ResponseField name="surnom" type="string" post={["optional"]}>
          Surnom de l’utilisateur.
        </ResponseField>

        <ResponseField name="phone_number" type="string" post={["optional"]}>
          Numéro de téléphone de l’utilisateur.
        </ResponseField>

        <ResponseField name="phone_verified" type="boolean" post={["optional"]}>
          Indique si l’utilisateur a vérifié son numéro de téléphone.
        </ResponseField>

        <ResponseField name="photo" type="string" post={["optional"]}>
          URL pointant vers la [photo de profil de l’utilisateur](https://auth0.com/docs/users/change-user-picture).
        </ResponseField>

        <ResponseField name="updated_at" type="string">
          Horodatage indiquant la date de la dernière mise à jour/modification du profil de l’utilisateur.
        </ResponseField>

        <ResponseField name="user_id" type="string">
          (unique) Identifiant unique de l’utilisateur.
        </ResponseField>

        <ResponseField name="user_metadata" type="dictionary">
          Champs personnalisés qui stockent des renseignements sur un utilisateur sans incidence sur ce à quoi il peut ou non accéder, comme son adresse professionnelle, son adresse personnelle ou ses préférences.
        </ResponseField>

        <ResponseField name="nom d’utilisateur" type="string" post={["optional"]}>
          (unique) Nom d’utilisateur.
        </ResponseField>

        <ResponseField name="enrolledFactors" type="array of objects" post={["optional"]}>
          Un tableau des facteurs d’authentification que l’utilisateur a déjà configurés.

          <Expandable title="Propriétés de enrolledFactors">
            <ResponseField name="type" type="string">
              Le type de facteur d’authentification, comme `push-notification`, `phone`, `email`, `otp`, `webauthn-roaming` et `webauthn-platform`.
            </ResponseField>

            <ResponseField name="options" type="dictionary" post={["optional"]}>
              Des options supplémentaires décrivant cette instance du facteur configuré.
            </ResponseField>
          </Expandable>
        </ResponseField>

        <ResponseField name="multifacteur" type="array of strings" post={["optional"]}>
          Liste des fournisseurs d’authentification multifacteur (MFA) auxquels l’utilisateur est inscrit. Ce tableau est mis à jour lorsque l’utilisateur s’inscrit à l’authentification multifacteur et lorsqu’un administrateur réinitialise les inscriptions MFA d’un utilisateur.
        </ResponseField>

        <ResponseField name="identités" type="array of objects">
          Contient les renseignements récupérés auprès du fournisseur d’identité avec lequel l’utilisateur s’est authentifié au départ. Les utilisateurs peuvent aussi lier leur profil à plusieurs fournisseurs d’identité; ces identités figureront alors également dans ce tableau. Le contenu de chaque objet de fournisseur d’identité varie selon le fournisseur.

          <Expandable title="propriétés de l’élément identities">
            <ResponseField name="connection" type="string" post={["optional"]}>
              Nom de la connexion Auth0 utilisée pour authentifier l’utilisateur.
            </ResponseField>

            <ResponseField name="isSocial" type="boolean" post={["optional"]}>
              Indique s’il s’agit d’une connexion sociale.
            </ResponseField>

            <ResponseField name="profileData" type="dictionary" post={["optional"]}>
              Renseignements sur l’utilisateur associés à la connexion. Lorsque des profils sont liés, il contient les renseignements utilisateur associés aux comptes secondaires.
            </ResponseField>

            <ResponseField name="provider" type="string" post={["optional"]}>
              Nom de l’entité qui authentifie l’utilisateur, comme Facebook, Google, SAML ou votre propre fournisseur.
            </ResponseField>

            <ResponseField name="user_id" type="string" post={["optional"]}>
              Identifiant unique de l’utilisateur pour cette connexion ou ce fournisseur.
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="autorisations_demandées" type="array of strings">
      Les scopes demandés (s’il y en a) dans la demande d’échange de jetons.
    </ResponseField>

    <ResponseField name="type_de_token_demandé" type="string">
      Le type de jeton qu’Auth0 doit générer. Par exemple : urn:ietf:params:oauth:token-type:access&#95;token.
    </ResponseField>

    <ResponseField name="subject_token" type="string">
      Le jeton du sujet fourni dans la requête d’échange de jeton.
    </ResponseField>

    <ResponseField name="subject_token_type" type="string">
      Le subject&#95;token&#95;type fourni dans la requête d’échange de jeton.
    </ResponseField>
  </Expandable>
</ResponseField>
