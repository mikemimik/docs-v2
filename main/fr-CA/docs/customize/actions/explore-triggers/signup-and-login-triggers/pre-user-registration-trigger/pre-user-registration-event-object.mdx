<div id="eventsecurity_context">
  ## `event.security_context`
</div>

<ResponseField name="event.security_context" type="object" post={["optional"]}>
  Un objet contenant des signatures d’empreinte numérique. Il n’est disponible que si le client utilise cloudflare. L’empreinte JA3/JA4 peut être nulle ou vide dans certains cas. Le cas le plus courant concerne les requêtes HTTP, puisque JA3 et JA4 sont calculées dans TLS. Elle peut aussi être vide si le Worker envoie des requêtes dans la même zone ou vers une zone qui n’est pas proxyfiée (ou vers un tiers).

  <Expandable title="propriétés de security_context" defaultOpen>
    <ResponseField name="ja3" type="string" post={["optional"]}>
      Signature d’empreinte JA3. Elle n’est disponible que si le client utilise une connexion TLS.
    </ResponseField>

    <ResponseField name="ja4" type="string" post={["optional"]}>
      Signature d’empreinte JA4. Elle n’est disponible que si le client utilise une connexion TLS.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="dictionary" post={["optional"]}>
  Détails sur la transaction en cours.

  <Expandable title="propriétés de la transaction" defaultOpen>
    <ResponseField name="acr_values" type="array of strings">
      Toute valeur `acr_values` fournie dans la requête d’authentification d’origine.
    </ResponseField>

    <ResponseField name="locale" type="string">
      La locale à utiliser pour cette transaction, déterminée en comparant les langues demandées par le navigateur aux paramètres de langue du tenant.
    </ResponseField>

    <ResponseField name="login_hint" type="string" post={["optional"]}>
      Indication destinée à l’Authorization Server au sujet de l’identifiant de connexion que l’end-user pourrait utiliser pour se connecter (au besoin).
    </ResponseField>

    <ResponseField name="prompt" type="array of strings" post={["optional"]}>
      Liste d’instructions indiquant si l’utilisateur peut être invité à effectuer une re-authentification et à donner son consentement.
    </ResponseField>

    <ResponseField name="protocol" type="string" post={["optional"]}>
      Valeurs possibles :

      * `oidc-basic-profile`
      * `oidc-ciba`
      * `oidc-ciba-web-link`
      * `oidc-implicit-profile`
      * `oauth2-device-code`
      * `oauth2-resource-owner`
      * `oauth2-resource-owner-jwt-bearer`
      * `oauth2-password`
      * `oauth2-webauthn`
      * `oauth2-access-token`
      * `oauth2-refresh-token`
      * `oauth2-token-exchange`
      * `oidc-hybrid-profile`
      * `samlp`
      * `wsfed`
      * `wstrust-usernamemixed`
    </ResponseField>

    <ResponseField name="redirect_uri" type="string" post={["optional"]}>
      L’URL vers laquelle Auth0 redirigera le navigateur une fois la transaction terminée.
    </ResponseField>

    <ResponseField name="requested_scopes" type="array of strings">
      Les scopes demandés (s’il y en a) au démarrage de ce flux d’authentification.
    </ResponseField>

    <ResponseField name="response_mode" type="string" post={["optional"]}>
      Indique à l’Authorization Server le mécanisme à utiliser pour renvoyer les paramètres depuis le authorization endpoint.

      Valeurs autorisées : `query`, `fragment`, `form_post`, `web_message`
    </ResponseField>

    <ResponseField name="response_type" type="array of strings" post={["optional"]}>
      Indique le type de credential qu’Auth0 renverra.
    </ResponseField>

    <ResponseField name="state" type="string" post={["optional"]}>
      Une chaîne alphanumérique opaque et arbitraire que votre application ajoute à la requête initiale et qu’Auth0 inclut lors de la redirection vers votre application.
    </ResponseField>

    <ResponseField name="ui_locales" type="array of strings">
      Les valeurs `ui_locales` fournies dans la requête d’authentification d’origine.
    </ResponseField>

    <ResponseField name="correlation_id" type="string" post={["optional"]}>
      Le Correlation ID peut être fourni dans la requête d’authentification initiale lorsque l’application redirige vers Universal Login. Vous pouvez utiliser cette valeur pour faire la correlation entre les logs et les requêtes de votre code Action et le parcours utilisateur.
    </ResponseField>
  </Expandable>
</ResponseField>
