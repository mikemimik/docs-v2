<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="object">
  Renseignements sur la transaction Credentials Exchange.

  <Expandable title="propriétés de la transaction" defaultOpen>
    <ResponseField name="correlation_id" type="string" post={["optional"]}>
      L’ID de corrélation peut être fourni dans la requête d’authentification initiale lorsque l’application redirige vers Universal Login. Vous pouvez utiliser cette valeur pour établir la corrélation entre les logs et les requêtes de votre code Action et le parcours de l’utilisateur.
    </ResponseField>

    <ResponseField name="requested_scopes" type="array of strings">
      Les scopes précisés, le cas échéant, lors de la demande du jeton d’accès.
    </ResponseField>

    <ResponseField name="target_scopes" type="array of strings" post={["optional"]}>
      &#91;Accès anticipé&#93; L’ensemble de scopes cibles actifs pour le jeton d’accès. Il est initialisé à partir des client grants, puis mis à jour immédiatement par les méthodes de portée cible de `api.transaction` dans les Actions en cours et suivantes. Une fois toutes les Actions terminées, ces scopes sont croisés avec le client grant. Les scopes qui ne figurent pas dans le grant sont omis silencieusement du jeton d’accès final.
    </ResponseField>
  </Expandable>
</ResponseField>
