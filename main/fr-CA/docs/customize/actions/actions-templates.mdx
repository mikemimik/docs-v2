title: Modèles pour les Actions
---

La galerie de modèles d’Actions propose divers modèles de départ pour créer une Action. Ces modèles fournissent des exemples de code couvrant divers déclencheurs et cas d’utilisation, comme <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-factor+Authentication">Authentification multifacteur</Tooltip>, le contrôle d’accès et l’enrichissement du profil.

Les modèles fournissent du code de départ que vous pouvez modifier et utiliser comme base dans un déclencheur. Une fois que vous avez sélectionné un modèle pour votre cas d’utilisation précis, vous pouvez ensuite personnaliser le code pour répondre aux besoins particuliers de votre application.

Pour obtenir un guide étape par étape sur l’utilisation des modèles d’Actions, consultez [Write Your First Action](/fr-CA/docs/customize/actions/write-your-first-action).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les modèles d’Actions sont aussi accessibles à l’adresse suivante : https://github.com/auth0/opensource-marketplace.
</Callout>

<div id="best-practices">
  ## Bonnes pratiques
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les modèles d’Actions sont conçus uniquement à des fins d’apprentissage.

  Ces modèles contiennent des extraits de code incomplets et ne doivent en aucun cas être utilisés dans un environnement de production. Il incombe aux développeurs de compléter l’Action Template de façon sécuritaire avant de le déployer dans un environnement de production.
</Callout>

* Assurez-vous que les modèles sont adaptés à votre environnement particulier. Comme le code est incomplet, vous devez personnaliser et compléter les exemples avant de les importer dans un environnement de production.
* Ne déployez pas un modèle sans l’avoir d’abord modifié et complété.
* Les modèles d’Actions contiennent souvent des balises dans les commentaires, qui ne doivent pas être modifiées ni supprimées.
* Utilisez toujours la version la plus récente de tout package `npm`.
* Par défaut, les modèles d’Actions ciblent les versions les plus récentes de Node.js. Après avoir créé un modèle, vous pouvez le modifier pour prendre en charge d’autres versions de Node.js au besoin.

Veuillez consulter les [directives de codage des Actions](/fr-CA/docs/customize/actions/action-coding-guidelines), car elles s’appliquent également ici.
