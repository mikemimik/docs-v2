sidebarTitle: Vue d’ensemble
title: Explorer les déclencheurs
---

Les déclencheurs sont des points du processus Auth0 où des Actions peuvent être ajoutées pour répondre à des besoins de personnalisation. Plusieurs Actions peuvent être placées dans un déclencheur et s’exécutent en séquence. À l’exception de l’Échange de jeton personnalisé, les déclencheurs peuvent être synchrones (bloquants) ou asynchrones (non bloquants). Chaque déclencheur possède des objets et des API qui lui sont propres. Ils peuvent varier selon votre connexion (comme une connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=passwordless">sans mot de passe</Tooltip>).

Pour explorer les objets de référence et les API propres aux Actions d’un déclencheur donné, sélectionnez le déclencheur ci-dessous et prenez connaissance de la définition d’Auth0 d’une [connexion sans mot de passe](/fr-CA/docs/authenticate/passwordless).

<div id="signup-and-login">
  #### Inscription et connexion
</div>

Lorsqu’un utilisateur est ajouté à une connexion de base de données ou sans mot de passe (Inscription), ou dans le cadre de la transaction d’authentification d’un utilisateur (Connexion).

| Déclencheur              | Description                                                                               |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se déclenche avant qu’un utilisateur soit créé.                                           |
| `post-user-registration` | Se déclenche de manière asynchrone après la création d’un utilisateur.                    |
| `post-login`             | Se déclenche après l’authentification d’un utilisateur, mais avant l’émission d’un jeton. |

<div id="mfa-notifications">
  #### Notifications MFA
</div>

Lorsque vous utilisez le SMS comme facteur pour <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip> (MFA) ou pour configurer un fournisseur personnalisé.

| déclencheur          | Description                                                                                                                              |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se déclenche lorsque vous utilisez un fournisseur personnalisé pour envoyer les messages liés au processus d’enrôlement et de challenge. |

<div id="password-reset">
  #### Réinitialisation du mot de passe
</div>

Lorsqu’une demande de réinitialisation du mot de passe est amorcée.

| Déclencheur                     | Description                                                                                        |
| ------------------------------- | -------------------------------------------------------------------------------------------------- |
| `post-change-password`          | Se déclenche après qu’un mot de passe a été modifié.                                               |
| `password-reset-post-challenge` | Se déclenche après la première étape de vérification et avant la réinitialisation du mot de passe. |

<div id="machine-to-machine">
  #### Machine to Machine
</div>

Lorsqu’une application, plutôt qu’un utilisateur, est authentifiée et autorisée à appeler vos API.

| Déclencheur            | Description                                          |
| ---------------------- | ---------------------------------------------------- |
| `credentials-exchange` | Se déclenche avant qu’un jeton d’accès soit renvoyé. |

<div id="custom-token-exchange">
  ### Échange de jeton personnalisé
</div>

Lorsqu’une requête d’échange de jeton personnalisé est reçue afin d’ouvrir la session d’un utilisateur.

| Déclencheur             | Description                                                                          |
| ----------------------- | ------------------------------------------------------------------------------------ |
| `custom-token-exchange` | Se déclenche à la première étape de la transaction, avant le déclencheur post-login. |
