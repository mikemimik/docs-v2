Ce guide vous montrera comment créer et déployer votre première Action à l’aide du <Tooltip tip="Auth0 Dashboard : l’interface principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Lorsque vous aurez terminé, vous devriez comprendre les fonctionnalités de base communes à toutes les Actions et vous serez familiarisé avec le modèle de programmation. Ce que vous apprendrez jette les bases de la création de toute Action, peu importe son objectif ou le flow.

<div id="the-goal-notify-slack-on-user-login">
  ## L’objectif : aviser Slack lors de la connexion d’un utilisateur
</div>

Vous créez une Action qui envoie un message à un canal Slack lorsqu’un utilisateur se connecte. Bien que ce tutoriel utilise le déclencheur Post Login, les renseignements fournis ici sont utiles pour les déclencheurs de tout type.

Pour accomplir cette tâche, vous ferez ce qui suit :

* [Créer une Action](#create-an-action)
* [Ajouter un Secret](#add-a-secret)
* [Ajouter une dépendance](#add-a-dependency)
* [Enregistrer le brouillon](#save-the-draft)
* [Ajouter une logique personnalisée](#add-custom-logic)
* [Tester une Action](#test-an-action)
* [Déployer une Action](#deploy-the-action)
* [Associer une Action à un Flow](#attach-the-action-to-a-flow)
* [Voir les résultats d’une Action dans les journaux du tenant](#observe-actions-in-tenant-logs)

<div id="prerequisites">
  ## Prérequis
</div>

Comme cette Action enverra des messages dans un canal Slack, vous devez [créer un webhook entrant pour un espace de travail Slack](https://api.slack.com/messaging/webhooks). Une fois que vous avez une URL de webhook Slack, vous pouvez poursuivre ce guide.

<div id="create-an-action">
  ## Créer une Action
</div>

Pour qu’une Action fonctionne dans un flow précis, vous devez d’abord créer l’Action, puis l’ajouter à un flow.

1. Accédez à [Auth0 Dashboard &gt; Actions &gt; Library,](https://manage.auth0.com/#/actions/library), puis sélectionnez **Create Action** &gt; **Build from scratch**.
2. Saisissez un **Name** et sélectionnez le déclencheur **Login / Post Login**, puisque vous allez ajouter une Action au flux Login, puis sélectionnez **Create.**

L’éditeur de code Actions s’affiche :

<Frame>![Éditeur de code Actions](/docs/images/cdy7uua7fh8z/12Q8OEuDI39eWuWsrVVwN0/ff7dd385d33aee704e1cb603f43c679b/2025-02-28_13-27-14.png)</Frame>

<div id="create-an-action-from-a-template">
  ## Créer une Action à partir d’un modèle
</div>

La galerie de modèles d’Actions propose différents modèles de départ pour vous aider à créer une Action. Pour créer une Action à partir d’un modèle :

1. Accédez à [Auth0 Dashboard &gt; Actions &gt; Library](https://manage.auth0.com/#/actions/library), puis sélectionnez **Create Action**.
2. Sélectionnez **Choose from template**.
3. Sélectionnez le modèle que vous souhaitez utiliser pour créer une Action.
4. Vous devriez maintenant voir un aperçu en lecture seule du code du modèle. Pour continuer, sélectionnez **Use this template**.
5. Entrez un nom, puis sélectionnez **Create**.

<div id="add-a-secret">
  ## Ajouter un Secret
</div>

Chaque Action peut contenir des valeurs Secret, idéales pour stocker des renseignements sensibles, comme les <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Access+Tokens">jetons d’accès</Tooltip> et les clés API. Stockons l’URL du webhook Slack comme Secret.

1. Sélectionnez l’icône en forme de clé dans la barre latérale gauche de l’éditeur de code
2. Sélectionnez **Add Secret**.
3. Donnez au Secret le nom suivant : `SLACK_WEBHOOK_URL`.
4. Collez l’URL du webhook fournie par Slack, puis sélectionnez **Create**.

Vous verrez que le Secret a été ajouté à l’Action et que vous pouvez utiliser ce nouveau secret grâce à l’autocomplétion du code en tapant `event.secrets`.

<Frame>![Actions Code Editor - Votre Secret a été ajouté dans la section Secrets.](/docs/images/cdy7uua7fh8z/4FyFtKRSIYuigxE9U8TkEJ/9598b5909694631ca673f748abf2268a/Screen_Shot_2022-02-09_at_3.24.05_PM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Une fois un Secret créé, sa valeur ne sera plus jamais affichée. Auth0 chiffre tous les Secrets et les stocke de façon sécurisée.
</Callout>

<div id="add-a-dependency">
  ## Ajouter une dépendance
</div>

Vous utiliserez le package [`@slack/webhook` `npm`](https://www.npmjs.com/package/@slack/webhook) pour faciliter l’envoi d’un message dans Slack. Vous pouvez utiliser presque n’importe quel package `npm` public dans une Action, à condition qu’il puisse être installé sans dépendre de [modules natifs complémentaires](https://www.npmjs.com/package/node-gyp).

Pour ajouter la dépendance :

1. Sélectionnez l’icône de dépendance (icône de cube) dans la barre latérale.
2. Dans **Name**, entrez `@slack/webhook`.

   Par défaut, votre Action utilisera la version la plus récente de la dépendance au moment de son ajout.
3. Sélectionnez **Create** pour ajouter la dépendance à l’Action :

   <Frame>![Actions Code Editor : Votre dépendance a été ajoutée.](/docs/images/cdy7uua7fh8z/Jmzvgydo204pBLikzuDJy/1d9a614b4c92f27afbbdd2b2af3b3c0b/Screen_Shot_2022-02-23_at_11.28.41_AM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Lorsque vous enregistrez cette Action, la version la plus récente de votre dépendance sera résolue et remplacée par un numéro de version précis afin d’éviter que de futures mises à jour du package ne brisent votre Action.
</Callout>

<div id="save-the-draft">
  ## Enregistrer le brouillon
</div>

Sélectionnez **Enregistrer le brouillon**. Votre Action est enregistrée, mais elle ne sera pas encore exécutée dans le cadre d’aucun parcours utilisateur. Avec Actions, vous pouvez créer, modifier et tester des Actions avant qu’elles n’aient une incidence sur le trafic de votre tenant Auth0. Une fois que le comportement de l’Action vous convient, vous pourrez déployer l’Action et l’ajouter plus tard à un Flow.

<div id="add-custom-logic">
  ## Ajouter une logique personnalisée
</div>

Maintenant que l’Action a été configurée avec un Secret et une dépendance, écrivons du code !

Chaque Action possède un objet `event` qui contient des informations contextuelles en lecture seule pertinentes pour le déclencheur associé. Si vous commencez à taper `event.` dans l’Editor, toutes les propriétés disponibles de `event` devraient s’afficher. Pour en savoir plus sur les informations disponibles pour chaque déclencheur, consultez [Explorer les Flows et les déclencheurs](/fr-CA/docs/customize/actions/explore-triggers).

Utilisons notre Action pour notifier Slack. Ajoutez ce code à votre action :

```javascript lines
const { IncomingWebhook } = require('@slack/webhook');

exports.onExecutePostLogin = async (event, api) => {
    const url = event.secrets.SLACK_WEBHOOK_URL;
    const webhook = new IncomingWebhook(url);

    // Envoyer la notification
    await webhook.send({ text: "Logging In..." });
};
```

Sélectionnez **Enregistrer comme brouillon** pour enregistrer votre Action.

<div id="test-the-action">
  ## Tester l’Action
</div>

Avant d’exécuter cette Action dans votre tenant avec du vrai trafic d’utilisateurs, testez-la pour vous assurer qu’elle fonctionne comme prévu.

1. Sélectionnez l’icône Test (triangle) dans la barre latérale. Dans la section Payload, vous verrez des données JSON modifiables associées au déclencheur que vous avez sélectionné lors de la création de l’Action.

   <Frame>![Actions Code Editor : panneau Test Runner](/docs/images/cdy7uua7fh8z/4MSu8EfpMLNJpcYtrwP7Jf/66480a0434c89eadb809012bca890037/Screen_Shot_2022-02-10_at_10.01.10_AM.png)</Frame>
2. Sélectionnez le bouton **Run** et vous devriez voir un message s’afficher dans votre canal Slack.

<div id="deploy-the-action">
  ## Déployer l’Action
</div>

Maintenant que nous avons confirmé que l’Action fonctionne comme prévu, il est temps de la déployer.

Sélectionnez **Deploy**. Le déploiement d’une Action crée un instantané de l’Action à ce moment précis et l’enregistre comme une **version d’Action**.

Sélectionnez **Version History**; vous devriez voir l’Action enregistrée en tant que version 1 :

<Frame>![Actions : historique des versions](/docs/images/cdy7uua7fh8z/53sn70cXoxUHibc3Bmk7Og/193af512ac2b58f3d60ab36ca5499d90/Screen_Shot_2022-02-10_at_10.02.47_AM.png)</Frame>

Vous pouvez maintenant mettre à jour votre Action et l’enregistrer sans modifier la version 1 de votre Action. Auth0 n’exécutera que la version actuellement déployée d’une Action; tant que vous ne déployez pas de nouveau l’Action, c’est la version 1 qu’Auth0 exécutera.

<div id="attach-the-action-to-a-flow">
  ## Associer l’Action à un Flow
</div>

La dernière étape pour configurer une nouvelle Action afin qu’elle s’exécute dans le trafic de votre tenant consiste à l’associer à un Flow.

1. Accédez à [Auth0 Dashboard &gt; Actions &gt; Flows](https://manage.auth0.com/#/actions/flows) pour afficher la liste des Flows disponibles :
2. Sélectionnez le [Login flow](/fr-CA/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) et faites glisser `my-slack-action` dans le flow.

   <Frame>![Actions Flows: Login Flow](/docs/images/cdy7uua7fh8z/6hU7sKWtGzSx9Z8dhsXbvp/0253fb4e9eac1133cc9626385622c54e/Login_Action_-_English.png)</Frame>
3. Sélectionnez **Apply**.

L’Action s’exécute maintenant sur le trafic réel des utilisateurs de votre tenant. Si vous souhaitez un environnement de test plus robuste, envisagez de [configurer plusieurs environnements Auth0](/fr-CA/docs/get-started/auth0-overview/create-tenants/set-up-multiple-environments).

<div id="observe-actions-in-tenant-logs">
  ## Consulter les Actions dans les journaux du tenant
</div>

Maintenant que votre Action s’exécute dans le Login flow, Auth0 consigne des renseignements sur chaque exécution d’Action dans les journaux du tenant. Pour les afficher, accédez à [Auth0 Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs), puis sélectionnez un événement **Successful Login**.

Vous devriez voir une vue **Action Details** qui contient des renseignements sur toutes les Actions exécutées dans ce flow.

<Frame>![Journaux du tenant : vue Action Details](/docs/images/cdy7uua7fh8z/61k82wmsXSb9M3wOExM4jH/51fae3a36a8ecddeec08bc45da468e2b/Action_logs_-_English.png)</Frame>
