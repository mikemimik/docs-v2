title: Migrer les connexions Office365 vers Windows Azure AD
---

Office365 est obsolète. Vous devriez migrer vos connexions Office365 vers des connexions Windows Azure AD.

Depuis le début, nous prenons en charge l’authentification des utilisateurs avec Office365. Office365 a toujours utilisé Windows Azure AD en arrière-plan, mais il n’existait pas de bonne UI pour créer une « application » dans Windows Azure AD. C’est pourquoi vous deviez la créer dans le Seller Dashboard. Dorénavant, Microsoft souhaite que vous utilisiez Windows Azure AD, et vous pouvez maintenant créer facilement un annuaire associé à votre compte Office365 et à l’application.

<div id="how-to-migrate-to-azure">
  ## Comment migrer vers Azure
</div>

1. Créez un abonnement à Windows Azure AD (gratuit)
2. Créez un annuaire (qui sera associé à votre compte Office365), ainsi qu’une application. Pour en savoir plus, consultez [Connect Your App to Microsoft Azure Active Directory](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

Si vous utilisiez `user_id` dans votre application, notez qu’il passera de `office365|....some-guid....` à `waad|...email....`.
