Il existe de nombreuses façons d’intégrer Auth0 aux produits et services Amazon, selon vos besoins précis.

<div id="product-integrations">
  ## Intégrations de produits
</div>

Pour en savoir plus sur l’utilisation des produits ci-dessous avec vos services Auth0 et AWS :

* [CloudFront](/fr-CA/docs/customize/custom-domains/self-managed-certificates/configure-aws-cloudfront-for-use-as-reverse-proxy) : Utilisez-le comme proxy inverse avec votre <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=custom+domain">domaine personnalisé</Tooltip>.
* [Simple Email Service (SES)](/fr-CA/docs/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider) : Gérez les communications par courriel avec vos utilisateurs.
* [EventBridge](https://marketplace.auth0.com/integrations/amazon-log-streaming) : Transmettez les journaux vers EventBridge.
* [Cognito](/fr-CA/docs/customize/integrations/aws/amazon-cognito) : Utilisez-le comme backend pour votre application.
* AWS S3 et DynamoDB :

  * Auth0 fournit des endpoints de passerelle VPC pour accéder à AWS S3 et à DynamoDB, ce qui peut être nécessaire pour [Actions](/fr-CA/docs/customize/actions). Cela nécessite des policies IAM basées sur `aws:SourceVpc`, comme décrit dans [Gateway endpoints for Amazon S3](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-s3.html) et [Gateway endpoints for Amazon DynamoDB](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-ddb.html). Veuillez communiquer avec [Auth0 support](https://www.support.auth0.com) pour obtenir plus de détails sur les identifiants VPC.

<div id="identity-and-access-management-iam-integrations">
  ## Intégrations de gestion des identités et des accès (IAM)
</div>

Nous avons plusieurs guides pratiques pour configurer AWS afin de fonctionner avec Auth0 :

* [Sécuriser les endpoints d’AWS API Gateway à l’aide d’authorizers personnalisés](/fr-CA/docs/customize/integrations/aws/aws-api-gateway-custom-authorizers) : Acceptez des <Tooltip tip="Jeton d’accès : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+tokens">jetons d’accès</Tooltip> émis par Auth0.
* [Utiliser les balises de session AWS avec les API et les ressources AWS](/fr-CA/docs/customize/integrations/aws/session-tags) : Étiquetez les ressources et attribuez aux utilisateurs des paires clé/valeur pour le RBAC.
* [Configurer AWS pour l’authentification unique](/fr-CA/docs/customize/integrations/aws/configure-amazon-web-services-for-sso) : Permettez à vos utilisateurs de se connecter à AWS à l’aide de n’importe quel <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> pris en charge.
* [Ajouter la connexion Amazon à votre application](/fr-CA/docs/customize/integrations/aws/configure-amazon-web-services-for-sso) : Permettez aux utilisateurs de se connecter à votre application à l’aide de leur profil Amazon.
