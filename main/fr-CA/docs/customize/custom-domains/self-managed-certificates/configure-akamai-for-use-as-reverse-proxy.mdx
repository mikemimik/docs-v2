<Card title="La disponibilité varie selon le plan Auth0">
  La disponibilité de cette fonctionnalité dépend à la fois de votre mise en œuvre de connexion et de votre plan Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Card>

Pour configurer un produit Akamai comme proxy inverse, suivez les étapes ci-dessous :

1. Vérifiez la propriété de votre domaine personnalisé. Pour en savoir plus, consultez [Verify Ownership](/docs/fr-CA/customize/custom-domains/auth0-managed-certificates#verify-ownership).
2. Accédez à [Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) pour récupérer les paramètres de distribution, de l’en-tête personnalisé d’origine, de `origin hostname` et du comportement de cache par défaut. Utilisez ensuite ces renseignements pour mettre à jour les paramètres de votre domaine personnalisé Akamai afin qu’ils pointent vers le domaine du serveur proxy inverse. Pour en savoir plus, consultez [Configure reserve proxy](/docs/fr-CA/customize/custom-domains/self-managed-certificates#configure-reverse-proxy).
3. Mettez à jour l’en-tête IP client du domaine personnalisé à l’aide du point de terminaison PATCH `/api/v2/custom-domains/{id}` de la Management API, comme indiqué ci-dessous :

   ```json lines
   { 
   "custom_client_ip_header": "true-client-ip" 
   }
   ```

   Pour en savoir plus, consultez [Update custom domain configuration](https://auth0.com/docs/api/management/v2#!/Custom_Domains/patch_custom_domains_by_id).

Pour savoir comment configurer Akamai comme proxy inverse à l’aide des renseignements ci-dessus, consultez la documentation d’Akamai ou le soutien à la clientèle.
