title: Configurer Azure CDN comme proxy inverse
---

<Card title="La disponibilité varie selon le plan Auth0">
  Votre plan Auth0 ou votre entente personnalisée détermine si cette fonctionnalité est offerte. Pour en savoir plus, consultez [Tarification](https://auth0.com/pricing).
</Card>

Pour configurer Azure CDN comme proxy inverse, un plan Azure CDN Premium est requis.

1. [Configurez les domaines personnalisés avec des certificats autogérés](/fr-CA/docs/customize/custom-domains/self-managed-certificates) si ce n&#39;est pas déjà fait. Prenez en note les valeurs **Origin Domain Name** et **cname-api-key**, car vous en aurez besoin plus tard.
2. Connectez-vous au [portail Azure](https://portal.azure.com/).
3. [Créez un nouveau profil Azure CDN et un nouveau point de terminaison](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint). Dans les paramètres du point de terminaison CDN, indiquez le **Origin Domain Name** provenant du Auth0 Dashboard.
4. [Ajoutez votre domaine personnalisé à votre point de terminaison Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup).
5. [Configurez HTTPS pour votre domaine personnalisé Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate). Ce processus exige que vous vérifiiez la propriété du domaine. Une fois cette étape terminée, le déploiement du certificat dans tous les points de présence (PoP) du CDN peut prendre jusqu&#39;à 6 heures.
6. Configurez la communication du domaine personnalisé avec Auth0 à l&#39;aide du moteur Rules d&#39;Azure CDN. (Pour en savoir plus, consultez [Remplacer le comportement HTTP à l&#39;aide du moteur Rules Azure CDN from Verizon Premium](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) dans la documentation Microsoft.) [Créez une nouvelle Rule Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) avec les paramètres suivants :

   | Paramètre        | Valeur                                                                                                       |
   | ---------------- | ------------------------------------------------------------------------------------------------------------ |
   | Name/Description | Auth0 Custom Domain                                                                                          |
   | Type of requests | Sélectionnez l&#39;option **Edge CName**, puis sélectionnez votre nom de domaine personnalisé dans la liste. |
7. Ajoutez les **Features** suivantes à votre Rule Azure CDN :
   Nous vous recommandons de créer une autre Rule Azure CDN pour rejeter l&#39;utilisation du CNAME **azureedge.net**.

   | Paramètre                    | Valeur                                                                                                               |
   | ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
   | Bypass Cache                 | **Activé**                                                                                                           |
   | Modify Client Request Header | Sélectionnez **Override**, entrez **cname-api-key** comme nom, puis la clé API CNAME fournie par Auth0 comme valeur. |
8. Une fois la Rule Azure CDN approuvée, le status passera de Pending XML à Active XML. À ce stade, Azure CDN publiera les rules et les certificats. Quand Azure aura terminé de traiter tous les changements, vous pourrez utiliser votre domaine personnalisé.
