La page de configuration de base vous permet de gérer les paramètres suivants pour Universal Login et Classic Login :

* **Moteur de connexion** : Passez de Universal Login à Classic Login.
* **Intégration par iframe** : Activez le chargement des flux de connexion dans des iframes pour les environnements de développement.

<div id="settings">
  ## Paramètres
</div>

<div id="universal-or-classic-login">
  ### Universal Login ou Classic Login
</div>

Choisissez l’expérience de connexion utilisée par votre tenant. La modification de ce paramètre a une incidence sur toutes les applications qui redirigent vers votre page de connexion Auth0. Pour en savoir plus, consultez [Universal Login vs. Classic Login](/fr-CA/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<Steps>
  <Step title="Accédez à configuration de base">
    Accédez à **Branding &gt; Universal Login &gt; Pro-code customizations** et sélectionnez **configuration de base**.
  </Step>

  <Step title="Sélectionnez le type de connexion">
    Dans **Settings**, sélectionnez **Universal Login** ou **Classic Login**.
  </Step>
</Steps>

<div id="iframe-embedding">
  ### intégration par iframe
</div>

Activez Universal Login ou Classic Login pour qu’ils se chargent lorsque l’application et le flux de connexion sont intégrés dans une iframe, comme avec des plateformes d’UI générative comme Vercel. Lorsqu’elle est activée, l’intégration par iframe s’applique uniquement aux environnements de développement.

<Warning>
  Lorsque l’intégration par iframe est activée, les en-têtes de sécurité qui protègent contre le détournement de clics sont désactivés.
</Warning>

<Steps>
  <Step title="Accéder à configuration de base">
    Accédez à **Branding &gt; Universal Login &gt; Pro-code customizations** et sélectionnez **configuration de base**.
  </Step>

  <Step title="Activer la prise en charge des iframe">
    Sous **Iframe settings**, activez la bascule **Enable support for generative UI platforms**.
  </Step>

  <Step title="Configurer les origines autorisées">
    Sous **Allowed iframe URLs for Login**, saisissez les origines autorisées.

    Pour v0, saisissez les origines suivantes :

    * `https://v0.app`
    * `https://*.v0.app`
    * `https://*.vusercontent.net`

    Lorsqu’une application est créée à partir d’une plateforme d’UI générative comme Vercel à l’aide d’une intégration Auth0 disponible, Auth0 préremplit les origines autorisées à partir de la plateforme. Vérifiez toujours les origines préremplies pour vous assurer qu’elles sont complètes. Pour en savoir plus, consultez [Integrate with Vercel](/fr-CA/docs/customize/integrations/integrate-with-vercel).
  </Step>
</Steps>
