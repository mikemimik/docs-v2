---
description: Découvrez comment utiliser les Actions Auth0 pour valider et stocker les données des utilisateurs finaux.
title: Utiliser les Actions pour valider et stocker les données des utilisateurs finaux recueillies grâce aux personnalisations de l’invite d’inscription
---

Vous pouvez utiliser la [personnalisation de l’invite d’inscription](/fr-CA/docs/customize/login-pages/universal-login/customize-signup-and-login-prompts) avec le déclencheur `pre-user-registration` pour ajouter à `user_metadata` des données fournies par l’utilisateur final (comme son numéro de téléphone ou son emplacement) saisies dans l’invite d’inscription. Vous pouvez aussi valider ces données et afficher une erreur de validation dans l’invite.

<div id="prerequisites">
  ### Prérequis
</div>

* Le tenant a un [domaine personnalisé](/fr-CA/docs/customize/custom-domains) vérifié
* Le tenant a un [gabarit de page](/fr-CA/docs/customize/login-pages/universal-login/customize-templates) configuré

<div id="add-a-field-to-the-signup-prompt">
  ### Ajouter un champ à l’invite d’inscription
</div>

Utilisez la [Management API](https://auth0.com/docs/api/management/v2/prompts/put-custom-text-by-language) pour insérer un champ personnalisé dans un prompt à l’un des [points d’entrée](/fr-CA/docs/customize/login-pages/universal-login/customize-signup-and-login-prompts#-signup-prompt-entry-points). Cet exemple ajoute le contenu suivant au point d’insertion `ulp-container-form-content-start` :

```html lines
<div class="ulp-field">
  <label for="first-name">First Name</label>
  <input type="text" name="ulp-first-name" id="first-name">
</div>
```

Le résultat est une section « Prénom » dans l’invite d’inscription :

<Frame>![](/docs/images/cdy7uua7fh8z/580LNGMxSkRlazJEdcVNpn/5fa98d4b414ac2133d1d6f4e327cea5a/Screenshot_2023-10-05_at_4.07.38_PM.png)</Frame>

<div id="create-an-action-in-the-pre-user-registration-trigger">
  ### Créer une Action dans le déclencheur Pre-User Registration
</div>

Vous pouvez créer une Action pre-user registration personnalisée en allant à [**Actions &gt; Bibliothèque &gt; Build Custom**](https://manage.auth0.com/dashboard/us/dev-6endizjt/actions/library)**.**

<Frame>![](/docs/images/cdy7uua7fh8z/52VSznphe2TyoJQnAn4nQg/ed54ca48dea191916a3173b648463fd8/Create_Action_-_EN.png)</Frame>

Dans l’éditeur de code, mettez à jour le Handler `onExecutePreUserRegistration` :

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const firstName = event.request.body['ulp-first-name'];
  api.user.setUserMetadata("firstName", firstName);
};
```

Au besoin, vous pouvez valider la saisie de l’utilisateur et retourner une erreur de validation en appelant la méthode `api.validation.error`, puis déployer l’Action :

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const firstName = event.request.body['ulp-first-name'];
  if(!firstName) {
    api.validation.error("invalid_payload", "Missing first name");
    return;
  }

  api.user.setUserMetadata("firstName", firstName);
};
```

<div id="add-the-action-to-the-flow">
  ### Ajouter l’action au flow
</div>

Accédez à [**Actions &gt; Flows &gt; Pre User Registration &gt; Add Action &gt; Custom**](https://manage.auth0.com/dashboard/us/dev-6endizjt/actions/flows/pre-user-signup/), puis faites glisser votre nouvelle action dans le flow d’inscription et sélectionnez **Apply**.

<Frame>![](/docs/images/cdy7uua7fh8z/5JmjwcdZCX7pDukNa90cgD/dae853b10cf4c41d2d3543a8e167aae7/3333.png)</Frame>

<div id="test-the-action">
  ### Testez l’Action
</div>

Créez un compte dans votre flow de test et laissez le champ **Prénom** vide. Vous verrez une erreur au moment de soumettre le formulaire :

<Frame>![](/docs/images/cdy7uua7fh8z/7qIWCsP8doFWPJGvMTNXJY/1225f9b3ff7ac01a7e9b59f03ca5e139/2024-01-31_16-10-10.png)</Frame>

Après avoir saisi un prénom dans le champ **Prénom**, vous pourrez soumettre le formulaire avec succès :

<Frame>![](/docs/images/cdy7uua7fh8z/6TaK1VQyKGELMGmhuRIbVA/b453fe2eb97c95d92e4fbd68ce3cbd6a/2024-01-31_16-12-01.png)</Frame>

<div id="verify-that-the-data-was-saved-on-user_metadata">
  ### Vérifiez que les données ont bien été enregistrées dans user_metadata
</div>

Accédez à [**User Management &gt; Users**](https://manage.auth0.com/dashboard/us/dev-6endizjt/users), puis confirmez que les données ont bien été enregistrées en consultant l’onglet Details :

<Frame>![](/docs/images/cdy7uua7fh8z/3x5lyrSJJgJQhbjsxoTmG6/033820476be7f67dc688da993fee7924/2024-01-31_16-13-35.png)</Frame>
