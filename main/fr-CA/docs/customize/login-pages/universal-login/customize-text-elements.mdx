<Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> se compose d’un ensemble de pages qui facilitent plusieurs actions liées au compte, comme la connexion, la configuration de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur l’Authorization Server d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=multi-factor+authentication">facteurs d’authentification multifacteur</Tooltip> ou la réinitialisation de mots de passe. Auth0 fournit le texte de ces pages en plusieurs langues.

Dans certains cas, vous souhaiterez peut-être modifier le texte de ces pages pour mieux l’adapter au ton ou aux besoins particuliers de votre application.

Vous pouvez personnaliser le texte affiché sur les pages de [Universal Login](/fr-CA/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) dans toutes les langues prises en charge, au moyen de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>. Les définitions, valeurs et variables suivantes s’appliquent aux deux méthodes.

<div id="configure-prompts-and-screens">
  ## Configurer les prompts et les écrans
</div>

Vous pouvez personnaliser n’importe lequel des [prompts et écrans](#prompt-values) disponibles à partir de l’Auth0 Dashboard ou au moyen de la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Utiliser le Dashboard">
    Vous pouvez utiliser le Dashboard pour personnaliser les textes des prompts du nouveau Universal Login.

    1. Dans le Dashboard, accédez à [**Branding &gt; Universal Login &gt; Advanced Options**](https://manage.auth0.com/#/custom_text), puis sélectionnez **Texte personnalisé**.
    2. Sélectionnez le **Prompt** et le **Screen** que vous souhaitez modifier, puis apportez les modifications voulues. Un aperçu en direct est disponible à droite.
    3. Lorsque vous êtes prêt à afficher vos modifications dans votre environnement de production, sélectionnez **Save Changes.**
  </Tab>

  <Tab title="Utiliser l’API Management">
    Vous pouvez utiliser la Management API pour personnaliser les invites de texte du nouveau Universal Login.

    1. Obtenez un [jeton d’accès à l’API de gestion](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les portées `read:prompts` et `update:prompts`.
    2. Appelez le point de terminaison [Get custom text for a prompt](https://auth0.com/docs/api/management/v2#!/Prompts/get_custom_text_by_language) de la Management API pour obtenir les données de configuration existantes pour le prompt et la langue indiqués.
    3. Modifiez les données de configuration au besoin, en vous servant des [paramètres pris en charge](#supported-parameters) et des [valeurs de prompt](#prompt-values) comme référence.
    4. Appelez le point de terminaison [Définir un texte personnalisé pour un prompt spécifique](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de la Management API avec les données de configuration mises à jour.

       <Warning>
         Le endpoint [Set custom text for a specific prompt](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de la Management API remplace toutes les données de configuration existantes du prompt indiqué par les données de configuration fournies dans le paramètre `body`.

         Si vous souhaitez configurer plusieurs valeurs `screen` pour un prompt donné, vous devez toutes les inclure dans un seul appel à la Management API.
       </Warning>

    #### Exemple

    Vous pouvez définir le composant de texte `description` de l&#39;écran `login` de l&#39;invite `login` à `Log in to ACME's website` en appelant le point de terminaison [Set custom text for a specific prompt](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de la Management API de la façon suivante :

      ```bash cURL
      curl --request PUT \
        --url 'https://{yourDomain}/api/v2/prompts/login/custom-text/en' \
        --header 'authorization: Bearer {yourMgmtApiAccessToken}' \
        --header 'content-type: application/json' \
        --data '{ "login": { "description": "Login to ACME'''s Website" } }'
      ```

      ```csharp C#
      var client = new RestClient("https://{yourDomain}/api/v2/prompts/login/custom-text/en");
      var request = new RestRequest(Method.PUT);
      request.AddHeader("content-type", "application/json");
      request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
      request.AddParameter("application/json", "{ "login": { "description": "Login to ACME's Website" } }", ParameterType.RequestBody);
      IRestResponse response = client.Execute(request);
      ```

      ```go Go
      package main

      import (
      	"fmt"
      	"strings"
      	"net/http"
      	"io/ioutil"
      )

      func main() {

      	url := "https://{yourDomain}/api/v2/prompts/login/custom-text/en"

      	payload := strings.NewReader("{ "login": { "description": "Login to ACME's Website" } }")

      	req, _ := http.NewRequest("PUT", url, payload)

      	req.Header.Add("content-type", "application/json")
      	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

      	res, _ := http.DefaultClient.Do(req)

      	defer res.Body.Close()
      	body, _ := ioutil.ReadAll(res.Body)

      	fmt.Println(res)
      	fmt.Println(string(body))

      }
      ```

<div id="supported-parameters">
  ### Paramètres pris en charge
</div>

| Paramètre  | Définition                                                                                                                                                                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `prompt`   | L’étape du flux de connexion. Pour obtenir la liste des valeurs disponibles, consultez [Valeurs de prompt](#prompt-values).                                                                                                                       |
| `language` | La langue à laquelle la personnalisation s’applique. Pour obtenir la liste des langues disponibles, consultez [Universal Login Internationalization](/fr-CA/docs/customize/internationalization-and-localization/universal-login-internationalization). |
| `screen`   | L’écran spécifique dans un prompt.                                                                                                                                                                                                                |
| `key`      | L’identifiant d’un composant de texte.                                                                                                                                                                                                            |
| `text`     | Le message affiché à l’utilisateur.                                                                                                                                                                                                               |

<div id="variables">
  ## Variables
</div>

Certains écrans contiennent des variables dans le texte qui sont remplacées au moment de l’exécution selon les informations de contexte. Comme les variables offertes diffèrent d’un écran à l’autre, rien ne garantit qu’une variable donnée fonctionnera ailleurs que sur son écran d’origine.

| Variable                      | Description                                                                                                      |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| $&#123;clientName&#125;       | Nom de l’application Auth0                                                                                       |
| $&#123;connectionName&#125;   | Nom de la connexion (p. ex. Google)                                                                              |
| $&#123;tenantName&#125;       | Nom du tenant Auth0                                                                                              |
| $&#123;organizationName&#125; | Nom de l’organisation de l’utilisateur si celui-ci s’authentifie dans le contexte d’une organisation             |
| $&#123;companyName&#125;      | Nom du tenant Auth0, ou nom de l’organisation si l’utilisateur s’authentifie dans le contexte d’une organisation |
| $&#123;userName&#125;         | Nom de l’utilisateur connecté                                                                                    |
| $&#123;email&#125;            | Adresse courriel de l’utilisateur connecté                                                                       |
| $&#123;appName&#125;          | Nom de l’application Guardian Push personnalisée                                                                 |
