Vous pouvez configurer ACUL à l’aide de l’Auth0 Dashboard, de l’Auth0 Management API et d’outils d’infrastructure as code (IaC). Chaque méthode modifie le même objet de configuration du rendu propre à chaque écran.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour configurer ACUL, assurez-vous que vos fichiers de configuration personnalisés et vos ressources sont hébergés sur un CDN public.
</Callout>

Pour comprendre l’objet de configuration du rendu et ses propriétés, consultez la [référence de la configuration du rendu d’ACUL](/docs/fr-CA/customize/login-pages/advanced-customizations/configure/reference).

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Vous pouvez utiliser le Dashboard pour personnaliser vos écrans avec ACUL :

A. Accédez à [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/dashboard/#/universal-login/customizations-new)

1. Sélectionnez **Customize authentication screens**.
   2. Sélectionnez l’<code>screen</code> à personnaliser.
   3. Dans **Settings &gt; Rendering mode**, sélectionnez **Advanced mode**.
   4. (Facultatif) Sélectionnez **Use custom page template**.
   5. (Facultatif) Sélectionnez **Advanced mode filters**.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/aculDashboard.png" alt="Page ACUL de l’Auth0 Dashboard" />
</Frame>

6. Sélectionnez **Head Tags** pour ajouter vos propres balises HTML `Head Tags`.
   7. (Facultatif) Sélectionnez **Additional Data** pour choisir quelles données additionnelles envoyer au navigateur.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Additional Data** vous permet d’ajouter des propriétés de données de contexte facultatives. Par exemple : les propriétés de données de contexte avec OPT-IN de [`user`](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/UserMembers).
</Callout>

B. Sélectionnez **Save &amp; Publish**.

<div id="infrastructure-as-code-iac">
  ## Infrastructure as code (IaC)
</div>

Configurez ACUL à l’aide d’outils d’IaC, notamment :

* La commande [`auth0 acul`](https://auth0.github.io/auth0-cli/auth0_acul.html) de l’**Auth0 CLI**.
  * La ressource [`auth0_prompt_screen_renderer`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) de l’**Auth0 Terraform Provider**.
  * La ressource [`prompts`](https://github.com/auth0/auth0-deploy-cli/blob/master/docs/resource-specific-documentation.md#prompts) de l’**Auth0 Deploy CLI**.

<div id="auth0-management-api">
  ## Auth0 Management API
</div>

Utilisez les points de terminaison de l’Auth0 Management API pour configurer individuellement les écrans de Universal Login ou mettre à jour plusieurs écrans au moyen d’un seul appel d’API. Vous devez inclure un [jeton d’accès à l’API de gestion](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) dans la requête.

**Configuration d’un écran individuel**

* Le [point de terminaison `GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/get-rendering) récupère la configuration d’un seul écran.

  * Le [point de terminaison `PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-rendering) configure ACUL pour un seul écran.

**Configuration groupée des écrans**

* Le [point de terminaison `GET /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/get-all-rendering) récupère la configuration de tous les écrans.

  * Le [point de terminaison `PATCH /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-bulk-rendering) met à jour plusieurs écrans en bloc.
