Auth0 propose plusieurs SDKs à utiliser avec ACUL.

<div id="acul-sdks">
  ## ACUL SDKs
</div>

* [Auth0 ACUL React SDK](/docs/fr-CA/libraries/acul/react-sdk)
  Fournit des Hooks React pour simplifier l’intégration aux applications React.

  * [Auth0 ACUL JS SDK](/docs/fr-CA/libraries/acul/js-sdk)
    La bibliothèque Javascript de base (écrite en TypeScript), qui convient à tout framework frontend ou au Javascript pur.

<div id="acul-sample-applications">
  ## Exemples d’applications ACUL
</div>

Les [exemples d’applications ACUL](https://github.com/auth0-samples/auth0-acul-samples) comprennent un pipeline de build complet, des workflows de déploiement et des implémentations pour les écrans courants de Universal Login. Ces exemples d’applications vous aident à développer vos écrans. Les exemples sont disponibles en React à l’aide des SDKs ACUL React et JS.

Vous pouvez utiliser la commande `auth0 acul init` pour initialiser un projet React de départ avec les SDKs ACUL React ou JS. Pour en savoir plus, consultez [Auth0 CLI](https://github.com/auth0/auth0-cli) sur GitHub.

<div id="acul-resources">
  ## Ressources ACUL
</div>

Auth0 propose plusieurs ressources pour personnaliser votre expérience Universal Login.

<div id="auth0-management-api">
  ### Auth0 Management API
</div>

Mettez à jour des écrans individuels d’Universal Login ou configurez plusieurs écrans à la fois avec la Management API. Pour utiliser l’Auth0 Management API, vous devez disposer d’un [jeton d’accès](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens).

**Configuration d’un écran individuel**

* Le point de terminaison [`GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/get-rendering) vous permet de récupérer la configuration d’un écran précis.

  * Le point de terminaison [`PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-rendering) vous permet de configurer ACUL pour un écran précis.

**Configuration groupée des écrans**

* Le point de terminaison [`GET /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/get-all-rendering) vous permet de récupérer la configuration de tous les écrans.

  * Le point de terminaison [`PATCH /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-bulk-rendering) vous permet de mettre à jour plusieurs écrans à la fois.

<div id="auth0-developer-tools">
  ### Outils de développement Auth0
</div>

Auth0 propose plusieurs outils pour simplifier la gestion des tenants, y compris la configuration ACUL.

* **Auth0 CLI**<p />
  Utilisez les commandes `acul` pour générer la structure de projets, développer localement et déployer vos configurations d’écran. Pour en savoir plus, consultez [Auth0 CLI](https://github.com/auth0/auth0-cli) sur GitHub.

  * **Auth0 Deploy CLI**<p />
    Gérez toute la configuration de votre tenant comme du code, y compris les paramètres de rendu ACUL définis dans des fichiers JSON. Pour en savoir plus, consultez [Auth0 Deploy CLI](https://github.com/auth0/auth0-deploy-cli) sur GitHub.

  * **Auth0 Terraform provider**<p />
    Définissez vos configurations d’écran ACUL. Pour en savoir plus, consultez la ressource [`promt_screen_renderer`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) sur Terraform.
