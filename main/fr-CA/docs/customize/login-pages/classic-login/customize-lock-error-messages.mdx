Vous pouvez personnaliser les messages d’erreur affichés dans certaines situations en fournissant l’option `languageDictionary`. La liste complète des champs `languageDictionary` que vous pouvez personnaliser se trouve dans le [fichier de dictionnaire anglais pour Lock](https://github.com/auth0/lock/blob/master/src/i18n/en.js) du GitHub repository. Voici un exemple de messages d’erreur personnalisés :

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample = `// Exemples de messages d’erreur personnalisés dans l’option languageDictionary
var options = {
  languageDictionary: {
    error: {
      login: {
        "lock.invalid_email_password": "Message personnalisé concernant des informations d’identification invalides",
        "lock.network": "Message personnalisé indiquant une erreur réseau et suggérant à l’utilisateur de vérifier sa connexion",
        "lock.unauthorized": "Message personnalisé concernant un problème de permissions",
        "too_many_attempts": "Message personnalisé indiquant que l’utilisateur a échoué à se connecter trop souvent."
      },
      signUp: {
        "invalid_password": "Message personnalisé indiquant qu’un mot de passe n’était pas valide",
        "user_exists": "Message personnalisé indiquant qu’un utilisateur existe déjà"
      }
    }
  }
};

// Initialisation de notre Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<div id="custom-errors-in-rules">
  ## Erreurs personnalisées dans les Rules
</div>

Si vous renvoyez des codes d’erreur personnalisés à partir d’une [Rule](/fr-CA/docs/customize/rules) ou d’un script de base de données personnalisé, vous pouvez gérer ces erreurs personnalisées :

* Dans l’URL de redirection de votre application, en lisant les paramètres de la chaîne de requête `error` et `error_mesage`.
* En redirigeant l’utilisateur vers vos pages hébergées avec un message d’erreur personnalisé, puis en affichant le message au moyen d’un [message flash](/fr-CA/docs/libraries/lock/lock-api-reference).
