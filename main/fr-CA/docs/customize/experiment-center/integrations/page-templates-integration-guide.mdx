Lorsqu’une expérience est active et qu’une variation est attribuée, Experiment Center injecte automatiquement le contexte de l’expérience dans le contexte de rendu du [gabarit de page](/fr-CA/docs/customize/login-pages/universal-login/customize-templates#customize-universal-login-page-templates). Les gabarits de page ne nécessitent aucune étape d’activation explicite : le contexte de l’expérience est transmis à tous les gabarits sans condition lorsqu’une expérience est active. Templates reçoit ce contexte sous la forme d’une variable locale nommée `experiment`.

<Warning>
  Pendant la bêta, Experiment Center fonctionne uniquement avec les tenants de développement. Les tenants de production ne sont pas pris en charge.
</Warning>

<div id="available-context-fields">
  ## Champs de contexte disponibles
</div>

Les champs suivants sont disponibles dans le gabarit de page lorsqu’une expérience est active :

| **Variable**               | **Type** | **Description**                                                           |
| -------------------------- | -------- | ------------------------------------------------------------------------- |
| `experiment.experiment_id` | string   | L’ID de l’expérience active                                               |
| `experiment.variation_id`  | string   | L’ID de la variation attribuée                                            |
| `experiment.config`        | object   | Configuration fusionnée : paramètres de base + surcharges de la variation |
| `experiment.is_control`    | boolean  | Indique s’il s’agit de la variation témoin                                |

Lorsqu’aucune expérience n’est active, `experiment` vaut `null` (ou n’est pas présent). Vérifiez toujours qu’il n’est pas nul avant de l’utiliser.

<div id="experimentconfig-structure">
  ### Structure de `experiment.config`
</div>

`experiment.config` est une structure clé-valeur où chaque clé correspond à un nom de paramètre et chaque valeur est un objet doté d’une propriété `value` :

```
experiment.config.my_param.value  → the parameter's value for this variation
```

Chaque paramètre défini pour le feature flag a une valeur dans `config`. L’Experiment Center fusionne les paramètres de la configuration de référence avec les overrides de la variation avant l’injection; vous n’avez donc jamais besoin de chercher vous-même la valeur par défaut.

<div id="read-a-parameter-in-a-template">
  ## Lire un paramètre dans un modèle
</div>

Les gabarits de page d’Auth0 Universal Login utilisent EJS. Les exemples ci-dessous utilisent la syntaxe EJS.

**Syntaxe EJS :**

```html
<%= experiment && experiment.config.heading_text.value %>
```

Avec une vérification de valeur nulle :

```html
<% if (experiment) { %>
  <h1><%= experiment.config.heading_text.value %></h1>
<% } else { %>
  <h1>Create your account</h1>
<% } %>
```

<div id="customize-signup-and-login-with-partials">
  ## Personnalisez Signup et Login avec des Partials
</div>

Lorsque vous personnalisez les prompts Signup et Login à l’aide de [partials](/fr-CA/docs/customize/login-pages/universal-login/customize-signup-and-login-prompts#manage-partials-programmatically), le contexte de l’expérience est transmis explicitement dans votre partial.

Dans le partial, la même variable `experiment` est disponible :

```html
<!-- partials/signup-header.ejs -->
<header>
  <h1>
    <% if (experiment && experiment.config.heading_text) { %>
      <%= experiment.config.heading_text.value %>
    <% } else { %>
      Create your account
    <% } %>
  </h1>
</header>
```

<div id="example-signup-headline-copy-variant">
  ## Exemple : variante du texte du titre de la page d’inscription
</div>

Cet exemple montre un feature flag avec un paramètre String qui contrôle le titre de la page d’inscription. La variation témoin utilise &quot;Créez votre compte&quot; ; le traitement utilise &quot;Inscrivez-vous en quelques secondes.&quot;

**Paramètres du feature flag :**

```json
{
  "signup_headline": {
    "type": "string",
    "value": "Create your account",
    "description": "Headline text on the signup screen"
  }
}
```

**Variation témoin :** aucune surcharge (hérite de `signup_headline: "Create your account"`)

**Variation de traitement :**

```json
{
  "overrides": {
    "signup_headline": { "value": "Join in seconds" }
  }
}
```

**Code du gabarit :**

```html
<!-- inscription.ejs -->
<div class="signup-container">
  <h1>
    <% if (experiment) { %>
      <%= experiment.config.signup_headline.value %>
    <% } else { %>
      Create your account
    <% } %>
  </h1>

  <form action="/usernamepassword/login" method="POST">
    <!-- champs du formulaire -->
  </form>
</div>
```

Les utilisateurs assignés à la variation témoin voient « Créez votre compte. » Les utilisateurs du traitement voient « Inscrivez-vous en quelques secondes. » Les utilisateurs ne participant à aucune expérience active voient aussi « Créez votre compte » (la branche else).

<div id="example-conditional-ui-element">
  ## Exemple : élément UI conditionnel
</div>

Cet exemple utilise un paramètre booléen pour afficher ou masquer un élément UI supplémentaire :

```html
<!-- login.ejs -->
<% if (experiment && experiment.config.show_social_proof.value === true) { %>
  <div class="social-proof">
    <p>Join 2 million people who sign in with their passkey.</p>
  </div>
<% } %>

<form action="/usernamepassword/login" method="POST">
  <!-- champs du formulaire -->
</form>
```

La comparaison `=== true` (plutôt qu’un test de type « truthy ») empêche l’élément de s’afficher lorsque `experiment` vaut `null` ou lorsque le paramètre est absent.

<div id="safe-defaults-when-no-experiment-is-active">
  ## Valeurs par défaut sûres lorsqu’aucune expérience n’est active
</div>

Lorsqu’aucune expérience n’est active, `experiment` est `null` dans le contexte du gabarit. Structurez vos gabarits pour qu’ils s’affichent toujours correctement, même sans contexte d’expérience :

```html
<h1>
  <%= (experiment && experiment.config.page_title && experiment.config.page_title.value)
      || "Default page title" %>
</h1>
```

Ou encore, utilisez des blocs if/else explicites (plus lisibles pour les conditions complexes) :

```html
<% if (experiment && experiment.config.page_title) { %>
  <h1><%= experiment.config.page_title.value %></h1>
<% } else { %>
  <h1>Default page title</h1>
<% } %>
```

Définissez en dur les valeurs de repli dans vos gabarits. Elles correspondent à la configuration de référence de la variation témoin, de sorte que vos gabarits se comportent correctement autant lorsqu’aucune expérience n’est en cours que lorsque la variation témoin est affectée.
