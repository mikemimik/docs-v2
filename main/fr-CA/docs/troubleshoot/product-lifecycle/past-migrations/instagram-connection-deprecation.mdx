---
description: Décrit les changements découlant de l’abandon par Instagram de son Authentication API.
title: Abandon de la connexion Instagram
---

Depuis le 31 mars 2020, Facebook a mis hors service les anciennes API d’Instagram au profit d’un nouvel ensemble d’API :

* [Instagram Graph API](https://developers.facebook.com/docs/instagram-api), conçue pour les comptes professionnels sur Instagram, et non pour l’authentification des utilisateurs finaux.
* [Instagram Basic Display API](https://developers.facebook.com/docs/instagram-basic-display-api), une API OAuth2 qui vous permet d’autoriser une application tierce à accéder aux données de base de votre compte Instagram.

Facebook exige que les applications utilisent Facebook Login pour l’authentification. Facebook n’approuvera pas les applications qui utilisent l’Instagram Basic Display API pour l’authentification.

<div id="actions">
  ## Actions
</div>

Pour permettre à vos utilisateurs actuels de continuer à accéder à votre application, vous devrez demander aux utilisateurs qui s&#39;authentifient avec Instagram de s&#39;authentifier autrement, puis utiliser la liaison de comptes pour associer la nouvelle identité à l&#39;ancienne.

Voici un exemple de flux :

* L&#39;utilisateur s&#39;authentifie avec Instagram.
* L&#39;application informe l&#39;utilisateur qu&#39;il ne pourra plus s&#39;authentifier avec Instagram et qu&#39;il devra le faire autrement.
* L&#39;application présente les options d&#39;authentification dont dispose l&#39;utilisateur, par exemple :

  * Facebook
  * Nom d’utilisateur et mot de passe
* Une fois que l&#39;utilisateur s&#39;est authentifié autrement, vous liez les comptes à l&#39;aide de la liaison de comptes.

Même si nous pouvions remplacer notre implémentation actuelle et utiliser le [flux OAuth Instagram Basic Display](https://developers.facebook.com/docs/instagram-basic-display-api/guides/getting-access-tokens-and-permissions), cela ne serait pas accepté par les politiques de Facebook. Vous devriez créer une application Instagram sur Facebook et, dans cette application, une notification indique :

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Notez que Basic Display n&#39;est pas un outil d&#39;authentification. Les données renvoyées par l&#39;API ne peuvent pas être utilisées pour authentifier les utilisateurs de votre application ni pour leur permettre d&#39;ouvrir une session dans votre application. Si votre application utilise des données d&#39;API pour authentifier des utilisateurs, elle sera rejetée lors de l&#39;examen de l&#39;application. Si vous avez besoin d&#39;une solution d&#39;authentification, utilisez plutôt Facebook Login.
</Callout>

Cela signifie que même si Auth0 implémentait ce flux, votre application Instagram ne serait pas approuvée par Facebook.

Si vous devez accéder aux données Instagram, vous devrez authentifier votre utilisateur autrement (par exemple, en utilisant Facebook Login ou un nom d’utilisateur/mot de passe) et implémenter le flux Instagram <Tooltip tip="OAuth 2.0 : framework d’autorisation qui définit les protocoles et workflows d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth">OAuth</Tooltip> dans votre application.
