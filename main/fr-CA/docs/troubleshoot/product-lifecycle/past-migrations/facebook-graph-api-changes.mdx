title: Modifications apportées à Facebook Graph API
---

Depuis le 1er août 2018, Facebook a modifié les autorisations et les champs de Facebook Graph API qu’il est possible de demander. Auth0 a mis à jour les connexions Facebook pour tenir compte de ces changements et a clarifié l’interface de la connexion. Consultez le [journal des modifications de Facebook Login : modifications récentes apportées à Facebook Login](https://developers.facebook.com/docs/facebook-login/changelog#2018-07-02) pour tous les détails et les dates clés.

<div id="features-affected">
  ## Fonctionnalités touchées
</div>

Il se peut que la mise à jour n’exige aucune modification de votre code ou de votre configuration, mais votre application pourrait recevoir des données de profil supplémentaires si les autorisations existantes le permettent. Gardez à l’esprit que :

* Si votre connexion Facebook est configurée pour demander l’une des autorisations supprimées, votre <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Access+Token">jeton d’accès</Tooltip> ne les inclura pas dans sa portée.
* Si votre application Facebook est marquée comme &quot;development&quot;, il se peut que vous voyiez encore temporairement une erreur lorsque vous testez la connexion.
* Si vous ajoutez de nouvelles autorisations à la connexion, les utilisateurs finaux devront donner leur consentement la prochaine fois qu’ils se connecteront. Consultez la documentation de Facebook pour savoir comment gérer les cas où des utilisateurs ne disposent pas d’une autorisation précise.

<div id="facebook-login-permissions">
  ### Autorisations de Facebook Login
</div>

Les [autorisations de Facebook Login](https://developers.facebook.com/docs/facebook-login/permissions) sont demandées par votre application lorsqu’un utilisateur se connecte avec Facebook. Si l’utilisateur se connecte pour la première fois ou si les autorisations ont changé, une fenêtre de consentement s’affichera dans Facebook pour présenter les nouvelles autorisations demandées. Une fois ces autorisations accordées, votre application peut alors agir au nom de cet utilisateur à l’aide d’un jeton d’accès Facebook.

L’interface de connexion Facebook a été mise à jour pour afficher à la fois le nom courant et le nom système de toutes les autorisations. Il est ainsi plus facile de trouver les autorisations dont vous avez besoin et de les faire correspondre au code que vous utilisez à l’aide de ces noms d’autorisation.

<div id="permissions-added">
  #### Autorisations ajoutées
</div>

Les autorisations suivantes ont été ajoutées à l’interface de la connexion Facebook :

* business&#95;management
* groups&#95;access&#95;member&#95;info
* leads&#95;retrieval
* pages&#95;manage&#95;instant&#95;articles
* publish&#95;to&#95;groups
* publish&#95;to&#95;groups
* user&#95;age&#95;range
* user&#95;gender
* user&#95;link

<div id="permissions-removed">
  #### Autorisations supprimées
</div>

Les autorisations suivantes ont été supprimées de l’interface de la connexion Facebook :

* read&#95;custom&#95;friendlists
* rsvp&#95;event
* user&#95;about&#95;me
* user&#95;actions-books
* user&#95;actions-fitness
* user&#95;actions-music
* user&#95;actions-news
* user&#95;actions-video
* user&#95;education&#95;history
* user&#95;games&#95;activity
* user&#95;relationship&#95;details
* user&#95;relationships
* user&#95;religion&#95;politics
* user&#95;website
* user&#95;work&#95;history

<div id="permissions-moved-to-deprecated">
  #### Autorisations déplacées vers Deprecated
</div>

Les autorisations suivantes ont été déplacées dans la section **Deprecated** et ne doivent pas être utilisées avec la dernière version de la Graph API :

* publish&#95;actions
* user&#95;managed&#95;groups

<div id="facebook-graph-api-fields">
  ### Champs du Facebook Graph API
</div>

Le [Facebook Graph API](https://developers.facebook.com/docs/graph-api/reference/v3.2/user) est utilisé après la connexion d’un utilisateur pour récupérer les données de profil de l’utilisateur Auth0. Les autorisations d’accès aux données utilisateur demandées déterminent quelles informations sont récupérées à partir du Graph API. Les champs renvoyés dépendent des autorisations demandées et de la présence de ces champs dans le profil Facebook de l’utilisateur.

Ce changement met à niveau le Graph API de la version v2.8 à la version v3.2 et demandera les champs de données utilisateur suivants lors de la connexion :

* address (ajouté)
* age&#95;range
* birthday
* context
* cover
* currency (ajouté)
* devices
* email
* favorite&#95;athletes
* favorite&#95;teams
* first&#95;name
* gender
* hometown
* id
* inspirational&#95;people
* install&#95;type (ajouté)
* installed
* is&#95;verified
* languages
* last&#95;name
* link
* locale
* location
* meeting&#95;for (ajouté)
* middle&#95;name
* name
* name&#95;format
* picture
* public&#95;key (ajouté)
* quotes
* security&#95;settings (ajouté)
* short&#95;name (ajouté)
* significant&#95;other
* sports (ajouté)
* third&#95;party&#95;id
* timezone
* updated&#95;time
* verified
* video&#95;upload&#95;limits (ajouté)
* viewer&#95;can&#95;send&#95;gift (ajouté)
