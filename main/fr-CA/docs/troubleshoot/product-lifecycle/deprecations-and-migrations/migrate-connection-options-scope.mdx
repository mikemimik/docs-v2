title: Migrer vers les portées des options de connexion de l’API de gestion

​Auth0 introduit deux nouvelles [portées](/fr-CA/docs/get-started/apis/scopes) de l’API de gestion pour les options de connexion : `read:connections_options` et `update:connections_options`.

Nous déprécions également la possibilité d’accéder au champ `options` ou de le mettre à jour sans les portées appropriées, ce qui entraîne les changements incompatibles suivants :

* Les réponses des points de terminaison [Get All Connections](/fr-CA/docs/api/management/v2/connections/get-connections) et [Get Connection by Id](/fr-CA/docs/api/management/v2/connections/get-connections-by-id) n’incluent pas le champ `options`, sauf si l’appelant a la portée `read:connections_options`.
* Les requêtes envoyées au point de terminaison [Update a Connection](/fr-CA/docs/api/management/v2/connections/patch-connections-by-id) échouent si le corps de la requête contient le champ `options` et que l’appelant n’a pas la portée `update:connections_options`. Les réponses réussies n’incluent pas le champ `options`, sauf si l’appelant a la portée `read:connections_options`.

Pour continuer à utiliser le champ `options`, vous devez mettre à jour les autorisations de vos applications afin d’inclure les nouvelles portées. Suivez les étapes ci-dessous pour déterminer si votre tenant et vos intégrations sont touchés, migrer vers les portées dédiées aux options et, enfin, ne plus utiliser le comportement déprécié.

<div id="identify-affected-tenants">
  ## Identifier les tenants concernés
</div>

Utilisez Auth0 Dashboard pour vérifier si un tenant est autorisé à accéder aux options de connexion sans les nouvelles portées et doit être migré.

1. Accédez à [Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Faites défiler la page jusqu’à la section **Migrations**.
3. Repérez la bascule **Allow Connections Management without Options Scopes** :
   A. Si la bascule est **activée**, votre tenant peut toujours accéder aux options de connexion sans les nouvelles portées, et vous devez effectuer la migration avant la date limite.
   B. Si la bascule est absente ou **désactivée**, aucune autre action n’est requise.

<div id="identify-affected-integrations">
  ## Identifier les intégrations touchées
</div>

Examinez les journaux du tenant `depnote` pour déterminer quelles applications et intégrations utilisent les points de terminaison concernés. L’exemple de requête suivant montre comment rechercher les journaux du tenant associés à l’avis de dépréciation propre à cette dépréciation :

```
type:depnote AND description:Allow\ Connections\ Management\ without\ Options\ Scopes*
```

Pour en savoir plus, consultez [Syntaxe des requêtes de recherche dans les journaux](docs/deploy-monitor/logs/log-search-query-syntax).

Une fois que vous avez les résultats de la requête, vérifiez le champ `client_id` pour identifier chaque application susceptible d’être touchée. Étant donné la nature des changements, la présence d’un avis de dépréciation pour une application cliente précise ne signifie pas nécessairement que l’application est touchée.

​Si le champ `client_id` est vide, cela signifie que la requête vers le point de terminaison touché provient d’une extension. Plusieurs extensions Auth0 peuvent générer dans les journaux du tenant des avis de dépréciation qui sont de faux positifs. Une fois les étapes propres aux extensions de la section Mettre à jour les extensions terminées, vous pouvez ignorer les avis de dépréciation dont l’identifiant client est vide.

<div id="update-application-grants">
  ## Mettre à jour les autorisations d’application
</div>

Pour les applications qui récupèrent ou mettent à jour le champ `options` à l’aide des points de terminaison concernés, mettez à jour leurs autorisations d’accès à l’API de gestion pour inclure les portées nécessaires.

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/dashboard/#/apis).
2. Sélectionnez **Auth0 API de gestion**, puis l’onglet **Machine to Machine Applications**.
3. À côté de l’application à mettre à jour, sélectionnez l’icône de flèche vers le bas pour afficher la section.
4. Dans la section Permissions, recherchez `“connections_options”`. Activez l’une ou les deux nouvelles portées. Ensuite, sélectionnez **Update** pour enregistrer vos modifications.
   <Frame>![Mettre à jour les permissions de l’API de gestion](/docs/images/cdy7uua7fh8z/update-connections-permissions.png)</Frame>

<div id="update-extensions">
  ## Mettre à jour les extensions
</div>

Mettez à jour l’extension [Delegated Administration Dashboard](/fr-CA/docs/customize/extensions/delegated-administration-extension/install-delegated-admin-extension) vers sa version la plus récente. Cette mise à jour est nécessaire, car l’extension doit avoir accès au champ des options de connexion pour fonctionner correctement. Si vous utilisez cette extension, assurez-vous de la mettre à jour afin d’en préserver le bon fonctionnement.

<div id="opt-out-finalize-migration">
  ## Désactivation (finaliser la migration)
</div>

Une fois que vos intégrations ne dépendent plus du comportement déprécié, nous vous recommandons de configurer votre tenant pour désactiver explicitement la gestion des connexions sans `options scopes`. Vous pouvez ainsi choisir exactement quand votre tenant adoptera le nouveau comportement, ce qui vous donne davantage de contrôle sur votre processus de migration.

1. Accédez à [Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Dans la section Migrations, désactivez la bascule **Allow Connections Management without Options Scopes**.

Si vous rencontrez des problèmes, vous pouvez réactiver temporairement le comportement précédent en rétablissant cette même bascule pendant que vous corrigez le problème.
