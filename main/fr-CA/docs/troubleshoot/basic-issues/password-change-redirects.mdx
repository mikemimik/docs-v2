---
description: Description de la façon de définir une URI de redirection après un changement de mot de passe
title: Redirections après changement de mot de passe
---

L’URI vers laquelle l’utilisateur est redirigé après une réinitialisation réussie du mot de passe dépend des conditions suivantes :

1. si votre tenant utilise Universal ou Classic Login;
2. si vous générez directement un ticket de changement de mot de passe au moyen de la Management API;
3. ou si vous envoyez un courriel de changement de mot de passe à partir de l’Authentication API, de la page de connexion ou d’une autre source.

Le diagramme de flux suivant vous aide à définir l’URI de redirection de votre tenant selon sa configuration.
