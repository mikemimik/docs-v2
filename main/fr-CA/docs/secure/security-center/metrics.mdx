Security Center utilise les événements du journal du tenant pour repérer des tendances qui indiquent habituellement des types d’attaques connus. Nous classons les tendances des événements du journal du tenant dans les catégories suivantes : menaces de bourrage d’identifiants, menaces d’attaques à l’inscription et menaces de contournement de l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=MFA">MFA</Tooltip>.

<Warning>
  La classification des codes de type d’événement peut changer. Évitez d’implanter des solutions qui reposent sur les définitions actuelles des codes d’événement du journal.
</Warning>

<div id="credential-stuffing">
  ## Bourrage d’identifiants
</div>

Nous détectons les menaces de bourrage d’identifiants dans une même heure à l’aide des codes d’événement suivants :

| Code d’événement | Événement                                                                                           |
| ---------------- | --------------------------------------------------------------------------------------------------- |
| `f`              | Échec de connexion d’un utilisateur                                                                 |
| `fu`             | Échec de connexion d’un utilisateur en raison d’un nom d’utilisateur invalide                       |
| `fp`             | Échec de connexion d’un utilisateur en raison d’un mot de passe invalide                            |
| `pwd_leak`       | Tentative de connexion avec un mot de passe divulgué                                                |
| `limit_wc`       | Adresse IP bloquée pour &gt;10 tentatives de connexion échouées sur un même compte                  |
| `limit_sul`      | Utilisateur bloqué pour &gt;20 connexions par minute à partir de la même adresse IP                 |
| `limit_mu`       | Adresse IP bloquée pour &gt;100 tentatives de connexion échouées ou &gt;50 tentatives d’inscription |

<div id="signup-attack">
  ## Attaque à l’inscription
</div>

Nous identifions les menaces d’attaque à l’inscription au cours d’une même heure à l’aide des codes d’événement suivants :

| Code d’événement | Événement              |
| ---------------- | ---------------------- |
| `fs`             | Échec de l’inscription |

<div id="mfa-bypass">
  ## contournement de l’MFA
</div>

Nous détectons les menaces de contournement de l’MFA sur une période d’une heure à l’aide des codes d’événement suivants :

| Code d’événement                | Événement                            |
| ------------------------------- | ------------------------------------ |
| `gd_send_email`                 | Courriel envoyé                      |
| `gd_send_pn`                    | Notification push envoyée            |
| `gd_send_sms`                   | SMS envoyé                           |
| `gd_send_voice`                 | Appel vocal effectué                 |
| `gd_auth_failed`                | Échec de l’authentification OTP      |
| `gd_auth_rejected`              | Authentification OTP rejetée         |
| `gd_otp_rate_limit_exceed`      | Trop d’échecs d’authentification OTP |
| `gd_recovery_failed`            | Échec de la récupération             |
| `gd_recovery_rate_limit_exceed` | Trop d’échecs de récupération        |
| `gd_webauthn_challenge_failed`. | Échec du navigateur WebAuthn         |
