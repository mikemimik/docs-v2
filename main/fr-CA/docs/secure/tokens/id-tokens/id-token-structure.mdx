title: Structure du jeton d’identité
---

<Tooltip tip="Jeton d’identité : jeton destiné au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=ID+tokens">Les jetons d’identité</Tooltip> suivent la norme [JSON Web Token (JWT)](/fr-CA/docs/secure/tokens/json-web-tokens), ce qui signifie que leur structure de base est conforme à la [structure JWT](/fr-CA/docs/secure/tokens/json-web-tokens/json-web-token-structure) typique et qu’ils contiennent des [claims JWT](/fr-CA/docs/secure/tokens/json-web-tokens/json-web-token-claims) standard concernant le jeton lui-même.

En plus de ce qui est requis pour <Tooltip tip="Jeton d’identité : jeton destiné au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=JWT">JWT</Tooltip>, les jetons d’identité contiennent aussi des claims sur l’utilisateur authentifié, qui sont prédéfinies par le protocole <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker leurs renseignements de connexion." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) et sont donc appelées claims OIDC standard. Voici quelques claims OIDC standard :

* `name`
* `nickname`
* `picture`
* `email`
* `email_verified`

Pour obtenir la liste complète des claims OIDC standard, consultez [OIDC specification: Standard Claims](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). Vous pouvez aussi créer des [claims personnalisées](/fr-CA/docs/secure/tokens/json-web-tokens/json-web-token-claims), c’est-à-dire des claims que vous définissez, contrôlez et ajoutez à un jeton à l’aide d’Auth0 Actions.

Vous contrôlez les claims OIDC incluses dans le jeton d’identité utilisé par votre application en ajoutant des [OpenID Connect Scopes](/fr-CA/docs/get-started/apis/scopes/openid-connect-scopes) précis dans un paramètre lorsque vous demandez des jetons pendant l’authentification des utilisateurs. Pour savoir comment demander un jeton d’identité, consultez [Get ID Tokens](/fr-CA/docs/secure/tokens/id-tokens/get-id-tokens).

<div id="sample-id-token">
  ## Exemple de jeton d’identité
</div>

Cet exemple montre le contenu d’un jeton d’identité. Remarquez que la valeur <Tooltip tip="Audience : identificateur unique de l’audience d’un token émis. Appelée aud dans un token, sa valeur contient l’ID d’une application (Client ID) pour un jeton d’identité ou d’une API (API Identifier) pour un Access Token." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=audience">audience</Tooltip> (située dans la claim `aud`) correspond à l’identificateur de l’application, ce qui signifie que seule cette application précise doit utiliser le token. Pour en savoir plus sur les claims de cet exemple, consultez [JSON Web Token Claims](/fr-CA/docs/secure/tokens/json-web-tokens/json-web-token-claims).

```json lines
{
  "iss": "http://my-domain.auth0.com",
  "sub": "auth0|123456",
  "aud": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "name": "Jane Doe",
  "given_name": "Jane",
  "family_name": "Doe",
  "gender": "female",
  "birthdate": "0000-10-31",
  "email": "janedoe@example.com",
  "picture": "http://example.com/janedoe/me.jpg"
}
```
