<Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Identity+providers">Fournisseurs d’identité</Tooltip> émettent des <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+tokens">jetons d’accès</Tooltip> tiers une fois que les utilisateurs se sont authentifiés auprès de ce fournisseur. Vous pouvez utiliser ces jetons d’accès pour appeler l’API du fournisseur tiers qui les a émis. Par exemple, vous pouvez utiliser un jeton d’accès émis après qu’un utilisateur s’est authentifié avec Facebook pour appeler la Facebook Graph API.

Pour les utilisateurs individuels, les jetons du fournisseur d’identité sont disponibles dans le tableau `identities` de l’objet `user`, sous l’élément correspondant à la connexion en question. Pour accéder de façon sécuritaire aux jetons d’un utilisateur précis, vous avez besoin d’un [jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) qui comprend la portée `read:user_idp_tokens`. Vous pouvez ensuite effectuer une requête HTTP GET au [point de terminaison Get a User](https://auth0.com/docs/api/management/v2/#!/Users/get_users_by_id) pour récupérer les jetons.

Pour les utilisateurs Admin des connexions d’entreprise configurées à l’aide d’un flux de consentement <Tooltip tip="OAuth 2.0 : framework d’autorisation qui définit les protocoles et workflows d’autorisation." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>, les jetons du fournisseur d’identité sont disponibles dans l’objet `connection`. Pour accéder de façon sécuritaire aux jetons d’une connexion, vous avez besoin d’un [jeton d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) qui comprend la portée `read:connections`. Vous pouvez ensuite effectuer une requête HTTP GET au [point de terminaison Get a Connection](https://auth0.com/docs/api/management/v2/connections/get-connections-by-id) pour récupérer les jetons.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le contenu des jetons d’accès tiers varie selon le fournisseur d’identité qui les émet. Comme ces jetons sont créés et gérés par un tiers (comme Facebook, GitHub, etc.), leur période de validité varie selon l’émetteur. Si vous croyez que ces jetons ont été compromis, vous devez les révoquer ou les réinitialiser auprès du tiers qui les a émis.
</Callout>

<div id="renew-third-party-tokens">
  ## Actualiser les jetons tiers
</div>

Il n’existe pas de méthode normalisée pour actualiser les jetons d’accès du fournisseur d’identité par l’intermédiaire d’Auth0. Le mécanisme d’actualisation des jetons d’accès du fournisseur d’identité varie selon le fournisseur. Pour certains fournisseurs d’identité, Auth0 peut stocker un [jeton d’actualisation](/docs/fr-CA/secure/tokens/refresh-tokens), que vous pouvez utiliser pour obtenir un nouveau jeton d’accès auprès du fournisseur d’identité. Voici une liste de quelques fournisseurs d’identité :

* BitBucket
* Google OAuth 2.0 (transmettez le paramètre `access_type=offline`, ainsi que le paramètre `connection_scope` avec les portées requises, lors de l’appel du point de terminaison Auth0 `/authorize`)
* Tout autre fournisseur d’identité OAuth 2.0
* SharePoint
* Azure AD

Vous pouvez récupérer des <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un jeton d’accès renouvelé sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> de la même manière que vous récupérez des jetons d’accès. Consultez les directives ci-dessus pour en savoir plus.

<div id="validate-third-party-tokens">
  ## Valider les jetons de tiers
</div>

En général, si vous avez reçu un jeton d’accès d’un fournisseur d’identité, vous n’avez pas besoin de le valider. Vous pouvez le transmettre à l’émetteur, qui s’occupe du reste.
