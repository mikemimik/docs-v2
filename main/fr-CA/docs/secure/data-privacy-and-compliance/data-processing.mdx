Ce document présente les données qu’Auth0 possède et explique comment elles sont traitées.

<div id="data-auth0-possesses">
  ## Données détenues par Auth0
</div>

Toutes les données qu’Auth0 possède sur un utilisateur final se trouvent dans le profil utilisateur Auth0. Les attributs précis contenus dans le profil utilisateur varient selon l’implémentation du client et dépendent de plusieurs facteurs, comme le type de connexion, le consentement de l’utilisateur durant le flux d’authentification, et le fait que vous ayez enrichi les profils utilisateur avec des renseignements supplémentaires.

<Warning>
  Les métadonnées Auth0 ne constituent pas un stockage de données sécurisé et ne doivent pas être utilisées pour stocker des renseignements sensibles, comme des secrets à risque élevé et des renseignements personnels identifiables (PII), tels que des numéros d’assurance sociale ou de carte de crédit. Auth0 recommande fortement à ses clients d’évaluer les données stockées dans les métadonnées et de n’y conserver que ce qui est nécessaire aux fins de la gestion des identités et des accès.
</Warning>

<div id="when-data-is-stored">
  ## Quand les données sont stockées
</div>

Les informations du profil utilisateur Auth0 sont stockées dans Auth0 lorsque vous utilisez une connexion de base de données. Si un utilisateur se connecte avec tout autre type de connexion (y compris des connexions de base de données personnalisées), Auth0 stocke les informations fournies par le <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=identity+provider">fournisseur d’identité</Tooltip> externe pour des requêtes ultérieures.

<div id="how-data-is-used">
  ## Comment les données sont utilisées
</div>

Les données personnelles stockées dans Auth0 servent uniquement à fournir ses services, notamment l’authentification des utilisateurs

<div id="how-can-i-delete-my-data">
  ## Comment puis-je supprimer mes données ?
</div>

En tant que responsable du traitement des données, le client est chargé de gérer et de supprimer les données au moyen des fonctionnalités en libre-service, notamment :

* [Supprimer ou réinitialiser des tenants](/docs/fr-CA/troubleshoot/customer-support/manage-subscriptions/delete-or-reset-tenant) - Utilisez cette fonctionnalité pour supprimer un tenant. Lorsque vous supprimez un tenant, le système supprime définitivement toutes les données du client associées.
* [Gérer les utilisateurs à l’aide de la Management API](/docs/fr-CA/manage-users/user-accounts/manage-users-using-the-management-api) - Utilisez l’API pour récupérer, créer, mettre à jour ou supprimer des utilisateurs avec la Management API.
* [Gérer les utilisateurs dans le Dashboard](/docs/fr-CA/manage-users/user-accounts/manage-users-using-the-dashboard) - Utilisez le Dashboard pour créer, consulter, modifier ou supprimer manuellement des utilisateurs.

<div id="when-is-customer-data-automatically-deleted">
  ## Quand les données du client sont-elles automatiquement supprimées?
</div>

Auth0 conserve les données du client tant que celui-ci est considéré comme actif.

Auth0 considère un client comme inactif lorsque les deux conditions suivantes sont remplies :

1. Le client utilise un Free Plan (y compris les clients qui ont choisi le Free Tier ou qui y sont passés).

2. Tous les tenants associés n’affichent aucune activité pendant une période continue de 150 jours.

Après 150 jours d’inactivité, le système planifie la suppression définitive des tenants du client inactif et de toutes les données associées.
