title: Configurer les notifications push pour MFA
---

Les SDKs [Guardian](/fr-CA/docs/secure/multi-factor-authentication/auth0-guardian) pour [iOS](/fr-CA/docs/secure/multi-factor-authentication/auth0-guardian/guardian-for-ios-sdk) et [Android](/fr-CA/docs/secure/multi-factor-authentication/auth0-guardian/guardian-for-android-sdk) vous permettent d’utiliser une application personnalisée pour les services de notifications push propres à chaque fournisseur. Vous pouvez configurer les notifications push pour la plateforme AWS Simple Notification Service (SNS) ou utiliser les services Direct to Vendor pour ajouter directement dans Auth0 les identifiants Firebase Cloud Messaging (FCM) et Apple Push Notification (APN).

<div id="aws-sns-configuration">
  ## Configuration d’AWS SNS
</div>

Avant de configurer les notifications push, suivez les prérequis ci-dessous :

1. [Créez une SNS Platform Application](https://console.aws.amazon.com/sns/v3/home?region=us-east-1#/mobile/push-notifications/platform-applications) à l’aide de la console de gestion AWS et notez son ARN.
2. Créez un AWS Access Key ID autorisé à créer des points de terminaison de Platform Application. Guardian crée automatiquement un point de terminaison de Platform Application avec le jeton d’appareil approprié dans le cadre d’un enrôlement réussi.

<div id="configure-push-notifications-with-aws-sns">
  ### Configurer les notifications push avec AWS SNS
</div>

Pour recevoir des notifications push de Guardian, vous devez d’abord remplacer les paramètres SNS par défaut de Guardian par les vôtres. Vous pourrez ensuite configurer les notifications push pour Android et iOS.

1. Dans l’Auth0 Dashboard, accédez à [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa) et sélectionnez **Push via Auth0 Guardian**. Si la méthode est inactive, utilisez le bouton bascule en haut à droite pour l’activer.
2. Sélectionnez **Custom** pour afficher des paramètres supplémentaires.
3. Dans la section Push Notification Service, choisissez **Amazon SNS.**

<Frame>![Auth0 Dashboard > Security > Multi-factor Auth > Guardian > Custom > Amazon SNS](/docs/images/cdy7uua7fh8z/6PKSyH99GJ2CalJoG38sfZ/fbe7229dcc6f7d303a582520001b3dc1/Amazon_SNS.png)</Frame>

1. Dans la section Amazon SNS Configuration, remplissez les champs suivants :

   | Champ                 | Description                         |
   | --------------------- | ----------------------------------- |
   | AWS Access Key ID     | Votre AWS Access Key ID.            |
   | AWS Secret Access Key | Votre AWS Secret Access Key.        |
   | AWS Region            | La région de votre application AWS. |
2. Sélectionnez **Save** pour enregistrer vos modifications.

<div id="configure-push-notifications-for-android-using-fcm">
  ## Configurer les notifications push pour Android à l’aide de FCM
</div>

<Tabs>
  <Tab title="Obtenir les identifiants FCM">
    Pour configurer les notifications push Android dans votre Auth0 Dashboard, commencez par récupérer les identifiants de votre serveur FCM. Pour en savoir plus, consultez la [documentation Firebase](https://firebase.google.com/docs/admin/setup#initialize-sdk) de Google.

    1. Accédez à la console Firebase et sélectionnez le projet approprié.
    2. Ouvrez le menu **Settings** et choisissez **Service Accounts**.
    3. Sélectionnez **Generate New Private Key** pour télécharger votre fichier d’identifiants.

    Une fois vos identifiants récupérés, vous pouvez configurer les notifications push dans votre Auth0 Dashboard.
  </Tab>

  <Tab title="Configurer Auth0 Dashboard">
    Après avoir récupéré vos identifiants, vous pouvez configurer les notifications push Android depuis votre Auth0 Dashboard.

    1. Accédez à [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian) et sélectionnez **Push Notification using Auth0 Guardian**.
    2. Dans la section **Push Notification App**, sélectionnez **Custom** pour afficher des paramètres supplémentaires.
    3. Dans la section de configuration de l’application Android, configurez les éléments suivants :

       | Paramètre                     | Description                                                                                                                                                                |
       | ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
       | Activer l’application Android | Permet à Auth0 d’envoyer des notifications push à votre application personnalisée. Si cette option n’est pas activée, les notifications push ne peuvent pas être envoyées. |
       | URL du Google Play Store      | Fournissez un lien vers l’application personnalisée que les utilisateurs doivent télécharger pour configurer MFA.                              |
       | Identifiants du serveur FCM   | Téléversez vos [identifiants du serveur FCM](https://firebase.google.com/docs/admin/setup#initialize-sdk).                                                                 |
    4. Sélectionnez **Save** pour enregistrer vos modifications.
  </Tab>
</Tabs>

<div id="configure-push-notifications-for-ios-using-apns">
  ## Configurer les notifications push pour iOS à l’aide d’APNs
</div>

<Tabs>
  <Tab title="Obtenir le certificat APN">
    Pour configurer les notifications push iOS dans votre Auth0 Dashboard, commencez par récupérer votre certificat APN.

    1. Générez un [nouveau certificat APN](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns#2947597) à partir de votre compte développeur Apple.

    2. Installez le certificat localement.

    3. Repérez le certificat **Apple Sandbox Push Services: {AppId}** dans vos Trousseaux d’accès.

    4. Exportez le certificat et la clé privée associée dans un fichier P12 sans mot de passe (nom par défaut : `Certificates.p12`).

    5. Pour générer un fichier P12 avec Triple DES plutôt qu’avec RC2, exécutez

       ```bash lines
       openssl pkcs12 -in Certificates.p12 -legacy -nocerts -nodes -out pk.pem -passin pass:"" &&
       openssl pkcs12 -in Certificates.p12 -legacy -nokeys -out cert.crt -passin pass:"" &&
       openssl pkcs12 -export -inkey pk.pem -in cert.crt -descert -out Certificate_3des.p12 -passout pass:"" &&
       rm pk.pem cert.crt
       ```

    6. Supprimez `Certificates.p12`, puis utilisez `Certificate_3des.p12` comme certificat APN dans l’Auth0 Dashboard.

    Une fois vos identifiants récupérés, vous pouvez configurer les notifications push dans votre Auth0 Dashboard.
  </Tab>

  <Tab title="Configurer l’Auth0 Dashboard">
    Une fois vos identifiants récupérés, vous pouvez configurer les notifications push iOS à partir de votre Auth0 Dashboard.

    1. Accédez à [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian) et sélectionnez **Push Notification using Auth0 Guardian**.
    2. Dans la section Push Notification App, sélectionnez **Custom** pour afficher des paramètres supplémentaires.
    3. Dans la section de configuration de l’application iOS, configurez les éléments suivants :

       | Champ               | Description                                                                                                                                                                                 |
       | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
       | Enable iOS App      | Permet à Auth0 d’envoyer des notifications push à votre application personnalisée. Si cette option n’est pas activée, les notifications push ne peuvent pas être envoyées.                  |
       | Apple App Store URL | Fournissez un lien vers l’application personnalisée que les utilisateurs doivent télécharger pour configurer la MFA.                                                                        |
       | APNs Bundle ID      | Fourni par la console de développement iOS.                                                                                                                                                 |
       | APNs Certificate    | Téléversez votre [certificat P12](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns). |
       | iOS App Environment | Définissez cette valeur selon la configuration de votre compte développeur Apple.                                                                                                           |
    4. Sélectionnez **Save** pour enregistrer vos modifications.
  </Tab>
</Tabs>
