Vous pouvez activer l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> (MFA) pour votre tenant dans l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

Lorsque vous activez la MFA, sélectionnez les facteurs (comme les notifications push ou les messages envoyés au téléphone) que vous souhaitez rendre accessibles aux utilisateurs et définissez une politique pour déterminer à quel moment vous exigez qu’ils s’authentifient.

<div id="select-factors">
  ## Sélectionner les facteurs
</div>

Auth0 prend en charge divers facteurs que vous pouvez activer pour permettre à vos utilisateurs de terminer leur authentification.

<div id="independent-factors">
  ### Facteurs indépendants
</div>

Vous devez activer et configurer au moins un facteur indépendant pour exiger l’authentification multifacteur (MFA) pour les utilisateurs de votre tenant. Vous pouvez activer simultanément plusieurs facteurs indépendants, selon vos besoins.

Les facteurs indépendants offerts sont :

* [WebAuthn avec FIDO Security Keys](/fr-CA/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)
* [Mot de passe à usage unique (OTP)](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)
* [Notification push avec Auth0 Guardian](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa)
* [Message téléphonique](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)
* [Cisco Duo Security](/fr-CA/docs/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="dependent-factors">
  ### Facteurs dépendants
</div>

Vous pouvez activer un ou plusieurs facteurs dépendants après avoir déjà activé et configuré au moins un facteur indépendant pour les utilisateurs de votre tenant.

Les facteurs dépendants offerts sont :

* [WebAuthn avec données biométriques FIDO](/fr-CA/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)
* [Email](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)
* [Code de récupération](/fr-CA/docs/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="enable-mfa-in-the-auth0-dashboard">
  ## Activer la MFA dans l’Auth0 Dashboard
</div>

1. Accédez à [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
2. Dans la section **Factors**, activez et configurez les facteurs que vous souhaitez rendre accessibles aux utilisateurs.
3. Dans la section **Define policies**, sélectionnez une politique pour déterminer à quel moment les utilisateurs devront utiliser la MFA.

   * **Never** : la MFA n’est requise pour aucune connexion.
   * **Use Adaptive MFA** : la MFA est requise selon l’évaluation des risques d’Auth0. Pour en savoir plus, consultez [Enable Adaptive MFA](/fr-CA/docs/secure/multi-factor-authentication/adaptive-mfa).
   * **Always** : la MFA est requise pour toutes les connexions.
4. Si vous choisissez la politique **Never** ou **Always**, la section **MFA Risk Assessors** s’affiche. Par défaut, le paramètre **Enable Adaptive MFA Risk Assessment** est désactivé. Activez la bascule pour évaluer et consigner le risque pour toutes les transactions de connexion dans les journaux de votre tenant.
5. Dans la section **Additional Settings**, configurez les options suivantes au besoin :

   * **Show Multi-factor Authentication options:** Activez cette bascule pour permettre aux utilisateurs de sélectionner des facteurs d’authentification lors de l’inscription. Lorsqu’elle est désactivée, Auth0 présente automatiquement aux utilisateurs les facteurs d’authentification les plus sécuritaires au moment de l’inscription.

     <Frame>![Vue de l’utilisateur final de Show Multi-factor Authentication options](/docs/images/cdy7uua7fh8z/3Mo3EXUk3ZFK4hZKck0yOO/37268f33f70d1e09032f0db317ac11d3/MFA_Options_-_English.png)</Frame>
   * **Customize MFA Factors using Actions** : Activez cette bascule pour personnaliser vos flux MFA à l’aide des [Actions](/fr-CA/docs/customize/actions) `post-login`. Cela vous permet d’imposer aux utilisateurs un facteur précis ou une séquence de facteurs, ainsi que d’utiliser des renseignements contextuels pour créer des expériences plus personnalisées. Pour en savoir plus, consultez [Customize MFA Selection for Universal Login](/fr-CA/docs/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
6. Cliquez sur **Save**.

<div id="customize-mfa-flow">
  ## Personnaliser le flux MFA
</div>

Vous pouvez utiliser Auth0 Actions pour personnaliser le flux MFA pour les utilisateurs de votre tenant. Par exemple, vous pouvez créer une Action pour imposer la MFA dans certaines circonstances ou pour forcer l’utilisation d’un facteur précis.

Pour en savoir plus, consultez [Personnaliser l’Adaptive MFA](/fr-CA/docs/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).
