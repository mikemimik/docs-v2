<Tooltip tip="Suspicious IP Throttling : forme de protection contre les attaques qui protège votre tenant contre les tentatives de connexion suspectes visant un trop grand nombre de comptes à partir d’une même adresse IP." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Suspicious+IP+Throttling">Suspicious IP Throttling</Tooltip> bloque le trafic provenant de toute adresse IP qui effectue rapidement un trop grand nombre de tentatives de connexion ou d’inscription. Cela contribue à protéger vos applications contre les attaques à haut débit qui ciblent plusieurs comptes.

Suspicious IP Throttling est activé par défaut lorsque vous créez votre tenant Auth0. Lorsqu’Auth0 détecte un nombre élevé de tentatives d’inscription ou de tentatives de connexion échouées à partir d’une adresse IP, il renvoie, pour les tentatives suivantes, le [code d’état HTTP 429 Too Many Requests](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/429) jusqu’à ce que cette adresse IP ne soit plus soumise à une limitation.

Pour en savoir plus sur les tentatives d’inscription et les tentatives de connexion échouées consignées dans les logs de votre tenant, consultez [Log Type Codes](/fr-CA/docs/deploy-monitor/logs/log-event-type-codes).

<div id="configure-suspicious-ip-throttling">
  ## Configurer Suspicious IP Throttling
</div>

Vous pouvez configurer Suspicious IP Throttling dans l’[Auth0 Dashboard](https://manage.auth0.com/#) ou à l’aide de l’[Auth0 Management API](https://auth0.com/docs/api/management/v2/attack-protection/patch-suspicious-ip-throttling).

<Warning>
  Auth0 recommande fortement de ne pas désactiver Suspicious IP Throttling.
</Warning>

1. Accédez à [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling**.
2. Sélectionnez la bascule en haut de la page pour activer ou désactiver Suspicious IP Throttling.

<div id="allow-trusted-ip-addresses-exceed-throttling-limits">
  ## Autoriser les adresses IP de confiance à dépasser les limites de limitation
</div>

Vous pouvez exempter jusqu’à 100 adresses IP distinctes et/ou plages CIDR (IPv4 ou IPv6) de Suspicious IP Throttling en les ajoutant à l’**IP AllowList**. Auth0 ne bloque pas les administrateurs du tenant et ne leur envoie pas d’alerte lorsque ces adresses IP dépassent les limites de limitation.

1. Accédez à [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling**.
2. Dans le champ **IP AllowList**, entrez les adresses IP et/ou les plages CIDR que vous souhaitez autoriser à effectuer un nombre illimité de tentatives de connexion et d’inscription. Séparez plusieurs adresses IP et/ou plages CIDR par des virgules.

<div id="configure-response">
  ## Configurer la réponse
</div>

Par défaut, Auth0 envoie un courriel aux administrateurs du tenant lorsqu&#39;une adresse IP est considérée comme suspecte.

Vous pouvez configurer la façon dont Auth0 réagit aux tentatives de connexion ou d&#39;inscription à cadence élevée.

<Warning>
  Si vous activez les fonctionnalités de protection contre les attaques sans activer de paramètres de réponse, le mode de surveillance sera activé et seuls les événements connexes seront consignés dans le journal du tenant. Pour en savoir plus, consultez [Afficher les événements du journal de la protection contre les attaques](/fr-CA/docs/secure/attack-protection/view-attack-protection-events).
</Warning>

1. Accédez à [Dashboard &gt; Sécurité &gt; Protection contre les attaques](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling.**
2. Repérez la section **Réponse**.
3. Dans la section **Paramètres de blocage**, activez **Limiter le trafic à cadence élevée ciblant trop de comptes** pour limiter le trafic provenant d&#39;adresses IP qui dépassent le seuil de connexion ou d&#39;inscription.
4. Dans la section **Notifications**, activez **Envoyer une notification à l&#39;administrateur du compte** pour qu&#39;un courriel soit automatiquement envoyé aux administrateurs du tenant lorsqu&#39;une adresse IP dépasse le seuil de connexion ou d&#39;inscription.
5. Sélectionnez **Enregistrer**.

<div id="configure-throttling-limits-and-rates">
  ## Configurer les limites et les taux de limitation
</div>

Vous pouvez personnaliser la façon dont Auth0 limite les adresses IP suspectes. Vous pouvez modifier :

* Le nombre maximal de tentatives de connexion et d’inscription échouées autorisées à partir d’une adresse IP donnée.
* Le taux auquel les adresses IP limitées autorisent de nouvelles tentatives de connexion et d’inscription.

Pour en savoir plus sur le nombre maximal de tentatives, le taux de limitation et leur fonctionnement, consultez [Fonctionnement de Suspicious IP Throttling](#how-suspicious-ip-throttling-works).

1. Accédez à [Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/#/security/attack-protection), puis sélectionnez **Suspicious IP Throttling.**

   <Frame>![Personnalisation de Suspicious IP Throttling dans le Dashboard Auth0](/docs/images/cdy7uua7fh8z/5yLYmDn0FvDB061WgTH2fk/0f3dce552a41aa5756a2eca788944965/Screen_Shot_2021-09-29_at_10.19.32_AM.png)</Frame>
2. Repérez la section **Detection**.
3. Sélectionnez **Custom** pour **Suspicious IP Thresholds.**
4. Configurez les paramètres de **Login Threshold** :

   * **Maximum Attempts** : Saisissez le nombre de tentatives de connexion échouées qu’une même adresse IP peut effectuer en une journée avant qu’Auth0 bloque la tentative suivante.
   * **Throttling Rate** : Saisissez le taux auquel de nouveaux jetons de connexion sont accordés.
5. Configurez les paramètres de **Signup Threshold** :

   * **Maximum Attempts** : Saisissez le nombre de tentatives d’inscription qu’une même adresse IP peut effectuer en une minute avant qu’Auth0 bloque la tentative suivante.
   * **Throttling Rate** : Saisissez le taux auquel de nouveaux jetons d’inscription sont accordés.
6. Sélectionnez **Save**.

<div id="how-suspicious-ip-throttling-works">
  ### Fonctionnement de la Suspicious IP Throttling
</div>

Auth0 comptabilise et autorise séparément les tentatives de connexion et d&#39;inscription. Les adresses IP suspendues pour toute tentative de connexion supplémentaire peuvent quand même essayer de s&#39;inscrire. Les adresses IP suspendues pour toute tentative d&#39;inscription supplémentaire peuvent quand même essayer de se connecter.

<div id="login-attempts">
  #### Tentatives de connexion
</div>

Auth0 applique une limitation à toute adresse IP qui effectue un trop grand nombre de tentatives de connexion infructueuses en une journée. Le taux de limitation des connexions détermine le nombre de tentatives de connexion qu’Auth0 accorde à une adresse IP, réparties uniformément sur 24 heures. Par exemple, un taux de limitation de 100 signifie qu’Auth0 accorde une nouvelle tentative environ toutes les 15 minutes.

<div id="signup-attempts">
  #### Tentatives d’inscription
</div>

Auth0 applique une limitation à toute adresse IP qui effectue un trop grand nombre de tentatives d’inscription en une minute. Par exemple, si une adresse IP effectue 50 tentatives d’inscription, Auth0 bloque les tentatives suivantes. Contrairement aux tentatives de connexion, les tentatives d’inscription n’ont pas besoin d’échouer.

Le taux de limitation des inscriptions détermine le nombre de tentatives d’inscription qu’Auth0 accorde à une adresse IP, réparties uniformément sur 24 heures. Par exemple, un taux de limitation de 72 000 signifie qu’Auth0 accorde une nouvelle tentative environ toutes les secondes.

<div id="malformed-requests-and-schema-validation-errors">
  #### Requêtes mal formées et erreurs de validation du schéma
</div>

Les requêtes de connexion et d’inscription qui ne respectent pas la structure ou le format requis ne sont pas considérées comme des tentatives valides et ne comptent donc pas dans le seuil configuré.

Pour en savoir plus sur les requêtes de connexion et d’inscription, consultez [Auth0 Authentication API](https://auth0.com/docs/api/authentication).

<div id="special-cases">
  ## Cas particuliers
</div>

Comme la Suspicious IP Throttling dépend de l’adresse IP de l’utilisateur, les cas d’utilisation suivants nécessitent une configuration supplémentaire :

* **Utilisation du <Tooltip tip="Resource Owner : entité (comme un utilisateur ou une application) capable d’accorder l’accès à une ressource protégée." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Resource+Owner">Resource Owner</Tooltip> Password Grant depuis le backend d’une application** : cet appel ne permet pas d’obtenir l’adresse IP de l’utilisateur; toutefois, pour que la Suspicious IP Throttling fonctionne correctement, vous pouvez configurer votre application afin d’envoyer l’adresse IP de l’utilisateur dans la requête. Consultez [Éviter les problèmes courants avec le Resource Owner Password Flow et la protection contre les attaques : envoyer l’adresse IP de l’utilisateur depuis votre serveur](/fr-CA/docs/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
* **Authentification d’un grand nombre d’utilisateurs à partir de la même adresse IP** : les utilisateurs qui passent par un proxy sont plus susceptibles d’atteindre les limites définies et de déclencher la limitation du débit. Vous pouvez éviter de déclencher cette limitation par erreur en configurant une AllowList pour l’adresse IP du proxy et sa plage CIDR. Consultez [Éviter les problèmes courants avec le Resource Owner Password Flow et la protection contre les attaques : configurer votre application pour faire confiance à l’adresse IP](/fr-CA/docs/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
