---
title: Comment activer ou désactiver le mode de surveillance pour les règles de la Tenant Access Control List
sidebarTitle: Activer le mode de surveillance
description: Vérifiez l’effet qu’une règle aurait sur le trafic entrant sans modifier le comportement actuel de la Tenant ACL en activant le mode de surveillance.
---

Lorsqu’une règle est en mode de surveillance, la Tenant ACL [évalue cette règle](./rule-evaluation) et génère un [événement du journal du tenant](./tenant-log-event-reference), mais n’exécute pas l’action de la règle et n’interrompt pas l’évaluation des règles et des listes suivantes.

Pour activer le mode de surveillance, définissez la propriété `log` de l’action de la règle sur `true` :

<Tabs>
  <Tab title="Management API">
    Appelez le [point de terminaison Partial update for an access control list de la Management API](/fr-CA/docs/api/management/v2/network-acls/patch-network-acls-by-id) avec le corps suivant :

    ```json Management API request body highlight={4}
    {
      "rule": {
        "action": {
          "log": true
        },
        "scope": "authentication"
      }
    }
    ```
  </Tab>

  <Tab title="Go SDK">
    ```go highlight={20}
    package main

    import (
    	"context"
    	"log"

    	"github.com/auth0/go-auth0"
    	"github.com/auth0/go-auth0/management"
    )

    func main() {
    	mgmt, err := management.New("{yourDomain}", management.WithClientCredentials("{yourClientId}", "{yourClientSecret}"))
    	if err != nil {
    		log.Fatal(err)
    	}

    	networkACL := &management.NetworkACL{
    		Rule: &management.NetworkACLRule{
    			Action: &management.NetworkACLRuleAction{
    				Log: auth0.Bool(true),
    			},
    			Scope: auth0.String("authentication"),
    		},
    	}

    	err = mgmt.NetworkACL.Patch(context.Background(), "{yourTenantAclRuleId}", networkACL)
    	if err != nil {
    		log.Fatal(err)
    	}
    	log.Println("Network ACL has been updated to enable monitoring mode")
    ```
  </Tab>

  <Tab title="Node SDK">
    ```javascript highlight={4}
    const updateNetworkAclPayload: Management.UpdateNetworkAclRequestContent = {
      rule: {
        action: {
          log: true,
        },
        scope: "authentication"
      }
    };

    const updateNetworkAcl = await client.networkAcls.update("{yourTenantAclRuleId}", updateNetworkAclPayload);
    ```
  </Tab>

  <Tab title="Terraform">
    Dans le bloc de ressource, sous [`rule.action`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/network_acl#nested-schema-for-ruleaction), définissez `log` sur `true` :

    ```terraform highlight={7}
    resource "auth0_network_acl" "example_acl" {
        description = "Example ACL"
        active = true
        priority = 1
        rule {
            action {
                block = true
                log = true
            }
            scope = "authentication"
            match {
              hostnames = ["example.com"]
            }
        }
    }
    ```
  </Tab>

  <Tab title="Deploy CLI">
    ```yaml highlight={8}
    exampleACLs:
      - description: Example ACL
        active: true
        priority: 1
        rule:
          action:
            block: true
            log: true
          match:
            hostnames:
              - "example.com"
          scope: authentication
    ```
  </Tab>

  <Tab title="Auth0 CLI">
    ```bash
    auth0 network-acl update {yourTenantAclRuleId} --action log
    ```
  </Tab>
</Tabs>

Vous pouvez désactiver le mode de surveillance en définissant de la même façon la propriété `log` sur `false`.
