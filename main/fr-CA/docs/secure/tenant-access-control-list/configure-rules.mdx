title: Comment configurer les règles de la Tenant Access Control List
sidebarTitle: Configurer les règles
description: Définissez le signal, la condition, l’action, la portée et la priorité d’une règle pour en définir le comportement. Utilisez la Management API ou les SDK, CLI ou le provider Terraform d’Auth0 pour configurer et gérer la Tenant ACL.
---

Une règle de Tenant ACL se compose des éléments suivants :

* **Signal** : Le signal est un renseignement d’identification fourni par la requête entrante, comme l’adresse IP, la géolocalisation ou l’agent utilisateur.

* **Condition** : La condition est la combinaison d’un opérateur (comme `match`) et d’un ensemble de valeurs (comme une liste d’adresses IP).

* **Action** : L’action est la directive qu’exécute votre règle si les critères sont remplis, par exemple autoriser, bloquer ou rediriger.

* **Scope** : La portée indique pour quels ensembles de points de terminaison la règle s’applique, y compris l’Authentication API, la Management API ou l’ensemble de votre tenant.

* **Priority** : La priorité définit l’ordre dans lequel la règle s’exécute par rapport aux autres règles.

Pour obtenir de l’information détaillée sur les propriétés de ces règles de Tenant ACL et sur la façon de les utiliser, consultez la [référence des paramètres du point de terminaison de la Management API](./reference).

<Note>
  La Tenant ACL utilise une [logique d’évaluation des règles](./rule-evaluation) cohérente pour déterminer, s’il y a lieu, quelle action appliquer à une requête donnée.

  Attribuez les priorités avec soin afin de créer des politiques granulaires de contrôle d’accès pour votre cas d’utilisation.
</Note>

<div id="create-and-delete-rules">
  ## Créer et supprimer des règles
</div>

Vous pouvez créer et supprimer des règles Tenant ACL à l’aide des points de terminaison suivants de la Management API. Vous avez besoin d’un [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les portées indiquées.

| Action              | Point de terminaison                                                                         | Portée requise        |
| ------------------- | -------------------------------------------------------------------------------------------- | --------------------- |
| Créer une règle     | [Create access control list](/fr-CA/docs/api/management/v2/network-acls/post-network-acls)         | `create:network_acls` |
| Supprimer une règle | [Delete access control list](/fr-CA/docs/api/management/v2/network-acls/delete-network-acls-by-id) | `delete:network_acls` |

<div id="modify-rules">
  ## Modifier des règles
</div>

Vous pouvez modifier les règles Tenant ACL à l’aide des points de terminaison Management API suivants. Vous devez disposer d’un [jeton d’accès à la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les portées indiquées.

| Action                  | Point de terminaison                                                                                                   | Portée requise        |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------- | --------------------- |
| Mettre à jour une règle | [Mise à jour partielle d’une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/patch-network-acls-by-id) | `update:network_acls` |
| Remplacer une règle     | [Mise à jour d’une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/put-network-acls-by-id)             | `update:network_acls` |

<div id="enable-or-disable-rules">
  ### Activer ou désactiver des règles
</div>

<Tabs>
  <Tab title="Management API">
    Vous pouvez activer ou désactiver une règle au moyen du point de terminaison [Update access control list](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) de la Management API.

    Définissez la propriété `active` de l’objet `rule` à `true` ou à `false`, respectivement.
  </Tab>

  <Tab title="Dashboard">
    Pour activer ou désactiver une règle à l’aide du Dashboard, accédez à [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control).

    Dans la liste des règles existantes, vous pouvez activer ou désactiver des règles précises à l’aide du bouton à bascule situé à côté de chacune d’elles.

    ![Activer ou désactiver une règle de Tenant ACL dans l’Auth0 Dashboard](/docs/images/cdy7uua7fh8z/29b92e102142d3cb20e69a40e292f6b7/Screenshot_2025-09-03_at_12.34.05.png)

    La colonne **statut** affiche le statut de la règle (**Active** ou **Disabled**).
  </Tab>
</Tabs>

<div id="view-rules">
  ## Afficher les règles
</div>

<Tabs>
  <Tab title="Management API">
    Vous pouvez consulter les règles Tenant ACL existantes à l’aide des endpoints suivants de la Management API. Vous avez besoin d’un [jeton d’accès à la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les portées indiquées.

    | Action                     | Endpoint                                                                                                                              | Portée requise      |
    | -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
    | Afficher une règle         | [Obtenir une entrée précise de liste de contrôle d’accès pour un tenant](/fr-CA/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls` |
    | Afficher toutes les règles | [Obtenir toutes les entrées de liste de contrôle d’accès pour un tenant](/fr-CA/docs/api/management/v2/network-acls/get-network-acls)       | `read:network_acls` |
  </Tab>

  <Tab title="Dashboard">
    Accédez à [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) pour consulter la liste de vos règles existantes.

    | Nom de colonne       | Description                                                                                   | Champ connexe dans la réponse de l’API |
    | -------------------- | --------------------------------------------------------------------------------------------- | -------------------------------------- |
    | Priorité             | Numéro ordinal indiquant à quel moment la règle est évaluée                                   | `priority`                             |
    | Nom                  | Description de l’objectif ou de la fonctionnalité de la règle                                 | `description`                          |
    | Action               | L’[action](/fr-CA/docs/secure/tenant-access-control-list/reference#actions) effectuée par la règle. | `action`                               |
    | Statut               | Valeur indiquant si la règle est active ou désactivée.                                        | `active`                               |
    | Dernière mise à jour | Horodatage indiquant la dernière mise à jour de la règle.                                     | `updated_at`                           |

    ![Liste des règles Tenant ACL dans le Dashboard Auth0](/docs/images/cdy7uua7fh8z/4093f9a39aba48bdf8d579c134dc4da5/Screenshot_2025-09-03_at_12.33.47.png)
  </Tab>
</Tabs>
