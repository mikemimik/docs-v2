    Ce guide explique comment intégrer Auth0 à toute API Python nouvelle ou existante construite avec [Django](https://www.djangoproject.com/).

    Si vous n&#39;avez pas encore créé d&#39;API dans votre Auth0 Dashboard, vous pouvez utiliser le sélecteur interactif pour créer une nouvelle
    API Auth0 ou sélectionner une API existante qui représente le projet que vous souhaitez intégrer.

    Vous pouvez également consulter notre guide de démarrage,
    qui vous aidera à configurer votre première API par l&#39;intermédiaire du Auth0 Dashboard.

    Chaque API dans Auth0 est configurée à l&#39;aide d&#39;un identifiant d&#39;API que le code de votre application utilisera comme audience pour valider le jeton d&#39;accès.

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Les permissions vous permettent de définir comment il est possible d’accéder aux ressources au nom de l’utilisateur à l’aide d’un jeton d’accès donné. Par
      exemple, vous pourriez choisir d’accorder un accès en lecture à la ressource `messages` si les utilisateurs ont le niveau
      d’accès Manager, et d’accorder un accès en écriture à cette ressource s’ils ont le niveau d’accès d’administrateur.

      Vous pouvez définir les permissions autorisées dans la vue **Permissions** de la section [APIs](https://manage.auth0.com/#/apis) de l’Auth0 Dashboard. L’exemple suivant
      utilise le scope `read:messages`.

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Vous utiliserez une bibliothèque appelée [Authlib](https://github.com/lepture/authlib) pour créer un [ResourceProtector](https://docs.authlib.org/en/latest/flask/1/resource-server.html), soit un type de [décorateur de vue Django](https://docs.djangoproject.com/en/4.0/topics/http/decorators/) qui protège vos ressources (vues d’API) à l’aide d’un
      validateur donné.

      Le validateur vérifiera le jeton d’accès que vous transmettez à la ressource en s’assurant qu’il possède une
      signature valide et des claims valides.

      Vous pouvez utiliser le validateur `JWTBearerTokenValidator` d’AuthLib avec quelques ajustements pour vous assurer qu’il respecte
      nos exigences de [validation des jetons d’accès](https://auth0.com/docs/secure/tokens/access-tokens/validate-access-tokens).

      Pour créer votre `Auth0JWTBearerTokenValidator`, vous devez lui fournir votre `domain` et votre
      `audience` (identifiant d’API). Il récupérera ensuite la clé publique requise pour vérifier la signature du jeton
      et la transmettra à la classe `JWTBearerTokenValidator`.

      Vous remplacerez ensuite `claims_options` de la classe pour vous assurer que les claims `expiry`,
      `audience` et `issue` du jeton sont validés conformément à nos exigences.

      Créez le fichier `apiexample/validator.py` à l’aide du code du panneau interactif.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Ensuite, vous créerez trois vues d’API dans `apiexample/views.py` :

      * `/api/public` : un endpoint public qui ne nécessite aucune authentification.
      * `/api/private` : un endpoint privé qui nécessite un JWT de jeton d’accès valide.
      * `/api/private-scoped` : un endpoint privé qui nécessite un JWT de jeton d’accès valide contenant la
        valeur `scope` indiquée.

      Les routes protégées auront un décorateur `require_auth`, qui est un `ResourceProtector`
      utilisant le `Auth0JWTBearerTokenValidator` que vous avez créé plus tôt.

      Pour créer le `Auth0JWTBearerTokenValidator`, vous lui transmettrez le domaine de votre tenant et
      l’identifiant d’API de l’API que vous avez créée plus tôt.

      Le décorateur `require_auth` sur la route `private_scoped` accepte un argument supplémentaire,
      `"read:messages"`, qui vérifie dans le jeton d’accès la permission (scope) que vous avez créée plus tôt.
    </Section>
