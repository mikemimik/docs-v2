<div id="by-jim-anderson">
  ##### Par Jim Anderson
</div>

Ce document vous aidera à résoudre les problèmes liés à votre configuration si votre API renvoie des réponses inattendues. Nous vous recommandons de vous connecter pour suivre ce guide Quickstart avec des exemples configurés pour votre compte.

Si vous avez correctement configuré la validation du JWT, votre API devrait renvoyer les bonnes réponses lorsque vous faites des requêtes. Toutefois, si votre API renvoie une réponse 401 (Non autorisé), c’est que la configuration de votre middleware JWT ne correspond pas au JWT transmis.

Ce document vous aidera à résoudre les problèmes de configuration de votre middleware JWT.

<div id="how-does-a-token-get-validated">
  ## Comment un jeton est-il validé ?
</div>

Lorsqu’il s’agit de valider un JWT, plusieurs éléments sont à prendre en compte :

1. **Le jeton est-il bien formé ?** Autrement dit, respecte-t-il la structure d’un JSON Web Token (JWT) ? Pour en savoir plus sur la structure d’un JWT, consultez [cette section sur la structure d’un JWT](/fr-CA/docs/secure/tokens/json-web-tokens#what-is-the-json-web-token-structure-)
2. **Le jeton a-t-il été altéré ?** La dernière partie d’un JWT est la signature. Elle sert à vérifier que le jeton a bien été signé par l’expéditeur et qu’il n’a subi aucune modification.
3. **Le jeton a-t-il été reçu pendant sa période de validité ?** Les JWT ne sont valides que pour une période précise (comme l’indique le claim `exp`).
4. **Le jeton provient-il bien de l’autorité attendue ?** Cela comporte 2 volets

   * **Vérification de la signature** : Peut-on confirmer que le JWT est correctement signé à l’aide de la clé émise par l’autorité émettrice ?
   * **Valeur de l’émetteur** : L’émetteur est défini dans le claim `iss`. Encore une fois, ce claim correspond-il à ce que votre application attend ?
5. **Le jeton est-il destiné à l’application actuelle ?** Autrement dit, le claim `aud` du JWT correspond-il à ce que votre application attend ?

<div id="inspecting-a-token">
  ## Inspecter un jeton
</div>

Pour inspecter rapidement un JWT, vous pouvez utiliser le site Web [JWT.io](https://jwt.io/). Il propose un débogueur pratique qui vous permet de vérifier rapidement qu’un JWT est bien formé, ainsi que d’examiner les valeurs des différentes claims.

<Frame>![Débogage d’un JWT sur JWT.io](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-rs256.png)</Frame>

Dans la capture d’écran ci-dessus, vous pouvez voir que le jeton a été signé à l’aide de l’algorithme **RS256**. L’**émetteur** du jeton est **https://jerrie.auth0.com/**, et l’**audience** est **https://rs256.test.api**.

Autrement dit, les valeurs indiquées dans votre configuration de validation du JWT doivent correspondre **exactement** — y compris la barre oblique finale de l’émetteur, comme

```
audience = "https://rs256.test.api",
domain = "https://jerrie.auth0.com/"
};
```

Pour un jeton signé avec HS256, la vue du débogueur sera légèrement différente :

Dans la capture d’écran ci-dessus, vous pouvez voir que le jeton a été signé avec l’algorithme **HS256**. L’**émetteur** du jeton est **https://jerrie.auth0.com/**, et l’**audience** est **https://hs256.test.api**.
