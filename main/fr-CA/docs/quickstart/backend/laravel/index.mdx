<div id="by-evan-sims">
  ##### Par Evan Sims
</div>

Le Laravel SDK d’Auth0 vous permet d’ajouter rapidement à votre application Laravel une autorisation fondée sur des jetons et un contrôle d’accès aux routes. Ce guide explique comment intégrer Auth0 à une application Laravel 9 ou 10, nouvelle ou existante. Nous vous recommandons de vous connecter pour suivre ce Quickstart avec des exemples configurés pour votre compte.

{/* <Card title="Voir sur Github" href="https://github.com/auth0-samples/laravel/tree/7.x/sample" icon="github">
  Prérequis : Laravel 9 / 10 | PHP 8.0+ | Composer
  </Card> */}

**Les applications backend se distinguent des applications Web traditionnelles en ce qu’elles ne gèrent pas l’authentification des utilisateurs et n’ont pas d’interface utilisateur. Elles fournissent une API avec laquelle d’autres applications peuvent interagir. Elles acceptent des [jetons d’accès](https://auth0.com/docs/secure/tokens/access-tokens) dans l’en-tête `Authorization` des requêtes afin de contrôler l’accès aux routes.**

Des applications front-end distinctes sont habituellement conçues pour interagir avec ces types de backends. Il peut s’agir d’[applications monopages](https://auth0.com/docs/quickstart/spa) ou d’[applications natives ou mobiles](https://auth0.com/docs/quickstart/native) (pour lesquelles Auth0 fournit aussi des SDKs !)

Lorsque les utilisateurs doivent interagir avec votre application backend, ils s’authentifient d’abord auprès d’Auth0 à l’aide de l’application front-end. L’application front-end récupère ensuite un jeton d’accès auprès d’Auth0, qu’elle peut utiliser pour envoyer des requêtes à votre application backend au nom de l’utilisateur.

Comme leur nom l’indique, les [jetons d’accès](https://auth0.com/docs/secure/tokens/access-tokens) sont conçus pour le contrôle d’accès (autorisation) et ne contiennent pas d’information sur l’utilisateur. **Les applications backend fonctionnent exclusivement avec des jetons d’accès.** Vous pouvez récupérer de l’information sur l’utilisateur qui a créé le jeton à l’aide de la [Management API](https://auth0.com/docs/api/management/v2), que nous présenterons plus loin.

<div id="laravel-installation">
  ## Installation de Laravel
</div>

**Si vous n’avez pas déjà mis en place une application Laravel**, ouvrez un terminal dans le répertoire de votre choix pour un nouveau projet et exécutez la commande suivante :

```bash lines
composer create-project --prefer-dist laravel/laravel auth0-laravel-api ^9.0
```

Toutes les commandes de ce guide supposent que vous les exécutez à partir du répertoire racine de votre projet Laravel; vous devriez donc vous placer dans le répertoire du nouveau projet avec `cd` :

```bash lines
cd auth0-laravel-api
```

<div id="sdk-installation">
  ## Installation du SDK
</div>

Exécutez la commande suivante dans le répertoire de votre projet pour installer le [Auth0 Laravel SDK](https://github.com/auth0/laravel-auth0):

```bash lines
composer require auth0/login:^7.8 --update-with-all-dependencies
```

Ensuite, générez un fichier de configuration SDK pour votre application :

```bash lines
php artisan vendor:publish --tag auth0
```

<div id="sdk-configuration">
  ## Configuration du SDK
</div>

Exécutez la commande suivante depuis le répertoire de votre projet pour télécharger l’[Auth0 CLI](https://github.com/auth0/auth0-cli):

```bash lines
curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
```

Authentifiez-vous ensuite dans le CLI avec votre compte Auth0, en choisissant &quot;en tant qu’utilisateur&quot; lorsqu’on vous le demande :

```bash lines
./auth0 login
```

Ensuite, créez une nouvelle application dans Auth0 :

```bash lines
./auth0 apps create \
  --name "My Laravel Backend" \
  --type "regular" \
  --auth-method "post" \
  --callbacks "http://localhost:8000/callback" \
  --logout-urls "http://localhost:8000" \
  --reveal-secrets \
  --no-input \
  --json > .auth0.app.json
```

Vous devriez également créer une nouvelle API :

```bash lines
./auth0 apis create \
  --name "My Laravel Backend API" \
  --identifier "https://github.com/auth0/laravel-auth0" \
  --offline-access \
  --no-input \
  --json > .auth0.api.json
```

Cela produit deux fichiers dans le répertoire de votre projet qui configurent le SDK.

Comme ces fichiers contiennent des identifiants d’authentification, il est important de les traiter comme des données sensibles. Assurez-vous de ne pas les verser dans votre système de gestion de versions. Si vous utilisez Git, vous devriez les ajouter à votre fichier `.gitignore` :

```bash lines
echo ".auth0.*.json" >> .gitignore
```

<div id="access-control">
  ## Contrôle d’accès
</div>

Vous pouvez utiliser le garde d’autorisation du SDK Auth0 pour restreindre l’accès aux routes de votre application.

Pour rejeter les requêtes qui ne contiennent pas de jeton d’accès valide dans l’en-tête `Authorization`, vous pouvez utiliser le middleware `auth` de Laravel :

```php lines
Route::get('/private', function () {
  return response()->json([
    'message' => 'Your token is valid; you are authorized.',
  ]);
})->middleware('auth');
```

Vous pouvez aussi exiger que le token fourni possède des [permissions](https://auth0.com/docs/manage-users/access-control/rbac) précises en combinant cette option avec le middleware `can` de Laravel :

```php lines
Route::get('/scope', function () {
    return response()->json([
      'message' => 'Your token is valid and has the `read:messages` permission; you are authorized.',
    ]);
})->middleware('auth')->can('read:messages');
```

<div id="token-information">
  ## Informations sur le token
</div>

Vous pouvez obtenir des informations sur le jeton d’accès fourni à l’aide de la façade `Auth` de Laravel ou de la fonction utilitaire `auth()`.

Par exemple, pour récupérer l’identifier et l’adresse courriel de l’utilisateur :

```php lines
Route::get('/', function () {
  if (! auth()->check()) {
    return response()->json([
      'message' => 'You did not provide a valid token.',
    ]);
  }

  return response()->json([
    'message' => 'Your token is valid; you are authorized.',
    'id' => auth()->id(),
    'token' => auth()?->user()?->getAttributes(),
  ]);
});
```

<div id="retrieve-user-information">
  ## Récupérer les renseignements sur l’utilisateur
</div>

Vous pouvez récupérer des renseignements sur l’utilisateur qui a créé le jeton d’accès depuis Auth0 à l’aide de l’[Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). Le SDK fournit un wrapper pratique pour cette API, accessible au moyen de la méthode `management()` du SDK.

**Avant d’effectuer des appels à l’Auth0 Management API, vous devez autoriser votre application à communiquer avec l’Auth0 Management API.** Pour ce faire, accédez à la [page API du tableau de bord Auth0](https://manage.auth0.com/#/apis/), choisissez `Auth0 Management API`, puis sélectionnez l’onglet `Machine to Machine Applications`. Autorisez votre application Laravel, puis cliquez sur la flèche vers le bas pour choisir les portées que vous souhaitez accorder.

Pour l’exemple suivant, vous devriez accorder la portée `read:users`. Vous trouverez une liste des points de terminaison de l’API et des portées requises dans [la documentation de l’Auth0 Management API](https://auth0.com/docs/api/management/v2).

```php lines
use Auth0\Laravel\Facade\Auth0;

Route::get('/me', function () {
  $user = auth()->id();
  $profile = cache()->get($user);

  if (null === $profile) {
    $endpoint = Auth0::management()->users();
    $profile = $endpoint->get($user);
    $profile = Auth0::json($profile);

    cache()->put($user, $profile, 120);
  }

  $name = $profile['name'] ?? 'Unknown';
  $email = $profile['email'] ?? 'Unknown';

  return response()->json([
    'name' => $name,
    'email' => $email,
  ]);
})->middleware('auth');
```

<Info>
  **Vous devriez mettre en cache les renseignements utilisateur dans votre application pendant de courtes périodes.** Cela réduit le nombre de requêtes envoyées par votre application à Auth0 et améliore les performances. Évitez de conserver les renseignements utilisateur dans votre application pendant de longues périodes, car ils risquent de devenir périmés. Évitez également de stocker dans des bases de données persistantes des renseignements utilisateur autres que l’identifiant de l’utilisateur.
</Info>

<div id="run-the-application">
  ## Lancer l’application
</div>

Vous êtes maintenant prêt à démarrer votre application Laravel pour qu’elle puisse accepter des requêtes :

```bash lines
php artisan serve
```

<div id="retrieve-a-test-token">
  ## Récupérer un jeton de test
</div>

Pour en savoir plus sur la [récupération de jetons d’accès, cliquez ici](https://auth0.com/docs/secure/tokens/access-tokens/get-access-tokens). Pour ce Quickstart, vous pouvez toutefois simplement utiliser un jeton d’accès depuis la [vue « test » des paramètres de votre API](https://manage.auth0.com/#/apis).

<Info>
  La route `/me` que nous avons créée ci-dessus ne fonctionnera pas avec un jeton de test, puisqu’aucun utilisateur réel n’y est associé.
</Info>

<div id="checkpoint">
  ## Vérification
</div>

Ouvrez un terminal et essayez d’envoyer des requêtes à votre application.

Commencez par envoyer une requête à la route publique :

```bash lines
curl --request GET \
  --url http://localhost:8000/api \
  --header 'Accept: application/json'
```

Ensuite, utilisez votre jeton d’accès dans un en-tête `Authorization` pour effectuer une requête vers une route protégée :

```bash lines
curl --request GET \
  --url http://localhost:8000/api/private \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

Enfin, essayez d’envoyer une requête à la route protégée par la portée, ce qui ne réussira que si la portée `read:messages` a été accordée au jeton d’accès :

```bash lines
curl --request GET \
  --url http://localhost:8000/api/scope \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```

Si vous rencontrez des problèmes, voici quelques pistes à essayer :

* Essayez d’exécuter `php artisan optimize:clear` pour vider le cache de Laravel.
* Assurez-vous que vos fichiers `.auth0.app.json` et `.auth0.api.json` se trouvent à la racine de votre projet.
* Assurez-vous d’avoir activé votre application Laravel comme Machine-to-Machine Application et de lui avoir accordé toutes les portées nécessaires pour l’`Auth0 Management API` dans le [Auth0 Dashboard](https://manage.auth0.com/#/apis/).

Vous avez des problèmes ? Consultez la [documentation](https://github.com/auth0/laravel-auth0) du SDK ou notre [centre de documentation](https://auth0.com/docs). Vous pouvez aussi visiter [la Community](https://community.auth0.com), où notre équipe et d’autres membres de la communauté pourront répondre à vos questions.
