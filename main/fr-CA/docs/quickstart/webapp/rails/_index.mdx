    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Pour utiliser les services Auth0, vous devez avoir une application configurée dans l’Auth0 Dashboard. L’application Auth0 est
      l’endroit où vous configurerez le fonctionnement de l’authentification pour le projet que vous développez.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle application Auth0 ou sélectionner une application existante qui représente
      le projet que vous souhaitez intégrer. Chaque application dans Auth0 se voit attribuer un client ID unique et alphanumérique
      que le code de votre application utilisera pour appeler les API Auth0 au moyen du SDK.

      Tous les paramètres que vous configurez à l’aide de ce Quickstart mettront automatiquement à jour votre Application dans le [Dashboard](https://manage.auth0.com/#/), où vous
      pourrez gérer vos Applications par la suite.

      Si vous préférez explorer une configuration complète, vous pouvez plutôt consulter un exemple d’application.

      ### Configurer les URL de rappel

      Une URL de rappel est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      authentification. Si elle n’est pas définie, les utilisateurs ne seront pas redirigés vers votre application après s’être connectés.

      <Info>
        Si vous suivez notre projet d’exemple, définissez cette valeur à
        `http://localhost:3000/auth/auth0/callback`.
      </Info>

      ### Configurer les URL de déconnexion

      Une URL de déconnexion est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      déconnexion. Si elle n’est pas définie, les utilisateurs ne pourront pas se déconnecter de votre application et recevront une erreur.

      <Info>
        Si vous suivez notre projet d’exemple, définissez cette valeur à `http://localhost:3000`.
      </Info>

      ### Configurer les origines Web autorisées

      Une origine Web autorisée est une URL que vous souhaitez autoriser à accéder à votre flux d’authentification. Elle doit
      contenir l’URL de votre projet. Si elle n’est pas correctement définie, votre projet ne pourra pas actualiser silencieusement
      les jetons d’authentification; vos utilisateurs seront donc déconnectés lors de leur prochaine visite dans votre application ou lorsqu’ils actualiseront une
      page.

      <Info>
        Si vous suivez notre projet d’exemple, définissez cette valeur à `http://localhost:3000`.
      </Info>

      Utilisez `omniauth-auth0`, une [stratégie OmniAuth](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication) personnalisée, pour gérer le processus d’authentification.

      Ajoutez les dépendances suivantes à votre `Gemfile` :

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # empêche les requêtes d'authentification falsifiées
      ```

      Une fois les gems ajoutés, installez-les avec `bundle install`.

      Créez un contrôleur Auth0 pour gérer le callback d&#39;authentification, l&#39;action `logout` et les méthodes permettant de
      construire l&#39;URL de déconnexion.

      Exécutez la commande :
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

      Dans la méthode callback, assignez le hachage des renseignements utilisateur — renvoyé dans
      `request.env['omniauth.auth']` — à la session active.

      Pour configurer logout, effacez tous les objets stockés dans la session en appelant la méthode `reset_session`
      dans l&#39;action `logout`. Redirigez ensuite vers l&#39;endpoint de logout d&#39;Auth0. Pour en savoir plus sur
      `reset_session`, consultez la [documentation Ruby on Rails ActionController](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668).

      Un utilisateur peut maintenant se connecter à votre application en accédant au point de terminaison `/auth/auth0`.

      <Warning>
        Pour [éviter les requêtes d’authentification falsifiées](https://github.com/cookpad/omniauth-rails_csrf_protection), utilisez les méthodes utilitaires `link_to` ou
        `button_to` avec la méthode `:post`.
      </Warning>

      ```ruby lines
      <!-- Placez un bouton de connexion n'importe où dans votre application -->
      <%= button_to 'Login', '/auth/auth0', method: :post %>
      ```

      <Note>
        ##### Checkpoint

        Ajoutez à votre application un bouton qui redirige l’utilisateur vers le endpoint `/auth/auth0` lorsqu’il le

        application une fois l’authentification réussie.

      moyen de vous déconnecter. Déconnectez un utilisateur en le redirigeant vers l’action `auth/logout`, qui le redirige

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9" isSingleColumn>
      Pour afficher le profil de l&#39;utilisateur, votre application doit fournir une route protégée. Vous pouvez utiliser un [Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) pour contrôler l&#39;accès aux routes pouvant être partagées entre plusieurs
      contrôleurs. Le concern doit automatiquement rediriger l&#39;utilisateur vers Auth0 lorsque celui-ci n&#39;est pas authentifié. Sinon, le
      concern doit renvoyer le profil de l&#39;utilisateur courant.

      Une fois que vous avez un Concern, incluez-le dans tout contrôleur qui nécessite qu&#39;un utilisateur soit connecté. Vous pouvez ensuite accéder à l&#39;
      utilisateur à partir de la session `session[:userinfo]`, comme dans l&#39;exemple suivant :

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      Une fois l’utilisateur chargé depuis la session, utilisez-le pour afficher des informations dans votre interface utilisateur :

      ```html lines
      <div>
      <p>Normalized User Profile:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>Full User Profile:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Note>
        ##### Point de reprise

        Ajoutez le concern `Secured` à votre application, puis incluez-le dans le contrôleur qui nécessite un
        utilisateur authentifié pour y accéder. Vérifiez qu’un utilisateur authentifié a accès aux actions de ce
        contrôleur et que les utilisateurs non authentifiés sont redirigés vers Auth0 pour s’authentifier.
      </Note>
