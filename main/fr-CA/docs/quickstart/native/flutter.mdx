Ce guide montre comment intégrer Auth0 à une application Flutter à l’aide du [Auth0 Flutter SDK](https://github.com/auth0/auth0-flutter). Il couvre la configuration pour les plateformes **Android**, **iOS** et **macOS**.

<Info>
  Le Auth0 Flutter SDK prend aussi en charge **Web** et **Windows** (bêta). Des Quickstarts distincts sont offerts pour ces plateformes :

  * [Quickstart Flutter Web](/docs/fr-CA/quickstart/spa/flutter) — pour les applications Flutter destinées au navigateur
  * [Quickstart Flutter Windows](/docs/fr-CA/quickstart/native/flutter-windows) — pour les applications de bureau Flutter sous Windows (bêta)
</Info>

<div id="get-started">
  ## Pour commencer
</div>

<Steps>
  <Step title="Créer un nouveau projet Flutter" stepNumber={1}>
    Créez un nouveau projet Flutter pour ce Quickstart.

    **Dans votre terminal :**

    1. Accédez à votre répertoire de travail
    2. Exécutez : `flutter create auth0_flutter_sample`
    3. Accédez au répertoire du projet : `cd auth0_flutter_sample`
    4. Ouvrez-le dans votre IDE :
       * **VS Code** : `code .`
       * **Android Studio** : `open -a "Android Studio" .`

    ```shellscript
    # Créer un nouveau projet Flutter
    flutter create auth0_flutter_sample

    # Accéder au répertoire du projet
    cd auth0_flutter_sample

    # Ouvrir dans VS Code
    code .
    ```

    <Tip>
      Cette commande crée une application Flutter moderne avec la structure de projet la plus récente. Exécutez `flutter doctor` pour vérifier que votre environnement est correctement configuré.
    </Tip>
  </Step>

  <Step title="Installer le SDK Flutter d’Auth0" stepNumber={2}>
    Ajoutez l’Auth0 Flutter SDK à votre projet à l’aide de la CLI de Flutter.

    ```shellscript
    flutter pub add auth0_flutter
    ```

    Cela ajoutera `auth0_flutter` aux dépendances de votre `pubspec.yaml` :

    ```yaml pubspec.yaml lines
    dependencies:
      auth0_flutter: ^2.0.0-beta.1
    ```

    <Tip>
      Le SDK Auth0 Flutter nécessite **Flutter 3.24.0+** et **Dart 3.5.0+**. Exécutez `flutter doctor` pour vérifier que votre environnement respecte ces exigences.
    </Tip>
  </Step>

  <Step title="Configurez votre Auth0 App" stepNumber={3}>
    La prochaine étape consiste à créer une nouvelle application dans votre Auth0 tenant et à configurer les callback URLs.

    1. Accédez à l’[Auth0 Dashboard](https://manage.auth0.com/dashboard/)
    2. Cliquez sur **Applications** &gt; **Applications** &gt; **Create Application**
    3. Dans la fenêtre contextuelle, saisissez un nom pour votre application, sélectionnez `Native` comme type d’application, puis cliquez sur **Create**
    4. Passez à l’onglet **Settings** de la page Application Details
    5. Prenez en note les valeurs **Domain** et **Client ID** - vous en aurez besoin plus tard

    Dans l’onglet **Settings**, configurez les URL suivantes en fonction de votre ou vos plateformes cibles :

    **Allowed Callback URLs :**

    <Tabs>
      <Tab title="Android">
        ```text
        https://{yourDomain}/android/{yourPackageName}/callback
        ```
      </Tab>

      <Tab title="iOS">
        ```text
        https://{yourDomain}/ios/{yourBundleIdentifier}/callback,
        {yourBundleIdentifier}://{yourDomain}/ios/{yourBundleIdentifier}/callback
        ```
      </Tab>

      <Tab title="macOS">
        ```text
        https://{yourDomain}/macos/{yourBundleIdentifier}/callback,
        {yourBundleIdentifier}://{yourDomain}/macos/{yourBundleIdentifier}/callback
        ```
      </Tab>
    </Tabs>

    **Allowed Logout URLs :**

    Ajoutez dans le champ **Allowed Logout URLs** les mêmes URL que celles utilisées dans la configuration des callbacks ci-dessus.

    <Info>
      Les **Allowed Callback URLs** constituent une mesure de sécurité essentielle pour garantir que les utilisateurs sont redirigés de façon sécuritaire vers votre application après l’authentication. Sans URL correspondante, le login process échouera.

      Les **Allowed Logout URLs** sont essentielles pour offrir une user experience fluide lors de la déconnexion. Sans URL correspondante, les utilisateurs ne seront pas redirigés vers votre application après le logout.

      Par exemple, si votre Auth0 domain est `example.us.auth0.com` et que le nom de package de votre application Android est `com.example.myapp`, votre callback URL Android serait : `https://example.us.auth0.com/android/com.example.myapp/callback`
    </Info>

    <Warning>
      **Important** : Assurez-vous que le nom du package (Android) ou le bundle identifier (iOS/macOS) dans vos callback URLs correspond bien à l’identifiant réel de votre application. Si l’authentication échoue, vérifiez que ces valeurs sont identiques.
    </Warning>
  </Step>

  <Step title="Configurez votre application" stepNumber={4}>
    Une configuration propre à chaque plateforme est requise pour activer le flux d’authentification. Suivez les instructions pour chaque plateforme ciblée.

    <Tabs>
      <Tab title="Android">
        Ouvrez le fichier `android/app/build.gradle` et ajoutez les variables de remplacement du manifeste suivantes dans `android > defaultConfig` :

        ```groovy android/app/build.gradle lines
        android {
            // ...
            defaultConfig {
                // Ajoutez la ligne suivante
                manifestPlaceholders += [auth0Domain: "{yourDomain}", auth0Scheme: "https"]
            }
            // ...
        }
        ```

        Remplacez `{yourDomain}` par votre domaine Auth0 (p. ex., `example.us.auth0.com`).

        **schéma HTTPS**

        Pour utiliser le schéma `https` avec votre URL de rappel, configurez les [Android App Links](https://auth0.com/docs/get-started/applications/enable-android-app-links-support) pour votre application.

        **Pour l’authentification biométrique (facultatif) :**

        Si vous comptez utiliser l’authentification biométrique, mettez à jour votre `MainActivity.kt` pour qu’il étende `FlutterFragmentActivity` :

        ```kotlin android/app/src/main/kotlin/.../MainActivity.kt lines
        import io.flutter.embedding.android.FlutterFragmentActivity

        class MainActivity: FlutterFragmentActivity() {
        }
        ```
      </Tab>

      <Tab title="iOS">
        Pour utiliser Universal Links (URL de rappel HTTPS) sur iOS 17.4+, configurez un domaine associé :

        1. Ouvrez votre application dans Xcode : `open ios/Runner.xcworkspace`
        2. Sélectionnez la cible **Runner** et accédez à **Signing &amp; Capabilities**
        3. Cliquez sur **+ Capability** et ajoutez **Associated Domains**
        4. Ajoutez l’entrée : `webcredentials:{yourDomain}`
        5. Dans le [Auth0 Dashboard](https://manage.auth0.com/#/applications), accédez à **Settings &gt; Advanced Settings &gt; Device Settings**
        6. Dans la section **iOS**, définissez votre **Team ID** et votre **App ID** (identificateur du bundle)

        <Info>
          La configuration de Universal Links est facultative. Ignorez cette étape si vous n’avez pas besoin de la prise en charge de Universal Links : le SDK utilisera automatiquement des schémas d’URL personnalisés.
        </Info>
      </Tab>

      <Tab title="macOS">
        Suivez les mêmes étapes que pour iOS, mais ouvrez plutôt `macos/Runner.xcworkspace`.
      </Tab>
    </Tabs>

    <Warning>
      **Android** : Assurez-vous que la valeur `auth0Domain` correspond exactement à votre domaine Auth0. Si l’authentification échoue, vérifiez que cette valeur est identique au domaine affiché dans votre Auth0 Dashboard.

      **iOS/macOS** : Universal Links nécessitent un compte Apple Developer payant ainsi qu’iOS 17.4+/macOS 14.4+. Sur les versions antérieures, le SDK utilisera automatiquement des schémas d’URL personnalisés.
    </Warning>
  </Step>

  <Step title="Implement Connexion et déconnexion" stepNumber={5}>
    [Universal Login](https://auth0.com/docs/authenticate/login/auth0-universal-login) est la façon la plus simple de configurer l’authentification dans votre application. Nous vous recommandons de l’utiliser pour profiter de la meilleure expérience, de la meilleure sécurité et de l’éventail de fonctionnalités le plus complet.

    **Implement Login :**

    Importez le Auth0 Flutter SDK et créez une instance `Auth0` :

    ```dart lib/auth_service.dart lines
    import 'package:auth0_flutter/auth0_flutter.dart';

    class AuthService {
      final auth0 = Auth0('{yourDomain}', '{yourClientId}');

      Future<Credentials> login() async {
        final credentials = await auth0.webAuthentication().login(useHTTPS: true);

        // Jeton d'accès -> credentials.accessToken
        // Jeton d'identité -> credentials.idToken
        // Profil utilisateur -> credentials.user

        return credentials;
      }
    }
    ```

    **Implement Déconnexion :**

    ```dart lib/auth_service.dart lines
    Future<void> logout() async {
      await auth0.webAuthentication().logout(useHTTPS: true);

      // L'utilisateur est maintenant déconnecté
      // Les informations d'identification ont été effacées du stockage sécurisé
    }
    ```

    <Info>
      **iOS/macOS** : Le paramètre `useHTTPS: true` active Universal Links sur iOS 17.4+ et macOS 14.4+ pour une sécurité renforcée.

      **Android** : Si vous utilisez un schéma personnalisé, transmettez-le à la méthode `login` afin que le SDK puisse rediriger correctement vers la page de connexion, puis revenir correctement :

      ```dart
      await auth0.webAuthentication(scheme: 'YOUR CUSTOM SCHEME').login();
      ```
    </Info>
  </Step>

  <Step title="Afficher les informations du profil utilisateur" stepNumber={6}>
    Le profil utilisateur est récupéré automatiquement lorsque l’utilisateur se connecte. L’objet `Credentials` contient une propriété `user` qui renferme toutes les informations du profil utilisateur, obtenues en décodant le jeton d’identification.

    ```dart lib/profile_screen.dart lines
    void displayUserProfile(Credentials credentials) {
      final user = credentials.user;
      
      print('User ID: ${user.sub}');
      print('Email: ${user.email}');
      print('Name: ${user.name}');
      print('Picture: ${user.pictureUrl}');
      print('Nickname: ${user.nickname}');
    }
    ```

    <Tip>
      Demandez les portées d’accès appropriées lors de la connexion pour accéder à des champs précis du profil utilisateur. Les portées d’accès par défaut sont `openid`, `profile`, `email` et `offline_access`.
    </Tip>
  </Step>

  <Step title="Exécutez votre application" stepNumber={7}>
    Compilez et exécutez votre application Flutter.

    **Dans votre terminal :**

    ```shellscript
    # Lister les appareils disponibles
    flutter devices

    # Exécuter sur Android
    flutter run -d android

    # Exécuter sur le simulateur iOS
    flutter run -d ios

    # Exécuter sur macOS
    flutter run -d macos
    ```

    **Flux attendu :**

    1. L’application s’ouvre sur votre interface de connexion
    2. L’utilisateur appuie sur **Se connecter** → le navigateur/l’onglet personnalisé s’ouvre avec Auth0 Universal Login
    3. L’utilisateur termine l’authentification
    4. Le navigateur redirige l’utilisateur vers votre application
    5. L’utilisateur est maintenant authentifié et les informations d’identification sont enregistrées
  </Step>
</Steps>

<Check>
  **Checkpoint**

  Vous devriez maintenant avoir une expérience de connexion avec Auth0 entièrement fonctionnelle dans votre application Flutter. L’application utilise une authentification sécurisée par navigateur et stocke automatiquement les identifiants pour assurer la persistance de la session.
</Check>

  ## Dépannage et utilisation avancée
</div>

<Accordion title="Problèmes courants et solutions">
  ### Incompatibilité d’URL de rappel

  **Symptôme** : Erreur &quot;redirect&#95;uri&#95;mismatch&quot; ou échec silencieux de l’authentification.

  **Solutions :**

  1. Vérifiez que les **Allowed Callback URLs** dans Auth0 Dashboard correspondent exactement à la configuration de votre application
  2. Vérifiez le schéma (`https://` vs `http://`)
  3. Assurez-vous que le nom du package (Android) ou le Bundle Identifier (iOS/macOS) est correct
  4. Vérifiez s’il y a une barre oblique à la fin

  ### Android : l’onglet personnalisé Chrome ne s’ouvre pas

  **Symptôme** : Rien ne se passe lors de l’appel à `login()`.

  **Correctif :**

  1. Vérifiez que `manifestPlaceholders` est correctement défini dans `build.gradle`
  2. Assurez-vous que la permission d’accès à Internet figure dans `AndroidManifest.xml` :
     ```xml
     <uses-permission android:name="android.permission.INTERNET" />
     ```
  3. Vérifiez que Chrome ou un autre navigateur est installé sur l’appareil

  ### iOS : alerte « Open in App »

  **Symptôme** : Une boîte de dialogue s’affiche et vous demande d’ouvrir dans votre application.

  **Correctif :** Il s’agit du comportement attendu avec `ASWebAuthenticationSession`. Pour la supprimer :

  * Utilisez Universal Links (nécessite iOS 17.4+ et un compte Apple Developer payant)
  * Ou définissez `useEphemeralSession: true` (désactive le SSO) :

  ```dart expandable
  await auth0.webAuthentication().login(
    useHTTPS: true,
    useEphemeralSession: true,
  );
  ```

  ### Authentification annulée par l’utilisateur

  Gérez ce cas correctement dans votre gestion des erreurs :

  ```dart expandable
  try {
    final credentials = await auth0.webAuthentication().login(useHTTPS: true);
    // Gérer une connexion réussie
  } on WebAuthenticationException catch (e) {
    if (e.code == 'USER_CANCELLED') {
      showMessage('La connexion a été annulée');
    } else {
      showMessage('Échec de la connexion : ${e.message}');
    }
  }
  ```
</Accordion>

<Accordion title="Gestion des informations d’identification">
  Auth0 Flutter SDK comprend un Credentials Manager intégré qui stocke de façon sécurisée les informations d’identification des utilisateurs. Sur les plateformes mobiles, les informations d’identification sont chiffrées et stockées dans l’espace de stockage sécurisé de la plateforme (Keychain sur iOS/macOS, SharedPreferences chiffré sur Android).

  ### Vérifier la présence d’informations d’identification stockées

  Avant d’inviter l’utilisateur à se connecter, vérifiez si des informations d’identification valides existent déjà :

  ```dart lib/auth_service.dart expandable lines
  Future<bool> checkAuthentication() async {
    return await auth0.credentialsManager.hasValidCredentials();
  }
  ```

  ### Récupérer les informations d’identification stockées

  Récupérez les informations d’identification pour accéder aux jetons d’accès ou aux renseignements de l’utilisateur. Le Credentials Manager renouvelle automatiquement les jetons expirés lorsque possible :

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> getCredentials() async {
    return await auth0.credentialsManager.credentials();
  }
  ```

  <Tip>
    Vous n’avez pas besoin de stocker manuellement les informations d’identification après la connexion — le SDK s’en charge automatiquement. Vous n’avez pas non plus besoin de renouveler manuellement les jetons ; le Credentials Manager s’en charge au besoin.
  </Tip>
</Accordion>

<Accordion title="Gestion des erreurs">
  Gérez les erreurs d’authentification correctement afin d’offrir une bonne expérience utilisateur.

  ```dart lib/auth_service.dart expandable lines
  import 'package:auth0_flutter/auth0_flutter.dart';

  Future<void> login() async {
    try {
      final credentials = await auth0.webAuthentication().login(useHTTPS: true);
      // Gérer une connexion réussie
    } on WebAuthenticationException catch (e) {
      if (e.code == 'USER_CANCELLED') {
        // L’utilisateur a annulé la connexion
        print('Connexion annulée par l’utilisateur');
      } else {
        // Gérer les autres erreurs
        print('Erreur de connexion : ${e.message}');
      }
    }
  }

  Future<Credentials> getCredentials() async {
    try {
      return await auth0.credentialsManager.credentials();
    } on CredentialsManagerException catch (e) {
      if (e.isNoCredentialsFound) {
        // Aucune information d’identification stockée, l’utilisateur doit se connecter
        throw Exception('Veuillez d’abord vous connecter');
      } else if (e.isTokenRenewFailed) {
        // Le jeton d’actualisation a expiré, une nouvelle authentification est requise
        return await auth0.webAuthentication().login(useHTTPS: true);
      }
      rethrow;
    }
  }
  ```
</Accordion>

<Accordion title="Intégration Flutter avancée">
  ### Sécurité renforcée des informations d’authentification grâce à la biométrie

  Implémentez l’authentification biométrique pour accéder aux informations d’authentification sur mobile :

  ```dart lib/secure_auth_service.dart expandable lines
  class SecureAuthService {
    final auth0 = Auth0('{yourDomain}', '{yourClientId}');
    
    Future<void> enableBiometrics() async {
      // Activer l’authentification locale (Face ID, Touch ID, empreinte digitale)
      await auth0.credentialsManager.enableLocalAuthentication(
        title: 'Authenticate to access your account',
        cancelTitle: 'Cancel',
        fallbackTitle: 'Use passcode',
      );
    }
    
    Future<Credentials> getCredentialsWithBiometrics() async {
      // Cela exigera maintenant une authentification biométrique
      return await auth0.credentialsManager.credentials();
    }
  }
  ```

  <Info>
    **Android** : exige que `MainActivity` étende `FlutterFragmentActivity`, comme configuré à l’étape 4.

    **iOS/macOS** : exige l’ajout de `NSFaceIDUsageDescription` à votre fichier `Info.plist`.
  </Info>

  ### Scopes personnalisés et audience

  Demandez des scopes précis et une audience pour votre API :

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> loginWithCustomScopes() async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      scopes: {'openid', 'profile', 'email', 'offline_access', 'read:posts'},
      audience: 'https://myapi.example.com',
      parameters: {'prompt': 'login'},
    );
  }
  ```

  ### Organizations (B2B/Enterprise)

  Authentifiez les utilisateurs au sein d’une organization précise :

  ```dart lib/auth_service.dart expandable lines
  Future<Credentials> loginWithOrganization(String organizationId) async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      organizationId: organizationId,
    );
  }

  // Ou inviter l’utilisateur à sélectionner une organization
  Future<Credentials> loginWithOrganizationName(String organizationName) async {
    return await auth0.webAuthentication().login(
      useHTTPS: true,
      organizationName: organizationName,
    );
  }
  ```
</Accordion>

<Accordion title="Déploiement en production">
  ### Préparation pour l’App Store

  * Configurez les Universal Links (iOS) et les App Links (Android) pour une authentification fluide
  * Effectuez des tests sur différentes tailles d’appareils et versions du système d’exploitation
  * Implémentez une gestion adéquate des erreurs en cas de défaillance réseau
  * Ajoutez des règles ProGuard pour Android si vous utilisez l’obscurcissement du code
  * Respectez les politiques propres aux plateformes App Store/Play Store

  ### Considérations de sécurité

  * Utilisez le Credentials Manager intégré pour stocker les informations d’authentification en production
  * Activez l’authentification biométrique pour les opérations sensibles
  * Envisagez l’épinglage de certificat pour renforcer la sécurité de l’API
  * Implémentez une gestion adéquate du renouvellement des tokens
  * Utilisez `useHTTPS: true` pour les Universal Links sur les plateformes prises en charge
</Accordion>
