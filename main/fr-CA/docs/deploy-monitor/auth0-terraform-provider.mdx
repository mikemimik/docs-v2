description: Découvrez comment utiliser l’Auth0 Terraform Provider pour représenter les configurations de votre tenant sous forme de code.
title: Auth0 Terraform Provider
---

Le Deploy CLI n’est pas le seul outil offert pour gérer la configuration de votre tenant Auth0 ; il existe aussi un [Terraform Provider officiellement pris en charge](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) est un outil tiers qui permet de représenter sous forme de code la configuration de vos ressources infonuagiques. Il repose sur un cadre de plugiciels bien établi qui prend en charge un large éventail de fournisseurs infonuagiques, dont Auth0.

L’Auth0 Terraform Provider est aussi offert pour [OpenTofu](https://opentofu.org/), l’outil libre compatible avec Terraform. Vous trouverez le fournisseur Auth0 dans le [registre OpenTofu](https://search.opentofu.org/provider/auth0/auth0/latest).

Le Deploy CLI et le Terraform Provider existent tous deux pour vous aider à gérer les configurations de vos tenants Auth0, mais chacun comporte ses propres avantages et inconvénients.

Vous pourriez envisager d’utiliser l’Auth0 Terraform Provider si :

* Vos workflows de développement utilisent déjà Terraform ou OpenTofu
* Vos besoins en gestion de tenant sont très précis ou ne concernent que quelques ressources particulières

Vous pourriez ne pas vouloir envisager l’Auth0 Terraform Provider si :

* Votre workflow de développement n’utilise pas Terraform ou OpenTofu, ce qui exige une configuration supplémentaire au départ
* Vos workflows de développement visent principalement la gestion en bloc de vos tenants
* Votre tenant comporte déjà beaucoup de ressources, ce qui peut demander des efforts importants pour les « importer »
