<div id="use-automated-deployment-for-actions">
  ## Utiliser le déploiement automatisé pour les Actions
</div>

Développer une Action dans l’éditeur de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Consulter le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> est un excellent moyen de l’implémenter et de la tester tant qu’elle est encore en cours de développement. Cependant, quand vient le moment de la déployer dans des environnements de test automatisés ou de production, il faut un mécanisme plus automatisé; copier-coller du code d’un tenant Auth0 à l’autre n’est pas une méthode satisfaisante.

<div id="set-up-version-control-for-actions">
  ### Configurer le contrôle de version pour Actions
</div>

Les modifications apportées à une Action dans l’Auth0 Dashboard sont enregistrées à l’état de brouillon jusqu’à leur déploiement. Après le déploiement, une version est créée, mais vous devez associer l’Action à un déclencheur pour qu’elle s’active. Nous vous recommandons d’utiliser le contrôle de version pour assurer la gestion des changements et la révision du code. [Actions prend en charge la gestion des versions](docs/customize/actions/manage-versions) de façon native dans le Dashboard.

Auth0 propose des méthodes de déploiement automatisé pour Actions entre des environnements de tenant Auth0. Les intégrations Auth0 suivantes vous permettent de mettre à jour des Actions à partir du système de contrôle de version, manuellement comme automatiquement (par exemple, lorsqu’un changement dans le système de contrôle de version est détecté) : [GitHub Actions](https://marketplace.auth0.com/integrations/github-actions), [GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline), [Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline) et [Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline).

<div id="use-deploy-cli-tool-for-action-import-and-export">
  ### Utiliser l’outil Deploy CLI pour importer et exporter des Actions
</div>

L’outil Auth0 Deploy CLI automatise le déploiement des Actions entre des tenants Auth0. Il fonctionne avec des fichiers YAML locaux ou fondés sur un répertoire, en combinaison avec l’Auth0 <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>, et permet d’exporter des Actions à partir d’un tenant Auth0 ainsi que de les importer dans un tenant Auth0. De plus, l’outil offre un contrôle programmatique sur l’ordre et la configuration d’environnement d’une Action dans le cadre de l’automatisation du déploiement. Pour en savoir plus, consultez [Deploy CLI Tool](/fr-CA/docs/deploy-monitor/deploy-cli-tool).

Comme bonne pratique, utilisez Auth0 Deploy CLI dans presque tous les cas de déploiement vers des environnements de test ou de production. Bien que les intégrations Marketplace permettent de détecter automatiquement les changements, Deploy CLI offre un contrôle précis sur ce qui est déployé, à quel moment et de quelle façon.

<div id="set-up-test-tenants">
  ## Configurer des tenants de test
</div>

Nous vous recommandons de configurer un tenant de test distinct dans Auth0 afin de tester en toute sécurité vos Actions avant de les déployer en Production. Pour en savoir plus, consultez [Configurer plusieurs environnements](/fr-CA/docs/get-started/auth0-overview/create-tenants/set-up-multiple-environments).
