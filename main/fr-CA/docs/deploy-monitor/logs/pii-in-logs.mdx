title: Renseignements personnels identifiables dans les journaux Auth0
---

De nombreux journaux du tenant Auth0 peuvent contenir des renseignements personnels identifiables (PII). Les événements d’authentification génèrent des entrées de journal qui peuvent inclure les renseignements personnels des utilisateurs. Et si vous utilisez l’Auth0 <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> ou le Dashboard pour ajouter des utilisateurs, Auth0 consigne les détails du compte utilisateur.

Ces champs dans les entrées de journal peuvent contenir des PII :

* Nom
* Numéro de téléphone (utilisé pour l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip>)
* Adresse courriel
* Adresse IP (qui peut révéler la localisation)
* Toute information utilisateur personnalisée que vous définissez

À aucun moment, Auth0 ne consigne de <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+tokens">jetons d’accès</Tooltip> d’Auth0 ni de ceux d’un <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip>. Lorsqu’un échange de code d’autorisation a lieu, les journaux n’affichent qu’un code partiel (par exemple : `code: 31XXXXX`).
