Accédez à la page [Auth0 Public Cloud Status](https://status.auth0.com) pour vérifier la disponibilité du service de la version de nuage public d’Auth0.

Sélectionnez une **Région** dans le menu déroulant pour voir l’état des services de base (comme l’authentification des utilisateurs et l’authentification machine à machine) et des services de soutien (comme <Tooltip tip="Authentification multifacteur (MFA) : Processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=MFA">MFA</Tooltip>, <Tooltip tip="Management API : Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>, les journaux du tenant et le CDN).

Pour en savoir plus sur les régions disponibles, consultez le tableau [Région, localité et sous-localité](/fr-CA/docs/get-started/auth0-overview/create-tenants#region-locality-and-sub-locality) dans [Créer des tenants](/fr-CA/docs/get-started/auth0-overview/create-tenants).

<div id="subscribe-to-public-cloud-status-updates">
  ## S’abonner aux mises à jour d’état du Public Cloud
</div>

Sélectionnez une **Région**, puis sélectionnez **Subscribe** pour recevoir des mises à jour. Vous pouvez choisir l’une des deux options suivantes pour consulter l’état :

1. Suivez [@auth0status](https://x.com/auth0status) sur X pour obtenir les plus récentes mises à jour d’état.
2. Abonnez-vous au flux Atom ou RSS d’Auth0 pour recevoir les mises à jour d’état qui touchent votre tenant. Remplacez `{YOUR_TENANT_DOMAIN}` par le **Domain** de votre tenant.

Pour trouver le **Domain** de votre tenant, accédez à [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/dashboard/#/applications), sélectionnez une application, puis accédez à **Settings**. Le **Domain** de votre tenant figure sous **Basic Information**.

<div id="check-auth0-status-for-private-cloud">
  ## Vérifier l’état d’Auth0 pour Private Cloud
</div>

Accédez à [Auth0 Public Cloud Status](https://status.auth0.com), puis sélectionnez **Private Cloud Login**.

Une fois authentifié, vous pouvez consulter l’état de tout environnement autre qu’un environnement de développement (par exemple, `travel0-prod-1`), s’il est disponible sur la plateforme convergée d’Auth0.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour accéder à l’état de votre environnement Private Cloud, vous devez être membre du tenant de soutien Public Cloud associé. Pour en savoir plus, veuillez consulter [Ouvrir et gérer les tickets de soutien](/fr-CA/docs/troubleshoot/customer-support/open-and-manage-support-tickets) et [Gérer l’accès au Dashboard](/fr-CA/docs/get-started/manage-dashboard-access).
</Callout>

Pour revenir à la page Auth0 Public Cloud Status, sélectionnez **Auth0 Public Cloud Status** dans la barre de navigation en haut à droite.

<div id="subscribe-to-private-cloud-status-updates">
  ## Abonnez-vous aux mises à jour du statut de Private Cloud
</div>

Sélectionnez un environnement, puis sélectionnez **Subscribe**.

Abonnez-vous au flux Atom ou RSS d’Auth0 en copiant l’URL générée dans la zone « Subscribe » pour recevoir des mises à jour de statut concernant votre environnement.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’URL générée du flux ATOM / RSS est propre à chaque environnement Private Cloud et contient des jetons qui peuvent faire l’objet d’une rotation en [contactant le soutien](https://support.auth0.com/).
</Callout>
