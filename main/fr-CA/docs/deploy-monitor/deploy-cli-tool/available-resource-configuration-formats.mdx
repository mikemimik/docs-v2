description: Découvrez les formats de configuration disponibles lors de l’utilisation de l’Auth0 Deploy CLI.
title: Formats de configuration des ressources disponibles
---

L’état des ressources Auth0 peut être représenté dans deux formats de fichier de configuration différents : YAML et répertoire (JSON). Lorsque vous utilisez la commande `export` du Deploy CLI, il vous sera demandé de choisir entre les deux.

<div id="yaml">
  ## YAML
</div>

Le format YAML se présente principalement sous la forme d’un fichier plat tenant.yaml, accompagné de fichiers de code supplémentaires pour des ressources comme les Actions et les gabarits de courriel. Ce fichier unique simplifie le suivi des changements au fil du temps dans le contrôle de version. De plus, il élimine une certaine ambiguïté liée aux noms de répertoires et de fichiers, qui n’est pas toujours évidente au premier abord.

<div id="directory-json">
  ## Répertoire (JSON)
</div>

Le format de répertoire sépare les types de ressources en répertoires distincts, chaque ressource étant stockée dans son propre fichier JSON. Ce format facilite la séparation conceptuelle entre chaque type de ressource, ainsi qu’entre les ressources individuelles elles-mêmes. De plus, le Deploy CLI reflète de près les structures de données définies dans l’[Auth0 Management API](https://auth0.com/docs/api/management/v2); il peut donc être utile de consulter les exemples JSON dans la documentation lorsque vous utilisez ce format.

<div id="how-to-choose">
  ## Comment choisir
</div>

Le choix du format à utiliser devrait avant tout relever d’une question de préférence. Les deux formats sont des solutions viables qui permettent d’accomplir la même tâche, mais avec des forces et des faiblesses légèrement différentes, décrites ci-dessus. Assurez-vous d’évaluer chacun en fonction de votre contexte. Il est important de noter que ce choix n’est pas définitif et qu’il est possible de passer de l’un à l’autre à l’aide de la commande `import`.
