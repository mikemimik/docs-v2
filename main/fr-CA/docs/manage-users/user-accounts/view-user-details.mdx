---
description: Comment afficher les utilisateurs et les détails de leur profil.
title: Afficher les détails de l’utilisateur
---

La page [Users](https://manage.auth0.com/#/users) répertorie les utilisateurs associés à vos applications. Pour ouvrir un utilisateur en particulier, cliquez sur sa photo de profil ou sur son nom dans la colonne « Name ». La page Détails de l’utilisateur s’ouvre et affiche les renseignements sur cet utilisateur.

La page Détails de l’utilisateur comporte des liens vers cinq onglets :

1. Détails
2. Appareils
3. Historique
4. Lieux
5. JSON brut

<div id="user-details-details-tab">
  ## Détails de l’utilisateur : onglet Détails
</div>

L’onglet Détails contient cinq sections qui offrent un aperçu général des informations figurant dans le profil de l’utilisateur :

* **Identité de l’utilisateur** présente en un coup d’œil des renseignements sur l’utilisateur, y compris l’identifiant utilisé, les Connections associées et les droits d’accès.
* **Metadata** affiche les informations `app_metadata` et `user_metadata`. Vous pouvez modifier ces valeurs.
* **<Tooltip tip="fournisseur d’identité (IdP)\u00a0: service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> Attributes** affiche les informations récupérées auprès du fournisseur d’authentification. Notez que les attributs du fournisseur d’identité sont en lecture seule.
* **Passkeys** affiche les [passkeys](/fr-CA/docs/authenticate/database-connections/passkeys) créées par l’utilisateur.
* **<Tooltip tip="Multi-factor authentication (MFA)\u00a0: processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-Factor+Authentication">Multi-Factor Authentication</Tooltip>** affiche les facteurs [MFA](/fr-CA/docs/secure/multi-factor-authentication) que l’utilisateur a configurés.
  Si vous activez [Multiple Custom Domains](/fr-CA/docs/customize/custom-domains/multiple-custom-domains), vous disposez de l’option **Domain** pour sélectionner le domaine de notification de votre tenant.

<div id="user-details-devices-tab">
  ## Détails de l’utilisateur : onglet Appareils
</div>

L’onglet Appareils répertorie les appareils pour lesquels l’utilisateur a demandé une authentification. Lorsqu’une autorisation est demandée sur un appareil, celui-ci est associé au compte de l’utilisateur.

Les informations de connexion de l’utilisateur sont associées au <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Refresh+Token">jeton d’actualisation</Tooltip> attribué à cet appareil. Pour révoquer le jeton d’actualisation, cliquez sur **Dissocier** à côté de l’appareil.

<div id="user-details-history-tab">
  ## Détails de l’utilisateur : onglet Historique
</div>

L’onglet Historique affiche un journal de l’activité du compte de l’utilisateur au cours des 2 derniers jours.

Les journaux comprennent des renseignements sur :

* Les événements qui se sont produits
* Le moment où les événements se sont produits
* Les applications associées aux événements
* Le fournisseur d’identité utilisé pour l’authentification
* Les adresses IP d’origine des événements
* L’endroit d’où proviennent les événements

<div id="user-details-locations-tab">
  ## Détails de l’utilisateur : onglet Lieux
</div>

L’onglet Lieux affiche une carte avec des repères indiquant le ou les lieux où l’utilisateur s’est connecté aux applications.

<div id="user-details-raw-json-tab">
  ## Détails de l’utilisateur : onglet Raw JSON
</div>

L’onglet Raw JSON affiche tous les renseignements contenus dans le profil de l’utilisateur au format JSON, afin que vous puissiez consulter rapidement tous les renseignements disponibles à son sujet.
