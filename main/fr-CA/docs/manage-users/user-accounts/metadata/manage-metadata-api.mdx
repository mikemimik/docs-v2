<div id="create-user-metadata">
  ### Créer des métadonnées de l’utilisateur
</div>

Pour créer un utilisateur avec les renseignements de profil suivants :

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing"
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

Effectuez l’appel `POST` suivant vers le point de terminaison [`/post_users`](https://auth0.com/docs/api/management/v2#!/Users/post_users) de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> afin de créer l’utilisateur et de définir les valeurs des propriétés :

<AuthCodeGroup>
  ```bash cURL
  curl --request POST \
    --url 'https://{yourDomain}/api/v2/users' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'content-type: application/json' \
    --data '{"email": "jane.doe@example.com", "user_metadata": {"hobby": "surfing"}, "app_metadata": {"plan": "full"}}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users");
  var request = new RestRequest(Method.POST);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  request.AddHeader("content-type", "application/json");
  request.AddParameter("application/json", "{"email": "jane.doe@example.com", "user_metadata": {"hobby": "surfing"}, "app_metadata": {"plan": "full"}}", ParameterType.RequestBody);
  IRestResponse response = client.Execute(request);
  ```

  ```go Go

  	req, _ := http.NewRequest("POST", url, payload)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
  	req.Header.Add("content-type", "application/json")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```


<div id="update-user-metadata">
  ### Mettre à jour les métadonnées de l’utilisateur
</div>

Vous pouvez mettre à jour les métadonnées d’un utilisateur en envoyant une requête `PATCH` au point de terminaison [`/patch_users_by_id`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) de la Management API.

Supposons que vous ayez créé un utilisateur avec les valeurs de métadonnées suivantes :

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing"
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

Pour mettre à jour `user_metadata` et ajouter l’adresse du domicile de l’utilisateur en tant que propriété de deuxième niveau :

```json lines
{
    "addresses": {
        "home": "123 Main Street, Anytown, ST 12345"
    }
}
```

Vous feriez la requête `PATCH` suivante :

Le profil de l’utilisateur s’affichera maintenant comme suit :

```json lines
{
    "email": "jane.doe@example.com",
    "user_metadata": {
        "hobby": "surfing",
        "addresses": {
            "home": "123 Main Street, Anytown, ST 12345"
        }
    },
    "app_metadata": {
        "plan": "full"
    }
}
```

<Warning>
  Lorsque vous envoyez un appel `PATCH` dans lequel la valeur d’une propriété est définie sur `null` (par exemple, `{user_metadata: {color: null}}`), Auth0 **supprime** la paire propriété/valeur de la base de données. De plus, si vous appliquez un patch aux métadonnées elles-mêmes à l’aide d’un objet vide, les métadonnées sont entièrement supprimées.
</Warning>

<div id="merge-user-metadata">
  ### Fusionner les métadonnées de l’utilisateur
</div>

Seules les propriétés au niveau racine sont fusionnées dans l’objet. Toutes les propriétés des niveaux inférieurs seront remplacées.

Par exemple, pour ajouter l’adresse professionnelle d’un utilisateur comme sous-propriété supplémentaire, vous devez inclure l’intégralité du contenu de la propriété `addresses`. Puisque l’objet `addresses` est une propriété de niveau racine, il sera fusionné dans l’objet JSON final représentant l’utilisateur, mais pas ses sous-propriétés.

```json lines
{
  "user_metadata": {
    "addresses": {
      "home": "123 Main Street, Anytown, ST 12345",
      "work": "100 Industrial Way, Anytown, ST 12345"
    }
  }
}
```

Par conséquent, l’appel `PATCH` correspondant à l’API serait le suivant :

  ```bash cURL
    --data '{"user_metadata": {"addresses": {"home": "123 Main Street, Anytown, ST 12345", "work": "100 Industrial Way, Anytown, ST 12345"}}}'
  ```

```json lines
{
  "user_metadata": {}
```

<div id="create-applications-with-client-metadata">
  ### Créer des applications avec les métadonnées du client
</div>

Un objet `clientMetadata` peut être inclus lors de la création d’une nouvelle application via l’endpoint des applications `POST /api/v2/`.

<div id="read-client-metadata">
  ### Consulter les métadonnées du client
</div>

Les métadonnées du client sont incluses dans les réponses des points de terminaison `GET /api/v2/clients` et `GET /api/v2/client/{id}`.

<div id="update-client-metadata">
  ### Mettre à jour les métadonnées du client
</div>

Les métadonnées du client peuvent être mises à jour au moyen du endpoint [`PATCH /api/v2/clients/{id}`](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id), en fournissant un objet d’application avec la propriété `clientMetadata`, dont la valeur est un objet contenant les métadonnées que vous souhaitez modifier.

**Application auparavant :**

```json lines
{
  ...
  "name": "myclient",
  "client_metadata": {
    "mycolor": "red",
    "myflavor": "grape"
  }
  ...
}
```

Requête : `PATCH /api/v2/client/myclientid123` avec le corps de la requête :

```json lines
{ "client_metadata": { "mycolor": "blue" } }
```

**Application après :**

```json lines
{
  "name": "myclient",
  "client_metadata": {
    "mycolor": "blue",
    "myflavor": "grape"
  }
  ...
}
```

<div id="delete-client-metadata-properties-and-values">
  ### Supprimer des propriétés et des valeurs de métadonnées du client
</div>

Les clés des métadonnées du client peuvent être supprimées en envoyant une requête PATCH, comme il est décrit dans « Update app&#95;metadata » ci-dessus, mais en indiquant `null` comme valeur de la clé. Ce comportement correspond à celui des propriétés `user_metadata` et `app_metadata` du endpoint `PATCH` [/api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).
