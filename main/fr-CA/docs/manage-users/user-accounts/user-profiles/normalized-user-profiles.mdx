description: Découvrez comment Auth0 normalise les propriétés utilisateur courantes dans le profil utilisateur.
title: Profils utilisateur normalisés
---

Le profil utilisateur normalisé est une méthode propre à Auth0 pour standardiser et stocker les renseignements liés aux utilisateurs. Comme chaque <Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">Identity Provider</Tooltip> (IdP) fournit un ensemble différent de renseignements sur un utilisateur, Auth0 normalise les propriétés de profil courantes dans une représentation de l’utilisateur indépendante du protocole lors du stockage des claims liés à l’utilisateur. Par exemple, `family_name` dans le profil utilisateur normalisé contient des renseignements qui peuvent être renvoyés par un IdP sous la forme de `surname` ou `last_name`.

Ainsi, les claims Auth0 inclus dans le profil utilisateur normalisé diffèrent de l’ensemble standard de claims qui peuvent être renvoyés dans les <Tooltip tip="ID Token : justificatif destiné au client lui-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=ID+Tokens">ID Tokens</Tooltip> à partir du [endpoint `oauth/token` de l’Authentication API](https://auth0.com/docs/api/authentication#get-token) ou dans la réponse de l’endpoint [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (qui respectent tous deux les exigences détaillées dans la spécification OIDC).

Pour en savoir plus sur les attributs inclus dans le profil utilisateur normalisé et comprendre comment les utiliser, consultez [Structure du profil utilisateur](/fr-CA/docs/manage-users/user-accounts/user-profiles/user-profile-structure).
