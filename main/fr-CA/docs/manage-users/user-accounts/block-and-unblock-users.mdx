---
title: Comment bloquer et débloquer des utilisateurs en tant qu’administrateur de tenant
sidebarTitle: Bloquer et débloquer des utilisateurs
description: Gérez la possibilité pour un utilisateur de se connecter à votre application en ajoutant ou en supprimant des blocages d’administrateur de tenant.
---

Les administrateurs de tenant peuvent bloquer des utilisateurs pour les empêcher d’accéder aux applications (et les débloquer pour leur redonner l’accès).

Si un utilisateur bloqué essaie de se connecter à une application, il est redirigé vers l’URL de rappel avec les paramètres d’erreur suivants :

```
error=unauthorized&error_description=user%20is%20blocked
```

Cet événement est également consigné dans vos [journaux du tenant](/fr-CA/docs/deploy-monitor/logs/log-event-type-codes) sous la forme d’un événement **Échec de connexion** (`f`) dont la description est `user is blocked`.

<div id="add-tenant-administrator-blocks">
  ## Ajouter des blocages d’administrateur de tenant
</div>

Pour ajouter un blocage d’administrateur de tenant à l’aide de l’Auth0 Dashboard :

1. Accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
2. Sélectionnez le nom de l’utilisateur que vous souhaitez bloquer.
3. Dans l’onglet **Details**, à côté de **Block user**, sélectionnez **Block**.

Cette option est aussi offerte dans le menu **...** à droite du nom de l’utilisateur.

Pour ajouter un blocage d’administrateur de tenant à l’aide de la Management API, utilisez le [Update a User endpoint](/fr-CA/docs/api/management/v2/users/patch-users-by-id) (`PATCH /users/{id}`) pour définir l’attribut [`blocked`](/fr-CA/docs/api/management/v2/users/patch-users-by-id#body-blocked) de l’utilisateur à `true`.

<div id="remove-tenant-administrator-blocks">
  ## Supprimer les blocages d’administrateur de tenant
</div>

Lorsqu’un administrateur de tenant bloque un utilisateur, le blocage n’expire pas automatiquement. Un administrateur de tenant doit débloquer l’utilisateur pour rétablir son accès aux applications.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les blocages liés à la [Brute-force protection](/fr-CA/docs/secure/attack-protection/brute-force-protection) sont gérés séparément des blocages d’administrateur de tenant. Pour en savoir plus, consultez [removing brute-force protection blocks](/fr-CA/docs/secure/attack-protection/brute-force-protection#removing-brute-force-protection-blocks).
</Callout>

Pour supprimer un blocage d’administrateur de tenant dans l’Auth0 Dashboard :

1. Accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
2. Sélectionnez le nom de l’utilisateur visé par le blocage d’administrateur de tenant.
3. Dans l’onglet **Details**, à côté de **Block user**, sélectionnez **Unblock**.

Cette option est aussi accessible dans le menu **...** à droite du nom de l’utilisateur.

Pour supprimer un blocage d’administrateur de tenant à l’aide de la Management API, utilisez le [Update a User endpoint](/fr-CA/docs/api/management/v2/users/patch-users-by-id) (`PATCH /users/{id}`) pour définir l’[`blocked` attribute](/fr-CA/docs/api/management/v2/users/patch-users-by-id#body-blocked) de l’utilisateur à `false`.

Si vous utilisez la Management API pour réinitialiser le mot de passe d’un utilisateur, cela supprime également les blocages d’administrateur de tenant.
