title: Configurer la migration automatique à partir de votre base de données
---

Configurez votre connexion de base de données pour migrer automatiquement vos utilisateurs d’un magasin d’utilisateurs externe vers Auth0 à l’aide de [scripts d’action de base de données personnalisés](/fr-CA/docs/authenticate/database-connections/custom-db/templates).

<Warning>
  Si vous essayez d’utiliser plus d’une méthode de migration (par exemple, la migration automatique suivie de l’importation en bloc d’utilisateurs), vous pourriez obtenir une erreur `DUPLICATED_USER`. Cette erreur indique que l’utilisateur existe dans le magasin d’utilisateurs interne d’Auth0, mais pas dans votre tenant. Pour corriger cette erreur, supprimez l’utilisateur à l’aide du point de terminaison [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de l’Auth0 Management API, puis réessayez l’importation.
</Warning>

<div id="configure-database-connection-as-custom">
  ## Configurer une connexion de base de données comme personnalisée
</div>

Créez une connexion de base de données, puis définissez-la comme personnalisée.

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) et sélectionnez la base de données à consulter.
2. Sélectionnez la vue **Custom Database**, puis activez la bascule **Use my own database**.

   <Frame>![Auth0 Dashboard Authentication Database Connection Custom Database Settings Use Own Database Enabled](/docs/images/cdy7uua7fh8z/11HPAdVwJMmnWbzMVjHCJ8/f75ff47f613a41778f266643f29ab2b1/2025-02-27_17-25-36.png)</Frame>
3. Sélectionnez la vue **Settings**, activez la bascule **Import Users to Auth0**, puis sélectionnez **Save**.

<div id="configure-database-action-scripts">
  ## Configurer les **Database Action Scripts**
</div>

Sélectionnez la vue **Custom Database**, puis repérez **Database Action Scripts**.

<Frame>![Dashboard authentification Database Connection onglet Custom Database Database Action Scripts](/docs/images/cdy7uua7fh8z/4LBIvvjaABo51It4eVCmX3/4ef14765c79cc5b7ecfa3f3155caa9eb/Screen_Shot_2021-05-18_at_8.54.55_PM.png)</Frame>

<div id="login">
  ### Login
</div>

Le script [Login](/fr-CA/docs/authenticate/database-connections/custom-db/templates/login) s’exécute chaque fois qu’un utilisateur qu’Auth0 ne trouve pas tente de se connecter. Il vérifie que l’utilisateur existe dans le magasin d’utilisateurs externe sans lui redemander son mot de passe.

<div id="get-user">
  ### Get User
</div>

Le script [Get User](/fr-CA/docs/authenticate/database-connections/custom-db/templates/get-user) s’exécute après l’un ou l’autre des scénarios suivants :

* Un utilisateur lance une tentative d’inscription.
* Un utilisateur lance un flux de changement de mot de passe.
* Un utilisateur saisit son identifiant de connexion lorsqu’il utilise Identifier First + Biometrics.
* Le point de terminaison [Update a User&#39;s Email or Username](https://auth0.com/docs/api/v2#!/Users/patch_users_by_id) ou [Create User](https://auth0.com/docs/api/management/v2#!/Users/post_users) de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> est appelé.

Si un utilisateur qui n’a pas encore été migré confirme un changement de mot de passe et se connecte avec succès, son profil utilisateur sera créé dans Auth0 avec le nouveau mot de passe. Ce profil utilisateur contient toutes les données renvoyées par le script Get User. Toutes les connexions ultérieures de cet utilisateur se feront directement dans Auth0.

<div id="verify-user-migration-is-complete">
  ## Vérifiez que la migration des utilisateurs est terminée
</div>

Vérifiez que tous les utilisateurs ont été migrés vers l’Auth0 user store à l’aide du <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de l’Auth0 Management API.

1. Accédez à [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users), puis consultez la liste des utilisateurs.
2. Utilisez le point de terminaison [List or Search Users](https://auth0.com/docs/api/v2#!/Users/get_users) de la Management API.

<div id="disconnect-external-user-store">
  ## Déconnecter le magasin d’utilisateurs externe
</div>

Reconfigurez les **Database Action Scripts** en fonctions sans effet. Cela empêchera Auth0 d’interroger votre magasin d’utilisateurs externe pour authentifier les utilisateurs.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Assurez-vous de laisser l’option **Import Users to Auth0** activée. Si vous désactivez cette option, Auth0 utilisera uniquement les scripts pour authentifier les utilisateurs et effectuer d’autres actions sur les utilisateurs, au lieu d’utiliser les utilisateurs importés localement.
</Callout>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database), puis sélectionnez votre connexion de base de données.

2. Passez à la vue **Custom Database**, puis repérez **Database Action Scripts.**

3. Mettez à jour le script **Login**.

   ```javascript lines
   function login (email, password, callback) {
     return callback(null, null);
   }
   ```

4. Mettez à jour le script **Get User**.

   ```javascript lines
   function getByEmail (email, callback) {
     return callback(null, null);
   }
   ```

<div id="troubleshoot-user-migration-issues">
  ## Résoudre les problèmes de migration des utilisateurs
</div>

Si vous rencontrez des problèmes de migration automatique, lisez d’abord [Meilleures pratiques relatives à la connexion de base de données personnalisée et au script d’action](/fr-CA/docs/authenticate/database-connections/custom-db/custom-database-connections-scripts).

<div id="user-already-exists">
  ### L’utilisateur existe déjà
</div>

Voici les scénarios les plus courants dans lesquels vous pourriez rencontrer ce problème :

* Vous supprimez un utilisateur de votre connexion de base de données, puis vous essayez de recréer cet utilisateur.
* Vous tentez de créer un nouvel utilisateur dans votre connexion de base de données alors que cet utilisateur existe déjà dans votre magasin d’utilisateurs externe.
* Le processus de migration de l’utilisateur a été interrompu.

Pendant le processus de migration de l’utilisateur, Auth0 crée d’abord un profil utilisateur partiel dans un magasin d’utilisateurs interne, puis crée un profil utilisateur complet dans votre connexion de base de données. Si un problème survient et empêche la création de ce profil utilisateur complet, vous pourriez obtenir l’erreur `The user already exists`.

Vous pouvez généralement résoudre ce problème en supprimant l’utilisateur de la connexion de base de données, en supprimant l’utilisateur du magasin d’utilisateurs interne, puis en relançant le processus de migration.

1. Vérifiez les instructions `console.log()` dans les [journaux en temps réel des Actions](/fr-CA/docs/customize/actions/actions-real-time-logs).
2. Supprimez l’utilisateur à l’aide du point de terminaison [Delete a User](https://auth0.com/docs/api/management/v2#!/Users/delete_users_by_id) de la Management API.
3. Supprimez l’utilisateur à l’aide du point de terminaison [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de la Management API.
4. Demandez à l’utilisateur de se connecter ou de changer son mot de passe pour relancer la migration.

<div id="missing-metadata">
  ### Métadonnées manquantes
</div>

Si le parcours de connexion ou de changement de mot de passe d’un utilisateur, qui lance le processus de migration, est interrompu, Auth0 pourrait ne pas être en mesure de transférer ses métadonnées (`user_metadata` ou `app_metadata`) avec le reste de ses données de profil.

Vous pouvez atténuer ce scénario en créant une [Action](/fr-CA/docs/customize/actions) qui vérifie si le profil de l’utilisateur ne contient pas de métadonnées, les récupère à partir du magasin d’utilisateurs externe, puis les enregistre dans Auth0.
