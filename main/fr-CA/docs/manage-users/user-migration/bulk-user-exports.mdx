Vous pouvez utiliser le endpoint `POST /api/v2/jobs/users-exports` pour créer une tâche qui exporte tous les utilisateurs associés à une [connexion](/fr-CA/docs/authenticate/identity-providers) ou tous les utilisateurs du tenant.

Pour obtenir la liste des champs du profil utilisateur qui peuvent être exportés, consultez [User Profile Structure](/fr-CA/docs/manage-users/user-accounts/user-profiles/user-profile-structure).

Lorsque vous créez votre tâche, vous devez fournir :

* l’[ID de la connexion](/fr-CA/docs/authenticate/identity-providers/locate-the-connection-id) à partir de laquelle vous souhaitez exporter des utilisateurs (facultatif)
* le format du fichier d’exportation (CSV ou JSON-compatible)
* le nombre maximal d’enregistrements utilisateur à exporter (facultatif ; si ce paramètre est omis, tous les enregistrements seront exportés)
* les champs liés à l’utilisateur (comme le user ID ou le nom) à inclure dans l’exportation

Vous aurez aussi besoin d’un [Management API Access Token](https://auth0.com/docs/api/management/v2/tokens) valide.

<div id="create-a-request-body">
  ## Créer le corps de la requête
</div>

Au besoin, repérez le `connection_id` et le nom de domaine de votre tenant Auth0 dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Créez un nouveau fichier texte contenant le corps de la requête ci-dessous :

```json lines
{
   "connection_id":"connection_id",
   "format":"csv",
   "limit":20,
   "fields":[
      {
         "name":"user_id"
      },
      {
         "name":"email"
      },
      {
         "name":"user_metadata.country"
      }
   ]
}
```

Mettez à jour le `connection_id` avec l’ID de votre connexion de base de données, ou supprimez-le pour exporter tous les utilisateurs du tenant.

<div id="include-user-metadata-in-exported-csv">
  ## Inclure les métadonnées utilisateur dans le CSV exporté
</div>

Si vous exportez des données utilisateur au format CSV et que vous voulez inclure des métadonnées, précisez chaque champ de métadonnées à exporter. Vous pouvez exporter jusqu’à 30 champs.

<Warning>
  Vous ne pouvez pas exporter l’intégralité de `app_metadata` ou de `user_metadata` en CSV. Vous devez préciser explicitement les champs des objets de métadonnées.

  Pour exporter `app_metadata` ou `user_metadata` en tant qu’objets uniques, utilisez le format compatible avec JSON et incluez le champ souhaité dans le paramètre `fields` du corps de la requête. Par exemple :

  `{"name":"app_metadata"}`

  Comme vous ne pouvez exporter qu’un maximum de 30 champs, il est recommandé d’utiliser le format JSON si les données utilisateur comportent de nombreux champs.
</Warning>

Par exemple, pour des métadonnées structurées comme ceci :

```json lines
{
  "consent": {
      "given": true,
      "date": "01/23/2019",
      "text_details": "{yourURL}"
  }
}
```

La requête d’exportation (pour les trois champs) ressemblerait à ceci :

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Dans les fichiers CSV d’exportation des utilisateurs, nous échappons les types de données chaîne de caractères conformément aux normes OWASP visant à atténuer l’injection CSV :

  * Les guillemets doubles sont précédés d’un autre guillemet double.
  * Chaque chaîne est précédée d’une apostrophe.
  * Chaque chaîne est mise entre guillemets doubles.

  Cela ne s’applique pas aux dates générées par Auth0 au format ISO 8601.
</Callout>

<div id="json-compatible-format">
  ### Format compatible avec JSON
</div>

Si vous exportez les données dans un format compatible avec JSON, vous n’avez qu’à fournir la propriété root; il n’est pas nécessaire de nommer chaque propriété interne individuellement, puisqu’elles seront incluses automatiquement.

En raison de leur grande taille, les fichiers d’exportation d’Auth0 utilisent le format [NDJSON](https://github.com/ndjson/ndjson-spec), tandis que la fonction d’importation attend un fichier JSON.

Avant de pouvoir importer des utilisateurs à partir d’une exportation générée par Auth0, vous devrez convertir le fichier de NDJSON en JSON à l’aide de la bibliothèque de votre choix (comme [jq](https://stedolan.github.io/jq/)).

Dans ce cas, pour le même exemple que précédemment, la requête ressemblerait à ceci :

<div id="find-export-data">
  ## Trouver les données exportées
</div>

Vous pouvez accéder à vos fichiers d’exportation à l’aide de l’URL fournie comme valeur du paramètre **location**. Le nom de votre tenant correspond aussi au nom de votre fichier. Par exemple, si le nom de votre tenant est `auth0docs`, votre fichier sera `auth0docs.csv` ou `auth0docs.json`. Lorsque vous accédez à l’URL, le téléchargement du fichier démarre automatiquement.

Le lien de téléchargement est valide pendant 60 secondes. Une fois ce délai écoulé, vous avez 24 heures pour refaire la demande avant l’expiration de la tâche.

<div id="job-cleanup">
  ## Nettoyage des tâches
</div>

Toutes les données associées à vos tâches sont automatiquement supprimées après 24 heures et ne sont plus accessibles ensuite. Par conséquent, **nous vous recommandons fortement d’enregistrer les résultats des tâches dans le système de stockage de votre choix**.

<div id="filter-example">
  ## Exemple de filtre
</div>

Vous voudrez peut-être filtrer votre fichier `.csv` exporté pour n’en conserver qu’un sous-ensemble précis de vos données, comme la date de la dernière connexion des utilisateurs à votre application.

1. Effectuez un appel `POST` au endpoint [Create export user&#39;s job](https://auth0.com/docs/api/management/v2/jobs/post-users-exports) de la Management API pour exporter les utilisateurs vers un fichier `.csv` :

   1. Créez une nouvelle tâche :

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/users-exports' \
      -H 'Content-Type: application/json' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN' \
      -d '{"format":"csv"}'
      ```

   2. Vérifiez le statut :

      ```bash lines
      curl -L 'https://{account.namespace}/api/v2/jobs/JOB_ID' \
      -H 'Accept: application/json' \
      -H 'Authorization: Bearer MGMT_API_TOKEN'
      ```

2. Accédez au fichier dans votre dossier de téléchargements sur votre machine locale.

3. Utilisez un outil en ligne de commande pour ajouter un filtre. Dans cet exemple, nous utilisons [Miller](https://miller.readthedocs.io/en/latest/installing-miller/) :

   ```bash wrap lines
   mlr --csv filter '$last_login >= "2024-01-01T00:00:00Z"' file.csv > filtered.csv
   ```
