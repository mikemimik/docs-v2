L’accès Machine-to-Machine à Organizations peut être accordé en associant directement un client grant à une organisation ou en autorisant l’accès à n’importe quelle organisation dans les paramètres du client grant. Ces deux scénarios peuvent être vérifiés dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou au moyen de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>, où vous pouvez consulter ou récupérer la liste des client grants associés à une organisation pour une application. Pour en savoir plus sur le fonctionnement des permissions d’accès M2M à Organizations, consultez [Configure Your Application for M2M Access](/fr-CA/docs/manage-users/organizations/organizations-for-m2m-applications/configure-your-application-for-m2m-access) et [Authorize M2M Access](/fr-CA/docs/manage-users/organizations/organizations-for-m2m-applications/authorize-m2m-access).

<div id="access-granted-via-direct-association">
  ## Accès accordé par association directe
</div>

Utilisez le [Auth0 Dashboard](https://manage.auth0.com/) et la [Management API](https://auth0.com/docs/api/management/v2) pour vérifier l’accès M2M des client grants directement associés à une organisation.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour afficher dans le Auth0 Dashboard les client grants d’application autorisés pour une organisation donnée :

    1. Accédez à **Organizations** et choisissez l’organisation que vous souhaitez examiner.
    2. Sélectionnez l’onglet **Machine-to-Machine Access**. Vous obtiendrez une liste paginée de toutes les applications qui peuvent accéder à une API pour cette organisation par association directe.
    3. Choisissez une application pour examiner les API autorisées répertoriées pour cette application.

    <Frame>![](/docs/images/cdy7uua7fh8z/5FVJzuu5yq8IHwpQKdYYU6/3468ee757c1e65fb51883213e7e7b476/image2.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Les client grants associés à une organisation peuvent être récupérés au moyen du point de terminaison de l’API [Organization Client Grants](https://auth0.com/docs/api/management/v2/organizations/get-organization-client-grants) :

    <AuthCodeBlock children={codeExample1} language="bash" />

    Sinon, si vous souhaitez obtenir des renseignements sur les applications qui peuvent accéder à au moins une des API d’une organisation, vous pouvez utiliser le query parameter `q` sur le [Get Clients point de terminaison de l’API](https://auth0.com/docs/api/management/v2/clients/get-clients) avec le term de recherche `client_grant.organization_id:{organization_id}`. Utilisez la syntaxe Lucene, semblable à la [User Search Query Syntax](/fr-CA/docs/manage-users/user-search/user-search-query-syntax).

<div id="access-granted-to-any-organization">
  ## Accès accordé à toute organisation
</div>

Utilisez le [Auth0 Dashboard](https://manage.auth0.com/) et la [Management API](https://auth0.com/docs/api/management/v2) pour auditer l’accès M2M des applications ayant accès à toute organisation.

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Accédez à **Organizations**.
    2. Sélectionnez l’onglet **Machine to Machine Access**. Vous obtiendrez une liste paginée de toutes les applications qui peuvent accéder à au moins une API d’une organisation.
    3. Choisissez une application pour examiner les API autorisées associées à cette application.

    <Frame>![](/docs/images/cdy7uua7fh8z/4H3VjVTz0ZGDvV5za9y33m/d0d193df808d7308469ce66d4357b5e7/image1.png)</Frame>
  </Tab>

<div id="search-applications-based-on-organization-access">
  ## Rechercher des applications selon l’accès à l’organisation
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les résultats de recherche présentent une cohérence éventuelle.
</Callout>

Le tableau suivant présente les termes de recherche pris en charge pour interroger les applications à l’aide du paramètre `q` au point de terminaison `/clients` :

| **Champ**                                        | **Description**                                                                                                          |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------ |
| `client_grant.organization_id:{organization_id}` | À utiliser pour rechercher des applications qui peuvent accéder à au moins une des API d’une organisation.               |
| `client_grant.allow_any_organization:true`       | À utiliser pour rechercher des applications qui peuvent accéder à au moins une des API de n’importe quelle organisation. |
