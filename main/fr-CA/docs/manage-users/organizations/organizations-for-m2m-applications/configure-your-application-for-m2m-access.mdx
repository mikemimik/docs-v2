Après avoir créé l’application, configurez-la pour l’accès machine à machine en suivant ces étapes :

1. [Définir le comportement de l’organisation](#define-organization-behavior) pour chaque API à laquelle vous devez accéder.
2. Si l’application ne doit accéder qu’à certaines organisations, c.-à-d. pas à toutes, [Autoriser l’accès M2M](/fr-CA/docs/manage-users/organizations/organizations-for-m2m-applications/authorize-m2m-access) pour chaque API à laquelle vous devez accéder pour ces organisations.

<div id="define-organization-behavior">
  ## Définir le comportement de l’organisation
</div>

Vous pouvez configurer, pour chaque API, la façon dont votre application utilise Organizations dans le cadre du flux Client Credentials, par exemple pour déterminer si l’application doit utiliser une organisation, si elle peut accéder à n’importe quelle organisation ou uniquement à celles auxquelles elle est explicitement associée.

Le tableau suivant décrit les champs permettant de définir le comportement de l’organisation pour l’accès M2M :

<div id="define-organization-behavior-table">
  ### Tableau de définition du comportement des organisations
</div>

| Champ                                               | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           | Correspondance API                                                                                                                                                                                                     |
| --------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Organization Support                                | Détermine comment cette application peut utiliser les organisations lors d’un accès par le Client Credentials Flow.<br /><br />Les options comprennent : <ul> <li>**None:** L’application *ne doit pas* utiliser d’organisations. Il s’agit de la valeur par défaut.</li> <li>**Optional:** L’application *peut choisir* d’utiliser ou non une organisation.</li> <li>**Required:** L’application *doit* utiliser une organisation.</li> </ul>                                                                                                        | Organization Support correspond à `organization_usage`.<br /><br />Options : <ul> <li>**None** correspond à `deny`</li> <li>**Optional** correspond à `allow`</li> <li>**Required** correspond à `required`</li> </ul> |
| Allow machine-to-machine access to any organization | Détermine si cette application peut accéder à n’importe quelle organisation ou si elle est limitée à un ensemble d’organisations explicitement associées lorsqu’elle utilise le Client Credentials Flow. Par défaut, ce comportement n’est pas autorisé.<br /><br />**Avertissement :** Activer l’accès machine à machine à n’importe quelle organisation permet à l’application d’accéder à n’importe quelle organisation sans y être explicitement associée. Cette option ne devrait être activée que pour les applications internes de confiance. | Allow machine-to-machine access to any organization correspond à `allow_any_organization`.<br /><br />Options : <ul> <li>Checked correspond à `true`</li> <li>Unchecked correspond à `false`</li> </ul>                |

<div id="define-organization-behavior-for-an-application">
  ### Définir le comportement de l’organisation pour une application
</div>

Pour définir le comportement de l’organisation pour une application, utilisez le [Auth0 Dashboard](https://manage.auth0.com/) ou la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour une application machine-to-machine, définissez le comportement de l’organisation à partir du Auth0 Dashboard :

    1. Accédez à **Auth0 Dashboard &gt; Applications**, puis sélectionnez l’application pour laquelle vous souhaitez configurer Organizations.
    2. Ouvrez l’onglet **APIs** et développez les détails du `client_grant` pour l’API dont vous voulez configurer l’accès.
    3. Configurez les paramètres appropriés comme indiqué dans le [tableau Définir le comportement de l’organisation](#define-organization-behavior-table).
    4. Sélectionnez **Enregistrer**.

    Sinon, s’il ne s’agit pas d’une application machine-to-machine :

    1. Accédez à **Auth0 Dashboard &gt; APIs**, puis sélectionnez l’API pour laquelle vous souhaitez configurer l’accès à Organizations.
    2. Ouvrez l’onglet **Machine To Machine Applications**. Il affiche toutes les applications, et pas seulement les applications Machine-to-Machine. Repérez votre application.
    3. Pour autoriser l’accès, cliquez sur la bascule de votre application. Cliquez ensuite sur votre application pour développer les détails de configuration.
    4. Configurez les paramètres appropriés comme indiqué dans le [tableau Définir le comportement de l’organisation](#define-organization-behavior-table).
    5. Sélectionnez **Enregistrer**.
  </Tab>

  <Tab title="Management API">
    Vous pouvez aussi définir le comportement de l’organisation pour une application au moyen de la Management API. L’objet `client_grant` contrôle l’accès de votre application à une API; vous devez configurer le `client_grant` pour chaque API à laquelle votre application doit accéder. Pour configurer le `client_grant` pour l’accès M2M, utilisez les endpoints [Create client grant](https://auth0.com/docs/api/management/v2/client-grants/post-client-grants) ou [Update client grant](https://auth0.com/docs/api/management/v2/client-grants/patch-client-grants-by-id), ainsi que le [tableau Définir le comportement de l’organisation](#define-organization-behavior-table), pour mettre à jour les paramètres appropriés.

<div id="set-default-organization">
  ## Définir l’organisation par défaut
</div>

Certains clients ne prennent pas en charge les champs non normalisés dans le [Client Credentials Flow](/fr-CA/docs/get-started/authentication-and-authorization-flow/client-credentials-flow). Ils ne peuvent donc pas envoyer le paramètre `organization` requis à l’endpoint `/oauth/token`. Pour ces clients, vous pouvez définir une organisation par défaut qui s’applique automatiquement à toute requête Client Credentials provenant de l’application lorsque `none` est spécifié et que la prise en charge des organisations est `required` par l’API.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’organisation par défaut s’applique lorsque `organization_usage=require` pour l’API (c.-à-d. l’audience cible).
</Callout>

Le tableau suivant explique les champs à configurer pour définir une organisation par défaut pour l’accès machine à machine :

<div id="set-default-organization-table">
  ### Tableau de définition de l’organisation par défaut
</div>

| Champ                                                  | Description                                                                                              | Correspondance API                                                                                                                                                                                                                                                                                                                                                                                      |
| ------------------------------------------------------ | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Organisation par défaut pour l’accès machine à machine | Définit l’organisation à appliquer aux requêtes client credentials qui ne contiennent pas d’organisation | Organisation par défaut pour l’accès machine à machine correspond à l’objet `default_organization` avec les propriétés suivantes : <ul> <li>`organization_id` : ID de l’organisation à utiliser</li> <li> `flows` : Le flux d’authentification à utiliser pour l’organisation par défaut.<br /><br />Ce paramètre est masqué dans l’UI et sa valeur par défaut est `["client_credentials"]` </li> </ul> |

<div id="set-default-organization-for-an-application">
  ### Définir l’organisation par défaut d’une application
</div>

Pour définir l’organisation par défaut d’une application, utilisez l’[Auth0 Dashboard](https://manage.auth0.com/) ou la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour activer une organisation par défaut pour une application dans l’Auth0 Dashboard :

    1. Accédez à **Auth0 Dashboard &gt; Applications**, puis sélectionnez l’application que vous souhaitez configurer.
    2. Sélectionnez l’onglet **Organizations**.
    3. Configurez les paramètres appropriés, comme indiqué dans le [tableau Définir l’organisation par défaut](#set-default-organization-table).
    4. Cliquez sur **Save.**
