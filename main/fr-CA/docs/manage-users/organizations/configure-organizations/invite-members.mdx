<Card title="La disponibilité varie selon le plan Auth0">
  La disponibilité de cette fonctionnalité dépend à la fois de votre implémentation de login et de votre plan Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Card>

Si vous souhaitez attribuer un membre à une [Organization](/fr-CA/docs/manage-users/organizations/organizations-overview), mais que l’utilisateur n’existe pas encore dans votre banque de données (ou qu’il existe déjà dans votre banque de données, sans être encore membre de votre Organization), vous pouvez l’inviter à l’Organization. Dans ce cas, l’utilisateur recevra un courriel contenant un lien qui lui permettra de créer un compte ou de se connecter et de joindre l’Organization, éventuellement avec des rôles prédéfinis, puis d’être redirigé vers votre application afin de pouvoir l’utiliser immédiatement.

<Warning>
  Les rôles des membres d’une Organization sont distincts des rôles attribués aux utilisateurs au moyen de la fonctionnalité [Role-Based Access Control](/fr-CA/docs/manage-users/access-control/rbac) d’Auth0. Lorsqu’un rôle prédéfini est appliqué à un utilisateur qui accepte une invitation à une Organization, ce rôle s’applique uniquement lorsque l’utilisateur se connecte par l’entremise de cette Organization. Vous pouvez [voir les rôles des membres d’une Organization](/fr-CA/docs/manage-users/organizations/configure-organizations/retrieve-member-roles) dans le Dashboard.
</Warning>

Pour configurer le flux d’invitation d’adhésion :

1. [Intégrer le flux d’invitation d’adhésion à votre application](#integrate-the-membership-invitation-flow-into-your-application)
2. [Personnaliser le flux d’invitation utilisateur](#customize-the-user-invitation-flow)
3. [Envoyer des invitations d’adhésion](#send-membership-invitations)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Limites**

  * Auth0 peut envoyer des invitations utilisateur uniquement par courriel. Si vous préférez envoyer l’invitation autrement, vous pouvez choisir de générer uniquement une URL d’invitation.
  * Les utilisateurs invités doivent se connecter ou créer un compte avec l’adresse courriel à laquelle l’invitation a été envoyée.
</Callout>

<div id="integrate-the-membership-invitation-flow-into-your-application">
  ## Intégrez le flux d’invitation d’adhésion à votre application
</div>

Pour que les invitations d’utilisateurs fonctionnent, vous devez configurer une route dans votre application dans le cadre du processus d’acceptation de l’invitation et [configurer une route de connexion par défaut associée](/fr-CA/docs/authenticate/login/auth0-universal-login/configure-default-login-routes) pour votre tenant ou votre application. Un lien vers l’URI configuré sera inclus dans le courriel d’invitation envoyé aux utilisateurs.

<div id="configure-the-uri">
  ### Configurer l’URI
</div>

Si l’URI doit être partagée entre toutes vos applications, vous pouvez définir l’**URL de connexion du locataire** dans [Auth0 Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced). Sinon, vous pouvez définir l’URI pour chaque application à l’aide du champ **Application Login URI** dans les paramètres de votre application, auxquels vous pouvez accéder depuis [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).

<div id="specify-route-behavior">
  ### Préciser le comportement de la route
</div>

La route de votre application doit accepter les paramètres `invitation` et `organization` dans la chaîne de requête, puis appeler le point de terminaison [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de l’API d’authentification Auth0 avec ces deux paramètres.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour prendre en charge les scénarios multilocataires où le nom de l’Organization est utilisé comme sous-domaine ou variable de chemin d’accès dans votre application (par exemple, `acme.myapp.com`), le paramètre `organization_name` est également inclus dans le lien d’invitation envoyé aux utilisateurs. Il n’est pas nécessaire d’envoyer ce paramètre au point de terminaison [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de l’API d’authentification Auth0.
</Callout>

Par exemple, si vous avez une application activée pour les Organizations avec une **Application Login URI** définie sur `https://myapp.com/login`, le lien dans l’invitation par courriel envoyée à l’utilisateur final serait : `https://myapp.com/login?invitation={invitation_ticket_id}&organization={organization_id}&organization_name={organization_name}`.

<div id="customize-the-user-invitation-flow">
  ## Personnaliser le processus d’invitation des utilisateurs
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour personnaliser le processus d’invitation des utilisateurs, vous devez utiliser des domaines personnalisés. Les forfaits Free comprennent un domaine personnalisé gratuit ; des domaines personnalisés supplémentaires sont offerts avec n’importe quel forfait payant.
</Callout>

Pour personnaliser l’écran d’invitation et le courriel reçu par l’utilisateur, modifiez l’<Tooltip tip="Universal Login: Your application redirects to Universal Login, hosted on Auth0's Authorization Server, to verify a user's identity." cta="View Glossary" href="/fr-CA/docs/glossary?term=Universal+Login">invite Universal Login</Tooltip> et les modèles de courriel. Pour en savoir plus, consultez les pages suivantes :

* [Modèles de page Universal Login](/fr-CA/docs/customize/login-pages/universal-login/customize-templates)
* [Internationalisation d’Universal Login](/fr-CA/docs/customize/internationalization-and-localization/universal-login-internationalization)
* [Personnaliser les modèles de courriel](/fr-CA/docs/customize/email/email-templates)

<div id="localization">
  ### Localisation
</div>

Comme la langue de l’utilisateur invité n’est pas connue d’avance, vous pouvez envoyer un paramètre `user_metadata` facultatif au point de terminaison d’invitation à l’aide de la <Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>. Lors de la localisation du processus d’invitation des utilisateurs, vous pouvez accéder à ce paramètre à partir du modèle de courriel **Invitation d’utilisateur** et [utiliser la même logique que pour d’autres modèles de courriel multilingues](/fr-CA/docs/customize/email/email-templates).

Pour que l’utilisateur invité voie l’invitation dans la langue appropriée, votre application devra envoyer le paramètre `ui_locales`, ou le navigateur de l’utilisateur invité devra envoyer l’en-tête `Accept-Language` approprié. Pour en savoir plus, consultez [Personnaliser les modèles de courriel : modèles de courriel multilingues](/fr-CA/docs/customize/email/email-templates).

<div id="send-membership-invitations">
  ## Envoyer des invitations d’adhésion
</div>

Vous pouvez [envoyer des invitations d’adhésion à une Organization](/fr-CA/docs/manage-users/organizations/configure-organizations/send-membership-invitations) aux utilisateurs à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la Management API. Si vous préférez envoyer l’invitation au moyen de votre propre service de courriel, vous pouvez choisir de générer uniquement une URL d’invitation. Sinon, l’utilisateur recevra un courriel contenant le lien, ce qui lui permettra d’ouvrir une session dans l’Organization au moyen d’une connexion configurée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Lorsqu’un utilisateur accepte une invitation, son adresse courriel sera marquée comme vérifiée.

  Si l’utilisateur accepte une invitation au moyen d’une connexion fédérée, le fournisseur d’identité doit renvoyer l’adresse courriel correspondant à celle à laquelle l’invitation a été envoyée. La vérification de l’adresse courriel ne sera pas remplacée par la valeur renvoyée par le fournisseur d’identité lors des connexions ultérieures.

  Si l’utilisateur n’accepte pas une invitation avant son expiration, l’invitation sera marquée comme expirée et finira par être supprimée de l’[Auth0 Dashboard](/fr-CA/docs/get-started/auth0-overview/dashboard).
</Callout>
