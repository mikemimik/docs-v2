<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Pour créer une organisation à l’aide de l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations).
2. Sélectionnez **Create Organization**.
3. Entrez les renseignements de base de votre organisation, puis sélectionnez **Add Organization** :

   | **Champ**        | **Description**                                                                                                                                                                                                                                                                                                                                                                                                        |
   | ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Name**         | Nom de l’organisation que vous souhaitez créer. Il s’agit du nom qu’un utilisateur final saisirait dans l’invite de préconnexion pour indiquer l’organisation par laquelle il veut se connecter. Identifiant logique unique. Peut contenir des lettres minuscules, des chiffres, des traits de soulignement (`_`) et des traits d’union (`-`). Peut commencer par un chiffre. Doit comporter entre 1 et 50 caractères. |
   | **Display Name** | Nom convivial à afficher.                                                                                                                                                                                                                                                                                                                                                                                              |
4. Repérez la section **Branding** et personnalisez votre organisation, puis sélectionnez **Save changes** :

   | **Champ**                 | **Description**                                                                                           |
   | ------------------------- | --------------------------------------------------------------------------------------------------------- |
   | **Organization Logo**     | Logo à afficher. La résolution minimale recommandée est de 200 pixels (largeur) sur 200 pixels (hauteur). |
   | **Primary Color**         | Couleur des éléments principaux.                                                                          |
   | **Page Background Color** | Couleur de l’arrière-plan.                                                                                |
5. Repérez la section **Metadata** et ajoutez à l’organisation toutes les paires clé-valeur de métadonnées nécessaires, puis sélectionnez **Add**.

<div id="management-api">
  ## Management API
</div>

Pour créer une organisation au moyen de la Management API :

Effectuez une requête `POST` vers l’endpoint `Create Organizations`. Assurez-vous de remplacer les valeurs fictives par les valeurs appropriées de votre tenant. Pour en savoir plus, consultez le tableau des paramètres ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Si votre domaine Auth0 correspond au nom de votre tenant, il s’agit de votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), suivi de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir du nom de votre domaine personnalisé.
</Callout>

| **Valeur**                 | **Description**                                                                                                                                                                                                                                                                                                                                                                                                    |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MGMT_API_ACCESS_TOKEN`    | [Access Token for the Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `create:organizations`.                                                                                                                                                                                                                                                                       |
| `ORG_NAME`                 | Nom de l’organisation que vous souhaitez créer. Il s’agit du nom qu’un utilisateur final saisirait dans l’invite de préconnexion pour indiquer l’organisation avec laquelle il souhaite se connecter. Identifiant logique unique. Peut contenir des lettres minuscules, des chiffres, des traits de soulignement (`_`) et des tirets (`-`). Peut commencer par un chiffre. Doit contenir entre 1 et 50 caractères. |
| `ORG_DISPLAY_NAME`         | Facultatif. Nom convivial de l’organisation pouvant être affiché dans le flux de connexion et les modèles de courriel.                                                                                                                                                                                                                                                                                             |
| `ORG_LOGO`                 | Facultatif. URL du logo de l’organisation.                                                                                                                                                                                                                                                                                                                                                                         |
| `ORG_PRIMARY_COLOR`        | Facultatif. Code de couleur HEX pour les éléments principaux.                                                                                                                                                                                                                                                                                                                                                      |
| `ORG_BACKGROUND_COLOR`     | Facultatif. Code de couleur HEX pour l’arrière-plan.                                                                                                                                                                                                                                                                                                                                                               |
| `KEY`/`VALUE`              | Facultatif. Paires clé-valeur de type String représentant les métadonnées de l’organisation. Maximum de 255 caractères chacune. Maximum de 25 paires de métadonnées.                                                                                                                                                                                                                                               |
| `CONNECTION_ID`            | Facultatif. ID de la connexion que vous souhaitez activer pour l’organisation indiquée. Les connexions activées s’affichent dans l’invite de connexion de l’organisation afin que les utilisateurs puissent accéder à votre ou vos applications par leur intermédiaire.                                                                                                                                            |
| `ASSIGN_MEMBERSHIP_OPTION` | Facultatif. Indique si vous souhaitez que les utilisateurs qui se connectent avec la connexion activée obtiennent automatiquement un statut de membre dans l’organisation indiquée. Lorsque la valeur est `true`, les utilisateurs obtiennent automatiquement ce statut. Lorsque la valeur est `false`, ils ne l’obtiennent pas automatiquement.                                                                   |

| --------------- | ----------------------- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| `201`           |                         | L’organisation a été créée avec succès.                                                                |                                                                                                      |
| `400`           | `invalid_body`          | Corps de la requête invalide. Le message varie selon la cause.                                         | La charge utile de la requête n’est pas valide.                                                      |
| `400`           | `invalid_query_string`  | Chaîne de requête invalide. Le message varie selon la cause.                                           | La chaîne de requête n’est pas valide.                                                               |
| `401`           |                         | Jeton invalide.                                                                                        |                                                                                                      |
| `401`           |                         | Signature invalide reçue pour la validation du JSON Web Token.                                         |                                                                                                      |
| `401`           |                         | Le client n’est pas global.                                                                            |                                                                                                      |
| `403`           | `insufficient_scope`    | Portée insuffisante ; une des portées suivantes est requise : `create:organizations`.                  | Tentative de lecture ou d’écriture d’un champ non autorisée avec les portées du bearer token fourni. |
| `409`           | `organization_conflict` | Une organisation portant le même nom existe déjà.                                                      | Une organisation portant le même nom existe déjà.                                                    |
