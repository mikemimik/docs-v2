Vous pouvez activer des connexions précises pour chaque [organisation](/docs/fr-CA/manage-users/organizations/organizations-overview) afin d’offrir aux utilisateurs différentes options de connexion. Après avoir activé une connexion, elle est ajoutée à l’écran de connexion de l’organisation, et les utilisateurs peuvent s’authentifier au moyen de cette connexion pour accéder à vos applications.

Pour activer une connexion pour une organisation, la connexion **doit** déjà exister dans votre tenant. Les connexions prises en charge comprennent les [connexions de base de données](/docs/fr-CA/authenticate/database-connections), les [connexions sociales](/docs/fr-CA/authenticate/identity-providers/social-identity-providers) et les [connexions d’entreprise](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers).

<div id="organization-properties">
  ## Propriétés de l’organisation
</div>

Lorsque vous utilisez des organizations, certaines connections comportent des propriétés supplémentaires que vous pouvez configurer :

| **Propriété**                                        | **Types de connexion**                   | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ---------------------------------------------------- | ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Nom de la connexion de l’organisation                | Tous les types de connexion              | Ce nom est fourni par l’administrateur de l’organisation à l’aide de la [My Organization API](/docs/fr-CA/manage-users/my-organization-api). S’il n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux administrateurs de l’organisation lorsqu’ils consultent cette connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Niveau d’accès des administrateurs de l’organisation | Tous les types de connexion              | Cette propriété détermine le niveau d’accès dont disposent les administrateurs de l’organisation pour cette connexion. Les valeurs possibles sont les suivantes : <ul><li>**Aucun** : géré par l’administrateur du tenant, non visible et non modifiable à l’aide de la My Organization API.</li> <li>**Lecture seule** : géré par l’administrateur du tenant, visible à l’aide de la My Organization API.</li> <li>**Limité** : les administrateurs de l’organisation peuvent modifier certains paramètres non sensibles à l’aide de la My Organization API.</li> <li>**Complet** : les administrateurs de l’organisation peuvent modifier tous les paramètres autorisés pour cette connexion, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile).</li></ul>                                                           |
| Appartenance lors de l’authentication                | Tous les types de connexion              | Lorsqu’elle est activée, cette propriété attribue automatiquement une Membership à l’organisation aux end-users la première fois qu’ils s’authentifient avec la connexion. L’appartenance lors de l’authentication est utile dans les scénarios où l’on peut présumer que tous les utilisateurs en mesure de s’authentifier avec une connexion donnée sont membres d’une organisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| Inscription à l’organisation                         | Connexions de base de données uniquement | Cette propriété détermine si les end-users peuvent accéder à un lien d’inscription dans l’écran de connexion, leur permettant d’obtenir une Membership à une organisation. Pour utiliser l’inscription à l’organisation, vous devez aussi activer Membership On Authentication. L’inscription à l’organisation est utile dans les scénarios où les utilisateurs ont besoin d’un accès libre-service pour rejoindre des organizations, par exemple : <ul><li>Lorsque les organisations servent à modéliser des groupes d’utilisateurs à adhésion ouverte, comme des espaces de travail.</li> <li>Dans les use cases interentreprises à consommateur (B2B2C) où les utilisateurs peuvent librement créer un compte.</li></ul>                                                                                                                                                                                |
| Afficher la connexion sous forme de bouton           | Connexions d’entreprise uniquement       | Facultatif. Cette propriété détermine si une connexion donnée s’affiche ou non comme option dans l’écran de connexion de l’organisation. Les end users peuvent aussi s’authentifier à l’aide du profil d’authentication [Identifier First with Home Realm Discovery](/docs/fr-CA/authenticate/login/auth0-universal-login/identifier-first), en combinaison avec le login flow d’organisation [Prompt for Credentials](/docs/fr-CA/manage-users/organizations/login-flows-for-organizations#configure-the-login-flow-for-your-application). Cela fonctionne uniquement si vous ne fournissez **pas** de `org_id` avec la authorization request.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Si cette option est désactivée, les end-users peuvent tout de même s’authentifier au moyen de la connexion en envoyant directement le paramètre de connexion dans la authorization request. </Callout> |

<div id="configure-organization-connections">
  ## Configurer les connexions d’organisation
</div>

Vous pouvez configurer les connexions des organisations à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    #### Auth0 Dashboard

    Pour activer une connection dans Auth0 Dashboard :

    1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations), puis sélectionnez l’Organization que vous souhaitez configurer.
    2. Sélectionnez l’onglet **Connections**, puis **Add Connections**.
    3. Choisissez la connection que vous souhaitez activer, puis sélectionnez **Add Connection**.
    4. Dans la section Organization Connection, configurez les deux properties requises pour utiliser une connection via la My Organization API :

    * Repérez **Organization Connection Name**. Si ce champ n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux Organization Admins lorsqu’ils consultent cette connection.
    * Repérez **Organization Admin Access Level** et, si ce champ n’est pas renseigné, choisissez le niveau d’accès que vous souhaitez accorder aux Organization Admins pour cette connection via la My Organization API.

    5. Dans la section Authentication, repérez **Membership On Authentication** et choisissez d’activer ou de désactiver l’auto-membership. Lorsqu’elle est activée, l’auto-membership ajoute automatiquement à l’Organization tous les users qui se connectent avec cette connection à titre de members.
    6. **Pour les database connections seulement** : Dans la section Organization Signup, choisissez d’activer ou de désactiver les inscriptions en self-service. Lorsqu’elle est activée, les users peuvent accéder à un lien d’inscription dans le login prompt pour créer leur compte et obtenir automatiquement une membership à l’Organization.

       * Pour activer cette property, vous devez d’abord activer **Membership on Authentication**.
       * **Remarque** : Organization Signup a priorité sur la configuration d’inscription de la Database connection. Lorsque Organization Signup est activé, les users peuvent quand même s’inscrire à une Organization ou accepter une invitation à une Organization, même si les inscriptions sont désactivées au niveau de la Database connection.
    7. **Pour les enterprise connections seulement** : Dans la section Connection button, activez au besoin la property **Display connection as a button** pour afficher la connection comme option dans le Organization login prompt.

       <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
         Si toutes les connections activées dans l’Organization sont des enterprise connections et qu’elles sont toutes masquées, Auth0 renvoie une erreur indiquant `Message: no connections enabled for the organization are visible` lorsque les users accèdent à l’application.
       </Callout>
    8. Sélectionnez **Save**.
  </Tab>

  <Tab title="Management API">
    #### Management API

    Pour activer une connexion via la Management API :

    Effectuez un appel `POST` vers le point de terminaison `Create Organization Connections`. Assurez-vous de remplacer les valeurs fictives suivantes par les informations appropriées :

    * Remplacez `{orgId}` par votre ID d’organisation.
    * Remplacez `{mgmtApiAccessToken}` par votre jeton d’accès à l’API de gestion.
    * Remplacez `{connectionId}` par l’ID d’une connexion précise.
    * Remplacez `{assignMembershipOption}` par `true` ou `false` selon votre sélection Membership on Authentication.
    * Remplacez `{isEnabled}` par `true` ou `false`, selon que vous souhaitez activer la connexion ou non.
    * **Pour les connexions de base de données seulement** : remplacez `{isSignupEnabled}` par `true` ou `false` selon votre sélection d’inscription.
    * **Pour les connexions d’entreprise seulement** : remplacez `{showAsButtonOption}` par `true` ou `false` selon votre sélection Connection Button.
    * **Pour utiliser My Organization API** : remplacez `{organizationAccessLevel}` et `{organizationConnectionName}` par les valeurs appropriées.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Trouvez votre domaine Auth0**

      Si votre domaine Auth0 est formé du nom de votre tenant, de votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), suivi de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et avait été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

      Si vous utilisez des [domaines personnalisés](/docs/fr-CA/customize/custom-domains), il doit s’agir de votre nom de domaine personnalisé.

    | `{orgId}`                      | ID de l’Organization pour laquelle vous souhaitez activer une connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
    | `{mgmtApiAccessToken}`         | [Jeton d’accès pour l’API de gestion](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) doté de la portée `create:organization_connections`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
    | `{connectionId}`               | ID de la connexion que vous souhaitez activer pour l’Organization indiquée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
    | `{assignMembershipOption}`     | Indique si vous voulez que les utilisateurs qui se connectent avec cette connexion obtiennent automatiquement l’appartenance à l’Organisation. Lorsque cette option est définie sur `true`, l’appartenance leur est accordée automatiquement. Lorsqu’elle est définie sur `false`, elle ne leur est pas accordée automatiquement.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
    | `{isEnabled}`                  | Permet d’activer ou de désactiver une connexion pour une Organization. En désactivant une connexion, il est possible de conserver les valeurs de `show_as_button`, `assign_membership_on_login`, etc., tout en laissant la connexion désactivée.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
    | `{isSignupEnabled}`            | Détermine si les utilisateurs peuvent accéder à un lien d’inscription libre-service dans le prompt de connexion. Lorsque cette valeur est définie à `true`, le lien d’inscription s’affiche dans le prompt. Lorsqu’elle est définie à `false`, le lien reste masqué. **Remarque** : pour activer cette option, vous devez aussi définir `{assignMembershipOption}` à `true`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
    | `{showAsButtonOption}`         | Indique si vous souhaitez qu’une connexion Enterprise précise s’affiche comme option sur l’invite de connexion de l’organisation. Lorsque cette option est définie sur `true`, la connexion s’affiche sous forme de bouton sur l’invite. Lorsqu’elle est définie sur `false`, la connexion est masquée sur l’invite.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
    | `{organizationAccessLevel}`    | Détermine le type d’accès dont disposera un administrateur d’Organization lorsqu’il utilise l’API My Organization. Lorsqu’elle est réglée sur `none`, la connection sera gérée par l’administrateur du tenant et ne sera ni visible ni modifiable via l’API My Organization. Lorsqu’elle est réglée sur `read-only`, la connection sera gérée par l’administrateur du tenant et sera visible via l’API My Organization, mais ne pourra pas être modifiée. Lorsqu’elle est réglée sur `limited`, les administrateurs d’Organization peuvent modifier certains paramètres sélectionnés non sensibles via l’API My Organization. Lorsqu’elle est réglée sur `full`, les administrateurs d’Organization peuvent modifier tous les paramètres autorisés pour cette connection, sous réserve des restrictions définies par le [Connection Profile](/docs/fr-CA/authenticate/enterprise-connections/connection-profile). |
    | `{organizationConnectionName}` | Ce nom est fourni par l’Admin de l’Organization au moyen de la My Organization API. S’il n’est pas renseigné, indiquez le nom que vous souhaitez afficher aux Admins de l’Organization lorsqu’ils consultent cette connexion.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |

    ##### Codes de statut de réponse

    Les codes de statut de réponse possibles sont les suivants :

    | **Code d’état** | **Code d’erreur**    | **Message**                                                                                            | **Cause**                                                                                          |
    | --------------- | -------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
    | `201`           |                      | Connexion ajoutée avec succès à une organisation.                                                      |                                                                                                    |
    | `401`           |                      | Jeton non valide.                                                                                      |                                                                                                    |
    | `401`           |                      | Signature non valide reçue lors de la validation du JSON Web Token.                                    |                                                                                                    |
    | `401`           |                      | Le client n’est pas global.                                                                            |                                                                                                    |
    | `403`           | `insufficient_scope` | Scope insuffisant ; l’une des valeurs suivantes est requise : `create:organizations_connections`.      | Tentative de lecture ou d’écriture d’un champ non autorisé par les portées du jeton Bearer fourni. |
    | `429`           |                      | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                    |
