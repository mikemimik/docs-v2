Lorsque vous représentez votre application dans Auth0, vous pouvez préciser quels types d’utilisateurs elle doit prendre en charge. Certaines applications permettent à des particuliers de se connecter avec des comptes personnels, tandis que d’autres sont destinées aux membres d’[Organizations](/docs/fr-CA/manage-users/organizations/organizations-overview). D’autres doivent prendre en charge les deux. C’est ce qu’on appelle le comportement de l’organisation, et il peut être défini pour chaque application que vous connectez à Auth0.

Par exemple, votre application pourrait avoir :

* Une page de destination marketing générique avec un bouton **Se connecter** qui amène vos utilisateurs vers le flux de connexion Auth0 sans Organization.
* Une URL distincte pour chacun de vos clients B2B (par exemple, les utilisateurs d’Acme vont à `acme.yourcompany.com`) qui redirige les utilisateurs vers Auth0 avec une Organization, afin qu’ils voient le bouton de connexion <Tooltip tip="Single Sign-On (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip> d’Acme.

Vous pouvez définir le comportement de l’organisation pour prendre en charge l’un ou l’autre de ces scénarios. De plus, vous pouvez le configurer de sorte que, si votre application exige qu’une Organization soit fournie, mais que votre utilisateur est accidentellement envoyé vers Auth0 sans Organization, il verra une invite lui permettant de saisir le nom de son Organization.

Vous pouvez définir le comportement de l’organisation à l’aide du <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.&quot;

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Pour définir le comportement de l’organisation dans l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/#/applications), puis sélectionnez l’application pour laquelle vous souhaitez configurer les organizations.
2. Passez à la vue Login Experience et configurez les paramètres appropriés :

| Champ                            | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           | Mappage API                                                                                                                                                                                                                                                                           |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Type d’utilisateurs**          | Détermine quelle catégorie d’utilisateurs peut se connecter à votre application.<br /><br />Les options comprennent : <ul> <li> **Individuals** : les utilisateurs peuvent s’inscrire avec un compte personnel et se connecter directement à votre application. Les utilisateurs individuels ne peuvent pas se connecter à l’aide d’une Organization. </li> <li> **Business Users** : les utilisateurs doivent être membres d’une Organization pour pouvoir se connecter. Lorsque cette option est sélectionnée, vous devez soit fournir une Organization lorsque vous redirigez les utilisateurs vers le point de terminaison `/authorize`, soit définir votre flux de connexion sur Prompt for Organization. </li> <li> **Both** : les utilisateurs peuvent se connecter comme membres d’une Organization ou s’inscrire avec un compte personnel. </li> </ul>          | **Type d’utilisateurs** correspond à `organization_usage`<br /><br />Options : <ul> <li>**Individuals** correspond à `deny`</li> <li>**Business Users** correspond à `require`</li> <li>**Both** correspond à `allow`</li> </ul>                                                      |
| **flux de connexion**                   | Détermine le login prompt initial présenté aux utilisateurs lorsqu’ils accèdent à votre application. Vous ne pouvez configurer ce champ que si Type of Users est défini sur Business Users ou Both.<br /><br />Les options comprennent : <ul> <li> **Prompt for Credentials** : les utilisateurs sont d’abord invités à fournir leurs credentials de connexion. Après s’être connectés, ils peuvent sélectionner leur Organization. </li> <li> **Prompt for Organization** : les utilisateurs sont d’abord invités à sélectionner leur Organization. Ils peuvent ensuite fournir leurs credentials pour se connecter. </li> <li> **No Prompt** : Auth0 ne détermine pas quel login prompt est présenté aux utilisateurs. C’est plutôt votre application qui doit envoyer à Auth0 les paramètres requis pour afficher le prompt approprié. </li> </ul> | **flux de connexion** correspond à `organization_require_behavior`<br /><br />Options : <ul> <li>**Prompt for Credentials** correspond à `post_login_prompt`</li> <li>**Prompt for Organization** correspond à `pre_login_prompt`</li> <li>**No Prompt** correspond à `no_prompt`</li> </ul> |
| **Découverte de l’organisation** | Définit la méthode utilisée pour identifier l’organisation de l’utilisateur avant l’authentification. Disponible uniquement avec le flux de connexion Prompt for Organization.<br /><br />Les options comprennent : <ul> <li> **Prompt for Organization Email** : les utilisateurs sont d’abord invités à fournir l’adresse courriel associée à leur Organization. </li> <li> **Prompt for Organization Name** : les utilisateurs sont d’abord invités à fournir le nom de leur Organization. </li> <li> **Prompt for Organization Email and Name** : les utilisateurs sont d’abord invités à fournir le nom de leur Organization ou l’adresse courriel associée à leur Organization. </li> </ul>                                                                                                                                                              | <ul> <li> **Prompt for Emails** : correspond à `email`. </li> <li> **Prompt for Organization** : correspond à `organization_name`. </li> <li> **Prompt for Organization Email and Name** : correspond à `email` et `organization_name`. </li> </ul>                                   |

3. Sélectionnez **Enregistrer les modifications**.&quot;

<div id="management-api">
  ## Management API
</div>

Effectuez une requête `PATCH` vers le [point de terminaison Update a Client](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id). Assurez-vous de remplacer les valeurs d’exemple `client_id`, `mgmt_api_access_token`, `organization_usage` et `organization_require_behavior` par votre <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=client+ID">ID client</Tooltip>, le <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, l’option d’utilisation de l’organisation et l’option de comportement de l’organisation, respectivement.

| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CLIENT_ID`                      | ID de l’application pour laquelle vous souhaitez ajouter un comportement de l’organisation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `MGMT_API_ACCESS_TOKEN`          | [Jetons d’accès pour la Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `update:clients`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `ORGANIZATION_USAGE`             | Indique si votre application peut prendre en charge des utilisateurs qui se connectent à une organisation. Les options comprennent : <ul> <li>`deny` : (Par défaut) Les utilisateurs ne peuvent pas se connecter au moyen d’une organisation.</li> <li>`allow` : Les utilisateurs peuvent se connecter avec ou sans organisation. Lorsque cette option est sélectionnée, vous devez fournir une organisation lorsque vous redirigez les utilisateurs vers le point de terminaison `/authorize`.</li> <li>`require` : Les utilisateurs doivent se connecter au moyen d’une organisation. Lorsque cette option est sélectionnée, vous devez soit fournir une organisation lorsque vous redirigez les utilisateurs vers le point de terminaison `/authorize`, soit définir `organization_require_behavior` à `pre_login_prompt` pour permettre aux utilisateurs de choisir une organisation avant de se connecter.</li> </ul> |
| `ORGANIZATION_REQUIRE_BEHAVIOR`  | Détermine le flux de connexion présenté aux utilisateurs qui accèdent à votre application. S’applique uniquement lorsque `organization_usage` est défini à `require` ou `allow`.<br /><br />Les options comprennent : <ul> <li>`no_prompt` : (Par défaut) N’affiche aucune invite. Les demandes sans paramètre d’organisation valide sont rejetées.</li> <li>`pre_login_prompt` : Invite les utilisateurs à sélectionner une organisation avant de pouvoir se connecter. Vous ne pouvez utiliser cette option que si `organization_usage` est défini à `require`.</li> <li>`post_login_prompt` : Invite les utilisateurs à se connecter avec leurs identifiants. Après la connexion, invite les utilisateurs à sélectionner leur organisation.</li> </ul>                                                                                                                                                                  |
| `ORGANIZATION_DISCOVERY_METHODS` | Détermine la méthode à utiliser pour identifier l’organisation d’un utilisateur avant l’authentification. S’applique uniquement lorsque `organization_require_behavior` est défini à `pre_login_prompt` et que `organization_usage` est défini à `require` ou `allow`. <ul> <li> `email` : (Par défaut) Les utilisateurs sont invités à saisir l’adresse courriel associée à leur organisation. </li> <li> `organization_name` : Les utilisateurs sont invités à saisir le nom associé à leur organisation. </li> <li> `Organization name` and `email` : Les utilisateurs sont invités à saisir l’adresse courriel ou le nom associé à leur organisation. </li> </ul>                                                                                                                                                                                                                                                      |&quot;
