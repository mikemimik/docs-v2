Lorsque vous travaillez avec les [organisations](/fr-CA/docs/manage-users/organizations/organizations-overview) par programmation, vous pourriez avoir besoin de récupérer la liste des organisations auxquelles un utilisateur a une adhésion.

<div id="management-api">
  ## Management API
</div>

<Warning>
  Jusqu’à 1000 membres d’organisation peuvent être affichés dans l’Auth0 Dashboard ou au moyen de la Management API, bien qu’il puisse en exister davantage.
</Warning>

Effectuez une requête `GET` vers le point de terminaison `Get User Organizations`. Assurez-vous de remplacer les valeurs d’espace réservé `USER_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID utilisateur et votre <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Jeton d’accès : information d’autorisation d’accès, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Access+Token">jeton d’accès</Tooltip>, respectivement.

  curl --request GET \

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Votre domaine Auth0 se compose du nom de votre tenant, de votre sous-domaine régional (sauf si votre tenant se trouve dans la région des États-Unis et a été créé avant juin 2020), puis de `.auth0.com`. Par exemple, si le nom de votre tenant était `travel0`, votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre tenant se trouvait aux États-Unis et a été créé avant juin 2020, votre nom de domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, utilisez le nom de votre domaine personnalisé.
</Callout>

| Valeur | Description |
| `USER_ID` | Identifiant de l’utilisateur dont vous souhaitez récupérer l’adhésion à une organisation. |
| `MGMT_API_ACCESS_TOKEN` | [jeton d’accès for the Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) avec la portée `read:organizations`. |

<div id="response-status-codes">
  ### Codes d’état de la réponse
</div>

Les codes d’état de la réponse possibles sont les suivants :

| Code d’état | Code d’erreur          | Message                                                                                                | Cause                                                                                               |
| ----------- | ---------------------- | ------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------- |
| `200`       |                        | Organizations récupérées avec succès.                                                                  |                                                                                                     |
| `400`       | `invalid_uri`          | `invalid_request_uri`                                                                                  | Le chemin n’est pas valide.                                                                         |
| `400`       | `invalid_query_string` | Chaîne de requête invalide. Le message varie selon la cause.                                           | La chaîne de requête n’est pas valide.                                                              |
| `401`       |                        | Jeton invalide.                                                                                        |                                                                                                     |
| `401`       |                        | Signature invalide reçue lors de la validation du JSON Web Token.                                      |                                                                                                     |
| `403`       | `insufficient_scope`   | Portée insuffisante; une des portées suivantes est requise : `read:organizations`.                     | Tentative de lecture ou d’écriture d’un champ non autorisé avec les portées du bearer token fourni. |
| `404`       |                        | Utilisateur introuvable                                                                                |                                                                                                     |
| `429`       |                        | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                     |
