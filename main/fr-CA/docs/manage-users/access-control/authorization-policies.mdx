description: Découvrez le concept des politiques d’autorisation et la façon dont elles s’appliquent dans Auth0.
title: Politiques d’autorisation
---

En arrière-plan, l’autorisation basée sur les rôles utilise une politique d’autorisation préconfigurée qui contient des conditions permettant au code de déterminer si un utilisateur peut accéder à une API protégée.

La politique d’autorisation détermine :

* comment définir et organiser les utilisateurs ou les rôles visés par la politique
* quelle logique et quelles conditions s’appliquent à la politique, et si leur résultat autorise ou refuse l’accès

Lorsque vous utilisez les fonctionnalités d’autorisation de base d’Auth0 et le [contrôle d’accès basé sur les rôles (RBAC)](/fr-CA/docs/manage-users/access-control/rbac), la politique comprend l’évaluation des rôles et des permissions attribués aux utilisateurs. Pour utiliser ces fonctionnalités, vous devez [activer le contrôle d’accès basé sur les rôles pour les API](/fr-CA/docs/get-started/apis/enable-role-based-access-control-for-apis).

Vous pouvez aussi personnaliser davantage la politique d’autorisation à l’aide des [Rules](/fr-CA/docs/customize/rules). Pour en savoir plus, consultez [Rules for Authorization Policies](/fr-CA/docs/manage-users/access-control/rules-for-authorization-policies).
