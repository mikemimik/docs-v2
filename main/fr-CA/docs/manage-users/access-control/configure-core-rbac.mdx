title: Configurer les fonctionnalités d’Authorization Core pour le contrôle d’accès basé sur les rôles
---

Les fonctionnalités d’Authorization Core d’Auth0 permettent le contrôle d’accès basé sur les rôles (RBAC) de vos API.

Pour que RBAC fonctionne correctement, vous devez l’activer pour votre API à l’aide du Dashboard ou de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>. Pour en savoir plus, consultez [Activer le contrôle d’accès basé sur les rôles pour les API](/fr-CA/docs/get-started/apis/enable-role-based-access-control-for-apis).

Les fonctionnalités d’Authorization Core sont différentes de celles de l’Authorization Extension. Pour comparer les deux produits, consultez [Authorization Core vs. Authorization Extension](/fr-CA/docs/manage-users/access-control/authorization-core-vs-authorization-extension).

Pour utiliser ces fonctionnalités le plus efficacement possible, vous devriez suivre les étapes suivantes :

1. [Enregistrer une API dans Auth0](/fr-CA/docs/get-started/auth0-overview/set-up-apis)
2. [Définir les permissions de l’API](/fr-CA/docs/get-started/apis/add-api-permissions)
3. [Créer des rôles](/fr-CA/docs/manage-users/access-control/configure-core-rbac/roles/create-roles)
4. [Attribuer des rôles aux utilisateurs](/fr-CA/docs/manage-users/access-control/configure-core-rbac/rbac-users/assign-roles-to-users)
5. [Attribuer des permissions aux utilisateurs](/fr-CA/docs/manage-users/access-control/configure-core-rbac/rbac-users/assign-permissions-to-users), au besoin.
