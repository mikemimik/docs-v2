Vous pouvez attribuer des [permissions](/docs/fr-CA/manage-users/access-control/rbac) à un utilisateur à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>. Les permissions attribuées peuvent être utilisées avec le jeu de fonctionnalités Authorization Core de l’API.

<Warning>
  L’attribution directe de permissions à un utilisateur vous prive des avantages du contrôle d’accès basé sur les rôles (RBAC) et n’est généralement pas recommandée.
</Warning>

<div id="prerequisites">
  ## Prérequis
</div>

* Pour que le contrôle d’accès basé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API dans le Dashboard ou au moyen de la Management API. La fonctionnalité Authorization Core diffère de l’Authorization Extension. Pour comparer les deux, consultez [Authorization Core vs. Authorization Extension](/docs/fr-CA/manage-users/access-control/authorization-core-vs-authorization-extension).
* [Configurer une API](/docs/fr-CA/get-started/auth0-overview/set-up-apis) dans le Dashboard.
* Les permissions sont sélectionnées parmi des valeurs prédéfinies. Si votre liste de rôles est vide, assurez-vous d’[ajouter des permissions](/docs/fr-CA/get-started/apis/add-api-permissions) à votre API.

<div id="dashboard">
  ## Dashboard
</div>

Il y a deux façons d’attribuer une permission à un utilisateur. Vous pouvez sélectionner un utilisateur dans la liste **Users**, puis lui attribuer une permission, ou accéder à la page **User Details** (profil d’utilisateur) d’un utilisateur précis et choisir une permission à attribuer dans l’onglet **Permissions**.

<div id="assign-permission-in-user-list">
  ### Attribuer des permissions depuis la liste des utilisateurs
</div>

1. Accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
2. Cliquez sur `...` à côté de l’utilisateur que vous souhaitez modifier, puis sélectionnez **Assign Permissions**.
3. Sélectionnez l’API à partir de laquelle vous souhaitez attribuer des permissions, puis sélectionnez les permissions à attribuer à l’utilisateur et cliquez sur **Add Permissions**.

<div id="assign-permissions-in-user-profile">
  ### Attribuer des permissions dans le profil utilisateur
</div>

Vous pouvez aussi attribuer des permissions aux utilisateurs à partir de leur page de profil.

1. Accédez à [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users), puis cliquez sur le nom de l’utilisateur à consulter.
2. Cliquez sur l’onglet **Permissions**, puis sur **Assign Permissions**.
3. Sélectionnez l’API à partir de laquelle vous souhaitez attribuer des permissions, puis sélectionnez les permissions à attribuer à l’utilisateur et cliquez sur **Add Permissions**.

<div id="management-api">
  ## Management API
</div>

Effectuez une requête `POST` vers le [point de terminaison d’attribution des permissions à l’utilisateur](https://auth0.com/docs/api/management/v2#!/Users/post_permissions). Assurez-vous de remplacer les valeurs fictives `USER_ID`, `MGMT_API_ACCESS_TOKEN`, `API_IDENTIFIER` et `PERMISSION_NAME` par, respectivement, votre ID utilisateur, le <Tooltip tip="Jeton d’accès : identifiant d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+token">jeton d’accès</Tooltip> de la Management API, le ou les identifiants d’API et le ou les noms de permission.

| Valeur                  | Description                                                                                                             |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | L’ID de l’utilisateur auquel vous voulez attribuer des permissions.                                                     |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) avec la portée `update:users`.   |
| `API_IDENTIFIER`        | Identifiant(s) de la ou des API associées à la ou aux permissions que vous souhaitez attribuer à l’utilisateur indiqué. |
| `PERMISSION_NAME`       | Nom(s) de la ou des permissions que vous souhaitez attribuer à l’utilisateur indiqué.                                   |
