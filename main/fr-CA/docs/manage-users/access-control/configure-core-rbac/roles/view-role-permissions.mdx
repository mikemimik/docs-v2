Vous pouvez consulter les [permissions](/fr-CA/docs/manage-users/access-control/rbac) ajoutées à un rôle à l’aide de <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>. Les permissions et les rôles attribués sont utilisés avec le jeu de fonctionnalités Authorization Core de l’API.

<div id="prerequisites">
  ## Prérequis
</div>

Pour que le contrôle d’accès basé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API dans le Dashboard ou au moyen de la Management API. La fonctionnalité Authorization Core se distingue de l’Authorization Extension. Pour comparer les deux, consultez [Authorization Core vs. Authorization Extension](/fr-CA/docs/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Dashboard
</div>

1. Accédez à [Dashboard &gt; User Management &gt; Roles](https://manage.auth0.com/#/roles), puis cliquez sur le nom du rôle que vous voulez consulter.
2. Cliquez sur la vue **Permissions**.

Les renseignements suivants s’affichent pour chaque permission :

| Colonne     | Description                                          |
| ----------- | ---------------------------------------------------- |
| Nom         | Nom de la permission tiré de sa définition.          |
| Description | Description de la permission tirée de sa définition. |
| API         | Nom de l’API à laquelle la permission est associée.  |

<div id="management-api">
  ## Management API
</div>

Effectuez un appel `GET` au [point de terminaison Get Role Permissions](https://auth0.com/docs/api/management/v2#!/Roles/get_role_permission). Assurez-vous de remplacer les valeurs d’exemple `ROLE_ID` et `MGMT_API_ACCESS_TOKEN` par l’identifiant de votre rôle et votre <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+token">jeton d’accès</Tooltip> pour la Management API, respectivement.

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/roles/ROLE_ID/permissions' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/roles/ROLE_ID/permissions");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"

  	req, _ := http.NewRequest("GET", url, nil)
  ```

| Valeur                  | Description                                                                                                         |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------- |
| `ROLE_ID`               | L’identifiant du rôle pour lequel vous souhaitez obtenir les permissions.                                           |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) avec le scope `read:roles`. |
