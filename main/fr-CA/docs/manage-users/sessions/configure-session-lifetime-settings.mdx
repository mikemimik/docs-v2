L’**expiration absolue** d’une session est définie au moment de sa création. Vous pouvez la modifier en configurant les paramètres de session dans le <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou à l’aide de la <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip>.

<div id="dashboard">
  ## Dashboard
</div>

1. Allez à [Dashboard &gt; Tenant Settings](https://manage.auth0.com/#/tenant) et sélectionnez la vue **Advanced**.
2. Faites défiler jusqu’à la section **Session Expiration**, repérez **Idle Session Lifetime** et **Maximum Session Lifetime**, entrez les paramètres souhaités, puis sélectionnez **Save**.

   <Frame>![Dashboard Tenant Settings Advanced login gestion de session](/docs/images/cdy7uua7fh8z/q2fGPzoUqCcXj7OxcHhjy/de15afcd26e8ea80937733034ffbd387/Screenshot_2024-11-06_at_11.02.16.png)</Frame>

| Paramètre                | Description                                                                                                                                                                                                                                                                                                      |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Idle Session Lifetime    | Délai (en minutes) après lequel la session d’un utilisateur expire s’il n’a pas interagi avec le serveur d’autorisation. Cette valeur est remplacée par les limites du système si elle dépasse 4 320 minutes (3 jours) pour Essential ou Professional, ou 144 000 minutes (100 jours) pour les forfaits Enterprise. |
| Maximum Session Lifetime | Délai (en minutes) après lequel un utilisateur doit se reconnecter, peu importe son activité. Cette valeur est remplacée par les limites du système si elle dépasse 43 200 minutes (30 jours) pour Essential ou Professional, ou 525 600 minutes (365 jours) pour les forfaits Enterprise.                       |

<div id="management-api">
  ## Management API
</div>

Faites un appel `PATCH` au [point de terminaison Tenant Settings](https://auth0.com/docs/api/management/v2#!/tenants/patch_settings). Assurez-vous de remplacer les valeurs fictives `MGMT_API_ACCESS_TOKEN`, `SESSION_LIFETIME` et `IDLE_SESSION_LIFETIME` par votre <Tooltip tip="Jeton d’accès : information d’authentification Authorization, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> de la Management API, la valeur de durée de vie de la session et la valeur de durée de vie de la session inactive, respectivement.

| Valeur                        | Description                                                                                                                                                                                                                                                                                                              |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MGMT_API_ACCESS_TOKEN`       | [Jeton d’accès pour la Management API](https://auth0.com/docs/api/management/v2/tokens) avec la portée `update:tenant_settings`.                                                                                                                                                                                         |
| `IDLE_SESSION_LIFETIME_VALUE` | Période (en heures) au terme de laquelle la session d’un utilisateur expirera s’il n’a pas interagi avec le serveur d’autorisation. Cette valeur sera remplacée par les limites du système si elle dépasse 72 heures (3 jours) pour Essential ou Professional, ou 2 400 heures (100 jours) pour les forfaits Enterprise. |
| `SESSION_LIFETIME_VALUE`      | Période (en heures) au terme de laquelle un utilisateur devra se reconnecter, peu importe son activité. Cette valeur sera remplacée par les limites du système si elle dépasse 720 heures (30 jours) pour Essential ou Professional, ou 8 760 heures (365 jours) pour les forfaits Enterprise.                           |
