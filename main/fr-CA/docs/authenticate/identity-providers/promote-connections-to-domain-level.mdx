Lorsque vous promouvez une connexion au niveau du domaine, toute [application tierce](/fr-CA/docs/get-started/applications/third-party-applications) de votre tenant peut l’utiliser pour authentifier les utilisateurs. Cela permet aux applications tierces d’accéder à vos fournisseurs d’identité. Les applications tierces peuvent uniquement utiliser des connexions au niveau du domaine.

Comme les clients enregistrés dynamiquement ne peuvent pas être associés individuellement à des connexions au moment de leur création, vous devez promouvoir les connexions au niveau du domaine lorsque vous utilisez la [Dynamic Client Registration](/fr-CA/docs/get-started/applications/dynamic-client-registration).

Vous pouvez promouvoir une connexion au niveau du domaine à l’aide du <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Vous pouvez promouvoir une connexion au domain level directement dans l’Auth0 Dashboard. Cette option est offerte pour tous les connection types (Database, Social, Enterprise et Passwordless).

    1. Accédez à **Dashboard &gt; Authentication** et sélectionnez le connection type (p. ex., **Database, Social, Enterprise**).
    2. Créez une nouvelle connexion ou sélectionnez une connexion existante à configurer.
    3. Accédez à l’onglet **Settings**.
    4. Faites défiler la page jusqu’au paramètre **Promote Connection to Domain Level**, qui se trouve généralement dans la section **Advanced**.
    5. Activez le bouton bascule.

    <Frame>![Promouvoir une connexion au niveau du domaine](/docs/images/cdy7uua7fh8z/promote-connection-to-domain-level.png)</Frame>

    6. Cliquez sur **Save Changes**.
  </Tab>

  <Tab title="Management API">
    Vous pouvez également promouvoir une connexion en effectuant un appel `PATCH` vers le [endpoint Update a Connection](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id) de la Management API. Assurez-vous de remplacer les valeurs fictives `CONNECTION_ID` et `MGMT_API_ACCESS_TOKEN` par vos propres informations.

    <AuthCodeGroup>
      ```bash cURL
      curl --request PATCH \
        --url 'https://{yourDomain}/api/v2/connections/CONNECTION_ID' \
        --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
        --header 'cache-control: no-cache' \
        --header 'content-type: application/json' \
        --data '{ "is_domain_connection": true }'
      ```

      ```csharp C#
      var client = new RestClient("https://{yourDomain}/api/v2/connections/CONNECTION_ID");
      var request = new RestRequest(Method.PATCH);
      request.AddHeader("content-type", "application/json");
      request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
      request.AddHeader("cache-control", "no-cache");
      request.AddParameter("application/json", "{ "is_domain_connection": true }", ParameterType.RequestBody);
      IRestResponse response = client.Execute(request);
      ```

      ```go Go
      package main

      import (
        "fmt"
        "strings"
        "net/http"
        "io/ioutil"
      )

      func main() {

        url := "https://{yourDomain}/api/v2/connections/CONNECTION_ID"

        payload := strings.NewReader("{ "is_domain_connection": true }")

        req, _ := http.NewRequest("PATCH", url, payload)

        req.Header.Add("content-type", "application/json")
        req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
        req.Header.Add("cache-control", "no-cache")

        res, _ := http.DefaultClient.Do(req)

        defer res.Body.Close()
        body, _ := ioutil.ReadAll(res.Body)

        fmt.Println(res)
        fmt.Println(string(body))

      }
      ```

      ```php PHP
      $curl = curl_init();

      curl_setopt_array($curl, [
        CURLOPT_URL => "https://{yourDomain}/api/v2/connections/CONNECTION_ID",
      ```

      ```python Python
      import http.client

      conn = http.client.HTTPSConnection("")

      payload = "{ "is_domain_connection": true }"

      headers = {
          'content-type': "application/json",
          'authorization': "Bearer MGMT_API_ACCESS_TOKEN",
          'cache-control': "no-cache"
          }

      conn.request("PATCH", "/{yourDomain}/api/v2/connections/CONNECTION_ID", payload, headers)

      res = conn.getresponse()
      data = res.read()

      print(data.decode("utf-8"))
      ```

      ```ruby Ruby
      require 'uri'
      require 'net/http'
      require 'openssl'

      url = URI("https://{yourDomain}/api/v2/connections/CONNECTION_ID")

      http.use_ssl = true

      request = Net::HTTP::Patch.new(url)
      request["content-type"] = 'application/json'
      request["authorization"] = 'Bearer MGMT_API_ACCESS_TOKEN'

      request.body = "{ "is_domain_connection": true }"

      response = http.request(request)
      ```

    | Valeur                  | Description                                                                                                                                              |
    | ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `CONNECTION_ID`         | L’identifiant de la connexion à promouvoir.                                                                                                              |
    | `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour l’API de gestion](https://auth0.com/docs/api/management/v2/tokens) avec la [portée](/fr-CA/docs/glossary?term=scope) `update:connections`. |
