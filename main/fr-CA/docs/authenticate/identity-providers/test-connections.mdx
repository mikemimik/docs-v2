En général, pour tester une connexion, vous devez vous connecter à l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Accédez à [Auth0 Dashboard](https://manage.auth0.com/#), puis sélectionnez **Authentication**, puis le type de connexion que vous voulez tester.
2. Repérez la connexion que vous voulez tester, puis sélectionnez **Try Connection** dans son menu **More Options** (**...**).
3. Connectez-vous avec le fournisseur d’identité.
4. Attendez que la page **It Works!** s’affiche avec le résultat.

Auth0 simule le flux d’authentification comme s’il s’agissait d’une application et affiche le profil d’utilisateur obtenu à la suite d’une authentification réussie.

<div id="test-partner-connections">
  ## Tester les connexions partenaires
</div>

Si vous testez une connexion qui appartient à quelqu’un d’autre et que vous n’avez pas d’identifiants de test fournis par cette personne, il n’est pas possible d’être connecté à l’Auth0 Dashboard. C’est fréquent avec les Enterprise connexions, comme les <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des informations d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des informations d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Providers">fournisseurs d’identité</Tooltip> (IdPs) ou Active Directory. La solution? Vos partenaires peuvent tester la nouvelle connexion.

1. Enregistrez une application de test.

   * Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis sélectionnez **Create Application**. Vous pouvez lui donner n’importe quel nom (par exemple, `Test App`).
   * Dans les paramètres de l’application nouvellement créée, définissez **Allowed Callback URLs** sur `http://jwt.io`.
   * Sélectionnez **Save Changes**.
2. Envoyez à votre partenaire le lien pour se connecter.

   <AuthCodeBlock children={codeExample} language="http" />

   Remplacez ces deux paramètres :

   * `client_id` : le Client ID de l’application créée à l’étape 1.
   * `connection` : le nom de la connexion que vous voulez tester.
3. Testez la connexion. Lorsque votre partenaire suit le lien, il est redirigé vers le fournisseur d’identité configuré (la connexion). Une fois l’authentification réussie, il est renvoyé vers [JWT.io](http://jwt.io), où toutes les propriétés de l’utilisateur sont décodées à partir du token.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’application de test n’est pas une vraie application. [JWT.io](http://jwt.io) est un site Web de test qui décode les tokens envoyés dans un fragment d’URL.
</Callout>
