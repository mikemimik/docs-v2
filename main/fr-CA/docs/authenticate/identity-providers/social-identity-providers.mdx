Le Social login est une méthode d’authentification qui permet aux utilisateurs de se connecter à une application à l’aide d’identifiants existants provenant d’un <Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> social, comme Google ou Facebook. Comme les utilisateurs ont souvent déjà enregistré leurs identifiants sociaux dans leur navigateur ou sur leur appareil, le Social login offre une expérience utilisateur fluide qui exige très peu d’interaction manuelle avec l’application.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’authentification des utilisateurs est distincte du fait de connecter et d’autoriser des applications auprès d’un fournisseur externe afin qu’elles puissent accéder à des API externes au nom de l’utilisateur. Pour en savoir plus, consultez [User authentication vs Connected Accounts](/fr-CA/docs/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).
</Callout>

Auth0 prend en charge le Social login pour les applications Web et les applications natives. Vous pouvez consulter la liste des fournisseurs d’identité sociaux pris en charge dans [Auth0 Marketplace](https://marketplace.auth0.com/categories/social-login) ou dans l’<Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> :

* La [catégorie Social Login d’Auth0 Marketplace](https://marketplace.auth0.com/categories/social-login) fournit des renseignements sur chaque connexion sociale prise en charge. Vous pouvez utiliser le bouton **Add Integration** pour lancer le processus de configuration dans votre tenant Auth0, et l’onglet **Installation** pour consulter la documentation d’implémentation.

* Dans [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social), sélectionnez **Create Connection** pour afficher une liste des fournisseurs d’identité sociaux pris en charge. Le **Setup Guide** présente les étapes à suivre pour configurer la connexion ainsi que des liens vers la documentation pertinente du fournisseur d’identité social.

Pour réussir l’implémentation, assurez-vous d’[enregistrer les developer keys](/fr-CA/docs/authenticate/identity-providers/social-identity-providers/devkeys) pour le fournisseur sélectionné.

Si un fournisseur en particulier n’est pas offert, vous pouvez aussi [configurer des custom social connections](/fr-CA/docs/authenticate/identity-providers/social-identity-providers/oauth2).
