---

Auth0 s’intègre à Active Directory (AD) au moyen de Lightweight Directory Access Protocol (LDAP), par l’intermédiaire d’un **Active Directory/LDAP Connector** que vous installez sur votre réseau.

Le **AD/LDAP Connector** (1) sert de passerelle entre votre **Active Directory/LDAP** (2) et le **service Auth0** (3). Cette passerelle est nécessaire, car AD/LDAP est généralement limité à votre réseau interne, tandis qu’Auth0 est un service infonuagique qui s’exécute dans un contexte complètement différent.

<Frame>![Schéma d’aperçu du connecteur AD/LDAP](/docs/images/cdy7uua7fh8z/2HT4cRvUDzA2OdEPlfgLDV/be32ffe82562dc8f07fdd2097f14d881/ldap-connect.png)</Frame>

Pour la [haute disponibilité et la répartition de charge](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-high-availability), vous pouvez installer plusieurs instances du connecteur. Toutes les connexions sont sortantes, du connecteur vers le serveur Auth0; il n’est donc généralement pas nécessaire de modifier votre pare-feu.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer :

* [Enregistrez votre application dans Auth0](/docs/fr-CA/get-started/auth0-overview/create-applications).

  * Sélectionnez un **Type d’application** approprié.
  * Ajoutez une **URL de rappel autorisée** : `{https://yourApp/callback}`.
  * Assurez-vous que les [**Types d’octroi**](/docs/fr-CA/get-started/applications/update-grant-types) de votre application incluent les flux appropriés.

<div id="steps">
  ## Étapes
</div>

Pour connecter votre application à Active Directory/LDAP, vous devez :

1. [Créer une connexion d’entreprise dans Auth0](#create-an-enterprise-connection-in-auth0) et télécharger le programme d’installation
2. [Installer le connecteur sur votre réseau](#install-the-connector-on-your-network)
3. [Activer la connexion d’entreprise pour votre application Auth0](#enable-the-enterprise-connection-for-your-auth0-application)
4. [Tester la connexion](#test-the-connection)

<div id="create-an-enterprise-connection-in-auth0">
  ## Créer une connexion d’entreprise dans Auth0
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), repérez **Active Directory / LDAP** et sélectionnez son `+`.

   <Frame>![Dashboard - Connections - Enterprise](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Saisissez les détails de votre connexion, puis sélectionnez **Create** :

| Champ                                          | Description                                                                                                                                                                                                                                                                                                                        |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Connection name**                            | Identifiant logique de votre connexion; il doit être unique dans votre tenant. Une fois défini, ce nom ne peut plus être modifié.                                                                                                                                                                                                  |
| **Display name** (facultatif)                  | Texte utilisé pour personnaliser le bouton de connexion de Universal Login. Lorsqu’il est défini, le bouton de connexion de Universal Login affiche : &quot;Continue with &#123;Display name&#125;&quot;.                                                                                                                          |
| **Logo URL** (facultatif)                      | URL de l’image utilisée pour personnaliser le bouton de connexion de Universal Login. Lorsqu’elle est définie, le bouton de connexion de Universal Login affiche l’image sous la forme d’un carré de 20 px sur 20 px.                                                                                                              |
| **IdP Domains** (facultatif)                   | Liste de domaines de courriel valides séparés par des virgules qui seront autorisés à se connecter à l’aide de cette connexion. Requis uniquement si vous utilisez le widget de connexion Lock.                                                                                                                                    |
| **Disable cache**                              | Lorsque cette option est activée, la mise en cache est désactivée.                                                                                                                                                                                                                                                                 |
| **Use client SSL certificate authentication**  | Lorsque cette option est activée, l’authentification par certificat SSL client est utilisée.                                                                                                                                                                                                                                       |
| **Use Windows Integrated Auth (Kerberos)**     | Lorsque cette option est activée, vous devrez saisir une plage d’adresses IP. Lorsque les utilisateurs se connectent à partir de ces adresses IP, Kerberos est utilisé; sinon, un nom d’utilisateur et un mot de passe AD/LDAP sont demandés. En règle générale, la plage d’adresses IP saisie correspond à des adresses intranet. |
| **Sync user profile attributes at each login** | Lorsque cette option est activée, Auth0 synchronise automatiquement les données du profil utilisateur à chaque connexion, ce qui garantit que les changements apportés dans la source de connexion sont automatiquement mis à jour dans Auth0.                                                                                     |

<Frame>![Saisir les détails de la connexion AD / LDAP](/docs/images/cdy7uua7fh8z/2IR80sIwNNyPlDMxIjWv8e/b6748243f90ce0e7d771a39cef24cb0f/LDAP_Connection__1_.png)</Frame>
3. Téléchargez le programme d’installation fourni et prenez en note la **Provisioning Ticket URL** fournie.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Nous proposons différentes versions du connecteur à installer sur les plateformes Windows ou Linux.
</Callout>

* [Installer l’AD/LDAP Connector sous Windows](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/install-configure-ad-ldap-connector)
* [Installer l’AD/LDAP Connector sur des plateformes autres que Microsoft](/docs/fr-CA/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/install-configure-ad-ldap-connector)
