---
description: Découvrez comment redemander à l’utilisateur d’accorder des permissions à votre application.
title: Gérer les permissions refusées lors de l’autorisation
---

Lorsque vos utilisateurs autorisent votre application, certains fournisseurs (comme Facebook) leur permettent de sélectionner les attributs qu’ils souhaitent partager.

Par défaut, cette sélection n’est effectuée que lorsque l’utilisateur autorise l’application pour la première fois. Si l’utilisateur choisit de ne pas autoriser certains attributs (comme son courriel) dont votre application a besoin, il ne pourra pas y accéder.

Dans de tels cas, l’utilisateur devra être invité de nouveau à accorder la permission pour les attributs requis afin de se connecter.

<div id="re-prompt-for-permissions">
  ## Demander à nouveau le consentement pour les permissions
</div>

Si vous définissez le paramètre **prompt=consent** lors de l’appel au point de terminaison [/authorize](https://auth0.com/docs/api/authentication/reference#social) de l’[API d’autorisation](https://auth0.com/docs/api/authentication), l’utilisateur sera invité à accorder de nouveau des permissions à votre application.

Ce paramètre peut aussi être défini dans Lock comme [paramètre d’authentification](/fr-CA/docs/libraries/lock/lock-authentication-parameters) avec **prompt: &#39;consent&#39;**.

Vous pouvez aussi le définir avec [Auth0.js](https://github.com/auth0/auth0.js) au moyen de **prompt: &#39;consent&#39;**.
