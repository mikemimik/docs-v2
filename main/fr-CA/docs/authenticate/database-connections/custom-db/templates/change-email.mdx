---
description: Décrit le script d’action de base de données personnalisée pour modifier l’adresse courriel d’un utilisateur.
title: Modèle de script Change Email
---

Le script Change Email implémente la fonction définie lorsqu’une adresse courriel d’utilisateur ou son état de vérification change. Nous vous recommandons de nommer cette fonction `changeEmail`.

Le script n’est utilisé que dans un [scénario d’authentification héritée](/fr-CA/docs/authenticate/database-connections/custom-db/overview-custom-db-connections) et il est requis si vous souhaitez mettre à jour l’adresse courriel d’un utilisateur (et/ou son état de vérification) dans Auth0 et dans votre base de données externe au cours de la même opération.

Le script Change Email ne peut pas être configuré dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Pour gérer ce script, vous devez utiliser l’<Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> d’Auth0, le point de terminaison [Create a connection](https://auth0.com/docs/api/management/v2#!/Connections/post_connections) ou [Update a connection](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id), ou encore l’[Auth0 Deploy CLI](/fr-CA/docs/deploy-monitor/deploy-cli-tool).

<Warning>
  Si vous gérez les Database Action Scripts d’une connexion de base de données personnalisée dans l’Auth0 Dashboard, le script Change Email sera automatiquement supprimé lorsque vous enregistrerez.

  Pour éviter que le script Change Email soit supprimé par erreur, utilisez l’Auth0 Management API ou l’Auth0 Deploy CLI pour gérer la connexion.

  Si vous modifiez manuellement l’adresse courriel d’un utilisateur dans votre base de données personnalisée, vous devez aussi appliquer un correctif à l’utilisateur séparément; Auth0 ne détectera pas automatiquement ce changement.
</Warning>

<div id="changeemail-function">
  ## Fonction changeEmail
</div>

La fonction changeEmail doit :

* Mettre à jour l’adresse courriel de l’utilisateur dans la base de données externe.
* Renvoyer une erreur si l’opération a échoué ou si une erreur s’est produite.

<div id="definition">
  ### Définition
</div>

La fonction `changeEmail` accepte quatre paramètres et retourne une fonction `callback` :

```js lines
changeEmail(email, newEmail, verified, callback): function
```

| Paramètre  | Type     | Description                                                                                                |
| ---------- | -------- | ---------------------------------------------------------------------------------------------------------- |
| `email`    | String   | Adresse courriel actuelle de l&#39;utilisateur.                                                            |
| `newEmail` | String   | Valeur à enregistrer comme nouvelle adresse courriel de l&#39;utilisateur dans la base de données externe. |
| `verified` | Boolean  | Statut de vérification de la nouvelle adresse courriel.                                                    |
| `callback` | Function | Sert à transmettre les données d&#39;erreur dans le pipeline.                                              |

<div id="example">
  ### Exemple
</div>

Voici un exemple en pseudo-JavaScript montrant comment vous pourriez implémenter la fonction `changeEmail` :

```javascript lines expandable
function (email, newEmail, verified, callback) {
  // Préparer l'appel API
  let options = {
    url: "https://example.com/api/users",
    action: "PATCH",
    body: {
      email: email,
      new_email: newEmail,
      email_verified: verified
    }
  };

  // Appeler l'API
  send(options, err => {
    // Retourner la valeur `false` dans le callback si l'opération a échoué
    if (err && err.id == "FAIL_CHANGE_EMAIL") {
      return callback(null, false);
    } else if (err) {
      // Retourner une erreur dans le callback si une erreur non spécifiée s'est produite
      return callback(new Error("My custom error message."));
    }

    // Retourner la valeur `true` dans le callback si l'opération a réussi
    return callback(null, true);
  });
}
```

<div id="callback-function">
  ## fonction callback
</div>

La fonction `callback` accepte deux paramètres et renvoie une fonction.

<div id="definition">
  ### Définition
</div>

```js lines
callback(error, operationResult): function
```

| Paramètre         | Type    | Requis     | Description                                                               |
| ----------------- | ------- | ---------- | ------------------------------------------------------------------------- |
| `error`           | Objet   | Requis     | Contient les données sur l’erreur.                                        |
| `operationResult` | Boolean | Facultatif | Indique le résultat de l’opération de modification de l’adresse courriel. |

<div id="return-a-success">
  ### Signaler une réussite
</div>

Si l’opération de changement d’adresse courriel a réussi, appelez la fonction `callback` en passant une valeur `null` comme paramètre `error` et une valeur `true` comme paramètre `operationResult` :

```js lines
return callback(null, true);
```

<div id="return-a-failure">
  ### Signaler un échec
</div>

Si l’opération de changement d’adresse courriel a échoué, retournez la fonction `callback` et transmettez la valeur `null` au paramètre `error` ainsi que la valeur `false` au paramètre `operationResult` :

```js lines
return callback(null, false);
```

<div id="return-an-error">
  ### Renvoyer une erreur
</div>

Si une erreur s&#39;est produite, appelez la fonction `callback` et transmettez les renseignements pertinents sur l&#39;erreur dans le paramètre `error` :

```js lines
return callback(new Error("My custom error message."));
```
