title: Surveiller les événements de clé d’accès dans les journaux du tenant
---

Vous pouvez surveiller les flux de clé d’accès associés à vos [connexions de base de données](/fr-CA/docs/deploy-monitor/logs) dans les [journaux du tenant](/fr-CA/docs/deploy-monitor/logs).

Vous pouvez consulter les événements de journal dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ([Monitoring &gt; Logs](https://manage.auth0.com/#/logs)) ou récupérer les journaux à l’aide de la [Management API](https://auth0.com/docs/api/management/v2/logs/get-logs).

| Code d’événement                | Scénario                                                                                                        | Notes                                                                                                                                                                                                                                                                                                        |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ss`                            | Lors de l’inscription, un utilisateur a enregistré une clé d’accès comme méthode d’authentification principale. | Cet événement de journal contient le champ de tableau `details.authentication_methods` avec la valeur `passkey`.<br /><br />Si un utilisateur s’est inscrit avec un mot de passe, ce champ de tableau contient la valeur `pwd`.                                                                              |
| `s`                             | Lors de la connexion, un utilisateur s’est authentifié avec une clé d’accès.                                    | Cet événement de journal contient le tableau `details.prompts`, qui répertorie les prompts qu’un utilisateur a parcourus pendant la connexion.<br /><br />Si un utilisateur s’est authentifié à l’aide d’une clé d’accès, le tableau contient un élément avec le champ suivant : `"performed_amr": ["phr"]`. |
| `gd_enrollment_complete`        | Un flux d’inscription progressive ou locale s’est terminé avec succès.                                          | Cet événement de journal contient le champ `details.authenticator.type` avec la valeur `passkey` si un utilisateur a inscrit une clé d’accès.                                                                                                                                                                |
| `gd_webauthn_enrollment_failed` | Une erreur s’est produite lors d’un flux d’inscription progressive ou locale.                                   | Cet événement de journal contient le champ de tableau `details.authentication_methods` avec la valeur `passkey` si un utilisateur a tenté d’inscrire une clé d’accès.                                                                                                                                        |
