Utilisez une connexion de base de données personnalisée lorsque vous souhaitez donner à Auth0 accès à votre propre référentiel de données d’identité indépendant (hérité), principalement pour l’authentification (en jouant le rôle d’un <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip>) ainsi que pour migrer les données utilisateur vers le référentiel de données d’Auth0.

L’[Extensibilité](/fr-CA/docs/customize/extensions) d’Auth0 vous permet d’ajouter une logique personnalisée afin de créer des solutions sur mesure pour la gestion des identités et des accès (IdAM). Auth0 offre cette extensibilité au moyen des [Actions](/fr-CA/docs/customize/actions) et des [scripts](/fr-CA/docs/authenticate/database-connections/custom-db/templates), tant pour les connexions de base de données personnalisées que pour la migration de bases de données personnalisées. Ces deux mécanismes sont mis en œuvre à l’aide de [Node.js](https://nodejs.org/en/), exécuté sur la plateforme Auth0 dans un tenant Auth0.

L’extensibilité d’Auth0 s’exécute à différents points du pipeline IAM :

* Les **Actions** sont des fonctions sécurisées, propres au tenant et versionnées, écrites en Node.js, qui s’exécutent à certains moments dans la plateforme Auth0. Les Actions servent à personnaliser et à étendre les capacités d’Auth0 grâce à une logique personnalisée. Pour en savoir plus, consultez [Comprendre le fonctionnement des Actions d’Auth0](/fr-CA/docs/customize/actions/actions-overview).

* Les **scripts d’action de base de données personnalisée** peuvent servir à intégrer un magasin d’identité utilisateur existant, ou être utilisés lorsqu’une [migration automatique des utilisateurs](/fr-CA/docs/manage-users/user-migration) à partir d’un magasin d’identité hérité est requise. Divers [modèles](/fr-CA/docs/authenticate/database-connections/custom-db/templates) sont fournis.

Quel que soit le cas d’utilisation, l’extensibilité d’Auth0 vous permet d’adapter les opérations IAM à vos besoins précis. Cependant, si elle n’est pas utilisée correctement, elle peut ouvrir la porte à une utilisation inappropriée ou involontaire, ce qui peut entraîner des problèmes par la suite. Pour aider à prévenir ce genre de situation, Auth0 fournit des [recommandations sur les pratiques exemplaires](/fr-CA/docs/authenticate/database-connections/custom-db/custom-database-connections-scripts) aux concepteurs comme aux responsables de la mise en œuvre, et nous vous recommandons de les lire au complet au moins une fois, même si vous avez déjà commencé votre parcours avec Auth0.
