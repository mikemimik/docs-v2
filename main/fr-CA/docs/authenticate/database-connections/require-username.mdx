title: Ajouter un nom d’utilisateur pour les connexions de base de données
---

Pour les connexions de base de données, vous pouvez permettre à vos utilisateurs de se connecter avec un nom d’utilisateur plutôt qu’avec leur adresse courriel.

<div id="require-username">
  ## Exiger un nom d’utilisateur
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database), puis sélectionnez le nom de la connexion que vous souhaitez modifier.
2. Repérez **Requires Username**, puis utilisez le bouton bascule pour activer ou désactiver l’obligation d’utiliser un nom d’utilisateur.

   <Frame>![Activer ou désactiver le paramètre Requires Username](/docs/images/cdy7uua7fh8z/1MlOrFmcSCOCRYKRxWz6xT/7a87dac16a31cbb6e608489c5843d6c7/2025-02-25_12-02-23.png)</Frame>

Pour voir l’effet de ce paramètre sur l’écran de connexion, sélectionnez la vue **Try Connection**. Remarquez qu’une fois **Requires Username** activé, les nouveaux utilisateurs doivent saisir un nom d’utilisateur et leur adresse courriel pour s’inscrire.

<Frame>![Formulaire de connexion avec nom d’utilisateur](/docs/images/cdy7uua7fh8z/AiB37UZqyBPb8h55PZtzm/73e6fd0033f63f865c40f189db1620ef/2025-01-29_15-22-11.png)</Frame>

Les utilisateurs peuvent ensuite ouvrir une session avec leur nom d’utilisateur ou leur adresse courriel. Les utilisateurs qui se sont inscrits alors que **Requires Username** était désactivé n’auront pas de valeur enregistrée dans le champ nom d’utilisateur et devront ouvrir une session avec leur adresse courriel.

<div id="username-length">
  ### Longueur du nom d’utilisateur
</div>

Par défaut, la longueur autorisée des noms d’utilisateur va de 1 à 15 caractères. Toutefois, dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou au moyen de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> v2, vous pouvez modifier les longueurs minimale et maximale (jusqu’à un maximum de 128 caractères).

<div id="allowed-characters">
  ### Caractères autorisés
</div>

Le champ de nom d’utilisateur accepte les caractères suivants :

* Les caractères alphanumériques (sans accents, automatiquement convertis en minuscules) ;
* Le caractère arobase (@) (mais les adresses courriel ne sont pas autorisées) ;
* Le caractère accent circonflexe (^) ;
* Le caractère signe dollar ($) ;
* Le caractère point (.) ;
* Le caractère point d’exclamation (!) ;
* Le caractère accent grave (&#96;) ;
* Le caractère trait d’union (-) ;
* Le caractère dièse (#) ;
* Le caractère signe plus (+) ;
* Le caractère apostrophe (&#39;) ;
* Le caractère tilde (~) ;
* Le caractère soulignement (&#95;) ;

Aucun autre caractère ou symbole n’est autorisé, et Auth0 ne valide ni n’assainit les entrées de base de données personnalisée.
