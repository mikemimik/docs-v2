---
description: Découvrez comment le pipeline conforme à OIDC affecte le Client Credentials Flow.
title: Client Credentials Flow avec OIDC
---

Le pipeline conforme à OIDC permet d’utiliser le [Client Credentials Flow](/fr-CA/docs/get-started/authentication-and-authorization-flow/client-credentials-flow), qui permet aux applications de s’authentifier elles-mêmes (plutôt qu’au nom d’un utilisateur) pour obtenir, de manière programmatique et sécuritaire, l’accès à une API.

Cet échange n’existe pas dans le pipeline hérité; on utilise plutôt le [flux du mot de passe du propriétaire de la ressource](/fr-CA/docs/get-started/authentication-and-authorization-flow/resource-owner-password-flow) pour le simuler en créant un **utilisateur de service**. Nous déconseillons fortement cette approche au profit du Client Credentials Flow, qui permet de définir des autorisations granulaires pour chaque application API.

Pour savoir comment exécuter un Client Credentials Flow, consultez [Call API Using the Client Credentials Flow](/fr-CA/docs/get-started/authentication-and-authorization-flow/client-credentials-flow/call-your-api-using-the-client-credentials-flow).
