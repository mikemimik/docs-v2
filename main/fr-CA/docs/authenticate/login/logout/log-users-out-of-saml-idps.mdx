title: Déconnecter les utilisateurs des fournisseurs d’identité SAML
---

La déconnexion unique (SLO) est une fonctionnalité qui permet à un utilisateur de mettre fin à plusieurs sessions d’authentification en effectuant une seule action de déconnexion.

Auth0 prend en charge le SLO lorsque vous [connectez votre application à un fournisseur d’identité SAML (IdP)](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/saml) et offre une prise en charge limitée du SLO lorsque vous [configurez Auth0 comme IdP SAML](/fr-CA/docs/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

<div id="configure-slo-when-application-is-connected-to-a-saml-idp">
  ## Configurer le SLO lorsqu’une application est connectée à un IdP SAML
</div>

Si vous souhaitez qu’Auth0 [déconnecte un utilisateur de son fournisseur d’identité](/fr-CA/docs/authenticate/login/logout/log-users-out-of-idps), incluez le paramètre `federated` lorsque vous appelez le [point de terminaison de Logout de l’Authentication API Auth0](https://auth0.com/docs/api/authentication#logout).

Vous devrez peut-être configurer des paramètres supplémentaires pour la connexion <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SAML">SAML</Tooltip> afin de vous assurer qu’Auth0 envoie la demande de déconnexion au point de terminaison de Logout de l’<Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=IdP">IdP</Tooltip> SAML :

1. Accédez à [Auth0 Dashboard &gt; Authentication &gt; Enterprise &gt; SAML](https://manage.auth0.com/#/connections/enterprise/samlp) et sélectionnez votre connexion.
2. Activez **Enable Sign Out**.
3. Saisissez l’URL de Logout SAML fournie par l’IdP SAML dans le champ **Sign Out URL**. Si vous laissez ce champ vide, Auth0 utilisera par défaut l’URL du champ **Sign In URL**.
4. Vérifiez que la valeur du champ **Protocol Binding** correspond à la liaison de protocole attendue par l’IdP SAML.

<div id="configure-slo-when-auth0-is-the-saml-idp">
  ## Configurer le SLO lorsque Auth0 est le SAML IdP
</div>

Lorsque Auth0 est le SAML IdP, il y a deux scénarios de logout à prendre en compte :

* À l’initiative de l’application
* À l’initiative de l’IdP

Dans les deux cas, vous devez [configurer le module complémentaire SAML2 Web App](/fr-CA/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) pour indiquer où envoyer les réponses de logout :

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et sélectionnez votre application.
2. Allez à l’onglet **Addons** et sélectionnez **SAML2 Web App**.
3. Allez à l’onglet **Settings**.
4. Repérez la section **Settings** et ajoutez (ou mettez à jour) l’objet `logout` avec les propriétés suivantes :

   * `callback` : Entrez l’URL de SLO de l’application.
   * `slo_enabled` : Entrez `true` si vous voulez activer le SLO à l’initiative de l’IdP, où Auth0 envoie une requête de logout SAML à l’application lorsque l’utilisateur se déconnecte de l’Auth0 SAML IdP. Sinon, entrez `false`.

     ```bash cURL lines
     {
       "logout": {
         "callback": "https://myapp.exampleco.com/saml/logout",
         "slo_enabled": true
       }
     }
     ```

Par défaut, Auth0 envoie les réponses de logout SAML avec la liaison de protocole `HTTP-POST`. Si votre application attend la liaison de protocole `HTTP-Redirect`, ajoutez (ou mettez à jour) la propriété `binding` comme suit :

```json JSON lines
{
  "binding": "urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect"
}
```

<div id="application-initiated-logout">
  ### Déconnexion initiée par l’application
</div>

Lorsque l’utilisateur initie la déconnexion depuis une application, celle-ci doit envoyer une requête au point de terminaison [SAML Logout](https://auth0.com/docs/api/authentication#saml-logout) de l’Authentication API afin de déclencher le SLO.

Si votre application ne prend pas en charge le SLO, mais qu’elle prend en charge une URL de redirection vers laquelle envoyer l’utilisateur après la déconnexion, définissez l’URL de redirection comme étant le point de terminaison [OIDC Logout](https://auth0.com/docs/api/authentication#oidc-logout) de l’Authentication API (ou le point de terminaison [Auth0 Logout](https://auth0.com/docs/api/authentication#auth0-logout) de l’Authentication API si vous utilisez la déconnexion Legacy). Auth0 n’informera pas les autres participants à la session qu’une déconnexion a été initiée, mais nous supprimerons la [session Auth0](/fr-CA/docs/manage-users/sessions/session-layers) de l’utilisateur.

<div id="idp-initiated-logout">
  ### Déconnexion initiée par l’IdP
</div>

Lorsqu’un utilisateur lance la déconnexion depuis un Auth0 SAML IdP, Auth0 envoie une réponse de déconnexion SAML à l’URL indiquée dans la propriété `logout.callback`, dans les Settings du module complémentaire SAML2 Web App. Vous ne pouvez spécifier qu’une seule URL pour cette propriété.

En raison de cette contrainte, Auth0 ne prend pas nativement en charge le scénario où un utilisateur doit être déconnecté de plusieurs applications connectées à votre Auth0 SAML IdP.

Vous pouvez obtenir un fonctionnement semblable au SLO en configurant vos applications pour qu’elles vérifient régulièrement l’état de la session Auth0 de l’utilisateur et mettent fin à la session de l’application en conséquence. Pour en savoir plus, consultez [les limites de durée de vie des sessions](/fr-CA/docs/manage-users/sessions/session-lifetime-limits).
