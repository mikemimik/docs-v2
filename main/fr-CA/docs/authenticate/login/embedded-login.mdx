title: Embedded Login
---

Embedded Login permet à vos utilisateurs de se connecter directement à votre application et de transmettre leurs identifiants au serveur Auth0 pour l’authentification. Nous ne recommandons pas d’utiliser Embedded Login. Pour en savoir plus, consultez [Centralized Universal Login vs. Embedded Login](/docs/fr-CA/authenticate/login/universal-vs-embedded-login).

Si vous décidez d’utiliser Embedded Login, vous devez [configurer votre application pour le partage de ressources entre origines multiples](/docs/fr-CA/get-started/applications/set-up-cors) et vous devriez [configurer un domaine personnalisé](/docs/fr-CA/customize/custom-domains). Vous pourrez ensuite implémenter le [SDK Lock](/docs/fr-CA/libraries/lock) ou le [SDK Auth0.js](/docs/fr-CA/libraries/auth0js) dans votre application, ou appeler directement l’[API d’authentification Auth0](https://auth0.com/docs/api/authentication).

Auth0 prend en charge Embedded Login avec des connexions <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip> pour plusieurs types d’application :

* [Embedded Passwordless Login in Native Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/native)
* [Embedded Passwordless Login in Regular Web Applications](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Embedded Passwordless Authentication for Single-Page Apps](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/spa)
