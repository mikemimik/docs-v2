---
title: "Intégrations d’authentification unique SAML"
description: "Décrit les options d’intégration du langage Security Assertion Markup Language (SAML) pour l’authentification unique (SSO)."
---

Lorsque vous mettez en place l’<Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=single+sign-on">authentification unique</Tooltip> (SSO), il est important de tenir compte des éléments suivants :

* Quel(s) système(s) serviront de source de référence pour les informations du profil utilisateur en cas de conflit entre deux sources ou plus.
* Quels attributs du profil utilisateur sont nécessaires pour chaque application.
* Comment les informations du profil utilisateur seront distribuées aux systèmes qui en ont besoin.

Vous configurez généralement une fédération SAML en configurant le SSO initié par le fournisseur de services. Le fournisseur de services renvoie une redirection du navigateur afin que l’utilisateur s’authentifie à l’aide de l’<Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=IdP">IdP</Tooltip>. Après l’authentification, le navigateur redirige l’utilisateur vers le fournisseur de services avec une assertion SAML contenant des renseignements sur le statut de l’authentification. Cette approche est couramment utilisée dans les scénarios destinés aux consommateurs.

Vous pouvez aussi configurer l’IdP pour qu’il initie le SSO plutôt que le fournisseur de services. Dans ce scénario, l’utilisateur accède à une URL sur l’IdP et est invité à s’authentifier, puis il est redirigé vers le fournisseur de services avec une assertion SAML. Cette approche est couramment utilisée dans les scénarios d’entreprise.

Nous avons de nombreuses [intégrations SSO qui utilisent Auth0 comme IdP](/fr-CA/docs/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider). Nous proposons également des [intégrations SSO pour d’autres IdP](/fr-CA/docs/authenticate/protocols/saml/saml-sso-integrations/configure-auth0-saml-service-provider).
