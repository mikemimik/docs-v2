  Nous vous recommandons fortement d’utiliser Actions pour étendre Auth0. Avec Actions, vous avez accès à de l’information de type enrichie, à de la documentation intégrée et à des packages `npm` publics, et vous pouvez connecter des intégrations externes qui améliorent votre expérience globale d’extensibilité. Pour en savoir plus sur ce qu’Actions offre, consultez [Comprendre le fonctionnement d’Auth0 Actions](/fr-CA/docs/customize/actions/actions-overview).

  Pour vous aider dans votre migration, nous proposons des guides pour vous aider à [migrer de Rules vers Actions](/fr-CA/docs/customize/actions/migrate/migrate-from-rules-to-actions) et à [migrer de Hooks vers Actions](/fr-CA/docs/customize/actions/migrate/migrate-from-hooks-to-actions). Nous avons aussi une page dédiée, [Passer à Actions](https://auth0.com/extensibility/movetoactions), qui présente des comparaisons de fonctionnalités, [une démo d’Actions](https://www.youtube.com/watch?v=UesFSY1klrI) et d’autres ressources pour vous accompagner dans votre migration.

  Pour en savoir plus sur la dépréciation des Rules et des Hooks, consultez notre billet de blogue : [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

Avec les [rules](/fr-CA/docs/customize/rules), vous pouvez gérer des cas plus complexes que ce qu’il est possible de faire avec les [connexions sans mot de passe](/fr-CA/docs/authenticate/passwordless) seules. Par exemple, vous pouvez ajouter des mesures supplémentaires pour mieux confirmer la possession d’une adresse courriel ou d’un appareil.

<div id="require-multi-factor-authentication-for-users-who-are-outside-the-corporate-network">
  ## Exiger l’authentification multifacteur pour les utilisateurs à l’extérieur du réseau d’entreprise
</div>

Supposons que vous vouliez exiger [l’authentification multifacteur (MFA)](/fr-CA/docs/secure/multi-factor-authentication) pour tout utilisateur qui accède à l’application au moyen d’une connexion <Tooltip tip="Sans mot de passe : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=passwordless">sans mot de passe</Tooltip> depuis l’extérieur de votre réseau d’entreprise.

À l’aide d’une règle, vous pouvez vérifier si un utilisateur s’authentifie au moyen d’une méthode sans mot de passe (`sms`, `email`) et, si l’adresse IP de sa session se trouve à l’extérieur du réseau d’entreprise désigné, lui demander un deuxième facteur d’authentification.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pourriez aussi déclencher cette règle en fonction d’autres critères, par exemple si l’adresse IP actuelle correspond à la liste d’autorisation d’IP de l’utilisateur ou si sa géolocalisation indique qu’il se trouve dans un pays différent de celui indiqué dans son profil utilisateur.
</Callout>

Pour ce faire, vous devez [créer la règle suivante](/fr-CA/docs/customize/rules/create-rules) :

```javascript lines
function(user, context, callback) {
  const ipaddr = require('ipaddr.js');
  const corp_network = "192.168.1.134/26";
  const current_ip = ipaddr.parse(context.request.ip);
  // la méthode d'auth est-elle sans mot de passe et l'IP est-elle hors du réseau d'entreprise ?
  const passwordlessOutside = context.authentication.methods.find(
    (method) => (
      ((method.name === 'sms') || (method.name === 'email')) && 
      (!current_ip.match(ipaddr.parseCIDR(corp_network)))
    )
  );

  // si oui, exiger l'AMF
  if (passwordlessOutside) {
    context.multifactor = {
      provider: 'any',
      allowRememberBrowser: false
    };
  }
  callback(null, user, context);
}
```
