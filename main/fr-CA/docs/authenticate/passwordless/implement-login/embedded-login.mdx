---
title: Authentification Passwordless avec connexion intégrée
description: Décrit comment mettre en œuvre l’authentification Passwordless avec Universal Login.
---

Pour intégrer l’interface de connexion à votre application, vous pouvez utiliser notre [API Passwordless pour la connexion intégrée](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/relevant-api-endpoints) ou nos SDKs.

Selon le type d’application que vous voulez créer, la mise en œuvre diffère :

* Pour les applications à page unique (SPA) (par exemple, Angular ou React), consultez [Authentification Passwordless intégrée pour les applications à page unique (SPA)](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/spa).
* Pour les applications natives (par exemple, iOS, Android ou les applications de bureau), consultez [Connexion Passwordless intégrée dans les applications natives](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/native).
* Pour les applications Web régulières (par exemple, NodeJS, Java, Rails ou .NET), consultez [Connexion Passwordless intégrée dans les applications Web régulières](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/webapps).

La connexion intégrée dans les applications Web régulières et les SPA nécessite la configuration de l’authentification cross-origin. Pour en savoir plus, consultez [Authentification cross-origin](/fr-CA/docs/authenticate/login/cross-origin-authentication).
