---

Avec les connexions <Tooltip tip="Sans mot de passe : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Passwordless">sans mot de passe</Tooltip>, les utilisateurs peuvent ouvrir une session sans mot de passe. À la place, ils peuvent utiliser différentes autres méthodes d’authentification. Les connexions sans mot de passe d’Auth0 prennent en charge les méthodes d’authentification suivantes :

| Facteur                                                                                 | Description                                                                                                                                                                                                                                                                               |
| --------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Courriel](/fr-CA/docs/authenticate/passwordless/authentication-methods/email-otp)            | L’utilisateur est invité à saisir son adresse courriel, puis Auth0 y envoie un code à usage unique. L’utilisateur saisit ensuite ce code dans votre application.                                                                                                                          |
| [Lien magique](/fr-CA/docs/authenticate/passwordless/authentication-methods/email-magic-link) | L’utilisateur est invité à saisir son adresse courriel, puis Auth0 y envoie un courriel contenant un lien. L’utilisateur clique sur le lien et ouvre directement une session dans votre application.                                                                                      |
| [SMS](/fr-CA/docs/authenticate/passwordless/authentication-methods/sms-otp)                   | L’utilisateur est invité à saisir son numéro de téléphone, puis Auth0 y envoie un code à usage unique. Par défaut, Auth0 utilise Twilio pour envoyer le code, mais si vous avez une passerelle SMS personnalisée, vous pouvez modifier votre connexion pour utiliser celle-ci à la place. |
