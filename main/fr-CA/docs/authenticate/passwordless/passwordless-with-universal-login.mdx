L’expérience <Tooltip tip="Universal Login : votre application est redirigée vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité de l’utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> prend en charge les [connexions Passwordless](/fr-CA/docs/authenticate/passwordless), qui permettent aux utilisateurs d’entrer un numéro de téléphone ou une adresse Email, puis de recevoir un mot de passe à usage unique (OTP) pour terminer l’authentification.

<div id="enable-passwordless-for-universal-login">
  ## Activer Passwordless pour Universal Login
</div>

Pour configurer Universal Login afin qu’il prenne en charge les connexions <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Passwordless">Passwordless</Tooltip>, vous devrez définir le Authentication Profile de façon à utiliser un parcours de connexion pris en charge, puis mettre à jour votre application pour indiquer la connexion au moment de l’authentification.

<div id="enable-identifier-first">
  ### Activer Identifier First
</div>

Dans le <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, accédez à [Authentication &gt; Authentication Profile.](https://manage.auth0.com/?/authentication-profiles)

Sélectionnez **Identifier First** ou **Identifier First + Biometrics**.

<div id="update-your-application">
  ### Mettez à jour votre application
</div>

Selon le type de connexions que vous avez activées pour votre application, vous devrez peut-être la mettre à jour afin d’indiquer une connexion Passwordless lors de la connexion :

| Connexion de base de données | Connexion Passwordless | Faut-il l’indiquer ? | Expérience utilisateur                                                                                       |
| ---------------------------- | ---------------------- | -------------------- | ------------------------------------------------------------------------------------------------------------ |
| Oui                          | SMS et/ou Email        | Oui                  | La connexion Passwordless indiquée est présentée lors de la connexion.                                       |
| Non                          | SMS et Email           | Non                  | La connexion Passwordless créée en premier est présentée.                                                    |
| Non                          | SMS ou Email           | Non                  | La connexion Passwordless est présentée lors de la connexion (avec toutes les connexions sociales activées). |

Pour indiquer une connexion Passwordless lors de la connexion, vous devez transmettre le paramètre `connection` et sa valeur (`sms` ou `email`) au [point de terminaison Login de l’Auth0 Authentication API](https://auth0.com/docs/api/authentication#login) :

`https://mytenant.us.auth0.com/authorize?client_id={id}&``connection={sms|email}``&scope=…&response_type=code&response_mode=query&state=…&redirect_uri=http%3A%2F%2Flocalhost%3A3000&code_challenge=…`

Si vous utilisez l’un de nos SDKs, vous pouvez indiquer une connexion Passwordless lors de l’initialisation. Par exemple :

```js lines
auth0 = await createAuth0Client({
     domain: config.domain,
     client_id: config.clientId,
     connection: "email"
  });

auth0 = await createAuth0Client({
     domain: config.domain,
     client_id: config.clientId,
     connection: "sms"
  });
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous utilisez Passwordless avec Universal Login et [Single Sign-On](/fr-CA/docs/authenticate/single-sign-on), les paramètres de connexion `sms` et `email` n’utilisent pas la session Auth0 existante, et l’utilisateur sera invité à se connecter.
</Callout>
