<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’authentification multifacteur (MFA) n’est pas compatible avec les transactions d’échange de jeton personnalisé dans lesquelles un acteur est défini au moyen de `api.authentication.setActor()`. Si la MFA est requise (par une politique ou une Action Post-Login) et que l’Action d’échange de jeton personnalisé définit un acteur, la transaction échoue avec une erreur `400` : `MFA is not supported using actor_token with the requested token exchange profile.`
</Callout>

Pour ajouter une protection contre le vol de jeton ou d’autres risques de sécurité, vous pouvez ajouter l’authentification multifacteur à une requête d’échange de jeton personnalisé à l’aide de l’une des méthodes suivantes :

* [Activer et configurer votre politique MFA ](/fr-CA/docs/secure/multi-factor-authentication/enable-mfa) pour votre tenant
* Utiliser `api.multifactor.enable()` dans une Action Post-Login

Lorsque vous ajoutez la MFA, le serveur d’autorisation Auth0 rejette l’appel initial d’échange de jeton personnalisé vers le endpoint `/token` avec une erreur `mfa_required`. Vous pouvez utiliser cette erreur dans l’[API MFA Auth0](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api) pour lancer la vérification d’un facteur afin d’obtenir les jetons d’accès, d’identité et d’actualisation Auth0 demandés.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’échange de jeton personnalisé ne prend pas en charge `api.authentication.challengeWith()` ni `api.authentication.enrollWith()`. Si vous utilisez ces méthodes avec votre Action Post-Login, la transaction échouera avec une erreur irrécupérable. Assurez-vous de ne pas utiliser ces méthodes lorsque `event.transaction.protocol==oauth2-token-exchange`, selon la valeur de `subject_token_type`.
</Callout>

Lorsqu’une transaction d’échange de jeton personnalisé est associée à une Organization, `api.multifactor.enable()` et les politiques MFA ne sont pas non plus pris en charge.

Pour en savoir plus sur l’utilisation du grant MFA, consultez [Authenticate Using the Resource Owner Password Flow with MFA](/fr-CA/docs/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa), car l’échange de jeton personnalisé suit le même modèle. Vous pouvez aussi consulter un exemple détaillé dans [Cas d’utilisation : Effectuer la MFA pendant un échange de jeton personnalisé](/fr-CA/docs/authenticate/custom-token-exchange/cte-example-use-cases#use-case%3A-perform-mfa-during-custom-token-exchange).
