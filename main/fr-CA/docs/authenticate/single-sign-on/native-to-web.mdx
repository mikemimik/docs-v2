La fonctionnalité <Tooltip tip="Authentification unique (SSO) : service qui, après la connexion d’un utilisateur à une application, le connecte automatiquement aux autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Single+Sign-On">Single Sign-On</Tooltip> (SSO) Native to Web d’Auth0 offre une expérience utilisateur fluide, permettant aux utilisateurs authentifiés de passer de votre [application native](/docs/fr-CA/authenticate/login/native-login) à votre application web.

Si votre application web s’appuie sur des WebViews intégrées ou des navigateurs externes pour offrir des fonctionnalités étendues ou avancées, vos utilisateurs peuvent passer de l’environnement natif à l’environnement web dans une même session authentifiée.

En liant les Session Transfer Tokens à l’appareil concerné au moyen de l’adresse IP ou de l’ASN, on préserve la continuité et la sécurité de la session, ce qui garantit que le contexte d’authentification demeure sécurisé tout au long de la transition.

<div id="how-it-works">
  ## Fonctionnement
</div>

<Frame>![Flux de travail Native to Web SSO](/docs/images/cdy7uua7fh8z/4gFE2RP5ZCEcukiNF3Gpae/f957034f63c36d5a11f6a64d988c00e9/sequence-diagram.png)</Frame>

1. Un utilisateur se connecte à votre application native.
2. Auth0 authentifie l’utilisateur et renvoie un jeton d’accès, un jeton d’actualisation et un jeton d’identité.
3. Votre application native appelle le point de terminaison [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow-with-pkce/get-token-pkce) d’Auth0 pour échanger un jeton d’actualisation contre un Session Transfer Token. Le Session Transfer Token est lié à une adresse IP ou à un ASN précis.
4. Auth0 renvoie le Session Transfer Token pour l’authentification avec une application web.
5. Auth0 autorise le Session Transfer Token dans le cookie ou comme paramètre d’URL, puis renvoie un [code d’autorisation](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow).
6. L’application web échange le code d’autorisation contre des jetons d’accès ou d’actualisation au point de terminaison [`/token`](https://auth0.com/docs/api/authentication/authorization-code-flow/get-token).
7. L’application web initialise une session pour l’utilisateur.

Découvrez comment [configurer et implémenter Native to Web SSO](/docs/fr-CA/authenticate/single-sign-on/native-to-web/configure-implement-native-to-web)

<div id="limitations">
  ## Limites
</div>

* Une fois que Native to Web SSO est activé dans une application, le paramètre `session_transfer_token` fonctionne uniquement avec Native to Web SSO.
* Les <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un nouvel access token sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Refresh+Tokens">Refresh Tokens</Tooltip> issus d’une transaction précédente de Session Transfer Token ne génèrent pas de nouveaux Session Transfer Tokens.
