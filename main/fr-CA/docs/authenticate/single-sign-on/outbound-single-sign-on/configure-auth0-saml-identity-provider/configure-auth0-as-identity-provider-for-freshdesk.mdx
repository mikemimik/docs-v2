description: Découvrez comment configurer Auth0 comme fournisseur d'identité à l'aide du module complémentaire SAML2 Web App pour Freshdesk.
title: Configurer Freshdesk comme fournisseur de services SAML
---

Utilisez la configuration <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d'échanger des renseignements d'authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SAML">SAML</Tooltip> suivante pour Freshdesk.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis créez une nouvelle application ou cliquez sur le nom d&#39;une application à mettre à jour.

2. Accédez à l&#39;onglet **Addons** et activez la bascule **SAML2 Web App**.

3. Dans l&#39;onglet **Settings**, définissez **Application Callback URL** sur : `https://{FD Domain}.freshdesk.com/login/saml`.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>

   1. Elle se trouve à l&#39;emplacement suivant :
      **Freshworks Dashboard &gt; Security &gt; SAML SSO &gt; Assertion Consumer Service(ACS) URL**
   2. L&#39;audience se trouve à l&#39;emplacement suivant :
      **Freshworks Dashboard &gt; Security &gt; SAML SSO &gt; Service Provider(SP) Entity ID URL**

4. Collez le code suivant dans la zone de texte **Settings**, puis cliquez sur **Debug**.

   ```json lines
   {
         "audience": "https://{FD Domain}.freshworks.com/sp/SAML/{xxxxxxxxxxxxxxxxxxx}/metadata",
         "mappings": {
       "email": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
       "given_name": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/givenname",
       "family_name": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/surname"
         },
         "createUpnClaim": false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities": false,
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
         ]
       }
   ```

5. Faites défiler jusqu&#39;au bas de la page et cliquez sur **Enable**.

6. Dans l&#39;onglet **Usage**, repérez **Identity Provider Metadata**, puis cliquez sur **Download** pour télécharger le fichier de métadonnées. Vous en aurez besoin lorsque vous configurerez Auth0 comme fournisseur d&#39;identité.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Usage Tab](/docs/images/cdy7uua7fh8z/3T90BDpyTXFUWDp1JkncBU/86bdba6bed8130b34b42b25c9bf2d232/dashboard-applications-applications-addons-saml2-web-app-usage.png)</Frame>
