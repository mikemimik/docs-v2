---
title: profil de connexion
description: Utilisez un profil de connexion pour préciser comment les paramètres privés d’une connexion Auth0 doivent être configurés lorsqu’elle est créée par des tiers.
---

Auth0 fournit des connexions Enterprise pour authentifier les utilisateurs auprès d’un fournisseur d’identité fédéré (IdP) externe, comme Okta, Microsoft Entra ID, Google Workspace, entre autres. Une connexion configurée comprend des paramètres propres au protocole pour intégrer l’authentification unique (SSO), le provisionnement des utilisateurs et les intégrations de logout avec l’IdP externe, ainsi que des paramètres privés qui régissent la façon dont la connexion interagit avec les fonctionnalités Universal Login et Organizations d’Auth0.

Le profil de connexion (CP) permet aux développeurs Auth0 de préciser comment les paramètres privés d’une connexion Auth0 doivent être configurés lorsqu’elle est créée par des tiers

<div id="how-it-works">
  ## Fonctionnement
</div>

* **Définition du profil** : Un administrateur crée un profil de connexion qui définit les valeurs de propriété à inscrire dans la connection chaque fois qu’elle est créée au moyen de l’une des fonctionnalités de l’administration déléguée d’Auth0.

* **Portée étendue** : Les profils sont actuellement associés aux flows Self-Service Enterprise Configuration et Okta Express Configuration, mais ils sont conçus pour être réutilisés plus largement, notamment pour le provisionnement, l’onboarding, la gestion des droits et les futures capacités d’Auth0.

<div id="connection-profile-properties">
  ## Propriétés de profil de connexion
</div>

Un profil de connexion est un objet JSON qui prend en charge ces propriétés configurables, lesquelles sont appliquées à toutes les connexions nouvellement créées.

| **Propriété**                             | **Description**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `connection_name_prefix_template`         | Représente le préfixe à utiliser pour nommer les connexions. Correspond au paramètre `name` d’une connexion. Cette valeur prend en charge la substitution de variables pour l’ID de l’organisation et le nom de l’organisation. Le flux de travail OIN doit substituer ces valeurs pour générer le préfixe final. Les variables sont placées entre accolades ({}). <p /> Variables prises en charge : <p /> <ul><li>`org_id` : l’ID de l’organisation</li><li>`org_name` : le [nom de l’organisation](/fr-CA/docs/manage-users/organizations/configure-organizations/create-organizations#auth0-dashboard)</li></ul> <p /> Exemple : `con-{org_id}-` |
| `enabled_features`                        | Cette liste précise les fonctionnalités prises en charge par la connexion configurée. Les fonctionnalités qui ne figurent pas dans la liste ne sont pas autorisées.<p /> Valeurs prises en charge : <ul><li>`scim` : lorsqu’elle est présente, SCIM peut être configuré sur la connexion.</li><li>`universal_logout` : lorsqu’elle est présente, la fonctionnalité Universal Logout peut être utilisée avec cette connexion.</li></ul>                                                                                                                                                                                                         |
| `organization.assign_membership_on_login` | Indique si les utilisateurs doivent automatiquement se voir attribuer une appartenance à l’organisation lors de la connexion. Cela correspond à la propriété `assign_membership_on_login` de la sous-ressource `enabled_connections` de l’organisation.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Pour les intégrations Express Configuration avec Okta, `optional` est traité comme `required`</Callout> <p />                                                                                                                                                                                                          |

| Valeur de profil de connexion  | Valeur `assign_membership_on_login` de Enable Connections                                                                                                                                                   |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `none`                        | `false`                                                                                                                                                                                                     |
| `optional`                    | `false`                                                                                                                                                                                                     |
| `required`                    | `true`                                                                                                                                                                                                      |
| `organization`                | Un objet contenant des paramètres appliqués à l’organisation sous laquelle la connexion est créée.                                                                                                          |
| `organization.show_as_button` | Indique si la connexion doit être affichée sous forme de bouton sur l’écran de connexion une fois qu’une organisation a été choisie. Cela correspond à la valeur `show_as_button` sur la connexion activée. |

| Valeur de profil de connexion | Valeur `show_as_button` de la connexion |
| ---------------------------- | --------------------------------------- |
| `none`                       | `false`                                 |
| `optional`                   | `true`                                  |
| `required`                   | `true`                                  |

<div id="example-connection-profile">
  ## Exemple de profil de connexion
</div>

```json
{
  "organization": {
    "show_as_button": "none",
    "assign_membership_on_login": "none"
  },
  "connection_name_prefix_template": "ec-{org_id}-",
  "enabled_features": [
    "scim",
    "universal_logout"
  ]
}
```

<div id="create-and-manage-connection-profiles">
  ## Créer et gérer les profils de connexion
</div>

Un profil de connexion est généré automatiquement lorsque vous utilisez la fonctionnalité [Express Configuration](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/okta/express-configuration) avec Okta. Vous pouvez aussi créer et gérer des profils de connexion à l’aide de l’Auth0 Management API.

<div id="configure-with-management-api">
  ### Configurer avec la Management API
</div>

Pour utiliser la Management API, vous devez obtenir un [jeton d’accès à la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens). Pour gérer les profils de connexion, les endpoints suivants de la Management API sont disponibles :

* `POST` `/api/v2/connection-profiles`
* `GET` `/api/v2/connection-profiles`
* `PATCH` `/api/v2/connection-profiles/{id}`
* `GET` `/api/v2/connection-profiles/{id}`
* `GET` `/api/v2/connection-profiles/templates`
