title: Créer une interface Delegated Admin avec My Organization API
description: Décrit comment utiliser Universal Components pour créer des interfaces d’administration déléguée avec la My Organization API d’Auth0.
sidebarTitle: Créer une interface Delegated Admin
---

Universal Components vous permet de créer un tableau de bord d’administration en libre-service dans votre propre application. Cela permet à vos clients B2B, désignés comme administrateurs de votre [Auth0 Organization](/fr-CA/docs/manage-users/organizations), de gérer eux-mêmes leur image de marque, leurs fournisseurs d’identité et leurs domaines vérifiés, sans avoir à accéder à Auth0 Dashboard ni à faire appel à une équipe de soutien pour des tâches administratives de base.

<div id="how-it-works">
  ## Fonctionnement
</div>

Universal Components s’appuie sur l’[My Organization API](/fr-CA/docs/api/myorganization), qui fonctionne avec des jetons propres à l’utilisateur, pour créer un [tableau de bord administratif SSO en libre-service](/fr-CA/docs/authenticate/enterprise-connections/self-service-enterprise-configuration). Lorsqu’un administrateur ouvre une session dans le tableau de bord intégré à votre application, le SDK Auth0 récupère un jeton d’accès contenant des scopes propres à l’Organization pour les privilèges d’administration. Les composants utilisent ce jeton pour appeler l’API au nom de l’utilisateur connecté, ce qui garantit que les administrateurs délégués ne peuvent modifier que l’Auth0 Organization à laquelle ils appartiennent.

<div id="available-components">
  ### Composants disponibles
</div>

Chaque composant est associé à des points de terminaison précis dans l’[My Organization API](/fr-CA/docs/api/myorganization) pour effectuer des tâches administratives.

| **Composant**                                                                                                                                                   | **Point de terminaison d’API**        |
| :-------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------ |
| **OrganizationDetailsEdit** — Modifier le nom de l’organisation, le nom d’affichage, les couleurs de l’image de marque et le logo.                              | `/my-org/details`                     |
| **SsoProviderTable** — Répertorier, activer, désactiver, supprimer et dissocier les fournisseurs d’identité SSO.                                                | `/my-org/identity-providers`          |
| **SsoProviderCreate** — Assistant en plusieurs étapes pour créer des fournisseurs SSO (Okta, ADFS, SAML, OIDC, Google Workspace, Azure AD, Ping Federate).      | `/my-org/identity-providers`          |
| **SsoProviderEdit** — Configurer les paramètres d’authentification, activer le provisionnement et gérer les jetons SCIM ainsi que les associations de domaines. | `/my-org/identity-providers/{idp_id}` |
| **DomainTable** — Créer, vérifier, supprimer et associer des domaines à des fournisseurs d’identité.                                                            | `/my-org/domains`                     |

<div id="configure-auth0-dashboard">
  ## Configurer Auth0 Dashboard
</div>

Avant d’utiliser les composants My Organization, vous devez configurer votre tenant Auth0 avec les [API](/fr-CA/docs/api), les applications et les [scopes](/fr-CA/docs/api/myorganization) appropriés.

    ## **Créer une application**

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Cet exemple utilise une application monopage (SPA). Pour les frameworks React à rendu côté serveur, consultez l’onglet Next.js.
    </Callout>

    1. Accédez à [**Auth0 Dashboard &gt; Applications &gt; Applications**](https://manage.auth0.com/#/applications). Sélectionnez **Create Application**.

    2. Choisissez **Applications Web monopage**.

    3. Dans l’onglet **Settings**, ajoutez `http://localhost:5173` dans les champs suivants :
       <br />A. Allowed Callback URLs
       <br />B. Allowed Logout URLs

    4. Dans **Expérience de connexion**, sélectionnez :
       <br />A. Utilisateurs d’entreprise.
       <br />B. (Facultatif) Invitation à indiquer l’organisation.

    ## **Activer l&#39;API My Organization**

    1. Accédez à [**Dashboard &gt; Applications &gt; APIs**](https://manage.auth0.com/#/applications).
    2. Sélectionnez **[My Organization API](/fr-CA/docs/api/myorganization)** et vérifiez qu&#39;elle est activée pour votre tenant.

    ## **Configurer l&#39;accès à l&#39;application**

    1. Accédez à l’onglet **Accès à l’application**.
    2. Sélectionnez **Modifier** pour l’application que vous souhaitez utiliser.
    3. Configurez les paramètres suivants :
       <br />A. [Profil de connexion](/fr-CA/docs/authenticate/enterprise-connections/connection-profile#connection-profile) : Sélectionnez ou créez un profil avec des mappages d’attributs de connexion.
       <br />B. [Profil des attributs utilisateur](/fr-CA/docs/authenticate/enterprise-connections/user-attribute-profile#user-attribute-profile) : Sélectionnez ou créez un profil avec des mappages d’attributs utilisateur.
       <br />C. Pris en charge [Fournisseurs d’identité](/fr-CA/docs/authenticate/identity-providers) : Activez les fournisseurs que vos clients peuvent utiliser.
       <br />D. Comportement de suppression de connexion : Choisissez **Autoriser** ou **Autoriser si vide**.
       <br />    • **Autoriser** : Les utilisateurs peuvent supprimer des connexions, ce qui supprime tous les utilisateurs de cette connexion.
       <br />    • **Autoriser si vide** : Les utilisateurs peuvent uniquement supprimer des connexions sans utilisateurs.
       <br />E. Autorisation d’accès des utilisateurs : Choisissez **Non autorisé**, **Autorisé** ou **Tous**.
       <br />    • **Non autorisé** : Aucune permission n’est autorisée.
       <br />    • **Autorisé** : Sélectionnez des permissions précises.
       <br />    • **Tous** : Inclut toutes les permissions actuelles et futures.
       <br />F. Autorisation d’accès des informations d’identification du client : Choisissez **Non autorisé**, **Autorisé** ou **Tous**.
    4. Sélectionnez **Enregistrer**.

    ## **Setup Database &amp; User**

    1. Accédez à [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) pour créer une connexion de base de données.
    2. Sélectionnez l’onglet **Applications** de la connexion, puis activez votre nouvelle application SPA.
    3. Créez un utilisateur de test dans cette connexion de base de données pour effectuer les premiers tests.

    ## **Configuration du rôle**

    1. Créez un rôle (p. ex., &quot;Admin de l’organisation&quot;).
    2. Ajoutez au rôle les portées requises de l’API My Organization API.

    <Accordion title="Autorisations requises">
      ```
          read:my_org:details
          update:my_org:details
          create:my_org:identity_providers
          read:my_org:identity_providers
          update:my_org:identity_providers
          delete:my_org:identity_providers
          update:my_org:identity_providers_detach
          create:my_org:identity_providers_domains
          delete:my_org:identity_providers_domains
          read:my_org:domains
          delete:my_org:domains
          create:my_org:domains
          update:my_org:domains
          read:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_scim_tokens
          delete:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_provisioning
          read:my_org:identity_providers_provisioning
          delete:my_org:identity_providers_provisioning
          read:my_org:configuration
      ```
    </Accordion>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Le token de l’utilisateur ne comprendra que les permissions présentes à la fois dans le rôle qui lui est attribué et dans les paramètres User Access Authorization configurés à l’étape précédente.
    </Callout>

    ## **Créer une organisation**

    1. Allez à [**Organizations &gt; Create Organization**](https://manage.auth0.com/#/organizations).
    2. Configurez l’organisation Auth0 :
       <br />A. Dans **Members** : ajoutez votre utilisateur test et attribuez-lui le rôle &quot;Organization Admin&quot;.
       <br />B. Dans **Connections** : activez votre connexion de base de données.

    ## **Configurer les variables d&#39;environnement**

    1. Créez un fichier `.env` à la racine de votre projet React :

    ```bash
    VITE_AUTH0_DOMAIN=your-domain.auth0.com
      VITE_AUTH0_CLIENT_ID=your-spa-client-id
    ```
