description: Décrit comment activer l’authentification unique (SSO) pour un tenant à l’aide de l’Auth0 Dashboard. À utiliser uniquement avec les tenants existants.
title: Activer l’authentification unique pour les tenants
---

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  À utiliser uniquement avec les tenants existants. Par défaut, le Seamless SSO est activé pour tous les nouveaux Auth0 tenants, mais les tenants existants peuvent choisir d’activer ou non cette fonctionnalité.
</Callout>

Vous pouvez activer le <Tooltip tip="authentification unique (SSO) : service qui, après la connexion d’un utilisateur à une application, le connecte automatiquement aux autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=SSO">SSO</Tooltip> pour votre tenant à l’aide de l’<Tooltip tip="authentification unique (SSO) : service qui, après la connexion d’un utilisateur à une application, le connecte automatiquement aux autres applications." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Si vous choisissez de ne pas activer le SSO au niveau du tenant, vous pouvez [l’activer par application](/docs/fr-CA/get-started/applications/enable-sso-for-applications). Si ce paramètre n’est pas disponible dans le Dashboard, c’est que Seamless SSO est déjà activé; ce réglage s’applique uniquement aux tenants existants.

1. Allez à [Dashboard &gt; Tenant Settings](https://manage.auth0.com/#/tenant).

   <Frame>![Dashboard Tenant Settings onglet General Settings](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. Allez à l’onglet **Advanced**.
3. Faites défiler jusqu’à la section **Log In Session Management**. Repérez **Enable Seamless SSO**.
4. Saisissez les valeurs des champs **Inactivity timeout** (4320 minutes par défaut) et **Require log in after** (10080 minutes par défaut).

Une fois terminé, vous devriez aussi [configurer vos paramètres de durée de session](/docs/fr-CA/manage-users/sessions/configure-session-lifetime-settings).
