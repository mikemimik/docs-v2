Vous pouvez effectuer périodiquement une rotation manuelle d’une clé de signature afin de changer la clé JSON Web Key (JWK) utilisée par les applications et les API pour valider les jetons. Si votre application ou API **ne prend pas en charge** ce changement de clé et qu’elle tente d’utiliser une clé de signature expirée pour vérifier un jeton, la requête d’authentification échouera.

<Warning>
  Auth0 recommande d’effectuer d’abord la rotation de la clé de signature dans un tenant de développement, puis de vérifier que vos applications et vos API fonctionnent toujours comme prévu. Une fois que vous avez confirmé que tout fonctionne correctement, effectuez la même rotation de clé de signature dans votre tenant de production.
</Warning>

Même si Auth0 ne signe qu’avec une seule clé de signature à la fois, le document de découverte <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les informations de connexion." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) de votre tenant contient toujours plusieurs clés. Le document de découverte OIDC inclut toujours à la fois la clé actuelle et la clé suivante, et il peut aussi inclure la clé précédente si celle-ci n’a pas encore été révoquée. Afin d’assurer une expérience sans interruption en cas d’urgence, votre application devrait pouvoir utiliser n’importe laquelle des clés indiquées dans le document. Pour en savoir plus sur les documents de découverte OpenID Connect, consultez [ensembles de clés Web JSON](/fr-CA/docs/secure/tokens/json-web-tokens/locate-json-web-key-sets).

<Warning>
  Pour vous laisser le temps de mettre à jour votre application avec la nouvelle clé de signature, tous les jetons signés avec la clé précédente demeureront valides jusqu’à ce que vous révoquiez la clé précédente. Pour en savoir plus, consultez [Revoke Signing Keys](/fr-CA/docs/get-started/tenant-settings/signing-keys/revoke-signing-keys).
</Warning>

Vous pouvez faire la rotation de la clé de signature de l’application de votre tenant dans le <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou au moyen de l’Auth0 <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>.&quot;

<div id="use-the-dashboard">
  ## Utilisez le Dashboard
</div>

1. Accédez à [Dashboard &gt; Settings &gt; Signing Keys](https://manage.auth0.com/#/tenant/signing_keys).

   <Frame>![Onglet Signing Keys dans les Tenant Settings du Dashboard](/docs/images/cdy7uua7fh8z/7r8t3EGctFmvkCgPrU0i2R/f79ac74dac5cded37b39bd8a8b80951d/dashboard-tenant-settings-signing-keys.png)</Frame>
2. Sous **Rotation Settings**, repérez **Rotate Signing Key**, puis sélectionnez **Rotate Key**.
3. Cliquez sur **Rotate** pour confirmer.

   <Frame>![Confirmation de la rotation dans l’onglet Signing Keys des Settings du Dashboard](/docs/images/cdy7uua7fh8z/6Ofp24mgt8ZrvaxF2wLA5N/5cca1a3267ffdc07f61d7990b4e85989/Rotating_Key_Warning_-_English.png)</Frame>

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

1. Pour obtenir la liste des clés de signature, effectuez un appel `GET` au point de terminaison [Obtenir toutes les clés de signature de l’application](https://auth0.com/docs/api/management/v2#!/Keys/get_signing_keys).
2. Pour faire pivoter la clé de signature, effectuez un appel `POST` au point de terminaison [Faire pivoter la clé de signature de l’application](https://auth0.com/docs/api/management/v2#!/Keys/post_signing_keys). Assurez-vous de remplacer la valeur d’espace réservé `MGMT_API_ACCESS_TOKEN` par votre jeton d’accès à la Management API.

<div id="key-rotation-impact">
  ## Impact de la rotation des clés
</div>

<div id="apis-and-api-gateways-accepting-access-tokens">
  ### API et passerelles API acceptant des jetons d’accès
</div>

La plupart des intergiciels et des passerelles API utilisent le point de terminaison JSON Web Key Set (JWKS) pour récupérer, à intervalles réguliers, les clés de signature actuelles et futures. Si vos intergiciels et/ou passerelles API **ne** prennent **pas** en charge ce point de terminaison et exigent que vous configuriez manuellement un fichier `*.cer`, vous devrez coordonner la rotation des clés de signature dans Auth0 avec la reconfiguration de vos intergiciels et de vos passerelles.

<div id="regular-web-applications">
  ### Applications web traditionnelles
</div>

Lorsque vous faites la rotation de la clé de signature dans Auth0, vous devrez coordonner la reconfiguration de vos applications qui utilisent <Tooltip tip="Web Service Federation (WS-Fed) : protocole de gestion des identités utilisateur entre domaines." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=WS-Fed">WS-Fed</Tooltip> ou <Tooltip tip="Web Service Federation (WS-Fed) : protocole de gestion des identités utilisateur entre domaines." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SAML">SAML</Tooltip>. Cela se produit généralement lorsque vous téléversez le nouveau certificat public ou reconfigurez l’application en saisissant l’URL des métadonnées WS-Fed/SAML. Cela modifiera la clé JWKS, qui est utilisée par les applications pour valider les jetons; assurez-vous que votre implémentation ne suppose pas que les clés JWKS ne changent jamais.
