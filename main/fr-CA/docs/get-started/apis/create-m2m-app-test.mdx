title: Créer des applications machine à machine pour les tests
---

Lorsque vous créez un compte, une instance par défaut de la <Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> est créée dans la section API de l’<Tooltip tip="Management API : Un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Une application machine à machine de test est également créée automatiquement. Les instructions ci-dessous vous permettent de créer une autre application de test à utiliser avec cette instance de la Management API, ainsi que d’utiliser le jeton de test généré avant de mettre en place votre propre configuration de Production.

Veuillez noter que le jeton de test sous API Explorer sert uniquement à tester l’accès à la Management API. Pour en savoir plus sur les jetons de la Management API, consultez [Get Management API Access Tokens for Production](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production). Si vous souhaitez utiliser le jeton de test, consultez [Get Management API Access Tokens for Testing](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-testing)

1. Accédez à [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) et sélectionnez **Auth0 Management API**.
2. Sélectionnez l’onglet **API Explorer**.
3. Cliquez sur **Créer et autoriser une application de test**. Une nouvelle application est créée et autorisée à accéder à la Management API.

<Frame>![Onglet Dashboard Applications APIs Auth0 Management API Explorer Autoriser et tester](/docs/images/cdy7uua7fh8z/6G5M1JY9iYnkYXfDHwTvR/d09119af4f7d6d45c9b39001fd189d2f/dashboard-applications-apis-mgmt-api-authorize-and-test.png)</Frame>

L’application créée dans les étapes ci-dessus s’est vu accorder toutes les portées de la Management API. Cela signifie qu’elle peut accéder à tous les points de terminaison à des fins de test. Toutefois, en général, les applications ne sont pas autorisées à accéder à toutes les portées, mais seulement à celles qui sont requises.

<Card title="Comment savoir quelles portées/autorisations sont requises pour la Management API ?">
  Chaque application machine à machine qui accède à une API doit se voir accorder un ensemble de [portées](/fr-CA/docs/get-started/apis/scopes/api-scopes). Les portées sont des autorisations qui doivent être accordées par le propriétaire. Chaque point de terminaison de l’[Auth0 Management API v2](https://auth0.com/docs/api/management/v2) exige des portées précises. Pour voir les portées/autorisations requises pour chaque point de terminaison, accédez au [Management API Explorer](https://auth0.com/docs/api/management/v2#!) et repérez le point de terminaison que vous souhaitez appeler. Chaque point de terminaison comporte une section appelée **Scopes** qui répertorie toutes les portées acceptées par ce point de terminaison.
</Card>

<div id="example-get-all-connections-endpoint">
  ## Exemple : point de terminaison Get All Connections
</div>

Le point de terminaison [Get All Connections](https://auth0.com/docs/api/management/v2#!/Clients/get_connections) accepte la portée `read:connections`, tandis que le point de terminaison [Create a Connection](https://auth0.com/docs/api/management/v2#!/Clients/post_connections) accepte la portée `write:connections`. Notre jeton machine-to-machine ne devrait avoir besoin que de la portée `read:connections` pour accéder aux données de ce point de terminaison.

Si vous avez plusieurs applications qui doivent accéder à la Management API, vous devriez créer des applications machine-to-machine distinctes pour chaque application dans Auth0, plutôt qu’une seule application machine-to-machine.
