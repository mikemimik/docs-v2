Utilisez l’onglet **Settings** dans <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, à [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis), pour configurer les API enregistrées que vos applications autorisées peuvent consommer. Pour configurer les paramètres d’une API, cliquez sur **...** à côté de l’API dans la liste et sélectionnez **Settings**, ou cliquez sur le nom de l’API. Pour savoir comment créer et enregistrer une API, consultez [Register APIs](/docs/fr-CA/get-started/auth0-overview/set-up-apis).

<Frame>![Liste des API dans Dashboard Applications](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>

<div id="settings">
  ## Paramètres
</div>

Utilisez les paramètres de cet onglet pour configurer l’expiration du token, le contrôle d’accès basé sur les rôles (RBAC) et d’autres paramètres d’accès. Cliquez sur **Enregistrer** au bas de l’onglet pour enregistrer les modifications.

<div id="general-settings">
  ### Paramètres généraux
</div>

Ces champs ont été définis lorsque vous avez enregistré l’API initialement, sauf dans le cas de la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Management+API">Management API</Tooltip> d’Auth0. Vous ne pouvez modifier que le **Name**.

<Frame>![Dashboard Applications APIs Settings Tab General Settings](/docs/images/cdy7uua7fh8z/QCaXd3FSeP6SYqLfEJWBr/fbb1bb5b2e12fb099073c33fb39cf44a/dashboard-applications-apis-settings-general-settings.png)</Frame>

* **Id** : Une chaîne alphanumérique unique générée par Auth0. Cette information est en lecture seule et vous n’en aurez besoin que si vous travaillez directement avec les [endpoints API Resource Servers de la Management API d’Auth0](https://auth0.com/docs/api/management/v2#!/Resource_Servers/get_resource_servers_by_id).
* **Name** : Un nom convivial pour l’API. N’affecte aucune fonctionnalité. Les caractères suivants ne sont pas autorisés : `< >`.
* **Identifier** : Un identifiant unique pour votre API. Cette valeur est définie lors de la création de l’API et ne peut pas être modifiée par la suite. Nous vous recommandons d’utiliser une URL, mais celle-ci n’a pas besoin d’être accessible publiquement; Auth0 n’appellera jamais votre API.

<div id="token-settings">
  ### Paramètres des jetons
</div>

<Frame>![Dashboard - API - Paramètres des jetons - Expiration](/docs/images/cdy7uua7fh8z/2FFfqqmntXL9CnGrSxea9M/810e5542553168ffcd6081a10bc2db2f/image2.png)</Frame>

* **Durée de vie maximale du <Tooltip tip="Access Token : information d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Access+Token">jeton d’accès</Tooltip> (secondes)** : Durée (en secondes) avant l’expiration d’un jeton d’accès. La valeur par défaut est de 86400 secondes (24 heures). La valeur maximale que vous pouvez définir est de 2592000 secondes (30 jours).
* **Durée de vie du jeton d’accès pour le flux implicite / hybride (secondes)** : Durée (en secondes) avant l’expiration d’un jeton d’accès émis au moyen d’un flux implicite ou hybride. La valeur par défaut est de 86400 secondes (24 heures). Cette valeur ne peut pas dépasser la durée de vie maximale du jeton d’accès.
* **Profil du <Tooltip tip="JSON Web Token (JWT) : format standard de ID Token (et souvent de jeton d’accès) utilisé pour représenter des claims de manière sécurisée entre deux parties." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT)** : Le profil détermine le format des jetons d’accès émis pour l’API. Les valeurs disponibles sont `Auth0` et `RFC 9068`. Pour en savoir plus, consultez [Profils des jetons d’accès](/docs/fr-CA/secure/tokens/access-tokens/access-token-profiles).
* **JSON Web Token (JWT) <Tooltip tip="Signing Algorithm : algorithme utilisé pour signer numériquement les jetons afin de garantir que le jeton n’a pas été altéré." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Signing+Algorithm">Signing Algorithm</Tooltip>** : Algorithme utilisé pour signer les jetons. La signature sert à vérifier que l’expéditeur du JWT est bien celui qu’il prétend être et à s’assurer que le message n’a pas été modifié en cours de route. Les valeurs disponibles sont [`HS256`](https://en.wikipedia.org/wiki/Symmetric-key_algorithm) et [`RS256`](https://en.wikipedia.org/wiki/Public-key_cryptography). Si vous sélectionnez `RS256` (recommandé), le jeton sera signé avec la clé privée de votre tenant. Cette valeur est définie lors de la création de votre API et ne peut pas être modifiée par la suite. Pour en savoir plus sur les algorithmes de signature et leur fonctionnement dans Auth0, consultez [Algorithmes de signature](/docs/fr-CA/get-started/applications/signing-algorithms).

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    La signature fait partie d’un JWT. Si vous ne connaissez pas bien la structure d’un JWT, veuillez consulter [Structure d’un JSON Web Token](/docs/fr-CA/secure/tokens/json-web-tokens/json-web-token-structure).
  </Callout>
* **JSON Web Encryption (JWE) :** Lorsqu’elle est activée, les jetons d’accès émis sont chiffrés au format JSON Web Encryption (JWE). Pour en savoir plus, consultez [JSON Web Encryption](/docs/fr-CA/secure/tokens/access-tokens/json-web-encryption).

<div id="rbac-settings">
  ### Paramètres RBAC
</div>

<Frame>![Auth0 Dashboard API Settings bascule RBAC](/docs/images/cdy7uua7fh8z/65tKb6aj0ktc2qXLUVlV3e/641adef615d6af9e5a3b588ff397af87/dashboard-apis-edit_view-settings_rbac-settings.png)</Frame>

* **Activer RBAC** : Activez ce paramètre pour appliquer les politiques RBAC à l’API. Pour en savoir plus, consultez [Role-Based Access Control](/docs/fr-CA/manage-users/access-control/rbac) et [Enable Role-Based Access Control for APIs](/docs/fr-CA/get-started/apis/enable-role-based-access-control-for-apis).
* **Ajouter les autorisations au jeton d’accès** : Activez ce paramètre pour ajouter la revendication d’autorisations au jeton d’accès. Ce paramètre n’est disponible que si vous activez RBAC. Vous pouvez configurer les autorisations dans l’onglet **Autorisations**.

<div id="access-settings">
  ### Paramètres d’accès
</div>

<Frame>![Onglet Paramètres d’accès des API Applications du Dashboard](/docs/images/cdy7uua7fh8z/JNlWACqoyfmpE89JwWLKy/5a2b595f939167b0e6e1f72b485b08e7/2025-02-25_14-53-21.png)</Frame>

* **Autoriser à ne pas demander le consentement de l’utilisateur** : Activez ce paramètre pour l’API afin de ne pas demander le consentement de l’utilisateur pour les applications marquées comme « de première partie ».
* **Autoriser l’accès hors ligne** : Activez ce paramètre pour permettre aux applications de demander des <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> pour l’API.

<div id="default-policy-setting">
  #### Paramètre Default Policy
</div>

<Warning>
  Le niveau d’assurance de l’authentification pour la **My Account API** est actuellement en Early Access avec une politique à option unique. En utilisant cette fonctionnalité, vous acceptez les conditions applicables de l’essai gratuit énoncées dans le [Master Subscription Agreement](https://www.okta.com/legal) d’Okta. Pour en savoir plus sur le cycle de lancement des produits d’Auth0, consultez [Product Release Stages](/docs/fr-CA/troubleshoot/product-lifecycle/product-release-stages). Pour participer au programme, communiquez avec [Auth0 Support](https://support.auth0.com).
</Warning>

<Frame>![Paramètre API Default Policy](/docs/images/api-settings-access-default-policy.png)</Frame>

* **Exiger la 2FA** : Lorsqu’elle est activée, Auth0 applique la Default Policy à la My Account API pour déclencher [Step-Up Authentication](/docs/fr-CA/secure/multi-factor-authentication/step-up-authentication). Les utilisateurs doivent s’authentifier à l’aide d’un deuxième facteur dans un délai de 15 minutes. Pour connaître le comportement complet et savoir comment l’activer, consultez [My Account API: Default Policy](/docs/fr-CA/manage-users/my-account-api#default-policy).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La Default Policy n’est pas offerte avec Classic Login.
</Callout>

<div id="permissions">
  ## Autorisations
</div>

Utilisez les paramètres de l’onglet **Autorisations** pour définir les autorisations (scopes) utilisées par l’API. Pour en savoir plus, consultez [Ajouter des autorisations d’API](/docs/fr-CA/get-started/apis/add-api-permissions) et [Supprimer des autorisations d’API](/docs/fr-CA/get-started/apis/delete-api-permissions).

<div id="machine-to-machine-applications">
  ## Applications machine à machine
</div>

Si vous avez des applications machine à machine, elles s’afficheront dans une liste dans cet onglet. Utilisez les bascules pour autoriser les applications de la liste. Pour en savoir plus, consultez [Enregistrer des applications machine à machine](/docs/fr-CA/get-started/auth0-overview/create-applications/machine-to-machine-apps).

  ## Test
