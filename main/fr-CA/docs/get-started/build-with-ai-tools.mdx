Accélérez votre flux de travail de développement en connectant directement des agents d’IA et des LLM à la base de connaissances d’Auth0. Obtenez des réponses adaptées au contexte, des exemples de code et des références d’API, avec des liens vers la documentation officielle.

Pour aider vos outils d’IA à indexer et à rechercher l’information dont vous avez besoin, Auth0 fournit :

* Fichiers de contexte LLM (`llms.txt`) : fichiers Markdown structurés pour l’ingestion en masse.
* Auth0 Docs MCP Server : MCP Server hébergé permettant aux agents d’IA d’interroger dynamiquement la base de connaissances.

<div id="llm-context-files">
  ## Fichiers de contexte LLM
</div>

Utilisez les fichiers de contexte LLM pour aider les LLM à comprendre l’ensemble du répertoire du site Auth0 et à en ingérer efficacement le contenu.

| Nom             | URL                                    | Description                                                                      |
| --------------- | -------------------------------------- | -------------------------------------------------------------------------------- |
| `llms.txt`      | `https://auth0.com/llms.txt`           | Résumés concis de tous les articles ; idéal pour obtenir rapidement le contexte. |
| `llms-full.txt` | `https://auth0.com/docs/llms-full.txt` | Contenu complet de tous les articles ; idéal pour une indexation approfondie.    |

<div id="auth0-docs-mcp-server">
  ## Auth0 Docs MCP Server
</div>

Connectez vos outils d’IA au serveur Auth0 Docs Model Context Protocol (MCP) pour accéder en temps réel à la documentation Auth0 la plus récente. Le serveur Auth0 Docs MCP Server expose l’outil `SearchAuth0Docs`, qui permet aux agents d’IA de rechercher dans la base de connaissances d’Auth0 et de renvoyer du contenu contextuel avec des liens pertinents vers la documentation du produit.

Le serveur Auth0 Docs MCP Server comprend les métadonnées d’endpoint suivantes :

```
Server Name: Auth0 Docs
Server URL: https://auth0.com/docs/mcp
Transport Type: HTTP
```

Découvrez comment connecter vos outils d’IA à Auth0 Docs MCP Server en suivant nos guides d’intégration.

<Tabs>
  <Tab title="ChatGPT">
    Pour connecter ChatGPT à Auth0 Docs MCP Server :

    1. Accédez à [Paramètres &gt; Connecteurs](https://chatgpt.com/#settings/Connectors).
    2. Sélectionnez **Créer** et fournissez les métadonnées de votre serveur MCP :
       * Nom du connecteur : Auth0 Docs MCP Server
       * Description : Indexe la base de connaissances d’Auth0 et permet d’y faire des recherches
       * URL du connecteur : `https://auth0.com/docs/mcp`
    3. Sélectionnez **Créer**.

    Si la connexion réussit, Auth0 Docs MCP Server devrait apparaître comme serveur MCP installé. Pour en savoir plus, consultez la documentation [Connect from ChatGPT](https://developers.openai.com/apps-sdk/deploy/connect-chatgpt/) d’OpenAI.
  </Tab>

  <Tab title="Claude">
    Pour connecter Claude au serveur MCP Auth0 Docs :

    1. Accédez à [Paramètres &gt; Connecteurs](https://claude.ai/settings/connectors).
    2. Sélectionnez **Ajouter un connecteur personnalisé**.
    3. Saisissez les renseignements suivants pour ajouter le serveur MCP Auth0 Docs :
       * Nom : Auth0 Docs MCP Server
       * URL : `https://auth0.com/docs/mcp`
    4. Sélectionnez **Ajouter**.

    Si l’opération réussit, le serveur MCP Auth0 Docs devrait s’afficher comme connecteur personnalisé. Pour en savoir plus, consultez la documentation [Connect to remote MCP Servers](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server) du Model Context Protocol.
  </Tab>

  <Tab title="Claude Code">
    Pour connecter Claude Code à l’Auth0 Docs MCP Server :

    Dans votre terminal, exécutez :

    ```bash lines
    claude mcp add --transport http auth0-docs-mcp-server https://auth0.com/docs/mcp
    ```

    Pour vérifier, exécutez :

    ```bash lines
    claude mcp list
    ```

    Si l’installation réussit, le serveur Auth0 Docs MCP Server devrait s’afficher comme serveur MCP installé. Pour en savoir plus, consultez la documentation de Claude sur [l’installation des serveurs MCP](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Codex">
    Pour connecter Codex à Auth0 Docs MCP Server :

    Dans votre terminal, exécutez :

    ```bash lines
    codex mcp add auth0-docs-mcp-server --url https://auth0.com/docs/mcp
    ```

    Pour vérifier, exécutez :

    ```bash lines
    codex mcp list
    ```

    Si tout se passe bien, le serveur Auth0 Docs MCP Server devrait figurer parmi les serveurs MCP installés. Pour en savoir plus, consultez la documentation de Codex sur le [Model Context Protocol](https://developers.openai.com/codex/mcp).
  </Tab>

  <Tab title="Cursor">
    Pour connecter Cursor au Auth0 Docs MCP Server :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir la palette de commandes et rechercher **Open MCP settings**.
    2. Sélectionnez **New MCP Server** pour modifier votre `mcp.json`.
    3. Ajoutez le Auth0 Docs MCP Server :

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "url": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    Si tout se passe bien, le Auth0 Docs MCP Server devrait apparaître parmi les serveurs MCP installés. Pour en savoir plus, consultez la documentation de Cursor sur [Installing MCP servers](https://cursor.com/docs/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    Pour connecter VS Code à Auth0 Docs MCP Server :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir la palette de commandes, puis recherchez **MCP: Add Server**.
    2. Sélectionnez **HTTP** comme type de serveur MCP.
    3. Entrez l’URL du serveur MCP (`https://auth0.com/docs/mcp`) et définissez l’ID du serveur sur `auth0-docs-mcp-server`.
    4. Sélectionnez **Global** ou **Workspace** pour rendre le serveur MCP accessible dans tous vos espaces de travail ou seulement dans votre espace de travail actuel.
    5. Appuyez sur **Enter** pour terminer la configuration. Vérifiez qu’Auth0 Docs MCP Server apparaît sous **Installed MCP Servers** dans **Extensions**.

    Pour en savoir plus, consultez [Add and manage MCP servers in VS Code](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>

  <Tab title="Windsurf">
    Pour connecter Windsurf à l’Auth0 Docs MCP Server :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir les paramètres de Windsurf et rechercher **MCP**.
    2. Sélectionnez **View Raw Config** pour modifier votre `mcp_config.json`.
    3. Ajoutez l’Auth0 Docs MCP Server :

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "serverUrl": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    4. Save et sélectionnez **Refresh** pour charger la nouvelle configuration du MCP.

    Pour vérifier, saisissez `What MCP servers do you have installed?` dans le clavardage de Cascade. Cascade devrait afficher Auth0 Docs MCP Server comme serveur MCP installé. Pour en savoir plus, consultez la documentation de Windsurf sur le [Model Context Protocol](https://docs.windsurf.com/plugins/cascade/mcp#model-context-protocol-mcp).
  </Tab>
</Tabs>

Mets à jour la configuration actuelle de mon serveur afin d’exiger OAuth 2.1 pour toutes les connexions entrantes et vérifie qu’elle utilise le profil de compatibilité du paramètre de ressource pour les jetons d’accès.

Si tu trouves des écarts susceptibles d’entraîner la divulgation du contexte utilisateur, suggère une correction et fournis le code mis à jour.

<div id="troubleshooting">
  ### Dépannage
</div>

* Aidez-moi à résoudre l’erreur `refresh token doesn’t exist` lors de l’utilisation du refresh token exchange. Utilisez les instructions de configuration de Token Vault comme référence.
* Le bouton de déconnexion ne met pas fin à la session du côté d’Auth0. Recherchez dans la documentation la structure correcte de l’URL de Federated Logout et mettez à jour mon gestionnaire de déconnexion pour l’utiliser.
* Mon processus de connexion échoue avec une `401 Unauthorized error`. Utilisez le Auth0 Docs MCP Server pour rechercher les causes courantes de cette erreur avec le SDK `auth0-node` et vérifiez s’il y a des incohérences entre mon `.env` et l’initialisation du client.
