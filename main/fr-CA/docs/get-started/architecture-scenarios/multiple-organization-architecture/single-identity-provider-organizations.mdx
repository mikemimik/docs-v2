---

Dans le scénario des Organizations avec un <Tooltip tip="Fournisseur d’identité (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) unique, chaque organisation qui utilise la fonctionnalité [Auth0 Organizations](/fr-CA/docs/manage-users/organizations/organizations-overview) utilise exactement une connexion Auth0. Cette fonctionnalité permet à chaque application de sélectionner l’organisation appropriée définie dans le tenant Auth0, qui authentifiera directement les informations d’identification de l’utilisateur ou transmettra la requête d’authentification à l’IdP approprié pour qu’il la prenne en charge.

Dans ce scénario, nous continuerons d’utiliser l’exemple [Travel0 Corporate Booking](/fr-CA/docs/get-started/architecture-scenarios/multiple-organization-architecture). Dans cet exemple, l’application de réservation d’entreprise correspond à votre application, et Travel0 à votre entreprise.

<Frame>![Scénarios d’architecture - Multitenancy - Diagramme - Utilisateurs isolés, applications partagées, orgs A et B](/docs/images/cdy7uua7fh8z/1YBaMaMozamiTv8hXmNB2D/21e9270a59021a12209bdcb4345147cb/isolated-users_shared-apps_org-ab.png)</Frame>

<Warning>
  La fonctionnalité Auth0 Organizations prend actuellement en charge les applications qui utilisent les protocoles [SAML](/fr-CA/docs/authenticate/protocols/saml/saml-configuration), [Open ID Connect](/fr-CA/docs/authenticate/protocols/openid-connect-protocol) et/ou le framework [OAuth 2](/fr-CA/docs/authenticate/protocols/oauth). Les applications qui nécessitent [WS-Fed](/fr-CA/docs/authenticate/protocols/ws-fed-protocol) ne sont pas prises en charge.
</Warning>

Peu importe la façon dont les informations d’identification sont traitées -- qu’elles soient stockées dans une connexion de base de données (comme pour Hoekstra &amp; Associates), validées au moyen d’une connexion d’entreprise (comme pour MetaHexa Bank) ou d’une connexion sociale -- votre application se comporte de la même manière. La fonctionnalité [Organizations](/fr-CA/docs/manage-users/organizations/organizations-overview) vous permet de configurer et de contrôler l’expérience.
