---

Les éléments de marque associés à une organisation sont extrêmement précieux, car ils offrent aux utilisateurs un environnement qu’ils connaissent et en lequel ils ont confiance. L’utilisation d’éléments de marque reconnus accroît aussi la confiance des utilisateurs en leur donnant l’assurance que les renseignements qu’ils fournissent (par exemple, leurs identifiants) seront traités de façon sécuritaire. Dans nos scénarios d’architecture, nous fournissons des conseils généraux sur plusieurs sujets liés à [B2B Branding](/fr-CA/docs/get-started/architecture-scenarios/business-to-business/branding), que nous vous recommandons de consulter en complément des conseils présentés ici.

<div id="universal-login">
  ## Universal Login
</div>

L’image de marque d’un environnement multi-organisation peut être plus complexe que celle d’un environnement standard, où vous n’avez qu’un seul ensemble d’éléments de marque à gérer. [Auth0 Organizations](/fr-CA/docs/manage-users/organizations/organizations-overview), toutefois, tire parti de l’[expérience Universal Login](/fr-CA/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), qui propose plusieurs [gabarits de page](/fr-CA/docs/customize/login-pages/universal-login/customize-templates) pour simplifier la personnalisation de l’image de marque. La configuration d’une Auth0 Organization permet également de [sélectionner une palette de couleurs de base pour l’image de marque](/fr-CA/docs/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Page de connexion
</div>

Lorsqu’une [connexion de base de données ou de base de données personnalisée](/fr-CA/docs/authenticate/database-connections) est utilisée, la page de connexion sert à recueillir les identifiants du premier facteur de l’utilisateur. Vous pouvez configurer la page de connexion à l’aide d’un ensemble de [variables de modèle](/fr-CA/docs/manage-users/organizations/create-first-organization) offertes pour faciliter la personnalisation propre à chaque organisation.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pouvez configurer la page de connexion pour afficher des connexions Enterprise et des connexions sociales pour l’authentification du premier facteur, de sorte que l’image de marque de la page de connexion soit pertinente dans ces cas.
</Callout>

<div id="password-reset-page">
  ### Page de réinitialisation du mot de passe
</div>

À l’instar de la [page de connexion](#login-page), la page de réinitialisation du mot de passe peut être configurée dans les scénarios qui utilisent une connexion de base de données ou de base de données personnalisée. Dans les scénarios faisant appel à une connexion d’entreprise ou à une connexion sociale, les changements de mot de passe sont gérés à l’extérieur d’Auth0 par le <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) en amont; une page de réinitialisation du mot de passe d’Auth0 n’est donc jamais affichée.

<div id="multi-factor-authentication-pages">
  ### Pages d’authentification multifacteur
</div>

<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip> (MFA) dans Auth0 est prise en charge pour toutes les méthodes utilisées pour obtenir les identifiants du premier facteur d’un utilisateur (c.-à-d. une connexion de base de données ou une connexion de base de données personnalisée, une connexion d’entreprise ou une connexion sociale). Si vous utilisez la MFA, vous voudrez probablement personnaliser l’image de marque des pages de l’expérience <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> associées à la MFA, ce qui peut se faire de façon semblable à la personnalisation de la [page de connexion](#login-page).

<div id="email-templates">
  ## Modèles de courriel
</div>

Comme pour les modèles de page Universal Login, un ensemble de [variables de modèle de courriel](/fr-CA/docs/customize/email/email-templates/supported-liquid-syntax#general-variables) est disponible pour faciliter la personnalisation des courriels propres à chaque organisation. Les modèles de courriel à personnaliser varient selon la façon dont les utilisateurs sont authentifiés (autrement dit, au moyen d’une connexion de base de données ou de base de données personnalisée, d’une connexion d’entreprise ou d’une connexion sociale). Pour en savoir plus sur la personnalisation des modèles de courriel lors de l’utilisation de la fonctionnalité Auth0 Organizations, consultez [Créez votre première organisation : modèles de courriel](/fr-CA/docs/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Domaines personnalisés
</div>

La fonctionnalité Auth0 Organizations ne prend en charge qu’un seul [domaine personnalisé](/fr-CA/docs/customize/custom-domains), défini au niveau du locataire Auth0. Nous vous recommandons donc d’utiliser un nom de <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou de fantaisie." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Custom+Domain">domaine personnalisé</Tooltip> qui n’est pas propre à une organisation. Si vous devez absolument mettre en place une nomenclature de domaine par organisation, il faudra alors déployer plusieurs locataires Auth0 en production. Comme l’utilisation de plusieurs locataires a des répercussions sur l’<Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO), la gestion du profil utilisateur, etc., vous devriez bien peser cette option avant de vous engager dans cette voie.
