Auth0 offre aux utilisateurs du tenant une expérience de recherche plus intuitive pour les Applications et les API enregistrées, avec la possibilité de créer des expressions de filtre pour les champs interrogeables dans Auth0 Dashboard. Au lieu de parcourir des listes paginées contenant des centaines de ressources, les utilisateurs de votre tenant peuvent effectuer des requêtes sur des champs comme :

* Le nom de votre application (Travel0-native)
* Les métadonnées de l’application ou de l’utilisateur (autorisations)
* Le type d’application (SPA)
* Une API par nom
* Une API par ID

Auth0 Dashboard affiche en temps réel les résultats correspondants parmi toutes vos ressources enregistrées.

<div id="how-it-works">
  ## Fonctionnement
</div>

La barre de recherche des sections [Applications](https://manage.auth0.com/#/applications) et [APIs](https://manage.auth0.com/#/apis) du Auth0 Dashboard utilise une interface de filtre guidé et la recherche booléenne. Elle fonctionne comme un menu déroulant qui affiche les champs pouvant être filtrés. Lorsque vous sélectionnez un champ et saisissez une valeur, celle-ci s’applique sous forme d’étiquette amovible au-dessus des résultats. Vous pouvez combiner jusqu’à cinq filtres à la fois.

Chaque filtre ajouté affine davantage les résultats. Pour préciser les résultats, appliquez la logique `AND/OR` entre différents champs. Les filtres se reflètent dans l’URL sous forme de paramètres de requête. Copiez l’URL à tout moment pour la partager avec un coéquipier ou ajoutez-la à vos signets pour revenir plus tard à la même vue filtrée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Exemple booléen**

  Si vous appliquez le filtre **Application type** `Native` OR `SPA`, ainsi que **First-party client** `true`, les résultats n’affichent que les applications first-party de type Native ou SPA. Les applications de type Regular Web Apps ou les applications third-party sont exclues.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportement de la recherche selon le type de champ
</div>

La recherche dans Auth0 Dashboard trouve des résultats en fonction du type de champ.

* **Champs String** (nom, identificateur) : la correspondance partielle est prise en charge. La saisie de `payments` permet de trouver « Payments-application », « Corp Payments Service » et d’autres résultats semblables — l’ordre des mots n’a pas d’importance. Pour exclure les résultats correspondant à un terme, faites-le précéder de `NOT`, `-` ou `No` — par exemple, la saisie de `NOT test` dans le champ du nom renvoie toutes les ressources dont le nom ne contient pas « test ». Les recherches avec caractères génériques exigent au moins trois caractères ; si vous en saisissez moins, le message « Minimum 3 characters » s’affiche et la requête n’est pas envoyée.

* **Champs de correspondance exacte**

Vous devez saisir la valeur complète de l’ID. Les ID partiels ne renvoient aucun résultat. Utilisez ces champs lorsque vous avez un Client ID précis provenant d’un log ou d’un ticket de soutien.

* **Champs Metadata**

Vous pouvez rechercher dans les métadonnées par clé et valeur, ou par clé seulement :

* **Key and value** : saisissez une clé et une valeur pour trouver les ressources où cette clé exacte correspond à cette valeur. Par exemple, la clé `environment` et la valeur `production` renvoient uniquement les ressources où `app_metadata.environment` est égal à `production`. La correspondance par préfixe est prise en charge pour les valeurs — la saisie de `prod` correspond à `production`, `prod-eu` et `prod-us`.
  * **Key only** : saisissez une clé et appuyez sur **Enter** sans valeur. Cela renvoie toutes les ressources pour lesquelles cette clé de métadonnées est définie, peu importe sa valeur. C’est utile pour trouver toutes les ressources qui utilisent une clé précise.

Vous pouvez aussi rechercher dans des clés de métadonnées imbriquées jusqu’à deux niveaux de profondeur au moyen de la notation par points. Par exemple, la clé `tier.bronze` et la valeur `true` correspondent aux ressources où `app_metadata.tier.bronze` est égal à `true`.

Pour exclure les résultats qui correspondent à une paire clé-valeur précise, faites précéder la valeur de `NOT`. Par exemple, la clé `color` et la valeur `NOT orange` renvoient toutes les ressources où `color` est défini à autre chose que `orange`. Les clés doivent correspondre exactement. La correspondance de type « contient » ou « se termine par » n’est pas prise en charge pour les métadonnées. Les valeurs sont indexées jusqu’à 64 caractères ; les valeurs plus longues ne peuvent pas faire l’objet d’une recherche.

* **Champs Enum**

Sélectionnez un ou plusieurs types d’Application dans une liste prédéfinie. Plusieurs sélections dans le même filtre utilisent une logique OR. Par exemple, sélectionner `Native` et `SPA` renvoie toutes les applications Native et SPA.

* **Champs Boolean**

Recherchez les clients first-party en sélectionnant `true` ou `false`. Utilisez cette option pour distinguer les applications first-party (celles autorisées à ignorer l’écran de consentement) des applications third-party.

<div id="searchable-fields">
  ### Champs de recherche
</div>

<div id="applications">
  #### Applications
</div>

| **Champ**                                    | **Description**                                                                            | **Type de correspondance**           |
| -------------------------------------------- | ------------------------------------------------------------------------------------------ | ------------------------------------ |
| **Nom**                                      | Le nom de l’application                                                                    | Correspondance partielle             |
| **Client ID** (`client_id`)                  | Le client ID unique                                                                        | Correspondance exacte                |
| **ID client externe** (`external_client_id`) | Un identifiant externe de l’application                                                    | Correspondance exacte                |
| **Metadata**                                 | Des métadonnées personnalisées sous forme de paires clé-valeur stockées dans l’application | Correspondance exacte ou par préfixe |
| **Application type**                         | Native, SPA, application Web traditionnelle ou Machine to Machine                          | Sélection dans une énumération       |
| **First-party client**                       | Indique si l’application est first-party ou third-party                                    | Booléen                              |

<div id="apis">
  #### API
</div>

| **Champ**       | **Description**                                                          | **Type de correspondance** |
| --------------- | ------------------------------------------------------------------------ | -------------------------- |
| **Nom**         | Le nom de l’API                                                          | Correspondance partielle   |
| **Identifiant** | L’identifiant d’audience unique (par exemple, `https://api.example.com`) | Correspondance partielle   |
| **Id**          | L’identifiant interne du système                                         | Correspondance exacte      |

<div id="limitations">
  ### Limites
</div>

* Vous pouvez appliquer un maximum de cinq filtres par recherche. La barre de recherche empêche l’ajout d’un sixième filtre.
* Les résultats de recherche peuvent ne pas refléter les modifications apportées au cours des dernières secondes, car l’index est à cohérence éventuelle.
* Les résultats s’affichent en moins de deux secondes dans des conditions normales. Si une requête expire, réduisez le nombre de filtres ou utilisez un terme de recherche plus précis.

<div id="example-queries">
  ## Exemples de requêtes
</div>

<div id="find-an-application-by-client-id">
  ### Trouver une application par Client ID
</div>

Utilisez cette procédure lorsque vous avez un `client_id` provenant d’un journal d’erreurs ou d’un ticket de soutien et que vous devez retrouver rapidement l’application.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **Client ID**.
3. Collez la valeur complète de `client_id`. L’application correspondante s’affiche immédiatement.

<div id="find-applications-using-metadata">
  ### Trouver des applications à l’aide des métadonnées
</div>

Si vos applications stockent des métadonnées personnalisées — comme des étiquettes d’environnement, des niveaux de forfait ou des identifiants externes — vous pouvez les rechercher à l’aide de ces valeurs.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **Metadata**.
3. Saisissez la clé de métadonnées, par exemple `environment`.
4. Saisissez la valeur, par exemple `production`. Les applications dont la valeur `metadata.environment` est `production` s’affichent dans les résultats. Pour effectuer une recherche par préfixe, saisissez une valeur partielle comme `prod`.

<div id="troubleshooting">
  ## Dépannage
</div>

**Aucun résultat trouvé**

Vérifiez que les valeurs de filtre sont orthographiées correctement. Pour les champs de type String, essayez un terme plus court ou plus général. Pour les champs de métadonnées, confirmez le nom exact de la clé — les clés sont sensibles à la casse et doivent correspondre exactement. La correspondance de type « contains » n’est pas prise en charge pour les métadonnées; utilisez plutôt un préfixe. Pour les champs à correspondance exacte, comme l’ID d’application, assurez-vous de coller la valeur complète de l’ID.

**Les résultats semblent obsolètes**

Les résultats de recherche sont à cohérence éventuelle et peuvent ne pas refléter les changements apportés au cours des dernières secondes. Si vous avez récemment créé ou mis à jour une ressource, attendez quelques secondes, puis relancez la recherche.

**La recherche a expiré**

Réduisez le nombre de filtres actifs ou utilisez un terme de recherche plus précis. Si le problème persiste, communiquez avec [Auth0 Support](https://support.auth0.com).
