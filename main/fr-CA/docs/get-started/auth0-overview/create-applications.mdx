Vous pouvez créer une application manuellement ou l’importer depuis l’URL d’un [Client ID Metadata Document (CIMD)](/fr-CA/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd) dans l’Auth0 Dashboard.

<div id="create-manually">
  ## Créer manuellement
</div>

Lorsque vous créez manuellement une application, Auth0 lui attribue un <Tooltip tip="ID client : Chaîne alphanumérique servant d'identifiant unique à une application." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+ID">ID client</Tooltip>, soit une chaîne alphanumérique qui sert d&#39;identifiant unique à l&#39;application. Vous utiliserez cet ID dans le code de votre application lorsque vous appellerez les API d’Auth0. Ce processus d&#39;enregistrement s&#39;appelle l&#39;enregistrement manuel du client.

Un autre élément important est le <Tooltip tip="Secret du client : Secret utilisé par un client (application) pour s'authentifier auprès du serveur d'autorisation; il ne doit être connu que du client et du serveur d'autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+secret">Secret du client</Tooltip>. Il doit rester confidentiel en tout temps. Si quelqu&#39;un obtient l&#39;accès au secret du client de votre application, cette personne pourrait usurper l&#39;identité de votre application et accéder à des ressources protégées. Il est important de sélectionner le bon type d&#39;application pour aider Auth0 à détecter certains risques de sécurité.

<div id="import-from-url">
  ## Importer à partir d’une URL
</div>

Vous pouvez enregistrer des applications en important, à partir d’une URL, un Client ID Metadata Document (CIMD) hébergé à l’externe. Un CIMD est un fichier JSON qui contient des métadonnées client et qui est hébergé sur un domaine HTTPS sécurisé contrôlé par l’application. Ce processus d’enregistrement d’application s’appelle l’enregistrement manuel CIMD. Pour en savoir plus, consultez [Register Applications with CIMD](/fr-CA/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd).

<div id="manual-client-registration-vs-manual-cimd-registration">
  ## Enregistrement manuel du client vs enregistrement manuel CIMD
</div>

Le tableau suivant compare l’enregistrement manuel du client et l’enregistrement manuel CIMD :

| Fonctionnalité                 | Enregistrement manuel du client                    | Enregistrement manuel CIMD                               |
| ------------------------------ | -------------------------------------------------- | -------------------------------------------------------- |
| **Client ID**                  | Généré par Auth0                                   | URL HTTPS (p. ex., `https://yourdomain.com/client.json`) |
| **Authentification**           | Client secret ou certificat                        | Clés asymétriques seulement (aucun secret)               |
| **Vérification de l’identité** | Aucune                                             | Propriété du domaine par hébergement HTTPS               |
| **Gestion des métadonnées**    | Mise à jour dans le Dashboard ou par l’API         | Le CIMD fait foi                                         |
| **Rotation des clés**          | Manuelle, une seule clé                            | Plusieurs clés prises en charge dans `jwks_uri`          |
| **Type d’application**         | Tous les types                                     | Applications tierces seulement                           |
| **Idéal pour**                 | Applications de première partie, services internes | Clients MCP, intégrations tierces                        |

<div id="create-application-in-auth0">
  ## Créer une application dans Auth0
</div>

Pour créer une application dans Auth0 :

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Sélectionnez **Create Application**.
3. Sélectionnez **Create it manually** ou **Import from URL**.
   * **Import from URL** : Suivez les instructions de [Register Applications with CIMD](/fr-CA/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd).
   * **Create it manually** : Donnez un nom descriptif à votre nouvelle application. Ensuite, sélectionnez un type d’application :
     * **Native Applications** : Ces applications comprennent des applis mobiles, de bureau ou hybrides qui s’exécutent nativement sur un appareil (p. ex., iOS, Android).
     * **Single-Page Web Applications** : Ces applications comprennent des applis Javascript qui exécutent la majeure partie de la logique de leur interface utilisateur dans un navigateur web et communiquent principalement avec un serveur web au moyen d’API (p. ex., AngularJS + Node.js ou React).
     * **Regular Web Applications** : Ces applications sont des applications web traditionnelles qui exécutent la majeure partie de leur logique sur le serveur (p. ex., Express.js, ASP.NET).
     * **Machine-to-Machine Applications** : Ces applications comprennent des applications non interactives, comme des outils en ligne de commande, des démons, des appareils IoT ou des services exécutés sur votre serveur principal.
4. Sélectionnez **Create**.
