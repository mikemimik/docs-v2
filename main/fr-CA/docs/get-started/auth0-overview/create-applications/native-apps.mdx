Pour intégrer Auth0 à une application native, vous devez d’abord enregistrer votre application dans Auth0 à l’aide de l’<Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Les applications natives doivent utiliser le flux de code d’autorisation avec PKCE pour assurer une authentification sécurisée. Pour en savoir plus, consultez [Authorization Code Flow with PKCE](/fr-CA/docs/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce). Ces applications peuvent inclure des applis mobiles, de bureau ou hybrides qui s’exécutent nativement sur un appareil (par exemple, iOS ou Android).

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Entrez un nom explicite pour votre application, sélectionnez **Native**, puis cliquez sur **Create**.

   Une fois l’application créée, Auth0 vous redirige vers **Application Settings**, qui comprend les onglets suivants :

   | Onglet des paramètres | Description                                                                                                                                                                                                                                                                                                                                                                                                                                     |
   | --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**       | Affiche toute la documentation offerte pour votre type d’application.                                                                                                                                                                                                                                                                                                                                                                           |
   | **Settings**          | Affiche tous les paramètres offerts pour votre application. Par défaut, la plupart des paramètres sont créés pour vous. Pour en savoir plus, consultez [Application Settings](/fr-CA/docs/get-started/applications/application-settings).                                                                                                                                                                                                             |
   | **Credentials**       | Affiche la méthode d’authentification de l’application et les informations d’identification configurées. Pour en savoir plus, consultez [Application Credentials](/fr-CA/docs/secure/application-credentials)                                                                                                                                                                                                                                         |
   | **Add-ons**           | Permet d’activer les plugiciels associés à une application. Les add-ons servent principalement aux intégrations SAML ou WS-Fed et ne s’appliquent généralement pas aux applications natives. Pour en savoir plus, consultez [Enable SAML2 Web App Addon](/fr-CA/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) et [Configure WS-Fed Applications](/fr-CA/docs/get-started/applications/update-application-connections). |
   | **Connections**       | Permet d’activer des connexions pour votre application. Les connexions sont des sources d’utilisateurs; elles peuvent être activées et partagées entre plusieurs applications. Pour en savoir plus, consultez [Connections](/fr-CA/docs/authenticate/identity-providers).                                                                                                                                                                             |
   | **Organizations**     | Permet d’authentifier les utilisateurs dans les organizations auxquelles ils appartiennent. Pour en savoir plus, consultez [Organizations](/fr-CA/docs/manage-users/organizations).                                                                                                                                                                                                                                                                   |
   | **Login Experience**  | Permet de configurer l’expérience de connexion des utilisateurs dans votre Organization. Pour en savoir plus, consultez [Understand how Auth0 Organizations Work](/fr-CA/docs/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                                                                    |

   Pour en savoir plus, consultez [Application Settings](/fr-CA/docs/get-started/applications/application-settings).
3. Si vous développez une appli mobile, fournissez les paramètres iOS/Android requis dans la section **Advanced Settings**, puis cliquez sur **Save Changes**.

   * Pour les applis iOS, [indiquez votre **Team ID** et votre **App Bundle Identifier**](/fr-CA/docs/get-started/applications/enable-universal-links-support-in-apple-xcode).
   * Pour les applis Android, [indiquez votre **App Package Name** et vos **Key Hashes**](/fr-CA/docs/get-started/applications/enable-android-app-links-support).

<div id="next-steps">
  ## Prochaines étapes
</div>

Une fois que vous avez enregistré et configuré votre application, voici quelques prochaines étapes courantes :

* Configurez une connection et activez-la pour votre application.
* Modifiez le code de votre application pour utiliser votre application enregistrée dans Auth0. Consultez nos [Auth0 Quickstarts](/fr-CA/docs/quickstarts), où vous trouverez des instructions détaillées et des exemples pour diverses technologies. Vous y apprendrez aussi à implémenter le login et le logout, à gérer vos sessions utilisateur, à récupérer et à afficher les renseignements du profil utilisateur, et plus encore.
* Utilisez les [API Auth0](/fr-CA/docs/api).

  * L’[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les fonctions principales liées à l’identité (par exemple, le login, le logout et la récupération du profil utilisateur). La plupart des utilisateurs utilisent cette API au moyen de nos Quickstarts, de la [bibliothèque auth0.js](/fr-CA/docs/libraries/auth0js) ou du [widget Lock](/fr-CA/docs/libraries/lock). Toutefois, si vous créez manuellement toute votre UI d’authentication, vous devrez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d’automatiser diverses tâches auxquelles vous pouvez aussi accéder dans le Dashboard d’Auth0 (par exemple : créer des utilisateurs, définir les types d’autorisation de l’application).
