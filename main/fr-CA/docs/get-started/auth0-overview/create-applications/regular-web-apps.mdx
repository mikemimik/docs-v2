Pour intégrer Auth0 à une application web traditionnelle, vous devez d’abord enregistrer votre application dans Auth0 à l’aide de l’<Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Les applications web traditionnelles sont des applications web classiques qui exécutent l’essentiel de leur logique côté serveur (par exemple, Express.js, ASP.NET). Pour en savoir plus sur les types d’application, consultez [Application Types](/fr-CA/docs/get-started/applications).

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Entrez un nom descriptif pour votre application, sélectionnez **application web traditionnelle**, puis cliquez sur **Create**.

   Une fois la nouvelle application créée, vous pouvez consulter la page **Application Settings**, qui comprend les onglets suivants :

   | Settings Tab         | Description                                                                                                                                                                                                                                                                                                                                                                                                                   |
   | -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**      | Affiche toute la documentation disponible pour votre type d’application.                                                                                                                                                                                                                                                                                                                                                      |
   | **Settings**         | Affiche tous les paramètres disponibles pour votre application. Par défaut, la plupart des paramètres sont créés pour vous. Pour en savoir plus, consultez [Application Settings](/fr-CA/docs/get-started/applications/application-settings).                                                                                                                                                                                       |
   | **Credentials**      | Affiche la méthode d’authentification de l’application et les informations d’identification configurées. Pour en savoir plus, consultez [Application Credentials](/fr-CA/docs/secure/application-credentials)                                                                                                                                                                                                                       |
   | **Add-ons**          | Vous permet d’activer les modules complémentaires associés à une application. Il s’agit d’applications web SAML ou WS-Fed pour lesquelles Auth0 génère des jetons d’accès. Pour en savoir plus, consultez [Enable SAML2 Web App Addon](/fr-CA/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) et [Configure WS-Fed Applications](/fr-CA/docs/get-started/applications/update-application-connections). |
   | **Connections**      | Vous permet d’activer des connexions pour votre application. Les connexions sont des sources d’utilisateurs; elles peuvent être activées et partagées entre plusieurs applications. Pour en savoir plus, consultez [Connections](/fr-CA/docs/authenticate/identity-providers).                                                                                                                                                      |
   | **Organizations**    | Vous permet d’authentifier des utilisateurs dans les organisations auxquelles ils appartiennent. Pour en savoir plus, consultez [Organizations](/fr-CA/docs/manage-users/organizations).                                                                                                                                                                                                                                            |
   | **Login Experience** | Vous permet de configurer l’expérience de connexion des utilisateurs de votre Organization. Pour en savoir plus, consultez [Understand how Auth0 Organizations Work](/fr-CA/docs/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                                               |

   Pour en savoir plus, consultez [Application Settings](/fr-CA/docs/get-started/applications/application-settings).
3. Faites défiler la page jusqu’au paramètre **Trust Token Endpoint IP Header**. Si l’architecture de votre application le permet, activez ce paramètre et cliquez sur **Save Changes**. Ce paramètre aide à vous protéger contre les attaques par force brute en permettant à Auth0 de se fier aux en-têtes IP de votre proxy.

<div id="next-steps">
  ## Prochaines étapes
</div>

Une fois votre application enregistrée et configurée, suivez les étapes suivantes :

1. Configurez une connexion et activez-la pour votre application.
2. Implémentez l’authentification dans le code de votre application à l’aide de l’application enregistrée dans Auth0. Consultez nos [Auth0 Quickstarts](/fr-CA/docs/quickstarts), où vous trouverez des instructions détaillées et des exemples pour diverses technologies. Vous y apprendrez aussi à implémenter la connexion et la déconnexion, à gérer les sessions utilisateur, à récupérer et à afficher les informations du profil utilisateur, et plus encore.

* Utilisez les [API Auth0](/fr-CA/docs/api).

  * L’[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les principales fonctions liées à l’identité (par exemple, la connexion, la déconnexion et la récupération du profil utilisateur). La plupart des utilisateurs utilisent cette API par l’intermédiaire de nos Quickstarts, de la [bibliothèque Auth0.js](/fr-CA/docs/libraries/auth0js) ou du [widget Lock](/fr-CA/docs/libraries/lock). Toutefois, si vous créez manuellement toute votre interface d’authentification, vous pouvez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d’automatiser diverses tâches auxquelles vous pouvez aussi accéder dans l’Auth0 Dashboard (par exemple, créer des utilisateurs et configurer les types d’autorisation de l’application).
