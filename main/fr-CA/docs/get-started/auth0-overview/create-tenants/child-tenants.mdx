Auth0 permet aux clients disposant d’un abonnement Enterprise de lier plusieurs tenants à un même abonnement Auth0 (ces tenants liés peuvent aussi être appelés tenants enfants).

Cette fonctionnalité peut être utile dans les situations suivantes :

* Séparer les tenants de développement et de production tout en conservant, dans les tenants de développement, le même accès aux fonctionnalités que dans les tenants de production
* Avoir plus d’un tenant de production sous un même abonnement Enterprise

<div id="linking-tenants">
  ## Lier des tenants
</div>

Un abonnement Enterprise permet automatiquement de lier des tenants à un abonnement Auth0 existant en sélectionnant l’option **Custom Agreement** dans le menu déroulant **Create Under** lors de la création de nouveaux tenants.

<Frame>![](/docs/images/cdy7uua7fh8z/7ggqdv5yw011z0TsdlaHOO/7707d826dd585e074bd0d8e25b30d9d6/publiccloudtenants.png)</Frame>

Si vous devez lier des tenants déjà créés qui ne font pas actuellement partie de votre abonnement Enterprise, communiquez avec votre équipe de compte ou ouvrez un ticket auprès d’[Auth0 Support](https://support.auth0.com).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous avez un abonnement en libre-service (non Enterprise), vous pouvez quand même lier des tenants. Pour en savoir plus, consultez [la gestion des tenants](/docs/fr-CA/get-started/auth0-teams/tenant-management).
</Callout>

<div id="usage-consolidation">
  ## Consolidation de l’utilisation
</div>

L’utilisation de tous les tenants liés est comptabilisée dans les limites de votre abonnement Enterprise. Vous pouvez consulter l’utilisation globale de tous les tenants liés dans vos rapports d’utilisation et de quotas.
