description: Décrit comment créer une intégration d’authentification unique (SSO) pour une application. À utiliser uniquement avec les tenants Legacy.
title: Activer les intégrations d’authentification unique pour les applications
---

Par défaut, l’<Tooltip tip="Single Sign-On (SSO) : service qui, après qu'un utilisateur a ouvert une session dans une application, connecte automatiquement cet utilisateur à d'autres applications." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=single+sign-on">authentification unique</Tooltip> (SSO) transparente est activée pour tous les nouveaux tenants Auth0; toutefois, les **tenants Legacy** peuvent [choisir d’activer ou non cette fonctionnalité au niveau du tenant](/fr-CA/docs/get-started/tenant-settings/enable-sso-for-legacy-tenants). Si vous n’avez pas activé le SSO au niveau du tenant, vous pouvez l’activer pour chaque application.

Vous pouvez activer le SSO pour votre application à l’aide du <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

<div id="prerequisite">
  ## Prérequis
</div>

Avant d’activer le SSO pour une application, vous devez d’abord créer et configurer une connexion pour chaque <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> que vous souhaitez utiliser. Pour les fournisseurs d’identité sociaux, assurez-vous que la connexion n’utilise pas de clés de développeur si vous utilisez l’[expérience Universal Login](/fr-CA/docs/authenticate/login/auth0-universal-login).

<div id="enable-sso">
  ## Activer l’authentification unique
</div>

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; SSO Integrations](https://manage.auth0.com/#/externalapps), puis sélectionnez le nom de l’intégration à consulter.
2. Faites défiler la page **Settings** jusqu’en bas, repérez **Use Auth0 instead of the IdP to do Single Sign-on**, activez la bascule, puis sélectionnez **Save Changes**.
