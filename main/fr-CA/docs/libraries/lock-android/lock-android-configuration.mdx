description: Modifier l’apparence et le comportement de Lock pour Android
title: 'Lock.Android : Configuration'
---

Vous pouvez utiliser les options ci-dessous pour configurer les fonctionnalités et l’apparence de Lock pour Android.

Ajoutez les options de configuration au Lock Builder à l’aide du format suivant :

```kotlin lines
val lock = Lock.newBuilder(account, callback)
    // Options de configuration
    .closable(true)
    .allowSignUp(false)
    .setPrivacyURL('http://example.com/privacy')
    .setTermsURL('http://example.com/terms')
    // Fin des options de configuration
    .build(this)
```

<div id="general-options">
  ## Options générales
</div>

* **closable(boolean)**: Définit si LockActivity peut être fermé. Par défaut, il ne peut pas l’être.
* **allowedConnections(List)**: Filtre les connexions autorisées dans la liste configurée dans le Dashboard. Par défaut, si cette valeur est vide, toutes les connexions définies dans le Dashboard seront disponibles.
* **hideMainScreenTitle(boolean)**: Si cette option est définie à `true`, l’en-tête de l’écran principal n’affichera pas le titre.

<div id="authentication-options">
  ## Options d’authentification
</div>

* **withAuthenticationParameters(Map&lt;String, Object&gt;)**: Définit des paramètres d’authentification supplémentaires à envoyer à chaque appel de connexion ou d’inscription. Le `scope` par défaut utilisé pour les appels d’authentification est `openid`. Si vous souhaitez en spécifier un autre, utilisez `withAuthenticationParameters` et ajoutez une valeur différente pour la clé `scope`.
* **withScope(String)**: Modifie le `scope` demandé lors de l’exécution d’une requête d’authentification.
* **withAudience(String)**: Modifie l’<Tooltip tip="Audience : identifiant unique de l’audience d’un jeton émis. Nommée aud dans un jeton, sa valeur contient l’ID soit d’une application (Client ID) pour un ID Token, soit d’une API (API Identifier) pour un Access Token." cta="Consulter le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip> demandée lors de l’exécution d’une requête d’authentification.

<div id="database-options">
  ## Options de base de données
</div>

* **withUsernameStyle(int)** : Définit s’il faut demander seulement le courriel, seulement le nom d’utilisateur ou les deux. Les valeurs acceptées sont `USERNAME` et `EMAIL`. Par défaut, cette option respecte la configuration du Dashboard pour le paramètre `requires_username`.
* **loginAfterSignUp(boolean)** : Indique si, après un événement `SignUp`, l’utilisateur doit être connecté automatiquement. La valeur par défaut est `true`.
* **initialScreen(int)** : Permet de personnaliser le formulaire qui s’affichera d’abord au lancement de Lock. Les valeurs acceptées sont `LOG_IN`, `SIGN_UP` et `FORGOT_PASSWORD`. Par défaut, `LOG_IN` est l’écran initial.
* **allowSignUp(boolean)** : Affiche le formulaire d’inscription si une connexion de base de données est configurée et autorisée dans le Dashboard. La valeur par défaut est `true`.
* **allowLogIn(boolean)** : Affiche le formulaire de connexion si une connexion de base de données est configurée. La valeur par défaut est `true`.
* **allowForgotPassword(boolean)** : Affiche le formulaire Mot de passe oublié si une connexion de base de données est configurée et autorisée dans le Dashboard. La valeur par défaut est `true`.
* **allowShowPassword(boolean)** : Affiche un bouton pour basculer la visibilité du contenu d’un champ Password. La valeur par défaut est `true`.
* **setDefaultDatabaseConnection(String)** : Définit quelle connexion de base de données sera utilisée par défaut. C’est utile si votre application comporte plusieurs connexions de base de données configurées.
* **enableEnterpriseWebAuthenticationFor(List)** : Filtre les connexions d’entreprise qui lanceront un flux d’authentification Web dans le navigateur lorsqu’on clique dessus. Seules les stratégies « ad », « adfs » et « waad » sont autorisées.
* **withSignUpFields(List)** : Affiche un deuxième écran avec des champs supplémentaires que l’utilisateur doit remplir après avoir saisi son nom d’utilisateur ou son courriel et son mot de passe dans l’écran d’inscription. Les valeurs envoyées de cette façon peuvent être stockées dans le profil utilisateur au moyen d’un attribut root ou de l’attribut `user_metadata`. Pour en savoir plus, consultez [Lock Android: Custom Fields at Signup](/docs/fr-CA/libraries/lock-android/lock-android-custom-fields-at-signup).
* **setPrivacyURL(String)** : Permet de personnaliser l’URL de la politique de confidentialité. La valeur par défaut est `https://auth0.com/privacy`.
* **setTermsURL(String)** : Permet de personnaliser l’URL des Conditions d’utilisation. La valeur par défaut est `https://auth0.com/terms`.
* **setSupportURL(String)** : Permet de définir une URL de soutien qui s’affichera si une erreur irrécupérable survient dans Lock.
* **setMustAcceptTerms(boolean)** : Oblige l’utilisateur à accepter les Conditions d’utilisation et la politique de confidentialité avant de s’inscrire. La valeur par défaut est `false`.
* **useLabeledSubmitButton(boolean)** : Si cette option est définie à `true`, le bouton d’envoi affichera un libellé correspondant au mode actuel (inscription/connexion) au lieu d’une icône. La valeur par défaut est `true`. Si l’option `hideMainScreenTitle` est définie à `true`, ce paramètre est ignoré et un libellé sera utilisé de toute façon.

<div id="oauth-options">
  ## Options OAuth
</div>

* **withAuthStyle(String, int)** : Personnalise l’apparence d’une connexion donnée (nom) avec un style précis. Consultez [ce document sur les connexions OAuth personnalisées](/docs/fr-CA/libraries/lock-android/lock-android-custom-theming) pour en savoir plus.
* **withConnectionScope(String, String...)** : Permet de préciser des scopes supplémentaires pour un nom de connexion donné, qui seront demandés en plus de ceux définis dans les paramètres de la connexion dans le [Auth0 Dashboard](https://manage.auth0.com/#). Les scopes ne sont validés d’aucune façon et doivent être reconnus par le fournisseur d’authentification donné. Pour obtenir la liste, consultez le [Auth0 Dashboard](https://manage.auth0.com/#) dans les paramètres de la connexion en question.
* **withScheme(String)** : Permet de modifier le schéma du `redirect_uri` envoyé dans l’appel d’autorisation. Par défaut, le schéma est `https`. Si ce paramètre est modifié, l’espace réservé du manifeste pour `auth0Scheme` ainsi que les « Allowed Callbacks URLs » dans le tableau de bord de l’application Auth0 doivent aussi être mis à jour.

<div id="passwordless-options">
  ## Options Passwordless
</div>

* **useCode()**: Envoyer un code au lieu d’un lien par courriel/SMS pour l’authentification <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Passwordless">Passwordless</Tooltip>.
* **useLink()**: Envoyer un lien au lieu d’un code par courriel/SMS pour l’authentification Passwordless.
* **rememberLastLogin(boolean)**: Indique si le courriel ou le numéro de téléphone utilisé lors de la dernière authentification réussie sera enregistré pour ouvrir automatiquement une session la prochaine fois qu’une authentification Passwordless sera demandée.
