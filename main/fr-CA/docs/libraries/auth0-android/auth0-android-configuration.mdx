Auth0.Android peut être configuré à l’aide de diverses options, présentées ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les options ci-dessous diffèrent de celles de la version précédente, V1. Consultez le guide de migration pour en savoir plus.
</Callout>

<div id="withconnection">
  ## withConnection
</div>

L’option `withConnection` vous permet de spécifier la connexion avec laquelle vous voulez vous authentifier.

```kotlin lines
WebAuthProvider.login(account)
                .withConnection("twitter")
                .start(this, callback)
```

<div id="withscope">
  ## withScope
</div>

L’utilisation des scopes vous permet de renvoyer des claims précis pour des champs précis dans votre requête. L’ajout de paramètres à `withScope` vous permet d’ajouter d’autres scopes. Consultez [Scopes](/docs/fr-CA/get-started/apis/scopes) pour en savoir plus.

```kotlin lines
WebAuthProvider.login(account)
                .withScope("openid profile email")
                .start(this, callback)
```

Le scope par défaut est `openid profile email`.

<div id="withconnectionscope">
  ## withConnectionScope
</div>

Il peut arriver que vous deviez vous authentifier avec des scopes de connexion précis, ou des permissions, auprès du fournisseur d’authentification concerné. Consultez [Adding Scopes for an External IDP](/docs/fr-CA/authenticate/identity-providers/adding-scopes-for-an-external-idp). Toutefois, si vous avez besoin d’un accès particulier dans un cas précis de votre application, vous pouvez transmettre des paramètres à `withConnectionScope`. Vous trouverez la liste complète des paramètres disponibles dans les Settings de cette connexion, dans le [Dashboard](https://manage.auth0.com/#), ou dans la documentation du fournisseur d’authentification. Le scope demandé ici s’ajoute à ceux précisés dans les Settings de la connexion dans le Dashboard.

```kotlin lines
WebAuthProvider.login(account)
                .withConnectionScope("email", "profile", "calendar:read")
                .start(this, callback)
```

<div id="withparameters">
  ## withParameters
</div>

Pour envoyer des paramètres supplémentaires dans la requête d’authentification, utilisez `withParameters.`

```kotlin lines
val parameters = mapOf("param1" to "value1")

WebAuthProvider.login(account)
                .withParameters(parameters)
                .start(this, callback)
```

<div id="withheaders">
  ## withHeaders
</div>

Pour envoyer des en-têtes personnalisés au point de terminaison d’autorisation, utilisez `withHeaders.`

```kotlin lines
val headers = mapOf("header1" to "value1")

WebAuthProvider.login(account)
                .withHeaders(headers)
                .start(this, callback)
```

<div id="withscheme">
  ## withScheme
</div>

Si vous n’utilisez pas les &quot;App Links&quot; d’Android ou si vous souhaitez utiliser un schéma différent pour l’URI de redirection, utilisez `withScheme`. Mettez à jour l’espace réservé `auth0Scheme` du Manifest dans le fichier `app/build.gradle`, ainsi que les **Allowed Callback URLs** de l’AllowList dans les paramètres de l’Application sur le [Dashboard](https://manage.auth0.com/#), afin qu’ils correspondent au schéma choisi.

```kotlin lines
WebAuthProvider.login(account)
                .withScheme("myapp")
                .start(this, callback)
```

<Warning>
  Le schéma doit être en minuscules et ne doit pas contenir de traits de soulignement.
</Warning>

<div id="withaudience">
  ## withAudience
</div>

Pour spécifier une <Tooltip tip="Audience : identifiant unique du destinataire d’un jeton émis. Appelée aud dans un jeton, sa valeur correspond à l’ID d’une application (Client ID) pour un ID Token ou d’une API (identifiant d’API) pour un jeton d’accès." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=audience">audience</Tooltip>, utilisez `withAudience`.

```kotlin lines
WebAuthProvider.login(account)
                .withAudience("https://YOUR_DOMAIN/userinfo")
                .start(this, callback)
```
