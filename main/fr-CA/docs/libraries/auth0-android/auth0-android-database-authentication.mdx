<Warning>
  L’authentification par nom d’utilisateur/courriel et mot de passe à partir d’applications natives est désactivée par défaut pour les nouveaux tenants depuis le 8 juin 2017. Il est recommandé d’utiliser Universal Login et d’effectuer plutôt une Web Authentication. Si vous souhaitez tout de même continuer, vous devrez d’abord activer le type d’octroi Password dans votre Dashboard. Consultez [Types d’octroi d’application](/docs/fr-CA/get-started/applications/application-grant-types) pour en savoir plus.
</Warning>

<div id="log-in-with-a-database-connection">
  ## Se connecter à l’aide d’une connexion de base de données
</div>

Pour vous connecter à l’aide d’une connexion de base de données, appelez `login` avec l’**adresse courriel** de l’utilisateur, son **mot de passe** et la **connexion** que vous voulez utiliser pour vous authentifier. La réponse est un objet `Credentials`.

```kotlin lines
authentication
    .login("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(payload: Credentials) {
            // Connecté!
        }

        override fun onFailure(error: AuthenticationException) {
            // Erreur!
        }
    })
```

La portée par défaut est `openid profile email`.

<div id="sign-up-with-a-database-connection">
  ## S’inscrire avec une connexion de base de données
</div>

Pour s’inscrire avec une connexion de base de données, appelez la méthode `signUp` en fournissant l’adresse courriel et le mot de passe de l’utilisateur, ainsi que le nom de la connexion.

```kotlin lines
authentication
    .signUp("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(result: Credentials) {
            // Inscrit et connecté!
        }

        override fun onFailure(error: AuthenticationException) {
            // Erreur!
        }
    });
```
