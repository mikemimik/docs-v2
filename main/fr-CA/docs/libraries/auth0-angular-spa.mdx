L’Auth0 Angular SDK est une bibliothèque JavaScript qui permet de mettre en œuvre l’authentification et l’autorisation dans des applications Angular avec Auth0. Il fournit un service, une garde d’authentification et un intercepteur HTTP pour vous permettre d’effectuer les tâches d’authentification courantes dans vos applications Angular.

L’Auth0 Angular SDK prend en charge les détails liés aux modes d’octroi et aux protocoles, l’expiration et le renouvellement des tokens, ainsi que le stockage des tokens et la mise en cache. En coulisses, il implémente [Universal Login](/docs/fr-CA/authenticate/login/auth0-universal-login) et le [Authorization Code Grant Flow with PKCE](/docs/fr-CA/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

La bibliothèque est [hébergée sur GitHub](https://github.com/auth0/auth0-angular), où vous pouvez [en savoir plus sur l’API](https://auth0.github.io/auth0-angular/).

<div id="installation">
  ## Installation
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le SDK Angular est compatible avec les SPA Angular à partir de la version 14.
</Callout>

Utilisez l’[Angular CLI](https://cli.angular.io/) pour installer le SDK dans votre projet :

`ng add @auth0/auth0-angular`

<div id="getting-started">
  ## Pour commencer
</div>

Commencez à intégrer le SDK en important `AuthModule` dans votre projet et en le configurant pour votre application cliente Auth0 :

```jsx lines
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';

// Importer le module depuis le SDK
import { AuthModule } from '@auth0/auth0-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,

    // Importer le module dans l'application, avec la configuration
    AuthModule.forRoot({
      domain: '{yourDomain}',
      clientId: 'YOUR_AUTH0_CLIENT_ID',
      authorizationParams: {
        redirect_uri: window.location.origin,
      },
    }),
  ],

  bootstrap: [AppComponent],
})
export class AppModule {}
```

<div id="login">
  ## Login
</div>

Le SDK prend entièrement en charge le système d’injection de dépendances d’Angular. Pour commencer, importez le type `AuthService` dans un composant et injectez-le dans le constructeur.

Ensuite, pour amorcer la connexion de l’utilisateur, appelez les méthodes `loginWithRedirect` ou `loginWithPopup` du service :

```jsx lines
import { Component } from '@angular/core';

// Importer le type AuthService depuis le SDK
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-login',
  template: '<button (click)="login">Log in</button>',
  styles: [],
})
export class LoginComponent {
  // Injecter le service d'authentification dans votre composant via le constructeur
  constructor(public auth: AuthService) {}

  login(): void {
    // Appeler cette méthode pour rediriger l'utilisateur vers la page de connexion
    this.auth.loginWithRedirect();
  }
}
```

<div id="logout">
  ## Déconnexion
</div>

Utilisez `logout` pour déconnecter vos utilisateurs. Assurez-vous que `returnTo` figure dans **Allowed Logout URLs** de votre <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

```jsx lines
import { Component } from '@angular/core';

// Importer le type AuthService depuis le SDK
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-logout',
  template: '<button (click)="logout">Log out</button>',
  styles: [],
})
export class LogoutComponent {
  // Injecter le service d'authentification dans votre composant via le constructeur
  constructor(@Inject(DOCUMENT) private doc: Document, public auth: AuthService) {}

  login(): void {
    // Appeler cette méthode pour rediriger l'utilisateur vers la page de connexion
    this.auth.logout({ 
      logoutParams: { 
        returnTo: this.doc.location.origin 
      }
    });
  }
}
```

<div id="authentication-state">
  ## État d’authentification
</div>

L’état d’authentification est exposé au moyen de l’observable `isAuthenticated$` :

```html lines
<div *ngIf="auth.isAuthenticated$ | async">
  <app-login></app-login>
</div>
```

<div id="user-profile">
  ## Profil utilisateur
</div>

Le profil de l’utilisateur authentifié est accessible au moyen de l’observable `user$` :

```html lines
<ul *ngIf="auth.user$ | async as user">
  <li>{{ user.name }}</li>
  <li>{{ user.email }}</li>
</ul>
```

En interne, cet observable découle de l’observable `isAuthenticated$`, il n’est donc pas nécessaire de vérifier manuellement si l’utilisateur est connecté. Il commencera à émettre des valeurs dès qu’un utilisateur authentifié sera présent.

<div id="protect-a-route">
  ## Protéger une route
</div>

Protégez un composant de route à l’aide de l’`AuthGuard` intégré. Lorsqu’un utilisateur non authentifié accède à cette route, il est redirigé vers la page de connexion, puis ramené à cette page après la connexion.

La garde d’authentification doit être ajoutée à une définition de route à l’aide du hook `canActivate` (ou `canActivateChild` si vous utilisez des routes enfants) :

```jsx lines expandable
import { NgModule } from '@angular/core';
import { Routes, RouterModule } from '@angular/router';
import { HomeComponent } from './unprotected/unprotected.component';
import { ProtectedComponent } from './protected/protected.component';

// Importer le garde d'authentification
import { AuthGuard } from '@auth0/auth0-angular';

const routes: Routes = [
  {
    path: 'protected',
    component: ProtectedComponent,

    // Protéger une route en enregistrant le garde d'authentification dans le hook `canActivate`
    canActivate: [AuthGuard],
  },
  {
    path: '',
    component: HomeComponent,
    pathMatch: 'full',
  },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule],
})
export class AppRoutingModule {}
```

Le garde d’authentification prend également en charge le hook `canLoad` pour empêcher le chargement des modules à chargement différé lorsque l’utilisateur n’est pas authentifié.

## Appeler une API

Le SDK comprend un `HttpInterceptor` intégré qui ajoute automatiquement des <Tooltip tip="Jeton d’accès : identifiant d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/docs/fr-CA/glossary?term=access+tokens">jetons d’accès</Tooltip> en injectant un en-tête `Authorization` lorsque des requêtes sont effectuées à l’aide du service `HttpClient` d’Angular.

Vous devez configurer le SDK pour indiquer à quelles routes de l’API cet en-tête doit être ajouté automatiquement, afin d’éviter que des jetons d’accès ne soient transmis à des destinataires non prévus.

<div id="import-the-http-interceptor">
  ### Importer l’intercepteur HTTP
</div>

Pour commencer, importez les types nécessaires pour intégrer l’intercepteur HTTP Auth0 dans votre application :

```jsx lines
// Importez le module intercepteur et les types Angular dont vous aurez besoin
import { HttpClientModule, HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthHttpInterceptor } from '@auth0/auth0-angular';

// Enregistrez l'intercepteur dans votre module d'application dans le tableau `providers`
@NgModule({
  declarations: [],
  imports: [
    BrowserModule,
    HttpClientModule,     // Enregistrez ceci pour pouvoir effectuer des appels d'API avec HttpClient
    AppRoutingModule,
    AuthModule.forRoot(...),
  ],
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthHttpInterceptor, multi: true },
  ],
  bootstrap: [AppComponent],
})
```

**Remarque :** L’intercepteur doit être enregistré manuellement dans le module de votre application (plutôt que automatiquement), car il est conçu pour être utilisé en chaîne avec d’autres intercepteurs, et nous ne pouvons pas garantir qu’il sera inséré au bon endroit.

<div id="configure-the-http-interceptor">
  ### Configurer l’intercepteur HTTP
</div>

Il existe plusieurs façons de le configurer selon vos besoins. Une route peut être définie au moyen d’une chaîne, d’un objet d’options qui influent sur le jeton envoyé, et elle peut aussi être mise en correspondance selon le verbe HTTP utilisé. Les URI de route peuvent également inclure un caractère générique pour désigner un groupe de routes.

**Remarque :** Si un appel HTTP est effectué sans correspondre à une route précisée dans la configuration, l’intercepteur sera ignoré et l’en-tête `Authorization` ne sera pas inclus.

Cet exemple montre des façons courantes d’indiquer au SDK quelles routes doivent inclure un en-tête `Authorization` :

```jsx lines expandable
// Modifiez votre configuration SDK existante pour inclure la config httpInterceptor
AuthModule.forRoot({
  domain: '{yourDomain}',
  clientId: 'YOUR_AUTH0_CLIENT_ID',
  authorizationParams: {
    redirect_uri: window.location.origin,
  },

  // La configuration de AuthHttpInterceptor
  httpInterceptor: {
    allowedList: [
      // Joindre des jetons d'accès à tous les appels vers '/api' (correspondance exacte)
      '/api',

      // Joindre des jetons d'accès à tous les appels commençant par '/api/'
      '/api/*',

      // Correspondre à tout ce qui commence par /api/accounts, mais aussi spécifier l'audience et la portée que doit
      // avoir le jeton d'accès joint
      {
        uri: '/api/accounts/*',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'read:accounts',
          }
        },
      },

      // Correspondance sur la méthode HTTP
      {
        uri: '/api/orders',
        httpMethod: 'post',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'write:orders',
          }
        },
      },

      // Utilisation d'un URI absolu
      {
        uri: 'https://your-domain.auth0.com/api/v2/users',
        tokenOptions: {
          authorizationParams: {
            audience: 'https://your-domain.com/api/v2/',
            scope: 'read:users',
          }
        },
      },
    ],
  },
});
```

**Remarque :** `tokenOptions` est transmis directement à la méthode `getTokenSilently` du SDK SPA sous-jacent. Veuillez consulter [la documentation](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html) pour plus de détails sur les propriétés acceptées.

<div id="call-an-api">
  ### Appeler une API
</div>

Effectuez votre appel à l’API à l’aide du service `HttpClient` d’Angular. Les jetons d’accès sont automatiquement inclus dans l’en-tête `Authorization` pour les routes correspondant à la configuration.

```jsx lines
export class MyComponent {
  constructor(private http: HttpClient) {}

  callApi(): void {
    this.http.get('/api').subscribe(result => console.log(result));
  }
}
```

Notez qu’il n’y a rien d’autre à faire dans le code appelant pour fournir des tokens ou toute autre information d’autorisation; tout cela est géré par l’intercepteur et la configuration du module.
