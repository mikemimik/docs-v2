Voici le scénario : après l’authentification de l’utilisateur, vous voulez stocker ses identifiants et les utiliser tant qu’ils sont valides. Une fois expirés, vous voudrez les actualiser à l’aide de `refreshToken` afin d’éviter de réafficher la page de connexion. Au lieu de le faire automatiquement, vous exigez que l’utilisateur valide son identité à l’aide de son empreinte digitale ou de la reconnaissance faciale.

Vous utiliserez l’utilitaire [Credentials Manager](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) dans [Auth0.swift](https://github.com/auth0/Auth0.swift/) pour simplifier la gestion des identifiants de l’utilisateur et effectuer l’authentification biométrique.

<div id="getting-started">
  ## Pour commencer
</div>

D’abord, importez le module `Auth0` :

`import Auth0`

<div id="credentials-manager">
  ### Credentials Manager
</div>

Avant de récupérer vos identifiants, vous pouvez aussi utiliser l’authentification biométrique (Face ID ou Touch ID) offerte sur votre appareil iOS.

Commencez par configurer le Credentials Manager. Activez ensuite la biométrie. Vous pouvez aussi fournir un titre à afficher dans l’invite.

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Nous vous recommandons fortement d’ajouter le paramètre [NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) au fichier `Info.plist` de votre projet afin d’indiquer pourquoi Face ID est utilisé. Dans certains cas, si vous ne fournissez pas de chaîne de description et que l’utilisateur tente de s’authentifier avec Face ID, la tentative peut échouer.

```xml lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

<div id="login">
  ### Login
</div>

Présentez la page <Tooltip tip="Universal Login : votre application redirige l’utilisateur vers Universal Login, hébergé sur l’Authorization Server d’Auth0, pour vérifier son identité." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> puis, une fois l’authentification réussie, transmettez les identifiants au Credentials Manager.

export const codeExample = `Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // Gérer l’erreur
        case .success(let credentials):
            // Stocker les identifiants en toute sécurité avec le Credentials Manager
            credentialsManager.store(credentials: credentials)
        }
}`;

<div id="renew-user-credentials">
  ### Renouveler les identifiants de l’utilisateur
</div>

Lorsque vous devez renouveler les identifiants de l’utilisateur, vous pouvez appeler la méthode `credentials` du Credentials Manager.

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // Gérer l'erreur
        // Revenir à l'écran de connexion
    }
    // Continuer à rediriger l'utilisateur car l'authentification a réussi
}
```

Il n’est pas nécessaire d’enregistrer manuellement les nouveaux identifiants, car le Credentials Manager s’en charge lors du renouvellement.
