Auth0.Android est une bibliothèque côté client que vous pouvez utiliser avec votre application Android pour authentifier les utilisateurs et accéder aux [API d’Auth0](/fr-CA/docs/api).

Consultez le [dépôt Auth0.Android](https://github.com/auth0/Auth0.Android) sur GitHub.

<div id="requirements">
  ## Configuration requise
</div>

La version 21 de l’API Android ou une version ultérieure est requise.

<div id="installation">
  ## Installation
</div>

Ajoutez le SDK à votre projet. La bibliothèque enverra des requêtes aux API authentification et Management d’Auth0.

<div id="add-auth0-to-gradle">
  ### Ajouter Auth0 à Gradle
</div>

Dans la section `dependencies` du fichier `build.gradle` de votre application, ajoutez ce qui suit :

```kotlin lines
dependencies {
  // Ajouter le SDK Android Auth0
  implementation 'com.auth0.android:auth0:2.+'
}
```

Si Android Studio signale une erreur avec le signe `+`, ou si vous souhaitez utiliser une version précise, consultez la dernière version sur [Maven](https://central.sonatype.com/artifact/com.auth0.android/auth0).

<Card title="Synchroniser le projet avec Gradle">
  N’oubliez pas de synchroniser à l’aide de l’invite d’Android Studio ou d’exécuter `./gradlew clean build` à partir de la ligne de commande.
</Card>

Dans la section `android`, ciblez respectivement le bytecode Java 8 pour Android et pour les plugins Kotlin.

```kotlin lines
android {
  compileOptions {
    sourceCompatibility JavaVersion.VERSION_1_8
    targetCompatibility JavaVersion.VERSION_1_8
  }
  kotlinOptions {
    jvmTarget = '1.8'
  }
}
```

<div id="permissions">
  ### Autorisations
</div>

Ouvrez le fichier `AndroidManifest.xml` de votre application et ajoutez l’autorisation suivante.

```text lines
<uses-permission android:name="android.permission.INTERNET" />
```

<div id="configure-for-universal-login">
  ## Configurer pour Universal Login
</div>

D’abord, allez dans [Dashboard &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur le nom de l’application que vous voulez modifier. Dans **Allowed Callback URLs**, assurez-vous que votre URL respecte le format suivant :
`https://YOUR_DOMAIN/android/{YOUR_APP_PACKAGE_NAME}/callback`

Ensuite, remplacez `{YOUR_APP_PACKAGE_NAME}` par le véritable nom de package de votre application. Vous le trouverez dans votre fichier `app/build.gradle`, dans la valeur `applicationId`.

Puis, dans votre fichier `app/build.gradle`, ajoutez les [manifest placeholders](https://developer.android.com/build/manage-manifests) pour les propriétés Auth0 Domain et Auth0 Scheme, que la bibliothèque utilisera pour enregistrer un intent-filter qui intercepte l’URI de rappel.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vous ne prévoyez pas utiliser la fonctionnalité Web authentification ni déclarer les manifest placeholders, vous pouvez redéclarer manuellement l’activité dans le fichier Manifest de votre application Android afin de la retirer du fichier Manifest fusionné. Gradle ne signalera ainsi plus l’absence des manifest placeholders.

  Veuillez [consulter la FAQ](https://github.com/auth0/auth0.android/blob/main/FAQ.md) pour en savoir plus.
</Callout>

```kotlin lines
apply plugin: 'com.android.application'

android {
    compileSdkVersion 30
    defaultConfig {
        applicationId "com.auth0.samples"
        minSdkVersion 21
        targetSdkVersion 30
        //...

        //---> Ajouter la ligne suivante
        manifestPlaceholders = [auth0Domain: "@string/com_auth0_domain", auth0Scheme: "https"]
        //<---
    }
    //...
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Il est recommandé de définir des ressources réutilisables comme `@string/com_auth0_domain` plutôt que de les coder en dur.
</Callout>

<div id="initialize-auth0">
  ## Initialiser Auth0
</div>

Créez un nouvel objet `Auth0` à l’aide de votre Auth0 <Tooltip tip="Client ID : Valeur d’identification fournie par Auth0 à votre ressource enregistrée." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+ID">client ID</Tooltip> et de votre domaine. Les objets s’en serviront plus tard pour interagir avec les points de terminaison d’Auth0.

```kotlin lines
val auth0 = Auth0("{yourClientId}", "YOUR_DOMAIN")
```

Vous pouvez également initialiser cet objet à l’aide d’un Context Android, comme dans [cet exemple d’utilisation](https://github.com/auth0/Auth0.Android/tree/master#usage).

<div id="reset-user-password">
  ### Réinitialiser le mot de passe d’un utilisateur
</div>

Pour lancer la réinitialisation du mot de passe d’un utilisateur, appelez `resetPassword` en indiquant l’adresse courriel de l’utilisateur et le nom de la connexion de base de données comme paramètres.

Les demandes de réinitialisation du mot de passe échouent en cas d’erreur liée au réseau, mais pas si l’adresse courriel indiquée n’existe pas dans la base de données (pour des raisons de sécurité).
