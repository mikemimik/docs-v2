Lock pour Android peut s’intégrer à vos applications Android natives pour offrir à vos utilisateurs une façon élégante de se connecter et de s’inscrire dans votre application. Il prend aussi en charge les <Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> sociaux comme Facebook, Google ou X, ainsi que des fournisseurs d’entreprise comme Active Directory.

Consultez le [dépôt Lock.Android](https://github.com/auth0/Lock.Android) sur GitHub.

<div id="requirements">
  ## Exigences
</div>

Pour utiliser l’UI de Lock ou votre propre UI au moyen de la [bibliothèque Auth0.Android](https://github.com/auth0/Auth0.Android), le niveau d’API Android minimal requis est 21 ou plus, et la version de Java doit être 8 ou supérieure. Vous aurez aussi besoin d’une application Auth0 de type &quot;Native&quot;.

Voici ce dont vous avez besoin dans `build.gradle` pour cibler le bytecode Java 8 pour les plugins Android et Kotlin, respectivement.

```kotlin lines
android {
    compileOptions {
        sourceCompatibility JavaVersion.VERSION_1_8
        targetCompatibility JavaVersion.VERSION_1_8
    }

    kotlinOptions {
        jvmTarget = '1.8'
    }
}
```

<div id="installation">
  ## Installation
</div>

Lock est offert sur [Maven Central](https://central.sonatype.com/). Pour commencer à utiliser Lock, ajoutez ces lignes à votre fichier de dépendances `build.gradle` :

`implementation 'com.auth0.android:lock:3.+'`

Vous pouvez consulter la version la plus récente dans le [Readme](https://github.com/auth0/Lock.Android#install) du dépôt ou dans [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock).

Après avoir ajouté votre dépendance Gradle, n’oubliez pas de synchroniser votre projet avec les fichiers Gradle.

<div id="dashboard-settings">
  ## Paramètres du Dashboard
</div>

Vous devez configurer quelques paramètres dans votre [Auth0 Dashboard](https://manage.auth0.com/#) avant de commencer.

<div id="callback-url">
  ### URL de rappel
</div>

Rendez-vous dans votre <Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> et accédez aux paramètres de l’application. Ajoutez l’URL suivante aux **Allowed Callback URLs** de l’application :

`https://{yourDomain}/android/{yourAppPackageName}/callback`

Remplacez `{yourAppPackageName}` par le nom de package réel de votre application, indiqué dans votre fichier `app/build.gradle` comme valeur `applicationId`.

<div id="keystores-and-key-hashes">
  ### Keystores et Key Hashes
</div>

Les applications Android doivent être signées avant de pouvoir être installées sur un appareil. À cette fin, l’IDE Android Studio génère, lors de son premier lancement, un « Android Debug Keystore » par défaut qu’il utilise pour signer les versions de développement. Ce [Keystore](https://developer.android.com/studio/publish/app-signing) sera probablement différent de celui de votre version de production, puisqu’il sert à vous identifier comme développeur.

Lorsque vous utilisez la fonctionnalité Web Authentication (c.-à-d. les connexions sociales), Lock est configuré par défaut pour tenter d’utiliser les Android App Links. Cela nécessite un paramètre supplémentaire dans le Dashboard de l’application Auth0. Utilisez notre [guide Android Keystores and Key Hashes](/fr-CA/docs/libraries/auth0-android/android-development-keystores-hashes) pour terminer cette étape.

<div id="configuring-the-sdk">
  ### Configuration du SDK
</div>

Dans votre fichier `app/build.gradle`, ajoutez les [manifest placeholders](https://developer.android.com/build/manage-manifests) pour les propriétés Auth0 Domain et Auth0 Scheme, qui seront utilisés en interne par la bibliothèque pour enregistrer un intent-filter capturant le résultat de l’authentification.

```kotlin lines
plugins {
    id "com.android.application"
    id "kotlin-android"
}

android {
    compileSdkVersion 30
    defaultConfig {
        applicationId "com.auth0.samples"
        minSdkVersion 21
        targetSdkVersion 30
        // ...

        // ---> Ajouter la ligne suivante
        manifestPlaceholders = [auth0Domain: "@string/com_auth0_domain", auth0Scheme: "https"]
        // <---
    }
}
```

C’est une bonne pratique d’ajouter ces valeurs au fichier `strings.xml` comme ressources de chaîne afin de pouvoir les référencer plus tard dans le code. Ce guide suivra cette pratique.

export const codeExample = `<resources>
    <string name="com_auth0_client_id">{yourClientId}</string>
    <string name="com_auth0_domain">{yourDomain}</string>
</resources>`;

<div id="sdk-usage">
  ### Utilisation du SDK
</div>

Dans l’activité où vous comptez appeler Lock, créez une instance de `Auth0` avec les informations de votre application. Le plus simple est de la créer en lui passant un contexte Android. Celui-ci utilisera les valeurs définies précédemment dans le fichier `strings.xml`. Pour que cela fonctionne, les ressources de chaîne doivent être définies à l’aide des mêmes clés que celles indiquées ci-dessus.

```text lines
val account = Auth0(context)
```

Déclarez une implémentation de `AuthenticationCallback` qui gérera les événements d’authentification de l’utilisateur. L’objet `Credentials` renvoyé en cas d’authentification réussie contiendra les jetons que votre application ou votre API utilisera. Consultez [Tokens](/fr-CA/docs/secure/tokens) pour plus de détails.

```kotlin lines
private val callback = object : AuthenticationCallback() {
    override fun onAuthentication(credentials: Credentials) {
        // Authentifié
    }

    override fun onError(error: AuthenticationException) {
        // Une exception s'est produite
    }
}
```

Préparez une nouvelle instance de Lock à l’aide de la classe Builder pour la configurer. Fournissez les renseignements du compte ainsi que l’implémentation du callback déclarée ci-dessus. Des valeurs comme <Tooltip tip="Audience : identifiant unique de l’audience d’un jeton émis. Nommée aud dans un jeton, sa valeur contient l’ID d’une application (ID client) pour un jeton d’identification ou d’une API (identifiant d’API) pour un jeton d’accès." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=audience">audience</Tooltip>, la portée et les connexions disponibles peuvent être configurées ici.

Une fois cela fait, créez l’instance Lock. Cette instance est conçue pour être réutilisée et doit être supprimée lorsqu’elle n’est plus nécessaire. Un bon endroit pour le faire est dans la méthode `onDestroy` de votre activité.

```kotlin lines expandable
// Cette activité affichera Lock
class MyActivity : AppCompatActivity() {

    private lateinit var lock: Lock

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val account = Auth0(this)
        // Instancier Lock une seule fois
        lock = Lock.newBuilder(account, callback)
            // Personnaliser Lock
            .build(this)
    }

    override fun onDestroy() {
        super.onDestroy()
        // Important ! Libérer Lock et ses ressources
        lock.onDestroy(this)
    }

    private val callback = object : AuthenticationCallback() {
        override fun onAuthentication(credentials: Credentials) {
            // Authentifié
        }

        override fun onError(error: AuthenticationException) {
            // Une exception s'est produite
        }
    }
}
```

Enfin, lancez le widget `Lock` depuis votre activité.

```kotlin lines
startActivity(lock.newIntent(this))
```

C&#39;est tout ! Lock s&#39;occupera du reste pour vous.

<div id="android-app-links-custom-scheme">
  #### Android App Links - schéma personnalisé
</div>

Le schéma d’URI de callback utilisé dans cet article et, par défaut, par Lock est `https`. C’est ce qui fonctionne le mieux avec Android Marshmallow (API 23) ou une version plus récente si vous utilisez les [Android App Links](https://developer.android.com/training/app-links). Toutefois, sur les versions antérieures d’Android, la boîte de dialogue du sélecteur d’intention peut s’afficher et inviter l’utilisateur à choisir soit votre application, soit le navigateur pour traiter l’intention. C’est ce qu’on appelle la « boîte de dialogue de désambiguïsation ». Vous pouvez modifier ce comportement en utilisant un schéma personnalisé unique afin que le système d’exploitation ouvre directement le lien dans votre application.

1. Mettez à jour la valeur d’espace réservé du Manifest `auth0Scheme` dans le fichier `app/build.gradle`, ou directement dans la définition du filtre d’intention du fichier `AndroidManifest.xml`, en remplaçant le schéma actuel par le nouveau.
2. Mettez à jour les « Allowed Callback URLs » dans les paramètres de votre Application du Auth0 Dashboard afin qu’elles correspondent à des URL qui commencent par le nouveau schéma.
3. Appelez `withScheme()` lors de la configuration de Lock avec le générateur, en lui transmettant le schéma que vous souhaitez utiliser.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La valeur du schéma doit être entièrement en minuscules. Un message d’avertissement sera consigné si ce n’est pas le cas, et l’authentification ne se terminera jamais.
</Callout>
