Le SDK Auth0-PHP s’intègre à vos applications PHP pour offrir des fonctionnalités simples de connexion et d’inscription des utilisateurs. Il prend en charge les <Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> sociaux comme Facebook, Google ou X, ainsi que des fournisseurs d’entreprise comme Active Directory. Le SDK fournit des méthodes pratiques pour accéder aux endpoints Authentication et Management d’Auth0.

Le dépôt Auth0-PHP est open source et [hébergé sur GitHub](https://github.com/auth0/auth0-PHP). Nous apprécions toutes les contributions, y compris les signalements de bogues, les propositions d’amélioration et les pull requests.

<div id="requirements">
  ## Prérequis
</div>

* PHP 7.4+ (8.0+ recommandé)
* [Composer](https://getcomposer.org/doc/00-intro.md)

<div id="installation">
  ## Installation
</div>

L’installation de l’Auth0 PHP SDK nécessite [Composer](https://getcomposer.org/doc/00-intro.md#installation-linux-unix-macos), l’outil standard de gestion des dépendances pour PHP. Composer vous permet de déclarer les bibliothèques dont votre projet a besoin et les installe pour vous. Assurez-vous que Composer est installé et accessible depuis votre terminal avant de poursuivre.

Ensuite, exécutez la commande shell suivante dans le répertoire de votre projet pour installer le SDK :

```php lines
composer require auth0/auth0-php
```

Cela créera un sous-dossier `vendor` dans votre projet et téléchargera toutes les dépendances nécessaires pour utiliser le SDK PHP. Un fichier `vendor/autoload.php`, nécessaire pour que le SDK fonctionne avec votre application, sera également créé, et nous l’importerons plus tard.

<div id="getting-started">
  ## Pour commencer
</div>

Pour utiliser les API d’Authentication et de Management d’Auth0, vous aurez besoin d’un compte Auth0 gratuit et d’une application :

1. Allez à [auth0.com/signup](https://auth0.com/signup) et créez votre compte.
2. Une fois dans le dashboard, allez à **Applications**, puis à **Create Application**.
3. Donnez un nom à votre application, sélectionnez **Regular Web Application**, puis **Create**
4. Cliquez sur l’onglet **Settings** pour obtenir les informations d’identification requises ci-dessous. Pour en savoir plus, consultez [Application Settings](/fr-CA/docs/get-started/applications/application-settings).

<div id="configure-the-sdk">
  ## Configurer le SDK
</div>

Vous devriez utiliser des [variables d’environnement](https://secure.php.net/manual/en/reserved.variables.environment.php) pour stocker et charger des identifiants Auth0 sensibles. Vous éviterez ainsi d’avoir à les coder en dur dans votre application. Créons un fichier `.env` à la racine du répertoire de notre projet pour y stocker les identifiants de notre application :

Le moyen le plus simple d’utiliser des variables d’environnement dans votre projet consiste à utiliser une bibliothèque comme [PHP Dotenv](https://github.com/josegonzalez/php-dotenv) avec un fichier `.env` local. Créez un fichier `.env` (assurez-vous qu’il n’est pas accessible publiquement et qu’il est exclu du contrôle de version) et ajoutez les valeurs suivantes :

export const codeExample = `# L’URL du domaine de notre tenant Auth0.
# Si nous utilisons un domaine personnalisé, assurez-vous d’utiliser cette valeur à la place.
AUTH0_DOMAIN='https://{yourDomain}'

# Le Client ID de notre application Auth0.
AUTH0_CLIENT_ID='{yourClientId}'

# Le Client Secret de notre application Auth0.
AUTH0_CLIENT_SECRET='{yourClientSecret}'

# Une longue valeur secrète que nous utiliserons pour chiffrer les cookies de session. Elle peut être générée à l’aide de \`openssl rand -hex 32\` dans notre shell.
AUTH0_COOKIE_SECRET='VOIR LE COMMENTAIRE CI-DESSUS'

# L’URL de base de notre application.
AUTH0_BASE_URL='http://127.0.0.1:3000'`;

Vous ne devriez jamais enregistrer ce fichier dans votre système de gestion de versions ni le partager de manière non sécurisée. Son contenu doit être manipulé avec soin et traité comme un mot de passe.
Comme PHP ne peut pas lire nativement notre fichier `.env`, vous devrez installer une bibliothèque PHP pour le faire. Dans cette documentation, nous utiliserons `vlucas/phpdotenv`, mais n&#39;importe quelle bibliothèque &quot;dotenv&quot; fera l&#39;affaire. Depuis le répertoire de votre projet, exécutez la commande shell suivante pour installer la bibliothèque :

```php lines
composer require vlucas/phpdotenv
```

<div id="initialize-the-sdk">
  ## Initialiser le SDK
</div>

Nous sommes maintenant prêts à configurer et à initialiser une instance du SDK dans notre nouvelle application PHP. Commençons par créer le fichier source PHP que nous utiliserons pour cette démonstration, `index.php`, puis utilisons l’extrait suivant pour démarrer :

```php lines
<?php

// Importer le chargeur automatique Composer pour rendre les classes du SDK accessibles :
require 'vendor/autoload.php';

// Charger nos variables d'environnement depuis le fichier .env :
(Dotenv\Dotenv::createImmutable(__DIR__))->load();

// Instancier maintenant la classe Auth0 avec notre configuration :
$auth0 = new \Auth0\SDK\Auth0([
    'domain' => $_ENV['AUTH0_DOMAIN'],
    'clientId' => $_ENV['AUTH0_CLIENT_ID'],
    'clientSecret' => $_ENV['AUTH0_CLIENT_SECRET'],
    'cookieSecret' => $_ENV['AUTH0_COOKIE_SECRET']
]);
```

Félicitations ! Votre application est maintenant configurée et prête à être utilisée avec Auth0.
