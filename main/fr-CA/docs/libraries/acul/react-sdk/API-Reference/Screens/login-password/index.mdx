---
title: "LoginPassword"
description: "Décrit tous les hooks et méthodes offerts pour personnaliser l’écran `login-password` de Universal Login."
---

L’écran `login-password` demande le mot de passe de l’utilisateur à la deuxième étape d’un flux d’authentification où l’identifiant est saisi en premier. Il permet aussi de passer à un fournisseur d’identité fédéré ou à une autre connexion.

<div id="import">
  ## Importation
</div>

Chaque écran a son propre ensemble de hooks et de méthodes. Le SDK prend en charge les **partial imports** et les **root imports** pour chaque écran.

* L’utilisation d’un partial import vous permet d’inclure uniquement le code nécessaire à votre use case.
* L’utilisation d’un root import vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez une build unifiée capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useLoginPassword } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useLoginPassword,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  login,
  federatedLogin,
  switchConnection,
} from '@auth0/auth0-acul-react/login-password';

function LoginPasswordForm() {
  const { login } = useLoginPassword();
  return (
    <button onClick={() => login({ username: 'user@example.com', password: 'secret' })}>
      Sign In
    </button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 sur l’écran `login-password`. Importez-les depuis `@auth0/auth0-acul-react/login-password`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les configurations d’identité visuelle, comme le logo, les couleurs et les paramètres du thème affichés sur l’écran `login-password`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/login-password';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit les configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `login-password`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/login-password';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organization de l’utilisateur si la connexion est associée à une Organization. Retourne `null` lorsqu’aucun contexte d’Organization n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/login-password';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>Aucun contexte d’organisation</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel du flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/login-password';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnLoginPassword">ScreenMembersOnLoginPassword</a></span>}>
  Ce hook contient des détails propres à l’écran `login-password`, notamment sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/login-password';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au tenant, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/login-password';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembersOnLoginPassword">TransactionMembersOnLoginPassword</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `login-password`, comme les connexions actives et l’état actuel du flux.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/login-password';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Ce hook contient les données non fiables transmises à l’écran, comme un identifiant prérempli provenant des paramètres d’URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/login-password';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Ce hook fournit des renseignements sur l’utilisateur actif, notamment son `username`, son `email` et les méthodes d’authentification disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/login-password';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useLoginPassword" type={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/LoginPasswordMembers">LoginPasswordMembers</a>}>
  Ce hook renvoie toutes les méthodes et tout le contexte disponibles sur l’écran `login-password`.
</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="federatedLogin" type="Promise<void>">
  Cette méthode lance l’authentification par l’intermédiaire d’un fournisseur d’identité fédéré, comme Google ou GitHub.

  ```jsx Example
  import { useLoginPassword } from '@auth0/auth0-acul-react/login-password';

  function SocialButton() {
    const { federatedLogin } = useLoginPassword();
    return (
      <button onClick={() => federatedLogin({ connection: 'google-oauth2' })}>
        Sign in with Google
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [FederatedLoginPayloadOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/FederatedLoginPayloadOptions).
    </ParamField>

    <ParamField body="connection" type="string" required>
      Le nom de la connexion sociale à utiliser (par ex. `google-oauth2`, `github`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="login" type="Promise<void>">
  Cette méthode envoie les informations d’identification de l’utilisateur pour terminer l’authentification sur l’écran `login-password`.

  ```jsx Example
  import { useLoginPassword } from '@auth0/auth0-acul-react/login-password';

  function LoginPasswordForm() {
    const { login } = useLoginPassword();
    return (
      <button onClick={() => login({ username: 'user@example.com', password: 'secret' })}>
        Sign In
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [LoginPasswordOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/LoginPasswordOptions).
    </ParamField>

    <ParamField body="username" type="string" required>
      L’identifiant (courriel, numéro de téléphone ou nom d’utilisateur) à utiliser pour s’authentifier.
    </ParamField>

    <ParamField body="password" type="string" required>
      Le mot de passe utilisé pour l’authentification.
    </ParamField>

    <ParamField body="captcha?" type="string">
      Valeur CAPTCHA requise lorsque la détection des robots est activée pour le tenant.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Champs personnalisés supplémentaires précédés du préfixe `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="switchConnection" type="Promise<void>">
  Cette méthode permet de passer à une autre connexion d’authentification sans redémarrer le flux.

  ```jsx Example
  import { useLoginPassword } from '@auth0/auth0-acul-react/login-password';

  function ConnectionSwitcher() {
    const { switchConnection } = useLoginPassword();
    return (
      <button onClick={() => switchConnection({ connection: 'enterprise-saml' })}>
        Use SSO
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SwitchConnectionOptionsLoginPassword](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/SwitchConnectionOptionsLoginPassword).
    </ParamField>

    <ParamField body="connection" type="string" required>
      Le nom de la connexion vers laquelle basculer. Utilisez `email` ou `sms` pour Passwordless, ou un nom de connexion DB pour l’authentification par mot de passe.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Champs personnalisés supplémentaires précédés du préfixe `ulp-` (par exemple, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<div id="commonutility-hooks">
  ## Hooks communs/utilitaires
</div>

<ParamField body={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">
  Ce hook récupère les options du thème actuel avec une configuration aplatie à partir du contexte de l’identité visuelle.
</ParamField>

<ParamField body={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Ce hook retourne une fonction permettant de changer la langue d’affichage sur l’écran ACUL actuel.
</ParamField>

<ParamField body={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Ce hook récupère le contexte et l’état de l’écran actuel.
</ParamField>

<ParamField body={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Ce hook lit et gère les erreurs du serveur, du client et du développeur affichées à l’écran.
</ParamField>
