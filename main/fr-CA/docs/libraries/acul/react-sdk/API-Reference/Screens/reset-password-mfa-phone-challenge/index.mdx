L’écran `reset-password-mfa-phone-challenge` s’affiche pendant le flux de réinitialisation du mot de passe lorsqu’une authentification multifacteur par téléphone est requise. Il invite l’utilisateur à choisir un mode de livraison (SMS ou appel vocal) pour recevoir un code de vérification et poursuivre le processus de réinitialisation.

<div id="import">
  ## Import
</div>

Chaque écran possède son propre ensemble de hooks et de méthodes. Le SDK prend en charge les **importations partielles** et les **importations à la racine** pour chaque écran.

* Une importation partielle vous permet d’inclure uniquement le code nécessaire à votre cas d’utilisation.
* Une importation à la racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez une compilation unifiée capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useResetPasswordMfaPhoneChallenge } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useResetPasswordMfaPhoneChallenge,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  continueMethod,
  tryAnotherMethod,
} from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';

function ResetPasswordMfaPhoneChallengeScreen() {
  const { continueMethod } = useResetPasswordMfaPhoneChallenge();
  return (
    <button onClick={() => continueMethod({ type: 'sms' })}>
      Send Code via SMS
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte d’Auth0 sur l’écran `reset-password-mfa-phone-challenge`. Importez-les depuis `@auth0/auth0-acul-react/reset-password-mfa-phone-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les configurations de branding, comme le logo, les couleurs et les paramètres de thème affichés sur l’écran `reset-password-mfa-phone-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit les configurations liées au client, comme `id`, `name` et `logoUrl`, pour l’écran `reset-password-mfa-phone-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organization de l’utilisateur si la réinitialisation du mot de passe est limitée à une Organization. Il retourne `null` lorsqu’aucun contexte d’Organization n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No Organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur le prompt actuel dans le flux d’authentification.


<div id="methods">
  ## Méthodes
</div>

<ParamField body="continueMethod" type="Promise<void>">
  Cette méthode soumet le mode de livraison choisi par l’utilisateur afin d’envoyer le code de vérification et de poursuivre l’étape MFA dans le flux de réinitialisation du mot de passe.

  ```jsx Example
  import { useResetPasswordMfaPhoneChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';

  function SendCodeForm() {
    const { continueMethod } = useResetPasswordMfaPhoneChallenge();
    return (
      <button onClick={() => continueMethod({ type: 'sms' })}>
        Send Code via SMS
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [ResetPasswordMfaPhoneChallengeContinueOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaPhoneChallengeContinueOptions).
    </ParamField>

    <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">
      Le mode de livraison sélectionné par l’utilisateur pour recevoir le code de vérification.

      * `sms` : Envoyer le code par message texte.
      * `voice` : Envoyer le code par appel vocal.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Cette méthode redirige l’utilisateur pour qu’il choisisse une autre méthode MFA.

  ```jsx Example
  import { useResetPasswordMfaPhoneChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-phone-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useResetPasswordMfaPhoneChallenge();
    return (
      <button onClick={() => tryAnotherMethod({ type: 'sms' })}>
        Try Another Method
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaPhoneChallengeTryAnotherMethodOptions).
    </ParamField>

    <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;">
      Le mode de livraison qui a d’abord été présenté ou sélectionné sur l’écran actuel.
      Ce paramètre est requis par le point de terminaison d’API pour traiter correctement l’action &#39;pick-authenticator&#39;.

      * `sms` : Indique que l’écran actuel correspondait au défi SMS.
      * `voice` : Indique que l’écran actuel correspondait au défi vocal.
    </ParamField>
  </Expandable>
</ParamField>
