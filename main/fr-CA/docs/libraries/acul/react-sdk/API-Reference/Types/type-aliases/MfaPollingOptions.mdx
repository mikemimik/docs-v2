title: "Alias de type : MfaPollingOptions"
---

```ts Example
type MfaPollingOptions = {
  intervalMs?: number;
  onCompleted?: () => void;
  onError?: (error: Error) => void;
}
```

<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer le processus d’interrogation d’un défi push MFA.

  Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
  à laquelle l’endpoint du défi est interrogé et la façon dont les réussites et les échecs sont gérés.

  <Expandable title="properties">
    <ParamField body="intervalMs" type="number">
      Intervalle facultatif, en **millisecondes**, entre des requêtes d’interrogation consécutives.

      * S’il est omis, l’intervalle par défaut interne du SDK est utilisé (habituellement 5000 ms).
      * Choisissez une valeur qui offre un bon équilibre entre la réactivité et les contraintes liées aux limites de débit.

      ```ts Default Value
      Propre à l’implémentation (généralement 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted" type="() => void">
      Fonction de rappel facultative exécutée une fois que le défi push MFA est approuvé
      avec succès et que l’interrogation est terminée.

      * Appelée exactement **une seule fois**, après quoi l’interrogation s’arrête automatiquement.
      * Utile pour faire progresser le flux de connexion (par exemple, appeler `continue()`).
    </ParamField>

    <ParamField body="onError" type="(error: Error) => void">
      Fonction de rappel facultative invoquée si une erreur se produit pendant l’interrogation.

      Reçoit un objet [ULError](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) contenant :

      * `status` — le code de statut HTTP de la requête échouée

      * `responseText` — le corps brut de la réponse, s’il est disponible

      * Appelée une fois par événement d’erreur; l’interrogation peut être relancée selon le type d’erreur.

      * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.

      **Propriétés**

      <ParamField body="error" type={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error">Error</a>}>
        Un objet d’erreur contenant le code de statut HTTP et le corps brut de la réponse de la requête d’interrogation échouée.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>
