La classe MfaPushChallengePush implémente la fonctionnalité de l’écran `mfa-push-challenge-push`. Cet écran s’affiche lorsqu’un utilisateur doit confirmer une notification push dans le cadre de l’authentification MFA.

<ParamField body="pollingManager" type={<span><a href="/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode lance et gère le polling pour un défi MFA push.

  Elle crée une session de polling qui vérifie de façon répétée le endpoint du défi MFA push à l’intervalle indiqué jusqu’à ce que le défi soit approuvé ou qu’une erreur survienne. Lorsque la condition d’approbation est remplie, le callback [onCompleted](/fr-CA/docs/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) fourni est appelé et le polling s’arrête automatiquement.

  Utilisez le [MfaPushPollingControl](/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) retourné pour démarrer, arrêter ou vérifier le statut du processus de polling à tout moment.

  Retourne une instance [MfaPushPollingControl](/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) qui expose :

  * `startPolling()` — Démarre ou reprend le polling.
  * `stopPolling()` — Annule immédiatement le polling.
  * `isRunning()` — Indique si le polling est actif.

  ```typescript Example
  import MfaPushChallengePush from '@auth0/auth0-acul-js/mfa-push-challenge-push';

  const mfaPushChallengePush = new MfaPushChallengePush();

  // Démarre le polling toutes les 5 secondes jusqu’à ce que le défi push soit approuvé
  const control = mfaPushChallengePush.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushChallengePush.continue(),
    onError: (error) => console.error('Erreur de polling:', error),
  });

  // Plus tard, annule le polling au besoin
  control.stopPolling();
  ```

  **Remarques**

  * Les réponses HTTP 429 (rate limit) sont gérées en interne : le polling attend automatiquement que la limite soit réinitialisée avant de réessayer.
  * Vous pouvez appeler `startPolling()` plusieurs fois en toute sécurité ; cela n’a aucun effet si le polling est déjà en cours.

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/fr-CA/docs/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalle optionnel, en millisecondes, entre des requests de polling consécutives. S’il est omis, l’intervalle par défaut interne du SDK est utilisé (généralement 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Callback optionnel exécuté une fois que le défi MFA push a été approuvé avec succès et que le polling est terminé. Il est appelé une seule fois, après quoi le polling s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Callback optionnel appelé si une erreur survient pendant le polling. Reçoit un objet `ULError` contenant `status` (code de statut HTTP) et `responseText` (corps brut de la réponse).
    </ParamField>
  </Expandable>
</ParamField>
