<div id="constructors">
  ## Constructeurs
</div>

Créez une instance du gestionnaire de l’écran de code QR d’inscription à la MFA push :

```typescript Example
import MfaPushEnrollmentQr from '@auth0/auth0-acul-js/mfa-push-enrollment-qr';

const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();
await mfaPushEnrollmentQr.pickAuthenticator();
```

  const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();

    language: 'fr',
  });

  **Paramètres de la méthode**

<ParamField body="pollingManager" type={<span><a href="/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode lance et gère le processus de polling pour l’inscription à MFA push.

  Elle crée une session de polling qui vérifie de façon répétée le endpoint d’inscription à MFA push à l’intervalle indiqué, jusqu’à ce que l’inscription soit approuvée ou qu’une erreur survienne. Lorsque la condition d’approbation est remplie, le callback [onCompleted](/fr-CA/docs/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) fourni est appelé et le polling s’arrête automatiquement.

  Utilisez le [MfaPushPollingControl](/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) retourné pour démarrer, arrêter ou vérifier le statut du processus de polling en tout temps.

  Retourne une instance de [MfaPushPollingControl](/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) qui expose :

  * `startPolling()` — Démarre ou reprend le polling.
  * `stopPolling()` — Annule le polling immédiatement.
  * `isRunning()` — Indique si le polling est actuellement actif.

  ```typescript Example
  import MfaPushEnrollmentQr from '@auth0/auth0-acul-js/mfa-push-enrollment-qr';

  const mfaPushEnrollmentQr = new MfaPushEnrollmentQr();

  // Démarrer le polling toutes les 5 secondes jusqu’à ce que l’inscription push soit approuvée
  const control = mfaPushEnrollmentQr.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushEnrollmentQr.continue(),
    onError: (error) => console.error('Polling error:', error),
  });

  // Plus tard, annuler le polling au besoin
  control.stopPolling();
  ```

  **Remarques**

  * Les réponses HTTP 429 (limite de débit) sont gérées à l’interne : le polling attend automatiquement que la limite de débit soit réinitialisée avant de réessayer.
  * Vous pouvez appeler `startPolling()` plusieurs fois sans risque ; cela n’a aucun effet si le polling est déjà en cours.

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/fr-CA/docs/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en millisecondes, entre deux requests de polling consécutives. S’il est omis, l’intervalle par défaut interne du SDK est utilisé (habituellement 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Callback facultatif exécuté lorsque l’inscription à MFA push est approuvée avec succès et que le polling est terminé. Il est appelé exactement une fois, après quoi le polling s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Callback facultatif appelé si une erreur survient pendant le polling. Reçoit un objet `ULError` contenant `status` (code de statut HTTP) et `responseText` (corps brut de la réponse).
    </ParamField>
  </Expandable>
</ParamField>
