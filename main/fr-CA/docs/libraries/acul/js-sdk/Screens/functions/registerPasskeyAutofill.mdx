---
title: "registerPasskeyAutofill"
---

> **registerPasskeyAutofill**(`inputId`?: `string`): `Promise`&lt;`void`&gt;

Initialise une requête passive d’identifiants WebAuthn à l’aide de `navigator.credentials.get()` avec `mediation: "conditional"`. Lorsqu’elle est prise en charge, cette fonctionnalité permet au navigateur d’afficher les clés d’accès enregistrées directement dans la liste déroulante de saisie semi-automatique du champ de nom d’utilisateur. La sélection d’une clé d’accès complète l’authentification sans nécessiter d’interaction supplémentaire de la part de l’utilisateur.

Appelez cette méthode une seule fois lors de l’initialisation de l’écran de connexion (par exemple, au chargement de la page). Si la médiation conditionnelle n’est pas prise en charge par le navigateur, la méthode n’effectue simplement aucune opération.

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="inputId?" type="string">
  L’ID de l’élément `<input>` du nom d’utilisateur (sans le `#`). Exemple : `"username"`. Lorsqu’il est fourni, le SDK valide que l’élément existe, confirme qu’il s’agit bien d’un `<input>` et remplace son attribut `autocomplete` par `"username webauthn"`.

  S’il est omis, vous devez configurer manuellement l’élément `<input>` :

  ```html
  <input id="username" autocomplete="username webauthn" />
  ```
</ParamField>

<div id="returns">
  ## Renvoie
</div>

`Promise`&lt;`void`&gt;

Se résout lorsque l’enregistrement conditionnel de l’interface utilisateur est terminé.

<div id="example">
  ## Exemple
</div>

```typescript
import LoginId from '@auth0/auth0-acul-js/login-id';

async function initializeLogin() {
  const loginId = new LoginId();
  // Assurez-vous que l'élément HTML input associé existe :
  // <input id="username" autocomplete="username webauthn" />
  await loginId.registerPasskeyAutofill('username');
}

initializeLogin().catch(console.error);
```

<div id="gotchas">
  ## Points à surveiller
</div>

* L’attribut `autocomplete` **doit contenir exactement** `"username webauthn"`. Si vous incluez des tokens non pertinents comme `"email"` ou `"text"`, les navigateurs n’afficheront pas le menu déroulant des clés d’accès.
* Le remplacement de l’attribut est intentionnel et nécessaire pour garantir un comportement uniforme d’un navigateur à l’autre. Ne comptez pas sur la fusion ou l’ajout aux valeurs `autocomplete` existantes.
* Si la médiation conditionnelle n’est pas prise en charge par le navigateur, le SDK n’effectuera simplement aucune opération.

<div id="remarks">
  ## Remarques
</div>

Cette méthode fait appel à l’utilitaire interne `registerPasskeyAutofill()` et retourne un `AbortController` en arrière-plan pour gérer le cycle de vie de la requête. Elle ne doit être appelée qu’une seule fois par cycle de vie de la page.
