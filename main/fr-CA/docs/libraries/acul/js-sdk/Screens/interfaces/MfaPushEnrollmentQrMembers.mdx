    <ParamField body="options" type={<span><a href="/docs/fr-CA/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions">MfaPollingOptions</a></span>}>
      <ParamField body="MfaPollingOptions" type="object">
        Options de configuration pour gérer un processus d’interrogation d’une notification push MFA.

        Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
        à laquelle le endpoint de défi est interrogé et la manière dont le succès ou l’échec est géré.

        <Expandable title="Propriétés">
          <ParamField body="intervalMs?" type="number">
            Intervalle facultatif, en **millisecondes**, entre deux requêtes d’interrogation consécutives.

            * S’il est omis, l’intervalle par défaut interne du SDK est utilisé (généralement 5000 ms).
            * Choisissez une valeur qui offre un bon équilibre entre réactivité et limites de débit.
          </ParamField>

          <ParamField body="onCompleted()?" type="() => void">
            Callback facultatif exécuté une fois le défi push MFA approuvé avec succès
            et l’interrogation terminée.

            * Appelé exactement **une** fois, après quoi l’interrogation s’arrête automatiquement.
            * Utile pour faire avancer le flux de connexion (p. ex., appeler `continue()`).
          </ParamField>

          <ParamField body="onError()?" type="(error) => void">
            Callback facultatif invoqué si une erreur survient pendant l’interrogation.

            Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/js-sdk/Screens/interfaces/Error) contenant :

            * `status` — le code d’état HTTP de la requête ayant échoué

            * `responseText` — le corps de la réponse brut, s’il est disponible

            * Appelé une fois par événement d’erreur; l’interrogation peut être relancée selon le type d’erreur.

            * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
