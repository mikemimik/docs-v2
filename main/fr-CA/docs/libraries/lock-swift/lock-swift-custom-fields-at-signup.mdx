---

**Lock v2 for iOS** vous permet de définir des champs supplémentaires que l’utilisateur doit remplir avant de créer un nouveau compte. Ces champs s’affichent après les champs de base (courriel, nom d’utilisateur, mot de passe).

<div id="adding-custom-fields">
  ## Ajout de champs personnalisés
</div>

Lors de l’inscription, les renseignements requis par défaut sont l’adresse courriel et le mot de passe de l’utilisateur. Vous pouvez élargir la collecte de données selon vos besoins. Les champs supplémentaires recueillis ici lors de l’inscription seront stockés dans le `user_metadata`. Pour en savoir plus, consultez [Métadonnées](/fr-CA/docs/manage-users/user-accounts/metadata).

```swift lines
.withOptions {
  $0.customSignupFields = [
    CustomTextField(name: "first_name", placeholder: "First Name", icon: LazyImage(name: "ic_person", bundle: Lock.bundle)),
    CustomTextField(name: "last_name", placeholder: "Last Name", icon: LazyImage(name: "ic_person", bundle: Lock.bundle))
  ]
}
```

Vous devez préciser l’icône à utiliser avec votre champ de texte personnalisé.

C&#39;est tout ! Si vous avez activé l’inscription des utilisateurs dans le Dashboard de l’application, après avoir rempli les champs de base (courriel/username, mot de passe) et cliqué sur Submit, ils seront invités à remplir les champs restants. L’utilisateur doit remplir tous les champs personnalisés avant de pouvoir terminer son inscription.

Lors d’une requête d’inscription ou de connexion d’un utilisateur, les champs supplémentaires seront ajoutés à l’attribut `user_metadata` dans le corps de la requête. Vous pouvez y accéder en consultant le profil de l’utilisateur à tout moment, même à partir du Dashboard dans la section Users.
