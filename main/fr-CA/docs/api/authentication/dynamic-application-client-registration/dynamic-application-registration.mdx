import { ResponseSchema } from '/snippets/ResponseSchema.jsx';

<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oidc/register`

En fournissant un nom et l’URL de rappel requise, vous pouvez enregistrer dynamiquement un client auprès d’Auth0. Aucun jeton n’est requis pour cette requête.

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_name" type="string">
  Le nom du client dynamique à créer.
</ParamField>

<ParamField body="grant_types" type="array">
  Liste des types d’autorisation que le client peut utiliser. S’ils ne sont pas fournis, le type d’autorisation `authorization_code` sera appliqué par défaut aux <a href="https://auth0.com/docs/get-started/applications/third-party-applications">applications tierces</a> avec des contrôles de sécurité renforcés.
</ParamField>

<ParamField body="redirect_uris" type="array" required>
  Tableau d’URL qu’Auth0 considérera comme valides pour la redirection à la fin d’un flux d’authentification.
</ParamField>

<ParamField body="token_endpoint_auth_method" type="string">
  Méthode d’authentification utilisée au point de terminaison des jetons.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="client_name" type="string">
    Le nom du client nouvellement enregistré.
  </ResponseField>

  <ResponseField name="client_id" type="string">
    L’identifiant unique du client.
  </ResponseField>

  <ResponseField name="client_secret" type="string">
    Le Client Secret.
  </ResponseField>

  <ResponseField name="client_secret_expires_at" type="integer">
    L’heure d’expiration du Client Secret.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                      |
| ------ | ------------------------------------------------ |
| 200    | Réponse réussie contenant les détails du client. |
