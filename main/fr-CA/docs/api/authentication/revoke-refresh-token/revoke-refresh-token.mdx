<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/revoke`

Utilisez ce point de terminaison pour invalider un Refresh Token s’il a été compromis.

Le comportement de ce point de terminaison dépend de l’état de la bascule [La révocation du Refresh Token supprime l’octroi](https://auth0.com/docs/secure/tokens/refresh-tokens/revoke-refresh-tokens#choose-whether-token-revocation-deletes-grants).
Si cette bascule est activée, chaque request de révocation invalide non seulement le token visé, mais aussi tous les autres tokens associés au même octroi d’autorisation. Cela signifie que **tous les Refresh Tokens émis pour le même utilisateur, la même application et la même audience seront révoqués**.
Si cette bascule est désactivée, seul le refresh token est révoqué, tandis que l’octroi demeure intact.

<div id="remarks">
  ## Remarques
</div>

* Pour les applications non confidentielles qui ne peuvent pas protéger le Client Secret (par exemple, les applications natives), le point de terminaison permet de ne pas transmettre de Client Secret, mais l’application elle-même doit avoir la propriété `tokenEndpointAuthMethod` définie sur `none`. Vous pouvez le faire soit dans l’interface ([Dashboard &gt; Applications &gt; Application Settings](https://manage.auth0.com/dashboard)), soit à l’aide de la [Management API](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id).

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre application.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT contenant une assertion signée avec les identifiants de votre application. Obligatoire lorsque Private Key JWT est la méthode d’authentification de l’application.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  La valeur est `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Obligatoire lorsque Private Key JWT est la méthode d’authentification de l’application.
</ParamField>

<ParamField body="client_secret" type="string">
  Le `client_secret` de votre application. Obligatoire lorsque Client Secret Basic ou Client Secret Post est la méthode d’authentification de l’application. Plus précisément, obligatoire pour les Regular Web Applications **uniquement**.
</ParamField>

<ParamField body="token" type="string" required>
  Le Refresh Token que vous souhaitez révoquer.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  Une réponse 200 sans erreur ne contient pas de corps. Les réponses d’erreur suivent ce format :

  <ResponseField name="error" type="string">
    Code d’erreur.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Description de l’erreur.
  </ResponseField>
</ResponseSchema>
