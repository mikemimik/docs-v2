<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/ro`

À partir des identifiants de l’utilisateur, ce point de terminaison authentifie l’utilisateur auprès du fournisseur et renvoie un objet JSON contenant l’Access Token et un ID Token.

<Note>
  Ce point de terminaison fait partie de l’ancien pipeline d’authentification et a été remplacé par le [Password Grant](#resource-owner-password). Pour en savoir plus sur le pipeline d’authentification le plus récent, consultez [Introducing OIDC Conformant Authentication](/fr-CA/api-auth/intro).
</Note>

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison fonctionne uniquement pour les connexions de base de données, les connexions Passwordless, Active Directory/LDAP, Windows Azure AD et ADFS.
* La portée `profile` demande l&#39;accès aux claims du profil par défaut de l&#39;utilisateur final, notamment `name`, `family_name`, `given_name` et d&#39;autres.
* La portée `email` demande l&#39;accès aux claims `email` et `email_verified`.
