---
description: Correcciones que debes hacer en tu aplicación de Auth0 antes de implementarla en producción
title: 'Verificaciones de preparación para producción: correcciones críticas'
---

Las siguientes verificaciones comprueban si has completado todos los requisitos necesarios para implementar correctamente en producción.

Para obtener más información sobre el resultado de estas verificaciones, consulta [Run Production Checks](/docs/es/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Verificación                                                                                                                                                                                                     | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Todas las Actions se ejecutan con una [versión recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)                                                       | Configura todas las [Actions](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) para que usen una versión recomendada de NodeJS y así aprovechar los parches de seguridad y las nuevas funciones.                                                                                                                                                                                                                                                                                            |
| [Permitir tokens de ID para la autenticación de Management API v2](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) está desactivado                              | La capacidad de usar ID Tokens para autorizar algunos de los endpoints de Users y Device Credentials de la Management API está quedando obsoleta. Después de completar la migración a tokens de acceso, asegúrate de que el interruptor [Allow ID Tokens for Management API v2 Authentication](https://manage.auth0.com/#/tenant/advanced) esté desactivado. Si no puedes ver esta configuración, tu tenant se creó después de que esta funcionalidad quedara obsoleta, por lo que ya está desactivada de forma predeterminada. |
| allowLegacyRoGrantTypes                                                                                                                                                                                          | El [endpoint oauth/ro heredado](/docs/es/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) en la configuración de migración del tenant está en desuso. Después de actualizar tus aplicaciones para que usen endpoints compatibles, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                               |
| allowOtherLegacyGrantTypes                                                                                                                                                                                       | El [endpoint heredado oauth/access&#95;token](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) de la configuración de migración del tenant está quedando obsoleto. Después de actualizar tus aplicaciones para usar endpoints compatibles, asegúrate de desactivar este endpoint.                                                                                                                                                                                                     |
| allowLegacyTokenInfoEndpoint                                                                                                                                                                                     | El [endpoint tokeninfo](/docs/es/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) heredado de la configuración de migración del tenant está quedando obsoleto. Después de actualizar tus aplicaciones para usar endpoints compatibles, asegúrate de desactivar este endpoint.                                                                                                                                                                                                                         |
| allowLegacyDelegationGrantTypes                                                                                                                                                                                  | El [endpoint de delegación](/docs/es/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation) heredado de la configuración de migración del tenant está quedando obsoleto. Después de actualizar sus aplicaciones para que usen endpoints compatibles, asegúrese de desactivar este endpoint.                                                                                                                                                                                                                   |
| [URLs de callback permitidas](/docs/es/authenticate/login/redirect-users-after-login) no deben incluir localhost                                                                                                    | Valida que las [URL de callback permitidas de la aplicación](https://manage.auth0.com/#/applications) no apunten a `localhost`, `127.0.0.1`, entre otros.                                                                                                                                                                                                                                                                                                                                                                       |
| [Orígenes permitidos (CORS)](/docs/es/authenticate/login/cross-origin-authentication) no deben ser Localhost                                                                                                        | Valida que la [URL de la ubicación](https://manage.auth0.com/#/applications) de la página no apunte a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                              |
| [Orígenes web permitidos](/docs/es/get-started/applications/application-settings) no deben incluir `localhost`                                                                                                      | Valida que las [URL de orígenes web permitidos](https://manage.auth0.com/#/applications) no apunten a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                              |
| [Proveedor de correo electrónico](/docs/es/customize/email/smtp-email-providers) debe estar configurado                                                                                                             | Verifica que se haya configurado el [proveedor de correo electrónico personalizado](https://manage.auth0.com/#/emails/provider).                                                                                                                                                                                                                                                                                                                                                                                                |
| Los Hooks se están dejando de usar y deben [migrarse a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions)                                                                                   | Los Hooks están en desuso. Le recomendamos encarecidamente que comience a usar [Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions) e inicie el proceso de migración. Los Hooks existentes seguirán activos hasta el 18 de noviembre de 2024.                                                                                                                                                                                                                                                               |
| La [autenticación multifactor](/docs/es/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) debe estar configurada (Dependencia: la MFA está configurada) | Si utiliza MFA con el factor [Phone Message](https://manage.auth0.com/#/mfa), configúrelo con Twilio o con un proveedor de entrega personalizado para evitar los límites de tasa del proveedor de entrega predeterminado.                                                                                                                                                                                                                                                                                                       |
| [Legacy User Profile](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) debe estar desactivado                                                                  | Se van a retirar los flujos de autenticación heredados que permiten que los ID Tokens y el endpoint `/userinfo` incluyan el perfil completo del usuario. Después de completar la migración a las nuevas API conformes con OIDC, asegúrate de que el interruptor [Legacy User Profile](https://manage.auth0.com/#/account/advanced) esté desactivado.                                                                                                                                                                            |
| Las Rules están en desuso y deben [migrarse a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions)                                                                                            | Rules están en proceso de eliminación. Recomendamos encarecidamente que comience a usar [Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) e inicie el proceso de migración. Las Rules existentes seguirán activas hasta el 18 de nov. de 2024.                                                                                                                                                                                                                                                           |
| [Conexiones sociales](/docs/es/authenticate/identity-providers/social-identity-providers/devkeys) no deben usar las claves de desarrollador de Auth0                                                                | Verifica que [Social Connections](https://manage.auth0.com/#/connections/social) no utilicen las claves de desarrollador predeterminadas de Auth0.                                                                                                                                                                                                                                                                                                                                                                              |
| Se debe configurar el correo electrónico de soporte                                                                                                                                                              | Garantiza que el [correo electrónico de soporte](https://manage.auth0.com/#/account) esté configurado en la configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                                           |
| La URL de soporte debe estar configurada                                                                                                                                                                         | Asegura que la [URL de soporte](https://manage.auth0.com/#/account) esté configurada en la configuración del tenant.                                                                                                                                                                                                                                                                                                                                                                                                            |
| Se debe configurar la etiqueta de entorno del tenant                                                                                                                                                             | Garantiza que la [etiqueta de entorno del tenant](https://support.auth0.com/tenants/public) esté configurada correctamente como Production, Staging o Development. Los cambios en esta etiqueta afectan el límite de solicitudes de tu tenant.                                                                                                                                                                                                                                                                                  |
| El tenant está configurado para usar una [versión predeterminada recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22)                                      | Actualice su tenant a una [versión recomendada de NodeJS](/docs/es/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) para beneficiarse de los parches de seguridad y las nuevas funciones.                                                                                                                                                                                                                                                                                                            |
| URI de inicio de sesión del tenant                                                                                                                                                                               | Asegúrate de que [Tenant Login URI](https://manage.auth0.com/#/tenant/advanced) esté configurado en Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                            |
| Usar un dominio personalizado en plantillas de correo electrónico de marca                                                                                                                                       | Asegúrese de que sus plantillas de correo electrónico usen un [dominio personalizado](/docs/es/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails). Recomendamos encarecidamente personalizar todos los correos electrónicos orientados al usuario para que usen su dominio personalizado y la identidad de marca de su empresa.                                                                                                                                                       |
