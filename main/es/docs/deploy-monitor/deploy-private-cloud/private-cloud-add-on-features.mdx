La opción de implementación de Private Cloud incluye dos funciones adicionales: Geo-failover y cumplimiento de PCI. Si tiene alguna pregunta sobre estas funciones, [póngase en contacto con el equipo de ventas de Auth0](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales).

<div id="private-cloud-geo-failover">
  ## Geo-failover de Private Cloud
</div>

La Geo-failover es una configuración activo-pasivo que permite a los clientes especificar una región secundaria de conmutación por error para usar en caso de una caída regional importante. En general, cualquier región del mismo proveedor de nube con una latencia inferior a 100 ms puede seleccionarse como región de conmutación por error. Sin embargo, en algunos casos, los proveedores de nube prefieren regiones de conmutación por error específicas. El equipo de incorporación de Auth0 trabajará con usted para seleccionar las mejores regiones para su caso de uso.

Además, se requiere una tercera región de &quot;desempate&quot; para uno de nuestros almacenes de datos. Esta tercera región nunca recibe tráfico real, pero es necesaria para designar a un nuevo líder en caso de una conmutación por error.

Si se detecta una falla regional, Auth0 activa la Geo-failover en nombre del cliente. Todo el proceso está automatizado una vez activado. Cuando se recibe la confirmación de que la región ya no presenta problemas, Auth0 revertirá la conmutación por error a la región primaria.

<div id="pci-compliance">
  ## Cumplimiento de PCI
</div>

El cumplimiento de PCI está disponible como complemento para los clientes que lo necesiten.
