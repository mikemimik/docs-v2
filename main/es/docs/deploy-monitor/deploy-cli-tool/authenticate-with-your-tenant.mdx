Hay tres métodos compatibles para autenticar Auth0 Deploy CLI con tu tenant.

<div id="client-credentials">
  ## Credenciales de cliente
</div>

La autenticación de [credenciales de cliente](/es/docs/get-started/authentication-and-authorization-flow/client-credentials-flow) requiere que proporciones directamente a la aplicación un par de <Tooltip tip="Client ID: Valor de identificación que Auth0 asigna a tu recurso registrado." cta="Ver glosario" href="/es/docs/glossary?term=client+ID">Client ID</Tooltip> y <Tooltip tip="Client ID: Valor de identificación que Auth0 asigna a tu recurso registrado." cta="Ver glosario" href="/es/docs/glossary?term=client+secret">secreto del cliente</Tooltip>. Esta opción es sencilla y ofrece la forma más rápida de configurar la herramienta.

Para configurar la autenticación con credenciales de cliente, establece los siguientes campos en tu [archivo de configuración](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                 | Valor                                          |
| --------------------- | ---------------------------------------------- |
| `AUTH0_CLIENT_ID`     | Client ID de tu aplicación de Auth0.           |
| `AUTH0_CLIENT_SECRET` | Secreto del cliente de tu aplicación de Auth0. |

<div id="private-key-jwt">
  ## clave privada JWT
</div>

La autenticación con [clave privada JWT](/es/docs/get-started/applications/configure-private-key-jwt) requiere que configures una clave privada para la aplicación y una clave pública para el tenant remoto. Esta opción es ideal si no quieres almacenar credenciales en Auth0.

Para configurar la autenticación con clave privada <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=JWT">JWT</Tooltip>, establece los siguientes campos en tu [archivo de configuración](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                            | Valor                                                           |
| -------------------------------- | --------------------------------------------------------------- |
| `AUTH0_CLIENT_SIGNING_KEY_PATH`  | Ruta relativa al certificado de la clave de firma del JWT.      |
| `AUTH0_CLIENT_SIGNING_ALGORITHM` | Opcional. Algoritmo de firma JWT utilizado para el certificado. |

<div id="access-token">
  ## Token de acceso
</div>

La [autenticación con token de acceso](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) requiere que proporciones directamente a la aplicación un <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> de Auth0. Esta opción implica una mayor responsabilidad para los desarrolladores, pero puede habilitar flujos de trabajo flexibles y específicos cuando sea necesario.

Para configurar la autenticación con token de acceso, pasa un token de acceso de Auth0 para Management API mediante la [variable de entorno](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) `AUTH0_ACCESS_TOKEN`.
