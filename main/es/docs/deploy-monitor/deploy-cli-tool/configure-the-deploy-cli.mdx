---
description: Aprende a configurar Auth0 Deploy CLI para adaptar la herramienta a tus necesidades.
title: Configurar Deploy CLI
---

Configurar Deploy CLI es fundamental para establecer las credenciales de Auth0 y, en general, ajustar el comportamiento de la herramienta a tus necesidades específicas. Hay dos maneras de configurar Deploy CLI:

1. Usa el archivo de configuración (`config.json`).
2. Configura variables de entorno.

<div id="configuration-file">
  ## Archivo de configuración
</div>

Se puede usar un archivo JSON independiente para configurar Deploy CLI. Por lo general, este archivo se encuentra en el directorio raíz de tu proyecto y debe llamarse `config.json`.

### Ejemplo

```json lines
{
  "AUTH0_DOMAIN": "<YOUR_TENANT_DOMAIN>",
  "AUTH0_CLIENT_ID": "{yourClientId}",
  "AUTH0_ALLOW_DELETE": false
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No se recomienda incluir las credenciales directamente en el código, ya que existe el riesgo de que se filtren secretos si este archivo llega a enviarse a un sistema público de control de versiones. Para mitigar este riesgo, como práctica recomendada, pasa las credenciales como [variables de entorno](#environment-variables).
</Callout>

<div id="environment-variables">
  ## Variables de entorno
</div>

De forma predeterminada, Deploy CLI usa variables de entorno, lo que permite pasar credenciales y otras configuraciones a la herramienta sin necesidad de publicarlas en el archivo `config.json`. Las variables de entorno pueden usarse para complementar el archivo `config.json` o sustituirlo por completo, según las necesidades del proyecto.

Los valores de configuración no primitivos, como `AUTH0_KEYWORD_REPLACE_MAPPINGS` y `AUTH0_EXCLUDED`, también pueden pasarse mediante variables de entorno, siempre que estos valores estén correctamente serializados como JSON.

Para deshabilitar el uso de variables de entorno en los comandos `import` o `export`, pasa el argumento `--env=false`.

<div id="examples">
  ### Ejemplos
</div>

<div id="deploying-configuration-for-yaml-formats-without-a-configjson-file">
  #### Despliegue de la configuración en formatos YAML sin un archivo config.json
</div>

```bash lines
$ export AUTH0_DOMAIN={yourDomain}
$ export AUTH0_CLIENT_ID={yourClientId}
$ export AUTH0_CLIENT_SECRET={yourClientSecret}
$ a0deploy import --input_file=local/tenant.yaml
```

<div id="disable-environment-variable-ingestion">
  #### Desactivar la ingesta de variables de entorno
</div>

```bash wrap lines
$ a0deploy export -c=config.json --format=yaml --output_folder=local --env=false
```

<div id="non-primitive-configuration-values">
  #### Valores no primitivos de configuración
</div>

```bash lines
$ export AUTH0_EXCLUDED='["actions","organizations"]'
$ export AUTH0_KEYWORD_REPLACE_MAPPINGS='{"ENVIRONMENT":"dev"}'
$ a0deploy export -c=config.json --format=yaml --output_folder=local
```

<div id="available-configuration-properties">
  ## Propiedades de configuración disponibles
</div>

<div id="auth0_domain">
  ### AUTH0_DOMAIN
</div>

String. El dominio del tenant de Auth0 de destino.

<div id="auth0_client_id">
  ### AUTH0_CLIENT_ID
</div>

String. El ID de la aplicación designada de Auth0 que se utiliza para hacer solicitudes a la API.

<div id="auth0_client_secret">
  ### AUTH0_CLIENT_SECRET
</div>

String. El secreto de la aplicación de Auth0 designada que se utiliza para hacer solicitudes a la API.

<div id="auth0_access_token">
  ### AUTH0_ACCESS_TOKEN
</div>

String. <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> de corta duración para <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de la aplicación de Auth0 especificada. Puede usarse en lugar de la combinación de <Tooltip tip="ID de cliente: valor de identificación dado a su recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=client+ID">ID de cliente</Tooltip> y <Tooltip tip="ID de cliente: valor de identificación dado a su recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=client+secret">secreto de cliente</Tooltip>.

<div id="auth0_client_signing_key_path">
  ### AUTH0_CLIENT_SIGNING_KEY_PATH
</div>

String. La ruta de la clave privada que utiliza el cliente para la autenticación <Tooltip tip="JSON Web Token (JWT): Formato estándar de token de ID (y, a menudo, de token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=JWT">JWT</Tooltip> con clave privada. Ruta relativa al directorio de trabajo. Tenga en cuenta también `AUTH0_CLIENT_SIGNING_ALGORITHM` para especificar el <Tooltip tip="Signing Algorithm: Algoritmo utilizado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/es/docs/glossary?term=signing+algorithm">algoritmo de firma</Tooltip>.

<div id="auth0_client_signing_algorithm">
  ### AUTH0_CLIENT_SIGNING_ALGORITHM
</div>

String. Especifica los algoritmos de firma de JWT que utiliza el cliente para la autenticación de clave privada JWT. Solo se usa en combinación con `AUTH0_CLIENT_SIGNING_KEY_PATH`.

Valores aceptados: `RS256`, `RS384`, `PS256`.

<div id="auth0_allow_delete">
  ### AUTH0_ALLOW_DELETE
</div>

Booleano. Cuando está habilitado, permite que la herramienta elimine recursos. Valor predeterminado: `false`.

<div id="auth0_excluded">
  ### AUTH0_EXCLUDED
</div>

Array de strings. Excluye tipos de recursos completos para que no se gestionen de forma bidireccional. Para obtener más información, consulta [Excluir recursos de la gestión](/es/docs/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

No se puede usar simultáneamente con [AUTH0&#95;INCLUDED&#95;ONLY](#auth0_included_only).

Valores posibles: `actions`, `attackProtection`, `branding`, `clientGrants`, `clients`, `connections`, `customDomains`, `databases`, `emailProvider`, `emailTemplates`, `guardianFactorProviders`, `guardianFactorTemplates`, `guardianFactors`, `guardianPhoneFactorMessageTypes`, `guardianPhoneFactorSelectedProvider`, `guardianPolicies`, `logStreams`, `migrations`, `organizations`, `pages`, `prompts`, `resourceServers`, `roles`, `tenant`, `triggers`.

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "AUTH0_EXCLUDED": ["organizations", "connections", "hooks"]
}
```

<div id="auth0_included_only">
  ### AUTH0_INCLUDED_ONLY
</div>

Array de cadenas. Especifica qué tipos de recursos se deben gestionar de forma bidireccional. Para obtener más información, consulta [Excluir recursos de la gestión](/es/docs/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

No se puede usar simultáneamente con [AUTH0&#95;EXCLUDED](#auth0_excluded).

Valores posibles: `actions`, `attackProtection`, `branding`, `clientGrants`, `clients`, `connections`, `customDomains`, `databases`, `emailProvider`, `emailTemplates`, `guardianFactorProviders`, `guardianFactorTemplates`, `guardianFactors`, `guardianPhoneFactorMessageTypes`, `guardianPhoneFactorSelectedProvider`, `guardianPolicies`, `logStreams`, `migrations`, `organizations`, `pages`, `prompts`, `resourceServers`, `roles`, `tenant`, `triggers`.

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "AUTH0_INCLUDED_ONLY": ["clients", "connections", "tenant", "branding"]
}
```

<div id="auth0_keyword_replace_mappings">
  ### AUTH0_KEYWORD_REPLACE_MAPPINGS
</div>

Mapa de palabras clave específicas para facilitar el reemplazo dinámico. Para obtener más información, consulta [Reemplazo de palabras clave](/es/docs/deploy-monitor/deploy-cli-tool/keyword-replacement).

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "ENVIRONMENT": "DEV",
  "ALLOWED_ORIGINS": ["https://dev.test-site.com", "localhost"]
}
```

<div id="auth0_preserve_keywords">
  ### AUTH0_PRESERVE_KEYWORDS
</div>

Boolean. Cuando está habilitado, intenta conservar los marcadores de reemplazo de palabras clave en los archivos de configuración locales durante la exportación. De lo contrario, los valores remotos sobrescribirán esos marcadores de palabras clave colocados manualmente.

Esta configuración requiere la presencia de archivos de configuración locales y de asignaciones de reemplazo de palabras clave definidas mediante la propiedad de configuración `AUTH0_KEYWORD_REPLACE_MAPPINGS`.

Consulte también: [Conservar palabras clave al exportar](/es/docs/deploy-monitor/deploy-cli-tool/keyword-replacement#preserve-keywords-on-export).

<div id="auth0_export_identifiers">
  ### AUTH0_EXPORT_IDENTIFIERS
</div>

Boolean. Cuando está habilitado, devuelve los identificadores de todos los recursos. Puede ser útil en determinados casos de depuración o mantenimiento de registros dentro de un contexto de tenant único. Predeterminado: `false`.

<div id="excluded_props">
  ### EXCLUDED_PROPS
</div>

Permite excluir de la gestión cualquier propiedad no deseada.

<div id="example">
  #### Ejemplo
</div>

```json lines
{
  "connections": ["options.twilio_token"]
}
```

<div id="auth0_audience">
  ### AUTH0_AUDIENCE
</div>

String. Valor independiente del valor de <Tooltip tip="Audience: Identificador único del audience de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/es/docs/glossary?term=audience">audience</Tooltip> al obtener un token de acceso para la Management API. Es útil cuando los endpoints predeterminados de la Management API no están expuestos públicamente.

<div id="auth0_excluded_rules">
  ### AUTH0_EXCLUDED_RULES
</div>

Lista de cadenas. Excluye la gestión de reglas específicas por ID. Nota: Esta configuración podría entrar en obsolescencia en el futuro. Para obtener más información, lee [Excluir recursos de la gestión](/es/docs/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_clients">
  ### AUTH0_EXCLUDED_CLIENTS
</div>

Arreglo de cadenas. Excluye la gestión de clientes específicos por ID. Nota: Esta configuración podría quedar obsoleta en el futuro. Para obtener más información, lee [Excluir recursos de la gestión](/es/docs/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_databases">
  ### AUTH0_EXCLUDED_DATABASES
</div>

Array de strings. Excluye la gestión de bases de datos específicas por ID. Nota: Esta configuración podría quedar sujeta a obsolescencia en el futuro. Para obtener más información, consulta [Excluir recursos de la gestión](/es/docs/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_connections">
  ### AUTH0_EXCLUDED_CONNECTIONS
</div>

Array de strings. Excluye de la gestión conexiones específicas por ID. Nota: esta configuración podría quedar obsoleta en el futuro. Para obtener más información, consulta [Excluir recursos de la gestión](/es/docs/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).

<div id="auth0_excluded_resource_servers">
  ### AUTH0_EXCLUDED_RESOURCE_SERVERS
</div>

Array de cadenas. Excluye de la administración <Tooltip tip="Resource Server: Servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/es/docs/glossary?term=resource+servers">servidores de recursos</Tooltip> específicos por ID. Nota: Esta configuración puede estar sujeta a obsolescencia en el futuro. Para obtener más información, consulta [Excluir recursos de la gestión](/es/docs/deploy-monitor/deploy-cli-tool/exclude-resources-from-management).
