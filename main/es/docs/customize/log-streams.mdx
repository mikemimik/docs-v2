<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los flujos de logs están pensados para fines de monitoreo y análisis. No recomendamos usar flujos de logs en la ruta crítica de tu aplicación ni para tomar decisiones en tiempo real.

  De forma predeterminada, es posible que los logs no se muestren en el orden exacto en que ocurrieron. Si estás transmitiendo logs del tenant y necesitas ordenarlos cronológicamente, te recomendamos implementar la ordenación de logs en el procesamiento de tu flujo.
</Callout>

El servicio de transmisión de logs de Auth0 te permite exportar los logs del tenant a un servicio de análisis de logs. Puedes elegir el tipo y el formato del contenido, y obtener una vista previa del payload desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. La transmisión de logs te permite reaccionar a eventos de log, como cambios de contraseña o nuevos registros, con tu propia lógica de negocio. Con el Auth0 Dashboard y la API, puedes pausar un flujo, ver y actualizar la configuración, y comprobar el estado del flujo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Al habilitar Log Streams, el Cliente pasa a ser el responsable del tratamiento de esa información y es responsable de los datos exportados desde la plataforma Auth0, incluidos qué datos se transmiten y cómo los utiliza el Cliente.
</Callout>

Puedes crear un nuevo flujo de logs en el Auth0 Dashboard con uno de los servicios compatibles, o configurar un flujo hacia un servicio que ya uses mediante un webhook personalizado.

<div id="delivery-attempts">
  ## Intentos de entrega
</div>

Auth0 entrega cada registro a tu servidor a medida que se genera en nuestro sistema. Garantizamos que los logs se entregan al menos una vez.

Cuando se genera un registro:

1. Intentamos entregar el registro a tu servidor hasta tres veces.
2. Si no logramos conectarnos con tu servidor en el tercer intento, registramos un error visible en la vista de **Health** del flujo de logs.
3. Reiniciamos este proceso para cualquier registro con error hasta que se resuelva el problema.

Si no logramos conectarnos con tu servidor durante 7 días consecutivos, pausaremos automáticamente el flujo. Tendrás que resolver el problema y luego reanudar el flujo manualmente.

<div id="log-retention">
  ## Retención de registros
</div>

El período de retención de registros de Auth0 depende de tu [nivel de suscripción](/es/docs/deploy-monitor/logs/log-data-retention).

Si por algún motivo tu servidor no recibe los registros enviados por Auth0, puedes eliminar el flujo y volver a crearlo con el campo Starting Cursor establecido en una fecha y hora concretas dentro del período de retención.

<div id="filtering-and-pii-obfuscation">
  ## Filtrado y ofuscación de PII
</div>

Los clientes pueden elegir qué [categorías de filtro](/es/docs/deploy-monitor/logs/log-event-filters) de sus registros transmitir y cuáles excluir para garantizar que el destino externo reciba solo los datos relevantes. También pueden optar por enmascarar o aplicar hash a la información de identificación personal (PII) en los registros transmitidos.

<div id="pii-obfuscation">
  ### Ofuscación de PII
</div>

Auth0 ofrece dos formas de ofuscar los campos del tenant log que contienen PII al transmitirlos a destinos third-party:

* **Masking:** los valores de los campos se reemplazan con asteriscos
* **xxHash:** un algoritmo hash rápido y no criptográfico

El cliente puede ofuscar los siguientes datos de registro y nombres de campo al transmitirlos:

| Datos        | Nombre de campo                                                             |
| ------------ | --------------------------------------------------------------------------- |
| `first_name` | `first_name`<br />`given_name`<br />`name`                                  |
| `last_name`  | `last_name`<br />`family_name`<br />`name`                                  |
| `email`      | `email`                                                                     |
| `phone`      | `phone`<br />`phone_number`<br />`old_phone_number`<br />`new_phone_number` |
| `address`    | `address`                                                                   |
| `username`   | `username`<br />`user_name`                                                 |
