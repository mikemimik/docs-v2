<Card title="Antes de empezar">
  Para migrar a Unified Phone Experience, debes:

  * Tener habilitadas y configuradas en tu tenant las notificaciones por SMS, voz o SMS+voz de la autenticación multifactor (MFA), o bien
  * Tener habilitado y configurado en tu tenant Passwordless SMS

  Si no has configurado ni MFA ni Passwordless, consulta [Configurar Unified Phone Experience de Auth0](/es/docs/customize/phone-messages/unified-phone/configure-unified-phone) para empezar desde el principio.

  Si tienes integraciones de MFA y Passwordless con proveedores distintos, debes usar un [proveedor telefónico personalizado](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider).

  Si utilizas Phone as ID, ya estás usando automáticamente Unified Phone Experience de Auth0 y no necesitas realizar ninguna acción adicional.
</Card>

Unified Phone Experience centraliza toda la configuración de tus proveedores telefónicos en un solo lugar dentro de tu tenant. No necesitas configurar un proveedor telefónico varias veces para distintos factores de autenticación por teléfono. Para obtener más información sobre los factores telefónicos para <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor adicional al nombre de usuario y la contraseña, como un código enviado por SMS." cta="Ver glosario" href="/es/docs/glossary?term=MFA">MFA</Tooltip>, consulta [Notificaciones por SMS y voz para MFA](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa).

Si ya tienes un proveedor telefónico configurado para [las notificaciones por SMS y voz de la autenticación multifactor (MFA)](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa), puedes migrar tu configuración a Unified Phone Experience y aplicarla a nivel de tenant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 no informa al usuario final si un mensaje telefónico se ha enviado correctamente con el proveedor configurado a nivel de tenant. Para solucionar problemas, revisa los [logs del tenant](/es/docs/deploy-monitor/logs).
</Callout>

<div id="update-your-phone-provider">
  ## Actualiza tu proveedor telefónico
</div>

Para usar Unified Phone Experience, necesitas:

* Verificar la configuración actual
* Configurar tu proveedor:

  * Configurar Twilio
  * Configurar un proveedor telefónico personalizado
* Actualizar tus factor telefónico y métodos de entrega
* Actualizar los endpoints de <Tooltip tip="Management API: Un producto para permitir que los clientes realicen tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>
* Actualizar los triggers de Auth0 Actions

<div id="configure-twilio-as-a-tenant-level-provider">
  ### Configura Twilio como proveedor a nivel del tenant
</div>

Si usas Twilio como proveedor telefónico para la autenticación multifactor, debes agregar el SID de Twilio y el AuthToken a la Unified Phone Experience. Puedes copiar el SID de Twilio de tu configuración actual, pero tendrás que obtener el valor del AuthToken desde Twilio.

1. Ve a [**Branding &gt; Proveedor telefónico**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona y configura tu proveedor:

   <Frame>![Branding > Proveedor telefónico > Twilio](/docs/images/cdy7uua7fh8z/6TESTIdmhUL0LEitACadIK/fcdc34e18a6b66b47370f6d754c085b5/2025-02-21_12-49-53.png)</Frame>

   1. Elige Twilio como proveedor telefónico.
   2. Elige los métodos de entrega.
   3. Ingresa la configuración de Twilio.

      <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
        Los factores de MFA deseados deben ser un subconjunto de los métodos de entrega que configures a nivel del tenant.
      </Callout>
3. **Guarda** los cambios.
4. Selecciona **Enviar mensaje de prueba** para probar el proveedor a nivel del tenant antes de usarlo en un entorno de producción.
5. Habilita un proveedor a nivel del tenant.

   1. Para MFA, ve a [**Security &gt; Autenticación multifactor &gt; Phone Message**.](https://manage.auth0.com/#/security/mfa)
   2. Selecciona **Usar proveedor de mensajería a nivel del tenant**.

      <Warning>
        Si no ves ningún interruptor en tu tenant, ya estás usando la Unified Phone Experience.
      </Warning>
6. **Guarda** los cambios.

<div id="configure-a-custom-provider-with-auth0-actions">
  ### Configura un proveedor personalizado con Auth0 Actions
</div>

La Unified Phone Experience usa el trigger de Actions `custom-phone-provider` para las notificaciones por teléfono, en lugar del trigger de Actions `send-phone-message` para las notificaciones telefónicas personalizadas.

No puedes migrar directamente tu trigger de Actions `send-phone-message` al trigger de Actions `custom-phone-provider` porque:

* No existe un mapeo 1:1 entre los tipos de mensaje y los tipos de notificación.
* Migrar los secrets y las dependencias asociados al trigger de Actions send-phone-message podría generar problemas de seguridad o afectar la experiencia del tenant.
* Una Action puede admitir más de un trigger, por lo que su migración podría resultar difícil. Por ejemplo, una Action puede admitir tanto los triggers `send-phone-message` como `post-login`.
* Los triggers de Actions `send-phone-message` y `custom-phone-provider` admiten distintos tipos de evento:

  * `send-phone-message` admite los tipos de mensaje SMS y voz. Para obtener más información, consulta [Action triggers: send-phone-message Event Object](/es/docs/customize/actions/explore-triggers/mfa-notifications-trigger/send-phone-message-event-object).
  * `custom-phone-provider` admite los tipos de notificación `otp_verify`, `otp_enroll`, `blocked_account`, `change_password` y `password_breach`. Para obtener más información, consulta [Actions triggers: custom-phone-provider Event Object](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/actions-triggers-custom-phone-provider-event-object).
* Solo puedes configurar una Action por proveedor telefónico personalizado. Si ya configuraste una Action para un proveedor telefónico personalizado, **debes** deshabilitarla antes de asociarla a una nueva Action.

Como resultado, debes convertir tu `send-phone-message` existente en un nuevo `custom-phone-provider` usando las nuevas variables contextuales:

Revise el ejemplo del proveedor telefónico personalizado:

```javascript lines expandable
const { fetch } = require('undici');
/**
* Controlador que se ejecuta al enviar una notificación por teléfono
* @param {Event} event - Detalles sobre el usuario y el contexto en el que está iniciando sesión.
* @param {CustomPhoneProviderAPI} api - Métodos y utilidades para modificar el comportamiento del envío de notificaciones por teléfono.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {

  if (event.notification.message_type.startsWith('otp')) {
    const body = {
      from: event.notification.from,
      action: event.notification.message_type === 'otp_verify' ? 'second-factor-authenticator' : 'enrollment',
      language: event.notification.locale,
      recipient: event.notification.recipient,
      message_type: event.notification.delivery_method,
      text: event.notification.as_text,
      code: event.notification.code,
    };

    const response = await fetch(event.secrets['MY_ENDPOINT'], {
      method: 'POST',
      headers: {
        authorization: `Bearer ${event.secrets['MY_SECRET']}`,
      },
      body: JSON.stringify(body),
    });

    console.log(response.status); // 200
    console.log(await response.text()); // ¡cuerpo de la respuesta de la solicitud!
  }

  return;
};
```

<div id="configure-mfa-custom-phone-provider">
  #### Configura `custom-phone-provider` de MFA
</div>

Para continuar y configurar el trigger `custom-phone-provider` de MFA:

1. Ve a [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Selecciona tu proveedor:

   1. Elige **Custom**.
   2. En **Provider Configuration**, agrega el código traducido de la Action `send-phone-message` en el editor de código integrado.

6. **Save** los cambios.

<div id="disable-phone-provider">
  ## Desactivar el proveedor telefónico
