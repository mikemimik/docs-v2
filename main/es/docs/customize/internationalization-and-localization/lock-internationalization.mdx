<div id="set-language-option">
  ## Configurar la opción de idioma
</div>

Para usar los siguientes ejemplos, primero tendrás que incluir Lock en tu página:

`<script src="https://cdn.auth0.com/js/lock/11.14/lock.min.js"></script>`

A continuación, define tu objeto `options` e incluye la opción `language`. La opción `language` debe ser una cadena que coincida con el nombre del archivo correspondiente en el directorio `i18n` [de Lock](https://github.com/auth0/lock/tree/master/src/i18n). Después, inicializa Lock.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample1 = `// Selecciona un idioma compatible
var options = {
  language: 'es'
};

// Inicializa Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample1} language="javascript" />

Los datos de traducción de Lock provienen de archivos de idioma con pares clave-valor que representan distintas traducciones. En algunos idiomas, puede que falten ciertos valores; en ese caso, verás la advertencia: `language does not have property <missing prop>`. Te animamos a enviar un [pull request](https://github.com/auth0/lock/tree/master/src/i18n) para añadir los valores que faltan. Como alternativa, puedes definir esos valores en las `options` de tu Lock (consulta más abajo).

<div id="replace-dictionary-terms">
  ## Reemplazar términos del diccionario
</div>

También puedes personalizar elementos específicos de tu diccionario mediante la opción `languageDictionary`. Esto resulta útil si estás usando uno de los idiomas compatibles, pero quieres cambiar la redacción concreta de algunos elementos. Por ejemplo, puedes reformular el `title` o modificar la forma en que se muestran otras etiquetas al usuario, dejando intacto el resto del texto del widget.

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample2 = `// Personaliza algunas propiedades de languageDictionary
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Iniciar sesión"
  },
};

// Inicializamos nuestro Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<AuthCodeBlock children={codeExample2} language="javascript" />

Para ver un ejemplo de los nombres de propiedad `languageDictionary` disponibles y de cómo estructurar un archivo `language`, consulta el [archivo de diccionario en inglés de Lock](https://github.com/auth0/lock/blob/master/src/i18n/en.js).

La opción `languageBaseUrl`, que acepta un valor de tipo cadena (una URL), reemplaza la URL de origen del idioma para las traducciones proporcionadas por Auth0. De forma predeterminada, usa la URL de la CDN de Auth0 `https://cdn.auth0.com`, ya que ahí se almacenan las traducciones de idioma proporcionadas. Si especificas otro valor, puedes usar tu propia fuente para las traducciones de idioma según sea necesario para tus aplicaciones. La fuente de idioma debe ser un archivo JavaScript.
