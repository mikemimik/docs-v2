<Warning>
  La fecha de End of Life (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos tenants creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Le recomendamos encarecidamente que use Actions para ampliar Auth0. Con Actions, tiene acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, lea [Entender cómo funcionan las Actions de Auth0](/es/docs/customize/actions/actions-overview).

  Para ayudarle con la migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/es/docs/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/es/docs/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarle en su proceso de migración.

  Para obtener más información sobre la descontinuación de Rules y Hooks, lea nuestra publicación del blog: [Preparación para el End of Life de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, solo debe crear nuevas Rules o Hooks en su entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar sus Rules a Actions, lea [Migrar de Rules a Actions](/es/docs/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar sus Hooks a Actions, lea [Migrar de Hooks a Actions](/es/docs/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Hooks incluye una gestión integrada de secretos para almacenar secretos de forma segura y, al mismo tiempo, ponerlos a disposición del código de manera práctica.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para acceder a un Hook Secret configurado desde un hook, use `context.webtask.secrets.SECRET_NAME`.
</Callout>

Puedes crear, actualizar, eliminar y ver secretos de Hooks desde el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulta:

* [Crear secretos de Hooks](/es/docs/customize/hooks/hook-secrets/create-hook-secrets)
* [Actualizar secretos de Hooks](/es/docs/customize/hooks/hook-secrets/update-hook-secrets)
* [Eliminar secretos de Hooks](/es/docs/customize/hooks/hook-secrets/delete-hook-secrets)
* [Ver secretos de Hooks](/es/docs/customize/hooks/hook-secrets/view-hook-secrets)

Los secretos de Hooks también se pueden importar o exportar con la [herramienta Deploy Command-Line Interface (CLI).](/es/docs/deploy-monitor/deploy-cli-tool)
