<Warning>
  La fecha de fin de vida útil (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para tenants nuevos creados a partir del **16 de octubre de 2023**. Los tenants existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida útil.

  Recomendamos encarecidamente que uses Actions para ampliar Auth0. Con Actions, tienes acceso a información de tipos detallada, documentación integrada y paquetes `npm` públicos, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Understand How Auth0 Actions Work](/docs/es/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías para [migrar de Rules a Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions). También tenemos una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funciones, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, consulta nuestra entrada del blog: [Preparing for Rules and Hooks End of Life](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, solo debes crear nuevas Rules o Hooks en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, consulta [Migrate from Rules to Actions](/docs/es/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, consulta [Migrate from Hooks to Actions](/docs/es/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Cuando ya no necesites los secretos de Hooks para un Hook determinado, puedes eliminarlos usando el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

Los secretos de Hooks también se pueden importar y exportar con la herramienta [Deploy Command-Line Interface (CLI)](/docs/es/deploy-monitor/deploy-cli-tool).

<div id="delete-hook-secrets-using-the-dashboard">
  ## Eliminar secretos de Hooks usando el Dashboard
</div>

1. Ve a la página de [Hooks](https://manage.auth0.com/#/hooks) en el [Auth0 Dashboard](https://manage.auth0.com/#/), y haz clic en el icono del lápiz junto al Hook que quieres editar.
2. En el editor del Hook, haz clic en el icono de la llave inglesa y luego en **Secrets**.
3. Busca el secreto de Hooks que quieres eliminar, haz clic en el icono de la papelera y luego en **Yes**.

<div id="delete-hook-secrets-using-the-management-api">
  ## Eliminar secretos de Hooks con la Management API
</div>

Realiza una solicitud `DELETE` al [endpoint Eliminar secretos de Hooks](https://auth0.com/docs/api/management/v2/#!/Hooks/delete_secrets). Asegúrate de reemplazar los valores de marcador de posición `HOOK_ID`, `HOOK_SECRET_NAME` y `MGMT_API_ACCESS_TOKEN` por tu ID de Hook, los nombres de tus secretos de Hooks y el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+token">token de acceso</Tooltip> de la Management API, respectivamente.

| Valor                   | Descripción                                                                                                                                                                                                               |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `HOOK_ID`               | ID del Hook para el que desea eliminar secretos.                                                                                                                                                                          |
| `MGMT_API_ACCESS_TOKEN` | Tokens de acceso para la Management API con el [scope](/docs/es/glossary?term=scope) `update:hooks`. Para obtener más información, consulte [Management API Access Tokens](https://auth0.com/docs/api/management/v2/tokens). |
| `HOOK_SECRET_NAME`      | Nombre(s) de los secretos que desea eliminar del Hook especificado. Este endpoint acepta un arreglo de nombres de secretos para eliminar.                                                                                 |
