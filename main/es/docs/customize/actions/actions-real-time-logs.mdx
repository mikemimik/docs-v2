description: Aprende a usar los logs en tiempo real de Auth0 Actions.
title: Logs en tiempo real de Actions
---

La función de logs en tiempo real de Actions muestra en tiempo real todos los logs del código personalizado de tu cuenta. Esto incluye toda la salida de `console.log` y las excepciones. Estos logs son útiles para depurar tus Actions, scripts de base de datos personalizados y conexiones sociales personalizadas.

<Warning>
  Al usar los logs en tiempo real de Actions, ten cuidado de no registrar información confidencial. Por ejemplo, los parámetros que reciben como entrada las funciones de Actions, los scripts de base de datos personalizados o las conexiones sociales personalizadas pueden contener información confidencial, ya sea explícita o implícita. Registrar objetos completos puede provocar la divulgación de información confidencial en los logs en tiempo real.

  Auth0 desaconseja firmemente estas prácticas. **Te recomendamos prestar atención a los objetos que registras y asegurarte de que no se registren datos confidenciales, así como evitar registrar explícitamente cualquier tipo de información confidencial.**
</Warning>

<div id="how-to-use">
  ## Cómo usarlo
</div>

Para ver los logs en tiempo real de tu código personalizado:

1. Ve a [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs).
2. Espera a que se conecte la transmisión. Los intentos de conexión se realizan automáticamente.
3. Una vez conectada, las entradas del log aparecerán en el panel principal a medida que se generen.

<div id="features">
  ## Funcionalidades
</div>

El panel de logs en tiempo real de Actions te permite supervisar la actividad del sistema a medida que ocurre. Esto proporciona información en vivo sobre las Actions ejecutadas a través del Dashboard, lo que facilita identificar problemas, confirmar operaciones correctas o simplemente observar el comportamiento del sistema en tiempo real. A continuación, consulta una lista detallada de las funcionalidades disponibles actualmente en esta característica.

<div id="live-log-streaming">
  ### Transmisión en tiempo real de registros
</div>

Las entradas de registro se transmiten en tiempo real a medida que Actions las genera. A medida que se ejecuta cada Action, la entrada de registro correspondiente aparece automáticamente en el panel de registros, sin necesidad de actualizar la página.

<div id="connection-status">
  ### Estado de la conexión
</div>

La parte superior del panel muestra el estado actual de la conexión entre tu navegador y el flujo de logs. Esto te ayuda a confirmar si los logs se están recibiendo activamente. Los estados más comunes incluyen:

| Estado                         | Descripción                                 |
| ------------------------------ | ------------------------------------------- |
| Conectando al flujo de logs... | Intentando establecer una conexión.         |
| CONECTADO                      | Conectado correctamente y recibiendo logs.  |
| DESCONECTADO                   | No está conectado al flujo en este momento. |

<div id="search-functionality">
  ### Función de búsqueda
</div>

Usa la barra de búsqueda en la parte superior del panel para filtrar las entradas del registro. Puedes buscar por:

* Palabras clave o frases
* Códigos de error específicos
* Mensajes de registro exactos

Esto resulta especialmente útil al revisar grandes volúmenes de registros en busca de una Action o de un problema específico.

<div id="options-menu">
  ### Menú de opciones (...)
</div>

Haz clic en el menú de **Opciones**, en la esquina superior derecha del panel de logs, para acceder a controles adicionales.

* **Hora local:** Alterna la visualización de la marca temporal de las entradas de logs entre la hora predeterminada del servidor (a menudo, UTC) y la hora local del navegador del usuario.
* **Descargar:** Permite descargar los logs que se muestran actualmente como un archivo JSON para analizarlos sin conexión, compartirlos o archivarlos.
* **Borrar logs:** Borra todas las entradas de logs visibles actualmente en la ventana de visualización. Esto no elimina los logs del almacenamiento del sistema, solo de la vista actual.

<div id="scroll-to-bottom">
  ### Ir al final
</div>

Cuando llegan nuevas entradas y quedan fuera de la vista, aparece un botón de **Ir al final** en el panel de registros. Al hacer clic en él, irás a la entrada de registro más reciente, para que siempre estés al día con la actividad más reciente.

<div id="debug-an-action">
  ## Depurar una Action
</div>

El siguiente ejemplo describe cómo crear una Action genérica de Hello World, ejecutarla y usar los logs en tiempo real de Actions para ver los resultados.

1. Sigue las instrucciones de [Write Your First Action](/docs/es/customize/actions/write-your-first-action) para escribir una Action de Post Login con el ejemplo de código siguiente:

   ```javascript lines
   /**
   * Controlador que se llamará durante la ejecución de un flujo PostLogin.
   *
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que inicia sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos se pueden usar para cambiar el comportamiento del inicio de sesión.
   */
   exports.onExecutePostLogin = async (event, api) => {
     console.log('Hello World!');
   };

   /**
   * Controlador que se invocará cuando esta Action se reanude después de una redirección externa. Si tu
   * función onExecutePostLogin no realiza una redirección, esta función puede ignorarse sin problema.
   *
   * @param {Event} event - Detalles sobre el usuario y el contexto en el que inicia sesión.
   * @param {PostLoginAPI} api - Interfaz cuyos métodos se pueden usar para cambiar el comportamiento del inicio de sesión.
   */
   // exports.onContinuePostLogin = async (event, api) => {
   // };
   ```

2. Abre una pestaña nueva y ve a [Auth0 Dashboard &gt; Monitoring &gt; Actions Logs](https://manage.auth0.com/#/tenant/action-logs)

3. Una vez conectada la transmisión, vuelve al editor de Actions y ejecuta una prueba.

4. Los resultados deberían aparecer en tiempo real en los logs en tiempo real de Actions.

<div id="supported-statements">
  ### Instrucciones compatibles
</div>

La siguiente tabla describe las instrucciones de consola de JavaScript compatibles con la función de registro en tiempo real:

| Instrucción       | Descripción                                                                                            | Caso de uso                                                                  |
| ----------------- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------- |
| `console.log()`   | Registro de información de uso general.                                                                | Mensajes informativos básicos, valores de variables.                         |
| `console.info()`  | Mensajes informativos que destacan el progreso o el estado de la aplicación.                           | Informar sobre operaciones correctas, hitos de la aplicación.                |
| `console.warn()`  | Indica posibles problemas o situaciones inesperadas que no son errores críticos.                       | Señalar problemas no críticos, funciones obsoletas.                          |
| `console.error()` | Informa de errores que se han producido en la aplicación.                                              | Indicar fallos, excepciones y problemas críticos.                            |
| `console.trace()` | Muestra un seguimiento de la pila e indica la secuencia de llamadas a funciones hasta el punto actual. | Depurar flujos de llamadas complejos, comprender el origen de los problemas. |
