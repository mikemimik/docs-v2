El objeto de API del trigger `post-user-registration` de Actions incluye:

<div id="apicache">
  ## `api.cache`
</div>

Almacena y recupera datos que se conservan entre ejecuciones.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina, si existe, un registro que describe un valor almacenado en la caché para la clave proporcionada.

Devuelve un objeto `CacheWriteResult` con `type: "success"` si se eliminó un valor de la caché. Si la operación falla, devuelve `type: "error"`. En caso de errores, el objeto devuelto tendrá una
propiedad `code` que indica la naturaleza del fallo.

| Parámetro | Descripción                                             |
| --------- | ------------------------------------------------------- |
| `key`     | *String*. La clave del registro almacenado en la caché. |

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en caché en la
`key` proporcionada, si existe. Si se encuentra un registro, el valor almacenado en caché
se puede encontrar en la propiedad `value` del objeto devuelto.

Devuelve un registro de caché si se encuentra un elemento en la caché para la
`key` proporcionada. Los registros de caché son objetos con una
propiedad `value` que contiene el valor almacenado en caché, así como una
propiedad `expires_at` que indica el vencimiento máximo del
registro en milisegundos desde la época Unix.

**Importante:**
Esta caché está diseñada para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores, incluso si todavía están dentro de su
duración.

| Parámetro | Descripción                                             |
| --------- | ------------------------------------------------------- |
| `key`     | *String*. La clave del registro almacenado en la caché. |

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, [options])`
</div>

Almacena o actualiza un valor de cadena en el caché con la clave especificada.

Los valores almacenados en este caché están limitados al trigger en el que se establecen.
Están sujetos a los
[Límites del caché de Actions](/es/docs/customize/actions/limitations).

Los valores almacenados de esta manera tendrán una duración de hasta los valores
especificados en `ttl` o `expires_at`. Si no se especifica
ninguna duración, se usará una duración predeterminada de 15 minutos. Las duraciones
no pueden superar la duración máxima indicada en
[Límites del caché de Actions](/es/docs/customize/actions/limitations).

| Parameter            | Description                                                                                                                                                                                                                                                                                                                                                                                             |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `key`                | *String*. La clave del registro almacenado en el caché.                                                                                                                                                                                                                                                                                                                                                 |
| `value`              | *String*. El valor del registro que se almacenará.                                                                                                                                                                                                                                                                                                                                                      |
| `options`            | *Objeto opcional*. Opciones para ajustar el comportamiento del caché.                                                                                                                                                                                                                                                                                                                                   |
| `options.expires_at` | *Número opcional*. La hora de vencimiento absoluta en milisegundos desde la epoch de Unix. Aunque los registros en caché pueden eliminarse antes, nunca permanecerán más allá del valor de `expires_at` proporcionado.<br />*Nota:* Este valor no debe proporcionarse si también se proporcionó un valor para `ttl`. Si se proporcionan ambas opciones, se usará el vencimiento más cercano de los dos. |
| `options.ttl`        | *Número opcional*. El valor de tiempo de vida de esta entrada de caché en milisegundos. Aunque los valores en caché pueden eliminarse antes, nunca permanecerán más allá del valor de `ttl` proporcionado.<br />*Nota:* Este valor no debe proporcionarse si también se proporcionó un valor para `expires_at`. Si se proporcionan ambas opciones, se usará el vencimiento más cercano de los dos.      |
