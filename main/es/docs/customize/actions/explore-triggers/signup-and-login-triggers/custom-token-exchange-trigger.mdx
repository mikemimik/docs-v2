Al usar esta funcionalidad, aceptas los términos aplicables de la prueba gratuita incluidos en el [Master Subscription Agreement de Okta](https://www.okta.com/legal/). Después de que el servidor de autorización de Auth0 valida que la solicitud de Custom Token Exchange es válida y corresponde a un [Custom Token Exchange Profile](/es/docs/authenticate/custom-token-exchange/configure-custom-token-exchange#create-custom-token-exchange-profile) existente, el trigger `custom-token-exchange` ejecuta la única Action asociada a ese perfil. Si la Action se completa correctamente y asigna un usuario a la transacción, las Actions de `Post Login` y el resto del pipeline de Auth0 se ejecutan a continuación para ese usuario autenticado.

<Frame>![](/docs/images/custom-token-exchange/cte_trigger_flow.png)</Frame>

Las Actions de este flujo son bloqueantes (sincrónicas), lo que significa que se ejecutan como parte del proceso de un trigger e impiden que el resto del pipeline de Auth0 se ejecute hasta que la Action se complete.

<div id="custom-token-exchange-vs-post-login-actions">
  ## Custom Token Exchange vs Actions de Post Login
</div>

El trigger de Actions de Post Login se ejecuta después de que se ejecute la Action de Custom Token Exchange para el usuario que establezcas para la transacción. Para otros casos de uso, como agregar claims personalizadas a los access tokens, usa el [trigger de Action de Post Login](/es/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger).

Custom Token Exchange usa el tipo de concesión de intercambio de tokens, en el que `event.transaction.protocol` se establece en `oauth2-token-exchange` en el trigger de Action de Post Login. Tanto Custom Token Exchange como el inicio de sesión social nativo usan el tipo de concesión de intercambio de tokens. Para distinguir entre ambos, usa el valor de `subject_token_type`, que debe corresponder a uno de tus perfiles de Custom Token Exchange.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El acceso anticipado de Custom Token Exchange no admite `api.authentication.challengeWith()` ni `api.authentication.enrollWith()`. Si usas esos métodos con tus Actions de Post Login, la transacción fallará con un error no recuperable. Asegúrate de omitir esos dos métodos cuando `event.transaction.protocol==oauth2-token-exchange`, según el valor de `subject_token_type`.
</Callout>

<div id="actor-support-for-delegation">
  ## Compatibilidad con actores para la delegación
</div>

El trigger `custom-token-exchange` admite escenarios de autorización delegada. La Action puede llamar a [`api.authentication.setActor()`](/es/docs/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger/custom-token-exchange-api-object#api-authentication-setactor-actor) para incluir un claim `act` en los tokens emitidos, con el fin de identificar a la entidad principal que actúa en nombre del usuario sujeto.

Se puede llamar a `setActor()` independientemente de si la solicitud incluye un `actor_token`. El parámetro `actor_token` es una forma de que la aplicación llamante pase información del actor a la Action, pero la identidad del actor también puede estar incrustada en el propio `subject_token` u obtenerse por otros medios personalizados dentro del código de la Action.

Cuando la solicitud sí incluye `actor_token` y `actor_token_type`, estos valores están disponibles en [`event.transaction`](/es/docs/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger/custom-token-exchange-event-object#event-transaction). Si `actor_token_type` es `urn:ietf:params:oauth:token-type:id_token` y el token es un token de ID de Auth0 válido del mismo tenant, el servidor valida automáticamente el token y rellena `event.transaction.actor_token_user` con el perfil de usuario del actor.

El claim `act` establecido mediante `setActor()` también está disponible en las Actions Post-Login a través de [`event.transaction.actor`](/es/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object#event-transaction).

<div id="triggers">
  ## Triggers
</div>

<div id="custom-token-exchange">
  ### Custom Token Exchange
</div>

El trigger `custom-token-exchange` es el primer paso del pipeline de Auth0. Esta función se ejecuta después de que se haya validado la solicitud de Custom Token Exchange recibida y se haya asignado correctamente a un perfil existente.

<div id="references">
  ### Referencias
</div>

* [objeto de evento](/es/docs/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger/custom-token-exchange-event-object): Proporciona información contextual sobre una solicitud de Intercambio de tokens, incluido el subject&#95;token, la dirección IP, el cliente y más.
* [objeto de API](/es/docs/customize/actions/explore-triggers/signup-and-login-triggers/custom-token-exchange-trigger/custom-token-exchange-api-object): Proporciona métodos para usar con tu Action de Intercambio de tokens, que te permite establecer el usuario, establecer el actor para la delegación, rechazar la transacción, establecer los metadatos del usuario y acceder a la caché de Actions, entre otras cosas.
