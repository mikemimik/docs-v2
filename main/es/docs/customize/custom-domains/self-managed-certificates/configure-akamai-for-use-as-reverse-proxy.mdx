<Card title="La disponibilidad varía según el plan de Auth0">
  La disponibilidad de esta función depende tanto de tu implementación específica de inicio de sesión como de tu plan de Auth0 o de tu acuerdo personalizado. Para obtener más información, consulta [Pricing](https://auth0.com/pricing).
</Card>

Para configurar un producto de Akamai como proxy inverso, completa lo siguiente:

1. Verifica la propiedad de tu dominio personalizado. Para obtener más información, consulta [Verify Ownership](/es/docs/customize/custom-domains/auth0-managed-certificates#verify-ownership).
2. Ve a [Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) para obtener la distribución, el encabezado personalizado de origen, el hostname de origen y la configuración predeterminada del comportamiento de caché. Luego, usa esta información para actualizar la configuración de tu dominio personalizado en Akamai de modo que apunte al dominio del servidor proxy inverso. Para obtener más información, consulta [Configurar proxy inverso](/es/docs/customize/custom-domains/self-managed-certificates#configure-reverse-proxy).
3. Actualiza el encabezado de IP del cliente del dominio personalizado mediante el endpoint PATCH `/api/v2/custom-domains/{id}` de la Management API, como se muestra a continuación:

   ```json lines
   { 
   "custom_client_ip_header": "true-client-ip" 
   }
   ```

   Para obtener más información, consulta [Update custom domain configuration](https://auth0.com/docs/api/management/v2#!/Custom_Domains/patch_custom_domains_by_id).

Para obtener más información sobre cómo configurar Akamai como proxy inverso con los datos anteriores, consulta la documentación de Akamai o ponte en contacto con el equipo de soporte.
