<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Debes tener una suscripción Enterprise para gestionar certificados en tu dominio personalizado. Para obtener más información, consulta [Auth0 Pricing](https://auth0.com/pricing) y [Login](/es/docs/authenticate/login).
</Callout>

Si decides gestionar los certificados de tus <Tooltip tip="Dominio personalizado: Dominio third-party con un nombre especializado o personalizado." cta="Ver glosario" href="/es/docs/glossary?term=custom+domains">dominios personalizados</Tooltip> sin ayuda de Auth0, debes saber cómo gestionar tu propio proxy y los certificados para las conexiones entrantes que requieren varios registros DNS en el dominio. Tendrás que comprar o proporcionar los certificados y encargarte tú mismo de las renovaciones. También necesitarás un proxy inverso en el que se instalará el certificado. Una vez que se verifique el dominio, aceptaremos tráfico desde el proxy.

Elige esta opción para:

* Tener más control sobre tus certificados (por ejemplo, para elegir tu propia CA o el vencimiento del certificado).
* Habilitar monitoreo adicional de tus llamadas a la API de Auth0.

Para configurar tu dominio personalizado con certificados autogestionados, debes proporcionar tu nombre de dominio a Auth0, verificar que eres el propietario de ese dominio y configurar el proxy inverso. Una vez configurado tu dominio personalizado, tendrás que ajustar las funciones de Auth0 para empezar a usarlo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Tu proxy hacia Auth0 debe usar TLS (SSL) versión 1.2 o una versión posterior.
</Callout>

<div id="provide-your-domain-name-to-auth0">
  ## Indique su nombre de dominio a Auth0
</div>

1. Vaya a [Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) o a [Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/custom_domains). Introduzca su dominio personalizado en el campo correspondiente y seleccione **Self-managed Certificates** en el menú desplegable.
2. Elija **Add Domain**.
   Solo puede añadir un dominio por tenant, aunque el botón **Add Domain** siga apareciendo después de añadir un dominio.

<div id="verify-ownership">
  ## Verificar la propiedad
</div>

Antes de poder usar el dominio con Auth0, tendrás que verificar que eres su propietario.

1. Ve a [Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/branding/custom_domains).

   <Frame>![Auth0 Branding Custom Domains Self-Managed Certificate Verify Domain](/docs/images/cdy7uua7fh8z/4XhEmQwKfoqydOldfHnnud/c63a58c72f79b4d19cd417f70ba262cc/Custom_Domains_-_Self_-_English.png)</Frame>

2. Agrega la información de verificación TXT al registro DNS de tu dominio.
   Estos pasos pueden variar según tu proveedor de dominios:

   1. Mantén abierta en tu navegador la página de dominios personalizados de Auth0 para poder copiar los valores.
   2. Inicia sesión en tu servicio de administración de dominios.
   3. Crea un registro nuevo y guárdalo con esta configuración:

      | Parámetro          | Valor                                                               |
      | ------------------ | ------------------------------------------------------------------- |
      | Tipo de registro   | TXT                                                                 |
      | Nombre             | Copia y pega el valor de **TXT Record** de tu dominio desde Auth0.  |
      | Time to Live (TTL) | Usa el valor predeterminado.                                        |
      | Valor              | Copia y pega el valor de **TXT Content** de tu dominio desde Auth0. |

3. Haz clic en **Verify** para continuar.

   Es posible que Auth0 tarde unos minutos en verificar tu dominio, según tu configuración de DNS.

   Si Auth0 verifica correctamente tu nombre de dominio, verás una ventana de confirmación.
   Guarda la información que se proporciona en esta ventana, en concreto el valor de `cname-api-key`, ya que esta es la **única** vez que verás este valor.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Por motivos de seguridad, este valor se almacena como un hash, por lo que no se puede recuperar. Si no guardas la `cname-api-key` en este momento, tendrás que volver a crear los dominios personalizados.
   </Callout>

   El proceso de verificación se habrá completado y, en un plazo de 1 a 2 minutos, tu dominio personalizado debería estar listo para usarse. Si no puedes completar el proceso de verificación en un plazo de tres días, repite estos pasos.

<div id="configure-reverse-proxy">
  ## Configurar el proxy inverso
</div>

El servidor proxy inverso recupera recursos de uno o más servidores en nombre de tu aplicación. Luego, esos recursos se devuelven a la aplicación como si se hubieran originado en el propio servidor proxy.

Puedes usar un servicio como [Cloudflare](/es/docs/customize/custom-domains/self-managed-certificates/configure-cloudflare-for-use-as-reverse-proxy), [Azure CDN](/es/docs/customize/custom-domains/self-managed-certificates/configure-azure-cdn-for-use-as-reverse-proxy), [Google Cloud Platform](/es/docs/customize/custom-domains/self-managed-certificates/configure-gcp-as-reverse-proxy) o [AWS Cloudfront](/es/docs/customize/custom-domains/self-managed-certificates/configure-aws-cloudfront-for-use-as-reverse-proxy) y configurar los ajustes de tu dominio personalizado. Añadirás el nuevo valor de CNAME a tu DNS para tu dominio personalizado, de modo que apunte al nombre de dominio del servidor proxy inverso para la distribución.

1. Después de crear la configuración del proxy inverso en tu servicio, ve a la pestaña [Auth0 Dashboard &gt; Branding &gt; Custom Domains](https://manage.auth0.com/#/branding/custom_domains).
2. Añade un nuevo registro CNAME a tu DNS para tu dominio personalizado, de modo que apunte al nombre de dominio del servicio para tu distribución. Puedes encontrarlo buscando el **ID de distribución** en la configuración de tu servidor proxy inverso.
   Una vez añadido, el registro CNAME debe estar presente en todo momento para evitar problemas durante la renovación del certificado.
3. La forma de configurar el servidor proxy variará según el servicio que uses. Es probable que tengas que configurar los siguientes tipos de ajustes:

* [Distribución](#distribution-settings)
* [Nombre de host de origen](#origin-hostname-settings)
* [Encabezados HTTP](#http-headers)
* [Comportamiento predeterminado de la caché](#default-cache-behavior-settings)

<div id="distribution-settings">
  ### Configuración de la distribución
</div>

| Configuración                           | Valor                                                                                                                                               |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Nombre de dominio de origen             | Configúralo con el valor de **Origin Domain Name** obtenido del Auth0 Dashboard durante el proceso de configuración de dominios personalizados.     |
| ID de origen                            | Una descripción del origen. Este valor te permite distinguir entre varios orígenes dentro de la misma distribución y, por lo tanto, debe ser único. |
| Política de protocolo de origen         | Establécela en `HTTPS Only`.                                                                                                                        |
| Nombres de dominio alternativos (CNAME) | Configúralos con tu nombre de dominio personalizado (el mismo que configuraste en el Auth0 Dashboard).                                              |

<div id="origin-hostname-settings">
  ### Configuración del hostname de origen
</div>

| Configuración                           | Valor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Hostname de origen                      | Introduce `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.us.auth0.com` y asegúrate de reemplazar `<CUSTOM_DOMAIN_ID>` por el valor del Nombre de dominio de origen (ID del dominio personalizado) que recibiste de Auth0 al configurar el nuevo dominio personalizado. Si tus tenants no están en la región de EE. UU., usa una de las siguientes opciones: <ul><li>EU: `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.eu.auth0.com`</li><li>AU: `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.au.auth0.com`</li><li>JP: `{yourTenant}-<CUSTOM_DOMAIN_ID>.edge.tenants.jp.auth0.com`</li></ul> Por ejemplo, si tu CUSTOM&#95;DOMAIN&#95;ID es:<br /><br />`cd_TXIdNgQ07HrAFVmz`<br /><br />Para un tenant de EE. UU., el hostname de origen debe ser:<br /><br />`US: {yourTenant}-cd-txIdngq07hrafvmz.edge.tenants.us.auth0.com` |
| Indicación del nombre de servidor (SNI) | Usa el nombre que proporcionaste para el hostname de origen.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

Para obtener más información sobre cómo recuperar los detalles de tu dominio personalizado, consulta [Get custom domain configurations](https://auth0.com/docs/api/management/v2/custom-domains/get-custom-domains).

<div id="http-headers">
  ### Encabezados HTTP
</div>

| Nombre del encabezado | Valor                                                                                                                                                                                                                                                                                                                                                                                   |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Host`                | El [hostname de origen](#origin-hostname-settings).                                                                                                                                                                                                                                                                                                                              |
| `cname-api-key`       | El valor de la clave de API de CNAME que se te proporcionó inmediatamente después de que verificaras la propiedad de tu nombre de dominio con Auth0. Debes establecer este encabezado para que Auth0 acepte tráfico desde tu proxy.                                                                                                                                                     |
| `X-Forwarded-For`     | La dirección IP del usuario final. No especifiques varias direcciones IP. <br /><br /> Puedes especificar un encabezado distinto para la dirección IP del usuario mediante el endpoint [Configurar un nuevo endpoint de dominio personalizado](/es/docs/api/management/v2/custom-domains/post-custom-domains) de la Management API para establecer la propiedad `custom_client_ip_header`. |

Auth0 usa la dirección IP del usuario final para funciones como la detección de anomalías, la limitación de solicitudes y MFA. Si tu proxy no reenvía la dirección IP del usuario final, Auth0 trata la dirección IP de tu proxy como la del cliente, lo que puede hacer que estas funciones no se comporten correctamente.

<div id="default-cache-behavior-settings">
  ### Configuración predeterminada del comportamiento de la caché
</div>

| Configuración                                                            | Valor                                                                                                                           |
| ------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| Política de protocolo del visor                                          | Seleccione **Redirigir HTTP a HTTPS**.                                                                                          |
| Métodos HTTP permitidos                                                  | Seleccione **GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE**.                                                                    |
| Almacenamiento en caché según los encabezados de solicitud seleccionados | Seleccione **Lista de elementos permitidos**.                                                                                   |
| Encabezados de la lista de elementos permitidos                          | Se deben permitir los siguientes encabezados: `User-Agent`, `Origin`, `Referer`, `Authorization`, `Accept` y `Accept-Language`. |
| Reenvío de cookies                                                       | Seleccione **Todos**.                                                                                                           |
| Reenvío y almacenamiento en caché de cadenas de consulta                 | Seleccione **Reenviar todo y almacenar en caché según todos**.                                                                  |

<div id="additional-steps-for-specific-auth0-features">
  ## Pasos adicionales para funciones específicas de Auth0
</div>

Hay pasos de configuración adicionales que debes completar según las funciones de Auth0 que estés utilizando. Para obtener más información, consulta [Configure Features to Use Custom Domains](/es/docs/customize/custom-domains/configure-features-to-use-custom-domains).

Cloudflare usa una función llamada [CNAME flattening](https://developers.cloudflare.com/dns/cname-flattening/), que afecta a la verificación de Auth0 y a la renovación de certificados por la forma en que gestiona los registros DNS. Te recomendamos desactivar CNAME flattening, a menos que sea absolutamente necesario.

Como esta función te da control sobre tus propios certificados, debes diseñar estrategias de mitigación (como un firewall de aplicaciones web) para bloquear solicitudes en caso de ataques. Para obtener más información, consulta [nuestro blog](https://auth0.com/blog/the-future-of-bot-protection-smarter-attacks-demand-a-layered-approach/) sobre el futuro de la protección contra bots.
