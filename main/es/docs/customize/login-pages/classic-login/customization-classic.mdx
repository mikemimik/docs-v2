description: Aprende a personalizar la página de inicio de sesión para la experiencia Classic Login.
title: Personalizar páginas de Classic Login
---

<Warning>
  En este momento, los esfuerzos de desarrollo activo de Auth0 están centrados en Universal Login, y Classic Login ya no recibe actualizaciones. Para personalizar tus páginas de inicio de sesión, Auth0 recomienda [Advanced Customizations for Universal Login](/docs/es/customize/login-pages/advanced-customizations).
</Warning>

[Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) es una experiencia de inicio de sesión alojada en Auth0 que se basa en JavaScript para personalizar páginas.

Desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, puedes configurar opciones básicas de personalización que se aplican a todas las páginas de inicio de sesión. Para una personalización avanzada, puedes modificar directamente la plantilla HTML de una o varias páginas.

Además, puedes gestionar el contenido de tu página con el [sistema de control de versiones](/docs/es/customize/login-pages/classic-login/version-control) que prefieras.

<div id="basic-customization">
  ## Personalización básica
</div>

Puedes configurar las siguientes opciones desde el Auth0 Dashboard en [Branding &gt; Universal Login &gt; pestaña Settings](https://manage.auth0.com/#/login_settings):

* Logotipo de la empresa (tamaño recomendado: 150 x 150 píxeles)
* Color principal
* Color de fondo

De manera predeterminada, estas opciones afectan a todas las páginas de Classic Login, como la pantalla de inicio de sesión o la página de restablecimiento de contraseña. Si modificas directamente la plantilla HTML de una o varias páginas y editas cualquier atributo relacionado con estas opciones, esos atributos tendrán prioridad sobre esta configuración.

<div id="advanced-customization">
  ## Personalización avanzada
</div>

Para personalizar aún más, puedes modificar directamente la plantilla HTML de las siguientes páginas:

* Login (ver abajo)
* [Restablecimiento de contraseña](/docs/es/customize/login-pages/classic-login/customize-password-reset-page)
* [autenticación multifactor](/docs/es/secure/multi-factor-authentication/customize-mfa/customize-mfa-classic-login)

<div id="responsibility-for-updates">
  ### Responsabilidad de las actualizaciones
</div>

**Cuando habilitas la personalización de una página específica, asumes la responsabilidad de mantener esa página**, ya que dejará de recibir actualizaciones automáticas de Auth0. Ese mantenimiento incluye actualizar los números de versión de los SDK de Auth0 o widgets incluidos.

Dado que por la página de inicio de sesión suele pasar información confidencial relacionada con la seguridad, la introducción de vulnerabilidades de cross-site scripting (XSS) es motivo de preocupación. Además, si tu organización usa [Home Realm Discovery](/docs/es/get-started/architecture-scenarios/business-to-business/authentication#home-realm-discovery) junto con Classic Login, una lista de las conexiones disponibles queda visible públicamente en la propiedad Auth0.setClient. Teniendo esto en cuenta, Auth0 te recomienda extremar la precaución al usar JavaScript de terceros en tu página de inicio de sesión.

<div id="enable-page-customization">
  ### Habilitar la personalización de la página
</div>

Para habilitar la personalización de una página específica, sigue estos pasos:

1. En el Auth0 Dashboard, ve a [Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings).
2. Selecciona una de las siguientes pestañas:

   * **Login**
   * **Restablecimiento de contraseña**
   * **Autenticación multifactor**
3. Activa el interruptor **Personalizar página** situado en la parte superior de la pestaña.
4. Repite los pasos 2 y 3 según sea necesario.

Después de habilitar la personalización de una página, puedes modificar su plantilla HTML en el editor de código situado debajo del interruptor.

<Warning>
  Si habilitas la personalización para inspeccionar el código de la página, pero decides no personalizarla, asegúrate de desactivar el interruptor **Personalizar página** al terminar para que Auth0 pueda renderizar las páginas predeterminadas.
</Warning>

<div id="customize-the-login-page">
  ### Personaliza la página de Login
</div>

Para personalizar la plantilla de la página de Login, primero debes elegir una plantilla base con la que trabajar.

1. En el Auth0 Dashboard, ve a [Branding &gt; Universal Login &gt; pestaña Login](https://manage.auth0.com/#/login_page).
2. Activa el interruptor **Customize Login Page**.
3. Sobre el editor de código HTML, selecciona una opción del menú **Plantillas predeterminadas**.

Las plantillas disponibles usan las siguientes bibliotecas:

* [Lock](/docs/es/libraries/lock)
* [Lock (modo passwordless)](/docs/es/libraries/lock)
* [Auth0.js](/docs/es/libraries/auth0js)
* [Authentication API](https://auth0.com/docs/api/authentication)

Puedes usar estas bibliotecas en Classic Login o integrarlas directamente en una aplicación. Para entender mejor cómo personalizar la experiencia de Login, consulta la documentación de la biblioteca que prefieras.

<div id="lock-template">
  #### Plantilla de Lock
</div>

Si planea usar la página de inicio de sesión predeterminada y solo necesita hacer cambios mínimos, puede usar la plantilla de Lock para modificar el comportamiento del widget de Lock en la página de inicio de sesión.

Por ejemplo, puede configurar el widget de Lock para:

* Redirigir a los usuarios a la página de registro en lugar de a la página de inicio de sesión de forma predeterminada
* Personalizar la página de inicio de sesión con distintos colores, textos o idiomas predeterminados
* Mostrar a los usuarios solo uno o varios métodos específicos de inicio de sesión

Para obtener más información sobre cómo personalizar el widget de Lock, consulte [Lock Configuration Options](/docs/es/libraries/lock/lock-configuration).

<div id="custom-login-form-template">
  #### Plantilla Custom Login Form
</div>

Si desea modificar considerablemente la página de inicio de sesión, puede usar la plantilla Custom Login Form como guía. Esta plantilla le muestra cómo obtener los valores que necesita con el SDK de Auth0.js. Después, puede modificar el estilo y el diseño de la página de inicio de sesión como desee. Cualquier personalización de CSS **debe** hacerse en línea, ya que no puede alojar un archivo CSS independiente en su tenant de Auth0.
