<Card title="Antes de empezar">
  Necesitas:

  * Un tenant de desarrollo de Auth0 configurado con [Universal Login](/docs/es/authenticate/login/auth0-universal-login) y un [dominio personalizado](/docs/es/customize/custom-domains).
  * Una [aplicación propia](/docs/es/get-started/auth0-overview/create-applications#create-applications) de Auth0.
  * [Identifier First Authentication](/docs/es/authenticate/login/auth0-universal-login/identifier-first) habilitado en tu tenant de Auth0.
  * [Node.js](http://Node.js) V22
  * [Auth0 CLI](https://auth0.github.io/auth0-cli/) instalado y configurado con tu tenant de desarrollo.
</Card>

Auth0 CLI te permite implementar y probar las pantallas de Advanced Customization for Universal Login (ACUL). Simplifica la personalización de pantallas de autenticación (como Login, Signup, Passwordless y la inscripción de clave de acceso) para que se ajusten exactamente a tus aplicaciones web.

Sigue estos pasos para crear un proyecto de ACUL y una pantalla `login_id` personalizada con Auth0 CLI:

<div id="1-initialize-an-acul-sample-application">
  ## 1. Inicializa una aplicación de ejemplo de ACUL
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si no has configurado tu tenant de desarrollo con Auth0 CLI, usa el comando `auth0 login` para configurar tu tenant.
</Callout>

Ve al directorio donde quieras crear tu proyecto y ejecuta el siguiente comando:

```bash
auth0 acul init "Your_App_Name"
```

1. Selecciona **React (con ACUL React SDK)**.
2. Selecciona la pantalla **login-id**.
3. En **Would you like to proceed with installing the required dependencies using &#39;npm install&#39;?**, escribe **y**.

Auth0 CLI crea un nuevo directorio del proyecto con el nombre de la aplicación proporcionado.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si no seleccionaste **y** para instalar las dependencias requeridas, ve al directorio de tu proyecto y ejecuta el comando `npm install`.
</Callout>

<div id="2-optional-add-additional-screens-to-your-project">
  ## 2. (Opcional) Añade pantallas adicionales a tu proyecto
</div>

Selecciona pantallas adicionales para tu proyecto ejecutando el siguiente comando:

```bash
auth0 acul screen add SCREEN_NAME 
```

<div id="3-launch-the-login-id-screen-with-universal-login-context-inspector">
  ## 3. Abre la pantalla Login Id con Universal Login Context Inspector
</div>

Usa el inspector de contexto de UL para abrir la pantalla Login ID:

1. Ve al directorio de tu proyecto y ejecuta el siguiente comando:

```bash
  auth0 acul dev
```

2. Confirma el directorio del proyecto
3. Selecciona el puerto para compartir los recursos locales; de forma predeterminada, el puerto `55444`

El Universal Login Context Inspector abre una nueva ventana del navegador con la pantalla Login Id.

<div id="4-update-the-sample-login-id-screen">
  ## 4. Actualiza la pantalla de ejemplo Login Id
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El comando auth0 acul dev compila las pantallas de ACUL, aloja tus recursos localmente y supervisa continuamente las actualizaciones del directorio de recursos, lo que te permite probar tus pantallas en tu entorno local. Para obtener más información, consulta la documentación del comando [Auth0 CLI acul](https://auth0.github.io/auth0-cli/auth0_acul_dev.html).
</Callout>

1. En el directorio de tu proyecto, edita `/src/index.css` y actualiza el widget del logotipo con la URL de tu logotipo:

`--ul-theme-widget-logo-url: "Your-Logo-URL";`

2. Guarda el cambio.

El servidor local de desarrollo se actualiza automáticamente con el cambio más reciente.

<div id="5-customize-the-context-data-for-your-authentication-screens">
  ## 5. Personaliza los datos de contexto de tus pantallas de autenticación
</div>

Con el Universal Login Context Inspector, puedes personalizar los datos de contexto predeterminados para probar tus pantallas de autenticación con configuraciones específicas del tenant.

1. Crea una versión local de los datos de contexto simulados para una pantalla de tu proyecto:

Selecciona el icono **Download JSON** para descargar el archivo de datos de contexto simulados.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/ULContextInspectorDownload.png" alt="Página de ACUL en Auth0 Dashboard" />
</Frame>

2. Cambia el nombre del archivo descargado y muévelo a `public/screens/{prompt}/{screen}`, y agrégalo al archivo `public/manifest.json` en el directorio de tu proyecto.

Por ejemplo, el archivo descargado `login-id-login-id-context.json` se renombra como `login-id.json` y se mueve a `/public/screens/login-id/login-id.json`.

3. Selecciona **Local Development** en **Data source** del Universal Login Context Inspector para cargar la versión local del contexto simulado de la pantalla.

Para obtener más información, consulta la documentación de [Auth0 CLI ACUL](https://auth0.github.io/auth0-cli/auth0_acul_dev.html).
