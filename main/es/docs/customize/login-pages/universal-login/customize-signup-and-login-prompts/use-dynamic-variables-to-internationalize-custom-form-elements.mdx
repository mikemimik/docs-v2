---
description: Aprende cómo las variables dinámicas pueden crear prompts de inicio de sesión en varios idiomas.
title: Usa variables dinámicas para internacionalizar elementos de formulario personalizados
---

Puedes renderizar [las personalizaciones del prompt de registro](/docs/es/customize/login-pages/universal-login/customize-signup-and-login-prompts) de forma diferente según los datos contextuales. Usa la variable `locale` para renderizar condicionalmente los campos del formulario y definir el comportamiento de validación.

El siguiente caso de uso utiliza la variable `locale` para renderizar un mensaje y una casilla de verificación de Términos de servicio. Puede sustituirse por cualquier variable expuesta en las [Plantillas de página](/docs/es/customize/login-pages/universal-login/customize-templates#available-variables).

<div id="prerequisites">
  ### Requisitos previos
</div>

* El tenant tiene un [Custom Domain](/docs/es/customize/custom-domains) verificado
* El tenant tiene configurado una plantilla de página

<div id="enable-fr-and-es-locales">
  ### Habilita los locales fr y es
</div>

En el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, ve a Settings &gt; General y habilita los locales:

<Frame>![](/docs/images/cdy7uua7fh8z/4yCGYTZ3RDyeULWIFcyHSv/01ce1985b1f8655aff0a0a2b4db6bee0/Language_Picker_-_EN.png)</Frame>

<div id="conditionally-render-custom-fields">
  ### Renderizar condicionalmente campos personalizados
</div>

Usa la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para agregar una plantilla parcial al contenedor del prompt de signup `form-content-end` de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>. Si el locale es `es` o `fr`, quizá quieras renderizar un mensaje de Términos de servicio y una casilla de verificación.

```liquid lines
{% if locale == 'fr' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      J'accepte les
      <a href="https://fr.example.com/tos">termes et conditions</a>
    </label>
  </div>
{% endif %}
{% if locale == 'es' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      Estoy de acuerdo con los
      <a href="https://es.example.com/tos">términos y condiciones</a>
    </label>
  </div>
{% endif %}
```

La siguiente solicitud se envía al endpoint **Establecer texto personalizado para un prompt específico** de la [Management API](https://auth0.com/docs/api/management/v2/prompts/put-custom-text-by-language):

```javascript lines
// PUT prompts/signup/partials

{
  "signup": {
    "form-content-end": "{% if locale..."
  }
}
```

Tu prompt de registro ahora muestra un mensaje de ToS y una casilla de verificación solo cuando el locale está configurado como `fr` o `es`:

<Frame>![](/docs/images/cdy7uua7fh8z/3zBadIHnkl9bIWP9BH1lLj/b2a6d9932be3bbb150b4dbbf76bc6599/2024-01-31_16-17-04.png)</Frame><Frame>![](/docs/images/cdy7uua7fh8z/3rwVd1h1Av7QzOEEWoCHhF/519a2eb5ac99c3801861c3372b81c0b9/2024-01-31_16-17-16.png)</Frame><Frame>![](/docs/images/cdy7uua7fh8z/2qYzAI6QVBLmqCW2FTAiGj/f19c59d79c67c63782b70995dbd223c3/2024-01-31_16-17-24.png)</Frame>

Para probar el resultado, ve a **Manage Dashboard** y dirígete a [Branding &gt; Universal Login &gt; Opciones de personalización](https://manage.auth0.com/dashboard/us/dev-6endizjt/universal-login/customizations/colors); luego, haz clic con el botón derecho en el **botón Try** y copia la dirección del enlace. Agrega los siguientes parámetros de consulta a la URL copiada y abre la nueva URL: `&screen_hint=signup&ui_locales=fr` (o `es`).

<div id="add-validation">
  ### Añadir validación
</div>

Cuando el `locale` sea `fr,`, puedes validar que la casilla esté marcada antes de continuar. Actualiza el fragmento de plantilla con el siguiente código de validación:

```liquid lines expandable
{% if locale == 'fr' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      J'accepte les
      <a href="https://fr.example.com/tos">termes et conditions</a>
    </label>

    <!-- NEW -->
    <div
      class="ulp-error-info"
      data-ulp-validation-function="requiredFunction"
      data-ulp-validation-event-listeners="change">
      Vous devez accepter les termes et conditions pour continuer
    </div>
    <!-- END NEW -->
  </div>
{% endif %}
{% if locale == 'es' %}
  <div class="ulp-field">
    <input
      type="checkbox"
      name="ulp-terms-of-service"
      id="terms-of-service">
    <label for="terms-of-service">
      Estoy de acuerdo con los
      <a href="https://es.example.com/tos">términos y condiciones</a>
    </label>
  </div>
{% endif %}

<!-- NEW -->
<script>
  function requiredFunction(element, formSubmitted) {
    if (! formSubmitted) {
      return true;
    }
    return element.checked;
  }
</script>
<!-- END NEW -->
```

Con esta validación, solo los usuarios con la `locale` `fr` deben dar su consentimiento a los ToS.

<Frame>![](/docs/images/cdy7uua7fh8z/7jBPuIe62PDNiIPUYzVVix/74de519659de4a8b4329165e799b1815/2024-01-31_16-20-54.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las validaciones de este caso de uso se realizan del lado del cliente y están pensadas para facilitar la experiencia del usuario. No se garantiza que se ejecuten; por ejemplo, el navegador del usuario puede tener Javascript desactivado. Además, las validaciones pueden ser modificadas por personas curiosas o malintencionadas. Para garantizar la integridad de la lógica de validación, las validaciones del lado del cliente deben combinarse con validaciones del lado del servidor.
</Callout>
