**Advanced Customizations for Universal Login (ACUL)** amplía las capacidades de [Universal Login](/es/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience#universal-login-experience) para admitir multimarca, configuraciones de seguridad complejas y la integración con numerosas plataformas de analítica.

<video autoPlay muted loop playsInline className="w-full aspect-video rounded-xl" src="/docs/images/videos/ACUL.mp4" />

Con ACUL, puedes escribir código en el framework que prefieras, integrar tus plataformas de analítica y pruebas A/B existentes, y definir la experiencia de usuario. Los [ACUL SDKs](/es/docs/customize/login-pages/advanced-customizations/reference) actúan como la conexión entre tu código personalizado y Universal Login.

<div id="benefits">
  ## Beneficios
</div>

* Amplía las capacidades de Universal Login de Auth0 para admitir varias marcas en un tenant.
* Aprovecha las funcionalidades de Auth0, como [Passkeys](/es/docs/authenticate/database-connections/passkeys), [Bot Detection](/es/docs/secure/attack-protection/bot-detection) y [MFA](/es/docs/secure/multi-factor-authentication), sin tener que rehacer tu lógica de autenticación.
* Integra analítica avanzada y pruebas A/B para optimizar la experiencia del usuario.
* Usa los ACUL SDKs para desarrollar con tu propia pila tecnológica. Puedes:
  * Usar cualquier biblioteca de CSS, JS o componentes de terceros
  * Integrar tu plataforma de analítica preferida, como Heap, Mixpanel, etc.
  * Desarrollar con tu framework de frontend preferido, como Vue o React

<div id="use-cases">
  ## Casos de uso
</div>

Te conviene usar ACUL con tu implementación de Universal Login si:

* Necesitas aplicar estándares de marca específicos a tus flujos de autenticación.
* Necesitas ofrecer una experiencia multimarca.
* Necesitas realizar pruebas A/B a nivel de interfaz de usuario u obtener análisis detallados.
* Necesitas ofrecer Universal Login localizado en idiomas que Auth0 no admite actualmente.
* Necesitas añadir lógica compleja o personalizada a tus flujos de autenticación.

Para obtener más información, consulta [ACUL Use Cases](/es/docs/customize/login-pages/advanced-customizations/acul-use-cases).

<div id="how-it-works">
  ## Cómo funciona
</div>

<img className="block dark:hidden" src="/docs/images/cdy7uua7fh8z/ACUL/Final-Light-ACUL.png" alt="Cómo funciona ACUL" />

<img className="hidden dark:block" src="/docs/images/cdy7uua7fh8z/ACUL/Final-Dark-ACUL.png" alt="Cómo funciona ACUL" />

La **fase de compilación** comienza cuando compilas tus pantallas de autenticación personalizadas en tu propio entorno.

Usa los [ACUL SDKs](/es/docs/customize/login-pages/advanced-customizations/reference) para enviar tu código personalizado y definir prompts y pantallas con la tecnología frontend que prefieras, como React o Angular. Cuando haces push de commits, tu pipeline de CI/CD ejecuta el proceso de compilación, compila tu aplicación en assets estáticos de JavaScript y CSS, y los sube a tu almacenamiento privado en la nube, que actúa como tu propia red de distribución de contenido (CDN).

La **fase de ejecución** comienza cuando tu aplicación redirige a los usuarios finales a tu página de inicio de sesión de Auth0.

Auth0 reconoce las pantallas de autenticación que configuraste para usar ACUL y sirve una página anfitriona mínima. El navegador del usuario carga esta página, que contiene etiquetas HTML que apuntan a las URL de tus assets en tu CDN. El navegador descarga tu JavaScript y CSS, verifica su integridad mediante los hashes de Subresource Integrity (SRI) y renderiza tu interfaz de usuario personalizada. A partir de ahí, el código de tu aplicación y el SDK de ACUL gestionan el resto de la interacción del usuario.

<div id="requirements">
  ## Requisitos
</div>

Para comenzar con Advanced Customizations for Universal Login, debe contar con lo siguiente:

* Un tenant de desarrollo de Auth0 configurado con [Universal Login](/es/docs/authenticate/login/auth0-universal-login) y un [dominio personalizado](/es/docs/customize/custom-domains).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le recomendamos crear un tenant para pruebas, desarrollo y producción con las etiquetas de entorno adecuadas.
</Callout>

* Una [Aplicación de primera parte](/es/docs/get-started/auth0-overview/create-applications#create-applications) de Auth0
  * Una CDN con un pipeline de CI/CD integrado, como GitHub, para alojar assets.
