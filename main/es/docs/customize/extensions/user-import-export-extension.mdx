description: Obtén información sobre la extensión User Import/Export, que te permite importar y exportar usuarios desde o hacia cualquier base de datos que hayas configurado en tu cuenta.
title: Extensión User Import / Export
---

<Warning>
  La **extensión User Import/Export** quedará obsoleta en septiembre de 2025. Ya puedes acceder a las mismas funcionalidades [directamente desde el Auth0 Management Dashboard](/es/docs/manage-users/user-migration/bulk-user-import-export).
</Warning>

La **extensión User Import / Export** te permite:

* Importar en bloque los usuarios de tu base de datos existente a Auth0
* Buscar y exportar algunos (o todos) los usuarios de tu base de datos de Auth0

Para ver una lista de los campos del perfil de usuario que se pueden importar y exportar, consulta [Atributos del perfil de usuario](/es/docs/manage-users/user-accounts/user-profiles/user-profile-structure).

Debes ser administrador del Dashboard para usar esta extensión.

<div id="install-the-extension">
  ## Instala la extensión
</div>

Para instalar esta extensión, ve a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y selecciona **User Import / Export**.

La extensión no requiere ninguna configuración adicional antes de instalarse, así que selecciona **Install** en la ventana emergente informativa para continuar.

<div id="use-the-extension">
  ## Usa la extensión
</div>

Después de instalar la extensión, la verás en tu lista de extensiones instaladas. Haz clic en **Import / Export Extension** para iniciarla.

La primera vez que inicies la extensión, se te pedirá que concedas permiso para que acceda a tu cuenta de Auth0 y realice las actividades indicadas.

<Frame>![Dashboard Extensions Autorizar la extensión Users Import/Export](/docs/images/cdy7uua7fh8z/5CCKmxUv1HN5nDj9jL8qel/d7f02ccf3f4be166b00bbf217e14144f/Screen_Shot_2021-05-18_at_9.11.01_PM.png)</Frame>

Haz clic en la **marca de verificación** para continuar.

Hay dos formas de usar esta extensión:

* Importar masivamente tus usuarios existentes de la base de datos a Auth0
* Buscar y exportar algunos o todos tus usuarios de la base de datos de Auth0

Ambos casos de uso se explican con más detalle a continuación.

<div id="import-users">
  ### Importar usuarios
</div>

De forma predeterminada, cada vez que abras la extensión, verás la pantalla de **Importación de usuarios** (si estás en la pantalla de exportación, puedes volver a esta pantalla haciendo clic en **Importar** en la barra de navegación de la izquierda).

Para importar tus usuarios, arrastra y suelta un archivo JSON válido ([esquema y ejemplos aquí](/es/docs/manage-users/user-migration/bulk-user-import-database-schema-and-examples)) en el área que dice **Suelta tu archivo aquí o haz clic para seleccionarlo**. También puedes hacer clic en esta área para buscar entre tus archivos y seleccionar el archivo JSON correspondiente. El archivo JSON debe contener la lista de usuarios que deseas importar.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si estás usando un archivo de exportación de un tenant de Auth0, debes convertir el archivo exportado de `ndjson` a [JSON](/es/docs/manage-users/user-migration/bulk-user-exports#json-compatible-format). Para conservar los mismos ID de usuario, debes eliminar el `prefijo auth0|` de todos los ID de usuario importados.

  El proceso de importación agrega automáticamente el `prefijo auth0|` a los ID de usuario importados. Si no eliminas el prefijo `auth0|` antes de importar, los ID de usuario aparecerán como `auth0|auth0|...`
</Callout>

Selecciona la conexión de base de datos en la que se importarán tus usuarios. Asegúrate de que la conexión que elijas esté habilitada para al menos una aplicación.

Haz clic en **Comenzar a importar usuarios** para iniciar el proceso de importación.

Cuando termine, verás el siguiente mensaje de **Completado**.

<Frame>![Dashboard Extensions Users Import Export Import Complete](/docs/images/cdy7uua7fh8z/7hyE3DWBJuFuAulAAnZlfN/23303644d4c706f4c0937bd363bf6699/import-complete.png)</Frame>

Una vez que hayas importado tus usuarios, puedes administrarlos individualmente desde [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).

<div id="export-users">
  ### Exportar usuarios
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 usa el formato ndjson debido al gran tamaño de los archivos de exportación. Antes de poder importar usuarios, tendrás que convertir **ndjson** a **json** con la biblioteca que prefieras (como [jq](https://stedolan.github.io/jq/)). Al exportar usuarios con la intención de importarlos más adelante, los nombres de los campos de usuario deben dejarse con sus valores predeterminados y no asignarse a un Nombre de columna.
</Callout>

Para exportar tus usuarios existentes de Auth0 asociados a conexiones de base de datos, selecciona **Exportar**.

<Frame>![Página Export Users de Dashboard Extensions Users Import Export](/docs/images/cdy7uua7fh8z/5wUhbGMjVQKIE9vx2SBIA3/a952aee6cc6027234631334c7174c4a8/export-users.png)</Frame>

En **Campos de usuario**, puedes decidir qué atributos o expresiones de usuario deben incluirse en la exportación. El atributo de usuario puede ser un valor estático como `user.user_metadata.name` o una expresión de JavaScript como `user.user_metadata.name || user.name`. Las expresiones se evaluarán durante la exportación. El valor de **nombre de columna** determina cómo se representará el valor en la exportación.

Puedes hacer clic en el botón **Agregar campos predeterminados** para seleccionar automáticamente los campos predeterminados y completar sus nombres de columna (esta también es una buena forma de visualizar cómo aparecerán los parámetros o las expresiones).

Puedes eliminar atributos o expresiones innecesarios haciendo clic en el icono de **papelera** correspondiente.

En **Configuración**, puedes:

* Configurar cómo se muestran tus usuarios exportados indicando un **Atributo de usuario** por el que deben ordenarse (así como si deben ordenarse en orden ascendente o descendente)
* Elegir tu **Formato de exportación**; puedes elegir entre archivos JSON y CSV

  <Frame>![Formato de exportación de configuración de Dashboard Extensions Users Import Export](/docs/images/cdy7uua7fh8z/1HVpo1IevSRJbFvwrEKkzf/761fa914f6c33cdee147e07b2c43f7c7/settings.png)</Frame>

Cuando estés listo, haz clic en **Exportar X usuarios** (donde `X` es la cantidad de usuarios que estás exportando).

Puedes descargar el archivo que contiene tus usuarios cuando se complete la exportación; es posible que necesites usar una herramienta de extracción de terceros si usas Windows 10 o una versión anterior.

<div id="update-users">
  ## Actualizar usuarios
</div>

La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0 permite actualizar masivamente los perfiles de usuario en `user_metadata` y `app_metadata` con la extensión User Import/Export.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Actualmente, Auth0 no admite la importación masiva para Passwordless. Usa la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) para actualizar los usuarios individualmente.
</Callout>

<div id="export-users">
  ### Exportar usuarios
</div>

Usa el [proceso de exportación anterior](/es/docs/customize/extensions/user-import-export-extension#export-users) para crear un archivo de exportación completo que incluya todos los atributos de tus usuarios.

Elige el formato JSON para la exportación.

Cuando la exportación se complete, descarga el archivo que contiene a tus usuarios.

<div id="update-your-configuration">
  ### Actualiza la configuración
</div>

Haz una **copia** del archivo JSON que descargaste en el paso anterior.

<Warning>
  Este proceso puede provocar la pérdida de datos. Crea una copia de seguridad de tu base de datos de usuarios en caso de que los datos se corrompan.
</Warning>

Abre la copia del archivo JSON en un editor de texto. El esquema de usuario debería ser similar al siguiente:

```json lines expandable
[
   {
      "Id":"auth0|626737a9813c000069daf462",
      "Nickname":"testuser1",
      "Name":"testuser1@auth0.com",
      "email":"testuser1@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-04-26T00:07:05.098Z",
      "Updated At":"2022-07-06T14:54:19.214Z"
   },
   {
      "Id":"auth0|62d1a1853bdcf11ff5dca672",
      "Nickname":"testuser2",
      "Name":"testuser2@auth0.com",
      "email":"testuser2@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-07-15T17:19:01.531Z",
      "Updated At":"2022-07-15T17:19:01.532Z"
   },
   {
      "Id":"auth0|62d1a1a524f32a4185c9ce23",
      "Nickname":"testuser3",
      "Name":"testuser3@auth0.com",
      "email":"testuser3@auth0.com",
      "Connection":"Username-Password-Authentication",
      "Created At":"2022-07-15T17:19:33.484Z",
      "Updated At":"2022-07-15T17:19:33.484Z"
   }
]
```

No cambie la dirección de correo electrónico del usuario, ya que es la clave de búsqueda de los usuarios en la base de datos.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El Id debe tener un `connection|id` único. Si el mismo Id de usuario existe en varias conexiones, la actualización no se realizará.
</Callout>

Para agregar un atributo, cree un campo nuevo en cada esquema de usuario y actualícelo con un valor. En el siguiente ejemplo, el atributo &quot;Family Name&quot; tiene el valor &quot;Smith&quot;, con una coma después del valor del nuevo atributo.

También puede agregar el atributo `email_verified` a varios usuarios y establecer su valor en `true`.

```json lines
{
   "Id":"auth0|626737a9813c000069daf462",
   "Nickname":"testuser1",
   "Family Name":"Smith",
   "Name":"testuser1@auth0.com",
   "email":"testuser1@auth0.com",
   "email_verified":true,
   "Connection":"Username-Password-Authentication",
   "Created At":"2022-04-26T00:07:05.098Z",
   "Updated At":"2022-07-06T14:54:19.214Z"
}
```

Guarda los cambios en tu archivo de usuarios.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este proceso no permite bloquear a varios usuarios a la vez. Debes bloquearlos individualmente en el Auth0 Dashboard o con la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).
</Callout>

<div id="import-updated-user-profile-data">
  ### Importar datos actualizados del perfil de usuario
</div>

El endpoint `users-import` te permite cargar tu archivo actualizado de perfiles de usuario y actualizar varios usuarios en un solo proceso. Ve al [endpoint Create import users job](https://auth0.com/docs/api/management/v2/#!/Jobs/post_users_imports).

Agrega tu archivo de usuarios al parámetro `users` y el ID de conexión de la base de datos que quieres actualizar.

El valor de `upsert` debe ser `true` para actualizar la información de usuarios que ya existen.
Para aplicar las actualizaciones, selecciona **Try** o usa Postman para importar los usuarios.

Usa el `job_id` para verificar el estado con el endpoint `jobs` [endpoint](https://auth0.com/docs/api/management/v2/#!/Jobs/get_jobs_by_id).
