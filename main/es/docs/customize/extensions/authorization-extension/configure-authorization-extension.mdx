<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 ofrece dos formas de implementar el control de acceso basado en roles (RBAC), que puedes usar en lugar del sistema interno de control de acceso de tu API o junto con él:

  * [Authorization Core](/es/docs/manage-users/access-control/rbac)
  * [Authorization Extension](/es/docs/customize/extensions/authorization-extension)

  El conjunto de funcionalidades de Authorization Core equivale al de Authorization Extension, mejora el rendimiento y la escalabilidad, y ofrece un sistema RBAC más flexible que Authorization Extension.

  Actualmente, ambos implementan las funcionalidades clave de RBAC y te permiten restringir los scopes personalizados definidos para una API a aquellos que se han asignado al usuario como permisos.
</Callout>

Antes de que la extensión pueda aplicar tu lógica de autorización, debes configurar cómo se comportará durante la transacción de inicio de sesión. La configuración quedará recogida en una [rule](/es/docs/customize/rules) que se ejecuta durante el runtime.

<div id="configure-extension">
  ## Configurar la extensión
</div>

1. Ve a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y selecciona **Auth0 Authorization**.
2. Elige **Configuración** en la esquina superior derecha del **Authorization Dashboard**.

   <Frame>![Dashboard - Dashboard de Authorization Extension - Configuración](/docs/images/cdy7uua7fh8z/r8Ic6OYMcVfXDvIKePLrb/ed9c75536d20c12149e8b8e62a8edc44/click-configuration.png)</Frame>
3. Accederás a la sección **Configuración de Rule** de la página **Configuración**. Todos los cambios que hagas en las secciones de **Contenido del token**, como los relacionados con grupos, roles y permisos, se reflejarán en la rule que exportes al finalizar este paso.

   <Frame>![Configuración de Authorization Extension](/docs/images/cdy7uua7fh8z/4ydHJBdKGs4XVLnRst66A3/1f61ad2e087252c6c3aa10add13d99cd/configuration0auth.png)</Frame>

<div id="apikey">
  ### ApiKey
</div>

La rule usa la ApiKey para comunicarse con la API de Authorization Extension y obtener la policy. La ApiKey se almacena como configuración de la rule y se creará automáticamente cuando se publique la rule. Para rotar la ApiKey, haz clic en el botón **Rotate**; cuando se rote la ApiKey, la configuración de la rule se actualizará automáticamente.

<div id="add-authorization-information-to-the-token-issued">
  ## Agregar información de autorización al token emitido
</div>

Puedes almacenar datos de autorización, como grupos, roles o permisos, en el token emitido por Auth0. Tu aplicación podrá usar esta información inspeccionando el token y realizar las acciones adecuadas según el contexto de autorización actual del usuario.

Para agregar información de grupos, roles y/o permisos al token emitido, activa el control deslizante junto a la opción que quieras incluir.

<Warning>
  Almacenar demasiados datos en el token puede causar problemas de rendimiento o incluso impedir su emisión. Asegúrate de almacenar solo lo necesario. Si necesitas tener disponible de inmediato una gran cantidad de datos de usuario, considera usar persistencia en lugar de agregar esos datos al token.
</Warning>

<div id="merge-authorization-data-from-idp">
  ### Combina los datos de autorización del IdP
</div>

Puede que tengas usuarios que reciban grupos, roles o permisos del <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip> (IdP) que estés usando, como Active Directory. Si quieres combinar estos elementos (para conservarlos) con los definidos en la Authorization Extension, asegúrate de habilitar las opciones de **Passthrough** correspondientes. Activa el control deslizante junto a cada combinación que quieras habilitar.

<div id="store-authorization-information-in-user-profiles">
  ### Almacenar información de autorización en los perfiles de usuario
</div>

Si tu contexto de autorización es amplio (por ejemplo, el usuario puede pertenecer a muchos grupos o tener asignados muchos permisos), puede resultarte útil almacenar parte de la información de autorización en los perfiles de los usuarios. Esto te permite almacenar menos información en el token, por lo que es menos probable que surjan problemas de rendimiento o incluso problemas con la emisión de tokens. La **persistencia** es el proceso mediante el cual almacenas información sobre grupos, roles y permisos en los perfiles de los usuarios.

Los datos se almacenarán en el campo `app_metadata` del usuario, y luego puedes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> o el [Dashboard](https://manage.auth0.com/#/users) para recuperar esta información después de que el usuario haya iniciado sesión.

<Frame>![Authorization Extension - configuración - Persistencia](/docs/images/cdy7uua7fh8z/5kfgcLhwwZkQ2Oh5uxaGxF/de5cf05429d1d1730a0165f4c61ac629/persistence.png)</Frame>

<div id="save-changes-to-rule">
  ## Guardar cambios en la rule
</div>

Una vez que hayas configurado tu rule, haz clic en **Publish Rule**. Esto crea una rule para tu tenant que se ejecuta después de cada inicio de sesión del usuario.

<div id="view-rule">
  ### Ver una rule
</div>

Si quieres ver la rule que has creado, puedes hacerlo en [Auth0 Dashboard &gt; pipeline de autenticación &gt; Rules](https://manage.auth0.com/#/rules).

Puedes abrir la rule para ver su configuración exacta.
