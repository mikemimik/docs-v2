Si eres un usuario con el rol **Delegated Admin - Administrator**, puedes administrar los distintos Extension Hooks y consultas que te permiten personalizar el comportamiento de la extensión de Administración delegada.

Para acceder al área de configuración:

1. Inicia sesión en el Dashboard de administración delegada.
2. Haz clic en tu nombre, en la esquina superior derecha. Verás un menú desplegable; haz clic en la opción **Configure**.

En la página de **configuración** a la que se te redirige, puedes administrar tus Hooks y consultas.

<div id="extension-hooks-context">
  ## Contexto de Extension Hooks
</div>

El objeto de contexto (**ctx**) expondrá algunas utilidades e información sobre la solicitud actual. Los siguientes métodos y propiedades están disponibles en cada Extension Hook:

* Registro
* Almacenamiento en caché
* Custom Data
* Payload y solicitud
* Llamadas remotas

<div id="logging">
  ### Registro
</div>

Para añadir un mensaje a los logs en tiempo real de Actions (que puedes consultar con logs en tiempo real de Actions), llama al método **log**:

```javascript lines
function(ctx, callback) {
  ctx.log('Logging action:', ctx.payload.action);
  return callback();
}
```

<div id="caching">
  ### Almacenamiento en caché
</div>

Para almacenar algo en caché (como una lista larga de departamentos), puedes guardarlo en el objeto **global** del contexto. Este objeto estará disponible hasta que se reinicie el contenedor de logs en tiempo real de Actions.

```javascript lines
function(ctx, callback) {
  ctx.global.departments = [ 'IT', 'HR', 'Finance' ];
  return callback();
}
```

<div id="custom-data">
  ### Custom Data
</div>

Puede almacenar datos personalizados en la extensión. Este campo está limitado a 400 KB de datos.

```javascript lines
var data = {
departments: [ 'IT', 'HR', 'Finance' ]
};

ctx.write(data)
.then(function() {
    // ...
})
.catch(function(err) {
    // ...
});
```

Para leer los datos:

```js lines
ctx.read()
.then(function(data) {
    // ...
})
.catch(function(err) {
    // ...
});
```

<div id="payload-and-request">
  ### Payload y solicitud
</div>

Cada Extension Hook expone el payload o la solicitud actual con información específica. La solicitud siempre contendrá información sobre el usuario que ha iniciado sesión en el Dashboard de Users:

```javascript lines
function(ctx, callback) {
  ctx.log('Current User:', ctx.request.user);
  return callback();
}
```

<div id="remote-calls">
  ### Llamadas remotas
</div>

Si quieres llamar a un servicio externo (como una API) para validar datos o cargar afiliaciones, puedes hacerlo con el módulo `request`.

```javascript lines
function(ctx, callback) {
var request = require('request');
    request('http://api.mycompany.com/departments', function (error, response, body) {
        if (error) {
        return callback(error);
        }

        // ...
    });
}
```

<div id="hook-contract">
  ### Contrato del Hook
</div>

* `ctx`: El objeto de contexto

  * `payload`: El objeto `payload`

    * `action`: La acción actual (por ejemplo, `delete:user`) que se está ejecutando
    * `user`: El usuario sobre el que se está ejecutando la acción
* `callback(error)`: La devolución de llamada a la que puedes pasar un error si se deniega el acceso

<div id="sample-use">
  ### Ejemplo de uso
</div>

Kelly gestiona el departamento de Finanzas y solo debería poder acceder a los usuarios de su propio departamento.

```javascript lines
function(ctx, callback) {
  if (ctx.payload.action === 'delete:user') {
    return callback(new Error('You are not allowed to delete users.'));
  }

  // Obtener el departamento de los metadatos del usuario actual.
  var department = ctx.request.user.app_metadata && ctx.request.user.app_metadata.department;
  if (!department || !department.length) {
    return callback(new Error('The current user is not part of any department.'));
  }

  // El departamento de TI puede acceder a todos los usuarios.
  if (department === 'IT') {
    return callback();
  }

  ctx.log('Verifying access:', ctx.payload.user.app_metadata.department, department);

  if (!ctx.payload.user.app_metadata.department || ctx.payload.user.app_metadata.department !== department) {
    return callback(new Error('You can only access users within your own department.'));
  }

  return callback();
}
```

Si este hook no está configurado, se podrá acceder a todos los usuarios.

Nombres de acciones admitidos:

* `read:user`
* `delete:user`
* `reset:password`
* `change:password`
* `change:username`
* `change:email`
* `read:devices`
* `read:logs`
* `remove:multifactor-provider`
* `block:user`
* `unblock:user`
* `send:verification-email`
