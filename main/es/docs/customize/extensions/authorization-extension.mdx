<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El conjunto de funcionalidades de [Authorization Core](/es/docs/manage-users/access-control/rbac) y [Authorization Extension](/es/docs/customize/extensions/authorization-extension) son características totalmente independientes. Para administrar grupos, roles o permisos, debes usar la funcionalidad en la que se crearon originalmente.
</Callout>

Authorization Extension ofrece compatibilidad con la autorización de usuarios mediante grupos, roles y permisos. Puedes definir el comportamiento esperado durante el proceso de inicio de sesión, y la configuración quedará reflejada en una regla que se ejecuta en tiempo de ejecución. Para obtener más información, consulta [Auth0 Rules](/es/docs/customize/rules).

Con Authorization Extension, puedes almacenar datos de autorización, como grupos, roles o permisos, en el token emitido por Auth0. Tu aplicación puede usar esta información inspeccionando el token y realizar las acciones adecuadas en función del contexto de autorización actual del usuario.

Con Authorization Extension, los roles y permisos se establecen por aplicación. Si necesitas los mismos roles o permisos en otra aplicación, tendrás que crearlos por separado. En cambio, el conjunto de funcionalidades de [Authorization Core](/es/docs/manage-users/access-control/rbac) ofrece mucha más flexibilidad con roles y permisos.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de poder usar la extensión, deberás:

1. [Instalar](/es/docs/customize/extensions/authorization-extension/install-authorization-extension) la extensión.
2. [Configurar](/es/docs/customize/extensions/authorization-extension/configure-authorization-extension) cómo se comportará la extensión durante el proceso de inicio de sesión.
3. [Configurar](/es/docs/customize/extensions/authorization-extension/set-up-authorization-extension-users) usuarios, grupos, roles y permisos.

<div id="manage-data">
  ## Gestionar datos
</div>

Puedes transferir datos fácilmente a la extensión o desde ella mediante un archivo JSON. Para obtener más información, consulta [Importar y exportar datos de Authorization Extension](/es/docs/customize/extensions/authorization-extension/import-and-export-authorization-extension-data).

<div id="add-functionality">
  ## Agregar funcionalidades
</div>

Una vez que la extensión esté en funcionamiento, puedes añadirle funcionalidades adicionales.

* [Habilitar el acceso a la API de la extensión](/es/docs/customize/extensions/authorization-extension/enable-api-access-to-authorization-extension): Aprende a automatizar el aprovisionamiento y a consultar en tiempo real el contexto de autorización de tus usuarios mediante la API de la extensión.
* [Explorar los endpoints de la API de Authorization Extension](https://auth0.com/docs/api/authorization-extension): Conoce los endpoints de la API de Authorization Extension y cómo utilizarlos.
* [Usar datos de Authorization Extension en reglas](/es/docs/customize/extensions/authorization-extension/use-rules-with-the-authorization-extension): Aprende a usar reglas para configurar lógica adicional en tus inicios de sesión.

<div id="upgrade-from-previous-versions">
  ## Actualizar desde versiones anteriores
</div>

<div id="upgrade-from-version-26-or-later">
  ### Actualización desde la versión 2.6 o posteriores
</div>

Las actualizaciones desde la versión 2.6 o posteriores no introducen cambios incompatibles y no requieren ninguna acción especial.

<div id="upgrade-from-versions-before-26">
  ### Actualización desde versiones anteriores a la 2.6
</div>

Authorization Extension 2.6 incluye cambios importantes derivados de una modificación en la lógica de almacenamiento y gestión de la API Key; por ello, al actualizar, debes realizar pasos adicionales, como se detalla a continuación. Si no completas estos pasos, al ejecutar la regla se producirá un error `InvalidApiKey` o `You are not allowed to access this application`. Para obtener más información, consulta el [registro de cambios en GitHub](https://github.com/auth0/auth0-authorization-extension/blob/master/CHANGELOG.md).

1. Ve a [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) y selecciona la vista **Installed Extensions**.
2. Busca **Auth0 Authorization**, selecciona **Upgrade** y confirma. Espera a que se complete la actualización.

<div id="rotate-the-extensions-api-key">
  #### Rota la clave de API de la extensión
</div>

1. Selecciona **Auth0 Authorization** para abrir la extensión.
2. En el menú desplegable de la esquina superior derecha del panel de la extensión, selecciona **Configuration**.
3. Busca la sección **API Key** y selecciona **Rotate**.

<div id="republish-the-extensions-rule">
  #### Volver a publicar la regla de la extensión
</div>

Selecciona **Publicar regla**.

<div id="delete-the-old-extension-rule-if-it-exists">
  #### Elimina la regla de extensión anterior si existe
</div>

1. Ve a [Auth0 Dashboard &gt; Auth Pipeline &gt; Rules](https://manage.auth0.com/#/rules).
2. Busca la regla `auth0-authz`. Si no existe, ya has terminado; de lo contrario, continúa con los siguientes pasos.
3. Busca la regla `auth0-authorization-extension` y arrástrala a la posición debajo de la regla `auth0-authz`.
4. Comprueba que la regla `auth0-authz`:

   * Fue creada por la Authorization Extension y no se ha modificado manualmente
   * No cambiará el flujo de autorización de forma que conceda acceso o privilegios a usuarios no deseados si se elimina
5. Si se cumplen las condiciones anteriores, usa el interruptor para deshabilitar la regla `auth0-authz`. Después de verificar que todo funciona correctamente, puedes decidir si dejar la regla deshabilitada o eliminarla por completo.
