---
title: Personaliza los mensajes de SMS y voz de la autenticación multifactor
description: Aprende a personalizar los mensajes de SMS y voz que Auth0 envía durante la inscripción y la verificación.
---

Para personalizar los mensajes de SMS y voz de la autenticación multifactor que envía Auth0:

1. Ve a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa) y haz clic en **Phone Message**.
2. En la sección **Plantilla de inscripción** o **Plantilla de verificación**, personaliza el mensaje.
3. Haz clic en **Save**.

La **Plantilla de inscripción** define el mensaje que se envía cuando un usuario inscribe un dispositivo nuevo por primera vez con MFA. La **Plantilla de verificación** define el mensaje que se envía cuando un usuario inicia sesión después de la inscripción.

<div id="syntax">
  ## Sintaxis
</div>

La sintaxis de [Liquid](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) es el motor de plantillas admitido para acceder a los atributos del usuario en las plantillas de SMS. Los siguientes atributos están disponibles:

| Atributo                        | Descripción                                                                                                                                                                                                                   |
| ------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `message_type`                  | Indica qué tipo de mensaje se envía: `sms` o `voice`.                                                                                                                                                                         |
| `code`                          | Código de inscripción/verificación. Al enviar mensajes de voz, los valores de las variables se separan con puntos entre los dígitos (`1.2.3.4.5`) para que los proveedores de mensajería de voz los pronuncien correctamente. |
| `locale`                        | Para Universal Login o API de MFA. Indica el idioma.                                                                                                                                                                          |
| `requestInfo.lang`              | Para la localización de Classic Login. Corresponde al encabezado accept-language del navegador. Por ejemplo: `es-AR,es;q=8.0`, `en-US,en`.                                                                                    |
| `tenant.friendlyName`           | El **nombre descriptivo** configurado en **Dashboard &gt; Configuración del tenant**.                                                                                                                                         |
| `custom_domain.domain`          | El nombre de dominio del tenant.                                                                                                                                                                                              |
| `custom_domain.domain_metadata` | Los campos de metadata del dominio personalizado (como pares clave-valor).                                                                                                                                                    |

Un ejemplo de cómo usar la sintaxis Liquid para enviar distintos mensajes de voz y SMS según el idioma:

```liquid lines
{% if message_type == "voice" %}
    {% if locale contains "fr" %}
        Bonjour, vous avez demandé à recevoir un code de vérification pour vous enregister avec {{tenant.friendly_name}}. Votre code est: {{pause}} {{code}}. Je répète, votre code est: {{pause}}{{code}}.
    {% elsif locale contains "es" %}
        Usted ha requerido un código de verificación para inscribirse con {{tenant.friendly_name}}. Su código es: {{pause}}{{code}}. Repito, su código es: {{pause}}{{code}}.
    {% else %}
        Hello, you requested a verification code to enroll with {{tenant.friendly_name}}. Your code is: {{pause}}{{code}}. I repeat, your code is: {{pause}}{{code}}.
    {% endif %}
{% else %}
    {% if locale contains "fr" %}
        {{code}} est votre code de vérification pour vous enregistrer avec {{tenant.friendly_name}}.
    {% elsif locale contains "es" %}
        {{code}} es su código para inscribirse con {{tenant.friendly_name}}.
    {% else %}
        {{code}} is your verification code to enroll with {{tenant.friendly_name}}.
    {% endif %}
{% endif %}
```

<div id="localization">
  ## Localización
</div>

En la Auth0 Authentication API, el [endpoint Get Code or Link](https://auth0.com/docs/api/authentication/passwordless/get-code-or-link), el [endpoint de solicitud de desafío](https://auth0.com/docs/api/authentication/muti-factor-authentication/request-mfa-challenge) de MFA y el [endpoint para agregar un autenticador](https://auth0.com/docs/api/authentication/muti-factor-authentication/add-an-authenticator) de MFA admiten el encabezado `x-request-language`.

Para usarlo, configura tus idiomas admitidos en **Tenant &gt; Settings** (en **Supported Languages**). Cuando se envía el encabezado `x-request-lanuage`, esa configuración regional está disponible en la variable `locale` de las plantillas de mensajes telefónicos de MFA. Consulta [Compatibilidad multilingüe de Passwordless](/es/docs/authenticate/passwordless/authentication-methods/sms-otp#multi-language-support) para obtener más información.
