Forms te permite añadir nodos y componentes, como campos, bloques y widgets, para que los usuarios interactúen con el formulario. Luego, puedes hacer referencia a los datos introducidos en los componentes en tus formularios y flujos.

<div id="form-nodes">
  ## Nodos de formulario
</div>

Hay cinco tipos de nodos de formulario: Inicio, Paso, Flujo, Router y Pantalla de finalización. Estos nodos cumplen una función muy específica en la estructura del formulario.

<Frame>![Dashboard >  Formularios > Editor de formularios](/docs/images/cdy7uua7fh8z/5V4GsHxxC2cjxx1aODCxPd/309850c72bbce224e7a8d3f5af34114c/form-editor-preview.png)</Frame>

<div id="step-node">
  ## Nodo de paso
</div>

Los nodos de paso son la representación visual del formulario. Pueden contener cualquier cantidad de componentes, como campos, bloques y widgets en los que el usuario puede completar sus datos.

<Frame>![Dashboard > Foms > Nodo de paso](/docs/images/cdy7uua7fh8z/2tbkcXkVon0BawCvYgnvIz/e2ce566320af18a6b08702f04f763126/form-step-node.png)</Frame>

<div id="field-components">
  ### Componentes de campo
</div>

Los campos son componentes de la interfaz de usuario en los que los usuarios introducen su información en Forms.

<Frame>![Dashboard > Forms > Campos](/docs/images/cdy7uua7fh8z/6fuBdJcbDybzcVaiByUkoQ/2ad29c8dbbcc6ddf850f5efc004a75e0/forms-fields.png)</Frame>

<div id="field-settings">
  #### Configuración de campos
</div>

Los campos tienen una serie de configuraciones predeterminadas que puedes ajustar según tus necesidades.

La configuración predeterminada de los campos es la siguiente:

| Configuración                 | Descripción                                  |
| ----------------------------- | -------------------------------------------- |
| ID                            | El identificador único                       |
| Etiqueta                      | Mostrar u ocultar una etiqueta               |
| Obligatorio                   | El valor de entrada es obligatorio           |
| Texto de ayuda                | Mostrar un texto de ayuda                    |
| Texto de marcador de posición | Texto visible para el usuario                |
| Valor predeterminado          | Valor predeterminado visible para el usuario |
| Transitorio                   | Habilitar o deshabilitar la máscara de datos |

<div id="field-validation-rules">
  #### Reglas de validación de campos
</div>

| Configuración           | Descripción                                                   |
| ----------------------- | ------------------------------------------------------------- |
| Long. mín. / Long. máx. | Exige una longitud mínima o máxima para el valor introducido. |
| Valor mín. / Valor máx. | Exige un valor numérico mínimo o máximo.                      |

<div id="available-fields-in-forms">
  #### Campos disponibles en Forms
</div>

Los campos disponibles, junto con su configuración específica y sus tipos de salida de datos, son:

<AccordionGroup>
  <Accordion title="Campo de texto">
    La entrada estándar permite que un usuario introduzca cualquier valor de texto.

    ##### Configuración del campo de texto

    La configuración del campo de texto es la siguiente:

    | Configuración | Descripción                                    |
    | ------------- | ---------------------------------------------- |
    | Multiline     | Habilita la entrada de texto en varias líneas. |

    ##### Valor de salida del campo de texto

    El tipo de datos del valor de salida del campo de texto es `string`.

    ```json lines
    {
      "text_field_id": "Auth0"
    }
    ```
  </Accordion>

  <Accordion title="Campo de correo electrónico">
    La entrada estándar permite que un usuario introduzca valores de correo electrónico como cadenas de texto.

    ##### Valor de salida del campo de correo electrónico

    El tipo de dato del valor de salida del campo de correo electrónico es un `string`.

    ```json lines
    {
      "email_field_id": "username@domain.com"
    }
    ```
  </Accordion>

  <Accordion title="Campo de teléfono">
    La entrada estándar permite a los usuarios introducir números de teléfono.

    ##### Ajustes del campo de teléfono

    Los ajustes del campo de teléfono son:

    | Settings                                  | Description                                                                                                    |
    | ----------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
    | Selector de país                          | Activa o desactiva el selector de país. De forma predeterminada, muestra la ubicación del usuario según su IP. |
    | Texto del marcador de posición del filtro | Texto visible para el usuario en la ventana de búsqueda de países.                                             |

    ##### Valor de salida del campo de teléfono

    El tipo de dato del valor de salida del campo de teléfono es `string` o, si el selector de país está habilitado, un `object` .

    El valor de salida del campo de teléfono como `string`:

    ```json lines
    {
      "phone_field_id": "8005550175"
    }
    ```

    El valor del tipo de datos de salida del campo de teléfono es un `object`:

    ```json lines
    {
      "phone_field_id": {
        "national_number": "8005550175",
        "national_format": "(800) 555-0175",
        "international_number": "+18005550175",
        "international_format": "+1 800-555-0175",
        "country_code_iso": "US",
        "country_code_number": "1"
      }
    }
    ```
  </Accordion>

  <Accordion title="Campo de opciones">
    Entrada estándar que permite a un usuario seleccionar valores de opción única o múltiple.

    ##### Configuración del campo de selección

    La configuración del campo de selección es:

    | Configuración                  | Descripción                                                                  |
    | ------------------------------ | ---------------------------------------------------------------------------- |
    | Opción múltiple                | Permite habilitar varias opciones.                                           |
    | Opción &quot;Otro&quot;        | Permite a los usuarios introducir su propia opción.                          |
    | Advanced &gt; Valores internos | Asigna un valor interno a cada opción.                                       |
    | Advanced &gt; Edición masiva   | Permite editar en bloque la etiqueta y los valores internos de las opciones. |

    ##### Valor de salida del campo de selección

    El tipo de dato del valor de salida del campo de selección es un `string` o, si la opción múltiple está habilitada, un `array of strings`.

    El valor de salida del campo de selección como `string`:

    ```json lines
    {
      "choice_field_id": "Option A"
    }
    ```

    El valor del tipo de datos de salida del campo de selección es un `array of strings`:

    ```json lines
    {
      "choice_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Campo de tarjetas">
    Campo de entrada estándar que permite al usuario seleccionar uno o varios valores de imagen.

    ##### Configuración del campo de tarjetas

    La configuración del campo de tarjetas es la siguiente:

    | Configuración                  | Descripción                            |
    | ------------------------------ | -------------------------------------- |
    | Opción múltiple                | Permite seleccionar varias opciones.   |
    | Ocultar etiquetas              | Oculta las etiquetas de las tarjetas.  |
    | Avanzado &gt; Valores internos | Asigna un valor interno a cada opción. |

    ##### Valor de salida del campo de tarjetas

    El tipo de dato del valor de salida del campo de tarjetas es un `string` o, si la opción múltiple está habilitada, un `array of strings`.

    El valor de salida del campo de tarjetas como `string`:

    ```json lines
    {
      "card_field_id": "Option A"
    }
    ```

    El campo de tarjetas devuelve un valor del tipo de dato `array of strings`:

    ```json lines
    {
      "card_field_id": ["Option A", "Option B"]
    }
    ```
  </Accordion>

  <Accordion title="Campo URL">
    La entrada estándar permite al usuario introducir un valor de URL.

    ##### Valor de salida del campo URL

    El tipo de dato del valor de salida del campo URL es una cadena:

    ```json lines
    {
      "url_field_id": "https://auth0.com"
    }
    ```
  </Accordion>

  <Accordion title="Campo de contraseña">
    Entrada estándar que permite al usuario introducir una contraseña o un valor secreto.

    ##### Configuración del campo de contraseña

    La configuración del campo de contraseña es la siguiente:

    | Configuración                    | Descripción                                                                                                                                                                                                                             |
    | -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Requerir contraseña segura       | Las contraseñas deben contener una letra mayúscula, una minúscula, un símbolo y un número.                                                                                                                                              |
    | Aplicar las directrices del NIST | La contraseña no puede tener menos de 8 caracteres ni ser una contraseña obtenida de filtraciones anteriores, una palabra del diccionario, una secuencia de caracteres repetidos o consecutivos, ni palabras específicas del contenido. |
    | Añadir medidor de seguridad      | Se muestra un medidor gráfico con una escala roja, amarilla y verde debajo del campo de contraseña para indicar la solidez de la contraseña introducida.                                                                                |
    | Hash                             | Seleccione un algoritmo para aplicar una función hash al valor introducido directamente en el navegador.                                                                                                                                |

    ##### Valor de salida del campo de contraseña

    El tipo de dato del valor de salida del campo de contraseña es una `string` enmascarada:

    ```json lines
    {
      "password_field_id": "███"
    }
    ```
  </Accordion>

  <Accordion title="Campo de pago">
    Entrada estándar que permite a un usuario introducir información de pago mediante un proveedor de pagos. Los pagos se procesan antes del nodo Ending.

    ##### Configuración del campo de pago

    La configuración del campo de pago es la siguiente:

    | Settings                  | Description                                                                                                                       |
    | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
    | Tipo de pago              | Selecciona pagos únicos (Charge) o recurrentes (Subscription).                                                                    |
    | Importe                   | Especifica el importe de un pago único (Charge).                                                                                  |
    | Moneda                    | Especifica la divisa de un pago único (Charge).                                                                                   |
    | Acción de suscripción     | Selecciona si deseas crear o actualizar una suscripción existente.                                                                |
    | Acción del cliente        | Selecciona si deseas crear o actualizar un cliente existente.                                                                     |
    | Opciones de visualización | Muestra y permite editar las etiquetas de Número de tarjeta, Fecha de vencimiento y Código de seguridad, así como los Trustmarks. |

    ##### Valor de salida del campo de pago

    El tipo de datos del valor de salida del campo de pago es una cadena o un objeto para los flujos posteriores al envío.

    El tipo de datos del valor de salida del campo de pago como `string`:

    ```json lines
    {
      "payment_field_id": "pm_1P19e..."
    }
    ```

    El tipo de dato del valor de salida del campo de pago para las suscripciones como `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "price_ids": [
          "price_1ONHR..."
        ],
        "subscription_id": "sub_1P1A...",
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```

    El tipo de dato del valor de salida del campo de pago de charge como `object` :

    ```json lines
    {
      "payment_field_id": {
        "payment_method_id": "pm_1P19e...",
        "customer_id": "cus_PqrM...",
        "amount": 100,
        "payment_intent_id": "pi_3P19e5..."
      }
    }
    ```
  </Accordion>

  <Accordion title="Campo legal">
    Entrada estándar que añade texto legal y una casilla de verificación.

    ##### Valor de salida del campo legal

    El tipo de dato del valor de salida del campo legal es `boolean`.

    ```json lines
    }
    ```

  <Accordion title="Campo de lista desplegable">

    ##### Valor de salida del campo desplegable

    ```json lines

      "dropdown_field_id": "Option A"
    }
    ```

    ```json lines
    {
      "dropdown_field_id": ["Option A", "Option B"]
    ```

  </Accordion>

  <Accordion title="Campo numérico">

  <Accordion title="Campo de fecha y hora">

    | Clave   | Descripción                               |

    | Formato | Seleccione el formato de fecha o de hora. |

<div id="blocks-components">
  ### Componentes de bloque
</div>

Los bloques son componentes de la interfaz de usuario que añaden funcionalidad a tus formularios, pero no recopilan información del usuario.

<Frame>![Dashboard > Formularios > Componentes > Bloques](/docs/images/cdy7uua7fh8z/7IijftJnuwcevL4bdHiaKw/f0376ca75303bb58d0df87e6af75e347/forms-block.png)</Frame>

<div id="available-blocks-in-forms">
  #### Bloques disponibles en Forms
</div>

Los bloques disponibles son:

<AccordionGroup>
  <Accordion title="Bloque de botón Siguiente">
    Un botón que permite a los usuarios continuar al siguiente nodo del formulario.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      * Antes de continuar, se aplican y validan las reglas de validación de los campos tanto del lado del cliente como del servidor. Si alguna de estas validaciones falla, se muestra un mensaje de error para que el usuario lo revise.
      * Si el siguiente nodo es un nodo de flujo, se muestra un indicador de carga hasta que el flujo finalice. Si el flujo falla o devuelve un mensaje de error personalizado, se muestra un mensaje de error.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Anterior">
    Un botón que permite a los usuarios volver al nodo del paso anterior.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un usuario selecciona el botón Anterior, las reglas de validación de los campos se vuelven a ejecutar, incluso si no se ha realizado ningún cambio en los datos introducidos antes de continuar.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Saltar">
    Un botón que permite a los usuarios omitir el paso actual y saltar directamente al nodo de destino.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Al omitir el paso actual, no se validará ni se recopilará la información completada en ese paso.
    </Callout>
  </Accordion>

  <Accordion title="Bloque de botón Reenviar">
    Un componente que permite a los usuarios volver a ejecutar un flujo para generar y enviar un nuevo código OTP

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si un usuario selecciona el botón Reenviar, el flujo asociado generará y enviará un nuevo código OTP.
    </Callout>

    | Configuración        | Descripción                                                                                                                                                                                                                                                                                                                                                |
    | -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | Alineación del texto | Selecciona la alineación del texto: izquierda, centro o derecha                                                                                                                                                                                                                                                                                            |
    | Texto                | Texto predeterminado que se muestra cuando los usuarios seleccionan el botón Reenviar                                                                                                                                                                                                                                                                      |
    | Texto del botón      | Texto del botón                                                                                                                                                                                                                                                                                                                                            |
    | Texto de espera      | Texto que se muestra cuando los usuarios han seleccionado el botón Reenviar. Sustituye la configuración `Text` y `Button text`. Usa la variable `“{{remaining_seconds}}”` para mostrar dinámicamente la cantidad de segundos restantes hasta que el botón de envío vuelva a estar habilitado. Por ejemplo: “Reenviar en `{{remaining_seconds}}` segundos”. |
    | Flujo                | Flujo que se ejecuta después de que el usuario selecciona el botón Reenviar.                                                                                                                                                                                                                                                                               |
    | Intentos máximos     | Número máximo de veces que el usuario puede seleccionar el botón Reenviar                                                                                                                                                                                                                                                                                  |
    | Tiempo de espera     | Tiempo de espera entre intentos, en segundos.                                                                                                                                                                                                                                                                                                              |
  </Accordion>

  <Accordion title="Bloque de texto enriquecido">
    Un bloque de texto enriquecido para personalizar el nodo del paso con información adicional.
  </Accordion>

  <Accordion title="Bloque divisor">
    Una línea para dividir las distintas secciones del nodo del paso. Puede contener un texto breve.
  </Accordion>

  <Accordion title="Bloque HTML">
    Un bloque HTML para crear una interfaz de usuario personalizada.
  </Accordion>

  <Accordion title="Bloque de imagen">
    Un bloque de imagen para personalizar el paso añadiendo imágenes.
  </Accordion>
</AccordionGroup>

<Frame>![Dashboard > Forms > Components > Widget](/docs/images/cdy7uua7fh8z/3J0IuWLP7utOPKSHc82IE9/a523afe0c240c63f714ef5fb102f8dab/forms-widget.png)</Frame>

<div id="available-widgets-in-forms">
  #### Widgets disponibles en Forms
</div>

El widget disponible es:

<AccordionGroup>
  <Accordion title="Widget de direcciones de Google">
    Un widget que permite a los usuarios introducir y validar su dirección.

    ##### Configuración del widget de direcciones de Google

    La configuración del widget de direcciones de Google incluye:

    | Configuración | Descripción                                                               |

    ```json lines
    {
      "google_address_widget_id": {
        "type": "Feature",
        "geometry": {
          "type": "Point",
          "coordinates": [
            -73.9654415,
            40.8054491
          ]
        },
        "properties": {
          "geocoding": {
            "type": "house",
            "housenumber": "2880",
    ```

  <Accordion title="Google reCAPTCHA">
    Un widget que ayuda a proteger su sitio web contra el spam y el abuso al verificar que el usuario es una persona y no un bot.

    El widget admite:

    * Basado en puntuación (v3)
    * Desafío (v2)

      * Insignia invisible de reCAPTCHA

    Diferencias entre `v2` y `v3`:

    * `v2` requiere la interacción del usuario mediante desafíos visibles, como marcar una casilla de verificación o resolver rompecabezas.
    * `v3` se ejecuta en segundo plano y proporciona una puntuación basada en el comportamiento del usuario, sin requerir su interacción. Al usar esta versión, asegúrese de implementar lógica de negocio adicional para gestionar la puntuación y determinar si se necesita alguna acción adicional.

    ##### Configuración del widget de Google reCAPTCHA

    La configuración del widget de Google reCAPTCHA incluye:

    | Configuración   | Descripción                                                                                                                                                                                                                                                  |
    | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | Clave del sitio | La clave pública del sitio que se usa para inicializar reCAPTCHA en su sitio web. Puede crearla en la [consola de Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) o en su proyecto de Google Cloud Platform.                                |
    | Clave secreta   | La clave secreta que se usa para comunicarse de forma segura con el servicio de reCAPTCHA del lado del servidor. Puede crearla en la [consola de Google reCAPTCHA](https://www.google.com/recaptcha/admin/create) o en su proyecto de Google Cloud Platform. |

    ##### Valor de salida de Google reCAPTCHA

    El tipo de dato del valor de salida del widget de Google reCAPTCHA es un `object`.

    Ejemplo de respuesta de `v2`:

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com"
      }
    }
    ```

    Ejemplo de respuesta de `v3`:

    ```json lines
    {
      "recaptcha_widget_id": {
        "success": true,
        "challenge_ts": "2025-03-26T11:22:18Z",
        "hostname": "auth.example.com",
        "score": 0.9
      }
    }
    ```
  </Accordion>

  <Accordion title="Credenciales verificables">
    Un widget que permite a los usuarios verificar su identidad mediante credenciales verificables almacenadas en su billetera digital.

    ##### Configuración del widget de Credenciales verificables

    La configuración del widget de Credenciales verificables incluye:

    | Configuración           | Descripción                                                                                                                                                                                                          |
    | ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | URL                     | El valor de la URL que se utiliza para generar el código QR. Este valor se devuelve en la propiedad `engagement` de la solicitud de verificación.                                                                    |
    | Texto del enlace        | Texto alternativo que se mostrará a los usuarios que no puedan escanear el código QR y prefieran abrir el enlace directamente en su dispositivo.                                                                     |
    | Tamaño                  | El tamaño del código QR.                                                                                                                                                                                             |
    | Token público           | El token de acceso necesario para consumir el endpoint de sondeo. Asegúrese de que el token se genere solo con el scope `read:verification_request` para evitar exponer el acceso a otros recursos.                  |
    | ID de verificación      | El ID de verificación que se genera al iniciar la solicitud de verificación.                                                                                                                                         |
    | Tiempo máximo de espera | El tiempo máximo de espera para que se complete el proceso de verificación. Una vez alcanzado el tiempo establecido, el sondeo se detendrá y se generará un error, independientemente del estado de la verificación. |

    ##### Valor de salida de Credenciales verificables

    El tipo de dato del valor de salida del widget de Credenciales verificables es un `object`.

    ```json lines
    {
      "verifiable_credentials_widget_id": {
        "state": "honored",
        "reason": "...", // El campo reason contiene información adicional (si está disponible) sobre el estado de la solicitud de verificación. 
        "presentation": {} // La presentación contiene los claims proporcionados por la billetera en respuesta a la definición de presentación.
      }
    }
    ```
  </Accordion>
</AccordionGroup>

<div id="router-node">
  ## Nodo Router
</div>

Los nodos Router te permiten agregar reglas para crear bifurcaciones de lógica condicional entre nodos.

De forma predeterminada, un nodo Router tiene una única regla de paso directo llamada **Caso predeterminado** que se conecta con otros nodos. Puedes agregar reglas adicionales basadas en un conjunto de condiciones que debe cumplir una variable para conectarse con otros nodos. Para obtener más información, consulta [Router.](/es/docs/customize/forms/routers)
