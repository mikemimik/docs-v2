<div id="configure-your-http-vault-connection-options">
  ## Configura las opciones de tu conexión HTTP de Vault
</div>

La conexión HTTP de Vault se puede configurar con diferentes opciones de authentication y luego usarse en tus HTTP Request Flow Actions.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configura tu conexión HTTP de Vault para Bearer Token
</div>

Para configurar una conexión HTTP de Vault con autorización **Bearer Token**, debes establecer los parámetros necesarios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro    | Descripción                                                 |
| ------------ | ----------------------------------------------------------- |
| Bearer token | Token estático utilizado para autorizar servicios externos. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configura tu conexión HTTP de Vault para OAuth con credenciales de cliente
</div>

Para configurar una conexión HTTP de Vault mediante la autorización de **credenciales de cliente**, debes establecer los parámetros obligatorios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro            | Descripción                                                                                        |
| -------------------- | -------------------------------------------------------------------------------------------------- |
| Endpoint del tenant  | Endpoint del tenant del servidor de autorización que emite los tokens de acceso.                   |
| ID de cliente        | ID de cliente de tu aplicación M2M autorizada para la API personalizada.                           |
| secreto de cliente   | secreto de cliente de tu aplicación M2M autorizada para la API personalizada.                      |
| Audiencia (opcional) | Audiencia de la API personalizada. Normalmente se usa para APIs personalizadas definidas en Auth0. |
| Recurso (opcional)   | Recurso de la API personalizada. Parámetro estándar con un significado similar al de la audiencia. |
| Scope (opcional)     | Lista de scopes en formato de cadena separada por espacios.                                        |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configura tu conexión HTTP de Vault para clave API
</div>

Para configurar una conexión HTTP de Vault con autorización mediante **clave API**, debes establecer los parámetros obligatorios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro | Descripción                                                                                                          |
| --------- | -------------------------------------------------------------------------------------------------------------------- |
| Nombre    | Nombre de la clave API que se añadirá como Encabezado o Parámetro de consulta.                                       |
| Valor     | Valor de la clave API que se añadirá como Encabezado o Parámetro de consulta.                                        |
| Agregar a | De forma predeterminada, es Encabezado. Determina si la clave API se agrega como Encabezado o Parámetro de consulta. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configura tu conexión HTTP de Vault para la autenticación básica
</div>

Para configurar una conexión HTTP de Vault con autorización **Basic Auth**, debes establecer los parámetros obligatorios.

<Frame>![Dashboard > Actions > formulario > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro         | Descripción                                                                                                                                |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Nombre de usuario | Nombre de usuario que se concatenará y codificará en base64, y luego se añadirá como parámetro del encabezado Authorization de tipo Basic. |
| Contraseña        | Contraseña que se concatenará y codificará en base64, y luego se añadirá como parámetro del encabezado Authorization de tipo Basic.        |

<Warning>
  La configuración de autenticación básica de la HTTP Vault Connection está destinada a reemplazar la opción actualmente integrada en la Action de solicitud HTTP de Flows.
</Warning>
