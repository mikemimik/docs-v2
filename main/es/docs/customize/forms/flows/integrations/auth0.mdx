<div id="custom-api-request">
  ## Solicitud de API personalizada
</div>

Envía una solicitud de API personalizada a los endpoints de la Auth0 API.

<Frame>![Dashboard > Personalizar > Flows > Integraciones > Auth0](/docs/images/cdy7uua7fh8z/66uV9iuhwfnXUdE1quLOyx/2ae3ff26335acbcd172cbb40618a5a90/Screenshot_2025-01-24_at_18.50.26.png)</Frame>

### Configuración de entrada

| Parameter  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Path       | La ruta absoluta del endpoint de la API.<br />( Ejemplo: `/api/v2/organizations` )<br />**Nota:** Algunos endpoints, por ejemplo, [ Obtener un usuario ](https://auth0.com/docs/api/management/v2/users/get-users-by-id), requieren parámetros codificados como URI cuando no se usan caracteres alfanuméricos, como `-`, `.`, `_` o `~` ([ caracteres ](https://datatracker.ietf.org/doc/html/rfc3986#section-2.3)).<br />Para ello, puedes usar la función encodeURIComponent:<br />`{{ functions.encodeURIComponent(context.user.user_id) }}` |
| Method     | Los métodos HTTP disponibles incluyen: <ul> <li>`GET`</li> <li>`POST`</li> <li>`PUT`</li> <li>`PATCH`</li> <li>`DELETE`</li> </ul>                                                                                                                                                                                                                                                                                                                                                                                                               |
| Body       | El cuerpo de la request.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Headers    | Los headers de la request.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| URL Params | Los parámetros de URL. Estos parámetros se codifican automáticamente en la URL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

<div id="output-object">
  ### Objeto de salida
</div>

Consulta [Auth0 Management API](https://auth0.com/docs/api/management/v2/introduction) para conocer las respuestas esperadas y los códigos de estado.

<div id="send-email">
  ## Enviar correo electrónico
</div>

Envía un correo electrónico personalizado con tu [proveedor de correo electrónico configurado](/docs/es/customize/email).

<Warning>
  [Proveedor de correo electrónico](/docs/es/customize/email) debe estar habilitado antes de usar esta Action.
</Warning>

<Frame>![](/docs/images/forms/flows/integrations/auth0-notifications-send-email.png)</Frame>

### Configuración de entrada

| Parámetro                                             | Descripción                                                                                                                                                                                                                                                       |
| ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Usar un correo electrónico de remitente personalizado | De forma predeterminada, usaremos el correo electrónico del remitente que hayas configurado en la configuración personalizada de tu proveedor de correo electrónico. Activa esta opción si quieres usar un correo electrónico o un nombre de remitente diferente. |
| Correo electrónico del destinatario                   | El correo electrónico del destinatario.                                                                                                                                                                                                                           |
| Variables personalizadas                              | Agrega variables personalizadas para insertarlas en tu plantilla de correo electrónico. Una vez configuradas, puedes consultarlas con la sintaxis `{{custom_vars.your_variable_key}}`.                                                                            |
| Asunto                                                | El asunto del correo electrónico. Admite variables personalizadas y sintaxis Liquid.                                                                                                                                                                              |
| Mensaje                                               | El mensaje del correo electrónico está en formato HTML, usa sintaxis Liquid y admite variables personalizadas.                                                                                                                                                    |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Asunto y Mensaje pueden usar una combinación de sintaxis Liquid con [variables comunes](https://auth0.com/docs/customize/email/email-templates/supported-liquid-syntax) o variables personalizadas mediante la sintaxis `{{custom_vars.your_variable_key}}`.

  Usa variables personalizadas para pasar información entre el contexto de Formularios / Flows y el contenido de Asunto y Mensaje.
</Callout>

<div id="output-object">
  ### Objeto de salida
</div>

| Propiedad | Tipo    | Descripción                                                                                                                  |
| --------- | ------- | ---------------------------------------------------------------------------------------------------------------------------- |
| success   | Boolean | Indica si el correo electrónico se puso en cola correctamente para su envío, independientemente de si la entrega falló o no. |

<div id="output-object-example">
  ### Ejemplo del objeto de salida
</div>

```json
{
  "success": true
}
```

<div id="send-sms">
  ## Enviar SMS
</div>

Envía un SMS personalizado con tu [proveedor de telefonía configurado](https://auth0.com/docs/customize/phone-messages/configure-phone-messaging-providers).

<Warning>
  [Phone Provider](https://auth0.com/docs/customize/phone-messages/configure-phone-messaging-providers) debe estar habilitado para poder enviar mensajes de texto antes de usar esta Action.
</Warning>

<Frame>![](/docs/images/forms/flows/integrations/auth0-notifications-send-sms.png)</Frame>

### Configuración de entrada

| Parameter        | Description                                                                                                                                                                                                                                       |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| From             | De forma predeterminada, usaremos el número de teléfono del remitente que hayas configurado en la configuración personalizada de tu proveedor telefónico. Introduce un valor personalizado si quieres usar otro número de teléfono del remitente. |
| To               | El número de teléfono del destinatario.                                                                                                                                                                                                           |
| Custom variables | Agrega variables personalizadas para insertarlas en tu plantilla telefónica. Una vez configuradas, puedes hacer referencia a ellas mediante la sintaxis `{{custom_vars.your_variable_key}}`.                                                      |
| Message          | El mensaje telefónico usa la sintaxis Liquid y admite variables personalizadas.                                                                                                                                                                   |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Mensaje puede usar una combinación de sintaxis Liquid con [variables comunes](https://auth0.com/docs/customize/email/email-templates/supported-liquid-syntax) o variables personalizadas mediante la sintaxis `{{custom_vars.your_variable_key}}`.

  Usa variables personalizadas para pasar información entre el contexto de Formularios / Flows y el contenido de Mensaje.
</Callout>

<div id="output-object">
  ### Objeto de salida
</div>

| Propiedad | Tipo    | Descripción                                                                                                            |
| --------- | ------- | ---------------------------------------------------------------------------------------------------------------------- |
| success   | Boolean | Indica si el mensaje telefónico se encoló correctamente para su envío, independientemente de si la entrega falló o no. |

### Ejemplo de objeto de salida

```json
{
  "success": true
}
```
