En Rules puedes almacenar en caché recursos costosos usando el objeto `global`. El objeto `global` sobrevive a cada ejecución, por lo que puedes reutilizar lo que almacenes allí en lugar de crearlo cada vez que se ejecute una Rule. Sin embargo, el entorno de Rules puede reciclarse en cualquier momento, así que tu código **debe** comprobar siempre que `global` contiene lo que esperas.

Este ejemplo muestra cómo usar el objeto `global` para mantener una conexión a MongoDB:

```javascript lines
//Si el objeto db existe, úsalo.
if (global.db){
  return query(global.db, callback);
}

//Si no, obtén la db (mongodb en este caso)
mongo('mongodb://user:pass@mymongoserver.com/my-db',  function (db){
  global.db = db;
  return query(db, callback);
});

//Realiza el trabajo
function query(db, cb){
  // Haz algo con db
  ...
  return cb(null, user, context);
}
```

Para obtener más información sobre cómo usar MongoDB, consulta [el manual de MongoDB](https://docs.mongodb.com/manual/).
