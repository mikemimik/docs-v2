---
title: Plantillas de correo electrónico
description: Habilita y personaliza los correos electrónicos enviados por los flujos de trabajo de Auth0 mediante plantillas de correo electrónico.
---

Cada correo electrónico enviado como parte de los flujos de trabajo de Auth0 tiene una plantilla de correo correspondiente que determina la dirección del remitente, el asunto, el cuerpo del mensaje y más.

Después de [configurar un proveedor externo de correo electrónico SMTP](/es/docs/customize/email/smtp-email-providers), puedes modificar las plantillas de correo electrónico para personalizar tus correos.

Los campos de contenido de las plantillas de correo electrónico admiten [Liquid](https://liquidjs.com/index.html), un lenguaje de plantillas de código abierto que amplía la funcionalidad de HTML. Liquid te permite:

* Rellenar variables dinámicamente con datos, por ejemplo, reemplazar `{{ user.name }}` por el nombre del usuario
* Implementar lógica, como sentencias `if`/`else`
* Modificar datos con filtros para manipular cadenas y matrices, operadores matemáticos y más

Puedes habilitar, deshabilitar y personalizar las plantillas de correo electrónico en la [página de plantillas de correo electrónico del Dashboard](https://manage.auth0.com/#/templates).

| Correo electrónico de alerta por filtración de contraseña | Auth0 detecta que el usuario está intentando acceder a la aplicación con una contraseña que un tercero ha filtrado                                                                                                                                           | Contenido informativo                                                                        |
| Correo electrónico de OTP sin contraseña                  | Un usuario inicia sesión con autenticación por correo electrónico <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">Passwordless</Tooltip> |                                                                                              |
| Correo electrónico de invitación de usuario               | Se invita a un usuario a una organización o aplicación                                                                                                                                                                                                       | Un enlace para unirse a la organización                                                      |
| Correo electrónico con código de verificación             | Has habilitado la verificación basada en códigos y un usuario se registra o inicia sesión por primera vez, o has habilitado la política de MFA adaptativa y hay una transacción con baja confianza para la que se debe verificar la titularidad de la cuenta | Un código que el usuario debe introducir durante la autenticación                            |
| Correo electrónico con enlace de verificación             | Un usuario se registra o inicia sesión por primera vez                                                                                                                                                                                                       | Un enlace para verificar el correo electrónico del usuario                                   |

<div id="email-template-limits">
  ## Límites de las plantillas de correo electrónico
</div>

* La personalización de plantillas de correo electrónico no está disponible cuando se usa el proveedor de correo integrado. Para usar plantillas de correo electrónico, [configura un proveedor de correo electrónico SMTP externo](/es/docs/customize/email/smtp-email-providers).

* Cada tipo de correo electrónico tiene exactamente una plantilla. Puedes usar Liquid para implementar condicionales y otra lógica para casos de uso, como correos electrónicos multilingües.

* Las plantillas de correo electrónico no admiten correos en texto sin formato. Todos los correos deben estar en HTML.

* En la plantilla de correo electrónico de inscripción en MFA, la variable `connection.name` no está disponible.

* En la plantilla de correo electrónico de OTP sin contraseña, solo está disponible la propiedad `email` del objeto `user`.

* A diferencia de otras plantillas de correo electrónico, la plantilla de correo electrónico de OTP sin contraseña se configura en la [página de configuración de correo electrónico de Passwordless Connections](https://manage.auth0.com/#/connections/passwordless), en lugar de en la página plantilla de correo electrónico.

* Por motivos de seguridad, la **dirección del remitente** en las plantillas de correo electrónico no puede contener `@auth0.com`. Si introduces una dirección de correo electrónico `@auth0.com` en From Addresses, tu tenant solo envía correos electrónicos predeterminados e ignora cualquier personalización. Los usuarios de Private Cloud pueden configurar una lista de denegación de dominios similar.
