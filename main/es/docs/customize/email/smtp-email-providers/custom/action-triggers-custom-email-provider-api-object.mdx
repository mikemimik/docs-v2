  ## `api.cache`
</div>

Modifica la caché.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

<ResponseField name="key" type="string">

</ResponseField>

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de cadena en la caché en la clave especificada.

Los valores almacenados en esta caché están limitados al trigger en el que
se establecen. Están sujetos a los [Límites de caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta forma tendrán un período de validez de *hasta* los valores
especificados en `ttl` o `expires_at`. Si no se especifica ningún período de validez, se usará
un período de validez predeterminado de 15 minutos. Los períodos de validez no pueden superar la duración
máxima indicada en [Límites de caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Esta caché está diseñada para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores incluso si aún están dentro del período de validez indicado.

<ResponseField name="key" type="string">
  La clave del registro que se almacenará.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se almacenará.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento de la caché.

  <Expandable title="propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora de vencimiento absoluta en milisegundos desde la época de Unix.
      Aunque los registros en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor `expires_at` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se
      proporcionó un valor para `ttl`. Si se proporcionan ambas opciones, se
      usará el vencimiento más próximo de los dos.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada de caché en milisegundos.
      Aunque los valores en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor `ttl` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se
      proporcionó un valor para `expires_at`. Si se proporcionan ambas opciones, se
      usará el vencimiento más próximo de los dos.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="apinotification">
  ## `api.notification`
</div>

Indica si el evento de notificación debe reintentarse o descartarse.
Consulta cada uno de estos métodos para obtener más información sobre el comportamiento real.
Si se realizan varias llamadas, solo se tiene en cuenta la última.

<div id="apinotificationdropreason">
  ### `api.notification.drop(reason)`
</div>

Cuando se invoca, el evento de notificación se considera fallido y sin recuperación:
Registraremos un error para este evento, pero no volveremos a enviarlo a la Action en el futuro.
Si necesitas que se vuelva a intentar este evento de notificación, considera llamar a retry en su lugar.

<ResponseField name="reason" type="string">
  este motivo se incluirá en la entrada de registro, lo que te ayudará a analizar mejor el error. Ten en cuenta que este campo está limitado a 1024 caracteres y se truncará si supera ese tamaño.
</ResponseField>

<div id="apinotificationretryreason">
  ### `api.notification.retry(reason)`
</div>

Cuando se llama, el evento de notificación se considera fallido, pero recuperable:
Registraremos un error para este evento, pero volveremos a intentarlo hasta 5 veces en los próximos minutos.
Si considera que este evento de notificación no debe reintentarse, le recomendamos llamar a drop en su lugar.

<ResponseField name="reason" type="string">
  este motivo formará parte de la entrada de registro, lo que le ayudará a analizar mejor el error. Tenga en cuenta que este campo está limitado a 1024 caracteres y se truncará si supera esa longitud.
</ResponseField>
