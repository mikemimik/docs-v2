<div id="prerequisites">
  ## Requisitos previos
</div>

* Debe tener una cuenta de [Microsoft 365](https://www.microsoft.com/en-us/microsoft-365) con una suscripción activa.

* Su suscripción a Microsoft 365 debe contar con licencia para enviar correos electrónicos a través de Exchange Online. Como mínimo, necesita una suscripción de pago al plan Microsoft Business Basic para usar Microsoft Graph API con Exchange Online.

* Debe configurar su proveedor para permitir conexiones entrantes desde direcciones IP específicas de [Auth0](/es/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-microsoft-365-exchange-online">
  ## Configurar Microsoft 365 Exchange Online
</div>

1. Inicia sesión en tu cuenta de [Microsoft Azure](https://azure.microsoft.com/) con tu cuenta de Microsoft 365.
2. Registra un recurso de [aplicación de Microsoft Entra ID](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application) si aún no tienes uno.
3. Ve al recurso de aplicación de Active Directory. Selecciona **Services &gt; App registrations** y, a continuación, la aplicación que usarás para enviar correos electrónicos.
4. Configura los permisos de aplicación necesarios en **Manage &gt; API permissions**.
5. Agrega el [permiso](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#api-permissions) `Mail.Send` seleccionando **Add a permission &gt; Microsoft Graph &gt; Application permissions**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Antes de configurar **Application permissions**, necesitas consentimiento administrativo.
   </Callout>

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     De forma predeterminada, `Mail.Send` otorga permiso para enviar correos como cualquier usuario de tu tenant. Para restringir el acceso a un buzón específico, usa [RBAC for Applications in Exchange Online](https://learn.microsoft.com/en-us/exchange/permissions-exo/application-rbac) para limitar el permiso a una entidad de servicio específica.
   </Callout>
6. Obtén las configuraciones necesarias del proveedor:

   1. Ve a **Información general** de la aplicación para obtener el [ID de cliente y el Tenant ID](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#application-id-client-id) de tu aplicación.
   2. Ve a **Manage &gt; Certificates &amp; secrets** y copia el **Value** para crear un [secreto](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#certificates--secrets) de la aplicación.
   3. Obtén el **User Email** de la organización que enviará los correos a través de la aplicación. Ve a **Users &gt; Active users**, selecciona tu usuario autorizado y copia su **User Email** desde sus propiedades.
7. Ve a Auth0 [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider).
8. Activa el interruptor **Use my own email provider**.
9. Haz clic en el logotipo de **Microsoft 365**.
10. Indica la dirección de correo electrónico en **From**. Luego, introduce el **Tenant ID**, el **ID de cliente** y el **Client Secret** del paso 6, y haz clic en **Save**.

<div id="send-a-test-email">
  ## Enviar un correo electrónico de prueba
</div>

Envía un correo electrónico de prueba con el botón **Send Test Email**. Si la configuración es correcta, recibirás un correo electrónico de confirmación. Si no recibes ningún correo después de unos minutos, consulta los [registros de Auth0](https://manage.auth0.com/#/logs) para ver los códigos de error.

Una vez completada la configuración, la página [Email Usage](https://admin.microsoft.com/#/reportsUsage/EmailActivity) de Microsoft Exchange muestra información sobre la entrega de todos los correos electrónicos enviados a tus usuarios.
