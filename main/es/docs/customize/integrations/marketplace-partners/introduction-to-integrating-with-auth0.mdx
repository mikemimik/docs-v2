title: Introducción a la integración con Auth0
---

Auth0 proporciona un servicio de autenticación y autorización basado en estándares abiertos como OIDC, OAuth2, <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip>, entre otros. Pero, además, ofrecemos herramientas para que nuestros clientes puedan externalizar la centralización y gestión de usuarios, ciertos componentes de la seguridad de las aplicaciones y la conexión de redes de aplicaciones a una única fuente de identidad.

Una parte importante de esto es permitir que partners, como usted, creen un ecosistema de integraciones que pueda utilizarse para mejorar y ampliar lo que ya ofrecemos. Para ello, proporcionamos puntos de extensibilidad y métodos que permiten la personalización. Esto se traduce en más funcionalidad para los clientes y una menor carga de mantenimiento para ellos.

Casi el 90 % de nuestros clientes amplían la plataforma Auth0 para adaptarla a sus necesidades específicas. Al trabajar con usted para resolver problemas comunes de identidad y casos de uso, podemos reducir el tiempo que tardan nuestros clientes en implementar sus flujos de trabajo de identidad.

<div id="what-can-be-extended">
  ## ¿Qué se puede ampliar?
</div>

A continuación, se muestra una lista de las diferentes formas en que Auth0 se puede personalizar. Algunos puntos de extensibilidad permiten instalar integraciones directamente, mientras que otros requieren una guía que el cliente pueda seguir, y algunos puntos de extensibilidad requieren una cuenta de pago de Auth0.

* **Inicio de sesión único en un servicio externo**

  * [SAML](/es/docs/customize/integrations/marketplace-partners/sso-integrations-for-partners)
  * OpenID Connect
  * WS-Fed
* [**Personalizar la UI de Auth0 de cara al usuario**](/es/docs/customize)

  * Cargar JavaScript y CSS personalizados en el formulario de inicio de sesión
  * plantillas de correo electrónico personalizadas
* **Fuentes de identidad**

  * [OpenID Connect](/es/docs/authenticate/identity-providers/enterprise-identity-providers/oidc)
  * [OAuth2](/es/docs/customize/integrations/marketplace-partners/social-connections-for-partners)
  * [SAML](/es/docs/authenticate/identity-providers/enterprise-identity-providers/saml)
  * [Active Directory/LDAP](/es/docs/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap)
  * [Código personalizado que apunta a una fuente de identidad remota](/es/docs/authenticate/database-connections/custom-db)
* [**Código personalizado después de una autenticación correcta**](/es/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)

  * Llamar a una API externa para enriquecer el perfil de usuario, bloquear inicios de sesión, etc.
  * [Redirigir a un servicio externo](/es/docs/customize/integrations/marketplace-partners/redirect-actions-for-partners)
  * Enriquecer el perfil de usuario almacenado en Auth0
  * Proporcionar un segundo factor de autenticación
  * [Autenticación escalonada](/es/docs/secure/multi-factor-authentication/step-up-authentication)
  * Activar MFA de Auth0
