<Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=SSO">SSO</Tooltip> (inicio de sesión único) usan una identidad de Auth0 para iniciar sesión, mediante <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip>, en un servicio externo. Esto permite que los clientes con aplicaciones internas usen SSO para acceder a proveedores externos con la identidad de Auth0.

Las [integraciones de SSO disponibles en Marketplace](https://marketplace.auth0.com/features/sso-integrations) definen tanto los valores estándar de configuración de SAML necesarios para conectar Auth0 con proveedores externos como los campos necesarios para valores adicionales específicos del cliente. En estas integraciones, Auth0 actúa como el <Tooltip tip="Identity Provider (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip> en un escenario iniciado por el proveedor de servicios (SP). Para obtener información sobre cómo configurar Auth0 para estas integraciones, consulta [Configurar Auth0 como proveedor de identidad SAML](/es/docs/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

La parte que proporcionarán nuestros socios es la configuración predeterminada de SAML necesaria para que esta transacción funcione. Esto consistirá en valores o campos solicitados por el cliente para algunos o todos los atributos enumerados en [Personalizar aserciones SAML: atributos de aserción SAML](/es/docs/authenticate/protocols/saml/saml-configuration/customize-saml-assertions).

Para crear y probar una integración, [crea un tenant](https://auth0.com/signup) en Auth0 y sigue la documentación anterior para crear una integración de SSO funcional con tu servicio.

Además, para ayudarte a crear tu integración, puedes aprovechar nuestra [integración de SSO](https://github.com/auth0-marketplace/TEMPLATE-sso).

<div id="submit-your-sso-integration">
  ## Envía tu integración de SSO
</div>

Si estás listo para convertirte en socio y enviar tu integración, crea una cuenta de socio en el [portal de socios](https://partners.marketplace.auth0.com/) y haz lo siguiente:

1. Completa el perfil de socio, que incluye una breve descripción de tu empresa, contenido legal y de marketing, el logotipo de la empresa, los términos del servicio, la política de privacidad, la información de contacto y el correo electrónico de soporte.
2. Envía tu integración, que incluye el código de integración, la guía de instalación y detalles como el nombre y la descripción. De forma opcional, puedes añadir 3 propuestas de valor a tu integración.

**Necesitaremos la siguiente configuración y documentación:**

* El JSON utilizado para configurar la aplicación con el complemento SAML en funcionamiento
* Una lista de campos que deben mostrarse al cliente que configura la integración (como **Callback URL**, **<Tooltip tip="Audience: Identificador único de la audiencia para un token emitido. Se denomina aud en un token, y su valor contiene el ID de una aplicación (Client ID) para un ID Token o de una API (API Identifier) para un Access Token." cta="Ver glosario" href="/es/docs/glossary?term=Audience">Audience</Tooltip>**, y **Signing Certificate**)
* Una guía de instalación escrita en Markdown.
