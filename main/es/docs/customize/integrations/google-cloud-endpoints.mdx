[Google Cloud Endpoints (GCE)](https://cloud.google.com/endpoints/) es un sistema de gestión de APIs que ofrece funciones para ayudarte a crear, mantener y proteger tus APIs. GCE usa [OpenAPI](https://www.openapis.org/) para definir los endpoints de tu API, las entradas y salidas, los errores y la descripción de seguridad.

Para obtener más información sobre la especificación OpenAPI, consulta el repositorio [OpenAPI Specification](https://github.com/OAI/OpenAPI-Specification) en GitHub.

En este tutorial se explica cómo proteger Google Cloud Endpoints con Auth0.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de empezar, necesitarás una API de GCE desplegada. Si aún no has creado una API, completa la [Cloud Endpoints Quickstart](https://cloud.google.com/endpoints/docs/quickstart-endpoints) que encontrarás en la documentación de Google.

La guía de inicio rápido te mostrará cómo crear una API de GCE sencilla con un único endpoint, `/airportName`, que devuelve el nombre de un aeropuerto a partir de su [código IATA](https://en.wikipedia.org/wiki/IATA_airport_code) de tres letras.

<div id="update-the-api-configuration">
  ## Actualiza la configuración de la API
</div>

A continuación, actualizaremos el archivo de configuración de OpenAPI de la API de GCE. En la API de ejemplo creada durante el Quickstart, este archivo es `openapi.yaml`.

<div id="add-security-definitions">
  ### Agregar definiciones de seguridad
</div>

Abre el archivo de configuración y agrega una nueva sección `securityDefinitions`. En esta sección, agrega una nueva definición (`auth0_jwt`) con los siguientes campos:

| Campo                | Descripción                                                                                                                                                                                                    |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `authorizationUrl`   | La URL de autorización; debe establecerse en `"https://{yourDomain}/authorize"`                                                                                                                                |
| `flow`               | El flow que usa el esquema de seguridad OAuth2. Los valores válidos son `"implicit"`, `"password"`, `"application"` o `"accessCode"`.                                                                          |
| `type`               | El tipo de esquema de seguridad. Los valores válidos son `"basic"`, `"apiKey"` o `"oauth2"`                                                                                                                    |
| `x-google-issuer`    | El emisor de una credencial; debe establecerse en `"https://{yourDomain}/"`                                                                                                                                    |
| `x-google-jwks_uri`  | El URI del conjunto de claves públicas para validar la firma del [JSON Web Token (JWT)](/docs/es/glossary?term=JSON+Web+Token+%28JWT%29). Establece este valor en `"https://\{yourDomain}/.well-known/jwks.json"` |
| `x-google-audiences` | El identificador de la API; asegúrate de que este valor coincida con el que definiste en el Auth0 Dashboard para la API.                                                                                       |

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample1 = `securityDefinitions:
  auth0_jwt:
    authorizationUrl: "https://{yourDomain}/authorize"
    flow: "implicit"
    type: "oauth2"
    x-google-issuer: "https://{yourDomain}/"
    x-google-jwks_uri: "https://{yourDomain}/.well-known/jwks.json"
    x-google-audiences: "{yourApiIdentifier}"`;

<div id="update-the-endpoint">
  ### Actualiza el endpoint
</div>

Ahora, actualiza el endpoint añadiendo un campo `security` con la `securityDefinition` que creamos en el paso anterior.

```yaml lines
paths:
  "/airportName":
    get:
      description: "Get the airport name for a given IATA code."
      operationId: "airportName"
      parameters:
        -
          name: iataCode
          in: query
          required: true
          type: string
      responses:
        200:
          description: "Success."
          schema:
            type: string
        400:
          description: "The IATA code is invalid or missing."
      security:
       - auth0_jwt: []
```

En el ejemplo anterior, el campo `security` le indica al proxy de GCE que nuestra ruta `/airportName` debe protegerse con la definición `auth0-jwt`.

Después de actualizar la configuración de OpenAPI, debería verse más o menos así:

Haz una solicitud `GET` a tu API con un encabezado Authorization de `Bearer {ACCESS_TOKEN}` para obtener acceso autorizado:

<AuthCodeGroup>
