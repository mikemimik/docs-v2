---
description: Aprende a extender Login by Auth0 WordPress Plugin con hooks, filtros y funciones.
title: Extiende Login by Auth0 WordPress Plugin
---

Los plugins de WordPress se pueden extender para adaptarse a tus necesidades específicas mediante Actions y filtros que ejecutan código personalizado en puntos concretos durante el runtime. En este documento se describen los hooks existentes en Login by Auth0 WordPress Plugin. Estaremos encantados de revisar y aprobar nuevos filtros y Actions que te ayuden a integrarte aún más con este plugin. Para obtener más información, consulta la sección Contributing en el [`README` del repositorio de GitHub del plugin](https://github.com/auth0/wordpress?tab=readme-ov-file#contributing).

<div id="wordpress-actions">
  ## Actions de WordPress
</div>

Las Actions de WordPress ejecutan código personalizado en puntos específicos del proceso. Para obtener más información sobre las Actions de WordPress, consulta [WordPress Plugin Handbook: Actions on wordpress.org](https://developer.wordpress.org/plugins/hooks/actions/).

<div id="auth0_before_login">
  ### auth0_before_login
</div>

Esta Action se ejecuta en `WP_Auth0_LoginManager` después de que un usuario se haya autenticado correctamente, pero antes de que inicie sesión en WordPress. Puede usarse para detener el proceso de inicio de sesión si es necesario mediante `wp_die()` o lanzando una excepción.

<div id="auth0_user_login">
  ### auth0_user_login
</div>

Esta Action se ejecuta en `WP_Auth0_LoginManager` después de que un usuario se haya autenticado correctamente y haya iniciado sesión en WordPress. Puede usarse para establecer valores específicos de metadatos, enviar notificaciones o hacer ping a otros servicios.

<div id="wpa0_user_created">
  ### wpa0_user_created
</div>

Esta Action se ejecuta en `WP_Auth0_Users` justo después de que se crea correctamente un usuario de WordPress. Puede usarse para cambiar los valores del usuario, establecer metadatos de usuario adicionales o activar otras Actions para usuarios nuevos.

<div id="filters">
  ## Filtros
</div>

Los filtros de WordPress también ejecutan código personalizado en puntos específicos durante el procesamiento, pero siempre devuelven un valor modificado del mismo tipo que se haya pasado. Para obtener más información sobre los filtros, consulta [WordPress Plugin Handbook: Filters on wordpress.org](https://developer.wordpress.org/plugins/hooks/filters/).

<div id="auth0_create_user_data">
  ### auth0_create_user_data
</div>

Este filtro se puede usar para modificar el arreglo de datos de usuario al crear un usuario nuevo en WordPress a partir de un usuario de Auth0 entrante. Recibe 2 parámetros:

* `$user_data` son los datos de usuario extraídos del perfil de Auth0.
* `$userinfo` es el perfil de usuario de Auth0.

Este filtro siempre debe devolver un arreglo de datos que se pasará a la función principal `wp_insert_user()`.

<div id="auth0_get_wp_user">
  ### auth0_get_wp_user
</div>

Este filtro se ejecuta después de que el plugin encuentra al usuario correspondiente para iniciar sesión (según el `user_id` de auth0) y se usa para reemplazar el comportamiento predeterminado con reglas de coincidencia personalizadas (por ejemplo, hacer que siempre coincida por correo electrónico).

Si el filtro devuelve null, buscará por correo electrónico. Para obtener más información, consulta [Integrate with WordPress](/es/docs/customize/integrations/cms/wordpress-plugin/integrate-with-wordpress).

<div id="auth0_verify_email_page">
  ### auth0_verify_email_page
</div>

Este filtro se ejecuta en `WP_Auth0_Email_Verification` para cambiar el HTML que se muestra cuando un usuario que está iniciando sesión necesita verificar su correo electrónico antes de obtener acceso al sitio. Ten en cuenta que este HTML se pasa a `wp_die()`, donde se modifica antes de mostrarse (consulta la definición de `_default_wp_die_handler()` en core para obtener más información).

<div id="auth0_get_auto_login_connection">
  ### auth0_get_auto_login_connection
</div>

Este filtro se usa en `WP_Auth0_LoginManager` para modificar la conexión que se utiliza en el proceso de inicio de sesión automático. La configuración de wp-admin se obtiene y luego se pasa por este filtro.

<div id="wp_auth0_get_option">
  ### wp_auth0_get_option
</div>

Este filtro lo utilizan funciones y métodos de obtención de opciones para modificar el valor de salida.

<div id="auth0_migration_ws_authenticated">
  ### auth0_migration_ws_authenticated
</div>

Este filtro se usa en `WP_Auth0_Routes` para modificar el objeto `WP_User` que se codifica como JSON y se devuelve a Auth0 durante una migración de usuario.

<div id="wpa0_should_create_user">
  ### wpa0_should_create_user
</div>

Este filtro se usa en `WP_Auth0_Users` para decidir si debe crearse un usuario. El valor inicial que se pasa es `TRUE`. Si por cualquier motivo se devuelve `FALSE`, se rechazará el registro y el usuario que se esté registrando verá un mensaje de error (`WP_Auth0_UsersRepo::create()`).

<div id="auth0_login_css">
  ### auth0_login_css
</div>

Este filtro se utiliza para modificar el CSS de la página de inicio de sesión, incluido el propio widget de inicio de sesión. Este filtro se ejecuta antes de recuperar el CSS de la página de Settings de wp-admin.

<div id="auth0_login_form_tpl">
  ### auth0_login_form_tpl
</div>

Filtra la plantilla que se usa para el formulario de inicio de sesión de Auth0. Debe devolver la ruta a un archivo que contenga HTML y sustituya lo que hay en `wp-content/plugins/auth0/templates/auth0-login-form.php`. El JS estándar de inicialización de Lock busca un atributo ID de `auth0-login-form` para instanciar el formulario de inicio de sesión, así que asegúrate de que esté presente o sustituye el archivo `wp-content/plugins/auth0/assets/js/lock-init.js` por uno propio.

<div id="auth0_settings_fields">
  ### auth0_settings_fields
</div>

Este filtro se utiliza para modificar un campo de formulario existente o para añadir uno nuevo. Debe devolver un arreglo `$options` modificado con tus cambios o incorporaciones. Los campos nuevos deben tener una función callback para el campo, como se muestra a continuación.

<div id="auth0_auth_scope">
  ### auth0_auth_scope
</div>

Este filtro permite a los desarrolladores añadir o cambiar el scope solicitado durante el inicio de sesión. Puede usarse para añadir [claims personalizadas](/es/docs/secure/tokens/json-web-tokens/create-custom-claims) o solicitar un <Tooltip tip="token de actualización: token que se utiliza para obtener un Access Token renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=Refresh+Token">token de actualización</Tooltip>.

<div id="auth0_nonce_cookie_name">
  ### auth0_nonce_cookie_name
</div>

Utiliza este filtro para modificar el nombre de la cookie que se usa para la validación de <Tooltip tip="Nonce: Número arbitrario emitido una sola vez en un protocolo de autenticación para detectar y prevenir ataques de repetición." cta="Ver glosario" href="/es/docs/glossary?term=nonce">nonce</Tooltip>. Consulta el filtro `auth0_state_cookie_name` que aparece a continuación para ver un ejemplo.

<div id="auth0_state_cookie_name">
  ### auth0_state_cookie_name
</div>

Usa este filtro para modificar el nombre de la cookie que se usa para el valor del parámetro state. Esto puede añadir un prefijo o un sufijo, o sustituir la cadena por completo. Para obtener más información sobre el parámetro state, consulta [Prevenir ataques y redirigir a los usuarios con parámetros state de OAuth 2.0](/es/docs/secure/attack-protection/state-parameters).
Asegúrate de usar caracteres válidos en cualquier modificación que realices:

`Un <cookie-name> puede contener cualquier carácter US-ASCII, excepto caracteres de control (CTL), espacios o tabulaciones. Tampoco debe contener un carácter separador como los siguientes: ( ) < > @ , ; : \ " / ? = { }.`

Para obtener más información sobre el encabezado de respuesta HTTP `Set-Cookie`, consulta [Set-Cookie en MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Set-Cookie).

<div id="auth0_settings_constant_prefix">
  ### auth0_settings_constant_prefix
</div>

Usa este filtro para cambiar el prefijo de la constante que se usa para sobrescribir la configuración del plugin. Ten en cuenta que este filtro **debe** ejecutarse antes de que se cargue el plugin de Auth0, por lo que tiene que estar ubicado en un plugin MU. Para obtener más información sobre los plugins MU, consulta [Must Use Plugins on wordpress.org](https://developer.wordpress.org/advanced-administration/plugins/mu-plugins/).

<div id="auth0_authorize_url_params">
  ### auth0_authorize_url_params
</div>

Este filtro permite a los desarrolladores ajustar los parámetros del endpoint `/authorize` según sea necesario. La función debe devolver un arreglo de tipo diccionario con parámetros de URL. Para obtener más información sobre cómo se usan estos parámetros, consulta [Authentication API Explorer: Login](https://auth0.com/docs/api/authentication#login).

<div id="auth0_authorize_url">
  ### auth0_authorize_url
</div>

Este filtro permite a los desarrolladores modificar la URL completa de `/authorize` antes de usarla. La función debe devolver una URL válida en forma de cadena. Para obtener más información sobre cómo se usa esta URL, consulta [Authentication API Explorer: Login](https://auth0.com/docs/api/authentication#login).

<div id="auth0_die_on_login_output">
  ### auth0_die_on_login_output
</div>

Este filtro te permite modificar o reemplazar el contenido HTML que se pasa a `wp_die()` cuando se produce un error durante el inicio de sesión. Este filtro no afecta el contenido de la página de verificación por correo electrónico (consulta auth0&#95;verify&#95;email&#95;page).

<div id="auth0_coo_auth0js_url">
  ### auth0_coo_auth0js_url
</div>

Este filtro te permite reemplazar la URL predeterminada de la CDN de Auth0.js al cargar la página de respaldo de COO.

<div id="auth0_slo_return_to">
  ### auth0_slo_return_to
</div>

Este filtro te permite sobrescribir la URL `returnTo` predeterminada al cerrar sesión en Auth0.

<div id="auth0_logout_url">
  ### auth0_logout_url
</div>

Este filtro te permite anular la URL de cierre de sesión de Auth0. Para obtener más información sobre cómo se usa, consulta [Cierre de sesión](/es/docs/authenticate/login/logout).

<div id="auth0_use_management_api_for_userinfo">
  ### auth0_use_management_api_for_userinfo
</div>

Este filtro determina si los datos del perfil de usuario recuperados de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> deben usarse cuando no estés usando el flujo de Login implícito. Devuelve un valor booleano `true` (predeterminado) para usar la API, o `false` para usar el <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+token">token de identidad</Tooltip>.

<div id="auth0_lock_options">
  ### auth0_lock_options
</div>

Este filtro puede usarse para modificar las opciones del formulario de inicio de sesión de Lock integrado que se usa en shortcodes, widgets y en la página wp-login.php cuando **Features &gt; <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Página de Universal Login</Tooltip>** está desactivada.

<div id="auth0_jwt_leeway">
  ### auth0_jwt_leeway
</div>

Este filtro te permite ajustar el tiempo de tolerancia utilizado para validar los tokens de ID y debe devolver un número entero de segundos.

<div id="auth0_jwt_max_age">
  ### auth0_jwt_max_age
</div>

Este filtro te permite ajustar el parámetro de URL `max_age` que se envía en la URL de autorización.

<div id="auth0_authorize_state">
  ### auth0_authorize_state
</div>

Este filtro te permite filtrar los datos de `state` antes de que se codifiquen y se utilicen en el inicio de sesión. Estos datos se verificarán después de un inicio de sesión correcto y se proporcionarán tal cual para su uso.
