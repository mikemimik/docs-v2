title: Migración de usuarios en Login by Auth0 WordPress Plugin
---

La funcionalidad de migración de usuarios utiliza una característica principal de Auth0 llamada bases de datos personalizadas, combinada con endpoints de URL en el plugin Login by Auth0, para permitir que los usuarios se autentiquen con cuentas de usuario de WordPress existentes. Para obtener más información sobre las bases de datos personalizadas, consulta [Custom Database Connections](/es/docs/authenticate/database-connections/custom-db).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando habilitas la migración de datos, el plugin expone dos endpoints seguros que permiten a Auth0 autenticar a los usuarios con cuentas de WordPress. Estos endpoints están protegidos con un token secreto y pueden configurarse para permitir únicamente las direcciones IP utilizadas por Auth0.

El flujo de inicio de sesión es el siguiente:

1. Un usuario intenta iniciar sesión con un formulario de inicio de sesión de Auth0 (integrado en tu sitio o alojado en Auth0).
2. Si Auth0 no puede encontrar un usuario asociado con las credenciales proporcionadas en tu conexión de base de datos, llama al endpoint de migración de tu sitio de WordPress con las credenciales del usuario y el token de migración.
3. El plugin busca un usuario en tu base de datos de WordPress con el nombre de usuario o correo electrónico proporcionado y verifica la contraseña.
4. Si el usuario puede autenticarse correctamente, Auth0 crea al usuario en la conexión de base de datos de tu sitio, lo autentica e inicia su sesión.
5. La próxima vez que el usuario inicie sesión, usará el usuario de Auth0 y se omitirá el endpoint de migración.

La Migración de usuarios debe configurarse cuando el sitio se conecta a Auth0 por primera vez. Si intentas activar o desactivar los scripts de base de datos personalizada para una conexión de base de datos que ya ha tenido usuarios, la operación fallará. Consulta la sección **solución de problemas** para obtener más información sobre cómo cambiar entre modos.

<div id="set-up-and-configuration">
  ## Instalación y configuración
</div>

La forma más sencilla de llevar a cabo la Migración de usuarios es usar el Setup Wizard al instalar el plugin por primera vez. Para obtener más información sobre el proceso, consulta [Install Login by Auth0](/es/docs/customize/integrations/cms/wordpress-plugin/install-login-by-auth0).

Si el Setup Wizard de Migración de usuarios no pudo completarse o si quieres ver el proceso con más detalle, sigue los pasos a continuación. Una vez más, se parte de cero con una conexión de base de datos que no tiene ningún usuario. El siguiente proceso debe realizarse en un sitio sin tráfico o con el modo de mantenimiento activado.

1. [Cree y configure correctamente una aplicación](/es/docs/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0), y cree y active una conexión de base de datos vacía para la aplicación. Pueden ser las mismas que se crearon en el proceso estándar del Setup Wizard, o puede crearlas desde cero. Para obtener más información sobre el Setup Wizard, lea [Instalar Login by Auth0](/es/docs/customize/integrations/cms/wordpress-plugin/install-login-by-auth0).
2. En la pantalla **Auth0 &gt; Settings** de WordPress, asegúrese de que el Domain, el ID de cliente y el secreto de cliente de la aplicación estén guardados en los campos correctos de la pestaña **Basic**.
3. En la vista **Advanced**, habilite la configuración **User Migration Endpoints** y seleccione **Save Changes**. Si está usando configuraciones basadas en constantes, establezca `AUTH0_ENV_MIGRATION_WS` en `true` y `AUTH0_ENV_MIGRATION_TOKEN` en una cadena aleatoria segura de al menos 16 caracteres, sin comillas simples ni barras invertidas.
4. Debajo de esta configuración, ahora debería ver un **Security Token**. Mantenga esta página abierta, ya que necesitará este valor más adelante en el proceso.
5. En el Auth0 Dashboard, vaya a la conexión de base de datos que quiere usar y habilite **Requires Username** e **Import Users to Auth0**.
6. Seleccione la vista **Custom Database** y habilite **Use my own database**.
7. Debajo de esta configuración, en **Scripts de acción de base de datos**, debería haber dos pestañas: una para **Login** y otra para **Get User**.
8. Seleccione la vista **Login**, borre el código existente, [copie el código db-login.js del repositorio de GitHub](https://raw.githubusercontent.com/auth0/wp-auth0/4.4.0/lib/scripts-js/db-login.js) y péguelo en el editor de código.
9. **Este paso es para las versiones 3.10.0 y anteriores:** Busque `{THE_WS_URL}` y reemplácelo por la URL del sitio de su instancia de WordPress, seguida de `/index.php?a0_action=migration-ws-login`. La URL del sitio se puede encontrar en la pantalla **Settings &gt; General** de wp-admin. Puede probarlo pegando la URL completa en su navegador. Debería ver `{"status":401,"error":"Unauthorized"}`.
10. **Este paso es para las versiones 3.10.0 y anteriores:** Busque `{THE_WS_TOKEN}` y reemplácelo por el token que aparece en la configuración **User Migration Endpoints**.
11. No debería haber errores en el editor. Si todo se ve bien, haga clic en **Save** en la parte superior.
12. **Este paso es para la versión 3.11.0 y posteriores:** Desplázate hacia abajo hasta **Configuración** y agrega las siguientes variables de configuración:

    * `endpointUrl` configurado con la URL del sitio de tu instancia de WordPress (**wp-admin &gt; Settings &gt; General &gt; campo &quot;Site URL&quot;**), seguida de `/index.php?a0_action=`.
    * `migrationToken` configurado con el valor del token de seguridad que aparece en el paso 4 anterior.
    * `userNamespace` configurado con el nombre de tu aplicación en Auth0 o con cualquier otro valor que solo incluya letras, números y guiones.

    <Frame>![Configuración de la base de datos personalizada en el WordPress Plugin](/docs/images/cdy7uua7fh8z/1icr0yX47nODUrzP7B6Xl2/b43621c3b6641f68bcb14ac38b237c11/auth0-custom-database-config.png)</Frame>
13. Haz clic en el botón **Try** en la parte superior y usa una cuenta de usuario válida de WordPress en el formulario que aparece. Deberías ver &quot;The profile is&quot; seguido de los datos del usuario. Si no es así, consulta la sección de **Solución de problemas** a continuación.
14. Selecciona la vista **Get User**, borra el código existente, [copia el código de db-get-user.js del repositorio de GitHub](https://raw.githubusercontent.com/auth0/wp-auth0/4.4.0/lib/scripts-js/db-get-user.js) y pégalo en el editor de código.
15. **Este paso es para la versión 3.10.0 y anteriores:** Busca `{THE_WS_URL}` y sustitúyelo por la URL del sitio de tu instalación de WordPress, seguida de /index.php?a0&#95;action=`migration-ws-get-user`. La URL del sitio se encuentra en la pantalla **Settings &gt; General** de wp-admin. Puedes probarlo pegando la URL completa en tu navegador. Deberías ver `{"status":401,"error":"Unauthorized"}`.
16. **Este paso es para la versión 3.10.0 y anteriores:** Busca `{THE_WS_TOKEN}` y sustitúyelo por el token que aparece en la configuración de **Endpoints de Migración de usuarios**.
17. No debería haber errores en el editor. Si todo se ve bien, haz clic en **Save**.
18. Haz clic en el botón **Try** en la parte superior y usa el correo electrónico de una cuenta de usuario válida de WordPress en el formulario que aparece. Deberías ver &quot;The profile is&quot; seguido de los datos del usuario. Si no es así, consulta la sección de **Solución de problemas** a continuación.
19. En una nueva sesión del navegador, ve a una página de inicio de sesión del sitio de WordPress e intenta iniciar sesión (el usuario no debería existir ya en la base de datos). Notarás que el proceso de inicio de sesión tarda un poco más de lo habitual la primera vez, pero debería completarse correctamente. Los inicios de sesión posteriores serán más rápidos.
20. (OPCIONAL) Para activar seguridad adicional en los endpoints de migración, ve a la pantalla **Auth0 &gt; Settings** en WordPress, actívala y luego haz clic en **Save Changes**. Intenta iniciar sesión con un usuario distinto para comprobar que Auth0 todavía puede acceder a los endpoints.

En este punto, la configuración de Migración de usuarios ya está completa, y los usuarios existentes de WordPress se irán migrando gradualmente a la conexión de base de datos de Auth0.

<div id="troubleshooting">
  ## Solución de problemas
</div>

Los problemas con la Migración de usuarios suelen deberse a unas pocas causas:

* URL o token incorrectos en los scripts de la base de datos personalizada.
* La lista de permitidos de IP está activada, pero con direcciones IP incorrectas.
* endpoints restringidos o almacenados en caché en tu instancia de WordPress.

La mejor manera de empezar a solucionar el problema es usar el botón **Try** del script **Login**, que se encuentra en la vista Custom Database de la conexión de base de datos que se está usando en [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database). A continuación, se muestran los mensajes de error que podrías ver y los pasos que debes seguir para solucionarlo.

<div id="unexpected-token-in-json-at-position-0">
  ### Token inesperado &lt; en JSON en la posición 0
</div>

Esto significa que el script personalizado no está recibiendo los datos en un formato que pueda usar. Lo más probable es que esto se deba a una URL de endpoint incorrecta en el script de base de datos.

Primero, copia la URL de la línea 10 del script y pégala en tu navegador. Si el endpoint es correcto, debería mostrar uno de estos dos mensajes:

`{"status":401,"error":"Unauthorized"}`

`// or`

`{"status":403,"error":"Forbidden"}`

Si lo que ves es la página de inicio o un error 404, la URL es incorrecta. Busca la URL de tu sitio en **Settings &gt; General &gt; Site URL** en el panel de administración de WordPress. Agrega `/index.php?a0_action=migration-ws-login` al final para el script de Login y `/index.php?a0_action=migration-ws-get-user` al final para el script de Get User.

* **Para las versiones 3.10.0 y anteriores**: El valor de la URL debería aparecer en el propio script como primer parámetro en la llamada a `request.post`.
* **Para las versiones 3.11.0 y posteriores**: El valor del token debe guardarse en una variable de configuración. Agrega lo siguiente en la primera línea de la función y usa el botón **Try** para ver qué se almacena en `endpointUrl`:

`callback(null, configuration);`

Si estás seguro de que las URL son correctas y el problema persiste, consulta con tu proveedor de hosting para asegurarte de que esas URL no estén en caché ni restringidas de ninguna manera.

<div id="wrong-email-or-password">
  ### Correo electrónico o contraseña incorrectos
</div>

Este es el error predeterminado que se muestra cuando algo más falla. La forma más sencilla de diagnosticar lo que está ocurriendo es mostrar temporalmente el error que se devuelve (por defecto, estos errores son opacos para evitar mostrar información que pueda dar pistas a los atacantes).

En la línea 30 del script de Login, cambia:

`callback(null);`

por:

`callback(wpUser.error);`

Guarda el script y vuelve a probar la conexión. Deberías ver uno de los siguientes mensajes y poder identificar el problema con los pasos que se indican a continuación. Una vez que lo hayas resuelto, vuelve a dejar el script como estaba.

<div id="forbidden">
  ### Prohibido
</div>

Esto significa que los endpoints de migración están desactivados en tu instalación de WordPress. En WordPress, ve a **Auth0 &gt; Settings &gt; Advanced** y activa **User Migration Endpoints**. Asegúrate de que el token que aparece allí sea el mismo que se usa en ambos scripts de base de datos personalizada:

* **Para las versiones 3.10.0 y anteriores**: El valor del token debe aparecer en el propio script después de `access_token:`
* **Para las versiones 3.11.0 y posteriores**: El valor del token debe guardarse en una variable de configuración. Agrega lo siguiente en la primera línea de la función y usa el botón **Try** para ver qué se almacena en `migrationToken`:

`callback(null, configuration);`

<div id="unauthorized">
  ### No autorizado
</div>

Esto significa que la lista de IP permitidas para la migración está activada, pero la dirección IP entrante no está en la lista. Justo debajo del script de Login deberías ver una lista de direcciones IP:

<Frame>![WordPress Migración de usuarios - Direcciones IP de Auth0](/docs/images/cdy7uua7fh8z/3RnxoRlvXm8FmNfu02fXVn/3672806c9cbf2bfe28503be8b8a63eda/auth0-incoming-ip-addresses.png)</Frame>

Asegúrate de que todas esas direcciones IP aparezcan en WordPress, en **Auth0 &gt; Settings &gt; Advanced**, dentro del plugin:

<Frame>![WordPress Migración de usuarios - Lista de IP permitidas](/docs/images/cdy7uua7fh8z/3w6C1IrsSC9hoivpSY122l/d117857524ef8d7872f364d3a7dbabda/migration-ip-whitelist-setting-field.png)</Frame>

Si una o más de las direcciones IP que aparecen en Auth0 no aparecen en WordPress, agrega las que falten en el campo y guarda la página de Settings. Además, [crea una publicación en Auth0 Community](https://community.auth0.com/tags/wordpress) (y etiquétala como &quot;wordpress&quot;) que incluya las direcciones IP que faltan, para que podamos solucionar el problema.

<div id="unauthorized-missing-authorization-header">
  ### No autorizado: falta el encabezado Authorization
</div>

El <Tooltip tip="Token de seguridad: artefacto firmado digitalmente que se usa para demostrar que el usuario se autenticó correctamente." cta="Ver glosario" href="/es/docs/glossary?term=security+token">token de seguridad</Tooltip> no está en el script de base de datos (línea 16), o tu servidor no está procesando los encabezados correctamente. Revisa el script de Login y asegúrate de que el token exista y coincida con el de WordPress. Si el token está ahí y es correcto, tendrás que hablar con tu proveedor de hosting para habilitar el procesamiento del encabezado `Authorization`. Si necesitas ayuda con la solución de problemas del servidor, lee [Apache 2.4 + PHP-FPM and Authorization headers on stackoverflow.com](https://stackoverflow.com/questions/17018586/apache-2-4-php-fpm-and-authorization-headers). Para ver cómo se recupera el token, consulta el [código del plugin en el repositorio de GitHub.](https://github.com/auth0/wp-auth0/blob/master/lib/WP_Auth0_Routes.php#L138)

<div id="invalid-token">
  ### Token no válido
</div>

El token de seguridad del script de base de datos es incorrecto. Revisa la línea 16 del script de Login y asegúrate de que el token coincida con el de WordPress.

<div id="invalid-credentials">
  ### Credenciales inválidas
</div>

La dirección de correo electrónico y/o la contraseña que estás usando son incorrectas. Verifica que estés introduciendo la dirección de correo electrónico correcta y que la contraseña también lo sea. Puedes restablecer la contraseña del usuario por otra para asegurarte de que tienes la correcta.

<div id="cannot-change-email-or-incorrect-user-data">
  ### No se puede cambiar el correo electrónico o hay datos de usuario incorrectos
</div>

Si usas más de una conexión de base de datos personalizada en tu tenant de Auth0 y no puedes cambiar la dirección de correo electrónico, o se están guardando datos de usuario para el usuario incorrecto, es probable que tengas IDs de usuario superpuestos en Auth0. Este problema se corrigió para los sitios nuevos que instalan la versión 3.11.0, pero en el caso de las conexiones creadas antes de eso, tendrás que corregirlo manualmente de una de las siguientes formas:

* Si no tienes datos de usuario almacenados que deban conservarse (si solo usas la conexión para permitir el inicio de sesión y no almacenas metadatos), puedes crear una nueva conexión de base de datos personalizada siguiendo los pasos anteriores (usando las notas de la versión 3.11.0) y cambiar la aplicación a esta nueva conexión (asegúrate de desactivar la conexión anterior). La migración se reiniciará y no habrá impacto en la experiencia de usuario.
* Si sí tienes datos en Auth0 que deben conservarse, puedes [usar nuestra extensión User Import/Export](/es/docs/manage-users/user-migration/user-import-export-extension) para ajustar los datos de usuario.
  1. Crea una nueva conexión de base de datos personalizada siguiendo los pasos anteriores (usando las notas de la versión 3.11.0).
  2. Exporta todos los usuarios de la conexión actual (te recomendamos poner tu sitio en modo de mantenimiento mientras realizas el cambio para que no se omita ningún usuario).
  3. Cambia todos los IDs de usuario para agregar el espacio de nombres usado al crear la nueva conexión. Los IDs de usuario deberían pasar de algo como `auth0|123` a `auth0|Your-WP-Site-Name|123`. Ajusta también cualquier otro campo necesario para que cumpla con el esquema de importación. Para obtener más información, consulta [Bulk User Import Database Schema and Examples](/es/docs/manage-users/user-migration/bulk-user-import-database-schema-and-examples).
  4. Activa la nueva conexión y desactiva la anterior para tu aplicación.
  5. Importa los nuevos datos de usuario en la nueva conexión y haz pruebas.
* Si tienes una cuenta de pago, puedes ponerte en contacto con nuestro equipo de soporte para ejecutar un script de actualización de la base de datos que cambie los IDs de usuario a una versión con espacio de nombres y agregue ese espacio de nombres a tu script de base de datos al mismo tiempo (paso 12 en **instalación y configuración** arriba).
