Aquí tienes algunas preguntas frecuentes sobre resolución de problemas. Si los elementos que aparecen a continuación no resuelven tu problema, considera estas alternativas:

* Si estás configurando el plugin por primera vez o tienes problemas para que los usuarios inicien sesión, consulta [Configure Login by Auth0](/docs/es/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0).
* Si encontraste un error en el código del plugin, [envía una incidencia](https://github.com/auth0/wordpress/issues) o [crea un pull request](https://github.com/auth0/wordpress/pulls) en GitHub.
* Si tienes preguntas sobre cómo usar Auth0 o el plugin, [publica tu consulta en nuestro sitio de la Community](https://community.auth0.com/) o [crea una solicitud en el foro de soporte de WordPress del plugin](https://wordpress.org/support/plugin/auth0/).
* Para consultar documentación adicional y encontrar más respuestas, [visita nuestro sitio de soporte](https://support.auth0.com/center/s/). Los clientes con un plan de pago de Auth0 pueden enviar un ticket de soporte para recibir una respuesta rápida.

<div id="im-seeing-the-error-message-invalid-state-that-prevents-me-from-logging-in">
  ## Me aparece el mensaje de error &quot;Invalid state&quot; y no puedo iniciar sesión
</div>

La validación de `state` es una función de seguridad añadida en la [versión 3.6.0](https://github.com/auth0/wordpress/releases/tag/3.6.0). Este mensaje de error suele deberse a una URL de devolución de llamada almacenada en caché (consulta la Configuración de la aplicación para ver qué URL de devolución de llamada no deben almacenarse en caché). Para obtener más información, lee [Solucionar problemas de errores &quot;Invalid state&quot; del plugin para WordPress](/docs/es/customize/integrations/cms/wordpress-plugin/troubleshoot-wordpress-plugin-invalid-state-errors).

<div id="im-having-an-issue-logging-in-or-changing-emailpassword-using-a-custom-database">
  ## Tengo un problema al iniciar sesión o al cambiar el correo electrónico o la contraseña con una base de datos personalizada
</div>

Para conocer las posibles soluciones, lee [Migración de usuarios en Login by Auth0 plugin para WordPress](/docs/es/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).

<div id="im-seeing-the-error-message-invalid-id-token-or-expired-id-token-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;token de identidad no válido&quot; o &quot;token de identidad vencido&quot;, que me impide iniciar sesión
</div>

Esto suele deberse a que el servidor tiene la hora incorrecta. Si el mensaje de error incluye &quot;used too early,&quot; significa que la hora de tu servidor está configurada en el futuro. Si dice que el token ha vencido, entonces la hora del servidor está configurada demasiado en el pasado. Es habitual que haya una diferencia horaria entre dos servidores. Ejecuta `echo date(DateTime::ISO8601)` en PHP en tu servidor y compáralo, incluidos los segundos, con la hora UTC actual. Si la hora de tu servidor difiere de la hora UTC por más de 60 segundos (el margen predeterminado), tendrás que establecer un margen mayor para compensar el desfase del reloj de tu servidor. Puedes pegar el siguiente código en el `functions.php` de tu tema o en cualquier otro lugar donde se ejecute después de que se cargue el plugin y antes de que se ejecute el hook de inicio de sesión:

```js wrap lines
add_filter( 'auth0_jwt_leeway', function( $default_leeway ) { return 120; } );
```

Esto daría un margen de 120 segundos. Puede que tenga que ajustarlo en función del desfase horario de su servidor.

<div id="i-see-the-error-message-this-account-does-not-have-an-email-associated-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;This account does not have an email associated...&quot; que me impide iniciar sesión
</div>

Si aparece este error, asegúrate de solicitar una dirección de correo electrónico de cada proveedor en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en **Connections &gt; Social** (expande cada proveedor). Ten en cuenta que no todos los proveedores devuelven direcciones de correo electrónico de los usuarios (p. ej., X). Si esto sucede, siempre puedes añadir una dirección de Email a cualquier usuario que haya iniciado sesión a través del Auth0 Dashboard (o de la API). Consulta **Users &gt; Edit**.

Para las conexiones que no proporcionan un indicador `email_verified` (algunas conexiones empresariales no lo incluyen) o para omitir esta validación en determinadas Social Connections, agrega la strategy de esa conexión en el campo &quot;Skip Strategies&quot;. Este campo se encuentra debajo del interruptor **Require Verified Email**, al que se accede desde **wp-admin** &gt; **Auth0** &gt; **Settings** &gt; **Advanced**.

**Este campo solo debe usarse si es necesario, ya que elude las medidas de seguridad recomendadas por Auth0.**

<div id="i-see-the-error-message-there-is-a-user-with-the-same-email-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;There is a user with the same email&quot; que me impide iniciar sesión
</div>

Esto significa que hay un usuario en WordPress que tiene el mismo correo electrónico que el que se está usando para iniciar sesión, asociado a un usuario diferente de Auth0. Si estás probando el plugin o quieres asociar el usuario existente con el nuevo usuario de Auth0:

1. Inicia sesión como administrador
2. Ve a **wp-admin &gt; Users** y busca el correo electrónico que se está usando
3. Ve al perfil del usuario y desplázate hasta el final
4. Haz clic en **Delete Auth0 Data** y confirma

Si tienes 2 cuentas de usuario en Auth0 con la misma dirección de correo electrónico, este mensaje de error seguirá apareciendo. Te recomendamos que [vincules cuentas de usuario](/docs/es/manage-users/user-accounts/user-account-linking/link-user-accounts).

<div id="i-see-the-error-message-failed-cross-origin-authentication-or-no-verifier-returned-from-client-in-my-browsers-console-logs-when-trying-to-log-in">
  ## Veo el mensaje de error &quot;Failed cross origin authentication&quot; o &quot;No verifier returned from client&quot; en los registros de la consola de mi navegador al intentar iniciar sesión
</div>

Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), selecciona tu aplicación de WordPress para ver su configuración y comprueba que **Allowed Callback URLs** y **Allowed Origins (CORS)** coincidan con tu sitio de WordPress.

Si usas un navegador basado en Chromium, asegúrate de no tener desactivadas las cookies de terceros. Para obtener más información, consulta [Autenticación de origen cruzado](/docs/es/authenticate/login/cross-origin-authentication).

<div id="i-need-to-rerun-the-setup-wizard-but-i-dont-see-that-menu-option-anymore">
  ## Necesito volver a ejecutar el Setup Wizard, pero ya no veo esa opción en el menú.
</div>

Esto significa que el plugin ya está configurado con un Domain, <Tooltip tip="ID de cliente: Valor de identificación otorgado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+ID">ID de cliente</Tooltip>, y <Tooltip tip="ID de cliente: Valor de identificación otorgado por Auth0 a tu recurso registrado." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>. Ejecutar el Setup Wizard por segunda vez puede producir resultados impredecibles. Si estás configurando WordPress por primera vez y quieres empezar de nuevo antes de que se haya producido algún login:

1. Ve a **wp-admin** &gt; **Auth0** &gt; **Settings** &gt; **Basic**.
2. Elimina el Domain y el ID de cliente. Desplázate hacia abajo y haz clic en **Save Changes**.
3. Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
4. Busca la aplicación que creó WordPress (su nombre debería ser el nombre de tu sitio de WordPress).
5. Selecciona la aplicación para ver su configuración. Desplázate hacia abajo y selecciona **Delete Application**.
6. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database).
7. Busca la conexión que creó WordPress (su nombre debería ser el nombre de tu sitio de WordPress con &quot;DB-&quot; al principio).
8. Selecciona la conexión para ver su configuración. Desplázate hacia abajo y selecciona **I Want To Delete This Connection.** Ten en cuenta que esto eliminará la conexión y todos los usuarios que se hayan creado en ella.
9. Vuelve a WordPress. Ahora verás la opción Setup Wizard en Auth0 dentro del menú de administración.

<div id="how-do-i-setup-passwordless-login">
  ## ¿Cómo configuro el inicio de sesión passwordless?
</div>

<Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Passwordless">Passwordless</Tooltip> El inicio de sesión passwordless es posible en cualquier sitio web habilitado con Auth0 mediante correo electrónico o SMS. Para que funcione en tu sitio de WordPress:

1. Activa **Passwordless Login** en la vista **Features** de la configuración del plugin y guarda los cambios.
2. Ve a [Auth0 Dashboard &gt; Authentication &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless).

* Para usar correo electrónico, activa la conexión **Email** y modifica la configuración si lo deseas. Esto habilitará el inicio de sesión con código por correo electrónico (se envía a los usuarios un código por correo que luego introducen en el formulario de login de tu sitio).
* Para usar un &quot;Magic Link&quot; (el enlace enviado por correo iniciará sesión automáticamente), agrega `{passwordlessMethod: 'code'}` al campo &quot;Extra Settings&quot; en la pestaña **Advanced** de la configuración del plugin.
* Para usar el inicio de sesión por SMS, activa la conexión **SMS** y sigue los pasos para configurar una cuenta de desarrollador de Twilio (según el uso, esto puede requerir una cuenta de pago de Twilio).

El formulario de login de Auth0 seleccionará un método passwordless según la conexión que hayas activado antes. Si tienes ambas conexiones activas, usará el correo electrónico de forma predeterminada. En ese caso, desactiva la conexión de correo electrónico para mostrar SMS o agrega `sms` al campo **Connections** en la pestaña **Advanced** de la configuración del plugin.

<div id="i-have-two-accounts-for-the-same-user-in-wordpress">
  ## Tengo dos cuentas para el mismo usuario en WordPress
</div>

En algunas situaciones, es posible que un usuario termine teniendo dos cuentas. WordPress le permite fusionar usuarios eliminando una de las cuentas y asignando el contenido de esa cuenta a otro usuario. Vaya a wp-admin &gt; Users, seleccione la cuenta que desea eliminar y, en el cuadro de diálogo de confirmación, seleccione otro usuario al que transferir el contenido.

<div id="my-configuration-is-wrong-and-i-cant-authenticate-using-auth0-is-there-another-way-to-access-the-plugin">
  ## Mi configuración es incorrecta y no puedo autenticarme con Auth0. ¿Hay alguna otra forma de acceder al plugin?
</div>

Se puede acceder al plugin con credenciales válidas de WordPress a través del inicio de sesión habitual de WordPress añadiendo `?wle` a la URL de inicio de sesión. Por ejemplo: `http://yourdomain.com/wp-login.php?wle`.

<div id="i-am-having-problems-when-a-user-logs-in-where-can-i-find-a-log-of-what-is-happening">
  ## Tengo problemas cuando un usuario inicia sesión. ¿Dónde puedo encontrar un registro de lo que está pasando?
</div>

El plugin proporciona un registro de errores donde puedes revisar qué ha ocurrido. Accede a él desde el subelemento **Registro de errores** del menú del plugin **Auth0**. Puedes encontrar información adicional en [Auth0 Dashboard &gt; monitoreo &gt; registros](https://manage.auth0.com/#/logs).

<div id="how-can-i-show-the-widget-or-shortcode-in-signup-mode-by-default">
  ## ¿Cómo puedo mostrar el widget o shortcode en modo de registro por defecto?
</div>

Puedes usar la opción `Extra configuration` del widget (o el atributo `extra_conf` en el shortcode) y añadir este JSON `{"mode":"signup" }`, lo que hará que el plugin se muestre en este modo.

<div id="when-using-a-plugin-to-force-the-login-the-user-is-not-logged-in">
  ## Al usar un plugin para forzar el inicio de sesión, el usuario no queda conectado
</div>

Esto suele deberse a una página almacenada en caché después de iniciar sesión. Consulta con tu proveedor de alojamiento qué estrategias puedes aplicar para mitigarlo, o intenta añadir un parámetro a la URL para evitar la caché. Para obtener más información, [consulta este gist en GitHub](https://gist.github.com/joshcanhelp/e3eb693749f0fe66aad097c3bbb3b415).

<div id="the-user-is-not-logged-in-when-using-the-force-login-plugin">
  ### El usuario no inicia sesión al usar el plugin &quot;Force Login&quot;
</div>

Esto se debe a que la URL de devolución de llamada no se ha añadido a la lista de permitidos. Prueba a añadir este código al filtro `my_forcelogin_whitelist`:

```javascript lines
function wp_auth0_forcelogin_whitelist( $whitelist ) {
  if ( ! empty( $_GET['auth0'] ) ) {
    $whitelist[] = site_url($_SERVER['REQUEST_URI']);
  }
  return $whitelist;
}
add_filter('v_forcelogin_whitelist', 'wp_auth0_forcelogin_whitelist', 10, 1);
```

<div id="how-can-i-redirect-the-users-to-a-specific-url-after-login">
  ## ¿Cómo puedo redirigir a los usuarios a una URL específica después del inicio de sesión?
</div>

Todos los redireccionamientos se comprueban con `wp_safe_redirect()` antes de realizarse. Si intentas redirigir a un dominio que no es tu dominio principal, añade ese dominio a la comprobación mediante el filtro `allowed_redirect_hosts` del núcleo de WordPress. Para obtener más información, consulta [apply&#95;filters( &#39;allowed&#95;redirect&#95;hosts&#39;, string[] $hosts, string $host ) en wordpress.org](https://developer.wordpress.org/reference/hooks/allowed_redirect_hosts/).

<div id="on-the-login-page">
  ### En la página de inicio de sesión
</div>

Este plugin aprovecha las funciones de WordPress para funcionar sin problemas con la configuración predeterminada. Para añadir una redirección, puedes agregar el parámetro de consulta `redirect_to` a la URL cuando dirijas al usuario a la página de inicio de sesión. El plugin redirigirá al usuario a esa URL después de iniciar sesión correctamente.

También puedes usar la opción **URL de redirección de inicio de sesión** en la página de configuración del plugin de Auth0. Esta URL se utilizará para redirigir al usuario cuando no se proporcione el parámetro `redirect_to`.

<div id="using-the-widget">
  ### Uso del widget
</div>

El widget redirigirá automáticamente a la misma página en la que estaba el usuario antes de la autenticación. Puedes cambiar este comportamiento con la opción `Redirect after login:`.

<div id="using-the-shortcode">
  ### Uso del shortcode
</div>

El shortcode redirigirá automáticamente a la misma página en la que se encontraba el usuario antes de la autenticación. Puedes cambiar este comportamiento mediante el atributo `redirect_to`.

<div id="how-can-i-migrate-my-wordpress-users-to-auth0">
  ## ¿Cómo puedo migrar mis usuarios de WordPress a Auth0?
</div>

La versión actual del plugin no ofrece una forma de migrar usuarios a Auth0 automáticamente, pero tiene algunas opciones:

* El plugin expone dos endpoints para marcar su conexión de base de datos personalizada en modo de **importación a Auth0**. Para obtener más información sobre los endpoints, lea [Configurar la migración automática desde su base de datos](/docs/es/manage-users/user-migration/configure-automatic-migration-from-your-database).
* Exporte sus datos de usuario a un archivo JSON y cárguelo para importarlos por lotes a Auth0. Al principio, sus usuarios tendrán que restablecer sus contraseñas al iniciar sesión con Auth0, ya que Auth0 no puede descifrar las contraseñas de WordPress durante la migración. Para saber cómo generar el archivo JSON, lea [Bulk User Imports](/docs/es/manage-users/user-migration/bulk-user-imports). Luego, tendrá que cargar el archivo mediante el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0 y su [endpoint Import Users](https://auth0.com/docs/api/management/v2/jobs/post-users-imports).
* [Use este script](https://gist.github.com/glena/b31716e3c8fe48927be2) y el endpoint XML-RPC de WordPress para configurar el flujo de migración mediante una conexión de base de datos personalizada. Para obtener más información sobre cómo configurar el flujo de migración, lea [Configurar la migración automática desde su base de datos](/docs/es/manage-users/user-migration/configure-automatic-migration-from-your-database). Para obtener más información sobre el endpoint XML-RPC de WordPress, lea [Soporte de XML-RPC en wordpress.org](https://codex.wordpress.org/XML-RPC_Support).

<div id="the-form_title-setting-is-ignored-when-i-set-up-the-dict-setting">
  ## La configuración `form_title` se ignora cuando configuro la opción `dict`
</div>

Internamente, el plugin usa la configuración `dict` para cambiar el título del widget de Auth0. Cuando configuras el campo `dict`, este sobrescribe `form&#95;title`.

Para cambiar `form&#95;title` en este caso, debes añadir el siguiente atributo al JSON de `dict`:

```json lines
{
    "signin": {
        "title": "The desired form title"
    }
}
```

<div id="how-can-i-modify-the-embedded-auth0-login-form">
  ## ¿Cómo puedo modificar el formulario de inicio de sesión integrado de Auth0?
</div>

Hay muchas opciones en la pestaña **Apariencia** de la página de configuración del plugin que pueden cambiar el aspecto del formulario de inicio de sesión integrado en su sitio (página `wp-login/php`, shortcodes o widgets). Para obtener más información sobre estas opciones, lea [Configure Login by Auth0](/docs/es/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0). Esto no afectará al formulario de inicio de sesión de la página de <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> alojada en Auth0.

También hay un campo llamado &quot;Configuración adicional&quot; en la pestaña **Avanzado** que acepta un objeto JSON válido con toda la configuración que desee establecer. Esto anulará cualquier cambio realizado en la pestaña **Apariencia**. Para ver todas las opciones de configuración disponibles, lea [Lock Configuration Options](/docs/es/libraries/lock/lock-configuration).

Las hojas de estilo externas y los archivos JS deben cargarse en su tema mediante el hook `wp_enqueue_scripts` para shortcodes/widgets y el hook `login_enqueue_scripts` para `wp-login.php`. Para obtener más información, lea [`do_action( 'wp_enqueue_scripts' )` en wordpress.org](https://developer.wordpress.org/reference/hooks/wp_enqueue_scripts/) y [`do_action( 'login_enqueue_scripts' )` en wordpress.org](https://developer.wordpress.org/reference/hooks/login_enqueue_scripts/).

<div id="the-session-expires-too-soon">
  ## La sesión expira demasiado pronto
</div>

El plugin de Auth0 no gestiona las sesiones; usa la configuración de WordPress. De forma predeterminada, las sesiones de usuario se mantienen activas durante dos días. Puedes habilitar la opción `Remember users session` en la pestaña **Advanced** de la configuración del plugin para permitir que las sesiones sigan activas durante un máximo de 14 horas.

<div id="how-do-i-implement-a-refresh-token">
  ## ¿Cómo implemento un token de actualización?
</div>

Implementamos parámetros adicionales en los métodos de inicio de sesión que usa el plugin para admitir <Tooltip tip="Token de actualización: token que se utiliza para obtener un nuevo token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=Refresh+Tokens">tokens de actualización</Tooltip>. Usa el filtro `auth0_auth_scope` combinado con la acción `auth0_user_login` para hacerlo.

<div id="profile-data-saved-in-wordpress-is-not-being-synced-to-the-auth0-user-account">
  ## Los datos del perfil guardados en WordPress no se sincronizan con la cuenta de usuario de Auth0.
</div>

Esta es una limitación actual del plugin, aunque estamos estudiando abordarla en una futura versión. La única excepción es la contraseña del usuario. Si la contraseña se cambia en WordPress y cumple la política de seguridad establecida para la conexión de base de datos en uso, esa contraseña también se actualizará para el usuario de Auth0. En una futura versión añadiremos un mensaje de error para detener el proceso si la contraseña no es lo bastante segura.

<div id="how-do-i-migrate-from-social-login-with-auth0-to-login-by-auth0">
  ## ¿Cómo migro de &quot;Social Login with Auth0&quot; a &quot;Login by Auth0&quot;?
</div>

Históricamente, Auth0 ha mantenido dos plugins de WordPress:

* [Login by Auth0](https://wordpress.org/plugins/auth0/)
* [Social Login with Auth0](https://wordpress.org/plugins/social-login-with-auth0/)

Estos dos plugins son prácticamente iguales, pero **Social Login with Auth** no recibirá más actualizaciones después de la versión 3.7.0 (publicada el 13 de agosto de 2018). Migrar de **Social Login with Auth** a **Login by Auth0** es sencillo y no provocará ninguna pérdida de datos ni en Auth0 ni en WordPress.

Cambiar de **Social Login with Auth** a **Login by Auth0** actualizará el número de versión que ves, así que asegúrate de probar este cambio primero en un servidor de staging o de desarrollo (igual que harías al actualizar el plugin en wp-admin). Además, es posible que los inicios de sesión no funcionen durante el proceso de migración, así que asegúrate de usar un plugin de modo de mantenimiento o de completar la migración durante horas de baja actividad.

La forma más sencilla de migrar es mediante (S)FTP:

1. Inicia sesión en el sitio de WordPress como administrador.
2. [Descarga Login by Auth0](https://downloads.wordpress.org/plugin/auth0.zip) y descomprímelo localmente.
3. Desactiva el plugin **Social Login with Auth0** desde la pantalla WordPress admin &gt; Plugins.
4. Inicia sesión en el servidor al que quieres migrar y ve a `wp-content/plugins`.
5. Mueve la carpeta `social-login-with-auth0` fuera de la carpeta de plugins para hacer una copia de seguridad del contenido.
6. Sube la nueva carpeta del plugin `auth0` al directorio de plugins.
7. Activa el nuevo plugin &quot;Login by Auth0&quot; desde la pantalla WordPress **Admin** &gt; **Plugins**.

Si no puedes acceder al sitio por FTP, también puedes realizar el proceso directamente desde el administrador de WordPress:

1. Inicia sesión en el sitio de WordPress como administrador.
2. Ve a **Auth0** &gt; **Import-Export Settings**.
3. Haz clic en **Export Settings** y luego en **Export**.
4. Desactiva el plugin **Social Login with Auth0** desde la pantalla WordPress **Admin** &gt; **Plugins**.
5. Elimina el plugin **Social Login with Auth0** y confirma.
6. Ve a **Plugins** &gt; **Add New** y busca &quot;Auth0&quot;.
7. En el plugin **Login by Auth0** (asegúrate de verificar el nombre), haz clic en **Install Now**.
8. Cuando termine, haz clic en **Activate**.
9. Revisa **Auth0** &gt; **Settings** y asegúrate de que se mantenga la configuración anterior. Si no es así:

   1. Ve a **Auth0** &gt; **Import-Export Settings**.
   2. Pega el JSON de configuración exportado anteriormente y haz clic en **Import**.
10. Elimina por completo el archivo JSON de configuración exportado (contiene información confidencial).

Ahora todo debería funcionar como se espera con el nuevo plugin y las actualizaciones se reanudarán con normalidad.
