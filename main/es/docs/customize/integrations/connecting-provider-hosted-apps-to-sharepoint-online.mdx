Auth0 puede simplificar enormemente el proceso de autenticación para las aplicaciones de SharePoint. Auth0 negociará un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">token de acceso</Tooltip> que luego podrás usar para llamar a las API de SharePoint.

No necesitarás ninguna biblioteca especial. Puedes usar cualquiera de los SDK compatibles con Auth0.

<div id="step-1-register-your-application-in-auth0">
  ## Paso 1. Registra tu aplicación en Auth0
</div>

Solo tienes que registrar una nueva aplicación en Auth0 como lo harías normalmente: **Applications &gt; NEW**. Elige cualquiera de los SDK disponibles para ver instrucciones detalladas. Ten a mano el `client_id`, ya que lo necesitarás en el siguiente paso.

<div id="step-2-create-a-package-for-your-application">
  ## Paso 2. Crea un paquete para tu aplicación
</div>

Debes obtener un **<Tooltip tip="ID de cliente: Valor de identificación otorgado a tu recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=Client+ID">ID de cliente</Tooltip>** y un **<Tooltip tip="ID de cliente: Valor de identificación otorgado a tu recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=Client+Secret">secreto de cliente</Tooltip>** para tu aplicación. Hay muchas formas de registrar tu aplicación, según el uso que se le vaya a dar.

[Registrar complementos de SharePoint (Microsoft Docs)](https://docs.microsoft.com/en-us/sharepoint/dev/sp-add-ins/register-sharepoint-add-ins) explica las distintas formas de registrar tu aplicación en SharePoint. En este paso del tutorial se usará la forma más sencilla: el autorregistro en un tenant específico (el tuyo).

<div id="open-sharepoint-online">
  ### Abrir SharePoint Online
</div>

La URL del panel es `https://{your Office365 tenant}.sharepoint.com/_layouts/15/appregnew.aspx`

  ### Genera un Client_Id y un ClientSecret:
</div>

<div id="complete-the-information-in-the-form">
  ### Completa la información del formulario:
</div>

Como Auth0 se encuentra entre tu aplicación y la infraestructura de Office 365, debes usar esta URL para la aplicación:

**Dominio de la aplicación**:

export const codeExample1 = `{yourDomain}`;

<AuthCodeBlock children={codeExample1} language="http" />

**URI de redirección**:

export const codeExample2 = `https://{yourDomain}/login/callback?SP_APP_TOKEN&connection=CONNECTION&client_id={yourClientId}&redirect_uri={https://yourApp/callback}`;

<AuthCodeBlock children={codeExample2} language="http" />

* `connection` es simplemente el nombre que usarás en las conexiones de Auth0 (como &quot;sharepoint&quot;).
* `client_id` identifica tu aplicación en Auth0 (creada en el paso 1).
* `redirect_uri` es la ubicación en tu aplicación donde tus usuarios acabarán llegando una vez completadas todas las negociaciones. Si no la especificas, siempre será la URL de callback de la aplicación definida en Auth0 (podría ser localhost)

<div id="package-the-app-and-upload-to-sharepoint">
  ### Empaquete la aplicación y súbala a SharePoint:
</div>

Complete la información del manifiesto de la aplicación en Visual Studio:

<Frame>![SharePoint - Visual Studio - Manifiesto de la aplicación](/docs/images/cdy7uua7fh8z/36dscLHlLHZjJx1rLgrQ7N/8d64ec7a51f4e053627ecb5dfd3c7707/90SEc.png)</Frame>

Tenga en cuenta que el `Query string` será exactamente igual que el `Redirect URI` que completó anteriormente. Luego, haga clic con el botón derecho en el proyecto y seleccione `Publish`:

Cree un **perfil de publicación** (tendrá que introducir el mismo **Client Id** y **secreto de cliente** obtenidos en el panel de SharePoint).

Haga clic en **Package** y suba el archivo resultante a SharePoint.

<div id="step-3-create-the-connection-in-auth0">
  ## Paso 3. Cree la conexión en Auth0
</div>

El último paso de la integración es agregar una conexión de SharePoint en Auth0.

Necesitará:

* `Connection Name`. Es un nombre arbitrario. Debe coincidir con el que ingresó en el paso 2.
* `Client Id` &amp; `secreto de cliente`. También deben coincidir con lo que ingresó en el paso 2.
* `Test SharePoint Site Url`. Esta es la URL del sitio de SP que se usa para probar la conexión (por ejemplo, al pulsar el botón &#39;Try&#39; en el Dashboard). Nunca se usa en tiempo de ejecución, porque los usuarios siempre seguirán el enlace a su sitio desde SharePoint.

Los usuarios instalarán su aplicación desde Office Marketplace. Cuando hagan clic en el enlace, se les redirigirá a Auth0, que negociará el token de acceso por usted y, finalmente, los dirigirá a su aplicación. Su aplicación recibirá un `User Profile` similar a este:

<Frame>![Office Marketplace - Redirect -  User Profile Example](/docs/images/cdy7uua7fh8z/4piqwmx1ONWMd3kjTFMq4I/714545f5cda3ab1d1dd5c620bc96b083/8Xp6x.png)</Frame>

Se incluirán las siguientes propiedades: `cacheKey`, `refresh_token`, `host` y `site`. Estas le permitirán volver a llamar a las API de SharePoint (como las listas).
