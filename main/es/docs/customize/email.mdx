---
title: Personalizar correos electrónicos
description: Administra y personaliza los correos electrónicos enviados por Auth0, incluida la verificación de cuentas, los mensajes de bienvenida y la detección de contraseñas vulneradas.
---

Auth0 ofrece varios flujos de trabajo basados en correo electrónico, como la [verificación de cuentas de usuario](/es/docs/manage-users/user-accounts/verify-emails) y la [detección de contraseñas vulneradas](/es/docs/secure/attack-protection/breached-password-detection).

Para usar estos flujos de trabajo, empieza por configurar un [proveedor de correo electrónico SMTP externo](/es/docs/customize/email/smtp-email-providers) con credenciales de un proveedor compatible (como tu clave de API de Mandrill o las claves de acceso de Amazon SES) o con los datos de un servidor SMTP.

Para administrar y personalizar aún más los correos electrónicos, puedes:

* **Modificar la apariencia y el contenido** de los correos electrónicos usando [plantillas de correo electrónico](/es/docs/customize/email/email-templates) y Liquid, un lenguaje de plantillas de código abierto.

* **Controlar por completo el proceso de entrega de correos electrónicos** escribiendo una <Tooltip tip="Action: Una función de Node.js que se ejecuta en puntos definidos durante el runtime de Auth0." cta="Ver glosario" href="/es/docs/glossary?term=Actions">Action</Tooltip> que envíe solicitudes a la API de tu proveedor de correo electrónico e implemente lógica personalizada para tu caso de uso.

* **Activar correos electrónicos o crear tickets fuera de los flujos de trabajo predeterminados** mediante la Management API.

Durante el desarrollo, puedes usar nuestro [proveedor de correo electrónico integrado](/es/docs/customize/email/smtp-email-providers#use-the-built-in-smtp-email-provider-for-development) para comprobar la entrega de correos electrónicos y obtener una vista previa de cómo se muestran durante las pruebas.
