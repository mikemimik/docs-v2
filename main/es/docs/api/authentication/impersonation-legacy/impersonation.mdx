<div id="endpoint">
  ## Endpoint
</div>

`POST /users/{user_id}/impersonate`

Usa este endpoint para obtener una URL de suplantación e iniciar sesión como otro usuario. Resulta útil para troubleshooting.

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo puede usarse con credenciales de **Global Client**.
* Para distinguir entre inicios de sesión reales e inicios de sesión por suplantación, el perfil del usuario suplantado incluirá propiedades adicionales: `impersonated` e `impersonator`. Por ejemplo: `"impersonated": true, "impersonator": {"user_id": "auth0|...", "email": "admin@example.com"}`.
* En una aplicación web estándar, debes configurar `additionalParameters`: establece `response_type` en `code`, `callback_url` en la URL de callback a la que Auth0 redirigirá con el código de autorización, y `scope` en los claims del JWT que quieres incluir en el JWT.
