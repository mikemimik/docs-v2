<div id="endpoint">
  ## Endpoint
</div>

`GET /authorize`

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este endpoint está **obsoleto** para la vinculación de cuentas. En su lugar, debe usarse [POST /api/v2/users/{id}/identities](/es/api/management/v2/users/post-identities). Para obtener más información, consulta el [aviso de migración](/es/migrations/past-migrations#account-linking-removal).
</Callout>

Llama a este endpoint cuando un usuario quiera vincular un segundo método de autenticación (por ejemplo, una conexión de base de datos de usuario/contraseña con Facebook).

Este endpoint iniciará el flujo de inicio de sesión para vincular una cuenta existente con una nueva. Esto devolverá una redirección 302 a la `conexión` que el usuario actual quiere añadir. El usuario se identifica mediante el token de acceso que se devolvió al iniciar sesión correctamente.

<div id="remarks">
  ### Observaciones
</div>

* El valor de `redirect_uri` debe especificarse como una URL de retorno válida en la [configuración de la aplicación](https://manage.auth0.com/dashboard).

<div id="query-parameters">
  ## Parámetros de consulta
</div>

<ParamField query="response_type" type="string" required>
  Usa `code` para los flujos del lado del servidor y `token` para los flujos del lado del cliente.
</ParamField>

<ParamField query="client_id" type="string" required>
  El `client_id` de tu aplicación.
</ParamField>

<ParamField query="connection" type="string">
  El nombre de la conexión configurada para tu aplicación. Si es null, redirigirá a la [página de inicio de sesión de Auth0](https://auth0.com/#/login_page) y mostrará el widget de inicio de sesión con la primera conexión de base de datos.
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  La URL a la que Auth0 redirigirá el navegador después de que el usuario haya concedido la autorización.
</ParamField>

<ParamField query="access_token" type="string" required>
  El token de acceso del usuario que ha iniciado sesión.
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                  |
| ------ | -------------------------------------------- |
| 302    | Redirige a la conexión para vincular cuentas |
