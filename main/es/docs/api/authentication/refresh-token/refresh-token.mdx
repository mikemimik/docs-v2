<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/token`

Usa este endpoint para actualizar un token de acceso con el token de actualización que obtuviste durante la autorización.

<div id="learn-more">
  ## Más información
</div>

* [Tokens de actualización](https://auth0.com/docs/secure/tokens/refresh-tokens)

<div id="headers">
  ## Encabezados
</div>

<ParamField header="DPoP" type="string">
  Una prueba DPoP para la solicitud. Es opcional y solo se requiere si tu aplicación usa Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="grant_type" type="string" required>
  Especifica el flujo que se está utilizando. Para renovar un token de acceso, debe establecerse en `refresh_token`.

  Valores permitidos: `refresh_token`
</ParamField>

<ParamField body="client_id" type="string" required>
  El ID de cliente de tu aplicación, que identifica a la aplicación que realiza la solicitud.
</ParamField>

<ParamField body="client_secret" type="string">
  El secreto de cliente de tu aplicación. Es obligatorio cuando el método de autenticación del endpoint de token está configurado como `Post` o `Basic`.
</ParamField>

<ParamField body="refresh_token" type="string" required>
  El token de actualización emitido previamente al cliente. Este token se utiliza para obtener un nuevo token de acceso.
</ParamField>

<ParamField body="scope" type="string">
  Una lista de permisos de scope solicitados, delimitada por espacios. Si se omite, se usarán los scopes originales; de lo contrario, puedes solicitar un conjunto reducido de scopes. Ten en cuenta que debe estar codificado como URL.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    El token de acceso recién emitido.
  </ResponseField>

  <ResponseField name="refresh_token" type="string">
    El token de actualización.
  </ResponseField>

  <ResponseField name="id_token" type="string">
    El token de ID, si corresponde.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    El tipo de token. Normalmente, `Bearer`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La vigencia del token de acceso en segundos.
  </ResponseField>

  <ResponseField name="scope" type="string">
    Los scopes del token de acceso.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                                                             |
| ------ | ------------------------------------------------------------------------------------------------------- |
| 200    | Token de acceso renovado correctamente.                                                                 |
| 400    | Solicitud incorrecta. Faltan parámetros obligatorios en la solicitud o esta tiene un formato no válido. |
| 401    | No autorizado. Las credenciales del cliente no son válidas o el token de actualización no es válido.    |
| 403    | Prohibido. El token de actualización no es válido o ha expirado.                                        |
