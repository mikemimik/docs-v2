<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/revoke`

Usa este endpoint para invalidar un token de actualización si se ha visto comprometido.

El comportamiento de este endpoint depende del estado de la opción [Refresh Token Revocation Deletes Grant](https://auth0.com/docs/secure/tokens/refresh-tokens/revoke-refresh-tokens#choose-whether-token-revocation-deletes-grants).
Si esta opción está habilitada, cada solicitud de revocación invalida no solo el token específico, sino también todos los demás tokens basados en el mismo grant de autorización. Esto significa que **todos los tokens de actualización emitidos para el mismo usuario, aplicación y audiencia se revocarán**.
Si esta opción está deshabilitada, solo se revoca el token de actualización, mientras que el grant permanece intacto.

<div id="remarks">
  ## Observaciones
</div>

* En las aplicaciones no confidenciales que no pueden mantener seguro el Client Secret (por ejemplo, las aplicaciones nativas), el endpoint permite omitir el Client Secret, pero la propia aplicación debe tener la propiedad `tokenEndpointAuthMethod` establecida en `none`. Puedes hacerlo desde la UI ([Dashboard &gt; Aplicaciones &gt; Configuración de la aplicación](https://manage.auth0.com/dashboard)) o mediante la [Management API](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id).

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu aplicación.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT que contiene una aserción firmada con las credenciales de tu aplicación. Es obligatorio cuando Private Key JWT es el método de autenticación de la aplicación.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  El valor es `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Es obligatorio cuando Private Key JWT es el método de autenticación de la aplicación.
</ParamField>

<ParamField body="client_secret" type="string">
  El `client_secret` de tu aplicación. Es obligatorio cuando Client Secret Basic o Client Secret Post es el método de autenticación de la aplicación. En concreto, es obligatorio **solo** para las Regular Web Applications.
</ParamField>

<ParamField body="token" type="string" required>
  El token de actualización que quieres revocar.
</ParamField>
