<div id="endpoint">
  ## Endpoint
</div>

`POST /mfa/associate`

Asocia o agrega un nuevo autenticador para la autenticación multifactor (MFA).

Si el usuario tiene autenticadores activos, se requiere un token de acceso con el `scope` `enroll` y el `audience` establecido en `https://{yourDomain}/mfa/` para usar este endpoint.

Si el usuario no tiene autenticadores activos, puedes usar el `mfa_token` del error `mfa_required` en lugar de un token de acceso para esta solicitud.

Después de agregar un autenticador, es necesario verificarlo. Para verificar el autenticador, usa los valores de respuesta de la solicitud a `/mfa/associate` en lugar de los valores devueltos por el endpoint `/mfa/challenge` y continúa con el flujo de verificación.

La primera vez que se agrega un autenticador, la respuesta incluye un campo `recovery_codes`. Puedes usar `recovery_codes` para completar la autenticación multifactor, como se muestra arriba en [Verificar con código de recuperación](/multi-factor-authentication/verify-with-recovery-code).

Para acceder a este endpoint, debes incluir un token de acceso en el encabezado Authorization, con los siguientes claims:

* `scope`: `enroll`
* `audience`: `https://{yourDomain}/mfa/`

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="client_id" type="string" required>
  El ID de cliente de tu aplicación.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT que contiene una aserción firmada con las credenciales de tu aplicación. Es obligatorio cuando Private Key JWT es el método de autenticación de tu aplicación.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  El valor es `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Es obligatorio cuando Private Key JWT es el método de autenticación de la aplicación.
</ParamField>

<ParamField body="client_secret" type="string" required>
  El secreto de cliente de tu aplicación. Es obligatorio cuando el campo Token Endpoint Authentication Method de tu Configuración de la aplicación es `Post` o `Basic`.
</ParamField>

<ParamField body="authenticator_types" type="array" required>
  El valor es un arreglo con los valores &#39;otp&#39; u &#39;oob&#39;.
</ParamField>

<ParamField body="oob_channels" type="array">
  Es obligatorio si `authenticator_types` incluye `oob`.
</ParamField>

<ParamField body="phone_number" type="string">
  El número de teléfono que se usará para SMS o voz. Es obligatorio si `oob_channels` incluye `sms` o `voice`.
</ParamField>

<div id="response-schema">
  ## Esquema de respuesta
</div>

<ResponseSchema>
  <ResponseField name="oob_code" type="string">
    El código para la autenticación fuera de banda.
  </ResponseField>

  <ResponseField name="binding_method" type="string">
    Indica el método de asociación.
  </ResponseField>

  <ResponseField name="authenticator_type" type="string">
    El tipo de autenticador.
  </ResponseField>

  <ResponseField name="oob_channels" type="string">
    Los canales OOB que utiliza el autenticador.
  </ResponseField>

  <ResponseField name="recovery_codes" type="array">
    Una matriz de códigos de recuperación que Auth0 genera para el usuario. Cuando un usuario agrega un autenticador, los códigos de recuperación solo se muestran una vez.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                      |
| ------ | ---------------------------------------------------------------- |
| 200    | Respuesta satisfactoria al agregar un autenticador.              |
| 400    | Solicitud incorrecta debido a parámetros faltantes o no válidos. |
| 401    | No autorizado: token de acceso o mfa&#95;token no válidos.       |
