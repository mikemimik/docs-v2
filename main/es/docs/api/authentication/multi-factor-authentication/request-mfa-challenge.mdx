<div id="endpoint">
  ## Endpoint
</div>

`POST /mfa/challenge`

Los endpoints de la API de autenticación multifactor (MFA) te permiten exigir MFA cuando los usuarios interactúan con [los endpoints de token](#get-token), así como inscribir y administrar los autenticadores de los usuarios.

Primero, solicita un desafío en función de los tipos de desafío compatibles con la aplicación y el usuario. Si sabes que se admite una contraseña de un solo uso (OTP), puedes omitir la solicitud de desafío.

A continuación, verifica la autenticación multifactor mediante el endpoint `/oauth/token` y el tipo de desafío especificado: una contraseña de un solo uso (OTP), un código de recuperación o un desafío fuera de banda (OOB).

Para obtener más información, consulta:

* [Autenticación multifactor y Resource Owner Password](https://auth0.com/docs/secure/multi-factor-authentication/authenticate-using-ropg-flow-with-mfa)
* [API de autenticación multifactor](https://auth0.com/docs/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api)
* [Autenticación multifactor en Auth0](https://auth0.com/docs/secure/multi-factor-authentication)

<div id="challenge-request">
  ## Solicitud de desafío
</div>

Solicita un desafío para la autenticación multifactor (MFA) en función de los tipos de desafío que admiten la aplicación y el usuario.

El `challenge_type` indica cómo recibirá el usuario el desafío y cómo demostrará que posee el factor. Los tipos de desafío admitidos incluyen:

* `otp`: para contraseña de un solo uso (OTP)
* `oob`: para mensajes SMS/de voz o fuera de banda (OOB)

Si el usuario admite OTP y no quieres solicitar un factor distinto, puedes omitir la solicitud de desafío y [verificar la autenticación multifactor con una contraseña de un solo uso](#verify-with-one-time-password-otp-).

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint no admite la inscripción; el usuario debe estar inscrito con el método preferido antes de solicitar un desafío.
* Auth0 elige el tipo de desafío en función de los tipos compatibles con la aplicación y de aquellos en los que esté inscrito el usuario.
* Se devuelve un error `unsupported_challenge_type` si tu aplicación no admite ninguno de los tipos de desafío en los que está inscrito el usuario.
* Se devuelve un error `unsupported_challenge_type` si el usuario no está inscrito.
* Si el usuario no está inscrito, recibirás un error `association_required`, lo que indica que debe inscribirse para usar MFA. Consulta [Agregar un autenticador](/multi-factor-authentication/add-an-authenticator) a continuación para saber cómo proceder.

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="mfa_token" type="string" required>
  El token recibido del error `mfa_required`.
</ParamField>

<ParamField body="client_id" type="string" required>
  El ID de cliente de tu aplicación.
</ParamField>

<ParamField body="client_secret" type="string">
  El secreto de cliente de tu aplicación.
</ParamField>

<ParamField body="challenge_type" type="string">
  Una lista, separada por espacios en blanco, de los tipos de desafío que acepta tu aplicación.

  Valores permitidos: `oob`, `otp`
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT que contiene una aserción firmada con las credenciales de tu aplicación.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  El valor es `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`.
</ParamField>

<ParamField body="authenticator_id" type="string">
  El ID del autenticador que se va a desafiar.
</ParamField>

<div id="response-schema">
  ## Esquema de la respuesta
</div>

<ResponseSchema statusCode="200">
  <ResponseField name="challenge_type" type="string">
    El tipo de desafío que emite este autenticador. Valores posibles: `otp`, `oob`.
  </ResponseField>

  <ResponseField name="oob_code" type="string">
    El código del desafío fuera de banda. Solo está presente cuando `challenge_type` es `oob`.
  </ResponseField>

  <ResponseField name="binding_method" type="string">
    El método utilizado para la vinculación. Solo está presente cuando corresponde. Valores posibles: `prompt`.
  </ResponseField>
</ResponseSchema>

<ResponseSchema statusCode="400">
  <ResponseField name="error" type="string">
    Código de error.
  </ResponseField>

  <ResponseField name="error_description" type="string">
    Descripción detallada del error.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                                                |
| ------ | ------------------------------------------------------------------------------------------ |
| 200    | Solicitud de desafío procesada correctamente                                               |
| 400    | Solicitud no válida, por ejemplo, un tipo de desafío no compatible o falta de inscripción. |
