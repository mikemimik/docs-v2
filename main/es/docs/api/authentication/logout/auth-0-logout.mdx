---
title: "Cierre de sesión en Auth0"
description: "Cierra la sesión de un usuario y, de forma opcional, redirígelo a una URL especificada después."
---

<div id="endpoint">
  ## Endpoint
</div>

`GET /v2/logout`

Usa este endpoint para cerrar la sesión de un usuario. Si quieres redirigir al usuario a una URL específica después de cerrar la sesión, indica esa URL en el parámetro `returnTo`. La URL debe estar incluida en la lista correspondiente de `Allowed Logout URLs`:

* Si se incluye el parámetro `client_id`, la URL de `returnTo` debe figurar en el conjunto de `Allowed Logout URLs` a nivel de la aplicación. Para obtener más información, consulta [Cerrar la sesión de usuarios en aplicaciones](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-applications).
* Si NO se incluye el parámetro `client_id`, la URL de `returnTo` debe figurar en el conjunto de `Allowed Logout URLs` a nivel del tenant. Para obtener más información, consulta [Cerrar la sesión de usuarios en Auth0](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-auth0).
* Si se incluye el parámetro `client_id` y NO se establece la URL de `returnTo`, el servidor redirige al usuario a la primera `Allowed Logout URL` configurada en el Dashboard. Para obtener más información, consulta [Cerrar la sesión de usuarios en aplicaciones](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-applications).

<div id="remarks">
  ### Consideraciones
</div>

* Cerrar la sesión del usuario en su proveedor de identidad no es una práctica habitual, así que tenga en cuenta la experiencia de usuario antes de usar el parámetro de cadena de consulta `federated`.
* El script de ejemplo de auth0.js utiliza la versión 8 de la biblioteca. Si está usando la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js).

<div id="query-parameters">
  ## Parámetros de consulta
</div>

<ParamField query="client_id" type="string">
  El `client_id` de tu aplicación.
</ParamField>

<ParamField query="returnTo" type="string" required>
  URL a la que se redirigirá al usuario después de cerrar sesión.
</ParamField>

<ParamField query="federated" type="string">
  Añade este parámetro de consulta a la URL de cierre de sesión para cerrar también la sesión del usuario en su proveedor de identidad.
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Estado | Descripción                                                             |
| ------ | ----------------------------------------------------------------------- |
| 200    | Cierre de sesión correcto                                               |
| 400    | Solicitud incorrecta, posiblemente por falta de parámetros obligatorios |
