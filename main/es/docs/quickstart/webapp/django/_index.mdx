    Auth0 te permite agregar autenticación y acceder a la información del perfil de usuario en tu aplicación. Esta guía
    muestra cómo integrar Auth0 con una aplicación de Python [Django](https://www.djangoproject.com/) mediante el SDK [Authlib](https://authlib.org/).

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configura una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente alfanumérico único
      que el código de tu aplicación usará para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con esta Quickstart se actualizará automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), donde podrás administrar tus Applications más adelante.

      Si prefieres ver una configuración completa, puedes consultar una aplicación de ejemplo.

      ### Configura las URLs de devolución de llamada

      Una URL de devolución de llamada es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no la configuras, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como
        `http://localhost:3000/callback`.
      </Info>

      ### Configura las URLs de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no la configuras, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como `http://localhost:3000`.
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Para esta integración, deberás agregar varias dependencias de bibliotecas, como Authlib. Crea un
      archivo `requirements.txt` en el directorio de tu proyecto e incluye lo siguiente:

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      Ejecuta el siguiente comando en tu shell para que estas dependencias queden disponibles:

      ```text lines
      pip install -r requirements.txt
      ```

      detalles de configuración.

      ```text lines
      AUTH0_CLIENT_ID={yourClientId}
      AUTH0_CLIENT_SECRET={yourClientSecret}
      AUTH0_DOMAIN={yourDomain}
      ```

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      En este ejemplo, agregarás cuatro rutas a tu aplicación: `login`, `callback`, `logout` e `index`.

      * `login` - Cuando los visitantes de tu aplicación accedan a la ruta `/login`, llegarán a Auth0 para
        comenzar el flujo de autenticación.
      * `callback` - Después de que tus usuarios terminen de iniciar sesión con Auth0, volverán a tu aplicación
        por la ruta `/callback`. Esta ruta guarda la sesión del usuario y evita que tenga
        que volver a iniciar sesión cuando regrese.
      * `logout` - La ruta `/logout` cierra la sesión de los usuarios en tu aplicación. Esta ruta borra
        la sesión del usuario en tu aplicación y redirige al endpoint de logout de Auth0 para garantizar que la sesión ya no
        permanezca guardada. Después, la aplicación redirige al usuario a tu ruta de inicio.
      * `index` - La ruta de inicio renderizará los detalles de un usuario autenticado o permitirá que los visitantes inicien sesión.
