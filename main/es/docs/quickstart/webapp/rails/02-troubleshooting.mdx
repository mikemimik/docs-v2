A continuación, se presentan algunos temas de resolución de problemas que podrías encontrar al usar la guía Quickstart de Rails.

<div id="using-a-reverse-proxy">
  ## Uso de un proxy inverso
</div>

El parámetro `redirect_uri` que OmniAuth genera al redirigir al inicio de sesión se basa en el encabezado `Host` que se pasa a Rails. Esto puede hacer que se pasen URL de devolución de llamada incorrectas al usar esta estrategia (y OmniAuth en general) con un proxy inverso. Puedes ajustar el host que usa OmniAuth con el siguiente fragmento:

```
OmniAuth.config.full_host = lambda do |env|
    scheme         = env['rack.url_scheme']
    local_host     = env['HTTP_HOST']
    forwarded_host = env['HTTP_X_FORWARDED_HOST']
    forwarded_host.blank? ? "#{scheme}://#{local_host}" : "#{scheme}://#{forwarded_host}"
end
```

[Consulta este hilo de StackOverflow para más información](https://stackoverflow.com/a/7135029/728480).

<div id="actioncontrollerinvalidauthenticitytoken">
  ## ActionController::InvalidAuthenticityToken
</div>

Esto probablemente se deba a que falta un token CSRF necesario para enviar la solicitud de inicio de sesión mediante POST. Si inspeccionas el botón de inicio de sesión en tu navegador, deberías ver algo como esto:

```erb lines
<a data-method="post" href="auth/auth0">Login</a>
```

... y en el elemento `<head>` de la página, deberías tener etiquetas meta de CSRF como estas:

```erb lines
<meta name="csrf-param" content="authenticity_token">
<meta name="csrf-token" content="UY2XpKwxzwBWalxFVJ8yKsao/33it7If09BnZewpHifVPSpFJd2LrA7xgQn6VQrhZNGjgZoLI3kV+bkQHtr+Rw==">
```

Con esos elementos en su lugar, Rails hará que el enlace de inicio de sesión envíe por POST el token CSRF al backend para verificarlo antes de redirigir al inicio de sesión.

<div id="actiondispatchcookiescookieoverflow">
  ## ActionDispatch::Cookies::CookieOverflow
</div>

Este error significa que se está usando una sesión basada en cookies y, como se está almacenando el perfil completo, se supera el tamaño máximo de 4 kb. Si no puedes acceder al perfil del usuario o recibes un error similar a `NoMethodError`, `undefined method '[]' for nil:NilClass`, intenta usar el almacenamiento en memoria en desarrollo.

Ve a `/config/initializers/session_store.rb` (o créalo si no existe) y agrega lo siguiente:

```
Rails.application.config.session_store :cache_store
```

Ve a `/config/environments/development.rb` y añade lo siguiente:

```
config.cache_store = :memory_store
```

Reinicia tu servidor de Rails para que estos cambios surtan efecto.

Se recomienda usar un almacenamiento en memoria, como MemCached, para las aplicaciones en producción.

<div id="ssl-issues">
  ## Problemas con SSL
</div>

En algunas configuraciones, es posible que Ruby no pueda encontrar los certificados de la entidad de certificación (certificados CA).

Descarga el paquete de certificados CA en el directorio del proyecto:

```bash lines
curl -L -o lib/ca-bundle.crt http://curl.haxx.se/ca/ca-bundle.crt
```

Añade este archivo de inicialización a `config/initializers/fix_ssl.rb`:

```rb lines
# config/initializers/fix_ssl.rb

require 'open-uri'
require 'net/https'

module Net
  class HTTP
    alias_method :original_use_ssl=, :use_ssl=

    def use_ssl=(flag)
      path = ( Rails.env == "development") ? "lib/ca-bundle.crt" : "/usr/lib/ssl/certs/ca-certificates.crt"
      self.ca_file = Rails.root.join(path).to_s
      self.verify_mode = OpenSSL::SSL::VERIFY_PEER
      self.original_use_ssl = flag
    end
  end
end
```

<div id="failure-messageinvalid_credentials">
  ## &quot;failure message=invalid_credentials&quot;
</div>

Este problema no se produce al trabajar en local, pero puede ocurrir en un entorno de preproducción o producción. El mensaje de error puede mostrarse así:

```
omniauth: (auth0) Authentication failure! invalid_credentials: OAuth2::Error, server_error: The redirect URI is wrong. You send [wrong url], and we expected [callback url set in your app settings]
```

Para resolverlo, añade lo siguiente a `config/environments/staging.rb` o `production.rb`:

```
OmniAuth.config.full_host = "http://www.example.com"
```

Sustituye `http://www.example.com` por la URL real que vayas a usar en tu aplicación.
