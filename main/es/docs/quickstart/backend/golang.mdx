  <Step title="Crear custom claims y un validador de JWT" stepNumber={5}>
    Las claims personalizadas te permiten extraer y validar datos especГ­ficos de la aplicaciГіn de los JWT. El validador es el componente central que verifica los tokens frente a Auth0.

    <Tabs>
      <Tab title="claims.go">
        ```go internal/auth/claims.go lines
        package auth

        import (
            "context"
            "fmt"
            "strings"
        )

        // CustomClaims contiene datos personalizados que queremos extraer del JWT.
        type CustomClaims struct {
            Scope string `json:"scope"`
        }

        // Validate garantiza que las claims personalizadas tengan el formato correcto.
        func (c *CustomClaims) Validate(ctx context.Context) error {
            if c.Scope == "" {
                return nil
            }

            if strings.TrimSpace(c.Scope) != c.Scope {
                return fmt.Errorf("scope claim has invalid whitespace")
            }

            if strings.Contains(c.Scope, "  ") {
                return fmt.Errorf("scope claim contains double spaces")
            }

            return nil
        }

        // HasScope comprueba si nuestras claims tienen un scope especГ­fico.
        func (c *CustomClaims) HasScope(expectedScope string) bool {
            if c.Scope == "" {
                return false
            }

            scopes := strings.Split(c.Scope, " ")
            for _, scope := range scopes {
                if scope == expectedScope {
                    return true
                }
            }
            return false
        }
        ```
      </Tab>

      <Tab title="validator.go">
        ```go internal/auth/validator.go lines
        package auth

        import (
            "fmt"
            "net/url"
            "time"

            "github.com/auth0/go-jwt-middleware/v3/jwks"
            "github.com/auth0/go-jwt-middleware/v3/validator"
        )

        func NewValidator(domain, audience string) (*validator.Validator, error) {
            // Construye la URL del issuer (debe incluir la barra diagonal final)
            issuerURL, err := url.Parse("https://" + domain + "/")
            if err != nil {
                return nil, fmt.Errorf("failed to parse issuer URL: %w", err)
            }

            // Inicializa el proveedor JWKS con el patrГіn de opciones de v3
            provider, err := jwks.NewCachingProvider(
                jwks.WithIssuerURL(issuerURL),
                jwks.WithCacheTTL(5*time.Minute),
            )
            if err != nil {
                return nil, fmt.Errorf("failed to create JWKS provider: %w", err)
            }

            // Crea el validador con el patrГіn de opciones de v3
            jwtValidator, err := validator.New(
                validator.WithKeyFunc(provider.KeyFunc),
                validator.WithAlgorithm(validator.RS256),
                validator.WithIssuer(issuerURL.String()),
                validator.WithAudience(audience),
                validator.WithCustomClaims(func() validator.CustomClaims {
                    return &CustomClaims{}
                }),
                validator.WithAllowedClockSkew(30*time.Second),
            )
            if err != nil {
                return nil, fmt.Errorf("failed to create validator: %w", err)
            }

            return jwtValidator, nil
        }
        ```
      </Tab>
    </Tabs>

    **Puntos clave:**

    * El mГ©todo `Validate` middleware llama automГЎticamente al mГ©todo `Validate` despuГ©s de procesar el JWT
    * `HasScope` procesa scopes separados por espacios para el control de acceso basado en permisos
    * El validador usa almacenamiento en cachГ© de JWKS (TTL de 5 min) y permite una desviaciГіn del reloj de 30 s
    * El algoritmo RS256 se configura explГ­citamente para evitar [ataques de confusiГіn de algoritmos](https://auth0.com/blog/critical-vulnerabilities-in-json-web-token-libraries/)
  </Step>

  <Step title="Crear middleware y controladores HTTP" stepNumber={6}>
    El middleware encapsula el validador de solicitudes HTTP. Los controladores muestran tres niveles de protecciГіn: pГєblico, privado y con permisos.

    <Tabs>
      <Tab title="middleware.go">
        ```go internal/auth/middleware.go lines
        package auth

        import (
            "log/slog"
            "net/http"

            jwtmiddleware "github.com/auth0/go-jwt-middleware/v3"
            "github.com/auth0/go-jwt-middleware/v3/validator"
        )

        func NewMiddleware(jwtValidator *validator.Validator) (*jwtmiddleware.JWTMiddleware, error) {
            return jwtmiddleware.New(
                jwtmiddleware.WithValidator(jwtValidator),
                jwtmiddleware.WithValidateOnOptions(false),
                jwtmiddleware.WithErrorHandler(func(w http.ResponseWriter, r *http.Request, err error) {
                    slog.Error("JWT validation failed", "error", err, "path", r.URL.Path)
                    w.Header().Set("Content-Type", "application/json")
                    w.WriteHeader(http.StatusUnauthorized)
                    w.Write([]byte(`{"message":"Failed to validate JWT."}`))
                }),
            )
        }
        ```
      </Tab>

      <Tab title="api.go">
        ```go internal/handlers/api.go expandable lines
        package handlers

        import (
            "encoding/json"
            "net/http"

            "github.com/yourorg/myapi/internal/auth"
            jwtmiddleware "github.com/auth0/go-jwt-middleware/v3"
            "github.com/auth0/go-jwt-middleware/v3/validator"
        )

        // PublicHandler - no requiere autenticaciГіn
        func PublicHandler(w http.ResponseWriter, r *http.Request) {
            response := map[string]string{
                "message": "Hello from a public endpoint! You don't need to be authenticated to see this.",
            }
            w.Header().Set("Content-Type", "application/json")
            json.NewEncoder(w).Encode(response)
        }

        // PrivateHandler - requiere un JWT vГЎlido
        func PrivateHandler(w http.ResponseWriter, r *http.Request) {
            response := map[string]string{
                "message": "Hello from a private endpoint! You need to be authenticated to see this.",
            }
            w.Header().Set("Content-Type", "application/json")
            json.NewEncoder(w).Encode(response)
        }

        // ScopedHandler - requiere el permiso 'read:messages'
        func ScopedHandler(w http.ResponseWriter, r *http.Request) {
            claims, err := jwtmiddleware.GetClaims[*validator.ValidatedClaims](r.Context())
            if err != nil {
                w.Header().Set("Content-Type", "application/json")
                w.WriteHeader(http.StatusUnauthorized)
                w.Write([]byte(`{"message":"Unauthorized."}`))
                return
            }

            customClaims, ok := claims.CustomClaims.(*auth.CustomClaims)
            if !ok || !customClaims.HasScope("read:messages") {
                w.Header().Set("Content-Type", "application/json")
                w.WriteHeader(http.StatusForbidden)
                w.Write([]byte(`{"message":"Insufficient scope."}`))
                return
            }

            response := map[string]string{
                "message": "Hello from a private endpoint! You need to be authenticated and have a scope of read:messages to see this.",
            }
            w.Header().Set("Content-Type", "application/json")
            json.NewEncoder(w).Encode(response)
        }
        ```
      </Tab>
    </Tabs>

    **Niveles de protecciГіn:**

    * **PГєblico** (`/api/public`) вЂ” No requiere autenticaciГіn
    * **Privado** (`/api/private`) вЂ” Requiere un JWT vГЎlido
    * **Con permisos** (`/api/private-scoped`) вЂ” Requiere un JWT vГЎlido + el permiso `read:messages`

  <Step title="Crea el servidor principal" stepNumber={7}>
    Integra todo en el punto de entrada principal con tiempos de espera preparados para producciГіn y un cierre ordenado:

    ```go cmd/server/main.go expandable lines
    package main

    import (
        "context"
        "log"
        "net/http"
        "os"
        "os/signal"
        "time"

        "github.com/yourorg/myapi/internal/auth"
        "github.com/yourorg/myapi/internal/config"
        "github.com/yourorg/myapi/internal/handlers"
        "github.com/joho/godotenv"
    )

    func main() {
        // Cargar variables de entorno desde el archivo .env
        if err := godotenv.Load(); err != nil {
            log.Println("No .env file found, using environment variables")
        }

        // Cargar configuraciГіn de Auth0
        cfg, err := config.LoadAuthConfig()
        if err != nil {
            log.Fatalf("Failed to load config: %v", err)
        }

        // Crear validador JWT
        jwtValidator, err := auth.NewValidator(cfg.Domain, cfg.Audience)
        if err != nil {
            log.Fatalf("Failed to create validator: %v", err)
        }

        // Crear middleware HTTP
        middleware, err := auth.NewMiddleware(jwtValidator)
        if err != nil {
            log.Fatalf("Failed to create middleware: %v", err)
        }

        // Configurar rutas
        mux := http.NewServeMux()
        mux.HandleFunc("/api/public", handlers.PublicHandler)
        mux.Handle("/api/private", middleware.CheckJWT(http.HandlerFunc(handlers.PrivateHandler)))
        mux.Handle("/api/private-scoped", middleware.CheckJWT(http.HandlerFunc(handlers.ScopedHandler)))

        // Configurar el servidor con tiempos de espera para producciГіn
        srv := &http.Server{
            Addr:         ":8080",
            Handler:      mux,
            ReadTimeout:  15 * time.Second,
            WriteTimeout: 15 * time.Second,
            IdleTimeout:  60 * time.Second,
        }

        // Iniciar el servidor en una goroutine
        go func() {
            log.Println("Server starting on :8080")
            if err := srv.ListenAndServe(); err != nil && err != http.ErrServerClosed {
                log.Fatalf("Server failed: %v", err)
            }
        }()

        // Apagado graceful
        quit := make(chan os.Signal, 1)
        signal.Notify(quit, os.Interrupt)
        <-quit

        log.Println("Shutting down server...")
        ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
        defer cancel()

        if err := srv.Shutdown(ctx); err != nil {
            log.Fatalf("Server forced to shutdown: %v", err)
        }

        log.Println("Server exited")
    }
    ```

<Accordion title="Ejemplos de cГіdigo del cliente" defaultOpen>
  <CodeGroup>
    ```bash cURL
    curl --request GET \
      --url http://localhost:8080/api/private \
      --header 'authorization: Bearer YOUR_ACCESS_TOKEN'
    ```

    ```go Go
    package main

    import (
        "fmt"
        "net/http"
        "io"
    )

    func main() {
        url := "http://localhost:8080/api/private"

        req, _ := http.NewRequest("GET", url, nil)
        req.Header.Add("authorization", "Bearer YOUR_ACCESS_TOKEN")

        res, _ := http.DefaultClient.Do(req)
        defer res.Body.Close()

        body, _ := io.ReadAll(res.Body)

        fmt.Println(res)
        fmt.Println(string(body))
    }
    ```

    ```javascript Node.js
    const axios = require("axios").default;

    const options = {
      method: 'GET',
      url: 'http://localhost:8080/api/private',

    axios.request(options).then(function (response) {

    }).catch(function (error) {
      console.error(error);
    });
    ```

    ```python Python

    conn = http.client.HTTPConnection("localhost", 8080)
    ```

<Accordion title="Seguridad de DPoP (Proof-of-Possession)">
  **DPoP (Demonstrating Proof-of-Possession)**, segГєn la RFC 9449, proporciona mayor seguridad al evitar el robo de tokens mediante la vinculaciГіn criptogrГЎfica a una clave.

  ```go internal/auth/middleware.go

      return jwtmiddleware.New(
          jwtmiddleware.WithValidator(jwtValidator),
          jwtmiddleware.WithDPoPMode(jwtmiddleware.DPoPRequired),
          jwtmiddleware.WithLogger(slog.Default()),
      )
  }
  ```

  **Modos de DPoP:**

  * `DPoPAllowed` (predeterminado) вЂ” Acepta tanto tokens Bearer como DPoP
  * `DPoPRequired` вЂ” Solo acepta tokens DPoP; rechaza Bearer
  * `DPoPDisabled` вЂ” Solo acepta tokens Bearer; rechaza DPoP

  <Note>
    Se recomienda DPoP para API financieras, API del sector salud y aplicaciones empresariales de alta seguridad. ObtГ©n mГЎs informaciГіn en la [documentaciГіn de DPoP](https://github.com/auth0/go-jwt-middleware#dpop-support).
  </Note>
</Accordion>

<Accordion title="ConfiguraciГіn de CORS">

<div id="troubleshooting">
  ## SoluciГіn de problemas
</div>

<AccordionGroup>
  <Accordion title="Problemas comunes y soluciones">
    ### &quot;Failed to validate JWT&quot; o 401 No autorizado

    **Problema:** La API no puede encontrar o validar el token de acceso.

    **Soluciones:**

    1. AsegГєrate de que el encabezado `Authorization` estГ© presente: `Authorization: Bearer YOUR_TOKEN`
    2. Comprueba que &quot;Bearer&quot; aparezca antes del token
    3. Verifica que el token no haya expirado
    4. AsegГєrate de que estГЎs usando un **token de acceso**, no un token de identidad

    ### &quot;aud claim mismatch&quot;

    **Problema:** La audiencia del token no coincide con tu API.

    **SoluciГіn:** Verifica que `AUTH0_AUDIENCE` coincida exactamente con el identificador de tu API en el Auth0 Dashboard. La audiencia NO debe tener una barra diagonal al final:

    ```bash
    # Correcto
    AUTH0_AUDIENCE=https://my-go-api.example.com

    # Incorrecto (con barra diagonal final)
    AUTH0_AUDIENCE=https://my-go-api.example.com/
    ```

    La aplicaciГіn cliente tambiГ©n debe solicitar un token con el parГЎmetro de audiencia correcto.

    ### &quot;unexpected signing method&quot;

    **Problema:** El algoritmo del token no coincide con la configuraciГіn del validador.

    **Soluciones:**

    1. Auth0 usa RS256 de forma predeterminada (asimГ©trico)
    2. AsegГєrate de que tu validador especifique `validator.RS256`
    3. Nunca uses `validator.HS256` para tokens de Auth0, salvo que se haya configurado especГ­ficamente

    ### Endpoint JWKS inaccesible

    **Problema:** El proveedor de cachГ© JWKS no puede alcanzar el endpoint de clave pГєblica de Auth0.

    **Soluciones:**

    1. Comprueba la conectividad de red con Auth0 (configuraciГіn de firewall/proxy)
    2. Prueba el endpoint JWKS manualmente: `curl https://YOUR_AUTH0_DOMAIN/.well-known/jwks.json`
    3. Verifica que la regiГіn de Auth0 sea la correcta (us/eu/au)

    ### Ruta de importaciГіn incorrecta

    **Problema:** `cannot find package "github.com/auth0/go-jwt-middleware/v3/..."`

    **SoluciГіn:** AsegГєrate de que todas las importaciones usen el sufijo `/v3`:

    ```go
    // Correcto
    import "github.com/auth0/go-jwt-middleware/v3/validator"

    // Incorrecto
    import "github.com/auth0/go-jwt-middleware/validator"
    ```

    ### Errores de desfase del reloj / token expirado

    **Problema:** El reloj del servidor estГЎ desincronizado, lo que hace que los tokens vГЎlidos parezcan expirados.

    **SoluciГіn:** El validador ya incluye una tolerancia de 30 s para el desfase del reloj. Si necesitas mГЎs, ajГєstala asГ­:

    ```go
    validator.WithAllowedClockSkew(60*time.Second)
    ```

    ### Error al extraer claims

    **Problema:** `Failed to retrieve claims` al usar genГ©ricos.

    **SoluciГіn:** AsegГєrate de estar usando el parГЎmetro de tipo correcto:

    ```go
    claims, err := jwtmiddleware.GetClaims[*validator.ValidatedClaims](r.Context())
    ```
  </Accordion>
</AccordionGroup>
