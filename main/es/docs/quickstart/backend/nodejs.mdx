<div id="get-started">
  ## Primeros pasos
</div>

En este quickstart se muestra cómo proteger endpoints de API de Express.js mediante tokens de acceso JWT. Crearás una API segura que valida los tokens de acceso de Auth0, protege rutas e implementa autorización basada en scopes.

<Steps>
  <Step title="Crear un proyecto nuevo" stepNumber={1}>
    Crea un nuevo directorio para tu API de Express e inicializa un proyecto de Node.js.

    ```shellscript
    mkdir auth0-express-api && cd auth0-express-api
    ```

    Inicializar el proyecto

    ```shellscript
    npm init -y
    ```

    Crear la estructura del proyecto

    ```shellscript
    touch server.js .env
    ```
  </Step>

  <Step title="Instala el SDK de express-oauth2-jwt-bearer" stepNumber={2}>
    Instalar las dependencias necesarias

    ```shellscript
    npm install express express-oauth2-jwt-bearer dotenv
    ```

    Añade scripts de inicio a tu `package.json`:

    ```json package.json
    {
      "scripts": {
        "start": "node server.js",
        "dev": "node --watch server.js"
      }
    }
    ```
  </Step>

  <Step title="Configura la API de Auth0" stepNumber={3}>
    A continuación, debes crear una nueva API en tu tenant de Auth0 y agregar las variables de entorno a tu proyecto.

    Tienes dos opciones para configurar tu API en Auth0: usar un comando de la CLI o hacerlo manualmente desde el Dashboard:

    <Tabs>
      <Tab title="CLI">
        Ejecuta el siguiente comando en el directorio raíz de tu proyecto para crear una API en Auth0:

        <CodeGroup>
          ```shellscript Mac
          # Instalar Auth0 CLI (si aún no está instalado)
          brew tap auth0/auth0-cli && brew install auth0

          # Crear Auth0 API
          auth0 apis create \
            --name "My Express API" \
            --identifier https://my-express-api.example.com
          ```

          ```powershell Windows
          # Instalar Auth0 CLI (si aún no está instalado)
          scoop bucket add auth0 https://github.com/auth0/scoop-auth0-cli.git
          scoop install auth0

          # Crear Auth0 API
          auth0 apis create `
            --name "My Express API" `
            --identifier https://my-express-api.example.com
          ```
        </CodeGroup>

        <Note>
          Este comando hará lo siguiente:

          1. Comprobará si ya estás autenticado (y te pedirá que inicies sesión si hace falta)
          2. Creará una API en Auth0 con el identificador especificado
          3. Mostrará los detalles de la API, incluidos el dominio y el identificador
        </Note>

        Una vez creada, copia los valores de **Identifier** y **Domain**, y luego crea tu archivo `.env`:

        ```bash .env
        AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
        AUTH0_AUDIENCE=YOUR_API_IDENTIFIER
        ```

        <Note>
          Reemplaza `YOUR_AUTH0_DOMAIN` por el dominio de tu tenant de Auth0 (por ejemplo, `dev-abc123.us.auth0.com`) y `YOUR_API_IDENTIFIER` por el identificador de tu API (por ejemplo, `https://my-express-api.example.com`).
        </Note>
      </Tab>

      <Tab title="Dashboard">
        1. Ve al [Auth0 Dashboard](https://manage.auth0.com/dashboard/)
        2. Ve a **Applications** → **APIs** → **Create API**
        3. Introduce un nombre para tu API (por ejemplo, &quot;My Express API&quot;)
        4. Define el **Identifier** (por ejemplo, `https://my-express-api.example.com`)
           * Esta es la audience de tu API y debe tener un formato de URL válido
           * No tiene que ser una URL real; es solo un identificador
        5. Deja **Signing Algorithm** como **RS256**
        6. Haz clic en **Create**
        7. Copia el valor de **Identifier** de la pestaña **Settings**

        Crea tu archivo `.env` con los siguientes valores:

        ```bash .env
        AUTH0_DOMAIN=YOUR_AUTH0_DOMAIN
        AUTH0_AUDIENCE=YOUR_API_IDENTIFIER
        ```

        <Note>
          Reemplaza `YOUR_AUTH0_DOMAIN` por el dominio de tu tenant de Auth0 (por ejemplo, `dev-abc123.us.auth0.com`) y `YOUR_API_IDENTIFIER` por el identificador de tu API del Dashboard (por ejemplo, `https://my-express-api.example.com`).
        </Note>
      </Tab>
    </Tabs>

    <Tip>
      Verifica que tu archivo `.env` exista: `cat .env` (Mac/Linux) o `type .env` (Windows)
    </Tip>
  </Step>

  <Step title="Configura el middleware JWT" stepNumber={4}>
    Crea tu servidor Express y configura la validación de JWT:

    ```javascript server.js {1-3,6-7,10-13} lines
    require('dotenv').config();
    const express = require('express');
    const { auth } = require('express-oauth2-jwt-bearer');

    const app = express();
    const port = process.env.PORT || 3001;

    // Configurar el middleware de validación JWT
    const checkJwt = auth({
      issuerBaseURL: `https://${process.env.AUTH0_DOMAIN}`,
      audience: process.env.AUTH0_AUDIENCE,
    });

    // Iniciar el servidor
    app.listen(port, () => {
      console.log(`API server running at http://localhost:${port}`);
    });
    ```

    **Qué hace esto:**

    * Crea un middleware de validación de JWT con tu dominio de Auth0 y la audiencia de la API
    * Valida los claims `iss` y `aud` de los tokens de acceso recibidos
    * Pone `checkJwt` a disposición para proteger rutas individuales
  </Step>

  <Step title="Crear rutas API" stepNumber={5}>
    Añade rutas públicas y protegidas a tu `server.js`:

    ```javascript server.js expandable lines
    require('dotenv').config();
    const express = require('express');
    const { auth, requiredScopes } = require('express-oauth2-jwt-bearer');

    const app = express();
    const port = process.env.PORT || 3001;

    // Configurar el middleware de validación JWT
    const checkJwt = auth({
      issuerBaseURL: `https://${process.env.AUTH0_DOMAIN}`,
      audience: process.env.AUTH0_AUDIENCE,
    });

    // Ruta pública - no requiere autenticación
    app.get('/api/public', (req, res) => {
      res.json({
        message: 'Hello from a public endpoint! You don\'t need to be authenticated to see this.',
        timestamp: new Date().toISOString(),
      });
    });

    // Ruta protegida - requiere un access token válido
    app.get('/api/private', checkJwt, (req, res) => {
      res.json({
        message: 'Hello from a protected endpoint! You successfully authenticated.',
        user: req.auth.payload.sub,
        timestamp: new Date().toISOString(),
      });
    });

    // Ruta protegida con scope - requiere el scope 'read:messages'
    app.get('/api/private-scoped', checkJwt, requiredScopes('read:messages'), (req, res) => {
      res.json({
        message: 'Hello from a scoped endpoint! You have the required permission.',
        user: req.auth.payload.sub,
        scope: req.auth.payload.scope,
        timestamp: new Date().toISOString(),
      });
    });

    // Middleware de manejo de errores
    app.use((err, req, res, next) => {
      const status = err.status || 500;
      const message = err.message || 'Internal Server Error';

      res.status(status).json({
        error: err.code || 'server_error',
        message: status === 401 ? 'Authentication required' : message,
      });
    });

    // Iniciar el servidor
    app.listen(port, () => {
      console.log(`API server running at http://localhost:${port}`);
    });
    ```

    **Puntos clave:**

    * Las rutas públicas no requieren autenticación
    * Las rutas protegidas usan el middleware `checkJwt` para exigir un JWT válido
    * Las rutas con scopes usan `requiredScopes()` para exigir permisos específicos en el token
    * `req.auth.payload` contiene los claims del JWT decodificado para las solicitudes autenticadas
    * El claim `sub` contiene el identificador único del usuario
  </Step>

  <Step title="Pon en marcha tu API" stepNumber={6}>
    Inicia el servidor de desarrollo:

    ```shellscript
    npm run dev
    ```

    Tu API ya se está ejecutando en [http://localhost:3001](http://localhost:3001).

    <Info>
      El indicador `--watch` de Node.js 18+ reinicia automáticamente el servidor cuando cambian los archivos.
    </Info>
  </Step>

  <Step title="Prueba tu API" stepNumber={7}>
    Prueba el endpoint público (no requiere autenticación):

    ```bash
    curl http://localhost:3001/api/public
    ```

    Deberías ver:

    ```json
    {
      "message": "Hello from a public endpoint! You don't need to be authenticated to see this.",
      "timestamp": "2024-01-15T10:30:00.000Z"
    }
    ```

    Prueba el endpoint protegido sin token (debería fallar):

    ```bash
    curl http://localhost:3001/api/private
    ```

    Deberías ver un error 401 Unauthorized:

    ```json
    {
      "error": "unauthorized",
      "message": "Authentication required"
    }
    ```

    Para probar con un token válido:

    1. Ve a [Auth0 Dashboard](https://manage.auth0.com/) → **Applications** → **APIs**
    2. Selecciona tu API → pestaña **Test**
    3. Copia el token de acceso generado

    Prueba tu endpoint protegido:

    ```bash
    curl http://localhost:3001/api/private \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
    ```

    Deberías ver:

    ```json
    {
      "message": "Hello from a protected endpoint! You successfully authenticated.",
      "user": "auth0|abc123...",
      "timestamp": "2024-01-15T10:30:00.000Z"
    }
    ```
  </Step>
</Steps>

<Check>
  **Verificación**

  Ahora deberías tener una API protegida. Tu API:

  1. Acepta solicitudes a endpoints públicos sin autenticación
  2. Rechaza solicitudes a endpoints protegidos sin un token válido
  3. Valida los tokens JWT con tu dominio de Auth0 y audience
  4. Proporciona información del usuario a partir de los claims del token mediante `req.auth.payload`
</Check>

  ## Uso avanzado
</div>

<Accordion title="Autorización basada en scopes">
  Los scopes permiten un control de acceso detallado. Puedes requerir scopes específicos para distintos endpoints.

  **Configura scopes en Auth0:**

  1. En el [Auth0 Dashboard](https://manage.auth0.com/), ve a **Applications** → **APIs** → Tu API
  2. Ve a la pestaña **Permissions**
  3. Agrega permisos como `read:messages`, `write:messages`, `admin:access`

  **Protege rutas con scopes:**

  ```javascript server.js
  const { auth, requiredScopes } = require('express-oauth2-jwt-bearer');

  // Requiere el scope 'read:messages'
  app.get('/api/messages', checkJwt, requiredScopes('read:messages'), (req, res) => {
    res.json({
      messages: [
        { id: 1, text: 'Hello!' },
        { id: 2, text: 'World!' },
      ],
    });
  });

  // Requiere el scope 'admin:access'
  app.get('/api/admin', checkJwt, requiredScopes('admin:access'), (req, res) => {
    res.json({
      message: 'Admin access granted',
      userId: req.auth.payload.sub,
    });
  });
  ```

  <Note>
    Si una solicitud no incluye el scope requerido, la API devuelve `403 Forbidden` con un error `insufficient_scope`. Asegúrate de que la aplicación cliente solicite los scopes correctos al obtener un token de acceso.
  </Note>
</Accordion>

<Accordion title="Validación de claims personalizadas">
  Además de los scopes, puedes validar claims personalizadas en el payload de JWT:

  ```javascript server.js
  const { auth, claimEquals, claimIncludes, claimCheck } = require('express-oauth2-jwt-bearer');

  // Requiere un valor exacto del claim
  app.get('/api/org/:orgId',
    checkJwt,
    claimEquals('org_id', 'org_123'),
    (req, res) => {
      res.json({ message: 'Organization access granted' });
    }
  );

  // Requiere que el claim incluya todos los valores especificados
  app.get('/api/roles',
    checkJwt,
    claimIncludes('roles', 'editor', 'viewer'),
    (req, res) => {
      res.json({ message: 'Role check passed' });
    }
  );

  // Lógica personalizada de validación de claims
  app.get('/api/premium',
    checkJwt,
    claimCheck((claims) => {
      return claims.subscription === 'premium' && claims.verified === true;
    }),
    (req, res) => {
      res.json({ message: 'Premium feature access granted' });
    }
  );
  ```

  <Note>
    Las claims personalizadas deben usar URL con espacio de nombres (por ejemplo, `https://myapp.com/roles`), a menos que sean claims estándar de OIDC. [Más información sobre las claims personalizadas](https://auth0.com/docs/secure/tokens/json-web-tokens/create-custom-claims).
  </Note>
</Accordion>

<Accordion title="Autenticación opcional (rutas mixtas públicas/privadas)">
  Permite acceso autenticado y anónimo en la misma ruta:

  ```javascript server.js
  const optionalAuth = auth({
    issuerBaseURL: `https://${process.env.AUTH0_DOMAIN}`,
    audience: process.env.AUTH0_AUDIENCE,
    authRequired: false,
  });

  app.get('/api/feed', optionalAuth, (req, res) => {
    if (req.auth) {
      res.json({
        message: `Welcome back, ${req.auth.payload.sub}!`,
        personalizedContent: true,
      });
    } else {
      res.json({
        message: 'Welcome, guest!',
        personalizedContent: false,
      });
    }
  });
  ```
</Accordion>

<Accordion title="Configuración de CORS">
  Habilita CORS para permitir solicitudes desde aplicaciones web:

  ```bash
  npm install cors
  ```

  ```javascript server.js
  const cors = require('cors');

  app.use(cors({
    origin: ['http://localhost:3000', 'http://localhost:5173'],
    allowedHeaders: ['Authorization', 'Content-Type'],
    exposedHeaders: ['WWW-Authenticate'],
  }));
  ```

  Para producción, especifica los orígenes exactos:

  ```javascript server.js
  app.use(cors({
    origin: [
      'https://myapp.com',
      'https://www.myapp.com'
    ],
    credentials: true,
    methods: ['GET', 'POST', 'PUT', 'DELETE'],
  }));
  ```
</Accordion>

<Accordion title="Manejo de errores personalizado">
  Agrega un manejo de errores exhaustivo para los errores de autenticación:

  ```javascript server.js
  const { UnauthorizedError, InvalidTokenError, InsufficientScopeError } = require('express-oauth2-jwt-bearer');

  app.use((err, req, res, next) => {
    if (err instanceof InsufficientScopeError) {
      return res.status(403).json({
        error: 'forbidden',
        message: 'You do not have permission to access this resource',
        required_scopes: err.requiredScopes,
      });
    }

    if (err instanceof InvalidTokenError) {
      return res.status(401).json({
        error: 'invalid_token',
        message: 'The provided token is invalid or expired',
      });
    }

    if (err instanceof UnauthorizedError) {
      return res.status(401).set(err.headers).json({
        error: 'unauthorized',
        message: 'Authentication required',
      });
    }

    next(err);
  });
  ```
</Accordion>

<Accordion title="Compatibilidad con TypeScript">
  Para proyectos con TypeScript, instala las definiciones de tipos y configura tu proyecto:

  ```bash
  npm install -D typescript @types/express @types/node
  ```

  Crea `server.ts`:

  ```typescript server.ts
  import 'dotenv/config';
  import express, { Request, Response, NextFunction } from 'express';
  import { auth, requiredScopes, UnauthorizedError } from 'express-oauth2-jwt-bearer';

  const app = express();
  const port = process.env.PORT || 3001;

  const checkJwt = auth({
    issuerBaseURL: `https://${process.env.AUTH0_DOMAIN}`,
    audience: process.env.AUTH0_AUDIENCE,
  });

  app.get('/api/public', (req: Request, res: Response) => {
    res.json({ message: 'Public endpoint - no authentication required' });
  });

  app.get('/api/private', checkJwt, (req: Request, res: Response) => {
    res.json({
      message: 'Private endpoint',
      user: req.auth?.payload.sub,
    });
  });

  app.use((err: Error, req: Request, res: Response, next: NextFunction) => {
    if (err instanceof UnauthorizedError) {
      res.status(err.status).set(err.headers).json({
        error: err.code || 'unauthorized',
        message: 'Authentication required',
      });
    } else {
      res.status(500).json({
        error: 'server_error',
        message: 'Internal Server Error',
      });
    }
  });

  app.listen(port, () => {
    console.log(`API server running at http://localhost:${port}`);
  });
  ```

  Agrega un archivo `tsconfig.json`:

  ```json tsconfig.json
  {
    "compilerOptions": {
      "target": "ES2020",
      "module": "commonjs",
      "strict": true,
      "esModuleInterop": true,
      "skipLibCheck": true,
      "outDir": "./dist"
    },
    "include": ["*.ts"]
  }
  ```

  Ejecútalo con: `npx ts-node server.ts`
</Accordion>

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="Problemas comunes y soluciones">
    ### &quot;No se encontró ningún token de autorización&quot;

    **Problema:** La API no puede encontrar el token de acceso en la solicitud.

    **Soluciones:**

    1. Asegúrate de que el encabezado `Authorization` esté presente: `Authorization: Bearer YOUR_TOKEN`
    2. Comprueba que &quot;Bearer&quot; esté incluido antes del token
    3. Verifica que el token no haya expirado

    ### &quot;Invalid token&quot; o &quot;jwt malformed&quot;

    **Problema:** El formato del token no es válido.

    **Soluciones:**

    1. Asegúrate de que estás usando un **token de acceso**, no un token de identidad
    2. El token debe obtenerse con el parámetro `audience` de tu API
    3. Comprueba que el token sea un JWT válido (debe tener tres partes separadas por puntos)

    ### Valor inesperado de &quot;iss&quot; o &quot;aud&quot;

    **Problema:** El issuer o la audience del token no coinciden con tu configuración.

    **Soluciones:**

    1. Decodifica tu token en [jwt.io](https://jwt.io)
    2. Comprueba que el claim `iss` coincida con `https://YOUR_AUTH0_DOMAIN/` (ten en cuenta la barra diagonal final)
    3. Comprueba que el claim `aud` coincida exactamente con tu `AUTH0_AUDIENCE`
    4. Verifica los valores de tu `.env`:

    ```bash
    AUTH0_DOMAIN=dev-abc123.us.auth0.com
    AUTH0_AUDIENCE=https://my-express-api.example.com
    ```

    ### &quot;You must provide an issuerBaseURL&quot; o &quot;audience is required&quot;

    **Problema:** Las variables de entorno no se están cargando.

    **Soluciones:**

    1. Asegúrate de que el archivo `.env` exista en la raíz de tu proyecto
    2. Verifica que `dotenv` esté instalado: `npm install dotenv`
    3. Agrega `require('dotenv').config()` al principio de tu archivo del servidor
    4. Comprueba que los nombres de las variables coincidan exactamente (distinguen entre mayúsculas y minúsculas)

    ### 401 No autorizado en todas las solicitudes

    **Posibles causas:**

    * El token ha expirado
    * La audience no coincide
    * El issuer no coincide

    **Pasos de depuración:**

    1. Decodifica tu token en [jwt.io](https://jwt.io)
    2. Comprueba que el claim `exp` no haya vencido
    3. Verifica que el claim `aud` coincida exactamente con tu `AUTH0_AUDIENCE`
    4. Verifica que el claim `iss` sea `https://{AUTH0_DOMAIN}/`
    5. Asegúrate de que el formato del encabezado Authorization sea `Bearer YOUR_TOKEN` (con espacio)

    ### 403 Prohibido con &quot;insufficient_scope&quot;

    **Problema:** El token no tiene los scopes requeridos.

    **Soluciones:**

    1. Verifica que los scopes estén definidos en tu Auth0 API (Dashboard → **Applications** → **APIs** → **Permissions**)
    2. Solicita los scopes al obtener el token
    3. Comprueba que el claim `scope` del token incluya los scopes requeridos

    ### Errores de CORS en el navegador

    **Problema:** El navegador bloquea las solicitudes a la API debido a la política de CORS.

    **Solución:** Instala y configura `cors`:

    ```bash
    npm install cors
    ```

    ```javascript
    const cors = require('cors');

    app.use(cors({
      origin: 'http://localhost:3000',
    }));
    ```
  </Accordion>
</AccordionGroup>
