---
title: Cómo configurar rules de la ACL del tenant
sidebarTitle: Configurar rules
description: Define la señal, la condición, la acción, el scope y la prioridad de una rule para establecer su comportamiento. Usa la Management API o los SDK, CLI o el proveedor de Terraform de Auth0 para configurar y administrar la ACL del tenant.
---

Una rule de la ACL del tenant se compone de los siguientes elementos:

* **Señal**: La señal es un dato identificador que proporciona la solicitud entrante, como la dirección IP, la geolocalización o el agente de usuario.

* **Condición**: La condición es la combinación de un operador (como `match`) y un conjunto de valores (como una lista de direcciones IP).

* **Action**: La acción es la instrucción que ejecuta tu rule si se cumplen los criterios, como permitir, bloquear o redirigir.

* **Scope**: El scope indica a qué conjuntos de endpoints se aplica la rule, incluida la Authentication API, la Management API o todo tu tenant.

* **Prioridad**: La prioridad define el orden en que la rule se ejecuta en relación con otras rules.

Para obtener información detallada sobre estas propiedades de las rules de la ACL del tenant y cómo usarlas, consulta la [referencia de parámetros del endpoint de la Management API](./reference).

<Note>
  La ACL del tenant usa una [lógica coherente de evaluación de rules](./rule-evaluation) para determinar qué acción, si corresponde, debe aplicarse a una solicitud determinada.

  Asigna cuidadosamente las prioridades para crear políticas de control de acceso granulares para tu caso de uso.
</Note>

<div id="create-and-delete-rules">
  ## Crear y eliminar rules
</div>

Puedes crear y eliminar rules de la ACL del tenant mediante los siguientes endpoints de la Management API. Necesitas un [token de acceso de la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los scopes indicados.

| Action             | Endpoint                                                                                              | Scope requerido       |
| ------------------ | ----------------------------------------------------------------------------------------------------- | --------------------- |
| Crear una rule    | [Crear lista de control de acceso](/es/docs/api/management/v2/network-acls/post-network-acls)            | `create:network_acls` |
| Eliminar una rule | [Eliminar lista de control de acceso](/es/docs/api/management/v2/network-acls/delete-network-acls-by-id) | `delete:network_acls` |

<div id="modify-rules">
  ## Modificar rules
</div>

Puedes modificar las rules de la ACL del tenant mediante los siguientes endpoints de la Management API. Necesitas un [token de acceso de la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los scopes indicados.

| Action               | Endpoint                                                                                                                 | Scope requerido       |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------ | --------------------- |
| Actualizar una rule | [Actualización parcial de una lista de control de acceso](/es/docs/api/management/v2/network-acls/patch-network-acls-by-id) | `update:network_acls` |
| Reemplazar una rule | [Actualizar lista de control de acceso](/es/docs/api/management/v2/network-acls/put-network-acls-by-id)                     | `update:network_acls` |

<div id="enable-or-disable-rules">
  ### Habilitar o deshabilitar rules
</div>

<Tabs>
  <Tab title="Management API">
    Puedes habilitar o deshabilitar una rule con el endpoint [Actualizar lista de control de acceso](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) de Management API.

    Establece la propiedad `active` del objeto `rule` en `true` o `false`, respectivamente.
  </Tab>

  <Tab title="Dashboard">
    Para habilitar o deshabilitar una rule desde el Dashboard, ve a [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control).

    En la lista de rules existentes, puedes habilitar o deshabilitar rules específicas con el interruptor que aparece junto a ellas.

    ![Habilitar o deshabilitar una rule de ACL del tenant en el Auth0 Dashboard](/docs/images/cdy7uua7fh8z/29b92e102142d3cb20e69a40e292f6b7/Screenshot_2025-09-03_at_12.34.05.png)

    La columna **Status** muestra el estado de la rule (**Active** o **Disabled**).
  </Tab>
</Tabs>

<div id="view-rules">
  ## Ver rules
</div>

<Tabs>
  <Tab title="Management API">
    Puedes ver las rules existentes de la ACL del tenant mediante los siguientes endpoints de la Management API. Necesitas un [token de acceso de la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los scopes indicados.

    | Action               | Endpoint                                                                                                                                    | Scope requerido     |
    | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
    | Ver una rule        | [Obtener una entrada específica de la lista de control de acceso de un tenant](/es/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls` |
    | Ver todas las rules | [Obtener todas las entradas de la lista de control de acceso de un tenant](/es/docs/api/management/v2/network-acls/get-network-acls)           | `read:network_acls` |
  </Tab>

  <Tab title="Dashboard">
    Ve a [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) para ver una lista de las rules que ya tienes.

    | Nombre de columna    | Descripción                                                                                  | Campo relacionado en la respuesta de la API |
    | -------------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------- |
    | Prioridad            | Número ordinal que indica cuándo se evalúa la rule                                          | `priority`                                  |
    | Nombre               | Descripción del propósito o la funcionalidad de la rule                                     | `description`                               |
    | Action               | La [acción](/es/docs/secure/tenant-access-control-list/reference#actions) que realiza la rule. | `action`                                    |
    | Estado               | Valor que indica si la rule está activa o deshabilitada.                                    | `active`                                    |
    | Última actualización | Marca de fecha y hora de la última actualización de la rule                                 | `updated_at`                                |

    ![Ver la lista de rules de la ACL del tenant en el Auth0 Dashboard](/docs/images/cdy7uua7fh8z/4093f9a39aba48bdf8d579c134dc4da5/Screenshot_2025-09-03_at_12.33.47.png)
  </Tab>
</Tabs>
