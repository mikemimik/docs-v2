title: Factores de la autenticación multifactor
---

Auth0 admite distintas opciones o factores para proteger el acceso a las cuentas de usuario mediante <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA).

Para seleccionar los factores de MFA que quieres usar en tu tenant, ve a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Debes habilitar al menos un factor en tu tenant para poder usar MFA. Los factores disponibles dependen de tu plan de suscripción. Para obtener más información, consulta [Auth0 Pricing](https://auth0.com/pricing/).

<div id="factors">
  ## Factores
</div>

Auth0 es compatible con los siguientes factores para implementar MFA.

<div id="push-notifications">
  ### Notificaciones push
</div>

Envíe a los usuarios notificaciones push a sus dispositivos previamente registrados, normalmente un teléfono móvil o una tableta, desde donde pueden permitir o denegar de inmediato el acceso a la cuenta con solo pulsar un botón. El factor push se ofrece con la aplicación móvil **Auth0 Guardian**, disponible tanto para iOS como para Android. Para obtener más información, lea [Auth0 Guardian](/es/docs/secure/multi-factor-authentication/auth0-guardian).

Si no desea que sus clientes tengan que descargar una aplicación aparte, Auth0 también proporciona un **Guardian SDK** para crear un flujo de segundo factor en su aplicación móvil existente. Para obtener más información, lea [Install Guardian SDK](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/install-guardian-sdk).

<div id="sms-notifications">
  ### Notificaciones por SMS
</div>

Envía a los usuarios un código de un solo uso por SMS, que se les pedirá que introduzcan antes de poder completar la autenticación. Para obtener más información, consulta [Configurar notificaciones por SMS y voz para MFA.](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)

<div id="voice-notifications">
  ### Notificaciones de voz
</div>

Envía a los usuarios un código de un solo uso mediante una llamada de voz, que luego deberán introducir antes de completar la autenticación. Para obtener más información, consulta [Configurar notificaciones de SMS y voz para MFA.](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)

<div id="one-time-passwords">
  ### Contraseñas de un solo uso
</div>

Las contraseñas de un solo uso permiten usar una aplicación de autenticación en el dispositivo personal del usuario, como Google Authenticator, para generar una contraseña de un solo uso que va cambiando con el tiempo y puede introducirse como segundo factor para validar una cuenta. Para obtener más información, lea [Configurar notificaciones de OTP para MFA.](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)

<div id="webauthn-with-security-keys">
  ### WebAuthn con Security Keys
</div>

Permite a los usuarios realizar MFA con Security Keys compatibles con FIDO (p. ej., [Yubikey](https://www.yubico.com/), [Google Titan](https://cloud.google.com/titan-security-key)). Para obtener más información, consulta [Configurar WebAuthn con Security Keys para MFA.](/es/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)

<div id="webauthn-with-device-biometrics">
  ### WebAuthn con biometría del dispositivo
</div>

Permita a los usuarios usar MFA con autenticadores de plataforma como Touch Bar de MacBook, Windows Hello, Touch ID/Face ID de iOS y el reconocimiento facial o de huella digital de Android. Para obtener más información, lea [Configure WebAuthn with Device Biometrics for MFA.](/es/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)

<div id="email-notifications">
  ### Notificaciones por correo electrónico
</div>

Permita que los usuarios usen MFA mediante contraseñas de un solo uso enviadas por correo electrónico, si tienen configurados [factores de autenticación independientes](/es/docs/secure/multi-factor-authentication/enable-mfa#independent-factors). Para obtener más información, lea [Configurar notificaciones por correo electrónico para MFA.](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)

<div id="cisco-duo-security">
  ### Cisco Duo Security
</div>

Cisco Duo es un proveedor multifacético y solo puede utilizarse si es el único factor disponible para el usuario. Usa tu cuenta de Duo para administrar la MFA con Auth0. Para obtener más información, consulta [Configurar Cisco Duo Security para MFA.](/es/docs/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="recovery-codes">
  ### Códigos de recuperación
</div>

Los códigos de recuperación son códigos únicos que permiten a los usuarios recuperar el acceso a su cuenta si no pueden acceder al dispositivo o a la cuenta que usaron para inscribirse en MFA. Para obtener más información, consulta [Configurar códigos de recuperación para MFA.](/es/docs/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="policies">
  ## Políticas
</div>

Las políticas determinan cuándo se le pedirá a un usuario que complete pasos adicionales para demostrar que es el titular de una cuenta determinada. Use las políticas para definir su propio nivel de riesgo aceptable. Puede elegir entre **Nunca**, **Usar <Tooltip tip="Adaptive Multi-factor Authentication: autenticación multifactor (MFA) que solo se activa para los usuarios cuando se determina que un intento de login es de baja confianza." cta="Ver glosario" href="/es/docs/glossary?term=Adaptive+MFA">Adaptive MFA</Tooltip>**, y **Siempre**.

Puede lograr configuraciones de autenticación multifactor más precisas (como por aplicación, por usuario, etc.) mediante Auth0 Actions. Para obtener más información, lea [Auth0 Actions](/es/docs/customize/actions).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las Rules que afectan a MFA tienen prioridad sobre la configuración de políticas en el Dashboard.
</Callout>

Para obtener más información, lea [Defined Authentication Policies on OpenID.](http://openid.net/specs/openid-provider-authentication-policy-extension-1_0.html#rfc.section.4)

<div id="use-cases">
  ## Casos de uso
</div>

Hay distintas formas de gestionar MFA según tu entorno:

* B2B: Tus clientes gestionan los factores de MFA de sus usuarios.
* B2C: Los usuarios finales gestionan sus propios factores de MFA desde una página de **My MFA Settings**.
* B2E: Tú gestionas los factores de MFA de tus usuarios.

Para conocer los endpoints de la API que puedes usar para crear una interfaz de usuario para gestionar factores de MFA, consulta [Gestionar factores del autenticador con la API de MFA.](/es/docs/secure/multi-factor-authentication/manage-mfa-auth0-apis/manage-authenticator-factors-mfa-api)

Las aplicaciones que permiten acceder a distintos tipos de recursos pueden requerir que los usuarios se autentiquen con un mecanismo de autenticación más sólido para acceder a recursos sensibles. Consulta [Autenticación escalonada](/es/docs/secure/multi-factor-authentication/step-up-authentication) para obtener más información.

Para crear una Action, ve a [Dashboard &gt; Actions &gt; Flows](https://manage.auth0.com/#/actions/flows) y define las condiciones para activar desafíos de autenticación. Usa Actions para exigir MFA a los usuarios de determinadas aplicaciones, o a usuarios con metadatos de usuario concretos o rangos de IP específicos, entre otros triggers.

Añade MFA contextual, que te permite definir condiciones arbitrarias que activarán desafíos de autenticación adicionales para tus clientes con el fin de reforzar la seguridad; por ejemplo, la ubicación geográfica (geofencing), la dirección o el tipo de red utilizada (filtrado de IP), la hora del día, el día de la semana o un cambio en la ubicación o el dispositivo que se usa para iniciar sesión.
