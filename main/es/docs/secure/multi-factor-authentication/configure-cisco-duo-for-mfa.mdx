Cisco Duo es un proveedor de authentication con múltiples funciones y solo puede usarse en tu tenant de Auth0 si todos los demás factores están deshabilitados. Tu cuenta de Duo puede admitir notificaciones push, SMS, OTP, devolución de llamada telefónica y más, según tu configuración.

Tampoco puedes habilitar Duo si hay otros factores habilitados. Duo solo está disponible para los usuarios cuando es el único factor habilitado.

La aplicación le solicitará al usuario el segundo factor con Duo y mostrará las opciones que hayas habilitado en tu cuenta de Duo.

Tus usuarios pueden descargar Duo desde [Google Play](https://play.google.com/store/apps/details?id=com.duosecurity.duomobile) o la [App Store](https://itunes.apple.com/us/app/duo-mobile/id422663827?mt=8) para usarlo como segundo factor.

<div id="configure-duo">
  ## Configurar Duo
</div>

Para configurar Duo Security, debes proporcionar las credenciales de Duo a tu aplicación en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Ve a [Dashboard &gt; Seguridad &gt; Multi-factor Auth &gt; Duo Security](https://manage.auth0.com/#/security/mfa/duo) y actívalo.
2. Introduce la información en los campos para vincular tu cuenta de Duo con Auth0.
3. Selecciona **Save**.

<div id="use-actions-to-enable-duo">
  ## Usa Actions para habilitar Duo
</div>

Para habilitar Duo en una Action, pasa `duo` como parámetro `provider` al habilitar la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuarios que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>.

```js lines
exports.onExecutePostLogin = async (event, api) => {
  api.multifactor.enable('duo', { allowRememberBrowser: false });
};
```

Duo no ofrece una opción para el comportamiento de &quot;Remember Me&quot;. La sesión de MFA de 30 días está configurada de forma fija para recordar al usuario después del inicio de sesión inicial.

Para obligar a tus usuarios a iniciar sesión con Duo cada vez, crea una rule con `allowRememberBrowser: false`.

	// ejecutar solo para los clientes especificados
	if (CLIENTS_WITH_MFA.includes(event.client.client_id)) {

		// descomenta la siguiente condición if si quieres solicitar un segundo factor solo a los usuarios que tengan user_metadata.use_mfa === true

<div id="current-limitations">
  ## Limitaciones actuales
</div>

* No puedes usar [tickets de inscripción en MFA de Auth0](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/create-custom-enrollment-tickets) para registrar usuarios con Duo. Da de alta a esos usuarios directamente en Duo.
* Si usas <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip>, debes habilitar Duo en una Action con `provider` configurado como `duo`, como se describió anteriormente. Puedes usar de forma condicional Duo o el proveedor integrado de Auth0 para aplicaciones específicas.
