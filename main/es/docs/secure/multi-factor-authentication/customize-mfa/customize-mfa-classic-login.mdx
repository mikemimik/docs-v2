title: Personaliza MFA para Classic Login
---

Puedes habilitar la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación del usuario que utiliza un factor adicional al username y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> en la sección [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Puedes personalizar las páginas de MFA con opciones básicas de branding en el <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación del usuario que utiliza un factor adicional al username y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Si necesitas una personalización más avanzada, puedes proporcionar tu propio HTML para la página de MFA. Al usar tu propio HTML, Classic Login utiliza el Auth0 MFA Widget con las siguientes limitaciones:

* No admite MFA por correo electrónico.
* Si los usuarios tienen configurado más de un factor, no pueden seleccionar cuál usar. El widget de MFA les pedirá que inicien sesión con el factor más seguro.
* No usa las funciones de internacionalización de <Tooltip tip="Universal Login: tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip>. Para obtener más información, consulta [Universal Login Internationalization](/es/docs/customize/internationalization-and-localization/universal-login-internationalization).

<div id="customize-html-for-the-mfa-page">
  ## Personaliza el HTML de la página de MFA
</div>

Para personalizar la página de MFA, ve a la pestaña [Dashboard &gt; Branding &gt; Universal Login &gt; Multi-factor Authentication](https://manage.auth0.com/#/mfa_page) y activa el interruptor **Customize MFA Page**. Después, puedes usar el editor de texto para modificar tu HTML, aplicar estilos a tu página con CSS y cambiar el JavaScript que se utiliza para obtener variables personalizadas. Cuando termines de hacer los cambios, asegúrate de hacer clic en **Save Changes**.

Las páginas de MFA funcionan sin necesidad de personalización, y Auth0 actualizará los scripts incluidos según sea necesario. Sin embargo, una vez que activas la personalización, eres responsable de actualizar y mantener el script (incluida la actualización de números de versión, como el del MFA Widget), ya que Auth0 ya no podrá actualizarlo automáticamente.

Para obtener más información sobre cómo modificar el texto en muchas áreas del proceso de MFA para Classic Login, consulta [diccionario de idiomas del tema de MFA](/es/docs/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary). Para obtener más información sobre las opciones de tema del MFA Widget, consulta [opciones de tema del widget de MFA](/es/docs/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options).

<div id="reset-page">
  ## Restablecer la página
</div>

Si quieres revertir a un diseño anterior, tienes dos opciones:

* Revertir a la última plantilla guardada haciendo clic en **Restablecer a la última guardada**;
* Revertir a la plantilla predeterminada que proporciona Auth0 haciendo clic en **Restablecer a la predeterminada**.

<div id="render-enrollment-invitations">
  ## Renderizar invitaciones de inscripción
</div>

Hay dos escenarios posibles en los que se puede renderizar la página. Si se ha dirigido a un usuario a esta página específicamente para la inscripción (por ejemplo, desde un correo electrónico con un enlace de inscripción), la propiedad `ticket` estará disponible. De lo contrario, estará disponible la propiedad `requestToken`.

<div id="html-and-liquid-syntax">
  ## Sintaxis de HTML y Liquid
</div>

La página alojada usa la sintaxis de Liquid para las plantillas. Para obtener más información, consulta [Liquid for Designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) en GitHub. Los siguientes parámetros están disponibles para ayudarte a renderizar tu página:

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

La mayoría de los parámetros que se usan en el MFA Widget deben pasarse a Guardian, como se muestra en la plantilla predeterminada disponible en el área de personalización. Si necesitas un mayor nivel de personalización, puedes usar [auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example).

Para obtener más información, consulta [Customize MFA User Pages](/es/docs/secure/multi-factor-authentication/customize-mfa).
