Auth0 Guardian es una aplicación móvil para dispositivos iOS y Android que permite a los usuarios completar la [autenticación multifactor (MFA)](/es/docs/secure/multi-factor-authentication) mediante notificaciones push o contraseñas temporales de un solo uso.

Auth0 Guardian puede enviar notificaciones push a los dispositivos registrados de los usuarios (normalmente teléfonos móviles o tabletas) o generar contraseñas de un solo uso directamente en la aplicación. Luego, los usuarios pueden responder rápidamente a estas notificaciones push u obtener una contraseña de un solo uso para completar su inicio de sesión.

Los usuarios pueden descargar la aplicación Auth0 Guardian desde la [Apple App Store](https://apps.apple.com/us/app/auth0-guardian/id1093447833) o la [Google Play Store](https://play.google.com/store/apps/details?id=com.auth0.guardian). Como alternativa, puedes integrar las funcionalidades de Auth0 Guardian en tu propia aplicación personalizada con el Guardian SDK.

<div id="push-notifications">
  ## Notificaciones push
</div>

Para usar notificaciones push, los usuarios deben tener instalada en su dispositivo la aplicación Auth0 Guardian o una aplicación personalizada creada con el Guardian SDK. Cuando un usuario intenta autenticarse, se envían notificaciones push a la aplicación instalada. El usuario debe responder a la notificación para completar su inicio de sesión, lo que demuestra que conoce su información de inicio de sesión y que posee el dispositivo configurado para la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación del usuario que usa un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=MFA">MFA</Tooltip>.

Las notificaciones push de Auth0 pueden implementarse mediante AWS Simple Notification Service (SNS) o uno o varios de los siguientes servicios directos de cada proveedor para configurar integraciones específicas del proveedor:

* Firebase Cloud Messaging (FCM)

  * Para obtener más información sobre cómo modificar el contenido de las notificaciones push, consulta [la documentación de FCM](https://firebase.google.com/docs/cloud-messaging/concept-options).
* Apple Push Notification (APN)

  * Para obtener más información sobre cómo modificar el contenido de las notificaciones push, consulta [la documentación de Apple](https://developer.apple.com/documentation/usernotifications/modifying-content-in-newly-delivered-notifications).

<div id="enroll-in-push-notifications">
  ### Registrarse en las notificaciones push
</div>

Cuando usas Auth0 Guardian para configurar las notificaciones push, se les pide a los usuarios que descarguen la aplicación móvil la primera vez que se registran o inician sesión en tu aplicación. Si usas el Guardian SDK para implementar notificaciones push en una aplicación personalizada, no se les pide a los usuarios que descarguen ninguna aplicación durante la inscripción.

Puedes habilitar las notificaciones push desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> en [Security &gt; Multi-factor Auth &gt; Push Notification using Auth0 Guardian](https://manage.auth0.com/#/security/mfa/guardian).

<Frame>![Auth0 Dashboard > Security > Multi-factor Auth > Notificación push con Auth0 Guardian](/docs/images/cdy7uua7fh8z/fwjSKFNhsvObzdPDiA79N/65850980f88029cb23eef5eaef4f9b20/Auth0_Dashboard_-_Push_Notification_screen.png)</Frame>

Cuando un usuario se registra o inicia sesión por primera vez en tu aplicación, [Universal Login](/es/docs/authenticate/login/auth0-universal-login) muestra un código QR que puede usar para registrar la aplicación Auth0 Guardian o una aplicación personalizada con Guardian SDK como factor de autenticación secundario. El usuario dispone de poco tiempo para escanear este código con la aplicación indicada y completar la inscripción.

Una vez que el usuario se registra, puede usar las notificaciones push como factor de autenticación. Cada vez que intente iniciar sesión en tu aplicación, recibirá una notificación push en su dispositivo a través de Auth0 Guardian o de una aplicación personalizada con Guardian SDK. El usuario debe aprobar esta solicitud para iniciar sesión correctamente en tu aplicación.

Para saber cómo restablecer MFA para usuarios que han perdido sus dispositivos o códigos de recuperación, consulta [Reset Multi-Factor Authentication and Recovery Codes](/es/docs/secure/multi-factor-authentication/reset-user-mfa).

<div id="temporary-one-time-passwords">
  ## Contraseñas temporales de un solo uso
</div>

La aplicación Auth0 Guardian y el Guardian SDK también admiten el uso de contraseñas temporales de un solo uso (OTP) como factores secundarios de autenticación. Tanto la aplicación como el SDK pueden generar OTP temporales que los usuarios pueden usar para completar desafíos de MFA en <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip>.

De forma predeterminada, la aplicación Auth0 Guardian muestra las aplicaciones que el usuario tiene registradas. Después de seleccionar una aplicación, se muestra el OTP correspondiente. Cada OTP de 6 dígitos es válido durante 30 segundos. Los OTP que están a punto de caducar se muestran en rojo. Al caducar, la aplicación genera inmediatamente un nuevo OTP. Para evitar errores de autenticación, los usuarios no deben usar OTP en rojo

En las aplicaciones personalizadas con Guardian SDK, la apariencia de los OTP puede variar.

<div id="use-temporary-one-time-passwords">
  ### Usa contraseñas temporales de un solo uso
</div>

Puedes usar las OTP de dos maneras principales:

* Como opción de respaldo para las notificaciones push en caso de que la app Auth0 Guardian u otra aplicación personalizada no reciba una solicitud push.
* Como desafío de MFA, si habilitas el [factor One-Time Password](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors) en tu tenant.

<AccordionGroup>
  <Accordion title="Método de respaldo: usa las OTP como alternativa a las notificaciones push">
    Si un usuario no recibe una notificación push a través de Auth0 Guardian u otra aplicación personalizada, puede usar una contraseña temporal de un solo uso para completar su desafío de MFA.

    **Ejemplo de flujo del usuario**:

    1. Un usuario intenta iniciar sesión en tu aplicación. Llega al prompt de MFA de Universal Login, pero no recibe una notificación push.
    2. En el prompt de MFA, el usuario selecciona el botón **Manually Enter Code**. Debe seleccionar esta opción para autenticarse con OTP.

       <Frame>![Ejemplo de un prompt de MFA que muestra el botón Manually Enter Code](/docs/images/cdy7uua7fh8z/7reEOLI5CWxbpRzgZY7gxg/58da830927a33545d9a576974ec3ba6e/MFA_-_Push_-_English.png)</Frame>
    3. El usuario abre Auth0 Guardian o la aplicación personalizada de Guardian SDK y selecciona la aplicación a la que intenta acceder.
    4. El usuario copia la OTP que aparece en la parte inferior de la página Account Detail.
    5. El usuario introduce la OTP en el prompt de MFA para completar el desafío.
  </Accordion>

  <Accordion title="Desafío de MFA: usa las OTP como desafío de MFA">
    Para permitir que los usuarios se autentiquen con OTP, debes habilitar **One-Time Password** como [factor de MFA](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors) para tu tenant. Una vez habilitado, los usuarios pueden completar los prompts de MFA con OTP al iniciar sesión mediante Universal Login. Para habilitar los factores de MFA, ve a [Auth0 Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).

    Los usuarios pueden inscribirse en OTP como factor de MFA la primera vez que se registran o inician sesión en tu aplicación. Según el tipo de dispositivo que utilicen, [Universal Login](/es/docs/authenticate/login/auth0-universal-login) muestra una de las siguientes opciones:

    * **Para dispositivos no móviles**: Universal Login muestra un código QR que el usuario puede escanear para continuar con la inscripción.

    <Frame>![Ejemplo de un prompt de inscripción de OTP que muestra el enlace Trouble Scanning?](/docs/images/cdy7uua7fh8z/3kbyOBiLOpJiY8vI0kUqwg/b10f58a3b3ace74de613e28517726069/OTP_Challenge_3_-_English.png)</Frame>

    * **Para dispositivos móviles**: Universal Login omite el código QR y, en su lugar, muestra directamente el código de inscripción.

    <Frame>![Pantalla de inscripción de contraseña temporal de un solo uso tal como se muestra en un dispositivo móvil. ](/docs/images/cdy7uua7fh8z/5QvQ9FLYnVZXINFAeLkIr9/04df9be004169ac06b8d9fee456dc599/OTP_prompt_-_English.png)</Frame>

    Con cualquiera de estos métodos, los usuarios pueden seguir los pasos que aparecen en pantalla para registrar la app Auth0 Guardian o una aplicación personalizada de Guardian SDK como factor de autenticación secundario. Normalmente, este proceso implica que el usuario agregue tu aplicación a la app correspondiente y obtenga la OTP generada para esa aplicación. Luego, introduce la OTP en el prompt de Universal Login para finalizar el proceso de inscripción.

    La próxima vez que el usuario intente iniciar sesión en tu aplicación después de completar la inscripción, podrá autenticarse introduciendo una OTP desde su app Auth0 Guardian o su aplicación personalizada de Guardian SDK en el desafío.
  </Accordion>
</AccordionGroup>

<div id="user-security-settings">
  ### Configuración de seguridad del usuario
</div>

En la app Auth0 Guardian, los usuarios pueden habilitar códigos de acceso y biometría como capas de seguridad de la aplicación en iOS y Android. Si un usuario habilita una o más de estas opciones, deberá completar estas verificaciones antes de poder responder a las notificaciones push u obtener contraseñas de un solo uso.

Para habilitar estas opciones en dispositivos iOS o Android, los usuarios pueden seguir los pasos que se indican a continuación.

<Tabs>
  <Tab title="iOS">
    Para habilitar la configuración de seguridad de la app Auth0 Guardian en iOS:

    1. En la app, selecciona el **icono de engranaje** para abrir el menú de Configuración.
    2. Selecciona **Código de acceso** para habilitar la protección con código de acceso.
    3. Establece y confirma un código de acceso de 6 dígitos.

    La app ahora está protegida con el código de acceso. El usuario debe introducir este código de acceso antes de poder responder a las notificaciones push u obtener OTP.

    Después de habilitar la protección con código de acceso, el usuario puede configurar la biometría del dispositivo como una medida de seguridad alternativa. Para habilitar la biometría del dispositivo:

    1. En la app, selecciona el **icono de engranaje** para abrir el menú de Configuración.
    2. Habilita la biometría del dispositivo seleccionando una de las opciones disponibles, como **Face ID** o **Touch ID**.
  </Tab>

  <Tab title="Android">
    Para habilitar la configuración de seguridad de la app Auth0 Guardian en Android:

    1. En la app, selecciona el **menú de tres puntos** para abrir el menú de Configuración.
    2. Activa el interruptor de la opción **Always Ask for Passphrase**.
    3. Introduce la frase de acceso que configuraste al configurar la app Auth0 Guardian.

    La app ahora está protegida con la frase de acceso de Auth0 Guardian. El usuario debe introducir esta frase de acceso antes de poder responder a las notificaciones push u obtener OTP.

    Después de habilitar la protección con frase de acceso, el usuario puede configurar la biometría del dispositivo como una medida de seguridad alternativa. Para habilitar la biometría del dispositivo:

    1. En la app, selecciona el **menú de tres puntos** para abrir el menú de Configuración.
    2. Activa el interruptor de la opción **Use Biometrics to unlock**.
  </Tab>
</Tabs>

En la sección Language del menú Settings, los usuarios pueden seleccionar su idioma preferido. De forma predeterminada, la aplicación usa el mismo idioma que el sistema del dispositivo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

<div id="guardian-app-themes">
  ### Temas de la app Guardian
</div>

La app Auth0 Guardian para iOS y Android admite temas claros y oscuros.

En la sección Theme del menú Settings, los usuarios pueden seleccionar las siguientes opciones:

* **System**: Usa el tema predeterminado del sistema
* **Light**: Activa el tema claro
* **Dark**: Activa el tema oscuro

<div id="customization-options-for-protected-applications">
  ### Opciones de personalización para aplicaciones protegidas
</div>

Para que a los usuarios les resulte más fácil distinguir sus aplicaciones protegidas en la aplicación Auth0 Guardian, cada aplicación puede personalizarse con un nombre, un color de acento y un icono diferentes.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El nombre predeterminado asignado a tu aplicación en la aplicación Auth0 Guardian se determina según tu [nombre del tenant](/es/docs/get-started/tenant-settings/find-your-tenant-name-or-tenant-id) o el Friendly Name configurado en la [configuración del tenant](/es/docs/get-started/tenant-settings). Del mismo modo, el logotipo predeterminado se obtiene de la Logo URL proporcionada en la configuración de tu tenant.

  Para editar la configuración de tu tenant, ve a [Auth0 Dashboard &gt; Settings &gt; General](https://manage.auth0.com/dashboard/#/tenant/general).
</Callout>

Para personalizar una aplicación protegida:

1. Selecciona una aplicación de la lista.

   * **Consejo**: También puedes deslizar el dedo hacia la izquierda sobre la aplicación para mostrar las opciones Edit y Delete.
2. En la página Account Detail, elige **Edit**.
3. Actualiza el nombre, el color y el icono de la aplicación según sea necesario.
4. Selecciona **Save**.

<div id="guardian-sdks">
  ## SDKs de Guardian
</div>

Puedes [instalar el SDK de Guardian](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/install-guardian-sdk) (disponible para [iOS](/es/docs/secure/multi-factor-authentication/auth0-guardian/guardian-for-ios-sdk) y [Android](/es/docs/secure/multi-factor-authentication/auth0-guardian/guardian-for-android-sdk)) para crear tu propia aplicación de autenticación multifactor con control total sobre la identidad visual y la experiencia de usuario. Con el SDK de Guardian, puedes crear tus propias aplicaciones móviles personalizadas que funcionen como Guardian o integrar algunas de sus funcionalidades, como recibir notificaciones push en tus aplicaciones móviles existentes. Un caso de uso típico podría ser el de una aplicación bancaria. Puedes usar el SDK de Guardian en tu aplicación móvil actual para recibir y confirmar notificaciones push cuando alguien realiza una transacción en un cajero automático.

<div id="use-actions-to-enable-multi-factor-with-auth0-guardian">
  ## Usa Actions para habilitar la autenticación multifactor con Auth0 Guardian
</div>

Para habilitar Auth0 Guardian en una Action, pasa `guardian` como parámetro `provider` al habilitar la autenticación multifactor:

```js lines
exports.onExecutePostLogin = async (event, api) => {
  api.multifactor.enable('guardian', { allowRememberBrowser: false });
};
```

Para obligar a tus usuarios a iniciar sesión con Auth0 Guardian siempre, crea la Action con `allowRememberBrowser: false`.

<div id="multi-factor-with-auth0-guardian-and-authorization-extension">
  ## Autenticación multifactor con Auth0 Guardian y Authorization Extension
</div>

Esta plantilla ofrece un ejemplo y un punto de partida para activar la autenticación multifactor con Auth0 Guardian mediante notificaciones push cuando se cumple una condición.

En el primer inicio de sesión, el usuario puede registrar el dispositivo. Consulta Habilitar la autenticación multifactor para obtener más información sobre M

```js lines
exports.onExecutePostLogin = async (event, api) => {
const groups = event.user.app_metadata.authorization.groups;
const GROUPS_WITH_MFA = {
// Agrega aquí los grupos que necesitan MFA
// Ejemplo
admins: true
};

const needsMFA = !!groups.find(function (group) {
return GROUPS_WITH_MFA[group];
});

if (needsMFA) {
// opcional, el valor predeterminado es true. Establece en false para forzar la autenticación de Guardian en cada inicio de sesión.
// Consulta https://auth0.com/docs/secure/multi-factor-authentication/customize-mfa#change-frequency-of-mfa-prompts para más detalles
api.multifactor.enable('guardian', { allowRememberBrowser: false });
}

};
```
