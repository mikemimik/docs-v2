Puedes habilitar la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) para tu tenant en el <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

Cuando habilites MFA, selecciona los factores (como las notificaciones push o los mensajes al teléfono) que quieras poner a disposición de los usuarios y define una política para determinar cuándo deben autenticarse.

<div id="select-factors">
  ## Selecciona los factores
</div>

Auth0 ofrece varios factores que puedes habilitar para que tus usuarios completen la autenticación.

<div id="independent-factors">
  ### Factores independientes
</div>

Debes habilitar y configurar al menos un factor independiente para requerir MFA para los usuarios de tu tenant. Puedes habilitar varios factores independientes simultáneamente si lo deseas.

Los factores independientes disponibles son:

* [WebAuthn con claves de seguridad FIDO](/es/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)
* [Contraseña de un solo uso (OTP)](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)
* [Notificación push con Auth0 Guardian](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa)
* [Mensaje telefónico](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)
* [Cisco Duo Security](/es/docs/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="dependent-factors">
  ### Factores dependientes
</div>

Puede habilitar uno o más factores dependientes una vez que haya habilitado y configurado al menos un factor independiente para los usuarios de su tenant.

Los factores dependientes disponibles son:

* [WebAuthn con biometría FIDO](/es/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)
* [Email](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)
* [Código de recuperación](/es/docs/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="enable-mfa-in-the-auth0-dashboard">
  ## Habilitar MFA en el Auth0 Dashboard
</div>

1. Ve a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
2. En la sección **Factors**, habilita y configura los factores que quieras poner a disposición de los usuarios.
3. En la sección **Define policies**, selecciona una política para determinar cuándo se solicitará MFA a los usuarios.

   * **Never**: No se requiere MFA para ningún inicio de sesión.
   * **Use MFA adaptativo**: Se requiere MFA según la evaluación de riesgo de Auth0. Para obtener más información, consulta [Enable MFA adaptativo](/es/docs/secure/multi-factor-authentication/adaptive-mfa).
   * **Always**: Se requiere MFA para todos los inicios de sesión.
4. Si eliges la política **Never** o **Always**, aparece la sección **MFA Risk Assessors**. De forma predeterminada, la configuración **Enable MFA adaptativo Risk Assessment** está deshabilitada. Activa el interruptor para evaluar y registrar el riesgo de todas las transacciones de inicio de sesión en los logs de tu tenant.
5. En la sección **Additional Settings**, configura las siguientes opciones según sea necesario:

   * **Show Multi-factor Authentication options:** Activa este interruptor para permitir que los usuarios seleccionen factores de autenticación durante la inscripción. Cuando está deshabilitado, Auth0 muestra automáticamente a los usuarios los factores de autenticación más seguros durante la inscripción.

     <Frame>![Vista del usuario final de Show Multi-factor Authentication options](/docs/images/cdy7uua7fh8z/3Mo3EXUk3ZFK4hZKck0yOO/37268f33f70d1e09032f0db317ac11d3/MFA_Options_-_English.png)</Frame>
   * **Customize MFA Factors using Actions**: Activa este interruptor para personalizar tus flujos de MFA mediante [Actions](/es/docs/customize/actions) de `post-login`. Esto te permite pedir a los usuarios que completen un factor específico o una secuencia de factores, además de usar información contextual para crear experiencias más personalizadas. Para obtener más información, consulta [Customize MFA Selection for Universal Login](/es/docs/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
6. Haz clic en **Save**.

<div id="customize-mfa-flow">
  ## Personaliza el flujo de MFA
</div>

Puedes usar Auth0 Actions para personalizar el flujo de MFA para los usuarios de tu tenant. Por ejemplo, puedes crear una Action para exigir MFA en circunstancias específicas o para obligar a usar un factor determinado.

Para obtener más información, consulta [Personaliza el MFA adaptativo](/es/docs/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).
