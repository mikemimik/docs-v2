<div id="use-the-dashboard">
  ## Usa el Dashboard
</div>

Para habilitar **WebAuthn con Device Biometrics**, ve a [Dashboard &gt; Security &gt; Multifactor Auth](https://manage.auth0.com/#/security/mfa). Tendrás que habilitar un factor adicional, ya que este no puede ser el único factor habilitado.

<div id="configure-relying-party">
  ## Configurar Relying Party
</div>

WebAuthn hace que el phishing sea imposible al vincular las credenciales al origen del navegador. Los usuarios tampoco pueden usar WebAuthn en un sitio en el que no se registraron.

Vincular las credenciales al origen significa que, si configuras un <Tooltip tip="Custom Domain: Dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/es/docs/glossary?term=custom+domain">dominio personalizado</Tooltip> o lo cambias, los usuarios que se registraron antes del cambio no podrán autenticarse.

WebAuthn define un [atributo de ID de Relying Party](https://www.w3.org/TR/webauthn-2/#relying-party-identifier) que te permite especificar el dominio que se usa para autenticar a los usuarios. Puedes establecerlo como cualquier sufijo de dominio registrable del origen del navegador. Por ejemplo, si el dominio personalizado es `login.example.com`, puedes configurar el ID de <Tooltip tip="Relying Party: Entidad (como un servicio o una aplicación) que depende de un proveedor de identidad de terceros para autenticar a un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Relying+Party">Relying Party</Tooltip> como `example.com`. Esto permite a los usuarios autenticarse en cualquier dominio `example.com` con sus credenciales de WebAuthn.

Auth0 te permite especificar el ID de Relying Party solo si tienes configurado un dominio personalizado. Si el dominio personalizado cambia, debes actualizar el ID de Relying Party.

<div id="device-recognition">
  ## Reconocimiento de dispositivos
</div>

Cuando habilitas WebAuthn con Device Biometrics, Auth0 intentará registrar progresivamente todos los dispositivos de un usuario final compatibles con WebAuthn. Los navegadores con Javascript deshabilitado o sin compatibilidad con autenticador de plataforma de WebAuthn no tendrán la opción de registrarse ni de autenticarse con Device Biometrics.

Dado que no existe una forma determinista de saber si un dispositivo específico se registró o no sin solicitar al usuario una verificación con WebAuthn, Auth0 se basa en el agente de usuario para decidir qué hacer. El comportamiento depende del sistema operativo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En el caso más habitual, en el que los usuarios siempre usan el mismo navegador y tienen un móvil y una computadora portátil o de escritorio, los matices de comportamiento que se describen a continuación no afectarán a la experiencia del usuario.
</Callout>

<div id="windows-and-ios-145">
  ### Windows e iOS 14.5+
</div>

En Windows e iOS 14.5+, el autenticador de plataforma de WebAuthn se registra a **nivel del sistema operativo**. Los usuarios pueden registrarse con un navegador e iniciar sesión con cualquier navegador. Si Auth0 detecta que los usuarios tienen un dispositivo registrado, se les ofrecerá la opción de autenticarse con Face ID / Touch ID / Windows Hello. Si se registraron con ese mismo dispositivo, podrán autenticarse. De lo contrario, no funcionará y tendrán que usar otro método de autenticación.

<div id="macos">
  ### macOS
</div>

En Mac, el autenticador de plataforma de WebAuthn se registra a nivel de sistema del **navegador**. A los usuarios se les pedirá que registren WebAuthn en cada navegador que utilicen. Si Auth0 detecta que el usuario se registró desde Chrome en una Mac, tendrá la opción de autenticarse con Touch ID cuando inicie sesión desde Chrome en una Mac. Si se registró desde esa misma Mac, podrá autenticarse.  De lo contrario, fallará y tendrá que usar otro método de autenticación. Si intenta registrarse desde Safari en la misma Mac, se le pedirá que complete <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación del usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=MFA">MFA</Tooltip> con el otro método de autenticación y luego **se le pedirá que se registre** con Touch ID.

<div id="android">
  ### Android
</div>

En Android, **solo Chrome** es compatible con los autenticadores de plataforma WebAuthn. Si Auth0 detecta que los usuarios tienen registrado un dispositivo Android, se les ofrecerá la opción de autenticarse con el reconocimiento facial o de huella digital de Android. Si se registraron con ese mismo dispositivo Android, podrán autenticarse. Si no, el proceso fallará y tendrán que usar otro método de autenticación.

<div id="device-support">
  ## Compatibilidad de dispositivos
</div>

El usuario debe tener activada otra inscripción en MFA antes de usar la Device Biometrics.

Para usar claves de Device Biometrics, el navegador debe tener Javascript habilitado y ser compatible con los autenticadores de plataforma de WebAuthn. Si no se cumplen esas condiciones, Auth0 no ofrecerá la opción de inscribirse ni de autenticarse con el dispositivo. Auth0 solicitará al usuario que use otro factor.

Las versiones más recientes de los navegadores y sistemas operativos más populares son compatibles con WebAuthn with Security Keys. Para obtener más detalles, consulta la [sección de compatibilidad de navegadores en webauthn.me](https://webauthn.me/browser-support).

<div id="limitations">
  ## Limitaciones
</div>

* No hay forma de registrar la Device Biometrics con WebAuthn fuera del prompt de inscripción progresiva.
* Al usar la [API de MFA](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-developer-resources/mfa-api), puedes listar y eliminar inscripciones de WebAuthn, pero no registrarlas.
* Los usuarios solo pueden registrar un dispositivo de cada tipo con WebAuthn y Device Biometrics (un teléfono, una tableta, una laptop o equipo de escritorio). Si un usuario quiere registrar otro dispositivo del mismo tipo, primero debe anularse la inscripción del dispositivo anterior.
