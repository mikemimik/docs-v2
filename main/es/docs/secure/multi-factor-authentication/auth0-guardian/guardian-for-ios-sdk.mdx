title: Guardian.swift iOS SDK
---

[Guardian.swift](https://github.com/auth0/Guardian.swift) te permite integrar el servicio de autenticación multifactor Guardian de Auth0 en tu propia app de iOS, transformándola en el propio segundo factor. Tus usuarios obtendrán todos los beneficios de nuestra <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación del usuario que utiliza un factor adicional a username y contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> directamente desde tu aplicación. Para obtener más información, consulta [Getting Started with Apple Push Notification Service](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa#configure-push-notifications-for-apple-using-apn-).

<div id="requirements">
  ## Requisitos
</div>

* Se requiere iOS 10+ y Swift 4.1 para usar Guardian.
* Para usar este SDK, debes configurar el servicio Guardian de tu tenant con tus propias credenciales de notificaciones push; de lo contrario, no recibirás ninguna notificación push. Para obtener más información, consulta [Configure Push Notifications for MFA](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa).

<div id="install-guardian-ios-sdk">
  ## Instalar el SDK de Guardian para iOS
</div>

<div id="cocoapods">
  ### CocoaPods
</div>

Guardian.swift está disponible mediante [CocoaPods](http://cocoapods.org). Para instalarlo, añade la siguiente línea a tu Podfile:

```bash lines
pod 'Guardian', '~> 1.1.0'
```

<div id="carthage">
  ### Carthage
</div>

Añade esta línea a tu `Cartfile`:

```bash lines
github "auth0/Guardian.swift" ~> 1.1.0
```

<div id="enable-guardian-push-notifications">
  ## Habilitar las notificaciones push de Guardian
</div>

1. Ve a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian).
2. Activa **Push Notification** para habilitar las notificaciones push.
3. [Configura las notificaciones push](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa#configure-push-notifications-for-apple-using-apn-).

<div id="usage">
  ## Uso
</div>

`Guardian` es el núcleo del SDK. Para usar el SDK, importa la biblioteca:

```swift lines
import Guardian
```

Establece el dominio de tu tenant. O usa el <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre específico o personalizado." cta="Ver glosario" href="/es/docs/glossary?term=custom+domain">dominio personalizado</Tooltip> si configuraste uno para tu tenant:

```swift lines
let domain = "<tenant>.<region>.auth0.com"
```

<div id="enroll">
  ### Inscribir
</div>

Una inscripción es un vínculo entre el segundo factor y una cuenta de Auth0. Cuando una cuenta está inscrita, tendrás que proporcionar el segundo factor necesario para verificar la identidad. Si tu aplicación todavía no usa notificaciones push o no estás familiarizado con ellas, consulta [Apple Push Notification Service Overview](https://developer.apple.com/go/?id=push-notifications) para obtener más información.

Para una inscripción necesitas la siguiente información, además del dominio de tu tenant:

| Variable               | Descripción                                                                                                                                    |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **URI de inscripción** | Valor codificado en el código QR escaneado desde Guardian Web Widget o en el ticket de inscripción que recibiste por correo electrónico o SMS. |
| **Token de APNS**      | Token de APNS de Apple para el dispositivo. Debe ser una cadena que contenga los 64 bytes (en formato hexadecimal).                            |
| **Par de claves**      | Un par de claves RSA (privada/pública) usado para acreditar tu identidad ante Auth0 Guardian.                                                  |

Después de tener la información, puedes inscribir tu dispositivo:

```swift lines
Guardian
        .enroll(forDomain: "{yourTenantDomain}",
                usingUri: "{enrollmentUri}",
                notificationToken: "{apnsToken}",
                signingKey: signingKey,
                verificationKey: verificationKey
                )
        .start { result in
            switch result {
            case .success(let enrolledDevice):
                // éxito, tenemos los datos del dispositivo de inscripción disponibles
            case .failure(let cause):
                // algo falló, verifica la causa para ver qué salió mal
            }
        }
```

Si todo sale bien, obtendrás la información de inscripción, que debe almacenarse de forma segura en tu aplicación. Esta información incluye el identificador de inscripción y el token de la API de Guardian asociado a tu dispositivo para actualizar o eliminar tu inscripción.

<div id="signing-and-verification-keys">
  #### Claves de firma y verificación
</div>

Guardian.swift proporciona una clase de utilidad para generar una clave de firma:

```swift lines
let signingKey = try DataRSAPrivateKey.new()
```

Esta clave solo existe en memoria, pero puedes obtener su representación en `Data` y almacenarla de forma segura, por ejemplo, en una SQLiteDB cifrada:

```javascript lines
// Almacenar datos
let data = signingKey.data
// realizar el almacenamiento

// Cargar desde el almacenamiento
let loadedKey = try DataRSAPrivateKey(data: data)
```

Pero si solo quieres guardarlo en el Keychain de iOS:

```swift wrap lines
let signingKey = try KeychainRSAPrivateKey.new(with: "com.myapp.mytag")
```

El ejemplo anterior crea una clave y la almacena automáticamente con la etiqueta proporcionada. Si quieres recuperarla, puedes usar la etiqueta:

```swift wrap lines
let signingKey = try KeychainRSAPrivateKey(tag: "com.myapp.mytag")
```

Para la clave de verificación, basta con obtenerla de cualquier `SigningKey`, por ejemplo:

```swift lines
let verificationKey = try signingKey.verificationKey()
```

<div id="allow-login-requests">
  ### Permitir solicitudes de inicio de sesión
</div>

Una vez que hayas completado la inscripción, recibirás una notificación push cada vez que el usuario tenga que validar su identidad con MFA. Guardian proporciona un método para procesar los datos recibidos de APNs y devolver una instancia de `Notification` lista para usar.

```swift lines
if let notification = Guardian.notification(from: notificationPayload) {
    // hemos recibido una notificación push de Guardian
}
```

Una vez que tengas la instancia de la notificación, puedes aprobar fácilmente la solicitud de autenticación con el método `allow`. También necesitarás cierta información del dispositivo inscrito que obtuviste anteriormente. Si tienes más de una inscripción, tendrás que encontrar la que tenga el mismo `id` que la notificación (la propiedad `enrollmentId`).

Cuando tengas la información, el parámetro `device` puede ser cualquier objeto que implemente el protocolo `AuthenticatedDevice`:

```swift lines
struct Authenticator: Guardian.AuthenticationDevice {
    let signingKey: SigningKey
    let localIdentifier: String
}
```

El identificador local es el ID local del dispositivo; de forma predeterminada, al inscribirlo se usa `UIDevice.current.identifierForVendor`. Luego, solo llama a:

```swift lines
Guardian
        .authentication(forDomain: "{yourTenantDomain}", device: device)
        .allow(notification: notification)
        .start { result in
            switch result {
            case .success:
                // la solicitud de autenticación fue aprobada exitosamente
            case .failure(let cause):
                // algo falló, revisa cause para ver qué salió mal
            }
        }
```

<div id="reject-login-requests">
  ### Rechazar solicitudes de inicio de sesión
</div>

Para denegar una solicitud de autenticación, llama a `reject`. También puedes enviar un motivo de rechazo opcional. El motivo de rechazo aparecerá en los registros de Guardian.

```swift lines
Guardian
        .authentication(forDomain: "{yourTenantDomain}", device: device)
        .reject(notification: notification)
        // o reject(notification: notification, withReason: "hacked")
        .start { result in
            switch result {
            case .success:
                // la solicitud de autenticación fue rechazada exitosamente
            case .failure(let cause):
                // algo falló, revisa cause para ver qué salió mal
            }
        }
```

<div id="unenroll">
  ### Cancelar la inscripción
</div>

Si quieres eliminar una inscripción, por ejemplo, para desactivar MFA, puedes hacer la siguiente solicitud:

```swift lines
Guardian
        .api(forDomain: "{yourTenantDomain}")
        .device(forEnrollmentId: "{userEnrollmentId}", token: "{enrollmentDeviceToken}")
        .delete()
        .start { result in
            switch result {
            case .success:
                // éxito, la inscripción fue eliminada
            case .failure(let cause):
                // algo falló, revisa cause para ver qué salió mal
            }
        }
```

<div id="set-up-mobile-only-otp-enrollment">
  ### Configurar la inscripción de OTP solo en dispositivos móviles
</div>

Puedes habilitar las contraseñas de un solo uso (OTP) como factor de MFA desde el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Esta opción no requiere un código QR y permite a los usuarios inscribirse manualmente.

Para invitar a un usuario a inscribirse, ve a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y selecciona un usuario. Luego, abre la pestaña Details y usa la sección Multi-Factor Authentication para enviar una invitación de inscripción.

<div id="connect-a-resource">
  #### Conectar un recurso
</div>

Puedes conectar un recurso con Auth0 Dashboard o con Guardian SDK.

<div id="use-auth0-dashboard">
  ##### Usa Auth0 Dashboard
</div>

1. Accede al prompt de inicio de sesión de Auth0 y copia el código proporcionado o una clave similar codificada en base32 obtenida de otra fuente. En el siguiente paso, introducirás este código en una aplicación de autenticación.

   <Frame>![Un ejemplo de prompt de inicio de sesión que muestra un código de un solo uso](/docs/images/cdy7uua7fh8z/1yoqiIuERVTwCU8yfx6IM8/047513dfe1d40a22ce811b131d5ea289/OTP_Challenge_2_-_English.png)</Frame>
2. Agrega el código que copiaste a una aplicación de autenticación, como Guardian.

<div id="use-the-sdk">
  ##### Usa el SDK
</div>

1. Importa la biblioteca Guardian.

   ```swift lines
   import Guardian
   ```

2. Crea un generador de código.

   ```swift lines
   let codeGenerator = try Guardian.totp(

      base32Secret: enrollmentCode,  // Código de inscripción introducido por el usuario

      algorithm: .sha1			// Algoritmo utilizado por TOTP

   )
   ```

3. Obtén el código generado.

   ```swift lines
   codeGenerator.code()
   ```

<div id="enter-one-time-code">
  #### Ingresa el código de un solo uso
</div>

En el prompt de inicio de sesión de Auth0, ingresa el código que generaste en el paso anterior.

<Frame>![Ejemplo de un prompt de inicio de sesión que muestra un código de un solo uso](/docs/images/cdy7uua7fh8z/1yoqiIuERVTwCU8yfx6IM8/047513dfe1d40a22ce811b131d5ea289/OTP_Challenge_2_-_English.png)</Frame>

Después de seleccionar Continuar, se muestra un mensaje que indica que tu aplicación se ha añadido como factor de autenticación para tu usuario.
