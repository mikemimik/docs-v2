<Card title="Antes de empezar">
  * Configura una [aplicación de máquina a máquina (M2M)](/es/docs/get-started/auth0-overview/create-applications/machine-to-machine-apps) y [otórgale acceso a la Auth0 Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los siguientes scopes:

    * `create:authentication_methods`
    * `read:authentication_methods`
    * `update:authentication_methods`
    * `delete:authentication_methods`
</Card>

La [Auth0 Management API](https://auth0.com/docs/api/management/v2) proporciona varios endpoints que puedes usar para administrar los métodos de autenticación MFA de tus usuarios. <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación del usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código vía SMS." cta="Ver glosario" href="/es/docs/glossary?term=MFA">MFA</Tooltip>

Este método se basa en la autenticación mediante una aplicación confidencial. Para obtener más información sobre las aplicaciones confidenciales y públicas, consulta [Aplicaciones confidenciales y públicas](/es/docs/get-started/applications/confidential-and-public-applications).

<div id="get-all-authentication-methods">
  ## Obtener todos los métodos de autenticación
</div>

Usa el endpoint [Obtiene una lista de métodos de autenticación](https://auth0.com/docs/api/management/v2#!/Users/get_authentication_methods) para obtener una lista de todos los métodos de autenticación en los que un usuario se ha registrado de forma total o parcial.

Este endpoint requiere el scope: `read:authentication_methods`.

### Ejemplos

La siguiente solicitud devuelve una lista de todos los métodos de autenticación de un usuario especificado.

<AuthCodeGroup>
  ```bash cURL lines
  curl --request GET \
    --url https://%7ByourDomain%7D/api/v2/users/%7BuserId%7D/authentication-methods \
    --header 'authorization: Bearer {yourMgmtApiAccessToken}'
  ```

  ```csharp C# lines
  var client = new RestClient("https://%7ByourDomain%7D/api/v2/users/%7BuserId%7D/authentication-methods");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go lines
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://%7ByourDomain%7D/api/v2/users/%7BuserId%7D/authentication-methods"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java lines

    .header("authorization", "Bearer {yourMgmtApiAccessToken}")
    .asString();
  ```

  ```javascript Node.JS lines

  var options = {
    method: 'GET',
    url: 'https://%7ByourDomain%7D/api/v2/users/%7BuserId%7D/authentication-methods',

  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP lines

    CURLOPT_URL => "https://%7ByourDomain%7D/api/v2/users/%7BuserId%7D/authentication-methods",

    CURLOPT_ENCODING => "",
    CURLOPT_MAXREDIRS => 10,
    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,

  ```

  ```ruby Ruby lines

  puts response.read_body
  ```

<div id="create-an-authentication-method">
  ## Crear un método de autenticación
</div>

Usa el endpoint [Crear un método de autenticación para un usuario determinado](https://auth0.com/docs/api/management/v2#!/Users/post_authentication_methods) para crear un método de autenticación para un usuario, como SMS, Email, contraseña de un solo uso (OTP) o WebAuthn with clave de seguridad. Para obtener más información sobre los factores de autenticación de MFA disponibles, consulta [Factores de autenticación multifactor](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors).

Este endpoint requiere el scope `create:authentication_methods`.

<Warning>
  Los métodos de autenticación creados a través de este endpoint se confirmarán automáticamente y estarán disponibles de inmediato. Verifica con el usuario que el método de autenticación esté configurado correctamente y que siga siendo válido.
</Warning>

  ### Contraseñas de un solo uso (OTP)
</div>
