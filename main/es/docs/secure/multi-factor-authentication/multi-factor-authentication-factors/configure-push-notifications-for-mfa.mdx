title: Configurar notificaciones push para MFA
---

Los SDK de [Guardian](/es/docs/secure/multi-factor-authentication/auth0-guardian) para [iOS](/es/docs/secure/multi-factor-authentication/auth0-guardian/guardian-for-ios-sdk) y [Android](/es/docs/secure/multi-factor-authentication/auth0-guardian/guardian-for-android-sdk) te permiten usar una aplicación desarrollada a medida con servicios de notificaciones push de proveedores específicos. Puedes configurar las notificaciones push para la plataforma AWS Simple Notification Service (SNS) o usar los servicios Direct to Vendor para agregar directamente en Auth0 las credenciales de Firebase Cloud Messaging (FCM) y Apple Push Notification (APN).

<div id="aws-sns-configuration">
  ## Configuración de AWS SNS
</div>

Antes de configurar las notificaciones push, sigue estos requisitos previos:

1. [Crea una aplicación de plataforma de SNS](https://console.aws.amazon.com/sns/v3/home?region=us-east-1#/mobile/push-notifications/platform-applications) con la consola de administración de AWS y anota su ARN.
2. Crea un AWS Access Key ID con autorización para crear endpoints de aplicaciones de plataforma. Guardian crea automáticamente un endpoint de aplicación de plataforma con el token de dispositivo correspondiente como parte de un registro correcto.

<div id="configure-push-notifications-with-aws-sns">
  ### Configurar las notificaciones push con AWS SNS
</div>

Para recibir notificaciones push de Guardian, primero debes reemplazar la configuración predeterminada de SNS de Guardian. Después, puedes configurar las notificaciones push para Android e iOS.

1. En el Auth0 Dashboard, ve a [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa) y selecciona **Push via Auth0 Guardian**. Si el método está inactivo, usa el interruptor de la esquina superior derecha para habilitarlo.
2. Selecciona **Custom** para mostrar opciones adicionales.
3. En la sección Push Notification Service, elige **Amazon SNS.**

<Frame>![Auth0 Dashboard > Security > Multi-factor Auth > Guardian > Custom > Amazon SNS](/docs/images/cdy7uua7fh8z/6PKSyH99GJ2CalJoG38sfZ/fbe7229dcc6f7d303a582520001b3dc1/Amazon_SNS.png)</Frame>

1. En la sección Amazon SNS Configuration, completa los siguientes campos:

   | Campo                 | Descripción                        |
   | --------------------- | ---------------------------------- |
   | AWS Access Key ID     | Tu AWS Access Key ID.              |
   | AWS Secret Access Key | Tu AWS Secret Access Key.          |
   | AWS Region            | La región de tu aplicación de AWS. |
2. Selecciona **Save** para guardar los cambios.

<div id="configure-push-notifications-for-android-using-fcm">
  ## Configurar las notificaciones push para Android con FCM
</div>

<Tabs>
  <Tab title="Obtener credenciales de FCM">
    Para configurar las notificaciones push de Android en tu Auth0 Dashboard, primero obtén las credenciales de tu servidor de FCM. Para más información, consulta la [documentación de Firebase](https://firebase.google.com/docs/admin/setup#initialize-sdk) de Google.

    1. Ve a la Consola de Firebase y selecciona el proyecto correspondiente.
    2. Abre el menú **Settings** y elige **Service Accounts**.
    3. Selecciona **Generate New Private Key** para descargar el archivo de credenciales.

    Una vez que hayas obtenido tus credenciales, podrás configurar las notificaciones push en tu Auth0 Dashboard.
  </Tab>

  <Tab title="Configurar Auth0 Dashboard">
    Después de obtener tus credenciales, puedes configurar las notificaciones push de Android desde tu Auth0 Dashboard.

    1. Ve a [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian) y selecciona **Push Notification using Auth0 Guardian**.
    2. En la sección Push Notification App, selecciona **Custom** para mostrar opciones adicionales.
    3. En la sección Android App Configuration, configura lo siguiente:

       | Configuración                    | Descripción                                                                                                                                            |
       | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
       | Habilitar aplicación de Android  | Permite que Auth0 envíe notificaciones push a tu aplicación personalizada. Si esta opción no está habilitada, no se podrán enviar notificaciones push. |
       | URL de Google Play Store         | Proporciona un enlace a la aplicación personalizada que los usuarios deben descargar para configurar MFA.                                              |
       | Credenciales del servidor de FCM | Sube tus [credenciales del servidor de FCM](https://firebase.google.com/docs/admin/setup#initialize-sdk).                                              |
    4. Selecciona **Guardar** para completar los cambios.
  </Tab>
</Tabs>

<div id="configure-push-notifications-for-ios-using-apns">
  ## Configurar las notificaciones push para iOS con APNs
</div>

<Tabs>
  <Tab title="Obtener certificado APN">
    Para configurar las notificaciones push de iOS en tu Auth0 Dashboard, primero obtén tu certificado APN.

    1. Genera un [nuevo certificado APN](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns#2947597) desde tu cuenta de desarrollador de Apple.

    2. Instala el certificado localmente.

    3. Busca el certificado **Apple Sandbox Push Services: {AppId}** en Acceso a Llaveros.

    4. Exporta el certificado y la clave privada asociada a un archivo P12 sin contraseña (nombre predeterminado: `Certificates.p12`).

    5. Para generar un archivo P12 con Triple DES en lugar de RC2, ejecuta

       ```bash lines
       openssl pkcs12 -in Certificates.p12 -legacy -nocerts -nodes -out pk.pem -passin pass:"" &&
       openssl pkcs12 -in Certificates.p12 -legacy -nokeys -out cert.crt -passin pass:"" &&
       openssl pkcs12 -export -inkey pk.pem -in cert.crt -descert -out Certificate_3des.p12 -passout pass:"" &&
       rm pk.pem cert.crt
       ```

    6. Elimina `Certificates.p12` y usa `Certificate_3des.p12` como certificado APN en Auth0 Dashboard.

    Después de obtener tus credenciales, puedes configurar las notificaciones push en tu Auth0 Dashboard.
  </Tab>

  <Tab title="Configurar Auth0 Dashboard">
    Después de obtener tus credenciales, puedes configurar las notificaciones push de iOS desde tu Auth0 Dashboard.

    1. Ve a [Security &gt; Multi-factor Auth](https://manage.auth0.com/#/guardian) y selecciona **Push Notification using Auth0 Guardian**.
    2. En la sección Push Notification App, selecciona **Custom** para mostrar opciones de configuración adicionales.
    3. En la sección iOS App Configuration, configura lo siguiente:

       | Field               | Description                                                                                                                                                                         |
       | ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
       | Enable iOS App      | Permite que Auth0 envíe notificaciones push a tu aplicación personalizada. Si esta opción no está habilitada, no se podrán enviar notificaciones push.                              |
       | Apple App Store URL | Proporciona un enlace a la aplicación personalizada que los usuarios deben descargar para configurar MFA.                                                                           |
       | APNs Bundle ID      | Lo proporciona iOS Developer Console.                                                                                                                                               |
       | APNs Certificate    | Sube tu [certificado P12](https://developer.apple.com/documentation/usernotifications/setting_up_a_remote_notification_server/establishing_a_certificate-based_connection_to_apns). |
       | iOS App Environment | Configúralo según la configuración de tu cuenta de desarrollador de Apple.                                                                                                          |
    4. Selecciona **Save** para completar los cambios.
  </Tab>
</Tabs>
