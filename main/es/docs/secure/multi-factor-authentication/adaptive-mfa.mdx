es una política de

<div id="how-it-works">
  ## Cómo funciona
</div>

Durante una transacción de inicio de sesión, Adaptive MFA calcula una puntuación de confianza general a partir del análisis de tres evaluaciones de riesgo:

| Evaluador                    | Señal de riesgo                                                                                                                                               | Cómo se calcula                                                                                                                                                                                                                                                                           |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `NewDevice`                  | El usuario intenta iniciar sesión desde un dispositivo que no se ha utilizado para acceder a la cuenta en los últimos 30 días.                                | El agente de usuario y las cookies del navegador identifican un dispositivo. En el momento del inicio de sesión, la información del dispositivo se compara con la lista de dispositivos de la cuenta.                                                                                     |
| `ImpossibleTravel`           | El usuario intenta iniciar sesión desde una ubicación geográfica que, en comparación con el último inicio de sesión, indica una situación de viaje imposible. | Se calcula la distancia entre la última ubicación válida y la ubicación del intento de inicio de sesión; la diferencia de tiempo entre ambos se utiliza para calcular una velocidad de desplazamiento hipotética. Esa velocidad se compara con una velocidad de desplazamiento razonable. |
| `UntrustedIP`                | El usuario intenta iniciar sesión desde una dirección IP que se sabe que está asociada a comportamientos sospechosos.                                         | Auth0 utiliza inteligencia obtenida de eventos de tráfico para determinar la probabilidad de que la dirección IP haya sido utilizada por actores maliciosos para perpetrar ataques de gran volumen.                                                                                       |
| Puntuación de riesgo general | Una combinación de los 3 factores anteriores.                                                                                                                 | Auth0 utiliza las 3 puntuaciones para asignar una puntuación general. Usa [Actions](/es/docs/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa) si quieres implementar tu propia lógica de negocio.                                                                     |

Cuando Adaptive MFA determina que la puntuación de confianza general es baja (es decir, que la transacción de inicio de sesión es de alto riesgo), requiere que el usuario verifique su identidad con MFA. Si el usuario no está inscrito en MFA, debe completar una verificación adicional antes de poder hacerlo.

Adaptive MFA incluye un flujo de seguridad integral que garantiza la autenticidad de los usuarios:

<Frame>![Diagrama del flujo de autenticación multifactor adaptativa de Auth0 Login](/docs/images/cdy7uua7fh8z/6NpeGW8jtQIma1jTVydW7U/d0f80c28226abfe10863f34d30f1a0be/Adaptive_MFA_Flow.png)</Frame>

Adaptive MFA ignora cualquier sesión de MFA existente (por ejemplo, si un usuario seleccionó **Remember this browser** durante un flujo de MFA anterior) y no permite que los usuarios omitan los desafíos de MFA.

  ## Soporte y limitaciones

<div id="authorization-flows">
  ### Flujos de autorización
</div>

Adaptive MFA es compatible con todos los flujos de autenticación y <Tooltip tip="Flujo de autorización: concesión de autorización (o workflow) especificada en el marco de OAuth 2.0." cta="Ver glosario" href="/es/docs/glossary?term=authorization+flows">flujos de autorización</Tooltip> que comienzan con el usuario final. Para obtener más información sobre los distintos flujos y protocolos, consulta [Flujos de autenticación y autorización](/es/docs/get-started/authentication-and-authorization-flow) y [Protocolos](/es/docs/authenticate/protocols).

| Protocolo     | Flujo                                                     | Compatible     |
| ------------- | --------------------------------------------------------- | -------------- |
| OIDC/OAuth2   | Flujo de código de autorización                           | Compatible     |
| OIDC/OAuth2   | Flujo de código de autorización con PKCE                  | Compatible     |
| OIDC/OAuth2   | Flujo implícito con Form Post                             | Compatible     |
| OIDC/OAuth2   | Flujo híbrido                                             | Compatible     |
| OIDC/OAuth2   | Credenciales de cliente                                   | No compatible  |
| OIDC/OAuth2   | Autorización de dispositivo                               | No compatible  |
| OIDC/OAuth2   | Contraseña del propietario del recurso (ROP)              | No compatible  |
| OIDC/OAuth2   | Custom Token Exchange                                     | No compatible  |
| SAML          | Iniciado por el proveedor de servicios (iniciado por SP)  | Compatible     |
| SAML          | Iniciado por el proveedor de identidad (iniciado por IdP) | No compatible* |
| WS-Federation | N/A                                                       | Compatible     |
| AD/LDAP       | N/A                                                       | Compatible     |

*Adaptive MFA no es compatible con los flujos <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip> iniciados por <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/es/docs/glossary?term=IdP">IdP</Tooltip>, pero puedes simular este flujo con aplicaciones OIDC. Para obtener más información, consulta [Configurar el inicio de sesión SAML iniciado por IdP para aplicaciones OIDC](/es/docs/authenticate/protocols/saml/saml-sso-integrations/configure-idp-initiated-saml-sign-on-to-oidc-apps).

<div id="social-connections">
  ### Conexiones sociales
</div>

Adaptive MFA es totalmente compatible con los tipos de conexión social en los que cada usuario tiene una dirección de correo electrónico disponible.

Adaptive MFA requiere una dirección de correo electrónico para completar el paso de verificación por correo electrónico que se produce cuando un usuario no está registrado en MFA. Si no hay una dirección de correo electrónico disponible, Adaptive MFA no puede realizar la verificación por correo electrónico y la transacción se bloqueará. Este escenario no supone un riesgo de seguridad, pero sí limita la funcionalidad de la característica.

Si has configurado una conexión social y esperas que haya una dirección de correo electrónico disponible, pero no es así, verifica tu configuración y confirma que se estén solicitando los scopes, claims y permisos correctos. Para obtener más información sobre las conexiones sociales compatibles y cómo instalarlas, consulta [Conexiones sociales en Auth0 Marketplace](https://marketplace.auth0.com/features/social-connections).
