---

**Las alertas de monitoreo de seguridad** permiten a los usuarios configurar umbrales para métricas de seguridad en su Security Center. Se pueden especificar valores numéricos de alerta, advertencia y recuperación para cada umbral, y también se pueden configurar notificaciones de alerta para los usuarios a fin de supervisar cuándo una métrica de amenazas supera un umbral establecido.

| Opción       | Descripción                                                                                                                                                |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Alerta       | Un valor numérico obligatorio que genera un aviso de alerta cuando la métrica evaluada supera el valor proporcionado.                                      |
| Advertencia  | Un valor numérico opcional que genera un aviso de advertencia cuando la métrica evaluada supera ese valor, si se proporciona.                              |
| Recuperación | Un valor numérico opcional que genera un aviso de recuperación cuando la métrica evaluada vuelve a un valor que ya no supera el umbral, si se proporciona. |

<div id="configure-and-update-alert-thresholds">
  #### Configurar y actualizar los umbrales de alerta
</div>

Los umbrales se calculan a partir de una media móvil ponderada para una métrica determinada y se pueden personalizar en tu <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Cada umbral definido puede verse en los gráficos de métricas del monitoreo de amenazas y se agrega por hora; si no se configura un umbral de recuperación, la recuperación se establece de forma predeterminada justo por debajo del umbral de advertencia o alerta definido.

<Warning>
  Los umbrales se aplican a la métrica original sin filtros. Cuando se aplican filtros, se muestran la línea del umbral original y las líneas de tendencia filtradas.
</Warning>

<Frame>![Una segunda captura de pantalla de nuestro producto de umbrales de Security Center](/docs/images/cdy7uua7fh8z/1Kd9d33WUCdlVa1SwEFkKQ/496722ce86bc00cc7f58d1e04e1fd006/MFA_success_rate_threshold_editor_-_english.png)</Frame>

1. Ve a [**Security &gt; Security Center &gt; Threat Monitoring**](https://manage.auth0.com/#/security/center/bot-detection) y elige un gráfico de métricas.
2. Selecciona el ícono **View Details** en la esquina superior derecha.
3. Ve al panel de umbrales que aparece debajo de la vista detallada del gráfico y elige **Create.**
4. Asigna un nombre al umbral y configura las siguientes opciones:

   * Cuándo el umbral debe activar una advertencia
   * Cuándo el umbral debe activar una alerta
   * Cuándo el umbral debe recuperarse
5. Si se han configurado destinos de notificación, estarán disponibles las siguientes opciones:

   * Selecciona un destino para recibir los avisos de alerta, advertencia y recuperación de la métrica,
   * Crea un destino nuevo seleccionando el **+**
   * Silencia temporalmente o de forma indefinida las notificaciones para todos los destinos del umbral en el menú desplegable **Mute Notifications**.
6. Selecciona **Save.**

Los umbrales también se pueden actualizar o eliminar en la pantalla de vista expandida. Los distintos umbrales del mismo gráfico están dentro del menú desplegable de la etiqueta umbral en la parte superior derecha.

<div id="manage-notification-destinations">
  #### Administrar destinos de notificación
</div>

Los destinos de notificación son endpoints a los que se envían avisos de alerta, advertencia y recuperación. Cada tenant está limitado a dos endpoints de destino, y se recomienda usar un editor de webhooks de terceros para personalizar el mensaje de la notificación.

1. Ve a la página Manage Destinations en tu Dashboard desde el panel de configuración de umbrales o en [**Security &gt; Security Center &gt; Manage Destinations**](https://manage.auth0.com/#/security/center/manage-destinations)
2. Selecciona el botón **New Destination** y proporciona los siguientes datos:

   1. Nombre
   2. URL de destino
   3. Token de Authorization
3. Selecciona **Save.** Para eliminar un endpoint de destino, ve a **More Actions &gt; Delete**.

<div id="notification-payload-data">
  #### Datos del payload de la notificación
</div>

Los campos raíz incluidos en el payload de la notificación son:

* `id`: ID de la alerta de Security Center
* `time`: fecha y hora UTC en que se superó el umbral
* `source`: URL del Dashboard del historial de alertas de Auth0 Security Center
* `data`: objeto de datos que contiene información relevante sobre la notificación de alerta

A continuación, se muestra el objeto de datos incluido en el payload de la notificación:

* `id`: ID de la alerta de Security Center
* `tenant`: tenant donde se originó la notificación de Security Center
* `evaluated_metric`: métrica de amenazas a la que se aplica el umbral
* `state`: estado de la notificación (ALERT/WARN/RECOVERED)
* `metric_value`: valor de la métrica evaluada durante los 60 minutos anteriores
* `alert_threshold`: valor del umbral de alerta configurado para la métrica de amenazas
* `warn_threshold`: valor del umbral de advertencia configurado para la métrica de amenazas
* `recovery_threshold`: valor del umbral de recuperación configurado para la métrica de amenazas
* `triggered_at`: fecha y hora UTC en que se superó el umbral.

<div id="view-alert-history">
  #### Ver el historial de alertas
</div>

Las notificaciones de alerta, advertencia y recuperación que se hayan producido pueden consultarse en [**Security &gt; Security Center &gt; Alert History**](https://manage.auth0.com/#/security/center/alert-history). Todas las notificaciones también se envían a tus destinos de notificación configurados.
