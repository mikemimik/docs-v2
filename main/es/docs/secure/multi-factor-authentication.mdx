<Card title="Información general">
  La autenticación multifactor (MFA) es un método adicional para proteger tu aplicación y las identidades de tus usuarios. La MFA añade una capa de seguridad durante el inicio de sesión que requiere que los usuarios proporcionen más de una credencial para demostrar su identidad digital. Los factores pueden ser:

  * Algo que eres, como un dato biométrico
  * Algo que sabes, como una contraseña
  * Algo que posees, como un dispositivo

  Lee estas preguntas y respuestas para ver si usar MFA con tu instancia de Auth0 es la opción adecuada para ti.
</Card>

<div id="what-is-multi-factor-authentication">
  ## ¿Qué es la autenticación multifactor?
</div>

<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Multi-factor+authentication">Autenticación multifactor</Tooltip> (MFA) es un método de verificación del usuario que requiere más de un tipo de validación de identidad. Evita que los <Tooltip tip="Bad Actors: Entidad (una persona o grupo) que representa una amenaza para la empresa o el entorno con la intención de causar daño." cta="Ver glosario" href="/es/docs/glossary?term=bad+actors">actores maliciosos</Tooltip> accedan a una cuenta incluso si han obtenido el nombre de usuario y la contraseña.

<div id="why-use-multi-factor-authentication">
  ## ¿Por qué usar la autenticación multifactor?
</div>

La MFA reduce la probabilidad de muchos tipos de ciberataques. Es habitual que terceros roben nombres de usuario y contraseñas o ataquen de forma automatizada las cuentas de usuario. Un factor de MFA adicional, como una huella dactilar o una contraseña de un solo uso, ayuda a impedir este tipo de vulneraciones.

<div id="how-does-multi-factor-authentication-work">
  ## ¿Cómo funciona la autenticación multifactor?
</div>

La MFA funciona exigiendo información de verificación adicional (conocida como factores). Los usuarios no pueden iniciar sesión usando solo nombres de usuario y contraseñas. Deben proporcionar una prueba de identidad adicional, como reconocimiento facial o notificaciones por SMS.

<div id="mfa-factors">
  ## Factores de MFA
</div>

<Warning>
  Los factores de MFA dependen de la disponibilidad del plan; algunos solo están disponibles en los planes Professional y Enterprise. Para obtener más información, consulta [Precios de Auth0](https://auth0.com/pricing).
</Warning>

Auth0 admite diversos factores de MFA, entre ellos:

* Notificaciones push
* Notificaciones por SMS
* Notificaciones de voz
* Contraseñas de un solo uso
* WebAuthn con claves de seguridad
* WebAuthn con biometría del dispositivo
* Notificaciones por correo electrónico
* Cisco Duo Security
* Códigos de recuperación

Para obtener más información, consulta [Factores de autenticación multifactor](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors).
