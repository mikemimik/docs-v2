---
title: Ver eventos de log de protección contra ataques
description: Describe cómo usar los datos de logs de tráfico del tenant para ver eventos de protección contra ataques.
---

Los logs de tu tenant contienen datos útiles que puedes usar para crear gráficos y analizar el perfil del tráfico que pasa por tu tenant. Esto resulta útil al evaluar la actividad de <Tooltip tip="Protección contra ataques: Funcionalidades que Auth0 proporciona para detectar y mitigar ataques, incluida la protección contra fuerza bruta, la limitación de IP sospechosas, la detección de contraseñas comprometidas, la detección de bots y la autenticación multifactor adaptativa." cta="Ver glosario" href="/es/docs/glossary?term=attack+protection">protección contra ataques</Tooltip>. Por ejemplo, puedes buscar los siguientes eventos para determinar si estás bajo ataque:

* Picos anormales de tráfico en el flujo de inicio de sesión que generan errores (como errores de nombre de usuario o contraseña incorrectos).
* Picos anormales de tráfico provenientes de ubicaciones de IP no esperadas.

Estos eventos suelen ocurrir sin que cambie mucho la tasa de inicios de sesión exitosos.

Puedes usar el campo `event` de los datos de logs de tu tenant para ver los datos de tráfico del tenant. Recomendamos crear un histograma diario de eventos fallidos de los siguientes tipos:

| Código de evento | Evento                                                                    |
| ---------------- | ------------------------------------------------------------------------- |
| `f`              | Inicio de sesión fallido                                                  |
| `fcoa`           | Autenticación de origen cruzado fallida                                   |
| `feccft`         | Intercambio fallido                                                       |
| `fepft`          | Intercambio fallido                                                       |
| `fsa`            | Autenticación silenciosa fallida                                          |
| `fu`             | Inicio de sesión fallido (correo electrónico/nombre de usuario no válido) |
| `pla`            | Evaluación de pre-login                                                   |
| `sepft`          | Intercambio exitoso                                                       |

Estos eventos fallidos dependen del flujo que hayas configurado con Auth0.

El siguiente ejemplo muestra un ataque de credential stuffing el 13/02, con un gran aumento de eventos de tipo `fu`, que corresponde a un nombre de usuario fallido (típico de un ataque de credential stuffing).

<div id="rate-of-errors-in-login-flow">
  ## Tasa de errores en el flujo de inicio de sesión
</div>

Busca un aumento repentino o una cantidad anormal de errores por nombre de usuario o contraseña incorrectos. Por ejemplo: ¿Esperarías &gt;30.000 errores por hora?

| Código de evento | Evento                                                                    |
| ---------------- | ------------------------------------------------------------------------- |
| `s`              | Inicio de sesión correcto                                                 |
| `fu`             | inicio de sesión fallido, correo electrónico/nombre de usuario no válido |
| `fp`             | inicio de sesión fallido, contraseña incorrecta                          |

Aquí tienes un ejemplo de cómo podrían verse los datos.

<Frame>![Gráfico de ejemplo de un aumento repentino de fallos de inicio de sesión en comparación con el tráfico normal](/docs/images/cdy7uua7fh8z/5PQBhfw1B1yx8S2BnHSDTk/ab753016110fbf957540aa5a534fea38/Login_events_by_type_-_English.png)</Frame>

<div id="rate-of-attack-protection-events">
  ## Tasa de eventos de protección contra ataques
</div>

Busque un volumen de tráfico anormalmente alto en eventos de protección contra ataques, como la <Tooltip tip="Detección de contraseñas vulneradas: forma de protección contra ataques en la que Auth0 notifica a sus usuarios si usan una combinación de nombre de usuario/contraseña que se ha visto comprometida en una filtración de datos en un sitio web o una aplicación de terceros." cta="Ver glosario" href="/es/docs/glossary?term=breached+password+detection">detección de contraseñas vulneradas</Tooltip> o los ataques de fuerza bruta en múltiples cuentas.

| Código de evento  | Evento                                                     |
| ----------------- | ---------------------------------------------------------- |
| `limit_mu`        | Dirección IP bloqueada                                     |
| `limit_wc`        | Cuenta bloqueada                                           |
| `pwd_leak`        | Contraseña vulnerada durante el inicio de sesión           |
| `signup_pwd_leak` | Contraseña vulnerada durante el registro                   |
| `reset_pwd_leak`  | Contraseña vulnerada durante la recuperación de contraseña |

Este es un ejemplo de cómo podrían verse los datos.

<Frame>![Gráfico de ejemplo de eventos de detección de anomalías](/docs/images/cdy7uua7fh8z/4hCSxSwOq5jmiwChEKnSuQ/731ba44b3be3fcb3f5a8e0cfac9ee89b/Types_-_English.png)</Frame>

<div id="number-of-ips-producing-errors-and-their-locations">
  ## Número de IP que generan errores y sus ubicaciones
</div>

Busca un número elevado de IP de ubicaciones que no tengan sentido. Por ejemplo: ¿Esperas recibir tráfico de 10.000 IP de Rusia todos los días? Observa los datos de las direcciones `ip` junto con el tráfico de eventos `fu` para determinar de dónde proviene el tráfico fallido.

Los datos de geolocalización de IP no están disponibles en los logs del tenant, a menos que puedas enriquecerlos desde otra fuente.
