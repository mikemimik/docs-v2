title: Agregar Bot Detection a flujos passwordless
---

Si usas flujos de login y signup con <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip>, puedes habilitar <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=Bot+Detection">Bot Detection</Tooltip> para renderizar un paso de CAPTCHA en los casos en que Auth0 determine que una solicitud es de alto riesgo.

<Warning>
  Si usas el [Lock SDK](/docs/es/libraries/lock) con la experiencia Classic Login, debes actualizar a la versión 11.35.0 o posterior para usar Bot Detection en flujos passwordless.

  Si usas el [Auth0.js SDK](/docs/es/libraries/auth0js), debes actualizar a la versión 9.20.0 o posterior para usar Bot Detection en flujos passwordless.
</Warning>

<div id="configure-bot-detection-for-passwordless-flows">
  ## Configurar Bot Detection para flujos passwordless
</div>

Puedes usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para configurar el comportamiento de la función CAPTCHA.

1. En el [Auth0 Dashboard](https://manage.auth0.com/dashboard), ve a [Security &gt; Attack Protection &gt; Bot Detection](https://manage.auth0.com/dashboard/#/security/attack-protection/bot-detection).
2. Busca la sección **Response**.
3. En la configuración **Enforce CAPTCHA for passwordless flows**, selecciona cuándo se debe exigir un CAPTCHA:

   * **Never**: Los usuarios no tienen que completar un CAPTCHA para iniciar sesión.
   * **When Risky**: Los usuarios tienen que completar un CAPTCHA si el inicio de sesión es de alto riesgo.
   * **Always**: Los usuarios siempre tienen que completar un CAPTCHA para iniciar sesión.
