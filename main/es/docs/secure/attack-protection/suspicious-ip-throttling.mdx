<Tooltip tip="Suspicious IP Throttling: Función de protección contra ataques que protege tu tenant frente a inicios de sesión sospechosos dirigidos a demasiadas cuentas desde una sola dirección IP." cta="Ver glosario" href="/docs/es/glossary?term=Suspicious+IP+Throttling">Suspicious IP Throttling</Tooltip> bloquea el tráfico de cualquier dirección IP que intente en poco tiempo demasiados inicios de sesión o registros. Esto ayuda a proteger tus aplicaciones frente a ataques de gran volumen dirigidos a múltiples cuentas.

Suspicious IP Throttling está habilitada de forma predeterminada cuando creas tu tenant de Auth0. Cuando Auth0 detecta un gran número de intentos de registro o de intentos fallidos de inicio de sesión desde una dirección IP, responde a los intentos posteriores con el [código de estado HTTP 429 Too Many Requests](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/429) hasta que esa dirección IP deje de estar sujeta a limitación.

Para obtener más información sobre los registros y los intentos fallidos de inicio de sesión registrados por tu tenant, consulta [Códigos de tipo de registro](/docs/es/deploy-monitor/logs/log-event-type-codes).

<div id="configure-suspicious-ip-throttling">
  ## Configurar Suspicious IP Throttling
</div>

Puedes configurar Suspicious IP Throttling en el [Auth0 Dashboard](https://manage.auth0.com/#) o con la [Auth0 Management API](https://auth0.com/docs/api/management/v2/attack-protection/patch-suspicious-ip-throttling).

<Warning>
  Auth0 recomienda encarecidamente no desactivar Suspicious IP Throttling.
</Warning>

1. Ve a [Dashboard &gt; Security &gt; protección contra ataques](https://manage.auth0.com/#/security/attack-protection) y selecciona **Suspicious IP Throttling**.
2. En la parte superior de la página, selecciona el interruptor para activar o desactivar Suspicious IP Throttling.

<div id="allow-trusted-ip-addresses-exceed-throttling-limits">
  ## Permita que las direcciones IP de confianza superen los límites de limitación
</div>

Puede hacer que hasta 100 direcciones IP individuales y/o rangos CIDR (IPv4 o IPv6) queden exentos de **Suspicious IP Throttling** agregándolos a la **IP AllowList**. Auth0 no bloquea ni alerta a los administradores del tenant cuando estas direcciones IP superan los límites de limitación.

1. Vaya a [Dashboard &gt; Security &gt; protección contra ataques](https://manage.auth0.com/#/security/attack-protection) y seleccione **Suspicious IP Throttling**.
2. En el campo **IP AllowList**, introduzca las direcciones IP y/o los rangos CIDR para los que desea permitir intentos ilimitados de inicio de sesión y registro. Separe varias direcciones IP y/o rangos CIDR con comas.

<div id="configure-response">
  ## Configurar la respuesta
</div>

De forma predeterminada, Auth0 envía correos electrónicos a los administradores del tenant cuando una dirección IP se marca como sospechosa.

Puedes configurar cómo responde Auth0 a intentos de inicio de sesión o registro a gran velocidad.

<Warning>
  Habilitar las funciones de protección contra ataques sin activar ninguna configuración de respuesta activa el modo de monitoreo, que solo registra los eventos relacionados en el log del tenant. Para obtener más información, consulta [Ver eventos de log de protección contra ataques](/docs/es/secure/attack-protection/view-attack-protection-events).
</Warning>

1. Ve a [Dashboard &gt; Security &gt; protección contra ataques](https://manage.auth0.com/#/security/attack-protection) y selecciona **Suspicious IP Throttling.**
2. Ubica la sección **Response**.
3. En la sección **Block Settings**, habilita **Limit high-velocity traffic targeting too many accounts** para limitar el tráfico de direcciones IP que superen el umbral de inicio de sesión o registro.
4. En la sección **Notifications**, habilita **Send notification to account administrator** para enviar automáticamente un correo electrónico a los administradores del tenant cuando una dirección IP supere el umbral de inicio de sesión o registro.
5. Selecciona **Save**.

<div id="configure-throttling-limits-and-rates">
  ## Configurar límites y tasas de limitación
</div>

Puedes personalizar cómo Auth0 aplica limitaciones a las direcciones IP sospechosas. Puedes cambiar:

* El número máximo de intentos fallidos de inicio de sesión y registro permitidos desde una dirección IP determinada.
* La tasa a la que las direcciones IP limitadas permiten nuevos intentos de inicio de sesión y registro.

Para obtener más información sobre el número máximo de intentos, la tasa de limitación y cómo funcionan, consulta [Cómo funciona Suspicious IP Throttling](#how-suspicious-ip-throttling-works).

1. Ve a [Dashboard &gt; Security &gt; protección contra ataques](https://manage.auth0.com/#/security/attack-protection) y selecciona **Suspicious IP Throttling.**

   <Frame>![Personalización de Suspicious IP Throttling en Auth0 Dashboard](/docs/images/cdy7uua7fh8z/5yLYmDn0FvDB061WgTH2fk/0f3dce552a41aa5756a2eca788944965/Screen_Shot_2021-09-29_at_10.19.32_AM.png)</Frame>
2. Busca la sección **Detection**.
3. Selecciona **Custom** en **Suspicious IP Thresholds.**
4. Configura los ajustes de **Login Threshold**:

   * **Maximum Attempts**: Introduce el número de intentos fallidos de inicio de sesión que una única dirección IP puede realizar en un día antes de que Auth0 bloquee el siguiente intento.
   * **Throttling Rate**: Introduce la tasa a la que se conceden nuevos tokens de inicio de sesión.
5. Configura los ajustes de **Signup Threshold**:

   * **Maximum Attempts**: Introduce el número de intentos de registro que una única dirección IP puede realizar en un minuto antes de que Auth0 bloquee el siguiente intento.
   * **Throttling Rate**: Introduce la tasa a la que se conceden nuevos tokens de registro.
6. Selecciona **Save**.

<div id="how-suspicious-ip-throttling-works">
  ### Cómo funciona la limitación para IP sospechosas
</div>

Auth0 contabiliza y permite por separado los intentos de inicio de sesión y de registro. Las direcciones IP suspendidas para seguir intentando iniciar sesión todavía pueden intentar registrarse. Las direcciones IP suspendidas para seguir intentando registrarse todavía pueden intentar iniciar sesión.

<div id="login-attempts">
  #### Intentos de inicio de sesión
</div>

Auth0 limita cualquier dirección IP que haga demasiados intentos fallidos de inicio de sesión en un día. La tasa de limitación de inicio de sesión determina cuántos intentos de inicio de sesión Auth0 concede a una dirección IP de forma uniforme a lo largo de 24 horas. Por ejemplo, una tasa de limitación de 100 significa que Auth0 concede un nuevo intento aproximadamente cada 15 minutos.

<div id="signup-attempts">
  #### Intentos de registro
</div>

Auth0 limita cualquier dirección IP que intente realizar demasiados registros en un minuto. Por ejemplo, si una dirección IP realiza 50 intentos de registro, Auth0 bloquea los intentos posteriores. A diferencia de los inicios de sesión, no es necesario que estos intentos de registro sean fallidos.

La tasa de limitación de registro determina cuántos intentos de registro permite Auth0 a una dirección IP, distribuidos de manera uniforme a lo largo de 24 horas. Por ejemplo, una tasa de limitación de 72,000 significa que Auth0 permite un nuevo intento aproximadamente cada segundo.

<div id="malformed-requests-and-schema-validation-errors">
  #### Solicitudes malformadas y errores de validación del esquema
</div>

Las solicitudes de inicio de sesión y de registro que no siguen la estructura o el formato correctos no se consideran intentos válidos y, por lo tanto, no cuentan para el umbral configurado.

Para obtener más información sobre las solicitudes de inicio de sesión y de registro, consulta [Auth0 Authentication API](https://auth0.com/docs/api/authentication).

<div id="special-cases">
  ## Casos especiales
</div>

Dado que la limitación de IP sospechosas depende de la dirección IP del usuario, los siguientes casos de uso requieren configuración adicional:

* **Usar la concesión de contraseña de <Tooltip tip="Resource Owner: Entidad (como un usuario o una aplicación) capaz de conceder acceso a un recurso protegido." cta="Ver glosario" href="/docs/es/glossary?term=Resource+Owner">Resource Owner</Tooltip> desde el backend de una aplicación**: esta llamada no obtiene la dirección IP del usuario; sin embargo, para que la limitación de IP sospechosas funcione correctamente, puedes configurar tu aplicación para enviar la dirección IP del usuario como parte de la solicitud. Consulta [Evita problemas comunes con el flujo de contraseña del propietario del recurso y protección contra ataques: envía la dirección IP del usuario desde tu servidor](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
* **Autenticar a una gran cantidad de usuarios desde la misma dirección IP**: los usuarios que están detrás de un proxy tienen más probabilidades de alcanzar los límites establecidos y activar la limitación. Puedes evitar activar la limitación por error configurando una AllowList para la IP y el rango CIDR del proxy. Consulta [Evita problemas comunes con el flujo de contraseña del propietario del recurso y protección contra ataques: configura tu aplicación para confiar en la dirección IP](/docs/es/get-started/authentication-and-authorization-flow/resource-owner-password-flow/avoid-common-issues-with-resource-owner-password-flow-and-attack-protection).
