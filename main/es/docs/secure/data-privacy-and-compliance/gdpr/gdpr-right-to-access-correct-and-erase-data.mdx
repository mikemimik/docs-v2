title: 'RGPD: Derecho a acceder, rectificar y eliminar datos'

De acuerdo con los artículos 15, 16, 17 y 19 del RGPD, los usuarios tienen derecho a obtener una copia de los datos personales que procesas, solicitar su rectificación si son inexactos y pedirte que elimines sus datos personales.

Con Auth0, puedes acceder a la información del usuario, editarla y eliminarla manualmente mediante el Dashboard o mediante programación con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

</Callout>

<div id="manual-process">
  ## Proceso manual
</div>

Puedes ver, editar y eliminar la información del usuario en [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users). Selecciona un usuario para ver su información. La información que puedes cambiar es:

| Campo              | Cómo editar                                                                                                                                                                            |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Email**          | Haz clic en Edit. Introduce el nuevo correo electrónico.                                                                                                                               |
| **Email verified** | Haz clic en **Edit** en el campo **Εmail**. Haz clic en el enlace **Set email as verified**.                                                                                           |
| **Metadata**       | Tanto los objetos `app_metadata` como `user_metadata` se pueden editar desde esta pantalla. Edita el JSON en la sección **Metadata** y guarda los cambios.                             |
| **Blocked**        | No se puede editar directamente. Haz clic en **Actions &gt; Block User** en la parte superior derecha de esta pantalla. Para desbloquearlo, haz clic en **Actions &gt; Unblock User**. |
| **Email**          | No se puede editar directamente. Haz clic en **Actions &gt; Change Email** en la parte superior derecha de esta pantalla.                                                              |
| **Password**       | No se puede editar directamente. Haz clic en **Actions &gt; Change Password** en la parte superior derecha de esta pantalla.                                                           |

Para eliminar un usuario, haz clic en **Actions &gt; Delete User**.

<div id="programmatic-process">
  ## Proceso programático
</div>

Puedes recuperar, editar y eliminar información de usuarios mediante nuestra API. Elige un endpoint que se ajuste a tus necesidades:

* Recuperar un usuario usando el ID como criterio de búsqueda
* Recuperar un usuario usando el Email como criterio de búsqueda
* Exportar todos los usuarios a un archivo mediante una tarea de larga duración
* Actualizar un usuario. Ten en cuenta que no todos los campos se pueden editar (consulta la siguiente sección). Recuerda que:

  * Las propiedades del nuevo objeto reemplazarán a las anteriores. Los campos `user_metadata` y `app_metadata` son una excepción a esta regla. Estas propiedades se combinan en lugar de reemplazarse, aunque la combinación solo ocurre en el primer nivel.
  * Si estás actualizando `email_verified`, `phone_verified`, `username` o `password`, debes establecer el parámetro `connection`.
  * Si estás actualizando `email` o `phone_number`, debes establecer los parámetros `connection` y `client_id`.
* Eliminar un usuario según el ID

Para llamar a cualquiera de los endpoints de la API, necesitarás un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> válido. Este token debe tener los permisos requeridos para cada endpoint. Cada endpoint en el explorador de la Management API tiene una sección de scopes que enumera el/los scope(s) que debe contener el token de acceso para poder acceder a él. Una vez que sepas a qué endpoint quieres acceder y tengas un token de acceso válido, estarás listo para enviar tu solicitud.

<div id="editable-data">
  ## Datos editables
</div>

La siguiente información del usuario puede actualizarse mediante la API:

* blocked
* email&#95;verified
* email
* verify&#95;email
* password
* phone&#95;number
* phone&#95;verified
* verify&#95;password
* user&#95;metadata
* app&#95;metadata
* username

La siguiente información del usuario **no** puede editarse:

* given&#95;name
* family&#95;name
* name
* nickname
* picture

<div id="searchable-fields">
  ## Campos de búsqueda
</div>

Puedes buscar usuarios mediante lo siguiente:

* Todos los campos del perfil de usuario normalizado
* La información del perfil del objeto `user_metadata`:

  * name
  * nickname
  * given&#95;name
  * family&#95;name

Es tu responsabilidad asegurarte de que el cliente se elimine o de que los datos se actualicen en cualquier otra base de datos a la que Auth0 no esté conectado.
