description: Obtén información sobre los scopes y endpoints disponibles para los tokens de la Management API para aplicaciones de una sola página (SPA).
title: Obtén tokens de acceso de la Management API para aplicaciones de una sola página
---

En ciertos casos, es posible que quieras usar la [Management API](https://auth0.com/docs/api/management/v2#!) de Auth0 para administrar tus aplicaciones y API en lugar del Dashboard de administración de Auth0.

Para llamar a cualquier endpoint de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>, debes autenticarte con un [token de acceso](/docs/es/secure/tokens/access-tokens) especializado llamado token de la Management API. Los tokens de la Management API son [JSON Web Tokens (JWT)](/docs/es/secure/tokens/json-web-tokens) que contienen permisos específicos otorgados (también conocidos como scopes) para los endpoints de la Management API que quieres invocar.

<div id="limitations">
  ## Limitaciones
</div>

Dado que las aplicaciones de una sola página (SPA) son <Tooltip tip="Cliente público: cliente (aplicación) que no puede almacenar credenciales de forma segura. Algunos ejemplos incluyen una aplicación nativa de escritorio o móvil y una aplicación web del lado del cliente basada en JavaScript (como una aplicación de una sola página (SPA))." cta="Ver glosario" href="/docs/es/glossary?term=public+clients">clientes públicos</Tooltip> y no pueden almacenar información confidencial de forma segura (como un **<Tooltip tip="Client Secret: secreto utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">Client Secret</Tooltip>**), deben obtener tokens de la Management API desde el frontend, a diferencia de otros [tipos de aplicación](/docs/es/get-started/applications). Esto significa que los tokens de la Management API para las SPA tienen ciertas limitaciones. En concreto, se emiten en el contexto del usuario que tiene una sesión iniciada en Auth0 en ese momento, lo que limita las actualizaciones únicamente a los datos del usuario autenticado. Aunque esto restringe el uso de la Management API, aún puede utilizarse para realizar acciones relacionadas con la actualización del perfil de usuario del usuario autenticado.

<Warning>
  Auth0 no recomienda exponer en el frontend tokens de la Management API que permitan a los usuarios cambiar sus metadatos. Esto puede permitir que los usuarios manipulen sus propios metadatos de una forma que podría perjudicar el funcionamiento de las aplicaciones. También permite que un cliente lance un ataque DoS contra la Management API de otra persona simplemente saturándola y alcanzando los límites de solicitudes.
</Warning>

<div id="available-scopes-and-endpoints">
  ## Scopes y endpoints disponibles
</div>

Con un token de la Management API emitido para una SPA, puedes acceder a los siguientes scopes (y, por lo tanto, endpoints).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No es posible cambiar la contraseña mediante el endpoint [PATCH /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id) con un token de la Management API emitido para una SPA.
</Callout>

| Scope del usuario actual                 | Endpoint                                                                                                                                                                                                                                                 |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `read:current_user`                      | [GET /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/get_users_by_id)<br />[GET /api/v2/users/{id}/enrollments](https://auth0.com/docs/api/management/v2#!/Users/get_enrollments)                                                   |
| `update:current_user_identities`         | [POST/api/v2/users/{id}/identities](https://auth0.com/docs/api/management/v2#!/Users/post_identities)<br />[DELETE /api/v2/users/{id}/identities/{provider}/{user_id}](https://auth0.com/docs/api/management/v2#!/Users/delete_user_identity_by_user_id) |
| `update:current_user_metadata`           | [PATCH /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id)                                                                                                                                                           |
| `create:current_user_metadata`           | [PATCH /api/v2/users/{id}](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id)                                                                                                                                                           |
| `create:current_user_device_credentials` | [POST /api/v2/device-credentials](https://auth0.com/docs/api/management/v2#!/Device_Credentials/post_device_credentials)                                                                                                                                 |
| `delete:current_user_device_credentials` | [DELETE /api/v2/device-credentials/{id}](https://auth0.com/docs/api/management/v2#!/Device_Credentials/delete_device_credentials_by_id)                                                                                                                  |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los scopes y endpoints anteriores están sujetos a [límites de tasa](/docs/es/troubleshoot/customer-support/operational-policies/rate-limit-policy).
</Callout>

<div id="use-management-api-token-to-call-management-api-from-a-spa">
  ## Usa un token de la Management API para llamar a la Management API desde una SPA
</div>

Puedes obtener un token de la Management API desde una SPA (usando la <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token, y su valor contiene el ID de una aplicación (Client ID) para un token de ID o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audience</Tooltip> de la Management API para generarlo) y usarlo para llamar a la Management API y recuperar el perfil de usuario completo del usuario que ha iniciado sesión.

1. [Obtén un token de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).

   1. Autentica al usuario redirigiéndolo al endpoint de Authorization, que es adonde se dirige a los usuarios cuando inician sesión o se registran.
   2. Cuando recibas el token de la Management API, estará en [formato JSON Web Token](/docs/es/secure/tokens/json-web-tokens/json-web-token-structure).
   3. Decodifícalo y revisa su contenido.
2. Llama a la Management API para recuperar el perfil de usuario del usuario autenticado desde el endpoint [Get User by ID](https://auth0.com/docs/api/management/v2#!/Users/get_users_by_id).

   1. Para llamar al endpoint, incluye el token codificado de la Management API que obtuviste en el encabezado `Authorization` de la solicitud.
   2. Asegúrate de reemplazar los marcadores de posición `USER_ID` y `MGMT_API_ACCESS_TOKEN` por el ID del usuario autenticado (el valor `sub` del token decodificado de la Management API) y el token de acceso de la Management API, respectivamente.
