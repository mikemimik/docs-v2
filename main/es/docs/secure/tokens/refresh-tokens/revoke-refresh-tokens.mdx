Puedes revocar <Tooltip tip="Token de actualización: token que se usa para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> si se ven comprometidos. Auth0 trata la revocación de tokens como si el token pudiera haber quedado expuesto a actores maliciosos.

Además, puedes considerar habilitar la [rotación de tokens de actualización](/docs/es/secure/tokens/refresh-tokens/refresh-token-rotation) para que, cada vez que un cliente intercambie un token de actualización para obtener un nuevo token de acceso, también reciba un nuevo token de actualización.

<div id="choose-whether-token-revocation-deletes-grants">
  ## Elija si la revocación de tokens elimina los grants
</div>

Un grant proporciona a una aplicación acceso a un recurso de otra entidad sin exponer las credenciales del usuario. Los tokens se emiten en el contexto de un grant.

Cuando se revoca un grant, también se revocan todos los tokens emitidos en el contexto de ese grant. Sin embargo, cuando se revoca un token, no necesariamente debe revocarse también el grant.

Puede elegir si la revocación de tokens elimina el grant asociado en la configuración de su tenant:

1. Vaya a [Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced).

2. En la sección **Settings**, active o desactive el interruptor **Refresh Token Revocation Deletes Grant**.

   * Active el interruptor para eliminar el grant subyacente cuando revoque el token de actualización.

   Cada solicitud de revocación invalida todos los demás tokens basados en el mismo grant de autorización, lo que revoca todos los tokens de actualización que se hayan emitido para el mismo usuario, aplicación y audiencia.

   * Desactive el interruptor para conservar el grant subyacente cuando revoque el token de actualización.

   Cuando se desvincula un dispositivo, solo se revoca el token de actualización asociado, y el grant permanece intacto.

Esta función está desactivada de forma predeterminada para todos los tenants creados después del 13 de enero de 2021.

<div id="revoke-a-refresh-token">
  ## Revocar un token de actualización
</div>

Puedes revocar un token de actualización de las siguientes formas:

* [Con el Auth0 Dashboard](#using-the-auth0-dashboard)
* [Con la Authentication API](#using-the-authenticaton-api)
* [Con la Management API](#using-the-management-api)

<div id="using-the-auth0-dashboard">
  ### Uso de Auth0 Dashboard
</div>

Puedes usar Auth0 Dashboard para revocar el acceso autorizado de un usuario a la aplicación que emitió el token. Esto invalida el token de actualización, lo cual es funcionalmente idéntico a revocar el propio token.

1. Ve a [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y selecciona el nombre del usuario que quieres ver.
2. Selecciona la pestaña **Authorized Applications**. Esta página muestra todas las aplicaciones a las que el usuario ha autorizado el acceso.
3. Para revocar el acceso del usuario a una aplicación autorizada y, de este modo, invalidar el token de actualización, haz clic en **Revoke**.

<div id="using-the-authentication-api">
  ### Uso de la Authentication API
</div>

Puedes usar el [endpoint Revocar un token de actualización](https://auth0.com/docs/api/authentication/revoke-refresh-token/revoke-refresh-token) de la Authentication API para revocar un token de actualización.

El endpoint admite el acceso sin el <Tooltip tip="Secreto del cliente: secreto que usa un cliente (aplicación) para autenticarse con el servidor de autorización; solo deberían conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto del cliente</Tooltip> en aplicaciones que no pueden mantenerlo seguro (como las aplicaciones nativas). La propia aplicación debe tener su [método de autenticación](/docs/es/get-started/applications/credentials) (`tokenEndpointAuthMethod`) configurado como `none`.

Cuando haces una solicitud, la API primero valida las credenciales de la aplicación y luego verifica si el token se emitió para la aplicación que realiza la solicitud de revocación. Si esta validación falla, la solicitud se rechaza y se informa a la aplicación del error.

A continuación, la API invalida el token. La invalidación se produce de inmediato y el token no puede volver a usarse después de la revocación. Cada solicitud de revocación invalida todos los tokens que se han emitido para el mismo grant de autorización.

Para obtener más información, incluidos los parámetros del cuerpo, los esquemas de respuesta y los ejemplos de código, consulta la documentación de la Authentication API del [endpoint Revocar un token de actualización](https://auth0.com/docs/api/authentication/revoke-refresh-token/revoke-refresh-token).

<div id="using-the-management-api-device-credentials-endpoints">
  ### Uso de los endpoints de credenciales de dispositivo de la Management API
</div>

Para revocar un token de actualización con los endpoints de credenciales de dispositivo de la Management API de Auth0:

1. Obtén el `id` del token de actualización que quieres revocar mediante el [endpoint para recuperar credenciales de dispositivo](https://auth0.com/docs/api/management/v2/device-credentials/get-device-credentials).

   Especifica `type=refresh_token` y un `user_id`, con un token de acceso que contenga el scope `read:device_credentials`. Para acotar los resultados, también puedes especificar el `client_id` asociado al token (si lo conoces).

2. Revoca el token de actualización mediante el [endpoint para eliminar una credencial de dispositivo](https://auth0.com/docs/api/management/v2/device-credentials/delete-device-credentials-by-id). Especifica un token de acceso que contenga el scope `delete:device_credentials` y el ID del token de actualización.

<div id="using-the-management-api-refresh-token-endpoints">
  ### Uso de los endpoints de token de actualización de la Management API
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los endpoints de token de actualización de la Auth0 Management API actualmente solo están disponibles para clientes con planes Enterprise. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Callout>

Los endpoints de token de actualización de la Management API sustituyen a los endpoints del recurso `/v2/device-credentials` y añaden propiedades ampliadas y operaciones de revocación masiva.

Puedes ver o revocar un token de actualización específico con los siguientes endpoints:

* Ver la información de un token de actualización por ID con el [endpoint Get a refresh token](https://auth0.com/docs/api/management/v2/refresh-tokens/get-refresh-token)
* Revocar un token de actualización por ID con el [endpoint Delete a refresh token](https://auth0.com/docs/api/management/v2/refresh-tokens/delete-refresh-token)
* Ver todos los tokens de actualización de un usuario con el [endpoint Get refresh tokens for a user](https://auth0.com/docs/api/management/v2/users/get-refresh-tokens-for-user)
* Revocar todos los tokens de actualización de un usuario con el [endpoint Delete refresh tokens for a user](https://auth0.com/docs/api/management/v2/users/delete-refresh-tokens-for-user)

<div id="considerations-and-limitations">
  ## Consideraciones y limitaciones
</div>

* Con el [Device Authorization Flow](/docs/es/get-started/authentication-and-authorization-flow/device-authorization-flow), la única forma de forzar que un dispositivo vuelva a autorizarse es revocar el token de actualización asignado al dispositivo. No se forzará al dispositivo a volver a autorizarse hasta que el token de acceso actual caduque y la application intente usar el token de actualización revocado. Para obtener más información, lee [Desvincular dispositivos de los usuarios](/docs/es/manage-users/user-accounts/unlink-devices-from-users).

* Al usar [Refresh Token Rotation](/docs/es/secure/tokens/refresh-tokens/refresh-token-rotation), si se usa un token previamente invalidado, se revocará de inmediato todo el conjunto de tokens de actualización emitidos desde que se emitió ese token invalidado, lo que obligará al usuario final a autenticarse de nuevo.

* Las operaciones de revocación (`DELETE`) de tokens de actualización se ejecutan de forma asíncrona y tienen consistencia eventual.

* Los tokens de actualización emitidos a partir del 21-09-2023 (22-02-2024 para los tenants de la región US-3) contienen la propiedad de ID de sesión (`session_id`) con el valor correspondiente. Los tokens de actualización emitidos antes de esta fecha contienen esta propiedad con un valor `null`.

Para obtener más información sobre las limitaciones de los tokens de actualización, lee [limitaciones de los tokens de actualización](/docs/es/secure/tokens/refresh-tokens).
