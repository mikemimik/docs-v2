title: Validar tokens de ID
---

Un [token de ID](/es/docs/secure/tokens/id-tokens), que contiene atributos del perfil del usuario, lo consume una aplicación y normalmente se utiliza para mostrar la interfaz de usuario. Auth0 emite todos los <Tooltip tip="Token de ID: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+tokens">tokens de ID</Tooltip> en formato [JSON web token (JWT)](/es/docs/secure/tokens/json-web-tokens).

Si alguna de estas comprobaciones falla, el token se considera no válido y la solicitud debe rechazarse.

1. [Valida el JWT](/es/docs/secure/tokens/json-web-tokens/validate-json-web-tokens).
2. Comprueba otras claims estándar. Si ya realizaste la validación estándar del JWT, ya habrás decodificado el [payload del JWT](/es/docs/secure/tokens/json-web-tokens/json-web-token-structure) y revisado sus claims estándar. Entre las claims adicionales que debes verificar en los tokens de ID se incluyen:

   * **Audiencia del token** (`aud`, string): El valor de audience del token debe coincidir con el ID de cliente de la aplicación, tal como se define en la [configuración de la aplicación](https://manage.auth0.com/#/applications), en el campo **Client ID**.
   * **Nonce** (`nonce`, string): Se recomienda incluir un nonce en la solicitud del token (es obligatorio para el flujo implícito) para ayudar a prevenir ataques de repetición. El valor de nonce del token debe coincidir exactamente con el nonce original enviado en la solicitud. Consulta [Mitigar ataques de repetición](/es/docs/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow) para obtener más información.
