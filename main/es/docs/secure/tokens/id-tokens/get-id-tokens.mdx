description: Describe cómo solicitar un token de ID al autenticar a los usuarios para que incluya claims sobre el usuario mediante la inclusión de scopes de OIDC.
title: Obtener tokens de ID
---

Para obtener un <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+token">token de ID</Tooltip>, debes solicitarlo al autenticar a los usuarios. Auth0 facilita que tu aplicación autentique a los usuarios mediante:

* [Quickstarts](/es/docs/quickstarts): La forma más sencilla de implementar la authentication, que puede mostrarte cómo usar [Universal Login](/es/docs/authenticate/login/auth0-universal-login), el widget Lock y los SDK de Auth0 específicos para cada lenguaje y framework. Nuestra [documentación de Lock](/es/docs/libraries/lock) y la [documentación de Auth0.js](/es/docs/libraries/auth0js) ofrecen información detallada sobre cómo recuperar un token de ID después de la autenticación.
* [Authentication API](https://auth0.com/docs/api/authentication): Si prefieres crear tu propia implementación, puedes llamar directamente a nuestra API. Primero, debes saber [qué flujo usar](/es/docs/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use) antes de seguir el tutorial del flujo correspondiente.

<div id="control-id-token-contents">
  ## Controla el contenido del token de ID
</div>

Puedes controlar qué claims sobre el usuario autenticado se incluyen en el token de ID que consume tu aplicación incluyendo [scopes específicos de OpenID Connect](/es/docs/get-started/apis/scopes/openid-connect-scopes) en el parámetro `scope` al solicitar tokens durante la autenticación de los usuarios.

También puedes crear [claims personalizados](/es/docs/secure/tokens/json-web-tokens/json-web-token-claims), es decir, claims que defines, controlas y agregas a un token mediante una rule. Al igual que con otros <Tooltip tip="JSON Web Token (JWT): formato estándar del token de ID (y, a menudo, también del token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=JWTs">JWTs</Tooltip>, debes [validar un token de ID](/es/docs/secure/tokens/id-tokens/validate-id-tokens) antes de asumir que se puede confiar en su contenido.

<div id="renew-id-tokens">
  ## Renovar tokens de ID
</div>

De forma predeterminada, un token de ID es válido durante 36000 segundos (10 horas). Si existen problemas de seguridad, puedes [acortar el tiempo de validez del token](/es/docs/secure/tokens/id-tokens/update-id-token-lifetime), pero recuerda que uno de los propósitos de este token es mejorar el rendimiento al almacenar en caché la información del usuario.

Una vez que un token de ID haya caducado, quizá quieras renovarlo. Para renovar el token de ID, puedes volver a autenticar al usuario con Auth0 o usar un [token de actualización](/es/docs/secure/tokens/refresh-tokens).
