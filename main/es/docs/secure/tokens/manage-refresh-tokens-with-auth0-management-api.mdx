<div id="management-api-endpoints">
  ## Endpoints de la Management API
</div>

La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0 proporciona endpoints para gestionar tokens de actualización de forma individual o en conjunto.

Estos endpoints complementan la [revocación alternativa de tokens de actualización](/es/docs/secure/tokens/refresh-tokens/revoke-refresh-tokens) y reemplazan los endpoints del recurso `device-credentials` con propiedades ampliadas y operaciones de revocación masiva.

<div id="refresh-token-resource">
  ### Recurso de token de actualización
</div>

Puede ver o revocar un token de actualización específico mediante los siguientes endpoints:

| Nombre                                                 | URL                                       | Scopes requeridos       |
| ------------------------------------------------------ | ----------------------------------------- | ----------------------- |
| Ver la información de un token de actualización por ID | `GET /api/v2/refresh-tokens/{tokenId}`    | `read:refresh_tokens`   |
| Revocar un token de actualización por ID               | `DELETE /api/v2/refresh-tokens/{tokenId}` | `delete:refresh_tokens` |

<div id="user-resource">
  ### Recurso del usuario
</div>

Puedes listar o revocar todos los tokens de actualización de un usuario específico con los siguientes endpoints:

| Nombre                                                  | URL                                            | scope(s) requeridos     |
| ------------------------------------------------------- | ---------------------------------------------- | ----------------------- |
| Listar todos los tokens de actualización de un usuario  | `GET /api/v2/users/{userId}/refresh-tokens`    | `read:refresh_tokens`   |
| Revocar todos los tokens de actualización de un usuario | `DELETE /api/v2/users/{userId}/refresh-tokens` | `delete:refresh_tokens` |

<div id="refresh-token-properties">
  ## Propiedades del token de actualización
</div>

Los <Tooltip tip="Endpoint de token: endpoint del servidor de autorización que se usa para solicitar tokens de forma programática." cta="Ver glosario" href="/es/docs/glossary?term=token+endpoints">endpoints de token</Tooltip> de actualización devuelven información relevante sobre el token y su historial.

| Campo                                                   | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID del token de actualización                           | Un identificador único y específico del tenant del token de actualización en la API. No es el token en sí, que es un secreto que se intercambia con el endpoint de token de la Authentication API.<br /><br />Los tokens de actualización pueden durar más que una sesión. El token de actualización conserva esta información independientemente del ciclo de vida de la sesión.                                                                                                                                                                                                                                                                 |
| ID de sesión                                            | El ID de sesión es un identificador persistente de la sesión en el tenant de Auth0. Ten en cuenta que el ID de sesión corresponde al claim `sid` que ya está presente en los tokens de ID y los tokens de cierre de sesión, y puede usarse para establecer referencias cruzadas entre estas entidades.<br /><br />El token de actualización contiene referencias a la sesión que se usó para crear el token. El ID de sesión puede usarse para recuperar información adicional con el endpoint de la Management API para [inspeccionar una sesión específica por ID](/es/docs/manage-users/sessions/manage-user-sessions-with-auth0-management-api). |
| Tiempo relevante                                        | Información sobre la creación y el vencimiento del token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Información del cliente y de los servidores de recursos | Información resumida sobre las aplicaciones y las API asociadas a un token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |

<div id="limitations">
  ## Limitaciones
</div>

* Las operaciones de revocación de token de actualización (`DELETE`) se ejecutan de forma asíncrona y tienen consistencia eventual.
* Los tokens de actualización emitidos el 21-09-2023 o después (22-02-2024 para tenants de la región US-3) contienen la propiedad de ID de sesión (`session_id`) con el valor correspondiente. Los tokens de actualización emitidos antes de esta fecha contienen esta propiedad con un valor `null`.
