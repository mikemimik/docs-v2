Todos los <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de access token) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=JWTs">JWTs</Tooltip> emitidos por Auth0 tienen [JSON Web Signatures (JWSs)](https://tools.ietf.org/html/rfc7515), lo que significa que están firmados en lugar de cifrados. Un JWS representa contenido protegido con <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de access token) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=digital+signatures">firmas digitales</Tooltip> o códigos de autenticación de mensajes (MAC) mediante estructuras de datos basadas en JSON.

Un JWT bien formado consta de tres cadenas concatenadas codificadas en Base64url, separadas por puntos (`.`):

* **encabezado JOSE**: contiene metadata sobre el tipo de token y los algoritmos criptográficos utilizados para proteger su contenido.
* **JWS payload** (conjunto de [claims](https://tools.ietf.org/html/rfc7519#section-4)): contiene afirmaciones de seguridad verificables, como la identidad del usuario y los permisos que tiene concedidos.
* **firma JWS**: se utiliza para validar que el token es fiable y no ha sido manipulado. Cuando uses un JWT, **debes** [verificar su firma](/es/docs/secure/tokens/json-web-tokens/validate-json-web-tokens) antes de almacenarlo y usarlo.

<div id="jose-header">
  ## Encabezado JOSE
</div>

Objeto JSON que contiene los parámetros que describen las operaciones criptográficas y los parámetros utilizados. El encabezado JOSE (JSON Object Signing and Encryption) se compone de un conjunto de parámetros de encabezado que normalmente consisten en un par nombre/valor: el algoritmo de hash utilizado (por ejemplo, HMAC SHA256 o RSA) y el tipo de JWT.

```json lines
{
      "alg": "HS256",
      "typ": "JWT"
    }
```

<div id="jws-payload">
  ## payload
</div>

La carga útil contiene afirmaciones sobre la entidad (normalmente, el usuario) y atributos adicionales de la entidad, que se denominan `claims`. En este ejemplo, nuestra entidad es un usuario.

```json lines
{
      "sub": "1234567890",
      "name": "John Doe",
      "admin": true
    }
```

Al trabajar con claims de JWT, debes tener en cuenta los distintos tipos de claims y las reglas de nomenclatura.

<div id="jws-signature">
  ## Firma JWS
</div>

La firma se utiliza para verificar que el remitente del JWT es quien dice ser y para garantizar que el mensaje no se haya modificado en el camino.

Para crear la firma, se toman el encabezado y el payload codificados en Base64, junto con un Secret, y se firman con el algoritmo especificado en el encabezado.

Por ejemplo, si está creando una firma para un token con el algoritmo HMAC SHA256, haría lo siguiente:

```js lines
HMACSHA256(
      base64UrlEncode(header) + "." +
      base64UrlEncode(payload),
      secret)
```
