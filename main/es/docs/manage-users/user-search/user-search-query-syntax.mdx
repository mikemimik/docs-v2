<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Este contenido describe el User Search actual de Auth0, versión 3.
</Callout>

Al buscar usuarios, puedes crear consultas con la [sintaxis de consultas de Lucene](https://lucene.apache.org/core/2_9_4/queryparsersyntax.html) para afinar la búsqueda.

La cadena de consulta se analiza en una serie de términos y operadores:

* Un término puede ser una sola palabra, como `jane` o `smith`.
* Un término puede ser una frase entre comillas dobles (`"green apple"`), que coincidirá con todas las palabras de la frase en el mismo orden.
* Un término sin nombre de campo no coincidirá con texto en los campos de [metadatos del usuario](/es/docs/manage-users/user-accounts/metadata).
* Se pueden agrupar varios términos entre paréntesis para formar subconsultas.
* Los valores de búsqueda de los campos de usuario normalizados (`email`, `name`, `given_name`, `family_name` y `nickname`) no distinguen entre mayúsculas y minúsculas. Todos los demás campos (incluidos todos los campos de `app_metadata`/`user_metadata`) sí distinguen entre mayúsculas y minúsculas.
* Los operadores (`AND`, `OR`, `NOT`) funcionan en todos los campos de usuario normalizados y en los campos de metadatos de raíz.
* Los operadores siempre deben escribirse en mayúsculas.

<div id="searchable-fields">
  ## Campos de búsqueda
</div>

Puede buscar usuarios usando todos los [campos del perfil de usuario normalizado](/es/docs/manage-users/user-accounts/user-profiles/normalized-user-profile-schema) y los campos que se indican a continuación:

| Campo de búsqueda | Tipo de dato                      | Descripción                                                                                                                                                                                                                                                                                            |
| ----------------- | --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `phone_number`    | texto                             | El número de teléfono del usuario. Solo es válido para usuarios con conexiones SMS.                                                                                                                                                                                                                    |
| `phone_verified`  | booleano                          | El valor `true/false` indica si el número de teléfono del usuario ha sido verificado. Solo es válido para usuarios con conexiones SMS.                                                                                                                                                                 |
| `logins_count`    | entero                            | La cantidad de veces que el usuario ha iniciado sesión. Si un usuario está bloqueado e inicia sesión, la sesión bloqueada se contabiliza en `logins_count` y actualiza el valor de `last_login`.                                                                                                       |
| `created_at`      | fecha y hora                      | La marca de tiempo de cuándo se creó por primera vez el perfil del usuario.                                                                                                                                                                                                                            |
| `updated_at`      | fecha y hora                      | La marca de tiempo de cuándo se actualizó o modificó por última vez el perfil del usuario.                                                                                                                                                                                                             |
| `last_login`      | fecha y hora                      | La marca de tiempo de cuándo el usuario inició sesión por última vez. Si esta propiedad se ejecuta dentro de una [Rule](/es/docs/customize/rules) con el objeto `user`, el valor se asociará con el inicio de sesión que activó la Rule (ya que las Rules se ejecutan después del inicio de sesión real). |
| `last_ip`         | texto (dirección IP válida)       | La dirección IP asociada con el último inicio de sesión del usuario.                                                                                                                                                                                                                                   |
| `blocked`         | booleano                          | El valor `true` o `false` indica si el usuario ha sido bloqueado. Nota: `true` *solo* devuelve usuarios bloqueados a través del Dashboard de Admin y la Management API; no devuelve usuarios bloqueados por la detección de anomalías por fuerza bruta.                                                |
| `email.domain`    | texto                             | La parte del dominio del correo electrónico del usuario.                                                                                                                                                                                                                                               |
| `organization_id` | texto (ID de organización válido) | La organización de la que el usuario es miembro                                                                                                                                                                                                                                                        |

Los campos de metadatos pueden usarse con:

* booleano
* numérico: entero o doble
* texto
* objetos: para buscar un valor escalar anidado dentro de otro objeto, use la ruta al campo. Por ejemplo, `app_metadata.subscription.plan:"gold"`
* arreglos: para buscar campos en objetos anidados dentro de arreglos, use la ruta al campo e ignore el nivel del arreglo. Por ejemplo, `user_metadata.addresses.city:"Paris"`

Los campos de metadatos que contienen un arreglo vacío, un objeto vacío o un valor `null` no se indexan y no se pueden buscar.

Las búsquedas por rango y con comodines no están disponibles en los campos `user_metadata`.

<div id="exact-match">
  ## Coincidencia exacta
</div>

Para encontrar coincidencias exactas, utiliza comillas dobles: `name:"jane smith"`.

Por ejemplo, para encontrar usuarios con el nombre `jane smith`, usa `q=name:"jane smith"`:

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3' \
    --header 'authorization: Bearer {yourMgmtApiAccessToken}'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse<String> response = Unirest.get("https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3")
    .header("authorization", "Bearer {yourMgmtApiAccessToken}")
    .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'GET',
    url: 'https://{yourDomain}/api/v2/users',
    params: {q: 'name:"jane smith"', search_engine: 'v3'},
    headers: {authorization: 'Bearer {yourMgmtApiAccessToken}'}
  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP
  $curl = curl_init();

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_ENCODING => "",
    CURLOPT_MAXREDIRS => 10,
    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
    CURLOPT_CUSTOMREQUEST => "GET",
    CURLOPT_HTTPHEADER => [
      "authorization: Bearer {yourMgmtApiAccessToken}"
    ],
  ]);

  $response = curl_exec($curl);
  $err = curl_error($curl);

  curl_close($curl);

  if ($err) {
    echo "cURL Error #:" . $err;
  } else {
    echo $response;
  }
  ```

  ```python Python
  import http.client

  conn = http.client.HTTPSConnection("")

  headers = { 'authorization': "Bearer {yourMgmtApiAccessToken}" }

  conn.request("GET", "/{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3", headers=headers)

  res = conn.getresponse()
  data = res.read()

  print(data.decode("utf-8"))
  ```

  ```ruby Ruby
  require 'uri'
  require 'net/http'
  require 'openssl'

  url = URI("https://{yourDomain}/api/v2/users?q=name%3A%22jane%20smith%22&search_engine=v3")

  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Get.new(url)
  request["authorization"] = 'Bearer {yourMgmtApiAccessToken}'

  response = http.request(request)
  puts response.read_body
  ```
</AuthCodeGroup>

<div id="wildcards">
  ## Comodines
</div>

Las búsquedas con comodines pueden realizarse sobre términos utilizando el carácter de asterisco (`*`) para reemplazar cero o más caracteres. Las búsquedas con comodines no están disponibles en los campos `user_metadata`.

<div id="examples">
  #### Ejemplos
</div>

* `name:john*` devuelve todos los usuarios con `john` al comienzo de su nombre.
* `name:j*` devuelve todos los usuarios con `j` al comienzo de su nombre.
* `q=name:john*` devuelve todos los usuarios cuyos nombres empiezan por `john`.
* Para la coincidencia por sufijo, los literales deben tener 3 caracteres o más. Por ejemplo, `name:*usa` está permitido, pero `name:*sa` no.

<div id="ranges">
  ## Rangos
</div>

Puedes usar rangos en tus consultas de User Search. Las búsquedas por rango no están disponibles en los campos de metadatos de usuario.

* Para rangos inclusivos, usa corchetes: `[min TO max]`.
* Para rangos excluyentes, usa llaves: `{min TO max}`.
* Se pueden combinar llaves y corchetes en la misma expresión de rango: `logins_count:[100 TO 200}`.
* Usa rangos en combinación con comodines. Por ejemplo, para encontrar todos los usuarios con más de 100 inicios de sesión, usa `q=logins_count:{100 TO *]`.

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/users?q=logins_count%3A%7B100%20TO%20*%5D&search_engine=v3' \
    --header 'authorization: Bearer {yourMgmtApiAccessToken}'
  ```

<div id="searchable-profile-attribute-examples">
  ## Ejemplos de atributos de perfil que se pueden buscar
</div>

Al buscar usuarios en la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0, puedes filtrarlos por `user_metadata` o `app_metadata`. Para ello, puedes usar la sintaxis de búsqueda de Lucene con el parámetro `q`.

Dado que el endpoint de Auth0 Management API para [listar o buscar usuarios](https://auth0.com/docs/api/management/v2#!/Users/get_users) está limitado a 1000 resultados (10 páginas de 100 registros), el filtrado es una forma útil de garantizar que se devuelvan los resultados más relevantes.

A continuación se muestra un ejemplo de `user_metadata` en un perfil de usuario:

```json lines
{
  "favorite_color": "blue",
  "approved": false,
  "preferredLanguage": "en",
  "preferences": {
    "fontSize": 13
  },
  "addresses":{
    "city":["Paris","Seattle"]
  }
}
```

<div id="filter-metadata-attributes">
  ### Filtrar atributos de metadatos
</div>

Para devolver un valor de `user_metadata`, actualiza la consulta `q` con un filtro por ese atributo.

Para los valores de `user_metadata`, puedes consultar el perfil directamente:

`q: _exists_:user_metadata.fav_color`

Esta consulta devuelve todos los perfiles de usuario que tienen el atributo `fav_color` en `user_metadata`.

<div id="filter-metadata-nested-object-attributes-and-values">
  ### Filtrar atributos y valores de objetos anidados de metadatos
</div>

También puedes buscar en objetos anidados dentro de `user_metadata`:

`q: _exists_:user_metadata.preferences.fontSize`

Esto consulta todos los perfiles de usuario que tienen `preferences.fontSize` configurado en `user_metadata`.

Para buscar los valores de un objeto anidado dentro de otro objeto, revisa la siguiente consulta:

`q: user_metadata.preferences.fontSize:13`

Esta consulta devuelve todos los perfiles de usuario cuyo atributo `fontSize` tiene el valor `13`.

<div id="filter-metadata-nested-array-values">
  ### Filtrar valores de arreglos anidados en metadatos
</div>

Puedes usar la siguiente consulta para buscar campos en arreglos anidados:

`q: user_metadata.addresses.city:"Seattle"`

Esto devuelve todos los perfiles de usuario que tienen el valor `Seattle` en los atributos `address.city` de `user_metadata`.
