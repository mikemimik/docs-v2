Los metadatos de la sesión hacen que los datos de contexto del usuario sean portátiles y visibles a lo largo de todo el ciclo de vida de la sesión y en los eventos de cierre de sesión.

Los sistemas de destino pueden usar la información de los metadatos de la sesión para realizar auditorías, hacer análisis y aplicar flujos de revocación, entre otros usos.

<Warning>
  Los metadatos de la sesión de Auth0 no son un almacén de datos seguro y no deben usarse para almacenar información sensible. Esto incluye secretos y PII de alto riesgo, como números de la seguridad social o de tarjetas de crédito. Se recomienda encarecidamente a los clientes de Auth0 que evalúen los datos almacenados en los metadatos y guarden únicamente los necesarios para fines de gestión de identidades y accesos. Para obtener más información, consulta [Cumplimiento del RGPD de Auth0](/docs/es/secure/data-privacy-and-compliance/gdpr).
</Warning>

<div id="add-organization-information-to-session-metadata">
  ## Agregar información de la organización a los metadatos de la sesión
</div>

Puedes usar [Actions](/docs/es/customize/actions) para almacenar los identificadores de la organización en los metadatos de una sesión con el método `api.session.setMetadata()` de [Post Login](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) y consultarlos mediante el objeto `event.session.metadata`.

Código de la Action de Post Login:

```javascript lines
/**
 * Post-Login Action (simple)
 * Agrega el contexto de organización a los metadatos de la sesión para que aparezca en Actions posteriores,
 * la Management API y (si está habilitado) el token de cierre de sesión por back-channel.
 */
exports.onExecutePostLogin = async (event, api) => {
  // Continuar solo si la transacción tiene como destino una organización
  if (!event.organization) return;

  // Mantener los valores cortos y solo de tipo string (los metadatos de la sesión requieren strings)
  const orgId = String(event.organization.id || "");
  const orgSlug = String(event.organization.name || "");
  const orgDisplay = String(event.organization.display_name || orgSlug);

  // Escrituras mínimas e idempotentes (solo algunas claves para no superar los límites)
  api.session.setMetadata("org_id", orgId);
  api.session.setMetadata("org_slug", orgSlug);
  api.session.setMetadata("org_name", orgDisplay);
};
```

Los metadatos de la sesión están disponibles para las Actions siguientes, pueden recuperarse mediante la [Management API](https://auth0.com/docs/api/management/v2) y pueden incluirse en el token de [OpenID Connect Back-Channel Logout](/docs/es/authenticate/login/logout/back-channel-logout).

* En las Actions siguientes, puedes consultar los datos mediante el objeto `event.session.metadata`:

```javascript lines
const orgId = event.session.metadata?.org_id;
```

* Si utilizas la Management API, puedes consultar los datos mediante el endpoint [/api/v2/sessions/{id}](https://auth0.com/docs/api/management/v2/sessions/get-session):

```bash lines
GET /api/v2/sessions/{id}
```

Ejemplo de respuesta:

```json lines
{
  "session_metadata": {
    "org_id": "org_abc123",
    "org_slug": "acme",
    "org_name": "Acme Corp"
  }
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las llamadas al endpoint `/api/v2/sessions/{id}` requieren un [token de acceso de la Management API](/docs/es/secure/tokens/access-tokens) con el `scope` `update:session`.
</Callout>

* Si tiene una aplicación configurada con [metadatos de la sesión para cierre de sesión OIDC por canal secundario](/docs/es/manage-users/sessions/session-metadata/configure-session-metadata), el `logout_token` incluye los metadatos de la sesión:

```json lines
{
  "events": { "http://schemas.openid.net/event/backchannel-logout": {} },
  "session_metadata": {
    "org_id": "org_abc123",
    "org_slug": "acme",
    "org_name": "Acme Corp"
  }
}
```
