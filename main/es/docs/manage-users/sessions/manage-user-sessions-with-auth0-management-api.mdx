Auth0 usa [sesiones](/docs/es/manage-users/sessions) para mantener el estado de autenticación de un usuario entre solicitudes.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los endpoints de gestión de sesiones de Auth0 Management API están disponibles para clientes con planes Enterprise. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Callout>

<div id="management-api-endpoints">
  ## Endpoints de Management API
</div>

La [Management API](https://auth0.com/docs/api/management/v2) ofrece acceso fuera de banda a los aspectos internos de las sesiones de usuario en la [Capa de sesión de Auth0](/docs/es/manage-users/sessions/session-layers), así como métodos de eliminación para forzar la finalización de las sesiones.

<div id="session-resource">
  ### Recurso de sesión
</div>

Puedes consultar o eliminar una sesión específica mediante los siguientes endpoints:

| Nombre                                 | URL                                   | scope(s) requerido(s) |
| -------------------------------------- | ------------------------------------- | --------------------- |
| Consultar una sesión específica por ID | `GET /api/v2/sessions/{sessionId}`    | `read:sessions`       |
| Eliminar una sesión específica por ID  | `DELETE /api/v2/sessions/{sessionId}` | `delete:sessions`     |

<div id="user-resource">
  ### Recurso del usuario
</div>

Puedes listar o eliminar todas las sesiones de un usuario determinado con los siguientes endpoints:

| Nombre                                            | URL                                      | scopes requeridos |
| ------------------------------------------------- | ---------------------------------------- | ----------------- |
| Listar los detalles de las sesiones de un usuario | `GET /api/v2/users/{userId}/sessions`    | `read:sessions`   |
| Eliminar todas las sesiones del usuario           | `DELETE /api/v2/users/{userId}/sessions` | `delete:sessions` |

<div id="session-properties">
  ## Propiedades de la sesión
</div>

Los endpoints de la sesión devuelven información relevante sobre la sesión y su historial.

| Campo                        | Descripción                                                                                                                                                                                                                                                                                     |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID de sesión                 | El ID de sesión es un identificador persistente de la sesión en el tenant de Auth0.<br /><br />Ten en cuenta que el ID de sesión corresponde al claim `sid`, que ya está presente en los token de ID y los tokens de cierre de sesión, y puede usarse para relacionar estas entidades entre sí. |
| Tiempo relevante             | Información sobre la creación de la sesión, la hora de autenticación y su vencimiento.                                                                                                                                                                                                          |
| Información del dispositivo  | La propiedad del dispositivo muestra detalles relacionados con el agente de usuario (por ejemplo, el navegador) utilizado en las interacciones con esta sesión de Auth0.                                                                                                                        |
| Información de autenticación | Contiene información resumida sobre los métodos utilizados para autenticarse en esta sesión.                                                                                                                                                                                                    |

Para obtener información detallada sobre estos campos, consulta la [documentación de la Management API](https://auth0.com/docs/api/management/v2).

<div id="oidc-back-channel-logout-initiators">
  ## Iniciadores del cierre de sesión OIDC por canal secundario
</div>

Los eventos de eliminación de sesión están vinculados al [cierre de sesión OIDC por canal secundario](/docs/es/authenticate/login/logout/back-channel-logout) a través del iniciador `session-deleted`. Para obtener más información, consulta [Iniciadores del cierre de sesión OIDC por canal secundario](/docs/es/authenticate/login/logout/back-channel-logout/oidc-back-channel-logout-initiators).

<div id="sessions-and-refresh-tokens">
  ## Sesiones y tokens de actualización
</div>

Las sesiones y los <Tooltip tip="token de actualización: Token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> trabajan en conjunto para reducir la fricción en la autenticación de los usuarios y optimizar la seguridad. Para obtener más información, consulta [Prácticas recomendadas para la gestión de sesiones de aplicaciones en el blog de Auth0](https://auth0.com/blog/application-session-management-best-practices).

Los tokens de actualización pueden permanecer activos después de que una sesión haya expirado o se haya eliminado, o después de que el usuario cierre sesión. Puedes usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> para [gestionar tokens de actualización](/docs/es/secure/tokens/manage-refresh-tokens-with-auth0-management-api) de forma independiente o junto con las sesiones.

<div id="limitations">
  ## Limitaciones
</div>

Las operaciones de Delete session se ejecutan de forma asíncrona y presentan consistencia eventual.
