Auth0 te permite asignar y quitar [roles](/es/docs/manage-users/access-control/rbac) a grupos empresariales aprovisionados con <Tooltip tip="System for Cross-domain Identity Management (SCIM): Un protocolo estandarizado para aprovisionar y administrar identidades y grupos de usuarios en distintos sistemas." cta="Ver glosario" href="/es/docs/glossary?term=SCIM">SCIM</Tooltip> usando el Auth0 Dashboard o Management API. Cuando se asignan roles a un grupo, todos los miembros heredan esos roles al iniciar sesión. Puedes administrar los roles de los grupos en dos ámbitos:

* **Grupos de organización** — Grupos empresariales dentro de una [Organización](/es/docs/manage-users/organizations). Requiere que [membresía automática](/es/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) esté habilitado en la conexión empresarial para que la herencia de roles surta efecto.
* **Grupos empresariales** — Grupos empresariales con alcance en tu tenant de Auth0.

<div id="assign-roles">
  ## Asignar roles
</div>

<div id="organization-groups">
  ### Grupos de organización
</div>

Asigna roles a Grupos empresariales aprovisionados mediante SCIM dentro de una organización. Los Grupos empresariales se sincronizan desde un proveedor de identidad empresarial (como Okta o Microsoft Entra ID) a través de SCIM. Cuando un miembro de la organización inicia sesión, hereda los roles asignados a los Grupos empresariales a los que pertenece.

La conexión empresarial de la organización debe tener habilitada la [membresía automática](/es/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) para que la herencia de roles de grupo surta efecto. Para obtener más información sobre cómo se aprovisionan los Grupos empresariales, consulta [Configurar SCIM entrante](/es/docs/authenticate/protocols/scim/configure-inbound-scim#use-groups-with-organizational-roles).

<Tabs>
  <Tab title="Dashboard">
    1. Ve a [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización.
    2. Selecciona la pestaña **Groups**.
    3. Selecciona el grupo al que quieras asignar roles.
    4. Selecciona **Assign Roles**.
    5. Busca y selecciona los roles que quieras asignar y, luego, selecciona **Assign**.

    Los roles asignados aparecen en los detalles del grupo.
  </Tab>

  <Tab title="Management API">
    Haz una llamada `POST` al endpoint [Assign Organization Group Roles](/es/docs/api/management/v2). Necesitas la siguiente información:

    | Parámetro                    | Descripción                                                                                                                                         |
    | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `org_id`                     | ID de la organización.                                                                                                                              |
    | `group_id`                   | ID del Grupo empresarial.                                                                                                                           |
    | `role_id`                    | ID del rol de la organización que se va a asignar. Repítelo en la matriz para asignar varios roles a la vez.                                        |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organization_group_roles`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/{ORG_ID}/groups/{GROUP_ID}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Si la solicitud se realiza correctamente, devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Grupos empresariales
</div>

Asigna roles a los Grupos empresariales aprovisionados mediante SCIM. Cuando se asigna un rol a un grupo, todos los miembros de ese grupo heredan el rol asignado al iniciar sesión.

Para obtener más información sobre cómo se aprovisionan los Grupos empresariales desde un proveedor de identidad empresarial, consulta [configurar SCIM entrante](/es/docs/authenticate/protocols/scim/configure-inbound-scim).

<Tabs>
  <Tab title="Dashboard">
    1. Ve a [Dashboard &gt; User Management &gt; Enterprise Groups](https://manage.auth0.com/#/groups).
    2. Selecciona el grupo al que quieras asignar roles.
    3. Selecciona la pestaña **Roles** y, después, **Assign Roles**.
    4. Busca y selecciona los roles que quieras asignar y, después, selecciona **Assign**.

    Los roles asignados aparecen en la pestaña **Roles** del grupo.
  </Tab>

  <Tab title="Management API">
    Haz una llamada `POST` al endpoint [Assign Roles to a Group](/es/docs/api/management/v2). Necesitas la siguiente información:

    | Parámetro                    | Descripción                                                                                                                          |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
    | `group_id`                   | ID del Grupo empresarial.                                                                                                            |
    | `role_id`                    | ID del rol que se va a asignar. Repítelo en la matriz para asignar varios roles a la vez.                                            |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:groups`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/{group_id}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Si la solicitud se realiza correctamente, devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="remove-roles">
  ## Eliminar roles
</div>

<div id="organization-groups">
  ### Grupos de organización
</div>

Quite roles de los Grupos empresariales aprovisionados mediante SCIM dentro de una organización. Una vez que se elimina un rol, los miembros de la organización de ese grupo dejarán de heredarlo al iniciar sesión.

La conexión empresarial de la organización debe tener habilitada la [membresía automática](/es/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) para que la herencia de roles del grupo surta efecto.

<Tabs>
  <Tab title="Dashboard">
    1. Vaya a [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y seleccione la organización.
    2. Seleccione la pestaña **Groups**.
    3. Seleccione el grupo que desea actualizar.
    4. Busque el rol que desea quitar y luego seleccione el icono de la papelera junto a él.
    5. Confirme la eliminación.

    El rol ya no aparecerá en los detalles del grupo.
  </Tab>

  <Tab title="Management API">
    Haga una llamada `DELETE` al endpoint [Delete Organization Group Roles](/es/docs/api/management/v2). Necesita la siguiente información:

    | Parámetro                    | Descripción                                                                                                                                         |
    | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `org_id`                     | ID de la organización.                                                                                                                              |
    | `group_id`                   | ID del Grupo empresarial.                                                                                                                           |
    | `role_id`                    | ID del rol de la organización que se va a quitar. Repítalo en el array para quitar varios roles a la vez.                                           |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `delete:organization_group_roles`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/ORG_ID/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Una solicitud correcta devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Grupos empresariales
</div>

Elimina roles de los grupos empresariales aprovisionados mediante SCIM. Una vez que se elimina un rol, los miembros del grupo dejarán de heredarlo cuando inicien sesión.

<Tabs>
  <Tab title="Dashboard">
    1. Ve a [Dashboard &gt; User Management &gt; Enterprise Groups](https://manage.auth0.com/#/groups).
    2. Selecciona el grupo que quieras actualizar.
    3. Selecciona la pestaña **Roles**.
    4. Busca el rol que quieras eliminar y, a continuación, selecciona el icono de papelera junto a él.
    5. Confirma la eliminación.

    El rol ya no aparece en la pestaña **Roles** del grupo.
  </Tab>

  <Tab title="Management API">
    Haz una llamada `DELETE` al endpoint [Remove Roles from a Group](/es/docs/api/management/v2). Necesitas la siguiente información:

    | Parámetro                    | Descripción                                                                                                                       |
    | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
    | `group_id`                   | ID del grupo empresarial.                                                                                                         |
    | `role_id`                    | ID del rol que se va a eliminar. Repítelo en el array para eliminar varios roles a la vez.                                        |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso de Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:groups`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Una solicitud correcta devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>
