Puedes asignar [permisos](/es/docs/manage-users/access-control/rbac) a un usuario mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Los permisos asignados pueden usarse con la funcionalidad API Authorization Core.

<Warning>
  Agregar permisos directamente a un usuario evita los beneficios del control de acceso basado en roles (RBAC) y, por lo general, no se recomienda.
</Warning>

<div id="prerequisites">
  ## Requisitos previos
</div>

* Para que el control de acceso basado en roles (RBAC) funcione correctamente, debes habilitarlo para tu API desde el Dashboard o mediante la Management API. La funcionalidad de Authorization Core es distinta de la de Authorization Extension. Para comparar ambas, consulta [Authorization Core vs. Authorization Extension](/es/docs/manage-users/access-control/authorization-core-vs-authorization-extension).
* [Configura una API](/es/docs/get-started/auth0-overview/set-up-apis) en el Dashboard.
* Los permisos se seleccionan de una lista de valores predefinidos. Si tu lista de roles está vacía, asegúrate de [agregar permisos](/es/docs/get-started/apis/add-api-permissions) a tu API.

<div id="dashboard">
  ## Dashboard
</div>

Hay dos formas de asignar un permiso a un usuario. Puedes elegir un usuario de la lista de **Users** y luego asignarle un permiso, o ir a la página **User Details** (perfil de usuario) de un usuario concreto y elegir en la pestaña **permiso** el permiso que quieres asignarle.

<div id="assign-permission-in-user-list">
  ### Asignar permisos en la lista de usuarios
</div>

1. Ve a [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users).
2. Haz clic en `...` junto al usuario que quieres modificar y selecciona **asignar permisos**.
3. Selecciona la API desde la que quieres asignar permisos, luego selecciona los permisos que quieres asignar al usuario y haz clic en **agregar permisos**.

<div id="assign-permissions-in-user-profile">
  ### Asignar permisos en el perfil de usuario
</div>

También puedes asignar permisos a los usuarios desde la página de su perfil individual.

1. Ve a [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y haz clic en el nombre del usuario que quieres ver.
2. Haz clic en la pestaña **Permisos** y, después, en **Asignar permisos**.
3. Selecciona la API desde la que quieres asignar permisos, selecciona los permisos que quieres asignarle al usuario y haz clic en **Agregar permisos**.

<div id="management-api">
  ## Management API
</div>

Haz una solicitud `POST` al [endpoint Assign User Permissions](https://auth0.com/docs/api/management/v2#!/Users/post_permissions). Asegúrate de reemplazar los valores de marcador de posición `USER_ID`, `MGMT_API_ACCESS_TOKEN`, `API_IDENTIFIER` y `PERMISSION_NAME` por tu ID de usuario, el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> de la Management API, los identificadores de la API y los nombres de los permisos, respectivamente.

| Valor                   | Descripción                                                                                                            |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | El ID del usuario al que quieres asignar permisos.                                                                     |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para la Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:users`. |
| `API_IDENTIFIER`        | Identificador(es) de la(s) API asociada(s) a los permisos que quieres asignar al usuario especificado.                 |
| `PERMISSION_NAME`       | Nombre(s) de los permisos que quieres asignar al usuario especificado.                                                 |
