Puedes eliminar los [roles](/es/docs/manage-users/access-control/rbac) asignados a un usuario desde el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Los roles asignados se utilizan con el conjunto de funciones API Authorization Core.

<div id="prerequisites">
  ## Requisitos previos
</div>

Para que el control de acceso basado en roles (RBAC) funcione correctamente, debes habilitarlo para tu API mediante el Dashboard o la Management API. La funcionalidad de Authorization Core es distinta de la de Authorization Extension. Para compararlas, consulta [Authorization Core vs. Authorization Extension](/es/docs/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Dashboard
</div>

1. Ve a [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y haz clic en el nombre del usuario para verlo.
2. Haz clic en la vista **Roles** y, a continuación, en el icono de la papelera junto al rol que quieres eliminar.

<div id="management-api">
  ## Management API
</div>

Realiza una solicitud `DELETE` al [endpoint Delete User Roles](https://auth0.com/docs/api/management/v2#!/Users/delete_user_roles). Asegúrate de reemplazar los valores de marcador de posición `USER_ID`, `MGMT_API_ACCESS_TOKEN` y `ROLE_ID` por tu ID de usuario, tu <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> de la Management API y los ID de rol, respectivamente.

  curl --request DELETE \

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \
    --header 'cache-control: no-cache' \
    --header 'content-type: application/json' \
    --data '{ "roles": [ "ROLE_ID", "ROLE_ID" ] }'

</AuthCodeGroup>

| Valor                   | Descripción                                                                                                            |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `USER_ID`               | El ID del usuario que se va a actualizar.                                                                              |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para la Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:users`. |
| `ROLE_ID`               | ID del rol o de los roles que quieres eliminar del usuario especificado.                                               |
