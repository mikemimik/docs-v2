---
description: Comprende el concepto de las políticas de autorización y cómo se aplican en Auth0.
title: Políticas de autorización
---

Entre bastidores, la autorización basada en roles utiliza una política de autorización preconfigurada que contiene condiciones que permiten al código evaluar si debe permitirse a un usuario acceder a una API protegida.

La política de autorización determina:

* cómo definir y organizar a los usuarios o roles a los que afecta la política
* qué lógica y condiciones se aplican a la política y si su resultado permite o deniega el acceso

Al utilizar la autorización principal de Auth0 y el [control de acceso basado en roles (RBAC)](/docs/es/manage-users/access-control/rbac), la política incluye la evaluación de los roles y permisos asignados a los usuarios. Para usar estas funciones, debes [habilitar el control de acceso basado en roles para las API](/docs/es/get-started/apis/enable-role-based-access-control-for-apis).

Puedes personalizar aún más la política de autorización mediante [Rules](/docs/es/customize/rules). Para obtener más información, consulta [Rules para políticas de autorización](/docs/es/manage-users/access-control/rules-for-authorization-policies).
