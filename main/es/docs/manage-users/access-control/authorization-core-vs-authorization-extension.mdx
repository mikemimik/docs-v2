description: Comprenda las diferencias entre la versión principal de RBAC de Auth0 y Authorization Extension.
title: Authorization Core vs. Authorization Extension
---

<Warning>
  Planeamos dejar obsoletos Rules y Hooks en 2024 y, en ese momento, las Extensions basadas en Rules y Hooks dejarán de tener soporte. Le recomendamos encarecidamente que migre a Actions para ampliar Auth0. Con Actions, tiene acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad.

  Para obtener más información sobre lo que ofrece Actions, lea [Understand How Auth0 Actions Work](/es/docs/customize/actions/actions-overview). También contamos con una página específica de [Move to Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarle en su proceso de migración.

  Para obtener más información sobre la obsolescencia de Rules y Hooks, lea nuestra entrada de blog: [Preparing for Rules and Hooks End of Life.](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/)
</Warning>

Actualmente, Auth0 ofrece dos formas de implementar el [control de acceso basado en roles (RBAC)](/es/docs/manage-users/access-control/rbac): nuestra implementación Core y Authorization Extension. Nuestra implementación Core mejora el rendimiento y la escalabilidad.

Recomendamos usar Authorization Core para la mayoría de las implementaciones. Si busca representar equipos, clientes empresariales o partners en una aplicación B2B o SaaS, le recomendamos representarlos como [Organizations](/es/docs/manage-users/organizations/organizations-overview) y usar Authorization Core. Authorization Extension no es compatible con Organizations.

Para ayudarle a decidir qué funcionalidad es la adecuada para su implementación, a continuación presentamos las diferencias entre ambas:

| Funcionalidad                                                                                                      | Authorization Core                                                                                          | Authorization Extension                                                                                                                                                       |
| ------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Rendimiento y escalabilidad mejorados                                                                              | Sí: lea [Política de límite de entidades](/es/docs/troubleshoot/customer-support/operational-policies/entity-limit-policy) | No: limitado a 500 KB de datos (1000 grupos, 3000 usuarios, donde cada usuario es miembro de 3 grupos; o 20 grupos, 7000 usuarios, donde cada usuario es miembro de 3 grupos) |
| Crear/editar/eliminar Roles                                                                                        | Sí                                                                                                          | Sí                                                                                                                                                                            |
| Los Roles pueden contener permisos de una o varias API                                                             | Sí                                                                                                          | No                                                                                                                                                                            |
| Se pueden asignar Users y Roles a grupos                                                                           | No                                                                                                          | Sí                                                                                                                                                                            |
| Los Roles están asociados a aplicaciones específicas                                                               | No                                                                                                          | Sí                                                                                                                                                                            |
| Crear/editar/eliminar Users                                                                                        | Sí                                                                                                          | Sí                                                                                                                                                                            |
| Buscar Users por usuario, correo electrónico o conexión                                                            | Sí                                                                                                          | Sí                                                                                                                                                                            |
| Buscar Users por proveedor de identidad, número de inicios de sesión, último inicio de sesión o número de teléfono | Sí                                                                                                          | No                                                                                                                                                                            |
| Buscar Users con sintaxis Lucene                                                                                   | Sí                                                                                                          | No                                                                                                                                                                            |
| Importación/exportación de usuarios mediante JSON                                                                  | Por ahora, no                                                                                               | Sí                                                                                                                                                                            |
| Crear políticas de autorización personalizadas                                                                     | Sí                                                                                                          | No                                                                                                                                                                            |
