---
title: Exportar hashes de contraseña y secretos de MFA
description: Aprende a solicitar, a través de un caso de soporte, una exportación cifrada con PGP de hashes de contraseña y secretos de MFA desde tu tenant de Auth0.
---

Cuando solicitas una exportación de hashes de contraseña o secretos de MFA desde tu tenant de Auth0, los datos se cifran de extremo a extremo con Pretty Good Privacy (PGP) antes de salir de los sistemas de Auth0. Solo tú, como titular de la clave privada correspondiente, puedes descifrar los datos exportados.

En esta página se explica todo el proceso, desde el envío de la solicitud hasta el descifrado del archivo de exportación.

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de enviar una solicitud, genera un par de claves PGP que cumpla los siguientes requisitos:

| Requisito                | Valor                                                                                                                                                                                                                    |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Longitud de la clave** | 4096 bits o más                                                                                                                                                                                                          |
| **Subclave de cifrado**  | La clave debe incluir al menos una subclave de cifrado RSA de 4096 bits. Auth0 usa esta subclave para cifrar tu exportación. Las demás subclaves pueden usar parámetros distintos; solo una debe cumplir este requisito. |
| **Frase de paso**        | Segura y única. Se recomienda usar una frase de paso generada aleatoriamente.                                                                                                                                            |
| **Caducidad**            | Establece una fecha de caducidad de al menos 7 días a partir de la fecha de generación. Para reutilizar la misma clave en solicitudes de exportación posteriores, ajusta la caducidad según corresponda.                 |

<div id="generate-your-pgp-key-pair">
  ## Genera tu par de claves PGP
</div>

En esta sección se utiliza [GnuPG (`gpg`)](https://gnupg.org/), la herramienta estándar de línea de comandos para OpenPGP. Para conocer otras herramientas u opciones avanzadas, consulta el [manual de GnuPG](https://www.gnupg.org/gph/en/manual/c14.html).

<div id="install-gnupg">
  ### Instalar GnuPG
</div>

<Tabs>
  <Tab title="macOS">
    ```bash
    brew install gnupg
    ```

    Requiere [Homebrew](https://brew.sh/). Para ver otras opciones de instalación, consulta la [página de descargas de GnuPG](https://gnupg.org/download/).
  </Tab>

  <Tab title="Linux">
    GnuPG viene preinstalado en la mayoría de las distribuciones. Si no es así:

    ```bash
    # Debian/Ubuntu
    sudo apt-get install gnupg

    # RHEL/Fedora
    sudo dnf install gnupg2
    ```
  </Tab>

  <Tab title="Windows">
    Instala [Gpg4win](https://www.gpg4win.org/), que incluye la herramienta de línea de comandos `gpg`. Después de instalarlo, reinicia la terminal para que el comando `gpg` esté en tu `PATH`.
  </Tab>
</Tabs>

<div id="generate-the-key">
  ### Genera la clave
</div>

Ejecuta el comando para generar la clave:

```bash
gpg --full-generate-key
```

Cuando se te solicite, proporciona los siguientes valores:

| Prompt                               | Value                                                                                                                                                                                                           |
| ------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Selecciona qué tipo de clave quieres | **`1` (RSA and RSA)**                                                                                                                                                                                           |
| Qué tamaño de clave quieres          | **`4096`**                                                                                                                                                                                                      |
| La clave es válida durante           | Una duración que coincida con el requisito de **Expiration** anterior (al menos 7 días)                                                                                                                         |
| Nombre real / correo electrónico     | Tu nombre y la dirección de correo electrónico exacta registrada para tu cuenta de administrador del tenant. Los alias o diferencias en el uso de mayúsculas y minúsculas pueden hacer que falle la validación. |
| Comentario                           | Etiqueta opcional no sensible que te ayude a identificar esta clave más adelante (por ejemplo, *&quot;Auth0 export key&quot;*).                                                                                 |
| Frase de contraseña                  | Una frase de contraseña segura, generada aleatoriamente.                                                                                                                                                        |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las versiones recientes de GnuPG usan ECC de forma predeterminada en el primer prompt. **Debes seleccionar explícitamente `1` (RSA and RSA).** Las claves ECC y EDDSA no son compatibles con la validación de exportación de Auth0.
</Callout>

<div id="export-your-public-key">
  ### Exporta tu clave pública
</div>

Una vez generada la clave, busca su ID de la clave y exporta la clave pública. Haz siempre referencia a la clave por su ID de la clave (no por correo electrónico). `gpg` devuelve la primera clave que coincide con el identificador que indicas, así que una búsqueda por correo electrónico puede devolver la clave equivocada si generaste más de una con la misma dirección.

Enumera tus claves:

```bash
gpg --list-keys
```

La salida incluye una línea `pub` seguida del ID de la clave de 16 caracteres:

```text
pub   rsa4096 2026-06-04 [SC] [expires: 2028-06-04]
      ABC123451E45G39A
      uid           [ultimate] Your Name <your.email@example.com>
sub   rsa4096 2026-06-04 [E]
```

Exporta la clave pública mediante el ID de la clave en formato ASCII-armored:

```bash
gpg --armor --export YOUR_KEY_ID > public_key.asc
```

Reemplaza `YOUR_KEY_ID` por la cadena alfanumérica que aparece debajo de la línea `pub` (por ejemplo, `ABC123451E45G39A`). El archivo de salida contiene el bloque de clave completo, incluidas las cabeceras de armadura:

```text
-----BEGIN PGP PUBLIC KEY BLOCK-----
...
-----END PGP PUBLIC KEY BLOCK-----
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No exportes ni compartas tu clave privada. Auth0 solo necesita tu clave pública para cifrar el archivo.
</Callout>

<div id="request-process">
  ## Proceso de solicitud
</div>

<Steps>
  <Step title="Enviar una solicitud de soporte">
    [Abre un caso de soporte](https://support.auth0.com) para solicitar una exportación de hash de contraseña o de secretos de MFA. Incluye lo siguiente en tu solicitud:

    * El **nombre del tenant** específico.
    * Tu **clave pública PGP** (el bloque completo con formato ASCII-armored de la sección anterior).
  </Step>

  <Step title="Esperar la revisión de elegibilidad">
    Auth0 revisa tu solicitud para determinar si cumple los requisitos. No todas las solicitudes califican para una exportación.

    Auth0 no proporciona plazos estimados para las exportaciones aprobadas, ya que su tramitación depende de la disponibilidad de recursos y accesos. Si necesitas que se complete antes de una fecha determinada, indícalo en tu solicitud.
  </Step>

  <Step title="Proporcionar la documentación requerida">
    Si tu solicitud se aprueba, Auth0 te enviará un mensaje con los requisitos y te pedirá lo siguiente:

    1. **Tu autorización**: una confirmación por escrito de que apruebas la exportación desde el tenant o los tenants incluidos en tu solicitud.
    2. **Una confirmación adicional de un administrador del tenant**: al menos otro administrador del tenant deberá ir en copia en el caso de soporte y confirmar la solicitud. Si eres el único administrador del tenant, este paso se omitirá.
    3. **Un formulario de acuse firmado**: el formulario **Hashed Password and MFA Secret Export Acknowledgment** debe estar firmado por un CISO, CSO o un representante de nivel ejecutivo (VP o superior) de tu organización. No se aceptan nombres escritos; se requiere una firma física o digital. Sube el formulario completado al caso de soporte.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si falta alguno de los elementos anteriores o está incompleto, es posible que no se procese tu solicitud.
    </Callout>
  </Step>

  <Step title="Esperar la preparación de la exportación">
    Una vez que Auth0 reciba toda la documentación requerida, ciframos y exportamos los datos de tu tenant con la clave pública PGP que proporcionaste.
  </Step>

  <Step title="Recibir tu enlace de descarga seguro">
    Recibirás un correo electrónico con un enlace de descarga seguro. El enlace:

    * **Requiere autenticación** con la cuenta de usuario que abrió el caso de soporte. Otros administradores del tenant no podrán descargar el archivo desde este enlace.
    * **Requiere que ese usuario siga teniendo el rol de administrador del tenant** en el momento de la descarga. Si pierde el rol, la descarga se bloqueará y será necesario enviar una nueva solicitud.
    * **Expira después de 3 días**. Descarga el archivo antes de que venza. Una vez vencido, deberás enviar una nueva solicitud.
  </Step>

  <Step title="Descargar y descifrar">
    Descarga el archivo cifrado (con extensión `.pgp` o `.gpg`) y descífralo con tu clave privada PGP y la frase de contraseña que configuraste al generar la clave:

    ```bash
    gpg --output DECRYPTED_EXPORT_FILE --decrypt ENCRYPTED_EXPORT_FILE.pgp
    ```

    Reemplaza `DECRYPTED_EXPORT_FILE` por el nombre de archivo de salida que prefieras y `ENCRYPTED_EXPORT_FILE.pgp` por el nombre del archivo descargado.
  </Step>
</Steps>

<div id="security-reminders">
  ## Recordatorios de seguridad
</div>

* **Nunca compartas tu clave privada ni tu frase de contraseña con nadie**, incluido el personal de soporte de Auth0 u Okta. Auth0 nunca te las pedirá.
* **Haz una copia de seguridad de tu clave privada y tu frase de contraseña** y guárdala de forma segura en un dispositivo sin conexión. Si las pierdes, no podrás descifrar tu exportación y deberás enviar una nueva solicitud.
* **No compartas el enlace de descarga.** Aunque la descarga requiere que el creador del caso se autentique con un rol de administrador activo del tenant, trata el enlace como confidencial.
