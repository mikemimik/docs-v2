title: Configurar la migración automática desde tu base de datos
---

Configura tu conexión de base de datos para migrar automáticamente a tus usuarios desde un repositorio de usuarios externo a Auth0 con [scripts de acción de base de datos personalizada](/es/docs/authenticate/database-connections/custom-db/templates).

<Warning>
  Si intentas usar más de un método de migración (por ejemplo, migración automática y después importación masiva de usuarios), es posible que se produzca un error `DUPLICATED_USER`. Este error indica que el usuario existe en el repositorio interno de usuarios de Auth0, pero no en tu tenant. Para corregirlo, elimina el usuario mediante el endpoint [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de Auth0 Management API y luego vuelve a intentar la importación.
</Warning>

<div id="configure-database-connection-as-custom">
  ## Configura la conexión de base de datos como personalizada
</div>

Crea una conexión de base de datos y, después, configúrala como personalizada.

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y selecciona la base de datos que quieres ver.
2. Selecciona la vista **Custom Database** y activa **Use my own database**.

   <Frame>![Auth0 Dashboard Authentication Database Connection Custom Database Settings Use Own Database Enabled](/docs/images/cdy7uua7fh8z/11HPAdVwJMmnWbzMVjHCJ8/f75ff47f613a41778f266643f29ab2b1/2025-02-27_17-25-36.png)</Frame>
3. Selecciona la vista **Settings**, activa **Import Users to Auth0** y selecciona **Save**.

<div id="configure-database-action-scripts">
  ## Configurar scripts de acción de base de datos
</div>

Selecciona la vista **Custom Database** y, a continuación, localiza **scripts de acción de base de datos**.

<Frame>![Dashboard Autenticación conexión de base de datos pestaña Custom Database scripts de acción de base de datos](/docs/images/cdy7uua7fh8z/4LBIvvjaABo51It4eVCmX3/4ef14765c79cc5b7ecfa3f3155caa9eb/Screen_Shot_2021-05-18_at_8.54.55_PM.png)</Frame>

<div id="login">
  ### Login
</div>

El script [Login](/es/docs/authenticate/database-connections/custom-db/templates/login) se ejecuta cada vez que un usuario que no figura en Auth0 intenta iniciar sesión. Verifica que el usuario exista en el repositorio de usuarios externo sin volver a pedirle la contraseña.

<div id="get-user">
  ### Get User
</div>

El script [Get User](/es/docs/authenticate/database-connections/custom-db/templates/get-user) se ejecuta en cualquiera de los siguientes casos:

* Un usuario intenta registrarse.
* Un usuario inicia un flujo de cambio de contraseña.
* Un usuario introduce su identificador de inicio de sesión al usar Identifier First + Biometrics.
* Se llama al endpoint [Update a User&#39;s Email or Username](https://auth0.com/docs/api/v2#!/Users/patch_users_by_id) o [Create User](https://auth0.com/docs/api/management/v2#!/Users/post_users) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

Si un usuario que no ha sido migrado confirma el cambio de contraseña e inicia sesión correctamente, su perfil de usuario se creará en Auth0 con la nueva contraseña. Este perfil de usuario contiene todos los datos que devuelve el script Get User. Todos los inicios de sesión posteriores de este usuario se realizarán directamente en Auth0.

<div id="verify-user-migration-is-complete">
  ## Verifique que la migración de usuarios esté completa
</div>

Verifique que todos los usuarios se hayan migrado al repositorio de usuarios de Auth0 mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la Auth0 Management API.

1. Vaya a [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) y revise la lista de usuarios.
2. Use el endpoint [List or Search Users](https://auth0.com/docs/api/v2#!/Users/get_users) de Management API.

<div id="disconnect-external-user-store">
  ## Desconectar el repositorio de usuarios externo
</div>

Reconfigura los scripts de acción de base de datos como funciones vacías. Esto evitará que Auth0 se comunique con tu repositorio de usuarios externo para autenticar a los usuarios.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Asegúrate de dejar activada la opción **Import Users to Auth0**. Si desactivas esta opción, Auth0 solo usará los scripts para autenticar y realizar otras acciones de usuario, en lugar de usar los usuarios que se importaron localmente.
</Callout>

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) y luego selecciona tu conexión de base de datos.

2. Cambia a la vista **Custom Database** y luego localiza **scripts de acción de base de datos.**

3. Actualiza el script **Login**.

   ```javascript lines
   function login (email, password, callback) {
     return callback(null, null);
   }
   ```

4. Actualiza el script **Get User**.

   ```javascript lines
   function getByEmail (email, callback) {
     return callback(null, null);
   }
   ```

<div id="troubleshoot-user-migration-issues">
  ## Solucionar problemas de migración de usuarios
</div>

Si tienes algún problema con la migración automática, primero lee [Prácticas recomendadas para la conexión de base de datos personalizada y el script de acción](/es/docs/authenticate/database-connections/custom-db/custom-database-connections-scripts).

<div id="user-already-exists">
  ### El usuario ya existe
</div>

Los escenarios más comunes en los que puedes encontrarte con este problema son:

* Eliminaste un usuario de tu conexión de base de datos y luego intentaste volver a crearlo.
* Intentas crear un usuario nuevo en tu conexión de base de datos cuando ese usuario ya existe en tu repositorio de usuarios externo.
* Se interrumpió el proceso de migración de ese usuario.

Durante el proceso de migración del usuario, Auth0 primero crea un perfil de usuario parcial en un repositorio de usuarios interno y luego crea un perfil de usuario completo en tu conexión de base de datos. Si se produce un problema que impide que se cree este perfil de usuario completo, es posible que te encuentres con el error `The user already exists`.

Normalmente, puedes resolver este problema eliminando al usuario de la conexión de base de datos, eliminándolo del repositorio de usuarios interno y luego volviendo a intentar el proceso de migración.

1. Revisa las sentencias `console.log()` con los [logs en tiempo real de Actions](/es/docs/customize/actions/actions-real-time-logs).
2. Elimina al usuario con el endpoint [Delete a User](https://auth0.com/docs/api/management/v2#!/Users/delete_users_by_id) de la Management API.
3. Elimina al usuario con el endpoint [Delete a Connection User](https://auth0.com/docs/api/management/v2#!/Connections/delete_users_by_email) de la Management API.
4. Indica al usuario que inicie sesión o cambie su contraseña para volver a intentar la migración.

<div id="missing-metadata">
  ### Metadatos faltantes
</div>

Si se interrumpe a un usuario durante el inicio de sesión o el flujo de cambio de contraseña que inicia el proceso de migración, es posible que Auth0 no pueda transferir sus metadatos (`user_metadata` o `app_metadata`) junto con el resto de los datos de su perfil.

Puede mitigar este caso creando una [Action](/es/docs/customize/actions) que verifique si al perfil del usuario le faltan metadatos, los recupere del almacén de usuarios externo y luego los guarde en Auth0.
