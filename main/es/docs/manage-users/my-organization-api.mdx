My Organization API de Auth0 proporciona una interfaz segura con alcance de organización que permite a tus clientes empresariales gestionar sus propias organizaciones dentro de tu tenant de Auth0. Esta API sirve como base técnica para la Administración delegada integrada y las integraciones API-first.

<div id="core-capabilities">
  ## Capacidades principales
</div>

Actualmente, la API admite la gestión de lo siguiente:

* detalles de [Auth0 Organization](/docs/es/manage-users/organizations) (nombre, marca y nombre para mostrar)
* configuración, propiedad y relaciones específicas de la organización
* [Proveedores de identidad (IdPs)](/docs/es/authenticate/identity-providers) y aprovisionamiento de [SCIM](/docs/es/authenticate/protocols/scim)
* configuración de dominios y de Home Realm Discovery (HRD)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  My Organization API te permite un control técnico profundo sobre tu integración. Para lograr la implementación más rápida, te recomendamos encarecidamente comenzar con los componentes de componente de interfaz de usuario integrable, los SDKs y las aplicaciones de ejemplo. Los componentes de componente de interfaz de usuario integrable y las aplicaciones de ejemplo reducen significativamente el tiempo y el esfuerzo necesarios para ofrecer una experiencia de autoservicio a tus clientes y usuarios finales.
</Callout>

<div id="set-up-my-organization-api">
  ## Configurar My Organization API
</div>

<div id="activate-the-my-organization-api-in-auth0-dashboard">
  ### Activa My Organization API en Auth0 Dashboard
</div>

1. Ve a **[Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis)**.
2. Busca el banner de My Organization API.
3. Selecciona **Activar**.
   <Frame>![Auth0 Dashboard>Authentication>APIs](/docs/images/cdy7uua7fh8z/my-org-api-activate.png)</Frame>
4. La API aparece en tu lista de APIs de Applications como My Organization API.

Una vez que actives My Organization API:

* Auth0 deshabilita la API para todas las aplicaciones cliente de forma predeterminada.
* Debes conceder acceso a las aplicaciones y los roles mediante [autorizaciones de cliente](/docs/es/get-started/applications/application-access-to-apis-client-grants) o políticas de [RBAC](/docs/es/manage-users/access-control/configure-core-rbac/enable-role-based-access-control-for-apis).
* Tus clientes empresariales pueden obtener los detalles de la organización o configurar IdPs en nombre de sus propias organizaciones.

<div id="default-settings">
  #### Configuración predeterminada
</div>

**Dominio de Auth0 frente a dominio personalizado**

My Organization API admite el uso de tu dominio canónico de Auth0 o de tu dominio personalizado, pero debes usar el mismo en todo el proceso, que incluye lo siguiente:

* Solicitar un token de acceso
* Establecer la audiencia o el valor `aud`
* Llamar al endpoint de My Organization API

Para obtener más información sobre el uso de dominios personalizados en Auth0, consulta [Custom Domains](/docs/es/customize/custom-domains).

**Políticas de acceso**

De forma predeterminada, My Organization API se activa con las siguientes políticas de acceso de aplicaciones a la API:

* `require_client_grant` para flujos de usuario
* `deny-all` para flujos de máquina a máquina

Para que una aplicación acceda a My Organization API en nombre del usuario, crea una autorización de cliente para esa aplicación a fin de definir los scopes máximos que puede solicitar. Como alternativa, puedes permitir que cualquier aplicación de tu tenant solicite cualquier scope cambiando los flujos de acceso de usuario a `allow_all`.

<Warning>
  Auth0 no recomienda usar `allow_all` para los flujos de acceso de usuario porque My Organization API expone información y operaciones sensibles. Debes seguir el principio de privilegio mínimo para asegurarte de que las aplicaciones solo obtengan acceso a lo que realmente necesitan, y así minimizar los posibles riesgos de seguridad.
</Warning>

Los permisos finales concedidos a la aplicación vendrán determinados por la intersección entre los scopes permitidos por la política de acceso de aplicaciones a la API, los permisos de control de acceso basado en roles (RBAC) asignados al usuario final y cualquier consentimiento otorgado por el usuario (si corresponde).

Para obtener más información sobre cómo administrar las políticas de acceso de aplicaciones a la API y las autorizaciones de cliente asociadas, consulta [Application Access to APIs: Client Grants](/docs/es/get-started/applications/application-access-to-apis-client-grants).

**Duración de los tokens**

My Organization API emite tokens de acceso con una duración fija de 600 segundos (10 minutos). Esta breve duración es una medida de seguridad deliberada diseñada para proteger tu tenant y sus recursos.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  My Organization API seguirá siendo siempre de activación opcional por motivos de seguridad. Deshabilitar la API elimina el acceso de todas las aplicaciones conectadas hasta que vuelva a habilitarse.
</Callout>

### Configurar los atributos de la aplicación cliente

[Crea una aplicación](/docs/es/get-started/auth0-overview/create-applications) en Auth0 para usarla con My Organization API. Una vez creada, ve a **[Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis)** y autoriza My Organization API, incluidos los scopes que quieres que use la aplicación.

Tu aplicación debe proporcionar el objeto `my_organization_configuration`; de lo contrario, My Organization API devolverá un error y rechazará la solicitud. Puedes usar las siguientes propiedades con el objeto `my_organization_configuration`:

| **Propiedad**                  | **Descripción**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `connection_profile_id`        | **ID del perfil de conexión.** ID del [perfil de conexión](/docs/es/authenticate/enterprise-connections/connection-profile) usado con la aplicación al usar My Organization API. Si no se proporciona, las funciones de My Organization API que requieren un perfil de conexión no funcionarán. Este ID debe hacer referencia a un perfil de conexión válido en el mismo tenant.                                                                                                                                                                                                                                                                                 |
| `user_attribute_profile_id`    | **ID del perfil de atributos de usuario**. ID del [perfil de atributos de usuario](/docs/es/authenticate/enterprise-connections/user-attribute-profile) usado con la aplicación al usar My Organization API. Si no se proporciona, las funciones de My Organization API que requieren un perfil de atributos de usuario no funcionarán. Este ID debe hacer referencia a un perfil de atributos de usuario válido en el mismo tenant.                                                                                                                                                                                                                                                             |
| `allowed_strategies`           | **Arreglo de strings.** Cada string es único y hace referencia a una strategy compatible. Las strategies compatibles —los valores de la enum— son las siguientes: `pingfederate`, `ad`, `adfs`, `waad`, `google-apps`, `okta`, `oidc` y `samlp`.                                                                                                                                                                                                                                                                                                                                                                                                              |
| `connection_deletion_behavior` | **Enum (allow, allow&#95;if&#95;empty).** Describe cómo se comporta My Organization API cuando un usuario final intenta eliminar una conexión a través de My Organization API desde esta aplicación. Los valores y la descripción de la enum son los siguientes: <p /> 1. `allow`: Si el usuario tiene el scope correcto, puede eliminar la conexión, lo que hace que se eliminen todos los usuarios que provienen de esa conexión. <p /> 2. `allow_if_empty`: Si el usuario tiene el scope correcto, solo puede eliminar la conexión si no hay usuarios en ella. Si hay usuarios, My Organization API devolverá un error y no continuará con la eliminación. |

<div id="generate-an-access-token">
  ## Generar un token de acceso
</div>

Solo se puede llamar a My Organization API con un token de acceso vinculado al usuario, obtenido mediante uno de los [flujos de OAuth 2.0](https://auth0.com/docs/get-started/authentication-and-authorization-flow).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si vas a permitir que My Organization API realice operaciones sensibles, te recomendamos encarecidamente que uses la [autenticación escalonada](/docs/es/secure/multi-factor-authentication/step-up-authentication) para aplicar políticas de seguridad adicionales mediante autenticación multifactor (MFA).
</Callout>

<div id="example-with-authorization-code-flow">
  ### Ejemplo con el flujo de código de autorización
</div>

Utiliza [flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow) para aplicaciones web confidenciales con un <Tooltip tip="Secreto de cliente: secreto que usa un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=Client+Secret">secreto de cliente</Tooltip>.

```bash
curl --request POST \
--url 'https://YOUR_DOMAIN/oauth/token' \
--header 'content-type: application/x-www-form-urlencoded' \
--data 'grant_type=authorization_code' \
--data 'client_id=YOUR_CLIENT_ID' \
--data 'client_secret=YOUR_CLIENT_SECRET' \
--data 'code=AUTH_CODE' \
--data 'redirect_uri=https://yourapp/callback' \
--data 'audience=https://YOUR_DOMAIN/my-org/'
```

**Respuesta de ejemplo**

```json
{
  "access_token": "eyJz93a...k4laUWw",
  "token_type": "Bearer",
  "expires_in": 600,
  "scope": "read:my_org:details update:my_org:identity_providers"
}
```

<div id="example-with-authorization-code-flow-with-pkce">
  ### Ejemplo con flujo de código de autorización con PKCE
</div>

Usa el [flujo de código de autorización con Proof Key for Code Exchange (PKCE)](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) para aplicaciones públicas sin secreto de cliente, aplicaciones de página única, aplicaciones móviles o nativas y herramientas de CLI.

```bash
curl --request POST \
--url 'https://YOUR_DOMAIN/oauth/token' \
--header 'content-type: application/x-www-form-urlencoded' \
--data 'grant_type=authorization_code' \
--data 'client_id=YOUR_CLIENT_ID' \
--data 'code=AUTH_CODE' \
--data 'code_verifier=CODE_VERIFIER' \
--data 'redirect_uri=https://yourapp/callback' \
--data 'audience=https://YOUR_DOMAIN/my-org/'
```

<div id="audience">
  ### Audiencia
</div>

La audiencia y la URL base de My Organization API son `https://{yourDomain}/my-org/`. Los tokens deben incluir la audiencia `https://YOUR_DOMAIN/my-org/`. Los tokens de otras API (como `/me` o `/api/v2/`) no funcionarán.

<div id="per-organization-rate-limits">
  ### Límites de frecuencia por organización
</div>

Además de los límites de frecuencia del plan de servicio, My Organization API también aplica límites de frecuencia por organización. Estos límites están diseñados para garantizar una asignación justa de los recursos y evitar que una sola organización afecte al rendimiento general de tu tenant. Al aplicar estos límites, mitigamos el efecto de “vecino ruidoso”, para asegurar que un aumento repentino de la actividad de una organización no consuma recursos compartidos ni afecte a otra dentro del mismo entorno. A cada organización se le asigna una cantidad específica de solicitudes por segundo (RPS), tanto para operaciones de lectura como de escritura.

| **Nivel**               | **Lectura por organización (RPS)** | **Escritura por organización (RPS)** |
| ----------------------- | ---------------------------------- | ------------------------------------ |
| **Free**                | 4                                  | 2                                    |
| **Public Self-Service** | 4                                  | 2                                    |
| **Public Enterprise**   | 8                                  | 4                                    |
| **Private Basic**       | 8                                  | 4                                    |
| **Private Performance** | 16                                 | 8                                    |

<div id="cross-origin-requests">
  ## Solicitudes de origen cruzado
</div>

Si tienes previsto llamar a My Organization API directamente desde una aplicación basada en navegador (como una Single Page Application) que se ejecuta en un dominio distinto al de tu tenant de Auth0, te encontrarás con las políticas de seguridad del navegador conocidas como [Cross-Origin Resource Sharing (CORS)](/docs/es/get-started/applications/set-up-cors). De forma predeterminada, los navegadores bloquean estas solicitudes de origen cruzado.

Para permitir que tu aplicación pueda hacer solicitudes a la API correctamente, debes añadir el dominio de tu aplicación (su “origen”) a la configuración de tu cliente:

1. Ve a [**Auth0 Dashboard &gt; Applications**](https://manage.auth0.com/#/applications). Selecciona la aplicación para verla.
2. En Cross-Origin Authentication, activa la opción Allow Cross-Origin Authentication.
3. Busca Allowed Origins (CORS) e introduce la URL de origen de tu aplicación.
4. Selecciona **Save**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si no necesitas usar CORS en tu aplicación, asegúrate de que Allow Cross-Origin Authentication esté desactivado. Añadir la URL de tu aplicación a esta lista le indica a Auth0 que confíe en las solicitudes procedentes de ese origen, lo que permite que tu aplicación del lado del cliente acceda a la API.
</Callout>

<div id="organization-connection-ownership">
  ## Titularidad de la conexión de la organización
</div>

La API introduce un modelo de titularidad para distinguir entre las conexiones administradas por el Admin del tenant y las autoadministradas por la organización. Esto se controla mediante la propiedad `organization_access_level`.

**Propiedad clave**: `organization_access_level`

| **Valor enum** | **Descripción**                                                                                                                                                                                                                                                                                            |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `none`         | Asociación asignada por el administrador del tenant. Esta conexión no se puede ver ni editar mediante la My Organization API.                                                                                                                                                                              |
| `readonly`     | Asociación asignada por el administrador del tenant. Se puede ver, pero no editar mediante la My Organization API.                                                                                                                                                                                         |
| `limited`      | Asociación asignada por el administrador del tenant. Se permiten modificaciones y acceso a información limitados mediante la My Organization API, concretamente en los atributos `show_as_button` e `is_enabled`.                                                                                          |
| `full`         | Asociación asignada por el administrador del tenant. Los usuarios de la My Organization API podrán modificar los atributos de la conexión específicos de la organización sin restricciones adicionales, concretamente los atributos `show_as_button`, `is_enabled`, `options`, `display_name` y `domains`. |

**Endpoints de Management API para conexiones**:

* `GET` [`/api/v2/organizations/{id}/connections`](https://auth0.com/docs/api/management/v2/organizations/get-organization-connection)
* `POST` [`/api/v2/organizations/{id}/connections`](https://auth0.com/docs/api/management/v2/organizations/post-organization-connection)
* `PATCH` [`/api/v2/organizations/{id}/connections/{id}`](https://auth0.com/docs/api/management/v2/organizations/patch-organization-connection)
* `DELETE` [`/api/v2/organizations/{id}/connections/{id}`](https://auth0.com/docs/api/management/v2/organizations/delete-organization-connection)

Cuando llames a los endpoints de `/connections`, usa los mismos scopes que usarías para los endpoints de `/enabled_connections`:

* `create:organization_connections`
* `read:organization_connections`
* `delete:organization_connections`
* `update:organization_connections`

Revisa los atributos adicionales del esquema:

| **Propiedad**                  | **Descripción**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `is_enabled`                   | **Boolean**. Habilita o deshabilita una conexión para una organización. Al deshabilitar la conexión, será posible conservar los valores del objeto de la conexión mientras la conexión siga deshabilitada.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `organization_access_level`    | **Enum (null, none, readonly, limited, full)**. Determina el tipo de acceso que tendrá un usuario al usar la My Organization API: 1. `null` 2. `none` - Asociación asignada por el administrador del tenant. Esta conexión no se puede ver ni editar mediante la My Organization API. 3. `readonly` - Asociación asignada por el administrador del tenant. Esta conexión se puede ver, pero no editar mediante la My Organization API. 4. `limited` - Asociación asignada por el administrador del tenant. Se permiten información y modificaciones limitadas mediante la My Organization API. 5. `full` - Asociación asignada por el administrador del tenant. Los usuarios de la My Organization API podrán modificar la conexión sin restricciones adicionales al usar la My Organization API. Nota: Independientemente de `organization_access_level`, las modificaciones realizadas mediante la My Organization API siempre están sujetas a las restricciones aplicadas por el perfil de la conexión. |
| `organization_connection_name` | **String**. Almacena el nombre proporcionado al crear la conexión. La My Organization API calcula este campo al evaluar `connection_name_prefix_template` del perfil de la conexión. Este campo solo es visible y modificable mediante la Management API.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

Notas:

* Estos endpoints aceptan un query parameter opcional `is_enabled=true/false` y, si está presente, solo muestran las conexiones que tienen el valor `is_enabled` especificado.
* `organization_access_level` solo se puede cambiar mediante la Management API.
* Si el atributo `name` no está establecido, debe completarse mediante la Management API antes de cambiar `organization_access_level` de `none` a cualquier otro valor.
