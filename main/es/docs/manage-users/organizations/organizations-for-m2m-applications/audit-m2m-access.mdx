El acceso de máquina a máquina a Organizations puede concederse asociando directamente una autorización de cliente a una organización o permitiendo el acceso a cualquier organización en la configuración de la autorización de cliente. Ambos casos pueden auditarse a través del <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>, donde puedes ver o recuperar una lista de autorizaciones de cliente asociadas a una organización para una aplicación. Para obtener más información sobre cómo funcionan los permisos de acceso M2M a Organizations, lee [Configure Your Application for M2M Access](/es/docs/manage-users/organizations/organizations-for-m2m-applications/configure-your-application-for-m2m-access) y [Authorize M2M Access](/es/docs/manage-users/organizations/organizations-for-m2m-applications/authorize-m2m-access).

<div id="access-granted-via-direct-association">
  ## Acceso otorgado mediante asociación directa
</div>

Usa el [Auth0 Dashboard](https://manage.auth0.com/) y la [Management API](https://auth0.com/docs/api/management/v2) para auditar el acceso M2M de las autorizaciones de cliente asociadas directamente a una organización.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para ver en el Auth0 Dashboard las autorizaciones de cliente de aplicación que se han autorizado para una organización específica:

    1. Ve a **Organizations** y elige la organización que quieres inspeccionar.
    2. Selecciona la pestaña **acceso de máquina a máquina**. Verás una lista paginada de todas las aplicaciones que pueden acceder a una API de esta organización mediante asociación directa.
    3. Elige una aplicación para revisar las API autorizadas que aparecen para esa aplicación.

    <Frame>![](/docs/images/cdy7uua7fh8z/5FVJzuu5yq8IHwpQKdYYU6/3468ee757c1e65fb51883213e7e7b476/image2.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Las autorizaciones de cliente asociadas a una organización se pueden recuperar mediante el endpoint de API [Organization Client Grants](https://auth0.com/docs/api/management/v2/organizations/get-organization-client-grants):

    <AuthCodeBlock children={codeExample1} language="bash" />

    Como alternativa, si quieres recuperar información sobre aplicaciones que pueden acceder al menos a una de las API de una organización, puedes usar el parámetro de consulta `q` en el [endpoint de API Get Clients](https://auth0.com/docs/api/management/v2/clients/get-clients) con el término de búsqueda `client_grant.organization_id:{organization_id}`. Usa la sintaxis de Lucene, que es similar a la de [User Search Query Syntax](/es/docs/manage-users/user-search/user-search-query-syntax).

    <AuthCodeBlock children={codeExample2} language="bash" />
  </Tab>
</Tabs>

<div id="access-granted-to-any-organization">
  ## Acceso concedido a cualquier organización
</div>

Usa el [Auth0 Dashboard](https://manage.auth0.com/) y la [Management API](https://auth0.com/docs/api/management/v2) para auditar el acceso M2M de las aplicaciones que tienen acceso concedido a cualquier organización.

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Ve a **Organizations**.
    2. Selecciona la pestaña **Machine to Machine Access**. Verás una lista paginada de todas las aplicaciones que pueden acceder al menos a una de las API de una organización.
    3. Elige una aplicación para revisar las API autorizadas que figuran para esa aplicación.

    <Frame>![](/docs/images/cdy7uua7fh8z/4H3VjVTz0ZGDvV5za9y33m/d0d193df808d7308469ce66d4357b5e7/image1.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Las autorizaciones de cliente con el parámetro `allow_any_organization` pueden recuperarse mediante el endpoint de API [Get Client Grants](https://auth0.com/docs/api/management/v2/client-grants/get-client-grants):

    <AuthCodeBlock children={codeExample3} language="bash" />

    Como alternativa, si quieres recuperar información sobre las aplicaciones que pueden acceder al menos a una de las API de cualquier organización, puedes usar el parámetro de consulta `q` en el endpoint de API [Get Client Grants](https://auth0.com/docs/api/management/v2/client-grants/get-client-grants) con el término de búsqueda `client_grant.allow_any_organization:true`. Usa la sintaxis de Lucene, que es similar a la [User Search Query Syntax](/es/docs/manage-users/user-search/user-search-query-syntax).

  </Tab>

<div id="search-applications-based-on-organization-access">
  ## Buscar aplicaciones según el acceso a organizaciones
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los resultados de la búsqueda tienen consistencia eventual.
</Callout>

La siguiente tabla muestra los términos de búsqueda admitidos para buscar aplicaciones con el parámetro `q` en el endpoint `/clients`:

| **Campo**                                        | **Descripción**                                                                                         |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------------- |
| `client_grant.organization_id:{organization_id}` | Se usa para buscar aplicaciones que pueden acceder al menos a una de las API de una organización.       |
| `client_grant.allow_any_organization:true`       | Se usa para buscar aplicaciones que pueden acceder al menos a una de las API de cualquier organización. |
