Al trabajar con [Auth0 Organizations](/docs/es/manage-users/organizations/organizations-overview) de forma programática, es posible que necesites obtener una lista de conexiones de organización.

Puedes ver las conexiones asociadas a una organización usando el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Ve a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y selecciona la organización.
2. Selecciona la vista **Connections**.

<div id="management-api">
  ## Management API
</div>

Haz una llamada `GET` al endpoint `Get Organization Connections`. Asegúrate de reemplazar los valores de marcador de posición `ORG_ID` y `MGMT_API_ACCESS_TOKEN` por tu ID de organización y tu <Tooltip tip="Token de acceso: credencial de Authorization, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">token de acceso</Tooltip> de Management API, respectivamente.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 se compone del nombre de tu tenant, tu subdominio regional (a menos que tu tenant esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu tenant fuera `travel0`, tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu tenant estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`).

  Si usas [dominios personalizados](/docs/es/customize/custom-domains), este debería ser el nombre de tu dominio personalizado.
</Callout>

| Valor                   | Descripción                                                                                                                                         |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de la organización para la que desea obtener las conexiones.                                                                                     |
| `MGMT_API_ACCESS_TOKEN` | [Access Token for the Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens) con el scope `read:organization_connections`. |

<div id="response-status-codes">
  ### Códigos de estado de respuesta
</div>

Los posibles códigos de estado de respuesta son los siguientes:

| Código de estado | Código de error      | Mensaje                                                                                                      | Causa                                                                                           |
| ---------------- | -------------------- | ------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------- |
| `200`            |                      | Conexiones recuperadas correctamente.                                                                        |                                                                                                 |
| `400`            | `invalid_body`       | Cuerpo de la solicitud no válido. El mensaje variará según la causa.                                         | El payload de la solicitud no es válido.                                                        |
| `401`            |                      | Token no válido.                                                                                             |                                                                                                 |
| `401`            |                      | Se recibió una firma no válida durante la validación del JSON Web Token.                                     |                                                                                                 |
| `401`            |                      | El cliente no es global.                                                                                     |                                                                                                 |
| `403`            | `insufficient_scope` | Scope insuficiente; se esperaba alguno de los siguientes: `read:organization_connections`.                   | Se intentó leer o escribir un campo no permitido con los scopes del bearer token proporcionado. |
| `429`            |                      | Demasiadas solicitudes. Revisa los encabezados X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                 |
