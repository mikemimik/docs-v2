Al representar tu aplicación en Auth0, puedes especificar qué tipos de usuarios debe admitir. Algunas aplicaciones permiten que las personas inicien sesión con cuentas personales, mientras que otras están pensadas para que las usen miembros de [organizaciones](/es/docs/manage-users/organizations/organizations-overview). Y algunas deben admitir ambas opciones. Esto se conoce como comportamiento de la organización y puede configurarse para cada aplicación que conectes a Auth0.

Por ejemplo, tu aplicación podría tener:

* Una página de destino de marketing genérica con un botón de **Log in** que lleve a tus usuarios al flujo de inicio de sesión de Auth0 sin una organización.
* Una URL distinta para cada uno de tus clientes B2B (por ejemplo, los usuarios de Acme van a `acme.yourcompany.com`) que redirija a los usuarios a Auth0 con una organización, para que vean el botón de Login de <Tooltip tip="inicio de sesión único (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=SSO">SSO</Tooltip> de Acme.

Puedes definir el comportamiento de la organización para permitir cualquiera de estos escenarios. Además, puedes configurar el comportamiento de la organización de modo que, si tu aplicación requiere que se proporcione una organización pero tu usuario es enviado accidentalmente a Auth0 sin una organización, vea un prompt para ingresar el nombre de su organización.

Puedes definir el comportamiento de la organización mediante el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para definir el comportamiento de la organización a través del Auth0 Dashboard:

1. Vaya a [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/#/applications) y seleccione la aplicación para la que desea configurar Organizations.
2. Cambie a la vista Experiencia de inicio de sesión y configure los ajustes correspondientes:

| Campo                         | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | Mapeo de API                                                                                                                                                                                                                                                                              |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Tipo de usuarios**          | Determina qué categoría de usuarios puede iniciar sesión en su aplicación.<br /><br />Las opciones incluyen: <ul> <li> **Usuarios individuales**: Los usuarios pueden registrarse con una cuenta personal e iniciar sesión directamente en su aplicación. Los usuarios individuales no pueden iniciar sesión mediante una organización. </li> <li> **Usuarios empresariales**: Los usuarios deben ser miembros de una organización para poder iniciar sesión. Cuando se selecciona esta opción, debe proporcionar una organización al redirigir a los usuarios al endpoint `/authorize` o configurar su Login Flow para solicitar una organización. </li> <li> **Ambos**: Los usuarios pueden iniciar sesión como miembros de una organización o registrarse con una cuenta personal. </li> </ul>                                                                                           | **Tipo de usuarios** corresponde a `organization_usage`<br /><br />Opciones: <ul> <li>**Usuarios individuales** corresponde a `deny`</li> <li>**Usuarios empresariales** corresponde a `require`</li> <li>**Ambos** corresponde a `allow`</li> </ul>                                      |
| **Login Flow**                | Determina el prompt de inicio de sesión inicial que se muestra a los usuarios cuando acceden a su aplicación. Solo puede configurar este campo si Tipo de usuarios está establecido en Usuarios empresariales o Ambos.<br /><br />Las opciones incluyen: <ul> <li> **Solicitar credenciales**: Primero se solicita a los usuarios que proporcionen sus credenciales de inicio de sesión. Después de iniciar sesión, los usuarios pueden seleccionar su organización. </li> <li> **Solicitar organización**: Primero se solicita a los usuarios que seleccionen su organización. Luego, pueden proporcionar sus credenciales para iniciar sesión. </li> <li> **Sin prompt**: Auth0 no determina qué prompt de inicio de sesión se muestra a los usuarios. En su lugar, su aplicación es responsable de enviar los parámetros necesarios a Auth0 para mostrar el prompt adecuado. </li> </ul> | **Login Flow** corresponde a `organization_require_behavior`<br /><br />Opciones: <ul> <li>**Solicitar credenciales** corresponde a `post_login_prompt`</li> <li>**Solicitar organización** corresponde a `pre_login_prompt`</li> <li>**Sin prompt** corresponde a `no_prompt`</li> </ul> |
| **Detección de organización** | Define el método utilizado para identificar la organización del usuario antes de la autenticación. Solo está disponible con el Login Flow Solicitar organización.<br /><br />Las opciones incluyen: <ul> <li> **Solicitar correo electrónico de la organización**: Primero se solicita a los usuarios que proporcionen la dirección de correo electrónico asociada a su organización. </li> <li> **Solicitar nombre de la organización**: Primero se solicita a los usuarios que proporcionen el nombre de su organización. </li> <li> **Solicitar correo electrónico y nombre de la organización**: Primero se solicita a los usuarios que proporcionen el nombre de su organización o el correo electrónico asociado a su organización. </li> </ul>                                                                                                                                       | <ul> <li> **Solicitar correos electrónicos**: corresponde a `email`. </li> <li> **Solicitar organización**: corresponde a `organization_name`. </li> <li> **Solicitar correo electrónico y nombre de la organización**: corresponde a `email` y `organization_name`. </li> </ul>          |

3. Seleccione **Save changes**.

<div id="management-api">
  ## Management API
</div>

Haz una llamada `PATCH` al [endpoint Actualizar un cliente](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id). Asegúrate de reemplazar los valores de marcador de posición `client_id`, `mgmt_api_access_token`, `organization_usage` y `organization_require_behavior` por tu <Tooltip tip="ID de cliente: valor de identificación que Auth0 asigna a tu recurso registrado." cta="Ver glosario" href="/es/docs/glossary?term=client+ID">ID de cliente</Tooltip>, el <Tooltip tip="ID de cliente: valor de identificación que Auth0 asigna a tu recurso registrado." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">token de acceso</Tooltip> de la Management API, la opción de uso de la organización y la opción de comportamiento de la organización, respectivamente.

| Value                            | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CLIENT_ID`                      | ID de la aplicación a la que desea agregar el comportamiento de organización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `MGMT_API_ACCESS_TOKEN`          | [Access Tokens para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:clients`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| `ORGANIZATION_USAGE`             | Determina si su aplicación puede admitir que los usuarios inicien sesión en una organización. Las opciones incluyen: <ul> <li>`deny`: (Predeterminado) Los usuarios no pueden iniciar sesión con una organización.</li> <li>`allow`: Los usuarios pueden iniciar sesión con o sin una organización. Cuando se selecciona esta opción, debe proporcionar una organización al redirigir a los usuarios al endpoint `/authorize`.</li> <li>`require`: Los usuarios deben iniciar sesión con una organización. Cuando se selecciona esta opción, debe proporcionar una organización al redirigir a los usuarios al endpoint `/authorize` o establecer `organization_require_behavior` en `pre_login_prompt` para permitir que los usuarios elijan una organización antes de iniciar sesión.</li> </ul> |
| `ORGANIZATION_REQUIRE_BEHAVIOR`  | Determina el Login Flow que se presenta a los usuarios que acceden a su aplicación. Solo se aplica cuando `organization_usage` está establecido en `require` o `allow`.<br /><br />Las opciones incluyen: <ul> <li>`no_prompt`: (Predeterminado) No se muestra ningún prompt. Las requests sin un parámetro de organización válido se rechazan.</li> <li>`pre_login_prompt`: Se solicita a los usuarios que seleccionen una organización antes de poder iniciar sesión. Solo puede usar esta opción si `organization_usage` está establecido en `require`.</li> <li>`post_login_prompt`: Se solicita a los usuarios que inicien sesión con sus credenciales. Después de iniciar sesión, se les solicita que seleccionen su organización.</li> </ul>                                                |
| `ORGANIZATION_DISCOVERY_METHODS` | Determina el método para identificar la organización de un usuario antes de la autenticación. Solo se aplica cuando `organization_require_behavior` es `pre_login_prompt` y `organization_usage` está establecido en `require` o `allow`. <ul> <li> `email`: (Predeterminado) Se solicita a los usuarios que introduzcan la dirección de correo electrónico asociada a su organización. </li> <li> `organization_name`: Se solicita a los usuarios que introduzcan el nombre asociado a su organización. </li> <li> `Organization name` y `email`: Se solicita a los usuarios la dirección de correo electrónico o el nombre asociados a su organización. </li> </ul>                                                                                                                              |
