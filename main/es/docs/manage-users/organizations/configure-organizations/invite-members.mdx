title: Invitar a miembros de una organización
---

<Card title="La disponibilidad varía según el plan de Auth0">
  Tanto tu implementación específica de inicio de sesión como tu plan de Auth0 o acuerdo personalizado determinan si esta funcionalidad está disponible. Para obtener más información, consulta [Pricing](https://auth0.com/pricing).
</Card>

Si quieres asignar un miembro a una [organización](/docs/es/manage-users/organizations/organizations-overview), pero el usuario aún no existe en tu almacén de datos (o existe, pero todavía no forma parte de tu organización), puedes invitarlo a la organización. En ese caso, el usuario recibirá un correo electrónico con un enlace que le permitirá crear una cuenta o iniciar sesión y unirse a la organización, opcionalmente con roles predefinidos, y será redirigido a tu aplicación para que pueda empezar a usarla de inmediato.

<Warning>
  Los roles de los miembros de una organización son independientes de los roles asignados a los usuarios mediante [Role-Based Access Control](/docs/es/manage-users/access-control/rbac) de Auth0. Cuando se aplica un rol predefinido a un usuario que acepta una invitación a una organización, el rol asignado solo se aplica cuando el usuario inicia sesión a través de esa organización. Puedes [ver los roles de los miembros de la organización](/docs/es/manage-users/organizations/configure-organizations/retrieve-member-roles) en el Dashboard.
</Warning>

Para configurar el flujo de invitación:

1. [Integrar el flujo de invitación de membresía en tu aplicación](#integrate-the-membership-invitation-flow-into-your-application)
2. [Personalizar el flujo de invitación de usuario](#customize-the-user-invitation-flow)
3. [Enviar invitaciones de membresía](#send-membership-invitations)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Limitaciones**

  * Auth0 solo puede enviar invitaciones de usuario por correo electrónico. Si prefieres enviar la invitación de otra forma, puedes optar por generar únicamente una URL de invitación.
  * Los usuarios invitados deben iniciar sesión o crear una cuenta con la dirección de correo electrónico a la que se envió la invitación.
</Callout>

<div id="integrate-the-membership-invitation-flow-into-your-application">
  ## Integra el flujo de invitación de membresía en tu aplicación
</div>

Para que las invitaciones de usuario funcionen, debes configurar una ruta en tu aplicación como parte del flujo de aceptación de la invitación y [configurar una ruta de inicio de sesión predeterminada asociada](/docs/es/authenticate/login/auth0-universal-login/configure-default-login-routes) para tu tenant o tu aplicación. En la invitación por correo electrónico que se envía a los usuarios, se incluirá un enlace al URI configurado.

<div id="configure-the-uri">
  ### Configura el URI
</div>

Si el URI debe compartirse en todas tus aplicaciones, puedes establecer la **Tenant Login URL** en [Auth0 Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced). De lo contrario, puedes establecer el URI para cada aplicación mediante **Application Login URI** en la configuración de la aplicación, a la que puedes acceder desde [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).

<div id="specify-route-behavior">
  ### Especificar el comportamiento de la ruta
</div>

La ruta de tu aplicación debe aceptar los parámetros `invitation` y `organization` en la cadena de consulta y, a continuación, llamar al endpoint [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de la Auth0 Authentication API con ambos parámetros.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para admitir escenarios multi-tenant en los que el nombre de la organización se use como subdominio o variable de ruta en tu aplicación (por ejemplo, `acme.myapp.com`), el parámetro `organization_name` también se incluye en el enlace de invitación que se envía a los usuarios. No es necesario enviar este parámetro al endpoint [Authorization](https://auth0.com/docs/api/authentication#authorize-application) de la Auth0 Authentication API.
</Callout>

Por ejemplo, si tienes una aplicación habilitada para organizaciones con un **Application Login URI** establecido en `https://myapp.com/login`, el enlace de la invitación por correo electrónico enviada al usuario final sería: `https://myapp.com/login?invitation={invitation_ticket_id}&organization={organization_id}&organization_name={organization_name}`.

<div id="customize-the-user-invitation-flow">
  ## Personaliza el flujo de invitación de usuarios
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para personalizar el flujo de invitación de usuarios, debes usar dominios personalizados. Los planes Free incluyen un dominio personalizado gratis; puedes obtener dominios personalizados adicionales con cualquier plan de pago.
</Callout>

Para personalizar el prompt de invitación de usuarios y el correo electrónico que recibe el usuario, modifica el prompt de <Tooltip tip="Universal Login: tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> y las plantillas de correo electrónico. Para obtener más información, consulta estas páginas:

* [Plantillas de la página de Universal Login](/docs/es/customize/login-pages/universal-login/customize-templates)
* [Internacionalización de Universal Login](/docs/es/customize/internationalization-and-localization/universal-login-internationalization)
* [Personalizar plantillas de correo electrónico](/docs/es/customize/email/email-templates)

<div id="localization">
  ### Localización
</div>

Dado que la configuración regional del usuario invitado se desconoce de antemano, puedes enviar un parámetro opcional `user_metadata` al endpoint de invitación mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>. Al localizar el flujo de invitación de usuarios, puedes acceder a este parámetro a través de la plantilla de correo electrónico **Invitación de usuario** y [aplicar la misma lógica que con otras plantillas de correo electrónico multilingües](/docs/es/customize/email/email-templates).

Para que el usuario invitado vea la invitación en el idioma adecuado, tu aplicación deberá enviar el parámetro `ui_locales`, o bien el navegador del usuario invitado deberá enviar el encabezado `Accept-Language` correspondiente. Para obtener más información, consulta [Personalizar plantillas de correo electrónico: Plantillas de correo electrónico multilingües](/docs/es/customize/email/email-templates).

<div id="send-membership-invitations">
  ## Enviar invitaciones de membresía
</div>

Puedes [enviar invitaciones de membresía a una organización](/docs/es/manage-users/organizations/configure-organizations/send-membership-invitations) a los usuarios usando el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la Management API. Si prefieres enviar la invitación con tu propio servicio de correo electrónico, puedes optar por generar solo una URL de invitación. De lo contrario, el usuario recibirá un correo electrónico con el enlace, que le permitirá iniciar sesión en la organización mediante una conexión configurada.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cuando un usuario acepta una invitación, su dirección de correo electrónico se marcará como verificada.

  Si el usuario acepta una invitación a través de un inicio de sesión federado, el proveedor de identidad debe devolver la misma dirección de correo electrónico a la que se envió la invitación. La verificación del correo electrónico no se sobrescribirá con el valor devuelto por el proveedor de identidad en inicios de sesión posteriores.

  Si el usuario no acepta una invitación antes de que venza, la invitación se marcará como vencida y, con el tiempo, se eliminará del [Auth0 Dashboard](/docs/es/get-started/auth0-overview/dashboard).
</Callout>
