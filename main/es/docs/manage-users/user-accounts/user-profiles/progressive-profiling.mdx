---
description: Comprende cómo el perfilado progresivo puede recopilar información sobre los usuarios con el tiempo, a medida que interactúan con tu sitio web o aplicación.
title: Comprende cómo funciona el perfilado progresivo
---

El perfilado progresivo es el proceso de recopilar de forma incremental información adicional sobre tus usuarios a medida que interactúan con tu sitio web o aplicación. Puedes recopilar información detallada de los usuarios solo cuando sea relevante para sus actividades.

Por ejemplo, podrías recopilar el nombre, el correo electrónico y la contraseña del usuario durante el registro, y luego pedirle el nombre de su empresa y su cargo la próxima vez que inicie sesión. También puedes usar [formularios de Auth0](/es/docs/customize/forms/configure-progressive-profile-form) para diseñar un flujo que recopile información de los usuarios.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cuando solicites información adicional a los usuarios, evita pedir datos que quizá ya tengas. Por ejemplo, si un usuario se registra a través de una conexión social que usa el protocolo OpenID Connect (como Google o Facebook), el proveedor de identidad (IdP) puede devolver más información del usuario de la que imaginas. Para obtener más información, consulta [Scopes de OpenID Connect](/es/docs/get-started/apis/scopes/openid-connect-scopes#standard-claims).
</Callout>

Implementar el perfilado progresivo mejora la experiencia de tus usuarios al:

* Acortar los formularios de registro.
* Evitar preguntas repetitivas.
* Recopilar información más relevante.
* Aumentar las tasas de conversión.

<div id="how-it-works">
  ## Cómo funciona
</div>

Cada vez que un usuario se autentica con Auth0, Auth0 actualiza su perfil de usuario. Los datos que se usan para actualizar su perfil pueden provenir de distintas fuentes:

* Atributos proporcionados por el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=IdP">IdP</Tooltip>.
* Atributos creados dinámicamente con [Auth0 Actions](/es/docs/customize/actions).
* Atributos devueltos al llamar a APIs como [FullContact](https://www.fullcontact.com/) y [Clearbit](https://clearbit.com/).
* Atributos gestionados por la aplicación.

<div id="use-action-triggers">
  ## Usar los triggers de Action
</div>

Con el trigger de Action `post-login`, puedes modificar los campos `user_metadata` y `app_metadata` de un usuario durante el flujo de inicio de sesión. Esto resulta útil para tareas como almacenar datos específicos de la aplicación en el perfil del usuario, registrar si se han realizado determinadas operaciones para un usuario o almacenar en caché los resultados de operaciones costosas en el perfil del usuario para reutilizarlos en futuros inicios de sesión. Para obtener más información, consulta [Administrar los metadatos del usuario con el trigger de Action `post-login`](/es/docs/manage-users/user-accounts/metadata/manage-user-metadata).

<div id="use-the-management-api">
  ## Usa la Management API
</div>

Puedes usar el endpoint [Update a User](https://auth0.com/docs/api/management/v2/#!/Users/patch_users_by_id) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0 para actualizar los datos del perfil de un usuario autenticado, incluidos los campos `app_metadata` y `user_metadata`. Para obtener más información, consulta [Administrar metadatos con la Management API](/es/docs/manage-users/user-accounts/metadata/manage-metadata-api).

<div id="use-the-lock-library">
  ## Usa la biblioteca Lock
</div>

Puedes usar la opción `additionalSignUpFields` para agregar campos personalizados a los formularios de registro. Cuando un usuario introduce datos en un campo personalizado, Auth0 almacena el valor en el campo `user_metadata` de su perfil. Para obtener más información, consulta [Opciones de configuración de Lock](/es/docs/libraries/lock/lock-configuration#additionalsignupfields-array-).
