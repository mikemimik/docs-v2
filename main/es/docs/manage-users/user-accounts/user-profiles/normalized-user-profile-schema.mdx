title: Esquema del perfil de usuario normalizado
---

A continuación se enumeran los atributos que Auth0 asigna a un esquema común.

Campos que siempre se generan:

* `name`: el nombre completo del usuario.
* `nickname`: de forma predeterminada, la parte local del correo electrónico del usuario.
* `picture`: la URL de la [foto del usuario](/es/docs/manage-users/user-accounts/change-user-picture). Si no está disponible, Auth0 usa la imagen de Gravatar asociada a la dirección de correo electrónico del usuario.
* `user_id`: el identificador único del usuario. Es único por conexión, pero es el mismo para todas las aplicaciones que se autentican a través de esa conexión.

De forma predeterminada, los atributos `name`, `nickname` y `picture` de un usuario proporcionados por <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente, ya que se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Si desea poder editar estos atributos, debe [configurar la sincronización de su conexión con Auth0](/es/docs/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo al crear el perfil de usuario. Los atributos raíz estarán entonces disponibles para [editarse individualmente](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) o [mediante importación masiva](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) usando la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

Campos que se generan cuando la información está disponible:

* `email`: la dirección de correo electrónico del usuario.
* `email_verified`: un valor booleano que indica si la dirección de correo electrónico del usuario ha sido verificada.
* `given_name`: el nombre del usuario.
* `family_name`: el apellido del usuario.

Al crear un usuario con el [endpoint Create a User de la Management API](https://auth0.com/docs/api/management/v2#!/Users/post_users), puede enviar `given_name` y `family_name`. De forma predeterminada, los atributos `given_name` y `family_name` de un usuario proporcionados por proveedores de identidad distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente, ya que se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Si desea poder editar estos atributos, debe [configurar la sincronización de su conexión con Auth0](/es/docs/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo al crear el perfil de usuario. Los atributos raíz estarán entonces disponibles para [editarse individualmente](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) o [mediante importación masiva](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) usando la Management API

Si está escribiendo un script de login para una [base de datos personalizada](/es/docs/authenticate/database-connections/custom-db), es responsable de devolver la información del perfil de usuario. Una propiedad `user_id` única e inmutable es obligatoria para identificar correctamente al usuario (consulte [Identificar usuarios de forma única](/es/docs/manage-users/user-accounts/identify-users)).

<div id="additional-attributes">
  ## Atributos adicionales
</div>

El perfil del usuario incluye un arreglo de identidades. En el caso más común (cuando se inicia sesión con un solo proveedor), el arreglo contiene un solo elemento. Si el usuario tiene varias cuentas vinculadas, el arreglo tendrá un elemento por cada cuenta asociada. Consulta [Vinculación de cuentas de usuario](/es/docs/manage-users/user-accounts/user-account-linking) para obtener más información.

El arreglo `identities` contiene los siguientes atributos:

* `connection`: el nombre de la conexión.
* `isSocial`: indica si el proveedor es un proveedor social.
* `provider`: el proveedor de la conexión.
* `user_id`: el identificador único del usuario para esta conexión.

Auth0 pasa a tu aplicación todas las demás propiedades proporcionadas por el proveedor de identidad, aunque no estén asignadas a los atributos estándar enumerados anteriormente.
