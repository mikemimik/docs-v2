---
description: Conozca los atributos raíz disponibles para el perfil de usuario normalizado de Auth0 y los métodos para actualizarlos.
title: Atributos raíz del perfil de usuario
---

El perfil de usuario normalizado de Auth0 contiene varios atributos raíz (atributos almacenados en el primer nivel, o nivel raíz, del objeto), algunos de los cuales se pueden actualizar. Para obtener más información sobre la estructura del perfil de usuario y sus atributos, consulte [Estructura del perfil de usuario.](/es/docs/manage-users/user-accounts/user-profiles/user-profile-structure)

Los métodos para actualizar los atributos raíz varían según el tipo de conexión.

<div id="auth0-as-the-identity-provider">
  ## Auth0 como proveedor de identidad
</div>

Cuando Auth0 es el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP), los suscriptores pueden:

* Definir atributos raíz [durante el registro del usuario](https://auth0.com/docs/api/management/guides/users/set-root-attributes-user-signup) (mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> o mediante el registro público)
  o [durante la importación](https://auth0.com/docs/api/management/guides/users/set-root-attributes-user-import)
* [Actualizar los atributos raíz individualmente](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) mediante la Management API

Auth0 es el IdP para los siguientes tipos de conexión:

* [Conexiones de base de datos estándar](/es/docs/authenticate/database-connections)
* [Conexiones de base de datos personalizadas](/es/docs/authenticate/database-connections/custom-db) con modo de importación
* [Conexiones passwordless](/es/docs/authenticate/passwordless)

<div id="upstream-identity-providers">
  ## Proveedores de identidad ascendentes
</div>

Cuando se usa un IdP ascendente (como Google o Facebook), los suscriptores tienen dos opciones:

* El IdP ascendente establece los atributos raíz cuando los usuarios se crean por primera vez y luego
  los actualiza automáticamente en cada inicio de sesión posterior. Este es el comportamiento predeterminado.
* El IdP ascendente establece los atributos raíz solo durante la creación del usuario y no
  los actualiza en los inicios de sesión posteriores, lo que permite a los suscriptores [actualizar los atributos raíz individualmente](https://auth0.com/docs/api/management/guides/users/update-root-attributes-users) mediante la Management API. Para habilitar esto, deberás [configurar la sincronización de tu conexión con Auth0](https://auth0.com/docs/api/management/guides/connections/configure-connection-sync).

Los proveedores de identidad ascendentes admiten los siguientes tipos de conexión:

* [Conexiones sociales](/es/docs/authenticate/identity-providers#social)
* [Conexiones empresariales](/es/docs/authenticate/identity-providers#enterprise)
* [Conexiones de identidad legal](/es/docs/authenticate/identity-providers#legal-identities)
