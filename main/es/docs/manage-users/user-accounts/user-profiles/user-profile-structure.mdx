title: Estructura del perfil de usuario

---

El perfil de usuario normalizado de Auth0 consta de varios componentes:

* **Details**: El objeto principal del perfil de usuario, que contiene información básica, como el nombre, el correo electrónico y la marca de tiempo del último inicio de sesión del usuario, en atributos predefinidos. Este objeto también puede contener información de la conexión de origen del usuario. La mayoría de los atributos del usuario son atributos raíz (atributos almacenados en el primer nivel, o nivel raíz, del objeto `user`), y algunos de ellos son editables.
* **Metadata**: Dos subobjetos que funcionan como almacenamiento secundario para guardar información adicional del usuario en atributos personalizables: `user_metadata` y `app_metadata`. Para obtener más información, incluido cuándo usar `app_metadata` y `user_metadata`, consulta [Comprende cómo funcionan los metadatos en los perfiles de usuario](/docs/es/manage-users/user-accounts/metadata).

<div id="user-profile-attributes">
  ## Atributos del perfil de usuario
</div>

Los siguientes atributos están disponibles en el perfil de usuario. Muchos son atributos raíz (atributos almacenados en el primer nivel, o nivel raíz, del objeto `user`) y algunos pueden actualizarse, importarse y exportarse, como se indica a continuación.

Si hay campos de usuario que Auth0 no debería almacenar por motivos de privacidad, puedes agregar a la Deny List los atributos que no quieres que se conserven en las bases de datos de Auth0. Para obtener más información, consulta [Agregar atributos de usuario a la Deny List](/docs/es/secure/security-guidance/data-security/denylist).

De forma predeterminada, los atributos del perfil de usuario proporcionados por <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente porque se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Para obtener más información sobre los atributos de los proveedores de identidad, consulta [Configurar la conexión del proveedor de identidad para actualizar el perfil de usuario](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Para poder editar los atributos raíz `name`, `nickname`, `given_name`, `family_name` o `picture` en el perfil de usuario normalizado, debes [configurar la sincronización de tu conexión con Auth0](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo al crear el perfil de usuario. Estos atributos raíz estarán entonces disponibles para editarse individualmente o mediante importaciones masivas

  ### Validación del perfil de usuario
</div>

Auth0 acepta un esquema de validación específico para ciertos campos del perfil de usuario. Consulta la siguiente tabla para ver la lista de requisitos al establecer valores para los atributos del perfil.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Todos los caracteres se convierten a minúsculas cuando se guardan en el perfil de usuario.

  Esto se aplica a todas las conexiones de base de datos de Auth0. Cuando Auth0 DB no es la fuente de referencia (por ejemplo, una base de datos personalizada con el modo de importación desactivado), así como en otros tipos de conexiones sociales y empresariales en los que Auth0 no es la fuente de referencia, el correo electrónico se conserva tal cual y no se convierte a minúsculas.
</Callout>

Otros tres campos no forman técnicamente parte del perfil de usuario, pero pueden ser de interés al importar usuarios:

* `password_hash` (text): Contraseña con hash para la conexión del usuario. Cuando se crean usuarios, Auth0 usa [bcrypt](https://auth0.com/blog/hashing-in-action-understanding-bcrypt/) para proteger la contraseña. Importar contraseñas con hash compatibles permite que los usuarios conserven sus contraseñas, lo que proporciona una experiencia más fluida. Las contraseñas compatibles deben tener hash con bcrypt {"$2a$"} o {"$2b$"} y tener 10 saltRounds. Ten en cuenta que solo puedes proporcionar este campo cuando el usuario se importa por primera vez; no puedes actualizarlo más adelante.
* `custom_password_hash` (object): Contraseña con hash para la conexión del usuario cuando se crea con un algoritmo alternativo. Se usa en lugar del campo `password_hash`. Durante el proceso de importación masiva, puedes actualizar `custom_password_hash` si el usuario no inició sesión con el `custom_password_hash` importado inicialmente.
