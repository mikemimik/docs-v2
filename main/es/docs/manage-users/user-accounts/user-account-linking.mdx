<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Connected Accounts permite vincular un único perfil de usuario de Auth0 con varias cuentas externas. Cuando habilitas Connected Accounts para un proveedor externo compatible, Auth0 añade automáticamente la cuenta de ese proveedor al perfil del usuario después de que este inicie sesión correctamente. Para obtener más información, consulta [Connected Accounts for Token Vault](/es/docs/secure/tokens/token-vault/connected-accounts-for-token-vault).
</Callout>

Auth0 admite la vinculación de cuentas de usuario de varios <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+providers">proveedores de identidad</Tooltip>. Esto permite que un usuario se autentique con cualquiera de sus cuentas y que tu aplicación lo siga reconociendo y asociando al mismo perfil de usuario.

<Card title="La disponibilidad varía según el plan de Auth0">
  Tanto la implementación concreta del inicio de sesión como tu plan de Auth0 o tu acuerdo personalizado influyen en si esta función está disponible. Para obtener más información, consulta [Pricing](https://auth0.com/pricing).
</Card>

Auth0 trata todas las identidades como separadas de forma predeterminada. Por ejemplo, si un usuario inicia sesión primero con la base de datos de Auth0 y luego a través de Google o Facebook, Auth0 interpretaría esos dos intentos como dos usuarios distintos. Puedes implementar una funcionalidad que permita a un usuario vincular cuentas de forma explícita. En este escenario, el usuario iniciaría sesión con un proveedor inicial, quizá Google. Tu aplicación proporcionaría un enlace o un botón para permitirle vincular otra cuenta a la primera. El usuario haría clic en ese enlace o botón y tu aplicación realizaría una llamada para que, cuando el usuario inicie sesión con el segundo proveedor, la segunda cuenta quede vinculada a la primera.

<div id="advantages-of-linking-accounts">
  ## Ventajas de vincular cuentas
</div>

* Permite a los usuarios iniciar sesión con cualquier proveedor de identidad sin crear un perfil independiente para cada uno
* Permite a los usuarios ya registrados usar un nuevo inicio de sesión social o <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=passwordless">passwordless</Tooltip> y seguir usando su perfil existente
* Permite a los usuarios que se registraron con un inicio de sesión passwordless vincularlo a una cuenta con un perfil más completo
* Permite que tus aplicaciones recuperen datos del perfil de usuario almacenados en varias conexiones

<div id="precautions">
  ## Precauciones
</div>

Vincular cuentas de forma insegura puede permitir que actores maliciosos accedan a cuentas de usuario legítimas. Ten en cuenta lo siguiente:

Tanto en la vinculación manual como en la automática de cuentas, tu tenant debe solicitar autenticación para **ambas** cuentas antes de que se produzca la vinculación.

Además, **toda** vinculación manual de cuentas debe pedir al usuario que introduzca sus credenciales.

<div id="how-it-works">
  ## Cómo funciona
</div>

El proceso de vinculación de cuentas fusiona dos perfiles de usuario existentes en uno solo. Al vincular cuentas, se debe especificar una **cuenta principal** y una **cuenta secundaria**.

En el siguiente ejemplo puedes ver cómo quedará el perfil vinculado resultante para las cuentas principal y secundaria de muestra.

Ten en cuenta que:

* El `user_id` y todas las demás propiedades principales del perfil siguen siendo las de la identidad principal
* La primera identidad del arreglo `user.identities` es la identidad principal
* La cuenta secundaria ahora está integrada en el arreglo `user.identities` del perfil principal
* Los atributos de la cuenta secundaria se colocan dentro del campo `profileData` de la identidad correspondiente dentro del arreglo
* Los `user_metadata` y `app_metadata` de la cuenta principal no han cambiado
* Los `user_metadata` y `app_metadata` de la cuenta secundaria se descartan
* No hay fusión automática de perfiles de usuario con identidades asociadas
* La cuenta secundaria se elimina de la lista de usuarios
* Si eliminas la cuenta principal, la cuenta secundaria también se elimina

<div id="metadata-merge">
  ### Fusión de metadatos
</div>

Los metadatos no se fusionan automáticamente durante la vinculación de cuentas. Si quieres fusionarlos, debes hacerlo manualmente mediante el endpoint [`/users/patch_users_by_id`](https://auth0.com/docs/api/v2#!/Users/patch_users_by_id) de la API de Management.

El [SDK de Auth0 para Node.js y APIv2](https://github.com/auth0/node-auth0/) también está disponible.

Para obtener más información, consulta [Comprende cómo funcionan los metadatos en los perfiles de usuario](/es/docs/manage-users/user-accounts/metadata)

<div id="scenarios">
  ## Escenarios
</div>

Hay dos formas distintas de implementar la vinculación de cuentas:

* Vinculación de cuentas iniciada por el usuario: permite a tus usuarios vincular sus cuentas desde una pantalla de administración en tu aplicación.
* Vinculación de cuentas sugerida: identifica las cuentas con la misma dirección de correo electrónico y solicita al usuario en tu aplicación que las vincule.

<div id="user-initiated-account-linking">
  ### Vinculación de cuentas iniciada por el usuario
</div>

Normalmente, un usuario autenticado iniciará la vinculación de cuentas. Tu aplicación debe incluir la UI, por ejemplo, un botón de **Vincular cuentas** en la página del perfil del usuario.

Para obtener más información sobre cómo implementar la vinculación de cuentas iniciada por el usuario en una Single Page Application, lee [Vinculación de cuentas del lado del cliente](/es/docs/manage-users/user-accounts/user-account-linking/user-initiated-account-linking-client-side-implementation). Para obtener más información sobre cómo implementar la vinculación de cuentas iniciada por el usuario en una Regular Web Application, lee [Vinculación de cuentas del lado del servidor](/es/docs/manage-users/user-accounts/user-account-linking/suggested-account-linking-server-side-implementation).

<div id="suggested-account-linking">
  ### Vinculación de cuentas sugerida
</div>

Puedes encontrar cuentas con el mismo correo electrónico y mostrarles a los usuarios un prompt para que las vinculen. Por ejemplo, un usuario puede crear una cuenta con Google con `user@gmail.com` y luego iniciar sesión con Facebook con una cuenta vinculada al mismo correo electrónico.

Si eso ocurre, puedes mostrar a los usuarios la lista de cuentas disponibles para que puedan vincularlas, autenticándose primero con la cuenta con la que las vincularán.

También puedes usar la [extensión Account Link](/es/docs/customize/extensions/account-link-extension) para lograr el mismo resultado.

<Warning>
  Anteriormente, en algunos casos, podías usar tokens de ID para vincular y desvincular cuentas de usuario. Esta funcionalidad está quedando obsoleta. Tendrás que usar tokens de acceso en todos los casos.
</Warning>
