---
description: Cómo ver a los usuarios y los detalles de su perfil.
title: Ver User Details
---

La página [Users](https://manage.auth0.com/#/users) muestra los usuarios asociados a tus aplicaciones. Para abrir un usuario en particular, haz clic en la foto de perfil o en el nombre del usuario en la columna &quot;Name&quot;. Se abrirá la página User Details y mostrará información de ese usuario.

La página User Details tiene enlaces a cinco pestañas:

1. Details
2. Devices
3. History
4. Locations
5. Raw JSON

<div id="user-details-details-tab">
  ## User Details: pestaña Details
</div>

La pestaña Details contiene tres secciones que ofrecen una visión general de alto nivel de la información del perfil del usuario:

* **Identidad del usuario** proporciona información del usuario de un vistazo, incluido el identificador que utilizó, las Connections asociadas y los derechos de acceso.
* **Metadata** muestra la información de `app_metadata` y `user_metadata`. Puedes editar estos valores.
* **<Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> Atributos** muestra la información recuperada del proveedor de autenticación. Ten en cuenta que los atributos del proveedor de identidad son de solo lectura.
* **Claves de acceso** muestra las [claves de acceso](/es/docs/authenticate/database-connections/passkeys) creadas por el usuario.
* **<Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación del usuario que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip>** muestra los factores de [MFA](/es/docs/secure/multi-factor-authentication) en los que está inscrito el usuario.
  Si habilitas [múltiples dominios personalizados](/es/docs/customize/custom-domains/multiple-custom-domains), tienes la opción **Domain** para seleccionar el dominio de notificación de tu tenant.

<div id="user-details-devices-tab">
  ## User Details: Pestaña Devices
</div>

La pestaña Devices enumera los dispositivos desde los que el usuario ha solicitado autenticación. Solicitar autorización en un dispositivo lo vincula con la cuenta del usuario.

Los datos de inicio de sesión del usuario están asociados con el <Tooltip tip="Token de actualización: token utilizado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/es/docs/glossary?term=Refresh+Token">token de actualización</Tooltip> asignado a ese dispositivo. Para revocar el token de actualización, haga clic en **Desvincular** junto al dispositivo.

<div id="user-details-history-tab">
  ## User Details: pestaña History
</div>

La pestaña History muestra un registro de la actividad de la cuenta del usuario de los últimos 2 días.

Los registros incluyen información sobre:

* Los eventos que ocurrieron
* Cuándo ocurrieron los eventos
* Las aplicaciones asociadas a los eventos
* El proveedor de identidad utilizado para la autenticación
* Las direcciones IP de origen de los eventos
* Dónde se originaron los eventos

<div id="user-details-locations-tab">
  ## User Details: pestaña Locations
</div>

La pestaña Locations muestra un mapa con marcadores que indican las ubicaciones desde las que el usuario inició sesión en las aplicaciones.

<div id="user-details-raw-json-tab">
  ## User Details: pestaña Raw JSON
</div>

La pestaña Raw JSON muestra toda la información contenida en el perfil del usuario en formato JSON, para que puedas consultar rápidamente toda la información disponible sobre ese usuario.
