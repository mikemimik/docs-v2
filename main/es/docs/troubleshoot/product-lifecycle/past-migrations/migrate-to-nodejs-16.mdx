---
description: Describe las funciones de Auth0 afectadas por la migración de Node.js v12 a Node.js v16 y proporciona recomendaciones para la migración.
title: Migrar a Node.js 16
---

El 30 de abril de 2022, [Node.js v12 dejó de contar con soporte a largo plazo (LTS)](https://github.com/nodejs/Release#release-schedule), lo que significa que el equipo de desarrollo de Node.js ya no incorpora de forma retroactiva correcciones críticas de seguridad para esta versión. Esto podría exponer su código de extensibilidad a vulnerabilidades de seguridad. Por lo tanto, Auth0 está migrando de Node 12 a Node 16.

<div id="recommendations">
  ## Recomendaciones
</div>

Aunque la actualización a Node 16 no introducirá ningún cambio incompatible en la biblioteca estándar de Node.js (Rules y los scripts de acción de Custom Database sí se ven afectados; consulte la sección [Cambios incompatibles: solo Rules y scripts de acción de Custom Database](#breaking-changes-rules-only)), recomendamos a los clientes que usan Node 12 mantenerse al día con las versiones de Node con soporte activo a largo plazo (LTS) por motivos de seguridad y cumplimiento normativo. Los clientes que aún usan Node 8 ya no cumplen con los requisitos de seguridad y deben migrar a Node 16 para eliminar riesgos de seguridad. Eliminamos el runtime de Node 8 el 22 de febrero de 2022 para los tenants de Public Cloud y en la versión de abril de 2022 de Private Cloud. Después de esas fechas, los tenants que sigan configurados con Node 8 corren el riesgo de sufrir una interrupción del servicio.

Actions está diseñado para admitir varias versiones de Node de forma simultánea y actualmente es compatible con Node 12 y Node 16. Además, las integraciones de Actions sin código seguirán un control de versiones de estándar del sector de &quot;siempre al día&quot;, con las versiones menores actualizadas automáticamente; el código subyacente se mantendrá siempre actualizado y en la versión más reciente de Node.js.

| Clientes que usan                       | Deben                                                                                                                                                                                                                        | Tarea                                                                         |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| Actions Node 12                         | Actualizar cualquier versión personalizada de Action existente a una nueva versión, que se actualizará a Node 16.                                                                                                            | Actualice todas las versiones personalizadas de Action lo antes posible.      |
| Legacy Node 12 (Rules/Hooks/Extensions) | Comenzar a migrar a Actions Node 16.                                                                                                                                                                                         | [Migrar a Actions Node 16](/es/docs/customize/actions/migrate) lo antes posible. |
| Legacy Node 8                           | Migrar de inmediato a Actions siempre que sea posible. En el caso de Rules y Hooks que no puedan migrarse de inmediato a Actions, debe, como mínimo, actualizar su código a Node 12 y la configuración del tenant a Node 12. | [Migrar a Actions Node 16](/es/docs/customize/actions/migrate) de inmediato.     |

<div id="tasks">
  ## Tareas
</div>

<div id="upgrade-actions">
  ### Actualizar Actions
</div>

Las Actions existentes creadas con Node 12 pueden actualizarse a Node 16 y, si es necesario, revertirse después a una versión anterior con Node 12. Actualiza las Actions a Node 16 [creando e implementando una nueva versión](/es/docs/customize/actions/manage-versions) configurada para usar Node 16 como runtime.

<div id="migrate-rules-and-hooks-to-actions">
  ### Migrar Rules y Hooks a Actions
</div>

Cuando sea posible, migra Rules y Hooks a Actions. Para determinar qué Rules y Hooks se pueden migrar, lee [Actions Limitations](/es/docs/customize/actions/limitations). Para saber cómo migrar tus Rules y Hooks a Actions, consulta nuestras guías disponibles en [Migrate to Actions](/es/docs/customize/actions/migrate).

<div id="enable-node-16-runtime">
  ### Habilitar el runtime de Node 16
</div>

Si utilizas algún producto de extensibilidad heredado, como Rules, Hooks o Extensions, debes habilitar el runtime de Node 16 a nivel del tenant. Te recomendamos cambiar primero tu tenant de desarrollo al runtime de Node 16, probar la configuración y, después, cambiar tu tenant de producción.

1. Habilita Node 16 en tu tenant de desarrollo desde el panel de **extensibilidad** de la página [Configuración avanzada del tenant](https://manage.auth0.com/#/tenant/advanced) del Dashboard. Selecciona **Node 16** en la lista desplegable **Runtime**.
2. Haz clic en **Guardar**.
3. Comprueba que no tengas los elementos que se indican a continuación o que hayas completado los pasos de migración de cada uno.
4. Prueba tu configuración. Cuando termines las pruebas, repite los pasos 1 y 2 anteriores con tu tenant de producción para habilitar Node 16 en producción.

<div id="breaking-changes-rules-and-custom-database-action-scripts-only">
  ### Cambios incompatibles: solo para Rules y scripts de acción de Custom Database
</div>

Rules se ejecuta en un sandbox de JavaScript. El sandbox admite el lenguaje ECMAScript 6 y una gran cantidad de módulos de Node.js (versión 16+). Si usa alguna de estas propiedades globales dentro de Rules o scripts de acción de Custom Database, [busque una alternativa en nuestra lista de módulos de sandbox compatibles](https://auth0-extensions.github.io/canirequire/) e impórtela explícitamente con `require`.

* `_`
* `async`
* `Auth0`
* `azure_storage`
* `bcrypt`
* `crypto`
* `couchbase`
* `cql`
* `ip`
* `jwt`
* `Knex`
* `mongo`
* `mysql`
* `mysql_pool`
* `ObjectID`
* `pbkdf2`
* `pg`
* `postgres`
* `Pubnub`
* `q`
* `querystring`
* `request`
* `sqlserver`
* `uuid`
* `xml2js`
* `xmldom`
* `xpath`
* `xtend`
