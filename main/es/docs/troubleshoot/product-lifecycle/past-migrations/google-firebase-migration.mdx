title: Migrar de Google Cloud Messaging a Firebase Cloud Messaging
---

A partir del 11 de abril de 2019, [Google declaró obsoleto](https://firebase.googleblog.com/2018/04/time-to-upgrade-from-gcm-to-fcm.html) Google Cloud Messaging (GCM) y lo reemplazó por Firebase Cloud Messaging (FCM).

<div id="features-affected">
  ## Funcionalidades afectadas
</div>

Los SDK de Guardian de Auth0 para iOS y Android te ayudan a crear aplicaciones móviles personalizadas con funcionalidades de Guardian, y ofrecen acceso seguro a la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) con notificaciones push.

La biblioteca Guardian Android SDK de Auth0 se creó para enviar notificaciones push mediante GCM. Ten en cuenta que las aplicaciones existentes deberían [seguir funcionando tal cual](https://aws.amazon.com/blogs/messaging-and-targeting/the-end-of-google-cloud-messaging-and-what-it-means-for-your-apps/).

<div id="actions">
  ## Actions
</div>

Consulta la [documentación de Google](https://developers.google.com/cloud-messaging/android/android-migrate-fcm) para saber cómo migrar de GCM a FCM.

La principal diferencia entre cómo se envían las notificaciones a GCM y a FCM está en el payload que se recibe en la notificación. Aunque los clientes existentes que usaban el SDK de Android podían adaptar el payload recibido antes de llamar al método del SDK, hemos actualizado la biblioteca para que acepte el nuevo payload, lo que simplifica la adopción de FCM. Más detalles [aquí](https://github.com/auth0/Guardian.Android/pull/84).

La versión 0.4.0 de Guardian Android SDK ya está disponible en Maven Central e incluye este cambio. La aplicación de ejemplo también se actualizó, por lo que puede probarse proporcionando el archivo `google-services.json` y un `guardian-url`.
