---
description: Describe los cambios en la API de Yahoo que afectan a la estructura del perfil de usuario.
title: Cambios en la API de Yahoo
---

Yahoo cambió la API que las aplicaciones deben usar para recuperar el perfil de usuario: pasó de su [API de Social Directory](https://developer.yahoo.com/oauth/social-directory-eol/) a un endpoint `/userinfo` de Yahoo. Este cambio implica que la estructura del perfil de usuario de los usuarios de Yahoo en Auth0 cambiará.

Anteriormente, Auth0 cargaba todos los datos del perfil que devolvía Yahoo y añadía estos campos adicionales, que se asignaban a partir del perfil de Yahoo.

Yahoo dejó de devolver `url`, `profileUrl`, `isConnected` y otros campos que se enumeran en la [documentación de Yahoo](https://developer.yahoo.com/oauth/social-directory-eol/) (consulta &quot;List Of Attributes Deprecated in Social Directory Profile Api&quot;). Esos otros campos tampoco formarán parte del perfil.

El endpoint `/userinfo` de Yahoo devuelve distintos campos según los permisos de la API que configures en la definición de la [aplicación de Yahoo](https://developer.yahoo.com/apps/). Yahoo te permite conceder uno de estos cuatro permisos en la sección de permisos **Profile (Social Directory)**:

* Lectura pública básica
* Lectura pública ampliada
* Lectura y escritura públicas
* Lectura y escritura públicas y privadas

<div id="actions">
  ## Actions
</div>

Al configurar la conexión de Yahoo en tu <Tooltip tip="Auth0 Dashboard: El principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, selecciona el atributo que corresponda a los permisos que concediste en la configuración de Yahoo. Si eliges un atributo que no coincide con lo que especificaste en Yahoo, la transacción de inicio de sesión fallará.

Si no seleccionas ningún permiso en la configuración de la conexión de Auth0, Auth0 solicitará de forma predeterminada el scope `openid`, que devuelve los campos del perfil correspondientes al permiso de API que especificaste en la aplicación de Yahoo. Por ejemplo, si tu aplicación de Yahoo está configurada con el permiso **Read Public Extended**, Yahoo devuelve los siguientes campos:

* `sub, name`
* `given_name`
* `family_name`
* `locale`
* `email`
* `email_verified`
* `birthdate`
* `profile_images`
* `picture`
* `preferred_username`
* `phone_number`
* `nickname`

Si usas la conexión de Yahoo para autenticar usuarios y obtener su información básica, tu aplicación seguirá funcionando sin cambios.

Si tu aplicación accede a campos del perfil de usuario que ya no están disponibles, tendrás que habilitar el **Atributo de conexión** correcto en el Auth0 Dashboard y ajustar el código de tu aplicación para usar los nombres de campo correctos.
