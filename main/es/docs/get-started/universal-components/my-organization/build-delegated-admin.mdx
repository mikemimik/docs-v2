title: Crea una interfaz de Administración delegada con My Organization API
description: Describe cómo usar Universal Components para crear interfaces de administración delegada con My Organization API de Auth0.
sidebarTitle: Crea una interfaz de Administración delegada
---

Universal Components te permite crear un Dashboard de administración de autoservicio dentro de tu propia aplicación. Esto permite que tus clientes B2B, a quienes has delegado como administradores de tu [Auth0 Organization](/es/docs/manage-users/organizations), gestionen su propia marca, proveedores de identidad y dominios verificados sin necesidad de acceder a Auth0 Dashboard ni de recurrir a un equipo de soporte para realizar tareas administrativas básicas.

<div id="how-it-works">
  ## Cómo funciona
</div>

Universal Components aprovecha la [My Organization API](/es/docs/api/myorganization), que opera con tokens de acceso de ámbito de usuario, para crear un [Dashboard administrativo de SSO de autoservicio](/es/docs/authenticate/enterprise-connections/self-service-enterprise-configuration). Cuando un administrador inicia sesión en el Dashboard integrado en tu aplicación, el SDK de Auth0 recupera un Access Token que contiene scopes específicos de la organización para privilegios administrativos. Los componentes usan este token para llamar a la API en nombre del usuario que ha iniciado sesión, lo que garantiza que los administradores delegados solo puedan modificar la Auth0 Organization específica a la que pertenecen.

<div id="available-components">
  ### Componentes disponibles
</div>

Cada componente se asigna a endpoints específicos de la [My Organization API](/es/docs/api/myorganization) para realizar tareas administrativas.

| **Componente**                                                                                                                                       | **endpoint de la API**                |
| :--------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------ |
| **OrganizationDetailsEdit** — Edita el nombre de la organización, el nombre visible, los colores de la marca y el logotipo.                          | `/my-org/details`                     |
| **SsoProviderTable** — Enumera, habilita, deshabilita, elimina y desvincula proveedores de identidad de SSO.                                         | `/my-org/identity-providers`          |
| **SsoProviderCreate** — Asistente de varios pasos para crear proveedores de SSO (Okta, ADFS, SAML, OIDC, Google Workspace, Azure AD, Ping Federate). | `/my-org/identity-providers`          |
| **SsoProviderEdit** — Configura los ajustes de autenticación, habilita el aprovisionamiento y gestiona tokens SCIM y asociaciones de dominios.       | `/my-org/identity-providers/{idp_id}` |
| **DomainTable** — Crea, verifica, elimina y asocia dominios con proveedores de identidad.                                                            | `/my-org/domains`                     |

<div id="configure-auth0-dashboard">
  ## Configura Auth0 Dashboard
</div>

Antes de usar cualquiera de los componentes de My Organization, debes configurar tu tenant de Auth0 con las [APIs](/es/docs/api), las aplicaciones y los [scopes](/es/docs/api/myorganization) correspondientes.

  <Tab title="React">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">

    ## **Crear aplicación**

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Este ejemplo usa una aplicación de una sola página (SPA). Para los frameworks de React con renderizado del lado del servidor, consulta la pestaña de Next.js.
    </Callout>

    1. Ve a [**Auth0 Dashboard &gt; Applications &gt; Applications**](https://manage.auth0.com/#/applications). Haz clic en **Create Application**.

    2. Selecciona **Single Page Web Applications**.

    3. En la pestaña **Settings**, agrega `http://localhost:5173` en los siguientes campos:
       <br />A. Allowed Callback URLs
       <br />B. Allowed Logout URLs

    4. En **Experiencia de inicio de sesión**, selecciona:
       <br />A. Usuarios de empresa.
       <br />B. (Opcional) Prompt para organización.

    ## **Habilitar la API My Organization**

    1. Ve a [**Dashboard &gt; Applications &gt; APIs**](https://manage.auth0.com/#/applications).
    2. Selecciona **[My Organization API](/es/docs/api/myorganization)** y asegúrate de que esté habilitada para tu tenant.

    ## **Configurar el acceso a la aplicación**

    1. Ve a la pestaña **Acceso a la aplicación**.
    2. Selecciona **Editar** en la aplicación que quieres usar.
    3. Configure los siguientes ajustes:
       <br />A. [Perfil de conexión](/es/docs/authenticate/enterprise-connections/connection-profile#connection-profile): Seleccione o cree un perfil con mapeo de atributos de conexión.
       <br />B. [User Attribute Profile](/es/docs/authenticate/enterprise-connections/user-attribute-profile#user-attribute-profile): Seleccione o cree un perfil con mapeo de atributos de usuario.
       <br />C. [Proveedores de identidad](/es/docs/authenticate/identity-providers) compatibles: Habilite los proveedores que sus clientes pueden usar.
       <br />D. Comportamiento de eliminación de la conexión: Elija **Allow** o **Allow if Empty**.
       <br />    • **Allow**: Los usuarios pueden eliminar conexiones, lo que elimina a todos los usuarios de esa conexión.
       <br />    • **Allow if Empty**: Los usuarios solo pueden eliminar conexiones que no tengan usuarios.
       <br />E. Autorización de acceso de usuarios: Elija **Unauthorized**, **Authorized** o **All**.
       <br />    • **Unauthorized**: No se permite ningún permiso.
       <br />    • **Authorized**: Seleccione permisos específicos.
       <br />    • **All**: Incluye todos los permisos actuales y futuros.
       <br />F. Autorización de acceso con credenciales de cliente: Elija **Unauthorized**, **Authorized** o **All**.
    4. Selecciona **Guardar**.

    ## **Setup de base de datos y usuario**

    1. Ve a [**Auth0 Dashboard &gt; Autenticación &gt; Base de datos**](https://manage.auth0.com/#/connections/database) para crear una conexión de base de datos.
    2. En la pestaña **Applications** de la conexión, habilita tu nueva aplicación SPA.
    3. Crea un usuario de prueba en esta base de datos para hacer las pruebas iniciales.

    ## **Setup Role**

    1. Cree un rol (p. ej., &quot;Administrador de la organización&quot;).
    2. Agregue los scopes requeridos de la API My Organization al rol.

    <Accordion title="Scopes requeridos">
      ```
          read:my_org:details
          update:my_org:details
          create:my_org:identity_providers
          read:my_org:identity_providers
          update:my_org:identity_providers
          delete:my_org:identity_providers
          update:my_org:identity_providers_detach
          create:my_org:identity_providers_domains
          delete:my_org:identity_providers_domains
          read:my_org:domains
          delete:my_org:domains
          create:my_org:domains
          update:my_org:domains
          read:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_scim_tokens
          delete:my_org:identity_providers_scim_tokens
          create:my_org:identity_providers_provisioning
          read:my_org:identity_providers_provisioning
          delete:my_org:identity_providers_provisioning
          read:my_org:configuration
      ```
    </Accordion>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      El token del usuario solo incluirá los permisos que existan tanto en el rol que tenga asignado como en la configuración de User Access Authorization definida en el paso anterior.
    </Callout>

    ## **Crear organización**

    1. Vaya a [**Organizations &gt; Crear organización**](https://manage.auth0.com/#/organizations).
    2. Configure la Organización de Auth0:
       <br />A. En **Members**: añada su usuario de prueba y asígnele el rol &quot;Organization Admin&quot;.
       <br />B. En **Connections**: habilite su conexión de base de datos.

    ## **Configurar variables de entorno**

    1. Crea un archivo `.env` en la raíz de tu proyecto React:

    ```bash
    VITE_AUTH0_DOMAIN=your-domain.auth0.com
      VITE_AUTH0_CLIENT_ID=your-spa-client-id
    ```
