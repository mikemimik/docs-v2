    **createAction**

    **Type:** `ComponentAction<void>`

    La prop `createAction` es obligatoria porque controla a dónde se redirige a los usuarios cuando hacen clic en &quot;Add Provider&quot;. Sin ella, la tabla no sabría cómo iniciar el flujo de creación del proveedor.

    **Propiedades:**

    * `disabled` - Desactiva el botón &quot;Añadir Provider&quot;
    * `onBefore()` - Se ejecuta antes de que se produzca la navegación. Devuelve `false` para impedirla (p. ej., si el usuario no tiene permisos).
    * `onAfter()` - Se ejecuta después de que `onBefore` se complete correctamente. Úsalo para ir a tu página de creación o hacer seguimiento de analíticas.

    **Ejemplo:**

    ```tsx
    // Navegar a la página de creación
    createAction={{
      onAfter: () => navigate("/providers/create"),
    }}

    // Verificar permisos antes de permitir la creación
    createAction={{
      onBefore: () => {
        if (!hasPermission("create:providers")) {
          alert("You don't have permission to create providers");
          return false;
        }
        return true;
      },
      onAfter: () => navigate("/providers/create"),
    }}

    // Registrar analíticas al intentar crear
    createAction={{
      onAfter: () => {
        analytics.track("Create Provider Started");
        navigate("/providers/create");
      },
    }}
    ```

    ***

    **editAction**

    **Type:** `ComponentAction<IdentityProvider>`

    La prop `editAction` es obligatoria porque controla a dónde se redirige a los usuarios cuando hacen clic en una fila de proveedor. La devolución de llamada recibe los datos del proveedor para que puedas navegar a la página de edición correspondiente.

    **Propiedades:**

    * `disabled` - Desactiva la navegación al hacer clic en la fila
    * `onBefore(provider)` - Se ejecuta antes de que se produzca la navegación. Devuelve `false` para impedirla (p. ej., para comprobaciones de acceso condicional).
    * `onAfter(provider)` - Se ejecuta después de que `onBefore` se complete correctamente. Úsalo para ir a la página de edición con los datos del Provider.

    **Ejemplo:**

    ```tsx
    // Navegar a la página de edición con el ID del provider
    editAction={{
      onAfter: (provider) => navigate(`/providers/${provider.id}`),
    }}

    // Registrar analíticas al seleccionar un provider
    editAction={{
      onAfter: (provider) => {
        analytics.track("Provider Selected", {
          id: provider.id,
          name: provider.name,
          strategy: provider.strategy,
        });
        navigate(`/providers/${provider.id}`);
      },
    }}
    ```

    **deleteAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla la eliminación permanente de un proveedor de SSO. Esta acción es destructiva: el proveedor se elimina por completo de tu tenant de Auth0.

    **Propiedades:**

    * `disabled` - Desactiva la opción de eliminar
    * `onBefore(provider)` - Se ejecuta antes de la eliminación. Devuelve `false` para impedirla (se recomienda para los cuadros de diálogo de confirmación).
    * `onAfter(provider)` - Se ejecuta después de que el Provider se haya eliminado correctamente. Úsalo para registrar el evento o mostrar una notificación.

    **Ejemplo:**

    ```tsx
    <SsoProviderTable
      createAction={{ onAfter: () => navigate("/providers/create") }}
      editAction={{ onAfter: (p) => navigate(`/providers/${p.id}`) }}
      deleteAction={{
        onBefore: (provider) => {
          return confirm(
            `Permanently delete "${provider.display_name}"? This cannot be undone.`,
          );
        },
        onAfter: (provider) => {
          analytics.track("Provider Deleted", { name: provider.name });
          toast.success("Provider deleted");
        },
      }}
    />
    ```

    ***

    **deleteFromOrganizationAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla la eliminación de un proveedor de la organización sin eliminarlo del tenant. El proveedor seguirá disponible para volver a añadirse más adelante.

    **Propiedades:**

    * `disabled` - Desactiva la opción de quitar
    * `onBefore(provider)` - Se ejecuta antes de quitarlo. Devuelve `false` para impedirlo (p. ej., para mostrar una confirmación).
    * `onAfter(provider)` - Se ejecuta después de que el proveedor se haya quitado correctamente de la organización.

    **Ejemplo:**

    ```tsx
    <SsoProviderTable
      createAction={{ onAfter: () => navigate("/providers/create") }}
      editAction={{ onAfter: (p) => navigate(`/providers/${p.id}`) }}
      deleteFromOrganizationAction={{
        onBefore: (provider) => {
          return confirm(
            `Remove "${provider.display_name}" from this organization?`,
          );
        },
        onAfter: (provider) => {
          toast.success(`${provider.display_name} removed from organization`);
        },
      }}
    />
    ```

    ***

    **enableProviderAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla el interruptor de activación/desactivación de los providers. Los providers desactivados permanecen configurados, pero los usuarios no pueden autenticarse a través de ellos.

    **Propiedades:**

    * `disabled` - Desactiva el interruptor
    * `onBefore(provider)` - Se ejecuta antes de activar o desactivar el interruptor. Devuelve `false` para evitar el cambio de estado.
    * `onAfter(provider)` - Se ejecuta después de que el proveedor se haya habilitado o deshabilitado correctamente.

    **Ejemplo:**

    ```tsx
    <SsoProviderTable
      createAction={{ onAfter: () => navigate("/providers/create") }}
      editAction={{ onAfter: (p) => navigate(`/providers/${p.id}`) }}
      enableProviderAction={{
        onBefore: (provider) => {
          if (!provider.is_enabled) {
            return confirm(`Enable "${provider.display_name}" for authentication?`);
          }
          return confirm(
            `Disable "${provider.display_name}"? Users won't be able to authenticate.`,
          );
        },
        onAfter: (provider) => {
          analytics.track(
            provider.is_enabled ? "Provider Enabled" : "Provider Disabled",
            {
              name: provider.name,
            },
          );
        },
      }}
    />
    ```

    **deleteAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla la eliminación permanente de un proveedor de SSO. Esta operación es destructiva: el proveedor se elimina por completo de tu tenant de Auth0.

    **Propiedades:**

    * `disabled` - Desactiva la opción de eliminar
    * `onBefore(provider)` - Se ejecuta antes de la eliminación. Devuelve `false` para impedirla (se recomienda para cuadros de diálogo de confirmación).
    * `onAfter(provider)` - Se ejecuta después de que el proveedor se haya eliminado correctamente. Úsalo para registrar el evento o mostrar una notificación.

    **Ejemplo:**

    ```tsx

      deleteAction={{
        onBefore: (provider) => {
          return confirm(
            `Permanently delete "${provider.display_name}"? This cannot be undone.`,
          );
        },
        onAfter: (provider) => {
          analytics.track("Provider Deleted", { name: provider.name });
          toast.success("Provider deleted");
        },
      }}
    />
    ```

    **deleteFromOrganizationAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla la eliminación de un proveedor de la organización sin eliminarlo del tenant. El proveedor seguirá disponible para volver a añadirse más adelante.

    * `onBefore(provider)` - Se ejecuta antes de la eliminación. Devuelve `false` para impedirla (por ejemplo, para mostrar una confirmación).

    **enableProviderAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla el interruptor de activación/desactivación de los proveedores. Los proveedores desactivados permanecen configurados, pero los usuarios no pueden autenticarse a través de ellos.

    **Propiedades:**

    * `disabled` - Deshabilita el interruptor
    * `onBefore(provider)` - Se ejecuta antes de activar o desactivar el interruptor. Devuelve `false` para impedir el cambio de estado.
    * `onAfter(provider)` - Se ejecuta después de que el proveedor se haya habilitado o deshabilitado correctamente.

    **Ejemplo:**

    ```tsx
    <SsoProviderTable
      createAction={{ onAfter: () => router.push("/providers/create") }}
      editAction={{ onAfter: (p) => router.push(`/providers/${p.id}`) }}
      enableProviderAction={{
        onBefore: (provider) => {
          if (!provider.is_enabled) {
            return confirm(`Enable "${provider.display_name}" for authentication?`);
          }
          return confirm(
            `Disable "${provider.display_name}"? Users won't be able to authenticate.`,
          );
        },
        onAfter: (provider) => {
          analytics.track(
            provider.is_enabled ? "Provider Enabled" : "Provider Disabled",
            {
              name: provider.name,
            },
          );
        },
      }}
    />
    ```

    **deleteAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla la eliminación permanente de un proveedor de SSO. Esta acción es destructiva: el proveedor se elimina por completo de tu tenant de Auth0.

    **Propiedades:**

    * `disabled` - Desactiva la opción de eliminación
    * `onBefore(provider)` - Se ejecuta antes de la eliminación. Devuelve `false` para evitarla (recomendado para diálogos de confirmación).
    * `onAfter(provider)` - Se ejecuta después de que el proveedor se haya eliminado correctamente. Úsalo para registrar el evento o mostrar una notificación.

    **Ejemplo:**

    ```tsx
    <SsoProviderTable
      createAction={{ onAfter: () => navigate("/providers/create") }}
      editAction={{ onAfter: (p) => navigate(`/providers/${p.id}`) }}
      deleteAction={{
        onBefore: (provider) => {
          return confirm(
            `Permanently delete "${provider.display_name}"? This cannot be undone.`,
          );
        },
        onAfter: (provider) => {
          analytics.track("Provider Deleted", { name: provider.name });
          toast.success("Provider deleted");
        },
      }}
    />
    ```

    ***

    **deleteFromOrganizationAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla la eliminación de un proveedor de la organización sin eliminarlo del tenant. El proveedor seguirá disponible para volver a añadirse más adelante.

    **Propiedades:**

    * `disabled` - Desactiva la opción de eliminar
    * `onBefore(provider)` - Se ejecuta antes de eliminarlo. Devuelve `false` para impedir la eliminación (p. ej., para mostrar una confirmación).
    * `onAfter(provider)` - Se ejecuta después de que el proveedor se elimine correctamente de la organización.

    **Ejemplo:**

    ```tsx
    <SsoProviderTable
      createAction={{ onAfter: () => navigate("/providers/create") }}
      editAction={{ onAfter: (p) => navigate(`/providers/${p.id}`) }}
      deleteFromOrganizationAction={{
        onBefore: (provider) => {
          return confirm(
            `Remove "${provider.display_name}" from this organization?`,
          );
        },
        onAfter: (provider) => {
          toast.success(`${provider.display_name} removed from organization`);
        },
      }}
    />
    ```

    ***

    **enableProviderAction**

    **Type:** `ComponentAction<IdentityProvider>`

    Controla el interruptor de activación/desactivación de los proveedores. Los proveedores desactivados permanecen configurados, pero los usuarios no pueden autenticarse a través de ellos.

    **Propiedades:**

    * `disabled` - Desactiva el interruptor
    * `onBefore(provider)` - Se ejecuta antes de cambiar el interruptor. Devuelve `false` para evitar el cambio de estado.
    * `onAfter(provider)` - Se ejecuta después de que el proveedor se haya habilitado o deshabilitado correctamente.

    **Ejemplo:**

    ```tsx
    <SsoProviderTable
      createAction={{ onAfter: () => navigate("/providers/create") }}
      editAction={{ onAfter: (p) => navigate(`/providers/${p.id}`) }}
      enableProviderAction={{
        onBefore: (provider) => {
          if (!provider.is_enabled) {
            return confirm(`Enable "${provider.display_name}" for authentication?`);
          }
          return confirm(
            `Disable "${provider.display_name}"? Users won't be able to authenticate.`,
          );
        },
        onAfter: (provider) => {
          analytics.track(
            provider.is_enabled ? "Provider Enabled" : "Provider Disabled",
            {
              name: provider.name,
            },
          );
        },
      }}
    />
    ```
