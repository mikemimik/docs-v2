`Auth0ComponentProvider` es la capa de orquestación de [Auth0 Universal Components](/es/docs/get-started/universal-components/universal-components-overview). Auth0 SDKs gestionan las sesiones y los tokens, y `Auth0ComponentProvider` traduce ese estado de identidad en un contexto de interfaz funcional y acorde con la marca. `Auth0ComponentProvider` se asegura de que los componentes, como el registro en MFA, tengan los permisos, los datos en caché y el estilo necesarios.

<div id="benefits">
  ## Beneficios
</div>

`Auth0ComponentProvider` es el contenedor raíz obligatorio para todos los Auth0 Universal Components. Los beneficios de usar este contenedor son:

* **Alineación de identidad**: Sirve de puente entre los [Auth0 SDKs](/es/docs/libraries) y la [My Account API](/es/docs/manage-users/my-account-api) para garantizar que las solicitudes se firmen con tokens con scope de usuario.
* **Optimización del rendimiento**: Implementa una caché compartida de **TanStack Query**, ajustada específicamente para workflows de identidad, para evitar llamadas redundantes a la API y saltos de diseño.
* **Consistencia del sistema de diseño**: Propaga variables de **Tailwind CSS** y temas compatibles con **Shadcn** por todo el árbol de componentes.
* **Retroalimentación global**: Gestiona un sistema unificado de notificaciones toast para alertas de seguridad y el estado del workflow.

<div id="integration-architecture">
  ## Arquitectura de integración
</div>

Coloca `Auth0ComponentProvider` dentro de tu proveedor de autenticación (`Auth0Provider`). Esta configuración permite acceder al estado de autenticación y a los métodos de obtención de token necesarios para llamar a las API de Auth0.

<div id="configure-auth0componentprovider">
  ## Configura `Auth0ComponentProvider`
</div>

<Tabs>
  <Tab title="React">
    ```tsx App.tsx
    import { Auth0Provider } from "@auth0/auth0-react";
    import { Auth0ComponentProvider } from "@auth0/universal-components-react/spa";

    function App() {
      return (
        <Auth0Provider
          domain="your-tenant.auth0.com"
          clientId="YOUR_CLIENT_ID"
          authorizationParams={{ redirect_uri: window.location.origin }}
          interactiveErrorHandler='popup' // Requerido para gestionar desafíos de autenticación escalonada mediante el popup de Universal Login
        >
          <Auth0ComponentProvider domain="your-tenant.auth0.com">
            {/* Tu aplicación con Auth0 Universal Components */}
          </Auth0ComponentProvider>
        </Auth0Provider>
      );
    }
    ```

    **toastSettings**

    La configuración de Toast admite dos tipos de proveedor: **Sonner** (predeterminado) o **custom**. Cada proveedor tiene su propia estructura de configuración para una mayor seguridad de tipos.

    <Tabs>
      <Tab title="Proveedor de Sonner (Predeterminado)">
        | Propiedad              | Tipo            | Predeterminado | Descripción                                                                                                                                                                               |
        | ---------------------- | --------------- | -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
        | `provider`             | `"sonner"`      | `"sonner"`     | Usa la biblioteca de toasts Sonner integrada                                                                                                                                              |
        | `settings.position`    | `ToastPosition` | `"top-right"`  | Posición en la que aparecen los toasts: &quot;top-left&quot;, &quot;top-right&quot;, &quot;bottom-left&quot;, &quot;bottom-right&quot;, &quot;top-center&quot;, &quot;bottom-center&quot; |
        | `settings.duration`    | `number`        | `4000`         | Duración en milisegundos antes de que el toast se cierre automáticamente (valor predeterminado de Sonner)                                                                                 |
        | `settings.maxToasts`   | `number`        | -              | Número máximo de toasts visibles a la vez                                                                                                                                                 |
        | `settings.dismissible` | `boolean`       | `true`         | Si los toasts pueden cerrarse manualmente mediante la interacción del usuario (valor predeterminado de Sonner)                                                                            |
        | `settings.closeButton` | `boolean`       | `true`         | Si se muestra un botón de cierre explícito en los toasts                                                                                                                                  |

        ```tsx Sonner Provider Example
        const toastSettings = {
          provider: 'sonner', // Opcional, este es el valor predeterminado
          settings: {
            position: 'top-center',
            duration: 6000,
            maxToasts: 5,
            dismissible: true,
            closeButton: true
          }
        };
        ```
      </Tab>

      <Tab title="Proveedor Custom">
        | Propiedad         | Tipo                         | Obligatorio | Descripción                                          |
        | ----------------- | ---------------------------- | ----------- | ---------------------------------------------------- |
        | `provider`        | `"custom"`                   | Sí          | Usa tu implementación personalizada de toast         |
        | `methods.success` | `(message: string) => void`  | No          | Controlador personalizado para toasts de éxito       |
        | `methods.error`   | `(message: string) => void`  | No          | Controlador personalizado para toasts de error       |
        | `methods.warning` | `(message: string) => void`  | No          | Controlador personalizado para toasts de advertencia |
        | `methods.info`    | `(message: string) => void`  | No          | Controlador personalizado para toasts informativos   |
        | `methods.dismiss` | `(toastId?: string) => void` | No          | Controlador personalizado para cerrar                |

        ```tsx Custom Provider Example
        const toastSettings = {
          provider: 'custom',
          methods: {
            success: (message: string) => {
              // Tu implementación personalizada de toast de éxito
              myToastLibrary.success(message);
            },
            error: (message: string) => {
              // Tu implementación personalizada de toast de error
              myToastLibrary.error(message);
            },
            warning: (message: string) => {
              // Tu implementación personalizada de toast de advertencia
              myToastLibrary.warning(message);
            },
            info: (message: string) => {
              // Tu implementación personalizada de toast informativo
              myToastLibrary.info(message);
            },
            dismiss: (toastId?: string) => {
              // Tu implementación personalizada para cerrar
              myToastLibrary.dismiss(toastId);
            }
          }
        };
        ```

    ## Estado y rendimiento

    Ajusta con precisión el almacenamiento en caché de TanStack Query para cada componente de Auth0. Los valores predeterminados mantienen los datos actualizados durante dos minutos, los recopilan como basura tras cinco minutos y omiten las recargas al enfocar la ventana.

    <table class="table">
      <thead>
        <tr>
          <th>Propiedad</th>
          <th>Tipo</th>
          <th>Predeterminado</th>
          <th>Descripción</th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            <code>enabled</code>
          </td>

          <td>
            <code>boolean</code>
          </td>

          <td>
            <code>true</code>
          </td>

          <td>
            Activa o desactiva por completo el almacenamiento en caché. Cuando se establece en <code>false</code>, se desactiva el uso de datos obsoletos y las entradas en caché se eliminan rápidamente.
          </td>
        </tr>

        <tr>
          <td>
            <code>staleTime</code>
          </td>

          <td>
            <code>number</code>
          </td>

          <td>
            <code>120000</code>
          </td>

          <td>
            Milisegundos antes de que los datos pasen a estar obsoletos. El valor predeterminado es de dos minutos. Auméntalo para paneles y redúcelo para flujos de trabajo críticos.
          </td>
        </tr>

        <tr>
          <td>
            <code>gcTime</code>
          </td>

          <td>
            <code>number</code>
          </td>

          <td>
            <code>300000</code>
          </td>

          <td>
            Milisegundos antes de que las consultas inactivas se eliminen mediante recolección de basura. El valor predeterminado es de cinco minutos.
          </td>
        </tr>

        <tr>
          <td>
            <code>refetchOnWindowFocus</code>
          </td>

          <td>
            <code>boolean | &quot;always&quot;</code>
          </td>

          <td>
            <code>false</code>
          </td>

          <td>
            Controla si las consultas se vuelven a obtener cuando el navegador recupera el foco. Usa{" "}
            <code>&quot;always&quot;</code> para garantizar la máxima frescura.
          </td>
        </tr>
      </tbody>
    </table>

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      **Desactivar el almacenamiento en caché:** Pasa `{ enabled: false }`. Esto establece automáticamente `staleTime` en 0 y reduce la ventana de recolección de basura a cinco segundos, de modo que cada renderizado obtenga datos actualizados.

      **Consejo:** Mantén activado el almacenamiento en caché, pero reduce `staleTime` al integrarlo con paneles de administración que requieren actualizaciones casi en tiempo real.
    </Callout>

    <Tabs>
      <Tab title="Configurar TTL">
        ```tsx
        <Auth0ComponentProvider
          domain="your-tenant.auth0.com"
          cacheConfig={{
            staleTime: 10 * 60 * 1000,
            gcTime: 15 * 60 * 1000,
            refetchOnWindowFocus: true,
          }}
        >
          <App />
        </Auth0ComponentProvider>
        ```
      </Tab>

      <Tab title="Desactivar la caché">
        ```tsx

          domain="your-tenant.auth0.com"
          cacheConfig={{ enabled: false }}
        >
          <App />
        </Auth0ComponentProvider>
        ```

    ## Configurar el modo proxy

    Para aplicaciones de Next.js, usa el modo proxy con autenticación del lado del servidor:

    ```tsx layout.tsx
    import { Auth0ComponentProvider } from "@auth0/universal-components-react/rwa";

    export default function RootLayout({ children }) {
      return (
        <html lang="en">
          <body>
            <Auth0ComponentProvider
              mode="proxy"
              domain="your-tenant.auth0.com"
              proxyConfig={{ baseUrl: "/api/auth" }}
            >
              {children}
            </Auth0ComponentProvider>
          </body>
        </html>

    }
    ```
