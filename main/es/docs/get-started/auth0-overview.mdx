title: Información general de Auth0
---

Auth0 es una solución flexible y lista para usar para añadir servicios de autenticación y autorización a tus aplicaciones. Tu equipo y tu organización pueden evitar el costo, el tiempo y el riesgo que implica crear su propia solución para autenticar y autorizar usuarios.

Echa un vistazo a algunos de los casos de uso de Auth0:

* Has creado una aplicación increíble y quieres añadir autenticación y autorización de usuarios. Tus usuarios deberían poder iniciar sesión con un identificador (nombre de usuario, correo electrónico o número de teléfono) y contraseña, o con sus cuentas sociales (como Facebook o X). Quieres recuperar el perfil del usuario después del inicio de sesión para poder personalizar la UI y aplicar tus políticas de autorización.
* Has creado una API y quieres protegerla con [OAuth 2.0](/es/docs/authenticate/protocols/oauth).
* Tienes más de una aplicación y quieres implementar [inicio de sesión único (SSO)](/es/docs/authenticate/single-sign-on).
* Has creado una aplicación frontend en JavaScript y una aplicación móvil, y quieres que ambas accedan de forma segura a tu API.
* Tienes una aplicación web que necesita autenticar usuarios mediante [Security Assertion Markup Language (SAML)](/es/docs/authenticate/protocols/saml/saml-configuration).
* Crees que las contraseñas ya no son seguras y quieres que tus usuarios inicien sesión con códigos de un solo uso enviados por correo electrónico o SMS.
* Si una de las direcciones de correo electrónico de tus usuarios se ve comprometida en una violación pública de la seguridad de los datos de algún sitio, quieres recibir una notificación y notificar a los usuarios y/o impedirles iniciar sesión en tu aplicación hasta que restablezcan su contraseña.
* Quieres actuar de forma proactiva para bloquear direcciones IP sospechosas si realizan intentos fallidos de inicio de sesión consecutivos, para evitar ataques DDoS.
* Formas parte de una gran organización que quiere federar su servicio de directorio empresarial existente para permitir que los empleados inicien sesión en varias aplicaciones internas y de terceros usando sus credenciales empresariales actuales.
* No quieres (o no sabes cómo) implementar tu propia solución de gestión de usuarios. Restablecimiento de contraseñas, creación, aprovisionamiento, bloqueo y eliminación de usuarios, y la UI para gestionar todo eso. Solo quieres centrarte en tu aplicación.
* Quieres aplicar [autenticación multifactor (MFA)](/es/docs/secure/multi-factor-authentication) cuando tus usuarios quieran acceder a datos sensibles.
* Buscas una solución de identidad que te ayude a mantenerte al día con los requisitos de cumplimiento en constante crecimiento de SOC2, RGPD, PCI DSS, HIPAA y otros.
* Quieres supervisar a los usuarios en tu sitio o aplicación. Planeas usar estos datos para crear embudos, medir la retención de usuarios y mejorar tu flujo de registro.
* Quieres una política de autorización sólida para permitir que tus usuarios accedan a recursos según [su relación](https://docs.fga.dev/authorization-concepts) con el recurso o [su rol](/es/docs/manage-users/access-control/rbac) en tu organización.
