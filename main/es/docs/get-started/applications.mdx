El término **aplicación** o **app** en Auth0 no implica ninguna característica de implementación en particular. Por ejemplo, puede ser una app nativa que se ejecuta en un dispositivo móvil, una aplicación de página única que se ejecuta en un navegador o una aplicación web tradicional que se ejecuta en un servidor.

Auth0 categoriza las apps según estas características:

* **Tipo de aplicación**: Para agregar autenticación a tu aplicación, debes registrarla en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y seleccionar uno de los siguientes tipos de aplicación:

  * **Aplicación web tradicional**: Aplicaciones web tradicionales que ejecutan la mayor parte de su lógica en el servidor (como Express.js o ASP.NET). Para aprender a configurar una aplicación web tradicional, lee [Registrar aplicaciones web clásicas](/docs/es/get-started/auth0-overview/create-applications/regular-web-apps).
  * **Aplicación web de página única (SPA)**: Aplicaciones JavaScript que ejecutan la mayor parte de la lógica de su interfaz de usuario en un navegador web y se comunican con un servidor web principalmente mediante API (como AngularJS + Node.js o React). Para aprender a configurar una aplicación web de página única, lee [Registrar aplicaciones web de página única](/docs/es/get-started/auth0-overview/create-applications/single-page-web-apps).
  * **Aplicación nativa**: Aplicaciones móviles o de escritorio que se ejecutan de forma nativa en un dispositivo (como iOS o Android). Para aprender a configurar una aplicación nativa, lee [Registrar aplicaciones nativas](/docs/es/get-started/auth0-overview/create-applications/native-apps).
  * **Aplicación máquina a máquina (M2M)**: Aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en tu backend. Normalmente, usarás esta opción si tienes un servicio que requiere acceso a una API. Para aprender a configurar una aplicación nativa, lee [Registrar aplicaciones máquina a máquina](/docs/es/get-started/auth0-overview/create-applications/machine-to-machine-apps).
* **Seguridad de las credenciales**: Según la [especificación OAuth 2.0](https://tools.ietf.org/html/rfc6749#section-2.1), las apps pueden clasificarse como públicas o confidenciales; las apps confidenciales pueden almacenar credenciales de forma segura, mientras que las apps públicas no. Para obtener más información, lee [Aplicaciones confidenciales y públicas](/docs/es/get-started/applications/confidential-and-public-applications).
* **Propiedad**: Que una app se clasifique como propia o de terceros depende de quién la posea y controle. Las apps propias están controladas por la misma organización o persona propietaria del dominio de Auth0. Las apps de terceros permiten que partes externas o partners accedan de forma segura a los recursos protegidos detrás de tu API. Para obtener más información, lee [aplicaciones propias y de terceros](/docs/es/get-started/applications/first-party-and-third-party-applications).

<div id="manage-applications-settings">
  ## Administrar la configuración de las aplicaciones
</div>

Registra las aplicaciones en [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications/\{yourClientId}/settings). Además de configurar aplicaciones en el Dashboard, también puedes configurarlas de forma programática, como se describe en la especificación [OpenID Connect (OIDC) Registro dinámico de clientes 1.0](https://openid.net/specs/openid-connect-registration-1_0.html).

Puedes establecer una configuración más compleja que permita a los usuarios iniciar sesión de forma diferente según la aplicación. Para obtener más información, consulta [Prácticas recomendadas para aplicaciones multi-tenant](/docs/es/get-started/auth0-overview/create-tenants/multi-tenant-apps-best-practices) y [Crear varios tenants](/docs/es/get-started/auth0-overview/create-tenants/create-multiple-tenants).

De forma predeterminada, Auth0 habilita todas las conexiones asociadas a tu tenant cuando creas una nueva aplicación. Para cambiar esto, [actualiza las conexiones de la aplicación](/docs/es/get-started/applications/update-application-connections) en **Configuración de la aplicación** en el Dashboard.

<div id="monitor-applications">
  ## Supervisar aplicaciones
</div>

Puede [supervisar aplicaciones](/docs/es/deploy-monitor/monitor/monitor-applications) y realizar pruebas de extremo a extremo con sus propias pruebas. Auth0 almacena [datos de registro](/docs/es/deploy-monitor/logs), incluidas las acciones de los administradores del Dashboard, las autenticaciones de usuarios exitosas y fallidas, y las solicitudes de cambio de contraseña. Puede usar el flujo de logs en [Auth0 Marketplace](https://marketplace.auth0.com/features/log-streaming) para exportar sus datos de registro y utilizar herramientas como Sumo Logic, Splunk o Mixpanel para analizar y almacenar sus datos de registro.

<div id="remove-applications">
  ## Eliminar aplicaciones
</div>

Puedes [eliminar una aplicación](/docs/es/get-started/applications/remove-applications) desde el Dashboard o mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip>.

<div id="manage-client-secrets">
  ## Gestionar secretos de cliente
</div>

Un <Tooltip tip="Servidor de autorización: servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=client+secret">secreto de cliente</Tooltip> es un secreto que solo conocen tu aplicación y el <Tooltip tip="Secreto de cliente: secreto que utiliza un cliente (aplicación) para autenticarse con el servidor de autorización; solo deben conocerlo el cliente y el servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>. Protege tus recursos al otorgar [tokens](/docs/es/secure/tokens) solo a quienes estén autorizados a solicitarlos.

Protege tus secretos de cliente y **nunca** los incluyas en aplicaciones móviles ni en aplicaciones basadas en navegador. Si tu secreto de cliente se ve comprometido en algún momento, deberías [rotarlo por uno nuevo](/docs/es/get-started/applications/rotate-client-secret) y actualizar todas las aplicaciones autorizadas con el nuevo secreto de cliente.

<div id="grant-types">
  ## Tipos de concesión
</div>

Auth0 ofrece muchos tipos de concesión o flujos de autenticación y autorización, y le permite indicar qué tipos de concesión son adecuados según la propiedad `grant_types` de su aplicación registrada en Auth0. Para obtener más información, consulte [tipo de concesión de aplicaciones](/docs/es/get-started/applications/application-grant-types).
