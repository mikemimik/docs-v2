---
title: "Revocar el acceso a las API mediante concesiones de aplicación"
---

Configurar la concesión de tu aplicación para que los tokens que emites tengan una duración corta significa que, cuando llegue el momento de revocar el acceso a un recurso protegido, simplemente puedes eliminar la concesión. En ese momento, la parte que tiene el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> solo dispone de un tiempo limitado entre la eliminación de la concesión y el vencimiento del token para hacer solicitudes adicionales a la API. Como esta es una opción fácil (y segura) de implementar, te recomendamos denegar el acceso a tus API y otros recursos protegidos revocando las concesiones de aplicación.

Si, por ejemplo, estás usando una aplicación [Machine-to-Machine](/es/docs/get-started/applications) para acceder a tu [API](/es/docs/get-started/apis) y tienes un socio que llama a tu API, puede ocurrir que, al final de tu contrato actual, tú y tu socio decidan no renovar la relación comercial. Ahora quieres quitarle a tu socio el acceso a tu API. Sin embargo, el problema es que le has dado un token de acceso que dura un mes.

* ¿Qué puedes hacer en esta situación?
* ¿Cómo podrías configurar tu entorno de Auth0 para que este tipo de situaciones sean más fáciles de manejar en el futuro?

<div id="application-grants">
  ## Concesiones de la aplicación
</div>

El principal problema en este escenario es el tiempo durante el cual el token de acceso de la API es válido: un mes.

De forma predeterminada, Auth0 emite tokens de acceso que duran 24 horas. Establecer la duración del token en 24 horas significa que su partner debe repetir el flujo de credenciales de cliente (o la concesión que haya implementado) para obtener un nuevo token de acceso cada 24 horas. Para denegarle el acceso a su partner cuando venza su contrato, puede simplemente eliminar la concesión de aplicación para que, cuando expire su token actual, no pueda solicitar uno nuevo.

Puede cambiar la duración de un token configurando la opción `token_lifetime`. La duración adecuada para su caso de uso concreto variará, pero le recomendamos establecer este valor lo más corto posible. Un buen punto de partida para determinarlo sería el margen de tiempo que considere aceptable entre la eliminación de la concesión y el último uso de la API.

<div id="delete-an-application-grant">
  ### Eliminar una concesión de aplicación
</div>

Para eliminar una concesión de aplicación, realiza la llamada `DELETE` correspondiente al <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>, concretamente a su [endpoint Delete an Application Grant](https://auth0.com/docs/api/management/v2#!/Client_Grants/delete_client_grants_by_id). Como parte de la llamada, deberás especificar el ID de la concesión de aplicación que quieres eliminar, que puedes obtener a través del [endpoint Get all Application Grants](https://auth0.com/docs/api/management/v2#!/Client_Grants/get_client_grants) de la Management API.

También puedes [actualizar los tipos de concesión de una aplicación](/es/docs/get-started/applications/update-grant-types) a través del <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.
