Los documentos de [OpenID Connect (OIDC) Discovery](https://openid.net/specs/openid-connect-discovery-1_0-final.html#RFC5785) contienen metadatos sobre el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip> (IdP). Agregar Discovery a tu SDK para dirigir tu aplicación al endpoint `./wellknown` y consumir información sobre tu IdP puede ayudarte a configurar tu integración con el IdP.

Integrar OIDC Discovery en tu SDK proporciona:

* Endpoints expuestos del IdP
* [Claims y scopes estándar compatibles con OIDC](/docs/es/get-started/apis/scopes/openid-connect-scopes) (esto excluye los [custom claims](/docs/es/secure/tokens/json-web-tokens/create-custom-claims) y los scopes definidos en tu tenant)
* Funcionalidades compatibles con el IdP

Puedes configurar aplicaciones con los documentos de Discovery de [OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-discovery-1_0.html), que se encuentran en: `https://{yourDomain}/.well-known/openid-configuration`.

<div id="sample-implementation">
  ### Implementación de ejemplo
</div>

Por ejemplo, así se configura el middleware de OIDC para Katana v3 (OWIN):

1. Instala el paquete de NuGet: **Microsoft.Owin.Security.OpenIdConnect** (v3.x.x)
2. Ve a `App_Start\Startup.Auth.cs` y sustituye tu implementación por la siguiente:

export const codeExample2 = `   app.UseCookieAuthentication(new CookieAuthenticationOptions
{
    AuthenticationType = CookieAuthenticationDefaults.AuthenticationType
});

app.UseOpenIdConnectAuthentication(new OpenIdConnectAuthenticationOptions
{
    Authority = "https://{yourDomain}/",
    ClientId = "{yourClientId}",
    SignInAsAuthenticationType = CookieAuthenticationDefaults.AuthenticationType,
    ResponseType = "token",
    Notifications = new OpenIdConnectAuthenticationNotifications
    {
        // OPCIONAL: puede leer o modificar los claims que se rellenan a partir del JWT
        SecurityTokenValidated = context =>
        {
            // agregar el Auth0 access token como claim
            var accessToken = context.ProtocolMessage.AccessToken;
            if (!string.IsNullOrEmpty(accessToken))
            {
                context.AuthenticationTicket.Identity.AddClaim(new Claim("access_token", accessToken));
            }
            return Task.FromResult(0);
        }
    }
});
`;

<div id="rsa-algorithm-for-jwts">
  ## Algoritmo RSA para JWT
</div>

El middleware de OIDC no admite <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y, a menudo, de token de acceso) que se utiliza para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/docs/es/glossary?term=JWTs">JWTs</Tooltip> firmados con claves simétricas. Asegúrate de configurar tu aplicación para que use el algoritmo RSA con claves públicas y privadas.

1. Ve a [Dashboard &gt; Settings](https://manage.auth0.com/#/applications/\{YOUR_AUTH0_CLIENT_ID}/settings).
2. Desplázate hacia abajo hasta **Advanced Settings**.
3. En la pestaña **OAuth**, establece `RS256` como **Json Web Token(JWT) Signature Algorithm** y haz clic en **Save**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si tienes una conexión empresarial de OIDC, puedes elegir entre [algoritmos de firma adicionales](/docs/es/authenticate/enterprise-connections/private-key-jwt-client-auth#configure-private-key-jwt-client-authentication) para clave privada JWT.
</Callout>

Con esta configuración, Auth0 emitirá JWT firmados con tu clave de firma privada. Tu aplicación los verificará con tu clave de firma pública.

<div id="configure-applications-with-oauth-20-authorization-server-metadata">
  ## Configura aplicaciones con los metadatos del servidor de autorización de OAuth 2.0
</div>

Si tu aplicación o SDK hace referencia a la especificación [OAuth RFC-8414](https://www.rfc-editor.org/rfc/rfc8414) de metadatos del <Tooltip tip="Servidor de autorización: servidor centralizado que ayuda a definir los límites del acceso de un usuario. Por ejemplo, tu servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Authorization+Server">servidor de autorización</Tooltip>, puedes usar el alias de <Tooltip tip="OAuth 2.0: marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">OAuth</Tooltip> para obtener metadatos del IdP: `/.well-known/oauth-authorization-server`. Por ejemplo, [Auth0 Model Context Protocol Server](/docs/es/get-started/auth0-mcp-server) recomienda que todas las aplicaciones OAuth hagan referencia a la especificación de metadatos del servidor de autorización de OAuth.
