---
title: Modo permisivo para aplicaciones de terceros
description: "Más información sobre el modo permisivo para aplicaciones de terceros."
---

Los clientes que tuvieran al menos un tenant que usara aplicaciones de terceros antes de abril de 2026 pueden tener aplicaciones que funcionen en modo permisivo. Este modo conserva el comportamiento preexistente para mantener la compatibilidad con versiones anteriores.

<div id="who-can-use-permissive-mode">
  ## Quién puede usar el modo permisivo
</div>

El modo permisivo está disponible para los clientes que tengan al menos un tenant con aplicaciones de terceros antes de abril de 2026. Todos los tenants de la cuenta de ese cliente pueden crear aplicaciones de terceros en modo permisivo, incluso aquellos que antes no tenían aplicaciones de terceros.

Estos clientes pueden:

* Seguir utilizando las aplicaciones de terceros existentes con su comportamiento actual
* Crear nuevas aplicaciones de terceros con `third_party_security_mode: "permissive"` mediante la Management API
* Crear nuevas aplicaciones con controles de seguridad reforzados en cualquier momento

Los clientes que no hayan usado previamente aplicaciones de terceros siempre crean aplicaciones con controles de seguridad reforzados. No pueden usar el modo permisivo.

<div id="what-permissive-mode-means">
  ## Qué significa el modo permisivo
</div>

Las aplicaciones de terceros en modo permisivo se comportan de forma similar a las aplicaciones de primera parte en cuanto a las funciones disponibles:

* Están disponibles la mayoría de los tipos de concesión (authorization code, implicit, client credentials, device code)
* Se admiten scopes de OIDC y tokens de ID
* El acceso a la API sigue la [política de acceso](/es/docs/get-started/apis/api-access-policies-for-applications) configurada en la API (no se exige una autorización de cliente)
* Formato estándar de ID de cliente (sin el prefijo `tpc_`)
* Se pueden configurar todas las propiedades del cliente

El modo permisivo requiere más esfuerzo de configuración por parte del administrador para aplicar los controles de seguridad. Usted es responsable de:

* Configurar [políticas de acceso a la API](/es/docs/get-started/apis/api-access-policies-for-applications) en cada API para controlar qué aplicaciones de terceros pueden acceder a ellas
* Revisar qué tipos de concesión y qué funciones están habilitados para cada aplicación
* Supervisar el comportamiento de las aplicaciones de terceros mediante los [logs del tenant](/es/docs/deploy-monitor/logs)

<div id="the-third_party_security_mode-property">
  ## La propiedad `third_party_security_mode`
</div>

Cada aplicación de terceros tiene una propiedad `third_party_security_mode` que indica su modo de seguridad. Esta propiedad se establece al crearse y no se puede cambiar.

| **Valor**    | **Significado**                                                                                                                     |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------- |
| `strict`     | Controles de seguridad reforzados. Compatibilidad con OAuth 2.1, autorización explícita de la API, conjunto de funciones limitado.  |
| `permissive` | Comportamiento preexistente. Todas las funciones están disponibles y el administrador configura manualmente los controles de seguridad. |

Las aplicaciones de terceros creadas antes de la introducción de los modos de seguridad devuelven `third_party_security_mode: "permissive"` en las respuestas de la API.

<div id="feature-comparison">
  ## Comparación de funcionalidades
</div>

| **Capacidad**                   | **Controles de seguridad reforzados (estrictos)**                                                                                             | **Comportamiento preexistente (permisivo)**               |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| **Tipos de concesión**          | `authorization_code`, `refresh_token`, `client_credentials`                                                                                   | La mayoría de los tipos de concesión (excepto `password`) |
| **PKCE**                        | Obligatorio                                                                                                                                   | Opcional                                                  |
| **OIDC**                        | No compatible. Está previsto para una futura versión.                                                                                         | Compatible                                                |
| **Autorización de API**         | Siempre requiere una autorización de cliente explícita                                                                                          | Sigue la política de acceso a la API                      |
| **Classic Login**               | No compatible                                                                                                                                 | Compatible                                                |
| **Endpoints heredados**         | No compatibles                                                                                                                                | Disponibles                                               |
| **Formato del ID de cliente**   | Prefijo `tpc_`                                                                                                                                | Formato estándar                                          |
| **Propiedades configurables**   | [Conjunto limitado de propiedades](/es/docs/get-started/applications/third-party-applications/security-controls#restricted-client-configuration) | Todas las propiedades                                     |
| **Capacidades futuras**         | Límites de tasa y futuras mejoras de seguridad y administración                                                                               | No disponibles                                            |
| **Creación mediante Dashboard** | Compatible                                                                                                                                    | No disponible a través del Dashboard                      |

<div id="dynamic-client-registration-in-permissive-mode">
  ## Registro dinámico de clientes en modo permisivo
</div>

Si tu cuenta cumple los requisitos para el modo permisivo y usas [registro dinámico de clientes](/es/docs/get-started/applications/dynamic-client-registration), puedes controlar el modo de seguridad de los clientes registrados dinámicamente mediante una configuración del tenant independiente.

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Ve a **Settings &gt; Advanced**.
    2. En **Dynamic Client Registration (DCR) Security Mode**, selecciona **Permissive**.
    3. Selecciona **Save**.

    <Frame>![Configuración avanzada del tenant en el Dashboard con el menú desplegable DCR Security Mode](/docs/images/third-party-applications/dcr-security-mode.png)</Frame>
  </Tab>

  <Tab title="Management API">
    ```bash
    curl --request PATCH \
      --url 'https://{yourDomain}/api/v2/tenants/settings' \
      --header 'Authorization: Bearer {YOUR_MANAGEMENT_API_TOKEN}' \
      --header 'Content-Type: application/json' \
      --data '{
        "dynamic_client_registration_security_mode": "permissive"
      }'
    ```
  </Tab>
</Tabs>

La configuración `dynamic_client_registration_security_mode` es independiente del valor predeterminado de las aplicaciones creadas mediante `POST /api/v2/clients`. Puedes configurarla en cualquier momento.

<Warning>
  Si estableces `dynamic_client_registration_security_mode` en `strict`, primero configura los [permisos predeterminados de la API](/es/docs/get-started/applications/application-access-to-apis-client-grants#default-permissions-for-third-party-applications). Sin ellos, los clientes registrados dinámicamente no podrán acceder a ninguna API.
</Warning>

<div id="adopt-enhanced-security-controls">
  ## Adopta controles de seguridad reforzados
</div>

Auth0 recomienda adoptar controles de seguridad reforzados para todas las nuevas aplicaciones de terceros. La propiedad `third_party_security_mode` no se puede cambiar después de crear una aplicación; no puedes convertir una aplicación permisiva existente en una estricta, ni viceversa. En su lugar, eliges qué modo de seguridad usar al crear nuevas aplicaciones en adelante.

Para entender cómo está cambiando la configuración predeterminada para las nuevas aplicaciones y cómo prepararte, lee [migrar a la seguridad mejorada para aplicaciones de terceros](/es/docs/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-to-enhanced-security-third-party-applications).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si necesitas reemplazar una aplicación permisiva existente por una estricta, debes crear una nueva aplicación. Esto invalida todos los tokens de actualización existentes y las autorizaciones de usuario de la aplicación anterior. Coordina la transición con el tercero para minimizar las interrupciones.
</Callout>
