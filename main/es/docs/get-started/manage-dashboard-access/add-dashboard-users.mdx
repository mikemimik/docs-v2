description: Aprende cómo agregar miembros del tenant para que accedan al Auth0 Dashboard.
title: Agregar miembros del tenant
---

Si eres administrador de un tenant, puedes agregar colegas al tenant y asignarles un rol específico para darles acceso a las funciones de <Tooltip tip="Auth0 Dashboard: producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> que necesiten.

<Warning>
  Si tu cuenta de administrador del tenant se autentica con nombre de usuario y contraseña, debes verificar tu dirección de correo electrónico antes de poder invitar a otros miembros.
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si Teams está aprovisionado para tu cuenta empresarial, administra los miembros de tu tenant a través de Auth0 Teams. Para obtener más información, consulta [Administración de miembros del equipo](/es/docs/get-started/auth0-teams/team-member-management).
</Callout>

Cuando agregas nuevos miembros del tenant, puedes asignarles roles que les den acceso sin poner en riesgo las aplicaciones de producción ni la información confidencial. Puedes asignar a un usuario más de un rol, y los permisos se combinarán. Para mayor protección, los usuarios pueden activar la [autenticación multifactor](/es/docs/get-started/manage-dashboard-access/add-change-remove-mfa) (<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=MFA">MFA</Tooltip>). Para obtener más información sobre las funciones del Dashboard a las que se puede acceder con cada rol, consulta [Acceso al Dashboard por rol](/es/docs/get-started/manage-dashboard-access/feature-access-by-role).

| Rol                             | Permisos                                                                                                                                                                                                  |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Admin**                       | Acceso de lectura y escritura a todos los recursos del Auth0 Dashboard, incluidas las extensiones.                                                                                                        |
| **Editor - Connections**        | Acceso de lectura, escritura y creación a todos los tipos de conexiones.                                                                                                                                  |
| **Editor - Key Management**     | Crear y administrar claves criptográficas.                                                                                                                                                                |
| **Editor - Organizations**      | Acceso de lectura, escritura y creación a la configuración de Organizations. Acceso de solo lectura a Users, Roles y Connections.                                                                         |
| **Editor - Specific Apps**      | Acceso de lectura y escritura únicamente a aplicaciones específicas.                                                                                                                                      |
| **Editor - Users**              | Operaciones de administración de usuarios (crear, eliminar, bloquear, desbloquear, restablecer MFA, restablecer contraseña, actualizar metadata, asignar roles, etc.) y acceso a logs.                    |
| **Viewer - Users**              | Acceso de solo lectura a los usuarios y a los logs relacionados con usuarios.                                                                                                                             |
| **Viewer - Config Settings**    | Acceso de solo lectura a todos los ajustes de configuración (aplicaciones, API, reglas, ajustes de seguridad, etc.), excepto a información confidencial, como secrets, facturación, usuarios y logs.      |
| **Support Access - Deprecated** | Acceso a tickets (enviar, ver y actualizar) y métricas agregadas.                                                                                                                                         |
| **Elevated Support Access**     | Acceso de lectura, escritura y creación a **todos** los tickets de soporte creados por **todos los usuarios** en un tenant (Tickets de suscripción en Auth0 Support Center). Acceso a métricas agregadas. |

<Card title="La disponibilidad varía según el plan de Auth0">
  Tanto tu implementación específica de login como tu plan de Auth0 o acuerdo personalizado influyen en si esta función está disponible. Para obtener más información, consulta [Precios](https://auth0.com/pricing).
</Card>

1. Ve a [Dashboard &gt; Settings &gt; Tenant Members](https://manage.auth0.com/#/tenant/admins).
2. Haz clic en **Add Member**.
3. En la pantalla **Add New Tenant Member**, introduce la dirección de correo electrónico del usuario y selecciona los roles que quieras asignarle.
4. Haz clic en **Invite**.
   Auth0 enviará al usuario un correo electrónico con un enlace en el que deberá hacer clic para aceptar la invitación. Antes de que el usuario acepte la invitación, aparecerá como pendiente en la lista de invitaciones. Si haces clic en `...`, verás **Remove invitation** o **Copy invitation link**.

<Card title="Tenants faltantes">
  Los usuarios pueden crear cuentas de Auth0 de distintas maneras. Por ejemplo, pueden registrarse con un proveedor social (p. ej., Google, GitHub) y luego volver a registrarse con su dirección de correo electrónico. Si los miembros de tu tenant no pueden ver todos sus tenants después de iniciar sesión, verifica si tienen varias cuentas de Auth0. Puedes ver el método de registro que usó el miembro del tenant del Dashboard en [Dashboard &gt; Tenant Settings &gt; Tenant Members](https://manage.auth0.com/#/tenant/admins).
</Card>

Cuando el usuario invitado haga clic en el enlace para aceptar la invitación, se le pedirá que inicie sesión con la dirección de correo electrónico invitada. Luego podrá elegir si iniciar sesión con un proveedor social o con su dirección de correo electrónico y contraseña. Si elige iniciar sesión con su dirección de correo electrónico y contraseña, ya debe haber creado una cuenta con Auth0; de lo contrario, primero tendrá que registrarse y luego volver a aceptar la invitación.

<Callout icon="clock" color="#F59E0B" iconType="regular">
  **Sincronización de Auth0 Support Center**

  Al usar [Auth0 Support Center](https://support.auth0.com) para crear un caso de soporte, el campo CC está limitado a los miembros actuales del tenant. Después de que un miembro del tenant acepte una invitación, pueden pasar hasta 12 horas antes de que se sincronice con Auth0 Support Center y esté disponible para añadirlo al campo CC.
</Callout>
