Acelera tu flujo de trabajo de desarrollo conectando agentes de IA y LLM directamente a la base de conocimientos de Auth0. Obtén respuestas contextualizadas, ejemplos de código y referencias de API con enlaces a la documentación oficial.

Para ayudar a tus herramientas de IA a indexar y buscar la información que necesitas, Auth0 ofrece:

* Archivos de contexto para LLM (`llms.txt`): archivos Markdown estructurados para la ingesta masiva.
* Servidor MCP de Auth0 Docs: servidor MCP alojado para que los agentes de IA consulten dinámicamente la base de conocimientos.

<div id="llm-context-files">
  ## Archivos de contexto de LLM
</div>

Usa archivos de contexto de LLM para ayudar a los LLM a comprender y procesar de forma eficiente todo el directorio del sitio de Auth0.

| Nombre          | URL                                    | Descripción                                                                         |
| --------------- | -------------------------------------- | ----------------------------------------------------------------------------------- |
| `llms.txt`      | `https://auth0.com/llms.txt`           | Resúmenes concisos de todos los artículos; ideal para obtener contexto rápidamente. |
| `llms-full.txt` | `https://auth0.com/docs/llms-full.txt` | Contenido completo de todos los artículos; ideal para una indexación exhaustiva.    |

<div id="auth0-docs-mcp-server">
  ## Servidor MCP de Auth0 Docs
</div>

Conecta tus herramientas de IA al servidor Model Context Protocol (MCP) de Auth0 Docs para acceder en tiempo real a la documentación más actualizada de Auth0. El servidor MCP de Auth0 Docs ofrece la herramienta `SearchAuth0Docs`, que permite a los agentes de IA buscar en la base de conocimientos de Auth0 y devolver contenido contextual con enlaces relevantes a la documentación del producto.

El servidor MCP de Auth0 Docs tiene los siguientes metadatos del endpoint:

```
Server Name: Auth0 Docs
Server URL: https://auth0.com/docs/mcp
Transport Type: HTTP
```

Aprende a conectar tus herramientas de IA al servidor MCP de Auth0 Docs con nuestras guías de integración.

<Tabs>
  <Tab title="ChatGPT">
    Para conectar ChatGPT al servidor MCP de Auth0 Docs:

    1. Ve a [Settings &gt; Connectors](https://chatgpt.com/#settings/Connectors).
    2. Selecciona **Create** e introduce los metadatos de tu servidor MCP:
       * Nombre del conector: Auth0 Docs MCP Server
       * Descripción: Indexa la base de conocimiento de Auth0 y permite buscar en ella
       * URL del conector: `https://auth0.com/docs/mcp`
    3. Selecciona **Create**.

    Si todo sale bien, el servidor MCP de Auth0 Docs debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de OpenAI sobre [Connect from ChatGPT](https://developers.openai.com/apps-sdk/deploy/connect-chatgpt/).
  </Tab>

  <Tab title="Claude">
    Para conectar Claude al servidor MCP de Auth0 Docs:

    1. Ve a [Settings &gt; Connectors](https://claude.ai/settings/connectors).
    2. Selecciona **Add custom connector**.
    3. Introduce la siguiente información para agregar el servidor MCP de Auth0 Docs:
       * Nombre: Auth0 Docs MCP Server
       * URL: `https://auth0.com/docs/mcp`
    4. Selecciona **Add**.

    Si todo sale bien, el servidor MCP de Auth0 Docs debería aparecer como un conector personalizado. Para obtener más información, consulta la documentación de Model Context Protocol sobre [Connect to remote MCP Servers](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    Para conectar Claude Code al servidor MCP de Auth0 Docs:

    En la terminal, ejecuta:

    ```bash lines
    claude mcp add --transport http auth0-docs-mcp-server https://auth0.com/docs/mcp
    ```

    Para comprobarlo, ejecuta:

    ```bash lines
    claude mcp list
    ```

    Si todo ha ido bien, Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Claude sobre [la instalación de servidores MCP](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Codex">
    Para conectar Codex al servidor MCP de la documentación de Auth0:

    En tu terminal, ejecuta:

    ```bash lines
    codex mcp add auth0-docs-mcp-server --url https://auth0.com/docs/mcp
    ```

    Para comprobarlo, ejecuta:

    ```bash lines
    codex mcp list
    ```

    Si todo va bien, Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Codex sobre [Model Context Protocol](https://developers.openai.com/codex/mcp).
  </Tab>

  <Tab title="Cursor">
    Para conectar Cursor al servidor MCP de Auth0 Docs:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la paleta de comandos y buscar **Open MCP settings**.
    2. Selecciona **New MCP Server** para editar el archivo `mcp.json`.
    3. Agrega el servidor MCP de Auth0 Docs:

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "url": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    Si la instalación se realizó correctamente, el Auth0 Docs MCP Server debería aparecer como un servidor MCP instalado. Para obtener más información, consulta la documentación de Cursor sobre [la instalación de servidores MCP](https://cursor.com/docs/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    Para conectar VS Code con el servidor MCP de Auth0 Docs:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la paleta de comandos y buscar **MCP: Add Server**.
    2. Selecciona **HTTP** como tipo de servidor MCP.
    3. Introduce la URL del servidor MCP (`https://auth0.com/docs/mcp`) y establece el ID del servidor como `auth0-docs-mcp-server`.
    4. Selecciona **Global** o **Workspace** para que el servidor MCP esté disponible en todos tus espacios de trabajo o solo en el espacio de trabajo actual.
    5. Pulsa **Enter** para completar la configuración. Verifica que el servidor MCP de Auth0 Docs aparezca en **Installed MCP Servers**, dentro de **Extensions**.

    Para obtener más información, consulta [Add and manage MCP servers in VS Code](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>

  <Tab title="Windsurf">
    Para conectar Windsurf con el servidor MCP de Auth0 Docs:

    1. Usa Command + Shift + P (Ctrl + Shift + P en Windows) para abrir la configuración de Windsurf y buscar **MCP**.
    2. Selecciona **View Raw Config** para editar tu `mcp_config.json`.
    3. Añade el servidor MCP de Auth0 Docs:

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "serverUrl": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    4. Guarda los cambios y selecciona **Refresh** para cargar la nueva configuración de MCP.

    Para comprobarlo, escribe `What MCP servers do you have installed?` en el chat de Cascade. Cascade debería mostrar el servidor MCP de Auth0 Docs como un servidor MCP instalado. Para obtener más información, consulta la documentación de Windsurf sobre [Model Context Protocol](https://docs.windsurf.com/plugins/cascade/mcp#model-context-protocol-mcp).
  </Tab>
</Tabs>

<div id="troubleshooting">
  ### Solución de problemas
</div>

* Ayúdame a solucionar un error de `refresh token doesn’t exist` al usar el intercambio de tokens de actualización. Usa como contexto las instrucciones de configuración de Token Vault.
* El botón de cierre de sesión no está cerrando la sesión del lado de Auth0. Busca en la documentación la estructura correcta de la URL de Federated Logout y actualiza mi controlador de cierre de sesión para usarla.
* Mi flujo de inicio de sesión está fallando con un `401 Unauthorized error`. Usa el servidor MCP de Auth0 Docs para buscar las causas comunes de este error con el SDK `auth0-node` y revisa mi `.env` y la inicialización del cliente para detectar desajustes.
