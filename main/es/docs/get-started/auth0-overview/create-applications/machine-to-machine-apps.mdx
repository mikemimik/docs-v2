Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debe registrar su aplicación en Auth0 mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Estas aplicaciones pueden incluir aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en el back-end.

Las aplicaciones M2M están vinculadas a una API y a sus permisos o [scopes](/docs/es/get-started/apis/scopes/api-scopes), que se seleccionan de valores predefinidos. Asegúrese de haber [registrado la API asociada](/docs/es/get-started/auth0-overview/set-up-apis) en Auth0 y de haber definido scopes para la API antes de comenzar este registro.

Si desea autorizar su aplicación para acceder únicamente a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0, no necesita registrar una API independiente; la Management API ya viene preconfigurada.

Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debe registrar su aplicación en Auth0 mediante Auth0 Dashboard.

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haga clic en **Create Application**.
2. Introduzca un nombre descriptivo para su aplicación, seleccione **Machine to Machine Applications** y haga clic en **Create**.
3. Seleccione la API que quiere poder llamar desde su aplicación.
4. A cada aplicación M2M que accede a una API se le debe otorgar un conjunto de permisos (o scopes) que debe conceder el usuario autenticado. Para obtener información sobre cómo agregar permisos, consulte [Agregar permisos de API](/docs/es/get-started/apis/add-api-permissions). Seleccione los **Permissions** que desea que se emitan como parte del token de acceso de su aplicación y haga clic en **Authorize**.

   <Warning>
     Use los scopes `update:client_grants` y `create:client_grants` solo con aplicaciones de alto privilegio, ya que permiten que el cliente se conceda permisos adicionales a sí mismo.
   </Warning>

   Una vez creada la nueva aplicación, puede configurar **Configuración de la aplicación**, que incluye las siguientes pestañas:

   | Pestaña de configuración | Descripción                                                                                                                                                                                                                                                                                                               |
   | ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**          | Muestra todos los ejemplos disponibles para aplicaciones ${application_type}. También muestra cómo llamar a su API con distintas tecnologías. Para obtener información sobre cómo aceptar y validar tokens de acceso en su API, consulte nuestros [Backend/API Quickstarts](/docs/es/quickstart/backend/nodejs/interactive). |
   | **Settings**             | Muestra toda la configuración disponible para su aplicación. De forma predeterminada, Auth0 crea la mayoría de estas opciones por usted.                                                                                                                                                                                  |
   | **Credentials**          | Muestra el método de autenticación de la aplicación y las credenciales configuradas. Para obtener más información, lea [Application Credentials](/docs/es/secure/application-credentials)                                                                                                                                    |
   | **APIs**                 | Le permite autorizar APIs adicionales para usarlas con su aplicación.                                                                                                                                                                                                                                                     |

   Para obtener más información, lea [Configuración de la aplicación](/docs/es/get-started/applications/application-settings).

<div id="next-steps">
  ## Próximos pasos
</div>

Una vez que hayas registrado y configurado tu aplicación, estos son algunos de los pasos siguientes más habituales:

* Configura una conexión con un proveedor de identidad y habilítala para tu aplicación (si es necesario para tu caso de uso).
* Modifica el código de tu aplicación para usar la aplicación que registraste en Auth0. Consulta nuestros [Auth0 Quickstarts](/docs/es/quickstarts), donde encontrarás instrucciones detalladas y ejemplos para diversas tecnologías. También aprenderás a implementar el inicio y cierre de sesión, gestionar las sesiones de tus usuarios, recuperar y mostrar información del perfil de usuario y mucho más.
* Usa las [API de Auth0](/docs/es/api).

  * La [Authentication API](https://auth0.com/docs/api/authentication) se encarga de todas las funciones principales relacionadas con la identidad (por ejemplo, iniciar sesión, cerrar sesión y obtener el perfil del usuario). La mayoría de los usuarios usan esta API a través de nuestros Quickstarts, la [biblioteca Auth0.js](/docs/es/libraries/auth0js) o el [widget Lock](/docs/es/libraries/lock). Sin embargo, si estás creando manualmente toda la interfaz de autenticación, tendrás que interactuar directamente con esta API.
  * La [Management API](https://auth0.com/docs/api/management/v2) te permite automatizar varias tareas a las que también puedes acceder desde el Dashboard de Auth0 (por ejemplo, crear usuarios y configurar tipos de concesión de aplicación).
