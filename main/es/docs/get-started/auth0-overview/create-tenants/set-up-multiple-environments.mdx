---
description: Describe cómo usar múltiples tenants de Auth0 para gestionar varios entornos.
title: Configurar múltiples entornos
---

Los entornos de desarrollo, staging y producción son fáciles de configurar en Auth0. Simplemente crea un tenant nuevo para cada entorno para garantizar el aislamiento entre ellos. Puedes cambiar fácilmente entre tenants con el selector de tenants del menú superior izquierdo del Dashboard. También puedes configurar distintos administradores para cada uno.

Los [rate limits](/es/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy) de producción solo se aplican a los tenants etiquetados como `Production`. Asegúrate de que la etiqueta de entorno de tu tenant esté configurada como `Production` antes de pasar a producción, y ten en cuenta las diferencias en los rate limits entre distintos tenants, entornos y estructuras de tarifas.

Puedes ponerles a tus entornos el nombre que prefieras. Para los entornos de producción, te recomendamos encarecidamente usar [custom domains](/es/docs/customize/custom-domains).

Si tienes un plan de suscripción Enterprise, puedes crear tenants secundarios [vinculando múltiples tenants a una sola suscripción](/es/docs/get-started/auth0-overview/create-tenants/child-tenants); así, cada tenant secundario obtiene acceso a las funciones de pago o mejoradas asociadas con la suscripción. Puedes asegurarte de que tus entornos de desarrollo, staging o pruebas tengan acceso a las mismas funciones que tu entorno de producción vinculándolos todos a la misma suscripción Enterprise.

<div id="tag-the-environment">
  ## Etiqueta el entorno
</div>

Para cada nuevo tenant que se cree, debes especificar su entorno. Puedes asignar etiquetas de entorno a tus tenants para diferenciar entre entornos de desarrollo, staging y producción.

Si tu tenant es de uso mixto, elige el entorno de nivel más alto. Por ejemplo, un tenant que se usa tanto para desarrollo como para producción debe configurarse como **Production**.

1. Para asignar una etiqueta de entorno a un tenant, ve a [Dashboard &gt; Settings &gt; General](https://manage.auth0.com/#/tenant/general).

   <Frame>![Pestaña General de la configuración del tenant en el Dashboard](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. En **Assign Environment Tag**, identifica el entorno de tu tenant como **Development**, **Staging** o **Production**.

   <Frame>![undefined](/docs/images/cdy7uua7fh8z/27OH1jFXce97CSjk7TPeD7/deafc344f675fb11ff5d3d589ffadee9/2025-02-26_14-27-31.png)</Frame>
3. Después de seleccionar el entorno, haz clic en **Save**.

<div id="migration">
  ## Migración
</div>

A través de la [Management API v2](https://auth0.com/docs/api/management/v2), puedes automatizar la migración de recursos ([rules](/es/docs/customize/rules), [conexiones](/es/docs/authenticate/identity-providers) de base de datos, etc.) entre tenants.

Para gestionar la configuración más fácilmente, guarda los valores de configuración en el [Dashboard](https://manage.auth0.com/#/rules) en lugar de incluirlos directamente en los scripts de tus **rules** o **db connections**.

Por ejemplo, supongamos que quieres definir una URL para los logs. Una forma de hacerlo es incluirla directamente en la regla:

```javascript lines
function(user, context, callback){
      var log_url = 'https://someurl/log';
      ...
    }
```

Sin embargo, este código no es portable, ya que es probable que esta URL cambie al pasar de desarrollo a producción.

La forma recomendada de trabajar con código que necesitas usar o mover de desarrollo a producción es a través de la sección [Rules](https://manage.auth0.com/#/rules). Si aún no has creado una regla, tendrás que hacerlo. (De lo contrario, ve directamente al paso 4).

1. Haz clic en **Create Your First Rule**.
2. Elige la plantilla **empty rule**.
3. Introduce un nombre para tu nueva regla y haz clic en **Save**.
4. Ve a [Dashboard &gt; Rules](https://manage.auth0.com/#/rules) y desplázate hasta la parte inferior de la página para establecer tus valores de configuración (usaremos `log_url` como nombre de clave y `https://someurl/log` como valor); luego, haz clic en **Create**.
5. Ahora puedes escribir tu regla. Edita la regla que creaste, introduce el siguiente código en el área de código y haz clic en **Save**.

   ```javascript lines
   function(user, context, callback){
         var log_url = configuration.log_url;
         ...
       }
   ```

Este código es portable y, cuando migres a producción, solo tendrás que cambiar esta configuración en lugar de buscar en tus scripts.

<div id="adldap-connectors">
  ## Conectores AD/LDAP
</div>

Si usa varios tenants de Auth0 con AD/LDAP, deberá crear una conexión AD/LDAP y configurar un conector AD/LDAP para cada tenant. Esto se debe a que cada conector AD/LDAP está vinculado a una conexión específica dentro de un tenant de Auth0.

Varios conectores AD/LDAP pueden apuntar al mismo directorio AD o LDAP, pero cada conector AD/LDAP solo puede ser utilizado por una única conexión dentro de un tenant de Auth0.

Si tiene varios directorios AD/LDAP en los que se autenticarán los usuarios (por ejemplo, para dar soporte a distintos departamentos o clientes, cada uno con su propio directorio), puede configurar varios conectores AD/LDAP dentro de cada tenant de Auth0.
