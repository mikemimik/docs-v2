En esta sección revisaremos toda la configuración que debemos realizar en el [Auth0 Dashboard](https://manage.auth0.com/#).

<div id="create-the-api">
  ## Crear la API
</div>

Ve a la [sección de APIs](https://manage.auth0.com/#/apis) del Dashboard y haz clic en el botón **Create API**.

Se te pedirá que proporciones los siguientes datos para tu API:

* **Name**: un nombre descriptivo para la API. No afecta a ninguna funcionalidad.
* **Identifier**: un identificador único para la API. Recomendamos usar una URL, pero ten en cuenta que no tiene que ser una URL disponible públicamente; Auth0 no llamará a tu API en ningún caso. Este valor no se puede modificar posteriormente.
* **<Tooltip tip="Signing Algorithm: Algoritmo utilizado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="View Glossary" href="/es/docs/glossary?term=Signing+Algorithm">Algoritmo de firma</Tooltip>**: el algoritmo con el que se firmarán los tokens. Los valores disponibles son `HS256` y `RS256`. Al seleccionar RS256, el token se firmará con la clave privada del tenant. Para obtener más información sobre los algoritmos de firma, consulta [Signing Algorithms](/es/docs/get-started/applications/signing-algorithms).

<Frame>![Dashboard - Applications - APIs - Create API - Popup](/docs/images/cdy7uua7fh8z/4kIH4BQc4vDel2zhJ6lwCE/094cff6e8e1788c39796eb28ab1925ca/2024-06-27_14-50-24.png)</Frame>

Completa la información requerida y haz clic en el botón **Create**.

<div id="configure-the-scopes">
  ## Configura los scopes
</div>

Una vez creada la aplicación, tendrás que configurar los scopes que las aplicaciones pueden solicitar durante la autorización.

En la configuración de tu API, ve a la pestaña Permisos. En esta sección puedes agregar los scopes para nuestro caso de uso: `read:timesheets`, `create:timesheets`, `delete:timesheets` y `approve:timesheets`.

<Frame>![Panel - Aplicaciones - APIs - Permisos](/docs/images/cdy7uua7fh8z/xxMOKezob6gNHJ0CcKhcg/ccbb9bbab29576a96ed8425329ecffb4/dashboard-apis-edit_view-permissions.png)</Frame>

<div id="create-the-application">
  ## Crear la aplicación
</div>

Hay cuatro tipos de aplicaciones en Auth0:

* **Aplicación nativa** (usada por aplicaciones móviles o de escritorio),
* **Aplicación web de una sola página**,
* **Aplicación web tradicional** y
* **Aplicación de máquina a máquina** (usada por CLI, daemons o servicios que se ejecutan en tu backend).

Para este caso, queremos crear una nueva aplicación para nuestra SPA, así que usaremos **aplicación web de página única** como tipo de aplicación.

Para crear una nueva aplicación, ve al [dashboard](https://manage.auth0.com/#) y haz clic en la opción de menú [Applications](https://manage.auth0.com/#/applications), a la izquierda. Luego haz clic en el botón **+ Create Application**.

Asigna un nombre a tu aplicación (usaremos `Timesheets SPA`) y selecciona `Single-Page Web App` como tipo.

Haz clic en **Create**.

Eso es todo por ahora. Cuando terminemos la implementación de la SPA, volveremos al dashboard y a la configuración de esta aplicación para hacer algunos cambios.

<div id="configure-the-authorization-extension">
  ## Configurar la Authorization Extension
</div>

Deberá asegurarse de que la Authorization Extension esté instalada en su tenant. Puede consultar la [documentación de la Authorization Extension](/es/docs/customize/extensions/authorization-extension) para obtener más información sobre cómo hacerlo.

<div id="define-permissions">
  ### Definir permisos
</div>

Ahora definirás los permisos necesarios, de acuerdo con los scopes que ya has definido: `read:timesheets`, `create:timesheets`, `delete:timesheets` y `approve:timesheets`.

En la Authorization Extension, haz clic en la pestaña **Permisos** y luego en el botón **Crear permiso**.

En el cuadro de diálogo, introduce los datos de cada permiso.

Asegúrate de que el nombre del permiso sea exactamente igual al scope correspondiente:

<Frame>![Panel - Extensiones - Authorization Extension - Crear permiso](/docs/images/cdy7uua7fh8z/6G4X14X3A73IDRwOrkmttq/e0c20c44a32e0b39a6d8fb72731ffbaf/create-permission.png)</Frame>

Continúa creando los permisos para todos los scopes restantes:

<div id="define-roles">
  ### Definir roles
</div>

A continuación, vamos a configurar los dos roles: employee y manager.

Ve a la pestaña **Roles**, haz clic en el botón **Create Role** y selecciona la aplicación **Timesheets SPA**.

Configura **Name** y **Description** como `Employee`, y selecciona los permisos `delete:timesheets`, `create:timesheets` y `read:timesheets`. Haz clic en **Save**.

A continuación, sigue el mismo proceso para crear el rol `Manager` y asegúrate de haber seleccionado todos los permisos.

<div id="assign-users-to-roles">
  ### Asignar usuarios a roles
</div>

Debe asignar a todos los usuarios el rol de `Manager` o el de `Employee`.

Para ello, vaya a la pestaña **Users** de Authorization Extension y seleccione un usuario.

En la pantalla de información del usuario, vaya a la pestaña **Roles**. Haga clic en **Add Role to User** y seleccione el rol adecuado.

<div id="configuring-the-authorization-extension">
  ### Configuración de la Authorization Extension
</div>

También deberá asegurarse de que la regla de la Authorization Extension esté publicada.

Para ello, haga clic en el avatar de su usuario, en la esquina superior derecha de la Authorization Extension, y seleccione **Configuration**.

Asegúrese de que **Permisos** esté habilitado y, a continuación, haga clic en **Publish Rule**.

<div id="create-a-rule-to-validate-token-scopes">
  ### Crear una Rule para validar los scopes del token
</div>

El último paso de este proceso es crear una Rule para comprobar si los scopes contenidos en un <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip> son válidos en función de los permisos asignados al usuario. Cualquier scope que no sea válido para un usuario debe eliminarse del Token de acceso.

En tu <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, ve a la pestaña **Rules**. Deberías ver la Rule creada por la Authorization Extension. Haz clic en el botón **Create Rule** y selecciona la plantilla **Empty Rule**. Puedes asignarle un nombre a la Rule, como **Access Token Scopes**, y luego especificar el siguiente código para la Rule:

```javascript lines
function (user, context, callback) {
  var permissions = user.permissions || [];
  var requestedScopes = context.request.body.scope || context.request.query.scope;
  var filteredScopes = requestedScopes.split(' ').filter( function(x) {
    return x.indexOf(':') < 0;
  });

  var allScopes = filteredScopes.concat(permissions);
  context.accessToken.scope = allScopes.join(' ');

  callback(null, user, context);
}
```

El código anterior garantizará que todos los tokens de acceso solo contengan los `scopes` con el formato correcto (p. ej., `action:area` o `delete:timesheets`) que sean válidos de acuerdo con los permisos del usuario. Cuando hayas terminado, puedes hacer clic en el botón **Save**.

Las reglas se ejecutan en el orden en que se muestran en la página Rules, así que asegúrate de que la nueva Rule que creaste esté ubicada debajo de la Rule de la Extensión de autorización, para que se ejecute después de la Rule de la Extensión de autorización.
