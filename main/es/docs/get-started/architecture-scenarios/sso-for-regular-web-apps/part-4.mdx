En este tutorial abordamos un escenario sencillo: una aplicación web tradicional, alojada en la nube, que usa Auth0 para la autenticación y aprovecha el almacén de usuarios existente de Active Directory. Vimos qué ofrece <Tooltip tip="OpenID: Estándar abierto para autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) y por qué era la mejor opción para este caso de uso, cómo funciona el flujo de autenticación, qué es un <Tooltip tip="OpenID: Estándar abierto para autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=ID+Token">ID Token</Tooltip> y cómo validarlo y manipularlo, cómo configurar aplicaciones y conexiones en el <Tooltip tip="Auth0 Dashboard: Auth0's main product to configure your services." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>, cómo implementar el inicio y cierre de sesión de usuarios con Lock, y cómo funcionan la gestión de sesiones y el control de acceso.

Empezamos describiendo el caso de uso y los requisitos, y luego explicamos cómo puede cumplirse cada uno y cuál fue el razonamiento detrás de cada decisión que se tomó.

Usamos ASP .NET Core para la implementación de ejemplo de la aplicación web, aunque esperamos que, después de leer este documento, puedas crear una aplicación web de este tipo con el framework que prefieras.
