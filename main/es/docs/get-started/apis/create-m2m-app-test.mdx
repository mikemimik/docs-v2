title: Crear aplicaciones de máquina a máquina para pruebas
---

Cuando creas una cuenta, se crea una instancia predeterminada de <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> en la sección API del <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. También se crea automáticamente una aplicación de prueba de máquina a máquina de ejemplo. Las siguientes instrucciones te permiten crear otra aplicación de prueba para usarla con esta instancia de la Management API y usar el token de prueba generado antes de crear tu propia configuración de producción.

Ten en cuenta que el token de prueba que se encuentra en API Explorer sirve únicamente para probar el acceso a la Management API. Para obtener más información sobre los tokens de la Management API, consulta [Obtener tokens de acceso de la Management API para producción](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production). Si te interesa usar el token de prueba, visita [Obtener tokens de acceso de la Management API para pruebas](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-testing)

1. Ve a [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) y selecciona **Auth0 Management API**.
2. Selecciona la pestaña **API Explorer**.
3. Haz clic en **Create &amp; Authorize a Test Application**. Se creará una nueva aplicación y quedará autorizada para acceder a la Management API.

<Frame>![Dashboard Applications APIs Auth0 Management API Explorer Tab Authorize and Test](/docs/images/cdy7uua7fh8z/6G5M1JY9iYnkYXfDHwTvR/d09119af4f7d6d45c9b39001fd189d2f/dashboard-applications-apis-mgmt-api-authorize-and-test.png)</Frame>

La aplicación creada en los pasos anteriores tiene concedidos todos los scopes de la Management API. Esto significa que puede acceder a todos los endpoints con fines de prueba. Sin embargo, por lo general las aplicaciones no tienen acceso a todos los scopes, sino solo a los que necesitan.

<Card title="¿Cómo puedo averiguar qué scopes/permisos se requieren para la Management API?">
  A cada aplicación de máquina a máquina que accede a una API se le debe conceder un conjunto de [Scopes](/es/docs/get-started/apis/scopes/api-scopes). Los scopes son permisos que debe conceder el propietario. Cada endpoint de [Auth0 Management API v2](https://auth0.com/docs/api/management/v2) requiere scopes específicos. Para ver los scopes/permisos requeridos para cada endpoint, ve a [Management API Explorer](https://auth0.com/docs/api/management/v2#!) y busca el endpoint que quieres llamar. Cada endpoint tiene una sección llamada **Scopes** que enumera todos los scopes que acepta.
</Card>

<div id="example-get-all-connections-endpoint">
  ## Ejemplo: endpoint Get All Connections
</div>

El endpoint [Get All Connections](https://auth0.com/docs/api/management/v2#!/Clients/get_connections) acepta el scope `read:connections`, mientras que el endpoint [Create a Connection](https://auth0.com/docs/api/management/v2#!/Clients/post_connections) acepta el scope `write:connections`. Nuestro token de máquina a máquina solo debería necesitar el scope `read:connections` para acceder a los datos de ese endpoint.

Si tienes varias aplicaciones que deben acceder a la Management API, deberías crear aplicaciones de máquina a máquina independientes para cada aplicación en Auth0, en lugar de una sola aplicación de máquina a máquina.
