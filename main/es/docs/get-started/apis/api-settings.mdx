description: Describe la configuración relacionada con las APIs disponibles en el Auth0 Dashboard.
title: Configuración de APIs
---

Usa la pestaña **Settings** en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis), para configurar las APIs registradas que puedes consumir desde tus aplicaciones autorizadas. Para configurar los ajustes de una API, haz clic en **...** junto a una API en la lista y selecciona **Settings**, o haz clic en el nombre de la API. Para obtener información sobre cómo crear y registrar una API, consulta [Registrar APIs](/es/docs/get-started/auth0-overview/set-up-apis).

<Frame>![Lista de APIs en Dashboard > Applications](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>

<div id="settings">
  ## Configuración
</div>

Usa las opciones de esta pestaña para configurar la expiración del token, el control de acceso basado en roles (RBAC) y otras opciones de acceso. Haz clic en **Guardar** al final de la pestaña para guardar los cambios.

<div id="general-settings">
  ### Configuración general
</div>

Estos campos se configuraron cuando registró la API inicialmente, excepto en el caso de la Auth0 <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Solo puede modificar el **Nombre**.

<Frame>![Dashboard Applications APIs Settings Tab Configuración general](/docs/images/cdy7uua7fh8z/QCaXd3FSeP6SYqLfEJWBr/fbb1bb5b2e12fb099073c33fb39cf44a/dashboard-applications-apis-settings-general-settings.png)</Frame>

* **Id**: Una cadena alfanumérica única generada por Auth0. Esta información es de solo lectura y solo la necesitará si va a trabajar directamente con los [endpoints de API Resource Servers de la Management API de Auth0](https://auth0.com/docs/api/management/v2#!/Resource_Servers/get_resource_servers_by_id).
* **Name**: Un nombre descriptivo para la API. No afecta a ninguna funcionalidad. No se permiten los siguientes caracteres: `< >`.
* **Identifier**: Un identificador único para su API. Este valor se establece al crear la API y no se puede modificar después. Recomendamos usar una URL, pero no tiene que ser una URL disponible públicamente; Auth0 no llamará a su API en ningún caso.

<div id="token-settings">
  ### Configuración de tokens
</div>

<Frame>![Dashboard - API - Configuración de tokens - Expiración](/docs/images/cdy7uua7fh8z/2FFfqqmntXL9CnGrSxea9M/810e5542553168ffcd6081a10bc2db2f/image2.png)</Frame>

* **Duración máxima del <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">token de acceso</Tooltip> (segundos)**: El tiempo (en segundos) que transcurre antes de que venza un token de acceso. El valor predeterminado es 86400 segundos (24 horas). El valor máximo que puedes establecer es 2592000 segundos (30 días).
* **Duración del token de acceso del flujo implícito / híbrido (segundos)**: El tiempo (en segundos) que transcurre antes de que venza un token de acceso emitido mediante un flujo implícito o híbrido. El valor predeterminado es 86400 segundos (24 horas). El valor no puede superar la duración máxima del token de acceso.
* **Perfil de <Tooltip tip="JSON Web Token (JWT): formato estándar de token de ID (y con frecuencia también de token de acceso) usado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT)**: El perfil determina el formato de los tokens de acceso emitidos para la API. Los valores disponibles son `Auth0` y `RFC 9068`. Para obtener más información, consulta [Perfiles de token de acceso](/es/docs/secure/tokens/access-tokens/access-token-profiles).
* **JSON Web Token (JWT) <Tooltip tip="Signing Algorithm: algoritmo utilizado para firmar digitalmente tokens y garantizar que el token no haya sido manipulado." cta="Ver glosario" href="/es/docs/glossary?term=Signing+Algorithm">Algoritmo de firma</Tooltip>**: El algoritmo con el que se firman los tokens. La firma se utiliza para verificar que el remitente del JWT es quien dice ser y para garantizar que el mensaje no se haya modificado durante el trayecto. Los valores disponibles son [`HS256`](https://en.wikipedia.org/wiki/Symmetric-key_algorithm) y [`RS256`](https://en.wikipedia.org/wiki/Public-key_cryptography). Si seleccionas `RS256` (recomendado), el token se firmará con la clave privada de tu tenant. Este valor se establece cuando se crea la API y no puede modificarse después. Para obtener más información sobre los algoritmos de firma y cómo funcionan en Auth0, consulta [Algoritmos de firma](/es/docs/get-started/applications/signing-algorithms).

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    La firma forma parte de un JWT. Si no estás familiarizado con la estructura de un JWT, consulta [Estructura de JSON Web Token](/es/docs/secure/tokens/json-web-tokens/json-web-token-structure).
  </Callout>
* **JSON Web Encryption (JWE):** Cuando se habilita, los tokens de acceso emitidos se cifran con el formato JSON Web Encryption (JWE). Para obtener más información, consulta [JSON Web Encryption](/es/docs/secure/tokens/access-tokens/json-web-encryption).

<div id="rbac-settings">
  ### Configuración de RBAC
</div>

<Frame>![Interruptor de RBAC en la configuración de la API en Auth0 Dashboard](/docs/images/cdy7uua7fh8z/65tKb6aj0ktc2qXLUVlV3e/641adef615d6af9e5a3b588ff397af87/dashboard-apis-edit_view-settings_rbac-settings.png)</Frame>

* **Habilitar RBAC**: Habilite esta configuración para que las políticas de RBAC se apliquen a la API. Para obtener más información, consulte [Control de acceso basado en roles](/es/docs/manage-users/access-control/rbac) y [Habilitar el control de acceso basado en roles para las API](/es/docs/get-started/apis/enable-role-based-access-control-for-apis).
* **Agregar Permiso al token de acceso**: Habilite esta configuración para agregar el claim Permiso al token de acceso. Esta configuración solo está disponible si habilita RBAC. Puede configurar permisos en la pestaña **Permiso**.

<div id="access-settings">
  ### Configuración de acceso
</div>

<Frame>![Dashboard Applications APIs pestaña Settings Configuración de acceso](/docs/images/cdy7uua7fh8z/JNlWACqoyfmpE89JwWLKy/5a2b595f939167b0e6e1f72b485b08e7/2025-02-25_14-53-21.png)</Frame>

* **Permitir omitir el consentimiento del usuario**: Activa esta configuración para que la API omita el consentimiento del usuario en las aplicaciones marcadas como &quot;de primera parte&quot;.
* **Permitir acceso sin conexión**: Activa esta configuración para permitir que las aplicaciones soliciten <Tooltip tip="Token de actualización: Token que se utiliza para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=refresh+tokens">tokens de actualización</Tooltip> para la API.

<div id="default-policy-setting">
  #### Configuración de la política predeterminada
</div>

<Warning>
  El nivel de seguridad de autenticación para la **My Account API** se encuentra actualmente en acceso anticipado con una política de una sola opción. Al usar esta funcionalidad, aceptas los términos aplicables de la prueba gratuita del [Master Subscription Agreement](https://www.okta.com/legal) de Okta. Para obtener más información sobre el ciclo de lanzamiento de productos de Auth0, consulta [Etapas de lanzamiento del producto](/es/docs/troubleshoot/product-lifecycle/product-release-stages). Para participar en el programa, contacta al [equipo de soporte de Auth0](https://support.auth0.com).
</Warning>

<Frame>![Configuración de la API de política predeterminada](/docs/images/api-settings-access-default-policy.png)</Frame>

* **Requerir 2FA**: Cuando está habilitado, Auth0 aplica la política predeterminada a la My Account API para activar la [autenticación escalonada](/es/docs/secure/multi-factor-authentication/step-up-authentication). Los usuarios deben autenticarse con un segundo factor en un plazo de 15 minutos. Para conocer el comportamiento completo y cómo habilitarla, consulta [My Account API: política predeterminada](/es/docs/manage-users/my-account-api#default-policy).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La política predeterminada no está disponible con Classic Login.
</Callout>

  ## Permisos
</div>

Usa las opciones de la pestaña **Permisos** para definir los permisos (`scopes`) que usará la API. Para obtener más información, consulta [Agregar permisos de API](/es/docs/get-started/apis/add-api-permissions) y [Eliminar permisos de API](/es/docs/get-started/apis/delete-api-permissions).

  ## Aplicaciones de máquina a máquina
</div>

Si tiene aplicaciones de máquina a máquina, aparecerán en una lista en esta pestaña. Use los interruptores para autorizar las aplicaciones de la lista.
