---
description: Describe algunas limitaciones del uso de las conexiones sin contraseña.
title: Limitaciones de las conexiones sin contraseña
---

El tipo de autenticación que desea usar depende de la versión de <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip> que haya configurado para su aplicación. Si usa la experiencia de Universal Login, puede configurar la [autenticación sin contraseña con WebAuthN y biometría del dispositivo](/es/docs/authenticate/login/auth0-universal-login/passwordless-login/webauthn-device-biometrics) o usar conexiones <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=passwordless">sin contraseña</Tooltip>, como se describe a continuación.

Puede configurar la autenticación sin contraseña por correo electrónico o SMS con los siguientes métodos:

1. Universal Login + parámetros de conexión sin contraseña
2. Classic Login + Lock
3. Classic Login + interfaz de usuario personalizada + Auth0.js

Para obtener más información sobre estas implementaciones, consulte [Configurar el correo electrónico o SMS para la autenticación sin contraseña](/es/docs/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="errors">
  ### Errores
</div>

Si usas un inicio de sesión integrado, te recomendamos usar <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o distintivo." cta="Ver glosario" href="/es/docs/glossary?term=custom+domains">dominios personalizados</Tooltip>. Para obtener más información, consulta [Dominios personalizados](/es/docs/customize/custom-domains).

<div id="considerations">
  ## Consideraciones
</div>

* No puedes crear usuarios sin contraseña desde el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Créalos directamente mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Users/post_users).
* Los <Tooltip tip="Refresh Token: Token usado para obtener un token de acceso renovado sin obligar a los usuarios a iniciar sesión de nuevo." cta="Ver glosario" href="/es/docs/glossary?term=Refresh+tokens">Refresh tokens</Tooltip> no se pueden obtener al usar un enlace mágico sin contraseña por correo electrónico, solo con un OTP.
