Puedes configurar la autenticación <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">Passwordless</Tooltip> para enviar una contraseña de un solo uso (OTP) a un usuario por SMS. Para obtener más información, consulta [Configurar correo electrónico o SMS para la autenticación Passwordless](/es/docs/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

Auth0 admite el uso de [Twilio](https://www.twilio.com) y un proveedor telefónico personalizado para enviar OTP. Si deseas usar un proveedor telefónico personalizado con Auth0 Actions, consulta [Configurar un proveedor telefónico personalizado](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) o [Configurar un proveedor telefónico personalizado con Terraform](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/configure-a-custom-phone-provider-with-terraform).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un usuario nuevo recibe un código y lo introduce por primera vez en tu aplicación, su perfil de usuario se crea en la conexión `sms` antes de que Auth0 lo autentique.

Si el número de teléfono al que se envió el OTP coincide con el de un usuario existente, Auth0 autentica al usuario:

<div id="configure-the-phone-provider">
  ## Configura el proveedor telefónico
</div>

1. En el Auth0 Dashboard, ve a [Branding &gt; Phone Provider](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Activa el interruptor de Phone Message Provider.

   <Frame>![Auth0 Dashboard > Branding > Phone Provider > Enable Phone Message Provider](/docs/images/cdy7uua7fh8z/1k93jvdxDq2FZLiRdvr5aR/d917431cd0265943c5118174a8673118/2025-02-24_16-43-14.png)</Frame>
3. Selecciona tu proveedor de mensajería telefónica.

   1. Para usar Twilio, sigue [Configura Twilio como proveedor de mensajería telefónica](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-twilio-as-a-phone-messaging-provider).
   2. Para usar un proveedor personalizado, sigue [Configura un proveedor telefónico personalizado](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) para usar Auth0 Actions.
4. Selecciona tu método de entrega.
5. **Save** los cambios.

<div id="configure-passwordless-sms-settings">
  ### Configurar los ajustes de SMS de passwordless
</div>

1. En **Mensaje**, introduce el texto del cuerpo del SMS.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     El marcador `@@password@@` se sustituirá automáticamente por la contraseña de un solo uso que se envía al usuario.
   </Callout>
2. Ajusta la configuración de **Vencimiento del OTP** y **Longitud del OTP**.

   * Solo se aceptará la última contraseña de un solo uso (o enlace) emitida. En cuanto se emita la más reciente, las anteriores quedarán invalidadas. Una vez utilizada, la más reciente también quedará invalidada.
   * Solo se permiten tres intentos fallidos de introducir la contraseña de un solo uso. Después de eso, tendrás que solicitar un código nuevo.
   * La contraseña de un solo uso emitida será válida (de forma predeterminada) durante tres minutos antes de caducar.
   * Si decides ampliar el tiempo que tarda en caducar tu contraseña de un solo uso, también deberías aumentar la longitud del código. De lo contrario, un atacante dispondrá de más tiempo para intentar adivinar un código corto.
3. Decide si quieres **Deshabilitar registros**. Si habilitas esta configuración, puedes permitir el acceso passwordless solo a usuarios existentes, pero también podrías exponer tu aplicación al riesgo de ataques de enumeración de usuarios. Para obtener más información, consulta [Prácticas recomendadas para las conexiones passwordless](/es/docs/authenticate/passwordless/best-practices).
4. Selecciona **Guardar**.

<div id="multi-language-support">
  #### Soporte multilingüe
</div>

El área de **mensaje** admite varios idiomas.

Para especificar un idioma, llama al [endpoint Get Code or Link de Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) y establece el valor del encabezado `x-request-language`. Si no se establece este encabezado, el idioma se extrae del encabezado `accept-language`, que el navegador configura automáticamente.

<div id="message-syntax">
  #### Sintaxis del mensaje
</div>

El campo **Mensaje** admite la sintaxis Liquid. Puedes usar esta sintaxis, combinada con valores de parámetros, para generar mediante programación elementos del mensaje.

Por ejemplo, puedes hacer referencia al parámetro `request_language` para cambiar el idioma del mensaje:

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ password }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ password }}
{% endif %}
```

Los siguientes parámetros están disponibles al definir la plantilla del mensaje:

| Parámetro            | Descripción                                                   |
| -------------------- | ------------------------------------------------------------- |
| `password` or `code` | La contraseña que se utilizará.                               |
| `phone_number`       | El número de teléfono del usuario.                            |
| `application.name`   | El nombre de la aplicación con la que el usuario se registra. |
| `request_language`   | El idioma solicitado para el contenido del mensaje.           |

Para obtener más información sobre el uso de Liquid, consulta [Liquid for Designers on GitHub](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers).
