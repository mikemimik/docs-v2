Puedes configurar una conexión <Tooltip tip="Passwordless: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">Passwordless</Tooltip> para enviar una contraseña de un solo uso (OTP) a un usuario por correo electrónico y completar la autenticación.

Para obtener más información, lee [Configurar Email o SMS para autenticación Passwordless](/es/docs/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un usuario nuevo recibe un OTP y lo introduce por primera vez en tu sitio web, su perfil de usuario se crea en la conexión `email` antes de que Auth0 lo autentique.

Si la dirección de correo electrónico a la que se envió el OTP pertenece a un usuario existente, Auth0 lo autentica:

<div id="configure-the-connection">
  ## Configura la conexión
</div>

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless) y habilita el interruptor **Email**.
2. Selecciona **Email** para abrir la ventana de configuración, cambia a la vista **Settings** y luego ingresa el texto de **From**, **Subject** y **Message** del correo electrónico.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Debes cambiar el valor de **From** por una dirección de correo electrónico que no use el dominio `auth0.com` para que Auth0 pueda enviar tus plantillas de correo electrónico personalizadas. Si no lo haces, Auth0 enviará la plantilla de correo electrónico predeterminada.
   </Callout>
3. Configura **OTP Expiry** y **OTP Length.**

   * Solo se aceptará el último OTP (o enlace) emitido. Una vez que se emita el más reciente, cualquier otro quedará invalidado. Una vez usado, el más reciente también quedará invalidado.
   * Solo se permiten tres intentos fallidos de introducir el OTP. Después de eso, tendrás que solicitar un nuevo código.
   * El OTP emitido será válido (de forma predeterminada) durante tres minutos antes de expirar.
   * Si decides ampliar el tiempo que tarda tu OTP en expirar, también deberías aumentar su longitud. De lo contrario, un atacante tendrá una ventana de tiempo mayor para intentar adivinar un código corto.
4. Decide si quieres **Disable Sign Ups**. Si habilitas esta configuración, puedes permitir el acceso passwordless solo a los usuarios existentes, pero podrías exponer tu aplicación al riesgo de ataques de enumeración de usuarios. Para obtener más información, consulta [Prácticas recomendadas para las conexiones passwordless](/es/docs/authenticate/passwordless/best-practices).
5. Selecciona **Save**.

<div id="multi-language-support">
  ### Compatibilidad multilingüe
</div>

El campo **Message** admite varios idiomas.

Para especificar un idioma, llama al [endpoint Get Code or Link de Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) y establece el valor del encabezado `x-request-language`. Si no se establece este encabezado, el idioma se extrae del encabezado `accept-language`, que el navegador configura automáticamente.

<div id="message-syntax">
  ### Sintaxis de Message
</div>

El área de **Message** admite la sintaxis Liquid. Puedes usar esta sintaxis, junto con los valores de los parámetros, para crear mediante programación elementos del mensaje.

Por ejemplo, puedes hacer referencia al parámetro `request_language` para cambiar el idioma del mensaje:

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ code }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ code }}
{% endif %}
```

Los siguientes parámetros están disponibles al definir la plantilla del mensaje:

| Parameter          | Description                                                                                                                                                                                 |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `code`             | La contraseña que se debe usar.                                                                                                                                                             |
| `link`             | El enlace de inicio de sesión generado.                                                                                                                                                     |
| `application.name` | El nombre de la aplicación con la que el usuario se está registrando.                                                                                                                       |
| `request_language` | El idioma solicitado para el contenido del mensaje.                                                                                                                                         |
| `operation`        | Indica si la plantilla se ha activado por una actualización del correo electrónico de un usuario mediante la API. Cuando se activa, el valor es `change_email`; de lo contrario, es `null`. |

Si el usuario ha iniciado sesión a través de una [organización](/es/docs/manage-users/organizations/organizations-overview), los siguientes parámetros también están disponibles:

* `organization.id`
* `organization.display_name`
* `organization.name`
* `organization.metadata`
* `organization.branding.logo_url`
* `organization.branding.colors.primary`
* `organization.branding.colors.page_background`

Para obtener más información sobre estos parámetros y sus valores, consulta el endpoint [Get organization](https://auth0.com/docs/api/management/v2#!/Organizations/get_organizations_by_id) de Auth0 <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

<div id="enable-applications">
  ## Habilitar aplicaciones
</div>

Cambia a la vista **Applications** y habilita las aplicaciones con las que quieras usar Passwordless Email.

<div id="email-providers">
  ## Proveedores de correo electrónico
</div>

Auth0 envía correos electrónicos desde su propio proveedor SMTP de forma predeterminada. El proveedor de correo electrónico integrado de Auth0 está diseñado únicamente para fines de prueba y no admite la personalización de las plantillas de correo electrónico.

[Configura tu propio proveedor de correo electrónico SMTP](/es/docs/customize/email/smtp-email-providers) para gestionar, supervisar y resolver problemas en tus comunicaciones por correo electrónico, además de personalizar las plantillas de correo electrónico.

Auth0 admite los siguientes proveedores de correo electrónico:

* [Mandrill](/es/docs/customize/email/smtp-email-providers/configure-mandrill-as-external-smtp-email-provider)
* [Amazon SES](/es/docs/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider)
* [Azure Communication Services](/es/docs/customize/email/smtp-email-providers/configure-azure-comm-service-as-smtp-email-provider)
* [SendGrid](/es/docs/customize/email/smtp-email-providers/configure-sendgrid-as-external-smtp-email-provider)
* [SparkPost](/es/docs/customize/email/smtp-email-providers/configure-sparkpost-as-external-smtp-email-provider)
* [Mailgun](/es/docs/customize/email/smtp-email-providers/configure-mailgun-as-external-smtp-email-provider)
* [Proveedor de correo electrónico SMTP externo personalizado](/es/docs/customize/email/smtp-email-providers/configure-custom-external-smtp-email-provider)
