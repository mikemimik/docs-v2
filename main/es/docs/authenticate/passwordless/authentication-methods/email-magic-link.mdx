Puedes configurar una conexión de <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">Passwordless</Tooltip> para enviar un Magic Link a un usuario por correo electrónico y completar la autenticación.

Para obtener más información, consulta [Configurar correo electrónico o SMS para la autenticación passwordless](/es/docs/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un usuario abre un Magic Link, Auth0 inicia su sesión directamente. Funciona de forma similar a recibir un correo electrónico con una contraseña de un solo uso (OTP), volver a tu aplicación e introducir el OTP, pero sin tener que realizar realmente esos pasos.

<div id="limitations">
  ### Limitaciones
</div>

Magic Links no son compatibles con <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip>.

Al usar Magic Links con Classic Login, tanto la solicitud inicial como la respuesta deben realizarse en el mismo navegador; de lo contrario, la transacción fallará. Esto es especialmente relevante para los usuarios de iOS, que no pueden cambiar su navegador web predeterminado.

Por ejemplo, el usuario podría hacer la solicitud inicial desde Chrome, pero cuando abra el Magic Link en su correo electrónico, iOS lo abrirá automáticamente en Safari (el navegador predeterminado). Si esto ocurre, la transacción fallará.

Luego, el usuario hace clic en el botón o enlace del correo electrónico y accede automáticamente a tu aplicación.

<div id="configure-the-connection">
  ## Configura la conexión
</div>

1. En el Auth0 Dashboard, ve a [Authentication &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless) y activa el interruptor **Email**.
2. Selecciona **Email** para abrir la ventana de configuración, cambia a la vista **Settings** e introduce el texto de **From**, **Subject** y **Message** del correo electrónico.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Debes cambiar el valor de **From** por una dirección de correo electrónico que no use el dominio `auth0.com` para que Auth0 pueda enviar tus plantillas de correo electrónico personalizadas. Si no lo haces, Auth0 enviará la plantilla de correo electrónico predeterminada.
   </Callout>
3. Introduce los **Authentication Parameters** que quieras incluir en el enlace de inicio de sesión generado.
4. Decide si quieres **Disable Sign Ups**. Si habilitas esta configuración, puedes permitir acceso passwordless solo a los usuarios existentes, pero también podrías exponer tu aplicación a ataques de enumeración de usuarios. Para obtener más información, consulta [prácticas recomendadas para las conexiones passwordless](/es/docs/authenticate/passwordless/best-practices).
5. Selecciona **Save**.

<div id="multi-language-support">
  ### Soporte multilingüe
</div>

El área de **Message** admite varios idiomas.

Para especificar un idioma, llama al [endpoint Get Code or Link de Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) y establece el valor del encabezado `x-request-language`. Si este encabezado no se establece, el idioma se extrae del encabezado `accept-language`, que el navegador configura automáticamente.

<div id="message-syntax">
  ### Sintaxis de Message
</div>

El campo **Message** admite la sintaxis Liquid. Puedes usar esta sintaxis, combinada con valores de parámetros, para crear mediante programación elementos del mensaje.

Por ejemplo, puedes hacer referencia al parámetro `request_language` para cambiar el idioma del mensaje:

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ code }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ code }}
{% endif %}
```

Los siguientes parámetros están disponibles al definir la plantilla del mensaje:

| Parameter          | Description                                                                                                                                                                                    |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `code`             | La contraseña que se debe usar.                                                                                                                                                                |
| `link`             | El enlace de inicio de sesión generado.                                                                                                                                                        |
| `application.name` | El nombre de la aplicación con la que el usuario se registra.                                                                                                                                  |
| `request_language` | El idioma solicitado para el contenido del mensaje.                                                                                                                                            |
| `operation`        | Indica si la plantilla se activó debido a una actualización del correo electrónico de un usuario a través de la API. Cuando se activa, el valor es `change_email`; de lo contrario, es `null`. |

<div id="update-classic-login-settings">
  ## Actualizar la configuración de Classic Login
</div>

Si usas Classic Login para tu aplicación, debes actualizar la plantilla de la página para implementar Magic Links.

1. En el Auth0 Dashboard, ve a [**Branding &gt; Universal Login &gt; Advanced Options**](https://manage.auth0.com/#/login_settings).

2. Cambia a la vista **Login**, busca el menú desplegable **Default Templates** y selecciona **Lock (passwordless).**

3. A continuación, actualiza la sección de código `Auth0LockPasswordless` con un parámetro nuevo. Agrega `passwordlessMethod` a tu plantilla.

   ```bash lines
   passwordlessMethod: 'link',
   ```

4. Selecciona **Save**.

Para obtener más información sobre las opciones de configuración de Embedded Login, consulta [Autenticación passwordless con Embedded Login](/es/docs/authenticate/passwordless/implement-login/embedded-login).

<div id="enable-applications">
  ## Habilitar aplicaciones
</div>

Cambia a la vista **Applications** y habilita las aplicaciones en las que quieras usar Passwordless Email.

<div id="email-providers">
  ## Proveedores de correo electrónico
</div>

Auth0 envía correos electrónicos desde su propio proveedor SMTP de forma predeterminada. El proveedor de correo electrónico integrado de Auth0 está diseñado únicamente para pruebas y no permite personalizar las plantillas de correo electrónico.

[Configura tu propio proveedor de correo electrónico SMTP](/es/docs/customize/email/smtp-email-providers) para gestionar, supervisar y solucionar los problemas de tus comunicaciones por correo electrónico, así como personalizar las plantillas de correo electrónico.

Auth0 admite los siguientes proveedores de correo electrónico:

* [Mandrill](/es/docs/customize/email/smtp-email-providers/configure-mandrill-as-external-smtp-email-provider)
* [Amazon SES](/es/docs/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider)
* [SendGrid](/es/docs/customize/email/smtp-email-providers/configure-sendgrid-as-external-smtp-email-provider)
* [SparkPost](/es/docs/customize/email/smtp-email-providers/configure-sparkpost-as-external-smtp-email-provider)
* [Mailgun](/es/docs/customize/email/smtp-email-providers/configure-mailgun-as-external-smtp-email-provider)
* [Proveedor externo de correo electrónico SMTP personalizado](/es/docs/customize/email/smtp-email-providers/configure-custom-external-smtp-email-provider)
