<div id="require-multi-factor-authentication-for-users-who-are-outside-the-corporate-network">
  ## Requerir autenticación multifactor para los usuarios que están fuera de la red corporativa
</div>

Supongamos que quieres exigir [autenticación multifactor (MFA)](/docs/es/secure/multi-factor-authentication) a cualquier usuario que acceda a la aplicación mediante una conexión <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">sin contraseña</Tooltip> desde fuera de tu red corporativa.

Con una regla, puedes comprobar si un usuario se está autenticando mediante un método sin contraseña (`sms`, `email`) y, si la IP de su sesión está fuera de la red corporativa designada, pedirle un segundo factor de autenticación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  También podrías activar esta regla en función de otros criterios, como si la IP actual coincide con la lista de IP permitidas del usuario o si la geolocalización del usuario indica que está en un país distinto del que figura en su perfil de usuario.
</Callout>

Para ello, [crea la siguiente regla](/docs/es/customize/rules/create-rules):

```javascript lines
function(user, context, callback) {
  const ipaddr = require('ipaddr.js');
  const corp_network = "192.168.1.134/26";
  const current_ip = ipaddr.parse(context.request.ip);
  // ¿el método de autenticación es sin contraseña y la IP está fuera de la red corporativa?
  const passwordlessOutside = context.authentication.methods.find(
    (method) => (
      ((method.name === 'sms') || (method.name === 'email')) && 
      (!current_ip.match(ipaddr.parseCIDR(corp_network)))
    )
  );

  // si es así, solicitar MFA
  if (passwordlessOutside) {
    context.multifactor = {
      provider: 'any',
      allowRememberBrowser: false
    };
  }
  callback(null, user, context);
}
```
