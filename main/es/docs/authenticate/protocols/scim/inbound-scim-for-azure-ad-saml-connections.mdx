<Card title="Antes de empezar">
  Para usar esta función, tu plan de Auth0 o tu acuerdo personalizado debe incluir Enterprise Connections. Para obtener más información, consulta la [página de precios de Auth0](https://auth0.com/pricing).

  Para probar una conexión de Inbound SCIM, debes instalar [Postman](https://www.postman.com/downloads/) en tu equipo.
</Card>

Para aprovisionar usuarios en tu endpoint SCIM de Auth0, debes configurar una aplicación empresarial no integrada en la galería en Microsoft Entra ID (antes Azure Active Directory).

<div id="configure-scim-settings-in-auth0">
  ## Configurar la configuración de SCIM en Auth0
</div>

1. En el [<Tooltip tip="..." cta="Ver glosario" href="/es/docs/glossary?term=Auth0%20Dashboard">Auth0 Dashboard</Tooltip>](https://manage.auth0.com/#/), ve a **Autenticación &gt; Empresarial &gt; SAML &gt; [YOUR&#95;AUTH0&#95;CONNECTION] &gt; Aprovisionamiento**.
2. Activa **Sincronizar perfiles de usuario mediante SCIM**. Esto abre una nueva sección en la misma página con configuración adicional.
3. Verifica la siguiente configuración:
   A. En la pestaña **Asignación**, asegúrate de que el campo **Atributo SCIM que contiene el ID de usuario** esté configurado como **userName**.
   B. En la sección **Asignaciones adicionales**, verifica los siguientes valores:
   ```json lines
   {
       "scim": "emails[type eq \"work\"].value",
       "auth0": "email"
   },
   ```
   También puedes consultar la [asignación de atributos](/es/docs/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) y asegurarte de que los atributos SCIM extendidos se asignen a los atributos de Auth0 que prefieras.
4. Si es necesario, selecciona **Guardar cambios**.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Obtener la URL del endpoint de SCIM y el token
</div>

Para configurar SCIM en un tenant de Microsoft Entra ID, necesitas una URL del endpoint de SCIM y un token de Auth0. Puedes obtener estos valores desde el Auth0 Dashboard:

1. En el [Auth0 Dashboard](https://manage.auth0.com/#/), ve a **Authentication &gt; empresarial &gt; SAML &gt; [your-connection] &gt; aprovisionamiento**.
2. En la sección **Sync user profiles using SCIM &gt; Setup**, copia la **SCIM Endpoint URL**.
3. En la misma sección, en **Bearer Token**, selecciona **Generate New Token**. Elige la fecha de vencimiento del token y los scopes, y luego selecciona **Generate Token**. La configuración predeterminada requiere los scopes que usa Azure AD: `get:users`, `post:users`, `patch:users` y `delete:users`. Si lo deseas, también puedes seleccionar `get:groups`, `post:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.
4. Copia el bearer token. Por seguridad, esta es la única vez que el token será visible.

También puedes obtener estos valores mediante programación usando los endpoints de SCIM de la Management API, como [`/v2/connections/{id}/scim-configuration/tokens`](https://auth0.com/docs/api/management/v2/connections/get-scim-tokens).

Para consultar las prácticas recomendadas, lee las [guías de implementación para configurar Inbound SCIM](/es/docs/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines).

<div id="configure-scim-in-entra-id-for-saml-apps">
  ## Configurar SCIM en Entra ID para aplicaciones SAML
</div>

Si la aplicación SAML aún no está registrada, siga [Configure IdP-Initiated SAML Login with Azure as the IdP](https://support.auth0.com/center/s/article/Configure-IdP-Initiated-SAML-Login-with-Azure-as-the-IdP) para registrar una aplicación empresarial personalizada que no sea de la galería en un tenant de Entra ID.

Luego, para configurar SCIM:

1. En el centro de administración de Microsoft Entra, vaya a **Entra ID &gt; Aplicaciones empresariales &gt; [YOUR&#95;APPLICATION]**.
2. Vaya a **Administrar &gt; Propiedades** y establezca **Se requiere asignación** en **Sí**.
3. Vaya a **Administrar &gt; Usuarios y grupos** y asigne los usuarios de Azure AD que quiera aprovisionar. Cuando asigna un <Tooltip tip="..." cta="Ver glosario" href="/es/docs/glossary?term=Group">grupo</Tooltip>, se aprovisionan los usuarios de ese grupo.
4. Vaya a **Administrar &gt; Aprovisionamiento** y seleccione **Comenzar**.
5. Establezca **Modo de aprovisionamiento** en **Automático**.
6. Expanda la sección **Credenciales de administrador** y configure los siguientes parámetros:
   * Establezca el campo **URL del tenant** con la URL del endpoint de SCIM que guardó en el paso anterior y agregue el query parameter `?aadOptscim062020` al final de la URL. Esta es la [solución alternativa recomendada por Microsoft para un problema conocido de SCIM 2.0](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/application-provisioning-config-problem-scim-compatibility).
   * Establezca **Token secreto** con el valor del token que guardó en el paso anterior y, a continuación, seleccione **Guardar**.
7. Expanda la sección **Asignaciones**.
8. Si quiere aprovisionar grupos de forma opcional, confirme que **Aprovisionar grupos de Entra ID** esté establecido en **Habilitado**. De lo contrario, establézcalo en **Deshabilitado**.
9. Luego, seleccione **Aprovisionar usuarios de Azure Active Directory** para abrir la pantalla **Asignación de atributos**.
   <Frame>![Asignación de atributos de Azure SAML](/docs/images/cdy7uua7fh8z/6JnGm4IeBglKiHNoVPX5vs/ddc17baeca48f02a6309b57ab87f845e/attributemap2.jpg)</Frame>
10. En la fila de la tabla de asignación de atributos que contiene `emails[type eq "work"].value` y `mail`, seleccione **Editar** para ir a la pantalla **Editar atributo** y configure los siguientes parámetros:
    * Establezca **Hacer coincidir objetos con este atributo** en **Sí**.
    * Establezca **Prioridad de coincidencia** en **2**.
11. Seleccione **Aceptar** para guardar la configuración. Esto cierra la pantalla **Editar atributo**. Luego, en la pantalla **Asignación de atributos**, seleccione **Guardar**.

<div id="testing">
  ## Pruebas
</div>

1. En la pantalla de información general de la aplicación empresarial, ve a **Manage &gt; aprovisionamiento &gt; Provision on Demand** para probar la conexión SCIM.
2. Ve a **Select a user or group** e introduce el nombre de un usuario o grupo que hayas asignado a la aplicación; luego, selecciona el elemento y elige **Provision**. Esto crea el usuario o grupo en el tenant de Auth0.
3. Establece **aprovisionamiento Status** en **On** para [aprovisionar todos los usuarios y grupos asignados](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status).
