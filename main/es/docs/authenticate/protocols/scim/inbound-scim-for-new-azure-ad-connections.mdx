<Card title="Antes de empezar">
  Asegúrate de que tu tenant de Auth0 esté conectado a Azure AD mediante el [tipo de conexión Microsoft Azure AD](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

  Esta integración requiere registrar dos aplicaciones en Azure AD: la integración de OpenID Connect y la integración de SCIM. Para agilizar este proceso de configuración para tus clientes, considera [publicar tu aplicación en la galería de aplicaciones de Azure Active Directory](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/v2-howto-app-gallery-listing#submit-your-application).
</Card>

Auth0 puede integrarse con Microsoft Azure Active Directory (ahora denominado Microsoft Entra ID) mediante el [tipo de conexión Microsoft Azure AD](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2), que utiliza el protocolo <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) para autenticar a los usuarios. Estas instrucciones son para conexiones nuevas de Azure AD; para conexiones más antiguas que usan identificadores de usuario de tipo pairwise (`sub)`), consulta [Conexiones antiguas](/es/docs/authenticate/protocols/scim/inbound-scim-for-older-azure-ad-connections).

<div id="configure-scim-settings-in-auth0">
  ## Configura la configuración de SCIM en Auth0
</div>

1. Abre el [Auth0 Dashboard](https://manage.auth0.com/#/), ve a **Authentication &gt; Enterprise &gt; Microsoft Azure AD &gt; [tu-conexión] &gt; Settings.**
2. Asegúrate de que **User ID Attribute Type** esté configurado como **User Object Identifier (oid)** y de que el interruptor **Use Common Endpoint** esté **Disabled**.
3. Selecciona la **pestaña Provisioning** y desactiva **Sync user profile attributes at each login**, a menos que quieras [sincronizar atributos adicionales al iniciar sesión](/es/docs/authenticate/protocols/scim/configure-inbound-scim).
4. En la misma sección, activa **Sync user profiles using SCIM**.
5. En la pestaña **Mapping**, asegúrate de que la opción **SCIM attribute containing the User ID** esté configurada como **externalId**.
6. Revisa los **Additional Mappings** para asegurarte de que los atributos SCIM extendidos estén asignados a los atributos de Auth0 que prefieras. Consulta [mapeo de atributos](/es/docs/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para obtener más información.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Obtén la URL del endpoint de SCIM y el token
</div>

En esta sección se usa el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, pero estos pasos también se pueden realizar con la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Consulta la sección [Pautas de implementación](/es/docs/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines) para conocer las prácticas recomendadas.

1. En el [Auth0 Dashboard](https://manage.auth0.com/#/), ve a la pestaña **Setup**, copia la **SCIM Endpoint URL** y pégala en un lugar seguro.
2. Genera un token de SCIM haciendo clic en **Generate New Token** y, si lo deseas, establece una fecha de expiración para el token.
3. Selecciona los scopes que quieres permitir. Los scopes predeterminados que requiere Azure AD son `get:users`, `post:users`, `patch:users` y `delete:users`. Opcionalmente, también puedes seleccionar `get:groups`, `post:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.

<div id="configure-scim-in-azure-ad-for-oidc-apps">
  ## Configurar SCIM en Azure AD para aplicaciones OIDC
</div>

1. Confirma que ya se haya registrado una aplicación OpenID Connect para gestionar la autenticación de usuarios en la sección **Microsoft Entra ID &gt; App registrations** del [portal de Azure](https://portal.azure.com).
2. Confirma que tu aplicación OpenID Connect tenga **Assignment Required** establecido en **Yes** en la sección **Microsoft Entra ID &gt; Enterprise applications &gt; [your-oidc-app] &gt; Manage &gt; Properties**, y que tenga usuarios asignados en la pestaña **Users and Groups**.
3. A continuación, registra una nueva aplicación **Non-gallery** en el [portal de Azure](https://portal.azure.com) desde **Microsoft Entra ID &gt; Enterprise applications &gt; New application &gt; Create your own application,** escribe un nombre para la aplicación y selecciona **Create**.
4. Ve a la pestaña **Users and Groups** y asigna los mismos usuarios y grupos de Azure AD que están asignados a la aplicación OpenID Connect registrada.
5. Selecciona la pestaña **Provisioning**, luego **Get started**, y elige **Automatic** como **Provisioning Mode.**
6. Selecciona **Admin Credentials** y luego introduce el valor de **SCIM Endpoint URL** que guardaste antes como **Tenant URL**. Al final de la URL, agrega el query parameter `?aadOptscim062020` para corregir los [problemas conocidos de Azure AD que se describen aquí.](https://learn.microsoft.com/en-us/azure/active-directory/app-provisioning/application-provisioning-config-problem-scim-compatibility)
7. Pega el valor del token en el campo **Secret Token** y selecciona **Save.**
8. Ve a **Mappings**.
9. Para aprovisionar grupos de forma opcional, confirma que **Provision Entra ID Groups** esté establecido en **Enabled**. De lo contrario, establécelo en **Disabled**.
10. Selecciona **Provision Microsoft Entra ID Users,** luego ve a **Attribute Mappings** y edita los atributos de la línea que contiene `externalId` y `mailNickname`.
11. En la pantalla **Edit Attribute**, cambia **Source attribute** a **objectId,** luego elige **OK**.
12. Vuelve a **Attribute Mappings** y selecciona la línea que contiene `emails[type eq \"work\"].value and mail`.
13. En la pantalla **Edit Attribute**, cambia **Match objects using this attribute** a **Yes**, luego establece **Matching precedence** en **2** y elige **OK.** La pantalla de mapeo de atributos se ve así mientras sigues usando la sección **Attribute Mappings** para configurar atributos SCIM adicionales:

<Frame>![Mapeo de atributos de SAML Azure](/docs/images/cdy7uua7fh8z/6JnGm4IeBglKiHNoVPX5vs/ddc17baeca48f02a6309b57ab87f845e/attributemap2.jpg)</Frame>

**Save** los mapeos de atributos y luego selecciona **X** en la esquina superior derecha para volver a la pantalla **Provisioning**.

<div id="testing">
  ## Pruebas
</div>

1. En la pantalla de información general de la aplicación empresarial, ve a **Manage &gt; Provisioning &gt; Provision on Demand** para probar la conexión SCIM.
2. Ve a **Select a user or group** e introduce el nombre de un usuario o grupo que hayas asignado a la aplicación; luego selecciona el elemento y elige **Provision**. Esto crea el usuario o grupo en el tenant de Auth0.
3. Establece **Provisioning Status** en **On** para [aprovisionar todos los usuarios y grupos asignados](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status).
