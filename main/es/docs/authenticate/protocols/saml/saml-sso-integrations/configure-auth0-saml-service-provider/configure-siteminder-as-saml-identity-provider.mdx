<div id="prerequisite">
  ## Requisito previo
</div>

Debes tener una cuenta de [SiteMinder](https://www.siteminder.com/login/).

<div id="configure-siteminder-settings">
  ## Configurar los ajustes de SiteMinder
</div>

Al configurar SiteMinder, usarás los valores predeterminados para la mayoría de las opciones. También necesitarás los siguientes valores relacionados con Auth0 en los pasos de configuración que aparecen a continuación:

* **EntityID:** `urn:auth0:{yourTenant}`
* **Assertion Consumer Service URL:** `https://{yourDomain}/login/callback`
* **URL de cierre de sesión:** `https://{yourDomain}/logout`
* **binding HTTP-Redirect** para la solicitud SAML
* **binding HTTP-POST** para la respuesta SAML

1. Inicia sesión en [SiteMinder](https://www.siteminder.com/login/).
2. Abre el **SAML Service Provider Dialog**.
3. Proporciona un nombre adecuado para este proveedor de servicios.
4. Define el `NameIdentifier`. Hay muchas formas de generarlo para los usuarios que se autentican con SiteMinder. Normalmente, este valor se asigna a una de las propiedades del usuario en el directorio de usuarios como `uid`.
5. Configura las propiedades SAML generales del proveedor de servicios:

| Configuración    | Descripción              |
| ---------------- | ------------------------ |
| **SP ID**        | `urn:auth0:{yourTenant}` |
| **SAML Version** | 2.0                      |
| **Skew Time:**   | 30 segundos              |

6. Configura la **Assertion Consumer Service URL**, que es la ubicación a la que SiteMinder enviará mediante POST el token SAML. Este proveedor de servicios ({yourTenant}) solo admite el binding `HTTP-POST` para respuestas SAML. Usa:
   `https://{yourDomain}/login/callback`
7. Marca la casilla **HTTP-Post**.
8. Configura propiedades de usuario adicionales para enviarlas en el token. Agrega cualquier otra propiedad que quieras compartir sobre el usuario autenticado a este proveedor de servicios. Los valores comunes son: `name`, `lastname`, `email address`, etc. Este proveedor de servicios usará el `NameIdentifier` definido anteriormente como identificador único del usuario. Estos atributos se tratarán como información de referencia.
9. Introduce la **SLO Location URL:**`https://{yourDomain}/logout`
10. (Opcional) El proveedor de servicios admite el cifrado de aserciones.
    Para usar esta opción, haz lo siguiente para descargar el certificado de clave pública del proveedor de servicios.

    1. En el Auth0 Dashboard, haz clic en **Connections** y luego en **Enterprise.**
    2. Haz clic en SAMLP Identity Provider.
    3. Haz clic en el icono de configuración (lápiz).
    4. En la ventana que aparece, el séptimo (último) punto con viñeta incluye enlaces para descargar el certificado en formato .pem o .cer.
    5. Descarga el certificado deseado y agrégalo al **Policy Server Keystore** de SiteMinder.
