title: Supervisar eventos de passkeys en los logs del tenant
---

Puedes monitorear los flujos de passkeys asociados con tus conexiones de base de datos a través de los [logs del tenant](/es/docs/deploy-monitor/logs).

Puedes ver los eventos de log en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ([monitoreo &gt; Logs](https://manage.auth0.com/#/logs)) o recuperar logs mediante la [Management API](https://auth0.com/docs/api/management/v2/logs/get-logs).

| Código de evento                | Escenario                                                                                       | Notas                                                                                                                                                                                                                                                                      |
| ------------------------------- | ----------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ss`                            | Durante el registro, un usuario registró una passkey como su método principal de autenticación. | Este evento de log contiene el campo de arreglo `details.authentication_methods` con el valor de entrada `passkey`.<br /><br />Si un usuario se registró con una contraseña, este campo de arreglo contiene el valor de entrada `pwd`.                                     |
| `s`                             | Durante el login, un usuario se autenticó con una passkey.                                      | Este evento de log contiene el arreglo `details.prompts`, que enumera los prompts por los que pasó un usuario durante el login.<br /><br />Si un usuario se autenticó con una passkey, el arreglo contiene un elemento con el siguiente campo: `"performed_amr": ["phr"]`. |
| `gd_enrollment_complete`        | Se completó correctamente un flujo de inscripción progresiva o local.                           | Este evento de log contiene el campo `details.authenticator.type` con el valor `passkey` si un usuario registró una passkey.                                                                                                                                               |
| `gd_webauthn_enrollment_failed` | Se produjo un error durante un flujo de inscripción progresiva o local.                         | Este evento de log contiene el campo de arreglo `details.authentication_methods` con el valor de entrada `passkey` si un usuario intentó registrar una passkey.                                                                                                            |
