---
title: Conexiones de base de datos personalizadas
description: Aprenda a autenticar usuarios con su base de datos como proveedor de identidad.
validatedOn: 2026-02-25
---

Use una conexión de base de datos personalizada cuando quiera dar a Auth0 acceso a su propio repositorio independiente (heredado) de datos de identidad, principalmente para la autenticación (cumpliendo la función de un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+provider">proveedor de identidad</Tooltip>) y para migrar datos de usuarios al almacén de datos de Auth0.

La [extensibilidad](/docs/es/customize/extensions) de Auth0 le permite agregar lógica personalizada para crear soluciones a medida de última milla para la gestión de identidades y accesos (IdAM). Auth0 ofrece extensibilidad mediante [Actions](/docs/es/customize/actions) y [scripts](/docs/es/authenticate/database-connections/custom-db/templates), tanto para conexiones de base de datos personalizadas como para la migración con bases de datos personalizadas. Cada una de estas opciones se implementa con [Node.js](https://nodejs.org/en/), que se ejecuta en la plataforma de Auth0 dentro de un tenant de Auth0.

La extensibilidad de Auth0 se ejecuta en distintos puntos del pipeline de IAM:

* **Actions** son funciones seguras, específicas del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma de Auth0. Las Actions se usan para personalizar y ampliar las capacidades de Auth0 con lógica personalizada. Para obtener más información, lea [Comprenda cómo funcionan las Actions de Auth0](/docs/es/customize/actions/actions-overview).

* **Los scripts de acción de base de datos personalizados** pueden usarse para integrarse con un almacén de identidad de usuario existente, o cuando se requiere la [migración automática de usuarios](/docs/es/manage-users/user-migration) desde un almacén de identidad heredado. Se proporcionan varias [plantillas](/docs/es/authenticate/database-connections/custom-db/templates).

Sea cual sea el caso de uso, la extensibilidad de Auth0 le permite adaptar las operaciones de IAM a sus necesidades exactas. Sin embargo, si no se usa correctamente, puede dar lugar a usos inadecuados o no intencionados que, con el tiempo, deriven en situaciones problemáticas. Para ayudar a prevenir estos problemas desde el principio, Auth0 proporciona [guía de prácticas recomendadas](/docs/es/authenticate/database-connections/custom-db/custom-database-connections-scripts) tanto para quienes diseñan como para quienes implementan, y recomendamos leerla completa al menos una vez, incluso si ya ha comenzado su recorrido con Auth0.
