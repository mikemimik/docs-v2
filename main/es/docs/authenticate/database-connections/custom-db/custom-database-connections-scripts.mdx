La extensibilidad permite a los administradores agregar inicios de sesión personalizados en Auth0 como mecanismo para crear soluciones de última milla para Identity and Access Management (IdAM). Cada tipo de extensibilidad usa Node.js y se ejecuta en la plataforma de Auth0 dentro de un tenant de Auth0.

La extensibilidad de Auth0 se presenta de varias formas:

* [Actions](/es/docs/customize/actions): Las Actions son funciones seguras, específicas del tenant y versionadas, escritas en Node.js, que se ejecutan en determinados puntos de la plataforma de Auth0. Usa Actions para personalizar y ampliar las capacidades de Auth0 con inicio de sesión personalizado.
* [Rules](/es/docs/customize/rules): Se ejecutan cuando se generan artefactos de autenticidad del usuario. Por ejemplo:

  * ID Token en OpenID Connect (OIDC)
  * Token de acceso en OAuth 2.0
  * Una aserción en SAML
* [Hooks](/es/docs/customize/hooks): Proporcionan extensibilidad adicional cuando hay un intercambio de artefactos no relacionados con el usuario y cuando se crean identidades de usuario, como en el prerregistro y el posregistro de usuarios.

A continuación se presentan algunas mejores prácticas para usar los productos de extensibilidad de Auth0:

* Los scripts, tanto para conexiones de bases de datos personalizadas como para migraciones, se usan para integrarse con un almacén de identidades de usuario existente o cuando se requiere la migración automática de usuarios desde un almacén de identidades independiente o heredado. Para obtener más información sobre la configuración, consulta [Create Custom Database Connections.](/es/docs/authenticate/database-connections/custom-db/create-db-connection)
* Para evitar la duplicación de usuarios, establece un `user_id` coherente en cada perfil de usuario devuelto. Además, asegúrate de no devolver la misma dirección de correo electrónico para dos usuarios diferentes en el script `get_user` o `login`, y de que ambos scripts devuelvan el mismo usuario.
* Para mantener la compatibilidad con versiones anteriores, `app_metadata` se llama `metadata` en los scripts de bases de datos personalizadas. Como la transferencia de datos puede interrumpirse, usa una Action para obtener los metadatos que deberían estar en `app_metadata` o `user_metadata` si faltan.
* Asegúrate de restringir el acceso a la <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Se denomina aud en un token, y su valor contiene el ID de una aplicación (Client ID) para un ID Token o de una API (API Identifier) para un token de acceso." cta="Ver glosario" href="/es/docs/glossary?term=audience">audiencia</Tooltip> con una [Action](/es/docs/customize/actions).

La extensibilidad de Auth0 es una capacidad integral y sofisticada para adaptar las operaciones de IdAM, pero también puede exponer tu base de datos a usos indebidos o no intencionados. Los documentos de esta sección destacan mejores prácticas tanto para diseñadores como para implementadores; te recomendamos leerlos completos al menos una vez, incluso si ya has comenzado tu recorrido con Auth0.
