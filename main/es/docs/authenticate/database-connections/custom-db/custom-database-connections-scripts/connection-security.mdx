<div id="access-legacy-identity-storage-via-custom-api">
  ## Acceda al almacén de identidades heredado mediante una API personalizada
</div>

Le recomendamos implementar una API para aplicar el principio de privilegio mínimo a su almacén de identidades heredado, en lugar de simplemente abrir un acceso general a través de internet.

Proteger el almacén de identidades heredado frente al acceso general es una práctica recomendada. Exponer directamente a internet una base de datos de identidades heredadas, por ejemplo, puede ser extremadamente problemático: las interfaces de bases de datos para SQL y similares son muy abiertas en cuanto a funcionalidad, lo que vulnera el principio de privilegio mínimo desde el punto de vista de la seguridad.

La alternativa es crear una API (personalizada) sencilla, protegida mediante el uso de un <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip>, a la que pueda llamar cada script de acción. Esta actuaría como interfaz del almacén de identidades heredado. Luego puede usarse el flujo de concesión de credenciales de cliente para obtener un token de acceso desde un script, y este puede almacenarse posteriormente en caché dentro del objeto `global` para reutilizarlo y mejorar el rendimiento. Después, la API puede ofrecer un número limitado de endpoints protegidos que realicen únicamente la funcionalidad de gestión de identidades heredadas necesaria (por ejemplo, leer usuarios o cambiar la contraseña).

De forma predeterminada, Auth0 le dará un token para cualquier API si se autentica correctamente e incluye la <Tooltip tip="Audiencia: Identificador único de la audiencia de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/es/docs/glossary?term=audience">audiencia</Tooltip> adecuada. Restringir el acceso a la API del almacén de identidades heredado limitando la asignación de tokens de acceso mediante el uso de una rule evitará usos no autorizados y mitigará varios escenarios de ataque, como aquellos en los que se intercepta la redirección a `/authorize` y se añade la audiencia de la API.

<Warning>
  Restringir el acceso a la API mediante Rule mitigará escenarios de ataque, como aquellos en los que se intercepta la redirección a `/authorize` y se añade la audiencia de la API, y garantizará que solo se conceda acceso mediante credenciales de cliente específicas.
</Warning>

<div id="access-to-legacy-identity-storage">
  ## Acceso al almacén de identidades heredado
</div>

Tanto si administra el acceso al almacén de identidades heredado mediante una API personalizada como si utiliza la interfaz nativa proporcionada, debe restringir el acceso a la lista de direcciones IP asociadas a su tenant de Auth0. Para ver la lista de direcciones IP, consulte [Direcciones IP de Auth0 para listas de permitidos](/es/docs/secure/security-guidance/data-security/allowlist). Agregar URL a la lista de permitidos restringe el acceso al almacén de identidades heredado y garantiza que solo se permitan los scripts de acciones de base de datos personalizada definidos en Auth0.

<Warning>
  La lista de permitidos de direcciones IP de Auth0 se comparte entre todos los tenants de Auth0 definidos en una región. Nunca use la lista de permitidos como único método para proteger el acceso a su almacén de identidades heredado; hacerlo podría generar vulnerabilidades de seguridad que permitan el acceso no autorizado a sus usuarios.
</Warning>
