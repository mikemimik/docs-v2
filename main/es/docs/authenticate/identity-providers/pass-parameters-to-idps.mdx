Puedes pasar parámetros específicos del proveedor a un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP) durante la autenticación. Los valores pueden ser estáticos para cada conexión o dinámicos para cada usuario.

<div id="limitations">
  ## Limitaciones
</div>

Para esta configuración, ten en cuenta las siguientes restricciones:

* Solo se aceptan [parámetros válidos de OAuth 2.0/OIDC](http://openid.net/specs/openid-connect-core-1_0.html#AuthorizationEndpoint).
* No todos los IdP admiten parámetros upstream. Confírmalo con el IdP correspondiente antes de continuar con tu implementación.
* Los IdP de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip> no admiten parámetros upstream.

<div id="static-parameters">
  ## Parámetros estáticos
</div>

Usa parámetros estáticos para configurar tu conexión y enviar un conjunto estándar de parámetros al IdP cuando un usuario inicia sesión.

Para configurar parámetros estáticos, llama al endpoint [Crear una conexión](https://auth0.com/docs/api/management/v2/#!/Connections/post_connections) o [Actualizar una conexión](https://auth0.com/docs/api/management/v2/#!/Connections/patch_connections_by_id) de la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0 y pasa el objeto `upstream_params` dentro del objeto `options`, con los parámetros que quieras enviar al IdP.

<div id="example-wordpress">
  ### Ejemplo: WordPress
</div>

WordPress te permite pasar un parámetro `blog` opcional a su endpoint de autorización de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip> y solicitar automáticamente acceso al blog especificado para los usuarios cuando inician sesión. Para obtener más información, consulta la [documentación de OAuth 2.0 de WordPress](https://developer.wordpress.com/docs/oauth2/).

Para seguir este ejemplo, necesitarás una [conexión social de WordPress](https://marketplace.auth0.com/integrations/wordpress-social-connection) que funcione.

<div id="dynamic-parameters">
  ## Parámetros dinámicos
</div>

Usa parámetros dinámicos para configurar tu conexión de modo que, cuando el usuario inicie sesión, envíe al IdP un conjunto de parámetros con valores específicos para ese usuario.

Para configurar parámetros dinámicos, llama al endpoint [Create a connection](https://auth0.com/docs/api/management/v2/#!/Connections/post_connections) o [actualizar una conexión](https://auth0.com/docs/api/management/v2/#!/Connections/patch_connections_by_id) de Management API de Auth0, pasa el objeto `upstream_params` dentro del objeto `options` con los parámetros que quieras enviar al IdP y especifica el campo al que corresponde el parámetro mediante el atributo `alias`.

Aquí tienes un ejemplo del objeto `options` que retomaremos más adelante en el ejemplo X:

```json lines
{
  "options": {
    "upstream_params": {
      "screen_name": {
        "alias": "login_hint"
      }
    }
  }
}
```

<div id="available-fields">
  ### Campos disponibles
</div>

Estos son los campos disponibles del atributo `alias`:

* `acr_values`
* `audience`
* `client_id`
* `display`
* `id_token_hint`
* `login_hint`
* `max_age`
* `prompt`
* `resource`
* `response_mode`
* `response_type`
* `ui_locales`
