Por lo general, para probar una conexión, debes iniciar sesión en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Ve a [Auth0 Dashboard](https://manage.auth0.com/#) y selecciona **Authentication**; luego, selecciona el tipo de conexión que quieres probar.
2. Busca la conexión que quieres probar y selecciona **Try Connection** en el menú **More Options** (**...**).
3. Inicia sesión con el proveedor de identidad.
4. Espera a que aparezca la página **It Works!**, que muestra el resultado.

Auth0 simula el flujo de autenticación como si fuera una aplicación y muestra el perfil de usuario que se obtiene tras una autenticación correcta.

<div id="test-partner-connections">
  ## Probar conexiones de partners
</div>

Si está probando una conexión que pertenece a otra persona y no tiene credenciales de prueba con ella, no es posible iniciar sesión en el Auth0 Dashboard. Esto es habitual al conectarse a conexiones empresariales, como <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip> <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin una contraseña." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Providers">proveedores de identidad</Tooltip> (IdP) o Active Directory. ¿La solución? Sus partners pueden probar la nueva conexión.

1. Registre una aplicación de prueba.

   * Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y seleccione **Create Application**. Puede ponerle cualquier nombre (por ejemplo, `Test App`).
   * En la configuración de la aplicación recién creada, establezca **Allowed Callback URLs** en `http://jwt.io`.
   * Seleccione **Save Changes**.
2. Envíe a su partner el enlace para iniciar sesión.

   <AuthCodeBlock children={codeExample} language="http" />

   Reemplace estos dos parámetros:

   * `client_id`: ID de cliente de la aplicación creada en el paso 1.
   * `connection`: Nombre de la conexión que desea probar.
3. Pruebe la conexión. Cuando su partner siga el enlace, se le redirigirá a su proveedor de identidad configurado (la conexión). Tras autenticarse correctamente, se le devolverá a [JWT.io](http://jwt.io), donde se decodificarán todas las propiedades del usuario a partir del token.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La aplicación de prueba no es una aplicación real. [JWT.io](http://jwt.io) es un sitio web de prueba que decodifica tokens enviados en un fragmento de URL.
</Callout>
