---
description: Enumera todos los requisitos previos para instalar y configurar el conector.
title: Requisitos del sistema del conector AD/LDAP
---

Normalmente, el conector AD/LDAP debe instalarlo un administrador de sistemas o un ingeniero de operaciones, y no un desarrollador, ya que suele requerir acceso a recursos de producción. A continuación, se incluye una lista de comprobación de los aspectos que deben tener en cuenta antes de la instalación propiamente dicha:

<div id="host-servers">
  ## Servidores host
</div>

El Connector puede instalarse en un servidor existente, incluso en un Domain Controller. Sin embargo, lo más habitual es instalarlo en máquinas virtuales aprovisionadas específicamente para el Connector. En cualquier caso, el servidor host debe contar con las siguientes especificaciones y configuraciones de hardware y software:

<div id="hardware-requirements">
  ### Requisitos de hardware
</div>

* **Arquitectura**: x86 o x86-64
* **Núcleos de CPU**: mín. 1; se recomiendan 2
* **Almacenamiento**: 500 MB de espacio libre en disco
* **Sistema operativo**: El conector puede ejecutarse en Windows o Linux. Se requiere Windows si se va a utilizar la autenticación Kerberos.
* **RAM**: mín. 2 GB

<div id="windows-version">
  ### Versión de Windows
</div>

Recomendamos usar una versión compatible de Windows Server, como Windows Server 2016 o Windows Server 2019. El conector también puede funcionar en Windows Server 2012 R2.

<div id="time-synchronization">
  ### Sincronización horaria
</div>

Es muy importante que el reloj del servidor host de Connector se sincronice automáticamente con un servidor NTP. De lo contrario, Connector no podrá iniciarse y mostrará un error de desfase del reloj.

<div id="outbound-connectivity">
  ## Conectividad de salida
</div>

El servidor host requiere conectividad de red de salida con los siguientes servicios:

<div id="auth0">
  ### Auth0
</div>

El conector debe instalarse en un servidor con conectividad saliente al servicio de Auth0 en: `https://{yourDomain}` por el puerto 443.

El conector puede instalarse y configurarse detrás de un servidor proxy, pero no lo recomendamos. Habilite un proxy mediante la variable de entorno o de configuración `HTTP_PROXY`.

<div id="ldap">
  ### LDAP
</div>

El Connector debe instalarse en un servidor con acceso al servidor LDAP a través del puerto 389 para LDAP o del 636 para LDAPS. Antes de instalar el Connector, debe conocer la cadena de conexión LDAP y el DN base necesarios para conectarse a su directorio LDAP.

<div id="inbound-connectivity">
  ## Conectividad entrante
</div>

No es necesario habilitar la conectividad entrante al Connector, a menos que esté habilitada la autenticación con Kerberos o mediante certificados. En estos casos, los servidores en los que está instalado el Connector deben ser accesibles desde los navegadores de los usuarios a través del puerto 443. Si hay instalada más de una instancia de Connector, debe usar un equilibrador de carga para dirigir el tráfico a uno u otro Connector.

Para obtener más información, consulte [Configure AD/LDAP Connector Authentication with Kerberos](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-with-kerberos) o [Configure AD/LDAP Connector Authentication with Client Certificates](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-client-certificates).

<div id="service-account">
  ## Cuenta de servicio
</div>

El Connector se ejecutará con una cuenta de servicio, que debe ser un usuario del dominio y contar, como mínimo, con acceso de lectura al directorio. Necesitarás el nombre de usuario y la contraseña de esta cuenta durante la instalación.

<div id="one-connector-per-auth0-tenant">
  ## Un Connector por tenant de Auth0
</div>

Si configura varios tenants de Auth0, por ejemplo para aislar los entornos de desarrollo y producción, deberá configurar una conexión AD/LDAP en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y un conector AD/LDAP para cada tenant de Auth0 que necesite esta forma de autenticación. Un Connector está vinculado a una conexión específica dentro de un tenant de Auth0.

Es posible tener varios Connectors dentro de un mismo tenant de Auth0 si tiene varios directorios AD/LDAP con los que se autenticarán los usuarios. Por ejemplo, para dar soporte a distintos departamentos o clientes, cada uno con su propio directorio. Además, varios Connectors pueden apuntar al mismo directorio AD o LDAP, pero un Connector solo puede utilizarse con una única Auth0 Connection dentro de un tenant de Auth0.

<div id="high-availability">
  ## Alta disponibilidad
</div>

El Connector puede instalarse en varios servidores host para garantizar alta disponibilidad y redundancia (la mayoría de las organizaciones suelen implementar dos), por si uno de los servidores deja de estar disponible. Cada servidor tendrá los mismos requisitos indicados anteriormente. No se requiere un equilibrador de carga, ya que el propio servidor de Auth0 se encarga de ello, a menos que habilites Kerberos o la autenticación basada en certificados de cliente.

Para obtener más información, consulta [Implementar conectores AD/LDAP en entornos de alta disponibilidad.](/docs/es/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/ad-ldap-high-availability)
