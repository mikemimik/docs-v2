description: Describe cómo controlar cómo se establece el campo `email_verified` para Azure AD y ADFS.
title: Verificación de correo electrónico para Azure AD y ADFS
---

El perfil del usuario de Auth0 tiene un campo `email_verified`, que puede establecerse de distintas maneras según el tipo de conexión. En las conexiones de base de datos, los usuarios deben pasar por un flujo de validación del correo electrónico para que este quede verificado. En las conexiones federadas, los <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=identity+providers">proveedores de identidad</Tooltip> pueden devolver el campo `email_verified` según sus propios criterios.

Azure AD y ADFS no pueden garantizar que los correos electrónicos que devuelven hayan sido verificados:

* En ADFS, el administrador puede configurar cualquier correo electrónico que quiera.
* En Azure AD, según cómo esté configurado el tenant de Azure AD, las direcciones de correo electrónico que devuelve Azure AD pueden corresponder o no a buzones de Office. Auth0 no puede saber si es así o no.

Sin embargo, si sabe cómo está configurado y administrado Azure AD o ADFS, puede decidir confiar en que los correos electrónicos de esas cuentas están verificados.

Para cubrir ambas necesidades, las conexiones de Azure AD y ADFS tienen una propiedad **Email Verification** con dos valores:

* Establecer siempre `email_verified` en `true`
* Establecer siempre `email_verified` en `false`

La conexión de Azure AD también tiene una propiedad **Use Common Endpoint**. Cuando está habilitada, el usuario puede autenticarse con cualquier tenant de Azure AD. Dado que no es posible confiar en que cualquier tenant de Azure AD devolverá correos electrónicos verificados, la propiedad **Email Verification** deberá establecerse en **Always set** `email_verified` **to** `false`.

Cuando la propiedad se establece en **Always set** `email_verified` **to** `false`, los usuarios tendrán `email_verified` establecido en `false` la próxima vez que inicien sesión, a menos que la [sincronización de los atributos del perfil de usuario en cada inicio de sesión](/docs/es/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) esté deshabilitada.

<div id="azure-adadfs-email-verification-migration-setting">
  ## Configuración de migración de Email Verification para Azure AD/ADFS
</div>

En versiones anteriores, Auth0 siempre establecía el campo `email_verified` en `true` en las conexiones de Azure AD y ADFS. Si antes usabas conexiones de Azure AD y ADFS, tendrás una configuración del tenant que anulará la configuración de la conexión para **Email Verification** y mantendrá el comportamiento anterior.

Puedes encontrar la nueva configuración del tenant en [Auth0 Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced). Ve a la sección **Migrations** y busca **Default to** `Email Verification` **setting for Azure AD/ADFS connections**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Esta configuración solo está disponible si:

  * el tenant es anterior a la fecha en que se introdujo la migración
  * el tenant tiene una o más conexiones activas de ADFS o Azure AD
</Callout>

<Frame>![Dashboard - Advanced Tenant Settings - Migrations](/docs/images/cdy7uua7fh8z/4K1rQIv6p96KPCYOrhjXz4/be0f7b104d743fd0d3d42d18df97e144/dashboard-tenant-edit_view-advanced_migrations_azure-adfs.png)</Frame>

Cuando esta configuración está deshabilitada, `email_verified` siempre será `true` para las conexiones de Azure AD/ADFS. Cuando está habilitada, usará la configuración de `Email Verification` a nivel de conexión.

<div id="email-verification-flow-for-azure-adadfs-connections">
  ## Flujo de verificación de correo electrónico para conexiones de Azure AD/ADFS
</div>

Si tu aplicación requiere que los correos electrónicos de los usuarios de una conexión de Azure AD/ADFS estén siempre verificados, puedes habilitar la opción **Enable email verification flow during login for Azure AD and ADFS connections** en la sección **Advanced Settings** del tenant.

Después de que el usuario se autentique por primera vez con un correo electrónico no verificado, Auth0 le pedirá que verifique su correo electrónico introduciendo un código de un solo uso que se enviará a su cuenta de correo electrónico:

<Frame>![Auth0 - Prompt de verificación de correo electrónico - Código de un solo uso](/docs/images/cdy7uua7fh8z/i92GJgntVuU5QK37adugH/b767d4db92de5bafd23113042eca2600/Verify_Email_-_English.png)</Frame>

Si el usuario completa este paso, el campo `email_verified` se establecerá en `true` y no se le volverá a pedir que verifique su correo electrónico, a menos que Azure AD o ADFS devuelvan una dirección de correo electrónico diferente para ese usuario.

Esta nueva pantalla se renderiza mediante la experiencia de <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip>, incluso si estás usando Classic Login. Para saber cómo personalizarla, consulta [Personalizar las páginas de Universal Login](/docs/es/customize/login-pages/universal-login/customize-templates).

Para saber cómo personalizar el correo electrónico que se envía a los usuarios, consulta [Verificar correos electrónicos con Auth0](/docs/es/manage-users/user-accounts/verify-emails).

<Warning>
  Cuando Azure AD no devuelve un claim `email`, Auth0 asigna [Azure UserPrincipalName](https://docs.microsoft.com/en-us/azure/active-directory/hybrid/plan-connect-userprincipalname) como dirección de correo electrónico. No hay garantía de que el valor de `UserPrincipalName` corresponda a un buzón de correo, por lo que Auth0 **no** mostrará el prompt de verificación de correo electrónico y el campo `email_verified` del usuario se establecerá en `false`.
</Warning>
