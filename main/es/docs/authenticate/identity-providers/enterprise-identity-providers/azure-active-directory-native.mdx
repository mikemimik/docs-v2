title: Conecta tu aplicación nativa a Microsoft Azure Active Directory usando Resource
---

Además de los flujos de WS-Federation y <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect, también es posible usar el flujo de [Resource Owner](/es/docs/get-started/authentication-and-authorization-flow/resource-owner-password-flow) con Azure AD. Este flujo te permite capturar y validar las credenciales de un usuario (correo electrónico y contraseña) en lugar de mostrar la página de inicio de sesión de Azure AD. Por motivos de seguridad y de <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SSO), este no es el enfoque recomendado; aun así, el flujo de <Tooltip tip="Resource Owner: Entidad (como un usuario o una aplicación) capaz de otorgar acceso a un recurso protegido." cta="Ver glosario" href="/es/docs/glossary?term=Resource+Owner">Resource Owner</Tooltip> puede ser útil en escenarios de aplicaciones móviles nativas o para procesar por lotes la autenticación con Azure AD.

Esta configuración requiere dos aplicaciones: una aplicación web y/o una API web, y una aplicación nativa. Desde el punto de vista de Azure AD, los usuarios se autenticarán mediante la aplicación nativa para obtener acceso a la aplicación web y/o a la API web.

<Frame>![Diagrama general de las aplicaciones de Azure AD que usan el flujo de Resource Owner](/docs/images/cdy7uua7fh8z/6Uy9Q35rn0d9rankvlJuBT/6a5fa5175db76fd1cab35b79f0a213ea/azure-ad-native-app.png)</Frame>

<div id="prerequisites">
  ## Requisitos previos
</div>

[Registra tu aplicación en Auth0](/es/docs/get-started/auth0-overview/create-applications):

1. Selecciona **Native** en **Application Type**.
2. Agrega una **Allowed Callback URL**. El formato de tu URL de devolución de llamada variará según tu plataforma. Para más información sobre el formato correspondiente a tu plataforma, consulta nuestros [Quickstarts nativos](/es/docs/quickstart/native).
3. Asegúrate de que los [Grant Types](/es/docs/get-started/applications/update-grant-types) de tu aplicación incluyan los flujos adecuados.

<div id="steps">
  ## Pasos
</div>

Para conectar tu aplicación mediante el flujo de Resource Owner, debes:

1. [Configurar tus aplicaciones en el portal de Microsoft Azure](#set-up-your-applications-in-the-microsoft-azure-portal).
2. [Crear una conexión empresarial en Auth0](#create-an-enterprise-connection-in-auth0).
3. [Habilitar la conexión empresarial para tu aplicación de Auth0](#enable-the-enterprise-connection-for-your-auth0-application).
4. [Probar la conexión](#test-the-connection).

<Card title="Cuenta de Microsoft Azure">
  Antes de continuar, necesitarás una cuenta válida de Microsoft Azure y deberás tener **tu propio** directorio de Microsoft Azure AD, del que debes ser administrador global.

  Si no tienes una cuenta de Microsoft Azure, puedes [registrarte gratis](https://azure.microsoft.com/en-us/free); después, si es necesario, configura un directorio de Azure AD siguiendo la guía de Microsoft [Quickstart: Create a new tenant in Azure Active Directory - Create a new tenant for your organization](https://docs.microsoft.com/en-us/azure/active-directory/fundamentals/active-directory-access-create-new-tenant#create-a-new-tenant-for-your-organization).

  Como alternativa, si tienes una cuenta de Office 365, puedes usar la instancia de Azure AD de esa cuenta en lugar de crear una nueva. Para acceder a la instancia de Azure AD de tu cuenta de Office 365:

  1. [Inicia sesión en Office 365](https://portal.office.com) y ve al [Centro de administración de Office 365](https://portal.office.com/adminportal/home#/homepage).
  2. Abre el panel del menú **Centros de administración**, situado en el menú de la izquierda, y haz clic en **Azure AD**.
</Card>

<div id="register-a-new-web-application">
  ### Registrar una nueva aplicación web
</div>

Para saber cómo registrar tu aplicación en Azure AD, consulta [Quickstart: Register an application with the Microsoft identity platform on Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-register-app).

<Warning>
  Si tienes más de un directorio de Azure AD, asegúrate de estar en el directorio correcto al registrar tu aplicación.
</Warning>

Mientras configuras tu aplicación, asegúrate de usar la siguiente configuración:

* Si quieres permitir usuarios de organizaciones externas (como otros directorios de Azure AD), selecciona **Accounts in any organizational directory (Any Azure AD directory - Multitenant)** al configurar **Supported account types**.
* Cuando se te pida configurar una **Redirect URI**, selecciona **Web** e introduce tu URL de devolución de llamada: `https://{yourDomain}/login/callback`.

<Card title="Encuentra tu dominio de Auth0 para las redirecciones">
  Si tu dominio de Auth0 no aparece arriba y no estás usando nuestra funcionalidad de [dominios personalizados](/es/docs/customize/custom-domains), tu dominio se compone del nombre de tu tenant, tu subdominio regional y `auth0.com`, separados por un punto (`.`).

  Por ejemplo, si el nombre de tu tenant es `exampleco-enterprises` y tu tenant está en la región de EE. UU., tu dominio de Auth0 sería `exampleco-enterprises.us.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.us.auth0.com/login/callback`.

  Sin embargo, si tu tenant está en la región de EE. UU. y se creó antes de junio de 2020, tu dominio de Auth0 sería `exampleco-enterprises.auth0.com` y tu **Redirect URI** sería `https://exampleco-enterprises.auth0.com/login/callback`.

  Si estás usando [dominios personalizados](/es/docs/customize/custom-domains), tu **Redirect URI** sería `https://<YOUR CUSTOM DOMAIN>/login/callback`.
</Card>

Durante este proceso, anota el **ID de la aplicación (cliente)** que Microsoft genera para tu aplicación (puedes encontrarlo en la pantalla **Información general** de la aplicación), ya que lo necesitarás más adelante en el tutorial.

<div id="configure-your-web-application-to-expose-an-api">
  ### Configura tu aplicación web para exponer una API

Mientras configuras tu aplicación, asegúrate de usar la siguiente configuración:

* Cuando se te pida establecer un **nombre del scope**, introduce `API.Access`.

Durante este proceso, anota el **Application ID URI** que Microsoft genera para tu aplicación, ya que lo necesitarás más adelante en el tutorial.

<Warning>

<div id="create-a-client-secret-for-your-native-application">
  ### Crea un secreto de cliente para tu aplicación nativa
</div>

Para saber cómo crear un <Tooltip tip="Secreto de cliente: Secret utilizado por un cliente (aplicación) para autenticarse con el servidor de autorización; solo el cliente y el servidor de autorización deben conocerlo, y debe ser lo suficientemente aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/es/docs/glossary?term=client+secret">secreto de cliente</Tooltip>, consulta [Quickstart: Configurar una aplicación cliente para acceder a API web - Agregar credenciales a tu aplicación web en Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-configure-app-access-web-apis#add-credentials-to-your-web-application).

Genera un **Secreto de cliente** y anota su valor. Lo necesitarás más adelante en el tutorial.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si configuras un secreto con vencimiento, asegúrate de **anotar la fecha de vencimiento**; tendrás que renovar la clave antes de ese día para evitar una interrupción del servicio.
</Callout>

<div id="add-permissions-for-your-native-application">
  ### Agregar permisos para tu aplicación nativa
</div>

Para obtener información sobre cómo agregar permisos para tu aplicación **Native**, consulta [Quickstart: Configure a client application to access web APIs - Add permissions to access web APIs on Microsoft Docs](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-configure-app-access-web-apis#add-permissions-to-access-web-apis).

Auth0 requiere que habilites los permisos mínimos necesarios (para Microsoft Graph API y para la aplicación web que configuraste para exponer una API) para que esta configuración funcione correctamente. Para obtener más información sobre los permisos de Microsoft Graph API, consulta [Microsoft Graph permissions reference on Microsoft Docs](https://learn.microsoft.com/en-us/graph/permissions-reference).

Mientras configuras tus permisos, asegúrate de usar la siguiente configuración para Microsoft Graph API:

* Cuando se te pida un tipo de permiso, elige **Delegated permissions**.

  * En **User**, selecciona **User.Read** para que tu aplicación pueda permitir que los usuarios inicien sesión y leer el perfil del usuario que ha iniciado sesión.
  * En **Directory**, selecciona **Directory.Read.All** para que tu aplicación pueda leer datos del directorio en nombre del usuario que ha iniciado sesión.

Para tu aplicación web que configuraste para exponer una API, asegúrate de usar la siguiente configuración:

* Cuando se te pida un tipo de permiso, elige **Delegated permissions**.

  * En **API**, selecciona **API.Access** para que tu aplicación pueda acceder a tu API en nombre del usuario.

<div id="configure-the-connection-in-auth0">
  ## Configura la conexión en Auth0
</div>

Después de crear ambas aplicaciones en Azure AD, puedes configurar la conexión de Auth0:

1. Ve a [Auth0 Dashboard &gt; Authentication &gt; Enterprise](https://manage.auth0.com/#/connections/enterprise), busca **Microsoft Azure AD** y selecciona el botón Agregar (**+**).

   <Frame>![Dashboard - Conexiones - Enterprise](/docs/images/cdy7uua7fh8z/1fSTcrZpkgkPR64NnI1lr8/b3454e60a4463e99353603fd11a71983/Enterprise_Connections_-_EN.png)</Frame>
2. Introduce los datos de tu conexión y selecciona **Create**:

| Campo                                                         | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Nombre de la conexión**                                     | Identificador lógico de tu conexión; debe ser único para tu tenant. Una vez establecido, este nombre no se puede cambiar.                                                                                                                                                                                                                                                                                                                                                                    |
| **Dominio de Microsoft Azure AD**                             | El nombre de dominio de tu Azure AD. Puedes encontrarlo en la página de información general del directorio de Azure AD en el portal de Microsoft Azure.                                                                                                                                                                                                                                                                                                                                      |
| **ID de cliente**                                             | Identificador único de tu aplicación registrada en Azure AD. Introduce el valor guardado de **Application (client) ID** de la aplicación nativa que registraste en Azure AD.                                                                                                                                                                                                                                                                                                                 |
| **secreto de cliente**                                        | Cadena que se usa para obtener acceso a tu aplicación registrada en Azure AD. Introduce el valor guardado de **Client secret** de la aplicación nativa que registraste en Azure AD.                                                                                                                                                                                                                                                                                                          |
| **Usar endpoint común** (opcional)                            | Cuando está habilitado, tu aplicación aceptará dinámicamente usuarios de directorios nuevos. Normalmente se habilita si seleccionaste una opción multiinquilino para **Supported account types** en la aplicación que registraste en Azure AD. Cuando está habilitado, Auth0 redirigirá a los usuarios al endpoint común de login de Azure, y Azure realizará *Home Realm Discovery* en función del dominio de la dirección de correo electrónico del usuario.                               |
| **Identity API**                                              | API que Auth0 usa para interactuar con los endpoints de Azure AD. Consulta las diferencias de comportamiento en la documentación de Microsoft [Why update to Microsoft identity platform (v2.0)](https://docs.microsoft.com/en-us/azure/active-directory/develop/azure-ad-endpoint-comparison). Selecciona `Azure Active Directory (v1)` y, en **App ID URI**, introduce el valor guardado de **Application ID URI** que se creó cuando configuraste tu aplicación web para exponer una API. |
| **Atributos**                                                 | Atributos básicos del usuario que inició sesión a los que tu aplicación puede acceder. Indica cuánta información quieres almacenar en el perfil de usuario de Auth0.                                                                                                                                                                                                                                                                                                                         |
| **Atributos extendidos** (opcional)                           | Atributos extendidos del usuario que inició sesión a los que tu aplicación puede acceder.                                                                                                                                                                                                                                                                                                                                                                                                    |
| **APIs de Auth0** (opcional)                                  | Cuando se selecciona, indica que necesitas poder hacer llamadas a la API de Users de Azure AD.                                                                                                                                                                                                                                                                                                                                                                                               |
| **Sincronizar atributos del perfil de usuario en cada login** | Cuando está habilitado, Auth0 sincroniza automáticamente los datos del perfil de usuario en cada login, lo que garantiza que los cambios realizados en el origen de la conexión se actualicen automáticamente en Auth0.                                                                                                                                                                                                                                                                      |
| **Verificación del correo electrónico**                       | Elige cómo Auth0 establece el campo `email_verified` en el perfil de usuario. Para obtener más información, consulta [Email Verification for Azure AD and ADFS](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azuread-adfs-email-verification).                                                                                                                                                                                                                        |

<div id="group-memberships-and-advanced-profile-information">
  ## Pertenencia a grupos e información avanzada del perfil
</div>

En este flujo nativo, Auth0 recibirá un <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">token de acceso</Tooltip> de Azure AD emitido para su aplicación web de Azure AD. Como resultado, funciones como cargar la pertenencia a grupos e información avanzada del perfil dejarán de funcionar, porque el token de acceso recibido de Azure AD ya no puede usarse para consultar la Graph API de Azure AD y obtener esta información adicional.

Sin embargo, si depende de la pertenencia a grupos y de la información avanzada del perfil, puede cambiar su configuración para adaptarla a sus necesidades:

1. En Azure, configure su aplicación nativa con permisos adicionales para Microsoft Graph API:

   * Cuando se le solicite un tipo de permiso, elija **Delegated permissions**.

     * En **Directory**, seleccione **Directory.AccessAsUser.All** para que su aplicación pueda acceder al directorio como el usuario que ha iniciado sesión.
2. En Auth0, modifique su conexión empresarial de Azure AD:

   * En **Identity API**, seleccione `Azure Active Directory (v1)` y, en **App ID URI**, introduzca el URI de la Graph API de Azure AD:

     ```text lines
     https://graph.windows.net
     ```
