---

Auth0 admite el inicio de sesión empresarial tanto para aplicaciones web como para aplicaciones nativas. El inicio de sesión empresarial es un método de autenticación que permite a los usuarios iniciar sesión en una aplicación con credenciales existentes de un proveedor de identidad empresarial, como Google Workspace o Microsoft Azure Active Directory (Entra ID). Esto es distinto de conectar y autorizar aplicaciones para un proveedor externo a fin de que puedan acceder a API externas en nombre del usuario. Para obtener más información, consulta [Autenticación de usuarios vs Connected Accounts](/es/docs/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).

Auth0 admite de forma nativa los siguientes proveedores empresariales:

* [Active Directory/LDAP](/es/docs/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap)
* [ADFS](/es/docs/authenticate/identity-providers/enterprise-identity-providers/adfs)
* [Azure Active Directory Native](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory-native)
* [Google Workspace](/es/docs/authenticate/identity-providers/enterprise-identity-providers/google-apps)
* [OpenID Connect](/es/docs/authenticate/identity-providers/enterprise-identity-providers/oidc)
* [Okta](/es/docs/authenticate/identity-providers/enterprise-identity-providers/okta)
* [PingFederate](/es/docs/authenticate/identity-providers/enterprise-identity-providers/ping-federate)
* [SAML](/es/docs/authenticate/identity-providers/enterprise-identity-providers/saml)
* [Azure Active Directory](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2)
* [User authentication vs Connected Accounts](/es/docs/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los planes B2B y Enterprise de Auth0 permiten conexiones empresariales de Okta ilimitadas. Para obtener más información, consulta la [página de precios de Auth0](https://auth0.com/pricing).
</Callout>

También puedes explorar conexiones empresariales compatibles con el respaldo de partners en el [Auth0 Marketplace](https://marketplace.auth0.com/features/enterprise-connections).

<div id="openid-connect-oidc-protocol">
  ## Protocolo OpenID Connect (OIDC)
</div>

Las conexiones empresariales que usan <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect u Okta Workforce como <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar las identidades de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip> pueden admitir lo siguiente:

* Proof Key for Code Exchange (PKCE)

  * Para obtener más información sobre PKCE, consulta [Protecting Apps with PKCE](https://www.oauth.com/oauth2-servers/pkce/).
* claims de atributos y mapeo de <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip>
* Integración con UserInfo

Actualmente, puedes implementar estas funciones para las conexiones de [OpenID Connect](/es/docs/authenticate/identity-providers/enterprise-identity-providers/oidc) u [Okta Workforce](/es/docs/authenticate/identity-providers/enterprise-identity-providers/okta). Para obtener más información, consulta [Configure PKCE and Claim Mapping for OIDC Connections](/es/docs/authenticate/identity-providers/enterprise-identity-providers/configure-pkce-claim-mapping-for-oidc).
