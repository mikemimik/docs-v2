Una vez que el usuario haya iniciado sesión, puedes obtener su perfil de usuario y después el `accessToken` asociado para llamar a las API del <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/docs/es/glossary?term=Identity+Provider">proveedor de identidad</Tooltip> (IdP), como se describe en [Llamar a una API de un proveedor de identidad](/docs/es/authenticate/identity-providers/calling-an-external-idp-api).

Sin embargo, si recibes `Access Denied` al llamar a la API del IdP, probablemente no hayas solicitado los permisos adecuados para el usuario durante el inicio de sesión. Puedes solicitar los permisos adecuados de una de estas dos maneras.

<div id="change-identity-provider-settings">
  ## Cambiar la configuración del proveedor de identidad
</div>

Para configurar los scopes/permisos que se le solicitarán al usuario, ve a [Auth0 Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social) y selecciona un IdP. Puedes seleccionar los permisos necesarios que aparecen en la pantalla de configuración.

Por ejemplo, si haces clic en la conexión **Google / Gmail**, puedes configurar permisos específicos de Google:

<Frame>![Permisos para Google](/docs/images/cdy7uua7fh8z/61ACa6hnMtO5aUjus0fCb7/31411373a18463f272107e1124445c60/dashboard-connections-social-create_google.png)</Frame>

<div id="pass-scopes-to-authorize-endpoint">
  ## Pasa scopes al endpoint de autorización
</div>

También puedes pasar los scopes/permisos que quieras solicitar como una lista separada por comas en el parámetro `connection_scope` al llamar al [endpoint de autorización](https://auth0.com/docs/api/authentication#login). Por ejemplo, si quieres solicitar los scopes `https://www.googleapis.com/auth/contacts.readonly` y `https://www.googleapis.com/auth/analytics` de Google, puedes pasarlos junto con el parámetro `connection` para asegurarte de que el usuario inicie sesión con su cuenta de Google:

export const codeExample = `https://{yourDomain}/authorize
  ?response_type=id_token
  &client_id={yourClientId}
  &redirect_uri={https://yourApp/callback}
  &scope=openid%20profile
  &connection=google-oauth2
  &connection_scope=https%3A%2F%2Fwww.googleapis.com%2Fauth%2Fanalytics%2Chttps%3A%2F%2Fwww.googleapis.com%2Fauth%2Fcontacts.readonly
  &nonce=abc`;
