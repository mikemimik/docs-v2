---
title: Perfil de conexión
description: Usa un perfil de conexión para especificar cómo deben configurarse los ajustes privados de una Conexión de Auth0 cuando la crean terceros.
---

Auth0 ofrece conexiones empresariales para autenticar a los usuarios con un proveedor de identidad federado externo (IdP), como Okta, Microsoft Entra ID, Google Workspace y otros. Una conexión configurada incluirá ajustes específicos del protocolo para integrar inicio de sesión único (SSO), aprovisionamiento de usuarios e integraciones de cierre de sesión con el IdP externo, así como ajustes privados que determinan cómo la conexión interactúa con las funciones Universal Login y Organizations de Auth0.

El perfil de conexión (CP) permite a los desarrolladores de Auth0 especificar cómo deben configurarse los ajustes privados de una Conexión de Auth0 cuando la crean terceros

<div id="how-it-works">
  ## Cómo funciona
</div>

* **Definición del perfil**: Un administrador crea un perfil de conexión que define los valores de las propiedades que deben escribirse en la conexión cada vez que se cree mediante una de las funcionalidades de Administración delegada de Auth0.

* **Alcance flexible**: Actualmente, los perfiles están vinculados a los flujos de Self-Service Enterprise Configuration y Okta Express Configuration, pero están diseñados para reutilizarse de forma más amplia y abarcar el aprovisionamiento, la incorporación, la gestión de derechos y futuras capacidades de Auth0.

<div id="connection-profile-properties">
  ## Propiedades del perfil de conexión
</div>

Un perfil de conexión es un objeto JSON que admite estas propiedades configurables, que se aplican a todas las conexiones recién creadas.

| **Propiedad**                             | **Descripción**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ----------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `connection_name_prefix_template`         | Representa el prefijo que debe usarse para asignar nombres a las conexiones. Se corresponde con el parámetro `name` de una conexión. Este valor admite sustitución de variables para el ID de la organización y el nombre de la organización. El flujo de trabajo de OIN debe sustituir estos valores para generar el prefijo final. Las variables se escriben entre llaves ({}). <p /> Variables admitidas: <p /> <ul><li>`org_id`: El ID de la organización</li><li>`org_name`: El [nombre de la organización](/es/docs/manage-users/organizations/configure-organizations/create-organizations#auth0-dashboard)</li></ul> <p /> Ejemplo: `con-{org_id}-` |
| `enabled_features`                        | Esta lista especifica las funcionalidades compatibles con la conexión configurada. Las funcionalidades que no estén en la lista no se permiten.<p /> Valores admitidos: <ul><li>`scim`: Cuando está presente, SCIM puede configurarse en la conexión.</li><li>`universal_logout`: Cuando está presente, la funcionalidad Universal Logout puede usarse con esta conexión.</li></ul>                                                                                                                                                                                                                                                                      |
| `organization.assign_membership_on_login` | Especifica si a los usuarios se les debe asignar automáticamente membresía en la organización al iniciar sesión. Se corresponde con la propiedad `assign_membership_on_login` del subrecurso `enabled_connections` de la organización.  <Callout icon="file-lines" color="#0EA5E9" iconType="regular"> Para las integraciones de Express Configuration con Okta, `optional` se trata como `required`</Callout> <p />                                                                                                                                                                                                                                     |

| Valor del perfil de conexión  | Valor de `assign_membership_on_login` en Conexiones habilitadas                                                                                                                                                 |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `none`                        | `false`                                                                                                                                                                                                         |
| `optional`                    | `false`                                                                                                                                                                                                         |
| `required`                    | `true`                                                                                                                                                                                                          |
| `organization`                | Un objeto que contiene ajustes que se aplican a la organización en la que se crea la conexión.                                                                                                                  |
| `organization.show_as_button` | Especifica si la conexión debe mostrarse como un botón en la pantalla de inicio de sesión una vez que se haya elegido una organización. Se corresponde con el valor `show_as_button` de la conexión habilitada. |

| Valor del perfil de conexión | Valor de `show_as_button` de la conexión |
| ---------------------------- | ---------------------------------------- |
| `none`                       | `false`                                  |
| `optional`                   | `true`                                   |
| `required`                   | `true`                                   |

<div id="example-connection-profile">
  ## Ejemplo de perfil de conexión
</div>

```json
{
  "organization": {
    "show_as_button": "none",
    "assign_membership_on_login": "none"
  },
  "connection_name_prefix_template": "ec-{org_id}-",
  "enabled_features": [
    "scim",
    "universal_logout"
  ]
}
```

<div id="create-and-manage-connection-profiles">
  ## Crear y administrar perfiles de conexión
</div>

Un perfil de conexión se genera automáticamente al usar la función [Configuración exprés](/es/docs/authenticate/identity-providers/enterprise-identity-providers/okta/express-configuration) con Okta. Además, puede crear y administrar perfiles de conexión con la Auth0 Management API.

<div id="configure-with-management-api">
  ### Configurar con Management API
</div>

Para usar la Management API, debes obtener un [token de acceso de la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens). Para administrar perfiles de conexión, están disponibles los siguientes endpoints de la Management API:

* `POST` `/api/v2/connection-profiles`
* `GET` `/api/v2/connection-profiles`
* `PATCH` `/api/v2/connection-profiles/{id}`
* `GET` `/api/v2/connection-profiles/{id}`
* `GET` `/api/v2/connection-profiles/templates`
