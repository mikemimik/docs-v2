* [Completar los prerrequisitos](#prerequisites)

<div id="prerequisites">
  ## Requisitos previos
</div>

Antes de configurar Custom Token Exchange, asegúrate de que tu aplicación cumpla los siguientes requisitos:

* Sea un cliente de primera parte
* Sea compatible con OIDC

Para configurar tu aplicación, ve a **[Applications &gt; Configuración avanzada &gt; OAuth](https://auth0.com/docs/get-started/applications/application-settings#oauth)** en el Auth0 Dashboard.

<div id="enable-custom-token-exchange-for-your-application">
  ## Habilita Custom Token Exchange para tu aplicación
</div>

Para habilitar Custom Token Exchange, [crea una nueva aplicación](/docs/es/get-started/auth0-overview/create-applications) o actualiza una existente desde Auth0 Dashboard o la Management API. Puedes crear varias aplicaciones para usar Custom Token Exchange.

Cuando creas una nueva aplicación:

1. De forma predeterminada, Custom Token Exchange está deshabilitado. Para habilitar Custom Token Exchange, usa la Management API para realizar una solicitud `POST` a [Crear un cliente](https://auth0.com/docs/api/management/v2/clients/post-clients) o una solicitud `PATCH` a [actualizar un cliente](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id). Establece el atributo `allow_any_profile_of_type` dentro de `token_exchange` en `["custom_authentication"]`:

```json lines
{
  "token_exchange": {
    "allow_any_profile_of_type": ["custom_authentication"]
  }
}
```

2. Habilita la conexión que quieras usar con Custom Token Exchange para la aplicación.
3. Asegúrate de que tu aplicación esté marcada como [First-Party](/docs/es/get-started/applications/first-party-and-third-party-applications) y configurada como OIDC Conformant en **[Dashboard &gt; Applications &gt; Advanced Settings &gt; OAuth](/docs/es/get-started/applications/application-settings#oauth)**.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las Custom DBs con el modo de importación `ON` solo son compatibles con las operaciones `setUserById()`.
</Callout>

Una vez que crees la aplicación, toma nota de `client_id` y `client_secret` para usarlos más adelante al llamar al endpoint `/oauth/token`.

<div id="configure-custom-token-exchange-profile">
  ## Configurar el perfil de Custom Token Exchange
</div>

Cada perfil de Custom Token Exchange establece un mapeo de uno a uno entre un `subject_token_type` y una Action, que contiene la lógica de código para un caso de uso específico.

Las solicitudes de Custom Token Exchange enviadas al endpoint `/oauth/token` con un valor específico de `subject_token_type` se asignarán al perfil de Custom Token Exchange correspondiente y se dirigirán a la Action asociada para su procesamiento.

Para crear un perfil de Custom Token Exchange, debes:

1. [Crear una Action para el perfil](#create-action-for-the-profile)
2. [Crear el perfil de Custom Token Exchange](#create-custom-token-exchange-profile)
3. [Administrar el perfil de Custom Token Exchange](#manage-custom-token-exchange-profile)

<div id="create-action-for-the-profile">
  ### Crear una Action para el perfil
</div>

Use los objetos Event y API de Custom Token Exchange para escribir una Action que:

* Decodifique y valide el `subject_token` según el `subject_token_type`. Esto le proporcionará información sobre el usuario para la transacción.
* Aplique cualquier política de autorización que necesite para la transacción.

Una vez que esté seguro de que la transacción puede continuar, defina el usuario. Auth0 emitirá entonces tokens de acceso, de ID y de actualización para este usuario como parte de la autenticación del usuario.

Para ver ejemplos de Actions de Custom Token Exchange, lea [Casos de uso de ejemplo y muestras de código](/docs/es/authenticate/custom-token-exchange/cte-example-use-cases).

Una vez que haya escrito su Action, agréguela y despliegúela en el Auth0 Dashboard.

1. Vaya a **Actions &gt; Biblioteca**.
2. Seleccione **Crear Action &gt; Crear desde cero**.
3. En el cuadro de diálogo **Crear Action**, introduzca un nombre y seleccione el trigger **Custom Token Exchange** de la lista desplegable.

<Frame>![](/docs/images/cdy7uua7fh8z/22vz9dsCFj5Ruot7U0HIVx/1c3dc4b562334dab9d6ac415028ea76e/Screenshot_2025-02-05_at_8.48.34_AM.png)</Frame>

4. Seleccione **Crear**.
5. **Despliegue** la Action.

<Frame>![](/docs/images/cdy7uua7fh8z/56NZA69Gmzha167xfgRD0W/302e59276815d2e2644ab2da3b9b5f1f/Screenshot_2025-02-03_at_10.29.17_AM.png)</Frame>

Después de desplegar la Action, copie el ID de la Action que Auth0 le asignó. Aún necesita agregar su lógica personalizada a la Action. Primero, obtenga el ID de la Action para crear el perfil de Custom Token Exchange.

6. Para obtener el ID de la Action en el Auth0 Dashboard, vaya a la URL de la ventana del navegador. El ID de la Action debe ser la última parte de la URL, como se muestra en la siguiente imagen:

<Frame>![](/docs/images/cdy7uua7fh8z/1Xx4UbgZR0FIuLC1KVvhKG/bde4469d770c2ff8d37c19895a0c8e66/Screenshot_2025-02-03_at_10.31.18_AM.png)</Frame>

También puede obtener el ID de la Action mediante la Management API. Primero, obtenga un token de la Management API para consumir la API. Luego, realice la siguiente solicitud `GET` al endpoint `/actions`:

```bash lines
curl --location 'https://{yourDomain}/api/v2/actions/actions?actionName={yourActionName}' \
--header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
```

Deberías recibir el ID de la Action en el cuerpo de la respuesta, dentro de `actions[0].id.` Necesitas el ID de la Action para crear el perfil de Custom Token Exchange.

<div id="create-custom-token-exchange-profile">
  ### Crear perfil de Custom Token Exchange
</div>

Para crear el perfil de Custom Token Exchange, usa la Management API para enviar una solicitud `POST` con los siguientes parámetros al endpoint `/token-exchange-profiles`:

```bash lines
curl --location 'https://{yourDomain}/api/v2/token-exchange-profiles' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
--data '{
    "name": "<YOUR_PROFILE_NAME>",
    "subject_token_type": "<YOUR_UNIQUE_PROFILE_TOKEN_TYPE_URI>",
    "action_id": "<YOUR_ACTION_ID>",
    "type": "custom_authentication"
}'
```

| Parámetro            | Descripción                                                                                                                                                                                                                                                                                                                                              |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `subject_token_type` | URI único del tipo de token de perfil que comienza con `https://` o `urn`<br /><br />Los siguientes espacios de nombres están reservados y no se pueden usar:<br /><br /><ul><li>`http://auth0.com`</li><li>`https://auth0.com`</li><li>`http://okta.com`</li><li>`https://okta.com`</li><li>`urn:ietf`</li><li>`urn:auth0`</li><li>`urn:okta`</li></ul> |
| `action_id`          | ID de la Action asociada con el perfil de Custom Token Exchange.                                                                                                                                                                                                                                                                                         |
| `type`               | Debe establecerse como `custom_authentication`.                                                                                                                                                                                                                                                                                                          |

Si has creado correctamente un Custom Token Exchange Profile, deberías recibir una respuesta como la siguiente:

```json lines
{
  "id":"tep_9xqewuejpa2RTltf",
  "name":"<YOUR_PROFILE_NAME>",
  "type":"custom_authentication",
  "subject_token_type":"<YOUR_UNIQUE_PROFILE_TOKEN_TYPE_URI>",
  "action_id":"<YOUR_ACTION_ID>",
  "created_at":"2025-01-30T13:19:00.616Z",
  "updated_at":"2025-01-30T13:19:00.616Z"
}
```

<div id="manage-custom-token-exchange-profile">
  ### Administrar el perfil de Custom Token Exchange
</div>

Para administrar tu perfil de Custom Token Exchange, usa la Management API para realizar solicitudes al endpoint `/token-exchange-profiles`.

Para obtener todos tus perfiles de Custom Token Exchange, realiza la siguiente solicitud `GET` al endpoint `/token-exchange-profiles`. El endpoint `/token-exchange-profiles` admite paginación por checkpoint si tienes varios perfiles.

```bash lines
curl --location 'https://{yourDomain}/api/v2/token-exchange-profiles' \
--header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
```

Para actualizar el nombre o el `subject_token_type` de un perfil existente, realiza la siguiente solicitud `PATCH` al endpoint `/token-exchange-profiles`.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Una vez creada la Action, no puedes modificar el ID de la Action.
</Callout>

```bash lines
curl --location --request PATCH 'https://{yourDomain}/api/v2/token-exchange-profiles/{yourProfileId}' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
--data '{
    "name": "external-idp-migration",
    "subject_token_type": "urn:partner0:external-idp-migration"
}'
```

Para eliminar un Custom Token Exchange Profile, haz la siguiente solicitud `DELETE` al endpoint `/token-exchange-profiles`:

```bash lines
curl --location --request DELETE 'https://{yourDomain}/api/v2/token-exchange-profiles/{yourProfileId}' \
--header 'Authorization: Bearer <YOUR_MANAGEMENT_API_TOKEN>' \
--data ''
```
