title: Inicio de sesión nativo
---

Las aplicaciones móviles nativas pueden usar flujos de inicio de sesión nativos o basados en navegador.

En un flujo de inicio de sesión **basado en navegador**, se muestra al usuario un navegador web y se le redirige a la página de inicio de sesión de Auth0 para registrarse o iniciar sesión. Por ejemplo, una aplicación de iOS abre un SafariViewController o una aplicación de Android abre una pestaña personalizada de Chrome.

En un flujo de inicio de sesión **nativo**, el usuario se registra o introduce sus credenciales directamente en la aplicación.

Independientemente de la opción que elijas, Auth0 admite ambas.

<div id="considerations">
  ### Consideraciones
</div>

* Riesgos de phishing/seguridad: un tercero no autorizado podría descompilar o interceptar el tráfico hacia/desde tu aplicación para obtener el <Tooltip tip="Client ID: Valor de identificación otorgado a tu recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=Client+ID">Client ID</Tooltip> y la URL de autenticación. Con esta información, podría crear una aplicación maliciosa, subirla a una tienda de aplicaciones y usarla para hacer phishing y obtener nombres de usuario, contraseñas y <Tooltip tip="Access Token: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Tokens">tokens de acceso</Tooltip>.
* <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=SSO">SSO</Tooltip>: los usuarios deben introducir sus credenciales en cada aplicación.

  * Es posible implementar SSO con aplicaciones nativas almacenando tokens de actualización en un llavero compartido, pero esto no cumple con las especificaciones de OAuth 2.0.
* Lleva más tiempo implementarlo
* No hay mejoras automáticas cuando Auth0 agrega nuevas funcionalidades; hay que actualizar el código de la aplicación para aprovecharlas, a diferencia de UL
* No cumple con las prácticas recomendadas de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de autorización." cta="Ver glosario" href="/es/docs/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip>

  * [RFC 8252 OAuth 2.0 para aplicaciones nativas](https://tools.ietf.org/html/rfc8252): las solicitudes de autorización desde aplicaciones nativas solo deben realizarse a través de agentes de usuario externos, principalmente el navegador del usuario

<div id="rate-limits">
  ### Límites de frecuencia
</div>

Los límites solo se aplican a las solicitudes relacionadas con los flujos de inicio de sesión social nativo, que se identifican según el cuerpo de la solicitud con los siguientes criterios iniciales:

| Tipo de solicitud    | Cuerpo                                               |
| -------------------- | ---------------------------------------------------- |
| `grant_type`         | `urn:ietf:params:oauth:grant-type:token-exchange`    |
| `subject_token_type` | `http://auth0.com/oauth/token-type/apple-authz-code` |

<div id="limits-for-production-tenants-of-paying-customers">
  #### Límites para los tenants de producción de clientes de pago
</div>

| Endpoint      | Ruta           | Se limita por                                         | Límite de solicitudes                             |
| ------------- | -------------- | ----------------------------------------------------- | ------------------------------------------------- |
| Obtener token | `/oauth/token` | Cualquier solicitud de inicio de sesión social nativa | 50 por minuto, con picos de hasta 500 solicitudes |

<div id="limits-for-non-production-tenants-of-paying-customers-and-all-tenants-of-free-customers">
  #### Límites para tenants de no producción de clientes de pago y todos los tenants de clientes gratuitos

| ------------- | -------------- | -------------------------------------------------- | -------------- |
| Obtener token | `/oauth/token` | Solicitudes de inicio de sesión social nativo e IP | 30 por minuto  |
