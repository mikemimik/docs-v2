---
description: Aprende cómo el flujo compatible con OIDC afecta a los tokens que se usan para proteger las API, incluidos los scopes y los claims.
title: Tokens de acceso con OIDC
---

Debido a que las aplicaciones y las API (recursos) se definen como entidades separadas en Auth0 con el flujo compatible con OIDC, puedes obtener <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+tokens">tokens de acceso</Tooltip> para tus API. En consecuencia, todas las API deben protegerse con tokens de acceso en lugar de <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=ID+tokens">tokens de ID</Tooltip>. Para obtener más información, consulta [Access Tokens](/es/docs/secure/tokens/access-tokens) y [ID Tokens](/es/docs/secure/tokens/id-tokens).

<div id="claims">
  ## Claims
</div>

El flujo compatible con OIDC estandariza los claims que puedes añadir a los tokens de ID y de acceso.

<div id="standard-claims">
  ### Claims estándar
</div>

La especificación de OIDC define un conjunto estándar de [](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) claims sobre los usuarios, como perfil y correo electrónico, que pueden devolverse en tokens de ID o en la respuesta del endpoint `/userinfo`.

Para obtener más información sobre la especificación de claims estándar de OIDC, lee la [Sección 5.1 Standard Claims](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims) de la [especificación de OpenID](https://openid.net).

<div id="custom-claims">
  ### Claims personalizadas
</div>

Con el flujo compatible con OIDC, todavía se pueden agregar claims personalizadas a los tokens de ID o tokens de acceso, pero deben ajustarse a un formato con espacio de nombres para evitar posibles conflictos con las claims estándar de OIDC.

Para obtener información sobre cómo agregar una claim personalizada en el flujo compatible con OIDC, lee [Crear claims personalizadas con espacio de nombres](/es/docs/secure/tokens/json-web-tokens/create-custom-claims).

<div id="scopes">
  ## Scopes
</div>

En el flujo compatible con OIDC, puedes configurar tus aplicaciones en Auth0 para que usen scopes y soliciten lo siguiente:

* Que los claims estándar de OIDC, como `profile` y `email`, se incluyan en el token de ID (si el usuario da su consentimiento para proporcionar esta información a la aplicación).
* Que los permisos compatibles con la API a la que quieren acceder se incluyan en el token de acceso. Por ejemplo, puedes definir la <Tooltip tip="Audience: Identificador único de la audience de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/es/docs/glossary?term=audience">audience</Tooltip> de tu API personalizada y los scopes requeridos, lo que te permitirá segmentar el acceso a distintas operaciones dentro de tu API.

Para obtener más información, consulta [Scopes de OpenID Connect](/es/docs/get-started/apis/scopes/openid-connect-scopes).
