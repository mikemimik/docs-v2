---
description: Aprende cómo el flujo compatible con OIDC afecta el uso de los tokens de actualización.
title: Tokens de actualización con OIDC
---

Con el flujo compatible con OIDC, los <Tooltip tip="Refresh Token: Token que se usa para obtener un nuevo token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=refresh+tokens">tokens de actualización</Tooltip>:

* Ya no se devolverán al usar la [concesión implícita](/es/docs/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post) para la autenticación.
* Se pueden usar con [aplicaciones confidenciales](/es/docs/get-started/applications/confidential-and-public-applications).
* Se pueden usar con [rotación del Refresh Token](/es/docs/secure/tokens/refresh-tokens/refresh-token-rotation) en aplicaciones públicas al usar el flujo de código de autorización con PKCE.
* Deben usar el endpoint [`/oauth/token`](https://auth0.com/docs/api/authentication#refresh-token) para obtener nuevos tokens, ya que el endpoint [`/delegation`](https://auth0.com/docs/api/authentication#delegation) está obsoleto.

Además, hay diferencias en la estructura de los tokens de actualización. Para obtener más información, consulta [Tokens de actualización](/es/docs/secure/tokens/refresh-tokens).

<div id="legacy-delegation">
  ## Heredado (delegation)
</div>

```json lines
POST /delegation
Content-Type: 'application/json'
{
  "grant_type": "urn:ietf:params:oauth:grant-type:jwt-bearer",
  "client_id": "...",
  "refresh_token": "...",
  "scope": "openid profile"
}
```

<div id="oidc-conformant-token-endpoint">
  ## Compatible con OIDC (endpoint de token)
</div>

```http lines
POST /oauth/token
Content-Type: application/x-www-form-urlencoded
grant_type=refresh_token&refresh_token=123&client_id=123&client_secret=123&scope=openid+profile&audience=https%3A%2F%2Fapi.example.com
```

* Los parámetros `audience` y `client_secret` son opcionales.
* `client_secret` no es necesario al solicitar un `refresh_token` para una aplicación pública.

<Warning>
  Los tokens de actualización deben mantenerse confidenciales durante el tránsito y el almacenamiento, y solo deben compartirse entre el servidor de autorización y el cliente al que se emitieron.
</Warning>
