---
description: Aprende cómo el flujo compatible con OIDC afecta tu uso de la delegación.
title: Delegación con OIDC
---

<Warning>
  De forma predeterminada, la delegación está deshabilitada para los tenants que no tuvieran un add-on en uso al 8 de junio de 2017. Los tenants antiguos que actualmente usan un add-on que requiere delegación pueden seguir usando esta funcionalidad. Si la funcionalidad de delegación cambia o se retira del servicio en algún momento, se notificará con antelación a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar.
</Warning>

Tradicionalmente, la delegación se usa para:

* Intercambiar un <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+token">token de identidad</Tooltip> emitido para una aplicación por uno nuevo emitido para otra aplicación.
* Obtener un token de identidad nuevo usando un [token de actualización](/es/docs/secure/tokens/refresh-tokens).
* Intercambiar un token de identidad por un token de API de terceros (p. ej., Firebase, AWS).

Como el flujo compatible con OIDC exige que los token de ID ya no se usen para proteger las API y que los <Tooltip tip="token de actualización: Token usado para obtener un token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=refresh+tokens">tokens de actualización</Tooltip> se usen solo en el endpoint [`/oauth/token`](https://auth0.com/docs/api/authentication#refresh-token), el endpoint [`/delegation`](https://auth0.com/docs/api/authentication#delegation) está obsoleto.

Las aplicaciones compatibles con OIDC no pueden ser el origen ni el destino de las solicitudes de delegación.

<div id="third-party-apis">
  ## APIs de terceros
</div>

Dado que no existe ningún mecanismo compatible con OIDC para obtener tokens de API de terceros (p. ej., Firebase, AWS), la delegación aún puede usarse para obtener tokens de API de terceros.
