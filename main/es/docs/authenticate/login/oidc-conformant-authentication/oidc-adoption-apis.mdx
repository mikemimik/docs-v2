---
description: Aprende cómo el flujo compatible con OIDC afecta el uso de API externas y de los tokens utilizados para protegerlas.
title: API externas con OIDC
---

Con el flujo compatible con OIDC, debes definir las aplicaciones y las API (recursos) como entidades separadas en Auth0. Entre las ventajas se incluyen:

* Integración de API más sencilla: las API ya no están vinculadas a las aplicaciones que las consumen.
* Casos de uso de integración de [máquina a máquina (M2M)](/es/docs/get-started/authentication-and-authorization-flow/client-credentials-flow): las aplicaciones pueden autenticarse por sí mismas (en lugar de actuar en nombre de un usuario) para obtener de forma programática y segura un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip>.

<div id="architecture">
  ## Arquitectura
</div>

Entre las soluciones sugeridas compatibles con OIDC para distintos escenarios se incluyen las siguientes:

* **Varias aplicaciones que invocan una API con un solo <Tooltip tip="ID de cliente: Valor de identificación otorgado a tu recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=client+ID">ID de cliente</Tooltip>**: Representa cada aplicación con una única aplicación de Auth0, y cada una de ellas puede interactuar con la API de la que dependen las aplicaciones.
* **Intercambiar tokens obtenidos por una aplicación por tokens para otra aplicación mediante delegation**: Usa una solución con varias aplicaciones, en la que cada aplicación se autentique en el mismo <Tooltip tip="Servidor de recursos: Servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/es/docs/glossary?term=resource+server">servidor de recursos</Tooltip>.
* **Las aplicaciones no dependen de APIs externas; solo se usan para autenticar usuarios**: No necesitas definir una API siempre que los <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+tokens">tokens de ID</Tooltip>:

  * Solo los procese la aplicación.
  * No se envíen a ningún servicio externo.
