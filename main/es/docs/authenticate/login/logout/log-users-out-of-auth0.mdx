Auth0 implementa <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/docs/es/glossary?term=OpenID">OpenID</Tooltip> Connect [RP-Initiated Logout 1.0](https://openid.net/specs/openid-connect-rpinitiated-1_0.html) para el cierre de sesión de los usuarios finales. Este estándar forma parte de la colección de [especificaciones finales](https://openid.net/developers/specs/) de OpenID Connect.

<div id="how-it-works">
  ## Cómo funciona
</div>

RP-Initiated Logout es un escenario en el que una <Tooltip tip="Relying Party: Entidad (como un servicio o una aplicación) que depende de un proveedor de identidad de terceros para autenticar a un usuario." cta="Ver glosario" href="/docs/es/glossary?term=relying+party">parte autenticadora</Tooltip> (usuario) solicita al proveedor OpenID (Auth0) que cierre su sesión.

1. El usuario inicia una solicitud de cierre de sesión en la aplicación.
2. La aplicación redirige al usuario al endpoint [OIDC Logout](https://auth0.com/docs/api/authentication#oidc-logout) de la Auth0 Authentication API.
3. Auth0 redirige al usuario al destino correspondiente en función de los [parámetros del endpoint de cierre de sesión de OIDC](#oidc-logout-endpoint-parameters) proporcionados.

<div id="configure-rp-initiated-logout">
  ## Configurar RP-Initiated Logout
</div>

Para configurar RP-Initiated Logout, debes asegurarte de que tu aplicación pueda localizar el parámetro `end_session_endpoint` en el [documento de metadatos de descubrimiento](https://openid.net/specs/openid-connect-discovery-1_0.html#ProviderConfig) de tu tenant de Auth0 y de que llame al endpoint de cierre de sesión de OIDC con los parámetros necesarios.

<div id="enable-endpoint-discovery">
  ### Habilitar el descubrimiento del endpoint
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para los tenants de Auth0 creados a partir del 14 de noviembre de 2023, **RP-Initiated Logout End Session Endpoint Discovery** está habilitado de forma predeterminada.
</Callout>

Puedes habilitar **RP-Initiated Logout End Session Endpoint Discovery** en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/docs/es/glossary?term=Management+API">Management API</Tooltip> de Auth0.

<Tabs>
  <Tab title="Dashboard">
    Para habilitar **RP-Initiated Logout End Session Endpoint Discovery** en el Dashboard:

    1. Ve a [Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced).
    2. Busca la sección **Login and Logout**.
    3. Activa el interruptor **RP-Initiated Logout End Session Endpoint Discovery**.
  </Tab>

  <Tab title="Management API">
    Para habilitar **RP-Initiated Logout End Session Endpoint Discovery** con la Management API:

    1. [Obtén un token de acceso de la Management API](/docs/es/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) que incluya el scope `update:tenant_settings`.
    2. Llama al endpoint [Update tenant settings](https://auth0.com/docs/api/management/v2/tenants/patch-settings) de la Management API con el siguiente payload:

       ```json JSON lines
       {
         "oidc_logout": {
           "rp_logout_end_session_endpoint_discovery": true
         }
       }
       ```
  </Tab>
</Tabs>

<div id="call-the-oidc-logout-endpoint">
  ### Llamar al endpoint de cierre de sesión de OIDC
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La mayoría de las [bibliotecas SDK de Auth0](/docs/es/libraries) cumplen con OIDC y son compatibles con RP-Initiated Logout por diseño.
</Callout>

Cuando llame al endpoint de cierre de sesión de OIDC, Auth0 recomienda proporcionar el parámetro `id_token_hint`.

Si su aplicación no puede almacenar de forma segura los <Tooltip tip="Token de ID: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+tokens">tokens de ID</Tooltip>, puede proporcionar en su lugar los parámetros `logout_hint` y `client_id`.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Puede llamar al endpoint de cierre de sesión de OIDC con información redundante.

  Por ejemplo, puede enviar los parámetros `id_token_hint` y `logout_hint`, o `id_token_hint` y `client_id`.

  En todos los casos, Auth0 comprueba que los datos del usuario y de la sesión sean coherentes, y devuelve un error si detecta alguna discrepancia.
</Callout>

<div id="oidc-logout-endpoint-parameters">
  #### Parámetros del endpoint de cierre de sesión de OIDC
</div>

El endpoint [OIDC Logout](https://auth0.com/docs/api/authentication#oidc-logout) de Authentication API admite los siguientes parámetros:

| Parameter                  | Required?   | Description                                                                                                                                                                               |
| -------------------------- | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id_token_hint`            | Recomendado | Token de identidad emitido previamente para el usuario. Indica qué usuario debe cerrar sesión.                                                                                            |
| `logout_hint`              | Opcional    | Valor del ID de sesión (`sid`) que indica qué usuario debe cerrar sesión.                                                                                                                 |
| `post_logout_redirect_uri` | Opcional    | Valor de URL de redirección que indica adónde redirigir al usuario después del cierre de sesión.                                                                                          |
| `client_id`                | Opcional    | ID de cliente de su aplicación.                                                                                                                                                           |
| `federated`                | Opcional    | Indica a Auth0 que cierre la sesión del usuario en su proveedor de identidad.                                                                                                             |
| `state`                    | Opcional    | Valor opaco que la aplicación agrega a la solicitud inicial de cierre de sesión y que Auth0 incluye al redirigir de vuelta a `post_logout_redirect_uri`.                                  |
| `ui_locales`               | Opcional    | Lista de locales separada por espacios que se usa para restringir la lista de idiomas de la solicitud. El primer locale de la lista debe coincidir con el locale habilitado en su tenant. |

<div id="id_token_hint-parameter">
  #### parámetro id_token_hint
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recomienda usar el parámetro `id_token_hint` al llamar al endpoint de cierre de sesión de OIDC.
</Callout>

El valor del parámetro `id_token_hint` debe ser el token de identidad que Auth0 emitió al usuario después de autenticarse.

El token de identidad contiene, para su verificación, el emisor (`iss`) de las claims registradas, la <Tooltip tip="Audience: Identificador único de la audiencia de un token emitido. Llamado aud en un token, su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip> (`aud`) y el ID de sesión de Auth0 (`sid`). Para obtener más información sobre las claims del token de identidad, consulta [Estructura del token de ID](/docs/es/secure/tokens/id-tokens/id-token-structure).

  ##### Ejemplos
</div>

<AuthCodeGroup>
  ```http HTTP
  https://{yourDomain}/oidc/logout?id_token_hint={yourIdToken}&post_logout_redirect_uri={yourCallbackUrl}
  ```

  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/oidc/logout' \
    --header 'content-type: application/x-www-form-urlencoded' \
    --data 'id_token_hint={yourIdToken}' \
    --data 'post_logout_redirect_uri={yourCallbackUrl}'
  ```
</AuthCodeGroup>

<div id="logout_hint-parameter">
  #### parámetro logout_hint
</div>

El valor del parámetro `logout_hint` debe ser el ID de sesión (`sid`) de la sesión actual del usuario en Auth0.

El ID de sesión (`sid`) se proporciona como un claim registrado dentro del token de identidad que Auth0 emitió al usuario después de autenticarse.

<Warning>
  Debes usar el ID de sesión (`sid`) asociado al token de identidad emitido por Auth0 cuando comenzó la sesión actual. Auth0 ignora las solicitudes con valores aleatorios o que no reflejan los datos de la sesión actual.
</Warning>

<div id="post_logout_redirect_uri-parameter">
  #### parámetro post_logout_redirect_uri
</div>

El valor del parámetro `post_logout_redirect_uri` debe ser una URL válida y codificada que se haya registrado en la lista de **Allowed Logout URLs** de tu:

1. [Configuración de la aplicación](/docs/es/get-started/applications/application-settings#application-uris): si proporcionas el parámetro `id_token_hint` o los parámetros `logout_hint` y `client_id`.
2. [Configuración del tenant](/docs/es/get-started/tenant-settings#login-and-logout): si proporcionas solo el parámetro `logout_hint`.

<div id="allowed-logout-urls-guidelines">
  ##### Pautas para Allowed Logout URLs
</div>

Cuando actualices **Allowed Logout URLs**, sigue estas pautas para evitar errores de validación:

* Separa varios valores de URL con una coma (`,`).
* Incluye el esquema de la URL (por ejemplo, `https://`).

Puedes usar un asterisco (`*`) como comodín para los subdominios (por ejemplo, `https://*.example.com`), pero te recomendamos no usar comodines en entornos de producción. Para obtener más información, consulta [Marcadores de posición para URL de subdominios](/docs/es/get-started/applications/wildcards-for-subdomains#wildcard-url-placeholders).

<div id="add-query-string-parameters-to-post_logout_redirect_uri">
  ##### Agregar parámetros de cadena de consulta a post_logout_redirect_uri
</div>

El endpoint de cierre de sesión de OIDC analiza los parámetros de cadena de consulta de la URL proporcionada en el parámetro `post_logout_redirect_uri`.

Debes incluir la **URL exacta con los parámetros de cadena de consulta** en tus **Allowed Logout URLs**; de lo contrario, la solicitud de cierre de sesión será rechazada. La URL debe coincidir exactamente, incluidos todos los nombres y valores de los parámetros de consulta.

Por ejemplo, si pasas `https://example.com/logout?myParam=1234` al parámetro `post_logout_redirect_uri` (codificado como `https%3A%2F%2Fexample.com%2Flogout%3FmyParam%3D1234`), debes incluir la URL completa `https://example.com/logout?myParam=1234` en tus **Allowed Logout URLs**.

<Warning>
  No se admiten valores dinámicos para los parámetros de consulta. Cada combinación única de nombres y valores de parámetros de consulta debe registrarse como una entrada independiente en tus **Allowed Logout URLs**.
</Warning>

<div id="ui_locales-parameter">
  #### parámetro ui_locales
</div>

El valor del parámetro `ui_locales` debe ser una lista de [locales compatibles](/docs/es/customize/internationalization-and-localization/universal-login-internationalization#new-universal-login-experience-localization) separada por espacios.

El primer valor de la lista debe coincidir con la [configuración del idioma predeterminado de tu tenant](/docs/es/get-started/tenant-settings#languages).

<div id="federated-parameter">
  #### parámetro `federated`
</div>

El parámetro `federated` no requiere ningún valor.

Si incluyes el parámetro `federated` al llamar al endpoint de cierre de sesión de OIDC, Auth0 intenta [cerrar la sesión del usuario en su proveedor de identidad](/docs/es/authenticate/login/logout/log-users-out-of-idps).

<div id="logout-consent-prompt">
  ## Solicitud de consentimiento de cierre de sesión
</div>

El estándar OIDC establece que el flujo de cierre de sesión debe interrumpirse para solicitar el consentimiento del usuario si el proveedor de OpenID no puede verificar que la solicitud la haya realizado el usuario.

Auth0 aplica este comportamiento mostrando una solicitud de consentimiento de cierre de sesión si detecta alguna de las siguientes condiciones:

* No se proporciona ni el parámetro `id_token_hint` ni `logout_hint`.
* El claim `sid` del token de identidad no se corresponde con la sesión del navegador de la solicitud.
* El valor del parámetro `logout_hint` no coincide con los datos de la sesión actual.

<Frame>![](/docs/images/cdy7uua7fh8z/5Sycg1aMJ1CLZwJj19Omv4/655fa33b439217d0410fba1335c63a63/Logout_-_English.png)</Frame>

Si el usuario confirma la solicitud de cierre de sesión, Auth0 continúa con el flujo de cierre de sesión.

<div id="disable-the-logout-consent-prompt">
  ### Desactivar la solicitud de consentimiento de cierre de sesión
</div>

Puede desactivar la solicitud de consentimiento de cierre de sesión. Si lo hace, Auth0 no intentará detectar comportamientos anómalos y aceptará automáticamente las solicitudes de cierre de sesión.

Para desactivar la solicitud de consentimiento de cierre de sesión en el Dashboard:

1. Vaya a [Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced).
2. Desactive el interruptor **Show RP-Initiated Logout End-User Confirmation**.
