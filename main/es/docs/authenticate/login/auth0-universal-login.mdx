Auth0 <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> proporciona la función esencial de un <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authorization+server">servidor de autorización</Tooltip>: el flujo de inicio de sesión. Cuando un usuario necesita demostrar su identidad para acceder a tu aplicación, puedes redirigirlo a Universal Login y dejar que Auth0 se encargue del proceso de autenticación.

Con Universal Login, no tienes que realizar ningún trabajo de integración para admitir distintos métodos de autenticación. Puedes empezar con un flujo de autenticación sencillo y luego añadir funciones adicionales, como el inicio de sesión sin contraseña, el inicio de sesión social o la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación del usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/docs/es/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA), para crear una experiencia de inicio de sesión segura y fácil de usar.

La configuración de Universal Login es dinámica y no requiere cambios a nivel de la aplicación, ya que las páginas web alojadas en el <Tooltip tip="Servidor de autenticación: Servidor que confirma o rechaza la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=authentication+server">servidor de autenticación</Tooltip> centralizado de Auth0 controlan por completo el flujo de inicio de sesión. Además, no necesitas actualizar manualmente tu código para que las applications se beneficien de las mejoras que Auth0 incorpora a Universal Login.

Desde el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, puedes personalizar la apariencia y el comportamiento de las páginas de inicio de sesión para crear una experiencia coherente y alineada con tu marca. Para casos de uso más avanzados, también puedes modificar el código de cada página por separado.

Para obtener más información sobre las diferencias entre Universal Login y el inicio de sesión integrado, consulta [Centralized Universal Login vs. Embedded Login](/docs/es/authenticate/login/universal-vs-embedded-login).

<div id="configure-universal-login">
  ## Configurar Universal Login
</div>

En tu Auth0 Dashboard, puedes elegir qué experiencia usar para las páginas de inicio de sesión predeterminadas en [Branding &gt; Universal Login &gt; Advanced Options](https://manage.auth0.com/#/universal-login/customizations-new).

Auth0 ofrece dos experiencias de inicio de sesión alojadas:

* **Universal Login** ofrece una experiencia más ágil para los usuarios y no requiere JavaScript para su personalización.
* **Classic Login** utiliza controles de JavaScript para cada página del flujo de inicio de sesión.

En este momento, los esfuerzos de desarrollo activo de Auth0 se centran en Universal Login, y Classic Login ya no recibe actualizaciones. A menos que tu caso de uso específico requiera la experiencia Classic, se recomienda implementar Universal Login.

Para obtener más información sobre las diferencias entre estas dos experiencias, consulta [Universal Login vs. Classic Login](/docs/es/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<div id="accessibility">
  ### Accesibilidad
</div>

Auth0 emplea los siguientes estándares para mejorar la accesibilidad de los flujos de Universal Login para las personas que utilizan tecnologías de asistencia:

* [Pautas de Accesibilidad para el Contenido Web (WCAG), versión 2.2 AA](https://www.w3.org/WAI/standards-guidelines/wcag/)
* [EN 301 549](https://www.etsi.org/human-factors-accessibility/en-301-549-v3-the-harmonized-european-standard-for-ict-accessibility)

Auth0 Universal Login incorpora estos estándares de forma predeterminada. A continuación, se indican algunas áreas que podrían verse afectadas por las personalizaciones de los clientes:

* Los errores de validación se muestran en línea en lugar de como información sobre herramientas, para que sean accesibles a los lectores de pantalla.
* Los colores predeterminados de los componentes de la interfaz se ajustan a las pautas de las WCAG sobre contraste de color.
* Los formularios usan autocorrección.
* Los atributos [ARIA](https://www.w3.org/WAI/standards-guidelines/aria/) y otros atributos de los componentes de la interfaz permiten navegar fácilmente con lectores de pantalla.
* Las direcciones de correo electrónico se validan del lado del cliente y siguen las mismas reglas de formato que la validación del lado del servidor.
* Las etiquetas HTML son accesibles y están asociadas a sus respectivos campos de entrada.
* Los requisitos de complejidad de la contraseña y los mensajes de error en línea son anunciados por los lectores de pantalla.
* Todas las pantallas del <Tooltip tip="Flujo de autorización: concesión de autorización (o flujo de trabajo) especificado en el marco de OAuth 2.0." cta="Ver glosario" href="/docs/es/glossary?term=authorization+flow">flujo de autorización</Tooltip> tienen un título de página claro y distintivo.
* Los campos obligatorios incluyen un indicador visual en la etiqueta.

<div id="customize-login-page">
  ### Personalizar la página de inicio de sesión
</div>

Después de habilitar Universal Login, puedes personalizar tu página de inicio de sesión directamente desde el Auth0 Dashboard. En concreto, puedes personalizar las plantillas y los temas de la página, las pantallas de registro e inicio de sesión, y otros elementos de texto. Para obtener más información, consulta Personalizar Universal Login.

Si decides implementar Classic Login, puedes personalizar las opciones básicas de imagen de marca desde el Auth0 Dashboard. Después, puedes configurar personalizaciones avanzadas con el SDK que uses para crear tu flujo de inicio de sesión. Para obtener más información, consulta [Personalizar Classic Login](/docs/es/customize/login-pages/classic-login).

<div id="implement-universal-login">
  ## Implement Universal Login
</div>

Después de habilitar Universal Login o Classic Login en tu tenant, sigue estos pasos para implementar la experiencia:

1. [Registra tu aplicación](/docs/es/get-started/auth0-overview/create-applications) en Auth0.
2. Configura tu aplicación para llamar al [endpoint de inicio de sesión](https://auth0.com/docs/api/authentication#login) de la Authentication API a fin de activar el flujo de inicio de sesión y gestionar la respuesta. y gestionar la respuesta. Puedes configurarlo directamente o usar uno de los [SDKs](/docs/es/libraries) de Auth0.
3. Crea una conexión de [base de datos](/docs/es/get-started/applications/set-up-database-connections), [Enterprise](/docs/es/authenticate/enterprise-connections) o [Social](/docs/es/authenticate/identity-providers/social-identity-providers) y habilítala para tu aplicación.

<div id="navigate-to-the-login-page">
  ### Ve a la página de inicio de sesión
</div>

Puedes acceder directamente al [endpoint de inicio de sesión](https://auth0.com/docs/api/authentication#login) desde cualquier navegador:

export const codeExample = `https://{yourDomain}/authorize?
  response_type=code|token&
  client_id={yourClientId}&
  connection={connectionName}&
  redirect_uri=https://{yourApp}/callback&
  state={yourState}`;

<AuthCodeBlock children={codeExample} language="bash" filename="cURL" />

La solicitud debe incluir los siguientes valores:

* `response_type` (puede ser `code` o `token`)
* `client_id`
* `redirect_uri`
* `state`

  * Para comprender mejor por qué este valor es obligatorio, consulta [Prevent Attacks and Redirect Users With OAuth 2.0 State Parameters](/docs/es/secure/attack-protection/state-parameters).

Opcionalmente, puedes incluir el parámetro `connection` para indicar a los usuarios que se autentiquen con la conexión especificada.

<div id="correlation-id">
  ### ID de correlación
</div>

<ReleaseStageNotice feature="ID de correlación" stage="ea" terms="true" contact="Soporte de Auth0 o tu gerente técnico de cuenta" />

Universal Login admite eventos correlacionados en todos los flujos de autenticación. Una vez que implementes Universal Login, tu aplicación genera un ID único que pasas como `correlation_id` al endpoint `/authorize`. El ID único se conserva durante los siguientes eventos de autenticación:

* Registro
* Inicio de sesión
* Inscripción en la autenticación multifactor (MFA) y desafíos
* Restablecimiento de contraseña

Una vez que se pasa `correlation_id`, puedes hacer seguimiento de los eventos de autenticación en [logs de Auth0](/docs/es/deploy-monitor/logs).

Para obtener más información, consulta [ID de correlación](/docs/es/authenticate/login/configure-correlation-id).
