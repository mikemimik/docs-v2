---
description: Aprende a configurar Auth0 como proveedor de identidad mediante el add-on SAML2 Web App para Cisco WebEx.
title: Configurar Cisco WebEx como proveedor de servicios SAML
---

Usa la siguiente configuración de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip> para Cisco WebEx.

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y crea una nueva aplicación o haz clic en el nombre de una aplicación para actualizarla.

2. Ve a la pestaña **Addons** y activa **SAML2 Web App**.

3. En la pestaña **Settings**, configura la **URL de devolución de llamada de la aplicación.** Ten en cuenta que Webex tiene una opción para aprovisionar automáticamente nuevos usuarios. Deberás enviar esa información como claims (`lastname`, etc.). Si integras Auth0 con los servicios de Cisco Spark, consulta [inicio de sesión único con Cisco Spark Services](https://collaborationhelp.cisco.com/article/en-us/lfu88u).

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>

4. Pega el siguiente código en el cuadro de texto **Settings** y haz clic en **Debug**.

   ```json lines
   {
          "mappings": {
        "email": ["email", "uid", "Name_ID"]
          },
         "createUpnClaim":       false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities":        false,
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
         "nameIdentifierProbes": [
        "email",
          ]
       }
   ```

5. Desplázate hasta la parte inferior de la pestaña y haz clic en **Enable**.

6. En la pestaña **Usage**, busca **Identity Provider Metadata** y haz clic en **Download** para descargar el archivo de metadatos. Lo necesitarás cuando configures Auth0 como proveedor de identidad en Cisco WebEx.
