Para obtener los mejores resultados al implementar <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=SSO">SSO</Tooltip> de nativo a web, Auth0 recomienda seguir las prácticas recomendadas que se indican a continuación. Estas pautas pueden ayudar a garantizar la integridad de la sesión:

<div id="use-post-login-actions-to-limit-session-lifetime">
  ## Usa Actions de Post Login para limitar la duración de la sesión
</div>

Usa los triggers de Action [`post-login`](/docs/es/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) para garantizar que las sesiones web creadas mediante SSO de nativo a web tengan una duración limitada de forma adecuada y caduquen rápidamente cuando estén inactivas. 
Puedes usar Actions `post-login` para detectar cuándo se inicia una sesión mediante un `session_transfer_token` y aplicar tiempos de espera por inactividad y absolutos más cortos:

```swift lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.session_transfer_token) {
    const now = Date.now();

    // Limitar la sesión a 30 minutos en total
    api.session.setExpiresAt(now + 30 * 60 * 1000);

    // Establecer el tiempo de inactividad en 15 minutos
    api.session.setIdleExpiresAt(now + 15 * 60 * 1000);
  }
};
```

<div id="bind-session_transfer_token-to-the-device-or-ip-address">
  ## Vincula `session_transfer_token` al dispositivo o a la dirección IP
</div>

Para reducir el riesgo de reutilización del token si se filtra, queda registrado o es interceptado, vincula siempre el [`session_transfer_token`](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) al entorno de origen mediante `enforce_device_binding`.

<div id="use-secure-cookies-over-query-parameters">
  ## Usa cookies seguras en lugar de parámetros de consulta
</div>

Envía el `session_transfer_token` a la aplicación web mediante una cookie segura y HTTPOnly con alcance en tu dominio de Auth0 para evitar que el token se registre accidentalmente o se comparta a través de URL, y para reducir la superficie de ataque frente a la interceptación del token. Si necesitas usar un parámetro de consulta (por ejemplo, para Chrome Custom Tabs), asegúrate de que la URL use HTTPS y elimina el token de la URL después de usarlo.

<div id="avoid-issuing-refresh-tokens-to-web-apps-unless-necessary">
  ## Evita emitir tokens de actualización a aplicaciones web salvo que sea necesario
</div>

Solo habilita [`allow_refresh_token`](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para aplicaciones web que realmente necesiten tokens de larga duración. En la mayoría de los casos, los <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=access+tokens">tokens de acceso</Tooltip> de corta duración, combinados con autenticación silenciosa, son suficientes y más seguros en entornos de navegador.

<div id="enable-allow-refresh-tokens-when-appropriate-to-set-the-refresh-tokens-as-online">
  ## Habilita Allow Refresh Tokens cuando corresponda para configurar los tokens de actualización como «online»
</div>

Para evitar credenciales huérfanas e impedir que los <Tooltip tip="Token de actualización: token utilizado para obtener un nuevo Access Token sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/docs/es/glossary?term=refresh+tokens">tokens de actualización</Tooltip> sigan activos después del cierre de sesión, usa la configuración [`allow_refresh_token`](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) para garantizar que los tokens de actualización emitidos mediante SSO de nativo a web queden vinculados a la sesión que los emitió. Si la sesión se revoca o caduca, el token de actualización se invalida automáticamente.

<div id="enable-enforce-cascade-revocation">
  ## Habilitar `enforce_cascade_revocation`
</div>

Para garantizar que se revoquen todas las sesiones web y los tokens de actualización asociados a un `session_transfer_token`, habilita [`enforce_cascade_revocation`](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) en la aplicación nativa. Esto es fundamental para garantizar la invalidación segura de sesiones entre aplicaciones.
