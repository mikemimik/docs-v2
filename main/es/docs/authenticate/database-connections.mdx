Auth0 proporciona conexiones de base de datos para autenticar a los usuarios con un identificador (correo electrónico, nombre de usuario o número de teléfono) y contraseña, o con [claves de acceso](/es/docs/authenticate/database-connections/passkeys/configure-passkey-policy). Estas credenciales se almacenan de forma segura en el repositorio de usuarios de Auth0 o en tu propia base de datos.

Puedes crear una nueva conexión de base de datos y administrar las existentes en [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database).

<div id="database-options">
  ## Opciones para Database
</div>

<div id="use-auth0-user-store">
  ### Usa el repositorio de usuarios de Auth0
</div>

Auth0 proporciona, de manera predeterminada, la infraestructura de base de datos para almacenar tus usuarios. Este escenario ofrece el mejor rendimiento para el proceso de autenticación, ya que todos los datos se almacenan en Auth0.

La base de datos alojada en Auth0 es altamente segura. Las contraseñas nunca se almacenan ni aparecen en los logs en texto plano, sino que se procesan con hash mediante **bcrypt**. También se pueden aplicar distintos niveles de requisitos de seguridad para las contraseñas. Para obtener más información, consulta [Fortaleza de la contraseña en Auth0 Database Connections](/es/docs/authenticate/database-connections/password-strength).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para las conexiones de base de datos, Auth0 limita la cantidad de intentos repetidos de inicio de sesión por usuario y dirección IP. Para obtener más información, consulta [Límites de inicio de sesión de la base de datos](/es/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy#database-login-limits).
</Callout>

Para usar la infraestructura de base de datos de Auth0 como tu repositorio de usuarios, tienes varias opciones para migrar tus usuarios a Auth0. La funcionalidad de [migración automática](/es/docs/manage-users/user-migration/configure-automatic-migration-from-your-database) agrega tus usuarios a la base de datos de Auth0 de uno en uno, a medida que inician sesión, y evita tener que pedirles que restablezcan sus contraseñas todos al mismo tiempo. O bien, usa Management API para crear una tarea para [importar usuarios](/es/docs/manage-users/user-migration/bulk-user-imports). Para obtener más información y ver ejemplos de migración, consulta [Escenarios de migración de usuarios](/es/docs/manage-users/user-migration/user-migration-scenarios).

<div id="use-your-own-database">
  ### Usa tu propia base de datos
</div>

Si ya tienes un repositorio de usuarios o quieres almacenar las credenciales de usuario en tu propio servidor, Auth0 te permite conectarte a una [base de datos personalizada](/es/docs/authenticate/database-connections/custom-db) o repositorio y usarlo como proveedor de identidad.

<Frame>![](/docs/images/cdy7uua7fh8z/3SNKwx0KzkUwbodqUQbtvW/df720f8871b56dd7ec399a24aa621c36/Authentication>Database>use-my-own-database.png)</Frame>

En este escenario, proporcionas el script de Login para autenticar al usuario, que se ejecutará cada vez que un usuario intente iniciar sesión. Opcionalmente, puedes [crear scripts](/es/docs/authenticate/database-connections/custom-db/templates) para el registro, la verificación del correo electrónico, el restablecimiento de contraseña y la eliminación de usuarios.

Los scripts son código Node.js. Auth0 proporciona [plantillas](/es/docs/authenticate/database-connections/custom-db/templates) para la mayoría de las bases de datos más comunes, como ASP.NET Membership Provider, MongoDB, MySQL, PostgreSQL, SQL Server y Windows Azure SQL Database, así como para servicios web a los que se accede mediante Basic Auth. En esencia, puedes conectarte a casi cualquier tipo de base de datos o servicio web con un script personalizado.
