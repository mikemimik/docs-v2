<Warning>
  La autenticación con Username/Email y Password desde aplicaciones nativas está deshabilitada de forma predeterminada para los tenants nuevos desde el 8 de junio de 2017. Se recomienda usar Universal Login y realizar la autenticación web en su lugar. Si aun así quieres continuar, primero tendrás que habilitar el Password Grant Type en tu Dashboard. Consulta [Tipos de concesión de aplicación](/es/docs/get-started/applications/application-grant-types) para obtener más información.
</Warning>

<div id="log-in-with-a-database-connection">
  ## Inicia sesión con una conexión de base de datos
</div>

Para iniciar sesión con una conexión de base de datos, llama a `login` con el **correo electrónico**, la **contraseña** y la **conexión** que quieres usar para autenticarte. La respuesta es un objeto `Credentials`.

```kotlin lines
authentication
    .login("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(payload: Credentials) {
            // ¡Sesión iniciada!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    })
```

El scope predeterminado es `openid profile email`.

<div id="sign-up-with-a-database-connection">
  ## Registrarse con una conexión de base de datos
</div>

Para registrarse con una conexión de base de datos, llama al método `signUp` y pásale el correo electrónico y la contraseña del usuario, además del nombre de la conexión.

```kotlin lines
authentication
    .signUp("username@domain.com", "a secret password", "my-database-connection")
    .start(object: Callback<Credentials, AuthenticationException> {
        override fun onSuccess(result: Credentials) {
            // ¡Registrado y con sesión iniciada!
        }

        override fun onFailure(error: AuthenticationException) {
            // ¡Error!
        }
    });
```
