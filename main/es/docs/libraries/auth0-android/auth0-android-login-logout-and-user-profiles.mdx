<div id="add-login-to-your-android-application">
  ## Añade el inicio de sesión a tu aplicación Android
</div>

Puedes hacer que el usuario inicie sesión usando el método `WebAuthProvider.login`:

```kotlin lines
WebAuthProvider.login(account)
  .withScheme(getString(R.string.com_auth0_scheme))
  .withScope("openid profile email")
  .start(this, object : Callback<Credentials, AuthenticationException> {
    override fun onFailure(exception: AuthenticationException) {
       // Autenticación fallida
     }

      override fun onSuccess(credentials: Credentials) {
         // Autenticación exitosa
       }
  })
```

El resultado de la autenticación se devolverá en la función de callback `onSuccess`.

Consulta la [configuración de Auth0.Android](/es/docs/libraries/auth0-android/auth0-android-configuration) para ver más opciones de la clase `WebAuthProvider`.

<div id="add-logout-to-your-android-application">
  ## Añade el cierre de sesión a tu aplicación Android
</div>

Para cerrar la sesión del usuario, llama al método `WebAuthProvider.logout`. El resultado del cierre de sesión se proporcionará en el callback `onSuccess`.

Este método elimina la cookie que el navegador estableció durante la autenticación, por lo que obliga a los usuarios a volver a introducir sus credenciales la próxima vez que intenten autenticarse.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si se cancela el cierre de sesión, quizá quieras llevar al usuario de vuelta al lugar donde estaba antes de intentar cerrar la sesión.
</Callout>

```kotlin lines
WebAuthProvider.logout(account)
  .withScheme("demo")
  .start(this, object: Callback<Void?, AuthenticationException> {
    override fun onSuccess(payload: Void?) {
      // ¡El usuario ha cerrado sesión!
    }

    override fun onFailure(error: AuthenticationException) {
      // ¡Algo salió mal!
    }
  })
```

<div id="show-the-users-profile">
  ## Mostrar el perfil del usuario
</div>

Usa la clase `AuthenticationAPIClient` para obtener el perfil del usuario desde Auth0. Para ello, necesitas:

* El <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, usada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> recibido durante la fase de inicio de sesión
* El scope `profile`, que debe incluirse al llamar a `WebAuthProvider.login`
* El scope `email` (si quieres obtener la dirección de correo electrónico del usuario)

Este ejemplo muestra una función que obtiene el perfil del usuario y lo muestra en pantalla:

```kotlin lines
var client = AuthenticationAPIClient(account)

// Usa el token de acceso recibido para llamar a `userInfo` y obtener el perfil desde Auth0.
client.userInfo(accessToken)
  .start(object : Callback<UserProfile, AuthenticationException> {
      override fun onFailure(exception: AuthenticationException) {
          // ¡Algo salió mal!
      }

      override fun onSuccess(profile: UserProfile) {
        // ¡Tenemos el perfil del usuario!
        val email = profile.email
        val name = profile.name
      }
})
```
