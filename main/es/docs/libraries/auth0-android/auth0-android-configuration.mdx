Auth0.Android se puede configurar con diversas opciones, que se enumeran a continuación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las opciones que aparecen a continuación han cambiado con respecto a la versión anterior, V1. Consulta la guía de migración para obtener más información.
</Callout>

<div id="withconnection">
  ## withConnection
</div>

La opción `withConnection` te permite especificar la conexión con la que quieres autenticarte.

```kotlin lines
WebAuthProvider.login(account)
                .withConnection("twitter")
                .start(this, callback)
```

<div id="withscope">
  ## withScope
</div>

Usar scopes te permite devolver claims específicos para determinados campos de tu request. Agregar parámetros a `withScope` te permite añadir más scopes. Consulta [Scopes](/docs/es/get-started/apis/scopes) para más detalles.

```kotlin lines
WebAuthProvider.login(account)
                .withScope("openid profile email")
                .start(this, callback)
```

El scope predeterminado es `openid profile email`.

<div id="withconnectionscope">
  ## withConnectionScope
</div>

En algunos casos, puede que necesites autenticarte con scopes de conexión específicos, o permisos, del proveedor de autenticación en cuestión. Consulta [Adding Scopes for an External IDP](/docs/es/authenticate/identity-providers/adding-scopes-for-an-external-idp). Sin embargo, si necesitas un acceso específico para una situación concreta en tu aplicación, puedes pasar parámetros a `withConnectionScope`. Puedes encontrar la lista completa de parámetros disponibles en la configuración de esa conexión en el [Dashboard](https://manage.auth0.com/#) o en la documentación del proveedor de autenticación. El scope solicitado aquí se añade a los ya especificados en la configuración de la conexión en el Dashboard.

```kotlin lines
WebAuthProvider.login(account)
                .withConnectionScope("email", "profile", "calendar:read")
                .start(this, callback)
```

<div id="withparameters">
  ## withParameters
</div>

Para enviar parámetros adicionales durante la autenticación, usa `withParameters.`

```kotlin lines
val parameters = mapOf("param1" to "value1")

WebAuthProvider.login(account)
                .withParameters(parameters)
                .start(this, callback)
```

<div id="withheaders">
  ## withHeaders
</div>

Para enviar headers personalizados al endpoint de autorización, usa `withHeaders.`

```kotlin lines
val headers = mapOf("header1" to "value1")

WebAuthProvider.login(account)
                .withHeaders(headers)
                .start(this, callback)
```

<div id="withscheme">
  ## withScheme
</div>

Si no usas Android &quot;App Links&quot; o quieres usar un esquema distinto para la URI de redirección, usa `withScheme`. Actualiza el marcador de posición `auth0Scheme` del manifiesto en el archivo `app/build.gradle` y la AllowList **Allowed Callback URLs** en la configuración de la aplicación del [Dashboard](https://manage.auth0.com/#) para que coincidan con el esquema elegido.

```kotlin lines
WebAuthProvider.login(account)
                .withScheme("myapp")
                .start(this, callback)
```

<Warning>
  El esquema debe estar en minúsculas y no debe contener guiones bajos.
</Warning>

<div id="withaudience">
  ## withAudience
</div>

Para especificar una <Tooltip tip="Audiencia: identificador único de la audiencia de un token emitido. Se llama aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un token de ID o de una API (identificador de API) para un token de acceso." cta="Ver glosario" href="/docs/es/glossary?term=audience">audiencia</Tooltip>, usa `withAudience`.

```kotlin lines
WebAuthProvider.login(account)
                .withAudience("https://YOUR_DOMAIN/userinfo")
                .start(this, callback)
```
