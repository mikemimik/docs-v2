<ParamField body="useErrors(options?: UseErrorOptions)" type={<span><a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult">UseErrorsResult</a></span>}>
  Hook de React para leer y gestionar errores en ACUL (Personalización avanzada de Universal Login).
  Agrupa los errores en tres tipos:

  * `auth0` — errores devueltos por Auth0 o por tu propio backend.
  * `validation` — errores de validación del lado del cliente (p. ej., entradas de formulario no válidas).
  * `configuration` — errores causados por una integración incorrecta o por un uso indebido del SDK.

  ### Funcionalidades clave

  * **Almacén de errores unificado** — muestra errores de auth0, validación y configuración desde un solo hook.
  * **Filtrado flexible** — consulta errores por tipo, campo o una combinación de ambos.
  * **Controles para descartar** — elimina errores individuales o borra todos de una vez.

  ## Parámetros

  <ParamField body="options" type={<span><a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorOptions">UseErrorOptions</a></span>}>
    Configuración opcional del hook.

    <Expandable title="propiedades">
      <ParamField body="includeDevErrors" type="boolean">
        Cuando es `true`, incluye en la lista detalles de errores específicos del desarrollo. El valor predeterminado es `false`.
      </ParamField>
    </Expandable>
  </ParamField>

  ## Devuelve

  [`UseErrorsResult`](/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult)

  Un objeto [`UseErrorsResult`](/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult) que contiene:

  * `errors` — la lista completa de errores de tipo [`ErrorsResult`](/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ErrorsResult), con utilidades:
    * `errors.byType(type, filter?)` — filtra por tipo de error y, opcionalmente, por campo.
    * `errors.byField(field, filter?)` — filtra por campo y, opcionalmente, por tipo.
  * `hasError` — `true` si actualmente hay algún error.
  * `dismiss(id)` — elimina un error específico por su ID.
  * `dismissAll()` — borra todos los errores rastreados.

  ## Pantallas compatibles

  * El hook `useErrors` está disponible en todas las pantallas de ACUL.

  ```tsx Example
  import { useErrors } from "@auth0/auth0-acul-react";

  export function SignupForm() {
    const { errors, hasError, dismiss, dismissAll } = useErrors();

    return (
      <div>
        {hasError && (
          <div className="mb-4">
            {errors.byType("auth0").map(err => (
              <div key={err.id} className="text-red-600">
                {err.message}
                <button onClick={() => dismiss(err.id)}>Dismiss</button>
              </div>
            ))}
          </div>
        )}

        <button onClick={dismissAll}>Clear All Errors</button>
      </div>
    );
  }
  ```

  Además de renderizar mensajes, puedes filtrar por campo o tipo:

  ```ts
  console.log(errors.byType('validation')); // todos los errores de validación
  console.log(errors.byType('validation', { field: 'username' })); // errores de validación para el campo 'username'
  console.log(errors.byField('username')); // todos los errores para el campo 'username'
  console.log(errors.byField('username', { type: 'auth0' })); // errores de auth0 para el campo 'username'
  ```

  ## Observaciones

  * `useErrors` no está vinculado a una pantalla específica — impórtalo directamente desde `@auth0/auth0-acul-react`.
  * Llama a `useErrors` en el nivel superior de tu componente; no lo llames de forma condicional ni dentro de controladores de eventos.
  * Usa `includeDevErrors: true` durante el desarrollo para mostrar errores por uso indebido del SDK; desactívalo en producción.
