La pantalla `mfa-phone-challenge` solicita al usuario que verifique su identidad introduciendo un código enviado a su teléfono mediante SMS o una llamada de voz.

<div id="import">
  ## Importación
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite la **importación parcial** y la **importación desde la raíz** para cada pantalla.

* La importación parcial te permite incluir solo el código que necesitas para tu caso de uso.
* La importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo que resulta útil si quieres una compilación unificada que gestione todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useMfaPhoneChallenge,
  // Hooks de contexto
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks de utilidad
  useChangeLanguage,
  // Métodos
  continueMethod,
  pickPhone,
  tryAnotherMethod,
} from "@auth0/auth0-acul-react/mfa-phone-challenge";

function MfaPhoneChallengeScreen() {
  const { continueMethod } = useMfaPhoneChallenge();
  return (
    <button onClick={() => continueMethod({ type: 'sms' })}>
      Send Code via SMS
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexto
</div>

Hooks con alcance de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `mfa-phone-challenge`. Impórtalos desde `@auth0/auth0-acul-react/mfa-phone-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona configuraciones de branding, como el logotipo, los colores y la configuración del tema que se muestran en la pantalla `mfa-phone-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `mfa-phone-challenge`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la organización del usuario si el flujo de MFA tiene alcance de organización. Devuelve `null` cuando no hay contexto de organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual en el flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnMfaPhoneChallenge">ScreenMembersOnMfaPhoneChallenge</a></span>}>
  Este hook contiene detalles específicos de la pantalla `mfa-phone-challenge`, incluida su configuración y contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y los metadatos asociados.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Este hook proporciona datos específicos de la transacción para la pantalla `mfa-phone-challenge`, como el estado actual del flujo de MFA.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Este hook gestiona los datos no confiables que se pasan a la pantalla, como los valores rellenados previamente a partir de parámetros de URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook proporciona información del usuario activo, incluidos `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/mfa-phone-challenge';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useMfaPhoneChallenge" type={<a href="/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPhoneChallengeMembers">MfaPhoneChallengeMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `mfa-phone-challenge`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="continueMethod" type="Promise<void>">
  Este método inicia el desafío por teléfono enviando un código de verificación al teléfono del usuario mediante el método de entrega especificado.

  ```jsx Example
  import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react/mfa-phone-challenge';

  function PhoneChallengeForm() {
    const { continueMethod } = useMfaPhoneChallenge();
    return (
      <button onClick={() => continueMethod({ type: 'sms' })}>
        Enviar código por SMS
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [MfaPhoneChallengeContinueOptions](/es/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPhoneChallengeContinueOptions).
    </ParamField>

    <ParamField body="type" type="&#x22;sms&#x22; | &#x22;voice&#x22;" required>
      El tipo de desafío que se debe realizar.

      * `sms`: Envía el código por mensaje de texto.
      * `voice`: Envía el código mediante una llamada de voz.

      #### Predeterminado

      ```ts
      'sms'
      ```
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="pickPhone" type="Promise<void>">
  Este método lleva a la pantalla de selección de número de teléfono para que el usuario pueda elegir qué número usar para el desafío.

  ```jsx Example
  import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react/mfa-phone-challenge';

  function PickPhoneButton() {
    const { pickPhone } = useMfaPhoneChallenge();
    return (
      <button onClick={() => pickPhone()}>
        Elegir número de teléfono
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Este método lleva a la pantalla de selección del método de MFA para que el usuario pueda elegir un factor de autenticación diferente.

  ```jsx Example
  import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react/mfa-phone-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useMfaPhoneChallenge();
    return (
      <button onClick={() => tryAnotherMethod()}>
        Probar otro método
      </button>
    );
  }
  ```
</ParamField>
