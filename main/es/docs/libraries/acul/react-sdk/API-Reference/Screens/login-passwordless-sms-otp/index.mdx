La pantalla `login-passwordless-sms-otp` solicita al usuario que introduzca un código de un solo uso enviado a su número de teléfono para completar la autenticación passwordless por SMS.

<div id="import">
  ## Importación
</div>

Cada pantalla tiene su propio conjunto de hooks y métodos. El SDK admite la **importación parcial** y la **importación desde la raíz** para cada pantalla.

* La importación parcial te permite incluir solo el código que necesitas para tu caso de uso específico.
* La importación desde la raíz te permite cargar todas las pantallas desde un único paquete, lo que resulta útil cuando quieres una versión unificada que gestione todas las pantallas posibles.

```jsx Import Example
// importación desde la raíz
import { useLoginPasswordlessSmsOtp } from '@auth0/auth0-acul-react';

// importación parcial
import {
  useLoginPasswordlessSmsOtp,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks comunes
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
  useResend,
  // Métodos
  submitOTP,
  resendOTP,
  switchConnection,
} from '@auth0/auth0-acul-react/login-passwordless-sms-otp';

function SmsOtpForm() {
  const { submitOTP } = useLoginPasswordlessSmsOtp();
  return (
    <button onClick={() => submitOTP({ code: '123456' })}>
      Verify Code
    </button>
  );
}
```

<div id="context-hooks">
  ## Context Hooks
</div>

Hooks con alcance de pantalla que proporcionan acceso de solo lectura a los datos de contexto de Auth0 en la pantalla `login-passwordless-sms-otp`. Impórtalos desde `@auth0/auth0-acul-react/login-passwordless-sms-otp`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Este hook proporciona configuraciones de branding, como el logotipo, los colores y los ajustes del tema que se muestran en la pantalla `login-passwordless-sms-otp`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Este hook proporciona configuraciones relacionadas con el cliente, como `id`, `name` y `logoUrl`, para la pantalla `login-passwordless-sms-otp`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Este hook proporciona información sobre la Organización del usuario si el login está limitado a una Organización. Devuelve `null` cuando no hay contexto de Organización.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Este hook contiene datos sobre el prompt actual en el flujo de autenticación.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembersOnLoginPasswordlessSmsOtp">ScreenMembersOnLoginPasswordlessSmsOtp</a></span>}>
  Este hook contiene detalles específicos de la pantalla `login-passwordless-sms-otp`, incluida su configuración y su contexto.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Este hook contiene datos relacionados con el tenant, como `id` y los metadatos asociados.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembersOnLoginPasswordlessSmsOtp">TransactionMembersOnLoginPasswordlessSmsOtp</a></span>}>
  Este hook proporciona datos específicos de la transaction para la pantalla `login-passwordless-sms-otp`, como las conexiones activas y el estado actual del flujo.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Este hook gestiona datos no confiables que se pasan a la pantalla, como un número de teléfono precompletado a partir de parámetros de URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Este hook proporciona información del usuario activo, incluidos `username`, `email` y los métodos de autenticación disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useLoginPasswordlessSmsOtp" type={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/LoginPasswordlessSmsOtpMembers">LoginPasswordlessSmsOtpMembers</a>}>
  Este hook devuelve todos los métodos y el contexto disponibles en la pantalla `login-passwordless-sms-otp`.
</ParamField>

<div id="methods">
  ## Métodos
</div>

<ParamField body="resendOTP" type="Promise<void>">
  Este método reenvía el código de un solo uso (OTP) al número de teléfono del usuario.

  ```jsx Example
  import { useLoginPasswordlessSmsOtp } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';

  function ResendButton() {
    const { resendOTP } = useLoginPasswordlessSmsOtp();
    return (
      <button onClick={() => resendOTP()}>
        Resend Code
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [CustomOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/CustomOptions).
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="submitOTP" type="Promise<void>">
  Este método envía el código de un solo uso para completar la autenticación passwordless por SMS.

  ```jsx Example
  import { useLoginPasswordlessSmsOtp } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';

  function SmsOtpForm() {
    const { submitOTP } = useLoginPasswordlessSmsOtp();
    return (
      <button onClick={() => submitOTP({ code: '123456' })}>
        Verify Code
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [SubmitOTPOptions](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/SubmitOTPOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      El código de un solo uso enviado al número de teléfono del usuario.
    </ParamField>

    <ParamField body="username?" type="string">
      El número de teléfono o identificador del usuario, si el flujo lo requiere.
    </ParamField>

    <ParamField body="captcha?" type="string">
      Valor de captcha requerido cuando la detección de bots está habilitada en el tenant.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="switchConnection" type="Promise<void>">
  Este método cambia a una conexión de autenticación alternativa sin reiniciar el flujo.

  ```jsx Example
  import { useLoginPasswordlessSmsOtp } from '@auth0/auth0-acul-react/login-passwordless-sms-otp';

  function ConnectionSwitcher() {
    const { switchConnection } = useLoginPasswordlessSmsOtp();
    return (
      <button onClick={() => switchConnection({ connection: 'enterprise-saml' })}>
        Use SSO
      </button>
    );
  }
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [SwitchConnectionOptionsSmsOtp](/docs/es/libraries/acul/react-sdk/API-Reference/Types/interfaces/SwitchConnectionOptionsSmsOtp).
    </ParamField>

    <ParamField body="connection" type="string" required>
      El nombre de la conexión a la que se cambiará. Usa &#39;email&#39; o &#39;sms&#39; para passwordless, o el nombre de una conexión de base de datos para la autenticación basada en contraseña.
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      Campos personalizados adicionales con el prefijo `ulp-` (por ejemplo, `'ulp-custom-field'`).
    </ParamField>
  </Expandable>
</ParamField>

<div id="commonutility-hooks">
  ## Hooks comunes y de utilidad
</div>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes">useAuth0Themes</a>} type="Hooks">
  Este hook obtiene las opciones del tema actual con la configuración del contexto de branding en formato plano.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage">useChangeLanguage</a>} type="Hooks">
  Este hook devuelve una función para cambiar el idioma que se muestra en la pantalla actual de ACUL.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen">useCurrentScreen</a>} type="Hooks">
  Este hook obtiene el contexto y el estado de la pantalla actual.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useErrors">useErrors</a>} type="Hooks">
  Este hook lee y gestiona los errores del servidor, del cliente y del desarrollador en la pantalla.
</ParamField>

<ParamField body={<a href="/docs/es/libraries/acul/react-sdk/API-Reference/Hooks/useResend">useResend</a>} type="Hooks">
  Este hook gestiona el estado de espera para el reenvío y la disponibilidad de las acciones de reenvío del código.
</ParamField>
