<ParamField body="MfaPollingOptions" type="object">
  Opciones de configuración para gestionar un proceso de polling de notificaciones push de MFA.

  Proporcione estas opciones al iniciar una sesión de polling para controlar con qué frecuencia
  se consulta el endpoint del desafío y cómo se gestionan los casos de éxito o error.

  ```ts Example
  export type MfaPollingOptions = {
    /**
     * Intervalo opcional, en **milisegundos**, entre solicitudes de polling consecutivas.
     *
     * - Si se omite, se usa el intervalo interno predeterminado del SDK (normalmente, 5000 ms).
     * - Elija un valor que equilibre la capacidad de respuesta con las consideraciones sobre límites de tasa.
     *
     * @defaultValue Específico de la implementación (normalmente, 5000 ms)
     */
    intervalMs?: number;

    /**
     * Devolución de llamada opcional que se ejecuta una vez que el desafío push de MFA
     * se aprueba correctamente y el polling finaliza.
     *
     * - Se ejecuta exactamente **una vez**, tras lo cual el polling se detiene automáticamente.
     * - Útil para avanzar en el flujo de inicio de sesión (p. ej., llamando a `continue()`).
     */
    onCompleted?: () => void;

    /**
     * Devolución de llamada opcional que se invoca si se produce un error durante el polling.
     *
     * Recibe un objeto {@link ULError} que contiene:
     * - `status` — el código de estado HTTP de la solicitud fallida
     * - `responseText` — el cuerpo de la respuesta sin procesar, si está disponible
     *
     * - Se invoca una vez por cada evento de error; el polling puede reintentarse según el tipo de error.
     * - Úselo para mostrar mensajes de error o activar lógica personalizada de reintento o cancelación.
     */
    onError?: (error: ULError) => void;
  };
  ```
