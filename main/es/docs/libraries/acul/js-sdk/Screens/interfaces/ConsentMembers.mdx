ConsentMembers

````ts Example
export interface ConsentMembers extends BaseMembers {
  /**
   * Proporciona acceso a las propiedades y datos específicos de la pantalla de consentimiento,
   * incluida la lista de `scopes` que se solicitan y el indicador `hideScopes`.
   * @type {ScreenMembersOnConsent}
   */
  screen: ScreenMembersOnConsent;

  /**
   * Envía la decisión del usuario de aceptar (conceder) los permisos solicitados.
   * Esta action realiza una solicitud POST al endpoint `/u/consent` con `action: "accept"`.
   *
   * @param {CustomOptions} [payload] - Opcional. Un objeto para cualquier par clave-valor personalizado
   *                                    que se enviará con la solicitud. Estos parámetros se incluirán
   *                                    en los datos del formulario enviados al servidor.
   * @returns {Promise<void>} Una Promise que se resuelve cuando la acción de aceptar se envía correctamente.
   *                          Si la operación es exitosa, Auth0 normalmente redirige al usuario de vuelta a la aplicación
   *                          o al siguiente paso en el flujo de autenticación.
   * @throws {Error} Lanza un error si el envío del formulario falla (p. ej., problema de red, estado no válido).
   *                 Los errores del lado del servidor (como "invalid_request") se reflejarán en `this.transaction.errors`
   *                 tras la operación, en lugar de lanzarse como errores de JavaScript.
   *
   * @example
   * ```typescript
   * // Se asume que 'consentManager' es una instancia de la clase SDK de la pantalla de consentimiento
   * try {
   *   await consentManager.accept();
   *   // Si la operación es exitosa, la página normalmente redirigirá.
   * } catch (error) {
   *   // Manejar errores inesperados durante el envío.
   *   console.error("Failed to submit consent acceptance:", error);
   * }
   * // Tras el await, verificar consentManager.transaction.errors para problemas de validación del lado del servidor.
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * Envía la decisión del usuario de denegar (rechazar) los permisos solicitados.
   * Esta action realiza una solicitud POST al endpoint `/u/consent` con `action: "deny"`.
   *
   * @param {CustomOptions} [payload] - Opcional. Un objeto para cualquier par clave-valor personalizado
   *                                    que se enviará con la solicitud. Estos parámetros se incluirán
   *                                    en los datos del formulario enviados al servidor.
   * @returns {Promise<void>} Una Promise que se resuelve cuando la acción de denegar se envía correctamente.
   *                          Si la operación es exitosa, Auth0 normalmente redirige al usuario, mostrando
   *                          potencialmente un mensaje de acceso denegado o devolviendo un error a la aplicación.
   * @throws {Error} Lanza un error si el envío del formulario falla (p. ej., problema de red, estado no válido).
   *                 Los errores del lado del servidor se reflejan en `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // Se asume que 'consentManager' es una instancia de la clase SDK de la pantalla de consentimiento
   * try {
   *   await consentManager.deny({ reason: "user_declined" }); // Ejemplo de opción personalizada
   *   // Si la operación es exitosa, la página normalmente redirigirá.
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // Tras el await, verificar consentManager.transaction.errors para problemas de validación del lado del servidor.
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}
````

<ParamField body="accept" type="Promise<void>">
  Envía la decisión del usuario de aceptar (conceder) los permisos solicitados.
  Esta acción realiza un envío al endpoint `/u/consent` con `action: "accept"`.

  Una promesa que se resuelve cuando la acción de aceptación se envía correctamente.
  Si todo sale bien, Auth0 normalmente redirige al usuario de vuelta a la aplicación
  o al siguiente paso del flujo de autenticación.

  #### Lanza

  Lanza un error si falla el envío del formulario (por ejemplo, por un problema de red o un `state` no válido).
  Los errores del lado del servidor (como &quot;invalid&#95;request&quot;) se reflejarán en `this.transaction.errors`
  después de la operación, en lugar de lanzarse como errores de JavaScript.

  ```typescript Example
  // Suponiendo que 'consentManager' es una instancia de la clase SDK de la pantalla de consentimiento
  try {
    await consentManager.accept();
    // Si se realiza correctamente, la página normalmente redirigirá.
  } catch (error) {
    // Gestiona errores inesperados durante el propio envío.
    console.error("Failed to submit consent acceptance:", error);
  }
  // Después del await, revisa consentManager.transaction.errors para detectar problemas de validación del lado del servidor.
  ```

  Los errores del lado del servidor se reflejan en `this.transaction.errors`.

    console.error("Failed to submit consent denial:", error);
