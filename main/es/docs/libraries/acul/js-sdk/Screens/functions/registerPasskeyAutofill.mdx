---
title: "registerPasskeyAutofill"
---

> **registerPasskeyAutofill**(`inputId`?: `string`): `Promise`&lt;`void`&gt;

Inicializa una solicitud pasiva de credencial WebAuthn mediante `navigator.credentials.get()` con `mediation: "conditional"`. Cuando es compatible, esto permite al navegador mostrar claves de acceso almacenadas directamente en el menú desplegable de autocompletar del campo de nombre de usuario. Al seleccionar una clave de acceso, la autenticación se completa sin necesidad de interacción adicional por parte del usuario.

Llama a esto una sola vez cuando se inicialice la pantalla de inicio de sesión (por ejemplo, al cargar la página). Si el navegador no admite Conditional Mediation, el método simplemente no realiza ninguna acción.

<div id="parameters">
  ## Parámetros
</div>

<ParamField body="inputId?" type="string">
  El ID del elemento `<input>` del nombre de usuario (sin `#`). Ejemplo: `"username"`. Cuando se proporciona, el SDK valida que el elemento exista, confirma que sea un `<input>` y sobrescribe su atributo `autocomplete` con `"username webauthn"`.

  Si se omite, debes configurar manualmente el elemento `<input>`:

  ```html
  <input id="username" autocomplete="username webauthn" />
  ```
</ParamField>

<div id="returns">
  ## Devuelve
</div>

`Promise`&lt;`void`&gt;

Se resuelve cuando finaliza el registro de la UI condicional.

<div id="example">
  ## Ejemplo
</div>

```typescript
import LoginId from '@auth0/auth0-acul-js/login-id';

async function initializeLogin() {
  const loginId = new LoginId();
  // Asegúrate de que el input HTML asociado exista:
  // <input id="username" autocomplete="username webauthn" />
  await loginId.registerPasskeyAutofill('username');
}

initializeLogin().catch(console.error);
```

<div id="gotchas">
  ## Consideraciones
</div>

* El atributo `autocomplete` **debe contener exactamente** `"username webauthn"`. Incluir tokens no relacionados, como `"email"` o `"text"`, impedirá que los navegadores muestren el menú desplegable de clave de acceso.
* Sobrescribir el atributo es intencional y necesario para garantizar un comportamiento coherente entre navegadores. No confíes en combinar ni ampliar los valores existentes de `autocomplete`.
* Si el navegador no admite Conditional Mediation, el SDK simplemente no realizará ninguna acción de forma segura.

<div id="remarks">
  ## Notas
</div>

Este método delega en la utilidad interna `registerPasskeyAutofill()` y devuelve un `AbortController` en segundo plano para gestionar la vigencia de la solicitud. Solo debe invocarse una vez por ciclo de vida de la página.
