La clase EmailVerificationResult implementa la funcionalidad de la pantalla `email-verification-result`. Esta pantalla muestra el resultado de un intento de verificación del correo electrónico.

<div id="constructors">
  ## Constructores
</div>

Crea una instancia del gestor de pantalla EmailVerificationResult:

```typescript Example
import EmailVerificationResult from '@auth0/auth0-acul-js/email-verification-result';
const emailVerificationResultManager = new EmailVerificationResult();
emailVerificationResultManager.changeLanguage({ language: 'fr' });
```

<div id="methods">
  ## Métodos
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma de visualización de la página de Universal Login.

  ```typescript Example
  import EmailVerificationResult from '@auth0/auth0-acul-js/email-verification-result';
  const emailVerificationResultManager = new EmailVerificationResult();
  emailVerificationResultManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/docs/es/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de configuración regional del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Si se establece en `'session'`, el idioma seleccionado se conserva durante toda la sesión.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/es/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Este método obtiene la lista de errores de transacción del contexto, o una lista vacía si no existe ninguno.
</ParamField>
