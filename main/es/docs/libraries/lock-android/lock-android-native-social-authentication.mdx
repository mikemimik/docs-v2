<div id="native-provider-google">
  ## Proveedor nativo - Google
</div>

Puedes usar Google AuthProvider para iniciar sesión con o sin **Lock**. Asegúrate de seguir las instrucciones de la sección [Setup](#setup).

[Lock-Google.Android](https://github.com/okta-graveyard/Lock-Google.Android)
requiere Android API 15 o posterior y Google Play Services 10.+

<div id="latest-version-of-lock-google">
  ### Última versión de Lock-Google
</div>

Lock-Google está disponible en [Maven Central](https://central.sonatype.com/). Para instalarlo, simplemente agrega la siguiente línea a tu `build.gradle`:

`compile 'com.auth0.android:lock-google:1.+'`

Puedes consultar la versión más reciente en el [Readme](https://github.com/okta-graveyard/Lock-Google.Android#install) del repositorio o en [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock-google).

<div id="lock-google-setup">
  ### Lock-Google Setup
</div>

<div id="google-cloud-console">
  #### Google Cloud Console
</div>

1. Ve a la [Google Cloud Console](https://console.cloud.google.com) y crea un proyecto nuevo.
2. Completa la [pantalla de consentimiento de OAuth](https://console.cloud.google.com/apis/credentials/consent) proporcionando al menos una dirección de correo electrónico válida y un nombre.
3. En el lado izquierdo verás el panel de navegación; haz clic en [Credentials](https://console.cloud.google.com/apis/credentials).
4. Crea una nueva credencial haciendo clic en el botón [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) y eligiendo **OAuth client ID**. A continuación, elige **Web Application** y asígnale un nombre como &quot;Auth0 Server Google-OAuth&quot;. Completa los **Authorized redirect URIs** rellenando el campo con tu URL de devolución de llamada, que debería verse así: `https://{yourDomain}/login/callback`. Asegúrate de pulsar ENTER antes de salir del campo y luego haz clic en el botón Create. Toma nota de los valores de `CLIENT ID` y `CLIENT SECRET`, ya que los usaremos más adelante.
5. Crea una nueva credencial haciendo clic en el botón [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) y eligiendo **OAuth client ID**. A continuación, elige **Android** y asígnale un nombre como &quot;Auth0 Android Google-OAuth&quot;. Obtén el **SHA-1** del certificado que usas para firmar tu aplicación y rellena con él el primer campo. Si necesitas ayuda para obtener el SHA-1, consulta [esta](#certificate-fingerprints) sección. Por último, completa el último campo con el **Package Name** de tu aplicación Android y luego haz clic en el botón Create. Toma nota del valor de `CLIENT ID`, ya que lo usaremos más adelante.

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Ve al Auth0 Dashboard y haz clic en [Social Connections](https://manage.auth0.com/#/connections/social).
2. Haz clic en **Google** y se abrirá un cuadro de diálogo.
3. Completa el campo &quot;Client ID&quot; con el valor de `CLIENT ID` obtenido en el paso 4 de la sección **Google Cloud Console** anterior.
4. Completa el campo &quot;Client Secret&quot; con el valor de `CLIENT SECRET` obtenido en el paso 4 de la sección **Google Cloud Console** anterior.
5. Completa el campo &quot;Allowed Mobile Client IDs&quot; con el `CLIENT ID` obtenido en el paso 5 de la sección **Google Cloud Console** anterior.
6. Haz clic en el botón Save.
7. Ve al Auth0 Dashboard y haz clic en [Applications](https://manage.auth0.com/#/applications). Si todavía no has creado una, hazlo primero y luego entra en la página de configuración de tu aplicación.
8. En la parte inferior de la página, haz clic en el enlace &quot;Show Advanced Settings&quot; y ve a la pestaña &quot;Device Settings&quot;.
9. En la sección de Android, completa **Package Name** con el nombre del paquete de tu aplicación. Por último, completa el campo **Key Hashes** con el SHA-256 del certificado que usas para firmar tu aplicación. Si necesitas ayuda para obtener el SHA-256, consulta [esta](#certificate-fingerprints) sección. Haz clic en el botón &quot;Save Changes&quot;.

<div id="android-application">
  #### Aplicación Android
</div>

1. En tu aplicación Android, crea un nuevo recurso String en el archivo `res/strings.xml`. Asígnale el nombre `google_server_client_id` y establece como valor el `CLIENT_ID` obtenido en el paso 5 de la sección de Setup de **Google Cloud Console** que aparece arriba.
2. Agrega los metadatos de versión de Google Play Services al archivo `AndroidManifest.xml`, dentro de la etiqueta Application.

   ```xml lines
   <meta-data
       android:name="com.google.android.gms.version"
       android:value="@integer/google_play_services_version" />
   ```
3. Agrega el permiso de Internet de Android a tu archivo `AndroidManifest.xml`.

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. Al crear una nueva instancia de `GoogleAuthProvider`, pasa el valor de `google_server_client_id` obtenido previamente como primer parámetro:

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private GoogleAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new GoogleAuthProvider(getString(R.string.google_server_client_id), client);
     }

     // ...
   }
   ```

Si necesitas más ayuda con la configuración del SDK de Google, consulta la [guía de inicio de sesión de Google para Android](https://developers.google.com/identity/android-credential-manager).

<div id="usage-without-lock">
  ### Uso sin Lock
</div>

Si piensas usar este proveedor sin **Lock**, asegúrate de sobrescribir el método `onActivityResult()` y redirigir la llamada a la instancia del proveedor. Por último, llama a start para iniciar el proceso de autenticación.

```kotlin lines
// Define tus propios códigos de solicitud
private static final int RC_PERMISSIONS = 101;
private static final int RC_AUTHENTICATION = 102;

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    if (provider.authorize(requestCode, resultCode, data)) {
        return;
    }
    super.onActivityResult(requestCode, resultCode, data);
}

private void beginAuthentication(){
  provider.start(this, callback, RC_PERMISSIONS, RC_AUTHENTICATION);
}
```

¡Eso es todo! Ya estás listo para ejecutar la aplicación e iniciar sesión con el proveedor nativo de Google.

<div id="additional-options">
  ### Opciones adicionales
</div>

<div id="using-a-custom-connection-name">
  #### Uso de un nombre de conexión personalizado
</div>

Para usar un nombre de conexión personalizado para la conexión social al autenticarte con Auth0, crea la instancia de GoogleAuthProvider mediante el segundo constructor:

```kotlin wrap lines
GoogleAuthProvider provider = new GoogleAuthProvider("my-connection", "google-server-client-id", client);
```

<div id="send-additional-authentication-parameters">
  #### Enviar parámetros de autenticación adicionales
</div>

Para enviar parámetros adicionales en la llamada de autenticación, usa `setParameters`.

```kotlin lines
Map<String, Object> parameters = new HashMap<>();
//Agregar entradas
provider.setParameters(parameters);
```

<div id="requesting-a-custom-google-scope">
  #### Solicitar un scope personalizado de Google
</div>

De forma predeterminada, se solicita el scope `Scopes.PLUS_LOGIN`. Puedes personalizar los scopes llamando a `setScopes` con la lista de scopes. Cada API de Google (Auth, Drive, Plus..) especifica su propia lista de scopes de Google.

```kotlin wrap lines
provider.setScopes(Arrays.asList(new Scope(Scopes.PLUS_ME), new Scope(Scopes.PLUS_LOGIN)));
```

<div id="requesting-custom-android-runtime-permissions">
  #### Solicitar permisos personalizados de runtime de Android
</div>

Este proveedor no necesita ningún permiso especial en el manifiesto de Android para autenticar al usuario. Pero, si tu caso de uso sí los necesita, puedes hacer que AuthProvider se encargue de gestionarlos. Usa el método `setRequiredPermissions`.

```kotlin wrap lines
provider.setRequiredPermissions(new String[]{"android.permission.GET_ACCOUNTS"});
```

Si no usas Lock, tendrás que gestionar tú mismo el resultado de la solicitud de permisos. Para ello, haz que tu actividad implemente la interfaz `ActivityCompat.OnRequestPermissionsResultCallback`. Cuando se invoque el método `onRequestPermissionsResult`, pasa el resultado al proveedor llamando a `provider.onRequestPermissionsResult`.

#### Cerrar sesión / borrar cuenta.

Para cerrar la sesión del usuario y hacer que la próxima vez se le pida que introduzca sus credenciales, llama a `clearSession`. Después de hacerlo, el estado del proveedor dejará de ser válido y tendrás que llamar a `start` de nuevo antes de intentar `authorize` un resultado. Llamar a `stop` tiene el mismo efecto.

`provider.clearSession();`

<div id="remember-the-last-login">
  #### Recordar el último inicio de sesión
</div>

De forma predeterminada, este proveedor recordará la última cuenta utilizada para iniciar sesión. Si quieres cambiar este comportamiento, utiliza el siguiente método.

`provider.rememberLastLogin(false);`

<div id="certificate-fingerprints">
  #### Huellas digitales del certificado
</div>

Al crear una nueva <Tooltip tip="OAuth 2.0: Marco de autorización que define los protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/docs/es/glossary?term=OAuth">credencial de OAuth</Tooltip> en Google Cloud Console, tendrás que proporcionar el SHA-1 del certificado que uses para firmar tu aplicación. Al completar la configuración de tu aplicación en el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/docs/es/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, también tendrás que proporcionar el valor SHA-256. Aquí tienes un ejemplo del comando de la terminal para obtener el valor y un resultado de ejemplo.

Comando:

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

Resultado de ejemplo:

`no71633JAC3qgzQYCbskprUr55k=`

Si necesitas ayuda, puedes seguir esta [guía de keystores](/docs/es/libraries/auth0-android/android-development-keystores-hashes) para obtener esos valores.

<div id="lock-facebook-setup">
  ### Setup de Lock-Facebook
</div>

<div id="facebook-developers-console">
  #### Consola de desarrolladores de Facebook
</div>

1. Ve a la [Consola de desarrolladores de Facebook](https://developers.facebook.com/) y crea una App nueva: elige &quot;Android&quot; y asígnale un nombre válido. Haz clic en &quot;Create new Facebook App ID&quot;.
2. Agrega el **Package Name** de tu aplicación y el nombre de la **clase Activity** en la que estás usando el proveedor, y haz clic en el botón Next.
3. Agrega los hashes de clave **SHA-1** codificados en Base64 de los certificados que usas para firmar tu aplicación y haz clic en el botón Next. Si necesitas ayuda para obtener el SHA-1, consulta [esta](#certificate-fingerprints) sección.
4. Por último, desplázate hasta la parte superior de la página y haz clic en el botón Skip Quickstart para ir a la página de tu app de Facebook.
5. En la parte superior de la página encontrarás los valores de `APP ID` y `APP SECRET`. Guárdalos, ya que los necesitarás más adelante.
6. En el lado izquierdo tienes el panel de navegación. Haz clic en Settings y luego en Basic. Activa el interruptor de **inicio de sesión único** y haz clic en el botón Save.
7. Haz clic en Settings y luego en Advanced. Activa el interruptor de **Native or desktop app?**.

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Ve al Auth0 Dashboard y haz clic en [Social Connections](https://manage.auth0.com/#/connections/social).
2. Haz clic en **Facebook** y se abrirá un cuadro de diálogo.
3. Completa el campo &quot;App ID&quot; con el valor de `APP ID` obtenido en el paso 5 de la sección **Facebook Developers Console** anterior.
4. Completa el campo &quot;App Secret&quot; con el valor de `APP SECRET` obtenido en el paso 5 de la sección **Facebook Developers Console** anterior.
5. Haz clic en el botón Save.
6. Ve al Auth0 Dashboard y haz clic en [Applications](https://manage.auth0.com/#/applications). Si aún no has creado una, hazlo primero y luego entra en la página de configuración de tu aplicación.
7. Al final de la página, haz clic en el enlace &quot;Show Advanced Settings&quot; y ve a la pestaña &quot;Device Settings&quot;.
8. En la sección Android, completa **Package Name** con el nombre del paquete de tu aplicación. Por último, completa el campo **Key Hashes** con el SHA-256 del certificado que usas para firmar tu aplicación. Si necesitas ayuda para obtener el SHA-256, consulta [esta](#certificate-fingerprints) sección. Haz clic en el botón &quot;Save Changes&quot;.

<div id="android-application">
  #### Aplicación Android
</div>

1. En tu aplicación Android, crea un nuevo recurso de tipo String en el archivo `res/strings.xml`. Asígnale el nombre `facebook_app_id` y establece como valor el `APP ID` obtenido en el paso 5 de la sección de Setup de **Facebook Developers Console** anterior.
2. Agrega `FacebookActivity` y los metadatos de `facebook_app_id` al archivo `AndroidManifest.xml`, dentro de la etiqueta Application.

   ```xml lines
   <activity
       android:name="com.facebook.FacebookActivity"
       android:configChanges="keyboard|keyboardHidden|screenLayout|screenSize|orientation"
       android:label="@string/app_name"/>
   <meta-data
       android:name="com.facebook.sdk.ApplicationId"
       android:value="@string/facebook_app_id" />
   ```
3. Agrega el permiso de acceso a Internet de Android a tu archivo `AndroidManifest.xml`.

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. Crea una nueva instancia de `FacebookAuthProvider`.

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private FacebookAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new FacebookAuthProvider(client);
     }

     // ...
   }
   ```

Si necesitas más ayuda con la configuración del Facebook SDK, consulta la [Guía de introducción](https://developers.facebook.com/docs/android/getting-started) de Facebook.

### Huellas digitales del certificado

Al crear una nueva credencial de OAuth en la consola de desarrolladores de Facebook, deberás proporcionar el SHA-1 del certificado que usas para firmar tu aplicación. Al completar la configuración de tu aplicación en el Auth0 Dashboard, también deberás proporcionar el valor SHA-256. Aquí tienes un ejemplo del comando de la terminal para obtener el valor y un ejemplo del resultado.

Comando:

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

Salida de ejemplo:

SHA1: BB:0D:AC:74:D3:21:E1:43:07:71:9B:62:90:AF:A1:66:6E:44:5D:75
SHA256: 15:B9:F9:33:9F:E4:E3:68:C2:10:49:17:5D:A8:77:12:7C:8E:57:E9:FF:B7:23:EA:CC:DD:56:08:06:C9:5E:33

Si necesitas ayuda, puedes seguir esta [guía sobre keystores](/docs/es/libraries/auth0-android/android-development-keystores-hashes) para obtener esos valores.
