Al crear una nueva credencial de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=OAuth">OAuth</Tooltip> para muchas conexiones, deberás proporcionar el SHA-1 del certificado que usas para firmar tu aplicación. Al completar la configuración de tu aplicación en el [Auth0 Dashboard](https://manage.auth0.com/#), también deberás proporcionar el valor SHA-256.

Localiza el certificado que usas para firmar tu aplicación. Si no tienes uno, puedes generarlo. En las aplicaciones de producción, deberías hacerlo.

Durante el desarrollo, puedes firmar tu aplicación con el certificado predeterminado `android.keystore`, que se generó automáticamente cuando instalaste el SDK. En este ejemplo, vamos a usar este keystore predeterminado. Para generar los hashes de clave con este keystore, sigue los ejemplos que aparecen a continuación.

<div id="generating-your-key-hashes">
  ## Generando tus hashes de clave
</div>

**En Windows:**

`keytool -list -v -keystore "%USERPROFILE%\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android`

**En Linux / macOS:**

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

<div id="using-your-key-hashes">
  ## Uso de tus hashes de clave
</div>

Una vez que tengas tus hashes de clave generados, copia el valor SHA256 resultante y ve a la configuración de tu aplicación en el [Auth0 Dashboard](https://manage.auth0.com/#/applications). Haz clic en &quot;Show Advanced Settings&quot; y, en la pestaña &quot;Device Settings&quot;, en &quot;Android&quot;, rellena &quot;App Package Name&quot; con el nombre del paquete de tu aplicación y el campo &quot;Key Hashes&quot; con el valor SHA256 que copiaste. No olvides guardar los cambios.

Si no agregas la URL de devolución de llamada a la configuración de la aplicación ni los hashes de clave a la configuración del dispositivo de la aplicación, el servidor de Auth0 no devolverá el resultado de la llamada a tu aplicación.
