---
description: Personalización de la interfaz de usuario de Lock.Android
title: 'Lock.Android: Tematización personalizada'
---

Puedes personalizar la apariencia de la interfaz de usuario de **Lock.Android**. Además de varios elementos, como el logotipo y el título del encabezado, puedes modificar algunos colores, botones y otros elementos para adaptarlos al tema de tu aplicación.

<div id="supported-attributes-list">
  ## Lista de atributos admitidos
</div>

| Nombre                 | Tipo                 | Descripción                                                                                                                                |
| ---------------------- | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Auth0.HeaderLogo       | drawable - reference | Logotipo drawable que se muestra dentro del encabezado. La resolución mínima recomendada es de 200 píxeles (ancho) por 200 píxeles (alto). |
| Auth0.HeaderTitle      | string - reference   | Texto que se mostrará como título dentro del encabezado.                                                                                   |
| Auth0.HeaderTitleColor | color - reference    | Color del texto del título.                                                                                                                |
| Auth0.HeaderBackground | color - reference    | Se usa como color de fondo del encabezado.                                                                                                 |
| Auth0.PrimaryColor     | color - reference    | Se usa como estado *normal* en widgets, como el botón Submit. También se usa como color de *acento*.                                       |
| Auth0.DarkPrimaryColor | color - reference    | Se usa como estado *presionado* en widgets, como el botón Submit.                                                                          |

<div id="create-a-new-resource-file">
  ## Crear un nuevo archivo de recursos
</div>

Primero, crea un nuevo `Theme` que herede de `Lock.Theme` y sobrescribe los atributos que quieras personalizar. Los atributos que no sobrescribas usarán por defecto los definidos en `Lock.Theme`.

```xml lines
<resources>
  <style name="MyTheme" parent="Lock.Theme">
    <item name="Auth0.HeaderLogo">@drawable/com_auth0_lock_header_logo</item>
    <item name="Auth0.HeaderTitle">@string/com_auth0_lock_header_title</item>
    <item name="Auth0.HeaderTitleColor">@color/com_auth0_lock_text</item>
    <item name="Auth0.HeaderBackground">@color/com_auth0_lock_header_background</item>
    <item name="Auth0.PrimaryColor">@color/com_auth0_lock_submit_normal</item>
    <item name="Auth0.DarkPrimaryColor">@color/com_auth0_lock_submit_pressed</item>
  </style>
</resources>
```

Para empezar a usar el nuevo tema, configura la Activity en el archivo `AndroidManifest.xml` con el atributo `android:theme`. Según si usas Classic Lock o <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">Passwordless</Tooltip> Lock, la declaración de la actividad que debes actualizar será distinta. Como la biblioteca Lock declara estas actividades internamente, debes volver a declararlas con el atributo especial tools`:replace`, que sobrescribirá la declaración de la biblioteca solo para el atributo de tema.

```xml lines
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.company.app">
    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        <!-- Classic -->
        <activity
            android:name="com.auth0.android.lock.LockActivity"
            android:theme="@style/MyTheme"
            tools:replace="android:theme" />
        <!-- Passwordless -->
        <activity
            android:name="com.auth0.android.lock.PasswordlessLockActivity"
            android:theme="@style/MyTheme"
            tools:replace="android:theme" />
    </application>
</manifest>
```

<Warning>
  Ten en cuenta que, si defines tu propio Tema en un archivo de recursos de estilo y olvidas especificar que el padre del Tema es `Lock.Theme`, o si olvidas indicar en el manifiesto cuál será el Tema de la Activity, la interfaz de usuario tendrá un aspecto realmente malo. Esto también puede ocurrir si los valores que especificas en tu Tema personalizado no son válidos.
</Warning>

<div id="custom-oauth-connection-buttons">
  ## Botones personalizados de conexión OAuth
</div>

Para personalizar el estilo de una conexión de un <Tooltip tip="proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip> de terceros, debes crear una nueva conexión con la [extensión](https://manage.auth0.com/#/extensions) `Custom Social Connections` y completar todos los campos obligatorios antes de guardar los cambios.

Para personalizar el estilo de una conexión de un proveedor de identidad de terceros en Lock, llama al constructor y pasa tanto `connectionName` como el `style` que quieras usar.

Primero, crea un estilo personalizado que extienda `Lock.Theme.AuthStyle.` Define el logotipo, el color de fondo y el nombre de la conexión con nombres de clave similares a los del ejemplo siguiente.

```xml lines
<style name="Style.Facebook" parent="Lock.Theme.AuthStyle">
    <item name="Auth0.BackgroundColor">@color/facebook_color</item>
    <item name="Auth0.Name">@string/facebook_name</item>
    <item name="Auth0.Logo">@drawable/facebook_logo</item>
</style>
```

En la configuración del builder, agrega el `AuthStyle` para el nombre de la conexión que quieres personalizar.

`builder.withAuthStyle("facebook", R.style.Style_Facebook)
.build(...);`

Cuando **Lock** necesite mostrar esa conexión en un **SocialButton**, primero buscará los estilos personalizados por el usuario. Si no encuentra ninguno, usará el estilo predeterminado de Lock. Siguiendo el ejemplo anterior, esto significa que para `facebook` usaría el color de fondo de Facebook, el logotipo de Facebook y &quot;`FACEBOOK"` como nombre.
