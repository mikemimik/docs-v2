---
description: Añadir campos adicionales al registro con Lock.Android
title: 'Lock.Android: Campos personalizados en el registro'
---

**Lock.Android** te permite especificar campos adicionales que el usuario debe completar antes de crear una nueva cuenta. Los campos adicionales se mostrarán en una segunda pantalla después de que el usuario complete los campos básicos (correo electrónico, nombre de usuario, contraseña).

<div id="create-the-custom-fields">
  ## Crea los campos personalizados
</div>

Crea un nuevo objeto `CustomField` pasando estos 4 parámetros obligatorios.

1. Icono: El `int` que apunta al recurso que quieres usar como icono (que sea pequeño).
2. Tipo: El `FieldType` que se usará en este campo. El tipo define la distribución del teclado y, a veces, la validación de la entrada.
3. Clave: El `String` que identifica este valor en el JSON resultante. ¡No debe repetirse! Si se repiten claves de campo, el segundo campo se eliminará de la lista.
4. Sugerencia: El `@StringRes` del texto que se mostrará como sugerencia en el campo.

Además, puedes especificar dónde se almacenará el campo en el perfil del usuario. Consulta la sección [almacenamiento](#storage) más abajo para obtener más detalles.

También puedes crear campos que no se mostrarán al usuario, pero que pueden usarse para incluir metadatos adicionales de forma dinámica después del registro.

```kotlin lines
// Campo regular
val nameField = CustomField(R.drawable.ic_name, FieldType.TYPE_NAME, "first_name", R.string.hint_first_name, CustomField.Storage.PROFILE_ROOT)

// Campo oculto
val androidField = HiddenField("android_version", Build.VERSION.SDK_INT.toString(), Storage.USER_METADATA)

val customFields = listOf(nameField, androidField)
```

Repite los pasos anteriores tantas veces como campos necesites.

<div id="use-the-custom-fields">
  ## Usa los campos personalizados
</div>

Pasa la lista de campos personalizados a la instancia de Lock mientras la creas, usando el método `withSignUpFields()`.

```kotlin lines
val lock = Lock.newBuilder(auth0, callback)
              .withSignUpFields(customFields)
              //...
              .build(this)
```

Si has habilitado el registro en el Dashboard de la aplicación, se les pedirá a los usuarios que completen los campos restantes después de completar los campos básicos (correo electrónico/nombre de usuario, contraseña) y hacer clic en **Submit**. El usuario debe completar todos los campos personalizados antes de poder finalizar el registro.

Durante el registro, los campos adicionales se adjuntan al atributo `user_metadata` o se establecen directamente en el perfil principal del usuario, según la opción de almacenamiento elegida. Puedes acceder a estas propiedades consultando el perfil del usuario en cualquier momento, incluso desde el Dashboard en la sección Users.

<div id="field-types">
  ## Tipos de campo
</div>

Cada campo personalizado solo puede tener asociado un `FieldType`.

* TYPE&#95;NAME
* TYPE&#95;NUMBER
* TYPE&#95;PHONE&#95;NUMBER
* TYPE&#95;EMAIL

<div id="storage">
  ## Almacenamiento
</div>

Cada campo personalizado solo puede tener un `Storage` asociado. Puedes elegir almacenarlo en el nivel raíz, en un atributo raíz del perfil, o dentro del atributo `user_metadata`. Para especificar la ubicación de almacenamiento, usa el constructor de cinco parámetros y pasa el parámetro `Storage` que prefieras. De forma predeterminada, los campos se almacenarán dentro del atributo `user_metadata`.

Opciones disponibles:

* PROFILE&#95;ROOT
* USER&#95;METADATA (predeterminado)

Para que los campos se guarden en el nivel raíz del perfil del usuario, sus claves deben coincidir con las que se enumeran en la [documentación del endpoint](https://auth0.com/docs/api/authentication#signup).
