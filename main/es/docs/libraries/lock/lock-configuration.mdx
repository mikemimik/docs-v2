Lock puede configurarse mediante el parámetro `options` que se envía al constructor. Estas opciones pueden modificar la forma en que se comporta el widget de Lock, cómo gestiona las conexiones, los campos adicionales de registro que requiere tu proyecto, los valores de idioma y texto, los colores y las imágenes del widget, entre muchas otras cosas. Consulta el índice que aparece a continuación si sabes lo que buscas, o explora las opciones para obtener más detalles.

  ## UI

<div id="ui-options">
  ## Opciones de la UI
</div>

### allowAutocomplete {Boolean}

Determina si los campos de correo electrónico o nombre de usuario permiten el autocompletado (`<input autocomplete />`). De forma predeterminada, es `false`.

`allowAutocomplete: true`

### allowPasswordAutocomplete {Boolean}

Determina si el campo de contraseña permitirá el autocompletado (`<input autocomplete />`). El valor predeterminado es `false`.

Establece `allowPasswordAutocomplete` en `true` para admitir gestores de contraseñas y evitar otros comportamientos no deseados.

`allowPasswordAutocomplete: true`

### allowShowPassword {Boolean}

Esta opción determina si se añade o no una casilla en la UI que, al seleccionarla, permite al usuario mostrar su Password mientras lo escribe. El valor predeterminado de esta opción es `false`.

`allowShowPassword: true`

Lock con `allowShowPassword` establecido en `true` y activado para mostrar el Password:

<Frame>![Opción de Lock: Permitir mostrar el Password](/docs/images/cdy7uua7fh8z/6G8LJaNjaMl5zUrwGi5UJQ/44dc951b70077b0ec6bc04d2f0413b12/lock-allowshowpassword.png)</Frame>

### allowedConnections {Arreglo}

Arreglo de conexiones que se utilizarán para las acciones `signin|signup|reset`. De forma predeterminada, se utilizan todas las conexiones habilitadas.

`allowedConnections: ['Username-Password-Authentication']`

`allowedConnections: ['twitter', 'facebook', 'linkedin']`

`allowedConnections: ['qraftlabs.com']`

Ejemplos de `allowedConnections`:

<Frame>![Opción de Lock: Conexiones permitidas - Database](/docs/images/cdy7uua7fh8z/1mllPDLFCCETwQ9MeGEf3W/763728971302656618ef3bd131ab83b1/lock-allowedconnections-database.png)</Frame><Frame>![Opción de Lock: Conexiones permitidas - Social](/docs/images/cdy7uua7fh8z/7uYN4ebrLLM8bjx1fee8j0/1e716cca3e1bae990063e9226dfe9ac2/lock-allowedconnections-social.png)</Frame>

### autoclose {Boolean}

Determina si Lock se cerrará automáticamente tras iniciar sesión correctamente. El valor predeterminado es false. Si Lock no es `closable`, no se cerrará, aunque esta opción esté configurada como true.

`autoclose: true`

### autofocus {Boolean}

Si es `true`, el foco se sitúa en el primer campo del widget. El valor predeterminado es `false` cuando se renderiza en un dispositivo móvil o si se proporciona la opción `container`; en todos los demás casos, el valor predeterminado es `true`.

`autofocus: false`

### avatar {Object}

De forma predeterminada, Gravatar se utiliza para obtener el avatar del usuario y el nombre para mostrar, pero puedes obtenerlos de cualquier fuente con la opción `avatar`.

```js lines
var options = {
  avatar: {
    url: function(email, cb) {
      // Obtiene la URL del avatar para el email ingresado por el usuario, Lock
      // precargará la imagen antes de mostrarla.
      // Ten en cuenta que en caso de error debes llamar a cb con el error en
      // el primer argumento en lugar de `null`.
      var url = obtainAvatarUrl(email);
      cb(null, url);
    },
    displayName: function(email, cb) {
      // Obtiene el nombre para mostrar para el email ingresado por el usuario.
      // Ten en cuenta que en caso de error debes llamar a cb con el error en
      // el primer argumento en lugar de `null`.
      var displayName = obtainDisplayName(email);
      cb(null, displayName);
    }
  }
};
```

O bien, si no quieres mostrar ningún avatar, simplemente pasa `null`.

`avatar: null`

Comportamiento predeterminado con Gravatar:

<Frame>![Lock Option: Avatar](/docs/images/cdy7uua7fh8z/69Nz0QoQifPWpfBZwX15J0/204aad93cde406f0ed3771ce38766f56/lock-avatar.png)</Frame>

### closable {Boolean}

Determina si se puede cerrar Lock o no. Cuando se proporciona la opción `container`, su valor siempre es `false`; de lo contrario, el valor predeterminado es `true`.

`closable: false`

<Frame>![Opción de Lock: Closable](/docs/images/cdy7uua7fh8z/2qwXtPQev1nlpyOQauXKBO/f967e1157972ecbf5b030b3be3600f2e/lock-closable.png)</Frame>

### container {String}

El `id` del elemento HTML donde se mostrará el widget.

Esto hace que el widget aparezca integrado en tu `div` en lugar de en una ventana modal emergente.

```html lines
<div id="hiw-login-container"></div>

<script>
  var options = {
    container: 'hiw-login-container'
  };

  // inicializar
  var lock = new Auth0Lock('xxxxxx', '<account>.auth0.com', options);

  // renderizar
  lock.show();
</script>
```

<Frame>![Lock Option: Contenedor](/docs/images/cdy7uua7fh8z/5L4S47Yfset2DR9pjcqVrX/1c4d1928cf75acf40129ffd01394bf33/lock-container.png)</Frame>

### flashMessage {Object}

Muestra un mensaje flash de `error` o `success` al mostrarse Lock. Este objeto tiene las siguientes propiedades:

* type &#123;String&#125;: El tipo de mensaje; los tipos admitidos son `error`, `info` y `success`
* text &#123;String&#125;: El texto que se debe mostrar.

```js lines
var options = {
  flashMessage: {
    type: 'success',
    text: 'Welcome!'
  }
};
```

### language {String}

Especifica el idioma del widget. El valor predeterminado es &quot;en&quot;. Consulta el [directorio de internacionalización](https://github.com/auth0/lock/blob/master/src/i18n/) para obtener una lista actualizada de los idiomas disponibles.

`language: 'es'`

<Frame>![Lock Option: Idioma](/docs/images/cdy7uua7fh8z/3u3lytCUBDbcKmXFV34OUv/5cf5d7e1af9593a3b094f32815a4da11/lock-language.png)</Frame>

### languageDictionary {Object}

Permite personalizar todos los textos que se muestran en Lock. El valor predeterminado es {"{}"}. Consulta la [especificación del diccionario de idiomas](https://github.com/auth0/lock/blob/master/src/i18n/en.js) en inglés para ver la lista completa de valores de `languageDictionary` que se pueden modificar con este objeto.

```js lines
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Log me in"
  },
};
```

<Frame>![Opción de Lock: diccionario de idiomas](/docs/images/cdy7uua7fh8z/5203ofwtagtIKGT4HnIPmk/8ad9cdd151a2cdb18b8a42ee47dbf679/lock-languagedictionary.png)</Frame>

Además, consulta también la página [Personalización de mensajes de error](/docs/es/customize/login-pages/classic-login/customize-lock-error-messages) o la página [Internacionalización](/docs/es/customize/internationalization-and-localization/lock-internationalization) para obtener más información sobre el uso de la opción `languageDictionary`.

### popupOptions {Object}

Permite personalizar la ubicación del popup en la pantalla. Se admite cualquier opción de posición y tamaño permitida por `window.open`. El valor predeterminado es {"{}"}.

Opciones para las características de `window.open`. Esto solo se aplica si `redirect` se establece en `false`.

```js lines
var options = {
  auth: {
      redirect: false
  },
  popupOptions: { width: 300, height: 400, left: 200, top: 300 }
};
```

### rememberLastLogin {Boolean}

Determina si se muestra o no una pantalla que te permite iniciar sesión rápidamente con la cuenta que usaste la última vez.
Solicita datos de <Tooltip tip="inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, lo inicia automáticamente en otras aplicaciones." cta="Ver glosario" href="/docs/es/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SSO) y habilita el mensaje **La última vez iniciaste sesión con**. El valor predeterminado es `true`. Esta información proviene de la sesión de Auth0 del usuario, por lo que esta funcionalidad durará lo mismo que su sesión de Auth0 (que es configurable).

`rememberLastLogin: false`

Los tenants nuevos [tienen Seamless SSO habilitado automáticamente](/docs/es/get-started/tenant-settings/enable-sso-for-legacy-tenants). Con esta opción habilitada, `rememberLastLogin` deja de ser relevante porque, si ya existe una sesión, la página de <Tooltip tip="Universal Login: tu aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/docs/es/glossary?term=Universal+Login">Universal Login</Tooltip> no se mostrará en absoluto. Se recomienda encarecidamente usar Seamless SSO porque ofrece una experiencia de autenticación fluida: los usuarios inician sesión una vez y no tendrán que volver a introducir credenciales cuando naveguen por las aplicaciones que has creado o por aplicaciones de terceros. Si el usuario no ha iniciado sesión, se lo redirigirá a la pantalla de inicio de sesión, como es de esperar. Además, el mensaje **La última vez iniciaste sesión con** no estará disponible en las siguientes circunstancias:

* Usaste Lock en una [Hosted Login Page](/docs/es/authenticate/login/auth0-universal-login) con la sesión establecida mediante [autenticación passwordless](/docs/es/libraries/lock/lock-configuration#passwordlessmethod-string-).
* Usaste Lock en un [escenario de embedded login](/docs/es/authenticate/login/universal-vs-embedded-login) donde `responseType: code` (lo que indica el [flujo de código de autorización](/docs/es/get-started/authentication-and-authorization-flow/authorization-code-flow), que se usa para Regular Web Apps).

### scrollGlobalMessagesIntoView {Boolean}

Indica si `globalMessage` debe desplazarse hasta quedar visible para el usuario. El valor predeterminado es `true`.

`scrollGlobalMessagesIntoView: false`

<div id="theme-options">
  ## Opciones del tema
</div>

Las opciones del tema se agrupan en la propiedad `theme` del objeto `options`.

```js lines
var options = {
  theme: {
    labeledSubmitButton: false,
    logo: "https://example.com/assets/logo.png",
    primaryColor: "green",
    authButtons: {
      connectionName: {
        displayName: "...",
        primaryColor: "...",
        foregroundColor: "...",
        icon: "https://.../logo.png"
      }
    }
  }
};
```

### authButtons {Object}

Permite personalizar los botones de Lock con conexiones personalizadas de OAuth2. Debe incluirse por nombre cada conexión personalizada cuyo botón quieras personalizar, cada una con su propio conjunto de parámetros. Los parámetros que se pueden personalizar se enumeran a continuación:

* **displayName** &#123;String&#125;: El nombre que se mostrará en lugar del nombre de la conexión al generar el título del botón, como `LOGIN WITH MYCONNECTION` para el inicio de sesión).
* **primaryColor** &#123;String&#125;: El color de fondo del botón. El valor predeterminado es `#eb5424`.
* **foregroundColor** &#123;String&#125;: El color del texto del botón. El valor predeterminado es `#FFFFFF`.
* **icon** &#123;String&#125;: La URL del icono de esta conexión. Por ejemplo: `http://site.com/logo.png`.

```js lines
var options = {
  theme: {
    authButtons: {
      "testConnection": {
        displayName: "Test Conn",
        primaryColor: "#b7b7b7",
        foregroundColor: "#000000",
        icon: "http://example.com/icon.png"
      },
      "testConnection2": {
        primaryColor: "#000000",
        foregroundColor: "#ffffff",
      }
    }
  }
};
```

### labeledSubmitButton {Boolean}

Esta opción indica si el botón de envío debe llevar una etiqueta, y su valor predeterminado es `true`. Si se establece en `false`, se mostrará un icono en su lugar.

```js lines
var options = {
  theme: {
    labeledSubmitButton: false
  }
};
```

<Frame>![Opción de Lock: botón de envío con etiqueta](/docs/images/cdy7uua7fh8z/5PNCvsbYa98pKZwWHVPS9S/39cb5ac63818ec5558c451b1e37f735f/lock-theme-labeledsubmitbutton.png)</Frame>

Si la etiqueta está configurada como true, que es el valor predeterminado, su texto se puede personalizar mediante la opción [languageDictionary](#languagedictionary-object-).

### logo {String}

El valor de `logo` es la URL de una imagen que se colocará en el encabezado de Lock y, de forma predeterminada, usa el logotipo de Auth0. Se recomienda una altura máxima de `58px` para ofrecer una mejor experiencia de usuario.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png'
  }
};
```

<Frame>![Personalización de la UI de Lock - Logotipo](/docs/images/cdy7uua7fh8z/FdOAKKy71rTbFjIlZEL8D/2eef96008095d85388daaa461707901d/lock-theme-logo.png)</Frame>

### primaryColor {String}

La propiedad `primaryColor` define el color principal de Lock; todos los colores utilizados en el widget se calcularán a partir de él. Esta opción resulta útil al proporcionar un `logo` personalizado, para garantizar que todos los colores combinen bien con la paleta de colores del `logo`. El valor predeterminado es `#ea5323`.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png',
    primaryColor: '#31324F'
  }
};
```

<Frame>![Personalización de la UI de Lock: color principal](/docs/images/cdy7uua7fh8z/1qQjeTmXJyIziv400VPaOn/48dd3cb8a72a6e88b2634b4b8a8bed63/lock-theme-primarycolor.png)</Frame>

<div id="authentication-options">
  ## Opciones de autenticación
</div>

Las opciones de autenticación se agrupan en la propiedad `auth` del objeto `options`.

El scope predeterminado que usa Lock es `openid profile email`.

```js lines
var options = {
  auth: {
   params: {
    param1: "value1",
    scope: "openid profile email"
   },
   autoParseHash: true,
   redirect: true,
   redirectUrl: "some url",
   responseMode: "form_post",
   responseType: "token",
   sso: true,
   connectionScopes: {
    connectionName: [ 'scope1', 'scope2' ]
   }
  }
};
```

### audience {String}

La opción `audience` indica qué API consumirá el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/docs/es/glossary?term=Access+Token">Token de acceso</Tooltip> que se recibe después de la autenticación.

export const codeExample = `var options = {
  auth: {
    audience: 'https://{yourDomain}/userinfo',
  }
}`;

### autoParseHash {Boolean}

Cuando `autoParseHash` se establece en `true`, Lock analizará la cadena `window.location.hash` al instanciarse. Si se establece en `false`, tendrás que reanudar manualmente la autenticación mediante el método [resumeAuth](/docs/es/libraries/lock/lock-api-reference).

```js lines
var options = {
  auth: {
    autoParseHash: false
  }
};
```

### connectionScopes {Object}

Esta opción te permite definir los scopes que se enviarán a la conexión oauth2/social para la autenticación.

```js lines
var options = {
  auth: {
    connectionScopes: {
      'facebook': ['scope1', 'scope2']
    }
  }
};
```

Se puede obtener del proveedor en cuestión una lista de los `scopes` específicos para tus conexiones sociales. Por ejemplo, la documentación de [Facebook for Developers](https://developers.facebook.com/docs/facebook-login/permissions/) incluye una lista de permisos distintos que se pueden solicitar para tu conexión.

### params {Object}

Puede enviar parámetros al iniciar sesión agregándolos al objeto options. El ejemplo siguiente agrega un parámetro `state` con el valor `foo` y también agrega un parámetro `scope` (que incluye el scope y, a continuación, los atributos solicitados).

```js lines
var options = {
  auth: {
    params: {
      state: 'foo',
      scope: 'openid email user_metadata app_metadata picture'
    }
  }
};
```

Para más información sobre los parámetros compatibles, consulta [Parámetros de autenticación de Lock](/docs/es/libraries/lock/lock-authentication-parameters).

### redirect {Boolean}

El valor predeterminado es true. Cuando se establece en true, se usa el modo de redirección. Si se establece en false, se usa el [modo popup](/docs/es/libraries/lock/lock-authentication-modes).

```js lines
var options = {
  auth: {
    redirect: false
  }
};
```

### redirectUrl {String}

La URL a la que Auth0 redirigirá después de la autenticación. El valor predeterminado es la cadena vacía &quot;&quot; (sin URL de redirección).

```js lines
var options = {
  auth: {
    redirectUrl: 'http://testurl.com'
  }
};
```

Cuando se proporciona `redirectUrl` (es decir, se establece con un valor no vacío), la opción `responseType` tomará por defecto el valor `code` si no se configura manualmente.

### responseMode {String}

Debe establecerse en `"form_post"` si quieres que el código o el token se transmitan mediante una solicitud HTTP POST a `redirectUrl`, en lugar de incluirse en los parámetros de consulta o en el fragmento de la URL.

De lo contrario, esta opción debe omitirse y, por defecto, se omite.

```js lines
var options = {
  auth: {
    responseMode: 'form_post'
  }
};
```

### responseType {String}

El valor de `responseType` debe establecerse en &quot;token&quot; para las aplicaciones de página única y, en caso contrario, en &quot;code&quot;. De forma predeterminada, es &quot;code&quot; cuando se proporciona redirectUrl y, de lo contrario, &quot;token&quot;.

```js lines
var options = {
  auth: {
    responseType: 'token'
  }
};
```

Cuando `responseType` se establece en `code`, Lock nunca mostrará el mensaje **La última vez que iniciaste sesión con** y siempre le pedirá al usuario sus credenciales.

## sso {Boolean}

El valor predeterminado es `true`. Si se establece en `true`, habilita el [inicio de sesión único](/docs/es/authenticate/single-sign-on) en Lock.

```js lines
var options = {
  auth: {
    sso: false
  }
};
```

<div id="database-options">
  ## Opciones de Database
</div>

### additionalSignUpFields {Arreglo}

Se pueden agregar campos de entrada adicionales a la pantalla de registro con la opción `additionalSignUpFields`. Cada opción que se agregue de esta manera se añadirá al `user_metadata` del usuario. Para obtener más información, consulta [Comprende cómo funcionan los metadatos en los perfiles de usuario](/docs/es/manage-users/user-accounts/metadata). Cada campo debe tener un `name` y un `placeholder`, y también se puede proporcionar una URL para `icon`. Además, se puede establecer un valor inicial con la opción `prefill`, que puede ser una cadena con el valor o una función que lo obtenga. Las demás opciones dependen del tipo de campo, que se define mediante la opción `type` y cuyo valor predeterminado es &quot;text&quot;.

<Card title="La opción additionalSignUpFields está pensada para usarse solo con registros de base de datos">
  Los `additionalSignUpFields` están pensados para usarse solo con registros de base de datos. Si también tienes registro con redes sociales, puedes solicitar la información adicional después de que los usuarios se registren (consulta esta [página sobre registro personalizado](/docs/es/libraries/custom-signup) para obtener más detalles). Puedes usar la clave de i18n `databaseAlternativeSignupInstructions` para mostrar estas instrucciones.
</Card>

Los nuevos campos se renderizan debajo de los campos de entrada habituales del registro, en el orden en que se proporcionan.

<div id="text-fields">
  #### Campos de texto
</div>

Los campos de texto son el tipo predeterminado de campo adicional para el registro. Ten en cuenta que también se puede proporcionar una función `validator`.

```js lines
var options = {
  additionalSignUpFields: [{
    name: "address",
    placeholder: "enter your address",
    // Las siguientes propiedades son opcionales
    icon: "https://example.com/assests/address_icon.png",
    prefill: "street 123",
    validator: function(address) {
      return {
         valid: address.length >= 10,
         hint: "Must have 10 or more chars" // opcional
      };
    }
  },
  {
    name: "full_name",
    placeholder: "Enter your full name"
  }]
}
```

Si no especificas un `validator`, el campo de texto será **obligatorio**. Si quieres que el campo de texto sea opcional, usa un validador que siempre devuelva `true`, así:

```js lines
var options = {
  additionalSignUpFields: [{
    name: "favorite_color",
    placeholder: "Enter your favorite color (optional)",
    validator: function() { 
      return true;
    }
  }]
}
```

Si desea guardar el valor del atributo en la raíz de su perfil, use `storage: 'root'`. Solo se puede almacenar de esta manera un subconjunto de valores. La lista de atributos que pueden agregarse a la raíz de su perfil está [aquí](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id). De forma predeterminada, cada campo adicional de registro se almacena dentro del objeto user&#95;metadata.

```js lines
var options = {
  additionalSignUpFields: [{
    name: "name",
    storage: "root"
  }]
};
```

<div id="select-field">
  #### Campo de selección
</div>

El campo de registro `type: "select"` te permitirá usar elementos de selección para que el usuario seleccione un valor.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "select",
    name: "location",
    placeholder: "choose your location",
    options: [
      {value: "us", label: "United States"},
      {value: "fr", label: "France"},
      {value: "ar", label: "Argentina"}
    ],
    // Las siguientes propiedades son opcionales
    icon: "https://example.com/assests/location_icon.png",
    prefill: "us"
  }]
}
```

Los elementos del arreglo `options` de los campos `select` deben seguir el siguiente formato:
`{label: “non empty string”, value: “non empty string”}`, y debe definirse al menos una opción.

Los valores de `options` y `prefill` pueden proporcionarse mediante una función:

```js lines
var options = {
  additionalSignUpFields: [{
    type: "select",
    name: "location",
    placeholder: "choose your location",
    options: function(cb) {
      // obtener opciones, en caso de error llama a cb con el error en el
      // primer argumento en lugar de null
      cb(null, options);
    },
    icon: "https://example.com/assests/location_icon.png",
    prefill: function(cb) {
      // obtener prefill, en caso de error llama a cb con el error en el
      // primer argumento en lugar de null
      cb(null, prefill);
    }
  }]
}
```

<div id="checkbox-field">
  #### Campo de casilla de verificación
</div>

El tercer tipo de campo de registro personalizado es `type: "checkbox"`. El valor de `prefill` puede definir el estado predeterminado de la casilla de verificación (`true` o `false`) y es obligatorio.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "checkbox",
    name: "newsletter",
    prefill: "true",
    placeholder: "I hereby agree that I want to receive marketing emails from your company"
  }]
}
```

<div id="hidden-field">
  #### Campo oculto
</div>

El campo de registro `type: "hidden"` te permitirá usar un campo oculto con un valor fijo.

```js lines
var options = {
  additionalSignUpFields: [{
    type: "hidden",
    name: "signup_code",
    value: "abc123"
  }]
}
```

Algunos casos de uso podrían aprovechar los datos de `additionalSignUpFields` en las plantillas de correo electrónico, por ejemplo, una opción de preferencia de idioma, cuyo valor podría usarse luego para establecer el idioma de las comunicaciones por correo electrónico generadas a partir de plantillas.

### allowLogin {Boolean}

Si se establece en `false`, el widget no mostrará la pantalla de inicio de sesión. Esto es útil si quieres usar el widget solo para registros (se ocultarán las pestañas de inicio de sesión y registro en la pantalla de registro) o para restablecer contraseñas (se ocultará el botón Atrás en la pantalla de recuperación de contraseña). En esos casos, también puede que necesites especificar las opciones `initialScreen`, `allowForgotPassword` y `allowSignUp`. El valor predeterminado es `true`.

`allowLogin: false`

<Frame>![Opción de Lock: Permitir inicio de sesión](/docs/images/cdy7uua7fh8z/6gI4aSLHXLzhcHiR9aWMET/d05c2a08a3895278b01150ecf894cb5f/lock-allowlogin.png)</Frame>

### allowForgotPassword {Boolean}

Cuando se establece en false, `allowForgotPassword` oculta el enlace &quot;¿No recuerdas tu contraseña?&quot; en la pantalla de login, lo que hace que no se pueda acceder a la pantalla de recuperación de contraseña. El valor predeterminado es true. Si usas una conexión de base de datos con una base de datos personalizada que no tiene un script para cambiar la contraseña, la pantalla de recuperación de contraseña no estará disponible.

`allowForgotPassword: false`

<Frame>![Lock Option: Permitir Forgot Password](/docs/images/cdy7uua7fh8z/1L6wJS3UnWK1VGKYMFhAB9/2ddcf2d462255e78a8b78958c2503809/lock-allowforgotpassword.png)</Frame>

### allowSignUp {Boolean}

Cuando se establece en `false`, oculta las pestañas de inicio de sesión y registro en la pantalla de inicio de sesión, por lo que no se puede acceder a la pantalla de registro. El valor predeterminado es `true`. Ten en cuenta que, si la conexión de base de datos tiene los registros deshabilitados o estás usando una base de datos personalizada que no tiene un script de creación, la pantalla de registro no estará disponible.

También ten en cuenta que esta opción **solo** controla la apariencia del lado del cliente y no impide por completo que visitantes anónimos decididos completen nuevos registros. Si quieres evitar por completo que nuevos usuarios se registren, debes usar la opción **Disable Sign Ups** en el Dashboard, en la configuración de la conexión.

`allowSignUp: false`

<Frame>![Lock Option: Permitir registro](/docs/images/cdy7uua7fh8z/2BNlYe99EFlRmIlhoLZiEB/f280d865cb16fa5fecda952271cb18bf/lock-allowsignup.png)</Frame>

### defaultDatabaseConnection {String}

Especifica la conexión de base de datos que se utilizará cuando haya más de una disponible.

`defaultDatabaseConnection: 'test-database'`

### initialScreen {String}

El nombre de la pantalla que se mostrará al abrir el widget. Los valores válidos son `login`, `signUp` y `forgotPassword`. Si no se especifica esta opción, el widget usará de forma predeterminada la primera pantalla disponible de esa lista.

`initialScreen: 'forgotPassword'`

### loginAfterSignUp {Boolean}

Determina si el usuario iniciará sesión automáticamente después de registrarse correctamente. El valor predeterminado es `true`.

`loginAfterSignUp: false`

### forgotPasswordLink {String}

Establece la URL de una página que permite al usuario restablecer su contraseña. Cuando se establece en una cadena no vacía, se redirigirá al usuario a la URL proporcionada al hacer clic en el enlace &quot;¿No recuerdas tu contraseña?&quot; en la pantalla de inicio de sesión.

`forgotPasswordLink: 'https://yoursite.com/reset-password'`

### showTerms {Boolean}

Si se establece en `true`, se muestra la cadena `languageDictionary.signUpTerms`. El valor predeterminado es `true`.

`showTerms: false`

### mustAcceptTerms {Boolean}

Si se establece en `true`, muestra una casilla de verificación junto a los términos y condiciones, que deben aceptarse antes de registrarse. Los términos y condiciones se pueden especificar mediante la opción `languageDictionary`. Esta opción solo surtirá efecto para los usuarios que se registren con conexiones de base de datos. El valor predeterminado es `false`.

`mustAcceptTerms: true`

### prefill {Object}

Permite establecer el valor inicial de los campos de correo electrónico y/o username. Si se omite, no se proporcionará ningún valor inicial.

```js lines
var options = {
  prefill: {
    email: "someone@auth0.com",
    username: "someone",
    phoneNumber: "+1234567890"
  }
};
```

### signUpLink {String}

Establece la URL a la que se accederá al hacer clic en el botón de registro. Cuando se establece como una cadena no vacía, esta opción fuerza `allowSignUp` a `true`.

`signUpLink: 'https://yoursite.com/signup'`

### usernameStyle {String}

Determina qué se usará para identificar al usuario en una conexión de base de datos que tenga activado el indicador `requires_username` (si no lo está, la opción `usernameStyle` se ignorará). Los valores posibles son `"username"` y `"email"`. De forma predeterminada, se permiten tanto `username` como `email`; al establecer esta opción, los inicios de sesión se limitarán a uno u otro.

`usernameStyle: 'username'`

### defaultEnterpriseConnection {String}

Especifica la conexión empresarial que permite iniciar sesión con un nombre de usuario y una contraseña, y que se utilizará cuando haya más de una conexión disponible o cuando exista una conexión de base de datos. Si se proporciona una `defaultDatabaseConnection`, se utilizará la conexión de base de datos y esta opción se ignorará.

`defaultEnterpriseConnection: 'test-database'`

### defaultADUsernameFromEmailPrefix {Boolean}

Obtiene el username provisional de AD a partir del prefijo del correo electrónico. El valor predeterminado es `true`.

`defaultADUsernameFromEmailPrefix: false`

<div id="passwordless-options">
  ## Opciones de Passwordless
</div>

### passwordlessMethod {String}

Al usar `Auth0LockPasswordless` con una conexión por correo electrónico, puedes usar esta opción para elegir entre enviar un [código](/docs/es/authenticate/passwordless) o un [Magic Link](/docs/es/authenticate/passwordless) para autenticar al usuario. Los valores disponibles para las conexiones por correo electrónico son `code` y `link`. El valor predeterminado es `code`. Las conexiones <Tooltip tip="Passwordless: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/docs/es/glossary?term=passwordless">passwordless</Tooltip> por SMS siempre usarán `code`.

`passwordlessMethod: code`

<div id="other-options">
  ## Otras opciones
</div>

### configurationBaseUrl {String}

Sobrescribe la URL base de la configuración de la aplicación. De forma predeterminada, usa el dominio proporcionado. Esta opción solo es necesaria si tu caso de uso específico requiere que tu aplicación no utilice el comportamiento predeterminado.

`configurationBaseUrl: "https://www.example.com"`

### languageBaseUrl {String}

Reemplaza la URL base del idioma para las traducciones proporcionadas por Auth0. De forma predeterminada, esta opción usa la URL de la CDN de Auth0 `https://cdn.auth0.com`, ya que allí se almacenan todas las traducciones incluidas. Si proporcionas otro valor, puedes usar otra fuente para las traducciones del idioma si lo necesitas.

`languageBaseUrl: "https://www.example.com"`

### hashCleanup {Boolean}

Cuando la opción `hashCleanup` está habilitada, elimina la parte del hash de la URL de devolución de llamada después de la autenticación del usuario. De forma predeterminada, es `true`.

`hashCleanup: false`

### connectionResolver {Function}

Cuando se utiliza, ofrece un punto de extensibilidad que permite elegir qué conexión usar en función de la información del username.

Recibe `username`, `context` y `callback` como parámetros. La devolución de llamada espera un objeto como: `{type: 'database', name: 'connection name'}`. **Esto solo funciona con conexiones de base de datos.** Ten en cuenta que esta función se ejecutará en el evento `onSubmit` del formulario, así que mantenla simple y rápida.

Esta funcionalidad está en beta. Si encuentras un error, abre un [issue](https://github.com/auth0/lock/issues/new) en GitHub.

### leeway {Integer}

La opción `leeway` se puede configurar como un número entero —un valor en segundos— que puede usarse para tener en cuenta la desviación del reloj en el vencimiento de los <Tooltip tip="token de ID: credencial destinada al propio cliente, y no a acceder a un recurso." cta="Ver glosario" href="/docs/es/glossary?term=ID+Token">token de ID</Tooltip>. Por lo general, el valor no suele ser mayor de uno o dos minutos como máximo.

`leeway: 30`
