title: Personalización de la UI de Lock

Puedes personalizar la apariencia de tu widget Lock de varias maneras. La forma mejor y más segura de hacerlo es con las opciones de JavaScript proporcionadas.

<div id="javascript-options">
  ## Opciones de JavaScript
</div>

Puedes configurar diversas personalizaciones de tu Lock mediante el parámetro `options` al instanciarlo. Algunas de ellas te permiten personalizar la UI. Las opciones de personalización de la UI aún están en desarrollo; esperamos añadir más con el tiempo.

Primero, definirás el objeto `options`, con las opciones que quieras personalizar. Luego tendrás que incluir ese objeto `options` como tercer parámetro al instanciar Lock; más información sobre esto a continuación.

<div id="theming-options">
  ### Opciones de tema
</div>

Actualmente hay un par de opciones de tema disponibles, agrupadas bajo la propiedad `theme`.

#### logo {String}

<Frame>![Personalización de la UI de Lock - Logotipo](/docs/images/cdy7uua7fh8z/FdOAKKy71rTbFjIlZEL8D/2eef96008095d85388daaa461707901d/lock-theme-logo.png)</Frame>

El valor de `logo` es la URL de una imagen que se colocará en la parte superior de Lock; de forma predeterminada, se usa el logotipo de Auth0. La resolución mínima recomendada es de 200 píxeles (ancho) por 200 píxeles (alto).

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png'
  }
};
```

#### primaryColor {String}

<Frame>![Personalización de la UI de Lock - color primario](/docs/images/cdy7uua7fh8z/1qQjeTmXJyIziv400VPaOn/48dd3cb8a72a6e88b2634b4b8a8bed63/lock-theme-primarycolor.png)</Frame>

La propiedad `primaryColor` define el color principal de Lock; todos los colores usados en el widget se calcularán a partir de él. Esta opción resulta útil al proporcionar un `logo` personalizado, para asegurar que todos los colores armonicen bien con la paleta del `logo`. El valor predeterminado es `#ea5323`.

```js lines
var options = {
  theme: {
    logo: 'https://example.com/logo.png',
    primaryColor: '#31324F'
  }
};
```

<div id="authbuttons-object">
  #### authButtons `{Object}`
</div>

Permite personalizar botones en Lock. Debe indicarse por nombre cada conexión personalizada cuyo botón quieras personalizar, cada una con su propio conjunto de parámetros. Los parámetros que se pueden personalizar se enumeran a continuación:

* **displayName** &#123;String&#125;: El nombre que se mostrará en lugar del nombre de la conexión al crear el título del botón, por ejemplo, `LOGIN WITH MYCONNECTION` para el inicio de sesión).
* **primaryColor** &#123;String&#125;: El color de fondo del botón. El valor predeterminado es `#eb5424`.
* **foregroundColor** &#123;String&#125;: El color del texto del botón. El valor predeterminado es `#FFFFFF`.
* **icon** &#123;String&#125;: La URL del ícono de esta conexión. Por ejemplo: `http://site.com/logo.png`.

```js lines
var options = {
  theme: {
    authButtons: {
      "testConnection": {
        displayName: "Test Conn",
        primaryColor: "#b7b7b7",
        foregroundColor: "#000000",
        icon: "http://example.com/icon.png"
      },
      "testConnection2": {
        primaryColor: "#000000",
        foregroundColor: "#ffffff",
      }
    }
  }
};
```

<div id="customizing-text">
  ### Personalización del texto
</div>

La opción `languageDictionary` permite personalizar todos los textos que se muestran en Lock. El valor predeterminado es {}. A continuación, puedes ver un ejemplo.

```js lines
var options = {
  languageDictionary: {
    emailInputPlaceholder: "something@youremail.com",
    title: "Log me in"
  },
};
```

<Frame>![Personalización del texto en la interfaz de usuario de Lock mediante la opción languageDictionary](/docs/images/cdy7uua7fh8z/iNvBLxxmea2tiWBhzwKX5/db0d7f86d0f7f95f70e5edcd34ab736f/lock-languagedictionary.png)</Frame>

Para obtener una lista completa de los elementos que se pueden personalizar con `languageDictionary`, consulta la [especificación del diccionario de idiomas en inglés](https://github.com/auth0/lock/blob/master/src/i18n/en.js) en el repositorio.

<div id="instantiating-lock">
  ### Crear una instancia de Lock
</div>

Por último, crea una instancia de Lock con el objeto `options` que has definido con tus opciones personalizadas.

`var lock = new Auth0Lock('{yourClientId}', '{yourDomain}', options);`

<div id="overriding-css">
  ## Sobrescribir CSS
</div>

No se recomienda personalizar tu Lock sobrescribiendo su CSS. El problema es que, con las nuevas versiones de Lock, algunos estilos pueden cambiar, lo que puede generar problemas no deseados si estás sobrescribiendo el CSS. Además, es fácil pasar por alto el uso de estilos en otras partes y, aunque el cambio pueda verse bien en una vista, puede que no se vea bien en otra.

Si aun así quieres sobrescribir el CSS para personalizar aún más el estilo de tu Lock, te recomendamos usar una versión de parche específica de Lock en lugar de una versión mayor o menor, para limitar la cantidad de resultados inesperados que pueden producirse cuando modificas los estilos y luego se publica otro parche que podría causar un comportamiento inesperado en tu UI debido a esos cambios. Esto puede hacerse asegurándote de especificar esa versión de parche (`x.y.z`) al incluir Lock o al descargarlo.

Además, por supuesto, te recomendamos probar exhaustivamente tus cambios de CSS para asegurarte de que la experiencia sea la que pretendes ofrecer a tus clientes.
